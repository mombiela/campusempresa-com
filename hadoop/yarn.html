Introducción a YARN

YARN, que significa Yet Another Resource Negotiator, es un componente central de Hadoop que gestiona recursos y programa trabajos en un clúster de Hadoop. Fue introducido en Hadoop 2.0 para superar las limitaciones del marco original de MapReduce.

Conceptos Clave

  • Gestión de Recursos: YARN gestiona los recursos del clúster de Hadoop, asegurando que las aplicaciones tengan los recursos necesarios para ejecutarse eficientemente.
  • Programación de Trabajos: YARN programa trabajos y asigna recursos dinámicamente según las necesidades de las aplicaciones.
  • Escalabilidad: YARN mejora la escalabilidad de Hadoop permitiendo que múltiples aplicaciones se ejecuten simultáneamente.

Arquitectura de YARN

La arquitectura de YARN consiste en varios componentes clave que trabajan juntos para gestionar recursos y programar trabajos.

Componentes

  • ResourceManager (RM): El demonio maestro que gestiona recursos y programa aplicaciones.
  • NodeManager (NM): El demonio por nodo que monitorea el uso de recursos e informa al ResourceManager.
  • ApplicationMaster (AM): Una biblioteca específica del marco que negocia recursos con el ResourceManager y trabaja con el NodeManager para ejecutar y monitorear tareas.
  • Contenedor: Un paquete lógico de recursos (memoria, CPU) asignado por el ResourceManager a una aplicación.

Diagrama de Arquitectura de YARN

+------------------+       +------------------+
|  ResourceManager |<----->|  NodeManager     |
|                  |       |                  |
+------------------+       +------------------+
        ^                          ^
        |                          |
        v                          v
+------------------+       +------------------+
| ApplicationMaster|       |  Contenedor      |
|                  |       |                  |
+------------------+       +------------------+

Cómo Funciona YARN

YARN opera separando la gestión de recursos y la programación/monitoreo de trabajos en demonios separados.

Flujo de Trabajo

  1. Envío del Cliente: Un cliente envía una aplicación al ResourceManager.
  2. Asignación de Recursos: El ResourceManager asigna un contenedor para el ApplicationMaster.
  3. Inicialización del ApplicationMaster: El ApplicationMaster se inicializa y negocia recursos para la aplicación.
  4. Ejecución de Tareas: El ApplicationMaster solicita contenedores al ResourceManager, y el NodeManager lanza las tareas.
  5. Monitoreo y Finalización: El ApplicationMaster monitorea el progreso de las tareas e informa al ResourceManager al completarlas.

Configuración de YARN

Para configurar YARN, necesitas modificar varios archivos de configuración en la configuración de Hadoop.

Archivos de Configuración Clave

  • yarn-site.xml: Contiene configuraciones específicas de YARN.
  • mapred-site.xml: Contiene configuraciones específicas de MapReduce.

Ejemplo de Configuración

<!-- yarn-site.xml -->
<configuration>
  <property>
    <name>yarn.resourcemanager.hostname</name>
    <value>resourcemanager.example.com</value>
  </property>
  <property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>8192</value>
  </property>
  <property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>4</value>
  </property>
</configuration>
<!-- mapred-site.xml -->
<configuration>
  <property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
  </property>
  <property>
    <name>mapreduce.jobhistory.address</name>
    <value>jobhistory.example.com:10020</value>
  </property>
</configuration>

Características Avanzadas de YARN

YARN proporciona varias características avanzadas para mejorar la gestión de recursos y la programación de trabajos.

Características

  • Políticas de Programación de Recursos: YARN soporta diferentes políticas de programación como FIFO, Capacity Scheduler y Fair Scheduler.
  • Preempción de Recursos: Permite la preempción de recursos de aplicaciones de menor prioridad a aplicaciones de mayor prioridad.
  • Alta Disponibilidad: YARN soporta alta disponibilidad para el ResourceManager para asegurar una operación continua.

Conclusión

YARN es un poderoso marco de gestión de recursos y programación de trabajos que mejora la escalabilidad y eficiencia de los clústeres de Hadoop. Al entender su arquitectura, flujo de trabajo y configuración, puedes gestionar y optimizar efectivamente tus aplicaciones de Hadoop. Las características avanzadas como las políticas de programación de recursos y la alta disponibilidad extienden aún más las capacidades de YARN, convirtiéndolo en un componente crítico del procesamiento moderno de big data.
