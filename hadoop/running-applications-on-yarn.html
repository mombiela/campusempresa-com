Ejecutar Aplicaciones en YARN
Planificadores de YARN ►

En esta sección, exploraremos cómo ejecutar aplicaciones en YARN (Yet Another Resource Negotiator), que es un componente central de Hadoop responsable de gestionar recursos y programar trabajos. Este tema cubrirá los conceptos básicos de YARN, cómo enviar aplicaciones y configuraciones avanzadas para optimizar el rendimiento.

Introducción a YARN

YARN es una capa de gestión de recursos para Hadoop que permite a múltiples motores de procesamiento de datos manejar datos almacenados en una sola plataforma, desbloqueando el potencial para una amplia gama de análisis de datos.

  • Gestión de Recursos: YARN gestiona recursos a través del clúster.
  • Programación de Trabajos: YARN programa trabajos y asigna recursos según los requisitos.

Componentes Clave de YARN

Entender los componentes clave de YARN es crucial para ejecutar aplicaciones de manera efectiva.

  • ResourceManager (RM): Gestiona recursos y programa aplicaciones.
  • NodeManager (NM): Gestiona recursos en nodos individuales e informa al RM.
  • ApplicationMaster (AM): Gestiona el ciclo de vida de una aplicación.
  • Contenedor: Una unidad de asignación de recursos (CPU, memoria) en un nodo.

Ejecutar una Aplicación Simple en YARN

Para ejecutar una aplicación en YARN, debes seguir estos pasos:

  1. Preparar la Aplicación: Empaqueta tu aplicación en un archivo JAR.
  2. Enviar la Aplicación: Usa el comando yarn para enviar tu aplicación.
  3. Monitorear la Aplicación: Usa la interfaz web de YARN o herramientas de línea de comandos para monitorear el progreso de la aplicación.

Ejemplo: Ejecutar una Aplicación WordCount en YARN

Vamos a ejecutar una simple aplicación WordCount en YARN.

  1. Preparar la Aplicación:

    • Asegúrate de tener un archivo JAR para la aplicación WordCount.
    • Ejemplo de JAR: wordcount.jar

  2. Enviar la Aplicación:

    yarn jar wordcount.jar org.apache.hadoop.examples.WordCount /input /output
    • yarn jar: Comando para ejecutar un archivo JAR en YARN.
    • wordcount.jar: El archivo JAR que contiene la aplicación WordCount.
    • org.apache.hadoop.examples.WordCount: La clase principal de la aplicación.
    • /input: Directorio de entrada en HDFS.
    • /output: Directorio de salida en HDFS.

  3. Monitorear la Aplicación:

    • Abre la interfaz web del ResourceManager de YARN (usualmente en http://<ResourceManager-Host>:8088).
    • Verifica el estado de tu aplicación en la pestaña "Applications".

Configuraciones Avanzadas de YARN

Para optimizar el rendimiento de tus aplicaciones en YARN, puedes ajustar varias configuraciones.

Asignación de Recursos

  • Configuración de Memoria:

    • yarn.nodemanager.resource.memory-mb: Memoria total disponible para contenedores en un nodo.
    • yarn.scheduler.maximum-allocation-mb: Asignación máxima de memoria para un solo contenedor.

  • Configuración de CPU:

    • yarn.nodemanager.resource.cpu-vcores: Núcleos de CPU totales disponibles para contenedores en un nodo.
    • yarn.scheduler.maximum-allocation-vcores: Asignación máxima de núcleos de CPU para un solo contenedor.

Ejemplo de Configuración en `yarn-site.xml`

<configuration>
    <property>
        <name>yarn.nodemanager.resource.memory-mb</name>
        <value>8192</value>
    </property>
    <property>
        <name>yarn.scheduler.maximum-allocation-mb</name>
        <value>4096</value>
    </property>
    <property>
        <name>yarn.nodemanager.resource.cpu-vcores</name>
        <value>8</value>
    </property>
    <property>
        <name>yarn.scheduler.maximum-allocation-vcores</name>
        <value>4</value>
    </property>
</configuration>

Conclusión

Ejecutar aplicaciones en YARN implica entender su arquitectura, preparar y enviar aplicaciones, y monitorear su progreso. Al configurar los ajustes de asignación de recursos de YARN, puedes optimizar el rendimiento de tus aplicaciones. Con estos conceptos básicos, ahora estás equipado para ejecutar y gestionar aplicaciones en YARN de manera efectiva.
