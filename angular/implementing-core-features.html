Introducción

En esta sección, profundizaremos en las características fundamentales de Angular, un poderoso framework para construir aplicaciones web dinámicas. Al final de este tema, tendrás una comprensión sólida de cómo implementar características esenciales como componentes, servicios, enrutamiento y formularios.

Componentes

Los componentes son los bloques de construcción de las aplicaciones Angular. Encapsulan el HTML, CSS y la lógica para una parte de la interfaz de usuario.

Creación de un Componente

Para crear un nuevo componente, utiliza el comando Angular CLI:

ng generate component my-component

Este comando genera cuatro archivos:

  • my-component.component.ts: El archivo TypeScript que contiene la lógica del componente.
  • my-component.component.html: La plantilla HTML para el componente.
  • my-component.component.css: Los estilos CSS para el componente.
  • my-component.component.spec.ts: El archivo de prueba para el componente.

Ejemplo de Componente

Aquí tienes un ejemplo simple de un componente:

// my-component.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  title = '¡Hola, Angular!';
}
<!-- my-component.component.html -->
<h1>{{ title }}</h1>

En este ejemplo, la propiedad title está vinculada a la plantilla usando la sintaxis de interpolación de Angular ({{ }}).

Servicios

Los servicios en Angular se utilizan para encapsular la lógica de negocio y el acceso a datos. Pueden ser inyectados en componentes y otros servicios.

Creación de un Servicio

Para crear un nuevo servicio, utiliza el comando Angular CLI:

ng generate service my-service

Este comando genera dos archivos:

  • my-service.service.ts: El archivo TypeScript que contiene la lógica del servicio.
  • my-service.service.spec.ts: El archivo de prueba para el servicio.

Ejemplo de Servicio

Aquí tienes un ejemplo simple de un servicio:

// my-service.service.ts
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MyService {
  getValue(): string {
    return 'Valor del Servicio';
  }
}

Para usar este servicio en un componente:

// my-component.component.ts
import { Component } from '@angular/core';
import { MyService } from './my-service.service';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  value: string;

  constructor(private myService: MyService) {
    this.value = this.myService.getValue();
  }
}
<!-- my-component.component.html -->
<p>{{ value }}</p>

Enrutamiento

El enrutamiento en Angular te permite navegar entre diferentes vistas o componentes.

Configuración del Enrutamiento

Primero, define tus rutas en el archivo app-routing.module.ts:

// app-routing.module.ts
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { MyComponent } from './my-component/my-component.component';

const routes: Routes = [
  { path: 'my-component', component: MyComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Luego, agrega la directiva <router-outlet> en tu plantilla principal:

<!-- app.component.html -->
<router-outlet></router-outlet>

Para navegar a una ruta programáticamente:

// some-component.component.ts
import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-some-component',
  templateUrl: './some-component.component.html',
  styleUrls: ['./some-component.component.css']
})
export class SomeComponent {
  constructor(private router: Router) {}

  navigateToMyComponent() {
    this.router.navigate(['/my-component']);
  }
}
<!-- some-component.component.html -->
<button (click)="navigateToMyComponent()">Ir a Mi Componente</button>

Formularios

Angular proporciona dos enfoques para manejar formularios: Formularios basados en plantillas y Formularios reactivos.

Formularios Basados en Plantillas

Los formularios basados en plantillas son fáciles de usar y adecuados para formularios simples.

<!-- template-driven-form.component.html -->
<form #myForm="ngForm" (ngSubmit)="onSubmit(myForm)">
  <label for="name">Nombre:</label>
  <input type="text" id="name" name="name" ngModel>
  <button type="submit">Enviar</button>
</form>
// template-driven-form.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-template-driven-form',
  templateUrl: './template-driven-form.component.html',
  styleUrls: ['./template-driven-form.component.css']
})
export class TemplateDrivenFormComponent {
  onSubmit(form: any) {
    console.log('¡Formulario Enviado!', form);
  }
}

Formularios Reactivos

Los formularios reactivos son más poderosos y adecuados para formularios complejos.

// reactive-form.component.ts
import { Component, OnInit } from '@angular/core';
import { FormBuilder, FormGroup } from '@angular/forms';

@Component({
  selector: 'app-reactive-form',
  templateUrl: './reactive-form.component.html',
  styleUrls: ['./reactive-form.component.css']
})
export class ReactiveFormComponent implements OnInit {
  myForm: FormGroup;

  constructor(private fb: FormBuilder) {}

  ngOnInit() {
    this.myForm = this.fb.group({
      name: ['']
    });
  }

  onSubmit() {
    console.log('¡Formulario Enviado!', this.myForm.value);
  }
}
<!-- reactive-form.component.html -->
<form [formGroup]="myForm" (ngSubmit)="onSubmit()">
  <label for="name">Nombre:</label>
  <input type="text" id="name" formControlName="name">
  <button type="submit">Enviar</button>
</form>

Conclusión

En este tema, cubrimos las características fundamentales de Angular, incluyendo componentes, servicios, enrutamiento y formularios. Estas características son fundamentales para construir aplicaciones Angular robustas y mantenibles. Al dominar estos conceptos, estarás bien equipado para abordar temas más avanzados y construir aplicaciones web sofisticadas.
