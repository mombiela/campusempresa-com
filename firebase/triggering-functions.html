Introducción a las Funciones de Firebase

Las Funciones de Firebase te permiten ejecutar código backend en respuesta a eventos desencadenados por características de Firebase y solicitudes HTTPS. Esto te permite extender la funcionalidad de tu aplicación sin gestionar tus propios servidores.

Configuración de las Funciones de Firebase

Antes de profundizar en la activación de funciones, necesitas configurar las Funciones de Firebase en tu proyecto.

  1. Instalar Firebase CLI:

    npm install -g firebase-tools

  2. Inicializar Firebase en tu proyecto:

    firebase init functions

  3. Instalar dependencias:

    cd functions
npm install

Tipos de Disparadores

Las Funciones de Firebase pueden ser activadas por varios eventos. Aquí están los principales tipos:

  • Disparadores HTTP: Activados por solicitudes HTTP.
  • Disparadores de Autenticación: Activados por eventos de Firebase Authentication.
  • Disparadores de Firestore: Activados por eventos de la base de datos Firestore.
  • Disparadores de la Base de Datos en Tiempo Real: Activados por eventos de la Base de Datos en Tiempo Real.
  • Disparadores de Almacenamiento: Activados por eventos de Cloud Storage.
  • Disparadores de Analytics: Activados por eventos de Firebase Analytics.
  • Disparadores de Pub/Sub: Activados por eventos de Google Cloud Pub/Sub.

Disparadores HTTP

Los disparadores HTTP te permiten crear funciones que pueden ser llamadas a través de solicitudes HTTP.

Ejemplo

const functions = require('firebase-functions');

exports.helloWorld = functions.https.onRequest((request, response) => {
    response.send("¡Hola desde Firebase!");
});

Explicación

  • functions.https.onRequest: Esto especifica que la función será activada por una solicitud HTTP.
  • request: Contiene información sobre la solicitud HTTP.
  • response: Se usa para enviar una respuesta de vuelta al cliente.

Disparadores de Firestore

Los disparadores de Firestore responden a cambios en los documentos de Firestore.

Ejemplo

exports.onUserCreate = functions.firestore
    .document('users/{userId}')
    .onCreate((snap, context) => {
        const newValue = snap.data();
        console.log('Nuevo usuario creado:', newValue);
    });

Explicación

  • functions.firestore.document('users/{userId}'): Especifica la ruta del documento de Firestore a observar.
  • onCreate: Activado cuando se crea un nuevo documento.
  • snap: Contiene la instantánea del documento creado.
  • context: Contiene metadatos sobre el evento.

Disparadores de la Base de Datos en Tiempo Real

Los disparadores de la Base de Datos en Tiempo Real responden a cambios en la Base de Datos en Tiempo Real.

Ejemplo

exports.onDataChange = functions.database
    .ref('/messages/{messageId}')
    .onWrite((change, context) => {
        const beforeData = change.before.val();
        const afterData = change.after.val();
        console.log('Datos cambiados de', beforeData, 'a', afterData);
    });

Explicación

  • functions.database.ref('/messages/{messageId}'): Especifica la ruta de la Base de Datos en Tiempo Real a observar.
  • onWrite: Activado cuando los datos son creados, actualizados o eliminados.
  • change: Contiene los datos antes y después del cambio.
  • context: Contiene metadatos sobre el evento.

Disparadores de Autenticación

Los disparadores de autenticación responden a eventos de Firebase Authentication.

Ejemplo

exports.onUserSignUp = functions.auth.user().onCreate((user) => {
    console.log('Nuevo usuario registrado:', user.email);
});

Explicación

  • functions.auth.user().onCreate: Activado cuando un nuevo usuario se registra.
  • user: Contiene información sobre el usuario recién creado.

Disparadores de Almacenamiento

Los disparadores de almacenamiento responden a cambios en Cloud Storage.

Ejemplo

exports.onFileUpload = functions.storage.object().onFinalize((object) => {
    console.log('Archivo subido:', object.name);
});

Explicación

  • functions.storage.object().onFinalize: Activado cuando se sube un archivo.
  • object: Contiene información sobre el archivo subido.

Conclusión

Las Funciones de Firebase proporcionan una forma poderosa de extender las capacidades del backend de tu aplicación sin gestionar tus propios servidores. Al entender y utilizar diferentes tipos de disparadores, puedes responder a varios eventos en tu proyecto de Firebase, desde solicitudes HTTP hasta cambios en la base de datos y eventos de autenticación de usuarios. Practica creando y desplegando diferentes tipos de funciones para volverte competente en el uso de las Funciones de Firebase.
