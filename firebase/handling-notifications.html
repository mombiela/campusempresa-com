Introducción

Firebase Cloud Messaging (FCM) es una solución de mensajería multiplataforma que te permite enviar mensajes de manera confiable y sin costo. Esta sección cubrirá cómo manejar notificaciones en tu aplicación usando Firebase, desde la configuración básica hasta la personalización avanzada.

Configuración de Firebase Cloud Messaging

Prerrequisitos

  • Un proyecto de Firebase configurado en la Consola de Firebase.
  • SDK de Firebase integrado en tu aplicación (iOS, Android o Web).

Añadiendo FCM a tu Aplicación

  1. iOS:

    • Añade el SDK de Firebase a tu proyecto usando CocoaPods.
    • Configura tu aplicación para notificaciones push.
    • Regístrate para notificaciones remotas en tu delegado de la aplicación.
    import Firebase
import UserNotifications

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate, UNUserNotificationCenterDelegate {
    func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        FirebaseApp.configure()
        UNUserNotificationCenter.current().delegate = self
        application.registerForRemoteNotifications()
        return true
    }
}

  2. Android:

    • Añade el SDK de Firebase a tu proyecto vía Gradle.
    • Configura tu aplicación para recibir notificaciones modificando el AndroidManifest.xml.
    <service
    android:name=".MyFirebaseMessagingService">
    <intent-filter>
        <action android:name="com.google.firebase.MESSAGING_EVENT"/>
    </intent-filter>
</service>

  3. Web:

    • Añade el SDK de Firebase a tu proyecto.
    • Solicita permiso para enviar notificaciones.
    import { getMessaging, getToken } from "firebase/messaging";

const messaging = getMessaging();
getToken(messaging, { vapidKey: "YOUR_PUBLIC_VAPID_KEY" }).then((currentToken) => {
    if (currentToken) {
        console.log('Token received: ', currentToken);
    } else {
        console.log('No registration token available.');
    }
}).catch((err) => {
    console.log('An error occurred while retrieving token. ', err);
});

Enviando Notificaciones

Usando la Consola de Firebase

  • Navega a la Consola de Firebase.
  • Selecciona tu proyecto y ve a la sección de Cloud Messaging.
  • Haz clic en "Enviar tu primer mensaje".
  • Rellena los detalles de la notificación y dirige tu audiencia.

Usando el SDK de Firebase Admin

  • Instala el SDK de Firebase Admin en tu entorno de servidor.
  • Envía una notificación usando el SDK.
const admin = require('firebase-admin');
admin.initializeApp();

const message = {
    notification: {
        title: 'Hello World',
        body: 'This is a test notification'
    },
    token: 'RECIPIENT_DEVICE_TOKEN'
};

admin.messaging().send(message)
    .then((response) => {
        console.log('Successfully sent message:', response);
    })
    .catch((error) => {
        console.log('Error sending message:', error);
    });

Manejo de Notificaciones en la Aplicación

Notificaciones en Primer Plano

  • iOS:

    • Implementa los métodos de UNUserNotificationCenterDelegate para manejar notificaciones mientras la aplicación está en primer plano.
    func userNotificationCenter(_ center: UNUserNotificationCenter, willPresent notification: UNNotification, withCompletionHandler completionHandler: @escaping (UNNotificationPresentationOptions) -> Void) {
    completionHandler([.alert, .sound])
}

  • Android:

    • Sobrescribe el método onMessageReceived en tu FirebaseMessagingService.
    public class MyFirebaseMessagingService extends FirebaseMessagingService {
    @Override
    public void onMessageReceived(RemoteMessage remoteMessage) {
        // Handle the message here
    }
}

  • Web:

    • Maneja mensajes usando el método onMessage de la instancia de Firebase Messaging.
    import { getMessaging, onMessage } from "firebase/messaging";

const messaging = getMessaging();
onMessage(messaging, (payload) => {
    console.log('Message received. ', payload);
});

Notificaciones en Segundo Plano

  • iOS & Android:

    • Las notificaciones son manejadas por el sistema y mostradas en la bandeja de notificaciones.

  • Web:

    • Usa un service worker para manejar notificaciones en segundo plano.
    self.addEventListener('push', function(event) {
    const data = event.data.json();
    const options = {
        body: data.body,
        icon: 'icon.png'
    };
    event.waitUntil(
        self.registration.showNotification(data.title, options)
    );
});

Personalización Avanzada

Cargas Útiles de Datos Personalizados

  • Envía datos personalizados junto con notificaciones para manejar acciones específicas o mostrar elementos de UI personalizados.
const message = {
    data: {
        key1: 'value1',
        key2: 'value2'
    },
    token: 'RECIPIENT_DEVICE_TOKEN'
};

admin.messaging().send(message)
    .then((response) => {
        console.log('Successfully sent message:', response);
    })
    .catch((error) => {
        console.log('Error sending message:', error);
    });

Canales de Notificación (Android)

  • Crea canales de notificación para categorizar notificaciones.
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
    NotificationChannel channel = new NotificationChannel("CHANNEL_ID", "Channel Name", NotificationManager.IMPORTANCE_DEFAULT);
    NotificationManager manager = getSystemService(NotificationManager.class);
    manager.createNotificationChannel(channel);
}

Conclusión

Manejar notificaciones con Firebase Cloud Messaging implica configurar los SDKs necesarios, enviar notificaciones a través de la Consola de Firebase o el SDK Admin, y manejar notificaciones en tu aplicación tanto en estados de primer plano como de segundo plano. La personalización avanzada permite experiencias de usuario a medida y una mejor gestión de notificaciones. Siguiendo estos pasos, puedes integrar y gestionar notificaciones de manera efectiva en tu aplicación.
