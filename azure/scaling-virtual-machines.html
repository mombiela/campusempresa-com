Introducción

Escalar máquinas virtuales (VMs) en Azure es una habilidad crucial para gestionar cargas de trabajo de manera eficiente. Este tema cubrirá los conceptos básicos del escalado de VMs, diferentes estrategias de escalado y técnicas avanzadas para asegurar un rendimiento óptimo y gestión de costos.

Conceptos Clave

  • Escalar Verticalmente vs. Escalar Horizontalmente: Entender la diferencia entre aumentar el tamaño de una VM (escalar verticalmente) y aumentar el número de VMs (escalar horizontalmente).
  • Conjuntos de Escala de VMs de Azure: Un servicio que te permite crear y gestionar un grupo de VMs balanceadas por carga.
  • Autoescalado: Ajustar automáticamente el número de instancias de VM basado en la demanda.
  • Escalado Manual: Ajustar manualmente el número de instancias de VM.

Escalar Verticalmente vs. Escalar Horizontalmente

Escalar Verticalmente

  • Definición: Aumentar el tamaño (CPU, memoria) de una VM existente.
  • Caso de Uso: Adecuado para aplicaciones que requieren más recursos pero no pueden distribuirse en múltiples VMs.
  • Ejemplo: 
    Supongamos que tienes una VM con 2 CPUs y 4GB de RAM. Escalar verticalmente podría implicar actualizar a una VM con 4 CPUs y 8GB de RAM.

Escalar Horizontalmente

  • Definición: Aumentar el número de instancias de VM.
  • Caso de Uso: Ideal para aplicaciones que pueden distribuirse en múltiples VMs, como servidores web.
  • Ejemplo: 
    Si tienes una VM manejando tráfico web, escalar horizontalmente podría implicar agregar más VMs para distribuir la carga.

Conjuntos de Escala de VMs de Azure

  • Definición: Los Conjuntos de Escala de VMs de Azure te permiten crear y gestionar un grupo de VMs idénticas y balanceadas por carga.
  • Beneficios:
    • Fácil de gestionar y escalar.
    • Integrado con Azure Load Balancer.
    • Soporta autoescalado.
  • Ejemplo: 
    Crear un Conjunto de Escala de VM usando Azure CLI:
```bash
az vmss create --resource-group myResourceGroup --name myScaleSet --image UbuntuLTS --upgrade-policy-mode automatic --admin-username azureuser --generate-ssh-keys

Autoescalado

  • Definición: Ajusta automáticamente el número de instancias de VM basado en reglas y métricas predefinidas.
  • Beneficios:
    • Rentable: Solo pagas por lo que usas.
    • Rendimiento: Asegura que tu aplicación pueda manejar cargas variables.
  • Ejemplo: 
    Configurar autoescalado para un Conjunto de Escala de VM:
```bash
az monitor autoscale create --resource-group myResourceGroup --resource myScaleSet --resource-type Microsoft.Compute/virtualMachineScaleSets --name myAutoscale --min-count 1 --max-count 10 --count 1
az monitor autoscale rule create --resource-group myResourceGroup --autoscale-name myAutoscale --condition "Percentage CPU > 75 avg 5m" --scale out 1
az monitor autoscale rule create --resource-group myResourceGroup --autoscale-name myAutoscale --condition "Percentage CPU < 25 avg 5m" --scale in 1

Escalado Manual

  • Definición: Ajustar manualmente el número de instancias de VM.
  • Beneficios:
    • Control total sobre las decisiones de escalado.
    • Útil para cargas de trabajo predecibles.
  • Ejemplo: 
    Escalar manualmente un Conjunto de Escala de VM usando Azure CLI:
```bash
az vmss scale --resource-group myResourceGroup --name myScaleSet --new-capacity 5

Resumen

Escalar máquinas virtuales en Azure implica entender la diferencia entre escalar verticalmente y escalar horizontalmente, utilizar Conjuntos de Escala de VMs de Azure e implementar autoescalado para mayor eficiencia. El escalado manual proporciona control para cargas de trabajo predecibles. Dominar estas técnicas asegura que tus aplicaciones puedan manejar cargas variables de manera eficiente mientras optimizas los costos.
