Desplegando una Aplicación Multi-Capa en Azure

Introducción

Desplegar una aplicación multi-capa implica configurar diferentes capas (tiers) de una aplicación, como la capa de presentación, la capa de lógica de negocio y la capa de datos, en Azure. Esta guía te llevará desde los conceptos básicos hasta estrategias avanzadas de despliegue.

Conceptos Clave

  • Arquitectura Multi-Capa: Una arquitectura de aplicación que separa diferentes funcionalidades en capas.
    • Capa de Presentación: Interfaz de usuario.
    • Capa de Lógica de Negocio: Procesamiento de la aplicación.
    • Capa de Datos: Gestión de la base de datos.
  • Servicios de Azure: Varios servicios proporcionados por Azure para soportar aplicaciones multi-capa.
    • Azure App Service: Para alojar aplicaciones web.
    • Azure SQL Database: Servicio de base de datos relacional gestionado.
    • Azure Virtual Machines: Para alojar aplicaciones y servicios personalizados.
    • Azure Load Balancer: Distribuye el tráfico de red entrante entre múltiples servidores.

Configuración del Entorno

Creando Grupos de Recursos

Los grupos de recursos en Azure son contenedores lógicos para los recursos desplegados en Azure.

az group create --name MyResourceGroup --location eastus

Desplegando la Capa de Presentación

Usa Azure App Service para alojar la aplicación web.

  1. Crear un Plan de App Service:

    az appservice plan create --name MyAppServicePlan --resource-group MyResourceGroup --sku B1 --is-linux

  2. Crear una Aplicación Web:

    az webapp create --resource-group MyResourceGroup --plan MyAppServicePlan --name MyWebApp --runtime "NODE|14-lts"

Desplegando la Capa de Lógica de Negocio

Usa Azure Virtual Machines o Azure Functions para la lógica de negocio.

  1. Crear una Máquina Virtual:

    az vm create --resource-group MyResourceGroup --name MyVM --image UbuntuLTS --admin-username azureuser --generate-ssh-keys

  2. Instalar el Software Necesario:

    az vm run-command invoke --command-id RunShellScript --name MyVM --resource-group MyResourceGroup --scripts "sudo apt-get update && sudo apt-get install -y nodejs"

Desplegando la Capa de Datos

Usa Azure SQL Database para la capa de datos.

  1. Crear un Servidor SQL:

    az sql server create --name myserver --resource-group MyResourceGroup --location eastus --admin-user myadmin --admin-password MyPassword123

  2. Crear una Base de Datos SQL:

    az sql db create --resource-group MyResourceGroup --server myserver --name mydatabase --service-objective S0

Conectando las Capas

Configurando la Aplicación Web para Conectarse a la Capa de Lógica de Negocio

Actualiza la configuración de la aplicación web para apuntar a la capa de lógica de negocio.

az webapp config appsettings set --resource-group MyResourceGroup --name MyWebApp --settings BUSINESS_LOGIC_URL=http://<VM_IP_ADDRESS>

Configurando la Capa de Lógica de Negocio para Conectarse a la Capa de Datos

Actualiza la capa de lógica de negocio para conectarse a la base de datos SQL.

const sql = require('mssql');

const config = {
    user: 'myadmin',
    password: 'MyPassword123',
    server: 'myserver.database.windows.net',
    database: 'mydatabase',
    options: {
        encrypt: true
    }
};

sql.connect(config).then(pool => {
    return pool.request().query('SELECT * FROM mytable');
}).then(result => {
    console.dir(result);
}).catch(err => {
    console.error(err);
});

Estrategias Avanzadas de Despliegue

Usando Azure DevOps para CI/CD

Configura pipelines de integración continua y despliegue continuo usando Azure DevOps.

  1. Crear un Nuevo Pipeline:

    • Navega a Azure DevOps.
    • Crea un nuevo pipeline y conéctalo a tu repositorio.
    • Define tus etapas de build y release.

  2. Ejemplo de Pipeline YAML:

    trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseNode@2
  inputs:
    version: '14.x'
- script: |
    npm install
    npm run build
  displayName: 'Build project'
- task: AzureWebApp@1
  inputs:
    azureSubscription: '<Azure Service Connection>'
    appName: 'MyWebApp'
    package: '$(Build.ArtifactStagingDirectory)/**/*.zip'

Escalando la Aplicación

Usa Azure Load Balancer para distribuir el tráfico y escalar la aplicación.

  1. Crear un Load Balancer:

    az network lb create --resource-group MyResourceGroup --name MyLoadBalancer --sku Basic --frontend-ip-name MyFrontEnd --backend-pool-name MyBackEndPool

  2. Agregar VMs al Load Balancer:

    az network lb address-pool address add --resource-group MyResourceGroup --lb-name MyLoadBalancer --pool-name MyBackEndPool --vnet-name MyVNet --nic-name MyVMNic

Conclusión

Desplegar una aplicación multi-capa en Azure implica configurar diferentes capas de la aplicación usando varios servicios de Azure. Siguiendo los pasos descritos en esta guía, puedes desplegar una aplicación multi-capa robusta y escalable. Estrategias avanzadas como el uso de Azure DevOps para CI/CD y Azure Load Balancer para escalar pueden mejorar aún más tu proceso de despliegue.
