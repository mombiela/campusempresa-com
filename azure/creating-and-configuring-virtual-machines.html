Introducción

En esta sección, exploraremos cómo crear y configurar Máquinas Virtuales (VMs) en Microsoft Azure. Este tema es esencial para cualquiera que busque aprovechar la computación en la nube para una infraestructura escalable y flexible.

Requisitos Previos

Antes de comenzar, asegúrate de tener:

  • Una suscripción activa a Azure.
  • Comprensión básica de los conceptos de computación en la nube.
  • Familiaridad con el portal de Azure.

Creación de una Máquina Virtual

Guía Paso a Paso

  1. Navegar al Portal de Azure:

    • Ve al Portal de Azure.
    • Inicia sesión con tu cuenta de Azure.

  2. Crear una Nueva Máquina Virtual:

    • Haz clic en "Crear un recurso" en el menú de la izquierda.
    • Selecciona "Compute" y luego "Máquina Virtual."

  3. Configurar Ajustes Básicos:

    • Suscripción: Elige tu suscripción.
    • Grupo de Recursos: Crea un nuevo grupo de recursos o selecciona uno existente.
    • Nombre de la Máquina Virtual: Ingresa un nombre único para tu VM.
    • Región: Selecciona la región donde deseas desplegar tu VM.
    • Opciones de Disponibilidad: Elige conjunto de disponibilidad o zona de disponibilidad si es necesario.
    • Imagen: Selecciona la imagen del sistema operativo (por ejemplo, Windows Server, Ubuntu).
    • Tamaño: Elige el tamaño de la VM según tus requisitos.

  4. Cuenta de Administrador:

    • Nombre de Usuario: Ingresa un nombre de usuario para la cuenta de administrador.
    • Contraseña: Ingresa una contraseña fuerte o usa una clave pública SSH para VMs Linux.

  5. Reglas de Puertos Entrantes:

    • Selecciona los puertos que deseas abrir (por ejemplo, RDP para Windows, SSH para Linux).

  6. Revisar + Crear:

    • Revisa tus configuraciones y haz clic en "Crear."

Ejemplo

# Ejemplo de Azure CLI para crear una VM
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Configuración de una Máquina Virtual

Accediendo a la VM

  • VM de Windows: Usa el Protocolo de Escritorio Remoto (RDP) para conectarte.
  • VM de Linux: Usa Secure Shell (SSH) para conectarte.

Instalación de Software

  • VM de Windows:

    • Usa la conexión RDP para instalar software manualmente.
    • Usa scripts de PowerShell para automatización.

  • VM de Linux:

    • Usa SSH para conectarte e instalar software usando gestores de paquetes como apt o yum.

Ejemplo

# Ejemplo de instalación de Apache en una VM Ubuntu
sudo apt update
sudo apt install apache2 -y

Configuración Avanzada

Redes

  • Red Virtual (VNet): Configura VNets para gestionar el tráfico de red.
  • Grupos de Seguridad de Red (NSG): Define reglas de seguridad para controlar el tráfico entrante y saliente.

Almacenamiento

  • Discos Administrados: Usa discos administrados para mejor rendimiento y fiabilidad.
  • Discos de Datos: Adjunta discos de datos adicionales para necesidades de almacenamiento.

Escalado

  • Conjuntos de Escala: Usa conjuntos de escala de VM para escalar automáticamente el número de VMs según la demanda.

Ejemplo

# Ejemplo de Azure CLI para agregar un disco de datos a una VM
az vm disk attach \
  --resource-group myResourceGroup \
  --vm-name myVM \
  --name myDataDisk \
  --size-gb 128

Monitoreo y Mantenimiento

Monitoreo

  • Azure Monitor: Usa Azure Monitor para rastrear el rendimiento y la salud de tus VMs.
  • Log Analytics: Recoge y analiza logs para resolución de problemas.

Mantenimiento

  • Actualizaciones: Actualiza regularmente tus VMs para asegurar seguridad y rendimiento.
  • Copias de Seguridad: Usa Azure Backup para crear copias de seguridad regulares de tus VMs.

Conclusión

Crear y configurar VMs en Azure es una habilidad fundamental para aprovechar la infraestructura en la nube. Siguiendo los pasos descritos en esta guía, puedes configurar VMs adaptadas a tus necesidades específicas, configurarlas para un rendimiento óptimo y asegurarte de que sean seguras y estén bien mantenidas. A medida que avances, explora configuraciones e integraciones más complejas para utilizar completamente las capacidades de Azure.
