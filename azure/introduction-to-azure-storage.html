Introducción al Almacenamiento de Azure
Almacenamiento de Blobs ►

El Almacenamiento de Azure es una solución de almacenamiento en la nube proporcionada por Microsoft Azure que ofrece opciones de almacenamiento altamente disponibles, seguras, duraderas y escalables. Esta sección te guiará a través de los conceptos básicos del Almacenamiento de Azure, sus componentes clave y cómo comenzar a usarlo.

Conceptos Clave del Almacenamiento de Azure

  • Blob Storage: Se utiliza para almacenar datos no estructurados como texto o datos binarios. Es ideal para servir imágenes o documentos directamente a un navegador, almacenar archivos para acceso distribuido y transmitir video y audio.
  • File Storage: Proporciona recursos compartidos de archivos totalmente gestionados en la nube que son accesibles a través del protocolo Server Message Block (SMB).
  • Queue Storage: Te permite almacenar grandes cantidades de mensajes que pueden ser accesibles desde cualquier lugar mediante llamadas autenticadas usando HTTP o HTTPS.
  • Table Storage: Un almacén de valores clave NoSQL para el desarrollo rápido utilizando conjuntos de datos masivos semi-estructurados.

Blob Storage

Blob Storage está diseñado para almacenar grandes cantidades de datos no estructurados. Soporta tres tipos de blobs:

  • Block Blobs: Optimizado para la transmisión y almacenamiento de objetos en la nube.
  • Append Blobs: Optimizado para operaciones de anexado, lo que los hace ideales para el registro de logs.
  • Page Blobs: Optimizado para operaciones de lectura/escritura aleatorias y utilizado para archivos de disco duro virtual (VHD).

Ejemplo: Subir un Blob

from azure.storage.blob import BlobServiceClient, BlobClient, ContainerClient

# Inicializar una conexión al servicio Blob
blob_service_client = BlobServiceClient.from_connection_string("your_connection_string")

# Crear un contenedor
container_name = "mycontainer"
container_client = blob_service_client.create_container(container_name)

# Subir un blob
blob_client = blob_service_client.get_blob_client(container=container_name, blob="myblob.txt")
with open("sample.txt", "rb") as data:
    blob_client.upload_blob(data)

File Storage

Azure File Storage te permite crear recursos compartidos de archivos en la nube que pueden ser accesibles a través del protocolo SMB. Esto es útil para aplicaciones que necesitan almacenamiento compartido accesible por múltiples máquinas virtuales.

Ejemplo: Montar un Recurso Compartido de Archivos de Azure

# Instalar cifs-utils si no está ya instalado
sudo apt-get install cifs-utils

# Montar el recurso compartido de archivos
sudo mount -t cifs //your_storage_account.file.core.windows.net/your_file_share /mnt/your_mount_point -o vers=3.0,username=your_storage_account,password=your_storage_account_key,dir_mode=0777,file_mode=0777,sec=ntlmssp

Queue Storage

Queue Storage se utiliza para almacenar y recuperar mensajes. Es útil para desacoplar componentes de una aplicación en la nube para mejorar la escalabilidad y la fiabilidad.

Ejemplo: Añadir un Mensaje a una Cola

from azure.storage.queue import QueueServiceClient

# Inicializar una conexión al servicio de Cola
queue_service_client = QueueServiceClient.from_connection_string("your_connection_string")

# Crear una cola
queue_name = "myqueue"
queue_client = queue_service_client.create_queue(queue_name)

# Añadir un mensaje a la cola
queue_client.send_message("Hello, Azure Queue!")

Table Storage

Table Storage es un almacén de datos NoSQL que permite el almacenamiento de grandes cantidades de datos estructurados. Es sin esquema, lo que lo hace flexible y escalable.

Ejemplo: Insertar una Entidad en una Tabla

from azure.data.tables import TableServiceClient, TableEntity

# Inicializar una conexión al servicio de Tabla
table_service_client = TableServiceClient.from_connection_string("your_connection_string")

# Crear una tabla
table_name = "mytable"
table_client = table_service_client.create_table_if_not_exists(table_name)

# Insertar una entidad
entity = TableEntity()
entity.PartitionKey = "partition1"
entity.RowKey = "row1"
entity["Name"] = "John Doe"
entity["Age"] = 30

table_client.create_entity(entity=entity)

Resumen

El Almacenamiento de Azure proporciona una variedad de soluciones de almacenamiento para satisfacer diferentes necesidades, desde el almacenamiento de datos no estructurados con Blob Storage hasta el almacenamiento de datos estructurados con Table Storage. Comprender estos servicios y cómo usarlos es crucial para construir aplicaciones en la nube escalables y fiables. Esta introducción cubrió los conceptos básicos de cada tipo de almacenamiento, junto con ejemplos para ayudarte a comenzar. A medida que avances, aprenderás características más avanzadas y mejores prácticas para usar el Almacenamiento de Azure de manera efectiva.
