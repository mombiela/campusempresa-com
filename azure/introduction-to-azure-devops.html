Introducción a Azure DevOps
Integración Continua y Despliegue Continuo (CI/CD) ►

Azure DevOps es un conjunto de herramientas de desarrollo proporcionadas por Microsoft para apoyar todo el ciclo de vida del desarrollo de software. Incluye servicios para planificar, desarrollar, entregar y operar aplicaciones. Este tema te introducirá a los conceptos y servicios básicos de Azure DevOps, proporcionando una base para una mayor exploración y dominio.

Conceptos Clave de Azure DevOps

  • Azure Boards: Herramientas de planificación ágil (tableros Kanban, backlogs, paneles de equipo) para rastrear el trabajo.
  • Azure Repos: Repositorios Git privados ilimitados alojados en la nube para tu proyecto.
  • Azure Pipelines: Construye, prueba y despliega con CI/CD que funciona con cualquier lenguaje, plataforma y nube.
  • Azure Test Plans: Solución integral de pruebas planificadas y exploratorias.
  • Azure Artifacts: Crea, aloja y comparte paquetes con tu equipo, y añade artefactos a tus pipelines de CI/CD.

Configuración de Azure DevOps

Creación de una Cuenta de Azure DevOps

  1. Ve al sitio web de Azure DevOps.
  2. Haz clic en "Start free" o "Sign in" si ya tienes una cuenta de Microsoft.
  3. Sigue las indicaciones para crear tu organización de Azure DevOps.

Creación de un Nuevo Proyecto

  1. Una vez que hayas iniciado sesión, haz clic en "New Project".
  2. Ingresa un nombre y una descripción para el proyecto.
  3. Elige la visibilidad (pública o privada).
  4. Haz clic en "Create".

Azure Boards

Azure Boards proporciona un conjunto rico de capacidades que incluyen soporte nativo para Scrum y Kanban, paneles personalizables e informes integrados.

Creación y Gestión de Elementos de Trabajo

  • Elementos de Trabajo: Unidades básicas de trabajo, como historias de usuario, tareas, errores.
  • Tableros: Visualiza y gestiona el trabajo con tableros Kanban.
  • Backlogs: Prioriza el trabajo y planifica sprints.

Ejemplo de creación de un elemento de trabajo:

1. Navega a Boards > Work Items.
2. Haz clic en "New Work Item" y selecciona el tipo (por ejemplo, Historia de Usuario).
3. Rellena los detalles y guarda.

Azure Repos

Azure Repos proporciona repositorios Git o Team Foundation Version Control (TFVC) para el control de versiones de tu código.

Creación de un Repositorio

  1. Navega a Repos > Files.
  2. Haz clic en "New Repository".
  3. Elige Git o TFVC.
  4. Inicializa con un README si lo deseas y haz clic en "Create".

Comandos Básicos de Git

# Clonar un repositorio
git clone https://dev.azure.com/tuorganizacion/tuprojecto/_git/turepo

# Añadir un archivo a la zona de preparación
git add nombrearchivo

# Confirmar cambios
git commit -m "Tu mensaje de confirmación"

# Empujar cambios a Azure Repos
git push origin main

Azure Pipelines

Azure Pipelines te permite construir, probar y desplegar código automáticamente.

Creación de un Pipeline

  1. Navega a Pipelines > Pipelines.
  2. Haz clic en "New Pipeline".
  3. Selecciona tu fuente de repositorio (por ejemplo, GitHub, Azure Repos).
  4. Configura tu pipeline usando YAML o el editor clásico.

Ejemplo de configuración de pipeline en YAML:

trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- script: echo Hello, world!
  displayName: 'Run a one-line script'

Azure Test Plans

Azure Test Plans proporciona herramientas para pruebas manuales y exploratorias.

Creación de un Plan de Pruebas

  1. Navega a Test Plans > Test Plans.
  2. Haz clic en "New Test Plan".
  3. Ingresa un nombre y una descripción.
  4. Añade casos de prueba y configura suites de prueba.

Azure Artifacts

Azure Artifacts te permite crear y compartir feeds de paquetes Maven, npm, NuGet y Python desde fuentes públicas y privadas.

Creación de un Feed

  1. Navega a Artifacts > Feeds.
  2. Haz clic en "New Feed".
  3. Ingresa un nombre y establece permisos.
  4. Haz clic en "Create".

Uso de Azure Artifacts en un Pipeline

Ejemplo de añadir un paquete NuGet a un pipeline:

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- task: NuGetCommand@2
  inputs:
    command: 'restore'
    feedsToUse: 'select'
    vstsFeed: 'your-feed-id'

Conclusión

Azure DevOps es un conjunto integral de herramientas diseñadas para apoyar todo el ciclo de vida del desarrollo de software. Al comprender y utilizar Azure Boards, Repos, Pipelines, Test Plans y Artifacts, puedes optimizar tu proceso de desarrollo, mejorar la colaboración y entregar software de alta calidad de manera más eficiente. Esta introducción proporciona una base para una mayor exploración y dominio de Azure DevOps.
