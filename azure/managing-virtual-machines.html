Introducción

Gestionar máquinas virtuales (VMs) en Azure es una habilidad fundamental para la computación en la nube. Este tema te guiará a través del proceso desde un nivel principiante hasta uno avanzado, cubriendo conceptos clave, ejemplos prácticos y ejercicios.

Empezando con Máquinas Virtuales en Azure

¿Qué es una Máquina Virtual?

  • Una máquina virtual (VM) es una emulación de un sistema informático.
  • Las VMs se ejecutan en hardware físico pero están aisladas de él.
  • Permiten ejecutar múltiples sistemas operativos en una sola máquina física.

Creando Tu Primera VM

  1. Inicia sesión en el Portal de Azure: Navega a Portal de Azure.
  2. Crear un Recurso: Haz clic en "Crear un recurso" y selecciona "Máquina Virtual".
  3. Configurar Configuraciones Básicas:
    • Suscripción: Elige tu suscripción de Azure.
    • Grupo de Recursos: Crea un nuevo grupo de recursos o selecciona uno existente.
    • Nombre de la VM: Dale un nombre único a tu VM.
    • Región: Selecciona la región donde deseas desplegar tu VM.
    • Imagen: Elige una imagen de sistema operativo (por ejemplo, Windows Server, Ubuntu).
    • Tamaño: Selecciona el tamaño de la VM según tus necesidades.
# Ejemplo: Comando de Azure CLI para crear una VM
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Conectándote a Tu VM

  • SSH para VMs Linux: Usa un cliente SSH para conectarte.
  • RDP para VMs Windows: Usa el Protocolo de Escritorio Remoto (RDP).
# Ejemplo: SSH en una VM Linux
ssh azureuser@<public-ip-address>

Tareas de Gestión Intermedia

Gestión de Tamaños de VM y Escalado

  • Redimensionar una VM: Cambia el tamaño de tu VM para escalar hacia arriba o hacia abajo según tus necesidades.
  • Autoescalado: Usa Conjuntos de Escala de Azure para escalar automáticamente las VMs según la demanda.
# Ejemplo: Redimensionar una VM usando Azure CLI
az vm resize \
  --resource-group myResourceGroup \
  --name myVM \
  --size Standard_DS3_v2

Gestión de Discos

  • Agregar Discos de Datos: Adjunta discos adicionales a tu VM para almacenamiento extra.
  • Tipos de Discos: Entiende la diferencia entre HDD Estándar, SSD Estándar y SSD Premium.
# Ejemplo: Adjuntar un nuevo disco de datos a una VM
az vm disk attach \
  --resource-group myResourceGroup \
  --vm-name myVM \
  --name myDataDisk \
  --size-gb 128

Redes

  • Redes Virtuales (VNet): Crea y gestiona VNets para controlar la configuración de red de la VM.
  • Grupos de Seguridad de Red (NSG): Configura NSGs para gestionar el tráfico entrante y saliente.
# Ejemplo: Crear una VNet y Subred
az network vnet create \
  --resource-group myResourceGroup \
  --name myVNet \
  --address-prefix 10.0.0.0/16 \
  --subnet-name mySubnet \
  --subnet-prefix 10.0.1.0/24

Tareas de Gestión Avanzada

Automatización con Plantillas de Azure Resource Manager (ARM)

  • Plantillas ARM: Define tu infraestructura como código usando plantillas JSON.
  • Desplegando Plantillas ARM: Usa Azure CLI o el Portal de Azure para desplegar plantillas.
{
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Compute/virtualMachines",
      "apiVersion": "2019-12-01",
      "name": "myVM",
      "location": "[resourceGroup().location]",
      "properties": {
        "hardwareProfile": {
          "vmSize": "Standard_DS1_v2"
        },
        "storageProfile": {
          "imageReference": {
            "publisher": "Canonical",
            "offer": "UbuntuServer",
            "sku": "18.04-LTS",
            "version": "latest"
          },
          "osDisk": {
            "createOption": "FromImage"
          }
        },
        "osProfile": {
          "computerName": "myVM",
          "adminUsername": "azureuser",
          "linuxConfiguration": {
            "disablePasswordAuthentication": true,
            "ssh": {
              "publicKeys": [
                {
                  "path": "/home/azureuser/.ssh/authorized_keys",
                  "keyData": "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQC..."
                }
              ]
            }
          }
        },
        "networkProfile": {
          "networkInterfaces": [
            {
              "id": "[resourceId('Microsoft.Network/networkInterfaces', 'myNic')]"
            }
          ]
        }
      }
    }
  ]
}

Monitoreo y Diagnóstico

  • Azure Monitor: Recoge y analiza datos de telemetría de tus VMs.
  • Log Analytics: Consulta y analiza registros para obtener información más profunda.
# Ejemplo: Habilitar monitoreo en una VM
az vm diagnostics set \
  --resource-group myResourceGroup \
  --vm-name myVM \
  --settings '{"storageAccount": "mystorageaccount"}'

Respaldo y Recuperación

  • Azure Backup: Configura políticas de respaldo para proteger tus VMs.
  • Recuperación ante Desastres: Usa Azure Site Recovery para replicación y conmutación por error de VMs.
# Ejemplo: Habilitar respaldo para una VM
az backup protection enable-for-vm \
  --resource-group myResourceGroup \
  --vault-name myRecoveryServicesVault \
  --vm myVM \
  --policy-name DefaultPolicy

Conclusión

Gestionar máquinas virtuales en Azure implica una variedad de tareas desde la creación y conexión básica hasta la automatización y monitoreo avanzados. Al dominar estas habilidades, puedes aprovechar efectivamente las capacidades de Azure para desplegar, escalar y mantener tu infraestructura virtual. Practica regularmente y explora la extensa documentación de Azure para profundizar tu comprensión y competencia.
