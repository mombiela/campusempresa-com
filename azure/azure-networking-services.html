Introducción a las Redes de Azure

Los Servicios de Redes de Azure proporcionan la infraestructura y las herramientas necesarias para conectar y gestionar tus recursos en la nube. Comprender estos servicios es crucial para construir soluciones en la nube seguras, escalables y eficientes.

Servicios Clave de Redes de Azure

  • Red Virtual (VNet)
  • Azure Load Balancer
  • Azure Application Gateway
  • Azure VPN Gateway
  • Azure ExpressRoute
  • Azure DNS
  • Azure Traffic Manager

Red Virtual (VNet)

Azure Virtual Network (VNet) es el bloque de construcción fundamental para tu red privada en Azure. VNet permite que muchos tipos de recursos de Azure, como las Máquinas Virtuales de Azure (VMs), se comuniquen de manera segura entre sí, con internet y con redes locales.

Conceptos Clave

  • Subredes: Divide tu VNet en múltiples subredes para una mejor organización y seguridad.
  • Grupos de Seguridad de Red (NSGs): Controla el tráfico entrante y saliente hacia las interfaces de red (NIC), VMs y subredes.
  • Peering: Conecta VNets entre sí dentro de la misma región o entre regiones.

Ejemplo: Creando una VNet

az network vnet create \
  --name MyVNet \
  --resource-group MyResourceGroup \
  --address-prefix 10.0.0.0/16 \
  --subnet-name MySubnet \
  --subnet-prefix 10.0.1.0/24

Este comando crea una VNet llamada MyVNet con un espacio de direcciones de 10.0.0.0/16 y una subred llamada MySubnet con un rango de direcciones de 10.0.1.0/24.

Azure Load Balancer

Azure Load Balancer distribuye el tráfico de red entrante entre múltiples VMs para asegurar que ninguna VM se sobrecargue.

Conceptos Clave

  • Load Balancer Público: Equilibra el tráfico de internet hacia las VMs.
  • Load Balancer Interno: Equilibra el tráfico dentro de una VNet.
  • Health Probes: Monitorea el estado de las VMs para asegurar que el tráfico solo se envíe a instancias saludables.

Ejemplo: Creando un Load Balancer

az network lb create \
  --resource-group MyResourceGroup \
  --name MyLoadBalancer \
  --frontend-ip-name MyFrontEndPool \
  --backend-pool-name MyBackEndPool \
  --sku Basic

Este comando crea un load balancer básico llamado MyLoadBalancer con pools de frontend y backend.

Azure Application Gateway

Azure Application Gateway es un equilibrador de carga de tráfico web que te permite gestionar el tráfico hacia tus aplicaciones web.

Conceptos Clave

  • Enrutamiento basado en URL: Enruta el tráfico basado en las rutas de URL.
  • Terminación SSL: Desencripta el tráfico SSL en el gateway.
  • Firewall de Aplicaciones Web (WAF): Protege tus aplicaciones web de amenazas comunes.

Ejemplo: Creando un Application Gateway

az network application-gateway create \
  --name MyAppGateway \
  --resource-group MyResourceGroup \
  --vnet-name MyVNet \
  --subnet MySubnet \
  --capacity 2 \
  --sku WAF_v2 \
  --http-settings-cookie-based-affinity Enabled \
  --frontend-port 80 \
  --routing-rule-type Basic

Este comando crea un application gateway llamado MyAppGateway con WAF habilitado y reglas de enrutamiento básicas.

Azure VPN Gateway

Azure VPN Gateway conecta tus redes locales a Azure a través de VPNs sitio a sitio.

Conceptos Clave

  • VPN Sitio a Sitio: Conecta tu red local a Azure.
  • VPN Punto a Sitio: Conecta dispositivos individuales a tu red de Azure.
  • ExpressRoute: Proporciona una conexión privada a Azure.

Ejemplo: Creando un VPN Gateway

az network vnet-gateway create \
  --resource-group MyResourceGroup \
  --name MyVPNGateway \
  --public-ip-address MyGatewayIP \
  --vnet MyVNet \
  --gateway-type Vpn \
  --vpn-type RouteBased \
  --sku VpnGw1 \
  --no-wait

Este comando crea un VPN gateway llamado MyVPNGateway con un tipo de VPN basado en rutas.

Azure ExpressRoute

Azure ExpressRoute proporciona una conexión privada entre tu infraestructura local y los centros de datos de Azure.

Conceptos Clave

  • Peering Privado: Conecta a los servicios de Azure usando direcciones IP privadas.
  • Peering Público: Conecta a los servicios de Azure usando direcciones IP públicas.
  • Peering de Microsoft: Conecta a servicios de Microsoft como Office 365.

Ejemplo: Creando un Circuito ExpressRoute

az network express-route create \
  --name MyExpressRoute \
  --resource-group MyResourceGroup \
  --bandwidth 200 \
  --peering-location "Silicon Valley" \
  --provider "Equinix" \
  --sku-family MeteredData \
  --sku-tier Standard

Este comando crea un circuito ExpressRoute llamado MyExpressRoute con un ancho de banda de 200 Mbps.

Azure DNS

Azure DNS es un servicio de alojamiento para dominios DNS, proporcionando resolución de nombres usando la infraestructura de Microsoft Azure.

Conceptos Clave

  • Zonas DNS: Contenedores para registros DNS.
  • Registros DNS: Mapean nombres de dominio a direcciones IP.

Ejemplo: Creando una Zona DNS

az network dns zone create \
  --resource-group MyResourceGroup \
  --name mydomain.com

Este comando crea una zona DNS llamada mydomain.com.

Azure Traffic Manager

Azure Traffic Manager es un equilibrador de carga de tráfico basado en DNS que te permite distribuir el tráfico de manera óptima a servicios en regiones globales de Azure.

Conceptos Clave

  • Métodos de Enrutamiento de Tráfico: Rendimiento, Prioridad, Ponderado, Geográfico, Multivalor, Subred.
  • Endpoints: Representan los destinos para el tráfico.

Ejemplo: Creando un Perfil de Traffic Manager

az network traffic-manager profile create \
  --name MyTrafficManagerProfile \
  --resource-group MyResourceGroup \
  --routing-method Performance \
  --unique-dns-name myapptrafficmanager \
  --ttl 30 \
  --protocol HTTP \
  --port 80 \
  --path "/"

Este comando crea un perfil de Traffic Manager llamado MyTrafficManagerProfile con enrutamiento basado en rendimiento.

Conclusión

Los Servicios de Redes de Azure proporcionan un conjunto completo de herramientas para conectar, gestionar y asegurar tus recursos en la nube. Desde Redes Virtuales y Load Balancers hasta VPN Gateways y Traffic Managers, comprender estos servicios es esencial para construir soluciones en la nube robustas y escalables. Al dominar estas herramientas, puedes asegurar que tus aplicaciones sean altamente disponibles, seguras y de alto rendimiento.
