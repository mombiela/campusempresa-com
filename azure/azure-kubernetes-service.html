Introducción al Servicio de Kubernetes de Azure (AKS)

Azure Kubernetes Service (AKS) es un servicio de orquestación de contenedores gestionado basado en el sistema de Kubernetes de código abierto. AKS simplifica el despliegue de un clúster de Kubernetes gestionado en Azure al descargar la sobrecarga operativa a Azure. Esta sección cubrirá los conceptos básicos de AKS, sus beneficios y conceptos clave.

  • ¿Qué es Kubernetes?

    • Una plataforma de código abierto para automatizar el despliegue, escalado y operaciones de contenedores de aplicaciones.
    • Gestiona aplicaciones en contenedores a través de múltiples hosts.
    • Proporciona infraestructura centrada en contenedores.

  • ¿Qué es AKS?

    • Un servicio de Kubernetes gestionado proporcionado por Azure.
    • Simplifica la gestión de clústeres de Kubernetes.
    • Se integra con servicios de Azure para monitoreo, escalado y seguridad.

  • Beneficios de AKS:

    • Servicio Gestionado: Azure maneja el plano de control de Kubernetes.
    • Escalabilidad: Escala aplicaciones fácilmente hacia arriba o hacia abajo.
    • Integración: Integración perfecta con otros servicios de Azure.
    • Seguridad: Funciones de seguridad integradas y cumplimiento con los estándares de Azure.

Configuración de AKS

Esta sección te guiará a través de la configuración de un clúster de AKS desde cero.

Requisitos Previos

  • Una suscripción activa de Azure.
  • Azure CLI instalado en tu máquina local.
  • Comprensión básica de los conceptos de Kubernetes.

Pasos para Crear un Clúster de AKS

  1. Instalar Azure CLI:

    curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

  2. Iniciar sesión en Azure:

    az login

  3. Crear un Grupo de Recursos:

    az group create --name myResourceGroup --location eastus

  4. Crear un Clúster de AKS:

    az aks create --resource-group myResourceGroup --name myAKSCluster --node-count 1 --enable-addons monitoring --generate-ssh-keys

  5. Obtener Credenciales de AKS:

    az aks get-credentials --resource-group myResourceGroup --name myAKSCluster

  6. Verificar el Clúster:

    kubectl get nodes

Desplegando Aplicaciones en AKS

Esta sección cubrirá cómo desplegar aplicaciones en tu clúster de AKS usando manifiestos de Kubernetes.

Creando un Despliegue

  • Ejemplo de YAML de Despliegue:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

  • Aplicar el Despliegue:

    kubectl apply -f nginx-deployment.yaml

  • Verificar el Despliegue:

    kubectl get deployments
kubectl get pods

Exponiendo la Aplicación

  • YAML del Servicio:

    apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: LoadBalancer
  selector:
    app: nginx
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

  • Aplicar el Servicio:

    kubectl apply -f nginx-service.yaml

  • Obtener la IP Externa:

    kubectl get services

Funciones Avanzadas de AKS

Esta sección profundizará en funciones más avanzadas de AKS, como escalado, monitoreo y seguridad.

Escalando Aplicaciones

  • Autoscaler Horizontal de Pods (HPA):

    apiVersion: autoscaling/v1
kind: HorizontalPodAutoscaler
metadata:
  name: nginx-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: nginx-deployment
  minReplicas: 1
  maxReplicas: 10
  targetCPUUtilizationPercentage: 50

  • Aplicar HPA:

    kubectl apply -f nginx-hpa.yaml

Monitoreo y Registro

  • Azure Monitor para Contenedores:

    • Se integra con AKS para proporcionar monitoreo y registro.
    • Se habilita automáticamente si se especifica durante la creación del clúster.

  • Visualización de Métricas:

    • Usa el Portal de Azure para ver métricas y registros.
    • Usa kubectl top para ver el uso de recursos.

Mejores Prácticas de Seguridad

  • Control de Acceso Basado en Roles (RBAC):

    • Usa RBAC para controlar el acceso a la API de Kubernetes.
    • Define roles y enlaces de roles.

  • Políticas de Red:

    • Usa políticas de red para controlar el tráfico entre pods.
    • Ejemplo de YAML de Política de Red: 
      apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-nginx
spec:
  podSelector:
    matchLabels:
      app: nginx
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: nginx
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: nginx

Conclusión

Azure Kubernetes Service (AKS) es una herramienta poderosa para gestionar aplicaciones en contenedores a escala. Al aprovechar AKS, puedes simplificar el despliegue, la gestión y el escalado de clústeres de Kubernetes. Este curso ha cubierto los conceptos básicos de AKS, la configuración de un clúster, el despliegue de aplicaciones y funciones avanzadas como escalado, monitoreo y seguridad. Con este conocimiento, estás bien equipado para comenzar a usar AKS para tus necesidades de orquestación de contenedores.
