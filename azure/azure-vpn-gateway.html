Introducción

Azure VPN Gateway es un componente crítico en la red de Azure que te permite conectar de manera segura tus redes locales con los recursos de Azure. Esta sección cubrirá los conceptos básicos de Azure VPN Gateway, sus tipos, configuraciones y características avanzadas.

Conceptos Clave

  • Puerta de Enlace de Red Virtual: Una puerta de enlace de red virtual está compuesta por dos o más máquinas virtuales que se configuran y despliegan automáticamente en una subred específica llamada subred de puerta de enlace.
  • Subred de Puerta de Enlace: Una subred específica en tu red virtual que contiene los recursos de la puerta de enlace de red virtual.
  • Tipos de Puerta de Enlace VPN: Hay dos tipos de puertas de enlace VPN: Basada en Rutas y Basada en Políticas.
  • Tipos de Conexión: Conexiones Sitio a Sitio (S2S), Punto a Sitio (P2S) y VNet a VNet.

Configuración de una Puerta de Enlace VPN

Prerrequisitos

  • Una suscripción de Azure.
  • Una red virtual con una subred de puerta de enlace.
  • Una dirección IP pública para la puerta de enlace VPN.

Pasos para Crear una Puerta de Enlace VPN

  1. Crear una Red Virtual:

    az network vnet create --name MyVNet --resource-group MyResourceGroup --address-prefix 10.1.0.0/16 --subnet-name MySubnet --subnet-prefix 10.1.0.0/24

  2. Crear una Subred de Puerta de Enlace:

    az network vnet subnet create --address-prefix 10.1.255.0/27 --name GatewaySubnet --resource-group MyResourceGroup --vnet-name MyVNet

  3. Crear una Dirección IP Pública:

    az network public-ip create --name MyGatewayIP --resource-group MyResourceGroup --allocation-method Dynamic

  4. Crear la Puerta de Enlace VPN:

    az network vnet-gateway create --name MyVPNGateway --public-ip-address MyGatewayIP --resource-group MyResourceGroup --vnet MyVNet --gateway-type Vpn --vpn-type RouteBased --sku VpnGw1 --no-wait

Configuración de Conexiones VPN

VPN Sitio a Sitio (S2S)

  • Definición: Conecta tu red local a una VNet de Azure a través de un túnel VPN IPsec/IKE.
  • Caso de Uso: Ideal para extender tu red local a Azure.

VPN Punto a Sitio (P2S)

  • Definición: Permite que clientes individuales se conecten a tu VNet de Azure desde cualquier lugar a través de internet.
  • Caso de Uso: Adecuado para trabajadores remotos que necesitan acceso seguro a recursos de Azure.

VPN VNet a VNet

  • Definición: Conecta dos VNets de Azure.
  • Caso de Uso: Útil para conectar VNets en diferentes regiones o suscripciones.

Ejemplo: Configuración de una VPN Sitio a Sitio

  1. Crear una Puerta de Enlace de Red Local:

    az network local-gateway create --gateway-ip-address <OnPremisesPublicIP> --name MyLocalGateway --resource-group MyResourceGroup --local-address-prefixes 10.2.0.0/16

  2. Crear la Conexión VPN:

    az network vpn-connection create --name MyS2SConnection --resource-group MyResourceGroup --vnet-gateway1 MyVPNGateway --local-gateway2 MyLocalGateway --shared-key 'MySharedKey'

Características Avanzadas

Alta Disponibilidad

  • Modo Activo-Activo: Despliega dos puertas de enlace VPN en una configuración activo-activo para alta disponibilidad.
  • Modo Activo-Pasivo: Configuración predeterminada donde una puerta de enlace está activa y la otra es pasiva.

BGP (Protocolo de Puerta de Enlace Fronteriza)

  • Definición: BGP se utiliza para intercambiar información de enrutamiento entre redes.
  • Caso de Uso: Útil para topologías de red complejas y enrutamiento dinámico.

Integración con ExpressRoute

  • Definición: ExpressRoute proporciona una conexión privada a Azure.
  • Caso de Uso: Para escenarios que requieren mayor ancho de banda y menor latencia que las conexiones VPN típicas.

Conclusión

Azure VPN Gateway es una forma versátil y segura de conectar tus redes locales, clientes remotos y otras VNets de Azure. Al entender los diferentes tipos de puertas de enlace VPN y conexiones, puedes diseñar una arquitectura de red robusta y escalable en Azure. Ya sea que estés configurando una simple VPN sitio a sitio o una conexión VNet a VNet multi-región compleja, Azure VPN Gateway proporciona las herramientas y características necesarias para satisfacer tus necesidades de red.
