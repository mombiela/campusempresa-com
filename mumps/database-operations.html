Introducción a la Base de Datos MUMPS

MUMPS (Massachusetts General Hospital Utility Multi-Programming System), también conocido como M, es un lenguaje poderoso diseñado para operaciones de base de datos. Es particularmente fuerte en el manejo de datos jerárquicos y multidimensionales.

Conceptos Clave

  • Variables Globales: Almacenamiento persistente en MUMPS, utilizado para almacenar datos a través de sesiones.
  • Variables Locales: Almacenamiento temporal, utilizado dentro de la sesión o rutina actual.
  • Nodos: Elementos de variables globales, formando una estructura jerárquica.
  • Subíndices: Índices utilizados para acceder a nodos en variables globales.

Operaciones Básicas de Base de Datos

Creación y Almacenamiento de Datos

Para almacenar datos en MUMPS, se utilizan variables globales. Aquí hay un ejemplo simple:

SET ^Patient(1, "Name") = "John Doe"
SET ^Patient(1, "Age") = 30
SET ^Patient(1, "Condition") = "Healthy"

Explicación:

  • ^Patient es la variable global.
  • 1 es el primer subíndice, representando el ID del paciente.
  • "Name", "Age", y "Condition" son subíndices adicionales que representan diferentes atributos del paciente.

Recuperación de Datos

Para recuperar datos, se utiliza el comando WRITE:

WRITE ^Patient(1, "Name")

Esto mostrará:

John Doe

Eliminación de Datos

Para eliminar un nodo o una rama entera, se utiliza el comando KILL:

KILL ^Patient(1, "Condition")

Esto eliminará el nodo "Condition" para el paciente 1.

Operaciones Intermedias de Base de Datos

Iteración Sobre Datos

Se puede iterar sobre nodos utilizando el bucle FOR y el comando ORDER:

SET patientID = ""
FOR  SET patientID = $ORDER(^Patient(patientID)) QUIT:patientID=""  DO
. WRITE "Patient ID: ", patientID, " Name: ", ^Patient(patientID, "Name"), !

Explicación:

  • $ORDER(^Patient(patientID)) recupera el siguiente subíndice en la variable global ^Patient.
  • QUIT:patientID="" sale del bucle cuando no hay más pacientes.

Recuperación Condicional

Se pueden recuperar datos condicionalmente utilizando declaraciones IF:

SET patientID = 1
IF ^Patient(patientID, "Age") > 25 WRITE "Patient is older than 25"

Operaciones Avanzadas de Base de Datos

Estructuras de Datos Multinivel

MUMPS soporta estructuras de datos complejas y multinivel. Aquí hay un ejemplo de un registro de paciente más complejo:

SET ^Patient(1, "Name") = "John Doe"
SET ^Patient(1, "Age") = 30
SET ^Patient(1, "Condition", "Primary") = "Healthy"
SET ^Patient(1, "Condition", "Secondary") = "None"

Transacciones

MUMPS soporta transacciones para asegurar la integridad de los datos. Utilice TSTART y TCOMMIT para gestionar transacciones:

TSTART
SET ^Patient(2, "Name") = "Jane Smith"
SET ^Patient(2, "Age") = 28
TCOMMIT

Si ocurre un error, se puede utilizar TROLLBACK para deshacer los cambios:

TSTART
SET ^Patient(3, "Name") = "Alice Brown"
IF $GET(^Patient(3, "Name"))="" DO
. TROLLBACK
. WRITE "Transaction failed", !
ELSE  DO
. TCOMMIT
. WRITE "Transaction successful", !

Resumen

En este tema, cubrimos lo esencial de las operaciones de base de datos en MUMPS, desde el almacenamiento y la recuperación de datos básicos hasta conceptos más avanzados como estructuras de datos multinivel y transacciones. Entender estas operaciones es crucial para gestionar y manipular datos de manera efectiva en MUMPS.

Puntos Clave

  • MUMPS utiliza variables globales para el almacenamiento persistente de datos.
  • Las operaciones básicas incluyen establecer, recuperar y eliminar datos.
  • Las operaciones intermedias implican iterar sobre datos y recuperación condicional.
  • Las operaciones avanzadas incluyen el manejo de estructuras de datos multinivel y la gestión de transacciones para la integridad de los datos.

Al dominar estas operaciones de base de datos, estarás bien equipado para manejar tareas complejas de gestión de datos en MUMPS.
