Clases y Objetos en F#

Introducción

Las clases y los objetos son conceptos fundamentales en la programación orientada a objetos (OOP). En F#, aunque el lenguaje es principalmente funcional, admite principios de OOP, lo que te permite definir y usar clases y objetos. Esta sección te guiará desde los conceptos básicos hasta el uso avanzado de clases y objetos en F#.

Conceptos Clave

  • Clase: Un plano para crear objetos, definiendo propiedades, métodos y eventos.
  • Objeto: Una instancia de una clase.
  • Constructor: Un método especial utilizado para inicializar objetos.
  • Herencia: Un mecanismo para crear una nueva clase utilizando las propiedades y métodos de una clase existente.
  • Encapsulación: La agrupación de datos con métodos que operan sobre esos datos.
  • Polimorfismo: La capacidad de procesar objetos de manera diferente según su tipo de datos o clase.

Definiendo una Clase

En F#, defines una clase usando la palabra clave type seguida del nombre de la clase y sus miembros.

type Person(name: string, age: int) =
    member this.Name = name
    member this.Age = age
    member this.Introduce() = 
        printfn "Hi, my name is %s and I am %d years old." this.Name this.Age

Explicación

  • type Person(name: string, age: int) define una clase llamada Person con dos parámetros: name y age.
  • member this.Name = name y member this.Age = age definen propiedades.
  • member this.Introduce() define un método que imprime un mensaje de saludo.

Creando un Objeto

Para crear un objeto, usas la palabra clave new seguida del nombre de la clase y los parámetros del constructor.

let person1 = new Person("Alice", 30)
person1.Introduce()

Explicación

  • let person1 = new Person("Alice", 30) crea una instancia de la clase Person.
  • person1.Introduce() llama al método Introduce en el objeto person1.

Herencia

La herencia te permite crear una nueva clase que reutiliza, extiende o modifica el comportamiento de otra clase.

type Employee(name: string, age: int, employeeId: string) =
    inherit Person(name, age)
    member this.EmployeeId = employeeId
    member this.Work() = 
        printfn "%s is working." this.Name

Explicación

  • inherit Person(name, age) indica que Employee hereda de Person.
  • member this.EmployeeId = employeeId añade una nueva propiedad específica para Employee.
  • member this.Work() define un nuevo método para Employee.

Encapsulación

La encapsulación se logra controlando el acceso a los miembros de la clase usando modificadores de acceso como private y public.

type BankAccount(initialBalance: decimal) =
    let mutable balance = initialBalance
    member this.Deposit(amount: decimal) =
        balance <- balance + amount
    member this.Withdraw(amount: decimal) =
        if amount <= balance then
            balance <- balance - amount
        else
            printfn "Insufficient funds"
    member this.Balance = balance

Explicación

  • let mutable balance = initialBalance define un campo mutable privado.
  • member this.Deposit(amount: decimal) y member this.Withdraw(amount: decimal) son métodos públicos para modificar el balance.
  • member this.Balance es una propiedad pública para acceder al balance.

Polimorfismo

El polimorfismo permite que los métodos procesen objetos de manera diferente según su clase.

type Shape() =
    abstract member Area: unit -> float

type Circle(radius: float) =
    inherit Shape()
    override this.Area() = Math.PI * radius * radius

type Square(side: float) =
    inherit Shape()
    override this.Area() = side * side

let shapes: Shape list = [ Circle(5.0); Square(4.0) ]
shapes |> List.iter (fun shape -> printfn "Area: %f" (shape.Area()))

Explicación

  • abstract member Area: unit -> float define un método abstracto en la clase Shape.
  • override this.Area() proporciona implementaciones específicas en Circle y Square.
  • shapes |> List.iter (fun shape -> printfn "Area: %f" (shape.Area())) itera sobre una lista de formas y imprime sus áreas.

Conclusión

En esta sección, cubrimos los conceptos básicos de clases y objetos en F#, incluyendo la definición de clases, creación de objetos, herencia, encapsulación y polimorfismo. Estos conceptos son cruciales para construir aplicaciones robustas y mantenibles. Practica creando tus propias clases y experimentando con estas características para profundizar tu comprensión.
