Introducción

En esta sección, exploraremos dos conceptos fundamentales de la programación orientada a objetos (OOP) en F#: Encapsulación y Polimorfismo. Estos conceptos ayudan a crear software robusto, mantenible y escalable.

Encapsulación

La encapsulación es el concepto de agrupar datos y métodos que operan sobre esos datos dentro de una sola unidad, típicamente una clase. Restringe el acceso directo a algunos de los componentes de un objeto, lo que puede prevenir la modificación accidental de datos.

Conceptos Clave de la Encapsulación

  • Ocultación de Datos: Restringir el acceso al estado interno del objeto.
  • Interfaz Pública: Proporcionar métodos para interactuar con los datos del objeto.

Ejemplo de Encapsulación en F#

type Person(name: string, age: int) =
    // Campos privados
    let mutable privateName = name
    let mutable privateAge = age

    // Métodos públicos
    member this.GetName() = privateName
    member this.GetAge() = privateAge
    member this.SetAge(newAge: int) =
        if newAge > 0 then
            privateAge <- newAge
        else
            printfn "La edad debe ser positiva"

// Uso
let person = Person("John", 30)
printfn "Nombre: %s" (person.GetName())
printfn "Edad: %d" (person.GetAge())
person.SetAge(35)
printfn "Edad Actualizada: %d" (person.GetAge())

En este ejemplo:

  • La clase Person encapsula los campos name y age.
  • Los campos son privados y solo pueden ser accedidos o modificados a través de métodos públicos (GetName, GetAge, SetAge).

Polimorfismo

El polimorfismo permite que objetos de diferentes clases sean tratados como objetos de una clase superior común. Es una forma de realizar una sola acción en diferentes formas.

Conceptos Clave del Polimorfismo

  • Sobrescritura de Métodos: Una subclase proporciona una implementación específica de un método que ya está definido en su superclase.
  • Implementación de Interfaces: Diferentes clases pueden implementar la misma interfaz de diferentes maneras.

Ejemplo de Polimorfismo en F#

Sobrescritura de Métodos

type Animal() =
    abstract member Speak: unit -> unit

type Dog() =
    inherit Animal()
    override this.Speak() = printfn "¡Guau!"

type Cat() =
    inherit Animal()
    override this.Speak() = printfn "¡Miau!"

// Uso
let animals: Animal list = [ Dog(); Cat() ]
animals |> List.iter (fun animal -> animal.Speak())

En este ejemplo:

  • Animal es una clase abstracta con un método abstracto Speak.
  • Las clases Dog y Cat heredan de Animal y proporcionan su propia implementación del método Speak.
  • Se crea una lista de objetos Animal, que contiene tanto objetos Dog como Cat. Se llama al método Speak en cada objeto, demostrando el polimorfismo.

Implementación de Interfaces

type IShape =
    abstract member Area: unit -> float

type Circle(radius: float) =
    interface IShape with
        member this.Area() = System.Math.PI * radius * radius

type Rectangle(width: float, height: float) =
    interface IShape with
        member this.Area() = width * height

// Uso
let shapes: IShape list = [ Circle(5.0); Rectangle(4.0, 6.0) ]
shapes |> List.iter (fun shape -> printfn "Área: %f" (shape.Area()))

En este ejemplo:

  • IShape es una interfaz con un método Area.
  • Las clases Circle y Rectangle implementan la interfaz IShape.
  • Se crea una lista de objetos IShape, que contiene tanto objetos Circle como Rectangle. Se llama al método Area en cada objeto, demostrando el polimorfismo.

Conclusión

La encapsulación y el polimorfismo son conceptos poderosos en F# que ayudan a crear código bien estructurado y flexible. La encapsulación asegura que el estado interno de un objeto esté protegido, mientras que el polimorfismo permite un código flexible y reutilizable al permitir que los objetos sean tratados como instancias de su clase padre o interfaz. Comprender y aplicar estos conceptos mejorará significativamente tu capacidad para escribir aplicaciones F# robustas y mantenibles.
