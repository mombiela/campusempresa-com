Introducción a los Patrones de Diseño

Los patrones de diseño son soluciones típicas a problemas comunes en el diseño de software. Son como planos que puedes personalizar para resolver un problema de diseño particular en tu código. En esta sección, exploraremos cómo implementar algunos de los patrones de diseño más comunes en F#.

Patrones Creacionales

Patrón Singleton

El patrón Singleton asegura que una clase tenga solo una instancia y proporciona un punto de acceso global a ella.

Ejemplo:

type Singleton private () =
    static let instance = Singleton()
    static member Instance = instance

// Uso
let singletonInstance = Singleton.Instance

Patrón Método de Fábrica

El patrón Método de Fábrica define una interfaz para crear un objeto pero permite que las subclases alteren el tipo de objetos que se crearán.

Ejemplo:

type IProduct =
    abstract member Operation: unit -> string

type ConcreteProductA() =
    interface IProduct with
        member this.Operation() = "Resultado de ConcreteProductA"

type ConcreteProductB() =
    interface IProduct with
        member this.Operation() = "Resultado de ConcreteProductB"

type Creator() =
    member this.FactoryMethod(productType: string) : IProduct =
        match productType with
        | "A" -> ConcreteProductA() :> IProduct
        | "B" -> ConcreteProductB() :> IProduct
        | _ -> failwith "Tipo de producto desconocido"

// Uso
let creator = Creator()
let productA = creator.FactoryMethod("A")
printfn "%s" (productA.Operation())

Patrones Estructurales

Patrón Adaptador

El patrón Adaptador permite que interfaces incompatibles trabajen juntas. Actúa como un puente entre dos interfaces incompatibles.

Ejemplo:

type ITarget =
    abstract member Request: unit -> string

type Adaptee() =
    member this.SpecificRequest() = "Solicitud específica."

type Adapter(adaptee: Adaptee) =
    interface ITarget with
        member this.Request() = adaptee.SpecificRequest()

// Uso
let adaptee = Adaptee()
let adapter = Adapter(adaptee)
printfn "%s" (adapter :> ITarget).Request()

Patrón Decorador

El patrón Decorador permite agregar comportamiento a un objeto individual, dinámicamente, sin afectar el comportamiento de otros objetos de la misma clase.

Ejemplo:

type IComponent =
    abstract member Operation: unit -> string

type ConcreteComponent() =
    interface IComponent with
        member this.Operation() = "ConcreteComponent"

type Decorator(component: IComponent) =
    interface IComponent with
        member this.Operation() = sprintf "Decorator(%s)" (component.Operation())

// Uso
let component = ConcreteComponent()
let decorator = Decorator(component)
printfn "%s" (decorator :> IComponent).Operation()

Patrones de Comportamiento

Patrón Observador

El patrón Observador define una dependencia de uno a muchos entre objetos para que cuando un objeto cambie de estado, todos sus dependientes sean notificados y actualizados automáticamente.

Ejemplo:

type IObserver =
    abstract member Update: string -> unit

type Subject() =
    let observers = System.Collections.Generic.List<IObserver>()
    member this.Attach(observer: IObserver) = observers.Add(observer)
    member this.Detach(observer: IObserver) = observers.Remove(observer)
    member this.Notify(message: string) =
        for observer in observers do
            observer.Update(message)

type ConcreteObserver(name: string) =
    interface IObserver with
        member this.Update(message: string) =
            printfn "Observer %s received message: %s" name message

// Uso
let subject = Subject()
let observer1 = ConcreteObserver("A")
let observer2 = ConcreteObserver("B")

subject.Attach(observer1)
subject.Attach(observer2)

subject.Notify("¡Hola Observadores!")

Patrón Estrategia

El patrón Estrategia define una familia de algoritmos, encapsula cada uno y los hace intercambiables. La estrategia permite que el algoritmo varíe independientemente de los clientes que lo usan.

Ejemplo:

type IStrategy =
    abstract member Execute: int -> int -> int

type ConcreteStrategyAdd() =
    interface IStrategy with
        member this.Execute(a: int, b: int) = a + b

type ConcreteStrategySubtract() =
    interface IStrategy with
        member this.Execute(a: int, b: int) = a - b

type Context(strategy: IStrategy) =
    member this.ExecuteStrategy(a: int, b: int) = strategy.Execute(a, b)

// Uso
let contextAdd = Context(ConcreteStrategyAdd())
let contextSubtract = Context(ConcreteStrategySubtract())

printfn "Add: %d" (contextAdd.ExecuteStrategy(3, 4))
printfn "Subtract: %d" (contextSubtract.ExecuteStrategy(3, 4))

Conclusión

Los patrones de diseño son herramientas poderosas en el kit de herramientas de un programador. Proporcionan soluciones probadas a problemas comunes y pueden mejorar significativamente la estructura y mantenibilidad de tu código. Al entender y aplicar estos patrones en F#, puedes escribir aplicaciones más robustas y escalables.
