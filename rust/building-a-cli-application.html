Introducción

En este módulo, exploraremos cómo construir una aplicación de Interfaz de Línea de Comandos (CLI) usando Rust. Las aplicaciones CLI son herramientas poderosas que permiten a los usuarios interactuar con el software a través de comandos de texto. Al final de este módulo, serás capaz de crear una aplicación CLI completamente funcional en Rust.

Prerrequisitos

Antes de comenzar este módulo, deberías tener:

  • Comprensión básica de la sintaxis y conceptos de Rust.
  • Rust instalado en tu máquina.
  • Familiaridad con el uso del terminal o símbolo del sistema.

Configurando Tu Proyecto

Para empezar, necesitamos crear un nuevo proyecto en Rust. Abre tu terminal y ejecuta el siguiente comando:

cargo new cli_app
cd cli_app

Este comando crea un nuevo proyecto en Rust llamado cli_app y navega dentro del directorio del proyecto.

Añadiendo Dependencias

Para construir aplicaciones CLI, utilizaremos el crate clap, que proporciona una manera simple de analizar argumentos de línea de comandos.

Añade lo siguiente a tu archivo Cargo.toml:

[dependencies]
clap = "3.0"

Aplicación CLI Básica

Comencemos creando una aplicación CLI básica que acepte un solo argumento y lo imprima.

main.rs

use clap::Parser;

/// Programa simple para saludar a una persona
#[derive(Parser)]
struct Cli {
    /// El nombre de la persona a saludar
    name: String,
}

fn main() {
    let args = Cli::parse();
    println!("¡Hola, {}!", args.name);
}

Explicación

  • use clap::Parser;: Importa el trait Parser del crate clap.
  • #[derive(Parser)]: Deriva el trait Parser para la estructura Cli, permitiéndole analizar argumentos de línea de comandos.
  • struct Cli: Define una estructura con un solo campo name para contener el argumento.
  • Cli::parse(): Analiza los argumentos de línea de comandos y devuelve una instancia de Cli.
  • println!: Imprime un mensaje de saludo usando el nombre proporcionado.

Manejo de Múltiples Argumentos

A continuación, extendamos nuestra aplicación para manejar múltiples argumentos y opciones.

main.rs

use clap::{Parser, ArgEnum};

#[derive(Parser)]
struct Cli {
    /// El nombre de la persona a saludar
    name: String,
    /// El número de veces para saludar
    #[clap(short, long, default_value_t = 1)]
    count: u8,
    /// El idioma a usar para el saludo
    #[clap(arg_enum, short, long, default_value_t = Language::English)]
    language: Language,
}

#[derive(Copy, Clone, PartialEq, Eq, PartialOrd, Ord, ArgEnum)]
enum Language {
    English,
    Spanish,
    French,
}

fn main() {
    let args = Cli::parse();
    for _ in 0..args.count {
        match args.language {
            Language::English => println!("Hello, {}!", args.name),
            Language::Spanish => println!("¡Hola, {}!", args.name),
            Language::French => println!("Bonjour, {}!", args.name),
        }
    }
}

Explicación

  • #[clap(short, long, default_value_t = 1)]: Especifica que count puede ser establecido usando -c o --count y por defecto es 1.
  • #[clap(arg_enum, short, long, default_value_t = Language::English)]: Especifica que language puede ser establecido usando -l o --language y por defecto es English.
  • enum Language: Define una enumeración para los idiomas soportados.
  • match args.language: Coincide con el idioma proporcionado e imprime el saludo en el idioma correspondiente.

Manejo de Errores

El manejo adecuado de errores es crucial para aplicaciones CLI robustas. Añadamos manejo de errores a nuestra aplicación.

main.rs

use clap::{Parser, ArgEnum};
use std::error::Error;

#[derive(Parser)]
struct Cli {
    name: String,
    #[clap(short, long, default_value_t = 1)]
    count: u8,
    #[clap(arg_enum, short, long, default_value_t = Language::English)]
    language: Language,
}

#[derive(Copy, Clone, PartialEq, Eq, PartialOrd, Ord, ArgEnum)]
enum Language {
    English,
    Spanish,
    French,
}

fn main() -> Result<(), Box<dyn Error>> {
    let args = Cli::parse();
    for _ in 0..args.count {
        match args.language {
            Language::English => println!("Hello, {}!", args.name),
            Language::Spanish => println!("¡Hola, {}!", args.name),
            Language::French => println!("Bonjour, {}!", args.name),
        }
    }
    Ok(())
}

Explicación

  • Result<(), Box<dyn Error>>: Cambia el tipo de retorno de main a un Result para manejar errores potenciales.
  • Ok(()): Indica una ejecución exitosa.

Características Avanzadas

Para aplicaciones CLI avanzadas, podrías necesitar subcomandos, archivos de configuración o análisis de argumentos más complejos. El crate clap soporta estas características.

Ejemplo de Subcomandos

use clap::{Parser, Subcommand};

#[derive(Parser)]
struct Cli {
    #[clap(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    Greet {
        name: String,
    },
    Farewell {
        name: String,
    },
}

fn main() {
    let args = Cli::parse();
    match args.command {
        Commands::Greet { name } => println!("¡Hola, {}!", name),
        Commands::Farewell { name } => println!("¡Adiós, {}!", name),
    }
}

Explicación

  • #[clap(subcommand)]: Especifica que command es un subcomando.
  • enum Commands: Define los subcomandos Greet y Farewell.

Conclusión

En este módulo, cubrimos los conceptos básicos de la construcción de una aplicación CLI en Rust usando el crate clap. Comenzamos con una aplicación simple, añadimos múltiples argumentos y opciones, implementamos manejo de errores y exploramos características avanzadas como subcomandos. Con estas habilidades, puedes crear aplicaciones CLI poderosas y fáciles de usar en Rust.
