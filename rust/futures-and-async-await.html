Futuros y Async/Await en Rust

Introducción

En esta sección, exploraremos la programación asincrónica en Rust utilizando Futuros y la sintaxis async/await. La programación asincrónica te permite escribir programas que pueden manejar muchas tareas concurrentemente, mejorando el rendimiento y la capacidad de respuesta.

Conceptos Clave

  • Futuros: Un futuro representa un valor que puede no estar disponible aún pero que será calculado eventualmente.
  • Async/Await: Sintaxis para trabajar con futuros de una manera más legible y mantenible.
  • Concurrencia vs. Paralelismo: La concurrencia trata de manejar muchas tareas a la vez, mientras que el paralelismo trata de hacer muchas tareas al mismo tiempo.

Futuros

¿Qué es un Futuro?

Un futuro es una abstracción para un valor que puede no estar disponible aún. Te permite escribir código no bloqueante que puede esperar a que el valor sea calculado.

Ejemplo de un Futuro

use futures::executor::block_on;

async fn hello_world() {
    println!("¡Hola, mundo!");
}

fn main() {
    let future = hello_world(); // Esto no ejecuta la función
    block_on(future); // Esto ejecuta la función
}
  • async fn hello_world() define una función asincrónica.
  • block_on(future) ejecuta el futuro hasta completarlo.

Async/Await

Funciones Asincrónicas

Una función async devuelve un futuro. Puedes usar la palabra clave await para esperar a que el futuro se complete.

Ejemplo de Async/Await

use std::time::Duration;
use tokio::time::sleep;

async fn say_hello() {
    sleep(Duration::from_secs(2)).await;
    println!("¡Hola, mundo!");
}

#[tokio::main]
async fn main() {
    say_hello().await;
}
  • sleep(Duration::from_secs(2)).await pausa la función por 2 segundos sin bloquear el hilo.
  • #[tokio::main] es una macro que configura el runtime de Tokio, que es necesario para ejecutar código asincrónico.

Combinando Futuros

Uniéndose a Futuros

Puedes ejecutar múltiples futuros concurrentemente usando la macro join!.

Ejemplo de Uniéndose a Futuros

use tokio::time::{sleep, Duration};

async fn task_one() {
    sleep(Duration::from_secs(1)).await;
    println!("Tarea uno completada");
}

async fn task_two() {
    sleep(Duration::from_secs(2)).await;
    println!("Tarea dos completada");
}

#[tokio::main]
async fn main() {
    tokio::join!(task_one(), task_two());
    println!("Ambas tareas completadas");
}
  • tokio::join!(task_one(), task_two()) ejecuta ambas tareas concurrentemente.

Manejo de Errores en Código Asincrónico

Usando `Result` con Funciones Asincrónicas

Las funciones asincrónicas pueden devolver tipos Result para el manejo de errores.

Ejemplo de Manejo de Errores

use tokio::time::{sleep, Duration};
use std::error::Error;

async fn might_fail() -> Result<(), Box<dyn Error>> {
    sleep(Duration::from_secs(1)).await;
    Err("Algo salió mal")?
}

#[tokio::main]
async fn main() {
    match might_fail().await {
        Ok(_) => println!("Éxito"),
        Err(e) => println!("Error: {}", e),
    }
}
  • Err("Algo salió mal")? propaga el error.
  • La declaración match maneja el resultado del futuro.

Conclusión

La programación asincrónica en Rust utilizando Futuros y la sintaxis async/await te permite escribir código eficiente y no bloqueante. Al entender cómo definir y ejecutar futuros, manejar errores y combinar múltiples futuros, puedes construir aplicaciones responsivas y de alto rendimiento.
