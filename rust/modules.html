Introducción a los Módulos

Los módulos en Rust son una forma de organizar el código en espacios de nombres separados, lo que facilita su gestión y comprensión. Permiten agrupar funciones, estructuras, enumeraciones y otros elementos relacionados. Esto ayuda a mantener una base de código limpia y organizada, especialmente a medida que tu proyecto crece.

Conceptos Clave

  • Declaración de Módulos: Cómo declarar un módulo.
  • Archivos de Módulos: Cómo dividir módulos en diferentes archivos.
  • Visibilidad: Cómo controlar la visibilidad de los elementos dentro de un módulo.
  • Uso de Módulos: Cómo usar elementos de otros módulos.

Declaración de Módulos

En Rust, puedes declarar un módulo usando la palabra clave mod. Esto se puede hacer dentro de un solo archivo o en varios archivos.

Ejemplo

mod my_module {
    pub fn say_hello() {
        println!("¡Hola desde my_module!");
    }
}

fn main() {
    my_module::say_hello();
}

En este ejemplo:

  • Declaramos un módulo llamado my_module.
  • Dentro de my_module, definimos una función pública say_hello.
  • En la función main, llamamos a my_module::say_hello.

Archivos de Módulos

Los módulos pueden dividirse en archivos separados para mantener la base de código organizada. El sistema de módulos en Rust te permite crear una estructura de directorios que refleje tu jerarquía de módulos.

Ejemplo

Asume la siguiente estructura de directorios:

src/
├── main.rs
└── my_module/
    └── mod.rs

En main.rs:

mod my_module;

fn main() {
    my_module::say_hello();
}

En my_module/mod.rs:

pub fn say_hello() {
    println!("¡Hola desde my_module!");
}

Aquí:

  • mod my_module; en main.rs le dice a Rust que busque el módulo en my_module/mod.rs.
  • La función say_hello se define en my_module/mod.rs.

Visibilidad

Por defecto, todos los elementos en un módulo son privados. Puedes controlar la visibilidad usando la palabra clave pub.

Ejemplo

mod my_module {
    pub fn public_function() {
        println!("Esta es una función pública.");
    }

    fn private_function() {
        println!("Esta es una función privada.");
    }
}

fn main() {
    my_module::public_function();
    // my_module::private_function(); // Esto causará un error en tiempo de compilación
}

En este ejemplo:

  • public_function es accesible fuera del módulo porque se declara con pub.
  • private_function no es accesible fuera del módulo.

Uso de Módulos

Para usar elementos de otros módulos, puedes usar la palabra clave use. Esto puede hacer que tu código sea más limpio y fácil de leer.

Ejemplo

mod my_module {
    pub fn say_hello() {
        println!("¡Hola desde my_module!");
    }
}

use my_module::say_hello;

fn main() {
    say_hello();
}

En este ejemplo:

  • Usamos use my_module::say_hello; para traer say_hello al ámbito.
  • Luego podemos llamar a say_hello directamente en la función main.

Conclusión

Los módulos en Rust son una característica poderosa que te ayuda a organizar tu código en piezas manejables. Al entender cómo declarar módulos, dividirlos en archivos, controlar la visibilidad y usar elementos de otros módulos, puedes escribir código Rust más limpio y mantenible. A medida que progreses, encontrarás que los módulos son esenciales para construir aplicaciones más grandes y complejas.
