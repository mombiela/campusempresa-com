Genéricos en Rust
Hilos ►

Los genéricos son una característica poderosa en Rust que te permite escribir código flexible y reutilizable. Al usar genéricos, puedes crear funciones, estructuras, enumeraciones y rasgos que pueden operar sobre muchos tipos diferentes sin sacrificar rendimiento o seguridad.

Conceptos Clave

  • Tipos Genéricos: Tipos de marcador de posición que pueden ser especificados cuando se usa la función, estructura o enumeración.
  • Parámetros de Tipo: Los tipos reales que reemplazan a los tipos genéricos cuando el código es compilado.
  • Rasgos y Límites: Restricciones que especifican qué capacidades debe tener un tipo para ser usado como un tipo genérico.

Funciones Genéricas

Las funciones genéricas te permiten escribir una función que puede operar sobre diferentes tipos sin duplicar código.

Ejemplo

fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list.iter() {
        if item > largest {
            largest = item;
        }
    }
    largest
}

En este ejemplo:

  • T es un tipo genérico.
  • PartialOrd es un rasgo que asegura que el tipo T puede ser comparado usando el operador >.

Explicación

  • fn largest<T: PartialOrd>(list: &[T]) -> &T declara una función llamada largest que toma una rebanada de tipo T y devuelve una referencia a un elemento de tipo T.
  • T: PartialOrd es un límite de rasgo que asegura que los elementos pueden ser comparados.

Estructuras Genéricas

Puedes definir estructuras con tipos genéricos para crear estructuras de datos que puedan almacenar diferentes tipos de datos.

Ejemplo

struct Point<T> {
    x: T,
    y: T,
}

impl<T> Point<T> {
    fn new(x: T, y: T) -> Self {
        Point { x, y }
    }
}

Explicación

  • struct Point<T> define una estructura con un tipo genérico T.
  • impl<T> Point<T> implementa métodos para la estructura Point.
  • fn new(x: T, y: T) -> Self es un método constructor que inicializa un Point con valores de tipo T.

Enumeraciones Genéricas

Las enumeraciones también pueden ser genéricas, permitiéndote definir variantes que pueden contener diferentes tipos.

Ejemplo

enum Option<T> {
    Some(T),
    None,
}

Explicación

  • enum Option<T> define una enumeración con un tipo genérico T.
  • Some(T) y None son las variantes de la enumeración. Some contiene un valor de tipo T, mientras que None no contiene ningún valor.

Rasgos y Límites

Los rasgos definen comportamientos compartidos que los tipos pueden implementar. Puedes usar límites de rasgo para especificar que un tipo genérico debe implementar un rasgo particular.

Ejemplo

fn print_area<T: Area>(shape: &T) {
    println!("El área es {}", shape.area());
}

trait Area {
    fn area(&self) -> f64;
}

struct Circle {
    radius: f64,
}

impl Area for Circle {
    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }
}

Explicación

  • fn print_area<T: Area>(shape: &T) define una función que toma una referencia a un tipo genérico T que implementa el rasgo Area.
  • trait Area define un rasgo con un método area.
  • struct Circle define una estructura que representa un círculo.
  • impl Area for Circle implementa el rasgo Area para la estructura Circle.

Resumen

Los genéricos en Rust te permiten escribir código flexible, reutilizable y seguro en cuanto a tipos. Al usar tipos genéricos, puedes crear funciones, estructuras, enumeraciones y rasgos que funcionen con muchos tipos diferentes mientras mantienes el rendimiento y la seguridad. Entender los genéricos es crucial para escribir código idiomático y eficiente en Rust.
