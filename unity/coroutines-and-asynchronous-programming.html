Corrutinas y Programación Asíncrona en Unity
Objetos Scriptables ►

Introducción

En esta sección, exploraremos las corrutinas y la programación asíncrona en Unity. Estos conceptos son cruciales para gestionar tareas que toman tiempo en completarse, como cargar activos, esperar la entrada del usuario o manejar solicitudes de red. Al final de este tema, comprenderás cómo implementar y usar corrutinas y métodos asíncronos para crear aplicaciones de Unity más eficientes y receptivas.

¿Qué son las Corrutinas?

Las corrutinas son métodos especiales en Unity que te permiten pausar la ejecución y reanudarla más tarde. Esto es particularmente útil para tareas que necesitan ser distribuidas a lo largo de múltiples cuadros.

Conceptos Clave

  • Instrucciones Yield: Comandos que le dicen a la corrutina cuándo pausar y reanudar.
  • IEnumerator: El tipo de retorno de las corrutinas.
  • StartCoroutine(): Método para iniciar una corrutina.
  • StopCoroutine(): Método para detener una corrutina.

Ejemplo Básico de Corrutina

using UnityEngine;

public class CoroutineExample : MonoBehaviour
{
    void Start()
    {
        // Iniciar la corrutina
        StartCoroutine(PrintNumbers());
    }

    IEnumerator PrintNumbers()
    {
        for (int i = 0; i < 5; i++)
        {
            Debug.Log(i);
            // Pausar la ejecución por 1 segundo
            yield return new WaitForSeconds(1);
        }
    }
}

En este ejemplo, la corrutina PrintNumbers imprime números del 0 al 4, pausando por un segundo entre cada impresión.

Instrucciones Yield

Las instrucciones yield se utilizan dentro de las corrutinas para especificar cuándo la corrutina debe pausar y reanudar.

Instrucciones Yield Comunes

  • WaitForSeconds(float seconds): Pausa la corrutina por un número especificado de segundos.
  • WaitForEndOfFrame(): Pausa la corrutina hasta el final del cuadro.
  • WaitForFixedUpdate(): Pausa la corrutina hasta el próximo ciclo de actualización fija.
  • WaitUntil(Func<bool> predicate): Pausa la corrutina hasta que la condición especificada sea verdadera.
  • WaitWhile(Func<bool> predicate): Pausa la corrutina mientras la condición especificada sea verdadera.

Ejemplo de Instrucción Yield

using UnityEngine;

public class YieldExample : MonoBehaviour
{
    void Start()
    {
        StartCoroutine(WaitForCondition());
    }

    IEnumerator WaitForCondition()
    {
        Debug.Log("Esperando 3 segundos...");
        yield return new WaitForSeconds(3);
        Debug.Log("¡3 segundos pasaron!");

        Debug.Log("Esperando hasta que se presione la tecla espacio...");
        yield return new WaitUntil(() => Input.GetKeyDown(KeyCode.Space));
        Debug.Log("¡Tecla espacio presionada!");
    }
}

En este ejemplo, la corrutina espera 3 segundos y luego espera hasta que se presione la tecla espacio.

Programación Asíncrona con Async/Await

Unity también soporta la programación asíncrona usando las palabras clave async y await, lo que puede hacer que tu código sea más limpio y fácil de leer.

Conceptos Clave

  • Métodos Asíncronos: Métodos marcados con la palabra clave async.
  • Palabra Clave Await: Se usa para pausar la ejecución de un método asíncrono hasta que la tarea esperada se complete.
  • Task: Representa una operación asíncrona.

Ejemplo Básico de Async/Await

using System.Threading.Tasks;
using UnityEngine;

public class AsyncExample : MonoBehaviour
{
    async void Start()
    {
        Debug.Log("Iniciando tarea asíncrona...");
        await WaitAndPrint();
        Debug.Log("¡Tarea asíncrona completada!");
    }

    async Task WaitAndPrint()
    {
        await Task.Delay(3000); // Esperar 3 segundos
        Debug.Log("¡3 segundos pasaron en el método asíncrono!");
    }
}

En este ejemplo, el método WaitAndPrint es un método asíncrono que espera 3 segundos antes de imprimir un mensaje.

Comparación: Corrutinas vs Async/Await

| Característica | Corrutinas | Async/Await | |---------------------|-------------------------------------|--------------------------------------| | Sintaxis | Usa IEnumerator y yield | Usa async y await | | Rendimiento | Ligero, sobrecarga mínima | Ligeramente más sobrecarga debido a tareas | | Legibilidad | Puede volverse complejo con yields anidados | Generalmente más legible y mantenible | | Casos de Uso | Bueno para operaciones basadas en cuadros | Bueno para operaciones dependientes de I/O |

Conclusión

Las corrutinas y la programación asíncrona son herramientas poderosas en Unity para manejar tareas que consumen tiempo sin congelar el hilo principal. Las corrutinas son ideales para operaciones basadas en cuadros, mientras que async/await es más adecuado para tareas dependientes de I/O. Comprender ambos enfoques te ayudará a escribir aplicaciones de Unity más eficientes y receptivas. Practica usando estas técnicas en tus proyectos para volverte competente en la gestión de operaciones asíncronas.
