Introducción

Apache Kafka es una plataforma de transmisión distribuida que te permite publicar y suscribirte a flujos de registros. La API del consumidor es una parte crucial de Kafka, permitiendo a las aplicaciones leer registros de los temas de Kafka. Esta sección proporcionará una descripción general de la API del consumidor, sus conceptos clave y cómo usarla de manera efectiva.

Conceptos clave

  • Grupo de consumidores: Un grupo de consumidores que trabajan juntos para consumir registros de un conjunto de temas. Cada consumidor en el grupo lee un subconjunto de las particiones.
  • Desplazamiento (Offset): Un identificador único para cada registro dentro de una partición. Los consumidores usan desplazamientos para llevar un registro de qué registros han sido consumidos.
  • Rebalanceo: El proceso de redistribuir particiones entre los consumidores en un grupo cuando cambia la membresía del grupo.
  • Confirmación (Commit): La acción de guardar el desplazamiento actual de un consumidor, asegurando que el consumidor pueda reanudar desde este punto en caso de una falla.

Configuración de un consumidor de Kafka

Para crear un consumidor de Kafka, necesitas configurar varias propiedades y usar la clase KafkaConsumer. A continuación se muestra un ejemplo básico en Java.

Ejemplo: Consumidor básico de Kafka en Java

import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.apache.kafka.clients.consumer.ConsumerRecords;
import org.apache.kafka.clients.consumer.KafkaConsumer;

import java.time.Duration;
import java.util.Collections;
import java.util.Properties;

public class BasicKafkaConsumer {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("group.id", "test-group");
        props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
        props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

        KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
        consumer.subscribe(Collections.singletonList("test-topic"));

        try {
            while (true) {
                ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
                for (ConsumerRecord<String, String> record : records) {
                    System.out.printf("offset = %d, key = %s, value = %s%n", record.offset(), record.key(), record.value());
                }
            }
        } finally {
            consumer.close();
        }
    }
}

Explicación

  • Configuración de propiedades: El objeto Properties se usa para configurar el consumidor. Las propiedades clave incluyen bootstrap.servers, group.id, key.deserializer y value.deserializer.
  • Inicialización de KafkaConsumer: La clase KafkaConsumer se instancia con las propiedades configuradas.
  • Suscripción: El consumidor se suscribe a un tema usando el método subscribe.
  • Sondeo (Polling): El método poll se usa para obtener registros del tema. El argumento Duration.ofMillis(100) especifica el tiempo de espera para la operación de sondeo.
  • Procesamiento de registros: Los registros se procesan en un bucle, imprimiendo el desplazamiento, la clave y el valor de cada registro.
  • Apagado ordenado: El consumidor se cierra en el bloque finally para asegurar que los recursos se liberen adecuadamente.

Características avanzadas del consumidor

Control manual de desplazamientos

Por defecto, Kafka confirma los desplazamientos automáticamente. Sin embargo, puedes controlar los desplazamientos manualmente para un control más preciso sobre el procesamiento de registros.

consumer.commitSync();

Oyentes de rebalanceo

Puedes implementar un ConsumerRebalanceListener para manejar reasignaciones de particiones durante el rebalanceo.

consumer.subscribe(Collections.singletonList("test-topic"), new ConsumerRebalanceListener() {
    @Override
    public void onPartitionsRevoked(Collection<TopicPartition> partitions) {
        // Manejar revocación de particiones
    }

    @Override
    public void onPartitionsAssigned(Collection<TopicPartition> partitions) {
        // Manejar asignación de particiones
    }
});

Manejo de errores

El manejo adecuado de errores es crucial para aplicaciones de consumidor robustas. Usa bloques try-catch y maneja excepciones específicas como WakeupException.

try {
    while (true) {
        ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
        // Procesar registros
    }
} catch (WakeupException e) {
    // Manejar apagado
} finally {
    consumer.close();
}

Conclusión

La API del consumidor de Kafka es una herramienta poderosa para leer registros de los temas de Kafka. Comprender sus conceptos clave, como los grupos de consumidores, los desplazamientos y el rebalanceo, es esencial para construir aplicaciones de consumidor eficientes y confiables. Al dominar tanto las características básicas como las avanzadas, puedes asegurar que tus consumidores de Kafka sean robustos y de alto rendimiento.
