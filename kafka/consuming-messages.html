Introducción

Apache Kafka es una plataforma de transmisión distribuida que te permite publicar y suscribirte a flujos de registros. En esta sección, nos enfocaremos en consumir mensajes de temas de Kafka. Cubriremos los conceptos básicos de los consumidores de Kafka, cómo configurarlos y técnicas avanzadas para un consumo eficiente de mensajes.

Conceptos Clave

  • Consumidor: Un cliente que lee registros de un tema de Kafka.
  • Grupo de Consumidores: Un grupo de consumidores que trabajan juntos para consumir mensajes de un tema.
  • Desplazamiento: Un identificador único para cada registro dentro de una partición, utilizado para rastrear la posición del consumidor.

Configuración de un Consumidor de Kafka

Para consumir mensajes de Kafka, necesitas configurar un consumidor de Kafka. Esto implica configurar las propiedades del consumidor y escribir código para leer mensajes de un tema de Kafka.

Configuración Básica del Consumidor

Aquí están las propiedades esenciales que necesitas configurar para un consumidor de Kafka:

  • bootstrap.servers: Una lista de pares host/puerto para usar al establecer la conexión inicial con el clúster de Kafka.
  • group.id: Una cadena única que identifica el grupo de consumidores al que pertenece este consumidor.
  • key.deserializer: El nombre de la clase del deserializador para la clave.
  • value.deserializer: El nombre de la clase del deserializador para el valor.

Ejemplo: Configuración Básica del Consumidor en Java

import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.clients.consumer.KafkaConsumer;
import org.apache.kafka.clients.consumer.ConsumerRecords;
import org.apache.kafka.clients.consumer.ConsumerRecord;

import java.util.Properties;
import java.util.Collections;

public class BasicConsumer {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "test-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringDeserializer");
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringDeserializer");

        KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
        consumer.subscribe(Collections.singletonList("test-topic"));

        while (true) {
            ConsumerRecords<String, String> records = consumer.poll(100);
            for (ConsumerRecord<String, String> record : records) {
                System.out.printf("offset = %d, key = %s, value = %s%n", record.offset(), record.key(), record.value());
            }
        }
    }
}

Funciones Avanzadas del Consumidor

Grupos de Consumidores y Paralelismo

  • Grupos de Consumidores: Múltiples consumidores pueden unirse a un grupo de consumidores para leer de un tema en paralelo. Cada partición en el tema se asigna a solo un consumidor dentro de un grupo, asegurando que cada mensaje se procese solo una vez.
  • Rebalanceo: Cuando un nuevo consumidor se une o deja el grupo, Kafka redistribuye automáticamente las particiones entre los consumidores.

Confirmación de Desplazamientos

  • Confirmación Automática de Desplazamientos: Kafka puede confirmar automáticamente los desplazamientos a intervalos regulares.
  • Confirmación Manual de Desplazamientos: Puedes confirmar manualmente los desplazamientos para tener más control sobre el procesamiento de mensajes.

Ejemplo: Confirmación Manual de Desplazamientos

import org.apache.kafka.clients.consumer.CommitFailedException;

// Dentro del bucle while
try {
    consumer.commitSync();
} catch (CommitFailedException e) {
    System.err.printf("Commit failed: %s%n", e.getMessage());
}

Manejo de Errores y Reintentos

  • Manejo de Errores de Deserialización: Implementa deserializadores personalizados para manejar errores de manera elegante.
  • Reintentos: Configura políticas de reintento para errores transitorios.

Ejemplo: Deserializador Personalizado

import org.apache.kafka.common.serialization.Deserializer;

public class CustomDeserializer implements Deserializer<MyObject> {
    @Override
    public MyObject deserialize(String topic, byte[] data) {
        try {
            // Lógica de deserialización personalizada
        } catch (Exception e) {
            // Manejar error de deserialización
        }
        return null;
    }
}

Conclusión

Consumir mensajes de Kafka es un aspecto fundamental de trabajar con esta poderosa plataforma de transmisión. Al comprender los conceptos básicos de los consumidores de Kafka, configurarlos correctamente y aprovechar funciones avanzadas como los grupos de consumidores y la confirmación manual de desplazamientos, puedes construir aplicaciones de consumo de mensajes robustas y eficientes. Siempre considera el manejo de errores y los reintentos para asegurar que tus aplicaciones consumidoras sean resilientes y confiables.
