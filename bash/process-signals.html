Introducción a las Señales de Proceso

En sistemas operativos tipo Unix, las señales son una forma de comunicación entre procesos utilizada para notificar a los procesos que un evento específico ha ocurrido. Las señales pueden ser enviadas por el sistema operativo, otros procesos, o el propio proceso.

  • Señales: Notificaciones enviadas a un proceso para activar acciones predefinidas.
  • Señales Comunes: SIGINT, SIGTERM, SIGKILL, SIGHUP, SIGSTOP, SIGCONT.

Señales Comunes y Sus Usos

Aquí hay una tabla de algunas señales comunes y sus usos típicos:

| Señal | Número | Descripción | Acción Predeterminada | |---------|--------|---------------------------------------|-----------------------| | SIGINT| 2 | Interrupción desde el teclado (Ctrl+C) | Terminar proceso | | SIGTERM| 15 | Señal de terminación | Terminar proceso | | SIGKILL| 9 | Señal de matar | Terminar forzosamente | | SIGHUP| 1 | Colgar detectado en terminal de control | Terminar proceso | | SIGSTOP| 19 | Detener el proceso | Detener proceso | | SIGCONT| 18 | Continuar si está detenido | Continuar proceso |

Envío de Señales

Puedes enviar señales a procesos usando el comando kill. A pesar de su nombre, kill puede enviar cualquier señal, no solo terminar procesos.

Ejemplos:

  • Enviando SIGTERM a un proceso:

    kill -SIGTERM <pid>

    o

    kill -15 <pid>

  • Enviando SIGKILL a un proceso:

    kill -SIGKILL <pid>

    o

    kill -9 <pid>

  • Enviando SIGINT a un proceso:

    kill -SIGINT <pid>

    o

    kill -2 <pid>

Manejo de Señales en Scripts de Bash

Puedes escribir scripts de Bash para manejar señales usando el comando trap. Esto te permite definir acciones personalizadas cuando tu script recibe señales específicas.

Ejemplo:

#!/bin/bash

# Definir un manejador para SIGINT
trap 'echo "Señal SIGINT capturada"; exit' SIGINT

echo "El script está en ejecución. Presiona Ctrl+C para activar SIGINT."

# Bucle infinito para mantener el script en ejecución
while true; do
  sleep 1
done

En este script:

  • El comando trap captura la señal SIGINT (usualmente enviada al presionar Ctrl+C).
  • Cuando se recibe SIGINT, el script imprime un mensaje y sale.

Ejercicios Prácticos

  1. Enviar una señal SIGTERM a un proceso:

    • Inicia un proceso de larga duración (por ejemplo, sleep 1000).
    • Usa el comando ps para encontrar el ID del proceso (PID) del comando sleep.
    • Envía una señal SIGTERM al proceso usando el comando kill.

  2. Manejar SIGINT en un script de Bash:

    • Escribe un script que se ejecute indefinidamente.
    • Usa el comando trap para capturar la señal SIGINT.
    • Imprime un mensaje y sal cuando se reciba SIGINT.

  3. Experimentar con otras señales:

    • Modifica el script para manejar SIGTERM y SIGHUP.
    • Observa el comportamiento cuando estas señales se envían al script.

Conclusión

Entender las señales de proceso es crucial para gestionar y controlar procesos en sistemas operativos tipo Unix. Las señales te permiten manejar interrupciones de manera elegante, terminar procesos y realizar tareas de limpieza. Al dominar el uso de señales y el comando trap en scripts de Bash, puedes crear scripts más robustos y receptivos.
