Uso de Expresiones LOD
Diseñando Tableros ►

Las Expresiones de Nivel de Detalle (LOD) en Tableau te permiten controlar la granularidad de tus datos. Te permiten realizar cálculos y agregaciones complejas a diferentes niveles de detalle, independientemente del nivel de detalle de la vista. Este tema te guiará desde los conceptos básicos hasta el uso avanzado de las Expresiones LOD en Tableau.

Introducción a las Expresiones LOD

  • Definición: Las Expresiones LOD son una característica poderosa en Tableau que te permiten calcular valores a nivel de la fuente de datos y a nivel de la visualización.
  • Tipos de Expresiones LOD:
    • FIXED: Calcula valores usando las dimensiones especificadas, ignorando el nivel de detalle de la vista.
    • INCLUDE: Calcula valores usando las dimensiones especificadas además de cualquier dimensión que esté en la vista.
    • EXCLUDE: Calcula valores usando las dimensiones especificadas, excluyendo ciertas dimensiones de la vista.

Expresiones LOD Básicas

Expresiones LOD FIXED

  • Sintaxis: { FIXED [Dimension1], [Dimension2], ... : Aggregate_Expression }
  • Ejemplo: Calcular las ventas totales por región, independientemente del nivel de detalle de la vista.
{ FIXED [Region] : SUM([Sales]) }
  • Explicación: Esta expresión calcula la suma de ventas para cada región, ignorando cualquier otra dimensión en la vista.

Expresiones LOD INCLUDE

  • Sintaxis: { INCLUDE [Dimension1], [Dimension2], ... : Aggregate_Expression }
  • Ejemplo: Calcular el promedio de ventas por cliente dentro de cada región.
{ INCLUDE [Customer Name] : AVG([Sales]) }
  • Explicación: Esta expresión calcula el promedio de ventas por cliente, pero incluye la dimensión del cliente además de las dimensiones ya presentes en la vista.

Expresiones LOD EXCLUDE

  • Sintaxis: { EXCLUDE [Dimension1], [Dimension2], ... : Aggregate_Expression }
  • Ejemplo: Calcular las ventas totales excluyendo la categoría de producto.
{ EXCLUDE [Category] : SUM([Sales]) }
  • Explicación: Esta expresión calcula la suma de ventas mientras excluye la dimensión de categoría de producto del cálculo.

Expresiones LOD Intermedias

Combinando Expresiones LOD

  • Ejemplo: Calcular la diferencia entre las ventas totales por región y el promedio de ventas por región.
{ FIXED [Region] : SUM([Sales]) } - { INCLUDE [Region] : AVG([Sales]) }
  • Explicación: Esta expresión primero calcula las ventas totales por región y luego resta el promedio de ventas por región, combinando tanto las expresiones LOD FIXED como INCLUDE.

Expresiones LOD Anidadas

  • Ejemplo: Calcular las ventas máximas por cliente dentro de cada región.
{ FIXED [Region] : MAX({ INCLUDE [Customer Name] : SUM([Sales]) }) }
  • Explicación: Esta expresión LOD anidada primero calcula la suma de ventas por cliente y luego encuentra las ventas máximas por cliente dentro de cada región.

Expresiones LOD Avanzadas

Uso de Expresiones LOD con Filtros

  • Ejemplo: Calcular las ventas totales por región, excluyendo ciertas categorías.
{ FIXED [Region] : SUM(IF [Category] != 'Furniture' THEN [Sales] END) }
  • Explicación: Esta expresión calcula la suma de ventas por región, excluyendo las ventas de la categoría 'Furniture'.

Expresiones LOD Dinámicas

  • Ejemplo: Calcular las ventas totales por región, ajustándose dinámicamente según las dimensiones seleccionadas por el usuario.
{ FIXED [Region], [User Selected Dimension] : SUM([Sales]) }
  • Explicación: Esta expresión permite a los usuarios seleccionar dinámicamente las dimensiones que se incluirán en el cálculo LOD.

Conclusión

Las Expresiones LOD en Tableau son una herramienta versátil que puede ayudarte a realizar cálculos y agregaciones complejas a diferentes niveles de detalle. Al dominar las expresiones LOD FIXED, INCLUDE y EXCLUDE, puedes obtener una comprensión más profunda de tus datos y crear visualizaciones más sofisticadas. Practica usando estas expresiones con varios conjuntos de datos para comprender completamente su comportamiento y aplicaciones.
