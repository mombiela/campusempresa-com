SharedPreferences en Android Studio
Almacenamiento Interno ►

SharedPreferences es una herramienta poderosa en el desarrollo de Android, que te permite almacenar y recuperar pequeñas cantidades de datos primitivos como pares clave-valor. Este tema te guiará desde los conceptos básicos hasta el uso avanzado de SharedPreferences en Android Studio.

Introducción a SharedPreferences

SharedPreferences se utiliza para:

  • Almacenar tipos de datos simples (boolean, float, int, long, string).
  • Persistir preferencias y configuraciones del usuario.
  • Cachear pequeñas cantidades de datos.

Conceptos Clave

  • SharedPreferences: Interfaz para acceder y modificar datos de preferencias.
  • Editor: Interfaz utilizada para modificar valores en un objeto SharedPreferences.

Uso Básico de SharedPreferences

Creación y Acceso a SharedPreferences

Para crear o acceder a SharedPreferences, utiliza el método getSharedPreferences().

// Creación o acceso a SharedPreferences
SharedPreferences sharedPreferences = getSharedPreferences("MyPrefs", MODE_PRIVATE);

Almacenamiento de Datos

Para almacenar datos, utiliza la interfaz SharedPreferences.Editor.

// Almacenamiento de datos en SharedPreferences
SharedPreferences.Editor editor = sharedPreferences.edit();
editor.putString("username", "JohnDoe");
editor.putInt("userAge", 25);
editor.apply(); // o editor.commit();

Recuperación de Datos

Para recuperar datos, utiliza el método get apropiado.

// Recuperación de datos de SharedPreferences
String username = sharedPreferences.getString("username", "defaultName");
int userAge = sharedPreferences.getInt("userAge", 0);

Uso Avanzado de SharedPreferences

Eliminación de Datos

Para eliminar un par clave-valor específico, utiliza el método remove().

// Eliminación de datos de SharedPreferences
SharedPreferences.Editor editor = sharedPreferences.edit();
editor.remove("username");
editor.apply();

Borrado de Todos los Datos

Para borrar todos los datos en SharedPreferences, utiliza el método clear().

// Borrado de todos los datos de SharedPreferences
SharedPreferences.Editor editor = sharedPreferences.edit();
editor.clear();
editor.apply();

Uso de SharedPreferences con Objetos Personalizados

SharedPreferences no soporta el almacenamiento directo de objetos personalizados. Necesitas serializar el objeto a una cadena JSON.

Ejemplo: Almacenamiento de un Objeto Personalizado

// Objeto personalizado
class User {
    String name;
    int age;

    // Constructor, getters y setters
}

// Convertir objeto a cadena JSON
Gson gson = new Gson();
User user = new User("JohnDoe", 25);
String json = gson.toJson(user);

// Almacenar cadena JSON en SharedPreferences
SharedPreferences.Editor editor = sharedPreferences.edit();
editor.putString("user", json);
editor.apply();

Ejemplo: Recuperación de un Objeto Personalizado

// Recuperar cadena JSON de SharedPreferences
String json = sharedPreferences.getString("user", "");

// Convertir cadena JSON de vuelta a objeto
User user = gson.fromJson(json, User.class);

Mejores Prácticas

  • Usa apply() en lugar de commit(): apply() es asíncrono y no bloquea el hilo principal, mientras que commit() es síncrono.
  • Mantén las claves organizadas: Usa una convención de nombres consistente para las claves para evitar conflictos.
  • Limita el uso de SharedPreferences: Úsalo solo para pequeñas cantidades de datos. Para conjuntos de datos más grandes, considera usar una base de datos.

Conclusión

SharedPreferences es una herramienta simple pero poderosa para almacenar pequeñas cantidades de datos en aplicaciones Android. Al entender su uso básico y avanzado, puedes gestionar eficazmente las preferencias del usuario y otros pequeños requisitos de datos en tu aplicación. Recuerda seguir las mejores prácticas para asegurar un almacenamiento de datos eficiente y organizado.
