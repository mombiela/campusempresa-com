Introducción a los Métodos

Los métodos son bloques de código que realizan una tarea específica y pueden ser llamados cuando sea necesario. Ayudan a organizar el código, hacerlo reutilizable y mejorar la legibilidad.

Conceptos Clave

  • Definición: Un método es una colección de declaraciones agrupadas para realizar una operación.
  • Sintaxis: La sintaxis básica para definir un método en Java es: 
    returnType methodName(parameters) {
    // cuerpo del método
}
  • Tipo de Retorno: Especifica el tipo de valor que el método devuelve. Usa void si el método no devuelve un valor.
  • Nombre del Método: Un identificador único para el método.
  • Parámetros: Entradas para el método, encerradas entre paréntesis. Si no se necesitan entradas, deja los paréntesis vacíos.
  • Cuerpo del Método: El bloque de código que define lo que hace el método.

Ejemplo

public class Main {
    // Método para sumar dos números
    public int add(int a, int b) {
        return a + b;
    }

    public static void main(String[] args) {
        Main obj = new Main();
        int result = obj.add(5, 3);
        System.out.println("Suma: " + result); // Salida: Suma: 8
    }
}

En este ejemplo:

  • add es un método que toma dos enteros como parámetros y devuelve su suma.
  • El método main es el punto de entrada del programa donde se llama al método add.

Tipos de Métodos

Métodos de Instancia

Los métodos de instancia pertenecen a una instancia de una clase. Requieren que se cree un objeto de la clase antes de que puedan ser llamados.

public class Calculator {
    public int multiply(int a, int b) {
        return a * b;
    }

    public static void main(String[] args) {
        Calculator calc = new Calculator();
        int product = calc.multiply(4, 5);
        System.out.println("Producto: " + product); // Salida: Producto: 20
    }
}

Métodos Estáticos

Los métodos estáticos pertenecen a la clase en lugar de a cualquier instancia. Pueden ser llamados sin crear un objeto de la clase.

public class MathUtils {
    public static int square(int num) {
        return num * num;
    }

    public static void main(String[] args) {
        int result = MathUtils.square(6);
        System.out.println("Cuadrado: " + result); // Salida: Cuadrado: 36
    }
}

Sobrecarga de Métodos

La sobrecarga de métodos permite que una clase tenga más de un método con el mismo nombre, siempre que sus listas de parámetros sean diferentes.

public class Printer {
    public void print(String s) {
        System.out.println(s);
    }

    public void print(int i) {
        System.out.println(i);
    }

    public static void main(String[] args) {
        Printer printer = new Printer();
        printer.print("¡Hola, Mundo!"); // Salida: ¡Hola, Mundo!
        printer.print(100);             // Salida: 100
    }
}

Conceptos Avanzados de Métodos

Recursión

La recursión es una técnica donde un método se llama a sí mismo para resolver un problema.

public class Factorial {
    public int factorial(int n) {
        if (n == 0) {
            return 1;
        } else {
            return n * factorial(n - 1);
        }
    }

    public static void main(String[] args) {
        Factorial fact = new Factorial();
        int result = fact.factorial(5);
        System.out.println("Factorial: " + result); // Salida: Factorial: 120
    }
}

Paso por Valor

Java usa paso por valor para pasar argumentos a los métodos. Esto significa que se pasa una copia del argumento al método.

public class PassByValue {
    public void modifyValue(int num) {
        num = 10;
    }

    public static void main(String[] args) {
        int x = 5;
        PassByValue obj = new PassByValue();
        obj.modifyValue(x);
        System.out.println("Valor de x: " + x); // Salida: Valor de x: 5
    }
}

Conclusión

Los métodos son bloques fundamentales en la programación en Java. Ayudan a organizar el código, hacerlo reutilizable y mejorar la legibilidad. Entender los diferentes tipos de métodos, la sobrecarga de métodos, la recursión y el concepto de paso por valor es crucial para escribir programas en Java eficientes y mantenibles. Practica escribiendo y llamando métodos para volverte competente en su uso efectivo.
