Introducción a Clases y Objetos

En Java, las clases y los objetos son conceptos fundamentales que forman la base de la programación orientada a objetos (OOP). Entender estos conceptos es crucial para escribir código eficiente y organizado.

Conceptos Clave

  • Clase: Un plano para crear objetos. Define un tipo de dato agrupando datos y métodos que trabajan sobre los datos en una sola unidad.
  • Objeto: Una instancia de una clase. Se crea a partir de una clase y puede usar los métodos definidos en la clase.

Definiendo una Clase

Una clase en Java se define usando la palabra clave class seguida del nombre de la clase y un par de llaves {}.

public class Car {
    // Campos (o variables de instancia)
    String color;
    String model;
    int year;

    // Métodos
    void displayDetails() {
        System.out.println("Model: " + model);
        System.out.println("Color: " + color);
        System.out.println("Year: " + year);
    }
}

Explicación

  • Campos: Variables que mantienen el estado del objeto.
  • Métodos: Funciones que definen el comportamiento del objeto.

Creando Objetos

Los objetos se crean usando la palabra clave new seguida del constructor de la clase.

public class Main {
    public static void main(String[] args) {
        // Creando un objeto de la clase Car
        Car myCar = new Car();
        
        // Estableciendo valores de los campos
        myCar.color = "Red";
        myCar.model = "Toyota";
        myCar.year = 2021;
        
        // Llamando a un método
        myCar.displayDetails();
    }
}

Explicación

  • Instanciación: Car myCar = new Car(); crea un nuevo objeto de la clase Car.
  • Accediendo a Campos y Métodos: Usa el operador punto . para acceder a los campos y métodos del objeto.

Constructores

Los constructores son métodos especiales que se llaman cuando un objeto es instanciado. Inicializan el objeto.

public class Car {
    String color;
    String model;
    int year;

    // Constructor
    public Car(String color, String model, int year) {
        this.color = color;
        this.model = model;
        this.year = year;
    }

    void displayDetails() {
        System.out.println("Model: " + model);
        System.out.println("Color: " + color);
        System.out.println("Year: " + year);
    }
}

Usando Constructores

public class Main {
    public static void main(String[] args) {
        // Creando un objeto usando el constructor
        Car myCar = new Car("Red", "Toyota", 2021);
        
        // Llamando a un método
        myCar.displayDetails();
    }
}

Explicación

  • Definición del Constructor: public Car(String color, String model, int year) inicializa los campos del objeto Car.
  • Creación de Objeto con Constructor: Car myCar = new Car("Red", "Toyota", 2021); inicializa el objeto con valores especificados.

Encapsulación

La encapsulación es el concepto de envolver datos (campos) y métodos en una sola unidad (clase) y restringir el acceso a algunos de los componentes del objeto.

Ejemplo con Encapsulación

public class Car {
    private String color;
    private String model;
    private int year;

    // Constructor
    public Car(String color, String model, int year) {
        this.color = color;
        this.model = model;
        this.year = year;
    }

    // Métodos Getter y Setter
    public String getColor() {
        return color;
    }

    public void setColor(String color) {
        this.color = color;
    }

    public String getModel() {
        return model;
    }

    public void setModel(String model) {
        this.model = model;
    }

    public int getYear() {
        return year;
    }

    public void setYear(int year) {
        this.year = year;
    }

    void displayDetails() {
        System.out.println("Model: " + model);
        System.out.println("Color: " + color);
        System.out.println("Year: " + year);
    }
}

Explicación

  • Campos Privados: Los campos se declaran como private para restringir el acceso directo.
  • Métodos Getter y Setter: Métodos públicos para acceder y actualizar los campos privados.

Herencia

La herencia es un mecanismo donde una clase adquiere las propiedades (campos) y comportamientos (métodos) de otra clase.

Ejemplo de Herencia

// Clase base
public class Vehicle {
    String brand;

    public void honk() {
        System.out.println("Beep! Beep!");
    }
}

// Clase derivada
public class Car extends Vehicle {
    String model;

    public void displayDetails() {
        System.out.println("Brand: " + brand);
        System.out.println("Model: " + model);
    }
}

Usando Herencia

public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.brand = "Toyota";
        myCar.model = "Corolla";
        
        myCar.honk(); // Método de la clase Vehicle
        myCar.displayDetails(); // Método de la clase Car
    }
}

Explicación

  • Clase Base (Vehicle): Contiene propiedades y métodos comunes.
  • Clase Derivada (Car): Hereda propiedades y métodos de la clase base y puede tener propiedades y métodos adicionales.

Polimorfismo

El polimorfismo permite que los métodos hagan cosas diferentes basadas en el objeto sobre el que están actuando, aunque compartan el mismo nombre.

Ejemplo de Polimorfismo

public class Animal {
    public void sound() {
        System.out.println("Animal makes a sound");
    }
}

public class Dog extends Animal {
    public void sound() {
        System.out.println("Dog barks");
    }
}

public class Cat extends Animal {
    public void sound() {
        System.out.println("Cat meows");
    }
}

Usando Polimorfismo

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Animal();
        Animal myDog = new Dog();
        Animal myCat = new Cat();
        
        myAnimal.sound();
        myDog.sound();
        myCat.sound();
    }
}

Explicación

  • Sobrescritura de Métodos: Las clases derivadas (Dog y Cat) proporcionan implementaciones específicas del método sound.
  • Comportamiento Polimórfico: El método sound se comporta de manera diferente según el tipo de objeto.

Resumen

  • Clases y Objetos: Conceptos centrales de OOP en Java.
  • Definiendo Clases: Usa la palabra clave class.
  • Creando Objetos: Usa la palabra clave new.
  • Constructores: Métodos especiales para inicializar objetos.
  • Encapsulación: Restringe el acceso a componentes del objeto usando campos privados y métodos getter/setter públicos.
  • Herencia: Mecanismo para adquirir propiedades y comportamientos de otra clase.
  • Polimorfismo: Permite que los métodos realicen diferentes tareas según el objeto sobre el que están actuando.

Entender estos conceptos te ayudará a escribir código más organizado, reutilizable y mantenible en Java.
