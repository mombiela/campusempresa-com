Herencia en Java
Polimorfismo ►

La herencia es un concepto fundamental en la programación orientada a objetos (OOP) que permite a una nueva clase heredar propiedades y comportamientos (métodos) de una clase existente. Esto promueve la reutilización del código y establece una jerarquía natural entre las clases.

Conceptos Clave

  • Superclase (Clase Padre): La clase cuyas propiedades y métodos son heredados.
  • Subclase (Clase Hija): La clase que hereda de la superclase.
  • Palabra clave extends: Se usa para indicar que una clase está heredando de otra clase.
  • Sobrescritura de Métodos: Una subclase puede proporcionar una implementación específica de un método que ya está definido en su superclase.
  • Palabra clave super: Se usa para referirse al objeto de la clase padre inmediata.

Ejemplo Básico de Herencia

Comencemos con un ejemplo simple para entender cómo funciona la herencia en Java.

// Superclase
class Animal {
    void eat() {
        System.out.println("Este animal come comida.");
    }
}

// Subclase
class Dog extends Animal {
    void bark() {
        System.out.println("El perro ladra.");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.eat();  // Método heredado
        myDog.bark(); // Método específico de la subclase
    }
}

Explicación

  • Superclase (Animal): Contiene un método eat().
  • Subclase (Dog): Hereda el método eat() de Animal y también tiene su propio método bark().
  • Clase Principal: Crea una instancia de Dog y llama a los métodos eat() y bark().

Sobrescritura de Métodos

La sobrescritura de métodos permite a una subclase proporcionar una implementación específica para un método que ya está definido en su superclase.

class Animal {
    void sound() {
        System.out.println("Este animal hace un sonido.");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("El perro ladra.");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Animal();
        myAnimal.sound(); // Salida: Este animal hace un sonido.

        Dog myDog = new Dog();
        myDog.sound(); // Salida: El perro ladra.
    }
}

Explicación

  • Sobrescritura de Métodos: El método sound() en Dog sobrescribe el método sound() en Animal.
  • Anotación @Override: Indica que un método está sobrescribiendo un método en su superclase.

Uso de la Palabra Clave `super`

La palabra clave super se usa para acceder a métodos y constructores de la superclase.

class Animal {
    void sound() {
        System.out.println("Este animal hace un sonido.");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        super.sound(); // Llama al método de la superclase
        System.out.println("El perro ladra.");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.sound(); // Salida: Este animal hace un sonido. El perro ladra.
    }
}

Explicación

  • super.sound(): Llama al método sound() de la clase Animal antes de ejecutar el método sound() en la clase Dog.

Herencia y Constructores

Los constructores no se heredan, pero un constructor de una subclase puede llamar a un constructor de la superclase usando la palabra clave super.

class Animal {
    Animal() {
        System.out.println("Animal es creado.");
    }
}

class Dog extends Animal {
    Dog() {
        super(); // Llama al constructor de la superclase
        System.out.println("Perro es creado.");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog(); // Salida: Animal es creado. Perro es creado.
    }
}

Explicación

  • Constructor de la Superclase: El constructor de Animal se llama primero.
  • Constructor de la Subclase: El constructor de Dog se llama después del constructor de la superclase.

Resumen

  • La herencia permite que una clase herede propiedades y métodos de otra clase, promoviendo la reutilización del código.
  • La sobrescritura de métodos permite que una subclase proporcione una implementación específica de un método ya definido en su superclase.
  • La palabra clave super se usa para referirse al objeto de la clase padre inmediata, permitiendo el acceso a métodos y constructores de la superclase.
  • Los constructores no se heredan pero pueden ser llamados desde una subclase usando la palabra clave super.

Al entender y utilizar la herencia, puedes crear una base de código más organizada y modular, haciendo que tus programas en Java sean más eficientes y fáciles de mantener.
