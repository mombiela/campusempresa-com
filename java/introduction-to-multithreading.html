Introducción a la Multihilación
Ciclo de Vida de un Hilo ►

La multihilación es una característica poderosa en Java que permite la ejecución concurrente de dos o más hilos para la máxima utilización de la CPU. Este tema te guiará a través de los conceptos básicos de la multihilación, sus beneficios y cómo implementarla en Java, progresando desde conceptos para principiantes hasta avanzados.

¿Qué es la Multihilación?

  • Definición: La multihilación es la capacidad de una CPU, o un solo núcleo en un procesador multinúcleo, para proporcionar múltiples hilos de ejecución concurrentemente, soportados por el sistema operativo.
  • Hilo: Un hilo es un proceso liviano, y la multihilación es el proceso de ejecutar múltiples hilos simultáneamente.

Beneficios de la Multihilación

  • Mejora del Rendimiento: Al permitir que múltiples hilos se ejecuten concurrentemente, los programas pueden realizar múltiples operaciones al mismo tiempo.
  • Compartición de Recursos: Los hilos dentro del mismo proceso comparten recursos como la memoria, lo que puede llevar a un uso más eficiente de los recursos.
  • Responsividad: La multihilación puede hacer que las aplicaciones sean más responsivas, especialmente en interfaces de usuario donde un hilo puede manejar la entrada del usuario mientras otro realiza tareas en segundo plano.

Creación de Hilos en Java

Extender la Clase Thread

Una forma de crear un hilo en Java es extendiendo la clase Thread.

class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running.");
    }

    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        t1.start(); // Start the thread
    }
}
  • Explicación:
    • La clase MyThread extiende la clase Thread.
    • El método run contiene el código que constituye el nuevo hilo.
    • Se llama al método start para comenzar la ejecución del hilo.

Implementar la Interfaz Runnable

Otra forma de crear un hilo es implementando la interfaz Runnable.

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Thread is running.");
    }

    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        Thread t1 = new Thread(myRunnable);
        t1.start(); // Start the thread
    }
}
  • Explicación:
    • La clase MyRunnable implementa la interfaz Runnable.
    • El método run contiene el código que constituye el nuevo hilo.
    • Se crea un objeto Thread con MyRunnable como su objetivo, y se llama al método start para comenzar la ejecución.

Ciclo de Vida del Hilo

Un hilo en Java puede estar en uno de los siguientes estados:

  • Nuevo: Un hilo que ha sido creado pero aún no ha comenzado.
  • Runnable: Un hilo que está listo para ejecutarse y está esperando tiempo de CPU.
  • Bloqueado: Un hilo que está bloqueado esperando un bloqueo de monitor.
  • Esperando: Un hilo que está esperando indefinidamente a que otro hilo realice una acción particular.
  • Esperando con Tiempo: Un hilo que está esperando a que otro hilo realice una acción hasta un tiempo de espera especificado.
  • Terminado: Un hilo que ha salido.

Sincronización

Cuando múltiples hilos acceden a recursos compartidos, pueden surgir problemas como la inconsistencia de datos. La sincronización se utiliza para controlar el acceso de múltiples hilos a recursos compartidos.

Método Sincronizado

class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public int getCount() {
        return count;
    }
}

class MyThread extends Thread {
    Counter counter;

    MyThread(Counter counter) {
        this.counter = counter;
    }

    public void run() {
        for (int i = 0; i < 1000; i++) {
            counter.increment();
        }
    }

    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();
        MyThread t1 = new MyThread(counter);
        MyThread t2 = new MyThread(counter);
        t1.start();
        t2.start();
        t1.join();
        t2.join();
        System.out.println("Count: " + counter.getCount());
    }
}
  • Explicación:
    • El método increment está sincronizado, asegurando que solo un hilo pueda ejecutarlo a la vez.
    • Dos hilos (t1 y t2) incrementan el contador concurrentemente.
    • El método join asegura que el hilo principal espere a que t1 y t2 terminen antes de imprimir el conteo.

Conceptos Avanzados

Pools de Hilos

Los pools de hilos gestionan un pool de hilos de trabajo, reduciendo la sobrecarga de crear y destruir hilos.

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Thread is running.");
    }

    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(5);
        for (int i = 0; i < 10; i++) {
            executor.execute(new MyRunnable());
        }
        executor.shutdown();
    }
}
  • Explicación:
    • ExecutorService se utiliza para crear un pool de hilos.
    • Executors.newFixedThreadPool(5) crea un pool de 5 hilos.
    • El método execute se utiliza para enviar tareas para su ejecución.

Callable y Future

Callable es similar a Runnable pero puede devolver un resultado y lanzar una excepción verificada.

import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.Future;

class MyCallable implements Callable<Integer> {
    public Integer call() throws Exception {
        return 123;
    }

    public static void main(String[] args) throws InterruptedException, ExecutionException {
        ExecutorService executor = Executors.newFixedThreadPool(5);
        Future<Integer> future = executor.submit(new MyCallable());
        System.out.println("Result: " + future.get());
        executor.shutdown();
    }
}
  • Explicación:
    • MyCallable implementa Callable y devuelve un entero.
    • El método submit se utiliza para enviar una tarea Callable.
    • El método get de Future se utiliza para recuperar el resultado.

Conclusión

La multihilación en Java es una característica compleja pero poderosa que puede mejorar enormemente el rendimiento y la responsividad de las aplicaciones. Al comprender los conceptos básicos de la creación de hilos, la sincronización y conceptos avanzados como los pools de hilos y Callable, puedes utilizar eficazmente la multihilación en tus programas Java. Practica con los ejemplos y ejercicios proporcionados para solidificar tu comprensión y convertirte en un experto en multihilación.
