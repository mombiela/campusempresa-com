Introducción a NIO

Java NIO (New I/O) es una API de E/S alternativa para Java que ofrece una forma diferente de trabajar con E/S en comparación con el paquete de E/S estándar (java.io). Fue introducida en Java 1.4 y proporciona una manera más eficiente y escalable de manejar operaciones de E/S.

Conceptos Clave de NIO

  • Buffers: Contenedores para datos que pueden ser leídos o escritos.
  • Channels: Una nueva abstracción de E/S primitiva que soporta la lectura y escritura en buffers.
  • Selectors: Permiten que un solo hilo gestione múltiples canales para operaciones de E/S no bloqueantes.

Buffers

Los buffers son fundamentales para NIO. Se utilizan para contener datos que se transfieren entre un canal y una aplicación.

  • Tipos de Buffers: ByteBuffer, CharBuffer, ShortBuffer, IntBuffer, LongBuffer, FloatBuffer, DoubleBuffer.
  • Operaciones de Buffer:
    • put(): Escribe datos en el buffer.
    • get(): Lee datos del buffer.
    • flip(): Prepara el buffer para la lectura después de escribir.
    • clear(): Limpia el buffer para escribir.
    • rewind(): Rebobina el buffer para volver a leer los datos.

Ejemplo de Uso de Buffers

import java.nio.ByteBuffer;

public class BufferExample {
    public static void main(String[] args) {
        ByteBuffer buffer = ByteBuffer.allocate(10);

        // Escribiendo datos en el buffer
        for (int i = 0; i < buffer.capacity(); i++) {
            buffer.put((byte) i);
        }

        // Volteando el buffer para preparar la lectura
        buffer.flip();

        // Leyendo datos del buffer
        while (buffer.hasRemaining()) {
            System.out.print(buffer.get() + " ");
        }
    }
}

Channels

Los canales se utilizan para la transferencia de datos entre buffers y entidades de E/S como archivos o sockets.

  • Tipos de Canales: FileChannel, DatagramChannel, SocketChannel, ServerSocketChannel.
  • Operaciones de Canal:
    • read(): Lee datos del canal en un buffer.
    • write(): Escribe datos de un buffer en el canal.
    • close(): Cierra el canal.

Ejemplo de Uso de Canales

import java.io.RandomAccessFile;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;

public class ChannelExample {
    public static void main(String[] args) throws Exception {
        RandomAccessFile file = new RandomAccessFile("example.txt", "rw");
        FileChannel channel = file.getChannel();

        ByteBuffer buffer = ByteBuffer.allocate(48);
        int bytesRead = channel.read(buffer);

        while (bytesRead != -1) {
            buffer.flip();
            while (buffer.hasRemaining()) {
                System.out.print((char) buffer.get());
            }
            buffer.clear();
            bytesRead = channel.read(buffer);
        }
        channel.close();
        file.close();
    }
}

Selectors

Los selectores permiten que un solo hilo gestione múltiples canales, lo cual es útil para operaciones de E/S no bloqueantes.

  • Operaciones de Selector:
    • select(): Bloquea hasta que al menos un canal esté listo para las operaciones que te interesan.
    • selectedKeys(): Devuelve el conjunto de claves que representan los canales que están listos para operaciones de E/S.
    • register(): Registra un canal con el selector.

Ejemplo de Uso de Selectores

import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.SelectionKey;
import java.nio.channels.Selector;
import java.nio.channels.ServerSocketChannel;
import java.nio.channels.SocketChannel;
import java.util.Iterator;

public class SelectorExample {
    public static void main(String[] args) throws IOException {
        Selector selector = Selector.open();
        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.bind(new InetSocketAddress("localhost", 8080));
        serverChannel.configureBlocking(false);
        serverChannel.register(selector, SelectionKey.OP_ACCEPT);

        while (true) {
            selector.select();
            Iterator<SelectionKey> keys = selector.selectedKeys().iterator();

            while (keys.hasNext()) {
                SelectionKey key = keys.next();
                keys.remove();

                if (key.isAcceptable()) {
                    ServerSocketChannel server = (ServerSocketChannel) key.channel();
                    SocketChannel client = server.accept();
                    client.configureBlocking(false);
                    client.register(selector, SelectionKey.OP_READ);
                } else if (key.isReadable()) {
                    SocketChannel client = (SocketChannel) key.channel();
                    ByteBuffer buffer = ByteBuffer.allocate(256);
                    client.read(buffer);
                    buffer.flip();
                    System.out.println(new String(buffer.array()).trim());
                    buffer.clear();
                }
            }
        }
    }
}

Conclusión

Java NIO proporciona una forma poderosa y flexible de manejar operaciones de E/S, ofreciendo mejor rendimiento y escalabilidad en comparación con la E/S tradicional. Entender buffers, canales y selectores es crucial para aprovechar todo el potencial de NIO. Al dominar estos conceptos, puedes construir aplicaciones eficientes y receptivas que manejen operaciones de E/S de manera efectiva.
