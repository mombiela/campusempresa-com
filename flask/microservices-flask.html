Microservicios con Flask

Introducción a los Microservicios

  • Definición: Los microservicios son un estilo arquitectónico que estructura una aplicación como una colección de servicios acoplados de manera flexible.
  • Beneficios:
    • Escalabilidad
    • Flexibilidad
    • Despliegue independiente
    • Mejora en el aislamiento de fallos

Configuración de Flask para Microservicios

  • Instalación: 
    pip install Flask
  • Aplicación básica de Flask: 
    from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "¡Hola, Microservicios!"

if __name__ == '__main__':
    app.run(debug=True)

Diseño de Microservicios

  • Límites del Servicio: Definir límites claros para cada microservicio.
  • Comunicación: Usar APIs RESTful para la comunicación entre servicios.
  • Gestión de Datos: Cada microservicio debe gestionar su propia base de datos.

Construcción de un Microservicio Simple

  • Creación de un Servicio de Usuarios: 
    from flask import Flask, jsonify, request

app = Flask(__name__)

users = []

@app.route('/users', methods=['GET'])
def get_users():
    return jsonify(users)

@app.route('/users', methods=['POST'])
def add_user():
    user = request.json
    users.append(user)
    return jsonify(user), 201

if __name__ == '__main__':
    app.run(debug=True)
  • Explicación:
    • @app.route('/users', methods=['GET']): Define una ruta para obtener todos los usuarios.
    • @app.route('/users', methods=['POST']): Define una ruta para agregar un nuevo usuario.
    • request.json: Recupera datos JSON del cuerpo de la solicitud.

Comunicación entre Servicios

  • APIs RESTful: Usar métodos HTTP (GET, POST, PUT, DELETE) para la comunicación.
  • Ejemplo: 
    import requests

response = requests.get('http://user-service/users')
users = response.json()
print(users)

Manejo de Errores y Registro

  • Manejo de Errores: 
    @app.errorhandler(404)
def not_found(error):
    return jsonify({'error': 'No encontrado'}), 404
  • Registro: 
    import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

@app.route('/users', methods=['POST'])
def add_user():
    user = request.json
    users.append(user)
    logger.info(f'Usuario agregado: {user}')
    return jsonify(user), 201

Pruebas de Microservicios

  • Pruebas Unitarias: 
    import unittest

class UserServiceTest(unittest.TestCase):
    def test_add_user(self):
        with app.test_client() as client:
            response = client.post('/users', json={'name': 'John'})
            self.assertEqual(response.status_code, 201)
            self.assertEqual(response.json, {'name': 'John'})

if __name__ == '__main__':
    unittest.main()
  • Pruebas de Integración: Probar la interacción entre múltiples servicios.

Estrategias de Despliegue

  • Docker: Contenerizar tus microservicios. 
    FROM python:3.8-slim

WORKDIR /app

COPY requirements.txt requirements.txt
RUN pip install -r requirements.txt

COPY . .

CMD ["python", "app.py"]
  • Kubernetes: Orquestar tus microservicios.

Conclusión

Los microservicios con Flask permiten servicios escalables, flexibles y desplegables de manera independiente. Al comprender los conceptos básicos de Flask, diseñar límites claros de servicio, manejar la comunicación entre servicios e implementar un manejo adecuado de errores y registros, puedes construir microservicios robustos. Las estrategias de prueba y despliegue aseguran aún más la fiabilidad y escalabilidad de tus aplicaciones.
