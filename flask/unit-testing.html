Pruebas Unitarias con Flask
Pruebas de Integración ►

Las pruebas unitarias son una parte crucial del desarrollo de aplicaciones web confiables y mantenibles. En esta sección, cubriremos cómo escribir pruebas unitarias para una aplicación Flask, asegurándonos de que tu código funcione como se espera y pueda manejar varios casos límite.

Introducción a las Pruebas Unitarias

  • Definición: Las pruebas unitarias implican probar componentes o funciones individuales de una aplicación en aislamiento para asegurarse de que funcionen correctamente.
  • Importancia: Ayuda a detectar errores temprano, simplifica la depuración y asegura la confiabilidad del código.
  • Herramientas: Usaremos el módulo unittest incorporado en Python y las utilidades de prueba de Flask.

Configuración del Entorno de Pruebas

Antes de escribir pruebas, necesitamos configurar nuestro entorno de pruebas.

  1. Instalar Dependencias:

    pip install Flask
pip install pytest

  2. Crear una Aplicación Flask:

    # app.py
from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/hello')
def hello():
    return jsonify(message="Hello, World!")

if __name__ == '__main__':
    app.run(debug=True)

  3. Crear un Directorio de Pruebas:

    mkdir tests

Escribiendo tu Primera Prueba

Vamos a escribir una prueba simple para verificar si nuestro endpoint /hello devuelve el mensaje correcto.

  1. Crear un Archivo de Prueba:

    # tests/test_app.py
import unittest
from app import app

class BasicTests(unittest.TestCase):

    def setUp(self):
        self.app = app.test_client()
        self.app.testing = True

    def test_hello(self):
        response = self.app.get('/hello')
        self.assertEqual(response.status_code, 200)
        self.assertEqual(response.json, {"message": "Hello, World!"})

if __name__ == "__main__":
    unittest.main()

  2. Ejecutar la Prueba:

    python -m unittest discover -s tests

Pruebas con Pytest

Aunque unittest es poderoso, muchos desarrolladores prefieren pytest por su simplicidad y características avanzadas.

  1. Convertir la Prueba a Pytest:

    # tests/test_app.py
import pytest
from app import app

@pytest.fixture
def client():
    app.testing = True
    return app.test_client()

def test_hello(client):
    response = client.get('/hello')
    assert response.status_code == 200
    assert response.json == {"message": "Hello, World!"}

  2. Ejecutar la Prueba con Pytest:

    pytest

Pruebas de Casos Límite y Manejo de Errores

Es esencial probar no solo el camino feliz sino también los casos límite y los escenarios de error.

  1. Agregar una Nueva Ruta con Manejo de Errores:

    # app.py
@app.route('/divide')
def divide():
    try:
        dividend = int(request.args.get('dividend'))
        divisor = int(request.args.get('divisor'))
        result = dividend / divisor
        return jsonify(result=result)
    except ZeroDivisionError:
        return jsonify(error="Cannot divide by zero"), 400
    except ValueError:
        return jsonify(error="Invalid input"), 400

  2. Escribir Pruebas para Casos Límite:

    # tests/test_app.py
def test_divide_success(client):
    response = client.get('/divide?dividend=10&divisor=2')
    assert response.status_code == 200
    assert response.json == {"result": 5.0}

def test_divide_by_zero(client):
    response = client.get('/divide?dividend=10&divisor=0')
    assert response.status_code == 400
    assert response.json == {"error": "Cannot divide by zero"}

def test_divide_invalid_input(client):
    response = client.get('/divide?dividend=ten&divisor=2')
    assert response.status_code == 400
    assert response.json == {"error": "Invalid input"}

Simulación y Aislamiento de Dependencias

A veces, necesitas aislar tus pruebas de dependencias externas como bases de datos o APIs.

  1. Usando unittest.mock: 
    from unittest.mock import patch

@patch('app.some_external_function')
def test_mocked_function(mock_function, client):
    mock_function.return_value = "mocked result"
    response = client.get('/some-endpoint')
    assert response.json == {"result": "mocked result"}

Conclusión

Las pruebas unitarias son una práctica esencial para desarrollar aplicaciones Flask robustas. Al configurar un entorno de pruebas adecuado, escribir pruebas para varios escenarios y usar herramientas como unittest y pytest, puedes asegurarte de que tu aplicación se comporte como se espera. Recuerda probar tanto el camino feliz como los casos límite, y usar simulaciones para aislar dependencias cuando sea necesario. ¡Felices pruebas!
