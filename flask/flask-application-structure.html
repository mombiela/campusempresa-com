Introducción

Flask es un micro framework web para Python, diseñado para ser simple pero poderoso. Entender la estructura de una aplicación Flask es crucial para desarrollar aplicaciones web escalables y mantenibles. Esta sección te guiará a través de la estructura típica de una aplicación Flask, explicando cada componente y su propósito.

Conceptos Clave

  • Aplicación Flask: El núcleo de tu aplicación web.
  • Rutas: Patrones de URL que se asignan a funciones específicas en tu aplicación.
  • Plantillas: Archivos HTML que son renderizados por Flask para generar páginas web dinámicas.
  • Archivos Estáticos: Archivos como CSS, JavaScript e imágenes que se sirven directamente al cliente.
  • Configuración: Configuraciones que controlan el comportamiento de tu aplicación Flask.

Estructura Básica de una Aplicación Flask

Una aplicación Flask básica típicamente consiste en los siguientes archivos y directorios:

mi_app_flask/
│
├── app.py
├── templates/
│   └── index.html
├── static/
│   ├── css/
│   ├── js/
│   └── images/
└── config.py

app.py

Este es el archivo principal que inicializa la aplicación Flask y define las rutas.

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def home():
    return render_template('index.html')

if __name__ == '__main__':
    app.run(debug=True)
  • Inicialización de Flask: app = Flask(__name__) inicializa la aplicación Flask.
  • Definición de Ruta: @app.route('/') define una ruta para la página de inicio.
  • Renderizar Plantilla: render_template('index.html') renderiza la plantilla index.html.

templates/index.html

Este es un archivo HTML que será renderizado por Flask.

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Inicio</title>
</head>
<body>
    <h1>¡Bienvenido a Mi App Flask!</h1>
</body>
</html>

static/

Este directorio contiene archivos estáticos como CSS, JavaScript e imágenes. Estos archivos se sirven directamente al cliente.

  • css/: Contiene archivos CSS.
  • js/: Contiene archivos JavaScript.
  • images/: Contiene archivos de imágenes.

config.py

Este archivo contiene configuraciones para tu aplicación Flask.

import os

class Config:
    SECRET_KEY = os.environ.get('SECRET_KEY') or 'you-will-never-guess'
    DEBUG = True

Estructura Avanzada de una Aplicación Flask

A medida que tu aplicación crece, puedes querer organizarla en una estructura más modular. Aquí hay un ejemplo de una estructura más avanzada:

mi_app_flask/
│
├── app/
│   ├── __init__.py
│   ├── routes.py
│   ├── models.py
│   ├── templates/
│   │   └── index.html
│   └── static/
│       ├── css/
│       ├── js/
│       └── images/
├── config.py
└── run.py

app/__init__.py

Este archivo inicializa la aplicación Flask y configura la configuración.

from flask import Flask
from config import Config

app = Flask(__name__)
app.config.from_object(Config)

from app import routes

app/routes.py

Este archivo define las rutas para tu aplicación.

from app import app
from flask import render_template

@app.route('/')
def home():
    return render_template('index.html')

app/models.py

Este archivo contiene los modelos de base de datos para tu aplicación. (Ejemplo con SQLAlchemy)

from app import app
from flask_sqlalchemy import SQLAlchemy

db = SQLAlchemy(app)

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(64), index=True, unique=True)
    email = db.Column(db.String(120), index=True, unique=True)

    def __repr__(self):
        return f'<User {self.username}>'

run.py

Este archivo se usa para ejecutar la aplicación Flask.

from app import app

if __name__ == '__main__':
    app.run(debug=True)

Conclusión

Entender la estructura de una aplicación Flask es esencial para construir aplicaciones web eficientes y mantenibles. Comenzar con una estructura básica y gradualmente moverse a una estructura más modular y avanzada te ayudará a gestionar mejor tu código a medida que tu aplicación crece. Al organizar adecuadamente tus rutas, plantillas, archivos estáticos y configuraciones, puedes crear una aplicación Flask escalable y mantenible.
