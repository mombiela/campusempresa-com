Introducción

La detección de colisiones es un componente esencial en la física de videojuegos. Permite determinar cuándo dos o más objetos en un entorno virtual se tocan o interactúan de alguna manera. Este proceso es crucial para la jugabilidad y la física realista en los videojuegos.

Conceptos Clave

  • Colisión: Ocurre cuando dos objetos ocupan el mismo espacio en un momento dado.
  • Bounding Box (Caja Envolvente): Una caja rectangular que encierra un objeto, utilizada para simplificar la detección de colisiones.
  • Bounding Sphere (Esfera Envolvente): Una esfera que encierra un objeto, otra simplificación para la detección de colisiones.
  • Algoritmo de Separación de Ejes (SAT): Un método para determinar si dos polígonos convexos están colisionando.
  • Broad Phase y Narrow Phase: Dos etapas en la detección de colisiones. La fase amplia (Broad Phase) reduce el número de posibles colisiones, y la fase estrecha (Narrow Phase) realiza una detección más precisa.

Tipos de Colisiones

Colisiones AABB (Axis-Aligned Bounding Box)

Las colisiones AABB son una de las formas más simples y rápidas de detección de colisiones. Se utilizan cajas rectangulares alineadas con los ejes del sistema de coordenadas.

class AABB:
    def __init__(self, x, y, width, height):
        self.x = x
        self.y = y
        self.width = width
        self.height = height

def check_collision(a, b):
    return (a.x < b.x + b.width and
            a.x + a.width > b.x and
            a.y < b.y + b.height and
            a.y + a.height > b.y)

# Ejemplo de uso
a = AABB(0, 0, 10, 10)
b = AABB(5, 5, 10, 10)
print(check_collision(a, b))  # Devuelve True

Colisiones de Esferas

Las colisiones de esferas son útiles cuando los objetos tienen una forma más circular. La detección se basa en la distancia entre los centros de las esferas.

import math

class Sphere:
    def __init__(self, x, y, radius):
        self.x = x
        self.y = y
        self.radius = radius

def check_collision(s1, s2):
    distance = math.sqrt((s1.x - s2.x)**2 + (s1.y - s2.y)**2)
    return distance < (s1.radius + s2.radius)

# Ejemplo de uso
s1 = Sphere(0, 0, 5)
s2 = Sphere(7, 0, 5)
print(check_collision(s1, s2))  # Devuelve True

Algoritmo de Separación de Ejes (SAT)

El SAT es más complejo pero muy efectivo para polígonos convexos. Se basa en proyectar los vértices de los polígonos en un eje y comprobar si hay una separación.

# Ejemplo simplificado de SAT en pseudocódigo
def sat_collision(poly1, poly2):
    for axis in poly1.get_axes() + poly2.get_axes():
        projection1 = poly1.project(axis)
        projection2 = poly2.project(axis)
        if not projection1.overlaps(projection2):
            return False
    return True

# poly1 y poly2 son objetos que tienen métodos get_axes() y project(axis)

Broad Phase y Narrow Phase

Broad Phase

La fase amplia reduce el número de posibles colisiones utilizando métodos como el barrido y poda (Sweep and Prune) o las cuadrículas espaciales.

Narrow Phase

La fase estrecha realiza una detección de colisiones más precisa entre los pares de objetos que pasaron la fase amplia.

Conclusión

La detección de colisiones es fundamental para la física de videojuegos, proporcionando la base para interacciones realistas entre objetos. Desde métodos simples como las colisiones AABB y de esferas hasta técnicas más avanzadas como el SAT, cada método tiene sus ventajas y desventajas. Comprender y aplicar estos conceptos es crucial para cualquier desarrollador de videojuegos.

Este tema proporciona una base sólida para la detección de colisiones en videojuegos. A medida que avances en el curso, exploraremos más técnicas y optimizaciones para mejorar la eficiencia y precisión de la detección de colisiones.
