Introducción

En este ejercicio, aprenderás a configurar un pipeline básico de CI/CD (Integración Continua y Entrega Continua) utilizando una herramienta popular como GitHub Actions. Este ejercicio está diseñado para ayudarte a comprender los conceptos fundamentales y a poner en práctica una configuración básica que puedes expandir en proyectos futuros.

Objetivos del Ejercicio

  • Configurar un repositorio de GitHub para CI/CD.
  • Crear un archivo de configuración de GitHub Actions.
  • Ejecutar pruebas automáticas en cada push al repositorio.
  • Desplegar automáticamente a un entorno de staging.

Requisitos Previos

  • Conocimiento básico de Git y GitHub.
  • Familiaridad con YAML.
  • Un proyecto simple en cualquier lenguaje de programación (usaremos un proyecto en Python para este ejemplo).

Paso 1: Crear un Repositorio en GitHub

  1. Ve a GitHub y crea una nueva cuenta si no tienes una.
  2. Crea un nuevo repositorio:
    • Nombre: ci-cd-demo
    • Descripción: Repositorio de demostración para configuración básica de CI/CD
    • Visibilidad: Pública o privada, según tu preferencia.

Paso 2: Configurar GitHub Actions

  1. Navega a la pestaña "Actions" en tu nuevo repositorio.
  2. Selecciona "Set up a workflow yourself" para crear un archivo de configuración desde cero.
  3. Crea un archivo llamado ci.yml en el directorio .github/workflows/.

Ejemplo de Archivo de Configuración

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.8'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run tests
      run: |
        pytest

Explicación del Archivo

  • name: Nombre del workflow.
  • on: Define el evento que desencadena el workflow (en este caso, un push a la rama main).
  • jobs: Define los trabajos que se ejecutarán.
    • build: Nombre del trabajo.
    • runs-on: Especifica el sistema operativo del runner.
    • steps: Lista de pasos a ejecutar.
      • Checkout code: Usa la acción actions/checkout@v2 para clonar el repositorio.
      • Set up Python: Usa la acción actions/setup-python@v2 para configurar Python.
      • Install dependencies: Instala las dependencias del proyecto.
      • Run tests: Ejecuta las pruebas utilizando pytest.

Paso 3: Ejecutar el Pipeline

  1. Realiza un commit y push de tu archivo ci.yml al repositorio.
  2. Navega a la pestaña "Actions" en tu repositorio para ver el progreso del pipeline.
  3. Verifica que todas las etapas (checkout, setup, install, test) se ejecuten correctamente.

Conclusión

En este ejercicio, has configurado un pipeline básico de CI/CD utilizando GitHub Actions. Has aprendido a:

  • Configurar un repositorio de GitHub para CI/CD.
  • Crear y entender un archivo de configuración de GitHub Actions.
  • Ejecutar pruebas automáticas en cada push al repositorio.

Este es solo el comienzo. A medida que te familiarices con estos conceptos, podrás expandir tu pipeline para incluir despliegues a producción, análisis de código estático, y mucho más.
