Introducción al Balanceo de Carga Elástico (ELB)

El Balanceo de Carga Elástico (ELB) es un servicio proporcionado por AWS que distribuye automáticamente el tráfico de aplicaciones entrante entre múltiples destinos, como instancias EC2, contenedores y direcciones IP. ELB ayuda a garantizar un alto nivel de disponibilidad y tolerancia a fallos para tus aplicaciones.

Conceptos Clave

  • Balanceador de Carga: Un dispositivo que distribuye el tráfico de red o de aplicaciones entre varios servidores.
  • Destino: El destino del tráfico, como instancias EC2, contenedores o direcciones IP.
  • Listener: Un proceso que verifica las solicitudes de conexión utilizando el protocolo y puerto que configures.
  • Grupo de Destinos: Un grupo de destinos a los que un balanceador de carga enruta las solicitudes.

Tipos de Balanceadores de Carga

AWS ofrece tres tipos de balanceadores de carga:

  1. Balanceador de Carga de Aplicaciones (ALB)

    • Mejor para tráfico HTTP y HTTPS.
    • Opera en la capa de aplicación (Capa 7).
    • Soporta enrutamiento avanzado, como enrutamiento basado en rutas y en hosts.

  2. Balanceador de Carga de Red (NLB)

    • Mejor para tráfico TCP, UDP y TLS.
    • Opera en la capa de transporte (Capa 4).
    • Capaz de manejar millones de solicitudes por segundo manteniendo latencias ultra bajas.

  3. Balanceador de Carga Clásico (CLB)

    • Opera tanto en la capa de aplicación (Capa 7) como en la capa de transporte (Capa 4).
    • Adecuado para aplicaciones construidas dentro de la red EC2-Classic.

Configuración de un Balanceador de Carga de Aplicaciones (ALB)

Guía Paso a Paso

  1. Crear un Balanceador de Carga

    • Navega al Panel de EC2.
    • Selecciona "Load Balancers" del menú de la izquierda.
    • Haz clic en "Create Load Balancer" y elige "Application Load Balancer".

  2. Configurar el Balanceador de Carga

    • Proporciona un nombre para tu balanceador de carga.
    • Selecciona el esquema (Expuesto a Internet o Interno).
    • Elige el tipo de dirección IP (IPv4 o Dualstack).
    • Configura los listeners (por defecto es HTTP en el puerto 80).

  3. Configurar la Seguridad

    • Si usas HTTPS, configura los certificados SSL.

  4. Configurar Grupos de Seguridad

    • Selecciona o crea un grupo de seguridad que permita tráfico entrante en los puertos de los listeners.

  5. Configurar el Enrutamiento

    • Crea un grupo de destinos.
    • Elige el tipo de destino (Instancias, direcciones IP o funciones Lambda).
    • Configura las verificaciones de salud (protocolo, ruta y puerto).

  6. Registrar Destinos

    • Añade las instancias o direcciones IP a las que deseas enrutar el tráfico.

  7. Revisar y Crear

    • Revisa tus configuraciones y haz clic en "Create".

Código de Ejemplo: Creando un ALB usando AWS CLI

aws elbv2 create-load-balancer \
    --name my-load-balancer \
    --subnets subnet-12345678 subnet-87654321 \
    --security-groups sg-12345678 \
    --scheme internet-facing \
    --type application

Código de Ejemplo: Registrando Destinos usando AWS CLI

aws elbv2 register-targets \
    --target-group-arn arn:aws:elasticloadbalancing:region:account-id:targetgroup/my-targets/1234567890123456 \
    --targets Id=i-1234567890abcdef0 Id=i-0abcdef1234567890

Características Avanzadas de ELB

Sesiones Persistentes

Las sesiones persistentes (afinidad de sesión) te permiten vincular la sesión de un usuario a un destino específico dentro del grupo de destinos. Esto asegura que todas las solicitudes de un usuario durante una sesión se envíen al mismo destino.

Balanceo de Carga entre Zonas

El balanceo de carga entre zonas distribuye las solicitudes entrantes de manera uniforme entre todos los destinos en todas las Zonas de Disponibilidad habilitadas. Esto mejora la tolerancia a fallos y la disponibilidad de tu aplicación.

Terminación SSL

La terminación SSL descifra las solicitudes SSL entrantes en el balanceador de carga antes de enviarlas a los destinos. Esto reduce la carga en tus destinos y simplifica la gestión de certificados SSL.

Integración con Auto Scaling

ELB se integra perfectamente con Auto Scaling, permitiéndote añadir o eliminar automáticamente destinos según la demanda. Esto asegura que tu aplicación pueda manejar niveles variables de tráfico sin intervención manual.

Monitoreo y Registro

Métricas de CloudWatch

ELB proporciona varias métricas de CloudWatch para monitorear el rendimiento y la salud de tu balanceador de carga, tales como:

  • RequestCount
  • HealthyHostCount
  • UnHealthyHostCount
  • Latency

Registros de Acceso

Los registros de acceso capturan información detallada sobre las solicitudes enviadas a tu balanceador de carga, incluyendo la hora en que se recibió la solicitud, la dirección IP del cliente y la ruta de la solicitud. Estos registros pueden almacenarse en S3 para su análisis.

Código de Ejemplo: Habilitando Registros de Acceso usando AWS CLI

aws elbv2 modify-load-balancer-attributes \
    --load-balancer-arn arn:aws:elasticloadbalancing:region:account-id:loadbalancer/app/my-load-balancer/50dc6c495c0c9188 \
    --attributes Key=access_logs.s3.enabled,Value=true Key=access_logs.s3.bucket,Value=my-logs-bucket Key=access_logs.s3.prefix,Value=my-app

Conclusión

El Balanceo de Carga Elástico (ELB) es un servicio poderoso que mejora la disponibilidad, tolerancia a fallos y escalabilidad de tus aplicaciones. Al entender los diferentes tipos de balanceadores de carga, configurar y configurar un ALB, y aprovechar las características avanzadas, puedes asegurar que tus aplicaciones sean robustas y puedan manejar niveles variables de tráfico de manera eficiente. Las capacidades de monitoreo y registro ayudan además a mantener la salud y el rendimiento de tus aplicaciones.
