Introducción

Los Planes de Ahorro y las Reservas de Instancias (RIs) de AWS son opciones de ahorro de costos proporcionadas por Amazon Web Services (AWS) para ayudar a los usuarios a reducir sus gastos en computación en la nube. Esta sección cubrirá los conceptos básicos de ambas opciones, sus diferencias y cómo elegir la adecuada para tus necesidades.

Conceptos Clave

Planes de Ahorro de AWS

  • Definición: Los Planes de Ahorro son modelos de precios flexibles que ofrecen ahorros significativos en el uso de AWS, a cambio de un compromiso con una cantidad constante de uso (medida en $/hora) por un período de 1 o 3 años.
  • Tipos de Planes de Ahorro:
    • Planes de Ahorro de Cómputo: Proporcionan la mayor flexibilidad y se aplican a cualquier uso de instancias EC2 sin importar la región, familia de instancias, sistema operativo o tenencia.
    • Planes de Ahorro de Instancias EC2: Ofrecen precios más bajos pero son menos flexibles, aplicándose solo a familias de instancias específicas dentro de una región.

Reservas de Instancias (RIs)

  • Definición: Las Reservas de Instancias proporcionan un descuento en comparación con los precios bajo demanda a cambio de comprometerse a usar un tipo específico de instancia en una región específica por un período de 1 o 3 años.
  • Tipos de RIs:
    • RIs Estándar: Ofrecen el mayor descuento pero son menos flexibles. Pueden modificarse para cambiar la Zona de Disponibilidad, el tamaño de la instancia (dentro de la misma familia) y el tipo de red.
    • RIs Convertibles: Ofrecen un descuento menor pero permiten cambiar el tipo de instancia, el sistema operativo y la tenencia.

Comparación de Planes de Ahorro y Reservas de Instancias

| Característica | Planes de Ahorro | Reservas de Instancias | |----------------------------|----------------------------------------|--------------------------------------| | Flexibilidad | Alta (especialmente Planes de Ahorro de Cómputo) | Media (RIs Convertibles) a Baja (RIs Estándar) | | Nivel de Descuento | Moderado a Alto | Alto (RIs Estándar) a Moderado (RIs Convertibles) | | Período de Compromiso | 1 o 3 años | 1 o 3 años | | Alcance | A nivel de cuenta | Específico para tipo de instancia y región | | Aplicabilidad | A través de EC2, Fargate, Lambda | Solo instancias EC2 |

Cómo Elegir entre Planes de Ahorro y Reservas de Instancias

Factores a Considerar

  • Predictibilidad de la Carga de Trabajo: Si tu carga de trabajo es predecible y estable, las RIs pueden ser más beneficiosas debido a sus mayores descuentos.
  • Necesidades de Flexibilidad: Si necesitas más flexibilidad en términos de tipos de instancias y regiones, los Planes de Ahorro, especialmente los Planes de Ahorro de Cómputo, son una mejor opción.
  • Presupuesto y Gestión de Costos: Evalúa tu presupuesto y cuánto estás dispuesto a comprometer por adelantado. Ambas opciones ofrecen ahorros significativos, pero el nivel de compromiso y el pago por adelantado pueden variar.

Escenarios de Ejemplo

  1. Escenario 1: Una startup con una carga de trabajo creciente y cambiante podría preferir los Planes de Ahorro de Cómputo por su flexibilidad.
  2. Escenario 2: Una gran empresa con cargas de trabajo estables y predecibles podría beneficiarse más de las RIs Estándar debido a los mayores descuentos.

Ejemplos Prácticos

Ejemplo 1: Comprar un Plan de Ahorro de Cómputo

import boto3

client = boto3.client('savingsplans')

response = client.create_savings_plan(
    savingsPlanOfferingId='string',  # ID de la oferta del Plan de Ahorro
    commitment='0.1',  # Cantidad de compromiso en $/hora
    upfrontPaymentAmount='0',  # Cantidad de pago por adelantado
    purchaseTime='2023-10-01T00:00:00Z',  # Hora de compra
    tags={
        'Project': 'MyProject'
    }
)

print(response)

Explicación:

  • Este código utiliza el SDK de AWS para Python (Boto3) para comprar un Plan de Ahorro de Cómputo.
  • El parámetro commitment especifica el compromiso por hora en dólares.
  • El parámetro upfrontPaymentAmount especifica cualquier pago por adelantado.

Ejemplo 2: Modificar una Reserva de Instancia

import boto3

client = boto3.client('ec2')

response = client.modify_reserved_instances(
    ReservedInstancesIds=[
        'ri-12345678',  # ID de la Reserva de Instancia
    ],
    TargetConfigurations=[
        {
            'AvailabilityZone': 'us-west-2a',
            'InstanceType': 'm5.large',
            'Platform': 'Linux/UNIX',
        },
    ]
)

print(response)

Explicación:

  • Este código modifica una Reserva de Instancia existente para cambiar su Zona de Disponibilidad y tipo de instancia.
  • El parámetro ReservedInstancesIds especifica el ID de la RI a modificar.
  • El parámetro TargetConfigurations especifica la nueva configuración para la RI.

Conclusión

Los Planes de Ahorro y las Reservas de Instancias de AWS son herramientas poderosas para gestionar y reducir tus costos en AWS. Al comprender sus diferencias y evaluar tus necesidades específicas, puedes elegir la opción correcta para maximizar tus ahorros. Los Planes de Ahorro ofrecen más flexibilidad, mientras que las Reservas de Instancias proporcionan mayores descuentos para cargas de trabajo predecibles. Utiliza los ejemplos proporcionados para comenzar a comprar y gestionar estas opciones de ahorro de costos en tu entorno de AWS.
