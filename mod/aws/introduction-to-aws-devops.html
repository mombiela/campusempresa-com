Descripción general

Amazon Web Services (AWS) proporciona un conjunto de herramientas y servicios que permiten prácticas de DevOps. Esta sección te introducirá a los conceptos fundamentales de AWS DevOps, las herramientas disponibles y cómo pueden ser utilizadas para automatizar y agilizar tus procesos de desarrollo y operaciones.

Conceptos clave

  • DevOps: Un conjunto de prácticas que combinan el desarrollo de software (Dev) y las operaciones de TI (Ops) para acortar el ciclo de vida del desarrollo y entregar software de alta calidad de manera continua.
  • Herramientas de AWS DevOps: Una colección de servicios proporcionados por AWS para facilitar las prácticas de DevOps, incluyendo CI/CD, infraestructura como código, monitoreo y registro.

Herramientas de AWS DevOps

AWS ofrece una variedad de herramientas para ayudar a implementar prácticas de DevOps. Aquí hay algunos de los servicios clave:

  • AWS CodePipeline: Un servicio de integración continua y entrega continua (CI/CD) para actualizaciones rápidas y confiables de aplicaciones e infraestructura.
  • AWS CodeBuild: Un servicio de compilación totalmente gestionado que compila código fuente, ejecuta pruebas y produce paquetes de software listos para su implementación.
  • AWS CodeDeploy: Un servicio que automatiza las implementaciones de código en cualquier instancia, incluidas las instancias de Amazon EC2 y los servidores locales.
  • AWS CodeCommit: Un servicio de control de código fuente totalmente gestionado que facilita a los equipos alojar repositorios Git seguros y escalables.
  • AWS CloudFormation: Un servicio que proporciona un lenguaje común para describir y aprovisionar todos los recursos de infraestructura en tu entorno en la nube.
  • Amazon CloudWatch: Un servicio de monitoreo y observabilidad diseñado para ingenieros de DevOps, desarrolladores, ingenieros de confiabilidad del sitio (SRE) y gerentes de TI.

Configuración de una canalización básica de CI/CD

Paso 1: Crear un repositorio de CodeCommit

Primero, crea un repositorio de CodeCommit para almacenar tu código fuente.

aws codecommit create-repository --repository-name MyDemoRepo --repository-description "Mi repositorio de demostración"

Paso 2: Configurar CodeBuild

Crea un proyecto de compilación en CodeBuild para compilar tu código y ejecutar pruebas.

{
  "name": "MyDemoBuildProject",
  "source": {
    "type": "CODECOMMIT",
    "location": "https://git-codecommit.us-east-1.amazonaws.com/v1/repos/MyDemoRepo"
  },
  "artifacts": {
    "type": "NO_ARTIFACTS"
  },
  "environment": {
    "type": "LINUX_CONTAINER",
    "image": "aws/codebuild/standard:4.0",
    "computeType": "BUILD_GENERAL1_SMALL"
  },
  "serviceRole": "arn:aws:iam::123456789012:role/service-role/codebuild-service-role"
}

Paso 3: Configurar CodeDeploy

Configura CodeDeploy para automatizar la implementación de tu aplicación.

{
  "applicationName": "MyDemoApplication",
  "deploymentGroupName": "MyDemoDeploymentGroup",
  "serviceRoleArn": "arn:aws:iam::123456789012:role/CodeDeployDemoRole",
  "deploymentConfigName": "CodeDeployDefault.OneAtATime",
  "ec2TagFilters": [
    {
      "Key": "Name",
      "Value": "MyDemoInstance",
      "Type": "KEY_AND_VALUE"
    }
  ]
}

Paso 4: Crear una CodePipeline

Finalmente, crea una canalización en CodePipeline para unir todo.

{
  "pipeline": {
    "name": "MyDemoPipeline",
    "roleArn": "arn:aws:iam::123456789012:role/AWS-CodePipeline-Service",
    "artifactStore": {
      "type": "S3",
      "location": "my-demo-bucket"
    },
    "stages": [
      {
        "name": "Source",
        "actions": [
          {
            "name": "Source",
            "actionTypeId": {
              "category": "Source",
              "owner": "AWS",
              "provider": "CodeCommit",
              "version": "1"
            },
            "outputArtifacts": [
              {
                "name": "SourceArtifact"
              }
            ],
            "configuration": {
              "RepositoryName": "MyDemoRepo",
              "BranchName": "main"
            }
          }
        ]
      },
      {
        "name": "Build",
        "actions": [
          {
            "name": "Build",
            "actionTypeId": {
              "category": "Build",
              "owner": "AWS",
              "provider": "CodeBuild",
              "version": "1"
            },
            "inputArtifacts": [
              {
                "name": "SourceArtifact"
              }
            ],
            "outputArtifacts": [
              {
                "name": "BuildArtifact"
              }
            ],
            "configuration": {
              "ProjectName": "MyDemoBuildProject"
            }
          }
        ]
      },
      {
        "name": "Deploy",
        "actions": [
          {
            "name": "Deploy",
            "actionTypeId": {
              "category": "Deploy",
              "owner": "AWS",
              "provider": "CodeDeploy",
              "version": "1"
            },
            "inputArtifacts": [
              {
                "name": "BuildArtifact"
              }
            ],
            "configuration": {
              "ApplicationName": "MyDemoApplication",
              "DeploymentGroupName": "MyDemoDeploymentGroup"
            }
          }
        ]
      }
    ]
  }
}

Conclusión

En esta sección, introdujimos los conceptos básicos de AWS DevOps y exploramos algunos de los servicios clave de AWS que apoyan las prácticas de DevOps. También recorrimos la configuración de una canalización básica de CI/CD utilizando AWS CodeCommit, CodeBuild, CodeDeploy y CodePipeline. Al aprovechar estas herramientas, puedes automatizar tu ciclo de vida de desarrollo de software, mejorar la eficiencia y asegurar implementaciones consistentes y confiables.
