Introducción al Esquema de GraphQL

Un esquema de GraphQL es una pieza central de cualquier implementación de servidor GraphQL. Define los tipos de datos que se pueden consultar y las relaciones entre esos tipos. Entender el esquema es crucial tanto para diseñar como para consultar una API de GraphQL.

Conceptos Clave

  • Tipos: Los bloques de construcción de un esquema de GraphQL. Los tipos definen la forma de los datos.
  • Consultas: Las operaciones de lectura que los clientes pueden realizar para obtener datos.
  • Mutaciones: Las operaciones de escritura que los clientes pueden realizar para modificar datos.
  • Resolvers: Funciones que resuelven los valores para los campos en el esquema.

Definiendo Tipos

En GraphQL, defines tipos usando la palabra clave type. Cada tipo tiene campos, y cada campo tiene un tipo.

type Book {
  id: ID!
  title: String!
  author: Author!
}

type Author {
  id: ID!
  name: String!
  books: [Book!]!
}

Explicación:

  • Book y Author son tipos personalizados.
  • ID! es un identificador único no anulable.
  • String! es una cadena de texto no anulable.
  • Author! indica que el campo author en Book no puede ser nulo.
  • [Book!]! indica una lista no anulable de objetos Book no anulables.

Consultas

Las consultas se definen usando la palabra clave type Query. Especifican qué datos se pueden obtener y cómo.

type Query {
  books: [Book]
  book(id: ID!): Book
  authors: [Author]
  author(id: ID!): Author
}

Explicación:

  • books devuelve una lista de objetos Book.
  • book(id: ID!) devuelve un único objeto Book basado en el id proporcionado.
  • authors devuelve una lista de objetos Author.
  • author(id: ID!) devuelve un único objeto Author basado en el id proporcionado.

Mutaciones

Las mutaciones se definen usando la palabra clave type Mutation. Especifican qué datos se pueden modificar y cómo.

type Mutation {
  addBook(title: String!, authorId: ID!): Book
  updateBook(id: ID!, title: String, authorId: ID): Book
  deleteBook(id: ID!): Book
}

Explicación:

  • addBook(title: String!, authorId: ID!) añade un nuevo Book y devuelve el Book añadido.
  • updateBook(id: ID!, title: String, authorId: ID) actualiza un Book existente y devuelve el Book actualizado.
  • deleteBook(id: ID!) elimina un Book existente y devuelve el Book eliminado.

Resolvers

Los resolvers son funciones que proporcionan la lógica para obtener o modificar los datos para cada campo en el esquema.

const resolvers = {
  Query: {
    books: () => getAllBooks(),
    book: (parent, args) => getBookById(args.id),
    authors: () => getAllAuthors(),
    author: (parent, args) => getAuthorById(args.id),
  },
  Mutation: {
    addBook: (parent, args) => addNewBook(args.title, args.authorId),
    updateBook: (parent, args) => updateExistingBook(args.id, args.title, args.authorId),
    deleteBook: (parent, args) => deleteExistingBook(args.id),
  },
  Book: {
    author: (book) => getAuthorById(book.authorId),
  },
  Author: {
    books: (author) => getBooksByAuthorId(author.id),
  },
};

Explicación:

  • Los resolvers de Query manejan la obtención de datos.
  • Los resolvers de Mutation manejan la modificación de datos.
  • Los resolvers específicos de campos como Book.author y Author.books manejan la obtención de datos anidados.

Resumen

En esta sección, cubrimos los conceptos básicos de los esquemas de GraphQL, incluyendo:

  • Definir tipos para dar forma a los datos.
  • Crear consultas para obtener datos.
  • Crear mutaciones para modificar datos.
  • Implementar resolvers para proporcionar la lógica para obtener y modificar datos.

Entender estos fundamentos es crucial para construir y consultar una API de GraphQL de manera efectiva. A medida que avances, aprenderás conceptos más avanzados como interfaces, uniones y escalares personalizados para mejorar aún más tus esquemas de GraphQL.
