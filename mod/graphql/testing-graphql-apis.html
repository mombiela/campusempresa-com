Pruebas de APIs GraphQL

Las pruebas son una parte crucial del desarrollo de APIs GraphQL robustas y confiables. Esta sección te guiará a través del proceso de prueba de APIs GraphQL, desde conceptos básicos hasta técnicas avanzadas.

Introducción a las Pruebas de APIs GraphQL

Las pruebas aseguran que tu API GraphQL se comporte como se espera y ayudan a detectar errores temprano en el proceso de desarrollo. Hay varios tipos de pruebas que puedes realizar en una API GraphQL:

  • Pruebas Unitarias: Prueban resolutores individuales y otras pequeñas unidades de código.
  • Pruebas de Integración: Prueban cómo diferentes partes de tu aplicación funcionan juntas.
  • Pruebas de Extremo a Extremo (E2E): Prueban todo el flujo de la aplicación desde el cliente hasta el servidor y de vuelta.

Herramientas para Probar APIs GraphQL

Varias herramientas pueden ayudarte a probar APIs GraphQL de manera efectiva:

  • Jest: Un marco de pruebas de JavaScript que funciona bien con GraphQL.
  • Utilidades de Prueba de Apollo Server: Herramientas proporcionadas por Apollo Server para facilitar las pruebas.
  • GraphQL Playground: Un IDE interactivo en el navegador para GraphQL que puede usarse para pruebas manuales.

Escribir Pruebas Unitarias para Resolutores GraphQL

Las pruebas unitarias se enfocan en resolutores individuales para asegurar que devuelvan los resultados esperados. Aquí te mostramos cómo puedes escribir pruebas unitarias usando Jest.

Ejemplo: Probar un Resolutor con Jest

Supongamos que tienes el siguiente resolutor:

const resolvers = {
  Query: {
    user: (parent, args, context, info) => {
      return context.dataSources.userAPI.getUserById(args.id);
    },
  },
};

Puedes escribir una prueba unitaria para este resolutor de la siguiente manera:

const { ApolloServer } = require('apollo-server');
const { createTestClient } = require('apollo-server-testing');
const gql = require('graphql-tag');

// Fuente de datos simulada
const mockUserAPI = {
  getUserById: jest.fn(),
};

// Crear un servidor de prueba
const server = new ApolloServer({
  typeDefs,
  resolvers,
  dataSources: () => ({
    userAPI: mockUserAPI,
  }),
});

const { query } = createTestClient(server);

describe('Query.user', () => {
  it('fetches a user by ID', async () => {
    // Preparar
    const user = { id: '1', name: 'John Doe' };
    mockUserAPI.getUserById.mockResolvedValue(user);

    // Actuar
    const res = await query({
      query: gql`
        query getUser($id: ID!) {
          user(id: $id) {
            id
            name
          }
        }
      `,
      variables: { id: '1' },
    });

    // Afirmar
    expect(res.data.user).toEqual(user);
  });
});

Escribir Pruebas de Integración

Las pruebas de integración aseguran que diferentes partes de tu aplicación funcionen correctamente juntas. Estas pruebas típicamente implican configurar un servidor de prueba y hacer solicitudes a él.

Ejemplo: Prueba de Integración con Apollo Server

const { ApolloServer } = require('apollo-server');
const { createTestClient } = require('apollo-server-testing');
const gql = require('graphql-tag');

// Define tu esquema y resolutores
const typeDefs = gql`
  type User {
    id: ID!
    name: String!
  }

  type Query {
    user(id: ID!): User
  }
`;

const resolvers = {
  Query: {
    user: (parent, args, context, info) => {
      return { id: args.id, name: 'John Doe' };
    },
  },
};

// Crear un servidor de prueba
const server = new ApolloServer({
  typeDefs,
  resolvers,
});

const { query } = createTestClient(server);

describe('Prueba de Integración', () => {
  it('fetches a user by ID', async () => {
    const res = await query({
      query: gql`
        query getUser($id: ID!) {
          user(id: $id) {
            id
            name
          }
        }
      `,
      variables: { id: '1' },
    });

    expect(res.data.user).toEqual({ id: '1', name: 'John Doe' });
  });
});

Escribir Pruebas de Extremo a Extremo (E2E)

Las pruebas de extremo a extremo simulan interacciones reales de usuarios con tu aplicación. Estas pruebas son típicamente más complejas e implican configurar toda la pila de la aplicación.

Ejemplo: Prueba E2E con Cypress

Cypress es una herramienta popular para pruebas E2E. Aquí tienes un ejemplo básico de cómo podrías usar Cypress para probar una API GraphQL.

  1. Instalar Cypress:

    npm install cypress --save-dev

  2. Escribir una Prueba de Cypress:

    // cypress/integration/graphql_spec.js
describe('GraphQL API', () => {
  it('fetches a user by ID', () => {
    cy.request({
      method: 'POST',
      url: 'http://localhost:4000/graphql',
      body: {
        query: `
          query getUser($id: ID!) {
            user(id: $id) {
              id
              name
            }
          }
        `,
        variables: { id: '1' },
      },
    }).then((response) => {
      expect(response.body.data.user).to.have.property('id', '1');
      expect(response.body.data.user).to.have.property('name', 'John Doe');
    });
  });
});

Conclusión

Probar APIs GraphQL es esencial para asegurar la fiabilidad y corrección de tu aplicación. Al escribir pruebas unitarias, pruebas de integración y pruebas de extremo a extremo, puedes detectar errores temprano y asegurar que tu API se comporte como se espera. Usar herramientas como Jest, las utilidades de prueba de Apollo Server y Cypress puede hacer que el proceso de prueba sea más eficiente y efectivo.
