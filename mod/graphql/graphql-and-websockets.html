Introducción a GraphQL y WebSockets

GraphQL es un lenguaje de consulta para APIs y un tiempo de ejecución para ejecutar esas consultas utilizando un sistema de tipos que defines para tus datos. WebSockets proporcionan un canal de comunicación full-duplex sobre una única conexión TCP, permitiendo la transferencia de datos en tiempo real entre cliente y servidor.

Combinar GraphQL con WebSockets permite actualizaciones y suscripciones en tiempo real, haciendo posible construir aplicaciones altamente interactivas y dinámicas.

Conceptos Clave

  • GraphQL: Un lenguaje de consulta para tu API y un tiempo de ejecución del lado del servidor para ejecutar consultas utilizando un sistema de tipos que defines para tus datos.
  • WebSockets: Un protocolo que proporciona canales de comunicación full-duplex sobre una única conexión TCP.
  • Suscripciones: Una característica de GraphQL que permite a los clientes suscribirse a eventos específicos y recibir actualizaciones en tiempo real.

Configuración de GraphQL con WebSockets

Para usar GraphQL con WebSockets, necesitas configurar un servidor GraphQL que soporte suscripciones. Una opción popular es Apollo Server.

Instalación de Dependencias

Primero, instala los paquetes necesarios:

npm install apollo-server graphql subscriptions-transport-ws

Configuración de Apollo Server con Soporte para WebSocket

Aquí tienes una configuración básica para un Apollo Server con soporte para WebSocket:

const { ApolloServer, gql } = require('apollo-server');
const { createServer } = require('http');
const { execute, subscribe } = require('graphql');
const { SubscriptionServer } = require('subscriptions-transport-ws');

// Define tu esquema
const typeDefs = gql`
  type Query {
    hello: String
  }

  type Subscription {
    time: String
  }
`;

// Define tus resolvers
const resolvers = {
  Query: {
    hello: () => '¡Hola mundo!',
  },
  Subscription: {
    time: {
      subscribe: (_, __, { pubsub }) => pubsub.asyncIterator('TIME'),
    },
  },
};

// Crea una instancia de ApolloServer
const server = new ApolloServer({
  typeDefs,
  resolvers,
  context: ({ req }) => ({
    pubsub,
  }),
});

// Crea un servidor HTTP
const httpServer = createServer(server);

// Configura WebSocket para suscripciones
server.installSubscriptionHandlers(httpServer);

// Inicia el servidor
httpServer.listen(4000, () => {
  console.log(`El servidor está corriendo en http://localhost:4000${server.graphqlPath}`);
  console.log(`Suscripciones listas en ws://localhost:4000${server.subscriptionsPath}`);
});

Publicación de Eventos

Para publicar eventos a los suscriptores, puedes usar una instancia de PubSub del paquete graphql-subscriptions:

const { PubSub } = require('graphql-subscriptions');
const pubsub = new PubSub();

// Publica un evento cada segundo
setInterval(() => {
  const time = new Date().toISOString();
  pubsub.publish('TIME', { time });
}, 1000);

Ejemplo: Aplicación de Chat en Tiempo Real

Vamos a construir una simple aplicación de chat en tiempo real usando GraphQL y WebSockets.

Definición del Esquema

Define el esquema para mensajes y suscripciones:

const typeDefs = gql`
  type Message {
    id: ID!
    content: String!
    user: String!
  }

  type Query {
    messages: [Message!]
  }

  type Mutation {
    postMessage(user: String!, content: String!): ID!
  }

  type Subscription {
    messages: [Message!]
  }
`;

Resolvers

Implementa los resolvers para consultas, mutaciones y suscripciones:

const messages = [];
const resolvers = {
  Query: {
    messages: () => messages,
  },
  Mutation: {
    postMessage: (parent, { user, content }) => {
      const id = messages.length;
      messages.push({ id, user, content });
      pubsub.publish('MESSAGES', { messages });
      return id;
    },
  },
  Subscription: {
    messages: {
      subscribe: () => pubsub.asyncIterator('MESSAGES'),
    },
  },
};

Implementación del Lado del Cliente

En el lado del cliente, puedes usar Apollo Client para conectarte al servidor y manejar suscripciones:

import { ApolloClient, InMemoryCache, gql } from '@apollo/client';
import { WebSocketLink } from '@apollo/client/link/ws';
import { split } from '@apollo/client';
import { getMainDefinition } from '@apollo/client/utilities';
import { HttpLink } from '@apollo/client';

// Crea un enlace HTTP
const httpLink = new HttpLink({
  uri: 'http://localhost:4000/graphql',
});

// Crea un enlace WebSocket
const wsLink = new WebSocketLink({
  uri: 'ws://localhost:4000/graphql',
  options: {
    reconnect: true,
  },
});

// Divide enlaces según el tipo de operación
const link = split(
  ({ query }) => {
    const definition = getMainDefinition(query);
    return (
      definition.kind === 'OperationDefinition' &&
      definition.operation === 'subscription'
    );
  },
  wsLink,
  httpLink,
);

// Crea una instancia de Apollo Client
const client = new ApolloClient({
  link,
  cache: new InMemoryCache(),
});

// Suscríbete a mensajes
client
  .subscribe({
    query: gql`
      subscription {
        messages {
          id
          content
          user
        }
      }
    `,
  })
  .subscribe({
    next({ data }) {
      console.log(data);
    },
  });

Conclusión

GraphQL y WebSockets juntos proporcionan un conjunto de herramientas poderoso para construir aplicaciones en tiempo real. Al aprovechar las suscripciones de GraphQL, puedes crear experiencias de usuario dinámicas e interactivas que se actualizan en tiempo real. Esta combinación es particularmente útil para aplicaciones como aplicaciones de chat, actualizaciones en vivo de deportes y herramientas colaborativas.

En esta sección, aprendiste cómo configurar un servidor GraphQL con soporte para WebSocket, implementar suscripciones y construir una aplicación de chat en tiempo real. Con estas habilidades, ahora puedes explorar casos de uso más avanzados y optimizar el manejo de datos en tiempo real.
