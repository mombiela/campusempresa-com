Introducción a las Interfaces

Las interfaces en GraphQL son una forma poderosa de definir un contrato que múltiples tipos pueden implementar. Permiten crear un conjunto de campos que diferentes tipos deben incluir, asegurando consistencia en tu esquema.

  • Definición: Una interfaz es un tipo abstracto que incluye un cierto conjunto de campos que un tipo debe implementar.
  • Propósito: Las interfaces se utilizan para definir una estructura común para diferentes tipos, permitiendo el polimorfismo en tu esquema de GraphQL.

Definiendo Interfaces

Para definir una interfaz en GraphQL, se utiliza la palabra clave interface seguida del nombre de la interfaz y los campos que incluye.

Ejemplo

interface Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
}

En este ejemplo:

  • Character es la interfaz.
  • Incluye tres campos: id, name y appearsIn.

Implementando Interfaces

Los tipos que implementan una interfaz deben incluir todos los campos definidos en la interfaz.

Ejemplo

type Human implements Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
  totalCredits: Int
}

type Droid implements Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
  primaryFunction: String
}

En este ejemplo:

  • Human y Droid ambos implementan la interfaz Character.
  • Ambos tipos incluyen los campos id, name y appearsIn como requiere la interfaz Character.
  • También incluyen campos adicionales específicos para cada tipo (totalCredits para Human y primaryFunction para Droid).

Consultando Interfaces

Al consultar una interfaz, puedes solicitar campos que son parte de la interfaz, y también puedes usar fragmentos en línea para solicitar campos específicos de los tipos que implementan.

Ejemplo

{
  characters {
    id
    name
    appearsIn
    ... on Human {
      totalCredits
    }
    ... on Droid {
      primaryFunction
    }
  }
}

En esta consulta:

  • characters es un campo que devuelve una lista de tipos de interfaz Character.
  • La consulta solicita los campos id, name y appearsIn, que son parte de la interfaz Character.
  • Los fragmentos en línea (... on Human y ... on Droid) se utilizan para solicitar campos específicos de los tipos Human y Droid.

Interfaces vs. Uniones

Aunque tanto las interfaces como las uniones permiten el polimorfismo, sirven para diferentes propósitos.

| Característica | Interfaz | Unión | |------------------|------------------------------------------------|-------------------------------------------------| | Campos Comunes | Sí | No | | Verificación de Tipo | Puede verificar campos específicos | Solo puede verificar el tipo | | Caso de Uso | Cuando los tipos comparten campos comunes | Cuando los tipos no comparten campos comunes | | Sintaxis | Palabra clave interface | Palabra clave union |

Ejemplo de Unión

union SearchResult = Human | Droid | Starship

{
  search(text: "R2") {
    ... on Human {
      name
    }
    ... on Droid {
      primaryFunction
    }
    ... on Starship {
      model
    }
  }
}

En este ejemplo:

  • SearchResult es una unión de Human, Droid y Starship.
  • La consulta utiliza fragmentos en línea para solicitar campos específicos de cada tipo en la unión.

Conclusión

Las interfaces en GraphQL proporcionan una forma robusta de imponer una estructura consistente en diferentes tipos. Al definir campos comunes en una interfaz e implementarlos en varios tipos, puedes asegurarte de que tu esquema permanezca organizado y predecible. Entender cómo definir, implementar y consultar interfaces es esencial para crear APIs de GraphQL flexibles y mantenibles.
