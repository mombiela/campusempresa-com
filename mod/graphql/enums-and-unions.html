Enumeraciones y Uniones en GraphQL
Interfaces ►

Introducción

En esta sección, exploraremos dos características importantes de GraphQL: Enumeraciones y Uniones. Estas características ayudan a definir esquemas más precisos y flexibles, haciendo que tu API de GraphQL sea más robusta y fácil de usar.

Enumeraciones en GraphQL

¿Qué son las Enumeraciones?

  • Las Enumeraciones (abreviadas como Enums) son un tipo especial de escalar que restringe el valor a un conjunto predefinido de constantes.
  • Son útiles para representar un conjunto fijo de valores posibles, como días de la semana, roles de usuario o códigos de estado.

Definiendo Enumeraciones

Para definir una Enumeración en GraphQL, usas la palabra clave enum seguida del nombre de la Enumeración y el conjunto de valores posibles.

enum Role {
  ADMIN
  USER
  GUEST
}

Usando Enumeraciones en el Esquema

Las Enumeraciones pueden ser usadas como tipos de campo en tu esquema de GraphQL. Aquí hay un ejemplo de cómo usar la enumeración Role en un tipo User:

type User {
  id: ID!
  name: String!
  role: Role!
}

Ejemplo de Consulta con Enumeraciones

Aquí tienes cómo podrías consultar un usuario y obtener su rol:

query {
  user(id: "1") {
    name
    role
  }
}

Ejemplo de Mutación con Enumeraciones

También puedes usar Enumeraciones en mutaciones. Por ejemplo, estableciendo el rol de un usuario:

mutation {
  updateUserRole(id: "1", role: ADMIN) {
    id
    name
    role
  }
}

Uniones en GraphQL

¿Qué son las Uniones?

  • Las Uniones permiten que un campo devuelva uno de varios tipos diferentes.
  • A diferencia de las interfaces, las uniones no comparten ningún campo en común.

Definiendo Uniones

Para definir una Unión, usas la palabra clave union seguida del nombre de la Unión y los posibles tipos que puede devolver.

union SearchResult = User | Post

Usando Uniones en el Esquema

Las Uniones pueden ser usadas como tipos de campo en tu esquema de GraphQL. Aquí hay un ejemplo de cómo usar la unión SearchResult en una consulta:

type Query {
  search(text: String!): [SearchResult!]!
}

Ejemplo de Consulta con Uniones

Aquí tienes cómo podrías consultar una búsqueda y obtener diferentes tipos de resultados:

query {
  search(text: "GraphQL") {
    ... on User {
      id
      name
    }
    ... on Post {
      id
      title
      content
    }
  }
}

Resolviendo Uniones

Cuando usas uniones, necesitas definir cómo resolver el tipo de unión en tu código del lado del servidor. Aquí hay un ejemplo en JavaScript usando Apollo Server:

const resolvers = {
  SearchResult: {
    __resolveType(obj, context, info) {
      if (obj.name) {
        return 'User';
      }
      if (obj.title) {
        return 'Post';
      }
      return null;
    },
  },
  Query: {
    search: (parent, args, context, info) => {
      // Implementa la lógica de búsqueda aquí
      return [
        { id: '1', name: 'Alice' },
        { id: '2', title: 'Introducción a GraphQL', content: '...' },
      ];
    },
  },
};

Conclusión

Las Enumeraciones y Uniones son características poderosas en GraphQL que te permiten crear esquemas más precisos y flexibles. Las Enumeraciones restringen un campo a un conjunto de valores predefinidos, haciendo tu API más predecible y fácil de usar. Las Uniones permiten que un campo devuelva diferentes tipos, proporcionando flexibilidad en cómo estructuras tus datos.

Al entender y utilizar Enumeraciones y Uniones, puedes mejorar la robustez y usabilidad de tus APIs de GraphQL, haciéndolas más intuitivas para que los clientes interactúen con ellas.
