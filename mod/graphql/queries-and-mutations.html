Introducción a GraphQL

GraphQL es un lenguaje de consulta para APIs y un entorno de ejecución para ejecutar esas consultas utilizando un sistema de tipos que defines para tus datos. Permite a los clientes solicitar exactamente los datos que necesitan, haciéndolo más eficiente que las APIs REST tradicionales.

Consultas

Las consultas en GraphQL se utilizan para obtener datos del servidor. Son análogas a las solicitudes GET en las APIs REST.

Estructura Básica de una Consulta

Una consulta básica de GraphQL se ve así:

{
  user(id: "1") {
    id
    name
    email
  }
}
  • Explicación:
    • La consulta solicita un user con un id de "1".
    • Se especifican los campos id, name y email, lo que significa que solo se devolverán estos campos.

Consultas Anidadas

GraphQL permite consultas anidadas, lo que te permite obtener datos relacionados en una sola solicitud.

{
  user(id: "1") {
    id
    name
    email
    posts {
      title
      content
    }
  }
}
  • Explicación:
    • Además de la información básica del usuario, esta consulta obtiene los posts asociados con el usuario.
    • Para cada post, solo se solicitan los campos title y content.

Argumentos

Se pueden pasar argumentos a los campos para filtrar o personalizar los datos devueltos.

{
  posts(limit: 10, sort: "DESC") {
    title
    content
  }
}
  • Explicación:
    • El campo posts acepta argumentos limit y sort para controlar el número de posts devueltos y su orden.

Alias

Los alias te permiten renombrar el resultado de un campo para evitar conflictos o hacer la respuesta más legible.

{
  firstUser: user(id: "1") {
    name
  }
  secondUser: user(id: "2") {
    name
  }
}
  • Explicación:
    • El campo user se consulta dos veces con diferentes valores de id, pero los resultados se renombran como firstUser y secondUser.

Mutaciones

Las mutaciones en GraphQL se utilizan para modificar datos en el servidor. Son análogas a las solicitudes POST, PUT, DELETE en las APIs REST.

Estructura Básica de una Mutación

Una mutación básica se ve así:

mutation {
  createUser(name: "John Doe", email: "john@example.com") {
    id
    name
    email
  }
}
  • Explicación:
    • La mutación createUser crea un nuevo usuario con el name y email especificados.
    • La respuesta incluye el id, name y email del usuario recién creado.

Actualización de Datos

Las mutaciones también se pueden utilizar para actualizar datos existentes.

mutation {
  updateUser(id: "1", email: "newemail@example.com") {
    id
    name
    email
  }
}
  • Explicación:
    • La mutación updateUser actualiza el email del usuario con id "1".
    • La respuesta incluye la información actualizada del usuario.

Eliminación de Datos

Las mutaciones también pueden eliminar datos.

mutation {
  deleteUser(id: "1") {
    id
    name
  }
}
  • Explicación:
    • La mutación deleteUser elimina al usuario con id "1".
    • La respuesta incluye el id y name del usuario eliminado.

Combinación de Consultas y Mutaciones

Puedes combinar múltiples consultas y mutaciones en una sola solicitud.

mutation {
  createUser(name: "Jane Doe", email: "jane@example.com") {
    id
    name
  }
}

query {
  users {
    id
    name
    email
  }
}
  • Explicación:
    • La mutación createUser crea un nuevo usuario.
    • La consulta users obtiene la lista de todos los usuarios.

Conclusión

  • Consultas: Se utilizan para obtener datos. Pueden ser anidadas, aceptar argumentos y usar alias.
  • Mutaciones: Se utilizan para modificar datos. Pueden crear, actualizar o eliminar datos.
  • Combinar consultas y mutaciones en una sola solicitud puede optimizar el uso de la red.

Entender cómo usar eficazmente las consultas y mutaciones en GraphQL te permite construir APIs eficientes y flexibles. Practica escribiendo tus propias consultas y mutaciones para familiarizarte con la sintaxis y las capacidades de GraphQL.
