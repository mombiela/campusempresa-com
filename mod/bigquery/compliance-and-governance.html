Introducción

BigQuery, como un almacén de datos completamente gestionado, proporciona características robustas para garantizar el cumplimiento y la gobernanza. Esta sección cubrirá los aspectos esenciales del cumplimiento y la gobernanza en BigQuery, desde conceptos básicos hasta prácticas avanzadas.

Conceptos Clave

  • Cumplimiento: Asegurar que el manejo y procesamiento de datos cumplan con los requisitos regulatorios.
  • Gobernanza: Establecer políticas y procedimientos para gestionar la calidad, seguridad y uso de los datos.

Seguridad y Privacidad de los Datos

Control de Acceso

  • Gestión de Identidad y Acceso (IAM): BigQuery utiliza IAM para gestionar el acceso a los recursos.
    • Roles: Roles predefinidos (por ejemplo, bigquery.user, bigquery.admin) y roles personalizados.
    • Permisos: Acciones específicas que se pueden realizar (por ejemplo, bigquery.tables.getData).
-- Ejemplo: Otorgar un rol a un usuario
GRANT `roles/bigquery.dataViewer` ON PROJECT `my-project` TO `user:example@example.com`;

Cifrado de Datos

  • Cifrado en Reposo: Los datos se cifran utilizando claves gestionadas por Google de forma predeterminada.
  • Claves de Cifrado Gestionadas por el Cliente (CMEK): Opción para usar tus propias claves de cifrado.
-- Ejemplo: Crear un conjunto de datos con CMEK
CREATE SCHEMA my_dataset
OPTIONS (
  location = 'US',
  default_kms_key_name = 'projects/my-project/locations/us/keyRings/my-keyring/cryptoKeys/my-key'
);

Gobernanza de Datos

Linaje de Datos

  • Seguimiento del Linaje de Datos: Comprender el flujo de datos desde la fuente hasta el destino.
    • Registros de Auditoría: Usar Cloud Audit Logs para rastrear el acceso y los cambios.
    • Catálogo de Datos: Usar Data Catalog para la gestión de metadatos y el seguimiento del linaje.

Calidad de los Datos

  • Validación de Datos: Implementar verificaciones para asegurar la precisión y consistencia de los datos.
    • Restricciones: Usar restricciones SQL para hacer cumplir la integridad de los datos.
    • Perfilado de Datos: Analizar los datos para comprender su estructura y calidad.
-- Ejemplo: Agregar una restricción NOT NULL
CREATE TABLE my_table (
  id INT64 NOT NULL,
  name STRING NOT NULL
);

Retención y Eliminación de Datos

  • Políticas de Retención: Establecer políticas para retener datos por un período específico.
  • Políticas de Eliminación: Asegurar que los datos se eliminen de forma segura después de su período de retención.
-- Ejemplo: Establecer un tiempo de expiración para una tabla
CREATE TABLE my_table (
  id INT64,
  name STRING
)
OPTIONS (
  expiration_timestamp = TIMESTAMP '2023-12-31 23:59:59 UTC'
);

Auditoría y Monitoreo

Registros de Auditoría

  • Tipos de Registros: Actividad Administrativa, Acceso a Datos, Evento del Sistema y Políticas Denegadas.
  • Análisis de Registros: Usar BigQuery para analizar los registros de auditoría para el monitoreo de cumplimiento y seguridad.
-- Ejemplo: Consultar registros de auditoría
SELECT
  protopayload_auditlog.methodName,
  protopayload_auditlog.resourceName,
  timestamp
FROM
  `my-project.audit_logs.cloudaudit_googleapis_com_activity_*`
WHERE
  protopayload_auditlog.methodName = 'google.bigquery.v2.JobService.InsertJob';

Monitoreo

  • Herramientas de Monitoreo: Usar Cloud Monitoring y Cloud Logging para monitorear el uso y rendimiento de BigQuery.
  • Alertas: Configurar alertas para eventos o umbrales específicos.
-- Ejemplo: Crear una política de alerta
gcloud alpha monitoring policies create \
  --display-name="Alerta de Fallo de Trabajo de BigQuery" \
  --combiner="OR" \
  --conditions="condition_threshold.json"

Conclusión

En este tema, exploramos los aspectos críticos del cumplimiento y la gobernanza en BigQuery. Cubrimos el control de acceso, el cifrado de datos, el linaje de datos, la calidad de los datos, las políticas de retención y la auditoría y monitoreo. Al implementar estas prácticas, puedes asegurar que tu entorno de BigQuery sea seguro, cumpla con las normativas y esté bien gobernado.
