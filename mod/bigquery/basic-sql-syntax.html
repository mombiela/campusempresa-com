Introducción

BigQuery es un almacén de datos totalmente gestionado y sin servidor que permite el análisis escalable de petabytes de datos. Utiliza SQL (Structured Query Language) para consultar y gestionar datos. Esta sección cubrirá la sintaxis básica de SQL necesaria para comenzar con BigQuery.

Conceptos clave

  • SQL (Structured Query Language): Un lenguaje estándar para acceder y manipular bases de datos.
  • BigQuery: Un servicio de la plataforma Google Cloud para almacenamiento y análisis de datos.
  • Dataset: Una colección de tablas.
  • Table: Una colección de filas y columnas.
  • Query: Una solicitud de datos o información de una base de datos.

Declaraciones básicas de SQL

Declaración SELECT

La declaración SELECT se utiliza para consultar datos de una base de datos.

SELECT column1, column2, ...
FROM table_name;

Ejemplo:

SELECT name, age
FROM employees;

Explicación: Esta consulta selecciona las columnas name y age de la tabla employees.

Cláusula WHERE

La cláusula WHERE se utiliza para filtrar registros.

SELECT column1, column2, ...
FROM table_name
WHERE condition;

Ejemplo:

SELECT name, age
FROM employees
WHERE age > 30;

Explicación: Esta consulta selecciona las columnas name y age de la tabla employees donde la edad es mayor de 30.

Cláusula ORDER BY

La cláusula ORDER BY se utiliza para ordenar el conjunto de resultados.

SELECT column1, column2, ...
FROM table_name
ORDER BY column1 [ASC|DESC];

Ejemplo:

SELECT name, age
FROM employees
ORDER BY age DESC;

Explicación: Esta consulta selecciona las columnas name y age de la tabla employees y ordena los resultados por edad en orden descendente.

Cláusula LIMIT

La cláusula LIMIT se utiliza para especificar el número de registros a devolver.

SELECT column1, column2, ...
FROM table_name
LIMIT number;

Ejemplo:

SELECT name, age
FROM employees
LIMIT 5;

Explicación: Esta consulta selecciona las columnas name y age de la tabla employees y devuelve solo los primeros 5 registros.

Funciones de agregación

Las funciones de agregación realizan un cálculo sobre un conjunto de valores y devuelven un solo valor.

  • COUNT(): Devuelve el número de filas.
  • SUM(): Devuelve la suma total de una columna numérica.
  • AVG(): Devuelve el valor promedio de una columna numérica.
  • MAX(): Devuelve el valor máximo.
  • MIN(): Devuelve el valor mínimo.

Ejemplo:

SELECT COUNT(*), AVG(age), MAX(age), MIN(age)
FROM employees;

Explicación: Esta consulta devuelve el número total de empleados, la edad promedio, la edad máxima y la edad mínima de la tabla employees.

Cláusula GROUP BY

La cláusula GROUP BY agrupa filas que tienen los mismos valores en filas de resumen.

SELECT column1, COUNT(*)
FROM table_name
GROUP BY column1;

Ejemplo:

SELECT department, COUNT(*)
FROM employees
GROUP BY department;

Explicación: Esta consulta agrupa a los empleados por departamento y devuelve el número de empleados en cada departamento.

Cláusula JOIN

La cláusula JOIN se utiliza para combinar filas de dos o más tablas.

  • INNER JOIN: Devuelve registros que tienen valores coincidentes en ambas tablas.
  • LEFT JOIN: Devuelve todos los registros de la tabla izquierda y los registros coincidentes de la tabla derecha.
  • RIGHT JOIN: Devuelve todos los registros de la tabla derecha y los registros coincidentes de la tabla izquierda.
  • FULL OUTER JOIN: Devuelve todos los registros cuando hay una coincidencia en la tabla izquierda o derecha.

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
INNER JOIN departments ON employees.department_id = departments.id;

Explicación: Esta consulta selecciona el name de la tabla employees y el department_name de la tabla departments donde el department_id en employees coincide con el id en departments.

Conclusión

Entender la sintaxis básica de SQL es esencial para consultar y gestionar datos en BigQuery. Esta sección cubrió declaraciones fundamentales de SQL como SELECT, WHERE, ORDER BY, LIMIT, funciones de agregación, GROUP BY y JOIN. Dominar estos conceptos básicos proporcionará una base sólida para consultas SQL más avanzadas y análisis de datos en BigQuery.
