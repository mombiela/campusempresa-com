Introducción

En esta sección, exploraremos los conceptos fundamentales de los tipos de datos y esquemas en BigQuery. Comprender estos conceptos es crucial para almacenar, consultar y analizar datos de manera eficiente. Cubriremos los tipos de datos básicos, cómo definir esquemas y temas avanzados como campos anidados y repetidos.

Tipos de Datos Básicos

BigQuery admite una variedad de tipos de datos para acomodar diferentes tipos de datos. Aquí están los principales tipos de datos que encontrarás:

  • STRING: Usado para datos de texto.
  • INT64: Usado para números enteros.
  • FLOAT64: Usado para números de punto flotante.
  • BOOL: Usado para valores booleanos (TRUE o FALSE).
  • BYTES: Usado para datos binarios.
  • DATE: Usado para fechas de calendario (sin hora).
  • DATETIME: Usado para fecha y hora (sin zona horaria).
  • TIMESTAMP: Usado para fecha y hora con zona horaria.
  • TIME: Usado para la hora del día (sin fecha).
  • GEOGRAPHY: Usado para datos geográficos.

Ejemplo

CREATE TABLE my_dataset.my_table (
  name STRING,
  age INT64,
  height FLOAT64,
  is_student BOOL,
  profile_picture BYTES,
  birth_date DATE,
  last_login TIMESTAMP,
  daily_schedule TIME
);

En este ejemplo, definimos una tabla con varias columnas, cada una usando un tipo de dato diferente.

Definiendo Esquemas

Un esquema en BigQuery es un plano que define la estructura de una tabla, incluidos los nombres y tipos de datos de las columnas. Los esquemas pueden definirse manualmente o inferirse automáticamente al cargar datos.

Definición Manual de Esquema

Al crear una tabla, puedes especificar el esquema explícitamente:

CREATE TABLE my_dataset.user_info (
  user_id INT64,
  user_name STRING,
  email STRING,
  signup_date DATE
);

Inferencia Automática de Esquema

BigQuery también puede inferir el esquema a partir de los datos cuando los cargas desde un archivo:

bq load --autodetect --source_format=CSV my_dataset.user_info gs://my_bucket/user_info.csv

Campos Anidados y Repetidos

BigQuery admite campos anidados y repetidos, lo que te permite almacenar estructuras de datos complejas dentro de una tabla. Esto es particularmente útil para manejar datos similares a JSON.

Campos Anidados

Los campos anidados son campos que contienen otros campos. Se definen usando el tipo de dato STRUCT.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  customer STRUCT<
    customer_id INT64,
    customer_name STRING,
    contact_info STRUCT<
      email STRING,
      phone STRING
    >
  >,
  order_date DATE
);

Campos Repetidos

Los campos repetidos son arreglos de valores. Se definen usando el tipo de dato ARRAY.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  items ARRAY<STRUCT<
    item_id INT64,
    item_name STRING,
    quantity INT64
  >>,
  order_date DATE
);

Combinando Campos Anidados y Repetidos

Puedes combinar campos anidados y repetidos para crear estructuras de datos aún más complejas.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  customer STRUCT<
    customer_id INT64,
    customer_name STRING
  >,
  items ARRAY<STRUCT<
    item_id INT64,
    item_name STRING,
    quantity INT64
  >>,
  order_date DATE
);

Conclusión

Comprender los tipos de datos y esquemas es fundamental para trabajar eficazmente con BigQuery. Al dominar estos conceptos, puedes diseñar estructuras de datos eficientes y escalables que satisfagan tus necesidades analíticas. Ya sea que estés manejando tablas simples o campos anidados y repetidos complejos, BigQuery proporciona la flexibilidad para manejar una amplia gama de escenarios de datos.
