Introducción

Integrar BigQuery con otros servicios de Google Cloud puede mejorar significativamente tus capacidades de procesamiento, análisis y visualización de datos. Esta sección cubrirá cómo conectar BigQuery con varios servicios de Google Cloud, incluyendo Google Cloud Storage, Google Cloud Pub/Sub, Google Data Studio y Google Cloud Functions.

Google Cloud Storage

Google Cloud Storage (GCS) es un servicio de almacenamiento de objetos escalable que te permite almacenar y recuperar cualquier cantidad de datos en cualquier momento.

Cargando Datos desde Google Cloud Storage a BigQuery

Para cargar datos desde GCS a BigQuery, puedes usar la interfaz web de BigQuery, la herramienta de línea de comandos bq o la API de BigQuery.

Usando la Interfaz Web de BigQuery

  1. Ve a la página de BigQuery en la Consola de Google Cloud.
  2. En el panel Explorador, selecciona el conjunto de datos donde deseas cargar los datos.
  3. Haz clic en el botón "Crear Tabla".
  4. En la sección "Fuente", selecciona "Google Cloud Storage" y proporciona el URI del archivo GCS.
  5. Configura los ajustes restantes y haz clic en "Crear Tabla".

Usando la Herramienta de Línea de Comandos bq

bq load --source_format=CSV mydataset.mytable gs://mybucket/myfile.csv
  • mydataset.mytable: El conjunto de datos y la tabla de destino en BigQuery.
  • gs://mybucket/myfile.csv: El URI del archivo en GCS.

Exportando Datos desde BigQuery a Google Cloud Storage

También puedes exportar datos desde BigQuery a GCS para respaldo o procesamiento adicional.

Usando la Interfaz Web de BigQuery

  1. Ve a la página de BigQuery en la Consola de Google Cloud.
  2. En el panel Explorador, selecciona la tabla que deseas exportar.
  3. Haz clic en el botón "Exportar".
  4. Proporciona el URI de destino en GCS y configura los ajustes de exportación.
  5. Haz clic en "Exportar".

Usando la Herramienta de Línea de Comandos bq

bq extract --destination_format=CSV mydataset.mytable gs://mybucket/myfile.csv
  • mydataset.mytable: El conjunto de datos y la tabla de origen en BigQuery.
  • gs://mybucket/myfile.csv: El URI de destino en GCS.

Google Cloud Pub/Sub

Google Cloud Pub/Sub es un servicio de mensajería para intercambiar datos de eventos entre aplicaciones y servicios.

Transmitiendo Datos desde Pub/Sub a BigQuery

Puedes crear una canalización de datos para transmitir datos desde Pub/Sub a BigQuery usando Dataflow.

Configurando la Canalización de Dataflow

  1. Crea un tema y una suscripción de Pub/Sub.
  2. Crea un trabajo de Dataflow para leer mensajes de la suscripción de Pub/Sub y escribirlos en BigQuery.

Código de ejemplo de una canalización de Dataflow en Python:

import apache_beam as beam
from apache_beam.options.pipeline_options import PipelineOptions
from apache_beam.io.gcp.bigquery import WriteToBigQuery

def run():
    options = PipelineOptions()
    p = beam.Pipeline(options=options)

    (p
     | 'Leer de Pub/Sub' >> beam.io.ReadFromPubSub(subscription='projects/myproject/subscriptions/mysubscription')
     | 'Transformar Datos' >> beam.Map(lambda x: {'field1': x['field1'], 'field2': x['field2']})
     | 'Escribir en BigQuery' >> WriteToBigQuery('myproject:mydataset.mytable'))

    p.run().wait_until_finish()

if __name__ == '__main__':
    run()
  • myproject:mydataset.mytable: El conjunto de datos y la tabla de destino en BigQuery.
  • projects/myproject/subscriptions/mysubscription: La suscripción de Pub/Sub.

Google Data Studio

Google Data Studio es una herramienta de visualización de datos que te permite crear paneles y reportes interactivos.

Conectando BigQuery a Google Data Studio

  1. Abre Google Data Studio.
  2. Haz clic en "Crear" y selecciona "Fuente de Datos".
  3. Elige "BigQuery" como el conector.
  4. Selecciona tu proyecto, conjunto de datos y tabla.
  5. Haz clic en "Conectar" y configura la fuente de datos.

Google Cloud Functions

Google Cloud Functions es un entorno de ejecución sin servidor para construir y conectar servicios en la nube.

Activando Trabajos de BigQuery con Cloud Functions

Puedes usar Cloud Functions para activar trabajos de BigQuery basados en eventos, como cargas de archivos a GCS.

Ejemplo de Cloud Function en Python:

from google.cloud import bigquery

def trigger_bigquery_job(event, context):
    client = bigquery.Client()
    query = """
    SELECT *
    FROM `myproject.mydataset.mytable`
    WHERE timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 1 DAY)
    """
    job = client.query(query)
    results = job.result()
    for row in results:
        print(row)
  • myproject.mydataset.mytable: El conjunto de datos y la tabla de destino en BigQuery.

Conclusión

Integrar BigQuery con otros servicios de Google Cloud puede agilizar tus flujos de trabajo de datos y mejorar tus capacidades analíticas. Ya sea que estés cargando datos desde Google Cloud Storage, transmitiendo datos con Pub/Sub, visualizando datos en Google Data Studio o automatizando tareas con Cloud Functions, estas integraciones pueden ayudarte a aprovechar todo el poder de Google Cloud.
