Creando Tu Primer Proyecto Cordova
Entendiendo la Estructura del Proyecto Cordova ►

En esta sección, te guiaremos a través del proceso de creación de tu primer proyecto Apache Cordova. Apache Cordova es un popular marco de desarrollo de aplicaciones móviles que te permite construir aplicaciones móviles usando HTML, CSS y JavaScript. Al final de esta sección, tendrás un proyecto básico de Cordova configurado y listo para un desarrollo posterior.

Prerrequisitos

Antes de comenzar, asegúrate de tener lo siguiente instalado en tu sistema:

  • Node.js y npm (Node Package Manager)
  • Cordova CLI (Command Line Interface)

Instalando Node.js y npm

Si aún no has instalado Node.js y npm, puedes descargarlos e instalarlos desde el sitio web oficial de Node.js.

Instalando Cordova CLI

Una vez que Node.js y npm estén instalados, puedes instalar Cordova CLI ejecutando el siguiente comando en tu terminal o símbolo del sistema:

npm install -g cordova

La bandera -g instala Cordova globalmente, haciéndolo accesible desde cualquier directorio en tu sistema.

Creando un Nuevo Proyecto Cordova

Para crear un nuevo proyecto Cordova, sigue estos pasos:

  1. Abre tu terminal o símbolo del sistema.
  2. Navega al directorio donde deseas crear tu proyecto.
  3. Ejecuta el siguiente comando:
cordova create MyApp com.example.myapp MyApp

Este comando crea un nuevo proyecto Cordova con los siguientes parámetros:

  • MyApp: El nombre del directorio para tu proyecto.
  • com.example.myapp: El identificador de dominio inverso para tu proyecto.
  • MyApp: El nombre de visualización de tu aplicación.

Estructura del Proyecto

Después de ejecutar el comando cordova create, tendrás un nuevo directorio llamado MyApp con la siguiente estructura:

MyApp/
├── config.xml
├── hooks/
├── platforms/
├── plugins/
└── www/
  • config.xml: El archivo de configuración para tu proyecto Cordova.
  • hooks/: Scripts que pueden ser activados en diferentes etapas del proceso de construcción de Cordova.
  • platforms/: Contiene código específico de la plataforma (por ejemplo, Android, iOS).
  • plugins/: Contiene los plugins de Cordova usados en tu proyecto.
  • www/: Contiene tus activos web (HTML, CSS, JavaScript).

Añadiendo Plataformas

Para construir tu proyecto Cordova para diferentes plataformas, necesitas añadir las plataformas deseadas. Por ejemplo, para añadir la plataforma Android, ejecuta el siguiente comando:

cordova platform add android

De manera similar, para añadir la plataforma iOS, ejecuta:

cordova platform add ios

Listando Plataformas Instaladas

Puedes listar las plataformas añadidas a tu proyecto ejecutando:

cordova platform ls

Construyendo el Proyecto

Una vez que hayas añadido las plataformas necesarias, puedes construir tu proyecto usando el siguiente comando:

cordova build

Este comando compila tu proyecto para todas las plataformas añadidas. También puedes construir para una plataforma específica especificando el nombre de la plataforma:

cordova build android

Ejecutando el Proyecto

Para ejecutar tu proyecto Cordova en un emulador o un dispositivo conectado, usa el siguiente comando:

cordova run android

Reemplaza android con ios si deseas ejecutar el proyecto en un dispositivo o emulador iOS.

Conclusión

En esta sección, has aprendido cómo crear tu primer proyecto Cordova, añadir plataformas, construir el proyecto y ejecutarlo en un emulador o dispositivo. Esta es la base para desarrollar aplicaciones móviles usando Apache Cordova. En las próximas secciones, profundizaremos en la personalización de tu proyecto, el uso de plugins y la optimización de tu aplicación para diferentes plataformas.
