Introducción

Apache Cordova proporciona una manera sencilla de acceder a las capacidades de geolocalización de un dispositivo. Esto te permite recuperar la ubicación geográfica del dispositivo, lo cual puede ser útil para una variedad de aplicaciones, tales como mapeo, navegación y servicios basados en la ubicación.

Configuración de la Geolocalización en Cordova

Instalación del Plugin de Geolocalización

Para usar la geolocalización en tu proyecto de Cordova, necesitas instalar el plugin de Geolocalización de Cordova. Esto se puede hacer usando el siguiente comando:

cordova plugin add cordova-plugin-geolocation

Añadiendo Permisos

Asegúrate de que tu aplicación tenga los permisos necesarios para acceder a la ubicación del dispositivo. Para Android, esto implica modificar el archivo AndroidManifest.xml, y para iOS, actualizar el archivo Info.plist.

Android

Añade los siguientes permisos a tu AndroidManifest.xml:

<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />

iOS

Añade las siguientes claves a tu Info.plist:

<key>NSLocationWhenInUseUsageDescription</key>
<string>Necesitamos tu ubicación para proporcionar mejores servicios.</string>
<key>NSLocationAlwaysUsageDescription</key>
<string>Necesitamos tu ubicación para proporcionar mejores servicios.</string>

Uso Básico de la Geolocalización

Obteniendo la Posición Actual

Para obtener la posición actual del dispositivo, usa el método navigator.geolocation.getCurrentPosition. Aquí hay un ejemplo básico:

document.addEventListener("deviceready", onDeviceReady, false);

function onDeviceReady() {
    navigator.geolocation.getCurrentPosition(onSuccess, onError);
}

function onSuccess(position) {
    console.log('Latitud: ' + position.coords.latitude);
    console.log('Longitud: ' + position.coords.longitude);
}

function onError(error) {
    console.error('Código de error: ' + error.code + '\nMensaje: ' + error.message);
}

Manejo de Errores

La función de callback onError maneja cualquier error que ocurra mientras se intenta recuperar la geolocalización. El objeto de error contiene un código y un mensaje.

  • error.code puede ser:
    • 1: Permiso denegado
    • 2: Posición no disponible
    • 3: Tiempo de espera agotado

Uso Avanzado de la Geolocalización

Observando la Posición

Para monitorear continuamente la posición del dispositivo, usa el método navigator.geolocation.watchPosition. Esto es útil para aplicaciones que necesitan rastrear el movimiento del usuario.

var watchID;

document.addEventListener("deviceready", onDeviceReady, false);

function onDeviceReady() {
    watchID = navigator.geolocation.watchPosition(onSuccess, onError, { timeout: 30000 });
}

function onSuccess(position) {
    console.log('Latitud: ' + position.coords.latitude);
    console.log('Longitud: ' + position.coords.longitude);
}

function onError(error) {
    console.error('Código de error: ' + error.code + '\nMensaje: ' + error.message);
}

function stopWatching() {
    if (watchID) {
        navigator.geolocation.clearWatch(watchID);
        watchID = null;
    }
}

Opciones de Geolocalización

Puedes personalizar el comportamiento de los métodos de geolocalización pasando un objeto de opciones. Las opciones incluyen:

  • enableHighAccuracy: Booleano, indica si la aplicación quiere los mejores resultados posibles.
  • timeout: Tiempo máximo (en milisegundos) que el dispositivo puede tardar en devolver una posición.
  • maximumAge: Edad máxima (en milisegundos) de una posición en caché que es aceptable devolver.

Ejemplo con opciones:

var options = {
    enableHighAccuracy: true,
    timeout: 5000,
    maximumAge: 0
};

navigator.geolocation.getCurrentPosition(onSuccess, onError, options);

Conclusión

Acceder a la geolocalización en Apache Cordova es sencillo con la ayuda del cordova-plugin-geolocation. Al entender el uso básico y avanzado de los métodos de geolocalización, puedes crear aplicaciones poderosas basadas en la ubicación. Siempre recuerda manejar los errores de manera adecuada y respetar la privacidad del usuario solicitando permisos apropiadamente.
