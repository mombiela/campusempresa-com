Introducción a las Máquinas de Estados Finitos

Las Máquinas de Estados Finitos (FSM, por sus siglas en inglés) son un modelo matemático utilizado para diseñar sistemas lógicos y de control. En el contexto de los videojuegos, las FSM son una herramienta poderosa para gestionar el comportamiento de los personajes no jugables (NPCs), permitiendo que estos reaccionen de manera coherente a diferentes situaciones del juego.

Conceptos Clave

  • Estado: Representa una condición o situación específica en la que puede encontrarse el sistema.
  • Transición: El cambio de un estado a otro, generalmente desencadenado por un evento o condición.
  • Evento: Una acción o condición que provoca una transición entre estados.
  • Acción: Una operación que se ejecuta cuando se entra, se sale o se permanece en un estado.

Estructura de una FSM

Una FSM se compone de:

  1. Conjunto de Estados: Todos los estados posibles en los que puede estar el sistema.
  2. Estado Inicial: El estado en el que comienza el sistema.
  3. Eventos/Condiciones: Factores que provocan transiciones entre estados.
  4. Transiciones: Las reglas que definen cómo y cuándo cambiar de un estado a otro.
  5. Acciones: Las operaciones que se ejecutan durante las transiciones o mientras se está en un estado.

Ejemplo de FSM en Videojuegos

Consideremos un NPC en un videojuego que puede estar en uno de los siguientes estados: Patrullando, Persiguiendo, Atacando, Huyendo.

Definición de Estados y Transiciones

Estado Evento/Condición Estado Siguiente Acción
Patrullando Ve al jugador Persiguiendo Comenzar a seguir al jugador
Persiguiendo Jugador fuera de vista Patrullando Volver a patrullar
Persiguiendo Cerca del jugador Atacando Iniciar ataque
Atacando Jugador fuera de alcance Persiguiendo Volver a seguir al jugador
Atacando Salud baja Huyendo Escapar del jugador
Huyendo Salud recuperada Patrullando Volver a patrullar

Ejemplo de Código en Python

A continuación, se presenta un ejemplo básico de cómo implementar una FSM para un NPC en Python:

class NPC:
    def __init__(self):
        self.state = 'Patrullando'
    
    def see_player(self):
        if self.state == 'Patrullando':
            self.state = 'Persiguiendo'
            print("Comenzar a seguir al jugador")
    
    def lose_sight_of_player(self):
        if self.state == 'Persiguiendo':
            self.state = 'Patrullando'
            print("Volver a patrullar")
    
    def reach_player(self):
        if self.state == 'Persiguiendo':
            self.state = 'Atacando'
            print("Iniciar ataque")
    
    def player_out_of_reach(self):
        if self.state == 'Atacando':
            self.state = 'Persiguiendo'
            print("Volver a seguir al jugador")
    
    def low_health(self):
        if self.state == 'Atacando':
            self.state = 'Huyendo'
            print("Escapar del jugador")
    
    def health_recovered(self):
        if self.state == 'Huyendo':
            self.state = 'Patrullando'
            print("Volver a patrullar")

# Ejemplo de uso
npc = NPC()
npc.see_player()  # Comenzar a seguir al jugador
npc.reach_player()  # Iniciar ataque
npc.low_health()  # Escapar del jugador
npc.health_recovered()  # Volver a patrullar

Ventajas y Desventajas de las FSM

Ventajas

  • Simplicidad: Fácil de entender y de implementar.
  • Claridad: Los estados y transiciones son explícitos, lo que facilita el mantenimiento y la depuración.
  • Determinismo: El comportamiento del sistema es predecible y reproducible.

Desventajas

  • Escalabilidad: Puede volverse complejo y difícil de manejar con un gran número de estados y transiciones.
  • Flexibilidad: Menos flexible para comportamientos muy dinámicos o adaptativos.

Conclusión

Las Máquinas de Estados Finitos son una herramienta fundamental en el desarrollo de IA para videojuegos, proporcionando una estructura clara y manejable para definir el comportamiento de los NPCs. Aunque tienen sus limitaciones, su simplicidad y claridad las hacen ideales para muchos escenarios comunes en el desarrollo de videojuegos. Con una comprensión sólida de las FSM, los desarrolladores pueden crear personajes más realistas y reactivos, mejorando la experiencia de juego para los usuarios.
