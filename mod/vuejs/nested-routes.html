Introducción a las rutas anidadas

Las rutas anidadas en Vue.js te permiten crear estructuras de navegación complejas y jerárquicas dentro de tu aplicación. Esto es particularmente útil para aplicaciones con múltiples capas de vistas, como paneles de control, perfiles de usuario o sitios de comercio electrónico con categorías y subcategorías de productos.

Conceptos clave

  • Rutas anidadas: Rutas dentro de rutas, permitiendo una estructura jerárquica.
  • Vista del enrutador: Un componente que renderiza el componente coincidente para una ruta dada.
  • Rutas hijas: Rutas que están anidadas dentro de una ruta principal.

Configuración de rutas anidadas

Para configurar rutas anidadas, necesitas definirlas en tu configuración de Vue Router. Aquí tienes una guía paso a paso:

  1. Instalar Vue Router: Si aún no lo has hecho, instala Vue Router en tu proyecto de Vue.js.

    npm install vue-router

  2. Definir rutas: Crea un archivo router.js (o modifica tu configuración de enrutador existente) para definir tus rutas anidadas.

Ejemplo: Rutas anidadas básicas

Considera una aplicación simple con un componente User que tiene rutas anidadas para Profile y Posts.

// router.js
import Vue from 'vue';
import Router from 'vue-router';
import Home from './components/Home.vue';
import User from './components/User.vue';
import UserProfile from './components/UserProfile.vue';
import UserPosts from './components/UserPosts.vue';

Vue.use(Router);

const routes = [
  { path: '/', component: Home },
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      }
    ]
  }
];

const router = new Router({
  routes
});

export default router;

Explicación

  • Ruta principal: /user/:id - Esta ruta renderizará el componente User.
  • Rutas hijas: /user/:id/profile y /user/:id/posts - Estas rutas renderizarán los componentes UserProfile y UserPosts respectivamente, dentro del componente User.

Vista del enrutador en el componente principal

Para renderizar los componentes hijos, necesitas incluir un <router-view> en el componente principal.

<!-- User.vue -->
<template>
  <div>
    <h2>Usuario {{ $route.params.id }}</h2>
    <router-view></router-view> <!-- Aquí se renderizarán los componentes hijos -->
  </div>
</template>

<script>
export default {
  name: 'User'
};
</script>

Rutas anidadas avanzadas

Para aplicaciones más complejas, podrías necesitar rutas anidadas más profundas o rutas anidadas dinámicas.

Ejemplo: Rutas anidadas más profundas

Imagina una aplicación donde un usuario tiene múltiples proyectos, y cada proyecto tiene múltiples tareas.

// router.js
import Vue from 'vue';
import Router from 'vue-router';
import Home from './components/Home.vue';
import User from './components/User.vue';
import UserProfile from './components/UserProfile.vue';
import UserPosts from './components/UserPosts.vue';
import Project from './components/Project.vue';
import Task from './components/Task.vue';

Vue.use(Router);

const routes = [
  { path: '/', component: Home },
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      },
      {
        path: 'project/:projectId',
        component: Project,
        children: [
          {
            path: 'task/:taskId',
            component: Task
          }
        ]
      }
    ]
  }
];

const router = new Router({
  routes
});

export default router;

Explicación

  • Rutas anidadas más profundas: El componente Project está anidado dentro del componente User, y el componente Task está anidado dentro del componente Project.
  • Segmentos dinámicos: Tanto projectId como taskId son segmentos dinámicos, permitiendo un enrutamiento dinámico basado en los IDs de proyecto y tarea.

Vista del enrutador en componentes anidados

Asegúrate de que cada componente principal incluya un <router-view> para renderizar sus componentes hijos.

<!-- Project.vue -->
<template>
  <div>
    <h3>Proyecto {{ $route.params.projectId }}</h3>
    <router-view></router-view> <!-- Aquí se renderizará el componente Task -->
  </div>
</template>

<script>
export default {
  name: 'Project'
};
</script>

Resumen

Las rutas anidadas en Vue.js proporcionan una forma poderosa de estructurar la navegación de tu aplicación. Al definir rutas dentro de rutas, puedes crear vistas complejas y jerárquicas que son fáciles de gestionar y navegar. Recuerda:

  • Definir rutas anidadas en tu configuración de enrutador.
  • Usar <router-view> en componentes principales para renderizar componentes hijos.
  • Utilizar segmentos dinámicos para un enrutamiento flexible y dinámico.

Al dominar las rutas anidadas, puedes construir aplicaciones sofisticadas y fáciles de usar con Vue.js.
