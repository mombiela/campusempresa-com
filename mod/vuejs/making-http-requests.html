En esta sección, cubriremos cómo realizar solicitudes HTTP en Vue.js. Esta es una habilidad crucial para interactuar con APIs y obtener datos para tus aplicaciones. Comenzaremos con lo básico y avanzaremos hacia conceptos más avanzados.

Introducción a las solicitudes HTTP en Vue.js

Las solicitudes HTTP se utilizan para comunicarse con servidores para obtener o enviar datos. En Vue.js, comúnmente usamos bibliotecas como Axios o la API Fetch para manejar estas solicitudes.

Conceptos clave

  • Métodos HTTP: GET, POST, PUT, DELETE, etc.
  • Endpoints: URLs a las que se envían las solicitudes.
  • Encabezados: Información adicional enviada con la solicitud.
  • Cuerpo: Datos enviados con solicitudes POST, PUT, DELETE.

Uso de Axios para solicitudes HTTP

Axios es un cliente HTTP basado en promesas para JavaScript, que funciona perfectamente con Vue.js.

Instalación de Axios

Primero, necesitas instalar Axios en tu proyecto Vue.js:

npm install axios

Realización de una solicitud GET

Aquí se muestra cómo realizar una simple solicitud GET para obtener datos de una API:

<template>
  <div>
    <h1>Datos de la API</h1>
    <ul>
      <li v-for="item in items" :key="item.id">{{ item.name }}</li>
    </ul>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  data() {
    return {
      items: []
    };
  },
  created() {
    axios.get('https://api.example.com/items')
      .then(response => {
        this.items = response.data;
      })
      .catch(error => {
        console.error('¡Hubo un error!', error);
      });
  }
};
</script>

Realización de una solicitud POST

Para enviar datos a una API, puedes usar una solicitud POST:

<template>
  <div>
    <h1>Agregar nuevo ítem</h1>
    <form @submit.prevent="addItem">
      <input v-model="newItem" placeholder="Ingrese el nombre del ítem" />
      <button type="submit">Agregar</button>
    </form>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  data() {
    return {
      newItem: ''
    };
  },
  methods: {
    addItem() {
      axios.post('https://api.example.com/items', { name: this.newItem })
        .then(response => {
          console.log('Ítem agregado:', response.data);
        })
        .catch(error => {
          console.error('¡Hubo un error!', error);
        });
    }
  }
};
</script>

Manejo de errores

Es importante manejar los errores de manera adecuada en tu aplicación. Axios proporciona una forma de capturar errores usando .catch().

axios.get('https://api.example.com/items')
  .then(response => {
    this.items = response.data;
  })
  .catch(error => {
    console.error('¡Hubo un error!', error);
    this.errorMessage = 'Error al cargar los datos';
  });

Uso de la API Fetch

La API Fetch es una API integrada en JavaScript para realizar solicitudes HTTP. Devuelve una promesa que se resuelve en el objeto Response.

Realización de una solicitud GET con Fetch

<template>
  <div>
    <h1>Datos de la API</h1>
    <ul>
      <li v-for="item in items" :key="item.id">{{ item.name }}</li>
    </ul>
  </div>
</template>

<script>
export default {
  data() {
    return {
      items: []
    };
  },
  created() {
    fetch('https://api.example.com/items')
      .then(response => response.json())
      .then(data => {
        this.items = data;
      })
      .catch(error => {
        console.error('¡Hubo un error!', error);
      });
  }
};
</script>

Realización de una solicitud POST con Fetch

<template>
  <div>
    <h1>Agregar nuevo ítem</h1>
    <form @submit.prevent="addItem">
      <input v-model="newItem" placeholder="Ingrese el nombre del ítem" />
      <button type="submit">Agregar</button>
    </form>
  </div>
</template>

<script>
export default {
  data() {
    return {
      newItem: ''
    };
  },
  methods: {
    addItem() {
      fetch('https://api.example.com/items', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({ name: this.newItem })
      })
      .then(response => response.json())
      .then(data => {
        console.log('Ítem agregado:', data);
      })
      .catch(error => {
        console.error('¡Hubo un error!', error);
      });
    }
  }
};
</script>

Conceptos avanzados

Interceptores en Axios

Los interceptores te permiten ejecutar tu código o modificar la solicitud/respuesta antes de que sea manejada por then o catch.

axios.interceptors.request.use(config => {
  // Hacer algo antes de que se envíe la solicitud
  config.headers.Authorization = 'Bearer token';
  return config;
}, error => {
  // Hacer algo con el error de la solicitud
  return Promise.reject(error);
});

axios.interceptors.response.use(response => {
  // Hacer algo con los datos de la respuesta
  return response;
}, error => {
  // Hacer algo con el error de la respuesta
  return Promise.reject(error);
});

Uso de Async/Await

La sintaxis async/await puede hacer que tu código sea más limpio y fácil de leer.

<template>
  <div>
    <h1>Datos de la API</h1>
    <ul>
      <li v-for="item in items" :key="item.id">{{ item.name }}</li>
    </ul>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  data() {
    return {
      items: []
    };
  },
  async created() {
    try {
      const response = await axios.get('https://api.example.com/items');
      this.items = response.data;
    } catch (error) {
      console.error('¡Hubo un error!', error);
    }
  }
};
</script>
