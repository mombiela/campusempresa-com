Introducción a Vuex

Vuex es un patrón de gestión de estado + biblioteca para aplicaciones Vue.js. Sirve como una tienda centralizada para todos los componentes en una aplicación, con reglas que aseguran que el estado solo pueda ser mutado de una manera predecible. Vuex se usa a menudo en aplicaciones de mediana a gran escala para gestionar el estado de manera más eficiente.

Configuración de Vuex

Antes de usar Vuex en tus componentes, necesitas configurarlo en tu proyecto Vue.js.

  1. Instalar Vuex:

    npm install vuex

  2. Crear una Tienda Vuex:

    // store.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
    state: {
        count: 0
    },
    mutations: {
        increment(state) {
            state.count++;
        }
    },
    actions: {
        increment({ commit }) {
            commit('increment');
        }
    },
    getters: {
        count: state => state.count
    }
});

  3. Integrar la Tienda con la Instancia de Vue:

    // main.js
import Vue from 'vue';
import App from './App.vue';
import store from './store';

new Vue({
    store,
    render: h => h(App)
}).$mount('#app');

Accediendo al Estado en Componentes

Para acceder al estado en tus componentes Vue, puedes usar el helper mapState de Vuex.

  1. Usando mapState:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
    </div>
</template>

<script>
import { mapState } from 'vuex';

export default {
    computed: {
        ...mapState(['count'])
    }
};
</script>

  2. Acceso Directo:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ $store.state.count }}</p>
    </div>
</template>

<script>
export default {
    computed: {
        count() {
            return this.$store.state.count;
        }
    }
};
</script>

Cometiendo Mutaciones

Para cambiar el estado, necesitas cometer mutaciones. Esto asegura que los cambios de estado sean rastreables y predecibles.

  1. Usando mapMutations:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">Increment</button>
    </div>
</template>

<script>
import { mapState, mapMutations } from 'vuex';

export default {
    computed: {
        ...mapState(['count'])
    },
    methods: {
        ...mapMutations(['increment'])
    }
};
</script>

  2. Cometido Directo:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">Increment</button>
    </div>
</template>

<script>
export default {
    computed: {
        count() {
            return this.$store.state.count;
        }
    },
    methods: {
        increment() {
            this.$store.commit('increment');
        }
    }
};
</script>

Despachando Acciones

Las acciones son similares a las mutaciones, pero en lugar de mutar el estado directamente, las acciones cometen mutaciones. Las acciones pueden contener operaciones asíncronas.

  1. Usando mapActions:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">Increment</button>
    </div>
</template>

<script>
import { mapState, mapActions } from 'vuex';

export default {
    computed: {
        ...mapState(['count'])
    },
    methods: {
        ...mapActions(['increment'])
    }
};
</script>

  2. Despacho Directo:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">Increment</button>
    </div>
</template>

<script>
export default {
    computed: {
        count() {
            return this.$store.state.count;
        }
    },
    methods: {
        increment() {
            this.$store.dispatch('increment');
        }
    }
};
</script>

Usando Getters

Los getters se usan para calcular el estado derivado basado en el estado de la tienda. Son similares a las propiedades computadas para las tiendas.

  1. Usando mapGetters:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
    </div>
</template>

<script>
import { mapGetters } from 'vuex';

export default {
    computed: {
        ...mapGetters(['count'])
    }
};
</script>

  2. Acceso Directo:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
    </div>
</template>

<script>
export default {
    computed: {
        count() {
            return this.$store.getters.count;
        }
    }
};
</script>

Conclusión

Usar Vuex en tus componentes Vue.js te permite gestionar el estado de una manera más estructurada y predecible. Al aprovechar el estado, las mutaciones, las acciones y los getters de Vuex, puedes crear aplicaciones que sean más fáciles de depurar y mantener. Recuerda usar los helpers proporcionados (mapState, mapMutations, mapActions y mapGetters) para mantener tus componentes limpios y concisos.
