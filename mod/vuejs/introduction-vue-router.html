Introducción a Vue Router
Configuración de Vue Router ►

Vue Router es el enrutador oficial para Vue.js, que permite a los desarrolladores crear aplicaciones de una sola página con múltiples vistas y navegación. Esta sección te introducirá a los conceptos básicos de Vue Router, su instalación y cómo usarlo para gestionar rutas en tus aplicaciones Vue.js.

¿Qué es Vue Router?

Vue Router es una biblioteca que se integra con Vue.js para proporcionar capacidades de enrutamiento. Te permite:

  • Definir rutas y asignarlas a componentes.
  • Navegar entre diferentes vistas.
  • Manejar enrutamiento dinámico y rutas anidadas.
  • Gestionar guardias de navegación y transiciones de rutas.

Instalando Vue Router

Para comenzar con Vue Router, necesitas instalarlo en tu proyecto Vue.js. Puedes hacerlo usando npm o yarn.

Usando npm:

npm install vue-router

Usando yarn:

yarn add vue-router

Configurando Vue Router

Una vez instalado, necesitas configurar Vue Router en tu aplicación Vue.js. Esto implica crear una instancia del enrutador y definir tus rutas.

Configuración Básica

  1. Crea un archivo router.js:
// src/router.js
import Vue from 'vue';
import Router from 'vue-router';
import Home from './components/Home.vue';
import About from './components/About.vue';

Vue.use(Router);

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About }
];

const router = new Router({
  routes
});

export default router;
  1. Integra el enrutador con tu instancia de Vue:
// src/main.js
import Vue from 'vue';
import App from './App.vue';
import router from './router';

new Vue({
  render: h => h(App),
  router
}).$mount('#app');

Definiendo Rutas

Las rutas se definen como objetos dentro de un array. Cada objeto de ruta típicamente contiene:

  • path: La ruta URL.
  • component: El componente a renderizar cuando la ruta coincide.

Ejemplo:

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
  { path: '/contact', component: Contact }
];

Navegando Entre Rutas

Vue Router proporciona varias formas de navegar entre rutas:

  • Usando el componente <router-link>:
<template>
  <div>
    <router-link to="/">Home</router-link>
    <router-link to="/about">About</router-link>
  </div>
</template>
  • Programáticamente usando this.$router.push:
methods: {
  goToAbout() {
    this.$router.push('/about');
  }
}

Enrutamiento Dinámico

El enrutamiento dinámico te permite definir rutas con segmentos dinámicos que pueden coincidir con múltiples rutas.

Ejemplo:

const routes = [
  { path: '/user/:id', component: User }
];

En el componente User, puedes acceder al segmento dinámico usando this.$route.params.id.

Ejemplo de Componente:

<template>
  <div>User ID: {{ $route.params.id }}</div>
</template>

<script>
export default {
  name: 'User'
};
</script>

Rutas Anidadas

Las rutas anidadas te permiten crear rutas dentro de rutas, lo cual es útil para crear diseños complejos.

Ejemplo:

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      }
    ]
  }
];

Guardias de Navegación

Las guardias de navegación se usan para controlar el acceso a ciertas rutas. Pueden definirse globalmente, por ruta o en el componente.

Ejemplo de Guardia Global:

router.beforeEach((to, from, next) => {
  if (to.path === '/protected' && !isAuthenticated()) {
    next('/login');
  } else {
    next();
  }
});

Resumen

En esta sección, aprendiste:

  • Qué es Vue Router y su propósito.
  • Cómo instalar y configurar Vue Router en un proyecto Vue.js.
  • Cómo definir y navegar entre rutas.
  • Los conceptos básicos del enrutamiento dinámico y las rutas anidadas.
  • Cómo usar guardias de navegación para controlar el acceso a las rutas.

Vue Router es una herramienta poderosa que mejora las capacidades de las aplicaciones Vue.js al proporcionar una navegación y enrutamiento fluidos. Entender estos conceptos básicos te ayudará a construir aplicaciones de una sola página más complejas y amigables para el usuario.
