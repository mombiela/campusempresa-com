Mocking y Stubbing en Vue.js
Optimizando Aplicaciones Vue.js ►

Mocking y stubbing son técnicas esenciales en las pruebas unitarias que ayudan a aislar el componente o función que se está probando al reemplazar dependencias con sustitutos controlados. Esto asegura que las pruebas sean confiables y se centren en la funcionalidad específica bajo prueba.

¿Qué es Mocking?

Mocking implica crear objetos que simulan el comportamiento de objetos reales. Estos objetos simulados pueden ser programados para devolver valores específicos o realizar ciertas acciones cuando son llamados durante una prueba.

Conceptos Clave de Mocking

  • Objetos Simulados: Objetos simulados que imitan el comportamiento de objetos reales.
  • Verificación de Comportamiento: Asegurarse de que los objetos simulados se usen como se espera.
  • Aislamiento: Probar componentes en aislamiento reemplazando dependencias con simulaciones.

¿Qué es Stubbing?

Stubbing es una técnica utilizada para reemplazar una función o método con un "stub" que devuelve resultados predefinidos. Esto es particularmente útil cuando se desea controlar la salida de una función durante una prueba.

Conceptos Clave de Stubbing

  • Funciones Stub: Funciones que devuelven valores predefinidos.
  • Entorno Controlado: Crear un entorno predecible para las pruebas.
  • Simplificación: Simplificar dependencias complejas utilizando stubs.

Mocking y Stubbing en Vue.js

En Vue.js, mocking y stubbing se utilizan comúnmente en pruebas unitarias para aislar componentes y asegurar que las pruebas sean enfocadas y confiables. La biblioteca vue-test-utils, junto con frameworks de prueba como Jest, proporciona herramientas para simular y reemplazar fácilmente dependencias.

Mocking con `vue-test-utils` y Jest

Ejemplo: Simulando un Servicio

Supongamos que tenemos un servicio que obtiene datos de usuario:

// userService.js
export const getUser = async (id) => {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
};

Podemos simular este servicio en nuestras pruebas:

// UserComponent.vue
<template>
  <div>{{ user.name }}</div>
</template>

<script>
import { getUser } from './userService';

export default {
  data() {
    return {
      user: {},
    };
  },
  async created() {
    this.user = await getUser(1);
  },
};
</script>
// UserComponent.test.js
import { shallowMount } from '@vue/test-utils';
import UserComponent from './UserComponent';
import * as userService from './userService';

jest.mock('./userService');

describe('UserComponent', () => {
  it('fetches and displays user data', async () => {
    userService.getUser.mockResolvedValue({ name: 'John Doe' });

    const wrapper = shallowMount(UserComponent);
    await wrapper.vm.$nextTick();

    expect(wrapper.text()).toContain('John Doe');
  });
});

Stubbing con `vue-test-utils`

Ejemplo: Reemplazando un Componente Hijo

Supongamos que tenemos un componente padre que usa un componente hijo:

// ChildComponent.vue
<template>
  <div>Child Component</div>
</template>
// ParentComponent.vue
<template>
  <div>
    <ChildComponent />
  </div>
</template>

<script>
import ChildComponent from './ChildComponent';

export default {
  components: {
    ChildComponent,
  },
};
</script>

Podemos reemplazar el componente hijo en nuestras pruebas:

// ParentComponent.test.js
import { shallowMount } from '@vue/test-utils';
import ParentComponent from './ParentComponent';

describe('ParentComponent', () => {
  it('renders the child component', () => {
    const wrapper = shallowMount(ParentComponent, {
      stubs: {
        ChildComponent: true,
      },
    });

    expect(wrapper.html()).toContain('<childcomponent-stub></childcomponent-stub>');
  });
});

Conclusión

Mocking y stubbing son técnicas poderosas en las pruebas de Vue.js que permiten a los desarrolladores aislar componentes y controlar dependencias. Al usar herramientas como vue-test-utils y Jest, puedes crear pruebas unitarias confiables y enfocadas que aseguren que tus componentes funcionen como se espera. Entender y aplicar estas técnicas te ayudará a escribir mejores pruebas y mantener un alto nivel de calidad en el código de tus aplicaciones Vue.js.
