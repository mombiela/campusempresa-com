Introducción a los Guardias de Navegación

Los Guardias de Navegación son una característica esencial en Vue.js que te permiten controlar la navegación de tu aplicación. Proporcionan hooks que pueden ser utilizados para proteger rutas, realizar acciones antes o después de la navegación, e incluso cancelar la navegación basándose en ciertas condiciones.

Tipos de Guardias de Navegación

Vue.js ofrece varios tipos de guardias de navegación:

  • Guardias Globales: Se aplican a todas las rutas en la aplicación.
  • Guardias por Ruta: Se aplican a rutas específicas.
  • Guardias en Componentes: Se aplican a componentes específicos.

Guardias Globales

Los guardias globales se definen a nivel del router y se aplican a todas las rutas.

Guardia `beforeEach`

La guardia beforeEach se llama antes de cada navegación. Puede ser utilizada para realizar comprobaciones o obtener datos antes de permitir que la navegación proceda.

const router = new VueRouter({
  routes: [
    // tus rutas aquí
  ]
});

router.beforeEach((to, from, next) => {
  // Realizar alguna lógica
  if (to.meta.requiresAuth && !isAuthenticated()) {
    next('/login');
  } else {
    next();
  }
});
  • to: El objeto de ruta de destino al que se está navegando.
  • from: La ruta actual desde la que se está navegando.
  • next: Una función que debe ser llamada para resolver el hook.

Guardia `beforeResolve`

La guardia beforeResolve es similar a beforeEach pero se llama justo antes de que la navegación sea confirmada, después de que todas las guardias beforeEach y los componentes de ruta asíncronos hayan sido resueltos.

router.beforeResolve((to, from, next) => {
  // Realizar alguna lógica
  next();
});

Guardia `afterEach`

La guardia afterEach se llama después de cada navegación. No afecta la navegación en sí y se utiliza típicamente para tareas de análisis o limpieza.

router.afterEach((to, from) => {
  // Realizar alguna lógica
  console.log(`Navegaste a ${to.path} desde ${from.path}`);
});

Guardias por Ruta

Las guardias por ruta se definen directamente en la configuración de la ruta y se aplican solo a rutas específicas.

const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    beforeEnter: (to, from, next) => {
      // Realizar alguna lógica
      if (!isAuthenticated()) {
        next('/login');
      } else {
        next();
      }
    }
  }
];

Guardias en Componentes

Las guardias en componentes se definen dentro del propio componente y se aplican solo cuando se está navegando hacia o desde el componente.

`beforeRouteEnter`

Se llama antes de que la ruta que renderiza este componente sea confirmada. No tiene acceso a this porque la instancia del componente aún no ha sido creada.

export default {
  beforeRouteEnter(to, from, next) {
    // Realizar alguna lógica
    next(vm => {
      // Acceder a la instancia del componente a través de `vm`
    });
  }
};

`beforeRouteUpdate`

Se llama cuando la ruta que renderiza este componente ha cambiado pero el componente se reutiliza en la nueva ruta.

export default {
  beforeRouteUpdate(to, from, next) {
    // Realizar alguna lógica
    next();
  }
};

`beforeRouteLeave`

Se llama cuando la ruta que renderiza este componente está a punto de ser abandonada.

export default {
  beforeRouteLeave(to, from, next) {
    // Realizar alguna lógica
    next();
  }
};

Resumen

Los Guardias de Navegación en Vue.js proporcionan una forma poderosa de controlar el flujo de tu aplicación. Utilizando guardias globales, por ruta y en componentes, puedes asegurarte de que tu aplicación se comporte como se espera durante la navegación. Ya sea que necesites comprobar la autenticación, obtener datos o realizar tareas de limpieza, los guardias de navegación ofrecen una solución flexible para manejar estos escenarios de manera efectiva.
