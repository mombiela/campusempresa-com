Pruebas Automatizadas en Android Studio
Perfilado y Ajuste de Rendimiento ►

Las pruebas automatizadas son un aspecto crucial del desarrollo de Android, asegurando que tu aplicación funcione como se espera y se mantenga estable con el tiempo. Este tema te guiará desde los conceptos básicos hasta los avanzados de las pruebas automatizadas en Android Studio.

Introducción a las Pruebas Automatizadas

Las pruebas automatizadas implican el uso de herramientas de software para ejecutar pruebas en tu aplicación automáticamente. Esto ayuda a:

  • Reducir el esfuerzo de pruebas manuales.
  • Asegurar una ejecución de pruebas consistente.
  • Identificar rápidamente errores y problemas.

Tipos de Pruebas Automatizadas

Hay tres tipos principales de pruebas automatizadas en el desarrollo de Android:

  1. Pruebas Unitarias: Prueban componentes o funciones individuales.
  2. Pruebas de Integración: Prueban la interacción entre diferentes componentes.
  3. Pruebas de UI: Prueban la interfaz de usuario y las interacciones del usuario.

Configuración de Pruebas en Android Studio

Antes de escribir pruebas, necesitas configurar tu proyecto de Android Studio para las pruebas.

Añadiendo Dependencias de Pruebas

Añade las siguientes dependencias a tu archivo build.gradle:

dependencies {
    // Dependencias de pruebas unitarias
    testImplementation 'junit:junit:4.13.2'
    testImplementation 'org.mockito:mockito-core:3.9.0'

    // Dependencias de pruebas de instrumentación de Android
    androidTestImplementation 'androidx.test.ext:junit:1.1.3'
    androidTestImplementation 'androidx.test.espresso:espresso-core:3.4.0'
}

Escribiendo Pruebas Unitarias

Las pruebas unitarias se centran en probar funciones o métodos individuales.

Ejemplo: Probando una Función de Utilidad

Considera una simple función de utilidad que suma dos números:

// Archivo: Utils.kt
fun add(a: Int, b: Int): Int {
    return a + b
}

Para probar esta función, crea una clase de prueba:

// Archivo: UtilsTest.kt
import org.junit.Assert.*
import org.junit.Test

class UtilsTest {

    @Test
    fun testAdd() {
        val result = add(2, 3)
        assertEquals(5, result)
    }
}

Ejecutando Pruebas Unitarias

Para ejecutar tus pruebas unitarias:

  • Haz clic derecho en la clase o método de prueba.
  • Selecciona "Run 'testAdd'".

Escribiendo Pruebas de Integración

Las pruebas de integración aseguran que diferentes partes de tu aplicación funcionen correctamente juntas.

Ejemplo: Probando un Repositorio

Considera un repositorio que obtiene datos de una red:

// Archivo: UserRepository.kt
class UserRepository(private val apiService: ApiService) {

    fun getUser(userId: String): User {
        return apiService.getUser(userId)
    }
}

Para probar esto, puedes usar un marco de trabajo de simulación como Mockito:

// Archivo: UserRepositoryTest.kt
import org.junit.Assert.*
import org.junit.Test
import org.mockito.Mockito.*

class UserRepositoryTest {

    @Test
    fun testGetUser() {
        val apiService = mock(ApiService::class.java)
        val user = User("1", "John Doe")
        `when`(apiService.getUser("1")).thenReturn(user)

        val userRepository = UserRepository(apiService)
        val result = userRepository.getUser("1")

        assertEquals(user, result)
    }
}

Escribiendo Pruebas de UI

Las pruebas de UI interactúan con la interfaz de usuario para asegurar que se comporte como se espera.

Ejemplo: Probando una Pantalla de Inicio de Sesión

Considera una simple pantalla de inicio de sesión con un campo de nombre de usuario y contraseña:

<!-- Archivo: activity_login.xml -->
<LinearLayout ...>
    <EditText
        android:id="@+id/username"
        ... />
    <EditText
        android:id="@+id/password"
        ... />
    <Button
        android:id="@+id/loginButton"
        ... />
</LinearLayout>

Para probar esta pantalla, usa Espresso:

// Archivo: LoginActivityTest.kt
import androidx.test.ext.junit.runners.AndroidJUnit4
import androidx.test.rule.ActivityTestRule
import androidx.test.espresso.Espresso.*
import androidx.test.espresso.assertion.ViewAssertions.*
import androidx.test.espresso.matcher.ViewMatchers.*
import org.junit.Rule
import org.junit.Test
import org.junit.runner.RunWith

@RunWith(AndroidJUnit4::class)
class LoginActivityTest {

    @Rule @JvmField
    val activityRule = ActivityTestRule(LoginActivity::class.java)

    @Test
    fun testLoginButton() {
        onView(withId(R.id.username)).perform(typeText("user"))
        onView(withId(R.id.password)).perform(typeText("password"))
        onView(withId(R.id.loginButton)).perform(click())

        // Añade afirmaciones para verificar el resultado
    }
}

Conclusión

Las pruebas automatizadas son una práctica esencial en el desarrollo de Android que ayuda a mantener la calidad del código y la estabilidad de la aplicación. Al escribir pruebas unitarias, pruebas de integración y pruebas de UI, puedes asegurar que tu aplicación se comporte como se espera bajo diversas condiciones. Comienza con pruebas simples y gradualmente avanza a escenarios más complejos a medida que te familiarices con los marcos de pruebas y herramientas disponibles en Android Studio.
