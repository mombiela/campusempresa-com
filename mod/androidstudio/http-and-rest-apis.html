Introducción a HTTP y APIs REST

En esta sección, cubriremos los conceptos básicos de HTTP y APIs REST, que son esenciales para comunicarse con servicios web en aplicaciones Android.

  • HTTP (Protocolo de Transferencia de Hipertexto): La base de cualquier intercambio de datos en la Web y un protocolo utilizado para transmitir solicitudes de hipertexto e información entre servidores y clientes.
  • REST (Transferencia de Estado Representacional): Un estilo arquitectónico para diseñar aplicaciones en red. Las APIs RESTful utilizan solicitudes HTTP para realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar).

Entendiendo HTTP

HTTP es un protocolo utilizado para enviar y recibir datos a través de la web. Aquí hay algunos conceptos clave:

  • Métodos HTTP:

    • GET: Recuperar datos de un servidor.
    • POST: Enviar datos a un servidor para crear un recurso.
    • PUT: Actualizar un recurso existente en el servidor.
    • DELETE: Eliminar un recurso del servidor.

  • Códigos de Estado HTTP:

    • 200 OK: La solicitud fue exitosa.
    • 201 Created: La solicitud fue exitosa y se creó un recurso.
    • 400 Bad Request: La solicitud no pudo ser entendida o faltaban parámetros requeridos.
    • 401 Unauthorized: La autenticación falló o el usuario no tiene permisos para la acción deseada.
    • 404 Not Found: No se pudo encontrar el recurso solicitado.
    • 500 Internal Server Error: Ocurrió un error en el servidor.

Introducción a las APIs REST

Las APIs REST te permiten interactuar con servicios web utilizando métodos HTTP. Son sin estado y pueden ser utilizadas para realizar operaciones CRUD.

  • Recursos y URIs: En REST, los recursos son identificados por URIs (Identificadores Uniformes de Recursos).

    • Ejemplo: https://api.example.com/users podría representar una colección de usuarios.

  • JSON (Notación de Objetos de JavaScript): Un formato de intercambio de datos ligero que es fácil de leer y escribir para los humanos y fácil de analizar y generar para las máquinas.

Realizando Solicitudes HTTP en Android

Para realizar solicitudes HTTP en Android, puedes usar bibliotecas como HttpURLConnection, OkHttp o Retrofit. Aquí, nos enfocaremos en Retrofit debido a su simplicidad y facilidad de uso.

Configurando Retrofit

Primero, agrega la dependencia de Retrofit a tu archivo build.gradle:

dependencies {
    implementation 'com.squareup.retrofit2:retrofit:2.9.0'
    implementation 'com.squareup.retrofit2:converter-gson:2.9.0'
}

Creando una Instancia de Retrofit

Crea una instancia de Retrofit para definir la URL base y la fábrica de convertidores:

Retrofit retrofit = new Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .addConverterFactory(GsonConverterFactory.create())
    .build();

Definiendo Endpoints de la API

Define los endpoints usando una interfaz:

public interface ApiService {
    @GET("users")
    Call<List<User>> getUsers();

    @POST("users")
    Call<User> createUser(@Body User user);
}

Realizando una Solicitud

Crea una instancia del servicio API y realiza una solicitud:

ApiService apiService = retrofit.create(ApiService.class);

// Realizando una solicitud GET
Call<List<User>> call = apiService.getUsers();
call.enqueue(new Callback<List<User>>() {
    @Override
    public void onResponse(Call<List<User>> call, Response<List<User>> response) {
        if (response.isSuccessful()) {
            List<User> users = response.body();
            // Manejar la lista de usuarios
        }
    }

    @Override
    public void onFailure(Call<List<User>> call, Throwable t) {
        // Manejar el fallo
    }
});

Manejo de Respuestas JSON

Usa la biblioteca Gson para analizar respuestas JSON en objetos Java. Asegúrate de que tus clases modelo coincidan con la estructura JSON:

public class User {
    private int id;
    private String name;
    private String email;

    // Getters y setters
}

Conclusión

En este tema, cubrimos los conceptos básicos de HTTP y APIs REST, incluyendo métodos HTTP, códigos de estado y cómo realizar solicitudes HTTP en Android usando Retrofit. Entender estos conceptos es crucial para integrar servicios web en tus aplicaciones Android. Practicando los ejemplos y ejercicios proporcionados, podrás comunicarte efectivamente con APIs RESTful y manejar datos JSON en tus aplicaciones.
