Introducción a JobScheduler

JobScheduler es una poderosa API introducida en Android 5.0 (Lollipop) que te permite programar tareas para que se ejecuten en momentos específicos o bajo ciertas condiciones. Es particularmente útil para tareas en segundo plano que necesitan ejecutarse periódicamente o cuando se cumplen ciertas condiciones, como la disponibilidad de red o la carga del dispositivo.

Conceptos Clave

  • JobService: Un componente que ejecuta el trabajo programado.
  • JobInfo: Contiene los criterios para activar el trabajo.
  • JobScheduler: El servicio del sistema que programa y gestiona los trabajos.

Configuración de JobScheduler

Añadiendo Permisos

Para usar JobScheduler, necesitas añadir los permisos apropiados a tu archivo AndroidManifest.xml:

<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED"/>

Creando un JobService

Un JobService es un servicio que se ejecuta en segundo plano cuando se activa el trabajo. Aquí tienes un ejemplo de un simple JobService:

import android.app.job.JobParameters;
import android.app.job.JobService;
import android.util.Log;

public class MyJobService extends JobService {
    private static final String TAG = "MyJobService";

    @Override
    public boolean onStartJob(JobParameters params) {
        Log.d(TAG, "Job started");
        // Realiza tu tarea en segundo plano aquí
        jobFinished(params, false);
        return true; // Devuelve true si el trabajo necesita seguir ejecutándose
    }

    @Override
    public boolean onStopJob(JobParameters params) {
        Log.d(TAG, "Job stopped");
        return false; // Devuelve true para reprogramar el trabajo
    }
}

Programando un Trabajo

Para programar un trabajo, necesitas crear una instancia de JobInfo y pasarla al JobScheduler. Aquí tienes un ejemplo:

import android.app.job.JobInfo;
import android.app.job.JobScheduler;
import android.content.ComponentName;
import android.content.Context;
import android.os.Build;

public class JobSchedulerUtil {
    public static void scheduleJob(Context context) {
        ComponentName componentName = new ComponentName(context, MyJobService.class);
        JobInfo.Builder builder = new JobInfo.Builder(1, componentName)
                .setRequiredNetworkType(JobInfo.NETWORK_TYPE_ANY)
                .setPersisted(true)
                .setPeriodic(15 * 60 * 1000); // 15 minutos

        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            builder.setRequiresBatteryNotLow(true);
        }

        JobScheduler jobScheduler = (JobScheduler) context.getSystemService(Context.JOB_SCHEDULER_SERVICE);
        jobScheduler.schedule(builder.build());
    }
}

Cancelando un Trabajo

Para cancelar un trabajo, puedes usar el método cancel de JobScheduler:

public class JobSchedulerUtil {
    public static void cancelJob(Context context, int jobId) {
        JobScheduler jobScheduler = (JobScheduler) context.getSystemService(Context.JOB_SCHEDULER_SERVICE);
        jobScheduler.cancel(jobId);
    }
}

Características Avanzadas de JobScheduler

Restricciones de Trabajo

JobScheduler te permite establecer varias restricciones para tus trabajos:

  • Tipo de Red: Requiere cualquier red o una red no medida.
  • Carga del Dispositivo: Requiere que el dispositivo esté cargando.
  • Dispositivo Inactivo: Requiere que el dispositivo esté inactivo.
  • Batería No Baja: Requiere que la batería no esté baja.

Criterios de Reintento

Puedes especificar criterios de reintento para controlar cómo debe reintentarse el trabajo si falla:

builder.setBackoffCriteria(3000, JobInfo.BACKOFF_POLICY_LINEAR);

Trabajos Persistentes

Los trabajos persistentes permanecen programados incluso después de que el dispositivo se reinicie:

builder.setPersisted(true);

Ejemplo: Sincronización de Datos Periódica

Aquí tienes un ejemplo completo de un trabajo de sincronización de datos periódica que se ejecuta cada 15 minutos:

import android.app.job.JobInfo;
import android.app.job.JobParameters;
import android.app.job.JobScheduler;
import android.app.job.JobService;
import android.content.ComponentName;
import android.content.Context;
import android.os.Build;
import android.util.Log;

public class DataSyncJobService extends JobService {
    private static final String TAG = "DataSyncJobService";

    @Override
    public boolean onStartJob(JobParameters params) {
        Log.d(TAG, "Data sync job started");
        // Realiza la sincronización de datos aquí
        jobFinished(params, false);
        return true;
    }

    @Override
    public boolean onStopJob(JobParameters params) {
        Log.d(TAG, "Data sync job stopped");
        return false;
    }

    public static void scheduleJob(Context context) {
        ComponentName componentName = new ComponentName(context, DataSyncJobService.class);
        JobInfo.Builder builder = new JobInfo.Builder(1, componentName)
                .setRequiredNetworkType(JobInfo.NETWORK_TYPE_ANY)
                .setPersisted(true)
                .setPeriodic(15 * 60 * 1000);

        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            builder.setRequiresBatteryNotLow(true);
        }

        JobScheduler jobScheduler = (JobScheduler) context.getSystemService(Context.JOB_SCHEDULER_SERVICE);
        jobScheduler.schedule(builder.build());
    }
}

Conclusión

JobScheduler es una API robusta para programar tareas en segundo plano en Android. Al entender cómo crear y gestionar trabajos, puedes asegurarte de que tu aplicación realice las operaciones necesarias en segundo plano de manera eficiente y efectiva. Recuerda considerar las restricciones y los criterios de reintento para optimizar el rendimiento y la fiabilidad de tus tareas programadas.
