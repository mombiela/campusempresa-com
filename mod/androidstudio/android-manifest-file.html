Introducción

El archivo de manifiesto de Android es un componente crucial de cualquier aplicación de Android. Proporciona información esencial sobre tu aplicación al sistema Android, que el sistema debe tener antes de que pueda ejecutar cualquier código de la aplicación.

Conceptos Clave

  • Nombre del Paquete: El identificador único para tu aplicación.
  • Componentes: Declara los componentes de la aplicación como actividades, servicios, receptores de difusión y proveedores de contenido.
  • Permisos: Especifica los permisos que la aplicación necesita para operar.
  • Características de Hardware y Software: Declara las características de hardware y software que la aplicación requiere.
  • Metadatos de la Aplicación: Proporciona información adicional sobre la aplicación, como el ícono, tema y versión de la aplicación.

Estructura del Archivo de Manifiesto de Android

El archivo de manifiesto de Android es un archivo XML llamado AndroidManifest.xml ubicado en el directorio raíz de tu proyecto de Android.

Estructura Básica

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.example.myapp">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
        
    </application>

</manifest>

Explicación

  • <manifest>: El elemento raíz del archivo de manifiesto.
  • xmlns:android: Declaración del espacio de nombres XML.
  • package: El nombre único del paquete de la aplicación.
  • <application>: Contiene configuraciones y declaraciones a nivel de aplicación.
    • android:allowBackup: Permite que los datos de la aplicación se respalden.
    • android:icon: El ícono de la aplicación.
    • android:label: El nombre de la aplicación.
    • android:roundIcon: El ícono redondo de la aplicación.
    • android:supportsRtl: Soporta diseños de derecha a izquierda.
    • android:theme: El tema de la aplicación.
  • <activity>: Declara una actividad.
    • android:name: El nombre de la clase de la actividad.
    • <intent-filter>: Especifica los tipos de intenciones a las que la actividad puede responder.
      • <action>: La acción que la actividad puede realizar.
      • <category>: La categoría de la actividad.

Declaración de Permisos

Los permisos se declaran en el archivo de manifiesto para informar al usuario qué datos o recursos accederá la aplicación.

Ejemplo

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.example.myapp">

    <uses-permission android:name="android.permission.INTERNET" />
    <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
        
    </application>

</manifest>

Explicación

  • <uses-permission>: Declara un permiso que la aplicación debe tener para operar.
    • android:name: El nombre del permiso.

Declaración de Características de Hardware y Software

Puedes declarar las características de hardware y software que tu aplicación requiere usando el elemento <uses-feature>.

Ejemplo

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.example.myapp">

    <uses-feature android:name="android.hardware.camera" android:required="true" />
    <uses-feature android:name="android.hardware.sensor.accelerometer" android:required="false" />

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
        
    </application>

</manifest>

Explicación

  • <uses-feature>: Declara una característica de hardware o software utilizada por la aplicación.
    • android:name: El nombre de la característica.
    • android:required: Especifica si la característica es requerida (true) u opcional (false).

Conclusión

El archivo de manifiesto de Android es una parte fundamental de cualquier aplicación de Android. Proporciona al sistema Android información esencial sobre la aplicación, como sus componentes, permisos y requisitos de hardware. Entender cómo configurar correctamente el archivo de manifiesto es crucial para desarrollar aplicaciones de Android robustas y funcionales.
