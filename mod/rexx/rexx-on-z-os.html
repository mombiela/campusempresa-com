Introducción a REXX

  • ¿Qué es REXX?

    • REXX (Restructured Extended Executor) es un lenguaje de programación de alto nivel diseñado para ser fácil de aprender y leer.
    • Es ampliamente utilizado para scripting y automatización en el sistema operativo mainframe z/OS de IBM.

  • ¿Por qué usar REXX en z/OS?

    • Simplifica tareas complejas con menos líneas de código.
    • Se integra perfectamente con otras utilidades y programas de z/OS.
    • Ideal tanto para principiantes como para programadores experimentados debido a su sintaxis sencilla.

Comenzando con REXX en z/OS

  • Configurando el Entorno

    • Asegúrate de tener acceso a un sistema z/OS.
    • Familiarízate con TSO (Time Sharing Option) e ISPF (Interactive System Productivity Facility).

  • Escribiendo tu Primer Script en REXX

    • Crea un nuevo miembro en un PDS (Partitioned Data Set) con una extensión .REXX.
    • Ejemplo: 
      /* Hola Mundo en REXX */
SAY '¡Hola, Mundo!'
    • Guarda y ejecuta el script usando el comando EXEC en TSO.

Sintaxis Básica y Comandos de REXX

  • Comentarios

    • Comentarios de una sola línea: /* Este es un comentario */

  • Variables

    • No es necesario declarar variables explícitamente.
    • Ejemplo: 
      name = 'Juan Pérez'
SAY 'Hola, ' name

  • Operaciones Básicas

    • Aritmética: +, -, *, /
    • Ejemplo: 
      a = 5
b = 3
c = a + b
SAY 'La suma de' a 'y' b 'es' c

  • Estructuras de Control

    • IF-THEN-ELSE 
      age = 18
IF age >= 18 THEN
  SAY 'Eres un adulto.'
ELSE
  SAY 'Eres un menor.'
    • Bucles DO 
      DO i = 1 TO 5
  SAY 'Iteración' i
END

Características Avanzadas de REXX

  • Funciones Integradas

    • Manipulación de Cadenas 
      str = '¡Hola, Mundo!'
SAY LENGTH(str)  /* Salida: 13 */
SAY SUBSTR(str, 1, 5)  /* Salida: Hola */

  • Operaciones de Entrada/Salida de Archivos

    • Leyendo de un archivo: 
      /* Leer un archivo línea por línea */
"ALLOCATE FILE(INPUT) DA('tu.dataset.name') SHR"
DO WHILE LINES('INPUT') > 0
  LINEIN('INPUT')
  SAY LINE
END
"FREE FILE(INPUT)"
    • Escribiendo en un archivo: 
      /* Escribir en un archivo */
"ALLOCATE FILE(OUTPUT) DA('tu.dataset.name') SHR"
CALL LINEOUT 'OUTPUT', 'Esta es una línea de texto.'
"FREE FILE(OUTPUT)"

  • Interfaz con Utilidades de z/OS

    • Comandos TSO

      ADDRESS TSO "LISTCAT ENTRIES('tu.dataset.name')"

    • Paneles ISPF

      ADDRESS ISPEXEC "DISPLAY PANEL(tu.panel.name)"

Manejo de Errores y Depuración

  • Captura de Errores

    • Ejemplo: 
      SIGNAL ON ERROR
/* Código que podría causar un error */
ERROR:
SAY 'Ocurrió un error.'

  • Técnicas de Depuración

    • Usa TRACE para seguir el flujo de ejecución. 
      TRACE R

Mejores Prácticas para Programar en REXX

  • Legibilidad del Código

    • Usa nombres de variables significativos.
    • Comenta tu código extensamente.

  • Programación Modular

    • Divide scripts grandes en procedimientos más pequeños y reutilizables.

  • Optimización del Rendimiento

    • Minimiza el uso de comandos externos y operaciones de E/S.

Conclusión

  • Resumen

    • REXX es un poderoso lenguaje de scripting en z/OS, que ofrece simplicidad e integración con utilidades de mainframe.
    • Desde la sintaxis básica hasta características avanzadas, REXX puede manejar una variedad de tareas de manera eficiente.
