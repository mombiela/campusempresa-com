Introducción a REXX

  • ¿Qué es REXX?
    • REXX (REstructured eXtended eXecutor) es un lenguaje de scripting desarrollado por IBM.
    • Conocido por su facilidad de uso y legibilidad.
  • Historia y Evolución
    • Se originó en los años 70 para mainframes de IBM.
    • Evolucionó para soportar múltiples plataformas, incluyendo Unix/Linux.

Configuración de REXX en Unix/Linux

  • Instalación
    • La mayoría de las distribuciones de Unix/Linux incluyen intérpretes de REXX como Regina REXX.
    • Ejemplo de comando de instalación para Regina REXX: 
      sudo apt-get install regina-rexx
  • Verificación de la Instalación
    • Verifica la instalación ejecutando: 
      rexx -v

Sintaxis Básica y Estructura

  • Programa Hola Mundo

    • Ejemplo: 
      /* Hola Mundo en REXX */
say "¡Hola, Mundo!"
    • Explicación:
      • /* ... */ se usa para comentarios.
      • say es una función incorporada para imprimir texto en la pantalla.

  • Variables y Tipos de Datos

    • Las variables son de tipo dinámico.
    • Ejemplo: 
      name = "Alice"
age = 30
say "Nombre:" name
say "Edad:" age

Estructuras de Control

  • Sentencias Condicionales

    • Sentencias if, else y select.
    • Ejemplo: 
      age = 20
if age >= 18 then
  say "Eres un adulto."
else
  say "Eres un menor."

  • Bucles

    • Bucles do.
    • Ejemplo: 
      do i = 1 to 5
  say "Número:" i
end

Funciones y Procedimientos

  • Funciones Incorporadas

    • Funciones comunes como length, substr, left, right.
    • Ejemplo: 
      text = "¡Hola, Mundo!"
say "Longitud:" length(text)
say "Subcadena:" substr(text, 1, 5)

  • Funciones Definidas por el Usuario

    • Ejemplo: 
      say "La suma de 5 y 3 es:" sum(5, 3)

sum: procedure
  parse arg a, b
  return a + b

Manejo de Archivos

  • Lectura de un Archivo

    • Ejemplo: 
      file = "example.txt"
call linein file
do while lines(file) > 0
  line = linein(file)
  say line
end
call lineout file

  • Escritura en un Archivo

    • Ejemplo: 
      file = "output.txt"
call lineout file, "Esta es una línea de texto."
call lineout file

Temas Avanzados

  • Interfaz con Comandos de Unix/Linux

    • Usando address para ejecutar comandos de shell.
    • Ejemplo: 
      address system "ls -l"

  • Manejo de Errores

    • Usando signal para manejar errores.
    • Ejemplo: 
      signal on error
call nonexistent_function
exit

error:
  say "Ocurrió un error."

  • Expresiones Regulares

    • Usando expresiones regulares para coincidencia de patrones.
    • Ejemplo: 
      text = "El rápido zorro marrón"
if text \= ".*rápido.*" then
  say "¡Coincidencia encontrada!"

Conclusión

  • Resumen
    • REXX es un lenguaje de scripting versátil adecuado para varias tareas en Unix/Linux.
    • Los conceptos clave incluyen sintaxis básica, estructuras de control, manejo de archivos e interfaz avanzada con comandos del sistema.
