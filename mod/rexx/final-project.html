Resumen

El proyecto final para este curso de REXX está diseñado para consolidar tu aprendizaje desde niveles principiantes hasta avanzados. Aplicarás los conceptos, técnicas y mejores prácticas que has aprendido a lo largo del curso para crear un script REXX integral. Este proyecto evaluará tu comprensión de la programación en REXX y tu capacidad para resolver problemas del mundo real utilizando REXX.

Requisitos del Proyecto

Tu proyecto final debe cumplir con los siguientes requisitos:

  • Longitud del Script: El script debe tener al menos 100 líneas de código.
  • Funcionalidad: El script debe realizar una tarea significativa, como procesamiento de datos, manipulación de archivos o automatización de una tarea repetitiva.
  • Uso de Características de REXX: El script debe demostrar el uso de varias características de REXX, incluyendo:
    • Variables y tipos de datos
    • Estructuras de control (bucles, condicionales)
    • Funciones y subrutinas
    • Operaciones de E/S de archivos
    • Manejo de errores
  • Documentación: El script debe estar bien documentado con comentarios que expliquen el propósito de cada sección del código.
  • Interacción con el Usuario: El script debe incluir alguna forma de interacción con el usuario, como solicitudes de entrada o argumentos de línea de comandos.

Ideas de Proyectos Ejemplares

Aquí tienes algunas ideas de proyectos ejemplares para inspirarte:

  • Analizador de Archivos de Registro: Crea un script que lea un archivo de registro, extraiga información relevante y genere un informe resumido.
  • Automatización de Tareas: Desarrolla un script que automatice una tarea repetitiva, como renombrar archivos en un directorio según un patrón específico.
  • Procesamiento de Datos: Escribe un script que procese un archivo CSV, realice transformaciones de datos y genere los resultados en un nuevo archivo.
  • Cuestionario Interactivo: Construye una aplicación de cuestionario interactivo que haga una serie de preguntas al usuario y proporcione retroalimentación basada en sus respuestas.

Código Ejemplar

A continuación, se muestra un ejemplo de un script REXX simple que lee un archivo de texto y cuenta el número de líneas:

/* Script REXX simple para contar líneas en un archivo */
parse arg filename

if filename = '' then do
  say 'Por favor, proporciona un nombre de archivo como argumento.'
  exit 1
end

lines = 0

/* Abre el archivo para lectura */
call stream filename, 'c', 'open read'

if stream(filename, 's') = 'READY:' then do
  do while lines = stream(filename, 'c', 'read')
    lines = lines + 1
  end
  call stream filename, 'c', 'close'
  say 'Número de líneas en el archivo:' lines
end
else do
  say 'No se pudo abrir el archivo.'
  exit 1
end

Explicación

  • parse arg filename: Esta línea recupera el nombre del archivo de los argumentos de la línea de comandos.
  • if filename = '' then do: Este bloque verifica si se proporcionó un nombre de archivo y sale si no es así.
  • lines = 0: Inicializa el contador de líneas.
  • call stream filename, 'c', 'open read': Abre el archivo para lectura.
  • if stream(filename, 's') = 'READY:' then do: Verifica si el archivo se abrió correctamente.
  • do while lines = stream(filename, 'c', 'read'): Lee cada línea del archivo e incrementa el contador de líneas.
  • call stream filename, 'c', 'close': Cierra el archivo después de leerlo.
  • say 'Número de líneas en el archivo:' lines: Muestra el número total de líneas.

Directrices de Envío

  • Documentación: Incluye un archivo README que explique el propósito de tu script, cómo ejecutarlo y cualquier dependencia o requisito.
