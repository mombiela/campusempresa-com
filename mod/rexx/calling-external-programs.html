Llamar a Programas Externos en REXX
Interfaz con Bases de Datos ►

En esta sección, exploraremos cómo llamar a programas externos desde un script REXX. Esta es una característica poderosa que permite a los scripts REXX interactuar con otros programas y realizar una amplia gama de tareas más allá de las capacidades del propio REXX.

Introducción

Llamar a programas externos puede ser útil para:

  • Automatizar tareas del sistema.
  • Integrar scripts REXX con otros programas.
  • Extender la funcionalidad de REXX aprovechando otras herramientas.

Sintaxis Básica

En REXX, puedes llamar a un programa externo usando la instrucción ADDRESS. La instrucción ADDRESS te permite especificar el entorno en el que se debe ejecutar el comando.

Ejemplo: Llamar a un Comando de Shell

Aquí hay un ejemplo simple de cómo llamar a un comando de shell desde un script REXX:

/* Script REXX para llamar a un comando de shell externo */
ADDRESS SYSTEM 'echo Hello, World!'

En este ejemplo:

  • ADDRESS SYSTEM especifica que el comando debe ejecutarse en el shell del sistema.
  • 'echo Hello, World!' es el comando que se está ejecutando.

Capturar Salida

Puedes capturar la salida de un comando externo usando la función LINEIN. Esto es útil cuando necesitas procesar la salida del comando dentro de tu script REXX.

Ejemplo: Capturar la Salida de un Comando

/* Script REXX para capturar la salida de un comando de shell */
ADDRESS SYSTEM 'dir > temp.txt'
output = LINEIN('temp.txt')
SAY output

En este ejemplo:

  • ADDRESS SYSTEM 'dir > temp.txt' ejecuta el comando dir y redirige la salida a un archivo llamado temp.txt.
  • output = LINEIN('temp.txt') lee la primera línea de la salida del archivo.
  • SAY output imprime la salida capturada.

Manejo de Errores

Al llamar a programas externos, es importante manejar los errores de manera adecuada. Puedes verificar el código de retorno del comando para determinar si se ejecutó correctamente.

Ejemplo: Verificar el Código de Retorno

/* Script REXX para verificar el código de retorno de un comando de shell */
ADDRESS SYSTEM 'nonexistent_command'
if RC \= 0 then
  SAY 'El comando falló con el código de retorno:' RC
else
  SAY 'El comando se ejecutó correctamente'

En este ejemplo:

  • ADDRESS SYSTEM 'nonexistent_command' intenta ejecutar un comando que no existe.
  • if RC \= 0 then verifica si el código de retorno (RC) no es igual a 0, lo que indica un error.
  • SAY 'El comando falló con el código de retorno:' RC imprime un mensaje de error si el comando falló.

Técnicas Avanzadas

Para casos de uso más avanzados, es posible que necesites interactuar con el programa externo de una manera más compleja, como pasar argumentos o manejar múltiples líneas de salida.

Ejemplo: Pasar Argumentos

/* Script REXX para pasar argumentos a un programa externo */
arg1 = 'Hello'
arg2 = 'World'
ADDRESS SYSTEM 'echo' arg1 arg2

En este ejemplo:

  • arg1 y arg2 son variables que contienen los argumentos a pasar.
  • ADDRESS SYSTEM 'echo' arg1 arg2 ejecuta el comando echo con los argumentos.

Ejemplo: Manejar Múltiples Líneas de Salida

/* Script REXX para manejar múltiples líneas de salida */
ADDRESS SYSTEM 'dir > temp.txt'
do while LINES('temp.txt') > 0
  line = LINEIN('temp.txt')
  SAY line
end

En este ejemplo:

  • do while LINES('temp.txt') > 0 itera mientras haya líneas para leer en el archivo.
  • line = LINEIN('temp.txt') lee cada línea de la salida.
  • SAY line imprime cada línea.

Conclusión

Llamar a programas externos desde scripts REXX amplía las capacidades de tus scripts al permitirles interactuar con el sistema y otros programas. Al entender la sintaxis básica, capturar la salida, manejar errores y usar técnicas avanzadas, puedes integrar efectivamente comandos externos en tus scripts REXX.

Practica estos conceptos con varios programas y comandos externos para volverte competente en aprovechar esta poderosa característica de REXX.
