Renderización de Plantillas
Motor de Plantillas Jinja2 ►

En esta sección, exploraremos cómo renderizar plantillas en Flask, un aspecto crucial para construir aplicaciones web dinámicas. Cubriremos los conceptos básicos de la renderización de plantillas, cómo pasar datos a las plantillas y técnicas avanzadas para crear plantillas reutilizables y mantenibles.

Introducción a las Plantillas

Las plantillas te permiten separar la capa de presentación de la lógica de negocio de tu aplicación. Flask utiliza Jinja2 como su motor de plantillas, que proporciona una forma poderosa de generar HTML dinámicamente.

Conceptos Clave

  • Motor de Plantillas: Un componente de software que combina plantillas con un modelo de datos para producir documentos.
  • Jinja2: El motor de plantillas predeterminado para Flask, que permite la generación de contenido dinámico.
  • Renderización: El proceso de generar el HTML final a partir de plantillas y datos.

Renderización Básica de Plantillas

Para renderizar una plantilla en Flask, utilizas la función render_template. Esta función toma el nombre del archivo de la plantilla y cualquier dato que desees pasarle.

Ejemplo

  1. Crear un Archivo de Plantilla: Guarda el siguiente código HTML en un archivo llamado index.html dentro de una carpeta llamada templates.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{{ title }}</title>
</head>
<body>
    <h1>{{ heading }}</h1>
    <p>{{ message }}</p>
</body>
</html>

  2. Renderizar la Plantilla en Flask: Usa la función render_template en tu ruta de Flask.

    from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def home():
    return render_template('index.html', title='Home Page', heading='Welcome to Flask', message='This is a basic template rendering example.')

if __name__ == '__main__':
    app.run(debug=True)

Explicación

  • Variables de Plantilla: {{ title }}, {{ heading }} y {{ message }} son marcadores de posición en la plantilla que serán reemplazados con los valores pasados desde la ruta de Flask.
  • Función render_template: Esta función toma el nombre del archivo de la plantilla y argumentos de palabra clave para los datos que se pasarán a la plantilla.

Pasar Datos a las Plantillas

Puedes pasar varios tipos de datos a las plantillas, incluyendo cadenas, listas, diccionarios y más.

Ejemplo con Lista y Diccionario

  1. Archivo de Plantilla: Guarda el siguiente código HTML en un archivo llamado data.html.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Data Example</title>
</head>
<body>
    <h1>Users</h1>
    <ul>
        {% for user in users %}
            <li>{{ user.name }} - {{ user.email }}</li>
        {% endfor %}
    </ul>
</body>
</html>

  2. Renderizar la Plantilla con Datos:

    @app.route('/users')
def users():
    users = [
        {'name': 'Alice', 'email': 'alice@example.com'},
        {'name': 'Bob', 'email': 'bob@example.com'},
        {'name': 'Charlie', 'email': 'charlie@example.com'}
    ]
    return render_template('data.html', users=users)

Explicación

  • Bucle en Plantillas: {% for user in users %} es un bucle de Jinja2 que itera sobre la lista de usuarios y renderiza el nombre y correo electrónico de cada usuario.
  • Contexto de Plantilla: La variable users se pasa a la plantilla y puede ser accedida dentro de la plantilla.

Características Avanzadas de Plantillas

Flask y Jinja2 ofrecen características avanzadas para hacer las plantillas más poderosas y mantenibles.

Herencia de Plantillas

La herencia de plantillas te permite crear una plantilla base que otras plantillas pueden extender. Esto es útil para mantener un diseño consistente en múltiples páginas.

  1. Plantilla Base: Guarda el siguiente código HTML en un archivo llamado base.html.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}My Website{% endblock %}</title>
</head>
<body>
    <header>
        <h1>My Website</h1>
    </header>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 My Website</p>
    </footer>
</body>
</html>

  2. Plantilla Hija: Guarda el siguiente código HTML en un archivo llamado child.html.

    {% extends "base.html" %}

{% block title %}Home Page{% endblock %}

{% block content %}
    <h2>Welcome to the Home Page</h2>
    <p>This is the content of the home page.</p>
{% endblock %}

  3. Renderizar la Plantilla Hija:

    @app.route('/home')
def home():
    return render_template('child.html')

Explicación

  • Bloques de Plantilla: {% block title %} y {% block content %} definen secciones de la plantilla que pueden ser sobrescritas por plantillas hijas.
  • Extender Plantillas: {% extends "base.html" %} indica que child.html extiende base.html.

Conclusión

Renderizar plantillas en Flask es una forma poderosa de crear páginas web dinámicas. Al entender los conceptos básicos de la renderización de plantillas, pasar datos a las plantillas y usar características avanzadas como la herencia de plantillas, puedes construir aplicaciones web mantenibles y escalables. Practica estos conceptos con diferentes tipos de datos y plantillas para volverte competente en la renderización de plantillas en Flask.
