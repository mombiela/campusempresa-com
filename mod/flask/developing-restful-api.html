Introducción a las APIs RESTful

  • Definición: REST (Transferencia de Estado Representacional) es un estilo arquitectónico para diseñar aplicaciones en red.
  • Conceptos Clave:
    • Recursos: Objetos o representaciones de datos con los que los clientes interactúan.
    • Métodos HTTP: GET, POST, PUT, DELETE, etc.
    • Sin Estado: Cada solicitud de un cliente contiene toda la información necesaria para procesar la solicitud.
    • Endpoints: URLs que representan recursos.

Configuración de Flask

  • Instalación: 
    pip install Flask
  • Aplicación Básica de Flask: 
    from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "¡Hola, Flask!"

if __name__ == '__main__':
    app.run(debug=True)

Creación de una API RESTful Simple

  • Creación de Endpoints: 
    from flask import Flask, jsonify, request

app = Flask(__name__)

@app.route('/api/resource', methods=['GET'])
def get_resource():
    return jsonify({"message": "Solicitud GET recibida"})

@app.route('/api/resource', methods=['POST'])
def create_resource():
    data = request.get_json()
    return jsonify({"message": "Solicitud POST recibida", "data": data}), 201

if __name__ == '__main__':
    app.run(debug=True)
  • Explicación:
    • @app.route('/api/resource', methods=['GET']): Define un endpoint GET.
    • @app.route('/api/resource', methods=['POST']): Define un endpoint POST.
    • request.get_json(): Analiza los datos de la solicitud JSON entrante.

Manejo de Operaciones CRUD

  • Operaciones CRUD:

    • Crear: POST
    • Leer: GET
    • Actualizar: PUT
    • Eliminar: DELETE

  • Ejemplo:

    from flask import Flask, jsonify, request

app = Flask(__name__)

resources = []

@app.route('/api/resources', methods=['GET'])
def get_resources():
    return jsonify(resources)

@app.route('/api/resources', methods=['POST'])
def create_resource():
    data = request.get_json()
    resources.append(data)
    return jsonify(data), 201

@app.route('/api/resources/<int:id>', methods=['PUT'])
def update_resource(id):
    data = request.get_json()
    resources[id] = data
    return jsonify(data)

@app.route('/api/resources/<int:id>', methods=['DELETE'])
def delete_resource(id):
    resources.pop(id)
    return '', 204

if __name__ == '__main__':
    app.run(debug=True)

  • Explicación:

    • resources = []: Una lista simple en memoria para almacenar recursos.
    • @app.route('/api/resources', methods=['GET']): Recupera todos los recursos.
    • @app.route('/api/resources', methods=['POST']): Crea un nuevo recurso.
    • @app.route('/api/resources/<int:id>', methods=['PUT']): Actualiza un recurso por ID.
    • @app.route('/api/resources/<int:id>', methods=['DELETE']): Elimina un recurso por ID.

Manejo de Errores

  • Respuestas de Error Personalizadas: 
    from flask import Flask, jsonify

app = Flask(__name__)

@app.errorhandler(404)
def not_found(error):
    return jsonify({"error": "Recurso no encontrado"}), 404

@app.errorhandler(500)
def internal_error(error):
    return jsonify({"error": "Error interno del servidor"}), 500

if __name__ == '__main__':
    app.run(debug=True)
  • Explicación:
    • @app.errorhandler(404): Manejador personalizado para errores 404.
    • @app.errorhandler(500): Manejador personalizado para errores 500.

Temas Avanzados

  • Autenticación y Autorización:
    • JWT (JSON Web Tokens): 
      from flask import Flask, jsonify, request
import jwt
import datetime

app = Flask(__name__)
app.config['SECRET_KEY'] = 'tu_clave_secreta'

@app.route('/login', methods=['POST'])
def login():
    auth_data = request.get_json()
    if auth_data['username'] == 'user' y auth_data['password'] == 'pass':
        token = jwt.encode({'user': auth_data['username'], 'exp': datetime.datetime.utcnow() + datetime.timedelta(minutes=30)}, app.config['SECRET_KEY'])
        return jsonify({'token': token})
    return jsonify({'message': 'Credenciales inválidas'}), 401

@app.route('/protected', methods=['GET'])
def protected():
    token = request.headers.get('x-access-tokens')
    if not token:
        return jsonify({'message': 'Falta el token'}), 401
    try:
        data = jwt.decode(token, app.config['SECRET_KEY'], algorithms=["HS256"])
    except:
        return jsonify({'message': 'Token inválido'}), 401
    return jsonify({'message': 'Ruta protegida accedida'})

if __name__ == '__main__':
    app.run(debug=True)
    • Explicación:
      • jwt.encode(): Codifica una carga útil en un token JWT.
      • jwt.decode(): Decodifica un token JWT.
      • @app.route('/login', methods=['POST']): Endpoint para autenticar y generar un token.
      • @app.route('/protected', methods=['GET']): Endpoint protegido que requiere un token válido.

Conclusión

Desarrollar una API RESTful con Flask implica entender los conceptos básicos de la arquitectura REST, configurar Flask, crear endpoints, manejar operaciones CRUD e implementar el manejo de errores. Temas avanzados como la autenticación y autorización pueden añadirse para asegurar la API. Con estos conceptos, puedes construir APIs robustas y escalables usando Flask.
