Introducción a las APIs RESTful

  • Definición: REST (Transferencia de Estado Representacional) es un estilo arquitectónico para diseñar aplicaciones en red.
  • Conceptos Clave:
    • Recursos: Objetos o representaciones de datos que pueden ser accedidos a través de una URL.
    • Métodos HTTP: GET, POST, PUT, DELETE, etc., usados para realizar operaciones CRUD.
    • Sin Estado: Cada solicitud de un cliente al servidor debe contener toda la información que el servidor necesita para cumplir con esa solicitud.
    • JSON: Formato común para enviar y recibir datos en APIs RESTful.

Configuración de Flask para APIs RESTful

  • Instalación: 
    pip install Flask
pip install Flask-RESTful
  • Aplicación Básica de Flask: 
    from flask import Flask
app = Flask(__name__)

@app.route('/')
def home():
    return "¡Hola, Flask!"

if __name__ == '__main__':
    app.run(debug=True)

Creando una API RESTful Simple

  • Creando Endpoints: 
    from flask import Flask, jsonify, request
app = Flask(__name__)

@app.route('/api/v1/resource', methods=['GET'])
def get_resource():
    data = {"message": "Hola, este es tu recurso"}
    return jsonify(data)

if __name__ == '__main__':
    app.run(debug=True)
  • Explicación:
    • @app.route('/api/v1/resource', methods=['GET']): Define el endpoint y el método HTTP.
    • jsonify(data): Convierte el diccionario de Python a una respuesta JSON.

Manejo de Operaciones CRUD

  • Ejemplo con Datos en Memoria: 
    from flask import Flask, jsonify, request

app = Flask(__name__)

resources = []

@app.route('/api/v1/resources', methods=['GET'])
def get_resources():
    return jsonify(resources)

@app.route('/api/v1/resources', methods=['POST'])
def add_resource():
    new_resource = request.get_json()
    resources.append(new_resource)
    return jsonify(new_resource), 201

@app.route('/api/v1/resources/<int:id>', methods=['PUT'])
def update_resource(id):
    updated_resource = request.get_json()
    resources[id] = updated_resource
    return jsonify(updated_resource)

@app.route('/api/v1/resources/<int:id>', methods=['DELETE'])
def delete_resource(id):
    resources.pop(id)
    return '', 204

if __name__ == '__main__':
    app.run(debug=True)
  • Explicación:
    • request.get_json(): Analiza los datos de la solicitud JSON entrante.
    • resources.append(new_resource): Añade un nuevo recurso a la lista.
    • resources[id] = updated_resource: Actualiza un recurso existente.
    • resources.pop(id): Elimina un recurso.

Manejo de Errores y Validación

  • Mensajes de Error Personalizados: 
    from flask import Flask, jsonify, request, abort

app = Flask(__name__)

resources = []

@app.route('/api/v1/resources/<int:id>', methods=['GET'])
def get_resource(id):
    if id >= len(resources) or id < 0:
        abort(404, description="Recurso no encontrado")
    return jsonify(resources[id])

@app.errorhandler(404)
def resource_not_found(e):
    return jsonify(error=str(e)), 404

if __name__ == '__main__':
    app.run(debug=True)
  • Explicación:
    • abort(404, description="Recurso no encontrado"): Aborta la solicitud con un código de estado 404 y un mensaje personalizado.
    • @app.errorhandler(404): Manejador de errores personalizado para errores 404.

Temas Avanzados

  • Autenticación y Autorización:
    • JWT (JSON Web Tokens): 
      from flask import Flask, jsonify, request
import jwt
import datetime

app = Flask(__name__)
app.config['SECRET_KEY'] = 'tu_clave_secreta'

@app.route('/login', methods=['POST'])
def login():
    auth_data = request.get_json()
    if auth_data['username'] == 'user' y auth_data['password'] == 'pass':
        token = jwt.encode({'user': auth_data['username'], 'exp': datetime.datetime.utcnow() + datetime.timedelta(minutes=30)}, app.config['SECRET_KEY'])
        return jsonify({'token': token})
    return jsonify({'message': 'Credenciales inválidas'}), 401

if __name__ == '__main__':
    app.run(debug=True)
    • Explicación:
      • jwt.encode(): Codifica un payload en un token JWT.
      • app.config['SECRET_KEY']: Clave secreta para codificar y decodificar tokens JWT.

Conclusión

  • Resumen:
    • Las APIs RESTful son una forma poderosa de diseñar aplicaciones en red.
    • Flask proporciona un marco simple y flexible para construir APIs RESTful.
    • Los conceptos clave incluyen recursos, métodos HTTP y sin estado.
    • Manejar operaciones CRUD, manejo de errores y autenticación son esenciales para construir APIs robustas.
