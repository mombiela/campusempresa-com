Introducción

Flask es un marco web ligero y flexible para Python, pero con gran flexibilidad viene la responsabilidad de asegurar que tu aplicación sea segura. Esta sección cubrirá las prácticas esenciales de seguridad para aplicaciones Flask, desde niveles principiantes hasta avanzados.

Prácticas Básicas de Seguridad

Validación de Entradas

  • Sanitizar Entradas de Usuario: Siempre valida y sanitiza las entradas de usuario para prevenir ataques de inyección. 
    from flask import request, escape

@app.route('/submit', methods=['POST'])
def submit():
    user_input = request.form['user_input']
    sanitized_input = escape(user_input)
    return f"Processed input: {sanitized_input}"

Configuración Segura

  • Variables de Entorno: Almacena información sensible como claves secretas y credenciales de base de datos en variables de entorno.

    import os

app.config['SECRET_KEY'] = os.environ.get('SECRET_KEY', 'default_secret_key')

  • Modo Debug: Nunca ejecutes tu aplicación en modo debug en producción.

    if __name__ == '__main__':
    app.run(debug=False)

HTTPS

  • Usa HTTPS: Asegúrate de que tu aplicación se sirva a través de HTTPS para encriptar los datos en tránsito. 
    from flask import Flask, redirect, request

app = Flask(__name__)

@app.before_request
def before_request():
    if not request.is_secure:
        return redirect(request.url.replace("http://", "https://"))

Prácticas Intermedias de Seguridad

Protección Contra Cross-Site Request Forgery (CSRF)

  • Tokens CSRF: Usa tokens CSRF para proteger contra ataques CSRF. 
    from flask_wtf.csrf import CSRFProtect

csrf = CSRFProtect(app)

Gestión de Sesiones

  • Cookies Seguras: Configura cookies seguras para prevenir el secuestro de sesiones. 
    app.config.update(
    SESSION_COOKIE_HTTPONLY=True,
    SESSION_COOKIE_SECURE=True,
)

Autenticación y Autorización

  • Flask-Login: Usa Flask-Login para la gestión de sesiones de usuario. 
    from flask_login import LoginManager

login_manager = LoginManager()
login_manager.init_app(app)

Prácticas Avanzadas de Seguridad

Política de Seguridad de Contenidos (CSP)

  • Encabezados CSP: Implementa encabezados CSP para prevenir ataques XSS. 
    @app.after_request
def set_csp(response):
    response.headers['Content-Security-Policy'] = "default-src 'self'"
    return response

Limitación de Tasa

  • Flask-Limiter: Usa Flask-Limiter para proteger contra ataques de fuerza bruta. 
    from flask_limiter import Limiter

limiter = Limiter(app, key_func=get_remote_address)

@app.route('/login')
@limiter.limit("5 per minute")
def login():
    return "Login Page"

Encabezados Seguros

  • Flask-Talisman: Usa Flask-Talisman para establecer varios encabezados HTTP para seguridad. 
    from flask_talisman import Talisman

Talisman(app)

Conclusión

Asegurar una aplicación Flask implica múltiples capas de defensa, desde la validación básica de entradas y configuraciones seguras hasta técnicas avanzadas como CSP y limitación de tasa. Siguiendo estas mejores prácticas, puedes reducir significativamente el riesgo de vulnerabilidades de seguridad en tu aplicación. Mantente siempre actualizado con las últimas tendencias de seguridad y monitorea continuamente tu aplicación en busca de posibles amenazas.
