Introducción

En este proyecto, construiremos un pipeline de Integración Continua y Despliegue Continuo (CI/CD) usando Docker. Este proyecto te ayudará a entender cómo automatizar la construcción, prueba y despliegue de aplicaciones usando contenedores Docker. Al final de este proyecto, tendrás un pipeline de CI/CD funcional que puede integrarse con cualquier aplicación.

Conceptos Clave

  • Integración Continua (CI): La práctica de integrar automáticamente cambios de código de múltiples contribuyentes en un repositorio compartido varias veces al día.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio que pasa las pruebas automatizadas a producción.
  • Docker: Una plataforma para desarrollar, enviar y ejecutar aplicaciones en contenedores.
  • Dockerfile: Un documento de texto que contiene todos los comandos para ensamblar una imagen.
  • Docker Compose: Una herramienta para definir y ejecutar aplicaciones Docker de múltiples contenedores.

Prerrequisitos

  • Entendimiento básico de Docker y Docker Compose.
  • Familiaridad con sistemas de control de versiones como Git.
  • Conocimiento básico de scripting en shell.

Configuración del Entorno

  1. Instalar Docker: Sigue la guía oficial de instalación de Docker para tu sistema operativo.

  2. Instalar Docker Compose: Sigue la guía oficial de instalación de Docker Compose para tu sistema operativo.

  3. Configurar un Repositorio Git: Crea un nuevo repositorio en GitHub o cualquier otro servicio de alojamiento Git.

Creando una Aplicación Simple

Vamos a crear una aplicación simple de Node.js que usaremos para nuestro pipeline de CI/CD.

  1. Inicializar un Proyecto de Node.js:

    mkdir my-node-app
cd my-node-app
npm init -y

  2. Crear un Archivo index.js:

    const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('¡Hola Mundo!');
});

app.listen(port, () => {
  console.log(`App escuchando en http://localhost:${port}`);
});

  3. Instalar Express:

    npm install express

Escribiendo un Dockerfile

Crea un Dockerfile en la raíz de tu directorio de proyecto.

# Usar la imagen oficial de Node.js.
FROM node:14

# Crear y cambiar al directorio de la aplicación.
WORKDIR /usr/src/app

# Copiar los manifiestos de dependencias de la aplicación a la imagen del contenedor.
COPY package*.json ./

# Instalar dependencias.
RUN npm install

# Copiar el código local a la imagen del contenedor.
COPY . .

# Ejecutar el servicio web al iniciar el contenedor.
CMD [ "node", "index.js" ]

# Exponer el puerto en el que se ejecuta la aplicación.
EXPOSE 3000

Creando un Archivo Docker Compose

Crea un archivo docker-compose.yml en la raíz de tu directorio de proyecto.

version: '3.8'

services:
  web:
    build: .
    ports:
      - "3000:3000"

Configuración de CI con GitHub Actions

Crea un archivo .github/workflows/ci.yml en tu repositorio.

name: CI

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v1

      - name: Build the Docker image
        run: docker build . --file Dockerfile --tag my-node-app:latest

      - name: Run tests
        run: docker run my-node-app:latest npm test

Configuración de CD con GitHub Actions

Extiende el archivo .github/workflows/ci.yml para incluir pasos de despliegue.

name: CI/CD

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v1

      - name: Build the Docker image
        run: docker build . --file Dockerfile --tag my-node-app:latest

      - name: Run tests
        run: docker run my-node-app:latest npm test

  deploy:
    runs-on: ubuntu-latest
    needs: build

    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Deploy to Docker Hub
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin
          docker tag my-node-app:latest my-dockerhub-username/my-node-app:latest
          docker push my-dockerhub-username/my-node-app:latest

Conclusión

En este proyecto, hemos configurado exitosamente un pipeline de CI/CD usando Docker y GitHub Actions. Creamos una aplicación simple de Node.js, escribimos un Dockerfile para contenerizar la aplicación y usamos Docker Compose para gestionar el contenedor. Luego configuramos GitHub Actions para automatizar la construcción, prueba y despliegue de nuestro contenedor Docker. Este pipeline de CI/CD puede adaptarse para ajustarse a cualquier aplicación, proporcionando un flujo de trabajo robusto y automatizado para la integración y despliegue continuo.
