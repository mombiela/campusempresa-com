¿Qué es Docker Compose?

Docker Compose es una herramienta para definir y ejecutar aplicaciones Docker de múltiples contenedores. Te permite gestionar múltiples contenedores como un solo servicio, facilitando la orquestación de aplicaciones complejas.

Conceptos Clave

  • Servicio: Un servicio define un contenedor en el contexto de Docker Compose. Especifica la imagen a usar, los puertos a exponer y otras configuraciones.
  • Volumen: Los volúmenes se utilizan para persistir datos generados y utilizados por los contenedores Docker.
  • Red: Las redes permiten la comunicación entre contenedores Docker. Docker Compose crea automáticamente una red predeterminada para que los servicios se comuniquen.

¿Por qué usar Docker Compose?

  • Configuración Simplificada: Define aplicaciones de múltiples contenedores con un solo archivo YAML.
  • Gestión Fácil: Inicia, detén y reconstruye servicios con comandos simples.
  • Consistencia del Entorno: Asegura que tus entornos de desarrollo, pruebas y producción sean consistentes.

Instalando Docker Compose

Docker Compose está incluido en Docker Desktop para Windows y Mac. Para Linux, puedes instalarlo usando los siguientes comandos:

sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

Verifica la instalación:

docker-compose --version

Creando un Archivo Docker Compose

Un archivo Docker Compose se escribe en formato YAML y típicamente se nombra docker-compose.yml.

Ejemplo: Aplicación Web Simple

Vamos a crear una aplicación web simple usando Docker Compose.

  1. Crea un directorio de proyecto:

    mkdir my-web-app
cd my-web-app

  2. Crea un archivo docker-compose.yml:

    version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"
  redis:
    image: redis:latest

Explicación

  • version: Especifica la versión del formato del archivo Docker Compose.
  • services: Define los servicios que componen tu aplicación.
    • web: Usa la imagen nginx:latest y mapea el puerto 8080 en el host al puerto 80 en el contenedor.
    • redis: Usa la imagen redis:latest.

Ejecutando Docker Compose

Para iniciar los servicios definidos en el archivo docker-compose.yml, usa el siguiente comando:

docker-compose up

Este comando:

  • Descargará las imágenes especificadas si no están disponibles localmente.
  • Creará e iniciará los contenedores.
  • Se adjuntará a los registros de los contenedores.

Para ejecutar los servicios en segundo plano (modo desacoplado), usa:

docker-compose up -d

Gestionando Servicios de Docker Compose

Deteniendo Servicios

Para detener los servicios, usa:

docker-compose down

Este comando detiene y elimina los contenedores, redes y volúmenes definidos en el archivo docker-compose.yml.

Visualizando Registros

Para ver los registros de un servicio específico, usa:

docker-compose logs <service_name>

Por ejemplo, para ver los registros del servicio web:

docker-compose logs web

Escalando Servicios

Docker Compose te permite escalar servicios. Por ejemplo, para escalar el servicio web a 3 instancias:

docker-compose up --scale web=3 -d

Conclusión

Docker Compose simplifica la gestión de aplicaciones Docker de múltiples contenedores. Al definir servicios, redes y volúmenes en un solo archivo YAML, puedes orquestar fácilmente aplicaciones complejas. Con comandos como docker-compose up y docker-compose down, la gestión del ciclo de vida de tu aplicación se vuelve sencilla. A medida que avances, puedes explorar características más avanzadas como variables de entorno, redes personalizadas y gestión de volúmenes para mejorar aún más tus configuraciones de Docker Compose.
