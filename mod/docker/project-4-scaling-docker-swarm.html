Introducción

En este proyecto, exploraremos Docker Swarm, una herramienta nativa de agrupamiento y orquestación para Docker. Docker Swarm te permite gestionar un clúster de nodos Docker como un único sistema virtual. Este proyecto te guiará a través de la configuración de un Docker Swarm, el despliegue de servicios y su escalado eficiente.

Conceptos Clave

  • Docker Swarm: Una herramienta de agrupamiento y orquestación para contenedores Docker.
  • Nodo: Una máquina individual en el clúster de Docker Swarm.
  • Nodo Manager: Un nodo que gestiona el swarm y toma decisiones sobre el clúster.
  • Nodo Worker: Un nodo que recibe y ejecuta tareas del nodo manager.
  • Servicio: Una definición de cómo debe comportarse un contenedor en el swarm.
  • Tarea: Un solo contenedor ejecutándose en el swarm.

Configuración de Docker Swarm

Inicializando el Swarm

Para inicializar un Docker Swarm, necesitas ejecutar el siguiente comando en la máquina que actuará como nodo manager:

docker swarm init --advertise-addr <MANAGER-IP>

Reemplaza <MANAGER-IP> con la dirección IP de tu nodo manager.

Añadiendo Nodos Worker

Después de inicializar el swarm, recibirás un comando para añadir nodos worker. Ejecuta este comando en cada nodo worker:

docker swarm join --token <SWARM-TOKEN> <MANAGER-IP>:2377

Reemplaza <SWARM-TOKEN> con el token proporcionado por el comando docker swarm init y <MANAGER-IP> con la dirección IP del nodo manager.

Despliegue de Servicios

Creando un Servicio

Para crear un servicio en Docker Swarm, usa el comando docker service create. Por ejemplo, para crear un servicio simple de Nginx, ejecuta:

docker service create --name my-nginx --replicas 3 -p 80:80 nginx

Este comando:

  • Creará un servicio llamado my-nginx.
  • Desplegará 3 réplicas del contenedor Nginx.
  • Mapeará el puerto 80 en el host al puerto 80 en el contenedor.

Listando Servicios

Para listar todos los servicios ejecutándose en el swarm, usa:

docker service ls

Inspeccionando un Servicio

Para obtener información detallada sobre un servicio, usa:

docker service inspect <SERVICE-ID>

Reemplaza <SERVICE-ID> con el ID del servicio que deseas inspeccionar.

Escalado de Servicios

Escalando Hacia Arriba

Para escalar un servicio hacia arriba, usa el comando docker service scale. Por ejemplo, para escalar el servicio my-nginx a 5 réplicas, ejecuta:

docker service scale my-nginx=5

Escalando Hacia Abajo

Para escalar un servicio hacia abajo, usa el mismo comando docker service scale. Por ejemplo, para escalar el servicio my-nginx a 2 réplicas, ejecuta:

docker service scale my-nginx=2

Monitoreo y Gestión del Swarm

Verificando el Estado de los Nodos

Para verificar el estado de todos los nodos en el swarm, usa:

docker node ls

Drenando un Nodo

Para drenar un nodo (detenerlo de recibir nuevas tareas), usa:

docker node update --availability drain <NODE-ID>

Reemplaza <NODE-ID> con el ID del nodo que deseas drenar.

Promoviendo y Degradando Nodos

Para promover un nodo worker a nodo manager, usa:

docker node promote <NODE-ID>

Para degradar un nodo manager a nodo worker, usa:

docker node demote <NODE-ID>

Conclusión

En este proyecto, aprendiste cómo configurar un Docker Swarm, desplegar y escalar servicios, y gestionar el swarm de manera efectiva. Docker Swarm proporciona una forma poderosa pero sencilla de orquestar contenedores, facilitando la gestión de aplicaciones a gran escala. Al dominar estos conceptos, puedes asegurar que tus aplicaciones sean escalables, resilientes y fáciles de gestionar.
