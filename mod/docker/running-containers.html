Ejecutando Contenedores
Ciclo de Vida del Contenedor ►

En esta sección, exploraremos cómo ejecutar contenedores usando Docker. Esta es una habilidad fundamental para cualquiera que trabaje con Docker, ya que los contenedores son la unidad central de despliegue de software en Docker.

Conceptos Clave

  • Contenedor: Un paquete ligero, autónomo y ejecutable que incluye todo lo necesario para ejecutar una pieza de software, incluyendo el código, el tiempo de ejecución, las bibliotecas y las herramientas del sistema.
  • Imagen: Una plantilla de solo lectura utilizada para crear contenedores. Las imágenes a menudo se basan en otras imágenes, con alguna personalización adicional.
  • Docker Hub: Un repositorio basado en la nube donde los usuarios de Docker pueden crear, probar, almacenar y distribuir imágenes de contenedores.

Ejecutando Tu Primer Contenedor

Para ejecutar un contenedor, necesitas una imagen de Docker. Docker Hub proporciona un vasto repositorio de imágenes preconstruidas que puedes usar.

Ejemplo: Ejecutando un Contenedor Hello World

  1. Descargar la Imagen: Primero, necesitas descargar la imagen desde Docker Hub.

    docker pull hello-world

  2. Ejecutar el Contenedor: Usa el comando docker run para crear y arrancar un contenedor desde la imagen.

    docker run hello-world

    Cuando ejecutas este comando, Docker:

    • Verificará si la imagen hello-world está disponible localmente. Si no, la descargará desde Docker Hub.
    • Creará un nuevo contenedor a partir de la imagen.
    • Arrancará el contenedor.
    • Ejecutará el comando especificado en la imagen (en este caso, imprimir "Hello from Docker!").

Entendiendo el Comando `docker run`

El comando docker run es versátil y puede usarse con varias opciones para personalizar el comportamiento del contenedor.

  • Sintaxis Básica:

    docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

  • Opciones Comunes:

    • -d: Ejecuta el contenedor en modo desacoplado (en segundo plano).
    • -p: Publica el/los puerto(s) del contenedor en el host.
    • -v: Monta un volumen.
    • --name: Asigna un nombre al contenedor.
    • -e: Establece variables de entorno.

Ejemplo: Ejecutando un Servidor Web

Vamos a ejecutar un servidor web simple usando la imagen nginx.

  1. Descargar la Imagen:

    docker pull nginx

  2. Ejecutar el Contenedor:

    docker run -d -p 8080:80 --name mynginx nginx

    Explicación:

    • -d: Ejecuta el contenedor en modo desacoplado.
    • -p 8080:80: Mapea el puerto 8080 en el host al puerto 80 en el contenedor.
    • --name mynginx: Nombra el contenedor mynginx.
    • nginx: La imagen a usar.

  3. Acceder al Servidor Web: Abre un navegador web y navega a http://localhost:8080. Deberías ver la página de bienvenida predeterminada de Nginx.

Gestionando Contenedores

Una vez que tienes contenedores en ejecución, necesitas gestionarlos. Docker proporciona varios comandos para ayudar con esto.

Listando Contenedores

  • Listar Contenedores en Ejecución:

    docker ps

  • Listar Todos los Contenedores (incluyendo los detenidos):

    docker ps -a

Deteniendo y Eliminando Contenedores

  • Detener un Contenedor en Ejecución:

    docker stop <container_id_or_name>

  • Eliminar un Contenedor Detenido:

    docker rm <container_id_or_name>

Ejemplo: Deteniendo y Eliminando el Contenedor Nginx

  1. Detener el Contenedor:

    docker stop mynginx

  2. Eliminar el Contenedor:

    docker rm mynginx

Conclusión

Ejecutar contenedores es una habilidad fundamental en Docker. Al entender cómo descargar imágenes, ejecutar contenedores y gestionarlos, puedes comenzar a aprovechar el poder de Docker para desplegar y gestionar aplicaciones de manera eficiente. Practica ejecutando diferentes contenedores y usando varias opciones con el comando docker run para familiarizarte más con Docker.
