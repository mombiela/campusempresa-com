Gestión de Contenedores

En esta sección, cubriremos cómo gestionar contenedores Docker de manera efectiva. Esto incluye iniciar, detener, inspeccionar y eliminar contenedores, así como gestionar los registros y la red de los contenedores. Al final de esta sección, tendrás una comprensión sólida de cómo controlar e interactuar con los contenedores Docker.

Iniciar y Detener Contenedores

Gestionar el ciclo de vida de los contenedores es fundamental para usar Docker. Aquí, aprenderemos cómo iniciar, detener y reiniciar contenedores.

Iniciar un Contenedor

Para iniciar un contenedor, utiliza el comando docker run. Este comando crea e inicia un nuevo contenedor a partir de una imagen especificada.

docker run -d --name my_container nginx
  • -d: Ejecuta el contenedor en modo desacoplado (en segundo plano).
  • --name my_container: Asigna un nombre al contenedor.
  • nginx: La imagen a usar para el contenedor.

Detener un Contenedor

Para detener un contenedor en ejecución, utiliza el comando docker stop seguido del nombre o ID del contenedor.

docker stop my_container

Reiniciar un Contenedor

Para reiniciar un contenedor detenido, utiliza el comando docker start.

docker start my_container

También puedes reiniciar un contenedor en ejecución utilizando el comando docker restart.

docker restart my_container

Inspeccionar Contenedores

Inspeccionar contenedores te permite ver información detallada sobre ellos, como su configuración, estado y uso de recursos.

Ver Detalles del Contenedor

Utiliza el comando docker inspect para obtener información detallada sobre un contenedor.

docker inspect my_container

Este comando devuelve un objeto JSON que contiene todos los detalles sobre el contenedor.

Verificar los Registros del Contenedor

Para ver los registros de un contenedor, utiliza el comando docker logs.

docker logs my_container

También puedes seguir los registros en tiempo real utilizando la bandera -f.

docker logs -f my_container

Eliminar Contenedores

Eliminar contenedores que ya no se necesitan ayuda a liberar recursos del sistema.

Eliminar un Contenedor Detenido

Para eliminar un contenedor detenido, utiliza el comando docker rm.

docker rm my_container

Eliminar un Contenedor en Ejecución

Para eliminar un contenedor en ejecución, necesitas detenerlo primero o usar la bandera -f para eliminarlo forzosamente.

docker rm -f my_container

Gestión de la Red de Contenedores

Docker proporciona varias opciones de red para conectar contenedores entre sí y con redes externas.

Listar Redes

Para listar todas las redes Docker disponibles, utiliza el comando docker network ls.

docker network ls

Crear una Red

Para crear una nueva red, utiliza el comando docker network create.

docker network create my_network

Conectar un Contenedor a una Red

Para conectar un contenedor en ejecución a una red, utiliza el comando docker network connect.

docker network connect my_network my_container

Desconectar un Contenedor de una Red

Para desconectar un contenedor de una red, utiliza el comando docker network disconnect.

docker network disconnect my_network my_container

Resumen

En esta sección, cubrimos los comandos y conceptos esenciales para gestionar contenedores Docker. Aprendiste cómo iniciar, detener y reiniciar contenedores, inspeccionar sus detalles, ver registros y eliminarlos cuando ya no son necesarios. Además, exploramos cómo gestionar la red de contenedores, incluyendo la creación de redes y la conexión o desconexión de contenedores de ellas. Dominar estas habilidades te permitirá controlar e interactuar efectivamente con los contenedores Docker en tus entornos de desarrollo y producción.
