Introducción

En esta sección, cubriremos los conceptos fundamentales de la construcción y ejecución de imágenes Docker. Las imágenes Docker son la base de los contenedores, y entender cómo crearlas y gestionarlas es crucial para cualquier usuario de Docker.

Conceptos Clave

  • Dockerfile: Un documento de texto que contiene todos los comandos para ensamblar una imagen.
  • Imagen: Un paquete ligero, autónomo y ejecutable que incluye todo lo necesario para ejecutar un software.
  • Contenedor: Una instancia en tiempo de ejecución de una imagen.

Construcción de Imágenes Docker

Escribiendo un Dockerfile

Un Dockerfile es un script que contiene una serie de instrucciones sobre cómo construir una imagen Docker. Aquí hay un ejemplo simple:

# Usar una imagen oficial de Python como imagen base
FROM python:3.8-slim

# Establecer el directorio de trabajo en el contenedor
WORKDIR /app

# Copiar el contenido del directorio actual al contenedor en /app
COPY . /app

# Instalar cualquier paquete necesario especificado en requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Hacer que el puerto 80 esté disponible para el mundo fuera de este contenedor
EXPOSE 80

# Definir una variable de entorno
ENV NAME World

# Ejecutar app.py cuando el contenedor se inicie
CMD ["python", "app.py"]

Construcción de la Imagen

Para construir una imagen a partir de un Dockerfile, usa el comando docker build. Aquí está la sintaxis:

docker build -t my-python-app .
  • -t my-python-app: Etiqueta la imagen con el nombre my-python-app.
  • .: Especifica el contexto de construcción, que es el directorio actual.

Entendiendo el Proceso de Construcción

Cuando ejecutas docker build, Docker realiza los siguientes pasos:

  1. Lee el Dockerfile.
  2. Ejecuta cada instrucción en el Dockerfile.
  3. Crea una nueva capa de imagen para cada instrucción.
  4. Etiqueta la imagen final con el nombre especificado.

Ejecución de Imágenes Docker

Ejecutando un Contenedor

Para ejecutar un contenedor a partir de una imagen, usa el comando docker run. Aquí hay un ejemplo:

docker run -p 4000:80 my-python-app
  • -p 4000:80: Mapea el puerto 4000 en el host al puerto 80 en el contenedor.
  • my-python-app: El nombre de la imagen a ejecutar.

Gestión de Contenedores

  • Listar contenedores en ejecución: docker ps
  • Detener un contenedor: docker stop <container_id>
  • Eliminar un contenedor: docker rm <container_id>

Ejemplo: Ejecutando una Aplicación Web

Supongamos que tienes una simple aplicación web Flask. Aquí está cómo puedes construirla y ejecutarla usando Docker:

  1. Crear un Dockerfile:

    FROM python:3.8-slim
WORKDIR /app
COPY . /app
RUN pip install --no-cache-dir -r requirements.txt
EXPOSE 5000
CMD ["python", "app.py"]

  2. Construir la imagen:

    docker build -t flask-app .

  3. Ejecutar el contenedor:

    docker run -p 5000:5000 flask-app

Conclusión

En esta sección, cubrimos los conceptos básicos de la construcción y ejecución de imágenes Docker. Aprendimos cómo escribir un Dockerfile, construir una imagen y ejecutar un contenedor. Estas habilidades son fundamentales para trabajar con Docker y desplegar aplicaciones en un entorno de contenedores. A medida que progreses, aprenderás técnicas más avanzadas para optimizar y gestionar imágenes y contenedores Docker.
