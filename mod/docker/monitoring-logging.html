Monitoreo y Registro en Docker
Mejores Prácticas de Seguridad ►

El monitoreo y el registro son aspectos cruciales para gestionar contenedores Docker de manera efectiva. Este tema te guiará desde los conceptos básicos hasta técnicas avanzadas para el monitoreo y registro en Docker.

Introducción al Monitoreo y Registro

El monitoreo y el registro son esenciales para:

  • Asegurar la salud y el rendimiento de tus aplicaciones.
  • Depurar y solucionar problemas.
  • Mantener la seguridad y el cumplimiento.

Conceptos Básicos

Monitoreo

  • Definición: El proceso de recopilar, analizar y usar información para rastrear el rendimiento y la salud de tus aplicaciones e infraestructura.
  • Métricas Clave: Uso de CPU, uso de memoria, I/O de disco, I/O de red, etc.

Registro

  • Definición: El proceso de registrar eventos de aplicaciones y sistemas para proporcionar un historial de actividad.
  • Tipos de Registros: Registros de aplicaciones, registros del sistema, registros del demonio de Docker, etc.

Configuración de Monitoreo Básico

Usando Docker Stats

Docker proporciona un comando simple para monitorear el uso de recursos del contenedor.

docker stats

Este comando muestra una transmisión en vivo de las estadísticas de uso de recursos del(los) contenedor(es).

Ejemplo de Salida

CONTAINER ID   NAME           CPU %     MEM USAGE / LIMIT     MEM %     NET I/O           BLOCK I/O         PIDS
d1f2c3b4e5f6   my_container   0.07%     12.34MiB / 1.952GiB   0.62%     1.23kB / 2.34kB   0B / 0B           2

Configuración de Registro Básico

Usando Docker Logs

Docker proporciona un comando para obtener los registros de un contenedor.

docker logs <container_id>

Ejemplo de Uso

docker logs d1f2c3b4e5f6

Este comando mostrará los registros del contenedor con ID d1f2c3b4e5f6.

Monitoreo Avanzado

Usando Prometheus y Grafana

Prometheus es un sistema de monitoreo de código abierto, y Grafana es una solución de análisis y monitoreo de código abierto.

Configuración de Prometheus

  1. Crear un archivo de configuración de Prometheus (prometheus.yml):

    global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'docker'
    static_configs:
      - targets: ['<docker_host>:<docker_port>']

  2. Ejecutar Prometheus como un contenedor Docker:

    docker run -d -p 9090:9090 -v /path/to/prometheus.yml:/etc/prometheus/prometheus.yml prom/prometheus

Configuración de Grafana

  1. Ejecutar Grafana como un contenedor Docker:

    docker run -d -p 3000:3000 grafana/grafana

  2. Configurar Grafana:

    • Accede a Grafana en http://localhost:3000.
    • Agrega Prometheus como una fuente de datos.
    • Crea paneles para visualizar métricas.

Usando cAdvisor

cAdvisor (Container Advisor) proporciona a los usuarios de contenedores una comprensión del uso de recursos y las características de rendimiento de sus contenedores en ejecución.

Ejecutando cAdvisor

docker run \
  --volume=/:/rootfs:ro \
  --volume=/var/run:/var/run:ro \
  --volume=/sys:/sys:ro \
  --volume=/var/lib/docker/:/var/lib/docker:ro \
  --publish=8080:8080 \
  --detach=true \
  --name=cadvisor \
  google/cadvisor:latest

Accede a cAdvisor en http://localhost:8080.

Registro Avanzado

Usando ELK Stack (Elasticsearch, Logstash, Kibana)

La pila ELK es un conjunto poderoso de herramientas para gestionar y analizar registros.

Configuración de Elasticsearch

docker run -d -p 9200:9200 -p 9300:9300 -e "discovery.type=single-node" elasticsearch:7.10.1

Configuración de Logstash

  1. Crear un archivo de configuración de Logstash (logstash.conf):

    input {
  docker {
    host => "unix:///var/run/docker.sock"
  }
}
output {
  elasticsearch {
    hosts => ["localhost:9200"]
  }
}

  2. Ejecutar Logstash como un contenedor Docker:

    docker run -d -v /path/to/logstash.conf:/usr/share/logstash/pipeline/logstash.conf logstash:7.10.1

Configuración de Kibana

docker run -d -p 5601:5601 kibana:7.10.1

Accede a Kibana en http://localhost:5601.

Conclusión

El monitoreo y el registro son críticos para mantener la salud, el rendimiento y la seguridad de tus aplicaciones Dockerizadas. Comenzando con comandos básicos como docker stats y docker logs, puedes progresar a configuraciones más avanzadas usando herramientas como Prometheus, Grafana, cAdvisor y la pila ELK. Al implementar estas prácticas, puedes asegurar que tus aplicaciones funcionen de manera fluida y eficiente, y puedes diagnosticar y resolver rápidamente cualquier problema que surja.
