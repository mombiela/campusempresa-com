Introducción a Docker Compose

Docker Compose es una herramienta para definir y ejecutar aplicaciones Docker de múltiples contenedores. Con Compose, utilizas un archivo YAML para configurar los servicios de tu aplicación. Luego, con un solo comando, creas y arrancas todos los servicios desde tu configuración.

Conceptos Clave

  • Servicio: Un servicio es un contenedor en tu aplicación. Cada servicio ejecuta una imagen y puede escalarse especificando el número de contenedores a ejecutar.
  • Volumen: Los volúmenes se utilizan para persistir datos generados y utilizados por los contenedores Docker.
  • Red: Las redes permiten que los contenedores se comuniquen entre sí, ya sea en el mismo host o a través de múltiples hosts.

Configuración de Docker Compose

Para comenzar con Docker Compose, necesitas instalar Docker y Docker Compose en tu máquina.

Instalación

  • Docker: Sigue la guía oficial de instalación de Docker para tu sistema operativo.
  • Docker Compose: Docker Compose está incluido en Docker Desktop para Windows y Mac. Para Linux, puedes instalarlo usando el siguiente comando: 
    sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

Creación de un Archivo Docker Compose

Un archivo Docker Compose es un archivo YAML que define servicios, redes y volúmenes para una aplicación Docker.

Estructura Básica

Aquí tienes un ejemplo básico de un archivo docker-compose.yml:

version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
  db:
    image: postgres:latest
    environment:
      POSTGRES_PASSWORD: example

Explicación

  • version: Especifica la versión del formato del archivo Compose.
  • services: Define los servicios a ejecutar.
    • web: Un servicio llamado web que utiliza la imagen nginx:latest.
      • ports: Mapea el puerto 80 en el host al puerto 80 en el contenedor.
    • db: Un servicio llamado db que utiliza la imagen postgres:latest.
      • environment: Establece variables de entorno para el contenedor.

Ejecutando Docker Compose

Para iniciar los servicios definidos en tu archivo docker-compose.yml, navega al directorio que contiene el archivo y ejecuta:

docker-compose up

Este comando iniciará todos los servicios en primer plano. Para ejecutarlos en segundo plano, usa:

docker-compose up -d

Gestión de Servicios

Docker Compose proporciona varios comandos para gestionar tus servicios.

Comandos Comunes

  • Iniciar Servicios: docker-compose start
  • Detener Servicios: docker-compose stop
  • Reiniciar Servicios: docker-compose restart
  • Ver Logs: docker-compose logs
  • Escalar Servicios: docker-compose up --scale web=3 -d

Ejemplo

Para escalar el servicio web a 3 instancias, usarías:

docker-compose up --scale web=3 -d

Uso de Volúmenes y Redes

Los volúmenes y redes son esenciales para la persistencia de datos y la comunicación entre servicios.

Volúmenes

Define volúmenes en tu archivo docker-compose.yml:

services:
  db:
    image: postgres:latest
    volumes:
      - db-data:/var/lib/postgresql/data
volumes:
  db-data:

Redes

Define redes personalizadas para tus servicios:

services:
  web:
    image: nginx:latest
    networks:
      - webnet
  db:
    image: postgres:latest
    networks:
      - webnet
networks:
  webnet:

Configuración Avanzada

Docker Compose permite configuraciones más avanzadas como el uso de archivos de entorno, la extensión de servicios y el uso de múltiples archivos Compose.

Archivos de Entorno

Puedes usar un archivo de entorno para establecer variables de entorno:

services:
  web:
    image: nginx:latest
    env_file:
      - .env

Extensión de Servicios

Puedes extender servicios existentes para reutilizar configuraciones:

services:
  base:
    image: nginx:latest
  web:
    extends:
      service: base
      file: common-services.yml

Múltiples Archivos Compose

Puedes usar múltiples archivos Compose para sobrescribir configuraciones:

docker-compose -f docker-compose.yml -f docker-compose.override.yml up -d

Conclusión

Gestionar proyectos con Docker Compose implica entender la estructura del archivo docker-compose.yml, ejecutar y gestionar servicios, y usar volúmenes y redes para la persistencia de datos y la comunicación. Con estas herramientas, puedes gestionar eficientemente aplicaciones Docker de múltiples contenedores, haciendo que tus procesos de desarrollo y despliegue sean más eficientes y efectivos.
