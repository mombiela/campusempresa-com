Introducción a las Redes en Docker

Las redes en Docker son un aspecto crucial de las aplicaciones contenedorizadas, permitiendo que los contenedores se comuniquen entre sí y con sistemas externos. Entender las redes en Docker es esencial para desplegar y gestionar aplicaciones de manera efectiva.

Conceptos Clave

  • Redes de Contenedores: Cómo se comunican los contenedores dentro de un host Docker.
  • Controladores de Red: Diferentes tipos de controladores de red proporcionados por Docker.
  • Redes Bridge: El controlador de red predeterminado para los contenedores Docker.
  • Redes Host: Los contenedores comparten el espacio de nombres de red del host.
  • Redes Overlay: Utilizadas para redes multi-host.
  • Redes Macvlan: Asigna una dirección MAC a cada contenedor para acceso directo a la red.
  • Configuración de Red: Personalización de la configuración de red para los contenedores.

Redes Bridge

Las redes bridge son el tipo de red predeterminado creado por Docker. Permiten que los contenedores se comuniquen entre sí en el mismo host.

Creando una Red Bridge

docker network create my_bridge_network

Ejecutando Contenedores en una Red Bridge

docker run -d --name container1 --network my_bridge_network nginx
docker run -d --name container2 --network my_bridge_network nginx

Inspeccionando la Red

docker network inspect my_bridge_network

Redes Host

Las redes host permiten que los contenedores usen directamente la pila de red del host. Esto puede ser útil para aplicaciones sensibles al rendimiento.

Ejecutando un Contenedor en la Red Host

docker run -d --name container3 --network host nginx

Redes Overlay

Las redes overlay permiten la comunicación entre contenedores a través de diferentes hosts Docker, lo cual es esencial para aplicaciones distribuidas.

Creando una Red Overlay

docker network create -d overlay my_overlay_network

Ejecutando Contenedores en una Red Overlay

docker run -d --name container4 --network my_overlay_network nginx
docker run -d --name container5 --network my_overlay_network nginx

Redes Macvlan

Las redes macvlan asignan una dirección MAC única a cada contenedor, permitiéndoles aparecer como dispositivos físicos en la red.

Creando una Red Macvlan

docker network create -d macvlan \
  --subnet=192.168.1.0/24 \
  --gateway=192.168.1.1 \
  -o parent=eth0 my_macvlan_network

Ejecutando un Contenedor en una Red Macvlan

docker run -d --name container6 --network my_macvlan_network nginx

Configuración de Red

Docker permite la personalización de la configuración de red para ajustarse a requisitos específicos.

Servidores DNS Personalizados

docker run -d --name container7 --network my_bridge_network --dns 8.8.8.8 nginx

Exponiendo Puertos

docker run -d --name container8 -p 8080:80 nginx

Resumen

Entender las redes en Docker es esencial para desplegar y gestionar aplicaciones contenedorizadas. Docker proporciona varios controladores de red, incluyendo bridge, host, overlay y macvlan, cada uno adecuado para diferentes casos de uso. Al dominar estos conceptos, puedes asegurar una comunicación eficiente y segura entre tus contenedores y sistemas externos.
