Introducción a Dockerfile

Un Dockerfile es un documento de texto que contiene todos los comandos que un usuario podría llamar en la línea de comandos para ensamblar una imagen. Usando docker build, los usuarios pueden crear una compilación automatizada que ejecuta varias instrucciones de línea de comandos en sucesión.

Conceptos clave de Dockerfile

  • FROM: Establece la imagen base para las instrucciones posteriores.
  • RUN: Ejecuta un comando en la shell.
  • CMD: Proporciona el comando predeterminado para el contenedor.
  • LABEL: Agrega metadatos a una imagen.
  • EXPOSE: Informa a Docker que el contenedor escucha en los puertos de red especificados en tiempo de ejecución.
  • ENV: Establece variables de entorno.
  • ADD: Copia nuevos archivos, directorios o URLs de archivos remotos desde src y los agrega al sistema de archivos del contenedor en la ruta dest.
  • COPY: Copia nuevos archivos o directorios desde src y los agrega al sistema de archivos del contenedor en la ruta dest.
  • ENTRYPOINT: Configura un contenedor que se ejecutará como un ejecutable.
  • VOLUME: Crea un punto de montaje con la ruta especificada y lo marca como que contiene volúmenes montados externamente desde el host nativo u otros contenedores.
  • USER: Establece el UID (o nombre de usuario) a usar al ejecutar la imagen y para cualquier instrucción RUN, CMD y ENTRYPOINT que le siga en el Dockerfile.
  • WORKDIR: Establece el directorio de trabajo para cualquier instrucción RUN, CMD, ENTRYPOINT, COPY y ADD que le siga en el Dockerfile.

Ejemplo básico de Dockerfile

Vamos a crear un Dockerfile simple para una aplicación Node.js.

# Usar una imagen oficial de Node.js como imagen base
FROM node:14

# Establecer el directorio de trabajo en el contenedor
WORKDIR /usr/src/app

# Copiar los archivos package.json y package-lock.json al directorio de trabajo
COPY package*.json ./

# Instalar los paquetes necesarios especificados en package.json
RUN npm install

# Copiar el resto del código de la aplicación al directorio de trabajo
COPY . .

# Hacer que el puerto 8080 esté disponible para el mundo exterior a este contenedor
EXPOSE 8080

# Definir la variable de entorno
ENV NODE_ENV=production

# Ejecutar la aplicación
CMD ["node", "app.js"]

Explicación del Dockerfile

  • FROM node:14: Esta línea especifica la imagen base. En este caso, es la imagen oficial de Node.js con la etiqueta 14.
  • WORKDIR /usr/src/app: Esto establece el directorio de trabajo dentro del contenedor en /usr/src/app.
  • COPY package.json ./*: Esto copia los archivos package.json y package-lock.json al directorio de trabajo.
  • RUN npm install: Esto ejecuta el comando npm install para instalar las dependencias.
  • COPY . .: Esto copia el resto del código de la aplicación al directorio de trabajo.
  • EXPOSE 8080: Esto expone el puerto 8080 al mundo exterior.
  • ENV NODE_ENV=production: Esto establece una variable de entorno NODE_ENV a production.
  • CMD ["node", "app.js"]: Esto especifica el comando para ejecutar la aplicación.

Conclusión

Entender los conceptos básicos de Dockerfile es crucial para crear imágenes Docker eficientes y efectivas. Al dominar las instrucciones clave y sus propósitos, puedes automatizar el proceso de construcción y asegurar que tus aplicaciones se ejecuten consistentemente en diferentes entornos. Practica escribiendo Dockerfiles para diferentes aplicaciones para obtener una mejor comprensión de estos conceptos.
