Creación de Imágenes Docker
Conceptos Básicos de Dockerfile ►

Crear imágenes Docker es una habilidad fundamental para cualquiera que trabaje con Docker. Este tema te guiará a través del proceso desde un nivel principiante hasta un nivel avanzado, cubriendo conceptos clave, ejemplos prácticos y ejercicios para solidificar tu comprensión.

Introducción a las Imágenes Docker

Las imágenes Docker son los planos para crear contenedores Docker. Contienen todo lo necesario para ejecutar una aplicación, incluyendo el código, el runtime, las bibliotecas, las variables de entorno y los archivos de configuración.

Conceptos Clave

  • Dockerfile: Un archivo de texto que contiene una serie de instrucciones sobre cómo construir una imagen Docker.
  • Imagen Base: El punto de partida para construir tu imagen Docker, a menudo una imagen oficial de Docker Hub.
  • Capas: Cada instrucción en un Dockerfile crea una nueva capa en la imagen, lo que ayuda en el almacenamiento en caché y la reutilización.
  • Etiquetado de Imágenes: Asignar una versión o identificador a una imagen, típicamente en el formato repository:tag.

Escribiendo un Dockerfile

Un Dockerfile es el núcleo de la creación de imágenes Docker. Define los pasos necesarios para ensamblar la imagen.

Estructura Básica del Dockerfile

Aquí hay un ejemplo simple de un Dockerfile para una aplicación Node.js:

# Usar una imagen oficial de Node.js como imagen base
FROM node:14

# Establecer el directorio de trabajo en el contenedor
WORKDIR /usr/src/app

# Copiar los archivos package.json y package-lock.json
COPY package*.json ./

# Instalar las dependencias de la aplicación
RUN npm install

# Copiar el resto del código de la aplicación
COPY . .

# Exponer el puerto de la aplicación
EXPOSE 8080

# Definir el comando para ejecutar la aplicación
CMD ["node", "app.js"]

Explicación

  • FROM node:14: Especifica la imagen base.
  • WORKDIR /usr/src/app: Establece el directorio de trabajo dentro del contenedor.
  • COPY package*.json ./: Copia los archivos de paquetes al directorio de trabajo.
  • RUN npm install: Instala las dependencias.
  • COPY . .: Copia el resto del código de la aplicación.
  • EXPOSE 8080: Expone el puerto 8080 para la aplicación.
  • CMD ["node", "app.js"]: Especifica el comando para ejecutar la aplicación.

Construcción de Imágenes Docker

Una vez que tienes un Dockerfile, puedes construir una imagen usando el comando docker build.

Construcción de una Imagen

Para construir una imagen desde el Dockerfile en el directorio actual, usa el siguiente comando:

docker build -t my-node-app:1.0 .

Explicación

  • docker build: El comando para construir una imagen Docker.
  • -t my-node-app:1.0: Etiqueta la imagen con el nombre my-node-app y la versión 1.0.
  • .: Especifica el contexto de construcción, que es el directorio actual.

Gestión de Imágenes Docker

Después de construir imágenes, necesitas gestionarlas de manera efectiva.

Listado de Imágenes

Para listar todas las imágenes Docker en tu sistema, usa:

docker images

Eliminación de Imágenes

Para eliminar una imagen, usa:

docker rmi my-node-app:1.0

Etiquetado de Imágenes

Para etiquetar una imagen existente con una nueva etiqueta, usa:

docker tag my-node-app:1.0 my-node-app:latest

Instrucciones Avanzadas de Dockerfile

A medida que te sientas más cómodo con Docker, puedes usar instrucciones más avanzadas para optimizar tus imágenes.

Construcciones Multi-Etapa

Las construcciones multi-etapa ayudan a crear imágenes más pequeñas y eficientes separando el entorno de construcción del entorno de ejecución.

# Etapa 1: Construcción
FROM node:14 AS build
WORKDIR /usr/src/app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# Etapa 2: Producción
FROM node:14-alpine
WORKDIR /usr/src/app
COPY --from=build /usr/src/app/dist ./dist
EXPOSE 8080
CMD ["node", "dist/app.js"]

Explicación

  • AS build: Nombra la primera etapa build.
  • COPY --from=build /usr/src/app/dist ./dist: Copia la aplicación construida desde la etapa build a la etapa de producción.

Conclusión

Crear imágenes Docker es una habilidad crucial para contenerizar aplicaciones. Al entender y practicar el uso de Dockerfiles, construir imágenes y gestionarlas, puedes desplegar y escalar tus aplicaciones de manera eficiente. A medida que avances, aprovechar técnicas como las construcciones multi-etapa puede optimizar aún más tus imágenes Docker para su uso en producción.
