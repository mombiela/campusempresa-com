Introducción

En esta sección, exploraremos el ciclo de vida de un contenedor Docker. Entender el ciclo de vida del contenedor es crucial para gestionar y orquestar contenedores de manera efectiva. Cubriremos las diversas etapas por las que pasa un contenedor desde su creación hasta su eliminación, junto con ejemplos prácticos y ejercicios.

Conceptos Clave

  • Imagen de Contenedor: Un paquete ligero, autónomo y ejecutable que incluye todo lo necesario para ejecutar una pieza de software.
  • Contenedor: Una instancia en tiempo de ejecución de una imagen.
  • Daemon de Docker: El servicio en segundo plano que se ejecuta en el host y gestiona los contenedores Docker.
  • CLI de Docker: Interfaz de Línea de Comandos utilizada para interactuar con el Daemon de Docker.

Etapas del Ciclo de Vida del Contenedor

El ciclo de vida de un contenedor Docker se puede desglosar en varias etapas:

  1. Crear
  2. Iniciar
  3. Ejecutar
  4. Pausar/Reanudar
  5. Detener
  6. Reiniciar
  7. Matar
  8. Eliminar

Crear

  • Descripción: El contenedor se crea a partir de una imagen Docker pero no se inicia.
  • Comando: docker create [OPTIONS] IMAGE [COMMAND] [ARG...]
  • Ejemplo: 
    docker create --name my_container ubuntu
  • Explicación: Este comando crea un contenedor llamado my_container a partir de la imagen ubuntu.

Iniciar

  • Descripción: El contenedor se inicia y comienza a ejecutarse.
  • Comando: docker start [OPTIONS] CONTAINER
  • Ejemplo: 
    docker start my_container
  • Explicación: Este comando inicia el my_container previamente creado.

Ejecutar

  • Descripción: Combina crear e iniciar en un solo paso.
  • Comando: docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
  • Ejemplo: 
    docker run --name my_container -d ubuntu
  • Explicación: Este comando crea e inicia un contenedor llamado my_container a partir de la imagen ubuntu en modo desacoplado (-d).

Pausar/Reanudar

  • Descripción: Suspende temporalmente todos los procesos en el contenedor.
  • Comandos:
    • Pausar: docker pause CONTAINER
    • Reanudar: docker unpause CONTAINER
  • Ejemplo: 
    docker pause my_container
docker unpause my_container
  • Explicación: Estos comandos pausan y reanudan el my_container, respectivamente.

Detener

  • Descripción: Detiene el contenedor de manera ordenada.
  • Comando: docker stop [OPTIONS] CONTAINER
  • Ejemplo: 
    docker stop my_container
  • Explicación: Este comando detiene el my_container de manera ordenada.

Reiniciar

  • Descripción: Detiene y luego inicia el contenedor.
  • Comando: docker restart [OPTIONS] CONTAINER
  • Ejemplo: 
    docker restart my_container
  • Explicación: Este comando reinicia el my_container.

Matar

  • Descripción: Detiene el contenedor de manera forzada.
  • Comando: docker kill [OPTIONS] CONTAINER
  • Ejemplo: 
    docker kill my_container
  • Explicación: Este comando detiene forzosamente el my_container.

Eliminar

  • Descripción: Elimina el contenedor.
  • Comando: docker rm [OPTIONS] CONTAINER
  • Ejemplo: 
    docker rm my_container
  • Explicación: Este comando elimina el my_container del host Docker.

Resumen

Entender el ciclo de vida de un contenedor Docker es esencial para una gestión efectiva de contenedores. Las etapas del ciclo de vida incluyen crear, iniciar, ejecutar, pausar, detener, reiniciar, matar y eliminar contenedores. Cada etapa tiene comandos Docker específicos asociados, que te permiten controlar el estado de tus contenedores. Al dominar estos comandos, puedes gestionar eficientemente tus aplicaciones en contenedores.

Ejercicios

  1. Crear e Iniciar un Contenedor:

    • Crea un contenedor a partir de la imagen nginx e inícialo.
    docker create --name web_server nginx
docker start web_server

  2. Pausar y Reanudar un Contenedor:

    • Pausa y luego reanuda el contenedor web_server.
    docker pause web_server
docker unpause web_server

  3. Detener y Eliminar un Contenedor:

    • Detén y luego elimina el contenedor web_server.
    docker stop web_server
docker rm web_server

Al practicar estos ejercicios, obtendrás experiencia práctica con el ciclo de vida del contenedor Docker, reforzando tu comprensión de cada etapa y sus comandos asociados.
