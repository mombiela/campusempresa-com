Introducción

La Integración Continua (CI) y el Despliegue Continuo (CD) son prácticas esenciales en el desarrollo de software moderno. Ayudan a automatizar el proceso de integración de cambios de código, pruebas y despliegue de aplicaciones. En esta sección, exploraremos cómo implementar una canalización CI/CD utilizando Azure DevOps.

Conceptos Clave

  • Integración Continua (CI): La práctica de integrar automáticamente los cambios de código de múltiples contribuyentes en un repositorio compartido varias veces al día.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio que pasa las pruebas automatizadas a producción.
  • Azure DevOps: Un conjunto de herramientas de desarrollo proporcionadas por Microsoft para apoyar todo el ciclo de vida del desarrollo de software.

Configuración de Azure DevOps

  1. Crear una Organización en Azure DevOps:

    • Ve al portal de Azure DevOps.
    • Inicia sesión con tu cuenta de Microsoft.
    • Haz clic en "Crear nueva organización" y sigue las indicaciones.

  2. Crear un Proyecto:

    • Una vez que tu organización esté configurada, haz clic en "Nuevo Proyecto."
    • Proporciona un nombre y una descripción para tu proyecto.
    • Elige la visibilidad (pública o privada) y haz clic en "Crear."

Configuración de Repositorios

  • Crear un Repositorio:

    • Navega a Repos > Archivos.
    • Haz clic en "Nuevo Repositorio."
    • Proporciona un nombre e inicialízalo con un archivo README.

  • Clonar el Repositorio:

    git clone https://dev.azure.com/tuorganizacion/tuprojecto/_git/turepositorio
cd turepositorio

Configuración de la Integración Continua (CI)

  1. Crear una Canalización de Construcción:

    • Navega a Pipelines > Pipelines.
    • Haz clic en "Nueva Canalización."
    • Selecciona el repositorio que deseas construir.
    • Elige la plantilla que se ajuste a tu proyecto (por ejemplo, ASP.NET Core, Node.js).

  2. Configurar el Archivo YAML:

    • Azure DevOps utiliza archivos YAML para definir la canalización de construcción.
    • Ejemplo para una aplicación Node.js: 
      trigger:
  - main

pool:
  vmImage: 'ubuntu-latest'

steps:
  - task: UseNode@2
    inputs:
      version: '14.x'
  - script: |
      npm install
      npm test
    displayName: 'Instalar y Probar'

  3. Ejecutar la Canalización:

    • Guarda y ejecuta la canalización.
    • Monitorea el proceso de construcción y asegúrate de que se complete con éxito.

Configuración del Despliegue Continuo (CD)

  1. Crear una Canalización de Despliegue:

    • Navega a Pipelines > Releases.
    • Haz clic en "Nueva Canalización."
    • Selecciona el artefacto de construcción de la canalización CI.
    • Añade etapas para el despliegue (por ejemplo, Dev, QA, Producción).

  2. Configurar Tareas de Despliegue:

    • Ejemplo para desplegar en un Azure App Service: 
      stages:
  - stage: Deploy
    jobs:
      - job: DeployJob
        pool:
          vmImage: 'ubuntu-latest'
        steps:
          - task: AzureWebApp@1
            inputs:
              azureSubscription: 'tu-suscripción-azure'
              appName: 'tu-nombre-app-service'
              package: '$(System.DefaultWorkingDirectory)/**/*.zip'

  3. Activar el Despliegue:

    • Define activadores para crear automáticamente un despliegue cuando una construcción se complete.
    • Monitorea el proceso de despliegue y asegúrate de que se complete con éxito.

Monitoreo y Mantenimiento

  • Monitorear Canalizaciones:

    • Utiliza los paneles de Azure DevOps para monitorear el estado de tus canalizaciones.
    • Configura alertas y notificaciones para fallos en la construcción y el despliegue.

  • Mantener Canalizaciones:

    • Actualiza regularmente las configuraciones de tus canalizaciones para adaptarte a los cambios en tu proyecto.
    • Optimiza el rendimiento de la canalización almacenando en caché las dependencias y paralelizando tareas.

Conclusión

Implementar una canalización CI/CD en Azure DevOps implica configurar repositorios, configurar canalizaciones de construcción y despliegue, y monitorear y mantener continuamente estas canalizaciones. Al automatizar los procesos de integración y despliegue, puedes asegurar una entrega más rápida y confiable de tus aplicaciones de software.
