Introducción a Azure Key Vault

Azure Key Vault es un servicio en la nube que proporciona un almacén seguro para secretos, claves y certificados. Ayuda a proteger claves criptográficas y secretos utilizados por aplicaciones y servicios en la nube.

Conceptos Clave

  • Secretos: Almacenar de manera segura y controlar estrictamente el acceso a tokens, contraseñas, certificados, claves API y otros secretos.
  • Claves: Gestionar y usar claves criptográficas para cifrado, descifrado y firma.
  • Certificados: Proveer, gestionar y desplegar certificados SSL/TLS públicos y privados.

Configuración de Azure Key Vault

Para comenzar a usar Azure Key Vault, necesitas crear un Key Vault en tu suscripción de Azure.

Pasos para Crear un Key Vault

  1. Navegar al Portal de Azure: Ve al Portal de Azure.
  2. Crear un Recurso: Haz clic en "Crear un recurso" y busca "Key Vault".
  3. Configurar Key Vault:
    • Nombre: Proporciona un nombre único para tu Key Vault.
    • Región: Selecciona la región donde deseas desplegar tu Key Vault.
    • Nivel de Precios: Elige el nivel de precios (Estándar o Premium).
    • Grupo de Recursos: Selecciona un grupo de recursos existente o crea uno nuevo.
  4. Revisar y Crear: Revisa tus configuraciones y haz clic en "Crear".

Ejemplo: Creando un Key Vault usando Azure CLI

# Variables
RESOURCE_GROUP="miGrupoDeRecursos"
KEY_VAULT_NAME="miKeyVault"
LOCATION="eastus"

# Crear Grupo de Recursos
az group create --name $RESOURCE_GROUP --location $LOCATION

# Crear Key Vault
az keyvault create --name $KEY_VAULT_NAME --resource-group $RESOURCE_GROUP --location $LOCATION

Gestión de Secretos en Azure Key Vault

Los secretos son valores que deseas mantener confidenciales, como claves API, contraseñas o cadenas de conexión.

Añadiendo un Secreto

Puedes añadir secretos a tu Key Vault usando el Portal de Azure, Azure CLI o Azure PowerShell.

Ejemplo: Añadiendo un Secreto usando Azure CLI

# Variables
SECRET_NAME="MiSecreto"
SECRET_VALUE="MiValorSecreto"

# Añadir Secreto
az keyvault secret set --vault-name $KEY_VAULT_NAME --name $SECRET_NAME --value $SECRET_VALUE

Recuperando un Secreto

Para recuperar un secreto, puedes usar Azure CLI o los SDKs de Azure.

Ejemplo: Recuperando un Secreto usando Azure CLI

# Recuperar Secreto
az keyvault secret show --vault-name $KEY_VAULT_NAME --name $SECRET_NAME

Gestión de Claves en Azure Key Vault

Las claves se usan para operaciones de cifrado, descifrado y firma.

Creando una Clave

Puedes crear claves en tu Key Vault usando el Portal de Azure, Azure CLI o Azure PowerShell.

Ejemplo: Creando una Clave usando Azure CLI

# Variables
KEY_NAME="MiClave"

# Crear Clave
az keyvault key create --vault-name $KEY_VAULT_NAME --name $KEY_NAME --protection software

Usando una Clave para Cifrado

Puedes usar las claves almacenadas en Azure Key Vault para cifrar datos.

Ejemplo: Cifrando Datos usando Azure CLI

# Variables
PLAINTEXT="Hola, Mundo!"

# Cifrar Datos
az keyvault key encrypt --vault-name $KEY_VAULT_NAME --name $KEY_NAME --algorithm RSA-OAEP --value $PLAINTEXT

Gestión de Certificados en Azure Key Vault

Los certificados se usan para asegurar comunicaciones y establecer identidad.

Importando un Certificado

Puedes importar certificados en tu Key Vault usando el Portal de Azure, Azure CLI o Azure PowerShell.

Ejemplo: Importando un Certificado usando Azure CLI

# Variables
CERT_NAME="MiCertificado"
CERT_FILE="/ruta/al/certificado.pfx"
CERT_PASSWORD="contraseña"

# Importar Certificado
az keyvault certificate import --vault-name $KEY_VAULT_NAME --name $CERT_NAME --file $CERT_FILE --password $CERT_PASSWORD

Políticas de Acceso y Seguridad

Las políticas de acceso en Azure Key Vault controlan el acceso a los secretos, claves y certificados almacenados en el almacén.

Configuración de Políticas de Acceso

Puedes configurar políticas de acceso para otorgar o denegar acceso a usuarios o aplicaciones específicas.

Ejemplo: Configuración de Políticas de Acceso usando Azure CLI

# Variables
USER_OBJECT_ID="id-objeto-usuario"

# Establecer Política de Acceso
az keyvault set-policy --name $KEY_VAULT_NAME --object-id $USER_OBJECT_ID --secret-permissions get list --key-permissions get list --certificate-permissions get list

Integración de Azure Key Vault con Aplicaciones

Puedes integrar Azure Key Vault con tus aplicaciones para gestionar de manera segura secretos, claves y certificados.

Ejemplo: Usando Azure Key Vault en una Aplicación .NET

using Azure.Identity;
using Azure.Security.KeyVault.Secrets;

var client = new SecretClient(new Uri("https://<TuNombreDeKeyVault>.vault.azure.net/"), new DefaultAzureCredential());
KeyVaultSecret secret = client.GetSecret("MiSecreto");

Console.WriteLine($"Valor del Secreto: {secret.Value}");

Conclusión

Azure Key Vault es una herramienta poderosa para gestionar secretos, claves y certificados de manera segura y escalable. Al entender cómo crear y gestionar Key Vaults, añadir y recuperar secretos, crear y usar claves, e importar certificados, puedes mejorar la seguridad de tus aplicaciones y servicios. Integrar Azure Key Vault con tus aplicaciones asegura que la información sensible se almacene de manera segura y sea accesible solo por entidades autorizadas.
