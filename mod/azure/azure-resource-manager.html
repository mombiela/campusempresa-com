Azure Resource Manager (ARM)
Despliegue de una Aplicación Multi-Capa ►

Azure Resource Manager (ARM) es el servicio de implementación y gestión para Azure. Proporciona una capa de gestión consistente que te permite crear, actualizar y eliminar recursos en tu cuenta de Azure.

Introducción a Azure Resource Manager

  • Definición: ARM es un marco de gestión que te permite implementar, gestionar y monitorear recursos de Azure como un grupo.
  • Beneficios:
    • Consistencia: Asegura que los recursos se implementen en un estado consistente.
    • Seguridad: Proporciona control de acceso basado en roles (RBAC) para asegurar los recursos.
    • Etiquetado: Te permite etiquetar recursos para una gestión y facturación más sencilla.
    • Plantillas: Soporta Infraestructura como Código (IaC) a través de plantillas ARM.

Conceptos Clave

  • Grupo de Recursos: Un contenedor que contiene recursos relacionados para una solución de Azure.
  • Plantilla ARM: Un archivo JSON que define la infraestructura y configuración para tu proyecto.
  • Implementación: El proceso de implementar recursos usando plantillas ARM.
  • Proveedor de Recursos: Un servicio que suministra recursos de Azure.

Trabajando con Grupos de Recursos

  • Creando un Grupo de Recursos:

    az group create --name myResourceGroup --location eastus

    Este comando crea un nuevo grupo de recursos llamado myResourceGroup en la región eastus.

  • Listando Grupos de Recursos:

    az group list --output table

    Este comando lista todos los grupos de recursos en tu suscripción en formato de tabla.

  • Eliminando un Grupo de Recursos:

    az group delete --name myResourceGroup --yes --no-wait

    Este comando elimina el grupo de recursos especificado y todos los recursos contenidos dentro de él.

Plantillas ARM

  • Estructura de una Plantilla ARM:

    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Storage/storageAccounts",
      "apiVersion": "2019-04-01",
      "name": "mystorageaccount",
      "location": "eastus",
      "sku": {
        "name": "Standard_LRS"
      },
      "kind": "StorageV2",
      "properties": {}
    }
  ]
}
    • $schema: Define la ubicación del archivo de esquema JSON que describe la versión del lenguaje de la plantilla.
    • contentVersion: Especifica la versión de la plantilla.
    • resources: Un array que contiene los recursos a ser implementados o actualizados.

  • Implementando una Plantilla ARM:

    az deployment group create --resource-group myResourceGroup --template-file template.json

    Este comando implementa los recursos definidos en template.json al grupo de recursos especificado.

Características Avanzadas de las Plantillas ARM

  • Parámetros: Te permiten pasar valores durante la implementación para personalizar las propiedades de los recursos.

    {
  "parameters": {
    "storageAccountType": {
      "type": "string",
      "defaultValue": "Standard_LRS",
      "allowedValues": [
        "Standard_LRS",
        "Standard_GRS",
        "Standard_ZRS",
        "Premium_LRS"
      ],
      "metadata": {
        "description": "Tipo de Cuenta de Almacenamiento"
      }
    }
  }
}

  • Variables: Almacenan valores que reutilizas en tu plantilla.

    {
  "variables": {
    "storageAccountName": "[concat('storage', uniqueString(resourceGroup().id))]"
  }
}

  • Salidas: Devuelven valores de los recursos implementados.

    {
  "outputs": {
    "storageAccountName": {
      "type": "string",
      "value": "[variables('storageAccountName')]"
    }
  }
}

Mejores Prácticas

  • Plantillas Modulares: Divide plantillas grandes en plantillas más pequeñas y reutilizables.
  • Control de Versiones: Almacena tus plantillas en un sistema de control de versiones como Git.
  • Pruebas: Prueba tus plantillas en un entorno de staging antes de implementarlas en producción.
  • Documentación: Documenta tus plantillas para hacerlas más fáciles de entender y mantener.

Conclusión

Azure Resource Manager (ARM) es una herramienta poderosa para gestionar recursos de Azure. Al entender y utilizar plantillas ARM, grupos de recursos y las diversas características de ARM, puedes lograr implementaciones consistentes, repetibles y automatizadas. Esto no solo mejora la eficiencia de la gestión de tus recursos de Azure, sino que también asegura que tu infraestructura sea escalable y mantenible.
