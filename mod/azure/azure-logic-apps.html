Azure Logic Apps
Aprendizaje Automático de Azure ►

Azure Logic Apps es un servicio en la nube que te ayuda a automatizar y orquestar tareas, procesos empresariales y flujos de trabajo. Este tema te guiará desde los conceptos básicos hasta los avanzados de Azure Logic Apps.

Introducción a Azure Logic Apps

  • ¿Qué son Azure Logic Apps?

    • Azure Logic Apps es un servicio basado en la nube que te permite automatizar flujos de trabajo e integrar aplicaciones, datos, servicios y sistemas.
    • Proporciona un diseñador visual para crear flujos de trabajo sin escribir código.

  • Características clave:

    • Diseñador visual: Interfaz de arrastrar y soltar para diseñar flujos de trabajo.
    • Conectores: Conectores preconstruidos para integrarse con varios servicios como Office 365, SQL Server y más.
    • Disparadores y acciones: Define qué inicia el flujo de trabajo (disparador) y qué sucede a continuación (acciones).

Configuración de tu primera Logic App

  • Requisitos previos:

    • Una cuenta de Azure.
    • Comprensión básica de conceptos en la nube.

  • Creación de una Logic App:

    1. Navegar al portal de Azure:
      • Ve al portal de Azure (https://portal.azure.com).
    2. Crear una Logic App:
      • Haz clic en "Crear un recurso" y busca "Logic App".
      • Haz clic en "Crear" y completa los detalles necesarios como grupo de recursos, nombre y región.
    3. Diseñar el flujo de trabajo:
      • Usa el diseñador visual para agregar un disparador y acciones.

Ejemplo: Enviar un correo electrónico cuando se sube un archivo a OneDrive

{
  "definition": {
    "$schema": "https://schema.management.azure.com/providers/Microsoft.Logic/schemas/2016-06-01/workflowdefinition.json#",
    "actions": {
      "Send_an_email_(V2)": {
        "inputs": {
          "body": {
            "Content": "A new file has been uploaded to OneDrive.",
            "Subject": "File Upload Notification",
            "To": "example@example.com"
          },
          "host": {
            "connection": {
              "name": "@parameters('$connections')['office365']['connectionId']"
            }
          }
        },
        "runAfter": {},
        "type": "ApiConnection"
      }
    },
    "triggers": {
      "When_a_file_is_created": {
        "inputs": {
          "host": {
            "connection": {
              "name": "@parameters('$connections')['onedrive']['connectionId']"
            }
          },
          "method": "get",
          "path": "/datasets/default/files"
        },
        "recurrence": {
          "frequency": "Minute",
          "interval": 5
        },
        "type": "ApiConnection"
      }
    }
  }
}

Conceptos avanzados de Logic Apps

  • Conectores personalizados:

    • Crea tus propios conectores para integrarte con servicios no disponibles en los conectores preconstruidos.
    • Útil para integrarse con sistemas internos o propietarios.

  • Manejo de errores:

    • Implementa manejo de errores para gestionar fallos en los flujos de trabajo.
    • Usa ámbitos y configura ajustes de ejecución posterior para manejar errores de manera elegante.

  • Flujos de trabajo con estado vs sin estado:

    • Con estado: Mantiene información de estado entre ejecuciones.
    • Sin estado: No mantiene información de estado, adecuado para escenarios de alto rendimiento.

Ejemplo: Manejo de errores con ámbitos

{
  "definition": {
    "actions": {
      "Scope": {
        "actions": {
          "Action1": {
            "type": "Http",
            "inputs": {
              "method": "GET",
              "uri": "https://example.com/api"
            }
          }
        },
        "runAfter": {},
        "type": "Scope"
      },
      "ErrorHandling": {
        "actions": {
          "SendEmail": {
            "type": "ApiConnection",
            "inputs": {
              "body": {
                "Content": "An error occurred in the workflow.",
                "Subject": "Error Notification",
                "To": "admin@example.com"
              },
              "host": {
                "connection": {
                  "name": "@parameters('$connections')['office365']['connectionId']"
                }
              }
            }
          }
        },
        "runAfter": {
          "Scope": [
            "Failed"
          ]
        },
        "type": "Scope"
      }
    },
    "triggers": {
      "manual": {
        "type": "Request",
        "kind": "Http",
        "inputs": {
          "schema": {}
        }
      }
    }
  }
}

Mejores prácticas

  • Flujos de trabajo modulares:

    • Divide flujos de trabajo complejos en componentes más pequeños y reutilizables.
    • Usa Logic Apps anidadas para una mejor manejabilidad.

  • Monitoreo y registro:

    • Habilita diagnósticos y registros para monitorear el rendimiento y la salud de tus Logic Apps.
    • Usa Azure Monitor y Application Insights para análisis detallados.

  • Seguridad:

    • Asegura tus Logic Apps usando Azure Active Directory (AAD) e identidades administradas.
    • Asegúrate de que los datos sensibles estén encriptados y el acceso esté controlado.

Conclusión

Azure Logic Apps proporciona una forma poderosa y flexible de automatizar flujos de trabajo e integrar varios servicios. Al comprender los conceptos básicos y explorar las características avanzadas, puedes crear flujos de trabajo eficientes y confiables para satisfacer las necesidades de tu negocio. Recuerda seguir las mejores prácticas para modularidad, monitoreo y seguridad para asegurar que tus Logic Apps sean robustas y mantenibles.
