Introducción

La Infraestructura Global de Azure es la columna vertebral de los servicios en la nube de Microsoft, proporcionando los recursos físicos y virtuales necesarios para ejecutar aplicaciones y servicios a nivel mundial. Comprender la infraestructura de Azure es crucial para diseñar, desplegar y gestionar aplicaciones de manera efectiva.

Conceptos Clave

  • Regiones: Áreas geográficas que contienen múltiples centros de datos.
  • Zonas de Disponibilidad: Ubicaciones físicamente separadas dentro de una región.
  • Centros de Datos: Instalaciones físicas que albergan servidores y equipos de red.
  • Ubicaciones de Borde: Sitios que proporcionan acceso de baja latencia a los servicios de Azure.
  • Grupos de Recursos: Contenedores lógicos para gestionar recursos.

Regiones

Las regiones de Azure son la base de su infraestructura global. Cada región es un conjunto de centros de datos desplegados dentro de una ubicación geográfica específica.

  • Ejemplos de Regiones de Azure:
    • Este de EE. UU.
    • Oeste de Europa
    • Sudeste Asiático

Elegir una Región

Al seleccionar una región, considere los siguientes factores:

  • Proximidad a los Usuarios: Elija una región cercana a su base de usuarios para reducir la latencia.
  • Requisitos de Cumplimiento: Algunas regiones cumplen con requisitos regulatorios específicos.
  • Disponibilidad de Servicios: No todos los servicios están disponibles en todas las regiones.

Zonas de Disponibilidad

Las Zonas de Disponibilidad son ubicaciones físicas únicas dentro de una región de Azure. Cada zona está compuesta por uno o más centros de datos equipados con energía, refrigeración y redes independientes.

  • Beneficios de las Zonas de Disponibilidad:
    • Alta Disponibilidad: Protege aplicaciones y datos de fallos en los centros de datos.
    • Aislamiento de Fallos: Cada zona está aislada para prevenir fallos en cascada.

Ejemplo: Desplegando una Máquina Virtual en una Zona de Disponibilidad

# Crear un grupo de recursos
az group create --name myResourceGroup --location eastus

# Crear una red virtual
az network vnet create --resource-group myResourceGroup --name myVnet --subnet-name mySubnet

# Crear una dirección IP pública
az network public-ip create --resource-group myResourceGroup --name myPublicIP

# Crear un grupo de seguridad de red
az network nsg create --resource-group myResourceGroup --name myNetworkSecurityGroup

# Crear una máquina virtual en una zona de disponibilidad específica
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --vnet-name myVnet \
  --subnet mySubnet \
  --public-ip-address myPublicIP \
  --nsg myNetworkSecurityGroup \
  --zone 1

Centros de Datos

Los centros de datos son las instalaciones físicas que albergan los servidores y equipos de red. Están diseñados para proporcionar alta disponibilidad y seguridad.

  • Características Clave:
    • Suministros de energía redundantes
    • Sistemas avanzados de refrigeración
    • Medidas robustas de seguridad física

Ubicaciones de Borde

Las ubicaciones de borde son sitios que almacenan contenido en caché más cerca de los usuarios, reduciendo la latencia y mejorando el rendimiento. Son parte de la Red de Entrega de Contenido (CDN) de Azure.

  • Casos de Uso:
    • Transmisión de medios
    • Entrega de contenido web
    • Aceleración de API

Grupos de Recursos

Los grupos de recursos son contenedores lógicos que ayudan a gestionar y organizar los recursos de Azure. Proporcionan una forma de agrupar recursos relacionados para una gestión y facturación más sencilla.

  • Mejores Prácticas:
    • Agrupar recursos por ciclo de vida y permisos.
    • Usar nombres significativos para los grupos de recursos.
    • Aplicar etiquetas para una mejor organización y gestión de costos.

Ejemplo: Creando un Grupo de Recursos

# Crear un grupo de recursos
az group create --name myResourceGroup --location eastus

Resumen

La Infraestructura Global de Azure es un sistema complejo y robusto diseñado para proporcionar alta disponibilidad, escalabilidad y seguridad para aplicaciones en la nube. Al comprender las regiones, zonas de disponibilidad, centros de datos, ubicaciones de borde y grupos de recursos, puede tomar decisiones informadas sobre dónde y cómo desplegar sus aplicaciones en Azure. Este conocimiento es esencial para optimizar el rendimiento, asegurar el cumplimiento y gestionar los costos de manera efectiva.
