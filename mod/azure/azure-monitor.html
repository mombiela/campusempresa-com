Introducción a Azure Monitor

Azure Monitor es una solución integral para recopilar, analizar y actuar sobre la telemetría de tus entornos en la nube y locales. Te ayuda a maximizar la disponibilidad y el rendimiento de tus aplicaciones y servicios.

Conceptos Clave

  • Datos de Telemetría: Datos recopilados de diversas fuentes, incluidas aplicaciones, sistemas operativos y recursos.
  • Métricas: Datos numéricos que describen el rendimiento de un sistema en un momento dado.
  • Registros: Registros de eventos que ocurren dentro de tu sistema.
  • Alertas: Notificaciones desencadenadas por condiciones específicas en tus métricas o registros.
  • Tableros: Vistas personalizables que muestran métricas y registros.

Configuración de Azure Monitor

Para comenzar a usar Azure Monitor, necesitas configurar tu entorno y configurar los componentes necesarios.

Pasos para Configurar Azure Monitor

  1. Crear una Cuenta de Azure: Si no tienes una cuenta de Azure, crea una en Azure Portal.
  2. Navegar a Azure Monitor: En el Portal de Azure, busca "Azure Monitor" en la barra de búsqueda.
  3. Habilitar la Monitorización de Recursos: Selecciona los recursos que deseas monitorear y habilita la monitorización.

Ejemplo: Habilitar la Monitorización para una Máquina Virtual

# Script de PowerShell para habilitar la monitorización de una VM
$vmName = "myVM"
$resourceGroup = "myResourceGroup"
Set-AzVMMonitoringExtension -ResourceGroupName $resourceGroup -VMName $vmName

Recopilación y Análisis de Datos

Azure Monitor recopila datos de varias fuentes y proporciona herramientas para analizarlos.

Fuentes de Datos

  • Application Insights: Monitorea aplicaciones en vivo y proporciona información sobre su rendimiento y uso.
  • Azure Log Analytics: Recopila y analiza datos de registro de varias fuentes.
  • Azure Metrics: Recopila y analiza datos métricos de recursos de Azure.

Análisis de Datos con Log Analytics

Log Analytics utiliza un lenguaje de consulta llamado Kusto Query Language (KQL) para analizar datos de registro.

Ejemplo: Consulta Básica en KQL

// Obtener los últimos 10 registros de la tabla 'Event'
Event
| take 10

Creación de Alertas y Notificaciones

Las alertas te notifican cuando se cumplen ciertas condiciones en tus métricas o registros.

Tipos de Alertas

  • Alertas de Métricas: Desencadenadas por valores de métricas.
  • Alertas de Registros: Desencadenadas por datos de registro.
  • Alertas de Registro de Actividades: Desencadenadas por registros de actividades de Azure.

Ejemplo: Crear una Alerta de Métrica

# Script de PowerShell para crear una alerta de métrica
$actionGroup = New-AzActionGroup -Name "myActionGroup" -ResourceGroupName "myResourceGroup" -ShortName "AG"
Add-AzMetricAlertRuleV2 -Name "myMetricAlert" -ResourceGroupName "myResourceGroup" -WindowSize (New-TimeSpan -Minutes 5) -Frequency (New-TimeSpan -Minutes 1) -TargetResourceId "/subscriptions/{subscription-id}/resourceGroups/myResourceGroup/providers/Microsoft.Compute/virtualMachines/myVM" -Condition "avg Percentage CPU > 80" -ActionGroupId $actionGroup.Id

Visualización de Datos con Tableros

Los tableros proporcionan una vista personalizable de tus métricas y registros.

Creación de un Tablero

  1. Navegar a Tableros: En el Portal de Azure, ve a la sección "Tableros".
  2. Crear un Nuevo Tablero: Haz clic en "Nuevo Tablero" y personalízalo agregando mosaicos.
  3. Agregar Métricas y Registros: Agrega mosaicos para métricas y registros para visualizar tus datos.

Ejemplo: Agregar un Mosaico de Métrica

# Script de PowerShell para agregar un mosaico de métrica a un tablero
$dashboard = New-AzPortalDashboard -Name "myDashboard" -ResourceGroupName "myResourceGroup"
Add-AzPortalDashboardTile -Dashboard $dashboard -TileType "metric" -MetricName "Percentage CPU" -ResourceId "/subscriptions/{subscription-id}/resourceGroups/myResourceGroup/providers/Microsoft.Compute/virtualMachines/myVM"

Conclusión

Azure Monitor es una herramienta poderosa para monitorear y gestionar el rendimiento y la disponibilidad de tus aplicaciones y servicios. Al comprender sus conceptos clave, configurarlo correctamente, recopilar y analizar datos, crear alertas y visualizar datos con tableros, puedes asegurarte de que tus sistemas funcionen de manera fluida y eficiente.
