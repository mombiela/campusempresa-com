Servicios de Cómputo de Azure
Servicios de Almacenamiento de Azure ►

Los Servicios de Cómputo de Azure proporcionan la infraestructura y las herramientas necesarias para ejecutar aplicaciones, gestionar cargas de trabajo y escalar recursos en la nube. Este tema cubrirá los diversos servicios de cómputo ofrecidos por Azure, desde máquinas virtuales básicas hasta orquestación avanzada de contenedores y computación sin servidor.

Introducción a los Servicios de Cómputo de Azure

Los Servicios de Cómputo de Azure abarcan una gama de ofertas que te permiten ejecutar aplicaciones y gestionar cargas de trabajo en la nube. Estos servicios incluyen:

  • Máquinas Virtuales (VMs)
  • Servicios de Aplicaciones
  • Azure Kubernetes Service (AKS)
  • Azure Functions
  • Azure Batch

Máquinas Virtuales (VMs)

Las Máquinas Virtuales de Azure proporcionan recursos de cómputo escalables bajo demanda. Son ideales para escenarios donde necesitas control total sobre el sistema operativo y el entorno.

Conceptos Clave

  • Tamaños de VM: Diferentes tamaños ofrecen diversas cantidades de CPU, memoria y almacenamiento.
  • Imágenes: Plantillas preconfiguradas para VMs, incluyendo distribuciones de Windows y Linux.
  • Discos: Opciones de almacenamiento persistente para VMs, incluyendo HDD Estándar, SSD Estándar y SSD Premium.
  • Redes: Redes virtuales, direcciones IP públicas y grupos de seguridad de red.

Ejemplo: Creación de una Máquina Virtual

# Crear un grupo de recursos
az group create --name myResourceGroup --location eastus

# Crear una máquina virtual
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Gestión de Máquinas Virtuales

  • Iniciar/Detener: Controlar el estado de tu VM.
  • Redimensionar: Cambiar el tamaño de tu VM para escalar hacia arriba o hacia abajo.
  • Instantáneas: Crear copias de seguridad en un punto en el tiempo de los discos de tu VM.

Servicios de Aplicaciones

Azure App Services es una plataforma completamente gestionada para construir, desplegar y escalar aplicaciones web y APIs.

Conceptos Clave

  • Aplicaciones Web: Alojar aplicaciones web utilizando varios frameworks como .NET, Java, PHP, Node.js y Python.
  • Aplicaciones API: Alojar APIs RESTful.
  • Aplicaciones Móviles: Servicios de backend para aplicaciones móviles.
  • Ranuras de Despliegue: Entornos de prueba antes del despliegue en producción.

Ejemplo: Desplegar una Aplicación Web

# Crear un plan de App Service
az appservice plan create --name myAppServicePlan --resource-group myResourceGroup --sku B1 --is-linux

# Crear una aplicación web
az webapp create --resource-group myResourceGroup --plan myAppServicePlan --name myUniqueAppName --runtime "NODE|14-lts"

Escalado y Monitoreo

  • Escalado: Escalar hacia arriba (aumentar recursos) o escalar hacia afuera (aumentar instancias).
  • Monitoreo: Usar Azure Monitor y Application Insights para métricas de rendimiento y uso.

Azure Kubernetes Service (AKS)

Azure Kubernetes Service simplifica el despliegue, gestión y escalado de aplicaciones en contenedores utilizando Kubernetes.

Conceptos Clave

  • Clusters: Un conjunto de VMs que ejecutan aplicaciones en contenedores.
  • Nodos: VMs individuales dentro de un cluster.
  • Pods: Las unidades desplegables más pequeñas en Kubernetes, que pueden contener uno o más contenedores.
  • Servicios: Definir cómo acceder a los pods.

Ejemplo: Crear un Cluster AKS

# Crear un grupo de recursos
az group create --name myResourceGroup --location eastus

# Crear un cluster AKS
az aks create --resource-group myResourceGroup --name myAKSCluster --node-count 1 --enable-addons monitoring --generate-ssh-keys

Gestión de AKS

  • Escalado: Ajustar el número de nodos en tu cluster.
  • Actualizaciones: Actualizar la versión de Kubernetes.
  • Monitoreo: Usar Azure Monitor para contenedores para rastrear el rendimiento y la salud.

Azure Functions

Azure Functions es un servicio de computación sin servidor que te permite ejecutar código bajo demanda sin gestionar infraestructura.

Conceptos Clave

  • Disparadores: Eventos que causan la ejecución de la función (por ejemplo, solicitudes HTTP, temporizadores, colas).
  • Enlaces: Conectar funciones a otros servicios de Azure (por ejemplo, almacenamiento, bases de datos).
  • Plan de Consumo: Escala automáticamente y cobra según el número de ejecuciones.

Ejemplo: Crear una Función Activada por HTTP

using System.IO;
using Microsoft.AspNetCore.Mvc;
using Microsoft.Azure.WebJobs;
using Microsoft.Azure.WebJobs.Extensions.Http;
using Microsoft.AspNetCore.Http;
using Microsoft.Extensions.Logging;
using Newtonsoft.Json;

public static class HttpTriggerFunction
{
    [FunctionName("HttpTriggerFunction")]
    public static async Task<IActionResult> Run(
        [HttpTrigger(AuthorizationLevel.Function, "get", "post", Route = null)] HttpRequest req,
        ILogger log)
    {
        log.LogInformation("C# HTTP trigger function processed a request.");

        string name = req.Query["name"];

        string requestBody = await new StreamReader(req.Body).ReadToEndAsync();
        dynamic data = JsonConvert.DeserializeObject(requestBody);
        name = name ?? data?.name;

        return name != null
            ? (ActionResult)new OkObjectResult($"Hello, {name}")
            : new BadRequestObjectResult("Please pass a name on the query string or in the request body");
    }
}

Escalado y Monitoreo

  • Escalado: Escala automáticamente según la demanda.
  • Monitoreo: Usar Application Insights para registros detallados y métricas de rendimiento.

Azure Batch

Azure Batch es un servicio para ejecutar aplicaciones de computación paralela y de alto rendimiento (HPC) a gran escala de manera eficiente en la nube.

Conceptos Clave

  • Trabajos: Una colección de tareas que se ejecutan en los nodos de cómputo.
  • Tareas: Unidades individuales de trabajo dentro de un trabajo.
  • Pools: Colecciones de nodos de cómputo que ejecutan las tareas.

Ejemplo: Crear un Trabajo Batch

# Crear una cuenta Batch
az batch account create --name mybatchaccount --resource-group myResourceGroup --location eastus

# Crear un pool Batch
az batch pool create --id mypool --vm-size Standard_A1_v2 --target-dedicated-nodes 2 --image canonical:ubuntuserver:18.04-LTS

# Crear un trabajo Batch
az batch job create --id myjob --pool-id mypool

# Añadir una tarea al trabajo
az batch task create --job-id myjob --task-id mytask --command-line "echo Hello, Batch!"

Gestión de Trabajos Batch

  • Escalado: Ajustar el número de nodos en el pool.
  • Monitoreo: Usar Azure Monitor para rastrear el rendimiento de trabajos y tareas.

Conclusión

Los Servicios de Cómputo de Azure proporcionan una suite completa de herramientas para ejecutar y gestionar aplicaciones en la nube. Desde máquinas virtuales básicas hasta funciones sin servidor avanzadas y orquestación de contenedores, Azure ofrece soluciones para una amplia gama de escenarios. Entender estos servicios y cómo usarlos efectivamente puede ayudarte a construir aplicaciones escalables, resilientes y eficientes.
