Introducción

En este módulo, exploraremos cómo gestionar y asegurar el almacenamiento en Microsoft Azure. Esto incluye comprender los diferentes tipos de almacenamiento disponibles, cómo gestionar cuentas de almacenamiento y las mejores prácticas para asegurar tus datos.

Tipos de Almacenamiento en Azure

Azure proporciona varios tipos de soluciones de almacenamiento para satisfacer diferentes necesidades:

  • Blob Storage: Utilizado para almacenar grandes cantidades de datos no estructurados como texto o datos binarios.
  • File Storage: Proporciona comparticiones de archivos totalmente gestionadas en la nube que son accesibles a través del protocolo SMB.
  • Queue Storage: Utilizado para almacenar grandes cantidades de mensajes que pueden ser accedidos desde cualquier lugar a través de llamadas autenticadas.
  • Table Storage: Un almacén NoSQL para almacenamiento sin esquema de datos estructurados.

Blob Storage

Blob Storage es ideal para:

  • Servir imágenes o documentos directamente a un navegador.
  • Almacenar archivos para acceso distribuido.
  • Transmitir video y audio.
  • Escribir en archivos de registro.

Ejemplo: Subiendo un Blob

from azure.storage.blob import BlobServiceClient, BlobClient, ContainerClient

# Inicializar una conexión al servicio Blob
blob_service_client = BlobServiceClient.from_connection_string("your_connection_string")

# Crear un contenedor
container_name = "mycontainer"
container_client = blob_service_client.create_container(container_name)

# Subir un archivo al contenedor
blob_client = blob_service_client.get_blob_client(container=container_name, blob="myfile.txt")
with open("localfile.txt", "rb") as data:
    blob_client.upload_blob(data)

File Storage

File Storage es ideal para:

  • Migrar comparticiones de archivos locales a la nube.
  • Almacenar configuraciones compartidas de aplicaciones.
  • Almacenar perfiles de usuario.

Ejemplo: Montando una Compartición de Archivos de Azure en Windows

# Montar la compartición de archivos
net use Z: \\<storage_account_name>.file.core.windows.net\<file_share_name> /u:<storage_account_name> <storage_account_key>

Queue Storage

Queue Storage es ideal para:

  • Desacoplar componentes de aplicaciones.
  • Construir aplicaciones flexibles y escalables.

Ejemplo: Añadiendo un Mensaje a una Cola

from azure.storage.queue import QueueServiceClient

# Inicializar una conexión al servicio de Colas
queue_service_client = QueueServiceClient.from_connection_string("your_connection_string")

# Crear una cola
queue_name = "myqueue"
queue_client = queue_service_client.create_queue(queue_name)

# Añadir un mensaje a la cola
queue_client.send_message("Hello, World!")

Table Storage

Table Storage es ideal para:

  • Almacenar datos estructurados que no requieren uniones complejas, claves foráneas o procedimientos almacenados.
  • Almacenar conjuntos de datos que requieren alta disponibilidad y escalabilidad.

Ejemplo: Insertando una Entidad en una Tabla

from azure.data.tables import TableServiceClient, TableEntity

# Inicializar una conexión al servicio de Tablas
table_service_client = TableServiceClient.from_connection_string("your_connection_string")

# Crear una tabla
table_name = "mytable"
table_client = table_service_client.create_table(table_name)

# Insertar una entidad
entity = TableEntity()
entity.PartitionKey = "partition1"
entity.RowKey = "row1"
entity["Name"] = "John Doe"
entity["Age"] = 30
table_client.create_entity(entity=entity)

Gestión de Cuentas de Almacenamiento

Las cuentas de almacenamiento proporcionan un espacio de nombres único para tus datos de Azure Storage. Puedes gestionar cuentas de almacenamiento usando el portal de Azure, Azure CLI o Azure PowerShell.

Creando una Cuenta de Almacenamiento usando Azure CLI

az storage account create \
  --name mystorageaccount \
  --resource-group myResourceGroup \
  --location eastus \
  --sku Standard_LRS

Gestión de Claves de Acceso

Las claves de acceso se utilizan para autenticar el acceso a tu cuenta de almacenamiento. Es importante gestionar estas claves de manera segura.

Rotación de Claves de Acceso

# Listar las claves de acceso
az storage account keys list --resource-group myResourceGroup --account-name mystorageaccount

# Regenerar una clave de acceso
az storage account keys renew --resource-group myResourceGroup --account-name mystorageaccount --key primary

Seguridad del Almacenamiento

Asegurar tu almacenamiento es crucial para proteger tus datos de accesos no autorizados y brechas de seguridad.

Mejores Prácticas para Asegurar el Almacenamiento

  • Usar Firmas de Acceso Compartido (SAS): SAS proporciona acceso limitado a tus recursos de almacenamiento sin exponer las claves de tu cuenta.
  • Habilitar Cifrado de Almacenamiento de Azure: Los datos se cifran y descifran de manera transparente usando cifrado AES de 256 bits.
  • Usar Puntos de Servicio de Red Virtual: Restringir el acceso a tu cuenta de almacenamiento a redes virtuales específicas.
  • Habilitar Protección Avanzada contra Amenazas: Detecta intentos inusuales y potencialmente dañinos de acceder o explotar tu cuenta de almacenamiento.

Ejemplo: Generando un Token SAS

from azure.storage.blob import generate_blob_sas, BlobSasPermissions
from datetime import datetime, timedelta

sas_token = generate_blob_sas(
    account_name="your_account_name",
    container_name="your_container_name",
    blob_name="your_blob_name",
    account_key="your_account_key",
    permission=BlobSasPermissions(read=True),
    expiry=datetime.utcnow() + timedelta(hours=1)
)
print(sas_token)

Al comprender estos conceptos, puedes gestionar y asegurar efectivamente tus datos en Azure, asegurando que sean accesibles y estén protegidos.
