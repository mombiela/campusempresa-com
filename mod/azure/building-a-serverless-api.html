Introducción

En este módulo, exploraremos cómo construir una API serverless usando Azure Functions. Comenzaremos con los conceptos básicos y gradualmente avanzaremos a temas más avanzados. Al final de este módulo, tendrás una comprensión sólida de cómo crear, desplegar y gestionar APIs serverless en Azure.

¿Qué es la Computación Serverless?

  • Definición: La computación serverless te permite construir y ejecutar aplicaciones y servicios sin pensar en servidores. El proveedor de la nube provisiona, escala y gestiona automáticamente la infraestructura necesaria para ejecutar el código.
  • Beneficios:
    • Sin gestión de servidores
    • Escalado automático
    • Precios por uso
    • Mayor productividad del desarrollador

Comenzando con Azure Functions

Prerrequisitos

  • Una cuenta activa de Azure
  • Comprensión básica de programación (preferiblemente en C#, JavaScript o Python)
  • Azure CLI instalado

Configurando Tu Entorno

  1. Instalar Azure Functions Core Tools: 
    npm install -g azure-functions-core-tools@3
  2. Crear una nueva Azure Function App: 
    func init MyFunctionApp --worker-runtime node
  3. Crear una nueva Función: 
    cd MyFunctionApp
func new
    Sigue las indicaciones para crear una función de disparador HTTP.

Ejemplo: Creando una Simple Función de Disparador HTTP

module.exports = async function (context, req) {
    context.log('HTTP trigger function processed a request.');

    const name = (req.query.name || (req.body && req.body.name));
    const responseMessage = name
        ? "Hello, " + name + ". This HTTP triggered function executed successfully."
        : "This HTTP triggered function executed successfully. Pass a name in the query string or in the request body for a personalized response.";

    context.res = {
        // status: 200, /* Defaults to 200 */
        body: responseMessage
    };
}
  • Explicación:
    • context: Proporciona métodos y propiedades para interactuar con el runtime.
    • req: Representa la solicitud HTTP.
    • context.res: Representa la respuesta HTTP.

Desplegando Tu Función en Azure

Pasos para Desplegar

  1. Iniciar sesión en Azure: 
    az login
  2. Crear un Grupo de Recursos: 
    az group create --name myResourceGroup --location eastus
  3. Crear una Function App en Azure: 
    az functionapp create --resource-group myResourceGroup --consumption-plan-location eastus --runtime node --functions-version 3 --name myFunctionApp --storage-account <NOMBRE_CUENTA_ALMACENAMIENTO>
  4. Desplegar Tu Función: 
    func azure functionapp publish myFunctionApp

Temas Avanzados

Autenticación y Autorización

  • Azure Active Directory (AAD): Usa AAD para asegurar tu API.
  • Claves API: Método simple para asegurar tu API.
  • Identidades Administradas: Accede de manera segura a otros recursos de Azure.

Integración con Otros Servicios de Azure

  • Azure Cosmos DB: Almacena y recupera datos.
  • Azure Storage: Almacena archivos y blobs.
  • Azure Event Grid: Arquitectura impulsada por eventos.

Monitoreo y Registro

  • Application Insights: Monitorea el rendimiento y uso.
  • Azure Monitor: Solución de monitoreo integral.
  • Registro: Usa context.log para registrar información.

Escalado y Optimización del Rendimiento

  • Escalado: Azure Functions escala automáticamente según la demanda.
  • Inicios en Frío: Mitiga los problemas de inicio en frío usando planes premium o instancias pre-calentadas.
  • Optimización del Rendimiento: Optimiza el código y la configuración de la función para un mejor rendimiento.

Conclusión

Construir una API serverless en Azure es una forma poderosa de crear aplicaciones escalables y rentables. Desde configurar tu entorno hasta desplegar y gestionar tus funciones, Azure proporciona una suite completa de herramientas y servicios para apoyar tu proceso de desarrollo. A medida que avances, integrar con otros servicios de Azure y optimizar el rendimiento te ayudará a construir aplicaciones serverless robustas y eficientes.
