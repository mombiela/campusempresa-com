Introducción a las Redes Virtuales (VNet)

Las Redes Virtuales (VNets) son componentes fundamentales para tu red privada en Azure. Permiten que muchos tipos de recursos de Azure, como las Máquinas Virtuales (VMs) de Azure, se comuniquen de manera segura entre sí, con internet y con redes locales.

Conceptos Clave

  • Aislamiento y Segmentación: Las VNets proporcionan aislamiento y segmentación dentro del entorno de Azure.
  • Subredes: Las VNets pueden dividirse en subredes para segmentar aún más la red.
  • Dirección IP: Las VNets utilizan direcciones IP privadas.
  • Grupos de Seguridad de Red (NSGs): Controlan el tráfico entrante y saliente hacia las interfaces de red (NIC), VMs y subredes.
  • Peering: Conecta VNets entre sí.
  • Puerta de Enlace VPN: Conecta VNets con redes locales.

Creación de una Red Virtual

Crear una VNet es el primer paso para configurar tu infraestructura de red en Azure.

Pasos para Crear una VNet

  1. Navegar al Portal de Azure: Ve al portal de Azure en https://portal.azure.com.

  2. Crear un Recurso: Haz clic en "Crear un recurso" y busca "Red Virtual".

  3. Configurar Configuraciones Básicas:

    • Nombre: Proporciona un nombre para tu VNet.
    • Espacio de Direcciones: Define el rango de direcciones IP (por ejemplo, 10.0.0.0/16).
    • Suscripción: Selecciona tu suscripción de Azure.
    • Grupo de Recursos: Elige un grupo de recursos existente o crea uno nuevo.
    • Ubicación: Selecciona la región de Azure.

  4. Configurar Subredes:

    • Nombre de la Subred: Proporciona un nombre para tu subred.
    • Rango de Direcciones de la Subred: Define el rango de direcciones IP para la subred (por ejemplo, 10.0.1.0/24).

  5. Revisar y Crear: Revisa tus configuraciones y haz clic en "Crear".

Código de Ejemplo: Creación de una VNet usando Azure CLI

# Crear un grupo de recursos
az group create --name MyResourceGroup --location eastus

# Crear una red virtual
az network vnet create \
  --name MyVNet \
  --resource-group MyResourceGroup \
  --address-prefix 10.0.0.0/16 \
  --subnet-name MySubnet \
  --subnet-prefix 10.0.1.0/24

Subredes

Las subredes te permiten segmentar la VNet en secciones más pequeñas y manejables.

Conceptos Clave

  • Rango de Direcciones de la Subred: Cada subred debe tener un rango de direcciones único dentro de la VNet.
  • Grupos de Seguridad de Red (NSGs): Aplica NSGs a subredes para controlar el tráfico.

Código de Ejemplo: Añadir una Subred usando Azure CLI

# Añadir una subred a una VNet existente
az network vnet subnet create \
  --address-prefix 10.0.2.0/24 \
  --name MySecondSubnet \
  --resource-group MyResourceGroup \
  --vnet-name MyVNet

Grupos de Seguridad de Red (NSGs)

Los NSGs se utilizan para filtrar el tráfico de red hacia y desde los recursos de Azure en una red virtual de Azure.

Conceptos Clave

  • Reglas Entrantes y Salientes: Define reglas para permitir o denegar tráfico.
  • Asociaciones: Los NSGs pueden asociarse con subredes o interfaces de red individuales.

Código de Ejemplo: Creación de un NSG y Adición de Reglas

# Crear un NSG
az network nsg create \
  --resource-group MyResourceGroup \
  --name MyNSG

# Crear una regla de seguridad entrante
az network nsg rule create \
  --resource-group MyResourceGroup \
  --nsg-name MyNSG \
  --name AllowHTTP \
  --protocol tcp \
  --priority 1000 \
  --destination-port-range 80 \
  --access allow

Peering de VNet

El peering de VNet conecta dos VNets, permitiendo que los recursos en cada VNet se comuniquen entre sí.

Conceptos Clave

  • Baja Latencia: Proporciona conectividad de baja latencia y alto ancho de banda.
  • Peering Transitivo: No es compatible; cada VNet debe estar emparejada directamente.

Código de Ejemplo: Creación de un Peering de VNet

# Emparejar VNet1 con VNet2
az network vnet peering create \
  --name VNet1ToVNet2 \
  --resource-group MyResourceGroup \
  --vnet-name VNet1 \
  --remote-vnet /subscriptions/{subscription-id}/resourceGroups/MyResourceGroup/providers/Microsoft.Network/virtualNetworks/VNet2 \
  --allow-vnet-access

Puerta de Enlace VPN

La Puerta de Enlace VPN se utiliza para enviar tráfico cifrado entre una VNet de Azure y una ubicación local a través de Internet pública.

Conceptos Clave

  • Tipos de Puerta de Enlace: Basada en políticas y basada en rutas.
  • Tipos de Conexión: Sitio a Sitio, Punto a Sitio y VNet a VNet.

Código de Ejemplo: Creación de una Puerta de Enlace VPN

# Crear una puerta de enlace VPN
az network vnet-gateway create \
  --resource-group MyResourceGroup \
  --name MyVPNGateway \
  --public-ip-address MyGatewayPublicIP \
  --vnet MyVNet \
  --gateway-type Vpn \
  --vpn-type RouteBased \
  --sku VpnGw1 \
  --no-wait

Conclusión

Las Redes Virtuales (VNets) son esenciales para crear una infraestructura de red segura y escalable en Azure. Al comprender y utilizar VNets, subredes, NSGs, peering de VNet y puertas de enlace VPN, puedes gestionar y asegurar eficazmente tus recursos de Azure. Este conocimiento fundamental es crucial tanto para principiantes como para usuarios avanzados para construir soluciones en la nube robustas.
