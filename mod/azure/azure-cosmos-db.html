Introducción a Azure Cosmos DB

Azure Cosmos DB es un servicio de base de datos multimodelo distribuido globalmente, diseñado para proporcionar alta disponibilidad, baja latencia y escalabilidad. Soporta múltiples modelos de datos como documento, clave-valor, gráfico y familia de columnas.

Características Clave

  • Distribución Global: Distribuye datos a través de múltiples regiones de Azure con un solo clic.
  • Soporte Multimodelo: Trabaja con modelos de datos de documento, clave-valor, gráfico y familia de columnas.
  • Escalabilidad Elástica: Escala el rendimiento y el almacenamiento de manera independiente y bajo demanda.
  • Baja Latencia: Garantiza tiempos de respuesta de un solo dígito en milisegundos.
  • SLAs Comprensivos: Ofrece SLAs para rendimiento, latencia, disponibilidad y consistencia.

Configuración de Azure Cosmos DB

Creación de una Cuenta de Cosmos DB

  1. Navegar al Portal de Azure: Ve al portal de Azure en portal.azure.com.
  2. Crear un Recurso: Haz clic en "Crear un recurso" y busca "Azure Cosmos DB".
  3. Configurar la Cuenta: Elige la API (Core (SQL), MongoDB, Cassandra, Gremlin, Table), establece el nombre de la cuenta, el grupo de recursos y la ubicación.
  4. Revisar y Crear: Revisa la configuración y haz clic en "Crear".

Ejemplo: Creación de una Cuenta de Cosmos DB usando Azure CLI

# Crear un grupo de recursos
az group create --name myResourceGroup --location eastus

# Crear una cuenta de Cosmos DB con la API SQL
az cosmosdb create --name myCosmosDBAccount --resource-group myResourceGroup --kind GlobalDocumentDB

Trabajando con Modelos de Datos

Modelo de Documento (API SQL)

El modelo de documento almacena datos en formato JSON y se accede a ellos mediante consultas similares a SQL.

Ejemplo: Insertar un Documento

{
  "id": "1",
  "name": "John Doe",
  "email": "john.doe@example.com",
  "age": 30
}

Ejemplo: Consultar Documentos

SELECT * FROM c WHERE c.age > 25

Modelo Clave-Valor (API Table)

El modelo clave-valor almacena datos como pares clave-valor, similar a Azure Table Storage.

Ejemplo: Insertar un Par Clave-Valor

{
  "PartitionKey": "Users",
  "RowKey": "1",
  "Name": "John Doe",
  "Email": "john.doe@example.com"
}

Modelo de Gráfico (API Gremlin)

El modelo de gráfico almacena datos como vértices y aristas, adecuado para consultas basadas en gráficos.

Ejemplo: Añadir un Vértice

g.addV('person').property('id', '1').property('name', 'John Doe')

Ejemplo: Añadir una Arista

g.V('1').addE('knows').to(g.V('2'))

Niveles de Consistencia

Azure Cosmos DB ofrece cinco niveles de consistencia para equilibrar entre consistencia y rendimiento:

| Nivel de Consistencia | Descripción | |-----------------------|-------------| | Fuerte | Garantiza la mayor consistencia asegurando que las lecturas siempre devuelvan la escritura más reciente. | | Obsolescencia Limitada| Garantiza que las lecturas se retrasen respecto a las escrituras por un intervalo de tiempo o número de versiones especificado. | | Sesión | Garantiza consistencia dentro de una sola sesión. | | Prefijo Consistente | Garantiza que las lecturas nunca vean escrituras fuera de orden. | | Eventual | Garantiza consistencia eventual con la menor latencia. |

Características Avanzadas

Particionamiento

El particionamiento ayuda a distribuir datos a través de múltiples servidores para asegurar escalabilidad y rendimiento.

Ejemplo: Configuración de una Clave de Partición

{
  "id": "1",
  "name": "John Doe",
  "email": "john.doe@example.com",
  "age": 30,
  "partitionKey": "email"
}

Change Feed

Change Feed te permite escuchar cambios (inserciones y actualizaciones) en tu contenedor de Cosmos DB.

Ejemplo: Usando Change Feed con Azure Functions

public static class CosmosDBTrigger
{
    [FunctionName("CosmosDBTrigger")]
    public static void Run([CosmosDBTrigger(
        databaseName: "myDatabase",
        collectionName: "myCollection",
        ConnectionStringSetting = "AzureWebJobsCosmosDBConnectionString",
        LeaseCollectionName = "leases")] IReadOnlyList<Document> input, ILogger log)
    {
        if (input != null && input.Count > 0)
        {
            log.LogInformation("Documentos modificados " + input.Count);
            log.LogInformation("Id del primer documento " + input[0].Id);
        }
    }
}

Conclusión

Azure Cosmos DB es un potente servicio de base de datos distribuido globalmente que soporta múltiples modelos de datos y ofrece alta disponibilidad, baja latencia y escalabilidad elástica. Al comprender sus características clave, configurar una cuenta, trabajar con diferentes modelos de datos y aprovechar características avanzadas como el particionamiento y el change feed, puedes construir aplicaciones robustas y escalables en Azure Cosmos DB.
