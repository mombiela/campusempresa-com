Restricciones Genéricas
Usando Genéricos con Interfaces ►

En TypeScript, los genéricos proporcionan una forma de crear componentes reutilizables que funcionan con una variedad de tipos de datos. Sin embargo, hay momentos en los que necesitas restringir los tipos que se pueden usar con un genérico. Aquí es donde entran en juego las restricciones genéricas.

¿Qué son las Restricciones Genéricas?

Las restricciones genéricas te permiten especificar que un tipo genérico debe conformarse a una cierta estructura o heredar de un tipo específico. Esto asegura que el tipo genérico tenga las propiedades o métodos que necesitas usar dentro de tu función o clase.

Conceptos Clave

  • Genéricos: Una forma de crear componentes que pueden trabajar con cualquier tipo de datos.
  • Restricciones: Limitaciones que restringen los tipos que se pueden usar con un genérico.

¿Por Qué Usar Restricciones Genéricas?

Usar restricciones genéricas puede ayudarte a:

  • Asegurar la seguridad de tipos restringiendo los tipos que se pueden usar.
  • Acceder a propiedades o métodos específicos del tipo genérico.
  • Mejorar la legibilidad y mantenibilidad del código definiendo claramente la estructura esperada de los tipos.

Cómo Aplicar Restricciones Genéricas

Puedes aplicar restricciones genéricas usando la palabra clave extends. Esta palabra clave te permite especificar que un tipo genérico debe extender un tipo o interfaz particular.

Ejemplo: Restricción Genérica Básica

Comencemos con un ejemplo simple donde queremos asegurarnos de que un tipo genérico tenga una propiedad length.

interface HasLength {
  length: number;
}

function logLength<T extends HasLength>(item: T): void {
  console.log(item.length);
}

logLength("¡Hola, Mundo!"); // Funciona porque el string tiene una propiedad length
logLength([1, 2, 3, 4, 5]); // Funciona porque el array tiene una propiedad length
// logLength(123); // Error: Argumento de tipo 'number' no es asignable al parámetro de tipo 'HasLength'

En este ejemplo:

  • Definimos una interfaz HasLength con una propiedad length.
  • La función genérica logLength usa T extends HasLength para asegurarse de que el tipo T tenga una propiedad length.
  • La función funciona con strings y arrays porque ambos tienen una propiedad length, pero lanzará un error si intentas pasar un número.

Usando Múltiples Restricciones

También puedes aplicar múltiples restricciones a un tipo genérico usando tipos de intersección.

Ejemplo: Múltiples Restricciones

interface HasName {
  name: string;
}

interface HasAge {
  age: number;
}

function logPerson<T extends HasName & HasAge>(person: T): void {
  console.log(`Nombre: ${person.name}, Edad: ${person.age}`);
}

const person = { name: "Alicia", age: 30, occupation: "Ingeniera" };
logPerson(person); // Funciona porque la persona tiene tanto propiedades name como age
// logPerson({ name: "Bob" }); // Error: Falta la propiedad 'age'

En este ejemplo:

  • Definimos dos interfaces, HasName y HasAge.
  • La función genérica logPerson usa T extends HasName & HasAge para asegurarse de que el tipo T tenga tanto propiedades name como age.
  • La función funciona con un objeto que tiene ambas propiedades pero lanzará un error si falta alguna propiedad.

Casos de Uso Prácticos

Ejemplo: Clases Genéricas con Restricciones

Las restricciones genéricas también son útiles en clases. Consideremos una clase que gestiona una colección de elementos, donde cada elemento debe tener una propiedad id.

interface Identifiable {
  id: number;
}

class ItemCollection<T extends Identifiable> {
  private items: T[] = [];

  addItem(item: T): void {
    this.items.push(item);
  }

  getItemById(id: number): T | undefined {
    return this.items.find(item => item.id === id);
  }
}

const collection = new ItemCollection<{ id: number; name: string }>();
collection.addItem({ id: 1, name: "Item 1" });
collection.addItem({ id: 2, name: "Item 2" });

console.log(collection.getItemById(1)); // { id: 1, name: "Item 1" }
console.log(collection.getItemById(3)); // undefined

En este ejemplo:

  • Definimos una interfaz Identifiable con una propiedad id.
  • La clase ItemCollection usa T extends Identifiable para asegurarse de que los elementos tengan una propiedad id.
  • La clase proporciona métodos para agregar elementos y recuperar elementos por su id.

Conclusión

Las restricciones genéricas en TypeScript son una característica poderosa que te permite crear código más robusto y seguro en términos de tipos. Al usar restricciones, puedes asegurarte de que tus genéricos tengan las propiedades o métodos necesarios, haciendo tu código más predecible y fácil de mantener.

  • Genéricos: Permiten componentes reutilizables para varios tipos de datos.
  • Restricciones: Restringen los tipos que se pueden usar con genéricos.
  • Uso Práctico: Aseguran la seguridad de tipos y acceso a propiedades o métodos necesarios.

Entender y usar restricciones genéricas de manera efectiva te ayudará a escribir código TypeScript más flexible y confiable.
