Tipos Any, Unknown y Never

Introducción

En TypeScript, entender los tipos any, unknown y never es crucial para escribir código robusto y seguro en términos de tipos. Estos tipos ayudan a gestionar diferentes escenarios donde la información de tipo es flexible, desconocida o imposible.

El Tipo `any`

El tipo any es un tipo poderoso pero potencialmente peligroso en TypeScript. Te permite optar por no realizar comprobaciones de tipo, lo que lo hace útil para escenarios donde no conoces el tipo de antemano o cuando migras código JavaScript a TypeScript.

Conceptos Clave

  • Flexibilidad: any puede contener cualquier valor, y puedes realizar cualquier operación sobre él sin que TypeScript se queje.
  • Seguridad de Tipo: Usar any elimina los beneficios de la comprobación de tipos de TypeScript, lo que puede llevar a errores en tiempo de ejecución.

Ejemplo

let value: any;

value = 5; // OK
value = "Hola"; // OK
value = true; // OK

// Sin comprobación de tipos
value.toUpperCase(); // No hay error, pero podría causar un error en tiempo de ejecución si value no es una cadena

El Tipo `unknown`

El tipo unknown es más seguro que any porque te obliga a realizar comprobaciones de tipo antes de realizar operaciones sobre el valor. Es útil cuando quieres aceptar cualquier valor pero aún así asegurar la seguridad de tipo.

Conceptos Clave

  • Seguridad de Tipo: A diferencia de any, no puedes realizar operaciones sobre un tipo unknown sin antes reducirlo a un tipo específico.
  • Guardias de Tipo: Debes usar guardias de tipo o aserciones de tipo para trabajar con valores unknown.

Ejemplo

let value: unknown;

value = 5; // OK
value = "Hola"; // OK
value = true; // OK

// Se requiere comprobación de tipos
if (typeof value === "string") {
    value.toUpperCase(); // OK
} else {
    // value.toUpperCase(); // Error: El objeto es de tipo 'unknown'
}

El Tipo `never`

El tipo never representa valores que nunca ocurren. A menudo se usa para funciones que nunca retornan o para comprobaciones exhaustivas de tipos.

Conceptos Clave

  • Código Inalcanzable: Las funciones que lanzan excepciones o tienen bucles infinitos pueden retornar never.
  • Comprobación de Exhaustividad: Útil en declaraciones switch para asegurar que se manejan todos los casos posibles.

Ejemplo

function error(message: string): never {
    throw new Error(message);
}

function infiniteLoop(): never {
    while (true) {}
}

// Comprobación de exhaustividad
type Shape = "circle" | "square";

function getArea(shape: Shape): number {
    switch (shape) {
        case "circle":
            return Math.PI * 1 * 1; // Ejemplo de cálculo de área
        case "square":
            return 1 * 1; // Ejemplo de cálculo de área
        default:
            const _exhaustiveCheck: never = shape;
            return _exhaustiveCheck; // Error si se añade una nueva forma pero no se maneja
    }
}

Conclusión

Entender los tipos any, unknown y never en TypeScript es esencial para escribir código flexible pero seguro en términos de tipos. Mientras que any proporciona flexibilidad, sacrifica la seguridad de tipo. El tipo unknown ofrece una alternativa más segura al exigir comprobaciones de tipo. El tipo never es útil para funciones que nunca retornan y para asegurar comprobaciones exhaustivas de tipos. Al dominar estos tipos, puedes escribir código TypeScript más robusto y mantenible.
