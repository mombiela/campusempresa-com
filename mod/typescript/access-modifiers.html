Modificadores de Acceso en TypeScript
Clases Abstractas ►

Los modificadores de acceso en TypeScript son palabras clave que establecen la accesibilidad de propiedades y métodos en las clases. Ayudan a encapsular los datos y controlar su visibilidad desde fuera de la clase.

Principales Modificadores de Acceso

TypeScript proporciona tres modificadores de acceso principales:

  • public
  • private
  • protected

Público

  • El modificador public es el nivel de acceso predeterminado.
  • Los miembros marcados como public pueden ser accedidos desde cualquier lugar, tanto dentro como fuera de la clase.
class Animal {
    public name: string;

    constructor(name: string) {
        this.name = name;
    }

    public makeSound() {
        console.log(`${this.name} hace un sonido.`);
    }
}

const dog = new Animal('Perro');
console.log(dog.name); // Accesible
dog.makeSound(); // Accesible

Privado

  • El modificador private restringe el acceso a los miembros dentro de la misma clase.
  • Los miembros marcados como private no pueden ser accedidos desde fuera de la clase.
class Animal {
    private name: string;

    constructor(name: string) {
        this.name = name;
    }

    private makeSound() {
        console.log(`${this.name} hace un sonido.`);
    }

    public revealName() {
        console.log(`El nombre del animal es ${this.name}`);
    }
}

const cat = new Animal('Gato');
// console.log(cat.name); // Error: La propiedad 'name' es privada y solo accesible dentro de la clase 'Animal'.
// cat.makeSound(); // Error: El método 'makeSound' es privado y solo accesible dentro de la clase 'Animal'.
cat.revealName(); // Accesible

Protegido

  • El modificador protected permite el acceso a los miembros dentro de la clase y sus subclases.
  • Los miembros marcados como protected no pueden ser accedidos desde fuera de la clase o sus subclases.
class Animal {
    protected name: string;

    constructor(name: string) {
        this.name = name;
    }

    protected makeSound() {
        console.log(`${this.name} hace un sonido.`);
    }
}

class Dog extends Animal {
    constructor(name: string) {
        super(name);
    }

    public bark() {
        console.log(`${this.name} ladra.`);
        this.makeSound(); // Accesible dentro de la subclase
    }
}

const dog = new Dog('Perro');
// console.log(dog.name); // Error: La propiedad 'name' es protegida y solo accesible dentro de la clase 'Animal' y sus subclases.
// dog.makeSound(); // Error: El método 'makeSound' es protegido y solo accesible dentro de la clase 'Animal' y sus subclases.
dog.bark(); // Accesible

Tabla Comparativa

| Modificador | Accesible dentro de la clase | Accesible dentro de la subclase | Accesible fuera de la clase | |-------------|------------------------------|---------------------------------|-----------------------------| | public | Sí | Sí | Sí | | private | Sí | No | No | | protected | Sí | Sí | No |

Conclusión

Los modificadores de acceso en TypeScript son esenciales para controlar la visibilidad y accesibilidad de los miembros de una clase. Ayudan a encapsular datos y asegurar que solo las partes previstas de tu código puedan interactuar con ciertas propiedades y métodos. Entender y usar los modificadores public, private y protected de manera efectiva puede llevar a un código más robusto y mantenible.
