Sobrecargas de Funciones en TypeScript
Funciones Flecha ►

La sobrecarga de funciones es una característica en TypeScript que te permite definir múltiples firmas para una única función. Esto puede ser particularmente útil cuando deseas que una función maneje diferentes tipos de parámetros o devuelva diferentes tipos de resultados según la entrada.

Conceptos Clave

  • Firma de Función: La combinación del nombre de una función y sus tipos de parámetros.
  • Firma de Sobrecarga: Diferentes versiones de una firma de función que una función puede aceptar.
  • Firma de Implementación: La implementación real de la función que maneja todas las sobrecargas.

¿Por Qué Usar Sobrecargas de Funciones?

  • Flexibilidad: Permite que una única función maneje diferentes tipos de entradas.
  • Legibilidad: Hace que el código sea más fácil de entender al definir claramente qué tipos de entradas puede aceptar una función.
  • Seguridad de Tipos: Asegura que la función se use correctamente según sus firmas definidas.

Sintaxis Básica

En TypeScript, las sobrecargas de funciones se definen escribiendo múltiples firmas de función seguidas de una única implementación.

// Firmas de sobrecarga
function add(a: number, b: number): number;
function add(a: string, b: string): string;

// Firma de implementación
function add(a: any, b: any): any {
    return a + b;
}

// Uso
console.log(add(1, 2)); // Salida: 3
console.log(add("Hello, ", "world!")); // Salida: Hello, world!

Ejemplo Detallado

Vamos a crear un ejemplo más detallado para entender mejor las sobrecargas de funciones. Supongamos que queremos crear una función que pueda concatenar dos cadenas o sumar dos números.

// Firmas de sobrecarga
function combine(a: number, b: number): number;
function combine(a: string, b: string): string;

// Firma de implementación
function combine(a: any, b: any): any {
    if (typeof a === "number" && typeof b === "number") {
        return a + b;
    } else if (typeof a === "string" && typeof b === "string") {
        return a + b;
    }
    throw new Error("Argumentos inválidos. Ambos argumentos deben ser del mismo tipo.");
}

// Uso
console.log(combine(10, 20)); // Salida: 30
console.log(combine("foo", "bar")); // Salida: foobar

Ejemplo Avanzado con Parámetros Opcionales

Las sobrecargas de funciones también pueden manejar parámetros opcionales. Vamos a crear una función que pueda devolver el nombre completo de una persona o solo el primer nombre si no se proporciona el apellido.

// Firmas de sobrecarga
function getName(firstName: string, lastName: string): string;
function getName(firstName: string): string;

// Firma de implementación
function getName(firstName: string, lastName?: string): string {
    if (lastName) {
        return `${firstName} ${lastName}`;
    } else {
        return firstName;
    }
}

// Uso
console.log(getName("John", "Doe")); // Salida: John Doe
console.log(getName("John")); // Salida: John

Conclusión

Las sobrecargas de funciones en TypeScript proporcionan una forma poderosa de crear funciones flexibles y seguras en cuanto a tipos. Al definir múltiples firmas para una única función, puedes manejar diferentes tipos de entradas y asegurar que tu función se use correctamente. Esta característica mejora la legibilidad y mantenibilidad del código, haciendo que sea más fácil trabajar con funciones complejas.

Al entender y utilizar las sobrecargas de funciones, puedes escribir código TypeScript más robusto y versátil, atendiendo a una variedad de casos de uso con una única función bien definida.
