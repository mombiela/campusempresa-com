Introducción a los Decoradores de Métodos

Los decoradores de métodos son una característica poderosa en TypeScript que te permiten modificar el comportamiento de los métodos en las clases. Son funciones que se aplican a los métodos para agregar, modificar o extender su funcionalidad sin alterar el código real del método.

Conceptos Clave

  • Función Decoradora: Una función que toma argumentos específicos y devuelve una función que modifica el método objetivo.
  • Objetivo: El prototipo de la clase.
  • Clave de Propiedad: El nombre del método que se está decorando.
  • Descriptor: El descriptor de propiedad del método, que contiene información sobre el método, como su valor, estado de escritura, etc.

Sintaxis Básica

Un decorador de método se define usando el símbolo @ seguido del nombre de la función decoradora, colocado encima de la definición del método.

function simpleDecorator(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    console.log("Decorador de Método llamado en:", propertyKey);
}

class ExampleClass {
    @simpleDecorator
    exampleMethod() {
        console.log("Método Ejemplo");
    }
}

En este ejemplo:

  • simpleDecorator es un decorador de método.
  • Registra el nombre del método que decora.
  • exampleMethod es el método que se está decorando.

Creando un Decorador de Método

Para crear un decorador de método, necesitas definir una función que tome tres parámetros:

  1. target: El prototipo de la clase.
  2. propertyKey: El nombre del método.
  3. descriptor: El descriptor de propiedad del método.

Ejemplo: Decorador de Registro

Vamos a crear un decorador que registre el tiempo de ejecución de un método.

function logExecutionTime(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    const originalMethod = descriptor.value;

    descriptor.value = function (...args: any[]) {
        console.log(`Ejecutando ${propertyKey} con argumentos: ${JSON.stringify(args)}`);
        const start = performance.now();
        const result = originalMethod.apply(this, args);
        const end = performance.now();
        console.log(`Tiempo de ejecución: ${end - start} milisegundos`);
        return result;
    };

    return descriptor;
}

class Calculator {
    @logExecutionTime
    add(a: number, b: number): number {
        return a + b;
    }
}

const calculator = new Calculator();
calculator.add(5, 3);

En este ejemplo:

  • logExecutionTime es un decorador que registra el tiempo de ejecución del método add.
  • Envuelve el método original, registra el tiempo de inicio, llama al método original, registra el tiempo de finalización y calcula la duración.

Uso Avanzado

Los decoradores de métodos pueden usarse para escenarios más avanzados como control de acceso, validación y más.

Ejemplo: Decorador de Control de Acceso

Vamos a crear un decorador que restrinja el acceso a un método basado en roles de usuario.

function authorize(roles: string[]) {
    return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) {
        const originalMethod = descriptor.value;

        descriptor.value = function (...args: any[]) {
            const userRole = args[0]; // Suponiendo que el primer argumento es el rol del usuario
            if (!roles.includes(userRole)) {
                throw new Error("No autorizado");
            }
            return originalMethod.apply(this, args);
        };

        return descriptor;
    };
}

class SecureService {
    @authorize(['admin'])
    secureMethod(userRole: string) {
        console.log("Método Seguro Accedido");
    }
}

const service = new SecureService();
try {
    service.secureMethod('user'); // Lanza error
} catch (error) {
    console.error(error.message);
}
service.secureMethod('admin'); // Funciona bien

En este ejemplo:

  • authorize es un decorador que restringe el acceso basado en roles de usuario.
  • Verifica si el rol del usuario está incluido en los roles permitidos antes de ejecutar el método.

Conclusión

Los decoradores de métodos en TypeScript proporcionan una forma flexible de modificar y extender el comportamiento de los métodos en las clases. Pueden usarse para registro, control de acceso, validación y muchos otros propósitos. Al entender cómo crear y aplicar decoradores de métodos, puedes escribir código más limpio y mantenible.

Resumen

  • Los decoradores de métodos son funciones que modifican el comportamiento de los métodos.
  • Toman tres parámetros: target, propertyKey y descriptor.
  • El uso básico implica registrar o modificar el comportamiento del método.
  • El uso avanzado puede incluir control de acceso, validación y más.

Al dominar los decoradores de métodos, puedes mejorar tus clases de TypeScript con funcionalidades poderosas y reutilizables.
