Introducción

Las declaraciones de variables son un concepto fundamental en TypeScript, como lo son en cualquier lenguaje de programación. Entender cómo declarar y usar variables de manera efectiva es crucial para escribir código limpio, eficiente y libre de errores.

Declaraciones Básicas de Variables

En TypeScript, puedes declarar variables usando tres palabras clave: var, let y const.

`var`

  • La palabra clave var se usa para declarar una variable que puede ser reasignada.
  • Las variables declaradas con var tienen un alcance de función o global, no de bloque.
var message = "¡Hola, Mundo!";
console.log(message); // Salida: ¡Hola, Mundo!

if (true) {
    var count = 10;
}
console.log(count); // Salida: 10 (accesible fuera del bloque)

`let`

  • La palabra clave let se usa para declarar una variable con alcance de bloque que puede ser reasignada.
  • Se prefiere sobre var debido a su comportamiento de alcance de bloque.
let message = "¡Hola, TypeScript!";
console.log(message); // Salida: ¡Hola, TypeScript!

if (true) {
    let count = 10;
    console.log(count); // Salida: 10 (accesible dentro del bloque)
}
console.log(count); // Error: count no está definido (no accesible fuera del bloque)

`const`

  • La palabra clave const se usa para declarar una variable con alcance de bloque que no puede ser reasignada.
  • Se usa para variables que deben permanecer constantes a lo largo del programa.
const message = "¡Hola, TypeScript!";
console.log(message); // Salida: ¡Hola, TypeScript!

message = "Nuevo Mensaje"; // Error: Asignación a una variable constante.

Anotaciones de Tipo

TypeScript te permite especificar el tipo de una variable usando anotaciones de tipo. Esto ayuda a detectar errores en tiempo de compilación y hace que el código sea más legible.

let message: string = "¡Hola, TypeScript!";
let count: number = 10;
let isActive: boolean = true;

Inferencia de Tipos

TypeScript puede inferir automáticamente el tipo de una variable basado en el valor asignado a ella. Esto significa que no siempre necesitas especificar explícitamente el tipo.

let message = "¡Hola, TypeScript!"; // TypeScript infiere que message es de tipo string
let count = 10; // TypeScript infiere que count es de tipo number
let isActive = true; // TypeScript infiere que isActive es de tipo boolean

Alcance de Variables

Entender el alcance de las variables es crucial para escribir código mantenible. El alcance determina la visibilidad y la vida útil de una variable.

Alcance Global

  • Las variables declaradas fuera de cualquier función o bloque están en el alcance global.
  • Pueden ser accedidas desde cualquier parte del código.
var globalMessage = "Soy global";

function showMessage() {
    console.log(globalMessage); // Salida: Soy global
}

Alcance de Función

  • Las variables declaradas dentro de una función están en el alcance de la función.
  • Solo pueden ser accedidas dentro de esa función.
function showMessage() {
    var localMessage = "Soy local";
    console.log(localMessage); // Salida: Soy local
}
console.log(localMessage); // Error: localMessage no está definido

Alcance de Bloque

  • Las variables declaradas con let o const dentro de un bloque (por ejemplo, dentro de una declaración if o un bucle) están en el alcance del bloque.
  • Solo pueden ser accedidas dentro de ese bloque.
if (true) {
    let blockMessage = "Soy de alcance de bloque";
    console.log(blockMessage); // Salida: Soy de alcance de bloque
}
console.log(blockMessage); // Error: blockMessage no está definido

Resumen

  • Usa var para variables con alcance de función o global, pero prefiere let y const para variables con alcance de bloque.
  • Usa let para variables que pueden ser reasignadas y const para variables que deben permanecer constantes.
  • Las anotaciones de tipo ayudan a detectar errores y hacen que el código sea más legible, pero TypeScript también puede inferir tipos.
  • Entender el alcance de las variables (global, de función y de bloque) es esencial para escribir código mantenible.

Dominando las declaraciones de variables en TypeScript, estableces una base sólida para escribir aplicaciones robustas y eficientes en TypeScript.
