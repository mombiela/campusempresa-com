Configuración de un Proyecto React con TypeScript

Introducción

Configurar un proyecto React con TypeScript puede parecer desalentador al principio, pero con las herramientas y pasos adecuados, se vuelve sencillo. Esta sección te guiará a través del proceso de creación de un nuevo proyecto React con TypeScript desde cero, asegurándote de entender cada paso en el camino.

Prerrequisitos

Antes de comenzar, asegúrate de tener lo siguiente instalado en tu máquina:

  • Node.js (se recomienda la versión LTS)
  • npm (viene con Node.js) o Yarn

Guía Paso a Paso

1. Crear un Nuevo Proyecto React

Primero, necesitamos crear un nuevo proyecto React. Podemos usar Create React App (CRA) con la plantilla de TypeScript para este propósito.

npx create-react-app my-typescript-app --template typescript

Este comando creará un nuevo directorio llamado my-typescript-app con un proyecto React configurado para usar TypeScript.

2. Navegar al Directorio de Tu Proyecto

Muévete al directorio del proyecto recién creado:

cd my-typescript-app

3. Entender la Estructura del Proyecto

Tu directorio del proyecto debería verse algo así:

my-typescript-app/
├── node_modules/
├── public/
├── src/
│   ├── App.css
│   ├── App.test.tsx
│   ├── App.tsx
│   ├── index.css
│   ├── index.tsx
│   ├── logo.svg
│   ├── react-app-env.d.ts
│   └── setupTests.ts
├── package.json
├── tsconfig.json
└── README.md
  • src/: Contiene tus componentes de TypeScript y React.
  • tsconfig.json: Archivo de configuración de TypeScript.
  • package.json: Lista de dependencias y scripts.

4. Configuración de TypeScript

El archivo tsconfig.json es crucial para los proyectos de TypeScript. Aquí hay una visión general básica de su contenido:

{
  "compilerOptions": {
    "target": "es5",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "strict": true,
    "forceConsistentCasingInFileNames": true,
    "noFallthroughCasesInSwitch": true,
    "module": "esnext",
    "moduleResolution": "node",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx"
  },
  "include": ["src"]
}
  • target: Especifica la versión objetivo de ECMAScript.
  • lib: Especifica los archivos de biblioteca a incluir.
  • strict: Habilita todas las opciones de verificación estricta de tipos.

5. Escribir Tu Primer Componente en TypeScript

Vamos a crear un componente simple en TypeScript.

Crea un nuevo archivo src/Hello.tsx:

import React from 'react';

interface HelloProps {
  name: string;
}

const Hello: React.FC<HelloProps> = ({ name }) => {
  return <h1>Hello, {name}!</h1>;
};

export default Hello;
  • HelloProps: Define las props para el componente.
  • React.FC<HelloProps>: Especifica que Hello es un componente funcional con props del tipo HelloProps.

6. Usar el Componente

Ahora, vamos a usar nuestro componente Hello en App.tsx.

import React from 'react';
import Hello from './Hello';

function App() {
  return (
    <div className="App">
      <Hello name="World" />
    </div>
  );
}

export default App;

7. Ejecutar el Proyecto

Para ver tu proyecto en acción, ejecuta:

npm start

Esto iniciará el servidor de desarrollo y abrirá tu proyecto en el navegador web predeterminado.

Conclusión

Configurar un proyecto React con TypeScript implica crear un nuevo proyecto con Create React App, entender la estructura del proyecto, configurar TypeScript y escribir tu primer componente en TypeScript. Con estos pasos, ahora tienes una base sólida para construir aplicaciones más complejas usando TypeScript y React.
