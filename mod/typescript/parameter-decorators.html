Introducción a los Decoradores de Parámetros

Los decoradores de parámetros son un tipo especial de decorador en TypeScript que se pueden usar para anotar parámetros dentro de un constructor de clase o método. Permiten modificar el comportamiento de los parámetros, proporcionando una forma de agregar metadatos o realizar validaciones.

Conceptos Clave

  • Decorador: Un tipo especial de declaración que se puede adjuntar a una clase, método, accesor, propiedad o parámetro.
  • Decorador de Parámetro: Un decorador que se aplica a un parámetro específico de un método o constructor.

Sintaxis y Uso

Los decoradores de parámetros se declaran justo antes del parámetro que están decorando. Reciben tres argumentos:

  1. target: El prototipo de la clase para miembros de instancia, o la función constructora para miembros estáticos.
  2. propertyKey: El nombre del método donde se declara el parámetro.
  3. parameterIndex: El índice del parámetro en la lista de parámetros de la función.

Ejemplo Básico

function logParameter(target: Object, propertyKey: string | symbol, parameterIndex: number) {
    console.log(`Parámetro en el índice ${parameterIndex} en el método ${String(propertyKey)} de la clase ${target.constructor.name}`);
}

class ExampleClass {
    method(@logParameter param1: string, @logParameter param2: number) {
        console.log(param1, param2);
    }
}

const example = new ExampleClass();
example.method('test', 42);

En este ejemplo:

  • logParameter es un decorador de parámetros.
  • Registra el índice y el nombre del parámetro que decora.

Aplicaciones Prácticas

Los decoradores de parámetros se pueden usar para varios propósitos, tales como:

  • Validación: Asegurarse de que los parámetros cumplan con ciertos criterios.
  • Inyección de Dependencias: Inyectar automáticamente dependencias en constructores de clase o métodos.
  • Metadatos: Agregar metadatos a los parámetros para su uso posterior.

Ejemplo de Validación

function required(target: Object, propertyKey: string | symbol, parameterIndex: number) {
    const existingRequiredParameters: number[] = Reflect.getOwnMetadata('required', target, propertyKey) || [];
    existingRequiredParameters.push(parameterIndex);
    Reflect.defineMetadata('required', existingRequiredParameters, target, propertyKey);
}

function validate(target: any, propertyName: string, descriptor: TypedPropertyDescriptor<Function>) {
    const method = descriptor.value!;
    descriptor.value = function (...args: any[]) {
        const requiredParameters: number[] = Reflect.getOwnMetadata('required', target, propertyName) || [];
        for (const parameterIndex of requiredParameters) {
            if (args[parameterIndex] === undefined) {
                throw new Error(`Falta el argumento requerido en el índice ${parameterIndex}`);
            }
        }
        return method.apply(this, args);
    };
}

class User {
    @validate
    setName(@required name: string) {
        console.log(`El nombre del usuario es ${name}`);
    }
}

const user = new User();
user.setName('John'); // Funciona bien
user.setName(); // Lanza un error: Falta el argumento requerido en el índice 0

En este ejemplo:

  • required es un decorador de parámetros que marca un parámetro como requerido.
  • validate es un decorador de métodos que verifica si se proporcionan todos los parámetros requeridos.

Conclusión

Los decoradores de parámetros en TypeScript proporcionan una forma poderosa de agregar metadatos, realizar validaciones y gestionar dependencias para los parámetros de métodos y constructores. Al comprender y utilizar los decoradores de parámetros, puedes escribir código más robusto y mantenible. Practica el uso de decoradores de parámetros en diferentes escenarios para comprender completamente su potencial y aplicaciones.
