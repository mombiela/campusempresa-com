Resolución de Módulos
Importaciones Dinámicas ►

La resolución de módulos es un concepto crítico en TypeScript que determina cómo el compilador encuentra y conecta los módulos en tu código. Entender la resolución de módulos ayuda a organizar tu base de código de manera eficiente y a evitar problemas comunes relacionados con las importaciones de módulos.

Conceptos Clave

  • Módulo: Un archivo que contiene código TypeScript. Los módulos pueden exportar e importar funcionalidades.
  • Resolución de Módulos: El proceso que utiliza el compilador de TypeScript para localizar y cargar módulos.
  • Especificador de Módulo: La cadena en la declaración import que le dice al compilador qué módulo cargar.
  • URL Base: Una opción del compilador que establece el directorio base para nombres de módulos no relativos.
  • Rutas: Una opción del compilador que te permite definir alias de módulos.

Estrategias de Resolución de Módulos

TypeScript utiliza dos estrategias principales para la resolución de módulos:

  1. Clásica: La estrategia predeterminada utilizada en TypeScript 1.6 y versiones anteriores.
  2. Node: La estrategia que imita la resolución de módulos de Node.js, adecuada para aplicaciones Node.js.

Resolución de Módulos Clásica

En la estrategia clásica, TypeScript intenta localizar el módulo buscando en el siguiente orden:

  1. Ruta Relativa: Si el especificador del módulo comienza con ./ o ../, TypeScript lo trata como una ruta relativa.
  2. Ruta No Relativa: Si el especificador del módulo no comienza con ./ o ../, TypeScript lo trata como una ruta no relativa y busca en el baseUrl y paths.

Resolución de Módulos Node

La estrategia Node es más compleja y sigue estos pasos:

  1. Ruta Relativa: Similar a la estrategia clásica, si el especificador del módulo comienza con ./ o ../, TypeScript lo trata como una ruta relativa.
  2. Ruta No Relativa: Para rutas no relativas, TypeScript busca en los directorios node_modules, comenzando desde el directorio actual y subiendo en el árbol de directorios.
  3. Extensiones de Archivo: TypeScript intenta resolver el módulo añadiendo las extensiones .ts, .tsx y .d.ts.
  4. Archivos Índice: Si el especificador del módulo apunta a un directorio, TypeScript busca un archivo index.ts, index.tsx o index.d.ts en ese directorio.

Configuración de la Resolución de Módulos

Puedes configurar la resolución de módulos en el archivo tsconfig.json usando la opción moduleResolution.

{
  "compilerOptions": {
    "moduleResolution": "node", // o "classic"
    "baseUrl": "./src",
    "paths": {
      "@app/*": ["app/*"]
    }
  }
}
  • moduleResolution: Especifica la estrategia de resolución de módulos (node o classic).
  • baseUrl: Establece el directorio base para resolver nombres de módulos no relativos.
  • paths: Define alias de módulos, permitiéndote mapear especificadores de módulos a rutas específicas.

Ejemplos

Importación de Módulo Relativo

// Archivo: src/utils/math.ts
export function add(a: number, b: number): number {
  return a + b;
}

// Archivo: src/index.ts
import { add } from './utils/math';

console.log(add(2, 3)); // Salida: 5

Importación de Módulo No Relativo con URL Base y Rutas

// Archivo: src/app/services/logger.ts
export function log(message: string): void {
  console.log(message);
}

// Archivo: src/index.ts
import { log } from '@app/services/logger';

log('Hello, World!'); // Salida: Hello, World!

Resolución de Módulos Node

// Archivo: src/index.ts
import * as fs from 'fs';

fs.writeFileSync('test.txt', 'Hello, World!');

Conclusión

Entender la resolución de módulos en TypeScript es esencial para gestionar dependencias y organizar tu base de código de manera efectiva. Configurando adecuadamente el archivo tsconfig.json y conociendo las diferencias entre las estrategias de resolución de módulos clásica y node, puedes evitar problemas comunes y asegurarte de que tus módulos sean correctamente localizados y cargados por el compilador de TypeScript.
