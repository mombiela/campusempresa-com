Introducción a los Decoradores de Clases

Los decoradores de clases son una característica poderosa en TypeScript que te permiten modificar o anotar clases. Son funciones que toman un constructor de clase como argumento y devuelven un nuevo constructor o modifican el existente.

Conceptos Clave

  • Función Decoradora: Una función que toma un constructor de clase como argumento.
  • Anotación: Añadir metadatos a una clase.
  • Modificación: Cambiar el comportamiento o las propiedades de una clase.

Sintaxis Básica de los Decoradores de Clases

Un decorador de clase se define con el símbolo @ seguido del nombre de la función decoradora.

function simpleDecorator(constructor: Function) {
    console.log("SimpleDecorator llamado.");
}

@simpleDecorator
class MyClass {
    constructor() {
        console.log("Instancia de MyClass creada.");
    }
}

const myInstance = new MyClass();
// Salida:
// SimpleDecorator llamado.
// Instancia de MyClass creada.

Explicación

  • La función simpleDecorator registra un mensaje cuando es llamada.
  • La sintaxis @simpleDecorator aplica el decorador a MyClass.
  • Cuando MyClass es instanciada, la función decoradora se ejecuta primero.

Creando Decoradores de Clases Útiles

Los decoradores de clases pueden ser utilizados para varios propósitos, como registro, validación o modificación de propiedades de la clase.

Ejemplo: Decorador de Registro

function logClass(constructor: Function) {
    console.log(`Clase ${constructor.name} creada.`);
}

@logClass
class User {
    constructor(public name: string) {}
}

const user = new User("Alice");
// Salida:
// Clase User creada.

Explicación

  • El decorador logClass registra el nombre de la clase cuando es creada.
  • La clase User está anotada con @logClass, por lo que cuando se crea una instancia de User, el decorador registra el nombre de la clase.

Uso Avanzado: Modificando el Comportamiento de la Clase

Los decoradores de clases también pueden modificar el comportamiento de la clase devolviendo un nuevo constructor.

Ejemplo: Añadiendo una Marca de Tiempo

function addTimestamp<T extends { new(...args: any[]): {} }>(constructor: T) {
    return class extends constructor {
        timestamp = new Date();
    }
}

@addTimestamp
class Document {
    constructor(public title: string) {}
}

const doc = new Document("Mi Documento");
console.log(doc.timestamp); // Muestra la fecha y hora actual

Explicación

  • El decorador addTimestamp añade una propiedad timestamp a la clase.
  • Devuelve una nueva clase que extiende la clase original y añade la propiedad timestamp.
  • Cuando se crea una instancia de Document, incluye la propiedad timestamp con la fecha y hora actual.

Combinando Múltiples Decoradores

Puedes aplicar múltiples decoradores a una sola clase. Se ejecutan en el orden en que son declarados, de arriba hacia abajo.

Ejemplo: Combinando Decoradores

function firstDecorator(constructor: Function) {
    console.log("FirstDecorator llamado.");
}

function secondDecorator(constructor: Function) {
    console.log("SecondDecorator llamado.");
}

@firstDecorator
@secondDecorator
class Product {
    constructor(public name: string) {}
}

const product = new Product("Laptop");
// Salida:
// FirstDecorator llamado.
// SecondDecorator llamado.

Explicación

  • firstDecorator y secondDecorator se aplican a la clase Product.
  • Ambos decoradores registran un mensaje cuando la clase es creada.
  • Los decoradores se ejecutan en el orden en que son declarados.

Conclusión

Los decoradores de clases en TypeScript proporcionan una forma flexible de modificar y anotar clases. Pueden ser utilizados para registro, validación, añadir propiedades, o incluso alterar el comportamiento de la clase. Entender cómo crear y aplicar decoradores de clases puede mejorar enormemente tus habilidades de programación en TypeScript.

Resumen

  • Los decoradores de clases son funciones que toman un constructor de clase como argumento.
  • Pueden ser utilizados para anotar o modificar clases.
  • Los decoradores se aplican usando el símbolo @.
  • Múltiples decoradores pueden ser combinados y se ejecutan en el orden en que son declarados.

Al dominar los decoradores de clases, puedes escribir código más modular, reutilizable y mantenible en TypeScript.
