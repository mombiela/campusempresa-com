Introducción a las Clases Abstractas

Las clases abstractas son un concepto fundamental en la programación orientada a objetos (OOP) que te permiten definir una clase base con funcionalidad común, mientras delegas algunos detalles de implementación a clases derivadas. En TypeScript, las clases abstractas proporcionan una forma de crear planos para otras clases.

Conceptos Clave

  • Clase Abstracta: Una clase que no puede ser instanciada directamente. Está destinada a ser subclaseada.
  • Método Abstracto: Un método que se declara en una clase abstracta pero no tiene una implementación. Las subclases están obligadas a proporcionar una implementación para estos métodos.
  • Herencia: Las clases abstractas se utilizan como clases base de las cuales otras clases pueden heredar.

Definiendo Clases Abstractas

Para definir una clase abstracta en TypeScript, usa la palabra clave abstract. Las clases abstractas pueden contener tanto métodos abstractos (sin implementación) como métodos concretos (con implementación).

Ejemplo

abstract class Animal {
    // Método abstracto (debe ser implementado en clases derivadas)
    abstract makeSound(): void;

    // Método concreto
    move(): void {
        console.log("Moviéndose...");
    }
}

En este ejemplo, Animal es una clase abstracta con un método abstracto makeSound y un método concreto move.

Implementando Clases Abstractas

Cuando una clase hereda de una clase abstracta, debe implementar todos los métodos abstractos de la clase padre.

Ejemplo

class Dog extends Animal {
    makeSound(): void {
        console.log("¡Guau!");
    }
}

const myDog = new Dog();
myDog.makeSound(); // Salida: ¡Guau!
myDog.move();      // Salida: Moviéndose...

En este ejemplo, Dog es una subclase de Animal y proporciona una implementación para el método makeSound.

Casos de Uso para Clases Abstractas

Las clases abstractas son útiles cuando deseas:

  • Definir una interfaz común para un grupo de clases relacionadas.
  • Proporcionar alguna funcionalidad compartida mientras obligas a las subclases a implementar métodos específicos.
  • Asegurarte de que ciertos métodos sean implementados en clases derivadas.

Comparación: Clases Abstractas vs Interfaces

Tanto las clases abstractas como las interfaces se utilizan para definir la estructura de una clase, pero tienen algunas diferencias.

| Característica | Clase Abstracta | Interfaz | |---------------------|---------------------------------------|------------------------------------| | Herencia | Puede heredar de una clase abstracta | Puede implementar múltiples interfaces | | Implementación de Métodos | Puede tener métodos concretos | No puede tener métodos concretos | | Propiedades | Puede tener propiedades con implementaciones | Solo declaraciones, sin implementaciones | | Instanciación | No puede ser instanciada directamente | No puede ser instanciada directamente |

Ejemplo de Interfaz

interface AnimalInterface {
    makeSound(): void;
    move(): void;
}

class Cat implements AnimalInterface {
    makeSound(): void {
        console.log("¡Miau!");
    }

    move(): void {
        console.log("Moviéndose silenciosamente...");
    }
}

En este ejemplo, Cat implementa la AnimalInterface y proporciona implementaciones para ambos métodos makeSound y move.

Conclusión

Las clases abstractas en TypeScript son una herramienta poderosa para crear un plano común para clases relacionadas. Te permiten definir métodos que deben ser implementados por las subclases, asegurando una interfaz consistente mientras proporcionan funcionalidad compartida. Entender cuándo y cómo usar clases abstractas, así como las diferencias entre clases abstractas e interfaces, es crucial para una programación efectiva en TypeScript.

Al dominar las clases abstractas, puedes crear bases de código más organizadas, mantenibles y escalables.
