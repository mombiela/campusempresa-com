Tipos Básicos
Anotaciones de Tipo ►

En esta sección, cubriremos los tipos básicos en TypeScript. Entender estos tipos es fundamental para escribir código robusto y seguro en cuanto a tipos. Exploraremos los siguientes tipos:

  • boolean
  • number
  • string
  • array
  • tuple
  • enum
  • any
  • void
  • null y undefined
  • never

Booleano

El tipo boolean representa un valor lógico que puede ser true o false.

let isDone: boolean = false;

Número

El tipo number se usa para representar tanto números enteros como de punto flotante.

let decimal: number = 6;
let hex: number = 0xf00d;
let binary: number = 0b1010;
let octal: number = 0o744;

Cadena

El tipo string se usa para representar datos textuales. Puedes usar comillas simples ('), comillas dobles ("), o literales de plantilla (`).

let color: string = "blue";
color = 'red';

let fullName: string = `John Doe`;
let age: number = 30;
let sentence: string = `Hello, my name is ${fullName}. I'll be ${age + 1} years old next month.`;

Arreglo

El tipo array puede escribirse de dos maneras. La primera forma usa el tipo seguido de [], y la segunda forma usa un tipo de arreglo genérico, Array<type>.

let list: number[] = [1, 2, 3];
let list: Array<number> = [1, 2, 3];

Tupla

El tipo tuple te permite expresar un arreglo con un número fijo de elementos cuyos tipos son conocidos pero pueden ser diferentes.

let x: [string, number];
x = ["hello", 10]; // OK
// x = [10, "hello"]; // Error

Enumeración

El tipo enum es una forma de dar nombres más amigables a conjuntos de valores numéricos.

enum Color {Red, Green, Blue}
let c: Color = Color.Green;

También puedes personalizar los valores numéricos:

enum Color {Red = 1, Green, Blue}
let c: Color = Color.Green;

O asignar valores arbitrarios:

enum Color {Red = 1, Green = 2, Blue = 4}
let c: Color = Color.Blue;

Cualquiera

El tipo any es una forma poderosa de trabajar con JavaScript existente, permitiéndote optar por no realizar la verificación de tipos.

let notSure: any = 4;
notSure = "maybe a string instead";
notSure = false; // okay, definitely a boolean

Vacío

El tipo void se usa para indicar la ausencia de cualquier tipo, comúnmente usado como el tipo de retorno de funciones que no devuelven un valor.

function warnUser(): void {
    console.log("This is my warning message");
}

Nulo e Indefinido

TypeScript tiene dos tipos especiales, null y undefined, que tienen sus propios tipos. Por defecto, null y undefined son subtipos de todos los demás tipos.

let u: undefined = undefined;
let n: null = null;

Nunca

El tipo never representa el tipo de valores que nunca ocurren. Por ejemplo, never es el tipo de retorno para una función que siempre lanza una excepción o una que nunca retorna.

function error(message: string): never {
    throw new Error(message);
}

function fail() {
    return error("Something failed");
}

function infiniteLoop(): never {
    while (true) {}
}

Conclusión

Entender y usar los tipos básicos en TypeScript es esencial para escribir código seguro en cuanto a tipos. Estos tipos te ayudan a detectar errores temprano en el proceso de desarrollo y hacen que tu código sea más legible y mantenible. A medida que avances en este curso, verás cómo estos tipos básicos forman la base para construcciones de tipos más complejas y cómo pueden usarse para crear aplicaciones robustas.
