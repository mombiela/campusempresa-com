Propiedades y Métodos Estáticos
Tipos de Intersección y Unión ►

Las propiedades y métodos estáticos son un concepto esencial en TypeScript y en la programación orientada a objetos. Pertenecen a la clase en sí misma en lugar de a las instancias de la clase. Este tema cubrirá los conceptos básicos de las propiedades y métodos estáticos, su uso y proporcionará ejemplos prácticos para solidificar tu comprensión.

¿Qué son las Propiedades y Métodos Estáticos?

  • Propiedades Estáticas: Estas son propiedades que pertenecen a la clase en sí misma, no a ninguna instancia específica de la clase.
  • Métodos Estáticos: Estos son métodos que pueden ser llamados en la clase en sí misma, no en instancias de la clase.

¿Por qué Usar Propiedades y Métodos Estáticos?

  • Datos Compartidos: Las propiedades estáticas pueden ser usadas para almacenar datos que son compartidos entre todas las instancias de una clase.
  • Funciones de Utilidad: Los métodos estáticos son a menudo usados para funciones de utilidad que no requieren ningún dato de las instancias de la clase.

Definiendo Propiedades y Métodos Estáticos

En TypeScript, las propiedades y métodos estáticos se definen usando la palabra clave static.

Ejemplo: Propiedades Estáticas

class MyClass {
    static staticProperty: string = "Soy una propiedad estática";

    static showStaticProperty() {
        console.log(MyClass.staticProperty);
    }
}

// Accediendo a la propiedad estática sin crear una instancia
console.log(MyClass.staticProperty); // Salida: Soy una propiedad estática

Ejemplo: Métodos Estáticos

class MathUtils {
    static add(a: number, b: number): number {
        return a + b;
    }
}

// Llamando al método estático sin crear una instancia
console.log(MathUtils.add(5, 3)); // Salida: 8

Casos de Uso Prácticos

Patrón Singleton

El patrón Singleton asegura que una clase tenga solo una instancia y proporciona un punto de acceso global a ella. Las propiedades y métodos estáticos son a menudo usados para implementar este patrón.

class Singleton {
    private static instance: Singleton;

    private constructor() {
        // Constructor privado para prevenir la instanciación
    }

    static getInstance(): Singleton {
        if (!Singleton.instance) {
            Singleton.instance = new Singleton();
        }
        return Singleton.instance;
    }

    showMessage(): void {
        console.log("¡Hola, soy una instancia singleton!");
    }
}

const singleton1 = Singleton.getInstance();
const singleton2 = Singleton.getInstance();

singleton1.showMessage(); // Salida: ¡Hola, soy una instancia singleton!
console.log(singleton1 === singleton2); // Salida: true

Clases de Utilidad

Las clases de utilidad a menudo contienen métodos estáticos que realizan tareas comunes.

class StringUtils {
    static toUpperCase(str: string): string {
        return str.toUpperCase();
    }

    static toLowerCase(str: string): string {
        return str.toLowerCase();
    }
}

console.log(StringUtils.toUpperCase("hola")); // Salida: HOLA
console.log(StringUtils.toLowerCase("MUNDO")); // Salida: mundo

Puntos Clave a Recordar

  • Las propiedades y métodos estáticos pertenecen a la clase en sí misma, no a las instancias de la clase.
  • Usa la palabra clave static para definir propiedades y métodos estáticos.
  • Los métodos estáticos son a menudo usados para funciones de utilidad y datos compartidos.
  • Las propiedades y métodos estáticos pueden ser accedidos directamente usando el nombre de la clase.

Conclusión

Las propiedades y métodos estáticos son herramientas poderosas en TypeScript que te permiten definir propiedades y métodos que pertenecen a la clase en sí misma en lugar de a las instancias de la clase. Son particularmente útiles para datos compartidos, funciones de utilidad y patrones de diseño como el patrón Singleton. Entender cómo usar las propiedades y métodos estáticos de manera efectiva puede ayudarte a escribir código más organizado y eficiente.
