Decoradores de Propiedades en TypeScript
Decoradores de Parámetro ►

Los decoradores de propiedades son una característica poderosa en TypeScript que te permiten modificar o anotar propiedades dentro de una clase. Proporcionan una forma de agregar metadatos o cambiar el comportamiento de las propiedades de manera declarativa. Este tema te guiará desde los conceptos básicos hasta el uso avanzado de los decoradores de propiedades.

Introducción a los Decoradores de Propiedades

  • Definición: Los decoradores de propiedades son funciones que se aplican a las propiedades de una clase.
  • Propósito: Pueden ser utilizados para agregar metadatos, realizar validaciones o modificar el comportamiento de las propiedades.

Sintaxis Básica

Un decorador de propiedades es una función que toma dos argumentos:

  1. El objeto objetivo (el prototipo de la clase para propiedades de instancia o la función constructora para propiedades estáticas).
  2. El nombre de la propiedad.
function MyPropertyDecorator(target: any, propertyKey: string) {
    console.log(`PropertyDecorator llamado en: ${propertyKey}`);
}

class ExampleClass {
    @MyPropertyDecorator
    myProperty: string;
}

En este ejemplo:

  • MyPropertyDecorator es un decorador simple que registra el nombre de la propiedad a la que se aplica.
  • @MyPropertyDecorator se usa para anotar myProperty en ExampleClass.

Casos de Uso Prácticos

Agregar Metadatos

Los decoradores de propiedades pueden ser utilizados para agregar metadatos a las propiedades, lo cual puede ser útil para varios propósitos como validación, serialización o ORM (Mapeo Objeto-Relacional).

function Required(target: any, propertyKey: string) {
    if (!target.constructor.requiredProperties) {
        target.constructor.requiredProperties = [];
    }
    target.constructor.requiredProperties.push(propertyKey);
}

class User {
    @Required
    name: string;

    @Required
    email: string;
}

console.log(User.requiredProperties); // Salida: ['name', 'email']

Validación

Puedes usar decoradores de propiedades para imponer reglas de validación en las propiedades.

function MinLength(length: number) {
    return function (target: any, propertyKey: string) {
        let value: string;

        const getter = () => value;
        const setter = (newVal: string) => {
            if (newVal.length < length) {
                throw new Error(`La longitud de ${propertyKey} debe ser al menos ${length}`);
            }
            value = newVal;
        };

        Object.defineProperty(target, propertyKey, {
            get: getter,
            set: setter,
            enumerable: true,
            configurable: true
        });
    };
}

class Person {
    @MinLength(3)
    name: string;
}

const person = new Person();
person.name = "Jo"; // Lanza un error: La longitud de name debe ser al menos 3

Uso Avanzado

Combinando Múltiples Decoradores

Puedes aplicar múltiples decoradores a una sola propiedad. Se aplican en el orden en que se declaran.

function FirstDecorator(target: any, propertyKey: string) {
    console.log(`FirstDecorator llamado en: ${propertyKey}`);
}

function SecondDecorator(target: any, propertyKey: string) {
    console.log(`SecondDecorator llamado en: ${propertyKey}`);
}

class Demo {
    @FirstDecorator
    @SecondDecorator
    demoProperty: string;
}

// Salida:
// SecondDecorator llamado en: demoProperty
// FirstDecorator llamado en: demoProperty

Usando Decoradores con Reflexión de Metadatos

TypeScript soporta la reflexión de metadatos a través de la biblioteca reflect-metadata, que puede ser utilizada para almacenar y recuperar metadatos sobre propiedades.

import 'reflect-metadata';

function ReflectMetadataExample(target: any, propertyKey: string) {
    Reflect.defineMetadata('exampleMetadata', 'someValue', target, propertyKey);
}

class Sample {
    @ReflectMetadataExample
    sampleProperty: string;
}

const metadataValue = Reflect.getMetadata('exampleMetadata', Sample.prototype, 'sampleProperty');
console.log(metadataValue); // Salida: 'someValue'

Conclusión

Los decoradores de propiedades en TypeScript proporcionan una forma poderosa de agregar metadatos, imponer validaciones y modificar el comportamiento de las propiedades de clase. Pueden ser utilizados en una variedad de escenarios, desde simples registros hasta validaciones complejas y gestión de metadatos. Al entender y utilizar los decoradores de propiedades, puedes escribir código más declarativo y mantenible.

En este tema, cubrimos:

  • La sintaxis básica y el propósito de los decoradores de propiedades.
  • Casos de uso prácticos, incluyendo agregar metadatos y validación.
  • Uso avanzado como combinar múltiples decoradores y usar reflexión de metadatos.
