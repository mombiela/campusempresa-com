Introducción

En TypeScript, las funciones pueden definirse con parámetros opcionales y predeterminados. Estas características mejoran la flexibilidad y robustez de tu código al permitir que las funciones manejen casos donde algunos argumentos podrían faltar o tener valores predeterminados.

Parámetros Opcionales

Los parámetros opcionales son parámetros que pueden o no ser proporcionados cuando se llama a una función. Se definen usando un signo de interrogación (?) después del nombre del parámetro.

Conceptos Clave

  • Los parámetros opcionales deben venir después de los parámetros requeridos en la firma de la función.
  • Si no se proporciona un parámetro opcional, su valor es undefined.

Ejemplo

function greet(name: string, greeting?: string): string {
    if (greeting) {
        return `${greeting}, ${name}!`;
    } else {
        return `¡Hola, ${name}!`;
    }
}

console.log(greet("Alice")); // Salida: ¡Hola, Alice!
console.log(greet("Bob", "Buenos días")); // Salida: ¡Buenos días, Bob!

En este ejemplo:

  • El parámetro greeting es opcional.
  • Si no se proporciona greeting, la función predetermina a decir "¡Hola!".

Parámetros Predeterminados

Los parámetros predeterminados te permiten especificar un valor predeterminado para un parámetro. Si no se proporciona el argumento, se usa el valor predeterminado.

Conceptos Clave

  • Los parámetros predeterminados pueden usarse en cualquier posición en la lista de parámetros.
  • Si se proporciona un valor para un parámetro predeterminado, este sobrescribe el valor predeterminado.

Ejemplo

function greet(name: string, greeting: string = "¡Hola"): string {
    return `${greeting}, ${name}!`;
}

console.log(greet("Alice")); // Salida: ¡Hola, Alice!
console.log(greet("Bob", "Buenos días")); // Salida: ¡Buenos días, Bob!

En este ejemplo:

  • El parámetro greeting tiene un valor predeterminado de "¡Hola".
  • Si no se proporciona greeting, se usa "¡Hola".

Combinando Parámetros Opcionales y Predeterminados

Puedes combinar parámetros opcionales y predeterminados en una función. Sin embargo, ten en cuenta que los parámetros opcionales deben venir después de los parámetros requeridos.

Ejemplo

function greet(name: string, greeting?: string, punctuation: string = "!"): string {
    if (greeting) {
        return `${greeting}, ${name}${punctuation}`;
    } else {
        return `¡Hola, ${name}${punctuation}`;
    }
}

console.log(greet("Alice")); // Salida: ¡Hola, Alice!
console.log(greet("Bob", "Buenos días")); // Salida: ¡Buenos días, Bob!
console.log(greet("Charlie", undefined, ".")); // Salida: ¡Hola, Charlie.

En este ejemplo:

  • El parámetro greeting es opcional.
  • El parámetro punctuation tiene un valor predeterminado de "!".
  • Si no se proporciona greeting, la función predetermina a decir "¡Hola".

Conclusión

Entender los parámetros opcionales y predeterminados en TypeScript te permite escribir funciones más flexibles y mantenibles. Los parámetros opcionales proporcionan una forma de manejar casos donde algunos argumentos podrían faltar, mientras que los parámetros predeterminados aseguran que tus funciones tengan valores predeterminados sensatos cuando no se proporcionan argumentos. Al combinar estas características, puedes crear funciones robustas que manejen una variedad de escenarios de entrada de manera elegante.
