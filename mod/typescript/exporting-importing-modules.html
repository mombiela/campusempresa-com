Exportar e Importar Módulos
Espacios de Nombres ►

En esta sección, exploraremos cómo exportar e importar módulos en TypeScript. Los módulos son una forma de organizar y encapsular el código, facilitando su gestión y reutilización. Entender cómo trabajar con módulos es crucial para construir aplicaciones escalables y mantenibles.

Conceptos Clave

  • Módulos: Un módulo es un archivo que contiene código TypeScript. Los módulos pueden exportar clases, funciones, variables e interfaces para ser usadas en otros módulos.
  • Exportar: Hacer que el código esté disponible para ser importado en otros módulos.
  • Importar: Traer código de otros módulos para usarlo en el módulo actual.
  • Exportaciones por Defecto: Una única exportación de un módulo que puede ser importada sin llaves.
  • Exportaciones Nombradas: Múltiples exportaciones de un módulo que deben ser importadas usando llaves.

Exportar Módulos

Exportaciones Nombradas

Las exportaciones nombradas permiten exportar múltiples elementos de un módulo. Cada elemento debe ser importado usando su nombre exacto.

// mathUtils.ts
export const add = (a: number, b: number): number => {
    return a + b;
};

export const subtract = (a: number, b: number): number => {
    return a - b;
};

Exportaciones por Defecto

Las exportaciones por defecto permiten exportar un único elemento de un módulo. Este elemento puede ser importado sin usar llaves.

// logger.ts
const log = (message: string): void => {
    console.log(message);
};

export default log;

Importar Módulos

Importar Exportaciones Nombradas

Para importar exportaciones nombradas, se usan llaves y los nombres exactos de las exportaciones.

// app.ts
import { add, subtract } from './mathUtils';

console.log(add(5, 3)); // Salida: 8
console.log(subtract(5, 3)); // Salida: 2

Importar Exportaciones por Defecto

Para importar exportaciones por defecto, simplemente se usa el nombre que se quiere asignar al módulo importado.

// app.ts
import log from './logger';

log('Este es un mensaje de registro'); // Salida: Este es un mensaje de registro

Importar Todas las Exportaciones

Se pueden importar todas las exportaciones de un módulo usando la sintaxis * as. Esto es útil cuando se quiere agrupar todas las exportaciones bajo un único espacio de nombres.

// app.ts
import * as MathUtils from './mathUtils';

console.log(MathUtils.add(5, 3)); // Salida: 8
console.log(MathUtils.subtract(5, 3)); // Salida: 2

Combinando Exportaciones por Defecto y Nombradas

Un módulo puede tener tanto exportaciones por defecto como nombradas. Se pueden importar juntas como se muestra a continuación:

// utils.ts
export const multiply = (a: number, b: number): number => {
    return a * b;
};

const divide = (a: number, b: number): number => {
    return a / b;
};

export default divide;
// app.ts
import divide, { multiply } from './utils';

console.log(multiply(6, 3)); // Salida: 18
console.log(divide(6, 3)); // Salida: 2

Conclusión

Entender cómo exportar e importar módulos en TypeScript es esencial para escribir código modular, mantenible y reutilizable. Usando exportaciones nombradas, exportaciones por defecto y técnicas de importación, puedes gestionar efectivamente las dependencias y organizar tu base de código. Practica estos conceptos con los ejemplos y ejercicios proporcionados para solidificar tu comprensión.
