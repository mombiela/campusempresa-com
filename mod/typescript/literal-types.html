Introducción a los Tipos Literales

Los tipos literales en TypeScript te permiten especificar valores exactos que una variable puede contener. Esta característica mejora la seguridad de tipos al asegurar que las variables solo acepten valores predefinidos.

Conceptos Clave

  • Tipos Literales: Estos son tipos que representan valores específicos en lugar de tipos generales.
  • Literales de Cadena: Tipos que solo pueden ser uno de un conjunto de valores de cadena.
  • Literales Numéricos: Tipos que solo pueden ser uno de un conjunto de valores numéricos.
  • Literales Booleanos: Tipos que solo pueden ser true o false.
  • Tipos de Unión: Combinación de múltiples tipos literales para permitir que una variable contenga cualquiera de varios valores.

Tipos Literales de Cadena

Los tipos literales de cadena te permiten especificar que una variable solo puede ser uno de un conjunto de valores de cadena.

type Direction = 'North' | 'South' | 'East' | 'West';

let direction: Direction;

direction = 'North'; // Válido
direction = 'South'; // Válido
direction = 'Up';    // Error: El tipo '"Up"' no es asignable al tipo 'Direction'.

Ejemplo Explicado

  • type Direction = 'North' | 'South' | 'East' | 'West';: Define un tipo Direction que solo puede ser una de las cuatro cadenas especificadas.
  • let direction: Direction;: Declara una variable direction del tipo Direction.
  • Asignar direction a 'North' o 'South' es válido, pero asignarlo a 'Up' resulta en un error de tipo.

Tipos Literales Numéricos

Los tipos literales numéricos restringen una variable a valores numéricos específicos.

type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;

let roll: DiceRoll;

roll = 1; // Válido
roll = 7; // Error: El tipo '7' no es asignable al tipo 'DiceRoll'.

Ejemplo Explicado

  • type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;: Define un tipo DiceRoll que solo puede ser uno de los seis números especificados.
  • let roll: DiceRoll;: Declara una variable roll del tipo DiceRoll.
  • Asignar roll a 1 es válido, pero asignarlo a 7 resulta en un error de tipo.

Tipos Literales Booleanos

Los tipos literales booleanos restringen una variable a true o false.

type IsEnabled = true | false;

let enabled: IsEnabled;

enabled = true;  // Válido
enabled = false; // Válido
enabled = 'yes'; // Error: El tipo '"yes"' no es asignable al tipo 'IsEnabled'.

Ejemplo Explicado

  • type IsEnabled = true | false;: Define un tipo IsEnabled que solo puede ser true o false.
  • let enabled: IsEnabled;: Declara una variable enabled del tipo IsEnabled.
  • Asignar enabled a true o false es válido, pero asignarlo a 'yes' resulta en un error de tipo.

Tipos de Unión

Los tipos de unión combinan múltiples tipos literales, permitiendo que una variable contenga cualquiera de varios valores.

type Status = 'success' | 'error' | 'loading';

let currentStatus: Status;

currentStatus = 'success'; // Válido
currentStatus = 'error';   // Válido
currentStatus = 'pending'; // Error: El tipo '"pending"' no es asignable al tipo 'Status'.

Ejemplo Explicado

  • type Status = 'success' | 'error' | 'loading';: Define un tipo Status que solo puede ser una de las tres cadenas especificadas.
  • let currentStatus: Status;: Declara una variable currentStatus del tipo Status.
  • Asignar currentStatus a 'success' o 'error' es válido, pero asignarlo a 'pending' resulta en un error de tipo.

Conclusión

Los tipos literales en TypeScript proporcionan una forma de especificar valores exactos que una variable puede contener, mejorando la seguridad de tipos y reduciendo errores. Al usar tipos literales de cadena, numéricos y booleanos, puedes crear código más preciso y predecible. Los tipos de unión extienden aún más esta capacidad al permitir que las variables acepten múltiples valores literales, haciendo tu código TypeScript tanto flexible como robusto.

Resumen

  • Los tipos literales restringen las variables a valores específicos.
  • Los literales de cadena, numéricos y booleanos son tipos comunes.
  • Los tipos de unión combinan múltiples literales para mayor flexibilidad.
  • Usar tipos literales mejora la seguridad de tipos y reduce errores en tiempo de ejecución.
