Introducción a las Pruebas
Pruebas Unitarias con Jest ►

Las pruebas son una parte crucial del desarrollo de software que asegura que tu código se comporte como se espera. En esta sección, exploraremos los conceptos básicos de las pruebas en TypeScript, cubriendo conceptos clave, herramientas y prácticas.

Por Qué las Pruebas son Importantes

  • Aseguramiento de Calidad: Asegura que el software cumpla con los estándares requeridos.
  • Detección de Errores: Identifica problemas temprano en el proceso de desarrollo.
  • Documentación: Las pruebas pueden servir como documentación de cómo se supone que debe funcionar el código.
  • Seguridad en la Refactorización: Te permite hacer cambios con la confianza de que no has roto la funcionalidad existente.

Tipos de Pruebas

  • Pruebas Unitarias: Prueban unidades o componentes individuales del código.
  • Pruebas de Integración: Prueban la interacción entre diferentes unidades o componentes.
  • Pruebas de Extremo a Extremo (E2E): Prueban el flujo completo de la aplicación de principio a fin.

Herramientas de Pruebas para TypeScript

  • Jest: Un marco de pruebas popular con soporte integrado para TypeScript.
  • Mocha: Un marco de pruebas flexible que se puede usar con varias bibliotecas de aserción.
  • Chai: Una biblioteca de aserción que a menudo se usa con Mocha.
  • Sinon: Una biblioteca para crear espías, mocks y stubs.

Configuración de un Entorno de Pruebas

Para comenzar con las pruebas en TypeScript, necesitas configurar tu entorno. Aquí, usaremos Jest como ejemplo.

Instalando Jest

Primero, instala Jest y su soporte para TypeScript:

npm install --save-dev jest ts-jest @types/jest

Configurando Jest

Crea un archivo jest.config.js para configurar Jest para TypeScript:

module.exports = {
  preset: 'ts-jest',
  testEnvironment: 'node',
  testMatch: ['**/?(*.)+(spec|test).[jt]s?(x)'],
};

Escribiendo Tu Primera Prueba

Vamos a escribir una función simple y una prueba para ella.

Función de Ejemplo

Crea un archivo sum.ts:

export function sum(a: number, b: number): number {
  return a + b;
}

Prueba de Ejemplo

Crea un archivo sum.test.ts:

import { sum } from './sum';

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Ejecutando la Prueba

Ejecuta la prueba usando el siguiente comando:

npx jest

Deberías ver una salida indicando que la prueba ha pasado.

Mejores Prácticas para las Pruebas

  • Escribe Pruebas Temprano: Comienza a escribir pruebas tan pronto como escribas tu código.
  • Mantén las Pruebas Pequeñas y Enfocadas: Cada prueba debe enfocarse en una sola funcionalidad.
  • Usa Nombres Descriptivos: Nombra tus pruebas claramente para describir lo que están probando.
  • Mockea Dependencias Externas: Usa mocks para aislar la unidad de código que se está probando.

Conclusión

Las pruebas son una habilidad esencial para cualquier desarrollador. Al escribir pruebas, aseguras que tu código TypeScript sea confiable, mantenible y libre de errores. Comienza con pruebas unitarias simples y gradualmente avanza a pruebas de integración y de extremo a extremo más complejas. Usa herramientas como Jest para agilizar el proceso de pruebas y sigue las mejores prácticas para hacer tus pruebas efectivas y eficientes.
