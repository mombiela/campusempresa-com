Introducción

En TypeScript, las clases e interfaces son bloques de construcción fundamentales para crear código robusto y mantenible. Esta sección te guiará desde los conceptos básicos hasta los avanzados, asegurando que entiendas cómo definir, implementar y utilizar clases e interfaces de manera efectiva.

Clases en TypeScript

Estructura Básica de una Clase

Una clase en TypeScript es un plano para crear objetos con propiedades y métodos predefinidos.

class Person {
    name: string;
    age: number;

    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }

    greet() {
        console.log(`Hola, mi nombre es ${this.name} y tengo ${this.age} años.`);
    }
}

const person1 = new Person('Alice', 30);
person1.greet(); // Salida: Hola, mi nombre es Alice y tengo 30 años.
  • Propiedades: Variables que contienen datos específicos de cada instancia de la clase.
  • Constructor: Un método especial para inicializar nuevos objetos.
  • Métodos: Funciones definidas dentro de una clase que operan sobre las propiedades de la clase.

Herencia

La herencia permite que una clase extienda otra clase, heredando sus propiedades y métodos.

class Employee extends Person {
    employeeId: number;

    constructor(name: string, age: number, employeeId: number) {
        super(name, age);
        this.employeeId = employeeId;
    }

    displayEmployeeId() {
        console.log(`ID de Empleado: ${this.employeeId}`);
    }
}

const employee1 = new Employee('Bob', 25, 12345);
employee1.greet(); // Salida: Hola, mi nombre es Bob y tengo 25 años.
employee1.displayEmployeeId(); // Salida: ID de Empleado: 12345
  • Palabra clave extends: Usada para crear una subclase.
  • Palabra clave super: Llama al constructor de la clase base.

Modificadores de Acceso

TypeScript proporciona modificadores de acceso para controlar la visibilidad de los miembros de la clase.

  • public: Accesible desde cualquier lugar.
  • private: Accesible solo dentro de la clase.
  • protected: Accesible dentro de la clase y sus subclases.
class Animal {
    public name: string;
    private age: number;
    protected species: string;

    constructor(name: string, age: number, species: string) {
        this.name = name;
        this.age = age;
        this.species = species;
    }

    public getAge() {
        return this.age;
    }
}

class Dog extends Animal {
    constructor(name: string, age: number, species: string) {
        super(name, age, species);
    }

    public getSpecies() {
        return this.species;
    }
}

const dog1 = new Dog('Buddy', 5, 'Canino');
console.log(dog1.name); // Salida: Buddy
console.log(dog1.getAge()); // Salida: 5
console.log(dog1.getSpecies()); // Salida: Canino

Interfaces en TypeScript

Definiendo Interfaces

Las interfaces definen la estructura a la que un objeto debe adherirse.

interface Person {
    name: string;
    age: number;
    greet(): void;
}

const person: Person = {
    name: 'Charlie',
    age: 40,
    greet() {
        console.log(`Hola, mi nombre es ${this.name} y tengo ${this.age} años.`);
    }
};

person.greet(); // Salida: Hola, mi nombre es Charlie y tengo 40 años.
  • Propiedades: Definen las propiedades requeridas y sus tipos.
  • Métodos: Definen los métodos requeridos y sus firmas.

Implementando Interfaces en Clases

Las clases pueden implementar interfaces para asegurar que se adhieren a una estructura específica.

interface Drivable {
    startEngine(): void;
    stopEngine(): void;
}

class Car implements Drivable {
    startEngine() {
        console.log('Motor encendido.');
    }

    stopEngine() {
        console.log('Motor apagado.');
    }
}

const myCar = new Car();
myCar.startEngine(); // Salida: Motor encendido.
myCar.stopEngine(); // Salida: Motor apagado.
  • Palabra clave implements: Asegura que la clase se adhiera a la estructura de la interfaz.

Extendiendo Interfaces

Las interfaces pueden extender otras interfaces, permitiendo estructuras más complejas.

interface Animal {
    name: string;
    age: number;
}

interface Mammal extends Animal {
    hasFur: boolean;
}

const mammal: Mammal = {
    name: 'Elefante',
    age: 10,
    hasFur: false
};

console.log(mammal); // Salida: { name: 'Elefante', age: 10, hasFur: false }

Conceptos Avanzados

Clases Abstractas

Las clases abstractas no pueden ser instanciadas directamente y están destinadas a ser subclaseadas.

abstract class Shape {
    abstract area(): number;

    displayArea() {
        console.log(`Área: ${this.area()}`);
    }
}

class Circle extends Shape {
    radius: number;

    constructor(radius: number) {
        super();
        this.radius = radius;
    }

    area(): number {
        return Math.PI * this.radius * this.radius;
    }
}

const circle = new Circle(5);
circle.displayArea(); // Salida: Área: 78.53981633974483
  • Palabra clave abstract: Usada para definir clases y métodos abstractos.

Tipos de Intersección

Combinando múltiples tipos en uno usando tipos de intersección.

interface CanFly {
    fly(): void;
}

interface CanSwim {
    swim(): void;
}

type AmphibiousVehicle = CanFly & CanSwim;

const vehicle: AmphibiousVehicle = {
    fly() {
        console.log('Volando.');
    },
    swim() {
        console.log('Nadando.');
    }
};

vehicle.fly(); // Salida: Volando.
vehicle.swim(); // Salida: Nadando.

Conclusión

Entender las clases e interfaces en TypeScript es crucial para construir aplicaciones escalables y mantenibles. Las clases proporcionan un plano para crear objetos, mientras que las interfaces definen la estructura a la que los objetos deben adherirse. Al dominar estos conceptos, puedes escribir código más robusto y seguro en cuanto a tipos.
