Tipos Condicionales en TypeScript
Tipos Mapeados ►

Introducción

Los tipos condicionales en TypeScript te permiten crear tipos que dependen de una condición. Son una característica poderosa que puede hacer que tus definiciones de tipos sean más flexibles y expresivas. Este tema te guiará desde lo básico hasta el uso avanzado de los tipos condicionales.

Sintaxis Básica

Los tipos condicionales siguen una sintaxis simple:

T extends U ? X : Y
  • T extends U es la condición.
  • Si la condición es verdadera, el tipo se resuelve a X.
  • Si la condición es falsa, el tipo se resuelve a Y.

Ejemplo

Comencemos con un ejemplo básico:

type IsString<T> = T extends string ? "Yes" : "No";

type A = IsString<string>; // "Yes"
type B = IsString<number>; // "No"

En este ejemplo:

  • IsString<string> se evalúa como "Yes" porque string extiende string.
  • IsString<number> se evalúa como "No" porque number no extiende string.

Tipos Condicionales con Genéricos

Los tipos condicionales se vuelven más poderosos cuando se usan con genéricos.

Ejemplo

Considera una función que devuelve diferentes tipos basados en una condición:

type ReturnType<T> = T extends () => infer R ? R : never;

type Func = () => string;
type Result = ReturnType<Func>; // string

Aquí:

  • ReturnType<T> verifica si T es un tipo de función.
  • Si es verdadero, extrae el tipo de retorno R usando infer.
  • Si es falso, se resuelve a never.

Tipos Condicionales Distributivos

Los tipos condicionales se distribuyen sobre uniones. Esto significa que si aplicas un tipo condicional a un tipo de unión, TypeScript aplicará el tipo condicional a cada miembro de la unión individualmente.

Ejemplo

type ToArray<T> = T extends any ? T[] : never;

type StrArray = ToArray<string>; // string[]
type NumArray = ToArray<number>; // number[]
type UnionArray = ToArray<string | number>; // string[] | number[]

En este ejemplo:

  • ToArray<string | number> se distribuye sobre la unión, resultando en string[] | number[].

Uso Avanzado

Los tipos condicionales pueden combinarse con otras características de TypeScript para crear definiciones de tipos complejas y poderosas.

Ejemplo: Filtrando Tipos

Puedes usar tipos condicionales para filtrar ciertos tipos de una unión.

type Exclude<T, U> = T extends U ? never : T;

type T1 = Exclude<"a" | "b" | "c", "a">; // "b" | "c"
type T2 = Exclude<string | number | (() => void), Function>; // string | number

Aquí:

  • Exclude<T, U> elimina tipos de T que son asignables a U.

Aplicación Práctica

Los tipos condicionales se usan a menudo en proyectos reales de TypeScript para crear tipos más precisos y flexibles.

Ejemplo: Deep Partial

Un tipo de utilidad común es DeepPartial, que hace que todas las propiedades de un tipo sean opcionales, recursivamente.

type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface User {
  name: string;
  address: {
    street: string;
    city: string;
  };
}

type PartialUser = DeepPartial<User>;
/*
PartialUser es:
{
  name?: string;
  address?: {
    street?: string;
    city?: string;
  };
}
*/

En este ejemplo:

  • DeepPartial<T> usa un tipo condicional para verificar si una propiedad es un objeto.
  • Si lo es, aplica DeepPartial recursivamente.

Conclusión

Los tipos condicionales en TypeScript proporcionan una forma poderosa de crear definiciones de tipos flexibles y expresivas. Te permiten definir tipos que dependen de condiciones, trabajar con genéricos e incluso distribuirse sobre uniones. Al dominar los tipos condicionales, puedes escribir código TypeScript más robusto y mantenible.
