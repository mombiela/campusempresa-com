Anotaciones de Tipo en TypeScript
Compilando TypeScript ►

Las anotaciones de tipo son una característica poderosa en TypeScript que te permiten especificar los tipos de variables, parámetros de funciones y valores de retorno. Esto ayuda a detectar errores temprano y hace que tu código sea más legible y mantenible.

¿Qué son las Anotaciones de Tipo?

Las anotaciones de tipo son una forma de declarar explícitamente el tipo de una variable, parámetro de función o valor de retorno. Esto ayuda a TypeScript a entender qué tipo de datos se están utilizando y puede prevenir muchos errores comunes de programación.

Conceptos Clave

  • Inferencia de Tipo: TypeScript a menudo puede inferir el tipo de una variable basado en su valor.
  • Anotaciones Explícitas: Puedes declarar explícitamente el tipo de una variable, parámetro de función o valor de retorno.
  • Tipos Básicos: Los tipos comunes incluyen string, number, boolean, array, tuple, enum, any, void, null y undefined.

Anotaciones de Tipo Básicas

Variables

Puedes anotar variables con un tipo específico usando dos puntos seguidos del nombre del tipo.

let username: string = "JohnDoe";
let age: number = 30;
let isAdmin: boolean = true;

Funciones

Anota los parámetros de las funciones y los tipos de retorno para asegurarte de que se usen correctamente.

function add(a: number, b: number): number {
    return a + b;
}

let result: number = add(5, 3);

Arrays

Anota arrays especificando el tipo de elementos que contienen.

let numbers: number[] = [1, 2, 3, 4, 5];
let names: string[] = ["Alice", "Bob", "Charlie"];

Anotaciones de Tipo Avanzadas

Tuplas

Las tuplas son arrays con un número fijo de elementos de tipos específicos.

let person: [string, number] = ["Alice", 25];

Enums

Los enums te permiten definir un conjunto de constantes con nombre.

enum Color {
    Red,
    Green,
    Blue
}

let favoriteColor: Color = Color.Green;

Any

El tipo any puede contener cualquier valor, pero debe usarse con moderación ya que omite la verificación de tipos.

let randomValue: any = 42;
randomValue = "Hello";
randomValue = true;

Void

El tipo void se usa para funciones que no retornan un valor.

function logMessage(message: string): void {
    console.log(message);
}

Null y Undefined

Puedes anotar explícitamente variables como null o undefined.

let emptyValue: null = null;
let notAssigned: undefined = undefined;

Anotaciones de Tipo en la Práctica

Ejemplo: Perfil de Usuario

Vamos a crear un perfil de usuario simple con anotaciones de tipo.

type UserProfile = {
    username: string;
    age: number;
    isAdmin: boolean;
};

let user: UserProfile = {
    username: "JohnDoe",
    age: 30,
    isAdmin: true
};

function printUserProfile(user: UserProfile): void {
    console.log(`Username: ${user.username}, Age: ${user.age}, Admin: ${user.isAdmin}`);
}

printUserProfile(user);

Ejemplo: Carrito de Compras

Un ejemplo más complejo que involucra arrays y funciones.

type Product = {
    id: number;
    name: string;
    price: number;
};

let cart: Product[] = [];

function addToCart(product: Product): void {
    cart.push(product);
}

function calculateTotal(cart: Product[]): number {
    return cart.reduce((total, product) => total + product.price, 0);
}

addToCart({ id: 1, name: "Laptop", price: 999.99 });
addToCart({ id: 2, name: "Mouse", price: 19.99 });

console.log(`Total: $${calculateTotal(cart)}`);

Conclusión

Las anotaciones de tipo en TypeScript proporcionan una forma robusta de asegurar que tu código sea seguro en cuanto a tipos y más fácil de entender. Al declarar explícitamente los tipos de variables, parámetros de funciones y valores de retorno, puedes detectar errores temprano y hacer que tu código sea más mantenible. A medida que te sientas más cómodo con TypeScript, encontrarás que las anotaciones de tipo son una herramienta invaluable en tu kit de herramientas de programación.
