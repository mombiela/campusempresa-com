Clases Genéricas
Restricciones Genéricas ►

En esta sección, exploraremos el concepto de Clases Genéricas en TypeScript. Los genéricos te permiten crear componentes flexibles y reutilizables que pueden trabajar con una variedad de tipos de datos. Esto es particularmente útil para crear estructuras de datos como listas, pilas y colas.

¿Qué son los Genéricos?

Los genéricos proporcionan una manera de crear clases, funciones e interfaces que pueden trabajar con diferentes tipos de datos sin sacrificar la seguridad de tipos. Te permiten definir un marcador de posición para un tipo que puede ser especificado más tarde.

Conceptos Clave

  • Parámetro de Tipo: Un marcador de posición para un tipo que se especifica cuando la clase genérica es instanciada.
  • Seguridad de Tipos: Asegura que los tipos de las variables sean conocidos en tiempo de compilación, reduciendo errores en tiempo de ejecución.
  • Reusabilidad: Las clases genéricas pueden ser reutilizadas con diferentes tipos de datos, haciendo tu código más flexible y mantenible.

Definiendo una Clase Genérica

Para definir una clase genérica, usas corchetes angulares (<>) para especificar el parámetro de tipo. Aquí hay un ejemplo simple de una clase genérica en TypeScript:

class Box<T> {
    private content: T;

    constructor(content: T) {
        this.content = content;
    }

    getContent(): T {
        return this.content;
    }

    setContent(content: T): void {
        this.content = content;
    }
}

En este ejemplo:

  • T es el parámetro de tipo.
  • La clase Box ahora puede contener cualquier tipo de contenido, especificado cuando la clase es instanciada.

Usando una Clase Genérica

Puedes crear instancias de la clase genérica con diferentes tipos:

let numberBox = new Box<number>(123);
console.log(numberBox.getContent()); // Salida: 123

let stringBox = new Box<string>("¡Hola, TypeScript!");
console.log(stringBox.getContent()); // Salida: ¡Hola, TypeScript!

Múltiples Parámetros de Tipo

También puedes definir una clase genérica con múltiples parámetros de tipo:

class Pair<K, V> {
    private key: K;
    private value: V;

    constructor(key: K, value: V) {
        this.key = key;
        this.value = value;
    }

    getKey(): K {
        return this.key;
    }

    getValue(): V {
        return this.value;
    }
}

Usando la clase Pair:

let pair = new Pair<string, number>("edad", 30);
console.log(pair.getKey()); // Salida: edad
console.log(pair.getValue()); // Salida: 30

Restricciones en Parámetros de Tipo

A veces podrías querer restringir los tipos que pueden ser usados como parámetros de tipo. Puedes lograr esto usando restricciones:

class Container<T extends { length: number }> {
    private content: T;

    constructor(content: T) {
        this.content = content;
    }

    getContentLength(): number {
        return this.content.length;
    }
}

En este ejemplo:

  • El parámetro de tipo T está restringido a tipos que tienen una propiedad length.

Usando la clase Container:

let stringContainer = new Container<string>("Hola");
console.log(stringContainer.getContentLength()); // Salida: 5

let arrayContainer = new Container<number[]>([1, 2, 3, 4]);
console.log(arrayContainer.getContentLength()); // Salida: 4

Resumen

Las clases genéricas en TypeScript proporcionan una manera poderosa de crear componentes flexibles y reutilizables. Al usar parámetros de tipo, puedes escribir clases que funcionen con cualquier tipo de datos mientras mantienes la seguridad de tipos. Esto hace tu código más robusto y fácil de mantener.

Puntos Clave

  • Los genéricos te permiten crear clases que pueden trabajar con cualquier tipo de datos.
  • Los parámetros de tipo son marcadores de posición para tipos que se especifican cuando la clase es instanciada.
  • Puedes definir múltiples parámetros de tipo y agregar restricciones a ellos.
  • Usar genéricos mejora la reusabilidad del código y la seguridad de tipos.

Al dominar los genéricos, puedes escribir código TypeScript más flexible y mantenible, haciendo tus aplicaciones más robustas y fáciles de extender.
