Manejo de Errores en TypeScript
Refactorización de Código TypeScript ►

El manejo de errores es un aspecto crucial de la programación que asegura que tu aplicación pueda manejar situaciones inesperadas de manera elegante. En TypeScript, el manejo de errores es similar a JavaScript pero con características adicionales de seguridad de tipos. Esta sección te guiará desde los conceptos básicos hasta los avanzados del manejo de errores en TypeScript.

Conceptos Básicos del Manejo de Errores

La Declaración `try...catch`

La declaración try...catch te permite capturar errores que ocurren durante la ejecución y manejarlos adecuadamente.

try {
    // Código que puede lanzar un error
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    // Manejar el error
    console.error("Ocurrió un error:", error);
}
  • Bloque try: Contiene el código que podría lanzar un error.
  • Bloque catch: Contiene el código para manejar el error.

Lanzar Errores

Puedes lanzar errores usando la declaración throw. Esto es útil para el manejo de errores personalizados.

function riskyOperation() {
    if (Math.random() > 0.5) {
        throw new Error("¡Algo salió mal!");
    }
    return "¡Éxito!";
}

try {
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    console.error("Se capturó un error:", error.message);
}

Manejo Avanzado de Errores

Clases de Errores Personalizadas

Crear clases de errores personalizadas puede ayudarte a manejar diferentes tipos de errores de manera más efectiva.

class CustomError extends Error {
    constructor(message: string) {
        super(message);
        this.name = "CustomError";
    }
}

function riskyOperation() {
    throw new CustomError("¡Este es un error personalizado!");
}

try {
    riskyOperation();
} catch (error) {
    if (error instanceof CustomError) {
        console.error("Se capturó un error personalizado:", error.message);
    } else {
        console.error("Se capturó un error desconocido:", error);
    }
}

Uso del Bloque `finally`

El bloque finally contiene código que se ejecutará independientemente de si se lanzó un error o no.

try {
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    console.error("Ocurrió un error:", error);
} finally {
    console.log("Esto siempre se ejecutará.");
}

Manejo de Errores con Promesas

Manejar errores en código asincrónico es crucial. Las promesas proporcionan un método catch para manejar errores.

function asyncOperation(): Promise<string> {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            if (Math.random() > 0.5) {
                resolve("¡Éxito!");
            } else {
                reject(new Error("¡La operación asincrónica falló!"));
            }
        }, 1000);
    });
}

asyncOperation()
    .then(result => {
        console.log(result);
    })
    .catch(error => {
        console.error("Se capturó un error:", error.message);
    });

Manejo de Errores con `async/await`

La sintaxis async/await facilita el trabajo con código asincrónico y el manejo de errores.

async function performAsyncOperation() {
    try {
        let result = await asyncOperation();
        console.log(result);
    } catch (error) {
        console.error("Se capturó un error:", error.message);
    }
}

performAsyncOperation();

Conclusión

El manejo de errores en TypeScript implica el uso de declaraciones try...catch, la creación de clases de errores personalizadas y el manejo de errores en código asincrónico usando Promesas y async/await. Al entender e implementar estas técnicas, puedes crear aplicaciones robustas que puedan manejar situaciones inesperadas de manera elegante.
