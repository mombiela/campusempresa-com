Introducción

El estilo y formato del código son cruciales para mantener la legibilidad, consistencia y colaboración en proyectos de TypeScript. Esta sección cubrirá las mejores prácticas, herramientas y directrices para ayudarte a escribir código TypeScript limpio y mantenible.

Conceptos Clave

  • Consistencia: Asegurarse de que tu código siga un estilo consistente en todo el proyecto.
  • Legibilidad: Escribir código que sea fácil de leer y entender.
  • Herramientas: Usar herramientas para automatizar el formato del código y hacer cumplir las directrices de estilo.

Mejores Prácticas

Convenciones de Nombres

  • Variables y Funciones: Usa camelCase. 
    let userName: string;
function getUserInfo() {}
  • Clases e Interfaces: Usa PascalCase. 
    class UserProfile {}
interface User {
  name: string;
  age: number;
}
  • Constantes: Usa UPPER_CASE con guiones bajos. 
    const MAX_USERS = 100;

Indentación y Espaciado

  • Usa 2 o 4 espacios para la indentación (elige uno y mantente con él).
  • Añade espacios alrededor de los operadores y después de las comas. 
    let sum = a + b;
let numbers = [1, 2, 3];

Punto y Coma

  • Usa siempre punto y coma para terminar las sentencias. 
    let user = 'John';
console.log(user);

Longitud de Línea

  • Mantén las líneas por debajo de 80 o 100 caracteres para mejorar la legibilidad.

Llaves y Bloques

  • Usa llaves para todas las estructuras de control, incluso bloques de una sola línea. 
    if (isValid) {
  console.log('Valid');
} else {
  console.log('Invalid');
}

Herramientas

Prettier

Prettier es un formateador de código opinado que soporta TypeScript. Hace cumplir un estilo consistente al analizar tu código y reimprimirlo con sus propias reglas.

  • Instalación: 
    npm install --save-dev prettier
  • Configuración: Crea un archivo .prettierrc en la raíz de tu proyecto. 
    {
  "semi": true,
  "singleQuote": true,
  "trailingComma": "all",
  "tabWidth": 2
}
  • Uso: 
    npx prettier --write src/

ESLint

ESLint es una herramienta de análisis estático de código para identificar patrones problemáticos en código JavaScript y TypeScript.

  • Instalación: 
    npm install --save-dev eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
  • Configuración: Crea un archivo .eslintrc.json. 
    {
  "parser": "@typescript-eslint/parser",
  "plugins": ["@typescript-eslint"],
  "extends": ["eslint:recommended", "plugin:@typescript-eslint/recommended"],
  "rules": {
    "semi": ["error", "always"],
    "quotes": ["error", "single"]
  }
}
  • Uso: 
    npx eslint src/**/*.ts

Ejemplo

Aquí hay un ejemplo de un archivo TypeScript bien formateado:

// Importando módulos necesarios
import { User } from './models/User';

// Constantes
const MAX_USERS = 100;

// Definición de clase
class UserProfile {
  private user: User;

  constructor(user: User) {
    this.user = user;
  }

  public getUserInfo(): string {
    return `Name: ${this.user.name}, Age: ${this.user.age}`;
  }
}

// Función para crear un nuevo perfil de usuario
function createUserProfile(name: string, age: number): UserProfile {
  const user: User = { name, age };
  return new UserProfile(user);
}

// Ejemplo de uso
const userProfile = createUserProfile('John Doe', 30);
console.log(userProfile.getUserInfo());

Conclusión

Adherirse a un estilo de código y directrices de formato consistentes es esencial para escribir código TypeScript limpio y mantenible. Siguiendo las mejores prácticas y aprovechando herramientas como Prettier y ESLint, puedes asegurarte de que tu base de código se mantenga legible y fácil de trabajar, tanto para ti como para otros.
