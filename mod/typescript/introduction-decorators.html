¿Qué son los Decoradores?

Los decoradores son un tipo especial de declaración que puede adjuntarse a una clase, método, accesor, propiedad o parámetro. Los decoradores son una propuesta de etapa 2 para JavaScript y están disponibles como una característica experimental en TypeScript. Proporcionan una forma de agregar tanto anotaciones como una sintaxis de meta-programación para declaraciones de clases y miembros.

¿Por qué Usar Decoradores?

  • Reutilización de Código: Los decoradores te permiten reutilizar lógica común en múltiples clases o métodos.
  • Separación de Preocupaciones: Ayudan a separar preocupaciones transversales como el registro, la validación y el manejo de errores de la lógica principal del negocio.
  • Mayor Legibilidad: Al usar decoradores, puedes hacer que tu código sea más legible y mantenible.

Tipos de Decoradores

  • Decoradores de Clase: Se utilizan para modificar o anotar una clase.
  • Decoradores de Método: Se utilizan para modificar o anotar un método.
  • Decoradores de Accesor: Se utilizan para modificar o anotar un getter o setter.
  • Decoradores de Propiedad: Se utilizan para modificar o anotar una propiedad.
  • Decoradores de Parámetro: Se utilizan para modificar o anotar un parámetro.

Sintaxis Básica

La sintaxis básica para un decorador en TypeScript es la siguiente:

function decoratorName(target: any, propertyKey?: string, descriptor?: PropertyDescriptor) {
    // Lógica del decorador aquí
}

Ejemplos

Decorador de Clase

Un decorador de clase se aplica al constructor de la clase. Puede usarse para modificar o reemplazar la definición de la clase.

function sealed(constructor: Function) {
    Object.seal(constructor);
    Object.seal(constructor.prototype);
}

@sealed
class Greeter {
    greeting: string;
    constructor(message: string) {
        this.greeting = message;
    }
    greet() {
        return "Hello, " + this.greeting;
    }
}

En este ejemplo, el decorador sealed sella la clase Greeter, evitando que se le agreguen nuevas propiedades.

Decorador de Método

Un decorador de método se aplica a un método de una clase. Puede usarse para modificar el comportamiento del método.

function log(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    const originalMethod = descriptor.value;

    descriptor.value = function (...args: any[]) {
        console.log(`Calling ${propertyKey} with arguments: ${JSON.stringify(args)}`);
        const result = originalMethod.apply(this, args);
        console.log(`Result: ${result}`);
        return result;
    };

    return descriptor;
}

class Calculator {
    @log
    add(a: number, b: number): number {
        return a + b;
    }
}

const calculator = new Calculator();
calculator.add(2, 3);

En este ejemplo, el decorador log registra el nombre del método y los argumentos antes y después de la ejecución del método.

Decorador de Propiedad

Un decorador de propiedad se aplica a una propiedad de una clase. Puede usarse para modificar la propiedad.

function readonly(target: any, propertyKey: string) {
    Object.defineProperty(target, propertyKey, {
        writable: false
    });
}

class Person {
    @readonly
    name: string;

    constructor(name: string) {
        this.name = name;
    }
}

const person = new Person("John");
person.name = "Doe"; // Error: No se puede asignar a 'name' porque es una propiedad de solo lectura.

En este ejemplo, el decorador readonly hace que la propiedad name sea de solo lectura.

Conclusión

Los decoradores en TypeScript proporcionan una forma poderosa de agregar anotaciones y capacidades de meta-programación a tus clases y sus miembros. Ayudan a mejorar la reutilización del código, la separación de preocupaciones y la legibilidad. Al entender y utilizar decoradores, puedes escribir aplicaciones TypeScript más mantenibles y escalables.
