Introducción a las Sentencias Case

Las sentencias case en Delphi/Object Pascal se utilizan para simplificar la lógica condicional compleja. Proporcionan una forma de ejecutar diferentes bloques de código según el valor de una variable o expresión.

Conceptos Clave

  • Estructura de la Sentencia Case: La estructura básica de una sentencia case.
  • Sentencias Case Simples: Uso de sentencias case con tipos ordinales.
  • Sentencias Case Anidadas: Uso de sentencias case dentro de otras sentencias case.
  • Sentencias Case con Cadenas: Uso de sentencias case con valores de cadena.
  • Caso Predeterminado: Manejo de valores inesperados.

Estructura de la Sentencia Case

La sintaxis general de una sentencia case en Delphi/Object Pascal es la siguiente:

case expression of
  value1: statement1;
  value2: statement2;
  value3: statement3;
  ...
else
  default_statement;
end;
  • expression: La variable o expresión que se está evaluando.
  • value1, value2, value3, ...: Posibles valores de la expresión.
  • statement1, statement2, statement3, ...: Sentencias a ejecutar para cada valor correspondiente.
  • else: Opcional. Se ejecuta si ninguno de los valores especificados coincide con la expresión.

Sentencias Case Simples

Un ejemplo de sentencia case simple con un tipo ordinal (entero):

var
  day: Integer;
begin
  day := 3;
  case day of
    1: WriteLn('Lunes');
    2: WriteLn('Martes');
    3: WriteLn('Miércoles');
    4: WriteLn('Jueves');
    5: WriteLn('Viernes');
    6: WriteLn('Sábado');
    7: WriteLn('Domingo');
  else
    WriteLn('Día inválido');
  end;
end;

Explicación

  • Se evalúa la variable day.
  • Dependiendo del valor de day, se ejecuta la sentencia WriteLn correspondiente.
  • Si day no está entre 1 y 7, se ejecuta la cláusula else, imprimiendo 'Día inválido'.

Sentencias Case Anidadas

Las sentencias case pueden anidarse dentro de otras sentencias case para manejar lógica más compleja:

var
  category, item: Integer;
begin
  category := 1;
  item := 2;
  case category of
    1: begin
      case item of
        1: WriteLn('Categoría 1, Ítem 1');
        2: WriteLn('Categoría 1, Ítem 2');
      else
        WriteLn('Ítem desconocido en Categoría 1');
      end;
    end;
    2: begin
      case item of
        1: WriteLn('Categoría 2, Ítem 1');
        2: WriteLn('Categoría 2, Ítem 2');
      else
        WriteLn('Ítem desconocido en Categoría 2');
      end;
    end;
  else
    WriteLn('Categoría desconocida');
  end;
end;

Explicación

  • La sentencia case externa evalúa category.
  • Dependiendo del valor de category, una sentencia case interna evalúa item.
  • Esta estructura permite un manejo detallado de combinaciones de category y item.

Sentencias Case con Cadenas

Delphi/Object Pascal también soporta sentencias case con valores de cadena:

var
  command: string;
begin
  command := 'start';
  case command of
    'start': WriteLn('Iniciando el proceso');
    'stop': WriteLn('Deteniendo el proceso');
    'pause': WriteLn('Pausando el proceso');
  else
    WriteLn('Comando desconocido');
  end;
end;

Explicación

  • Se evalúa la variable command.
  • Dependiendo del valor de command, se ejecuta la sentencia WriteLn correspondiente.
  • Si command no coincide con ningún valor especificado, se ejecuta la cláusula else, imprimiendo 'Comando desconocido'.

Caso Predeterminado

La cláusula else en una sentencia case actúa como un caso predeterminado, manejando cualquier valor no listado explícitamente:

var
  grade: Char;
begin
  grade := 'B';
  case grade of
    'A': WriteLn('Excelente');
    'B': WriteLn('Bueno');
    'C': WriteLn('Promedio');
    'D': WriteLn('Debajo del promedio');
    'F': WriteLn('Reprobado');
  else
    WriteLn('Calificación inválida');
  end;
end;

Explicación

  • Se evalúa la variable grade.
  • Dependiendo del valor de grade, se ejecuta la sentencia WriteLn correspondiente.
  • Si grade no coincide con ningún valor especificado, se ejecuta la cláusula else, imprimiendo 'Calificación inválida'.

Conclusión

Las sentencias case en Delphi/Object Pascal proporcionan una forma poderosa de manejar múltiples condiciones con código claro y legible. Pueden usarse con tipos ordinales, cadenas y pueden anidarse para lógica compleja. La cláusula else asegura que los valores inesperados se manejen de manera adecuada. Al dominar las sentencias case, puedes escribir código más eficiente y mantenible.
