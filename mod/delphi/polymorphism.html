Polimorfismo en Delphi/Object Pascal
Encapsulación ►

El polimorfismo es un concepto fundamental en la programación orientada a objetos que permite que los objetos sean tratados como instancias de su clase padre en lugar de su clase real. Esto permite que una única interfaz represente diferentes formas subyacentes (tipos de datos). En Delphi/Object Pascal, el polimorfismo se logra a través de la herencia y las interfaces.

Conceptos Clave del Polimorfismo

  • Herencia: Permite que una clase herede propiedades y métodos de otra clase.
  • Sobrescritura de Métodos: Permite que una subclase proporcione una implementación específica de un método que ya está definido en su superclase.
  • Interfaces: Definen métodos que deben ser implementados por cualquier clase que implemente la interfaz, permitiendo que diferentes clases sean tratadas de manera uniforme.

Herencia y Sobrescritura de Métodos

La herencia es el mecanismo por el cual una clase (la subclase) puede heredar las propiedades y métodos de otra clase (la superclase). La sobrescritura de métodos permite que la subclase proporcione una implementación específica de un método que ya está definido en su superclase.

Ejemplo de Herencia y Sobrescritura de Métodos

program PolymorphismExample;

{$APPTYPE CONSOLE}

uses
  SysUtils;

type
  TAnimal = class
  public
    procedure Speak; virtual;
  end;

  TDog = class(TAnimal)
  public
    procedure Speak; override;
  end;

  TCat = class(TAnimal)
  public
    procedure Speak; override;
  end;

procedure TAnimal.Speak;
begin
  WriteLn('El animal hace un sonido.');
end;

procedure TDog.Speak;
begin
  WriteLn('El perro ladra.');
end;

procedure TCat.Speak;
begin
  WriteLn('El gato maúlla.');
end;

var
  Animal: TAnimal;

begin
  Animal := TDog.Create;
  Animal.Speak; // Salida: El perro ladra.

  Animal := TCat.Create;
  Animal.Speak; // Salida: El gato maúlla.

  ReadLn;
end.

En este ejemplo:

  • TAnimal es la clase base con un método virtual Speak.
  • TDog y TCat son subclases que sobrescriben el método Speak.
  • La variable Animal puede contener referencias a objetos TDog y TCat, y el método Speak correcto se llama según el tipo real del objeto.

Interfaces

Las interfaces en Delphi/Object Pascal proporcionan una forma de definir métodos que deben ser implementados por cualquier clase que implemente la interfaz. Esto permite que diferentes clases sean tratadas de manera uniforme.

Ejemplo de Interfaces

program InterfaceExample;

{$APPTYPE CONSOLE}

uses
  SysUtils;

type
  ISpeakable = interface
    ['{8D3E4F5A-6B7C-4D8E-9F1A-2B3C4D5E6F7A}']
    procedure Speak;
  end;

  TDog = class(TInterfacedObject, ISpeakable)
  public
    procedure Speak;
  end;

  TCat = class(TInterfacedObject, ISpeakable)
  public
    procedure Speak;
  end;

procedure TDog.Speak;
begin
  WriteLn('El perro ladra.');
end;

procedure TCat.Speak;
begin
  WriteLn('El gato maúlla.');
end;

var
  Speakable: ISpeakable;

begin
  Speakable := TDog.Create;
  Speakable.Speak; // Salida: El perro ladra.

  Speakable := TCat.Create;
  Speakable.Speak; // Salida: El gato maúlla.

  ReadLn;
end.

En este ejemplo:

  • ISpeakable es una interfaz con un método Speak.
  • TDog y TCat implementan la interfaz ISpeakable.
  • La variable Speakable puede contener referencias a cualquier objeto que implemente ISpeakable, y el método Speak correcto se llama según el tipo real del objeto.

Resumen

El polimorfismo en Delphi/Object Pascal permite un código flexible y reutilizable al permitir que los objetos sean tratados como instancias de su clase padre o interfaz. Los conceptos clave incluyen la herencia, la sobrescritura de métodos y las interfaces. Al comprender y utilizar el polimorfismo, puedes escribir aplicaciones más mantenibles y escalables.

  • Herencia permite que las clases hereden propiedades y métodos de otras clases.
  • Sobrescritura de Métodos permite que las subclases proporcionen implementaciones específicas de métodos definidos en sus superclases.
  • Interfaces definen métodos que deben ser implementados por cualquier clase que implemente la interfaz, permitiendo un tratamiento uniforme de diferentes clases.

Al dominar estos conceptos, puedes aprovechar el poder del polimorfismo para crear aplicaciones más dinámicas y flexibles en Delphi/Object Pascal.
