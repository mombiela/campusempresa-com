Recursión en Delphi/Object Pascal
Arreglos ►

La recursión es un concepto fundamental en la informática y la programación donde una función se llama a sí misma para resolver instancias más pequeñas del mismo problema. Este tema cubrirá los conceptos básicos de la recursión, proporcionará ejemplos y profundizará en usos más avanzados en Delphi/Object Pascal.

¿Qué es la Recursión?

  • Definición: La recursión ocurre cuando una función se llama a sí misma directa o indirectamente para resolver un problema.
  • Caso Base: La condición bajo la cual la función recursiva deja de llamarse a sí misma para evitar bucles infinitos.
  • Caso Recursivo: La parte de la función donde ocurre la recursión, dividiendo el problema en instancias más pequeñas.

Ejemplo Básico: Cálculo del Factorial

El factorial de un entero no negativo n es el producto de todos los enteros positivos menores o iguales a n. Se denota por n!.

Función Factorial en Delphi/Object Pascal

function Factorial(n: Integer): Integer;
begin
  if n = 0 then
    Result := 1  // Caso base: 0! es 1
  else
    Result := n * Factorial(n - 1);  // Caso recursivo
end;
  • Explicación:
    • La función Factorial toma un entero n.
    • Si n es 0, devuelve 1 (caso base).
    • De lo contrario, devuelve n multiplicado por el factorial de n-1 (caso recursivo).

Conceptos Clave

  • Caso Base: Esencial para evitar la recursión infinita.
  • Caso Recursivo: La función se llama a sí misma con un argumento más pequeño o más simple.
  • Desbordamiento de Pila: Demasiadas llamadas recursivas pueden llevar a un error de desbordamiento de pila.

Ejemplo Avanzado: Secuencia de Fibonacci

La secuencia de Fibonacci es una serie de números donde cada número es la suma de los dos anteriores, generalmente comenzando con 0 y 1.

Función Fibonacci en Delphi/Object Pascal

function Fibonacci(n: Integer): Integer;
begin
  if n <= 1 then
    Result := n  // Caso base: Fibonacci(0) = 0, Fibonacci(1) = 1
  else
    Result := Fibonacci(n - 1) + Fibonacci(n - 2);  // Caso recursivo
end;
  • Explicación:
    • La función Fibonacci toma un entero n.
    • Si n es 0 o 1, devuelve n (caso base).
    • De lo contrario, devuelve la suma del Fibonacci de n-1 y n-2 (caso recursivo).

Optimización de la Recursión

Memoización

La memoización es una técnica para optimizar funciones recursivas almacenando los resultados de llamadas a funciones costosas y reutilizándolos cuando ocurren los mismos inputs nuevamente.

Fibonacci Memoizado en Delphi/Object Pascal

var
  Memo: array of Integer;

function FibonacciMemo(n: Integer): Integer;
begin
  if Memo[n] <> -1 then
    Result := Memo[n]
  else
  begin
    if n <= 1 then
      Memo[n] := n
    else
      Memo[n] := FibonacciMemo(n - 1) + FibonacciMemo(n - 2);
    Result := Memo[n];
  end;
end;

procedure InitializeMemoization(MaxN: Integer);
var
  i: Integer;
begin
  SetLength(Memo, MaxN + 1);
  for i := 0 to MaxN do
    Memo[i] := -1;
end;
  • Explicación:
    • Memo es un array utilizado para almacenar valores de Fibonacci previamente calculados.
    • FibonacciMemo verifica si el resultado para n ya está calculado.
    • Si no, calcula el valor y lo almacena en Memo.
    • InitializeMemoization inicializa el array Memo con -1 para indicar valores no calculados.

Ejercicios

  1. Recursión Básica: Escribe una función recursiva para calcular la suma de los primeros n números naturales.
  2. Inversión de Cadenas: Implementa una función recursiva para invertir una cadena.
  3. Torre de Hanoi: Resuelve el problema de la Torre de Hanoi usando recursión.

Resumen

  • La recursión es una herramienta poderosa en la programación, permitiendo que las funciones se llamen a sí mismas para resolver problemas.
  • Entender el caso base y el caso recursivo es crucial para escribir funciones recursivas correctas.
  • Técnicas avanzadas como la memoización pueden optimizar soluciones recursivas, haciéndolas más eficientes.
  • Practicar con varios problemas ayuda a solidificar la comprensión de la recursión en Delphi/Object Pascal.

Dominando la recursión, puedes resolver problemas complejos de manera más elegante y eficiente en Delphi/Object Pascal.
