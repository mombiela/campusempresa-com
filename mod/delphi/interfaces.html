Introducción a las Interfaces

Las interfaces en Delphi/Object Pascal proporcionan una forma de definir métodos que deben ser implementados por cualquier clase que soporte la interfaz. Son similares a las clases abstractas pero son más flexibles y pueden ser usadas para lograr herencia múltiple.

Conceptos Clave

  • Definición: Una interfaz es un tipo de referencia que define un conjunto de métodos sin implementarlos.
  • Implementación: Las clases que implementan una interfaz deben proporcionar implementaciones concretas para todos sus métodos.
  • Herencia Múltiple: Una clase puede implementar múltiples interfaces, permitiendo una forma de herencia múltiple.

Definiendo Interfaces

Para definir una interfaz en Delphi/Object Pascal, usa la palabra clave interface seguida de las firmas de los métodos.

type
  IShape = interface
    ['{8D5B3E1A-2B9D-4B6A-8D3E-1A2B3C4D5E6F}']
    function GetArea: Double;
    function GetPerimeter: Double;
  end;

Explicación

  • GUID: La cadena entre corchetes es un identificador único global (GUID) que identifica de manera única la interfaz.
  • Firmas de Métodos: La interfaz declara los métodos GetArea y GetPerimeter sin proporcionar implementaciones.

Implementando Interfaces

Una clase que implementa una interfaz debe proporcionar implementaciones concretas para todos los métodos declarados en la interfaz.

type
  TCircle = class(TInterfacedObject, IShape)
  private
    FRadius: Double;
  public
    constructor Create(ARadius: Double);
    function GetArea: Double;
    function GetPerimeter: Double;
  end;

constructor TCircle.Create(ARadius: Double);
begin
  FRadius := ARadius;
end;

function TCircle.GetArea: Double;
begin
  Result := Pi * FRadius * FRadius;
end;

function TCircle.GetPerimeter: Double;
begin
  Result := 2 * Pi * FRadius;
end;

Explicación

  • TInterfacedObject: Esta es una clase base que proporciona conteo de referencias y otras características necesarias para la gestión de interfaces.
  • Constructor: Inicializa el radio del círculo.
  • GetArea y GetPerimeter: Proporcionan implementaciones concretas para los métodos declarados en la interfaz IShape.

Usando Interfaces

Una vez que una interfaz es implementada, puedes usarla para interactuar con objetos de una manera polimórfica.

var
  Shape: IShape;
  Circle: TCircle;
begin
  Circle := TCircle.Create(5.0);
  Shape := Circle;
  Writeln('Área: ', Shape.GetArea:0:2);
  Writeln('Perímetro: ', Shape.GetPerimeter:0:2);
end;

Explicación

  • Polimorfismo: La variable Shape de tipo IShape puede contener cualquier objeto que implemente la interfaz IShape.
  • Llamadas a Métodos: Los métodos GetArea y GetPerimeter son llamados en la referencia de la interfaz Shape.

Temas Avanzados

Herencia de Interfaces

Las interfaces pueden heredar de otras interfaces, permitiendo jerarquías más complejas.

type
  IColoredShape = interface(IShape)
    function GetColor: TColor;
  end;

Explicación

  • Herencia: IColoredShape hereda de IShape y añade un método adicional GetColor.

Conteo de Referencias y Gestión de Memoria

Delphi usa el conteo de referencias para gestionar automáticamente la vida útil de las instancias de interfaces.

  • Conteo de Referencias: Cada referencia de interfaz incrementa el conteo de referencias, y cuando el conteo llega a cero, el objeto es destruido automáticamente.
  • TInterfacedObject: Proporciona la implementación necesaria para el conteo de referencias.

Conclusión

Las interfaces en Delphi/Object Pascal son una característica poderosa que permite un código flexible y reutilizable. Permiten polimorfismo, herencia múltiple y una separación clara de las definiciones de métodos de sus implementaciones. Al entender y usar interfaces, puedes crear aplicaciones más modulares y mantenibles.
