Introducción a la POO en Delphi/Object Pascal
Clases y Objetos ►

Resumen

La Programación Orientada a Objetos (POO) es un paradigma que utiliza "objetos" para diseñar aplicaciones y programas informáticos. Delphi, que utiliza el lenguaje Object Pascal, es una herramienta poderosa para implementar conceptos de POO. Esta sección te introducirá a los conceptos básicos de la POO en Delphi, cubriendo conceptos clave, sintaxis y ejemplos prácticos.

Conceptos Clave de la POO

  • Clase: Un plano para crear objetos (una estructura de datos particular), que contiene métodos y propiedades.
  • Objeto: Una instancia de una clase.
  • Encapsulación: La agrupación de datos con los métodos que operan sobre esos datos.
  • Herencia: El mecanismo por el cual una clase puede heredar características de otra clase.
  • Polimorfismo: La capacidad de presentar la misma interfaz para diferentes formas subyacentes (tipos de datos).

Clases y Objetos

Definiendo una Clase

En Delphi, una clase se define usando la palabra clave class. Aquí hay un ejemplo simple:

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
  public
    procedure SetName(AName: string);
    function GetName: string;
    procedure SetAge(AAge: Integer);
    function GetAge: Integer;
  end;

Creando un Objeto

Para crear un objeto, usas el método Create de la clase:

var
  Person: TPerson;
begin
  Person := TPerson.Create;
  Person.SetName('John Doe');
  Person.SetAge(30);
  ShowMessage(Person.GetName + ' tiene ' + IntToStr(Person.GetAge) + ' años.');
  Person.Free;
end;

Encapsulación

La encapsulación se logra utilizando especificadores de acceso como private, protected y public. En el ejemplo anterior, FName y FAge son privados, lo que significa que solo pueden ser accedidos dentro de la clase TPerson.

Herencia

La herencia permite que una clase herede propiedades y métodos de otra clase. Aquí hay un ejemplo:

type
  TEmployee = class(TPerson)
  private
    FEmployeeID: string;
  public
    procedure SetEmployeeID(AEmployeeID: string);
    function GetEmployeeID: string;
  end;

Polimorfismo

El polimorfismo en Delphi se logra típicamente a través de la sobrescritura de métodos y las interfaces. Aquí hay un ejemplo de sobrescritura de métodos:

type
  TPerson = class
  public
    procedure Speak; virtual;
  end;

  TEmployee = class(TPerson)
  public
    procedure Speak; override;
  end;

procedure TPerson.Speak;
begin
  ShowMessage('Hola, soy una persona.');
end;

procedure TEmployee.Speak;
begin
  ShowMessage('Hola, soy un empleado.');
end;

Ejemplo Práctico

Vamos a crear una aplicación simple que demuestre estos conceptos:

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
  public
    procedure SetName(AName: string);
    function GetName: string;
    procedure SetAge(AAge: Integer);
    function GetAge: Integer;
    procedure Speak; virtual;
  end;

  TEmployee = class(TPerson)
  private
    FEmployeeID: string;
  public
    procedure SetEmployeeID(AEmployeeID: string);
    function GetEmployeeID: string;
    procedure Speak; override;
  end;

procedure TPerson.SetName(AName: string);
begin
  FName := AName;
end;

function TPerson.GetName: string;
begin
  Result := FName;
end;

procedure TPerson.SetAge(AAge: Integer);
begin
  FAge := AAge;
end;

function TPerson.GetAge: Integer;
begin
  Result := FAge;
end;

procedure TPerson.Speak;
begin
  ShowMessage('Hola, soy una persona.');
end;

procedure TEmployee.SetEmployeeID(AEmployeeID: string);
begin
  FEmployeeID := AEmployeeID;
end;

function TEmployee.GetEmployeeID: string;
begin
  Result := FEmployeeID;
end;

procedure TEmployee.Speak;
begin
  ShowMessage('Hola, soy un empleado.');
end;

var
  Person: TPerson;
  Employee: TEmployee;
begin
  Person := TPerson.Create;
  Person.SetName('John Doe');
  Person.SetAge(30);
  Person.Speak;
  Person.Free;

  Employee := TEmployee.Create;
  Employee.SetName('Jane Smith');
  Employee.SetAge(28);
  Employee.SetEmployeeID('E12345');
  Employee.Speak;
  Employee.Free;
end;

Conclusión

En esta sección, introdujimos los conceptos fundamentales de la Programación Orientada a Objetos en Delphi/Object Pascal. Cubrimos:

  • La definición y creación de clases y objetos.
  • Encapsulación, herencia y polimorfismo.
  • Ejemplos prácticos para ilustrar estos conceptos.

Entender estos conceptos básicos proporcionará una base sólida para temas más avanzados en la programación con Delphi. A medida que progreses, verás cómo se aplican estos principios en aplicaciones del mundo real, haciendo tu código más modular, reutilizable y fácil de mantener.
