Definición y Llamada de Funciones en Delphi/Object Pascal
Paso de Parámetros ►

Introducción

En Delphi/Object Pascal, las funciones son bloques de construcción fundamentales que te permiten encapsular código en unidades reutilizables. Este tema te guiará a través de la definición y llamada de funciones, comenzando desde lo básico y avanzando hacia escenarios más complejos.

Definición Básica de Funciones y Llamadas

Definiendo una Función Simple

En Delphi/Object Pascal, una función se define usando la palabra clave function. Aquí tienes un ejemplo simple:

function AddNumbers(a, b: Integer): Integer;
begin
  Result := a + b;
end;
  • Palabra clave function: Indica el inicio de una definición de función.
  • AddNumbers: El nombre de la función.
  • (a, b: Integer): Parámetros de la función, ambos de tipo Integer.
  • : Integer: El tipo de retorno de la función.
  • begin...end: El bloque donde se implementa la lógica de la función.
  • Result: Un identificador especial usado para almacenar el valor de retorno de la función.

Llamando a una Función

Para llamar a una función, simplemente usa su nombre seguido de los argumentos entre paréntesis:

var
  sum: Integer;
begin
  sum := AddNumbers(5, 3);
  WriteLn('The sum is: ', sum);
end;
  • sum := AddNumbers(5, 3);: Llama a la función AddNumbers con 5 y 3 como argumentos y almacena el resultado en sum.

Funciones con Diferentes Tipos de Parámetros

Pasando Parámetros por Valor

Por defecto, los parámetros en Delphi/Object Pascal se pasan por valor, lo que significa que la función recibe una copia de los argumentos.

function Multiply(a, b: Integer): Integer;
begin
  Result := a * b;
end;

Pasando Parámetros por Referencia

Para permitir que una función modifique los argumentos, puedes pasar parámetros por referencia usando la palabra clave var:

procedure Swap(var x, y: Integer);
var
  temp: Integer;
begin
  temp := x;
  x := y;
  y := temp;
end;
  • procedure: Usado en lugar de function cuando no se necesita un valor de retorno.
  • var x, y: Integer: Parámetros pasados por referencia.

Parámetros Predeterminados

Puedes proporcionar valores predeterminados para los parámetros, haciéndolos opcionales al llamar a la función:

function Greet(name: String = 'Guest'): String;
begin
  Result := 'Hello, ' + name + '!';
end;
  • name: String = 'Guest': name tiene un valor predeterminado de 'Guest'.

Características Avanzadas de las Funciones

Sobrecarga de Funciones

Delphi/Object Pascal soporta la sobrecarga de funciones, permitiendo múltiples funciones con el mismo nombre pero diferentes parámetros:

function Max(a, b: Integer): Integer; overload;
begin
  if a > b then
    Result := a
  else
    Result := b;
end;

function Max(a, b, c: Integer): Integer; overload;
begin
  Result := Max(Max(a, b), c);
end;
  • overload: Indica que la función está sobrecargada.

Funciones Recursivas

Una función puede llamarse a sí misma, lo que se conoce como recursión. Esto es útil para problemas que pueden descomponerse en problemas más pequeños y similares:

function Factorial(n: Integer): Integer;
begin
  if n = 0 then
    Result := 1
  else
    Result := n * Factorial(n - 1);
end;
  • Factorial(n - 1): La función se llama a sí misma con un valor decrementado de n.

Ejercicios

  1. Creación Básica de Funciones: Define una función SubtractNumbers que tome dos enteros y devuelva su diferencia. Llama a esta función y muestra el resultado.
  2. Parámetros por Referencia: Escribe un procedimiento Increment que tome un entero por referencia y aumente su valor en 1. Prueba este procedimiento.
  3. Parámetros Predeterminados: Crea una función WelcomeMessage que tome un parámetro de cadena con un valor predeterminado de 'User' y devuelva un mensaje de bienvenida. Llama a esta función con y sin un argumento.
  4. Sobrecarga de Funciones: Implementa funciones sobrecargadas Min que encuentren el mínimo de dos y tres enteros respectivamente. Prueba estas funciones.
  5. Función Recursiva: Escribe una función recursiva Fibonacci que devuelva el enésimo número de Fibonacci. Prueba esta función con diferentes valores de n.

Conclusión

Entender cómo definir y llamar funciones en Delphi/Object Pascal es crucial para escribir código modular y mantenible. Las funciones te ayudan a encapsular la lógica, haciendo que tus programas sean más fáciles de entender y depurar. Al dominar las características básicas y avanzadas de las funciones, puedes escribir código más eficiente y flexible.
