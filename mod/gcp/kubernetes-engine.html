Introducción a Kubernetes Engine

Google Kubernetes Engine (GKE) es un entorno gestionado y listo para producción para desplegar aplicaciones en contenedores. Proporciona una forma poderosa y flexible de gestionar, desplegar y escalar aplicaciones utilizando Kubernetes.

Conceptos Clave

  • Kubernetes: Un sistema de código abierto para automatizar el despliegue, escalado y gestión de aplicaciones en contenedores.
  • GKE: Un servicio gestionado de Kubernetes proporcionado por Google Cloud Platform.
  • Contenedores: Paquetes de software ligeros, autónomos y ejecutables que incluyen todo lo necesario para ejecutar una pieza de software, incluyendo el código, tiempo de ejecución, bibliotecas y dependencias.

Configuración de GKE

Prerrequisitos

  • Una cuenta de Google Cloud Platform.
  • Conocimiento básico de Docker y Kubernetes.
  • Google Cloud SDK instalado en tu máquina local.

Pasos para Configurar GKE

  1. Crear un Proyecto GCP:

    gcloud projects create my-gke-project
gcloud config set project my-gke-project

  2. Habilitar la API de Kubernetes Engine:

    gcloud services enable container.googleapis.com

  3. Crear un Clúster GKE:

    gcloud container clusters create my-cluster --zone us-central1-a

  4. Obtener Credenciales de Autenticación para el Clúster:

    gcloud container clusters get-credentials my-cluster --zone us-central1-a

  5. Verificar el Clúster:

    kubectl get nodes

Desplegando Aplicaciones en GKE

Creando un Despliegue

  • Archivo YAML de Despliegue:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: gcr.io/my-gke-project/my-app:latest
        ports:
        - containerPort: 80

  • Aplicar el Despliegue:

    kubectl apply -f deployment.yaml

Exponiendo el Despliegue

  • Archivo YAML de Servicio:

    apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  type: LoadBalancer
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

  • Aplicar el Servicio:

    kubectl apply -f service.yaml

Escalando Aplicaciones

Escalado Manual

  • Escalar el Despliegue: 
    kubectl scale deployment my-app --replicas=5

Autoescalado

  • Habilitar Autoescalado: 
    kubectl autoscale deployment my-app --cpu-percent=80 --min=1 --max=10

Monitoreo y Registro

Integración con Stackdriver

  • Habilitar Stackdriver Logging y Monitoring:

    gcloud container clusters create my-cluster --zone us-central1-a --enable-stackdriver-kubernetes

  • Ver Registros:

    gcloud logging read "resource.type=k8s_container AND resource.labels.cluster_name=my-cluster"

Temas Avanzados

Definiciones de Recursos Personalizados (CRDs)

  • Definir un CRD:

    apiVersion: apiextensions.k8s.io/v1
kind: CustomResourceDefinition
metadata:
  name: crontabs.stable.example.com
spec:
  group: stable.example.com
  versions:
  - name: v1
    served: true
    storage: true
  scope: Namespaced
  names:
    plural: crontabs
    singular: crontab
    kind: CronTab
    shortNames:
    - ct

  • Aplicar el CRD:

    kubectl apply -f crd.yaml

Políticas de Red

  • Definir una Política de Red:

    apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-same-namespace
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector: {}

  • Aplicar la Política de Red:

    kubectl apply -f network-policy.yaml

Conclusión

Google Kubernetes Engine (GKE) simplifica el despliegue, gestión y escalado de aplicaciones en contenedores utilizando Kubernetes. Al aprovechar GKE, los desarrolladores pueden centrarse en escribir código y desplegar aplicaciones sin preocuparse por la infraestructura subyacente. Este curso cubrió los conceptos básicos de la configuración de GKE, el despliegue de aplicaciones, el escalado, el monitoreo y temas avanzados como CRDs y políticas de red. Con este conocimiento, estás bien equipado para gestionar y desplegar aplicaciones en GKE de manera efectiva.
