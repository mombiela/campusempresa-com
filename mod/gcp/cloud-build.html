Introducción a Cloud Build

Google Cloud Build es una plataforma de integración continua y entrega continua (CI/CD) que automatiza el proceso de construcción, prueba y despliegue de aplicaciones. Soporta una amplia gama de lenguajes y entornos, lo que lo convierte en una herramienta versátil para los desarrolladores.

Conceptos Clave

  • Disparadores de Construcción: Inician automáticamente las construcciones basadas en eventos como commits de código.
  • Pasos de Construcción: Definen la secuencia de operaciones en un proceso de construcción.
  • Artefactos de Construcción: Salidas generadas por el proceso de construcción, como binarios o imágenes de Docker.
  • Configuración de Construcción: Archivos YAML o JSON que especifican los pasos de construcción y el entorno.

Configuración de Cloud Build

Para comenzar con Cloud Build, necesitas configurar tu proyecto de Google Cloud y habilitar la API de Cloud Build.

Pasos para Configurar

  1. Crear un Proyecto de Google Cloud:

    • Ve a la Google Cloud Console.
    • Haz clic en el menú desplegable de proyectos y selecciona "Nuevo Proyecto".
    • Ingresa un nombre de proyecto y haz clic en "Crear".

  2. Habilitar la API de Cloud Build:

    • Navega a la Biblioteca de API.
    • Busca "Cloud Build API" y haz clic en ella.
    • Haz clic en "Habilitar".

  3. Instalar Cloud SDK:

    • Sigue las instrucciones en la página de instalación de Google Cloud SDK.

  4. Autenticarse con Google Cloud:

    gcloud auth login
gcloud config set project [PROJECT_ID]

Creando una Construcción Simple

Vamos a crear una configuración de construcción simple para entender los conceptos básicos.

Ejemplo: Construyendo una Imagen de Docker

  1. Crear un archivo cloudbuild.yaml:

    steps:
- name: 'gcr.io/cloud-builders/docker'
  args: ['build', '-t', 'gcr.io/[PROJECT_ID]/my-app', '.']
images:
- 'gcr.io/[PROJECT_ID]/my-app'

  2. Ejecutar la Construcción:

    gcloud builds submit --config cloudbuild.yaml .

Explicación

  • steps: Define las acciones a tomar. Aquí, usamos el constructor de Docker para construir una imagen.
  • name: Especifica la imagen del constructor.
  • args: Argumentos pasados al constructor. En este caso, estamos construyendo una imagen de Docker.
  • images: Especifica las imágenes que se subirán al Registro de Contenedores.

Configuraciones Avanzadas de Construcción

A medida que te sientas más cómodo con Cloud Build, puedes crear configuraciones de construcción más complejas.

Ejemplo: Construcción Multi-Paso

steps:
- name: 'gcr.io/cloud-builders/npm'
  args: ['install']
- name: 'gcr.io/cloud-builders/npm'
  args: ['test']
- name: 'gcr.io/cloud-builders/docker'
  args: ['build', '-t', 'gcr.io/[PROJECT_ID]/my-app', '.']
images:
- 'gcr.io/[PROJECT_ID]/my-app'

Explicación

  • npm install: Instala dependencias.
  • npm test: Ejecuta pruebas.
  • docker build: Construye la imagen de Docker.

Disparadores de Construcción

Los disparadores de construcción te permiten automatizar las construcciones basadas en eventos como commits de código.

Creando un Disparador de Construcción

  1. Navegar a los Disparadores de Cloud Build:

    • Ve a la página de Disparadores de Cloud Build.

  2. Crear un Disparador:

    • Haz clic en "Crear Disparador".
    • Selecciona tu repositorio y rama.
    • Especifica el archivo de configuración de construcción (cloudbuild.yaml).

Ejemplo: Configuración de Disparador

triggerTemplate:
  projectId: [PROJECT_ID]
  repoName: [REPO_NAME]
  branchName: 'main'

Conclusión

Cloud Build es una herramienta poderosa para automatizar los procesos de construcción, prueba y despliegue. Al entender los conceptos básicos y explorar gradualmente configuraciones avanzadas, puedes optimizar tu flujo de trabajo de desarrollo y asegurar construcciones consistentes y confiables. Comienza con configuraciones de construcción simples y progresivamente incorpora pasos y disparadores más complejos para aprovechar al máximo las capacidades de Cloud Build.
