Firestore es una base de datos flexible y escalable para el desarrollo de aplicaciones móviles, web y de servidor de Firebase y Google Cloud Platform (GCP). Es una base de datos de documentos NoSQL que te permite almacenar, sincronizar y consultar datos para tus aplicaciones.

Introducción a Firestore

  • ¿Qué es Firestore?

    • Firestore es una base de datos de documentos NoSQL.
    • Es parte de la plataforma Firebase e integra perfectamente con otros servicios de Firebase y GCP.
    • Firestore está diseñado para manejar datos a gran escala y actualizaciones en tiempo real.

  • Características Clave:

    • Sincronización en tiempo real.
    • Soporte sin conexión.
    • Consistencia fuerte.
    • Modelo de datos flexible.
    • Capacidades de consulta poderosas.

Configuración de Firestore

  • Requisitos Previos:

    • Una cuenta de Google Cloud Platform.
    • Configuración del proyecto Firebase.

  • Pasos para Configurar Firestore:

    1. Crear un Proyecto Firebase:
      • Ve a la Consola de Firebase.
      • Haz clic en "Agregar proyecto" y sigue las instrucciones de configuración.
    2. Habilitar Firestore:
      • En la Consola de Firebase, navega a la sección de Base de Datos.
      • Selecciona Firestore y haz clic en "Crear base de datos".
      • Elige una ubicación para tu base de datos y configura las reglas de seguridad.

Modelo de Datos de Firestore

  • Documentos y Colecciones:

    • Firestore almacena datos en documentos, que están organizados en colecciones.
    • Un documento es un conjunto de pares clave-valor.
    • Las colecciones son contenedores para documentos y pueden contener subcolecciones.

  • Estructura de Ejemplo:

    - users (colección)
  - userID (documento)
    - name: "John Doe"
    - email: "john.doe@example.com"
    - posts (subcolección)
      - postID (documento)
        - title: "My First Post"
        - content: "Hello, world!"

Operaciones Básicas

  • Agregar Datos:

    // Inicializar Firestore
const db = firebase.firestore();

// Agregar un nuevo documento en la colección "users"
db.collection("users").add({
    name: "John Doe",
    email: "john.doe@example.com"
})
.then((docRef) => {
    console.log("Documento escrito con ID: ", docRef.id);
})
.catch((error) => {
    console.error("Error al agregar el documento: ", error);
});

  • Leer Datos:

    // Obtener un documento de la colección "users"
db.collection("users").doc("userID").get().then((doc) => {
    if (doc.exists) {
        console.log("Datos del documento:", doc.data());
    } else {
        console.log("¡No existe tal documento!");
    }
}).catch((error) => {
    console.log("Error al obtener el documento:", error);
});

  • Actualizar Datos:

    // Actualizar un documento existente en la colección "users"
db.collection("users").doc("userID").update({
    email: "new.email@example.com"
})
.then(() => {
    console.log("¡Documento actualizado exitosamente!");
})
.catch((error) => {
    console.error("Error al actualizar el documento: ", error);
});

  • Eliminar Datos:

    // Eliminar un documento de la colección "users"
db.collection("users").doc("userID").delete().then(() => {
    console.log("¡Documento eliminado exitosamente!");
}).catch((error) => {
    console.error("Error al eliminar el documento: ", error);
});

Características Avanzadas

  • Actualizaciones en Tiempo Real:

    // Escuchar actualizaciones en tiempo real de un documento
db.collection("users").doc("userID")
  .onSnapshot((doc) => {
      console.log("Datos actuales: ", doc.data());
  });

  • Consultas:

    // Consultar documentos en la colección "users" donde el nombre es "John Doe"
db.collection("users").where("name", "==", "John Doe")
  .get()
  .then((querySnapshot) => {
      querySnapshot.forEach((doc) => {
          console.log(doc.id, " => ", doc.data());
      });
  })
  .catch((error) => {
      console.log("Error al obtener los documentos: ", error);
  });

  • Transacciones:

    // Realizar una transacción para actualizar un documento
db.runTransaction((transaction) => {
    return transaction.get(db.collection("users").doc("userID")).then((doc) => {
        if (!doc.exists) {
            throw "¡El documento no existe!";
        }

        // Actualizar el campo de correo electrónico
        transaction.update(db.collection("users").doc("userID"), { email: "new.email@example.com" });
    });
}).then(() => {
    console.log("¡Transacción comprometida exitosamente!");
}).catch((error) => {
    console.log("Transacción fallida: ", error);
});

Reglas de Seguridad

  • Configurar Reglas de Seguridad:

    • Las reglas de seguridad de Firestore controlan el acceso a tu base de datos.
    • Puedes definir reglas para permitir o denegar el acceso basado en las propiedades de la solicitud.

  • Ejemplo de Regla de Seguridad:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Conclusión

Firestore es una poderosa base de datos NoSQL que ofrece sincronización en tiempo real, soporte sin conexión y un modelo de datos flexible. Al entender sus operaciones básicas, características avanzadas y reglas de seguridad, puedes usar Firestore de manera efectiva para construir aplicaciones escalables y responsivas. Ya sea que estés trabajando en un proyecto pequeño o en una aplicación a gran escala, Firestore proporciona las herramientas que necesitas para gestionar tus datos eficientemente.
