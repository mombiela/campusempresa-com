Introducción a IAM

La Gestión de Identidades y Accesos (IAM) es un componente crítico de Google Cloud Platform (GCP) que te permite gestionar el acceso a los recursos definiendo quién (identidad) tiene qué acceso (rol) a qué recursos.

Conceptos Clave

  • Identidad: Representa a usuarios, grupos o servicios que necesitan acceso a los recursos de GCP.
  • Rol: Una colección de permisos que pueden ser asignados a identidades.
  • Permiso: Define qué acciones pueden realizarse en un recurso.
  • Política: Una colección de enlaces que asocian roles con identidades.

Conceptos Básicos de IAM

Comprendiendo las Identidades

  • Cuentas de Google: Usuarios individuales con una cuenta de Google.
  • Cuentas de Servicio: Cuentas especiales utilizadas por aplicaciones o VMs para realizar llamadas API autorizadas.
  • Grupos de Google: Colecciones de cuentas de Google.
  • Dominios de Identidad en la Nube: Dominios gestionados para organizaciones.

Comprendiendo los Roles

  • Roles Primitivos: Roles básicos que incluyen Propietario, Editor y Visor.
  • Roles Predefinidos: Roles granulares creados por Google que proporcionan permisos específicos.
  • Roles Personalizados: Roles definidos por el usuario que proporcionan control detallado sobre los permisos.

Asignando Roles a Identidades

Para asignar roles a identidades, creas un enlace de política IAM. Aquí tienes un ejemplo de cómo asignar el rol de Visor a un usuario:

bindings:
- members:
  - user:example-user@gmail.com
  role: roles/viewer

Conceptos Intermedios de IAM

Gestión de Cuentas de Servicio

Las cuentas de servicio son cruciales para las aplicaciones que se ejecutan en GCP. Aquí tienes cómo crearlas y gestionarlas:

Creando una Cuenta de Servicio

gcloud iam service-accounts create my-service-account \
    --display-name "My Service Account"

Asignando Roles a una Cuenta de Servicio

gcloud projects add-iam-policy-binding my-project \
    --member "serviceAccount:my-service-account@my-project.iam.gserviceaccount.com" \
    --role "roles/editor"

Uso de Políticas IAM

Las políticas IAM definen quién tiene qué tipo de acceso. Aquí tienes un ejemplo de una política IAM:

{
  "bindings": [
    {
      "role": "roles/viewer",
      "members": [
        "user:example-user@gmail.com"
      ]
    },
    {
      "role": "roles/editor",
      "members": [
        "serviceAccount:my-service-account@my-project.iam.gserviceaccount.com"
      ]
    }
  ]
}

Conceptos Avanzados de IAM

Roles Personalizados

Los roles personalizados te permiten definir permisos específicos adaptados a tus necesidades.

Creando un Rol Personalizado

gcloud iam roles create myCustomRole \
    --project my-project \
    --title "My Custom Role" \
    --permissions compute.instances.start,compute.instances.stop \
    --stage GA

Auditoría y Monitoreo de IAM

Monitorear las actividades de IAM es esencial para la seguridad y el cumplimiento.

Habilitando Registros de Auditoría

gcloud logging sinks create my-audit-logs \
    storage.googleapis.com/my-audit-logs-bucket \
    --log-filter='resource.type="gce_instance"'

Mejores Prácticas

  • Principio de Menor Privilegio: Otorga solo los permisos necesarios para que un usuario realice su trabajo.
  • Auditorías Regulares: Revisa regularmente las políticas y roles de IAM.
  • Usa Grupos: Gestiona el acceso asignando roles a grupos en lugar de a usuarios individuales.

Conclusión

IAM en GCP es una herramienta poderosa para gestionar el acceso a tus recursos en la nube. Al comprender y utilizar identidades, roles y políticas, puedes asegurar que tus recursos estén seguros y que los usuarios tengan el nivel adecuado de acceso. Recuerda seguir las mejores prácticas como el principio de menor privilegio y las auditorías regulares para mantener un entorno seguro.
