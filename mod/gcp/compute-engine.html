Compute Engine
App Engine ►

Compute Engine de Google Cloud Platform es una potente Infraestructura como Servicio (IaaS) que te permite ejecutar máquinas virtuales en la infraestructura de Google. Este tema te guiará desde los conceptos básicos hasta los avanzados de Compute Engine, ayudándote a entender cómo aprovecharlo para tus aplicaciones.

Introducción a Compute Engine

  • ¿Qué es Compute Engine?

    • Compute Engine es un servicio que proporciona máquinas virtuales (VMs) escalables y de alto rendimiento en Google Cloud Platform.
    • Te permite ejecutar cargas de trabajo a gran escala en máquinas virtuales alojadas en la infraestructura de Google.

  • Características Clave

    • VMs personalizables
    • Tipos de máquinas predefinidos
    • Discos persistentes
    • Balanceo de carga global
    • Escalado automático
    • Integración con otros servicios de GCP

Empezando con Compute Engine

  • Configurando tu Primera Instancia de VM
    • Paso 1: Crear un Proyecto
      • Ve a la Consola de GCP.
      • Crea un nuevo proyecto o selecciona uno existente.
    • Paso 2: Habilitar la API de Compute Engine
      • Navega al panel de API y Servicios.
      • Habilita la API de Compute Engine.
    • Paso 3: Crear una Instancia de VM
      • Ve a la sección de Compute Engine.
      • Haz clic en "Crear Instancia".
      • Configura la instancia (nombre, región, zona, tipo de máquina, disco de arranque, etc.).
      • Haz clic en "Crear".
# Ejemplo: Creando una instancia de VM usando la herramienta de línea de comandos gcloud
gcloud compute instances create my-instance \
    --zone=us-central1-a \
    --machine-type=e2-medium \
    --image-family=debian-10 \
    --image-project=debian-cloud
  • Conectándote a tu Instancia de VM
    • Usa SSH para conectarte a tu instancia de VM desde la Consola de GCP.
    • Alternativamente, usa la herramienta de línea de comandos gcloud:
# Ejemplo: Conectándose a una instancia de VM usando gcloud
gcloud compute ssh my-instance --zone=us-central1-a

Gestionando Instancias de VM

  • Ciclo de Vida de la Instancia
    • Iniciando y Deteniendo Instancias
      • Inicia, detén y reinicia instancias desde la Consola de GCP o usando comandos gcloud.
# Ejemplo: Iniciando una instancia de VM
gcloud compute instances start my-instance --zone=us-central1-a

# Ejemplo: Deteniendo una instancia de VM
gcloud compute instances stop my-instance --zone=us-central1-a
  • Eliminando Instancias
    • Elimina instancias cuando ya no sean necesarias para evitar incurrir en costos.
# Ejemplo: Eliminando una instancia de VM
gcloud compute instances delete my-instance --zone=us-central1-a
  • Plantillas y Grupos de Instancias
    • Plantillas de Instancias
      • Define una plantilla para instancias de VM para asegurar la consistencia.
    • Grupos de Instancias Administrados
      • Usa grupos de instancias administrados para autoescalado y balanceo de carga.
# Ejemplo: Creando una plantilla de instancia
gcloud compute instance-templates create my-template \
    --machine-type=e2-medium \
    --image-family=debian-10 \
    --image-project=debian-cloud

# Ejemplo: Creando un grupo de instancias administrado
gcloud compute instance-groups managed create my-group \
    --base-instance-name=my-instance \
    --template=my-template \
    --size=3 \
    --zone=us-central1-a

Características Avanzadas de Compute Engine

  • Tipos de Máquinas Personalizadas
    • Crea tipos de máquinas personalizadas para optimizar costos y rendimiento para cargas de trabajo específicas.
# Ejemplo: Creando una instancia de VM con un tipo de máquina personalizada
gcloud compute instances create custom-instance \
    --custom-cpu=4 \
    --custom-memory=10GB \
    --zone=us-central1-a \
    --image-family=debian-10 \
    --image-project=debian-cloud
  • Discos Persistentes
    • Adjunta y gestiona discos persistentes para almacenar datos más allá del ciclo de vida de una instancia de VM.
# Ejemplo: Creando y adjuntando un disco persistente
gcloud compute disks create my-disk --size=200GB --zone=us-central1-a
gcloud compute instances attach-disk my-instance --disk=my-disk --zone=us-central1-a
  • Redes
    • Configura VPCs, subredes y reglas de firewall para controlar el tráfico de red hacia y desde tus instancias de VM.
# Ejemplo: Creando una regla de firewall para permitir tráfico SSH
gcloud compute firewall-rules create allow-ssh \
    --allow=tcp:22 \
    --network=default
  • Balanceo de Carga
    • Distribuye el tráfico entre múltiples instancias de VM usando balanceo de carga global.
# Ejemplo: Creando un balanceador de carga HTTP global
gcloud compute backend-services create my-backend-service \
    --protocol=HTTP \
    --global

gcloud compute url-maps create my-url-map \
    --default-service=my-backend-service

gcloud compute target-http-proxies create my-http-proxy \
    --url-map=my-url-map

gcloud compute forwarding-rules create my-http-rule \
    --global \
    --target-http-proxy=my-http-proxy \
    --ports=80

Mejores Prácticas

  • Seguridad

    • Usa roles de IAM y cuentas de servicio para controlar el acceso.
    • Habilita OS Login para la gestión del acceso SSH.
    • Actualiza y parchea regularmente tus instancias de VM.

  • Gestión de Costos

    • Usa descuentos por uso sostenido y contratos de uso comprometido.
    • Monitorea y optimiza el uso de recursos.

  • Optimización del Rendimiento

    • Usa tipos de máquinas personalizadas para cargas de trabajo específicas.
    • Aprovecha el autoescalado para cargas de trabajo dinámicas.

Conclusión

Compute Engine de Google Cloud Platform ofrece un entorno robusto y flexible para ejecutar máquinas virtuales. Desde crear tu primera instancia de VM hasta gestionar cargas de trabajo complejas con tipos de máquinas personalizadas, discos persistentes y balanceo de carga, Compute Engine proporciona las herramientas necesarias para construir aplicaciones escalables y eficientes. Siguiendo las mejores prácticas para seguridad, gestión de costos y optimización del rendimiento, puedes maximizar los beneficios de usar Compute Engine en tus proyectos.
