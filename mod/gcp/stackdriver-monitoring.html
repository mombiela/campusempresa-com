Introducción a Stackdriver Monitoring

Stackdriver Monitoring, ahora conocido como Google Cloud Monitoring, es una herramienta poderosa proporcionada por Google Cloud Platform (GCP) para monitoreo, registro y diagnóstico. Te ayuda a obtener información sobre el rendimiento, tiempo de actividad y salud general de tus aplicaciones e infraestructura.

Conceptos Clave

  • Métricas: Mediciones cuantitativas de tus recursos y aplicaciones.
  • Tableros: Representaciones visuales de métricas.
  • Alertas: Notificaciones desencadenadas por condiciones específicas en tus métricas.
  • Verificaciones de Tiempo de Actividad: Verificaciones regulares para asegurar que tus servicios estén disponibles y sean receptivos.
  • Grupos: Colecciones de recursos monitoreados.

Configuración de Stackdriver Monitoring

Para empezar a usar Stackdriver Monitoring, necesitas configurarlo en tu proyecto de GCP.

Pasos para Configurar

  1. Habilitar la API de Monitoreo:

    • Ve a la Consola de GCP.
    • Navega al panel de API y Servicios.
    • Habilita la "API de Cloud Monitoring".

  2. Crear un Espacio de Trabajo:

    • Ve a la sección de Monitoreo en la Consola de GCP.
    • Crea un nuevo espacio de trabajo o selecciona uno existente.
    • Vincula tu proyecto de GCP al espacio de trabajo.

  3. Instalar el Agente de Monitoreo (para VMs):

    • SSH en tu instancia de VM.
    • Ejecuta los siguientes comandos para instalar el agente: 
      curl -sSO https://dl.google.com/cloudagents/add-monitoring-agent-repo.sh
sudo bash add-monitoring-agent-repo.sh
sudo apt-get update
sudo apt-get install stackdriver-agent
sudo service stackdriver-agent start

Monitoreo de Métricas

Las métricas son el núcleo de Stackdriver Monitoring. Proporcionan los datos que visualizas y analizas.

Métricas Comunes

  • Utilización de CPU: Mide el porcentaje de CPU utilizado.
  • Uso de Memoria: Mide la cantidad de memoria utilizada.
  • I/O de Disco: Mide las operaciones de entrada/salida en el disco.
  • Tráfico de Red: Mide la cantidad de datos enviados y recibidos a través de la red.

Visualización de Métricas

  1. Navegar a Metrics Explorer:

    • Ve a la sección de Monitoreo en la Consola de GCP.
    • Selecciona "Metrics Explorer".

  2. Seleccionar una Métrica:

    • Elige el tipo de recurso (por ejemplo, Instancia de VM).
    • Selecciona la métrica específica que deseas ver (por ejemplo, Utilización de CPU).

  3. Personalizar la Vista:

    • Usa filtros para reducir los datos.
    • Ajusta el rango de tiempo para ver datos históricos.

Creación de Tableros

Los tableros te permiten visualizar métricas de una manera personalizable e interactiva.

Pasos para Crear un Tablero

  1. Navegar a Tableros:

    • Ve a la sección de Monitoreo en la Consola de GCP.
    • Selecciona "Tableros".

  2. Crear un Nuevo Tablero:

    • Haz clic en "Crear Tablero".
    • Dale un nombre a tu tablero.

  3. Agregar Widgets:

    • Haz clic en "Agregar Gráfico".
    • Selecciona la métrica y el tipo de recurso.
    • Personaliza el tipo de gráfico (por ejemplo, gráfico de líneas, gráfico de barras).
    • Guarda el widget en el tablero.

Configuración de Alertas

Las alertas te notifican cuando se cumplen condiciones específicas, ayudándote a responder a problemas de manera rápida.

Pasos para Configurar Alertas

  1. Navegar a Alertas:

    • Ve a la sección de Monitoreo en la Consola de GCP.
    • Selecciona "Alertas".

  2. Crear una Política de Alerta:

    • Haz clic en "Crear Política".
    • Define las condiciones (por ejemplo, Utilización de CPU > 80%).
    • Configura los canales de notificación (por ejemplo, correo electrónico, SMS).

  3. Guardar la Política:

    • Revisa la configuración de la política.
    • Guarda la política para activarla.

Verificaciones de Tiempo de Actividad

Las verificaciones de tiempo de actividad aseguran que tus servicios estén disponibles y sean receptivos desde múltiples ubicaciones.

Pasos para Configurar Verificaciones de Tiempo de Actividad

  1. Navegar a Verificaciones de Tiempo de Actividad:

    • Ve a la sección de Monitoreo en la Consola de GCP.
    • Selecciona "Verificaciones de Tiempo de Actividad".

  2. Crear una Verificación de Tiempo de Actividad:

    • Haz clic en "Crear Verificación de Tiempo de Actividad".
    • Define los parámetros de la verificación (por ejemplo, URL, frecuencia).
    • Configura las ubicaciones desde las cuales se realizarán las verificaciones.

  3. Guardar la Verificación de Tiempo de Actividad:

    • Revisa la configuración.
    • Guarda la verificación de tiempo de actividad para comenzar a monitorear.

Conclusión

Stackdriver Monitoring es una herramienta esencial para mantener la salud y el rendimiento de tus aplicaciones e infraestructura en GCP. Al comprender y utilizar métricas, tableros, alertas y verificaciones de tiempo de actividad, puedes asegurar que tus servicios funcionen de manera fluida y eficiente. Comienza configurando lo básico y explora gradualmente las funciones avanzadas para aprovechar al máximo el poder de Stackdriver Monitoring.
