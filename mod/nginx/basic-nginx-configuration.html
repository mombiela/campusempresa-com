Introducción a NGINX

NGINX es un servidor HTTP de alto rendimiento, proxy inverso y servidor proxy IMAP/POP3. Es conocido por su estabilidad, conjunto de características ricas, configuración simple y bajo consumo de recursos.

Conceptos Clave

  • Bloque de Servidor: Define un servidor virtual utilizado para manejar solicitudes.
  • Bloque de Ubicación: Define cómo procesar conjuntos específicos de solicitudes.
  • Directivas: Instrucciones que controlan el comportamiento de NGINX.

Instalando NGINX

Antes de configurar NGINX, necesitas instalarlo. Aquí se explica cómo instalar NGINX en un sistema basado en Debian:

sudo apt update
sudo apt install nginx

Estructura Básica del Archivo de Configuración

El archivo de configuración principal de NGINX se encuentra típicamente en /etc/nginx/nginx.conf. Aquí tienes una estructura básica:

user www-data;
worker_processes auto;
pid /run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    include /etc/nginx/conf.d/*.conf;
    include /etc/nginx/sites-enabled/*;
}

Explicación:

  • user: Define el usuario y grupo bajo los cuales se ejecutarán los procesos de trabajo de NGINX.
  • worker_processes: Especifica el número de procesos de trabajo.
  • events: Contiene directivas que afectan el procesamiento de conexiones.
  • http: Contexto principal para la configuración HTTP.

Configurando un Bloque de Servidor

Un bloque de servidor define un servidor virtual. Aquí tienes un ejemplo de configuración para un bloque de servidor básico:

server {
    listen 80;
    server_name example.com www.example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }

    error_page 404 /404.html;
    location = /404.html {
        internal;
    }
}

Explicación:

  • listen: Especifica el puerto en el que NGINX escuchará.
  • server_name: Define los nombres de dominio a los que responderá este bloque de servidor.
  • location: Define cómo procesar las solicitudes para la URL raíz (/).
  • root: Especifica el directorio raíz para el servidor.
  • index: Define los archivos de índice que se buscarán cuando se solicite un directorio.
  • error_page: Personaliza la página de error para un código de estado HTTP específico.

Configurando un Bloque de Ubicación

Los bloques de ubicación te permiten definir cómo NGINX debe procesar conjuntos específicos de solicitudes. Aquí tienes un ejemplo:

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }

    location /images/ {
        root /var/www/media;
    }

    location /api/ {
        proxy_pass http://localhost:3000;
    }
}

Explicación:

  • location /images/: Las solicitudes a /images/ se servirán desde /var/www/media.
  • location /api/: Las solicitudes a /api/ se enviarán a http://localhost:3000.

Directivas Comunes

Aquí tienes algunas directivas comunes que podrías usar en tu configuración de NGINX:

| Directiva | Descripción | |-----------------|--------------------------------------------------| | listen | Especifica el puerto y la dirección para escuchar.| | server_name | Define los nombres de dominio para el bloque de servidor.| | root | Establece el directorio raíz para las solicitudes.| | index | Especifica los archivos de índice para los directorios.| | error_page | Personaliza las páginas de error. | | proxy_pass | Reenvía solicitudes a otro servidor. | | location | Define cómo procesar URIs de solicitud específicos.|

Conclusión

Entender la configuración básica de NGINX es crucial para configurar un servidor web estable y eficiente. Al dominar los bloques de servidor, los bloques de ubicación y las directivas comunes, puedes gestionar y servir contenido web de manera efectiva.
