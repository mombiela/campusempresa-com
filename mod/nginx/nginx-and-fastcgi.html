Introducción a FastCGI

FastCGI es un protocolo para la interfaz de programas interactivos con un servidor web. Es una mejora sobre la antigua Interfaz de Gateway Común (CGI) y proporciona mejor rendimiento y escalabilidad.

  • CGI vs FastCGI:
    • CGI: Cada solicitud genera un nuevo proceso, lo que lleva a un alto overhead.
    • FastCGI: Mantiene los procesos vivos para manejar múltiples solicitudes, reduciendo el overhead.

Configuración de NGINX con FastCGI

Para usar FastCGI con NGINX, necesitas configurar NGINX para pasar solicitudes a un servidor FastCGI.

Instalación de FastCGI

Primero, asegúrate de tener un servidor FastCGI instalado. Para aplicaciones PHP, puedes usar php-fpm (Gestor de Procesos FastCGI de PHP).

sudo apt-get install php-fpm

Configuración de NGINX para Usar FastCGI

Edita tu archivo de configuración de NGINX para incluir parámetros de FastCGI.

server {
    listen 80;
    server_name example.com;

    root /var/www/html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
    }
}
  • Explicación:
    • server: Define un servidor virtual.
    • listen 80: Escucha en el puerto 80.
    • server_name example.com: El nombre del dominio.
    • root /var/www/html: El directorio raíz para el servidor.
    • location /: Maneja solicitudes a la raíz.
    • try_files $uri $uri/ =404: Intenta servir el archivo, directorio, o devuelve un 404.
    • location ~ \.php$: Maneja archivos PHP.
    • include snippets/fastcgi-php.conf: Incluye la configuración de FastCGI.
    • fastcgi_pass unix:/var/run/php/php7.4-fpm.sock: Pasa la solicitud al socket de PHP-FPM.

Parámetros de FastCGI

Puedes ajustar los parámetros de FastCGI para optimizar el rendimiento.

location ~ \.php$ {
    include fastcgi_params;
    fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
}
  • Explicación:
    • fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name: Establece el parámetro del nombre del script para FastCGI.

Configuración Avanzada de FastCGI

Para configuraciones más avanzadas, puedes ajustar los tamaños de los buffers y los tiempos de espera de FastCGI.

Tamaños de Buffers

Ajustar los tamaños de los buffers puede ayudar a manejar respuestas grandes.

location ~ \.php$ {
    include fastcgi_params;
    fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
    fastcgi_buffers 16 16k;
    fastcgi_buffer_size 32k;
}
  • Explicación:
    • fastcgi_buffers 16 16k: Establece el número y tamaño de los buffers de FastCGI.
    • fastcgi_buffer_size 32k: Establece el tamaño del buffer de FastCGI.

Tiempos de Espera

Establecer tiempos de espera apropiados puede prevenir solicitudes colgadas.

location ~ \.php$ {
    include fastcgi_params;
    fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
    fastcgi_connect_timeout 60s;
    fastcgi_send_timeout 60s;
    fastcgi_read_timeout 60s;
}
  • Explicación:
    • fastcgi_connect_timeout 60s: Tiempo de espera para conectar a FastCGI.
    • fastcgi_send_timeout 60s: Tiempo de espera para enviar datos a FastCGI.
    • fastcgi_read_timeout 60s: Tiempo de espera para leer datos de FastCGI.

Conclusión

NGINX y FastCGI trabajan juntos para proporcionar una forma poderosa y eficiente de servir contenido dinámico. Al entender y configurar los parámetros de FastCGI, puedes optimizar el rendimiento de tu servidor y manejar cargas de tráfico altas de manera efectiva. Esta sección cubrió los conceptos básicos de la configuración y uso de NGINX con FastCGI, así como configuraciones avanzadas para un mejor rendimiento.
