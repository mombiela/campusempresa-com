Introducción a Jenkins y Servicios en la Nube

En esta sección, exploraremos cómo Jenkins se integra con varios servicios en la nube para mejorar los pipelines de Integración Continua y Despliegue Continuo (CI/CD). Los servicios en la nube proporcionan recursos escalables y flexibles que pueden aprovecharse para mejorar la eficiencia y la fiabilidad de tus pipelines de Jenkins.

Conceptos Clave

  • Servicios en la Nube: Plataformas que proporcionan varios servicios informáticos a través de internet, como AWS, Azure y Google Cloud.
  • Pipelines CI/CD: Flujos de trabajo automatizados que integran cambios de código, ejecutan pruebas y despliegan aplicaciones.
  • Plugins de Jenkins: Extensiones que añaden funcionalidad a Jenkins, incluidas integraciones con servicios en la nube.

Configuración de Jenkins con Servicios en la Nube

Prerrequisitos

  • Conocimiento básico de Jenkins y pipelines CI/CD.
  • Una cuenta con un proveedor de servicios en la nube (por ejemplo, AWS, Azure, Google Cloud).

Instalación de Jenkins en un Servicio en la Nube

  1. AWS EC2:

    • Lanza una instancia EC2.
    • Conéctate a la instancia mediante SSH e instala Jenkins.
    • Configura los grupos de seguridad para permitir el acceso a Jenkins.
    # Actualiza el índice de paquetes
sudo apt-get update

# Instala Java
sudo apt-get install openjdk-11-jdk -y

# Añade el repositorio y la clave de Jenkins
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'

# Instala Jenkins
sudo apt-get update
sudo apt-get install jenkins -y

# Inicia Jenkins
sudo systemctl start jenkins
sudo systemctl enable jenkins

  2. Azure VM:

    • Crea una Máquina Virtual en Azure.
    • Conéctate a la VM mediante SSH y sigue pasos similares para instalar Jenkins como en AWS.

  3. Google Cloud Compute Engine:

    • Crea una instancia de Compute Engine.
    • Conéctate a la instancia mediante SSH e instala Jenkins.

Configuración de Jenkins con Plugins de la Nube

  • Plugin de AWS: Permite que Jenkins interactúe con los servicios de AWS.
  • Plugin de Azure: Integra Jenkins con los servicios de Azure.
  • Plugin de Google Cloud: Conecta Jenkins con los servicios de Google Cloud.

Ejemplo: Configuración del Plugin de AWS

  1. Instala el plugin de AWS desde el Administrador de Plugins de Jenkins.

  2. Configura las credenciales de AWS en Jenkins:

    • Ve a Manage Jenkins > Manage Credentials.
    • Añade las credenciales de AWS (ID de Clave de Acceso y Clave de Acceso Secreta).

  3. Usa el plugin de AWS en un pipeline de Jenkins:

    pipeline {
    agent any
    environment {
        AWS_ACCESS_KEY_ID = credentials('aws-access-key-id')
        AWS_SECRET_ACCESS_KEY = credentials('aws-secret-access-key')
    }
    stages {
        stage('Deploy to S3') {
            steps {
                script {
                    sh '''
                    aws s3 cp my-app.zip s3://my-bucket/my-app.zip
                    '''
                }
            }
        }
    }
}

Integraciones Avanzadas de Jenkins y la Nube

Uso de Jenkins con Kubernetes

  • Kubernetes: Una plataforma de código abierto para automatizar el despliegue, escalado y operaciones de contenedores de aplicaciones.
  • Plugin de Kubernetes para Jenkins: Permite que Jenkins aprovisione dinámicamente pods de Kubernetes para ejecutar trabajos.

Ejemplo: Pipeline de Jenkins con Kubernetes

  1. Instala el plugin de Kubernetes en Jenkins.

  2. Configura los detalles del clúster de Kubernetes en Jenkins.

  3. Crea un pipeline de Jenkins que use pods de Kubernetes:

    pipeline {
    agent {
        kubernetes {
            yaml """
            apiVersion: v1
            kind: Pod
            spec:
              containers:
              - name: jnlp
                image: jenkins/inbound-agent
                args: ['\$(JENKINS_SECRET)', '\$(JENKINS_NAME)']
              - name: maven
                image: maven:3.6.3-jdk-11
                command:
                - cat
                tty: true
            """
        }
    }
    stages {
        stage('Build') {
            steps {
                container('maven') {
                    sh 'mvn clean install'
                }
            }
        }
    }
}

Jenkins y Arquitecturas Serverless

  • Serverless: Un modelo de ejecución de computación en la nube donde el proveedor de la nube gestiona dinámicamente la asignación de recursos de máquina.
  • AWS Lambda: Un servicio de computación serverless que ejecuta código en respuesta a eventos.

Ejemplo: Activar AWS Lambda desde Jenkins

  1. Instala el plugin de AWS Lambda en Jenkins.

  2. Configura las credenciales de AWS y los detalles de la función Lambda.

  3. Crea un pipeline de Jenkins para invocar una función Lambda:

    pipeline {
    agent any
    environment {
        AWS_ACCESS_KEY_ID = credentials('aws-access-key-id')
        AWS_SECRET_ACCESS_KEY = credentials('aws-secret-access-key')
    }
    stages {
        stage('Invoke Lambda') {
            steps {
                script {
                    sh '''
                    aws lambda invoke --function-name myLambdaFunction --payload '{"key": "value"}' response.json
                    '''
                }
            }
        }
    }
}

Conclusión

Integrar Jenkins con servicios en la nube puede mejorar significativamente tus pipelines de CI/CD al aprovechar la escalabilidad, flexibilidad y diversas herramientas proporcionadas por las plataformas en la nube. Ya sea que estés desplegando aplicaciones en AWS, gestionando recursos en Azure o ejecutando contenedores en Kubernetes, Jenkins ofrece una amplia gama de plugins e integraciones para optimizar tus flujos de trabajo. Al entender y utilizar estas integraciones, puedes construir pipelines de CI/CD más eficientes, fiables y escalables.
