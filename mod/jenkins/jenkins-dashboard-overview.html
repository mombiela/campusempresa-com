Introducción

El panel de Jenkins es el centro neurálgico para gestionar tu instancia de Jenkins. Proporciona una vista completa de todos tus proyectos, compilaciones y configuraciones. Comprender el panel es crucial para usar Jenkins de manera efectiva, ya seas un principiante o un usuario avanzado.

Componentes clave del panel de Jenkins

Menú principal

  • Dashboard: La vista predeterminada que muestra todos tus trabajos y sus estados.
  • New Item: Crear un nuevo trabajo o proyecto.
  • People: Ver y gestionar cuentas de usuario.
  • Build History: Una línea de tiempo de todas las compilaciones en todos los trabajos.
  • Manage Jenkins: Acceder a configuraciones y ajustes administrativos.
  • My Views: Personalizar vistas para trabajos o proyectos específicos.
  • Credentials: Gestionar credenciales para acceder a sistemas externos.
  • Logout: Cerrar sesión en Jenkins.

Lista de trabajos

  • Muestra todos los trabajos configurados en Jenkins.
  • Muestra el estado de cada trabajo (por ejemplo, éxito, fallo, en progreso).
  • Proporciona acceso rápido a acciones específicas del trabajo como compilar, configurar y eliminar.

Cola de compilación

  • Lista los trabajos que están esperando ser ejecutados.
  • Muestra el estado actual y la posición en la cola.

Estado del ejecutor de compilación

  • Muestra el estado de los ejecutores de compilación (inactivo, compilando, etc.).
  • Indica qué trabajos se están ejecutando actualmente.

Navegando por el panel

Ver detalles del trabajo

Haz clic en cualquier nombre de trabajo para ver información detallada sobre ese trabajo, incluyendo:

  • Build History: Una lista de todas las compilaciones del trabajo.
  • Workspace: Archivos y directorios utilizados por el trabajo.
  • Configure: Configuraciones y ajustes para el trabajo.
  • Build Now: Iniciar una nueva compilación manualmente.

Configurando Jenkins

Accede a la sección "Manage Jenkins" para:

  • Configurar ajustes del sistema.
  • Gestionar plugins.
  • Configurar seguridad y permisos de usuario.
  • Monitorear registros del sistema y rendimiento.

Ejemplo: Crear un nuevo trabajo

  1. Haz clic en "New Item" en el menú principal.
  2. Ingresa un nombre para el trabajo.
  3. Selecciona el tipo de trabajo (por ejemplo, proyecto Freestyle, Pipeline).
  4. Haz clic en "OK" y configura los ajustes del trabajo según sea necesario.
  5. Guarda la configuración y regresa al panel.
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}
  • Este ejemplo muestra un script simple de Jenkins Pipeline con tres etapas: Build, Test y Deploy.
  • Cada etapa contiene pasos que realizan acciones específicas, como imprimir mensajes en la consola.

Resumen

El panel de Jenkins es una herramienta esencial para gestionar tu entorno Jenkins. Proporciona una visión clara de todos los trabajos, sus estados y los historiales de compilación. Familiarizándote con los componentes y la navegación del panel, puedes gestionar y monitorear eficientemente tus proyectos en Jenkins. Ya sea que estés creando nuevos trabajos, configurando ajustes o monitoreando compilaciones, el panel es tu interfaz principal para todas las actividades en Jenkins.
