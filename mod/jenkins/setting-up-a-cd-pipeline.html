Introducción

La Entrega Continua (CD) es una práctica de desarrollo de software donde los cambios en el código se preparan automáticamente para su lanzamiento a producción. Jenkins es una herramienta popular para implementar canalizaciones de CD. Esta sección te guiará a través de la configuración de una canalización de CD usando Jenkins, desde niveles principiantes hasta avanzados.

Prerrequisitos

Antes de configurar una canalización de CD, asegúrate de tener lo siguiente:

  • Jenkins instalado y en funcionamiento.
  • Comprensión básica de los trabajos y canalizaciones de Jenkins.
  • Repositorio de código fuente (por ejemplo, GitHub, GitLab).
  • Aplicación para desplegar (por ejemplo, una aplicación web simple).

Conceptos Básicos

  • Entrega Continua (CD): La práctica de desplegar automáticamente los cambios en el código a un entorno de preparación o producción.
  • Canalización: Una serie de pasos automatizados para construir, probar y desplegar código.
  • Jenkinsfile: Un archivo de texto que contiene la definición de una Canalización de Jenkins y se registra en el control de versiones.

Configuración de una Canalización de CD Básica

Paso 1: Crear un Trabajo de Canalización en Jenkins

  1. Abre Jenkins y navega a Nuevo Elemento.
  2. Ingresa un nombre para tu canalización y selecciona Pipeline, luego haz clic en OK.
  3. En la configuración de la canalización, desplázate hacia abajo hasta la sección Pipeline.
  4. Selecciona Pipeline script from SCM.
  5. Elige tu SCM (por ejemplo, Git) y proporciona la URL del repositorio.
  6. Especifica la ruta del script (por ejemplo, Jenkinsfile).

Paso 2: Escribir un Jenkinsfile Básico

Crea un Jenkinsfile en tu repositorio con el siguiente contenido:

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Agrega pasos de construcción aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Agrega pasos de prueba aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Agrega pasos de despliegue aquí
            }
        }
    }
}

Paso 3: Activar la Canalización

  1. Guarda el Jenkinsfile en tu repositorio.
  2. Vuelve a Jenkins y activa la canalización haciendo clic en Build Now.

Conceptos Intermedios

Agregar Notificaciones

Para recibir notificaciones sobre el estado de la canalización, puedes usar complementos como Email Extension o Slack Notification.

Ejemplo: Notificación por Correo Electrónico

Agrega los siguientes pasos al Jenkinsfile:

post {
    success {
        mail to: 'team@example.com',
             subject: "Pipeline Success: ${currentBuild.fullDisplayName}",
             body: "Good news! The pipeline has succeeded."
    }
    failure {
        mail to: 'team@example.com',
             subject: "Pipeline Failure: ${currentBuild.fullDisplayName}",
             body: "Unfortunately, the pipeline has failed. Please check the logs."
    }
}

Etapas Paralelas

Para acelerar la canalización, puedes ejecutar etapas en paralelo.

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        echo 'Running unit tests...'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        echo 'Running integration tests...'
                    }
                }
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}

Conceptos Avanzados

Uso de Docker en Canalizaciones

Puedes usar contenedores Docker para proporcionar un entorno consistente para tus construcciones.

pipeline {
    agent {
        docker {
            image 'maven:3.6.3-jdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }

    stages {
        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}

Desplegar en Kubernetes

Para despliegues avanzados, puedes desplegar tu aplicación en un clúster de Kubernetes.

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            steps {
                withKubeConfig([credentialsId: 'kubeconfig-credentials']) {
                    sh 'kubectl apply -f deployment.yaml'
                }
            }
        }
    }
}

Conclusión

Configurar una canalización de CD en Jenkins implica crear un trabajo de canalización, escribir un Jenkinsfile y configurar etapas para construir, probar y desplegar. A medida que avanzas, puedes agregar notificaciones, ejecutar etapas en paralelo, usar Docker y desplegar en Kubernetes. La Entrega Continua con Jenkins ayuda a automatizar el proceso de despliegue, asegurando lanzamientos más rápidos y confiables.
