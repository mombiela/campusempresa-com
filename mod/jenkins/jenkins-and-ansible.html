Introducción a Jenkins y Ansible

En esta sección, presentaremos Jenkins y Ansible, explicando qué son y cómo pueden usarse juntos para automatizar y optimizar tus procesos DevOps.

  • Jenkins: Un servidor de automatización de código abierto que ayuda a automatizar las partes del desarrollo de software relacionadas con la construcción, prueba y despliegue, facilitando la integración continua y la entrega continua (CI/CD).
  • Ansible: Una herramienta de automatización de código abierto utilizada para la gestión de configuraciones, despliegue de aplicaciones y automatización de tareas. Utiliza un lenguaje simple y legible por humanos (YAML) para describir trabajos de automatización.

Configuración de Jenkins

Antes de integrar Jenkins con Ansible, necesitas tener Jenkins configurado y en funcionamiento.

Instalación de Jenkins

  1. Descargar Jenkins: Visita el sitio web de Jenkins y descarga el instalador apropiado para tu sistema operativo.
  2. Instalar Jenkins: Sigue las instrucciones de instalación para tu sistema operativo.
  3. Iniciar Jenkins: Una vez instalado, inicia el servicio de Jenkins.

Configuración de Jenkins

  1. Configuración Inicial: Abre un navegador web y navega a http://localhost:8080. Sigue el asistente de configuración para completar la configuración inicial.
  2. Instalar Plugins: Durante la configuración, instala los plugins sugeridos o selecciona manualmente los plugins que necesitas.

Configuración de Ansible

Ansible necesita estar instalado en la máquina donde Jenkins está ejecutándose.

Instalación de Ansible

  1. Usando pip: 
    pip install ansible
  2. Usando el gestor de paquetes:
    • Ubuntu: 
      sudo apt update
sudo apt install ansible
    • CentOS: 
      sudo yum install ansible

Integración de Jenkins con Ansible

Para usar Ansible dentro de Jenkins, necesitas configurar Jenkins para ejecutar playbooks de Ansible.

Instalación del Plugin de Ansible en Jenkins

  1. Navegar al Tablero de Jenkins: Ve a Manage Jenkins > Manage Plugins.
  2. Instalar el Plugin de Ansible: Busca el plugin "Ansible" e instálalo.

Configuración de Ansible en Jenkins

  1. Agregar Instalación de Ansible: Ve a Manage Jenkins > Global Tool Configuration.
  2. Configurar Ansible: En la sección de Ansible, agrega una nueva instalación de Ansible proporcionando un nombre y la ruta al ejecutable de Ansible.

Creación de un Trabajo de Jenkins para Ejecutar Playbooks de Ansible

  1. Crear un Nuevo Trabajo: Desde el tablero de Jenkins, haz clic en New Item, ingresa un nombre y selecciona Freestyle project.
  2. Configurar el Trabajo:
    • Gestión de Código Fuente: Configura tu repositorio de código fuente (por ejemplo, Git).
    • Entorno de Construcción: Marca Provide Node & Label si es necesario.
    • Pasos de Construcción: Agrega un paso de construcción para Invoke Ansible Playbook.

Ejemplo: Ejecutar un Playbook Simple de Ansible

  1. Crear un Playbook: Crea un playbook simple de Ansible llamado example-playbook.yml. 
    ---
- name: Example Playbook
  hosts: localhost
  tasks:
    - name: Ensure Apache is installed
      apt:
        name: apache2
        state: present
  2. Configurar el Trabajo de Jenkins:
    • Paso de Construcción: Agrega Invoke Ansible Playbook y configúralo para usar example-playbook.yml.

Ejecutar el Trabajo

  1. Construir el Trabajo: Haz clic en Build Now para ejecutar el trabajo.
  2. Verificar la Salida: Monitorea la salida de la consola para asegurarte de que el playbook se ejecute correctamente.

Integración Avanzada de Jenkins y Ansible

Para casos de uso más avanzados, puedes integrar Jenkins y Ansible para manejar escenarios de despliegue complejos.

Usar Pipelines de Jenkins con Ansible

  1. Crear un Trabajo de Pipeline: Desde el tablero de Jenkins, haz clic en New Item, ingresa un nombre y selecciona Pipeline.
  2. Definir el Script del Pipeline: 
    pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/your-repo.git'
            }
        }
        stage('Run Ansible Playbook') {
            steps {
                ansiblePlaybook playbook: 'example-playbook.yml'
            }
        }
    }
}

Ejemplo: Desplegar una Aplicación Web

  1. Playbook para Despliegue: 
    ---
- name: Deploy Web Application
  hosts: webservers
  tasks:
    - name: Ensure Nginx is installed
      apt:
        name: nginx
        state: present
    - name: Deploy application code
      copy:
        src: /path/to/app
        dest: /var/www/html
  2. Script del Pipeline: 
    pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/your-repo.git'
            }
        }
        stage('Run Ansible Playbook') {
            steps {
                ansiblePlaybook playbook: 'deploy-webapp.yml'
            }
        }
    }
}

Conclusión

Integrar Jenkins y Ansible te permite automatizar procesos de despliegue complejos, asegurando consistencia y fiabilidad. Siguiendo esta guía, deberías poder configurar Jenkins, configurarlo para usar Ansible, y crear trabajos y pipelines que ejecuten playbooks de Ansible. Esta integración es una herramienta poderosa en el kit de herramientas DevOps, permitiendo flujos de trabajo CI/CD sin problemas.
