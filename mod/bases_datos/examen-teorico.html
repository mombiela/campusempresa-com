Introducción

El examen teórico es una parte crucial del curso sobre Fundamentos de Bases de Datos. Este examen evaluará tu comprensión de los conceptos clave, las estructuras y las prácticas esenciales en el manejo de bases de datos. A continuación, se detallan los temas que serán cubiertos en el examen, junto con ejemplos y ejercicios para ayudarte a prepararte de manera efectiva.

Temas Clave

1. Conceptos Básicos de Bases de Datos

  • Definición de Base de Datos: Conjunto de datos organizados de manera que se pueda acceder, gestionar y actualizar fácilmente.
  • Sistemas de Gestión de Bases de Datos (SGBD): Software que permite la creación, gestión y manipulación de bases de datos.
  • Tipos de Bases de Datos: Relacionales, NoSQL, jerárquicas, en red, etc.

2. Modelos de Datos

  • Modelo Relacional: Basado en tablas (relaciones) que contienen filas y columnas.
  • Modelo Entidad-Relación (ER): Utiliza entidades y relaciones para representar datos y sus interacciones.
  • Modelo NoSQL: Incluye bases de datos de documentos, clave-valor, columnas y grafos.

3. Lenguaje de Consulta Estructurado (SQL)

  • Consultas Básicas: SELECT, INSERT, UPDATE, DELETE.
  • Consultas Avanzadas: JOIN, GROUP BY, HAVING, subconsultas.
  • Definición de Datos: CREATE, ALTER, DROP.

Ejemplo de Código SQL

-- Crear una tabla
CREATE TABLE empleados (
    id INT PRIMARY KEY,
    nombre VARCHAR(100),
    puesto VARCHAR(50),
    salario DECIMAL(10, 2)
);

-- Insertar datos en la tabla
INSERT INTO empleados (id, nombre, puesto, salario) VALUES
(1, 'Juan Pérez', 'Desarrollador', 50000.00),
(2, 'Ana Gómez', 'Analista', 55000.00);

-- Consultar datos de la tabla
SELECT * FROM empleados WHERE salario > 50000;

4. Normalización

  • Primera Forma Normal (1NF): Eliminar duplicados y asegurar que cada columna contenga valores atómicos.
  • Segunda Forma Normal (2NF): Eliminar dependencias parciales.
  • Tercera Forma Normal (3NF): Eliminar dependencias transitivas.

5. Transacciones y Concurrencia

  • ACID: Atomicidad, Consistencia, Aislamiento, Durabilidad.
  • Control de Concurrencia: Métodos para asegurar que múltiples transacciones puedan ocurrir simultáneamente sin causar inconsistencias.

6. Seguridad en Bases de Datos

  • Autenticación y Autorización: Control de acceso a los datos.
  • Cifrado: Protección de datos sensibles.
  • Auditoría: Registro de actividades y cambios en la base de datos.

Ejercicios Prácticos

  1. Crear y Consultar Tablas:

    • Crea una tabla llamada clientes con columnas para id, nombre, email y fecha_registro.
    • Inserta al menos tres registros en la tabla.
    • Realiza una consulta para obtener todos los clientes registrados en el último mes.

  2. Normalización:

    • Dada una tabla pedidos con columnas id_pedido, cliente, producto, cantidad, precio_total, normaliza la tabla hasta la 3NF.

  3. Transacciones:

    • Escribe una transacción que inserte un nuevo pedido en la tabla pedidos y actualice el inventario en la tabla productos. Asegúrate de que la transacción sea atómica.

Conclusión

El examen teórico sobre Fundamentos de Bases de Datos evaluará tu comprensión de los conceptos esenciales y tu capacidad para aplicar estos conocimientos en situaciones prácticas. Asegúrate de revisar cada uno de los temas clave, practicar con ejemplos de código y resolver los ejercicios propuestos. Esto te preparará para enfrentar el examen con confianza y éxito.
