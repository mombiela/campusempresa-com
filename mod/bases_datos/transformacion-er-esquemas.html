Transformación de Diagramas ER a Esquemas Relacionales
Conceptos de Normalización ►

Introducción

En esta sección, aprenderemos cómo transformar un Diagrama de Entidad-Relación (ER) en un esquema relacional. Este proceso es fundamental para diseñar bases de datos relacionales a partir de modelos conceptuales, asegurando que la estructura de la base de datos sea eficiente y coherente.

Conceptos Clave

  • Diagrama ER: Representación gráfica de entidades y sus relaciones.
  • Esquema Relacional: Conjunto de tablas que representan las entidades y relaciones de un diagrama ER en una base de datos relacional.
  • Entidad: Objeto del mundo real que se distingue de otros objetos.
  • Atributo: Propiedad o característica de una entidad.
  • Clave Primaria: Atributo o conjunto de atributos que identifican de manera única una entidad.
  • Clave Foránea: Atributo en una tabla que se refiere a la clave primaria de otra tabla.

Pasos para la Transformación

1. Transformación de Entidades Simples

Cada entidad simple en el diagrama ER se convierte en una tabla en el esquema relacional.

  • Ejemplo:

    • Entidad Estudiante con atributos ID, Nombre, Edad.

    • Tabla resultante:

      CREATE TABLE Estudiante (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Edad INT
);

2. Transformación de Relaciones

Las relaciones entre entidades se transforman en tablas adicionales o se incorporan en las tablas existentes mediante claves foráneas.

Relaciones Uno a Uno (1:1)

  • Ejemplo:

    • Entidades Persona y Pasaporte con una relación 1:1.

    • Tabla resultante:

      CREATE TABLE Persona (
    PersonaID INT PRIMARY KEY,
    Nombre VARCHAR(100)
);

CREATE TABLE Pasaporte (
    PasaporteID INT PRIMARY KEY,
    PersonaID INT,
    FechaExpiracion DATE,
    FOREIGN KEY (PersonaID) REFERENCES Persona(PersonaID)
);

Relaciones Uno a Muchos (1:N)

  • Ejemplo:

    • Entidades Profesor y Curso con una relación 1:N.

    • Tabla resultante:

      CREATE TABLE Profesor (
    ProfesorID INT PRIMARY KEY,
    Nombre VARCHAR(100)
);

CREATE TABLE Curso (
    CursoID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    ProfesorID INT,
    FOREIGN KEY (ProfesorID) REFERENCES Profesor(ProfesorID)
);

Relaciones Muchos a Muchos (M:N)

  • Ejemplo:

    • Entidades Estudiante y Curso con una relación M:N.

    • Tabla resultante:

      CREATE TABLE Estudiante (
    EstudianteID INT PRIMARY KEY,
    Nombre VARCHAR(100)
);

CREATE TABLE Curso (
    CursoID INT PRIMARY KEY,
    Nombre VARCHAR(100)
);

CREATE TABLE EstudianteCurso (
    EstudianteID INT,
    CursoID INT,
    PRIMARY KEY (EstudianteID, CursoID),
    FOREIGN KEY (EstudianteID) REFERENCES Estudiante(EstudianteID),
    FOREIGN KEY (CursoID) REFERENCES Curso(CursoID)
);

3. Transformación de Atributos Compuestos

Los atributos compuestos se descomponen en atributos simples.

  • Ejemplo:

    • Atributo compuesto Dirección con sub-atributos Calle, Ciudad, Código Postal.

    • Tabla resultante:

      CREATE TABLE Cliente (
    ClienteID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Calle VARCHAR(100),
    Ciudad VARCHAR(100),
    CodigoPostal VARCHAR(10)
);

4. Transformación de Atributos Multivaluados

Los atributos multivaluados se transforman en tablas separadas.

  • Ejemplo:

    • Entidad Empleado con atributo multivaluado Teléfonos.

    • Tabla resultante:

      CREATE TABLE Empleado (
    EmpleadoID INT PRIMARY KEY,
    Nombre VARCHAR(100)
);

CREATE TABLE Telefono (
    EmpleadoID INT,
    NumeroTelefono VARCHAR(15),
    PRIMARY KEY (EmpleadoID, NumeroTelefono),
    FOREIGN KEY (EmpleadoID) REFERENCES Empleado(EmpleadoID)
);

Ejercicio Práctico

Transforma el siguiente diagrama ER en un esquema relacional:

  • Entidades: Autor (ID, Nombre), Libro (ISBN, Título, Año), Editorial (ID, Nombre).
  • Relaciones:
    • Autor escribe Libro (M:N).
    • Libro es publicado por Editorial (N:1).

Conclusión

La transformación de diagramas ER a esquemas relacionales es un paso crucial en el diseño de bases de datos. Este proceso asegura que la estructura de la base de datos sea lógica y eficiente, facilitando la implementación y el mantenimiento de la base de datos. Practicar estos pasos con diferentes diagramas ER ayudará a consolidar estos conceptos y habilidades.
