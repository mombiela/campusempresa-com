Creación y Uso de Servicios
Inyección de Dependencias ►

En Angular, los servicios son un concepto fundamental utilizado para compartir datos, lógica y funciones entre diferentes componentes de una aplicación. Este tema te guiará a través de la creación y uso de servicios, desde niveles básicos hasta avanzados.

¿Qué es un Servicio?

  • Definición: Un servicio es una clase con un propósito específico. Se utiliza típicamente para realizar tareas como obtener datos de un servidor, validar la entrada del usuario o registrar logs.
  • Propósito: Los servicios ayudan a mantener tus componentes ligeros y enfocados en sus tareas principales, como mostrar datos y manejar la interacción del usuario.

Creación de un Servicio Básico

Guía Paso a Paso

  1. Generar un Servicio: Usa Angular CLI para generar un servicio.

    ng generate service my-service

    Este comando crea dos archivos: my-service.service.ts y my-service.service.spec.ts.

  2. Definir el Servicio: Abre my-service.service.ts y define el servicio.

    import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MyService {
  constructor() { }

  getData() {
    return '¡Hola desde MyService!';
  }
}

Explicación

  • @Injectable({ providedIn: 'root' }): Este decorador hace que el servicio esté disponible en toda la aplicación.
  • getData(): Un método simple que devuelve una cadena.

Usar un Servicio en un Componente

Guía Paso a Paso

  1. Inyectar el Servicio: Abre un archivo de componente (por ejemplo, app.component.ts) e inyecta el servicio.

    import { Component, OnInit } from '@angular/core';
import { MyService } from './my-service.service';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent implements OnInit {
  message: string;

  constructor(private myService: MyService) {}

  ngOnInit() {
    this.message = this.myService.getData();
  }
}

  2. Mostrar los Datos: Vincula los datos en la plantilla del componente (por ejemplo, app.component.html).

    <div>
  {{ message }}
</div>

Explicación

  • constructor(private myService: MyService): El servicio se inyecta en el componente.
  • ngOnInit(): El método getData() del servicio se llama cuando el componente se inicializa.

Características Avanzadas del Servicio

Inyección de Dependencias (DI)

  • Inyectores Jerárquicos: El sistema DI de Angular permite que los servicios se proporcionen en diferentes niveles (raíz, módulo, componente).
  • Ejemplo: Proporcionar un servicio a nivel de componente. 
    @Component({
  selector: 'app-child',
  templateUrl: './child.component.html',
  styleUrls: ['./child.component.css'],
  providers: [MyService]
})
export class ChildComponent {
  constructor(private myService: MyService) {}
}

Servicios HTTP

  • Obtener Datos de una API: Usa HttpClient de Angular para obtener datos de un servidor. 
    import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) {}

  fetchData(): Observable<any> {
    return this.http.get<any>(this.apiUrl);
  }
}

Explicación

  • HttpClient: Servicio incorporado de Angular para hacer solicitudes HTTP.
  • fetchData(): Método para obtener datos de la URL de la API especificada.

Usar el Servicio HTTP en un Componente

  1. Inyectar el Servicio: Inyecta el DataService en un componente.

    import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  templateUrl: './data.component.html',
  styleUrls: ['./data.component.css']
})
export class DataComponent implements OnInit {
  data: any;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.dataService.fetchData().subscribe(response => {
      this.data = response;
    });
  }
}

  2. Mostrar los Datos: Vincula los datos obtenidos en la plantilla del componente.

    <div *ngIf="data">
  <pre>{{ data | json }}</pre>
</div>

Explicación

  • fetchData().subscribe(response => { this.data = response; }): Se suscribe al observable devuelto por fetchData() y asigna la respuesta a la propiedad data.

Resumen

  • Servicios: Esenciales para compartir datos y lógica entre componentes.
  • Creación de Servicios: Usa Angular CLI y el decorador @Injectable.
  • Uso de Servicios: Inyecta servicios en componentes y llama a sus métodos.
  • Características Avanzadas: Inyección de dependencias en diferentes niveles, uso de HttpClient para llamadas a API.

Al dominar los servicios en Angular, puedes crear aplicaciones más modulares, mantenibles y testeables.
