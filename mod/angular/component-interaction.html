Interacción entre Componentes
Directivas Incorporadas ►

La interacción entre componentes es un concepto fundamental en Angular que permite a diferentes componentes comunicarse y compartir datos. Este tema cubrirá varios métodos de interacción entre componentes, que van desde la simple comunicación entre padre e hijo hasta técnicas más avanzadas como el uso de servicios y la gestión de estado.

Comunicación de Padre a Hijo

La comunicación de padre a hijo es una de las formas más comunes de interacción entre componentes en Angular. Esto se logra típicamente usando propiedades de entrada.

Propiedades de Entrada

  • Definición: Las propiedades de entrada permiten a un componente padre pasar datos a un componente hijo.
  • Uso: Usa el decorador @Input para definir una propiedad de entrada en el componente hijo.

Ejemplo

Plantilla del Componente Padre (parent.component.html):

<app-child [childProperty]="parentData"></app-child>

Clase del Componente Padre (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  parentData = 'Datos del padre';
}

Clase del Componente Hijo (child.component.ts):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-child',
  template: '<p>{{ childProperty }}</p>',
})
export class ChildComponent {
  @Input() childProperty: string;
}

En este ejemplo, el componente padre pasa los parentData al componente hijo usando la propiedad de entrada childProperty.

Comunicación de Hijo a Padre

La comunicación de hijo a padre se logra a menudo usando propiedades de salida y emisores de eventos.

Propiedades de Salida y Emisores de Eventos

  • Definición: Las propiedades de salida permiten a un componente hijo emitir eventos que un componente padre puede escuchar.
  • Uso: Usa el decorador @Output junto con un EventEmitter para definir una propiedad de salida en el componente hijo.

Ejemplo

Clase del Componente Hijo (child.component.ts):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-child',
  template: '<button (click)="sendData()">Enviar Datos</button>',
})
export class ChildComponent {
  @Output() dataEmitter = new EventEmitter<string>();

  sendData() {
    this.dataEmitter.emit('Datos del hijo');
  }
}

Plantilla del Componente Padre (parent.component.html):

<app-child (dataEmitter)="receiveData($event)"></app-child>

Clase del Componente Padre (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  receiveData(data: string) {
    console.log(data);
  }
}

En este ejemplo, el componente hijo emite un evento con datos, que el componente padre escucha y maneja.

Comunicación entre Hermanos

Los componentes hermanos pueden comunicarse a través de un componente padre compartido o un servicio común.

Usando un Componente Padre Compartido

  • Definición: Los componentes hermanos pueden comunicarse pasando datos a través de su componente padre compartido.
  • Uso: El componente padre actúa como mediador, recibiendo datos de un hijo y pasándolos a otro.

Ejemplo

Plantilla del Componente Padre (parent.component.html):

<app-sibling-one (dataEmitter)="receiveData($event)"></app-sibling-one>
<app-sibling-two [siblingData]="sharedData"></app-sibling-two>

Clase del Componente Padre (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  sharedData: string;

  receiveData(data: string) {
    this.sharedData = data;
  }
}

Clase del Componente Hermano Uno (sibling-one.component.ts):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-sibling-one',
  template: '<button (click)="sendData()">Enviar Datos</button>',
})
export class SiblingOneComponent {
  @Output() dataEmitter = new EventEmitter<string>();

  sendData() {
    this.dataEmitter.emit('Datos del hermano uno');
  }
}

Clase del Componente Hermano Dos (sibling-two.component.ts):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-sibling-two',
  template: '<p>{{ siblingData }}</p>',
})
export class SiblingTwoComponent {
  @Input() siblingData: string;
}

Usando un Servicio

  • Definición: Un servicio puede ser usado para compartir datos y facilitar la comunicación entre componentes hermanos.
  • Uso: Crea un servicio con datos compartidos y métodos para actualizar y recuperar esos datos.

Ejemplo

Servicio Compartido (shared.service.ts):

import { Injectable } from '@angular/core';
import { Subject } from 'rxjs';

@Injectable({
  providedIn: 'root',
})
export class SharedService {
  private dataSubject = new Subject<string>();
  data$ = this.dataSubject.asObservable();

  sendData(data: string) {
    this.dataSubject.next(data);
  }
}

Clase del Componente Hermano Uno (sibling-one.component.ts):

import { Component } from '@angular/core';
import { SharedService } from './shared.service';

@Component({
  selector: 'app-sibling-one',
  template: '<button (click)="sendData()">Enviar Datos</button>',
})
export class SiblingOneComponent {
  constructor(private sharedService: SharedService) {}

  sendData() {
    this.sharedService.sendData('Datos del hermano uno');
  }
}

Clase del Componente Hermano Dos (sibling-two.component.ts):

import { Component, OnInit } from '@angular/core';
import { SharedService } from './shared.service';

@Component({
  selector: 'app-sibling-two',
  template: '<p>{{ siblingData }}</p>',
})
export class SiblingTwoComponent implements OnInit {
  siblingData: string;

  constructor(private sharedService: SharedService) {}

  ngOnInit() {
    this.sharedService.data$.subscribe(data => {
      this.siblingData = data;
    });
  }
}

En este ejemplo, el SharedService facilita la comunicación entre SiblingOneComponent y SiblingTwoComponent.

Conclusión

La interacción entre componentes es un aspecto crucial para construir aplicaciones Angular. Entender cómo comunicarse efectivamente entre componentes usando propiedades de entrada y salida, componentes padres compartidos y servicios te permitirá crear aplicaciones más dinámicas y receptivas. A medida que progreses de niveles principiantes a avanzados, dominar estas técnicas mejorará significativamente tu capacidad para gestionar y estructurar tus proyectos en Angular.
