Introducción

La Interfaz de Línea de Comandos de Angular (CLI) es una herramienta poderosa que ayuda a los desarrolladores a crear, construir y gestionar aplicaciones Angular. Esta sección cubrirá las mejores prácticas para usar Angular CLI de manera efectiva, desde comandos básicos hasta configuraciones avanzadas.

Empezando con Angular CLI

  • Instalación: Asegúrate de tener Node.js instalado, luego instala Angular CLI globalmente usando: 
    npm install -g @angular/cli
  • Creando un Nuevo Proyecto: Usa el comando ng new para crear un nuevo proyecto Angular: 
    ng new my-angular-app
    • Sigue las indicaciones para configurar tu proyecto con enrutamiento, hojas de estilo y más.

Estructura y Organización del Proyecto

  • Convenciones de Nombres Consistentes: Usa nombres consistentes y descriptivos para archivos y directorios.
    • Ejemplo: Usa kebab-case para nombres de archivos (my-component.component.ts).
  • Arquitectura Modular: Organiza tu aplicación en módulos para mejorar la mantenibilidad y escalabilidad.
    • Ejemplo: Crea módulos de características para diferentes partes de tu aplicación (user.module.ts, admin.module.ts).

Uso de Comandos de Angular CLI

  • Generando Componentes, Servicios y Módulos: Usa el comando ng generate (o ng g) para crear nuevos componentes, servicios y módulos. 
    ng generate component my-component
ng generate service my-service
ng generate module my-module
  • Sirviendo la Aplicación: Usa el comando ng serve para ejecutar tu aplicación localmente. 
    ng serve
    • Usa la bandera --open para abrir automáticamente la aplicación en tu navegador predeterminado.
    ng serve --open

Gestión de Configuración y Entornos

  • Archivos de Entorno: Usa archivos de entorno para gestionar diferentes configuraciones para desarrollo, staging y producción.
    • Ejemplo: environment.ts para desarrollo y environment.prod.ts para producción.
  • Configuración Personalizada: Modifica el archivo angular.json para personalizar las configuraciones de construcción y servicio.
    • Ejemplo: Añadiendo una nueva configuración de construcción para staging.
    "configurations": {
  "staging": {
    "fileReplacements": [
      {
        "replace": "src/environments/environment.ts",
        "with": "src/environments/environment.staging.ts"
      }
    ]
  }
}

Optimización de Construcciones

  • Construcciones de Producción: Siempre usa la bandera --prod al construir para producción para habilitar optimizaciones como la compilación Ahead-of-Time (AOT) y la minificación. 
    ng build --prod
  • Carga Diferida: Implementa la carga diferida para módulos de características para mejorar el tiempo de carga inicial.
    • Ejemplo: Define rutas con carga diferida.
    const routes: Routes = [
  { path: 'feature', loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) }
];

Pruebas y Linting

  • Pruebas Unitarias: Usa Angular CLI para ejecutar pruebas unitarias con Karma y Jasmine. 
    ng test
  • Pruebas de Extremo a Extremo: Usa Protractor para pruebas de extremo a extremo. 
    ng e2e
  • Linting: Usa el comando ng lint para ejecutar verificaciones de linting y asegurar la calidad del código. 
    ng lint

Despliegue

  • Desplegando en Firebase: Usa Angular CLI para desplegar tu aplicación en Firebase Hosting. 
    ng add @angular/fire
ng deploy
  • Otras Opciones de Despliegue: Configura Angular CLI para el despliegue en otras plataformas como AWS, Azure o Netlify.

Conclusión

Usar Angular CLI de manera efectiva puede mejorar significativamente tu flujo de trabajo de desarrollo, mejorar la calidad del código y agilizar el proceso de despliegue. Siguiendo estas mejores prácticas, puedes asegurar que tus aplicaciones Angular estén bien estructuradas, sean mantenibles y estén optimizadas para el rendimiento.
