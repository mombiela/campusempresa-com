El enrutamiento es una característica central en Angular que te permite navegar entre diferentes vistas o componentes en tu aplicación. Esta sección te guiará a través de la configuración de rutas desde un nivel principiante hasta un nivel avanzado.

Introducción al Enrutamiento en Angular

El enrutamiento en Angular permite la navegación de una vista a otra mientras los usuarios realizan tareas en tu aplicación. Ayuda a crear Aplicaciones de Página Única (SPAs) donde la página no se recarga, pero la vista cambia.

Conceptos Clave

  • Router Module: El módulo de Angular para el enrutamiento.
  • Routes: Objetos de configuración que definen la ruta y el componente a mostrar.
  • Router Outlet: Una directiva que actúa como un marcador de posición para el componente enrutado.
  • Router Link: Una directiva para enlazar a rutas.

Configuración de Rutas Básicas

Para comenzar con el enrutamiento, necesitas importar el RouterModule y definir tus rutas.

Guía Paso a Paso

  1. Instalar Angular Router (si no está ya instalado):

    npm install @angular/router

  2. Importar RouterModule en tu módulo principal (app.module.ts):

    import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  { path: 'home', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppModule { }

  3. Agregar Router Outlet en tu plantilla principal (app.component.html):

    <nav>
  <a routerLink="/home">Home</a>
  <a routerLink="/about">About</a>
</nav>
<router-outlet></router-outlet>

Explicación

  • RouterModule.forRoot(routes): Configura el enrutador en el nivel raíz de la aplicación.
  • routerLink: Directiva para enlazar la etiqueta de anclaje a una ruta.
  • router-outlet: Marcador de posición para el componente enrutado.

Técnicas Avanzadas de Enrutamiento

A medida que tu aplicación crece, puedes necesitar características de enrutamiento más avanzadas como rutas anidadas, guardias de ruta y carga diferida.

Rutas Anidadas

Las rutas anidadas te permiten definir rutas dentro de otras rutas.

const routes: Routes = [
  {
    path: 'dashboard',
    component: DashboardComponent,
    children: [
      { path: 'stats', component: StatsComponent },
      { path: 'reports', component: ReportsComponent }
    ]
  }
];

Guardias de Ruta

Las guardias de ruta se utilizan para controlar el acceso a las rutas basadas en ciertas condiciones.

  1. Crear una Guardia:

    import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot } from '@angular/router';

@Injectable({
  proporcionado en: 'root'
})
export class AuthGuard implements CanActivate {
  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): boolean {
    // Agrega tu lógica de autenticación aquí
    return true; // o false basado en la condición
  }
}

  2. Aplicar la Guardia a una Ruta:

    const routes: Routes = [
  { path: 'admin', component: AdminComponent, canActivate: [AuthGuard] }
];

Carga Diferida

La carga diferida ayuda a cargar módulos de características solo cuando son necesarios, mejorando el rendimiento de la aplicación.

  1. Crear un Módulo de Características:

    @NgModule({
  declarations: [FeatureComponent],
  imports: [CommonModule, RouterModule.forChild([{ path: '', component: FeatureComponent }])]
})
export class FeatureModule { }

  2. Configurar la Carga Diferida en las Rutas:

    const routes: Routes = [
  { path: 'feature', loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) }
];

Resumen

Configurar rutas en Angular implica configurar el RouterModule con definiciones de rutas y usar directivas como routerLink y router-outlet para navegar y mostrar componentes. A medida que avanzas, puedes implementar rutas anidadas, guardias de ruta y carga diferida para optimizar y asegurar tu aplicación. Entender estos conceptos te ayudará a construir aplicaciones Angular robustas y escalables.
