Introducción a los Componentes

Los componentes son los bloques de construcción de cualquier aplicación Angular. Encapsulan la vista, la lógica y los estilos, lo que facilita la gestión y reutilización del código.

Conceptos Clave

  • Componente: Una clase que interactúa con la vista (HTML) y define el comportamiento de una parte de la interfaz de usuario.
  • Plantilla: La parte HTML del componente que define la vista.
  • Estilos: CSS o SCSS que define el aspecto y la sensación del componente.
  • Metadatos: Información sobre el componente, como su selector, plantilla y estilos.

Creación de un Componente Básico

Para crear un nuevo componente en Angular, puedes usar Angular CLI.

Usando Angular CLI

Ejecuta el siguiente comando en tu terminal:

ng generate component my-component

Este comando creará una nueva carpeta llamada my-component con los siguientes archivos:

  • my-component.component.ts: El archivo TypeScript que contiene la clase del componente.
  • my-component.component.html: La plantilla HTML para el componente.
  • my-component.component.css: Los estilos CSS para el componente.
  • my-component.component.spec.ts: El archivo de prueba para el componente.

Estructura del Componente

Desglosemos el archivo generado my-component.component.ts:

import { Component } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  // La lógica del componente va aquí
}
  • Decorador @Component: Este decorador proporciona metadatos sobre el componente.
    • selector: La etiqueta HTML personalizada que representa el componente.
    • templateUrl: La ruta al archivo de la plantilla HTML.
    • styleUrls: La ruta al(los) archivo(s) CSS.

Usando el Componente

Para usar el componente recién creado, agrega su selector a la plantilla de otro componente, como app.component.html:

<app-my-component></app-my-component>

Interacción de Componentes

Los componentes a menudo necesitan interactuar entre sí. Esto se puede hacer usando:

  • Input: Pasar datos de un componente padre a un componente hijo.
  • Output: Emitir eventos desde un componente hijo a un componente padre.

Ejemplo de Input

En el componente hijo (my-component):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  @Input() data: string;
}

En el componente padre (app.component.html):

<app-my-component [data]="parentData"></app-my-component>

En la clase del componente padre (app.component.ts):

export class AppComponent {
  parentData = 'Hola desde el Padre';
}

Ejemplo de Output

En el componente hijo (my-component):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  @Output() notify: EventEmitter<string> = new EventEmitter<string>();

  sendNotification() {
    this.notify.emit('Notificación desde el Hijo');
  }
}

En la plantilla del componente hijo (my-component.component.html):

<button (click)="sendNotification()">Notificar al Padre</button>

En el componente padre (app.component.html):

<app-my-component (notify)="onNotify($event)"></app-my-component>

En la clase del componente padre (app.component.ts):

export class AppComponent {
  onNotify(message: string) {
    console.log(message);
  }
}

Características Avanzadas del Componente

Ganchos del Ciclo de Vida

Angular proporciona ganchos del ciclo de vida que te permiten intervenir en eventos clave del ciclo de vida de un componente:

  • ngOnInit(): Llamado una vez que el componente está inicializado.
  • ngOnChanges(): Llamado cuando una propiedad de entrada cambia.
  • ngOnDestroy(): Llamado justo antes de que el componente sea destruido.

Ejemplo:

import { Component, OnInit, OnDestroy } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent implements OnInit, OnDestroy {
  ngOnInit() {
    console.log('Componente Inicializado');
  }

  ngOnDestroy() {
    console.log('Componente Destruido');
  }
}

ViewEncapsulation

Angular proporciona tres estrategias de encapsulación:

  • Emulated: Predeterminado, emula el comportamiento nativo del shadow DOM.
  • None: Sin encapsulación, los estilos son globales.
  • ShadowDom: Usa el shadow DOM nativo.

Ejemplo:

import { Component, ViewEncapsulation } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css'],
  encapsulation: ViewEncapsulation.Emulated // Predeterminado
})
export class MyComponent {}

Conclusión

Los componentes son fundamentales para las aplicaciones Angular. Encapsulan la vista, la lógica y los estilos, lo que facilita la gestión y reutilización del código. Al comprender cómo crear e interactuar con los componentes, puedes construir aplicaciones Angular complejas y escalables. Recuerda aprovechar Angular CLI para una creación eficiente de componentes y usar los ganchos del ciclo de vida para gestionar el comportamiento del componente de manera efectiva.
