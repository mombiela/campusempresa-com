Añadiendo Funcionalidades Avanzadas
Pruebas y Despliegue ►

En esta sección, exploraremos cómo añadir funcionalidades avanzadas a tus aplicaciones Angular. Esto incluirá temas como la carga diferida, la gestión de estado con NgRx y técnicas avanzadas de enrutamiento. Al final de esta sección, deberías ser capaz de mejorar el rendimiento y la escalabilidad de tus aplicaciones Angular.

Carga Diferida

La carga diferida es un patrón de diseño que retrasa la carga de recursos hasta que realmente se necesitan. Esto puede mejorar significativamente el rendimiento de tu aplicación Angular al reducir el tiempo de carga inicial.

Conceptos Clave

  • Módulos: Las aplicaciones Angular están divididas en módulos. La carga diferida te permite cargar estos módulos solo cuando son necesarios.
  • Rutas: La carga diferida a menudo se implementa utilizando el sistema de enrutamiento de Angular.

Ejemplo

Primero, crea un nuevo módulo que quieras cargar de forma diferida:

ng generate module lazy --route lazy --module app.module

Este comando creará un nuevo módulo llamado LazyModule y lo configurará para que se cargue de forma diferida.

A continuación, configura las rutas en app-routing.module.ts:

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: 'lazy',
    loadChildren: () => import('./lazy/lazy.module').then(m => m.LazyModule)
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Ahora, el LazyModule solo se cargará cuando el usuario navegue a la ruta /lazy.

Gestión de Estado con NgRx

NgRx es una biblioteca de gestión de estado para aplicaciones Angular, inspirada en Redux. Ayuda a gestionar el estado de tu aplicación de una manera predecible.

Conceptos Clave

  • Store: La única fuente de verdad para el estado de tu aplicación.
  • Acciones: Eventos que describen algo que ocurrió en la aplicación.
  • Reductores: Funciones que manejan acciones y actualizan el estado.
  • Selectores: Funciones que seleccionan una parte del estado.

Ejemplo

Primero, instala NgRx:

ng add @ngrx/store

Crea una acción en src/app/state/counter.actions.ts:

import { createAction } from '@ngrx/store';

export const increment = createAction('[Counter Component] Increment');
export const decrement = createAction('[Counter Component] Decrement');
export const reset = createAction('[Counter Component] Reset');

Crea un reductor en src/app/state/counter.reducer.ts:

import { createReducer, on } from '@ngrx/store';
import { increment, decrement, reset } from './counter.actions';

export const initialState = 0;

const _counterReducer = createReducer(initialState,
  on(increment, state => state + 1),
  on(decrement, state => state - 1),
  on(reset, state => 0),
);

export function counterReducer(state, action) {
  return _counterReducer(state, action);
}

Registra el reductor en app.module.ts:

import { StoreModule } from '@ngrx/store';
import { counterReducer } from './state/counter.reducer';

@NgModule({
  imports: [
    // otros imports
    StoreModule.forRoot({ count: counterReducer })
  ],
  // otras configuraciones
})
export class AppModule { }

Usa el store en un componente:

import { Component } from '@angular/core';
import { Store } from '@ngrx/store';
import { increment, decrement, reset } from './state/counter.actions';

@Component({
  selector: 'app-counter',
  template: `
    <button (click)="increment()">Incrementar</button>
    <div>Cuenta Actual: {{ count$ | async }}</div>
    <button (click)="decrement()">Decrementar</button>
    <button (click)="reset()">Reiniciar</button>
  `
})
export class CounterComponent {
  count$ = this.store.select('count');

  constructor(private store: Store<{ count: number }>) {}

  increment() {
    this.store.dispatch(increment());
  }

  decrement() {
    this.store.dispatch(decrement());
  }

  reset() {
    this.store.dispatch(reset());
  }
}

Técnicas Avanzadas de Enrutamiento

Las técnicas avanzadas de enrutamiento pueden ayudarte a crear aplicaciones Angular más dinámicas y flexibles.

Conceptos Clave

  • Guardias: Se utilizan para controlar el acceso a las rutas.
  • Resolutores: Se utilizan para pre-cargar datos antes de navegar a una ruta.
  • Rutas Hijas: Rutas anidadas que permiten estructuras de enrutamiento más complejas.

Ejemplo

Crea una guardia en src/app/auth.guard.ts:

import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, UrlTree } from '@angular/router';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): Observable<boolean | UrlTree> | Promise<boolean | UrlTree> | boolean | UrlTree {
    // Implementa tu lógica de autenticación aquí
    return true; // o false basado en la autenticación
  }
}

Configura la guardia en app-routing.module.ts:

import { AuthGuard } from './auth.guard';

const routes: Routes = [
  {
    path: 'protected',
    component: ProtectedComponent,
    canActivate: [AuthGuard]
  }
];

Conclusión

Añadir funcionalidades avanzadas a tu aplicación Angular puede mejorar enormemente su rendimiento, escalabilidad y experiencia de usuario. Al implementar la carga diferida, la gestión de estado con NgRx y técnicas avanzadas de enrutamiento, puedes crear aplicaciones más eficientes y mantenibles. Continúa explorando estas funcionalidades e intégralas en tus proyectos para aprovechar al máximo el poder de Angular.
