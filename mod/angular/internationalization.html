Internacionalización (i18n) en Angular
Angular y Web Workers ►

La internacionalización (i18n) es el proceso de diseñar y preparar tu aplicación para que sea utilizable en diferentes idiomas y regiones sin requerir cambios de ingeniería en el código fuente. Angular proporciona un soporte robusto para i18n, lo que facilita la creación de aplicaciones multilingües.

Introducción a la Internacionalización (i18n)

  • Definición: La internacionalización es el proceso de diseñar una aplicación para que pueda adaptarse a varios idiomas y regiones sin requerir cambios en el código.
  • Importancia: Permite que tu aplicación llegue a una audiencia más amplia al soportar múltiples idiomas y formatos regionales.

Configuración de Angular para i18n

Para comenzar con i18n en Angular, necesitas configurar tu proyecto para soportar traducciones.

Paso 1: Instalar Angular CLI

Asegúrate de tener instalado Angular CLI:

npm install -g @angular/cli

Paso 2: Crear un Nuevo Proyecto Angular

Crea un nuevo proyecto Angular si aún no tienes uno:

ng new my-i18n-app
cd my-i18n-app

Paso 3: Añadir Marcadores i18n en Plantillas

Usa los atributos i18n de Angular para marcar el texto para traducción en tus plantillas:

<h1 i18n="@@homeTitle">Welcome to My App</h1>
<p i18n="@@homeDescription">This is a sample application for demonstrating i18n in Angular.</p>
  • Atributo i18n: Marca el texto para traducción.
  • @@id: Identificador personalizado opcional para la unidad de traducción.

Extracción de Cadenas de Traducción

Después de marcar el texto para traducción, necesitas extraer estas cadenas en un archivo de traducción.

Paso 1: Extraer Cadenas

Usa Angular CLI para extraer las cadenas marcadas:

ng extract-i18n

Este comando genera un archivo messages.xlf en el directorio raíz de tu proyecto.

Paso 2: Traducir Cadenas

Traduce las cadenas extraídas en el archivo messages.xlf:

<trans-unit id="homeTitle" datatype="html">
  <source>Welcome to My App</source>
  <target>Bienvenido a Mi Aplicación</target>
</trans-unit>
<trans-unit id="homeDescription" datatype="html">
  <source>This is a sample application for demonstrating i18n in Angular.</source>
  <target>Esta es una aplicación de muestra para demostrar i18n en Angular.</target>
</trans-unit>

Configuración de Angular para Múltiples Locales

Para soportar múltiples locales, necesitas configurar Angular para usar diferentes archivos de traducción según el local del usuario.

Paso 1: Actualizar `angular.json`

Modifica el archivo angular.json para incluir diferentes locales:

"projects": {
  "my-i18n-app": {
    ...
    "i18n": {
      "sourceLocale": "en",
      "locales": {
        "es": "src/locale/messages.es.xlf"
      }
    }
  }
}

Paso 2: Compilar para Diferentes Locales

Compila tu aplicación para diferentes locales:

ng build --localize

Este comando genera compilaciones separadas para cada local especificado en el archivo angular.json.

Cambio Dinámico de Locales

Para aplicaciones más avanzadas, es posible que desees cambiar los locales dinámicamente en tiempo de ejecución.

Paso 1: Instalar Dependencias

Instala los paquetes necesarios de Angular:

npm install @ngx-translate/core @ngx-translate/http-loader

Paso 2: Configurar `app.module.ts`

Configura el cargador de traducción en tu AppModule:

import { HttpClient } from '@angular/common/http';
import { TranslateLoader, TranslateModule } from '@ngx-translate/core';
import { TranslateHttpLoader } from '@ngx-translate/http-loader';

export function HttpLoaderFactory(http: HttpClient) {
  return new TranslateHttpLoader(http);
}

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,
    HttpClientModule,
    TranslateModule.forRoot({
      loader: {
        provide: TranslateLoader,
        useFactory: HttpLoaderFactory,
        deps: [HttpClient]
      }
    })
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Paso 3: Cargar Traducciones en Componentes

Carga y usa traducciones en tus componentes:

import { Component } from '@angular/core';
import { TranslateService } from '@ngx-translate/core';

@Component({
  selector: 'app-root',
  template: `
    <div>
      <h1>{{ 'homeTitle' | translate }}</h1>
      <p>{{ 'homeDescription' | translate }}</p>
      <button (click)="switchLanguage('en')">English</button>
      <button (click)="switchLanguage('es')">Español</button>
    </div>
  `
})
export class AppComponent {
  constructor(private translate: TranslateService) {
    this.translate.setDefaultLang('en');
  }

  switchLanguage(language: string) {
    this.translate.use(language);
  }
}

Conclusión

La internacionalización (i18n) en Angular es una característica poderosa que te permite crear aplicaciones que pueden adaptarse fácilmente a diferentes idiomas y regiones. Siguiendo los pasos descritos en esta guía, puedes configurar tu aplicación Angular para i18n, extraer y traducir cadenas, configurar múltiples locales e incluso cambiar locales dinámicamente en tiempo de ejecución. Esto asegura que tu aplicación pueda llegar a una audiencia global con un esfuerzo mínimo.
