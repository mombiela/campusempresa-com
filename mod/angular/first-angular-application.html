Introducción

En esta sección, te guiaremos a través de la creación de tu primera aplicación Angular. Cubriremos los conceptos básicos de la configuración del entorno Angular, la creación de un nuevo proyecto y la comprensión de la estructura del proyecto. Al final de esta sección, tendrás una aplicación Angular simple ejecutándose en tu máquina local.

Configuración del Entorno Angular

Antes de que puedas crear una aplicación Angular, necesitas configurar tu entorno de desarrollo.

Requisitos Previos

  • Node.js y npm: Angular requiere Node.js y npm (Node Package Manager). Puedes descargarlos e instalarlos desde nodejs.org.
  • Angular CLI: La Interfaz de Línea de Comandos de Angular (CLI) es una herramienta poderosa para inicializar, desarrollar, esbozar y mantener aplicaciones Angular.

Instalación de Angular CLI

Para instalar Angular CLI, abre tu terminal o símbolo del sistema y ejecuta el siguiente comando:

npm install -g @angular/cli

Este comando instala Angular CLI globalmente en tu sistema.

Creación de un Nuevo Proyecto Angular

Una vez que Angular CLI esté instalado, puedes crear un nuevo proyecto Angular.

Guía Paso a Paso

  1. Abre tu terminal o símbolo del sistema.

  2. Navega al directorio donde deseas crear tu proyecto.

  3. Ejecuta el siguiente comando para crear un nuevo proyecto Angular:

    ng new my-first-angular-app
    • Se te pedirá que elijas si deseas agregar enrutamiento Angular. Escribe y para sí o n para no.
    • También se te pedirá que selecciones un formato de hoja de estilos (CSS, SCSS, SASS, LESS, Stylus). Elige tu opción preferida.

  4. Navega al directorio de tu proyecto:

    cd my-first-angular-app

  5. Sirve la aplicación:

    ng serve
    • Abre tu navegador y navega a http://localhost:4200/. Deberías ver la página de bienvenida predeterminada de Angular.

Comprensión de la Estructura del Proyecto

Cuando creas un nuevo proyecto Angular, Angular CLI genera una estructura de proyecto para ti. Aquí están los componentes clave:

  • e2e/: Pruebas de extremo a extremo para la aplicación.
  • node_modules/: Contiene los paquetes npm instalados para el proyecto.
  • src/: El código fuente de la aplicación.
    • app/: Contiene el módulo principal de la aplicación y el componente.
    • assets/: Contiene activos estáticos como imágenes y estilos.
    • environments/: Contiene archivos de configuración específicos del entorno.
  • angular.json: Archivo de configuración para Angular CLI.
  • package.json: Lista los paquetes npm de los que depende el proyecto.
  • tsconfig.json: Archivo de configuración de TypeScript.

Creación de un Componente Simple

Los componentes son los bloques de construcción de una aplicación Angular. Vamos a crear un componente simple.

Guía Paso a Paso

  1. Genera un nuevo componente:

    ng generate component hello-world

    Este comando crea un nuevo directorio hello-world dentro del directorio app con cuatro archivos:

    • hello-world.component.ts: El archivo TypeScript para la lógica del componente.
    • hello-world.component.html: La plantilla HTML para el componente.
    • hello-world.component.css: El archivo CSS para los estilos del componente.
    • hello-world.component.spec.ts: El archivo de prueba para el componente.

  2. Modifica la plantilla del componente: Abre hello-world.component.html y reemplaza su contenido con:

    <h1>¡Hola, Mundo!</h1>

  3. Usa el componente en la aplicación principal: Abre app.component.html y agrega la siguiente línea:

    <app-hello-world></app-hello-world>

    Esto renderizará el mensaje ¡Hola, Mundo! en tu aplicación principal.

  4. Sirve la aplicación:

    ng serve

    Abre tu navegador y navega a http://localhost:4200/. Deberías ver el mensaje ¡Hola, Mundo!.

Conclusión

¡Felicidades! Has creado con éxito tu primera aplicación Angular. Has aprendido cómo configurar el entorno Angular, crear un nuevo proyecto, comprender la estructura del proyecto y crear un componente simple. Estos son los pasos fundamentales para construir aplicaciones Angular más complejas. En las siguientes secciones, profundizaremos en las características de Angular y las mejores prácticas.
