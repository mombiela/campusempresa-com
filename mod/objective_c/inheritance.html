Herencia en Objective-C
Polimorfismo ►

La herencia es un concepto fundamental en la programación orientada a objetos que permite a una clase heredar propiedades y métodos de otra clase. Esto ayuda en la reutilización del código y en establecer una relación jerárquica entre las clases.

¿Qué es la Herencia?

  • Definición: La herencia permite que una nueva clase (subclase) herede las propiedades y métodos de una clase existente (superclase).
  • Propósito: Promueve la reutilización del código y establece una jerarquía natural entre las clases.

Conceptos Básicos

  • Superclase: La clase cuyas propiedades y métodos son heredados.
  • Subclase: La clase que hereda propiedades y métodos de la superclase.
  • @interface y @implementation: Palabras clave utilizadas para definir clases en Objective-C.

Creando una Superclase

Comencemos creando una superclase simple llamada Person.

// Person.h
#import <Foundation/Foundation.h>

@interface Person : NSObject

@property NSString *name;
@property NSInteger age;

- (void)displayInfo;

@end

// Person.m
#import "Person.h"

@implementation Person

- (void)displayInfo {
    NSLog(@"Name: %@, Age: %ld", self.name, (long)self.age);
}

@end
  • Explicación:
    • @interface Person : NSObject: Declara una nueva clase Person que hereda de NSObject.
    • @property: Declara las propiedades name y age.
    • - (void)displayInfo;: Declara un método para mostrar la información de la persona.
    • @implementation Person: Implementa la clase Person.
    • NSLog: Muestra el nombre y la edad en la consola.

Creando una Subclase

Ahora, creemos una subclase llamada Student que hereda de Person.

// Student.h
#import "Person.h"

@interface Student : Person

@property NSString *schoolName;

- (void)displayStudentInfo;

@end

// Student.m
#import "Student.h"

@implementation Student

- (void)displayStudentInfo {
    [self displayInfo];
    NSLog(@"School: %@", self.schoolName);
}

@end
  • Explicación:
    • @interface Student : Person: Declara una nueva clase Student que hereda de Person.
    • @property: Declara una nueva propiedad schoolName.
    • - (void)displayStudentInfo;: Declara un método para mostrar la información del estudiante.
    • @implementation Student: Implementa la clase Student.
    • [self displayInfo]: Llama al método displayInfo de la superclase Person.

Usando la Subclase

Veamos cómo usar la subclase Student en un programa.

#import <Foundation/Foundation.h>
#import "Student.h"

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Student *student = [[Student alloc] init];
        student.name = @"John Doe";
        student.age = 20;
        student.schoolName = @"XYZ University";
        
        [student displayStudentInfo];
    }
    return 0;
}
  • Explicación:
    • Student *student = [[Student alloc] init];: Crea una instancia de la clase Student.
    • student.name = @"John Doe";: Establece la propiedad name heredada de Person.
    • student.age = 20;: Establece la propiedad age heredada de Person.
    • student.schoolName = @"XYZ University";: Establece la propiedad schoolName.
    • [student displayStudentInfo];: Llama al método displayStudentInfo para mostrar toda la información.

Sobrescribiendo Métodos

Una subclase puede sobrescribir métodos de su superclase para proporcionar implementaciones específicas.

// Student.m
#import "Student.h"

@implementation Student

- (void)displayInfo {
    NSLog(@"Student Name: %@, Age: %ld", self.name, (long)self.age);
}

- (void)displayStudentInfo {
    [self displayInfo];
    NSLog(@"School: %@", self.schoolName);
}

@end
  • Explicación:
    • - (void)displayInfo: Sobrescribe el método displayInfo de Person para proporcionar una implementación específica para Student.

Resumen

  • La herencia permite que una clase herede propiedades y métodos de otra clase.
  • La superclase proporciona las propiedades y métodos base.
  • La subclase puede agregar nuevas propiedades y métodos o sobrescribir los existentes.
  • Esto promueve la reutilización del código y ayuda a crear una estructura jerárquica.

Al comprender y utilizar la herencia, puedes crear un código más organizado y mantenible en Objective-C.
