Introducción

En esta sección, cubriremos cómo implementar varias funciones en Objective-C. Esto incluirá la creación de clases, métodos, propiedades y la comprensión de protocolos y categorías. Al final de esta sección, podrás crear y gestionar funciones complejas en tus proyectos de Objective-C.

Creación de Clases y Objetos

Las clases son el plano para los objetos en Objective-C. Aquí te mostramos cómo puedes crear una clase e instanciar un objeto.

Definición de una Clase

Para definir una clase, necesitas crear un archivo de cabecera (.h) y un archivo de implementación (.m).

Archivo de Cabecera: Person.h

#import <Foundation/Foundation.h>

@interface Person : NSObject

@property (nonatomic, strong) NSString *name;
@property (nonatomic, assign) NSInteger age;

- (void)displayInfo;

@end

Archivo de Implementación: Person.m

#import "Person.h"

@implementation Person

- (void)displayInfo {
    NSLog(@"Name: %@, Age: %ld", self.name, (long)self.age);
}

@end

Instanciación de un Objeto

Puedes crear una instancia de la clase Person y usar sus propiedades y métodos.

Person *person = [[Person alloc] init];
person.name = @"John Doe";
person.age = 30;
[person displayInfo]; // Salida: Name: John Doe, Age: 30

Métodos

Los métodos en Objective-C son funciones que pertenecen a una clase. Pueden ser métodos de instancia o métodos de clase.

Métodos de Instancia

Los métodos de instancia operan sobre una instancia de la clase.

- (void)displayInfo {
    NSLog(@"Name: %@, Age: %ld", self.name, (long)self.age);
}

Métodos de Clase

Los métodos de clase se llaman sobre la clase en sí, no sobre una instancia.

+ (void)classMethodExample {
    NSLog(@"This is a class method");
}

Propiedades

Las propiedades en Objective-C proporcionan una manera de definir los atributos de una clase. Generan automáticamente métodos getter y setter.

Atributos de Propiedad

  • nonatomic: Acceso no atómico.
  • strong: Referencia fuerte, usada para objetos.
  • weak: Referencia débil, usada para evitar ciclos de retención.
  • assign: Usada para tipos de datos primitivos.
@property (nonatomic, strong) NSString *name;
@property (nonatomic, assign) NSInteger age;

Protocolos

Los protocolos en Objective-C son similares a las interfaces en otros lenguajes. Definen una lista de métodos que una clase puede implementar.

Definición de un Protocolo

@protocol Drivable <NSObject>

- (void)startEngine;
- (void)stopEngine;

@end

Implementación de un Protocolo

@interface Car : NSObject <Drivable>

@end

@implementation Car

- (void)startEngine {
    NSLog(@"Engine started");
}

- (void)stopEngine {
    NSLog(@"Engine stopped");
}

@end

Categorías

Las categorías te permiten agregar métodos a clases existentes sin modificar la clase original.

Definición de una Categoría

@interface NSString (ReverseString)

- (NSString *)reverseString;

@end

@implementation NSString (ReverseString)

- (NSString *)reverseString {
    NSUInteger len = [self length];
    NSMutableString *reversedString = [NSMutableString stringWithCapacity:len];
    
    while (len > 0) {
        [reversedString appendString:[NSString stringWithFormat:@"%C", [self characterAtIndex:--len]]];
    }
    
    return reversedString;
}

@end

Uso de una Categoría

NSString *originalString = @"Objective-C";
NSString *reversedString = [originalString reverseString];
NSLog(@"Reversed String: %@", reversedString); // Salida: C-evitcejbO

Conclusión

En esta sección, cubrimos los conceptos básicos de la implementación de funciones en Objective-C, incluyendo la creación de clases, métodos, propiedades, protocolos y categorías. Estos conceptos forman la base para construir aplicaciones robustas y mantenibles en Objective-C. Practica estos conceptos creando tus propias clases y métodos, e intenta implementar protocolos y categorías para extender la funcionalidad de clases existentes.
