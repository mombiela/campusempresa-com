Introducción

En esta sección, describiremos el proyecto en el que trabajarás a lo largo de este curso. Este proyecto te ayudará a aplicar los conceptos que aprendas en un entorno práctico, reforzando tu comprensión de Objective-C desde niveles principiantes hasta avanzados.

Descripción del proyecto

Desarrollarás una Aplicación de Lista de Tareas simple pero completa. Esta aplicación permitirá a los usuarios:

  • Añadir nuevas tareas.
  • Marcar tareas como completadas.
  • Eliminar tareas.
  • Ver una lista de todas las tareas.

Requisitos del proyecto

Para completar con éxito este proyecto, necesitarás cumplir con los siguientes requisitos:

Requisitos funcionales

  • Gestión de tareas: Los usuarios deben poder añadir, editar, eliminar y marcar tareas como completadas.
  • Interfaz de usuario: La aplicación debe tener una interfaz fácil de usar que permita una navegación y gestión de tareas sencillas.
  • Persistencia de datos: Las tareas deben guardarse entre sesiones. Esto significa que cuando el usuario cierre y vuelva a abrir la aplicación, sus tareas deben seguir ahí.

Requisitos no funcionales

  • Rendimiento: La aplicación debe ser receptiva y manejar las entradas del usuario de manera eficiente.
  • Escalabilidad: La aplicación debe poder manejar un número creciente de tareas sin degradación del rendimiento.
  • Mantenibilidad: El código debe estar bien organizado y documentado para facilitar futuras actualizaciones y mantenimiento.

Herramientas y tecnologías

Para construir este proyecto, necesitarás las siguientes herramientas y tecnologías:

  • Xcode: El entorno de desarrollo integrado (IDE) para macOS que contiene un conjunto de herramientas de desarrollo de software.
  • Objective-C: El lenguaje de programación principal utilizado en este curso.
  • Cocoa Touch Framework: Un marco de interfaz de usuario para construir aplicaciones iOS.

Conceptos clave

Aquí hay algunos conceptos clave que necesitarás entender para completar este proyecto:

  • Clases y objetos: Entender cómo definir y usar clases y objetos en Objective-C.
  • Gestión de memoria: Usar la Contabilidad Automática de Referencias (ARC) para gestionar la memoria.
  • Diseño de interfaz de usuario: Usar Interface Builder en Xcode para diseñar la interfaz de usuario.
  • Modelo-Vista-Controlador (MVC): Aplicar el patrón de diseño MVC para estructurar tu código.
  • Persistencia de datos: Usar listas de propiedades, Core Data u otros métodos para guardar datos entre sesiones.

Código de ejemplo

A continuación, se muestra un ejemplo de cómo podrías definir una clase Task simple en Objective-C:

#import <Foundation/Foundation.h>

@interface Task : NSObject

@property (nonatomic, strong) NSString *title;
@property (nonatomic, assign) BOOL isCompleted;

- (instancetype)initWithTitle:(NSString *)title;
- (void)markAsCompleted;

@end

@implementation Task

- (instancetype)initWithTitle:(NSString *)title {
    self = [super init];
    if (self) {
        _title = title;
        _isCompleted = NO;
    }
    return self;
}

- (void)markAsCompleted {
    self.isCompleted = YES;
}

@end

Explicación

  • @interface y @implementation: Estas palabras clave se usan para declarar y definir la clase Task.
  • Propiedades: title y isCompleted son propiedades de la clase Task.
  • Métodos: initWithTitle: es un método inicializador, y markAsCompleted es un método para marcar la tarea como completada.

Conclusión

En esta sección, hemos descrito el proyecto en el que trabajarás, incluyendo los requisitos funcionales y no funcionales, herramientas y tecnologías, y conceptos clave. Al final de este curso, tendrás una aplicación de Lista de Tareas completamente funcional y una sólida comprensión de la programación en Objective-C. Este proyecto servirá como una aplicación práctica de los conceptos que aprendas, ayudándote a reforzar tus habilidades y conocimientos.
