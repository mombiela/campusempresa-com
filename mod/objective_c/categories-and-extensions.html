Introducción

En Objective-C, las categorías y extensiones son características poderosas que te permiten agregar métodos a clases existentes sin modificar su código fuente. Esto puede ser particularmente útil para organizar el código, extender la funcionalidad y mantener bases de código limpias y modulares.

Categorías

Las categorías te permiten agregar métodos a clases existentes. A menudo se utilizan para:

  • Organizar métodos en grupos lógicos.
  • Agregar métodos a clases para las que no tienes el código fuente.
  • Separar detalles de implementación de la interfaz principal de la clase.

Creando una Categoría

Para crear una categoría, necesitas definirla en un archivo de encabezado e implementarla en un archivo de implementación correspondiente.

Archivo de Encabezado (.h)

// NSString+ReversedString.h
#import <Foundation/Foundation.h>

@interface NSString (ReversedString)

- (NSString *)reversedString;

@end

Archivo de Implementación (.m)

// NSString+ReversedString.m
#import "NSString+ReversedString.h"

@implementation NSString (ReversedString)

- (NSString *)reversedString {
    NSUInteger length = [self length];
    NSMutableString *reversedString = [NSMutableString stringWithCapacity:length];
    
    for (NSInteger i = length - 1; i >= 0; i--) {
        [reversedString appendFormat:@"%C", [self characterAtIndex:i]];
    }
    
    return reversedString;
}

@end

Usando una Categoría

Para usar los métodos definidos en una categoría, simplemente importa el archivo de encabezado de la categoría y llama a los métodos en instancias de la clase.

#import "NSString+ReversedString.h"

NSString *originalString = @"Objective-C";
NSString *reversedString = [originalString reversedString];
NSLog(@"Reversed String: %@", reversedString); // Salida: "C-evitcejbO"

Puntos Clave

  • Las categorías no pueden agregar variables de instancia.
  • Los métodos agregados por categorías están disponibles para todas las instancias de la clase.
  • Si un método de categoría tiene el mismo nombre que un método en la clase original, el método de la categoría sobrescribirá el método original.

Extensiones

Las extensiones son similares a las categorías, pero se utilizan para agregar métodos y propiedades privadas a una clase. Normalmente se declaran en el archivo de implementación de la clase que extienden.

Creando una Extensión

Las extensiones se declaran en el archivo de implementación de la clase que extienden y no requieren un archivo de encabezado separado.

Archivo de Implementación (.m)

// MyClass.m
#import "MyClass.h"

@interface MyClass ()

@property (nonatomic, strong) NSString *privateProperty;

- (void)privateMethod;

@end

@implementation MyClass

- (void)publicMethod {
    NSLog(@"Public method called");
    [self privateMethod];
}

- (void)privateMethod {
    NSLog(@"Private method called");
}

@end

Usando una Extensión

Las extensiones se utilizan internamente dentro de la clase que extienden y no son visibles para otras clases.

MyClass *myObject = [[MyClass alloc] init];
[myObject publicMethod]; // Salida: "Public method called" seguido de "Private method called"
// [myObject privateMethod]; // Esto causará un error en tiempo de compilación

Puntos Clave

  • Las extensiones pueden agregar tanto métodos como propiedades.
  • Los métodos y propiedades agregados por extensiones son privados para la clase.
  • Las extensiones se utilizan típicamente para ocultar detalles de implementación.

Resumen

Las categorías y extensiones son herramientas esenciales en Objective-C para extender y organizar tu código. Las categorías te permiten agregar métodos a clases existentes, facilitando la gestión y extensión de la funcionalidad sin alterar la clase original. Las extensiones, por otro lado, proporcionan una forma de agregar métodos y propiedades privadas, ayudando a encapsular detalles de implementación y mantener una interfaz limpia.

Al comprender y utilizar categorías y extensiones, puedes escribir código Objective-C más modular, mantenible y organizado.
