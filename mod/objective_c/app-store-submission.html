Introducción

Enviar una aplicación a la App Store es un paso crucial en el proceso de desarrollo de aplicaciones. Esta sección te guiará a través de los pasos necesarios para preparar tu aplicación en Objective-C para su envío, asegurando que cumpla con las directrices y requisitos de Apple.

Requisitos previos

Antes de comenzar el proceso de envío, asegúrate de tener lo siguiente:

  • Una cuenta de desarrollador de Apple.
  • Una aplicación completa que cumpla con las Directrices de Revisión de la App Store de Apple.
  • Activos necesarios para la aplicación (iconos, capturas de pantalla, etc.).
  • Un Mac con Xcode instalado.

Preparando tu aplicación para el envío

App Store Connect

App Store Connect es la plataforma donde gestionas tus aplicaciones, incluyendo el envío, actualizaciones y análisis.

  1. Crear una cuenta en App Store Connect:

    • Ve a App Store Connect e inicia sesión con tu cuenta de desarrollador de Apple.

  2. Registrar tu aplicación:

    • Navega a "Mis Apps" y haz clic en el botón "+" para agregar una nueva aplicación.
    • Rellena la información requerida como Nombre de la App, Idioma Principal, ID del Paquete y SKU.

Información de la aplicación

Rellena los detalles necesarios sobre tu aplicación:

  • Nombre de la App: El nombre de tu aplicación tal como aparecerá en la App Store.
  • Descripción: Una descripción detallada de las características y funcionalidades de tu aplicación.
  • Palabras clave: Palabras clave que describen tu aplicación para mejorar su búsqueda.
  • URL de soporte: Una URL donde los usuarios pueden obtener soporte para tu aplicación.
  • URL de marketing: Una URL para el marketing de tu aplicación (opcional).

Activos de la aplicación

Prepara y sube los activos requeridos:

  • Icono de la App: Asegúrate de que el icono de tu aplicación cumpla con las especificaciones de Apple.
  • Capturas de pantalla: Proporciona capturas de pantalla para todos los tamaños de dispositivos que tu aplicación soporta.
  • Vista previa de la App (opcional): Un video corto que demuestre tu aplicación.

Versión de la aplicación

Especifica el número de versión de tu aplicación:

  • Número de versión: Sigue la versión semántica (por ejemplo, 1.0.0).
  • Número de compilación: Incrementa este número con cada nueva compilación que envíes.

Configurando tu proyecto de Xcode

Identificadores de la aplicación y perfiles de aprovisionamiento

Asegúrate de que tu proyecto de Xcode esté configurado correctamente:

  • Identificador del paquete: Este debe coincidir con el ID del Paquete que registraste en App Store Connect.
  • Perfil de aprovisionamiento: Asegúrate de tener el perfil de aprovisionamiento correcto para la distribución.

Firma de código

La firma de código es esencial para el envío de la aplicación:

  • Firma automática: Habilita la firma automática en Xcode para que Xcode gestione tus certificados y perfiles.
  • Firma manual: Si prefieres la firma manual, asegúrate de tener los certificados y perfiles de aprovisionamiento correctos.

Archivando tu aplicación

Crea un archivo de tu aplicación para su envío:

  1. Abre tu proyecto en Xcode.
  2. Selecciona un dispositivo objetivo (no un simulador).
  3. Ve a Product > Archive.
  4. Una vez creado el archivo, se abrirá la ventana del Organizador.

Subiendo tu aplicación

Sube tu aplicación a App Store Connect:

  1. En la ventana del Organizador, selecciona tu archivo y haz clic en "Distribute App".
  2. Elige "App Store Connect" como el método de distribución.
  3. Sigue las indicaciones para validar y subir tu aplicación.

Enviando tu aplicación para revisión

Envío a App Store Connect

  1. Una vez que tu aplicación esté subida, vuelve a App Store Connect.
  2. Selecciona tu aplicación y navega a la pestaña "App Store".
  3. Completa los campos restantes, como precios y disponibilidad.
  4. Envía tu aplicación para revisión.

Proceso de revisión

Apple revisará tu aplicación para asegurarse de que cumpla con sus directrices:

  • Tiempo de revisión: El proceso de revisión puede tardar unos días a una semana.
  • Comentarios: Si tu aplicación es rechazada, Apple proporcionará comentarios sobre lo que necesita ser corregido.

Conclusión

Enviar una aplicación a la App Store implica varios pasos, desde preparar tu aplicación y activos hasta configurar tu proyecto de Xcode y finalmente enviar tu aplicación para revisión. Siguiendo este proceso estructurado, puedes asegurar un envío sin problemas y aumentar las posibilidades de que tu aplicación sea aprobada. Recuerda leer y cumplir cuidadosamente con las directrices de Apple para evitar errores comunes y rechazos.
