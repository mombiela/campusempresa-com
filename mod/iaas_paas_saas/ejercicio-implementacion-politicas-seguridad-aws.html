Introducción

En este ejercicio, aprenderemos a implementar políticas de seguridad en AWS utilizando IAM (Identity and Access Management). Las políticas de seguridad son esenciales para controlar el acceso a los recursos en la nube y garantizar que solo los usuarios autorizados puedan realizar acciones específicas.

Objetivos del Ejercicio

  • Comprender el concepto de políticas de seguridad en AWS.
  • Crear y asignar políticas de IAM.
  • Aplicar políticas a usuarios, grupos y roles.
  • Verificar la efectividad de las políticas implementadas.

Requisitos Previos

  • Cuenta de AWS activa.
  • Conocimientos básicos de AWS IAM.
  • Acceso a la consola de administración de AWS.

Pasos para Implementar Políticas de Seguridad en AWS

1. Crear una Política de IAM

Para crear una política de IAM, sigue estos pasos:

  1. Inicia sesión en la consola de administración de AWS.
  2. Navega a IAM.
  3. En el panel de navegación, selecciona Policies.
  4. Haz clic en Create policy.
  5. Utiliza el editor visual o JSON para definir la política.

Ejemplo de Política JSON

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "s3:ListBucket",
            "Resource": "arn:aws:s3:::example_bucket"
        },
        {
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:PutObject"
            ],
            "Resource": "arn:aws:s3:::example_bucket/*"
        }
    ]
}

Esta política permite listar el contenido del bucket example_bucket y obtener y poner objetos dentro de ese bucket.

2. Asignar la Política a un Usuario

  1. En el panel de navegación de IAM, selecciona Users.
  2. Elige el usuario al que deseas asignar la política.
  3. En la pestaña Permissions, haz clic en Add permissions.
  4. Selecciona Attach policies directly.
  5. Busca y selecciona la política que creaste anteriormente.
  6. Haz clic en Next: Review y luego en Add permissions.

3. Verificar la Política

Para verificar que la política se ha aplicado correctamente:

  1. Inicia sesión con el usuario al que asignaste la política.
  2. Intenta realizar acciones permitidas y no permitidas según la política.
  3. Verifica que solo las acciones permitidas en la política se pueden realizar.

Ejercicio Práctico

Tarea 1: Crear una Política de Solo Lectura para S3

  1. Crea una política que permita solo acciones de lectura (s3:GetObject) en un bucket específico.
  2. Asigna esta política a un nuevo usuario.

Tarea 2: Crear una Política de Administración para EC2

  1. Crea una política que permita todas las acciones (ec2:*) en los recursos de EC2.
  2. Asigna esta política a un grupo de administradores.

Tarea 3: Verificar Políticas

  1. Inicia sesión con los usuarios correspondientes y verifica que las políticas aplicadas funcionan como se espera.
  2. Documenta cualquier problema encontrado y cómo lo solucionaste.

Conclusión

En este ejercicio, hemos aprendido a crear y asignar políticas de seguridad en AWS utilizando IAM. Las políticas de seguridad son fundamentales para proteger los recursos en la nube y garantizar que solo los usuarios autorizados puedan acceder a ellos. Practicar la creación y asignación de políticas te ayudará a gestionar mejor la seguridad en tus entornos de AWS.
