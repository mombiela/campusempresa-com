Introducción a los Shaders
Texturas y Mapeo de Texturas ►

Los shaders son una parte fundamental de la programación moderna de OpenGL. Te permiten controlar la tubería gráfica y crear efectos visuales impresionantes. Esta sección te introducirá a los shaders, sus tipos y cómo usarlos en OpenGL.

¿Qué son los Shaders?

Los shaders son pequeños programas que se ejecutan en la GPU. Están escritos en GLSL (OpenGL Shading Language) y se utilizan para controlar la tubería de renderizado. Hay dos tipos principales de shaders en OpenGL:

  • Vertex Shaders: Procesan los atributos de cada vértice.
  • Fragment Shaders: Procesan el color de cada píxel.

Vertex Shaders

Los vertex shaders son el primer paso en la tubería gráfica. Procesan los atributos de cada vértice, como la posición, el color y las coordenadas de textura.

Ejemplo de Vertex Shader

#version 330 core
layout(location = 0) in vec3 aPos; // La variable de posición tiene la posición de atributo 0

void main()
{
    gl_Position = vec4(aPos, 1.0); // Establece la variable de salida a la posición de entrada
}
  • #version 330 core: Especifica la versión de GLSL.
  • layout(location = 0) in vec3 aPos: Define una variable de entrada aPos con ubicación 0.
  • gl_Position: Variable de salida incorporada que determina la posición del vértice.

Fragment Shaders

Los fragment shaders se ejecutan para cada píxel y determinan el color final del píxel.

Ejemplo de Fragment Shader

#version 330 core
out vec4 FragColor; // Variable de salida

void main()
{
    FragColor = vec4(1.0, 0.5, 0.2, 1.0); // Establece el color de salida a naranja
}
  • out vec4 FragColor: Define una variable de salida FragColor.
  • vec4(1.0, 0.5, 0.2, 1.0): Establece el color a naranja (formato RGBA).

Compilación y Enlace de Shaders

Para usar shaders en OpenGL, necesitas compilarlos y enlazarlos en un programa de shaders.

Código de Ejemplo para Compilar y Enlazar Shaders

// Código fuente del vertex shader
const char* vertexShaderSource = "#version 330 core\n"
    "layout(location = 0) in vec3 aPos;\n"
    "void main()\n"
    "{\n"
    "   gl_Position = vec4(aPos, 1.0);\n"
    "}\0";

// Código fuente del fragment shader
const char* fragmentShaderSource = "#version 330 core\n"
    "out vec4 FragColor;\n"
    "void main()\n"
    "{\n"
    "   FragColor = vec4(1.0, 0.5, 0.2, 1.0);\n"
    "}\0";

// Compilar vertex shader
unsigned int vertexShader = glCreateShader(GL_VERTEX_SHADER);
glShaderSource(vertexShader, 1, &vertexShaderSource, NULL);
glCompileShader(vertexShader);

// Comprobar errores de compilación del vertex shader
int success;
char infoLog[512];
glGetShaderiv(vertexShader, GL_COMPILE_STATUS, &success);
if (!success)
{
    glGetShaderInfoLog(vertexShader, 512, NULL, infoLog);
    std::cout << "ERROR::SHADER::VERTEX::COMPILATION_FAILED\n" << infoLog << std::endl;
}

// Compilar fragment shader
unsigned int fragmentShader = glCreateShader(GL_FRAGMENT_SHADER);
glShaderSource(fragmentShader, 1, &fragmentShaderSource, NULL);
glCompileShader(fragmentShader);

// Comprobar errores de compilación del fragment shader
glGetShaderiv(fragmentShader, GL_COMPILE_STATUS, &success);
if (!success)
{
    glGetShaderInfoLog(fragmentShader, 512, NULL, infoLog);
    std::cout << "ERROR::SHADER::FRAGMENT::COMPILATION_FAILED\n" << infoLog << std::endl;
}

// Enlazar shaders
unsigned int shaderProgram = glCreateProgram();
glAttachShader(shaderProgram, vertexShader);
glAttachShader(shaderProgram, fragmentShader);
glLinkProgram(shaderProgram);

// Comprobar errores de enlace
glGetProgramiv(shaderProgram, GL_LINK_STATUS, &success);
if (!success) {
    glGetProgramInfoLog(shaderProgram, 512, NULL, infoLog);
    std::cout << "ERROR::SHADER::PROGRAM::LINKING_FAILED\n" << infoLog << std::endl;
}

// Limpiar shaders ya que no son necesarios después del enlace
glDeleteShader(vertexShader);
glDeleteShader(fragmentShader);
  • glCreateShader: Crea un objeto shader.
  • glShaderSource: Establece el código fuente del shader.
  • glCompileShader: Compila el shader.
  • glCreateProgram: Crea un programa de shaders.
  • glAttachShader: Adjunta un shader al programa.
  • glLinkProgram: Enlaza el programa de shaders.
  • glDeleteShader: Elimina el objeto shader después del enlace.

Usando Shaders en OpenGL

Una vez que los shaders están compilados y enlazados en un programa, puedes usarlos en tu bucle de renderizado.

Código de Ejemplo para Usar el Programa de Shaders

// Usar el programa de shaders
glUseProgram(shaderProgram);

// Bucle de renderizado
while (!glfwWindowShouldClose(window))
{
    // Comandos de renderizado aquí...

    // Intercambiar buffers y sondear eventos de IO
    glfwSwapBuffers(window);
    glfwPollEvents();
}
  • glUseProgram: Activa el programa de shaders para el renderizado.

Conclusión

Los shaders son una herramienta poderosa en OpenGL que te permiten controlar la tubería gráfica y crear efectos visuales complejos. Al entender los vertex y fragment shaders, y aprender cómo compilarlos y enlazarlos, puedes comenzar a crear tus propios efectos gráficos personalizados. Practica escribiendo shaders simples y gradualmente avanza hacia shaders más complejos para aprovechar al máximo el poder de los shaders en OpenGL.
