Concurrencia: Tareas
Objetos Protegidos ►

La concurrencia es un concepto fundamental en Ada, que permite a los programas realizar múltiples operaciones simultáneamente. Este tema te guiará a través de los conceptos básicos de las tareas en Ada, desde la creación de tareas simples hasta mecanismos de control de concurrencia más avanzados.

Introducción a las Tareas

En Ada, las tareas son el medio principal para lograr la concurrencia. Una tarea es un hilo de control separado que puede ejecutarse concurrentemente con otras tareas.

Conceptos Clave

  • Declaración de Tareas: Cómo declarar una tarea en Ada.
  • Cuerpo de la Tarea: La implementación del comportamiento de la tarea.
  • Activación de la Tarea: Cuándo y cómo las tareas comienzan a ejecutarse.
  • Terminación de la Tarea: Cómo las tareas completan su ejecución.

Declaración y Cuerpo de la Tarea

Para declarar una tarea en Ada, se utiliza la palabra clave task. El cuerpo de la tarea define lo que hará la tarea cuando se ejecute.

Ejemplo

task My_Task is
    -- Declaración de la tarea
end My_Task;

task body My_Task is
begin
    -- Cuerpo de la tarea (implementación)
    for I in 1 .. 10 loop
        Ada.Text_IO.Put_Line("La tarea está corriendo: " & Integer'Image(I));
    end loop;
end My_Task;

En este ejemplo:

  • My_Task se declara como una tarea.
  • El cuerpo de la tarea contiene un bucle que imprime un mensaje diez veces.

Activación de la Tarea

Las tareas se activan automáticamente cuando el bloque o subprograma que las contiene se elabora. Esto significa que las tareas comienzan a ejecutarse tan pronto como el programa alcanza el punto donde se declara la tarea.

Ejemplo

procedure Main is
    task My_Task is
    end My_Task;

    task body My_Task is
    begin
        Ada.Text_IO.Put_Line("¡La tarea está activada!");
    end My_Task;
begin
    Ada.Text_IO.Put_Line("Procedimiento principal iniciado.");
end Main;

Salida:

Procedimiento principal iniciado.
¡La tarea está activada!

Comunicación entre Tareas

Las tareas a menudo necesitan comunicarse entre sí. Ada proporciona varios mecanismos para la comunicación entre tareas, incluyendo objetos protegidos y rendezvous.

Objetos Protegidos

Los objetos protegidos proporcionan una forma segura para que las tareas compartan datos. Aseguran que solo una tarea pueda acceder a los datos protegidos a la vez.

Ejemplo

protected Shared_Data is
    procedure Write(Value : Integer);
    function Read return Integer;
private
    Data : Integer := 0;
end Shared_Data;

protected body Shared_Data is
    procedure Write(Value : Integer) is
    begin
        Data := Value;
    end Write;

    function Read return Integer is
    begin
        return Data;
    end Read;
end Shared_Data;

task Writer is
end Writer;

task body Writer is
begin
    Shared_Data.Write(42);
end Writer;

task Reader is
end Reader;

task body Reader is
begin
    Ada.Text_IO.Put_Line("Valor leído: " & Integer'Image(Shared_Data.Read));
end Reader;

En este ejemplo:

  • Shared_Data es un objeto protegido con un procedimiento para escribir datos y una función para leer datos.
  • La tarea Writer escribe un valor en Shared_Data.
  • La tarea Reader lee el valor de Shared_Data.

Rendezvous

Rendezvous es un mecanismo donde una tarea puede llamar a una entrada de otra tarea, permitiéndoles sincronizarse y intercambiar datos.

Ejemplo

task Server is
    entry Request(Value : Integer);
end Server;

task body Server is
begin
    accept Request(Value : Integer) do
        Ada.Text_IO.Put_Line("El servidor recibió: " & Integer'Image(Value));
    end Request;
end Server;

task Client is
end Client;

task body Client is
begin
    Server.Request(99);
end Client;

En este ejemplo:

  • La tarea Server tiene una entrada Request que puede ser llamada por otras tareas.
  • La tarea Client llama a la entrada Request de Server, pasando el valor 99.

Conclusión

La concurrencia en Ada es una característica poderosa que te permite escribir programas que pueden realizar múltiples tareas simultáneamente. Al entender la declaración de tareas, activación, comunicación y mecanismos de sincronización como objetos protegidos y rendezvous, puedes gestionar efectivamente las operaciones concurrentes en tus programas Ada.

En las siguientes secciones, profundizaremos en mecanismos avanzados de control de concurrencia y mejores prácticas para escribir programas concurrentes eficientes y seguros en Ada.
