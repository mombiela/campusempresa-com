Interfaz con Otros Lenguajes
Sistemas en Tiempo Real ►

La interfaz con otros lenguajes es una habilidad crucial para los programadores de Ada, permitiéndoles aprovechar bibliotecas y sistemas existentes escritos en diferentes lenguajes de programación. Este tema cubrirá los conceptos básicos de la interfaz de Ada con C, ya que es el escenario más común, y proporcionará información sobre la interfaz con otros lenguajes.

Introducción a la Interfaz

La interfaz permite que los programas Ada llamen funciones y usen bibliotecas escritas en otros lenguajes, y viceversa. Esto es particularmente útil para:

  • Reutilizar código existente.
  • Aprovechar bibliotecas especializadas.
  • Integrarse con APIs a nivel de sistema.

Interfaz con C

C es el lenguaje más común con el que Ada se interconecta debido a su uso generalizado y compatibilidad. Ada proporciona una forma sencilla de interconectar con C a través de las directivas pragma Import, pragma Export y pragma Convention.

Llamar Funciones de C desde Ada

Para llamar una función de C desde Ada, necesitas:

  1. Declarar la función de C en Ada.
  2. Usar pragma Import para enlazar la función de C.

Ejemplo

Supongamos que tienes una función de C int add(int a, int b);:

Código C:

// add.c
int add(int a, int b) {
    return a + b;
}

Código Ada:

-- add.ads
with Interfaces.C; use Interfaces.C;

function Add (A, B : C.int) return C.int;
pragma Import (C, Add, "add");

-- main.adb
with Ada.Text_IO; use Ada.Text_IO;
with Add;

procedure Main is
begin
    Put_Line ("5 + 3 = " & Integer'Image (Add (5, 3)));
end Main;

Exportar Funciones de Ada a C

Para llamar una función de Ada desde C, necesitas:

  1. Declarar la función de Ada.
  2. Usar pragma Export para hacer la función disponible para C.

Ejemplo

Supongamos que quieres llamar una función de Ada desde C:

Código Ada:

-- my_functions.ads
with Interfaces.C; use Interfaces.C;

procedure Print_Message (Message : in C.char_array);
pragma Export (C, Print_Message, "print_message");

-- my_functions.adb
with Ada.Text_IO; use Ada.Text_IO;

procedure Print_Message (Message : in Interfaces.C.char_array) is
begin
    Ada.Text_IO.Put_Line (Message.all);
end Print_Message;

Código C:

// main.c
extern void print_message(const char* message);

int main() {
    print_message("Hello from C!");
    return 0;
}

Mapeo de Tipos de Datos

Al interconectar con C, es esencial mapear los tipos de Ada a sus correspondientes tipos de C. Aquí hay una tabla para los mapeos de tipos comunes:

Interfaz con Otros Lenguajes

Aunque C es el lenguaje más común con el que Ada se interconecta, también es posible interconectar con otros lenguajes como Fortran, C++ y Java. El proceso es similar pero puede requerir consideraciones adicionales para características específicas del lenguaje.

Interfaz con Fortran

Fortran se usa a menudo en la computación científica. Ada puede interconectarse con Fortran usando pragmas similares:

procedure Fortran_Subroutine;
pragma Import (Fortran, Fortran_Subroutine, "fortran_subroutine");

Interfaz con C++

La interfaz con C++ puede ser más compleja debido a características como clases y mangling de nombres. Sin embargo, las llamadas a funciones básicas pueden manejarse de manera similar a C:

procedure Cpp_Function;
pragma Import (C, Cpp_Function, "_Z12cpp_functionv"); -- Nombre mangled

Interfaz con Java

La interfaz con Java generalmente implica usar la Interfaz Nativa de Java (JNI). Esto es más avanzado y requiere entender las convenciones de JNI.

Conclusión

Interconectar Ada con otros lenguajes, especialmente C, es una característica poderosa que te permite extender las capacidades de tus programas Ada. Al entender cómo declarar y enlazar funciones, mapear tipos de datos y usar las pragmas apropiadas, puedes integrar efectivamente Ada con una variedad de otros lenguajes de programación. Esta habilidad es invaluable para aprovechar bases de código y bibliotecas existentes, haciendo tus aplicaciones Ada más versátiles y poderosas.
