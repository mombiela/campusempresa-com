Introducción

El paso de parámetros es un concepto fundamental en la programación que te permite pasar datos a subprogramas (procedimientos y funciones). En Ada, el paso de parámetros es versátil y proporciona varios modos para controlar cómo se pasan y manipulan los datos.

Modos de Parámetros

Ada admite tres modos de parámetros principales:

  • In: Pasa datos al subprograma.
  • Out: Pasa datos fuera del subprograma.
  • In Out: Pasa datos tanto al subprograma como fuera de él.

Modo In

El modo in se utiliza cuando deseas pasar datos a un subprograma sin permitir que el subprograma modifique los datos originales.

procedure Print_Value(Value : in Integer) is
begin
   Put_Line("The value is: " & Integer'Image(Value));
end Print_Value;
  • Concepto Clave: El parámetro in es de solo lectura dentro del subprograma.

Modo Out

El modo out se utiliza cuando deseas que el subprograma modifique el parámetro y devuelva el resultado.

procedure Increment(Value : out Integer) is
begin
   Value := Value + 1;
end Increment;
  • Concepto Clave: El parámetro out es de solo escritura dentro del subprograma. Se utiliza para devolver un valor al llamador.

Modo In Out

El modo in out permite que el subprograma lea y modifique el parámetro.

procedure Double_Value(Value : in out Integer) is
begin
   Value := Value * 2;
end Double_Value;
  • Concepto Clave: El parámetro in out puede ser leído y modificado dentro del subprograma.

Mecanismos de Paso de Parámetros

Ada utiliza diferentes mecanismos para implementar el paso de parámetros:

  • Por Copia: El valor real se copia al subprograma.
  • Por Referencia: Se pasa una referencia a los datos reales al subprograma.

Por Copia

Normalmente se utiliza para parámetros in. El subprograma trabaja con una copia del parámetro real.

procedure Show_Copy(Value : in Integer) is
   Temp : Integer := Value;
begin
   Temp := Temp + 1;
   Put_Line("Temp value: " & Integer'Image(Temp));
   Put_Line("Original value: " & Integer'Image(Value));
end Show_Copy;

Por Referencia

Normalmente se utiliza para parámetros out y in out. El subprograma trabaja directamente con el parámetro real.

procedure Modify_Reference(Value : in out Integer) is
begin
   Value := Value + 10;
end Modify_Reference;

Ejemplos y Ejercicios

Ejemplo 1: Usando el Modo `in`

procedure Display_Sum(A, B : in Integer) is
begin
   Put_Line("Sum: " & Integer'Image(A + B));
end Display_Sum;

-- Uso
begin
   Display_Sum(5, 10);  -- Salida: Sum: 15
end;

Ejemplo 2: Usando el Modo `out`

procedure Initialize(Value : out Integer) is
begin
   Value := 0;
end Initialize;

-- Uso
declare
   Num : Integer;
begin
   Initialize(Num);
   Put_Line("Initialized value: " & Integer'Image(Num));  -- Salida: Initialized value: 0
end;

Ejemplo 3: Usando el Modo `in out`

procedure Square(Value : in out Integer) is
begin
   Value := Value * Value;
end Square;

-- Uso
declare
   Num : Integer := 4;
begin
   Square(Num);
   Put_Line("Squared value: " & Integer'Image(Num));  -- Salida: Squared value: 16
end;

Ejercicio 1

Escribe un procedimiento Swap que tome dos parámetros in out y cambie sus valores.

procedure Swap(A, B : in out Integer) is
   Temp : Integer;
begin
   Temp := A;
   A := B;
   B := Temp;
end Swap;

-- Prueba tu procedimiento
declare
   X : Integer := 5;
   Y : Integer := 10;
begin
   Swap(X, Y);
   Put_Line("X: " & Integer'Image(X));  -- Salida: X: 10
   Put_Line("Y: " & Integer'Image(Y));  -- Salida: Y: 5
end;

Conclusión

Entender el paso de parámetros en Ada es crucial para escribir subprogramas eficientes y efectivos. Al dominar los modos in, out y in out, puedes controlar cómo se pasan y manipulan los datos dentro de tus programas. Practica con los ejemplos y ejercicios proporcionados para solidificar tu comprensión de estos conceptos.
