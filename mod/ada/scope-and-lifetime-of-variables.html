Alcance y Duración de las Variables en Ada
Arreglos ►

Entender el alcance y la duración de las variables es crucial para escribir programas eficientes y sin errores en Ada. Este tema te guiará a través de los conceptos desde un nivel principiante hasta uno avanzado.

Introducción al Alcance y la Duración

  • Alcance: La región del programa donde una variable es accesible.
  • Duración: El tiempo durante el cual una variable existe en la memoria durante la ejecución del programa.

Alcance de las Variables

Alcance Local

Las variables declaradas dentro de un bloque (por ejemplo, un procedimiento, función o bucle) son locales a ese bloque.

procedure LocalScopeExample is
begin
   declare
      X : Integer := 10;
   begin
      -- X es accesible aquí
      Put_Line("X = " & Integer'Image(X));
   end;
   -- X no es accesible aquí
end LocalScopeExample;
  • Concepto Clave: Las variables declaradas dentro de un bloque no son accesibles fuera de ese bloque.

Alcance Global

Las variables declaradas fuera de cualquier bloque son globales y accesibles desde cualquier parte del programa.

package GlobalScopeExample is
   X : Integer := 10;
end GlobalScopeExample;

with GlobalScopeExample;
procedure UseGlobalScope is
begin
   Put_Line("X = " & Integer'Image(GlobalScopeExample.X));
end UseGlobalScope;
  • Concepto Clave: Las variables globales pueden ser accedidas desde cualquier parte del programa que with el paquete.

Alcances Anidados

Ada soporta alcances anidados, donde los bloques internos pueden acceder a variables de los bloques externos.

procedure NestedScopeExample is
   X : Integer := 10;
begin
   declare
      Y : Integer := 20;
   begin
      -- Tanto X como Y son accesibles aquí
      Put_Line("X = " & Integer'Image(X) & ", Y = " & Integer'Image(Y));
   end;
   -- Solo X es accesible aquí
end NestedScopeExample;
  • Concepto Clave: Los bloques internos pueden acceder a variables de los bloques externos, pero no viceversa.

Duración de las Variables

Duración Estática

Las variables globales tienen una duración estática, lo que significa que existen durante toda la duración del programa.

package StaticLifetimeExample is
   X : Integer := 10;
end StaticLifetimeExample;

with StaticLifetimeExample;
procedure UseStaticLifetime is
begin
   Put_Line("X = " & Integer'Image(StaticLifetimeExample.X));
end UseStaticLifetime;
  • Concepto Clave: Las variables globales se crean cuando el programa comienza y se destruyen cuando el programa termina.

Duración Dinámica

Las variables locales tienen una duración dinámica, lo que significa que existen solo durante la ejecución del bloque en el que se declaran.

procedure DynamicLifetimeExample is
begin
   declare
      X : Integer := 10;
   begin
      -- X existe aquí
      Put_Line("X = " & Integer'Image(X));
   end;
   -- X ya no existe aquí
end DynamicLifetimeExample;
  • Concepto Clave: Las variables locales se crean cuando se entra en el bloque y se destruyen cuando se sale del bloque.

Conceptos Avanzados

Tipos de Acceso y Asignación en el Heap

En Ada, puedes usar tipos de acceso (punteros) para la asignación dinámica de memoria, lo que permite que las variables tengan una duración más allá del alcance en el que fueron creadas.

procedure AccessTypeExample is
   type Int_Ptr is access all Integer;
   Ptr : Int_Ptr;
begin
   Ptr := new Integer'(10);
   Put_Line("Value = " & Integer'Image(Ptr.all));
   -- La memoria debe ser desalojada manualmente
   Free(Ptr);
end AccessTypeExample;
  • Concepto Clave: Las variables asignadas en el heap usando tipos de acceso existen hasta que son desalojadas explícitamente.

Tipos Controlados

Los tipos controlados en Ada te permiten definir comportamientos personalizados de inicialización, finalización y asignación, lo cual puede ser útil para gestionar recursos.

with Ada.Finalization; use Ada.Finalization;

procedure ControlledTypeExample is
   type My_Controlled is new Controlled with record
      X : Integer := 10;
   end record;

   procedure Initialize(Object : in out My_Controlled) is
   begin
      Put_Line("Initializing X to " & Integer'Image(Object.X));
   end Initialize;

   procedure Finalize(Object : in out My_Controlled) is
   begin
      Put_Line("Finalizing X with value " & Integer'Image(Object.X));
   end Finalize;

   Obj : My_Controlled;
begin
   -- Obj se inicializa y finaliza automáticamente
end ControlledTypeExample;
  • Concepto Clave: Los tipos controlados proporcionan ganchos para la inicialización y finalización personalizada, ayudando a gestionar la duración de recursos complejos.

Resumen

  • Alcance determina dónde se puede acceder a una variable.
    • Alcance local: Las variables son accesibles solo dentro del bloque en el que se declaran.
    • Alcance global: Las variables son accesibles en todo el programa.
    • Alcance anidado: Los bloques internos pueden acceder a variables de los bloques externos.
  • Duración determina cuánto tiempo existe una variable en la memoria.
    • Duración estática: Las variables globales existen durante toda la duración del programa.
    • Duración dinámica: Las variables locales existen solo durante la ejecución de su bloque.
    • Los tipos de acceso y los tipos controlados proporcionan mecanismos avanzados para gestionar la duración de las variables.

Entender estos conceptos es esencial para escribir programas robustos y mantenibles en Ada.
