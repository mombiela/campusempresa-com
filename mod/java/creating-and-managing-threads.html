Introducción a los Hilos

Los hilos permiten que un programa opere de manera más eficiente al hacer múltiples cosas al mismo tiempo. Son particularmente útiles en Java para realizar tareas en segundo plano, mejorar el rendimiento y gestionar eventos asíncronos.

  • Definición: Un hilo es un proceso ligero que puede ejecutarse concurrentemente con otros hilos dentro del mismo programa.
  • Casos de Uso: Tareas en segundo plano, procesamiento paralelo, manejo de múltiples solicitudes de clientes en un servidor, etc.

Creación de Hilos en Java

Hay dos formas principales de crear un hilo en Java:

  1. Extendiendo la clase Thread
  2. Implementando la interfaz Runnable

Extendiendo la Clase `Thread`

Para crear un hilo extendiendo la clase Thread, debes seguir estos pasos:

  1. Extender la clase Thread.
  2. Sobrescribir el método run().
  3. Crear una instancia de tu clase y llamar al método start().
class MyThread extends Thread {
    public void run() {
        System.out.println("El hilo está corriendo.");
    }

    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start(); // Esto llamará al método run()
    }
}

Implementando la Interfaz `Runnable`

Para crear un hilo implementando la interfaz Runnable, sigue estos pasos:

  1. Implementar la interfaz Runnable.
  2. Sobrescribir el método run().
  3. Crear una instancia de la clase Thread, pasando tu objeto Runnable a su constructor.
  4. Llamar al método start() en la instancia de Thread.
class MyRunnable implements Runnable {
    public void run() {
        System.out.println("El hilo está corriendo.");
    }

    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        Thread thread = new Thread(myRunnable);
        thread.start(); // Esto llamará al método run()
    }
}

Gestión de Hilos

Gestionar hilos implica controlar su ejecución y asegurarse de que se ejecuten eficientemente sin conflictos.

Ciclo de Vida del Hilo

Un hilo puede estar en uno de los siguientes estados:

  • Nuevo: Cuando un hilo es creado pero aún no ha comenzado.
  • Ejecutable: Cuando el hilo está listo para ejecutarse y esperando tiempo de CPU.
  • Bloqueado/Esperando: Cuando el hilo está esperando un recurso o que otro hilo realice una tarea.
  • Esperando con Tiempo: Cuando el hilo está esperando por un período especificado.
  • Terminado: Cuando el hilo ha terminado de ejecutarse.

Métodos de Hilo

Algunos métodos importantes para gestionar hilos incluyen:

  • start(): Inicia la ejecución del hilo.
  • sleep(long millis): Hace que el hilo duerma por el número especificado de milisegundos.
  • join(): Espera a que el hilo muera.
  • interrupt(): Interrumpe el hilo.
  • isAlive(): Verifica si el hilo aún está corriendo.

Ejemplo: Usando `sleep` y `join`

class MyRunnable implements Runnable {
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("Hilo: " + i);
            try {
                Thread.sleep(1000); // Dormir por 1 segundo
            } catch (InterruptedException e) {
                System.out.println("Hilo interrumpido.");
            }
        }
    }

    public static void main(String[] args) {
        Thread thread = new Thread(new MyRunnable());
        thread.start();
        try {
            thread.join(); // Esperar a que el hilo termine
        } catch (InterruptedException e) {
            System.out.println("Hilo principal interrumpido.");
        }
        System.out.println("Hilo principal terminado.");
    }
}

Sincronización

Cuando múltiples hilos acceden a recursos compartidos, la sincronización es crucial para prevenir la inconsistencia de datos.

Métodos Sincronizados

Puedes usar la palabra clave synchronized para hacer que los métodos sean seguros para hilos.

class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public int getCount() {
        return count;
    }
}

class MyRunnable implements Runnable {
    private Counter counter;

    public MyRunnable(Counter counter) {
        this.counter = counter;
    }

    public void run() {
        for (int i = 0; i < 1000; i++) {
            counter.increment();
        }
    }

    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();
        Thread t1 = new Thread(new MyRunnable(counter));
        Thread t2 = new Thread(new MyRunnable(counter));
        t1.start();
        t2.start();
        t1.join();
        t2.join();
        System.out.println("Cuenta final: " + counter.getCount());
    }
}

Gestión Avanzada de Hilos

Para una gestión más avanzada de hilos, Java proporciona el paquete java.util.concurrent, que incluye clases como ExecutorService, Future y Callable.

Usando `ExecutorService`

El marco ExecutorService simplifica la gestión de hilos al manejar la creación de hilos y la programación de tareas.

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("El hilo está corriendo.");
    }

    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        executor.submit(new MyRunnable());
        executor.submit(new MyRunnable());
        executor.shutdown();
    }
}

Conclusión

Crear y gestionar hilos en Java es esencial para construir aplicaciones eficientes y de alto rendimiento. Al entender los conceptos básicos de la creación de hilos, la sincronización y la gestión avanzada de hilos, puedes desarrollar programas robustos con múltiples hilos. Practica con los ejemplos proporcionados y explora el paquete java.util.concurrent para obtener más características avanzadas.
