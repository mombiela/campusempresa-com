Ciclo de Vida de un Hilo en Java

Introducción

En Java, un hilo es un proceso ligero que permite la ejecución concurrente de código. Comprender el ciclo de vida de un hilo es crucial para escribir aplicaciones multihilo eficientes y efectivas. Esta sección cubrirá los diferentes estados en los que puede estar un hilo, cómo transicionar entre estos estados y proporcionará ejemplos de código para ilustrar estos conceptos.

Estados del Hilo

Un hilo en Java puede estar en uno de los siguientes estados:

  1. Nuevo
  2. Ejecutable
  3. Bloqueado
  4. Esperando
  5. Esperando con Tiempo
  6. Terminado

Nuevo

  • Un hilo está en el estado "Nuevo" cuando se crea pero aún no se ha iniciado.
  • Permanece en este estado hasta que se llama al método start().
Thread thread = new Thread(() -> {
    System.out.println("El hilo está en ejecución");
});
// El hilo está en estado Nuevo

Runnable

  • Un hilo entra en el estado "Runnable" cuando se llama al método start().
  • En este estado, el hilo está listo para ejecutarse y está esperando tiempo de CPU.
thread.start(); // El hilo ahora está en estado Runnable

Bloqueado

  • Un hilo entra en el estado "Bloqueado" cuando está esperando un bloqueo de monitor para entrar en un bloque/método sincronizado.
  • Transiciona de vuelta al estado "Runnable" una vez que adquiere el bloqueo.
synchronized (someObject) {
    // El hilo está en estado Bloqueado si no puede adquirir el bloqueo
}

Esperando

  • Un hilo entra en el estado "Esperando" cuando está esperando indefinidamente a que otro hilo realice una acción particular.
  • Los métodos que pueden causar que un hilo entre en este estado incluyen Object.wait(), Thread.join() y LockSupport.park().
synchronized (someObject) {
    someObject.wait(); // El hilo está en estado Esperando
}

Esperando con Tiempo

  • Un hilo entra en el estado "Esperando con Tiempo" cuando está esperando a que otro hilo realice una acción durante un tiempo de espera especificado.
  • Los métodos que pueden causar que un hilo entre en este estado incluyen Thread.sleep(), Object.wait(long timeout), Thread.join(long millis), LockSupport.parkNanos() y LockSupport.parkUntil().
Thread.sleep(1000); // El hilo está en estado Esperando con Tiempo durante 1 segundo

Terminado

  • Un hilo entra en el estado "Terminado" cuando ha completado su ejecución o ha sido terminado explícitamente.
  • Una vez que un hilo está en este estado, no puede ser reiniciado.
// El hilo está en estado Terminado después de que el método run() se completa

Transiciones del Ciclo de Vida del Hilo

La siguiente tabla resume las transiciones entre los diferentes estados del hilo:

| Estado Actual | Método de Transición | Siguiente Estado | |---------------|-----------------------|------------------------| | Nuevo | start() | Runnable | | Runnable | Adquiere tiempo de CPU| Ejecutando | | Ejecutando | yield() | Runnable | | Ejecutando | wait() | Esperando | | Ejecutando | sleep() | Esperando con Tiempo | | Ejecutando | join() | Esperando/Esperando con Tiempo | | Ejecutando | Completa run() | Terminado | | Esperando | notify()/notifyAll()| Runnable | | Esperando con Tiempo | Expira el tiempo | Runnable | | Bloqueado | Adquiere bloqueo | Runnable |

Ejemplo de Código

A continuación se muestra un ejemplo completo que demuestra los diferentes estados de un hilo:

public class ThreadLifecycleDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            try {
                System.out.println("El hilo está en estado Runnable");
                Thread.sleep(1000); // Estado Esperando con Tiempo
                synchronized (ThreadLifecycleDemo.class) {
                    ThreadLifecycleDemo.class.wait(); // Estado Esperando
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("El hilo está en estado Terminado");
        });

        System.out.println("El hilo está en estado Nuevo");
        thread.start(); // Estado Runnable

        try {
            Thread.sleep(500); // El hilo principal duerme para permitir que el hilo hijo comience
            synchronized (ThreadLifecycleDemo.class) {
                ThreadLifecycleDemo.class.notify(); // Notificar al hilo en espera
            }
            thread.join(); // El hilo principal espera a que el hilo hijo termine
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

Conclusión

Comprender el ciclo de vida del hilo es esencial para gestionar los estados y transiciones del hilo de manera efectiva en una aplicación Java multihilo. Al dominar estos conceptos, puedes escribir programas más eficientes y robustos que hagan un mejor uso de los recursos del sistema.
