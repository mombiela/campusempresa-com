Introducción

Las pruebas y la depuración son habilidades cruciales para cualquier programador de Java. Esta sección te guiará a través de los fundamentos de las pruebas y la depuración, desde conceptos básicos hasta técnicas avanzadas.

Pruebas Unitarias con JUnit

Las pruebas unitarias implican probar componentes individuales de tu código para asegurarte de que funcionen como se espera.

Conceptos Clave

  • Prueba Unitaria: Una prueba que verifica el comportamiento de una pequeña parte de la aplicación, típicamente un solo método.
  • JUnit: Un marco popular para escribir y ejecutar pruebas unitarias en Java.

Escribiendo Tu Primera Prueba Unitaria

Para comenzar con JUnit, necesitas agregar la biblioteca JUnit a tu proyecto. Aquí hay un ejemplo simple:

import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
}

Explicación

  • import org.junit.Test: Importa la anotación Test de JUnit.
  • import static org.junit.Assert.assertEquals: Importa el método assertEquals para las afirmaciones.
  • @Test: Marca el método testAddition como un método de prueba.
  • assertEquals(5, result): Verifica si el resultado es igual a 5.

Ejecutando Pruebas

Puedes ejecutar tus pruebas usando tu IDE o una herramienta de construcción como Maven o Gradle. La mayoría de los IDEs tienen soporte incorporado para ejecutar pruebas JUnit.

Desarrollo Guiado por Pruebas (TDD)

El Desarrollo Guiado por Pruebas es un proceso de desarrollo de software donde escribes pruebas antes de escribir el código real.

Conceptos Clave

  • Rojo-Verde-Refactorizar: El ciclo de TDD:
    1. Rojo: Escribir una prueba que falle.
    2. Verde: Escribir código para que la prueba pase.
    3. Refactorizar: Mejorar el código manteniendo la prueba pasando.

Ejemplo

  1. Rojo: Escribir una prueba que falle. 
    @Test
public void testSubtraction() {
    Calculator calculator = new Calculator();
    int result = calculator.subtract(5, 3);
    assertEquals(2, result);
}
  2. Verde: Escribir código para que la prueba pase. 
    public int subtract(int a, int b) {
    return a - b;
}
  3. Refactorizar: Mejorar el código si es necesario.

Técnicas de Depuración

La depuración es el proceso de identificar y corregir errores en tu código.

Conceptos Clave

  • Puntos de Interrupción: Puntos en tu código donde la ejecución se detendrá, permitiéndote inspeccionar el estado de la aplicación.
  • Paso Sobre: Ejecutar la línea de código actual y detenerse en la siguiente línea.
  • Paso Dentro: Entrar en el método que se está llamando y detenerse en la primera línea de ese método.
  • Paso Fuera: Completar el método actual y detenerse en la línea donde se llamó al método.

Usando un IDE para Depurar

La mayoría de los IDEs modernos proporcionan herramientas de depuración robustas. Aquí te mostramos cómo usarlas:

  1. Establecer un Punto de Interrupción: Haz clic en el margen izquierdo junto al número de línea donde deseas pausar la ejecución.
  2. Iniciar Depuración: Ejecuta tu aplicación en modo de depuración.
  3. Inspeccionar Variables: Pasa el cursor sobre las variables para ver sus valores actuales.
  4. Usar Controles de Depuración: Usa los botones de paso sobre, paso dentro y paso fuera para navegar a través de tu código.

Ejemplo

Considera el siguiente código con un error:

public class Calculator {
    public int divide(int a, int b) {
        return a / b;
    }
}

Para depurar un error de división por cero:

  1. Establece un punto de interrupción en return a / b;.
  2. Ejecuta la aplicación en modo de depuración.
  3. Cuando la ejecución se detenga, inspecciona los valores de a y b.
  4. Identifica que b es cero y corrige el código para manejar este caso.

Técnicas Avanzadas de Pruebas

A medida que te sientas más cómodo con las pruebas y la depuración básicas, puedes explorar técnicas más avanzadas.

Simulación con Mockito

Mockito es un marco que te permite crear objetos simulados para propósitos de prueba.

Ejemplo

import static org.mockito.Mockito.*;
import org.junit.Test;

public class ServiceTest {

    @Test
    public void testService() {
        // Crear un objeto simulado
        Database mockDatabase = mock(Database.class);
        
        // Definir el comportamiento del objeto simulado
        when(mockDatabase.getData()).thenReturn("Mock Data");
        
        // Usar el objeto simulado en tu prueba
        Service service = new Service(mockDatabase);
        String result = service.getData();
        
        assertEquals("Mock Data", result);
    }
}

Explicación

  • mock(Database.class): Crea un objeto simulado de la clase Database.
  • when(mockDatabase.getData()).thenReturn("Mock Data"): Define el comportamiento del objeto simulado.
  • Service service = new Service(mockDatabase): Inyecta el objeto simulado en el servicio.

Conclusión

Las pruebas y la depuración son habilidades esenciales para cualquier programador de Java. Al dominar las pruebas unitarias con JUnit, adoptar el Desarrollo Guiado por Pruebas y utilizar herramientas de depuración en tu IDE, puedes escribir código más confiable y mantenible. A medida que avances, explorar marcos como Mockito para simulación puede mejorar aún más tus capacidades de prueba. Practica estas técnicas regularmente para volverte competente en identificar y corregir problemas en tu código.
