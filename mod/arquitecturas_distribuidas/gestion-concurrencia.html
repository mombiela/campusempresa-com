Gestión de Concurrencia
Bases de Datos NoSQL ►

La gestión de concurrencia es un aspecto crucial en las arquitecturas distribuidas, ya que permite que múltiples procesos o hilos de ejecución operen simultáneamente sin conflictos. Este tema aborda los conceptos fundamentales, técnicas y herramientas utilizadas para manejar la concurrencia en sistemas distribuidos.

Conceptos Clave

  • Concurrencia: La capacidad de un sistema para ejecutar múltiples tareas al mismo tiempo.
  • Paralelismo: La ejecución simultánea de múltiples tareas en diferentes procesadores o núcleos.
  • Condiciones de carrera: Situaciones donde el resultado de un proceso depende del orden de ejecución de otros procesos.
  • Sección crítica: Parte del código que debe ser ejecutada de manera exclusiva por un solo hilo o proceso a la vez.
  • Bloqueo (Lock): Mecanismo para asegurar que solo un hilo o proceso pueda acceder a una sección crítica a la vez.
  • Deadlock (Interbloqueo): Situación donde dos o más procesos se bloquean mutuamente esperando recursos que nunca serán liberados.
  • Starvation (Hambre): Situación donde un proceso nunca obtiene los recursos necesarios para continuar su ejecución.

Técnicas de Gestión de Concurrencia

Bloqueos y Semáforos

  • Mutex (Mutual Exclusion): Un tipo de bloqueo que permite que solo un hilo acceda a una sección crítica a la vez.
  • Semáforos: Variables que se utilizan para controlar el acceso a recursos compartidos. Pueden ser binarios (0 o 1) o contar (permiten un número específico de accesos simultáneos).

Ejemplo de Mutex en Python

import threading

# Recurso compartido
counter = 0
counter_lock = threading.Lock()

def increment_counter():
    global counter
    with counter_lock:
        counter += 1

threads = []
for i in range(10):
    thread = threading.Thread(target=increment_counter)
    threads.append(thread)
    thread.start()

for thread in threads:
    thread.join()

print(f"Final counter value: {counter}")

Monitores

  • Monitores: Estructuras que combinan la mutua exclusión y la sincronización de hilos. Proveen métodos que son ejecutados en exclusión mutua.

Algoritmos de Consenso

  • Paxos: Un algoritmo de consenso que asegura que un grupo de nodos en un sistema distribuido acuerden un valor propuesto.
  • Raft: Un algoritmo de consenso más sencillo y comprensible que Paxos, utilizado para la replicación de logs.

Transacciones Distribuidas

  • Two-Phase Commit (2PC): Un protocolo que asegura que todas las partes de una transacción distribuida acuerden en realizar la transacción o revertirla.
  • Three-Phase Commit (3PC): Una mejora del 2PC que añade una fase adicional para reducir la posibilidad de interbloqueos.

Herramientas y Bibliotecas

  • Zookeeper: Un servicio centralizado para mantener la configuración y proporcionar servicios de sincronización.
  • Redis: Una base de datos en memoria que soporta operaciones atómicas y estructuras de datos avanzadas.
  • Akka: Un toolkit para construir aplicaciones concurrentes y distribuidas en la JVM.

Ejemplo de Concurrencia con Akka en Scala

import akka.actor.{Actor, ActorSystem, Props}

case object Increment
case object GetCount

class Counter extends Actor {
  var count = 0

  def receive = {
    case Increment => count += 1
    case GetCount => sender() ! count
  }
}

object Main extends App {
  val system = ActorSystem("CounterSystem")
  val counter = system.actorOf(Props[Counter], "counter")

  counter ! Increment
  counter ! Increment
  counter ! GetCount

  system.terminate()
}

Conclusión

La gestión de concurrencia es esencial para el correcto funcionamiento de sistemas distribuidos. Comprender y aplicar técnicas como bloqueos, semáforos, monitores y algoritmos de consenso es crucial para evitar problemas como condiciones de carrera, interbloqueos y hambre. Herramientas y bibliotecas como Zookeeper, Redis y Akka facilitan la implementación de soluciones concurrentes y distribuidas, permitiendo a los desarrolladores construir sistemas robustos y eficientes.
