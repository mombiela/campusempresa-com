Introducción

Los sistemas distribuidos son una colección de computadoras independientes que parecen un único sistema coherente para los usuarios. En este tema, exploraremos los diferentes modelos de sistemas distribuidos, sus características, ventajas y desventajas, y cómo se aplican en la práctica.

Modelos de Sistemas Distribuidos

1. Modelo Cliente-Servidor

  • Descripción: En este modelo, los clientes solicitan servicios y los servidores proporcionan esos servicios.
  • Características:
    • Centralización de recursos en el servidor.
    • Clientes ligeros que solicitan servicios.
    • Comunicación a través de protocolos de red.
  • Ventajas:
    • Fácil de gestionar y mantener.
    • Escalabilidad vertical (mejora del servidor).
  • Desventajas:
    • Punto único de fallo.
    • Escalabilidad limitada por el servidor.

Ejemplo de Código

# Ejemplo de un servidor simple en Python usando sockets
import socket

def start_server():
    server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    server_socket.bind(('localhost', 8080))
    server_socket.listen(5)
    print("Servidor iniciado y escuchando en el puerto 8080")

    while True:
        client_socket, address = server_socket.accept()
        print(f"Conexión establecida con {address}")
        client_socket.send(b'Hola desde el servidor!')
        client_socket.close()

start_server()

2. Modelo Peer-to-Peer (P2P)

  • Descripción: En este modelo, cada nodo actúa tanto como cliente como servidor, compartiendo recursos directamente entre ellos.
  • Características:
    • Descentralización.
    • Cada nodo tiene capacidades equivalentes.
    • Comunicación directa entre nodos.
  • Ventajas:
    • Alta disponibilidad y redundancia.
    • Escalabilidad horizontal (añadir más nodos).
  • Desventajas:
    • Gestión y coordinación complejas.
    • Seguridad y confiabilidad pueden ser un problema.

Ejemplo de Código

# Ejemplo de un nodo P2P simple en Python usando sockets
import socket
import threading

def handle_client(client_socket):
    request = client_socket.recv(1024)
    print(f"Recibido: {request.decode('utf-8')}")
    client_socket.send(b'Hola desde el nodo P2P!')
    client_socket.close()

def start_node():
    node_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    node_socket.bind(('localhost', 8081))
    node_socket.listen(5)
    print("Nodo P2P iniciado y escuchando en el puerto 8081")

    while True:
        client_socket, address = node_socket.accept()
        print(f"Conexión establecida con {address}")
        client_handler = threading.Thread(target=handle_client, args=(client_socket,))
        client_handler.start()

start_node()

3. Modelo de Computación en la Nube

  • Descripción: Este modelo proporciona recursos de computación y almacenamiento a través de internet.
  • Características:
    • Servicios bajo demanda.
    • Escalabilidad elástica.
    • Pago por uso.
  • Ventajas:
    • Reducción de costos de infraestructura.
    • Flexibilidad y escalabilidad.
  • Desventajas:
    • Dependencia del proveedor de servicios.
    • Preocupaciones de seguridad y privacidad.

Ejemplo de Uso

# Ejemplo de uso de AWS S3 para almacenamiento en la nube
import boto3

def upload_to_s3(file_name, bucket, object_name=None):
    s3_client = boto3.client('s3')
    try:
        s3_client.upload_file(file_name, bucket, object_name or file_name)
        print(f"Archivo {file_name} subido a {bucket}")
    except Exception as e:
        print(f"Error subiendo archivo: {e}")

upload_to_s3('mi_archivo.txt', 'mi-bucket')

Comparación de Modelos

| Característica | Cliente-Servidor | Peer-to-Peer (P2P) | Computación en la Nube | |----------------------|--------------------------|-------------------------|--------------------------| | Centralización | Sí | No | Sí | | Escalabilidad | Vertical | Horizontal | Elástica | | Punto único de fallo | Sí | No | Dependencia del proveedor| | Gestión | Sencilla | Compleja | Moderada | | Costos | Iniciales altos | Distribuidos | Pago por uso |

Conclusión

Los modelos de sistemas distribuidos ofrecen diferentes enfoques para la gestión y distribución de recursos. La elección del modelo adecuado depende de las necesidades específicas del sistema, incluyendo la escalabilidad, la gestión, la seguridad y los costos. Comprender estos modelos es crucial para diseñar arquitecturas distribuidas eficientes y efectivas.
