Introducción a JUnit

JUnit es un popular marco de pruebas para Java que permite a los desarrolladores escribir y ejecutar pruebas repetibles. Es una herramienta esencial para garantizar la calidad y fiabilidad del código.

Conceptos Clave

  • Caso de Prueba: Una unidad única de prueba.
  • Suite de Pruebas: Una colección de casos de prueba.
  • Aserciones: Declaraciones que verifican si una condición es verdadera.
  • Anotaciones: Marcadores especiales en el código que proporcionan metadatos al marco de JUnit.

Configuración de JUnit

Antes de que puedas comenzar a escribir pruebas, necesitas configurar JUnit en tu proyecto.

Añadiendo JUnit a Tu Proyecto

Para un proyecto Maven, añade la siguiente dependencia a tu archivo pom.xml:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-engine</artifactId>
    <version>5.7.0</version>
    <scope>test</scope>
</dependency>

Para un proyecto Gradle, añade lo siguiente a tu archivo build.gradle:

testImplementation 'org.junit.jupiter:junit-jupiter-engine:5.7.0'

Escribiendo Tu Primera Prueba

Comencemos con un caso de prueba simple.

Ejemplo: Probando un Método de Suma

Supongamos que tenemos un método simple que suma dos enteros:

public class Calculator {
    public int sum(int a, int b) {
        return a + b;
    }
}

Ahora, escribamos una prueba para este método:

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    public void testSum() {
        Calculator calculator = new Calculator();
        int result = calculator.sum(2, 3);
        assertEquals(5, result, "2 + 3 debería ser igual a 5");
    }
}

Explicación

  • @Test: Marca el método como un caso de prueba.
  • assertEquals(expected, actual, message): Verifica si el valor actual coincide con el valor esperado.

Técnicas Avanzadas de Pruebas

Pruebas Parametrizadas

JUnit te permite ejecutar la misma prueba con diferentes parámetros.

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

public class CalculatorTest {

    @ParameterizedTest
    @CsvSource({
        "1, 1, 2",
        "2, 3, 5",
        "3, 7, 10"
    })
    public void testSum(int a, int b, int expected) {
        Calculator calculator = new Calculator();
        assertEquals(expected, calculator.sum(a, b));
    }
}

Explicación

  • @ParameterizedTest: Marca el método como una prueba parametrizada.
  • @CsvSource: Proporciona los parámetros para la prueba.

Pruebas de Excepciones

También puedes probar si tu código lanza correctamente excepciones.

import static org.junit.jupiter.api.Assertions.assertThrows;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    public void testDivideByZero() {
        Calculator calculator = new Calculator();
        assertThrows(ArithmeticException.class, () -> calculator.divide(1, 0));
    }
}

Explicación

  • assertThrows(expectedType, executable): Verifica si el ejecutable lanza la excepción esperada.

Mejores Prácticas

  • Escribe Pruebas Claras y Concisas: Cada prueba debe centrarse en un solo comportamiento.
  • Usa Nombres Significativos: Los nombres de los métodos de prueba deben describir claramente lo que están probando.
  • Mantén las Pruebas Independientes: Las pruebas no deben depender unas de otras.
  • Ejecuta Pruebas Frecuentemente: Integra las pruebas en tu flujo de trabajo de desarrollo.

Resumen

JUnit es un marco poderoso que te ayuda a escribir y ejecutar pruebas para tu código Java. Al entender los conceptos básicos de los casos de prueba, las aserciones y las anotaciones, puedes comenzar a escribir pruebas efectivas. A medida que te sientas más cómodo, puedes explorar técnicas avanzadas como las pruebas parametrizadas y las pruebas de excepciones. Seguir las mejores prácticas garantizará que tus pruebas sean mantenibles y fiables.
