Migrar de JUnit 4 a JUnit 5
Pruebas de Aplicaciones Web ►

Introducción

Migrar de JUnit 4 a JUnit 5 puede parecer desalentador, pero ofrece numerosos beneficios como una arquitectura mejorada, nuevas características y mejor extensibilidad. Esta sección te guiará a través del proceso de migración paso a paso, asegurando una transición sin problemas.

Diferencias Clave Entre JUnit 4 y JUnit 5

Antes de sumergirse en el proceso de migración, es esencial entender las diferencias clave entre JUnit 4 y JUnit 5.

  • Arquitectura: JUnit 5 está compuesto por tres sub-proyectos: JUnit Platform, JUnit Jupiter y JUnit Vintage.
  • Anotaciones: Algunas anotaciones han cambiado o se han añadido.
  • Aserciones y Suposiciones: Mejoradas y más flexibles.
  • Modelo de Extensión: Más poderoso y extensible.

Tabla Comparativa

| Característica/Aspecto | JUnit 4 | JUnit 5 | |------------------------|----------------------------------|----------------------------------| | Arquitectura | Monolítico | Modular (Platform, Jupiter, Vintage) | | Anotaciones | @Test, @Before, @After | @Test, @BeforeEach, @AfterEach, @DisplayName | | Aserciones | assertEquals, assertTrue | assertEquals, assertTrue, assertAll, assertThrows | | Suposiciones | assumeTrue, assumeFalse | assumeTrue, assumeFalse, assumingThat | | Modelo de Extensión | Reglas | Extensiones (más poderosas) |

Proceso de Migración Paso a Paso

1. Actualizar Dependencias

Primero, actualiza las dependencias de tu proyecto para incluir JUnit 5. Si estás usando Maven, actualiza tu pom.xml:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-engine</artifactId>
    <version>5.8.1</version>
    <scope>test</scope>
</dependency>

Para Gradle, actualiza tu build.gradle:

testImplementation 'org.junit.jupiter:junit-jupiter-engine:5.8.1'

2. Reemplazar Anotaciones

Reemplaza las anotaciones de JUnit 4 con sus equivalentes en JUnit 5:

  • @Before -> @BeforeEach
  • @After -> @AfterEach
  • @BeforeClass -> @BeforeAll
  • @AfterClass -> @AfterAll
  • @Ignore -> @Disabled

Ejemplo

JUnit 4:

import org.junit.Before;
import org.junit.After;
import org.junit.Test;

public class ExampleTest {
    @Before
    public void setUp() {
        // código de configuración
    }

    @After
    public void tearDown() {
        // código de limpieza
    }

    @Test
    public void testSomething() {
        // código de prueba
    }
}

JUnit 5:

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.AfterEach;
import org.junit.jupiter.api.Test;

public class ExampleTest {
    @BeforeEach
    public void setUp() {
        // código de configuración
    }

    @AfterEach
    public void tearDown() {
        // código de limpieza
    }

    @Test
    public void testSomething() {
        // código de prueba
    }
}

3. Actualizar Aserciones y Suposiciones

JUnit 5 proporciona aserciones y suposiciones más flexibles.

Ejemplo

JUnit 4:

import static org.junit.Assert.assertEquals;
import static org.junit.Assume.assumeTrue;

public class ExampleTest {
    @Test
    public void testSomething() {
        assumeTrue(System.getProperty("os.name").startsWith("Windows"));
        assertEquals(4, 2 + 2);
    }
}

JUnit 5:

import static org.junit.jupiter.api.Assertions.assertEquals;
import static org.junit.jupiter.api.Assumptions.assumeTrue;

public class ExampleTest {
    @Test
    public void testSomething() {
        assumeTrue(System.getProperty("os.name").startsWith("Windows"));
        assertEquals(4, 2 + 2);
    }
}

4. Utilizar JUnit Vintage

Si tienes una gran base de código y no puedes migrar todas las pruebas de una vez, usa JUnit Vintage para ejecutar pruebas de JUnit 4 junto con pruebas de JUnit 5.

Configuración de Maven

<dependency>
    <groupId>org.junit.vintage</groupId>
    <artifactId>junit-vintage-engine</artifactId>
    <version>5.8.1</version>
    <scope>test</scope>
</dependency>

Configuración de Gradle

testImplementation 'org.junit.vintage:junit-vintage-engine:5.8.1'

Conclusión

Migrar de JUnit 4 a JUnit 5 implica actualizar dependencias, reemplazar anotaciones y aprovechar nuevas características y mejoras. Siguiendo los pasos descritos en esta guía, puedes asegurar una transición sin problemas y aprovechar las poderosas capacidades que ofrece JUnit 5.
