Introducción a JUnit y Maven

JUnit es un popular marco de pruebas para aplicaciones Java, mientras que Maven es una herramienta de automatización de compilación utilizada principalmente para proyectos Java. Combinar JUnit con Maven permite una gestión eficiente de dependencias y pruebas automatizadas.

Configuración de Maven para JUnit

Para comenzar a usar JUnit con Maven, necesitas configurar un proyecto Maven e incluir JUnit como una dependencia.

Creación de un Proyecto Maven

  1. Usando la Línea de Comandos:

    mvn archetype:generate -DgroupId=com.example -DartifactId=JUnitWithMaven -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false

    Este comando crea un nuevo proyecto Maven con el ID de grupo y el ID de artefacto especificados.

  2. Usando un IDE (por ejemplo, IntelliJ IDEA, Eclipse):

    • Abre tu IDE y selecciona "Crear Nuevo Proyecto".
    • Elige "Maven" como el tipo de proyecto.
    • Rellena los detalles necesarios (ID de Grupo, ID de Artefacto, etc.) y finaliza la configuración.

Añadiendo la Dependencia de JUnit

Abre el archivo pom.xml en tu proyecto Maven y añade la dependencia de JUnit:

<dependencies>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter-engine</artifactId>
        <version>5.8.1</version>
        <scope>test</scope>
    </dependency>
</dependencies>

Esta configuración añade JUnit 5 a tu proyecto, que es la última versión de JUnit.

Escribiendo Tu Primera Prueba

Crea una clase Java simple y escribe una prueba para ella usando JUnit.

Ejemplo de Clase Java

package com.example;

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

Ejemplo de Prueba JUnit

Crea una clase de prueba en el directorio src/test/java:

package com.example;

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {
    @Test
    public void testAdd() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 debería ser igual a 5");
    }
}

Ejecutando Pruebas con Maven

Para ejecutar tus pruebas, usa el siguiente comando Maven:

mvn test

Este comando compila tu código y ejecuta todas las pruebas en el directorio src/test/java.

Características Avanzadas de JUnit con Maven

Explora características más avanzadas de JUnit y cómo gestionarlas con Maven.

Pruebas Parametrizadas

Las pruebas parametrizadas te permiten ejecutar la misma prueba con diferentes parámetros.

Ejemplo

package com.example;

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

public class CalculatorTest {
    @ParameterizedTest
    @CsvSource({
        "1, 1, 2",
        "2, 3, 5",
        "3, 7, 10"
    })
    public void testAdd(int a, int b, int expected) {
        Calculator calculator = new Calculator();
        assertEquals(expected, calculator.add(a, b));
    }
}

Suites de Pruebas

Las suites de pruebas te permiten agrupar múltiples clases de prueba y ejecutarlas juntas.

Ejemplo

package com.example;

import org.junit.platform.suite.api.SelectClasses;
import org.junit.platform.suite.api.Suite;

@Suite
@SelectClasses({CalculatorTest.class, AnotherTest.class})
public class AllTests {
}

Integración Continua con Maven

Integra tu proyecto Maven con una herramienta de CI como Jenkins para automatizar tu proceso de pruebas.

Configuración de Jenkins

  1. Instalar Jenkins: Sigue la guía oficial de instalación de Jenkins.
  2. Crear un Nuevo Trabajo:
    • Selecciona "Nuevo Elemento" y elige "Proyecto de estilo libre".
    • Configura la gestión del código fuente (por ejemplo, Git).
    • Añade un paso de compilación para ejecutar los objetivos de Maven (clean test).

Conclusión

Combinar JUnit con Maven agiliza el proceso de escribir, gestionar y ejecutar pruebas en proyectos Java. Al configurar un proyecto Maven, añadir dependencias de JUnit y utilizar características avanzadas de JUnit, puedes asegurar que tu código sea robusto y confiable. Integrar con herramientas de CI como Jenkins mejora aún más tu flujo de trabajo de desarrollo al automatizar el proceso de pruebas.
