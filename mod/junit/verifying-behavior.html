En esta sección, exploraremos cómo verificar el comportamiento de tu código utilizando JUnit. Verificar el comportamiento es un aspecto crucial de las pruebas unitarias, ya que asegura que tus métodos no solo produzcan la salida correcta, sino que también interactúen con otros componentes de la manera esperada.

Conceptos Clave

  • Verificación de Comportamiento: Asegurarse de que un método interactúe correctamente con sus dependencias.
  • Mocks y Stubs: Herramientas para simular el comportamiento de objetos complejos.
  • Aserciones: Declaraciones que verifican si una condición es verdadera.

Uso de Mocks y Stubs

Los mocks y stubs son esenciales para aislar la unidad de código que estás probando. Permiten simular el comportamiento de objetos complejos y enfocarse en las interacciones.

Creación de Mocks

Para crear un objeto mock, puedes usar bibliotecas como Mockito. Aquí tienes un ejemplo simple:

import static org.mockito.Mockito.*;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        // Crear un objeto mock de UserRepository
        UserRepository mockRepository = mock(UserRepository.class);

        // Definir el comportamiento del objeto mock
        when(mockRepository.findUserById(1)).thenReturn(new User(1, "John Doe"));

        // Usar el objeto mock en tu prueba
        UserService userService = new UserService(mockRepository);
        User user = userService.getUser(1);

        // Verificar el comportamiento
        assertEquals("John Doe", user.getName());
        verify(mockRepository).findUserById(1);
    }
}

En este ejemplo:

  • Creamos un objeto mock de UserRepository.
  • Definimos el comportamiento del objeto mock usando when y thenReturn.
  • Usamos el objeto mock en nuestra clase UserService.
  • Verificamos el comportamiento usando verify.

Stubbing de Métodos

El stubbing se usa para especificar el comportamiento de un objeto mock. Es particularmente útil cuando necesitas simular diferentes escenarios.

import static org.mockito.Mockito.*;

public class UserServiceTest {

    @Test
    public void testGetUserNotFound() {
        // Crear un objeto mock de UserRepository
        UserRepository mockRepository = mock(UserRepository.class);

        // Definir el comportamiento del objeto mock para un escenario específico
        when(mockRepository.findUserById(2)).thenReturn(null);

        // Usar el objeto mock en tu prueba
        UserService userService = new UserService(mockRepository);
        User user = userService.getUser(2);

        // Verificar el comportamiento
        assertNull(user);
        verify(mockRepository).findUserById(2);
    }
}

En este ejemplo:

  • Stubbeamos el método findUserById para que devuelva null cuando el ID del usuario es 2.
  • Verificamos que el método fue llamado con el parámetro correcto.

Aserciones

Las aserciones se usan para verificar si una condición es verdadera. JUnit proporciona varios métodos de aserción para verificar el comportamiento de tu código.

Aserciones Comunes

| Método de Aserción | Descripción | |----------------------------|---------------------------------------------| | assertEquals(expected, actual) | Verifica si dos valores son iguales. | | assertTrue(condition) | Verifica si una condición es verdadera. | | assertFalse(condition) | Verifica si una condición es falsa. | | assertNull(object) | Verifica si un objeto es nulo. | | assertNotNull(object) | Verifica si un objeto no es nulo. |

Ejemplo

public class CalculatorTest {

    @Test
    public void testAdd() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);

        // Verificar el resultado usando aserciones
        assertEquals(5, result);
    }

    @Test
    public void testDivide() {
        Calculator calculator = new Calculator();
        int result = calculator.divide(10, 2);

        // Verificar el resultado usando aserciones
        assertEquals(5, result);
    }
}

En este ejemplo:

  • Usamos assertEquals para verificar que los métodos add y divide devuelvan los resultados esperados.

Conclusión

Verificar el comportamiento es un aspecto fundamental de las pruebas unitarias con JUnit. Al usar mocks, stubs y aserciones, puedes asegurarte de que tus métodos interactúen correctamente con sus dependencias y produzcan los resultados esperados. Esto no solo ayuda a identificar errores temprano, sino que también hace que tu código sea más robusto y mantenible.
