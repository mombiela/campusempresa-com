Pruebas de Código Legado
Descripción del Proyecto ►

Introducción

Probar código legado puede ser un desafío, pero es crucial para mantener y mejorar la calidad de tu software. El código legado a menudo carece de pruebas, lo que dificulta su refactorización o extensión sin introducir errores. Esta sección te guiará a través del proceso de probar código legado usando JUnit, desde entender los conceptos básicos hasta implementar técnicas avanzadas.

Conceptos Clave

  • Código Legado: Código heredado de una versión anterior del software que puede no tener pruebas.
  • Refactorización: El proceso de reestructurar el código existente sin cambiar su comportamiento externo.
  • Cobertura de Pruebas: Una medida de cuánto del código es probado por pruebas automatizadas.
  • Mocking: Crear objetos que simulan el comportamiento de objetos reales de manera controlada.

Desafíos de Probar Código Legado

  • Falta de pruebas existentes.
  • Acoplamiento y dependencias estrechas.
  • Mala documentación.
  • Riesgo de introducir errores durante la refactorización.

Estrategias para Probar Código Legado

  1. Pruebas de Caracterización: Escribir pruebas para entender el comportamiento actual del código.
  2. Refactorización Incremental: Refactorizar el código gradualmente mientras se añaden pruebas.
  3. Inyección de Dependencias: Usar inyección de dependencias para hacer el código más testeable.
  4. Mocking y Stubbing: Usar mocks y stubs para aislar el código bajo prueba.

Escribir Pruebas de Caracterización

Las pruebas de caracterización te ayudan a entender lo que hace el código legado. Capturan el comportamiento actual, lo cual puede ser útil al refactorizar.

Ejemplo

public class LegacyCalculator {
    public int add(int a, int b) {
        return a + b;
    }
}

// Prueba de Caracterización
import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class LegacyCalculatorTest {
    @Test
    public void testAdd() {
        LegacyCalculator calculator = new LegacyCalculator();
        assertEquals(5, calculator.add(2, 3));
    }
}

Explicación:

  • Creamos una prueba simple para el método add para capturar su comportamiento actual.
  • Esta prueba asegura que cualquier cambio futuro en el método add no alterará su comportamiento esperado.

Refactorización Incremental

Refactoriza el código en pequeños pasos mientras aseguras que las pruebas existentes pasen. Este enfoque minimiza el riesgo de introducir errores.

Ejemplo

Refactorizar un método estrechamente acoplado introduciendo inyección de dependencias.

Antes de la Refactorización:

public class OrderService {
    private Database database = new Database();

    public void placeOrder(Order order) {
        database.save(order);
    }
}

Después de la Refactorización:

public class OrderService {
    private Database database;

    public OrderService(Database database) {
        this.database = database;
    }

    public void placeOrder(Order order) {
        database.save(order);
    }
}

Prueba para el Código Refactorizado

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class OrderServiceTest {
    @Test
    public void testPlaceOrder() {
        Database mockDatabase = mock(Database.class);
        OrderService orderService = new OrderService(mockDatabase);
        Order order = new Order();

        orderService.placeOrder(order);

        verify(mockDatabase).save(order);
    }
}

Explicación:

  • Refactorizamos OrderService para usar inyección de dependencias.
  • Creamos un objeto mock Database para probar OrderService en aislamiento.

Mocking y Stubbing

Usa frameworks de mocking como Mockito para crear mocks y stubs para dependencias. Esto te permite probar el código en aislamiento.

Ejemplo con Mockito

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class PaymentServiceTest {
    @Test
    public void testProcessPayment() {
        PaymentGateway mockGateway = mock(PaymentGateway.class);
        PaymentService paymentService = new PaymentService(mockGateway);
        Payment payment = new Payment();

        when(mockGateway.process(payment)).thenReturn(true);

        boolean result = paymentService.processPayment(payment);

        assertTrue(result);
        verify(mockGateway).process(payment);
    }
}

Explicación:

  • Creamos un objeto mock PaymentGateway.
  • Stub del método process para que retorne true.
  • Verificamos que el método process fue llamado con el objeto de pago esperado.

Conclusión

Probar código legado es una habilidad crítica para mantener y mejorar la calidad del software. Al escribir pruebas de caracterización, refactorizar incrementalmente y usar frameworks de mocking, puedes probar y refactorizar efectivamente el código legado. Estas técnicas te ayudarán a asegurar que tu código permanezca confiable y mantenible con el tiempo.
