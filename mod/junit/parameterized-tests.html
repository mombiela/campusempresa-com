Pruebas Parametrizadas
Creación de Suites de Pruebas ►

Las pruebas parametrizadas te permiten ejecutar la misma prueba varias veces con diferentes entradas. Esto es particularmente útil para probar funciones que deben comportarse de la misma manera para una variedad de entradas.

Introducción a las Pruebas Parametrizadas

  • Definición: Las pruebas parametrizadas permiten a un desarrollador ejecutar la misma prueba con diferentes conjuntos de parámetros.
  • Propósito: Ayudan a reducir la duplicación de código y mejorar la cobertura de pruebas.

Configuración de Pruebas Parametrizadas en JUnit

Para usar pruebas parametrizadas en JUnit, necesitas incluir la dependencia junit-jupiter-params en tu proyecto. Aquí hay un ejemplo para un proyecto Maven:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-params</artifactId>
    <version>5.8.1</version>
    <scope>test</scope>
</dependency>

Escribiendo una Prueba Parametrizada

Paso 1: Importar las Anotaciones Necesarias

Necesitas importar las siguientes anotaciones:

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

Paso 2: Crear un Método de Prueba Parametrizada

Aquí hay un ejemplo simple que prueba si un número es par:

import static org.junit.jupiter.api.Assertions.assertTrue;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

public class EvenNumberTest {

    @ParameterizedTest
    @ValueSource(ints = {2, 4, 6, 8, 10})
    void testIsEven(int number) {
        assertTrue(number % 2 == 0);
    }
}
  • Explicación:
    • @ParameterizedTest: Marca el método como una prueba parametrizada.
    • @ValueSource(ints = {2, 4, 6, 8, 10}): Proporciona los parámetros para el método de prueba. En este caso, proporciona un array de enteros.

Pruebas Parametrizadas Avanzadas

Usando `@CsvSource`

Puedes usar @CsvSource para pasar múltiples argumentos a tu método de prueba:

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

public class StringUtilsTest {

    @ParameterizedTest
    @CsvSource({
        "hello, HELLO",
        "world, WORLD",
        "junit, JUNIT"
    })
    void testToUpperCase(String input, String expected) {
        assertEquals(expected, input.toUpperCase());
    }
}
  • Explicación:
    • @CsvSource: Te permite pasar múltiples conjuntos de argumentos al método de prueba.
    • El método de prueba testToUpperCase toma dos parámetros: input y expected.

Usando `@MethodSource`

Para escenarios más complejos, puedes usar @MethodSource para proporcionar parámetros desde un método:

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.MethodSource;
import java.util.stream.Stream;

public class MathUtilsTest {

    @ParameterizedTest
    @MethodSource("provideNumbersForAddition")
    void testAddition(int a, int b, int expected) {
        assertEquals(expected, a + b);
    }

    static Stream<Arguments> provideNumbersForAddition() {
        return Stream.of(
            Arguments.of(1, 1, 2),
            Arguments.of(2, 3, 5),
            Arguments.of(3, 5, 8)
        );
    }
}
  • Explicación:
    • @MethodSource("provideNumbersForAddition"): Especifica el método que proporciona los parámetros.
    • El método provideNumbersForAddition devuelve un Stream de Arguments.

Conclusión

Las pruebas parametrizadas en JUnit son una característica poderosa que puede ayudarte a escribir pruebas más eficientes y completas. Al usar anotaciones como @ValueSource, @CsvSource y @MethodSource, puedes ejecutar fácilmente la misma prueba con diferentes entradas, reduciendo la duplicación de código y mejorando la cobertura de pruebas.

  • Puntos Clave:
    • Las pruebas parametrizadas permiten ejecutar la misma prueba con diferentes entradas.
    • Ayudan a reducir la duplicación de código y mejorar la cobertura de pruebas.
    • JUnit proporciona varias anotaciones (@ValueSource, @CsvSource, @MethodSource) para facilitar las pruebas parametrizadas.

Al dominar las pruebas parametrizadas, puedes mejorar significativamente la robustez y mantenibilidad de tus suites de pruebas.
