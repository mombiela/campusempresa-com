Introducción a JUnit y Gradle

JUnit es un popular marco de pruebas para Java, mientras que Gradle es una poderosa herramienta de automatización de compilación. Combinar JUnit con Gradle permite procesos de prueba eficientes y automatizados.

Configuración de JUnit con Gradle

Para comenzar con JUnit en un proyecto Gradle, necesitas configurar tu archivo build.gradle.

Configuración Paso a Paso

  1. Crear un Proyecto Gradle:

    • Si no tienes un proyecto Gradle, crea uno usando el siguiente comando: 
      gradle init --type java-application

  2. Agregar Dependencia de JUnit:

    • Abre tu archivo build.gradle y agrega la dependencia de JUnit: 
      dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.8.1'
}

  3. Aplicar el Plugin de Java:

    • Asegúrate de que el plugin de Java esté aplicado en tu build.gradle: 
      plugins {
    id 'java'
}

  4. Configurar la Tarea de Prueba:

    • Configura la tarea de prueba para usar la Plataforma JUnit: 
      test {
    useJUnitPlatform()
}

Ejemplo de Archivo `build.gradle`

plugins {
    id 'java'
}

repositories {
    mavenCentral()
}

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.8.1'
}

test {
    useJUnitPlatform()
}

Escribir y Ejecutar Pruebas con JUnit y Gradle

Una vez que tu proyecto esté configurado, puedes comenzar a escribir y ejecutar pruebas.

Escribir una Prueba Simple

Crea una clase de prueba en el directorio src/test/java:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

Ejecutar Pruebas

  • Línea de Comando:
    • Ejecuta las pruebas usando el siguiente comando: 
      gradle test
  • Integración con IDE:
    • La mayoría de los IDEs como IntelliJ IDEA o Eclipse pueden ejecutar tareas de Gradle directamente. Haz clic derecho en la clase o método de prueba y selecciona "Run".

Configuración Avanzada

Para proyectos más complejos, podrías necesitar configuraciones avanzadas.

Configuraciones Personalizadas de Prueba

  • Ejecución Paralela de Pruebas: 
    test {
    maxParallelForks = 4
}
  • Filtrado de Pruebas: 
    test {
    include '**/CalculatorTest.class'
}

Compilaciones de Múltiples Proyectos

Si tienes una compilación de múltiples proyectos, configura cada subproyecto:

subprojects {
    apply plugin: 'java'

    repositories {
        mavenCentral()
    }

    dependencies {
        testImplementation 'org.junit.jupiter:junit-jupiter:5.8.1'
    }

    test {
        useJUnitPlatform()
    }
}

Conclusión

Integrar JUnit con Gradle agiliza el proceso de pruebas, facilitando la escritura, ejecución y gestión de pruebas. Siguiendo los pasos de configuración, puedes aprovechar todo el poder de ambas herramientas para mantener alta calidad y fiabilidad en tus proyectos Java.
