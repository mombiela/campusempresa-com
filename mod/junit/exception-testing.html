Introducción a las Pruebas de Excepciones

Las pruebas de excepciones son un aspecto crucial de las pruebas unitarias que aseguran que tu código maneje las condiciones de error de manera adecuada. En JUnit, puedes probar si tu código lanza las excepciones esperadas bajo condiciones específicas. Esto ayuda a verificar la robustez y fiabilidad de tu aplicación.

Conceptos Clave

  • Manejo de Excepciones: Mecanismo para manejar errores en tiempo de ejecución, asegurando el flujo normal de la aplicación.
  • Excepciones Esperadas: Excepciones específicas que se anticipa que tu código lance bajo ciertas condiciones.
  • Anotaciones de JUnit: Marcadores especiales en JUnit para especificar excepciones esperadas.

Pruebas de Excepciones con JUnit 4

En JUnit 4, puedes usar la anotación @Test con el parámetro expected para especificar la excepción que esperas que una prueba lance.

Ejemplo

import org.junit.Test;
import static org.junit.Assert.*;

public class ExceptionTest {

    @Test(expected = IllegalArgumentException.class)
    public void testExceptionIsThrown() {
        MyClass myClass = new MyClass();
        myClass.methodThatThrowsException();
    }
}

En este ejemplo:

  • @Test(expected = IllegalArgumentException.class): Esta anotación le dice a JUnit que espere una IllegalArgumentException cuando se llame a methodThatThrowsException.
  • Si se lanza la excepción, la prueba pasa. Si no, la prueba falla.

Pruebas de Excepciones con JUnit 5

JUnit 5 introduce el método assertThrows, que proporciona más flexibilidad y control sobre las pruebas de excepciones.

Ejemplo

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertThrows;

public class ExceptionTest {

    @Test
    public void testExceptionIsThrown() {
        MyClass myClass = new MyClass();
        assertThrows(IllegalArgumentException.class, () -> {
            myClass.methodThatThrowsException();
        });
    }
}

En este ejemplo:

  • assertThrows(IllegalArgumentException.class, () -> { ... }): Este método afirma que la expresión lambda proporcionada lanza una IllegalArgumentException.
  • Te permite capturar e inspeccionar la excepción si es necesario.

Pruebas Avanzadas de Excepciones

Para escenarios más avanzados, es posible que necesites verificar el mensaje de la excepción u otras propiedades.

Ejemplo

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class AdvancedExceptionTest {

    @Test
    public void testExceptionMessage() {
        MyClass myClass = new MyClass();
        Exception exception = assertThrows(IllegalArgumentException.class, () -> {
            myClass.methodThatThrowsException();
        });

        String expectedMessage = "Invalid argument";
        String actualMessage = exception.getMessage();

        assertTrue(actualMessage.contains(expectedMessage));
    }
}

En este ejemplo:

  • El método assertThrows captura la excepción.
  • La prueba luego verifica que el mensaje de la excepción contenga el texto esperado.

Resumen

Las pruebas de excepciones son una parte esencial de las pruebas unitarias que aseguran que tu aplicación maneje los errores como se espera. JUnit proporciona diferentes formas de probar excepciones:

  • JUnit 4 usa la anotación @Test(expected = Exception.class).
  • JUnit 5 usa el método más flexible assertThrows.

Dominando las pruebas de excepciones, puedes escribir código más robusto y fiable, asegurando que tu aplicación se comporte correctamente bajo condiciones de error.
