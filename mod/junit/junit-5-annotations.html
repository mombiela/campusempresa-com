JUnit 5 es un potente marco de pruebas para aplicaciones Java. Introduce varias anotaciones que ayudan a escribir y organizar pruebas. Este tema cubrirá las anotaciones esenciales de JUnit 5, sus propósitos y cómo usarlas de manera efectiva.

Anotaciones clave en JUnit 5

JUnit 5 proporciona una variedad de anotaciones para controlar el ciclo de vida de las pruebas y especificar comportamientos de prueba. Aquí están las anotaciones más comúnmente usadas:

  • @Test
  • @BeforeEach
  • @AfterEach
  • @BeforeAll
  • @AfterAll
  • @DisplayName
  • @Disabled
  • @RepeatedTest
  • @ParameterizedTest
  • @ExtendWith

@Test

La anotación @Test se usa para marcar un método como un método de prueba.

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@BeforeEach

La anotación @BeforeEach se usa para especificar un método que debe ejecutarse antes de cada método de prueba.

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @Test
    void addition() {
        assertEquals(2, calculator.add(1, 1));
    }
}

@AfterEach

La anotación @AfterEach se usa para especificar un método que debe ejecutarse después de cada método de prueba.

import org.junit.jupiter.api.AfterEach;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @AfterEach
    void tearDown() {
        calculator = null;
    }

    @Test
    void addition() {
        assertEquals(2, calculator.add(1, 1));
    }
}

@BeforeAll

La anotación @BeforeAll se usa para especificar un método que debe ejecutarse una vez antes de todos los métodos de prueba en la clase. Este método debe ser estático.

import org.junit.jupiter.api.BeforeAll;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @BeforeAll
    static void initAll() {
        // Código de inicialización
    }

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@AfterAll

La anotación @AfterAll se usa para especificar un método que debe ejecutarse una vez después de todos los métodos de prueba en la clase. Este método debe ser estático.

import org.junit.jupiter.api.AfterAll;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @BeforeAll
    static void initAll() {
        // Código de inicialización
    }

    @AfterAll
    static void tearDownAll() {
        // Código de limpieza
    }

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@DisplayName

La anotación @DisplayName se usa para proporcionar un nombre personalizado para el método de prueba.

import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    @DisplayName("Adición de dos números")
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@Disabled

La anotación @Disabled se usa para deshabilitar un método de prueba o una clase de prueba.

import org.junit.jupiter.api.Disabled;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    @Disabled("Deshabilitado hasta que se solucione el error #42")
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@RepeatedTest

La anotación @RepeatedTest se usa para repetir una prueba un número especificado de veces.

import org.junit.jupiter.api.RepeatedTest;

public class CalculatorTest {

    @RepeatedTest(5)
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@ParameterizedTest

La anotación @ParameterizedTest se usa para ejecutar la misma prueba varias veces con diferentes argumentos.

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

public class CalculatorTest {

    @ParameterizedTest
    @ValueSource(ints = {1, 2, 3, 4, 5})
    void isPositive(int number) {
        assertTrue(number > 0);
    }
}

@ExtendWith

La anotación @ExtendWith se usa para registrar extensiones para una clase de prueba o un método de prueba.

import org.junit.jupiter.api.extension.ExtendWith;
import org.junit.jupiter.api.Test;

@ExtendWith(MyExtension.class)
public class CalculatorTest {

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

Resumen

Las anotaciones de JUnit 5 proporcionan una forma poderosa de controlar el ciclo de vida y el comportamiento de tus pruebas. Entender y usar estas anotaciones de manera efectiva puede ayudarte a escribir pruebas más organizadas, legibles y mantenibles. Aquí tienes un resumen rápido de las anotaciones clave cubiertas:

  • @Test: Marca un método como un método de prueba.
  • @BeforeEach: Ejecuta un método antes de cada método de prueba.
  • @AfterEach: Ejecuta un método después de cada método de prueba.
  • @BeforeAll: Ejecuta un método una vez antes de todos los métodos de prueba (debe ser estático).
  • @AfterAll: Ejecuta un método una vez después de todos los métodos de prueba (debe ser estático).
  • @DisplayName: Proporciona un nombre personalizado para el método de prueba.
  • @Disabled: Deshabilita un método o clase de prueba.
  • @RepeatedTest: Repite una prueba un número especificado de veces.
  • @ParameterizedTest: Ejecuta la misma prueba varias veces con diferentes argumentos.
  • @ExtendWith: Registra extensiones para una clase o método de prueba.

Dominando estas anotaciones, puedes aprovechar al máximo las capacidades de JUnit 5 para crear suites de pruebas robustas y eficientes.
