Introducción al TDD

El Desarrollo Guiado por Pruebas (TDD) es un proceso de desarrollo de software donde escribes pruebas antes de escribir el código real. Este enfoque ayuda a asegurar que tu código sea confiable, mantenible y libre de errores.

  • Ciclo Rojo-Verde-Refactorizar:
    1. Rojo: Escribe una prueba que falle.
    2. Verde: Escribe el código mínimo necesario para pasar la prueba.
    3. Refactorizar: Mejora el código mientras mantienes la prueba pasando.

Beneficios del TDD

  • Mejora de la Calidad del Código: Escribir pruebas primero te obliga a pensar en los requisitos y el diseño antes de la implementación.
  • Reducción de Errores: Las pruebas detectan errores temprano en el proceso de desarrollo.
  • Documentación: Las pruebas sirven como documentación de cómo se supone que debe funcionar el código.
  • Confianza en la Refactorización: Con un conjunto de pruebas, puedes refactorizar el código con la confianza de que no romperás la funcionalidad existente.

Configuración de JUnit

Antes de sumergirte en el TDD, necesitas configurar JUnit en tu proyecto.

Configuración de Maven

Agrega la siguiente dependencia a tu archivo pom.xml:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.7.0</version>
    <scope>test</scope>
</dependency>

Configuración de Gradle

Agrega lo siguiente a tu archivo build.gradle:

testImplementation 'org.junit.jupiter:junit-jupiter-api:5.7.0'
testRuntimeOnly 'org.junit.jupiter:junit-jupiter-engine:5.7.0'

Escribiendo tu Primera Prueba

Comencemos con un ejemplo simple para entender el proceso de TDD.

Ejemplo: Suma de Calculadora

  1. Rojo: Escribe una prueba que falle.
import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {
    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        assertEquals(5, calculator.add(2, 3));
    }
}
  • Esta prueba fallará porque la clase Calculator y el método add aún no existen.
  1. Verde: Escribe el código mínimo necesario para pasar la prueba.
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}
  • Ahora, la prueba debería pasar porque el método add devuelve la suma correcta.
  1. Refactorizar: Mejora el código mientras mantienes la prueba pasando.
  • En este ejemplo simple, puede que no haya mucho que refactorizar, pero siempre busca oportunidades para mejorar la legibilidad y mantenibilidad del código.

Conceptos Avanzados de TDD

Mocking y Stubbing

Al escribir pruebas, puede que necesites aislar la unidad de código que estás probando. Aquí es donde entran en juego el mocking y el stubbing.

  • Mocking: Crear una versión falsa de un objeto para controlar su comportamiento.
  • Stubbing: Proporcionar respuestas predefinidas a las llamadas de métodos.

Ejemplo: Usando Mockito

Agrega la dependencia de Mockito a tu proyecto:

<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>3.6.0</version>
    <scope>test</scope>
</dependency>

Ejemplo de mocking de un servicio:

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {
    @Test
    public void testGetUser() {
        UserRepository mockRepo = mock(UserRepository.class);
        when(mockRepo.findById(1)).thenReturn(new User(1, "John Doe"));

        UserService userService = new UserService(mockRepo);
        User user = userService.getUser(1);

        assertEquals("John Doe", user.getName());
    }
}

Pruebas Parametrizadas

JUnit te permite ejecutar la misma prueba con diferentes parámetros.

Ejemplo:

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;
import static org.junit.jupiter.api.Assertions.assertTrue;

public class ParameterizedTestExample {
    @ParameterizedTest
    @ValueSource(strings = {"racecar", "radar", "level"})
    public void testIsPalindrome(String word) {
        assertTrue(isPalindrome(word));
    }

    public boolean isPalindrome(String word) {
        return new StringBuilder(word).reverse().toString().equals(word);
    }
}

Conclusión

El Desarrollo Guiado por Pruebas (TDD) es una metodología poderosa que puede mejorar significativamente la calidad y mantenibilidad de tu código. Al escribir pruebas primero, aseguras que tu código cumpla con los requisitos y esté libre de errores. Usando JUnit, puedes implementar fácilmente TDD en tus proyectos Java, y con técnicas avanzadas como mocking y pruebas parametrizadas, puedes manejar escenarios más complejos. Recuerda el ciclo Rojo-Verde-Refactorizar y aplícalo consistentemente para obtener los beneficios del TDD.
