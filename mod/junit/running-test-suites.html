Ejecutar Conjuntos de Pruebas
Organización de Pruebas ►

En esta sección, exploraremos cómo ejecutar conjuntos de pruebas usando JUnit. Los conjuntos de pruebas te permiten agrupar múltiples clases de prueba y ejecutarlas juntas, lo cual es particularmente útil para gestionar proyectos grandes. Cubriremos los conceptos básicos de la creación y ejecución de conjuntos de pruebas, así como algunas técnicas avanzadas.

¿Qué es un Conjunto de Pruebas?

Un conjunto de pruebas es una colección de casos de prueba que están destinados a ejecutarse juntos. Esto es útil para:

  • Organizar pruebas en grupos lógicos.
  • Ejecutar múltiples pruebas con un solo comando.
  • Asegurarse de que todas las pruebas pasen antes de desplegar el código.

Crear un Conjunto de Pruebas

Para crear un conjunto de pruebas en JUnit, necesitas usar la anotación @Suite junto con las anotaciones @SelectClasses o @SelectPackages.

Usando `@SelectClasses`

La anotación @SelectClasses te permite especificar clases de prueba individuales para incluir en el conjunto.

import org.junit.platform.runner.JUnitPlatform;
import org.junit.platform.suite.api.SelectClasses;
import org.junit.platform.suite.api.Suite;
import org.junit.runner.RunWith;

@RunWith(JUnitPlatform.class)
@Suite
@SelectClasses({TestClass1.class, TestClass2.class})
public class MyTestSuite {
}

En este ejemplo:

  • @RunWith(JUnitPlatform.class) le dice a JUnit que use el ejecutor de la Plataforma JUnit.
  • @Suite indica que esta clase es un conjunto de pruebas.
  • @SelectClasses especifica las clases de prueba para incluir en el conjunto.

Usando `@SelectPackages`

La anotación @SelectPackages te permite incluir todas las clases de prueba en uno o más paquetes.

import org.junit.platform.runner.JUnitPlatform;
import org.junit.platform.suite.api.SelectPackages;
import org.junit.platform.suite.api.Suite;
import org.junit.runner.RunWith;

@RunWith(JUnitPlatform.class)
@Suite
@SelectPackages("com.example.tests")
public class MyPackageTestSuite {
}

En este ejemplo:

  • @SelectPackages("com.example.tests") incluye todas las clases de prueba en el paquete com.example.tests.

Ejecutar el Conjunto de Pruebas

Una vez que hayas creado tu conjunto de pruebas, puedes ejecutarlo usando tu IDE, herramienta de construcción o línea de comandos.

Ejecutar con un IDE

La mayoría de los IDEs como IntelliJ IDEA, Eclipse y NetBeans te permiten ejecutar conjuntos de pruebas directamente:

  • Haz clic derecho en la clase del conjunto de pruebas.
  • Selecciona "Run" o "Run as JUnit Test".

Ejecutar con Maven

Si estás usando Maven, puedes ejecutar tu conjunto de pruebas con el siguiente comando:

mvn test -Dtest=MyTestSuite

Ejecutar con Gradle

Si estás usando Gradle, puedes ejecutar tu conjunto de pruebas con el siguiente comando:

gradle test --tests "MyTestSuite"

Técnicas Avanzadas

Filtrar Pruebas

Puedes filtrar qué pruebas ejecutar dentro de un conjunto usando etiquetas o filtros personalizados.

Usando Etiquetas

Primero, etiqueta tus métodos de prueba:

import org.junit.jupiter.api.Tag;
import org.junit.jupiter.api.Test;

public class TaggedTests {

    @Test
    @Tag("fast")
    void fastTest() {
        // código de prueba rápida
    }

    @Test
    @Tag("slow")
    void slowTest() {
        // código de prueba lenta
    }
}

Luego, configura tu conjunto para incluir o excluir etiquetas específicas:

import org.junit.platform.runner.JUnitPlatform;
import org.junit.platform.suite.api.IncludeTags;
import org.junit.platform.suite.api.SelectClasses;
import org.junit.platform.suite.api.Suite;
import org.junit.runner.RunWith;

@RunWith(JUnitPlatform.class)
@Suite
@SelectClasses(TaggedTests.class)
@IncludeTags("fast")
public class FastTestsSuite {
}

Ejecución Paralela

JUnit 5 soporta la ejecución paralela de pruebas, lo cual puede configurarse en el archivo junit-platform.properties:

junit.jupiter.execution.parallel.enabled = true
junit.jupiter.execution.parallel.mode.default = concurrent

Esta configuración habilitará la ejecución paralela para todas las pruebas, incluyendo aquellas en conjuntos de pruebas.

Conclusión

Ejecutar conjuntos de pruebas en JUnit te permite gestionar y ejecutar múltiples pruebas de manera eficiente. Al agrupar pruebas en conjuntos, puedes asegurar una prueba exhaustiva y agilizar tu flujo de trabajo de desarrollo. Ya sea que estés ejecutando pruebas desde un IDE, Maven o Gradle, JUnit proporciona opciones flexibles para satisfacer tus necesidades. Técnicas avanzadas como el filtrado y la ejecución paralela mejoran aún más tus capacidades de prueba.
