Introducción

Las aplicaciones de Interfaz de Línea de Comandos (CLI) son herramientas poderosas que permiten a los usuarios interactuar con el software a través de comandos de texto. En esta sección, aprenderemos cómo construir una aplicación CLI usando Go. Cubriremos los conceptos básicos, exploraremos bibliotecas populares y construiremos una herramienta CLI simple desde cero.

Conceptos Clave

  • Aplicación CLI: Un programa que procesa comandos ingresados por los usuarios en una terminal o símbolo del sistema.
  • Flags: Argumentos de línea de comandos que modifican el comportamiento de la aplicación CLI.
  • Subcomandos: Comandos dentro de una aplicación CLI que realizan tareas específicas.
  • Bibliotecas: Paquetes que proporcionan utilidades para simplificar la creación de aplicaciones CLI.

Configurando Tu Entorno

Antes de comenzar a construir nuestra aplicación CLI, asegúrate de tener Go instalado en tu máquina. Puedes descargarlo desde el sitio web oficial de Go.

Aplicación CLI Básica

Comencemos con una aplicación CLI simple que acepte un nombre como argumento e imprima un saludo.

Ejemplo: CLI Hola Mundo

package main

import (
    "flag"
    "fmt"
)

func main() {
    name := flag.String("name", "World", "a name to say hello to")
    flag.Parse()
    fmt.Printf("Hello, %s!\n", *name)
}

Explicación

  • flag.String: Define un flag de cadena con un valor predeterminado ("World") y una descripción.
  • flag.Parse: Analiza los flags de la línea de comandos.
  • fmt.Printf: Imprime la cadena formateada en la consola.

Usando una Biblioteca CLI

Aunque el paquete flag de la biblioteca estándar es útil, existen bibliotecas más poderosas como cobra y urfave/cli que proporcionan características adicionales y mejor estructura para aplicaciones CLI complejas.

Ejemplo: Usando Cobra

Primero, instala el paquete cobra:

go get -u github.com/spf13/cobra/cobra

Ahora, vamos a crear una aplicación CLI simple usando cobra.

package main

import (
    "fmt"
    "github.com/spf13/cobra"
)

func main() {
    var rootCmd = &cobra.Command{
        Use:   "greet",
        Short: "Greet es una herramienta CLI para saludar a las personas",
        Run: func(cmd *cobra.Command, args []string) {
            fmt.Println("Hello, World!")
        },
    }

    var name string
    var greetCmd = &cobra.Command{
        Use:   "hello",
        Short: "Imprime un mensaje de saludo",
        Run: func(cmd *cobra.Command, args []string) {
            fmt.Printf("Hello, %s!\n", name)
        },
    }

    greetCmd.Flags().StringVarP(&name, "name", "n", "World", "nombre para saludar")
    rootCmd.AddCommand(greetCmd)
    rootCmd.Execute()
}

Explicación

  • cobra.Command: Define un comando con uso, descripción y una función para ejecutar.
  • Flags().StringVarP: Añade un flag de cadena al comando.
  • AddCommand: Añade un subcomando al comando raíz.
  • Execute: Ejecuta el comando raíz.

Construyendo una Aplicación CLI Más Compleja

Construyamos una aplicación CLI más compleja con múltiples subcomandos y flags.

Ejemplo: CLI de Gestión de Tareas

package main

import (
    "fmt"
    "github.com/spf13/cobra"
)

var tasks = []string{}

func main() {
    var rootCmd = &cobra.Command{Use: "task"}

    var addCmd = &cobra.Command{
        Use:   "add [task]",
        Short: "Añadir una nueva tarea",
        Args:  cobra.MinimumNArgs(1),
        Run: func(cmd *cobra.Command, args []string) {
            task := args[0]
            tasks = append(tasks, task)
            fmt.Printf("Tarea añadida: %s\n", task)
        },
    }

    var listCmd = &cobra.Command{
        Use:   "list",
        Short: "Listar todas las tareas",
        Run: func(cmd *cobra.Command, args []string) {
            fmt.Println("Tareas:")
            for i, task := range tasks {
                fmt.Printf("%d: %s\n", i+1, task)
            }
        },
    }

    rootCmd.AddCommand(addCmd, listCmd)
    rootCmd.Execute()
}

Explicación

  • Args: cobra.MinimumNArgs(1): Asegura que el comando reciba al menos un argumento.
  • tasks: Un slice para almacenar tareas.
  • addCmd: Añade una nueva tarea a la lista.
  • listCmd: Lista todas las tareas.

Conclusión

Construir aplicaciones CLI en Go puede ser sencillo con las herramientas y bibliotecas adecuadas. Comenzamos con un ejemplo básico usando la biblioteca estándar y luego exploramos la biblioteca cobra para construir aplicaciones más complejas. Al entender los flags, subcomandos y cómo estructurar tu aplicación, puedes crear herramientas CLI poderosas y fáciles de usar.
