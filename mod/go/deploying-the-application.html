Desplegando la Aplicación

Desplegar una aplicación Go implica varios pasos, desde construir el ejecutable hasta configurar el entorno donde la aplicación se ejecutará. Esta sección te guiará a través del proceso de desplegar una aplicación Go, cubriendo tanto estrategias básicas como avanzadas de despliegue.

Construyendo la Aplicación Go

Antes de desplegar, necesitas construir tu aplicación Go en un binario ejecutable.

  • Construyendo para el sistema actual:

    go build -o myapp

    Este comando compila el código fuente de Go y produce un ejecutable llamado myapp para el sistema operativo y la arquitectura actuales.

  • Compilando para diferentes sistemas:

    GOOS=linux GOARCH=amd64 go build -o myapp-linux

    Este comando establece el sistema operativo objetivo a Linux y la arquitectura a AMD64, produciendo un ejecutable llamado myapp-linux.

Preparando el Entorno

Asegúrate de que el entorno objetivo esté listo para ejecutar tu aplicación Go.

  • Configurando el servidor:

    • Elige un servidor (por ejemplo, AWS, DigitalOcean, Google Cloud).
    • Instala las dependencias necesarias (por ejemplo, Docker, sistemas de bases de datos).

  • Variables de entorno:

    • Almacena información sensible como claves API y credenciales de bases de datos en variables de entorno.
    • Ejemplo en un archivo .env: 
      DB_HOST=localhost
DB_USER=root
DB_PASS=password

Desplegando con Docker

Docker simplifica el despliegue empaquetando tu aplicación y sus dependencias en un contenedor.

  • Creando un Dockerfile:

    # Usa la imagen oficial de Golang como imagen base
FROM golang:1.19-alpine

# Establece el Directorio de Trabajo Actual dentro del contenedor
WORKDIR /app

# Copia los archivos go mod y sum
COPY go.mod go.sum ./

# Descarga todas las dependencias. Las dependencias se almacenarán en caché si los archivos go.mod y go.sum no cambian
RUN go mod download

# Copia el código fuente desde el directorio actual al Directorio de Trabajo dentro del contenedor
COPY . .

# Construye la aplicación Go
RUN go build -o myapp

# Comando para ejecutar el ejecutable
CMD ["./myapp"]

  • Construyendo y ejecutando el contenedor Docker:

    docker build -t myapp .
docker run -p 8080:8080 myapp

Desplegando a un Servicio en la Nube

Desplegando a AWS Elastic Beanstalk

  • Crear una aplicación Elastic Beanstalk:

    • Usa la Consola de Administración de AWS para crear una nueva aplicación Elastic Beanstalk.
    • Elige una plataforma (por ejemplo, Docker si usas un contenedor Docker).

  • Desplegar la aplicación:

    • Empaqueta tu código de aplicación y archivos de configuración.
    • Usa la CLI de Elastic Beanstalk para desplegar: 
      eb init -p docker myapp
eb create myapp-env
eb deploy

Desplegando a Google Cloud Run

  • Containerizar la aplicación:

    • Asegúrate de tener un Dockerfile como se mostró anteriormente.

  • Desplegar usando la CLI de gcloud:

    gcloud builds submit --tag gcr.io/PROJECT-ID/myapp
gcloud run deploy --image gcr.io/PROJECT-ID/myapp --platform managed

Conclusión

Desplegar una aplicación Go implica construir el ejecutable, preparar el entorno y elegir una estrategia de despliegue. Ya sea que uses Docker, AWS Elastic Beanstalk o Google Cloud Run, la clave es asegurarte de que tu aplicación esté empaquetada correctamente y el entorno objetivo esté configurado adecuadamente. Siguiendo estos pasos, puedes desplegar tus aplicaciones Go de manera eficiente y confiable.
