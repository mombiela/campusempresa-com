Introducción a JSON y XML

JSON (JavaScript Object Notation) y XML (eXtensible Markup Language) son dos formatos populares para el intercambio de datos. Ambos son ampliamente utilizados en servicios web, APIs y archivos de configuración.

  • JSON: Ligero, fácil de leer y escribir, y ampliamente utilizado en aplicaciones web.
  • XML: Más verboso, soporta estructuras de datos complejas y se utiliza en varias aplicaciones empresariales.

Trabajando con JSON en Go

Codificación y Decodificación de JSON

Go proporciona el paquete encoding/json para trabajar con datos JSON. Este paquete te permite codificar estructuras de datos de Go en JSON y decodificar datos JSON en estructuras de datos de Go.

Codificación de Estructuras de Datos de Go a JSON

Para codificar una estructura de datos de Go a JSON, puedes usar la función json.Marshal.

package main

import (
    "encoding/json"
    "fmt"
)

type Person struct {
    Name string `json:"name"`
    Age  int    `json:"age"`
}

func main() {
    person := Person{Name: "Alice", Age: 30}
    jsonData, err := json.Marshal(person)
    if err != nil {
        fmt.Println(err)
        return
    }
    fmt.Println(string(jsonData))
}

Decodificación de JSON a Estructuras de Datos de Go

Para decodificar datos JSON en una estructura de datos de Go, puedes usar la función json.Unmarshal.

package main

import (
    "encoding/json"
    "fmt"
)

type Person struct {
    Name string `json:"name"`
    Age  int    `json:"age"`
}

func main() {
    jsonData := `{"name":"Alice","age":30}`
    var person Person
    err := json.Unmarshal([]byte(jsonData), &person)
    if err != nil {
        fmt.Println(err)
        return
    }
    fmt.Printf("%+v\n", person)
}

Manejo de JSON con Estructuras Complejas

JSON puede representar estructuras de datos complejas como objetos anidados y arreglos. El paquete encoding/json de Go también puede manejar estos.

package main

import (
    "encoding/json"
    "fmt"
)

type Address struct {
    Street string `json:"street"`
    City   string `json:"city"`
}

type Person struct {
    Name    string   `json:"name"`
    Age     int      `json:"age"`
    Address Address  `json:"address"`
    Hobbies []string `json:"hobbies"`
}

func main() {
    jsonData := `{
        "name": "Alice",
        "age": 30,
        "address": {
            "street": "123 Main St",
            "city": "Wonderland"
        },
        "hobbies": ["reading", "biking"]
    }`
    var person Person
    err := json.Unmarshal([]byte(jsonData), &person)
    if err != nil {
        fmt.Println(err)
        return
    }
    fmt.Printf("%+v\n", person)
}

Trabajando con XML en Go

Codificación y Decodificación de XML

Go proporciona el paquete encoding/xml para trabajar con datos XML. Este paquete te permite codificar estructuras de datos de Go en XML y decodificar datos XML en estructuras de datos de Go.

Codificación de Estructuras de Datos de Go a XML

Para codificar una estructura de datos de Go a XML, puedes usar la función xml.Marshal.

package main

import (
    "encoding/xml"
    "fmt"
)

type Person struct {
    XMLName xml.Name `xml:"person"`
    Name    string   `xml:"name"`
    Age     int      `xml:"age"`
}

func main() {
    person := Person{Name: "Alice", Age: 30}
    xmlData, err := xml.Marshal(person)
    if err != nil {
        fmt.Println(err)
        return
    }
    fmt.Println(string(xmlData))
}

Decodificación de XML a Estructuras de Datos de Go

Para decodificar datos XML en una estructura de datos de Go, puedes usar la función xml.Unmarshal.

package main

import (
    "encoding/xml"
    "fmt"
)

type Person struct {
    XMLName xml.Name `xml:"person"`
    Name    string   `xml:"name"`
    Age     int      `xml:"age"`
}

func main() {
    xmlData := `<person><name>Alice</name><age>30</age></person>`
    var person Person
    err := xml.Unmarshal([]byte(xmlData), &person)
    if err != nil {
        fmt.Println(err)
        return
    }
    fmt.Printf("%+v\n", person)
}

Manejo de XML con Estructuras Complejas

XML puede representar estructuras de datos complejas como elementos anidados y atributos. El paquete encoding/xml de Go también puede manejar estos.

package main

import (
    "encoding/xml"
    "fmt"
)

type Address struct {
    Street string `xml:"street"`
    City   string `xml:"city"`
}

type Person struct {
    XMLName xml.Name `xml:"person"`
    Name    string   `xml:"name"`
    Age     int      `xml:"age"`
    Address Address  `xml:"address"`
    Hobbies []string `xml:"hobbies>hobby"`
}

func main() {
    xmlData := `<person>
        <name>Alice</name>
        <age>30</age>
        <address>
            <street>123 Main St</street>
            <city>Wonderland</city>
        </address>
        <hobbies>
            <hobby>reading</hobby>
            <hobby>biking</hobby>
        </hobbies>
    </person>`
    var person Person
    err := xml.Unmarshal([]byte(xmlData), &person)
    if err != nil {
        fmt.Println(err)
        return
    }
    fmt.Printf("%+v\n", person)
}

JSON vs. XML: Una Comparación

Característica JSON XML
Sintaxis Ligero, fácil de leer Verboso, basado en etiquetas
Tipos de Datos Soporta tipos de datos básicos Soporta tipos de datos complejos
Legibilidad Legible por humanos Legible por humanos pero verboso
Soporte de Esquemas No tiene soporte de esquemas Soporta esquemas (XSD)
Casos de Uso APIs web, archivos de configuración Aplicaciones empresariales, archivos de configuración

Conclusión

En esta sección, exploramos cómo trabajar con JSON y XML en Go. Cubrimos la codificación y decodificación de estructuras de datos, el manejo de estructuras complejas y comparamos los dos formatos. JSON es generalmente preferido para aplicaciones web debido a su simplicidad y facilidad de uso, mientras que XML se utiliza en escenarios que requieren una representación de datos más compleja y validación de esquemas. Entender ambos formatos y cómo trabajar con ellos en Go es esencial para cualquier programador que maneje el intercambio de datos.
