Introducción

Las pruebas y la depuración son habilidades cruciales para cualquier programador. En esta sección, cubriremos los conceptos básicos de las pruebas y la depuración en Go, progresando desde técnicas para principiantes hasta técnicas avanzadas. Al final de esta sección, deberías sentirte cómodo escribiendo pruebas, utilizando las herramientas de prueba de Go y depurando tus programas en Go de manera efectiva.

Pruebas en Go

Conceptos Básicos de las Pruebas

  • Paquete de Pruebas de Go: Go tiene un paquete de pruebas incorporado testing que proporciona soporte para pruebas automatizadas.
  • Funciones de Prueba: Las funciones de prueba en Go se identifican por el prefijo Test seguido del nombre de la función que se está probando.

Ejemplo de una Prueba Básica

package main

import (
    "testing"
)

func Add(a, b int) int {
    return a + b
}

func TestAdd(t *testing.T) {
    result := Add(2, 3)
    expected := 5
    if result != expected {
        t.Errorf("Add(2, 3) = %d; want %d", result, expected)
    }
}
  • Ejecutar Pruebas: Usa el comando go test para ejecutar pruebas en tu paquete de Go.

Pruebas Basadas en Tablas

  • Concepto: Las pruebas basadas en tablas usan una tabla de entradas y salidas esperadas para ejecutar múltiples casos de prueba en una sola función de prueba.
  • Ventajas: Facilitan la adición de nuevos casos de prueba y mejoran la legibilidad de las pruebas.

Ejemplo de Prueba Basada en Tablas

package main

import (
    "testing"
)

func Multiply(a, b int) int {
    return a * b
}

func TestMultiply(t *testing.T) {
    tests := []struct {
        a, b, expected int
    }{
        {2, 3, 6},
        {4, 5, 20},
        {0, 10, 0},
    }

    for _, tt := range tests {
        result := Multiply(tt.a, tt.b)
        if result != tt.expected {
            t.Errorf("Multiply(%d, %d) = %d; want %d", tt.a, tt.b, result, tt.expected)
        }
    }
}

Pruebas de Rendimiento

  • Propósito: Las pruebas de rendimiento ayudan a medir el rendimiento de tu código.
  • Funciones de Rendimiento: Se identifican por el prefijo Benchmark y toman un parámetro *testing.B.

Ejemplo de Prueba de Rendimiento

package main

import (
    "testing"
)

func BenchmarkAdd(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Add(2, 3)
    }
}
  • Ejecutar Pruebas de Rendimiento: Usa el comando go test -bench . para ejecutar pruebas de rendimiento.

Cobertura de Pruebas

  • Concepto: La cobertura de pruebas mide el porcentaje de tu código que está cubierto por pruebas.
  • Comando: Usa go test -cover para ver la cobertura de pruebas de tu paquete.

Depuración en Go

Uso de Sentencias Print

  • Técnica Básica: Insertar sentencias print (fmt.Println) en tu código para entender el flujo y el estado de las variables.
  • Limitaciones: Puede ser un proceso que consume tiempo y desordenar el código.

Uso del Depurador Delve

  • Delve: Un depurador poderoso para Go que te permite establecer puntos de interrupción, inspeccionar variables y controlar el flujo de ejecución.
  • Instalación: Instala Delve usando go install github.com/go-delve/delve/cmd/dlv@latest.

Comandos Básicos en Delve

  • Iniciar Delve: dlv debug para iniciar la depuración de tu programa.
  • Establecer Puntos de Interrupción: break main.go:10 para establecer un punto de interrupción en la línea 10 de main.go.
  • Ejecutar el Programa: continue para ejecutar el programa hasta el siguiente punto de interrupción.
  • Inspeccionar Variables: print variableName para inspeccionar el valor de una variable.
  • Avanzar en el Código: step para entrar en funciones, next para avanzar sobre funciones y stepout para salir de la función actual.

Ejemplo de Sesión de Depuración

Conclusión

Las pruebas y la depuración son prácticas esenciales para escribir código en Go que sea confiable y mantenible. Al dominar el paquete de pruebas de Go, escribir pruebas efectivas y usar herramientas como Delve para la depuración, puedes mejorar significativamente la calidad de tu código. Recuerda escribir pruebas para tu código regularmente y usar herramientas de depuración para identificar y solucionar problemas rápidamente.
