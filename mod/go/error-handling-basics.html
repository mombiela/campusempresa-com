Conceptos Básicos de Manejo de Errores
Errores Personalizados ►

El manejo de errores es un aspecto crucial de la programación, y Go proporciona una forma única y efectiva de gestionar errores. Esta sección cubrirá los conceptos básicos del manejo de errores en Go, incluyendo cómo crear, manejar y propagar errores.

Conceptos Clave

  • Tipo de Error: En Go, los errores están representados por el tipo incorporado error.
  • Creación de Errores: Usa la función errors.New o la función fmt.Errorf para crear nuevos valores de error.
  • Manejo de Errores: Verifica los errores usando simples sentencias if.
  • Propagación de Errores: Devuelve errores desde funciones para propagarlos en la pila de llamadas.
  • Errores Personalizados: Implementa la interfaz error para crear tipos de error personalizados.

Creación de Errores

En Go, puedes crear un error usando la función errors.New del paquete errors o la función fmt.Errorf del paquete fmt.

Usando `errors.New`

package main

import (
    "errors"
    "fmt"
)

func main() {
    err := errors.New("algo salió mal")
    if err != nil {
        fmt.Println(err)
    }
}

Usando `fmt.Errorf`

package main

import (
    "fmt"
)

func main() {
    err := fmt.Errorf("ocurrió un error: %v", "algo salió mal")
    if err != nil {
        fmt.Println(err)
    }
}

Manejo de Errores

El manejo de errores en Go se hace típicamente usando sentencias if. Si una función devuelve un error, debes verificar si el error es nil para determinar si ocurrió un error.

package main

import (
    "fmt"
    "strconv"
)

func main() {
    _, err := strconv.Atoi("123a")
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Conversión exitosa")
    }
}

Propagación de Errores

Al escribir funciones, debes devolver errores para permitir que el llamador los maneje apropiadamente. Esto se conoce como propagación de errores.

package main

import (
    "errors"
    "fmt"
)

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("no se puede dividir por cero")
    }
    return a / b, nil
}

func main() {
    result, err := divide(4, 0)
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Resultado:", result)
    }
}

Errores Personalizados

Puedes crear tipos de error personalizados implementando la interfaz error. Esto te permite agregar contexto adicional a tus errores.

package main

import (
    "fmt"
)

type MyError struct {
    Code    int
    Message string
}

func (e *MyError) Error() string {
    return fmt.Sprintf("Error %d: %s", e.Code, e.Message)
}

func doSomething(flag bool) error {
    if !flag {
        return &MyError{Code: 123, Message: "algo salió mal"}
    }
    return nil
}

func main() {
    err := doSomething(false)
    if err != nil {
        fmt.Println(err)
    }
}

Resumen

  • Go usa el tipo incorporado error para el manejo de errores.
  • Crea errores usando errors.New o fmt.Errorf.
  • Maneja errores usando sentencias if para verificar si el error es nil.
  • Propaga errores devolviéndolos desde funciones.
  • Implementa la interfaz error para crear tipos de error personalizados.

Entender estos conceptos básicos te ayudará a escribir programas en Go más robustos y mantenibles. A medida que progreses, aprenderás técnicas más avanzadas para el manejo de errores, como usar el paquete errors para envolver y desenvolver errores.
