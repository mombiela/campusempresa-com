Introducción

En esta sección, cubriremos los pasos esenciales y las mejores prácticas para construir y desplegar aplicaciones Go. Esto incluye compilar tu código Go, gestionar dependencias, crear binarios ejecutables y desplegar tu aplicación en diferentes entornos.

Construcción de Aplicaciones Go

Compilación de Código Go

  • Comando Go Build: El comando go build compila los paquetes y dependencias.

    // main.go
package main

import "fmt"

func main() {
    fmt.Println("¡Hola, Mundo!")
}

    $ go build main.go
    • Este comando genera un archivo ejecutable llamado main (o main.exe en Windows).

  • Comando Go Install: El comando go install compila e instala el paquete.

    $ go install main.go
    • Este comando coloca el ejecutable en el directorio $GOPATH/bin.

Gestión de Dependencias

  • Módulos Go: Los módulos Go son el estándar para la gestión de dependencias en Go.
    • Inicializar un nuevo módulo: 
      $ go mod init example.com/myapp
    • Añadir una dependencia: 
      $ go get github.com/gin-gonic/gin
    • Limpiar dependencias: 
      $ go mod tidy

Creación de Binarios Ejecutables

  • Compilación Cruzada: Go soporta la compilación cruzada para diferentes sistemas operativos y arquitecturas. 
    $ GOOS=linux GOARCH=amd64 go build -o myapp-linux main.go
$ GOOS=windows GOARCH=amd64 go build -o myapp.exe main.go
    • Este comando crea binarios para Linux y Windows.

Despliegue de Aplicaciones Go

Despliegue en un Servidor

  • Copiar Binarios: Simplemente copia el binario compilado al servidor de destino. 
    $ scp myapp-linux user@server:/path/to/deploy
  • Ejecutar la Aplicación: SSH en el servidor y ejecuta el binario. 
    $ ssh user@server
$ ./myapp-linux

Uso de Docker para Despliegue

  • Dockerfile: Crea un Dockerfile para contenerizar tu aplicación Go. 
    # Dockerfile
FROM golang:1.17-alpine AS builder
WORKDIR /app
COPY . .
RUN go build -o myapp

FROM alpine:latest
WORKDIR /root/
COPY --from=builder /app/myapp .
CMD ["./myapp"]
  • Construcción de la Imagen Docker: 
    $ docker build -t myapp:latest .
  • Ejecución del Contenedor Docker: 
    $ docker run -p 8080:8080 myapp:latest

Despliegue en Servicios en la Nube

  • Google Cloud Run: Desplegar una aplicación Go contenerizada en Google Cloud Run.

    • Construir y enviar la imagen Docker a Google Container Registry: 
      $ docker build -t gcr.io/[PROJECT-ID]/myapp .
$ docker push gcr.io/[PROJECT-ID]/myapp
    • Desplegar en Cloud Run: 
      $ gcloud run deploy --image gcr.io/[PROJECT-ID]/myapp --platform managed

  • Heroku: Desplegar una aplicación Go en Heroku.

    • Crear un Procfile: 
      web: ./myapp
    • Desplegar usando Git: 
      $ git init
$ heroku create
$ git add .
$ git commit -m "Commit inicial"
$ git push heroku master

Conclusión

Construir y desplegar aplicaciones Go implica varios pasos, desde compilar el código hasta gestionar dependencias y crear binarios ejecutables. El despliegue puede realizarse en varias plataformas, incluyendo servidores, contenedores Docker y servicios en la nube. Siguiendo las mejores prácticas y utilizando las herramientas adecuadas, puedes asegurar que tus aplicaciones Go se construyan y desplieguen de manera eficiente y confiable.
