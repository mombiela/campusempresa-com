Introducción a INNER JOIN

En SQL, la cláusula INNER JOIN se utiliza para combinar filas de dos o más tablas basándose en una columna relacionada entre ellas. Devuelve solo las filas que tienen valores coincidentes en ambas tablas.

Conceptos Clave

  • Condición de Unión: La condición que especifica cómo las filas de una tabla se emparejan con las filas de otra tabla.
  • Conjunto de Resultados: El conjunto de filas devuelto por la operación INNER JOIN.
  • Clave Primaria: Un identificador único para una fila en una tabla.
  • Clave Foránea: Una columna o un conjunto de columnas en una tabla que se refiere a la clave primaria en otra tabla.

Sintaxis Básica

La sintaxis básica para un INNER JOIN es la siguiente:

SELECT columnas
FROM tabla1
INNER JOIN tabla2
ON tabla1.columna = tabla2.columna;

Ejemplo

Considera dos tablas, Employees y Departments:

Tabla Employees

| EmployeeID | Nombre | DepartmentID | |------------|----------|--------------| | 1 | Alice | 101 | | 2 | Bob | 102 | | 3 | Charlie | 103 | | 4 | David | 101 |

Tabla Departments

| DepartmentID | NombreDepartamento | |--------------|--------------------| | 101 | HR | | 102 | IT | | 103 | Finance | | 104 | Marketing |

Consulta INNER JOIN

Para obtener una lista de empleados junto con los nombres de sus departamentos, puedes usar la siguiente consulta INNER JOIN:

SELECT Employees.Nombre, Departments.NombreDepartamento
FROM Employees
INNER JOIN Departments
ON Employees.DepartmentID = Departments.DepartmentID;

Conjunto de Resultados

| Nombre | NombreDepartamento | |---------|--------------------| | Alice | HR | | Bob | IT | | Charlie | Finance | | David | HR |

Uso Avanzado

Uniendo Múltiples Tablas

Puedes unir más de dos tablas usando múltiples cláusulas INNER JOIN. Por ejemplo:

SELECT Employees.Nombre, Departments.NombreDepartamento, Projects.NombreProyecto
FROM Employees
INNER JOIN Departments ON Employees.DepartmentID = Departments.DepartmentID
INNER JOIN EmployeeProjects ON Employees.EmployeeID = EmployeeProjects.EmployeeID
INNER JOIN Projects ON EmployeeProjects.ProjectID = Projects.ProjectID;

Usando Alias

Los alias pueden hacer que tus consultas SQL sean más legibles:

SELECT e.Nombre, d.NombreDepartamento
FROM Employees AS e
INNER JOIN Departments AS d
ON e.DepartmentID = d.DepartmentID;

Filtrando Resultados

Puedes agregar una cláusula WHERE para filtrar los resultados:

SELECT e.Nombre, d.NombreDepartamento
FROM Employees AS e
INNER JOIN Departments AS d
ON e.DepartmentID = d.DepartmentID
WHERE d.NombreDepartamento = 'IT';

Ejercicios

  1. INNER JOIN Básico: Escribe una consulta para unir las tablas Students y Courses para listar todos los estudiantes junto con los cursos en los que están inscritos.
  2. Múltiples INNER JOINs: Escribe una consulta para unir las tablas Orders, Customers y Products para listar todos los pedidos junto con los nombres de los clientes y los detalles de los productos.
  3. Filtrando con INNER JOIN: Escribe una consulta para unir las tablas Employees y Departments y filtrar los resultados para mostrar solo los empleados en el departamento de 'Finance'.

Conclusión

La cláusula INNER JOIN es una herramienta poderosa en SQL para combinar filas de múltiples tablas basándose en una columna relacionada. Al comprender y usar INNER JOIN, puedes recuperar y analizar datos de bases de datos relacionales de manera eficiente. Practica con diferentes escenarios y conjuntos de datos para dominar el uso de INNER JOIN en tus consultas SQL.
