Claves Primarias y Claves Foráneas
Índices ►

Introducción

En SQL, las claves primarias y las claves foráneas son conceptos fundamentales utilizados para establecer y reforzar relaciones entre tablas. Comprender estas claves es crucial para diseñar bases de datos robustas y eficientes.

Claves Primarias

Una clave primaria es un identificador único para un registro en una tabla. Asegura que cada registro pueda ser identificado y recuperado de manera única.

Conceptos Clave

  • Unicidad: Cada valor en la(s) columna(s) de la clave primaria debe ser único.
  • No nulidad: Las columnas de la clave primaria no pueden contener valores NULL.
  • Simple o Compuesta: Una clave primaria puede consistir en una sola columna o en múltiples columnas (clave compuesta).

Ejemplo

Considera una tabla Students donde cada estudiante tiene un ID único:

CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    BirthDate DATE
);

En este ejemplo:

  • StudentID es la clave primaria.
  • Identifica de manera única a cada estudiante en la tabla Students.

Claves Foráneas

Una clave foránea es una columna o un conjunto de columnas en una tabla que identifica de manera única una fila de otra tabla. Establece un vínculo entre los datos en las dos tablas.

Conceptos Clave

  • Integridad Referencial: Asegura que el valor en la(s) columna(s) de la clave foránea coincida con un valor en la(s) columna(s) de la clave primaria de la tabla referenciada.
  • Relación Padre-Hijo: La tabla que contiene la clave foránea es la tabla hija, y la tabla que contiene la clave primaria es la tabla padre.

Ejemplo

Considera una tabla Enrollments que registra qué estudiantes están inscritos en qué cursos:

CREATE TABLE Enrollments (
    EnrollmentID INT PRIMARY KEY,
    StudentID INT,
    CourseID INT,
    EnrollmentDate DATE,
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID)
);

En este ejemplo:

  • StudentID en la tabla Enrollments es una clave foránea.
  • Hace referencia a StudentID en la tabla Students, estableciendo una relación entre las dos tablas.

Combinando Claves Primarias y Foráneas

Para ilustrar cómo funcionan juntas las claves primarias y foráneas, agreguemos una tabla Courses y modifiquemos la tabla Enrollments para incluir una referencia de clave foránea a Courses.

Ejemplo

CREATE TABLE Courses (
    CourseID INT PRIMARY KEY,
    CourseName VARCHAR(100),
    Credits INT
);

CREATE TABLE Enrollments (
    EnrollmentID INT PRIMARY KEY,
    StudentID INT,
    CourseID INT,
    EnrollmentDate DATE,
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
    FOREIGN KEY (CourseID) REFERENCES Courses(CourseID)
);

En este ejemplo:

  • CourseID en la tabla Courses es la clave primaria.
  • CourseID en la tabla Enrollments es una clave foránea que hace referencia a CourseID en la tabla Courses.
  • Esta configuración asegura que cada registro de inscripción esté vinculado a un estudiante válido y a un curso válido.

Resumen

  • Claves Primarias: Aseguran que cada registro en una tabla sea único e identificable.
  • Claves Foráneas: Establecen relaciones entre tablas, asegurando la integridad referencial.
  • Uso: Las claves primarias se utilizan para identificar registros de manera única, mientras que las claves foráneas vinculan registros entre tablas.

Comprender e implementar correctamente las claves primarias y foráneas es esencial para mantener la integridad de los datos y establecer relaciones claras dentro de tu base de datos.
