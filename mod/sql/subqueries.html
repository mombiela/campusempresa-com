Introducción a las Subconsultas

Las subconsultas, también conocidas como consultas internas o consultas anidadas, son consultas dentro de otra consulta SQL. Se utilizan para realizar operaciones que requieren múltiples pasos, como filtrar, agregar o unir datos de una manera más compleja.

Conceptos Clave

  • Subconsulta: Una consulta anidada dentro de otra consulta.
  • Consulta Externa: La consulta principal que contiene la subconsulta.
  • Subconsulta de una sola fila: Devuelve una sola fila de resultados.
  • Subconsulta de múltiples filas: Devuelve múltiples filas de resultados.
  • Subconsulta Correlacionada: Una subconsulta que hace referencia a columnas de la consulta externa.

Tipos de Subconsultas

Subconsultas de una sola fila

Las subconsultas de una sola fila devuelven una fila de resultados y a menudo se utilizan con operadores de comparación como =, >, <, >=, <=, y <>.

Ejemplo

SELECT employee_id, first_name, last_name
FROM employees
WHERE department_id = (
    SELECT department_id
    FROM departments
    WHERE department_name = 'Sales'
);
  • Explicación: Esta consulta recupera el employee_id, first_name y last_name de los empleados que trabajan en el departamento de 'Ventas'. La subconsulta encuentra el department_id para 'Ventas', y la consulta externa utiliza este department_id para filtrar empleados.

Subconsultas de múltiples filas

Las subconsultas de múltiples filas devuelven más de una fila y a menudo se utilizan con operadores como IN, ANY o ALL.

Ejemplo

SELECT employee_id, first_name, last_name
FROM employees
WHERE department_id IN (
    SELECT department_id
    FROM departments
    WHERE location_id = 1700
);
  • Explicación: Esta consulta recupera el employee_id, first_name y last_name de los empleados que trabajan en departamentos ubicados en location_id 1700. La subconsulta devuelve todos los department_id en la ubicación 1700, y la consulta externa filtra empleados basándose en estos department_id.

Subconsultas Correlacionadas

Las subconsultas correlacionadas hacen referencia a columnas de la consulta externa y se evalúan una vez por cada fila procesada por la consulta externa.

Ejemplo

SELECT employee_id, first_name, last_name
FROM employees e
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department_id = e.department_id
);
  • Explicación: Esta consulta recupera el employee_id, first_name y last_name de los empleados cuyo salario es mayor que el salario promedio de su departamento. La subconsulta calcula el salario promedio para cada departamento, y la consulta externa compara el salario de cada empleado con este promedio.

Ejercicios Prácticos

Ejercicio 1: Subconsulta de una sola fila

Escribe una consulta para encontrar el nombre del empleado que tiene el salario más alto.

SELECT first_name, last_name
FROM employees
WHERE salary = (
    SELECT MAX(salary)
    FROM employees
);

Ejercicio 2: Subconsulta de múltiples filas

Escribe una consulta para encontrar los nombres de los empleados que trabajan en el mismo departamento que 'John Doe'.

SELECT first_name, last_name
FROM employees
WHERE department_id = (
    SELECT department_id
    FROM employees
    WHERE first_name = 'John' AND last_name = 'Doe'
);

Ejercicio 3: Subconsulta Correlacionada

Escribe una consulta para encontrar los nombres de los empleados que ganan más que el salario promedio en su departamento.

SELECT first_name, last_name
FROM employees e
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department_id = e.department_id
);

Conclusión

Las subconsultas son herramientas poderosas en SQL que permiten una recuperación de datos más compleja y flexible. Se pueden utilizar para realizar operaciones que serían difíciles o imposibles con una sola consulta. Comprender cómo usar subconsultas de una sola fila, de múltiples filas y correlacionadas mejorará tu capacidad para escribir consultas SQL eficientes y efectivas. Practica el uso de subconsultas en diferentes escenarios para volverte competente en su aplicación.
