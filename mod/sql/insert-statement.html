Declaración INSERT
Sentencia UPDATE ►

La declaración INSERT en SQL se utiliza para agregar nuevas filas de datos a una tabla. Esta es una operación fundamental en la gestión de bases de datos, que te permite poblar tablas con los datos necesarios para tus aplicaciones.

Conceptos Clave

  • Sintaxis: La estructura básica de la declaración INSERT.
  • Inserción de una Sola Fila: Cómo insertar una sola fila en una tabla.
  • Inserción de Múltiples Filas: Cómo insertar múltiples filas en una sola declaración.
  • Insertar Datos desde Otra Tabla: Usar INSERT INTO ... SELECT para copiar datos de una tabla a otra.
  • Manejo de Valores NULL: Cómo manejar columnas que pueden tener valores NULL.
  • Manejo de Errores: Errores comunes y cómo manejarlos.

Sintaxis

La sintaxis básica para la declaración INSERT es la siguiente:

INSERT INTO nombre_tabla (columna1, columna2, columna3, ...)
VALUES (valor1, valor2, valor3, ...);
  • nombre_tabla: El nombre de la tabla donde deseas insertar los datos.
  • columna1, columna2, columna3, ...: Las columnas en la tabla donde se insertarán los datos.
  • valor1, valor2, valor3, ...: Los valores que se insertarán en las columnas especificadas.

Inserción de una Sola Fila

Para insertar una sola fila en una tabla, puedes usar el siguiente ejemplo:

INSERT INTO empleados (nombre, apellido, correo, fecha_contratacion)
VALUES ('John', 'Doe', 'john.doe@example.com', '2023-10-01');

En este ejemplo:

  • La tabla empleados tiene columnas nombre, apellido, correo y fecha_contratacion.
  • La cláusula VALUES especifica los datos que se insertarán en estas columnas.

Inserción de Múltiples Filas

Puedes insertar múltiples filas en una tabla en una sola declaración INSERT:

INSERT INTO empleados (nombre, apellido, correo, fecha_contratacion)
VALUES 
    ('Jane', 'Smith', 'jane.smith@example.com', '2023-10-02'),
    ('Alice', 'Johnson', 'alice.johnson@example.com', '2023-10-03'),
    ('Bob', 'Brown', 'bob.brown@example.com', '2023-10-04');

Este ejemplo inserta tres filas en la tabla empleados de una sola vez.

Insertar Datos desde Otra Tabla

También puedes insertar datos en una tabla desde otra tabla usando la declaración INSERT INTO ... SELECT:

INSERT INTO empleados_archivo (nombre, apellido, correo, fecha_contratacion)
SELECT nombre, apellido, correo, fecha_contratacion
FROM empleados
WHERE fecha_contratacion < '2023-01-01';

En este ejemplo:

  • Los datos se copian de la tabla empleados a la tabla empleados_archivo.
  • Solo se insertan las filas donde fecha_contratacion es anterior al 1 de enero de 2023.

Manejo de Valores NULL

Si una columna puede aceptar valores NULL, puedes insertar NULL explícitamente u omitir la columna de la declaración INSERT:

INSERT INTO empleados (nombre, apellido, correo)
VALUES ('Charlie', 'Davis', NULL);

O:

INSERT INTO empleados (nombre, apellido)
VALUES ('Charlie', 'Davis');

En ambos casos, la columna correo tendrá un valor NULL para la nueva fila.

Manejo de Errores

Los errores comunes que podrías encontrar al usar la declaración INSERT incluyen:

  • Error de Clave Duplicada: Ocurre cuando intentas insertar una fila con una clave primaria que ya existe.
  • Incompatibilidad de Tipo de Datos: Sucede cuando el valor que se está insertando no coincide con el tipo de datos de la columna.
  • Violación de Restricción: Ocurre cuando los datos que se están insertando violan una restricción de la tabla, como una restricción de clave foránea.

Ejemplo de manejo de un error de clave duplicada:

INSERT INTO empleados (id, nombre, apellido, correo)
VALUES (1, 'David', 'Wilson', 'david.wilson@example.com')
ON CONFLICT (id) DO NOTHING;

En este ejemplo, si ya existe una fila con id 1, la cláusula ON CONFLICT evita la inserción y no hace nada.

Conclusión

La declaración INSERT es una herramienta poderosa en SQL para agregar nuevos datos a tus tablas. Al comprender su sintaxis y varios casos de uso, puedes gestionar eficientemente los datos de tu base de datos. Ya sea que estés insertando una sola fila, múltiples filas o copiando datos de otra tabla, dominar la declaración INSERT es esencial para cualquier programador de SQL.
