Introducción a las Auto Uniones

Una auto unión es una unión regular, pero la tabla se une consigo misma. Esto es útil cuando necesitas comparar filas dentro de la misma tabla o cuando necesitas encontrar relaciones entre filas en la misma tabla.

Conceptos Clave

  • Auto Unión: Una unión donde una tabla se une consigo misma.
  • Alias: Se usa para darle a una tabla un nombre temporal para diferenciar entre las instancias original y unida de la tabla.
  • Clave Primaria: Un identificador único para cada fila en una tabla.
  • Clave Foránea: Un campo en una tabla que identifica de manera única una fila en otra tabla (o en la misma tabla en el caso de auto uniones).

Cuándo Usar Auto Uniones

  • Encontrar relaciones jerárquicas (por ejemplo, relaciones empleado-gerente).
  • Comparar filas dentro de la misma tabla.
  • Encontrar duplicados o patrones dentro de una tabla.

Sintaxis de las Auto Uniones

La sintaxis de una auto unión es similar a una unión regular, pero debes usar alias de tabla para diferenciar entre las instancias de la tabla.

SELECT A.column1, B.column2
FROM table_name A, table_name B
WHERE A.common_field = B.common_field;

Ejemplo: Relación Empleado-Gerente

Considera una tabla employees donde cada empleado tiene un manager_id que referencia el employee_id de su gerente.

Tabla: employees

employee_id employee_name manager_id
1 John Doe NULL
2 Jane Smith 1
3 Emily Davis 2
4 Michael Brown 1

Consulta

Para encontrar los nombres de los empleados junto con los nombres de sus gerentes:

SELECT e1.employee_name AS Employee, e2.employee_name AS Manager
FROM employees e1
LEFT JOIN employees e2 ON e1.manager_id = e2.employee_id;

Resultado

Empleado Gerente
John Doe NULL
Jane Smith John Doe
Emily Davis Jane Smith
Michael Brown John Doe

Ejercicios

  1. Auto Unión Básica: Escribe una consulta para encontrar todos los pares de empleados que comparten el mismo gerente.
  2. Datos Jerárquicos: Escribe una consulta para listar todos los empleados y sus gerentes directos e indirectos.
  3. Detección de Duplicados: Escribe una consulta para encontrar filas duplicadas en una tabla basada en una columna específica.

Ejercicio 1: Auto Unión Básica

SELECT e1.employee_name AS Employee1, e2.employee_name AS Employee2, e1.manager_id
FROM employees e1
JOIN employees e2 ON e1.manager_id = e2.manager_id
WHERE e1.employee_id != e2.employee_id;

Ejercicio 2: Datos Jerárquicos

WITH RECURSIVE EmployeeHierarchy AS (
    SELECT employee_id, employee_name, manager_id
    FROM employees
    WHERE manager_id IS NULL
    UNION ALL
    SELECT e.employee_id, e.employee_name, eh.employee_id
    FROM employees e
    JOIN EmployeeHierarchy eh ON e.manager_id = eh.employee_id
)
SELECT * FROM EmployeeHierarchy;

Ejercicio 3: Detección de Duplicados

SELECT t1.*
FROM table_name t1
JOIN table_name t2 ON t1.column_to_check = t2.column_to_check
WHERE t1.id != t2.id;

Conclusión

Las auto uniones son una herramienta poderosa en SQL que te permiten comparar filas dentro de la misma tabla. Son particularmente útiles para datos jerárquicos y para encontrar relaciones dentro de una sola tabla. Al dominar las auto uniones, puedes realizar consultas complejas que implican comparar y analizar datos dentro del mismo conjunto de datos.
