Introducción

En SQL, las uniones se utilizan para combinar filas de dos o más tablas basadas en una columna relacionada entre ellas. Entender las uniones es crucial para consultar bases de datos relacionales de manera efectiva.

Tipos de Uniones

Unión Interna

Una unión interna devuelve solo las filas que tienen valores coincidentes en ambas tablas.

Sintaxis:

SELECT columnas
FROM tabla1
INNER JOIN tabla2
ON tabla1.columna_comun = tabla2.columna_comun;

Ejemplo:

Considera dos tablas, employees y departments:

employees | employee_id | name | department_id | |-------------|--------|---------------| | 1 | Alice | 1 | | 2 | Bob | 2 | | 3 | Charlie| 3 |

departments | department_id | department_name | |---------------|-----------------| | 1 | HR | | 2 | IT | | 4 | Marketing |

SELECT employees.name, departments.department_name
FROM employees
INNER JOIN departments
ON employees.department_id = departments.department_id;

Resultado: | name | department_name | |--------|-----------------| | Alice | HR | | Bob | IT |

Unión Izquierda (Externa)

Una unión izquierda devuelve todas las filas de la tabla izquierda, y las filas coincidentes de la tabla derecha. Si no se encuentra coincidencia, se devuelven valores NULL para las columnas de la tabla derecha.

Sintaxis:

SELECT columnas
FROM tabla1
LEFT JOIN tabla2
ON tabla1.columna_comun = tabla2.columna_comun;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
LEFT JOIN departments
ON employees.department_id = departments.department_id;

Resultado: | name | department_name | |---------|-----------------| | Alice | HR | | Bob | IT | | Charlie | NULL |

Unión Derecha (Externa)

Una unión derecha devuelve todas las filas de la tabla derecha, y las filas coincidentes de la tabla izquierda. Si no se encuentra coincidencia, se devuelven valores NULL para las columnas de la tabla izquierda.

Sintaxis:

SELECT columnas
FROM tabla1
RIGHT JOIN tabla2
ON tabla1.columna_comun = tabla2.columna_comun;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
RIGHT JOIN departments
ON employees.department_id = departments.department_id;

Resultado: | name | department_name | |--------|-----------------| | Alice | HR | | Bob | IT | | NULL | Marketing |

Unión Completa (Externa)

Una unión completa devuelve todas las filas cuando hay una coincidencia en cualquiera de las tablas izquierda o derecha. Si no hay coincidencia, el resultado es NULL en el lado que no tiene coincidencia.

Sintaxis:

SELECT columnas
FROM tabla1
FULL OUTER JOIN tabla2
ON tabla1.columna_comun = tabla2.columna_comun;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
FULL OUTER JOIN departments
ON employees.department_id = departments.department_id;

Resultado: | name | department_name | |---------|-----------------| | Alice | HR | | Bob | IT | | Charlie | NULL | | NULL | Marketing |

Unión Cruzada

Una unión cruzada devuelve el producto cartesiano de las dos tablas, es decir, devuelve todas las combinaciones posibles de filas de las tablas.

Sintaxis:

SELECT columnas
FROM tabla1
CROSS JOIN tabla2;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
CROSS JOIN departments;

Resultado: | name | department_name | |---------|-----------------| | Alice | HR | | Alice | IT | | Alice | Marketing | | Bob | HR | | Bob | IT | | Bob | Marketing | | Charlie | HR | | Charlie | IT | | Charlie | Marketing |

Conclusión

Las uniones son un aspecto fundamental de SQL que te permiten combinar datos de múltiples tablas basadas en columnas relacionadas. Entender los diferentes tipos de uniones—interna, izquierda, derecha, completa y cruzada—te permitirá escribir consultas más complejas y eficientes. Practica usando estas uniones con varios conjuntos de datos para volverte competente en la consulta de SQL.
