Requisitos y Entregables
Evaluación del Proyecto ►

En esta sección del curso sobre Gestión del Cambio, abordaremos los conceptos de requisitos y entregables, fundamentales para asegurar el éxito de cualquier proyecto de cambio en una organización. Entenderemos qué son, cómo se definen y gestionan, y su importancia en el contexto de la gestión del cambio.

Requisitos

Los requisitos son las condiciones o capacidades que deben cumplirse o alcanzarse en un proyecto. En el contexto de la gestión del cambio, los requisitos pueden ser tanto técnicos como no técnicos y deben ser claramente definidos y documentados.

Tipos de Requisitos

  • Funcionales: Describen lo que el sistema debe hacer.
    • Ejemplo: "El sistema debe permitir a los usuarios iniciar sesión con un nombre de usuario y contraseña."
  • No Funcionales: Describen cómo debe comportarse el sistema.
    • Ejemplo: "El sistema debe ser capaz de manejar 1000 usuarios concurrentes."
  • Requisitos de Negocio: Describen las necesidades de la organización.
    • Ejemplo: "El nuevo sistema debe reducir el tiempo de procesamiento de pedidos en un 20%."
  • Requisitos de Usuario: Describen las necesidades y expectativas de los usuarios finales.
    • Ejemplo: "Los usuarios deben poder acceder al sistema desde dispositivos móviles."

Ejemplo de Documento de Requisitos

# Documento de Requisitos del Proyecto XYZ

## Requisitos Funcionales
1. El sistema debe permitir a los usuarios registrarse con una dirección de correo electrónico y una contraseña.
2. El sistema debe enviar un correo de confirmación después del registro.

## Requisitos No Funcionales
1. El sistema debe ser accesible 24/7.
2. El tiempo de respuesta del sistema no debe exceder los 2 segundos.

## Requisitos de Negocio
1. El sistema debe integrarse con el sistema de gestión de inventarios existente.
2. El sistema debe generar informes mensuales de ventas.

## Requisitos de Usuario
1. Los usuarios deben poder recuperar su contraseña mediante un enlace enviado a su correo electrónico.
2. Los usuarios deben poder personalizar su perfil.

Entregables

Los entregables son los productos, servicios o resultados que se deben completar y entregar durante un proyecto. En la gestión del cambio, los entregables pueden incluir documentos, software, informes, entre otros.

Tipos de Entregables

  • Documentación: Manuales de usuario, guías de instalación, documentos de requisitos.
    • Ejemplo: "Manual de usuario del nuevo sistema de gestión de inventarios."
  • Software: Aplicaciones, módulos, actualizaciones.
    • Ejemplo: "Versión 1.0 del sistema de gestión de inventarios."
  • Informes: Análisis de impacto, informes de progreso, evaluaciones post-implementación.
    • Ejemplo: "Informe de análisis de impacto del cambio en el proceso de ventas."
  • Capacitación: Programas de formación, talleres, seminarios.
    • Ejemplo: "Taller de capacitación para el uso del nuevo sistema de gestión de inventarios."

Ejemplo de Lista de Entregables

# Lista de Entregables del Proyecto XYZ

1. Documento de Requisitos del Sistema
2. Manual de Usuario
3. Versión 1.0 del Sistema de Gestión de Inventarios
4. Informe de Análisis de Impacto
5. Programa de Capacitación para Usuarios Finales

Comparación de Requisitos y Entregables

| Aspecto | Requisitos | Entregables | |---------------|-------------------------------------------------|------------------------------------------------| | Definición | Condiciones o capacidades necesarias | Productos, servicios o resultados a entregar | | Tipos | Funcionales, No Funcionales, Negocio, Usuario | Documentación, Software, Informes, Capacitación| | Ejemplo | "El sistema debe permitir iniciar sesión" | "Manual de Usuario del sistema" | | Propósito | Establecer lo que se necesita | Proveer lo que se ha desarrollado o producido |

Conclusión

En resumen, los requisitos y entregables son componentes esenciales en la gestión del cambio. Los requisitos definen lo que se necesita para que el proyecto tenga éxito, mientras que los entregables son los resultados tangibles que se producen a lo largo del proyecto. Una clara comprensión y gestión de ambos elementos es crucial para asegurar que el cambio se implemente de manera efectiva y cumpla con las expectativas de todas las partes interesadas.
