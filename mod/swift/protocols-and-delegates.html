Introducción a los Protocolos

Los protocolos en Swift definen un esquema de métodos, propiedades y otros requisitos que se adaptan a una tarea o funcionalidad particular. Las clases, estructuras y enumeraciones pueden adoptar y conformarse a estos protocolos para proporcionar implementaciones reales de los requisitos.

Conceptos Clave

  • Definición de Protocolo: Un protocolo se define usando la palabra clave protocol.
  • Adopción y Conformidad: Los tipos (clases, estructuras, enumeraciones) adoptan un protocolo listándolo después de su nombre de tipo, separado por dos puntos.
  • Requisitos de Métodos y Propiedades: Los protocolos pueden requerir que se implementen métodos y propiedades específicos.

Ejemplo

protocol Greetable {
    var greeting: String { get }
    func greet()
}

struct Person: Greetable {
    var greeting: String
    
    func greet() {
        print(greeting)
    }
}

let person = Person(greeting: "¡Hola, Mundo!")
person.greet() // Salida: ¡Hola, Mundo!

Explicación

  • Definición de Protocolo: protocol Greetable define un protocolo con una propiedad greeting y un método greet().
  • Adopción y Conformidad: struct Person: Greetable adopta el protocolo Greetable y proporciona implementaciones para la propiedad y el método requeridos.

Características Avanzadas de los Protocolos

Los protocolos en Swift ofrecen características avanzadas como herencia de protocolos, composición de protocolos y extensiones de protocolos.

Herencia de Protocolos

Un protocolo puede heredar de uno o más otros protocolos, agregando sus propios requisitos además de los heredados.

protocol Named {
    var name: String { get }
}

protocol Aged {
    var age: Int { get }
}

protocol Person: Named, Aged {}

struct User: Person {
    var name: String
    var age: Int
}

let user = User(name: "Alice", age: 30)
print(user.name) // Salida: Alice
print(user.age)  // Salida: 30

Composición de Protocolos

Puedes combinar múltiples protocolos en un solo requisito usando la composición de protocolos.

func wishHappyBirthday(to celebrator: Named & Aged) {
    print("¡Feliz cumpleaños, \(celebrator.name), ahora tienes \(celebrator.age)!")
}

let birthdayPerson = User(name: "Bob", age: 25)
wishHappyBirthday(to: birthdayPerson) // Salida: ¡Feliz cumpleaños, Bob, ahora tienes 25!

Extensiones de Protocolos

Las extensiones de protocolos te permiten proporcionar implementaciones predeterminadas para cualquier requisito de método o propiedad computada de ese protocolo.

extension Greetable {
    func greet() {
        print("¡Hola!")
    }
}

struct Robot: Greetable {
    var greeting: String
}

let robot = Robot(greeting: "Beep Boop")
robot.greet() // Salida: ¡Hola!

Delegados

La delegación es un patrón de diseño que permite a una clase o estructura delegar algunas de sus responsabilidades a una instancia de otro tipo. Este patrón se usa a menudo para manejar eventos o fuentes de datos.

Conceptos Clave

  • Protocolo de Delegado: Define un protocolo que describe las responsabilidades a delegar.
  • Propiedad del Delegado: La clase delegante mantiene una referencia débil al delegado para evitar ciclos de retención.
  • Métodos del Delegado: El delegado implementa los métodos del protocolo para manejar las tareas delegadas.

Ejemplo

protocol TaskDelegate: AnyObject {
    func taskDidStart()
    func taskDidFinish()
}

class Task {
    weak var delegate: TaskDelegate?
    
    func start() {
        delegate?.taskDidStart()
        // Realizar tarea...
        delegate?.taskDidFinish()
    }
}

class TaskHandler: TaskDelegate {
    func taskDidStart() {
        print("Tarea iniciada")
    }
    
    func taskDidFinish() {
        print("Tarea finalizada")
    }
}

let task = Task()
let handler = TaskHandler()
task.delegate = handler
task.start()
// Salida:
// Tarea iniciada
// Tarea finalizada

Explicación

  • Protocolo de Delegado: protocol TaskDelegate define los métodos taskDidStart y taskDidFinish.
  • Propiedad del Delegado: weak var delegate: TaskDelegate? en la clase Task mantiene una referencia débil al delegado.
  • Métodos del Delegado: TaskHandler se conforma a TaskDelegate e implementa los métodos requeridos.

Conclusión

Los protocolos y delegados son herramientas poderosas en Swift que promueven la reutilización del código y la separación de responsabilidades. Los protocolos definen un esquema de métodos y propiedades, mientras que los delegados permiten que los objetos se comuniquen y manejen tareas de manera desacoplada. Al dominar estos conceptos, puedes escribir código Swift más modular, flexible y mantenible.
