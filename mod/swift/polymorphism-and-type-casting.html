Polimorfismo y Conversión de Tipos en Swift
Protocolos y Delegados ►

Introducción

El polimorfismo y la conversión de tipos son conceptos fundamentales en la programación orientada a objetos que permiten un código más flexible y reutilizable. En Swift, estos conceptos se implementan de una manera que es tanto poderosa como fácil de usar. Esta sección cubrirá los conceptos básicos del polimorfismo y la conversión de tipos, proporcionará ejemplos e incluirá ejercicios para solidificar tu comprensión.

Polimorfismo

El polimorfismo permite que los objetos sean tratados como instancias de su clase padre en lugar de su clase real. Esto permite que una sola función opere en diferentes tipos de objetos.

Conceptos Clave

  • Polimorfismo: La capacidad de diferentes clases para responder a la misma llamada de método de diferentes maneras.
  • Sobrescritura de Métodos: Las subclases pueden proporcionar una implementación específica de un método que ya está definido en su superclase.
  • Despacho Dinámico: El proceso de seleccionar qué implementación de una operación polimórfica (método) llamar en tiempo de ejecución.

Ejemplo

class Animal {
    func makeSound() {
        print("Algún sonido genérico de animal")
    }
}

class Dog: Animal {
    override func makeSound() {
        print("Ladrido")
    }
}

class Cat: Animal {
    override func makeSound() {
        print("Maullido")
    }
}

let animals: [Animal] = [Dog(), Cat(), Animal()]

for animal in animals {
    animal.makeSound()
}

En este ejemplo:

  • Tenemos una clase base Animal con un método makeSound.
  • Dog y Cat son subclases de Animal que sobrescriben el método makeSound.
  • Un array de tipo Animal contiene instancias de Dog, Cat y Animal.
  • Al iterar a través del array, se llama al método makeSound sobrescrito basado en el tipo real del objeto.

Conversión de Tipos

La conversión de tipos en Swift se utiliza para verificar el tipo de una instancia o para tratar esa instancia como una superclase o subclase diferente desde algún lugar en su jerarquía de clases.

Conceptos Clave

  • Operador is: Verifica si una instancia es de un cierto tipo de subclase.
  • Operador as?: Intenta convertir a una subclase y devuelve un opcional.
  • Operador as!: Convierte forzosamente a una subclase y puede causar un error en tiempo de ejecución si la conversión falla.
  • Operador as: Se utiliza para la conversión ascendente y la conversión de tipos.

Ejemplo

class Vehicle {
    var name: String
    init(name: String) {
        self.name = name
    }
}

class Car: Vehicle {
    var numberOfDoors: Int
    init(name: String, numberOfDoors: Int) {
        self.numberOfDoors = numberOfDoors
        super.init(name: name)
    }
}

class Bicycle: Vehicle {
    var hasBasket: Bool
    init(name: String, hasBasket: Bool) {
        self.hasBasket = hasBasket
        super.init(name: name)
    }
}

let vehicles: [Vehicle] = [Car(name: "Sedán", numberOfDoors: 4), Bicycle(name: "Bicicleta de Montaña", hasBasket: true)]

for vehicle in vehicles {
    if let car = vehicle as? Car {
        print("Coche: \(car.name), Puertas: \(car.numberOfDoors)")
    } else if let bicycle = vehicle as? Bicycle {
        print("Bicicleta: \(bicycle.name), Tiene Cesta: \(bicycle.hasBasket)")
    }
}

En este ejemplo:

  • Tenemos una clase base Vehicle con subclases Car y Bicycle.
  • Un array de tipo Vehicle contiene instancias de Car y Bicycle.
  • Usamos as? para convertir de manera segura cada Vehicle a su tipo específico de subclase y imprimir propiedades específicas.

Ejercicios

  1. Ejercicio de Polimorfismo: Crea una clase base Shape con un método area(). Crea subclases Circle y Rectangle que sobrescriban el método area(). Crea un array de objetos Shape e imprime el área de cada forma.
  2. Ejercicio de Conversión de Tipos: Crea una clase base Employee con subclases Manager y Developer. Crea un array de objetos Employee y usa la conversión de tipos para imprimir propiedades específicas de Manager y Developer.

Conclusión

El polimorfismo y la conversión de tipos son herramientas esenciales en Swift que permiten un código más flexible y reutilizable. El polimorfismo permite que los objetos sean tratados como instancias de su clase padre, mientras que la conversión de tipos te permite verificar y convertir tipos en tiempo de ejecución. Al dominar estos conceptos, puedes escribir programas en Swift más dinámicos y eficientes.
