Introducción

En JavaScript, manejar operaciones asíncronas es crucial para construir aplicaciones responsivas y eficientes. Dos métodos principales para gestionar código asíncrono son los callbacks y las promesas. Esta sección cubrirá ambos conceptos, comenzando desde lo básico y avanzando hacia un uso más avanzado.

Callbacks

¿Qué es un Callback?

  • Un callback es una función pasada a otra función como un argumento, que luego se invoca dentro de la función externa para completar algún tipo de rutina o acción.

Ejemplo Básico de un Callback

function fetchData(callback) {
    setTimeout(() => {
        const data = { name: "John", age: 30 };
        callback(data);
    }, 1000);
}

function displayData(data) {
    console.log(`Name: ${data.name}, Age: ${data.age}`);
}

fetchData(displayData);
  • fetchData es una función asíncrona que toma un callback.
  • setTimeout simula una operación asíncrona.
  • displayData es la función callback que procesa los datos.

Manejo de Errores con Callbacks

function fetchData(callback) {
    setTimeout(() => {
        const error = false; // Cambiar a true para simular un error
        if (error) {
            callback('Error occurred', null);
        } else {
            const data = { name: "John", age: 30 };
            callback(null, data);
        }
    }, 1000);
}

function displayData(error, data) {
    if (error) {
        console.log(error);
    } else {
        console.log(`Name: ${data.name}, Age: ${data.age}`);
    }
}

fetchData(displayData);
  • Patrón de callback con error primero: el primer argumento es un error, y el segundo es el resultado.

Promesas

¿Qué es una Promesa?

  • Una promesa es un objeto que representa la eventual finalización o falla de una operación asíncrona.

Ejemplo Básico de una Promesa

function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const data = { name: "John", age: 30 };
            resolve(data);
        }, 1000);
    });
}

fetchData().then(data => {
    console.log(`Name: ${data.name}, Age: ${data.age}`);
});
  • fetchData devuelve una promesa.
  • resolve se llama cuando la operación es exitosa.
  • then se usa para manejar el valor resuelto.

Manejo de Errores con Promesas

function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const error = false; // Cambiar a true para simular un error
            if (error) {
                reject('Error occurred');
            } else {
                const data = { name: "John", age: 30 };
                resolve(data);
            }
        }, 1000);
    });
}

fetchData()
    .then(data => {
        console.log(`Name: ${data.name}, Age: ${data.age}`);
    })
    .catch(error => {
        console.log(error);
    });
  • reject se llama cuando la operación falla.
  • catch se usa para manejar el valor rechazado.

Comparación: Callbacks vs Promesas

| Característica | Callbacks | Promesas | |-------------------|------------------------------------|------------------------------------| | Sintaxis | Función pasada como argumento | Objeto con then y catch | | Manejo de Errores | Patrón de callback con error primero| Método catch | | Legibilidad | Puede llevar a "infierno de callbacks" | Más legible y mantenible | | Encadenamiento | Difícil de encadenar múltiples llamadas | Fácil encadenamiento con then |

Promesas Avanzadas

Encadenamiento de Promesas

function fetchData() {
    return new Promise((resolve) => {
        setTimeout(() => {
            const data = { name: "John", age: 30 };
            resolve(data);
        }, 1000);
    });
}

function processData(data) {
    return new Promise((resolve) => {
        setTimeout(() => {
            data.processed = true;
            resolve(data);
        }, 1000);
    });
}

fetchData()
    .then(data => processData(data))
    .then(processedData => {
        console.log(processedData);
    });
  • Las promesas pueden encadenarse para realizar operaciones asíncronas secuenciales.

Usando `Promise.all`

function fetchData1() {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve("Data from API 1");
        }, 1000);
    });
}

function fetchData2() {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve("Data from API 2");
        }, 1500);
    });
}

Promise.all([fetchData1(), fetchData2()])
    .then(results => {
        console.log(results); // ["Data from API 1", "Data from API 2"]
    });
  • Promise.all ejecuta múltiples promesas en paralelo y se resuelve cuando todas las promesas se resuelven.

Conclusión

Los callbacks y las promesas son conceptos fundamentales para manejar operaciones asíncronas en JavaScript. Mientras que los callbacks son directos, pueden llevar a un código complejo y difícil de mantener. Las promesas ofrecen una forma más robusta y legible de gestionar tareas asíncronas, con mejor manejo de errores y capacidades de encadenamiento. Entender ambos métodos es crucial para escribir código JavaScript eficiente y mantenible.
