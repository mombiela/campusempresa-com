La API Fetch proporciona una forma moderna, flexible y poderosa de realizar solicitudes de red en JavaScript. Es un reemplazo para la antigua API XMLHttpRequest (XHR) y está integrada en los navegadores modernos.

Introducción a la API Fetch

  • ¿Qué es la API Fetch?

    • Una interfaz moderna para realizar solicitudes HTTP.
    • Devuelve Promesas, lo que facilita el trabajo con operaciones asíncronas.
    • Soporta varios métodos HTTP como GET, POST, PUT, DELETE, etc.

  • ¿Por qué usar la API Fetch?

    • Sintaxis más simple y limpia en comparación con XMLHttpRequest.
    • Mejor soporte para manejar datos JSON.
    • Soporte integrado para Promesas, lo que facilita el manejo del código asíncrono.

Uso Básico

Realizando una Solicitud GET

Para obtener datos de un servidor, puedes usar la función fetch. Aquí hay un ejemplo básico de cómo hacer una solicitud GET:

fetch('https://api.example.com/data')
  .then(response => response.json()) // Analiza el JSON de la respuesta
  .then(data => console.log(data)) // Maneja los datos
  .catch(error => console.error('Error:', error)); // Maneja cualquier error
  • Explicación:
    • fetch('https://api.example.com/data'): Inicia una solicitud GET a la URL especificada.
    • .then(response => response.json()): Convierte la respuesta a JSON.
    • .then(data => console.log(data)): Muestra los datos en la consola.
    • .catch(error => console.error('Error:', error)): Captura y muestra cualquier error.

Realizando una Solicitud POST

Para enviar datos a un servidor, puedes usar el método POST:

fetch('https://api.example.com/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ key: 'value' }) // Convierte los datos a una cadena JSON
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));
  • Explicación:
    • method: 'POST': Especifica el método HTTP.
    • headers: { 'Content-Type': 'application/json' }: Establece el tipo de contenido a JSON.
    • body: JSON.stringify({ key: 'value' }): Convierte el objeto JavaScript a una cadena JSON.

Uso Avanzado

Manejo de Diferentes Tipos de Respuesta

Fetch puede manejar varios tipos de respuesta como texto, blob y arrayBuffer:

fetch('https://api.example.com/data')
  .then(response => response.text()) // Para respuesta en texto plano
  .then(text => console.log(text))
  .catch(error => console.error('Error:', error));

Manejo de Errores HTTP

Fetch no rechaza automáticamente los errores HTTP (por ejemplo, 404 o 500). Necesitas manejarlos manualmente:

fetch('https://api.example.com/data')
  .then(response => {
    if (!response.ok) {
      throw new Error('La respuesta de la red no fue satisfactoria ' + response.statusText);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Usando Async/Await con Fetch

La sintaxis async/await puede hacer que tu código sea más legible:

async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    if (!response.ok) {
      throw new Error('La respuesta de la red no fue satisfactoria ' + response.statusText);
    }
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error:', error);
  }
}

fetchData();

Comparación: API Fetch vs. XMLHttpRequest

| Característica | API Fetch | XMLHttpRequest | |---------------------------|------------------------------------|----------------------------------| | Sintaxis | Más limpia y concisa | Verbosa y compleja | | Promesas | Soporte integrado | Requiere manejo manual | | Manejo de Respuestas | Más fácil con métodos integrados | Requiere análisis manual | | Manejo de Errores | Manual pero directo | Manual y complejo | | Características Modernas | Soporta streaming y otras características web modernas | Limitado |

Conclusión

La API Fetch es una herramienta poderosa y flexible para realizar solicitudes HTTP en JavaScript. Su sintaxis moderna y soporte integrado para Promesas la convierten en una opción preferida sobre la antigua XMLHttpRequest. Al comprender tanto el uso básico como el avanzado, puedes manejar eficazmente varias operaciones de red en tus aplicaciones.
