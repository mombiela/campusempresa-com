Introducción a la Programación Asíncrona

  • Programación Asíncrona: Permite que el programa inicie una tarea potencialmente prolongada y continúe con otras tareas antes de que la primera termine.
  • Callbacks: Funciones pasadas como argumentos a otras funciones para ser ejecutadas más tarde.
  • Promises: Objetos que representan la eventual finalización (o falla) de una operación asíncrona y su valor resultante.

Entendiendo las Promesas

  • Estados de una Promesa:
    • Pending: Estado inicial, ni cumplida ni rechazada.
    • Fulfilled: Operación completada con éxito.
    • Rejected: Operación fallida.
  • Creando una Promesa: 
    const myPromise = new Promise((resolve, reject) => {
  // Operación asíncrona
  if (/* operación exitosa */) {
    resolve('¡Éxito!');
  } else {
    reject('¡Fallo!');
  }
});

Introducción a Async/Await

  • Funciones Async: Funciones declaradas con la palabra clave async, que devuelven una Promesa.
  • Palabra clave Await: Usada dentro de una función async para pausar la ejecución hasta que una Promesa sea resuelta o rechazada.

Usando Async/Await

  • Sintaxis Básica: 
    async function fetchData() {
  try {
    let response = await fetch('https://api.example.com/data');
    let data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error:', error);
  }
}
fetchData();
    • async function fetchData() {}: Declara una función async.
    • await fetch('https://api.example.com/data'): Pausa la ejecución hasta que la Promesa fetch sea resuelta.
    • try...catch: Maneja cualquier error que ocurra durante las operaciones async.

Manejo de Errores con Async/Await

  • Usando try...catch: 
    async function getData() {
  try {
    let response = await fetch('https://api.example.com/data');
    if (!response.ok) {
      throw new Error('La respuesta de la red no fue satisfactoria');
    }
    let data = await response.json();
    return data;
  } catch (error) {
    console.error('Error en la obtención:', error);
  }
}

Encadenando Funciones Async

  • Ejecución Secuencial: 
    async function processData() {
  let data1 = await fetchData1();
  let data2 = await fetchData2(data1);
  let result = await processData(data2);
  return result;
}

Ejecución Paralela con Async/Await

  • Usando Promise.all: 
    async function fetchMultipleData() {
  try {
    let [data1, data2] = await Promise.all([
      fetch('https://api.example.com/data1').then(response => response.json()),
      fetch('https://api.example.com/data2').then(response => response.json())
    ]);
    console.log(data1, data2);
  } catch (error) {
    console.error('Error al obtener los datos:', error);
  }
}
fetchMultipleData();

Patrones Avanzados

  • Reintentando Solicitudes Fallidas: 
    async function fetchWithRetry(url, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      let response = await fetch(url);
      if (!response.ok) throw new Error('La respuesta de la red no fue satisfactoria');
      return await response.json();
    } catch (error) {
      if (i === retries - 1) throw error;
    }
  }
}

Conclusión

  • Async/Await simplifica el trabajo con Promesas, haciendo que el código asíncrono sea más fácil de leer y escribir.
  • Manejo de Errores es sencillo con bloques try...catch.
  • Ejecución Secuencial y Paralela puede ser gestionada efectivamente usando async/await y Promise.all.
  • Patrones Avanzados como reintentar solicitudes fallidas pueden ser implementados para manejar escenarios más complejos.

Al dominar async/await, puedes escribir código JavaScript asíncrono más limpio y mantenible.
