Introducción a los Módulos

Los módulos de JavaScript te permiten descomponer tu código en piezas más pequeñas y reutilizables. Esto hace que tu código esté más organizado, sea más fácil de mantener y ayuda a evitar conflictos de nombres.

Conceptos Clave

  • Módulo: Un archivo que contiene código JavaScript que puede ser importado y usado en otros archivos JavaScript.
  • Exportar: La palabra clave usada para hacer funciones, objetos o valores primitivos disponibles para importar en otros módulos.
  • Importar: La palabra clave usada para traer funciones, objetos o valores primitivos de otros módulos.

Creación y Exportación de Módulos

Para crear un módulo, simplemente escribes código JavaScript en un archivo separado y usas la palabra clave export para hacer partes del código disponibles para importar.

Exportaciones Nombradas

Las exportaciones nombradas te permiten exportar múltiples valores de un módulo.

// math.js
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;

Exportaciones por Defecto

Las exportaciones por defecto se usan para exportar un solo valor de un módulo.

// greet.js
const greet = (name) => `¡Hola, ${name}!`;
export default greet;

Importación de Módulos

Para usar los valores exportados en otro archivo, usas la palabra clave import.

Importación de Exportaciones Nombradas

Al importar exportaciones nombradas, necesitas usar los mismos nombres que los valores exportados.

// main.js
import { add, subtract } from './math.js';

console.log(add(2, 3)); // 5
console.log(subtract(5, 2)); // 3

Importación de Exportaciones por Defecto

Al importar exportaciones por defecto, puedes usar cualquier nombre que desees.

// main.js
import greet from './greet.js';

console.log(greet('Alice')); // ¡Hola, Alice!

Importación de Todas las Exportaciones Nombradas

Puedes importar todas las exportaciones nombradas de un módulo usando la sintaxis *.

// main.js
import * as math from './math.js';

console.log(math.add(2, 3)); // 5
console.log(math.subtract(5, 2)); // 3

Combinando Exportaciones por Defecto y Nombradas

Un módulo puede tener tanto exportaciones por defecto como nombradas.

// utils.js
export const multiply = (a, b) => a * b;
const divide = (a, b) => a / b;
export default divide;
// main.js
import divide, { multiply } from './utils.js';

console.log(multiply(2, 3)); // 6
console.log(divide(6, 2)); // 3

Conceptos Avanzados de Módulos

Importaciones Dinámicas

Las importaciones dinámicas te permiten cargar módulos condicionalmente o bajo demanda.

// main.js
if (condition) {
  import('./moduleA.js').then(moduleA => {
    moduleA.doSomething();
  });
} else {
  import('./moduleB.js').then(moduleB => {
    moduleB.doSomethingElse();
  });
}

Agrupadores de Módulos

Herramientas como Webpack, Rollup y Parcel pueden agrupar múltiples módulos en un solo archivo para su despliegue.

Módulos ES vs CommonJS

JavaScript tiene dos sistemas principales de módulos: Módulos ES (ESM) y CommonJS (CJS). ESM es el estándar para JavaScript moderno, mientras que CJS se usa principalmente en Node.js.

| Característica | Módulos ES (ESM) | CommonJS (CJS) | |------------------|------------------|----------------| | Sintaxis | import/export| require/module.exports | | Exportación por Defecto | export default | module.exports = | | Estático/Dinámico| Estático | Dinámico | | Soporte en Navegadores | Sí | No |

Conclusión

Los módulos son una característica poderosa en JavaScript que te ayudan a organizar y gestionar tu código de manera más efectiva. Al entender cómo crear, exportar e importar módulos, puedes escribir código más limpio y mantenible. A medida que avances, puedes explorar importaciones dinámicas y agrupadores de módulos para optimizar aún más tus proyectos.
