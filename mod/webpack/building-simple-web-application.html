Introducción a Webpack

  • ¿Qué es Webpack?

    • Webpack es un empaquetador de módulos para aplicaciones JavaScript.
    • Toma módulos con dependencias y genera activos estáticos que representan esos módulos.

  • ¿Por qué usar Webpack?

    • Gestiona eficientemente las dependencias.
    • Optimiza el código para producción.
    • Soporta características y herramientas modernas de JavaScript.

Configurando Webpack

  • Instalando Webpack

    • Instalar Webpack y Webpack CLI vía npm: 
      npm install --save-dev webpack webpack-cli

  • Creando la Estructura del Proyecto

    • Estructura básica del proyecto: 
      my-app/
├── src/
│   └── index.js
├── dist/
├── package.json
└── webpack.config.js

Configurando Webpack

  • Configuración Básica

    • Crear webpack.config.js en el directorio raíz: 
      const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  mode: 'development'
};

  • Ejecutando Webpack

    • Agregar un script a package.json: 
      "scripts": {
  "build": "webpack"
}
    • Ejecutar el proceso de construcción: 
      npm run build

Cargando y Empaquetando Activos

  • Manejando CSS

    • Instalar loaders: 
      npm install --save-dev style-loader css-loader
    • Actualizar webpack.config.js: 
      module.exports = {
  // ...otras configuraciones
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      }
    ]
  }
};

  • Manejando Imágenes

    • Instalar file-loader: 
      npm install --save-dev file-loader
    • Actualizar webpack.config.js: 
      module.exports = {
  // ...otras configuraciones
  module: {
    rules: [
      {
        test: /\.(png|svg|jpg|gif)$/,
        use: ['file-loader']
      }
    ]
  }
};

Configuración Avanzada

  • Usando Plugins

    • Instalar HTML Webpack Plugin: 
      npm install --save-dev html-webpack-plugin
    • Actualizar webpack.config.js: 
      const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  // ...otras configuraciones
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};

  • División de Código

    • Modificar webpack.config.js para habilitar la división de código: 
      module.exports = {
  // ...otras configuraciones
  optimization: {
    splitChunks: {
      chunks: 'all'
    }
  }
};

Herramientas de Desarrollo

  • Usando Webpack Dev Server
    • Instalar Webpack Dev Server: 
      npm install --save-dev webpack-dev-server
    • Actualizar webpack.config.js: 
      module.exports = {
  // ...otras configuraciones
  devServer: {
    contentBase: './dist'
  }
};
    • Agregar un script a package.json: 
      "scripts": {
  "start": "webpack serve --open"
}
    • Iniciar el servidor de desarrollo: 
      npm start

Conclusión

  • Resumen

    • Webpack es una herramienta poderosa para empaquetar aplicaciones JavaScript.
    • Simplifica la gestión de dependencias y optimiza el código para producción.
    • Usando loaders y plugins, Webpack puede manejar varios tipos de activos y mejorar la experiencia de desarrollo.

  • Próximos Pasos

    • Explorar características más avanzadas de Webpack como tree shaking y plugins personalizados.
    • Integrar Webpack con otras herramientas y frameworks como React o Vue.js.
    • Continuar practicando construyendo aplicaciones y configuraciones más complejas.
