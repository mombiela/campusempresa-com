Caché en Webpack
Sacudida de Árbol ►

El caché es un concepto crucial en el desarrollo web que ayuda a mejorar el rendimiento de tus aplicaciones web al almacenar datos a los que se accede con frecuencia. En esta sección, exploraremos cómo Webpack maneja el caché, desde técnicas básicas hasta avanzadas.

Introducción al Caché

  • Definición: El caché es el proceso de almacenar copias de archivos o datos en un caché, o ubicación de almacenamiento temporal, para que puedan ser accedidos más rápidamente.
  • Importancia: Reduce los tiempos de carga, disminuye la carga del servidor y mejora la experiencia del usuario.

Caché Básico con Webpack

Nombres de Archivos de Salida

Para aprovechar el caché, necesitas asegurarte de que los nombres de los archivos de salida cambien cada vez que el contenido cambie. Esto se puede hacer usando hashes de contenido.

module.exports = {
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
};
  • [name]: El nombre del punto de entrada.
  • [contenthash]: Un hash único basado en el contenido del archivo.

Ejemplo

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
};

Esta configuración asegura que los nombres de los archivos de salida cambien cada vez que el contenido cambie, ayudando a los navegadores a almacenar en caché los archivos de manera efectiva.

Técnicas Avanzadas de Caché

SplitChunksPlugin

El SplitChunksPlugin de Webpack se puede usar para dividir tu código en fragmentos más pequeños, que pueden ser almacenados en caché por separado.

module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
    },
  },
};

Ejemplo

const path = require('path');

module.exports = {
  entry: {
    app: './src/index.js',
  },
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
  optimization: {
    splitChunks: {
      chunks: 'all',
    },
  },
};

Grupos de Caché

Puedes personalizar aún más el comportamiento de SplitChunksPlugin usando grupos de caché.

module.exports = {
  optimization: {
    splitChunks: {
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};

Ejemplo

const path = require('path');

module.exports = {
  entry: {
    app: './src/index.js',
  },
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
  optimization: {
    splitChunks: {
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};

Gestión del Caché a Largo Plazo

Identificadores de Módulos

Webpack usa identificadores de módulos para rastrear módulos. Para el caché a largo plazo, puedes usar el HashedModuleIdsPlugin para asegurar que los identificadores de módulos sean consistentes entre compilaciones.

const webpack = require('webpack');

module.exports = {
  plugins: [
    new webpack.HashedModuleIdsPlugin(),
  ],
};

Ejemplo

const path = require('path');
const webpack = require('webpack');

module.exports = {
  entry: {
    app: './src/index.js',
  },
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
  optimization: {
    splitChunks: {
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
  plugins: [
    new webpack.HashedModuleIdsPlugin(),
  ],
};

Resumen

El caché es una técnica poderosa para mejorar el rendimiento de las aplicaciones web. Al usar las características de Webpack como hashes de contenido, SplitChunksPlugin y HashedModuleIdsPlugin, puedes gestionar efectivamente el caché y asegurar que tus usuarios tengan una experiencia rápida y fluida.

  • Caché Básico: Usa hashes de contenido en los nombres de archivos.
  • Caché Avanzado: Usa SplitChunksPlugin y grupos de caché.
  • Caché a Largo Plazo: Usa HashedModuleIdsPlugin para identificadores de módulos consistentes.

Al dominar estas técnicas, puedes mejorar significativamente el rendimiento y la fiabilidad de tus aplicaciones web.
