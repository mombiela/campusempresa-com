Introducción a Webpack

Webpack es un potente empaquetador de módulos para aplicaciones JavaScript. Toma módulos con dependencias y genera activos estáticos que representan esos módulos. Entender cómo planificar y configurar Webpack es crucial para flujos de trabajo eficientes de desarrollo y producción.

Conceptos Clave

  • Punto de Entrada: El punto de inicio para que Webpack construya su gráfico de dependencias interno.
  • Salida: Dónde se emitirán los archivos empaquetados.
  • Cargadores: Transformaciones aplicadas al código fuente de un módulo.
  • Plugins: Extienden o personalizan el proceso de construcción de Webpack.
  • Modo: Define el entorno (desarrollo o producción) para optimizar la construcción en consecuencia.

Configuración Inicial

Instalando Webpack

Para comenzar con Webpack, necesitas instalarlo a través de npm (Node Package Manager).

npm install --save-dev webpack webpack-cli

Configuración Básica

Crea un archivo webpack.config.js en la raíz de tu proyecto. Este archivo contendrá la configuración para Webpack.

// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js', // Punto de entrada
  output: {
    filename: 'bundle.js', // Archivo de salida
    path: path.resolve(__dirname, 'dist') // Directorio de salida
  },
  mode: 'development' // Modo
};

Explicación:

  • entry: Especifica el punto de entrada de tu aplicación.
  • output: Define el nombre del archivo de salida y el directorio.
  • mode: Establece el modo en 'development' para una mejor experiencia de depuración.

Uso de Cargadores

Los cargadores te permiten preprocesar archivos mientras los importas o cargas. Por ejemplo, puedes usar Babel para transpilar JavaScript moderno a versiones más antiguas para compatibilidad.

Instalando Babel Loader

npm install --save-dev babel-loader @babel/core @babel/preset-env

Configurando Babel Loader

Actualiza tu webpack.config.js para incluir el cargador de Babel.

// webpack.config.js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  mode: 'development',
  module: {
    rules: [
      {
        test: /\.js$/, // Aplica esta regla a archivos .js
        exclude: /node_modules/, // Excluye node_modules
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env'] // Usa el preset-env
          }
        }
      }
    ]
  }
};

Uso de Plugins

Los plugins pueden realizar una gama más amplia de tareas como optimización de paquetes, gestión de activos e inyección de variables de entorno.

Instalando HTML Webpack Plugin

npm install --save-dev html-webpack-plugin

Configurando HTML Webpack Plugin

Actualiza tu webpack.config.js para incluir el HTML Webpack Plugin.

// webpack.config.js
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  mode: 'development',
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html' // Archivo de plantilla
    })
  ]
};

Desarrollo vs. Producción

Webpack puede configurarse de manera diferente para entornos de desarrollo y producción para optimizar el proceso de construcción.

Configuración de Configuraciones Específicas del Entorno

Crea archivos de configuración separados para desarrollo y producción.

webpack.dev.js

// webpack.dev.js
const { merge } = require('webpack-merge');
const common = require('./webpack.common.js');

module.exports = merge(common, {
  mode: 'development',
  devtool: 'inline-source-map', // Mapas de origen para depuración
  devServer: {
    contentBase: './dist'
  }
});

webpack.prod.js

// webpack.prod.js
const { merge } = require('webpack-merge');
const common = require('./webpack.common.js');

module.exports = merge(common, {
  mode: 'production',
  optimization: {
    minimize: true // Minimiza la salida
  }
});

webpack.common.js

// webpack.common.js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};

Ejecutando Diferentes Configuraciones

Usa scripts de npm para ejecutar diferentes configuraciones.

// package.json
{
  "scripts": {
    "start": "webpack serve --config webpack.dev.js",
    "build": "webpack --config webpack.prod.js"
  }
}

Conclusión

Configurar Webpack implica entender sus conceptos básicos como puntos de entrada, salida, cargadores, plugins y modos. Siguiendo los pasos descritos en esta sección, puedes crear una configuración básica de Webpack y extenderla para entornos de desarrollo y producción. Este conocimiento fundamental te ayudará a construir configuraciones de Webpack más complejas y optimizadas a medida que avances en el curso.
