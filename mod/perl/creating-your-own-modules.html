Creación de tus propios módulos
Referencias ►

Crear tus propios módulos en Perl es una habilidad esencial que te permite organizar tu código, promover la reutilización y mantener una base de código limpia. Este tema te guiará a través del proceso de creación, uso y distribución de tus propios módulos Perl, desde niveles principiantes hasta avanzados.

Introducción a los módulos Perl

  • ¿Qué es un módulo?

    • Un módulo en Perl es un paquete que puede ser reutilizado en otros scripts Perl.
    • Típicamente contiene una colección de funciones y variables relacionadas.

  • ¿Por qué usar módulos?

    • Reutilización de código.
    • Mejor organización.
    • Mantenimiento y depuración más fáciles.

Creación de un módulo Perl simple

Guía paso a paso

  1. Crear un nuevo archivo para tu módulo

    • El nombre del archivo debe coincidir con el nombre del paquete y tener una extensión .pm.
    • Ejemplo: MyModule.pm.

  2. Definir el paquete

    • Usa la palabra clave package para definir el módulo.
    • Ejemplo: 
      package MyModule;

  3. Agregar funciones y variables

    • Define cualquier función o variable que quieras incluir en tu módulo.
    • Ejemplo: 
      package MyModule;

use strict;
use warnings;

sub hello {
    return "¡Hola, Mundo!";
}

1;  # El final del módulo debe devolver un valor verdadero

  4. Guardar el archivo

    • Guarda el archivo en un directorio donde Perl pueda encontrarlo, típicamente en el mismo directorio que tu script o en un directorio listado en @INC.

Usar tu módulo

  1. Incluir el módulo en tu script

    • Usa la palabra clave use para incluir tu módulo.
    • Ejemplo: 
      use MyModule;

print MyModule::hello();

  2. Ejecutar tu script

    • Ejecuta tu script para ver la salida de tu módulo.

Características avanzadas del módulo

Exportar funciones

  • Exportar funciones automáticamente

    • Usa el módulo Exporter para exportar funciones automáticamente.
    • Ejemplo: 
      package MyModule;
use strict;
use warnings;
use Exporter 'import';

our @EXPORT = qw(hello);

sub hello {
    return "¡Hola, Mundo!";
}

1;

  • Usar funciones exportadas

    • Ejemplo: 
      use MyModule;

print hello();  # No es necesario usar MyModule::hello()

Módulos orientados a objetos

  • Crear un módulo orientado a objetos

    • Define un constructor y métodos.
    • Ejemplo: 
      package MyModule;
use strict;
use warnings;

sub new {
    my $class = shift;
    my $self = {
        name => shift,
    };
    bless $self, $class;
    return $self;
}

sub get_name {
    my $self = shift;
    return $self->{name};
}

1;

  • Usar un módulo orientado a objetos

    • Ejemplo: 
      use MyModule;

my $obj = MyModule->new("Perl");
print $obj->get_name();  # Salida: Perl

Distribuir tu módulo

Crear una distribución CPAN

  1. Preparar tu módulo

    • Asegúrate de que tu módulo tenga un nombre de paquete y una versión adecuados.
    • Ejemplo: 
      package MyModule;
our $VERSION = '1.00';

  2. Crear un Makefile.PL

    • Usa ExtUtils::MakeMaker para crear un Makefile.PL.
    • Ejemplo: 
      use ExtUtils::MakeMaker;

WriteMakefile(
    NAME         => 'MyModule',
    VERSION_FROM => 'MyModule.pm',
    AUTHOR       => 'Tu Nombre <tu@example.com>',
    ABSTRACT     => 'Un módulo de ejemplo simple',
);

  3. Construir y probar tu módulo

    • Ejecuta los siguientes comandos: 
      perl Makefile.PL
make
make test

  4. Subir a CPAN

    • Sigue las directrices de CPAN para subir tu módulo.

Conclusión

Crear tus propios módulos en Perl es una forma poderosa de mejorar tus habilidades de programación y contribuir a la comunidad Perl. Al organizar tu código en módulos reutilizables, puedes mejorar el mantenimiento del código, promover la reutilización y compartir tu trabajo con otros. Ya sea que estés creando módulos procedurales simples o avanzados orientados a objetos, los principios y técnicas cubiertos en este tema te ayudarán a desarrollar módulos Perl robustos y eficientes.
