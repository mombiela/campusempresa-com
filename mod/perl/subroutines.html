Introducción a las Subrutinas

Las subrutinas, también conocidas como funciones o métodos en otros lenguajes de programación, son bloques de código reutilizables que realizan una tarea específica. En Perl, las subrutinas ayudan a organizar el código, reducir la redundancia y mejorar la legibilidad.

Conceptos Clave

  • Definición: Una subrutina se define usando la palabra clave sub seguida del nombre de la subrutina y un bloque de código.
  • Llamada a una Subrutina: Las subrutinas se llaman por su nombre seguido de paréntesis.
  • Argumentos: Las subrutinas pueden aceptar argumentos, que se pasan como una lista.
  • Valores de Retorno: Las subrutinas pueden devolver valores usando la palabra clave return.

Definiendo y Llamando Subrutinas

Para definir una subrutina en Perl, usa la palabra clave sub seguida del nombre de la subrutina y un bloque de código.

sub say_hello {
    print "¡Hola, Mundo!\n";
}

Para llamar a la subrutina, simplemente usa su nombre seguido de paréntesis:

say_hello();

Ejemplo

sub greet {
    my $name = shift;  # Recupera el primer argumento
    print "¡Hola, $name!\n";
}

greet("Alice");  # Salida: ¡Hola, Alice!

Pasando Argumentos a Subrutinas

Los argumentos se pasan a las subrutinas como una lista. Dentro de la subrutina, puedes acceder a estos argumentos usando el array especial @_.

Ejemplo

sub add {
    my ($a, $b) = @_;  # Desempaqueta los argumentos
    return $a + $b;
}

my $sum = add(3, 4);  # $sum ahora contiene el valor 7
print "Suma: $sum\n";  # Salida: Suma: 7

Devolviendo Valores desde Subrutinas

Las subrutinas pueden devolver valores usando la palabra clave return. Si return se omite, la última expresión evaluada se devuelve por defecto.

Ejemplo

sub multiply {
    my ($x, $y) = @_;
    return $x * $y;
}

my $product = multiply(5, 6);  # $product ahora contiene el valor 30
print "Producto: $product\n";  # Salida: Producto: 30

Características Avanzadas de las Subrutinas

Sensibilidad al Contexto

Las subrutinas en Perl pueden comportarse de manera diferente según el contexto en el que se llamen (contexto escalar o de lista).

Ejemplo

sub context_example {
    return wantarray ? (1, 2, 3) : "escalar";
}

my @array = context_example();  # Contexto de lista
print "@array\n";  # Salida: 1 2 3

my $scalar = context_example();  # Contexto escalar
print "$scalar\n";  # Salida: escalar

Subrutinas Anónimas

Las subrutinas anónimas son subrutinas sin nombre, a menudo usadas como argumentos para otras funciones o para crear cierres.

Ejemplo

my $anon_sub = sub {
    my ($x, $y) = @_;
    return $x - $y;
};

my $result = $anon_sub->(10, 3);  # $result ahora contiene el valor 7
print "Resultado: $result\n";  # Salida: Resultado: 7

Conclusión

Las subrutinas en Perl son herramientas poderosas para organizar y reutilizar código. Al entender cómo definir, llamar y pasar argumentos a subrutinas, así como cómo manejar valores de retorno y sensibilidad al contexto, puedes escribir programas en Perl más modulares y mantenibles. Practica creando y usando subrutinas para volverte competente en su uso.
