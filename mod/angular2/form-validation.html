Introducción a la Validación de Formularios

La validación de formularios es un aspecto crucial del desarrollo web, asegurando que las entradas del usuario sean correctas y seguras antes de procesarlas. Angular 2+ proporciona herramientas poderosas para manejar la validación de formularios de manera eficiente.

Tipos de Formularios en Angular

Angular soporta dos tipos de formularios:

  • Formularios basados en plantillas: Más fáciles de usar, adecuados para formularios simples.
  • Formularios reactivos: Más flexibles y escalables, adecuados para formularios complejos.

Formularios Basados en Plantillas

Los formularios basados en plantillas dependen en gran medida de las directivas de Angular y se definen en la plantilla.

Creación de un Formulario Básico Basado en Plantillas

<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Nombre:</label>
    <input type="text" id="name" name="name" ngModel required>
  </div>
  <div>
    <label for="email">Correo Electrónico:</label>
    <input type="email" id="email" name="email" ngModel required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>

Validación de Formularios Basados en Plantillas

  • Requerido: Asegura que el campo no esté vacío.
  • Correo Electrónico: Asegura que el campo contenga una dirección de correo electrónico válida.

Mostrando Errores de Validación

<div *ngIf="userForm.controls.name?.invalid && userForm.controls.name?.touched">
  <small *ngIf="userForm.controls.name?.errors?.required">El nombre es obligatorio.</small>
</div>
<div *ngIf="userForm.controls.email?.invalid && userForm.controls.email?.touched">
  <small *ngIf="userForm.controls.email?.errors?.required">El correo electrónico es obligatorio.</small>
  <small *ngIf="userForm.controls.email?.errors?.email">Dirección de correo electrónico no válida.</small>
</div>

Formularios Reactivos

Los formularios reactivos son más robustos y proporcionan un mayor control sobre la validación de formularios.

Creación de un Formulario Reactivo Básico

import { Component } from '@angular/core';
import { FormBuilder, FormGroup, Validators } from '@angular/forms';

@Component({
  selector: 'app-user-form',
  templateUrl: './user-form.component.html'
})
export class UserFormComponent {
  userForm: FormGroup;

  constructor(private fb: FormBuilder) {
    this.userForm = this.fb.group({
      name: ['', Validators.required],
      email: ['', [Validators.required, Validators.email]]
    });
  }

  onSubmit() {
    if (this.userForm.valid) {
      console.log(this.userForm.value);
    }
  }
}

Plantilla para Formulario Reactivo

<form [formGroup]="userForm" (ngSubmit)="onSubmit()">
  <div>
    <label for="name">Nombre:</label>
    <input type="text" id="name" formControlName="name">
    <div *ngIf="userForm.controls.name.invalid && userForm.controls.name.touched">
      <small *ngIf="userForm.controls.name.errors.required">El nombre es obligatorio.</small>
    </div>
  </div>
  <div>
    <label for="email">Correo Electrónico:</label>
    <input type="email" id="email" formControlName="email">
    <div *ngIf="userForm.controls.email.invalid && userForm.controls.email.touched">
      <small *ngIf="userForm.controls.email.errors.required">El correo electrónico es obligatorio.</small>
      <small *ngIf="userForm.controls.email.errors.email">Dirección de correo electrónico no válida.</small>
    </div>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>

Validadores Personalizados

Los validadores personalizados te permiten crear lógica de validación específica.

Creación de un Validador Personalizado

import { AbstractControl, ValidationErrors } from '@angular/forms';

export function forbiddenNameValidator(control: AbstractControl): ValidationErrors | null {
  const forbidden = /admin/.test(control.value);
  return forbidden ? { forbiddenName: { value: control.value } } : null;
}

Uso de Validadores Personalizados

this.userForm = this.fb.group({
  name: ['', [Validators.required, forbiddenNameValidator]],
  email: ['', [Validators.required, Validators.email]]
});

Mostrando Errores de Validación Personalizados

<div *ngIf="userForm.controls.name.invalid && userForm.controls.name.touched">
  <small *ngIf="userForm.controls.name.errors.required">El nombre es obligatorio.</small>
  <small *ngIf="userForm.controls.name.errors.forbiddenName">El nombre no puede ser 'admin'.</small>
</div>

Resumen

La validación de formularios en Angular 2+ puede manejarse usando formularios basados en plantillas o formularios reactivos. Los formularios basados en plantillas son más simples y adecuados para casos de uso básicos, mientras que los formularios reactivos ofrecen más control y escalabilidad para escenarios complejos. Los validadores personalizados proporcionan flexibilidad para implementar lógica de validación específica. Entender estos conceptos te ayudará a crear formularios robustos y fáciles de usar en tus aplicaciones Angular.
