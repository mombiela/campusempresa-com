Introducción a las Rutas Hijas

Las rutas hijas en Angular te permiten crear rutas anidadas dentro de una ruta principal. Esto es útil para construir aplicaciones complejas con múltiples niveles de navegación. Entender las rutas hijas es esencial para crear aplicaciones modulares y mantenibles en Angular.

Conceptos Clave

  • Ruta Principal: La ruta principal que contiene una o más rutas hijas.
  • Ruta Hija: Una ruta anidada dentro de una ruta principal.
  • Módulo de Enrutador: Módulo de Angular que proporciona capacidades de navegación y manipulación de URL.
  • Configuración de Rutas: Un array de definiciones de rutas que especifican los caminos y componentes.

Configuración de Rutas Hijas

Paso 1: Definir Componentes Principal e Hijo

Primero, crea los componentes que servirán como principal e hijo en tu estructura de enrutamiento.

// parent.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  template: `
    <h2>Componente Principal</h2>
    <router-outlet></router-outlet>
  `
})
export class ParentComponent {}

// child.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `<h3>Componente Hijo</h3>`
})
export class ChildComponent {}

Paso 2: Configurar Rutas

A continuación, configura las rutas en tu AppRoutingModule.

// app-routing.module.ts
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { ParentComponent } from './parent.component';
import { ChildComponent } from './child.component';

const routes: Routes = [
  {
    path: 'parent',
    component: ParentComponent,
    children: [
      {
        path: 'child',
        component: ChildComponent
      }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule {}

Paso 3: Actualizar el Módulo de la Aplicación

Asegúrate de que tus componentes y el módulo de enrutamiento estén declarados en tu AppModule.

// app.module.ts
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppRoutingModule } from './app-routing.module';
import { AppComponent } from './app.component';
import { ParentComponent } from './parent.component';
import { ChildComponent } from './child.component';

@NgModule({
  declarations: [
    AppComponent,
    ParentComponent,
    ChildComponent
  ],
  imports: [
    BrowserModule,
    AppRoutingModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule {}

Paso 4: Navegar a las Rutas Hijas

Para navegar a la ruta hija, puedes usar la directiva de enlace del enrutador de Angular.

<!-- app.component.html -->
<nav>
  <a routerLink="/parent">Principal</a>
  <a routerLink="/parent/child">Hijo</a>
</nav>
<router-outlet></router-outlet>

Resumen

Las rutas hijas en Angular te permiten crear estructuras de navegación anidadas, haciendo tu aplicación más modular y fácil de gestionar. Al definir componentes principales e hijos, configurar rutas y usar la directiva de enlace del enrutador, puedes implementar efectivamente rutas hijas en tu aplicación Angular. Entender y utilizar las rutas hijas es crucial para construir aplicaciones Angular escalables y mantenibles.
