Introducción al Enrutamiento en Angular

El enrutamiento en Angular te permite navegar entre diferentes vistas o componentes dentro de tu aplicación. Es un aspecto crucial de las aplicaciones de una sola página (SPAs) ya que permite al usuario moverse entre diferentes partes de la aplicación sin recargar toda la página.

Conceptos Clave del Enrutamiento en Angular

  • Router Module: El módulo de Angular que proporciona los servicios y directivas necesarios para el enrutamiento.
  • Routes: Objetos de configuración que definen la asignación entre rutas URL y componentes.
  • Router Outlet: Una directiva que actúa como un marcador de posición para el componente enrutado.
  • Router Link: Una directiva para enlazar a diferentes rutas.

Configuración de Rutas Básicas

Para configurar el enrutamiento en una aplicación Angular, sigue estos pasos:

  1. Instalar Angular Router: Si no está ya incluido, instala el paquete Angular Router.

    npm install @angular/router

  2. Importar RouterModule: Importa RouterModule y Routes en tu módulo principal de la aplicación.

    import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { RouterModule, Routes } from '@angular/router';

import { AppComponent } from './app.component';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  declarations: [
    AppComponent,
    HomeComponent,
    AboutComponent
  ],
  imports: [
    BrowserModule,
    RouterModule.forRoot(routes)
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

  3. Agregar Router Outlet: Agrega la directiva <router-outlet> en la plantilla de tu componente principal.

    <!-- app.component.html -->
<nav>
  <a routerLink="/">Home</a>
  <a routerLink="/about">About</a>
</nav>
<router-outlet></router-outlet>

  4. Crear Componentes: Asegúrate de que los componentes a los que deseas enrutar estén creados.

    // home.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-home',
  template: `<h1>Home</h1>`
})
export class HomeComponent { }

// about.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-about',
  template: `<h1>About</h1>`
})
export class AboutComponent { }

Técnicas Avanzadas de Enrutamiento

Rutas Anidadas

Las rutas anidadas te permiten definir rutas dentro de otras rutas, creando una jerarquía.

const routes: Routes = [
  { path: '', component: HomeComponent },
  { 
    path: 'about', 
    component: AboutComponent,
    children: [
      { path: 'team', component: TeamComponent },
      { path: 'history', component: HistoryComponent }
    ]
  }
];

Guardias de Ruta

Las guardias de ruta se utilizan para controlar el acceso a ciertas rutas basadas en condiciones.

import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, Router } from '@angular/router';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  constructor(private router: Router) {}

  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): boolean {
    const isAuthenticated = // lógica para verificar si el usuario está autenticado
    if (!isAuthenticated) {
      this.router.navigate(['/login']);
      return false;
    }
    return true;
  }
}

// Aplicando la guardia a una ruta
const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'protected', component: ProtectedComponent, canActivate: [AuthGuard] }
];

Carga Diferida

La carga diferida ayuda a cargar módulos de características solo cuando son necesarios, mejorando el rendimiento de la aplicación.

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'feature', loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) }
];

Resumen

Configurar rutas en Angular implica configurar el RouterModule con definiciones de rutas, usar la directiva <router-outlet> para mostrar componentes enrutados y emplear técnicas avanzadas como rutas anidadas, guardias de ruta y carga diferida para mejorar las capacidades de enrutamiento de la aplicación. Al dominar estos conceptos, puedes crear una navegación robusta y eficiente dentro de tus aplicaciones Angular.
