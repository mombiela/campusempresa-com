Pruebas de Servicios en Angular 2+
Pruebas de Extremo a Extremo ►

Las pruebas de servicios son un aspecto crucial del desarrollo en Angular. Aseguran que los servicios, que son responsables de manejar la lógica de negocio y la recuperación de datos, funcionen correctamente. Este tema te guiará desde los conceptos básicos hasta los avanzados de las pruebas de servicios en Angular 2+.

Introducción a las Pruebas de Servicios

  • ¿Qué son las Pruebas de Servicios?

    • Las pruebas de servicios implican probar los servicios en tu aplicación Angular para asegurar que funcionen como se espera.
    • Se enfocan en la lógica de negocio y los aspectos de manejo de datos de tu aplicación.

  • ¿Por qué son Importantes las Pruebas de Servicios?

    • Aseguran la fiabilidad y corrección de la lógica de negocio.
    • Ayudan a identificar errores temprano en el ciclo de desarrollo.
    • Facilitan un mantenimiento y refactorización más sencillos.

Configuración del Entorno de Pruebas

Antes de sumergirnos en la escritura de pruebas, configuremos el entorno de pruebas.

  • Instalación de Paquetes Necesarios
    • Angular CLI viene con Jasmine y Karma preconfigurados para pruebas.
    • Asegúrate de tener instalado Angular CLI: 
      npm install -g @angular/cli
    • Crea un nuevo proyecto Angular si no tienes uno: 
      ng new my-angular-app
cd my-angular-app

Escribiendo Pruebas Básicas de Servicios

  • Creando un Servicio

    • Genera un nuevo servicio usando Angular CLI: 
      ng generate service my-service
    • Ejemplo de servicio (my-service.service.ts): 
      import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MyService {
  constructor() { }

  getValue(): string {
    return '¡Hola, Mundo!';
  }
}

  • Escribiendo una Prueba para el Servicio

    • Ejemplo de prueba (my-service.service.spec.ts): 
      import { TestBed } from '@angular/core/testing';
import { MyService } from './my-service.service';

describe('MyService', () => {
  let service: MyService;

  beforeEach(() => {
    TestBed.configureTestingModule({});
    service = TestBed.inject(MyService);
  });

  it('should be created', () => {
    expect(service).toBeTruthy();
  });

  it('should return "¡Hola, Mundo!"', () => {
    expect(service.getValue()).toBe('¡Hola, Mundo!');
  });
});

Probando Servicios con Dependencias

  • Servicio con Dependencia

    • Ejemplo de servicio con dependencia (data.service.ts): 
      import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  constructor(private http: HttpClient) { }

  fetchData(): Observable<any> {
    return this.http.get('https://api.example.com/data');
  }
}

  • Simulando Dependencias

    • Usa HttpClientTestingModule para simular solicitudes HTTP: 
      import { TestBed } from '@angular/core/testing';
import { HttpClientTestingModule, HttpTestingController } from '@angular/common/http/testing';
import { DataService } from './data.service';

describe('DataService', () => {
  let service: DataService;
  let httpMock: HttpTestingController;

  beforeEach(() => {
    TestBed.configureTestingModule({
      imports: [HttpClientTestingModule],
      providers: [DataService]
    });
    service = TestBed.inject(DataService);
    httpMock = TestBed.inject(HttpTestingController);
  });

  it('should fetch data', () => {
    const dummyData = [{ id: 1, name: 'John' }];

    service.fetchData().subscribe(data => {
      expect(data.length).toBe(1);
      expect(data).toEqual(dummyData);
    });

    const req = httpMock.expectOne('https://api.example.com/data');
    expect(req.request.method).toBe('GET');
    req.flush(dummyData);
  });

  afterEach(() => {
    httpMock.verify();
  });
});

Técnicas Avanzadas de Pruebas de Servicios

  • Probando Escenarios de Error

    • Ejemplo de prueba de manejo de errores: 
      it('should handle error', () => {
  service.fetchData().subscribe(
    () => fail('should have failed with the 500 error'),
    (error: any) => {
      expect(error.status).toBe(500);
    }
  );

  const req = httpMock.expectOne('https://api.example.com/data');
  req.flush('Something went wrong', { status: 500, statusText: 'Server Error' });
});

  • Usando Espías para Simular Métodos

    • Ejemplo de uso de espías: 
      it('should call getValue method', () => {
  const spy = spyOn(service, 'getValue').and.returnValue('Mock Value');
  expect(service.getValue()).toBe('Mock Value');
  expect(spy).toHaveBeenCalled();
});

Conclusión

Las pruebas de servicios en Angular 2+ son esenciales para asegurar la fiabilidad y corrección de la lógica de negocio de tu aplicación. Al configurar un entorno de pruebas adecuado, escribir pruebas básicas y avanzadas, y usar técnicas como la simulación de dependencias y el manejo de errores, puedes crear aplicaciones Angular robustas y mantenibles. Recuerda probar a fondo y regularmente para detectar problemas temprano y mantener una alta calidad de código.
