Pruebas y Despliegue

Introducción

En esta sección, cubriremos los aspectos esenciales de las pruebas y el despliegue de aplicaciones Angular. Las pruebas aseguran que tu aplicación funcione como se espera, mientras que el despliegue es el proceso de hacer que tu aplicación esté disponible para los usuarios. Esta sección se divide en dos partes principales: Pruebas y Despliegue.

Pruebas

Por qué las Pruebas son Importantes

  • Asegura la Calidad del Código: Ayuda a detectar errores y fallos temprano.
  • Facilita la Refactorización: Hace más fácil cambiar el código sin romper la funcionalidad.
  • Mejora la Mantenibilidad: El código bien probado es más fácil de mantener y extender.

Tipos de Pruebas

  • Pruebas Unitarias: Prueban componentes o servicios individuales.
  • Pruebas de Integración: Prueban la interacción entre diferentes partes de la aplicación.
  • Pruebas de Extremo a Extremo (E2E): Prueban el flujo completo de la aplicación de principio a fin.

Configuración del Entorno de Pruebas

Angular viene con soporte integrado para pruebas usando Jasmine y Karma.

Instalación de Paquetes Necesarios

npm install --save-dev jasmine-core karma karma-chrome-launcher karma-jasmine karma-jasmine-html-reporter

Escribiendo Pruebas Unitarias

Las pruebas unitarias se enfocan en probar componentes o servicios individuales.

Ejemplo: Probando un Componente

import { ComponentFixture, TestBed } from '@angular/core/testing';
import { MyComponent } from './my-component.component';

describe('MyComponent', () => {
  let component: MyComponent;
  let fixture: ComponentFixture<MyComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ MyComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(MyComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should have a default title', () => {
    expect(component.title).toBe('Default Title');
  });
});

Ejecutando Pruebas

Para ejecutar tus pruebas, usa el siguiente comando:

ng test

Escribiendo Pruebas E2E

Las pruebas E2E se escriben usando Protractor.

Ejemplo: Prueba E2E

import { browser, by, element } from 'protractor';

describe('My App', () => {
  it('should display the welcome message', () => {
    browser.get('/');
    expect(element(by.css('h1')).getText()).toEqual('Welcome to My App!');
  });
});

Ejecutando Pruebas E2E

Para ejecutar tus pruebas E2E, usa el siguiente comando:

ng e2e

Despliegue

Preparación para el Despliegue

  • Optimización de la Construcción: Usa Angular CLI para construir una versión optimizada de tu aplicación.
  • Configuración del Entorno: Configura diferentes entornos (desarrollo, producción).

Construyendo la Aplicación

Para construir tu aplicación para producción, usa el siguiente comando:

ng build --prod

Opciones de Despliegue

  • Alojamiento Estático: Despliega tu aplicación Angular en servicios de alojamiento estático como GitHub Pages, Firebase Hosting o Netlify.
  • Alojamiento en Servidor: Despliega tu aplicación en un servidor usando servicios como AWS, Azure o Heroku.

Desplegando en GitHub Pages

  1. Instalar Angular CLI GitHub Pages Deploy: 
    npm install -g angular-cli-ghpages
  2. Construir el Proyecto: 
    ng build --prod --base-href "https://<username>.github.io/<repository-name>/"
  3. Desplegar: 
    ngh --dir=dist/<project-name>

Desplegando en Firebase Hosting

  1. Instalar Firebase Tools: 
    npm install -g firebase-tools
  2. Iniciar Sesión en Firebase: 
    firebase login
  3. Inicializar Firebase en tu Proyecto: 
    firebase init
  4. Construir el Proyecto: 
    ng build --prod
  5. Desplegar: 
    firebase deploy

Conclusión

Las pruebas y el despliegue son aspectos críticos para desarrollar aplicaciones Angular robustas y confiables. Al escribir pruebas exhaustivas, puedes asegurar que tu aplicación se comporte como se espera y sea mantenible. Desplegar tu aplicación correctamente asegura que sea accesible para tus usuarios. Con las herramientas y técnicas cubiertas en esta sección, estás bien equipado para probar y desplegar tus aplicaciones Angular de manera efectiva.
