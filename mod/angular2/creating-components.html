Introducción a los Componentes

Los componentes son los bloques de construcción de cualquier aplicación Angular. Encapsulan el HTML, CSS y la lógica para una parte de la interfaz de usuario. Entender cómo crear y usar componentes es fundamental para dominar Angular.

Conceptos Clave

  • Clase del Componente: Define la lógica y los datos para el componente.
  • Plantilla del Componente: Define la vista HTML para el componente.
  • Estilos del Componente: Define los estilos CSS para el componente.
  • Metadatos: Decora la clase para definir cómo debe ser procesada, instanciada y utilizada en tiempo de ejecución.

Creación de un Componente Básico

Guía Paso a Paso

  1. Generar un Componente: Usa Angular CLI para generar un nuevo componente.

    ng generate component my-component

    Este comando crea una nueva carpeta con cuatro archivos: my-component.component.ts, my-component.component.html, my-component.component.css, y my-component.component.spec.ts.

  2. Clase del Componente: Ubicada en my-component.component.ts.

    import { Component } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  title = 'Hello World';
}
    • @Component Decorador: Define los metadatos para el componente.
    • selector: La etiqueta HTML utilizada para incluir este componente en otras plantillas.
    • templateUrl: Ruta al archivo de plantilla HTML.
    • styleUrls: Ruta al(los) archivo(s) CSS.

  3. Plantilla del Componente: Ubicada en my-component.component.html.

    <h1>{{ title }}</h1>
    • Interpolación: {{ title }} enlaza la propiedad title de la clase del componente al HTML.

  4. Estilos del Componente: Ubicada en my-component.component.css.

    h1 {
  color: blue;
}

  5. Usando el Componente: Agrega el selector del componente a la plantilla de un componente padre.

    <app-my-component></app-my-component>

Características Avanzadas del Componente

Propiedades de Entrada y Salida

Los componentes pueden comunicarse entre sí usando los decoradores @Input y @Output.

Propiedad de Entrada

Permite que un componente padre pase datos a un componente hijo.

// child.component.ts
import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `<p>{{ childMessage }}</p>`
})
export class ChildComponent {
  @Input() childMessage: string;
}
<!-- parent.component.html -->
<app-child [childMessage]="'Hello from Parent'"></app-child>

Propiedad de Salida

Permite que un componente hijo envíe datos a un componente padre.

// child.component.ts
import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `<button (click)="sendMessage()">Send Message</button>`
})
export class ChildComponent {
  @Output() messageEvent = new EventEmitter<string>();

  sendMessage() {
    this.messageEvent.emit('Hello Parent');
  }
}
<!-- parent.component.html -->
<app-child (messageEvent)="receiveMessage($event)"></app-child>
<p>{{ message }}</p>
// parent.component.ts
export class ParentComponent {
  message: string;

  receiveMessage($event: string) {
    this.message = $event;
  }
}

Ganchos del Ciclo de Vida

Angular proporciona ganchos del ciclo de vida que te permiten aprovechar eventos clave en el ciclo de vida de un componente.

Ganchos Comunes del Ciclo de Vida

  • ngOnInit: Llamado una vez que el componente está inicializado.
  • ngOnChanges: Llamado cuando una propiedad de entrada cambia.
  • ngOnDestroy: Llamado justo antes de que el componente sea destruido.
import { Component, OnInit, OnChanges, OnDestroy, Input } from '@angular/core';

@Component({
  selector: 'app-lifecycle',
  template: `<p>Lifecycle Component</p>`
})
export class LifecycleComponent implements OnInit, OnChanges, OnDestroy {
  @Input() data: string;

  ngOnInit() {
    console.log('Component Initialized');
  }

  ngOnChanges() {
    console.log('Input Property Changed');
  }

  ngOnDestroy() {
    console.log('Component Destroyed');
  }
}

Resumen

Crear componentes en Angular implica definir una clase de componente, plantilla y estilos. Los componentes pueden comunicarse entre sí usando propiedades de entrada y salida. Entender los ganchos del ciclo de vida te permite gestionar el ciclo de vida del componente de manera efectiva. Dominar estos conceptos es esencial para construir aplicaciones Angular robustas y mantenibles.
