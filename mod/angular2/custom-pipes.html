Introducción a los Pipes

Los pipes en Angular son una forma de transformar datos en plantillas. Se utilizan para formatear datos mostrados al usuario. Angular proporciona varios pipes integrados, como DatePipe, UpperCasePipe, LowerCasePipe, CurrencyPipe y PercentPipe.

¿Por qué Pipes personalizados?

Aunque los pipes integrados cubren muchos escenarios comunes, hay ocasiones en las que necesitas crear pipes personalizados para manejar transformaciones de datos específicas que son únicas para tu aplicación.

Creación de un Pipe personalizado

Crear un pipe personalizado implica los siguientes pasos:

  1. Generar el Pipe: Utiliza Angular CLI para generar un nuevo pipe.
  2. Implementar la lógica del Pipe: Define la lógica de transformación en la clase del pipe.
  3. Registrar el Pipe: Asegúrate de que el pipe esté declarado en un módulo de Angular.

Paso 1: Generar el Pipe

Utiliza Angular CLI para generar un nuevo pipe:

ng generate pipe custom

Este comando crea un nuevo archivo custom.pipe.ts con una estructura básica.

Paso 2: Implementar la lógica del Pipe

Edita el archivo del pipe generado para implementar la lógica de transformación. Por ejemplo, vamos a crear un pipe que convierte una cadena a mayúsculas iniciales (la primera letra de cada palabra está en mayúscula).

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'titleCase'
})
export class TitleCasePipe implements PipeTransform {
  transform(value: string): string {
    if (!value) return value;
    return value.replace(/\w\S*/g, (txt) => {
      return txt.charAt(0).toUpperCase() + txt.substr(1).toLowerCase();
    });
  }
}

Paso 3: Registrar el Pipe

Asegúrate de que el pipe esté declarado en un módulo de Angular, típicamente en app.module.ts.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { TitleCasePipe } from './title-case.pipe';

@NgModule({
  declarations: [
    AppComponent,
    TitleCasePipe
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Usar el Pipe personalizado

Una vez que el pipe personalizado está creado y registrado, puedes usarlo en tus plantillas como cualquier otro pipe.

<p>{{ 'hello world' | titleCase }}</p>

Esto mostrará:

Hello World

Pipes parametrizados

Los pipes personalizados también pueden aceptar parámetros. Por ejemplo, vamos a crear un pipe que formatea un número con un número especificado de decimales.

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'decimalFormat'
})
export class DecimalFormatPipe implements PipeTransform {
  transform(value: number, decimalPlaces: number): string {
    return value.toFixed(decimalPlaces);
  }
}

Uso en una plantilla:

<p>{{ 1234.5678 | decimalFormat:2 }}</p>

Esto mostrará:

1234.57

Pipes puros vs impuros

  • Pipes puros: Son sin estado y solo dependen de sus argumentos de entrada. Solo se llaman cuando Angular detecta un cambio en el valor de entrada.
  • Pipes impuros: Pueden tener efectos secundarios y se llaman en cada ciclo de detección de cambios.

Para crear un pipe impuro, establece la propiedad pure a false en el decorador @Pipe.

@Pipe({
  name: 'impurePipe',
  pure: false
})
export class ImpurePipe implements PipeTransform {
  // Implementación
}

Resumen

  • Pipes personalizados: Te permiten crear lógica de transformación de datos reutilizable.
  • Pasos para crear: Generar, implementar y registrar el pipe.
  • Uso: Usa pipes personalizados en plantillas como pipes integrados.
  • Pipes parametrizados: Aceptan argumentos adicionales para transformaciones más flexibles.
  • Pipes puros vs impuros: Los pipes puros son sin estado, mientras que los pipes impuros pueden tener efectos secundarios y se llaman con más frecuencia.

Al dominar los pipes personalizados, puedes mejorar significativamente las capacidades de presentación de datos de tus aplicaciones Angular.
