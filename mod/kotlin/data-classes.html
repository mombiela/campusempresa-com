Introducción a las Clases de Datos

Las clases de datos en Kotlin son una forma concisa de crear clases que se utilizan principalmente para contener datos. Proporcionan automáticamente varios métodos útiles, como equals(), hashCode(), toString(), y copy(), lo que las hace muy convenientes para manejar datos.

Conceptos Clave

  • Constructor Primario: Las clases de datos deben tener al menos un parámetro de constructor primario.
  • Propiedades: Todos los parámetros del constructor primario deben estar marcados como val o var.
  • Métodos Generados: Kotlin genera automáticamente los métodos equals(), hashCode(), toString(), copy(), y componentN() para las clases de datos.
  • Inmutabilidad: Por defecto, las clases de datos son inmutables si usas val para las propiedades.

Creando una Clase de Datos

Para definir una clase de datos, usa la palabra clave data antes de la declaración de la clase.

data class User(val name: String, val age: Int)

Ejemplo Explicado

  • data: Palabra clave que indica que esta es una clase de datos.
  • User: Nombre de la clase de datos.
  • val name: String: Propiedad inmutable name de tipo String.
  • val age: Int: Propiedad inmutable age de tipo Int.

Métodos Generados

`toString()`

Proporciona una representación en cadena del objeto.

val user = User("Alice", 30)
println(user) // Salida: User(name=Alice, age=30)

`equals()` y `hashCode()`

Se utilizan para comparar objetos y para hashing.

val user1 = User("Alice", 30)
val user2 = User("Alice", 30)
println(user1 == user2) // Salida: true
println(user1.hashCode() == user2.hashCode()) // Salida: true

`copy()`

Crea una copia del objeto, opcionalmente modificando algunas propiedades.

val user1 = User("Alice", 30)
val user2 = user1.copy(age = 31)
println(user2) // Salida: User(name=Alice, age=31)

`componentN()`

Permite declaraciones de desestructuración.

val user = User("Alice", 30)
val (name, age) = user
println("Name: $name, Age: $age") // Salida: Name: Alice, Age: 30

Características Avanzadas

Valores Predeterminados

Puedes proporcionar valores predeterminados para las propiedades en el constructor primario.

data class User(val name: String = "Unknown", val age: Int = 0)
val user = User()
println(user) // Salida: User(name=Unknown, age=0)

Herencia de Clases de Datos

Las clases de datos no pueden ser abstractas, abiertas, selladas o internas. Sin embargo, pueden implementar interfaces.

interface Printable {
    fun print()
}

data class User(val name: String, val age: Int) : Printable {
    override fun print() {
        println("User(name=$name, age=$age)")
    }
}

Copiando con Clases de Datos Anidadas

Al copiar clases de datos anidadas, solo se copia el objeto de nivel superior. Los objetos anidados no se copian profundamente.

data class Address(val city: String, val country: String)
data class User(val name: String, val age: Int, val address: Address)

val address = Address("New York", "USA")
val user1 = User("Alice", 30, address)
val user2 = user1.copy()

println(user1.address === user2.address) // Salida: true (misma referencia)

Conclusión

Las clases de datos en Kotlin proporcionan una forma poderosa y concisa de manejar clases centradas en datos. Generan automáticamente métodos útiles, reduciendo el código repetitivo y mejorando la legibilidad. Comprender y utilizar eficazmente las clases de datos puede mejorar significativamente tus habilidades de programación en Kotlin, especialmente cuando se trata de aplicaciones con muchos datos.
