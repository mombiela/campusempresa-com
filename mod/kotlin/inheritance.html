Herencia en Kotlin
Interfaces ►

La herencia es un concepto fundamental en la programación orientada a objetos que permite a una clase heredar propiedades y funciones de otra clase. En Kotlin, la herencia es sencilla y proporciona una forma poderosa de reutilizar código y crear una estructura jerárquica de clases.

Conceptos Clave

  • Clase Superclase (Clase Padre): La clase cuyas propiedades y funciones son heredadas.
  • Clase Subclase (Clase Hija): La clase que hereda propiedades y funciones de la superclase.
  • Palabra clave open: Por defecto, las clases en Kotlin son finales y no pueden ser heredadas. La palabra clave open se usa para hacer una clase heredable.
  • Palabra clave override: Se usa para sobrescribir una función o propiedad en la subclase.

Creando una Superclase

En Kotlin, necesitas marcar una clase como open para permitir que otras clases hereden de ella.

open class Animal {
    open fun sound() {
        println("Animal makes a sound")
    }
}

Creando una Subclase

Una subclase hereda de una superclase usando el símbolo :.

class Dog : Animal() {
    override fun sound() {
        println("Dog barks")
    }
}

Ejemplo: Herencia Básica

Veamos un ejemplo completo de herencia en Kotlin.

// Superclase
open class Animal {
    open fun sound() {
        println("Animal makes a sound")
    }
}

// Subclase
class Dog : Animal() {
    override fun sound() {
        println("Dog barks")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()  // Output: Dog barks
}

En este ejemplo:

  • Animal es la superclase con una función abierta sound.
  • Dog es la subclase que sobrescribe la función sound.

Propiedades en la Herencia

También puedes heredar propiedades de la superclase.

open class Animal {
    open val type: String = "Unknown"
}

class Dog : Animal() {
    override val type: String = "Dog"
}

fun main() {
    val myDog = Dog()
    println(myDog.type)  // Output: Dog
}

Llamando a Funciones de la Superclase

Puedes llamar a la función de la superclase usando la palabra clave super.

open class Animal {
    open fun sound() {
        println("Animal makes a sound")
    }
}

class Dog : Animal() {
    override fun sound() {
        super.sound()  // Llama a la función de la superclase
        println("Dog barks")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()
    // Output:
    // Animal makes a sound
    // Dog barks
}

Clases Abstractas

A veces, quieres crear una superclase que no debería ser instanciada directamente. Para esto, puedes usar clases abstractas.

abstract class Animal {
    abstract fun sound()
}

class Dog : Animal() {
    override fun sound() {
        println("Dog barks")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()  // Output: Dog barks
}

En este ejemplo:

  • Animal es una clase abstracta con una función abstracta sound.
  • Dog es una subclase que proporciona una implementación para la función sound.

Resumen

  • La herencia permite a una clase heredar propiedades y funciones de otra clase.
  • Usa la palabra clave open para hacer una clase heredable.
  • Usa la palabra clave override para sobrescribir funciones y propiedades en la subclase.
  • Las clases abstractas pueden ser usadas para definir clases que no deberían ser instanciadas directamente pero pueden ser heredadas.

Al entender y utilizar la herencia, puedes crear código más modular, reutilizable y mantenible en Kotlin.
