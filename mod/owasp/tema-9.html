Introducción

El control de acceso es un mecanismo fundamental en la seguridad de aplicaciones web. Se refiere a las políticas y procedimientos que determinan quién puede acceder a qué recursos en una aplicación. El problema de "Broken Access Control" ocurre cuando estas políticas no se implementan correctamente, permitiendo a usuarios no autorizados acceder a recursos restringidos.

Conceptos Clave

  • Control de Acceso: Conjunto de mecanismos que restringen el acceso a recursos a usuarios autorizados.
  • Autenticación vs. Autorización:
    • Autenticación: Verificación de la identidad del usuario.
    • Autorización: Determinación de los permisos del usuario autenticado.
  • Principio de Menor Privilegio: Los usuarios deben tener solo los permisos necesarios para realizar sus tareas.

Tipos Comunes de Fallos en el Control de Acceso

  • Bypass de Autorización: Acceso a recursos sin pasar por los controles de autorización.
  • Escalada de Privilegios: Un usuario obtiene permisos más altos de los que debería tener.
  • Acceso Directo a Objetos: Acceso a recursos mediante la manipulación de URLs, parámetros o formularios.

Ejemplos de Broken Access Control

Ejemplo 1: Acceso Directo a Objetos

Un usuario puede acceder a los datos de otro usuario manipulando la URL.

# URL original para acceder a los datos del usuario
http://example.com/user/123

# Manipulación de la URL para acceder a los datos de otro usuario
http://example.com/user/124

Ejemplo 2: Escalada de Privilegios

Un usuario regular intenta acceder a una página de administración.

# URL para usuarios regulares
http://example.com/user/dashboard

# URL para administradores
http://example.com/admin/dashboard

Si el control de acceso no está bien implementado, un usuario regular podría acceder a la página de administración.

Mejores Prácticas para Prevenir Broken Access Control

  • Implementar controles de acceso en el servidor: No confiar en controles de acceso implementados solo en el cliente.
  • Utilizar listas de control de acceso (ACL): Definir claramente qué usuarios tienen acceso a qué recursos.
  • Revisar y auditar permisos regularmente: Asegurarse de que los permisos asignados sean los correctos.
  • Principio de Menor Privilegio: Asignar solo los permisos necesarios a cada usuario.
  • Validar el acceso en cada solicitud: No asumir que un usuario tiene acceso a un recurso basado en una solicitud anterior.

Ejercicio Práctico

Ejercicio 1: Identificación de Fallos en el Control de Acceso

Dado el siguiente código, identifique y corrija los problemas de control de acceso.

# Código original
def get_user_data(user_id):
    user = db.get_user(user_id)
    return user.data

# Código corregido
def get_user_data(current_user, user_id):
    if current_user.id != user_id and not current_user.is_admin:
        raise PermissionError("Access Denied")
    user = db.get_user(user_id)
    return user.data

Ejercicio 2: Implementación de Control de Acceso

Implemente un control de acceso básico para una aplicación web en Python usando Flask.

from flask import Flask, request, jsonify

app = Flask(__name__)

# Simulación de base de datos de usuarios
users_db = {
    1: {"id": 1, "name": "Alice", "role": "user"},
    2: {"id": 2, "name": "Bob", "role": "admin"}
}

def get_current_user():
    # Simulación de autenticación
    user_id = int(request.headers.get('X-User-ID'))
    return users_db.get(user_id)

@app.route('/admin', methods=['GET'])
def admin_dashboard():
    current_user = get_current_user()
    if current_user['role'] != 'admin':
        return jsonify({"error": "Access Denied"}), 403
    return jsonify({"message": "Welcome to the admin dashboard"})

if __name__ == '__main__':
    app.run(debug=True)

Conclusión

El control de acceso es una parte crítica de la seguridad de las aplicaciones web. Los fallos en el control de acceso pueden permitir a usuarios no autorizados acceder a recursos restringidos, lo que puede tener consecuencias graves. Implementar controles de acceso robustos y seguir las mejores prácticas puede ayudar a prevenir estos problemas y proteger la integridad y confidencialidad de los datos.
