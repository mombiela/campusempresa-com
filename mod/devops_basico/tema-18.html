Introducción a los contenedores

Los contenedores son una tecnología que permite empaquetar aplicaciones y sus dependencias en un entorno aislado y reproducible. Esto facilita la portabilidad y la consistencia entre diferentes entornos de desarrollo, prueba y producción.

  • Aislamiento: Cada contenedor opera de manera independiente, lo que reduce los conflictos entre aplicaciones.
  • Portabilidad: Los contenedores pueden ejecutarse en cualquier sistema que soporte la tecnología de contenedores, como Docker.
  • Eficiencia: Los contenedores comparten el mismo sistema operativo, lo que los hace más ligeros que las máquinas virtuales.

Docker: La plataforma de contenedores más popular

Docker es una plataforma que permite a los desarrolladores crear, desplegar y ejecutar aplicaciones en contenedores.

  • Docker Engine: El motor que permite la creación y ejecución de contenedores.
  • Docker Hub: Un repositorio en línea para compartir y almacenar imágenes de contenedores.

Instalación de Docker

Para comenzar a trabajar con Docker, primero necesitas instalarlo en tu sistema.

Instalación en Linux

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io

Instalación en Windows y macOS

Visita Docker Desktop y sigue las instrucciones para tu sistema operativo.

Creación de contenedores

Para crear un contenedor, primero necesitas una imagen de Docker. Las imágenes son plantillas de solo lectura que contienen el sistema operativo y las aplicaciones necesarias.

Ejemplo: Crear un contenedor de Nginx

  1. Descargar la imagen de Nginx: 
    docker pull nginx
  2. Ejecutar un contenedor de Nginx: 
    docker run --name mynginx -d -p 8080:80 nginx
    • --name mynginx: Asigna un nombre al contenedor.
    • -d: Ejecuta el contenedor en segundo plano.
    • -p 8080:80: Mapea el puerto 80 del contenedor al puerto 8080 del host.

Verificar el contenedor en ejecución

docker ps

Gestión de contenedores

Una vez que tienes contenedores en ejecución, necesitas saber cómo gestionarlos.

Listar contenedores

  • Contenedores en ejecución: 
    docker ps
  • Todos los contenedores (incluidos los detenidos): 
    docker ps -a

Detener y eliminar contenedores

  • Detener un contenedor: 
    docker stop mynginx
  • Eliminar un contenedor: 
    docker rm mynginx

Inspeccionar contenedores

Para obtener detalles sobre un contenedor específico:

docker inspect mynginx

Volúmenes y redes en Docker

Los volúmenes y las redes son componentes esenciales para la gestión avanzada de contenedores.

Volúmenes

Los volúmenes permiten persistir datos más allá del ciclo de vida de un contenedor.

  • Crear un volumen: 
    docker volume create myvolume
  • Montar un volumen en un contenedor: 
    docker run -d -v myvolume:/data --name mynginx nginx

Redes

Las redes permiten la comunicación entre contenedores.

  • Crear una red: 
    docker network create mynetwork
  • Conectar un contenedor a una red: 
    docker run -d --network mynetwork --name mynginx nginx

Conclusión

La creación y gestión de contenedores es una habilidad fundamental en DevOps. Docker facilita este proceso al proporcionar herramientas robustas para el desarrollo, despliegue y gestión de aplicaciones en contenedores. Con el conocimiento básico de Docker, puedes empezar a construir entornos de desarrollo más eficientes y portables.

Recuerda practicar estos comandos y conceptos para familiarizarte con el flujo de trabajo de Docker. ¡La práctica constante te ayudará a dominar la gestión de contenedores!
