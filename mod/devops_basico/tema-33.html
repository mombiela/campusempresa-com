Introducción

El proyecto final del curso de DevOps Básico tiene como objetivo integrar y aplicar los conocimientos adquiridos a lo largo del curso. Los estudiantes trabajarán en un proyecto práctico que simula un entorno real de DevOps, permitiéndoles experimentar con las herramientas y prácticas esenciales de DevOps.

Objetivos del Proyecto

  • Implementar un pipeline de CI/CD: Configurar un pipeline de Integración Continua y Despliegue Continuo utilizando herramientas como Jenkins, GitLab CI, o GitHub Actions.
  • Automatización de Infraestructura: Utilizar herramientas de infraestructura como código (IaC) como Terraform o Ansible para automatizar la provisión de recursos.
  • Monitoreo y Logging: Implementar soluciones de monitoreo y logging para asegurar la observabilidad del sistema.
  • Gestión de Contenedores: Utilizar Docker para la creación y gestión de contenedores, y Kubernetes para la orquestación de contenedores.

Requisitos del Proyecto

Pipeline de CI/CD

  • Repositorio de Código: Configurar un repositorio en GitHub o GitLab.
  • Integración Continua: Configurar un pipeline que ejecute pruebas automáticas cada vez que se realice un commit.
  • Despliegue Continuo: Configurar el pipeline para desplegar automáticamente la aplicación en un entorno de staging y, posteriormente, en producción.
# Ejemplo de pipeline en GitLab CI
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - echo "Building the application..."
    - ./build.sh

test:
  stage: test
  script:
    - echo "Running tests..."
    - ./run_tests.sh

deploy:
  stage: deploy
  script:
    - echo "Deploying to production..."
    - ./deploy.sh
  only:
    - master

Automatización de Infraestructura

  • Provisión de Recursos: Utilizar Terraform para crear y gestionar recursos en la nube (AWS, Azure, GCP).
  • Configuración de Servidores: Utilizar Ansible para configurar servidores y desplegar aplicaciones.
# Ejemplo de configuración de Terraform
provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "WebServer"
  }
}

Monitoreo y Logging

  • Monitoreo: Implementar Prometheus para la recolección de métricas y Grafana para la visualización.
  • Logging: Configurar ELK Stack (Elasticsearch, Logstash, Kibana) para la gestión de logs.
# Ejemplo de configuración de Prometheus
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

Gestión de Contenedores

  • Docker: Crear Dockerfiles para la aplicación y sus dependencias.
  • Kubernetes: Desplegar la aplicación en un clúster de Kubernetes.
# Ejemplo de Dockerfile
FROM node:14
WORKDIR /app
COPY package.json .
RUN npm install
COPY . .
CMD ["npm", "start"]
# Ejemplo de configuración de Kubernetes
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: web
        image: myapp:latest
        ports:
        - containerPort: 80

Conclusión

El proyecto final es una oportunidad para aplicar de manera práctica los conceptos y herramientas aprendidos durante el curso de DevOps Básico. Al completar este proyecto, los estudiantes habrán adquirido experiencia en la configuración de pipelines de CI/CD, la automatización de infraestructura, el monitoreo y logging, y la gestión de contenedores. Este proyecto no solo consolidará sus conocimientos, sino que también les proporcionará una base sólida para enfrentar desafíos reales en el ámbito de DevOps.
