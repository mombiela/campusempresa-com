Introducción a Kubernetes

Kubernetes es una plataforma de orquestación de contenedores de código abierto que automatiza el despliegue, la escala y las operaciones de aplicaciones en contenedores. Es una herramienta esencial en el ecosistema DevOps para gestionar aplicaciones en contenedores de manera eficiente.

Conceptos Clave

  • Cluster: Un conjunto de nodos (máquinas) que ejecutan aplicaciones en contenedores gestionadas por Kubernetes.
  • Nodo: Una máquina individual dentro del cluster, puede ser física o virtual.
  • Pod: La unidad más pequeña y básica de despliegue en Kubernetes, que puede contener uno o más contenedores.
  • Service: Un recurso que define una política de acceso a los pods, permitiendo la comunicación entre ellos y con el exterior.
  • Deployment: Un recurso que gestiona la creación y escalado de pods, asegurando que el estado deseado de la aplicación se mantenga.

Despliegue Básico de una Aplicación

Para desplegar una aplicación en Kubernetes, se deben seguir varios pasos clave:

1. Crear un archivo de configuración YAML

El archivo YAML define los recursos que Kubernetes debe crear y gestionar. A continuación, se muestra un ejemplo básico de un archivo de configuración para desplegar una aplicación Nginx.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

2. Aplicar el archivo de configuración

Utiliza el comando kubectl apply para aplicar el archivo de configuración y crear los recursos en el cluster.

kubectl apply -f nginx-deployment.yaml

3. Verificar el despliegue

Comprueba que los pods se han creado y están en ejecución.

kubectl get pods

4. Exponer la aplicación

Para que la aplicación sea accesible desde fuera del cluster, se debe crear un Service. Aquí hay un ejemplo de un archivo YAML para un Service tipo LoadBalancer.

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer

Aplica el archivo de configuración del Service:

kubectl apply -f nginx-service.yaml

5. Verificar el Service

Comprueba que el Service se ha creado y está asignando una IP externa.

kubectl get services

Ejercicio Práctico

  1. Crear un archivo de configuración YAML para una aplicación simple de tu elección.
  2. Desplegar la aplicación en un cluster de Kubernetes utilizando kubectl apply.
  3. Crear y aplicar un Service para exponer la aplicación.
  4. Verificar que la aplicación está en ejecución y accesible desde fuera del cluster.

Conclusión

El despliegue de aplicaciones en Kubernetes implica la creación y gestión de varios recursos como Pods, Deployments y Services. A través de archivos de configuración YAML y comandos kubectl, los profesionales de TI pueden automatizar y gestionar eficientemente sus aplicaciones en contenedores. Practicar estos pasos básicos es fundamental para dominar el despliegue en Kubernetes y avanzar en el camino de DevOps.
