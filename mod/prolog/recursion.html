Introducción a la Recursión

La recursión es un concepto fundamental en Prolog y muchos otros lenguajes de programación. Involucra una función que se llama a sí misma para resolver instancias más pequeñas del mismo problema. En Prolog, la recursión se usa a menudo para recorrer listas, árboles y otras estructuras de datos.

Conceptos Clave

  • Caso Base: La condición bajo la cual las llamadas recursivas se detienen.
  • Caso Recursivo: La parte de la función que incluye la llamada recursiva.

Recursión Básica

Comencemos con un ejemplo simple: calcular el factorial de un número.

Ejemplo de Factorial

El factorial de un número \( n \) se define como:

  • \( 0! = 1 \) (Caso Base)
  • \( n! = n \times (n-1)! \) (Caso Recursivo)

Aquí tienes cómo implementarlo en Prolog:

% Caso Base
factorial(0, 1).

% Caso Recursivo
factorial(N, Result) :-
    N > 0,
    N1 is N - 1,
    factorial(N1, Result1),
    Result is N * Result1.

Explicación

  • El caso base factorial(0, 1) establece que el factorial de 0 es 1.
  • El caso recursivo factorial(N, Result) verifica si \( N \) es mayor que 0, luego calcula \( N-1 \) y llama recursivamente a factorial con \( N-1 \). Finalmente, multiplica \( N \) por el resultado de la llamada recursiva.

Recursión con Listas

Prolog sobresale en el manejo de listas a través de la recursión. Veamos un ejemplo de sumar todos los elementos en una lista.

Suma de Elementos de una Lista

Para sumar los elementos de una lista, definimos:

  • La suma de una lista vacía es 0 (Caso Base).
  • La suma de una lista es la cabeza de la lista más la suma de la cola (Caso Recursivo).
% Caso Base
sum_list([], 0).

% Caso Recursivo
sum_list([Head|Tail], Sum) :-
    sum_list(Tail, SumTail),
    Sum is Head + SumTail.

Explicación

  • El caso base sum_list([], 0) establece que la suma de una lista vacía es 0.
  • El caso recursivo sum_list([Head|Tail], Sum) divide la lista en su cabeza y cola, suma recursivamente la cola, y añade la cabeza a esta suma.

Recursión Avanzada

Secuencia de Fibonacci

La secuencia de Fibonacci es un ejemplo clásico de recursión, donde cada número es la suma de los dos anteriores.

% Casos Base
fibonacci(0, 0).
fibonacci(1, 1).

% Caso Recursivo
fibonacci(N, Result) :-
    N > 1,
    N1 is N - 1,
    N2 is N - 2,
    fibonacci(N1, Result1),
    fibonacci(N2, Result2),
    Result is Result1 + Result2.

Explicación

  • Los casos base fibonacci(0, 0) y fibonacci(1, 1) definen los dos primeros números de la secuencia.
  • El caso recursivo fibonacci(N, Result) calcula los números Fibonacci \( N-1 \) y \( N-2 \) y los suma.

Optimización de la Recursión con Acumuladores

La recursión puede optimizarse usando acumuladores para evitar cálculos redundantes. Vamos a optimizar el ejemplo de Fibonacci.

% Predicado auxiliar con acumulador
fibonacci_acc(N, Result) :-
    fibonacci_acc(N, 0, 1, Result).

% Caso Base
fibonacci_acc(0, A, _, A).

% Caso Recursivo
fibonacci_acc(N, A, B, Result) :-
    N > 0,
    N1 is N - 1,
    Sum is A + B,
    fibonacci_acc(N1, B, Sum, Result).

Explicación

  • El predicado auxiliar fibonacci_acc/2 inicializa los acumuladores.
  • El caso base fibonacci_acc(0, A, _, A) devuelve el acumulador cuando \( N \) es 0.
  • El caso recursivo fibonacci_acc(N, A, B, Result) actualiza los acumuladores y se llama a sí mismo con \( N-1 \).

Conclusión

La recursión es una herramienta poderosa en Prolog para resolver problemas que involucran estructuras repetitivas como listas y árboles. Al entender los casos base y recursivos, puedes implementar soluciones recursivas eficientes. Optimizar la recursión con técnicas como los acumuladores puede mejorar aún más el rendimiento. Practica estos conceptos con varios problemas para dominar la recursión en Prolog.
