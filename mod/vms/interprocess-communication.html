Comunicación Interprocesos
Operaciones de Entrada/Salida de Archivos ►

La Comunicación Interprocesos (IPC) es un concepto fundamental en los sistemas operativos, que permite a los procesos intercambiar datos y sincronizar sus acciones. En OpenVMS, los mecanismos de IPC son robustos y versátiles, proporcionando varios métodos para que los procesos se comuniquen de manera efectiva. Este tema cubrirá IPC desde niveles principiantes hasta avanzados, asegurando una comprensión integral de los conceptos y las implementaciones prácticas.

Introducción a la Comunicación Interprocesos

  • Definición: IPC es un mecanismo que permite a los procesos comunicarse entre sí y sincronizar sus acciones.
  • Importancia: IPC es crucial para entornos multitarea donde los procesos necesitan compartir datos y recursos de manera eficiente.

Mecanismos de IPC en OpenVMS

OpenVMS proporciona varios mecanismos de IPC, incluyendo:

  1. Buzones
  2. Secciones Globales
  3. Banderas de Evento
  4. Nombres Lógicos
  5. Colas

Buzones

Los buzones son una forma simple y efectiva para que los procesos envíen y reciban mensajes.

  • Concepto: Un buzón es un búfer al que los procesos pueden escribir y leer.
  • Uso: Adecuado para mensajes pequeños y notificaciones.

Ejemplo: Creación y Uso de un Buzón

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    unsigned short channel;
    char msg[] = "¡Hola, OpenVMS!";
    char buffer[20];
    unsigned short iosb[4];

    // Crear un buzón
    int status = sys$crembx(0, &channel, 0, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error al crear el buzón: %d\n", status);
        return 1;
    }

    // Escribir en el buzón
    status = sys$qiow(0, channel, IO$_WRITEVBLK, iosb, 0, 0, msg, sizeof(msg) - 1, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error al escribir en el buzón: %d\n", status);
        return 1;
    }

    // Leer del buzón
    status = sys$qiow(0, channel, IO$_READVBLK, iosb, 0, 0, buffer, sizeof(buffer) - 1, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error al leer del buzón: %d\n", status);
        return 1;
    }

    buffer[iosb[1]] = '\0'; // Terminar la cadena con nulo
    printf("Mensaje recibido: %s\n", buffer);

    // Desasignar el canal
    sys$dassgn(channel);

    return 0;
}

Secciones Globales

Las secciones globales permiten a los procesos compartir memoria.

  • Concepto: Una sección global es un segmento de memoria compartida que múltiples procesos pueden mapear en su espacio de direcciones.
  • Uso: Adecuado para compartir grandes cantidades de datos y estructuras de datos complejas.

Ejemplo: Creación y Uso de una Sección Global

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    struct dsc$descriptor_s name = { 0, DSC$K_DTYPE_T, DSC$K_CLASS_S, 0 };
    unsigned long gsd[2];
    unsigned long *shared_memory;
    unsigned long status;

    // Definir el nombre de la sección global
    name.dsc$w_length = strlen("MI_SECCION_GLOBAL");
    name.dsc$a_pointer = "MI_SECCION_GLOBAL";

    // Crear la sección global
    status = sys$crmpsc(gsd, 0, 0, SEC$M_GBL, 0, 0, &name, 0, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error al crear la sección global: %d\n", status);
        return 1;
    }

    // Mapear la sección global en el espacio de direcciones del proceso
    status = sys$mgblsc(gsd, &shared_memory, 0, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error al mapear la sección global: %d\n", status);
        return 1;
    }

    // Usar la memoria compartida
    shared_memory[0] = 42;
    printf("Valor de la memoria compartida: %lu\n", shared_memory[0]);

    // Desmapear la sección global
    sys$deltva(&shared_memory, 0, 0);

    return 0;
}

Banderas de Evento

Las banderas de evento se utilizan para la sincronización de procesos.

  • Concepto: Una bandera de evento es una bandera binaria que los procesos pueden establecer, limpiar y esperar.
  • Uso: Adecuado para señalización entre procesos.

Ejemplo: Uso de Banderas de Evento

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    unsigned long efn = 0; // Número de bandera de evento
    unsigned long status;

    // Establecer la bandera de evento
    status = sys$setef(efn);
    if (status != SS$_NORMAL) {
        printf("Error al establecer la bandera de evento: %d\n", status);
        return 1;
    }

    // Esperar la bandera de evento
    status = sys$waitfr(efn);
    if (status != SS$_NORMAL) {
        printf("Error al esperar la bandera de evento: %d\n", status);
        return 1;
    }

    printf("Bandera de evento señalada.\n");

    return 0;
}

Nombres Lógicos

Los nombres lógicos se utilizan para definir nombres simbólicos para recursos del sistema.

  • Concepto: Un nombre lógico es una cadena que se asigna a un recurso del sistema, como un archivo o dispositivo.
  • Uso: Adecuado para abstraer nombres de recursos.

Ejemplo: Creación y Uso de Nombres Lógicos

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    struct dsc$descriptor_s logname = { 0, DSC$K_DTYPE_T, DSC$K_CLASS_S, 0 };
    struct dsc$descriptor_s logvalue = { 0, DSC$K_DTYPE_T, DSC$K_CLASS_S, 0 };
    unsigned long status;

    // Definir el nombre lógico y el valor
    logname.dsc$w_length = strlen("MI_NOMBRE_LOGICO");
    logname.dsc$a_pointer = "MI_NOMBRE_LOGICO";
    logvalue.dsc$w_length = strlen("MI_VALOR_LOGICO");
    logvalue.dsc$a_pointer = "MI_VALOR_LOGICO";

    // Crear el nombre lógico
    status = sys$crelnm(0, &logname, &logvalue, 0);
    if (status != SS$_NORMAL) {
        printf("Error al crear el nombre lógico: %d\n", status);
        return 1;
    }

    // Traducir el nombre lógico
    char buffer[256];
    struct dsc$descriptor_s result = { sizeof(buffer) - 1, DSC$K_DTYPE_T, DSC$K_CLASS_S, buffer };
    status = sys$trnlnm(0, &logname, &result, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error al traducir el nombre lógico: %d\n", status);
        return 1;
    }

    buffer[result.dsc$w_length] = '\0'; // Terminar la cadena con nulo
    printf("Valor del nombre lógico: %s\n", buffer);

    return 0;
}

Colas

Las colas se utilizan para el paso de mensajes entre procesos.

  • Concepto: Una cola es una estructura de datos que contiene mensajes para que los procesos los lean y escriban.
  • Uso: Adecuado para el paso de mensajes complejos y control de trabajos.

Ejemplo: Uso de Colas

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    struct dsc$descriptor_s queue_name = { 0, DSC$K_DTYPE_T, DSC$K_CLASS_S, 0 };
    unsigned short iosb[4];
    char msg[] = "¡Hola, Cola!";
    char buffer[20];
    unsigned long status;

    // Definir el nombre de la cola
    queue_name.dsc$w_length = strlen("MI_COLA");
    queue_name.dsc$a_pointer = "MI_COLA";

    // Crear la cola
    status = sys$crembx(0, &queue_name, 0, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error al crear la cola: %d\n", status);
        return 1;
    }

    // Escribir en la cola
    status = sys$qiow(0, queue_name, IO$_WRITEVBLK, iosb, 0, 0, msg, sizeof(msg) - 1, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {

        return 1;
    }

    // Leer de la cola
    status = sys$qiow(0, queue_name, IO$_READVBLK, iosb, 0, 0, buffer, sizeof(buffer) - 1, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error al leer de la cola: %d\n", status);
        return 1;
    }

    buffer[iosb[1]] = '\0'; // Terminar la cadena con nulo
    printf("Mensaje recibido: %s\n", buffer);

    return 0;
}

Conclusión

La Comunicación Interprocesos en OpenVMS es una característica poderosa que permite a los procesos comunicarse y sincronizarse de manera efectiva. Al comprender y utilizar los diversos mecanismos de IPC, como buzones, secciones globales, banderas de evento, nombres lógicos y colas, los programadores pueden desarrollar aplicaciones robustas y eficientes. Cada método tiene sus casos de uso específicos y ventajas, haciendo de OpenVMS una plataforma versátil para la comunicación entre procesos.
