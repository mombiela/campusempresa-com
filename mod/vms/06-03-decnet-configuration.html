Configuración de DECnet
Siguiente ►

Introducción

DECnet es un conjunto de protocolos de red desarrollados por Digital Equipment Corporation (DEC) que permite la comunicación entre sistemas OpenVMS y otros sistemas compatibles. En este módulo, aprenderemos cómo configurar DECnet en un sistema OpenVMS, cubriendo desde la instalación hasta la configuración básica y avanzada.

Objetivos

Al finalizar este tema, deberías ser capaz de:

  • Comprender los conceptos básicos de DECnet.
  • Instalar y configurar DECnet en un sistema OpenVMS.
  • Realizar configuraciones básicas y avanzadas de DECnet.
  • Solucionar problemas comunes relacionados con DECnet.

Conceptos Básicos de DECnet

Antes de entrar en la configuración, es importante entender algunos conceptos clave:

  • Nodos: Un nodo es cualquier dispositivo en la red DECnet, como una computadora o un router.
  • Dirección de Nodo: Cada nodo en la red DECnet tiene una dirección única, generalmente en el formato area.node.
  • Áreas: Las áreas son subredes dentro de la red DECnet, permitiendo una organización jerárquica.
  • Fases de DECnet: DECnet ha evolucionado a través de varias fases, siendo las más comunes la Fase IV y la Fase V.

Instalación de DECnet

Paso 1: Verificar la Instalación de DECnet

Primero, verifica si DECnet ya está instalado en tu sistema OpenVMS. Puedes hacerlo ejecutando el siguiente comando en el terminal:

$ @SYS$MANAGER:NET$CONFIGURE

Si DECnet no está instalado, el sistema te lo indicará y te guiará a través del proceso de instalación.

Paso 2: Instalar DECnet

Si DECnet no está instalado, sigue estos pasos para instalarlo:

  1. Inserta el medio de instalación de OpenVMS.
  2. Ejecuta el siguiente comando para iniciar el proceso de instalación:
$ @SYS$UPDATE:VMSINSTAL DECNET
  1. Sigue las instrucciones en pantalla para completar la instalación.

Configuración Básica de DECnet

Paso 1: Configurar la Dirección del Nodo

Cada nodo en la red DECnet necesita una dirección única. La dirección se compone de un número de área y un número de nodo. Por ejemplo, 1.1 donde 1 es el área y 1 es el nodo.

Para configurar la dirección del nodo, ejecuta el siguiente comando:

$ @SYS$MANAGER:NET$CONFIGURE

Sigue las instrucciones en pantalla para configurar la dirección del nodo. Asegúrate de que la dirección que elijas no esté en uso por otro nodo en la red.

Paso 2: Configurar el Nombre del Nodo

El nombre del nodo es un identificador único que se utiliza para identificar el nodo en la red. Para configurar el nombre del nodo, sigue las instrucciones en el script de configuración:

$ @SYS$MANAGER:NET$CONFIGURE

Introduce el nombre del nodo cuando se te solicite. Asegúrate de que el nombre sea único en la red.

Paso 3: Configurar los Parámetros de Red

Durante el proceso de configuración, se te pedirá que configures varios parámetros de red, como la máscara de subred y la puerta de enlace predeterminada. Introduce los valores adecuados según la configuración de tu red.

Configuración Avanzada de DECnet

Configuración de Rutas Estáticas

En algunas redes, puede ser necesario configurar rutas estáticas para asegurar que los paquetes de red se envíen a través de la ruta correcta. Para configurar una ruta estática, utiliza el siguiente comando:

$ MCR NCP SET ROUTE destination-node /GATEWAY=gateway-node

Por ejemplo, para configurar una ruta estática al nodo 2.1 a través del nodo 1.2, utiliza:

$ MCR NCP SET ROUTE 2.1 /GATEWAY=1.2

Configuración de Servicios de Red

DECnet permite la configuración de varios servicios de red, como el servicio de terminal remoto (RTS) y el servicio de transferencia de archivos (FTS). Para configurar estos servicios, utiliza el siguiente comando:

$ MCR NCP SET OBJECT object-name /FILE=file-spec

Por ejemplo, para configurar el servicio de terminal remoto, utiliza:

$ MCR NCP SET OBJECT RTS /FILE=SYS$SYSTEM:RTS.EXE

Solución de Problemas Comunes

Verificar el Estado de DECnet

Para verificar el estado de DECnet, utiliza el siguiente comando:

$ MCR NCP SHOW EXECUTOR

Este comando mostrará información sobre el estado actual de DECnet en tu sistema.

Verificar la Conectividad de Red

Para verificar la conectividad de red entre dos nodos, utiliza el siguiente comando:

$ MCR NCP TELL node-name SHOW KNOWN LINKS

Este comando mostrará información sobre los enlaces de red conocidos entre los nodos.

Reiniciar DECnet

Si encuentras problemas con DECnet, a veces puede ser útil reiniciar el servicio. Para hacerlo, utiliza el siguiente comando:

$ @SYS$MANAGER:NET$SHUTDOWN
$ @SYS$MANAGER:NET$STARTUP
