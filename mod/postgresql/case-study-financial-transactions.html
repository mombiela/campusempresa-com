Introducción

En este estudio de caso, exploraremos cómo gestionar transacciones financieras utilizando PostgreSQL. Esto incluirá la creación de tablas, inserción de datos, realización de consultas y aseguramiento de la integridad de los datos. Al final de esta sección, tendrás una comprensión sólida de cómo manejar datos financieros en una base de datos relacional.

Configuración de la Base de Datos

Creación de la Base de Datos

Primero, vamos a crear una nueva base de datos para nuestras transacciones financieras.

CREATE DATABASE financial_transactions;

Conexión a la Base de Datos

Conéctate a la base de datos recién creada.

\c financial_transactions;

Creación de Tablas

Tabla de Cuentas

Necesitamos una tabla para almacenar la información de las cuentas.

CREATE TABLE accounts (
    account_id SERIAL PRIMARY KEY,
    account_name VARCHAR(100) NOT NULL,
    balance NUMERIC(15, 2) NOT NULL DEFAULT 0.00
);
  • account_id: Identificador único para cada cuenta.
  • account_name: Nombre del titular de la cuenta.
  • balance: Saldo actual de la cuenta.

Tabla de Transacciones

A continuación, creamos una tabla para almacenar los detalles de las transacciones.

CREATE TABLE transactions (
    transaction_id SERIAL PRIMARY KEY,
    account_id INT REFERENCES accounts(account_id),
    transaction_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    amount NUMERIC(15, 2) NOT NULL,
    transaction_type VARCHAR(10) CHECK (transaction_type IN ('credit', 'debit'))
);
  • transaction_id: Identificador único para cada transacción.
  • account_id: Clave foránea que referencia la tabla accounts.
  • transaction_date: Fecha y hora de la transacción.
  • amount: Monto de la transacción.
  • transaction_type: Tipo de transacción, ya sea 'credit' o 'debit'.

Inserción de Datos

Añadiendo Cuentas

Añadamos algunas cuentas a nuestra tabla accounts.

INSERT INTO accounts (account_name, balance) VALUES
('Alice', 1000.00),
('Bob', 1500.00),
('Charlie', 2000.00);

Añadiendo Transacciones

Ahora, añadiremos algunas transacciones.

INSERT INTO transactions (account_id, amount, transaction_type) VALUES
(1, 200.00, 'credit'),
(2, 300.00, 'debit'),
(3, 150.00, 'credit'),
(1, 50.00, 'debit');

Consultas de Datos

Visualización de Saldos de Cuentas

Para ver el saldo actual de todas las cuentas:

SELECT * FROM accounts;

Visualización de Transacciones

Para ver todas las transacciones:

SELECT * FROM transactions;

Uniendo Tablas

Para obtener una vista detallada de las transacciones junto con los nombres de las cuentas:

SELECT t.transaction_id, a.account_name, t.transaction_date, t.amount, t.transaction_type
FROM transactions t
JOIN accounts a ON t.account_id = a.account_id;

Asegurando la Integridad de los Datos

Uso de Transacciones

Para asegurar la integridad de los datos durante operaciones complejas, utiliza transacciones.

BEGIN;

UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
INSERT INTO transactions (account_id, amount, transaction_type) VALUES (1, 100.00, 'debit');

COMMIT;

Ejemplo de Rollback

Si algo sale mal, puedes hacer un rollback de la transacción.

BEGIN;

UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
-- Simular un error
-- INSERT INTO transactions (account_id, amount, transaction_type) VALUES (1, 'invalid_amount', 'debit');

ROLLBACK;

Conclusión

En este estudio de caso, hemos cubierto los conceptos básicos de la gestión de transacciones financieras en PostgreSQL. Creamos tablas para cuentas y transacciones, insertamos datos, realizamos consultas y aseguramos la integridad de los datos utilizando transacciones. Este conocimiento fundamental te ayudará a gestionar datos financieros de manera efectiva en una base de datos relacional.
