Asegurando Conexiones
Respaldo y Restauración ►

Asegurar las conexiones a tu base de datos PostgreSQL es crucial para proteger datos sensibles y garantizar que solo usuarios autorizados puedan acceder a la base de datos. Esta sección te guiará a través de varios métodos y mejores prácticas para asegurar las conexiones de PostgreSQL, desde técnicas básicas hasta avanzadas.

Introducción a la Seguridad de Conexiones

  • ¿Por qué asegurar las conexiones?

    • Proteger datos sensibles del acceso no autorizado.
    • Prevenir brechas de datos y posibles problemas legales.
    • Garantizar la integridad y confidencialidad de los datos.

  • Amenazas comunes:

    • Ataques de intermediario (man-in-the-middle).
    • Acceso no autorizado.
    • Intercepción de datos.

Medidas de Seguridad Básicas

Usando Autenticación por Contraseña

PostgreSQL soporta varios métodos de autenticación, pero la autenticación por contraseña es uno de los métodos más comunes y directos.

  • Configurando pg_hba.conf: El archivo pg_hba.conf controla la autenticación del cliente. Aquí se muestra cómo configurarlo para la autenticación por contraseña:

    # TYPE  DATABASE        USER            ADDRESS                 METHOD
host    all             all             0.0.0.0/0               md5
    • TYPE: Especifica el tipo de conexión (por ejemplo, host para conexiones TCP/IP).
    • DATABASE: Especifica el nombre de la base de datos (all para todas las bases de datos).
    • USER: Especifica el nombre del usuario (all para todos los usuarios).
    • ADDRESS: Especifica el rango de direcciones IP del cliente (0.0.0.0/0 para todas las direcciones).
    • METHOD: Especifica el método de autenticación (md5 para autenticación por contraseña).

Imponiendo Contraseñas Fuertes

  • Usando la extensión pgcrypto: La extensión pgcrypto de PostgreSQL puede ser utilizada para imponer contraseñas fuertes.

    CREATE EXTENSION pgcrypto;

    • Ejemplo de hash de una contraseña:

      SELECT crypt('mi_contraseña_segura', gen_salt('bf'));

Medidas de Seguridad Intermedias

Habilitando Conexiones SSL

Secure Sockets Layer (SSL) encripta los datos transmitidos entre el cliente y el servidor.

  • Configurando SSL en postgresql.conf:

    ssl = on
ssl_cert_file = 'server.crt'
ssl_key_file = 'server.key'

  • Generando certificados SSL:

    openssl req -new -text -out server.req
openssl rsa -in privkey.pem -out server.key
openssl req -x509 -in server.req -text -key server.key -out server.crt

  • Actualizando pg_hba.conf para SSL:

    hostssl all all 0.0.0.0/0 md5

Usando Certificados de Cliente

Los certificados de cliente añaden una capa extra de seguridad al requerir que los clientes proporcionen un certificado para autenticarse.

  • Generando certificados de cliente:

    openssl req -new -text -out client.req
openssl rsa -in privkey.pem -out client.key
openssl req -x509 -in client.req -text -key client.key -out client.crt

  • Configurando pg_hba.conf para certificados de cliente:

    hostssl all all 0.0.0.0/0 cert

Medidas de Seguridad Avanzadas

Usando Autenticación LDAP

Lightweight Directory Access Protocol (LDAP) puede ser utilizado para autenticación centralizada.

  • Configurando LDAP en pg_hba.conf:

    host all all 0.0.0.0/0 ldap ldapserver=ldap.ejemplo.com ldapbasedn="dc=ejemplo,dc=com"

Implementando Seguridad a Nivel de Fila (RLS)

La Seguridad a Nivel de Fila (RLS) te permite controlar el acceso a filas específicas en una tabla.

  • Habilitando RLS:

    ALTER TABLE empleados ENABLE ROW LEVEL SECURITY;

  • Creando una política:

    CREATE POLICY politica_empleados ON empleados
USING (departamento = current_setting('miapp.departamento_actual'));

  • Estableciendo el departamento actual:

    SET miapp.departamento_actual = 'Ventas';

Resumen

Asegurar las conexiones a tu base de datos PostgreSQL implica múltiples capas de medidas de seguridad, desde la autenticación básica por contraseña hasta técnicas avanzadas como SSL, certificados de cliente, autenticación LDAP y Seguridad a Nivel de Fila. Al implementar estas prácticas, puedes mejorar significativamente la seguridad de tu base de datos PostgreSQL y proteger tus datos del acceso no autorizado y amenazas potenciales.
