Introducción

Consultar datos es una de las operaciones fundamentales en cualquier sistema de gestión de bases de datos. En PostgreSQL, se utiliza el lenguaje SQL para recuperar, manipular y gestionar datos. Esta sección cubrirá los conceptos básicos de la consulta de datos, desde simples sentencias SELECT hasta técnicas de consulta más avanzadas.

Sentencias SELECT Básicas

La sentencia SELECT se utiliza para consultar datos de una base de datos. Los datos devueltos se almacenan en una tabla de resultados, a veces llamada conjunto de resultados.

Sintaxis

SELECT column1, column2, ...
FROM table_name;

Ejemplo

SELECT first_name, last_name
FROM employees;

Esta consulta recupera las columnas first_name y last_name de la tabla employees.

Filtrar Datos con WHERE

La cláusula WHERE se utiliza para filtrar registros que cumplen una cierta condición.

Sintaxis

SELECT column1, column2, ...
FROM table_name
WHERE condition;

Ejemplo

SELECT first_name, last_name
FROM employees
WHERE department = 'Sales';

Esta consulta recupera el first_name y last_name de los empleados que trabajan en el departamento de Ventas.

Ordenar Datos con ORDER BY

La cláusula ORDER BY se utiliza para ordenar el conjunto de resultados en orden ascendente o descendente.

Sintaxis

SELECT column1, column2, ...
FROM table_name
ORDER BY column1 [ASC|DESC], column2 [ASC|DESC], ...;

Ejemplo

SELECT first_name, last_name
FROM employees
ORDER BY last_name ASC;

Esta consulta recupera el first_name y last_name de los empleados y los ordena por last_name en orden ascendente.

Limitar Resultados con LIMIT

La cláusula LIMIT se utiliza para especificar el número de registros a devolver.

Sintaxis

SELECT column1, column2, ...
FROM table_name
LIMIT number;

Ejemplo

SELECT first_name, last_name
FROM employees
ORDER BY last_name ASC
LIMIT 10;

Esta consulta recupera los primeros 10 empleados ordenados por last_name en orden ascendente.

Combinar Condiciones con AND, OR y NOT

Puedes combinar múltiples condiciones en una cláusula WHERE usando AND, OR y NOT.

Ejemplo

SELECT first_name, last_name
FROM employees
WHERE department = 'Sales' AND salary > 50000;

Esta consulta recupera el first_name y last_name de los empleados que trabajan en el departamento de Ventas y tienen un salario mayor a 50,000.

Usar Funciones de Agregación

Las funciones de agregación realizan un cálculo sobre un conjunto de valores y devuelven un solo valor. Las funciones de agregación comunes incluyen COUNT, SUM, AVG, MAX y MIN.

Ejemplo

SELECT COUNT(*)
FROM employees
WHERE department = 'Sales';

Esta consulta devuelve el número de empleados en el departamento de Ventas.

Agrupar Datos con GROUP BY

La sentencia GROUP BY agrupa filas que tienen los mismos valores en filas de resumen.

Sintaxis

SELECT column1, aggregate_function(column2)
FROM table_name
GROUP BY column1;

Ejemplo

SELECT department, COUNT(*)
FROM employees
GROUP BY department;

Esta consulta devuelve el número de empleados en cada departamento.

Filtrar Grupos con HAVING

La cláusula HAVING se utiliza para filtrar grupos basados en una condición.

Ejemplo

SELECT department, COUNT(*)
FROM employees
GROUP BY department
HAVING COUNT(*) > 10;

Esta consulta devuelve los departamentos con más de 10 empleados.

Unir Tablas

Las uniones se utilizan para combinar filas de dos o más tablas basadas en una columna relacionada.

Tipos de Uniones

  • INNER JOIN: Devuelve registros que tienen valores coincidentes en ambas tablas.
  • LEFT JOIN (o LEFT OUTER JOIN): Devuelve todos los registros de la tabla izquierda y los registros coincidentes de la tabla derecha.
  • RIGHT JOIN (o RIGHT OUTER JOIN): Devuelve todos los registros de la tabla derecha y los registros coincidentes de la tabla izquierda.
  • FULL JOIN (o FULL OUTER JOIN): Devuelve todos los registros cuando hay una coincidencia en la tabla izquierda o derecha.

Ejemplo

SELECT employees.first_name, employees.last_name, departments.department_name
FROM employees
INNER JOIN departments ON employees.department_id = departments.department_id;

Esta consulta recupera el first_name y last_name de los empleados junto con su department_name.

Subconsultas

Una subconsulta es una consulta anidada dentro de otra consulta.

Ejemplo

SELECT first_name, last_name
FROM employees
WHERE department_id = (SELECT department_id FROM departments WHERE department_name = 'Sales');

Esta consulta recupera el first_name y last_name de los empleados que trabajan en el departamento de Ventas.

Conclusión

Consultar datos en PostgreSQL implica usar varias cláusulas y funciones SQL para recuperar y manipular datos de manera efectiva. Desde sentencias SELECT básicas hasta uniones y subconsultas complejas, dominar estas técnicas es esencial para una gestión eficiente de bases de datos y análisis de datos. Practica estas consultas para volverte competente en extraer información significativa de tus bases de datos PostgreSQL.
