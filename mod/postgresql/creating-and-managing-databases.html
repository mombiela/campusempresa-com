Creación de Bases de Datos

  • Crear una Base de Datos:

    CREATE DATABASE mydatabase;

    Explicación: Este comando crea una nueva base de datos llamada mydatabase.

  • Listar Bases de Datos:

    \l

    Explicación: Este comando lista todas las bases de datos en la instancia de PostgreSQL.

  • Conectarse a una Base de Datos:

    \c mydatabase

    Explicación: Este comando conecta a la base de datos mydatabase.

Gestión de Bases de Datos

  • Renombrar una Base de Datos:

    ALTER DATABASE mydatabase RENAME TO newdatabase;

    Explicación: Este comando renombra mydatabase a newdatabase.

  • Eliminar una Base de Datos:

    DROP DATABASE mydatabase;

    Explicación: Este comando elimina la base de datos mydatabase. Usar con precaución ya que esta acción es irreversible.

Usuarios y Permisos

  • Crear un Usuario:

    CREATE USER myuser WITH PASSWORD 'mypassword';

    Explicación: Este comando crea un nuevo usuario myuser con la contraseña mypassword.

  • Otorgar Permisos:

    GRANT ALL PRIVILEGES ON DATABASE mydatabase TO myuser;

    Explicación: Este comando otorga todos los privilegios sobre mydatabase a myuser.

  • Revocar Permisos:

    REVOKE ALL PRIVILEGES ON DATABASE mydatabase FROM myuser;

    Explicación: Este comando revoca todos los privilegios sobre mydatabase de myuser.

Tablas y Esquemas

  • Crear una Tabla:

    CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    position VARCHAR(50),
    salary NUMERIC
);

    Explicación: Este comando crea una tabla llamada employees con columnas para id, name, position y salary.

  • Listar Tablas:

    \dt

    Explicación: Este comando lista todas las tablas en la base de datos actual.

  • Crear un Esquema:

    CREATE SCHEMA myschema;

    Explicación: Este comando crea un nuevo esquema llamado myschema.

  • Usar un Esquema:

    SET search_path TO myschema;

    Explicación: Este comando establece la ruta de búsqueda a myschema, haciéndolo el esquema predeterminado para los comandos subsecuentes.

Gestión Avanzada de Bases de Datos

  • Respaldo y Restauración:

    • Respaldar una Base de Datos:

      pg_dump mydatabase > mydatabase_backup.sql

      Explicación: Este comando crea un respaldo de mydatabase y lo guarda en mydatabase_backup.sql.

    • Restaurar una Base de Datos:

      psql mydatabase < mydatabase_backup.sql

      Explicación: Este comando restaura mydatabase desde el archivo mydatabase_backup.sql.

Optimización del Rendimiento

  • Indexación:

    CREATE INDEX idx_name ON employees (name);

    Explicación: Este comando crea un índice en la columna name de la tabla employees para mejorar el rendimiento de las consultas.

  • Optimización de Consultas:

    • Usar EXPLAIN para analizar planes de consulta: 
      EXPLAIN SELECT * FROM employees WHERE name = 'John Doe';
      Explicación: Este comando muestra el plan de ejecución para la consulta, ayudando a identificar cuellos de botella en el rendimiento.

  • Vacuuming:

    VACUUM FULL;

    Explicación: Este comando recupera almacenamiento y optimiza la base de datos eliminando tuplas muertas.

Conclusión

En este tema, cubrimos los aspectos esenciales de la creación y gestión de bases de datos en PostgreSQL, desde la creación básica de bases de datos hasta técnicas avanzadas de optimización del rendimiento. Al comprender estos conceptos, puedes gestionar eficazmente tus bases de datos PostgreSQL, asegurando que sean seguras, eficientes y escalables.
