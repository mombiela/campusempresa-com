Crear tablas es un aspecto fundamental al trabajar con PostgreSQL. Este tema te guiará a través del proceso desde una perspectiva para principiantes hasta técnicas más avanzadas.

Introducción a las Tablas

Las tablas son la estructura principal utilizada para almacenar datos en una base de datos relacional. Cada tabla consta de filas y columnas, donde cada columna representa un atributo específico de los datos, y cada fila representa un registro.

Creación Básica de Tablas

Para crear una tabla en PostgreSQL, utilizas la declaración CREATE TABLE. Aquí hay un ejemplo simple:

CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    email VARCHAR(100),
    hire_date DATE
);

Explicación:

  • CREATE TABLE employees: Este comando crea una nueva tabla llamada employees.
  • id SERIAL PRIMARY KEY: Crea una columna id que se auto-incrementa con cada nuevo registro y sirve como clave primaria.
  • first_name VARCHAR(50): Crea una columna first_name que puede almacenar hasta 50 caracteres.
  • last_name VARCHAR(50): Crea una columna last_name que puede almacenar hasta 50 caracteres.
  • email VARCHAR(100): Crea una columna email que puede almacenar hasta 100 caracteres.
  • hire_date DATE: Crea una columna hire_date para almacenar valores de fecha.

Tipos de Datos

Entender los tipos de datos es crucial al crear tablas. Aquí hay algunos tipos de datos comunes utilizados en PostgreSQL:

Tipo de Dato Descripción
INTEGER Números enteros
SERIAL Entero auto-incremental
VARCHAR(n) Cadena de longitud variable con un límite de n caracteres
TEXT Cadena de longitud variable sin límite
DATE Fecha de calendario (año, mes, día)
BOOLEAN Valores verdaderos o falsos

Restricciones

Las restricciones son reglas aplicadas a las columnas para garantizar la integridad de los datos. Aquí hay algunas restricciones comunes:

  • PRIMARY KEY: Identifica de manera única cada registro.
  • NOT NULL: Asegura que una columna no pueda tener un valor NULL.
  • UNIQUE: Asegura que todos los valores en una columna sean únicos.
  • CHECK: Asegura que todos los valores en una columna cumplan una condición específica.
  • FOREIGN KEY: Asegura la integridad referencial entre dos tablas.

Ejemplo con Restricciones:

CREATE TABLE departments (
    department_id SERIAL PRIMARY KEY,
    department_name VARCHAR(100) NOT NULL UNIQUE,
    manager_id INTEGER,
    CHECK (department_name <> '')
);

Explicación:

  • department_id SERIAL PRIMARY KEY: Clave primaria auto-incremental.
  • department_name VARCHAR(100) NOT NULL UNIQUE: El nombre del departamento debe ser único y no puede ser NULL.
  • manager_id INTEGER: ID del gerente, que puede estar vinculado a otra tabla.
  • CHECK (department_name <> ''): Asegura que el nombre del departamento no sea una cadena vacía.

Creación Avanzada de Tablas

Uso de Claves Foráneas

Las claves foráneas se utilizan para vincular dos tablas juntas. Aquí hay un ejemplo:

CREATE TABLE projects (
    project_id SERIAL PRIMARY KEY,
    project_name VARCHAR(100) NOT NULL,
    start_date DATE,
    end_date DATE,
    department_id INTEGER,
    FOREIGN KEY (department_id) REFERENCES departments(department_id)
);

Explicación:

  • department_id INTEGER: Esta columna almacenará el ID del departamento.
  • FOREIGN KEY (department_id) REFERENCES departments(department_id): Esto refuerza una relación entre la tabla projects y la tabla departments.

Claves Primarias Compuestas

A veces, puedes necesitar una clave primaria que consista en múltiples columnas:

CREATE TABLE project_assignments (
    project_id INTEGER,
    employee_id INTEGER,
    assignment_date DATE,
    PRIMARY KEY (project_id, employee_id)
);

Explicación:

  • PRIMARY KEY (project_id, employee_id): Esto crea una clave primaria compuesta utilizando tanto project_id como employee_id.

Conclusión

Crear tablas en PostgreSQL implica entender la estructura de tus datos y aplicar los tipos de datos y restricciones apropiados para asegurar la integridad de los datos. Desde la creación básica de tablas hasta técnicas avanzadas como claves foráneas y claves primarias compuestas, dominar estos conceptos es esencial para una gestión efectiva de bases de datos.

Al practicar estos ejemplos y entender los principios subyacentes, estarás bien equipado para diseñar e implementar esquemas de bases de datos robustos en PostgreSQL.
