Integración Continua y Despliegue Continuo (CI/CD) en React Native
Construcción de una Aplicación de Tareas ►

La Integración Continua (CI) y el Despliegue Continuo (CD) son prácticas esenciales en el desarrollo de software moderno. Ayudan a garantizar que los cambios en el código se prueben y desplieguen automáticamente, reduciendo el riesgo de errores y mejorando la velocidad de entrega. Esta sección te guiará a través de la configuración de CI/CD para un proyecto de React Native, desde niveles principiantes hasta avanzados.

Introducción a CI/CD

  • Integración Continua (CI): La práctica de integrar automáticamente los cambios de código de múltiples contribuyentes en un repositorio compartido varias veces al día.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio que pasa todas las etapas de la canalización de producción a los usuarios finales.

Configuración de una Canalización CI/CD

Elegir un Servicio CI/CD

Hay varios servicios CI/CD disponibles, cada uno con sus propias fortalezas. Algunas opciones populares incluyen:

  • GitHub Actions
  • CircleCI
  • Travis CI
  • Bitrise

Para este curso, utilizaremos GitHub Actions debido a su integración perfecta con los repositorios de GitHub.

Canalización CI/CD Básica con GitHub Actions

  1. Crear un Repositorio en GitHub: Comienza creando un nuevo repositorio en GitHub para tu proyecto de React Native.

  2. Agregar un Archivo de Flujo de Trabajo: En tu repositorio, crea un directorio .github/workflows y añade un nuevo archivo llamado ci.yml.

name: CI

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

Explicación del Archivo de Flujo de Trabajo

  • name: El nombre del flujo de trabajo.
  • on: Especifica los eventos que desencadenan el flujo de trabajo. Aquí, se ejecuta en los empujes y solicitudes de extracción a la rama main.
  • jobs: Define los trabajos a ejecutar. En este caso, tenemos un solo trabajo llamado build.
  • runs-on: Especifica el tipo de máquina en la que se ejecutará el trabajo. ubuntu-latest es una opción comúnmente utilizada.
  • steps: Los pasos individuales a ejecutar en el trabajo.
    • Checkout repository: Utiliza la acción actions/checkout para clonar el repositorio.
    • Set up Node.js: Utiliza la acción actions/setup-node para configurar Node.js.
    • Install dependencies: Ejecuta npm install para instalar las dependencias del proyecto.
    • Run tests: Ejecuta npm test para ejecutar la suite de pruebas.

Canalización CI/CD Avanzada

Agregar Linting y Formateo

Para garantizar la calidad del código, puedes agregar pasos para linting y formateo.

    - name: Run ESLint
      run: npm run lint

    - name: Run Prettier
      run: npm run format

Construcción y Despliegue de la Aplicación

Para desplegar una aplicación de React Native, es posible que desees construir la aplicación y desplegarla en un servicio como Firebase App Distribution o la App Store.

Construcción de la Aplicación

    - name: Build Android
      run: |
        cd android
        ./gradlew assembleRelease

    - name: Build iOS
      run: |
        cd ios
        xcodebuild -workspace YourApp.xcworkspace -scheme YourApp -sdk iphoneos -configuration AppStoreDistribution archive -archivePath $PWD/build/YourApp.xcarchive

Despliegue de la Aplicación

Para desplegar en Firebase App Distribution:

    - name: Deploy to Firebase App Distribution
      uses: wzieba/Firebase-Distribution-Github-Action@v1
      with:
        appId: ${{ secrets.FIREBASE_APP_ID }}
        token: ${{ secrets.FIREBASE_TOKEN }}
        groups: testers
        file: android/app/build/outputs/apk/release/app-release.apk

Conclusión

Configurar una canalización CI/CD para tu proyecto de React Native puede mejorar significativamente tu flujo de trabajo de desarrollo al automatizar los procesos de prueba y despliegue. Comenzar con una configuración básica de CI y agregar gradualmente características más avanzadas como linting, formateo, construcción y despliegue asegura que tu código mantenga una alta calidad y que tu aplicación llegue a los usuarios de manera rápida y confiable.

Al integrar CI/CD en tus proyectos de React Native, puedes:

  • Detectar errores temprano con pruebas automatizadas.
  • Asegurar una calidad de código consistente con linting y formateo.
  • Desplegar actualizaciones más rápido y de manera más confiable.

Recuerda, la clave para un CI/CD efectivo es comenzar simple e iterar, agregando más complejidad a medida que tu proyecto crece y tus necesidades evolucionan.
