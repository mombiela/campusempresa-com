Introducción

Las pruebas y la depuración son aspectos cruciales del desarrollo de software. Aseguran que tu código funcione como se espera y ayudan a identificar y corregir problemas. En esta sección, cubriremos los conceptos básicos de las pruebas y la depuración en Groovy, progresando desde niveles principiantes hasta avanzados.

Pruebas Unitarias con Groovy

¿Qué son las Pruebas Unitarias?

  • Las pruebas unitarias implican probar componentes o funciones individuales de tu código para asegurar que funcionen correctamente.
  • Ayudan a detectar errores temprano en el proceso de desarrollo.

Configuración del Entorno de Pruebas

  • Groovy utiliza JUnit, un popular marco de pruebas para Java, que se integra perfectamente con Groovy.
  • Para comenzar, necesitas incluir JUnit en las dependencias de tu proyecto.
// build.gradle
dependencies {
    testImplementation 'junit:junit:4.13.2'
}

Escribiendo Tu Primera Prueba

  • Crea una clase de prueba y anótala con @Test.
  • Usa afirmaciones para verificar los resultados esperados.
import org.junit.Test
import static org.junit.Assert.*

class CalculatorTest {
    @Test
    void testAddition() {
        def calculator = new Calculator()
        assertEquals(5, calculator.add(2, 3))
    }
}

class Calculator {
    int add(int a, int b) {
        return a + b
    }
}

Ejecutando Pruebas

  • Puedes ejecutar tus pruebas usando Gradle o un IDE como IntelliJ IDEA o Eclipse.
  • En Gradle, usa el comando ./gradlew test.

Técnicas Avanzadas de Pruebas

Mocking y Stubbing

  • Mocking y stubbing son técnicas utilizadas para simular el comportamiento de objetos complejos.
  • Groovy proporciona el paquete groovy.mock.interceptor para crear mocks.
import groovy.mock.interceptor.*

class UserService {
    def userRepository

    def getUserById(id) {
        return userRepository.findById(id)
    }
}

class UserServiceTest {
    @Test
    void testGetUserById() {
        def userRepository = Mock(UserRepository)
        userRepository.findById(1) >> new User(id: 1, name: 'John Doe')

        def userService = new UserService(userRepository: userRepository)
        def user = userService.getUserById(1)

        assertEquals('John Doe', user.name)
    }
}

Pruebas Parametrizadas

  • Las pruebas parametrizadas te permiten ejecutar la misma prueba con diferentes entradas.
  • JUnit proporciona las anotaciones @RunWith y @Parameters para este propósito.
import org.junit.Test
import org.junit.runner.RunWith
import org.junit.runners.Parameterized

@RunWith(Parameterized)
class CalculatorParameterizedTest {
    int a, b, expected

    CalculatorParameterizedTest(int a, int b, int expected) {
        this.a = a
        this.b = b
        this.expected = expected
    }

    @Parameterized.Parameters
    static data() {
        return [
            [1, 2, 3],
            [2, 3, 5],
            [3, 3, 6]
        ]
    }

    @Test
    void testAddition() {
        def calculator = new Calculator()
        assertEquals(expected, calculator.add(a, b))
    }
}

Depuración de Código Groovy

Usando Declaraciones Print

  • La forma más simple de depuración es usar declaraciones print para mostrar los valores de las variables y el flujo del programa.
def add(a, b) {
    println "Adding $a and $b"
    return a + b
}

println add(2, 3)

Usando un Depurador

  • Los IDEs modernos como IntelliJ IDEA y Eclipse vienen con depuradores integrados.
  • Establece puntos de interrupción en tu código y ejecuta el depurador para recorrer tu código línea por línea.

Técnicas Comunes de Depuración

  • Puntos de Interrupción: Pausa la ejecución en líneas específicas.
  • Paso Sobre: Ejecuta la línea actual y pasa a la siguiente.
  • Paso Dentro: Entra en las llamadas a métodos.
  • Ver Variables: Monitorea los valores de las variables en tiempo real.

Conclusión

Las pruebas y la depuración son habilidades esenciales para cualquier programador. En Groovy, puedes aprovechar herramientas poderosas como JUnit para pruebas y IDEs modernos para depuración para asegurar que tu código sea robusto y libre de errores. Al dominar estas técnicas, estarás bien equipado para manejar cualquier problema que surja durante el desarrollo.
