Introducción a las Transformaciones AST

Las Transformaciones del Árbol de Sintaxis Abstracta (AST) son una característica poderosa en Groovy que permite a los desarrolladores modificar la estructura del código en tiempo de compilación. Esto se puede usar para inyectar código, realizar optimizaciones o imponer estándares de codificación.

  • AST: Árbol de Sintaxis Abstracta, una representación en árbol de la estructura sintáctica abstracta del código fuente.
  • Transformación: Modificar el AST para cambiar cómo se comporta o se compila el código.

Conceptos Clave

  • Metaprogramación en tiempo de compilación: Las transformaciones AST ocurren durante la fase de compilación, permitiendo modificaciones antes de que el código se ejecute.
  • Anotaciones: Groovy usa anotaciones para activar transformaciones AST.
  • Fases: El proceso de compilación de Groovy se divide en varias fases, y las transformaciones se pueden aplicar en diferentes etapas.

Transformaciones AST Comunes

Groovy proporciona varias transformaciones AST integradas que se pueden usar para simplificar el código y agregar funcionalidad.

@ToString

Genera un método toString() para una clase.

import groovy.transform.ToString

@ToString
class Person {
    String name
    int age
}

def person = new Person(name: 'John', age: 30)
println person.toString() // Output: Person(John, 30)

@EqualsAndHashCode

Genera métodos equals() y hashCode().

import groovy.transform.EqualsAndHashCode

@EqualsAndHashCode
class Person {
    String name
    int age
}

def person1 = new Person(name: 'John', age: 30)
def person2 = new Person(name: 'John', age: 30)
assert person1 == person2 // true

@Immutable

Hace que una clase sea inmutable generando campos finales y proporcionando un constructor.

import groovy.transform.Immutable

@Immutable
class Point {
    int x, y
}

def point = new Point(10, 20)
assert point.x == 10
assert point.y == 20

Transformaciones AST Personalizadas

Crear transformaciones AST personalizadas permite modificaciones más avanzadas y específicas.

Escribir una Transformación AST Simple

  1. Crear la Anotación: Definir una anotación para marcar dónde se debe aplicar la transformación.
import java.lang.annotation.ElementType
import java.lang.annotation.Retention
import java.lang.annotation.RetentionPolicy
import java.lang.annotation.Target

@Retention(RetentionPolicy.SOURCE)
@Target([ElementType.TYPE])
@interface MyTransformation {}
  1. Implementar la Transformación: Crear una clase que implemente ASTTransformation.
import org.codehaus.groovy.ast.*
import org.codehaus.groovy.ast.builder.AstBuilder
import org.codehaus.groovy.control.*
import org.codehaus.groovy.transform.*

@GroovyASTTransformation(phase = CompilePhase.CANONICALIZATION)
class MyTransformationImpl implements ASTTransformation {
    void visit(ASTNode[] nodes, SourceUnit source) {
        nodes.each { node ->
            if (node instanceof ClassNode) {
                ClassNode classNode = (ClassNode) node
                // Ejemplo: Agregar un nuevo método a la clase
                classNode.addMethod(
                    new MethodNode(
                        'newMethod',
                        ACC_PUBLIC,
                        ClassHelper.VOID_TYPE,
                        [] as Parameter[],
                        [] as ClassNode[],
                        new BlockStatement(
                            [new ExpressionStatement(new ConstantExpression('¡Hola desde la Transformación AST!'))],
                            new VariableScope()
                        )
                    )
                )
            }
        }
    }
}
  1. Aplicar la Anotación: Usar la anotación en tu código.
@MyTransformation
class Example {}

def example = new Example()
example.newMethod() // Output: ¡Hola desde la Transformación AST!

Conclusión

Las Transformaciones AST en Groovy proporcionan un mecanismo robusto para modificar el código en tiempo de compilación, habilitando capacidades de metaprogramación poderosas. Al entender y utilizar tanto las transformaciones integradas como las personalizadas, los desarrolladores pueden escribir código más conciso, mantenible y eficiente. A medida que avances en Groovy, dominar las transformaciones AST mejorará significativamente tu capacidad para manipular y optimizar tu base de código.
