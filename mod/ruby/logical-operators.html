Introducción

Los operadores lógicos son fundamentales en la programación ya que te permiten tomar decisiones basadas en múltiples condiciones. En Ruby, los operadores lógicos se utilizan para combinar dos o más condiciones y producir un único resultado booleano. Esta sección cubrirá los operadores lógicos básicos en Ruby, su uso y ejemplos para ayudarte a entender cómo aplicarlos en tu código.

Operadores Lógicos Básicos

Ruby proporciona tres operadores lógicos principales:

  • && (AND)
  • || (OR)
  • ! (NOT)

Operador AND (`&&`)

El operador && devuelve true si ambos operandos son verdaderos. Si cualquiera de los operandos es falso, devuelve false.

Ejemplo:

a = true
b = false

result = a && b
puts result # Output: false

Explicación:

  • a && b evalúa a false porque b es false.

Operador OR (`||`)

El operador || devuelve true si al menos uno de los operandos es verdadero. Devuelve false solo si ambos operandos son falsos.

Ejemplo:

a = true
b = false

result = a || b
puts result # Output: true

Explicación:

  • a || b evalúa a true porque a es true.

Operador NOT (`!`)

El operador ! niega el valor booleano de su operando. Si el operando es true, devuelve false, y viceversa.

Ejemplo:

a = true

result = !a
puts result # Output: false

Explicación:

  • !a evalúa a false porque a es true.

Combinando Operadores Lógicos

Puedes combinar múltiples operadores lógicos para formar condiciones complejas. Se pueden usar paréntesis para controlar el orden de evaluación.

Ejemplo:

a = true
b = false
c = true

result = (a && b) || c
puts result # Output: true

Explicación:

  • (a && b) evalúa a false porque b es false.
  • false || c evalúa a true porque c es true.

Evaluación de Cortocircuito

Ruby utiliza la evaluación de cortocircuito para los operadores lógicos. Esto significa que la evaluación se detiene tan pronto como se determina el resultado.

Cortocircuito del Operador AND

Si el primer operando de && es false, el segundo operando no se evalúa porque el resultado ya es false.

Ejemplo:

a = false
b = true

result = a && (b = false)
puts result # Output: false
puts b      # Output: true

Explicación:

  • a && (b = false) deja de evaluarse después de a porque a es false.
  • b permanece true porque (b = false) no se ejecuta.

Cortocircuito del Operador OR

Si el primer operando de || es true, el segundo operando no se evalúa porque el resultado ya es true.

Ejemplo:

a = true
b = false

result = a || (b = true)
puts result # Output: true
puts b      # Output: false

Explicación:

  • a || (b = true) deja de evaluarse después de a porque a es true.
  • b permanece false porque (b = true) no se ejecuta.

Valores Verdaderos y Falsos

En Ruby, cada valor es "verdadero" o "falso". Solo nil y false se consideran falsos; todo lo demás es verdadero.

Ejemplo:

a = nil
b = 0
c = ""

result = a || b || c
puts result # Output: 0

Explicación:

  • a es nil (falso), por lo que la evaluación continúa.
  • b es 0 (verdadero), por lo que la evaluación se detiene y se devuelve 0.

Ejercicios

  1. Escribe un script en Ruby que verifique si un número está dentro de un rango específico (por ejemplo, entre 10 y 20) usando operadores lógicos.
  2. Crea un método en Ruby que tome dos argumentos booleanos y devuelva true si al menos uno de ellos es true.
  3. Escribe un script en Ruby que use operadores lógicos para determinar si una cadena no está vacía y no contiene solo espacios en blanco.

Conclusión

Los operadores lógicos son herramientas esenciales para controlar el flujo de tus programas en Ruby. Al entender cómo usar &&, || y !, puedes crear condiciones más complejas y dinámicas. Recuerda considerar la evaluación de cortocircuito y el concepto de valores verdaderos y falsos al trabajar con operadores lógicos. Practica con los ejercicios proporcionados para solidificar tu comprensión.
