¿Qué son las Excepciones?

Las excepciones son eventos que pueden alterar el flujo normal de un programa. Se utilizan para manejar errores y otras condiciones excepcionales de manera controlada. En Ruby, las excepciones son objetos que representan estas condiciones de error.

Conceptos Clave

  • Manejo de Excepciones: El proceso de responder a la ocurrencia de excepciones.
  • Raise: La acción de crear una excepción.
  • Rescue: La acción de manejar una excepción.
  • Ensure: Un bloque que siempre se ejecutará, independientemente de si se levantó una excepción o no.

Levantando Excepciones

En Ruby, puedes levantar una excepción usando la palabra clave raise. Esto se puede hacer con o sin un argumento.

Ejemplo: Levantando una Excepción

# Levantando una excepción sin un argumento
raise "¡Ha ocurrido un error!"

# Levantando una excepción con un tipo específico
raise ArgumentError, "¡Argumento inválido!"

Manejando Excepciones

Para manejar excepciones, Ruby proporciona el bloque begin-rescue. Esto te permite capturar y gestionar excepciones de manera elegante.

Ejemplo: Manejando una Excepción

begin
  # Código que puede levantar una excepción
  result = 10 / 0
rescue ZeroDivisionError => e
  # Código para manejar la excepción
  puts "Error: #{e.message}"
end

Explicación

  • begin: Inicia el bloque de código donde pueden ocurrir excepciones.
  • rescue: Define el bloque de código a ejecutar si se levanta una excepción.
  • ZeroDivisionError: El tipo específico de excepción que se está rescatando.
  • e: La variable que contiene el objeto de excepción.

Asegurando la Ejecución

El bloque ensure se utiliza para garantizar que cierto código siempre se ejecute, independientemente de si se levantó una excepción o no.

Ejemplo: Usando Ensure

begin
  # Código que puede levantar una excepción
  file = File.open("example.txt", "r")
  # Realizar operaciones en el archivo
rescue Errno::ENOENT => e
  # Manejar la excepción de archivo no encontrado
  puts "Archivo no encontrado: #{e.message}"
ensure
  # Asegurar que el archivo se cierre
  file.close if file
end

Tipos Comunes de Excepciones

Aquí hay algunas excepciones comunes que podrías encontrar en Ruby:

Tipo de Excepción Descripción
StandardError El predeterminado para la mayoría de los errores
ArgumentError Se levanta cuando los argumentos son incorrectos
NoMethodError Se levanta cuando no se encuentra un método
RuntimeError Un error genérico levantado por raise
ZeroDivisionError Se levanta al dividir por cero
TypeError Se levanta cuando un objeto es del tipo incorrecto

Excepciones Personalizadas

Puedes definir tus propias excepciones creando una nueva clase que herede de StandardError.

Ejemplo: Excepción Personalizada

class MyCustomError < StandardError
end

begin
  # Código que puede levantar una excepción personalizada
  raise MyCustomError, "¡Algo salió mal!"
rescue MyCustomError => e
  # Manejar la excepción personalizada
  puts "Error personalizado: #{e.message}"
end

Resumen

  • Las excepciones se utilizan para manejar errores y condiciones excepcionales en Ruby.
  • Usa raise para crear excepciones y rescue para manejarlas.
  • El bloque ensure garantiza que cierto código siempre se ejecute.
  • Ruby proporciona una variedad de tipos de excepciones integradas, y también puedes crear excepciones personalizadas.

Al comprender y utilizar las excepciones, puedes hacer que tus programas en Ruby sean más robustos y resistentes a errores.
