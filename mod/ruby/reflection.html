Reflexión en Ruby
DSLs (Lenguajes Específicos de Dominio) ►

La reflexión es una característica poderosa en Ruby que permite a un programa inspeccionar y modificar su propia estructura y comportamiento en tiempo de ejecución. Esta capacidad es esencial para la metaprogramación, donde el código puede definir dinámicamente métodos, clases y otras estructuras.

Conceptos Clave

  • Introspección: La capacidad de examinar las propiedades de los objetos, como su clase, métodos y variables de instancia.
  • Invocación Dinámica de Métodos: Llamar a métodos cuyos nombres no se conocen hasta el tiempo de ejecución.
  • Modificación de Clases y Módulos: Agregar o alterar métodos y propiedades de clases y módulos durante la ejecución.

Introspección

La introspección es el proceso de examinar las propiedades de los objetos en tiempo de ejecución. Ruby proporciona varios métodos para facilitar la introspección.

Ejemplo: Inspeccionando la Clase y Métodos de un Objeto

class Person
  attr_accessor :name, :age

  def initialize(name, age)
    @name = name
    @age = age
  end

  def greet
    "Hello, my name is #{@name}."
  end
end

person = Person.new("Alice", 30)

# Inspeccionando la clase del objeto
puts person.class # Output: Person

# Listando todos los métodos del objeto
puts person.methods.sort

# Listando variables de instancia
puts person.instance_variables # Output: [:@name, :@age]

# Obteniendo el valor de una variable de instancia
puts person.instance_variable_get(:@name) # Output: Alice

Invocación Dinámica de Métodos

La invocación dinámica de métodos te permite llamar a métodos cuyos nombres se determinan en tiempo de ejecución. Esto se puede hacer usando el método send.

Ejemplo: Usando `send` para Invocar Métodos Dinámicamente

class Calculator
  def add(a, b)
    a + b
  end

  def subtract(a, b)
    a - b
  end
end

calc = Calculator.new

# Nombres de métodos como símbolos
method_name = :add
puts calc.send(method_name, 5, 3) # Output: 8

method_name = :subtract
puts calc.send(method_name, 5, 3) # Output: 2

Modificación de Clases y Módulos

Ruby te permite modificar clases y módulos existentes en tiempo de ejecución. Esto puede ser útil para agregar nuevos métodos o alterar los existentes.

Ejemplo: Agregando Métodos a una Clase

class String
  def shout
    self.upcase + "!!!"
  end
end

puts "hello".shout # Output: HELLO!!!

Ejemplo: Usando `define_method` para Crear Métodos Dinámicamente

class Person
  [:name, :age, :gender].each do |attr|
    define_method(attr) do
      instance_variable_get("@#{attr}")
    end

    define_method("#{attr}=") do |value|
      instance_variable_set("@#{attr}", value)
    end
  end
end

person = Person.new
person.name = "Bob"
person.age = 25
person.gender = "Male"

puts person.name   # Output: Bob
puts person.age    # Output: 25
puts person.gender # Output: Male

Conclusión

La reflexión en Ruby proporciona herramientas poderosas para la introspección, la invocación dinámica de métodos y la modificación de clases y módulos en tiempo de ejecución. Estas capacidades son esenciales para la metaprogramación y pueden mejorar enormemente la flexibilidad y dinamismo de tu código. Al comprender y utilizar la reflexión, puedes escribir programas Ruby más adaptables e inteligentes.
