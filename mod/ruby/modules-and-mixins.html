Introducción a los Módulos

Los módulos en Ruby son una forma de agrupar métodos, constantes y variables de clase relacionadas. Tienen dos propósitos principales: espacios de nombres y mixins.

  • Espacios de nombres: Evita conflictos de nombres agrupando clases, métodos y constantes relacionadas.
  • Mixins: Permite compartir métodos entre múltiples clases.

Creando un Módulo

Para definir un módulo, usa la palabra clave module seguida del nombre del módulo. Los nombres de los módulos son constantes y deben comenzar con una letra mayúscula.

module Greetings
  def say_hello
    "¡Hola!"
  end

  def say_goodbye
    "¡Adiós!"
  end
end

Usando Módulos para Espacios de Nombres

Los espacios de nombres ayudan a organizar el código y evitar conflictos con nombres de métodos y clases.

module MathTools
  PI = 3.14159

  def self.square(x)
    x * x
  end
end

puts MathTools::PI          # => 3.14159
puts MathTools.square(3)    # => 9

Mixins con Módulos

Los mixins permiten incluir métodos de módulos en clases. Esto se hace usando las palabras clave include o extend.

Incluyendo Módulos

Cuando incluyes un módulo, sus métodos se convierten en métodos de instancia de la clase.

module Greetings
  def say_hello
    "¡Hola!"
  end
end

class Person
  include Greetings
end

person = Person.new
puts person.say_hello  # => "¡Hola!"

Extendiendo Módulos

Cuando extiendes un módulo, sus métodos se convierten en métodos de clase.

module Greetings
  def say_hello
    "¡Hola!"
  end
end

class Person
  extend Greetings
end

puts Person.say_hello  # => "¡Hola!"

Características Avanzadas de los Módulos

Constantes del Módulo

Los módulos pueden contener constantes que pueden ser accedidas usando el operador de resolución de ámbito ::.

module Config
  MAX_USERS = 100
end

puts Config::MAX_USERS  # => 100

Métodos del Módulo

Los módulos pueden tener métodos definidos con self. para hacerlos métodos del módulo.

module MathTools
  def self.square(x)
    x * x
  end
end

puts MathTools.square(4)  # => 16

Incluyendo Múltiples Módulos

Una clase puede incluir múltiples módulos, permitiéndole heredar métodos de todos los módulos incluidos.

module Greetings
  def say_hello
    "¡Hola!"
  end
end

module Farewells
  def say_goodbye
    "¡Adiós!"
  end
end

class Person
  include Greetings
  include Farewells
end

person = Person.new
puts person.say_hello   # => "¡Hola!"
puts person.say_goodbye # => "¡Adiós!"

Ejercicios Prácticos

  1. Crea un módulo llamado MathOperations con métodos para suma, resta, multiplicación y división. Incluye este módulo en una clase y demuestra su uso.
module MathOperations
  def add(a, b)
    a + b
  end

  def subtract(a, b)
    a - b
  end

  def multiply(a, b)
    a * b
  end

  def divide(a, b)
    a / b
  end
end

class Calculator
  include MathOperations
end

calc = Calculator.new
puts calc.add(5, 3)        # => 8
puts calc.subtract(5, 3)   # => 2
puts calc.multiply(5, 3)   # => 15
puts calc.divide(6, 3)     # => 2
  1. Define un módulo Trackable con un método track que imprima "Tracking...". Incluye este módulo en una clase Vehicle y llama al método track.
module Trackable
  def track
    "Tracking..."
  end
end

class Vehicle
  include Trackable
end

car = Vehicle.new
puts car.track  # => "Tracking..."

Conclusión

Los módulos y mixins en Ruby proporcionan una forma poderosa de organizar y reutilizar código. Al usar módulos para espacios de nombres, puedes evitar conflictos de nombres y mantener tu base de código limpia. Los mixins te permiten compartir métodos entre múltiples clases, promoviendo la reutilización del código y reduciendo la duplicación. Entender y usar efectivamente los módulos y mixins es esencial para escribir aplicaciones Ruby mantenibles y escalables.
