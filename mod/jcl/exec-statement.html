Instrucción EXEC en JCL
Declaración DD ►

La instrucción EXEC es uno de los componentes más críticos en el Lenguaje de Control de Trabajos (JCL). Se utiliza para especificar la ejecución de un programa o un procedimiento dentro de un trabajo. Este tema cubrirá la instrucción EXEC desde un nivel principiante hasta avanzado, proporcionando explicaciones claras, ejemplos y ejercicios.

Introducción a la Instrucción EXEC

  • Propósito: La instrucción EXEC se utiliza para identificar el programa o procedimiento que un paso de trabajo ejecutará.
  • Sintaxis: La sintaxis básica de la instrucción EXEC es la siguiente: 
    //stepname EXEC PGM=program-name
    • stepname: Un nombre único para el paso de trabajo.
    • PGM=program-name: Especifica el nombre del programa a ejecutar.

Ejemplo Básico

//STEP1    EXEC PGM=IEFBR14
  • Explicación:
    • STEP1: El nombre del paso de trabajo.
    • PGM=IEFBR14: Especifica que se ejecutará el programa IEFBR14 (un programa ficticio).

Uso de Parámetros en la Instrucción EXEC

Parámetros Comunes

  • PARM: Se utiliza para pasar parámetros al programa. 
    //STEP1    EXEC PGM=MYPROG,PARM='parameter-string'
  • ACCT: Especifica información contable. 
    //STEP1    EXEC PGM=MYPROG,ACCT='account-info'
  • REGION: Especifica la cantidad de memoria asignada al paso de trabajo. 
    //STEP1    EXEC PGM=MYPROG,REGION=4M

Ejemplo con Parámetros

//STEP1    EXEC PGM=MYPROG,PARM='TEST',REGION=4M
  • Explicación:
    • PARM='TEST': Pasa la cadena 'TEST' al programa MYPROG.
    • REGION=4M: Asigna 4 megabytes de memoria al paso de trabajo.

Uso Avanzado de la Instrucción EXEC

Ejecución de Procedimientos

  • Sintaxis: Para ejecutar un procedimiento, use la siguiente sintaxis: 
    //stepname EXEC procname
    • procname: El nombre del procedimiento a ejecutar.

Ejemplo de Ejecución de un Procedimiento

//STEP1    EXEC PROC=MYPROC
  • Explicación:
    • PROC=MYPROC: Especifica que se ejecutará el procedimiento MYPROC.

Sobrescribir Parámetros del Procedimiento

  • Sintaxis: Para sobrescribir parámetros en un procedimiento, use la siguiente sintaxis: 
    //stepname EXEC procname
//stepname.ddname DD parameter-override
    • ddname: El nombre de la instrucción DD dentro del procedimiento.
    • parameter-override: El nuevo valor del parámetro.

Ejemplo de Sobrescribir Parámetros del Procedimiento

//STEP1    EXEC PROC=MYPROC
//STEP1.INPUT DD DSN=NEW.DATASET,DISP=SHR
  • Explicación:
    • INPUT: La instrucción DD dentro del procedimiento MYPROC.
    • DSN=NEW.DATASET,DISP=SHR: Sobrescribe el nombre del dataset y la disposición.

Ejercicios

  1. Instrucción EXEC Básica:

    • Escriba una instrucción EXEC para ejecutar un programa llamado MYPROG con un nombre de paso STEP1.

  2. Uso de Parámetros:

    • Modifique la instrucción EXEC para pasar la cadena de parámetros 'RUN' y asignar 2 megabytes de memoria.

  3. Ejecución de un Procedimiento:

    • Escriba una instrucción EXEC para ejecutar un procedimiento llamado MYPROC con un nombre de paso STEP2.

  4. Sobrescribir Parámetros del Procedimiento:

    • Sobrescriba la instrucción DD OUTPUT en el procedimiento MYPROC para usar el dataset NEW.OUTPUT.DATA.

Conclusión

La instrucción EXEC es una parte fundamental del JCL, permitiéndole especificar la ejecución de programas y procedimientos dentro de un trabajo. Comprender su sintaxis y parámetros es crucial para gestionar eficazmente los pasos de trabajo. Al dominar la instrucción EXEC, puede controlar el flujo de ejecución y la asignación de recursos de sus trabajos JCL, haciendo que sus flujos de trabajo sean más eficientes y manejables.
