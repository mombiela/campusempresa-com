◄Definiendo Conjuntos de Datos
Disposición de Conjuntos de Datos
Parámetros del Conjunto de Datos ►

En esta sección, cubriremos el concepto de Disposición de Conjuntos de Datos en JCL (Lenguaje de Control de Trabajos). Entender cómo gestionar conjuntos de datos es crucial para la ejecución eficiente de trabajos y el manejo de datos en entornos de mainframe.

Conceptos Clave

  • Conjunto de Datos: Un conjunto de datos es una colección de registros de datos almacenados en un disco o cinta.
  • Disposición: La disposición se refiere al estado o condición de un conjunto de datos antes y después de que se ejecute un paso del trabajo.

Parámetro DISP

El parámetro DISP en JCL especifica el estado de un conjunto de datos al inicio y al final de un paso del trabajo. Se utiliza en la declaración DD (Definición de Datos).

Sintaxis de DISP

El parámetro DISP tiene la siguiente sintaxis:

DISP=(estado, terminación-normal, terminación-anormal)
  • estado: Especifica el estado del conjunto de datos antes de que se ejecute el paso del trabajo.
  • terminación-normal: Especifica la acción a tomar si el paso del trabajo termina normalmente.
  • terminación-anormal: Especifica la acción a tomar si el paso del trabajo termina anormalmente.

Valores de Estado

  • NEW: Indica que el conjunto de datos se va a crear.
  • OLD: Indica que el conjunto de datos ya existe y se va a usar.
  • MOD: Indica que el conjunto de datos ya existe y se va a añadir.
  • SHR: Indica que el conjunto de datos ya existe y se puede compartir.

Valores de Terminación

  • DELETE: Elimina el conjunto de datos.
  • KEEP: Mantiene el conjunto de datos.
  • CATLG: Cataloga el conjunto de datos.
  • UNCATLG: Descataloga el conjunto de datos.

Ejemplo de Parámetro DISP

//MYDATA  DD  DSN=MY.DATASET, DISP=(NEW,CATLG,DELETE)

En este ejemplo:

  • El conjunto de datos MY.DATASET se crea si no existe.
  • Si el paso del trabajo termina normalmente, el conjunto de datos se cataloga.
  • Si el paso del trabajo termina anormalmente, el conjunto de datos se elimina.

Escenarios Comunes de DISP

Creación de un Nuevo Conjunto de Datos

//NEWDS   DD  DSN=NEW.DATASET, DISP=(NEW,CATLG,DELETE), 
//            SPACE=(CYL,(5,1)), UNIT=SYSDA
  • NEW: Indica que el conjunto de datos se va a crear.
  • CATLG: Cataloga el conjunto de datos al terminar normalmente.
  • DELETE: Elimina el conjunto de datos al terminar anormalmente.

Uso de un Conjunto de Datos Existente

//OLDDATA DD  DSN=EXISTING.DATASET, DISP=(OLD,KEEP,KEEP)
  • OLD: Indica que el conjunto de datos ya existe.
  • KEEP: Mantiene el conjunto de datos independientemente del estado de terminación del paso del trabajo.

Modificación de un Conjunto de Datos Existente

//MODDATA DD  DSN=EXISTING.DATASET, DISP=(MOD,CATLG,DELETE)
  • MOD: Indica que el conjunto de datos se va a añadir.
  • CATLG: Cataloga el conjunto de datos al terminar normalmente.
  • DELETE: Elimina el conjunto de datos al terminar anormalmente.

Compartir un Conjunto de Datos Existente

//SHRDATA DD  DSN=EXISTING.DATASET, DISP=SHR
  • SHR: Indica que el conjunto de datos se puede compartir.

Resumen

Entender el parámetro DISP es esencial para gestionar conjuntos de datos de manera efectiva en JCL. Te permite controlar el estado de los conjuntos de datos antes y después de la ejecución del trabajo, asegurando que los datos se manejen adecuadamente según el éxito o fracaso del trabajo. Al dominar el parámetro DISP, puedes crear, usar, modificar y compartir conjuntos de datos de manera eficiente, lo cual es una habilidad fundamental para cualquier programador de JCL.
