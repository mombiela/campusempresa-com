Introducción a la Declaración DD

La declaración DD (Definición de Datos) es un componente crucial en JCL (Lenguaje de Control de Trabajos) utilizado para describir los conjuntos de datos que un trabajo utilizará. Especifica los archivos de entrada y salida, sus atributos y cómo deben manejarse durante la ejecución del trabajo.

Conceptos Clave

  • DDNAME: El nombre utilizado para referirse al conjunto de datos dentro del JCL.
  • DSNAME (DSN): El nombre real del conjunto de datos.
  • DISP: La disposición del conjunto de datos, indicando su estado antes y después de que el trabajo se ejecute.
  • UNIT: El tipo de dispositivo en el que reside el conjunto de datos.
  • SPACE: La cantidad de espacio a asignar para el conjunto de datos.
  • DCB: Parámetros del Bloque de Control de Datos, que definen los atributos del conjunto de datos como el formato y la longitud del registro.

Estructura Básica de una Declaración DD

Una declaración DD simple podría verse así:

//MYDATA  DD  DSN=MY.DATA.SET, DISP=SHR

Explicación

  • //MYDATA: El DDNAME, utilizado para referirse a este conjunto de datos en el JCL.
  • DD: Indica que esta es una declaración DD.
  • DSN=MY.DATA.SET: Especifica el nombre del conjunto de datos.
  • DISP=SHR: Especifica la disposición, en este caso, acceso compartido.

Explicación Detallada de los Parámetros DD

DSNAME (DSN)

El parámetro DSNAME especifica el nombre del conjunto de datos. Puede ser un nombre completamente calificado o un nombre relativo.

//INPUT  DD  DSN=MY.INPUT.DATA, DISP=OLD

DISP (Disposición)

El parámetro DISP tiene tres sub-parámetros:

  • Estado: NEW, OLD, MOD, o SHR.
  • Disposición Normal: Qué sucede con el conjunto de datos si el trabajo se completa exitosamente.
  • Disposición Anormal: Qué sucede con el conjunto de datos si el trabajo falla.
//OUTPUT  DD  DSN=MY.OUTPUT.DATA, DISP=(NEW,CATLG,DELETE)

UNIT

El parámetro UNIT especifica el tipo de dispositivo en el que reside el conjunto de datos.

//TEMP  DD  UNIT=SYSDA, SPACE=(CYL,(5,5))

SPACE

El parámetro SPACE especifica la cantidad de espacio a asignar para un nuevo conjunto de datos.

//TEMP  DD  DSN=MY.TEMP.DATA, DISP=(NEW,DELETE), UNIT=SYSDA, SPACE=(CYL,(5,5))

DCB (Bloque de Control de Datos)

El parámetro DCB define los atributos del conjunto de datos, como el formato del registro (RECFM), la longitud lógica del registro (LRECL) y el tamaño del bloque (BLKSIZE).

//MYFILE  DD  DSN=MY.FILE.DATA, DISP=SHR, DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)

Uso Avanzado de la Declaración DD

Concatenación de Conjuntos de Datos

Se pueden concatenar múltiples conjuntos de datos en una sola declaración DD.

//MULTI  DD  DSN=DATA.SET1, DISP=SHR
//       DD  DSN=DATA.SET2, DISP=SHR

Referencia a Conjuntos de Datos Temporales

Se pueden crear y referenciar conjuntos de datos temporales dentro del mismo trabajo.

//STEP1  DD  DSN=&&TEMP, DISP=(NEW,PASS), UNIT=SYSDA, SPACE=(CYL,(1,1))
//STEP2  DD  DSN=&&TEMP, DISP=(OLD,DELETE)

Uso de SYSOUT

El parámetro SYSOUT dirige la salida a una clase de salida del sistema, típicamente para impresión o registro.

//PRINT  DD  SYSOUT=A

Ejemplos y Ejercicios

Ejemplo 1: Declaración DD Básica

//STEP1  EXEC PGM=MYPROG
//INFILE  DD  DSN=MY.INPUT.DATA, DISP=SHR
//OUTFILE DD  DSN=MY.OUTPUT.DATA, DISP=(NEW,CATLG,DELETE), UNIT=SYSDA, SPACE=(CYL,(10,5))

Ejercicio 1

Escribe una declaración DD para crear un nuevo conjunto de datos USER.TEST.DATA con los siguientes atributos:

  • Disposición: Nuevo, catalogado si es exitoso, eliminado si falla.
  • Unidad: SYSDA
  • Espacio: 5 cilindros primarios, 2 cilindros secundarios.
  • Formato de registro: Bloque fijo (FB)
  • Longitud lógica del registro: 80
  • Tamaño del bloque: 800

Solución

//TESTDATA  DD  DSN=USER.TEST.DATA, DISP=(NEW,CATLG,DELETE), UNIT=SYSDA, SPACE=(CYL,(5,2)), DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)

Conclusión

La declaración DD es una parte fundamental del JCL, permitiéndote definir y gestionar los conjuntos de datos utilizados por tus trabajos. Entender sus parámetros y cómo usarlos efectivamente es crucial para un control eficiente de trabajos y gestión de datos en un entorno de mainframe. Practica escribiendo y modificando declaraciones DD para volverte competente en JCL.
