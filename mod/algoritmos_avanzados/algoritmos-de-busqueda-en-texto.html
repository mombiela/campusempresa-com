Algoritmos de Búsqueda en Texto

Introducción

Los algoritmos de búsqueda en texto son fundamentales en el campo de la informática y tienen aplicaciones en diversas áreas como la recuperación de información, la bioinformática y la compresión de datos. Este tema abordará los algoritmos más importantes y eficientes para buscar patrones en textos.

Algoritmos Clásicos de Búsqueda en Texto

Búsqueda Ingenua

El algoritmo de búsqueda ingenua es el más simple y directo, pero también el menos eficiente. Se basa en comparar el patrón con todas las posibles posiciones en el texto.

  • Ventajas:

    • Fácil de implementar.
    • Funciona bien para textos y patrones pequeños.

  • Desventajas:

    • Ineficiente para textos largos.
    • Complejidad de tiempo: O((n-m+1) * m), donde n es la longitud del texto y m es la longitud del patrón.
def naive_search(text, pattern):
    n = len(text)
    m = len(pattern)
    for i in range(n - m + 1):
        j = 0
        while j < m and text[i + j] == pattern[j]:
            j += 1
        if j == m:
            print(f"Pattern found at index {i}")

text = "ABABDABACDABABCABAB"
pattern = "ABABCABAB"
naive_search(text, pattern)

Algoritmo de Knuth-Morris-Pratt (KMP)

El algoritmo KMP mejora la eficiencia al evitar comparaciones redundantes mediante el uso de un arreglo de prefijos.

  • Ventajas:

    • Más eficiente que la búsqueda ingenua.
    • Complejidad de tiempo: O(n + m).

  • Desventajas:

    • Más complejo de implementar.
def kmp_search(text, pattern):
    def compute_lps(pattern):
        lps = [0] * len(pattern)
        length = 0
        i = 1
        while i < len(pattern):
            if pattern[i] == pattern[length]:
                length += 1
                lps[i] = length
                i += 1
            else:
                if length != 0:
                    length = lps[length - 1]
                else:
                    lps[i] = 0
                    i += 1
        return lps

    n = len(text)
    m = len(pattern)
    lps = compute_lps(pattern)
    i = 0
    j = 0
    while i < n:
        if pattern[j] == text[i]:
            i += 1
            j += 1
        if j == m:
            print(f"Pattern found at index {i - j}")
            j = lps[j - 1]
        elif i < n and pattern[j] != text[i]:
            if j != 0:
                j = lps[j - 1]
            else:
                i += 1

text = "ABABDABACDABABCABAB"
pattern = "ABABCABAB"
kmp_search(text, pattern)

Algoritmo de Boyer-Moore

El algoritmo Boyer-Moore es uno de los más eficientes para la búsqueda de patrones en texto, utilizando dos heurísticas: la heurística de mal carácter y la heurística de buen sufijo.

  • Ventajas:

    • Muy eficiente en la práctica.
    • Complejidad de tiempo promedio: O(n/m).

  • Desventajas:

    • Complejo de implementar.
def boyer_moore_search(text, pattern):
    def bad_char_heuristic(pattern):
        bad_char = [-1] * 256
        for i in range(len(pattern)):
            bad_char[ord(pattern[i])] = i
        return bad_char

    def good_suffix_heuristic(pattern):
        m = len(pattern)
        good_suffix = [0] * (m + 1)
        border_pos = [0] * (m + 1)
        i = m
        j = m + 1
        border_pos[i] = j
        while i > 0:
            while j <= m and pattern[i - 1] != pattern[j - 1]:
                if good_suffix[j] == 0:
                    good_suffix[j] = j - i
                j = border_pos[j]
            i -= 1
            j -= 1
            border_pos[i] = j
        j = border_pos[0]
        for i in range(m + 1):
            if good_suffix[i] == 0:
                good_suffix[i] = j
            if i == j:
                j = border_pos[j]
        return good_suffix

    bad_char = bad_char_heuristic(pattern)
    good_suffix = good_suffix_heuristic(pattern)
    n = len(text)
    m = len(pattern)
    s = 0
    while s <= n - m:
        j = m - 1
        while j >= 0 and pattern[j] == text[s + j]:
            j -= 1
        if j < 0:
            print(f"Pattern found at index {s}")
            s += good_suffix[0]
        else:
            s += max(good_suffix[j + 1], j - bad_char[ord(text[s + j])])

text = "ABABDABACDABABCABAB"
pattern = "ABABCABAB"
boyer_moore_search(text, pattern)

Comparación de Algoritmos

| Algoritmo | Complejidad de Tiempo | Ventajas | Desventajas | |-------------------|-----------------------|-----------------------------------|----------------------------------| | Búsqueda Ingenua | O((n-m+1) * m) | Fácil de implementar | Ineficiente para textos largos | | KMP | O(n + m) | Evita comparaciones redundantes | Más complejo de implementar | | Boyer-Moore | O(n/m) (promedio) | Muy eficiente en la práctica | Complejo de implementar |

Conclusión

Los algoritmos de búsqueda en texto son herramientas esenciales en la informática. La elección del algoritmo adecuado depende del contexto y las necesidades específicas de la aplicación. La búsqueda ingenua es simple pero ineficiente, mientras que KMP y Boyer-Moore ofrecen mejoras significativas en eficiencia a costa de una mayor complejidad de implementación. Con una comprensión sólida de estos algoritmos, los profesionales pueden abordar una amplia gama de problemas de búsqueda en texto de manera efectiva.
