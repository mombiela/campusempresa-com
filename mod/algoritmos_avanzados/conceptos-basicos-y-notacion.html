Introducción

En esta sección, cubriremos los conceptos fundamentales y la notación utilizada en el estudio de algoritmos avanzados. Es crucial comprender estos conceptos básicos para poder abordar temas más complejos de manera efectiva.

Conceptos Clave

  • Algoritmo: Un conjunto de instrucciones bien definidas y finitas que resuelven un problema específico.
  • Eficiencia: Medida de los recursos que un algoritmo consume, principalmente tiempo y espacio.
  • Complejidad Temporal: Tiempo que tarda un algoritmo en ejecutarse, generalmente expresado en función del tamaño de la entrada (n).
  • Complejidad Espacial: Cantidad de memoria que un algoritmo necesita durante su ejecución.
  • Notación Big O: Representa el peor caso de la complejidad temporal de un algoritmo.
  • Notación Big Ω: Representa el mejor caso de la complejidad temporal de un algoritmo.
  • Notación Big Θ: Representa el caso promedio de la complejidad temporal de un algoritmo.

Notación Asintótica

La notación asintótica es una herramienta fundamental para describir la eficiencia de los algoritmos. A continuación, se presentan las notaciones más comunes:

Notación Big O

La notación Big O describe el límite superior del tiempo de ejecución de un algoritmo. Se utiliza para expresar el peor caso.

Ejemplo:

def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

En este caso, la complejidad temporal es O(n), donde n es el tamaño del arreglo arr.

Notación Big Ω

La notación Big Ω describe el límite inferior del tiempo de ejecución de un algoritmo. Se utiliza para expresar el mejor caso.

Ejemplo:

def find_min(arr):
    min_val = arr[0]
    for i in range(1, len(arr)):
        if arr[i] < min_val:
            min_val = arr[i]
    return min_val

En este caso, la complejidad temporal es Ω(n), ya que en el mejor caso, el algoritmo aún debe revisar todos los elementos del arreglo.

Notación Big Θ

La notación Big Θ describe tanto el límite superior como el límite inferior del tiempo de ejecución de un algoritmo. Se utiliza para expresar el caso promedio.

Ejemplo:

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

En este caso, la complejidad temporal es Θ(log n), donde n es el tamaño del arreglo arr.

Comparación de Notaciones

| Notación | Descripción | Ejemplo de Complejidad | |----------|-------------|------------------------| | O(f(n)) | Peor caso | O(n^2) | | Ω(f(n)) | Mejor caso | Ω(n) | | Θ(f(n)) | Caso promedio | Θ(log n) |

Ejemplos Prácticos

Ejemplo 1: Ordenamiento de Burbuja

El algoritmo de ordenamiento de burbuja tiene una complejidad temporal de O(n^2) en el peor caso.

def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr

Ejemplo 2: Ordenamiento Rápido

El algoritmo de ordenamiento rápido tiene una complejidad temporal de O(n log n) en el mejor caso y O(n^2) en el peor caso.

def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort(left) + middle + quick_sort(right)

Conclusión

Comprender los conceptos básicos y la notación es esencial para el estudio de algoritmos avanzados. La notación asintótica nos permite analizar y comparar la eficiencia de diferentes algoritmos de manera formal. A medida que avancemos en el curso, estos conceptos serán fundamentales para entender y diseñar algoritmos más complejos.
