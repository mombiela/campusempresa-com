Introducción

En esta sección del curso sobre Algoritmos Avanzados, exploraremos dos técnicas fundamentales para resolver problemas complejos: la recursión y la programación dinámica. Ambas técnicas son esenciales para diseñar algoritmos eficientes y son ampliamente utilizadas en diversas áreas de la informática.

Recursión

La recursión es una técnica en la que una función se llama a sí misma para resolver subproblemas más pequeños del problema original. Es especialmente útil para problemas que pueden dividirse en subproblemas similares.

Conceptos Clave

  • Caso Base: La condición que detiene la recursión.
  • Caso Recursivo: La parte de la función que incluye la llamada recursiva.
  • Pila de Llamadas: La estructura de datos que maneja las llamadas recursivas.

Ejemplo de Código: Factorial

def factorial(n):
    if n == 0:
        return 1  # Caso Base
    else:
        return n * factorial(n - 1)  # Caso Recursivo

# Ejemplo de uso
print(factorial(5))  # Salida: 120

Explicación del Ejemplo

  • Caso Base: Si n es 0, la función retorna 1.
  • Caso Recursivo: La función se llama a sí misma con n - 1 hasta que alcanza el caso base.

Programación Dinámica

La programación dinámica es una técnica de optimización que resuelve problemas almacenando los resultados de subproblemas ya resueltos para evitar cálculos redundantes. Se utiliza principalmente para problemas de optimización y de toma de decisiones.

Conceptos Clave

  • Memoización: Almacenamiento de resultados de subproblemas en una estructura de datos (como un diccionario) para reutilizarlos.
  • Tabulación: Construcción de una tabla (normalmente un array) de abajo hacia arriba para almacenar los resultados de subproblemas.

Ejemplo de Código: Fibonacci con Memoización

def fibonacci(n, memo={}):
    if n in memo:
        return memo[n]  # Retorna el resultado almacenado
    if n <= 1:
        return n  # Caso Base
    memo[n] = fibonacci(n - 1, memo) + fibonacci(n - 2, memo)  # Caso Recursivo
    return memo[n]

# Ejemplo de uso
print(fibonacci(10))  # Salida: 55

Ejemplo de Código: Fibonacci con Tabulación

def fibonacci_tab(n):
    if n <= 1:
        return n
    fib = [0] * (n + 1)
    fib[1] = 1
    for i in range(2, n + 1):
        fib[i] = fib[i - 1] + fib[i - 2]
    return fib[n]

# Ejemplo de uso
print(fibonacci_tab(10))  # Salida: 55

Comparación: Recursión vs Programación Dinámica

| Aspecto | Recursión | Programación Dinámica | |------------------------|---------------------------------|----------------------------------| | Eficiencia | Puede ser ineficiente | Más eficiente debido a la reutilización de subproblemas | | Facilidad de Implementación | Suele ser más intuitiva y fácil de implementar | Puede ser más compleja debido a la necesidad de almacenamiento de subproblemas | | Uso de Memoria | Puede consumir mucha memoria debido a la pila de llamadas | Consume memoria adicional para almacenar resultados de subproblemas |

Ejercicios

  1. Implementar el Algoritmo de Euclides: Escribe una función recursiva para encontrar el máximo común divisor (MCD) de dos números.
  2. Problema de la Mochila (Knapsack Problem): Implementa una solución utilizando programación dinámica para el problema de la mochila 0/1.
  3. Secuencia de Longest Common Subsequence (LCS): Escribe una función que utilice programación dinámica para encontrar la longitud de la subsecuencia común más larga entre dos cadenas.

Conclusión

La recursión y la programación dinámica son técnicas poderosas para resolver problemas complejos de manera eficiente. La recursión es intuitiva y fácil de implementar, pero puede ser ineficiente para problemas grandes. La programación dinámica, aunque más compleja, optimiza el rendimiento al evitar cálculos redundantes. Comprender cuándo y cómo utilizar estas técnicas es crucial para diseñar algoritmos avanzados y eficientes.
