Introducción

COBOL (Common Business-Oriented Language) es uno de los lenguajes de programación de alto nivel más antiguos, utilizado principalmente en sistemas de negocios, finanzas y administración para empresas y gobiernos. Comprender la estructura de un programa COBOL es fundamental para escribir código eficiente y sin errores. Esta sección te guiará a través de la estructura básica de un programa COBOL, incluyendo sus divisiones, secciones y párrafos.

Estructura Básica de un Programa COBOL

Un programa COBOL se divide en cuatro divisiones principales:

  1. División de Identificación
  2. División de Entorno
  3. División de Datos
  4. División de Procedimientos

Cada división tiene un propósito específico y contiene varias secciones y párrafos.

División de Identificación

La División de Identificación es la primera división en un programa COBOL. Proporciona metadatos sobre el programa.

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.
AUTHOR. John Doe.
  • PROGRAM-ID: Especifica el nombre del programa.
  • AUTHOR: Indica el autor del programa.

División de Entorno

La División de Entorno especifica el entorno en el que se ejecutará el programa. Contiene dos secciones: Sección de Configuración y Sección de Entrada-Salida.

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
SOURCE-COMPUTER. IBM-370.
OBJECT-COMPUTER. IBM-370.

INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT INPUT-FILE ASSIGN TO 'input.txt'.
    SELECT OUTPUT-FILE ASSIGN TO 'output.txt'.
  • SOURCE-COMPUTER: Especifica la computadora en la que se compila el programa fuente.
  • OBJECT-COMPUTER: Especifica la computadora en la que se ejecutará el programa.
  • FILE-CONTROL: Gestiona las asignaciones de archivos.

División de Datos

La División de Datos se utiliza para definir las variables y estructuras de datos. Contiene varias secciones, pero las más comúnmente usadas son la Sección de Archivos y la Sección de Almacenamiento de Trabajo.

DATA DIVISION.
FILE SECTION.
FD  INPUT-FILE.
01  INPUT-RECORD PIC X(100).

FD  OUTPUT-FILE.
01  OUTPUT-RECORD PIC X(100).

WORKING-STORAGE SECTION.
01  WS-NAME PIC X(20).
01  WS-AGE  PIC 99.
  • FILE SECTION: Define la estructura de los archivos.
  • WORKING-STORAGE SECTION: Define variables y constantes utilizadas en el programa.

División de Procedimientos

La División de Procedimientos contiene la lógica del programa. Consiste en una serie de declaraciones que manipulan los datos definidos en la División de Datos.

PROCEDURE DIVISION.
MAIN-PARA.
    OPEN INPUT INPUT-FILE.
    OPEN OUTPUT OUTPUT-FILE.
    PERFORM READ-INPUT UNTIL END-OF-FILE.
    CLOSE INPUT-FILE.
    CLOSE OUTPUT-FILE.
    STOP RUN.

READ-INPUT.
    READ INPUT-FILE INTO INPUT-RECORD
        AT END
            MOVE 'YES' TO END-OF-FILE
        NOT AT END
            MOVE INPUT-RECORD TO OUTPUT-RECORD
            WRITE OUTPUT-RECORD.
  • MAIN-PARA: El párrafo principal donde comienza la ejecución.
  • READ-INPUT: Un párrafo que lee registros del archivo de entrada y los escribe en el archivo de salida.

Conclusión

Comprender la estructura de un programa COBOL es esencial para escribir código efectivo y mantenible. Las cuatro divisiones principales—Identificación, Entorno, Datos y Procedimientos—cada una tiene un propósito específico y deben incluirse en cada programa COBOL. Al dominar estas divisiones y sus respectivas secciones, estarás bien equipado para abordar tareas de programación COBOL más complejas.
