División de Procedimientos
Declaraciones Condicionales ►

La División de Procedimientos en COBOL es donde ocurre el procesamiento real de los datos. Esta es la sección del programa donde escribes las instrucciones que le dicen a la computadora qué hacer con los datos definidos en la División de Datos. Entender la División de Procedimientos es crucial para escribir programas COBOL efectivos.

Conceptos Clave

  • Párrafos y Secciones: La División de Procedimientos está organizada en párrafos y secciones. Una sección es una colección de párrafos, y un párrafo es una colección de sentencias.
  • Sentencias: Estas son las instrucciones individuales que realizan operaciones como mover datos, realizar cálculos y controlar el flujo del programa.
  • Flujo de Control: COBOL proporciona varios constructos para controlar el flujo del programa, incluyendo sentencias IF, bucles PERFORM y sentencias GO TO.

Estructura de la División de Procedimientos

La División de Procedimientos comienza con el encabezado PROCEDURE DIVISION y es seguida por secciones, párrafos y sentencias.

Estructura Básica

PROCEDURE DIVISION.
    SECTION-1.
    PARAGRAPH-1.
        DISPLAY 'Hello, World!'.
        STOP RUN.

Secciones y Párrafos

  • Secciones: Agrupan párrafos relacionados.
  • Párrafos: Contienen las sentencias ejecutables.
PROCEDURE DIVISION.
    INITIALIZE-SECTION.
    INITIALIZE-PARAGRAPH.
        DISPLAY 'Initializing Program'.
    
    PROCESS-SECTION.
    PROCESS-PARAGRAPH.
        DISPLAY 'Processing Data'.
    
    TERMINATE-SECTION.
    TERMINATE-PARAGRAPH.
        DISPLAY 'Terminating Program'.
        STOP RUN.

Sentencias Comunes

Sentencia DISPLAY

La sentencia DISPLAY se utiliza para mostrar texto en la consola.

PROCEDURE DIVISION.
    DISPLAY 'Hello, World!'.
    STOP RUN.

Sentencia MOVE

La sentencia MOVE asigna valores a variables.

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-NAME PIC X(20).

PROCEDURE DIVISION.
    MOVE 'John Doe' TO WS-NAME.
    DISPLAY WS-NAME.
    STOP RUN.

Sentencia IF

La sentencia IF se utiliza para el procesamiento condicional.

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-AGE PIC 99.

PROCEDURE DIVISION.
    MOVE 25 TO WS-AGE.
    IF WS-AGE > 18
        DISPLAY 'Adult'
    ELSE
        DISPLAY 'Minor'.
    STOP RUN.

Sentencia PERFORM

La sentencia PERFORM se utiliza para ejecutar un párrafo o sección.

PROCEDURE DIVISION.
    PERFORM INITIALIZE-PARAGRAPH.
    PERFORM PROCESS-PARAGRAPH.
    PERFORM TERMINATE-PARAGRAPH.
    STOP RUN.

INITIALIZE-PARAGRAPH.
    DISPLAY 'Initializing Program'.

PROCESS-PARAGRAPH.
    DISPLAY 'Processing Data'.

TERMINATE-PARAGRAPH.
    DISPLAY 'Terminating Program'.

Sentencia GO TO

La sentencia GO TO transfiere el control a otra parte del programa.

PROCEDURE DIVISION.
    DISPLAY 'Start'.
    GO TO END-PARAGRAPH.
    DISPLAY 'This will not be displayed'.
    
END-PARAGRAPH.
    DISPLAY 'End'.
    STOP RUN.

Constructos de Control de Flujo

Sentencias IF Anidadas

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-SCORE PIC 99.

PROCEDURE DIVISION.
    MOVE 85 TO WS-SCORE.
    IF WS-SCORE >= 90
        DISPLAY 'Grade: A'
    ELSE
        IF WS-SCORE >= 80
            DISPLAY 'Grade: B'
        ELSE
            DISPLAY 'Grade: C'.
    STOP RUN.

Bucle PERFORM UNTIL

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-COUNTER PIC 99 VALUE 1.

PROCEDURE DIVISION.
    PERFORM UNTIL WS-COUNTER > 5
        DISPLAY 'Counter: ' WS-COUNTER
        ADD 1 TO WS-COUNTER
    END-PERFORM.
    STOP RUN.

Conclusión

La División de Procedimientos es el corazón de un programa COBOL, donde ocurre la lógica y el procesamiento de datos real. Al entender cómo estructurar secciones, párrafos y sentencias, y al dominar los constructos de control de flujo como las sentencias IF y los bucles PERFORM, puedes escribir programas COBOL eficientes y efectivos. Practica escribiendo diferentes tipos de sentencias y estructuras de control para volverte competente en la programación COBOL.
