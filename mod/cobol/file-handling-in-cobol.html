Manejo de Archivos en COBOL
Procesamiento de Archivos Secuenciales ►

El manejo de archivos es un aspecto crucial de la programación en COBOL, ya que a menudo se utiliza para aplicaciones comerciales que requieren un procesamiento de datos extenso. Este tema te guiará a través de los conceptos básicos del manejo de archivos en COBOL, desde la comprensión de los tipos de archivos hasta las operaciones avanzadas con archivos.

Introducción al Manejo de Archivos

En COBOL, los archivos se utilizan para almacenar datos que pueden ser leídos y escritos por el programa. Comprender cómo manejar archivos es esencial para cualquier programador de COBOL.

Conceptos Clave

  • Tipos de Archivos: Secuencial, Indexado y Relativo.
  • Organización de Archivos: Cómo se estructuran los datos dentro del archivo.
  • Modos de Acceso a Archivos: Entrada, Salida, E/S y Extender.
  • Control de Archivos: Uso de las sentencias SELECT y ASSIGN.
  • Descripción de Archivos: Uso de la entrada FD (Descripción de Archivos) en la División de Datos.

Tipos de Archivos

COBOL soporta tres tipos principales de archivos:

  1. Archivos Secuenciales: Los registros se almacenan uno tras otro.
  2. Archivos Indexados: Los registros se almacenan con un índice, permitiendo un acceso más rápido.
  3. Archivos Relativos: Los registros se almacenan en posiciones relativas, permitiendo un acceso directo.

Archivos Secuenciales

Los archivos secuenciales son el tipo más simple de archivo en COBOL. Los registros se leen y escriben de manera secuencial.

Ejemplo: Leyendo un Archivo Secuencial

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadSeqFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD InputFile.
01 InputRecord PIC X(80).

WORKING-STORAGE SECTION.
01 WS-EOF PIC X VALUE 'N'.

PROCEDURE DIVISION.
    OPEN INPUT InputFile
    PERFORM UNTIL WS-EOF = 'Y'
        READ InputFile INTO InputRecord
            AT END
                MOVE 'Y' TO WS-EOF
            NOT AT END
                DISPLAY InputRecord
        END-READ
    END-PERFORM
    CLOSE InputFile
    STOP RUN.

Archivos Indexados

Los archivos indexados permiten un acceso más rápido a los registros utilizando un índice.

Ejemplo: Leyendo un Archivo Indexado

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadIdxFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT IndexedFile ASSIGN TO 'indexed.dat'
    ORGANIZATION IS INDEXED
    ACCESS MODE IS RANDOM
    RECORD KEY IS KeyField.

DATA DIVISION.
FILE SECTION.
FD IndexedFile.
01 IndexedRecord.
   05 KeyField PIC X(10).
   05 DataField PIC X(70).

WORKING-STORAGE SECTION.
01 WS-Key PIC X(10).

PROCEDURE DIVISION.
    OPEN I-O IndexedFile
    MOVE 'somekey' TO WS-Key
    READ IndexedFile
        INVALID KEY
            DISPLAY 'Record not found'
        NOT INVALID KEY
            DISPLAY IndexedRecord
    END-READ
    CLOSE IndexedFile
    STOP RUN.

Archivos Relativos

Los archivos relativos almacenan registros en posiciones relativas específicas.

Ejemplo: Leyendo un Archivo Relativo

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadRelFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT RelativeFile ASSIGN TO 'relative.dat'
    ORGANIZATION IS RELATIVE
    ACCESS MODE IS RANDOM
    RELATIVE KEY IS RelKey.

DATA DIVISION.
FILE SECTION.
FD RelativeFile.
01 RelativeRecord.
   05 RelKey PIC 9(4).
   05 DataField PIC X(76).

WORKING-STORAGE SECTION.
01 WS-RelKey PIC 9(4).

PROCEDURE DIVISION.
    OPEN I-O RelativeFile
    MOVE 1234 TO WS-RelKey
    READ RelativeFile
        INVALID KEY
            DISPLAY 'Record not found'
        NOT INVALID KEY
            DISPLAY RelativeRecord
    END-READ
    CLOSE RelativeFile
    STOP RUN.

Modos de Acceso a Archivos

COBOL proporciona diferentes modos para acceder a los archivos:

  • INPUT: Acceso de solo lectura.
  • OUTPUT: Acceso de solo escritura, crea un nuevo archivo.
  • I-O: Acceso de lectura y escritura.
  • EXTEND: Añadir registros al final del archivo.

Ejemplo: Escribiendo en un Archivo Secuencial

IDENTIFICATION DIVISION.
PROGRAM-ID. WriteSeqFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT OutputFile ASSIGN TO 'output.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD OutputFile.
01 OutputRecord PIC X(80).

WORKING-STORAGE SECTION.
01 WS-Data PIC X(80) VALUE 'This is a test record'.

PROCEDURE DIVISION.
    OPEN OUTPUT OutputFile
    MOVE WS-Data TO OutputRecord
    WRITE OutputRecord
    CLOSE OutputFile
    STOP RUN.

Control y Descripción de Archivos

Control de Archivos

Las sentencias SELECT y ASSIGN en la sección de Control de Archivos de la División de Entorno se utilizan para asociar un archivo con un medio de almacenamiento físico.

Descripción de Archivos

La entrada FD (Descripción de Archivos) en la División de Datos describe la estructura de los registros del archivo.

Ejemplo: Control y Descripción de Archivos

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT MyFile ASSIGN TO 'myfile.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD MyFile.
01 MyRecord.
   05 Field1 PIC X(10).
   05 Field2 PIC 9(5).

Conclusión

El manejo de archivos en COBOL es una habilidad fundamental para cualquier programador de COBOL. Al comprender los diferentes tipos de archivos, modos de acceso y cómo controlar y describir archivos, puedes gestionar eficientemente los datos dentro de tus programas COBOL. Practica con los ejemplos proporcionados y experimenta con diferentes operaciones de archivos para profundizar tu comprensión.
