Subrutinas y Funciones en COBOL

Introducción

Las subrutinas y funciones son componentes esenciales en la programación COBOL, permitiendo un código modular, reutilizable y organizado. Esta sección cubrirá los conceptos básicos de subrutinas y funciones, su sintaxis, uso y mejores prácticas desde niveles principiantes hasta avanzados.

Conceptos Clave

  • Subrutinas: También conocidas como párrafos o secciones, son bloques de código que realizan tareas específicas y pueden ser llamados desde diferentes partes del programa.
  • Funciones: Operaciones predefinidas o definidas por el usuario que devuelven un valor y pueden ser usadas dentro de expresiones.

Subrutinas en COBOL

Estructura Básica

Las subrutinas en COBOL se definen en la PROCEDURE DIVISION. Pueden organizarse en párrafos o secciones.

Párrafos

Un párrafo es un bloque de código nombrado que puede ser ejecutado usando la instrucción PERFORM.

IDENTIFICATION DIVISION.
PROGRAM-ID. SubroutineExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM1 PIC 9(2) VALUE 10.
01 NUM2 PIC 9(2) VALUE 20.
01 RESULT PIC 9(3).

PROCEDURE DIVISION.
    PERFORM ADD-NUMBERS
    DISPLAY "El resultado es: " RESULT
    STOP RUN.

ADD-NUMBERS.
    ADD NUM1 TO NUM2 GIVING RESULT.

Secciones

Las secciones son bloques más grandes que pueden contener múltiples párrafos.

IDENTIFICATION DIVISION.
PROGRAM-ID. SectionExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM1 PIC 9(2) VALUE 10.
01 NUM2 PIC 9(2) VALUE 20.
01 RESULT PIC 9(3).

PROCEDURE DIVISION.
    PERFORM CALCULATION-SECTION
    DISPLAY "El resultado es: " RESULT
    STOP RUN.

CALCULATION-SECTION.
ADD-NUMBERS.
    ADD NUM1 TO NUM2 GIVING RESULT.

Llamando Subrutinas

  • PERFORM: Ejecuta un párrafo o sección.
  • THRU: Ejecuta un rango de párrafos.
PERFORM ADD-NUMBERS
PERFORM CALCULATION-SECTION THRU END-CALCULATION

Funciones en COBOL

Funciones Intrínsecas

COBOL proporciona un conjunto de funciones integradas para operaciones comunes.

Ejemplo: Función NUMVAL

Convierte una cadena numérica a un valor numérico.

IDENTIFICATION DIVISION.
PROGRAM-ID. FunctionExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM-STRING PIC X(5) VALUE '12345'.
01 NUM-VALUE PIC 9(5).

PROCEDURE DIVISION.
    COMPUTE NUM-VALUE = FUNCTION NUMVAL(NUM-STRING)
    DISPLAY "El valor numérico es: " NUM-VALUE
    STOP RUN.

Funciones Definidas por el Usuario

Las funciones definidas por el usuario pueden ser creadas usando las instrucciones FUNCTION-ID y END FUNCTION.

IDENTIFICATION DIVISION.
FUNCTION-ID. MyFunction.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 INPUT-NUMBER PIC 9(2).
01 RESULT PIC 9(3).

PROCEDURE DIVISION.
    ENTRY "MyFunction" USING INPUT-NUMBER
    ADD INPUT-NUMBER TO INPUT-NUMBER GIVING RESULT
    EXIT FUNCTION RESULT.
END FUNCTION MyFunction.

Llamando Funciones Definidas por el Usuario

Las funciones definidas por el usuario pueden ser llamadas usando la palabra clave FUNCTION.

IDENTIFICATION DIVISION.
PROGRAM-ID. CallFunctionExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 INPUT-NUMBER PIC 9(2) VALUE 10.
01 RESULT PIC 9(3).

PROCEDURE DIVISION.
    COMPUTE RESULT = FUNCTION MyFunction(INPUT-NUMBER)
    DISPLAY "El resultado es: " RESULT
    STOP RUN.

Mejores Prácticas

  • Modularidad: Descomponer tareas complejas en subrutinas o funciones más pequeñas.
  • Reutilización: Escribir subrutinas y funciones que puedan ser reutilizadas en diferentes programas.
  • Claridad: Usar nombres significativos para subrutinas y funciones para mejorar la legibilidad del código.
  • Documentación: Comentar el código para explicar el propósito y uso de subrutinas y funciones.

Resumen

Las subrutinas y funciones son herramientas poderosas en COBOL que ayudan a crear código organizado, modular y reutilizable. Entender su estructura, uso y mejores prácticas es crucial para una programación eficiente en COBOL. Al dominar estos conceptos, puedes escribir programas COBOL más mantenibles y escalables.
