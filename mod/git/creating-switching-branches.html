Introducción

Las ramas en Git te permiten divergir de la línea principal de desarrollo y continuar trabajando sin afectar la base de código principal. Esto es particularmente útil para desarrollar nuevas características, corregir errores o experimentar con nuevas ideas.

Conceptos Clave

  • Rama: Un puntero a un commit específico en el repositorio.
  • Rama Principal: La rama predeterminada en un repositorio Git, típicamente llamada main o master.
  • Rama de Característica: Una rama creada para desarrollar una nueva característica.
  • Cambio de Ramas: Moverse de una rama a otra para trabajar en diferentes partes del proyecto.

Creación de una Rama

Para crear una nueva rama, usa el comando git branch seguido del nombre de la nueva rama.

Ejemplo

git branch feature-branch

Este comando crea una nueva rama llamada feature-branch pero no cambia a ella.

Explicación

  • git branch: El comando para crear una nueva rama.
  • feature-branch: El nombre de la nueva rama.

Cambio de Ramas

Para cambiar a una rama diferente, usa el comando git checkout seguido del nombre de la rama a la que quieres cambiar.

Ejemplo

git checkout feature-branch

Este comando cambia tu directorio de trabajo a la feature-branch.

Explicación

  • git checkout: El comando para cambiar de ramas.
  • feature-branch: El nombre de la rama a la que quieres cambiar.

Crear y Cambiar en un Solo Paso

Puedes crear y cambiar a una nueva rama en un solo paso usando la bandera -b con el comando git checkout.

Ejemplo

git checkout -b new-feature

Este comando crea una nueva rama llamada new-feature y cambia a ella inmediatamente.

Explicación

  • -b: Una bandera que le dice a Git que cree una nueva rama y cambie a ella.
  • new-feature: El nombre de la nueva rama.

Ver Ramas

Para ver una lista de todas las ramas en tu repositorio, usa el comando git branch sin argumentos.

Ejemplo

git branch

Este comando lista todas las ramas y resalta la rama actual con un asterisco (*).

Explicación

  • git branch: El comando para listar todas las ramas.

Eliminar una Rama

Una vez que una rama ya no es necesaria, puedes eliminarla usando el comando git branch -d.

Ejemplo

git branch -d feature-branch

Este comando elimina la feature-branch.

Explicación

  • -d: Una bandera que le dice a Git que elimine la rama especificada.
  • feature-branch: El nombre de la rama a eliminar.

Resumen

  • Creación de una Rama: Usa git branch <branch-name>.
  • Cambio de Ramas: Usa git checkout <branch-name>.
  • Crear y Cambiar en un Solo Paso: Usa git checkout -b <branch-name>.
  • Ver Ramas: Usa git branch.
  • Eliminar una Rama: Usa git branch -d <branch-name>.

Entender cómo crear y cambiar ramas en Git es fundamental para gestionar diferentes líneas de desarrollo en tus proyectos. Practica estos comandos para volverte competente en el manejo de ramas en Git.
