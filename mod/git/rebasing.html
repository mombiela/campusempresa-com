Rebasear
Cherry-Picking ►

Rebasear es una característica poderosa en Git que te permite reescribir el historial de commits. Puede ser utilizada para simplificar una serie de commits, actualizar ramas y mantener un historial de proyecto más limpio. Este tema cubrirá los conceptos básicos del rebaseo, cómo realizar un rebase y técnicas avanzadas de rebaseo.

¿Qué es el Rebaseo?

El rebaseo es el proceso de mover o combinar una secuencia de commits a un nuevo commit base. Es una alternativa a la fusión y puede ser utilizada para mantener un historial de proyecto lineal.

Conceptos Clave

  • Commit Base: El commit en el que una rama se desvía de otra rama.
  • Rama Upstream: La rama sobre la cual estás rebaseando tus cambios.
  • Rebase vs. Merge: Mientras que la fusión crea un nuevo commit que combina los cambios de dos ramas, el rebaseo mueve toda la rama para comenzar desde un nuevo commit base.

Rebaseo Básico

Cómo Rebasear

Para rebasear una rama, utilizas el comando git rebase. Aquí tienes un ejemplo simple:

# Cambiar a la rama que quieres rebasear
git checkout feature-branch

# Rebasear sobre la rama principal
git rebase main

Ejemplo Explicado

  1. Cambiar a la rama: Primero cambias a la rama que quieres rebasear.
  2. Rebasear sobre la rama principal: Este comando moverá los commits de feature-branch para comenzar desde el último commit en la rama main.

Manejo de Conflictos

Durante un rebaseo, podrías encontrar conflictos. Git pausará el proceso de rebaseo y te permitirá resolver estos conflictos.

# Resolver conflictos en tu editor
# Después de resolver los conflictos, añade los cambios
git add .

# Continuar con el proceso de rebaseo
git rebase --continue

Si deseas abortar el proceso de rebaseo, puedes usar:

git rebase --abort

Rebaseo Interactivo

El rebaseo interactivo te permite editar, reordenar, combinar o eliminar commits. Esto es útil para limpiar tu historial de commits antes de fusionarlo en la rama principal.

Cómo Realizar un Rebaseo Interactivo

# Iniciar un rebaseo interactivo
git rebase -i HEAD~n

Reemplaza n con el número de commits que quieres rebasear. Esto abrirá un editor con una lista de commits.

Ejemplo Explicado

  1. Pick: Usa esto para mantener un commit tal como está.
  2. Reword: Usa esto para cambiar el mensaje del commit.
  3. Edit: Usa esto para detener y editar el commit.
  4. Squash: Usa esto para combinar este commit con el anterior.
  5. Fixup: Similar a squash, pero descarta el mensaje del commit.
  6. Drop: Usa esto para eliminar el commit.

Ejemplo

pick 1234567 Mensaje del commit 1
squash 89abcde Mensaje del commit 2
reword fghijkl Mensaje del commit 3

En este ejemplo:

  • El segundo commit será combinado con el primero.
  • El mensaje del tercer commit será editado.

Técnicas Avanzadas de Rebaseo

Rebasear Sobre Otra Rama

Puedes rebasear tu rama sobre cualquier otra rama, no solo la rama principal.

git rebase another-branch

Usando `--onto`

La opción --onto te permite rebasear un rango de commits sobre otra rama.

# Rebasear commits de feature-branch sobre another-branch
git rebase --onto another-branch main feature-branch

Ejemplo Explicado

  1. main: La rama desde la cual se toman los commits.
  2. feature-branch: La rama que contiene los commits a ser rebaseados.
  3. another-branch: La rama sobre la cual los commits serán rebaseados.

Resumen

Rebasear es una herramienta poderosa en Git que ayuda a mantener un historial de proyecto limpio y lineal. Puede ser utilizada para:

  • Mover una rama a un nuevo commit base.
  • Limpiar el historial de commits con rebaseo interactivo.
  • Resolver conflictos de manera controlada.

Aunque el rebaseo puede ser complejo, entender sus conceptos básicos y practicar con ejemplos simples te ayudará a dominar esta característica esencial de Git. Recuerda siempre comunicarte con tu equipo cuando realices rebaseos en ramas compartidas para evitar interrumpir el flujo de trabajo.
