Introducción a las Ramas

Las ramas en Git son una característica poderosa que te permite divergir de la línea principal de desarrollo y continuar trabajando sin afectar esa línea principal. Esto es particularmente útil para trabajar en nuevas características, correcciones de errores o experimentos en aislamiento.

Conceptos Clave

  • Rama: Una rama en Git es simplemente un puntero ligero y movible a uno de estos commits.
  • Master/Main: La rama predeterminada en la mayoría de los repositorios. A menudo es la línea principal de desarrollo.
  • HEAD: Un puntero que representa tu directorio de trabajo actual. Usualmente apunta al último commit en tu rama actual.

Creando y Gestionando Ramas

Creando una Rama

Para crear una nueva rama, usas el comando git branch seguido del nombre de la rama que quieres crear.

git branch nueva-caracteristica

Este comando crea una nueva rama llamada nueva-caracteristica pero no cambia a ella. Para cambiar a la nueva rama, usas el comando git checkout:

git checkout nueva-caracteristica

Alternativamente, puedes crear y cambiar a una nueva rama en un solo comando:

git checkout -b nueva-caracteristica

Listando Ramas

Para listar todas las ramas en tu repositorio, usa el comando git branch sin ningún argumento:

git branch

Esto mostrará todas las ramas y resaltará la rama actual con un asterisco (*).

Eliminando una Rama

Una vez que has terminado de trabajar en una rama, podrías querer eliminarla. Para eliminar una rama, usa la opción -d con el comando git branch:

git branch -d nueva-caracteristica

Si la rama no ha sido fusionada, Git te impedirá eliminarla. Para forzar la eliminación de una rama, usa la opción -D:

git branch -D nueva-caracteristica

Fusionando Ramas

Fusión Fast-Forward

Una fusión fast-forward ocurre cuando hay un camino directo desde la rama actual a la rama objetivo. Git simplemente mueve el puntero hacia adelante.

git checkout main
git merge nueva-caracteristica

Fusión de Tres Vías

Cuando las ramas han divergido, Git realiza una fusión de tres vías. Esto implica crear un nuevo commit que combine los cambios de ambas ramas.

git checkout main
git merge nueva-caracteristica

Si hay conflictos, Git te pedirá que los resuelvas manualmente.

Estrategias de Ramas

Flujo de Trabajo de Ramas de Características

  • Crea una nueva rama para cada característica o corrección de errores.
  • Fusiona la rama de características de nuevo en la rama principal una vez que el trabajo esté completo.

Flujo de Trabajo Gitflow

  • Usa múltiples ramas para diferentes etapas de desarrollo (por ejemplo, característica, desarrollo, lanzamiento, corrección rápida).
  • Proporciona un marco robusto para gestionar proyectos más grandes.

Flujo de Trabajo de Forking

  • Cada desarrollador hace un fork del repositorio principal y trabaja en su propia copia.
  • Los cambios se integran de nuevo en el repositorio principal a través de pull requests.

Ejercicios Prácticos

Ejercicio 1: Creando y Cambiando Ramas

  1. Crea una nueva rama llamada experimento.
  2. Cambia a la rama experimento.
  3. Haz algunos cambios y haz commit de ellos.
  4. Cambia de nuevo a la rama main.

Ejercicio 2: Fusionando Ramas

  1. Crea una nueva rama llamada caracteristica-1.
  2. Haz algunos cambios y haz commit de ellos.
  3. Cambia de nuevo a la rama main.
  4. Fusiona caracteristica-1 en main.

Ejercicio 3: Resolviendo Conflictos de Fusión

  1. Crea una nueva rama llamada rama-conflicto.
  2. Haz cambios conflictivos en main y rama-conflicto.
  3. Intenta fusionar rama-conflicto en main y resuelve los conflictos.

Resumen

Entender las ramas en Git es crucial para un control de versiones efectivo. Las ramas te permiten trabajar en diferentes características, correcciones de errores o experimentos en aislamiento. Puedes crear, cambiar, listar y eliminar ramas fácilmente. Fusionar ramas puede ser sencillo o complejo, dependiendo de los cambios realizados. Al dominar las estrategias y flujos de trabajo de ramas, puedes gestionar tus proyectos de manera más eficiente y colaborar con otros sin problemas.
