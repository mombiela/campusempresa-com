Configuración del Repositorio del Proyecto
Implementando Funcionalidades y Colaboración ►

Configurar un repositorio de proyecto es una habilidad fundamental para cualquier desarrollador. Esta sección te guiará a través del proceso de inicializar un repositorio Git, agregar archivos y hacer tu primer commit. Al final de esta sección, tendrás una comprensión sólida de cómo configurar y gestionar un repositorio Git.

Inicializando un Repositorio Git

Para empezar a usar Git, necesitas inicializar un repositorio en tu directorio de proyecto. Este proceso crea un directorio oculto .git que rastrea todos los cambios en tus archivos.

Pasos para Inicializar un Repositorio

  1. Navega a tu directorio de proyecto: 
    cd path/to/your/project
  2. Inicializa el repositorio: 
    git init
    Este comando configura los archivos y directorios necesarios para que Git comience a rastrear tu proyecto.

Agregando Archivos al Repositorio

Una vez que tu repositorio está inicializado, necesitas agregar archivos a él. Este proceso implica preparar los archivos y luego hacer commit de ellos al repositorio.

Preparando Archivos

Preparar archivos significa prepararlos para un commit. Puedes preparar archivos individualmente o todos a la vez.

  • Prepara un solo archivo: 
    git add filename
  • Prepara todos los archivos: 
    git add .

Haciendo Commit de Archivos

Después de preparar los archivos, el siguiente paso es hacer commit de ellos. Un commit es una instantánea de tu proyecto en un punto específico en el tiempo.

  • Haz commit de los archivos preparados: 
    git commit -m "Initial commit"

Creando un Archivo .gitignore

Un archivo .gitignore especifica qué archivos y directorios Git debe ignorar. Esto es útil para excluir archivos temporales, artefactos de compilación y otros archivos que no necesitan ser rastreados.

Ejemplo de Archivo .gitignore

# Ignorar el directorio node_modules
node_modules/

# Ignorar archivos de registro
*.log

# Ignorar artefactos de compilación
dist/

Para crear un archivo .gitignore:

  1. Crea el archivo: 
    touch .gitignore
  2. Edita el archivo: 
    nano .gitignore
  3. Agrega patrones para ignorar (como se muestra en el ejemplo anterior).

Viendo el Estado del Repositorio

Para ver el estado actual de tu repositorio, usa el comando git status. Este comando muestra qué archivos están preparados, no preparados y no rastreados.

  • Verifica el estado del repositorio: 
    git status

Resumen

En esta sección, aprendiste cómo configurar un repositorio Git desde cero. Ahora sabes cómo:

  • Inicializar un repositorio Git.
  • Preparar y hacer commit de archivos.
  • Crear y usar un archivo .gitignore.
  • Verificar el estado de tu repositorio.

Estas habilidades fundamentales son esenciales para gestionar cualquier proyecto con Git. En las siguientes secciones, profundizaremos en características y flujos de trabajo más avanzados de Git.
