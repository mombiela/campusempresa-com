Introducción

Git es un poderoso sistema de control de versiones que ayuda a los desarrolladores a gestionar los cambios en su base de código. En esta sección, cubriremos tres comandos fundamentales de Git: add, commit y status. Entender estos comandos es esencial para cualquiera que esté comenzando con Git.

Comando `add` de Git

El comando add se utiliza para agregar cambios en el directorio de trabajo al área de preparación. Este es el primer paso en el proceso de realizar commits de cambios al repositorio.

Conceptos clave

  • Directorio de trabajo: El directorio donde se encuentran los archivos de tu proyecto.
  • Área de preparación: Un área temporal donde se reúnen los cambios antes de hacer un commit.
  • Archivos rastreados: Archivos de los que Git es consciente y está rastreando cambios.

Uso

# Agregar un solo archivo al área de preparación
git add <file_name>

# Agregar todos los cambios en el directorio actual al área de preparación
git add .

Ejemplo

# Agregar un solo archivo
git add index.html

# Agregar todos los cambios en el directorio actual
git add .

Comando `commit` de Git

El comando commit se utiliza para guardar cambios desde el área de preparación al repositorio. Cada commit es una instantánea de tu proyecto en un punto específico en el tiempo.

Conceptos clave

  • Mensaje de commit: Una breve descripción de los cambios que se están realizando.
  • Hash SHA-1: Un identificador único para cada commit.

Uso

# Realizar un commit de los cambios con un mensaje
git commit -m "Tu mensaje de commit aquí"

Ejemplo

# Realizar un commit de los cambios con un mensaje descriptivo
git commit -m "Añadida nueva característica a la página principal"

Comando `status` de Git

El comando status muestra el estado del directorio de trabajo y del área de preparación. Muestra qué cambios han sido preparados, cuáles no, y qué archivos no están siendo rastreados por Git.

Conceptos clave

  • Archivos no rastreados: Archivos que no están siendo rastreados por Git.
  • Archivos modificados: Archivos que han sido cambiados pero aún no preparados.
  • Archivos preparados: Archivos que han sido añadidos al área de preparación.

Uso

# Verificar el estado del directorio de trabajo y del área de preparación
git status

Ejemplo

# Mostrar el estado del repositorio
git status

Resumen

Entender los comandos add, commit y status es crucial para un control de versiones efectivo con Git. Aquí tienes un resumen rápido:

  • Usa git add para mover cambios al área de preparación.
  • Usa git commit para guardar los cambios preparados en el repositorio con un mensaje descriptivo.
  • Usa git status para verificar el estado de tu directorio de trabajo y área de preparación.

Al dominar estos comandos, estarás bien encaminado para gestionar eficientemente tu base de código con Git.
