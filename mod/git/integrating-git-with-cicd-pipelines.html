Introducción a CI/CD

  • Integración Continua (CI): La práctica de integrar automáticamente los cambios de código de múltiples contribuyentes en un repositorio compartido varias veces al día.
  • Entrega Continua (CD): La práctica de preparar automáticamente los cambios de código para una liberación a producción.
  • Despliegue Continuo: Extiende CD al desplegar automáticamente cada cambio que pasa todas las etapas del pipeline de producción a los clientes.

Importancia de Integrar Git con CI/CD

  • Automatización: Reduce la intervención manual, minimizando el error humano.
  • Consistencia: Asegura que el código esté siempre en un estado desplegable.
  • Retroalimentación: Proporciona retroalimentación inmediata sobre la calidad del código y problemas de integración.
  • Colaboración: Facilita una mejor colaboración entre los miembros del equipo.

Configuración de un Pipeline de CI/CD con Git

Elegir una Herramienta de CI/CD

  • Herramientas Populares:
    • Jenkins
    • GitLab CI/CD
    • CircleCI
    • Travis CI
    • GitHub Actions

Ejemplo: Usando GitHub Actions para CI/CD

  1. Crear un Repositorio en GitHub:

    • Inicializa un nuevo repositorio en GitHub o usa uno existente.
    • Asegúrate de que tu repositorio tenga un archivo .gitignore y un README.md.

  2. Agregar un Archivo de Workflow:

    • Crea un directorio llamado .github/workflows en la raíz de tu repositorio.
    • Agrega un nuevo archivo llamado ci.yml dentro de este directorio.

  3. Definir el Workflow:

    name: CI Pipeline

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

  4. Explicación del Workflow:

    • name: El nombre del workflow.
    • on: Especifica los eventos que desencadenan el workflow. Aquí, se desencadena en eventos de push y pull_request a la rama main.
    • jobs: Define los trabajos a ejecutar. En este ejemplo, hay un solo trabajo llamado build.
    • runs-on: Especifica el tipo de máquina en la que se ejecutará el trabajo. ubuntu-latest es un entorno comúnmente usado.
    • steps: Lista los pasos a ejecutar en el trabajo.
      • Checkout code: Usa la acción actions/checkout para clonar el repositorio.
      • Set up Node.js: Usa la acción actions/setup-node para configurar Node.js.
      • Install dependencies: Ejecuta npm install para instalar las dependencias del proyecto.
      • Run tests: Ejecuta npm test para ejecutar el conjunto de pruebas.

Ejemplo: Usando Jenkins para CI/CD

  1. Instalar Jenkins:

    • Sigue la guía oficial de instalación de Jenkins para tu sistema operativo.

  2. Crear un Nuevo Trabajo en Jenkins:

    • Abre Jenkins y crea un nuevo proyecto Freestyle.

  3. Configurar la Gestión del Código Fuente:

    • En la sección "Source Code Management", selecciona "Git".
    • Ingresa la URL del repositorio y las credenciales si es necesario.

  4. Agregar Pasos de Construcción:

    • En la sección "Build", agrega pasos de construcción como "Execute shell" para ejecutar comandos como:
    npm install
npm test

  5. Configurar Acciones Post-Construcción:

    • Agrega acciones post-construcción como "Archive the artifacts" o "Publish JUnit test result report" para manejar los resultados de la construcción.

Mejores Prácticas para Integrar Git con CI/CD

  • Estrategia de Ramas: Usa una estrategia de ramas consistente como GitFlow o GitHub Flow.
  • Mensajes de Commit: Escribe mensajes de commit claros y descriptivos.
  • Pruebas Automatizadas: Asegúrate de que tu pipeline de CI incluya pruebas automatizadas.
  • Paridad de Entornos: Mantén los entornos de desarrollo, staging y producción lo más similares posible.
  • Seguridad: Asegura tu pipeline de CI/CD gestionando adecuadamente secretos y credenciales.

Conclusión

Integrar Git con pipelines de CI/CD es crucial para el desarrollo de software moderno. Automatiza el proceso de integrar, probar y desplegar código, asegurando que tu aplicación esté siempre en un estado desplegable. Siguiendo las mejores prácticas y usando herramientas como GitHub Actions o Jenkins, puedes optimizar tu flujo de trabajo de desarrollo y mejorar la colaboración dentro de tu equipo.
