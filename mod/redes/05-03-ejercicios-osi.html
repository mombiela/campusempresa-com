Ejercicios del Modelo OSI
Siguiente ►

En esta sección, pondremos en práctica los conceptos aprendidos sobre el Modelo OSI. Realizaremos una serie de ejercicios que te ayudarán a reforzar tu comprensión de cada capa del modelo y su funcionamiento.

Ejercicio 1: Identificación de Capas

Enunciado:

A continuación, se presentan varias tareas relacionadas con la comunicación en red. Identifica a qué capa del Modelo OSI pertenece cada tarea.

  1. Convertir señales digitales en señales eléctricas.
  2. Establecer una conexión lógica entre dos dispositivos.
  3. Enviar paquetes de datos a través de diferentes redes.
  4. Asegurar que los datos lleguen sin errores y en el orden correcto.
  5. Gestionar el diálogo entre dos aplicaciones.
  6. Traducir datos entre diferentes formatos.
  7. Proveer servicios de red a las aplicaciones.

Solución:

  1. Capa Física: Convertir señales digitales en señales eléctricas.
  2. Capa de Enlace de Datos: Establecer una conexión lógica entre dos dispositivos.
  3. Capa de Red: Enviar paquetes de datos a través de diferentes redes.
  4. Capa de Transporte: Asegurar que los datos lleguen sin errores y en el orden correcto.
  5. Capa de Sesión: Gestionar el diálogo entre dos aplicaciones.
  6. Capa de Presentación: Traducir datos entre diferentes formatos.
  7. Capa de Aplicación: Proveer servicios de red a las aplicaciones.

Ejercicio 2: Análisis de Tráfico de Red

Enunciado:

Analiza el siguiente tráfico de red y determina a qué capa del Modelo OSI pertenece cada uno de los siguientes elementos:

  • Dirección IP de origen y destino.
  • Número de puerto de origen y destino.
  • Datos de usuario.
  • Encabezado Ethernet.
  • Control de flujo y errores.

Solución:

  • Dirección IP de origen y destino: Capa de Red.
  • Número de puerto de origen y destino: Capa de Transporte.
  • Datos de usuario: Capa de Aplicación.
  • Encabezado Ethernet: Capa de Enlace de Datos.
  • Control de flujo y errores: Capa de Transporte.

Ejercicio 4: Descripción de Funciones

Enunciado:

Describe las funciones principales de cada capa del Modelo OSI.

Solución:

  1. Capa Física: Se encarga de la transmisión de bits a través de un medio físico. Define las características eléctricas, mecánicas, de procedimiento y funcionales para activar, mantener y desactivar el enlace físico entre sistemas finales.
  2. Capa de Enlace de Datos: Proporciona un enlace fiable entre dos nodos conectados físicamente. Se encarga de la detección y corrección de errores en la capa física.
  3. Capa de Red: Gestiona el direccionamiento y la entrega de paquetes entre dispositivos en diferentes redes. Utiliza direcciones IP para determinar la ruta óptima.
  4. Capa de Transporte: Asegura la transferencia de datos de extremo a extremo. Proporciona control de flujo, segmentación y reensamblaje, y control de errores.
  5. Capa de Sesión: Gestiona y controla las conexiones entre aplicaciones. Establece, mantiene y termina sesiones entre aplicaciones.
  6. Capa de Presentación: Traduce datos entre el formato de la red y el formato que puede entender una aplicación. Se encarga de la codificación, compresión y encriptación de datos.
  7. Capa de Aplicación: Proporciona servicios de red directamente a las aplicaciones del usuario. Incluye protocolos como HTTP, FTP, SMTP, etc.

Ejercicio 5: Caso Práctico

Enunciado:

Imagina que estás configurando una red para una pequeña empresa. Describe cómo interactuarían las capas del Modelo OSI cuando un empleado envía un correo electrónico a otro empleado dentro de la misma red.

Solución:

  1. Capa de Aplicación: El empleado utiliza un cliente de correo electrónico (por ejemplo, Outlook) para redactar y enviar el correo. El cliente de correo utiliza el protocolo SMTP para enviar el mensaje.
  2. Capa de Presentación: El mensaje de correo electrónico se convierte en un formato estándar que puede ser entendido por el servidor de correo.
  3. Capa de Sesión: Se establece una sesión entre el cliente de correo del empleado y el servidor de correo para gestionar la transferencia del mensaje.
  4. Capa de Transporte: El mensaje se divide en segmentos. Cada segmento se envía con un número de puerto específico que identifica el servicio de correo electrónico.
  5. Capa de Red: Los segmentos se encapsulan en paquetes con direcciones IP de origen y destino. Los paquetes se enrutan a través de la red interna de la empresa.
  6. Capa de Enlace de Datos: Los paquetes se encapsulan en tramas con direcciones MAC de origen y destino. Las tramas se transmiten a través de la red local.
  7. Capa Física: Las tramas se convierten en señales eléctricas y se transmiten a través del cableado de la red hasta el servidor de correo.

Conclusión

En esta sección, hemos practicado la identificación y descripción de las funciones de cada capa del Modelo OSI, así como la correspondencia entre los modelos OSI y TCP/IP. Estos ejercicios te ayudarán a consolidar tu comprensión del Modelo OSI y su aplicación práctica en redes de comunicación. En el siguiente módulo, aplicaremos estos conceptos al Modelo TCP/IP.
