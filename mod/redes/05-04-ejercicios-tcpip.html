Ejercicios del Modelo TCP/IP
Siguiente ►

En esta sección, vamos a poner en práctica los conceptos aprendidos sobre el Modelo TCP/IP. A través de una serie de ejercicios, reforzaremos nuestro entendimiento de las capas, los protocolos y las funciones dentro del modelo TCP/IP. Cada ejercicio incluye una explicación detallada y una solución para que puedas verificar tu comprensión.

Ejercicio 1: Identificación de Capas y Protocolos

Enunciado

Asocia cada uno de los siguientes protocolos con la capa correspondiente del modelo TCP/IP:

  1. HTTP
  2. TCP
  3. IP
  4. Ethernet
  5. DNS

Solución

Protocolo Capa TCP/IP
HTTP Capa de Aplicación
TCP Capa de Transporte
IP Capa de Internet
Ethernet Capa de Acceso a la Red
DNS Capa de Aplicación

Explicación

  • HTTP (HyperText Transfer Protocol): Protocolo de la capa de aplicación utilizado para la transmisión de documentos web.
  • TCP (Transmission Control Protocol): Protocolo de la capa de transporte que proporciona una comunicación fiable y ordenada.
  • IP (Internet Protocol): Protocolo de la capa de internet que se encarga de direccionar y enrutar los paquetes de datos.
  • Ethernet: Protocolo de la capa de acceso a la red utilizado para la comunicación en redes locales (LAN).
  • DNS (Domain Name System): Protocolo de la capa de aplicación que traduce nombres de dominio en direcciones IP.

Ejercicio 2: Análisis de Tráfico de Red

Enunciado

Analiza el siguiente fragmento de tráfico de red y determina a qué capa del modelo TCP/IP pertenece cada parte:

Ethernet Header: 00:1A:2B:3C:4D:5E -> 00:5E:4D:3C:2B:1A
IP Header: 192.168.1.1 -> 192.168.1.2
TCP Header: Source Port: 443, Destination Port: 80
HTTP Data: GET /index.html HTTP/1.1

Solución

Fragmento de Tráfico Capa TCP/IP
Ethernet Header Capa de Acceso a la Red
IP Header Capa de Internet
TCP Header Capa de Transporte
HTTP Data Capa de Aplicación

Explicación

  • Ethernet Header: Pertenece a la capa de acceso a la red, ya que define la comunicación a nivel de hardware.
  • IP Header: Pertenece a la capa de internet, encargada de la dirección y el enrutamiento de los paquetes.
  • TCP Header: Pertenece a la capa de transporte, que gestiona la transmisión de datos entre hosts.
  • HTTP Data: Pertenece a la capa de aplicación, que maneja los protocolos de alto nivel utilizados por las aplicaciones.

Ejercicio 3: Configuración de Red

Enunciado

Configura una red simple con dos computadoras (A y B) conectadas a través de un switch. Asigna las siguientes direcciones IP y configura la máscara de subred:

  • Computadora A: IP 192.168.1.10, Máscara de subred 255.255.255.0
  • Computadora B: IP 192.168.1.20, Máscara de subred 255.255.255.0

Solución

  1. Configuración en Computadora A: 
    ifconfig eth0 192.168.1.10 netmask 255.255.255.0
  2. Configuración en Computadora B: 
    ifconfig eth0 192.168.1.20 netmask 255.255.255.0

Explicación

  • ifconfig: Comando utilizado en sistemas Unix/Linux para configurar interfaces de red.
  • eth0: Nombre de la interfaz de red.
  • 192.168.1.10 / 192.168.1.20: Direcciones IP asignadas a las computadoras A y B respectivamente.
  • netmask 255.255.255.0: Máscara de subred que define la red local.

Ejercicio 4: Resolución de Problemas de Conectividad

Enunciado

Supongamos que la computadora A no puede comunicarse con la computadora B. Realiza los siguientes pasos para diagnosticar y resolver el problema:

  1. Verifica la configuración de IP en ambas computadoras.
  2. Realiza un ping desde la computadora A a la computadora B.
  3. Comprueba la configuración del switch.

Solución

  1. Verificación de IP:

    # En Computadora A
ifconfig eth0
# En Computadora B
ifconfig eth0

    Asegúrate de que las direcciones IP y las máscaras de subred estén configuradas correctamente.

  2. Ping:

    # Desde Computadora A
ping 192.168.1.20

    Si el ping no tiene éxito, puede haber un problema de conectividad física o de configuración.

  3. Configuración del Switch:

    • Asegúrate de que ambos puertos del switch estén activos y configurados correctamente.
    • Verifica que los cables de red estén conectados correctamente.

Explicación

  • ifconfig: Permite verificar la configuración de la interfaz de red.
  • ping: Herramienta de diagnóstico que envía paquetes ICMP para comprobar la conectividad.
  • Switch: Dispositivo de red que conecta múltiples dispositivos en una red local.

Conclusión

En esta sección, hemos practicado la identificación de capas y protocolos del modelo TCP/IP, el análisis de tráfico de red, la configuración de direcciones IP y la resolución de problemas de conectividad. Estos ejercicios te ayudarán a consolidar tu comprensión del modelo TCP/IP y a aplicar los conceptos en situaciones prácticas. Asegúrate de revisar cada solución y explicación para reforzar tu aprendizaje y estar preparado para el siguiente tema.
