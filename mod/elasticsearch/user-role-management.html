Gestión de Usuarios y Roles en Elasticsearch
Registro de Auditoría ►

La gestión de usuarios y roles es un aspecto crucial para asegurar tu clúster de Elasticsearch. Este tema te guiará desde los conceptos básicos hasta los avanzados, asegurando que entiendas cómo gestionar usuarios y roles de manera efectiva.

Introducción a la Gestión de Usuarios y Roles

Elasticsearch proporciona un modelo de seguridad robusto que incluye autenticación de usuarios y control de acceso basado en roles (RBAC). Esto asegura que solo los usuarios autorizados puedan acceder y realizar acciones en tus datos.

Conceptos Clave

  • Usuario: Una cuenta individual que puede autenticarse y realizar acciones en Elasticsearch.
  • Rol: Un conjunto de permisos que pueden asignarse a los usuarios para controlar su acceso a los recursos.
  • RBAC (Control de Acceso Basado en Roles): Un método para regular el acceso a los recursos basado en los roles asignados a los usuarios individuales.

Creación y Gestión de Usuarios

Los usuarios en Elasticsearch son entidades que pueden autenticarse y realizar acciones basadas en sus roles asignados.

Creación de un Usuario

Para crear un usuario, puedes usar la interfaz de usuario de Kibana o la API de Elasticsearch. Aquí, nos enfocaremos en usar la API.

PUT /_security/user/john_doe
{
  "password" : "password123",
  "roles" : [ "admin", "read_only" ],
  "full_name" : "John Doe",
  "email" : "john.doe@example.com"
}
  • Endpoint: /security/user/{username}
  • Campos:
    • password: La contraseña del usuario.
    • roles: Un array de roles asignados al usuario.
    • full_name: El nombre completo del usuario.
    • email: La dirección de correo electrónico del usuario.

Gestión de Usuarios

Puedes actualizar, eliminar o recuperar información del usuario usando la API de Elasticsearch.

  • Actualizar Usuario:

    POST /_security/user/john_doe/_password
{
  "password" : "new_password123"
}

  • Eliminar Usuario:

    DELETE /_security/user/john_doe

  • Recuperar Información del Usuario:

    GET /_security/user/john_doe

Creación y Gestión de Roles

Los roles definen los permisos que los usuarios tienen dentro del clúster de Elasticsearch.

Creación de un Rol

Los roles pueden crearse usando la API de Elasticsearch.

PUT /_security/role/read_only
{
  "cluster": ["monitor"],
  "indices": [
    {
      "names": [ "index1", "index2" ],
      "privileges": [ "read" ]
    }
  ]
}
  • Endpoint: /security/role/{role_name}
  • Campos:
    • cluster: Permisos a nivel de clúster.
    • indices: Permisos específicos de índices.

Gestión de Roles

Puedes actualizar, eliminar o recuperar información del rol usando la API de Elasticsearch.

  • Actualizar Rol:

    PUT /_security/role/read_only
{
  "cluster": ["monitor"],
  "indices": [
    {
      "names": [ "index1", "index2" ],
      "privileges": [ "read", "write" ]
    }
  ]
}

  • Eliminar Rol:

    DELETE /_security/role/read_only

  • Recuperar Información del Rol:

    GET /_security/role/read_only

Asignación de Roles a Usuarios

Los roles pueden asignarse a los usuarios durante la creación del usuario o actualizarse posteriormente.

Asignación de Roles Durante la Creación del Usuario

Como se muestra en el ejemplo de creación de usuario:

PUT /_security/user/john_doe
{
  "password" : "password123",
  "roles" : [ "admin", "read_only" ],
  "full_name" : "John Doe",
  "email" : "john.doe@example.com"
}

Actualización de Roles de Usuario

Puedes actualizar los roles asignados a un usuario usando la siguiente llamada a la API:

POST /_security/user/john_doe/_roles
{
  "roles" : [ "admin", "read_only", "new_role" ]
}

Mejores Prácticas

  • Usar Contraseñas Fuertes: Asegúrate de que las contraseñas de los usuarios sean fuertes y se actualicen regularmente.
  • Principio de Menor Privilegio: Asigna los permisos mínimos necesarios a los usuarios.
  • Auditorías Regulares: Revisa regularmente los roles y permisos de los usuarios para asegurarte de que estén actualizados y sean apropiados.

Conclusión

La gestión de usuarios y roles en Elasticsearch es esencial para asegurar tu clúster y garantizar que solo los usuarios autorizados puedan acceder y realizar acciones en tus datos. Al entender cómo crear, gestionar y asignar roles a los usuarios, puedes implementar un modelo de seguridad robusto que proteja tu entorno de Elasticsearch.
