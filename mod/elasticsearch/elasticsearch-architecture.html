Introducción

Elasticsearch es un motor de búsqueda y análisis distribuido y RESTful capaz de resolver un número creciente de casos de uso. Comprender su arquitectura es crucial para desplegar y gestionar eficazmente los clústeres de Elasticsearch. Esta sección cubrirá los componentes y conceptos fundamentales de la arquitectura de Elasticsearch.

Componentes Clave

Clúster

  • Definición: Un clúster es una colección de uno o más nodos (servidores) que juntos contienen todos tus datos y proporcionan capacidades de indexación y búsqueda federadas.
  • Nombre del Clúster: Cada clúster se identifica por un nombre único, que por defecto es "elasticsearch". Los nodos solo pueden unirse a clústeres con el mismo nombre.

Nodo

  • Definición: Un nodo es un servidor único que forma parte del clúster, almacena datos y participa en las capacidades de indexación y búsqueda del clúster.
  • Roles del Nodo:
    • Nodo Maestro: Responsable de la configuración y operaciones a nivel de clúster.
    • Nodo de Datos: Almacena datos y ejecuta operaciones relacionadas con los datos, como CRUD, búsqueda y agregaciones.
    • Nodo de Ingesta: Preprocesa documentos antes de la indexación.
    • Nodo Coordinador: Enruta solicitudes, maneja la fase de reducción de búsqueda y distribuye la indexación en masa.

Índice

  • Definición: Un índice es una colección de documentos que tienen características algo similares.
  • Estructura: Un índice se identifica por un nombre y contiene múltiples tipos (obsoleto en versiones más recientes), que a su vez contienen múltiples documentos.

Documento

  • Definición: Un documento es una unidad básica de información que puede ser indexada. Se expresa en formato JSON.
  • Ejemplo: 
    {
  "user": "john_doe",
  "post_date": "2023-10-01",
  "message": "Elasticsearch es increíble!"
}

Fragmento

  • Definición: Un índice puede dividirse en múltiples piezas llamadas fragmentos. Cada fragmento es un "índice" completamente funcional e independiente que puede alojarse en cualquier nodo del clúster.
  • Tipos:
    • Fragmento Primario: Fragmento original donde se indexan los datos.
    • Fragmento Réplica: Copia del fragmento primario, utilizada para conmutación por error y aumento del rendimiento de búsqueda.

Réplica

  • Definición: Una réplica es una copia de un fragmento, proporcionando redundancia y aumentando el rendimiento de búsqueda.
  • Configuración de Réplica: Configurable a nivel de índice, permitiendo especificar el número de réplicas para cada fragmento primario.

Flujo de Datos en Elasticsearch

Proceso de Indexación

  1. Envío de Documento: Se envía un documento a un índice.
  2. Enrutamiento: El documento se enruta a un fragmento primario.
  3. Ingesta: Si se utiliza un nodo de ingesta, el documento se preprocesa.
  4. Almacenamiento: El documento se almacena en el fragmento primario y se replica en los fragmentos réplica.

Proceso de Búsqueda

  1. Enrutamiento de Solicitud: Se envía una solicitud de búsqueda a un nodo coordinador.
  2. Distribución de Consulta: El nodo coordinador reenvía la consulta a los fragmentos relevantes.
  3. Ejecución: Cada fragmento ejecuta la consulta y devuelve resultados al nodo coordinador.
  4. Agregación: El nodo coordinador agrega los resultados y devuelve la respuesta final.

Ejemplo: Creación de un Índice y Adición de un Documento

Creación de un Índice

PUT /my_index
{
  "settings": {
    "number_of_shards": 3,
    "number_of_replicas": 2
  }
}
  • Explicación: Este comando crea un índice llamado my_index con 3 fragmentos primarios y 2 réplicas para cada fragmento.

Adición de un Documento

POST /my_index/_doc/1
{
  "user": "john_doe",
  "post_date": "2023-10-01",
  "message": "Elasticsearch es increíble!"
}
  • Explicación: Este comando añade un documento con ID 1 al índice my_index.

Conclusión

Comprender la arquitectura de Elasticsearch es fundamental para gestionar y escalar eficientemente tus soluciones de búsqueda y análisis. Componentes clave como clústeres, nodos, índices, documentos, fragmentos y réplicas forman la columna vertebral de Elasticsearch. Al comprender estos conceptos, puedes diseñar, desplegar y solucionar problemas de clústeres de Elasticsearch para satisfacer tus necesidades específicas.
