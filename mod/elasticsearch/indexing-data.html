Indexación de Datos
Buscando Datos ►

La indexación de datos en Elasticsearch es un concepto fundamental que te permite almacenar, buscar y analizar grandes volúmenes de datos rápidamente y en tiempo casi real. Este tema cubrirá los conceptos básicos de la indexación, cómo crear y gestionar índices, y técnicas avanzadas para optimizar tu proceso de indexación de datos.

Introducción a la Indexación

La indexación en Elasticsearch es el proceso de añadir datos a un índice. Un índice es una colección de documentos que comparten características similares. Cada documento es un objeto JSON que contiene campos, que son los datos que deseas buscar y analizar.

Conceptos Clave

  • Índice: Una colección de documentos.
  • Documento: Un objeto JSON que contiene datos.
  • Campo: Un par clave-valor dentro de un documento.
  • Mapeo: Define cómo los documentos y sus campos se almacenan e indexan.

Creación de un Índice

Para crear un índice en Elasticsearch, puedes usar la solicitud PUT. Aquí tienes un ejemplo simple:

PUT /mi_primer_indice
{
  "settings": {
    "number_of_shards": 1,
    "number_of_replicas": 1
  }
}
  • number_of_shards: Define el número de fragmentos primarios.
  • number_of_replicas: Define el número de fragmentos de réplica.

Ejemplo Explicado

En el ejemplo anterior, creamos un índice llamado mi_primer_indice con 1 fragmento primario y 1 fragmento de réplica. Los fragmentos son las unidades básicas de almacenamiento en Elasticsearch, y las réplicas son copias de los fragmentos para la tolerancia a fallos.

Añadiendo Documentos a un Índice

Para añadir un documento a un índice, usas la solicitud POST o PUT. Aquí tienes un ejemplo:

POST /mi_primer_indice/_doc/1
{
  "nombre": "John Doe",
  "edad": 30,
  "ocupación": "Ingeniero de Software"
}
  • /mi_primer_indice/_doc/1: Especifica el índice y el ID del documento.
  • nombre, edad, ocupación: Campos dentro del documento.

Ejemplo Explicado

En este ejemplo, añadimos un documento con ID 1 al índice mi_primer_indice. El documento contiene tres campos: nombre, edad y ocupación.

Actualización de Documentos

Para actualizar un documento existente, puedes usar la solicitud POST con el endpoint _update. Aquí tienes un ejemplo:

POST /mi_primer_indice/_update/1
{
  "doc": {
    "edad": 31
  }
}

Ejemplo Explicado

En este ejemplo, actualizamos el documento con ID 1 en el índice mi_primer_indice, cambiando el campo edad a 31.

Eliminación de Documentos

Para eliminar un documento, usas la solicitud DELETE. Aquí tienes un ejemplo:

DELETE /mi_primer_indice/_doc/1

Ejemplo Explicado

En este ejemplo, eliminamos el documento con ID 1 del índice mi_primer_indice.

Técnicas Avanzadas de Indexación

Indexación en Masa

Para un mejor rendimiento, especialmente cuando se trata de grandes conjuntos de datos, puedes usar la API _bulk para indexar múltiples documentos en una sola solicitud. Aquí tienes un ejemplo:

POST /_bulk
{ "index": { "_index": "mi_primer_indice", "_id": "2" } }
{ "nombre": "Jane Doe", "edad": 25, "ocupación": "Científica de Datos" }
{ "index": { "_index": "mi_primer_indice", "_id": "3" } }
{ "nombre": "Alice Smith", "edad": 28, "ocupación": "Gerente de Producto" }

Ejemplo Explicado

En este ejemplo, usamos la API _bulk para añadir dos documentos al índice mi_primer_indice en una sola solicitud.

Alias de Índice

Los alias de índice te permiten usar un solo nombre para referirte a uno o más índices. Esto puede ser útil para gestionar versiones de índices y realizar reindexaciones sin tiempo de inactividad. Aquí tienes un ejemplo:

POST /_aliases
{
  "actions": [
    { "add": { "index": "mi_primer_indice", "alias": "indice_actual" } }
  ]
}

Ejemplo Explicado

En este ejemplo, creamos un alias llamado indice_actual que apunta al índice mi_primer_indice.

Conclusión

La indexación de datos en Elasticsearch es un paso crucial para almacenar y recuperar datos de manera eficiente. Al entender los conceptos básicos de la creación de índices, la adición, actualización y eliminación de documentos, así como técnicas avanzadas como la indexación en masa y los alias de índice, puedes optimizar tu uso de Elasticsearch para un mejor rendimiento y escalabilidad.
