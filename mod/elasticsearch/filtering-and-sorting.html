Filtrado y Ordenación en Elasticsearch
Agregaciones ►

Introducción

En esta sección, exploraremos cómo filtrar y ordenar datos en Elasticsearch. Filtrar y ordenar son operaciones esenciales para consultar grandes conjuntos de datos de manera eficiente. Cubriremos lo básico y gradualmente pasaremos a conceptos más avanzados.

Filtrado de Datos

El filtrado te permite reducir los resultados de búsqueda basados en criterios específicos. Elasticsearch proporciona varias formas de filtrar datos.

Filtros Básicos

Los filtros básicos son directos y fáciles de usar. Incluyen filtros de término, rango y existencia.

  • Filtro de Término: Filtra documentos que contienen un término exacto en un campo proporcionado.

    {
  "query": {
    "term": {
      "status": "active"
    }
  }
}

  • Filtro de Rango: Filtra documentos que tienen campos dentro de un cierto rango.

    {
  "query": {
    "range": {
      "age": {
        "gte": 30,
        "lte": 40
      }
    }
  }
}

  • Filtro de Existencia: Filtra documentos que contienen un campo específico.

    {
  "query": {
    "exists": {
      "field": "email"
    }
  }
}

Filtros Compuestos

Los filtros compuestos combinan múltiples filtros usando operadores lógicos como must, should y must_not.

  • Consulta Bool: Combina múltiples filtros usando lógica booleana.

    {
  "query": {
    "bool": {
      "must": [
        { "term": { "status": "active" } },
        { "range": { "age": { "gte": 30, "lte": 40 } } }
      ],
      "must_not": [
        { "term": { "role": "admin" } }
      ]
    }
  }
}

Filtros Avanzados

Los filtros avanzados proporcionan más flexibilidad y poder.

  • Filtro Anidado: Filtra documentos que contienen objetos anidados o arreglos.

    {
  "query": {
    "nested": {
      "path": "comments",
      "query": {
        "bool": {
          "must": [
            { "match": { "comments.author": "John" } },
            { "range": { "comments.date": { "gte": "2022-01-01" } } }
          ]
        }
      }
    }
  }
}

Ordenación de Datos

La ordenación te permite ordenar tus resultados de búsqueda basados en campos específicos.

Ordenación Básica

La ordenación básica es simple e implica ordenar por un solo campo.

  • Ordenar por Campo: Ordena documentos por un campo especificado en orden ascendente o descendente.

    {
  "sort": [
    { "age": { "order": "asc" } }
  ]
}

Ordenación de Múltiples Niveles

La ordenación de múltiples niveles te permite ordenar por múltiples campos.

  • Ordenar por Múltiples Campos: Ordena documentos por múltiples campos en un orden especificado.

    {
  "sort": [
    { "age": { "order": "asc" } },
    { "name": { "order": "desc" } }
  ]
}

Ordenación de Campos Anidados

La ordenación también puede aplicarse a campos anidados dentro de documentos.

  • Ordenar por Campo Anidado: Ordena documentos basados en campos dentro de objetos anidados.

    {
  "sort": [
    {
      "comments.date": {
        "order": "desc",
        "nested": {
          "path": "comments"
        }
      }
    }
  ]
}

Combinando Filtrado y Ordenación

A menudo, necesitarás filtrar y ordenar datos simultáneamente. Elasticsearch te permite combinar estas operaciones sin problemas.

  • Filtrar y Ordenar: Combina filtrado y ordenación en una sola consulta.

    {
  "query": {
    "bool": {
      "must": [
        { "term": { "status": "active" } },
        { "range": { "age": { "gte": 30, "lte": 40 } } }
      ]
    }
  },
  "sort": [
    { "age": { "order": "asc" } }
  ]
}

Conclusión

El filtrado y la ordenación son características poderosas en Elasticsearch que te permiten consultar y organizar tus datos de manera eficiente. Al entender y utilizar filtros básicos, compuestos y avanzados, así como técnicas de ordenación, puedes realizar consultas complejas y recuperar datos en un orden significativo. Practica estos conceptos con varios conjuntos de datos para volverte competente en la gestión de consultas en Elasticsearch.
