Asegurando Elasticsearch
Gestión de Usuarios y Roles ►

Asegurar Elasticsearch es crucial para proteger tus datos y asegurar que solo los usuarios autorizados tengan acceso a tu clúster. Este tema cubrirá varios aspectos de la seguridad en Elasticsearch, desde la autenticación básica hasta configuraciones de seguridad avanzadas.

Autenticación Básica

La autenticación básica es la forma más simple de asegurar tu clúster de Elasticsearch. Implica configurar un nombre de usuario y una contraseña para controlar el acceso.

Configuración de la Autenticación Básica

  1. Instalar el Plugin X-Pack: X-Pack es un plugin que proporciona características de seguridad para Elasticsearch.

    bin/elasticsearch-plugin install x-pack

  2. Configurar el Archivo Elasticsearch.yml: Añade las siguientes líneas a tu archivo de configuración elasticsearch.yml.

    xpack.security.enabled: true

  3. Iniciar Elasticsearch: Reinicia tu instancia de Elasticsearch para aplicar los cambios.

    bin/elasticsearch

  4. Configurar Usuarios Integrados: Usa la herramienta elasticsearch-setup-passwords para establecer contraseñas para los usuarios integrados.

    bin/elasticsearch-setup-passwords interactive

  5. Probar la Autenticación: Usa curl para probar la autenticación.

    curl -u elastic:your_password http://localhost:9200

Control de Acceso Basado en Roles (RBAC)

RBAC te permite definir roles y asignarlos a usuarios, controlando qué acciones pueden realizar y a qué datos pueden acceder.

Creación de Roles

  1. Definir Roles en un Archivo JSON: Crea un archivo JSON para definir roles.

    {
  "cluster": ["all"],
  "indices": [
    {
      "names": ["*"],
      "privileges": ["read", "write"]
    }
  ]
}

  2. Añadir Roles Usando la API de Roles: Usa la API de Roles para añadir roles.

    curl -X POST "localhost:9200/_security/role/my_role" -H 'Content-Type: application/json' -d @role.json

Asignación de Roles a Usuarios

  1. Crear un Usuario: Usa la API de Usuarios para crear un usuario y asignarle un rol. 
    curl -X POST "localhost:9200/_security/user/my_user" -H 'Content-Type: application/json' -d '{
  "password" : "password",
  "roles" : [ "my_role" ],
  "full_name" : "User Name",
  "email" : "user@example.com"
}'

Seguridad de la Capa de Transporte (TLS)

TLS encripta la comunicación entre nodos de Elasticsearch y entre clientes y nodos, asegurando la integridad y confidencialidad de los datos.

Habilitación de TLS

  1. Generar Certificados: Usa la herramienta elasticsearch-certutil para generar certificados.

    bin/elasticsearch-certutil cert --ca

  2. Configurar Elasticsearch para TLS: Añade las siguientes líneas a tu archivo elasticsearch.yml.

    xpack.security.transport.ssl.enabled: true
xpack.security.transport.ssl.verification_mode: certificate
xpack.security.transport.ssl.key: /path/to/your/node.key
xpack.security.transport.ssl.certificate: /path/to/your/node.crt
xpack.security.transport.ssl.certificate_authorities: [ "/path/to/your/ca.crt" ]

  3. Reiniciar Elasticsearch: Reinicia tu instancia de Elasticsearch para aplicar los cambios.

    bin/elasticsearch

Auditoría

La auditoría te ayuda a rastrear el acceso y los cambios en tu clúster de Elasticsearch, proporcionando un rastro de acciones para propósitos de seguridad y cumplimiento.

Habilitación de la Auditoría

  1. Configurar la Auditoría en Elasticsearch.yml: Añade las siguientes líneas a tu archivo elasticsearch.yml.

    xpack.security.audit.enabled: true
xpack.security.audit.outputs: [ index, logfile ]

  2. Revisar los Registros de Auditoría: Los registros de auditoría se almacenarán en las ubicaciones de salida especificadas, como un índice o un archivo de registro.

Conclusión

Asegurar Elasticsearch implica múltiples capas de protección, incluyendo autenticación básica, control de acceso basado en roles, seguridad de la capa de transporte y auditoría. Al implementar estas medidas de seguridad, puedes asegurar que tu clúster de Elasticsearch esté protegido contra accesos no autorizados y brechas de datos. Mantén siempre tus configuraciones de seguridad actualizadas y revisa regularmente tus políticas de seguridad para adaptarte a nuevas amenazas.
