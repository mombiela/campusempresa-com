Introducción

Las operaciones de copia de seguridad y restauración son críticas para mantener la integridad y disponibilidad de los datos en Elasticsearch. Esta sección cubrirá los conceptos fundamentales, herramientas y técnicas para realizar copias de seguridad y restauraciones en Elasticsearch, desde niveles básicos hasta avanzados.

Conceptos Clave

  • Instantánea: Una instantánea es una copia de seguridad de un índice o un clúster en Elasticsearch. Captura el estado de los datos en un momento específico.
  • Repositorio: Un repositorio es una ubicación de almacenamiento donde se guardan las instantáneas. Puede ser un sistema de archivos compartido, Amazon S3, HDFS, etc.
  • Restaurar: El proceso de recuperar datos de una instantánea.

Configuración de un Repositorio

Antes de tomar una instantánea, necesitas configurar un repositorio donde se almacenarán las instantáneas.

Ejemplo: Registrar un Repositorio de Sistema de Archivos

PUT /_snapshot/my_backup
{
  "type": "fs",
  "settings": {
    "location": "/mount/backups/my_backup"
  }
}
  • Explicación:
    • PUT /_snapshot/my_backup: Crea un nuevo repositorio de instantáneas llamado my_backup.
    • "type": "fs": Especifica que el tipo de repositorio es un sistema de archivos.
    • "location": "/mount/backups/my_backup": La ruta donde se almacenarán las instantáneas.

Creación de una Instantánea

Una vez configurado el repositorio, puedes crear una instantánea de tus índices.

Ejemplo: Crear una Instantánea

PUT /_snapshot/my_backup/snapshot_1
{
  "indices": "index_1,index_2",
  "ignore_unavailable": true,
  "include_global_state": false
}
  • Explicación:
    • PUT /_snapshot/my_backup/snapshot_1: Crea una instantánea llamada snapshot_1 en el repositorio my_backup.
    • "indices": "index_1,index_2": Especifica los índices a incluir en la instantánea.
    • "ignore_unavailable": true: Ignora los índices que no están disponibles en el momento de la instantánea.
    • "include_global_state": false: Excluye el estado del clúster de la instantánea.

Restauración de una Instantánea

Restaurar una instantánea implica recuperar los datos de una instantánea de vuelta al clúster.

Ejemplo: Restaurar una Instantánea

POST /_snapshot/my_backup/snapshot_1/_restore
{
  "indices": "index_1",
  "ignore_unavailable": true,
  "include_global_state": false,
  "rename_pattern": "index_(.+)",
  "rename_replacement": "restored_index_$1"
}
  • Explicación:
    • POST /_snapshot/my_backup/snapshot_1/_restore: Inicia el proceso de restauración para snapshot_1 desde el repositorio my_backup.
    • "indices": "index_1": Especifica los índices a restaurar.
    • "ignore_unavailable": true: Ignora los índices que no están disponibles en la instantánea.
    • "include_global_state": false: Excluye el estado del clúster de la restauración.
    • "rename_pattern": "index_(.+)": Un patrón regex para coincidir con los nombres de los índices.
    • "rename_replacement": "restored_index_$1": El patrón de reemplazo para renombrar los índices restaurados.

Temas Avanzados

Gestión del Ciclo de Vida de Instantáneas (SLM)

SLM automatiza el proceso de tomar instantáneas basado en una política definida.

Ejemplo: Crear una Política de SLM

PUT /_slm/policy/daily-snapshots
{
  "schedule": "0 30 1 * * ?", 
  "name": "<daily-snap-{now/d}>",
  "repository": "my_backup",
  "config": {
    "indices": ["index_1", "index_2"],
    "ignore_unavailable": false,
    "include_global_state": true
  },
  "retention": {
    "expire_after": "30d",
    "min_count": 5,
    "max_count": 50
  }
}
  • Explicación:
    • PUT /_slm/policy/daily-snapshots: Crea una nueva política de SLM llamada daily-snapshots.
    • "schedule": "0 30 1 * * ?": Expresión cron para ejecutar la política diariamente a la 1:30 AM.
    • "name": "<daily-snap-{now/d}>": Patrón de nombres para las instantáneas.
    • "repository": "my_backup": El repositorio donde se almacenarán las instantáneas.
    • "config": Configuración para las instantáneas.
      • "indices": ["index_1", "index_2"]: Índices a incluir en las instantáneas.
      • "ignore_unavailable": false: No ignorar los índices no disponibles.
      • "include_global_state": true: Incluir el estado del clúster en las instantáneas.
    • "retention": Política de retención para las instantáneas.
      • "expire_after": "30d": Expirar instantáneas con más de 30 días.
      • "min_count": 5: Mantener al menos 5 instantáneas.
      • "max_count": 50: Mantener como máximo 50 instantáneas.

Conclusión

Entender cómo realizar copias de seguridad y restauraciones en Elasticsearch es esencial para la gestión de datos y la recuperación ante desastres. Configurando repositorios, creando instantáneas y restaurando datos, puedes asegurar la integridad y disponibilidad de tus datos. Las características avanzadas como la Gestión del Ciclo de Vida de Instantáneas (SLM) automatizan y agilizan aún más estos procesos, facilitando el mantenimiento de tu clúster de Elasticsearch.
