Instantánea y Restauración
Plugins de Elasticsearch ►

Introducción

La función de Instantánea y Restauración es crucial en Elasticsearch, ya que te permite hacer copias de seguridad de los datos de tu clúster y restaurarlos cuando sea necesario. Esto es esencial para la recuperación de datos, la migración y para asegurar la integridad de los datos a lo largo del tiempo.

Conceptos Clave

  • Instantánea: Una copia de seguridad de los datos almacenados en un clúster de Elasticsearch en un momento específico.
  • Repositorio: Un lugar de almacenamiento donde se guardan las instantáneas. Esto puede ser un sistema de archivos compartido, Amazon S3, HDFS, etc.
  • Restaurar: El proceso de recuperar datos de una instantánea.

Configuración de un Repositorio

Antes de tomar instantáneas, necesitas configurar un repositorio donde se almacenarán las instantáneas.

Ejemplo: Creación de un Repositorio de Sistema de Archivos

PUT /_snapshot/my_backup
{
  "type": "fs",
  "settings": {
    "location": "/mount/backups/my_backup"
  }
}
  • type: Especifica el tipo de repositorio. Aquí, fs significa sistema de archivos.
  • location: La ruta donde se almacenarán las instantáneas.

Tomando una Instantánea

Una vez que el repositorio está configurado, puedes tomar una instantánea de tu clúster.

Ejemplo: Tomando una Instantánea

PUT /_snapshot/my_backup/snapshot_1
{
  "indices": "index_1,index_2",
  "ignore_unavailable": true,
  "include_global_state": false
}
  • indices: Especifica qué índices incluir en la instantánea.
  • ignore_unavailable: Si es verdadero, el proceso de instantánea ignorará los índices que falten o estén cerrados.
  • include_global_state: Si es falso, el estado global del clúster (por ejemplo, plantillas, configuraciones persistentes) no se incluye en la instantánea.

Restaurando desde una Instantánea

Para restaurar datos desde una instantánea, necesitas especificar la instantánea y los índices que deseas restaurar.

Ejemplo: Restaurando una Instantánea

POST /_snapshot/my_backup/snapshot_1/_restore
{
  "indices": "index_1",
  "ignore_unavailable": true,
  "include_global_state": false,
  "rename_pattern": "index_(.+)",
  "rename_replacement": "restored_index_$1"
}
  • indices: Especifica qué índices restaurar.
  • ignore_unavailable: Si es verdadero, el proceso de restauración ignorará los índices que falten en la instantánea.
  • include_global_state: Si es falso, el estado global del clúster no se restaura.
  • rename_pattern: Un patrón regex para coincidir con los nombres de los índices.
  • rename_replacement: Una cadena de reemplazo para renombrar los índices durante la restauración.

Monitoreo de Instantáneas y Restauraciones

Puedes monitorear el estado de tus instantáneas y restauraciones para asegurarte de que se completen con éxito.

Ejemplo: Verificando el Estado de una Instantánea

GET /_snapshot/my_backup/snapshot_1

Ejemplo: Verificando el Estado de una Restauración

GET /_cat/recovery?v

Mejores Prácticas

  • Copias de Seguridad Regulares: Programa instantáneas regulares para asegurarte de tener siempre una copia de seguridad reciente.
  • Pruebas de Restauración: Prueba periódicamente la restauración desde instantáneas para asegurarte de que tu proceso de copia de seguridad funcione correctamente.
  • Repositorios Seguros: Asegúrate de que tus repositorios de instantáneas sean seguros y tengan controles de acceso apropiados.

Conclusión

La función de Instantánea y Restauración en Elasticsearch es una herramienta poderosa que te ayuda a gestionar la seguridad e integridad de tus datos. Configurando repositorios, tomando instantáneas regulares y probando restauraciones, puedes asegurarte de que tus datos estén siempre protegidos y sean recuperables.
