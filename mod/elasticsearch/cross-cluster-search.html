Búsqueda entre Clústeres
Instantáneas y Restauración ►

La Búsqueda entre Clústeres (CCS) es una característica poderosa en Elasticsearch que te permite buscar a través de múltiples clústeres como si fueran un solo clúster. Esto es particularmente útil para organizaciones que tienen datos distribuidos en diferentes ubicaciones geográficas o necesitan gestionar datos a través de múltiples clústeres para escalabilidad y redundancia.

Conceptos Clave

  • Clúster: Una colección de uno o más nodos (servidores) que juntos contienen todos tus datos y proporcionan capacidades de indexación y búsqueda federadas.
  • Búsqueda entre Clústeres: La capacidad de realizar consultas de búsqueda a través de múltiples clústeres de Elasticsearch.
  • Clúster Remoto: Un clúster que está configurado para ser buscado desde otro clúster.

Configuración de la Búsqueda entre Clústeres

Configuración de Clústeres Remotos

Para habilitar la Búsqueda entre Clústeres, primero necesitas configurar los clústeres remotos. Esto implica configurar los clústeres para que se reconozcan entre sí.

  1. Agregar Configuraciones de Clúster Remoto: Puedes agregar configuraciones de clúster remoto en el archivo de configuración elasticsearch.yml o a través de la API de configuraciones del clúster.

    cluster:
  remote:
    cluster_one:
      seeds: ["host1:9300", "host2:9300"]
    cluster_two:
      seeds: ["host3:9300", "host4:9300"]

  2. Usando la API de Configuraciones del Clúster: Puedes agregar dinámicamente clústeres remotos usando la siguiente llamada a la API:

    PUT /_cluster/settings
{
  "persistent": {
    "cluster.remote.cluster_one.seeds": ["host1:9300", "host2:9300"],
    "cluster.remote.cluster_two.seeds": ["host3:9300", "host4:9300"]
  }
}

Consultas a Través de Clústeres

Una vez que los clústeres remotos están configurados, puedes realizar búsquedas a través de ellos usando la siguiente sintaxis:

GET /cluster_one:index_name,cluster_two:index_name/_search
{
  "query": {
    "match_all": {}
  }
}

En este ejemplo, cluster_one:index_name y cluster_two:index_name especifican los índices en los respectivos clústeres.

Ejemplo: Consulta de Búsqueda entre Clústeres

Veamos un ejemplo práctico donde buscamos documentos que contienen el término "Elasticsearch" a través de dos clústeres.

GET /cluster_one:logs-2023.01,cluster_two:logs-2023.01/_search
{
  "query": {
    "match": {
      "message": "Elasticsearch"
    }
  }
}

Explicación

  • GET /cluster_one:logs-2023.01,cluster_two:logs-2023.01/_search: Esta línea especifica que la búsqueda debe realizarse en el índice logs-2023.01 en ambos cluster_one y cluster_two.
  • "query": { "match": { "message": "Elasticsearch" } }: Esta parte de la consulta especifica que estamos buscando documentos donde el campo message contiene el término "Elasticsearch".

Consideraciones de Rendimiento

Al realizar Búsquedas entre Clústeres, es importante considerar lo siguiente:

  • Latencia de Red: Buscar a través de clústeres ubicados en diferentes regiones geográficas puede introducir latencia de red.
  • Utilización de Recursos: Las Búsquedas entre Clústeres pueden ser intensivas en recursos. Asegúrate de que tus clústeres estén adecuadamente provisionados para manejar la carga.
  • Seguridad: Asegúrate de que la comunicación segura (por ejemplo, TLS) esté configurada entre clústeres para proteger los datos en tránsito.

Conclusión

La Búsqueda entre Clústeres en Elasticsearch es una característica poderosa que te permite buscar a través de múltiples clústeres sin problemas. Al configurar clústeres remotos y usar la sintaxis de consulta adecuada, puedes aprovechar esta capacidad para gestionar y buscar datos distribuidos de manera efectiva. Siempre considera las implicaciones de rendimiento y seguridad al configurar la Búsqueda entre Clústeres para asegurar operaciones óptimas y seguras.
