Acceso a Bases de Datos en F#
Visualización de Datos ►

Acceder a bases de datos es una habilidad crucial para cualquier programador. En esta sección, cubriremos cómo interactuar con bases de datos usando F#. Comenzaremos con lo básico y gradualmente avanzaremos a temas más complejos.

Introducción al Acceso a Bases de Datos

  • ¿Qué es una Base de Datos?
    • Una base de datos es una colección estructurada de datos que puede ser fácilmente accesible, gestionada y actualizada.
  • ¿Por qué Usar Bases de Datos?
    • Las bases de datos proporcionan una manera sistemática de almacenar, recuperar y gestionar datos de manera eficiente.
  • Tipos de Bases de Datos:
    • Bases de Datos Relacionales (por ejemplo, SQL Server, PostgreSQL)
    • Bases de Datos NoSQL (por ejemplo, MongoDB, Redis)

Configuración de tu Entorno

Antes de comenzar a programar, asegúrate de tener lo siguiente:

  • F# instalado en tu máquina.
  • Un servidor de base de datos (por ejemplo, SQL Server, PostgreSQL) instalado y en funcionamiento.
  • Un IDE o editor de texto (por ejemplo, Visual Studio, JetBrains Rider).

Conectándose a una Base de Datos

Usando ADO.NET

ADO.NET es un conjunto de clases que exponen servicios de acceso a datos para programadores del .NET Framework. Aquí se muestra cómo conectarse a una base de datos SQL Server usando ADO.NET en F#.

Ejemplo: Conectándose a SQL Server

open System.Data.SqlClient

let connectionString = "Server=your_server;Database=your_database;User Id=your_username;Password=your_password;"

let connectToDatabase() =
    use connection = new SqlConnection(connectionString)
    connection.Open()
    printfn "Conexión abierta exitosamente"
    // Realizar operaciones de base de datos aquí
    connection.Close()
    printfn "Conexión cerrada exitosamente"

connectToDatabase()
  • Explicación:
    • SqlConnection se usa para establecer una conexión con la base de datos.
    • connectionString contiene la información necesaria para conectarse a la base de datos.
    • connection.Open() abre la conexión.
    • connection.Close() cierra la conexión.

Ejecutando Comandos SQL

Ejecutando Consultas

Una vez conectado, puedes ejecutar comandos SQL para interactuar con la base de datos.

Ejemplo: Ejecutando una Consulta SELECT

let executeSelectQuery() =
    use connection = new SqlConnection(connectionString)
    connection.Open()
    let command = new SqlCommand("SELECT * FROM your_table", connection)
    use reader = command.ExecuteReader()
    while reader.Read() do
        printfn "Columna1: %s, Columna2: %s" (reader.GetString(0)) (reader.GetString(1))
    connection.Close()

executeSelectQuery()
  • Explicación:
    • SqlCommand se usa para ejecutar una consulta SQL.
    • ExecuteReader ejecuta el comando y devuelve un SqlDataReader para leer los resultados.
    • reader.Read() lee cada fila del conjunto de resultados.

Insertando Datos

Ejemplo: Ejecutando un Comando INSERT

let executeInsertCommand() =
    use connection = new SqlConnection(connectionString)
    connection.Open()
    let command = new SqlCommand("INSERT INTO your_table (Column1, Column2) VALUES (@value1, @value2)", connection)
    command.Parameters.AddWithValue("@value1", "SampleValue1")
    command.Parameters.AddWithValue("@value2", "SampleValue2")
    let rowsAffected = command.ExecuteNonQuery()
    printfn "%d filas insertadas" rowsAffected
    connection.Close()

executeInsertCommand()
  • Explicación:
    • Parameters.AddWithValue se usa para agregar parámetros al comando SQL para prevenir inyecciones SQL.
    • ExecuteNonQuery ejecuta el comando y devuelve el número de filas afectadas.

Temas Avanzados

Usando Entity Framework Core

Entity Framework Core (EF Core) es un ORM (Object-Relational Mapper) que simplifica el acceso a bases de datos al permitirte trabajar con datos como objetos.

Ejemplo: Configurando EF Core

  1. Instalar Paquetes de EF Core:

    dotnet add package Microsoft.EntityFrameworkCore
dotnet add package Microsoft.EntityFrameworkCore.SqlServer

  2. Definir un Modelo:

    open Microsoft.EntityFrameworkCore

type YourEntity() =
    [<Key>]
    member val Id = 0 with get, set
    member val Column1 = "" with get, set
    member val Column2 = "" with get, set

type YourDbContext() =
    inherit DbContext()
    [<DefaultValue>]
    val mutable yourEntities : DbSet<YourEntity>
    member this.YourEntities with get() = this.yourEntities and set v = this.yourEntities <- v

    override this.OnConfiguring(optionsBuilder: DbContextOptionsBuilder) =
        optionsBuilder.UseSqlServer("Server=your_server;Database=your_database;User Id=your_username;Password=your_password;") |> ignore

  3. Realizando Operaciones CRUD:

    let addEntity() =
    use context = new YourDbContext()
    let entity = YourEntity(Column1 = "SampleValue1", Column2 = "SampleValue2")
    context.YourEntities.Add(entity) |> ignore
    context.SaveChanges() |> ignore
    printfn "Entidad agregada exitosamente"

addEntity()
  • Explicación:
    • DbContext es la clase principal para interactuar con la base de datos usando EF Core.
    • DbSet representa una colección de entidades.
    • El método OnConfiguring se usa para configurar la conexión a la base de datos.
    • El método Add agrega una nueva entidad al contexto.
    • El método SaveChanges guarda los cambios en la base de datos.

Conclusión

En esta sección, cubrimos los conceptos básicos del acceso a bases de datos en F#, incluyendo la conexión a una base de datos, la ejecución de comandos SQL y el uso de Entity Framework Core para escenarios más avanzados. Al dominar estas técnicas, puedes interactuar eficientemente con bases de datos en tus aplicaciones F#. Practica estos ejemplos y explora características más avanzadas para profundizar tu comprensión.
