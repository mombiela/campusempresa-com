Introducción

La configuración de equipos de red es una habilidad fundamental para cualquier profesional de TI. Este tema abarca los conceptos esenciales y las prácticas recomendadas para configurar routers, switches y otros dispositivos de red. Aprenderás a configurar estos equipos para asegurar una comunicación eficiente y segura dentro de una infraestructura TI.

Conceptos Clave

  • Router: Dispositivo que dirige el tráfico de datos entre diferentes redes.
  • Switch: Dispositivo que conecta múltiples dispositivos dentro de la misma red y dirige el tráfico de datos entre ellos.
  • Firewall: Dispositivo o software que controla el acceso a la red y protege contra amenazas externas.
  • VLAN (Virtual LAN): Segmentación lógica de una red física en múltiples redes virtuales.
  • IP Addressing: Asignación de direcciones IP a dispositivos para identificarlos en la red.
  • Subnetting: Técnica para dividir una red IP en subredes más pequeñas.

Configuración de Routers

Configuración Básica

  1. Acceso al Router:

    • Conéctate al router mediante consola, SSH o Telnet.
    • Ejemplo de acceso por consola: 
      console> enable
console# configure terminal

  2. Asignación de Direcciones IP:

    • Configura la interfaz del router con una dirección IP. 
      interface GigabitEthernet0/0
ip address 192.168.1.1 255.255.255.0
no shutdown

  3. Configuración de Rutas Estáticas:

    • Añade rutas estáticas para dirigir el tráfico a redes específicas. 
      ip route 192.168.2.0 255.255.255.0 192.168.1.2

Configuración Avanzada

  1. Configuración de NAT (Network Address Translation):

    • Permite que múltiples dispositivos en una red local compartan una única dirección IP pública. 
      ip nat inside source list 1 interface GigabitEthernet0/0 overload
access-list 1 permit 192.168.1.0 0.0.0.255

  2. Configuración de DHCP (Dynamic Host Configuration Protocol):

    • Asigna automáticamente direcciones IP a dispositivos en la red. 
      ip dhcp pool LAN
network 192.168.1.0 255.255.255.0
default-router 192.168.1.1
dns-server 8.8.8.8

Configuración de Switches

Configuración Básica

  1. Acceso al Switch:

    • Conéctate al switch mediante consola, SSH o Telnet.
    • Ejemplo de acceso por consola: 
      console> enable
console# configure terminal

  2. Configuración de VLANs:

    • Crea y asigna VLANs a puertos específicos. 
      vlan 10
name Sales
interface GigabitEthernet0/1
switchport mode access
switchport access vlan 10

  3. Configuración de Trunking:

    • Permite que múltiples VLANs pasen a través de un solo enlace. 
      interface GigabitEthernet0/24
switchport mode trunk
switchport trunk allowed vlan 10,20

Configuración Avanzada

  1. Configuración de STP (Spanning Tree Protocol):

    • Previene bucles en la red. 
      spanning-tree mode rapid-pvst
spanning-tree vlan 10 priority 4096

  2. Configuración de Port Security:

    • Limita el número de direcciones MAC que pueden conectarse a un puerto. 
      interface GigabitEthernet0/1
switchport port-security
switchport port-security maximum 2
switchport port-security violation restrict
switchport port-security mac-address sticky

Comparación de Dispositivos de Red

| Dispositivo | Función Principal | Ejemplo de Configuración | |-------------|-------------------|--------------------------| | Router | Dirige tráfico entre redes | ip route 192.168.2.0 255.255.255.0 192.168.1.2 | | Switch | Conecta dispositivos dentro de una red | switchport access vlan 10 | | Firewall | Controla acceso y protege la red | access-list 100 permit ip any any |

Conclusión

La configuración de equipos de red es esencial para asegurar una comunicación eficiente y segura dentro de una infraestructura TI. Desde la configuración básica de routers y switches hasta la implementación de características avanzadas como NAT, DHCP, VLANs y STP, cada paso es crucial para mantener una red robusta y funcional. Practicar estos conceptos y configuraciones te permitirá gestionar y optimizar redes de manera efectiva.
