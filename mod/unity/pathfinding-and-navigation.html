Introducción

La búsqueda de caminos y la navegación son aspectos cruciales del desarrollo de juegos, especialmente en la creación de personajes de IA inteligentes y receptivos. En Unity, estos conceptos se implementan utilizando el sistema NavMesh, que permite a los personajes navegar por el mundo del juego de manera eficiente.

Conceptos Clave

  • NavMesh: Una malla de navegación que representa las superficies transitables en el mundo del juego.
  • NavMesh Agent: Un componente que permite a los personajes navegar por el NavMesh.
  • NavMesh Obstacles: Objetos que pueden afectar dinámicamente el NavMesh, haciendo que los agentes naveguen alrededor de ellos.
  • Algoritmos de Búsqueda de Caminos: Técnicas utilizadas para encontrar el camino más corto de un punto a otro.

Configuración de NavMesh

Creación de un NavMesh

  1. Hornear el NavMesh:
    • Ve a Window > AI > Navigation.
    • Selecciona la pestaña Bake.
    • Ajusta configuraciones como Agent Radius, Agent Height, y Max Slope.
    • Haz clic en Bake para generar el NavMesh.
// Ejemplo: Hornear NavMesh en código (uso avanzado)
using UnityEngine;
using UnityEngine.AI;

public class NavMeshBaker : MonoBehaviour
{
    void Start()
    {
        NavMeshSurface surface = GetComponent<NavMeshSurface>();
        surface.BuildNavMesh();
    }
}
  1. Componente NavMesh Surface:
    • Adjunta el componente NavMesh Surface a un GameObject.
    • Configura el componente y haz clic en Bake.

Añadir Agentes NavMesh

  1. Adjuntar NavMesh Agent:

    • Selecciona el GameObject del personaje.
    • Añade el componente NavMesh Agent.

  2. Configurar NavMesh Agent:

    • Establece propiedades como Speed, Angular Speed, y Stopping Distance.
// Ejemplo: Mover un NavMesh Agent a una posición objetivo
using UnityEngine;
using UnityEngine.AI;

public class AgentMover : MonoBehaviour
{
    public Transform target;

    void Start()
    {
        NavMeshAgent agent = GetComponent<NavMeshAgent>();
        agent.SetDestination(target.position);
    }
}

Navegación Dinámica

NavMesh Obstacles

  • Obstáculos Estáticos: Objetos que no se mueven pero afectan el NavMesh.
  • Obstáculos Dinámicos: Objetos que se mueven y actualizan dinámicamente el NavMesh.
// Ejemplo: Añadir un NavMesh Obstacle
using UnityEngine;

public class ObstacleCreator : MonoBehaviour
{
    void Start()
    {
        GameObject obstacle = GameObject.CreatePrimitive(PrimitiveType.Cube);
        obstacle.AddComponent<NavMeshObstacle>();
        obstacle.GetComponent<NavMeshObstacle>().carving = true;
    }
}

Off-Mesh Links

  • Enlaces Manuales: Enlaces colocados manualmente para saltos, escaladas, etc.
  • Enlaces Automáticos: Enlaces generados automáticamente para una navegación fluida.
// Ejemplo: Crear un Off-Mesh Link
using UnityEngine;
using UnityEngine.AI;

public class OffMeshLinkCreator : MonoBehaviour
{

Conclusión

La búsqueda de caminos y la navegación son esenciales para crear una IA inteligente y receptiva en Unity. Al comprender y utilizar NavMesh, NavMesh Agents y técnicas avanzadas de búsqueda de caminos, puedes crear sistemas de navegación complejos y dinámicos en tus juegos.
