Introducción a las Mónadas

Las mónadas son una forma poderosa y flexible de manejar cálculos en Haskell. Proporcionan una manera de estructurar programas de manera genérica. Esta sección cubrirá tres mónadas comunes: Maybe, Either y List.

Mónada Maybe

La mónada Maybe se utiliza para representar cálculos que podrían fallar. Encapsula un valor opcional.

Definición

data Maybe a = Nothing | Just a

Conceptos Clave

  • Nothing: Representa un fallo o ausencia de valor.
  • Just a: Representa un cálculo exitoso con un valor a.

Ejemplo de Uso

safeDivide :: Double -> Double -> Maybe Double
safeDivide _ 0 = Nothing
safeDivide x y = Just (x / y)

Instancia de Mónada

instance Monad Maybe where
    return = Just
    Nothing >>= _ = Nothing
    Just x >>= f = f x

Ejemplo con Notación `do`

maybeExample :: Maybe Double
maybeExample = do
    x <- Just 10
    y <- Just 2
    safeDivide x y

Mónada Either

La mónada Either se utiliza para cálculos que pueden fallar con un mensaje de error. Encapsula un valor que puede ser de dos tipos.

Definición

data Either a b = Left a | Right b

Conceptos Clave

  • Left a: Representa un error con un valor a.
  • Right b: Representa un cálculo exitoso con un valor b.

Ejemplo de Uso

safeDivide :: Double -> Double -> Either String Double
safeDivide _ 0 = Left "Division by zero"
safeDivide x y = Right (x / y)

Instancia de Mónada

instance Monad (Either e) where
    return = Right
    Left l >>= _ = Left l
    Right r >>= f = f r

Ejemplo con Notación `do`

eitherExample :: Either String Double
eitherExample = do
    x <- Right 10
    y <- Right 2
    safeDivide x y

Mónada List

La mónada List se utiliza para cálculos que pueden devolver múltiples resultados. Encapsula una lista de valores.

Definición

data [] a = [] | a : [a]

Conceptos Clave

  • []: Representa una lista vacía.
  • a : [a]: Representa una lista con cabeza a y cola [a].

Ejemplo de Uso

pairs :: [a] -> [b] -> [(a, b)]
pairs xs ys = [(x, y) | x <- xs, y <- ys]

Instancia de Mónada

instance Monad [] where
    return x = [x]
    xs >>= f = concat (map f xs)

Ejemplo con Notación `do`

listExample :: [(Int, Char)]
listExample = do
    x <- [1, 2, 3]
    y <- ['a', 'b']
    return (x, y)

Comparación de Mónadas

Mónada Propósito Tipo de Éxito Tipo de Fallo
Maybe Valor opcional Just a Nothing
Either Cálculo con mensaje de error Right b Left a
List Múltiples resultados [a] [] (lista vacía)

Conclusión

Mónadas como Maybe, Either y List proporcionan abstracciones poderosas para manejar diferentes tipos de cálculos en Haskell. Entender estas mónadas y su uso puede mejorar enormemente tu capacidad para escribir programas robustos y flexibles en Haskell. Practica usando estas mónadas en varios escenarios para familiarizarte con sus patrones e idioms.
