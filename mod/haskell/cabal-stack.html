Introducción a las Herramientas de Construcción en Haskell

Haskell, como muchos lenguajes de programación, tiene su propio conjunto de herramientas para gestionar dependencias, construir proyectos y manejar la distribución de paquetes. Las dos principales herramientas de construcción en el ecosistema de Haskell son Cabal y Stack. Entender estas herramientas es crucial para un desarrollo eficiente en Haskell.

Cabal

Cabal es un sistema para construir y empaquetar bibliotecas y programas en Haskell. Es parte de la Plataforma Haskell y ha existido durante mucho tiempo.

Conceptos Clave de Cabal

  • Archivo de Descripción del Paquete: Cada paquete de Haskell tiene un archivo .cabal que describe el paquete, sus dependencias y otros metadatos.
  • Hackage: El repositorio central de paquetes para bibliotecas de Haskell.
  • Sandboxing: Una forma de aislar las dependencias de un proyecto para evitar conflictos.

Comandos Básicos de Cabal

  • Inicializar un nuevo proyecto:

    cabal init

    Este comando crea un nuevo archivo .cabal y configura la estructura del proyecto.

  • Construir un proyecto:

    cabal build

    Este comando compila el proyecto y sus dependencias.

  • Ejecutar un proyecto:

    cabal run

    Este comando construye y ejecuta el proyecto.

  • Instalar dependencias:

    cabal install --only-dependencies

    Este comando instala todas las dependencias listadas en el archivo .cabal.

Ejemplo: Creando un Proyecto Simple con Cabal

  1. Inicializar el proyecto:

    cabal init

    Sigue las indicaciones para configurar el proyecto.

  2. Editar el archivo .cabal:

    name:                my-project
version:             0.1.0.0
build-type:          Simple
cabal-version:       >=1.10

library
  exposed-modules:     MyLib
  build-depends:       base >=4.7 && <5
  hs-source-dirs:      src
  default-language:    Haskell2010

  3. Crear el directorio de fuentes y un módulo simple:

    mkdir src
echo "module MyLib where\n\nhello :: String\nhello = \"Hello, World!\"" > src/MyLib.hs

  4. Construir y ejecutar el proyecto:

    cabal build
cabal repl

Stack

Stack es una herramienta de construcción más reciente que tiene como objetivo proporcionar una experiencia más amigable para el usuario y una mejor gestión de dependencias.

Conceptos Clave de Stack

  • Stackage: Un conjunto curado de paquetes que se sabe que funcionan bien juntos.
  • Snapshots: Conjuntos específicos de versiones de paquetes que se prueban juntos.
  • Configuración del Proyecto: Stack utiliza un archivo stack.yaml para configurar el proyecto.

Comandos Básicos de Stack

  • Inicializar un nuevo proyecto:

    stack new my-project

    Este comando crea un nuevo proyecto con una plantilla predeterminada.

  • Construir un proyecto:

    stack build

    Este comando compila el proyecto y sus dependencias.

  • Ejecutar un proyecto:

    stack run

    Este comando construye y ejecuta el proyecto.

  • Instalar dependencias:

    stack setup

    Este comando configura el compilador e instala dependencias.

Ejemplo: Creando un Proyecto Simple con Stack

  1. Inicializar el proyecto:

    stack new my-project
cd my-project

  2. Editar el archivo stack.yaml:

    resolver: lts-18.18

packages:
- .
extra-deps: []

  3. Editar el archivo package.yaml:

    name: my-project
version: 0.1.0.0
dependencies:
- base >= 4.7 && < 5

library:
  source-dirs: src
  exposed-modules: MyLib

  4. Crear el directorio de fuentes y un módulo simple:

    mkdir src
echo "module MyLib where\n\nhello :: String\nhello = \"Hello, World!\"" > src/MyLib.hs

  5. Construir y ejecutar el proyecto:

    stack build
stack ghci

Comparación de Cabal y Stack

Característica Cabal Stack
Repositorio de Paquetes Hackage Stackage
Gestión de Dependencias Manual, con sandboxing Automática, con snapshots curados
Archivo de Configuración Archivo .cabal stack.yaml y package.yaml
Facilidad de Uso Requiere más configuración manual Más amigable para el usuario, especialmente para principiantes
Aislamiento Sandboxing (opcional) Siempre aislado

Conclusión

Tanto Cabal como Stack son herramientas poderosas para gestionar proyectos en Haskell, cada una con sus propias fortalezas. Cabal ofrece flexibilidad y ha sido un pilar en la comunidad de Haskell durante años, mientras que Stack proporciona una experiencia más simplificada y amigable para el usuario, especialmente para principiantes. Entender ambas herramientas te dará una base sólida para gestionar proyectos en Haskell de manera efectiva.
