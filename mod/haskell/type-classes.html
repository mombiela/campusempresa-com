Introducción a las Clases de Tipos

Las clases de tipos en Haskell son una característica poderosa que te permite definir funciones que pueden operar en una variedad de tipos diferentes. Son similares a las interfaces en otros lenguajes, pero más flexibles y poderosas.

  • Clase de Tipo: Una colección de tipos que soportan ciertas operaciones.
  • Instancia: Una implementación específica de una clase de tipo para un tipo particular.

Ejemplo Básico: La Clase de Tipo `Eq`

La clase de tipo Eq se usa para tipos que soportan pruebas de igualdad.

class Eq a where
    (==) :: a -> a -> Bool
    (/=) :: a -> a -> Bool
  • class Eq a where: Define una clase de tipo Eq con una variable de tipo a.
  • (==) y (/=): Funciones que deben ser implementadas para cualquier tipo que sea una instancia de Eq.

Creando Instancias

Para hacer que un tipo sea una instancia de una clase de tipo, proporcionas implementaciones para las funciones requeridas.

data Color = Red | Green | Blue

instance Eq Color where
    Red == Red = True
    Green == Green = True
    Blue == Blue = True
    _ == _ = False
  • data Color = Red | Green | Blue: Define un nuevo tipo de datos Color.
  • instance Eq Color where: Hace que Color sea una instancia de Eq definiendo cómo funciona == para Color.

Clases de Tipos Comunes

Aquí hay algunas de las clases de tipos más comúnmente usadas en Haskell:

| Clase de Tipo | Descripción | |---------------|--------------| | Eq | Pruebas de igualdad | | Ord | Operaciones de comparación | | Show | Conversión a una cadena | | Read | Análisis desde una cadena | | Num | Operaciones numéricas |

La Clase de Tipo `Ord`

La clase de tipo Ord es para tipos que tienen un orden.

class Eq a => Ord a where
    compare :: a -> a -> Ordering
    (<) :: a -> a -> Bool
    (<=) :: a -> a -> Bool
    (>) :: a -> a -> Bool
    (>=) :: a -> a -> Bool
  • class Eq a => Ord a where: Ord es una subclase de Eq, lo que significa que cualquier tipo que sea una instancia de Ord también debe ser una instancia de Eq.

Ejemplo: Instancia `Ord` para `Color`

instance Ord Color where
    compare Red Red = EQ
    compare Red _ = LT
    compare _ Red = GT
    compare Green Green = EQ
    compare Green _ = LT
    compare _ Green = GT
    compare Blue Blue = EQ
  • compare: Define cómo comparar dos valores Color.

Clases de Tipos Avanzadas

Functor

La clase de tipo Functor se usa para tipos que pueden ser mapeados.

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • class Functor f where: Define una clase de tipo Functor con una variable de tipo f.
  • fmap: Una función que aplica una función a cada elemento en una estructura.

Ejemplo: Instancia `Functor` para `Maybe`

instance Functor Maybe where
    fmap _ Nothing = Nothing
    fmap f (Just x) = Just (f x)
  • instance Functor Maybe where: Hace que Maybe sea una instancia de Functor.
  • fmap _ Nothing = Nothing: Si el valor es Nothing, fmap devuelve Nothing.
  • fmap f (Just x) = Just (f x): Si el valor es Just x, fmap aplica f a x.

Ejercicios

  1. Crea una instancia de Eq para un tipo de datos personalizado Shape con constructores Circle, Square y Triangle.
  2. Implementa la clase de tipo Ord para el tipo Shape.
  3. Define una instancia de Functor para un tipo de datos personalizado Box que envuelve un solo valor.

Conclusión

Las clases de tipos en Haskell proporcionan una forma de definir interfaces genéricas que pueden ser implementadas por diferentes tipos. Son una piedra angular del sistema de tipos de Haskell, permitiendo polimorfismo y reutilización de código. Al entender y usar clases de tipos, puedes escribir código más flexible y reutilizable.
