Introducción a los Funtores Aplicativos

Los Funtores Aplicativos son una característica poderosa en Haskell que permiten la aplicación de funciones elevadas sobre un contexto computacional. Son más poderosos que los Funtores pero menos poderosos que los Monads. Entender los Funtores Aplicativos es crucial para escribir código Haskell conciso y expresivo.

Conceptos Clave

  • Functor: Una clase de tipo que permite aplicar una función a valores envueltos en un contexto.
  • Functor Aplicativo: Extiende Functor permitiendo que las funciones que están envueltas en un contexto se apliquen a valores envueltos en un contexto.
  • Pure: Una función que toma un valor y lo devuelve envuelto en un contexto.
  • <*> (Apply): Un operador que aplica una función envuelta en un contexto a un valor envuelto en un contexto.

Repaso de Functor

Antes de sumergirnos en los Funtores Aplicativos, repasemos rápidamente los Funtores.

class Functor f where
    fmap :: (a -> b) -> f a -> f b

Ejemplo con Maybe:

fmap (+1) (Just 2) -- Resultado: Just 3
fmap (+1) Nothing  -- Resultado: Nothing

Conceptos Básicos de Functor Aplicativo

La clase de tipo Applicative se define de la siguiente manera:

class Functor f => Applicative f where
    pure :: a -> f a
    (<*>) :: f (a -> b) -> f a -> f b

Pure

La función pure toma un valor y lo envuelve en un contexto.

pure 3 :: Maybe Int -- Resultado: Just 3
pure 3 :: [Int]     -- Resultado: [3]

Apply (<*>)

El operador <*> aplica una función envuelta en un contexto a un valor envuelto en un contexto.

(Just (+1)) <*> (Just 2) -- Resultado: Just 3
Nothing <*> (Just 2)     -- Resultado: Nothing
(Just (+1)) <*> Nothing  -- Resultado: Nothing

Ejemplo con Listas

Las listas también son Funtores Aplicativos.

pure (+1) <*> [1, 2, 3] -- Resultado: [2, 3, 4]
[(+1), (*2)] <*> [1, 2, 3] -- Resultado: [2, 3, 4, 2, 4, 6]

Combinando Funtores Aplicativos

Los Funtores Aplicativos pueden combinarse para aplicar funciones a múltiples valores envueltos.

pure (+) <*> Just 3 <*> Just 5 -- Resultado: Just 8
pure (+) <*> Just 3 <*> Nothing -- Resultado: Nothing

Ejemplo con Funciones

Las funciones en sí mismas pueden ser Funtores Aplicativos.

pure (+) <*> (+3) <*> (*2) $ 5 -- Resultado: 16
-- Explicación: ((+3) 5) + ((*2) 5) = 8 + 10 = 18

Ejemplo Práctico: Validación

Los Funtores Aplicativos son útiles en escenarios como la validación donde múltiples cálculos necesitan combinarse.

import Control.Applicative

data Validation e a = Failure e | Success a deriving (Show)

instance Functor (Validation e) where
    fmap _ (Failure e) = Failure e
    fmap f (Success a) = Success (f a)

instance Monoid e => Applicative (Validation e) where
    pure = Success
    (Failure e1) <*> (Failure e2) = Failure (e1 <> e2)
    (Failure e) <*> _ = Failure e
    _ <*> (Failure e) = Failure e
    (Success f) <*> (Success a) = Success (f a)

validateName :: String -> Validation [String] String
validateName name
    | null name = Failure ["El nombre no puede estar vacío"]
    | otherwise = Success name

validateAge :: Int -> Validation [String] Int
validateAge age
    | age <= 0 = Failure ["La edad debe ser positiva"]
    | otherwise = Success age

validatePerson :: String -> Int -> Validation [String] (String, Int)
validatePerson name age = (,) <$> validateName name <*> validateAge age

-- Ejemplo de uso:
-- validatePerson "" 0
-- Resultado: Failure ["El nombre no puede estar vacío","La edad debe ser positiva"]
-- validatePerson "Alice" 30
-- Resultado: Success ("Alice", 30)

Conclusión

Los Funtores Aplicativos proporcionan un punto intermedio entre los Funtores y los Monads, permitiendo un código más flexible y expresivo. Son particularmente útiles para escenarios donde múltiples cálculos independientes necesitan combinarse, como la validación. Entender los Funtores Aplicativos mejorará tu capacidad para escribir código Haskell conciso y poderoso.
