Introducción a Currying y Aplicación Parcial

Currying y la aplicación parcial son conceptos fundamentales en Haskell que permiten un código más flexible y modular. Permiten que las funciones se apliquen en etapas, lo que puede llevar a un código más legible y reutilizable.

Conceptos Clave

  • Currying: El proceso de transformar una función que toma múltiples argumentos en una secuencia de funciones, cada una tomando un solo argumento.
  • Aplicación Parcial: El proceso de fijar algunos argumentos de una función, produciendo otra función de menor aridad.

Entendiendo Currying

Currying lleva el nombre del matemático Haskell Curry. En Haskell, todas las funciones son curried por defecto.

Ejemplo de Currying

Considera una función simple que suma dos números:

add :: Int -> Int -> Int
add x y = x + y

Esta función puede ser curried:

addCurried :: Int -> (Int -> Int)
addCurried x = \y -> x + y

En Haskell, add y addCurried son esencialmente lo mismo debido al currying inherente de Haskell.

Usando Funciones Curried

Puedes aplicar argumentos uno a la vez:

let addFive = add 5
addFive 3  -- Resultado: 8

Aquí, add 5 devuelve una nueva función que suma 5 a su argumento.

Aplicación Parcial

La aplicación parcial está estrechamente relacionada con currying. Involucra aplicar una función a algunos de sus argumentos, produciendo otra función que toma los argumentos restantes.

Ejemplo de Aplicación Parcial

Usando la misma función add:

let addTen = add 10
addTen 7  -- Resultado: 17

Aquí, add 10 es una función parcialmente aplicada que suma 10 a cualquier número dado.

Casos de Uso Prácticos

La aplicación parcial es útil para crear funciones especializadas a partir de otras más generales:

multiply :: Int -> Int -> Int
multiply x y = x * y

let double = multiply 2
double 8  -- Resultado: 16

let triple = multiply 3
triple 5  -- Resultado: 15

Comparación de Currying y Aplicación Parcial

| Concepto | Descripción | Ejemplo | |--------------------|-----------------------------------------------------------------------------|----------------------------------------------| | Currying | Transformar una función con múltiples argumentos en una secuencia de funciones | addCurried x = \y -> x + y | | Aplicación Parcial | Aplicar una función a algunos de sus argumentos para obtener una nueva función | let addFive = add 5 |

Uso Avanzado

Funciones de Orden Superior

Currying y la aplicación parcial brillan cuando se usan con funciones de orden superior:

map (add 3) [1, 2, 3]  -- Resultado: [4, 5, 6]

Aquí, add 3 se aplica parcialmente y luego se mapea sobre una lista.

Composición de Funciones

Puedes componer funciones curried para construir operaciones más complejas:

let increment = add 1
let double = multiply 2
let incrementThenDouble = double . increment

incrementThenDouble 3  -- Resultado: 8

Ejercicios

  1. Práctica de Currying: Escribe una función curried que concatene tres cadenas. 
    concatThree :: String -> String -> String -> String
concatThree a b c = a ++ b ++ c
  2. Práctica de Aplicación Parcial: Crea una función parcialmente aplicada a partir de una función curried que multiplica tres números. 
    multiplyThree :: Int -> Int -> Int -> Int
multiplyThree x y z = x * y * z

let multiplyByTwoAndThree = multiplyThree 2 3
multiplyByTwoAndThree 4  -- Resultado: 24

Conclusión

Currying y la aplicación parcial son técnicas poderosas en Haskell que mejoran la modularidad y reutilización del código. Al entender y utilizar estos conceptos, puedes escribir programas en Haskell más concisos y flexibles. Practica estas técnicas con varias funciones para volverte competente en su uso.
