Introducción

El estilo y formato del código son cruciales en la programación en Haskell para asegurar la legibilidad, mantenibilidad y colaboración. Esta sección cubrirá las mejores prácticas para escribir código Haskell limpio y consistente, desde reglas básicas de formato hasta directrices estilísticas avanzadas.

Reglas Básicas de Formato

Indentación

  • Consistencia: Usa una indentación consistente a lo largo de tu código. El estándar es de 2 o 4 espacios por nivel de indentación.
  • No Tabulaciones: Evita usar tabulaciones; usa espacios para la indentación.
-- Buena indentación
main :: IO ()
main = do
  putStrLn "¡Hola, Mundo!"
  putStrLn "¡Bienvenido a Haskell!"

-- Mala indentación
main :: IO ()
main = do
 putStrLn "¡Hola, Mundo!"
  putStrLn "¡Bienvenido a Haskell!"

Longitud de Línea

  • Limitar Longitud de Línea: Mantén las líneas con un máximo de 80 caracteres para mejorar la legibilidad.
  • División de Líneas: Si una línea excede los 80 caracteres, divídela en múltiples líneas.
-- Buena longitud de línea
longFunctionName :: Int -> Int -> Int -> Int
longFunctionName x y z = x + y + z

-- Mala longitud de línea
longFunctionName :: Int -> Int -> Int -> Int = x + y + z

Espacios en Blanco

  • Alrededor de Operadores: Usa espacios alrededor de los operadores para una mejor legibilidad.
  • Dentro de Paréntesis: Evita espacios dentro de los paréntesis.
-- Buen uso de espacios en blanco
sum = a + b * c

-- Mal uso de espacios en blanco
sum=a+b*c

Convenciones de Nombres

Variables y Funciones

  • Camel Case: Usa camelCase para nombres de variables y funciones.
  • Nombres Descriptivos: Elige nombres descriptivos que transmitan el propósito de la variable o función.
-- Buen nombramiento
calculateSum :: Int -> Int -> Int
calculateSum firstNumber secondNumber = firstNumber + secondNumber

-- Mal nombramiento
cs :: Int -> Int -> Int
cs x y = x + y

Tipos y Constructores

  • Pascal Case: Usa PascalCase para nombres de tipos y constructores.
-- Buen nombramiento
data Person = Person { firstName :: String, lastName :: String }

-- Mal nombramiento
data person = person { firstname :: String, lastname :: String }

Comentarios

Comentarios en Línea

  • Propósito: Usa comentarios en línea para explicar lógica compleja o detalles importantes.
  • Ubicación: Coloca los comentarios en la misma línea o encima del código que describen.
-- Buen comentario en línea
factorial :: Int -> Int
factorial n
  | n == 0    = 1  -- Caso base: 0! es 1
  | otherwise = n * factorial (n - 1)

-- Mal comentario en línea
factorial :: Int -> Int
factorial n
  | n == 0 = 1 -- Caso base: 0! es 1
  | otherwise = n * factorial (n - 1)

Comentarios en Bloque

  • Uso: Usa comentarios en bloque para grandes secciones de código o documentación a nivel de módulo.
  • Formato: Inicia y termina los comentarios en bloque con {- y -}.
{- 
  Esta función calcula el factorial de un número.
  Usa recursión para computar el resultado.
-}
factorial :: Int -> Int
factorial n
  | n == 0    = 1
  | otherwise = n * factorial (n - 1)

Directrices Estilísticas Avanzadas

Composición de Funciones

  • Usar Composición: Prefiere la composición de funciones sobre llamadas a funciones anidadas para mayor claridad.
-- Buena composición de funciones
processData = map toUpper . filter isLetter

-- Malas llamadas a funciones anidadas
processData xs = map toUpper (filter isLetter xs)

Estilo Point-Free

  • Point-Free: Usa el estilo point-free donde sea apropiado para hacer el código más conciso.
-- Buen estilo point-free
sumSquares = sum . map (^2)

-- Mal estilo point-free
sumSquares xs = sum (map (^2) xs)

Conclusión

Adherirse a las directrices de estilo y formato del código en Haskell es esencial para escribir código limpio, legible y mantenible. Siguiendo una indentación consistente, convenciones de nombres y prácticas de comentarios, puedes mejorar la calidad de tus programas en Haskell y facilitar la colaboración con otros desarrolladores. Recuerda usar técnicas avanzadas como la composición de funciones y el estilo point-free con prudencia para mejorar la claridad y concisión del código.
