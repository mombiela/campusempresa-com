Introducción

En Haskell, las operaciones de Entrada/Salida (E/S) se manejan de manera diferente en comparación con los lenguajes imperativos. Haskell es un lenguaje puramente funcional, lo que significa que las funciones no tienen efectos secundarios. Sin embargo, las operaciones de E/S inherentemente tienen efectos secundarios, por lo que Haskell utiliza un tipo especial llamado IO para gestionar estas operaciones. Esta sección cubrirá los conceptos básicos de E/S en Haskell, incluyendo la lectura y escritura en la consola, el trabajo con archivos y la comprensión del tipo IO.

Conceptos clave

  • Funciones puras vs. Acciones de E/S: Comprender la diferencia entre funciones puras y acciones de E/S.
  • El tipo IO: Introducción al tipo IO y cómo encapsula los efectos secundarios.
  • Funciones básicas de E/S: Uso de putStrLn, getLine, readFile y writeFile.
  • Combinación de acciones de E/S: Uso de la notación do para secuenciar acciones de E/S.

Funciones puras vs. Acciones de E/S

En Haskell, las funciones puras siempre producen la misma salida para la misma entrada y no tienen efectos secundarios. Las acciones de E/S, por otro lado, interactúan con el mundo exterior y pueden producir resultados diferentes cada vez que se ejecutan.

-- Ejemplo de función pura
add :: Int -> Int -> Int
add x y = x + y

-- Ejemplo de acción de E/S
main :: IO ()
main = do
    putStrLn "Introduce tu nombre:"
    name <- getLine
    putStrLn ("¡Hola, " ++ name ++ "!")

El tipo `IO`

El tipo IO se utiliza para representar acciones que realizan E/S. Una acción IO es un valor que, cuando se ejecuta, interactúa con el mundo exterior y produce un resultado.

-- Ejemplo de una acción IO
main :: IO ()
main = putStrLn "¡Hola, Mundo!"

En el ejemplo anterior, putStrLn "¡Hola, Mundo!" es una acción IO que imprime una cadena en la consola.

Funciones básicas de E/S

`putStrLn`

putStrLn se utiliza para imprimir una cadena seguida de una nueva línea en la consola.

main :: IO ()
main = putStrLn "¡Hola, Mundo!"

`getLine`

getLine lee una línea de entrada de la consola.

main :: IO ()
main = do
    putStrLn "Introduce tu nombre:"
    name <- getLine
    putStrLn ("¡Hola, " ++ name ++ "!")

`readFile` y `writeFile`

readFile lee el contenido de un archivo como una cadena, y writeFile escribe una cadena en un archivo.

main :: IO ()
main = do
    contents <- readFile "example.txt"
    putStrLn contents

writeExample :: IO ()
writeExample = do
    writeFile "example.txt" "Este es un archivo de ejemplo."

Combinación de acciones de E/S

Para realizar múltiples acciones de E/S en secuencia, utilizamos la notación do. Esto nos permite escribir una serie de acciones que se ejecutan en orden.

main :: IO ()
main = do
    putStrLn "Introduce el primer número:"
    num1 <- getLine
    putStrLn "Introduce el segundo número:"
    num2 <- getLine
    let sum = (read num1 :: Int) + (read num2 :: Int)
    putStrLn ("La suma es: " ++ show sum)

En este ejemplo, leemos dos números del usuario, calculamos su suma e imprimimos el resultado.

Conclusión

Comprender las operaciones básicas de E/S en Haskell es crucial para interactuar con el mundo exterior. El tipo IO encapsula los efectos secundarios, permitiendo que Haskell mantenga su pureza funcional. Al usar funciones como putStrLn, getLine, readFile y writeFile, y combinarlas con la notación do, puedes realizar operaciones de E/S complejas de una manera estructurada y legible.
