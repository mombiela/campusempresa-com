Introducción

Hackage y Stackage son dos componentes esenciales del ecosistema Haskell que ayudan a gestionar y distribuir bibliotecas y paquetes de Haskell. Comprender estas herramientas es crucial para los desarrolladores de Haskell, desde principiantes hasta usuarios avanzados.

Hackage

Hackage es el archivo central de paquetes para bibliotecas de Haskell. Aloja una vasta colección de paquetes de Haskell contribuidos por la comunidad.

Conceptos Clave

  • Repositorio de Paquetes: Hackage sirve como un repositorio donde los desarrolladores pueden subir y compartir sus paquetes de Haskell.
  • Cabal: El sistema de construcción y gestor de paquetes para Haskell, que interactúa con Hackage para descargar y gestionar dependencias.
  • Metadatos: Cada paquete en Hackage incluye metadatos como versión, dependencias y documentación.

Usando Hackage

Para usar Hackage, normalmente interactúas con él a través de la herramienta de línea de comandos cabal.

Ejemplo: Instalando un Paquete desde Hackage

cabal update
cabal install <nombre-del-paquete>

Explicación:

  1. cabal update actualiza la lista local de paquetes disponibles desde Hackage.
  2. cabal install <nombre-del-paquete> instala el paquete especificado.

Publicando un Paquete en Hackage

Para publicar tu propio paquete en Hackage, necesitas seguir estos pasos:

  1. Prepara tu paquete: Asegúrate de que tu paquete tenga un archivo .cabal con todos los metadatos necesarios.
  2. Construye y prueba: Usa cabal build y cabal test para asegurarte de que tu paquete se construya y funcione correctamente.
  3. Sube: Usa cabal upload para subir tu paquete a Hackage.

Ejemplo: Subiendo un Paquete

cabal sdist
cabal upload dist/<nombre-del-paquete-versión>.tar.gz

Explicación:

  1. cabal sdist crea una distribución de fuente de tu paquete.
  2. cabal upload sube el archivo tarball generado a Hackage.

Stackage

Stackage es una fuente estable de paquetes de Haskell que proporciona conjuntos curados de paquetes que se sabe que funcionan bien juntos.

Conceptos Clave

  • LTS (Soporte a Largo Plazo): Stackage proporciona lanzamientos LTS, que son instantáneas estables de paquetes de Haskell.
  • Estabilidad: Stackage asegura que los paquetes en una instantánea sean compatibles entre sí, reduciendo problemas de dependencias.
  • Stack: Una herramienta de construcción que trabaja con Stackage para gestionar las dependencias del proyecto.

Usando Stackage

Para usar Stackage, normalmente interactúas con él a través de la herramienta de línea de comandos stack.

Ejemplo: Creando un Nuevo Proyecto con Stack

stack new my-project
cd my-project
stack setup
stack build

Explicación:

  1. stack new my-project crea un nuevo proyecto de Haskell usando una plantilla.
  2. cd my-project cambia el directorio al nuevo proyecto.
  3. stack setup instala la versión apropiada de GHC (Glasgow Haskell Compiler).
  4. stack build construye el proyecto usando la instantánea de Stackage.

Configurando Instantáneas de Stackage

Puedes especificar qué instantánea de Stackage usar en tu archivo stack.yaml.

Ejemplo: Especificando una Instantánea

resolver: lts-18.18

Explicación:

  • El campo resolver especifica la versión de la instantánea de Stackage a usar para tu proyecto.

Conclusión

Hackage y Stackage son herramientas fundamentales en el ecosistema Haskell que facilitan la gestión de paquetes y aseguran la estabilidad en los proyectos de Haskell. Hackage sirve como un repositorio de paquetes integral, mientras que Stackage proporciona conjuntos curados y estables de paquetes. Al dominar estas herramientas, los desarrolladores de Haskell pueden gestionar eficientemente las dependencias y mantener proyectos robustos.
