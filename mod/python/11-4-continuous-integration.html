11-4: Integración Continua

Introducción a la Integración Continua (CI)

La Integración Continua (CI) es una práctica de desarrollo de software en la que los desarrolladores integran su código en un repositorio compartido varias veces al día. Cada integración es verificada por una construcción automatizada (incluyendo pruebas) para detectar errores lo antes posible.

Beneficios de la Integración Continua

  • Detección temprana de errores: Al integrar y probar el código frecuentemente, los errores se detectan y corrigen rápidamente.
  • Mejora de la calidad del código: Las pruebas automatizadas aseguran que el código cumple con los estándares de calidad.
  • Reducción de conflictos de integración: Integrar frecuentemente reduce los conflictos que pueden surgir cuando múltiples desarrolladores trabajan en el mismo proyecto.
  • Entrega continua: Facilita la entrega continua de software, permitiendo lanzamientos más rápidos y frecuentes.

Herramientas de Integración Continua

Existen varias herramientas populares para implementar CI, entre ellas:

Herramienta Descripción
Jenkins Una de las herramientas de CI más populares y de código abierto.
Travis CI Integración continua basada en la nube, especialmente popular en proyectos de código abierto.
CircleCI Plataforma de CI/CD que ofrece integración y entrega continua.
GitLab CI Integración continua integrada en GitLab.
GitHub Actions Permite automatizar flujos de trabajo directamente desde GitHub.

Configuración de Jenkins para CI

A continuación, se muestra cómo configurar Jenkins para un proyecto de Python:

Paso 1: Instalación de Jenkins

  1. Descargar e instalar Jenkins: Puedes descargar Jenkins desde su sitio oficial.
  2. Iniciar Jenkins: Una vez instalado, inicia Jenkins y accede a la interfaz web en http://localhost:8080.

Paso 2: Configuración Inicial

  1. Instalar plugins necesarios: Durante la configuración inicial, Jenkins te pedirá que instales plugins. Asegúrate de instalar los plugins para Git y Python.
  2. Crear un nuevo trabajo: Ve a "Nuevo Item" y selecciona "Proyecto de estilo libre".

Paso 3: Configuración del Proyecto

  1. Configurar el repositorio de Git:

    • En la sección "Source Code Management", selecciona "Git".
    • Ingresa la URL del repositorio de tu proyecto.

  2. Configurar el entorno de construcción:

    • En la sección "Build Environment", selecciona "Provide Node & npm bin/ folder to PATH" si estás usando Node.js.
    • Para Python, asegúrate de que el entorno virtual esté configurado correctamente.

  3. Agregar pasos de construcción:

    • En la sección "Build", agrega un paso de "Execute shell" y escribe los comandos necesarios para instalar dependencias y ejecutar pruebas. Por ejemplo: 
      # Crear un entorno virtual
python3 -m venv venv
source venv/bin/activate

# Instalar dependencias
pip install -r requirements.txt

# Ejecutar pruebas
pytest

Paso 4: Ejecutar y Verificar

  1. Guardar y ejecutar el trabajo: Guarda la configuración y ejecuta el trabajo.
  2. Verificar resultados: Revisa la salida de la consola para asegurarte de que las pruebas se ejecutaron correctamente.

Ejercicio Práctico

Objetivo: Configurar un pipeline de CI básico usando GitHub Actions para un proyecto de Python.

Paso 1: Crear un archivo de flujo de trabajo

  1. En tu repositorio de GitHub, crea un directorio .github/workflows.
  2. Dentro de este directorio, crea un archivo ci.yml.

Paso 2: Definir el flujo de trabajo

name: CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.x'

    - name: Install dependencies
      run: |
        python -m venv venv
        source venv/bin/activate
        pip install -r requirements.txt

    - name: Run tests
      run: |
        source venv/bin/activate
        pytest

Paso 3: Confirmar y empujar los cambios

  1. Guarda el archivo ci.yml.
  2. Confirma y empuja los cambios a tu repositorio de GitHub.

Solución del Ejercicio

Una vez que hayas empujado los cambios, GitHub Actions debería iniciar automáticamente el flujo de trabajo definido en ci.yml. Puedes verificar el estado del flujo de trabajo en la pestaña "Actions" de tu repositorio.

Conclusión

La Integración Continua es una práctica esencial para mantener la calidad y la estabilidad del software. Herramientas como Jenkins y GitHub Actions facilitan la implementación de CI, permitiendo a los desarrolladores detectar y corregir errores rápidamente. En esta lección, aprendiste a configurar Jenkins y GitHub Actions para un proyecto de Python, lo que te permitirá integrar y probar tu código de manera continua y automática.
