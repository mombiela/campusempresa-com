11-1: Empaquetado y Distribución de Aplicaciones

Introducción

El empaquetado y la distribución de aplicaciones son pasos cruciales para llevar tu software desde el entorno de desarrollo hasta los usuarios finales. En esta sección, aprenderás cómo empaquetar tu aplicación Python para su distribución, utilizando herramientas estándar como setuptools y wheel. También veremos cómo publicar tu paquete en el Python Package Index (PyPI) para que otros puedan instalarlo fácilmente.

Objetivos de Aprendizaje

  • Comprender la importancia del empaquetado y la distribución de aplicaciones.
  • Aprender a usar setuptools para crear un paquete Python.
  • Generar archivos de distribución utilizando wheel.
  • Publicar un paquete en PyPI.

Conceptos Clave

  1. Empaquetado: Proceso de preparar tu código para su distribución.
  2. Distribución: Proceso de hacer que tu paquete esté disponible para otros.
  3. setuptools: Herramienta estándar para empaquetar proyectos Python.
  4. wheel: Formato de distribución binaria para Python.
  5. PyPI: Repositorio oficial de paquetes Python.

Paso 1: Estructura del Proyecto

Primero, asegúrate de que tu proyecto tenga una estructura clara y organizada. Aquí hay un ejemplo básico:

mi_proyecto/
│
├── mi_paquete/
│   ├── __init__.py
│   ├── modulo1.py
│   └── modulo2.py
│
├── tests/
│   ├── __init__.py
│   └── test_modulo1.py
│
├── setup.py
└── README.md

Paso 2: Crear el Archivo `setup.py`

El archivo setup.py es el corazón del empaquetado de tu proyecto. Aquí es donde defines los metadatos y las dependencias de tu paquete.

from setuptools import setup, find_packages

setup(
    name="mi_proyecto",
    version="0.1.0",
    author="Tu Nombre",
    author_email="tu_email@example.com",
    description="Una breve descripción de tu proyecto",
    long_description=open('README.md').read(),
    long_description_content_type="text/markdown",
    url="https://github.com/tu_usuario/mi_proyecto",
    packages=find_packages(),
    classifiers=[
        "Programming Language :: Python :: 3",
        "License :: OSI Approved :: MIT License",
        "Operating System :: OS Independent",
    ],
    python_requires='>=3.6',
    install_requires=[
        "numpy",
        "pandas",
    ],
)

Paso 3: Generar Archivos de Distribución

Para generar los archivos de distribución, necesitas instalar wheel si aún no lo has hecho:

pip install wheel

Luego, desde la raíz de tu proyecto, ejecuta:

python setup.py sdist bdist_wheel

Esto creará un directorio dist/ con los archivos .tar.gz y .whl.

Paso 4: Publicar en PyPI

Para publicar tu paquete en PyPI, primero necesitas crear una cuenta en PyPI. Luego, instala twine:

pip install twine

Finalmente, sube tus archivos de distribución:

twine upload dist/*

Ejercicio Práctico

  1. Crear un Proyecto de Ejemplo:

    • Crea un proyecto con la estructura mencionada anteriormente.
    • Escribe un simple módulo Python y un archivo de prueba.

  2. Escribir el Archivo setup.py:

    • Completa el archivo setup.py con los metadatos de tu proyecto.

  3. Generar y Subir el Paquete:

    • Genera los archivos de distribución.
    • Publica tu paquete en PyPI (puedes usar el repositorio de prueba de PyPI para practicar: https://test.pypi.org/).

Solución del Ejercicio

  1. Estructura del Proyecto:
mi_proyecto/
│
├── mi_paquete/
│   ├── __init__.py
│   └── saludo.py
│
├── tests/
│   ├── __init__.py
│   └── test_saludo.py
│
├── setup.py
└── README.md
  1. Archivo saludo.py:
def hola(nombre):
    return f"Hola, {nombre}!"
  1. Archivo test_saludo.py:
import unittest
from mi_paquete.saludo import hola

class TestSaludo(unittest.TestCase):
    def test_hola(self):
        self.assertEqual(hola("Mundo"), "Hola, Mundo!")

if __name__ == '__main__':
    unittest.main()
  1. Archivo setup.py:
from setuptools import setup, find_packages

setup(
    name="mi_proyecto",
    version="0.1.0",
    author="Tu Nombre",
    author_email="tu_email@example.com",
    description="Una breve descripción de tu proyecto",
    long_description=open('README.md').read(),
    long_description_content_type="text/markdown",
    url="https://github.com/tu_usuario/mi_proyecto",
    packages=find_packages(),
    classifiers=[
        "Programming Language :: Python :: 3",
        "License :: OSI Approved :: MIT License",
        "Operating System :: OS Independent",
    ],
    python_requires='>=3.6',
    install_requires=[],
)
