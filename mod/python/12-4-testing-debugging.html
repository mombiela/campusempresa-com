12-4: Pruebas y Depuración

En esta sección, nos enfocaremos en la importancia de las pruebas y la depuración en el desarrollo de software. Aprenderemos cómo aplicar técnicas de pruebas y depuración a nuestro proyecto final para asegurar que funcione correctamente y esté libre de errores.

Objetivos de la Sección

  • Comprender la importancia de las pruebas y la depuración.
  • Aprender a escribir pruebas unitarias para el proyecto.
  • Utilizar herramientas de depuración para identificar y corregir errores.
  • Integrar pruebas y depuración en el flujo de trabajo de desarrollo.

1. Introducción a las Pruebas

Las pruebas son una parte esencial del desarrollo de software. Nos ayudan a asegurar que nuestro código funcione como se espera y a identificar errores antes de que lleguen a producción.

Tipos de Pruebas:

  • Pruebas Unitarias: Verifican el funcionamiento de unidades individuales de código, como funciones o métodos.
  • Pruebas de Integración: Verifican que diferentes módulos o servicios funcionen juntos correctamente.
  • Pruebas de Sistema: Verifican el sistema completo para asegurar que cumple con los requisitos.
  • Pruebas de Aceptación: Verifican que el sistema cumple con los criterios de aceptación definidos por el cliente.

2. Escribir Pruebas Unitarias

Las pruebas unitarias son fundamentales para asegurar que cada parte de nuestro código funcione correctamente de manera aislada. En Python, podemos usar el módulo unittest para escribir y ejecutar pruebas unitarias.

Ejemplo de Prueba Unitaria:

import unittest

def suma(a, b):
    return a + b

class TestSuma(unittest.TestCase):
    def test_suma_positivos(self):
        self.assertEqual(suma(1, 2), 3)
    
    def test_suma_negativos(self):
        self.assertEqual(suma(-1, -1), -2)
    
    def test_suma_cero(self):
        self.assertEqual(suma(0, 0), 0)

if __name__ == '__main__':
    unittest.main()

Explicación:

  • Importamos el módulo unittest.
  • Definimos una función suma que queremos probar.
  • Creamos una clase TestSuma que hereda de unittest.TestCase.
  • Definimos métodos de prueba dentro de la clase, cada uno probando un caso específico.
  • Usamos self.assertEqual para verificar que el resultado de la función sea el esperado.
  • Ejecutamos las pruebas con unittest.main().

3. Técnicas de Depuración

La depuración es el proceso de identificar y corregir errores en el código. Algunas técnicas comunes incluyen:

  • Impresión de Mensajes de Depuración: Usar print para mostrar valores de variables y el flujo del programa.
  • Uso de Herramientas de Depuración: Utilizar depuradores integrados en IDEs o herramientas como pdb.
  • Revisión de Código: Revisar el código manualmente o con la ayuda de compañeros para identificar errores.

4. Uso de pdb para Depuración

pdb es el depurador integrado de Python. Nos permite pausar la ejecución del programa y examinar el estado del mismo.

Ejemplo de Uso de pdb:

import pdb

def suma(a, b):
    pdb.set_trace()  # Pausa la ejecución aquí
    return a + b

resultado = suma(1, 2)
print(resultado)

Comandos Básicos de pdb:

  • n (next): Ejecuta la siguiente línea de código.
  • c (continue): Continúa la ejecución hasta el siguiente punto de interrupción.
  • q (quit): Sale del depurador.
  • p (print): Imprime el valor de una variable.

5. Ejercicio Práctico

Ejercicio:

  1. Escribe una función multiplica(a, b) que multiplique dos números.
  2. Escribe pruebas unitarias para la función multiplica.
  3. Introduce un error intencional en la función y usa pdb para depurarlo.

Solución:

import unittest
import pdb

def multiplica(a, b):
    pdb.set_trace()  # Pausa la ejecución aquí
    return a * b  # Introduce un error intencional aquí, por ejemplo, return a + b

class TestMultiplica(unittest.TestCase):
    def test_multiplica_positivos(self):
        self.assertEqual(multiplica(2, 3), 6)
    
    def test_multiplica_negativos(self):
        self.assertEqual(multiplica(-2, -3), 6)
    
    def test_multiplica_cero(self):
        self.assertEqual(multiplica(0, 5), 0)

if __name__ == '__main__':
    unittest.main()

6. Conclusión

En esta sección, hemos aprendido la importancia de las pruebas y la depuración en el desarrollo de software. Hemos visto cómo escribir pruebas unitarias utilizando unittest y cómo usar pdb para depurar nuestro código. Estas habilidades son esenciales para asegurar que nuestro proyecto final funcione correctamente y esté libre de errores.
