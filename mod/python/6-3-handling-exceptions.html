6-3: Manejo de Excepciones

Introducción

El manejo de excepciones es una parte crucial de la programación, ya que permite a los desarrolladores gestionar errores y situaciones inesperadas de manera controlada. En Python, las excepciones son manejadas mediante las palabras clave try, except, else y finally.

Conceptos Clave

  1. Excepciones: Eventos que ocurren durante la ejecución de un programa y que interrumpen el flujo normal del programa.
  2. Bloque try: Se utiliza para envolver el código que puede generar una excepción.
  3. Bloque except: Se utiliza para manejar la excepción si ocurre.
  4. Bloque else: Se ejecuta si no ocurre ninguna excepción.
  5. Bloque finally: Se ejecuta siempre, ocurra o no una excepción.

Sintaxis Básica

try:
    # Código que puede generar una excepción
    pass
except SomeException as e:
    # Código para manejar la excepción
    pass
else:
    # Código que se ejecuta si no ocurre ninguna excepción
    pass
finally:
    # Código que se ejecuta siempre
    pass

Ejemplo Práctico

Vamos a ver un ejemplo práctico de manejo de excepciones en Python.

def divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError as e:
        print("Error: No se puede dividir por cero.")
        return None
    else:
        print("La división fue exitosa.")
        return result
    finally:
        print("Operación de división completada.")

# Prueba de la función
print(divide(10, 2))  # Salida esperada: 5.0
print(divide(10, 0))  # Salida esperada: Error: No se puede dividir por cero.

Explicación del Código

  1. Bloque try: Intenta ejecutar la división a / b.
  2. Bloque except: Si ocurre una excepción de tipo ZeroDivisionError, se imprime un mensaje de error y se retorna None.
  3. Bloque else: Si no ocurre ninguna excepción, se imprime un mensaje de éxito y se retorna el resultado de la división.
  4. Bloque finally: Se ejecuta siempre, independientemente de si ocurrió una excepción o no, imprimiendo un mensaje de finalización de la operación.

Ejercicios Prácticos

Ejercicio 1

Escribe una función que lea un archivo y maneje posibles excepciones como FileNotFoundError y IOError.

def read_file(filename):
    try:
        with open(filename, 'r') as file:
            content = file.read()
            return content
    except FileNotFoundError:
        print("Error: El archivo no fue encontrado.")
    except IOError:
        print("Error: Ocurrió un error de entrada/salida.")
    finally:
        print("Operación de lectura de archivo completada.")

# Prueba de la función
print(read_file("archivo_existente.txt"))
print(read_file("archivo_inexistente.txt"))

Ejercicio 2

Modifica la función divide para que maneje una excepción genérica y otra específica.

def divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Error: No se puede dividir por cero.")
        return None
    except Exception as e:
        print(f"Error inesperado: {e}")
        return None
    else:
        print("La división fue exitosa.")
        return result
    finally:
        print("Operación de división completada.")

# Prueba de la función
print(divide(10, 2))  # Salida esperada: 5.0
print(divide(10, 0))  # Salida esperada: Error: No se puede dividir por cero.
print(divide("10", 2))  # Salida esperada: Error inesperado: unsupported operand type(s) for /: 'str' and 'int'

Retroalimentación sobre Errores Comunes

  1. No manejar excepciones específicas: Es una buena práctica manejar excepciones específicas antes de manejar excepciones genéricas.
  2. Olvidar el bloque finally: Asegúrate de usar el bloque finally para liberar recursos o realizar acciones que deben ejecutarse sin importar si ocurrió una excepción.

Conclusión

El manejo de excepciones es esencial para crear programas robustos y confiables. Al utilizar bloques try, except, else y finally, puedes controlar el flujo de tu programa y manejar errores de manera efectiva. Practica estos conceptos con los ejercicios proporcionados para reforzar tu comprensión.
