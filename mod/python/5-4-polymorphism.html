Módulo 5: Programación Orientada a Objetos

5-4 Polimorfismo

El polimorfismo es uno de los cuatro pilares de la programación orientada a objetos (POO), junto con la encapsulación, la herencia y el abstraccionismo. En términos simples, el polimorfismo permite que una misma interfaz pueda ser utilizada para diferentes tipos de objetos. Esto significa que una función puede tomar diferentes formas según el contexto en el que se utilice.

Conceptos Clave

  1. Polimorfismo en Funciones: Permite que una función procese datos de diferentes tipos.
  2. Polimorfismo en Métodos: Permite que diferentes clases implementen métodos con el mismo nombre pero con comportamientos diferentes.
  3. Polimorfismo en Operadores: Permite que los operadores se comporten de manera diferente según los operandos.

Ejemplo Práctico

Vamos a ilustrar el polimorfismo mediante un ejemplo con clases de animales.

class Animal:
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

def make_animal_speak(animal):
    print(animal.speak())

dog = Dog()
cat = Cat()

make_animal_speak(dog)  # Output: Woof!
make_animal_speak(cat)  # Output: Meow!

En este ejemplo:

  • La clase base Animal define un método speak que no hace nada (pass).
  • Las clases Dog y Cat heredan de Animal y sobrescriben el método speak para proporcionar su propia implementación.
  • La función make_animal_speak acepta un objeto de tipo Animal y llama al método speak del objeto pasado.

Ejercicio Práctico

Ejercicio 1:

Crea una clase base llamada Shape con un método area que no haga nada. Luego, crea dos clases derivadas, Circle y Rectangle, que sobrescriban el método area para calcular el área de un círculo y un rectángulo, respectivamente.

import math

class Shape:
    def area(self):
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        return math.pi * self.radius ** 2

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height

# Prueba tus clases
circle = Circle(5)
rectangle = Rectangle(4, 6)

print(f"Área del círculo: {circle.area()}")  # Output: Área del círculo: 78.53981633974483
print(f"Área del rectángulo: {rectangle.area()}")  # Output: Área del rectángulo: 24

Solución:

import math

class Shape:
    def area(self):
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        return math.pi * self.radius ** 2

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height

# Prueba tus clases
circle = Circle(5)
rectangle = Rectangle(4, 6)

print(f"Área del círculo: {circle.area()}")  # Output: Área del círculo: 78.53981633974483
print(f"Área del rectángulo: {rectangle.area()}")  # Output: Área del rectángulo: 24

Errores Comunes

  1. No Sobrescribir Métodos: Asegúrate de que las clases derivadas sobrescriban los métodos de la clase base.
  2. No Usar la Clase Base: Utiliza la clase base para definir la interfaz común que las clases derivadas deben implementar.
  3. No Manejar Diferentes Tipos de Objetos: Asegúrate de que las funciones que utilizan polimorfismo puedan manejar diferentes tipos de objetos correctamente.

Consejos Adicionales

  • Utiliza Interfaces Claras: Define métodos claros y concisos en la clase base para que las clases derivadas los implementen.
  • Prueba con Diferentes Tipos de Objetos: Asegúrate de probar tus funciones polimórficas con diferentes tipos de objetos para verificar que se comportan correctamente.

Conclusión

El polimorfismo es una herramienta poderosa en la programación orientada a objetos que permite que una misma interfaz pueda ser utilizada para diferentes tipos de objetos. Esto facilita la reutilización del código y mejora la flexibilidad y mantenibilidad de los programas. En la siguiente sección, exploraremos el concepto de encapsulación, otro pilar fundamental de la POO.
