Multithreading y Multiprocessing en Python
Asyncio para Programación Asíncrona ►

Introducción

En esta sección, exploraremos los conceptos de multithreading y multiprocessing en Python. Estas técnicas son esenciales para mejorar el rendimiento de tus programas al permitirles realizar múltiples tareas concurrentemente.

Multithreading

El multithreading permite que un programa ejecute múltiples hilos (unidades más pequeñas de un proceso) simultáneamente. Esto es particularmente útil para tareas dependientes de I/O.

Conceptos Clave

  • Hilo: La unidad más pequeña de un proceso que puede ser programada para ejecución.
  • GIL (Global Interpreter Lock): Un mutex que protege el acceso a los objetos de Python, evitando que múltiples hilos ejecuten bytecodes de Python a la vez.
  • Seguridad de Hilos: Asegurar que los datos compartidos sean accedidos por solo un hilo a la vez.

Ejemplo Básico

import threading

def print_numbers():
    for i in range(5):
        print(i)

def print_letters():
    for letter in 'abcde':
        print(letter)

# Creando hilos
thread1 = threading.Thread(target=print_numbers)
thread2 = threading.Thread(target=print_letters)

# Iniciando hilos
thread1.start()
thread2.start()

# Esperando a que los hilos completen
thread1.join()
thread2.join()

print("¡Hecho!")

Explicación:

  • Definimos dos funciones: print_numbers y print_letters.
  • Creamos dos hilos, cada uno apuntando a una de las funciones.
  • Iniciamos ambos hilos y esperamos a que completen usando join().

Ejemplo de Seguridad de Hilos

import threading

counter = 0
lock = threading.Lock()

def increment_counter():
    global counter
    for _ in range(1000):
        with lock:
            counter += 1

threads = []
for _ in range(10):
    thread = threading.Thread(target=increment_counter)
    threads.append(thread)
    thread.start()

for thread in threads:
    thread.join()

print(f"Valor final del contador: {counter}")

Explicación:

  • Usamos un Lock para asegurar que solo un hilo pueda incrementar el contador a la vez, previniendo condiciones de carrera.

Multiprocessing

El multiprocessing implica usar múltiples procesos, cada uno con su propio intérprete de Python y espacio de memoria. Esto es útil para tareas dependientes de CPU.

Conceptos Clave

  • Proceso: Un programa independiente en ejecución.
  • IPC (Inter-Process Communication): Mecanismos para que los procesos se comuniquen y sincronicen sus acciones.
  • Memoria Compartida: Memoria que puede ser accedida por múltiples procesos.

Ejemplo Básico

import multiprocessing

def print_numbers():
    for i in range(5):
        print(i)

def print_letters():
    for letter in 'abcde':
        print(letter)

# Creando procesos
process1 = multiprocessing.Process(target=print_numbers)
process2 = multiprocessing.Process(target=print_letters)

# Iniciando procesos
process1.start()
process2.start()

# Esperando a que los procesos completen
process1.join()
process2.join()

print("¡Hecho!")

Explicación:

  • Similar al ejemplo de hilos, pero usando multiprocessing.Process en lugar de threading.Thread.

Ejemplo de Memoria Compartida

import multiprocessing

def increment_counter(counter):
    for _ in range(1000):
        counter.value += 1

if __name__ == "__main__":
    counter = multiprocessing.Value('i', 0)
    processes = []
    
    for _ in range(10):
        process = multiprocessing.Process(target=increment_counter, args=(counter,))
        processes.append(process)
        process.start()
    
    for process in processes:
        process.join()
    
    print(f"Valor final del contador: {counter.value}")

Explicación:

  • Usamos multiprocessing.Value para crear un entero compartido ('i' significa entero) que puede ser modificado de manera segura por múltiples procesos.

Tabla Comparativa

| Característica | Multithreading | Multiprocessing | |------------------|-----------------------------------------|-----------------------------------------| | Tipo de Concurrencia | A nivel de hilos | A nivel de procesos | | Espacio de Memoria | Compartido | Separado | | Adecuado Para | Tareas dependientes de I/O | Tareas dependientes de CPU | | Impacto del GIL | Afectado por el GIL | No afectado por el GIL | | Comunicación | Más fácil (memoria compartida) | Más complejo (mecanismos de IPC) |

Conclusión

El multithreading y el multiprocessing son herramientas poderosas para mejorar el rendimiento de los programas en Python. El multithreading es adecuado para tareas dependientes de I/O, mientras que el multiprocessing es mejor para tareas dependientes de CPU. Entender las diferencias y los casos de uso apropiados para cada uno puede ayudarte a escribir código en Python más eficiente y efectivo.
