8-4: Requests para Solicitudes HTTP

Introducción

La biblioteca requests es una de las herramientas más populares y fáciles de usar en Python para realizar solicitudes HTTP. Permite a los desarrolladores interactuar con servicios web y APIs de manera sencilla y eficiente.

Instalación

Para instalar la biblioteca requests, puedes usar pip:

pip install requests

Realizando Solicitudes HTTP

Solicitud GET

La solicitud GET se utiliza para solicitar datos de un servidor. Aquí hay un ejemplo básico:

import requests

response = requests.get('https://api.github.com')

# Verificar el estado de la respuesta
print(response.status_code)

# Imprimir el contenido de la respuesta
print(response.text)

Explicación:

  • requests.get(url) realiza una solicitud GET a la URL especificada.
  • response.status_code devuelve el código de estado HTTP de la respuesta.
  • response.text devuelve el contenido de la respuesta como una cadena.

Solicitud POST

La solicitud POST se utiliza para enviar datos al servidor. Aquí hay un ejemplo:

import requests

url = 'https://httpbin.org/post'
data = {'key': 'value'}

response = requests.post(url, data=data)

# Verificar el estado de la respuesta
print(response.status_code)

# Imprimir el contenido de la respuesta
print(response.json())

Explicación:

  • requests.post(url, data=data) realiza una solicitud POST a la URL especificada con los datos proporcionados.
  • response.json() convierte el contenido de la respuesta en un objeto JSON.

Manejo de Parámetros en la URL

Puedes pasar parámetros en la URL utilizando el argumento params:

import requests

url = 'https://api.github.com/search/repositories'
params = {'q': 'requests+language:python'}

response = requests.get(url, params=params)

# Verificar el estado de la respuesta
print(response.status_code)

# Imprimir el contenido de la respuesta
print(response.json())

Explicación:

  • params es un diccionario que contiene los parámetros de la URL.
  • requests.get(url, params=params) añade los parámetros a la URL automáticamente.

Manejo de Cabeceras HTTP

Puedes añadir cabeceras HTTP personalizadas utilizando el argumento headers:

import requests

url = 'https://api.github.com/user'
headers = {'Authorization': 'token YOUR_ACCESS_TOKEN'}

response = requests.get(url, headers=headers)

# Verificar el estado de la respuesta
print(response.status_code)

# Imprimir el contenido de la respuesta
print(response.json())

Explicación:

  • headers es un diccionario que contiene las cabeceras HTTP.
  • requests.get(url, headers=headers) añade las cabeceras a la solicitud.

Manejo de Errores

Es importante manejar errores al realizar solicitudes HTTP. Aquí hay un ejemplo de cómo hacerlo:

import requests

try:
    response = requests.get('https://api.github.com/invalid-endpoint')
    response.raise_for_status()
except requests.exceptions.HTTPError as err:
    print(f'HTTP error occurred: {err}')
except Exception as err:
    print(f'Other error occurred: {err}')
else:
    print('Success!')

Explicación:

  • response.raise_for_status() lanza una excepción si la solicitud HTTP no fue exitosa.
  • requests.exceptions.HTTPError maneja errores HTTP específicos.
  • Exception maneja cualquier otro tipo de error.

Ejercicio Práctico

Ejercicio: Realiza una solicitud GET a la API de GitHub para obtener información sobre un repositorio específico. Luego, imprime el nombre del repositorio, la descripción y el número de estrellas.

Solución:

import requests

url = 'https://api.github.com/repos/psf/requests'

response = requests.get(url)

if response.status_code == 200:
    data = response.json()
    print(f"Nombre del repositorio: {data['name']}")
    print(f"Descripción: {data['description']}")
    print(f"Estrellas: {data['stargazers_count']}")
else:
    print(f"Error: {response.status_code}")

Resumen

En esta sección, hemos aprendido a:

  • Instalar y utilizar la biblioteca requests para realizar solicitudes HTTP.
  • Realizar solicitudes GET y POST.
  • Manejar parámetros en la URL y cabeceras HTTP.
  • Manejar errores al realizar solicitudes HTTP.

Con estos conocimientos, puedes interactuar con APIs y servicios web de manera efectiva en tus proyectos de Python. ¡Ahora estás listo para explorar más bibliotecas y herramientas en Python!
