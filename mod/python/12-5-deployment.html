Módulo 12: Proyecto Final

12-5: Despliegue

Objetivos del Tema

En esta sección, aprenderás cómo desplegar tu aplicación Python en un entorno de producción. Cubriremos los siguientes puntos:

  • Preparación de la aplicación para el despliegue.
  • Opciones de alojamiento.
  • Configuración del servidor.
  • Despliegue continuo.

Preparación de la Aplicación para el Despliegue

Antes de desplegar tu aplicación, es crucial asegurarte de que esté lista para un entorno de producción. Aquí hay algunos pasos clave:

  1. Revisar Dependencias:

    • Asegúrate de que todas las dependencias estén listadas en un archivo requirements.txt.
    pip freeze > requirements.txt

  2. Configuración de Variables de Entorno:

    • Utiliza variables de entorno para manejar configuraciones sensibles como claves API, contraseñas y configuraciones específicas del entorno.
    import os
SECRET_KEY = os.getenv('SECRET_KEY')

  3. Pruebas Exhaustivas:

    • Asegúrate de que tu aplicación pase todas las pruebas unitarias y de integración.
    python -m unittest discover

Opciones de Alojamiento

Hay varias opciones para alojar tu aplicación Python. Aquí hay algunas populares:

Proveedor Descripción
Heroku Plataforma como servicio (PaaS) que facilita el despliegue de aplicaciones.
AWS (EC2) Servicio de computación en la nube que ofrece instancias virtuales.
DigitalOcean Proveedor de infraestructura en la nube con precios competitivos.
PythonAnywhere Alojamiento específico para aplicaciones Python.

Configuración del Servidor

Para este ejemplo, utilizaremos Heroku debido a su simplicidad y popularidad.

  1. Instalar Heroku CLI:

    • Descarga e instala la CLI de Heroku desde aquí.

  2. Iniciar Sesión en Heroku:

    heroku login

  3. Crear una Aplicación en Heroku:

    heroku create nombre-de-tu-aplicacion

  4. Configurar el Archivo Procfile:

    • Crea un archivo Procfile en el directorio raíz de tu proyecto con el siguiente contenido:
    web: gunicorn nombre_de_tu_aplicacion:app

  5. Desplegar la Aplicación:

    • Inicializa un repositorio Git si no lo has hecho ya.
    git init
git add .
git commit -m "Preparando para el despliegue"
    • Despliega tu aplicación a Heroku.
    git push heroku master

  6. Configurar Variables de Entorno en Heroku:

    heroku config:set SECRET_KEY=tu_clave_secreta

Despliegue Continuo

El despliegue continuo (CD) es una práctica que permite a los desarrolladores desplegar código automáticamente en producción cada vez que se realiza un cambio en el repositorio.

  1. Configurar Integración Continua (CI):

    • Utiliza herramientas como GitHub Actions, Travis CI o CircleCI para configurar pipelines de CI/CD.

  2. Ejemplo de Configuración con GitHub Actions:

    • Crea un archivo .github/workflows/deploy.yml en tu repositorio con el siguiente contenido:
    name: Deploy to Heroku

on:
  push:
    branches:
      - master

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.x'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Deploy to Heroku
      env:
        HEROKU_API_KEY: ${{ secrets.HEROKU_API_KEY }}
      run: |
        git remote add heroku https://git.heroku.com/nombre-de-tu-aplicacion.git
        git push heroku master

Conclusión

En esta sección, has aprendido cómo preparar tu aplicación para el despliegue, las opciones de alojamiento disponibles, cómo configurar un servidor y cómo implementar el despliegue continuo. Ahora estás listo para poner tu aplicación en producción y asegurarte de que esté disponible para los usuarios.

Ejercicio Práctico

Despliega tu aplicación en Heroku:

  1. Sigue los pasos mencionados anteriormente para desplegar tu aplicación en Heroku.
  2. Configura variables de entorno y asegúrate de que tu aplicación funcione correctamente en producción.
  3. Implementa un pipeline de CI/CD utilizando GitHub Actions.

Solución:

  • Sigue los pasos detallados en la sección de Configuración del Servidor y Despliegue Continuo.
  • Verifica que tu aplicación esté accesible en la URL proporcionada por Heroku.
  • Asegúrate de que todas las pruebas pasen antes de cada despliegue.
