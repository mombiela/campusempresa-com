10-1: Pruebas Unitarias con `unittest`

Introducción a las Pruebas Unitarias

Las pruebas unitarias son una parte fundamental del desarrollo de software. Permiten verificar que las distintas partes de tu código funcionen correctamente de manera aislada. En Python, el módulo unittest proporciona un marco para crear y ejecutar pruebas unitarias.

Objetivos de esta lección

  • Comprender la importancia de las pruebas unitarias.
  • Aprender a utilizar el módulo unittest para escribir y ejecutar pruebas.
  • Conocer las mejores prácticas para escribir pruebas efectivas.

Conceptos Clave

  1. Pruebas Unitarias: Verifican el funcionamiento de una unidad de código (por ejemplo, una función o un método).
  2. Módulo unittest: Biblioteca estándar de Python para escribir y ejecutar pruebas unitarias.
  3. Casos de Prueba: Clases que contienen métodos de prueba.
  4. Métodos de Prueba: Métodos dentro de un caso de prueba que verifican el comportamiento del código.

Configuración del Entorno

Asegúrate de tener Python instalado en tu sistema. No necesitas instalar nada adicional, ya que unittest es parte de la biblioteca estándar de Python.

Escribiendo tu Primera Prueba Unitaria

Vamos a comenzar con un ejemplo simple. Supongamos que tenemos una función que suma dos números:

# archivo: calculadora.py
def suma(a, b):
    return a + b

Ahora, escribiremos una prueba unitaria para esta función.

# archivo: test_calculadora.py
import unittest
from calculadora import suma

class TestCalculadora(unittest.TestCase):
    
    def test_suma(self):
        self.assertEqual(suma(1, 2), 3)
        self.assertEqual(suma(-1, 1), 0)
        self.assertEqual(suma(-1, -1), -2)

if __name__ == '__main__':
    unittest.main()

Explicación del Código

  1. Importaciones:

    • import unittest: Importa el módulo unittest.
    • from calculadora import suma: Importa la función suma desde el archivo calculadora.py.

  2. Clase de Prueba:

    • class TestCalculadora(unittest.TestCase): Define una clase de prueba que hereda de unittest.TestCase.

  3. Método de Prueba:

    • def test_suma(self): Define un método de prueba que verifica la función suma.
    • self.assertEqual(suma(1, 2), 3): Verifica que suma(1, 2) sea igual a 3.
    • self.assertEqual(suma(-1, 1), 0): Verifica que suma(-1, 1) sea igual a 0.
    • self.assertEqual(suma(-1, -1), -2): Verifica que suma(-1, -1) sea igual a -2.

  4. Ejecución de Pruebas:

    • if __name__ == '__main__': unittest.main(): Ejecuta todas las pruebas cuando se ejecuta el archivo.

Ejecutando las Pruebas

Para ejecutar las pruebas, abre una terminal y navega al directorio donde se encuentra test_calculadora.py. Luego, ejecuta el siguiente comando:

python test_calculadora.py

Deberías ver una salida similar a esta:

...
----------------------------------------------------------------------
Ran 1 test in 0.001s

OK

Ejercicio Práctico

Escribe una función que multiplique dos números y crea una prueba unitaria para verificar su correcto funcionamiento.

  1. Función de Multiplicación:
# archivo: calculadora.py
def multiplicacion(a, b):
    return a * b
  1. Prueba Unitaria:
# archivo: test_calculadora.py
import unittest
from calculadora import multiplicacion

class TestCalculadora(unittest.TestCase):
    
    def test_multiplicacion(self):
        self.assertEqual(multiplicacion(2, 3), 6)
        self.assertEqual(multiplicacion(-1, 1), -1)
        self.assertEqual(multiplicacion(-1, -1), 1)

if __name__ == '__main__':
    unittest.main()

Solución del Ejercicio

Para verificar tu solución, ejecuta el archivo test_calculadora.py de la misma manera que antes. Deberías ver una salida similar a esta:

...
----------------------------------------------------------------------
Ran 1 test in 0.001s

OK

Resumen

En esta lección, aprendiste a:

  • Comprender la importancia de las pruebas unitarias.
  • Utilizar el módulo unittest para escribir y ejecutar pruebas.
  • Escribir casos de prueba y métodos de prueba.
  • Ejecutar pruebas unitarias desde la línea de comandos.

En la siguiente lección, exploraremos técnicas de depuración para identificar y corregir errores en tu código.

Errores Comunes y Consejos

  • No heredar de unittest.TestCase: Asegúrate de que tus clases de prueba hereden de unittest.TestCase.
  • Olvidar el if __name__ == '__main__': unittest.main(): Esto es necesario para ejecutar las pruebas cuando el archivo se ejecuta directamente.
  • No usar self.assertEqual correctamente: Asegúrate de que el primer argumento sea el valor obtenido y el segundo el valor esperado.
