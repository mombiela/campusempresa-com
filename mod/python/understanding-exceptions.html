Introducción a las Excepciones

Las excepciones son eventos que pueden alterar el flujo normal de un programa. Se utilizan típicamente para manejar errores u otras condiciones excepcionales que ocurren durante la ejecución de un programa.

Conceptos Clave

  • Excepción: Un error que ocurre durante la ejecución de un programa.
  • Manejo de Excepciones: El proceso de responder a las excepciones cuando ocurren.
  • Bloque Try: Un bloque de código que se prueba para detectar errores mientras se ejecuta.
  • Bloque Except: Un bloque de código que se ejecuta si ocurre un error en el bloque try.
  • Bloque Finally: Un bloque de código que se ejecuta independientemente de si ocurre una excepción o no.

Manejo Básico de Excepciones

En Python, las excepciones se manejan usando los bloques try, except y opcionalmente finally.

Ejemplo

try:
    # Código que puede generar una excepción
    result = 10 / 0
except ZeroDivisionError:
    # Código para manejar la excepción
    print("¡No se puede dividir por cero!")
finally:
    # Código que se ejecutará sin importar qué
    print("Ejecución completada.")

Explicación

  • El bloque try contiene código que podría lanzar una excepción.
  • El bloque except captura y maneja la excepción.
  • El bloque finally contiene código que siempre se ejecutará, independientemente de si se lanzó una excepción o no.

Manejo de Múltiples Excepciones

Puedes manejar múltiples excepciones especificando múltiples bloques except.

Ejemplo

try:
    value = int(input("Introduce un número: "))
    result = 10 / value
except ValueError:
    print("¡Entrada no válida! Por favor, introduce un número.")
except ZeroDivisionError:
    print("¡No se puede dividir por cero!")
finally:
    print("Ejecución completada.")

Explicación

  • Múltiples bloques except te permiten manejar diferentes tipos de excepciones por separado.
  • Esto hace que tu código sea más robusto y fácil de depurar.

Usando Else con Try-Except

El bloque else puede ser usado para ejecutar código si no se levantan excepciones en el bloque try.

Ejemplo

try:
    value = int(input("Introduce un número: "))
    result = 10 / value
except ZeroDivisionError:
    print("¡No se puede dividir por cero!")
else:
    print(f"El resultado es {result}")
finally:
    print("Ejecución completada.")

Explicación

  • El bloque else se ejecuta solo si el bloque try no lanza una excepción.
  • Esto puede ser útil para código que solo debe ejecutarse si no ocurrieron errores.

Lanzando Excepciones

Puedes lanzar excepciones usando la palabra clave raise.

Ejemplo

def check_age(age):
    if age < 18:
        raise ValueError("La edad debe ser al menos 18.")
    return "Acceso concedido."

try:
    print(check_age(15))
except ValueError as e:
    print(e)

Explicación

  • La palabra clave raise se usa para desencadenar manualmente una excepción.
  • Esto puede ser útil para hacer cumplir ciertas condiciones en tu código.

Excepciones Personalizadas

Puedes definir tus propias excepciones creando una nueva clase que herede de la clase Exception incorporada.

Ejemplo

class CustomError(Exception):
    pass

def check_value(value):
    if value < 0:
        raise CustomError("El valor no puede ser negativo.")
    return value

try:
    print(check_value(-5))
except CustomError as e:
    print(e)

Explicación

  • Las excepciones personalizadas te permiten crear mensajes de error y rutinas de manejo más específicos.
  • Esto puede hacer que tu código sea más legible y mantenible.

Resumen

  • Las excepciones se utilizan para manejar errores y otras condiciones excepcionales en un programa.
  • Los bloques try, except y finally se utilizan para manejar excepciones.
  • Múltiples excepciones pueden ser manejadas usando múltiples bloques except.
  • El bloque else puede ser usado para ejecutar código si no se levantan excepciones.
  • Las excepciones pueden ser lanzadas manualmente usando la palabra clave raise.
  • Las excepciones personalizadas pueden ser creadas definiendo nuevas clases que hereden de la clase Exception.

Entender y usar efectivamente las excepciones es crucial para escribir programas en Python robustos y resistentes a errores.
