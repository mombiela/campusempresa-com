5-5: Encapsulación

Introducción a la Encapsulación

La encapsulación es uno de los cuatro pilares fundamentales de la Programación Orientada a Objetos (POO), junto con la herencia, el polimorfismo y la abstracción. La encapsulación se refiere a la práctica de restringir el acceso directo a algunos de los componentes de un objeto y puede prevenir la modificación accidental o no autorizada de datos.

Conceptos Clave

  • Encapsulación: Proceso de envolver los datos (variables) y el código (métodos) que opera sobre los datos en una sola unidad, como una clase.
  • Modificadores de Acceso: En Python, los modificadores de acceso se implementan mediante convenciones de nombres:
    • Público: Accesible desde cualquier lugar.
    • Protegido: Indicado con un guion bajo simple _, sugiere que no debe ser accedido directamente fuera de la clase y sus subclases.
    • Privado: Indicado con dos guiones bajos __, restringe el acceso a la propia clase.

Ejemplo Práctico

A continuación, se presenta un ejemplo práctico para ilustrar la encapsulación en Python:

class CuentaBancaria:
    def __init__(self, titular, saldo):
        self.titular = titular  # Atributo público
        self.__saldo = saldo    # Atributo privado

    def depositar(self, cantidad):
        if cantidad > 0:
            self.__saldo += cantidad
        else:
            print("La cantidad a depositar debe ser positiva.")

    def retirar(self, cantidad):
        if 0 < cantidad <= self.__saldo:
            self.__saldo -= cantidad
        else:
            print("Fondos insuficientes o cantidad inválida.")

    def obtener_saldo(self):
        return self.__saldo

# Uso de la clase
cuenta = CuentaBancaria("Juan Pérez", 1000)
print(cuenta.titular)  # Acceso permitido
print(cuenta.obtener_saldo())  # Acceso permitido mediante método público

cuenta.depositar(500)
print(cuenta.obtener_saldo())  # Saldo actualizado

cuenta.retirar(200)
print(cuenta.obtener_saldo())  # Saldo actualizado

# Intento de acceso directo al atributo privado
# print(cuenta.__saldo)  # Esto generará un error de atributo

Explicación del Código

  1. Definición de la Clase:

    • CuentaBancaria es una clase que representa una cuenta bancaria.
    • __init__ es el método constructor que inicializa los atributos titular y __saldo.

  2. Atributos Públicos y Privados:

    • titular es un atributo público, accesible desde cualquier lugar.
    • __saldo es un atributo privado, accesible solo dentro de la clase.

  3. Métodos Públicos:

    • depositar permite agregar fondos a la cuenta.
    • retirar permite retirar fondos de la cuenta.
    • obtener_saldo devuelve el saldo actual de la cuenta.

  4. Uso de la Clase:

    • Se crea una instancia de CuentaBancaria con un titular y un saldo inicial.
    • Se accede al atributo público titular y al saldo mediante el método obtener_saldo.
    • Se realizan operaciones de depósito y retiro, actualizando el saldo.

Ejercicio Práctico

Ejercicio: Implementa una clase Empleado que encapsule los atributos nombre, edad y salario. Proporciona métodos para establecer y obtener el salario, asegurando que el salario no pueda ser negativo.

class Empleado:
    def __init__(self, nombre, edad, salario):
        self.nombre = nombre
        self.edad = edad
        self.__salario = salario

    def establecer_salario(self, salario):
        if salario >= 0:
            self.__salario = salario
        else:
            print("El salario no puede ser negativo.")

    def obtener_salario(self):
        return self.__salario

# Uso de la clase
empleado = Empleado("Ana Gómez", 30, 50000)
print(empleado.nombre)  # Acceso permitido
print(empleado.obtener_salario())  # Acceso permitido mediante método público

empleado.establecer_salario(55000)
print(empleado.obtener_salario())  # Salario actualizado

empleado.establecer_salario(-1000)  # Intento de establecer un salario negativo
print(empleado.obtener_salario())  # Salario no actualizado

Solución del Ejercicio

  1. Definición de la Clase:

    • Empleado es una clase que representa a un empleado.
    • __init__ es el método constructor que inicializa los atributos nombre, edad y __salario.

  2. Atributos Públicos y Privados:

    • nombre y edad son atributos públicos.
    • __salario es un atributo privado.

  3. Métodos Públicos:

    • establecer_salario permite establecer el salario, asegurando que no sea negativo.
    • obtener_salario devuelve el salario actual del empleado.

Conclusión

La encapsulación es una técnica fundamental en la POO que ayuda a proteger los datos y a mantener la integridad de los objetos. Al restringir el acceso directo a los atributos y proporcionar métodos controlados para interactuar con ellos, se puede evitar la manipulación indebida de los datos y asegurar que los objetos se comporten de manera predecible.

En el siguiente tema, exploraremos cómo la encapsulación se combina con otros principios de la POO para crear sistemas robustos y mantenibles.
