11-2: Control de Versiones con Git

Introducción

El control de versiones es una práctica esencial en el desarrollo de software que permite gestionar los cambios en el código fuente a lo largo del tiempo. Git es uno de los sistemas de control de versiones más populares y utilizados en la industria. En esta sección, aprenderás los conceptos básicos de Git, cómo configurarlo y cómo utilizarlo para gestionar tus proyectos de programación.

Conceptos Clave

  • Repositorio (Repository): Un repositorio es un lugar donde se almacena el historial completo de cambios de un proyecto.
  • Commit: Un commit es una instantánea del estado del proyecto en un momento dado.
  • Branch (Rama): Una rama es una línea de desarrollo independiente.
  • Merge: La acción de combinar cambios de diferentes ramas.
  • Clone: Crear una copia local de un repositorio remoto.
  • Push: Enviar cambios locales a un repositorio remoto.
  • Pull: Obtener cambios desde un repositorio remoto a tu copia local.

Configuración Inicial

Instalación de Git

  1. Windows:

    • Descarga el instalador desde git-scm.com y sigue las instrucciones.

  2. macOS:

    • Usa Homebrew: brew install git
    • O descarga el instalador desde git-scm.com.

  3. Linux:

    • Debian/Ubuntu: sudo apt-get install git
    • Fedora: sudo dnf install git
    • Arch: sudo pacman -S git

Configuración Básica

Después de instalar Git, configura tu nombre de usuario y correo electrónico:

git config --global user.name "Tu Nombre"
git config --global user.email "tuemail@example.com"

Comandos Básicos de Git

Crear un Nuevo Repositorio

Para iniciar un nuevo repositorio en un directorio existente:

cd nombre-del-directorio
git init

Clonar un Repositorio

Para clonar un repositorio remoto:

git clone https://github.com/usuario/nombre-del-repositorio.git

Ver el Estado del Repositorio

Para ver el estado actual del repositorio:

git status

Añadir Cambios al Área de Staging

Para añadir archivos al área de staging:

git add nombre-del-archivo

Para añadir todos los archivos modificados:

git add .

Realizar un Commit

Para realizar un commit de los cambios en el área de staging:

git commit -m "Mensaje del commit"

Ver el Historial de Commits

Para ver el historial de commits:

git log

Trabajar con Ramas

Para crear una nueva rama:

git branch nombre-de-la-rama

Para cambiar a una rama existente:

git checkout nombre-de-la-rama

Para crear y cambiar a una nueva rama en un solo paso:

git checkout -b nombre-de-la-rama

Fusionar Ramas

Para fusionar una rama en la rama actual:

git merge nombre-de-la-rama

Enviar Cambios a un Repositorio Remoto

Para enviar cambios a un repositorio remoto:

git push origin nombre-de-la-rama

Obtener Cambios de un Repositorio Remoto

Para obtener cambios de un repositorio remoto:

git pull origin nombre-de-la-rama

Ejercicio Práctico

  1. Crear un Nuevo Repositorio:

    • Crea un nuevo directorio y navega a él.
    • Inicializa un nuevo repositorio de Git.

  2. Añadir y Confirmar Cambios:

    • Crea un archivo llamado hola_mundo.py y añade el siguiente código:

      print("Hola, Mundo!")

    • Añade el archivo al área de staging y realiza un commit.

  3. Crear una Nueva Rama:

    • Crea una nueva rama llamada nueva-funcionalidad.
    • Cambia a la nueva rama.

  4. Modificar y Fusionar:

    • Modifica hola_mundo.py para que imprima otro mensaje.
    • Realiza un commit de los cambios.
    • Cambia de nuevo a la rama main y fusiona los cambios de nueva-funcionalidad.

Solución del Ejercicio

  1. Crear un Nuevo Repositorio:

    mkdir mi-proyecto
cd mi-proyecto
git init

  2. Añadir y Confirmar Cambios:

    echo 'print("Hola, Mundo!")' > hola_mundo.py
git add hola_mundo.py
git commit -m "Añadir archivo hola_mundo.py"

  3. Crear una Nueva Rama:

    git checkout -b nueva-funcionalidad

  4. Modificar y Fusionar:

    echo 'print("Nueva funcionalidad añadida")' >> hola_mundo.py
git add hola_mundo.py
git commit -m "Añadir nueva funcionalidad a hola_mundo.py"
git checkout main
git merge nueva-funcionalidad

Resumen

En esta sección, hemos cubierto los conceptos básicos y comandos esenciales de Git, una herramienta fundamental para el control de versiones en el desarrollo de software. Aprendiste cómo configurar Git, crear y clonar repositorios, realizar commits, trabajar con ramas y fusionar cambios. Estos conocimientos te permitirán gestionar tus proyectos de programación de manera más eficiente y colaborativa.
