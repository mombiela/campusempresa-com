Introducción a Asyncio

Asyncio es una biblioteca en Python utilizada para escribir código concurrente usando la sintaxis async/await. Está diseñada para manejar operaciones de E/S asíncronas, lo que la hace ideal para tareas como solicitudes de red, E/S de archivos y otras operaciones que pueden realizarse concurrentemente.

Conceptos Clave

  • Programación Asíncrona: Un paradigma de programación que permite operaciones no bloqueantes, permitiendo que un programa maneje múltiples tareas a la vez.
  • Bucle de Eventos: El núcleo de cada aplicación asyncio. Ejecuta tareas y callbacks asíncronos, realiza operaciones de E/S de red y gestiona subprocesos.
  • Corutinas: Funciones especiales definidas con async def que pueden ser pausadas y reanudadas, permitiendo que otras operaciones se ejecuten concurrentemente.
  • Tareas: Envoltorios para corutinas que permiten que sean programadas y ejecutadas por el bucle de eventos.
  • Futuros: Objetos que representan el resultado de una operación asíncrona.

Configuración de Asyncio

Para usar asyncio, necesitas importarlo en tu script de Python:

import asyncio

Escribiendo Tu Primera Corutina

Una corutina se define usando la sintaxis async def. Aquí hay un ejemplo simple:

import asyncio

async def say_hello():
    print("Hola")
    await asyncio.sleep(1)
    print("Mundo")

# Ejecutando la corutina
asyncio.run(say_hello())

Explicación

  • async def say_hello(): Define una función asíncrona.
  • await asyncio.sleep(1): Pausa la corutina por 1 segundo, permitiendo que otras tareas se ejecuten.
  • asyncio.run(say_hello()): Ejecuta la corutina hasta que se complete.

Usando el Bucle de Eventos

El bucle de eventos es responsable de ejecutar tareas asíncronas. Puedes obtener el bucle de eventos actual usando asyncio.get_event_loop() y ejecutar tareas con loop.run_until_complete().

import asyncio

async def main():
    print("Inicio")
    await asyncio.sleep(2)
    print("Fin")

loop = asyncio.get_event_loop()
loop.run_until_complete(main())

Creando y Ejecutando Múltiples Tareas

Puedes ejecutar múltiples corutinas concurrentemente usando asyncio.create_task().

import asyncio

async def task1():
    await asyncio.sleep(1)
    print("Tarea 1 completada")

async def task2():
    await asyncio.sleep(2)
    print("Tarea 2 completada")

async def main():
    task1 = asyncio.create_task(task1())
    task2 = asyncio.create_task(task2())
    
    await task1
    await task2

asyncio.run(main())

Explicación

  • asyncio.create_task(): Programa la ejecución de una corutina.
  • await task1, await task2: Espera a que las tareas se completen.

Manejo de Excepciones en Asyncio

Puedes manejar excepciones en tareas asyncio usando bloques try-except.

import asyncio

async def faulty_task():
    await asyncio.sleep(1)
    raise ValueError("Ocurrió un error")

async def main():
    try:
        await faulty_task()
    except ValueError as e:
        print(f"Se capturó una excepción: {e}")

asyncio.run(main())

Uso Avanzado: Sincronización de Tareas

Asyncio proporciona varios primitivos de sincronización como asyncio.Lock, asyncio.Event y asyncio.Queue.

Ejemplo: Usando asyncio.Lock

import asyncio

lock = asyncio.Lock()

async def critical_section():
    async with lock:
        print("Entrando en sección crítica")
        await asyncio.sleep(1)
        print("Saliendo de sección crítica")

async def main():
    await asyncio.gather(critical_section(), critical_section())

asyncio.run(main())

Explicación

  • asyncio.Lock(): Crea un objeto de bloqueo.
  • async with lock: Adquiere el bloqueo antes de entrar en la sección crítica y lo libera después de salir.

Conclusión

Asyncio es una biblioteca poderosa para escribir código concurrente en Python. Al entender sus conceptos básicos como el bucle de eventos, corutinas, tareas y primitivos de sincronización, puedes manejar eficientemente operaciones asíncronas en tus aplicaciones.
