5-1: Clases y Objetos

Introducción

La Programación Orientada a Objetos (POO) es un paradigma de programación que utiliza "objetos" y "clases" para estructurar el código de manera más modular y reutilizable. En Python, la POO es una característica fundamental que permite crear programas más complejos y organizados.

Conceptos Clave

  1. Clase: Es una plantilla o molde que define las propiedades y comportamientos (métodos) que los objetos creados a partir de la clase tendrán.
  2. Objeto: Es una instancia de una clase. Cada objeto puede tener diferentes valores para las propiedades definidas en la clase.
  3. Atributos: Son las propiedades o datos que pertenecen a una clase o a un objeto.
  4. Métodos: Son funciones definidas dentro de una clase que describen los comportamientos de los objetos de esa clase.

Definición de una Clase

Para definir una clase en Python, se utiliza la palabra clave class seguida del nombre de la clase y dos puntos. Dentro de la clase, se definen los atributos y métodos.

class Persona:
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad

    def saludar(self):
        print(f"Hola, mi nombre es {self.nombre} y tengo {self.edad} años.")

Explicación del Código

  • class Persona:: Define una nueva clase llamada Persona.
  • def __init__(self, nombre, edad):: El método __init__ es un constructor que se llama automáticamente cuando se crea una nueva instancia de la clase. El parámetro self se refiere a la instancia actual de la clase.
  • self.nombre = nombre: Asigna el valor del parámetro nombre al atributo nombre del objeto.
  • self.edad = edad: Asigna el valor del parámetro edad al atributo edad del objeto.
  • def saludar(self):: Define un método llamado saludar que imprime un saludo utilizando los atributos del objeto.

Creación de Objetos

Para crear un objeto de una clase, se llama a la clase como si fuera una función, pasando los argumentos necesarios para el constructor.

persona1 = Persona("Juan", 30)
persona2 = Persona("Ana", 25)

Uso de Métodos

Una vez que se ha creado un objeto, se pueden llamar a sus métodos utilizando la notación de punto.

persona1.saludar()  # Salida: Hola, mi nombre es Juan y tengo 30 años.
persona2.saludar()  # Salida: Hola, mi nombre es Ana y tengo 25 años.

Ejercicio Práctico

Ejercicio 1

Define una clase Coche con los siguientes atributos y métodos:

  • Atributos:
    • marca
    • modelo
    • año
  • Métodos:
    • detalles: Imprime los detalles del coche.
    • arrancar: Imprime un mensaje indicando que el coche ha arrancado.
class Coche:
    def __init__(self, marca, modelo, año):
        self.marca = marca
        self.modelo = modelo
        self.año = año

    def detalles(self):
        print(f"Marca: {self.marca}, Modelo: {self.modelo}, Año: {self.año}")

    def arrancar(self):
        print(f"El coche {self.marca} {self.modelo} ha arrancado.")

Solución

# Crear instancias de la clase Coche
coche1 = Coche("Toyota", "Corolla", 2020)
coche2 = Coche("Ford", "Mustang", 2018)

# Llamar a los métodos
coche1.detalles()  # Salida: Marca: Toyota, Modelo: Corolla, Año: 2020
coche1.arrancar()  # Salida: El coche Toyota Corolla ha arrancado.

coche2.detalles()  # Salida: Marca: Ford, Modelo: Mustang, Año: 2018
coche2.arrancar()  # Salida: El coche Ford Mustang ha arrancado.

Errores Comunes y Consejos

  1. Olvidar el parámetro self en los métodos: Todos los métodos de instancia deben tener self como primer parámetro.
  2. No inicializar los atributos en el constructor: Asegúrate de inicializar todos los atributos necesarios en el método __init__.
  3. Confundir métodos de instancia con métodos de clase: Los métodos de instancia requieren self y operan sobre la instancia actual, mientras que los métodos de clase utilizan cls y operan sobre la clase en sí.

Conclusión

En esta sección, hemos aprendido los conceptos básicos de la Programación Orientada a Objetos en Python, incluyendo cómo definir clases, crear objetos y utilizar métodos. Estos fundamentos son esenciales para escribir código más modular y reutilizable. En la próxima sección, exploraremos los atributos y métodos en mayor detalle.
