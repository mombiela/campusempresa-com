Introducción

El lenguaje ensamblador es un lenguaje de programación de bajo nivel que está estrechamente relacionado con el código máquina. Proporciona una forma de escribir programas que son altamente eficientes y optimizados para hardware específico. En esta sección, cubriremos cómo escribir un programa simple de "Hola Mundo" en lenguaje ensamblador.

Conceptos Clave

  • Registros: Pequeñas ubicaciones de almacenamiento dentro de la CPU utilizadas para realizar operaciones.
  • Instrucciones: Comandos que le dicen a la CPU qué hacer.
  • Llamadas al Sistema: Solicitudes hechas por un programa al sistema operativo para realizar tareas específicas.

Escribiendo "Hola Mundo" en Ensamblador

Para escribir un programa de "Hola Mundo" en ensamblador, necesitamos entender la estructura básica de un programa en ensamblador y cómo usar llamadas al sistema para imprimir texto en la pantalla.

Estructura Básica de un Programa en Ensamblador

Un programa en ensamblador típicamente consta de las siguientes secciones:

  1. Sección de Datos: Define constantes y datos inicializados.
  2. Sección BSS: Define datos no inicializados.
  3. Sección de Texto: Contiene el código real.

Ejemplo: Programa "Hola Mundo"

A continuación se muestra un ejemplo de un programa "Hola Mundo" escrito en NASM (Netwide Assembler) para Linux:

section .data
    hello db 'Hola, Mundo!', 0x0A  ; Define la cadena con un carácter de nueva línea

section .text
    global _start                  ; Punto de entrada para el programa

_start:
    ; Escribir la cadena en stdout
    mov eax, 4                     ; Número de llamada al sistema para sys_write
    mov ebx, 1                     ; Descriptor de archivo 1 (stdout)
    mov ecx, hello                 ; Puntero a la cadena
    mov edx, 13                    ; Longitud de la cadena
    int 0x80                       ; Llamar al kernel

    ; Salir del programa
    mov eax, 1                     ; Número de llamada al sistema para sys_exit
    xor ebx, ebx                   ; Código de salida 0
    int 0x80                       ; Llamar al kernel

Explicación del Código

  • Sección de Datos:

    • hello db 'Hola, Mundo!', 0x0A: Define una cadena "Hola, Mundo!" seguida de un carácter de nueva línea (0x0A).

  • Sección de Texto:

    • global _start: Define el punto de entrada del programa.
    • _start:: Etiqueta que marca el comienzo del programa.
    • mov eax, 4: Mueve el número de llamada al sistema para sys_write al registro eax.
    • mov ebx, 1: Mueve el descriptor de archivo para stdout al registro ebx.
    • mov ecx, hello: Mueve la dirección de la cadena al registro ecx.
    • mov edx, 13: Mueve la longitud de la cadena al registro edx.
    • int 0x80: Interrupción para llamar al kernel y ejecutar la llamada al sistema.
    • mov eax, 1: Mueve el número de llamada al sistema para sys_exit al registro eax.
    • xor ebx, ebx: Establece el código de salida en 0 haciendo un XOR de ebx consigo mismo.
    • int 0x80: Interrupción para llamar al kernel y salir del programa.

Conclusión

Escribir un programa de "Hola Mundo" en lenguaje ensamblador proporciona una comprensión fundamental de cómo funciona la programación de bajo nivel. Al aprender a manipular registros y usar llamadas al sistema, obtienes una visión de los mecanismos internos de la CPU y el sistema operativo. Este conocimiento es crucial para escribir código eficiente y optimizado en ensamblador.
