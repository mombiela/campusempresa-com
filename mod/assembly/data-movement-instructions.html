Instrucciones de Movimiento de Datos
Instrucciones Aritméticas ►

Las instrucciones de movimiento de datos son fundamentales en la programación en lenguaje ensamblador. Permiten la transferencia de datos entre registros, memoria y puertos de E/S. Comprender estas instrucciones es crucial para una programación en ensamblador eficiente y efectiva.

Conceptos Clave

  • Registros: Pequeñas ubicaciones de almacenamiento dentro de la CPU utilizadas para la manipulación rápida de datos.
  • Memoria: Espacio de almacenamiento más grande utilizado para contener datos e instrucciones.
  • Valores Inmediatos: Valores constantes incrustados directamente dentro de las instrucciones.
  • Modos de Direccionamiento: Métodos para especificar la ubicación de los datos.

Instrucciones Básicas de Movimiento de Datos

Instrucción MOV

La instrucción MOV se utiliza para copiar datos de una ubicación a otra. Es la instrucción de movimiento de datos más comúnmente utilizada.

Sintaxis:

MOV destino, fuente

Ejemplos:

MOV AX, BX      ; Copiar el valor en BX a AX
MOV AX, 1234h   ; Cargar el valor inmediato 1234h en AX
MOV [SI], AX    ; Copiar el valor en AX a la ubicación de memoria apuntada por SI
MOV CX, [DI]    ; Copiar el valor de la ubicación de memoria apuntada por DI a CX

Instrucciones PUSH y POP

Las instrucciones PUSH y POP se utilizan para guardar y restaurar datos desde la pila.

Sintaxis:

PUSH fuente
POP destino

Ejemplos:

PUSH AX         ; Guardar el valor en AX en la pila
POP BX          ; Restaurar el valor de la pila en BX

Instrucción XCHG

La instrucción XCHG intercambia los valores de dos operandos.

Sintaxis:

XCHG operando1, operando2

Ejemplos:

XCHG AX, BX     ; Intercambiar los valores de AX y BX
XCHG [SI], AX   ; Intercambiar el valor en AX con el valor en la ubicación de memoria apuntada por SI

Instrucciones Avanzadas de Movimiento de Datos

Instrucción LEA

La instrucción LEA (Load Effective Address) carga la dirección del operando fuente en el registro de destino.

Sintaxis:

LEA destino, fuente

Ejemplos:

LEA BX, [SI+2]  ; Cargar la dirección efectiva de SI+2 en BX
LEA AX, [BX+DI] ; Cargar la dirección efectiva de BX+DI en AX

Instrucciones LDS y LES

Las instrucciones LDS (Load Pointer Using DS) y LES (Load Pointer Using ES) cargan un puntero lejano desde la memoria en el registro especificado y en el registro de segmento DS o ES.

Sintaxis:

LDS destino, fuente
LES destino, fuente

Ejemplos:

LDS SI, [1234h] ; Cargar un puntero lejano desde la ubicación de memoria 1234h en SI y DS
LES DI, [5678h] ; Cargar un puntero lejano desde la ubicación de memoria 5678h en DI y ES

Modos de Direccionamiento

Comprender los modos de direccionamiento es esencial para un movimiento de datos efectivo en lenguaje ensamblador.

  • Direccionamiento Inmediato: El operando es un valor constante.

    MOV AX, 5      ; El valor inmediato 5 se mueve a AX

  • Direccionamiento de Registro: El operando es un registro.

    MOV AX, BX     ; El valor en BX se mueve a AX

  • Direccionamiento Directo: El operando es una dirección de memoria.

    MOV AX, [1234h] ; El valor en la dirección de memoria 1234h se mueve a AX

  • Direccionamiento Indirecto: El operando es una ubicación de memoria apuntada por un registro.

    MOV AX, [SI]   ; El valor en la ubicación de memoria apuntada por SI se mueve a AX

  • Direccionamiento Indexado: El operando es una ubicación de memoria calculada usando un registro base y un índice.

    MOV AX, [BX+SI] ; El valor en la ubicación de memoria calculada por BX+SI se mueve a AX

Conclusión

Las instrucciones de movimiento de datos son la columna vertebral de la programación en lenguaje ensamblador. Permiten la transferencia de datos entre registros, memoria y puertos de E/S, lo cual es esencial para la operación de cualquier programa. Al dominar estas instrucciones y comprender los diversos modos de direccionamiento, puedes escribir código en ensamblador más eficiente y efectivo. Practica estas instrucciones con diferentes modos de direccionamiento para obtener una comprensión más profunda y mejorar tus habilidades de programación.
