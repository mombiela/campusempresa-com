Introducción

En esta sección, cubriremos el proceso de envío y revisión de tus proyectos en lenguaje Assembly. Entender este proceso es crucial para asegurar que tu código cumpla con los estándares requeridos y reciba retroalimentación constructiva.

Conceptos Clave

  • Directrices de Envío: Reglas y formatos para enviar tu código Assembly.
  • Criterios de Revisión: Estándares utilizados para evaluar tu código.
  • Mecanismos de Retroalimentación: Cómo se proporciona la retroalimentación y cómo usarla para mejorar.

Directrices de Envío

Al enviar tus proyectos en Assembly, sigue estas directrices para asegurar que tu código sea revisado de manera eficiente:

  1. Convenciones de Nombres de Archivos:

    • Usa nombres descriptivos para tus archivos.
    • Incluye tu nombre o identificador en el nombre del archivo.
    • Ejemplo: juan_perez_proyecto1.asm

  2. Documentación del Código:

    • Comenta tu código de manera exhaustiva.
    • Incluye un comentario de encabezado con tu nombre, fecha y una breve descripción del programa.
    • Ejemplo: 
      ; Autor: Juan Pérez
; Fecha: 2023-10-01
; Descripción: Este programa suma dos números y almacena el resultado en un registro.

  3. Formato de Envío:

    • Envía tu código en un solo archivo .asm a menos que se especifique lo contrario.
    • Si se requieren múltiples archivos, comprímelos en un archivo .zip o .tar.

  4. Compatibilidad de la Plataforma:

    • Asegúrate de que tu código se ejecute en el ensamblador y plataforma especificados.
    • Menciona cualquier requisito o dependencia específica en tu envío.

Criterios de Revisión

Tu código Assembly será revisado basado en los siguientes criterios:

  • Corrección:

    • ¿El código realiza la tarea prevista correctamente?
    • ¿Hay errores lógicos o bugs?

  • Eficiencia:

    • ¿El código está optimizado para el rendimiento?
    • ¿Hay instrucciones innecesarias u operaciones redundantes?

  • Legibilidad:

    • ¿El código está bien organizado y es fácil de leer?
    • ¿Los comentarios y la documentación son suficientes?

  • Adherencia a las Directrices:

    • ¿El código sigue las directrices de envío?
    • ¿Se cumplen las convenciones de nombres y los estándares de formato?

Mecanismos de Retroalimentación

Después de que tu código sea revisado, recibirás retroalimentación. Aquí te explicamos cómo interpretarla y usarla:

  1. Formato de Retroalimentación:

    • La retroalimentación se proporcionará como comentarios en línea dentro de tu código o como un documento de revisión separado.
    • Busca comentarios que indiquen áreas de mejora o que destaquen secciones bien escritas.

  2. Puntos Comunes de Retroalimentación:

    • Errores Lógicos: Errores en la lógica de tu código.
    • Sugerencias de Optimización: Consejos para hacer tu código más eficiente.
    • Mejoras en la Legibilidad: Sugerencias para mejorar los comentarios o la estructura de tu código.

  3. Usar la Retroalimentación para Mejorar:

    • Revisa toda la retroalimentación cuidadosamente y entiende las sugerencias.
    • Realiza los cambios necesarios en tu código basándote en la retroalimentación.
    • Vuelve a enviar tu código si es necesario, incorporando la retroalimentación.

Ejemplo de Envío y Revisión

Ejemplo de Envío de Código

; Autor: Juan Pérez
; Fecha: 2023-10-01
; Descripción: Este programa suma dos números y almacena el resultado en un registro.

section .data
    num1 db 5
    num2 db 10

section .text
    global _start

_start:
    mov al, [num1]   ; Cargar num1 en el registro AL
    add al, [num2]   ; Sumar num2 a AL
    ; El resultado ahora está en AL

    ; Salir del programa
    mov eax, 1       ; Número de llamada al sistema (sys_exit)
    int 0x80         ; Llamar al kernel

Ejemplo de Retroalimentación de Revisión

  • Corrección: El código suma correctamente dos números y almacena el resultado en el registro AL.
  • Eficiencia: Considera usar movzx para extender a cero los valores byte y evitar posibles problemas con números más grandes.
  • Legibilidad: Buen uso de comentarios, pero considera agregar más detalles sobre el propósito de cada sección.
  • Adherencia a las Directrices: El nombre del archivo y la documentación son apropiados.

Conclusión

Entender el proceso de envío y revisión es esencial para mejorar tus habilidades de programación en Assembly. Siguiendo las directrices de envío, cumpliendo con los criterios de revisión y utilizando efectivamente la retroalimentación, puedes mejorar la calidad de tu código y aprender de cada proyecto. Este proceso no solo te ayuda a escribir mejor código, sino que también te prepara para escenarios de programación del mundo real donde las revisiones de código son una práctica estándar.
