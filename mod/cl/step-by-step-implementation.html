Introducción al CL (Lenguaje de Control)

El Lenguaje de Control (CL) es un lenguaje de scripting utilizado en los sistemas iSeries (AS/400) de IBM. Se utiliza principalmente para operaciones del sistema, control de trabajos y tareas de automatización. Esta sección te guiará a través de la implementación paso a paso de programas CL, desde niveles principiantes hasta avanzados.

Nivel Principiante

Entendiendo los Conceptos Básicos

  • ¿Qué es CL?

    • CL se utiliza para controlar la ejecución de programas y comandos del sistema en los sistemas IBM iSeries.
    • Es similar a los scripts de shell en Unix/Linux pero específico para el entorno de IBM.

  • Estructura Básica de un Programa CL

    • Un programa CL consiste en una serie de comandos que se ejecutan secuencialmente.
    • Cada comando realiza una tarea específica, como llamar a otro programa, manipular archivos o controlar el flujo de trabajos.

Escribiendo Tu Primer Programa CL

  • Ejemplo: Programa Hola Mundo 
    PGM
DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
CHGVAR VAR(&MSG) VALUE('¡Hola, Mundo!')
SNDPGMMSG MSG(&MSG)
ENDPGM
    • Explicación:
      • PGM y ENDPGM definen el inicio y el fin del programa.
      • DCL declara una variable &MSG de tipo carácter con una longitud de 50.
      • CHGVAR cambia el valor de &MSG a '¡Hola, Mundo!'.
      • SNDPGMMSG envía el mensaje almacenado en &MSG a la cola de mensajes del programa.

Ejecutando un Programa CL

  • Pasos para Ejecutar:
    1. Guarda el código fuente del programa en un archivo físico de origen.
    2. Compila el código fuente usando el comando CRTCLPGM.
    3. Ejecuta el programa compilado usando el comando CALL.

Nivel Intermedio

Variables y Tipos de Datos

  • Tipos de Datos Comunes:

    • *CHAR (Carácter)
    • *DEC (Decimal)
    • *INT (Entero)
    • *LGL (Lógico)

  • Ejemplo: Usando Diferentes Tipos de Datos

    PGM
DCL VAR(&NAME) TYPE(*CHAR) LEN(20)
DCL VAR(&AGE) TYPE(*DEC) LEN(3 0)
DCL VAR(&ISADULT) TYPE(*LGL)

CHGVAR VAR(&NAME) VALUE('John Doe')
CHGVAR VAR(&AGE) VALUE(25)
CHGVAR VAR(&ISADULT) VALUE('1')

IF COND(&AGE *GE 18) THEN(DO)
  SNDPGMMSG MSG('Adulto: ' *CAT &NAME)
ENDDO
ENDPGM
    • Explicación:
      • Declara variables para nombre, edad y una bandera lógica para la adultez.
      • Verifica si la edad es mayor o igual a 18 y envía un mensaje en consecuencia.

Estructuras de Control

  • Declaraciones Condicionales:

    • IF, ELSE, DO, ENDDO

  • Bucles:

    • DOFOR, DOWHILE, DOUNTIL

  • Ejemplo: Bucle con DOFOR

    PGM
DCL VAR(&I) TYPE(*INT) LEN(3)

DOFOR VAR(&I) FROM(1) TO(5)
  SNDPGMMSG MSG('Iteración: ' *CAT &I)
ENDDO
ENDPGM
    • Explicación:
      • Bucle de 1 a 5, enviando un mensaje para cada iteración.

Nivel Avanzado

Manejo Avanzado de Errores

  • Usando MONMSG (Monitor de Mensajes)

    • MONMSG se utiliza para manejar errores y excepciones en programas CL.

  • Ejemplo: Manejo de Errores

    PGM
DCL VAR(&FILE) TYPE(*CHAR) LEN(10)

CHGVAR VAR(&FILE) VALUE('MYFILE')
OVRDBF FILE(&FILE) TOFILE(LIBRARY/&FILE)
MONMSG MSGID(CPF0000) EXEC(DO)
  SNDPGMMSG MSG('Error al sobrescribir el archivo: ' *CAT &FILE)
ENDDO

ENDPGM
    • Explicación:
      • Intenta sobrescribir un archivo de base de datos y monitorea cualquier error.
      • Si ocurre un error, envía un mensaje de error.

Operaciones Avanzadas con Archivos

  • Lectura y Escritura de Archivos

    • Usa comandos como RCVF (Recibir Archivo) y SNDF (Enviar Archivo) para operaciones con archivos.

  • Ejemplo: Leyendo un Archivo

    PGM
DCLF FILE(MYLIB/MYFILE)

RCVF
MONMSG MSGID(CPF0864) EXEC(GOTO CMDLBL(END))

SNDPGMMSG MSG('Registro: ' *CAT &FIELD1 *CAT ' ' *CAT &FIELD2)

ENDPGM
    • Explicación:
      • Declara un archivo y lee un registro.
      • Si se alcanza el fin del archivo, salta al final del programa.
      • Envía un mensaje con los campos del registro.

Conclusión

En este tema, cubrimos la implementación paso a paso de programas CL, comenzando desde conceptos básicos y progresando hacia características más avanzadas. Al entender la estructura, variables, estructuras de control, manejo de errores y operaciones con archivos, puedes crear programas CL robustos y eficientes para automatizar tareas en sistemas IBM iSeries. Practica con los ejemplos proporcionados y construye gradualmente scripts más complejos para mejorar tus habilidades.
