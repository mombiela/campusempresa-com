Introducción a CL (Lenguaje de Control)

El Lenguaje de Control (CL) es un lenguaje de scripting utilizado en los sistemas AS/400 de IBM (ahora conocidos como IBM i). Se utiliza principalmente para controlar la ejecución de programas y gestionar los recursos del sistema. Comprender la sintaxis y estructura básica de CL es esencial para cualquiera que desee trabajar con sistemas IBM i.

Conceptos Clave

  • Comandos: Las unidades fundamentales de CL, utilizadas para realizar tareas específicas.
  • Sentencias: Instrucciones escritas en CL que pueden incluir comandos, etiquetas y estructuras de control.
  • Variables: Utilizadas para almacenar datos que pueden ser manipulados dentro de un programa CL.
  • Estructuras de Control: Construcciones como bucles y condicionales que controlan el flujo del programa.

Sintaxis Básica

Comandos

Los comandos en CL son instrucciones predefinidas que realizan tareas específicas. Cada comando tiene una sintaxis específica y un conjunto de parámetros.

Ejemplo:

SNDMSG MSG('¡Hola, Mundo!') TOUSR(USER1)

Explicación:

  • SNDMSG es el comando para enviar un mensaje.
  • MSG es un parámetro que especifica el contenido del mensaje.
  • TOUSR es un parámetro que especifica el destinatario.

Sentencias

Un programa CL consiste en una serie de sentencias. Cada sentencia puede ser un comando, una etiqueta o una estructura de control.

Ejemplo:

PGM
    SNDMSG MSG('Iniciando el proceso') TOUSR(USER1)
    /* Comandos adicionales van aquí */
ENDPGM

Explicación:

  • PGM marca el inicio de un programa CL.
  • SNDMSG es una sentencia de comando.
  • ENDPGM marca el final del programa CL.

Variables

Las variables en CL se utilizan para almacenar datos que pueden ser utilizados y manipulados dentro del programa. Se definen utilizando el comando DCL (Declarar).

Ejemplo:

DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
CHGVAR VAR(&MSG) VALUE('¡Hola, Mundo!')
SNDMSG MSG(&MSG) TOUSR(USER1)

Explicación:

  • DCL declara una variable llamada &MSG de tipo carácter con una longitud de 50.
  • CHGVAR cambia el valor de la variable &MSG a '¡Hola, Mundo!'.
  • SNDMSG envía el mensaje almacenado en &MSG a USER1.

Estructuras de Control

Las estructuras de control en CL incluyen bucles y sentencias condicionales que controlan el flujo del programa.

Sentencia IF

La sentencia IF se utiliza para ejecutar un bloque de código solo si una condición especificada es verdadera.

Ejemplo:

DCL VAR(&STATUS) TYPE(*CHAR) LEN(10)
CHGVAR VAR(&STATUS) VALUE('SUCCESS')

IF COND(&STATUS *EQ 'SUCCESS') THEN(DO)
    SNDMSG MSG('Proceso completado con éxito') TOUSR(USER1)
ENDDO

Explicación:

  • IF COND verifica si la variable &STATUS es igual a 'SUCCESS'.
  • THEN(DO) comienza un bloque de código para ejecutar si la condición es verdadera.
  • ENDDO marca el final del bloque.

Bucle DO

El bucle DO se utiliza para repetir un bloque de código un número especificado de veces.

Ejemplo:

DCL VAR(&I) TYPE(*DEC) LEN(2 0)
DCL VAR(&MSG) TYPE(*CHAR) LEN(50)

DOFOR VAR(&I) FROM(1) TO(5)
    CHGVAR VAR(&MSG) VALUE('Iteración ' *CAT &I)
    SNDMSG MSG(&MSG) TOUSR(USER1)
ENDDO

Explicación:

  • DOFOR inicia un bucle con la variable &I que va de 1 a 5.
  • CHGVAR cambia el valor de &MSG a 'Iteración ' seguido del valor actual de &I.
  • SNDMSG envía el mensaje almacenado en &MSG a USER1.
  • ENDDO marca el final del bucle.

Conclusión

Comprender la sintaxis y estructura básica de CL es crucial para escribir y gestionar eficazmente scripts en sistemas IBM i. Esto incluye saber cómo usar comandos, sentencias, variables y estructuras de control. Con estos fundamentos, puedes comenzar a crear programas CL más complejos y funcionales.
