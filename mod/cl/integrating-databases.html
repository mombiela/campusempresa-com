Integración con Bases de Datos
Estudios de Caso y Mejores Prácticas ►

Introducción

La integración con bases de datos es un aspecto crucial al trabajar con Control Language (CL) en sistemas IBM i. Este tema te guiará a través de los conceptos básicos de la integración de bases de datos, progresando hacia técnicas más avanzadas. Al final de esta sección, serás capaz de realizar operaciones de bases de datos utilizando comandos CL de manera efectiva.

Conceptos Clave

  • Archivos de Base de Datos: Entender los tipos de archivos de base de datos (físicos y lógicos) y sus roles.
  • Comandos CL: Aprender los comandos CL utilizados para operaciones de bases de datos.
  • Manejo de Errores: Implementar manejo de errores en operaciones de bases de datos.
  • Técnicas Avanzadas: Explorar técnicas avanzadas de integración de bases de datos.

Archivos de Base de Datos

Archivos Físicos

Los archivos físicos son los archivos de base de datos reales que almacenan datos.

CRTPF FILE(MYLIB/MYFILE) RCDLEN(100)
  • CRTPF: Comando para crear un archivo físico.
  • FILE: Especifica la biblioteca y el nombre del archivo.
  • RCDLEN: Define la longitud del registro.

Archivos Lógicos

Los archivos lógicos proporcionan una vista diferente o una ruta de acceso a los datos en archivos físicos.

CRTLF FILE(MYLIB/MYLOGICAL) SRCFILE(MYLIB/QDDSSRC)
  • CRTLF: Comando para crear un archivo lógico.
  • FILE: Especifica la biblioteca y el nombre del archivo lógico.
  • SRCFILE: Especifica el archivo fuente que contiene las DDS (Especificaciones de Descripción de Datos).

Comandos CL para Operaciones de Base de Datos

Añadir Registros

Usa el comando ADDPFTRG para añadir registros a un archivo físico.

ADDPFTRG FILE(MYLIB/MYFILE) TRGTIME(*AFTER) TRGEVENT(*INSERT) PGM(MYLIB/MYPGM)
  • ADDPFTRG: Comando para añadir un disparador a un archivo físico.
  • FILE: Especifica la biblioteca y el nombre del archivo.
  • TRGTIME: Especifica cuándo debe activarse el disparador.
  • TRGEVENT: Especifica el evento que activa el disparador.
  • PGM: Especifica el programa a ejecutar cuando se activa el disparador.

Actualizar Registros

Usa el comando UPDPF para actualizar registros en un archivo físico.

UPDPF FILE(MYLIB/MYFILE) MBR(MYMBR) FROMFILE(MYLIB/NEWFILE)
  • UPDPF: Comando para actualizar un archivo físico.
  • FILE: Especifica la biblioteca y el nombre del archivo.
  • MBR: Especifica el miembro a actualizar.
  • FROMFILE: Especifica el archivo fuente para la actualización.

Eliminar Registros

Usa el comando DLTPF para eliminar registros de un archivo físico.

DLTPF FILE(MYLIB/MYFILE)
  • DLTPF: Comando para eliminar un archivo físico.
  • FILE: Especifica la biblioteca y el nombre del archivo.

Manejo de Errores

Implementar manejo de errores asegura que tus programas CL puedan gestionar excepciones de manera elegante.

MONMSG MSGID(CPF0000) EXEC(DO)
  /* Código de manejo de errores aquí */
ENDDO
  • MONMSG: Comando para monitorear mensajes.
  • MSGID: Especifica el ID del mensaje a monitorear.
  • EXEC: Especifica el comando a ejecutar si se recibe el mensaje.

Técnicas Avanzadas

Usar SQL en CL

Puedes incrustar sentencias SQL dentro de programas CL para operaciones de bases de datos más complejas.

RUNSQL SQL('INSERT INTO MYLIB.MYTABLE (COL1, COL2) VALUES (''Value1'', ''Value2'')')
  • RUNSQL: Comando para ejecutar una sentencia SQL.
  • SQL: Especifica la sentencia SQL a ejecutar.

Usar Áreas de Datos

Las áreas de datos pueden ser usadas para almacenar y recuperar datos de manera eficiente.

CRTDTAARA DTAARA(MYLIB/MYDATA) TYPE(*CHAR) LEN(100)
  • CRTDTAARA: Comando para crear un área de datos.
  • DTAARA: Especifica la biblioteca y el nombre del área de datos.
  • TYPE: Especifica el tipo de datos.
  • LEN: Especifica la longitud del área de datos.

Conclusión

Integrarse con bases de datos usando CL implica entender los tipos de archivos de base de datos, dominar los comandos CL esenciales para operaciones de bases de datos, implementar manejo de errores y explorar técnicas avanzadas como usar SQL dentro de programas CL. Con estas habilidades, puedes gestionar y manipular datos de manera eficiente en sistemas IBM i.
