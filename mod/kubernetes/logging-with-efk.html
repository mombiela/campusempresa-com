Introducción a la pila EFK

La pila EFK es un conjunto poderoso de herramientas utilizadas para el registro y monitoreo en entornos de Kubernetes. Consiste en:

  • Elasticsearch: Un motor de búsqueda y análisis.
  • Fluentd: Un recolector de datos para registro unificado.
  • Kibana: Una herramienta de visualización para datos de Elasticsearch.

¿Por qué usar EFK en Kubernetes?

  • Registro centralizado: Recopila registros de todos los pods y nodos en un solo lugar.
  • Escalabilidad: Maneja grandes volúmenes de registros de manera eficiente.
  • Búsqueda y análisis: Usa Elasticsearch para buscar y analizar registros.
  • Visualización: Usa Kibana para crear paneles y visualizar datos de registros.

Configuración de la pila EFK en Kubernetes

Requisitos previos

  • Un clúster de Kubernetes en funcionamiento.
  • La herramienta de línea de comandos kubectl configurada para interactuar con tu clúster.

Paso 1: Desplegar Elasticsearch

Elasticsearch almacenará e indexará los registros recopilados del clúster de Kubernetes.

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: elasticsearch
spec:
  serviceName: "elasticsearch"
  replicas: 1
  selector:
    matchLabels:
      app: elasticsearch
  template:
    metadata:
      labels:
        app: elasticsearch
    spec:
      containers:
      - name: elasticsearch
        image: docker.elastic.co/elasticsearch/elasticsearch:7.10.1
        ports:
        - containerPort: 9200
          name: http
        - containerPort: 9300
          name: transport
        env:
        - name: discovery.type
          value: single-node

Paso 2: Desplegar Fluentd

Fluentd recopilará registros de los nodos de Kubernetes y los reenviará a Elasticsearch.

apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: fluentd
  labels:
    k8s-app: fluentd-logging
spec:
  selector:
    matchLabels:
      name: fluentd
  template:
    metadata:
      labels:
        name: fluentd
    spec:
      containers:
      - name: fluentd
        image: fluent/fluentd:v1.11.5-debian-1.0
        env:
        - name: FLUENT_ELASTICSEARCH_HOST
          value: "elasticsearch"
        - name: FLUENT_ELASTICSEARCH_PORT
          value: "9200"
        volumeMounts:
        - name: varlog
          mountPath: /var/log
        - name: varlibdockercontainers
          mountPath: /var/lib/docker/containers
          readOnly: true
      volumes:
      - name: varlog
        hostPath:
          path: /var/log
      - name: varlibdockercontainers
        hostPath:
          path: /var/lib/docker/containers

Paso 3: Desplegar Kibana

Kibana proporcionará una interfaz web para visualizar los registros almacenados en Elasticsearch.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: kibana
spec:
  replicas: 1
  selector:
    matchLabels:
      app: kibana
  template:
    metadata:
      labels:
        app: kibana
    spec:
      containers:
      - name: kibana
        image: docker.elastic.co/kibana/kibana:7.10.1
        ports:
        - containerPort: 5601
        env:
        - name: ELASTICSEARCH_HOSTS
          value: "http://elasticsearch:9200"

Paso 4: Exponer Kibana

Crea un servicio para exponer Kibana y poder acceder a él desde tu navegador.

apiVersion: v1
kind: Service
metadata:
  name: kibana
spec:
  selector:
    app: kibana
  ports:
  - protocol: TCP
    port: 5601
    targetPort: 5601
  type: NodePort

Accediendo a Kibana

  • Obtén el NodePort asignado al servicio Kibana: 
    kubectl get svc kibana
  • Accede a Kibana usando la IP del nodo y el NodePort en tu navegador: 
    http://<NodeIP>:<NodePort>

Conclusión

La pila EFK proporciona una solución robusta para el registro y monitoreo en entornos de Kubernetes. Al desplegar Elasticsearch, Fluentd y Kibana, puedes centralizar, buscar y visualizar registros de todo tu clúster de Kubernetes, facilitando el monitoreo y la resolución de problemas de tus aplicaciones.
