Introducción al Trazado Distribuido

El trazado distribuido es un método utilizado para rastrear y monitorear solicitudes a medida que fluyen a través de varios servicios en una arquitectura de microservicios. Ayuda a identificar cuellos de botella en el rendimiento, comprender las dependencias de los servicios y depurar problemas complejos.

¿Qué es Jaeger?

Jaeger es una herramienta de trazado distribuido de extremo a extremo de código abierto que ayuda a monitorear y solucionar problemas en sistemas distribuidos basados en microservicios. Se utiliza para:

  • Optimización del rendimiento
  • Análisis de la causa raíz
  • Análisis de dependencias de servicios

Configuración de Jaeger en Kubernetes

Para comenzar a usar Jaeger en tu clúster de Kubernetes, necesitas desplegar los componentes de Jaeger. Esto se puede hacer utilizando Helm, un gestor de paquetes para Kubernetes.

Prerrequisitos

  • Un clúster de Kubernetes en funcionamiento
  • Helm instalado en tu máquina local

Instalación de Jaeger con Helm

  1. Agregar el repositorio Helm de Jaeger:

    helm repo add jaegertracing https://jaegertracing.github.io/helm-charts
helm repo update

  2. Instalar Jaeger:

    helm install jaeger jaegertracing/jaeger

  3. Verificar la instalación:

    kubectl get all -l app.kubernetes.io/instance=jaeger

Instrumentación de tu Aplicación

Para trazar solicitudes, necesitas instrumentar el código de tu aplicación para enviar datos de trazado a Jaeger.

Ejemplo: Instrumentación de una Aplicación en Python

  1. Instalar la biblioteca cliente de Jaeger:

    pip install jaeger-client

  2. Agregar trazado a tu aplicación:

    from jaeger_client import Config
from flask import Flask

app = Flask(__name__)

def init_tracer(service_name='my-service'):
    config = Config(
        config={
            'sampler': {'type': 'const', 'param': 1},
            'logging': True,
        },
        service_name=service_name,
    )
    return config.initialize_tracer()

tracer = init_tracer()

@app.route('/')
def hello():
    with tracer.start_span('hello-world') as span:
        span.set_tag('example-tag', 'hello-world')
        return 'Hello, World!'

if __name__ == '__main__':
    app.run()

Visualización de Trazas en la Interfaz de Jaeger

Una vez que tu aplicación esté instrumentada y en funcionamiento, puedes ver las trazas en la interfaz de Jaeger.

  1. Acceder a la interfaz de Jaeger:

    kubectl port-forward svc/jaeger-query 16686:16686

    Abre tu navegador y navega a http://localhost:16686.

  2. Buscar trazas:

    • Selecciona tu servicio del menú desplegable.
    • Haz clic en "Find Traces" para ver los datos de trazado.

Configuración Avanzada

Jaeger proporciona varias opciones de configuración para personalizar su comportamiento.

Estrategias de Muestreo

El muestreo determina qué trazas se recopilan y envían a Jaeger. Las estrategias comunes incluyen:

  • Muestreo Constante: Siempre muestrea un porcentaje fijo de trazas.
  • Muestreo Probabilístico: Muestrea trazas basadas en una probabilidad.

Ejemplo: Configuración de Muestreo en Kubernetes

  1. Crear un ConfigMap para el muestreo:

    apiVersion: v1
kind: ConfigMap
metadata:
  name: jaeger-sampling
data:
  sampling.json: |
    {
      "service_strategies": [
        {
          "service": "my-service",
          "type": "probabilistic",
          "param": 0.1
        }
      ]
    }

  2. Aplicar el ConfigMap:

    kubectl apply -f jaeger-sampling-configmap.yaml

  3. Actualizar el despliegue de Jaeger para usar el ConfigMap:

    spec:
  containers:
  - name: jaeger-agent
    volumeMounts:
    - name: sampling-config
      mountPath: /etc/jaeger/sampling
      subPath: sampling.json
  volumes:
  - name: sampling-config
    configMap:
      name: jaeger-sampling

Conclusión

El trazado con Jaeger en un entorno de Kubernetes proporciona poderosas ideas sobre tu arquitectura de microservicios. Siguiendo los pasos descritos, puedes configurar Jaeger, instrumentar tus aplicaciones y comenzar a analizar trazas para optimizar el rendimiento y solucionar problemas. Las configuraciones avanzadas como las estrategias de muestreo te permiten ajustar el trazado para adaptarlo a tus necesidades.
