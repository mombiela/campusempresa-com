Introducción a Helm

Helm es un gestor de paquetes para Kubernetes que te ayuda a gestionar aplicaciones en Kubernetes. Helm utiliza un formato de empaquetado llamado charts, que son colecciones de archivos que describen un conjunto relacionado de recursos de Kubernetes.

Conceptos Clave

  • Charts: Paquetes de Helm que contienen todas las definiciones de recursos necesarias para ejecutar una aplicación, herramienta o servicio dentro de un clúster de Kubernetes.
  • Repositorios: Ubicaciones donde se almacenan y comparten los charts.
  • Releases: Instancias de charts que se ejecutan en un clúster de Kubernetes.

Instalación de Helm

Para comenzar con Helm, necesitas instalarlo en tu máquina local. Sigue estos pasos:

  1. Descargar Helm:

    curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

  2. Verificar la Instalación:

    helm version

Charts de Helm

Los charts de Helm son la piedra angular de Helm. Definen la estructura y configuración de tus aplicaciones en Kubernetes.

Creación de un Chart de Helm

Para crear un nuevo chart de Helm, utiliza el siguiente comando:

helm create mychart

Este comando genera una estructura de directorios como esta:

mychart/
  Chart.yaml          # Un archivo YAML que contiene información sobre el chart
  values.yaml         # Los valores de configuración predeterminados para este chart
  charts/             # Un directorio que contiene cualquier chart del cual depende este chart
  templates/          # Un directorio de plantillas que, cuando se combinan con valores, generan archivos de manifiesto válidos de Kubernetes

Entendiendo Chart.yaml

El archivo Chart.yaml contiene metadatos sobre el chart:

apiVersion: v2
name: mychart
description: Un chart de Helm para Kubernetes
version: 0.1.0
appVersion: "1.0"

Uso de values.yaml

El archivo values.yaml te permite especificar valores predeterminados para tus plantillas:

replicaCount: 1
image:
  repository: nginx
  tag: stable
  pullPolicy: IfNotPresent
service:
  type: ClusterIP
  port: 80

Despliegue de Aplicaciones con Helm

Desplegar una aplicación usando Helm es sencillo. Puedes usar el comando helm install para desplegar un chart.

Instalación de un Chart

helm install myrelease mychart
  • myrelease: El nombre que deseas darle a esta release.
  • mychart: La ruta al chart que deseas instalar.

Actualización de una Release

Para actualizar una release existente, usa el comando helm upgrade:

helm upgrade myrelease mychart

Desinstalación de una Release

Para desinstalar una release, usa el comando helm uninstall:

helm uninstall myrelease

Repositorios de Helm

Los repositorios de Helm son ubicaciones donde se almacenan y comparten los charts. El repositorio predeterminado de Helm es Helm Hub, pero también puedes agregar otros repositorios.

Adición de un Repositorio

helm repo add stable https://charts.helm.sh/stable

Listado de Repositorios

helm repo list

Búsqueda de Charts

helm search repo nginx

Funciones Avanzadas de Helm

Helm ofrece varias funciones avanzadas que pueden ayudarte a gestionar aplicaciones complejas.

Hooks de Helm

Los hooks de Helm te permiten intervenir en ciertos puntos del ciclo de vida de una release, como antes o después de una actualización.

Plugins de Helm

Los plugins de Helm extienden la funcionalidad de Helm. Puedes instalar plugins usando el comando helm plugin.

Plantillas de Helm

Helm utiliza plantillas de Go para generar manifiestos de Kubernetes. Puedes usar funciones de plantillas para manipular datos y crear configuraciones dinámicas.

Ejemplo de una Plantilla

apiVersion: v1
kind: Pod
metadata:
  name: {{ .Release.Name }}-pod
spec:
  containers:
    - name: {{ .Chart.Name }}
      image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"

Conclusión

Helm es una herramienta poderosa para gestionar aplicaciones en Kubernetes. Simplifica el proceso de despliegue y proporciona una forma consistente de empaquetar y compartir aplicaciones. Al entender los charts de Helm, los repositorios y las funciones avanzadas, puedes gestionar eficazmente tus aplicaciones en Kubernetes desde el desarrollo hasta la producción.
