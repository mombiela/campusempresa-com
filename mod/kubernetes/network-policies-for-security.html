Introducción

Las Políticas de Red de Kubernetes son cruciales para asegurar tu clúster controlando la comunicación entre pods y servicios. Este tema te guiará desde los conceptos básicos hasta los avanzados, asegurando que entiendas cómo implementar y gestionar políticas de red de manera efectiva.

¿Qué son las Políticas de Red?

Las Políticas de Red en Kubernetes son reglas que definen cómo los pods se comunican entre sí y con otros puntos finales de la red. Se implementan utilizando el recurso NetworkPolicy de Kubernetes.

Conceptos Clave

  • Selector de Pods: Especifica el grupo de pods al que se aplica la política.
  • Reglas de Ingreso: Definen el tráfico entrante permitido hacia los pods seleccionados.
  • Reglas de Egreso: Definen el tráfico saliente permitido desde los pods seleccionados.
  • Namespaces: Las políticas de red pueden aplicarse dentro de un namespace específico o a través de múltiples namespaces.

Ejemplo Básico de Política de Red

Comencemos con un ejemplo simple para permitir el tráfico solo desde pods específicos.

Ejemplo: Permitir Tráfico desde Pods Específicos

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-specific-pods
  namespace: default
spec:
  podSelector:
    matchLabels:
      role: db
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: frontend

Explicación

  • apiVersion: Especifica la versión de la API.
  • kind: Define el tipo de recurso, que es NetworkPolicy.
  • metadata: Contiene el nombre y el namespace de la política.
  • spec: La especificación de la política.
    • podSelector: Selecciona pods con la etiqueta role: db.
    • ingress: Permite el tráfico entrante desde pods con la etiqueta role: frontend.

Conceptos Avanzados de Políticas de Red

A medida que te sientas más cómodo con las políticas básicas, puedes explorar características más avanzadas.

Combinando Reglas de Ingreso y Egreso

Puedes definir tanto reglas de ingreso como de egreso en una sola política.

Ejemplo: Reglas Combinadas de Ingreso y Egreso

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: combined-policy
  namespace: default
spec:
  podSelector:
    matchLabels:
      role: backend
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: frontend
  egress:
  - to:
    - podSelector:
        matchLabels:
          role: db

Explicación

  • podSelector: Selecciona pods con la etiqueta role: backend.
  • ingress: Permite el tráfico entrante desde pods con la etiqueta role: frontend.
  • egress: Permite el tráfico saliente hacia pods con la etiqueta role: db.

Uso de Selectores de Namespace

Las políticas de red también pueden usar selectores de namespace para controlar el tráfico a través de diferentes namespaces.

Ejemplo: Selector de Namespace

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: namespace-selector
  namespace: default
spec:
  podSelector:
    matchLabels:
      role: api
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          environment: production

Explicación

  • namespaceSelector: Selecciona namespaces con la etiqueta environment: production.
  • ingress: Permite el tráfico entrante desde cualquier pod en namespaces etiquetados como environment: production.

Mejores Prácticas para Políticas de Red

  • Menor Privilegio: Comienza con una política predeterminada de denegación total y luego agrega reglas específicas de permiso.
  • Consistencia de Etiquetas: Usa un etiquetado consistente en tus pods y namespaces para simplificar la gestión de políticas.
  • Pruebas: Prueba tus políticas de red en un entorno de staging antes de aplicarlas en producción.

Conclusión

Las Políticas de Red son una herramienta poderosa para asegurar tu clúster de Kubernetes controlando la comunicación entre pods. Al entender e implementar estas políticas, puedes asegurar que tus aplicaciones sean tanto seguras como conformes con las políticas organizacionales. Comienza con políticas básicas y gradualmente avanza hacia configuraciones más complejas a medida que tus necesidades evolucionen.
