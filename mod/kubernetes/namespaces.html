Espacios de nombres en Kubernetes
ConfigMaps y Secrets ►

Los espacios de nombres son un concepto fundamental en Kubernetes, proporcionando un mecanismo para aislar y gestionar recursos dentro de un clúster. Este tema cubrirá los espacios de nombres desde un nivel principiante hasta uno avanzado, asegurando que entiendas su propósito, cómo crearlos y gestionarlos, y casos de uso avanzados.

Introducción a los espacios de nombres

Los espacios de nombres en Kubernetes se utilizan para dividir los recursos del clúster entre múltiples usuarios. Proporcionan un ámbito para los nombres, permitiéndote crear recursos con el mismo nombre en diferentes espacios de nombres.

Conceptos clave

  • Aislamiento: Los espacios de nombres permiten el aislamiento de recursos dentro de un clúster.
  • Gestión de recursos: Ayudan en la gestión de recursos como cuotas de CPU y memoria.
  • Control de acceso: Los espacios de nombres pueden usarse para aplicar diferentes controles de acceso a diferentes partes del clúster.

Escenario de ejemplo

Imagina que tienes un clúster de Kubernetes utilizado por múltiples equipos: Desarrollo, Pruebas y Producción. Usando espacios de nombres, puedes crear entornos separados para cada equipo, asegurando que sus recursos y configuraciones no interfieran entre sí.

Creación y gestión de espacios de nombres

Crear y gestionar espacios de nombres en Kubernetes es sencillo. Puedes usar la herramienta de línea de comandos kubectl para realizar estas acciones.

Creación de un espacio de nombres

Para crear un espacio de nombres, puedes usar el siguiente comando:

kubectl create namespace <nombre-del-namespace>

Por ejemplo, para crear un espacio de nombres para el equipo de desarrollo:

kubectl create namespace desarrollo

Listado de espacios de nombres

Para listar todos los espacios de nombres en tu clúster, usa:

kubectl get namespaces

Eliminación de un espacio de nombres

Para eliminar un espacio de nombres, usa el siguiente comando:

kubectl delete namespace <nombre-del-namespace>

Por ejemplo, para eliminar el espacio de nombres de desarrollo:

kubectl delete namespace desarrollo

Trabajando con espacios de nombres

Una vez creados los espacios de nombres, puedes desplegar recursos en ellos y gestionarlos en consecuencia.

Despliegue de recursos en un espacio de nombres

Para desplegar un recurso en un espacio de nombres específico, puedes especificar el espacio de nombres en el archivo de definición del recurso o usar la bandera --namespace con kubectl.

Ejemplo de definición de recurso

apiVersion: v1
kind: Pod
metadata:
  name: mi-pod
  namespace: desarrollo
spec:
  containers:
  - name: mi-contenedor
    image: nginx

Uso de kubectl con espacio de nombres

kubectl apply -f mi-pod.yaml --namespace=desarrollo

Cambio entre espacios de nombres

Para cambiar entre espacios de nombres en tu contexto de kubectl, puedes usar:

kubectl config set-context --current --namespace=<nombre-del-namespace>

Por ejemplo, para cambiar al espacio de nombres de desarrollo:

kubectl config set-context --current --namespace=desarrollo

Gestión avanzada de espacios de nombres

La gestión avanzada de espacios de nombres implica establecer cuotas de recursos, límites y aplicar políticas de red.

Cuotas de recursos

Las cuotas de recursos aseguran que un espacio de nombres no use más de los recursos asignados.

Ejemplo de cuota de recursos

apiVersion: v1
kind: ResourceQuota
metadata:
  name: cpu-mem-quota
  namespace: desarrollo
spec:
  hard:
    requests.cpu: "1"
    requests.memory: 1Gi
    limits.cpu: "2"
    limits.memory: 2Gi

Aplica la cuota de recursos con:

kubectl apply -f resource-quota.yaml

Políticas de red

Las políticas de red controlan la comunicación entre pods dentro de un espacio de nombres.

Ejemplo de política de red

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-same-namespace
  namespace: desarrollo
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector: {}
  egress:
  - to:
    - podSelector: {}

Aplica la política de red con:

kubectl apply -f network-policy.yaml

Conclusión

Los espacios de nombres son una característica poderosa en Kubernetes que ayudan a organizar y gestionar recursos dentro de un clúster. Proporcionan aislamiento, gestión de recursos y control de acceso, facilitando el manejo de entornos complejos. Al entender cómo crear, gestionar y usar espacios de nombres de manera efectiva, puedes asegurar que tu clúster de Kubernetes esté bien organizado y seguro.

En resumen:

  • Los espacios de nombres proporcionan aislamiento y gestión de recursos.
  • Puedes crear, listar y eliminar espacios de nombres usando kubectl.
  • Despliega recursos en espacios de nombres específicos y cambia de contexto según sea necesario.
  • Usa características avanzadas como cuotas de recursos y políticas de red para gestionar espacios de nombres de manera efectiva.
