Introducción a los Pods

Los pods son los objetos más pequeños y simples de Kubernetes. Representan una única instancia de un proceso en ejecución en tu clúster. Los pods pueden contener uno o más contenedores, que están estrechamente acoplados y comparten el mismo espacio de nombres de red, almacenamiento y ciclo de vida.

Conceptos Clave

  • Pod de Contenedor Único: El tipo más común de pod, que contiene un solo contenedor.
  • Pod de Múltiples Contenedores: Un pod que contiene múltiples contenedores, que trabajan juntos como una sola unidad.
  • Almacenamiento Compartido: Los contenedores dentro de un pod pueden compartir volúmenes de almacenamiento.
  • Red Compartida: Los contenedores dentro de un pod comparten la misma dirección IP y espacio de puertos.
  • Ciclo de Vida: Los pods son efímeros y pueden ser creados, destruidos y recreados.

Creación de un Pod

Para crear un pod, lo defines en un archivo YAML y luego usas el comando kubectl para aplicarlo a tu clúster.

Ejemplo: Pod de Contenedor Único

Aquí hay un ejemplo de un archivo YAML para un pod de contenedor único:

apiVersion: v1
kind: Pod
metadata:
  name: my-single-container-pod
spec:
  containers:
  - name: my-container
    image: nginx:latest
    ports:
    - containerPort: 80

Explicación:

  • apiVersion: Especifica la versión de la API de Kubernetes.
  • kind: Especifica el tipo de objeto de Kubernetes (Pod en este caso).
  • metadata: Contiene datos que ayudan a identificar de manera única el objeto, incluyendo un nombre.
  • spec: Define el estado deseado del pod.
    • containers: Lista los contenedores dentro del pod.
      • name: El nombre del contenedor.
      • image: La imagen del contenedor a ejecutar.
      • ports: Los puertos que el contenedor expone.

Para crear este pod, guarda el YAML en un archivo llamado pod.yaml y ejecuta:

kubectl apply -f pod.yaml

Ejemplo: Pod de Múltiples Contenedores

Aquí hay un ejemplo de un archivo YAML para un pod de múltiples contenedores:

apiVersion: v1
kind: Pod
metadata:
  name: my-multi-container-pod
spec:
  containers:
  - name: app-container
    image: nginx:latest
    ports:
    - containerPort: 80
  - name: sidecar-container
    image: busybox
    command: ['sh', '-c', 'echo Hello Kubernetes! && sleep 3600']

Explicación:

  • Este pod contiene dos contenedores: app-container y sidecar-container.
  • app-container ejecuta un servidor Nginx.
  • sidecar-container ejecuta un contenedor BusyBox que imprime un mensaje y luego duerme.

Gestión de Pods

Visualización de Pods

Para listar todos los pods en el espacio de nombres predeterminado, usa:

kubectl get pods

Descripción de Pods

Para obtener información detallada sobre un pod específico, usa:

kubectl describe pod <nombre-del-pod>

Eliminación de Pods

Para eliminar un pod, usa:

kubectl delete pod <nombre-del-pod>

Conceptos Avanzados de Pods

Ciclo de Vida del Pod

  • Pending: El pod ha sido aceptado por el sistema de Kubernetes pero una o más de las imágenes de los contenedores no han sido creadas.
  • Running: El pod ha sido vinculado a un nodo, y todos los contenedores han sido creados.
  • Succeeded: Todos los contenedores en el pod han terminado exitosamente.
  • Failed: Todos los contenedores en el pod han terminado, y al menos un contenedor ha terminado en fallo.
  • Unknown: No se pudo obtener el estado del pod.

Contenedores de Inicialización

Los contenedores de inicialización se ejecutan antes que los contenedores de la aplicación y pueden contener scripts de configuración o realizar tareas de inicialización.

Ejemplo:

apiVersion: v1
kind: Pod
metadata:
  name: my-pod-with-init
spec:
  initContainers:
  - name: init-myservice
    image: busybox
    command: ['sh', '-c', 'echo Initializing... && sleep 5']
  containers:
  - name: my-container
    image: nginx:latest
    ports:
    - containerPort: 80

Contexto de Seguridad del Pod

El contexto de seguridad define configuraciones de privilegios y control de acceso para un pod o contenedor.

Ejemplo:

apiVersion: v1
kind: Pod
metadata:
  name: my-secure-pod
spec:
  securityContext:
    runAsUser: 1000
    fsGroup: 2000
  containers:
  - name: my-container
    image: nginx:latest
    ports:
    - containerPort: 80

Resumen

Los pods son los bloques de construcción fundamentales de las aplicaciones en Kubernetes. Pueden contener uno o más contenedores que comparten almacenamiento, red y ciclo de vida. Entender cómo crear, gestionar y configurar pods es esencial para desplegar aplicaciones en un clúster de Kubernetes. A medida que avances, aprenderás sobre configuraciones de pods más complejas y gestión del ciclo de vida, que son cruciales para construir aplicaciones robustas y seguras.
