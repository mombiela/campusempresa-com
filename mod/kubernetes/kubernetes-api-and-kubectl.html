Introducción

En esta sección, exploraremos la API de Kubernetes y kubectl, la herramienta de línea de comandos para interactuar con los clústeres de Kubernetes. Comprender estas herramientas es esencial para gestionar y automatizar tareas dentro de un entorno de Kubernetes.

API de Kubernetes

La API de Kubernetes es la interfaz a través de la cual los usuarios, componentes externos y componentes internos se comunican entre sí. Es una API RESTful que te permite realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en recursos de Kubernetes.

Conceptos Clave

  • Servidor API: El componente central del plano de control de Kubernetes que expone la API de Kubernetes.
  • Recursos: Objetos en la API de Kubernetes, como Pods, Servicios, Despliegues, etc.
  • Endpoints: URLs específicas que representan diferentes operaciones en recursos.

Ejemplo: Accediendo a la API de Kubernetes

Puedes interactuar con la API de Kubernetes directamente usando solicitudes HTTP. Aquí tienes un ejemplo de cómo listar todos los Pods en un espacio de nombres usando curl:

curl -X GET https://<kubernetes-api-server>/api/v1/namespaces/default/pods \
  -H "Authorization: Bearer <your-access-token>" \
  -H "Accept: application/json"

kubectl

kubectl es la herramienta de línea de comandos para interactuar con la API de Kubernetes. Simplifica el proceso de gestión de recursos de Kubernetes proporcionando un conjunto de comandos que abstraen las llamadas a la API subyacentes.

Conceptos Clave

  • Comandos: Los comandos de kubectl siguen el patrón kubectl <comando> <recurso> <nombre>.
  • Configuración: kubectl usa un archivo de configuración (usualmente ubicado en ~/.kube/config) para acceder al clúster de Kubernetes.
  • Contextos: Los contextos en el archivo de configuración te permiten cambiar entre diferentes clústeres y espacios de nombres.

Comandos Básicos

Aquí tienes algunos comandos básicos de kubectl para empezar:

  • Obtener Pods: Lista todos los Pods en el espacio de nombres por defecto.

    kubectl get pods

  • Describir Pod: Obtén información detallada sobre un Pod específico.

    kubectl describe pod <pod-name>

  • Crear Recurso: Crea un recurso a partir de un archivo YAML.

    kubectl apply -f <file.yaml>

  • Eliminar Recurso: Elimina un recurso específico.

    kubectl delete <resource-type> <resource-name>

Comandos Avanzados

Para operaciones más avanzadas, kubectl proporciona comandos adicionales:

  • Escalar Despliegue: Escala un despliegue a un número especificado de réplicas.

    kubectl scale deployment <deployment-name> --replicas=<number>

  • Actualización Continua: Realiza una actualización continua en un despliegue.

    kubectl rollout restart deployment <deployment-name>

  • Reenvío de Puertos: Reenvía un puerto local a un puerto en un Pod.

    kubectl port-forward <pod-name> <local-port>:<pod-port>

Gestión de Configuración

Gestionar múltiples clústeres y espacios de nombres se puede hacer usando contextos en el archivo de configuración de kubectl.

  • Ver Contexto Actual:

    kubectl config current-context

  • Establecer Contexto:

    kubectl config use-context <context-name>

  • Listar Contextos:

    kubectl config get-contexts

Conclusión

Comprender la API de Kubernetes y kubectl es fundamental para gestionar eficazmente los clústeres de Kubernetes. La API proporciona una interfaz poderosa para la automatización e integración, mientras que kubectl ofrece una forma fácil de usar para interactuar con el clúster. Dominar estas herramientas mejorará significativamente tu capacidad para desplegar, gestionar y solucionar problemas de aplicaciones en Kubernetes.
