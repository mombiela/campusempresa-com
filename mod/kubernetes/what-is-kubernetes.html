Introducción

Kubernetes, a menudo abreviado como K8s, es una plataforma de código abierto diseñada para automatizar el despliegue, escalado y operación de contenedores de aplicaciones. Originalmente desarrollado por Google, Kubernetes ahora es mantenido por la Cloud Native Computing Foundation (CNCF).

Conceptos Clave

  • Orquestación de Contenedores: Kubernetes gestiona el ciclo de vida de las aplicaciones en contenedores a través de un clúster de nodos.
  • Clúster: Un conjunto de nodos (máquinas) que ejecutan aplicaciones en contenedores gestionadas por Kubernetes.
  • Nodo: Una única máquina en un clúster de Kubernetes, que puede ser una máquina física o virtual.
  • Pod: La unidad de despliegue más pequeña en Kubernetes, que puede contener uno o más contenedores.
  • Servicio: Una abstracción que define un conjunto lógico de Pods y una política para acceder a ellos.
  • Namespace: Una forma de dividir los recursos del clúster entre múltiples usuarios.

¿Por qué Kubernetes?

  • Escalabilidad: Escalar automáticamente las aplicaciones hacia arriba y hacia abajo según la demanda.
  • Alta Disponibilidad: Asegurar que las aplicaciones estén siempre en funcionamiento y accesibles.
  • Eficiencia de Recursos: Optimizar el uso de los recursos de hardware subyacentes.
  • Portabilidad: Ejecutar aplicaciones de manera consistente en diferentes entornos (local, nube, híbrido).

Arquitectura Básica

La arquitectura de Kubernetes consta de los siguientes componentes:

| Componente | Descripción | |------------------|-----------------------------------------------------------------------------| | Nodo Maestro | Gestiona el clúster de Kubernetes. | | Nodos Trabajadores | Ejecutan las aplicaciones en contenedores. | | etcd | Un almacén de clave-valor consistente y altamente disponible utilizado para datos de configuración.| | kube-apiserver | Expone la API de Kubernetes. | | kube-scheduler | Asigna cargas de trabajo a los nodos según la disponibilidad de recursos. | | kube-controller-manager | Gestiona varios controladores que regulan el estado del clúster. | | kubelet | Un agente que se ejecuta en cada nodo trabajador y asegura que los contenedores estén en funcionamiento.| | kube-proxy | Mantiene reglas de red en los nodos. |

Ejemplo: Desplegando una Aplicación Simple

Paso 1: Crear un Deployment

Un Deployment asegura que un número especificado de réplicas de pods estén en funcionamiento en cualquier momento.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

Paso 2: Aplicar el Deployment

Usa la herramienta de línea de comandos kubectl para aplicar la configuración del deployment.

kubectl apply -f nginx-deployment.yaml

Paso 3: Verificar el Deployment

Verifica el estado del deployment para asegurar que esté funcionando correctamente.

kubectl get deployments
kubectl get pods

Paso 4: Exponer el Deployment

Crea un Servicio para exponer el deployment al tráfico externo.

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer

Aplica la configuración del servicio:

kubectl apply -f nginx-service.yaml

Conclusión

Kubernetes es una herramienta poderosa para gestionar aplicaciones en contenedores a escala. Proporciona un marco robusto para desplegar, escalar y mantener aplicaciones, asegurando alta disponibilidad y utilización eficiente de recursos. Al comprender los conceptos básicos y la arquitectura de Kubernetes, puedes comenzar a aprovechar sus capacidades para agilizar tus procesos de despliegue de aplicaciones.
