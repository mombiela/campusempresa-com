Kustomize
Kubeadm ►

Kustomize es una herramienta para personalizar configuraciones de Kubernetes. Te permite gestionar y personalizar manifiestos de Kubernetes sin la necesidad de usar plantillas. Este tema te guiará desde los conceptos básicos de Kustomize hasta su uso avanzado, proporcionando explicaciones claras, ejemplos y ejercicios.

Introducción a Kustomize

Kustomize está integrado en kubectl y proporciona una forma de gestionar recursos de Kubernetes de manera declarativa. Te permite definir un conjunto base de recursos y luego crear superposiciones para personalizar estos recursos para diferentes entornos.

Conceptos Clave

  • Base: Un conjunto de manifiestos de Kubernetes que definen los recursos comunes.
  • Overlay: Personalizaciones aplicadas sobre los manifiestos base para diferentes entornos (por ejemplo, desarrollo, staging, producción).
  • kustomization.yaml: El archivo de configuración utilizado por Kustomize para definir cómo deben personalizarse los recursos.

Empezando con Kustomize

Instalando Kustomize

Kustomize está integrado en kubectl, pero también puedes instalarlo como una herramienta independiente.

Usando kubectl

Kustomize está incluido en kubectl a partir de la versión 1.14. Puedes usarlo directamente con el comando kubectl kustomize.

kubectl kustomize <path-to-kustomization-directory>

Instalación Independiente

Para instalar Kustomize como una herramienta independiente, sigue las instrucciones en la página de GitHub de Kustomize.

Creando una Base

Una base es un conjunto de manifiestos de Kubernetes que definen los recursos comunes. Vamos a crear una base simple para un Deployment y un Service.

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:1.0.0
        ports:
        - containerPort: 80
# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

Creando un kustomization.yaml

Crea un archivo kustomization.yaml para definir la base.

# kustomization.yaml
resources:
- deployment.yaml
- service.yaml

Aplicando la Base

Para aplicar la base, usa el siguiente comando:

kubectl apply -k <path-to-base-directory>

Creando Overlays

Las superposiciones te permiten personalizar los recursos base para diferentes entornos.

Ejemplo: Superposición de Desarrollo

Crea un directorio para la superposición de desarrollo y un archivo kustomization.yaml dentro de él.

# overlays/development/kustomization.yaml
resources:
- ../../base

patchesStrategicMerge:
- deployment-patch.yaml

Crea un archivo de parche para personalizar el Deployment para el entorno de desarrollo.

# overlays/development/deployment-patch.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 1
  template:
    spec:
      containers:
      - name: my-app
        image: my-app:1.0.1

Aplicando la Superposición

Para aplicar la superposición de desarrollo, usa el siguiente comando:

kubectl apply -k overlays/development

Funciones Avanzadas de Kustomize

Usando ConfigMaps y Secrets

Kustomize te permite generar ConfigMaps y Secrets a partir de archivos o literales.

# kustomization.yaml
configMapGenerator:
- name: my-config
  files:
  - config.properties

secretGenerator:
- name: my-secret
  literals:
  - username=admin
  - password=secret

Etiquetas y Anotaciones Comunes

Puedes agregar etiquetas y anotaciones comunes a todos los recursos.

# kustomization.yaml
commonLabels:
  app: my-app
commonAnnotations:
  environment: development

Transformaciones

Kustomize soporta varias transformaciones, como namePrefix, nameSuffix y transformaciones de imágenes.

# kustomization.yaml
namePrefix: dev-
nameSuffix: -v1

images:
- name: my-app
  newName: my-app
  newTag: 1.0.1

Conclusión

Kustomize es una herramienta poderosa para gestionar y personalizar configuraciones de Kubernetes. Te permite definir un conjunto base de recursos y crear superposiciones para diferentes entornos, facilitando la gestión de despliegues complejos. Al entender y utilizar Kustomize, puedes optimizar tus flujos de trabajo de Kubernetes y mejorar la mantenibilidad de tus configuraciones.

Resumen

  • Base: Conjunto común de manifiestos de Kubernetes.
  • Overlay: Personalizaciones para diferentes entornos.
  • kustomization.yaml: Archivo de configuración para Kustomize.
  • Características Clave: Generación de ConfigMap y Secret, etiquetas y anotaciones comunes, transformaciones.

Practica usando Kustomize creando tus propias bases y superposiciones, y explora sus características avanzadas para aprovechar al máximo sus capacidades en tus despliegues de Kubernetes.
