Exámenes de Práctica y Preguntas de Muestra

Introducción

Los exámenes de práctica y las preguntas de muestra son herramientas esenciales para dominar Ansible. Ayudan a reforzar el aprendizaje, identificar áreas de mejora y aumentar la confianza. Esta sección proporciona una variedad de preguntas de práctica y exámenes simulados, que van desde niveles principiantes hasta avanzados, para ayudarte a solidificar tu comprensión de Ansible.

Nivel Principiante

Conceptos Clave

  • Comandos básicos de Ansible
  • Archivos de inventario
  • Playbooks y sintaxis YAML
  • Módulos y tareas

Preguntas de Muestra

  1. ¿Qué comando se usa para verificar la versión de Ansible instalada en tu sistema?

    ansible --version

  2. Escribe un archivo de inventario simple para dos hosts: webserver y dbserver.

    [web]
webserver

[db]
dbserver

  3. Crea un playbook básico para instalar nginx en webserver.

    ---
- name: Instalar nginx en webserver
  hosts: web
  tasks:
    - name: Instalar nginx
      apt:
        name: nginx
        state: present

  4. Explica el propósito del archivo ansible.cfg.

    • El archivo ansible.cfg se usa para configurar ajustes de Ansible como la ubicación del inventario, el usuario remoto y la configuración de SSH.

Nivel Intermedio

Conceptos Clave

  • Roles y estructura de roles
  • Variables y hechos
  • Manejadores y notificaciones
  • Condicionales y bucles

Preguntas de Muestra

  1. ¿Cuál es el propósito de los roles en Ansible?

    • Los roles ayudan a organizar los playbooks agrupando tareas, variables, archivos, plantillas y manejadores en componentes reutilizables.

  2. Escribe un playbook que use una variable para instalar un paquete especificado por el usuario.

    ---
- name: Instalar un paquete
  hosts: all
  vars:
    package_name: "default_package"
  tasks:
    - name: Instalar el paquete especificado
      apt:
        name: "{{ package_name }}"
        state: present

  3. Crea un manejador para reiniciar nginx y notifícalo en una tarea.

    ---
- name: Instalar y configurar nginx
  hosts: web
  tasks:
    - name: Instalar nginx
      apt:
        name: nginx
        state: present
      notify:
        - Reiniciar nginx

  handlers:
    - name: Reiniciar nginx
      service:
        name: nginx
        state: restarted

  4. Explica cómo usar condicionales en tareas de Ansible. Proporciona un ejemplo.

    • Los condicionales en Ansible se usan para ejecutar tareas basadas en ciertas condiciones. Se definen usando la palabra clave when.
    - name: Instalar nginx solo en Ubuntu
  apt:
    name: nginx
    state: present
  when: ansible_os_family == "Debian"

Nivel Avanzado

Conceptos Clave

  • Módulos personalizados
  • Ansible Vault
  • Inventario dinámico
  • Estrategias avanzadas de playbooks

Preguntas de Muestra

  1. Describe cómo crear un módulo personalizado de Ansible.

    • Los módulos personalizados pueden ser creados usando Python o cualquier otro lenguaje de scripting. Deben devolver un objeto JSON con claves específicas como changed y msg.

  2. Escribe un playbook que use Ansible Vault para cifrar datos sensibles.

    ---
- name: Desencriptar y usar datos sensibles
  hosts: all
  vars_files:
    - vault.yml
  tasks:
    - name: Usar datos sensibles
      debug:
        msg: "El secreto es {{ secret_value }}"
    • Para cifrar el archivo vault.yml: 
      ansible-vault encrypt vault.yml

  3. Explica el propósito del inventario dinámico y proporciona un script de ejemplo.

    • El inventario dinámico permite que Ansible consulte fuentes de datos externas para generar inventarios dinámicamente.
    # dynamic_inventory.py
import json

inventory = {
    "web": {
        "hosts": ["webserver1", "webserver2"]
    },
    "db": {
        "hosts": ["dbserver1"]
    }
}

print(json.dumps(inventory))
    • Para usar el script: 
      ansible-playbook -i dynamic_inventory.py playbook.yml

  4. Discute estrategias avanzadas de playbooks como serial, max_fail_percentage y delegate_to.

    • serial: Controla el número de hosts que se gestionarán a la vez.
    • max_fail_percentage: Define el porcentaje máximo de hosts que pueden fallar antes de que se aborte el play.
    • delegate_to: Ejecuta una tarea en un host diferente al especificado en la directiva hosts.

Conclusión

Los exámenes de práctica y las preguntas de muestra son invaluables para dominar Ansible. Al trabajar con estas preguntas, puedes identificar tus fortalezas y áreas de mejora, asegurándote de estar bien preparado para escenarios del mundo real y exámenes de certificación. Sigue practicando y explorando las vastas capacidades de Ansible para volverte competente en automatización y gestión de configuración.
