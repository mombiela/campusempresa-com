Plugins de Ansible
Manejo de Errores y Depuración ►

Los plugins de Ansible son una característica poderosa que extiende la funcionalidad de Ansible. Permiten personalizar y mejorar el comportamiento de Ansible de varias maneras. Esta sección cubrirá los diferentes tipos de plugins de Ansible, cómo usarlos y cómo crear tus propios plugins.

Tipos de Plugins de Ansible

Ansible soporta varios tipos de plugins, cada uno con un propósito diferente:

  • Plugins de Acción: Modifican el comportamiento de los módulos de Ansible.
  • Plugins de Búsqueda: Recuperan datos de fuentes externas.
  • Plugins de Filtro: Transforman datos dentro de los playbooks de Ansible.
  • Plugins de Callback: Personalizan la salida de las ejecuciones de Ansible.
  • Plugins de Conexión: Gestionan la conexión a hosts remotos.
  • Plugins de Inventario: Generan dinámicamente inventarios a partir de fuentes externas.
  • Plugins de Variables: Cargan variables desde fuentes externas.
  • Plugins de Prueba: Proveen pruebas adicionales para Jinja2.

Usando Plugins de Ansible

Los plugins de Ansible pueden ser usados directamente en tus playbooks o roles. Aquí hay algunos ejemplos de cómo usar diferentes tipos de plugins:

Plugins de Acción

Los plugins de acción te permiten modificar el comportamiento de los módulos. Por ejemplo, puedes usar un plugin de acción para añadir registro adicional o para cambiar cómo un módulo maneja errores.

---
- name: Ejemplo de playbook usando un plugin de acción
  hosts: localhost
  tasks:
    - name: Usar el plugin de acción personalizado
      my_custom_action:
        param1: value1
        param2: value2

Plugins de Búsqueda

Los plugins de búsqueda se usan para recuperar datos de fuentes externas. Por ejemplo, puedes usar un plugin de búsqueda para leer datos de un archivo o una base de datos.

---
- name: Ejemplo de playbook usando un plugin de búsqueda
  hosts: localhost
  tasks:
    - name: Recuperar datos usando un plugin de búsqueda
      debug:
        msg: "{{ lookup('file', '/path/to/file.txt') }}"

Plugins de Filtro

Los plugins de filtro se usan para transformar datos dentro de los playbooks de Ansible. Por ejemplo, puedes usar un plugin de filtro para formatear una fecha o para manipular cadenas.

---
- name: Ejemplo de playbook usando un plugin de filtro
  hosts: localhost
  tasks:
    - name: Transformar datos usando un plugin de filtro
      debug:
        msg: "{{ '2023-10-01' | to_datetime | date('long') }}"

Plugins de Callback

Los plugins de callback te permiten personalizar la salida de las ejecuciones de Ansible. Por ejemplo, puedes usar un plugin de callback para enviar notificaciones o para registrar resultados en un archivo.

---
- name: Ejemplo de playbook usando un plugin de callback
  hosts: localhost
  tasks:
    - name: Ejecutar una tarea con un callback personalizado
      command: echo "Hello, World!"

Plugins de Conexión

Los plugins de conexión gestionan la conexión a hosts remotos. Por ejemplo, puedes usar un plugin de conexión para conectarte a un host usando SSH o para gestionar conexiones a servicios en la nube.

---
- name: Ejemplo de playbook usando un plugin de conexión
  hosts: remote_host
  connection: local
  tasks:
    - name: Ejecutar una tarea en el host remoto
      command: echo "Running on remote host"

Plugins de Inventario

Los plugins de inventario generan dinámicamente inventarios a partir de fuentes externas. Por ejemplo, puedes usar un plugin de inventario para generar inventarios desde la API de un proveedor de nube.

---
plugin: aws_ec2
regions:
  - us-east-1
filters:
  instance-state-name: running

Plugins de Variables

Los plugins de variables cargan variables desde fuentes externas. Por ejemplo, puedes usar un plugin de variables para cargar variables desde un archivo o una base de datos.

---
- name: Ejemplo de playbook usando un plugin de variables
  hosts: localhost
  vars_files:
    - /path/to/vars.yml
  tasks:
    - name: Usar variables cargadas
      debug:
        msg: "{{ my_variable }}"

Plugins de Prueba

Los plugins de prueba proveen pruebas adicionales para Jinja2. Por ejemplo, puedes usar un plugin de prueba para verificar si una variable es una dirección IP válida.

---
- name: Ejemplo de playbook usando un plugin de prueba
  hosts: localhost
  tasks:
    - name: Verificar si una variable es una dirección IP válida
      debug:
        msg: "La variable es una dirección IP válida"
      when: my_variable is ipaddr

Creando Plugins Personalizados de Ansible

Crear plugins personalizados te permite extender la funcionalidad de Ansible para satisfacer tus necesidades específicas. Aquí hay un ejemplo básico de cómo crear un plugin de filtro personalizado:

  1. Crea una estructura de directorios para tu plugin:
my_plugins/
└── filter_plugins/
    └── my_custom_filter.py
  1. Implementa el plugin de filtro personalizado en my_custom_filter.py:
def to_uppercase(value):
    return value.upper()

class FilterModule(object):
    def filters(self):
        return {
            'to_uppercase': to_uppercase
        }
  1. Usa el plugin de filtro personalizado en tu playbook:
---
- name: Ejemplo de playbook usando un plugin de filtro personalizado
  hosts: localhost
  tasks:
    - name: Transformar datos usando el plugin de filtro personalizado
      debug:
        msg: "{{ 'hello, world!' | to_uppercase }}"

Conclusión

Los plugins de Ansible son una característica versátil y poderosa que te permite extender y personalizar el comportamiento de Ansible. Al entender los diferentes tipos de plugins y cómo usarlos, puedes mejorar tus flujos de trabajo de automatización y adaptar Ansible a tus necesidades específicas. Ya sea que estés usando plugins integrados o creando los tuyos propios, los plugins proveen una manera flexible de integrar Ansible con sistemas externos y de realizar transformaciones de datos complejas.
