Introducción a las Pruebas Unitarias

Las pruebas unitarias son un método de prueba de software donde se prueban unidades o componentes individuales de un software. El propósito es validar que cada unidad del software funcione como se espera.

  • Unidad: La parte más pequeña testeable de cualquier software.
  • Prueba Unitaria: Un fragmento de código escrito por un desarrollador que ejecuta una pequeña parte de la aplicación y verifica un comportamiento o estado específico.

¿Por qué Pruebas Unitarias?

  • Asegura la confiabilidad del código.
  • Facilita el mantenimiento del código.
  • Ayuda a detectar errores temprano.
  • Proporciona documentación del código.

Resumen de Jasmine y Karma

Jasmine

Jasmine es un marco de desarrollo dirigido por comportamiento para probar código JavaScript. No depende de otros marcos de JavaScript y no requiere un DOM.

  • Características:
    • Sintaxis limpia y fácil.
    • Soporta pruebas asíncronas.
    • Matchers integrados para necesidades comunes de prueba.

Karma

Karma es un ejecutor de pruebas desarrollado por el equipo de AngularJS. Permite ejecutar código JavaScript en múltiples navegadores reales.

  • Características:
    • Ejecuta pruebas en navegadores reales.
    • Proporciona soporte para integración continua.
    • Muestra resultados de pruebas en la línea de comandos.

Configuración de Jasmine y Karma en un Proyecto Ionic

Paso 1: Instalar Dependencias

Primero, necesitas instalar Jasmine y Karma junto con sus plugins necesarios.

npm install --save-dev jasmine-core karma karma-jasmine karma-chrome-launcher

Paso 2: Configurar Karma

Crea un archivo karma.conf.js en la raíz de tu proyecto.

module.exports = function(config) {
  config.set({
    basePath: '',
    frameworks: ['jasmine'],
    files: [
      'src/**/*.spec.ts'
    ],
    preprocessors: {
      'src/**/*.spec.ts': ['webpack']
    },
    webpack: {
      // configuración de webpack
    },
    reporters: ['progress'],
    port: 9876,
    colors: true,
    logLevel: config.LOG_INFO,
    autoWatch: true,
    browsers: ['Chrome'],
    singleRun: false,
    concurrency: Infinity
  });
};

Paso 3: Crear una Prueba de Ejemplo

Crea un archivo de prueba de ejemplo example.spec.ts en el directorio src.

describe('Prueba de Ejemplo', () => {
  it('debería devolver verdadero', () => {
    expect(true).toBe(true);
  });
});

Paso 4: Ejecutar las Pruebas

Ejecuta las pruebas usando el siguiente comando:

npx karma start karma.conf.js

Escribir Pruebas con Jasmine

Estructura Básica de una Prueba

Una prueba básica en Jasmine se estructura usando bloques describe e it.

describe('MiComponente', () => {
  it('debería hacer algo', () => {
    // Preparar
    let value = true;
    
    // Actuar
    let result = value;
    
    // Afirmar
    expect(result).toBe(true);
  });
});

Matchers

Jasmine proporciona una variedad de matchers para validar tus pruebas.

  • toBe: Verifica igualdad estricta.
  • toEqual: Verifica igualdad profunda.
  • toBeTruthy: Verifica si el valor es verdadero.
  • toBeFalsy: Verifica si el valor es falso.
  • toContain: Verifica si un array o string contiene un valor.

Pruebas Asíncronas

Jasmine soporta pruebas asíncronas usando el callback done o async/await.

it('debería obtener datos de forma asíncrona', (done) => {
  fetchData().then(data => {
    expect(data).toBeDefined();
    done();
  });
});

Técnicas Avanzadas de Prueba

Espías

Los espías se usan para rastrear llamadas a funciones o métodos.

it('debería llamar al método', () => {
  let myObj = {
    myMethod: () => {}
  };
  spyOn(myObj, 'myMethod');
  
  myObj.myMethod();
  
  expect(myObj.myMethod).toHaveBeenCalled();
});

Simulación de Dependencias

La simulación se usa para imitar el comportamiento de dependencias complejas.

class MockService {
  getData() {
    return of(['data1', 'data2']);
  }
}

describe('MiComponente', () => {
  let component: MiComponente;
  let service: MockService;

  beforeEach(() => {
    service = new MockService();
    component = new MiComponente(service);
  });

  it('debería usar el servicio simulado', () => {
    component.ngOnInit();
    expect(component.data.length).toBe(2);
  });
});

Conclusión

Las pruebas unitarias son una parte crucial del proceso de desarrollo, asegurando que los componentes individuales funcionen como se espera. Jasmine y Karma proporcionan un marco robusto para escribir y ejecutar pruebas en un proyecto Ionic. Siguiendo los pasos y ejemplos proporcionados, puedes configurar y escribir pruebas unitarias efectivas, llevando a un código más confiable y mantenible.
