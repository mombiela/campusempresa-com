Introducción a las Pruebas de Extremo a Extremo

Las pruebas de extremo a extremo (E2E) son una metodología utilizada para probar si el flujo de una aplicación se desempeña según lo diseñado de principio a fin. El propósito de las pruebas E2E es identificar las dependencias del sistema y asegurar que la integridad de los datos se mantenga entre varios componentes y sistemas del sistema.

  • Conceptos Clave:
    • Flujo de Usuario: Probar el recorrido completo del usuario.
    • Dependencias del Sistema: Asegurar que todas las partes del sistema funcionen juntas.
    • Integridad de los Datos: Verificar la consistencia de los datos en toda la aplicación.

¿Qué es Protractor?

Protractor es un marco de pruebas de extremo a extremo para aplicaciones Angular y AngularJS. Está construido sobre WebDriverJS, que utiliza eventos nativos y controladores específicos del navegador para interactuar con tu aplicación como lo haría un usuario.

  • Características Clave:
    • Conciencia de Angular: Protractor entiende los elementos específicos de Angular.
    • Espera Automática: Protractor espera automáticamente a que la aplicación esté estable.
    • Integración Perfecta: Funciona bien con Jasmine, Mocha y otros marcos de pruebas.

Configuración de Protractor en un Proyecto Ionic

Para comenzar con Protractor en un proyecto Ionic, necesitas instalar Protractor y configurarlo.

Paso 1: Instalar Protractor

Primero, instala Protractor globalmente usando npm:

npm install -g protractor

Paso 2: Actualizar WebDriver

Actualiza el administrador de WebDriver para asegurarte de tener la última versión:

webdriver-manager update

Paso 3: Configurar Protractor

Crea un archivo protractor.conf.js en la raíz de tu proyecto:

exports.config = {
  framework: 'jasmine',
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['e2e/**/*.spec.js'],
  capabilities: {
    browserName: 'chrome'
  },
  onPrepare: function() {
    browser.ignoreSynchronization = true;
  }
};

Paso 4: Iniciar WebDriver

Inicia el servidor de WebDriver:

webdriver-manager start

Escribiendo Tu Primera Prueba con Protractor

Vamos a escribir una prueba simple para verificar si la página de inicio de tu aplicación Ionic se carga correctamente.

Ejemplo de Prueba

Crea un archivo e2e/homepage.spec.js:

describe('Ionic App Homepage', function() {
  it('should have a title', function() {
    browser.get('http://localhost:8100');
    expect(browser.getTitle()).toEqual('My Ionic App');
  });
});

Explicación

  • describe: Un conjunto de pruebas.
  • it: Una prueba individual.
  • browser.get: Navega a la URL especificada.
  • expect: Una afirmación para verificar si el título es el esperado.

Ejecutando Tus Pruebas con Protractor

Para ejecutar tus pruebas, usa el siguiente comando:

protractor protractor.conf.js

Características Avanzadas de Protractor

A medida que te sientas más cómodo con Protractor, puedes aprovechar sus características avanzadas para escribir pruebas más completas.

Manejo de Operaciones Asíncronas

Protractor espera automáticamente a que Angular termine de renderizar y realizar llamadas HTTP. Sin embargo, para aplicaciones no Angular o escenarios específicos, es posible que necesites manejar operaciones asíncronas manualmente.

Ejemplo con Operaciones Asíncronas

it('should wait for an element to be visible', function() {
  var EC = protractor.ExpectedConditions;
  var button = element(by.id('myButton'));
  browser.wait(EC.visibilityOf(button), 5000);
  button.click();
});

Objetos de Página

Los Objetos de Página son un patrón de diseño que te ayuda a escribir pruebas más limpias encapsulando la estructura y las operaciones de la página.

Ejemplo de Objeto de Página

Crea un archivo e2e/pages/homepage.js:

var HomePage = function() {
  this.title = element(by.css('h1'));
  this.get = function() {
    browser.get('http://localhost:8100');
  };
};

module.exports = new HomePage();

Usa el Objeto de Página en tu prueba:

var HomePage = require('./pages/homepage');

describe('Ionic App Homepage', function() {
  it('should have a title', function() {
    HomePage.get();
    expect(HomePage.title.getText()).toEqual('Welcome to My Ionic App');
  });
});

Conclusión

Las pruebas de extremo a extremo con Protractor en un proyecto Ionic aseguran que tu aplicación funcione como se espera desde la perspectiva del usuario. Al configurar Protractor, escribir pruebas y aprovechar características avanzadas como operaciones asíncronas y Objetos de Página, puedes crear suites de pruebas robustas y mantenibles. Esto no solo mejora la calidad de tu aplicación, sino que también mejora el flujo de trabajo de desarrollo al detectar problemas temprano en el proceso.
