¿Qué son los Patrones de Comportamiento?

Los patrones de comportamiento son un tipo de patrones de diseño que se centran en cómo los objetos interactúan y se comunican entre sí. Estos patrones ayudan a definir cómo se distribuyen las responsabilidades entre los objetos y cómo se coordinan sus acciones para lograr una funcionalidad específica.

Características Clave:

  • Interacción entre Objetos: Se enfocan en cómo los objetos colaboran y se comunican.
  • Distribución de Responsabilidades: Ayudan a definir roles claros y responsabilidades entre los objetos.
  • Flexibilidad y Reusabilidad: Promueven soluciones flexibles y reutilizables para problemas comunes de interacción.

Clasificación de los Patrones de Comportamiento

Los patrones de comportamiento se pueden clasificar en varias categorías según su propósito y la forma en que gestionan las interacciones entre objetos. A continuación, se presenta una lista de los patrones de comportamiento más comunes que se estudiarán en este módulo:

  1. Chain of Responsibility: Permite que varios objetos tengan la oportunidad de manejar una solicitud, pasando la solicitud a lo largo de una cadena de manejadores.
  2. Command: Encapsula una solicitud como un objeto, lo que permite parametrizar a los clientes con diferentes solicitudes, encolar o registrar solicitudes, y soportar operaciones que se pueden deshacer.
  3. Interpreter: Define una representación para su gramática y un intérprete que usa la representación para interpretar oraciones en el lenguaje.
  4. Iterator: Proporciona una forma de acceder secuencialmente a los elementos de un objeto agregado sin exponer su representación subyacente.
  5. Mediator: Define un objeto que encapsula cómo interactúan un conjunto de objetos. Promueve el acoplamiento débil al evitar que los objetos se refieran entre sí explícitamente.
  6. Memento: Sin violar la encapsulación, captura y externaliza el estado interno de un objeto para que el objeto pueda ser restaurado a este estado más tarde.
  7. Observer: Define una dependencia de uno a muchos entre objetos para que cuando un objeto cambie de estado, todos sus dependientes sean notificados y actualizados automáticamente.
  8. State: Permite que un objeto altere su comportamiento cuando su estado interno cambia. El objeto parecerá cambiar su clase.
  9. Strategy: Define una familia de algoritmos, encapsula cada uno y los hace intercambiables. Permite que el algoritmo varíe independientemente de los clientes que lo usan.
  10. Template Method: Define el esqueleto de un algoritmo en una operación, diferiendo algunos pasos a las subclases. Permite que las subclases redefinan ciertos pasos de un algoritmo sin cambiar la estructura del algoritmo.
  11. Visitor: Representa una operación que se realiza sobre los elementos de una estructura de objetos. Permite definir una nueva operación sin cambiar las clases de los elementos sobre los que opera.

Ejemplo de Uso de un Patrón de Comportamiento

Para ilustrar cómo funcionan los patrones de comportamiento, veamos un ejemplo sencillo del patrón Observer.

Ejemplo: Patrón Observer

El patrón Observer define una relación de dependencia uno-a-muchos entre objetos, de manera que cuando un objeto cambia de estado, todos sus dependientes son notificados y actualizados automáticamente.

Código en Python:

class Subject:
    def __init__(self):
        self._observers = []

    def attach(self, observer):
        self._observers.append(observer)

    def detach(self, observer):
        self._observers.remove(observer)

    def notify(self):
        for observer in self._observers:
            observer.update(self)

class ConcreteSubject(Subject):
    def __init__(self, state):
        super().__init__()
        self._state = state

    @property
    def state(self):
        return self._state

    @state.setter
    def state(self, value):
        self._state = value
        self.notify()

class Observer:
    def update(self, subject):
        pass

class ConcreteObserver(Observer):
    def update(self, subject):
        print(f"Observer: Reacted to the state change to {subject.state}")

# Uso del patrón Observer
subject = ConcreteSubject(0)
observer1 = ConcreteObserver()
observer2 = ConcreteObserver()

subject.attach(observer1)
subject.attach(observer2)

subject.state = 1  # Notifica a los observadores
subject.state = 2  # Notifica a los observadores

Explicación del Código:

  1. Subject: Clase base que mantiene una lista de observadores y proporciona métodos para adjuntar, separar y notificar a los observadores.
  2. ConcreteSubject: Extiende Subject y mantiene el estado que interesa a los observadores. Cuando el estado cambia, notifica a todos los observadores.
  3. Observer: Clase base para los observadores que define el método update que será llamado cuando el Subject cambie de estado.
  4. ConcreteObserver: Implementa el método update para reaccionar a los cambios en el Subject.

Conclusión

Los patrones de comportamiento son esenciales para gestionar la interacción y comunicación entre objetos en un sistema de software. Proporcionan soluciones reutilizables y flexibles para problemas comunes de diseño, mejorando la mantenibilidad y escalabilidad del código.
