Módulo 4: Patrones de Comportamiento

4-11 Template Method

El patrón Template Method es un patrón de diseño de comportamiento que define el esqueleto de un algoritmo en una operación, diferiendo algunos pasos a las subclases. Permite que las subclases redefinan ciertos pasos de un algoritmo sin cambiar la estructura del mismo.

Conceptos Clave

  • Algoritmo: Secuencia de pasos para realizar una tarea.
  • Método Template: Método que define la estructura del algoritmo.
  • Métodos Abstractos: Métodos que deben ser implementados por las subclases.
  • Métodos Concretos: Métodos que tienen una implementación por defecto y pueden ser sobrescritos por las subclases.

Estructura

  1. Clase Abstracta: Define el método template y los métodos abstractos.
  2. Subclases Concretas: Implementan los métodos abstractos definidos en la clase abstracta.

Ejemplo en Código

A continuación, se muestra un ejemplo en Python para ilustrar el patrón Template Method.

from abc import ABC, abstractmethod

class DataProcessor(ABC):
    def process(self):
        self.read_data()
        self.process_data()
        self.save_data()

    @abstractmethod
    def read_data(self):
        pass

    @abstractmethod
    def process_data(self):
        pass

    @abstractmethod
    def save_data(self):
        pass

class CSVDataProcessor(DataProcessor):
    def read_data(self):
        print("Reading data from CSV file")

    def process_data(self):
        print("Processing CSV data")

    def save_data(self):
        print("Saving processed data to CSV file")

class JSONDataProcessor(DataProcessor):
    def read_data(self):
        print("Reading data from JSON file")

    def process_data(self):
        print("Processing JSON data")

    def save_data(self):
        print("Saving processed data to JSON file")

# Uso del patrón Template Method
csv_processor = CSVDataProcessor()
csv_processor.process()

json_processor = JSONDataProcessor()
json_processor.process()

Explicación del Código

  1. Clase Abstracta DataProcessor:

    • Define el método process que es el método template.
    • Declara métodos abstractos read_data, process_data y save_data que deben ser implementados por las subclases.

  2. Subclase CSVDataProcessor:

    • Implementa los métodos abstractos para leer, procesar y guardar datos en formato CSV.

  3. Subclase JSONDataProcessor:

    • Implementa los métodos abstractos para leer, procesar y guardar datos en formato JSON.

  4. Uso del Patrón:

    • Se crean instancias de CSVDataProcessor y JSONDataProcessor y se llama al método process en cada una de ellas.

Ventajas del Patrón Template Method

  • Reutilización de Código: Permite reutilizar la estructura del algoritmo en varias subclases.
  • Flexibilidad: Las subclases pueden redefinir ciertos pasos del algoritmo sin cambiar su estructura.
  • Mantenimiento: Facilita el mantenimiento del código al centralizar la estructura del algoritmo en una clase abstracta.

Desventajas del Patrón Template Method

  • Complejidad: Puede aumentar la complejidad del código debido a la introducción de múltiples clases y métodos abstractos.
  • Rigidez: La estructura del algoritmo está fijada en la clase abstracta, lo que puede limitar la flexibilidad en algunos casos.

Ejercicio Práctico

  1. Ejercicio 1: Implementa una clase XMLDataProcessor que herede de DataProcessor y que implemente los métodos abstractos para leer, procesar y guardar datos en formato XML.
  2. Ejercicio 2: Modifica la clase DataProcessor para incluir un nuevo método abstracto validate_data y actualiza las subclases para implementar este nuevo método.

Conclusión

El patrón Template Method es una herramienta poderosa para definir la estructura de un algoritmo y permitir que las subclases implementen los detalles específicos. Este patrón es útil cuando se desea reutilizar la estructura del algoritmo en varias subclases y mantener el código organizado y fácil de mantener.
