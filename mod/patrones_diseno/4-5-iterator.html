Módulo 4: Patrones de Comportamiento

4-5: Iterator

El patrón Iterator es uno de los patrones de diseño de comportamiento que permite recorrer los elementos de una colección sin exponer su representación subyacente. Este patrón proporciona una forma de acceder secuencialmente a los elementos de una colección sin necesidad de conocer su estructura interna.

Conceptos Clave

  • Iterador: Objeto que permite recorrer una colección y acceder a sus elementos.
  • Colección: Conjunto de elementos que se pueden recorrer mediante un iterador.
  • Interfaz de Iterador: Define las operaciones necesarias para recorrer una colección (por ejemplo, next(), hasNext()).

Ventajas del Patrón Iterator

  • Encapsulamiento: Oculta la estructura interna de la colección.
  • Flexibilidad: Permite recorrer diferentes tipos de colecciones de manera uniforme.
  • Simplicidad: Simplifica el código del cliente al proporcionar una interfaz estándar para el recorrido.

Desventajas del Patrón Iterator

  • Sobrecarga: Puede añadir complejidad adicional si no se necesita un recorrido flexible.
  • Rendimiento: En algunas implementaciones, el uso de iteradores puede ser menos eficiente que el acceso directo a los elementos.

Ejemplo de Implementación en Java

A continuación, se presenta un ejemplo de cómo implementar el patrón Iterator en Java.

// Interfaz de Iterador
public interface Iterator {
    boolean hasNext();
    Object next();
}

// Interfaz de Colección
public interface Container {
    Iterator getIterator();
}

// Implementación de la Colección
public class NameRepository implements Container {
    public String names[] = {"John", "Jane", "Jack", "Jill"};

    @Override
    public Iterator getIterator() {
        return new NameIterator();
    }

    private class NameIterator implements Iterator {
        int index;

        @Override
        public boolean hasNext() {
            return index < names.length;
        }

        @Override
        public Object next() {
            if (this.hasNext()) {
                return names[index++];
            }
            return null;
        }
    }
}

// Uso del Iterador
public class IteratorPatternDemo {
    public static void main(String[] args) {
        NameRepository namesRepository = new NameRepository();

        for (Iterator iter = namesRepository.getIterator(); iter.hasNext();) {
            String name = (String) iter.next();
            System.out.println("Name: " + name);
        }
    }
}

Explicación del Código

  1. Interfaz de Iterador: Define los métodos hasNext() y next().
  2. Interfaz de Colección: Define el método getIterator() que devuelve un iterador.
  3. Implementación de la Colección: NameRepository implementa la interfaz Container y proporciona una implementación del método getIterator().
  4. Clase Interna NameIterator: Implementa la interfaz Iterator y proporciona las implementaciones de hasNext() y next().
  5. Uso del Iterador: En IteratorPatternDemo, se crea una instancia de NameRepository y se utiliza el iterador para recorrer los nombres.

Ejercicio Práctico

  1. Implementación Propia: Implementa tu propia versión del patrón Iterator para una colección de números enteros.
  2. Extensión del Ejemplo: Modifica el ejemplo anterior para que el iterador pueda recorrer la colección en orden inverso.

Conclusión

El patrón Iterator es una herramienta poderosa para recorrer colecciones de manera uniforme y encapsulada. Al separar la lógica de recorrido de la estructura de la colección, este patrón facilita el mantenimiento y la extensión del código. Es especialmente útil en situaciones donde se necesita acceder a los elementos de una colección de manera secuencial sin exponer su estructura interna.
