Módulo 2: Patrones Creacionales

2-5: Builder

Introducción al Patrón Builder

El patrón Builder es un patrón de diseño creacional que se utiliza para construir objetos complejos paso a paso. A diferencia de otros patrones creacionales que se enfocan en la creación de objetos a partir de una sola interfaz, el patrón Builder permite la construcción de un objeto a través de una serie de pasos bien definidos. Este patrón es especialmente útil cuando el objeto que se está creando puede tener múltiples representaciones o configuraciones.

Conceptos Clave

  • Separación de la construcción y representación: El patrón Builder separa la construcción de un objeto de su representación final, permitiendo que el mismo proceso de construcción pueda crear diferentes representaciones.
  • Director: Una clase que controla el proceso de construcción utilizando un objeto Builder.
  • Builder: Una interfaz o clase abstracta que define los pasos necesarios para construir el objeto.
  • Producto: El objeto complejo que se está construyendo.

Estructura del Patrón Builder

  1. Director: Controla el proceso de construcción.
  2. Builder: Define la interfaz para crear las partes del objeto.
  3. ConcreteBuilder: Implementa la interfaz Builder y construye las partes del objeto.
  4. Producto: El objeto final que se construye.

Ejemplo de Código

Vamos a implementar un ejemplo simple del patrón Builder en Python para construir una Pizza.

# Producto
class Pizza:
    def __init__(self):
        self.dough = None
        self.sauce = None
        self.topping = None

    def __str__(self):
        return f'Dough: {self.dough}, Sauce: {self.sauce}, Topping: {self.topping}'

# Builder
class PizzaBuilder:
    def __init__(self):
        self.pizza = Pizza()

    def set_dough(self, dough):
        self.pizza.dough = dough
        return self

    def set_sauce(self, sauce):
        self.pizza.sauce = sauce
        return self

    def set_topping(self, topping):
        self.pizza.topping = topping
        return self

    def build(self):
        return self.pizza

# Director
class Director:
    def __init__(self, builder):
        self._builder = builder

    def construct_pizza(self):
        self._builder.set_dough("cross").set_sauce("tomato").set_topping("cheese")
        return self._builder.build()

# Uso del patrón Builder
builder = PizzaBuilder()
director = Director(builder)
pizza = director.construct_pizza()
print(pizza)

Explicación del Código

  1. Producto (Pizza): La clase Pizza representa el objeto complejo que queremos construir. Tiene atributos para la masa, la salsa y el topping.
  2. Builder (PizzaBuilder): La clase PizzaBuilder proporciona métodos para establecer cada parte de la pizza y un método build para retornar la pizza completa.
  3. Director (Director): La clase Director controla el proceso de construcción de la pizza. Utiliza el PizzaBuilder para establecer las partes de la pizza y luego construye la pizza completa.

Ventajas del Patrón Builder

  • Control sobre el proceso de construcción: Permite un control detallado sobre el proceso de construcción del objeto.
  • Reutilización de código: Los mismos pasos de construcción pueden ser reutilizados para crear diferentes representaciones del objeto.
  • Facilidad de mantenimiento: Separar la construcción de la representación facilita el mantenimiento del código.

Desventajas del Patrón Builder

  • Complejidad adicional: Introduce más clases y puede hacer que el código sea más complejo.
  • No siempre necesario: Puede ser excesivo para objetos simples que no requieren una construcción paso a paso.

Ejercicio Práctico

Implemente el patrón Builder para construir una Computer con los siguientes atributos: CPU, RAM, Storage, y GPU. Proporcione métodos en el Builder para establecer cada uno de estos atributos y un método build para retornar la Computer completa.

# Producto
class Computer:
    def __init__(self):
        self.cpu = None
        self.ram = None
        self.storage = None
        self.gpu = None

    def __str__(self):
        return f'CPU: {self.cpu}, RAM: {self.ram}, Storage: {self.storage}, GPU: {self.gpu}'

# Builder
class ComputerBuilder:
    def __init__(self):
        self.computer = Computer()

    def set_cpu(self, cpu):
        self.computer.cpu = cpu
        return self

    def set_ram(self, ram):
        self.computer.ram = ram
        return self

    def set_storage(self, storage):
        self.computer.storage = storage
        return self

    def set_gpu(self, gpu):
        self.computer.gpu = gpu
        return self

    def build(self):
        return self.computer

# Director
class ComputerDirector:
    def __init__(self, builder):
        self._builder = builder

    def construct_computer(self):
        self._builder.set_cpu("Intel i9").set_ram("32GB").set_storage("1TB SSD").set_gpu("NVIDIA RTX 3080")
        return self._builder.build()

# Uso del patrón Builder
builder = ComputerBuilder()
director = ComputerDirector(builder)
computer = director.construct_computer()
print(computer)

Conclusión

El patrón Builder es una herramienta poderosa para la construcción de objetos complejos de manera controlada y paso a paso. Al separar la construcción de la representación, permite una mayor flexibilidad y reutilización del código. Sin embargo, es importante evaluar si su uso es necesario para el problema en cuestión, ya que puede añadir complejidad adicional al diseño del software.
