Introducción

El patrón Chain of Responsibility es un patrón de diseño de comportamiento que permite a múltiples objetos manejar una solicitud sin acoplar explícitamente el remitente de la solicitud a su receptor. Este patrón crea una cadena de objetos receptores y pasa la solicitud a lo largo de la cadena hasta que un objeto la maneja.

Objetivos de Aprendizaje

  • Comprender el propósito y la estructura del patrón Chain of Responsibility.
  • Identificar cuándo y cómo aplicar este patrón en el desarrollo de software.
  • Implementar el patrón Chain of Responsibility en un ejemplo práctico.

Conceptos Clave

  • Responsabilidad: Cada objeto en la cadena tiene la oportunidad de manejar la solicitud.
  • Desacoplamiento: El remitente de la solicitud no necesita conocer cuál objeto en la cadena manejará la solicitud.
  • Flexibilidad: Es fácil añadir o cambiar los manejadores en la cadena sin modificar el código del remitente.

Estructura del Patrón

El patrón Chain of Responsibility se compone de los siguientes componentes:

  • Handler (Manejador): Define una interfaz para manejar las solicitudes.
  • ConcreteHandler (Manejador Concreto): Maneja las solicitudes que le corresponden y pasa la solicitud al siguiente manejador en la cadena si no puede manejarla.
  • Client (Cliente): Inicia la solicitud a uno de los manejadores en la cadena.

Diagrama de Clases

| Componente | Descripción | |------------------|-----------------------------------------------------------------------------| | Handler | Interfaz que declara el método para manejar las solicitudes. | | ConcreteHandler1 | Implementa el método de manejo de solicitudes y pasa la solicitud si no puede manejarla. | | ConcreteHandler2 | Similar a ConcreteHandler1, maneja solicitudes específicas. | | Client | Inicia la solicitud a uno de los manejadores en la cadena. |

Ejemplo Práctico

Vamos a implementar un ejemplo simple en Python donde diferentes manejadores procesan una solicitud de soporte técnico.

Código de Ejemplo

class Handler:
    def __init__(self, successor=None):
        self._successor = successor

    def handle(self, request):
        raise NotImplementedError("Debe implementar el método handle")

class LevelOneSupport(Handler):
    def handle(self, request):
        if request == "simple issue":
            return "LevelOneSupport: Resolved simple issue"
        elif self._successor:
            return self._successor.handle(request)

class LevelTwoSupport(Handler):
    def handle(self, request):
        if request == "complex issue":
            return "LevelTwoSupport: Resolved complex issue"
        elif self._successor:
            return self._successor.handle(request)

class LevelThreeSupport(Handler):
    def handle(self, request):
        if request == "critical issue":
            return "LevelThreeSupport: Resolved critical issue"
        elif self._successor:
            return self._successor.handle(request)

# Configuración de la cadena de responsabilidad
level_three = LevelThreeSupport()
level_two = LevelTwoSupport(level_three)
level_one = LevelOneSupport(level_two)

# Cliente hace una solicitud
requests = ["simple issue", "complex issue", "critical issue", "unknown issue"]

for req in requests:
    result = level_one.handle(req)
    print(f"Request: {req} -> {result}")

Explicación del Código

  • Handler: Clase base que define la interfaz handle.
  • LevelOneSupport, LevelTwoSupport, LevelThreeSupport: Manejadores concretos que implementan la lógica para manejar diferentes tipos de solicitudes.
  • Configuración de la cadena: Se crea una cadena de manejadores donde LevelOneSupport pasa la solicitud a LevelTwoSupport si no puede manejarla, y así sucesivamente.
  • Cliente: Realiza varias solicitudes y la cadena de manejadores procesa cada una.

Conclusión

El patrón Chain of Responsibility es útil para situaciones donde múltiples objetos pueden manejar una solicitud y el remitente no necesita saber cuál objeto la manejará. Este patrón promueve el desacoplamiento y la flexibilidad en el manejo de solicitudes. Al implementar este patrón, es importante diseñar cuidadosamente la cadena de manejadores para asegurar que todas las solicitudes sean procesadas adecuadamente.
