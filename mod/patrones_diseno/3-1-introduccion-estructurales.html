¿Qué son los Patrones Estructurales?

Los patrones estructurales son un tipo de patrones de diseño que se centran en cómo se componen las clases y los objetos para formar estructuras más grandes y complejas. Estos patrones ayudan a garantizar que, cuando se combinan, las clases y los objetos funcionen juntos de manera eficiente y flexible.

Características Clave de los Patrones Estructurales

  1. Composición de Objetos y Clases: Facilitan la creación de relaciones entre objetos y clases.
  2. Flexibilidad y Reutilización: Promueven la reutilización del código y la flexibilidad en la estructura del software.
  3. Separación de Intereses: Ayudan a separar las preocupaciones del sistema, haciendo que el código sea más manejable y mantenible.

Tipos de Patrones Estructurales

A continuación, se presenta una lista de los patrones estructurales más comunes que se estudiarán en este módulo:

  1. Adapter: Permite que clases con interfaces incompatibles trabajen juntas.
  2. Bridge: Desacopla una abstracción de su implementación, permitiendo que ambas varíen independientemente.
  3. Composite: Compone objetos en estructuras de árbol para representar jerarquías parte-todo.
  4. Decorator: Añade responsabilidades adicionales a un objeto de manera dinámica.
  5. Facade: Proporciona una interfaz simplificada a un conjunto de interfaces en un subsistema.
  6. Flyweight: Utiliza el almacenamiento compartido para soportar eficientemente un gran número de objetos de grano fino.
  7. Proxy: Proporciona un sustituto o marcador de posición para otro objeto para controlar el acceso a él.

Ejemplo de un Patrón Estructural: Adapter

Para ilustrar cómo funciona un patrón estructural, consideremos el patrón Adapter. Este patrón permite que dos clases incompatibles trabajen juntas al convertir la interfaz de una clase en otra que el cliente espera.

Ejemplo en Código

Supongamos que tenemos una clase EuropeanPlug que tiene un método connect y queremos usarla con una clase AmericanSocket que espera un método plugIn.

# Clase existente con una interfaz incompatible
class EuropeanPlug:
    def connect(self):
        return "European plug connected"

# Clase que espera una interfaz diferente
class AmericanSocket:
    def plugIn(self):
        return "American socket plugged in"

# Adaptador que permite que EuropeanPlug trabaje con AmericanSocket
class PlugAdapter:
    def __init__(self, european_plug):
        self.european_plug = european_plug

    def plugIn(self):
        return self.european_plug.connect()

# Uso del adaptador
european_plug = EuropeanPlug()
adapter = PlugAdapter(european_plug)
print(adapter.plugIn())  # Salida: European plug connected

Explicación del Código

  1. EuropeanPlug: Clase existente con un método connect.
  2. AmericanSocket: Clase que espera un método plugIn.
  3. PlugAdapter: Clase adaptadora que toma una instancia de EuropeanPlug y proporciona el método plugIn esperado por AmericanSocket.
  4. Uso del Adaptador: Creamos una instancia de EuropeanPlug, la envolvemos en un PlugAdapter y llamamos al método plugIn, que internamente llama al método connect de EuropeanPlug.

Conclusión

Los patrones estructurales son esenciales para crear sistemas de software flexibles y reutilizables. Al comprender y aplicar estos patrones, los desarrolladores pueden construir aplicaciones más robustas y mantenibles.
