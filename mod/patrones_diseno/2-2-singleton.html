Módulo 2: Patrones Creacionales

2-2 Singleton

El patrón Singleton es uno de los patrones creacionales más conocidos y utilizados en el desarrollo de software. Su objetivo principal es asegurar que una clase tenga una única instancia y proporcionar un punto de acceso global a dicha instancia.

Conceptos Clave

  • Instancia Única: Garantiza que solo exista una instancia de la clase.
  • Acceso Global: Proporciona un punto de acceso global a la instancia única.
  • Control de Concurrencia: Asegura que la instancia única sea accesible de manera segura en entornos concurrentes.

Ventajas del Patrón Singleton

  1. Control de Acceso a la Instancia Única: Facilita el control sobre cómo y cuándo se crea la instancia.
  2. Reducción de Uso de Recursos: Evita la creación de múltiples instancias, lo que puede ser costoso en términos de memoria y recursos.
  3. Consistencia Global: Proporciona un estado global consistente a través de la aplicación.

Desventajas del Patrón Singleton

  1. Dificultad en Pruebas Unitarias: Puede complicar las pruebas unitarias debido a su naturaleza global.
  2. Riesgo de Estado Global: Puede llevar a un estado global mutable, lo que puede ser problemático en aplicaciones grandes.
  3. Problemas de Concurrencia: Necesita manejo cuidadoso en entornos concurrentes para evitar problemas de sincronización.

Implementación del Patrón Singleton

A continuación, se muestra una implementación básica del patrón Singleton en Java:

public class Singleton {
    // Instancia única de la clase
    private static Singleton instancia;

    // Constructor privado para evitar la creación de instancias
    private Singleton() {}

    // Método público para obtener la instancia única
    public static Singleton getInstancia() {
        if (instancia == null) {
            instancia = new Singleton();
        }
        return instancia;
    }
}

Explicación del Código

  1. Instancia Única: La variable instancia es estática y privada, lo que asegura que solo exista una instancia de la clase.
  2. Constructor Privado: El constructor es privado para evitar la creación de instancias desde fuera de la clase.
  3. Método getInstancia: Este método proporciona un punto de acceso global a la instancia única. Si la instancia no existe, se crea; de lo contrario, se devuelve la instancia existente.

Ejemplo de Uso

public class Main {
    public static void main(String[] args) {
        Singleton singleton1 = Singleton.getInstancia();
        Singleton singleton2 = Singleton.getInstancia();

        // Verificar que ambas referencias apuntan a la misma instancia
        System.out.println(singleton1 == singleton2); // Debería imprimir "true"
    }
}

Control de Concurrencia

Para asegurar que el patrón Singleton funcione correctamente en entornos concurrentes, es necesario implementar mecanismos de sincronización. Aquí hay una versión mejorada del patrón Singleton con sincronización:

public class Singleton {
    private static Singleton instancia;

    private Singleton() {}

    public static synchronized Singleton getInstancia() {
        if (instancia == null) {
            instancia = new Singleton();
        }
        return instancia;
    }
}

Ejercicio Práctico

  1. Implementación Básica: Implementa el patrón Singleton en tu lenguaje de programación preferido.
  2. Control de Concurrencia: Mejora tu implementación para manejar entornos concurrentes.
  3. Pruebas Unitarias: Escribe pruebas unitarias para verificar que tu implementación del patrón Singleton funciona correctamente.

Conclusión

El patrón Singleton es una herramienta poderosa para controlar la creación de instancias y proporcionar un punto de acceso global a una instancia única. Sin embargo, debe usarse con cuidado para evitar problemas de estado global y concurrencia. Con una implementación adecuada, el patrón Singleton puede mejorar significativamente la eficiencia y consistencia de tu aplicación.
