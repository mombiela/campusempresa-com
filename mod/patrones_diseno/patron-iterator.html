Introducción

El Patrón Iterator es uno de los patrones de diseño de comportamiento más utilizados en el desarrollo de software. Su propósito principal es proporcionar una forma de acceder secuencialmente a los elementos de una colección sin exponer su representación subyacente.

Objetivos del Patrón Iterator

  • Separación de responsabilidades: Permite que la colección y el iterador tengan responsabilidades separadas.
  • Acceso uniforme: Proporciona una interfaz uniforme para recorrer diferentes tipos de colecciones.
  • Encapsulamiento: Oculta la complejidad de la estructura interna de la colección.

Componentes del Patrón Iterator

El Patrón Iterator generalmente se compone de los siguientes elementos:

  • Iterador (Iterator): Define la interfaz para acceder y recorrer los elementos.
  • Iterador Concreto (Concrete Iterator): Implementa la interfaz del iterador y mantiene el estado de la posición actual en la colección.
  • Colección (Aggregate): Define una interfaz para crear un iterador.
  • Colección Concreta (Concrete Aggregate): Implementa la interfaz de la colección y devuelve una instancia del iterador concreto.

Ejemplo de Implementación en Java

A continuación, se presenta un ejemplo de cómo implementar el Patrón Iterator en Java.

Interfaz del Iterador

public interface Iterator {
    boolean hasNext();
    Object next();
}

Iterador Concreto

public class ConcreteIterator implements Iterator {
    private List<Object> collection;
    private int position = 0;

    public ConcreteIterator(List<Object> collection) {
        this.collection = collection;
    }

    @Override
    public boolean hasNext() {
        return position < collection.size();
    }

    @Override
    public Object next() {
        if (this.hasNext()) {
            return collection.get(position++);
        }
        return null;
    }
}

Interfaz de la Colección

public interface Aggregate {
    Iterator createIterator();
}

Colección Concreta

import java.util.List;
import java.util.ArrayList;

public class ConcreteAggregate implements Aggregate {
    private List<Object> items = new ArrayList<>();

    public void addItem(Object item) {
        items.add(item);
    }

    @Override
    public Iterator createIterator() {
        return new ConcreteIterator(items);
    }
}

Uso del Patrón Iterator

public class Main {
    public static void main(String[] args) {
        ConcreteAggregate collection = new ConcreteAggregate();
        collection.addItem("Item 1");
        collection.addItem("Item 2");
        collection.addItem("Item 3");

        Iterator iterator = collection.createIterator();
        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }
    }
}

Ventajas y Desventajas

Ventajas

  • Simplicidad: Facilita el recorrido de colecciones sin necesidad de conocer su estructura interna.
  • Flexibilidad: Permite cambiar la colección subyacente sin modificar el código que la recorre.
  • Reusabilidad: Los iteradores pueden ser reutilizados para diferentes colecciones.

Desventajas

  • Sobrecarga: Puede añadir complejidad adicional en casos donde no es necesario.
  • Rendimiento: En algunos casos, puede afectar el rendimiento debido a la creación de múltiples objetos iteradores.

Conclusión

El Patrón Iterator es una herramienta poderosa para manejar colecciones de manera eficiente y flexible. Al separar la lógica de recorrido de la estructura de la colección, se logra un código más limpio y mantenible. Este patrón es especialmente útil en situaciones donde se requiere un acceso secuencial a los elementos de una colección sin exponer su estructura interna.
