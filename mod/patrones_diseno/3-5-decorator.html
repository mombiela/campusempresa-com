Módulo 3: Patrones Estructurales

3-5 Decorator

Introducción al Patrón Decorator

El patrón Decorator es un patrón estructural que permite añadir funcionalidades a un objeto de manera dinámica. Este patrón es útil cuando se desea extender las capacidades de una clase sin modificar su código original, lo que promueve el principio de abierto/cerrado (Open/Closed Principle) de SOLID.

Conceptos Clave

  • Componente: La interfaz o clase abstracta que define las operaciones que pueden ser decoradas.
  • Componente Concreto: La implementación concreta del componente.
  • Decorador: Una clase que implementa la misma interfaz que el componente y contiene una referencia a un componente.
  • Decorador Concreto: Una clase que extiende la funcionalidad del componente decorado.

Estructura del Patrón Decorator

La estructura del patrón Decorator se puede visualizar de la siguiente manera:

+-------------------+       +-------------------+
|   Component       |<------|   Decorator       |
|-------------------|       |-------------------|
| + operation()     |       | + operation()     |
+-------------------+       +-------------------+
        ^                          ^
        |                          |
+-------------------+       +-------------------+
| ConcreteComponent |       | ConcreteDecorator |
|-------------------|       |-------------------|
| + operation()     |       | + operation()     |
+-------------------+       +-------------------+

Ejemplo de Código

Vamos a implementar un ejemplo en Python para ilustrar el patrón Decorator. Supongamos que tenemos una clase Coffee y queremos añadirle diferentes tipos de condimentos (decoradores) como leche y azúcar.

# Componente
class Coffee:
    def cost(self):
        return 5

# Decorador
class CoffeeDecorator:
    def __init__(self, coffee):
        self._coffee = coffee

    def cost(self):
        return self._coffee.cost()

# Decorador Concreto
class MilkDecorator(CoffeeDecorator):
    def cost(self):
        return self._coffee.cost() + 1

# Decorador Concreto
class SugarDecorator(CoffeeDecorator):
    def cost(self):
        return self._coffee.cost() + 0.5

# Uso del patrón Decorator
if __name__ == "__main__":
    my_coffee = Coffee()
    print(f"Cost of plain coffee: ${my_coffee.cost()}")

    my_coffee_with_milk = MilkDecorator(my_coffee)
    print(f"Cost of coffee with milk: ${my_coffee_with_milk.cost()}")

    my_coffee_with_milk_and_sugar = SugarDecorator(my_coffee_with_milk)
    print(f"Cost of coffee with milk and sugar: ${my_coffee_with_milk_and_sugar.cost()}")

Explicación del Código

  1. Componente: Coffee es la clase base que define el método cost().
  2. Decorador: CoffeeDecorator es la clase decoradora que contiene una referencia a un objeto Coffee.
  3. Decoradores Concretos: MilkDecorator y SugarDecorator son clases que extienden la funcionalidad de CoffeeDecorator añadiendo sus propios costos.

Ventajas del Patrón Decorator

  • Flexibilidad: Permite añadir funcionalidades de manera dinámica y flexible.
  • Reutilización: Los decoradores pueden ser reutilizados en diferentes contextos.
  • Cumplimiento del Principio de Abierto/Cerrado: Las clases pueden extenderse sin necesidad de modificarlas.

Desventajas del Patrón Decorator

  • Complejidad: Puede añadir complejidad al código debido a la creación de múltiples pequeñas clases.
  • Dificultad en Depuración: Puede ser más difícil de depurar y rastrear debido a la cantidad de objetos decoradores.

Conclusión

El patrón Decorator es una poderosa herramienta para extender la funcionalidad de objetos de manera dinámica y flexible. Es especialmente útil cuando se desea adherir al principio de abierto/cerrado, permitiendo que las clases sean extendidas sin ser modificadas. Sin embargo, es importante usarlo con cuidado para evitar añadir complejidad innecesaria al sistema.

En el próximo tema, exploraremos el patrón Facade, que simplifica la interacción con sistemas complejos proporcionando una interfaz unificada.
