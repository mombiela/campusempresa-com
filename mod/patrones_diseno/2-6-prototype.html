Módulo 2: Patrones Creacionales

Prototype

¿Qué es el Patrón Prototype?

El patrón Prototype es un patrón de diseño creacional que permite la creación de nuevos objetos mediante la clonación de una instancia existente, conocida como prototipo. Este patrón es útil cuando la creación directa de un objeto es costosa o compleja.

Conceptos Clave

  • Prototipo: Una instancia preexistente que se utiliza como plantilla para crear nuevos objetos.
  • Clonación: El proceso de copiar un objeto existente para crear uno nuevo.
  • Shallow Copy (Copia Superficial): Copia los valores de los atributos del objeto original al nuevo objeto, pero no copia los objetos referenciados.
  • Deep Copy (Copia Profunda): Copia los valores de los atributos y también los objetos referenciados, creando copias independientes.

Ventajas del Patrón Prototype

  • Reducción de Costos: Evita la costosa creación de objetos desde cero.
  • Flexibilidad: Permite la creación de objetos complejos de manera sencilla.
  • Desempeño: Mejora el desempeño al evitar la inicialización repetitiva de objetos.

Desventajas del Patrón Prototype

  • Complejidad: Puede ser complejo implementar la clonación profunda.
  • Mantenimiento: Requiere mantener el método de clonación en sincronía con las clases.

Ejemplo de Implementación en Java

A continuación, se presenta un ejemplo de cómo implementar el patrón Prototype en Java.

// Interfaz Prototype con el método clone
public interface Prototype {
    Prototype clone();
}

// Clase concreta que implementa la interfaz Prototype
public class ConcretePrototype implements Prototype {
    private String attribute;

    public ConcretePrototype(String attribute) {
        this.attribute = attribute;
    }

    // Implementación del método clone
    @Override
    public Prototype clone() {
        return new ConcretePrototype(this.attribute);
    }

    public String getAttribute() {
        return attribute;
    }

    public void setAttribute(String attribute) {
        this.attribute = attribute;
    }
}

// Clase Cliente que utiliza el patrón Prototype
public class Client {
    public static void main(String[] args) {
        // Crear un objeto prototipo
        ConcretePrototype prototype = new ConcretePrototype("Valor inicial");

        // Clonar el objeto prototipo
        ConcretePrototype clone = (ConcretePrototype) prototype.clone();

        // Mostrar los atributos de los objetos
        System.out.println("Prototipo: " + prototype.getAttribute());
        System.out.println("Clon: " + clone.getAttribute());

        // Modificar el clon y mostrar los atributos nuevamente
        clone.setAttribute("Nuevo valor");
        System.out.println("Prototipo después de modificar el clon: " + prototype.getAttribute());
        System.out.println("Clon después de modificar: " + clone.getAttribute());
    }
}

Explicación del Código

  1. Interfaz Prototype: Define el método clone que debe ser implementado por las clases que deseen utilizar el patrón Prototype.
  2. Clase ConcretePrototype: Implementa la interfaz Prototype y proporciona una implementación del método clone. En este caso, la clonación es una copia superficial.
  3. Clase Client: Demuestra cómo utilizar el patrón Prototype. Se crea un objeto prototipo y luego se clona. Se muestran los atributos de ambos objetos antes y después de modificar el clon.

Ejercicio Práctico

  1. Objetivo: Implementar el patrón Prototype para una clase Person que tenga atributos como nombre, edad y dirección.
  2. Pasos:
    • Crear la interfaz Prototype con el método clone.
    • Implementar la clase Person que implemente la interfaz Prototype.
    • Crear un objeto Person y clonarlo.
    • Modificar los atributos del clon y verificar que el objeto original no se vea afectado.
public interface Prototype {
    Prototype clone();
}

public class Person implements Prototype {
    private String name;
    private int age;
    private String address;

    public Person(String name, int age, String address) {
        this.name = name;
        this.age = age;
        this.address = address;
    }

    @Override
    public Prototype clone() {
        return new Person(this.name, this.age, this.address);
    }

    // Getters y Setters
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public String getAddress() {
        return address;
    }

    public void setAddress(String address) {
        this.address = address;
    }
}

public class Client {
    public static void main(String[] args) {
        // Crear un objeto Person
        Person person1 = new Person("John Doe", 30, "123 Main St");

        // Clonar el objeto Person
        Person person2 = (Person) person1.clone();

        // Mostrar los atributos de los objetos
        System.out.println("Person 1: " + person1.getName() + ", " + person1.getAge() + ", " + person1.getAddress());
        System.out.println("Person 2: " + person2.getName() + ", " + person2.getAge() + ", " + person2.getAddress());

        // Modificar el clon y mostrar los atributos nuevamente
        person2.setName("Jane Doe");
        person2.setAge(25);
        person2.setAddress("456 Elm St");
        System.out.println("Person 1 después de modificar el clon: " + person1.getName() + ", " + person1.getAge() + ", " + person1.getAddress());
        System.out.println("Person 2 después de modificar: " + person2.getName() + ", " + person2.getAge() + ", " + person2.getAddress());
    }
}

Conclusión

El patrón Prototype es una herramienta poderosa para la creación de objetos cuando la inicialización directa es costosa o compleja. Permite la clonación de objetos existentes, proporcionando flexibilidad y mejorando el desempeño. Sin embargo, es importante considerar la complejidad adicional que puede introducir, especialmente en la implementación de clonaciones profundas. Con la práctica y la comprensión adecuada, el patrón Prototype puede ser una adición valiosa a tu conjunto de herramientas de diseño de software.
