Introducción

El Patrón Interpreter es uno de los patrones de diseño de comportamiento que se utiliza para definir una representación gramatical para un lenguaje y un intérprete que utiliza la representación para interpretar sentencias en el lenguaje. Este patrón es útil cuando se necesita evaluar o interpretar sentencias en un lenguaje específico de dominio (DSL).

Conceptos Clave

  • Gramática: Conjunto de reglas que define la estructura de un lenguaje.
  • Expresión: Una instancia de la gramática que puede ser evaluada.
  • Contexto: Información global que puede ser utilizada durante la interpretación.
  • Interprete: Componente que procesa las expresiones basadas en la gramática definida.

Estructura del Patrón Interpreter

El patrón Interpreter generalmente se compone de las siguientes clases:

  1. AbstractExpression: Define la interfaz para interpretar el contexto.
  2. TerminalExpression: Implementa una operación interpretativa asociada con los elementos terminales de la gramática.
  3. NonTerminalExpression: Implementa una operación interpretativa asociada con los elementos no terminales de la gramática.
  4. Context: Contiene información global que puede ser utilizada durante la interpretación.

Diagrama de Clases

Clase Descripción
AbstractExpression Define la interfaz para interpretar el contexto.
TerminalExpression Implementa la operación interpretativa para elementos terminales.
NonTerminalExpression Implementa la operación interpretativa para elementos no terminales.
Context Contiene información global que puede ser utilizada durante la interpretación.

Ejemplo de Código

A continuación, se muestra un ejemplo simple del Patrón Interpreter en Python. Este ejemplo interpreta una expresión matemática simple.

class Context:
    def __init__(self, expression):
        self.expression = expression
        self.index = 0

class AbstractExpression:
    def interpret(self, context):
        pass

class NumberExpression(AbstractExpression):
    def interpret(self, context):
        start = context.index
        while context.index < len(context.expression) and context.expression[context.index].isdigit():
            context.index += 1
        return int(context.expression[start:context.index])

class AddExpression(AbstractExpression):
    def __init__(self, left, right):
        self.left = left
        self.right = right

    def interpret(self, context):
        return self.left.interpret(context) + self.right.interpret(context)

class SubtractExpression(AbstractExpression):
    def __init__(self, left, right):
        self.left = left
        self.right = right

    def interpret(self, context):
        return self.left.interpret(context) - self.right.interpret(context)

# Uso del patrón Interpreter
context = Context("3+5-2")
expression = AddExpression(NumberExpression(), SubtractExpression(NumberExpression(), NumberExpression()))
result = expression.interpret(context)
print(f"Resultado: {result}")  # Salida: Resultado: 6

Explicación del Código

  • Context: Mantiene la expresión y el índice actual.
  • AbstractExpression: Define la interfaz para las expresiones.
  • NumberExpression: Interpreta números en la expresión.
  • AddExpression: Interpreta la operación de suma.
  • SubtractExpression: Interpreta la operación de resta.

En el ejemplo, se crea un contexto con la expresión "3+5-2". Luego, se construye una expresión compuesta utilizando AddExpression y SubtractExpression. Finalmente, se interpreta la expresión completa y se obtiene el resultado.

Conclusión

El Patrón Interpreter es útil para definir y evaluar lenguajes específicos de dominio. Aunque puede no ser adecuado para lenguajes complejos debido a la dificultad de mantener una gramática extensa, es muy efectivo para lenguajes simples y bien definidos. Este patrón permite una fácil extensión y modificación de la gramática y las reglas de interpretación.
