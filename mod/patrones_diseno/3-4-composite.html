Módulo 3: Patrones Estructurales

Composite

Introducción al Patrón Composite

El patrón Composite es un patrón estructural que permite tratar objetos individuales y composiciones de objetos de manera uniforme. Este patrón es útil cuando necesitas representar jerarquías de objetos y quieres que el código que usa estos objetos trate tanto a los objetos individuales como a las composiciones de manera uniforme.

Conceptos Clave

  • Componente: Define la interfaz para los objetos en la composición.
  • Hoja (Leaf): Representa los objetos individuales en la composición. Una hoja no tiene hijos.
  • Compuesto (Composite): Define un comportamiento para los componentes que tienen hijos. Almacena subcomponentes y delega las operaciones a estos subcomponentes.

Estructura del Patrón Composite

La estructura del patrón Composite se puede representar de la siguiente manera:

Component
  + Operation()
  + Add(Component)
  + Remove(Component)
  + GetChild(int)

Leaf
  + Operation()

Composite
  + Operation()
  + Add(Component)
  + Remove(Component)
  + GetChild(int)
  - children: List<Component>

Ejemplo de Código

Vamos a implementar un ejemplo en Python para ilustrar cómo funciona el patrón Composite.

Definición de la Interfaz Componente

from abc import ABC, abstractmethod

class Component(ABC):
    @abstractmethod
    def operation(self):
        pass

Implementación de la Hoja

class Leaf(Component):
    def __init__(self, name):
        self.name = name

    def operation(self):
        print(f"Leaf {self.name} operation")

Implementación del Compuesto

class Composite(Component):
    def __init__(self):
        self._children = []

    def add(self, component):
        self._children.append(component)

    def remove(self, component):
        self._children.remove(component)

    def get_child(self, index):
        return self._children[index]

    def operation(self):
        print("Composite operation")
        for child in self._children:
            child.operation()

Uso del Patrón Composite

if __name__ == "__main__":
    # Crear hojas
    leaf1 = Leaf("A")
    leaf2 = Leaf("B")
    
    # Crear compuesto
    composite = Composite()
    composite.add(leaf1)
    composite.add(leaf2)
    
    # Operación en el compuesto
    composite.operation()

Ventajas del Patrón Composite

  • Simplicidad: Simplifica el código cliente al tratar objetos individuales y compuestos de manera uniforme.
  • Flexibilidad: Facilita la adición de nuevos tipos de componentes.

Desventajas del Patrón Composite

  • Complejidad: Puede introducir complejidad adicional en el diseño del sistema.
  • Rendimiento: Puede afectar el rendimiento si la estructura de la composición es muy grande.

Conclusión

El patrón Composite es una poderosa herramienta para manejar jerarquías de objetos de manera uniforme. Al entender y aplicar este patrón, puedes simplificar el manejo de estructuras complejas y mejorar la flexibilidad de tu código. Sin embargo, es importante considerar las posibles desventajas, como la complejidad adicional y el impacto en el rendimiento.
