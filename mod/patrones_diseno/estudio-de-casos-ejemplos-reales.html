Introducción

En esta sección, exploraremos cómo los patrones de diseño se aplican en situaciones del mundo real. Analizaremos casos específicos y veremos cómo los patrones de diseño pueden resolver problemas comunes en el desarrollo de software. Este enfoque práctico te ayudará a entender mejor la utilidad y la implementación de estos patrones.

Caso 1: Sistema de Gestión de Pedidos

Contexto

Una empresa de comercio electrónico necesita un sistema para gestionar los pedidos de sus clientes. El sistema debe ser flexible y escalable para manejar diferentes tipos de pedidos y métodos de pago.

Problema

El sistema actual es monolítico y difícil de mantener. Cada vez que se agrega un nuevo tipo de pedido o método de pago, se requiere una reescritura significativa del código.

Solución: Patrón de Diseño Estrategia

El patrón de diseño Estrategia permite definir una familia de algoritmos, encapsular cada uno de ellos y hacerlos intercambiables. Este patrón permite que el algoritmo varíe independientemente de los clientes que lo utilizan.

Implementación

  1. Definir la interfaz de la estrategia:

    public interface PaymentStrategy {
    void pay(int amount);
}

  2. Implementar diferentes estrategias de pago:

    public class CreditCardPayment implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " using Credit Card.");
    }
}

public class PayPalPayment implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " using PayPal.");
    }
}

  3. Contexto que utiliza la estrategia:

    public class Order {
    private PaymentStrategy paymentStrategy;

    public void setPaymentStrategy(PaymentStrategy paymentStrategy) {
        this.paymentStrategy = paymentStrategy;
    }

    public void processOrder(int amount) {
        paymentStrategy.pay(amount);
    }
}

  4. Uso del patrón Estrategia:

    public class Main {
    public static void main(String[] args) {
        Order order = new Order();

        // Pagar con tarjeta de crédito
        order.setPaymentStrategy(new CreditCardPayment());
        order.processOrder(100);

        // Pagar con PayPal
        order.setPaymentStrategy(new PayPalPayment());
        order.processOrder(200);
    }
}

Beneficios

  • Flexibilidad: Fácil de agregar nuevos métodos de pago sin modificar el código existente.
  • Mantenibilidad: Separación clara de las responsabilidades.

Caso 2: Sistema de Notificaciones

Contexto

Una aplicación de mensajería necesita enviar notificaciones a los usuarios a través de diferentes canales como email, SMS y notificaciones push.

Problema

El código para enviar notificaciones está disperso y duplicado en varias partes de la aplicación, lo que dificulta su mantenimiento y escalabilidad.

Solución: Patrón de Diseño Observador

El patrón Observador define una dependencia uno-a-muchos entre objetos, de manera que cuando un objeto cambia de estado, todos sus dependientes son notificados y actualizados automáticamente.

Implementación

  1. Definir la interfaz del observador:

    public interface Observer {
    void update(String message);
}

  2. Implementar diferentes observadores:

    public class EmailNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("Email notification: " + message);
    }
}

public class SMSNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("SMS notification: " + message);
    }
}

public class PushNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("Push notification: " + message);
    }
}

  3. Definir el sujeto que notifica a los observadores:

    import java.util.ArrayList;
import java.util.List;

public class NotificationService {
    private List<Observer> observers = new ArrayList<>();

    public void addObserver(Observer observer) {
        observers.add(observer);
    }

    public void removeObserver(Observer observer) {
        observers.remove(observer);
    }

    public void notifyObservers(String message) {
        for (Observer observer : observers) {
            observer.update(message);
        }
    }
}

  4. Uso del patrón Observador:

    public class Main {
    public static void main(String[] args) {
        NotificationService notificationService = new NotificationService();

        Observer emailNotifier = new EmailNotifier();
        Observer smsNotifier = new SMSNotifier();
        Observer pushNotifier = new PushNotifier();

        notificationService.addObserver(emailNotifier);
        notificationService.addObserver(smsNotifier);
        notificationService.addObserver(pushNotifier);

        notificationService.notifyObservers("New message received!");
    }
}

Beneficios

  • Desacoplamiento: Los observadores están desacoplados del sujeto.
  • Escalabilidad: Fácil de agregar nuevos tipos de notificaciones.

Conclusión

El estudio de casos reales nos muestra cómo los patrones de diseño pueden resolver problemas comunes en el desarrollo de software. Al aplicar patrones como Estrategia y Observador, podemos crear sistemas más flexibles, mantenibles y escalables. Estos ejemplos prácticos ilustran la importancia de los patrones de diseño y cómo pueden ser implementados en situaciones del mundo real.
