Módulo 5: Aplicación de Patrones de Diseño

Estudio de Casos

En esta sección, analizaremos varios estudios de casos para entender cómo se aplican los patrones de diseño en situaciones del mundo real. Revisaremos ejemplos concretos y discutiremos cómo los patrones de diseño pueden resolver problemas específicos en el desarrollo de software.

Caso 1: Sistema de Gestión de Pedidos

Contexto

Una empresa de comercio electrónico necesita un sistema para gestionar los pedidos de sus clientes. El sistema debe ser capaz de manejar diferentes tipos de pedidos (por ejemplo, pedidos normales, pedidos urgentes) y permitir la integración con múltiples proveedores de servicios de pago.

Patrones Aplicados

  1. Factory Method
  2. Strategy
  3. Observer

Implementación

Factory Method

El patrón Factory Method se utiliza para crear diferentes tipos de pedidos sin especificar la clase exacta del objeto que se va a crear.

from abc import ABC, abstractmethod

class Pedido(ABC):
    @abstractmethod
    def procesar_pedido(self):
        pass

class PedidoNormal(Pedido):
    def procesar_pedido(self):
        print("Procesando pedido normal")

class PedidoUrgente(Pedido):
    def procesar_pedido(self):
        print("Procesando pedido urgente")

class PedidoFactory(ABC):
    @abstractmethod
    def crear_pedido(self):
        pass

class PedidoNormalFactory(PedidoFactory):
    def crear_pedido(self):
        return PedidoNormal()

class PedidoUrgenteFactory(PedidoFactory):
    def crear_pedido(self):
        return PedidoUrgente()

Strategy

El patrón Strategy se utiliza para definir una familia de algoritmos (en este caso, métodos de pago) y hacerlos intercambiables.

from abc import ABC, abstractmethod

class MetodoPago(ABC):
    @abstractmethod
    def pagar(self, monto):
        pass

class PagoTarjeta(MetodoPago):
    def pagar(self, monto):
        print(f"Pagando {monto} con tarjeta de crédito")

class PagoPayPal(MetodoPago):
    def pagar(self, monto):
        print(f"Pagando {monto} con PayPal")

class ContextoPago:
    def __init__(self, metodo_pago: MetodoPago):
        self._metodo_pago = metodo_pago

    def ejecutar_pago(self, monto):
        self._metodo_pago.pagar(monto)

Observer

El patrón Observer se utiliza para notificar a los clientes sobre el estado de sus pedidos.

class Cliente:
    def actualizar(self, mensaje):
        print(f"Notificación al cliente: {mensaje}")

class Pedido:
    def __init__(self):
        self._observadores = []

    def agregar_observador(self, observador):
        self._observadores.append(observador)

    def notificar_observadores(self, mensaje):
        for observador in self._observadores:
            observador.actualizar(mensaje)

    def procesar_pedido(self):
        # Lógica de procesamiento del pedido
        self.notificar_observadores("Su pedido ha sido procesado")

Caso 2: Sistema de Gestión de Usuarios

Contexto

Una aplicación web necesita un sistema para gestionar usuarios y sus roles. El sistema debe permitir la creación, actualización y eliminación de usuarios, así como la asignación de roles específicos.

Patrones Aplicados

  1. Singleton
  2. Composite
  3. Decorator

Implementación

Singleton

El patrón Singleton se utiliza para asegurar que solo haya una instancia del gestor de usuarios.

class GestorUsuarios:
    _instancia = None

    def __new__(cls):
        if cls._instancia is None:
            cls._instancia = super(GestorUsuarios, cls).__new__(cls)
        return cls._instancia

    def __init__(self):
        self.usuarios = []

    def agregar_usuario(self, usuario):
        self.usuarios.append(usuario)

Composite

El patrón Composite se utiliza para gestionar la jerarquía de roles de los usuarios.

class Rol:
    def __init__(self, nombre):
        self.nombre = nombre
        self.sub_roles = []

    def agregar_sub_rol(self, rol):
        self.sub_roles.append(rol)

    def mostrar_roles(self, nivel=0):
        print("  " * nivel + self.nombre)
        for sub_rol in self.sub_roles:
            sub_rol.mostrar_roles(nivel + 1)

Decorator

El patrón Decorator se utiliza para añadir funcionalidades adicionales a los usuarios, como permisos especiales.

class Usuario:
    def __init__(self, nombre):
        self.nombre = nombre

    def obtener_descripcion(self):
        return self.nombre

class DecoradorUsuario(Usuario):
    def __init__(self, usuario):
        self._usuario = usuario

    def obtener_descripcion(self):
        return self._usuario.obtener_descripcion()

class UsuarioConPermisoEspecial(DecoradorUsuario):
    def obtener_descripcion(self):
        return f"{self._usuario.obtener_descripcion()} con permiso especial"

Conclusión

Los estudios de casos presentados demuestran cómo los patrones de diseño pueden ser aplicados para resolver problemas comunes en el desarrollo de software. Al utilizar patrones como Factory Method, Strategy, Observer, Singleton, Composite y Decorator, los desarrolladores pueden crear sistemas más flexibles, mantenibles y escalables. Es fundamental entender el contexto y los requisitos específicos del problema para seleccionar y aplicar el patrón de diseño más adecuado.
