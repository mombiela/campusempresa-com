Módulo 4: Patrones de Comportamiento

4-10: Strategy

El patrón de diseño Strategy es un patrón de comportamiento que permite definir una familia de algoritmos, encapsular cada uno de ellos y hacerlos intercambiables. Este patrón permite que el algoritmo varíe independientemente de los clientes que lo utilizan.

Conceptos Clave

  1. Contexto: La clase que utiliza una estrategia.
  2. Estrategia: La interfaz común para todas las estrategias concretas.
  3. Estrategia Concreta: Implementaciones específicas de la interfaz de estrategia.

Ejemplo en Código

Vamos a implementar un ejemplo en Python para ilustrar el patrón Strategy. Supongamos que tenemos una aplicación que puede ordenar listas usando diferentes algoritmos de ordenación.

from abc import ABC, abstractmethod

# Estrategia
class SortStrategy(ABC):
    @abstractmethod
    def sort(self, data):
        pass

# Estrategia Concreta A
class BubbleSortStrategy(SortStrategy):
    def sort(self, data):
        n = len(data)
        for i in range(n):
            for j in range(0, n-i-1):
                if data[j] > data[j+1]:
                    data[j], data[j+1] = data[j+1], data[j]
        return data

# Estrategia Concreta B
class QuickSortStrategy(SortStrategy):
    def sort(self, data):
        if len(data) <= 1:
            return data
        pivot = data[len(data) // 2]
        left = [x for x in data if x < pivot]
        middle = [x for x in data if x == pivot]
        right = [x for x in data if x > pivot]
        return self.sort(left) + middle + self.sort(right)

# Contexto
class SortContext:
    def __init__(self, strategy: SortStrategy):
        self._strategy = strategy

    def set_strategy(self, strategy: SortStrategy):
        self._strategy = strategy

    def sort(self, data):
        return self._strategy.sort(data)

# Uso del patrón Strategy
data = [5, 2, 9, 1, 5, 6]

# Usando BubbleSort
context = SortContext(BubbleSortStrategy())
print("BubbleSort:", context.sort(data.copy()))

# Usando QuickSort
context.set_strategy(QuickSortStrategy())
print("QuickSort:", context.sort(data.copy()))

Explicación del Código

  1. SortStrategy: Es una clase abstracta que define la interfaz sort.
  2. BubbleSortStrategy y QuickSortStrategy: Son implementaciones concretas de la interfaz SortStrategy.
  3. SortContext: Es la clase que utiliza una estrategia para ordenar los datos. Permite cambiar la estrategia en tiempo de ejecución mediante el método set_strategy.

Ventajas del Patrón Strategy

  • Flexibilidad: Permite cambiar el algoritmo utilizado en tiempo de ejecución.
  • Mantenimiento: Facilita la adición de nuevos algoritmos sin modificar el código existente.
  • Encapsulamiento: Cada algoritmo está encapsulado en su propia clase.

Desventajas del Patrón Strategy

  • Aumento de Clases: Puede aumentar el número de clases en el sistema.
  • Comunicación Compleja: Puede ser necesario que el contexto y las estrategias intercambien información adicional.

Conclusión

El patrón Strategy es una poderosa herramienta para diseñar software flexible y extensible. Al encapsular algoritmos en clases separadas, permite cambiar y agregar nuevas estrategias sin modificar el código cliente. Este patrón es especialmente útil en situaciones donde se necesita variar el comportamiento de un algoritmo en tiempo de ejecución.
