Introducción

El Patrón Command es un patrón de diseño de comportamiento que convierte una solicitud en un objeto independiente que contiene toda la información sobre la solicitud. Este patrón permite parametrizar los métodos con diferentes solicitudes, retrasar o poner en cola la ejecución de una solicitud y soportar operaciones que pueden deshacerse.

Conceptos Clave

  • Command: Interfaz que declara un método para ejecutar una operación.
  • ConcreteCommand: Implementa la interfaz Command y define la relación entre el receptor y una acción.
  • Client: Crea un objeto ConcreteCommand y lo configura con el receptor.
  • Invoker: Pide al objeto Command que ejecute la solicitud.
  • Receiver: Conoce cómo realizar las operaciones asociadas a llevar a cabo una solicitud.

Estructura

| Componente | Descripción | |----------------|-----------------------------------------------------------------------------| | Command | Interfaz que declara el método execute. | | ConcreteCommand| Implementa la interfaz Command y define la acción a realizar. | | Client | Crea y configura los objetos Command. | | Invoker | Invoca el método execute en el objeto Command. | | Receiver | Realiza las acciones necesarias para cumplir la solicitud del Command. |

Ejemplo de Código

A continuación, se presenta un ejemplo en Python para ilustrar el Patrón Command:

# Command Interface
class Command:
    def execute(self):
        pass

# ConcreteCommand
class LightOnCommand(Command):
    def __init__(self, light):
        self.light = light

    def execute(self):
        self.light.on()

class LightOffCommand(Command):
    def __init__(self, light):
        self.light = light

    def execute(self):
        self.light.off()

# Receiver
class Light:
    def on(self):
        print("The light is on")

    def off(self):
        print("The light is off")

# Invoker
class RemoteControl:
    def __init__(self):
        self.command = None

    def set_command(self, command):
        self.command = command

    def press_button(self):
        self.command.execute()

# Client
if __name__ == "__main__":
    light = Light()
    light_on = LightOnCommand(light)
    light_off = LightOffCommand(light)

    remote = RemoteControl()
    remote.set_command(light_on)
    remote.press_button()

    remote.set_command(light_off)
    remote.press_button()

Explicación del Código

  1. Command Interface: La clase Command define la interfaz con el método execute.
  2. ConcreteCommand: Las clases LightOnCommand y LightOffCommand implementan la interfaz Command y definen las acciones específicas para encender y apagar la luz.
  3. Receiver: La clase Light contiene los métodos on y off que realizan las acciones reales.
  4. Invoker: La clase RemoteControl mantiene una referencia a un objeto Command y llama a su método execute cuando se presiona el botón.
  5. Client: En el bloque if __name__ == "__main__":, se crean los objetos Light, LightOnCommand, LightOffCommand y RemoteControl. El cliente configura el RemoteControl con los comandos y simula la pulsación del botón.

Conclusión

El Patrón Command es una poderosa herramienta para desacoplar el objeto que invoca una operación del objeto que realmente la ejecuta. Esto permite una mayor flexibilidad y extensibilidad en el diseño de software, facilitando la implementación de operaciones que pueden ser parametrizadas, deshechas o puestas en cola. Este patrón es especialmente útil en aplicaciones que requieren un alto grado de flexibilidad y control sobre las acciones que se ejecutan.
