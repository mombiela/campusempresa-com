Módulo 4: Patrones de Comportamiento

Chain of Responsibility

El patrón Chain of Responsibility (Cadena de Responsabilidad) es un patrón de diseño de comportamiento que permite que un objeto pase una solicitud a una cadena de manejadores. Al recibir una solicitud, cada manejador decide si procesa la solicitud o la pasa al siguiente manejador en la cadena.

Conceptos Clave

  1. Manejador (Handler): Interfaz o clase abstracta que define un método para manejar las solicitudes.
  2. Manejadores Concretos (Concrete Handlers): Clases que implementan la interfaz del manejador y procesan las solicitudes o las pasan al siguiente manejador.
  3. Cliente (Client): El objeto que inicia la solicitud.

Estructura

La estructura del patrón Chain of Responsibility se puede representar con el siguiente diagrama:

Client -> Handler1 -> Handler2 -> Handler3 -> ... -> HandlerN

Ejemplo en Código

Vamos a implementar un ejemplo simple en Python para ilustrar cómo funciona el patrón Chain of Responsibility. Supongamos que estamos desarrollando un sistema de soporte técnico donde las solicitudes de soporte pueden ser manejadas por diferentes niveles de soporte: Soporte Básico, Soporte Avanzado y Soporte Experto.

class Handler:
    def __init__(self, successor=None):
        self._successor = successor

    def handle(self, request):
        raise NotImplementedError("Subclasses must implement 'handle' method")

class BasicSupport(Handler):
    def handle(self, request):
        if request == "basic":
            print("BasicSupport: Handling request.")
        elif self._successor:
            self._successor.handle(request)

class AdvancedSupport(Handler):
    def handle(self, request):
        if request == "advanced":
            print("AdvancedSupport: Handling request.")
        elif self._successor:
            self._successor.handle(request)

class ExpertSupport(Handler):
    def handle(self, request):
        if request == "expert":
            print("ExpertSupport: Handling request.")
        elif self._successor:
            self._successor.handle(request)

# Configuración de la cadena de responsabilidad
expert_support = ExpertSupport()
advanced_support = AdvancedSupport(expert_support)
basic_support = BasicSupport(advanced_support)

# Cliente que envía solicitudes
requests = ["basic", "advanced", "expert", "unknown"]

for request in requests:
    print(f"\nClient: Sending request '{request}'")
    basic_support.handle(request)

Explicación del Código

  1. Handler: Clase base que define la interfaz para manejar solicitudes y mantener una referencia al siguiente manejador.
  2. BasicSupport, AdvancedSupport, ExpertSupport: Clases concretas que implementan el método handle para procesar solicitudes específicas.
  3. Cadena de Responsabilidad: Se configura al crear instancias de los manejadores y encadenarlas.
  4. Cliente: Envía diferentes solicitudes que son manejadas por la cadena de responsabilidad.

Ventajas y Desventajas

Ventajas:

  • Desacoplamiento: Los objetos que envían solicitudes no necesitan conocer los detalles de los manejadores.
  • Flexibilidad: Es fácil añadir o cambiar manejadores en la cadena sin modificar el código del cliente.

Desventajas:

  • No Garantiza el Manejo: No hay garantía de que una solicitud será manejada si ningún manejador puede procesarla.
  • Depuración Compleja: Puede ser difícil depurar y seguir el flujo de la solicitud a través de la cadena.

Conclusión

El patrón Chain of Responsibility es útil cuando se desea desacoplar el emisor de una solicitud de sus receptores y permitir que múltiples objetos tengan la oportunidad de manejar la solicitud. Este patrón es especialmente útil en sistemas donde las solicitudes pueden ser manejadas por diferentes niveles de procesamiento o diferentes componentes.

En el próximo tema, exploraremos el patrón Command, que encapsula una solicitud como un objeto, permitiendo parametrizar a los clientes con diferentes solicitudes, encolar solicitudes o registrar solicitudes para deshacer operaciones.
