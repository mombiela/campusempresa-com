Módulo 2: Patrones Creacionales

Factory Method

El patrón Factory Method es uno de los patrones creacionales más utilizados en el desarrollo de software. Este patrón proporciona una interfaz para crear objetos en una superclase, pero permite a las subclases alterar el tipo de objetos que se crearán.

¿Qué es el Factory Method?

El Factory Method es un patrón de diseño que define una interfaz para crear un objeto, pero permite a las subclases decidir qué clase instanciar. Este patrón permite a una clase delegar la responsabilidad de la creación de objetos a sus subclases.

Conceptos Clave

  • Interfaz de Creación: Define el método que será utilizado para crear objetos.
  • Subclases Concretas: Implementan el método de creación para instanciar objetos específicos.
  • Producto: El objeto que es creado por el método de fábrica.

Ventajas del Factory Method

  1. Desacoplamiento: Separa el código que crea objetos del código que los utiliza.
  2. Extensibilidad: Facilita la adición de nuevas clases de productos sin modificar el código existente.
  3. Control sobre la Instanciación: Permite controlar cómo y cuándo se crean los objetos.

Desventajas del Factory Method

  1. Complejidad: Puede aumentar la complejidad del código debido a la necesidad de crear múltiples clases y subclases.
  2. Sobrecarga de Subclases: Requiere la creación de subclases adicionales para cada tipo de producto.

Ejemplo de Implementación

A continuación, se presenta un ejemplo de cómo implementar el patrón Factory Method en Python.

from abc import ABC, abstractmethod

# Producto
class Product(ABC):
    @abstractmethod
    def operation(self) -> str:
        pass

# Producto Concreto 1
class ConcreteProduct1(Product):
    def operation(self) -> str:
        return "Resultado del ConcreteProduct1"

# Producto Concreto 2
class ConcreteProduct2(Product):
    def operation(self) -> str:
        return "Resultado del ConcreteProduct2"

# Creador
class Creator(ABC):
    @abstractmethod
    def factory_method(self) -> Product:
        pass

    def some_operation(self) -> str:
        product = self.factory_method()
        result = f"Creador: El mismo código del creador ha trabajado con {product.operation()}"
        return result

# Creador Concreto 1
class ConcreteCreator1(Creator):
    def factory_method(self) -> Product:
        return ConcreteProduct1()

# Creador Concreto 2
class ConcreteCreator2(Creator):
    def factory_method(self) -> Product:
        return ConcreteProduct2()

# Cliente
def client_code(creator: Creator) -> None:
    print(f"Cliente: No conozco la clase del creador, pero aún funciona.\n"
          f"{creator.some_operation()}")

if __name__ == "__main__":
    print("App: Lanzado con ConcreteCreator1.")
    client_code(ConcreteCreator1())
    print("\n")

    print("App: Lanzado con ConcreteCreator2.")
    client_code(ConcreteCreator2())

Explicación del Código

  1. Producto: Product es una clase abstracta que define la interfaz para los productos.
  2. Productos Concretos: ConcreteProduct1 y ConcreteProduct2 son implementaciones concretas de Product.
  3. Creador: Creator es una clase abstracta que declara el método factory_method que debe ser implementado por las subclases.
  4. Creadores Concretos: ConcreteCreator1 y ConcreteCreator2 son implementaciones concretas de Creator que crean instancias de ConcreteProduct1 y ConcreteProduct2, respectivamente.
  5. Cliente: client_code es una función que utiliza el creador sin conocer su clase concreta.

Conclusión

El patrón Factory Method es una herramienta poderosa para manejar la creación de objetos en un sistema de software. Al utilizar este patrón, se puede lograr un alto grado de flexibilidad y extensibilidad en el código, permitiendo que nuevas clases de productos se integren fácilmente sin modificar el código existente. Sin embargo, es importante tener en cuenta que este patrón puede aumentar la complejidad del sistema debido a la necesidad de crear múltiples subclases.
