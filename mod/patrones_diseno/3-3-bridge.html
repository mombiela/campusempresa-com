Módulo 3: Patrones Estructurales

3-3: Bridge

El patrón Bridge es un patrón de diseño estructural que desacopla una abstracción de su implementación, permitiendo que ambas evolucionen independientemente. Este patrón es útil cuando se desea evitar un vínculo fuerte entre una abstracción y su implementación, facilitando la extensión y modificación del código.

Conceptos Clave

  • Abstracción: La interfaz o clase abstracta que define operaciones de alto nivel.
  • Implementación: La interfaz o clase abstracta que define operaciones de bajo nivel.
  • ConcretoAbstracción: Una clase que extiende la Abstracción y utiliza una Implementación concreta.
  • ConcretoImplementación: Una clase que implementa la interfaz de Implementación.

Estructura

El patrón Bridge se compone de los siguientes componentes:

  1. Abstracción: Define la interfaz de alto nivel que utiliza la implementación.
  2. Implementación: Define la interfaz de bajo nivel que la Abstracción utiliza.
  3. ConcretoAbstracción: Extiende la Abstracción y utiliza una Implementación concreta.
  4. ConcretoImplementación: Implementa la interfaz de Implementación.

Diagrama UML

Abstracción
+ Implementación: Implementación
+ Operación()

ConcretoAbstracción
+ Operación()

Implementación

ConcretoImplementaciónA
+ OperaciónImplementación()

ConcretoImplementaciónB
+ OperaciónImplementación()

Ejemplo en Código

Supongamos que estamos desarrollando una aplicación de formas geométricas que pueden ser dibujadas en diferentes plataformas (por ejemplo, Windows y Linux). Queremos que las formas y las plataformas sean independientes entre sí.

# Implementación
class DrawingAPI:
    def draw_circle(self, x, y, radius):
        pass

class WindowsAPI(DrawingAPI):
    def draw_circle(self, x, y, radius):
        print(f"Drawing circle on Windows at ({x}, {y}) with radius {radius}")

class LinuxAPI(DrawingAPI):
    def draw_circle(self, x, y, radius):
        print(f"Drawing circle on Linux at ({x}, {y}) with radius {radius}")

# Abstracción
class Shape:
    def __init__(self, drawing_api):
        self.drawing_api = drawing_api

    def draw(self):
        pass

class Circle(Shape):
    def __init__(self, x, y, radius, drawing_api):
        super().__init__(drawing_api)
        self.x = x
        self.y = y
        self.radius = radius

    def draw(self):
        self.drawing_api.draw_circle(self.x, self.y, self.radius)

# Uso
windows_api = WindowsAPI()
linux_api = LinuxAPI()

circle1 = Circle(1, 2, 3, windows_api)
circle2 = Circle(4, 5, 6, linux_api)

circle1.draw()  # Output: Drawing circle on Windows at (1, 2) with radius 3
circle2.draw()  # Output: Drawing circle on Linux at (4, 5) with radius 6

Ventajas del Patrón Bridge

  1. Desacoplamiento: Separa la abstracción de su implementación, permitiendo que ambas evolucionen independientemente.
  2. Extensibilidad: Facilita la adición de nuevas abstracciones e implementaciones sin modificar el código existente.
  3. Flexibilidad: Permite cambiar la implementación en tiempo de ejecución.

Desventajas del Patrón Bridge

  1. Complejidad: Introduce más clases y puede hacer que el diseño sea más complejo.
  2. Sobrecarga: Puede ser excesivo para aplicaciones simples donde el desacoplamiento no es necesario.

Conclusión

El patrón Bridge es una poderosa herramienta para desacoplar la abstracción de su implementación, permitiendo que ambas evolucionen de manera independiente. Este patrón es especialmente útil en sistemas que requieren alta flexibilidad y extensibilidad. Aunque puede introducir cierta complejidad adicional, los beneficios en términos de mantenimiento y escalabilidad suelen justificar su uso.

En el próximo tema, exploraremos el patrón Composite, que permite tratar objetos individuales y compuestos de manera uniforme.
