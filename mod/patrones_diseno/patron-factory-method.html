Introducción

El patrón Factory Method es uno de los patrones de diseño creacionales más importantes y utilizados en el desarrollo de software. Su principal objetivo es definir una interfaz para crear un objeto, pero permite a las subclases alterar el tipo de objetos que se crearán.

Conceptos Clave

  • Propósito: Proveer una interfaz para la creación de objetos en una superclase, mientras permite a las subclases modificar el tipo de objetos que se crearán.
  • Aplicabilidad: Se utiliza cuando una clase no puede anticipar el tipo de objetos que debe crear o cuando una clase quiere que sus subclases especifiquen los objetos que crea.
  • Participantes:
    • Product: Define la interfaz de los objetos que el método de fábrica crea.
    • ConcreteProduct: Implementa la interfaz Product.
    • Creator: Declara el método de fábrica, que devuelve un objeto de tipo Product.
    • ConcreteCreator: Sobrescribe el método de fábrica para devolver una instancia de ConcreteProduct.

Ejemplo de Código

A continuación, se presenta un ejemplo en Python para ilustrar el patrón Factory Method:

from abc import ABC, abstractmethod

# Product
class Button(ABC):
    @abstractmethod
    def render(self):
        pass

# ConcreteProduct
class WindowsButton(Button):
    def render(self):
        return "Render a button in Windows style"

class HTMLButton(Button):
    def render(self):
        return "Render a button in HTML style"

# Creator
class Dialog(ABC):
    @abstractmethod
    def create_button(self) -> Button:
        pass

    def render(self):
        button = self.create_button()
        return button.render()

# ConcreteCreator
class WindowsDialog(Dialog):
    def create_button(self) -> Button:
        return WindowsButton()

class WebDialog(Dialog):
    def create_button(self) -> Button:
        return HTMLButton()

# Cliente
def client_code(dialog: Dialog):
    print(dialog.render())

# Uso
if __name__ == "__main__":
    dialog = WindowsDialog()
    client_code(dialog)

    dialog = WebDialog()
    client_code(dialog)

Explicación del Código

  • Product: Button es una interfaz abstracta que define el método render.
  • ConcreteProduct: WindowsButton y HTMLButton son implementaciones concretas de Button.
  • Creator: Dialog es una clase abstracta que declara el método de fábrica create_button.
  • ConcreteCreator: WindowsDialog y WebDialog son subclases de Dialog que implementan el método de fábrica para crear instancias de WindowsButton y HTMLButton, respectivamente.
  • Cliente: La función client_code utiliza el método render de Dialog para mostrar el botón creado.

Comparación con Otros Patrones Creacionales

| Patrón | Propósito | Ejemplo de Uso | |-----------------------|---------------------------------------------------------------------------|-------------------------------------------------------------------------------| | Factory Method | Define una interfaz para crear un objeto, pero deja que las subclases decidan qué clase instanciar. | Crear diferentes tipos de botones (Windows, HTML) sin cambiar el código del cliente. | | Abstract Factory | Proveer una interfaz para crear familias de objetos relacionados o dependientes sin especificar sus clases concretas. | Crear familias de productos (botones, checkboxes) para diferentes plataformas (Windows, Mac). | | Singleton | Asegurar que una clase tenga solo una instancia y proporcionar un punto de acceso global a ella. | Administrar una única instancia de una clase de configuración en toda la aplicación. |

Conclusión

El patrón Factory Method es una herramienta poderosa para la creación de objetos en un sistema de software. Permite a las subclases decidir qué objetos crear, promoviendo la flexibilidad y la extensibilidad del código. Al comprender y aplicar este patrón, los desarrolladores pueden diseñar sistemas más robustos y mantenibles.
