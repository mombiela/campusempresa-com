Módulo 3: Patrones Estructurales

3-6: Facade

¿Qué es el Patrón Facade?

El patrón Facade es un patrón de diseño estructural que proporciona una interfaz simplificada a un conjunto de interfaces en un subsistema. Este patrón define una interfaz de alto nivel que hace que el subsistema sea más fácil de usar.

Objetivos del Patrón Facade

  1. Simplificación: Proporcionar una interfaz simple y fácil de usar para un subsistema complejo.
  2. Desacoplamiento: Reducir la dependencia entre el cliente y el subsistema, promoviendo un diseño más flexible y mantenible.
  3. Encapsulamiento: Ocultar la complejidad del subsistema y exponer solo lo necesario.

Estructura del Patrón Facade

El patrón Facade se compone de los siguientes elementos:

  1. Facade: La clase que proporciona una interfaz simplificada al subsistema.
  2. Subsistema: Un conjunto de clases que implementan la funcionalidad del subsistema. El cliente interactúa con el subsistema a través de la Facade.

Ejemplo en Código

Supongamos que tenemos un sistema de cine que incluye varias clases para gestionar la proyección de películas, el sonido y la iluminación. Vamos a crear una Facade para simplificar la interacción con este sistema.

# Subsistema
class Projector:
    def on(self):
        print("Projector is on")

    def off(self):
        print("Projector is off")

    def play(self, movie):
        print(f"Playing movie: {movie}")

class SoundSystem:
    def on(self):
        print("Sound system is on")

    def off(self):
        print("Sound system is off")

    def set_volume(self, level):
        print(f"Setting volume to {level}")

class Lights:
    def dim(self):
        print("Lights are dimmed")

    def brighten(self):
        print("Lights are brightened")

# Facade
class HomeTheaterFacade:
    def __init__(self):
        self.projector = Projector()
        self.sound_system = SoundSystem()
        self.lights = Lights()

    def watch_movie(self, movie):
        print("Get ready to watch a movie...")
        self.lights.dim()
        self.projector.on()
        self.sound_system.on()
        self.sound_system.set_volume(5)
        self.projector.play(movie)

    def end_movie(self):
        print("Shutting down the home theater...")
        self.projector.off()
        self.sound_system.off()
        self.lights.brighten()

# Cliente
def main():
    home_theater = HomeTheaterFacade()
    home_theater.watch_movie("Inception")
    home_theater.end_movie()

if __name__ == "__main__":
    main()

Explicación del Código

  1. Subsistema: Las clases Projector, SoundSystem y Lights representan el subsistema.
  2. Facade: La clase HomeTheaterFacade proporciona una interfaz simplificada para interactuar con el subsistema.
  3. Cliente: La función main actúa como cliente que utiliza la Facade para ver una película y luego apagar el sistema.

Ventajas del Patrón Facade

  1. Simplicidad: Facilita el uso del subsistema al proporcionar una interfaz más simple.
  2. Desacoplamiento: Reduce la dependencia entre el cliente y las clases del subsistema.
  3. Mantenimiento: Facilita el mantenimiento y la evolución del sistema al encapsular la complejidad.

Desventajas del Patrón Facade

  1. Sobrecarga de la Facade: Si la Facade se vuelve demasiado grande, puede convertirse en un punto de congestión.
  2. Ocultación de Funcionalidad: Puede ocultar funcionalidades importantes del subsistema que podrían ser necesarias para algunos clientes.

Conclusión

El patrón Facade es una herramienta poderosa para simplificar la interacción con subsistemas complejos. Al proporcionar una interfaz de alto nivel, facilita el uso y mantenimiento del sistema, promoviendo un diseño más limpio y desacoplado. Sin embargo, es importante usarlo con cuidado para no ocultar funcionalidades críticas o sobrecargar la Facade con demasiadas responsabilidades.
