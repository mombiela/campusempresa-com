Introducción

El Patrón Mediator es un patrón de diseño de comportamiento que se utiliza para reducir las dependencias entre los objetos de un programa. En lugar de que los objetos se comuniquen directamente entre sí, lo hacen a través de un mediador central. Este patrón es útil para simplificar la comunicación en sistemas complejos y mejorar la mantenibilidad del código.

Conceptos Clave

  • Mediador: Un objeto que encapsula cómo interactúan un conjunto de objetos. Promueve el acoplamiento débil al evitar que los objetos se refieran entre sí explícitamente.
  • Colegas: Los objetos que se comunican entre sí a través del mediador.

Ventajas del Patrón Mediator

  • Reducción de dependencias: Los objetos no necesitan conocer los detalles de otros objetos, solo necesitan conocer al mediador.
  • Facilidad de mantenimiento: Al centralizar la lógica de comunicación, es más fácil modificar y mantener el código.
  • Mejora de la legibilidad: La lógica de interacción se encuentra en un solo lugar, lo que facilita la comprensión del flujo del programa.

Desventajas del Patrón Mediator

  • Complejidad del Mediador: El mediador puede volverse complejo si maneja muchas interacciones.
  • Posible cuello de botella: Si el mediador se convierte en un punto central de comunicación, puede afectar el rendimiento.

Ejemplo de Código

A continuación, se presenta un ejemplo en Python para ilustrar el uso del Patrón Mediator.

class Mediator:
    def notify(self, sender, event):
        pass

class ConcreteMediator(Mediator):
    def __init__(self, component1, component2):
        self._component1 = component1
        self._component1.mediator = self
        self._component2 = component2
        self._component2.mediator = self

    def notify(self, sender, event):
        if event == "A":
            print("Mediator reacts on A and triggers following operations:")
            self._component2.do_c()
        elif event == "D":
            print("Mediator reacts on D and triggers following operations:")
            self._component1.do_b()
            self._component2.do_c()

class BaseComponent:
    def __init__(self, mediator=None):
        self._mediator = mediator

    @property
    def mediator(self):
        return self._mediator

    @mediator.setter
    def mediator(self, mediator):
        self._mediator = mediator

class Component1(BaseComponent):
    def do_a(self):
        print("Component 1 does A.")
        self.mediator.notify(self, "A")

    def do_b(self):
        print("Component 1 does B.")

class Component2(BaseComponent):
    def do_c(self):
        print("Component 2 does C.")

    def do_d(self):
        print("Component 2 does D.")
        self.mediator.notify(self, "D")

# Uso del patrón Mediator
component1 = Component1()
component2 = Component2()
mediator = ConcreteMediator(component1, component2)

print("Client triggers operation A.")
component1.do_a()

print("\nClient triggers operation D.")
component2.do_d()

Explicación del Código

  1. Mediator: Define una interfaz para la comunicación entre componentes.
  2. ConcreteMediator: Implementa la interfaz del mediador y coordina la comunicación entre los componentes.
  3. BaseComponent: Clase base para los componentes que interactúan a través del mediador.
  4. Component1 y Component2: Componentes concretos que realizan acciones y notifican al mediador cuando ocurre un evento.

Conclusión

El Patrón Mediator es una herramienta poderosa para gestionar la comunicación entre objetos en sistemas complejos. Al centralizar la lógica de interacción en un mediador, se reduce el acoplamiento entre los componentes, lo que facilita la mantenibilidad y la escalabilidad del código. Sin embargo, es importante tener en cuenta que el mediador puede volverse un punto de complejidad y un posible cuello de botella si no se gestiona adecuadamente.
