Módulo 3: Patrones Estructurales

Proxy

¿Qué es el Patrón Proxy?

El patrón Proxy es un patrón de diseño estructural que proporciona un sustituto o marcador de posición para otro objeto. Un proxy controla el acceso a un objeto, permitiendo realizar ciertas acciones antes o después de que la solicitud llegue al objeto real.

Tipos de Proxy

  1. Proxy Virtual: Retrasa la creación y la inicialización de objetos costosos hasta que realmente se necesiten.
  2. Proxy Remoto: Representa un objeto que reside en un espacio de direcciones diferente (por ejemplo, en un servidor remoto).
  3. Proxy de Protección: Controla el acceso a un objeto, permitiendo diferentes permisos a diferentes usuarios.
  4. Proxy de Caché: Almacena resultados de operaciones costosas para reutilizarlos en futuras solicitudes.

Estructura del Patrón Proxy

El patrón Proxy generalmente incluye los siguientes componentes:

  • Subject: Define la interfaz común para RealSubject y Proxy.
  • RealSubject: La clase que realiza el trabajo real.
  • Proxy: Mantiene una referencia a RealSubject y controla el acceso a él.

Diagrama UML

+-------------+       +-------------+
|   Subject   |<------|    Proxy    |
+-------------+       +-------------+
| +request()  |       | +request()  |
+-------------+       +-------------+
                           |
                           |
                           v
                    +-------------+
                    | RealSubject |
                    +-------------+
                    | +request()  |
                    +-------------+

Ejemplo en Código

A continuación, se presenta un ejemplo en Python para ilustrar el patrón Proxy:

from abc import ABC, abstractmethod

# Subject
class Subject(ABC):
    @abstractmethod
    def request(self):
        pass

# RealSubject
class RealSubject(Subject):
    def request(self):
        print("RealSubject: Handling request.")

# Proxy
class Proxy(Subject):
    def __init__(self, real_subject: RealSubject):
        self._real_subject = real_subject

    def request(self):
        if self.check_access():
            self._real_subject.request()
            self.log_access()

    def check_access(self) -> bool:
        print("Proxy: Checking access prior to firing a real request.")
        return True

    def log_access(self):
        print("Proxy: Logging the time of request.")

# Cliente
def client_code(subject: Subject):
    subject.request()

# Uso del patrón Proxy
print("Client: Executing the client code with a real subject:")
real_subject = RealSubject()
client_code(real_subject)

print("\nClient: Executing the same client code with a proxy:")
proxy = Proxy(real_subject)
client_code(proxy)

Explicación del Código

  1. Subject: Define la interfaz común request().
  2. RealSubject: Implementa la interfaz request() y realiza la operación real.
  3. Proxy: Implementa la misma interfaz y controla el acceso a RealSubject. En este ejemplo, el proxy realiza una verificación de acceso y registra la solicitud.
  4. Cliente: Utiliza el Subject para realizar operaciones. Puede trabajar tanto con RealSubject como con Proxy.

Ventajas y Desventajas del Patrón Proxy

Ventajas:

  • Controla el acceso a un objeto.
  • Puede agregar funcionalidad adicional sin cambiar el objeto real.
  • Puede mejorar el rendimiento mediante la carga diferida y el almacenamiento en caché.

Desventajas:

  • Puede agregar complejidad adicional al diseño.
  • Puede introducir una latencia adicional debido a la sobrecarga del proxy.

Conclusión

El patrón Proxy es una herramienta poderosa para controlar el acceso a objetos y agregar funcionalidades adicionales sin modificar el objeto real. Es especialmente útil en situaciones donde el acceso a un objeto debe ser controlado o donde la creación de un objeto es costosa y puede ser diferida hasta que sea realmente necesario.
