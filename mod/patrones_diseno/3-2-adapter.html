Módulo 3: Patrones Estructurales

3-2: Adapter

¿Qué es el Patrón Adapter?

El patrón Adapter es un patrón de diseño estructural que permite que dos interfaces incompatibles trabajen juntas. Este patrón actúa como un puente entre dos interfaces, permitiendo que una clase o un conjunto de clases se adapten a una interfaz esperada por el cliente.

Conceptos Clave

  • Cliente: La clase que espera una interfaz específica.
  • Adaptee: La clase que tiene una interfaz incompatible con la del cliente.
  • Adapter: La clase que convierte la interfaz del Adaptee en una interfaz compatible con la del Cliente.

Ejemplo del Mundo Real

Imagina que tienes un enchufe europeo y un dispositivo con un enchufe americano. No puedes conectar el dispositivo directamente a la toma de corriente europea. Necesitas un adaptador que convierta el enchufe americano en uno europeo.

Ejemplo en Código

Supongamos que tenemos una clase EuropeanSocket que tiene un método provideElectricity(), y una clase AmericanPlug que tiene un método plugIn(). Queremos usar el AmericanPlug con el EuropeanSocket.

# Adaptee
class EuropeanSocket:
    def provideElectricity(self):
        return "Electricity from European Socket"

# Target Interface
class AmericanPlug:
    def plugIn(self):
        pass

# Adapter
class SocketAdapter(AmericanPlug):
    def __init__(self, european_socket):
        self.european_socket = european_socket

    def plugIn(self):
        return self.european_socket.provideElectricity()

# Cliente
def main():
    european_socket = EuropeanSocket()
    adapter = SocketAdapter(european_socket)
    print(adapter.plugIn())

if __name__ == "__main__":
    main()

Explicación del Código

  1. Adaptee (EuropeanSocket): La clase que tiene la interfaz incompatible.
  2. Target Interface (AmericanPlug): La interfaz esperada por el cliente.
  3. Adapter (SocketAdapter): La clase que adapta la interfaz de EuropeanSocket a la interfaz AmericanPlug.
  4. Cliente (main): La función que utiliza el SocketAdapter para interactuar con el EuropeanSocket.

Ventajas del Patrón Adapter

  • Reutilización de Código: Permite reutilizar clases existentes que no tienen la interfaz requerida.
  • Flexibilidad: Facilita la integración de nuevas clases sin modificar el código existente.
  • Desacoplamiento: Reduce el acoplamiento entre el cliente y las clases que no tienen la interfaz esperada.

Desventajas del Patrón Adapter

  • Complejidad Adicional: Puede añadir complejidad al sistema debido a la introducción de nuevas clases.
  • Rendimiento: Puede haber una ligera penalización en el rendimiento debido a la llamada adicional a través del adaptador.

Ejercicio Práctico

  1. Descripción: Implementa un adaptador que permita a una clase OldPrinter con un método printOld() ser utilizada por una clase NewPrinter que espera un método printNew().
  2. Objetivo: Practicar la implementación del patrón Adapter para adaptar interfaces incompatibles.
# Adaptee
class OldPrinter:
    def printOld(self):
        return "Printing from Old Printer"

# Target Interface
class NewPrinter:
    def printNew(self):
        pass

# Adapter
class PrinterAdapter(NewPrinter):
    def __init__(self, old_printer):
        self.old_printer = old_printer

    def printNew(self):
        return self.old_printer.printOld()

# Cliente
def main():
    old_printer = OldPrinter()
    adapter = PrinterAdapter(old_printer)
    print(adapter.printNew())

if __name__ == "__main__":
    main()

Conclusión

El patrón Adapter es una herramienta poderosa para integrar clases con interfaces incompatibles, permitiendo la reutilización de código y mejorando la flexibilidad del sistema. Aunque puede añadir cierta complejidad, sus beneficios en términos de desacoplamiento y reutilización de código suelen superar las desventajas. Practicar la implementación de este patrón te ayudará a comprender mejor cómo y cuándo utilizarlo en tus proyectos de software.
