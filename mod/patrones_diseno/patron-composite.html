Introducción

El patrón Composite es un patrón de diseño estructural que permite tratar objetos individuales y compuestos de manera uniforme. Este patrón es útil cuando necesitas representar jerarquías de objetos y deseas que los clientes interactúen con estos objetos de manera consistente, sin preocuparse si están tratando con un objeto simple o compuesto.

Conceptos Clave

  • Componente: Define la interfaz para todos los objetos en la composición, tanto para los objetos simples como para los compuestos.
  • Hoja: Representa los objetos individuales en la composición. Una hoja no tiene hijos.
  • Compuesto: Define el comportamiento de los componentes que tienen hijos. Implementa operaciones relacionadas con los hijos y puede agregar, eliminar o acceder a ellos.

Estructura del Patrón Composite

El patrón Composite se puede visualizar con la siguiente estructura:

Componente Hoja Compuesto
+ operación() + operación() + operación()
+ agregar(Componente)
+ eliminar(Componente)
+ obtenerHijo(int)

Ejemplo de Código

A continuación, se presenta un ejemplo en Java para ilustrar cómo implementar el patrón Composite.

// Interfaz Componente
interface Componente {
    void operacion();
}

// Clase Hoja
class Hoja implements Componente {
    private String nombre;

    public Hoja(String nombre) {
        this.nombre = nombre;
    }

    @Override
    public void operacion() {
        System.out.println("Hoja " + nombre + " operacion realizada.");
    }
}

// Clase Compuesto
class Compuesto implements Componente {
    private List<Componente> hijos = new ArrayList<>();

    @Override
    public void operacion() {
        for (Componente hijo : hijos) {
            hijo.operacion();
        }
    }

    public void agregar(Componente componente) {
        hijos.add(componente);
    }

    public void eliminar(Componente componente) {
        hijos.remove(componente);
    }

    public Componente obtenerHijo(int index) {
        return hijos.get(index);
    }
}

// Uso del patrón Composite
public class Cliente {
    public static void main(String[] args) {
        Componente hoja1 = new Hoja("1");
        Componente hoja2 = new Hoja("2");

        Compuesto compuesto = new Compuesto();
        compuesto.agregar(hoja1);
        compuesto.agregar(hoja2);

        Compuesto compuestoMayor = new Compuesto();
        compuestoMayor.agregar(compuesto);

        compuestoMayor.operacion();
    }
}

Explicación del Código

  1. Interfaz Componente: Define la operación operacion() que será implementada tanto por las hojas como por los compuestos.
  2. Clase Hoja: Implementa la interfaz Componente y define la operación específica para las hojas.
  3. Clase Compuesto: Implementa la interfaz Componente y maneja una lista de hijos que también son componentes. Implementa métodos para agregar, eliminar y obtener hijos.
  4. Cliente: Crea instancias de hojas y compuestos, y demuestra cómo se pueden combinar y operar de manera uniforme.

Conclusión

El patrón Composite es una poderosa herramienta para manejar jerarquías de objetos de manera uniforme. Permite a los clientes tratar objetos individuales y compuestos de la misma manera, simplificando el código y mejorando la flexibilidad. Este patrón es especialmente útil en aplicaciones que manejan estructuras de árbol, como sistemas de archivos, interfaces gráficas y representaciones de documentos.

Recuerda que la clave del patrón Composite es la recursividad y la capacidad de tratar objetos individuales y compuestos de manera uniforme. Practica implementando este patrón en diferentes escenarios para comprender mejor su utilidad y aplicación.
