Introducción

En este tema, aprenderemos sobre el manejo de peticiones y respuestas en el contexto de una REST API. Este conocimiento es fundamental para cualquier profesional de TI que trabaje con servicios web, ya que las APIs RESTful son una de las formas más comunes de comunicación entre sistemas.

Conceptos Clave

  • Peticiones HTTP: Son solicitudes enviadas por el cliente al servidor para realizar alguna acción.
  • Respuestas HTTP: Son las respuestas enviadas por el servidor al cliente después de procesar una petición.
  • Métodos HTTP: GET, POST, PUT, DELETE, PATCH, etc., que definen la acción a realizar.
  • Códigos de Estado HTTP: Indican el resultado de la petición (200 OK, 404 Not Found, 500 Internal Server Error, etc.).
  • Headers HTTP: Información adicional enviada en las peticiones y respuestas (Content-Type, Authorization, etc.).
  • Cuerpo de la Petición/Respuesta: Datos enviados en el cuerpo de la petición o respuesta, generalmente en formato JSON.

Peticiones HTTP

Estructura de una Petición HTTP

Una petición HTTP típica incluye:

  • Línea de solicitud: Contiene el método HTTP, la URL y la versión del protocolo.
  • Headers: Información adicional sobre la petición.
  • Cuerpo: Datos enviados con la petición (opcional, depende del método HTTP).

Ejemplo de una petición HTTP GET:

GET /api/usuarios HTTP/1.1
Host: ejemplo.com
Authorization: Bearer token123
Accept: application/json

Ejemplo de una petición HTTP POST:

POST /api/usuarios HTTP/1.1
Host: ejemplo.com
Content-Type: application/json
Authorization: Bearer token123

{
  "nombre": "Juan Perez",
  "email": "juan.perez@ejemplo.com"
}

Respuestas HTTP

Estructura de una Respuesta HTTP

Una respuesta HTTP típica incluye:

  • Línea de estado: Contiene la versión del protocolo, el código de estado y el mensaje de estado.
  • Headers: Información adicional sobre la respuesta.
  • Cuerpo: Datos enviados con la respuesta (opcional, depende del código de estado).

Ejemplo de una respuesta HTTP 200 OK:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "nombre": "Juan Perez",
  "email": "juan.perez@ejemplo.com"
}

Ejemplo de una respuesta HTTP 404 Not Found:

HTTP/1.1 404 Not Found
Content-Type: application/json

{
  "error": "Usuario no encontrado"
}

Métodos HTTP

Principales Métodos HTTP

  • GET: Recupera información del servidor.
  • POST: Envía datos al servidor para crear un nuevo recurso.
  • PUT: Actualiza un recurso existente en el servidor.
  • DELETE: Elimina un recurso del servidor.
  • PATCH: Aplica modificaciones parciales a un recurso.

| Método | Descripción | Idempotente | Seguro | |--------|-----------------------------------|-------------|--------| | GET | Recupera información | Sí | Sí | | POST | Crea un nuevo recurso | No | No | | PUT | Actualiza un recurso existente | Sí | No | | DELETE | Elimina un recurso | Sí | No | | PATCH | Modifica parcialmente un recurso | No | No |

Códigos de Estado HTTP

Principales Códigos de Estado

  • 2xx (Éxito): La petición fue recibida, entendida y aceptada.
    • 200 OK: La petición fue exitosa.
    • 201 Created: Un nuevo recurso fue creado.
  • 4xx (Errores del Cliente): La petición contiene sintaxis incorrecta o no puede ser procesada.
    • 400 Bad Request: La petición es inválida.
    • 401 Unauthorized: La autenticación es requerida.
    • 404 Not Found: El recurso no fue encontrado.
  • 5xx (Errores del Servidor): El servidor falló al cumplir una petición aparentemente válida.
    • 500 Internal Server Error: Error genérico del servidor.
    • 503 Service Unavailable: El servicio no está disponible.

Headers HTTP

Principales Headers

  • Content-Type: Indica el tipo de contenido del cuerpo de la petición/respuesta.
  • Authorization: Contiene credenciales para autenticar al cliente.
  • Accept: Indica qué tipos de contenido puede procesar el cliente.

Ejemplo de uso de headers:

GET /api/usuarios HTTP/1.1
Host: ejemplo.com
Authorization: Bearer token123
Accept: application/json

Ejemplos Prácticos

Ejemplo de una Petición y Respuesta Completa

Petición POST para crear un usuario:

POST /api/usuarios HTTP/1.1
Host: ejemplo.com
Content-Type: application/json
Authorization: Bearer token123

{
  "nombre": "Juan Perez",
  "email": "juan.perez@ejemplo.com"
}

Respuesta del servidor:

HTTP/1.1 201 Created
Content-Type: application/json

{
  "id": 1,
  "nombre": "Juan Perez",
  "email": "juan.perez@ejemplo.com"
}

Conclusión

El manejo de peticiones y respuestas es un aspecto crucial en el desarrollo y consumo de APIs RESTful. Comprender la estructura de las peticiones y respuestas, los métodos HTTP, los códigos de estado y los headers es esencial para interactuar eficazmente con cualquier API. Con esta base, estarás mejor preparado para desarrollar y consumir servicios web de manera eficiente y efectiva.
