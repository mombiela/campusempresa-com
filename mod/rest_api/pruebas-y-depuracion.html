Introducción

En esta sección, aprenderás cómo realizar pruebas y depuración de una REST API. Las pruebas y la depuración son esenciales para garantizar que tu API funcione correctamente y sea robusta frente a errores y excepciones.

Tipos de pruebas

Pruebas Unitarias

Las pruebas unitarias se centran en probar componentes individuales de la API, como funciones o métodos. Estas pruebas son cruciales para asegurar que cada parte de tu API funcione como se espera.

  • Objetivo: Verificar que cada unidad de código funcione correctamente.
  • Herramientas comunes: Jest, Mocha, Chai (para Node.js), JUnit (para Java).
// Ejemplo de prueba unitaria con Jest
const request = require('supertest');
const app = require('../app');

describe('GET /api/resource', () => {
  it('should return a list of resources', async () => {
    const res = await request(app).get('/api/resource');
    expect(res.statusCode).toEqual(200);
    expect(res.body).toHaveProperty('data');
  });
});

Pruebas de Integración

Las pruebas de integración verifican que diferentes módulos o servicios de tu API funcionen juntos correctamente.

  • Objetivo: Asegurar que los componentes integrados funcionen como un todo.
  • Herramientas comunes: Supertest, Postman, Newman.
// Ejemplo de prueba de integración con Supertest
const request = require('supertest');
const app = require('../app');

describe('POST /api/resource', () => {
  it('should create a new resource', async () => {
    const res = await request(app)
      .post('/api/resource')
      .send({
        name: 'New Resource',
        description: 'This is a new resource'
      });
    expect(res.statusCode).toEqual(201);
    expect(res.body).toHaveProperty('id');
  });
});

Pruebas Funcionales

Las pruebas funcionales se centran en verificar que la API cumpla con los requisitos funcionales especificados.

  • Objetivo: Validar que la API haga lo que se supone que debe hacer.
  • Herramientas comunes: Postman, Newman, SoapUI.
// Ejemplo de prueba funcional con Postman
{
  "info": {
    "name": "API Functional Test",
    "schema": "https://schema.getpostman.com/json/collection/v2.1.0/collection.json"
  },
  "item": [
    {
      "name": "Get Resources",
      "request": {
        "method": "GET",
        "header": [],
        "url": {
          "raw": "http://localhost:3000/api/resource",
          "protocol": "http",
          "host": ["localhost"],
          "port": "3000",
          "path": ["api", "resource"]
        }
      },
      "response": []
    }
  ]
}

Herramientas de depuración

Logs

Los logs son una herramienta fundamental para la depuración. Permiten registrar eventos y errores que ocurren durante la ejecución de la API.

  • Librerías comunes: Winston, Morgan (para Node.js), Log4j (para Java).
// Ejemplo de uso de Winston para logging
const winston = require('winston');

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' })
  ]
});

logger.info('This is an informational message');
logger.error('This is an error message');

Debuggers

Los depuradores permiten pausar la ejecución del código y examinar el estado de la aplicación en tiempo real.

  • Herramientas comunes: Node.js Debugger, Chrome DevTools, Visual Studio Code Debugger.
// Ejemplo de uso del depurador en Node.js
const http = require('http');

const server = http.createServer((req, res) => {
  debugger; // Punto de interrupción
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(3000, '127.0.0.1', () => {
  console.log('Server running at http://127.0.0.1:3000/');
});

Ejercicios prácticos

  1. Prueba Unitaria: Escribe una prueba unitaria para una función que suma dos números.
  2. Prueba de Integración: Crea una prueba de integración para un endpoint que crea un nuevo usuario.
  3. Prueba Funcional: Usa Postman para crear una colección de pruebas funcionales para tu API.
  4. Depuración: Añade logs a tu API y usa un depurador para identificar y corregir un error en un endpoint específico.

Conclusión

Las pruebas y la depuración son componentes esenciales en el desarrollo de una REST API robusta y confiable. Al implementar pruebas unitarias, de integración y funcionales, y al utilizar herramientas de depuración efectivas, puedes asegurar que tu API funcione correctamente y esté preparada para manejar errores y excepciones.
