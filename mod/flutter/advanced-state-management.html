Introducción

En esta sección, profundizaremos en técnicas avanzadas de gestión del estado en Flutter. La gestión del estado es crucial para construir aplicaciones responsivas y mantenibles. Exploraremos varias soluciones de gestión del estado, sus casos de uso y cómo implementarlas de manera efectiva.

Conceptos Clave

  • Gestión del Estado: El proceso de gestionar el estado de una aplicación, asegurando que la interfaz de usuario refleje el estado actual.
  • Widgets con Estado: Widgets que mantienen el estado y pueden reconstruirse cuando el estado cambia.
  • Soluciones de Gestión del Estado: Varias bibliotecas y patrones para gestionar el estado, como Provider, Riverpod, Bloc y Redux.

Provider

Provider es una solución popular de gestión del estado en Flutter. Está construida sobre InheritedWidget y ofrece una forma sencilla de gestionar el estado.

Uso Básico

import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

void main() {
  runApp(
    ChangeNotifierProvider(
      create: (context) => Counter(),
      child: MyApp(),
    ),
  );
}

class Counter with ChangeNotifier {
  int _count = 0;

  int get count => _count;

  void increment() {
    _count++;
    notifyListeners();
  }
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Provider Example')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: <Widget>[
              Text('You have pushed the button this many times:'),
              Consumer<Counter>(
                builder: (context, counter, child) {
                  return Text(
                    '${counter.count}',
                    style: Theme.of(context).textTheme.headline4,
                  );
                },
              ),
            ],
          ),
        ),
        floatingActionButton: FloatingActionButton(
          onPressed: () => context.read<Counter>().increment(),
          tooltip: 'Increment',
          child: Icon(Icons.add),
        ),
      ),
    );
  }
}
  • ChangeNotifierProvider: Proporciona una instancia de un ChangeNotifier a sus descendientes.
  • Consumer: Escucha los cambios en la instancia proporcionada y se reconstruye cuando ocurren cambios.
  • context.read: Accede a la instancia proporcionada sin escuchar cambios.

Riverpod

Riverpod es una solución de gestión del estado más avanzada y flexible en comparación con Provider. Está diseñada para superar algunas de las limitaciones de Provider.

Uso Básico

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

void main() {
  runApp(ProviderScope(child: MyApp()));
}

final counterProvider = StateProvider<int>((ref) => 0);

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Riverpod Example')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: <Widget>[
              Text('You have pushed the button this many times:'),
              Consumer(
                builder: (context, watch, child) {
                  final count = watch(counterProvider).state;
                  return Text(
                    '$count',
                    style: Theme.of(context).textTheme.headline4,
                  );
                },
              ),
            ],
          ),
        ),
        floatingActionButton: FloatingActionButton(
          onPressed: () => context.read(counterProvider).state++,
          tooltip: 'Increment',
          child: Icon(Icons.add),
        ),
      ),
    );
  }
}
  • ProviderScope: Un widget que almacena el estado de todos los proveedores.
  • StateProvider: Un proveedor que expone un estado mutable.
  • context.read: Accede al estado del proveedor sin escuchar cambios.
  • watch: Escucha los cambios en el estado del proveedor y reconstruye el widget cuando ocurren cambios.

Bloc (Business Logic Component)

Bloc es una biblioteca de gestión del estado que ayuda a separar la lógica de negocio de la interfaz de usuario. Utiliza streams para gestionar el estado y los eventos.

Uso Básico

import 'package:flutter/material.dart';
import 'package:flutter_bloc/flutter_bloc.dart';

void main() {
  runApp(MyApp());
}

class CounterCubit extends Cubit<int> {
  CounterCubit() : super(0);

  void increment() => emit(state + 1);
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: BlocProvider(
        create: (context) => CounterCubit(),
        child: CounterPage(),
      ),
    );
  }
}

class CounterPage extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Bloc Example')),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text('You have pushed the button this many times:'),
            BlocBuilder<CounterCubit, int>(
              builder: (context, count) {
                return Text(
                  '$count',
                  style: Theme.of(context).textTheme.headline4,
                );
              },
            ),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () => context.read<CounterCubit>().increment(),
        tooltip: 'Increment',
        child: Icon(Icons.add),
      ),
    );
  }
}
  • Cubit: Una versión más simple de Bloc que gestiona el estado sin eventos.
  • BlocProvider: Proporciona una instancia de un Cubit o Bloc a sus descendientes.
  • BlocBuilder: Escucha los cambios en el estado del Cubit o Bloc y reconstruye el widget cuando ocurren cambios.

Redux

Redux es un contenedor de estado predecible para aplicaciones Dart y Flutter. Está basado en el patrón Redux popularizado por el ecosistema de JavaScript.

Uso Básico

import 'package:flutter/material.dart';
import 'package:flutter_redux/flutter_redux.dart';
import 'package:redux/redux.dart';

void main() {
  final store = Store<int>(counterReducer, initialState: 0);
  runApp(MyApp(store: store));
}

int counterReducer(int state, dynamic action) {
  if (action == Actions.Increment) {
    return state + 1;
  }
  return state;
}

enum Actions { Increment }

class MyApp extends StatelessWidget {
  final Store<int> store;

  MyApp({required this.store});

  @override
  Widget build(BuildContext context) {
    return StoreProvider(
      store: store,
      child: MaterialApp(
        home: Scaffold(
          appBar: AppBar(title: Text('Redux Example')),
          body: Center(
            child: Column(
              mainAxisAlignment: MainAxisAlignment.center,
              children: <Widget>[
                Text('You have pushed the button this many times:'),
                StoreConnector<int, String>(
                  converter: (store) => store.state.toString(),
                  builder: (context, count) {
                    return Text(
                      count,
                      style: Theme.of(context).textTheme.headline4,
                    );
                  },
                ),
              ],
            ),
          ),
          floatingActionButton: StoreConnector<int, VoidCallback>(
            converter: (store) {
              return () => store.dispatch(Actions.Increment);
            },
            builder: (context, callback) {
              return FloatingActionButton(
                onPressed: callback,
                tooltip: 'Increment',
                child: Icon(Icons.add),
              );
            },
          ),
        ),
      ),
    );
  }
}
  • Store: Contiene el estado de la aplicación.
  • Reducer: Una función que toma el estado actual y una acción, y devuelve un nuevo estado.
  • StoreProvider: Proporciona la tienda a sus descendientes.
  • StoreConnector: Conecta la tienda a un widget, permitiendo que se reconstruya cuando el estado cambia.

Conclusión

La gestión avanzada del estado en Flutter implica entender e implementar varias soluciones como Provider, Riverpod, Bloc y Redux. Cada una tiene sus propias fortalezas y casos de uso. Al dominar estas técnicas, puedes construir aplicaciones Flutter más escalables, mantenibles y de alto rendimiento. Elige la solución que mejor se adapte a las necesidades y complejidad de tu proyecto.
