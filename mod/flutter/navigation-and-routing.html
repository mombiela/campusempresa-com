Introducción

La navegación y el enrutamiento son conceptos fundamentales en el desarrollo de aplicaciones móviles. En Flutter, estos conceptos te permiten moverte entre diferentes pantallas (o páginas) en tu aplicación. Esta sección cubrirá los conceptos básicos de navegación y enrutamiento, y gradualmente avanzará hacia temas más avanzados.

Navegación Básica

Usando Navigator

El widget Navigator es la forma principal de gestionar una pila de rutas en Flutter. Cada ruta es una pantalla o página en tu aplicación.

  • Pushing a Route: Añadir una nueva ruta a la pila.
  • Popping a Route: Eliminar la ruta actual de la pila.

Ejemplo: Pushing y Popping Rutas

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: FirstScreen(),
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(builder: (context) => SecondScreen()),
            );
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            Navigator.pop(context);
          },
        ),
      ),
    );
  }
}

En este ejemplo:

  • Navigator.push se usa para navegar a la SecondScreen.
  • Navigator.pop se usa para regresar a la FirstScreen.

Rutas Nombradas

Las rutas nombradas proporcionan una forma de navegar usando identificadores de cadena. Esto puede hacer que tu código sea más limpio y fácil de gestionar, especialmente en aplicaciones más grandes.

Definiendo Rutas Nombradas

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      initialRoute: '/',
      routes: {
        '/': (context) => FirstScreen(),
        '/second': (context) => SecondScreen(),
      },
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.pushNamed(context, '/second');
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            Navigator.pop(context);
          },
        ),
      ),
    );
  }
}

En este ejemplo:

  • Las rutas nombradas se definen en el widget MaterialApp.
  • Navigator.pushNamed se usa para navegar a la SecondScreen.

Pasar Datos Entre Pantallas

A menudo, necesitas pasar datos entre pantallas. Esto se puede hacer usando los métodos de Navigator.

Ejemplo: Pasar Datos

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: FirstScreen(),
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(
                builder: (context) => SecondScreen(data: 'Hello from FirstScreen'),
              ),
            );
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  final String data;

  SecondScreen({required this.data});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: Text(data),
      ),
    );
  }
}

En este ejemplo:

  • Los datos se pasan de FirstScreen a SecondScreen a través del constructor.

Navegación Avanzada

Usando Navigator 2.0

El Navigator 2.0 de Flutter proporciona más control sobre la pila de navegación y es adecuado para escenarios de navegación complejos.

  • API basada en páginas: Permite una navegación más flexible y declarativa.
  • Router: Gestiona el estado de navegación de la aplicación.

Ejemplo: Configuración Básica de Navigator 2.0

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp.router(
      routerDelegate: MyRouterDelegate(),
      routeInformationParser: MyRouteInformationParser(),
    );
  }
}

class MyRouterDelegate extends RouterDelegate with ChangeNotifier, PopNavigatorRouterDelegateMixin {
  final GlobalKey<NavigatorState> navigatorKey;

  MyRouterDelegate() : navigatorKey = GlobalKey<NavigatorState>();

  @override
  Widget build(BuildContext context) {
    return Navigator(
      key: navigatorKey,
      pages: [
        MaterialPage(child: FirstScreen()),
        if (showSecondScreen) MaterialPage(child: SecondScreen()),
      ],
      onPopPage: (route, result) {
        if (!route.didPop(result)) {
          return false;
        }
        showSecondScreen = false;
        notifyListeners();
        return true;
      },
    );
  }

  bool showSecondScreen = false;

  @override
  Future<void> setNewRoutePath(configuration) async {
    if (configuration == '/second') {
      showSecondScreen = true;
    } else {
      showSecondScreen = false;
    }
  }
}

class MyRouteInformationParser extends RouteInformationParser<String> {
  @override
  Future<String> parseRouteInformation(RouteInformation routeInformation) async {
    return routeInformation.location ?? '/';
  }

  @override
  RouteInformation restoreRouteInformation(String configuration) {
    return RouteInformation(location: configuration);
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            (Router.of(context).routerDelegate as MyRouterDelegate).showSecondScreen = true;
            (Router.of(context).routerDelegate as MyRouterDelegate).notifyListeners();
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            (Router.of(context).routerDelegate as MyRouterDelegate).showSecondScreen = false;
            (Router.of(context).routerDelegate as MyRouterDelegate).notifyListeners();
          },
        ),
      ),
    );
  }
}

En este ejemplo:

  • RouterDelegate y RouteInformationParser se usan para gestionar el estado de navegación.
  • El widget Navigator se usa para construir la pila de navegación basada en el estado.

Conclusión

La navegación y el enrutamiento son esenciales para construir aplicaciones multi-pantalla en Flutter. Este tema cubrió:

  • Navegación básica usando Navigator.
  • Rutas nombradas para una navegación más limpia.
  • Pasar datos entre pantallas.
  • Navegación avanzada usando Navigator 2.0.

Entender estos conceptos te ayudará a construir aplicaciones Flutter robustas y amigables para el usuario. Practica implementando diferentes escenarios de navegación en tus proyectos para solidificar tu comprensión.
