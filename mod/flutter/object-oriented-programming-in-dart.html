Introducción a la Programación Orientada a Objetos (POO)

La Programación Orientada a Objetos (POO) es un paradigma de programación basado en el concepto de "objetos", que pueden contener datos y código: datos en forma de campos (a menudo conocidos como atributos o propiedades), y código en forma de procedimientos (a menudo conocidos como métodos).

Conceptos Clave de la POO

  • Clases y Objetos
  • Herencia
  • Polimorfismo
  • Encapsulación
  • Abstracción

Clases y Objetos en Dart

Una clase es un plano para crear objetos. Un objeto es una instancia de una clase.

Definiendo una Clase

class Animal {
  String name;
  int age;

  // Constructor
  Animal(this.name, this.age);

  // Método
  void makeSound() {
    print('$name hace un sonido.');
  }
}

Creando un Objeto

void main() {
  Animal cat = Animal('Whiskers', 2);
  cat.makeSound(); // Salida: Whiskers hace un sonido.
}

Herencia

La herencia permite que una clase herede propiedades y métodos de otra clase.

Ejemplo de Herencia

class Animal {
  String name;
  int age;

  Animal(this.name, this.age);

  void makeSound() {
    print('$name hace un sonido.');
  }
}

class Dog extends Animal {
  String breed;

  Dog(String name, int age, this.breed) : super(name, age);

  void bark() {
    print('$name ladra.');
  }
}

void main() {
  Dog dog = Dog('Buddy', 3, 'Golden Retriever');
  dog.makeSound(); // Salida: Buddy hace un sonido.
  dog.bark();      // Salida: Buddy ladra.
}

Polimorfismo

El polimorfismo permite que los métodos hagan diferentes cosas según el objeto sobre el que actúan.

Ejemplo de Polimorfismo

class Animal {
  void makeSound() {
    print('Algún sonido genérico de animal');
  }
}

class Dog extends Animal {
  @override
  void makeSound() {
    print('Ladrido');
  }
}

class Cat extends Animal {
  @override
  void makeSound() {
    print('Maullido');
  }
}

void main() {
  Animal myDog = Dog();
  Animal myCat = Cat();

  myDog.makeSound(); // Salida: Ladrido
  myCat.makeSound(); // Salida: Maullido
}

Encapsulación

La encapsulación es el concepto de envolver datos y métodos que trabajan sobre los datos dentro de una unidad, por ejemplo, una clase en Dart.

Ejemplo de Encapsulación

class Person {
  String _name; // Propiedad privada
  int _age;    // Propiedad privada

  Person(this._name, this._age);

  // Getter para name
  String get name => _name;

  // Setter para name
  set name(String name) {
    _name = name;
  }

  // Getter para age
  int get age => _age;

  // Setter para age
  set age(int age) {
    if (age > 0) {
      _age = age;
    }
  }
}

void main() {
  Person person = Person('John', 30);
  print(person.name); // Salida: John
  person.age = 35;
  print(person.age);  // Salida: 35
}

Abstracción

La abstracción es el concepto de ocultar los detalles de implementación complejos y mostrar solo las características necesarias de un objeto.

Ejemplo de Abstracción

abstract class Shape {
  void draw(); // Método abstracto
}

class Circle extends Shape {
  @override
  void draw() {
    print('Dibujando un Círculo');
  }
}

class Square extends Shape {
  @override
  void draw() {
    print('Dibujando un Cuadrado');
  }
}

void main() {
  Shape circle = Circle();
  Shape square = Square();

  circle.draw(); // Salida: Dibujando un Círculo
  square.draw(); // Salida: Dibujando un Cuadrado
}

Conclusión

La Programación Orientada a Objetos en Dart es un paradigma poderoso que ayuda a estructurar el código de una manera más modular, reutilizable y mantenible. Al entender y aplicar los principios de clases y objetos, herencia, polimorfismo, encapsulación y abstracción, puedes crear aplicaciones robustas y escalables en Flutter.
