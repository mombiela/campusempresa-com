Introducción

Las declaraciones case en Bash son una herramienta poderosa para manejar múltiples condiciones. Te permiten ejecutar diferentes bloques de código basados en el valor de una variable, haciendo que tus scripts sean más legibles y mantenibles.

Conceptos Clave

  • Sintaxis: La estructura básica de una declaración case.
  • Patrones: Cómo hacer coincidir diferentes patrones.
  • Fall-through: Entender cómo las declaraciones case manejan múltiples coincidencias.
  • Caso por Defecto: Usar el caso por defecto para manejar patrones no coincidentes.

Sintaxis

La sintaxis básica de una declaración case en Bash es la siguiente:

case expresión in
    patrón1)
        # comandos
        ;;
    patrón2)
        # comandos
        ;;
    *)
        # comandos por defecto
        ;;
esac
  • expresión: La variable o valor que estás probando.
  • patrón1, patrón2, etc.: Patrones para hacer coincidir con la expresión.
  • ;;: Termina cada bloque de comandos.
  • *): El caso por defecto, ejecutado si no hay coincidencias con los patrones.
  • esac: Termina la declaración case.

Ejemplos

Ejemplo Básico

Aquí hay un ejemplo simple para ilustrar cómo funcionan las declaraciones case:

#!/bin/bash

echo "Introduce un número entre 1 y 3:"
read number

case $number in
    1)
        echo "Has introducido uno."
        ;;
    2)
        echo "Has introducido dos."
        ;;
    3)
        echo "Has introducido tres."
        ;;
    *)
        echo "Número inválido."
        ;;
esac

Usando Patrones

Puedes usar comodines y rangos en los patrones:

#!/bin/bash

echo "Introduce un carácter:"
read char

case $char in
    [a-z])
        echo "Has introducido una letra minúscula."
        ;;
    [A-Z])
        echo "Has introducido una letra mayúscula."
        ;;
    [0-9])
        echo "Has introducido un dígito."
        ;;
    *)
        echo "Has introducido un carácter especial."
        ;;
esac

Múltiples Patrones

Puedes hacer coincidir múltiples patrones con un solo bloque de comandos:

#!/bin/bash

echo "Introduce un día de la semana:"
read day

case $day in
    "Lunes"|"Martes"|"Miércoles"|"Jueves"|"Viernes")
        echo "Es un día laborable."
        ;;
    "Sábado"|"Domingo")
        echo "Es el fin de semana."
        ;;
    *)
        echo "Día inválido."
        ;;
esac

Uso Avanzado

Declaraciones Case Anidadas

Puedes anidar declaraciones case dentro de otras para una lógica más compleja:

#!/bin/bash

echo "Introduce una fruta (manzana, plátano, cereza):"
read fruit

case $fruit in
    "manzana")
        echo "Has elegido manzana."
        ;;
    "plátano")
        echo "Has elegido plátano."
        ;;
    "cereza")
        echo "Has elegido cereza."
        ;;
    *)
        echo "Fruta desconocida."
        ;;
esac

Usando Variables en Patrones

Puedes usar variables dentro de los patrones para hacer tus declaraciones case más dinámicas:

#!/bin/bash

echo "Introduce una extensión de archivo (por ejemplo, .txt, .sh):"
read extension

case $extension in
    *.txt)
        echo "Este es un archivo de texto."
        ;;
    *.sh)
        echo "Este es un script de shell."
        ;;
    *)
        echo "Tipo de archivo desconocido."
        ;;
esac

Conclusión

Las declaraciones case en Bash proporcionan una forma flexible y legible de manejar múltiples condiciones. Al entender la sintaxis, usar patrones efectivamente y aprovechar características avanzadas como declaraciones case anidadas y patrones de variables, puedes escribir scripts más eficientes y mantenibles. Practica con los ejemplos proporcionados e intenta crear tus propias declaraciones case para solidificar tu comprensión.
