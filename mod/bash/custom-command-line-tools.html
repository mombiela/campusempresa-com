Introducción

Crear herramientas de línea de comandos personalizadas puede mejorar significativamente tu productividad y agilizar tareas repetitivas. En esta sección, exploraremos cómo crear, gestionar y utilizar herramientas de línea de comandos personalizadas usando Bash.

Conceptos Clave

  • Shebang (#!): La primera línea en un script que especifica el intérprete.
  • Permisos de Script: Hacer que los scripts sean ejecutables.
  • Argumentos de Línea de Comandos: Pasar argumentos a los scripts.
  • Variables de Entorno: Usar y gestionar variables de entorno.
  • Funciones: Crear bloques de código reutilizables dentro de los scripts.
  • Manejo de Errores: Gestionar errores y estados de salida.

Creando un Script Simple

Ejemplo: Script de Hola Mundo

#!/bin/bash
# Este es un simple script de Hola Mundo
echo "¡Hola, Mundo!"
  • Explicación:
    • #!/bin/bash: Línea Shebang que especifica que el script debe ejecutarse con Bash.
    • echo "¡Hola, Mundo!": Imprime "¡Hola, Mundo!" en la terminal.

Haciendo el Script Ejecutable

chmod +x hola_mundo.sh
  • Explicación:
    • chmod +x hola_mundo.sh: Cambia los permisos del archivo para hacer el script ejecutable.

Argumentos de Línea de Comandos

Ejemplo: Script de Saludo con Argumentos

#!/bin/bash
# Este script saluda al usuario con su nombre
echo "¡Hola, $1!"
  • Explicación:
    • $1: Representa el primer argumento pasado al script.

Ejecutando el Script con Argumentos

./saludo.sh Alicia
  • Salida:
    • ¡Hola, Alicia!

Variables de Entorno

Ejemplo: Usando Variables de Entorno

#!/bin/bash
# Este script demuestra el uso de variables de entorno
echo "El usuario actual es $USER"
echo "El directorio home es $HOME"
  • Explicación:
    • $USER: Variable de entorno para el usuario actual.
    • $HOME: Variable de entorno para el directorio home.

Funciones en Scripts

Ejemplo: Función para Sumar Dos Números

#!/bin/bash
# Función para sumar dos números
sumar_numeros() {
  local suma=$(( $1 + $2 ))
  echo "La suma es: $suma"
}

# Llamando a la función
sumar_numeros 3 5
  • Explicación:
    • sumar_numeros() { ... }: Define una función llamada sumar_numeros.
    • local suma=$(( $1 + $2 )): Calcula la suma de dos argumentos.
    • echo "La suma es: $suma": Imprime la suma.

Manejo de Errores

Ejemplo: Comprobando la Existencia de un Archivo

#!/bin/bash
# Este script comprueba si un archivo existe
if [ -f "$1" ]; then
  echo "El archivo $1 existe."
else
  echo "El archivo $1 no existe."
  exit 1
fi
  • Explicación:
    • [ -f "$1" ]: Comprueba si el archivo especificado por el primer argumento existe.
    • exit 1: Sale del script con un código de estado de 1 si el archivo no existe.

Conclusión

Crear herramientas de línea de comandos personalizadas en Bash puede mejorar enormemente tu flujo de trabajo al automatizar tareas repetitivas y proporcionar soluciones rápidas a problemas comunes. Al entender los conceptos básicos de scripting, manejar argumentos de línea de comandos, usar variables de entorno, definir funciones e implementar manejo de errores, puedes construir herramientas poderosas y eficientes adaptadas a tus necesidades. Practica estos conceptos con varios ejemplos para volverte competente en la creación de tus propias herramientas de línea de comandos personalizadas.
