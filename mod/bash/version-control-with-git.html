Introducción al Control de Versiones

El control de versiones es un sistema que registra los cambios en un archivo o conjunto de archivos a lo largo del tiempo para que puedas recordar versiones específicas más tarde. Git es un sistema de control de versiones distribuido que permite a múltiples desarrolladores trabajar en un proyecto simultáneamente sin sobrescribir los cambios de los demás.

Configuración de Git

Antes de poder usar Git, necesitas instalarlo y configurar tu información de usuario.

Instalando Git

  • En Linux: Usa tu gestor de paquetes (por ejemplo, sudo apt-get install git para sistemas basados en Debian).
  • En macOS: Usa Homebrew (brew install git).
  • En Windows: Descarga el instalador desde el sitio web oficial de Git.

Configurando Git

Después de la instalación, configura tu nombre de usuario y correo electrónico de Git:

git config --global user.name "Tu Nombre"
git config --global user.email "tu.email@ejemplo.com"

Comandos Básicos de Git

Aquí hay algunos comandos fundamentales de Git para comenzar:

Inicializando un Repositorio

Para iniciar un nuevo repositorio de Git, navega a tu directorio de proyecto y ejecuta:

git init

Clonando un Repositorio

Para clonar un repositorio existente:

git clone https://github.com/username/repository.git

Verificando el Estado del Repositorio

Para ver el estado de tus archivos:

git status

Añadiendo Cambios

Para añadir cambios al área de preparación:

git add filename
# o para añadir todos los cambios
git add .

Confirmando Cambios

Para confirmar tus cambios con un mensaje:

git commit -m "Tu mensaje de confirmación"

Viendo el Historial de Confirmaciones

Para ver el historial de confirmaciones:

git log

Ramas y Fusiones

Las ramas te permiten trabajar en diferentes partes de un proyecto simultáneamente.

Creando una Rama

Para crear una nueva rama:

git branch nombre_rama

Cambiando de Rama

Para cambiar a una rama diferente:

git checkout nombre_rama

Fusionando Ramas

Para fusionar una rama en tu rama actual:

git merge nombre_rama

Repositorios Remotos

Los repositorios remotos son versiones de tu proyecto que están alojadas en internet o en una red.

Añadiendo un Remoto

Para añadir un repositorio remoto:

git remote add origin https://github.com/username/repository.git

Enviando Cambios

Para enviar tus cambios al repositorio remoto:

git push origin nombre_rama

Extrayendo Cambios

Para extraer cambios del repositorio remoto:

git pull origin nombre_rama

Manejo de Conflictos

Los conflictos ocurren cuando los cambios en diferentes ramas se contradicen entre sí.

Resolviendo Conflictos

  1. Abre los archivos en conflicto y resuelve los conflictos manualmente.
  2. Después de resolver, añade los archivos resueltos:
git add filename
  1. Confirma los cambios:
git commit -m "Conflicto de fusión resuelto"

Comandos Avanzados de Git

Para un uso más avanzado, aquí hay algunos comandos adicionales:

Guardando Cambios Temporalmente

Para guardar cambios temporalmente sin confirmar:

git stash

Para aplicar los cambios guardados:

git stash apply

Rebase

Para rebasear tu rama actual sobre otra rama:

git rebase nombre_rama

Resumen

En esta sección, cubrimos los conceptos básicos del uso de Git para el control de versiones, incluyendo la configuración de Git, comandos básicos, ramas y fusiones, trabajo con repositorios remotos, manejo de conflictos y algunos comandos avanzados. Entender estos conceptos te ayudará a gestionar tu código de manera más efectiva y colaborar con otros sin problemas.
