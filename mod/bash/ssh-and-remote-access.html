Introducción a SSH

Secure Shell (SSH) es un protocolo utilizado para conectarse de manera segura a sistemas remotos a través de una red. Proporciona un canal seguro sobre una red no segura mediante el uso de cifrado.

  • ¿Qué es SSH?

    • SSH significa Secure Shell.
    • Se utiliza para el inicio de sesión remoto seguro y otros servicios de red seguros sobre una red insegura.
    • SSH utiliza criptografía de clave pública para autenticar la computadora remota y permitir que esta autentique al usuario.

  • ¿Por qué usar SSH?

    • Comunicación segura: Cifra los datos transferidos entre el cliente y el servidor.
    • Autenticación: Utiliza criptografía de clave pública para una autenticación segura.
    • Reenvío de puertos: Puede tunelizar otros protocolos a través de SSH.

Configuración de SSH

Para usar SSH, necesitas tanto un cliente como un servidor. El cliente inicia la conexión y el servidor responde.

Instalación de SSH

  • En Ubuntu/Debian: 
    sudo apt-get update
sudo apt-get install openssh-server
  • En CentOS/RHEL: 
    sudo yum install openssh-server
sudo systemctl start sshd
sudo systemctl enable sshd

Iniciar y habilitar el servicio SSH

  • En Ubuntu/Debian: 
    sudo systemctl start ssh
sudo systemctl enable ssh
  • En CentOS/RHEL: 
    sudo systemctl start sshd
sudo systemctl enable sshd

Conectarse a un servidor remoto

Para conectarse a un servidor remoto usando SSH, necesitas la dirección IP o el nombre de host del servidor y una cuenta de usuario válida en el servidor.

Comando básico de SSH

ssh username@hostname
  • username: La cuenta de usuario en el servidor remoto.
  • hostname: La dirección IP o el nombre de dominio del servidor remoto.

Ejemplo

ssh john@192.168.1.10

Este comando se conectará al servidor en 192.168.1.10 usando el usuario john.

Autenticación basada en claves SSH

La autenticación basada en claves SSH es más segura que la autenticación basada en contraseñas. Utiliza un par de claves criptográficas: una clave privada y una clave pública.

Generar claves SSH

ssh-keygen -t rsa -b 4096 -C "tu_email@ejemplo.com"
  • -t rsa: Especifica el tipo de clave a crear (RSA).
  • -b 4096: Especifica el número de bits en la clave (4096 bits).
  • -C "tu_email@ejemplo.com": Añade una etiqueta a la clave.

Copiar la clave pública al servidor

ssh-copy-id username@hostname
  • username: La cuenta de usuario en el servidor remoto.
  • hostname: La dirección IP o el nombre de dominio del servidor remoto.

Ejemplo

ssh-copy-id john@192.168.1.10

Este comando copiará tu clave pública al servidor en 192.168.1.10 para el usuario john.

Gestión de configuraciones de SSH

Puedes gestionar múltiples conexiones SSH usando el archivo de configuración de SSH ubicado en ~/.ssh/config.

Ejemplo de configuración

Host miservidor
    HostName 192.168.1.10
    User john
    IdentityFile ~/.ssh/id_rsa
  • Host: Un apodo para tu servidor.
  • HostName: La dirección IP o el nombre de dominio del servidor.
  • User: La cuenta de usuario en el servidor.
  • IdentityFile: La ruta a tu clave privada.

Conectarse usando la configuración

ssh miservidor

Este comando usará la configuración definida para miservidor.

Reenvío de puertos con SSH

SSH puede reenviar puertos desde la máquina local a la máquina remota, o viceversa.

Reenvío de puertos locales

ssh -L puerto_local:direccion_remota:puerto_remoto username@hostname
  • puerto_local: El puerto local a reenviar.
  • direccion_remota: La dirección de la máquina remota.
  • puerto_remoto: El puerto en la máquina remota.

Ejemplo

ssh -L 8080:localhost:80 john@192.168.1.10

Este comando reenvía el puerto local 8080 al puerto 80 en 192.168.1.10.

Conclusión

SSH es una herramienta poderosa para el acceso remoto seguro y la gestión de servidores. Al comprender cómo configurar y usar SSH, puedes conectarte y gestionar sistemas remotos de manera segura. La autenticación basada en claves y las configuraciones de SSH pueden simplificar y asegurar tus conexiones, mientras que el reenvío de puertos puede extender la funcionalidad de SSH a otros servicios.
