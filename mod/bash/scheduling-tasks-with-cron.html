Introducción a cron

cron es un programador de tareas basado en el tiempo en sistemas operativos tipo Unix. Los usuarios pueden programar trabajos (comandos o scripts) para que se ejecuten periódicamente en momentos, fechas o intervalos fijos. Esto es particularmente útil para automatizar tareas repetitivas como copias de seguridad, mantenimiento del sistema y otras tareas rutinarias.

Entendiendo la Sintaxis de cron

Un trabajo cron se define usando una sintaxis específica en un archivo llamado crontab (tabla cron). Cada línea en un archivo crontab representa una tarea programada y sigue este formato:

* * * * * command_to_execute
- - - - -
| | | | |
| | | | +---- Día de la semana (0 - 7) (domingo es tanto 0 como 7)
| | | +------ Mes (1 - 12)
| | +-------- Día del mes (1 - 31)
| +---------- Hora (0 - 23)
+------------ Minuto (0 - 59)

Ejemplo

Para ejecutar un script todos los días a las 2:30 AM, escribirías:

30 2 * * * /path/to/script.sh

Gestión de Archivos crontab

Ver el crontab Actual

Para ver el crontab del usuario actual, usa:

crontab -l

Editar el crontab

Para editar el crontab, usa:

crontab -e

Este comando abre el archivo crontab en el editor de texto predeterminado. Aquí puedes agregar, modificar o eliminar trabajos cron.

Eliminar el crontab

Para eliminar el crontab del usuario actual, usa:

crontab -r

Patrones Comunes de Programación

Aquí hay algunos patrones comunes de programación que podrías usar:

| Programación | Sintaxis | Descripción | |----------------------------------|---------------------|-------------------------------------------| | Cada minuto | * * * * * | Se ejecuta cada minuto | | Cada hora | 0 * * * * | Se ejecuta al inicio de cada hora | | Todos los días a medianoche | 0 0 * * * | Se ejecuta todos los días a medianoche | | Todos los domingos a medianoche | 0 0 * * 0 | Se ejecuta todos los domingos a medianoche| | Primer día de cada mes a las 3 AM| 0 3 1 * * | Se ejecuta a las 3 AM el primer día del mes|

Uso Avanzado de cron

Redirigir Salida

Para capturar la salida de un trabajo cron, puedes redirigirla a un archivo:

30 2 * * * /path/to/script.sh >> /path/to/logfile.log 2>&1

Este ejemplo agrega tanto la salida estándar como el error estándar a logfile.log.

Usando Variables de Entorno

Puedes establecer variables de entorno en el archivo crontab:

SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin

30 2 * * * /path/to/script.sh

Establecer SHELL y PATH asegura que el trabajo cron se ejecute en el entorno correcto.

Cadenas Especiales

cron también soporta cadenas especiales para reemplazar los cinco campos de tiempo y fecha:

  • @reboot : Ejecutar una vez, al inicio.
  • @yearly o @annually : Ejecutar una vez al año, "0 0 1 1 *".
  • @monthly : Ejecutar una vez al mes, "0 0 1 * *".
  • @weekly : Ejecutar una vez a la semana, "0 0 * * 0".
  • @daily o @midnight : Ejecutar una vez al día, "0 0 * * *".
  • @hourly : Ejecutar una vez por hora, "0 * * * *".

Ejemplo:

@daily /path/to/daily_script.sh

Depuración de Trabajos cron

Verificación de Logs de cron

Los logs de cron se pueden encontrar en /var/log/syslog o /var/log/cron dependiendo de la configuración de tu sistema. Puedes usar grep para filtrar entradas relacionadas con cron:

grep cron /var/log/syslog

Problemas Comunes

  • Permisos: Asegúrate de que el script tenga permisos de ejecución.
  • Variables de Entorno: El entorno de cron es diferente de tu entorno de shell. Establece explícitamente las variables de entorno necesarias.
  • Problemas de Ruta: Usa rutas absolutas para comandos y scripts.

Conclusión

Programar tareas con cron es una manera poderosa de automatizar tareas repetitivas en sistemas tipo Unix. Entender la sintaxis y cómo gestionar archivos crontab te permite programar trabajos de manera eficiente. Con características avanzadas como la redirección de salida, variables de entorno y cadenas especiales, cron proporciona flexibilidad y control sobre la programación de tareas. Siempre recuerda revisar los logs y permisos para solucionar cualquier problema con tus trabajos cron.
