Introducción

Los Futuros y las Promesas son constructos poderosos en Scala que te permiten trabajar con cálculos concurrentes y asíncronos. Son esenciales para escribir código no bloqueante y manejar tareas que pueden completarse en algún momento en el futuro.

Futuros

Un Futuro representa un valor que puede no estar disponible aún. Es un marcador de posición para un resultado que eventualmente será calculado.

Creando Futuros

Puedes crear un Futuro usando el objeto compañero Future y un ExecutionContext implícito.

import scala.concurrent._
import ExecutionContext.Implicits.global

val future: Future[Int] = Future {
  // Simular un cálculo de larga duración
  Thread.sleep(1000)
  42
}

Trabajando con Futuros

Los Futuros proporcionan varios métodos para trabajar con sus resultados una vez que están disponibles.

onComplete

El método onComplete te permite especificar una función de retorno que se ejecutará cuando el futuro se complete.

future.onComplete {
  case Success(value) => println(s"El resultado es $value")
  case Failure(exception) => println(s"Ocurrió un error: ${exception.getMessage}")
}

map y flatMap

Puedes transformar el resultado de un futuro usando map y flatMap.

val transformedFuture: Future[String] = future.map(value => s"La respuesta es $value")

for-comprehensions

Las for-comprehensions proporcionan una forma más legible de encadenar múltiples futuros.

val future1 = Future { 10 }
val future2 = Future { 20 }

val combinedFuture: Future[Int] = for {
  value1 <- future1
  value2 <- future2
} yield value1 + value2

Promesas

Una Promesa es un contenedor de asignación única y escribible que completa un Futuro. Puedes pensar en una Promesa como una forma de controlar manualmente la finalización de un Futuro.

Creando Promesas

Puedes crear una Promesa usando el objeto compañero Promise.

val promise: Promise[Int] = Promise[Int]()
val futureFromPromise: Future[Int] = promise.future

Completando Promesas

Puedes completar una Promesa llamando a success o failure.

promise.success(42)
// o
promise.failure(new Exception("Algo salió mal"))

Ejemplo: Usando Promesas para Completar Futuros

Aquí hay un ejemplo que demuestra cómo usar una Promesa para completar un Futuro.

import scala.concurrent._
import ExecutionContext.Implicits.global

val promise = Promise[Int]()
val future = promise.future

future.onComplete {
  case Success(value) => println(s"El resultado es $value")
  case Failure(exception) => println(s"Ocurrió un error: ${exception.getMessage}")
}

// Simular algún cálculo asíncrono
Future {
  Thread.sleep(1000)
  promise.success(42)
}

Comparación: Futuros vs Promesas

| Característica | Futuro | Promesa | |------------------|---------------------------------|---------------------------------| | Creación | Creado con objeto Future | Creado con objeto Promise | | Finalización | Completado por el cálculo | Completado manualmente por el usuario | | Legibilidad | Solo lectura | Lectura-escritura | | Caso de Uso | Resultado de cálculo asíncrono | Control manual sobre la finalización |

Conclusión

Los Futuros y las Promesas son herramientas esenciales para manejar la programación asíncrona y concurrente en Scala. Los Futuros te permiten trabajar con valores que estarán disponibles en el futuro, mientras que las Promesas te dan control sobre cuándo y cómo se proporcionan esos valores. Al entender y usar estos constructos, puedes escribir código más eficiente y no bloqueante.
