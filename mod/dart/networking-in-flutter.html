Redes en Flutter
Pruebas Unitarias ►

Las redes son un aspecto crucial de las aplicaciones móviles modernas, permitiéndoles obtener datos de internet, comunicarse con servidores e interactuar con varios servicios web. En esta sección, exploraremos cómo manejar las redes en Flutter usando Dart.

Introducción a las Redes en Flutter

  • Solicitudes HTTP: Aprende cómo hacer solicitudes HTTP para obtener datos de internet.
  • Análisis de JSON: Entiende cómo analizar datos JSON recibidos de servicios web.
  • Manejo de Errores: Implementa el manejo de errores para solicitudes de red.
  • Programación Asíncrona: Utiliza async y await de Dart para manejar operaciones asíncronas.

Haciendo Solicitudes HTTP

Para hacer solicitudes HTTP en Flutter, usamos el paquete http. Primero, agrega el paquete http a tu archivo pubspec.yaml:

dependencies:
  http: ^0.13.3

Solicitud GET

Una solicitud GET se usa para obtener datos de un servidor. Aquí hay un ejemplo de cómo hacer una solicitud GET:

import 'package:http/http.dart' as http;
import 'dart:convert';

void fetchData() async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

  if (response.statusCode == 200) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to load data');
  }
}
  • Explicación:
    • http.get envía una solicitud GET a la URL especificada.
    • response.statusCode verifica el estado de la respuesta.
    • jsonDecode analiza la respuesta JSON en un objeto Dart.

Solicitud POST

Una solicitud POST se usa para enviar datos a un servidor. Aquí hay un ejemplo:

import 'package:http/http.dart' as http;
import 'dart:convert';

void sendData() async {
  final response = await http.post(
    Uri.parse('https://jsonplaceholder.typicode.com/posts'),
    headers: <String, String>{
      'Content-Type': 'application/json; charset=UTF-8',
    },
    body: jsonEncode(<String, String>{
      'title': 'foo',
      'body': 'bar',
      'userId': '1',
    }),
  );

  if (response.statusCode == 201) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to send data');
  }
}
  • Explicación:
    • http.post envía una solicitud POST a la URL especificada.
    • headers especifica el tipo de contenido.
    • jsonEncode convierte el objeto Dart en una cadena JSON.

Análisis de JSON

Al trabajar con APIs, los datos a menudo se devuelven en formato JSON. Aquí se muestra cómo analizar datos JSON:

Ejemplo de Datos JSON

{
  "userId": 1,
  "id": 1,
  "title": "Sample Title",
  "body": "This is a sample body."
}

Analizando JSON en Dart

import 'dart:convert';

class Post {
  final int userId;
  final int id;
  final String title;
  final String body;

  Post({required this.userId, required this.id, required this.title, required this.body});

  factory Post.fromJson(Map<String, dynamic> json) {
    return Post(
      userId: json['userId'],
      id: json['id'],
      title: json['title'],
      body: json['body'],
    );
  }
}

void parseJson() {
  String jsonString = '{"userId": 1, "id": 1, "title": "Sample Title", "body": "This is a sample body."}';
  Map<String, dynamic> jsonMap = jsonDecode(jsonString);

  Post post = Post.fromJson(jsonMap);
  print('Title: ${post.title}');
}
  • Explicación:
    • La clase Post representa la estructura JSON.
    • Post.fromJson es un constructor de fábrica que crea una instancia de Post a partir de un mapa JSON.
    • jsonDecode convierte la cadena JSON en un mapa Dart.

Manejo de Errores

Manejar errores es esencial para redes robustas. Aquí se muestra cómo manejar errores en solicitudes HTTP:

void fetchDataWithErrorHandling() async {
  try {
    final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

    if (response.statusCode == 200) {
      var data = jsonDecode(response.body);
      print(data);
    } else {
      print('Server error: ${response.statusCode}');
    }
  } catch (e) {
    print('Network error: $e');
  }
}
  • Explicación:
    • El bloque try-catch se usa para manejar excepciones.
    • response.statusCode se verifica para manejar errores del servidor.

Programación Asíncrona

Dart usa async y await para manejar operaciones asíncronas. Aquí hay una breve descripción:

  • async: Marca una función como asíncrona.
  • await: Espera a que una operación asíncrona se complete.

Ejemplo

Future<void> fetchData() async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

  if (response.statusCode == 200) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to load data');
  }
}
  • Explicación:
    • Future<void> indica que la función devuelve un Future.
    • await se usa para esperar a que la solicitud HTTP se complete.

Conclusión

En esta sección, cubrimos los conceptos básicos de las redes en Flutter usando Dart. Aprendimos cómo hacer solicitudes GET y POST, analizar datos JSON, manejar errores y usar programación asíncrona. Estas habilidades son esenciales para construir aplicaciones móviles modernas y basadas en datos.

Al dominar estos conceptos, podrás crear aplicaciones Flutter que puedan interactuar con servicios web y proporcionar contenido dinámico a los usuarios.
