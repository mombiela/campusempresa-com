Herencia en Dart
Mixins ►

La herencia es un concepto fundamental en la programación orientada a objetos que permite a una clase heredar propiedades y métodos de otra clase. Esto ayuda en la reutilización del código y en el establecimiento de una jerarquía natural entre las clases.

Conceptos Clave

  • Superclase (Clase Padre): La clase cuyas propiedades y métodos son heredados.
  • Subclase (Clase Hija): La clase que hereda de otra clase.
  • Palabra clave extends: Se utiliza para indicar que una clase está heredando de otra clase.
  • Sobrescritura de Métodos: La capacidad de una subclase para proporcionar una implementación específica de un método que ya está definido en su superclase.
  • Palabra clave super: Se utiliza para referirse a la superclase y acceder a sus propiedades y métodos.

Herencia Básica

En Dart, puedes crear una subclase usando la palabra clave extends. Aquí hay un ejemplo simple:

class Animal {
  void makeSound() {
    print('Animal hace un sonido');
  }
}

class Dog extends Animal {
  void bark() {
    print('El perro ladra');
  }
}

void main() {
  Dog dog = Dog();
  dog.makeSound(); // Método heredado
  dog.bark(); // Método de la subclase
}

Explicación

  • Clase Animal: Esta es la superclase con un método makeSound.
  • Clase Dog: Esta es la subclase que extiende Animal y agrega un nuevo método bark.
  • Función main: Crea una instancia de Dog y llama tanto al método heredado makeSound como al método bark.

Sobrescritura de Métodos

Una subclase puede sobrescribir un método de su superclase para proporcionar una implementación específica. Usa la anotación @override para indicar que estás sobrescribiendo un método.

class Animal {
  void makeSound() {
    print('Animal hace un sonido');
  }
}

class Dog extends Animal {
  @override
  void makeSound() {
    print('El perro ladra');
  }
}

void main() {
  Dog dog = Dog();
  dog.makeSound(); // Salida: El perro ladra
}

Explicación

  • Anotación @override: Indica que el método makeSound en Dog está sobrescribiendo el método en Animal.
  • Salida: Cuando se llama a makeSound en un objeto Dog, se ejecuta el método sobrescrito en Dog.

Uso de la Palabra Clave `super`

La palabra clave super se usa para referirse a la superclase y puede usarse para llamar a sus métodos y constructores.

class Animal {
  void makeSound() {
    print('Animal hace un sonido');
  }
}

class Dog extends Animal {
  @override
  void makeSound() {
    super.makeSound(); // Llama al método de la superclase
    print('El perro ladra');
  }
}

void main() {
  Dog dog = Dog();
  dog.makeSound(); 
  // Salida:
  // Animal hace un sonido
  // El perro ladra
}

Explicación

  • super.makeSound(): Llama al método makeSound de la superclase Animal antes de ejecutar el método makeSound en Dog.

Constructores y Herencia

Cuando se instancia una subclase, el constructor de la superclase se llama primero. Puedes llamar explícitamente a un constructor de la superclase usando la palabra clave super.

class Animal {
  String name;
  
  Animal(this.name);
  
  void display() {
    print('Animal: $name');
  }
}

class Dog extends Animal {
  String breed;
  
  Dog(String name, this.breed) : super(name);
  
  @override
  void display() {
    super.display();
    print('Raza: $breed');
  }
}

void main() {
  Dog dog = Dog('Buddy', 'Golden Retriever');
  dog.display();
  // Salida:
  // Animal: Buddy
  // Raza: Golden Retriever
}

Explicación

  • Constructor de Animal: Toma un parámetro name.
  • Constructor de Dog: Toma parámetros name y breed y llama al constructor de la superclase con super(name).
  • Método display: Sobrescrito en Dog para incluir información adicional sobre la raza.

Resumen

La herencia en Dart te permite crear una jerarquía de clases donde las subclases heredan propiedades y métodos de sus superclases. Los conceptos clave incluyen la palabra clave extends para definir la herencia, la sobrescritura de métodos y la palabra clave super para acceder a los métodos y constructores de la superclase. Entender estos conceptos es crucial para escribir código reutilizable y mantenible en Dart.
