Gestión de Estado en Dart
Redes en Flutter ►

La gestión de estado es un aspecto crucial para construir aplicaciones responsivas y dinámicas. En Dart, especialmente cuando se trabaja con Flutter, gestionar el estado de tu aplicación de manera eficiente puede impactar significativamente en el rendimiento y la experiencia del usuario. Este tema te guiará desde los conceptos básicos hasta los avanzados de la gestión de estado en Dart.

Introducción a la Gestión de Estado

La gestión de estado se refiere al manejo del estado de una aplicación, que incluye los datos y el estado de la interfaz de usuario. Entender cómo gestionar el estado de manera efectiva es esencial para construir aplicaciones escalables y mantenibles.

Conceptos Clave

  • Estado: Los datos que tu aplicación necesita rastrear.
  • Widget con Estado: Un widget que mantiene un estado que puede cambiar durante la vida útil del widget.
  • Widget sin Estado: Un widget que no mantiene ningún estado.

Widgets con Estado y sin Estado

Widgets sin Estado

Los widgets sin estado son inmutables, lo que significa que sus propiedades no pueden cambiar una vez que se crean. Son ideales para contenido estático.

import 'package:flutter/material.dart';

class MyStatelessWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Ejemplo de Widget sin Estado'),
      ),
      body: Center(
        child: Text('Este es un widget sin estado'),
      ),
    );
  }
}

Widgets con Estado

Los widgets con estado mantienen un estado que puede cambiar con el tiempo. Son útiles para contenido dinámico.

import 'package:flutter/material.dart';

class MyStatefulWidget extends StatefulWidget {
  @override
  _MyStatefulWidgetState createState() => _MyStatefulWidgetState();
}

class _MyStatefulWidgetState extends State<MyStatefulWidget> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Ejemplo de Widget con Estado'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text('Has presionado el botón esta cantidad de veces:'),
            Text('$_counter', style: Theme.of(context).textTheme.headline4),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _incrementCounter,
        tooltip: 'Incrementar',
        child: Icon(Icons.add),
      ),
    );
  }
}

Técnicas de Gestión de Estado

setState

La forma más simple de gestionar el estado en Flutter es usando setState. Este método se usa dentro de un widget con estado para actualizar la interfaz de usuario.

void _incrementCounter() {
  setState(() {
    _counter++;
  });
}

InheritedWidget

InheritedWidget es una clase base para widgets que propagan información de manera eficiente hacia abajo en el árbol. Es útil para compartir estado entre muchos widgets.

import 'package:flutter/material.dart';

class MyInheritedWidget extends InheritedWidget {
  final int counter;
  final Widget child;

  MyInheritedWidget({this.counter, this.child}) : super(child: child);

  @override
  bool updateShouldNotify(MyInheritedWidget oldWidget) {
    return oldWidget.counter != counter;
  }

  static MyInheritedWidget of(BuildContext context) {
    return context.dependOnInheritedWidgetOfExactType<MyInheritedWidget>();
  }
}

Provider

Provider es un envoltorio alrededor de InheritedWidget para hacerlo más fácil de usar y más reutilizable. Es una de las soluciones de gestión de estado más populares en Flutter.

import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

void main() {
  runApp(
    ChangeNotifierProvider(
      create: (context) => Counter(),
      child: MyApp(),
    ),
  );
}

class Counter with ChangeNotifier {
  int _count = 0;

  int get count => _count;

  void increment() {
    _count++;
    notifyListeners();
  }
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Ejemplo de Provider'),
        ),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: <Widget>[
              Text('Has presionado el botón esta cantidad de veces:'),
              Consumer<Counter>(
                builder: (context, counter, child) {
                  return Text('${counter.count}', style: Theme.of(context).textTheme.headline4);
                },
              ),
            ],
          ),
        ),
        floatingActionButton: FloatingActionButton(
          onPressed: () {
            Provider.of<Counter>(context, listen: false).increment();
          },
          tooltip: 'Incrementar',
          child: Icon(Icons.add),
        ),
      ),
    );
  }
}

Gestión de Estado Avanzada

Bloc (Componente de Lógica de Negocio)

Bloc es un patrón que ayuda a separar la lógica de negocio de la interfaz de usuario. Utiliza streams para gestionar el estado.

import 'package:flutter_bloc/flutter_bloc.dart';

enum CounterEvent { increment }

class CounterBloc extends Bloc<CounterEvent, int> {
  CounterBloc() : super(0);

  @override
  Stream<int> mapEventToState(CounterEvent event) async* {
    switch (event) {
      case CounterEvent.increment:
        yield state + 1;
        break;
    }
  }
}

Riverpod

Riverpod es una solución de gestión de estado similar a Provider que ofrece más flexibilidad y robustez.

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

final counterProvider = StateProvider((ref) => 0);

void main() {
  runApp(ProviderScope(child: MyApp()));
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Ejemplo de Riverpod'),
        ),
        body: Center(
          child: Consumer(
            builder: (context, watch, child) {
              final count = watch(counterProvider).state;
              return Text('$count', style: Theme.of(context).textTheme.headline4);
            },
          ),
        ),
        floatingActionButton: FloatingActionButton(
          onPressed: () {
            context.read(counterProvider).state++;
          },
          tooltip: 'Incrementar',
          child: Icon(Icons.add),
        ),
      ),
    );
  }
}

Conclusión

La gestión de estado es un concepto fundamental en el desarrollo con Dart y Flutter. Desde el simple setState hasta patrones avanzados como Bloc y Riverpod, entender estas técnicas te ayudará a construir aplicaciones eficientes y mantenibles. Practica implementando estos patrones en tus proyectos para obtener una comprensión más profunda y elegir la mejor solución para tus necesidades.
