Introducción

En los sistemas operativos, la sincronización es crucial para gestionar la ejecución concurrente de procesos y threads. Los algoritmos de sincronización aseguran que los recursos compartidos se utilicen de manera segura y eficiente, evitando condiciones de carrera y otros problemas de concurrencia.

Conceptos Clave

  • Condición de carrera: Ocurre cuando dos o más procesos acceden y manipulan datos compartidos de manera concurrente, y el resultado depende del orden de ejecución.
  • Sección crítica: Parte del código donde se accede a recursos compartidos y que debe ejecutarse de manera atómica para evitar inconsistencias.
  • Mutex (Mutual Exclusion): Mecanismo que asegura que solo un proceso o thread pueda acceder a una sección crítica a la vez.
  • Semáforo: Variable o abstracto de programación que se utiliza para controlar el acceso a un recurso común por múltiples procesos en un entorno concurrente.
  • Monitores: Estructura de sincronización que permite que solo un thread acceda a una sección crítica a la vez, encapsulando tanto la mutual exclusion como la condición de sincronización.

Algoritmos de Sincronización

Mutex

Un mutex es una herramienta básica para la sincronización que asegura la exclusión mutua. Solo un thread puede poseer el mutex a la vez, y otros threads deben esperar hasta que el mutex sea liberado.

#include <pthread.h>
#include <stdio.h>

pthread_mutex_t lock;

void *thread_function(void *arg) {
    pthread_mutex_lock(&lock);
    // Sección crítica
    printf("Thread %d está en la sección crítica\n", *(int *)arg);
    pthread_mutex_unlock(&lock);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    pthread_mutex_init(&lock, NULL);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    pthread_mutex_destroy(&lock);
    return 0;
}

Semáforos

Los semáforos son más flexibles que los mutexes y pueden ser utilizados para sincronizar múltiples threads y procesos. Existen dos tipos de semáforos: binarios y contadores.

#include <pthread.h>
#include <semaphore.h>
#include <stdio.h>

sem_t semaphore;

void *thread_function(void *arg) {
    sem_wait(&semaphore);
    // Sección crítica
    printf("Thread %d está en la sección crítica\n", *(int *)arg);
    sem_post(&semaphore);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    sem_init(&semaphore, 0, 1);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    sem_destroy(&semaphore);
    return 0;
}

Monitores

Los monitores encapsulan tanto la exclusión mutua como la condición de sincronización, proporcionando una forma más estructurada de manejar la concurrencia.

class Monitor {
    private int count = 0;

    public synchronized void increment() {
        count++;
        System.out.println("Count incrementado: " + count);
    }

    public synchronized void decrement() {
        count--;
        System.out.println("Count decrementado: " + count);
    }
}

public class MonitorExample {
    public static void main(String[] args) {
        Monitor monitor = new Monitor();

        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                monitor.increment();
            }
        });

        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                monitor.decrement();
            }
        });

        t1.start();
        t2.start();
    }
}

Conclusión

Los algoritmos de sincronización son fundamentales para el correcto funcionamiento de sistemas operativos que manejan múltiples procesos y threads. Entender cómo y cuándo utilizar mutexes, semáforos y monitores es esencial para evitar problemas de concurrencia y asegurar la integridad de los datos compartidos. A través de ejemplos prácticos, hemos visto cómo implementar y utilizar estos mecanismos en diferentes lenguajes de programación.
