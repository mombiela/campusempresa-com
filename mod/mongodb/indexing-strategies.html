Introducción a la Indexación

Los índices son estructuras de datos especiales que almacenan una pequeña porción del conjunto de datos de la colección en una forma fácil de recorrer. El índice almacena el valor de un campo específico o conjunto de campos, ordenado por el valor del campo. Los índices soportan la ejecución eficiente de consultas en MongoDB.

¿Por qué Usar Índices?

  • Mejora del Rendimiento: Los índices mejoran significativamente el rendimiento de las consultas.
  • Recuperación Eficiente de Datos: Permiten que MongoDB recupere solo los documentos necesarios.
  • Reducción del Consumo de Recursos: Los índices reducen la cantidad de datos que MongoDB necesita escanear.

Tipos de Índices

MongoDB soporta varios tipos de índices, cada uno adecuado para diferentes casos de uso.

Índice de Campo Único

Un índice de campo único se crea en un solo campo de un documento.

db.collection.createIndex({ field: 1 })
  • Ejemplo: Creando un índice ascendente en el campo name. 
    db.users.createIndex({ name: 1 })

Índice Compuesto

Un índice compuesto se crea en múltiples campos.

db.collection.createIndex({ field1: 1, field2: -1 })
  • Ejemplo: Creando un índice compuesto en age (ascendente) y score (descendente). 
    db.students.createIndex({ age: 1, score: -1 })

Índice Multikey

Un índice multikey se crea en un campo de tipo array.

db.collection.createIndex({ arrayField: 1 })
  • Ejemplo: Creando un índice en el campo array tags. 
    db.articles.createIndex({ tags: 1 })

Índice de Texto

Un índice de texto soporta consultas de búsqueda de texto en contenido de tipo string.

db.collection.createIndex({ field: "text" })
  • Ejemplo: Creando un índice de texto en el campo description. 
    db.products.createIndex({ description: "text" })

Índice Geoespacial

Un índice geoespacial soporta consultas para formas y datos geoespaciales.

db.collection.createIndex({ location: "2dsphere" })
  • Ejemplo: Creando un índice 2dsphere en el campo location. 
    db.places.createIndex({ location: "2dsphere" })

Índice Hashed

Un índice hashed se usa para particionamiento y soporta consultas de igualdad.

db.collection.createIndex({ field: "hashed" })
  • Ejemplo: Creando un índice hashed en el campo user_id. 
    db.accounts.createIndex({ user_id: "hashed" })

Estrategias de Indexación

Elegir la estrategia de indexación correcta es crucial para optimizar el rendimiento de las consultas.

Entendiendo los Patrones de Consulta

  • Analizar Consultas: Usa el método explain() para entender la ejecución de la consulta. 
    db.collection.find({ field: value }).explain("executionStats")

Selección de Índices

  • Consultas de Igualdad: Usa índices de campo único o compuesto.
  • Consultas de Rango: Usa índices compuestos con el campo de rango como el último campo.
  • Operaciones de Ordenación: Crea índices que coincidan con el orden de la ordenación.
  • Consultas Multikey: Usa índices multikey para campos de tipo array.

Mantenimiento de Índices

  • Tamaño del Índice: Monitorea el tamaño del índice para evitar un uso excesivo de memoria.
  • Reconstrucción de Índices: Reconstruye periódicamente los índices para mantener el rendimiento. 
    db.collection.reIndex()

Mejores Prácticas de Indexación

  • Limitar el Número de Índices: Demasiados índices pueden degradar el rendimiento de escritura.
  • Usar Índices Dispersos: Para campos que no están presentes en todos los documentos. 
    db.collection.createIndex({ field: 1 }, { sparse: true })
  • Usar Consultas Cubiertas: Asegúrate de que las consultas puedan ser satisfechas usando solo índices.

Conclusión

Los índices son una herramienta poderosa en MongoDB que puede mejorar significativamente el rendimiento de las consultas cuando se usan correctamente. Entender los diferentes tipos de índices y cómo aplicarlos según los patrones de consulta es esencial para una gestión eficiente de la base de datos. El mantenimiento regular y la selección estratégica de índices son clave para optimizar el rendimiento y el uso de recursos.
