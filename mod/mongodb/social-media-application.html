Introducción

En este curso, exploraremos cómo usar MongoDB para construir una aplicación de redes sociales. Comenzaremos con los conceptos básicos de MongoDB y progresivamente avanzaremos hacia temas más avanzados. Al final de este curso, tendrás una comprensión sólida de MongoDB y cómo usarlo para gestionar datos para una aplicación de redes sociales.

Empezando con MongoDB

¿Qué es MongoDB?

  • MongoDB es una base de datos NoSQL que almacena datos en documentos similares a JSON.
  • Está diseñada para la escalabilidad y flexibilidad, lo que la hace ideal para aplicaciones web modernas.

Instalando MongoDB

  • Sigue la guía oficial de instalación de MongoDB para tu sistema operativo.
  • Verifica la instalación ejecutando mongo --version en tu terminal.

Operaciones Básicas de MongoDB

  • Conectándose a MongoDB: Usa la shell de MongoDB o un cliente como MongoDB Compass.
  • Operaciones CRUD:
    • Crear: Insertar documentos en una colección.
    • Leer: Consultar documentos de una colección.
    • Actualizar: Modificar documentos existentes.
    • Eliminar: Eliminar documentos de una colección.
// Conectándose a MongoDB
const { MongoClient } = require('mongodb');
const uri = "tu_cadena_de_conexión_mongodb";
const client = new MongoClient(uri, { useNewUrlParser: true, useUnifiedTopology: true });

async function run() {
    try {
        await client.connect();
        console.log("Conectado a MongoDB");
    } finally {
        await client.close();
    }
}
run().catch(console.dir);

Diseñando el Esquema de la Aplicación de Redes Sociales

Colección de Usuarios

  • Campos:
    • username: String, identificador único para el usuario.
    • email: String, dirección de correo única.
    • password: String, contraseña cifrada.
    • profile: Objeto que contiene información del perfil del usuario.
const userSchema = {
    username: "john_doe",
    email: "john@example.com",
    password: "hashed_password",
    profile: {
        fullName: "John Doe",
        bio: "Desarrollador de Software",
        avatar: "url_to_avatar"
    }
};

Colección de Publicaciones

  • Campos:
    • userId: ObjectId, referencia al usuario que creó la publicación.
    • content: String, el contenido de la publicación.
    • createdAt: Date, marca de tiempo de cuando se creó la publicación.
    • likes: Array de ObjectIds, referencias a usuarios que les gustó la publicación.
    • comments: Array de objetos de comentarios.
const postSchema = {
    userId: ObjectId("user_id"),
    content: "Esta es una publicación de muestra",
    createdAt: new Date(),
    likes: [ObjectId("user_id_1"), ObjectId("user_id_2")],
    comments: [
        {
            userId: ObjectId("commenter_id"),
            comment: "¡Buena publicación!",
            createdAt: new Date()
        }
    ]
};

Colección de Comentarios

  • Campos:
    • postId: ObjectId, referencia a la publicación.
    • userId: ObjectId, referencia al usuario que comentó.
    • comment: String, el texto del comentario.
    • createdAt: Date, marca de tiempo de cuando se creó el comentario.
const commentSchema = {
    postId: ObjectId("post_id"),
    userId: ObjectId("user_id"),
    comment: "Este es un comentario",
    createdAt: new Date()
};

Características Avanzadas de MongoDB

Indexación

  • Propósito: Mejorar el rendimiento de las consultas.
  • Tipos:
    • Índice de Campo Único
    • Índice Compuesto
    • Índice de Texto
// Creando un índice en el campo username
db.users.createIndex({ username: 1 });

// Creando un índice compuesto en email y username
db.users.createIndex({ email: 1, username: 1 });

Marco de Agregación

  • Propósito: Realizar procesamiento y análisis de datos complejos.
  • Etapas:
    • $match: Filtrar documentos.
    • $group: Agrupar documentos por un campo especificado.
    • $sort: Ordenar documentos.
// Pipeline de agregación para obtener el número de publicaciones por usuario
db.posts.aggregate([
    { $group: { _id: "$userId", postCount: { $sum: 1 } } },
    { $sort: { postCount: -1 } }
]);

Transacciones

  • Propósito: Asegurar la consistencia de datos a través de múltiples operaciones.
  • Uso: Usa transacciones para operaciones que necesitan ser atómicas.
async function createPostWithTransaction(client, userId, content) {
    const session = client.startSession();
    session.startTransaction();
    try {
        const postsCollection = client.db("social_media").collection("posts");
        const result = await postsCollection.insertOne({ userId, content, createdAt: new Date() }, { session });
        await session.commitTransaction();
        console.log("Publicación creada exitosamente");
    } catch (error) {
        await session.abortTransaction();
        console.error("Transacción abortada debido a un error: ", error);
    } finally {
        session.endSession();
    }
}
