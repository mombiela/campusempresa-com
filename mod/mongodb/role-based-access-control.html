Introducción a RBAC

El Control de Acceso Basado en Roles (RBAC) es un método para regular el acceso a recursos de una computadora o red basado en los roles de los usuarios individuales dentro de una empresa. En MongoDB, RBAC se utiliza para gestionar permisos y asegurar que los usuarios tengan los niveles de acceso apropiados para realizar sus tareas.

Conceptos Clave

  • Rol: Una colección de privilegios que pueden ser asignados a un usuario.
  • Privilegio: Una acción específica que un usuario puede realizar en un recurso.
  • Usuario: Un individuo al que se le asigna uno o más roles.

Configuración de RBAC en MongoDB

Creación de Usuarios

Para crear un usuario en MongoDB, se utiliza el método db.createUser(). A cada usuario se le debe asignar al menos un rol.

db.createUser({
  user: "nombreUsuario",
  pwd: "contraseña",
  roles: [
    { role: "readWrite", db: "exampleDB" }
  ]
});
  • user: El nombre de usuario.
  • pwd: La contraseña para el usuario.
  • roles: Un array de roles asignados al usuario.

Roles Incorporados

MongoDB proporciona varios roles incorporados que cubren casos de uso comunes. Algunos de los roles incorporados clave incluyen:

Rol Descripción
read Otorga acceso de solo lectura a la base de datos.
readWrite Otorga acceso de lectura y escritura a la base de datos.
dbAdmin Proporciona privilegios administrativos en la base de datos.
userAdmin Permite la gestión de usuarios dentro de la base de datos.
clusterAdmin Otorga privilegios administrativos en el clúster.

Roles Personalizados

También puedes crear roles personalizados para cumplir con requisitos específicos.

db.createRole({
  role: "rolPersonalizado",
  privileges: [
    { resource: { db: "exampleDB", collection: "" }, actions: ["find", "insert"] }
  ],
  roles: []
});
  • role: El nombre del rol personalizado.
  • privileges: Un array de privilegios, donde cada privilegio especifica el recurso y las acciones permitidas.
  • roles: Un array de roles que este rol hereda.

Gestión de Roles de Usuario

Asignación de Roles a Usuarios

Puedes asignar roles adicionales a un usuario existente utilizando el método db.grantRolesToUser().

db.grantRolesToUser("nombreUsuario", [{ role: "dbAdmin", db: "exampleDB" }]);

Revocación de Roles de Usuarios

Para revocar roles de un usuario, utiliza el método db.revokeRolesFromUser().

db.revokeRolesFromUser("nombreUsuario", [{ role: "readWrite", db: "exampleDB" }]);

Ejemplos Prácticos

Ejemplo 1: Creación de un Usuario de Solo Lectura

db.createUser({
  user: "usuarioSoloLectura",
  pwd: "contraseñaSoloLectura",
  roles: [
    { role: "read", db: "exampleDB" }
  ]
});

Esto crea un usuario usuarioSoloLectura con acceso de solo lectura a exampleDB.

Ejemplo 2: Creación de un Usuario con Rol Personalizado

db.createRole({
  role: "analistaDatos",
  privileges: [
    { resource: { db: "exampleDB", collection: "" }, actions: ["find"] }
  ],
  roles: []
});

db.createUser({
  user: "usuarioAnalista",
  pwd: "contraseñaAnalista",
  roles: [
    { role: "analistaDatos", db: "exampleDB" }
  ]
});

Esto crea un rol personalizado analistaDatos con acceso de solo lectura y lo asigna a usuarioAnalista.

Conclusión

RBAC en MongoDB es una característica poderosa que te permite gestionar los permisos de los usuarios de manera efectiva. Al entender y utilizar roles, privilegios y usuarios, puedes asegurar que tu despliegue de MongoDB sea seguro y que los usuarios tengan los niveles de acceso apropiados para realizar sus tareas. Ya sea utilizando roles incorporados o creando roles personalizados, RBAC proporciona la flexibilidad necesaria para manejar varios escenarios de control de acceso.
