Introducción a las Transacciones

Las transacciones en MongoDB te permiten ejecutar múltiples operaciones en aislamiento y asegurar que todas tengan éxito o todas fallen. Esto es crucial para mantener la integridad de los datos, especialmente en aplicaciones que requieren propiedades de atomicidad, consistencia, aislamiento y durabilidad (ACID).

Conceptos Clave

  • Atomicidad: Asegura que una serie de operaciones se completen en su totalidad o no se completen en absoluto.
  • Consistencia: Garantiza que la base de datos permanezca en un estado válido antes y después de la transacción.
  • Aislamiento: Asegura que las transacciones se procesen de manera segura e independiente al mismo tiempo sin interferencias.
  • Durabilidad: Asegura que una vez que una transacción se compromete, permanecerá así, incluso en caso de una falla del sistema.

Transacciones de Documento Único

MongoDB proporciona automáticamente atomicidad para operaciones de documento único. Esto significa que cualquier operación de actualización, inserción o eliminación en un solo documento es atómica por defecto.

Ejemplo

db.collection.updateOne(
  { _id: 1 },
  { $set: { status: "active" } }
);

En este ejemplo, la operación de actualización en el documento con _id: 1 es atómica.

Transacciones de Múltiples Documentos

A partir de MongoDB 4.0, se admiten transacciones de múltiples documentos, lo que te permite agrupar múltiples operaciones en una sola transacción.

Ejemplo

const session = client.startSession();

session.startTransaction();
try {
  db.collection1.insertOne({ _id: 1, name: "Alice" }, { session });
  db.collection2.insertOne({ _id: 2, name: "Bob" }, { session });
  
  session.commitTransaction();
} catch (error) {
  session.abortTransaction();
  throw error;
} finally {
  session.endSession();
}

En este ejemplo:

  1. Se inicia una sesión.
  2. Se inicia una transacción.
  3. Se realizan dos operaciones de inserción dentro de la transacción.
  4. La transacción se compromete si ambas operaciones tienen éxito.
  5. Si ocurre un error, la transacción se aborta.

Opciones de Transacción

Las transacciones se pueden configurar con varias opciones para controlar su comportamiento.

Ejemplo

const session = client.startSession();
const transactionOptions = {
  readConcern: { level: "snapshot" },
  writeConcern: { w: "majority" },
  readPreference: "primary"
};

session.startTransaction(transactionOptions);
try {
  db.collection1.insertOne({ _id: 1, name: "Alice" }, { session });
  db.collection2.insertOne({ _id: 2, name: "Bob" }, { session });
  
  session.commitTransaction();
} catch (error) {
  session.abortTransaction();
  throw error;
} finally {
  session.endSession();
}

En este ejemplo, la transacción se configura con opciones específicas de readConcern, writeConcern y readPreference.

Manejo de Errores en Transacciones

El manejo adecuado de errores es crucial para asegurar que las transacciones se comprometan o aborten correctamente.

Ejemplo

const session = client.startSession();

session.startTransaction();
try {
  db.collection1.insertOne({ _id: 1, name: "Alice" }, { session });
  db.collection2.insertOne({ _id: 2, name: "Bob" }, { session });
  
  session.commitTransaction();
} catch (error) {
  session.abortTransaction();
  console.error("Transacción abortada debido a un error: ", error);
} finally {
  session.endSession();
}

En este ejemplo, cualquier error durante la transacción causará que se aborte, y el error se registrará.

Conclusión

Las transacciones en MongoDB proporcionan una forma poderosa de asegurar la integridad y consistencia de los datos a través de múltiples operaciones. Al comprender y utilizar operaciones atómicas de documento único y transacciones de múltiples documentos, puedes construir aplicaciones robustas que manejen manipulaciones de datos complejas de manera confiable. Recuerda siempre manejar los errores de manera adecuada para mantener la estabilidad y confiabilidad de tus operaciones de base de datos.
