Introducción a la Tubería de Agregación

La Tubería de Agregación es un marco poderoso en MongoDB para la agregación de datos, que te permite procesar registros de datos y devolver resultados calculados. Es similar al concepto de tuberías de procesamiento de datos en sistemas tipo Unix, donde los datos se pasan a través de una serie de etapas, cada una realizando una operación específica.

Conceptos Clave

  • Etapas: Cada etapa en la tubería realiza una operación en los documentos de entrada y pasa el resultado a la siguiente etapa.
  • Operadores: Los operadores definen las operaciones que se realizarán en los datos dentro de cada etapa.
  • Tubería: Una secuencia de etapas a través de las cuales fluyen los datos.

Etapas Básicas en la Tubería de Agregación

$match

Filtra los documentos para pasar solo aquellos que coinciden con la(s) condición(es) especificada(s) a la siguiente etapa.

db.orders.aggregate([
  { $match: { status: "A" } }
])

Ejemplo: Esta etapa filtra documentos donde el campo status es "A".

$group

Agrupa documentos de entrada por una expresión de identificador especificada y aplica las expresiones de acumulador a cada grupo.

db.orders.aggregate([
  { $group: { _id: "$cust_id", total: { $sum: "$amount" } } }
])

Ejemplo: Esta etapa agrupa documentos por cust_id y calcula el monto total para cada cliente.

$project

Reconfigura cada documento en el flujo, como agregando nuevos campos o eliminando campos existentes.

db.orders.aggregate([
  { $project: { item: 1, total: { $multiply: ["$price", "$quantity"] } } }
])

Ejemplo: Esta etapa proyecta el campo item y calcula un nuevo campo total multiplicando price y quantity.

$sort

Ordena todos los documentos de entrada y los devuelve a la tubería en orden ordenado.

db.orders.aggregate([
  { $sort: { total: -1 } }
])

Ejemplo: Esta etapa ordena los documentos por el campo total en orden descendente.

$limit

Limita el número de documentos pasados a la siguiente etapa.

db.orders.aggregate([
  { $limit: 5 }
])

Ejemplo: Esta etapa limita la salida a los primeros 5 documentos.

$skip

Omite los primeros N documentos y pasa los documentos restantes a la siguiente etapa.

db.orders.aggregate([
  { $skip: 10 }
])

Ejemplo: Esta etapa omite los primeros 10 documentos.

Etapas y Operadores Avanzados

$unwind

Descompone un campo de matriz de los documentos de entrada para generar un documento por cada elemento.

db.orders.aggregate([
  { $unwind: "$items" }
])

Ejemplo: Esta etapa descompone el campo de matriz items, generando un documento por cada elemento.

$lookup

Realiza una unión externa izquierda a otra colección en la misma base de datos para filtrar documentos de la colección "unida" para su procesamiento.

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "cust_id",
      foreignField: "cust_id",
      as: "customer_info"
    }
  }
])

Ejemplo: Esta etapa une la colección orders con la colección customers basada en el campo cust_id.

$facet

Procesa múltiples tuberías de agregación dentro de una sola etapa en el mismo conjunto de documentos de entrada.

db.orders.aggregate([
  {
    $facet: {
      "categorizedByPrice": [
        { $match: { price: { $exists: 1 } } },
        { $bucket: { groupBy: "$price", boundaries: [0, 50, 100, 200], default: "Other" } }
      ],
      "categorizedByQuantity": [
        { $match: { quantity: { $exists: 1 } } },
        { $bucket: { groupBy: "$quantity", boundaries: [0, 20, 50, 100], default: "Other" } }
      ]
    }
  }
])

Ejemplo: Esta etapa categoriza documentos por precio y cantidad usando dos tuberías separadas.

Ejemplos Prácticos

Ejemplo 1: Informe de Ventas

Generar un informe de ventas que muestre el monto total de ventas para cada categoría de producto.

db.sales.aggregate([
  { $match: { date: { $gte: ISODate("2023-01-01"), $lt: ISODate("2024-01-01") } } },
  { $group: { _id: "$category", totalSales: { $sum: "$amount" } } },
  { $sort: { totalSales: -1 } }
])

Explicación: Esta tubería filtra ventas dentro del año 2023, las agrupa por categoría, calcula el monto total de ventas para cada categoría y ordena los resultados en orden descendente.

Ejemplo 2: Frecuencia de Compras de Clientes

Encontrar el número de compras realizadas por cada cliente.

db.orders.aggregate([
  { $group: { _id: "$cust_id", purchaseCount: { $sum: 1 } } },
  { $sort: { purchaseCount: -1 } }
])

Explicación: Esta tubería agrupa pedidos por ID de cliente, cuenta el número de compras para cada cliente y ordena los resultados en orden descendente.

Conclusión

La Tubería de Agregación en MongoDB es una herramienta versátil y poderosa para el procesamiento y análisis de datos. Al encadenar múltiples etapas juntas, puedes realizar transformaciones y cálculos de datos complejos de manera eficiente. Comprender las etapas básicas y avanzadas, junto con ejemplos prácticos, te ayudará a aprovechar todo el potencial del marco de agregación de MongoDB en tus aplicaciones.
