Introducción

MongoDB es una base de datos NoSQL popular conocida por su flexibilidad y escalabilidad. Comprender su terminología básica es crucial para trabajar eficazmente con MongoDB. Esta sección cubrirá los términos y conceptos fundamentales que necesitas para comenzar.

Conceptos Clave

Base de Datos

  • Definición: Una base de datos en MongoDB es un contenedor para colecciones.
  • Ejemplo: Una base de datos llamada myDatabase puede ser creada usando el siguiente comando: 
    use myDatabase

Colección

  • Definición: Una colección es un grupo de documentos de MongoDB, similar a una tabla en bases de datos relacionales.
  • Ejemplo: Creando una colección llamada users: 
    db.createCollection("users")

Documento

  • Definición: Un documento es un conjunto de pares clave-valor, similar a una fila en una tabla.
  • Ejemplo: Un documento simple en la colección users: 
    {
  "name": "John Doe",
  "age": 30,
  "email": "john.doe@example.com"
}

Campo

  • Definición: Un campo es un par clave-valor en un documento.
  • Ejemplo: En el documento anterior, name, age y email son campos.

BSON

  • Definición: BSON (Binary JSON) es la representación binaria de documentos similares a JSON. MongoDB usa BSON para almacenar documentos.
  • Ejemplo: BSON permite más tipos de datos que JSON, como Date y Binary.

Índice

  • Definición: Un índice es una estructura de datos que mejora la velocidad de las operaciones de recuperación de datos.
  • Ejemplo: Creando un índice en el campo email en la colección users: 
    db.users.createIndex({ email: 1 })

Clave Primaria

  • Definición: Una clave primaria es un identificador único para un documento. En MongoDB, el campo _id sirve como clave primaria.
  • Ejemplo: El campo _id se agrega automáticamente si no se especifica: 
    {
  "_id": ObjectId("507f1f77bcf86cd799439011"),
  "name": "John Doe",
  "age": 30,
  "email": "john.doe@example.com"
}

Conjunto de Réplicas

  • Definición: Un conjunto de réplicas es un grupo de servidores de MongoDB que mantienen el mismo conjunto de datos, proporcionando redundancia y alta disponibilidad.
  • Ejemplo: Una configuración típica de conjunto de réplicas incluye un nodo primario y dos nodos secundarios.

Fragmentación

  • Definición: La fragmentación es un método para distribuir datos a través de múltiples servidores para soportar grandes conjuntos de datos y operaciones de alto rendimiento.
  • Ejemplo: Fragmentando una colección basada en un campo específico: 
    sh.shardCollection("myDatabase.users", { "email": 1 })

Resumen

Comprender estos términos básicos de MongoDB es esencial para trabajar eficazmente con la base de datos. Aquí tienes un resumen rápido:

  • Base de Datos: Contenedor para colecciones.
  • Colección: Grupo de documentos.
  • Documento: Conjunto de pares clave-valor.
  • Campo: Par clave-valor en un documento.
  • BSON: Representación binaria de documentos similares a JSON.
  • Índice: Estructura de datos para acelerar la recuperación de datos.
  • Clave Primaria: Identificador único para un documento.
  • Conjunto de Réplicas: Grupo de servidores de MongoDB con el mismo conjunto de datos.
  • Fragmentación: Distribución de datos a través de múltiples servidores.

Con estos conceptos en mente, estarás mejor preparado para profundizar en MongoDB y utilizar todo su potencial.
