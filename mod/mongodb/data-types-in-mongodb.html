Introducción

En MongoDB, entender los tipos de datos es crucial para un diseño y consulta de bases de datos efectivos. MongoDB soporta un conjunto rico de tipos de datos que te permiten almacenar y manipular datos en varias formas. Esta sección cubrirá los principales tipos de datos disponibles en MongoDB, proporcionando explicaciones, ejemplos y ejercicios para ayudarte a comprender su uso.

Tipos de Datos Clave

Cadena

  • Descripción: Usado para almacenar datos de texto.
  • Ejemplo: 
    { "name": "Alice" }
  • Uso: Comúnmente usado para almacenar nombres, direcciones y otra información textual.

Entero

  • Descripción: Usado para almacenar datos numéricos sin componente fraccional.
  • Ejemplo: 
    { "age": 30 }
  • Uso: Ideal para contar, indexar y almacenar números enteros.

Doble

  • Descripción: Usado para almacenar números de punto flotante.
  • Ejemplo: 
    { "price": 19.99 }
  • Uso: Adecuado para almacenar precios, medidas y otros valores decimales.

Booleano

  • Descripción: Representa un valor binario, ya sea true o false.
  • Ejemplo: 
    { "isActive": true }
  • Uso: Útil para indicadores de estado, banderas y condiciones binarias.

Arreglo

  • Descripción: Usado para almacenar listas de valores.
  • Ejemplo: 
    { "tags": ["mongodb", "database", "nosql"] }
  • Uso: Ideal para almacenar múltiples valores en un solo campo, como etiquetas, categorías o listas de ítems.

Objeto

  • Descripción: Usado para almacenar documentos embebidos.
  • Ejemplo: 
    { 
  "address": {
    "street": "123 Main St",
    "city": "Anytown",
    "zip": "12345"
  }
}
  • Uso: Útil para anidar datos relacionados dentro de un solo documento.

Fecha

  • Descripción: Usado para almacenar fechas y horas.
  • Ejemplo: 
    { "createdAt": ISODate("2023-10-01T00:00:00Z") }
  • Uso: Esencial para marcar eventos con tiempo, programar y rastrear datos basados en tiempo.

ObjectId

  • Descripción: Un identificador único para documentos.
  • Ejemplo: 
    { "_id": ObjectId("507f1f77bcf86cd799439011") }
  • Uso: Generado automáticamente para cada documento en una colección, asegurando unicidad.

Nulo

  • Descripción: Representa la ausencia de un valor.
  • Ejemplo: 
    { "middleName": null }
  • Uso: Útil para campos que son opcionales o desconocidos.

Datos Binarios

  • Descripción: Usado para almacenar datos binarios.
  • Ejemplo: 
    { "file": BinData(0, "base64encodedstring") }
  • Uso: Adecuado para almacenar imágenes, archivos y otros contenidos binarios.

Expresión Regular

  • Descripción: Usado para almacenar expresiones regulares.
  • Ejemplo: 
    { "pattern": /abc/i }
  • Uso: Útil para la coincidencia de patrones dentro de cadenas.

Comparación de Tipos de Datos

| Tipo de Dato | Descripción | Ejemplo | |-------------------|---------------------------------------|----------------------------------------------| | Cadena | Datos de texto | { "name": "Alice" } | | Entero | Números enteros | { "age": 30 } | | Doble | Números de punto flotante | { "price": 19.99 } | | Booleano | Valores binarios | { "isActive": true } | | Arreglo | Listas de valores | { "tags": ["mongodb", "database", "nosql"] } | | Objeto | Documentos embebidos | { "address": { "street": "123 Main St" } } | | Fecha | Fechas y horas | { "createdAt": ISODate("2023-10-01T00:00:00Z") } | | ObjectId | Identificador único | { "_id": ObjectId("507f1f77bcf86cd799439011") } | | Nulo | Ausencia de valor | { "middleName": null } | | Datos Binarios | Contenido binario | { "file": BinData(0, "base64encodedstring") } | | Expresión Regular | Coincidencia de patrones | { "pattern": /abc/i } |

Ejercicios

  1. Crear un Documento con Varios Tipos de Datos:

    • Crea un documento de MongoDB que incluya al menos cinco tipos de datos diferentes.
    • Ejemplo: 
      {
  "name": "John Doe",
  "age": 25,
  "isActive": true,
  "tags": ["developer", "blogger"],
  "address": {
    "street": "456 Elm St",
    "city": "Somewhere",
    "zip": "67890"
  },
  "createdAt": ISODate("2023-10-01T00:00:00Z")
}

  2. Consultar Documentos por Tipo de Dato:

    • Escribe una consulta para encontrar todos los documentos donde el campo isActive sea true.
    • Ejemplo: 
      db.collection.find({ "isActive": true })

  3. Actualizar un Documento con un Nuevo Tipo de Dato:

    • Añade un nuevo campo lastLogin con la fecha actual a un documento existente.
    • Ejemplo: 
      db.collection.updateOne(
  { "name": "John Doe" },
  { $set: { "lastLogin": new Date() } }
)

Conclusión

Entender y usar efectivamente los tipos de datos de MongoDB es fundamental para diseñar bases de datos robustas y eficientes. Cada tipo de dato sirve un propósito específico y puede influir en cómo almacenas, consultas y manipulas datos. Al dominar estos tipos de datos, puedes asegurar que tus aplicaciones de MongoDB sean tanto eficientes como escalables.
