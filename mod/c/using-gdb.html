Usando GDB
Optimización de Código ►

El Depurador de GNU (GDB) es una herramienta poderosa para depurar programas en C. Esta sección te guiará a través de los conceptos básicos del uso de GDB, desde su configuración hasta el uso de sus características avanzadas.

Introducción a GDB

GDB te permite:

  • Inspeccionar lo que el programa está haciendo en un punto específico en el tiempo.
  • Cambiar variables y ejecutar código línea por línea.
  • Establecer puntos de interrupción para pausar la ejecución en líneas específicas.
  • Rastrear el flujo de ejecución para entender cómo el programa alcanza un cierto estado.

Configuración de GDB

Para usar GDB, necesitas compilar tu programa en C con información de depuración. Esto se hace usando la opción -g con gcc.

gcc -g -o miprograma miprograma.c

Este comando compila miprograma.c en un ejecutable llamado miprograma con información de depuración incluida.

Comandos Básicos de GDB

Iniciando GDB

Para iniciar GDB con tu programa compilado:

gdb miprograma

Ejecutando el Programa

Para ejecutar el programa dentro de GDB:

(gdb) run

Estableciendo Puntos de Interrupción

Los puntos de interrupción te permiten pausar la ejecución de tu programa en líneas o funciones específicas.

  • Para establecer un punto de interrupción en una línea específica:
(gdb) break 10
  • Para establecer un punto de interrupción al inicio de una función:
(gdb) break main

Inspeccionando Variables

Para inspeccionar el valor de una variable:

(gdb) print nombre_variable

Avanzando a Través del Código

  • Para ejecutar la siguiente línea de código:
(gdb) next
  • Para entrar en una llamada de función:
(gdb) step

Continuando la Ejecución

Para continuar ejecutando el programa después de alcanzar un punto de interrupción:

(gdb) continue

Saliendo de GDB

Para salir de GDB:

(gdb) quit

Características Avanzadas de GDB

Puntos de Interrupción Condicionales

Puedes establecer puntos de interrupción que solo se activan cuando una cierta condición es verdadera.

(gdb) break 10 if x == 5

Puntos de Observación

Los puntos de observación pausan la ejecución cuando el valor de una variable cambia.

(gdb) watch nombre_variable

Rastreo

Para ver la pila de llamadas y entender cómo el programa llegó al punto actual:

(gdb) backtrace

Examinando la Memoria

Para examinar la memoria en una dirección específica:

(gdb) x /nfu dirección
  • n es el número de unidades a mostrar.
  • f es el formato (por ejemplo, x para hexadecimal, d para decimal).
  • u es el tamaño de la unidad (por ejemplo, b para bytes, h para medias palabras).

Sesión de Ejemplo

Aquí hay una sesión de GDB de ejemplo para ilustrar los comandos:

$ gcc -g -o ejemplo ejemplo.c
$ gdb ejemplo
(gdb) break main

(gdb) run
Starting program: /path/to/ejemplo 

Breakpoint 1, main () at ejemplo.c:5
5       int x = 0;
(gdb) print x
$1 = 0
(gdb) next
6       x = 5;
(gdb) print x
$2 = 0
(gdb) next
7       printf("%d\n", x);
(gdb) print x
$3 = 5
(gdb) continue
5
[Inferior 1 (process 1234) exited normally]
(gdb) quit

Conclusión

GDB es una herramienta esencial para depurar programas en C. Al dominar sus características básicas y avanzadas, puedes mejorar significativamente tu capacidad para diagnosticar y corregir errores. Practica usando GDB con diferentes programas para familiarizarte más con sus comandos y capacidades.
