Introducción al Switch Case

La declaración switch en C es una declaración de control que permite probar una variable para igualdad contra una lista de valores. Cada valor se llama un caso, y la variable que se está cambiando se verifica para cada caso.

Conceptos Clave

  • Declaración Switch: Una declaración de control que prueba una variable contra múltiples casos.
  • Caso: Un valor posible para la variable que se está probando.
  • Declaración Break: Se usa para terminar un caso en la declaración switch.
  • Caso Default: Un caso opcional que se ejecuta si ninguno de los casos especificados coincide con la variable.

Sintaxis del Switch Case

La sintaxis básica de una declaración switch case en C es la siguiente:

switch (expresión) {
    case constante1:
        // código a ejecutar si la expresión es igual a constante1
        break;
    case constante2:
        // código a ejecutar si la expresión es igual a constante2
        break;
    // puedes tener cualquier número de declaraciones case
    default:
        // código a ejecutar si la expresión no coincide con ningún caso
}

Ejemplo

Aquí hay un ejemplo simple para demostrar el uso de la declaración switch case:

#include <stdio.h>

int main() {
    int day = 3;

    switch (day) {
        case 1:
            printf("Lunes\n");
            break;
        case 2:
            printf("Martes\n");
            break;
        case 3:
            printf("Miércoles\n");
            break;
        case 4:
            printf("Jueves\n");
            break;
        case 5:
            printf("Viernes\n");
            break;
        case 6:
            printf("Sábado\n");
            break;
        case 7:
            printf("Domingo\n");
            break;
        default:
            printf("Día inválido\n");
    }

    return 0;
}

Explicación

  • La declaración switch evalúa la expresión day.
  • Luego compara day con cada caso.
  • Cuando day coincide con un caso (en este ejemplo, day es 3, por lo que coincide con case 3), se ejecuta el bloque de código correspondiente.
  • La declaración break evita que la ejecución continúe al siguiente caso.
  • Si ningún caso coincide, se ejecuta el caso default.

Uso Avanzado

Switch Case Anidado

Puedes anidar declaraciones switch dentro de otras. Esto es útil para procesos de toma de decisiones más complejos.

#include <stdio.h>

int main() {
    char grade = 'B';

    switch (grade) {
        case 'A':
        case 'B':
            printf("¡Buen trabajo!\n");
            switch (grade) {
                case 'A':
                    printf("¡Excelente!\n");
                    break;
                case 'B':
                    printf("¡Bien hecho!\n");
                    break;
            }
            break;
        case 'C':
            printf("Aprobaste.\n");
            break;
        default:
            printf("Mejor suerte la próxima vez.\n");
    }

    return 0;
}

Comportamiento de Caída

En C, si omites la declaración break, el programa continúa ejecutando el siguiente caso. Esto se conoce como comportamiento de caída.

#include <stdio.h>

int main() {
    int num = 2;

    switch (num) {
        case 1:
            printf("Uno\n");
        case 2:
            printf("Dos\n");
        case 3:
            printf("Tres\n");
        default:
            printf("Default\n");
    }

    return 0;
}

Explicación

  • En este ejemplo, dado que no hay una declaración break después de case 2, el programa continuará ejecutando case 3 y el caso default, resultando en la salida: 
    Dos
Tres
Default

Resumen

  • La declaración switch es una herramienta poderosa para manejar múltiples condiciones basadas en el valor de una sola variable.
  • Cada case representa un valor posible para la variable.
  • La declaración break es crucial para evitar el comportamiento de caída.
  • El caso default es opcional pero útil para manejar valores inesperados.
  • El uso avanzado incluye declaraciones switch anidadas y caídas intencionales para una lógica específica.

Al comprender y utilizar la declaración switch case, puedes escribir código más legible y mantenible para escenarios complejos de toma de decisiones.
