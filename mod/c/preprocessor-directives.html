Directivas del Preprocesador en C
Macros y Funciones Inline ►

Las directivas del preprocesador son comandos que dan instrucciones al compilador para preprocesar el código fuente antes de que comience la compilación real. Son una parte crucial del lenguaje de programación C, permitiendo la compilación condicional, definiciones de macros, inclusión de archivos y más.

Conceptos Clave

  • Directivas del Preprocesador: Instrucciones que se ejecutan antes de la compilación real del código.
  • Sintaxis: Las directivas del preprocesador comienzan con un símbolo # y no terminan con un punto y coma.
  • Directivas Comunes: #include, #define, #undef, #ifdef, #ifndef, #if, #else, #elif, #endif, #error, #pragma.

Directivas Comunes del Preprocesador

`#include`

La directiva #include se usa para incluir el contenido de un archivo o biblioteca en el programa.

#include <stdio.h>  // Incluye la biblioteca estándar de entrada-salida
#include "myheader.h"  // Incluye un archivo de cabecera definido por el usuario
  • Corchetes Angulares < >: Usados para archivos de la biblioteca estándar.
  • Comillas Dobles " ": Usadas para archivos definidos por el usuario.

`#define`

La directiva #define se usa para crear definiciones de macros. Puede definir constantes o macros que pueden tomar argumentos.

#define PI 3.14159  // Define una constante
#define SQUARE(x) ((x) * (x))  // Define una macro que calcula el cuadrado de un número

`#undef`

La directiva #undef se usa para desdefinir una macro.

#undef PI  // Desdefine la macro PI

Compilación Condicional

La compilación condicional permite que partes del código se compilen o se ignoren según ciertas condiciones.

#ifdef y #ifndef

#ifdef DEBUG
    printf("El modo de depuración está activado.\n");
#endif

#ifndef RELEASE
    printf("El modo de lanzamiento está desactivado.\n");
#endif
  • #ifdef: Compila el código si la macro está definida.
  • #ifndef: Compila el código si la macro no está definida.

#if, #else, #elif, y #endif

#define VERSION 2

#if VERSION == 1
    printf("Versión 1\n");
#elif VERSION == 2
    printf("Versión 2\n");
#else
    printf("Versión desconocida\n");
#endif
  • #if: Inicia un bloque condicional.
  • #else: Proporciona una alternativa si la condición #if es falsa.
  • #elif: Else if, proporciona otra condición para verificar.
  • #endif: Termina el bloque condicional.

`#error`

La directiva #error genera un error de compilación con un mensaje especificado.

#if VERSION > 3
    #error "¡Número de versión demasiado alto!"
#endif

`#pragma`

La directiva #pragma se usa para proporcionar información adicional al compilador. Su comportamiento es específico del compilador.

#pragma message("Compilando el código...")

Ejemplos y Ejercicios

Ejemplo 1: Usando `#include` y `#define`

#include <stdio.h>
#define GREETING "¡Hola, Mundo!"

int main() {
    printf("%s\n", GREETING);
    return 0;
}

Ejemplo 2: Compilación Condicional

#include <stdio.h>
#define DEBUG

int main() {
#ifdef DEBUG
    printf("La depuración está habilitada.\n");
#endif
    printf("El programa está en ejecución.\n");
    return 0;
}

Ejercicio 1

Escribe un programa que use #define para crear una macro para calcular el área de un círculo. Usa la macro en la función main para calcular e imprimir el área de un círculo con un radio de 5.

Ejercicio 2

Modifica el programa anterior para incluir compilación condicional. Define una macro DEBUG y usa #ifdef para imprimir información adicional de depuración si DEBUG está definido.

Resumen

Las directivas del preprocesador en C son herramientas poderosas que te permiten controlar el proceso de compilación. Te permiten incluir archivos, definir constantes y macros, y compilar código condicionalmente. Entender y usar estas directivas de manera efectiva puede mejorar en gran medida la flexibilidad y el mantenimiento de tus programas en C.
