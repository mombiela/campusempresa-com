Introducción

En esta sección, proporcionaremos una descripción general del proyecto en el que trabajarás a lo largo de este curso. Este proyecto te ayudará a aplicar los conceptos y técnicas que aprendas en la programación MUMPS (M), desde niveles principiantes hasta avanzados. Al final de este curso, tendrás una comprensión completa de MUMPS y serás capaz de desarrollar aplicaciones robustas utilizando este lenguaje.

Descripción del Proyecto

El proyecto en el que trabajaremos es un Sistema de Gestión de Pacientes (SGP) simple pero completo. Este sistema permitirá a los usuarios:

  • Agregar nuevos pacientes.
  • Actualizar la información de los pacientes.
  • Recuperar los detalles de los pacientes.
  • Eliminar registros de pacientes.

El proyecto se dividirá en varios módulos, cada uno enfocado en diferentes aspectos de la programación MUMPS. A medida que avances en el curso, irás construyendo y mejorando el SGP.

Requisitos del Proyecto

Para completar con éxito este proyecto, necesitarás cumplir con los siguientes requisitos:

Requisitos Técnicos

  • Entorno MUMPS: Asegúrate de tener un entorno MUMPS funcional. Puedes usar GT.M u otra implementación de MUMPS.
  • Editor de Texto: Usa un editor de texto que soporte resaltado de sintaxis MUMPS. Ejemplos incluyen VS Code con las extensiones apropiadas o cualquier IDE específico para MUMPS.
  • Conocimientos Básicos de MUMPS: Familiaridad con la sintaxis y comandos básicos de MUMPS. Cubriremos estos en las secciones para principiantes del curso.

Requisitos Funcionales

  • Gestión de Datos de Pacientes: El sistema debe ser capaz de manejar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) para los registros de pacientes.
  • Validación de Datos: Implementar validación básica para asegurar la integridad de los datos (por ejemplo, no campos vacíos, formatos de fecha válidos).
  • Interfaz de Usuario: Una interfaz simple basada en texto para interactuar con el sistema.

Conceptos Clave

Aquí hay algunos conceptos clave que necesitarás entender e implementar en tu proyecto:

  • Variables Globales: MUMPS usa variables globales para almacenar datos de manera persistente. Aprenderás a usarlas para almacenar registros de pacientes.
  • Subrutinas y Funciones: Modulariza tu código usando subrutinas y funciones para una mejor mantenibilidad.
  • Manejo de Errores: Implementa manejo de errores para gestionar situaciones inesperadas de manera elegante.
  • Entrada/Salida de Archivos: Aprende a leer y escribir archivos para la persistencia de datos.

Código de Ejemplo

Aquí hay un ejemplo simple para ilustrar cómo podrías comenzar agregando un nuevo registro de paciente:

AddPatient
    WRITE "Ingrese ID del Paciente: "
    READ PatientID
    WRITE "Ingrese Nombre del Paciente: "
    READ PatientName
    WRITE "Ingrese Fecha de Nacimiento del Paciente (AAAAMMDD): "
    READ PatientDOB
    
    ; Almacenar los datos en variables globales
    SET ^Patient(PatientID, "Name") = PatientName
    SET ^Patient(PatientID, "DOB") = PatientDOB
    
    WRITE "¡Registro de paciente agregado exitosamente!", !
    QUIT

Explicación

  • WRITE y READ: Estos comandos se usan para operaciones de entrada y salida.
  • Variables Globales: ^Patient(PatientID, "Name") y ^Patient(PatientID, "DOB") se usan para almacenar la información del paciente de manera persistente.

Resumen

En esta sección, hemos delineado el proyecto en el que trabajarás a lo largo del curso. El Sistema de Gestión de Pacientes te ayudará a aplicar los conceptos de programación MUMPS en un contexto práctico. También hemos discutido los requisitos técnicos y funcionales, los conceptos clave, y proporcionado un ejemplo de código simple para que comiences. A medida que avances en el curso, construirás sobre esta base para crear una aplicación completamente funcional.
