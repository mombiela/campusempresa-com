Pruebas y Validación
Mejores Prácticas y Estándares de Codificación ►

Las pruebas y la validación son aspectos cruciales del desarrollo de software, asegurando que tu código MUMPS (M) sea confiable, eficiente y libre de errores. Esta sección te guiará a través de los conceptos básicos de las pruebas y la validación, progresando hacia técnicas más avanzadas.

Introducción a las Pruebas en MUMPS

Las pruebas son el proceso de ejecutar un programa con la intención de encontrar errores. Ayuda a asegurar que el código se comporte como se espera bajo diversas condiciones.

Conceptos Clave

  • Pruebas Unitarias: Pruebas de unidades o componentes individuales de un programa.
  • Pruebas de Integración: Pruebas de la interacción entre unidades o componentes integrados.
  • Pruebas del Sistema: Pruebas del sistema de software completo e integrado.
  • Pruebas de Regresión: Reejecución de pruebas para asegurar que los cambios no hayan introducido nuevos errores.
  • Validación: Asegurar que el software cumpla con los requisitos y expectativas de los usuarios finales.

Pruebas Unitarias en MUMPS

Las pruebas unitarias se enfocan en verificar las partes más pequeñas de la aplicación, como funciones o procedimientos, para asegurar que funcionen correctamente.

Ejemplo: Prueba Unitaria Simple

Vamos a crear una función simple y una prueba unitaria correspondiente.

; Función para sumar dos números
ADD(a, b)
    Q a + b

; Prueba Unitaria para la función ADD
TESTADD
    N result
    S result=$$ADD(2,3)
    I result=5 W "Prueba Aprobada",! Q
    W "Prueba Fallida",!
    Q
  • Explicación:
    • ADD(a, b): Una función que toma dos argumentos y devuelve su suma.
    • TESTADD: Una prueba unitaria que verifica si ADD(2, 3) devuelve 5.

Ejecutando la Prueba

Para ejecutar la prueba, simplemente llama al procedimiento TESTADD desde el prompt de MUMPS.

D TESTADD

Pruebas de Integración en MUMPS

Las pruebas de integración aseguran que los diferentes módulos o servicios utilizados por tu aplicación funcionen bien juntos.

Ejemplo: Prueba de Integración

Supongamos que tenemos dos funciones, ADD y MULTIPLY, y queremos probar su integración.

; Función para multiplicar dos números
MULTIPLY(a, b)
    Q a * b

; Prueba de Integración para las funciones ADD y MULTIPLY
TESTINTEGRATION
    N sum, product
    S sum=$$ADD(2,3)
    S product=$$MULTIPLY(sum,4)
    I product=20 W "Prueba de Integración Aprobada",! Q
    W "Prueba de Integración Fallida",!
    Q
  • Explicación:
    • MULTIPLY(a, b): Una función que toma dos argumentos y devuelve su producto.
    • TESTINTEGRATION: Una prueba de integración que verifica si MULTIPLY($$ADD(2, 3), 4) devuelve 20.

Ejecutando la Prueba

Para ejecutar la prueba de integración, llama al procedimiento TESTINTEGRATION desde el prompt de MUMPS.

D TESTINTEGRATION

Pruebas del Sistema en MUMPS

Las pruebas del sistema implican probar todo el sistema en su conjunto para asegurar que cumpla con los requisitos especificados.

Ejemplo: Prueba del Sistema

Considera un sistema simple que calcula el precio total de los artículos en un carrito de compras.

; Función para calcular el precio total
TOTALPRICE(cart)
    N total, i
    S total=0
    F i=1:1:$L(cart,",") S total=total+$P(cart,",",i)
    Q total

; Prueba del Sistema para la función TOTALPRICE
TESTSYSTEM
    N cart, total
    S cart="10,20,30"
    S total=$$TOTALPRICE(cart)
    I total=60 W "Prueba del Sistema Aprobada",! Q
    W "Prueba del Sistema Fallida",!
    Q
  • Explicación:
    • TOTALPRICE(cart): Una función que toma una lista de precios separada por comas y devuelve su total.
    • TESTSYSTEM: Una prueba del sistema que verifica si TOTALPRICE("10,20,30") devuelve 60.

Ejecutando la Prueba

Para ejecutar la prueba del sistema, llama al procedimiento TESTSYSTEM desde el prompt de MUMPS.

D TESTSYSTEM

Pruebas de Regresión en MUMPS

Las pruebas de regresión aseguran que los nuevos cambios no afecten negativamente la funcionalidad existente.

Ejemplo: Prueba de Regresión

Después de modificar la función ADD para manejar números negativos, necesitamos asegurarnos de que aún funcione para números positivos.

; Función ADD modificada para manejar números negativos
ADD(a, b)
    Q a + b

; Prueba de Regresión para la función ADD
TESTREGRESSION
    N result
    S result=$$ADD(2,3)
    I result=5 W "Prueba de Regresión Aprobada",! Q
    W "Prueba de Regresión Fallida",!
    Q
  • Explicación:
    • TESTREGRESSION: Una prueba de regresión que verifica si ADD(2, 3) aún devuelve 5 después de las modificaciones.

Ejecutando la Prueba

Para ejecutar la prueba de regresión, llama al procedimiento TESTREGRESSION desde el prompt de MUMPS.

D TESTREGRESSION

Conclusión

Las pruebas y la validación son prácticas esenciales en la programación MUMPS para asegurar la calidad y confiabilidad del código. Al implementar pruebas unitarias, pruebas de integración, pruebas del sistema y pruebas de regresión, puedes verificar sistemáticamente que tu código funcione como se espera y cumpla con los requisitos del usuario. Las pruebas regulares ayudan a identificar problemas temprano, haciendo más fácil mantener y mejorar tus aplicaciones MUMPS.
