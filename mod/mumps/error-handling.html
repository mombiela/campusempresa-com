Manejo de Errores en MUMPS (M)
Creación y Uso de Rutinas ►

El manejo de errores es un aspecto crucial de la programación en cualquier lenguaje, incluido MUMPS. Un manejo adecuado de errores asegura que tus programas puedan manejar situaciones inesperadas de manera elegante y continuar operando o fallar de manera segura. Esta sección cubrirá el manejo de errores desde niveles principiantes hasta avanzados.

Introducción al Manejo de Errores

El manejo de errores en MUMPS implica detectar y gestionar errores que ocurren durante la ejecución de un programa. MUMPS proporciona mecanismos para atrapar errores y tomar acciones apropiadas.

Conceptos Clave

  • Captura de Errores: El proceso de capturar errores cuando ocurren.
  • $ZTRAP: Una variable especial utilizada para especificar la rutina de manejo de errores.
  • $ECODE: Una variable especial que contiene el código de error cuando ocurre un error.
  • $ETRAP: Un mecanismo de manejo de errores más nuevo y flexible en comparación con $ZTRAP.

Manejo Básico de Errores con $ZTRAP

Configuración de $ZTRAP

La variable $ZTRAP se utiliza para especificar una etiqueta a la que se debe transferir el control si ocurre un error.

SET $ZTRAP="ERROR^MYPROG"

En este ejemplo, si ocurre un error, el control se transfiere a la etiqueta ERROR en la rutina MYPROG.

Ejemplo

MYPROG ;
    NEW X,Y
    SET $ZTRAP="ERROR^MYPROG"
    SET X=10,Y=0
    WRITE "Result: ",X/Y,!
    QUIT

ERROR ;
    WRITE "An error occurred: ",$ZERROR,!
    QUIT

En este ejemplo:

  • El programa intenta dividir X por Y, que es cero, causando un error de división por cero.
  • El control se transfiere a la etiqueta ERROR, donde se muestra el mensaje de error.

Manejo Intermedio de Errores con $ECODE

Uso de $ECODE

La variable $ECODE contiene el código de error cuando ocurre un error. Puedes usarla para determinar el tipo de error y tomar la acción apropiada.

SET $ECODE=",<DIVIDE>,"

Ejemplo

MYPROG ;
    NEW X,Y
    SET $ZTRAP="ERROR^MYPROG"
    SET X=10,Y=0
    WRITE "Result: ",X/Y,!
    QUIT

ERROR ;
    IF $ECODE[",<DIVIDE>," DO
    . WRITE "Division by zero error",!
    ELSE  DO
    . WRITE "An unexpected error occurred: ",$ZERROR,!
    SET $ECODE=""
    QUIT

En este ejemplo:

  • El programa verifica si el código de error contiene <DIVIDE>, indicando un error de división por cero.
  • Se muestran mensajes apropiados según el tipo de error.

Manejo Avanzado de Errores con $ETRAP

Configuración de $ETRAP

La variable $ETRAP permite un manejo de errores más flexible. Puede contener código ejecutable MUMPS que se ejecuta cuando ocurre un error.

SET $ETRAP="DO ERROR^MYPROG"

Ejemplo

MYPROG ;
    NEW X,Y
    SET $ETRAP="DO ERROR^MYPROG"
    SET X=10,Y=0
    WRITE "Result: ",X/Y,!
    QUIT

ERROR ;
    WRITE "An error occurred: ",$ZERROR,!
    SET $ECODE=""
    QUIT

En este ejemplo:

  • $ETRAP se configura para ejecutar la rutina ERROR cuando ocurre un error.
  • Se ejecuta la rutina de manejo de errores y se muestra el mensaje de error.

Resumen

El manejo de errores en MUMPS es esencial para crear programas robustos y confiables. Esta sección cubrió:

  • Manejo básico de errores usando $ZTRAP.
  • Manejo intermedio de errores usando $ECODE.
  • Manejo avanzado de errores usando $ETRAP.

Al comprender e implementar estos mecanismos de manejo de errores, puedes asegurar que tus programas MUMPS puedan manejar situaciones inesperadas de manera elegante y continuar operando o fallar de manera segura.
