Introducción

En esta sección del curso sobre Metodologías Ágiles, realizaremos un ejercicio práctico para simular un sprint. Este ejercicio está diseñado para proporcionar una experiencia práctica en la planificación, ejecución y revisión de un sprint, siguiendo las mejores prácticas de Scrum.

Objetivos del Ejercicio

  • Comprender el ciclo completo de un sprint.
  • Practicar la planificación y estimación de tareas.
  • Experimentar con la ejecución y seguimiento del progreso.
  • Realizar una revisión y retrospectiva del sprint.

Materiales Necesarios

  • Herramienta de gestión de proyectos (JIRA, Trello, etc.)
  • Plantillas de planificación de sprint
  • Historias de usuario predefinidas
  • Equipo de trabajo (puede ser simulado)

Planificación del Sprint

Paso 1: Definición del Sprint

  • Duración del Sprint: 2 semanas
  • Objetivo del Sprint: Implementar y probar una nueva funcionalidad en la aplicación web.

Paso 2: Selección de Historias de Usuario

  • Historia de Usuario 1: Como usuario, quiero poder registrarme en la aplicación para acceder a funcionalidades exclusivas.
  • Historia de Usuario 2: Como usuario registrado, quiero poder iniciar sesión para acceder a mi cuenta.
  • Historia de Usuario 3: Como usuario, quiero poder restablecer mi contraseña en caso de olvidarla.

Paso 3: Estimación de Tareas

Utilizaremos la técnica de Planning Poker para estimar las tareas. Cada miembro del equipo asignará puntos de historia a cada tarea basada en su complejidad.

| Historia de Usuario | Tarea | Estimación (Puntos de Historia) | |---------------------|-------|---------------------------------| | Registro de Usuario | Crear formulario de registro | 3 | | | Validar datos de entrada | 2 | | | Guardar datos en la base de datos | 3 | | Inicio de Sesión | Crear formulario de inicio de sesión | 2 | | | Validar credenciales | 3 | | | Manejar sesiones de usuario | 3 | | Restablecimiento de Contraseña | Crear formulario de restablecimiento | 2 | | | Enviar correo de restablecimiento | 3 | | | Actualizar contraseña en la base de datos | 2 |

Ejecución del Sprint

Paso 4: Desarrollo

Cada miembro del equipo tomará una tarea del backlog y comenzará a trabajar en ella. Utilizaremos una herramienta de gestión de proyectos para seguir el progreso.

Ejemplo de Código: Validación de Datos de Registro

def validate_registration_data(username, email, password):
    if not username or not email or not password:
        return False, "All fields are required."
    if len(password) < 8:
        return False, "Password must be at least 8 characters long."
    if "@" not in email:
        return False, "Invalid email address."
    return True, "Validation successful."

Paso 5: Daily Stand-up

Cada día, el equipo se reunirá brevemente para discutir:

  • Lo que hicieron ayer.
  • Lo que planean hacer hoy.
  • Cualquier impedimento que estén enfrentando.

Revisión del Sprint

Paso 6: Sprint Review

Al final del sprint, el equipo presentará el trabajo completado a los stakeholders. Se demostrará la nueva funcionalidad implementada y se recogerá feedback.

Paso 7: Sprint Retrospective

El equipo se reunirá para discutir:

  • Lo que salió bien durante el sprint.
  • Lo que no salió bien.
  • Qué se puede mejorar para el próximo sprint.

Conclusión

Este ejercicio práctico de simulación de un sprint proporciona una experiencia completa del ciclo de vida de un sprint en Scrum. Al seguir estos pasos, los profesionales de TI pueden mejorar su comprensión y habilidades en la gestión de proyectos ágiles, lo que les permitirá aplicar estos conocimientos en sus entornos de trabajo reales.
