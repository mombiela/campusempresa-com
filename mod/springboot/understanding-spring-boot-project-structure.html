Entendiendo la Estructura de un Proyecto Spring Boot
Visión General de las Anotaciones de Spring Boot ►

En esta sección, exploraremos la estructura de un proyecto Spring Boot. Comprender la disposición y el propósito de cada componente es crucial para desarrollar, mantener y escalar tus aplicaciones de manera efectiva.

Componentes Clave de un Proyecto Spring Boot

Un proyecto típico de Spring Boot consta de varios componentes y directorios clave. Vamos a desglosar cada uno de estos:

1. `src/main/java`

Este directorio contiene el código fuente principal de tu aplicación.

  • Clase de Aplicación: Este es el punto de entrada de tu aplicación Spring Boot. Generalmente está anotado con @SpringBootApplication.

    package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

  • Clases Controladoras: Estas manejan las solicitudes HTTP entrantes y devuelven respuestas. Generalmente están anotadas con @RestController.

    package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
    @GetMapping("/hello")
    public String sayHello() {
        return "¡Hola, Mundo!";
    }
}

  • Clases de Servicio: Estas contienen la lógica de negocio y están anotadas con @Service.

    package com.example.demo.service;

import org.springframework.stereotype.Service;

@Service
public class HelloService {
    public String getGreeting() {
        return "¡Hola, Mundo!";
    }
}

  • Clases de Repositorio: Estas interactúan con la base de datos y están anotadas con @Repository.

    package com.example.demo.repository;

import org.springframework.data.jpa.repository.JpaRepository;
import com.example.demo.model.User;

public interface UserRepository extends JpaRepository<User, Long> {
}

2. `src/main/resources`

Este directorio contiene archivos de configuración y recursos estáticos.

  • application.properties o application.yml: Configuraciones de tu aplicación.

    server.port=8081
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=secret

  • Recursos Estáticos: Archivos como HTML, CSS y JavaScript que se sirven directamente al cliente.

    src/main/resources/static/
├── css/
├── js/
└── images/

  • Plantillas: Plantillas HTML utilizadas por Thymeleaf u otros motores de plantillas.

    src/main/resources/templates/
├── index.html
└── about.html

3. `src/test/java`

Este directorio contiene casos de prueba para tu aplicación.

  • Pruebas Unitarias: Prueban componentes individuales como servicios y controladores.

    package com.example.demo;

import static org.assertj.core.api.Assertions.assertThat;

import com.example.demo.controller.HelloController;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;

@SpringBootTest
public class DemoApplicationTests {

    @Autowired
    private HelloController helloController;

    @Test
    public void contextLoads() throws Exception {
        assertThat(helloController).isNotNull();
    }
}

  • Pruebas de Integración: Prueban la interacción entre múltiples componentes.

    package com.example.demo;

import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.get;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.status;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.content;

import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.web.servlet.AutoConfigureMockMvc;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.test.web.servlet.MockMvc;

@SpringBootTest
@AutoConfigureMockMvc
public class HelloControllerIntegrationTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    public void shouldReturnDefaultMessage() throws Exception {
        this.mockMvc.perform(get("/hello"))
            .andExpect(status().isOk())
            .andExpect(content().string("¡Hola, Mundo!"));
    }
}

4. `pom.xml` o `build.gradle`

Este archivo se utiliza para gestionar las dependencias del proyecto y las configuraciones de construcción.

  • Maven (pom.xml):

    <project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>demo</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <packaging>jar</packaging>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        <!-- Añadir otras dependencias aquí -->
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>

  • Gradle (build.gradle):

    plugins {
    id 'org.springframework.boot' version '2.5.4'
    id 'io.spring.dependency-management' version '1.0.11.RELEASE'
    id 'java'
}

group = 'com.example'
version = '0.0.1-SNAPSHOT'
sourceCompatibility = '11'

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    // Añadir otras dependencias aquí
}

test {
    useJUnitPlatform()
}

Resumen

Comprender la estructura de un proyecto Spring Boot es fundamental para un desarrollo eficiente. Aquí tienes un resumen rápido de lo que hemos cubierto:

  • src/main/java: Contiene el código principal de la aplicación, incluyendo controladores, servicios y repositorios.
  • src/main/resources: Contiene archivos de configuración y recursos estáticos.
  • src/test/java: Contiene pruebas unitarias y de integración.
  • Archivos de Construcción: pom.xml para Maven o build.gradle para Gradle, utilizados para gestionar dependencias y configuraciones de construcción.

Familiarizándote con estos componentes, podrás navegar y gestionar tus proyectos Spring Boot de manera más efectiva.
