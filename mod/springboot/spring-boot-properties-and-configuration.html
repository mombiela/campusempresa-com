Introducción

Spring Boot simplifica la configuración de aplicaciones Spring proporcionando una amplia gama de propiedades y opciones de configuración. Esta sección cubrirá cómo usar estas propiedades y configuraciones para personalizar tu aplicación Spring Boot.

Conceptos Clave

  • Propiedades de la Aplicación: Configuración centralizada usando application.properties o application.yml.
  • Configuración Externa: Cargar propiedades desde fuentes externas.
  • Perfiles: Configuraciones específicas del entorno.
  • Propiedades de Configuración: Vincular propiedades a objetos Java.
  • Propiedades Personalizadas: Definir y usar propiedades personalizadas.

Propiedades de la Aplicación

Spring Boot te permite definir propiedades en un archivo centralizado, típicamente application.properties o application.yml.

Usando `application.properties`

El archivo application.properties es un archivo de configuración simple de pares clave-valor.

server.port=8081
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=secret

Usando `application.yml`

El archivo application.yml usa el formato YAML, que es más legible para datos jerárquicos.

server:
  port: 8081

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: secret

Configuración Externa

Spring Boot puede cargar propiedades desde fuentes externas como variables de entorno, argumentos de línea de comandos y archivos de configuración externos.

Variables de Entorno

Puedes establecer variables de entorno para sobrescribir propiedades definidas en application.properties.

export SERVER_PORT=8082

Argumentos de Línea de Comandos

Puedes pasar propiedades como argumentos de línea de comandos al iniciar tu aplicación Spring Boot.

java -jar myapp.jar --server.port=8083

Perfiles

Los perfiles te permiten definir diferentes configuraciones para diferentes entornos (por ejemplo, desarrollo, pruebas, producción).

Definiendo Perfiles

Puedes definir archivos de propiedades específicos del perfil, como application-dev.properties y application-prod.properties.

# application-dev.properties
server.port=8081
spring.datasource.url=jdbc:mysql://localhost:3306/devdb

# application-prod.properties
server.port=8080
spring.datasource.url=jdbc:mysql://localhost:3306/proddb

Activando Perfiles

Puedes activar perfiles usando la propiedad spring.profiles.active.

# application.properties
spring.profiles.active=dev

Propiedades de Configuración

Spring Boot te permite vincular propiedades a objetos Java usando la anotación @ConfigurationProperties.

Ejemplo

Define una clase de propiedades de configuración:

import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {
    private String name;
    private String description;

    // getters y setters
}

Define propiedades en application.properties:

app.name=MyApp
app.description=This is my application.

Usa las propiedades en tu aplicación:

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class AppController {

    @Autowired
    private AppProperties appProperties;

    @GetMapping("/info")
    public String getAppInfo() {
        return appProperties.getName() + ": " + appProperties.getDescription();
    }
}

Propiedades Personalizadas

Puedes definir y usar propiedades personalizadas en tu aplicación Spring Boot.

Definiendo Propiedades Personalizadas

Agrega propiedades personalizadas a application.properties:

custom.property1=value1
custom.property2=value2

Accediendo a Propiedades Personalizadas

Usa la anotación @Value para inyectar propiedades personalizadas:

import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

@Component
public class CustomProperties {

    @Value("${custom.property1}")
    private String property1;

    @Value("${custom.property2}")
    private String property2;

    // getters y setters
}

Conclusión

Spring Boot proporciona una forma flexible y poderosa de gestionar propiedades y configuraciones de la aplicación. Al entender cómo usar application.properties o application.yml, configuraciones externas, perfiles, propiedades de configuración y propiedades personalizadas, puedes personalizar y gestionar eficazmente tus aplicaciones Spring Boot para diferentes entornos y requisitos.
