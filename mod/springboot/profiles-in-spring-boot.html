Introducción a los Perfiles

Los perfiles de Spring Boot proporcionan una forma de segregar partes de la configuración de tu aplicación y hacerla disponible solo en ciertos entornos. Esto es particularmente útil cuando necesitas ejecutar tu aplicación con diferentes configuraciones para entornos de desarrollo, prueba y producción.

Conceptos Clave

  • Perfil: Un agrupamiento lógico y nombrado de configuraciones.
  • Configuración Específica del Entorno: Diferentes configuraciones para diferentes entornos (por ejemplo, application-dev.properties, application-prod.properties).
  • Activación: Mecanismo para activar un perfil específico.

Creación y Uso de Perfiles

Definición de Perfiles

Los perfiles pueden definirse en archivos de propiedades o archivos YAML. Por ejemplo:

  • application-dev.properties
  • application-prod.properties

Ejemplo: Archivos de Propiedades

Crea diferentes archivos de propiedades para diferentes entornos.

application-dev.properties

server.port=8081
spring.datasource.url=jdbc:h2:mem:devdb
spring.datasource.username=sa
spring.datasource.password=password

application-prod.properties

server.port=8080
spring.datasource.url=jdbc:mysql://prod-db-server/mydb
spring.datasource.username=produser
spring.datasource.password=prodpassword

Activación de Perfiles

Los perfiles pueden activarse de varias maneras:

  • Línea de Comando: Usando -Dspring.profiles.active=dev
  • Variable de Entorno: Configurando SPRING_PROFILES_ACTIVE=dev
  • Propiedades de la Aplicación: Añadiendo spring.profiles.active=dev en application.properties

Ejemplo: Activación de Perfil vía Línea de Comando

java -jar myapp.jar --spring.profiles.active=dev

Ejemplo: Activación de Perfil vía Variable de Entorno

export SPRING_PROFILES_ACTIVE=dev

Uso de Perfiles en el Código

También puedes usar perfiles directamente en tu código de Spring Boot para cargar condicionalmente beans o configuraciones.

Ejemplo: Creación Condicional de Beans

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Profile;

@Configuration
public class DataSourceConfig {

    @Bean
    @Profile("dev")
    public DataSource devDataSource() {
        return new H2DataSource("jdbc:h2:mem:devdb", "sa", "password");
    }

    @Bean
    @Profile("prod")
    public DataSource prodDataSource() {
        return new MySQLDataSource("jdbc:mysql://prod-db-server/mydb", "produser", "prodpassword");
    }
}

Gestión Avanzada de Perfiles

Múltiples Perfiles

Puedes activar múltiples perfiles simultáneamente separándolos con comas.

Ejemplo: Activación de Múltiples Perfiles

java -jar myapp.jar --spring.profiles.active=dev,debug

Beans Específicos de Perfil

Puedes definir beans que solo deberían estar disponibles en ciertos perfiles usando la anotación @Profile.

Ejemplo: Beans Específicos de Perfil

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Profile;

@Configuration
public class AppConfig {

    @Bean
    @Profile("test")
    public TestService testService() {
        return new TestService();
    }

    @Bean
    @Profile("prod")
    public ProdService prodService() {
        return new ProdService();
    }
}

Resumen

Los perfiles de Spring Boot te permiten gestionar fácilmente diferentes configuraciones para diferentes entornos. Al definir archivos de propiedades específicos del entorno y activar perfiles a través de varios métodos, puedes asegurarte de que tu aplicación se comporte correctamente en entornos de desarrollo, prueba y producción. Los perfiles también pueden usarse directamente en tu código para cargar condicionalmente beans y configuraciones, proporcionando una forma flexible y poderosa de gestionar las configuraciones de la aplicación.

Al dominar los perfiles, puedes crear aplicaciones Spring Boot más robustas y adaptables que sean más fáciles de gestionar y desplegar en diferentes entornos.
