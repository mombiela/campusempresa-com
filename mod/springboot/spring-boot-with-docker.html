Introducción a Docker

  • ¿Qué es Docker?
    • Docker es una plataforma que permite a los desarrolladores automatizar el despliegue de aplicaciones dentro de contenedores ligeros y portátiles.
  • ¿Por qué usar Docker con Spring Boot?
    • Simplifica el despliegue.
    • Asegura consistencia a través de diferentes entornos.
    • Facilita la escalabilidad y gestión de aplicaciones.

Configuración de Docker

  • Instalación de Docker
    • Sigue la guía oficial de instalación de Docker para tu sistema operativo: Instalación de Docker
  • Comandos básicos de Docker
    • docker --version: Verificar la versión de Docker.
    • docker run hello-world: Ejecutar un contenedor de prueba para verificar la instalación.

Creación de una aplicación Spring Boot

  • Inicializar un proyecto Spring Boot
    • Usa Spring Initializr para crear un nuevo proyecto Spring Boot con las dependencias necesarias.
    • Ejemplo de dependencias: Spring Web, Spring Data JPA, H2 Database.
// Ejemplo de aplicación Spring Boot
package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

Dockerizar una aplicación Spring Boot

  • Creación de un Dockerfile
    • Un Dockerfile es un script que contiene una serie de instrucciones sobre cómo construir una imagen Docker para tu aplicación.
# Usar una imagen oficial de JDK como imagen base
FROM openjdk:11-jdk-slim

# Establecer el directorio de trabajo en el contenedor
WORKDIR /app

# Copiar el archivo JAR ejecutable dentro del contenedor
COPY target/demo-0.0.1-SNAPSHOT.jar app.jar

# Ejecutar el archivo JAR
ENTRYPOINT ["java", "-jar", "app.jar"]

  • Construcción de la imagen Docker

    • Navega al directorio que contiene el Dockerfile y ejecuta el siguiente comando: 
      docker build -t demo-app .

  • Ejecución del contenedor Docker

    • Ejecuta el contenedor Docker usando la imagen que acabas de construir: 
      docker run -p 8080:8080 demo-app

Gestión de contenedores Docker

  • Comandos comunes de Docker
    • docker ps: Listar contenedores en ejecución.
    • docker stop <container_id>: Detener un contenedor en ejecución.
    • docker rm <container_id>: Eliminar un contenedor detenido.
    • docker images: Listar todas las imágenes Docker.
    • docker rmi <image_id>: Eliminar una imagen Docker.

Conceptos avanzados de Docker

  • Docker Compose
    • Docker Compose es una herramienta para definir y ejecutar aplicaciones Docker de múltiples contenedores.
    • Ejemplo de docker-compose.yml para una aplicación Spring Boot con una base de datos PostgreSQL:
version: '3.8'
services:
  app:
    image: demo-app
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8080:8080"
    depends_on:
      - db
  db:
    image: postgres:13
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: demo
    ports:
      - "5432:5432"
  • Escalado de servicios
    • Docker Compose te permite escalar servicios usando la bandera --scale: 
      docker-compose up --scale app=3

Conclusión

  • Resumen

    • Docker simplifica el despliegue y la gestión de aplicaciones Spring Boot.
    • Al contenerizar tu aplicación, aseguras consistencia a través de diferentes entornos.
    • Docker Compose facilita la gestión de aplicaciones de múltiples contenedores, haciendo más fácil escalar y gestionar dependencias.

  • Próximos pasos

    • Explora características más avanzadas de Docker como redes, volúmenes y Docker Swarm.
    • Integra Docker con pipelines de CI/CD para despliegue automatizado.
    • Experimenta con Kubernetes para orquestar contenedores Docker a gran escala.
