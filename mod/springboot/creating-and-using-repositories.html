Creación y Uso de Repositorios en Spring Boot

Introducción

En esta sección, exploraremos cómo crear y usar repositorios en Spring Boot. Los repositorios son una parte crucial del módulo Spring Data JPA, que simplifica la implementación de capas de acceso a datos proporcionando un conjunto de interfaces y anotaciones.

Conceptos Clave

  • Spring Data JPA: Parte de la familia más grande de Spring Data, Spring Data JPA facilita la implementación de repositorios basados en JPA.
  • Interfaz de Repositorio: Una interfaz que extiende una de las interfaces de repositorio de Spring Data (por ejemplo, CrudRepository, JpaRepository).
  • Entidad: Una clase anotada con @Entity que representa una tabla en la base de datos.
  • Operaciones CRUD: Operaciones básicas (Crear, Leer, Actualizar, Eliminar) proporcionadas por los repositorios de Spring Data.

Configuración del Proyecto

Antes de profundizar en la creación de repositorios, asegúrate de tener un proyecto Spring Boot configurado con las dependencias necesarias.

Dependencias

Agrega las siguientes dependencias a tu pom.xml si estás usando Maven:

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>
    <dependency>
        <groupId>com.h2database</groupId>
        <artifactId>h2</artifactId>
        <scope>runtime</scope>
    </dependency>
</dependencies>

Creación de una Entidad

Primero, vamos a crear una clase de entidad que se mapee a una tabla de la base de datos.

import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;

@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private String email;

    // Getters y Setters
}
  • @Entity: Especifica que la clase es una entidad y está mapeada a una tabla de la base de datos.
  • @Id: Especifica la clave primaria de una entidad.
  • @GeneratedValue: Proporciona la estrategia de generación para la clave primaria.

Creación de una Interfaz de Repositorio

A continuación, crea una interfaz de repositorio para la entidad User.

import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository<User, Long> {
}
  • JpaRepository<User, Long>: Extiende la interfaz JpaRepository, proporcionando operaciones CRUD para la entidad User. El primer parámetro es el tipo de entidad, y el segundo es el tipo de la clave primaria.

Uso del Repositorio

Ahora, vamos a usar el UserRepository en una clase de servicio para realizar operaciones CRUD.

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Optional;

@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;

    public User createUser(User user) {
        return userRepository.save(user);
    }

    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    public Optional<User> getUserById(Long id) {
        return userRepository.findById(id);
    }

    public User updateUser(Long id, User userDetails) {
        User user = userRepository.findById(id).orElseThrow(() -> new RuntimeException("User not found"));
        user.setName(userDetails.getName());
        user.setEmail(userDetails.getEmail());
        return userRepository.save(user);
    }

    public void deleteUser(Long id) {
        userRepository.deleteById(id);
    }
}
  • @Autowired: Inyecta el bean UserRepository.
  • save(): Guarda una entidad dada.
  • findAll(): Recupera todas las entidades.
  • findById(): Recupera una entidad por su ID.
  • deleteById(): Elimina una entidad por su ID.

Funcionalidades Avanzadas del Repositorio

Spring Data JPA proporciona varias funcionalidades avanzadas para mejorar la funcionalidad del repositorio.

Métodos de Consulta Personalizados

Puedes definir métodos de consulta personalizados en tu interfaz de repositorio.

import java.util.List;

public interface UserRepository extends JpaRepository<User, Long> {
    List<User> findByName(String name);
}
  • findByName(String name): Encuentra usuarios por su nombre. Spring Data JPA genera automáticamente la consulta basada en el nombre del método.

Consultas JPQL y Nativas

También puedes usar JPQL (Java Persistence Query Language) o consultas SQL nativas.

import org.springframework.data.jpa.repository.Query;

public interface UserRepository extends JpaRepository<User, Long> {
    @Query("SELECT u FROM User u WHERE u.email = ?1")
    User findByEmail(String email);

    @Query(value = "SELECT * FROM User WHERE email = ?1", nativeQuery = true)
    User findByEmailNative(String email);
}
  • @Query: Define una consulta JPQL o SQL nativa.

Conclusión

En esta sección, cubrimos los conceptos básicos de la creación y uso de repositorios en Spring Boot. Comenzamos con la configuración del proyecto, la creación de una entidad y luego la creación de una interfaz de repositorio. También exploramos cómo usar el repositorio en una clase de servicio y vimos algunas características avanzadas como métodos de consulta personalizados y consultas JPQL/nativas. Entender estos conceptos es crucial para construir capas de acceso a datos robustas y mantenibles en tus aplicaciones Spring Boot.
