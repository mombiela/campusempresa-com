Introducción

El estilo y formato del código son aspectos cruciales para escribir código limpio, legible y mantenible. En Rust, adherirse a un estilo consistente ayuda a asegurar que tu código sea comprensible no solo para ti, sino también para otros desarrolladores que puedan trabajar en tu base de código. Esta sección cubrirá los principios clave del estilo y formato del código en Rust, incluyendo convenciones de nomenclatura, indentación y el uso de la herramienta de formato integrada de Rust, rustfmt.

Convenciones de Nomenclatura

Las convenciones de nomenclatura en Rust ayudan a que el código sea más legible y consistente. Aquí están las convenciones clave:

  • Variables y Funciones: Usa snake_case.

    let my_variable = 10;
fn my_function() {
    // cuerpo de la función
}

  • Constantes y Estáticos: Usa SCREAMING_SNAKE_CASE.

    const MAX_SIZE: u32 = 100;
static GLOBAL_COUNT: u32 = 0;

  • Structs, Enums y Traits: Usa CamelCase.

    struct MyStruct {
    field: u32,
}

enum MyEnum {
    VariantOne,
    VariantTwo,
}

trait MyTrait {
    fn do_something(&self);
}

Indentación y Espaciado

La correcta indentación y espaciado hacen que tu código sea más legible. Rust usa 4 espacios por nivel de indentación.

  • Indentación: Usa 4 espacios, no tabs.

    fn main() {
    let x = 5;
    if x > 0 {
        println!("x es positivo");
    }
}

  • Espaciado: Usa espacios alrededor de los operadores y después de las comas.

    let sum = a + b;
let array = [1, 2, 3, 4];

Longitud de Línea

Mantén las líneas con un máximo de 100 caracteres. Esto ayuda a asegurar que el código sea legible en varios tamaños de pantalla y en diferentes entornos.

  • Ejemplo: 
    // Esta línea tiene exactamente 100 caracteres de largo
let long_variable_name = "Esta es una cadena muy larga que debería dividirse en múltiples líneas si excede los 100 caracteres.";

Comentarios

Los comentarios son esenciales para explicar el propósito del código. Rust soporta dos tipos de comentarios:

  • Comentarios de Línea: Usa // para comentarios de una sola línea.

    // Este es un comentario de una sola línea
let x = 5; // Este es un comentario en línea

  • Comentarios de Bloque: Usa /* ... */ para comentarios de múltiples líneas.

    /*
 * Este es un comentario de bloque.
 * Puede abarcar múltiples líneas.
 */
let y = 10;

Usando `rustfmt`

Rust proporciona una herramienta integrada llamada rustfmt para formatear automáticamente tu código según las guías de estilo oficiales de Rust. Esta herramienta ayuda a mantener la consistencia y legibilidad en toda tu base de código.

  • Instalación:

    rustup component add rustfmt

  • Uso:

    cargo fmt

Ejemplo de Código Bien Formateado

Aquí hay un ejemplo de un programa en Rust bien formateado:

// Este es un programa simple en Rust que demuestra el estilo y formato de código adecuado.

fn main() {
    let numbers = vec![1, 2, 3, 4, 5];

    for number in &numbers {
        if is_even(*number) {
            println!("{} es par", number);
        } else {
            println!("{} es impar", number);
        }
    }
}

fn is_even(num: i32) -> bool {
    num % 2 == 0
}

Conclusión

Adherirse a guías de estilo y formato de código consistentes es esencial para escribir código en Rust que sea limpio, legible y mantenible. Siguiendo las convenciones de nomenclatura, reglas de indentación y usando herramientas como rustfmt, puedes asegurar que tu código sea fácil de entender y trabajar con él. Recuerda, el código bien formateado no solo te beneficia a ti, sino también a tus colaboradores y futuros mantenedores de la base de código.
