Ejecutando Pruebas
Depuración ►

Las pruebas son una parte crucial del desarrollo de software, asegurando que tu código se comporte como se espera. Rust proporciona un potente marco de pruebas integrado en el lenguaje. Esta sección te guiará a través de los conceptos básicos de escribir y ejecutar pruebas en Rust, progresando hacia técnicas de prueba más avanzadas.

Introducción a las Pruebas en Rust

El marco de pruebas de Rust está integrado en el lenguaje, lo que facilita escribir y ejecutar pruebas. Las pruebas son funciones que verifican la corrección de tu código.

  • Funciones de Prueba: Funciones anotadas con el atributo #[test].
  • Aserciones de Prueba: Macros como assert!, assert_eq! y assert_ne! para verificar condiciones.

Ejemplo: Función de Prueba Básica

Aquí tienes un ejemplo simple de una función de prueba:

// src/lib.rs

pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
    }
}
  • #[cfg(test)]: Este atributo asegura que el módulo de prueba solo se compile al ejecutar pruebas.
  • #[test]: Marca la función como una función de prueba.
  • assert_eq!: Verifica si los dos argumentos son iguales.

Ejecutando Pruebas

Para ejecutar pruebas, usa el comando cargo test. Este comando compila tu código en modo de prueba y ejecuta todas las funciones de prueba.

$ cargo test

Salida de `cargo test`

Cuando ejecutas cargo test, verás una salida similar a esta:

running 1 test
test tests::test_add ... ok

test result: ok. 1 passed; 0 failed; 0 ignored; 0 measured; 0 filtered out
  • Resumen de Pruebas: Muestra el número de pruebas pasadas, fallidas, ignoradas, etc.

Escribiendo Pruebas Más Complejas

A medida que tu base de código crezca, necesitarás pruebas más complejas. Rust soporta varias estrategias de prueba:

  • Pruebas Unitarias: Prueban unidades individuales de código (funciones, métodos).
  • Pruebas de Integración: Prueban la interacción entre múltiples partes de tu código.

Pruebas Unitarias

Las pruebas unitarias se escriben en el mismo archivo que el código que prueban, típicamente dentro de un módulo tests.

// src/lib.rs

pub fn multiply(a: i32, b: i32) -> i32 {
    a * b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_multiply() {
        assert_eq!(multiply(2, 3), 6);
    }
}

Pruebas de Integración

Las pruebas de integración se colocan en el directorio tests en la raíz de tu proyecto. Cada archivo en este directorio se compila como un crate separado.

// tests/integration_test.rs

extern crate my_crate;

#[test]
fn test_integration() {
    assert_eq!(my_crate::add(2, 3), 5);
}
  • extern crate: Se usa para traer la biblioteca al alcance.

Técnicas Avanzadas de Prueba

Ignorando Pruebas

A veces puede que quieras deshabilitar temporalmente una prueba. Usa el atributo #[ignore].

#[test]
#[ignore]
fn expensive_test() {
    // código que toma mucho tiempo en ejecutarse
}

Ejecutando Pruebas Específicas

Para ejecutar una prueba específica, usa la bandera --test seguida del nombre de la prueba.

$ cargo test test_add

Fixtures de Prueba

Los fixtures de prueba son configuraciones comunes para múltiples pruebas. Rust no tiene soporte incorporado para fixtures, pero puedes lograr esto usando funciones de configuración.

#[cfg(test)]
mod tests {
    use super::*;

    fn setup() -> i32 {
        // código de configuración
        42
    }

    #[test]
    fn test_with_setup() {
        let value = setup();
        assert_eq!(value, 42);
    }
}

Conclusión

Las pruebas son una parte integral de la programación en Rust, asegurando que tu código sea confiable y mantenible. El marco de pruebas integrado de Rust facilita escribir y ejecutar pruebas, desde simples pruebas unitarias hasta complejas pruebas de integración. Al dominar estas técnicas de prueba, estarás bien equipado para desarrollar aplicaciones robustas en Rust.

  • Puntos Clave:
    • Usa #[test] para marcar funciones de prueba.
    • Ejecuta pruebas con cargo test.
    • Escribe pruebas unitarias en el mismo archivo y pruebas de integración en el directorio tests.
    • Usa técnicas avanzadas como ignorar pruebas y configurar fixtures para escenarios más complejos.
