Manejo de Errores con Result
Manejo de Errores con Option ►

El manejo de errores es un aspecto crucial de la programación, asegurando que tu código pueda manejar situaciones inesperadas de manera elegante. En Rust, el tipo Result es una herramienta poderosa para gestionar errores. Este tema te guiará desde los conceptos básicos hasta el uso avanzado de Result para el manejo de errores en Rust.

Introducción a `Result`

El tipo Result es un enum que representa éxito (Ok) o fracaso (Err). Se define como:

enum Result<T, E> {
    Ok(T),
    Err(E),
}
  • T: El tipo del valor devuelto en caso de éxito.
  • E: El tipo del error devuelto en caso de fallo.

Uso Básico

Comencemos con un ejemplo simple:

fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("División por cero"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    match divide(4.0, 2.0) {
        Ok(result) => println!("Resultado: {}", result),
        Err(e) => println!("Error: {}", e),
    }

    match divide(4.0, 0.0) {
        Ok(result) => println!("Resultado: {}", result),
        Err(e) => println!("Error: {}", e),
    }
}

En este ejemplo:

  • La función divide devuelve un Result<f64, String>.
  • Si la división es exitosa, devuelve Ok(result).
  • Si hay un error (división por cero), devuelve Err(error_message).

Manejo de `Result` con `match`

La declaración match es una forma común de manejar Result:

fn main() {
    let result = divide(10.0, 2.0);

    match result {
        Ok(value) => println!("Éxito: {}", value),
        Err(error) => println!("Error: {}", error),
    }
}

Usando `unwrap` y `expect`

Para prototipos rápidos, puedes usar unwrap y expect para manejar Result:

  • unwrap: Devuelve el valor dentro de Ok o entra en pánico si es Err.
  • expect: Similar a unwrap, pero te permite proporcionar un mensaje de pánico personalizado.
fn main() {
    let result = divide(10.0, 2.0).unwrap();
    println!("Resultado: {}", result);

    let result = divide(10.0, 0.0).expect("Fallo al dividir");
}

Nota: Usar unwrap y expect no se recomienda para código de producción ya que pueden causar que tu programa entre en pánico.

Manejo Avanzado de Errores

Encadenando Métodos con `and_then` y `or_else`

Puedes encadenar múltiples operaciones que devuelven Result usando and_then y or_else:

  • and_then: Encadena otra operación si el resultado es Ok.
  • or_else: Encadena otra operación si el resultado es Err.
fn add_one(x: f64) -> Result<f64, String> {
    Ok(x + 1.0)
}

fn main() {
    let result = divide(10.0, 2.0)
        .and_then(add_one)
        .or_else(|e| Err(format!("Fallo: {}", e)));

    match result {
        Ok(value) => println!("Éxito: {}", value),
        Err(error) => println!("Error: {}", error),
    }
}

Usando el Operador `?`

El operador ? es una abreviatura para propagar errores. Solo se puede usar en funciones que devuelven Result.

fn divide_and_add(a: f64, b: f64) -> Result<f64, String> {
    let result = divide(a, b)?;
    add_one(result)
}

fn main() {
    match divide_and_add(10.0, 2.0) {
        Ok(value) => println!("Éxito: {}", value),
        Err(error) => println!("Error: {}", error),
    }
}

Tipos de Error Personalizados

Para aplicaciones más complejas, podrías querer definir tipos de error personalizados:

#[derive(Debug)]
enum MathError {
    DivisionByZero,
    NegativeLogarithm,
}

fn divide(a: f64, b: f64) -> Result<f64, MathError> {
    if b == 0.0 {
        Err(MathError::DivisionByZero)
    } else {
        Ok(a / b)
    }
}

fn main() {
    match divide(10.0, 0.0) {
        Ok(value) => println!("Éxito: {}", value),
        Err(error) => println!("Error: {:?}", error),
    }
}

Resumen

  • El tipo Result es esencial para el manejo de errores en Rust.
  • Usa match para manejar Result explícitamente.
  • unwrap y expect pueden usarse para prototipos rápidos pero no se recomiendan para producción.
  • El operador ? simplifica la propagación de errores.
  • Los tipos de error personalizados proporcionan más flexibilidad y claridad en el manejo de errores.

Dominando Result, puedes escribir código en Rust robusto y resistente a errores.
