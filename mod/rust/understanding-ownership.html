Entendiendo la Propiedad
Referencias y Préstamo ►

La propiedad es una de las características más únicas y poderosas de Rust. Permite a Rust gestionar la memoria de manera segura y eficiente sin un recolector de basura. Este tema te guiará a través de los conceptos básicos de propiedad, préstamo y tiempos de vida, proporcionándote una base sólida para escribir código seguro y eficiente en Rust.

Conceptos Clave de la Propiedad

  • Propiedad: Cada valor en Rust tiene una variable que se llama su propietario.
  • Préstamo: Referencias a un valor sin tomar posesión.
  • Tiempos de Vida: Asegurando que las referencias sean válidas mientras se usen.

Reglas de la Propiedad

  1. Cada valor en Rust tiene un solo propietario.
  2. Cuando el propietario sale del ámbito, el valor se elimina.
  3. La propiedad puede ser transferida (movida) a otra variable.

Ejemplo: Propiedad Básica

fn main() {
    let s1 = String::from("hello"); // s1 posee el String
    let s2 = s1; // La propiedad del String se mueve a s2
    // println!("{}", s1); // Esto causaría un error en tiempo de compilación porque s1 ya no posee el String
    println!("{}", s2); // Esto es válido
}

En este ejemplo:

  • s1 inicialmente posee el String.
  • La propiedad se transfiere a s2 cuando s1 se asigna a s2.
  • Intentar usar s1 después de la transferencia resulta en un error en tiempo de compilación.

Préstamo

El préstamo te permite referenciar un valor sin tomar posesión. Esto se hace usando referencias (&).

Ejemplo: Préstamo

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1); // Prestando s1
    println!("La longitud de '{}' es {}.", s1, len); // s1 aún puede ser usado aquí
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

En este ejemplo:

  • s1 es prestado por la función calculate_length.
  • s1 aún puede ser usado después de la llamada a la función porque la propiedad no se transfiere.

Préstamo Mutable

Rust impone que puedes tener una referencia mutable o cualquier número de referencias inmutables, pero no ambas simultáneamente.

Ejemplo: Préstamo Mutable

fn main() {
    let mut s = String::from("hello");
    change(&mut s); // Préstamo mutable
    println!("{}", s); // s aún puede ser usado aquí
}

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}

En este ejemplo:

  • s es mutable y prestado de manera mutable por change.
  • change modifica el String añadiendo ", world".

Tiempos de Vida

Los tiempos de vida aseguran que las referencias sean válidas mientras se usen. Rust usa tiempos de vida para prevenir referencias colgantes.

Ejemplo: Tiempos de Vida

fn main() {
    let r;
    {
        let x = 5;
        r = &x;
    }
    // println!("r: {}", r); // Esto causaría un error en tiempo de compilación
}

En este ejemplo:

  • x sale del ámbito antes de que r sea usado, llevando a una referencia colgante.
  • Rust previene esto imponiendo tiempos de vida.

Resumen

  • Propiedad: Asegura que cada valor tenga un solo propietario, y los valores se eliminan cuando el propietario sale del ámbito.
  • Préstamo: Permite referencias a un valor sin tomar posesión, habilitando acceso seguro.
  • Tiempos de Vida: Aseguran que las referencias sean válidas mientras se usen, previniendo referencias colgantes.

Entender estos conceptos es crucial para escribir código seguro y eficiente en Rust. Practica estos conceptos con varios ejemplos para solidificar tu comprensión.
