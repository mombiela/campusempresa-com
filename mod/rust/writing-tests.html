Escribir Pruebas en Rust
Ejecutando Pruebas ►

Las pruebas son una parte crucial del desarrollo de software. Aseguran que tu código funcione como se espera y ayudan a prevenir errores. Rust tiene un marco de pruebas incorporado que hace que escribir y ejecutar pruebas sea sencillo. Esta sección te guiará a través de la escritura de pruebas en Rust, desde niveles básicos hasta avanzados.

Introducción a las Pruebas en Rust

  • Por Qué las Pruebas son Importantes: Las pruebas ayudan a detectar errores temprano, aseguran la fiabilidad del código y mejoran la mantenibilidad.
  • El Marco de Pruebas de Rust: Rust incluye un marco de pruebas incorporado que se integra perfectamente con Cargo, el gestor de paquetes y sistema de construcción de Rust.

Escribir Pruebas Básicas

Crear una Función de Prueba

En Rust, las funciones de prueba se anotan con el atributo #[test]. Aquí hay un ejemplo simple:

#[cfg(test)]
mod tests {
    #[test]
    fn it_works() {
        let result = 2 + 2;
        assert_eq!(result, 4);
    }
}
  • Explicación:
    • #[cfg(test)]: Este atributo asegura que el módulo de prueba solo se compile al ejecutar pruebas.
    • #[test]: Marca la función como una función de prueba.
    • assert_eq!: Un macro que verifica si dos valores son iguales. Si no lo son, la prueba falla.

Ejecutar Pruebas

Para ejecutar pruebas, usa el siguiente comando:

cargo test

Este comando compila tu código en modo de prueba y ejecuta todas las funciones de prueba.

Escribir Pruebas Más Complejas

Probar Panics

A veces, quieres asegurarte de que una función genere un pánico bajo ciertas condiciones. Usa el atributo #[should_panic] para esto:

#[cfg(test)]
mod tests {
    #[test]
    #[should_panic]
    fn it_panics() {
        panic!("Esta función debería generar un pánico");
    }
}
  • Explicación:
    • #[should_panic]: Marca la prueba como una que debería generar un pánico. Si la función no genera un pánico, la prueba falla.

Usar Result en Pruebas

Las funciones de prueba pueden devolver Result<T, E>, permitiendo el uso del operador ? para el manejo de errores:

#[cfg(test)]
mod tests {
    #[test]
    fn it_works_with_result() -> Result<(), String> {
        if 2 + 2 == 4 {
            Ok(())
        } else {
            Err(String::from("Las matemáticas están rotas"))
        }
    }
}
  • Explicación:
    • Devolver Result<(), String> permite el uso de ? para un manejo de errores conciso.
    • La prueba pasa si se devuelve Ok(()) y falla si se devuelve Err.

Organizar Pruebas

Módulos de Prueba

Organiza las pruebas en módulos para mantener tu código limpio y mantenible:

#[cfg(test)]
mod tests {
    mod math_tests {
        #[test]
        fn addition_works() {
            assert_eq!(2 + 2, 4);
        }
    }

    mod string_tests {
        #[test]
        fn string_contains() {
            assert!(String::from("hello").contains("ell"));
        }
    }
}
  • Explicación:
    • Agrupa pruebas relacionadas en submódulos para una mejor organización.

Pruebas de Integración

Las pruebas de integración se colocan en el directorio tests en la raíz de tu proyecto. Estas pruebas son externas a tu biblioteca y prueban la API pública:

// tests/integration_test.rs
use my_crate;

#[test]
fn it_adds_two() {
    assert_eq!(my_crate::add_two(2), 4);
}
  • Explicación:
    • Las pruebas de integración usan la API pública de tu crate.
    • Colócalas en el directorio tests.

Técnicas Avanzadas de Pruebas

Mocking y Stubbing

Rust no tiene soporte incorporado para mocking, pero puedes usar crates como mockall o mockito para este propósito.

Pruebas Basadas en Propiedades

Las pruebas basadas en propiedades verifican que una propiedad se mantenga para una amplia gama de entradas. El crate proptest se usa comúnmente para esto:

use proptest::prelude::*;

proptest! {
    #[test]
    fn test_addition(a in 0..1000, b in 0..1000) {
        assert_eq!(a + b, b + a);
    }
}
  • Explicación:
    • El macro proptest! genera pruebas para un rango de entradas.
    • Asegura que la propiedad (conmutatividad de la adición) se mantenga para todas las entradas generadas.

Conclusión

Las pruebas son una parte esencial de la programación en Rust. Al escribir pruebas, aseguras que tu código sea fiable y mantenible. Comienza con pruebas básicas usando #[test] y assert_eq!, luego avanza a escenarios más complejos como probar panics y usar Result<T, E>. Organiza tus pruebas en módulos y usa pruebas de integración para probar tu API pública. Para pruebas avanzadas, explora mocking, stubbing y pruebas basadas en propiedades.

Al incorporar estas prácticas, puedes escribir código Rust robusto y fiable.
