Introducción a Modelos y Bases de Datos

En Django, los modelos son clases de Python que representan tablas de bases de datos. Cada atributo del modelo corresponde a un campo de la base de datos. El Mapeo Objeto-Relacional (ORM) de Django te permite interactuar con la base de datos usando código Python en lugar de escribir consultas SQL.

Creación de Modelos

Para definir un modelo, creas una clase que hereda de django.db.models.Model. Cada atributo de la clase representa un campo de la base de datos.

from django.db import models

class Author(models.Model):
    name = models.CharField(max_length=100)
    birth_date = models.DateField()

Conceptos Clave

  • Clase Modelo: Una clase de Python que representa una tabla de base de datos.
  • Tipos de Campos: Django proporciona varios tipos de campos como CharField, DateField, IntegerField, etc.
  • Clase Meta: Se usa para definir metadatos para el modelo, como el orden y los nombres descriptivos.

Tipos de Campos

Django ofrece una variedad de tipos de campos para adaptarse a diferentes tipos de datos.

Tipo de Campo Descripción
CharField Para cadenas de tamaño pequeño a grande
TextField Para campos de texto grandes
IntegerField Para valores enteros
FloatField Para números de punto flotante
DateField Para valores de fecha
DateTimeField Para valores de fecha y hora
BooleanField Para valores verdadero/falso
ForeignKey Para crear relaciones de muchos a uno

Ejemplo

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(Author, on_delete=models.CASCADE)
    published_date = models.DateField()
    price = models.FloatField()
    in_stock = models.BooleanField(default=True)

Migraciones

Las migraciones son la forma en que Django propaga los cambios que realizas en tus modelos al esquema de la base de datos.

Crear y Aplicar Migraciones

  1. Crear Migraciones: Generar archivos de migración basados en los cambios en tus modelos. 
    python manage.py makemigrations
  2. Aplicar Migraciones: Aplicar las migraciones generadas a la base de datos. 
    python manage.py migrate

Ejemplo

# Crear una nueva migración basada en los cambios detectados en tus modelos
python manage.py makemigrations

# Aplicar la migración a la base de datos
python manage.py migrate

Consultas a la Base de Datos

El ORM de Django proporciona una API poderosa para consultar la base de datos.

Consultas Básicas

  • Todos los Objetos: Recuperar todos los objetos de un modelo. 
    books = Book.objects.all()
  • Filtrar Objetos: Recuperar objetos que coinciden con ciertos criterios. 
    books = Book.objects.filter(author__name='John Doe')
  • Obtener un Solo Objeto: Recuperar un solo objeto que coincida con los criterios. 
    book = Book.objects.get(id=1)

Consultas Avanzadas

  • Encadenar Filtros: Combinar múltiples filtros. 
    books = Book.objects.filter(author__name='John Doe').filter(in_stock=True)
  • Excluir Objetos: Excluir objetos que coinciden con ciertos criterios. 
    books = Book.objects.exclude(price__gt=20.00)

Ejemplo

# Recuperar todos los libros
all_books = Book.objects.all()

# Recuperar libros de un autor específico
john_doe_books = Book.objects.filter(author__name='John Doe')

# Recuperar un solo libro por su ID
book = Book.objects.get(id=1)

# Recuperar libros que están en stock y son del autor John Doe
in_stock_books = Book.objects.filter(author__name='John Doe', in_stock=True)

# Excluir libros que tienen un precio superior a $20
affordable_books = Book.objects.exclude(price__gt=20.00)

Conclusión

En esta sección, cubrimos los conceptos básicos de los modelos y bases de datos en Django. Aprendiste cómo crear modelos, definir campos y realizar consultas básicas y avanzadas usando el ORM de Django. Entender estos conceptos es crucial para construir aplicaciones web robustas y escalables con Django.
