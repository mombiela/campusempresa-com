Introducción

En Django, las plantillas se utilizan para generar contenido HTML dinámico. Las plantillas pueden incluir variables y lógica para mostrar datos de tus vistas. Esta sección cubrirá cómo pasar contexto a las plantillas y cómo usar etiquetas de plantilla para agregar lógica a tus plantillas.

Contexto en Plantillas de Django

El contexto es un diccionario que asigna nombres de variables a objetos de Python. Este diccionario se pasa al motor de plantillas para renderizar la plantilla con datos dinámicos.

Pasar Contexto desde Vistas

Para pasar contexto a una plantilla, necesitas incluirlo en la función render en tu vista.

from django.shortcuts import render

def my_view(request):
    context = {
        'name': 'John Doe',
        'age': 30,
        'hobbies': ['Reading', 'Cycling', 'Hiking']
    }
    return render(request, 'my_template.html', context)

En el ejemplo anterior, el diccionario context contiene tres claves: name, age y hobbies. Estas pueden ser accedidas en la plantilla.

Acceder al Contexto en Plantillas

En tu plantilla, puedes acceder a las variables del contexto usando dobles llaves {{ }}.

<!DOCTYPE html>
<html>
<head>
    <title>My Template</title>
</head>
<body>
    <h1>Hello, {{ name }}!</h1>
    <p>You are {{ age }} years old.</p>
    <h2>Your Hobbies:</h2>
    <ul>
        {% for hobby in hobbies %}
            <li>{{ hobby }}</li>
        {% endfor %}
    </ul>
</body>
</html>

Etiquetas de Plantilla

Las etiquetas de plantilla proporcionan lógica en la plantilla. Están encerradas en {% %} y pueden ser usadas para bucles, condicionales y más.

Etiquetas de Plantilla Comunes

Aquí hay algunas etiquetas de plantilla comúnmente usadas:

  • {% for %}: Iterar sobre una secuencia.
  • {% if %}: Declaración condicional.
  • {% block %}: Definir un bloque que puede ser sobrescrito en plantillas hijas.
  • {% extends %}: Extender una plantilla base.
  • {% include %}: Incluir otra plantilla.

Ejemplos de Etiquetas de Plantilla

Iterar con {% for %}

<ul>
    {% for item in item_list %}
        <li>{{ item }}</li>
    {% endfor %}
</ul>

Declaraciones Condicionales con {% if %}

{% if user.is_authenticated %}
    <p>Welcome, {{ user.username }}!</p>
{% else %}
    <p>Please log in.</p>
{% endif %}

Extender Plantillas con {% extends %} y {% block %}

<!-- base.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
    <header>
        <h1>My Site</h1>
    </header>
    <main>
        {% block content %}{% endblock %}
    </main>
</body>
</html>
<!-- child.html -->
{% extends "base.html" %}

{% block title %}Home{% endblock %}

{% block content %}
    <p>Welcome to the homepage!</p>
{% endblock %}

Incluir Plantillas con {% include %}

<!-- sidebar.html -->
<div class="sidebar">
    <h2>Sidebar</h2>
    <p>This is the sidebar content.</p>
</div>
<!-- main.html -->
<!DOCTYPE html>
<html>
<head>
    <title>Main Page</title>
</head>
<body>
    <h1>Main Content</h1>
    {% include "sidebar.html" %}
</body>
</html>

Conclusión

Entender cómo pasar contexto y usar etiquetas de plantilla es crucial para crear plantillas dinámicas y reutilizables en Django. Al dominar estos conceptos, puedes construir aplicaciones web más eficientes y mantenibles. Practica creando diferentes vistas y plantillas, experimentando con varios datos de contexto y usando etiquetas de plantilla para controlar la lógica de renderizado.
