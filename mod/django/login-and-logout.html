Inicio de sesión y cierre de sesión en Django
Registro de Usuarios ►

Introducción

En esta sección, cubriremos cómo implementar la funcionalidad de inicio y cierre de sesión de usuarios en una aplicación Django. Esto incluye configurar las vistas, formularios y plantillas necesarios, así como configurar el sistema de autenticación incorporado de Django.

Configuración de la autenticación

Instalando Django

Antes de comenzar, asegúrate de tener Django instalado. Si no, puedes instalarlo usando pip:

pip install django

Creando un proyecto y una aplicación Django

Primero, crea un nuevo proyecto Django y una aplicación dentro de él:

django-admin startproject myproject
cd myproject
django-admin startapp myapp

Configurando el proyecto

Agrega la nueva aplicación a la lista INSTALLED_APPS en myproject/settings.py:

INSTALLED_APPS = [
    ...
    'myapp',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    ...
]

Implementando el inicio de sesión

Creando la vista de inicio de sesión

Crea una vista para manejar el inicio de sesión de usuarios en myapp/views.py:

from django.shortcuts import render, redirect
from django.contrib.auth import authenticate, login
from django.contrib import messages

def login_view(request):
    if request.method == 'POST':
        username = request.POST['username']
        password = request.POST['password']
        user = authenticate(request, username=username, password=password)
        if user is not None:
            login(request, user)
            return redirect('home')
        else:
            messages.error(request, 'Nombre de usuario o contraseña inválidos')
    return render(request, 'myapp/login.html')

Creando la plantilla de inicio de sesión

Crea una plantilla para la página de inicio de sesión en myapp/templates/myapp/login.html:

<!DOCTYPE html>
<html>
<head>
    <title>Inicio de sesión</title>
</head>
<body>
    <h2>Inicio de sesión</h2>
    {% if messages %}
        <ul>
            {% for message in messages %}
                <li>{{ message }}</li>
            {% endfor %}
        </ul>
    {% endif %}
    <form method="post">
        {% csrf_token %}
        <label for="username">Nombre de usuario:</label>
        <input type="text" name="username" id="username" required><br>
        <label for="password">Contraseña:</label>
        <input type="password" name="password" id="password" required><br>
        <button type="submit">Iniciar sesión</button>
    </form>
</body>
</html>

Actualizando URLs

Agrega un patrón de URL para la vista de inicio de sesión en myapp/urls.py:

from django.urls import path
from .views import login_view

urlpatterns = [
    path('login/', login_view, name='login'),
]

Implementando el cierre de sesión

Creando la vista de cierre de sesión

Crea una vista para manejar el cierre de sesión de usuarios en myapp/views.py:

from django.contrib.auth import logout

def logout_view(request):
    logout(request)
    return redirect('login')

Actualizando URLs

Agrega un patrón de URL para la vista de cierre de sesión en myapp/urls.py:

from .views import logout_view

urlpatterns = [
    ...
    path('logout/', logout_view, name='logout'),
]

Protegiendo vistas con inicio de sesión requerido

Para asegurar que ciertas vistas solo sean accesibles para usuarios autenticados, usa el decorador login_required. Por ejemplo, en myapp/views.py:

from django.contrib.auth.decorators import login_required

@login_required
def home_view(request):
    return render(request, 'myapp/home.html')

Actualiza el patrón de URL para la vista de inicio en myapp/urls.py:

from .views import home_view

urlpatterns = [
    ...
    path('', home_view, name='home'),
]

Conclusión

En esta sección, cubrimos los conceptos básicos de la implementación de la funcionalidad de inicio y cierre de sesión en una aplicación Django. Creamos vistas, plantillas y patrones de URL para manejar la autenticación de usuarios. Además, aprendimos cómo proteger vistas usando el decorador login_required. Con estos fundamentos, ahora puedes gestionar sesiones de usuario y asegurar partes de tu aplicación de manera efectiva.
