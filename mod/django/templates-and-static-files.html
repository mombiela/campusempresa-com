Plantillas y Archivos Estáticos
Modelos y Bases de Datos ►

En esta sección, exploraremos cómo Django maneja las plantillas y los archivos estáticos. Las plantillas se utilizan para generar contenido HTML dinámico, mientras que los archivos estáticos se utilizan para servir CSS, JavaScript e imágenes. Comprender estos conceptos es crucial para construir una aplicación web completamente funcional.

Plantillas

Las plantillas de Django son una forma poderosa de generar HTML dinámicamente. Permiten separar la capa de presentación de la lógica de negocio, haciendo que tu código sea más limpio y mantenible.

Conceptos Clave

  • Motor de Plantillas: Django utiliza su propio motor de plantillas por defecto, pero se puede configurar para usar otros como Jinja2.
  • Etiquetas de Plantilla: Sintaxis especial dentro de las plantillas para realizar lógica como bucles y condicionales.
  • Filtros de Plantilla: Se utilizan para modificar variables para su visualización.
  • Herencia de Plantillas: Permite crear una plantilla base y extenderla en otras plantillas.

Creación y Uso de Plantillas

  1. Crear un Directorio de Plantillas: Por defecto, Django busca plantillas en un directorio llamado templates dentro de cada aplicación.

    myapp/
├── templates/
│   └── myapp/
│       └── index.html

  2. Configurar la Configuración de Plantillas: Asegúrate de que tu archivo settings.py esté configurado para encontrar tus plantillas.

    TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [os.path.join(BASE_DIR, 'templates')],
        'APP_DIRS': True,
        'OPTIONS': {
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',
            ],
        },
    },
]

  3. Crear una Vista para Renderizar la Plantilla: Usa vistas de Django para renderizar plantillas.

    from django.shortcuts import render

def index(request):
    return render(request, 'myapp/index.html', {'message': '¡Hola, Mundo!'})

  4. Usar Etiquetas y Filtros de Plantilla: Ejemplo de una plantilla usando etiquetas y filtros.

    <!-- templates/myapp/index.html -->
<!DOCTYPE html>
<html>
<head>
    <title>Mi Aplicación Django</title>
</head>
<body>
    <h1>{{ message|upper }}</h1>
    {% if user.is_authenticated %}
        <p>Bienvenido, {{ user.username }}!</p>
    {% else %}
        <p>Por favor, inicia sesión.</p>
    {% endif %}
</body>
</html>

Herencia de Plantillas

La herencia de plantillas te permite crear una plantilla base que otras plantillas pueden extender.

  1. Crear una Plantilla Base:

    <!-- templates/base.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}Mi Sitio{% endblock %}</title>
</head>
<body>
    <header>
        <h1>Mi Sitio</h1>
    </header>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 Mi Sitio</p>
    </footer>
</body>
</html>

  2. Extender la Plantilla Base:

    <!-- templates/myapp/index.html -->
{% extends "base.html" %}

{% block title %}Inicio{% endblock %}

{% block content %}
    <h2>Bienvenido a Mi Sitio</h2>
    <p>{{ message }}</p>
{% endblock %}

Archivos Estáticos

Los archivos estáticos incluyen CSS, JavaScript e imágenes que se sirven al cliente. Django proporciona una forma de gestionar estos archivos de manera eficiente.

Conceptos Clave

  • STATIC_URL: La URL a usar cuando se hace referencia a archivos estáticos.
  • STATICFILES_DIRS: Una lista de directorios donde Django también buscará archivos estáticos.
  • collectstatic: Un comando de gestión para recopilar archivos estáticos en una sola ubicación para su despliegue.

Configuración de Archivos Estáticos

  1. Configurar Archivos Estáticos en settings.py:

    STATIC_URL = '/static/'
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]

  2. Organizar Archivos Estáticos:

    myapp/
├── static/
│   └── myapp/
│       ├── css/
│       │   └── styles.css
│       ├── js/
│       │   └── scripts.js
│       └── images/
│           └── logo.png

  3. Usar Archivos Estáticos en Plantillas:

    <!-- templates/myapp/index.html -->
<!DOCTYPE html>
<html>
<head>
    <title>Mi Aplicación Django</title>
    <link rel="stylesheet" type="text/css" href="{% static 'myapp/css/styles.css' %}">
</head>
<body>
    <h1>{{ message }}</h1>
    <img src="{% static 'myapp/images/logo.png' %}" alt="Logo">
    <script src="{% static 'myapp/js/scripts.js' %}"></script>
</body>
</html>

  4. Recopilar Archivos Estáticos para el Despliegue:

    python manage.py collectstatic

Conclusión

En esta sección, cubrimos los conceptos básicos de las plantillas y los archivos estáticos en Django. Aprendiste cómo crear y usar plantillas, incluyendo la herencia de plantillas, y cómo gestionar archivos estáticos para tu aplicación web. Dominar estos conceptos te permitirá construir aplicaciones web dinámicas y visualmente atractivas con Django.
