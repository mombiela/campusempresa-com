Preparación para el Despliegue
Desplegando en Heroku ►

Desplegar una aplicación Django implica varios pasos para asegurar que tu aplicación sea segura, escalable y funcione bien en un entorno de producción. Esta sección te guiará a través de los pasos esenciales necesarios para preparar tu proyecto Django para el despliegue.

Conceptos Clave

  • Configuración del Entorno: Configurar diferentes ajustes para desarrollo y producción.
  • Gestión de Archivos Estáticos: Manejo de archivos estáticos como CSS, JavaScript e imágenes.
  • Configuración de la Base de Datos: Configurar una base de datos lista para producción.
  • Mejoras de Seguridad: Implementar las mejores prácticas de seguridad.
  • Configuración del Servidor Web: Configurar un servidor web para servir tu aplicación Django.
  • Monitoreo y Registro: Configurar el monitoreo y registro para tu aplicación.

Configuración del Entorno

Gestión de Configuraciones

Las configuraciones de Django deben ser diferentes para los entornos de desarrollo y producción. Esto se puede lograr utilizando variables de entorno y archivos de configuración separados.

Ejemplo: Uso de Variables de Entorno

# settings.py
import os

SECRET_KEY = os.getenv('DJANGO_SECRET_KEY', 'default-secret-key')
DEBUG = os.getenv('DJANGO_DEBUG', 'False') == 'True'
ALLOWED_HOSTS = os.getenv('DJANGO_ALLOWED_HOSTS', '').split(',')

Ejemplo: Archivos de Configuración Separados

# settings/base.py
import os

SECRET_KEY = os.getenv('DJANGO_SECRET_KEY', 'default-secret-key')
ALLOWED_HOSTS = os.getenv('DJANGO_ALLOWED_HOSTS', '').split(',')

# settings/development.py
from .base import *
DEBUG = True

# settings/production.py
from .base import *
DEBUG = False

Gestión de Archivos Estáticos

En un entorno de producción, los archivos estáticos deben ser recolectados y servidos eficientemente.

Recolectando Archivos Estáticos

python manage.py collectstatic

Sirviendo Archivos Estáticos

  • Usando un CDN: Descargar archivos estáticos a una Red de Entrega de Contenidos.
  • Usando un Servidor Web: Configurar tu servidor web (por ejemplo, Nginx) para servir archivos estáticos.

Ejemplo: Configuración de Nginx

server {
    location /static/ {
        alias /path/to/staticfiles/;
    }
}

Configuración de la Base de Datos

Para producción, se recomienda usar una base de datos robusta como PostgreSQL.

Ejemplo: Configuración de PostgreSQL

# settings.py
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': os.getenv('DB_NAME'),
        'USER': os.getenv('DB_USER'),
        'PASSWORD': os.getenv('DB_PASSWORD'),
        'HOST': os.getenv('DB_HOST'),
        'PORT': os.getenv('DB_PORT', '5432'),
    }
}

Mejoras de Seguridad

Configuraciones Clave de Seguridad

  • SECRET_KEY: Asegúrate de que se mantenga en secreto y no esté codificada.
  • DEBUG: Configurado en False en producción.
  • ALLOWED_HOSTS: Especifica los hosts permitidos.
  • HTTPS: Usa HTTPS para cifrar los datos en tránsito.

Ejemplo: Configuraciones de Seguridad

# settings.py
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True

Configuración del Servidor Web

Usando Gunicorn

Gunicorn es un servidor HTTP WSGI popular para servir aplicaciones Django.

Ejemplo: Ejecutando Gunicorn

gunicorn myproject.wsgi:application --bind 0.0.0.0:8000

Usando Nginx como Proxy Inverso

Nginx puede ser usado para reenviar solicitudes a Gunicorn.

Ejemplo: Configuración de Nginx

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Monitoreo y Registro

Configuración del Registro

Configura Django para registrar errores y eventos importantes.

Ejemplo: Configuración del Registro

# settings.py
LOGGING = {
    'version': 1,
    'disable_existing_loggers': False,
    'handlers': {
        'file': {
            'level': 'DEBUG',
            'class': 'logging.FileHandler',
            'filename': '/path/to/django/debug.log',
        },
    },
    'loggers': {
        'django': {
            'handlers': ['file'],
            'level': 'DEBUG',
            'propagate': True,
        },
    },
}

Herramientas de Monitoreo

  • Sentry: Para el seguimiento de errores.
  • Prometheus y Grafana: Para el monitoreo del rendimiento.

Conclusión

Preparar una aplicación Django para el despliegue implica configurar los ajustes del entorno, gestionar archivos estáticos, configurar una base de datos lista para producción, mejorar la seguridad, configurar un servidor web y configurar el monitoreo y registro. Siguiendo estos pasos, puedes asegurar que tu aplicación Django esté lista para un entorno de producción, proporcionando un servicio seguro, escalable y eficiente a tus usuarios.
