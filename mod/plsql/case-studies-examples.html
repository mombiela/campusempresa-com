Introducción

En esta sección, exploraremos varios estudios de caso y ejemplos para entender cómo se puede aplicar PL/SQL en escenarios del mundo real. Estos ejemplos irán desde niveles principiantes hasta avanzados, ayudándote a comprender las aplicaciones prácticas de PL/SQL.

Nivel Principiante

Ejemplo 1: Bloque PL/SQL Simple

Un bloque PL/SQL simple que imprime un mensaje.

BEGIN
    DBMS_OUTPUT.PUT_LINE('¡Hola, PL/SQL!');
END;
/
  • Explicación:
    • BEGIN y END definen el inicio y el final del bloque PL/SQL.
    • DBMS_OUTPUT.PUT_LINE es un procedimiento incorporado para imprimir mensajes en la consola.

Ejemplo 2: Declaración y Asignación de Variables

Un bloque PL/SQL que declara una variable y le asigna un valor.

DECLARE
    v_message VARCHAR2(50);
BEGIN
    v_message := '¡Bienvenido a PL/SQL!';
    DBMS_OUTPUT.PUT_LINE(v_message);
END;
/
  • Explicación:
    • La sección DECLARE se usa para declarar variables.
    • v_message es una variable de tipo VARCHAR2 con una longitud máxima de 50 caracteres.
    • La variable se le asigna un valor y luego se imprime.

Nivel Intermedio

Ejemplo 3: Uso de Cursores

Un bloque PL/SQL que utiliza un cursor explícito para obtener y mostrar nombres de empleados.

DECLARE
    CURSOR emp_cursor IS
        SELECT first_name, last_name FROM employees;
    v_first_name employees.first_name%TYPE;
    v_last_name employees.last_name%TYPE;
BEGIN
    OPEN emp_cursor;
    LOOP
        FETCH emp_cursor INTO v_first_name, v_last_name;
        EXIT WHEN emp_cursor%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_first_name || ' ' || v_last_name);
    END LOOP;
    CLOSE emp_cursor;
END;
/
  • Explicación:
    • CURSOR emp_cursor IS declara un cursor para seleccionar nombres y apellidos de la tabla employees.
    • OPEN emp_cursor abre el cursor.
    • FETCH recupera los datos en variables.
    • EXIT WHEN emp_cursor%NOTFOUND sale del bucle cuando no se encuentran más filas.
    • CLOSE emp_cursor cierra el cursor.

Ejemplo 4: Manejo de Excepciones

Un bloque PL/SQL que demuestra el manejo de excepciones.

DECLARE
    v_num NUMBER;
BEGIN
    v_num := 10 / 0;
EXCEPTION
    WHEN ZERO_DIVIDE THEN
        DBMS_OUTPUT.PUT_LINE('Error: División por cero');
END;
/
  • Explicación:
    • El bloque BEGIN intenta dividir un número por cero.
    • El bloque EXCEPTION captura la excepción ZERO_DIVIDE e imprime un mensaje de error.

Nivel Avanzado

Ejemplo 5: Creación y Uso de Procedimientos

Un procedimiento PL/SQL que calcula el salario total de los empleados en un departamento.

CREATE OR REPLACE PROCEDURE calculate_total_salary (p_dept_id IN NUMBER, p_total_salary OUT NUMBER) IS
BEGIN
    SELECT SUM(salary) INTO p_total_salary
    FROM employees
    WHERE department_id = p_dept_id;
END;
/
  • Explicación:
    • CREATE OR REPLACE PROCEDURE define un nuevo procedimiento.
    • p_dept_id es un parámetro de entrada para el ID del departamento.
    • p_total_salary es un parámetro de salida para el salario total.
    • La declaración SELECT calcula el salario total para el departamento especificado.

Ejemplo 6: Uso de Triggers

Un trigger que registra cambios en la tabla employees.

CREATE OR REPLACE TRIGGER log_employee_changes
AFTER INSERT OR UPDATE OR DELETE ON employees
FOR EACH ROW
BEGIN
    IF INSERTING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'INSERT', :NEW.employee_id);
    ELSIF UPDATING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'UPDATE', :NEW.employee_id);
    ELSIF DELETING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'DELETE', :OLD.employee_id);
    END IF;
END;
/
  • Explicación:
    • CREATE OR REPLACE TRIGGER define un nuevo trigger.
    • El trigger se activa AFTER INSERT OR UPDATE OR DELETE en la tabla employees.
    • FOR EACH ROW especifica que el trigger actúa sobre cada fila afectada.
    • Las declaraciones IF verifican el tipo de operación DML y registran los cambios en consecuencia.

Conclusión

A través de estos estudios de caso y ejemplos, hemos cubierto una variedad de aplicaciones de PL/SQL desde bloques simples hasta procedimientos y triggers avanzados. Al practicar estos ejemplos, obtendrás una comprensión más profunda de cómo usar PL/SQL de manera efectiva en varios escenarios. Continúa experimentando con estos conceptos para solidificar tu conocimiento y mejorar tus habilidades de programación.
