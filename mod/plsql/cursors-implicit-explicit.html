Introducción

En PL/SQL, los cursores se utilizan para manejar los resultados de las consultas. Permiten recuperar y manipular datos obtenidos por consultas SQL. Hay dos tipos de cursores en PL/SQL: implícitos y explícitos. Comprender estos cursores es crucial para una programación de bases de datos eficiente.

Cursores Implícitos

Los cursores implícitos son creados automáticamente por Oracle cada vez que se ejecuta una instrucción SQL. Son más simples de usar pero ofrecen menos control en comparación con los cursores explícitos.

Conceptos Clave

  • Manejo Automático: Los cursores implícitos son gestionados automáticamente por PL/SQL.
  • Alcance: Solo están disponibles durante la ejecución de la instrucción SQL.
  • Uso: Comúnmente utilizados para consultas de una sola fila.

Ejemplo

DECLARE
  v_employee_name employees.first_name%TYPE;
BEGIN
  SELECT first_name INTO v_employee_name FROM employees WHERE employee_id = 101;
  DBMS_OUTPUT.PUT_LINE('Employee Name: ' || v_employee_name);
END;
/

En este ejemplo:

  • Se utiliza un cursor implícito para la instrucción SELECT.
  • El resultado se almacena en la variable v_employee_name.

Cursores Explícitos

Los cursores explícitos son definidos y controlados por el programador. Proporcionan más flexibilidad y se utilizan para consultas de múltiples filas.

Conceptos Clave

  • Manejo Manual: Necesitas declarar, abrir, recuperar y cerrar los cursores explícitos.
  • Control: Ofrecen más control sobre la ejecución de la consulta y la recuperación de datos.
  • Uso: Adecuados para consultas de múltiples filas y manipulación de datos complejos.

Pasos para Usar Cursores Explícitos

  1. Declarar el Cursor: Definir la consulta SQL.
  2. Abrir el Cursor: Ejecutar la consulta SQL.
  3. Recuperar Datos: Recuperar el conjunto de resultados fila por fila.
  4. Cerrar el Cursor: Liberar los recursos.

Ejemplo

DECLARE
  CURSOR emp_cursor IS
    SELECT first_name, last_name FROM employees WHERE department_id = 10;
  v_first_name employees.first_name%TYPE;
  v_last_name employees.last_name%TYPE;
BEGIN
  OPEN emp_cursor;
  LOOP
    FETCH emp_cursor INTO v_first_name, v_last_name;
    EXIT WHEN emp_cursor%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE('Employee: ' || v_first_name || ' ' || v_last_name);
  END LOOP;
  CLOSE emp_cursor;
END;
/

En este ejemplo:

  • El cursor emp_cursor se declara para seleccionar empleados del departamento 10.
  • El cursor se abre y los datos se recuperan en un bucle hasta que todas las filas se procesan.
  • Luego, el cursor se cierra.

Comparación de Cursores Implícitos y Explícitos

| Característica | Cursor Implícito | Cursor Explícito | |---------------------|----------------------------------------------|----------------------------------------------| | Declaración | Automática | Manual | | Control | Limitado | Completo | | Uso | Consultas de una sola fila | Consultas de múltiples filas | | Gestión de Recursos | Automática | Manual (abrir, recuperar, cerrar) | | Flexibilidad | Menos flexible | Más flexible |

Conclusión

Los cursores son esenciales para manejar los resultados de las consultas en PL/SQL. Los cursores implícitos son fáciles de usar para consultas simples de una sola fila, mientras que los cursores explícitos proporcionan mayor control y flexibilidad para consultas complejas de múltiples filas. Comprender cuándo y cómo usar cada tipo de cursor te ayudará a escribir código PL/SQL eficiente y efectivo.
