Introducción a PL/SQL

PL/SQL (Procedural Language/Structured Query Language) es la extensión procedural de Oracle Corporation para SQL y la base de datos relacional Oracle. Combina el poder de manipulación de datos de SQL con el poder de procesamiento de los lenguajes procedurales.

Conceptos Clave

  • Estructura de Bloques: PL/SQL se basa en una estructura de bloques. Cada bloque consta de tres secciones: Declaración, Ejecución y Manejo de Excepciones.
  • Variables: Se utilizan para almacenar datos que pueden ser manipulados durante la ejecución del programa.
  • Estructuras de Control: Incluyen declaraciones condicionales y bucles para controlar el flujo de ejecución.
  • Cursores: Se utilizan para recuperar múltiples filas de una consulta.
  • Manejo de Excepciones: Mecanismos para manejar errores en tiempo de ejecución.

Estructura de Bloques en PL/SQL

Un bloque PL/SQL tiene la siguiente estructura:

DECLARE
   -- Sección de declaración: variables, constantes, cursores, etc.
BEGIN
   -- Sección de ejecución: sentencias SQL y PL/SQL
EXCEPTION
   -- Sección de manejo de excepciones: código de manejo de errores
END;
/

Ejemplo de un Bloque PL/SQL Simple

DECLARE
   v_message VARCHAR2(50);
BEGIN
   v_message := '¡Hola, PL/SQL!';
   DBMS_OUTPUT.PUT_LINE(v_message);
EXCEPTION
   WHEN OTHERS THEN
      DBMS_OUTPUT.PUT_LINE('Ocurrió un error.');
END;
/
  • DECLARE: Esta sección es opcional y se utiliza para declarar variables, constantes y cursores.
  • BEGIN: Esta sección contiene las sentencias ejecutables.
  • EXCEPTION: Esta sección es opcional y se utiliza para manejar excepciones.
  • END: Marca el final del bloque PL/SQL.
  • /: Se utiliza para ejecutar el bloque en SQL*Plus o SQL Developer.

Variables

Las variables en PL/SQL se utilizan para almacenar datos temporalmente. Deben ser declaradas en la sección de declaración.

Declaración de Variables

DECLARE
   v_name VARCHAR2(30);
   v_age NUMBER;
BEGIN
   v_name := 'John Doe';
   v_age := 25;
   DBMS_OUTPUT.PUT_LINE('Nombre: ' || v_name);
   DBMS_OUTPUT.PUT_LINE('Edad: ' || v_age);
END;
/
  • v_name: Una variable de tipo VARCHAR2.
  • v_age: Una variable de tipo NUMBER.

Estructuras de Control

PL/SQL soporta varias estructuras de control para gestionar el flujo de ejecución.

Declaraciones Condicionales

IF-THEN-ELSE

DECLARE
   v_score NUMBER := 85;
BEGIN
   IF v_score >= 90 THEN
      DBMS_OUTPUT.PUT_LINE('Calificación: A');
   ELSIF v_score >= 80 THEN
      DBMS_OUTPUT.PUT_LINE('Calificación: B');
   ELSE
      DBMS_OUTPUT.PUT_LINE('Calificación: C');
   END IF;
END;
/

Bucles

Bucle FOR

BEGIN
   FOR i IN 1..5 LOOP
      DBMS_OUTPUT.PUT_LINE('Iteración: ' || i);
   END LOOP;
END;
/

Bucle WHILE

DECLARE
   v_counter NUMBER := 1;
BEGIN
   WHILE v_counter <= 5 LOOP
      DBMS_OUTPUT.PUT_LINE('Contador: ' || v_counter);
      v_counter := v_counter + 1;
   END LOOP;
END;
/

Cursores

Los cursores se utilizan para manejar múltiples filas devueltas por una consulta.

Cursor Implícito

BEGIN
   FOR rec IN (SELECT first_name, last_name FROM employees WHERE department_id = 10) LOOP
      DBMS_OUTPUT.PUT_LINE('Nombre: ' || rec.first_name || ' ' || rec.last_name);
   END LOOP;
END;
/

Cursor Explícito

DECLARE
   CURSOR emp_cursor IS SELECT first_name, last_name FROM employees WHERE department_id = 10;
   v_first_name employees.first_name%TYPE;
   v_last_name employees.last_name%TYPE;
BEGIN
   OPEN emp_cursor;
   LOOP
      FETCH emp_cursor INTO v_first_name, v_last_name;
      EXIT WHEN emp_cursor%NOTFOUND;
      DBMS_OUTPUT.PUT_LINE('Nombre: ' || v_first_name || ' ' || v_last_name);
   END LOOP;
   CLOSE emp_cursor;
END;
/

Manejo de Excepciones

Las excepciones se utilizan para manejar errores que ocurren durante la ejecución de un bloque PL/SQL.

Ejemplo de Manejo de Excepciones

DECLARE
   v_num NUMBER;
BEGIN
   v_num := 10 / 0; -- Esto causará un error de división por cero
EXCEPTION
   WHEN ZERO_DIVIDE THEN
      DBMS_OUTPUT.PUT_LINE('Error: División por cero');
   WHEN OTHERS THEN
      DBMS_OUTPUT.PUT_LINE('Ocurrió un error inesperado');
END;
/

Conclusión

Entender la sintaxis básica de PL/SQL es crucial para escribir código eficiente y libre de errores. Este tema cubrió los bloques fundamentales de PL/SQL, incluyendo la estructura de bloques, declaración de variables, estructuras de control, cursores y manejo de excepciones.
