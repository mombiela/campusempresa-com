Mejores Prácticas para el Manejo de Errores
Revisión y Refactorización de Código ►

El manejo de errores es un aspecto crucial de la programación en PL/SQL. Un manejo adecuado de errores asegura que tus programas puedan manejar situaciones inesperadas de manera elegante y proporcionar retroalimentación significativa a los usuarios. Este tema cubrirá las mejores prácticas para el manejo de errores en PL/SQL, desde niveles principiantes hasta avanzados.

Introducción al Manejo de Errores

  • Definición: El manejo de errores en PL/SQL implica detectar y gestionar errores que ocurren durante la ejecución de un bloque PL/SQL.
  • Tipos de Errores:
    • Errores de Sintaxis: Detectados en tiempo de compilación.
    • Errores de Ejecución: Detectados durante la ejecución.
    • Errores Lógicos: Resultan de una lógica incorrecta en el código.

Manejo Básico de Errores con EXCEPTION

PL/SQL proporciona el bloque EXCEPTION para manejar errores de ejecución.

Sintaxis

BEGIN
  -- Tu código aquí
EXCEPTION
  WHEN exception_name THEN
    -- Código de manejo de errores aquí
END;

Ejemplo

BEGIN
  -- Intentar dividir por cero
  DECLARE
    num1 NUMBER := 10;
    num2 NUMBER := 0;
    result NUMBER;
  BEGIN
    result := num1 / num2;
  EXCEPTION
    WHEN ZERO_DIVIDE THEN
      DBMS_OUTPUT.PUT_LINE('Error: División por cero.');
  END;
END;
  • Explicación: En este ejemplo, se captura la excepción ZERO_DIVIDE y se imprime un mensaje.

Excepciones Predefinidas Comúnmente Usadas

  • NO_DATA_FOUND: Se genera cuando una instrucción SELECT INTO no devuelve filas.
  • TOO_MANY_ROWS: Se genera cuando una instrucción SELECT INTO devuelve más de una fila.
  • INVALID_CURSOR: Se genera cuando se intenta una operación ilegal con un cursor.

Ejemplo

BEGIN
  DECLARE
    v_name VARCHAR2(50);
  BEGIN
    SELECT name INTO v_name FROM employees WHERE employee_id = 9999;
  EXCEPTION
    WHEN NO_DATA_FOUND THEN
      DBMS_OUTPUT.PUT_LINE('Error: No se encontró un empleado con ID 9999.');
  END;
END;

Excepciones Definidas por el Usuario

Puedes definir tus propias excepciones usando la palabra clave EXCEPTION.

Sintaxis

DECLARE
  my_exception EXCEPTION;
BEGIN
  -- Tu código aquí
  RAISE my_exception;
EXCEPTION
  WHEN my_exception THEN
    -- Código de manejo de errores aquí
END;

Ejemplo

DECLARE
  insufficient_funds EXCEPTION;
  balance NUMBER := 500;
  withdrawal_amount NUMBER := 600;
BEGIN
  IF withdrawal_amount > balance THEN
    RAISE insufficient_funds;
  END IF;
EXCEPTION
  WHEN insufficient_funds THEN
    DBMS_OUTPUT.PUT_LINE('Error: Fondos insuficientes.');
END;

Técnicas Avanzadas de Manejo de Errores

Uso de SQLCODE y SQLERRM

  • SQLCODE: Devuelve el número de error.
  • SQLERRM: Devuelve el mensaje de error.

Ejemplo

BEGIN
  DECLARE
    num1 NUMBER := 10;
    num2 NUMBER := 0;
    result NUMBER;
  BEGIN
    result := num1 / num2;
  EXCEPTION
    WHEN OTHERS THEN
      DBMS_OUTPUT.PUT_LINE('Código de error: ' || SQLCODE);
      DBMS_OUTPUT.PUT_LINE('Mensaje de error: ' || SQLERRM);
  END;
END;
  • Explicación: La cláusula WHEN OTHERS captura todas las excepciones, y SQLCODE y SQLERRM proporcionan información detallada del error.

Propagación de Excepciones

  • RAISE_APPLICATION_ERROR: Te permite propagar excepciones con mensajes de error personalizados.

Ejemplo

CREATE OR REPLACE PROCEDURE check_balance(p_balance NUMBER) IS
BEGIN
  IF p_balance < 0 THEN
    RAISE_APPLICATION_ERROR(-20001, 'No se permite un saldo negativo.');
  END IF;
END;
  • Explicación: El procedimiento RAISE_APPLICATION_ERROR genera una excepción con un código de error y un mensaje personalizados.

Mejores Prácticas

  • Siempre Maneja Excepciones Conocidas: Asegúrate de manejar las excepciones que esperas que puedan ocurrir.
  • Usa WHEN OTHERS con Moderación: Evita usar WHEN OTHERS sin un registro adecuado o sin volver a generar la excepción.
  • Registra Errores: Usa mecanismos de registro para grabar detalles de errores para depuración y auditoría.
  • Proporciona Mensajes Significativos: Asegúrate de que los mensajes de error sean claros y útiles para el usuario final.
  • Evita Fallos Silenciosos: No suprimas excepciones sin manejarlas adecuadamente.

Conclusión

Un manejo efectivo de errores en PL/SQL es esencial para crear aplicaciones robustas y confiables. Siguiendo las mejores prácticas como manejar excepciones conocidas, usar mensajes de error significativos y registrar errores, puedes asegurar que tus programas PL/SQL puedan manejar situaciones inesperadas de manera elegante y proporcionar retroalimentación valiosa a los usuarios.
