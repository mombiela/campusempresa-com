Introducción

Firebase Hosting es una herramienta poderosa para desplegar aplicaciones web. Mientras que las características básicas de hosting son suficientes para muchos proyectos, las características avanzadas pueden mejorar significativamente el rendimiento, la seguridad y la experiencia del usuario de tu aplicación. Esta sección cubrirá estas características avanzadas, incluyendo dominios personalizados, reescrituras y redirecciones de URL, encabezados personalizados y el despliegue de múltiples sitios.

Dominios Personalizados

Los dominios personalizados te permiten usar tu propio nombre de dominio en lugar del subdominio predeterminado de Firebase.

  • Configuración de un Dominio Personalizado:

    1. Ve a la Consola de Firebase.
    2. Selecciona tu proyecto y navega a la sección de Hosting.
    3. Haz clic en "Agregar Dominio Personalizado".
    4. Sigue las instrucciones para verificar la propiedad del dominio y configurar los registros DNS.

  • Ejemplo:

    Tipo: CNAME
Nombre: www
Valor: your-project-id.firebaseapp.com

Reescrituras y Redirecciones de URL

Las reescrituras y redirecciones de URL son esenciales para gestionar la estructura de URL de tu aplicación y asegurar una experiencia de usuario fluida.

  • Reescrituras: Mapean un patrón de URL a un recurso específico, como una Función de Cloud o un archivo estático.

  • Redirecciones: Redirigen automáticamente una URL a otra, útil para mantener el SEO y los marcadores de usuario.

  • Ejemplo de Configuración:

    {
  "hosting": {
    "rewrites": [
      {
        "source": "/api/**",
        "function": "api"
      }
    ],
    "redirects": [
      {
        "source": "/old-page",
        "destination": "/new-page",
        "type": 301
      }
    ]
  }
}

Encabezados Personalizados

Los encabezados personalizados pueden ser utilizados para mejorar la seguridad, controlar el almacenamiento en caché y gestionar tipos de contenido.

  • Encabezados Comunes:

    • Content-Security-Policy: Previene ataques XSS.
    • Cache-Control: Gestiona el comportamiento del almacenamiento en caché.
    • Strict-Transport-Security: Refuerza el uso de HTTPS.

  • Ejemplo de Configuración:

    {
  "hosting": {
    "headers": [
      {
        "source": "**/*.@(js|css)",
        "headers": [
          {
            "key": "Cache-Control",
            "value": "max-age=31536000"
          }
        ]
      },
      {
        "source": "/**",
        "headers": [
          {
            "key": "Strict-Transport-Security",
            "value": "max-age=31536000; includeSubDomains"
          }
        ]
      }
    ]
  }
}

Desplegar Múltiples Sitios

Firebase Hosting te permite desplegar múltiples sitios dentro de un solo proyecto de Firebase. Esto es útil para gestionar diferentes entornos (por ejemplo, staging y producción) o alojar múltiples aplicaciones.

  • Pasos para Desplegar Múltiples Sitios:

    1. Define cada sitio en tu archivo firebase.json.
    2. Usa el comando firebase use --add para configurar múltiples configuraciones de sitio.
    3. Despliega cada sitio usando el comando firebase deploy --only hosting:<site-name>.

  • Ejemplo de Configuración:

    {
  "hosting": [
    {
      "target": "site1",
      "public": "public-site1",
      "ignore": ["firebase.json", "**/.*", "**/node_modules/**"]
    },
    {
      "target": "site2",
      "public": "public-site2",
      "ignore": ["firebase.json", "**/.*", "**/node_modules/**"]
    }
  ]
}

Conclusión

Las características avanzadas de hosting en Firebase proporcionan herramientas poderosas para mejorar tus aplicaciones web. Los dominios personalizados, las reescrituras y redirecciones de URL, los encabezados personalizados y la capacidad de desplegar múltiples sitios ofrecen flexibilidad y control sobre tu entorno de hosting. Al dominar estas características, puedes optimizar el rendimiento, la seguridad y la experiencia del usuario de tu aplicación.
