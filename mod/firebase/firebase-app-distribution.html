Firebase App Distribution es una herramienta poderosa que permite a los desarrolladores distribuir sus aplicaciones a los testers de manera rápida y eficiente. Esta sección cubrirá todo, desde los conceptos básicos de la configuración de Firebase App Distribution hasta características avanzadas y mejores prácticas.

Introducción a Firebase App Distribution

Firebase App Distribution proporciona una manera de distribuir versiones preliminares de tu aplicación a los testers. Esto ayuda a obtener retroalimentación temprana e identificar errores antes de que la aplicación se lance oficialmente.

Conceptos Clave

  • Gestión de Testers: Añadir y gestionar testers que recibirán las versiones de tu aplicación.
  • Distribución de Builds: Subir y distribuir builds a los testers.
  • Recopilación de Retroalimentación: Recoger retroalimentación de los testers para mejorar tu aplicación.
  • Integración con CI/CD: Automatizar el proceso de distribución usando herramientas de Integración Continua/Despliegue Continuo.

Configuración de Firebase App Distribution

Requisitos Previos

  • Un proyecto de Firebase.
  • Una aplicación registrada en tu proyecto de Firebase.
  • Firebase CLI instalado en tu máquina.

Pasos para Configurar

  1. Añadir Firebase a Tu Aplicación:

    • Sigue las instrucciones en la consola de Firebase para añadir Firebase a tu aplicación iOS o Android.

  2. Instalar Firebase CLI:

    npm install -g firebase-tools

  3. Autenticar Firebase CLI:

    firebase login

  4. Inicializar Firebase en Tu Proyecto:

    firebase init

  5. Añadir App Distribution a Tu Proyecto:

    • En la consola de Firebase, navega a App Distribution y sigue las instrucciones de configuración.

Distribuyendo Tu Primer Build

Subiendo un Build

  1. Construir Tu Aplicación:

    • Para Android: 
      ./gradlew assembleDebug
    • Para iOS: 
      xcodebuild -scheme YourScheme -sdk iphoneos -configuration Debug archive -archivePath YourApp.xcarchive

  2. Distribuir el Build:

    firebase appdistribution:distribute path/to/your/app.apk --app your-app-id --groups testers

Añadiendo Testers

  • Navega a la sección de App Distribution en la consola de Firebase.
  • Haz clic en "Testers & Groups".
  • Añade testers individuales o crea grupos y añade testers a estos grupos.

Recopilando Retroalimentación

Habilitando la Recopilación de Retroalimentación

  • Asegúrate de que los testers tengan instalada la aplicación Firebase App Tester.
  • Los testers pueden proporcionar retroalimentación directamente a través de la aplicación, la cual será visible en la consola de Firebase.

Viendo la Retroalimentación

  • Navega a la sección de App Distribution en la consola de Firebase.
  • Haz clic en el build específico para ver la retroalimentación de los testers.

Integración con CI/CD

Usando Fastlane

  • Instalar Fastlane:

    gem install fastlane

  • Añadir Fastlane a Tu Proyecto:

    fastlane init

  • Configurar Fastlane para Firebase App Distribution:

    lane :distribute do
  firebase_app_distribution(
    app: "your-app-id",
    testers: "tester1@example.com, tester2@example.com",
    release_notes: "This is a test build"
  )
end

Usando GitHub Actions

  • Crear un Workflow de GitHub Actions: 
    name: Distribute App

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up JDK 1.8
      uses: actions/setup-java@v1
      with:
        java-version: 1.8

    - name: Build with Gradle
      run: ./gradlew build

    - name: Distribute to Firebase App Distribution
      uses: wzieba/Firebase-Distribution-Github-Action@v1
      with:
        appId: ${{ secrets.FIREBASE_APP_ID }}
        token: ${{ secrets.FIREBASE_TOKEN }}
        groups: testers
        file: app/build/outputs/apk/debug/app-debug.apk

Mejores Prácticas

  • Actualizar Regularmente a los Testers: Mantén a tus testers comprometidos actualizándolos regularmente con nuevos builds.
  • Automatizar la Distribución: Usa herramientas de CI/CD para automatizar el proceso de distribución, asegurando que los testers siempre tengan el último build.
  • Recoger y Actuar sobre la Retroalimentación: Recoge activamente retroalimentación de los testers y úsala para mejorar tu aplicación antes del lanzamiento oficial.

Conclusión

Firebase App Distribution es una herramienta esencial para cualquier equipo de desarrollo de aplicaciones. Simplifica el proceso de distribuir builds preliminares a los testers, recopilar retroalimentación e integrarse con pipelines de CI/CD. Siguiendo los pasos descritos en esta sección, puedes gestionar eficientemente tu proceso de distribución de aplicaciones y asegurarte de que tu aplicación esté completamente probada antes de llegar a tus usuarios.
