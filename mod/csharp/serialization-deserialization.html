Serialización y Deserialización en C#
Introducción a LINQ ►

La serialización y deserialización son conceptos esenciales en C# para convertir objetos en un formato que pueda ser fácilmente almacenado o transmitido y luego reconstruirlos de nuevo en objetos. Este tema cubrirá los aspectos básicos y avanzados de la serialización y deserialización en C#.

¿Qué es la Serialización?

La serialización es el proceso de convertir un objeto en una secuencia de bytes para almacenar el objeto o transmitirlo a la memoria, una base de datos o un archivo. Su propósito principal es guardar el estado de un objeto para que pueda ser recreado cuando sea necesario.

Conceptos Clave

  • Estado del Objeto: Los datos contenidos dentro de un objeto.
  • Secuencia de Bytes: Una secuencia de bytes que representa el estado del objeto.
  • Almacenamiento/Transmisión: El medio donde se almacenan o transmiten los datos serializados.

Ejemplo

using System;
using System.IO;
using System.Runtime.Serialization.Formatters.Binary;

[Serializable]
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}

class Program
{
    static void Main()
    {
        Person person = new Person { Name = "John", Age = 30 };

        // Serializar
        BinaryFormatter formatter = new BinaryFormatter();
        using (FileStream stream = new FileStream("person.dat", FileMode.Create))
        {
            formatter.Serialize(stream, person);
        }

        Console.WriteLine("Objeto Person serializado.");
    }
}

¿Qué es la Deserialización?

La deserialización es el proceso inverso de la serialización. Involucra convertir una secuencia de bytes de nuevo en un objeto. Esto es útil para reconstruir el objeto desde su estado almacenado o transmitido.

Conceptos Clave

  • Reconstrucción: El proceso de crear un objeto a partir de una secuencia de bytes.
  • Secuencia de Bytes: Los datos serializados que representan el estado del objeto.

Ejemplo

using System;
using System.IO;
using System.Runtime.Serialization.Formatters.Binary;

[Serializable]
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}

class Program
{
    static void Main()
    {
        Person person;

        // Deserializar
        BinaryFormatter formatter = new BinaryFormatter();
        using (FileStream stream = new FileStream("person.dat", FileMode.Open))
        {
            person = (Person)formatter.Deserialize(stream);
        }

        Console.WriteLine($"Objeto Person deserializado. Nombre: {person.Name}, Edad: {person.Age}");
    }
}

Tipos de Serialización

Serialización Binaria

  • Descripción: Convierte un objeto en un formato binario.
  • Caso de Uso: Eficiente para almacenamiento y transmisión, pero no legible por humanos.
  • Ejemplo: Usando BinaryFormatter.

Serialización XML

  • Descripción: Convierte un objeto en un formato XML.
  • Caso de Uso: Legible por humanos y útil para servicios web.
  • Ejemplo: Usando XmlSerializer.
using System;
using System.IO;
using System.Xml.Serialization;

public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}

class Program
{
    static void Main()
    {
        Person person = new Person { Name = "John", Age = 30 };

        // Serializar
        XmlSerializer serializer = new XmlSerializer(typeof(Person));
        using (FileStream stream = new FileStream("person.xml", FileMode.Create))
        {
            serializer.Serialize(stream, person);
        }

        Console.WriteLine("Objeto Person serializado a XML.");
    }
}

Serialización JSON

  • Descripción: Convierte un objeto en un formato JSON.
  • Caso de Uso: Legible por humanos, ampliamente usado en aplicaciones web.
  • Ejemplo: Usando JsonSerializer.
using System;
using System.Text.Json;

public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
}

class Program
{
    static void Main()
    {
        Person person = new Person { Name = "John", Age = 30 };

        // Serializar
        string jsonString = JsonSerializer.Serialize(person);
        Console.WriteLine("Objeto Person serializado a JSON:");
        Console.WriteLine(jsonString);
    }
}

Temas Avanzados

Serialización Personalizada

  • Descripción: Implementar lógica personalizada para la serialización y deserialización.
  • Caso de Uso: Cuando la serialización por defecto no cumple con los requisitos.
  • Ejemplo: Implementando la interfaz ISerializable.
using System;
using System.IO;
using System.Runtime.Serialization;
using System.Runtime.Serialization.Formatters.Binary;

[Serializable]
public class Person : ISerializable
{
    public string Name { get; set; }
    public int Age { get; set; }

    public Person() { }

    protected Person(SerializationInfo info, StreamingContext context)
    {
        Name = info.GetString("Name");
        Age = info.GetInt32("Age");
    }

    public void GetObjectData(SerializationInfo info, StreamingContext context)
    {
        info.AddValue("Name", Name);
        info.AddValue("Age", Age);
    }
}

class Program
{
    static void Main()
    {
        Person person = new Person { Name = "John", Age = 30 };

        // Serializar
        BinaryFormatter formatter = new BinaryFormatter();
        using (FileStream stream = new FileStream("person_custom.dat", FileMode.Create))
        {
            formatter.Serialize(stream, person);
        }

        // Deserializar
        using (FileStream stream = new FileStream("person_custom.dat", FileMode.Open))
        {
            person = (Person)formatter.Deserialize(stream);
        }

        Console.WriteLine($"Objeto Person deserializado. Nombre: {person.Name}, Edad: {person.Age}");
    }
}

Resumen

La serialización y deserialización son técnicas poderosas en C# para convertir objetos a y desde un formato almacenable o transmisible. Entender los diferentes tipos de serialización (binaria, XML, JSON) y cómo implementar la serialización personalizada puede mejorar enormemente tu capacidad para gestionar datos en tus aplicaciones. Al dominar estos conceptos, puedes asegurar que tus aplicaciones puedan almacenar y transmitir datos de manera eficiente mientras mantienen la integridad y estructura de tus objetos.
