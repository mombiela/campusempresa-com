Introducción a las Tareas y TPL

La Biblioteca de Paralelismo de Tareas (TPL) es un conjunto de tipos públicos y APIs en el espacio de nombres System.Threading.Tasks que simplifica el proceso de añadir paralelismo y concurrencia a las aplicaciones. Es un componente clave del .NET Framework y está diseñada para facilitar la escritura de código paralelo eficiente, escalable y mantenible.

Conceptos Clave

  • Tarea: Representa una operación asincrónica.
  • Biblioteca de Paralelismo de Tareas (TPL): Proporciona una abstracción de nivel superior para la programación paralela en comparación con el threading tradicional.
  • Programación Asincrónica: Permite que las tareas se ejecuten independientemente del flujo principal del programa, mejorando la capacidad de respuesta y el rendimiento.

Creación y Ejecución de Tareas

Crear y ejecutar tareas en C# es sencillo con la TPL.

Ejemplo: Creación de una Tarea Simple

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task myTask = new Task(() => Console.WriteLine("¡Hola desde la Tarea!"));
        myTask.Start();
        myTask.Wait(); // Espera a que la tarea se complete
    }
}
  • Explicación:
    • Task myTask = new Task(...): Crea una nueva tarea.
    • myTask.Start(): Inicia la tarea.
    • myTask.Wait(): Espera a que la tarea se complete antes de continuar.

Ejemplo: Usando Task.Run

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task.Run(() => Console.WriteLine("¡Hola desde Task.Run!")).Wait();
    }
}
  • Explicación:
    • Task.Run(...): Una forma abreviada de crear e iniciar una tarea en un solo paso.
    • .Wait(): Asegura que el hilo principal espere a que la tarea se complete.

Continuaciones de Tareas

Las tareas pueden encadenarse usando continuaciones, lo que permite especificar acciones a realizar después de que una tarea se complete.

Ejemplo: Continuación de Tareas

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task firstTask = Task.Run(() => Console.WriteLine("Primera Tarea"));
        Task continuation = firstTask.ContinueWith(t => Console.WriteLine("Tarea de Continuación"));
        continuation.Wait();
    }
}
  • Explicación:
    • firstTask.ContinueWith(...): Especifica una tarea de continuación que se ejecuta después de que firstTask se complete.

Manejo de Excepciones en Tareas

Las tareas pueden lanzar excepciones, y manejarlas adecuadamente es crucial para aplicaciones robustas.

Ejemplo: Manejo de Excepciones

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task task = Task.Run(() => { throw new InvalidOperationException("¡Error en la Tarea!"); });

        try
        {
            task.Wait();
        }
        catch (AggregateException ex)
        {
            foreach (var innerEx in ex.InnerExceptions)
            {
                Console.WriteLine(innerEx.Message);
            }
        }
    }
}
  • Explicación:
    • AggregateException: Cuando una tarea lanza una excepción, se envuelve en una AggregateException.
    • ex.InnerExceptions: Contiene todas las excepciones lanzadas por la tarea.

Programación Paralela con TPL

La TPL también proporciona métodos para bucles paralelos y otros constructos paralelos.

Ejemplo: Parallel.For

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Parallel.For(0, 10, i =>
        {
            Console.WriteLine($"Procesando {i}");
        });
    }
}
  • Explicación:
    • Parallel.For(...): Ejecuta un bucle for en el que las iteraciones pueden ejecutarse en paralelo.

Ejemplo: Parallel.ForEach

using System;
using System.Collections.Generic;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        List<int> numbers = new List<int> { 1, 2, 3, 4, 5 };

        Parallel.ForEach(numbers, number =>
        {
            Console.WriteLine($"Procesando {number}");
        });
    }
}
  • Explicación:
    • Parallel.ForEach(...): Ejecuta un bucle foreach en el que las iteraciones pueden ejecutarse en paralelo.

Características Avanzadas de las Tareas

Para escenarios avanzados, la TPL proporciona características más sofisticadas como la cancelación de tareas y el reporte de progreso.

Ejemplo: Cancelación de Tareas

using System;
using System.Threading;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        CancellationTokenSource cts = new CancellationTokenSource();

        Task task = Task.Run(() =>
        {
            for (int i = 0; i < 10; i++)
            {
                if (cts.Token.IsCancellationRequested)
                {
                    Console.WriteLine("La tarea fue cancelada");
                    return;
                }
                Console.WriteLine($"Procesando {i}");
                Thread.Sleep(500);
            }
        }, cts.Token);

        Thread.Sleep(2000); // Simula algún trabajo
        cts.Cancel(); // Solicita la cancelación

        try
        {
            task.Wait();
        }
        catch (AggregateException ex)
        {
            foreach (var innerEx in ex.InnerExceptions)
            {
                Console.WriteLine(innerEx.Message);
            }
        }
    }
}
  • Explicación:
    • CancellationTokenSource: Proporciona una forma de señalar la cancelación.
    • cts.Token.IsCancellationRequested: Verifica si se ha solicitado la cancelación.

Conclusión

La Biblioteca de Paralelismo de Tareas (TPL) en C# proporciona una forma poderosa y flexible de escribir código paralelo y asincrónico. Al comprender las tareas, continuaciones, manejo de excepciones y características avanzadas como la cancelación, puedes crear aplicaciones eficientes y receptivas. Ya seas un principiante o un programador avanzado, dominar la TPL mejorará significativamente tu capacidad para escribir código C# de alto rendimiento.
