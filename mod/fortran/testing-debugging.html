Pruebas y Depuración en Fortran
Presentación y Revisión ►

Introducción

Las pruebas y la depuración son aspectos cruciales del desarrollo de software, asegurando que tus programas en Fortran funcionen correctamente y de manera eficiente. Esta sección cubrirá los conceptos básicos de las pruebas y la depuración, progresando desde niveles principiantes hasta avanzados.

Conceptos Básicos de las Pruebas

¿Qué son las Pruebas?

  • Definición: Las pruebas son el proceso de ejecutar un programa con la intención de encontrar errores.
  • Propósito: Asegurar que el programa se comporte como se espera bajo diversas condiciones.

Tipos de Pruebas

  • Pruebas Unitarias: Pruebas de componentes o funciones individuales.
  • Pruebas de Integración: Pruebas de partes combinadas de una aplicación para asegurar que funcionen juntas.
  • Pruebas del Sistema: Pruebas del sistema completo como un todo.
  • Pruebas de Aceptación: Pruebas para asegurar que el sistema cumpla con los requisitos.

Escribir Casos de Prueba

  • Identificar Escenarios de Prueba: Determinar qué necesita ser probado.
  • Crear Datos de Prueba: Preparar datos de entrada para las pruebas.
  • Definir Resultados Esperados: Especificar cuál debería ser la salida correcta.

Ejemplo de un Caso de Prueba Simple

program test_addition
  implicit none
  integer :: result

  result = add(2, 3)
  if (result /= 5) then
    print *, "Prueba Fallida: Se esperaba 5, se obtuvo ", result
  else
    print *, "Prueba Exitosa"
  end if
end program test_addition

function add(a, b)
  integer :: add
  integer, intent(in) :: a, b
  add = a + b
end function add
  • Explicación: Este programa prueba la función add. Verifica si sumar 2 y 3 resulta en 5.

Conceptos Básicos de la Depuración

¿Qué es la Depuración?

  • Definición: La depuración es el proceso de identificar, analizar y eliminar errores de un programa.
  • Propósito: Corregir errores y asegurar que el programa funcione correctamente.

Técnicas Comunes de Depuración

  • Sentencias Print: Insertar sentencias print para mostrar valores de variables y el flujo del programa.
  • Uso de Depuradores: Utilizar herramientas como GDB (GNU Debugger) para una depuración más avanzada.
  • Revisión de Código: Revisar el código con compañeros para encontrar posibles problemas.

Ejemplo de Depuración con Sentencias Print

program debug_example
  implicit none
  integer :: a, b, result

  a = 5
  b = 0
  print *, "Antes de la división: a =", a, ", b =", b
  result = a / b
  print *, "Después de la división: result =", result
end program debug_example
  • Explicación: Este programa fallará debido a una división por cero. Las sentencias print ayudan a identificar los valores de a y b antes de que ocurra el error.

Técnicas Avanzadas de Pruebas

Pruebas Automatizadas

  • Definición: Uso de scripts y herramientas para ejecutar pruebas automáticamente.
  • Herramientas: Frameworks como pFUnit para Fortran.

Ejemplo de Pruebas Automatizadas con pFUnit

module test_addition
  use pfunit
  implicit none

  @test
  subroutine test_add
    integer :: result
    result = add(2, 3)
    @assertEqual(result, 5)
  end subroutine test_add

  function add(a, b)
    integer :: add
    integer, intent(in) :: a, b
    add = a + b
  end function add
end module test_addition
  • Explicación: Este módulo utiliza pFUnit para automatizar la prueba de la función add.

Técnicas Avanzadas de Depuración

Uso de GDB con Fortran

  • Establecer Puntos de Interrupción: Pausar la ejecución en líneas específicas.
  • Inspeccionar Variables: Verificar los valores de las variables en diferentes puntos.
  • Ejecutar Código Paso a Paso: Ejecutar el código línea por línea para encontrar errores.

Ejemplo de Uso de GDB

  1. Compilar con Información de Depuración: 
    gfortran -g -o debug_example debug_example.f90
  2. Ejecutar GDB: 
    gdb ./debug_example
  3. Establecer Punto de Interrupción y Ejecutar: 
    (gdb) break debug_example.f90:6
(gdb) run

Conclusión

Las pruebas y la depuración son habilidades esenciales para cualquier programador de Fortran. Comenzando con técnicas básicas como escribir casos de prueba y usar sentencias print, puedes progresar a métodos más avanzados como pruebas automatizadas con pFUnit y el uso de GDB para la depuración. Al dominar estas técnicas, puedes asegurar que tus programas en Fortran sean robustos, confiables y eficientes.
