Subrutinas en Fortran
Funciones ►

Las subrutinas son un concepto fundamental en Fortran, que te permite encapsular y reutilizar código. Esta sección te guiará desde los conceptos básicos hasta el uso avanzado de subrutinas en Fortran.

Introducción a las Subrutinas

Las subrutinas en Fortran son similares a las funciones en otros lenguajes de programación. Te permiten descomponer tu programa en piezas más pequeñas y manejables. Las subrutinas pueden tomar argumentos, realizar operaciones y devolver resultados.

Conceptos Clave

  • Definición: Una subrutina se define usando la palabra clave SUBROUTINE.
  • Argumentos: Las subrutinas pueden aceptar argumentos de entrada y devolver resultados a través de estos argumentos.
  • Llamada: Las subrutinas se invocan usando la instrucción CALL.
  • Ámbito: Las variables dentro de una subrutina son locales a esa subrutina a menos que se compartan explícitamente.

Estructura Básica de una Subrutina

Aquí tienes un ejemplo simple para ilustrar la estructura básica de una subrutina en Fortran.

PROGRAM Main
    CALL Greet
END PROGRAM Main

SUBROUTINE Greet
    PRINT *, "¡Hola, Mundo!"
END SUBROUTINE Greet

Explicación

  • PROGRAM Main: El bloque principal del programa.
  • CALL Greet: Invoca la subrutina Greet.
  • SUBROUTINE Greet: Define la subrutina llamada Greet.
  • PRINT: Muestra "¡Hola, Mundo!" en la consola.
  • END SUBROUTINE Greet: Marca el final de la subrutina.

Subrutinas con Argumentos

Las subrutinas pueden tomar argumentos para realizar operaciones sobre datos de entrada.

PROGRAM Main
    INTEGER :: num1, num2, result
    num1 = 10
    num2 = 20
    CALL Add(num1, num2, result)
    PRINT *, "La suma es: ", result
END PROGRAM Main

SUBROUTINE Add(a, b, sum)
    INTEGER, INTENT(IN) :: a, b
    INTEGER, INTENT(OUT) :: sum
    sum = a + b
END SUBROUTINE Add

Explicación

  • Argumentos: a, b y sum son argumentos de la subrutina Add.
  • INTENT(IN): Especifica que a y b son argumentos de entrada.
  • INTENT(OUT): Especifica que sum es un argumento de salida.
  • sum = a + b: Realiza la suma y almacena el resultado en sum.

Características Avanzadas de las Subrutinas

Argumentos Opcionales

Las subrutinas pueden tener argumentos opcionales, que no son necesarios cuando se llama a la subrutina.

PROGRAM Main
    CALL PrintMessage("¡Hola, Mundo!")
    CALL PrintMessage()
END PROGRAM Main

SUBROUTINE PrintMessage(msg)
    CHARACTER(LEN=*), OPTIONAL :: msg
    IF (PRESENT(msg)) THEN
        PRINT *, msg
    ELSE
        PRINT *, "No se proporcionó ningún mensaje."
    END IF
END SUBROUTINE PrintMessage

Explicación

  • OPTIONAL: Indica que msg es un argumento opcional.
  • PRESENT: Verifica si se proporcionó el argumento opcional msg.

Subrutinas Recursivas

Las subrutinas pueden llamarse a sí mismas recursivamente.

PROGRAM Main
    INTEGER :: result
    result = Factorial(5)
    PRINT *, "El factorial de 5 es: ", result
END PROGRAM Main

RECURSIVE SUBROUTINE Factorial(n, result)
    INTEGER, INTENT(IN) :: n
    INTEGER, INTENT(OUT) :: result
    IF (n <= 1) THEN
        result = 1
    ELSE
        CALL Factorial(n - 1, result)
        result = n * result
    END IF
END SUBROUTINE Factorial

Explicación

  • RECURSIVE: Indica que la subrutina puede llamarse a sí misma.
  • Factorial: Calcula el factorial de n usando recursión.

Resumen

Las subrutinas en Fortran son herramientas poderosas para organizar y reutilizar código. Pueden tomar argumentos, devolver resultados e incluso llamarse a sí mismas recursivamente. Al dominar las subrutinas, puedes escribir programas en Fortran más modulares, legibles y mantenibles.

Puntos Clave

  • Las subrutinas ayudan a descomponer programas complejos en piezas manejables.
  • Pueden aceptar argumentos de entrada y devolver resultados.
  • Los argumentos opcionales y la recursión añaden flexibilidad al uso de subrutinas.

Practica escribiendo y usando subrutinas para volverte competente en la programación en Fortran.
