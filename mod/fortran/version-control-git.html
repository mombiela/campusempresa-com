Introducción al Control de Versiones

  • Definición: El control de versiones es un sistema que registra los cambios en un archivo o conjunto de archivos a lo largo del tiempo para que puedas recordar versiones específicas más tarde.
  • Importancia: Permite que múltiples personas trabajen en un proyecto simultáneamente, rastrea cambios y ayuda a gestionar el historial del proyecto.

Empezando con Git

  • ¿Qué es Git?: Git es un sistema de control de versiones distribuido diseñado para manejar todo, desde proyectos pequeños hasta muy grandes, con velocidad y eficiencia.
  • Instalación:
    • Windows: Descarga desde git-scm.com y sigue las instrucciones de instalación.
    • macOS: Instala a través de Homebrew con brew install git.
    • Linux: Usa el gestor de paquetes, por ejemplo, sudo apt-get install git para sistemas basados en Debian.

Comandos Básicos de Git

  • Configurando Git: 
    git config --global user.name "Tu Nombre"
git config --global user.email "tu.email@ejemplo.com"
  • Inicializando un Repositorio: 
    git init
  • Clonando un Repositorio: 
    git clone https://github.com/username/repository.git
  • Verificando el Estado: 
    git status
  • Añadiendo Archivos: 
    git add filename
  • Realizando Commit de Cambios: 
    git commit -m "Mensaje del commit"
  • Pushing de Cambios: 
    git push origin main

Trabajando con Ramas

  • Creando una Rama: 
    git branch new-branch
  • Cambiando de Rama: 
    git checkout new-branch
  • Fusionando Ramas: 
    git checkout main
git merge new-branch

Colaborando con Git

  • Forking de un Repositorio: Haz un fork de un repositorio en GitHub para crear tu propia copia.
  • Pull Requests: Envía cambios al repositorio original a través de pull requests.
  • Manejando Conflictos de Fusión:
    • Identificar Conflicto: Git marcará los conflictos en los archivos.
    • Resolver Conflicto: Edita los archivos para resolver los conflictos.
    • Commit de la Resolución: 
      git add resolved-file
git commit -m "Resolved merge conflict"

Funciones Avanzadas de Git

  • Guardando Cambios: 
    git stash
git stash pop
  • Rebasing: 
    git rebase main
  • Etiquetado: 
    git tag -a v1.0 -m "Versión 1.0"
git push origin v1.0

Ejemplo: Gestionando un Proyecto en Fortran con Git

Ejemplo Paso a Paso

  1. Inicializar Repositorio: 
    git init fortran-project
cd fortran-project
  2. Crear un Archivo Fortran: 
    ! hello.f90
PROGRAM HelloWorld
  PRINT *, "Hello, World!"
END PROGRAM HelloWorld
  3. Añadir y Realizar Commit: 
    git add hello.f90
git commit -m "Add HelloWorld program"
  4. Crear una Nueva Rama para la Función: 
    git checkout -b feature-addition
  5. Modificar el Archivo Fortran: 
    ! hello.f90
PROGRAM HelloWorld
  PRINT *, "Hello, World!"
  PRINT *, "This is a new feature."
END PROGRAM HelloWorld
  6. Añadir y Realizar Commit de los Cambios: 
    git add hello.f90
git commit -m "Add new feature to HelloWorld program"
  7. Fusionar la Rama de la Función: 
    git checkout main
git merge feature-addition

Resumen

En este tema, cubrimos los conceptos básicos del control de versiones con Git, incluyendo la instalación, comandos básicos, ramas, colaboración y funciones avanzadas. También proporcionamos un ejemplo práctico de cómo gestionar un proyecto en Fortran usando Git. Al dominar estas habilidades, puedes gestionar eficientemente tu base de código, colaborar con otros y mantener un historial robusto del proyecto.
