Introducción

En esta sección, profundizaremos en arquitecturas avanzadas de Redes Neuronales Convolucionales (CNN) utilizando TensorFlow. Estas arquitecturas han sido diseñadas para abordar tareas complejas de reconocimiento de imágenes y han establecido puntos de referencia en varias competiciones. Comprender estas arquitecturas te ayudará a construir modelos más eficientes y precisos.

Conceptos Clave

  • Redes Residuales (ResNet)
  • Redes Inception
  • MobileNets
  • EfficientNet

Redes Residuales (ResNet)

ResNet introduce el concepto de aprendizaje residual para abordar el problema del gradiente que se desvanece en redes profundas.

Características Clave

  • Conexiones de Salto: Permite que la red aprenda funciones de identidad, facilitando el entrenamiento de redes muy profundas.
  • Bloques Residuales: Consiste en unas pocas capas convolucionales con una conexión de salto que pasa por alto una o más capas.

Ejemplo de Código

import tensorflow as tf
from tensorflow.keras.layers import Input, Conv2D, BatchNormalization, Activation, Add
from tensorflow.keras.models import Model

def residual_block(x, filters, kernel_size=3, stride=1):
    shortcut = x
    x = Conv2D(filters, kernel_size, strides=stride, padding='same')(x)
    x = BatchNormalization()(x)
    x = Activation('relu')(x)
    x = Conv2D(filters, kernel_size, strides=1, padding='same')(x)
    x = BatchNormalization()(x)
    x = Add()([x, shortcut])
    x = Activation('relu')(x)
    return x

input_tensor = Input(shape=(224, 224, 3))
x = Conv2D(64, 7, strides=2, padding='same')(input_tensor)
x = BatchNormalization()(x)
x = Activation('relu')(x)
x = residual_block(x, 64)
model = Model(inputs=input_tensor, outputs=x)
model.summary()

Explicación

  • Conv2D: Capa convolucional con filtros y tamaño de kernel especificados.
  • BatchNormalization: Normaliza la salida de la capa anterior.
  • Activation: Aplica la función de activación ReLU.
  • Add: Añade la conexión de salto a la salida de las capas convolucionales.

Redes Inception

Las Redes Inception tienen como objetivo mejorar la utilización de los recursos computacionales dentro de la red.

Características Clave

  • Módulos Inception: Combina múltiples operaciones convolucionales con diferentes tamaños de kernel en paralelo.
  • Reducción de Dimensionalidad: Utiliza convoluciones 1x1 para reducir el número de parámetros.

Ejemplo de Código

from tensorflow.keras.layers import concatenate

def inception_module(x, filters):
    branch1x1 = Conv2D(filters, 1, padding='same', activation='relu')(x)
    
    branch3x3 = Conv2D(filters, 1, padding='same', activation='relu')(x)
    branch3x3 = Conv2D(filters, 3, padding='same', activation='relu')(branch3x3)
    
    branch5x5 = Conv2D(filters, 1, padding='same', activation='relu')(x)
    branch5x5 = Conv2D(filters, 5, padding='same', activation='relu')(branch5x5)
    
    branch_pool = tf.keras.layers.MaxPooling2D(3, strides=1, padding='same')(x)
    branch_pool = Conv2D(filters, 1, padding='same', activation='relu')(branch_pool)
    
    x = concatenate([branch1x1, branch3x3, branch5x5, branch_pool], axis=-1)
    return x

input_tensor = Input(shape=(224, 224, 3))
x = inception_module(input_tensor, 32)
model = Model(inputs=input_tensor, outputs=x)
model.summary()

Explicación

  • concatenate: Fusiona las salidas de diferentes ramas.
  • Módulo Inception: Combina múltiples capas convolucionales con diferentes tamaños de kernel y una capa de pooling.

MobileNets

MobileNets están diseñadas para aplicaciones de visión móvil y embebida.

Características Clave

  • Convoluciones Separables en Profundidad: Reduce el número de parámetros y el cómputo.
  • Multiplicador de Ancho: Controla el ancho de la red.
  • Multiplicador de Resolución: Controla la resolución de la imagen de entrada.

Ejemplo de Código

from tensorflow.keras.layers import DepthwiseConv2D

def depthwise_separable_conv_block(x, filters, stride=1):
    x = DepthwiseConv2D(kernel_size=3, strides=stride, padding='same')(x)
    x = BatchNormalization()(x)
    x = Activation('relu')(x)
    x = Conv2D(filters, kernel_size=1, strides=1, padding='same')(x)
    x = BatchNormalization()(x)
    x = Activation('relu')(x)
    return x

input_tensor = Input(shape=(224, 224, 3))
x = depthwise_separable_conv_block(input_tensor, 64)
model = Model(inputs=input_tensor, outputs=x)
model.summary()

Explicación

  • DepthwiseConv2D: Aplica un único filtro convolucional por canal de entrada.
  • Convoluciones Separables en Profundidad: Combina convoluciones en profundidad y puntuales para reducir el cómputo.

EfficientNet

EfficientNet escala el ancho, la profundidad y la resolución de la red de manera equilibrada.

Características Clave

  • Escalado Compuesto: Equilibra la profundidad, el ancho y la resolución de la red.
  • EfficientNet-B0 a B7: Diferentes versiones con creciente complejidad y precisión.

Ejemplo de Código

import tensorflow as tf
from tensorflow.keras.applications import EfficientNetB0

model = EfficientNetB0(weights='imagenet', include_top=False, input_shape=(224, 224, 3))
model.summary()

Explicación

  • EfficientNetB0: Modelo EfficientNet preentrenado.
  • include_top=False: Excluye la capa de clasificación final.

Conclusión

En esta sección, exploramos arquitecturas avanzadas de CNN como ResNet, Redes Inception, MobileNets y EfficientNet. Cada arquitectura tiene sus características y ventajas únicas, lo que las hace adecuadas para diferentes tipos de tareas de reconocimiento de imágenes. Comprender estas arquitecturas te permitirá construir modelos más eficientes y precisos utilizando TensorFlow.
