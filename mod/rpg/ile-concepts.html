Introducción a ILE

Integrated Language Environment (ILE) es un modelo de programación introducido por IBM para los sistemas AS/400 (ahora IBM i). Permite la integración de diferentes lenguajes de programación y proporciona una forma más modular y eficiente de desarrollar aplicaciones.

Conceptos Clave de ILE

  • Módulos: La unidad más pequeña de código en ILE, que puede ser escrita en diferentes lenguajes (RPG, COBOL, C, etc.). Los módulos son compilados pero no ejecutables por sí solos.
  • Programas: Objetos ejecutables creados al enlazar uno o más módulos juntos.
  • Programas de Servicio: Colecciones de procedimientos y funciones reutilizables que pueden ser compartidos entre diferentes programas.
  • Directorio de Enlace: Una lista de módulos y programas de servicio que pueden ser utilizados durante el proceso de enlace.
  • Grupos de Activación: Contenedores lógicos que gestionan el entorno de ejecución para programas y programas de servicio ILE.

Módulos

Los módulos son los bloques de construcción de ILE. Contienen procedimientos y funciones que pueden ser reutilizados en diferentes programas.

Creación de un Módulo

Para crear un módulo en RPG, se utiliza el comando CRTRPGMOD. Aquí hay un ejemplo simple de un módulo RPG:

**FREE
Dcl-Proc HelloWorld;
  Dcl-S message Char(50);
  message = '¡Hola, Mundo!';
  Dsply message;
End-Proc;

En este ejemplo:

  • Dcl-Proc declara un procedimiento llamado HelloWorld.
  • Dcl-S declara una variable llamada message.
  • Dsply muestra el mensaje en la pantalla.

Programas

Los programas se crean al enlazar uno o más módulos juntos. Esto se hace utilizando el comando CRTPGM.

Creación de un Programa

Aquí hay un ejemplo de cómo crear un programa a partir de un módulo:

  1. Crear el módulo:

    CRTRPGMOD MODULE(MYLIB/HELLOMOD) SRCFILE(MYLIB/QRPGLESRC) SRCMBR(HELLOMOD)

  2. Crear el programa:

    CRTPGM PGM(MYLIB/HELLOPGM) MODULE(MYLIB/HELLOMOD)

Programas de Servicio

Los programas de servicio son colecciones de procedimientos que pueden ser utilizados por múltiples programas. Se crean utilizando el comando CRTSRVPGM.

Creación de un Programa de Servicio

Aquí hay un ejemplo de cómo crear un programa de servicio:

  1. Crear el módulo:

    CRTRPGMOD MODULE(MYLIB/MYSRVPGM) SRCFILE(MYLIB/QRPGLESRC) SRCMBR(MYSRVPGM)

  2. Crear el programa de servicio:

    CRTSRVPGM SRVPGM(MYLIB/MYSRVPGM) MODULE(MYLIB/MYSRVPGM)

Directorio de Enlace

Un directorio de enlace se utiliza para simplificar el proceso de enlace de módulos y programas de servicio. Contiene una lista de módulos y programas de servicio que pueden ser referenciados durante el proceso de enlace.

Creación de un Directorio de Enlace

Aquí hay un ejemplo de cómo crear y utilizar un directorio de enlace:

  1. Crear el directorio de enlace:

    CRTBNDDIR BNDDIR(MYLIB/MYBNDDIR)

  2. Añadir un módulo o programa de servicio al directorio de enlace:

    ADDBNDDIRE BNDDIR(MYLIB/MYBNDDIR) OBJ((MYLIB/HELLOMOD *MODULE))

  3. Utilizar el directorio de enlace al crear un programa:

    CRTPGM PGM(MYLIB/HELLOPGM) BNDDIR(MYLIB/MYBNDDIR)

Grupos de Activación

Los grupos de activación gestionan el entorno de ejecución para programas y programas de servicio ILE. Controlan recursos como la memoria y el uso de archivos.

Tipos de Grupos de Activación

  • Grupos de Activación Nombrados: Nombrados explícitamente por el desarrollador.
  • *NEW: Cada llamada crea un nuevo grupo de activación.
  • *CALLER: Utiliza el grupo de activación del programa que llama.
  • *DFTACTGRP: Utiliza el grupo de activación predeterminado, típicamente para programas OPM (Original Program Model).

Especificación de un Grupo de Activación

Aquí hay un ejemplo de cómo especificar un grupo de activación en un programa RPG:

**FREE
Ctl-Opt ActGrp('MYACTGRP');

Dcl-Proc HelloWorld;
  Dcl-S message Char(50);
  message = '¡Hola, Mundo!';
  Dsply message;
End-Proc;

En este ejemplo, la directiva Ctl-Opt especifica que el programa debe ejecutarse en el grupo de activación llamado MYACTGRP.

Conclusión

Entender los conceptos y la arquitectura de ILE es crucial para desarrollar aplicaciones eficientes y modulares en sistemas IBM i. Al dominar módulos, programas, programas de servicio, directorios de enlace y grupos de activación, puedes crear aplicaciones robustas y mantenibles. Practica creando y enlazando estos componentes para solidificar tu comprensión y mejorar tus habilidades de programación en RPG.
