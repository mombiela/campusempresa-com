Programas de Servicio
Directorios de Enlace ►

Los Programas de Servicio son una característica poderosa en RPG que te permite encapsular y reutilizar código en múltiples aplicaciones. Este tema te guiará a través de la comprensión, creación y uso de programas de servicio desde un nivel principiante hasta uno avanzado.

Introducción a los Programas de Servicio

Los Programas de Servicio en RPG son similares a las bibliotecas de enlace dinámico (DLL) en otros lenguajes de programación. Te permiten agrupar procedimientos y funciones relacionados en un solo módulo que puede ser utilizado por múltiples programas.

  • Encapsulación: Agrupa procedimientos relacionados.
  • Reusabilidad: Usa los mismos procedimientos en diferentes programas sin duplicar código.
  • Mantenibilidad: Actualiza procedimientos en un solo lugar, y todos los programas que usan el programa de servicio usarán automáticamente el código actualizado.

Creación de un Programa de Servicio

Definiendo Procedimientos

Para crear un programa de servicio, primero necesitas definir los procedimientos que deseas incluir. Aquí tienes un ejemplo simple de cómo definir procedimientos en un módulo RPG:

**free
// Procedimiento para sumar dos números
dcl-proc AddNumbers;
  dcl-pi *n int(10);
    num1 int(10);
    num2 int(10);
  end-pi;

  return num1 + num2;
end-proc;

// Procedimiento para restar dos números
dcl-proc SubtractNumbers;
  dcl-pi *n int(10);
    num1 int(10);
    num2 int(10);
  end-pi;

  return num1 - num2;
end-proc;

Creación del Módulo

Compila el código anterior en un módulo. En IBM i, puedes usar el comando CRTRPGMOD:

CRTRPGMOD MODULE(MYLIB/MYSRVPGM) SRCFILE(MYLIB/QRPGLESRC) SRCMBR(MYSRVPGM)

Vinculación del Módulo en un Programa de Servicio

Una vez que tengas tu módulo, necesitas crear un programa de servicio usando el comando CRTSRVPGM:

CRTSRVPGM SRVPGM(MYLIB/MYSRVPGM) MODULE(MYLIB/MYSRVPGM) EXPORT(*ALL)

Uso de un Programa de Servicio

Vinculación de un Programa de Servicio a un Programa

Para usar los procedimientos en un programa de servicio, necesitas vincular el programa de servicio a tu programa RPG. Aquí tienes un ejemplo de cómo llamar a los procedimientos desde un programa de servicio:

**free
// Prototipo para el procedimiento AddNumbers
dcl-pr AddNumbers int(10);
  num1 int(10);
  num2 int(10);
end-pr;

// Prototipo para el procedimiento SubtractNumbers
dcl-pr SubtractNumbers int(10);
  num1 int(10);
  num2 int(10);
end-pr;

dcl-s result int(10);

result = AddNumbers(10, 5);
dsply result; // Muestra 15

result = SubtractNumbers(10, 5);
dsply result; // Muestra 5

*inlr = *on;

Directorio de Vinculación

Necesitas especificar el programa de servicio en el directorio de vinculación. Usa el comando CRTBNDCL:

CRTBNDCL PGM(MYLIB/MYPGM) SRCFILE(MYLIB/QCLLESRC) SRCMBR(MYPGM) BNDDIR(MYLIB/MYBNDDIR)

Asegúrate de que el directorio de vinculación incluya el programa de servicio:

ADDBNDDIRE BNDDIR(MYLIB/MYBNDDIR) OBJ((MYLIB/MYSRVPGM *SRVPGM))

Uso Avanzado

Versionado de Programas de Servicio

A medida que tus aplicaciones evolucionan, puede que necesites actualizar tus programas de servicio. El versionado te permite gestionar cambios sin romper aplicaciones existentes.

  • Compatibilidad hacia atrás: Asegúrate de que las nuevas versiones de los procedimientos no rompan la funcionalidad existente.
  • Listas de Exportación: Usa listas de exportación para controlar qué procedimientos están disponibles para los programas que llaman.

Ejemplo de Listas de Exportación

Define una lista de exportación en tu programa de servicio:

CRTSRVPGM SRVPGM(MYLIB/MYSRVPGM) MODULE(MYLIB/MYSRVPGM) EXPORT(*SRCFILE) SRCFILE(MYLIB/QSRVSRC) SRCMBR(EXPORTS)

En el miembro de origen EXPORTS:

AddNumbers
SubtractNumbers

Vinculación Dinámica

La vinculación dinámica te permite llamar a procedimientos en un programa de servicio en tiempo de ejecución, proporcionando flexibilidad en cómo usas los programas de servicio.

**free
dcl-pr DynamicAddNumbers extpgm('MYSRVPGM');
  num1 int(10);
  num2 int(10);
end-pr;

dcl-s result int(10);

result = DynamicAddNumbers(10, 5);
dsply result; // Muestra 15

*inlr = *on;

Conclusión

Los Programas de Servicio en RPG proporcionan una forma robusta de encapsular y reutilizar código, haciendo que tus aplicaciones sean más modulares y mantenibles. Al comprender cómo crear, usar y gestionar programas de servicio, puedes mejorar significativamente la eficiencia y escalabilidad de tus aplicaciones RPG. A medida que avances, considera explorar características más complejas como el versionado y la vinculación dinámica para aprovechar al máximo el poder de los programas de servicio.
