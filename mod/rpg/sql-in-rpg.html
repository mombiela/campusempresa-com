Introducción a SQL en RPG
SQL Embebido ►

Visión General

SQL (Structured Query Language) es una herramienta poderosa para gestionar y manipular bases de datos. En el contexto de RPG (Report Program Generator), integrar SQL puede mejorar significativamente las capacidades de procesamiento de datos. Esta sección te introducirá a los conceptos básicos de usar SQL dentro de programas RPG.

Conceptos Clave

  • SQL Embebido: Sentencias SQL embebidas directamente dentro del código RPG.
  • Precompiladores SQL: Herramientas que procesan sentencias SQL embebidas antes de que el programa RPG sea compilado.
  • SQLCA (Área de Comunicación SQL): Una estructura usada para manejar el estado de ejecución de SQL y errores.
  • Operaciones CRUD: Operaciones básicas de SQL - Crear, Leer, Actualizar, Eliminar.

Configuración de SQL en RPG

Para usar SQL en RPG, necesitas asegurarte de que tu entorno esté configurado correctamente:

  1. Incluir Precompilador SQL: Asegúrate de que tu compilador RPG soporte precompilación SQL.
  2. Conexión a la Base de Datos: Establece una conexión a tu base de datos.

Operaciones Básicas de SQL en RPG

Seleccionar Datos

Para recuperar datos de una base de datos, usas la sentencia SELECT.

Dcl-S CustomerName Char(50);
Exec SQL
    SELECT Name INTO :CustomerName
    FROM Customers
    WHERE CustomerID = 1;
  • Explicación:
    • Dcl-S CustomerName Char(50);: Declara una variable para almacenar el nombre del cliente.
    • Exec SQL: Indica el inicio de una sentencia SQL embebida.
    • SELECT Name INTO :CustomerName: Selecciona el campo Name de la tabla Customers y lo almacena en la variable CustomerName.
    • WHERE CustomerID = 1;: Especifica la condición para filtrar los datos.

Insertar Datos

Para añadir nuevos registros a una tabla, usa la sentencia INSERT.

Dcl-S CustomerID Int(10);
Dcl-S CustomerName Char(50);
CustomerID = 2;
CustomerName = 'John Doe';

Exec SQL
    INSERT INTO Customers (CustomerID, Name)
    VALUES (:CustomerID, :CustomerName);
  • Explicación:
    • Dcl-S CustomerID Int(10);: Declara una variable para el ID del cliente.
    • Dcl-S CustomerName Char(50);: Declara una variable para el nombre del cliente.
    • CustomerID = 2;: Asigna un valor a la variable CustomerID.
    • CustomerName = 'John Doe';: Asigna un valor a la variable CustomerName.
    • INSERT INTO Customers (CustomerID, Name): Especifica la tabla y las columnas en las que insertar datos.
    • VALUES (:CustomerID, :CustomerName);: Proporciona los valores a insertar.

Actualizar Datos

Para modificar registros existentes, usa la sentencia UPDATE.

Dcl-S CustomerID Int(10);
Dcl-S NewName Char(50);
CustomerID = 2;
NewName = 'Jane Doe';

Exec SQL
    UPDATE Customers
    SET Name = :NewName
    WHERE CustomerID = :CustomerID;
  • Explicación:
    • Dcl-S CustomerID Int(10);: Declara una variable para el ID del cliente.
    • Dcl-S NewName Char(50);: Declara una variable para el nuevo nombre.
    • CustomerID = 2;: Asigna un valor a la variable CustomerID.
    • NewName = 'Jane Doe';: Asigna un valor a la variable NewName.
    • UPDATE Customers: Especifica la tabla a actualizar.
    • SET Name = :NewName: Establece el nuevo valor para la columna Name.
    • WHERE CustomerID = :CustomerID;: Especifica la condición para identificar el registro a actualizar.

Eliminar Datos

Para eliminar registros de una tabla, usa la sentencia DELETE.

Dcl-S CustomerID Int(10);
CustomerID = 2;

Exec SQL
    DELETE FROM Customers
    WHERE CustomerID = :CustomerID;
  • Explicación:
    • Dcl-S CustomerID Int(10);: Declara una variable para el ID del cliente.
    • CustomerID = 2;: Asigna un valor a la variable CustomerID.
    • DELETE FROM Customers: Especifica la tabla de la que eliminar.
    • WHERE CustomerID = :CustomerID;: Especifica la condición para identificar el registro a eliminar.

Manejo de Errores con SQLCA

SQLCA (Área de Comunicación SQL) se usa para manejar errores e información de estado para operaciones SQL.

Dcl-DS SQLCA;
  SQLCODE Int(10);
  SQLSTATE Char(5);
End-DS;

Exec SQL
    SELECT Name INTO :CustomerName
    FROM Customers
    WHERE CustomerID = 1;

If SQLCA.SQLCODE <> 0;
  // Manejar error
  Dsply ('Error: ' + %Char(SQLCA.SQLCODE));
EndIf;
  • Explicación:
    • Dcl-DS SQLCA;: Declara la estructura de datos SQLCA.
    • SQLCODE Int(10);: Declara una variable para almacenar el código de retorno de SQL.
    • SQLSTATE Char(5);: Declara una variable para almacenar el estado de SQL.
    • If SQLCA.SQLCODE <> 0;: Verifica si hubo un error durante la operación SQL.
    • Dsply ('Error: ' + %Char(SQLCA.SQLCODE));: Muestra el código de error.

Conclusión

Integrar SQL con RPG puede mejorar enormemente tu capacidad para gestionar y manipular datos dentro de tus aplicaciones. Al entender y utilizar operaciones básicas de SQL como SELECT, INSERT, UPDATE y DELETE, puedes realizar una amplia gama de tareas de procesamiento de datos. Además, manejar errores usando SQLCA asegura que tus programas puedan gestionar y responder de manera adecuada a los problemas que surjan durante la ejecución de SQL.
