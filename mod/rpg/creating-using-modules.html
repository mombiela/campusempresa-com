Introducción

Los módulos en RPG (Report Program Generator) son una forma de organizar y reutilizar código. Ayudan a descomponer programas complejos en piezas manejables, mejorando la legibilidad, mantenibilidad y reutilización. Esta sección te guiará a través de los conceptos básicos de la creación y uso de módulos en RPG, desde niveles principiantes hasta avanzados.

¿Qué es un Módulo?

  • Un módulo es una unidad de código autónoma que realiza una tarea específica.
  • Los módulos pueden ser compilados por separado y luego enlazados para formar un programa completo.
  • Promueven la reutilización de código y el diseño modular.

Creación de un Módulo Simple

Guía Paso a Paso

  1. Definir el Módulo:

    • Usa la especificación H para definir el módulo.
    • Usa la especificación P para definir procedimientos dentro del módulo.

  2. Escribir el Código:

    • Escribe la lógica para el procedimiento dentro del módulo.

  3. Compilar el Módulo:

    • Usa los comandos de compilación apropiados para compilar el módulo.

Ejemplo

H NOMAIN

P AddNumbers       B                   EXPORT
D AddNumbers       PI            10I 0
D   num1           10I 0 VALUE
D   num2           10I 0 VALUE

 /FREE
     RETURN num1 + num2;
 /END-FREE
P AddNumbers       E
  • Explicación:
    • H NOMAIN: Indica que este es un módulo y no un programa independiente.
    • P AddNumbers B EXPORT: Comienza la definición de un procedimiento llamado AddNumbers y lo hace disponible para que otros programas lo usen.
    • D AddNumbers PI 10I 0: Define la interfaz del procedimiento con un tipo de retorno de entero de 10 dígitos.
    • D num1 10I 0 VALUE: Define el primer parámetro del procedimiento.
    • D num2 10I 0 VALUE: Define el segundo parámetro del procedimiento.
    • La lógica dentro de /FREE y /END-FREE calcula la suma de num1 y num2 y devuelve el resultado.

Uso de un Módulo

Guía Paso a Paso

  1. Importar el Módulo:

    • Usa la especificación H para importar el módulo.
    • Usa la especificación D para declarar el procedimiento.

  2. Llamar al Procedimiento:

    • Llama al procedimiento desde tu programa principal.

Ejemplo

H DFTACTGRP(*NO) ACTGRP(*NEW)

D AddNumbers       PR            10I 0
D   num1           10I 0 VALUE
D   num2           10I 0 VALUE

 /COPY QRPGLESRC,MYMODULE

D result           S             10I 0

 /FREE
     result = AddNumbers(5: 10);
     dsply result;
     *INLR = *ON;
 /END-FREE
  • Explicación:
    • H DFTACTGRP(*NO) ACTGRP(*NEW): Define el grupo de activación para el programa.
    • D AddNumbers PR 10I 0: Declara el prototipo para el procedimiento AddNumbers.
    • /COPY QRPGLESRC,MYMODULE: Incluye el código fuente del módulo.
    • result = AddNumbers(5: 10): Llama al procedimiento AddNumbers con los parámetros 5 y 10.
    • dsply result: Muestra el resultado.
    • *INLR = *ON: Termina el programa.

Conceptos Avanzados de Módulos

Diseño Modular

  • Descompone tareas complejas en módulos más pequeños y reutilizables.
  • Usa módulos para encapsular lógica de negocio, acceso a datos y funciones utilitarias.

Manejo de Errores en Módulos

  • Implementa manejo de errores dentro de los módulos para asegurar robustez.
  • Usa funciones integradas como MONITOR, ON-ERROR y ENDMON para manejar excepciones.

Ejemplo

P SafeDivide       B                   EXPORT
D SafeDivide       PI            10I 0
D   num1           10I 0 VALUE
D   num2           10I 0 VALUE

D result           S             10I 0

 /FREE
     MONITOR;
         result = num1 / num2;
     ON-ERROR;
         result = 0;
     ENDMON;
     RETURN result;
 /END-FREE
P SafeDivide       E
  • Explicación:
    • MONITOR: Comienza el bloque de manejo de errores.
    • ON-ERROR: Especifica la acción a tomar si ocurre un error.
    • ENDMON: Termina el bloque de manejo de errores.

Conclusión

Los módulos en RPG son esenciales para crear código organizado, mantenible y reutilizable. Al descomponer programas complejos en módulos más pequeños, puedes mejorar la estructura y legibilidad de tu código. Esta sección cubrió los conceptos básicos de la creación y uso de módulos, así como algunos conceptos avanzados como el diseño modular y el manejo de errores. Practica creando y usando módulos para volverte competente en la programación modular con RPG.
