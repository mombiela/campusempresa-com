Introducción

En este estudio de caso, exploraremos el desarrollo de un Sistema de Gestión de Inventario utilizando RPG (Report Program Generator). Este sistema ayudará a gestionar los niveles de inventario, rastrear existencias y generar informes. Cubriremos todo el proceso desde conceptos básicos hasta características avanzadas, asegurándonos de que tanto principiantes como programadores avanzados puedan seguirlo.

Conceptos Clave

  • Gestión de Inventario: El proceso de ordenar, almacenar y usar el inventario de una empresa.
  • Programación RPG: Un lenguaje de programación de alto nivel utilizado para aplicaciones empresariales.
  • Interacción con Bases de Datos: Cómo interactuar con bases de datos para almacenar y recuperar datos de inventario.
  • Interfaz de Usuario: Crear interfaces amigables para interactuar con el sistema.

Configuración del Entorno

Antes de comenzar a programar, asegúrate de tener lo siguiente:

  • Un sistema IBM i o un compilador RPG.
  • Acceso a una base de datos (por ejemplo, DB2).
  • Comprensión básica de la sintaxis y estructura de RPG.

Sistema Básico de Gestión de Inventario

Diseño de la Base de Datos

Primero, necesitamos diseñar las tablas de la base de datos para nuestro sistema de inventario. Crearemos una tabla llamada INVENTORY con los siguientes campos:

  • ITEM_ID (Clave Primaria)
  • ITEM_NAME
  • QUANTITY
  • PRICE
CREATE TABLE INVENTORY (
    ITEM_ID INT PRIMARY KEY,
    ITEM_NAME VARCHAR(100),
    QUANTITY INT,
    PRICE DECIMAL(10, 2)
);

Programa Básico en RPG

A continuación, escribiremos un programa RPG simple para agregar artículos al inventario.

H DFTACTGRP(*NO) ACTGRP(*NEW)

D ITEM_ID       S              10I 0
D ITEM_NAME     S             100A
D QUANTITY      S              10I 0
D PRICE         S              10P 2

C     *ENTRY    PLIST
C               PARM                    ITEM_ID
C               PARM                    ITEM_NAME
C               PARM                    QUANTITY
C               PARM                    PRICE

C/EXEC SQL
C+   INSERT INTO INVENTORY (ITEM_ID, ITEM_NAME, QUANTITY, PRICE)
C+   VALUES (:ITEM_ID, :ITEM_NAME, :QUANTITY, :PRICE)
C/END-EXEC

C     *INLR     SETON

Explicación

  • H DFTACTGRP(*NO) ACTGRP(*NEW): Especifica el grupo de activación para el programa.
  • D: Declara variables.
  • *C ENTRY PLIST: Define la lista de parámetros para el programa.
  • C/EXEC SQL: Comienza una declaración SQL embebida.
  • C+ INSERT INTO INVENTORY: Comando SQL para insertar un nuevo registro en la tabla INVENTORY.
  • C/END-EXEC: Termina la declaración SQL embebida.
  • *C INLR SETON: Activa el indicador de último registro, finalizando el programa.

Características Intermedias

Actualización del Inventario

Ahora agregaremos funcionalidad para actualizar la cantidad de un artículo existente.

H DFTACTGRP(*NO) ACTGRP(*NEW)

D ITEM_ID       S              10I 0
D NEW_QUANTITY  S              10I 0

C     *ENTRY    PLIST
C               PARM                    ITEM_ID
C               PARM                    NEW_QUANTITY

C/EXEC SQL
C+   UPDATE INVENTORY
C+   SET QUANTITY = :NEW_QUANTITY
C+   WHERE ITEM_ID = :ITEM_ID
C/END-EXEC

C     *INLR     SETON

Explicación

  • C+ UPDATE INVENTORY: Comando SQL para actualizar el campo QUANTITY de un registro existente en la tabla INVENTORY.

Características Avanzadas

Generación de Informes

Crearemos un programa para generar un informe de todos los artículos en el inventario.

H DFTACTGRP(*NO) ACTGRP(*NEW)

D ITEM_ID       S              10I 0
D ITEM_NAME     S             100A
D QUANTITY      S              10I 0
D PRICE         S              10P 2

C/EXEC SQL
C+   DECLARE C1 CURSOR FOR
C+   SELECT ITEM_ID, ITEM_NAME, QUANTITY, PRICE
C+   FROM INVENTORY
C/END-EXEC

C/EXEC SQL
C+   OPEN C1
C/END-EXEC

C/EXEC SQL
C+   FETCH C1 INTO :ITEM_ID, :ITEM_NAME, :QUANTITY, :PRICE
C/END-EXEC

C     DOU       SQLCOD <> 0
C     ITEM_ID   DSPLY
C     ITEM_NAME DSPLY
C     QUANTITY  DSPLY
C     PRICE     DSPLY

C/EXEC SQL
C+   FETCH C1 INTO :ITEM_ID, :ITEM_NAME, :QUANTITY, :PRICE
C/END-EXEC
C     ENDDO

C/EXEC SQL
C+   CLOSE C1
C/END-EXEC

C     *INLR     SETON

Explicación

  • DECLARE C1 CURSOR FOR: Declara un cursor para seleccionar todos los registros de la tabla INVENTORY.
  • OPEN C1: Abre el cursor.
  • FETCH C1 INTO: Recupera la siguiente fila del cursor en las variables especificadas.
  • DOU SQLCOD <> 0: Bucle hasta que el código SQL no sea cero (indicando que no hay más filas).
  • DSPLY: Muestra los valores de las variables.

Conclusión

En este estudio de caso, hemos construido un Sistema Básico de Gestión de Inventario utilizando RPG. Comenzamos con la configuración del entorno, creando una base de datos y escribiendo programas básicos para agregar y actualizar artículos de inventario. Luego pasamos a características más avanzadas como la generación de informes. Este enfoque integral asegura que tengas una comprensión sólida de los conceptos de programación RPG básicos y avanzados aplicados a un escenario del mundo real.
