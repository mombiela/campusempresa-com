SQL dinámico en RPG
Transacciones de Base de Datos ►

El SQL dinámico es una característica poderosa en RPG que te permite construir y ejecutar sentencias SQL dinámicamente en tiempo de ejecución. Esto es particularmente útil cuando la sentencia SQL exacta no puede determinarse hasta que el programa esté en ejecución.

Conceptos clave

  • SQL estático: Las sentencias SQL están codificadas en el programa y no cambian.
  • SQL dinámico: Las sentencias SQL se construyen y ejecutan en tiempo de ejecución, permitiendo mayor flexibilidad.

Ventajas del SQL dinámico

  • Flexibilidad: Permite la construcción de consultas SQL basadas en la entrada del usuario u otras condiciones en tiempo de ejecución.
  • Adaptabilidad: Puede manejar requisitos de consulta variables sin cambiar el código fuente.

Desventajas del SQL dinámico

  • Complejidad: Más complejo de escribir y depurar en comparación con el SQL estático.
  • Rendimiento: Puede ser más lento debido a la sobrecarga de construir y analizar sentencias SQL en tiempo de ejecución.
  • Seguridad: Mayor riesgo de ataques de inyección SQL si no se maneja adecuadamente.

Construcción de SQL dinámico en RPG

Ejemplo básico

Aquí hay un ejemplo simple de construcción y ejecución de una sentencia SQL dinámica en RPG:

Dcl-S sqlStmt Varchar(500);
Dcl-S custId Char(10);
Dcl-S custName Char(50);

custId = 'CUST001';

sqlStmt = 'SELECT CUSTNAME FROM CUSTOMERS WHERE CUSTID = ''' + custId + '''';

Exec SQL
  Prepare stmt1 From :sqlStmt;

Exec SQL
  Declare c1 Cursor For stmt1;

Exec SQL
  Open c1;

Exec SQL
  Fetch c1 Into :custName;

Exec SQL
  Close c1;

Dsply custName;

Explicación

  1. Declarar Variables: Declarar las variables necesarias para la sentencia SQL y la recuperación de datos.
  2. Construir Sentencia SQL: Construir la sentencia SQL dinámicamente usando concatenación de cadenas.
  3. Preparar Sentencia: Usar Prepare para compilar la sentencia SQL.
  4. Declarar Cursor: Declarar un cursor para la sentencia preparada.
  5. Abrir Cursor: Abrir el cursor para ejecutar la sentencia SQL.
  6. Recuperar Datos: Recuperar datos del cursor en variables RPG.
  7. Cerrar Cursor: Cerrar el cursor después de recuperar los datos.

Manejo de Entrada del Usuario

Al tratar con la entrada del usuario, es crucial prevenir la inyección SQL. Usa marcadores de parámetros para incluir de manera segura la entrada del usuario en las sentencias SQL.

Dcl-S sqlStmt Varchar(500);
Dcl-S custId Char(10);
Dcl-S custName Char(50);

custId = 'CUST001';

sqlStmt = 'SELECT CUSTNAME FROM CUSTOMERS WHERE CUSTID = ?';

Exec SQL
  Prepare stmt1 From :sqlStmt;

Exec SQL
  Declare c1 Cursor For stmt1;

Exec SQL
  Open c1 Using :custId;

Exec SQL
  Fetch c1 Into :custName;

Exec SQL
  Close c1;

Dsply custName;

Explicación

  1. Marcador de Parámetro: Usa ? como un marcador de posición para la variable en la sentencia SQL.
  2. Abrir Cursor con Parámetros: Usa Using para pasar el valor de la variable de manera segura a la sentencia SQL.

SQL dinámico avanzado

Uso de SQL dinámico con múltiples parámetros

Dcl-S sqlStmt Varchar(500);
Dcl-S custId Char(10);
Dcl-S orderDate Date;
Dcl-S orderAmount Packed(7:2);

custId = 'CUST001';
orderDate = %Date('2023-01-01');

sqlStmt = 'SELECT ORDERAMOUNT FROM ORDERS WHERE CUSTID = ? AND ORDERDATE = ?';

Exec SQL
  Prepare stmt1 From :sqlStmt;

Exec SQL
  Declare c1 Cursor For stmt1;

Exec SQL
  Open c1 Using :custId, :orderDate;

Exec SQL
  Fetch c1 Into :orderAmount;

Exec SQL
  Close c1;

Dsply %Char(orderAmount);

Explicación

  1. Múltiples Parámetros: Usa múltiples marcadores de posición ? para diferentes variables.
  2. Pasar Múltiples Variables: Usa Using para pasar múltiples variables a la sentencia SQL.

Conclusión

El SQL dinámico en RPG proporciona la flexibilidad para construir y ejecutar sentencias SQL en tiempo de ejecución, convirtiéndolo en una herramienta poderosa para manejar requisitos de consulta complejos y variables. Sin embargo, viene con desafíos como una mayor complejidad, posibles problemas de rendimiento y riesgos de seguridad. El manejo adecuado de la entrada del usuario y la comprensión de las compensaciones son esenciales para usar eficazmente el SQL dinámico en tus programas RPG.
