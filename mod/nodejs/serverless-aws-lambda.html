Computación sin Servidor con AWS Lambda
Descripción General del Proyecto y Configuración ►

Introducción a la Computación sin Servidor

  • Definición: La computación sin servidor te permite construir y ejecutar aplicaciones y servicios sin pensar en servidores. AWS Lambda es un servicio de computación sin servidor que te permite ejecutar código sin aprovisionar o gestionar servidores.
  • Beneficios:
    • Sin gestión de servidores
    • Escalado flexible
    • Alta disponibilidad
    • Rentable

Configuración de AWS Lambda

  • Requisitos previos:

    • Cuenta de AWS
    • Conocimiento básico de la Consola de Administración de AWS
    • Node.js instalado en tu máquina local

  • Pasos para Configurar:

    1. Crear una Cuenta de AWS: Si aún no tienes una, regístrate en AWS.
    2. Instalar AWS CLI: Sigue las instrucciones aquí.
    3. Configurar AWS CLI: Ejecuta aws configure e ingresa tu AWS Access Key ID, Secret Access Key, región y formato de salida.

Creando tu Primera Función Lambda

  • Creando una Función Lambda a través de la Consola de AWS:

    1. Ve a la Consola de Administración de AWS.
    2. Navega al servicio Lambda.
    3. Haz clic en "Crear función".
    4. Elige "Autor desde cero".
    5. Ingresa un nombre para la función.
    6. Elige "Node.js" como el tiempo de ejecución.
    7. Haz clic en "Crear función".

  • Código de Ejemplo:

    exports.handler = async (event) => {
    const response = {
        statusCode: 200,
        body: JSON.stringify('¡Hola desde Lambda!'),
    };
    return response;
};

  • Desplegando la Función:

    1. En el editor de funciones, pega el código de ejemplo.
    2. Haz clic en "Desplegar".

Invocando Funciones Lambda

  • Invocación Manual:

    • Usa el botón "Probar" en la Consola de AWS para invocar manualmente la función.

  • Invocación Automatizada:

    • Configura disparadores como API Gateway, eventos de S3, flujos de DynamoDB, etc.

  • Ejemplo con API Gateway:

    1. Crea una API en API Gateway.
    2. Crea un recurso y método.
    3. Integra el método con la función Lambda.

Conceptos Avanzados de Lambda

  • Variables de Entorno:

    • Almacena configuraciones y secretos.
    • Ejemplo: 
      const apiKey = process.env.API_KEY;

  • Capas:

    • Bibliotecas de código reutilizables que pueden ser compartidas entre múltiples funciones Lambda.
    • Ejemplo: Empaquetar funciones utilitarias comunes.

  • Concurrencia y Escalado:

    • Configura concurrencia reservada para controlar el número de instancias.
    • Entiende el comportamiento y los límites de escalado.

  • Manejo de Errores:

    • Usa bloques try-catch en tu código.
    • Configura Colas de Mensajes Muertos (DLQ) para invocaciones fallidas.

Monitoreo y Registro

  • Registros de CloudWatch:

    • Captura automáticamente los registros de las funciones Lambda.
    • Ejemplo de registro en Lambda: 
      console.log('Este es un mensaje de registro');

  • Métricas:

    • Monitorea el conteo de invocaciones, duración, conteo de errores, etc.
    • Configura Alarmas de CloudWatch para métricas críticas.

Mejores Prácticas

  • Optimización de Código:

    • Minimiza el tamaño del paquete.
    • Usa prácticas de codificación eficientes para reducir el tiempo de ejecución.

  • Seguridad:

    • Usa roles de IAM con el menor privilegio.
    • Encripta variables de entorno y datos sensibles.

  • Gestión de Costos:

    • Monitorea el uso y configura alertas de facturación.
    • Optimiza el tiempo de ejecución de las funciones para reducir costos.

Conclusión

La computación sin servidor con AWS Lambda proporciona una forma poderosa y flexible de construir y desplegar aplicaciones sin la sobrecarga de gestionar servidores. Al aprovechar AWS Lambda con Node.js, puedes crear soluciones escalables y rentables que son fáciles de mantener y desplegar. Este curso ha cubierto los conceptos básicos de configuración y uso de AWS Lambda, conceptos avanzados y mejores prácticas para asegurarte de que puedas utilizar eficazmente la computación sin servidor en tus proyectos.
