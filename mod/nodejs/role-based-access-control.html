Control de Acceso Basado en Roles en Node.js
Introducción a WebSockets ►

El Control de Acceso Basado en Roles (RBAC) es un método de regulación del acceso a recursos basado en los roles de los usuarios individuales dentro de una organización. Este tema te guiará a través de la implementación de RBAC en una aplicación Node.js, desde conceptos básicos hasta técnicas avanzadas.

Introducción al Control de Acceso Basado en Roles

  • Definición: RBAC es un mecanismo de control de acceso neutral en cuanto a políticas, definido en torno a roles y privilegios.
  • Propósito: Simplificar la gestión de permisos asignando roles a los usuarios y asociando permisos con estos roles.
  • Beneficios:
    • Simplifica la gestión de permisos de usuario.
    • Mejora la seguridad al garantizar que los usuarios solo tengan los permisos que necesitan.
    • Escala bien con el crecimiento de la aplicación.

Configuración de un Proyecto Node.js

Antes de profundizar en RBAC, configuremos un proyecto básico de Node.js.

  1. Inicializar un nuevo proyecto Node.js:

    mkdir rbac-example
cd rbac-example
npm init -y

  2. Instalar paquetes necesarios:

    npm install express jsonwebtoken bcryptjs

  3. Crear la estructura del proyecto:

    mkdir src
touch src/app.js src/routes.js src/controllers.js src/middleware.js

Conceptos Básicos de RBAC

Roles y Permisos

  • Roles: Un rol es una colección de permisos. Ejemplos incluyen admin, editor y viewer.
  • Permisos: Los permisos son derechos de acceso específicos a recursos, como read, write y delete.

Ejemplo de Mapeo de Roles y Permisos

| Rol | Permisos | |--------|----------------------| | Admin | read, write, delete | | Editor | read, write | | Viewer | read |

Implementación de RBAC en Node.js

Paso 1: Definir Roles y Permisos

Cree un archivo src/roles.js para definir roles y sus permisos asociados.

const roles = {
    admin: ['read', 'write', 'delete'],
    editor: ['read', 'write'],
    viewer: ['read']
};

module.exports = roles;

Paso 2: Middleware para Verificación de Roles

Cree un middleware para verificar si un usuario tiene el rol requerido para acceder a un recurso.

// src/middleware.js
const roles = require('./roles');

function authorize(requiredRole) {
    return (req, res, next) => {
        const userRole = req.user.role; // Suponiendo que req.user es poblado por el middleware de autenticación
        if (roles[userRole] && roles[userRole].includes(requiredRole)) {
            next();
        } else {
            res.status(403).json({ message: 'Forbidden' });
        }
    };
}

module.exports = { authorize };

Paso 3: Aplicar Middleware a Rutas

Use el middleware en sus rutas para proteger los puntos finales basados en roles.

// src/routes.js
const express = require('express');
const { authorize } = require('./middleware');
const router = express.Router();

router.get('/admin', authorize('admin'), (req, res) => {
    res.send('Admin content');
});

router.get('/editor', authorize('editor'), (req, res) => {
    res.send('Editor content');
});

router.get('/viewer', authorize('viewer'), (req, res) => {
    res.send('Viewer content');
});

module.exports = router;

Paso 4: Integrar Rutas en la Aplicación

Integre las rutas en su archivo principal de la aplicación.

// src/app.js
const express = require('express');
const app = express();
const routes = require('./routes');

// Middleware para simular la autenticación de usuario
app.use((req, res, next) => {
    req.user = { role: 'admin' }; // Simular un usuario autenticado con rol 'admin'
    next();
});

app.use('/api', routes);

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});

Técnicas Avanzadas de RBAC

Asignación Dinámica de Roles

Los roles pueden asignarse dinámicamente basándose en datos de usuario de una base de datos.

// src/middleware.js
const roles = require('./roles');
const User = require('./models/User'); // Suponiendo que tiene un modelo de Usuario

async function authorize(requiredRole) {
    return async (req, res, next) => {
        const user = await User.findById(req.user.id); // Obtener usuario de la base de datos
        const userRole = user.role;
        if (roles[userRole] && roles[userRole].includes(requiredRole)) {
            next();
        } else {
            res.status(403).json({ message: 'Forbidden' });
        }
    };
}

module.exports = { authorize };

Jerarquías de Roles

Implemente jerarquías de roles donde los roles superiores heredan permisos de roles inferiores.

const roles = {
    admin: ['read', 'write', 'delete'],
    editor: ['read', 'write'],
    viewer: ['read']
};

function getPermissions(role) {
    const roleHierarchy = {
        admin: ['editor', 'viewer'],
        editor: ['viewer'],
        viewer: []
    };

    let permissions = new Set(roles[role]);
    (roleHierarchy[role] || []).forEach(subRole => {
        roles[subRole].forEach(permission => permissions.add(permission));
    });

    return Array.from(permissions);
}

module.exports = { getPermissions };

Conclusión

El Control de Acceso Basado en Roles es un método poderoso para gestionar los permisos de usuario en una aplicación Node.js. Al definir roles y permisos, crear middleware para la autorización y aplicar estos conceptos a sus rutas, puede construir un sistema de control de acceso seguro y escalable. Las técnicas avanzadas como la asignación dinámica de roles y las jerarquías de roles mejoran aún más la flexibilidad y robustez de su implementación de RBAC.
