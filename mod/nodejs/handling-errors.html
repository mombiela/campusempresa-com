Manejo de Errores
Módulo HTTP ►

El manejo de errores es un aspecto crítico para construir aplicaciones robustas y confiables en Node.js. Este tema te guiará a través de los fundamentos del manejo de errores, desde conceptos básicos hasta técnicas avanzadas.

Introducción al Manejo de Errores

El manejo de errores es el proceso de responder y gestionar los errores que ocurren durante la ejecución de un programa. En Node.js, los errores pueden ocurrir por diversas razones, como entrada de usuario inválida, solicitudes de red fallidas o errores en el código.

Conceptos Clave

  • Objeto Error: En Node.js, los errores se representan mediante el objeto Error, que contiene información sobre el error.
  • Errores Sincrónicos vs Asincrónicos: Los errores pueden ocurrir tanto en código sincrónico como asincrónico. Manejar estos errores requiere diferentes enfoques.
  • Bloque Try-Catch: Una forma común de manejar errores sincrónicos.
  • Callbacks con Error Primero: Un patrón utilizado para manejar errores en código asincrónico.
  • Promises y Async/Await: Formas modernas de manejar errores asincrónicos.

Manejo de Errores Sincrónicos

Los errores sincrónicos ocurren durante la ejecución de una función y pueden ser capturados usando bloques try-catch.

Ejemplo

function divide(a, b) {
    if (b === 0) {
        throw new Error('División por cero');
    }
    return a / b;
}

try {
    const result = divide(4, 0);
    console.log(result);
} catch (error) {
    console.error('Error:', error.message);
}

En este ejemplo, si b es cero, se lanza un error y el bloque catch lo maneja registrando el mensaje de error.

Manejo de Errores Asincrónicos

Los errores asincrónicos ocurren en operaciones como solicitudes de red, E/S de archivos o temporizadores. Manejar estos errores requiere técnicas diferentes.

Callbacks con Error Primero

Node.js utiliza el patrón de callback con error primero para manejar errores asincrónicos.

Ejemplo

const fs = require('fs');

fs.readFile('nonexistentfile.txt', 'utf8', (err, data) => {
    if (err) {
        console.error('Error:', err.message);
        return;
    }
    console.log(data);
});

En este ejemplo, si el archivo no existe, el objeto err contendrá el error y se registrará el mensaje de error.

Promesas

Las promesas proporcionan una forma más limpia de manejar operaciones y errores asincrónicos.

Ejemplo

const fs = require('fs').promises;

fs.readFile('nonexistentfile.txt', 'utf8')
    .then(data => {
        console.log(data);
    })
    .catch(err => {
        console.error('Error:', err.message);
    });

Async/Await

Async/Await es azúcar sintáctico sobre promesas, haciendo que el código asincrónico parezca sincrónico.

Ejemplo

const fs = require('fs').promises;

async function readFile() {
    try {
        const data = await fs.readFile('nonexistentfile.txt', 'utf8');
        console.log(data);
    } catch (err) {
        console.error('Error:', err.message);
    }
}

readFile();

Manejo Avanzado de Errores

Clases de Error Personalizadas

Crear clases de error personalizadas puede ayudar a categorizar y manejar diferentes tipos de errores de manera más efectiva.

Ejemplo

class ValidationError extends Error {
    constructor(message) {
        super(message);
        this.name = 'ValidationError';
    }
}

function validateUser(user) {
    if (!user.name) {
        throw new ValidationError('El nombre es requerido');
    }
    // Otra lógica de validación
}

try {
    validateUser({});
} catch (error) {
    if (error instanceof ValidationError) {
        console.error('Error de Validación:', error.message);
    } else {
        console.error('Error:', error.message);
    }
}

Middleware de Manejo de Errores en Express

En una aplicación Express, puedes usar middleware para manejar errores globalmente.

Ejemplo

const express = require('express');
const app = express();

app.get('/', (req, res) => {
    throw new Error('Algo salió mal');
});

// Middleware de manejo de errores
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).send('¡Algo se rompió!');
});

app.listen(3000, () => {
    console.log('El servidor está corriendo en el puerto 3000');
});

Conclusión

Manejar errores de manera efectiva es crucial para construir aplicaciones confiables en Node.js. Al comprender e implementar las diversas técnicas de manejo de errores discutidas en este tema, puedes asegurarte de que tu aplicación pueda manejar situaciones inesperadas de manera elegante y proporcionar una mejor experiencia de usuario.

  • Errores Sincrónicos: Usa bloques try-catch.
  • Errores Asincrónicos: Usa callbacks con error primero, promesas o async/await.
  • Errores Personalizados: Crea clases de error personalizadas para una mejor categorización de errores.
  • Middleware de Express: Usa middleware para el manejo global de errores en aplicaciones Express.

Dominando estas técnicas, estarás bien equipado para manejar errores en tus aplicaciones Node.js, desde scripts simples hasta servidores web complejos.
