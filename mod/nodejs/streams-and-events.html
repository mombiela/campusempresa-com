Streams y Eventos en Node.js
Callbacks ►

Introducción a Streams

Streams son una característica poderosa en Node.js que te permite manejar datos de manera eficiente. Son instancias de EventEmitter y pueden ser usadas para leer o escribir datos continuamente.

Conceptos Clave

  • Readable Streams: Usados para leer datos.
  • Writable Streams: Usados para escribir datos.
  • Duplex Streams: Pueden ser usados tanto para leer como para escribir.
  • Transform Streams: Un tipo de duplex stream donde la salida se calcula en base a la entrada.

Ejemplo: Leyendo desde un Readable Stream

const fs = require('fs');
const readableStream = fs.createReadStream('example.txt', 'utf8');

readableStream.on('data', (chunk) => {
  console.log(`Received ${chunk.length} bytes of data.`);
  console.log(chunk);
});

readableStream.on('end', () => {
  console.log('No more data.');
});
  • Explicación: Este código lee datos de example.txt en fragmentos y los registra en la consola. El evento data se emite cuando un fragmento de datos está disponible, y el evento end se emite cuando no hay más datos para leer.

Writable Streams

Writable streams se usan para escribir datos a un destino.

Ejemplo: Escribiendo a un Writable Stream

const fs = require('fs');
const writableStream = fs.createWriteStream('output.txt');

writableStream.write('Hello, world!\n');
writableStream.write('Writing data to a file.\n');
writableStream.end('Finished writing.');
  • Explicación: Este código escribe datos a output.txt. El método write se usa para escribir datos, y el método end se usa para señalar el fin de la escritura.

Duplex y Transform Streams

Duplex streams pueden leer y escribir datos, mientras que transform streams pueden modificar o transformar los datos a medida que se leen o escriben.

Ejemplo: Transform Stream

const { Transform } = require('stream');

class UppercaseTransform extends Transform {
  _transform(chunk, encoding, callback) {
    this.push(chunk.toString().toUpperCase());
    callback();
  }
}

const uppercase = new UppercaseTransform();

process.stdin.pipe(uppercase).pipe(process.stdout);
  • Explicación: Este código crea un transform stream que convierte los datos de entrada a mayúsculas. El stream process.stdin se canaliza a través del transform stream y luego a process.stdout.

Introducción a Eventos

Node.js está construido alrededor de una arquitectura impulsada por eventos. La clase EventEmitter está en el núcleo de esta arquitectura.

Conceptos Clave

  • EventEmitter: Una clase que te permite crear, emitir y escuchar eventos.
  • on(event, listener): Añade un oyente para el evento especificado.
  • emit(event, [arg1], [arg2], [...]): Emite el evento especificado.

Ejemplo: EventEmitter Básico

const EventEmitter = require('events');
const myEmitter = new EventEmitter();

myEmitter.on('event', () => {
  console.log('An event occurred!');
});

myEmitter.emit('event');
  • Explicación: Este código crea una instancia de EventEmitter, añade un oyente para el evento event, y luego emite el evento.

Manejo Avanzado de Eventos

Puedes pasar argumentos a los oyentes de eventos y manejar múltiples eventos.

Ejemplo: Pasando Argumentos a Oyentes de Eventos

const EventEmitter = require('events');
const myEmitter = new EventEmitter();

myEmitter.on('greet', (name) => {
  console.log(`Hello, ${name}!`);
});

myEmitter.emit('greet', 'Alice');
  • Explicación: Este código emite el evento greet con un argumento, y el oyente registra un saludo personalizado.

Combinando Streams y Eventos

Streams y eventos a menudo trabajan juntos en aplicaciones Node.js.

Ejemplo: Usando Streams con EventEmitter

const fs = require('fs');
const EventEmitter = require('events');
const myEmitter = new EventEmitter();

const readableStream = fs.createReadStream('example.txt', 'utf8');

readableStream.on('data', (chunk) => {
  myEmitter.emit('dataReceived', chunk);
});

myEmitter.on('dataReceived', (data) => {
  console.log(`Data received: ${data}`);
});
  • Explicación: Este código lee datos de un archivo y emite un evento personalizado dataReceived cada vez que se lee un fragmento de datos. El oyente del evento registra los datos recibidos.

Conclusión

Streams y eventos son conceptos fundamentales en Node.js que permiten un manejo eficiente de datos y programación impulsada por eventos. Entender cómo usar readable, writable, duplex, y transform streams, así como cómo trabajar con la clase EventEmitter, es crucial para construir aplicaciones Node.js escalables y eficientes.

Al dominar estos conceptos, puedes manejar grandes cantidades de datos y crear aplicaciones responsivas que reaccionen a varios eventos sin problemas.
