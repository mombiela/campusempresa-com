Introducción a las Expresiones Regulares

Las expresiones regulares (regex) son secuencias de caracteres que definen un patrón de búsqueda. Se utilizan comúnmente para la coincidencia y manipulación de cadenas. En PowerShell, las regex se pueden usar para buscar, reemplazar y validar cadenas.

Conceptos Clave

  • Coincidencia de Patrones: Identificar si una cadena contiene un patrón especificado.
  • Clases de Caracteres: Conjuntos de caracteres encerrados en corchetes [] que coinciden con cualquier carácter dentro del conjunto.
  • Cuantificadores: Especifican el número de ocurrencias de un carácter o grupo.
  • Anclas: Aseguran la posición dentro de una cadena (por ejemplo, inicio o fin).
  • Grupos y Captura: Encierran parte de la regex en paréntesis () para crear grupos.

Sintaxis Básica y Uso

Coincidencia de Patrones

Para comprobar si una cadena coincide con un patrón, use el operador -match.

$string = "Hola Mundo"
if ($string -match "Mundo") {
    Write-Output "¡Patrón encontrado!"
}

Clases de Caracteres

Las clases de caracteres permiten coincidir con cualquiera de varios caracteres.

$string = "Hola Mundo"
if ($string -match "[aeiou]") {
    Write-Output "¡La cadena contiene una vocal!"
}

Cuantificadores

Los cuantificadores especifican cuántas veces debe coincidir un carácter o grupo.

  • * : 0 o más veces
  • + : 1 o más veces
  • ? : 0 o 1 vez
  • {n} : Exactamente n veces
  • {n,} : n o más veces
  • {n,m} : Entre n y m veces
$string = "Holaaaa Mundo"
if ($string -match "a{2,}") {
    Write-Output "¡La cadena contiene dos o más 'a' consecutivas!"
}

Anclas

Las anclas se utilizan para especificar la posición dentro de la cadena.

  • ^ : Inicio de la cadena
  • $ : Fin de la cadena
$string = "Hola Mundo"
if ($string -match "^Hola") {
    Write-Output "¡La cadena comienza con 'Hola'!"
}
if ($string -match "Mundo$") {
    Write-Output "¡La cadena termina con 'Mundo'!"
}

Grupos y Captura

Los grupos se utilizan para crear sub-patrones dentro de una regex.

$string = "Hola123"
if ($string -match "([A-Za-z]+)(\d+)") {
    Write-Output "Letras: $($matches[1]), Números: $($matches[2])"
}

Técnicas Avanzadas

Lookahead y Lookbehind

Las afirmaciones lookahead y lookbehind permiten coincidir con un patrón solo si está (o no está) seguido o precedido por otro patrón.

  • Lookahead Positivo: (?=...)
  • Lookahead Negativo: (?!...)
  • Lookbehind Positivo: (?<=...)
  • Lookbehind Negativo: (?<!...)
$string = "foo123bar"
if ($string -match "foo(?=\d{3})") {
    Write-Output "¡Encontrado 'foo' seguido de tres dígitos!"
}

Grupos No Capturantes

Los grupos no capturantes se utilizan para agrupar parte de una regex sin capturarla.

$string = "Hola123"
if ($string -match "(?:Hola)(\d+)") {
    Write-Output "Números: $($matches[1])"
}

Grupos de Captura Nombrados

Los grupos de captura nombrados permiten asignar un nombre a un grupo.

$string = "Hola123"
if ($string -match "(?<letters>[A-Za-z]+)(?<numbers>\d+)") {
    Write-Output "Letras: $($matches['letters']), Números: $($matches['numbers'])"
}

Ejemplos Prácticos

Validación de Direcciones de Correo Electrónico

$email = "ejemplo@ejemplo.com"
$regex = "^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$"
if ($email -match $regex) {
    Write-Output "¡Dirección de correo válida!"
} else {
    Write-Output "¡Dirección de correo no válida!"
}

Extracción de Datos de Texto

$text = "ID de Pedido: 12345, Fecha: 2023-10-01"
if ($text -match "ID de Pedido: (\d+), Fecha: (\d{4}-\d{2}-\d{2})") {
    $orderId = $matches[1]
    $date = $matches[2]
    Write-Output "ID de Pedido: $orderId, Fecha: $date"
}

Resumen

Las expresiones regulares son herramientas poderosas para el procesamiento de texto en PowerShell. Al comprender la sintaxis básica y las técnicas avanzadas, puede realizar manipulaciones y validaciones de cadenas complejas de manera eficiente. Practique con diferentes patrones y casos de uso para volverse competente en el uso de regex en sus scripts de PowerShell.
