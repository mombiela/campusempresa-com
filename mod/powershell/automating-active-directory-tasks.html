Introducción

Active Directory (AD) es un servicio crítico para la gestión de usuarios, computadoras y otros recursos en un entorno Windows. PowerShell proporciona una forma poderosa de automatizar muchas de las tareas asociadas con Active Directory, haciendo la administración más eficiente y menos propensa a errores.

En este tema, cubriremos:

  • Conexión a Active Directory
  • Gestión de Usuarios
  • Gestión de Grupos
  • Gestión de Computadoras
  • Técnicas Avanzadas de Automatización

Conexión a Active Directory

Antes de poder gestionar objetos de Active Directory, necesitas establecer una conexión con tu entorno AD.

Instalación del Módulo de Active Directory

Primero, asegúrate de que el módulo de Active Directory para PowerShell esté instalado. Puedes instalarlo usando el siguiente comando:

Install-WindowsFeature -Name RSAT-AD-PowerShell

Importación del Módulo de Active Directory

Una vez instalado, importa el módulo en tu sesión de PowerShell:

Import-Module ActiveDirectory

Conexión a un Controlador de Dominio

Para conectarte a un controlador de dominio específico, usa el cmdlet Get-ADDomainController:

$domainController = Get-ADDomainController -Discover -Service "PrimaryDC"

Gestión de Usuarios

La gestión de usuarios es una tarea común en Active Directory. PowerShell facilita la creación, modificación y eliminación de cuentas de usuario.

Creación de un Nuevo Usuario

Para crear un nuevo usuario, usa el cmdlet New-ADUser:

New-ADUser -Name "John Doe" -GivenName "John" -Surname "Doe" -SamAccountName "jdoe" -UserPrincipalName "jdoe@domain.com" -Path "OU=Users,DC=domain,DC=com" -AccountPassword (ConvertTo-SecureString "P@ssw0rd" -AsPlainText -Force) -Enabled $true

Modificación de un Usuario Existente

Para modificar un usuario existente, usa el cmdlet Set-ADUser:

Set-ADUser -Identity "jdoe" -Title "Senior Developer" -Department "IT"

Eliminación de un Usuario

Para eliminar un usuario, usa el cmdlet Remove-ADUser:

Remove-ADUser -Identity "jdoe"

Gestión de Grupos

Los grupos son esenciales para la gestión de permisos y control de acceso. PowerShell proporciona cmdlets para crear, modificar y eliminar grupos.

Creación de un Nuevo Grupo

Para crear un nuevo grupo, usa el cmdlet New-ADGroup:

New-ADGroup -Name "Developers" -GroupScope Global -Path "OU=Groups,DC=domain,DC=com"

Adición de un Usuario a un Grupo

Para añadir un usuario a un grupo, usa el cmdlet Add-ADGroupMember:

Add-ADGroupMember -Identity "Developers" -Members "jdoe"

Eliminación de un Usuario de un Grupo

Para eliminar un usuario de un grupo, usa el cmdlet Remove-ADGroupMember:

Remove-ADGroupMember -Identity "Developers" -Members "jdoe" -Confirm:$false

Gestión de Computadoras

La gestión de cuentas de computadoras es otra tarea crítica en Active Directory.

Creación de una Nueva Computadora

Para crear una nueva cuenta de computadora, usa el cmdlet New-ADComputer:

New-ADComputer -Name "PC01" -Path "OU=Computers,DC=domain,DC=com" -Enabled $true

Restablecimiento de una Cuenta de Computadora

Para restablecer una cuenta de computadora, usa el cmdlet Reset-ADComputer:

Reset-ADComputer -Identity "PC01"

Eliminación de una Computadora

Para eliminar una cuenta de computadora, usa el cmdlet Remove-ADComputer:

Remove-ADComputer -Identity "PC01"

Técnicas Avanzadas de Automatización

Para una automatización más avanzada, puedes combinar múltiples cmdlets y usar técnicas de scripting para realizar tareas complejas.

Creación Masiva de Usuarios

Puedes crear múltiples usuarios desde un archivo CSV usando un script:

$users = Import-Csv -Path "C:\Users\new_users.csv"
foreach ($user in $users) {
    New-ADUser -Name $user.Name -GivenName $user.GivenName -Surname $user.Surname -SamAccountName $user.SamAccountName -UserPrincipalName $user.UserPrincipalName -Path $user.Path -AccountPassword (ConvertTo-SecureString $user.Password -AsPlainText -Force) -Enabled $true
}

Tareas Programadas

Puedes programar scripts de PowerShell para que se ejecuten en momentos específicos usando el Programador de Tareas.

$action = New-ScheduledTaskAction -Execute "PowerShell.exe" -Argument "-File C:\Scripts\DailyUserReport.ps1"
$trigger = New-ScheduledTaskTrigger -Daily -At 6am
Register-ScheduledTask -Action $action -Trigger $trigger -TaskName "DailyUserReport" -Description "Genera un informe diario de actividades de usuarios"

Conclusión

PowerShell es una herramienta poderosa para automatizar tareas de Active Directory. Al dominar los cmdlets y técnicas cubiertos en este tema, puedes agilizar significativamente tus procesos de gestión de AD. Ya sea que estés creando usuarios, gestionando grupos o automatizando flujos de trabajo complejos, PowerShell proporciona la flexibilidad y el control que necesitas para gestionar eficientemente tu entorno de Active Directory.
