Introducción a la Infraestructura como Código (IaC)

  • Definición: La Infraestructura como Código (IaC) es el proceso de gestionar y aprovisionar infraestructura informática a través de scripts legibles por máquina, en lugar de mediante la configuración de hardware físico o herramientas de configuración interactivas.
  • Beneficios:
    • Consistencia y repetibilidad
    • Control de versiones
    • Automatización y eficiencia
    • Reducción del riesgo de error humano

Empezando con PowerShell

  • ¿Qué es PowerShell?: Un marco de automatización de tareas y gestión de configuración de Microsoft, que consiste en una consola de línea de comandos y un lenguaje de scripting asociado.

  • Instalando PowerShell:

    • En Windows: PowerShell viene preinstalado.
    • En macOS/Linux: Usa gestores de paquetes como Homebrew o apt-get.
    # macOS
brew install --cask powershell

# Linux (Ubuntu)
sudo apt-get install -y powershell

Sintaxis Básica de PowerShell

  • Cmdlets: Comandos de PowerShell, pronunciados "command-lets".

    • Ejemplo: Get-Command, Get-Help, Get-Process
    # Listar todos los cmdlets disponibles
Get-Command

# Obtener ayuda para un cmdlet específico
Get-Help Get-Process

  • Variables: Almacenan datos que pueden ser usados y manipulados.

    $greeting = "¡Hola, Mundo!"
Write-Output $greeting

  • Pipelines: Pasan la salida de un cmdlet como entrada a otro.

    Get-Process | Sort-Object CPU -Descending

Escribiendo Scripts de PowerShell

  • Conceptos Básicos de Scripts: Los scripts de PowerShell son archivos de texto con una extensión .ps1.

  • Ejecutando Scripts:

    • Asegúrate de que la política de ejecución permita la ejecución de scripts: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned
    • Ejecuta un script: .\script.ps1
    # Script de ejemplo: hello.ps1
$name = "Usuario de PowerShell"
Write-Output "¡Hola, $name!"

Infraestructura como Código con PowerShell

  • PowerShell Desired State Configuration (DSC):
    • Definición: Una plataforma de gestión en PowerShell que te permite gestionar tu infraestructura de TI y desarrollo con configuración como código.
    • Componentes:
      • Configuración: Define el estado deseado.
      • Recursos: Los bloques de construcción de DSC, como archivos, servicios y paquetes.
      • Local Configuration Manager (LCM): El motor que aplica la configuración.

Escribiendo una Configuración Básica de DSC

  • Ejemplo: Asegurar que un archivo exista con contenido específico.

    Configuration SampleConfig {
    Node "localhost" {
        File ExampleFile {
            DestinationPath = "C:\Example\example.txt"
            Contents = "¡Hola, DSC!"
            Ensure = "Present"
        }
    }
}

# Generar el archivo MOF
SampleConfig

# Aplicar la configuración
Start-DscConfiguration -Path .\SampleConfig -Wait -Verbose

Configuraciones Avanzadas de DSC

  • Configuraciones Compuestas: Configuraciones reutilizables que pueden ser incluidas en otras configuraciones.

  • Configuraciones Parciales: Permiten que diferentes equipos gestionen diferentes partes de la configuración.

    Configuration WebServerConfig {
    Import-DscResource -ModuleName PSDesiredStateConfiguration

    Node "WebServer" {
        WindowsFeature IIS {
            Name = "Web-Server"
            Ensure = "Present"
        }

        File WebContent {
            DestinationPath = "C:\inetpub\wwwroot\index.html"
            Contents = "<html><body>¡Hola, Mundo!</body></html>"
            Ensure = "Present"
        }
    }
}

# Generar y aplicar la configuración
WebServerConfig
Start-DscConfiguration -Path .\WebServerConfig -Wait -Verbose

Mejores Prácticas para IaC con PowerShell

  • Control de Versiones: Usa Git u otro sistema de control de versiones para rastrear cambios.

  • Modularización: Divide las configuraciones en módulos más pequeños y reutilizables.

  • Pruebas: Usa Pester, un marco de pruebas para PowerShell, para escribir pruebas para tus configuraciones.

    Describe "SampleConfig" {
    It "Debería crear el archivo de ejemplo" {
        $filePath = "C:\Example\example.txt"
        Test-Path $filePath | Should -Be $true
    }
}

Conclusión

La Infraestructura como Código con PowerShell, particularmente usando DSC, permite una gestión robusta, repetible y automatizada de tu infraestructura. Al aprovechar las capacidades de scripting de PowerShell, puedes asegurar que tu infraestructura esté siempre en el estado deseado, reduciendo el riesgo de deriva de configuración y error humano. A medida que avances, considera integrar estas prácticas con pipelines de CI/CD para mejorar aún más tus flujos de trabajo de automatización.
