Sentencias Switch en PowerShell
Manejo de Errores ►

Las sentencias Switch en PowerShell proporcionan una forma de ejecutar diferentes bloques de código basados en el valor de una variable o expresión. Son particularmente útiles para manejar múltiples condiciones de manera más eficiente que usando múltiples sentencias if.

Conceptos Clave

  • Sintaxis de la Sentencia Switch: La estructura básica de una sentencia switch.
  • Coincidencia de Patrones: Uso de comodines y expresiones regulares dentro de las sentencias switch.
  • Múltiples Condiciones: Manejo de múltiples casos y casos predeterminados.
  • Características Avanzadas: Uso de las opciones -file, -regex y -wildcard.

Sintaxis de la Sentencia Switch

La sintaxis básica de una sentencia switch en PowerShell es la siguiente:

switch ($variable) {
    case 'value1' {
        # Código a ejecutar si $variable es 'value1'
    }
    case 'value2' {
        # Código a ejecutar si $variable es 'value2'
    }
    default {
        # Código a ejecutar si $variable no coincide con ningún caso
    }
}

Ejemplo

$day = 'Monday'

switch ($day) {
    'Monday' { "Inicio de la semana laboral." }
    'Friday' { "Fin de la semana laboral." }
    default { "Día entre semana." }
}

En este ejemplo:

  • Si $day es 'Monday', se mostrará "Inicio de la semana laboral."
  • Si $day es 'Friday', se mostrará "Fin de la semana laboral."
  • Para cualquier otro valor de $day, se mostrará "Día entre semana."

Coincidencia de Patrones

Las sentencias switch de PowerShell soportan la coincidencia de patrones usando comodines y expresiones regulares.

Comodines

$fruit = 'apple'

switch -Wildcard ($fruit) {
    'a*' { "Empieza con 'a'" }
    '*e' { "Termina con 'e'" }
    default { "Sin coincidencia" }
}

Expresiones Regulares

$fruit = 'banana'

switch -Regex ($fruit) {
    '^b.*a$' { "Empieza con 'b' y termina con 'a'" }
    'n{2}' { "Contiene doble 'n'" }
    default { "Sin coincidencia" }
}

Múltiples Condiciones

Puedes manejar múltiples condiciones especificando múltiples valores en un solo caso.

$number = 3

switch ($number) {
    1 { "Uno" }
    2 { "Dos" }
    3 { "Tres" }
    4 { "Cuatro" }
    default { "Número no en el rango 1-4" }
}

Características Avanzadas

Uso de `-file`

La opción -file te permite leer la entrada desde un archivo y procesar cada línea.

switch -File "C:\path\to\file.txt" {
    'Error' { "Se encontró un error" }
    'Warning' { "Se encontró una advertencia" }
    default { "Información general" }
}

Uso de `-regex` y `-wildcard`

Puedes combinar las opciones -regex y -wildcard para manejar escenarios más complejos.

$input = 'error: file not found'

switch -Regex ($input) {
    'error:.*' { "Ocurrió un error" }
    'warning:.*' { "Ocurrió una advertencia" }
    default { "No se encontraron problemas" }
}

Conclusión

Las sentencias switch en PowerShell son una herramienta poderosa para manejar múltiples condiciones y coincidencias de patrones. Pueden simplificar tus scripts al reducir la necesidad de múltiples sentencias if y hacer tu código más legible. Al entender la sintaxis básica, las capacidades de coincidencia de patrones y las características avanzadas, puedes usar efectivamente las sentencias switch para controlar el flujo de tus scripts de PowerShell.
