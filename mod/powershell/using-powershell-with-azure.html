Introducción a Azure PowerShell

Azure PowerShell es un conjunto de cmdlets diseñados para gestionar recursos de Azure directamente desde la línea de comandos de PowerShell. Esta sección cubrirá los conceptos básicos de Azure PowerShell y cómo empezar.

Conceptos Clave

  • Cmdlets: Comandos especializados en el entorno de PowerShell.
  • Azure Resource Manager (ARM): El servicio de despliegue y gestión para Azure.
  • Módulos: Paquetes que contienen cmdlets, proveedores, funciones, flujos de trabajo, variables y alias.

Instalando Azure PowerShell

Para usar Azure PowerShell, necesitas instalar el módulo Az.

# Instalar el módulo Az
Install-Module -Name Az -AllowClobber -Scope CurrentUser

Autenticación con Azure

Antes de gestionar recursos de Azure, necesitas autenticarte.

# Iniciar sesión en tu cuenta de Azure
Connect-AzAccount

Gestión de Recursos de Azure

Esta sección cubrirá cómo gestionar varios recursos de Azure usando PowerShell.

Creación de un Grupo de Recursos

Los grupos de recursos son contenedores que contienen recursos relacionados para una solución de Azure.

# Crear un nuevo grupo de recursos
New-AzResourceGroup -Name "MyResourceGroup" -Location "EastUS"

Creación de una Máquina Virtual

Las máquinas virtuales (VMs) son uno de los recursos más comunes en Azure.

# Crear una nueva máquina virtual
New-AzVM -ResourceGroupName "MyResourceGroup" -Name "MyVM" -Location "EastUS" -VirtualNetworkName "MyVNet" -SubnetName "MySubnet" -SecurityGroupName "MyNSG" -PublicIpAddressName "MyPublicIP" -OpenPorts 80,3389

Gestión de Cuentas de Almacenamiento

Las cuentas de almacenamiento proporcionan un espacio de nombres único para almacenar y acceder a objetos de datos de Azure Storage.

# Crear una nueva cuenta de almacenamiento
New-AzStorageAccount -ResourceGroupName "MyResourceGroup" -Name "mystorageaccount" -Location "EastUS" -SkuName "Standard_LRS"

Azure PowerShell Avanzado

Esta sección cubrirá temas más avanzados y mejores prácticas para usar Azure PowerShell.

Automatización de Tareas con Scripts

Automatiza tareas repetitivas escribiendo scripts de PowerShell.

# Ejemplo de script para automatizar la creación de VM
$resourceGroupName = "MyResourceGroup"
$location = "EastUS"
$vmName = "MyVM"

New-AzResourceGroup -Name $resourceGroupName -Location $location
New-AzVM -ResourceGroupName $resourceGroupName -Name $vmName -Location $location -VirtualNetworkName "MyVNet" -SubnetName "MySubnet" -SecurityGroupName "MyNSG" -PublicIpAddressName "MyPublicIP" -OpenPorts 80,3389

Uso de Azure PowerShell con Pipelines CI/CD

Integra scripts de Azure PowerShell en pipelines CI/CD para despliegues automatizados.

# Ejemplo de script de pipeline de Azure DevOps
trigger:
- main

pool:
  vmImage: 'windows-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- powershell: |
    Install-Module -Name Az -AllowClobber -Scope CurrentUser
    Connect-AzAccount -ServicePrincipal -Tenant $env:tenantId -Credential (New-Object System.Management.Automation.PSCredential($env:servicePrincipalId, (ConvertTo-SecureString $env:servicePrincipalKey -AsPlainText -Force)))
    New-AzResourceGroup -Name "MyResourceGroup" -Location "EastUS"
  env:
    tenantId: $(tenantId)
    servicePrincipalId: $(servicePrincipalId)
    servicePrincipalKey: $(servicePrincipalKey)

Resumen

En este tema, cubrimos los conceptos básicos de usar PowerShell con Azure, incluyendo:

  • Instalación y autenticación con Azure PowerShell.
  • Gestión de recursos de Azure como grupos de recursos, máquinas virtuales y cuentas de almacenamiento.
  • Automatización de tareas con scripts de PowerShell.
  • Integración de Azure PowerShell en pipelines CI/CD.

Al dominar estas habilidades, puedes gestionar y automatizar eficientemente tus recursos de Azure usando PowerShell.
