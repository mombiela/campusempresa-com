Introducción

PowerShell es un marco de automatización de tareas y gestión de configuración de Microsoft, que consiste en una consola de línea de comandos y el lenguaje de scripting asociado. Esta sección te guiará a través del proceso de instalación y configuración de PowerShell en tu sistema, asegurándote de tener el entorno adecuado para comenzar a escribir scripts y automatizar tareas.

Instalación de PowerShell

Windows

PowerShell viene preinstalado en Windows 10 y versiones posteriores. Sin embargo, es posible que desees instalar la última versión de PowerShell (PowerShell Core) para obtener características adicionales y soporte multiplataforma.

  1. Descargar PowerShell Core:

    • Visita la página de lanzamientos de PowerShell en GitHub.
    • Descarga el último paquete .msi para Windows.

  2. Instalar PowerShell Core:

    • Ejecuta el archivo .msi descargado.
    • Sigue los pasos del asistente de instalación.
    • Asegúrate de que la opción para agregar PowerShell al PATH del sistema esté marcada.

  3. Verificar la Instalación:

    • Abre un símbolo del sistema o el PowerShell de Windows existente.
    • Escribe pwsh y presiona Enter.
    • Deberías ver el prompt de PowerShell Core, lo que indica una instalación exitosa.

macOS

PowerShell Core está disponible para macOS y se puede instalar usando Homebrew.

  1. Instalar Homebrew (si no está ya instalado):

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

  2. Instalar PowerShell Core:

    brew install --cask powershell

  3. Verificar la Instalación:

    • Abre Terminal.
    • Escribe pwsh y presiona Enter.
    • Deberías ver el prompt de PowerShell Core.

Linux

PowerShell Core se puede instalar en varias distribuciones de Linux. A continuación se muestran los pasos para Ubuntu:

  1. Descargar e Instalar las claves GPG del repositorio de Microsoft:

    wget -q https://packages.microsoft.com/config/ubuntu/20.04/packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb

  2. Actualizar la lista de productos:

    sudo apt-get update

  3. Instalar PowerShell:

    sudo apt-get install -y powershell

  4. Verificar la Instalación:

    • Abre Terminal.
    • Escribe pwsh y presiona Enter.
    • Deberías ver el prompt de PowerShell Core.

Configuración de PowerShell

Configuración de la Política de Ejecución

La política de ejecución determina las condiciones bajo las cuales PowerShell carga archivos de configuración y ejecuta scripts. Por defecto, la política de ejecución está configurada en Restricted, lo que impide que se ejecuten scripts.

  1. Verificar la Política de Ejecución Actual:

    Get-ExecutionPolicy

  2. Configurar la Política de Ejecución:

    Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
    • RemoteSigned permite ejecutar scripts creados localmente pero requiere que los scripts descargados de internet estén firmados por un editor de confianza.
    • El alcance CurrentUser aplica la política solo al usuario actual.

Instalación de Módulos de PowerShell

Los módulos son paquetes de cmdlets, proveedores, funciones, flujos de trabajo, variables y alias. La PowerShell Gallery es el repositorio central para contenido de PowerShell.

  1. Instalar un Módulo:

    Install-Module -Name Az -Scope CurrentUser -Repository PSGallery -Force
    • Az es un módulo para gestionar recursos de Azure.
    • -Scope CurrentUser instala el módulo para el usuario actual.
    • -Repository PSGallery especifica el repositorio fuente.

  2. Verificar la Instalación del Módulo:

    Get-Module -ListAvailable -Name Az

Personalización del Perfil de PowerShell

El perfil de PowerShell es un script que se ejecuta cada vez que PowerShell se inicia. Puedes usarlo para personalizar tu entorno.

  1. Ubicar la Ruta del Perfil:

    $PROFILE

  2. Crear o Editar el Perfil:

    if (!(Test-Path -Path $PROFILE)) {
    New-Item -ItemType File -Path $PROFILE -Force
}
notepad $PROFILE

  3. Agregar Personalizaciones:

    • Ejemplo: Agregar un alias para un comando usado frecuentemente.
    Set-Alias ll Get-ChildItem

Conclusión

Siguiendo los pasos descritos en esta sección, deberías tener PowerShell instalado y configurado en tu sistema. También has aprendido cómo configurar la política de ejecución, instalar módulos y personalizar tu perfil de PowerShell. Con esta configuración, estás listo para comenzar a explorar y utilizar PowerShell para tareas de automatización y scripting.
