Introducción a las Pruebas
Pruebas Unitarias con Jest ►

Las pruebas son una parte crucial del desarrollo de software, asegurando que tu aplicación funcione como se espera y esté libre de errores. En esta sección, te introduciremos a las pruebas en el contexto de las aplicaciones React, cubriendo los conceptos básicos y avanzando gradualmente hacia conceptos más avanzados.

Por Qué las Pruebas son Importantes

  • Confiabilidad: Asegura que tu aplicación se comporte como se espera.
  • Mantenibilidad: Facilita la refactorización del código sin romper la funcionalidad.
  • Documentación: Sirve como documentación de cómo se supone que debe funcionar tu código.
  • Confianza: Proporciona confianza a los desarrolladores y partes interesadas de que la aplicación es estable.

Tipos de Pruebas

  • Pruebas Unitarias: Prueban componentes o funciones individuales en aislamiento.
  • Pruebas de Integración: Prueban cómo diferentes partes de la aplicación funcionan juntas.
  • Pruebas de Extremo a Extremo (E2E): Prueban toda la aplicación desde la perspectiva del usuario.

Herramientas de Pruebas para React

  • Jest: Un marco de pruebas de JavaScript mantenido por Facebook, comúnmente utilizado con React.
  • React Testing Library: Una biblioteca para probar componentes React, enfocándose en probar interacciones de usuario.
  • Cypress: Un marco de pruebas de extremo a extremo.

Configuración del Entorno de Pruebas

Para comenzar con las pruebas en una aplicación React, necesitas configurar tu entorno de pruebas. La mayoría de los proyectos React usan Jest y React Testing Library.

Instalando Jest y React Testing Library

Primero, asegúrate de tener un proyecto React configurado. Luego, instala los paquetes necesarios:

npm install --save-dev jest @testing-library/react @testing-library/jest-dom

Configurando Jest

Jest usualmente requiere una configuración mínima. Crea un archivo jest.config.js en la raíz de tu proyecto:

module.exports = {
  setupFilesAfterEnv: ['<rootDir>/setupTests.js'],
  testEnvironment: 'jsdom',
};

Crea un archivo setupTests.js para configurar React Testing Library:

import '@testing-library/jest-dom/extend-expect';

Escribiendo Tu Primera Prueba

Vamos a escribir una prueba simple para un componente React.

Creando un Componente Simple

Crea un componente HelloWorld.js:

import React from 'react';

const HelloWorld = () => {
  return <h1>Hello, World!</h1>;
};

export default HelloWorld;

Escribiendo una Prueba para el Componente

Crea un archivo HelloWorld.test.js:

import React from 'react';
import { render, screen } from '@testing-library/react';
import HelloWorld from './HelloWorld';

test('renders Hello, World! text', () => {
  render(<HelloWorld />);
  const helloWorldElement = screen.getByText(/Hello, World!/i);
  expect(helloWorldElement).toBeInTheDocument();
});

Explicación

  • render: Renderiza el componente en un DOM virtual para pruebas.
  • screen: Proporciona utilidades para consultar el DOM renderizado.
  • getByText: Encuentra un elemento por su contenido de texto.
  • expect: Realiza afirmaciones sobre los resultados de la prueba.
  • toBeInTheDocument: Verifica si el elemento está presente en el DOM.

Conceptos Avanzados de Pruebas

Simulación de Funciones y Módulos

A veces, necesitas simular funciones o módulos para aislar el componente que se está probando.

Simulación de una Función

const mockFunction = jest.fn();

mockFunction();
expect(mockFunction).toHaveBeenCalled();

Simulación de un Módulo

jest.mock('axios');

import axios from 'axios';
import { fetchData } from './api';

test('fetches data from API', async () => {
  axios.get.mockResolvedValue({ data: 'mockData' });
  const data = await fetchData();
  expect(data).toBe('mockData');
});

Pruebas de Instantáneas

Las pruebas de instantáneas capturan la salida renderizada de un componente y la comparan con una instantánea guardada.

Creando una Prueba de Instantánea

import React from 'react';
import renderer from 'react-test-renderer';
import HelloWorld from './HelloWorld';

test('matches snapshot', () => {
  const tree = renderer.create(<HelloWorld />).toJSON();
  expect(tree).toMatchSnapshot();
});

Conclusión

Las pruebas son una práctica esencial en el desarrollo de React, asegurando que tu aplicación sea confiable y mantenible. Al comprender los conceptos básicos de las pruebas unitarias, de integración y de extremo a extremo, y al usar herramientas como Jest y React Testing Library, puedes escribir pruebas efectivas para tus componentes React. A medida que avances, aprenderás a simular funciones y módulos, y a usar pruebas de instantáneas para mejorar aún más tu cobertura de pruebas. ¡Felices pruebas!
