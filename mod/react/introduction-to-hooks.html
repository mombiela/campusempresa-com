¿Qué son los Hooks?

Los Hooks son una característica introducida en React 16.8 que te permiten usar estado y otras características de React sin escribir una clase. Te permiten gestionar el estado y los efectos secundarios en componentes funcionales, haciendo tu código más conciso y fácil de entender.

Conceptos Clave

  • State Hook (useState): Te permite agregar estado a los componentes funcionales.
  • Effect Hook (useEffect): Te permite realizar efectos secundarios en componentes funcionales.
  • Custom Hooks: Te permiten extraer la lógica del componente en funciones reutilizables.

¿Por qué usar Hooks?

Los Hooks simplifican la forma en que escribes componentes al:

  • Eliminar la necesidad de componentes de clase.
  • Facilitar el compartir lógica entre componentes.
  • Mejorar la legibilidad y mantenibilidad del código.

State Hook (`useState`)

El hook useState te permite agregar estado a tus componentes funcionales. Aquí hay un ejemplo básico:

import React, { useState } from 'react';

function Counter() {
  // Declarar una variable de estado llamada "count" con un valor inicial de 0
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Has hecho clic {count} veces</p>
      <button onClick={() => setCount(count + 1)}>
        Haz clic en mí
      </button>
    </div>
  );
}

export default Counter;

Explicación

  • useState devuelve un array con dos elementos: el valor actual del estado y una función para actualizarlo.
  • Usamos desestructuración de arrays para asignar estos valores a count y setCount.
  • La función setCount se llama cuando se hace clic en el botón, actualizando el estado.

Effect Hook (`useEffect`)

El hook useEffect te permite realizar efectos secundarios en tus componentes, como la obtención de datos, suscripciones o cambios manuales en el DOM.

import React, { useState, useEffect } from 'react';

function DataFetcher() {
  const [data, setData] = useState(null);

  useEffect(() => {
    // Obtener datos de una API
    fetch('https://api.example.com/data')
      .then(response => response.json())
      .then(data => setData(data));
  }, []); // El array vacío significa que este efecto se ejecuta solo una vez, similar a componentDidMount

  return (
    <div>
      {data ? <p>Datos: {data}</p> : <p>Cargando...</p>}
    </div>
  );
}

export default DataFetcher;

Explicación

  • useEffect toma una función que contiene el código del efecto secundario.
  • El segundo argumento es un array de dependencias. Si está vacío, el efecto se ejecuta solo una vez después del renderizado inicial.

Custom Hooks

Los hooks personalizados te permiten extraer y reutilizar lógica en múltiples componentes. Son simplemente funciones de JavaScript cuyos nombres comienzan con "use" y que pueden llamar a otros hooks.

import { useState, useEffect } from 'react';

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(url)
      .then(response => response.json())
      .then(data => {
        setData(data);
        setLoading(false);
      });
  }, [url]);

  return { data, loading };
}

export default useFetch;

Ejemplo de Uso

import React from 'react';
import useFetch from './useFetch';

function App() {
  const { data, loading } = useFetch('https://api.example.com/data');

  return (
    <div>
      {loading ? <p>Cargando...</p> : <p>Datos: {data}</p>}
    </div>
  );
}

export default App;

Explicación

  • useFetch es un hook personalizado que obtiene datos de una URL dada.
  • Devuelve un objeto que contiene los datos obtenidos y un estado de carga.
  • El componente App usa este hook personalizado para obtener y mostrar datos.

Conclusión

Los Hooks son una adición poderosa a React que simplifican la gestión del estado y los efectos secundarios en componentes funcionales. Al entender y utilizar hooks como useState, useEffect y hooks personalizados, puedes escribir código React más limpio y mantenible. A medida que continúes explorando los hooks, descubrirás que ofrecen una forma flexible y eficiente de manejar la lógica del componente.
