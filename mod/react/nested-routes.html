Introducción a las Rutas Anidadas

Las rutas anidadas te permiten definir rutas dentro de otras rutas, habilitando estructuras de navegación más complejas y jerárquicas en tus aplicaciones React. Esto es particularmente útil para crear menús de navegación de varios niveles o cuando necesitas renderizar diferentes componentes basados en la ruta URL.

Conceptos Clave

  • Jerarquía de Rutas: Organizar rutas en una relación padre-hijo.
  • Componente Outlet: Un componente marcador de posición utilizado para renderizar rutas hijas.
  • Configuración de Rutas: Definir rutas anidadas en tu configuración de rutas.

Configuración de Rutas Anidadas

Configuración Básica

Primero, asegúrate de tener react-router-dom instalado en tu proyecto:

npm install react-router-dom

Estructura de Ejemplo

Considera una aplicación simple con la siguiente estructura:

  • Inicio
  • Acerca de
  • Tablero
    • Perfil
    • Configuración

Definiendo Rutas

Así es como puedes definir estas rutas usando react-router-dom:

import React from 'react';
import { BrowserRouter as Router, Routes, Route, Link, Outlet } from 'react-router-dom';

function App() {
  return (
    <Router>
      <nav>
        <Link to="/">Inicio</Link>
        <Link to="/about">Acerca de</Link>
        <Link to="/dashboard">Tablero</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/dashboard" element={<Dashboard />}>
          <Route path="profile" element={<Profile />} />
          <Route path="settings" element={<Settings />} />
        </Route>
      </Routes>
    </Router>
  );
}

function Home() {
  return <h2>Inicio</h2>;
}

function About() {
  return <h2>Acerca de</h2>;
}

function Dashboard() {
  return (
    <div>
      <h2>Tablero</h2>
      <nav>
        <Link to="profile">Perfil</Link>
        <Link to="settings">Configuración</Link>
      </nav>
      <Outlet />
    </div>
  );
}

function Profile() {
  return <h2>Perfil</h2>;
}

function Settings() {
  return <h2>Configuración</h2>;
}

export default App;

Explicación

  • Router: Envuelve toda la aplicación para habilitar el enrutamiento.
  • Routes: Define la configuración de rutas.
  • Route: Especifica rutas individuales. El atributo path define la ruta URL, y el atributo element especifica el componente a renderizar.
  • Outlet: Utilizado en el componente Dashboard para renderizar rutas anidadas (Profile y Settings).

Conceptos Avanzados

Rutas Anidadas Dinámicas

También puedes crear rutas anidadas dinámicas donde la ruta incluye parámetros. Por ejemplo:

<Route path="/dashboard/:userId" element={<Dashboard />}>
  <Route path="profile" element={<Profile />} />
  <Route path="settings" element={<Settings />} />
</Route>

Usando el Hook `useParams`

Para acceder a los parámetros de la ruta, usa el hook useParams de react-router-dom:

import { useParams } from 'react-router-dom';

function Dashboard() {
  let { userId } = useParams();
  return (
    <div>
      <h2>Tablero para el Usuario {userId}</h2>
      <nav>
        <Link to="profile">Perfil</Link>
        <Link to="settings">Configuración</Link>
      </nav>
      <Outlet />
    </div>
  );
}

Protegiendo Rutas Anidadas

Puedes proteger rutas anidadas usando un componente de orden superior o un componente envolvente que verifique la autenticación:

function ProtectedRoute({ children }) {
  const isAuthenticated = // lógica para verificar si el usuario está autenticado
  return isAuthenticated ? children : <Navigate to="/login" />;
}

// Uso
<Route path="/dashboard" element={<ProtectedRoute><Dashboard /></ProtectedRoute>}>
  <Route path="profile" element={<Profile />} />
  <Route path="settings" element={<Settings />} />
</Route>

Conclusión

Las rutas anidadas en React te permiten crear estructuras de navegación complejas y jerárquicas, haciendo tu aplicación más organizada y fácil de navegar. Al entender e implementar rutas anidadas, puedes mejorar la experiencia del usuario y mantener una arquitectura de enrutamiento limpia. Recuerda usar el componente Outlet para renderizar rutas hijas y considera técnicas avanzadas como rutas dinámicas y protección de rutas para aplicaciones más robustas.
