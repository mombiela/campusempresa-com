Introducción

Los componentes controlados son un concepto fundamental en React que se refiere a componentes donde React controla el estado de los elementos del formulario. Esto significa que los datos del formulario son manejados por el componente de React, lo que facilita su gestión y manipulación.

Conceptos Clave

  • Gestión de Estado: En los componentes controlados, el estado de los elementos del formulario es gestionado por el componente de React.
  • Manejo de Eventos: Los componentes controlados dependen de los manejadores de eventos para actualizar el estado.
  • Única Fuente de Verdad: El estado dentro del componente de React sirve como la única fuente de verdad para los datos del formulario.

Ejemplo de un Componente Controlado

Comencemos con un ejemplo simple de un campo de entrada controlado:

import React, { useState } from 'react';

function ControlledInput() {
  const [inputValue, setInputValue] = useState('');

  const handleChange = (event) => {
    setInputValue(event.target.value);
  };

  return (
    <div>
      <input type="text" value={inputValue} onChange={handleChange} />
      <p>Current Input: {inputValue}</p>
    </div>
  );
}

export default ControlledInput;

Explicación

  • Hook useState: Usamos el hook useState para crear una variable de estado inputValue y una función setInputValue para actualizarla.
  • Función handleChange: Esta función actualiza el estado cada vez que el valor del input cambia.
  • Elemento Input: El atributo value del elemento input se establece en inputValue, convirtiéndolo en un componente controlado. El manejador de eventos onChange llama a handleChange para actualizar el estado.

Componentes Controlados vs No Controlados

| Característica | Componentes Controlados | Componentes No Controlados | |--------------------------|------------------------------------------------|------------------------------------------------| | Gestión de Estado | Gestionado por el estado del componente de React | Gestionado por el DOM | | Manejo de Eventos | Requiere manejadores de eventos para actualizar el estado | Usa refs para acceder directamente a los elementos del DOM | | Única Fuente de Verdad | Estado dentro del componente | El estado es gestionado por el DOM | | Facilidad de Pruebas | Más fácil de probar debido al estado predecible | Más difícil de probar debido a la dependencia del DOM |

Ejemplo Avanzado: Formulario Controlado

Vamos a crear un ejemplo más complejo con un formulario controlado que contiene múltiples campos de entrada:

import React, { useState } from 'react';

function ControlledForm() {
  const [formData, setFormData] = useState({
    name: '',
    email: '',
    message: ''
  });

  const handleChange = (event) => {
    const { name, value } = event.target;
    setFormData({
      ...formData,
      [name]: value
    });
  };

  const handleSubmit = (event) => {
    event.preventDefault();
    console.log('Form Data Submitted:', formData);
  };

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label>Name:</label>
        <input type="text" name="name" value={formData.name} onChange={handleChange} />
      </div>
      <div>
        <label>Email:</label>
        <input type="email" name="email" value={formData.email} onChange={handleChange} />
      </div>
      <div>
        <label>Message:</label>
        <textarea name="message" value={formData.message} onChange={handleChange}></textarea>
      </div>
      <button type="submit">Submit</button>
    </form>
  );
}

export default ControlledForm;

Explicación

  • Inicialización del Estado: El estado formData se inicializa con un objeto que contiene los campos name, email y message.
  • Función handleChange: Esta función actualiza el campo correspondiente en el estado formData basado en el atributo name del input.
  • Función handleSubmit: Esta función previene el comportamiento predeterminado de envío del formulario y registra los datos del formulario en la consola.
  • Elementos del Formulario: El atributo value de cada elemento del formulario está vinculado al campo correspondiente en el estado formData, y el manejador de eventos onChange actualiza el estado.

Conclusión

Los componentes controlados son esenciales para gestionar los datos de los formularios en aplicaciones React. Al controlar el estado dentro del componente de React, se obtiene más control sobre el comportamiento del formulario y se pueden manipular y validar fácilmente los datos. Entender los componentes controlados es crucial para construir aplicaciones React robustas y mantenibles.
