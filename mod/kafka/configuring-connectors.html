Apache Kafka Connect es una herramienta para transmitir datos de manera escalable y confiable entre Apache Kafka y otros sistemas. Configurar conectores es una parte crucial de la configuración de Kafka Connect. Este tema te guiará a través del proceso desde niveles principiantes hasta avanzados.

Introducción a los Conectores de Kafka

Los conectores de Kafka son plugins que te ayudan a mover grandes colecciones de datos dentro y fuera de Kafka. Vienen en dos tipos:

  • Conectores de Origen: Importan datos de sistemas externos a temas de Kafka.
  • Conectores de Destino: Exportan datos de temas de Kafka a sistemas externos.

Configuración Básica

Configuración de Kafka Connect

Antes de configurar conectores, necesitas configurar Kafka Connect. Aquí tienes una configuración básica para el modo standalone:

# connect-standalone.properties
bootstrap.servers=localhost:9092
key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter
offset.storage.file.filename=/tmp/connect.offsets

Configuración de un Conector de Origen

Una configuración de conector de origen típicamente incluye las siguientes propiedades:

  • name: Nombre único para el conector.
  • connector.class: La clase Java para el conector.
  • tasks.max: Número máximo de tareas a usar para este conector.
  • topics: Lista de temas de Kafka a los que escribir datos.

Ejemplo de configuración para un FileStreamSourceConnector:

# file-source.properties
name=file-source-connector
connector.class=org.apache.kafka.connect.file.FileStreamSourceConnector
tasks.max=1
file=/path/to/input/file.txt
topic=example-topic

Configuración de un Conector de Destino

Una configuración de conector de destino incluye propiedades similares pero se enfoca en leer de temas de Kafka y escribir a sistemas externos.

Ejemplo de configuración para un FileStreamSinkConnector:

# file-sink.properties
name=file-sink-connector
connector.class=org.apache.kafka.connect.file.FileStreamSinkConnector
tasks.max=1
file=/path/to/output/file.txt
topics=example-topic

Configuración Avanzada

Modo Distribuido

Para entornos de producción, deberías usar Kafka Connect en modo distribuido. Este modo proporciona escalabilidad y tolerancia a fallos.

Ejemplo de configuración para el modo distribuido:

# connect-distributed.properties
bootstrap.servers=localhost:9092
group.id=connect-cluster
key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter
config.storage.topic=connect-configs
offset.storage.topic=connect-offsets
status.storage.topic=connect-status

Convertidores Personalizados

Puedes usar convertidores personalizados para manejar diferentes formatos de datos. Por ejemplo, usando AvroConverter:

key.converter=io.confluent.connect.avro.AvroConverter
key.converter.schema.registry.url=http://localhost:8081
value.converter=io.confluent.connect.avro.AvroConverter
value.converter.schema.registry.url=http://localhost:8081

Transformaciones

Las Transformaciones de Mensaje Único (SMTs) te permiten modificar mensajes a medida que fluyen a través de Kafka Connect. Aquí tienes un ejemplo de cómo agregar una marca de tiempo a cada mensaje:

transforms=InsertTimestamp
transforms.InsertTimestamp.type=org.apache.kafka.connect.transforms.InsertField$Value
transforms.InsertTimestamp.timestamp.field=timestamp

Monitoreo y Gestión

API REST

Kafka Connect proporciona una API REST para gestionar conectores. Puedes usarla para crear, configurar y monitorear conectores.

Ejemplo para crear un conector usando curl:

curl -X POST -H "Content-Type: application/json" --data '{
  "name": "file-source-connector",
  "config": {
    "connector.class": "org.apache.kafka.connect.file.FileStreamSourceConnector",
    "tasks.max": "1",
    "file": "/path/to/input/file.txt",
    "topic": "example-topic"
  }
}' http://localhost:8083/connectors

Métricas

Kafka Connect expone métricas a través de JMX. Puedes monitorear estas métricas para asegurarte de que tus conectores estén funcionando sin problemas.

Conclusión

Configurar conectores en Kafka Connect implica configurar el entorno, definir propiedades del conector y, opcionalmente, usar características avanzadas como convertidores personalizados y transformaciones. Ya sea que estés ejecutando en modo standalone o distribuido, entender estas configuraciones te ayudará a transmitir datos de manera efectiva entre Kafka y otros sistemas. Monitorear y gestionar tus conectores a través de la API REST y las métricas también es crucial para mantener una canalización de datos saludable.
