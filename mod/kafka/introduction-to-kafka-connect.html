Visión General

Kafka Connect es un marco para conectar Kafka con sistemas externos como bases de datos, almacenes de clave-valor, índices de búsqueda y sistemas de archivos. Simplifica el proceso de integración de Kafka con otros sistemas, permitiendo tuberías de datos escalables y confiables.

Conceptos Clave

  • Conector: Un conector es un componente reutilizable que captura datos de un sistema fuente o envía datos a un sistema destino.
    • Conector de Fuente: Captura datos de un sistema externo y los escribe en temas de Kafka.
    • Conector de Sumidero: Lee datos de temas de Kafka y los escribe en un sistema externo.
  • Tarea: Una tarea es una unidad de trabajo que realiza un conector. Los conectores pueden configurarse para ejecutar múltiples tareas en paralelo para escalabilidad.
  • Trabajador: Un trabajador es un proceso JVM que ejecuta conectores y tareas. Los trabajadores pueden ejecutarse en modo autónomo o distribuido.
  • Modo Autónomo: Adecuado para configuraciones simples de un solo nodo.
  • Modo Distribuido: Adecuado para configuraciones a gran escala y tolerantes a fallos.

Configuración de Kafka Connect

Prerrequisitos

  • Apache Kafka instalado y en funcionamiento.
  • Kit de Desarrollo de Java (JDK) instalado.

Ejemplo de Modo Autónomo

  1. Crear un archivo de propiedades para el conector:

    name=local-file-source
connector.class=FileStreamSource
tasks.max=1
file=/path/to/input/file.txt
topic=connect-test

  2. Ejecutar Kafka Connect en modo autónomo:

    bin/connect-standalone.sh config/connect-standalone.properties config/connect-file-source.properties

Ejemplo de Modo Distribuido

  1. Crear un archivo de propiedades para el trabajador:

    bootstrap.servers=localhost:9092
group.id=connect-cluster
key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter
config.storage.topic=connect-configs
offset.storage.topic=connect-offsets
status.storage.topic=connect-status

  2. Iniciar el trabajador de Kafka Connect:

    bin/connect-distributed.sh config/connect-distributed.properties

  3. Enviar una configuración de conector:

    curl -X POST -H "Content-Type: application/json" --data '{
    "name": "local-file-source",
    "config": {
        "connector.class": "FileStreamSource",
        "tasks.max": "1",
        "file": "/path/to/input/file.txt",
        "topic": "connect-test"
    }
}' http://localhost:8083/connectors

Conectores Comunes

Conectores de Fuente

  • Conector de Fuente JDBC: Captura datos de bases de datos relacionales.
  • Conector de Fuente de Archivo: Captura datos de archivos.

Conectores de Sumidero

  • Conector de Sumidero JDBC: Escribe datos en bases de datos relacionales.
  • Conector de Sumidero Elasticsearch: Escribe datos en Elasticsearch.

Ejemplo: Conector de Fuente JDBC

Configuración

  1. Crear un archivo de propiedades para el Conector de Fuente JDBC:

    name=my-jdbc-source
connector.class=io.confluent.connect.jdbc.JdbcSourceConnector
tasks.max=1
connection.url=jdbc:mysql://localhost:3306/mydb
connection.user=myuser
connection.password=mypassword
table.whitelist=mytable
mode=incrementing
incrementing.column.name=id
topic.prefix=mydb-

  2. Iniciar el conector:

    bin/connect-standalone.sh config/connect-standalone.properties config/my-jdbc-source.properties

Monitoreo y Gestión

  • API REST: Kafka Connect proporciona una API REST para gestionar conectores, tareas y trabajadores.
  • Métricas: Kafka Connect expone métricas a través de JMX para monitorear el rendimiento y la salud.

Conclusión

Kafka Connect es una herramienta poderosa para integrar Kafka con varios sistemas externos. Al comprender sus conceptos clave y configuraciones, puedes construir tuberías de datos escalables y confiables. Ya sea que trabajes en modo autónomo o distribuido, Kafka Connect simplifica el proceso de ingestión y exportación de datos, convirtiéndose en un componente esencial en el ecosistema de Kafka.
