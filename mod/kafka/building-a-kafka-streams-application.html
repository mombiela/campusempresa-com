Introducción a Kafka Streams

Kafka Streams es una biblioteca cliente para construir aplicaciones y microservicios, donde los datos de entrada y salida se almacenan en clústeres de Kafka. Combina la simplicidad de escribir y desplegar aplicaciones estándar de Java y Scala en el lado del cliente con los beneficios de la tecnología de clúster del lado del servidor de Kafka.

  • Conceptos Clave:
    • Stream: Un conjunto de datos no acotado y continuamente actualizado.
    • Table: Una vista de un stream como un changelog, representando el estado más reciente.
    • Topology: La representación lógica de los pasos de procesamiento del stream.
    • Processor: Un nodo en la topología que procesa registros.

Configurando Tu Entorno

Antes de construir una aplicación de Kafka Streams, necesitas configurar tu entorno de desarrollo.

  • Requisitos Previos:
    • Java Development Kit (JDK) 8 o superior.
    • Apache Kafka instalado y en funcionamiento.
    • Maven o Gradle para la gestión de dependencias.

Ejemplo: Configuración de Maven

Crea un archivo pom.xml con las dependencias necesarias.

Construyendo Tu Primera Aplicación de Kafka Streams

Vamos a construir una simple aplicación de Kafka Streams que lee de un tema de entrada, procesa los datos y escribe en un tema de salida.

Paso 1: Definir la Topología

Crea una clase Java para definir la topología de procesamiento del stream.

import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsConfig;
import org.apache.kafka.streams.kstream.KStream;

import java.util.Properties;

public class SimpleKafkaStreamsApp {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(StreamsConfig.APPLICATION_ID_CONFIG, "streams-example");
        props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, "org.apache.kafka.common.serialization.Serdes$StringSerde");
        props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, "org.apache.kafka.common.serialization.Serdes$StringSerde");

        StreamsBuilder builder = new StreamsBuilder();
        KStream<String, String> source = builder.stream("input-topic");
        source.to("output-topic");

        KafkaStreams streams = new KafkaStreams(builder.build(), props);
        streams.start();
    }
}

Paso 2: Ejecutar la Aplicación

  • Asegúrate de que Kafka esté en funcionamiento.
  • Crea los temas de entrada y salida usando las herramientas CLI de Kafka.
  • Ejecuta la aplicación Java.
$ kafka-topics.sh --create --topic input-topic --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1
$ kafka-topics.sh --create --topic output-topic --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1
$ java -cp target/kafka-streams-example-1.0-SNAPSHOT.jar com.example.SimpleKafkaStreamsApp

Conceptos Avanzados de Kafka Streams

Una vez que te sientas cómodo con lo básico, puedes explorar características más avanzadas.

  • Operaciones con Estado:
    • Agregaciones
    • Joins
    • Ventanas

Ejemplo: Aplicación de Conteo de Palabras

Un ejemplo común es una aplicación de conteo de palabras que cuenta las ocurrencias de palabras en el stream de entrada.

import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsConfig;
import org.apache.kafka.streams.kstream.KStream;
import org.apache.kafka.streams.kstream.KTable;
import org.apache.kafka.streams.kstream.Materialized;
import org.apache.kafka.streams.kstream.Produced;
import org.apache.kafka.common.serialization.Serdes;

import java.util.Arrays;
import java.util.Properties;

public class WordCountApp {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(StreamsConfig.APPLICATION_ID_CONFIG, "wordcount-example");
        props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, Serdes.String().getClass().getName());
        props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, Serdes.String().getClass().getName());

        StreamsBuilder builder = new StreamsBuilder();
        KStream<String, String> textLines = builder.stream("input-topic");
        KTable<String, Long> wordCounts = textLines
            .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
            .groupBy((key, value) -> value)
            .count(Materialized.as("counts-store"));

        wordCounts.toStream().to("output-topic", Produced.with(Serdes.String(), Serdes.Long()));

        KafkaStreams streams = new KafkaStreams(builder.build(), props);
        streams.start();
    }
}

Conclusión

Construir una aplicación de Kafka Streams implica configurar tu entorno, definir la topología y ejecutar la aplicación. A medida que avances, puedes aprovechar las operaciones con estado para construir aplicaciones de procesamiento de streams más complejas. Kafka Streams proporciona un marco poderoso y flexible para el procesamiento de datos en tiempo real, convirtiéndolo en una herramienta valiosa para cualquier desarrollador que trabaje con datos en streaming.
