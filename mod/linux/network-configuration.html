Introducción

La configuración de red en Linux implica configurar y gestionar interfaces de red, direcciones IP, rutas y configuraciones de DNS. Este tema te guiará desde tareas básicas hasta avanzadas de configuración de red en Linux.

Configuración Básica de Red

Visualización de Interfaces de Red

Para ver las interfaces de red actuales y sus estados, puedes usar el comando ip.

ip a
  • Explicación: El comando ip a lista todas las interfaces de red y sus detalles, como direcciones IP y estado (activado o desactivado).

Configuración de una Dirección IP Estática

Para configurar una dirección IP estática, puedes usar el comando ip.

sudo ip addr add 192.168.1.100/24 dev eth0
sudo ip link set dev eth0 up
  • Explicación:
    • ip addr add 192.168.1.100/24 dev eth0 asigna la dirección IP 192.168.1.100 con una máscara de subred de 255.255.255.0 a la interfaz eth0.
    • ip link set dev eth0 up activa la interfaz eth0.

Configuración de DNS

Las configuraciones de DNS se configuran típicamente en el archivo /etc/resolv.conf.

sudo nano /etc/resolv.conf

Añade las siguientes líneas para configurar los servidores DNS:

nameserver 8.8.8.8
nameserver 8.8.4.4
  • Explicación: Las líneas nameserver especifican las direcciones IP de los servidores DNS a utilizar.

Configuración Intermedia de Red

Uso de NetworkManager

NetworkManager es una herramienta para gestionar configuraciones de red en Linux.

Verificación del Estado de NetworkManager

sudo systemctl status NetworkManager
  • Explicación: Este comando verifica si NetworkManager está en ejecución.

Configuración de una Interfaz de Red con NetworkManager

nmcli con add type ethernet ifname eth0 con-name my-eth0 ip4 192.168.1.100/24 gw4 192.168.1.1
nmcli con up my-eth0
  • Explicación:
    • nmcli con add añade una nueva conexión llamada my-eth0 para la interfaz eth0 con la dirección IP y la puerta de enlace especificadas.
    • nmcli con up my-eth0 activa la conexión my-eth0.

Configuración de Rutas

Las rutas determinan cómo se reenvían los paquetes entre redes.

Añadir una Ruta Estática

sudo ip route add 192.168.2.0/24 via 192.168.1.1
  • Explicación: Este comando añade una ruta a la red 192.168.2.0/24 a través de la puerta de enlace 192.168.1.1.

Configuración Avanzada de Red

Uso de `netplan` para Configuración de Red

netplan es una utilidad para configurar ajustes de red en Ubuntu y otras distribuciones que usan netplan.

Configuración de netplan

Edita el archivo de configuración en /etc/netplan/.

sudo nano /etc/netplan/01-netcfg.yaml

Añade la siguiente configuración:

network:
  version: 2
  ethernets:
    eth0:
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

Aplica la configuración:

sudo netplan apply
  • Explicación: Esta configuración YAML establece una dirección IP estática, puerta de enlace y servidores DNS para la interfaz eth0.

Configuración de Firewall con `iptables`

iptables es una herramienta poderosa para configurar reglas de filtrado de paquetes de red.

Comandos Básicos de iptables

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
sudo iptables -A INPUT -j DROP
  • Explicación:
    • iptables -A INPUT -p tcp --dport 22 -j ACCEPT permite conexiones entrantes SSH.
    • iptables -A INPUT -j DROP descarta todo el tráfico entrante restante.

Uso de `firewalld` para Gestión de Firewall

firewalld es una herramienta de gestión de firewall dinámica con iptables como su backend.

Configuración de firewalld

sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload
  • Explicación:
    • firewall-cmd --permanent --add-service=ssh permite el servicio SSH de manera permanente.
    • firewall-cmd --reload recarga la configuración del firewall.

Conclusión

La configuración de red en Linux abarca desde tareas básicas como configurar direcciones IP y servidores DNS hasta tareas avanzadas como configurar firewalls y enrutamiento. El dominio de estas habilidades es esencial para gestionar sistemas Linux de manera efectiva. Al comprender y practicar estas configuraciones, puedes asegurar que tus sistemas Linux estén correctamente conectados y seguros.
