Introducción a SSH

Secure Shell (SSH) es un protocolo utilizado para conectarse de manera segura a sistemas remotos a través de una red. Proporciona un canal seguro sobre una red no segura mediante el uso de cifrado.

  • ¿Qué es SSH?

    • SSH significa Secure Shell.
    • Se utiliza para inicio de sesión remoto seguro y otros servicios de red seguros sobre una red insegura.
    • SSH utiliza criptografía de clave pública para autenticar la computadora remota y permitir que esta autentique al usuario.

  • ¿Por qué usar SSH?

    • Comunicación segura entre cliente y servidor.
    • Transferencia de datos cifrada.
    • Autenticación de usuarios y dispositivos.
    • Transferencia segura de archivos (SCP, SFTP).

Configuración de SSH

Para usar SSH, necesitas tener un cliente SSH en tu máquina local y un servidor SSH en la máquina remota.

Instalación de SSH

  • En Ubuntu/Debian:

    sudo apt update
sudo apt install openssh-server

  • En CentOS/RHEL:

    sudo yum install openssh-server
sudo systemctl start sshd
sudo systemctl enable sshd

  • En Windows:

    • Usa un cliente SSH como PuTTY o habilita el cliente OpenSSH incorporado en Windows 10.

Iniciar y habilitar el servicio SSH

  • En Ubuntu/Debian:

    sudo systemctl start ssh
sudo systemctl enable ssh

  • En CentOS/RHEL:

    sudo systemctl start sshd
sudo systemctl enable sshd

Conectarse a un servidor remoto

Para conectarse a un servidor remoto usando SSH, necesitas la dirección IP o el nombre de host del servidor y el nombre de usuario.

Comando básico de SSH

ssh username@hostname_or_ip
  • Ejemplo: 
    ssh user@192.168.1.10

Uso de claves SSH para autenticación

Las claves SSH proporcionan una forma más segura de iniciar sesión en un servidor SSH.

Generación de claves SSH

ssh-keygen -t rsa -b 2048
  • Este comando genera un par de claves: una clave privada y una clave pública.

Copiar la clave pública al servidor

ssh-copy-id username@hostname_or_ip
  • Ejemplo: 
    ssh-copy-id user@192.168.1.10

Uso avanzado de SSH

Reenvío de puertos

SSH puede usarse para reenviar puertos, lo que te permite acceder de manera segura a servicios en una red remota.

  • Reenvío de puerto local:

    ssh -L local_port:remote_address:remote_port username@hostname_or_ip
    • Ejemplo: 
      ssh -L 8080:localhost:80 user@192.168.1.10

  • Reenvío de puerto remoto:

    ssh -R remote_port:local_address:local_port username@hostname_or_ip
    • Ejemplo: 
      ssh -R 8080:localhost:80 user@192.168.1.10

Túnel SSH

El túnel SSH te permite crear una conexión segura entre tu máquina local y una máquina remota a través de un servidor SSH.

  • Reenvío de puerto dinámico (Proxy SOCKS): 
    ssh -D local_port username@hostname_or_ip
    • Ejemplo: 
      ssh -D 8080 user@192.168.1.10

Archivo de configuración de SSH

Puedes simplificar las conexiones SSH utilizando el archivo de configuración de SSH (~/.ssh/config).

  • Ejemplo de archivo de configuración:

    Host myserver
    HostName 192.168.1.10
    User user
    Port 22
    IdentityFile ~/.ssh/id_rsa

  • Conectarse usando el archivo de configuración:

    ssh myserver

Resumen

SSH es una herramienta poderosa para el acceso remoto seguro y la gestión de servidores. Al comprender los conceptos básicos de SSH, configurar claves SSH y utilizar funciones avanzadas como el reenvío de puertos y el túnel, puedes mejorar la seguridad y la eficiencia de tus operaciones remotas. Siempre asegúrate de que tu servidor SSH esté configurado y asegurado adecuadamente para prevenir accesos no autorizados.
