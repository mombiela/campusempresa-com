Introducción a YARN

YARN (Yet Another Resource Negotiator) es un componente central de Hadoop que gestiona recursos y programa trabajos. Permite que múltiples motores de procesamiento de datos, como procesamiento por lotes, procesamiento interactivo y procesamiento en tiempo real, se ejecuten y procesen datos almacenados en HDFS.

Conceptos Clave de YARN

  • ResourceManager (RM): El demonio maestro que gestiona recursos y programa aplicaciones.
  • NodeManager (NM): El agente por nodo responsable de gestionar contenedores, monitorear su uso de recursos (CPU, memoria, disco, red) e informar al ResourceManager.
  • ApplicationMaster (AM): Una biblioteca específica del marco que negocia recursos con el ResourceManager y trabaja con los NodeManager(s) para ejecutar y monitorear tareas.
  • Contenedor: Una colección de recursos físicos (CPU, memoria, etc.) asignados por el ResourceManager para una aplicación específica.

Arquitectura de YARN

YARN sigue una arquitectura maestro-esclavo con el ResourceManager como el maestro y los NodeManagers como los esclavos.

Componentes del ResourceManager

  • Scheduler: Asigna recursos a varias aplicaciones en ejecución según la disponibilidad de recursos y las políticas configuradas.
  • ApplicationManager: Gestiona el ciclo de vida de las aplicaciones, incluyendo la aceptación de envíos de trabajos, la negociación del primer contenedor para ejecutar el ApplicationMaster y el reinicio del ApplicationMaster en caso de fallo.

Componentes del NodeManager

  • ContainerManager: Gestiona el ciclo de vida de los contenedores en un nodo, incluyendo el inicio, la detención y el monitoreo del uso de recursos.
  • Monitoreo de Recursos: Monitorea el uso de recursos de los contenedores e informa al ResourceManager.

Asignación de Recursos en YARN

YARN utiliza un planificador enchufable para asignar recursos. Los planificadores más comunes son:

  • Planificador FIFO: Simple y directo, procesa trabajos en el orden en que llegan.
  • Planificador de Capacidad: Permite compartir recursos entre múltiples organizaciones, asegurando una garantía de capacidad mínima.
  • Planificador Justo: Asigna recursos de manera justa entre todos los trabajos en ejecución, asegurando que ningún trabajo se quede sin recursos.

Ejemplo de Configuración para el Planificador de Capacidad

<property>
  <name>yarn.scheduler.capacity.root.queues</name>
  <value>default,production</value>
</property>
<property>
  <name>yarn.scheduler.capacity.root.default.capacity</name>
  <value>50</value>
</property>
<property>
  <name>yarn.scheduler.capacity.root.production.capacity</name>
  <value>50</value>
</property>

En este ejemplo, se configuran dos colas (default y production), cada una con el 50% de la capacidad total del clúster.

Ejecución de Aplicaciones en YARN

Cuando se envía una aplicación a YARN, ocurren los siguientes pasos:

  1. Envío: El cliente envía la aplicación al ResourceManager.
  2. Lanzamiento del ApplicationMaster: El ResourceManager asigna un contenedor para el ApplicationMaster y lo inicia.
  3. Solicitud de Recursos: El ApplicationMaster solicita recursos para las tareas de la aplicación.
  4. Ejecución de Tareas: Los NodeManagers lanzan contenedores para ejecutar las tareas.
  5. Monitoreo y Finalización: El ApplicationMaster monitorea las tareas e informa el estado al cliente. Una vez que todas las tareas se completan, el ApplicationMaster se apaga.

Ejemplo de Envío de una Aplicación YARN

yarn jar my-app.jar com.example.MyApp

Este comando envía la aplicación my-app.jar a YARN, donde com.example.MyApp es la clase principal.

Monitoreo y Gestión de Aplicaciones YARN

YARN proporciona varias herramientas para monitorear y gestionar aplicaciones:

  • Interfaz Web del ResourceManager: Accesible a través de http://<ResourceManager-host>:8088, proporciona información sobre aplicaciones en ejecución, métricas del clúster y uso de recursos.
  • Herramientas de Línea de Comandos: Comandos como yarn application -list, yarn application -status <appId> y yarn application -kill <appId> permiten a los usuarios gestionar aplicaciones desde la línea de comandos.

Ejemplo de Comprobación del Estado de una Aplicación

yarn application -status application_1234567890123_0001

Este comando recupera el estado de la aplicación con el ID application_1234567890123_0001.

Conclusión

YARN es una capa de gestión de recursos poderosa en Hadoop que permite una asignación eficiente de recursos y programación de trabajos. Comprender su arquitectura, componentes y mecanismos de asignación de recursos es crucial para gestionar y ejecutar aplicaciones de manera efectiva en un clúster de Hadoop. Al aprovechar las capacidades de YARN, los usuarios pueden optimizar la utilización de recursos y asegurar un procesamiento justo y eficiente de sus cargas de trabajo de datos.
