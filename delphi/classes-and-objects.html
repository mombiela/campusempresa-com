Introducción a Clases y Objetos

Las clases y los objetos son conceptos fundamentales en la Programación Orientada a Objetos (OOP). En Delphi/Object Pascal, estos conceptos te permiten crear estructuras de datos y funcionalidades complejas de manera modular y reutilizable.

Conceptos Clave

  • Clase: Un plano para crear objetos. Define un tipo de objeto según los atributos (campos) y comportamientos (métodos) que debe tener.
  • Objeto: Una instancia de una clase. Representa una implementación específica de la clase con valores y comportamientos reales.

Definiendo una Clase

Para definir una clase en Delphi/Object Pascal, se utiliza la palabra clave class. Aquí hay un ejemplo básico:

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
  public
    procedure SetName(AName: string);
    function GetName: string;
    procedure SetAge(AAge: Integer);
    function GetAge: Integer;
  end;

Explicación

  • type: Declara un nuevo tipo.
  • TPerson: El nombre de la clase.
  • private: Sección donde se declaran los campos privados. Estos campos solo son accesibles dentro de la clase misma.
  • public: Sección donde se declaran los métodos públicos. Estos métodos pueden ser accedidos desde fuera de la clase.

Creando y Usando Objetos

Para crear un objeto, se utiliza el método Create de la clase. Aquí hay un ejemplo de cómo crear y usar un objeto de la clase TPerson:

var
  Person: TPerson;
begin
  Person := TPerson.Create;
  Person.SetName('John Doe');
  Person.SetAge(30);
  WriteLn('Name: ', Person.GetName);
  WriteLn('Age: ', Person.GetAge);
  Person.Free;
end;

Explicación

  • Person: Una variable de tipo TPerson.
  • TPerson.Create: Crea una instancia de la clase TPerson.
  • SetName y SetAge: Métodos para establecer el nombre y la edad de la persona.
  • GetName y GetAge: Métodos para obtener el nombre y la edad de la persona.
  • Free: Libera la memoria asignada para el objeto.

Constructores y Destructores

Los constructores y destructores son métodos especiales utilizados para inicializar y limpiar objetos.

Ejemplo de Constructor

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
  public
    constructor Create(AName: string; AAge: Integer);
    destructor Destroy; override;
    procedure DisplayInfo;
  end;

constructor TPerson.Create(AName: string; AAge: Integer);
begin
  FName := AName;
  FAge := AAge;
end;

destructor TPerson.Destroy;
begin
  // Código de limpieza aquí
  inherited Destroy;
end;

procedure TPerson.DisplayInfo;
begin
  WriteLn('Name: ', FName);
  WriteLn('Age: ', FAge);
end;

Usando el Constructor

var
  Person: TPerson;
begin
  Person := TPerson.Create('Jane Doe', 25);
  Person.DisplayInfo;
  Person.Free;
end;

Explicación

  • constructor Create: Inicializa el objeto con valores específicos.
  • destructor Destroy: Limpia el objeto antes de que sea destruido.
  • override: Indica que el método sobrescribe un método en la clase base.

Herencia

La herencia permite que una clase herede campos y métodos de otra clase.

Ejemplo de Herencia

type
  TEmployee = class(TPerson)
  private
    FEmployeeID: Integer;
  public
    constructor Create(AName: string; AAge: Integer; AEmployeeID: Integer);
    procedure DisplayInfo; override;
  end;

constructor TEmployee.Create(AName: string; AAge: Integer; AEmployeeID: Integer);
begin
  inherited Create(AName, AAge);
  FEmployeeID := AEmployeeID;
end;

procedure TEmployee.DisplayInfo;
begin
  inherited DisplayInfo;
  WriteLn('Employee ID: ', FEmployeeID);
end;

Usando la Clase Heredada

var
  Employee: TEmployee;
begin
  Employee := TEmployee.Create('John Smith', 40, 12345);
  Employee.DisplayInfo;
  Employee.Free;
end;

Explicación

  • TEmployee = class(TPerson): TEmployee hereda de TPerson.
  • inherited Create: Llama al constructor de la clase base.
  • override: Indica que el método sobrescribe un método en la clase base.

Polimorfismo

El polimorfismo permite que los métodos sean llamados en objetos de diferentes clases a través de una interfaz común.

Ejemplo de Polimorfismo

type
  TPerson = class
  public
    procedure Speak; virtual;
  end;

  TEmployee = class(TPerson)
  public
    procedure Speak; override;
  end;

procedure TPerson.Speak;
begin
  WriteLn('Person speaking');
end;

procedure TEmployee.Speak;
begin
  WriteLn('Employee speaking');
end;

var
  Person: TPerson;
  Employee: TEmployee;
begin
  Person := TPerson.Create;
  Employee := TEmployee.Create;
  Person.Speak; // Imprime: Person speaking
  Employee.Speak; // Imprime: Employee speaking
  Person := Employee;
  Person.Speak; // Imprime: Employee speaking
  Person.Free;
  Employee.Free;
end;

Explicación

  • virtual: Indica que el método puede ser sobrescrito en una clase derivada.
  • override: Indica que el método sobrescribe un método en la clase base.
  • Person := Employee: Demuestra el polimorfismo asignando un objeto Employee a una variable Person.

Conclusión

Las clases y los objetos son herramientas poderosas en Delphi/Object Pascal que te permiten crear código modular, reutilizable y mantenible. Al entender y utilizar conceptos como constructores, destructores, herencia y polimorfismo, puedes construir aplicaciones complejas con facilidad. Practica estos conceptos con varios ejemplos para solidificar tu comprensión y convertirte en un experto en Programación Orientada a Objetos con Delphi/Object Pascal.
