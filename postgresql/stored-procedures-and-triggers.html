Introducción

Los procedimientos almacenados y los triggers son componentes esenciales en PostgreSQL que te permiten automatizar y encapsular la lógica de negocio a nivel de la base de datos. Esta sección te guiará desde los conceptos básicos hasta el uso avanzado de procedimientos almacenados y triggers.

Procedimientos Almacenados

¿Qué son los Procedimientos Almacenados?

  • Los procedimientos almacenados son un conjunto de sentencias SQL que pueden ser almacenadas en la base de datos y ejecutadas como una unidad.
  • Ayudan a encapsular lógica de negocio compleja, mejorando el rendimiento y asegurando la reutilización del código.

Creando un Procedimiento Almacenado Simple

Para crear un procedimiento almacenado en PostgreSQL, utilizas la sentencia CREATE PROCEDURE.

CREATE PROCEDURE add_numbers(IN num1 INT, IN num2 INT, OUT sum INT)
LANGUAGE plpgsql
AS $$
BEGIN
    sum := num1 + num2;
END;
$$;
  • Los parámetros IN son entradas para el procedimiento.
  • Los parámetros OUT son salidas del procedimiento.
  • LANGUAGE plpgsql especifica que el procedimiento está escrito en PL/pgSQL.

Ejecutando un Procedimiento Almacenado

Para ejecutar un procedimiento almacenado, utilizas la sentencia CALL.

CALL add_numbers(5, 10, sum);

Ventajas de los Procedimientos Almacenados

  • Rendimiento: Reducción de la comunicación cliente-servidor.
  • Seguridad: Encapsulación de la lógica de negocio.
  • Mantenibilidad: Gestión centralizada del código.

Procedimientos Almacenados Avanzados

Los procedimientos almacenados también pueden incluir estructuras de control como bucles y sentencias condicionales.

CREATE PROCEDURE calculate_factorial(IN num INT, OUT result INT)
LANGUAGE plpgsql
AS $$
DECLARE
    i INT := 1;
BEGIN
    result := 1;
    WHILE i <= num LOOP
        result := result * i;
        i := i + 1;
    END LOOP;
END;
$$;

Triggers

¿Qué son los Triggers?

  • Los triggers son tipos especiales de procedimientos almacenados que se ejecutan automáticamente o "disparan" cuando ocurren ciertos eventos en la base de datos.
  • Los eventos pueden ser INSERT, UPDATE o DELETE.

Creando un Trigger Simple

Para crear un trigger, necesitas definir una función de trigger y luego vincularla a un evento de tabla.

Paso 1: Crear una Función de Trigger

CREATE FUNCTION log_changes() RETURNS trigger AS $$
BEGIN
    IF TG_OP = 'INSERT' THEN
        INSERT INTO audit_log(table_name, operation, new_data)
        VALUES (TG_TABLE_NAME, TG_OP, row_to_json(NEW));
    ELSIF TG_OP = 'UPDATE' THEN
        INSERT INTO audit_log(table_name, operation, old_data, new_data)
        VALUES (TG_TABLE_NAME, TG_OP, row_to_json(OLD), row_to_json(NEW));
    ELSIF TG_OP = 'DELETE' THEN
        INSERT INTO audit_log(table_name, operation, old_data)
        VALUES (TG_TABLE_NAME, TG_OP, row_to_json(OLD));
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

Paso 2: Vincular la Función de Trigger a un Evento de Tabla

CREATE TRIGGER audit_trigger
AFTER INSERT OR UPDATE OR DELETE ON your_table
FOR EACH ROW EXECUTE FUNCTION log_changes();

Ventajas de los Triggers

  • Automatización: Aplicación automática de reglas de negocio.
  • Consistencia: Asegura la integridad de los datos.
  • Auditabilidad: Seguimiento de cambios en los datos.

Triggers Avanzados

Los triggers pueden ser utilizados para operaciones complejas como actualizaciones en cascada o mantenimiento de tablas de resumen.

CREATE FUNCTION update_summary() RETURNS trigger AS $$
BEGIN
    UPDATE summary_table
    SET total = total + NEW.amount
    WHERE id = NEW.id;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_summary_trigger
AFTER INSERT ON transactions
FOR EACH ROW EXECUTE FUNCTION update_summary();

Conclusión

Los procedimientos almacenados y los triggers son herramientas poderosas en PostgreSQL que ayudan a automatizar y encapsular lógica de negocio compleja. Mejoran el rendimiento, la seguridad y la mantenibilidad de tus aplicaciones de base de datos. Al dominar estas herramientas, puedes crear sistemas de base de datos más eficientes y robustos.
