Introducción

Los tipos de datos son fundamentales para cualquier sistema de bases de datos, incluyendo PostgreSQL. Definen el tipo de datos que se pueden almacenar en una columna de una tabla. Entender los tipos de datos es crucial para diseñar bases de datos eficientes y escribir consultas efectivas.

Conceptos Clave

  • Tipo de Datos: Una clasificación que especifica qué tipo de valor puede contener una columna.
  • Categorías de Tipos: PostgreSQL soporta varias categorías de tipos de datos, incluyendo numéricos, de caracteres, de fecha/hora, y más.
  • Restricciones de Tipo: Reglas que limitan el tipo de datos que se pueden almacenar en una columna.

Tipos Numéricos

Los tipos de datos numéricos se utilizan para almacenar números. PostgreSQL soporta varios tipos numéricos:

| Tipo de Datos | Descripción | Tamaño de Almacenamiento | |---------------|-------------|--------------------------| | smallint | Entero de pequeño rango | 2 bytes | | integer o int | Opción típica para enteros | 4 bytes | | bigint | Entero de gran rango | 8 bytes | | decimal o numeric | Precisión especificada por el usuario | Variable | | real | Punto flotante de precisión simple | 4 bytes | | double precision | Punto flotante de doble precisión | 8 bytes | | serial | Entero auto-incremental | 4 bytes | | bigserial | Entero auto-incremental grande | 8 bytes |

Ejemplo

CREATE TABLE numbers_example (
    small_num smallint,
    medium_num integer,
    large_num bigint,
    precise_num numeric(10, 2),
    float_num real,
    double_num double precision
);

Explicación:

  • small_num puede almacenar enteros pequeños.
  • medium_num es adecuado para valores enteros típicos.
  • large_num puede almacenar enteros muy grandes.
  • precise_num almacena números con hasta 10 dígitos, 2 de los cuales son después del punto decimal.
  • float_num y double_num almacenan números de punto flotante con precisión simple y doble, respectivamente.

Tipos de Caracteres

Los tipos de datos de caracteres se utilizan para almacenar texto. PostgreSQL proporciona varias opciones:

| Tipo de Datos | Descripción | Tamaño de Almacenamiento | |---------------|-------------|--------------------------| | char(n) | Tipo de carácter de longitud fija | n bytes | | varchar(n) | Tipo de carácter de longitud variable | Hasta n bytes | | text | Tipo de carácter de longitud variable sin límite | Variable |

Ejemplo

CREATE TABLE text_example (
    fixed_char char(10),
    variable_char varchar(50),
    unlimited_text text
);

Explicación:

  • fixed_char almacena una cadena de longitud fija de 10 caracteres.
  • variable_char puede almacenar una cadena de hasta 50 caracteres.
  • unlimited_text puede almacenar una cadena de cualquier longitud.

Tipos de Fecha/Hora

Los tipos de datos de fecha y hora se utilizan para almacenar fechas, horas y marcas de tiempo.

| Tipo de Datos | Descripción | Tamaño de Almacenamiento | |---------------|-------------|--------------------------| | date | Fecha del calendario (año, mes, día) | 4 bytes | | time | Hora del día (sin zona horaria) | 8 bytes | | timestamp | Fecha y hora (sin zona horaria) | 8 bytes | | timestamptz | Fecha y hora con zona horaria | 8 bytes | | interval | Intervalo de tiempo | 16 bytes |

Ejemplo

CREATE TABLE datetime_example (
    event_date date,
    event_time time,
    event_timestamp timestamp,
    event_timestamptz timestamptz,
    duration interval
);

Explicación:

  • event_date almacena una fecha.
  • event_time almacena una hora del día.
  • event_timestamp almacena una fecha y hora.
  • event_timestamptz almacena una fecha y hora con información de zona horaria.
  • duration almacena un intervalo de tiempo.

Tipo Booleano

El tipo de datos booleano se utiliza para almacenar valores verdadero/falso.

| Tipo de Datos | Descripción | Tamaño de Almacenamiento | |---------------|-------------|--------------------------| | boolean | Booleano lógico (verdadero/falso) | 1 byte |

Ejemplo

CREATE TABLE boolean_example (
    is_active boolean
);

Explicación:

  • is_active puede almacenar un valor verdadero o falso.

Tipos de Arreglo

PostgreSQL permite definir columnas como arreglos de longitud variable.

Ejemplo

CREATE TABLE array_example (
    integer_array integer[],
    text_array text[]
);

Explicación:

  • integer_array puede almacenar un arreglo de enteros.
  • text_array puede almacenar un arreglo de cadenas de texto.

Tipos JSON

PostgreSQL soporta tipos de datos JSON para almacenar datos JSON.

| Tipo de Datos | Descripción | Tamaño de Almacenamiento | |---------------|-------------|--------------------------| | json | Datos JSON textuales | Variable | | jsonb | Datos JSON binarios, más eficientes para indexación | Variable |

Ejemplo

CREATE TABLE json_example (
    json_data json,
    jsonb_data jsonb
);

Explicación:

  • json_data almacena datos JSON en formato de texto.
  • jsonb_data almacena datos JSON en un formato binario, que es más eficiente para la indexación y consulta.

Conclusión

Entender los tipos de datos en PostgreSQL es esencial para diseñar bases de datos eficientes y efectivas. Al elegir el tipo de datos adecuado para cada columna, puedes asegurar la integridad de los datos, optimizar el almacenamiento y mejorar el rendimiento de las consultas. Este tema cubrió los tipos de datos más comúnmente utilizados, incluyendo numéricos, de caracteres, de fecha/hora, booleanos, de arreglo y tipos JSON. A medida que avances en tu viaje con PostgreSQL, encontrarás tipos de datos más especializados y aprenderás a usarlos efectivamente.
