Proyecto Final: Aplicación de Metodologías Ágiles en un Proyecto Real

Introducción

En este proyecto final, los estudiantes aplicarán las metodologías ágiles aprendidas durante el curso en un proyecto real. El objetivo es consolidar los conocimientos teóricos mediante la práctica y la resolución de problemas reales que podrían encontrarse en un entorno profesional.

Objetivos del Proyecto

  • Aplicar los principios y prácticas de metodologías ágiles.
  • Desarrollar habilidades de trabajo en equipo y comunicación.
  • Gestionar un proyecto desde la planificación hasta la entrega.
  • Evaluar y mejorar continuamente el proceso de desarrollo.

Descripción del Proyecto

Los estudiantes trabajarán en equipos para desarrollar una aplicación web utilizando una metodología ágil, como Scrum o Kanban. El proyecto incluirá todas las fases del ciclo de vida del desarrollo de software, desde la recopilación de requisitos hasta la entrega final.

Fases del Proyecto

  1. Planificación Inicial

    • Definir el alcance del proyecto.
    • Crear un backlog de producto.
    • Establecer roles y responsabilidades dentro del equipo.

  2. Desarrollo Iterativo

    • Dividir el trabajo en sprints (si se usa Scrum) o en tareas (si se usa Kanban).
    • Realizar reuniones diarias de seguimiento (Daily Stand-ups).
    • Implementar y probar las funcionalidades planificadas.

  3. Revisión y Retrospectiva

    • Realizar revisiones de sprint para demostrar el progreso.
    • Conducir retrospectivas para identificar áreas de mejora.

  4. Entrega Final

    • Preparar la documentación del proyecto.
    • Realizar una presentación final del producto desarrollado.

Herramientas Recomendadas

  • Gestión de Proyectos: Jira, Trello
  • Control de Versiones: Git, GitHub
  • Comunicación: Slack, Microsoft Teams
  • Desarrollo y Pruebas: Visual Studio Code, Postman

Ejemplo de Aplicación

Para ilustrar cómo se puede aplicar una metodología ágil en un proyecto real, consideremos el desarrollo de una aplicación de gestión de tareas.

Planificación Inicial

  • Backlog de Producto:

    • Crear tareas.
    • Editar tareas.
    • Eliminar tareas.
    • Asignar tareas a usuarios.
    • Marcar tareas como completadas.

  • Roles y Responsabilidades:

    • Product Owner: Responsable de definir y priorizar el backlog.
    • Scrum Master: Facilita las reuniones y elimina impedimentos.
    • Equipo de Desarrollo: Implementa las funcionalidades.

Desarrollo Iterativo

  • Sprint 1:

    • Crear tareas.
    • Editar tareas.

  • Sprint 2:

    • Eliminar tareas.
    • Asignar tareas a usuarios.

  • Sprint 3:

    • Marcar tareas como completadas.
    • Pruebas y corrección de errores.

Revisión y Retrospectiva

  • Revisión de Sprint:

    • Demostración de las funcionalidades completadas.
    • Feedback del Product Owner.

  • Retrospectiva:

    • ¿Qué salió bien?
    • ¿Qué se puede mejorar?
    • Plan de acción para el próximo sprint.

Conclusión

El proyecto final es una oportunidad para que los estudiantes apliquen los conocimientos adquiridos sobre metodologías ágiles en un entorno práctico. A través de la planificación, el desarrollo iterativo, y las revisiones continuas, los estudiantes podrán experimentar de primera mano los beneficios de las metodologías ágiles y desarrollar habilidades clave para su carrera profesional en TI.
