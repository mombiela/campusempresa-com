Introducción a los Protocolos

Los protocolos en Swift definen un esquema de métodos, propiedades y otros requisitos que se adaptan a una tarea o funcionalidad particular. Cualquier clase, estructura o enumeración puede adoptar protocolos para proporcionar una implementación real de esos requisitos.

Conceptos Clave

  • Declaración de Protocolo: Cómo declarar un protocolo.
  • Adopción de un Protocolo: Cómo hacer que una clase, estructura o enumeración cumpla con un protocolo.
  • Herencia de Protocolo: Cómo los protocolos pueden heredar de otros protocolos.
  • Requisitos Opcionales del Protocolo: Cómo definir requisitos opcionales en los protocolos.

Declaración de Protocolo

Para declarar un protocolo, usa la palabra clave protocol seguida del nombre del protocolo y un conjunto de requisitos.

protocol Describable {
    var description: String { get }
    func describe() -> String
}

Adopción de un Protocolo

Un tipo adopta un protocolo enumerando el nombre del protocolo después del nombre del tipo, separado por dos puntos.

struct Car: Describable {
    var description: String
    var make: String
    var model: String

    func describe() -> String {
        return "Este coche es un \(make) \(model)."
    }
}

Herencia de Protocolo

Los protocolos pueden heredar de uno o más otros protocolos, y un protocolo puede requerir que los tipos conformantes implementen propiedades o métodos de múltiples protocolos.

protocol Identifiable {
    var id: String { get }
}

protocol Nameable {
    var name: String { get }
}

protocol Person: Identifiable, Nameable {
    var age: Int { get }
}

Requisitos Opcionales del Protocolo

Los protocolos pueden definir requisitos opcionales usando el atributo @objc y la palabra clave optional. Esto solo está disponible para protocolos que están marcados con el atributo @objc.

@objc protocol OptionalProtocol {
    @objc optional func optionalMethod()
}

Introducción a las Extensiones

Las extensiones añaden nueva funcionalidad a una clase, estructura, enumeración o tipo de protocolo existente. Esto incluye la capacidad de extender tipos para los cuales no tienes acceso al código fuente original.

Conceptos Clave

  • Declaración de Extensión: Cómo declarar una extensión.
  • Añadir Propiedades Computadas: Cómo añadir propiedades computadas usando extensiones.
  • Añadir Métodos: Cómo añadir nuevos métodos usando extensiones.
  • Conformidad de Protocolo con Extensiones: Cómo usar extensiones para cumplir con protocolos.

Declaración de Extensión

Para declarar una extensión, usa la palabra clave extension seguida del nombre del tipo que deseas extender.

extension Int {
    var isEven: Bool {
        return self % 2 == 0
    }
}

Añadir Propiedades Computadas

Las extensiones pueden añadir propiedades computadas pero no pueden añadir propiedades almacenadas.

extension Double {
    var km: Double { return self * 1_000.0 }
    var m: Double { return self }
    var cm: Double { return self / 100.0 }
    var mm: Double { return self / 1_000.0 }
}

Añadir Métodos

Las extensiones pueden añadir nuevos métodos a tipos existentes.

extension String {
    func reversedString() -> String {
        return String(self.reversed())
    }
}

Conformidad de Protocolo con Extensiones

Las extensiones pueden ser usadas para hacer que un tipo existente cumpla con un protocolo.

protocol Greetable {
    func greet()
}

extension String: Greetable {
    func greet() {
        print("¡Hola, \(self)!")
    }
}

Resumen

En esta sección, exploramos los conceptos de protocolos y extensiones en Swift. Los protocolos definen un esquema de métodos y propiedades que pueden ser adoptados por clases, estructuras y enumeraciones. Las extensiones nos permiten añadir nueva funcionalidad a tipos existentes, incluyendo la capacidad de cumplir con protocolos. Entender estos conceptos es crucial para escribir código flexible y reutilizable en Swift.
