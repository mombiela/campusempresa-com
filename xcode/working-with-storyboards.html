Introducción a los Storyboards

Los storyboards son una representación visual de la interfaz de usuario de tu aplicación en Xcode. Te permiten diseñar y organizar los elementos de la UI y los controladores de vista de tu aplicación de manera visual.

  • ¿Qué es un Storyboard?

    • Un archivo que contiene la representación visual de la UI de tu aplicación.
    • Te permite diseñar el flujo de tu aplicación usando escenas y segues.
    • Proporciona una manera de ver toda la UI de tu aplicación en un solo lugar.

  • Ventajas de Usar Storyboards

    • Representación visual de la UI de la aplicación.
    • Más fácil de gestionar y entender el flujo de la aplicación.
    • Simplifica el proceso de diseñar interfaces de usuario complejas.

Creación y Configuración de Storyboards

Creación de un Nuevo Storyboard

  1. Abre Xcode y crea un nuevo proyecto.
  2. Selecciona la plantilla "Single View App".
  3. En el navegador de proyectos, haz clic derecho en el archivo "Main.storyboard" y selecciona "New File".
  4. Elige "Storyboard" de las opciones de plantilla y haz clic en "Next".
  5. Nombra tu archivo de storyboard y haz clic en "Create".

Añadiendo Controladores de Vista

  • Arrastra y suelta un View Controller desde la Biblioteca de Objetos al storyboard.
  • Establece el controlador de vista inicial seleccionando el controlador de vista y marcando la casilla "Is Initial View Controller" en el Inspector de Atributos.

Configuración de los Controladores de Vista

  • Estableciendo la Clase: Asigna una clase personalizada a tu controlador de vista.

    class MyViewController: UIViewController {
    override func viewDidLoad() {
        super.viewDidLoad()
        // Código de inicialización personalizado
    }
}
    • En el Inspector de Identidad, establece la clase del controlador de vista a MyViewController.

  • Añadiendo Elementos de UI: Arrastra y suelta elementos de UI (por ejemplo, botones, etiquetas) desde la Biblioteca de Objetos al controlador de vista.

    • Personaliza las propiedades usando el Inspector de Atributos.

Trabajando con Segues

Los segues definen las transiciones entre controladores de vista en un storyboard.

Creación de un Segue

  1. Arrastra con control desde un elemento de UI (por ejemplo, un botón) a otro controlador de vista.
  2. Selecciona el tipo de segue (por ejemplo, Show, Present Modally).

Configuración de Segues

  • Identificador: Establece un identificador para el segue en el Inspector de Atributos.
  • Realizando un Segue Programáticamente: 
    performSegue(withIdentifier: "showDetail", sender: self)

Pasando Datos Entre Controladores de Vista

  • Preparar para Segue: Sobrescribe el método prepare(for:sender:) para pasar datos. 
    override func prepare(for segue: UIStoryboardSegue, sender: Any?) {
    if segue.identifier == "showDetail" {
        let destinationVC = segue.destination as! DetailViewController
        destinationVC.data = "Some data"
    }
}

Auto Layout y Constraints

Auto Layout te permite crear interfaces de usuario responsivas y adaptativas.

Añadiendo Constraints

  • Selecciona un elemento de UI y haz clic en el botón "Add New Constraints" en el Interface Builder.
  • Establece las constraints deseadas (por ejemplo, leading, trailing, top, bottom).

Usando Stack Views

  • Los stack views simplifican el proceso de crear layouts complejos.
  • Arrastra y suelta un Stack View desde la Biblioteca de Objetos.
  • Añade elementos de UI al stack view y configura sus propiedades (por ejemplo, axis, distribution).

Técnicas Avanzadas de Storyboard

Segues Personalizados

  • Crea clases de segue personalizadas subclasificando UIStoryboardSegue. 
    class CustomSegue: UIStoryboardSegue {
    override func perform() {
        // Código de transición personalizado
    }
}

Referencias de Storyboard

  • Usa referencias de storyboard para dividir storyboards grandes en partes más pequeñas y manejables.
  • Arrastra y suelta una Referencia de Storyboard desde la Biblioteca de Objetos.
  • Establece el storyboard y el controlador de vista referenciados en el Inspector de Atributos.

Conclusión

Los storyboards son una herramienta poderosa en Xcode para diseñar y gestionar la interfaz de usuario de tu aplicación. Proporcionan una manera visual de diseñar tu UI, definir transiciones y gestionar el flujo de tu aplicación. Al entender cómo crear y configurar storyboards, añadir controladores de vista, trabajar con segues y usar Auto Layout, puedes construir interfaces de usuario complejas y responsivas de manera eficiente. Técnicas avanzadas como segues personalizados y referencias de storyboard mejoran aún más tu capacidad para gestionar proyectos grandes. Con práctica, te volverás competente en el uso de storyboards para crear aplicaciones intuitivas y visualmente atractivas.
