Introducción

Ansible es una poderosa herramienta de automatización que puede gestionar tanto servidores Linux como Windows. Esta sección te guiará a través del proceso de gestión de servidores Windows usando Ansible, desde la configuración básica hasta configuraciones avanzadas.

Requisitos Previos

Antes de comenzar a gestionar servidores Windows con Ansible, asegúrate de tener lo siguiente:

  • Una máquina de control con Ansible instalado (Linux o macOS).
  • Servidores Windows con WinRM habilitado.
  • Conocimiento básico de Ansible y YAML.

Configuración del Entorno

Instalación de Paquetes Requeridos

Para gestionar servidores Windows, necesitas instalar el paquete pywinrm en tu máquina de control.

pip install "pywinrm>=0.3.0"

Configuración de WinRM en Servidores Windows

WinRM (Windows Remote Management) debe estar configurado en tus servidores Windows. Puedes usar el siguiente script de PowerShell para habilitar y configurar WinRM:

# Habilitar WinRM
winrm quickconfig -force

# Configurar el servicio WinRM para que inicie automáticamente
Set-Service -Name WinRM -StartupType Automatic

# Permitir tráfico no cifrado (opcional, para propósitos de prueba)
winrm set winrm/config/service '@{AllowUnencrypted="true"}'

# Permitir autenticación básica (opcional, para propósitos de prueba)
winrm set winrm/config/service/auth '@{Basic="true"}'

# Crear un listener de WinRM
winrm create winrm/config/Listener?Address=*+Transport=HTTP

# Verificar la configuración
winrm get winrm/config

Configuración del Inventario

Definición de Hosts Windows

En tu archivo de inventario de Ansible, define tus hosts Windows y especifica los detalles de conexión.

[windows]
windows_server ansible_host=192.168.1.10

[windows:vars]
ansible_user=Administrator
ansible_password=YourPassword
ansible_connection=winrm
ansible_winrm_transport=basic
ansible_port=5985
ansible_winrm_server_cert_validation=ignore

Playbook Básico para Windows

Creación de un Playbook Simple

Crea un playbook simple de Ansible para probar la conexión a tu servidor Windows.

---
- name: Test connection to Windows server
  hosts: windows
  tasks:
    - name: Ping the Windows server
      win_ping:

Ejecuta el playbook con el siguiente comando:

ansible-playbook test_connection.yml

Gestión de Características de Windows

Instalación de Características de Windows

Puedes usar el módulo win_feature para instalar características de Windows.

---
- name: Install IIS on Windows server
  hosts: windows
  tasks:
    - name: Install IIS
      win_feature:
        name: Web-Server
        state: present

Eliminación de Características de Windows

De manera similar, puedes eliminar características de Windows usando el módulo win_feature.

---
- name: Remove IIS from Windows server
  hosts: windows
  tasks:
    - name: Remove IIS
      win_feature:
        name: Web-Server
        state: absent

Gestión de Archivos y Directorios

Copia de Archivos

Usa el módulo win_copy para copiar archivos al servidor Windows.

---
- name: Copy file to Windows server
  hosts: windows
  tasks:
    - name: Copy a file
      win_copy:
        src: /path/to/local/file.txt
        dest: C:\path\to\remote\file.txt

Gestión de Directorios

Usa el módulo win_file para crear o eliminar directorios.

---
- name: Manage directories on Windows server
  hosts: windows
  tasks:
    - name: Create a directory
      win_file:
        path: C:\path\to\directory
        state: directory

    - name: Delete a directory
      win_file:
        path: C:\path\to\directory
        state: absent

Configuraciones Avanzadas

Gestión de Servicios de Windows

Usa el módulo win_service para gestionar servicios de Windows.

---
- name: Manage Windows services
  hosts: windows
  tasks:
    - name: Ensure the service is running
      win_service:
        name: wuauserv
        start_mode: auto
        state: started

    - name: Stop a service
      win_service:
        name: wuauserv
        state: stopped

Ejecución de Scripts de PowerShell

Usa el módulo win_shell o win_command para ejecutar scripts de PowerShell.

---
- name: Execute PowerShell script on Windows server
  hosts: windows
  tasks:
    - name: Run PowerShell script
      win_shell: |
        Get-Process | Where-Object { $_.CPU -gt 100 }

Resumen

Gestionar servidores Windows con Ansible implica configurar WinRM, configurar el inventario y usar varios módulos de Ansible para realizar tareas. Siguiendo esta guía, puedes automatizar la gestión de servidores Windows, desde tareas básicas como copiar archivos hasta configuraciones avanzadas como gestionar servicios y ejecutar scripts de PowerShell. Con práctica, podrás aprovechar todo el potencial de Ansible para optimizar la gestión de tus servidores Windows.
