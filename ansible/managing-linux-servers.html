Introducción a Ansible

  • ¿Qué es Ansible?
    • Ansible es una herramienta de automatización de código abierto para tareas de TI como la gestión de configuraciones, el despliegue de aplicaciones y la automatización de tareas.
  • ¿Por qué usar Ansible para gestionar servidores Linux?
    • Arquitectura sin agentes
    • Sintaxis simple en YAML
    • Idempotencia (asegura que las tareas solo se apliquen cuando sea necesario)

Configuración de Ansible

  • Instalación
    • En una máquina Linux, instala Ansible usando el gestor de paquetes: 
      sudo apt update
sudo apt install ansible
  • Configuración
    • Archivo de configuración de Ansible (ansible.cfg): 
      [defaults]
inventory = ./hosts
  • Archivo de Inventario
    • Define tus nodos gestionados en un archivo de inventario (hosts): 
      [webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com

Comandos Básicos de Ansible

  • Hacer ping a todos los hosts 
    ansible all -m ping
  • Ejecutar un comando en todos los hosts 
    ansible all -a "/bin/echo hello"

Escritura de Playbooks

  • Introducción a los Playbooks
    • Los playbooks son archivos YAML que definen una serie de tareas a ejecutar en los nodos gestionados.
  • Estructura Básica de un Playbook 
    ---
- name: Ensure Apache is installed
  hosts: webservers
  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present
  • Ejecutar un Playbook 
    ansible-playbook playbook.yml

Características Avanzadas de los Playbooks

  • Variables 
    vars:
  http_port: 80
tasks:
  - name: Ensure Apache is listening on port 80
    lineinfile:
      path: /etc/apache2/ports.conf
      regexp: '^Listen'
      line: "Listen {{ http_port }}"
  • Handlers 
    tasks:
  - name: Install Apache
    apt:
      name: apache2
      state: present
    notify: Restart Apache

handlers:
  - name: Restart Apache
    service:
      name: apache2
      state: restarted
  • Templates
    • Uso de plantillas Jinja2 para gestionar archivos de configuración: 
      tasks:
  - name: Deploy Apache configuration
    template:
      src: templates/apache2.conf.j2
      dest: /etc/apache2/apache2.conf

Roles y Reutilización

  • Introducción a los Roles
    • Los roles te permiten organizar los playbooks en componentes reutilizables.
  • Crear un Rol 
    ansible-galaxy init myrole
  • Usar un Rol en un Playbook 
    ---
- name: Apply myrole to webservers
  hosts: webservers
  roles:
    - myrole

Gestión de Múltiples Entornos

  • Variables específicas del entorno
    • Define variables para diferentes entornos en archivos separados: 
      # group_vars/staging.yml
http_port: 8080

# group_vars/production.yml
http_port: 80
  • Uso de Playbooks específicos del entorno 
    ansible-playbook -i staging playbook.yml
ansible-playbook -i production playbook.yml

Ansible Vault

  • Encriptar Datos Sensibles
    • Encripta datos sensibles usando Ansible Vault: 
      ansible-vault encrypt vars/secrets.yml
  • Uso de Archivos Encriptados en Playbooks 
    vars_files:
  - vars/secrets.yml

Conclusión

Gestionar servidores Linux con Ansible simplifica y automatiza muchas tareas administrativas. Al entender los conceptos básicos de Ansible, escribir playbooks y usar características avanzadas como roles y Ansible Vault, puedes gestionar y escalar tu infraestructura de manera eficiente. Este curso ha proporcionado una visión general completa desde temas para principiantes hasta avanzados, equipándote con el conocimiento para aprovechar Ansible en tu entorno.
