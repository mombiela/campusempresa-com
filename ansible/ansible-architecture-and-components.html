Introducción

Ansible es una herramienta de automatización de código abierto utilizada para la gestión de configuraciones, despliegue de aplicaciones y automatización de tareas. Comprender su arquitectura y componentes es crucial para usar Ansible de manera efectiva en escenarios del mundo real. Esta sección cubrirá la arquitectura central y los componentes clave de Ansible.

Arquitectura de Ansible

Conceptos Clave

  • Arquitectura sin Agentes: Ansible no requiere que se instale ningún software agente en los nodos gestionados. Utiliza SSH para la comunicación.
  • Modelo Basado en Push: Ansible empuja configuraciones desde una máquina central (nodo de control) a los nodos gestionados.
  • Idempotencia: Ansible asegura que aplicar la misma configuración múltiples veces no cambiará el sistema después de la primera aplicación.

Nodo de Control y Nodos Gestionados

  • Nodo de Control: La máquina donde Ansible está instalado y desde la cual se ejecutan los comandos.
  • Nodos Gestionados: Las máquinas objetivo que son gestionadas por Ansible.

Inventario

  • Archivo de Inventario: Un archivo que lista los nodos gestionados. Puede ser un archivo de texto simple o un script de inventario dinámico. 
    [webservers]
web1.ejemplo.com
web2.ejemplo.com

[dbservers]
db1.ejemplo.com
db2.ejemplo.com

Componentes de Ansible

Playbooks

  • Playbooks: Archivos YAML que definen una serie de tareas a ser ejecutadas en los nodos gestionados. 
    - name: Instalar y arrancar Apache
  hosts: webservers
  tasks:
    - name: Instalar Apache
      apt:
        name: apache2
        state: present
    - name: Arrancar Apache
      service:
        name: apache2
        state: started

Módulos

  • Módulos: Scripts reutilizables e independientes que Ansible ejecuta en tu nombre. Pueden gestionar recursos del sistema, instalar software y más.
    • Ejemplo: apt, yum, service, copy, template.

Roles

  • Roles: Una forma de organizar playbooks y otros archivos para facilitar la reutilización y el intercambio.
    • Estructura del directorio: 
      roles/
  common/
    tasks/
      main.yml
    handlers/
      main.yml
    templates/
    files/
    vars/
      main.yml
    defaults/
      main.yml
    meta/
      main.yml

Plugins

  • Plugins: Extienden la funcionalidad central de Ansible. Los tipos incluyen:
    • Plugins de Acción: Modifican el comportamiento de las tareas de Ansible.
    • Plugins de Búsqueda: Recuperan datos de fuentes externas.
    • Plugins de Filtro: Transforman datos dentro de plantillas.
    • Plugins de Callback: Personalizan la salida o el registro.

Variables y Hechos

  • Variables: Se utilizan para almacenar valores que pueden ser reutilizados a lo largo de los playbooks. 
    vars:
  http_port: 80
  • Hechos: Información recopilada automáticamente sobre los nodos gestionados.
    • Ejemplo: ansible_facts, ansible_os_family.

Manejadores

  • Manejadores: Tareas especiales que son desencadenadas por otras tareas. Típicamente se utilizan para reinicios de servicios. 
    handlers:
  - name: restart apache
    service:
      name: apache2
      state: restarted

Conclusión

Comprender la arquitectura y los componentes de Ansible es fundamental para aprovechar su máximo potencial. El modelo sin agentes y basado en push simplifica la gestión de configuraciones, mientras que componentes como playbooks, módulos, roles y plugins proporcionan herramientas poderosas para la automatización. Al dominar estos elementos, puedes gestionar y automatizar eficientemente tu infraestructura de TI.
