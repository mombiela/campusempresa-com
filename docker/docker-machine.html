Docker Machine
Docker Compose vs Kubernetes ►

Docker Machine es una herramienta que te permite instalar Docker Engine en hosts virtuales y gestionar los hosts con comandos docker-machine. Puedes usar Machine para crear hosts Docker en tu máquina local, en proveedores de nube y en tu centro de datos.

Conceptos Clave

  • Docker Machine: Una herramienta para aprovisionar y gestionar hosts Docker.
  • Driver: Un plugin que permite a Docker Machine crear hosts en varias plataformas (por ejemplo, VirtualBox, AWS, DigitalOcean).
  • Machine: Un host Docker creado y gestionado por Docker Machine.
  • Variables de Entorno: Variables que configuran el cliente Docker para comunicarse con Docker Engine en una máquina específica.

Instalando Docker Machine

Para instalar Docker Machine, sigue estos pasos:

  1. Descarga el binario de Docker Machine: 
    curl -L https://github.com/docker/machine/releases/download/v0.16.2/docker-machine-$(uname -s)-$(uname -m) >/tmp/docker-machine
  2. Haz que el binario sea ejecutable: 
    chmod +x /tmp/docker-machine
  3. Mueve el binario a tu PATH: 
    sudo mv /tmp/docker-machine /usr/local/bin/docker-machine

Creando un Host Docker

Para crear un host Docker, usa el comando docker-machine create junto con un driver. Por ejemplo, para crear un host Docker usando VirtualBox:

docker-machine create --driver virtualbox my-docker-host

Explicación:

  • --driver virtualbox: Especifica el driver a usar (VirtualBox en este caso).
  • my-docker-host: El nombre del host Docker que se está creando.

Gestionando Hosts Docker

Una vez que has creado hosts Docker, puedes gestionarlos usando varios comandos docker-machine.

Listando Máquinas Docker

Para listar todas las máquinas Docker:

docker-machine ls

Iniciando y Deteniendo Máquinas Docker

Para iniciar una máquina Docker:

docker-machine start my-docker-host

Para detener una máquina Docker:

docker-machine stop my-docker-host

Eliminando una Máquina Docker

Para eliminar una máquina Docker:

docker-machine rm my-docker-host

Configurando el Cliente Docker

Para configurar el cliente Docker para usar una máquina Docker específica, necesitas establecer variables de entorno. Docker Machine proporciona una manera conveniente de hacer esto:

eval $(docker-machine env my-docker-host)

Explicación:

  • docker-machine env my-docker-host: Muestra las variables de entorno necesarias.
  • eval $(...): Evalúa la salida en la sesión actual del shell.

Ejemplo de Flujo de Trabajo

Aquí hay un ejemplo de flujo de trabajo para crear, gestionar y usar una máquina Docker:

  1. Crea una máquina Docker: 
    docker-machine create --driver virtualbox my-docker-host
  2. Lista las máquinas Docker: 
    docker-machine ls
  3. Configura el cliente Docker: 
    eval $(docker-machine env my-docker-host)
  4. Ejecuta un contenedor Docker: 
    docker run hello-world
  5. Detén la máquina Docker: 
    docker-machine stop my-docker-host
  6. Elimina la máquina Docker: 
    docker-machine rm my-docker-host

Conclusión

Docker Machine simplifica el proceso de crear y gestionar hosts Docker en varias plataformas. Al entender cómo instalar Docker Machine, crear y gestionar hosts Docker, y configurar el cliente Docker, puedes configurar y usar eficientemente entornos Docker adaptados a tus necesidades. Este conocimiento es esencial tanto para entornos de desarrollo como de producción, haciendo de Docker Machine una herramienta valiosa en tu kit de herramientas Docker.
