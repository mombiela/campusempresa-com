Introducción

Los algoritmos de aproximación son técnicas utilizadas para encontrar soluciones cercanas a la óptima para problemas de optimización que son difíciles o imposibles de resolver exactamente en un tiempo razonable. Estos algoritmos son especialmente útiles en problemas NP-completos, donde encontrar la solución exacta puede ser computacionalmente inviable.

Conceptos Clave

  • Problemas NP-completos: Problemas para los cuales no se conoce un algoritmo que pueda resolverlos en tiempo polinomial.
  • Ratio de Aproximación: Mide cuán cerca está la solución encontrada por el algoritmo de aproximación de la solución óptima.
  • Algoritmos Greedy: Algoritmos que construyen una solución paso a paso, eligiendo la opción que parece la mejor en cada paso.
  • Algoritmos de Programación Dinámica: Algoritmos que resuelven problemas dividiéndolos en subproblemas más pequeños y combinando sus soluciones.

Ejemplos de Algoritmos de Aproximación

Problema del Viajante de Comercio (TSP)

El problema del viajante de comercio es un problema NP-completo donde se busca encontrar el camino más corto que visita una serie de ciudades y regresa al punto de partida.

Algoritmo Greedy de Aproximación

  1. Descripción: Selecciona la ciudad más cercana no visitada en cada paso.
  2. Ratio de Aproximación: No garantiza una solución óptima, pero es fácil de implementar.
def tsp_greedy(graph, start):
    n = len(graph)
    visited = [False] * n
    path = [start]
    visited[start] = True
    current = start

    while len(path) < n:
        next_city = None
        min_dist = float('inf')
        for city in range(n):
            if not visited[city] and graph[current][city] < min_dist:
                next_city = city
                min_dist = graph[current][city]
        path.append(next_city)
        visited[next_city] = True
        current = next_city

    path.append(start)  # Return to the starting point
    return path

# Ejemplo de uso
graph = [
    [0, 10, 15, 20],
    [10, 0, 35, 25],
    [15, 35, 0, 30],
    [20, 25, 30, 0]
]
start = 0
print(tsp_greedy(graph, start))

Problema de la Mochila (Knapsack Problem)

El problema de la mochila es otro problema NP-completo donde se busca maximizar el valor de los objetos que se pueden llevar en una mochila con una capacidad limitada.

Algoritmo de Aproximación por Fracción

  1. Descripción: Permite fraccionar los objetos y selecciona los objetos en orden de valor por unidad de peso.
  2. Ratio de Aproximación: Puede alcanzar una solución óptima en el caso de la versión fraccionaria del problema.
def fractional_knapsack(weights, values, capacity):
    index = list(range(len(values)))
    ratio = [v/w for v, w in zip(values, weights)]
    index.sort(key=lambda i: ratio[i], reverse=True)

    max_value = 0
    fractions = [0] * len(values)
    for i in index:
        if weights[i] <= capacity:
            fractions[i] = 1
            max_value += values[i]
            capacity -= weights[i]
        else:
            fractions[i] = capacity / weights[i]
            max_value += values[i] * fractions[i]
            break

    return max_value, fractions

# Ejemplo de uso
weights = [10, 20, 30]
values = [60, 100, 120]
capacity = 50
print(fractional_knapsack(weights, values, capacity))

Conclusión

Los algoritmos de aproximación son herramientas poderosas para abordar problemas complejos donde las soluciones exactas son impracticables. Aunque no siempre garantizan una solución óptima, proporcionan soluciones suficientemente buenas en un tiempo razonable. Es crucial entender las limitaciones y las garantías de rendimiento de estos algoritmos para aplicarlos adecuadamente en problemas del mundo real.
