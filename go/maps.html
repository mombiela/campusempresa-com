Mapas en Go
Structs ►

Los mapas son un tipo de dato incorporado en Go que asocia claves con valores. Son similares a los diccionarios en Python o las tablas hash en otros lenguajes. Esta sección cubrirá los conceptos básicos de los mapas, cómo usarlos y algunas técnicas avanzadas.

Introducción a los Mapas

  • Los mapas son colecciones de pares clave-valor.
  • Las claves deben ser de un tipo que sea comparable (por ejemplo, cadenas, enteros).
  • Los valores pueden ser de cualquier tipo.
  • Los mapas proporcionan búsquedas, actualizaciones y eliminaciones rápidas.

Creación e Inicialización de Mapas

Puedes crear un mapa usando la función make o un literal de mapa.

// Usando la función make
var m1 map[string]int
m1 = make(map[string]int)

// Usando un literal de mapa
m2 := map[string]int{
    "Alice": 25,
    "Bob":   30,
}

Agregar y Recuperar Elementos

Para agregar o actualizar elementos en un mapa, usa el operador de asignación. Para recuperar un elemento, usa la clave.

m := make(map[string]int)
m["Alice"] = 25
m["Bob"] = 30

// Recuperar valor
age := m["Alice"]
fmt.Println("La edad de Alice:", age) // Salida: La edad de Alice: 25

Comprobación de Existencia

Para comprobar si una clave existe en un mapa, usa la asignación de dos valores.

age, exists := m["Charlie"]
if exists {
    fmt.Println("La edad de Charlie:", age)
} else {
    fmt.Println("Charlie no está en el mapa")
}

Eliminación de Elementos

Para eliminar un elemento de un mapa, usa la función delete.

delete(m, "Bob")

Iteración Sobre Mapas

Puedes iterar sobre un mapa usando un bucle for.

for key, value := range m {
    fmt.Printf("%s: %d\n", key, value)
}

Técnicas Avanzadas de Mapas

Mapas Anidados

Los mapas pueden contener otros mapas como valores, lo que permite estructuras de datos más complejas.

nestedMap := map[string]map[string]int{
    "Grupo1": {
        "Alice": 25,
        "Bob":   30,
    },
    "Grupo2": {
        "Charlie": 35,
        "Dave":    40,
    },
}

Mapas con Estructuras

Los mapas también pueden usar estructuras como valores para datos más estructurados.

type Person struct {
    Age  int
    City string
}

people := map[string]Person{
    "Alice": {25, "Nueva York"},
    "Bob":   {30, "San Francisco"},
}

Consideraciones de Rendimiento

  • Los mapas proporcionan una complejidad de tiempo promedio de O(1) para búsquedas, inserciones y eliminaciones.
  • Sin embargo, los mapas no son seguros para hilos. Usa mecanismos de sincronización como sync.Mutex o sync.RWMutex cuando accedas a mapas desde múltiples goroutines.

Errores Comunes

  • Valor Cero: El valor cero de un mapa es nil. Un mapa nil se comporta como un mapa vacío al leer, pero causa un pánico en tiempo de ejecución al escribir.
  • Tipos de Clave: Solo los tipos que son comparables pueden usarse como claves. Por ejemplo, las slices no pueden usarse como claves.

Conclusión

Los mapas son una estructura de datos poderosa y flexible en Go, que proporciona un almacenamiento y recuperación de clave-valor eficientes. Entender cómo crear, manipular e iterar sobre mapas es esencial para una programación efectiva en Go. Al dominar los mapas, puedes manejar una amplia gama de tareas de almacenamiento y recuperación de datos de manera eficiente.
