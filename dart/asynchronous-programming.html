Programación Asíncrona en Dart
Streams ►

La programación asíncrona es un concepto clave en el desarrollo de software moderno, que permite a las aplicaciones manejar múltiples tareas simultáneamente sin bloquear el hilo principal de ejecución. En Dart, la programación asíncrona se maneja principalmente usando Future, async y await.

Introducción a la Programación Asíncrona

  • Sincrónico vs Asíncrono:

    • Sincrónico: Las tareas se ejecutan una tras otra. Cada tarea espera a que la anterior se complete.
    • Asíncrono: Las tareas pueden ejecutarse concurrentemente, permitiendo que el programa continúe ejecutando otras tareas mientras espera que algunas operaciones se completen.

  • ¿Por qué Programación Asíncrona?:

    • Mejora el rendimiento al no bloquear el hilo principal.
    • Esencial para tareas como solicitudes de red, entrada/salida de archivos y temporizadores.

Futuros en Dart

Un Future representa un valor potencial o un error que estará disponible en algún momento en el futuro.

  • Creando un Future:

    Future<String> fetchUserOrder() {
  // Imagina que esta función está obteniendo el pedido del usuario de una base de datos o una API
  return Future.delayed(Duration(seconds: 2), () => 'Large Latte');
}

  • Manejando un Future:

    void main() {
  fetchUserOrder().then((order) {
    print('Tu pedido es: $order');
  }).catchError((error) {
    print('Algo salió mal: $error');
  });
}

Async y Await

Las palabras clave async y await hacen que el código asíncrono se vea más como código sincrónico, mejorando la legibilidad.

  • Usando async y await:

    Future<void> printOrderMessage() async {
  var order = await fetchUserOrder();
  print('Tu pedido es: $order');
}

void main() {
  printOrderMessage();
}

  • Manejo de Errores con async y await:

    Future<void> printOrderMessage() async {
  try {
    var order = await fetchUserOrder();
    print('Tu pedido es: $order');
  } catch (error) {
    print('Algo salió mal: $error');
  }
}

void main() {
  printOrderMessage();
}

Streams en Dart

Los Streams proporcionan una forma de manejar una secuencia de eventos asíncronos.

  • Creando un Stream:

    Stream<int> countStream(int max) async* {
  for (int i = 1; i <= max; i++) {
    await Future.delayed(Duration(seconds: 1));
    yield i;
  }
}

  • Escuchando un Stream:

    void main() {
  var stream = countStream(5);
  stream.listen((count) {
    print(count);
  });
}

  • Manejo de Errores en Streams:

    void main() {
  var stream = countStream(5);
  stream.listen(
    (count) {
      print(count);
    },
    onError: (error) {
      print('Error: $error');
    },
    onDone: () {
      print('El Stream ha terminado');
    },
  );
}

Comparación entre Futuros y Streams

| Característica | Future | Stream | |----------------|---------------------------------|--------------------------------| | Valor Único | Devuelve un solo valor o error | Puede emitir múltiples valores o errores a lo largo del tiempo | | Finalización | Se completa con un valor o error | Puede continuar emitiendo valores hasta que termine | | Uso | Ideal para operaciones asíncronas únicas | Ideal para manejar secuencias de eventos asíncronos |

Conclusión

La programación asíncrona en Dart es una herramienta poderosa que permite a los desarrolladores escribir código eficiente y no bloqueante. Al entender y utilizar Future, async, await y Stream, puedes manejar tareas asíncronas de manera efectiva, haciendo que tus aplicaciones sean más receptivas y con mejor rendimiento. Practica con los ejemplos y ejercicios proporcionados para solidificar tu comprensión de estos conceptos.
