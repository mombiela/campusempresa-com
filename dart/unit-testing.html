Pruebas Unitarias en Dart
Pruebas de Widgets en Flutter ►

Las pruebas unitarias son un aspecto crucial del desarrollo de software que asegura que las unidades individuales de código funcionen como se espera. En Dart, las pruebas unitarias son facilitadas por el paquete test, que proporciona un marco para escribir y ejecutar pruebas.

Introducción a las Pruebas Unitarias

  • Definición: Las pruebas unitarias implican probar componentes o funciones individuales de un programa para asegurar que funcionen como se espera.
  • Propósito: Detectar errores temprano, asegurar la fiabilidad del código y facilitar la refactorización.

Configuración de Pruebas Unitarias en Dart

Para comenzar con las pruebas unitarias en Dart, necesitas agregar el paquete test a tu proyecto.

  1. Agregar el paquete test:

    dev_dependencies:
  test: ^1.16.0

  2. Instalar dependencias:

    dart pub get

Escribiendo Tu Primera Prueba

Vamos a escribir una prueba simple para entender los conceptos básicos.

  1. Crear un archivo Dart para las pruebas:

    touch test/example_test.dart

  2. Escribir una prueba simple:

    // test/example_test.dart
import 'package:test/test.dart';

void main() {
  test('String split', () {
    var string = 'Hello, world!';
    var result = string.split(',');

    expect(result, equals(['Hello', ' world!']));
  });
}

Explicación:

  • Importando el paquete test: Esto es necesario para usar las funciones de prueba.
  • Definiendo una prueba: La función test toma una descripción y una función de callback que contiene la lógica de la prueba.
  • Aserciones: La función expect verifica si el resultado actual coincide con el resultado esperado.

Agrupando Pruebas

Agrupar pruebas relacionadas puede hacer que tu suite de pruebas sea más organizada y legible.

  1. Usando la función group: 
    // test/example_test.dart
import 'package:test/test.dart';

void main() {
  group('String tests', () {
    test('String split', () {
      var string = 'Hello, world!';
      var result = string.split(',');

      expect(result, equals(['Hello', ' world!']));
    });

    test('String trim', () {
      var string = '  Hello  ';
      var result = string.trim();

      expect(result, equals('Hello'));
    });
  });
}

Explicación:

  • Agrupando: La función group ayuda a organizar las pruebas lógicamente.
  • Múltiples pruebas: Puedes agregar múltiples pruebas dentro de un grupo para una mejor estructura.

Ejecutando Pruebas

Para ejecutar tus pruebas, usa el siguiente comando:

dart test

Técnicas Avanzadas de Pruebas

Mocking

El mocking se usa para simular el comportamiento de objetos complejos.

  1. Agregar el paquete mockito:

    dev_dependencies:
  mockito: ^5.0.0

  2. Crear una clase mock:

    // lib/counter.dart
class Counter {
  int value = 0;

  void increment() {
    value++;
  }
}

  3. Escribir una prueba con un mock:

    // test/counter_test.dart
import 'package:test/test.dart';
import 'package:mockito/mockito.dart';
import 'package:your_package/counter.dart';

class MockCounter extends Mock implements Counter {}

void main() {
  test('Counter increment', () {
    var counter = MockCounter();

    when(counter.increment()).thenReturn(null);
    counter.increment();

    verify(counter.increment()).called(1);
  });
}

Explicación:

  • Clase mock: MockCounter extiende Mock e implementa Counter.
  • Comportamiento mock: Las funciones when y verify se usan para definir y verificar el comportamiento del mock.

Conclusión

Las pruebas unitarias en Dart son una forma poderosa de asegurar que tu código funcione correctamente. Comenzando con pruebas simples y gradualmente incorporando técnicas más avanzadas como agrupación y mocking, puedes construir una suite de pruebas robusta que ayude a mantener la calidad y fiabilidad del código.

  • Puntos Clave:
    • Las pruebas unitarias ayudan a detectar errores temprano y aseguran la fiabilidad del código.
    • El paquete test proporciona un marco para escribir y ejecutar pruebas.
    • Agrupar pruebas y usar mocking puede mejorar tu estrategia de pruebas.

Siguiendo estas pautas, puedes implementar efectivamente pruebas unitarias en tus proyectos de Dart y asegurar que tu código se mantenga mantenible y libre de errores.
