Introducción a `$lookup`

MongoDB es una base de datos NoSQL que no soporta joins tradicionales de SQL. Sin embargo, MongoDB proporciona la etapa de agregación $lookup para realizar operaciones de join entre colecciones.

  • Propósito: $lookup se usa para unir documentos de diferentes colecciones.
  • Sintaxis: 
    {
  $lookup: {
    from: <colección a unir>,
    localField: <campo de los documentos de entrada>,
    foreignField: <campo de los documentos de la colección "from">,
    as: <campo de salida del array>
  }
}

Uso Básico de `$lookup`

Comencemos con un ejemplo simple para entender cómo funciona $lookup.

Escenario de Ejemplo

Tenemos dos colecciones: orders y customers.

Colección orders

[
  { "_id": 1, "product": "Laptop", "customerId": 101 },
  { "_id": 2, "product": "Phone", "customerId": 102 },
  { "_id": 3, "product": "Tablet", "customerId": 101 }
]

Colección customers

[
  { "_id": 101, "name": "John Doe", "email": "john@example.com" },
  { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
]

Realizando un Join

Queremos unir la colección orders con la colección customers para incluir detalles del cliente en cada pedido.

Pipeline de Agregación

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  }
])

Resultado

[
  {
    "_id": 1,
    "product": "Laptop",
    "customerId": 101,
    "customerDetails": [
      { "_id": 101, "name": "John Doe", "email": "john@example.com" }
    ]
  },
  {
    "_id": 2,
    "product": "Phone",
    "customerId": 102,
    "customerDetails": [
      { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
    ]
  },
  {
    "_id": 3,
    "product": "Tablet",
    "customerId": 101,
    "customerDetails": [
      { "_id": 101, "name": "John Doe", "email": "john@example.com" }
    ]
  }
]

Uso Avanzado de `$lookup`

Desenrollando el Array Unido

Por defecto, $lookup crea un array de documentos coincidentes. Para aplanar este array, usamos la etapa $unwind.

Ejemplo

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  },
  { $unwind: "$customerDetails" }
])

Resultado

[
  {
    "_id": 1,
    "product": "Laptop",
    "customerId": 101,
    "customerDetails": { "_id": 101, "name": "John Doe", "email": "john@example.com" }
  },
  {
    "_id": 2,
    "product": "Phone",
    "customerId": 102,
    "customerDetails": { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
  },
  {
    "_id": 3,
    "product": "Tablet",
    "customerId": 101,
    "customerDetails": { "_id": 101, "name": "John Doe", "email": "john@example.com" }
  }
]

Manejo de Documentos No Coincidentes

Para incluir documentos que no tienen una coincidencia en la colección unida, use la opción preserveNullAndEmptyArrays con $unwind.

Ejemplo

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  },
  { $unwind: { path: "$customerDetails", preserveNullAndEmptyArrays: true } }
])

Resultado

Si hubiera pedidos con customerId no presentes en la colección customers, esos pedidos aún aparecerían en el resultado con customerDetails establecido en null.

Resumen

  • Join Básico: Use $lookup para unir documentos de diferentes colecciones.
  • Aplanando Resultados: Use $unwind para aplanar el array creado por $lookup.
  • Manejo de Documentos No Coincidentes: Use preserveNullAndEmptyArrays con $unwind para incluir documentos sin coincidencias.

Al dominar $lookup, puede realizar eficazmente operaciones de join en MongoDB, permitiéndole combinar datos de múltiples colecciones de una manera flexible y poderosa.
