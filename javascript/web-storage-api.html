API de Almacenamiento Web
API Fetch ►

La API de Almacenamiento Web proporciona mecanismos mediante los cuales los navegadores pueden almacenar pares clave/valor, de una manera mucho más intuitiva que usando cookies. Esta API es esencial para construir aplicaciones web modernas que requieren almacenamiento de datos persistente en el lado del cliente.

Conceptos Clave

  • Almacenamiento Local: Almacena datos sin fecha de expiración. Los datos no se eliminan cuando se cierra el navegador y pueden ser accedidos posteriormente.
  • Almacenamiento de Sesión: Almacena datos durante la duración de la sesión de la página. Los datos se eliminan cuando la sesión de la página termina (es decir, cuando se cierra el navegador).

Almacenamiento Local

El Almacenamiento Local está diseñado para almacenar datos que necesitan persistir incluso después de que el usuario cierre el navegador. Es ideal para almacenar preferencias del usuario, configuraciones y otros datos que deben estar disponibles a través de sesiones.

Operaciones Básicas

Establecer Datos

Para almacenar datos en el Almacenamiento Local, usa el método setItem.

localStorage.setItem('username', 'JohnDoe');

Obtener Datos

Para recuperar datos del Almacenamiento Local, usa el método getItem.

let username = localStorage.getItem('username');
console.log(username); // Salida: JohnDoe

Eliminar Datos

Para eliminar un ítem específico del Almacenamiento Local, usa el método removeItem.

localStorage.removeItem('username');

Limpiar Todos los Datos

Para limpiar todos los datos del Almacenamiento Local, usa el método clear.

localStorage.clear();

Ejemplo: Almacenando Preferencias del Usuario

// Almacenando preferencias del usuario
localStorage.setItem('theme', 'dark');
localStorage.setItem('fontSize', '16px');

// Recuperando preferencias del usuario
let theme = localStorage.getItem('theme');
let fontSize = localStorage.getItem('fontSize');

console.log(`Theme: ${theme}, Font Size: ${fontSize}`); // Salida: Theme: dark, Font Size: 16px

Almacenamiento de Sesión

El Almacenamiento de Sesión está diseñado para almacenar datos que solo deben estar disponibles durante la duración de la sesión de la página. Es útil para almacenar datos temporales que no necesitan persistir a través de sesiones.

Operaciones Básicas

Establecer Datos

Para almacenar datos en el Almacenamiento de Sesión, usa el método setItem.

sessionStorage.setItem('sessionID', '123456');

Obtener Datos

Para recuperar datos del Almacenamiento de Sesión, usa el método getItem.

let sessionID = sessionStorage.getItem('sessionID');
console.log(sessionID); // Salida: 123456

Eliminar Datos

Para eliminar un ítem específico del Almacenamiento de Sesión, usa el método removeItem.

sessionStorage.removeItem('sessionID');

Limpiar Todos los Datos

Para limpiar todos los datos del Almacenamiento de Sesión, usa el método clear.

sessionStorage.clear();

Ejemplo: Almacenando Datos Temporales

// Almacenando datos temporales
sessionStorage.setItem('currentPage', 'home');
sessionStorage.setItem('cartItems', JSON.stringify(['item1', 'item2', 'item3']));

// Recuperando datos temporales
let currentPage = sessionStorage.getItem('currentPage');
let cartItems = JSON.parse(sessionStorage.getItem('cartItems'));

console.log(`Current Page: ${currentPage}, Cart Items: ${cartItems}`); // Salida: Current Page: home, Cart Items: item1,item2,item3

Comparación entre Almacenamiento Local y Almacenamiento de Sesión

| Característica | Almacenamiento Local | Almacenamiento de Sesión | |---------------------|---------------------------------------|---------------------------------------| | Persistencia de Datos | Persistente a través de sesiones | Solo durante la duración de la sesión | | Límite de Almacenamiento | Típicamente 5-10 MB por origen | Típicamente 5-10 MB por origen | | Casos de Uso | Preferencias del usuario, configuraciones | Datos temporales, información específica de la sesión | | Alcance de Acceso | Cualquier ventana/pestaña del mismo origen | Solo la ventana/pestaña que lo creó |

Conclusión

La API de Almacenamiento Web es una herramienta poderosa para el almacenamiento de datos del lado del cliente en aplicaciones web. Al comprender las diferencias entre el Almacenamiento Local y el Almacenamiento de Sesión, los desarrolladores pueden tomar decisiones informadas sobre dónde almacenar datos según sus requisitos de persistencia. El Almacenamiento Local es ideal para datos que necesitan persistir a través de sesiones, mientras que el Almacenamiento de Sesión es adecuado para datos temporales que deben ser eliminados cuando la sesión termina.
