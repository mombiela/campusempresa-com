Introducción

Las pruebas y la depuración son habilidades cruciales para cualquier desarrollador de JavaScript. Esta sección cubrirá los conceptos básicos de las pruebas y la depuración, progresando desde niveles principiantes hasta avanzados. Al final de esta sección, podrás escribir pruebas para tu código y depurar problemas de manera efectiva.

Pruebas en JavaScript

¿Qué son las Pruebas?

  • Definición: Las pruebas son el proceso de evaluar un sistema o sus componentes para determinar si satisfacen los requisitos especificados.
  • Propósito: Identificar defectos, asegurar la calidad y verificar que el código funcione como se espera.

Tipos de Pruebas

  • Pruebas Unitarias: Pruebas de unidades individuales o componentes de un software.
  • Pruebas de Integración: Pruebas de la interacción entre unidades o componentes integrados.
  • Pruebas de Extremo a Extremo: Pruebas del flujo completo de una aplicación de principio a fin.

Herramientas para Pruebas

  • Jest: Un marco de pruebas de JavaScript mantenido por Facebook.
  • Mocha: Un marco de pruebas de JavaScript rico en características que se ejecuta en Node.js.
  • Chai: Una biblioteca de aserciones BDD / TDD para Node.js y el navegador.

Escribiendo tu Primera Prueba con Jest

  1. Instalar Jest:

    npm install --save-dev jest

  2. Crear un Archivo de Prueba:

    // sum.js
function sum(a, b) {
    return a + b;
}
module.exports = sum;

    // sum.test.js
const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
    expect(sum(1, 2)).toBe(3);
});

  3. Ejecutar la Prueba:

    npx jest

Explicación

  • sum.js: Una función simple que suma dos números.
  • sum.test.js: Un archivo de prueba que importa la función sum y prueba si sum(1, 2) es igual a 3.
  • test(): Una función proporcionada por Jest para definir un caso de prueba.
  • expect(): Una función de aserción para verificar si el resultado coincide con el valor esperado.

Depuración en JavaScript

¿Qué es la Depuración?

  • Definición: La depuración es el proceso de identificar, analizar y eliminar errores o bugs del software.
  • Propósito: Asegurar que el software funcione sin problemas y correctamente.

Técnicas Comunes de Depuración

  • Registro en Consola: Usar console.log() para imprimir variables y rastrear el flujo del programa.
  • Puntos de Interrupción: Usar puntos de interrupción en un depurador para pausar la ejecución e inspeccionar el estado de la aplicación.
  • Manejo de Errores: Usar bloques try...catch para manejar errores de manera elegante.

Uso del Registro en Consola

  • Ejemplo: 
    function greet(name) {
    console.log('greet function called with:', name);
    return `Hello, ${name}!`;
}

greet('Alice');

Uso de Puntos de Interrupción en Chrome DevTools

  1. Abrir DevTools: Haz clic derecho en la página web y selecciona "Inspeccionar" o presiona Ctrl+Shift+I.
  2. Establecer un Punto de Interrupción: Navega a la pestaña "Sources", encuentra tu archivo JavaScript y haz clic en el número de línea donde deseas establecer un punto de interrupción.
  3. Ejecutar el Código: Actualiza la página o desencadena la función para alcanzar el punto de interrupción e inspeccionar variables.

Manejo de Errores con try...catch

  • Ejemplo: 
    function parseJSON(jsonString) {
    try {
        const result = JSON.parse(jsonString);
        console.log('Parsing successful:', result);
        return result;
    } catch (error) {
        console.error('Parsing error:', error.message);
        return null;
    }
}

parseJSON('{"name": "Alice"}'); // JSON válido
parseJSON('Invalid JSON'); // JSON inválido

Conclusión

Las pruebas y la depuración son prácticas esenciales para mantener un código JavaScript de alta calidad. Al escribir pruebas, puedes asegurarte de que tu código se comporte como se espera, y al depurar de manera efectiva, puedes identificar y solucionar problemas rápidamente. Dominar estas habilidades te convertirá en un desarrollador más competente y confiable.
