Introducción

En esta sección del curso sobre Algoritmos, exploraremos las estrategias de diseño de algoritmos. Estas estrategias son fundamentales para resolver problemas de manera eficiente y efectiva. Aprenderemos sobre diferentes enfoques y técnicas que se pueden aplicar para diseñar algoritmos robustos y optimizados.

Estrategias Comunes de Diseño de Algoritmos

1. Divide y Vencerás

  • Descripción: Esta estrategia implica dividir un problema en subproblemas más pequeños, resolver cada subproblema de manera recursiva y luego combinar las soluciones de los subproblemas para resolver el problema original.
  • Ejemplo Clásico: Algoritmo de ordenación Merge Sort.
def merge_sort(arr):
    if len(arr) > 1:
        mid = len(arr) // 2
        L = arr[:mid]
        R = arr[mid:]

        merge_sort(L)
        merge_sort(R)

        i = j = k = 0

        while i < len(L) and j < len(R):
            if L[i] < R[j]:
                arr[k] = L[i]
                i += 1
            else:
                arr[k] = R[j]
                j += 1
            k += 1

        while i < len(L):
            arr[k] = L[i]
            i += 1
            k += 1

        while j < len(R):
            arr[k] = R[j]
            j += 1
            k += 1

arr = [12, 11, 13, 5, 6, 7]
merge_sort(arr)
print("Sorted array is:", arr)

2. Programación Dinámica

  • Descripción: Esta técnica se utiliza para resolver problemas complejos dividiéndolos en subproblemas más simples y almacenando los resultados de estos subproblemas para evitar cálculos redundantes.
  • Ejemplo Clásico: Problema de la mochila (Knapsack Problem).
def knapsack(W, wt, val, n):
    K = [[0 for x in range(W + 1)] for x in range(n + 1)]

    for i in range(n + 1):
        for w in range(W + 1):
            if i == 0 or w == 0:
                K[i][w] = 0
            elif wt[i-1] <= w:
                K[i][w] = max(val[i-1] + K[i-1][w-wt[i-1]], K[i-1][w])
            else:
                K[i][w] = K[i-1][w]

    return K[n][W]

val = [60, 100, 120]
wt = [10, 20, 30]
W = 50
n = len(val)
print(knapsack(W, wt, val, n))

3. Algoritmos Greedy

  • Descripción: Esta estrategia construye una solución paso a paso, eligiendo en cada paso la opción que parece ser la mejor en ese momento.
  • Ejemplo Clásico: Problema de la moneda (Coin Change Problem).
def coin_change(coins, amount):
    coins.sort(reverse=True)
    count = 0
    for coin in coins:
        while amount >= coin:
            amount -= coin
            count += 1
    return count

coins = [1, 2, 5, 10, 20, 50]
amount = 93
print(coin_change(coins, amount))

4. Backtracking

  • Descripción: Esta técnica implica explorar todas las posibles soluciones y retroceder cuando se detecta que una solución no es válida.
  • Ejemplo Clásico: Problema de las N-Reinas.
def is_safe(board, row, col):
    for i in range(col):
        if board[row][i] == 1:
            return False
    for i, j in zip(range(row, -1, -1), range(col, -1, -1)):
        if board[i][j] == 1:
            return False
    for i, j in zip(range(row, len(board), 1), range(col, -1, -1)):
        if board[i][j] == 1:
            return False
    return True

def solve_n_queens(board, col):
    if col >= len(board):
        return True
    for i in range(len(board)):
        if is_safe(board, i, col):
            board[i][col] = 1
            if solve_n_queens(board, col + 1):
                return True
            board[i][col] = 0
    return False

n = 4
board = [[0] * n for _ in range(n)]
if solve_n_queens(board, 0):
    for row in board:
        print(row)
else:
    print("No solution found")

Comparación de Estrategias

| Estrategia | Ventajas | Desventajas | |----------------------|-----------------------------------------------|----------------------------------------------| | Divide y Vencerás | Eficiente para problemas grandes y complejos | Puede tener sobrecarga de recursión | | Programación Dinámica| Evita cálculos redundantes | Requiere espacio adicional para almacenamiento| | Algoritmos Greedy | Simple y fácil de implementar | No siempre produce la solución óptima | | Backtracking | Encuentra todas las soluciones posibles | Puede ser muy lento para problemas grandes |

Conclusión

Las estrategias de diseño de algoritmos son herramientas poderosas que permiten abordar una amplia variedad de problemas de manera eficiente. Cada estrategia tiene sus propias ventajas y desventajas, y la elección de la estrategia adecuada depende del problema específico que se esté tratando de resolver. A través de la práctica y la experiencia, los profesionales de TI pueden aprender a seleccionar y aplicar la estrategia más adecuada para cada situación.
