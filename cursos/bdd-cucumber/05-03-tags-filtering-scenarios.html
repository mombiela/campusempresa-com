Etiquetas y Filtrado de Escenarios

Siguiente ►
►

En este módulo, aprenderemos cómo utilizar etiquetas en Cucumber para organizar y filtrar escenarios de prueba. Las etiquetas son una herramienta poderosa que permite ejecutar subconjuntos específicos de pruebas, lo cual es especialmente útil en proyectos grandes o cuando se necesita ejecutar pruebas específicas bajo ciertas condiciones.

¿Qué son las Etiquetas en Cucumber?

Las etiquetas en Cucumber son anotaciones que se pueden agregar a los escenarios, características o incluso a los antecedentes para clasificarlos. Estas etiquetas son útiles para:

  • Organizar pruebas: Agrupar escenarios relacionados.
  • Filtrar pruebas: Ejecutar solo un subconjunto de pruebas.
  • Controlar el flujo de ejecución: Ejecutar pruebas bajo ciertas condiciones o entornos.

Sintaxis de las Etiquetas

Las etiquetas se definen usando el símbolo @ seguido del nombre de la etiqueta. Pueden colocarse antes de una característica, escenario o antecedentes.

@regression
Feature: User login

  @smoke
  Scenario: Successful login
    Given the user is on the login page
    When the user enters valid credentials
    Then the user should be redirected to the dashboard

  @regression @slow
  Scenario: Unsuccessful login
    Given the user is on the login page
    When the user enters invalid credentials
    Then an error message should be displayed

Filtrado de Escenarios con Etiquetas

Cucumber permite ejecutar escenarios específicos utilizando etiquetas. Esto se hace mediante la línea de comandos o configuraciones en los archivos de construcción.

Ejecución de Escenarios con Etiquetas

Para ejecutar escenarios con etiquetas específicas, puedes usar el siguiente comando:

cucumber --tags @smoke

Este comando ejecutará solo los escenarios etiquetados con @smoke.

Combinación de Etiquetas

Puedes combinar etiquetas para ejecutar escenarios que cumplan con múltiples condiciones:

  • AND: Ejecutar escenarios que tengan todas las etiquetas especificadas. 
    cucumber --tags @regression --tags @slow
  • OR: Ejecutar escenarios que tengan al menos una de las etiquetas especificadas. 
    cucumber --tags @regression,@smoke
  • NOT: Excluir escenarios con una etiqueta específica. 
    cucumber --tags ~@slow

Ejercicio Práctico

Ejercicio

  1. Crea un archivo de características con al menos tres escenarios.
  2. Asigna diferentes etiquetas a cada escenario.
  3. Ejecuta los escenarios utilizando diferentes combinaciones de etiquetas.

Solución

@feature1
Feature: User management

  @create
  Scenario: Create a new user
    Given the admin is on the user creation page
    When the admin enters valid user details
    Then the user should be created successfully

  @delete
  Scenario: Delete a user
    Given the admin is on the user management page
    When the admin deletes a user
    Then the user should be removed from the system

  @update
  Scenario: Update user information
    Given the admin is on the user management page
    When the admin updates user details
    Then the user information should be updated

  • Ejecuta solo el escenario de creación de usuario:

    cucumber --tags @create

  • Ejecuta escenarios de creación y eliminación de usuario:

    cucumber --tags @create,@delete

  • Ejecuta todos los escenarios excepto el de actualización:

    cucumber --tags ~@update

Conclusión

Las etiquetas en Cucumber son una herramienta esencial para gestionar y ejecutar pruebas de manera eficiente. Permiten una gran flexibilidad al seleccionar qué pruebas ejecutar, lo cual es crucial en entornos de desarrollo y pruebas continuas. Al dominar el uso de etiquetas, puedes optimizar el flujo de trabajo de pruebas y asegurar que las pruebas más relevantes se ejecuten en el momento adecuado. En el siguiente módulo, exploraremos cómo integrar Cucumber con sistemas de integración continua para automatizar aún más el proceso de pruebas.
