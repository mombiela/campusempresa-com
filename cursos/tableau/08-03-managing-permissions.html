Gestionando permisos

Siguiente ►
►

En este tema, aprenderás cómo gestionar permisos en Tableau Server y Tableau Online. La gestión de permisos es crucial para asegurar que los usuarios tengan acceso adecuado a los datos y funcionalidades que necesitan, sin comprometer la seguridad y privacidad de la información.

Objetivos de Aprendizaje

Al final de este tema, deberías ser capaz de:

  1. Comprender los conceptos básicos de permisos en Tableau.
  2. Configurar permisos para usuarios y grupos.
  3. Gestionar permisos a nivel de proyecto, libro de trabajo y fuente de datos.
  4. Resolver conflictos de permisos.

Conceptos Básicos de Permisos

¿Qué son los permisos?

Los permisos en Tableau determinan qué acciones pueden realizar los usuarios en los proyectos, libros de trabajo y fuentes de datos. Los permisos se pueden configurar a nivel de usuario o grupo y se aplican a diferentes niveles de contenido.

Niveles de Permisos

  1. Proyecto: Controla el acceso a todos los contenidos dentro del proyecto.
  2. Libro de trabajo: Controla el acceso a un libro de trabajo específico.
  3. Fuente de datos: Controla el acceso a una fuente de datos específica.

Tipos de Permisos

  • Ver: Permite ver el contenido.
  • Interactuar: Permite interactuar con el contenido (filtrar, resaltar, etc.).
  • Editar: Permite editar el contenido.
  • Eliminar: Permite eliminar el contenido.
  • Publicar: Permite publicar contenido nuevo.
  • Descargar: Permite descargar el contenido.

Configuración de Permisos

Paso 1: Acceder a la Configuración de Permisos

  1. Navega al proyecto, libro de trabajo o fuente de datos en Tableau Server o Tableau Online.
  2. Haz clic en el icono de permisos (generalmente representado por un candado o engranaje).

Paso 2: Añadir Usuarios o Grupos

  1. En la ventana de permisos, haz clic en "Añadir usuario/grupo".
  2. Selecciona el usuario o grupo al que deseas asignar permisos.

Paso 3: Configurar Permisos

  1. Selecciona los permisos que deseas otorgar al usuario o grupo.
  2. Haz clic en "Guardar" para aplicar los cambios.

Ejemplo Práctico

Supongamos que tienes un proyecto llamado "Ventas" y deseas otorgar permisos de visualización a un grupo de usuarios llamado "Equipo de Ventas".

1. Navega al proyecto "Ventas".
2. Haz clic en el icono de permisos.
3. Haz clic en "Añadir usuario/grupo".
4. Selecciona "Equipo de Ventas".
5. Marca la casilla "Ver".
6. Haz clic en "Guardar".

Gestión de Permisos a Diferentes Niveles

Permisos a Nivel de Proyecto

Los permisos a nivel de proyecto se heredan por defecto a todos los contenidos dentro del proyecto. Esto significa que si configuras permisos en un proyecto, esos permisos se aplicarán automáticamente a todos los libros de trabajo y fuentes de datos dentro de ese proyecto.

Permisos a Nivel de Libro de Trabajo y Fuente de Datos

Puedes sobrescribir los permisos heredados configurando permisos específicos para libros de trabajo o fuentes de datos individuales.

Ejemplo de Sobrescritura de Permisos

Si deseas que un usuario específico tenga permisos de edición en un libro de trabajo dentro del proyecto "Ventas", pero no en otros contenidos del proyecto:

1. Navega al libro de trabajo específico.
2. Haz clic en el icono de permisos.
3. Añade el usuario específico.
4. Marca la casilla "Editar".
5. Haz clic en "Guardar".

Resolución de Conflictos de Permisos

Principios de Resolución

  1. Permisos Explícitos: Los permisos configurados explícitamente para un usuario tienen prioridad sobre los permisos heredados.
  2. Permisos de Grupo: Si un usuario pertenece a varios grupos con diferentes permisos, se aplicará el permiso más permisivo.

Ejemplo de Conflicto

Si un usuario pertenece a dos grupos, uno con permiso de "Ver" y otro con permiso de "Editar", el usuario tendrá permiso de "Editar" debido a que es el permiso más permisivo.

Ejercicio Práctico

Ejercicio 1: Configuración de Permisos

  1. Crea un proyecto llamado "Marketing".
  2. Añade un grupo llamado "Equipo de Marketing".
  3. Configura permisos para que el "Equipo de Marketing" pueda ver e interactuar con todos los contenidos dentro del proyecto "Marketing".

Solución

1. Crea el proyecto "Marketing".
2. Navega al proyecto "Marketing".
3. Haz clic en el icono de permisos.
4. Haz clic en "Añadir usuario/grupo".
5. Selecciona "Equipo de Marketing".
6. Marca las casillas "Ver" e "Interactuar".
7. Haz clic en "Guardar".

Conclusión

En esta sección, has aprendido cómo gestionar permisos en Tableau Server y Tableau Online. La correcta configuración de permisos es esencial para asegurar que los usuarios tengan acceso adecuado a los datos y funcionalidades que necesitan, sin comprometer la seguridad y privacidad de la información. Ahora estás preparado para gestionar permisos a diferentes niveles y resolver conflictos de permisos de manera efectiva.
