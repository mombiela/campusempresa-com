Comandos de Gestión del Sistema

Siguiente ►
►

En este tema, aprenderemos sobre los comandos esenciales de gestión del sistema en OpenVMS. Estos comandos son fundamentales para administrar y mantener un sistema OpenVMS de manera eficiente. Cubriremos los siguientes aspectos:

  1. Gestión de Usuarios y Grupos
  2. Gestión de Recursos del Sistema
  3. Gestión de Servicios y Procesos del Sistema
  4. Monitoreo del Sistema
  5. Mantenimiento del Sistema

  1. Gestión de Usuarios y Grupos

Comandos Básicos

  • ADD USER: Añade un nuevo usuario al sistema.
  • MODIFY USER: Modifica las propiedades de un usuario existente.
  • DELETE USER: Elimina un usuario del sistema.

Ejemplo Práctico

$ ADD USER JSMITH /PASSWORD=secret /PRIVILEGES=(NETMBX, TMPMBX)

Explicación:

  • ADD USER JSMITH: Añade un usuario con el nombre JSMITH.
  • /PASSWORD=secret: Establece la contraseña del usuario como secret.
  • /PRIVILEGES=(NETMBX, TMPMBX): Asigna los privilegios NETMBX y TMPMBX al usuario.

Ejercicio

Tarea: Añade un usuario llamado JDoe con la contraseña password123 y los privilegios NETMBX y TMPMBX.

Solución:

$ ADD USER JDoe /PASSWORD=password123 /PRIVILEGES=(NETMBX, TMPMBX)

  1. Gestión de Recursos del Sistema

Comandos Básicos

  • SHOW MEMORY: Muestra el estado de la memoria del sistema.
  • SHOW DEVICE: Muestra información sobre los dispositivos del sistema.
  • SHOW QUOTA: Muestra las cuotas de recursos asignadas a los usuarios.

Ejemplo Práctico

$ SHOW MEMORY

Explicación:

  • SHOW MEMORY: Muestra el estado actual de la memoria del sistema, incluyendo la memoria libre y utilizada.

Ejercicio

Tarea: Muestra la información sobre los dispositivos del sistema.

Solución:

$ SHOW DEVICE

  1. Gestión de Servicios y Procesos del Sistema

Comandos Básicos

  • START/STOP SERVICE: Inicia o detiene un servicio del sistema.
  • SHOW PROCESS: Muestra información sobre los procesos en ejecución.
  • KILL PROCESS: Termina un proceso específico.

Ejemplo Práctico

$ SHOW PROCESS /ID=12345

Explicación:

  • SHOW PROCESS /ID=12345: Muestra información detallada sobre el proceso con el ID 12345.

Ejercicio

Tarea: Termina el proceso con el ID 67890.

Solución:

$ KILL PROCESS /ID=67890

  1. Monitoreo del Sistema

Comandos Básicos

  • MONITOR SYSTEM: Monitorea el rendimiento del sistema en tiempo real.
  • SHOW SYSTEM: Muestra el estado general del sistema.

Ejemplo Práctico

$ MONITOR SYSTEM

Explicación:

  • MONITOR SYSTEM: Inicia el monitoreo en tiempo real del rendimiento del sistema, mostrando métricas como el uso de CPU, memoria y disco.

Ejercicio

Tarea: Muestra el estado general del sistema.

Solución:

$ SHOW SYSTEM

  1. Mantenimiento del Sistema

Comandos Básicos

  • BACKUP: Realiza copias de seguridad de los datos del sistema.
  • RESTORE: Restaura datos desde una copia de seguridad.
  • UPDATE: Aplica actualizaciones al sistema.

Ejemplo Práctico

$ BACKUP /IMAGE DKA0: [000000] MKA500:BACKUP.SAV /SAVE_SET

Explicación:

  • BACKUP /IMAGE DKA0: [000000] MKA500:BACKUP.SAV /SAVE_SET: Realiza una copia de seguridad de la unidad DKA0: y la guarda en MKA500:BACKUP.SAV.

Ejercicio

Tarea: Restaura datos desde la copia de seguridad MKA500:BACKUP.SAV a la unidad DKA0:.

Solución:

$ RESTORE MKA500:BACKUP.SAV DKA0: /IMAGE

Conclusión

En esta sección, hemos cubierto los comandos esenciales para la gestión del sistema en OpenVMS. Estos comandos te permitirán administrar usuarios y grupos, gestionar recursos del sistema, controlar servicios y procesos, monitorear el rendimiento del sistema y realizar tareas de mantenimiento. Asegúrate de practicar estos comandos para familiarizarte con su uso y mejorar tus habilidades en la gestión de sistemas OpenVMS.
