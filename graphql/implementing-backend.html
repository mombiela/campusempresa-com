En esta sección, profundizaremos en el proceso de implementación del backend para una API GraphQL. Esto incluye configurar el servidor, definir el esquema y crear resolutores. Al final de esta sección, tendrás una comprensión sólida de cómo construir un backend funcional de GraphQL.

Configurando el Servidor

Para empezar, necesitamos configurar un servidor que pueda manejar solicitudes GraphQL. Usaremos Node.js con el framework Express y la biblioteca Apollo Server.

Instalando Dependencias

Primero, instalemos los paquetes necesarios:

npm init -y
npm install express apollo-server-express graphql

Creando el Servidor

A continuación, crea un archivo llamado server.js y agrega el siguiente código para configurar el servidor:

const express = require('express');
const { ApolloServer, gql } = require('apollo-server-express');

const app = express();

// Definir un esquema simple
const typeDefs = gql`
  type Query {
    hello: String
  }
`;

// Definir resolutores
const resolvers = {
  Query: {
    hello: () => '¡Hola, mundo!',
  },
};

// Crear una instancia de Apollo Server
const server = new ApolloServer({ typeDefs, resolvers });

// Aplicar el middleware de Apollo GraphQL y establecer la ruta a /graphql
server.applyMiddleware({ app, path: '/graphql' });

// Iniciar el servidor
app.listen({ port: 4000 }, () =>
  console.log(`Servidor listo en http://localhost:4000${server.graphqlPath}`)
);

Explicación

  • Dependencias: Importamos express, ApolloServer y gql de sus respectivos paquetes.
  • Definición del Esquema: Definimos un esquema simple con un solo tipo Query que devuelve una cadena.
  • Resolutores: Creamos un resolutor para la consulta hello que devuelve una cadena estática.
  • Apollo Server: Creamos una instancia de ApolloServer con el esquema y los resolutores, luego lo aplicamos como middleware a la aplicación Express.
  • Inicio del Servidor: Finalmente, iniciamos el servidor en el puerto 4000.

Definiendo el Esquema

El esquema es una parte crucial de cualquier API GraphQL. Define los tipos, consultas y mutaciones que los clientes pueden ejecutar.

Esquema Básico

Ampliemos nuestro esquema para incluir tipos y relaciones más complejas. Actualiza los typeDefs en server.js:

const typeDefs = gql`
  type Book {
    title: String
    author: Author
  }

  type Author {
    name: String
    books: [Book]
  }

  type Query {
    books: [Book]
    authors: [Author]
  }
`;

Explicación

  • Tipos: Definimos los tipos Book y Author. Cada Book tiene un title y un author, y cada Author tiene un name y una lista de books.
  • Consultas: Añadimos consultas para obtener listas de libros y autores.

Creando Resolutores

Los resolutores proporcionan la lógica para obtener los datos definidos en el esquema.

Resolutores Básicos

Actualiza los resolvers en server.js:

const books = [
  { title: 'The Awakening', author: 'Kate Chopin' },
  { title: 'City of Glass', author: 'Paul Auster' },
];

const authors = [
  { name: 'Kate Chopin', books: ['The Awakening'] },
  { name: 'Paul Auster', books: ['City of Glass'] },
];

const resolvers = {
  Query: {
    books: () => books,
    authors: () => authors,
  },
  Book: {
    author: (book) => authors.find((author) => author.name === book.author),
  },
  Author: {
    books: (author) => books.filter((book) => book.author === author.name),
  },
};

Explicación

  • Datos Estáticos: Usamos arreglos estáticos para libros y autores por simplicidad.
  • Resolutores de Consultas: Proporcionamos resolutores para las consultas books y authors que devuelven los datos estáticos.
  • Resolutores de Campos: Añadimos resolutores de campos para Book.author y Author.books para resolver las relaciones entre libros y autores.

Conclusión

En esta sección, cubrimos los conceptos básicos de la configuración de un backend GraphQL usando Node.js, Express y Apollo Server. Definimos un esquema simple y creamos resolutores para obtener datos. Este conocimiento fundamental te permitirá construir APIs GraphQL más complejas a medida que avances en el curso.

Al entender cómo configurar el servidor, definir el esquema y crear resolutores, ahora estás equipado para manejar los componentes principales de un backend GraphQL. En las siguientes secciones, exploraremos temas más avanzados como la autenticación, el manejo de errores y la optimización del rendimiento.
