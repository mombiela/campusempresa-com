Unión de Esquemas en GraphQL
Agrupación y Caché ►

La unión de esquemas es una técnica avanzada de GraphQL que te permite combinar múltiples esquemas de GraphQL en un único esquema unificado. Esto es particularmente útil para arquitecturas de microservicios o al integrar APIs de terceros.

¿Qué es la Unión de Esquemas?

La unión de esquemas implica fusionar múltiples esquemas de GraphQL en uno solo. Esto se puede hacer combinando las definiciones de tipos y los resolutores de diferentes esquemas, permitiéndote consultar datos de múltiples fuentes en una sola solicitud.

Conceptos Clave

  • Esquemas: Esquemas individuales de GraphQL que definen tipos, consultas, mutaciones y suscripciones.
  • Resolutores: Funciones que resuelven los valores para los campos en tu esquema.
  • Unión: El proceso de fusionar múltiples esquemas en uno.

¿Por Qué Usar la Unión de Esquemas?

  • Modularidad: Divide tu API de GraphQL en piezas más pequeñas y manejables.
  • Reutilización: Reutiliza esquemas y resolutores existentes en diferentes proyectos.
  • Escalabilidad: Escala tu aplicación distribuyendo la carga entre múltiples servicios.
  • Integración: Integra fácilmente APIs de terceros en tu esquema de GraphQL.

Ejemplo Básico

Comencemos con un ejemplo simple donde tenemos dos esquemas de GraphQL separados: uno para usuarios y otro para publicaciones.

Esquema de Usuario

# userSchema.graphql
type User {
  id: ID!
  name: String!
}

type Query {
  user(id: ID!): User
}

Esquema de Publicación

# postSchema.graphql
type Post {
  id: ID!
  title: String!
  authorId: ID!
}

type Query {
  post(id: ID!): Post
}

Fusionando Esquemas

Para unir estos esquemas, podemos usar una biblioteca como graphql-tools en JavaScript.

// index.js
const { makeExecutableSchema, mergeSchemas } = require('graphql-tools');
const { ApolloServer } = require('apollo-server');

// Definir definiciones de tipos y resolutores para el esquema de usuario
const userTypeDefs = `
  type User {
    id: ID!
    name: String!
  }

  type Query {
    user(id: ID!): User
  }
`;

const userResolvers = {
  Query: {
    user: (_, { id }) => ({ id, name: `User ${id}` }),
  },
};

// Definir definiciones de tipos y resolutores para el esquema de publicación
const postTypeDefs = `
  type Post {
    id: ID!
    title: String!
    authorId: ID!
  }

  type Query {
    post(id: ID!): Post
  }
`;

const postResolvers = {
  Query: {
    post: (_, { id }) => ({ id, title: `Post ${id}`, authorId: `User ${id}` }),
  },
};

// Crear esquemas ejecutables
const userSchema = makeExecutableSchema({ typeDefs: userTypeDefs, resolvers: userResolvers });
const postSchema = makeExecutableSchema({ typeDefs: postTypeDefs, resolvers: postResolvers });

// Fusionar esquemas
const schema = mergeSchemas({
  schemas: [userSchema, postSchema],
});

// Crear Apollo Server
const server = new ApolloServer({ schema });

server.listen().then(({ url }) => {
  console.log(`Servidor listo en ${url}`);
});

Unión de Esquemas Avanzada

Añadiendo Relaciones

Para hacer el esquema unido más útil, puedes añadir relaciones entre tipos en diferentes esquemas. Por ejemplo, vinculando publicaciones a sus autores.

# Extender el tipo User para incluir publicaciones
extend type User {
  posts: [Post]
}

# Extender el tipo Post para incluir autor
extend type Post {
  author: User
}

Resolutores para Relaciones

Necesitas añadir resolutores para manejar estas relaciones.

const userResolvers = {
  Query: {
    user: (_, { id }) => ({ id, name: `User ${id}` }),
  },
  User: {
    posts: (user) => [{ id: 1, title: 'Post 1', authorId: user.id }],
  },
};

const postResolvers = {
  Query: {
    post: (_, { id }) => ({ id, title: `Post ${id}`, authorId: `User ${id}` }),
  },
  Post: {
    author: (post) => ({ id: post.authorId, name: `User ${post.authorId}` }),
  },
};

Resumen

La unión de esquemas es una técnica poderosa en GraphQL que te permite:

  • Combinar múltiples esquemas en un único esquema unificado.
  • Mejorar la modularidad, reutilización y escalabilidad de tus APIs de GraphQL.
  • Integrar APIs de terceros sin problemas.

Al entender e implementar la unión de esquemas, puedes construir arquitecturas de GraphQL más flexibles y mantenibles, especialmente en sistemas complejos o distribuidos.
