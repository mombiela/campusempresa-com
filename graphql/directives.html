Directivas en GraphQL
Composición de Esquemas ►

Las directivas son una característica poderosa en GraphQL que te permiten alterar dinámicamente la ejecución de tus consultas y esquemas. Proporcionan una forma de personalizar el comportamiento de tu servidor GraphQL y las interacciones del cliente. En esta sección, exploraremos qué son las directivas, cómo usarlas y algunos casos de uso comunes.

¿Qué son las Directivas?

Las directivas son anotaciones especiales en GraphQL que pueden adjuntarse a campos, fragmentos u operaciones para modificar su comportamiento. Están precedidas por un símbolo @ y pueden usarse para incluir o excluir partes de una consulta de manera condicional, entre otras cosas.

Conceptos Clave

  • Anotación: Las directivas son anotaciones que pueden añadirse a consultas y esquemas de GraphQL.
  • Ejecución Condicional: Permiten la inclusión o exclusión condicional de campos y fragmentos.
  • Directivas Personalizadas: Puedes definir tus propias directivas para extender la funcionalidad de tu servidor GraphQL.

Directivas Incorporadas

GraphQL viene con dos directivas incorporadas: @include y @skip.

Directiva @include

La directiva @include incluye condicionalmente un campo o fragmento solo si el argumento especificado es verdadero.

Sintaxis

directive @include(if: Boolean!) on FIELD | FRAGMENT_SPREAD | INLINE_FRAGMENT

Ejemplo

query GetUser($includeAddress: Boolean!) {
  user(id: "1") {
    name
    email
    address @include(if: $includeAddress) {
      street
      city
      country
    }
  }
}

En este ejemplo, el campo address solo se incluirá en la respuesta si la variable $includeAddress es verdadera.

Directiva @skip

La directiva @skip excluye condicionalmente un campo o fragmento si el argumento especificado es verdadero.

Sintaxis

directive @skip(if: Boolean!) on FIELD | FRAGMENT_SPREAD | INLINE_FRAGMENT

Ejemplo

query GetUser($skipAddress: Boolean!) {
  user(id: "1") {
    name
    email
    address @skip(if: $skipAddress) {
      street
      city
      country
    }
  }
}

En este ejemplo, el campo address se excluirá de la respuesta si la variable $skipAddress es verdadera.

Directivas Personalizadas

Puedes definir tus propias directivas para agregar comportamiento personalizado a tu servidor GraphQL. Las directivas personalizadas se definen en el esquema y pueden usarse para implementar características como autenticación, registro o transformación de datos.

Definiendo una Directiva Personalizada

Para definir una directiva personalizada, necesitas agregarla a tu esquema usando la palabra clave directive.

Ejemplo

directive @auth(requires: Role = ADMIN) on FIELD_DEFINITION

enum Role {
  ADMIN
  USER
}

En este ejemplo, definimos una directiva @auth que toma un argumento requires de tipo Role.

Usando una Directiva Personalizada

Una vez definida, puedes usar la directiva personalizada en tu esquema.

Ejemplo

type Query {
  secretData: String @auth(requires: ADMIN)
}

En este ejemplo, el campo secretData está protegido por la directiva @auth, que requiere que el usuario tenga un rol ADMIN.

Implementando la Lógica de una Directiva Personalizada

Para implementar la lógica de una directiva personalizada, necesitas escribir un resolver de directiva en el código de tu servidor.

Ejemplo (JavaScript con Apollo Server)

const { SchemaDirectiveVisitor } = require('apollo-server');
const { defaultFieldResolver } = require('graphql');

class AuthDirective extends SchemaDirectiveVisitor {
  visitFieldDefinition(field) {
    const { resolve = defaultFieldResolver } = field;
    const { requires } = this.args;

    field.resolve = async function (...args) {
      const context = args[2];
      if (!context.user || !context.user.roles.includes(requires)) {
        throw new Error("Not authorized");
      }
      return resolve.apply(this, args);
    };
  }
}

const typeDefs = `
  directive @auth(requires: Role = ADMIN) on FIELD_DEFINITION

  enum Role {
    ADMIN
    USER
  }

  type Query {
    secretData: String @auth(requires: ADMIN)
  }
`;

const resolvers = {
  Query: {
    secretData: () => "This is secret data",
  },
};

const server = new ApolloServer({
  typeDefs,
  resolvers,
  schemaDirectives: {
    auth: AuthDirective,
  },
  context: ({ req }) => {
    // Mock user context
    return {
      user: {
        roles: ['USER'],
      },
    };
  },
});

server.listen().then(({ url }) => {
  console.log(`🚀 Server ready at ${url}`);
});

En este ejemplo, la clase AuthDirective extiende SchemaDirectiveVisitor y sobrescribe el método visitFieldDefinition para implementar la lógica de autorización.

Conclusión

Las directivas en GraphQL son una herramienta versátil que puede ayudarte a controlar la ejecución de tus consultas y extender la funcionalidad de tu servidor GraphQL. Al entender y utilizar tanto las directivas incorporadas como las personalizadas, puedes crear aplicaciones GraphQL más dinámicas y poderosas. Ya sea que necesites incluir campos condicionalmente, hacer cumplir la autorización o implementar comportamientos personalizados, las directivas proporcionan una solución flexible para satisfacer tus necesidades.
