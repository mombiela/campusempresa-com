Descripción del Proyecto

El proyecto final del curso de Fundamentos de Bases de Datos tiene como objetivo aplicar los conocimientos adquiridos a lo largo del curso en un proyecto práctico. Los estudiantes deberán diseñar, implementar y consultar una base de datos relacional que resuelva un problema específico. Este proyecto permitirá a los estudiantes demostrar su comprensión de los conceptos clave de bases de datos, así como sus habilidades en diseño y consulta de bases de datos.

Requisitos del Proyecto

  • Tema del Proyecto: Los estudiantes deben elegir un tema para su base de datos. Puede ser cualquier tema que les interese, como una base de datos para una biblioteca, un sistema de gestión de inventarios, una red social, etc.
  • Diseño de la Base de Datos:
    • Crear un diagrama entidad-relación (ER) que represente las entidades, relaciones y atributos de la base de datos.
    • Normalizar las tablas hasta la tercera forma normal (3NF).
  • Implementación:
    • Crear las tablas en un sistema de gestión de bases de datos (DBMS) como MySQL, PostgreSQL o SQLite.
    • Insertar datos de prueba en las tablas.
  • Consultas:
    • Escribir y ejecutar consultas SQL para recuperar información de la base de datos.
    • Incluir consultas que utilicen SELECT, JOIN, GROUP BY, HAVING y ORDER BY.

Entregables

  1. Documento de Diseño:
    • Diagrama ER.
    • Descripción de las entidades y relaciones.
    • Justificación de la normalización.
  2. Script SQL:
    • Código para crear las tablas.
    • Código para insertar datos de prueba.
    • Consultas SQL.
  3. Informe Final:
    • Descripción del tema elegido.
    • Explicación del diseño de la base de datos.
    • Resultados de las consultas SQL.
    • Conclusiones y aprendizajes.

Ejemplo de Proyecto

Tema: Sistema de Gestión de Biblioteca

Diagrama ER

Descripción de Entidades y Relaciones

  • Entidad Libro:
    • Atributos: ISBN, Título, Autor, Año de Publicación, Género
  • Entidad Usuario:
    • Atributos: ID Usuario, Nombre, Correo Electrónico, Fecha de Registro
  • Entidad Préstamo:
    • Atributos: ID Préstamo, ID Usuario, ISBN, Fecha de Préstamo, Fecha de Devolución
  • Relaciones:
    • Un Usuario puede tomar prestados múltiples Libros.
    • Un Libro puede ser prestado a múltiples Usuarios en diferentes momentos.

Script SQL

-- Creación de tablas
CREATE TABLE Libro (
    ISBN VARCHAR(13) PRIMARY KEY,
    Titulo VARCHAR(255) NOT NULL,
    Autor VARCHAR(255) NOT NULL,
    Año_Publicacion INT,
    Genero VARCHAR(100)
);

CREATE TABLE Usuario (
    ID_Usuario INT PRIMARY KEY AUTO_INCREMENT,
    Nombre VARCHAR(255) NOT NULL,
    Correo_Electronico VARCHAR(255) UNIQUE NOT NULL,
    Fecha_Registro DATE NOT NULL
);

CREATE TABLE Prestamo (
    ID_Prestamo INT PRIMARY KEY AUTO_INCREMENT,
    ID_Usuario INT,
    ISBN VARCHAR(13),
    Fecha_Prestamo DATE NOT NULL,
    Fecha_Devolucion DATE,
    FOREIGN KEY (ID_Usuario) REFERENCES Usuario(ID_Usuario),
    FOREIGN KEY (ISBN) REFERENCES Libro(ISBN)
);

-- Inserción de datos de prueba
INSERT INTO Libro (ISBN, Titulo, Autor, Año_Publicacion, Genero) VALUES
('978-3-16-148410-0', 'El Quijote', 'Miguel de Cervantes', 1605, 'Novela'),
('978-0-14-044913-6', 'La Odisea', 'Homero', -800, 'Épica');

INSERT INTO Usuario (Nombre, Correo_Electronico, Fecha_Registro) VALUES
('Juan Pérez', 'juan.perez@example.com', '2023-01-15'),
('María López', 'maria.lopez@example.com', '2023-02-20');

INSERT INTO Prestamo (ID_Usuario, ISBN, Fecha_Prestamo, Fecha_Devolucion) VALUES
(1, '978-3-16-148410-0', '2023-03-01', '2023-03-15'),
(2, '978-0-14-044913-6', '2023-03-05', NULL);

-- Consultas SQL
-- 1. Obtener todos los libros prestados actualmente
SELECT L.Titulo, U.Nombre, P.Fecha_Prestamo
FROM Prestamo P
JOIN Libro L ON P.ISBN = L.ISBN
JOIN Usuario U ON P.ID_Usuario = U.ID_Usuario
WHERE P.Fecha_Devolucion IS NULL;

-- 2. Contar el número de préstamos por usuario
SELECT U.Nombre, COUNT(P.ID_Prestamo) AS Numero_Prestamos
FROM Usuario U
JOIN Prestamo P ON U.ID_Usuario = P.ID_Usuario
GROUP BY U.Nombre
ORDER BY Numero_Prestamos DESC;

Conclusión

El proyecto final es una excelente oportunidad para consolidar los conocimientos adquiridos durante el curso de Fundamentos de Bases de Datos. Al completar este proyecto, los estudiantes habrán demostrado su capacidad para diseñar, implementar y consultar una base de datos relacional. Además, habrán ganado experiencia práctica que será valiosa en su carrera profesional en el campo de TI.
