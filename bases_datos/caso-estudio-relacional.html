Introducción

En este caso de estudio, exploraremos cómo diseñar y gestionar una base de datos relacional para una pequeña empresa de comercio electrónico. Este ejercicio te ayudará a comprender los conceptos fundamentales de las bases de datos relacionales y cómo aplicarlos en un escenario del mundo real.

Requisitos del Sistema

Antes de comenzar con el diseño de la base de datos, es crucial entender los requisitos del sistema. La empresa de comercio electrónico necesita:

  • Gestión de Productos: Información sobre los productos que venden, incluyendo nombre, descripción, precio y stock.
  • Gestión de Clientes: Información sobre los clientes, incluyendo nombre, dirección, correo electrónico y número de teléfono.
  • Gestión de Pedidos: Información sobre los pedidos realizados por los clientes, incluyendo fecha del pedido, productos comprados y cantidad.
  • Gestión de Pagos: Información sobre los pagos realizados por los clientes, incluyendo método de pago y estado del pago.

Diseño de la Base de Datos

Entidades y Relaciones

Para satisfacer los requisitos del sistema, identificamos las siguientes entidades y sus relaciones:

  • Producto: Representa los artículos que se venden.
  • Cliente: Representa a los compradores.
  • Pedido: Representa las órdenes de compra realizadas por los clientes.
  • Pago: Representa los pagos realizados por los clientes.

Las relaciones entre estas entidades son las siguientes:

  • Un cliente puede realizar múltiples pedidos.
  • Un pedido puede incluir múltiples productos.
  • Un pedido puede tener un pago asociado.

Esquema de la Base de Datos

A continuación, se presenta el esquema de la base de datos con las tablas y sus relaciones:

| Tabla | Columnas | |----------|--------------------------------------------------------------------------| | Producto | id_producto (PK), nombre, descripción, precio, stock | | Cliente | id_cliente (PK), nombre, direccion, correo_electronico, telefono | | Pedido | id_pedido (PK), id_cliente (FK), fecha_pedido | | Pedido_Producto | id_pedido (FK), id_producto (FK), cantidad | | Pago | id_pago (PK), id_pedido (FK), metodo_pago, estado_pago |

Diagrama Entidad-Relación (ERD)

Un diagrama ERD visualiza las entidades y sus relaciones:

Cliente (1) ---- (N) Pedido
Pedido (1) ---- (N) Pedido_Producto
Producto (1) ---- (N) Pedido_Producto
Pedido (1) ---- (1) Pago

Implementación en SQL

A continuación, se muestra cómo crear las tablas en SQL:

CREATE TABLE Producto (
    id_producto INT PRIMARY KEY,
    nombre VARCHAR(100),
    descripcion TEXT,
    precio DECIMAL(10, 2),
    stock INT
);

CREATE TABLE Cliente (
    id_cliente INT PRIMARY KEY,
    nombre VARCHAR(100),
    direccion TEXT,
    correo_electronico VARCHAR(100),
    telefono VARCHAR(15)
);

CREATE TABLE Pedido (
    id_pedido INT PRIMARY KEY,
    id_cliente INT,
    fecha_pedido DATE,
    FOREIGN KEY (id_cliente) REFERENCES Cliente(id_cliente)
);

CREATE TABLE Pedido_Producto (
    id_pedido INT,
    id_producto INT,
    cantidad INT,
    PRIMARY KEY (id_pedido, id_producto),
    FOREIGN KEY (id_pedido) REFERENCES Pedido(id_pedido),
    FOREIGN KEY (id_producto) REFERENCES Producto(id_producto)
);

CREATE TABLE Pago (
    id_pago INT PRIMARY KEY,
    id_pedido INT,
    metodo_pago VARCHAR(50),
    estado_pago VARCHAR(50),
    FOREIGN KEY (id_pedido) REFERENCES Pedido(id_pedido)
);

Ejemplos de Consultas

Insertar Datos

INSERT INTO Producto (id_producto, nombre, descripcion, precio, stock) VALUES
(1, 'Laptop', 'Laptop de 15 pulgadas', 1200.00, 10),
(2, 'Mouse', 'Mouse inalámbrico', 25.00, 50);

INSERT INTO Cliente (id_cliente, nombre, direccion, correo_electronico, telefono) VALUES
(1, 'Juan Perez', 'Calle Falsa 123', 'juan.perez@example.com', '555-1234');

INSERT INTO Pedido (id_pedido, id_cliente, fecha_pedido) VALUES
(1, 1, '2023-10-01');

INSERT INTO Pedido_Producto (id_pedido, id_producto, cantidad) VALUES
(1, 1, 1),
(1, 2, 2);

INSERT INTO Pago (id_pago, id_pedido, metodo_pago, estado_pago) VALUES
(1, 1, 'Tarjeta de Crédito', 'Completado');

Consultar Datos

-- Obtener todos los productos
SELECT * FROM Producto;

-- Obtener todos los pedidos de un cliente específico
SELECT * FROM Pedido WHERE id_cliente = 1;

-- Obtener detalles de un pedido específico
SELECT p.nombre, pp.cantidad
FROM Pedido_Producto pp
JOIN Producto p ON pp.id_producto = p.id_producto
WHERE pp.id_pedido = 1;

Conclusión

Este caso de estudio ha demostrado cómo diseñar y gestionar una base de datos relacional para una empresa de comercio electrónico. Hemos cubierto desde la identificación de requisitos hasta la implementación en SQL, proporcionando una visión completa del proceso. Practicar con estos conceptos te ayudará a desarrollar una comprensión sólida de las bases de datos relacionales y su aplicación en escenarios del mundo real.
