Introducción a Kubernetes

Kubernetes es una plataforma de orquestación de contenedores de código abierto que automatiza la implementación, el escalado y la gestión de aplicaciones en contenedores. Fue desarrollado originalmente por Google y ahora es mantenido por la Cloud Native Computing Foundation (CNCF).

Conceptos Clave

  • Nodo: Una máquina (física o virtual) en el clúster de Kubernetes.
  • Clúster: Conjunto de nodos gestionados por Kubernetes.
  • Pod: Unidad mínima de despliegue en Kubernetes, que puede contener uno o más contenedores.
  • Servicio: Abstracción que define un conjunto lógico de Pods y una política para acceder a ellos.
  • Namespace: Mecanismo para aislar recursos dentro de un clúster.
  • ConfigMap y Secret: Mecanismos para inyectar configuraciones y datos sensibles en los Pods.

Arquitectura de Kubernetes

Kubernetes sigue una arquitectura maestro-esclavo:

  • Master Node: Gestiona el clúster y coordina las actividades.

    • API Server: Punto de entrada para todas las operaciones de administración del clúster.
    • etcd: Almacén de datos distribuido que guarda la configuración del clúster.
    • Controller Manager: Ejecuta los controladores que regulan el estado del clúster.
    • Scheduler: Asigna Pods a los nodos disponibles.

  • Worker Nodes: Ejecutan las aplicaciones en contenedores.

    • Kubelet: Agente que asegura que los contenedores están corriendo en un Pod.
    • Kube-proxy: Gestiona las reglas de red en los nodos.
    • Container Runtime: Software que ejecuta los contenedores (por ejemplo, Docker).

Despliegue de Aplicaciones en Kubernetes

Creación de un Pod

Un Pod es la unidad básica de ejecución en Kubernetes. A continuación se muestra un ejemplo de un archivo YAML para crear un Pod:

apiVersion: v1
kind: Pod
metadata:
  name: my-pod
spec:
  containers:
  - name: my-container
    image: nginx:latest
    ports:
    - containerPort: 80

Para desplegar este Pod, se utiliza el comando kubectl:

kubectl apply -f my-pod.yaml

Creación de un Deployment

Un Deployment proporciona actualizaciones declarativas para Pods y ReplicaSets. Aquí hay un ejemplo de un archivo YAML para un Deployment:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-container
        image: nginx:latest
        ports:
        - containerPort: 80

Para desplegar este Deployment, se utiliza el comando:

kubectl apply -f my-deployment.yaml

Servicios en Kubernetes

Un Servicio en Kubernetes expone uno o más Pods a través de una dirección IP única. Aquí hay un ejemplo de un archivo YAML para un Servicio:

apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
  type: ClusterIP

Para crear este Servicio, se utiliza el comando:

kubectl apply -f my-service.yaml

Ejercicios Prácticos

  1. Crear un Pod: Escribe un archivo YAML para crear un Pod que ejecute una aplicación simple (por ejemplo, un servidor web Nginx) y despliega el Pod en tu clúster.
  2. Crear un Deployment: Escribe un archivo YAML para un Deployment que despliegue tres réplicas de una aplicación y despliega el Deployment en tu clúster.
  3. Exponer un Servicio: Escribe un archivo YAML para un Servicio que exponga el Deployment creado anteriormente y despliega el Servicio en tu clúster.

Conclusión

Kubernetes es una herramienta poderosa para la orquestación de contenedores, que facilita la gestión de aplicaciones en contenedores a gran escala. Comprender los conceptos básicos y la arquitectura de Kubernetes es esencial para cualquier profesional de DevOps. Con la práctica, podrás desplegar, escalar y gestionar aplicaciones de manera eficiente en un entorno de producción.
