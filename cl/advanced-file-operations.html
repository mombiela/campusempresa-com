Introducción

En esta sección, profundizaremos en las operaciones avanzadas de archivos utilizando el Lenguaje de Control (CL) en IBM i (anteriormente AS/400). Este tema asume que tienes un entendimiento básico de CL y estás familiarizado con las operaciones fundamentales de archivos. Cubriremos técnicas avanzadas como sobrescrituras de archivos, operaciones de áreas de datos y el uso de APIs para la manipulación de archivos.

Sobrescrituras de Archivos

Las sobrescrituras de archivos te permiten cambiar los atributos de un archivo en tiempo de ejecución sin modificar el programa que utiliza el archivo.

Conceptos Clave

  • OVRDBF (Sobrescribir con Archivo de Base de Datos): Redirige las operaciones de archivo a un archivo o miembro diferente.
  • OVRPRTF (Sobrescribir con Archivo de Impresora): Cambia los atributos de un archivo de impresora.
  • DLTOVR (Eliminar Sobrescritura): Elimina la sobrescritura.

Ejemplo: OVRDBF

/* Sobrescribir el archivo MYFILE para usar el miembro MYMEMBER */
OVRDBF FILE(MYFILE) TOFILE(MYLIB/MYFILE) MBR(MYMEMBER)

/* Llamar al programa que usa MYFILE */
CALL PGM(MYPGM)

/* Eliminar la sobrescritura después de la llamada al programa */
DLTOVR FILE(MYFILE)

Explicación:

  • El comando OVRDBF sobrescribe el archivo MYFILE para usar el miembro MYMEMBER en la biblioteca MYLIB.
  • El comando CALL ejecuta el programa MYPGM que ahora usará el archivo sobrescrito.
  • DLTOVR elimina la sobrescritura, restaurando los atributos originales del archivo.

Operaciones de Áreas de Datos

Las áreas de datos se utilizan para almacenar pequeñas cantidades de datos que pueden ser accesibles por múltiples programas.

Conceptos Clave

  • CRTDTAARA (Crear Área de Datos): Crea un área de datos.
  • RTVDTAARA (Recuperar Área de Datos): Recupera el valor de un área de datos.
  • CHGDTAARA (Cambiar Área de Datos): Cambia el valor de un área de datos.
  • DLTDTAARA (Eliminar Área de Datos): Elimina un área de datos.

Ejemplo: Operaciones de Áreas de Datos

/* Crear un área de datos llamada MYDTAARA */
CRTDTAARA DTAARA(MYLIB/MYDTAARA) TYPE(*CHAR) LEN(10) VALUE('INITIAL')

/* Recuperar el valor del área de datos */
RTVDTAARA DTAARA(MYLIB/MYDTAARA) RTNVAR(&MYVAR)

/* Cambiar el valor del área de datos */
CHGDTAARA DTAARA(MYLIB/MYDTAARA) VALUE('NEWVALUE')

/* Eliminar el área de datos */
DLTDTAARA DTAARA(MYLIB/MYDTAARA)

Explicación:

  • CRTDTAARA crea un área de datos MYDTAARA en la biblioteca MYLIB con un valor inicial de 'INITIAL'.
  • RTVDTAARA recupera el valor de MYDTAARA en la variable &MYVAR.
  • CHGDTAARA cambia el valor de MYDTAARA a 'NEWVALUE'.
  • DLTDTAARA elimina el área de datos MYDTAARA.

Uso de APIs para la Manipulación de Archivos

IBM i proporciona varias APIs para la manipulación avanzada de archivos, como leer y escribir registros programáticamente.

Conceptos Clave

  • QDBRTVFD (Recuperar Descripción del Archivo): Recupera información de la descripción del archivo.
  • QDBGETM (Obtener Información del Miembro): Recupera información sobre un miembro de archivo.

Ejemplo: Uso de la API QDBRTVFD

/* Declarar variables */
DCL &FILEDSC *CHAR 5000
DCL &ERRCODE *CHAR 8

/* Llamar a la API QDBRTVFD para recuperar la descripción del archivo */
CALL PGM(QDBRTVFD) PARM('MYLIB' 'MYFILE' &FILEDSC &ERRCODE)

Explicación:

  • Los comandos DCL declaran variables para almacenar la descripción del archivo y el código de error.
  • El comando CALL invoca la API QDBRTVFD para recuperar la descripción del archivo MYFILE en la biblioteca MYLIB.

Conclusión

En esta sección, exploramos operaciones avanzadas de archivos en CL, incluyendo sobrescrituras de archivos, operaciones de áreas de datos y el uso de APIs para la manipulación de archivos. Dominar estas técnicas mejorará tu capacidad para gestionar y manipular archivos de manera efectiva en un entorno IBM i. Practica estos comandos y ejemplos para solidificar tu comprensión y aplicarlos a escenarios del mundo real.
