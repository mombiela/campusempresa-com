Introducción

Crear comandos personalizados en Control Language (CL) te permite extender la funcionalidad de tu sistema IBM i definiendo nuevos comandos que pueden simplificar tareas complejas, automatizar procesos y mejorar la productividad. Este tema te guiará a través del proceso de creación de comandos personalizados desde un nivel principiante hasta un nivel avanzado.

Conceptos Clave

  • Definición de Comando: El proceso de especificar la sintaxis y los parámetros de un nuevo comando.
  • Programa de Procesamiento de Comandos (CPP): El programa que se ejecuta cuando se ejecuta el comando.
  • Archivo de Fuente de Comandos: Un archivo fuente donde se almacena la definición del comando.
  • Definición de Parámetros: Especificar los parámetros que aceptará el comando.

Nivel Principiante

Creación de un Comando Simple

Para crear un comando personalizado simple, sigue estos pasos:

  1. Crear un Archivo Físico de Fuente:

    CRTSRCPF FILE(QGPL/QCMDSRC) RCDLEN(112)

    Este comando crea un archivo físico de fuente llamado QCMDSRC en la biblioteca QGPL con una longitud de registro de 112.

  2. Agregar un Miembro al Archivo de Fuente:

    ADDPFM FILE(QGPL/QCMDSRC) MBR(MYCMD) SRCTYPE(CMD)

    Este comando agrega un miembro llamado MYCMD al archivo QCMDSRC con un tipo de fuente CMD.

  3. Editar la Fuente del Comando: Usa la Utilidad de Entrada de Fuente (SEU) u otro editor para ingresar la definición del comando:

    CMD        PROMPT('Mi Comando Personalizado')
PARM       KWD(MSG) TYPE(*CHAR) LEN(50) PROMPT('Mensaje')

    Esto define un comando llamado MYCMD con un solo parámetro MSG, que es una cadena de 50 caracteres.

  4. Crear el Comando:

    CRTCMD CMD(QGPL/MYCMD) PGM(QGPL/MYPGM) SRCFILE(QGPL/QCMDSRC)

    Este comando crea el comando personalizado MYCMD en la biblioteca QGPL y lo asocia con un programa de procesamiento de comandos (CPP) llamado MYPGM.

Escribir el Programa de Procesamiento de Comandos

Crea un programa CL simple para procesar el comando:

PGM        PARM(&MSG)
DCL        VAR(&MSG) TYPE(*CHAR) LEN(50)
SNDPGMMSG  MSG(&MSG)
ENDPGM

Este programa toma el parámetro MSG y lo envía como un mensaje de programa.

Nivel Intermedio

Agregar Múltiples Parámetros

Mejora tu comando agregando más parámetros:

CMD        PROMPT('Comando Personalizado Mejorado')
PARM       KWD(MSG) TYPE(*CHAR) LEN(50) PROMPT('Mensaje')
PARM       KWD(USER) TYPE(*NAME) PROMPT('Nombre de Usuario')

Este comando ahora acepta un mensaje y un nombre de usuario.

Lógica Condicional en CPP

Modifica el CPP para incluir lógica condicional:

PGM        PARM(&MSG &USER)
DCL        VAR(&MSG) TYPE(*CHAR) LEN(50)
DCL        VAR(&USER) TYPE(*NAME)
IF         COND(&USER *NE '') THEN(DO)
   SNDPGMMSG  MSG('Hola ' *CAT &USER *TCAT ' - ' *CAT &MSG)
ENDDO
ELSE       CMD(SNDPGMMSG MSG(&MSG))
ENDPGM

Este programa envía un mensaje personalizado si se proporciona el parámetro USER.

Nivel Avanzado

Uso de Programas de Salida de Procesamiento de Comandos

Crea un programa de salida de procesamiento de comandos para manejar escenarios más complejos:

CMD        PROMPT('Comando Personalizado Avanzado')
PARM       KWD(MSG) TYPE(*CHAR) LEN(50) PROMPT('Mensaje')
PARM       KWD(USER) TYPE(*NAME) PROMPT('Nombre de Usuario')
PARM       KWD(OPTION) TYPE(*CHAR) LEN(10) VALUES('INFO' 'WARN' 'ERROR') PROMPT('Opción')

Este comando incluye un parámetro OPTION con valores predefinidos.

CPP Avanzado con Programa de Salida

Escribe un CPP avanzado que use un programa de salida:

PGM        PARM(&MSG &USER &OPTION)
DCL        VAR(&MSG) TYPE(*CHAR) LEN(50)
DCL        VAR(&USER) TYPE(*NAME)
DCL        VAR(&OPTION) TYPE(*CHAR) LEN(10)

IF         COND(&OPTION *EQ 'INFO') THEN(DO)
   SNDPGMMSG  MSG('INFO: ' *CAT &MSG)
ENDDO
ELSEIF     COND(&OPTION *EQ 'WARN') THEN(DO)
   SNDPGMMSG  MSG('ADVERTENCIA: ' *CAT &MSG)
ENDDO
ELSEIF     COND(&OPTION *EQ 'ERROR') THEN(DO)
   SNDPGMMSG  MSG('ERROR: ' *CAT &MSG)
ENDDO

IF         COND(&USER *NE '') THEN(DO)
   SNDPGMMSG  MSG('Usuario: ' *CAT &USER)
ENDDO

ENDPGM

Este programa procesa el parámetro OPTION para enviar diferentes tipos de mensajes.

Conclusión

Crear comandos personalizados en CL puede mejorar significativamente la funcionalidad y usabilidad de tu sistema IBM i. Comenzar con comandos simples e incorporar gradualmente lógica y parámetros más complejos te permite adaptar los comandos para satisfacer necesidades específicas. Al dominar estas técnicas, puedes automatizar tareas, imponer estándares y mejorar la eficiencia general del sistema.
