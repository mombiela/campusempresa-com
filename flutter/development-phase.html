Fase de Desarrollo
Fase de Pruebas ►

La fase de desarrollo en Flutter implica escribir, probar y refinar el código para construir una aplicación móvil funcional y eficiente. Esta fase es crucial ya que sienta las bases para el rendimiento, la usabilidad y la experiencia general del usuario de la aplicación. En esta sección, cubriremos los conceptos clave, herramientas y prácticas que son esenciales para desarrollar una aplicación Flutter desde cero.

Conceptos Clave

  • Widgets: Los bloques de construcción de una aplicación Flutter. Todo en Flutter es un widget, desde estructuras de diseño hasta elementos de la interfaz de usuario.
  • Gestión de Estado: Técnicas para gestionar el estado de tu aplicación, asegurando que la interfaz de usuario refleje el estado actual de la aplicación.
  • Navegación y Enrutamiento: Métodos para navegar entre diferentes pantallas o páginas en tu aplicación.
  • Programación Asíncrona: Manejo de operaciones asíncronas usando Future, async y await.
  • Pruebas: Escribir pruebas para asegurar que tu aplicación funcione como se espera.

Widgets

Los widgets son los componentes centrales de una aplicación Flutter. Describen cómo debería verse su vista dada su configuración y estado actuales.

Ejemplo: Estructura Básica de un Widget

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Fase de Desarrollo en Flutter'),
        ),
        body: Center(
          child: Text('¡Hola, Flutter!'),
        ),
      ),
    );
  }
}

En este ejemplo:

  • MyApp es un widget sin estado.
  • MaterialApp es un widget que envuelve varios widgets comúnmente requeridos para aplicaciones de diseño material.
  • Scaffold proporciona un marco para la estructura básica del diseño visual de material design.
  • AppBar y Text son widgets utilizados para crear una interfaz de usuario simple.

Gestión de Estado

Gestionar el estado es crucial para construir aplicaciones interactivas. Flutter proporciona varias formas de gestionar el estado, incluyendo setState, InheritedWidget, Provider y más.

Ejemplo: Usando `setState`

class Counter extends StatefulWidget {
  @override
  _CounterState createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: <Widget>[
        Text('Has presionado el botón esta cantidad de veces:'),
        Text(
          '$_counter',
          style: Theme.of(context).textTheme.headline4,
        ),
        ElevatedButton(
          onPressed: _incrementCounter,
          child: Text('Incrementar'),
        ),
      ],
    );
  }
}

En este ejemplo:

  • Counter es un widget con estado.
  • _CounterState gestiona el estado del widget Counter.
  • _incrementCounter actualiza el estado usando setState.

Navegación y Enrutamiento

Flutter proporciona un sistema de navegación robusto para moverse entre pantallas. El widget Navigator gestiona una pila de objetos de ruta.

Ejemplo: Navegación Básica

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: FirstScreen(),
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Primera Pantalla'),
      ),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(builder: (context) => SecondScreen()),
            );
          },
          child: Text('Ir a la Segunda Pantalla'),
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Segunda Pantalla'),
      ),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            Navigator.pop(context);
          },
          child: Text('Volver a la Primera Pantalla'),
        ),
      ),
    );
  }
}

En este ejemplo:

  • Navigator.push se usa para navegar a la SecondScreen.
  • Navigator.pop se usa para volver a la FirstScreen.

Programación Asíncrona

Flutter utiliza las características asíncronas de Dart para realizar tareas como obtener datos de internet.

Ejemplo: Obtener Datos

import 'package:flutter/material.dart';
import 'dart:async';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: DataScreen(),
    );
  }
}

class DataScreen extends StatefulWidget {
  @override
  _DataScreenState createState() => _DataScreenState();
}

class _DataScreenState extends State<DataScreen> {
  Future<String> fetchData() async {
    await Future.delayed(Duration(seconds: 2));
    return 'Datos cargados';
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Obtención Asíncrona de Datos'),
      ),
      body: Center(
        child: FutureBuilder<String>(
          future: fetchData(),
          builder: (context, snapshot) {
            if (snapshot.connectionState == ConnectionState.waiting) {
              return CircularProgressIndicator();
            } else if (snapshot.hasError) {
              return Text('Error: ${snapshot.error}');
            } else {
              return Text('Resultado: ${snapshot.data}');
            }
          },
        ),
      ),
    );
  }
}

En este ejemplo:

  • fetchData simula una llamada de red.
  • FutureBuilder se usa para construir la interfaz de usuario basada en el estado del futuro.

Pruebas

Las pruebas aseguran que tu aplicación funcione como se espera. Flutter soporta pruebas unitarias, pruebas de widgets y pruebas de integración.

Ejemplo: Prueba Unitaria Simple

import 'package:flutter_test/flutter_test.dart';

void main() {
  test('Prueba de incremento del contador', () {
    int counter = 0;

    counter++;

    expect(counter, 1);
  });
}

En este ejemplo:

  • Una prueba unitaria simple verifica si el contador incrementa correctamente.

Conclusión

La fase de desarrollo en Flutter implica entender y utilizar widgets, gestionar el estado, navegar entre pantallas, manejar operaciones asíncronas y escribir pruebas. Dominar estos conceptos te permitirá construir aplicaciones Flutter robustas y eficientes. A medida que avances, descubrirás que estas habilidades fundamentales son cruciales para abordar temas más avanzados y crear aplicaciones complejas.
