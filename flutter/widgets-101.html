Introducción a los Widgets

Los widgets son los bloques de construcción de una aplicación Flutter. Describen cómo debería verse su vista dada su configuración y estado actuales. Los widgets son inmutables y se utilizan para construir la interfaz de usuario.

Tipos de Widgets

Flutter proporciona una amplia gama de widgets, pero se pueden categorizar en dos tipos:

  • Widgets Sin Estado: Estos widgets no tienen ningún estado interno. Son inmutables y se reconstruyen cuando su configuración cambia.
  • Widgets Con Estado: Estos widgets mantienen un estado interno que puede cambiar con el tiempo. Son mutables y pueden reconstruirse a sí mismos cuando su estado cambia.

Creando un Widget Sin Estado

Un widget sin estado es un widget que no requiere un estado mutable. Aquí hay un ejemplo de un widget sin estado simple:

import 'package:flutter/material.dart';

class MyStatelessWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Ejemplo de Widget Sin Estado'),
      ),
      body: Center(
        child: Text('¡Hola, Flutter!'),
      ),
    );
  }
}

void main() => runApp(MaterialApp(
  home: MyStatelessWidget(),
));

Explicación

  • MyStatelessWidget extiende StatelessWidget.
  • El método build describe la parte de la interfaz de usuario representada por el widget.
  • Scaffold proporciona un marco para la estructura básica del diseño visual de material design.
  • AppBar es una barra de aplicaciones de material design.
  • Center centra su widget hijo.
  • Text muestra una cadena de texto.

Creando un Widget Con Estado

Un widget con estado es un widget que tiene un estado mutable. Aquí hay un ejemplo de un widget con estado simple:

import 'package:flutter/material.dart';

class MyStatefulWidget extends StatefulWidget {
  @override
  _MyStatefulWidgetState createState() => _MyStatefulWidgetState();
}

class _MyStatefulWidgetState extends State<MyStatefulWidget> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Ejemplo de Widget Con Estado'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text('Has presionado el botón esta cantidad de veces:'),
            Text(
              '$_counter',
              style: Theme.of(context).textTheme.headline4,
            ),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _incrementCounter,
        tooltip: 'Incrementar',
        child: Icon(Icons.add),
      ),
    );
  }
}

void main() => runApp(MaterialApp(
  home: MyStatefulWidget(),
));

Explicación

  • MyStatefulWidget extiende StatefulWidget.
  • _MyStatefulWidgetState es la clase de estado que extiende State<MyStatefulWidget>.
  • _counter es una variable entera privada que contiene el conteo actual.
  • _incrementCounter es un método que incrementa _counter y llama a setState para actualizar la UI.
  • El método build describe la parte de la interfaz de usuario representada por el widget.
  • Column organiza sus hijos en una matriz vertical.
  • FloatingActionButton es un botón circular que flota sobre el contenido.

Widgets Comúnmente Usados

Aquí hay algunos widgets comúnmente usados en Flutter:

  • Container: Un widget de conveniencia que combina widgets comunes de pintura, posicionamiento y tamaño.
  • Row: Dispone sus hijos en una matriz horizontal.
  • Column: Dispone sus hijos en una matriz vertical.
  • Stack: Permite superponer widgets unos sobre otros.
  • ListView: Una lista de widgets desplazable dispuesta linealmente.
  • GridView: Una matriz 2D desplazable de widgets.

Ejemplo de un Widget Container

Container(
  height: 100.0,
  width: 100.0,
  color: Colors.blue,
  child: Center(
    child: Text('Hola'),
  ),
)

Ejemplo de un Widget Row

Row(
  mainAxisAlignment: MainAxisAlignment.spaceEvenly,
  children: <Widget>[
    Icon(Icons.star, color: Colors.red),
    Icon(Icons.star, color: Colors.green),
    Icon(Icons.star, color: Colors.blue),
  ],
)

Ciclo de Vida del Widget

Entender el ciclo de vida de un widget es crucial para gestionar el estado y los recursos de manera efectiva.

Ciclo de Vida del Widget Con Estado

  1. createState: Llamado cuando el framework crea un widget con estado.
  2. initState: Llamado cuando este objeto se inserta en el árbol.
  3. didChangeDependencies: Llamado cuando una dependencia de este objeto State cambia.
  4. build: Describe la parte de la interfaz de usuario representada por este widget.
  5. didUpdateWidget: Llamado cada vez que la configuración del widget cambia.
  6. setState: Notifica al framework que el estado interno ha cambiado.
  7. deactivate: Llamado cuando este objeto es removido del árbol.
  8. dispose: Llamado cuando este objeto es removido permanentemente del árbol.

Conclusión

Los widgets son los bloques de construcción principales de una aplicación Flutter. Entender la diferencia entre widgets sin estado y con estado, así como el ciclo de vida de los widgets con estado, es esencial para construir aplicaciones Flutter eficientes y efectivas. Al dominar estos conceptos, puedes crear interfaces de usuario dinámicas y responsivas con facilidad.
