Introducción a la API de Google Analytics

  • ¿Qué es la API de Google Analytics?

    • Un conjunto de herramientas proporcionadas por Google para acceder a los datos de Google Analytics de manera programática.
    • Permite la automatización de la recuperación de datos, informes personalizados e integración con otras aplicaciones.

  • ¿Por qué usar la API de Google Analytics?

    • Automatizar tareas repetitivas.
    • Recuperar grandes conjuntos de datos de manera eficiente.
    • Integrar datos de Google Analytics con otros sistemas empresariales.
    • Crear paneles e informes personalizados.

Configuración de la API de Google Analytics

  • Requisitos previos:

    • Una cuenta de Google.
    • Acceso a una cuenta de Google Analytics.
    • Conocimientos básicos de programación (preferiblemente en Python o JavaScript).

  • Pasos para habilitar la API:

    1. Crear un proyecto en Google Cloud Console:
      • Visita el Google Cloud Console.
      • Crea un nuevo proyecto o selecciona uno existente.
    2. Habilitar la API de Google Analytics:
      • Navega a la Biblioteca de API.
      • Busca "Google Analytics API" y habilítala.
    3. Configurar credenciales OAuth 2.0:
      • Ve a la página de "Credenciales".
      • Crea IDs de cliente OAuth 2.0.
      • Configura la pantalla de consentimiento y descarga el archivo JSON de credenciales.

Autenticación y Autorización de Solicitudes

  • Autenticación OAuth 2.0:
    • Usa el archivo JSON de credenciales para autenticar tu aplicación.
    • Ejemplo en Python usando la biblioteca google-auth:
from google.oauth2 import service_account
from googleapiclient.discovery import build

# Cargar credenciales desde el archivo JSON
credentials = service_account.Credentials.from_service_account_file(
    'path/to/credentials.json',
    scopes=['https://www.googleapis.com/auth/analytics.readonly']
)

# Construir el objeto de servicio de la API de Informes de Analytics V4
analytics = build('analyticsreporting', 'v4', credentials=credentials)

Realización de Solicitudes a la API

  • Estructura Básica de una Solicitud a la API:
    • Definir el ID de vista (perfil).
    • Especificar el rango de fechas.
    • Establecer las métricas y dimensiones.
    • Ejemplo de solicitud para obtener el número de sesiones:
response = analytics.reports().batchGet(
    body={
        'reportRequests': [
            {
                'viewId': 'YOUR_VIEW_ID',
                'dateRanges': [{'startDate': '7daysAgo', 'endDate': 'today'}],
                'metrics': [{'expression': 'ga:sessions'}]
            }
        ]
    }
).execute()

# Imprimir la respuesta
print(response)

Análisis y Uso de la Respuesta de la API

  • Entendiendo la Estructura de la Respuesta:

    • La respuesta es un objeto JSON que contiene datos del informe.
    • Los componentes clave incluyen columnHeader, data y rows.

  • Extracción de Datos de la Respuesta:

    • Ejemplo de análisis de la respuesta para obtener el número de sesiones:
for report in response.get('reports', []):
    columnHeader = report.get('columnHeader', {})
    metricHeaders = columnHeader.get('metricHeader', {}).get('metricHeaderEntries', [])
    rows = report.get('data', {}).get('rows', [])

    for row in rows:
        metrics = row.get('metrics', [])
        for metric in metrics:
            for i, value in enumerate(metric.get('values')):
                print(f"{metricHeaders[i].get('name')}: {value}")

Uso Avanzado

  • Combinación de Múltiples Métricas y Dimensiones:
    • Ejemplo de una solicitud más compleja:
response = analytics.reports().batchGet(
    body={
        'reportRequests': [
            {
                'viewId': 'YOUR_VIEW_ID',
                'dateRanges': [{'startDate': '30daysAgo', 'endDate': 'today'}],
                'metrics': [{'expression': 'ga:sessions'}, {'expression': 'ga:pageviews'}],
                'dimensions': [{'name': 'ga:country'}, {'name': 'ga:browser'}]
            }
        ]
    }
).execute()

# Imprimir la respuesta
print(response)
  • Manejo de Grandes Conjuntos de Datos con Paginación:
    • Usa el pageToken para recuperar páginas adicionales de datos.
    • Ejemplo:
page_token = None
while True:
    response = analytics.reports().batchGet(
        body={
            'reportRequests': [
                {
                    'viewId': 'YOUR_VIEW_ID',
                    'dateRanges': [{'startDate': '30daysAgo', 'endDate': 'today'}],
                    'metrics': [{'expression': 'ga:sessions'}],
                    'pageToken': page_token
                }
            ]
        }
    ).execute()

    # Procesar la respuesta
    print(response)

    # Verificar si hay más páginas
    page_token = response.get('reports', [])[0].get('nextPageToken')
    if not page_token:
        break

Conclusión

  • Resumen:

    • La API de Google Analytics es una herramienta poderosa para automatizar la recuperación de datos y crear informes personalizados.
    • La configuración implica crear un proyecto, habilitar la API y configurar credenciales OAuth 2.0.
    • Realizar solicitudes requiere especificar el ID de vista, rango de fechas, métricas y dimensiones.
    • El uso avanzado incluye la combinación de múltiples métricas y dimensiones y el manejo de grandes conjuntos de datos con paginación.

  • Próximos Pasos:

    • Experimenta con diferentes métricas y dimensiones.
    • Integra la API con otras fuentes de datos y herramientas de visualización.
    • Explora características y capacidades adicionales de la API de Google Analytics.
