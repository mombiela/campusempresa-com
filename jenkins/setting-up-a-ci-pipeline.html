Introducción a la Integración Continua (CI)

  • Definición: La Integración Continua (CI) es una práctica de desarrollo donde los desarrolladores integran código en un repositorio compartido con frecuencia, idealmente varias veces al día.
  • Beneficios:
    • Detección temprana de errores.
    • Mejora de la calidad del software.
    • Entrega más rápida de actualizaciones.

Introducción a Jenkins

  • ¿Qué es Jenkins?: Jenkins es un servidor de automatización de código abierto que ayuda a automatizar partes del proceso de desarrollo de software, incluyendo la construcción, prueba y despliegue de código.
  • Características Clave:
    • Extensible con plugins.
    • Soporta construcciones distribuidas.
    • Fácil de instalar y configurar.

Configuración de Jenkins

  • Instalación:
    • Windows: Descarga el instalador de Jenkins desde el sitio web oficial y sigue el asistente de configuración.
    • Linux: Usa gestores de paquetes como apt o yum para instalar Jenkins.
    • Docker: Ejecuta Jenkins en un contenedor Docker usando el comando: 
      docker run -p 8080:8080 -p 50000:50000 jenkins/jenkins:lts
  • Configuración Inicial:
    • Accede a Jenkins vía http://localhost:8080.
    • Desbloquea Jenkins usando la contraseña inicial de administrador.
    • Instala los plugins sugeridos.
    • Crea el primer usuario administrador.

Creación de tu Primer Trabajo en Jenkins

  • Proyecto Freestyle:
    • Navega a New Item > Freestyle project.
    • Ingresa un nombre para el trabajo y haz clic en OK.
    • Configura el trabajo:
      • Gestión de Código Fuente: Conéctate a un repositorio Git.
      • Disparadores de Construcción: Configura disparadores como Poll SCM o Build periodically.
      • Pasos de Construcción: Añade pasos de construcción como Execute shell o Invoke Ant.
      • Acciones Post-construcción: Configura acciones como Archive the artifacts o Email notifications.

Ejemplo: Construcción Simple con Script Shell

echo "Construyendo el proyecto..."
# Simular proceso de construcción
sleep 5
echo "¡Construcción completada!"

Configuración de una Canalización de CI con Jenkins

  • Pipeline como Código: Usa Jenkins Pipeline (Jenkinsfile) para definir tu canalización de CI como código.
  • Creación de un Trabajo de Pipeline:
    • Navega a New Item > Pipeline.
    • Ingresa un nombre para el trabajo y haz clic en OK.
    • Configura el pipeline:
      • Definición del Pipeline: Elige Pipeline script from SCM si tu Jenkinsfile está en un repositorio, o Pipeline script para escribirlo directamente.

Ejemplo de Jenkinsfile

pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/your-repo/your-project.git'
            }
        }
        stage('Build') {
            steps {
                sh './build.sh'
            }
        }
        stage('Test') {
            steps {
                sh './test.sh'
            }
        }
        stage('Deploy') {
            steps {
                sh './deploy.sh'
            }
        }
    }
    post {
        success {
            echo '¡Pipeline completado con éxito!'
        }
        failure {
            echo '¡Pipeline fallido!'
        }
    }
}

Conceptos Avanzados de Canalización de CI

  • Etapas Paralelas: Ejecuta múltiples etapas en paralelo para acelerar la canalización.
  • Ejecución Condicional: Usa condiciones para ejecutar etapas basadas en ciertos criterios.
  • Bibliotecas Compartidas: Reutiliza código común de la canalización en múltiples Jenkinsfiles.

Ejemplo: Etapas Paralelas

pipeline {
    agent any
    stages {
        stage('Parallel Stage') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        sh './run-unit-tests.sh'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        sh './run-integration-tests.sh'
                    }
                }
            }
        }
    }
}

Conclusión

  • Resumen:
    • Jenkins es una herramienta poderosa para configurar canalizaciones de CI.
    • Comienza con configuraciones básicas de trabajos y gradualmente avanza a canalizaciones más complejas.
    • Usa Jenkinsfile para definir tu canalización de CI como código para un mejor control de versiones y reutilización.
  • Próximos Pasos:
    • Explora los plugins de Jenkins para extender la funcionalidad.
    • Integra Jenkins con otras herramientas como Docker, Kubernetes y servicios en la nube.
    • Mejora continuamente tu canalización de CI para adaptarse a las necesidades de tu proyecto.
