Introducción a Jenkins

Jenkins es un servidor de automatización de código abierto que permite a los desarrolladores construir, probar y desplegar sus aplicaciones de manera confiable y eficiente. Esta sección cubrirá los conceptos básicos de Jenkins, su arquitectura y cómo encaja en la tubería CI/CD.

Configuración de Jenkins

Antes de desplegar aplicaciones, necesitas configurar Jenkins. Esta sección te guiará a través de la instalación y configuración inicial de Jenkins.

  • Instalación: Jenkins puede ser instalado en varios sistemas operativos, incluyendo Windows, macOS y Linux.
  • Configuración: La configuración inicial incluye configurar la seguridad, configurar cuentas de usuario e instalar los plugins necesarios.

Ejemplo: Instalando Jenkins en Ubuntu

# Actualizar el índice de paquetes
sudo apt update

# Instalar Java (Jenkins requiere Java para ejecutarse)
sudo apt install openjdk-11-jre

# Agregar el repositorio Debian de Jenkins
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'

# Actualizar el índice de paquetes nuevamente
sudo apt update

# Instalar Jenkins
sudo apt install jenkins

# Iniciar Jenkins
sudo systemctl start jenkins

# Habilitar Jenkins para que inicie al arrancar
sudo systemctl enable jenkins

Creando tu Primer Trabajo en Jenkins

Un trabajo en Jenkins es una tarea o un conjunto de tareas que Jenkins ejecuta. Esta sección te guiará a través de la creación de tu primer trabajo en Jenkins.

  • Proyecto Freestyle: Un tipo de proyecto básico que te permite configurar y ejecutar tareas simples.
  • Proyecto Pipeline: Un tipo de proyecto más avanzado que utiliza un Jenkinsfile para definir los pasos de tu tubería CI/CD.

Ejemplo: Creando un Proyecto Freestyle

  1. Abre Jenkins en tu navegador web.
  2. Haz clic en "Nuevo Ítem" en el menú de la izquierda.
  3. Ingresa un nombre para tu proyecto y selecciona "Proyecto Freestyle".
  4. Haz clic en "OK" y configura los ajustes de tu proyecto.
  5. Agrega pasos de construcción, como ejecutar comandos de shell o ejecutar scripts.
  6. Guarda tu proyecto y haz clic en "Construir Ahora" para ejecutarlo.

Tuberías de Jenkins

Las Tuberías de Jenkins son una forma poderosa de definir tus procesos de construcción, prueba y despliegue como código. Esta sección te introducirá a las Tuberías de Jenkins y cómo crearlas.

  • Pipeline Declarativo: Una sintaxis más simple y legible para definir tuberías.
  • Pipeline Escrita: Una sintaxis más flexible y poderosa para casos de uso avanzados.

Ejemplo: Pipeline Declarativo

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Construyendo...'
                // Agrega pasos de construcción aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Probando...'
                // Agrega pasos de prueba aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Desplegando...'
                // Agrega pasos de despliegue aquí
            }
        }
    }
}

Integrando Jenkins con Sistemas de Control de Versiones

Jenkins puede integrarse con varios sistemas de control de versiones (VCS) como Git, SVN y Mercurial. Esta sección se centrará en integrar Jenkins con Git.

  • Plugin de Git: El plugin de Git permite a Jenkins extraer código de un repositorio Git.
  • Webhooks: Los webhooks pueden desencadenar trabajos de Jenkins automáticamente cuando se realizan cambios en el repositorio.

Ejemplo: Configurando Jenkins con Git

  1. Instala el plugin de Git desde el gestor de plugins de Jenkins.
  2. Crea un nuevo trabajo en Jenkins o configura uno existente.
  3. En la sección "Gestión de Código Fuente", selecciona "Git".
  4. Ingresa la URL del repositorio y las credenciales si es necesario.
  5. Configura los desencadenadores de construcción, como sondear el repositorio o usar webhooks.

Desplegando Aplicaciones

Desplegar aplicaciones es una parte crítica de la tubería CI/CD. Esta sección cubrirá diferentes estrategias de despliegue y cómo implementarlas en Jenkins.

  • Despliegue Continuo: Desplegar automáticamente cada cambio que pase las pruebas.
  • Entrega Continua: Preparar automáticamente cada cambio para el despliegue pero requerir aprobación manual para desplegar.
  • Despliegue Azul-Verde: Desplegar en un nuevo entorno (azul) mientras se mantiene el entorno antiguo (verde) en funcionamiento, luego cambiar el tráfico al nuevo entorno.

Ejemplo: Desplegando una Aplicación Web

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Construyendo...'
                // Agrega pasos de construcción aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Probando...'
                // Agrega pasos de prueba aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Desplegando...'
                sh 'scp -r ./build user@server:/path/to/deploy'
                sh 'ssh user@server "systemctl restart webapp"'
            }
        }
    }
}

Monitoreo y Notificaciones

Monitorear tus trabajos de Jenkins y recibir notificaciones sobre su estado es crucial para mantener una tubería CI/CD saludable.

  • Estado de Construcción: Jenkins proporciona varias formas de monitorear el estado de tus construcciones, incluyendo la interfaz web, el historial de construcciones y la salida de consola.
  • Notificaciones: Jenkins puede enviar notificaciones por correo electrónico, Slack u otras herramientas de comunicación.

Ejemplo: Configurando Notificaciones por Correo Electrónico

  1. Instala el Plugin de Extensión de Correo Electrónico desde el gestor de plugins de Jenkins.
  2. Configura el servidor SMTP en la configuración global de Jenkins.
  3. En tu trabajo de Jenkins, agrega una "Acción Post-construcción" para enviar una notificación por correo electrónico.
  4. Configura los destinatarios y el contenido del correo electrónico.

Conclusión

Desplegar aplicaciones con Jenkins implica configurar Jenkins, crear trabajos, definir tuberías, integrar con sistemas de control de versiones e implementar estrategias de despliegue. Siguiendo estos pasos, puedes automatizar tu tubería CI/CD y asegurar despliegues de aplicaciones confiables y eficientes. Recuerda monitorear tus construcciones y configurar notificaciones para mantenerte informado sobre el estado de tus despliegues.
