Introducción a Jenkins y Kubernetes

Jenkins y Kubernetes son herramientas poderosas que, cuando se combinan, pueden mejorar significativamente tu pipeline de CI/CD. Jenkins es un servidor de automatización de código abierto que ayuda a automatizar las partes del desarrollo de software relacionadas con la construcción, prueba y despliegue, facilitando la integración continua y la entrega continua. Kubernetes, por otro lado, es una plataforma de orquestación de contenedores de código abierto que automatiza el despliegue, escalado y gestión de aplicaciones en contenedores.

¿Por qué usar Jenkins con Kubernetes?

  • Escalabilidad: Kubernetes puede escalar los agentes de Jenkins hacia arriba y hacia abajo según la carga de trabajo.
  • Aislamiento: Cada trabajo de Jenkins puede ejecutarse en su propio entorno aislado.
  • Gestión de Recursos: Kubernetes gestiona los recursos de manera eficiente, asegurando el uso óptimo de CPU y memoria.
  • Alta Disponibilidad: Kubernetes asegura que Jenkins esté altamente disponible y sea resistente a fallos.

Configuración de Jenkins en Kubernetes

Prerrequisitos

  • Comprensión básica de Jenkins y Kubernetes.
  • Un clúster de Kubernetes en funcionamiento.
  • Herramienta de línea de comandos kubectl instalada y configurada.
  • Gestor de paquetes Helm instalado.

Pasos para Desplegar Jenkins en Kubernetes

  1. Agregar el Repositorio Helm de Jenkins:

    helm repo add jenkins https://charts.jenkins.io
helm repo update

  2. Instalar Jenkins Usando Helm:

    helm install jenkins jenkins/jenkins --namespace jenkins --create-namespace

  3. Acceder a Jenkins:

    • Obtener la contraseña de administrador de Jenkins: 
      printf $(kubectl get secret --namespace jenkins jenkins -o jsonpath="{.data.jenkins-admin-password}" | base64 --decode); echo
    • Acceder a Jenkins usando el servicio de Kubernetes: 
      kubectl --namespace jenkins port-forward svc/jenkins 8080:8080
    • Abre tu navegador y ve a http://localhost:8080.

Configuración de Jenkins para Usar Agentes de Kubernetes

Instalación del Plugin de Kubernetes

  1. Ve a Manage Jenkins > Manage Plugins.
  2. Busca Kubernetes e instala el plugin.

Configuración de la Nube de Kubernetes en Jenkins

  1. Ve a Manage Jenkins > Configure System.
  2. Desplázate hacia abajo hasta Cloud y haz clic en Add a new cloud > Kubernetes.
  3. Rellena los detalles del clúster de Kubernetes:
    • Kubernetes URL: La URL de tu servidor API de Kubernetes.
    • Jenkins URL: La URL de tu instancia de Jenkins.
    • Credenciales: Agrega credenciales de Kubernetes si es necesario.
  4. Configura las Plantillas de Pods:
    • Define las plantillas de pods que Jenkins usará para crear agentes.
    • Especifica las imágenes de contenedores y los límites de recursos.

Ejemplo de Configuración de Plantilla de Pod

apiVersion: v1
kind: Pod
metadata:
  name: jenkins-agent
spec:
  containers:
  - name: jnlp
    image: jenkins/inbound-agent:latest
    args: ['$(JENKINS_SECRET)', '$(JENKINS_NAME)']
  - name: maven
    image: maven:3.6.3-jdk-8
    command: ['cat']
    tty: true
  - name: docker
    image: docker:19.03.12
    command: ['cat']
    tty: true

Ejecución de Trabajos de Jenkins en Kubernetes

Creación de un Trabajo de Pipeline Simple

  1. Ve a New Item > Pipeline.
  2. En la definición del Pipeline, usa el siguiente script: 
    pipeline {
    agent {
        kubernetes {
            yaml """
            apiVersion: v1
            kind: Pod
            spec:
              containers:
              - name: maven
                image: maven:3.6.3-jdk-8
                command:
                - cat
                tty: true
            """
        }
    }
    stages {
        stage('Build') {
            steps {
                container('maven') {
                    sh 'mvn --version'
                }
            }
        }
    }
}

Ejemplo Avanzado de Pipeline

pipeline {
    agent {
        kubernetes {
            yaml """
            apiVersion: v1
            kind: Pod
            spec:
              containers:
              - name: maven
                image: maven:3.6.3-jdk-8
                command:
                - cat
                tty: true
              - name: docker
                image: docker:19.03.12
                command:
                - cat
                tty: true
            """
        }
    }
    stages {
        stage('Build') {
            steps {
                container('maven') {
                    sh 'mvn clean install'
                }
            }
        }
        stage('Docker Build') {
            steps {
                container('docker') {
                    sh 'docker build -t myapp:latest .'
                }
            }
        }
    }
}

Conclusión

Integrar Jenkins con Kubernetes proporciona un pipeline de CI/CD robusto, escalable y eficiente. Al aprovechar las capacidades de orquestación de Kubernetes, Jenkins puede escalar dinámicamente sus agentes, asegurando una utilización óptima de los recursos y aislamiento. Esta configuración no solo mejora el rendimiento y la fiabilidad de tus procesos de CI/CD, sino que también simplifica la gestión de los entornos de construcción.
