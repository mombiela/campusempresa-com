Nivel Principiante

Comprensión de los Conceptos Básicos de Jenkins

  • Pregunta 1: ¿Qué es Jenkins y para qué se utiliza principalmente?

    • Respuesta: Jenkins es un servidor de automatización de código abierto utilizado para automatizar partes del proceso de desarrollo de software, como la construcción, prueba y despliegue de código.

  • Pregunta 2: ¿Cómo se instala Jenkins en una máquina local?

    • Respuesta: Jenkins se puede instalar en una máquina local descargando el archivo WAR de Jenkins desde el sitio web oficial y ejecutándolo usando un comando de Java: 
      java -jar jenkins.war
      Alternativamente, Jenkins se puede instalar usando gestores de paquetes como apt para sistemas basados en Debian o brew para macOS.

Configuración Básica

  • Pregunta 3: Describe los pasos para crear un nuevo trabajo en Jenkins.

    • Respuesta:
      1. Abre el panel de control de Jenkins.
      2. Haz clic en "Nuevo Elemento".
      3. Ingresa un nombre para el trabajo.
      4. Selecciona el tipo de trabajo (por ejemplo, Proyecto Freestyle).
      5. Haz clic en "OK".
      6. Configura los ajustes del trabajo y guarda.

  • Pregunta 4: ¿Cómo configuras un trabajo de Jenkins para obtener código de un repositorio Git?

    • Respuesta:
      1. En la página de configuración del trabajo, ve a la sección "Gestión de Código Fuente".
      2. Selecciona "Git".
      3. Ingresa la URL del repositorio.
      4. Proporciona credenciales si es necesario.
      5. Especifica la rama a construir (opcional).

Nivel Intermedio

Pipeline como Código

  • Pregunta 5: ¿Qué es un Pipeline de Jenkins y por qué es útil?

    • Respuesta: Un Pipeline de Jenkins es un conjunto de plugins que soporta la implementación e integración de pipelines de entrega continua en Jenkins. Permite definir todo el proceso de construcción, incluyendo las etapas de construcción, prueba y despliegue, como código, lo que facilita su gestión y control de versiones.

  • Pregunta 6: Escribe un script simple de Pipeline de Jenkins para construir un proyecto Java usando Maven.

    • Respuesta: 
      pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }
    }
}

Gestión de Nodos de Jenkins

  • Pregunta 7: Explica la diferencia entre un maestro de Jenkins y un agente de Jenkins.

    • Respuesta: El maestro de Jenkins es la unidad de control central que gestiona el entorno de Jenkins, maneja la programación de trabajos y proporciona la interfaz de usuario. Los agentes de Jenkins (o esclavos) son máquinas que realizan las tareas de construcción reales según lo indicado por el maestro.

  • Pregunta 8: ¿Cómo añades un nuevo nodo agente a Jenkins?

    • Respuesta:
      1. Ve a "Administrar Jenkins" > "Administrar Nodos y Nubes".
      2. Haz clic en "Nuevo Nodo".
      3. Ingresa un nombre para el nodo y selecciona "Agente Permanente".
      4. Configura los ajustes del nodo, como el directorio raíz remoto y el método de lanzamiento.
      5. Guarda y lanza el agente.

Nivel Avanzado

Técnicas Avanzadas de Pipeline

  • Pregunta 9: ¿Cómo puedes usar Jenkins para desplegar una aplicación en un clúster de Kubernetes?
    • Respuesta:
      1. Instala el plugin de Kubernetes en Jenkins.
      2. Configura las credenciales de Kubernetes en Jenkins.
      3. Crea un script de Pipeline de Jenkins que use el plugin de Kubernetes para desplegar la aplicación:
      pipeline {
    agent any
    environment {
        KUBECONFIG = credentials('kubeconfig')
    }
    stages {
        stage('Deploy') {
            steps {
                sh 'kubectl apply -f deployment.yaml'
            }
        }
    }
}

Seguridad y Mejores Prácticas

  • Pregunta 10: ¿Cuáles son algunas de las mejores prácticas para asegurar una instancia de Jenkins?
    • Respuesta:
      • Usa HTTPS para asegurar la comunicación.
      • Habilita y configura la autenticación y autorización.
      • Actualiza regularmente Jenkins y sus plugins.
      • Usa control de acceso basado en roles (RBAC) para limitar los permisos de los usuarios.
      • Realiza copias de seguridad de las configuraciones y datos de Jenkins regularmente.

Conclusión

Este conjunto de preguntas de examen de muestra cubre una variedad de temas desde conceptos básicos de Jenkins hasta técnicas avanzadas de pipeline y mejores prácticas de seguridad. Al practicar estas preguntas, los estudiantes pueden reforzar su comprensión de Jenkins y prepararse para escenarios del mundo real donde necesiten implementar y gestionar Jenkins en un entorno profesional.
