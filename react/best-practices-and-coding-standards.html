Introducción

En esta sección, cubriremos las mejores prácticas y estándares de codificación para desarrollar aplicaciones React. Adherirse a estas prácticas asegura que tu código sea limpio, mantenible y escalable.

Mejores Prácticas Generales

Usar Componentes Funcionales

  • Por qué: Los componentes funcionales son más simples y fáciles de leer.
  • Ejemplo: 
    // Componente Funcional
const Greeting = ({ name }) => <h1>Hello, {name}!</h1>;

// Componente de Clase
class Greeting extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

Usar Hooks para Estado y Efectos Secundarios

  • Por qué: Los hooks te permiten usar estado y otras características de React sin escribir una clase.
  • Ejemplo: 
    import React, { useState, useEffect } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `Count: ${count}`;
  }, [count]);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
};

Mantener los Componentes Pequeños y Enfocados

  • Por qué: Los componentes más pequeños son más fáciles de probar y reutilizar.
  • Ejemplo: 
    // Bueno: Componente pequeño y enfocado
const Button = ({ onClick, label }) => (
  <button onClick={onClick}>{label}</button>
);

// Malo: Componente grande y desenfocado
const App = () => (
  <div>
    <button onClick={() => alert('Clicked!')}>Click me</button>
    <p>Some unrelated content</p>
  </div>
);

Estándares de Codificación

Convenciones de Nombres

  • Componentes: Usar PascalCase. 
    // Correcto
const MyComponent = () => <div>My Component</div>;

// Incorrecto
const myComponent = () => <div>My Component</div>;
  • Props y Estado: Usar camelCase. 
    const MyComponent = ({ userName }) => <div>{userName}</div>;

Estructura de Archivos

  • Por qué: Una estructura de archivos bien organizada hace que tu proyecto sea más fácil de navegar.
  • Ejemplo: 
    src/
├── components/
│   ├── Button.js
│   ├── Header.js
├── pages/
│   ├── HomePage.js
│   ├── AboutPage.js
├── App.js
├── index.js

Formato Consistente

  • Por qué: Un formato consistente mejora la legibilidad.
  • Herramientas: Usar Prettier o ESLint. 
    // .prettierrc
{
  "singleQuote": true,
  "trailingComma": "es5"
}

Evitar Estilos en Línea

  • Por qué: Los estilos en línea pueden hacer que tus componentes sean más difíciles de mantener.
  • Ejemplo: 
    // Bueno: Usar clases CSS
const MyComponent = () => <div className="my-component">Content</div>;

// Malo: Usar estilos en línea
const MyComponent = () => <div style={{ color: 'red' }}>Content</div>;

Optimización de Rendimiento

Usar React.memo

  • Por qué: Previene renderizados innecesarios.
  • Ejemplo: 
    import React from 'react';

const MyComponent = React.memo(({ value }) => {
  return <div>{value}</div>;
});

Usar useCallback y useMemo

  • Por qué: Memorizar funciones y valores para evitar cálculos innecesarios.
  • Ejemplo: 
    import React, { useCallback, useMemo } from 'react';

const MyComponent = ({ items }) => {
  const computeExpensiveValue = useCallback(() => {
    // cálculo costoso
  }, [items]);

  const memoizedValue = useMemo(() => computeExpensiveValue(), [computeExpensiveValue]);

  return <div>{memoizedValue}</div>;
};

Conclusión

Siguiendo estas mejores prácticas y estándares de codificación, puedes crear aplicaciones React que no solo sean funcionales, sino también limpias, mantenibles y eficientes. La consistencia en tu base de código hará que sea más fácil trabajar en proyectos de manera colaborativa y asegurar la escalabilidad a largo plazo.
