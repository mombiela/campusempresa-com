Introducción

La simulación y el espionaje son técnicas esenciales en las pruebas de aplicaciones React. Ayudan a aislar componentes y funciones para asegurar que las pruebas sean confiables y enfocadas. Esta sección cubrirá los conceptos básicos de la simulación y el espionaje, su importancia y cómo implementarlos utilizando bibliotecas de pruebas populares como Jest y Enzyme.

Conceptos Clave

  • Simulación: Crear una versión falsa de una función o módulo para controlar su comportamiento durante las pruebas.
  • Espionaje: Monitorear llamadas a una función para verificar cómo se utilizó durante las pruebas.

Por Qué la Simulación y el Espionaje son Importantes

  • Aislamiento: Asegura que las pruebas se centren en el componente o función que se está probando, no en sus dependencias.
  • Control: Permite simular diferentes escenarios y casos límite.
  • Rendimiento: Reduce la complejidad y el tiempo de ejecución de las pruebas al evitar implementaciones reales.

Simulación en React

La simulación se utiliza a menudo para reemplazar dependencias con versiones controladas. Jest proporciona funciones integradas para crear simulaciones.

Simulación de Funciones

Para simular una función, puedes usar jest.fn().

// Función original
const fetchData = () => {
  return fetch('/api/data').then(response => response.json());
};

// Función simulada
const fetchDataMock = jest.fn().mockResolvedValue({ data: 'mocked data' });

// Usando la simulación en una prueba
test('fetchData returns mocked data', async () => {
  const data = await fetchDataMock();
  expect(data).toEqual({ data: 'mocked data' });
});

Simulación de Módulos

Puedes simular módulos enteros usando jest.mock().

// Módulo original
// api.js
export const fetchData = () => {
  return fetch('/api/data').then(response => response.json());
};

// Archivo de prueba
import { fetchData } from './api';

jest.mock('./api', () => ({
  fetchData: jest.fn().mockResolvedValue({ data: 'mocked data' })
}));

test('fetchData returns mocked data', async () => {
  const data = await fetchData();
  expect(data).toEqual({ data: 'mocked data' });
});

Espionaje en React

El espionaje te permite rastrear llamadas a una función e inspeccionar cómo se utilizó. Jest proporciona jest.spyOn() para este propósito.

Espionaje en Métodos

Puedes espiar métodos de objetos para verificar su uso.

class User {
  getName() {
    return 'John Doe';
  }
}

test('getName method is called', () => {
  const user = new User();
  const spy = jest.spyOn(user, 'getName');
  
  user.getName();
  
  expect(spy).toHaveBeenCalled();
});

Espionaje en Métodos de Componentes React

El espionaje también se puede usar para monitorear métodos de componentes React.

import React from 'react';
import { shallow } from 'enzyme';

class MyComponent extends React.Component {
  componentDidMount() {
    this.fetchData();
  }

  fetchData() {
    // lógica de obtención de datos
  }

  render() {
    return <div>My Component</div>;
  }
}

test('fetchData is called on componentDidMount', () => {
  const spy = jest.spyOn(MyComponent.prototype, 'fetchData');
  shallow(<MyComponent />);
  
  expect(spy).toHaveBeenCalled();
});

Conclusión

La simulación y el espionaje son técnicas poderosas que mejoran la confiabilidad y eficiencia de tus pruebas. Al aislar componentes y controlar sus dependencias, puedes crear pruebas más enfocadas y rápidas. Ya sea que estés simulando funciones, módulos o espiando métodos, estas herramientas son invaluables para mantener aplicaciones React de alta calidad.
