Introducción a las Excepciones

En Python, las excepciones son errores que ocurren durante la ejecución de un programa. Cuando ocurre un error, Python detiene el programa y genera un mensaje de error. Esto se conoce como una excepción.

Conceptos Clave

  • Excepción: Un error que ocurre durante la ejecución de un programa.
  • Manejo: El proceso de responder a las excepciones de una manera controlada.
  • Bloque Try: El código que podría lanzar una excepción se coloca dentro de este bloque.
  • Bloque Except: El código que maneja la excepción se coloca dentro de este bloque.
  • Bloque Finally: El código que siempre se ejecutará, independientemente de si ocurrió una excepción o no.

Manejo Básico de Excepciones

Para manejar excepciones, se utilizan los bloques try y except.

try:
    # Código que puede causar una excepción
    result = 10 / 0
except ZeroDivisionError:
    # Código que se ejecuta si ocurre una excepción
    print("¡No puedes dividir por cero!")

En este ejemplo:

  • El código dentro del bloque try intenta dividir por cero, lo que genera un ZeroDivisionError.
  • El bloque except captura la excepción e imprime un mensaje.

Manejo de Múltiples Excepciones

Puedes manejar múltiples excepciones especificando múltiples bloques except.

try:
    value = int(input("Introduce un número: "))
    result = 10 / value
except ValueError:
    print("¡Entrada no válida! Por favor, introduce un número válido.")
except ZeroDivisionError:
    print("¡No puedes dividir por cero!")

En este ejemplo:

  • Si el usuario introduce un valor no entero, se genera un ValueError.
  • Si el usuario introduce cero, se genera un ZeroDivisionError.
  • Cada excepción es manejada por su respectivo bloque except.

Uso del Bloque Finally

El bloque finally se utiliza para ejecutar código independientemente de si ocurre una excepción o no.

try:
    file = open("example.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("¡Archivo no encontrado!")
finally:
    file.close()
    print("Archivo cerrado.")

En este ejemplo:

  • El bloque finally asegura que el archivo se cierre, ya sea que ocurra una excepción o no.

Lanzar Excepciones

Puedes lanzar excepciones usando la palabra clave raise.

def check_age(age):
    if (age < 0):
        raise ValueError("¡La edad no puede ser negativa!")
    return age

try:
    age = check_age(-1)
except ValueError as e:
    print(e)

En este ejemplo:

  • La función check_age lanza un ValueError si la edad es negativa.
  • El bloque except captura la excepción e imprime el mensaje de error.

Excepciones Personalizadas

Puedes crear excepciones personalizadas definiendo una nueva clase que herede de la clase Exception.

class CustomError(Exception):
    pass

try:
    raise CustomError("¡Este es un error personalizado!")
except CustomError as e:
    print(e)

En este ejemplo:

  • Se define una excepción personalizada CustomError.
  • La excepción personalizada se lanza y se captura en el bloque except.

Conclusión

Manejar excepciones es una parte crucial de escribir programas robustos en Python. Al usar los bloques try, except y finally, puedes gestionar errores de manera elegante y asegurarte de que tu programa continúe funcionando sin problemas. Además, lanzar y crear excepciones personalizadas permite un manejo de errores más preciso y adaptado a tus necesidades específicas.
