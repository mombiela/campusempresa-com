Introducción a las Pruebas Unitarias

Las pruebas unitarias son una técnica de prueba de software donde se prueban unidades o componentes individuales de un software en aislamiento. El objetivo es validar que cada unidad del software funcione como se espera. En Python, el módulo unittest proporciona un marco para crear y ejecutar pruebas unitarias.

¿Por qué Pruebas Unitarias?

  • Detección Temprana de Errores: Identifica problemas temprano en el ciclo de desarrollo.
  • Calidad del Código: Asegura que el código funcione como se pretende.
  • Confianza en la Refactorización: Permite a los desarrolladores refactorizar el código sin temor a introducir nuevos errores.
  • Documentación: Sirve como documentación para la base de código.

Empezando con unittest

El módulo unittest es parte de la biblioteca estándar de Python, por lo que no se requiere instalación adicional. A continuación, se muestra un ejemplo simple para comenzar.

Ejemplo Básico

import unittest

def add(a, b):
    return a + b

class TestAddFunction(unittest.TestCase):
    def test_add_integers(self):
        self.assertEqual(add(1, 2), 3)
    
    def test_add_floats(self):
        self.assertEqual(add(1.1, 2.2), 3.3)
    
    def test_add_strings(self):
        self.assertEqual(add('hello', ' world'), 'hello world')

if __name__ == '__main__':
    unittest.main()

Explicación

  • import unittest: Importa el módulo unittest.
  • función add: Una función simple para sumar dos números o concatenar dos cadenas.
  • clase TestAddFunction: Hereda de unittest.TestCase y contiene métodos de prueba.
  • test_add_integers, test_add_floats, test_add_strings: Métodos de prueba que usan self.assertEqual para verificar si la función add funciona correctamente.
  • unittest.main(): Ejecuta las pruebas cuando se ejecuta el script.

Conceptos Clave

  • Caso de Prueba: La unidad más pequeña de prueba. Verifica una respuesta específica a un conjunto particular de entradas.
  • Suite de Pruebas: Una colección de casos de prueba, suites de prueba o ambas.
  • Ejecutor de Pruebas: Un componente que orquesta la ejecución de pruebas y proporciona el resultado al usuario.

Características Avanzadas

Configuración y Desmontaje

A veces, necesitas configurar un entorno particular antes de ejecutar las pruebas y limpiarlo después. unittest proporciona los métodos setUp y tearDown para este propósito.

import unittest

class TestExample(unittest.TestCase):
    def setUp(self):
        self.value = 42

    def tearDown(self):
        del self.value

    def test_value(self):
        self.assertEqual(self.value, 42)

if __name__ == '__main__':
    unittest.main()

Simulación

La simulación es útil para aislar la unidad de trabajo que se está probando reemplazando dependencias con objetos simulados.

from unittest.mock import MagicMock
import unittest

class Database:
    def fetch_data(self):
        pass  # Imagina que este método obtiene datos de una base de datos

class DataProcessor:
    def __init__(self, db):
        self.db = db

    def process(self):
        data = self.db.fetch_data()
        return data * 2

class TestDataProcessor(unittest.TestCase):
    def test_process(self):
        mock_db = MagicMock()
        mock_db.fetch_data.return_value = 21
        processor = DataProcessor(mock_db)
        self.assertEqual(processor.process(), 42)

if __name__ == '__main__':
    unittest.main()

Pruebas Parametrizadas

Puedes ejecutar la misma prueba con diferentes parámetros utilizando la característica subTest del módulo unittest.

import unittest

class TestParameterized(unittest.TestCase):
    def test_add(self):
        test_cases = [
            (1, 2, 3),
            (1.1, 2.2, 3.3),
            ('hello', ' world', 'hello world')
        ]
        for a, b, expected in test_cases:
            with self.subTest(a=a, b=b, expected=expected):
                self.assertEqual(add(a, b), expected)

if __name__ == '__main__':
    unittest.main()

Conclusión

Las pruebas unitarias son una práctica esencial para mantener un código de alta calidad. El módulo unittest en Python proporciona un marco robusto para escribir y ejecutar pruebas. Al comprender y utilizar sus características, puedes asegurarte de que tu código sea confiable, mantenible y libre de errores. ¡Felices pruebas!
