Introducción

El empaquetado de proyectos Python es una habilidad esencial para los desarrolladores que desean distribuir su código a otros. Esta sección te guiará a través del proceso de empaquetado de tus proyectos Python, desde lo básico hasta técnicas más avanzadas.

¿Por qué Empaquetar Tus Proyectos Python?

  • Distribución: Comparte fácilmente tu código con otros.
  • Gestión de Dependencias: Asegúrate de que todas las bibliotecas y dependencias necesarias estén incluidas.
  • Control de Versiones: Gestiona diferentes versiones de tu proyecto.
  • Instalación: Simplifica el proceso de instalación para los usuarios.

Conceptos Básicos

  • Paquete: Una colección de módulos Python.
  • Módulo: Un solo archivo Python que contiene código.
  • Distribución: Un archivo de archivo versionado que se puede instalar.

Configuración de Tu Proyecto

  1. Estructura del Directorio: Organiza los archivos de tu proyecto. 
    mi_proyecto/
├── mi_proyecto/
│   ├── __init__.py
│   ├── modulo1.py
│   └── modulo2.py
├── tests/
│   ├── __init__.py
│   └── test_modulo1.py
├── README.md
├── setup.py
└── requirements.txt
  2. README.md: Proporciona una descripción de tu proyecto.
  3. requirements.txt: Lista todas las dependencias. 
    numpy==1.21.0
pandas==1.3.0

Creación de setup.py

El archivo setup.py es el núcleo de cada proyecto Python. Contiene metadatos sobre tu proyecto e instrucciones sobre cómo instalarlo.

Ejemplo de setup.py

from setuptools import setup, find_packages

setup(
    name='mi_proyecto',
    version='0.1.0',
    packages=find_packages(),
    install_requires=[
        'numpy==1.21.0',
        'pandas==1.3.0',
    ],
    entry_points={
        'console_scripts': [
            'mi_proyecto=mi_proyecto.modulo1:main',
        ],
    },
    author='Tu Nombre',
    author_email='tu.email@ejemplo.com',
    description='Una breve descripción de tu proyecto',
    long_description=open('README.md').read(),
    long_description_content_type='text/markdown',
    url='https://github.com/tuusuario/mi_proyecto',
    classifiers=[
        'Programming Language :: Python :: 3',
        'License :: OSI Approved :: MIT License',
        'Operating System :: OS Independent',
    ],
    python_requires='>=3.6',
)

Explicación

  • name: El nombre de tu proyecto.
  • version: La versión actual de tu proyecto.
  • packages: Una lista de todos los paquetes de importación de Python que deben incluirse en el paquete de distribución.
  • install_requires: Una lista de dependencias que son necesarias para tu proyecto.
  • entry_points: Define puntos de entrada para scripts de línea de comandos.
  • author: Tu nombre.
  • author_email: Tu dirección de correo electrónico.
  • description: Un resumen corto de tu proyecto.
  • long_description: Una descripción detallada, usualmente el contenido de tu archivo README.
  • url: La URL de la página principal de tu proyecto.
  • classifiers: Una lista de clasificadores que describen tu proyecto.
  • python_requires: La versión de Python requerida para tu proyecto.

Construcción y Distribución de Tu Paquete

  1. Construye Tu Paquete:

    python setup.py sdist bdist_wheel
    • sdist: Distribución de Fuente.
    • bdist_wheel: Distribución Construida.

  2. Sube a PyPI:

    pip install twine
twine upload dist/*
    • twine: Una utilidad para publicar paquetes Python en PyPI.

Instalación de Tu Paquete

Los usuarios pueden instalar tu paquete usando pip:

pip install mi_proyecto

Conclusión

El empaquetado de tus proyectos Python te permite compartir tu trabajo con otros, gestionar dependencias y simplificar el proceso de instalación. Siguiendo los pasos descritos en esta sección, puedes crear, construir y distribuir tus paquetes Python de manera eficiente. Recuerda mantener tu archivo setup.py actualizado con los metadatos y dependencias correctos para asegurar una experiencia de instalación fluida para tus usuarios.
