Introducción

En Python, las excepciones son una forma de manejar errores que ocurren durante la ejecución de un programa. Lanzar excepciones te permite señalar que ha ocurrido un error y proporcionar información sobre la naturaleza del error. Esta sección cubrirá los conceptos básicos del lanzamiento de excepciones, cómo crear excepciones personalizadas y técnicas avanzadas para manejar excepciones.

Conceptos Clave

  • Excepción: Un error que ocurre durante la ejecución de un programa.
  • Lanzar una Excepción: El acto de señalar que ha ocurrido un error usando la declaración raise.
  • Excepciones Personalizadas: Excepciones definidas por el usuario que extienden la clase base Exception.

Lanzamiento de Excepciones

Sintaxis Básica

Para lanzar una excepción en Python, usas la declaración raise seguida de una instancia de una clase de excepción.

# Ejemplo: Lanzando una excepción básica
raise Exception("Esta es una excepción genérica")

Excepciones Comunes Incorporadas

Python proporciona varias excepciones incorporadas que puedes usar para señalar diferentes tipos de errores:

  • ValueError: Se lanza cuando una función recibe un argumento del tipo correcto pero de valor inapropiado.
  • TypeError: Se lanza cuando una operación o función se aplica a un objeto de tipo inapropiado.
  • IndexError: Se lanza cuando un subíndice de secuencia está fuera de rango.
  • KeyError: Se lanza cuando una clave de diccionario no se encuentra.
# Ejemplo: Lanzando un ValueError
def divide(a, b):
    if b == 0:
        raise ValueError("No se puede dividir por cero")
    return a / b

try:
    result = divide(10, 0)
except ValueError as e:
    print(e)

Creación de Excepciones Personalizadas

Puedes crear tus propias excepciones definiendo una nueva clase que herede de la clase Exception.

# Ejemplo: Creando una excepción personalizada
class CustomError(Exception):
    def __init__(self, message):
        self.message = message

# Lanzando la excepción personalizada
raise CustomError("Este es un error personalizado")

Uso de Excepciones Personalizadas

Las excepciones personalizadas pueden ser usadas para proporcionar mensajes de error más específicos y para manejar errores de una manera más granular.

# Ejemplo: Usando una excepción personalizada en una función
class NegativeNumberError(Exception):
    pass

def check_positive(number):
    if number < 0:
        raise NegativeNumberError("No se permiten números negativos")

try:
    check_positive(-5)
except NegativeNumberError as e:
    print(e)

Técnicas Avanzadas

Encadenamiento de Excepciones

Puedes encadenar excepciones para proporcionar más contexto sobre el error. Esto se hace usando la palabra clave from.

# Ejemplo: Encadenamiento de excepciones
try:
    raise ValueError("Error inicial")
except ValueError as e:
    raise CustomError("Ocurrió un error personalizado") from e

Relanzamiento de Excepciones

A veces puede que quieras capturar una excepción, realizar algunas acciones y luego relanzar la misma excepción.

# Ejemplo: Relanzamiento de una excepción
try:
    raise ValueError("Ocurrió un error")
except ValueError as e:
    print("Manejando el error")
    raise  # Relanza la misma excepción

Conclusión

Lanzar excepciones es una parte fundamental del manejo de errores en Python. Al entender cómo lanzar excepciones incorporadas, crear excepciones personalizadas y usar técnicas avanzadas como el encadenamiento y el relanzamiento, puedes escribir código más robusto y mantenible. Recuerda siempre proporcionar mensajes de error significativos para ayudar con la depuración y manejar las excepciones de manera elegante para asegurar una buena experiencia de usuario.
