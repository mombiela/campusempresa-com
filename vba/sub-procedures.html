Introducción a los Procedimientos Sub

Los Procedimientos Sub, comúnmente conocidos como "Subs", son un concepto fundamental en la programación VBA. Permiten encapsular un conjunto de instrucciones que se pueden ejecutar como una unidad. Esto ayuda a organizar el código, hacerlo más legible y reutilizar bloques de código sin reescribirlos.

Conceptos Clave

  • Definición: Un Procedimiento Sub es un bloque de código que realiza una tarea específica pero no devuelve un valor.
  • Sintaxis: La sintaxis básica para un Procedimiento Sub es: 
    Sub NombreDelProcedimiento()
    ' Código a ejecutar
End Sub
  • Llamar a un Sub: Puedes llamar a un Procedimiento Sub desde otro procedimiento o evento.

Crear y Usar Procedimientos Sub

Ejemplo Básico

Comencemos con un ejemplo simple de un Procedimiento Sub que muestra un cuadro de mensaje.

Sub MostrarMensaje()
    MsgBox "¡Hola, Mundo!"
End Sub

Para llamar a este Procedimiento Sub, simplemente puedes escribir:

Sub LlamarMostrarMensaje()
    MostrarMensaje
End Sub

Pasar Argumentos

Los Procedimientos Sub también pueden aceptar parámetros, lo que te permite pasar datos a ellos.

Sub SaludarUsuario(nombreUsuario As String)
    MsgBox "¡Hola, " & nombreUsuario & "!"
End Sub

Llamando a este Sub con un argumento:

Sub LlamarSaludarUsuario()
    SaludarUsuario "Alicia"
End Sub

Parámetros Opcionales

Puedes definir parámetros opcionales en un Procedimiento Sub, lo que permite al llamador omitir algunos argumentos.

Sub MostrarMensaje(mensaje As String, Optional titulo As String = "Título Predeterminado")
    MsgBox mensaje, , titulo
End Sub

Llamando a este Sub con y sin el parámetro opcional:

Sub LlamarMostrarMensaje()
    MostrarMensaje "Este es un mensaje", "Título Personalizado"
    MostrarMensaje "Este es otro mensaje"
End Sub

Procedimientos Sub Avanzados

Usando ByRef y ByVal

Los parámetros en VBA pueden pasarse por referencia (ByRef) o por valor (ByVal). Por defecto, los parámetros se pasan por referencia.

  • ByRef: El procedimiento llamado puede modificar el valor de la variable.
  • ByVal: El procedimiento llamado obtiene una copia del valor de la variable y no puede modificar la variable original.
Sub ModificarValor(ByRef num As Integer)
    num = num * 2
End Sub

Sub ProbarModificarValor()
    Dim miNum As Integer
    miNum = 10
    ModificarValor miNum
    MsgBox miNum ' El resultado será 20
End Sub

Manejo de Errores en Procedimientos Sub

El manejo de errores es crucial para aplicaciones VBA robustas. Puedes usar declaraciones On Error para gestionar errores dentro de los Procedimientos Sub.

Sub DivisionSegura(dividendo As Double, divisor As Double)
    On Error GoTo ManejadorDeErrores
    MsgBox "Resultado: " & (dividendo / divisor)
    Exit Sub

ManejadorDeErrores:
    MsgBox "Error: No se permite la división por cero."
End Sub

Llamando a este Sub con manejo de errores:

Sub ProbarDivisionSegura()
    DivisionSegura 10, 0
End Sub

Resumen

Los Procedimientos Sub en VBA son esenciales para escribir código organizado, reutilizable y mantenible. Permiten:

  • Encapsular código en bloques manejables.
  • Pasar datos a través de parámetros, incluidos los opcionales.
  • Manejar errores de manera elegante.
  • Elegir entre pasar argumentos por referencia o por valor.

Al dominar los Procedimientos Sub, puedes mejorar significativamente la estructura y eficiencia de tus proyectos VBA.
