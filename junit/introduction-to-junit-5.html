Introducción a JUnit 5
Anotaciones de JUnit 5 ►

JUnit 5 es la última versión del popular framework de pruebas Java JUnit. Introduce varias características nuevas y mejoras sobre sus predecesores, haciéndolo más poderoso y flexible para escribir y ejecutar pruebas. Esta sección proporcionará una visión general de JUnit 5, su arquitectura y uso básico.

¿Qué es JUnit 5?

JUnit 5 es un framework para escribir y ejecutar pruebas en Java. Se utiliza para asegurar que tu código se comporte como se espera y ayuda a identificar errores temprano en el proceso de desarrollo. JUnit 5 está compuesto por tres módulos principales:

  • JUnit Platform: La base para lanzar frameworks de pruebas en la JVM.
  • JUnit Jupiter: El nuevo modelo de programación y modelo de extensión para escribir pruebas y extensiones en JUnit 5.
  • JUnit Vintage: Proporciona soporte para ejecutar pruebas de JUnit 3 y JUnit 4 en la plataforma JUnit 5.

Características Clave de JUnit 5

  • Arquitectura Modular: JUnit 5 está dividido en varios módulos, permitiendo mayor flexibilidad y extensibilidad.
  • Nuevas Anotaciones: Introduce nuevas anotaciones para un mejor control sobre la ejecución de pruebas.
  • Modelo de Extensión: Proporciona un poderoso modelo de extensión para personalizar la ejecución de pruebas.
  • Pruebas Dinámicas: Soporta pruebas dinámicas que pueden ser generadas en tiempo de ejecución.
  • Aserciones Mejoradas: Métodos de aserción mejorados para pruebas más expresivas.

Configuración de JUnit 5

Para empezar a usar JUnit 5, necesitas agregar las dependencias necesarias a tu proyecto. Aquí hay un ejemplo de cómo configurar JUnit 5 con Maven:

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.junit</groupId>
            <artifactId>junit-bom</artifactId>
            <version>5.8.1</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<dependencies>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter-api</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter-engine</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>

Para Gradle, puedes agregar las siguientes dependencias:

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter-api:5.8.1'
    testRuntimeOnly 'org.junit.jupiter:junit-jupiter-engine:5.8.1'
}

Escribiendo Tu Primera Prueba

Vamos a escribir una prueba simple usando JUnit 5. Considera una clase Calculator con un método add:

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

Ahora, vamos a escribir una prueba para el método add:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    void testAdd() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 debería ser igual a 5");
    }
}

Explicación

  • @Test: Marca el método como un método de prueba.
  • assertEquals: Asegura que el valor esperado (5) es igual al valor actual (resultado).

Anotaciones Comunes en JUnit 5

JUnit 5 introduce varias nuevas anotaciones para controlar el ciclo de vida y comportamiento de las pruebas:

  • @Test: Marca un método como un método de prueba.
  • @BeforeEach: Se ejecuta antes de cada método de prueba.
  • @AfterEach: Se ejecuta después de cada método de prueba.
  • @BeforeAll: Se ejecuta una vez antes de todos los métodos de prueba en la clase.
  • @AfterAll: Se ejecuta una vez después de todos los métodos de prueba en la clase.
  • @Disabled: Desactiva un método de prueba o clase.

Ejemplo

import org.junit.jupiter.api.*;

public class LifecycleTest {

    @BeforeAll
    static void initAll() {
        System.out.println("Antes de todas las pruebas");
    }

    @BeforeEach
    void init() {
        System.out.println("Antes de cada prueba");
    }

    @Test
    void testOne() {
        System.out.println("Prueba uno");
    }

    @Test
    void testTwo() {
        System.out.println("Prueba dos");
    }

    @AfterEach
    void tearDown() {
        System.out.println("Después de cada prueba");
    }

    @AfterAll
    static void tearDownAll() {
        System.out.println("Después de todas las pruebas");
    }
}

Conclusión

JUnit 5 es un framework de pruebas poderoso y flexible que proporciona un enfoque moderno para escribir y ejecutar pruebas en Java. Su arquitectura modular, nuevas anotaciones y características mejoradas lo convierten en una herramienta esencial para los desarrolladores de Java. En esta introducción, cubrimos los conceptos básicos de JUnit 5, incluyendo sus características clave, configuración y escritura de pruebas simples. A medida que avances en este curso, aprenderás características y técnicas más avanzadas para aprovechar al máximo JUnit 5.
