Introducción

JUnit es un marco de pruebas popular para Java que permite a los desarrolladores escribir y ejecutar pruebas repetibles. Esta sección te guiará a través del proceso de configuración de JUnit en tu entorno de desarrollo, ya sea que estés utilizando un Entorno de Desarrollo Integrado (IDE) o una herramienta de automatización de compilación.

Requisitos previos

Antes de configurar JUnit, asegúrate de tener lo siguiente:

  • Kit de Desarrollo de Java (JDK) instalado.
  • Un IDE como IntelliJ IDEA, Eclipse o NetBeans, o una herramienta de compilación como Maven o Gradle.

Configuración de JUnit en un IDE

IntelliJ IDEA

  1. Crear un Nuevo Proyecto:

    • Abre IntelliJ IDEA.
    • Selecciona "New Project" y elige "Java".
    • Sigue las indicaciones para crear tu proyecto.

  2. Agregar JUnit al Proyecto:

    • Haz clic derecho en la carpeta del proyecto en el Explorador de Proyectos.
    • Selecciona "Add Framework Support".
    • Elige "JUnit" y haz clic en "OK".

  3. Verificar la Dependencia de JUnit:

    • Abre el archivo pom.xml (si usas Maven) o el archivo build.gradle (si usas Gradle) para asegurarte de que JUnit esté listado como una dependencia.

Eclipse

  1. Crear un Nuevo Proyecto Java:

    • Abre Eclipse.
    • Ve a File > New > Java Project.
    • Sigue las indicaciones para crear tu proyecto.

  2. Agregar la Biblioteca JUnit:

    • Haz clic derecho en la carpeta del proyecto en el Explorador de Proyectos.
    • Selecciona Build Path > Add Libraries.
    • Elige "JUnit" y haz clic en "Next".
    • Selecciona la versión de JUnit (se recomienda JUnit 5) y haz clic en "Finish".

  3. Verificar la Dependencia de JUnit:

    • Asegúrate de que la biblioteca JUnit esté listada en la sección Referenced Libraries en el Explorador de Proyectos.

Configuración de JUnit con Herramientas de Compilación

Maven

  1. Crear un Proyecto Maven:

    • Abre tu terminal o símbolo del sistema.
    • Ejecuta el siguiente comando para crear un nuevo proyecto Maven: 
      mvn archetype:generate -DgroupId=com.example -DartifactId=my-app -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false

  2. Agregar la Dependencia de JUnit:

    • Abre el archivo pom.xml en tu directorio de proyecto.
    • Agrega la siguiente dependencia dentro de la etiqueta <dependencies>: 
      <dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-engine</artifactId>
    <version>5.8.1</version>
    <scope>test</scope>
</dependency>

  3. Verificar la Dependencia:

    • Ejecuta mvn clean install para asegurarte de que las dependencias se descarguen e instalen correctamente.

Gradle

  1. Crear un Proyecto Gradle:

    • Abre tu terminal o símbolo del sistema.
    • Ejecuta el siguiente comando para crear un nuevo proyecto Gradle: 
      gradle init --type java-application

  2. Agregar la Dependencia de JUnit:

    • Abre el archivo build.gradle en tu directorio de proyecto.
    • Agrega la siguiente dependencia dentro del bloque dependencies: 
      testImplementation 'org.junit.jupiter:junit-jupiter-engine:5.8.1'

  3. Verificar la Dependencia:

    • Ejecuta gradle build para asegurarte de que las dependencias se descarguen e instalen correctamente.

Escribiendo tu Primera Prueba

Una vez que JUnit esté configurado, puedes escribir tu primera prueba. Aquí tienes un ejemplo simple:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 should equal 5");
    }
}

Explicación

  • Sentencias de Importación: Importa las clases necesarias de JUnit.
  • Anotación de Prueba: Usa @Test para indicar que el método es un método de prueba.
  • Aserciones: Usa assertEquals para verificar si el resultado esperado coincide con el resultado real.

Conclusión

Configurar JUnit es un proceso sencillo, ya sea que estés utilizando un IDE o una herramienta de compilación como Maven o Gradle. Una vez configurado, puedes comenzar a escribir y ejecutar pruebas para asegurarte de que tu código funcione como se espera. Este conocimiento fundamental será crucial a medida que avances a temas más avanzados de JUnit.
