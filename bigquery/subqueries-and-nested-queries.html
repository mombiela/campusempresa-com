Subconsultas y Consultas Anidadas en BigQuery
Expresiones de tabla comunes (CTEs) ►

Introducción

Las subconsultas y consultas anidadas son herramientas poderosas en SQL que te permiten realizar operaciones complejas de recuperación de datos. En BigQuery, estos conceptos son esenciales para escribir consultas eficientes y legibles. Esta sección te guiará desde lo básico hasta el uso avanzado de subconsultas y consultas anidadas en BigQuery.

¿Qué son las Subconsultas?

Una subconsulta es una consulta anidada dentro de otra consulta. Puede ser utilizada en varias partes de una sentencia SQL, como las cláusulas SELECT, FROM, WHERE y HAVING.

Conceptos Clave

  • Consulta Interna: La consulta dentro de la consulta principal.
  • Consulta Externa: La consulta principal que contiene la subconsulta.
  • Subconsulta Correlacionada: Una subconsulta que hace referencia a columnas de la consulta externa.

Ejemplo Básico

SELECT name
FROM `bigquery-public-data.usa_names.usa_1910_2013`
WHERE state = 'TX'
AND year = (SELECT MAX(year) FROM `bigquery-public-data.usa_names.usa_1910_2013` WHERE state = 'TX');
  • Explicación: Esta consulta recupera los nombres del estado de Texas para el año más reciente disponible en el conjunto de datos.

Tipos de Subconsultas

Subconsultas Escalares

Una subconsulta escalar devuelve un solo valor. Puede ser utilizada en lugares donde se espera un solo valor.

SELECT name, year
FROM `bigquery-public-data.usa_names.usa_1910_2013`
WHERE year = (SELECT MAX(year) FROM `bigquery-public-data.usa_names.usa_1910_2013`);
  • Explicación: Esta consulta recupera nombres y años donde el año es el máximo año en el conjunto de datos.

Subconsultas Correlacionadas

Una subconsulta correlacionada hace referencia a columnas de la consulta externa. Se evalúa una vez por cada fila procesada por la consulta externa.

SELECT name, year, state
FROM `bigquery-public-data.usa_names.usa_1910_2013` AS outer
WHERE year = (SELECT MAX(year) FROM `bigquery-public-data.usa_names.usa_1910_2013` AS inner WHERE inner.state = outer.state);
  • Explicación: Esta consulta recupera nombres, años y estados donde el año es el máximo año para cada estado.

Consultas Anidadas

Las consultas anidadas son subconsultas que se utilizan dentro de otra consulta. Pueden ser utilizadas para descomponer consultas complejas en partes más simples.

Uso de Consultas Anidadas en la Cláusula FROM

SELECT state, AVG(year) AS avg_year
FROM (
  SELECT state, year
  FROM `bigquery-public-data.usa_names.usa_1910_2013`
  WHERE name = 'John'
)
GROUP BY state;
  • Explicación: Esta consulta primero selecciona el estado y el año para el nombre 'John' y luego calcula el año promedio para cada estado.

Uso de Consultas Anidadas en la Cláusula SELECT

SELECT name,
  (SELECT MAX(year) FROM `bigquery-public-data.usa_names.usa_1910_2013` WHERE state = 'TX') AS max_year
FROM `bigquery-public-data.usa_names.usa_1910_2013`
WHERE state = 'TX';
  • Explicación: Esta consulta selecciona nombres y el año máximo para el estado de Texas.

Uso Avanzado

Combinación de Múltiples Subconsultas

SELECT name, year, state
FROM `bigquery-public-data.usa_names.usa_1910_2013`
WHERE year = (SELECT MAX(year) FROM `bigquery-public-data.usa_names.usa_1910_2013`)
AND state IN (SELECT DISTINCT state FROM `bigquery-public-data.usa_names.usa_1910_2013` WHERE name = 'John');
  • Explicación: Esta consulta recupera nombres, años y estados donde el año es el máximo año y el estado es uno donde aparece el nombre 'John'.

Consideraciones de Rendimiento

  • Índices: Asegúrate de que las columnas utilizadas en subconsultas estén indexadas para un mejor rendimiento.
  • Optimización de Consultas: Usa EXPLAIN para entender el plan de ejecución de la consulta y optimizar en consecuencia.
  • Evitar Subconsultas Correlacionadas: Cuando sea posible, evita las subconsultas correlacionadas ya que pueden ser menos eficientes.

Conclusión

Las subconsultas y consultas anidadas son esenciales para escribir consultas SQL complejas y eficientes en BigQuery. Entender cómo usarlas efectivamente puede mejorar en gran medida tus capacidades de recuperación de datos. Practica escribiendo diferentes tipos de subconsultas y consultas anidadas para volverte competente en su uso.
