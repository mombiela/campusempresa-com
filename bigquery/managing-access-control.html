Gestión del Control de Acceso
Encriptación de datos ►

En esta sección, exploraremos cómo gestionar el control de acceso en BigQuery. El control de acceso es crucial para asegurar que solo los usuarios autorizados puedan acceder o modificar tus datos. Cubriremos los conceptos básicos del control de acceso, cómo establecer permisos y técnicas avanzadas para gestionar el acceso.

Introducción al Control de Acceso

El control de acceso en BigQuery se gestiona a través de Identity and Access Management (IAM). IAM te permite controlar quién tiene acceso a tus recursos de BigQuery y qué pueden hacer con esos recursos.

  • Roles de IAM: Los roles son una colección de permisos. BigQuery proporciona roles predefinidos que puedes asignar a los usuarios.
  • Políticas de IAM: Las políticas se utilizan para otorgar roles a usuarios, grupos o cuentas de servicio.

Configuración de Roles de IAM

BigQuery proporciona varios roles predefinidos que puedes usar para gestionar el acceso. Estos roles van desde permisos amplios hasta muy específicos.

Roles Predefinidos

  • Viewer: Acceso de solo lectura a conjuntos de datos y tablas.
  • Editor: Acceso de lectura y escritura a conjuntos de datos y tablas.
  • Owner: Control total sobre conjuntos de datos y tablas, incluida la capacidad de otorgar acceso a otros.
  • BigQuery Admin: Control total sobre todos los recursos de BigQuery en un proyecto.

Roles Personalizados

Si los roles predefinidos no satisfacen tus necesidades, puedes crear roles personalizados con permisos específicos.

Asignación de Roles

Para asignar roles, necesitas actualizar la política de IAM para tu proyecto, conjunto de datos o tabla.

Ejemplo: Asignar un Rol a un Usuario

# Asignar el rol BigQuery Data Viewer a un usuario para un conjunto de datos específico
gcloud projects add-iam-policy-binding my-project \
    --member=user:example-user@gmail.com \
    --role=roles/bigquery.dataViewer

Ejemplo: Asignar un Rol a una Cuenta de Servicio

# Asignar el rol BigQuery Data Editor a una cuenta de servicio para un conjunto de datos específico
gcloud projects add-iam-policy-binding my-project \
    --member=serviceAccount:my-service-account@my-project.iam.gserviceaccount.com \
    --role=roles/bigquery.dataEditor

Gestión de Permisos de Conjuntos de Datos y Tablas

También puedes establecer permisos a nivel de conjunto de datos y tabla para un control más granular.

Ejemplo: Otorgar Acceso a un Conjunto de Datos

# Otorgar el rol BigQuery Data Viewer a un usuario para un conjunto de datos específico
bq update --dataset --add_iam_policy_binding \
    my-project:my_dataset \
    user:example-user@gmail.com:roles/bigquery.dataViewer

Ejemplo: Otorgar Acceso a una Tabla

# Otorgar el rol BigQuery Data Editor a un usuario para una tabla específica
bq update --table --add_iam_policy_binding \
    my-project:my_dataset.my_table \
    user:example-user@gmail.com:roles/bigquery.dataEditor

Técnicas Avanzadas de Control de Acceso

Uso de Listas de Control de Acceso (ACLs)

Las ACLs proporcionan una forma de gestionar el acceso a un nivel más granular que los roles de IAM. Puedes usar ACLs para otorgar acceso a columnas específicas dentro de una tabla.

Ejemplo: Establecer Permisos a Nivel de Columna

GRANT SELECT(column_name) ON TABLE my_dataset.my_table TO 'user:example-user@gmail.com';

Uso de Vistas Autorizadas

Las vistas autorizadas te permiten compartir resultados de consultas con usuarios específicos sin darles acceso a las tablas subyacentes.

Ejemplo: Crear una Vista Autorizada

  1. Crear una vista:

    CREATE VIEW my_dataset.my_view AS
SELECT column1, column2
FROM my_dataset.my_table;

  2. Autorizar la vista:

    bq update --view --add_iam_policy_binding 
            my-project:my_dataset.my_view 
            user:example-user@gmail.com:roles/bigquery.dataViewer

Conclusión

Gestionar el control de acceso en BigQuery es esencial para mantener la seguridad e integridad de tus datos. Al entender y utilizar roles de IAM, políticas, ACLs y vistas autorizadas, puedes asegurar que solo los usuarios autorizados tengan acceso a tus recursos de BigQuery. Practica asignando roles y estableciendo permisos para volverte competente en la gestión del control de acceso en BigQuery.
