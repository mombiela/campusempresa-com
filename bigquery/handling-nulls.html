Manejo de NULLs en BigQuery
Cargando datos en BigQuery ►

Introducción a los NULLs

  • Definición: En SQL, NULL representa un valor faltante o indefinido.
  • Importancia: Entender cómo manejar los valores NULL es crucial para un análisis y manipulación de datos precisos.

Identificación de NULLs

  • IS NULL: Se utiliza para verificar si un valor es NULL.
  • IS NOT NULL: Se utiliza para verificar si un valor no es NULL.
SELECT *
FROM my_table
WHERE column_name IS NULL;
SELECT *
FROM my_table
WHERE column_name IS NOT NULL;

Manejo de NULLs en Consultas

Usando COALESCE

  • Propósito: Reemplaza NULL con un valor especificado.
  • Sintaxis: COALESCE(expression, replacement_value)
SELECT COALESCE(column_name, 'default_value') AS column_name
FROM my_table;

Usando IFNULL

  • Propósito: Similar a COALESCE, pero solo toma dos argumentos.
  • Sintaxis: IFNULL(expression, replacement_value)
SELECT IFNULL(column_name, 'default_value') AS column_name
FROM my_table;

Usando NULLIF

  • Propósito: Devuelve NULL si dos expresiones son iguales.
  • Sintaxis: NULLIF(expression1, expression2)
SELECT NULLIF(column_name, 'value_to_compare') AS column_name
FROM my_table;

Agregación con NULLs

  • COUNT: COUNT(column_name) ignora los valores NULL, mientras que COUNT(*) los incluye.
  • SUM, AVG, MIN, MAX: Estas funciones ignoran los valores NULL.
SELECT COUNT(column_name) AS non_null_count,
       COUNT(*) AS total_count
FROM my_table;

Manejo Avanzado de NULLs

Usando Sentencias CASE

  • Propósito: Proporciona un manejo condicional más complejo de los valores NULL.
  • Sintaxis: CASE WHEN condition THEN result ELSE result END
SELECT CASE 
           WHEN column_name IS NULL THEN 'No Value'
           ELSE column_name
       END AS column_name
FROM my_table;

NULLs en Joins

  • LEFT JOIN: Incluye todos los registros de la tabla izquierda y los registros coincidentes de la tabla derecha. Si no hay coincidencia, se devuelven valores NULL para las columnas de la tabla derecha.
  • INNER JOIN: Excluye los registros con valores NULL en la condición de unión.
SELECT a.*, b.*
FROM table_a a
LEFT JOIN table_b b
ON a.id = b.id;

NULLs en Tipos de Datos

  • Cadena: NULL es diferente de una cadena vacía ''.
  • Numérico: NULL es diferente de cero 0.

| Tipo de Dato | Ejemplo NULL | Ejemplo No-NULL | |--------------|--------------|-----------------| | Cadena | NULL | '' | | Numérico | NULL | 0 |

Conclusión

Manejar los valores NULL de manera efectiva es esencial para un análisis de datos preciso en BigQuery. Al usar funciones como COALESCE, IFNULL y NULLIF, así como al entender cómo los valores NULL interactúan con las uniones y agregaciones, puedes asegurarte de que tus consultas devuelvan los resultados correctos. Siempre ten en cuenta cómo se tratan los valores NULL en diferentes contextos para evitar resultados inesperados en tu análisis de datos.
