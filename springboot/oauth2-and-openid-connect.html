Introducción a OAuth2 y OpenID Connect

OAuth2 y OpenID Connect son protocolos utilizados para autorización y autenticación, respectivamente. Son esenciales para asegurar aplicaciones web modernas y APIs.

  • OAuth2: Un marco de autorización que permite a aplicaciones de terceros obtener acceso limitado a los recursos de un usuario sin exponer credenciales.
  • OpenID Connect: Una capa de identidad sobre OAuth2 que verifica la identidad del usuario final basada en la autenticación realizada por un servidor de autorización.

Configuración de Spring Boot para OAuth2

Para comenzar a usar OAuth2 en una aplicación Spring Boot, necesitas incluir las dependencias necesarias y configurar la aplicación.

Añadiendo Dependencias

Añade las siguientes dependencias a tu archivo pom.xml:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-oauth2-client</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>

Configuración del Cliente OAuth2

Crea un archivo application.yml para configurar tu cliente OAuth2:

spring:
  security:
    oauth2:
      client:
        registration:
          google:
            client-id: YOUR_CLIENT_ID
            client-secret: YOUR_CLIENT_SECRET
            scope: profile, email
            redirect-uri: "{baseUrl}/login/oauth2/code/{registrationId}"
        provider:
          google:
            authorization-uri: https://accounts.google.com/o/oauth2/auth
            token-uri: https://oauth2.googleapis.com/token
            user-info-uri: https://www.googleapis.com/oauth2/v3/userinfo
            user-name-attribute: sub

Implementación del Inicio de Sesión con OAuth2

Para implementar el inicio de sesión con OAuth2, necesitas crear una clase de configuración de seguridad.

Configuración de Seguridad

Crea una clase SecurityConfig:

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;
import org.springframework.security.oauth2.client.oidc.userinfo.OidcUserService;
import org.springframework.security.oauth2.client.registration.ClientRegistrationRepository;
import org.springframework.security.oauth2.client.web.OAuth2LoginAuthenticationFilter;
import org.springframework.security.oauth2.client.web.OAuth2LoginAuthenticationProvider;

@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/", "/login**").permitAll()
                .anyRequest().authenticated()
                .and()
            .oauth2Login()
                .userInfoEndpoint()
                .oidcUserService(oidcUserService());
    }

    @Bean
    public OidcUserService oidcUserService() {
        return new OidcUserService();
    }
}

Entendiendo OpenID Connect

OpenID Connect extiende OAuth2 para proporcionar autenticación. Utiliza tokens de ID para transmitir información de identidad del usuario.

Estructura del Token de ID

Un token de ID es un JSON Web Token (JWT) que contiene información del usuario. Tiene tres partes:

  1. Encabezado: Contiene metadatos sobre el token.
  2. Cuerpo: Contiene información del usuario y reclamaciones.
  3. Firma: Verifica la autenticidad del token.

Ejemplo de Cuerpo de un Token de ID

{
  "iss": "https://accounts.google.com",
  "sub": "110169484474386276334",
  "aud": "YOUR_CLIENT_ID",
  "iat": 1516239022,
  "exp": 1516242622,
  "email": "user@example.com",
  "email_verified": true,
  "name": "John Doe",
  "picture": "https://lh3.googleusercontent.com/a-/AOh14Gg",
  "given_name": "John",
  "family_name": "Doe",
  "locale": "en"
}

Configuración y Personalización Avanzada

Para casos de uso avanzados, puede que necesites personalizar el comportamiento de OAuth2 y OpenID Connect.

Servicio de Usuario OAuth2 Personalizado

Puedes crear un servicio de usuario OAuth2 personalizado para procesar la información del usuario:

import org.springframework.security.oauth2.client.userinfo.DefaultOAuth2UserService;
import org.springframework.security.oauth2.core.user.OAuth2User;
import org.springframework.security.oauth2.core.user.OAuth2UserAuthority;
import org.springframework.security.oauth2.core.OAuth2AuthenticationException;
import org.springframework.security.oauth2.client.userinfo.OAuth2UserRequest;
import org.springframework.security.oauth2.client.userinfo.OAuth2UserService;
import org.springframework.stereotype.Service;

import java.util.Collections;
import java.util.Map;

@Service
public class CustomOAuth2UserService implements OAuth2UserService<OAuth2UserRequest, OAuth2User> {

    @Override
    public OAuth2User loadUser(OAuth2UserRequest userRequest) throws OAuth2AuthenticationException {
        OAuth2UserService<OAuth2UserRequest, OAuth2User> delegate = new DefaultOAuth2UserService();
        OAuth2User oAuth2User = delegate.loadUser(userRequest);

        Map<String, Object> attributes = oAuth2User.getAttributes();
        return new DefaultOAuth2User(
                Collections.singleton(new OAuth2UserAuthority(attributes)),
                attributes,
                "name");
    }
}

Configuración de Seguridad Personalizada

Actualiza el SecurityConfig para usar el servicio de usuario personalizado:

@Override
protected void configure(HttpSecurity http) throws Exception {
    http
        .authorizeRequests()
            .antMatchers("/", "/login**").permitAll()
            .anyRequest().authenticated()
            .and()
        .oauth2Login()
            .userInfoEndpoint()
            .userService(customOAuth2UserService());
}

@Bean
public CustomOAuth2UserService customOAuth2UserService() {
    return new CustomOAuth2UserService();
}

Conclusión

OAuth2 y OpenID Connect son protocolos poderosos para asegurar aplicaciones. Al integrarlos con Spring Boot, puedes implementar fácilmente mecanismos seguros de autenticación y autorización. Este tema cubrió los conceptos básicos de la configuración de OAuth2 y OpenID Connect, la implementación del inicio de sesión y la personalización del comportamiento para casos de uso avanzados. Con este conocimiento, puedes construir aplicaciones seguras y robustas usando Spring Boot.
