Introducción a los Servicios en Angular

Los servicios en Angular son objetos singleton que se instancian solo una vez durante la vida de una aplicación. Se utilizan para compartir datos, lógica y funciones entre diferentes componentes de una aplicación Angular.

Conceptos Clave

  • Servicio: Una clase con un propósito específico, a menudo utilizada para manejar operaciones de datos, lógica de negocio u otras funcionalidades reutilizables.
  • Inyección de Dependencias (DI): Un patrón de diseño utilizado para implementar IoC (Inversión de Control), permitiendo que una clase reciba sus dependencias de una fuente externa en lugar de crearlas por sí misma.
  • Singleton: Un patrón de diseño que restringe la instanciación de una clase a una única instancia.

Creación de un Servicio Básico

Para crear un servicio en Angular, puedes usar Angular CLI o crear el archivo del servicio manualmente.

Usando Angular CLI

ng generate service my-service

Creación Manual de un Servicio

  1. Crea un nuevo archivo my-service.service.ts.
  2. Define la clase del servicio y decórala con el decorador @Injectable.
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class MyService {
  constructor() { }

  getGreeting(): string {
    return '¡Hola, Mundo!';
  }
}

Explicación

  • @Injectable({ providedIn: 'root' }): Este decorador marca la clase como un servicio que puede ser inyectado. La sintaxis providedIn: 'root' asegura que el servicio esté disponible en toda la aplicación como un singleton.

Uso de un Servicio en un Componente

Para usar el servicio en un componente, necesitas inyectarlo en el constructor del componente.

Ejemplo de Componente

  1. Crea un nuevo componente my-component.component.ts.
  2. Inyecta el servicio en el componente.
import { Component, OnInit } from '@angular/core';
import { MyService } from './my-service.service';

@Component({
  selector: 'app-my-component',
  template: `<h1>{{ greeting }}</h1>`,
})
export class MyComponent implements OnInit {
  greeting: string;

  constructor(private myService: MyService) { }

  ngOnInit(): void {
    this.greeting = this.myService.getGreeting();
  }
}

Explicación

  • constructor(private myService: MyService): Esta sintaxis inyecta el MyService en el componente.
  • ngOnInit(): Gancho de ciclo de vida donde llamamos al método del servicio para obtener el mensaje de saludo.

Características Avanzadas del Servicio

Manejo de Solicitudes HTTP

Los servicios a menudo se utilizan para manejar solicitudes HTTP para interactuar con APIs de backend.

Ejemplo: Solicitud HTTP GET

  1. Importa HttpClientModule en app.module.ts.
import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [
    HttpClientModule,
    // otros imports
  ],
})
export class AppModule { }
  1. Actualiza el servicio para usar HttpClient.
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root',
})
export class MyService {
  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) { }

  getData(): Observable<any> {
    return this.http.get<any>(this.apiUrl);
  }
}
  1. Usa el servicio en un componente para obtener datos.
import { Component, OnInit } from '@angular/core';
import { MyService } from './my-service.service';

@Component({
  selector: 'app-my-component',
  template: `<div *ngIf="data">{{ data | json }}</div>`,
})
export class MyComponent implements OnInit {
  data: any;

  constructor(private myService: MyService) { }

  ngOnInit(): void {
    this.myService.getData().subscribe(response => {
      this.data = response;
    });
  }
}

Explicación

  • HttpClientModule: Módulo necesario para hacer solicitudes HTTP.
  • HttpClient: Servicio incorporado de Angular para hacer solicitudes HTTP.
  • Observable: Utilizado para manejar flujos de datos asíncronos.

Conclusión

Los servicios en Angular son herramientas poderosas para compartir datos y lógica entre componentes. Aprovechando el sistema de inyección de dependencias de Angular, los servicios pueden ser fácilmente creados y utilizados en toda la aplicación. Ya sea manejando operaciones de datos simples o solicitudes HTTP complejas, los servicios ayudan a mantener tu código modular y mantenible.
