Formularios basados en plantillas en Angular 2+

Introducción

Los formularios basados en plantillas son una característica poderosa en Angular que permite a los desarrolladores crear y gestionar formularios utilizando la sintaxis de plantillas de Angular. Este enfoque es particularmente útil para formularios más simples y proporciona una forma más declarativa de construir formularios en comparación con los formularios reactivos.

Conceptos clave

  • FormsModule: El módulo de Angular que necesita ser importado para usar formularios basados en plantillas.
  • Directiva ngModel: Vincula los controles del formulario a las propiedades en la clase del componente.
  • Validación de formularios: Validadores integrados y validadores personalizados para asegurar la integridad de los datos del formulario.
  • Envío de formularios: Manejo del envío de formularios y acceso a los datos del formulario.

Configuración de formularios basados en plantillas

Para comenzar a usar formularios basados en plantillas, necesitas importar el FormsModule en tu módulo de Angular.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { FormsModule } from '@angular/forms';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, FormsModule],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Creación de un formulario básico

Aquí tienes un ejemplo simple de un formulario basado en plantillas:

Plantilla HTML

<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Nombre:</label>
    <input type="text" id="name" name="name" ngModel required>
  </div>
  <div>
    <label for="email">Correo electrónico:</label>
    <input type="email" id="email" name="email" ngModel required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>

Clase del componente

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html'
})
export class AppComponent {
  onSubmit(form: any) {
    console.log('¡Formulario enviado!', form);
  }
}

Validación de formularios

Angular proporciona varios validadores integrados para asegurar la integridad de los datos del formulario. Estos incluyen required, minlength, maxlength, pattern y email.

Ejemplo con validación

<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Nombre:</label>
    <input type="text" id="name" name="name" ngModel required minlength="3">
    <div *ngIf="userForm.controls.name?.invalid && userForm.controls.name?.touched">
      <small *ngIf="userForm.controls.name?.errors?.required">El nombre es obligatorio.</small>
      <small *ngIf="userForm.controls.name?.errors?.minlength">El nombre debe tener al menos 3 caracteres.</small>
    </div>
  </div>
  <div>
    <label for="email">Correo electrónico:</label>
    <input type="email" id="email" name="email" ngModel required email>
    <div *ngIf="userForm.controls.email?.invalid && userForm.controls.email?.touched">
      <small *ngIf="userForm.controls.email?.errors?.required">El correo electrónico es obligatorio.</small>
      <small *ngIf="userForm.controls.email?.errors?.email">Dirección de correo electrónico no válida.</small>
    </div>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>

Validadores personalizados

También puedes crear validadores personalizados para manejar lógica de validación más compleja.

Ejemplo de validador personalizado

import { Directive } from '@angular/core';
import { NG_VALIDATORS, Validator, AbstractControl, ValidationErrors } from '@angular/forms';

@Directive({
  selector: '[appCustomValidator]',
  providers: [{ provide: NG_VALIDATORS, useExisting: CustomValidatorDirective, multi: true }]
})
export class CustomValidatorDirective implements Validator {
  validate(control: AbstractControl): ValidationErrors | null {
    const isValid = /* tu lógica de validación personalizada */;
    return isValid ? null : { customError: true };
  }
}

Uso del validador personalizado en la plantilla

<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="customField">Campo personalizado:</label>
    <input type="text" id="customField" name="customField" ngModel appCustomValidator>
    <div *ngIf="userForm.controls.customField?.invalid && userForm.controls.customField?.touched">
      <small *ngIf="userForm.controls.customField?.errors?.customError">Error de validación personalizada.</small>
    </div>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>

Envío de formularios

El manejo del envío de formularios implica capturar los datos del formulario y procesarlos según sea necesario.

Ejemplo de envío de formularios

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html'
})
export class AppComponent {
  onSubmit(form: any) {
    console.log('Datos del formulario:', form.value);
    // Procesar los datos del formulario aquí
  }
}

Conclusión

Los formularios basados en plantillas en Angular proporcionan una forma sencilla de crear y gestionar formularios utilizando la sintaxis de plantillas de Angular. Son ideales para formularios más simples y ofrecen características de validación integradas. Al comprender los conceptos clave y seguir los ejemplos proporcionados, puedes utilizar eficazmente los formularios basados en plantillas en tus aplicaciones Angular.
