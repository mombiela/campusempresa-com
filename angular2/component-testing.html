Las pruebas de componentes son un aspecto crucial para desarrollar aplicaciones robustas en Angular. Este tema te guiará a través de los fundamentos de las pruebas de componentes, desde la configuración de tu entorno hasta la escritura y ejecución de pruebas para tus componentes de Angular.

Introducción a las Pruebas de Componentes

Las pruebas de componentes implican probar componentes individuales de tu aplicación Angular para asegurarse de que funcionen como se espera. Este tipo de pruebas ayuda a detectar errores temprano y garantiza que cada componente funcione correctamente en aislamiento.

Conceptos Clave

  • Pruebas Unitarias: Pruebas de unidades o componentes individuales en aislamiento.
  • Test Bed: La API principal de Angular para probar componentes.
  • Fixtures: Un contenedor alrededor de un componente y su plantilla.
  • Spies: Funciones simuladas que rastrean llamadas y argumentos.

Configuración del Entorno de Pruebas

Antes de que puedas comenzar a escribir pruebas, necesitas configurar tu entorno de pruebas.

Instalación de Dependencias

Angular CLI viene con soporte integrado para pruebas usando Jasmine y Karma. Asegúrate de tener los paquetes necesarios instalados:

ng new my-angular-app
cd my-angular-app
ng add @angular/cli

Configuración de Archivos de Prueba

Angular CLI genera un archivo src/test.ts, que es el punto de entrada principal para ejecutar pruebas. Asegúrate de que tu archivo angular.json incluya la siguiente configuración:

"test": {
  "builder": "@angular-devkit/build-angular:karma",
  "options": {
    "main": "src/test.ts",
    "polyfills": "src/polyfills.ts",
    "tsConfig": "src/tsconfig.spec.json",
    "karmaConfig": "src/karma.conf.js"
  }
}

Escribiendo Tu Primera Prueba de Componente

Vamos a escribir una prueba simple para un componente.

Componente de Ejemplo

Considera un componente básico de Angular:

// src/app/hello.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-hello',
  template: '<h1>Hello, {{name}}!</h1>'
})
export class HelloComponent {
  name: string = 'World';
}

Archivo de Prueba

Crea un archivo de prueba para el componente:

// src/app/hello.component.spec.ts
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { HelloComponent } from './hello.component';

describe('HelloComponent', () => {
  let component: HelloComponent;
  let fixture: ComponentFixture<HelloComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ HelloComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(HelloComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should display the correct name', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('h1').textContent).toContain('Hello, World!');
  });
});

Explicación

  • Configuración de TestBed: Configura el entorno de pruebas para el componente.
  • Fixture: Proporciona acceso a la instancia del componente y su plantilla.
  • Pruebas Básicas: Verifica si el componente se crea y si muestra el nombre correcto.

Pruebas Avanzadas de Componentes

A medida que te sientas más cómodo con las pruebas básicas, puedes comenzar a explorar escenarios más avanzados.

Pruebas de Interacciones de Componentes

Los componentes a menudo interactúan con servicios. Puedes usar spies para simular estas interacciones.

Servicio de Ejemplo

// src/app/data.service.ts
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  getData() {
    return 'Real Data';
  }
}

Componente Usando el Servicio

// src/app/data.component.ts
import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  template: '<p>{{data}}</p>'
})
export class DataComponent implements OnInit {
  data: string;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.data = this.dataService.getData();
  }
}

Archivo de Prueba con Spies

// src/app/data.component.spec.ts
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { DataComponent } from './data.component';
import { DataService } from './data.service';

class MockDataService {
  getData() {
    return 'Mock Data';
  }
}

describe('DataComponent', () => {
  let component: DataComponent;
  let fixture: ComponentFixture<DataComponent>;
  let dataService: DataService;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ DataComponent ],
      providers: [{ provide: DataService, useClass: MockDataService }]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(DataComponent);
    component = fixture.componentInstance;
    dataService = TestBed.inject(DataService);
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should display mock data', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('p').textContent).toContain('Mock Data');
  });
});

Explicación

  • Servicio Simulado: Una implementación simulada del DataService.
  • Reemplazo de Proveedor: Reemplaza el servicio real con el servicio simulado en el test bed.
  • Prueba: Verifica que el componente muestra datos del servicio simulado.

Conclusión

Las pruebas de componentes en Angular son esenciales para garantizar la fiabilidad y corrección de tu aplicación. Al configurar un entorno de pruebas adecuado, escribir pruebas básicas y avanzar a escenarios más complejos, puedes detectar errores temprano y mantener una alta calidad de código. Recuerda:

  • Comienza con pruebas simples para verificar la creación del componente y su funcionalidad básica.
  • Usa mocks y spies para probar interacciones con servicios.
  • Expande continuamente tu cobertura de pruebas a medida que tu aplicación crece.

Con estas prácticas, estarás bien equipado para escribir pruebas de componentes efectivas en Angular.
