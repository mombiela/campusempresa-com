Introducción

En esta sección, aprenderás a crear tu primera aplicación Angular. Cubriremos los siguientes conceptos clave:

  • Configuración del entorno Angular
  • Creación de un nuevo proyecto Angular
  • Comprensión de la estructura del proyecto
  • Ejecución de la aplicación Angular
  • Componentes básicos de Angular

Configuración del Entorno Angular

Antes de que puedas crear una aplicación Angular, necesitas configurar tu entorno de desarrollo.

Requisitos Previos

  • Node.js y npm: Angular requiere Node.js y npm (Node Package Manager). Puedes descargarlos e instalarlos desde nodejs.org.
  • Angular CLI: La Interfaz de Línea de Comandos de Angular (CLI) es una herramienta poderosa para inicializar, desarrollar, estructurar y mantener aplicaciones Angular.

Instalación de Angular CLI

Para instalar Angular CLI, abre tu terminal o símbolo del sistema y ejecuta:

npm install -g @angular/cli

Creación de un Nuevo Proyecto Angular

Una vez que Angular CLI esté instalado, puedes crear un nuevo proyecto Angular.

Pasos para Crear un Nuevo Proyecto

  1. Abre tu terminal y navega al directorio donde deseas crear tu proyecto.
  2. Ejecuta el siguiente comando para crear un nuevo proyecto Angular: 
    ng new my-first-angular-app
  3. Navega al directorio del proyecto: 
    cd my-first-angular-app

Comprensión de la Estructura del Proyecto

Cuando creas un nuevo proyecto Angular, Angular CLI genera un conjunto de archivos y carpetas. Aquí tienes una breve descripción de los componentes principales:

  • e2e/: Pruebas de extremo a extremo para la aplicación.
  • node_modules/: Contiene los paquetes npm instalados para el proyecto.
  • src/: El código fuente de la aplicación.
    • app/: Contiene el código principal de la aplicación.
    • assets/: Contiene activos estáticos como imágenes y estilos.
    • environments/: Contiene archivos de configuración específicos del entorno.
  • angular.json: Archivo de configuración para Angular CLI.
  • package.json: Lista los paquetes npm de los que depende el proyecto.
  • tsconfig.json: Archivo de configuración de TypeScript.

Ejecución de la Aplicación Angular

Para ejecutar tu aplicación Angular, usa Angular CLI:

ng serve

Este comando compilará la aplicación y arrancará un servidor de desarrollo. Por defecto, la aplicación estará disponible en http://localhost:4200/.

Componentes Básicos de Angular

Las aplicaciones Angular se construyen utilizando componentes. Un componente en Angular es una clase TypeScript con una plantilla HTML y estilos.

Creación de un Nuevo Componente

Puedes crear un nuevo componente usando Angular CLI:

ng generate component my-component

Este comando generará un nuevo componente con los siguientes archivos:

  • my-component.component.ts: El archivo TypeScript que contiene la lógica del componente.
  • my-component.component.html: La plantilla HTML para el componente.
  • my-component.component.css: Los estilos para el componente.
  • my-component.component.spec.ts: El archivo de pruebas para el componente.

Componente de Ejemplo

Aquí tienes un ejemplo de un componente Angular simple:

app.component.ts

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'my-first-angular-app';
}

app.component.html

<div style="text-align:center">
  <h1>
    ¡Bienvenido a {{ title }}!
  </h1>
</div>

Explicación

  • @Component: Este decorador marca la clase como un componente Angular y proporciona metadatos sobre el componente.
  • selector: La etiqueta HTML personalizada que representa este componente.
  • templateUrl: La ruta al archivo de plantilla HTML.
  • styleUrls: La ruta al(los) archivo(s) CSS.

Conclusión

¡Felicidades! Has creado con éxito tu primera aplicación Angular. Aprendiste cómo configurar el entorno Angular, crear un nuevo proyecto, comprender la estructura del proyecto, ejecutar la aplicación y crear un componente Angular básico. Este conocimiento fundamental te ayudará a medida que continúes construyendo aplicaciones Angular más complejas.
