Pruebas Unitarias en Angular 2+
Pruebas de Componentes ►

Las pruebas unitarias son una parte crítica del desarrollo de aplicaciones robustas y mantenibles. En esta sección, cubriremos los conceptos básicos de las pruebas unitarias en Angular 2+ y progresaremos hacia temas más avanzados.

Introducción a las Pruebas Unitarias

  • Definición: Las pruebas unitarias implican probar componentes individuales o fragmentos de código para asegurarse de que funcionen como se espera.
  • Importancia: Ayuda a identificar errores temprano, facilita la refactorización y asegura la fiabilidad del código.

Configuración de Pruebas Unitarias en Angular

Angular utiliza Jasmine para escribir pruebas y Karma como el ejecutor de pruebas. Estas herramientas están integradas en Angular CLI.

Instalación de Angular CLI

npm install -g @angular/cli

Creación de un Nuevo Proyecto Angular

ng new my-angular-app
cd my-angular-app

Ejecución de Pruebas

ng test

Este comando iniciará el ejecutor de pruebas Karma y ejecutará todas las pruebas en tu proyecto.

Escribiendo Tu Primera Prueba

Estructura Básica de una Prueba

Una prueba básica en Angular se ve así:

import { TestBed } from '@angular/core/testing';
import { AppComponent } from './app.component';

describe('AppComponent', () => {
  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [
        AppComponent
      ],
    }).compileComponents();
  });

  it('should create the app', () => {
    const fixture = TestBed.createComponent(AppComponent);
    const app = fixture.componentInstance;
    expect(app).toBeTruthy();
  });
});
  • describe: Define un conjunto de pruebas.
  • beforeEach: Se ejecuta antes de cada prueba en el conjunto.
  • it: Define una sola prueba.
  • expect: Hace una afirmación sobre el código.

Pruebas de Componentes

Ejemplo de Prueba de Componente

Vamos a probar un componente simple:

import { Component } from '@angular/core';

@Component({
  selector: 'app-hello',
  template: '<h1>Hello, {{name}}!</h1>'
})
export class HelloComponent {
  name: string = 'World';
}

Escribiendo Pruebas para el Componente

import { TestBed, ComponentFixture } from '@angular/core/testing';
import { HelloComponent } from './hello.component';

describe('HelloComponent', () => {
  let component: HelloComponent;
  let fixture: ComponentFixture<HelloComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ HelloComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(HelloComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should have the default name as "World"', () => {
    expect(component.name).toBe('World');
  });

  it('should render the name in the template', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('h1').textContent).toContain('Hello, World!');
  });
});

Pruebas de Servicios

Ejemplo de Prueba de Servicio

Vamos a probar un servicio simple:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  getData() {
    return 'data';
  }
}

Escribiendo Pruebas para el Servicio

import { TestBed } from '@angular/core/testing';
import { DataService } from './data.service';

describe('DataService', () => {
  let service: DataService;

  beforeEach(() => {
    TestBed.configureTestingModule({});
    service = TestBed.inject(DataService);
  });

  it('should be created', () => {
    expect(service).toBeTruthy();
  });

  it('should return "data"', () => {
    expect(service.getData()).toBe('data');
  });
});

Temas Avanzados

Simulación de Dependencias

Al probar componentes o servicios que tienen dependencias, es posible que necesites simular esas dependencias.

Simulación de un Servicio

import { TestBed } from '@angular/core/testing';
import { DataService } from './data.service';
import { AnotherService } from './another.service';

class MockAnotherService {
  getValue() {
    return 'mock value';
  }
}

describe('DataService with Mock', () => {
  let service: DataService;

  beforeEach(() => {
    TestBed.configureTestingModule({
      providers: [
        DataService,
        { provide: AnotherService, useClass: MockAnotherService }
      ]
    });
    service = TestBed.inject(DataService);
  });

  it('should use the mock service', () => {
    const anotherService = TestBed.inject(AnotherService);
    expect(anotherService.getValue()).toBe('mock value');
  });
});

Pruebas de Código Asíncrono

Angular proporciona utilidades para probar código asíncrono, como fakeAsync y tick.

Ejemplo

import { fakeAsync, tick } from '@angular/core/testing';

it('should handle asynchronous code', fakeAsync(() => {
  let value = false;
  setTimeout(() => {
    value = true;
  }, 1000);
  tick(1000);
  expect(value).toBe(true);
}));

Resumen

Las pruebas unitarias son una práctica esencial en el desarrollo de Angular. Al escribir pruebas para componentes, servicios y manejar código asíncrono, puedes asegurar que tu aplicación sea fiable y mantenible. Comienza con pruebas básicas y gradualmente incorpora técnicas más avanzadas como la simulación de dependencias y la prueba de código asíncrono para cubrir todos los aspectos de tu aplicación.
