Inyectores Jerárquicos en Angular 2+
Configuración de Rutas ►

En Angular, la inyección de dependencias (DI) es un patrón poderoso utilizado para gestionar cómo se instancian e interactúan los componentes y servicios. Los inyectores jerárquicos son un concepto clave en el sistema de DI de Angular, permitiendo una gestión de dependencias flexible y eficiente.

Conceptos Clave

  • Inyección de Dependencias (DI): Un patrón de diseño en el cual una clase solicita dependencias de fuentes externas en lugar de crearlas.
  • Inyector: Un localizador de servicios que es responsable de proporcionar instancias de dependencias.
  • Inyectores Jerárquicos: El sistema de DI de Angular es jerárquico, lo que significa que los inyectores pueden anidarse, y los inyectores hijos pueden anular las dependencias proporcionadas por los inyectores padres.

Cómo Funcionan los Inyectores Jerárquicos

Los inyectores de Angular forman una estructura de árbol que se asemeja al árbol de componentes. Cada componente tiene su propio inyector, y estos inyectores están organizados jerárquicamente.

  • Inyector Raíz: El inyector de nivel superior que se crea cuando la aplicación inicia. Proporciona servicios singleton que están disponibles en toda la aplicación.
  • Inyectores Hijos: Inyectores que se crean para cada componente. Estos pueden anular los servicios proporcionados por los inyectores padres.

Ejemplo: Inyector Jerárquico Básico

Comencemos con un ejemplo simple para ilustrar cómo funcionan los inyectores jerárquicos.

app.module.ts

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { ParentComponent } from './parent/parent.component';
import { ChildComponent } from './child/child.component';
import { MyService } from './my.service';

@NgModule({
  declarations: [
    AppComponent,
    ParentComponent,
    ChildComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [MyService],
  bootstrap: [AppComponent]
})
export class AppModule { }

my.service.ts

import { Injectable } from '@angular/core';

@Injectable()
export class MyService {
  getValue(): string {
    return 'Valor del Servicio desde el Inyector Raíz';
  }
}

parent.component.ts

import { Component } from '@angular/core';
import { MyService } from '../my.service';

@Component({
  selector: 'app-parent',
  template: `
    <div>
      Componente Padre: {{ value }}
      <app-child></app-child>
    </div>
  `,
  providers: [MyService] // Esto creará una nueva instancia de MyService para este componente y sus hijos
})
export class ParentComponent {
  value: string;

  constructor(private myService: MyService) {
    this.value = this.myService.getValue();
  }
}

child.component.ts

import { Component } from '@angular/core';
import { MyService } from '../.my.service';

@Component({
  selector: 'app-child',
  template: `
    <div>
      Componente Hijo: {{ value }}
    </div>
  `
})
export class ChildComponent {
  value: string;

  constructor(private myService: MyService) {
    this.value = this.myService.getValue();
  }
}

Explicación

  • Inyector Raíz: El MyService proporcionado en AppModule está disponible en toda la aplicación.
  • Inyector Padre: El MyService proporcionado en ParentComponent anula el MyService del inyector raíz para el ParentComponent y sus hijos.
  • Inyector Hijo: El ChildComponent usa el MyService proporcionado por su padre, ParentComponent.

En este ejemplo, tanto el ParentComponent como el ChildComponent usarán la instancia de MyService proporcionada por el inyector del ParentComponent, no el inyector raíz.

Uso Avanzado

Proporcionando Diferentes Instancias

Puedes proporcionar diferentes instancias de un servicio en diferentes niveles de la jerarquía de inyectores.

Ejemplo: Diferentes Instancias

// parent.component.ts
@Component({
  selector: 'app-parent',
  template: `
    <div>
      Componente Padre: {{ value }}
      <app-child></app-child>
    </div>
  `,
  providers: [{ provide: MyService, useClass: ParentService }]
})
export class ParentComponent {
  value: string;

  constructor(private myService: MyService) {
    this.value = this.myService.getValue();
  }
}

// parent.service.ts
@Injectable()
export class ParentService extends MyService {
  getValue(): string {
    return 'Valor del Servicio desde el Inyector Padre';
  }
}

En este ejemplo, ParentComponent usa ParentService, mientras que ChildComponent usa la misma instancia proporcionada por ParentComponent.

Resumen

Los inyectores jerárquicos en Angular permiten una gestión de dependencias flexible y eficiente al crear un árbol de inyectores que se asemeja al árbol de componentes. Esto permite:

  • Servicios con Alcance: Los servicios pueden tener alcance a componentes o módulos específicos.
  • Anulación de Servicios: Los inyectores hijos pueden anular los servicios proporcionados por los inyectores padres.
  • Gestión Eficiente de Recursos: Los servicios singleton pueden ser compartidos en toda la aplicación, mientras que otros servicios pueden ser instanciados según sea necesario.

Entender los inyectores jerárquicos es crucial para construir aplicaciones Angular escalables y mantenibles. Al aprovechar esta poderosa característica, puedes asegurar que tus servicios se proporcionen de la manera más eficiente y lógica.
