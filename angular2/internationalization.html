Introducción a la Internacionalización (i18n)

La internacionalización (i18n) es el proceso de diseñar y preparar tu aplicación para que sea adaptable a varios idiomas y regiones sin requerir cambios de ingeniería en el código fuente. Angular proporciona un soporte robusto para i18n, lo que facilita la creación de aplicaciones que pueden ser traducidas a diferentes idiomas.

Conceptos Clave de i18n

  • Localización (l10n): El proceso de traducir tu aplicación internacionalizada a un idioma y región específicos.
  • Archivos de Traducción: Archivos que contienen el texto traducido para diferentes idiomas.
  • Herramientas de i18n de Angular: Herramientas proporcionadas por Angular para extraer, gestionar y aplicar traducciones.

Configuración de i18n en Angular

Paso 1: Marcar Texto para Traducción

Angular proporciona el atributo i18n para marcar texto para traducción en tus plantillas.

<h1 i18n="@@homeTitle">¡Bienvenido a nuestra aplicación!</h1>
<p i18n="@@homeDescription">Esta es una aplicación de ejemplo para demostrar i18n en Angular.</p>
  • i18n="@@homeTitle": El atributo i18n marca el texto para traducción.
  • @@homeTitle: Un ID personalizado opcional para la traducción.

Paso 2: Extraer Mensajes de Traducción

Usa Angular CLI para extraer el texto marcado en un archivo de traducción.

ng extract-i18n --output-path src/locale

Este comando genera un archivo messages.xlf en la ruta de salida especificada.

Paso 3: Traducir Mensajes

Traduce los mensajes extraídos a los idiomas deseados. Por ejemplo, crea messages.fr.xlf para traducciones al francés.

<trans-unit id="homeTitle" datatype="html">
  <source>Welcome to our application!</source>
  <target>Bienvenue dans notre application!</target>
</trans-unit>
<trans-unit id="homeDescription" datatype="html">
  <source>This is a sample application to demonstrate i18n in Angular.</source>
  <target>Ceci est une application d'exemple pour démontrer i18n dans Angular.</target>
</trans-unit>

Paso 4: Configurar Angular para Usar Traducciones

Modifica angular.json para incluir los archivos de traducción y configurar las opciones de compilación.

"projects": {
  "your-app-name": {
    "i18n": {
      "sourceLocale": "en",
      "locales": {
        "fr": "src/locale/messages.fr.xlf"
      }
    },
    ...
  }
}

Paso 5: Compilar y Servir la Aplicación

Compila la aplicación para una localización específica.

ng build --localize

Sirve la aplicación para una localización específica.

ng serve --configuration=fr

Funciones Avanzadas de i18n

Pluralización

Maneja formas plurales en traducciones usando la sintaxis de mensajes ICU de Angular.

<p i18n="@@itemCount">
  {itemCount, plural, 
    =0 {No items}
    =1 {One item}
    other {# items}
  }
</p>

Parámetros Dinámicos

Incluye valores dinámicos en las traducciones.

<p i18n="@@welcomeUser">¡Bienvenido, {{ userName }}!</p>

Datos de Localización Personalizados

Agrega datos de localización personalizados para idiomas o regiones específicos.

import { registerLocaleData } from '@angular/common';
import localeFr from '@angular/common/locales/fr';

registerLocaleData(localeFr, 'fr');

Conclusión

La internacionalización en Angular 2+ es una característica poderosa que te permite crear aplicaciones que pueden ser fácilmente adaptadas a diferentes idiomas y regiones. Siguiendo los pasos descritos en este tema, puedes marcar texto para traducción, extraer y traducir mensajes, y configurar tu aplicación para usar estas traducciones. Funciones avanzadas como la pluralización y los parámetros dinámicos mejoran aún más la flexibilidad de tu aplicación internacionalizada.
