Introducción a la Arquitectura de Angular

Angular es una plataforma y un framework para construir aplicaciones cliente de una sola página utilizando HTML y TypeScript. Angular está escrito en TypeScript y proporciona un conjunto de herramientas y bibliotecas para construir y gestionar aplicaciones Angular.

Conceptos Clave de la Arquitectura de Angular

  • Módulos: Las aplicaciones Angular son modulares y utilizan NgModules para organizar una aplicación en bloques cohesivos de funcionalidad.
  • Componentes: Los bloques de construcción básicos de una aplicación Angular. Cada componente consta de una plantilla HTML, una hoja de estilos CSS y una clase TypeScript.
  • Plantillas: Definen la vista para un componente utilizando HTML.
  • Servicios e Inyección de Dependencias: Los servicios se utilizan para compartir datos y lógica entre componentes. Angular utiliza la inyección de dependencias para proporcionar servicios a los componentes.
  • Enrutamiento: El enrutador de Angular permite la navegación de una vista a otra a medida que los usuarios realizan tareas en la aplicación.

Módulos de Angular

Los módulos son una forma de agrupar componentes, directivas, pipes y servicios que están relacionados, de tal manera que puedan combinarse con otros módulos para crear una aplicación.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }
  • declarations: Los componentes, directivas y pipes que pertenecen a este módulo.
  • imports: Otros módulos cuyas clases exportadas son necesarias por las plantillas de componentes declaradas en este módulo.
  • providers: Creadores de servicios que este módulo contribuye a la colección global de servicios.
  • bootstrap: La vista principal de la aplicación, llamada el componente raíz, que aloja todas las demás vistas de la aplicación.

Componentes

Los componentes son el bloque de construcción más básico de una aplicación Angular. Controlan una porción de la pantalla llamada vista.

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'my-angular-app';
}
  • selector: El selector de elementos CSS del componente.
  • templateUrl: La ubicación del archivo de plantilla del componente.
  • styleUrls: La ubicación de los estilos CSS privados del componente.

Plantillas

Las plantillas se utilizan para definir la vista de un componente. Utilizan HTML junto con la sintaxis de plantillas de Angular.

<h1>{{ title }}</h1>
<p>¡Bienvenido a {{ title }}!</p>
  • Interpolación: {{ title }} es un ejemplo de interpolación, que permite incrustar expresiones en la plantilla.

Servicios e Inyección de Dependencias

Los servicios se utilizan para compartir datos y lógica entre componentes. Angular utiliza la inyección de dependencias para proporcionar servicios a los componentes.

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class DataService {
  getData() {
    return ['Data1', 'Data2', 'Data3'];
  }
}
  • @Injectable: Marca una clase como disponible para ser proporcionada e inyectada como una dependencia.

Enrutamiento

El enrutador de Angular permite la navegación de una vista a otra a medida que los usuarios realizan tareas en la aplicación.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }
  • Routes: Definen las rutas y los componentes que deben mostrarse para cada ruta.

Resumen

Entender la arquitectura de Angular es crucial para construir aplicaciones eficientes y escalables. Los componentes clave de la arquitectura de Angular incluyen módulos, componentes, plantillas, servicios y enrutamiento. Cada uno de estos elementos juega un papel vital en la estructura y funcionalidad de una aplicación Angular. Al dominar estos conceptos, puedes crear aplicaciones robustas y mantenibles.
