Declaración SELECT
Filtrando Datos con WHERE ►

La declaración SELECT es uno de los comandos más fundamentales y ampliamente utilizados en SQL. Te permite recuperar datos de una o más tablas en una base de datos. Este tema cubrirá la declaración SELECT desde un nivel principiante hasta un nivel avanzado, asegurando que entiendas su sintaxis, uso y varias características avanzadas.

Introducción a SELECT

La declaración SELECT se usa para consultar la base de datos y recuperar datos. La sintaxis básica de una declaración SELECT es:

SELECT column1, column2, ...
FROM table_name;

Uso Básico

  • Seleccionar Todas las Columnas: Para seleccionar todas las columnas de una tabla, usa el asterisco (*).
SELECT * FROM employees;
  • Seleccionar Columnas Específicas: Para seleccionar columnas específicas, enumera los nombres de las columnas separados por comas.
SELECT first_name, last_name FROM employees;

Filtrar Datos con WHERE

La cláusula WHERE se usa para filtrar registros que cumplen ciertas condiciones.

SELECT first_name, last_name
FROM employees
WHERE department = 'Sales';

Ordenar Datos con ORDER BY

La cláusula ORDER BY se usa para ordenar el conjunto de resultados en orden ascendente o descendente.

SELECT first_name, last_name
FROM employees
ORDER BY last_name ASC;
  • Orden Ascendente: ASC (por defecto)
  • Orden Descendente: DESC

Limitar Resultados con LIMIT

La cláusula LIMIT se usa para especificar el número de registros a devolver.

SELECT first_name, last_name
FROM employees
ORDER BY last_name ASC
LIMIT 10;

Características Intermedias de SELECT

Alias

Los alias se usan para dar un nombre temporal a una tabla o una columna.

  • Alias de Columna:
SELECT first_name AS fname, last_name AS lname
FROM employees;
  • Alias de Tabla:
SELECT e.first_name, e.last_name
FROM employees AS e;

Funciones Agregadas

Las funciones agregadas realizan un cálculo sobre un conjunto de valores y devuelven un solo valor.

  • COUNT: Cuenta el número de filas.
SELECT COUNT(*) FROM employees;
  • SUM: Calcula la suma de una columna numérica.
SELECT SUM(salary) FROM employees;
  • AVG: Calcula el valor promedio de una columna numérica.
SELECT AVG(salary) FROM employees;
  • MAX y MIN: Encuentran los valores máximo y mínimo.
SELECT MAX(salary) FROM employees;
SELECT MIN(salary) FROM employees;

GROUP BY

La declaración GROUP BY agrupa filas que tienen los mismos valores en filas de resumen.

SELECT department, COUNT(*)
FROM employees
GROUP BY department;

HAVING

La cláusula HAVING se usa para filtrar grupos basados en una condición.

SELECT department, COUNT(*)
FROM employees
GROUP BY department
HAVING COUNT(*) > 5;

Características Avanzadas de SELECT

Subconsultas

Una subconsulta es una consulta dentro de otra consulta.

  • Subconsulta en SELECT:
SELECT first_name, last_name, (SELECT department FROM departments WHERE departments.id = employees.department_id) AS department
FROM employees;
  • Subconsulta en WHERE:
SELECT first_name, last_name
FROM employees
WHERE department_id = (SELECT id FROM departments WHERE name = 'Sales');

Joins

Los joins se usan para combinar filas de dos o más tablas basadas en una columna relacionada.

  • INNER JOIN: Devuelve registros que tienen valores coincidentes en ambas tablas.
SELECT employees.first_name, employees.last_name, departments.name
FROM employees
INNER JOIN departments ON employees.department_id = departments.id;
  • LEFT JOIN: Devuelve todos los registros de la tabla izquierda y los registros coincidentes de la tabla derecha.
SELECT employees.first_name, employees.last_name, departments.name
FROM employees
LEFT JOIN departments ON employees.department_id = departments.id;
  • RIGHT JOIN: Devuelve todos los registros de la tabla derecha y los registros coincidentes de la tabla izquierda.
SELECT employees.first_name, employees.last_name, departments.name
FROM employees
RIGHT JOIN departments ON employees.department_id = departments.id;
  • FULL OUTER JOIN: Devuelve todos los registros cuando hay una coincidencia en la tabla izquierda o derecha.
SELECT employees.first_name, employees.last_name, departments.name
FROM employees
FULL OUTER JOIN departments ON employees.department_id = departments.id;

UNION

El operador UNION se usa para combinar los conjuntos de resultados de dos o más declaraciones SELECT.

SELECT first_name, last_name FROM employees
UNION
SELECT first_name, last_name FROM managers;

Funciones de Ventana

Las funciones de ventana realizan un cálculo a través de un conjunto de filas de la tabla que están de alguna manera relacionadas con la fila actual.

SELECT first_name, last_name, salary, 
       RANK() OVER (PARTITION BY department_id ORDER BY salary DESC) AS rank
FROM employees;

Conclusión

La declaración SELECT es una herramienta poderosa en SQL que te permite recuperar y manipular datos de diversas maneras. Desde la recuperación básica de datos hasta consultas complejas que involucran joins, subconsultas y funciones de ventana, dominar la declaración SELECT es esencial para cualquier programador de SQL. Practica usando estas características para volverte competente en la consulta de bases de datos de manera efectiva.
