Introducción a las CTEs

Las Expresiones de Tabla Comunes (CTEs) son una característica poderosa en SQL que te permite crear conjuntos de resultados temporales que pueden ser referenciados dentro de una declaración SELECT, INSERT, UPDATE o DELETE. Pueden hacer que las consultas complejas sean más fáciles de leer y mantener.

  • Definición: Una CTE es un conjunto de resultados temporales que se define dentro del ámbito de ejecución de una única declaración SELECT, INSERT, UPDATE o DELETE.
  • Sintaxis: La sintaxis básica para una CTE es la siguiente: 
    WITH cte_name AS (
    -- Consulta de la CTE
    SELECT column1, column2
    FROM table_name
    WHERE condition
)
-- Consulta principal
SELECT column1, column2
FROM cte_name;

Beneficios de Usar CTEs

  • Mejor Legibilidad: Las CTEs pueden descomponer consultas complejas en partes más simples.
  • Reusabilidad: Puedes referenciar la CTE múltiples veces dentro de la consulta principal.
  • Recursión: Las CTEs soportan consultas recursivas, que son útiles para datos jerárquicos.

Ejemplo Básico de una CTE

Comencemos con un ejemplo simple para entender cómo funcionan las CTEs.

Ejemplo: Seleccionando Empleados de un Departamento

Supongamos que tenemos una tabla employees y queremos seleccionar empleados del departamento de 'Ventas'.

WITH SalesEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Sales'
)
SELECT employee_id, first_name, last_name
FROM SalesEmployees;
  • Explicación:
    • La CTE SalesEmployees se define para seleccionar empleados del departamento de 'Ventas'.
    • La consulta principal luego selecciona de la CTE SalesEmployees.

Uso Avanzado de CTEs

Múltiples CTEs

Puedes definir múltiples CTEs en una sola consulta separándolas con comas.

WITH SalesEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Sales'
),
MarketingEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Marketing'
)
SELECT * FROM SalesEmployees
UNION ALL
SELECT * FROM MarketingEmployees;

CTEs Recursivas

Las CTEs recursivas son útiles para datos jerárquicos, como organigramas o estructuras de árbol.

Ejemplo: Jerarquía Organizacional

Supongamos que tenemos una tabla employees con una columna manager_id que referencia al gerente del empleado.

WITH RECURSIVE OrgChart AS (
    SELECT employee_id, first_name, last_name, manager_id
    FROM employees
    WHERE manager_id IS NULL
    UNION ALL
    SELECT e.employee_id, e.first_name, e.last_name, e.manager_id
    FROM employees e
    INNER JOIN OrgChart o ON e.manager_id = o.employee_id
)
SELECT * FROM OrgChart;
  • Explicación:
    • El caso base selecciona empleados sin gerente (manager_id IS NULL).
    • La parte recursiva une la tabla employees con la CTE OrgChart para encontrar empleados gestionados por aquellos en el OrgChart.

Ejercicios Prácticos

  1. CTE Básica: Crea una CTE para seleccionar productos con un precio mayor a $100 de una tabla products.
  2. Múltiples CTEs: Crea dos CTEs para seleccionar empleados de los departamentos de 'RRHH' y 'TI', luego combina los resultados.
  3. CTE Recursiva: Crea una CTE recursiva para mostrar una jerarquía de categorías de una tabla categories con category_id y parent_category_id.

Conclusión

Las CTEs son una herramienta versátil en SQL que puede simplificar consultas complejas, mejorar la legibilidad y manejar datos jerárquicos de manera eficiente. Al dominar las CTEs, puedes escribir consultas SQL más mantenibles y eficientes.

  • Puntos Clave:
    • Las CTEs proporcionan una forma de crear conjuntos de resultados temporales.
    • Mejoran la legibilidad y mantenibilidad de las consultas.
    • Las CTEs recursivas son poderosas para trabajar con datos jerárquicos.

Practica usando CTEs en tus consultas SQL para volverte más competente y aprovechar su potencial al máximo.
