Uso de Nombres Lógicos y Símbolos

Introducción

En OpenVMS, los nombres lógicos y los símbolos son herramientas esenciales para gestionar los recursos del sistema y simplificar los procedimientos de comandos. Este tema te guiará a través de los conceptos básicos de los nombres lógicos y los símbolos, avanzando hacia un uso más avanzado.

Nombres Lógicos

¿Qué son los Nombres Lógicos?

  • Los nombres lógicos son alias para especificaciones de archivos, nombres de dispositivos u otras cadenas.
  • Simplifican los procedimientos de comandos y scripts proporcionando una referencia abreviada.

Creación de Nombres Lógicos

  • Usa el comando DEFINE para crear un nombre lógico.
  • Sintaxis: DEFINE logical_name equivalence_name
$ DEFINE MYDISK DKA0:[USER.DIRECTORY]
  • En este ejemplo, MYDISK es un nombre lógico para el directorio DKA0:[USER.DIRECTORY].

Uso de Nombres Lógicos

  • Los nombres lógicos pueden ser usados en lugar de sus nombres equivalentes en los comandos.
$ DIRECTORY MYDISK
  • Este comando lista el contenido de DKA0:[USER.DIRECTORY].

Tipos de Nombres Lógicos

  • Nombres Lógicos de Proceso: Válidos solo dentro del proceso actual.
  • Nombres Lógicos de Trabajo: Válidos para todos los procesos en el trabajo actual.
  • Nombres Lógicos de Grupo: Válidos para todos los procesos en el grupo actual.
  • Nombres Lógicos de Sistema: Válidos para todos los procesos en el sistema.

Gestión de Nombres Lógicos

  • Usa el comando SHOW LOGICAL para mostrar los nombres lógicos.
$ SHOW LOGICAL MYDISK
  • Usa el comando DEASSIGN para eliminar un nombre lógico.
$ DEASSIGN MYDISK

Símbolos

¿Qué son los Símbolos?

  • Los símbolos son nombres definidos por el usuario que representan comandos o secuencias de comandos.
  • Son similares a las variables en otros lenguajes de programación.

Creación de Símbolos

  • Usa el operador = para definir un símbolo.
  • Sintaxis: symbol_name = value
$ LISTDIR = "DIRECTORY"
  • En este ejemplo, LISTDIR es un símbolo para el comando DIRECTORY.

Uso de Símbolos

  • Los símbolos pueden ser usados en lugar de sus valores definidos en los comandos.
$ LISTDIR MYDISK
  • Este comando es equivalente a DIRECTORY MYDISK.

Símbolos Locales vs. Globales

  • Símbolos Locales: Válidos solo dentro del procedimiento de comandos actual.
  • Símbolos Globales: Válidos en todos los procedimientos de comandos en el proceso actual.

Gestión de Símbolos

  • Usa el comando SHOW SYMBOL para mostrar los símbolos.
$ SHOW SYMBOL LISTDIR
  • Usa el comando DELETE para eliminar un símbolo.
$ DELETE SYMBOL LISTDIR

Uso Avanzado

Tablas de Nombres Lógicos

  • Los nombres lógicos se almacenan en tablas de nombres lógicos.
  • Las tablas comunes incluyen LNM$PROCESS_TABLE, LNM$JOB_TABLE, LNM$GROUP_TABLE y LNM$SYSTEM_TABLE.

Traducción de Nombres Lógicos

  • Usa la función léxica F$TRNLNM para traducir nombres lógicos en procedimientos de comandos.
$ MYDIR = F$TRNLNM("MYDISK")
$ DIRECTORY 'MYDIR'

Sustitución de Símbolos

  • Usa comillas simples (') para sustituir símbolos dentro de procedimientos de comandos.
$ MYCOMMAND = "SHOW TIME"
$ 'MYCOMMAND'

Combinación de Nombres Lógicos y Símbolos

  • Los nombres lógicos y los símbolos pueden ser combinados para procedimientos de comandos más complejos.
$ DEFINE MYDISK DKA0:[USER.DIRECTORY]
$ LISTDIR = "DIRECTORY"
$ 'LISTDIR' MYDISK

Conclusión

Los nombres lógicos y los símbolos son herramientas poderosas en OpenVMS que ayudan a agilizar los procedimientos de comandos y gestionar los recursos del sistema de manera eficiente. Al comprender y utilizar estas características, puedes crear scripts más legibles, mantenibles y eficientes. Practica creando y usando nombres lógicos y símbolos para volverte competente en su aplicación.
