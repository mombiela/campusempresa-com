Introducción

Los trabajos por lotes son esenciales para automatizar tareas repetitivas en OpenVMS. Este tema te guiará a través de los conceptos básicos de la creación y gestión de trabajos por lotes, avanzando hacia técnicas más avanzadas.

¿Qué es un Trabajo por Lotes?

  • Definición: Un trabajo por lotes es un proceso no interactivo que se ejecuta en segundo plano.
  • Propósito: Automatizar tareas repetitivas, programar tareas para que se ejecuten en momentos específicos y gestionar los recursos del sistema de manera eficiente.

Creación de un Trabajo por Lotes Simple

Escribir un Archivo de Comandos por Lotes

Un archivo de comandos por lotes contiene una serie de comandos DCL (Digital Command Language) para ser ejecutados.

Ejemplo:

$! Ejemplo de archivo de comandos por lotes
$ WRITE SYS$OUTPUT "Iniciando trabajo por lotes..."
$ DIRECTORY SYS$LOGIN
$ WRITE SYS$OUTPUT "Trabajo por lotes completado."
  • Explicación:
    • $! es un comentario.
    • WRITE SYS$OUTPUT envía texto a la terminal.
    • DIRECTORY SYS$LOGIN lista los archivos en el directorio de inicio del usuario.

Enviar un Trabajo por Lotes

Usa el comando SUBMIT para enviar un trabajo por lotes.

Ejemplo:

$ SUBMIT my_batch_job.com
  • Explicación:
    • SUBMIT es el comando para enviar un trabajo por lotes.
    • my_batch_job.com es el nombre del archivo de comandos por lotes.

Gestión de Trabajos por Lotes

Monitoreo de Trabajos por Lotes

Usa el comando SHOW QUEUE para monitorear trabajos por lotes.

Ejemplo:

$ SHOW QUEUE/BATCH
  • Explicación:
    • SHOW QUEUE muestra el estado de las colas.
    • /BATCH filtra la visualización para mostrar solo las colas de lotes.

Eliminar un Trabajo por Lotes

Usa el comando DELETE/ENTRY para eliminar un trabajo por lotes.

Ejemplo:

$ DELETE/ENTRY=12345
  • Explicación:
    • DELETE/ENTRY elimina un trabajo específico.
    • 12345 es el número de entrada del trabajo.

Técnicas Avanzadas de Trabajos por Lotes

Programación de Trabajos por Lotes

Usa el calificador /AFTER con el comando SUBMIT para programar un trabajo por lotes.

Ejemplo:

$ SUBMIT/AFTER="TOMORROW+08:00" my_batch_job.com
  • Explicación:
    • /AFTER="TOMORROW+08:00" programa el trabajo para que se ejecute a las 8 AM del día siguiente.

Uso de Nombres Lógicos y Símbolos

Los nombres lógicos y símbolos pueden hacer que los trabajos por lotes sean más flexibles.

Ejemplo:

$ DEFINE MYDIR SYS$LOGIN
$ DIRECTORY MYDIR
  • Explicación:
    • DEFINE MYDIR SYS$LOGIN crea un nombre lógico MYDIR para el directorio de inicio.
    • DIRECTORY MYDIR usa el nombre lógico para listar el contenido del directorio.

Manejo de Errores en Trabajos por Lotes

Usa comandos condicionales para manejar errores.

Ejemplo:

$ SET NOON
$ COPY file1.txt file2.txt
$ IF $STATUS .NE. 1 THEN WRITE SYS$OUTPUT "Error al copiar el archivo."
  • Explicación:
    • SET NOON evita que el trabajo se detenga en caso de errores.
    • IF $STATUS .NE. 1 verifica si el último comando falló.

Resumen

  • Trabajos por lotes automatizan tareas y gestionan recursos del sistema.
  • Crear trabajos por lotes implica escribir archivos de comandos DCL y usar el comando SUBMIT.
  • Gestionar trabajos por lotes incluye monitorear con SHOW QUEUE y eliminar con DELETE/ENTRY.
  • Técnicas avanzadas incluyen programación, uso de nombres lógicos y manejo de errores.

Al dominar estos conceptos, puedes crear y gestionar trabajos por lotes en OpenVMS de manera eficiente, mejorando tu productividad y capacidades de gestión del sistema.
