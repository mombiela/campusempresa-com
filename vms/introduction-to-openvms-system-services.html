Introducción a los Servicios del Sistema OpenVMS
Uso de Llamadas a Servicios del Sistema ►

Visión General

OpenVMS (Open Virtual Memory System) es un sistema operativo de memoria virtual basado en multiprocesamiento y multiusuario, diseñado para su uso en procesamiento por tiempo compartido, procesamiento por lotes y procesamiento de transacciones. Este tema te introducirá a los servicios del sistema proporcionados por OpenVMS, que son esenciales para interactuar con el sistema operativo a un nivel bajo.

Conceptos Clave

  • Servicios del Sistema: Son rutinas proporcionadas por el sistema operativo OpenVMS que las aplicaciones pueden llamar para realizar varias tareas como control de procesos, operaciones de archivos y comunicación entre procesos.
  • Llamada a Servicios del Sistema: Los servicios del sistema se llaman típicamente usando una sintaxis específica en lenguajes de programación como C o ensamblador.

Servicios del Sistema Comunes

Aquí están algunos de los servicios del sistema más comúnmente utilizados en OpenVMS:

  • $CREPRC: Crear un proceso.
  • $DELPRC: Eliminar un proceso.
  • $GETJPI: Obtener información de trabajo/proceso.
  • $QIO: Poner en cola una solicitud de E/S.
  • $SNDJBC: Enviar un trabajo a una cola de lotes.

Llamada a Servicios del Sistema en C

Para llamar a un servicio del sistema en C, típicamente se incluyen los archivos de encabezado apropiados y se usa el nombre de la función del servicio del sistema. Aquí hay un ejemplo de llamada al servicio $GETJPI para obtener información sobre un proceso.

Ejemplo: Obtener Información del Proceso

#include <stdio.h>
#include <descrip.h>
#include <ssdef.h>
#include <jpidef.h>
#include <lib$routines.h>
#include <starlet.h>

int main() {
    unsigned long status;
    unsigned long pid = 0; // 0 significa proceso actual
    unsigned long item_code = JPI$_PID; // Obtener el ID del proceso
    unsigned long result;

    status = sys$getjpiw(0, &pid, 0, &item_code, &result, 0, 0, 0);

    if (status == SS$_NORMAL) {
        printf("ID del Proceso: %lu\n", result);
    } else {
        printf("Error al recuperar la información del proceso\n");
    }

    return 0;
}
  • Archivos de Encabezado: Incluir los encabezados necesarios para los servicios del sistema.
  • Variables: Definir variables para el estado, ID del proceso, código de ítem y resultado.
  • sys$getjpiw: Llamar al servicio del sistema $GETJPI.
  • Verificación de Estado: Comprobar si la llamada fue exitosa e imprimir el ID del proceso.

Uso de Servicios del Sistema en Ensamblador

Para aquellos que prefieren o necesitan usar el lenguaje ensamblador, aquí hay un ejemplo de llamada al servicio $CREPRC para crear un nuevo proceso.

Ejemplo: Crear un Proceso en Ensamblador

        .TITLE  CREATE_PROCESS
        .IDENT  /V1.0/

        .MACRO  $CREPRC, PID, NAME, STATUS
        MOV     R0, PID
        MOV     R1, NAME
        MOV     R2, STATUS
        CALLS   #3, SYS$CREPRC
        .ENDM

        .ENTRY  START
        .PSECT  DATA, RW, NOEXE

PID:    .LONG   0
NAME:   .ASCID  /MYPROCESS/
STATUS: .LONG   0

        .PSECT  CODE, RO, EXEC

START:  $CREPRC PID, NAME, STATUS
        BR      EXIT

EXIT:   HALT

        .END    START
  • .MACRO: Definir una macro para el servicio del sistema $CREPRC.
  • Instrucciones MOV: Mover parámetros a los registros.
  • CALLS: Llamar al servicio del sistema.
  • .PSECT: Definir secciones para datos y código.
  • HALT: Terminar el programa.

Conclusión

Entender y usar los servicios del sistema OpenVMS es crucial para una programación efectiva a nivel de sistema en la plataforma OpenVMS. Esta introducción cubrió los conceptos básicos de lo que son los servicios del sistema, cómo llamarlos en C y ensamblador, y proporcionó ejemplos para ilustrar su uso. A medida que progreses, encontrarás servicios y escenarios más complejos, pero el conocimiento fundamental proporcionado aquí servirá como un punto de partida sólido.
