Introducción a Terraform Cloud
Configurando Terraform Cloud ►

Terraform Cloud es un servicio gestionado proporcionado por HashiCorp que ofrece un entorno colaborativo para que los equipos gestionen su infraestructura como código. Esta sección te introducirá a Terraform Cloud, sus características y cómo empezar.

Conceptos Clave

  • Terraform Cloud: Un servicio gestionado que proporciona un entorno colaborativo para que los equipos usen Terraform.
  • Workspaces: Entornos aislados dentro de Terraform Cloud donde puedes gestionar una colección de recursos de infraestructura.
  • Gestión de Estado Remoto: Terraform Cloud almacena tus archivos de estado de forma remota, asegurando consistencia y colaboración.
  • Integración con VCS: Integración con Sistemas de Control de Versiones (VCS) como GitHub, GitLab y Bitbucket para ejecuciones automáticas de Terraform.
  • Run Triggers: Disparadores automáticos de ejecuciones de Terraform basados en cambios en tu VCS.
  • Políticas de Sentinel: Marco de políticas como código para hacer cumplir la conformidad y la gobernanza.

Características de Terraform Cloud

  • Colaboración: Varios miembros del equipo pueden trabajar en la misma infraestructura.
  • Operaciones Remotas: Las operaciones de Terraform se ejecutan de forma remota, reduciendo la necesidad de ejecución local.
  • Gestión de Estado: Almacenamiento de estado seguro y consistente.
  • Aplicación de Políticas: Usa Sentinel para aplicar políticas y conformidad.
  • Estimación de Costos: Estima el costo de tus cambios de infraestructura antes de aplicarlos.
  • Notificaciones: Recibe notificaciones sobre el estado de tus ejecuciones.

Empezando con Terraform Cloud

Paso 1: Regístrate en Terraform Cloud

  1. Visita el sitio web de Terraform Cloud.
  2. Regístrate para una cuenta gratuita o inicia sesión si ya tienes una.

Paso 2: Crea una Organización

  1. Después de iniciar sesión, crea una nueva organización.
  2. Proporciona un nombre para tu organización y haz clic en "Create Organization".

Paso 3: Crea un Workspace

  1. Dentro de tu organización, crea un nuevo workspace.
  2. Elige un nombre para tu workspace y selecciona el repositorio VCS que contiene tus archivos de configuración de Terraform.

Paso 4: Configura la Integración con VCS

  1. Conecta tu proveedor de VCS (por ejemplo, GitHub, GitLab).
  2. Autoriza a Terraform Cloud para acceder a tu repositorio.
  3. Selecciona el repositorio y la rama que contiene tu configuración de Terraform.

Paso 5: Ejecuta Terraform

  1. Empuja cambios a tu repositorio VCS.
  2. Terraform Cloud detectará automáticamente los cambios y comenzará una ejecución.
  3. Revisa el plan y aplica los cambios si todo se ve bien.

Ejemplo: Configuración Básica de Terraform Cloud

Aquí hay un ejemplo simple de una configuración de Terraform que aprovisiona una instancia EC2 de AWS. Esta configuración se almacenará en tu repositorio VCS y será gestionada por Terraform Cloud.

provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "TerraformExample"
  }
}

Explicación

  • provider "aws": Configura el proveedor de AWS con la región especificada.
  • resource "aws_instance" "example": Define un recurso de instancia EC2 de AWS.
    • ami: El ID de la Imagen de Máquina de Amazon (AMI) a usar para la instancia.
    • instance_type: El tipo de instancia a lanzar.
    • tags: Etiquetas para asignar a la instancia.

Conclusión

Terraform Cloud proporciona una plataforma poderosa para gestionar tu infraestructura como código con colaboración mejorada, operaciones remotas y aplicación de políticas. Al integrarse con tu VCS, Terraform Cloud automatiza el proceso de aplicar cambios de infraestructura, asegurando consistencia y reduciendo el riesgo de errores. En las siguientes secciones, profundizaremos en características avanzadas y mejores prácticas para usar Terraform Cloud de manera efectiva.
