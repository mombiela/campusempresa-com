Variables y Salidas en Terraform
Expresiones de Terraform ►

Introducción

En esta sección, exploraremos cómo usar variables y salidas en Terraform. Las variables te permiten parametrizar tus configuraciones de Terraform, haciéndolas más flexibles y reutilizables. Las salidas te permiten extraer y mostrar información desde tu estado de Terraform.

Variables

¿Qué son las Variables?

Las variables en Terraform se utilizan para parametrizar tus configuraciones. Te permiten definir valores que pueden ser reutilizados en todos tus archivos de configuración.

Definiendo Variables

Las variables se definen en un archivo .tf usando la palabra clave variable. Aquí hay un ejemplo:

variable "instance_type" {
  description = "Tipo de instancia EC2"
  type        = string
  default     = "t2.micro"
}
  • description: Una breve explicación de para qué es la variable.
  • type: El tipo de la variable (por ejemplo, string, number, list, map).
  • default: Un valor predeterminado para la variable si no se proporciona ninguno.

Usando Variables

Una vez definidas, las variables pueden ser usadas en tus archivos de configuración de Terraform. Aquí hay un ejemplo de cómo usar la variable instance_type:

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = var.instance_type
}

Tipos de Variables

Terraform soporta varios tipos de variables:

  • string: Un solo valor de cadena.
  • number: Un valor numérico.
  • bool: Un valor booleano (true o false).
  • list: Una lista de valores.
  • map: Un mapa de pares clave-valor.

Proporcionando Valores para Variables

Los valores para las variables pueden ser proporcionados de varias maneras:

  1. Flags de línea de comandos:

    terraform apply -var="instance_type=t2.large"

  2. Archivo de definiciones de variables (.tfvars):

    instance_type = "t2.large"

  3. Variables de entorno:

    export TF_VAR_instance_type="t2.large"

Ejemplo: Usando una Variable de Lista

Aquí hay un ejemplo de definición y uso de una variable de lista:

variable "availability_zones" {
  description = "Lista de zonas de disponibilidad"
  type        = list(string)
  default     = ["us-west-1a", "us-west-1b"]
}

resource "aws_instance" "example" {
  ami               = "ami-0c55b159cbfafe1f0"
  instance_type     = "t2.micro"
  availability_zone = var.availability_zones[0]
}

Salidas

¿Qué son las Salidas?

Las salidas en Terraform se utilizan para mostrar información desde tu estado de Terraform. Pueden ser útiles para depuración, o para proporcionar información a otras configuraciones o usuarios.

Definiendo Salidas

Las salidas se definen en un archivo .tf usando la palabra clave output. Aquí hay un ejemplo:

output "instance_id" {
  description = "El ID de la instancia EC2"
  value       = aws_instance.example.id
}
  • description: Una breve explicación de para qué es la salida.
  • value: El valor a ser mostrado, que puede referenciar recursos y variables.

Usando Salidas

Las salidas pueden ser vistas después de ejecutar terraform apply usando el comando terraform output:

terraform output instance_id

Ejemplo: Múltiples Salidas

Aquí hay un ejemplo de definición de múltiples salidas:

output "instance_id" {
  description = "El ID de la instancia EC2"
  value       = aws_instance.example.id
}

output "instance_public_ip" {
  description = "La IP pública de la instancia EC2"
  value       = aws_instance.example.public_ip
}

Resumen

En esta sección, cubrimos los conceptos básicos de variables y salidas en Terraform. Las variables te permiten parametrizar tus configuraciones, haciéndolas más flexibles y reutilizables. Las salidas te permiten extraer y mostrar información desde tu estado de Terraform. Al entender y utilizar estas características, puedes crear configuraciones de Terraform más dinámicas e informativas.

Puntos Clave

  • Las variables pueden ser definidas, usadas y proporcionadas valores de múltiples maneras.
  • Las salidas ayudan a extraer y mostrar información útil desde el estado de Terraform.
  • El uso adecuado de variables y salidas puede mejorar significativamente la flexibilidad y legibilidad de tus configuraciones de Terraform.
