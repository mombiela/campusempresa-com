Introducción

La planificación de tareas es un componente crucial en el desarrollo de inteligencia artificial (IA) para videojuegos. Permite que los personajes no jugadores (NPCs) tomen decisiones complejas y realicen acciones de manera organizada y eficiente. En esta sección, exploraremos los conceptos clave, técnicas y ejemplos prácticos para implementar la planificación de tareas en videojuegos.

Conceptos Clave

  • Tareas y Subtareas: Las tareas son acciones que un NPC debe realizar, y pueden descomponerse en subtareas más pequeñas.
  • Árboles de Comportamiento (Behavior Trees): Estructuras jerárquicas que organizan tareas y subtareas de manera lógica.
  • Planificación Basada en Objetivos (Goal-Oriented Action Planning, GOAP): Técnica que permite a los NPCs seleccionar y planificar acciones basadas en objetivos específicos.
  • Máquinas de Estados Finite (Finite State Machines, FSM): Modelos que representan estados y transiciones entre ellos, útiles para tareas secuenciales.

Árboles de Comportamiento

Los árboles de comportamiento son una herramienta poderosa para organizar y gestionar las tareas de los NPCs. Se componen de nodos que representan acciones y condiciones.

Ejemplo de Árbol de Comportamiento

class Node:
    def __init__(self, name):
        self.name = name
        self.children = []

    def add_child(self, child):
        self.children.append(child)

class BehaviorTree:
    def __init__(self, root):
        self.root = root

    def execute(self):
        return self.root.execute()

class TaskNode(Node):
    def execute(self):
        print(f"Executing task: {self.name}")
        for child in self.children:
            child.execute()

# Crear nodos de tareas
root = TaskNode("Root Task")
subtask1 = TaskNode("Subtask 1")
subtask2 = TaskNode("Subtask 2")

# Construir el árbol de comportamiento
root.add_child(subtask1)
root.add_child(subtask2)

# Ejecutar el árbol de comportamiento
tree = BehaviorTree(root)
tree.execute()

En este ejemplo, hemos creado un árbol de comportamiento simple con una tarea raíz y dos subtareas. Al ejecutar el árbol, se imprimen las tareas en orden jerárquico.

Planificación Basada en Objetivos (GOAP)

GOAP es una técnica avanzada que permite a los NPCs planificar sus acciones basándose en objetivos específicos. Los NPCs evalúan sus objetivos y seleccionan las acciones más adecuadas para alcanzarlos.

Ejemplo de GOAP

class Action:
    def __init__(self, name, cost):
        self.name = name
        self.cost = cost

    def execute(self):
        print(f"Executing action: {self.name}")

class Goal:
    def __init__(self, name):
        self.name = name
        self.actions = []

    def add_action(self, action):
        self.actions.append(action)

    def plan(self):
        # Ordenar acciones por costo
        self.actions.sort(key=lambda x: x.cost)
        for action in self.actions:
            action.execute()

# Crear acciones
action1 = Action("Action 1", 10)
action2 = Action("Action 2", 5)

# Crear objetivo y añadir acciones
goal = Goal("Reach Goal")
goal.add_action(action1)
goal.add_action(action2)

# Planificar y ejecutar acciones
goal.plan()

En este ejemplo, hemos creado un objetivo con dos acciones. Las acciones se ordenan por costo y se ejecutan en consecuencia.

Máquinas de Estados Finite (FSM)

Las FSM son útiles para modelar comportamientos secuenciales y estados de los NPCs. Cada estado representa una tarea o acción específica, y las transiciones definen cómo moverse entre estados.

Ejemplo de FSM

class State:
    def __init__(self, name):
        self.name = name

    def execute(self):
        print(f"Executing state: {self.name}")

class FSM:
    def __init__(self):
        self.states = {}
        self.current_state = None

    def add_state(self, state):
        self.states[state.name] = state

    def set_state(self, state_name):
        self.current_state = self.states[state_name]

    def execute(self):
        if self.current_state:
            self.current_state.execute()

# Crear estados
state1 = State("State 1")
state2 = State("State 2")

# Crear FSM y añadir estados
fsm = FSM()
fsm.add_state(state1)
fsm.add_state(state2)

# Establecer estado inicial y ejecutar
fsm.set_state("State 1")
fsm.execute()
fsm.set_state("State 2")
fsm.execute()

En este ejemplo, hemos creado una FSM con dos estados y hemos demostrado cómo cambiar y ejecutar estados.

Conclusión

La planificación de tareas es esencial para crear NPCs inteligentes y realistas en videojuegos. Las técnicas como los árboles de comportamiento, GOAP y FSM proporcionan diferentes enfoques para organizar y gestionar las acciones de los NPCs. Con una comprensión sólida de estas técnicas, los desarrolladores pueden crear experiencias de juego más inmersivas y desafiantes.

Recuerda practicar con los ejemplos proporcionados y experimentar con diferentes configuraciones para comprender mejor cómo funcionan estas técnicas en la práctica.
