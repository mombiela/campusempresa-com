Introducción a la Búsqueda en Elasticsearch

Elasticsearch es un motor de búsqueda poderoso que te permite almacenar, buscar y analizar grandes volúmenes de datos rápidamente y en tiempo casi real. Esta sección te introducirá a los conceptos básicos de la búsqueda de datos en Elasticsearch.

  • Búsqueda de texto completo: Elasticsearch sobresale en la búsqueda de texto completo, lo que significa buscar texto dentro de documentos.
  • Búsqueda estructurada: Elasticsearch también soporta la búsqueda estructurada, que implica buscar campos específicos dentro de documentos.
  • Filtrado: El filtrado te permite reducir los resultados de búsqueda basándote en criterios específicos.

Consultas de Búsqueda Básicas

Elasticsearch utiliza un lenguaje de consulta llamado Query DSL (Domain Specific Language) para buscar datos. Comencemos con algunas consultas de búsqueda básicas.

Consulta Match

La consulta match se usa para buscar un término específico en un campo.

GET /my_index/_search
{
  "query": {
    "match": {
      "message": "search term"
    }
  }
}
  • Explicación: Esta consulta busca documentos en el índice my_index donde el campo message contiene el término "search term".

Consulta Term

La consulta term se usa para coincidencias exactas.

GET /my_index/_search
{
  "query": {
    "term": {
      "status": "active"
    }
  }
}
  • Explicación: Esta consulta busca documentos donde el campo status coincide exactamente con "active".

Consultas de Búsqueda Avanzadas

A medida que te sientas más cómodo con las consultas básicas, puedes comenzar a explorar capacidades de búsqueda más avanzadas.

Consulta Bool

La consulta bool te permite combinar múltiples consultas usando lógica booleana.

GET /my_index/_search
{
  "query": {
    "bool": {
      "must": [
        { "match": { "message": "search term" } },
        { "term": { "status": "active" } }
      ],
      "filter": [
        { "range": { "date": { "gte": "2023-01-01" } } }
      ]
    }
  }
}
  • Explicación: Esta consulta busca documentos que deben coincidir tanto en el campo message que contiene "search term" como en el campo status que coincide exactamente con "active". Además, filtra los resultados para incluir solo documentos donde el campo date es mayor o igual a "2023-01-01".

Agregaciones

Las agregaciones te permiten realizar cálculos complejos y análisis de datos.

GET /my_index/_search
{
  "aggs": {
    "status_count": {
      "terms": {
        "field": "status"
      }
    }
  }
}
  • Explicación: Esta consulta realiza una agregación para contar el número de documentos para cada valor único en el campo status.

Filtrado vs. Consulta

Entender la diferencia entre filtrado y consulta es crucial para optimizar tus búsquedas.

| Característica | Filtrado | Consulta | |----------------|----------|----------| | Caso de Uso | Reducir resultados basados en criterios | Buscar documentos relevantes basados en la puntuación de relevancia | | Rendimiento | Más rápido, ya que no implica puntuación | Más lento, ya que implica puntuación | | Ejemplo | filter en consulta bool | must en consulta bool |

Paginación

Elasticsearch soporta la paginación para manejar grandes conjuntos de resultados de búsqueda.

From y Size

Los parámetros from y size controlan la paginación.

GET /my_index/_search
{
  "from": 10,
  "size": 10,
  "query": {
    "match_all": {}
  }
}
  • Explicación: Esta consulta recupera la segunda página de resultados, asumiendo que cada página contiene 10 resultados.

Ordenación

La ordenación te permite ordenar tus resultados de búsqueda basándote en campos específicos.

GET /my_index/_search
{
  "sort": [
    { "date": { "order": "desc" } }
  ],
  "query": {
    "match_all": {}
  }
}
  • Explicación: Esta consulta ordena los resultados de búsqueda por el campo date en orden descendente.

Conclusión

En este tema, cubrimos los conceptos básicos de la búsqueda de datos en Elasticsearch, incluyendo consultas de búsqueda básicas y avanzadas, filtrado, paginación y ordenación. Al entender estos conceptos, puedes buscar y analizar efectivamente los datos almacenados en Elasticsearch. A medida que progreses, puedes explorar consultas y agregaciones más complejas para aprovechar todo el poder de Elasticsearch.
