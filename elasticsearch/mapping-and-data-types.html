Introducción al Mapeo en Elasticsearch

El mapeo en Elasticsearch es el proceso de definir cómo se almacenan e indexan un documento y los campos que contiene. Es similar a definir un esquema en una base de datos relacional. Entender el mapeo es crucial para una indexación y consulta de datos eficiente.

Conceptos Clave

  • Documento: Un objeto JSON que se almacena en Elasticsearch.
  • Índice: Una colección de documentos.
  • Campo: Un par clave-valor en un documento.
  • Mapeo: El proceso de definir cómo se almacenan e indexan los documentos y sus campos.

Mapeo Básico

Cuando indexas un documento en Elasticsearch sin definir explícitamente un mapeo, Elasticsearch creará automáticamente un mapeo dinámico para ti. Sin embargo, para un mejor control y rendimiento, a menudo es beneficioso definir tus propios mapeos.

Ejemplo de Mapeo Dinámico

PUT /my_index/_doc/1
{
  "name": "John Doe",
  "age": 30,
  "email": "john.doe@example.com"
}

En este ejemplo, Elasticsearch inferirá automáticamente los tipos de datos de los campos.

Definiendo Mapeos Explícitos

Puedes definir mapeos explícitos al crear un índice para asegurar que los campos se indexen correctamente.

PUT /my_index
{
  "mappings": {
    "properties": {
      "name": {
        "type": "text"
      },
      "age": {
        "type": "integer"
      },
      "email": {
        "type": "keyword"
      }
    }
  }
}

En este ejemplo:

  • name se define como text, adecuado para búsqueda de texto completo.
  • age se define como integer, adecuado para operaciones numéricas.
  • email se define como keyword, adecuado para coincidencias exactas.

Tipos de Datos en Elasticsearch

Elasticsearch soporta una variedad de tipos de datos, cada uno optimizado para diferentes tipos de datos y consultas.

Tipos de Datos Básicos

  • Tipos de Cadena:
    • text: Usado para búsqueda de texto completo.
    • keyword: Usado para coincidencias exactas.
  • Tipos Numéricos:
    • integer: Entero de 32 bits.
    • long: Entero de 64 bits.
    • float: Punto flotante de precisión simple.
    • double: Punto flotante de doble precisión.
  • Fecha: Almacena fechas en varios formatos.
  • Booleano: Almacena true o false.

Ejemplo de Diferentes Tipos de Datos

PUT /my_index
{
  "mappings": {
    "properties": {
      "title": {
        "type": "text"
      },
      "price": {
        "type": "float"
      },
      "release_date": {
        "type": "date"
      },
      "in_stock": {
        "type": "boolean"
      }
    }
  }
}

En este ejemplo:

  • title es un campo text para búsqueda de texto completo.
  • price es un campo float para operaciones numéricas.
  • release_date es un campo date para operaciones de fecha.
  • in_stock es un campo boolean para valores verdadero/falso.

Técnicas Avanzadas de Mapeo

Objetos Anidados

Los objetos anidados te permiten indexar arrays de objetos y consultarlos como si fueran documentos separados.

PUT /my_index
{
  "mappings": {
    "properties": {
      "user": {
        "type": "nested",
        "properties": {
          "first_name": {
            "type": "text"
          },
          "last_name": {
            "type": "text"
          }
        }
      }
    }
  }
}

Campos Múltiples

Los campos múltiples permiten que un solo campo se indexe de múltiples maneras para diferentes tipos de consultas.

PUT /my_index
{
  "mappings": {
    "properties": {
      "name": {
        "type": "text",
        "fields": {
          "keyword": {
            "type": "keyword"
          }
        }
      }
    }
  }
}

En este ejemplo, el campo name se indexa tanto como text para búsqueda de texto completo como keyword para coincidencias exactas.

Conclusión

Entender y definir mapeos en Elasticsearch es esencial para una indexación y consulta de datos eficiente. Al definir mapeos explícitos, puedes optimizar tus índices de Elasticsearch para un mejor rendimiento y resultados de búsqueda más precisos. Recuerda elegir los tipos de datos apropiados para tus campos y considera técnicas avanzadas como objetos anidados y campos múltiples para manejar estructuras de datos complejas.
