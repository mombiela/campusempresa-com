Introducción a Logstash

Logstash es una poderosa herramienta de procesamiento de datos que puede ingerir datos de varias fuentes, transformarlos y enviarlos a su destino deseado, como Elasticsearch. Esta sección te introducirá a los conceptos básicos de Logstash y cómo se integra con Elasticsearch.

  • ¿Qué es Logstash?

    • Una tubería de procesamiento de datos del lado del servidor.
    • Ingresa datos de múltiples fuentes simultáneamente.
    • Transforma datos usando plugins de filtro.
    • Envía datos a varios destinos, incluyendo Elasticsearch.

  • ¿Por qué usar Logstash con Elasticsearch?

    • Integración perfecta para la ingestión de datos.
    • Procesamiento y transformación de datos en tiempo real.
    • Capacidades mejoradas de enriquecimiento de datos.

Configuración de Logstash

Antes de profundizar en el uso de Logstash con Elasticsearch, necesitas configurar Logstash en tu sistema.

  • Pasos de instalación:

    1. Descarga Logstash desde el sitio web oficial de Elastic.
    2. Extrae el archivo descargado.
    3. Navega al directorio de Logstash.
    4. Ejecuta Logstash usando el comando:
    bin/logstash -e 'input { stdin { } } output { stdout {} }'

  • Configuración básica:

    • Los archivos de configuración de Logstash están escritos en un formato similar a JSON.
    • Un archivo de configuración básico consta de tres secciones: input, filter y output.

Configuración de Logstash para Elasticsearch

Para enviar datos de Logstash a Elasticsearch, necesitas configurar adecuadamente las secciones input y output.

  • Sección de entrada:

    • Define la fuente de los datos.
    • Ejemplo: 
      input {
  file {
    path => "/path/to/your/logfile.log"
    start_position => "beginning"
  }
}

  • Sección de filtro:

    • Se utiliza para transformar y enriquecer los datos.
    • Ejemplo: 
      filter {
  grok {
    match => { "message" => "%{COMBINEDAPACHELOG}" }
  }
  date {
    match => [ "timestamp" , "dd/MMM/yyyy:HH:mm:ss Z" ]
  }
}

  • Sección de salida:

    • Define el destino para los datos procesados.
    • Ejemplo: 
      output {
  elasticsearch {
    hosts => ["localhost:9200"]
    index => "logstash-%{+YYYY.MM.dd}"
  }
}

Ejemplo práctico: Ingestión de registros de Apache

Vamos a recorrer un ejemplo práctico de cómo ingerir registros de Apache en Elasticsearch usando Logstash.

  1. Crear un archivo de configuración de Logstash:

    input {
  file {
    path => "/var/log/apache2/access.log"
    start_position => "beginning"
  }
}

filter {
  grok {
    match => { "message" => "%{COMBINEDAPACHELOG}" }
  }
  date {
    match => [ "timestamp" , "dd/MMM/yyyy:HH:mm:ss Z" ]
  }
}

output {
  elasticsearch {
    hosts => ["localhost:9200"]
    index => "apache-logs-%{+YYYY.MM.dd}"
  }
}

  2. Ejecutar Logstash con el archivo de configuración:

    bin/logstash -f /path/to/logstash.conf

  3. Verificar los datos en Elasticsearch:

    • Usa Kibana o la API REST de Elasticsearch para verificar que los datos se han ingerido correctamente.
    • Ejemplo de llamada a la API REST: 
      curl -X GET "localhost:9200/apache-logs-*/_search?pretty"

Características avanzadas de Logstash

A medida que te sientas más cómodo con Logstash, puedes explorar sus características avanzadas para mejorar tu tubería de procesamiento de datos.

  • Procesamiento condicional:

    • Aplica filtros y salidas de manera condicional.
    • Ejemplo: 
      filter {
  if [status] == 404 {
    mutate { add_tag => "not_found" }
  }
}

  • Múltiples tuberías:

    • Ejecuta múltiples tuberías en una sola instancia de Logstash.
    • Ejemplo de configuración: 
      - pipeline.id: apache_logs
  path.config: "/etc/logstash/conf.d/apache.conf"

- pipeline.id: syslogs
  path.config: "/etc/logstash/conf.d/syslog.conf"

Conclusión

Usar Logstash con Elasticsearch proporciona una solución robusta para ingerir, transformar y almacenar datos. Al entender los conceptos básicos de la configuración de Logstash y explorar características avanzadas, puedes construir tuberías de datos eficientes que se adapten a tus necesidades específicas. Ya sea que estés manejando archivos de registro, métricas u otros tipos de datos, Logstash y Elasticsearch juntos ofrecen una combinación poderosa para el procesamiento y análisis de datos en tiempo real.
