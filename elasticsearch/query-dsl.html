Introducción al Query DSL

Elasticsearch Query DSL (Domain Specific Language) es una forma poderosa y flexible de consultar datos almacenados en Elasticsearch. Te permite construir consultas complejas usando sintaxis JSON.

  • ¿Qué es Query DSL?
    • Un lenguaje de consulta basado en JSON.
    • Permite consultas complejas y anidadas.
    • Soporta filtrado, agregación y búsqueda de texto completo.

Consultas Básicas

Consulta Match

La consulta match se usa para buscar un término específico en un campo.

{
  "query": {
    "match": {
      "field_name": "search_term"
    }
  }
}

Consulta Term

La consulta term se usa para coincidencias exactas.

{
  "query": {
    "term": {
      "field_name": "exact_term"
    }
  }
}

Consulta Range

La consulta range se usa para encontrar documentos dentro de un rango específico.

{
  "query": {
    "range": {
      "field_name": {
        "gte": 10,
        "lte": 20
      }
    }
  }
}

Consultas Compuestas

Consulta Bool

La consulta bool te permite combinar múltiples consultas usando lógica booleana.

{
  "query": {
    "bool": {
      "must": [
        { "match": { "field_name": "search_term" } }
      ],
      "filter": [
        { "range": { "field_name": { "gte": 10, "lte": 20 } } }
      ]
    }
  }
}
  • must: Todas las condiciones deben ser verdaderas.
  • filter: Las condiciones deben ser verdaderas, pero no afectan la puntuación.
  • should: Al menos una condición debería ser verdadera.
  • must_not: Las condiciones no deben ser verdaderas.

Agregaciones

Las agregaciones te permiten realizar cálculos complejos y análisis de datos.

Agregación Terms

La agregación terms se usa para agrupar documentos por un campo específico.

{
  "aggs": {
    "group_by_field": {
      "terms": {
        "field": "field_name"
      }
    }
  }
}

Agregación Date Histogram

La agregación date histogram se usa para agrupar documentos por intervalos de fecha.

{
  "aggs": {
    "sales_over_time": {
      "date_histogram": {
        "field": "date_field",
        "interval": "month"
      }
    }
  }
}

Consultas Avanzadas

Consulta Nested

La consulta nested se usa para buscar dentro de objetos anidados.

{
  "query": {
    "nested": {
      "path": "nested_path",
      "query": {
        "bool": {
          "must": [
            { "match": { "nested_field": "search_term" } }
          ]
        }
      }
    }
  }
}

Consulta Script

La consulta script te permite usar scripts personalizados para consultas más complejas.

{
  "query": {
    "script": {
      "script": {
        "source": "doc['field_name'].value > 10",
        "lang": "painless"
      }
    }
  }
}

Conclusión

Elasticsearch Query DSL es una herramienta versátil y poderosa para consultar y analizar datos. Al entender las consultas básicas y avanzadas, puedes aprovechar Elasticsearch al máximo. Practica con diferentes tipos de consultas y agregaciones para volverte competente en el uso de Elasticsearch Query DSL.

  • Puntos Clave:
    • Query DSL está basado en JSON y soporta consultas complejas.
    • Las consultas básicas incluyen match, term y range.
    • Las consultas compuestas como las bool permiten combinar múltiples condiciones.
    • Las agregaciones permiten análisis de datos avanzados.
    • Las consultas avanzadas incluyen nested y script.

Al dominar estos conceptos, estarás bien equipado para manejar una amplia gama de tareas de búsqueda y análisis en Elasticsearch.
