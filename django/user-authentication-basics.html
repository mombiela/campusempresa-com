Introducción

La autenticación de usuarios es un aspecto crucial del desarrollo web, asegurando que los usuarios sean quienes dicen ser. En Django, el framework proporciona herramientas robustas para manejar la autenticación de manera eficiente. Esta sección te guiará a través de los conceptos básicos de la autenticación de usuarios en Django, desde la configuración hasta la implementación de funciones avanzadas.

Configuración de la Autenticación en Django

Instalación de Django

Antes de profundizar en la autenticación, asegúrate de tener Django instalado. Puedes instalarlo usando pip:

pip install django

Creación de un Proyecto Django

Crea un nuevo proyecto Django y navega dentro de él:

django-admin startproject myproject
cd myproject

Creación de una Aplicación Django

Crea una nueva aplicación dentro de tu proyecto para manejar la autenticación de usuarios:

python manage.py startapp accounts

Añadir la Aplicación a INSTALLED_APPS

Añade la nueva aplicación a la lista INSTALLED_APPS en settings.py:

# myproject/settings.py

INSTALLED_APPS = [
    ...
    'accounts',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    ...
]

Modelo de Usuario y Vistas de Autenticación

Uso del Modelo de Usuario Incorporado de Django

Django proporciona un modelo de usuario incorporado que puedes usar para la autenticación. Puedes importarlo de la siguiente manera:

from django.contrib.auth.models import User

Creación de la Vista de Registro de Usuario

Crea una vista para manejar el registro de usuarios en views.py:

# accounts/views.py

from django.shortcuts import render, redirect
from django.contrib.auth.forms import UserCreationForm

def register(request):
    if request.method == 'POST':
        form = UserCreationForm(request.POST)
        if form.is_valid():
            form.save()
            return redirect('login')
    else:
        form = UserCreationForm()
    return render(request, 'accounts/register.html', {'form': form})

Creación de la Plantilla de Registro de Usuario

Crea una plantilla para el registro de usuarios en templates/accounts/register.html:

<!-- templates/accounts/register.html -->

<!DOCTYPE html>
<html>
<head>
    <title>Register</title>
</head>
<body>
    <h2>Register</h2>
    <form method="post">
        {% csrf_token %}
        {{ form.as_p }}
        <button type="submit">Register</button>
    </form>
</body>
</html>

Añadir Patrones de URL

Añade patrones de URL para dirigir las solicitudes a la vista de registro:

# accounts/urls.py

from django.urls import path
from . import views

urlpatterns = [
    path('register/', views.register, name='register'),
]

Incluye las URLs de accounts en el urls.py del proyecto principal:

# myproject/urls.py

from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('accounts/', include('accounts.urls')),
]

Inicio y Cierre de Sesión de Usuario

Creación de la Vista de Inicio de Sesión

Django proporciona vistas incorporadas para el inicio y cierre de sesión. Puedes usar LoginView y LogoutView de django.contrib.auth.views:

# accounts/urls.py

from django.contrib.auth import views as auth_views

urlpatterns = [
    ...
    path('login/', auth_views.LoginView.as_view(template_name='accounts/login.html'), name='login'),
    path('logout/', auth_views.LogoutView.as_view(), name='logout'),
]

Creación de la Plantilla de Inicio de Sesión

Crea una plantilla para el inicio de sesión de usuarios en templates/accounts/login.html:

<!-- templates/accounts/login.html -->

<!DOCTYPE html>
<html>
<head>
    <title>Login</title>
</head>
<body>
    <h2>Login</h2>
    <form method="post">
        {% csrf_token %}
        {{ form.as_p }}
        <button type="submit">Login</button>
    </form>
</body>
</html>

Protección de Vistas con Login Required

Uso del Decorador Login Required

Para restringir el acceso a ciertas vistas, usa el decorador login_required:

# accounts/views.py

from django.contrib.auth.decorators import login_required

@login_required
def profile(request):
    return render(request, 'accounts/profile.html')

Creación de la Plantilla de Perfil

Crea una plantilla para el perfil de usuario en templates/accounts/profile.html:

<!-- templates/accounts/profile.html -->

<!DOCTYPE html>
<html>
<head>
    <title>Profile</title>
</head>
<body>
    <h2>Profile</h2>
    <p>Welcome, {{ user.username }}!</p>
</body>
</html>

Añadir el Patrón de URL para el Perfil

Añade el patrón de URL para la vista de perfil:

# accounts/urls.py

urlpatterns = [
    ...
    path('profile/', views.profile, name='profile'),
]

Resumen

En esta sección, aprendiste los conceptos básicos de la autenticación de usuarios en Django. Cubrimos:

  • Configuración de un proyecto y una aplicación Django.
  • Uso del modelo de usuario incorporado de Django.
  • Creación de vistas de registro, inicio de sesión y perfil.
  • Protección de vistas con el decorador login_required.

Al entender estos fundamentos, puedes construir aplicaciones web más seguras y amigables para el usuario. En las próximas secciones, profundizaremos en temas de autenticación más avanzados, como modelos de usuario personalizados y autenticación de terceros.
