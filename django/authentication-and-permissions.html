Introducción

En esta sección, exploraremos cómo Django maneja la autenticación y los permisos. La autenticación es el proceso de verificar la identidad de un usuario, mientras que los permisos determinan lo que un usuario autenticado puede hacer. Cubriremos los siguientes temas:

  • Sistema de autenticación incorporado de Django
  • Modelo de usuario y gestión de usuarios
  • Permisos y autorización
  • Personalización de la autenticación y los permisos

Sistema de Autenticación Incorporado de Django

Django proporciona un sistema de autenticación robusto y flexible listo para usar. Esto incluye inicio de sesión de usuario, cierre de sesión, gestión de contraseñas y más.

Conceptos Clave

  • Modelo de Usuario: Representa a un usuario en el sistema.
  • Autenticación: Verificar la identidad de un usuario.
  • Autorización: Determinar lo que un usuario autenticado puede hacer.
  • Sesiones: Mecanismo para persistir el estado del usuario a través de las solicitudes.

Configuración Básica

Para usar el sistema de autenticación de Django, asegúrate de que django.contrib.auth y django.contrib.contenttypes estén incluidos en tu configuración INSTALLED_APPS.

# settings.py
INSTALLED_APPS = [
    ...
    'django.contrib.auth',
    'django.contrib.contenttypes',
    ...
]

Modelo de Usuario y Gestión de Usuarios

Creación y Gestión de Usuarios

Django proporciona un modelo User predeterminado que se puede usar para crear y gestionar usuarios.

Creación de un Usuario

Puedes crear un usuario usando el shell de Django o dentro de tus vistas.

# Usando el shell de Django
from django.contrib.auth.models import User

user = User.objects.create_user('john', 'john@example.com', 'password123')
user.save()

Autenticación de un Usuario

Para autenticar a un usuario, usa la función authenticate y luego inicia sesión con la función login.

from django.contrib.auth import authenticate, login

def my_view(request):
    username = request.POST['username']
    password = request.POST['password']
    user = authenticate(request, username=username, password=password)
    if user is not None:
        login(request, user)
        # Redirigir a una página de éxito.
    else:
        # Devolver un mensaje de error de 'inicio de sesión no válido'.

Gestión de Usuarios en Admin

La interfaz de administración de Django proporciona una manera conveniente de gestionar usuarios. Asegúrate de haber creado un superusuario para acceder a la interfaz de administración.

python manage.py createsuperuser

Permisos y Autorización

Permisos Incorporados

Django proporciona una manera simple de gestionar permisos usando los modelos User y Group.

Permisos Predeterminados

Cada modelo en Django obtiene automáticamente tres permisos: agregar, cambiar y eliminar.

# Verificación de permisos
user.has_perm('app_label.add_modelname')
user.has_perm('app_label.change_modelname')
user.has_perm('app_label.delete_modelname')

Permisos Personalizados

Puedes definir permisos personalizados en tus modelos.

# models.py
from django.db import models

class MyModel(models.Model):
    ...
    class Meta:
        permissions = [
            ("can_publish", "Puede publicar publicaciones"),
        ]

Uso de Permisos en Vistas

Puedes usar el decorador permission_required para restringir el acceso a vistas basadas en permisos.

from django.contrib.auth.decorators import permission_required

@permission_required('app_label.can_publish')
def my_view(request):
    ...

Personalización de la Autenticación y los Permisos

Modelo de Usuario Personalizado

Para requisitos de usuario más complejos, puedes crear un modelo de usuario personalizado.

# models.py
from django.contrib.auth.models import AbstractUser

class CustomUser(AbstractUser):
    # Agregar campos adicionales aquí
    bio = models.TextField(blank=True, null=True)

Backends de Autenticación Personalizados

Puedes crear backends de autenticación personalizados para implementar lógica de autenticación personalizada.

# backends.py
from django.contrib.auth.backends import BaseBackend
from django.contrib.auth.models import User

class EmailBackend(BaseBackend):
    def authenticate(self, request, username=None, password=None, **kwargs):
        try:
            user = User.objects.get(email=username)
            if user.check_password(password):
                return user
        except User.DoesNotExist:
            return None

    def get_user(self, user_id):
        try:
            return User.objects.get(pk=user_id)
        except User.DoesNotExist:
            return None

Uso de Backends Personalizados

Agrega tu backend personalizado a la configuración AUTHENTICATION_BACKENDS.

# settings.py
AUTHENTICATION_BACKENDS = [
    'django.contrib.auth.backends.ModelBackend',  # predeterminado
    'path.to.EmailBackend',
]

Conclusión

El sistema de autenticación y permisos de Django es poderoso y flexible, lo que te permite gestionar la autenticación y autorización de usuarios con facilidad. Al comprender las características incorporadas y aprender cómo personalizarlas, puedes implementar medidas de seguridad robustas en tus aplicaciones Django.
