Introducción a la Autenticación Multifactor (MFA)

La Autenticación Multifactor (MFA) es un sistema de seguridad que requiere más de un método de autenticación para verificar la identidad del usuario para un inicio de sesión u otra transacción. MFA combina dos o más credenciales independientes: lo que el usuario sabe (contraseña), lo que el usuario tiene (token de seguridad) y lo que el usuario es (verificación biométrica).

Conceptos Clave

  • Algo que Sabes: Típicamente una contraseña o PIN.
  • Algo que Tienes: Un dispositivo físico como un smartphone o un token de hardware.
  • Algo que Eres: Verificación biométrica como huellas dactilares o reconocimiento facial.

Importancia de MFA en AWS

Los entornos de AWS a menudo contienen datos sensibles y aplicaciones críticas. Implementar MFA agrega una capa adicional de seguridad, haciendo significativamente más difícil que usuarios no autorizados obtengan acceso.

Beneficios de MFA

  • Seguridad Mejorada: Reduce el riesgo de cuentas comprometidas.
  • Cumplimiento: Ayuda a cumplir con los requisitos regulatorios.
  • Responsabilidad del Usuario: Asegura que solo los usuarios autorizados puedan realizar ciertas acciones.

Configuración de MFA en AWS

Requisitos Previos

  • Una cuenta de AWS.
  • Acceso a la Consola de Administración de AWS.
  • Un dispositivo MFA compatible (por ejemplo, un smartphone con una aplicación MFA como Google Authenticator o un dispositivo de hardware MFA).

Pasos para Habilitar MFA para Usuarios IAM de AWS

  1. Iniciar sesión en la Consola de Administración de AWS.
  2. Navegar al Panel de IAM:
    • Ir al servicio IAM desde la Consola de Administración de AWS.
  3. Seleccionar Usuarios:
    • Elegir el usuario para el cual deseas habilitar MFA.
  4. Gestionar Dispositivo MFA:
    • En la sección de detalles del usuario, haz clic en la pestaña "Credenciales de seguridad".
    • Haz clic en "Gestionar" junto a la sección "Dispositivo MFA asignado".
  5. Elegir Tipo de Dispositivo MFA:
    • Seleccionar "Dispositivo MFA virtual" o "Dispositivo MFA de hardware".
  6. Configurar el Dispositivo MFA:
    • Para un dispositivo MFA virtual, usar una aplicación MFA para escanear el código QR proporcionado por AWS.
    • Para un dispositivo MFA de hardware, ingresar el número de serie del dispositivo.
  7. Asignar Dispositivo MFA:
    • Ingresar los códigos de autenticación generados por el dispositivo MFA para completar la configuración.

Ejemplo: Habilitación de Dispositivo MFA Virtual

1. Abre la consola IAM en https://console.aws.amazon.com/iam/.
2. En el panel de navegación, elige Usuarios.
3. Elige el nombre del usuario para el que deseas habilitar MFA.
4. Elige la pestaña Credenciales de seguridad.
5. En la sección Autenticación multifactor (MFA), elige Gestionar.
6. Elige Dispositivo MFA virtual y luego elige Continuar.
7. Abre tu aplicación MFA en tu smartphone y escanea el código QR que se muestra en la consola de AWS.
8. Ingresa los dos códigos de autenticación consecutivos generados por tu aplicación MFA.
9. Elige Asignar MFA.

Configuraciones Avanzadas de MFA

MFA para la Cuenta Root

  • Altamente Recomendado: La cuenta root tiene acceso sin restricciones a tus recursos de AWS.
  • Pasos: Similar a habilitar MFA para usuarios IAM pero realizado a través de la configuración de seguridad de la cuenta root.

MFA para AWS CLI y SDK

  • Credenciales de Seguridad Temporales: Usar AWS STS (Servicio de Token de Seguridad) para generar credenciales temporales que requieran MFA.
  • Ejemplo: 
    aws sts get-session-token --serial-number arn-del-dispositivo-mfa --token-code código-del-token

Imposición de MFA con Políticas IAM

  • Ejemplo de Política IAM: 
    {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Deny",
            "Action": "*",
            "Resource": "*",
            "Condition": {
                "BoolIfExists": {
                    "aws:MultiFactorAuthPresent": "false"
                }
            }
        }
    ]
}

Solución de Problemas de MFA

  • Dispositivo MFA Perdido: Usar la cuenta root para deshabilitar MFA para el usuario afectado y reconfigurarlo.
  • Problemas de Sincronización: Asegurarse de que la hora en tu dispositivo MFA esté sincronizada con los servidores de AWS.
  • Códigos de Respaldo: AWS no proporciona códigos de respaldo; considera tener un dispositivo MFA secundario.

Conclusión

La Autenticación Multifactor (MFA) es una medida de seguridad crítica para proteger tu entorno de AWS. Al requerir múltiples formas de verificación, MFA mejora significativamente la seguridad de tus recursos de AWS. Configurar MFA es sencillo y se puede hacer a través de la Consola de Administración de AWS, CLI o SDK. Las configuraciones avanzadas y las políticas pueden imponer aún más el uso de MFA, asegurando prácticas de seguridad robustas en toda tu infraestructura de AWS.
