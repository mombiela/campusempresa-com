Amazon S3 (Simple Storage Service) es un servicio de almacenamiento de objetos escalable proporcionado por AWS. Está diseñado para almacenar y recuperar cualquier cantidad de datos desde cualquier lugar en la web. Este tema te guiará a través de las características básicas y avanzadas de Amazon S3, ayudándote a entender cómo aprovecharlo para tus necesidades de almacenamiento.

Introducción a Amazon S3

  • ¿Qué es Amazon S3?

    • Amazon S3 es un servicio de almacenamiento de objetos que ofrece escalabilidad, disponibilidad de datos, seguridad y rendimiento líderes en la industria.
    • Está diseñado para almacenar y recuperar cualquier cantidad de datos desde cualquier lugar en la web.

  • Conceptos Clave:

    • Buckets: Contenedores para almacenar objetos.
    • Objetos: Archivos y metadatos almacenados en buckets.
    • Keys: Identificador único para un objeto dentro de un bucket.
    • Regiones: Ubicaciones geográficas donde se almacenan los buckets.

Configuración de Amazon S3

  • Creación de un Bucket S3:
    1. Inicia sesión en la Consola de Administración de AWS.
    2. Navega al servicio S3.
    3. Haz clic en "Crear bucket."
    4. Ingresa un nombre único para el bucket y selecciona una región.
    5. Configura opciones como versionado, registro y cifrado.
    6. Revisa y crea el bucket.
import boto3

# Crear un cliente S3
s3 = boto3.client('s3')

# Crear un bucket
s3.create_bucket(Bucket='mi-nuevo-bucket', CreateBucketConfiguration={
    'LocationConstraint': 'us-west-1'})
  • Subir Objetos a S3:
    • Puedes subir archivos a tu bucket S3 usando la Consola de Administración de AWS, AWS CLI o SDKs.
# Subir un archivo al bucket
s3.upload_file('archivo_local.txt', 'mi-nuevo-bucket', 'archivo_remoto.txt')

Gestión de Buckets y Objetos S3

  • Políticas y Permisos de Bucket:
    • Controla el acceso a tus buckets y objetos S3 usando políticas de bucket, políticas IAM y ACLs (Listas de Control de Acceso).
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::mi-nuevo-bucket/*"
    }
  ]
}
  • Versionado:
    • Habilita el versionado para mantener múltiples versiones de un objeto en el mismo bucket.
# Habilitar el versionado en un bucket
s3.put_bucket_versioning(Bucket='mi-nuevo-bucket', VersioningConfiguration={
    'Status': 'Enabled'})
  • Políticas de Ciclo de Vida:
    • Define reglas para gestionar el ciclo de vida de los objetos, como la transición de objetos a diferentes clases de almacenamiento o su expiración después de un cierto período.
{
  "Rules": [
    {
      "ID": "Mover a Glacier después de 30 días",
      "Prefix": "",
      "Status": "Enabled",
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "GLACIER"
        }
      ]
    }
  ]
}

Características Avanzadas de Amazon S3

  • Clases de Almacenamiento S3:
    • Standard: Almacenamiento de propósito general para datos accedidos frecuentemente.
    • Intelligent-Tiering: Mueve automáticamente los datos al nivel de acceso más rentable.
    • Standard-IA (Acceso Infrecuente): Para datos accedidos con menos frecuencia.
    • One Zone-IA: Opción de menor costo para datos accedidos infrecuentemente almacenados en una sola zona de disponibilidad.
    • Glacier: Almacenamiento de bajo costo para datos archivados.
    • Glacier Deep Archive: Almacenamiento de menor costo para retención a largo plazo.

| Clase de Almacenamiento | Caso de Uso | Disponibilidad | Durabilidad | |-------------------------|--------------------------------------|----------------|---------------------| | Standard | Datos accedidos frecuentemente | 99.99% | 99.999999999% | | Intelligent-Tiering | Patrones de acceso desconocidos o cambiantes | 99.9% | 99.999999999% | | Standard-IA | Datos accedidos infrecuentemente | 99.9% | 99.999999999% | | One Zone-IA | Datos no críticos accedidos infrecuentemente | 99.5% | 99.999999999% | | Glacier | Datos archivados | 99.99% | 99.999999999% | | Glacier Deep Archive | Datos archivados a largo plazo | 99.99% | 99.999999999% |

  • Replicación entre Regiones:
    • Replica automáticamente objetos entre diferentes regiones de AWS para redundancia y recuperación ante desastres.
# Habilitar la replicación entre regiones
s3.put_bucket_replication(Bucket='mi-nuevo-bucket', ReplicationConfiguration={
    'Role': 'arn:aws:iam::id-cuenta:role/rol-replicacion',
    'Rules': [
        {
            'Status': 'Enabled',
            'Prefix': '',
            'Destination': {
                'Bucket': 'arn:aws:s3:::bucket-destino'
            }
        }
    ]
})
  • Notificaciones de Eventos:
    • Configura S3 para enviar notificaciones a AWS Lambda, SNS o SQS cuando ocurran ciertos eventos, como la creación o eliminación de objetos.
# Configurar notificaciones de eventos
s3.put_bucket_notification_configuration(Bucket='mi-nuevo-bucket', NotificationConfiguration={
    'LambdaFunctionConfigurations': [
        {
            'LambdaFunctionArn': 'arn:aws:lambda:region:id-cuenta:function:mi-funcion',
            'Events': ['s3:ObjectCreated:*']
        }
    ]
})

Conclusión

Amazon S3 es un servicio de almacenamiento versátil y poderoso que puede manejar una amplia gama de necesidades de almacenamiento, desde datos accedidos frecuentemente hasta archivos a largo plazo. Al entender sus conceptos básicos, configurar y gestionar buckets y objetos, y aprovechar características avanzadas, puedes optimizar tu estrategia de almacenamiento y asegurar que tus datos sean seguros, disponibles y rentables.
