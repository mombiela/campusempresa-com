Introducción a AWS CloudFormation

AWS CloudFormation es un servicio que te ayuda a modelar y configurar tus recursos de Amazon Web Services para que puedas dedicar menos tiempo a gestionar esos recursos y más tiempo a centrarte en tus aplicaciones. Creas una plantilla que describe todos los recursos de AWS que deseas (como instancias de Amazon EC2 o instancias de Amazon RDS DB), y AWS CloudFormation se encarga de aprovisionar y configurar esos recursos por ti.

Conceptos Clave

  • Plantillas: Archivos de texto formateados en JSON o YAML que describen los recursos de AWS que deseas crear.
  • Pilas: Una colección de recursos de AWS que puedes gestionar como una sola unidad. Creas, actualizas y eliminas una colección de recursos creando, actualizando y eliminando pilas.
  • Conjuntos de Cambios: Una vista previa de los cambios que AWS CloudFormation realizará en tu pila cuando la actualices.
  • Recursos: Los componentes de AWS que conforman tu pila, como instancias EC2, buckets S3 y bases de datos RDS.
  • Parámetros: Valores de entrada que puedes pasar a tu plantilla en tiempo de ejecución para personalizar tu pila.
  • Salidas: Valores que se devuelven cada vez que ves las propiedades de tu pila.

Creando Tu Primera Plantilla de CloudFormation

Para comenzar con AWS CloudFormation, necesitas crear una plantilla. A continuación, se muestra un ejemplo simple de una plantilla de CloudFormation escrita en YAML que crea un bucket S3.

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  MyS3Bucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      BucketName: my-first-cloudformation-bucket

Explicación

  • AWSTemplateFormatVersion: Especifica la versión de la plantilla de AWS CloudFormation que estás utilizando.
  • Resources: Define los recursos de AWS que deseas crear.
  • MyS3Bucket: Nombre lógico para el bucket S3.
  • Type: Especifica el tipo de recurso de AWS (en este caso, un bucket S3).
  • Properties: Define las propiedades para el recurso (en este caso, el nombre del bucket).

Desplegando una Pila

Una vez que tienes tu plantilla, puedes desplegarla como una pila usando la Consola de Administración de AWS, AWS CLI o AWS SDKs.

Usando la Consola de Administración de AWS

  1. Abre la Consola de Administración de AWS.
  2. Navega al servicio CloudFormation.
  3. Haz clic en "Crear pila".
  4. Sube tu archivo de plantilla.
  5. Sigue las indicaciones para crear la pila.

Usando AWS CLI

aws cloudformation create-stack --stack-name my-first-stack --template-body file://template.yaml

Usando AWS SDK (Ejemplo en Python)

import boto3

client = boto3.client('cloudformation')

with open('template.yaml', 'r') as template_file:
    template_body = template_file.read()

response = client.create_stack(
    StackName='my-first-stack',
    TemplateBody=template_body
)

print(response)

Actualizando una Pila

Para actualizar una pila, modificas la plantilla y luego aplicas los cambios.

Ejemplo: Añadiendo una Tabla DynamoDB

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  MyS3Bucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      BucketName: my-first-cloudformation-bucket
  MyDynamoDBTable:
    Type: 'AWS::DynamoDB::Table'
    Properties:
      TableName: my-first-dynamodb-table
      AttributeDefinitions:
        - AttributeName: Id
          AttributeType: S
      KeySchema:
        - AttributeName: Id
          KeyType: HASH
      ProvisionedThroughput:
        ReadCapacityUnits: 5
        WriteCapacityUnits: 5

Aplicando la Actualización

aws cloudformation update-stack --stack-name my-first-stack --template-body file://updated-template.yaml

Funciones Avanzadas

Pilas Anidadas

Las pilas anidadas te permiten crear pilas como parte de otras pilas, lo que te permite reutilizar plantillas de CloudFormation.

StackSets

StackSets te permiten crear, actualizar o eliminar pilas en múltiples cuentas y regiones de AWS con una sola operación.

Funciones Intrínsecas

Las funciones intrínsecas en las plantillas de CloudFormation te permiten asignar valores a propiedades que no están disponibles hasta el tiempo de ejecución.

Ejemplo: Usando Ref y GetAtt

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  MyS3Bucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      BucketName: !Ref BucketNameParameter
  MyEC2Instance:
    Type: 'AWS::EC2::Instance'
    Properties:
      ImageId: ami-0ff8a91507f77f867
      InstanceType: t2.micro
      SecurityGroups:
        - !GetAtt MySecurityGroup.GroupId

Conclusión

AWS CloudFormation es una herramienta poderosa para automatizar el despliegue y la gestión de recursos de AWS. Al usar plantillas, puedes definir tu infraestructura como código, lo que facilita su replicación y gestión. Ya seas un principiante o un usuario avanzado, entender CloudFormation puede mejorar significativamente tu capacidad para gestionar entornos de AWS de manera eficiente.
