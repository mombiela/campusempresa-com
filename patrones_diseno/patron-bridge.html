Introducción

El Patrón Bridge es un patrón de diseño estructural que desacopla una abstracción de su implementación, permitiendo que ambas varíen de manera independiente. Este patrón es útil cuando se desea evitar un crecimiento exponencial de clases debido a la combinación de múltiples variantes de abstracción e implementación.

Conceptos Clave

  • Abstracción: La interfaz de alto nivel que define las operaciones.
  • Implementación: La interfaz que define las operaciones primitivas.
  • Refinamiento de Abstracción: Una extensión de la abstracción que añade más operaciones.
  • Implementación Concreta: Una implementación específica de la interfaz de implementación.

Estructura del Patrón Bridge

El Patrón Bridge se compone de los siguientes componentes:

  • Abstracción: Define la interfaz y mantiene una referencia a un objeto de tipo Implementación.
  • Implementación: Define la interfaz para todas las implementaciones concretas.
  • Refinamiento de Abstracción: Extiende la interfaz de Abstracción.
  • Implementación Concreta: Implementa la interfaz de Implementación.

Diagrama UML

| Componente | Descripción | |---------------------------|-----------------------------------------------------------------------------| | Abstraction | Define la interfaz y mantiene una referencia a un objeto de tipo Implementor. | | Implementor | Define la interfaz para todas las implementaciones concretas. | | RefinedAbstraction | Extiende la interfaz de Abstraction. | | ConcreteImplementor | Implementa la interfaz de Implementor. |

Ejemplo de Código

A continuación, se presenta un ejemplo en Python para ilustrar el Patrón Bridge:

# Implementor
class DrawingAPI:
    def draw_circle(self, x, y, radius):
        pass

# ConcreteImplementor 1
class DrawingAPI1(DrawingAPI):
    def draw_circle(self, x, y, radius):
        print(f"API1.circle at {x}:{y} radius {radius}")

# ConcreteImplementor 2
class DrawingAPI2(DrawingAPI):
    def draw_circle(self, x, y, radius):
        print(f"API2.circle at {x}:{y} radius {radius}")

# Abstraction
class Shape:
    def __init__(self, drawing_api):
        self.drawing_api = drawing_api

    def draw(self):
        pass

    def resize_by_percentage(self, pct):
        pass

# RefinedAbstraction
class CircleShape(Shape):
    def __init__(self, x, y, radius, drawing_api):
        super().__init__(drawing_api)
        self.x = x
        self.y = y
        self.radius = radius

    def draw(self):
        self.drawing_api.draw_circle(self.x, self.y, self.radius)

    def resize_by_percentage(self, pct):
        self.radius *= (1 + pct / 100)

# Uso del patrón Bridge
circle1 = CircleShape(1, 2, 3, DrawingAPI1())
circle2 = CircleShape(5, 7, 11, DrawingAPI2())

circle1.draw()
circle2.draw()

circle1.resize_by_percentage(50)
circle1.draw()

Explicación del Código

  • DrawingAPI: Es la interfaz de implementación que define el método draw_circle.
  • DrawingAPI1 y DrawingAPI2: Son implementaciones concretas de DrawingAPI que proporcionan diferentes maneras de dibujar un círculo.
  • Shape: Es la abstracción que mantiene una referencia a un objeto de tipo DrawingAPI.
  • CircleShape: Es una extensión de Shape que añade operaciones específicas para un círculo, como draw y resize_by_percentage.

Conclusión

El Patrón Bridge es una solución efectiva para desacoplar una abstracción de su implementación, permitiendo que ambas evolucionen de manera independiente. Esto es especialmente útil en sistemas complejos donde las combinaciones de abstracción e implementación pueden crecer exponencialmente. Al aplicar este patrón, se mejora la flexibilidad y la mantenibilidad del código.
