Introducción

El Patrón Decorator es un patrón de diseño estructural que permite añadir funcionalidades a un objeto de manera dinámica. Este patrón es útil cuando queremos extender las capacidades de una clase sin modificar su código original, promoviendo así el principio de abierto/cerrado (Open/Closed Principle) de SOLID.

Conceptos Clave

  • Composición sobre herencia: El Patrón Decorator favorece la composición en lugar de la herencia para extender las funcionalidades de los objetos.
  • Interfaz común: Tanto el objeto original como los decoradores implementan una interfaz común, lo que permite que los decoradores puedan ser intercambiables.
  • Encapsulamiento: Los decoradores encapsulan el objeto original y pueden añadir o modificar su comportamiento.

Estructura

El Patrón Decorator se compone de los siguientes elementos:

  • Componente (Component): Define la interfaz común para los objetos que pueden tener responsabilidades adicionales.
  • Componente Concreto (ConcreteComponent): Implementa la interfaz del Componente. Es el objeto al que se le pueden añadir responsabilidades adicionales.
  • Decorador (Decorator): Mantiene una referencia al Componente y define una interfaz que cumple con la del Componente.
  • Decorador Concreto (ConcreteDecorator): Extiende la funcionalidad del Componente, añadiendo responsabilidades adicionales.

Ejemplo de Código

A continuación, se muestra un ejemplo en Python para ilustrar cómo funciona el Patrón Decorator.

from abc import ABC, abstractmethod

# Componente
class Beverage(ABC):
    @abstractmethod
    def cost(self) -> float:
        pass

# Componente Concreto
class Espresso(Beverage):
    def cost(self) -> float:
        return 1.99

# Decorador
class AddonDecorator(Beverage, ABC):
    def __init__(self, beverage: Beverage):
        self._beverage = beverage

# Decorador Concreto
class Milk(AddonDecorator):
    def cost(self) -> float:
        return self._beverage.cost() + 0.50

class Mocha(AddonDecorator):
    def cost(self) -> float:
        return self._beverage.cost() + 0.75

# Uso del Patrón Decorator
beverage = Espresso()
print(f"Cost of Espresso: ${beverage.cost()}")

beverage_with_milk = Milk(beverage)
print(f"Cost of Espresso with Milk: ${beverage_with_milk.cost()}")

beverage_with_milk_and_mocha = Mocha(beverage_with_milk)
print(f"Cost of Espresso with Milk and Mocha: ${beverage_with_milk_and_mocha.cost()}")

Explicación del Código

  1. Componente: Beverage es una clase abstracta que define el método cost.
  2. Componente Concreto: Espresso es una implementación concreta de Beverage.
  3. Decorador: AddonDecorator es una clase abstracta que implementa Beverage y contiene una referencia a un objeto Beverage.
  4. Decoradores Concretos: Milk y Mocha son decoradores concretos que añaden funcionalidades adicionales al objeto Beverage.

Comparación: Herencia vs Decorator

| Característica | Herencia | Decorator | |----------------------|-----------------------------------|------------------------------------| | Extensibilidad | Limitada a la jerarquía de clases | Muy flexible, se pueden combinar múltiples decoradores | | Complejidad | Puede volverse compleja con muchas subclases | Más sencilla, ya que se añaden funcionalidades de manera modular | | Reutilización | Menos reutilizable | Alta reutilización de decoradores |

Conclusión

El Patrón Decorator es una poderosa herramienta para añadir funcionalidades a los objetos de manera dinámica y flexible. Al favorecer la composición sobre la herencia, este patrón permite crear sistemas más modulares y fáciles de mantener. Es especialmente útil en situaciones donde las funcionalidades adicionales pueden ser combinadas de diferentes maneras, proporcionando una gran flexibilidad en el diseño del software.
