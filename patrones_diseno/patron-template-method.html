Patrón Template Method
Patrón Visitor ►

Introducción

El patrón Template Method es un patrón de diseño de comportamiento que define el esqueleto de un algoritmo en una operación, diferiendo algunos pasos a las subclases. Permite que las subclases redefinan ciertos pasos de un algoritmo sin cambiar la estructura del mismo.

Conceptos Clave

  • Clase Abstracta: Define el esqueleto del algoritmo y contiene el método template.
  • Método Template: Un método en la clase abstracta que define el algoritmo y llama a los métodos abstractos.
  • Métodos Abstractos: Métodos que deben ser implementados por las subclases.
  • Ganancia: Promueve la reutilización de código y la extensión del comportamiento sin modificar el código existente.

Estructura del Patrón

  1. Clase Abstracta: Contiene el método template y los métodos abstractos.
  2. Subclases Concretas: Implementan los métodos abstractos definidos en la clase abstracta.

Diagrama de Clases

| Clase Abstracta | Subclase Concreta | |-----------------|-------------------| | + templateMethod() | + metodoConcreto1() | | + metodoAbstracto1() | + metodoConcreto2() | | + metodoAbstracto2() | |

Ejemplo de Código

A continuación, se presenta un ejemplo en Python para ilustrar el patrón Template Method:

from abc import ABC, abstractmethod

class AbstractClass(ABC):
    def template_method(self):
        self.base_operation1()
        self.required_operations1()
        self.base_operation2()
        self.hook1()
        self.required_operations2()
        self.base_operation3()
        self.hook2()

    def base_operation1(self):
        print("AbstractClass dice: Estoy haciendo la mayor parte del trabajo")

    def base_operation2(self):
        print("AbstractClass dice: Pero permito que las subclases sobrescriban algunas operaciones")

    def base_operation3(self):
        print("AbstractClass dice: Pero yo soy quien llama a las operaciones")

    @abstractmethod
    def required_operations1(self):
        pass

    @abstractmethod
    def required_operations2(self):
        pass

    def hook1(self):
        pass

    def hook2(self):
        pass

class ConcreteClass1(AbstractClass):
    def required_operations1(self):
        print("ConcreteClass1 dice: Implementé Operation1")

    def required_operations2(self):
        print("ConcreteClass1 dice: Implementé Operation2")

class ConcreteClass2(AbstractClass):
    def required_operations1(self):
        print("ConcreteClass2 dice: Implementé Operation1")

    def required_operations2(self):
        print("ConcreteClass2 dice: Implementé Operation2")

    def hook1(self):
        print("ConcreteClass2 dice: Sobrescribí Hook1")

def client_code(abstract_class: AbstractClass):
    abstract_class.template_method()

if __name__ == "__main__":
    print("El mismo código de cliente puede funcionar con diferentes subclases:")
    client_code(ConcreteClass1())
    print("")
    print("El mismo código de cliente puede funcionar con diferentes subclases:")
    client_code(ConcreteClass2())

Explicación del Código

  • AbstractClass: Define el método template_method que establece el esqueleto del algoritmo. Incluye métodos abstractos (required_operations1 y required_operations2) que deben ser implementados por las subclases.
  • ConcreteClass1 y ConcreteClass2: Implementan los métodos abstractos definidos en AbstractClass. ConcreteClass2 también sobrescribe un método hook1 opcional.
  • client_code: Función que acepta una instancia de AbstractClass y llama al método template_method.

Conclusión

El patrón Template Method es útil cuando se desea definir el esqueleto de un algoritmo en una clase base y permitir que las subclases implementen detalles específicos. Esto promueve la reutilización de código y facilita la extensión del comportamiento sin modificar el código existente. Es especialmente útil en situaciones donde se tienen algoritmos similares con pasos específicos que varían entre implementaciones.
