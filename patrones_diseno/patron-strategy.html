Introducción al Patrón Strategy

El Patrón Strategy es un patrón de diseño de comportamiento que permite definir una familia de algoritmos, encapsular cada uno de ellos y hacerlos intercambiables. Este patrón permite que el algoritmo varíe independientemente de los clientes que lo utilizan.

Conceptos Clave

  • Encapsulación de Algoritmos: Cada algoritmo se encapsula en su propia clase.
  • Intercambiabilidad: Los algoritmos pueden ser intercambiados sin modificar el código del cliente.
  • Contexto: La clase que utiliza el algoritmo y que puede cambiar de estrategia en tiempo de ejecución.

Estructura del Patrón Strategy

El Patrón Strategy se compone de los siguientes componentes:

  1. Estrategia (Strategy): Interfaz común para todos los algoritmos.
  2. Estrategias Concretas (ConcreteStrategy): Implementaciones específicas de la interfaz de estrategia.
  3. Contexto (Context): Clase que utiliza una estrategia concreta.

Diagrama UML

| Componente | Descripción | |------------------|-----------------------------------------------------------------------------| | Strategy | Interfaz común para todos los algoritmos. | | ConcreteStrategyA | Implementación concreta de la estrategia A. | | ConcreteStrategyB | Implementación concreta de la estrategia B. | | Context | Clase que utiliza una estrategia concreta y puede cambiarla en tiempo de ejecución. |

Ejemplo de Código en Python

A continuación, se presenta un ejemplo de cómo implementar el Patrón Strategy en Python.

Definición de la Interfaz de Estrategia

from abc import ABC, abstractmethod

class Strategy(ABC):
    @abstractmethod
    def execute(self, data):
        pass

Implementaciones Concretas de Estrategia

class ConcreteStrategyA(Strategy):
    def execute(self, data):
        return sorted(data)

class ConcreteStrategyB(Strategy):
    def execute(self, data):
        return sorted(data, reverse=True)

Clase Contexto

class Context:
    def __init__(self, strategy: Strategy):
        self._strategy = strategy

    def set_strategy(self, strategy: Strategy):
        self._strategy = strategy

    def execute_strategy(self, data):
        return self._strategy.execute(data)

Uso del Patrón Strategy

if __name__ == "__main__":
    data = [5, 2, 9, 1]

    context = Context(ConcreteStrategyA())
    print("Orden Ascendente:", context.execute_strategy(data))

    context.set_strategy(ConcreteStrategyB())
    print("Orden Descendente:", context.execute_strategy(data))

Ventajas y Desventajas

Ventajas

  • Flexibilidad: Permite cambiar de algoritmo en tiempo de ejecución.
  • Mantenimiento: Facilita la adición de nuevas estrategias sin modificar el código existente.
  • Reutilización: Las estrategias pueden ser reutilizadas en diferentes contextos.

Desventajas

  • Complejidad: Aumenta la complejidad del código debido a la introducción de múltiples clases.
  • Sobrecarga: Puede introducir una ligera sobrecarga debido a la delegación de tareas.

Conclusión

El Patrón Strategy es una poderosa herramienta para manejar algoritmos intercambiables y encapsular comportamientos específicos. Al utilizar este patrón, los desarrolladores pueden crear sistemas más flexibles y mantenibles, permitiendo que los algoritmos varíen independientemente de los clientes que los utilizan. Sin embargo, es importante considerar la complejidad adicional que puede introducir en el diseño del sistema.
