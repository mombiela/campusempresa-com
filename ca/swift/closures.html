Introducció a les Closures

Les closures són blocs de funcionalitat autònoms que es poden passar i utilitzar en el teu codi. A Swift, les closures poden capturar i emmagatzemar referències a variables i constants del context en què es defineixen. Aquesta característica es coneix com "tancar sobre" aquestes constants i variables, d'aquí el nom "closures."

Conceptes Clau

  • Expressions de Closure: Closures sense nom escrites amb una sintaxi lleugera.
  • Captura de Valors: Les closures poden capturar i emmagatzemar referències a variables i constants del context circumdant.
  • Sintaxi de Closure Final: Una comoditat sintàctica per passar una closure com a últim argument d'una funció.
  • Closures d'Escapament: Closures que es criden després que la funció a la qual es van passar retorni.
  • Autoclosures: Closures que es creen automàticament per embolicar una expressió passada com a argument a una funció.

Sintaxi Bàsica de les Closures

Les closures a Swift tenen una sintaxi neta i concisa. Aquí tens un exemple bàsic d'una closure:

let greetingClosure = {
    print("Hola, Món!")
}

greetingClosure()  // Sortida: Hola, Món!

Expressions de Closure

Les expressions de closure són una manera d'escriure closures en línia de manera breu i clara. Aquí tens un exemple d'una expressió de closure utilitzada per ordenar una matriu:

let names = ["Chris", "Alex", "Ewa", "Barry", "Daniella"]

let sortedNames = names.sorted(by: { (s1: String, s2: String) -> Bool in
    return s1 < s2
})

print(sortedNames)  // Sortida: ["Alex", "Barry", "Chris", "Daniella", "Ewa"]

Inferència de Tipus del Context

Swift pot inferir els tipus de paràmetres i el tipus de retorn quan la closure s'utilitza com a argument d'una funció:

let sortedNames = names.sorted(by: { s1, s2 in return s1 < s2 })

Noms de Paràmetres Abreujats

Swift proporciona noms de paràmetres abreujats per fer les closures més concises:

let sortedNames = names.sorted(by: { $0 < $1 })

Sintaxi de Closure Final

Si una closure és l'últim argument d'una funció, pots utilitzar la sintaxi de closure final:

let sortedNames = names.sorted { $0 < $1 }

Captura de Valors

Les closures poden capturar i emmagatzemar referències a variables i constants del context circumdant:

func makeIncrementer(incrementAmount: Int) -> () -> Int {
    var total = 0
    let incrementer: () -> Int = {
        total += incrementAmount
        return total
    }
    return incrementer
}

let incrementByTwo = makeIncrementer(incrementAmount: 2)
print(incrementByTwo())  // Sortida: 2
print(incrementByTwo())  // Sortida: 4

Closures d'Escapament

Una closure d'escapament és una closure que es crida després que la funció a la qual es va passar retorni. Declares una closure d'escapament utilitzant la paraula clau @escaping:

var completionHandlers: [() -> Void] = []

func someFunctionWithEscapingClosure(completionHandler: @escaping () -> Void) {
    completionHandlers.append(completionHandler)
}

func someFunction() {
    someFunctionWithEscapingClosure {
        print("Aquesta és una closure d'escapament.")
    }
}

someFunction()
completionHandlers.first?()  // Sortida: Aquesta és una closure d'escapament.

Autoclosures

Una autoclosure és una closure que es crea automàticament per embolicar una expressió passada com a argument a una funció. No pren cap argument i retorna el valor de l'expressió:

func serve(customer customerProvider: @autoclosure () -> String) {
    print("Ara servint \(customerProvider())!")
}

serve(customer: "John Doe")  // Sortida: Ara servint John Doe!

Conclusió

Les closures són una característica poderosa a Swift que et permeten encapsular funcionalitat i passar-la pel teu codi. Poden capturar i emmagatzemar referències a variables i constants del context en què es defineixen, fent-les increïblement flexibles. Entendre les closures és essencial per dominar Swift, ja que s'utilitzen extensament en moltes APIs i patrons de Swift.
