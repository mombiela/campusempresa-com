Introducció a la Concurrència i Multithreading

La concurrència i el multithreading són conceptes essencials en la programació moderna, permetent que les aplicacions realitzin múltiples tasques simultàniament, millorant el rendiment i la capacitat de resposta. En Swift, aquests conceptes es implementen utilitzant diverses eines i tècniques.

Conceptes Clau

  • Concurrència: La capacitat d'un programa per gestionar múltiples tasques al mateix temps.
  • Multithreading: La capacitat d'una CPU per proporcionar múltiples fils d'execució de manera concurrent.
  • Fil: La unitat més petita de processament que es pot realitzar en un sistema operatiu.
  • GCD (Grand Central Dispatch): Una API de baix nivell per gestionar operacions concurrents.
  • OperationQueue: Una abstracció de nivell superior per gestionar operacions concurrents.

Grand Central Dispatch (GCD)

GCD és una eina poderosa per gestionar tasques concurrents en Swift. Proporciona una manera senzilla i eficient d'executar codi de manera asíncrona.

Conceptes Bàsics de GCD

  • Dispatch Queue: Una cua que gestiona l'execució de tasques.
    • Main Queue: S'executa en el fil principal, utilitzat per actualitzacions de la interfície d'usuari.
    • Global Queues: Cues concurrents proporcionades pel sistema.
    • Custom Queues: Cues creades pel desenvolupador.

Creació i Ús de Dispatch Queues

// Creació d'una cua serial personalitzada
let serialQueue = DispatchQueue(label: "com.example.serialQueue")

// Creació d'una cua concurrent personalitzada
let concurrentQueue = DispatchQueue(label: "com.example.concurrentQueue", attributes: .concurrent)

// Execució asíncrona d'una tasca en una cua global
DispatchQueue.global().async {
    // Realitzar alguna tasca en segon pla
    print("Tasques executant-se en segon pla")
}

// Execució síncrona d'una tasca en la cua principal
DispatchQueue.main.sync {
    // Realitzar alguna tasca en el fil principal
    print("Tasques executant-se en el fil principal")
}

Dispatch Groups

Els Dispatch Groups permeten agrupar múltiples tasques i ser notificat quan totes es completen.

let group = DispatchGroup()

DispatchQueue.global().async(group: group) {
    // Tasca 1
    print("Tasca 1 iniciada")
    sleep(2)
    print("Tasca 1 completada")
}

DispatchQueue.global().async(group: group) {
    // Tasca 2
    print("Tasca 2 iniciada")
    sleep(1)
    print("Tasca 2 completada")
}

group.notify(queue: DispatchQueue.main) {
    print("Totes les tasques completades")
}

OperationQueue

OperationQueue és una abstracció de nivell superior sobre GCD, proporcionant més control sobre l'execució de tasques concurrents.

Conceptes Bàsics d'OperationQueue

  • Operation: Una unitat de treball que es pot afegir a una OperationQueue.
  • OperationQueue: Una cua que gestiona l'execució d'objectes Operation.

Creació i Ús d'OperationQueue

let operationQueue = OperationQueue()

let operation1 = BlockOperation {
    print("Operació 1 iniciada")
    sleep(2)
    print("Operació 1 completada")
}

let operation2 = BlockOperation {
    print("Operació 2 iniciada")
    sleep(1)
    print("Operació 2 completada")
}

operationQueue.addOperation(operation1)
operationQueue.addOperation(operation2)

Dependències Entre Operacions

Les operacions poden tenir dependències, assegurant que una operació no comenci fins que una altra hagi acabat.

let operation1 = BlockOperation {
    print("Operació 1 iniciada")
    sleep(2)
    print("Operació 1 completada")
}

let operation2 = BlockOperation {
    print("Operació 2 iniciada")
    sleep(1)
    print("Operació 2 completada")
}

operation2.addDependency(operation1)

let operationQueue = OperationQueue()
operationQueue.addOperations([operation1, operation2], waitUntilFinished: false)

Temes Avançats

Seguretat de Fil

Assegurar que els recursos compartits s'accedeixin de manera segura en un entorn concurrent és crucial en la programació concurrent.

class ThreadSafeCounter {
    private var value = 0
    private let queue = DispatchQueue(label: "com.example.threadSafeCounter")

    func increment() {
        queue.sync {
            value += 1
        }
    }

    func getValue() -> Int {
        return queue.sync {
            return value
        }
    }
}

Async/Await (Swift 5.5+)

Swift 5.5 va introduir async/await per gestionar codi asíncron d'una manera més llegible i mantenible.

func fetchData() async -> String {
    // Simular una crida de xarxa
    await Task.sleep(2 * 1_000_000_000) // 2 segons
    return "Dades recuperades"
}

Task {
    let data = await fetchData()
    print(data)
}

Conclusió

La concurrència i el multithreading són eines poderoses que poden millorar significativament el rendiment i la capacitat de resposta de les teves aplicacions Swift. Entenent i utilitzant GCD, OperationQueue i tècniques avançades com async/await, pots gestionar efectivament les tasques concurrents i assegurar que les teves aplicacions funcionin sense problemes. Recorda sempre gestionar els recursos compartits amb cura per evitar condicions de carrera i assegurar la seguretat de fil.
