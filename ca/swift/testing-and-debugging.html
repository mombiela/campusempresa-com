Introducció

Les proves i la depuració són habilitats crucials per a qualsevol desenvolupador de Swift. Aquesta secció et guiarà a través dels conceptes bàsics de les proves i la depuració en Swift, des d'escriure el teu primer cas de prova fins a utilitzar eines avançades de depuració.

Proves Unitàries amb XCTest

Què són les Proves Unitàries?

  • Les proves unitàries impliquen provar components individuals del teu codi per assegurar-te que funcionen com s'espera.
  • En Swift, el framework XCTest és comunament utilitzat per a les proves unitàries.

Configuració de XCTest

  1. Crear un Objectiu de Prova:
    • Quan crees un nou projecte a Xcode, pots afegir un objectiu de prova seleccionant l'opció "Include Unit Tests".
  2. Importar XCTest:
    • Importa el framework XCTest en els teus arxius de prova.
    import XCTest

Escriure el Teu Primer Cas de Prova

  • Un cas de prova és una classe que hereta de XCTestCase.
  • Cada mètode de prova hauria de començar amb la paraula test.
import XCTest
@testable import YourProjectName

class MyFirstTests: XCTestCase {
    func testAddition() {
        let sum = 2 + 3
        XCTAssertEqual(sum, 5, "Sum should be 5")
    }
}

Executar Proves

  • Pots executar proves prement Cmd+U a Xcode o seleccionant el navegador de proves i fent clic al botó d'execució al costat del teu cas de prova.

Desenvolupament Guiat per Proves (TDD)

  • TDD és un procés de desenvolupament de programari on escrius proves abans d'escriure el codi real.
  • El cicle és: Escriure una prova -> Executar la prova (hauria de fallar) -> Escriure el codi -> Executar la prova (hauria de passar) -> Refactoritzar.

Exemple de TDD

  1. Escriure una Prova: 
    func testMultiplication() {
    let product = multiply(2, 3)
    XCTAssertEqual(product, 6, "Product should be 6")
}
  2. Executar la Prova (falla):
    • La funció multiply encara no existeix.
  3. Escriure el Codi: 
    func multiply(_ a: Int, _ b: Int) -> Int {
    return a * b
}
  4. Executar la Prova (passa):
    • Ara la prova hauria de passar.
  5. Refactoritzar:
    • Neteja el codi si és necessari.

Depuració a Xcode

Punts de Trencament

  • Els punts de trencament et permeten pausar l'execució del teu codi en una línia específica.
  • Pots afegir un punt de trencament fent clic a la vora al costat del número de línia a Xcode.

LLDB (Depurador de Baix Nivell)

  • LLDB és un depurador potent integrat a Xcode.
  • Pots utilitzar comandes LLDB a la consola de depuració per inspeccionar variables, controlar l'execució i més.

Comandes Comunes de LLDB

  • po: Imprimeix la descripció de l'objecte. 
    po myVariable
  • p: Imprimeix el valor d'una variable. 
    p myVariable
  • bt: Imprimeix la traça de la pila del fil actual. 
    bt

Consells de Depuració

  • Utilitza Declaracions Print: De vegades, simples declaracions print poden ajudar-te a entendre què està fallant. 
    print("Value of myVariable: \\(myVariable)")
  • Comprova si és Nil: Assegura't que les variables opcionals no són nil abans d'utilitzar-les. 
    if let value = myOptional {
    // Utilitza value
} else {
    print("myOptional is nil")
}

Tècniques Avançades de Prova

Mocking i Stubbing

  • Mocking: Crear objectes que simulen el comportament d'objectes reals.
  • Stubbing: Proporcionar respostes predefinides a les crides de mètodes.

Exemple de Mocking

  • Utilitza protocols per definir el comportament dels teus objectes.
  • Crea classes mock que compleixin aquests protocols.
protocol NetworkService {
    func fetchData() -> String
}

class MockNetworkService: NetworkService {
    func fetchData() -> String {
        return "Mock Data"
    }
}

Proves de Rendiment

  • XCTest també et permet escriure proves de rendiment per mesurar l'eficiència del teu codi.
func testPerformanceExample() {
    self.measure {
        // Codi per mesurar el temps
        let _ = (0..<1000).map { $0 * $0 }
    }
}

Conclusió

Les proves i la depuració són habilitats essencials per a qualsevol desenvolupador de Swift. En dominar les proves unitàries amb XCTest, adoptar pràctiques de TDD i utilitzar les potents eines de depuració de Xcode, pots escriure codi més fiable i mantenible. Recorda, l'objectiu no és només trobar i arreglar errors, sinó entendre millor el teu codi i assegurar-te que es comporta com s'espera sota diverses condicions.
