Visió general

En aquesta secció final del nostre curs de Swift, consolidarem tot el que hem après des de nivells principiants fins a avançats. Això inclourà una revisió completa dels conceptes clau, una presentació del projecte final i una sessió de preguntes i respostes per abordar qualsevol pregunta pendent.

Revisió de conceptes clau

Revisem alguns dels temes més importants coberts en aquest curs:

  • Variables i constants
  • Tipus de dades
  • Flux de control
  • Funcions

  • Programació orientada a objectes (OOP)
  • Protocols i extensions
  • Gestió d'errors

Variables i constants

Les variables i constants són els blocs de construcció de qualsevol programa en Swift.

var variableName = "Hello, Swift!"
let constantName = "Hello, World!"
  • var s'utilitza per declarar una variable que es pot canviar.
  • let s'utilitza per declarar una constant que no es pot canviar.

Tipus de dades

Swift té diversos tipus de dades, incloent Int, Double, String i Bool.

let age: Int = 25
let pi: Double = 3.14159
let name: String = "Alice"
let isSwiftFun: Bool = true

Flux de control

El flux de control inclou declaracions condicionals i bucles.

if isSwiftFun {
    print("Swift is fun!")
} else {
    print("Swift is not fun.")
}

for i in 1...5 {
    print(i)
}

Funcions

Les funcions són blocs de codi reutilitzables.

func greet(name: String) -> String {
    return "Hello, \(name)!"
}

let greeting = greet(name: "Alice")
print(greeting)

Closures

Les closures són blocs de funcionalitat autònoms que es poden passar i utilitzar en el teu codi.

let closure = { (name: String) -> String in
    return "Hello, \(name)!"
}

print(closure("Bob"))

Programació orientada a objectes (OOP)

La OOP implica crear classes i objectes.

class Person {
    var name: String
    var age: Int
    
    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }
    
    func greet() -> String {
        return "Hello, my name is \(name)."
    }
}

let person = Person(name: "Alice", age: 25)
print(person.greet())

Protocols i extensions

Els protocols defineixen un pla de mètodes, propietats i altres requisits.

protocol Greetable {
    func greet() -> String
}

extension Person: Greetable {}

Gestió d'errors

Swift proporciona una gestió robusta d'errors utilitzant do, try i catch.

enum CustomError: Error {
    case invalidInput
}

func checkInput(input: Int) throws {
    if input < 0 {
        throw CustomError.invalidInput
    }
}

do {
    try checkInput(input: -1)
} catch {

}

Concurrència

La concurrència en Swift es gestiona utilitzant Grand Central Dispatch (GCD) i cues d'operacions.

DispatchQueue.global().async {
    // Tasca en segon pla
    DispatchQueue.main.async {
        // Actualitzar la interfície d'usuari en el fil principal
    }
}

Exemple de projecte: Aplicació de llista de tasques

Aquí tens un exemple simple del que podria ser el teu projecte final:

import UIKit

class ToDoItem {
    var title: String
    var isCompleted: Bool
    
    init(title: String, isCompleted: Bool = false) {
        self.title = title
        self.isCompleted = isCompleted
    }
}

class ToDoList {
    var items: [ToDoItem] = []
    
    func addItem(title: String) {
        let newItem = ToDoItem(title: title)
        items.append(newItem)
    }
    
    func completeItem(at index: Int) {
        items[index].isCompleted = true
    }
}

let toDoList = ToDoList()
toDoList.addItem(title: "Finish Swift course")
toDoList.completeItem(at: 0)

for item in toDoList.items {
    print("\(item.title) - Completed: \(item.isCompleted)")
}
