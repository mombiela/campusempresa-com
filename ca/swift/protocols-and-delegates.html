Introducció als protocols

Els protocols en Swift defineixen un pla de mètodes, propietats i altres requisits que s'adapten a una tasca o funcionalitat particular. Les classes, estructures i enumeracions poden adoptar i conformar-se a aquests protocols per proporcionar implementacions reals dels requisits.

Conceptes clau

  • Definició de protocol: Un protocol es defineix utilitzant la paraula clau protocol.
  • Adopció i conformitat: Els tipus (classes, estructures, enumeracions) adopten un protocol llistant-lo després del seu nom de tipus, separat per un dos punts.
  • Requisits de mètodes i propietats: Els protocols poden requerir mètodes i propietats específiques per ser implementats.

Exemple

protocol Greetable {
    var greeting: String { get }
    func greet()
}

struct Person: Greetable {
    var greeting: String
    
    func greet() {
        print(greeting)
    }
}

let person = Person(greeting: "Hello, World!")
person.greet() // Output: Hello, World!

Explicació

  • Definició de protocol: protocol Greetable defineix un protocol amb una propietat greeting i un mètode greet().
  • Adopció i conformitat: struct Person: Greetable adopta el protocol Greetable i proporciona implementacions per a la propietat i el mètode requerits.

Característiques avançades dels protocols

Els protocols de Swift ofereixen característiques avançades com la herència de protocols, la composició de protocols i les extensions de protocols.

Herència de protocols

Un protocol pot heretar d'un o més altres protocols, afegint els seus propis requisits a sobre dels heretats.

protocol Named {
    var name: String { get }
}

protocol Aged {
    var age: Int { get }
}

protocol Person: Named, Aged {}

struct User: Person {
    var name: String
    var age: Int
}

let user = User(name: "Alice", age: 30)
print(user.name) // Output: Alice
print(user.age)  // Output: 30

Composició de protocols

Pots combinar múltiples protocols en un sol requisit utilitzant la composició de protocols.

func wishHappyBirthday(to celebrator: Named & Aged) {
    print("Happy Birthday, \(celebrator.name), you are now \(celebrator.age)!")
}

let birthdayPerson = User(name: "Bob", age: 25)
wishHappyBirthday(to: birthdayPerson) // Output: Happy Birthday, Bob, you are now 25!

Extensions de protocols

Les extensions de protocols et permeten proporcionar implementacions per defecte per a qualsevol requisit de mètode o propietat calculada d'aquest protocol.

extension Greetable {
    func greet() {
        print("Hi there!")
    }
}

struct Robot: Greetable {
    var greeting: String
}

let robot = Robot(greeting: "Beep Boop")
robot.greet() // Output: Hi there!

Delegats

La delegació és un patró de disseny que permet a una classe o estructura delegar algunes de les seves responsabilitats a una instància d'un altre tipus. Aquest patró s'utilitza sovint per gestionar esdeveniments o fonts de dades.

Conceptes clau

  • Protocol de delegat: Defineix un protocol que descriu les responsabilitats a delegar.
  • Propietat de delegat: La classe delegant manté una referència dèbil al delegat per evitar cicles de retenció.
  • Mètodes de delegat: El delegat implementa els mètodes del protocol per gestionar les tasques delegades.

Exemple

protocol TaskDelegate: AnyObject {
    func taskDidStart()
    func taskDidFinish()
}

class Task {
    weak var delegate: TaskDelegate?
    
    func start() {
        delegate?.taskDidStart()
        // Perform task...
        delegate?.taskDidFinish()
    }
}

class TaskHandler: TaskDelegate {
    func taskDidStart() {
        print("Task started")
    }
    
    func taskDidFinish() {
        print("Task finished")
    }
}

let task = Task()
let handler = TaskHandler()
task.delegate = handler
task.start()
// Output:
// Task started
// Task finished

Explicació

  • Protocol de delegat: protocol TaskDelegate defineix els mètodes taskDidStart i taskDidFinish.
  • Propietat de delegat: weak var delegate: TaskDelegate? a la classe Task manté una referència dèbil al delegat.
  • Mètodes de delegat: TaskHandler es conforma al TaskDelegate i implementa els mètodes requerits.

Conclusió

Els protocols i els delegats són eines poderoses en Swift que promouen la reutilització del codi i la separació de responsabilitats. Els protocols defineixen un pla de mètodes i propietats, mentre que els delegats permeten que els objectes es comuniquin i gestionin tasques de manera desacoblada. En dominar aquests conceptes, pots escriure codi Swift més modular, flexible i mantenible.
