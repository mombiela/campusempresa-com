Inicialització en Swift
Herència ►

La inicialització és un concepte fonamental en Swift que implica preparar una instància d'una classe, estructura o enumeració per al seu ús. Aquest tema cobrirà els conceptes bàsics de la inicialització, incloent inicialitzadors per defecte, inicialitzadors personalitzats i tècniques avançades d'inicialització.

Inicialitzadors per Defecte

Quan crees una nova instància d'una classe o estructura, Swift proporciona automàticament un inicialitzador per defecte si no has definit cap inicialitzador personalitzat.

Exemple

struct Person {
    var name: String
    var age: Int
}

let person = Person(name: "Alice", age: 30)
print(person.name) // Output: Alice
print(person.age)  // Output: 30

En aquest exemple, l'estructura Person té un inicialitzador per defecte membre a membre perquè no s'ha definit cap inicialitzador personalitzat.

Inicialitzadors Personalitzats

Pots definir inicialitzadors personalitzats per configurar les teves instàncies amb valors inicials específics o realitzar qualsevol configuració necessària abans que una instància estigui llesta per al seu ús.

Exemple

struct Person {
    var name: String
    var age: Int

    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }
}

let person = Person(name: "Bob", age: 25)
print(person.name) // Output: Bob
print(person.age)  // Output: 25

En aquest exemple, es defineix un inicialitzador personalitzat per inicialitzar les propietats name i age.

Paràmetres de l'Inicialitzador

Els inicialitzadors poden tenir paràmetres igual que les funcions, permetent-te passar valors quan crees una instància.

Exemple

struct Rectangle {
    var width: Double
    var height: Double

    init(width: Double, height: Double) {
        self.width = width
        self.height = height
    }

    func area() -> Double {
        return width * height
    }
}

let rectangle = Rectangle(width: 10.0, height: 5.0)
print(rectangle.area()) // Output: 50.0

Aquí, l'estructura Rectangle té un inicialitzador que pren width i height com a paràmetres.

Inicialitzadors Fallibles

De vegades, la inicialització pot fallar. Swift et permet definir inicialitzadors fallibles que poden retornar nil si la inicialització falla.

Exemple

struct Temperature {
    var celsius: Double

    init?(fahrenheit: Double) {
        guard fahrenheit >= -459.67 else {
            return nil
        }
        self.celsius = (fahrenheit - 32) * 5 / 9
    }
}

if let temp = Temperature(fahrenheit: -500) {
    print(temp.celsius)
} else {
    print("Invalid temperature")
}
// Output: Invalid temperature

En aquest exemple, l'inicialitzador retorna nil si el valor Fahrenheit proporcionat està per sota del zero absolut.

Delegació d'Inicialitzadors per a Tipus de Valor

Per a tipus de valor com estructures i enumeracions, pots delegar la inicialització a un altre inicialitzador dins del mateix tipus.

Exemple

struct Color {
    var red: Double
    var green: Double
    var blue: Double

    init(white: Double) {
        self.red = white
        self.green = white
        self.blue = white
    }

    init(red: Double, green: Double, blue: Double) {
        self.red = red
        self.green = green
        self.blue = blue
    }
}

let gray = Color(white: 0.5)
print(gray.red, gray.green, gray.blue) // Output: 0.5 0.5 0.5

En aquest exemple, l'estructura Color té dos inicialitzadors, i un d'ells delega a l'altre.

Herència i Sobreescriptura d'Inicialitzadors

Les classes poden heretar inicialitzadors de la seva superclasse i també poden sobreescriure'ls per proporcionar un comportament d'inicialització personalitzat.

Exemple

class Vehicle {
    var speed: Double

    init(speed: Double) {
        self.speed = speed
    }
}

class Car: Vehicle {
    var model: String

    init(speed: Double, model: String) {
        self.model = model
        super.init(speed: speed)
    }
}

let car = Car(speed: 120.0, model: "Sedan")
print(car.speed) // Output: 120.0
print(car.model) // Output: Sedan

En aquest exemple, la classe Car hereta de la classe Vehicle i sobreescriu l'inicialitzador per incloure una propietat addicional model.

Conclusió

La inicialització és una part crucial de la creació d'instàncies en Swift. Comprenent els inicialitzadors per defecte, inicialitzadors personalitzats, inicialitzadors fallibles, delegació d'inicialitzadors i herència d'inicialitzadors, pots gestionar eficaçment la configuració de les teves instàncies. Aquest coneixement t'ajudarà a escriure codi Swift més robust i flexible.
