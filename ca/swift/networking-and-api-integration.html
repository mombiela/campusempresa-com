Introducció

Les xarxes i la integració d'API són habilitats essencials per als desenvolupadors moderns d'iOS. Aquesta secció et guiarà a través dels conceptes bàsics de les xarxes, fer sol·licituds HTTP, analitzar JSON i integrar APIs a les teves aplicacions Swift. Al final d'aquest tema, seràs capaç d'obtenir dades d'internet i mostrar-les a la teva aplicació.

Conceptes Clau

  • Xarxes: El procés de connectar i comunicar-se amb altres ordinadors a través d'una xarxa.
  • API (Interfície de Programació d'Aplicacions): Un conjunt de regles que permeten que diferents entitats de programari es comuniquin entre si.
  • HTTP (Protocol de Transferència d'Hipertext): El protocol utilitzat per transmetre pàgines web a través d'internet.
  • JSON (Notació d'Objectes de JavaScript): Un format lleuger d'intercanvi de dades que és fàcil de llegir i escriure per als humans, i fàcil de parsejar i generar per a les màquines.

Fer Sol·licituds HTTP

Per interactuar amb serveis web, necessites fer sol·licituds HTTP. Swift proporciona diverses maneres de fer aquestes sol·licituds, sent URLSession la més comuna.

Sol·licitud GET Bàsica

Una sol·licitud GET s'utilitza per recuperar dades d'un servidor.

import Foundation

let url = URL(string: "https://api.example.com/data")!

let task = URLSession.shared.dataTask(with: url) { data, response, error in
    if let error = error {
        print("Error: \(error)")
        return
    }
    
    guard let data = data else {
        print("No data")
        return
    }
    
    // Analitzar les dades JSON
    do {
        if let json = try JSONSerialization.jsonObject(with: data, options: []) as? [String: Any] {
            print("Resposta JSON: \(json)")
        }
    } catch {
        print("Error analitzant JSON: \(error)")
    }
}

task.resume()

Explicació

  • URL: El punt final al qual estàs intentant arribar.
  • URLSession.shared.dataTask: Crea una tasca que recupera el contingut de l'URL especificat.
  • data, response, error: Els paràmetres del gestor de finalització que proporcionen les dades, la resposta i la informació d'error.
  • JSONSerialization: Converteix les dades JSON en un diccionari de Swift.

Analitzar JSON

Un cop hagis recuperat les dades, necessites analitzar-les. El protocol Codable de Swift fa que això sigui fàcil.

Utilitzant Codable

Defineix una struct que conformi Codable per representar les dades JSON.

import Foundation

struct User: Codable {
    let id: Int
    let name: String
    let email: String
}

let url = URL(string: "https://api.example.com/users/1")!

let task = URLSession.shared.dataTask(with: url) { data, response, error in
    if let error = error {
        print("Error: \(error)")
        return
    }
    
    guard let data = data else {
        print("No data")
        return
    }
    
    // Decodificar les dades JSON
    do {
        let user = try JSONDecoder().decode(User.self, from: data)
        print("Usuari: \(user)")
    } catch {
        print("Error decodificant JSON: \(error)")
    }
}

task.resume()

Explicació

  • User struct: Representa l'objecte JSON.
  • JSONDecoder: Decodifica les dades JSON en la struct User.

Xarxes Avançades

Per a tasques de xarxes més complexes, potser necessitis manejar diferents mètodes HTTP, capçaleres i dades de cos.

Sol·licitud POST amb Dades de Cos

Una sol·licitud POST s'utilitza per enviar dades a un servidor.

import Foundation

struct Post: Codable {
    let title: String
    let body: String
    let userId: Int
}

let url = URL(string: "https://jsonplaceholder.typicode.com/posts")!
var request = URLRequest(url: url)
request.httpMethod = "POST"
request.setValue("application/json", forHTTPHeaderField: "Content-Type")

let newPost = Post(title: "Hello, World!", body: "This is a new post.", userId: 1)
let jsonData = try! JSONEncoder().encode(newPost)

let task = URLSession.shared.uploadTask(with: request, from: jsonData) { data, response, error in
    if let error = error {
        print("Error: \(error)")
        return
    }
    
    guard let data = data else {
        print("No data")
        return
    }
    
    // Analitzar la resposta JSON
    do {
        let postResponse = try JSONDecoder().decode(Post.self, from: data)
        print("Resposta del Post: \(postResponse)")
    } catch {
        print("Error decodificant JSON: \(error)")
    }
}

task.resume()

Explicació

  • URLRequest: Representa una sol·licitud de càrrega d'URL d'una manera independent del protocol i l'esquema d'URL.
  • httpMethod: Especifica el mètode HTTP (GET, POST, etc.).
  • setValue: Estableix el valor del camp de capçalera HTTP donat.
  • uploadTask: Crea una tasca que carrega dades a l'URL especificat.

Conclusió

Les xarxes i la integració d'API són crucials per construir aplicacions d'iOS dinàmiques i basades en dades. Entenent com fer sol·licituds HTTP, analitzar JSON i manejar diferents mètodes HTTP, pots integrar efectivament diversos serveis web a les teves aplicacions Swift. Practica aquests conceptes amb diferents APIs per solidificar la teva comprensió i millorar les teves habilitats de desenvolupament d'aplicacions.
