Gestió d'Errors a Swift
Gestió de Memòria i ARC ►

La gestió d'errors és un aspecte crític de la programació que permet als desenvolupadors gestionar i respondre als errors de manera elegant. A Swift, la gestió d'errors està dissenyada per ser clara i concisa, facilitant als desenvolupadors l'escriptura de codi robust i resistent als errors.

Introducció a la Gestió d'Errors

  • Definició: La gestió d'errors és el procés de respondre i gestionar els errors que es produeixen durant l'execució d'un programa.
  • Importància: Una gestió d'errors adequada assegura que la teva aplicació pugui gestionar situacions inesperades sense bloquejar-se.

Conceptes Bàsics de la Gestió d'Errors

Protocol Error

A Swift, els errors es representen per valors de tipus que conformen el protocol Error.

enum NetworkError: Error {
    case badURL
    case requestFailed
    case unknown
}

Llançament d'Errors

Les funcions i mètodes poden llançar errors utilitzant la paraula clau throw.

func fetchData(from url: String) throws {
    guard url == "https://valid.url" else {
        throw NetworkError.badURL
    }
    // Lògica de recuperació de dades
}

Gestió d'Errors

Els errors es poden gestionar utilitzant blocs do-catch.

do {
    try fetchData(from: "https://invalid.url")
} catch NetworkError.badURL {
    print("URL invàlida.")
} catch {
    print("S'ha produït un error desconegut.")
}

Propagació d'Errors

Les funcions que poden llançar errors han de ser marcades amb la paraula clau throws. Els errors es poden propagar a la funció que crida.

func performNetworkRequest() throws {
    try fetchData(from: "https://invalid.url")
}

do {
    try performNetworkRequest()
} catch {
    print("S'ha produït un error: \(error)")
}

Gestió d'Errors Avançada

Tipus d'Errors Personalitzats

Pots definir tipus d'errors personalitzats per proporcionar més context sobre l'error.

enum FileError: Error {
    case fileNotFound(String)
    case unreadable
    case encodingFailed
}

func readFile(named fileName: String) throws {
    throw FileError.fileNotFound(fileName)
}

Múltiples Clausules Catch

Pots gestionar diferents errors per separat utilitzant múltiples clausules catch.

do {
    try readFile(named: "nonexistent.txt")
} catch FileError.fileNotFound(let fileName) {
    print("Fitxer no trobat: \(fileName)")
} catch FileError.unreadable {
    print("El fitxer no es pot llegir.")
} catch {
    print("S'ha produït un error desconegut.")
}

Convertir Errors a Valors Optionals

Utilitza try? per convertir errors a valors opcionals.

let data = try? fetchData(from: "https://invalid.url")
// data serà nil si es llança un error

Desactivar la Propagació d'Errors

Utilitza try! per desactivar la propagació d'errors i bloquejar-se si es produeix un error.

let data = try! fetchData(from: "https://valid.url")
// Això es bloquejarà si fetchData llança un error

Accions de Neteja

Utilitza defer per executar accions de neteja independentment de si es llança un error.

func processFile() throws {
    defer {
        print("Accions de neteja")
    }
    // Lògica de processament de fitxers
    throw FileError.unreadable
}

Resum

  • La gestió d'errors de Swift es basa en el protocol Error.
  • Els errors es llancen utilitzant la paraula clau throw i es gestionen utilitzant blocs do-catch.
  • Les funcions que llancen errors han de ser marcades amb throws.
  • Les tècniques avançades inclouen tipus d'errors personalitzats, múltiples clausules catch, convertir errors a valors opcionals i utilitzar defer per a accions de neteja.

Dominant la gestió d'errors a Swift, pots escriure codi més robust i mantenible que gestioni de manera elegant les situacions inesperades.
