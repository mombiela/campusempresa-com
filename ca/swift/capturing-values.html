Capturar valors és un concepte important en Swift, especialment quan es treballa amb closures. Aquest tema et guiarà a través dels conceptes bàsics de capturar valors, com funciona i les seves implicacions en la programació en Swift.

Què és Capturar Valors?

  • Definició: Capturar valors es refereix a la capacitat d'un closure per capturar i emmagatzemar referències a variables i constants del context circumdant en què es defineix.
  • Propòsit: Això permet que el closure utilitzi i modifiqui aquests valors fins i tot després que el context original hagi sortit de l'àmbit.

Com Funciona Capturar Valors

Quan un closure captura un valor, crea una referència forta a aquest valor. Això significa que el closure pot accedir i modificar el valor fins i tot si l'àmbit original on es va definir el valor ja no existeix.

Exemple: Capturant Valors en un Closure

func makeIncrementer(incrementAmount: Int) -> () -> Int {
    var total = 0
    
    let incrementer: () -> Int = {
        total += incrementAmount
        return total
    }
    
    return incrementer
}

let incrementByTwo = makeIncrementer(incrementAmount: 2)
print(incrementByTwo()) // Output: 2
print(incrementByTwo()) // Output: 4
  • Explicació:
    • makeIncrementer és una funció que pren un incrementAmount i retorna un closure.
    • El closure captura la variable total i el paràmetre incrementAmount.
    • Cada vegada que es crida el closure retornat, incrementa total per incrementAmount i retorna el nou total.

Capturar Valors i Gestió de Memòria

  • Referències Fortes: Els valors capturats són referenciats fortament pel closure, cosa que pot portar a cicles de retenció si no es gestionen adequadament.
  • Cicles de Retenció: Un cicle de retenció es produeix quan dos o més objectes mantenen referències fortes entre si, impedint que siguin desal·locats.

Exemple: Cicle de Retenció

class SomeClass {
    var value = 0
    lazy var incrementer: () -> Int = {
        self.value += 1
        return self.value
    }
}

let instance = SomeClass()
print(instance.incrementer()) // Output: 1
  • Explicació:
    • SomeClass té una propietat incrementer que és un closure que captura self.
    • Això crea un cicle de referència forta entre la instància de SomeClass i el closure, portant a un cicle de retenció.

Trencant Cicles de Retenció

  • Referències Weak i Unowned: Utilitza referències weak o unowned per trencar cicles de retenció.
    • weak: Utilitza quan la referència capturada pot convertir-se en nil.
    • unowned: Utilitza quan la referència capturada mai es convertirà en nil.

Exemple: Utilitzant Referències Weak

class SomeClass {
    var value = 0
    lazy var incrementer: () -> Int = { [weak self] in
        self?.value += 1
        return self?.value ?? 0
    }
}

let instance = SomeClass()
print(instance.incrementer()) // Output: 1
  • Explicació:
    • El closure captura self dèbilment, evitant un cicle de referència forta.
    • Si self es converteix en nil, el closure retorna de manera segura 0.

Resum

Capturar valors en closures de Swift et permet mantenir referències a variables i constants del context circumdant. Tot i que això és una característica poderosa, requereix una gestió acurada per evitar problemes de memòria com els cicles de retenció. Entenent com utilitzar referències weak i unowned, pots gestionar efectivament la memòria i assegurar que les teves aplicacions en Swift funcionin eficientment.
