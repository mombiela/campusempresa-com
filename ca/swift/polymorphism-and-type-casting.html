Introducció

El polimorfisme i la conversió de tipus són conceptes fonamentals en la programació orientada a objectes que permeten un codi més flexible i reutilitzable. En Swift, aquests conceptes s'implementen d'una manera que és tant potent com fàcil d'utilitzar. Aquesta secció cobrirà les bases del polimorfisme i la conversió de tipus, proporcionarà exemples i inclourà exercicis per consolidar la teva comprensió.

Polimorfisme

El polimorfisme permet que els objectes es tractin com a instàncies de la seva classe pare en lloc de la seva classe real. Això permet que una sola funció operi sobre diferents tipus d'objectes.

Conceptes Clau

  • Polimorfisme: La capacitat de diferents classes de respondre a la mateixa crida de mètode de diferents maneres.
  • Sobreescriptura de Mètodes: Les subclasses poden proporcionar una implementació específica d'un mètode que ja està definit en la seva superclasse.
  • Despatx Dinàmic: El procés de seleccionar quina implementació d'una operació polimòrfica (mètode) cridar en temps d'execució.

Exemple

class Animal {
    func makeSound() {
        print("Some generic animal sound")
    }
}

class Dog: Animal {
    override func makeSound() {
        print("Bark")
    }
}

class Cat: Animal {
    override func makeSound() {
        print("Meow")
    }
}

let animals: [Animal] = [Dog(), Cat(), Animal()]

for animal in animals {
    animal.makeSound()
}

En aquest exemple:

  • Tenim una classe base Animal amb un mètode makeSound.
  • Dog i Cat són subclasses de Animal que sobreescriuen el mètode makeSound.
  • Una matriu de tipus Animal conté instàncies de Dog, Cat i Animal.
  • Quan iterem a través de la matriu, el mètode makeSound sobreescrit es crida en funció del tipus real de l'objecte.

Conversió de Tipus

La conversió de tipus en Swift s'utilitza per comprovar el tipus d'una instància o per tractar aquesta instància com una superclasse o subclasse diferent d'algun lloc de la seva jerarquia de classes.

Conceptes Clau

  • Operador is: Comprova si una instància és d'un cert tipus de subclasse.
  • Operador as?: Intenta convertir a un tipus de subclasse i retorna un opcional.
  • Operador as!: Converteix forçosament a un tipus de subclasse i pot causar un error en temps d'execució si la conversió falla.
  • Operador as: S'utilitza per a la conversió ascendent i la conversió de tipus.

Exemple

class Vehicle {
    var name: String
    init(name: String) {
        self.name = name
    }
}

class Car: Vehicle {
    var numberOfDoors: Int
    init(name: String, numberOfDoors: Int) {
        self.numberOfDoors = numberOfDoors
        super.init(name: name)
    }
}

class Bicycle: Vehicle {
    var hasBasket: Bool
    init(name: String, hasBasket: Bool) {
        self.hasBasket = hasBasket
        super.init(name: name)
    }
}

let vehicles: [Vehicle] = [Car(name: "Sedan", numberOfDoors: 4), Bicycle(name: "Mountain Bike", hasBasket: true)]

for vehicle in vehicles {
    if let car = vehicle as? Car {
        print("Car: \(car.name), Doors: \(car.numberOfDoors)")
    } else if let bicycle = vehicle as? Bicycle {
        print("Bicycle: \(bicycle.name), Has Basket: \(bicycle.hasBasket)")
    }
}

En aquest exemple:

  • Tenim una classe base Vehicle amb subclasses Car i Bicycle.
  • Una matriu de tipus Vehicle conté instàncies de Car i Bicycle.
  • Utilitzem as? per convertir de manera segura cada Vehicle al seu tipus de subclasse específic i imprimir propietats específiques.

Exercicis

  1. Exercici de Polimorfisme: Crea una classe base Shape amb un mètode area(). Crea subclasses Circle i Rectangle que sobreescriguin el mètode area(). Crea una matriu d'objectes Shape i imprimeix l'àrea de cada forma.
  2. Exercici de Conversió de Tipus: Crea una classe base Employee amb subclasses Manager i Developer. Crea una matriu d'objectes Employee i utilitza la conversió de tipus per imprimir propietats específiques de Manager i Developer.

Conclusió

El polimorfisme i la conversió de tipus són eines essencials en Swift que permeten un codi més flexible i reutilitzable. El polimorfisme permet que els objectes es tractin com a instàncies de la seva classe pare, mentre que la conversió de tipus et permet comprovar i convertir tipus en temps d'execució. En dominar aquests conceptes, podràs escriure programes en Swift més dinàmics i eficients.
