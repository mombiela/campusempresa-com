Els objectes protegits en Ada proporcionen un mecanisme per a la programació concurrent segura i eficient. S'utilitzen per protegir dades compartides de l'accés concurrent, assegurant que només una tasca pot accedir a les dades protegides alhora. Això és crucial en sistemes en temps real i embeguts on la integritat de les dades i la sincronització són fonamentals.

Conceptes clau

  • Objecte protegit: Un constructe que encapsula dades i operacions, assegurant l'exclusió mútua i la sincronització.
  • Tipus protegit: La definició d'un objecte protegit, especificant les seves dades i operacions.
  • Procediments protegits: Operacions que modifiquen les dades protegides.
  • Funcions protegides: Operacions que llegeixen les dades protegides.
  • Entrades protegides: Operacions que inclouen mecanismes de sincronització, com ara esperar que una condició sigui certa.

Definició d'un tipus protegit

Un tipus protegit es defineix utilitzant la paraula clau protected. Inclou la declaració de dades i operacions (procediments, funcions i entrades).

protected type Shared_Counter is
   procedure Increment;
   function Value return Integer;
private
   Counter : Integer := 0;
end Shared_Counter;

Explicació

  • Declaració del tipus protegit: El tipus Shared_Counter inclou un procediment Increment i una funció Value.
  • Secció privada: La variable Counter és privada, el que significa que només pot ser accedida pels procediments i funcions dins del tipus protegit.

Implementació d'un tipus protegit

La implementació del tipus protegit especifica el comportament de les seves operacions.

protected body Shared_Counter is
   procedure Increment is
   begin
      Counter := Counter + 1;
   end Increment;

   function Value return Integer is
   begin
      return Counter;
   end Value;
end Shared_Counter;

Explicació

  • Procediment Increment: Augmenta el Counter en 1.
  • Funció Value: Retorna el valor actual de Counter.

Ús d'objectes protegits

Per utilitzar un objecte protegit, es declara una instància del tipus protegit i es criden les seves operacions.

procedure Main is
   Counter : Shared_Counter;
begin
   Counter.Increment;
   Ada.Text_IO.Put_Line("Counter Value: " & Integer'Image(Counter.Value));
end Main;

Explicació

  • Declaració del Counter: Es declara una instància de Shared_Counter.
  • Crides a operacions: Es crida el procediment Increment per augmentar el comptador, i es crida la funció Value per recuperar i imprimir el valor del comptador.

Entrades protegides

Les entrades protegides s'utilitzen per a operacions que requereixen sincronització, com ara esperar que una condició sigui certa.

protected type Synchronized_Buffer is
   entry Write(Item : in Integer);
   entry Read(Item : out Integer);
private
   Buffer : Integer;
   Full : Boolean := False;
end Synchronized_Buffer;

protected body Synchronized_Buffer is
   entry Write(Item : in Integer) when not Full is
   begin
      Buffer := Item;
      Full := True;
   end Write;

   entry Read(Item : out Integer) when Full is
   begin
      Item := Buffer;
      Full := False;
   end Read;
end Synchronized_Buffer;

Explicació

  • Entrada Write: Espera fins que el buffer no estigui ple abans d'escriure un element.
  • Entrada Read: Espera fins que el buffer estigui ple abans de llegir un element.
  • Condicions de sincronització: La clàusula when especifica la condició que ha de ser certa perquè l'entrada pugui continuar.

Conclusió

Els objectes protegits en Ada són una eina poderosa per gestionar l'accés concurrent a dades compartides. Proporcionen una manera estructurada d'assegurar la integritat de les dades i la sincronització, que és essencial en sistemes en temps real i embeguts. En encapsular dades i operacions dins d'un tipus protegit, Ada assegura que només una tasca pot accedir a les dades protegides alhora, prevenint condicions de carrera i assegurant una programació concurrent segura.
