Introducció als Sistemes en Temps Real

Els sistemes en temps real són sistemes que requereixen una resposta oportuna als esdeveniments externs. Aquests sistemes sovint s'utilitzen en aplicacions crítiques on els retards poden conduir a fallades del sistema o a una degradació significativa del rendiment.

Conceptes Clau

  • Determinisme: La capacitat d'un sistema per produir la mateixa sortida donada la mateixa entrada dins d'un període de temps previsible.
  • Latència: El retard de temps entre l'ocurrència d'un esdeveniment i la resposta del sistema.
  • Jitter: La variabilitat en el temps de resposta.
  • Planificació: El mètode pel qual les tasques s'assignen per executar-se en el processador.

Ada i Sistemes en Temps Real

Ada és un llenguatge de programació dissenyat amb suport integrat per a sistemes en temps real. Proporciona característiques que ajuden a escriure aplicacions en temps real fiables i eficients.

Característiques Clau d'Ada per a Sistemes en Temps Real

  • Tasques: El model de concurrència d'Ada permet la creació de tasques que poden executar-se en paral·lel.
  • Objectes Protegits: Mecanismes per assegurar l'accés segur als recursos compartits.
  • Rellotge en Temps Real: Instal·lacions per gestionar el temps i els retards.
  • Planificació de Prioritats: Suport per assignar prioritats a les tasques.

Tasques en Ada

Les tasques en Ada són similars als fils en altres llenguatges de programació. Permeten l'execució concurrent del codi.

Exemple: Creació i Execució de Tasques

with Ada.Text_IO; use Ada.Text_IO;

procedure Main is
   task type Worker is
      entry Start;
   end Worker;

   task body Worker is
   begin
      accept Start;
      Put_Line("La tasca Worker s'està executant.");
   end Worker;

   Worker_Task : Worker;
begin
   Worker_Task.Start;
   Put_Line("La tasca principal s'està executant.");
end Main;
  • Explicació: Aquest exemple demostra la creació d'una tasca (Worker) i la seva interacció amb el programa principal. La tasca Worker té un punt d'entrada Start que és cridat pel programa principal.

Objectes Protegits

Els objectes protegits en Ada proporcionen una manera de compartir dades de manera segura entre tasques.

Exemple: Ús d'Objectes Protegits

with Ada.Text_IO; use Ada.Text_IO;

protected Counter is
   procedure Increment;
   function Value return Integer;
private
   Count : Integer := 0;
end Counter;

protected body Counter is
   procedure Increment is
   begin
      Count := Count + 1;
   end Increment;

   function Value return Integer is
   begin
      return Count;
   end Value;
end Counter;

procedure Main is
begin
   Counter.Increment;
   Put_Line("Valor del comptador: " & Integer'Image(Counter.Value));
end Main;
  • Explicació: Aquest exemple mostra un objecte protegit Counter amb una procedura Increment i una funció Value. L'objecte protegit assegura que la variable Count es modifiqui i accedeixi de manera segura.

Rellotge en Temps Real i Retards

Ada proporciona instal·lacions per gestionar rellotges en temps real i retards, que són essencials per als sistemes en temps real.

Exemple: Ús de Retards

with Ada.Text_IO; use Ada.Text_IO;
with Ada.Real_Time; use Ada.Real_Time;

procedure Main is
   Start_Time : Time := Clock;
   Delay_Duration : constant Duration := 0.5; -- 500 mil·lisegons
begin
   Put_Line("Iniciant retard...");
   delay Delay_Duration;
   Put_Line("Retard completat.");
   Put_Line("Temps transcorregut: " & Duration'Image(To_Duration(Clock - Start_Time)));
end Main;
  • Explicació: Aquest exemple demostra com utilitzar el rellotge en temps real i els retards en Ada. El programa es retarda durant 500 mil·lisegons i després imprimeix el temps transcorregut.

Planificació de Prioritats

Ada suporta la planificació de prioritats, permetent assignar diferents prioritats a les tasques.

Exemple: Assignació de Prioritats a les Tasques

with Ada.Text_IO; use Ada.Text_IO;
with Ada.Real_Time; use Ada.Real_Time;
with Ada.Dispatching; use Ada.Dispatching;

procedure Main is
   task type Worker(Priority : Integer) is
      pragma Priority(Priority);
      entry Start;
   end Worker;

   task body Worker is
   begin
      accept Start;
      Put_Line("La tasca Worker amb prioritat " & Integer'Image(Priority) & " s'està executant.");
   end Worker;

   High_Priority_Task : Worker(10);
   Low_Priority_Task : Worker(5);
begin
   High_Priority_Task.Start;
   Low_Priority_Task.Start;
end Main;
  • Explicació: Aquest exemple mostra com crear tasques amb diferents prioritats utilitzant la directiva pragma Priority. Les tasques High_Priority_Task i Low_Priority_Task tenen assignades les prioritats 10 i 5, respectivament.

Conclusió

Els sistemes en temps real requereixen un control precís sobre el temps i l'execució de tasques. Ada proporciona característiques robustes com les tasques, els objectes protegits, la gestió del rellotge en temps real i la planificació de prioritats per donar suport al desenvolupament d'aplicacions en temps real fiables. Aprofitant aquestes característiques, els desenvolupadors poden crear sistemes que compleixin els estrictes requisits de rendiment i fiabilitat en temps real.
