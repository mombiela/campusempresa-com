La gestió d'excepcions és un aspecte crucial de la programació que permet als desenvolupadors gestionar i respondre als errors en temps d'execució de manera elegant. En Ada, la gestió d'excepcions és robusta i proporciona diversos mecanismes per assegurar que els programes puguin gestionar situacions inesperades de manera efectiva.

Introducció a la Gestió d'Excepcions

La gestió d'excepcions en Ada et permet:

  • Detectar errors en temps d'execució.
  • Proporcionar camins d'execució alternatius.
  • Assegurar l'estabilitat i la fiabilitat del programa.

Conceptes Bàsics

Què és una Excepció?

Una excepció és un esdeveniment que ocorre durant l'execució d'un programa que interromp el flux normal d'instruccions. Exemples inclouen divisió per zero, accés a un array fora dels límits, o errors d'E/S de fitxers.

Generar i Gestionar Excepcions

En Ada, les excepcions poden ser generades explícitament pel programador o implícitament pel sistema d'execució. Gestionar aquestes excepcions assegura que el programa pugui continuar operant o finalitzar de manera elegant.

Sintaxi i Estructura

Declarar Excepcions

En Ada, les excepcions es declaren utilitzant la paraula clau exception.

My_Exception : exception;

Generar Excepcions

Pots generar una excepció utilitzant la instrucció raise.

raise My_Exception;

Gestionar Excepcions

Les excepcions es gestionen utilitzant els blocs begin, exception, i end.

begin
   -- Codi que podria generar una excepció
exception
   when My_Exception =>
      -- Codi per gestionar My_Exception
   when others =>
      -- Codi per gestionar totes les altres excepcions
end;

Exemple: Gestió Bàsica d'Excepcions

Vegem un exemple senzill on gestionem un error de divisió per zero.

with Ada.Text_IO; use Ada.Text_IO;

procedure Division_Example is
   X, Y, Result : Integer;
begin
   X := 10;
   Y := 0;
   begin
      Result := X / Y;
      Put_Line("Result: " & Integer'Image(Result));
   exception
      when Constraint_Error =>
         Put_Line("Error: Division by zero is not allowed.");
   end;
end Division_Example;

Explicació

  • El bloc begin conté el codi que podria generar una excepció.
  • El bloc exception gestiona el Constraint_Error que es genera quan es produeix una divisió per zero.
  • La clàusula when others es pot utilitzar per gestionar qualsevol altra excepció no explícitament anomenada.

Gestió Avançada d'Excepcions

Propagar Excepcions

Les excepcions es poden propagar a procediments que les criden. Si una excepció no es gestiona en el bloc actual, es passa a la pila de crides.

procedure Inner_Procedure is
begin
   raise My_Exception;
end Inner_Procedure;

procedure Outer_Procedure is
begin
   Inner_Procedure;
exception
   when My_Exception =>
      Put_Line("My_Exception was handled in Outer_Procedure.");
end Outer_Procedure;

Excepcions Definides per l'Usuari

Pots definir les teves pròpies excepcions per gestionar condicions d'error específiques en la teva aplicació.

File_Not_Found : exception;

procedure Open_File(FileName : String) is
begin
   -- Codi per obrir el fitxer
   if FileName = "" then
      raise File_Not_Found;
   end if;
exception
   when File_Not_Found =>
      Put_Line("Error: The specified file was not found.");
end Open_File;

Informació sobre Excepcions

Pots obtenir informació sobre una excepció utilitzant el paquet Ada.Exceptions.

with Ada.Exceptions; use Ada.Exceptions;
with Ada.Text_IO; use Ada.Text_IO;

procedure Exception_Info is
begin
   raise My_Exception;
exception
   when E : others =>
      Put_Line("Exception occurred: " & Exception_Information(E));
end Exception_Info;

Conclusió

La gestió d'excepcions en Ada és una característica poderosa que ajuda a escriure programes robustos i resistents als errors. Entenent com declarar, generar i gestionar excepcions, pots assegurar que els teus programes en Ada puguin gestionar situacions inesperades de manera elegant. Recorda utilitzar gestors d'excepcions específics sempre que sigui possible i la clàusula when others com a recurs per a errors imprevistos.
