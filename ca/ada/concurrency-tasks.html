La concurrència és un concepte fonamental en Ada, que permet als programes realitzar múltiples operacions simultàniament. Aquest tema et guiarà a través dels conceptes bàsics de les tasques en Ada, des de la creació de tasques simples fins a mecanismes de control de concurrència més avançats.

Introducció a les Tasques

En Ada, les tasques són el mitjà principal per aconseguir la concurrència. Una tasca és un fil de control separat que pot executar-se concurrentment amb altres tasques.

Conceptes Clau

  • Declaració de Tasca: Com declarar una tasca en Ada.
  • Cos de la Tasca: La implementació del comportament de la tasca.
  • Activació de la Tasca: Quan i com les tasques comencen a executar-se.
  • Terminació de la Tasca: Com les tasques completen la seva execució.

Declaració i Cos de la Tasca

Per declarar una tasca en Ada, utilitzes la paraula clau task. El cos de la tasca defineix què farà la tasca quan s'executi.

Exemple

task My_Task is
    -- Declaració de la tasca
end My_Task;

task body My_Task is
begin
    -- Cos de la tasca (implementació)
    for I in 1 .. 10 loop
        Ada.Text_IO.Put_Line("La tasca s'està executant: " & Integer'Image(I));
    end loop;
end My_Task;

En aquest exemple:

  • My_Task es declara com una tasca.
  • El cos de la tasca conté un bucle que imprimeix un missatge deu vegades.

Activació de la Tasca

Les tasques s'activen automàticament quan el bloc o subprograma que les conté s'elabora. Això significa que les tasques comencen a executar-se tan aviat com el programa arriba al punt on es declara la tasca.

Exemple

procedure Main is
    task My_Task is
    end My_Task;

    task body My_Task is
    begin
        Ada.Text_IO.Put_Line("La tasca està activada!");
    end My_Task;
begin
    Ada.Text_IO.Put_Line("Procediment principal iniciat.");
end Main;

Sortida:

Procediment principal iniciat.
La tasca està activada!

Comunicació entre Tasques

Les tasques sovint necessiten comunicar-se entre elles. Ada proporciona diversos mecanismes per a la comunicació entre tasques, incloent objectes protegits i rendezvous.

Objectes Protegits

Els objectes protegits proporcionen una manera segura perquè les tasques comparteixin dades. Asseguren que només una tasca pugui accedir a les dades protegides alhora.

Exemple

protected Shared_Data is
    procedure Write(Value : Integer);
    function Read return Integer;
private
    Data : Integer := 0;
end Shared_Data;

protected body Shared_Data is
    procedure Write(Value : Integer) is
    begin
        Data := Value;
    end Write;

    function Read return Integer is
    begin
        return Data;
    end Read;
end Shared_Data;

task Writer is
end Writer;

task body Writer is
begin
    Shared_Data.Write(42);
end Writer;

task Reader is
end Reader;

task body Reader is
begin
    Ada.Text_IO.Put_Line("Valor llegit: " & Integer'Image(Shared_Data.Read));
end Reader;

En aquest exemple:

  • Shared_Data és un objecte protegit amb un procediment per escriure dades i una funció per llegir dades.
  • La tasca Writer escriu un valor a Shared_Data.
  • La tasca Reader llegeix el valor de Shared_Data.

Rendezvous

Rendezvous és un mecanisme on una tasca pot cridar una entrada d'una altra tasca, permetent-los sincronitzar-se i intercanviar dades.

Exemple

task Server is
    entry Request(Value : Integer);
end Server;

task body Server is
begin
    accept Request(Value : Integer) do
        Ada.Text_IO.Put_Line("El servidor ha rebut: " & Integer'Image(Value));
    end Request;
end Server;

task Client is
end Client;

task body Client is
begin
    Server.Request(99);
end Client;

En aquest exemple:

  • La tasca Server té una entrada Request que pot ser cridada per altres tasques.
  • La tasca Client crida l'entrada Request del Server, passant el valor 99.

Conclusió

La concurrència en Ada és una característica poderosa que et permet escriure programes que poden realitzar múltiples tasques simultàniament. Entenent la declaració de tasques, activació, comunicació i mecanismes de sincronització com els objectes protegits i rendezvous, pots gestionar efectivament operacions concurrents en els teus programes Ada.
