Introducció a l'Optimització de Codi

L'optimització de codi és el procés de modificar el codi per fer-lo més eficient, tant en termes de velocitat d'execució com d'ús de memòria. Això és crucial en la programació en C, on el rendiment sovint és una preocupació clau.

Conceptes Clau

  • Eficiència: Millorar la velocitat i reduir la petjada de memòria del teu codi.
  • Llegibilitat: Assegurar que el codi optimitzat segueixi sent comprensible.
  • Compromisos: Equilibrar entre l'optimització i la mantenibilitat del codi.

Tècniques Bàsiques d'Optimització

Optimització de Bucles

Els bucles són una font comuna d'ineficiència. Optimitzar els bucles pot millorar significativament el rendiment.

Exemple: Desenrotllament de Bucles

El desenrotllament de bucles redueix la sobrecàrrega del control del bucle executant múltiples iteracions en un sol cicle de bucle.

// Bucle original
for (int i = 0; i < 100; i++) {
    array[i] = 0;
}

// Bucle desenrotllat
for (int i = 0; i < 100; i += 4) {
    array[i] = 0;
    array[i+1] = 0;
    array[i+2] = 0;
    array[i+3] = 0;
}

Inlining de Funcions

Fer inlining de funcions petites pot reduir la sobrecàrrega de les crides a funcions.

// Funció original
int add(int a, int b) {
    return a + b;
}

// Funció inline
#define add(a, b) ((a) + (b))

int result = add(3, 4);

Evitar Càlculs Innecessaris

Reutilitzar valors calculats prèviament en lloc de recalcular-los pot estalviar temps.

// Codi ineficient
for (int i = 0; i < 100; i++) {
    int square = i * i;
    printf("%d\n", square);
}

// Codi optimitzat
int square;
for (int i = 0; i < 100; i++) {
    square = i * i;
    printf("%d\n", square);
}

Tècniques Intermèdies d'Optimització

Gestió de Memòria

L'ús eficient de la memòria pot impactar molt el rendiment dels programes en C.

Exemple: Ús d'Assignació Estàtica

L'assignació estàtica pot ser més ràpida que l'assignació dinàmica perquè evita la sobrecàrrega de funcions de gestió de memòria com malloc i free.

// Assignació dinàmica
int *array = (int *)malloc(100 * sizeof(int));

// Assignació estàtica
int array[100];

Localitat de Dades

Millorar la localitat de dades pot augmentar el rendiment de la memòria cau.

// Mala localitat de dades
struct Point {
    int x;
    int y;
};
struct Point points[100];

// Millor localitat de dades
int x[100];
int y[100];

Tècniques Avançades d'Optimització

Optimització d'Algorismes

Escollir l'algorisme adequat pot tenir un impacte profund en el rendiment.

Exemple: Algorismes de Classificació

  • Bubble Sort: Simple però ineficient per a conjunts de dades grans.
  • Quick Sort: Més complex però molt més ràpid per a conjunts de dades grans.
// Bubble Sort
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        for (int j = 0; j < n-i-1; j++) {
            if (arr[j] > arr[j+1]) {
                int temp = arr[j];
                arr[j] = arr[j+1];
                arr[j+1] = temp;
            }
        }
    }
}

// Quick Sort
void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}

Flags d'Optimització del Compilador

Utilitzar flags d'optimització del compilador pot optimitzar automàticament el teu codi.

Exemple: Flags d'Optimització de GCC

  • -O1: Optimització bàsica.
  • -O2: Més optimització.
  • -O3: Optimització màxima.
  • -Os: Optimitzar per mida.
gcc -O2 -o optimized_program program.c

Conclusió

L'optimització de codi en C és un procés multifacètic que implica millorar l'eficiència del teu codi mentre es manté la llegibilitat i la manejabilitat. Aplicant tècniques bàsiques, intermèdies i avançades d'optimització, pots millorar significativament el rendiment dels teus programes en C. Recorda sempre fer un perfil del teu codi abans i després de l'optimització per assegurar-te que els teus canvis tenen l'efecte desitjat.
