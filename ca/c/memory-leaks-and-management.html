Introducció a la Gestió de Memòria

La gestió de memòria és un aspecte crucial de la programació en C. Involucra l'assignació, ús i alliberament de memòria de manera eficient i sense errors. Entendre la gestió de memòria ajuda a prevenir fuites de memòria, que poden portar a un ús ineficient dels recursos i a fallades del programa.

Assignació Dinàmica de Memòria

L'assignació dinàmica de memòria permet als programes sol·licitar memòria en temps d'execució. Això es fa utilitzant funcions de la biblioteca estàndard de C.

Funcions Clau

  • malloc(): Assigna un nombre especificat de bytes i retorna un punter a la memòria assignada.
  • calloc(): Assigna memòria per a una matriu d'elements, els inicialitza a zero i retorna un punter a la memòria.
  • realloc(): Canvia la mida de la memòria prèviament assignada.
  • free(): Allibera la memòria prèviament assignada.

Exemple

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    int n, i;

    // Nombre d'elements
    n = 5;

    // Assignar memòria dinàmicament utilitzant malloc()
    ptr = (int*)malloc(n * sizeof(int));

    // Comprovar si la memòria ha estat assignada correctament per malloc o no
    if (ptr == NULL) {
        printf("Memòria no assignada.\n");
        exit(0);
    } else {
        // La memòria ha estat assignada correctament
        printf("Memòria assignada correctament utilitzant malloc.\n");

        // Obtenir els elements de la matriu
        for (i = 0; i < n; ++i) {
            ptr[i] = i + 1;
        }

        // Imprimir els elements de la matriu
        printf("Els elements de la matriu són: ");
        for (i = 0; i < n; ++i) {
            printf("%d ", ptr[i]);
        }

        // Alliberar la memòria
        free(ptr);
        printf("\nMemòria alliberada correctament.\n");
    }

    return 0;
}

Fuites de Memòria

Una fuita de memòria ocorre quan un programa assigna memòria però no l'allibera després d'usar-la. Això pot portar a un augment gradual en l'ús de memòria, eventualment causant que el programa es quedi sense memòria i falli.

Causes Comunes

  • Oblidar de cridar free() després de malloc(), calloc(), o realloc().
  • Perdre el punter a la memòria assignada abans d'alliberar-la.

Exemple d'una Fuita de Memòria

#include <stdio.h>
#include <stdlib.h>

void createMemoryLeak() {
    int *ptr = (int*)malloc(10 * sizeof(int));
    // La memòria és assignada però no alliberada
}

int main() {
    createMemoryLeak();
    // Ocorre una fuita de memòria aquí
    return 0;
}

Millors Pràctiques per a la Gestió de Memòria

  • Sempre emparellar malloc(), calloc(), i realloc() amb free().
  • Utilitzar eines com Valgrind per detectar fuites de memòria.
  • Inicialitzar els punters a NULL i comprovar abans d'alliberar.
  • Evitar perdre els punters a la memòria assignada dinàmicament.

Exemple amb Gestió de Memòria Adequada

#include <stdio.h>
#include <stdlib.h>

void createMemory() {
    int *ptr = (int*)malloc(10 * sizeof(int));
    if (ptr == NULL) {
        printf("Memòria no assignada.\n");
        exit(0);
    }
    // Utilitzar la memòria assignada
    for (int i = 0; i < 10; i++) {
        ptr[i] = i;
    }
    // Alliberar la memòria assignada
    free(ptr);
}

int main() {
    createMemory();
    return 0;
}

Eines per Detectar Fuites de Memòria

  • Valgrind: Una eina de programació per a depuració de memòria, detecció de fuites de memòria i perfilatge.
  • AddressSanitizer: Un detector d'errors de memòria ràpid.

Utilitzant Valgrind

Per utilitzar Valgrind per comprovar fuites de memòria:

  1. Compila el teu programa amb informació de depuració: gcc -g -o myprogram myprogram.c
  2. Executa Valgrind: valgrind --leak-check=full ./myprogram

Conclusió

La gestió de memòria és una habilitat fonamental per als programadors de C. L'ús adequat de les funcions d'assignació i alliberament de memòria dinàmica és essencial per evitar fuites de memòria. Seguint les millors pràctiques i utilitzant eines com Valgrind, pots assegurar un ús eficient i sense errors de la memòria en els teus programes.
