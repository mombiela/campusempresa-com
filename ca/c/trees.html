Introducció als Arbres

Els arbres són una estructura de dades fonamental en informàtica, utilitzada per representar relacions jeràrquiques. Són particularment útils per organitzar dades que tenen una jerarquia natural, com ara sistemes de fitxers, estructures organitzatives i més.

Conceptes Clau

  • Node: La unitat bàsica d'un arbre, que conté dades i referències a altres nodes.
  • Arrel: El node més alt d'un arbre.
  • Fill: Un node directament connectat a un altre node quan es mou lluny de l'arrel.
  • Pare: La noció inversa d'un fill.
  • Fulla: Un node sense fills.
  • Subarbre: Un arbre format per un node i els seus descendents.
  • Arbre Binari: Un arbre on cada node té com a màxim dos fills.
  • Arbre Binari de Cerca (BST): Un arbre binari on el fill esquerre conté només nodes amb valors menors que el node pare, i el fill dret només nodes amb valors més grans que el node pare.

Estructura Bàsica d'un Arbre en C

Definició d'un Node

En C, un node d'arbre es pot definir utilitzant una struct. Aquí hi ha un exemple d'una estructura bàsica de node per a un arbre binari:

#include <stdio.h>
#include <stdlib.h>

// Definir l'estructura per a un node d'arbre
struct Node {
    int data;
    struct Node* left;
    struct Node* right;
};

// Funció per crear un nou node
struct Node* createNode(int data) {
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = data;
    newNode->left = NULL;
    newNode->right = NULL;
    return newNode;
}

Creació d'un Arbre Simple

Creem un arbre binari simple amb tres nodes:

int main() {
    // Crear el node arrel
    struct Node* root = createNode(1);
    
    // Crear fills esquerre i dret
    root->left = createNode(2);
    root->right = createNode(3);
    
    // Imprimir l'arrel i els seus fills
    printf("Arrel: %d\n", root->data);
    printf("Fill Esquerre: %d\n", root->left->data);
    printf("Fill Dret: %d\n", root->right->data);
    
    return 0;
}

Arbre Binari de Cerca (BST)

Inserció en un BST

Inserir un node en un BST implica col·locar el nou node en la posició correcta per mantenir les propietats del BST.

struct Node* insert(struct Node* node, int data) {
    // Si l'arbre està buit, retorna un nou node
    si (node == NULL) return createNode(data);

    // Altrament, recorre l'arbre
    si (data < node->data)
        node->left = insert(node->left, data);
    altrament si (data > node->data)
        node->right = insert(node->right, data);

    // Retorna el punter del node (sense canvis)
    return node;
}

Cerca en un BST

Cercar un valor en un BST implica comparar el valor objectiu amb el valor del node actual i decidir si anar a l'esquerra o a la dreta.

struct Node* search(struct Node* root, int data) {
    // Casos Base: l'arrel és nul·la o les dades són presents a l'arrel
    si (root == NULL || root->data == data)
       return root;
    
    // Les dades són més grans que les de l'arrel
    si (root->data < data)
       return search(root->right, data);

    // Les dades són més petites que les de l'arrel
    return search(root->left, data);
}

Recorruts d'Arbre

El recorregut d'un arbre es refereix al procés de visitar tots els nodes d'un arbre en un ordre específic.

Recorregut Inordre

El recorregut inordre visita els nodes en el següent ordre: subarbre esquerre, arrel, subarbre dret.

void inorderTraversal(struct Node* root) {
    si (root != NULL) {
        inorderTraversal(root->left);
        printf("%d ", root->data);
        inorderTraversal(root->right);
    }
}

Recorregut Preordre

El recorregut preordre visita els nodes en el següent ordre: arrel, subarbre esquerre, subarbre dret.

void preorderTraversal(struct Node* root) {
    si (root != NULL) {
        printf("%d ", root->data);
        preorderTraversal(root->left);
        preorderTraversal(root->right);
    }
}

Recorregut Postordre

El recorregut postordre visita els nodes en el següent ordre: subarbre esquerre, subarbre dret, arrel.

void postorderTraversal(struct Node* root) {
    si (root != NULL) {
        postorderTraversal(root->left);
        postorderTraversal(root->right);
        printf("%d ", root->data);
    }
}

Temes Avançats

Equilibrat d'Arbres

Equilibrar un arbre assegura que l'arbre es mantingui eficient per a operacions com la inserció, eliminació i cerca. Els arbres equilibrats comuns inclouen els arbres AVL i els arbres Red-Black.

Arbres AVL

Un arbre AVL és un arbre binari de cerca autoequilibrat on la diferència entre les altures dels subarbres esquerre i dret no pot ser més gran que un per a tots els nodes.

Arbres Red-Black

Un arbre Red-Black és un tipus d'arbre binari de cerca autoequilibrat on cada node conté un bit extra per denotar el color del node, ja sigui vermell o negre.

Conclusió

Els arbres són una estructura de dades versàtil i essencial en informàtica. Entendre els conceptes bàsics de les estructures d'arbres, els arbres binaris de cerca i els recorreguts d'arbres és crucial per resoldre molts problemes complexos de manera eficient. A mesura que avancis, explorar arbres equilibrats com els AVL i els Red-Black millorarà encara més la teva capacitat per gestionar dades jeràrquiques de manera efectiva.
