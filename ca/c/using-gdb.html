El GNU Debugger (GDB) és una eina potent per depurar programes en C. Aquesta secció et guiarà a través dels conceptes bàsics d'ús de GDB, des de la seva configuració fins a l'ús de les seves característiques avançades.

Introducció a GDB

GDB et permet:

  • Inspeccionar què està fent el programa en un moment específic.
  • Canviar variables i executar codi línia per línia.
  • Establir punts de ruptura per pausar l'execució en línies específiques.
  • Traçar el flux d'execució per entendre com el programa arriba a un cert estat.

Configuració de GDB

Per utilitzar GDB, has de compilar el teu programa en C amb informació de depuració. Això es fa utilitzant el flag -g amb gcc.

gcc -g -o myprogram myprogram.c

Aquesta comanda compila myprogram.c en un executable anomenat myprogram amb informació de depuració inclosa.

Comandes bàsiques de GDB

Iniciar GDB

Per iniciar GDB amb el teu programa compilat:

gdb myprogram

Executar el programa

Per executar el programa dins de GDB:

(gdb) run

Establir punts de ruptura

Els punts de ruptura et permeten pausar l'execució del teu programa en línies o funcions específiques.

  • Per establir un punt de ruptura en una línia específica:
(gdb) break 10
  • Per establir un punt de ruptura a l'inici d'una funció:
(gdb) break main

Inspeccionar variables

Per inspeccionar el valor d'una variable:

(gdb) print variable_name

Passar línia a línia pel codi

  • Per executar la següent línia de codi:
(gdb) next
  • Per entrar en una crida de funció:
(gdb) step

Continuar l'execució

Per continuar executant el programa després d'arribar a un punt de ruptura:

(gdb) continue

Sortir de GDB

Per sortir de GDB:

(gdb) quit

Característiques avançades de GDB

Punts de ruptura condicionals

Pots establir punts de ruptura que només es disparen quan una certa condició és certa.

(gdb) break 10 if x == 5

Watchpoints

Els watchpoints pausen l'execució quan el valor d'una variable canvia.

(gdb) watch variable_name

Backtrace

Per veure la pila de crides i entendre com el programa ha arribat al punt actual:

(gdb) backtrace

Examinar la memòria

Per examinar la memòria en una adreça específica:

(gdb) x /nfu address
  • n és el nombre d'unitats a mostrar.
  • f és el format (per exemple, x per hexadecimal, d per decimal).
  • u és la mida de la unitat (per exemple, b per bytes, h per mitges paraules).

Sessió d'exemple

Aquí tens una sessió d'exemple de GDB per il·lustrar les comandes:

$ gcc -g -o example example.c
$ gdb example
(gdb) break main
Breakpoint 1 at 0x400526: file example.c, line 5.
(gdb) run
Starting program: /path/to/example 

Breakpoint 1, main () at example.c:5
5       int x = 0;
(gdb) print x
$1 = 0
(gdb) next
6       x = 5;
(gdb) print x
$2 = 0
(gdb) next
7       printf("%d\n", x);
(gdb) print x
$3 = 5
(gdb) continue
5
[Inferior 1 (process 1234) exited normally]
(gdb) quit

Conclusió

GDB és una eina essencial per depurar programes en C. En dominar les seves característiques bàsiques i avançades, pots millorar significativament la teva capacitat per diagnosticar i corregir errors. Practica utilitzant GDB amb diferents programes per sentir-te més còmode amb les seves comandes i capacitats.
