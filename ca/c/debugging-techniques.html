La depuració és una habilitat essencial per a qualsevol programador. Aquesta secció cobrirà diverses tècniques de depuració en C, des de nivells per a principiants fins a nivells avançats. Al final d'aquest tema, estaràs equipat amb les eines i els coneixements per identificar i corregir errors de manera efectiva en els teus programes en C.

Introducció a la Depuració

  • Definició: La depuració és el procés d'identificar, analitzar i eliminar errors o bugs d'un programa.
  • Importància: La depuració assegura que el teu programa funcioni correctament i eficientment.

Tècniques Bàsiques de Depuració

Declaracions de Print

Una de les tècniques de depuració més simples i comunes és utilitzar declaracions de print per seguir el flux d'execució i els valors de les variables.

#include <stdio.h>

int main() {
    int a = 5;
    int b = 10;
    int sum = a + b;
    
    printf("a: %d\n", a);
    printf("b: %d\n", b);
    printf("sum: %d\n", sum);
    
    return 0;
}
  • Explicació: Imprimint els valors de a, b i sum, pots verificar que les variables continguin els valors esperats.

Utilitzant Assertions

Les assertions s'utilitzen per verificar les suposicions fetes pel programa i poden ajudar a detectar errors aviat.

#include <assert.h>

int main() {
    int x = 10;
    assert(x == 10); // Això passarà
    assert(x == 5);  // Això fallarà i el programa es tancarà
    
    return 0;
}
  • Explicació: Si la condició en la declaració assert és falsa, el programa es tancarà i imprimirà un missatge d'error.

Tècniques Intermèdies de Depuració

Utilitzant un Depurador (GDB)

El GNU Debugger (GDB) és una eina poderosa per depurar programes en C.

  • Iniciant GDB: Compila el teu programa amb el flag -g per incloure informació de depuració. 
    gcc -g -o myprogram myprogram.c
gdb myprogram
  • Comandes Bàsiques:
    • run: Inicia el programa.
    • break: Estableix un punt de ruptura.
    • next: Executa la següent línia de codi.
    • print: Imprimeix el valor d'una variable.
    • continue: Continua l'execució fins al següent punt de ruptura.

Sessió d'Exemple amb GDB

#include <stdio.h>

int main() {
    int a = 5;
    int b = 10;
    int sum = a + b;
    
    printf("Sum: %d\n", sum);
    
    return 0;
}
  • Comandes GDB: 
    (gdb) break main
Breakpoint 1 at 0x400526: file myprogram.c, line 5.
(gdb) run
Starting program: /path/to/myprogram

Breakpoint 1, main () at myprogram.c:5
5       int a = 5;
(gdb) next
6       int b = 10;
(gdb) print a
$1 = 5
(gdb) continue
Continuing.
Sum: 15

Program exited normally.

Tècniques Avançades de Depuració

Depuració de Memòria amb Valgrind

Valgrind és una eina per detectar fuites de memòria i problemes de gestió de memòria.

  • Executant Valgrind: 
    valgrind --leak-check=full ./myprogram

Exemple de Sortida de Valgrind

==12345== HEAP SUMMARY:
==12345==     in use at exit: 0 bytes in 0 blocks
==12345==   total heap usage: 1 allocs, 1 frees, 1,024 bytes allocated
==12345== 
==12345== All heap blocks were freed -- no leaks are possible
==12345== 
==12345== For counts of detected and suppressed errors, rerun with: -v
==12345== ERROR SUMMARY: 0 errors from 0 contexts (suppressed: 0 from 0)
  • Explicació: Valgrind informa de qualsevol fuita de memòria o error, ajudant-te a assegurar que el teu programa gestiona la memòria correctament.

Utilitzant Eines d'Anàlisi Estàtica

Les eines d'anàlisi estàtica analitzen el teu codi sense executar-lo, identificant problemes potencials.

  • Eines Comunes:
    • cppcheck
    • clang-tidy

Exemple d'Anàlisi Estàtica amb `cppcheck`

cppcheck myprogram.c

  • Sortida:

    [myprogram.c:10]: (style) Variable 'sum' is assigned a value that is never used.

  • Explicació: cppcheck identifica que la variable sum està assignada però mai utilitzada, la qual cosa podria indicar un possible error o ineficiència.

Conclusió

La depuració és una habilitat crítica per a qualsevol programador en C. Dominant aquestes tècniques, des de declaracions de print bàsiques fins a eines avançades com GDB i Valgrind, pots identificar i resoldre problemes en el teu codi de manera eficient. Recorda, la clau per a una depuració efectiva és un enfocament sistemàtic: entendre el problema, aïllar la causa i aplicar la tècnica adequada per solucionar-lo.
