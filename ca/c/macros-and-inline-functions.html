Introducció

En la programació en C, les macros i les funcions inline són dues tècniques utilitzades per optimitzar el rendiment del codi. Ambdues s'utilitzen per reduir la sobrecàrrega de les crides a funcions, però tenen mecanismes i casos d'ús diferents. Aquesta secció cobrirà els conceptes bàsics de les macros i les funcions inline, les seves diferències i com utilitzar-les de manera efectiva.

Macros

Les macros són una característica del preprocesador de C que permet definir codi que es pot reutilitzar al llarg del programa. Les macros es defineixen utilitzant la directiva #define.

Definició de Macros

  • Sintaxi: 
    #define NOM_MACRO text_de_reemplaçament
  • Exemple: 
    #define PI 3.14159
#define QUADRAT(x) ((x) * (x))

Ús de Macros

  • Exemple: 
    #include <stdio.h>

#define PI 3.14159
#define QUADRAT(x) ((x) * (x))

int main() {
    double radi = 5.0;
    double area = PI * QUADRAT(radi);
    printf("Àrea del cercle: %f\n", area);
    return 0;
}

Avantatges de les Macros

  • Rendiment: No hi ha sobrecàrrega de crida a funció.
  • Flexibilitat: Es poden utilitzar per a constants, expressions i fins i tot blocs de codi.

Desavantatges de les Macros

  • Depuració: Més difícil de depurar ja que són expandides pel preprocesador.
  • Seguretat de Tipus: No hi ha verificació de tipus, cosa que pot portar a comportaments inesperats.

Funcions Inline

Les funcions inline són una característica que permet al compilador inserir el codi de la funció directament en el codi que la crida, reduint la sobrecàrrega de la crida a funció.

Definició de Funcions Inline

  • Sintaxi: 
    inline tipus_de_retorn nom_funció(paràmetres) {
    // cos de la funció
}
  • Exemple: 
    inline int quadrat(int x) {
    return x * x;
}

Ús de Funcions Inline

  • Exemple: 
    #include <stdio.h>

inline int quadrat(int x) {
    return x * x;
}

int main() {
    int num = 5;
    printf("El quadrat de %d és %d\n", num, quadrat(num));
    return 0;
}

Avantatges de les Funcions Inline

  • Seguretat de Tipus: La verificació de tipus és realitzada pel compilador.
  • Depuració: Més fàcil de depurar en comparació amb les macros.
  • Mantenibilitat: Més fàcil de llegir i mantenir.

Desavantatges de les Funcions Inline

  • Suport del Compilador: No tots els compiladors poden inlinar la funció com s'espera.
  • Mida del Codi: Pot augmentar la mida del binari si s'abusa.

Macros vs Funcions Inline

| Característica | Macros | Funcions Inline | |------------------|----------------------------------|------------------------------------| | Definició | Directiva del preprocesador | Directiva del compilador | | Seguretat de Tipus | No | Sí | | Depuració | Difícil | Més fàcil | | Rendiment | No hi ha sobrecàrrega de crida a funció | Reducció de la sobrecàrrega de crida a funció | | Flexibilitat | Alta (pot incloure blocs de codi) | Limitada a comportament de funció | | Mida del Codi| Sense impacte | Pot augmentar si s'abusa |

Conclusió

Les macros i les funcions inline són eines poderoses en la programació en C per optimitzar el rendiment. Les macros ofereixen flexibilitat però manquen de seguretat de tipus i poden ser més difícils de depurar. Les funcions inline proporcionen seguretat de tipus i són més fàcils de depurar, però no sempre són inlinades pel compilador i poden augmentar la mida del codi si s'abusa. Entendre quan i com utilitzar cadascuna pot ajudar a escriure codi més eficient i mantenible.
