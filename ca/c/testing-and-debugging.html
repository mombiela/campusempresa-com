Introducció

Les proves i la depuració són habilitats crucials per a qualsevol programador. En aquesta secció, cobrirem els conceptes bàsics de les proves i la depuració en C, progressant des de tècniques per a principiants fins a tècniques avançades. Al final d'aquesta secció, seràs capaç d'identificar i corregir errors en el teu codi i escriure proves per assegurar-te que el teu codi funciona com s'espera.

Conceptes Bàsics de la Depuració

Què és la Depuració?

  • La depuració és el procés d'identificar i eliminar errors del programari.
  • Els errors en el programari es coneixen comunament com a "bugs".

Tipus Comuns de Bugs

  • Errors de Sintaxi: Errors en el codi que violen les regles del llenguatge C.
  • Errors d'Execució: Errors que es produeixen mentre el programa s'està executant, sovint causant que el programa es bloquegi.
  • Errors Lògics: Errors que fan que el programa funcioni incorrectament però no es bloquegi.

Eines per a la Depuració

  • Instruccions printf: Simples i efectives per a programes petits.
  • Eines de Depuració: Eines més avançades com GDB (GNU Debugger).

Ús de printf per a la Depuració

#include <stdio.h>

int main() {
    int a = 5;
    int b = 0;
    int result;

    printf("Abans de la divisió\n");
    result = a / b; // Això causarà un error d'execució (divisió per zero)
    printf("Després de la divisió\n");

    return 0;
}
  • En aquest exemple, el programa es bloquejarà abans d'imprimir "Després de la divisió" a causa d'un error de divisió per zero.

Ús de GDB per a la Depuració

  • Iniciar GDB: gdb ./your_program
  • Establir Punts de Trencament: break main
  • Executar el Programa: run
  • Inspeccionar Variables: print variable_name
  • Passar pel Codi: step o next

Conceptes Bàsics de les Proves

Què són les Proves?

  • Les proves són el procés d'executar un programa amb la intenció de trobar errors.
  • Asseguren que el programari es comporti com s'espera.

Tipus de Proves

  • Proves Unitàries: Proves d'unitats o components individuals d'un programa.
  • Proves d'Integració: Proves de parts combinades d'una aplicació per determinar si funcionen correctament juntes.
  • Proves de Sistema: Proves del sistema complet com un tot.

Escriure Proves Unitàries en C

  • Utilitza biblioteques com CUnit o Unity per escriure proves unitàries.

Exemple d'una Prova Unitària Simple

#include <assert.h>

int add(int a, int b) {
    return a + b;
}

void test_add() {
    assert(add(2, 3) == 5);
    assert(add(-1, 1) == 0);
    assert(add(-1, -1) == -2);
}

int main() {
    test_add();
    printf("Totes les proves han passat!\n");
    return 0;
}
  • En aquest exemple, la funció assert s'utilitza per comprovar si la funció add retorna els resultats esperats.

Tècniques Avançades de Depuració

Depuració de Memòria

  • Valgrind: Una eina per detectar fuites de memòria i errors de memòria.
  • Ús: valgrind ./your_program

Exemple de Detecció de Fuites de Memòria

#include <stdlib.h>

int main() {
    int *ptr = (int *)malloc(sizeof(int) * 10);
    // S'ha oblidat alliberar la memòria assignada
    return 0;
}
  • Executar aquest programa amb Valgrind detectarà la fuita de memòria.

Depuració de Fallades de Segmentació

  • Causes Comunes: Desreferenciar punters nuls o no inicialitzats, accedir a memòria fora dels límits.
  • Ús de GDB: Estableix punts de trencament i passa pel codi per trobar la línia exacta que causa la fallada de segmentació.

Tècniques Avançades de Proves

Desenvolupament Guiat per Proves (TDD)

  • Escriu proves abans d'escriure el codi real.
  • Assegura que el codi sempre es provi i funcioni com s'espera.

Mocking i Stubbing

  • Mocking: Crear una versió falsa d'una funció o mòdul per provar una altra part del codi.
  • Stubbing: Substituir una funció per una versió simplificada que retorna valors fixos.

Exemple de Mocking

#include <assert.h>

int get_data_from_db() {
    // Funció original que obté dades d'una base de dades
    return 100;
}

// Funció mock
int mock_get_data_from_db() {
    return 42; // Retorna un valor fix per a les proves
}

void test_function_using_db() {
    int (*original_function)() = get_data_from_db;
    get_data_from_db = mock_get_data_from_db; // Substituir per mock

    assert(get_data_from_db() == 42);

    get_data_from_db = original_function; // Restaurar funció original
}

int main() {
    test_function_using_db();
    printf("Totes les proves han passat!\n");
    return 0;
}

Conclusió

Les proves i la depuració són habilitats essencials per a qualsevol programador. En dominar aquestes tècniques, pots assegurar-te que els teus programes en C siguin fiables, eficients i lliures d'errors. Comença amb la depuració bàsica utilitzant printf i GDB, després passa a escriure proves unitàries i utilitzar eines avançades com Valgrind. Finalment, explora tècniques avançades com TDD i mocking per millorar encara més les teves capacitats de proves i depuració.
