Introducció

Les piles i les cues són estructures de dades fonamentals en la informàtica. S'utilitzen per emmagatzemar i gestionar dades en un ordre particular. Entendre aquestes estructures és crucial per resoldre diversos problemes computacionals de manera eficient.

Piles

Conceptes clau

  • LIFO (Last In, First Out): L'últim element afegit a la pila és el primer que es treu.
  • Push: Afegir un element a la part superior de la pila.
  • Pop: Treure l'element superior de la pila.
  • Peek/Top: Veure l'element superior sense treure'l.

Operacions de pila en C

Definició de l'estructura

#define MAX 100

typedef struct {
    int data[MAX];
    int top;
} Stack;

Inicialitzar la pila

void initialize(Stack *s) {
    s->top = -1;
}

Operació Push

void push(Stack *s, int value) {
    if (s->top == MAX - 1) {
        printf("Stack overflow\n");
        return;
    }
    s->data[++(s->top)] = value;
}

Operació Pop

int pop(Stack *s) {
    if (s->top == -1) {
        printf("Stack underflow\n");
        return -1; // Retorna un valor d'error
    }
    return s->data[(s->top)--];
}

Operació Peek

int peek(Stack *s) {
    if (s->top == -1) {
        printf("Stack is empty\n");
        return -1; // Retorna un valor d'error
    }
    return s->data[s->top];
}

Exemple d'ús

#include <stdio.h>

int main() {
    Stack s;
    initialize(&s);

    push(&s, 10);
    push(&s, 20);
    push(&s, 30);

    printf("L'element superior és %d\n", peek(&s));
    printf("L'element tret és %d\n", pop(&s));
    printf("L'element superior és %d\n", peek(&s));

    return 0;
}

Cues

Conceptes clau

  • FIFO (First In, First Out): El primer element afegit a la cua és el primer que es treu.
  • Enqueue: Afegir un element al final de la cua.
  • Dequeue: Treure l'element frontal de la cua.
  • Front: Veure l'element frontal sense treure'l.
  • Rear: Veure l'element del final sense treure'l.

Operacions de cua en C

Definició de l'estructura

#define MAX 100

typedef struct {
    int data[MAX];
    int front;
    int rear;
} Queue;

Inicialitzar la cua

void initialize(Queue *q) {
    q->front = -1;
    q->rear = -1;
}

Operació Enqueue

void enqueue(Queue *q, int value) {
    if (q->rear == MAX - 1) {
        printf("Queue overflow\n");
        return;
    }
    if (q->front == -1) q->front = 0;
    q->data[++(q->rear)] = value;
}

Operació Dequeue

int dequeue(Queue *q) {
    if (q->front == -1 || q->front > q->rear) {
        printf("Queue underflow\n");
        return -1; // Retorna un valor d'error
    }
    return q->data[(q->front)++];
}

Operació Front

int front(Queue *q) {
    if (q->front == -1 || q->front > q->rear) {
        printf("Queue is empty\n");
        return -1; // Retorna un valor d'error
    }
    return q->data[q->front];
}

Exemple d'ús

#include <stdio.h>

int main() {
    Queue q;
    initialize(&q);

    enqueue(&q, 10);
    enqueue(&q, 20);
    enqueue(&q, 30);

    printf("L'element frontal és %d\n", front(&q));
    printf("L'element tret és %d\n", dequeue(&q));
    printf("L'element frontal és %d\n", front(&q));

    return 0;
}

Comparació de piles i cues

| Característica | Pila (LIFO) | Cua (FIFO) | |----------------|---------------------|---------------------| | Inserció | Push (top) | Enqueue (rear) | | Eliminació | Pop (top) | Dequeue (front) | | Accés | Peek/Top (top) | Front (front) |

Conclusió

Les piles i les cues són estructures de dades essencials que ajuden a gestionar dades en ordres específics. Les piles segueixen el principi LIFO, cosa que les fa adequades per a tasques com invertir dades i algorismes de backtracking. Les cues segueixen el principi FIFO, cosa que les fa ideals per a la programació de tasques i la gestió de buffers. Entendre com implementar i utilitzar aquestes estructures en C és fonamental per a una programació i resolució de problemes eficients.
