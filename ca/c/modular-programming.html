La programació modular és una tècnica de disseny de programari que posa èmfasi en separar la funcionalitat d'un programa en mòduls independents i intercanviables. Cada mòdul conté tot el necessari per executar només un aspecte de la funcionalitat desitjada.

Introducció a la Programació Modular

  • Definició: La programació modular és un mètode d'organitzar programes grans en unitats més petites, manejables i reutilitzables anomenades mòduls.
  • Beneficis:
    • Mantenibilitat: Més fàcil de gestionar i actualitzar.
    • Reutilització: Els mòduls es poden reutilitzar en diferents programes.
    • Escalabilitat: Simplifica l'afegit de noves funcionalitats.
    • Depuració: Més fàcil d'aïllar i corregir errors.

Creació de Mòduls en C

En C, la programació modular s'aconsegueix utilitzant funcions i fitxers separats. Cada mòdul normalment consisteix en un fitxer de capçalera (.h) i un fitxer de codi font (.c).

Fitxers de Capçalera

  • Propòsit: Declarar les funcions i variables que estan disponibles per a altres mòduls.
  • Sintaxi: Els fitxers de capçalera utilitzen l'extensió .h i contenen prototips de funcions, definicions de macros i definicions de tipus.

Exemple:

// math_operations.h
#ifndef MATH_OPERATIONS_H
#define MATH_OPERATIONS_H

int add(int a, int b);
int subtract(int a, int b);

#endif // MATH_OPERATIONS_H

Fitxers de Codi Font

  • Propòsit: Definir les funcions declarades en els fitxers de capçalera.
  • Sintaxi: Els fitxers de codi font utilitzen l'extensió .c i contenen la implementació real de les funcions.

Exemple:

// math_operations.c
#include "math_operations.h"

int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

Ús de Mòduls

Per utilitzar les funcions d'un mòdul en una altra part del teu programa, inclou el fitxer de capçalera del mòdul i enllaça el fitxer de codi font corresponent durant la compilació.

Exemple:

// main.c
#include <stdio.h>
#include "math_operations.h"

int main() {
    int x = 10, y = 5;
    printf("Add: %d\n", add(x, y));
    printf("Subtract: %d\n", subtract(x, y));
    return 0;
}

Compilació de Programes Multi-Fitxer

Per compilar un programa amb múltiples fitxers de codi font, utilitza la següent comanda:

gcc main.c math_operations.c -o program

Conceptes Avançats de Programació Modular

Enllaç Estàtic i Dinàmic

  • Enllaç Estàtic: Combina tots els mòduls en un únic executable en temps de compilació.
  • Enllaç Dinàmic: Els mòduls s'enllacen en temps d'execució, permetent executables més petits i actualitzacions més fàcils.

Encapsulació

  • Definició: Amagar els detalls d'implementació interna d'un mòdul i exposar només les interfícies necessàries.
  • Propòsit: Millora la modularitat i redueix les dependències entre mòduls.

Exemple:

// math_operations.h
#ifndef MATH_OPERATIONS_H
#define MATH_OPERATIONS_H

int add(int a, int b);
int subtract(int a, int b);

#endif // MATH_OPERATIONS_H

// math_operations.c
#include "math_operations.h"
#include <stdio.h>

static void log_operation(const char* operation) {
    printf("Performing: %s\n", operation);
}

int add(int a, int b) {
    log_operation("addition");
    return a + b;
}

int subtract(int a, int b) {
    log_operation("subtraction");
    return a - b;
}

En aquest exemple, log_operation és una funció estàtica, el que significa que només és accessible dins de math_operations.c.

Conclusió

La programació modular en C és una tècnica poderosa que ajuda a organitzar el codi en unitats manejables, reutilitzables i mantenibles. Utilitzant fitxers de capçalera i fitxers de codi font, pots crear interfícies ben definides i encapsular els detalls d'implementació. Aquest enfocament no només fa que el teu codi sigui més fàcil d'entendre i mantenir, sinó que també facilita la col·laboració i la reutilització del codi.

Dominant la programació modular, estaràs ben preparat per gestionar projectes més grans i complexos de manera eficient.
