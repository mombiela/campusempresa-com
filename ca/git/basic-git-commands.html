Introducció

Git és un potent sistema de control de versions que ajuda els desenvolupadors a gestionar els canvis en el seu codi. En aquesta secció, cobrirem tres comandes fonamentals de Git: add, commit i status. Entendre aquestes comandes és essencial per a qualsevol persona que comenci amb Git.

Comanda `add` de Git

La comanda add s'utilitza per afegir canvis en el directori de treball a l'àrea de preparació. Aquest és el primer pas en el procés de comprometre canvis al repositori.

Conceptes clau

  • Directori de treball: El directori on es troben els fitxers del teu projecte.
  • Àrea de preparació: Una àrea temporal on es recullen els canvis abans de comprometre'ls.
  • Fitxers seguits: Fitxers que Git coneix i dels quals està seguint els canvis.

Ús

# Afegeix un sol fitxer a l'àrea de preparació
git add <file_name>

# Afegeix tots els canvis en el directori actual a l'àrea de preparació
git add .

Exemple

# Afegint un sol fitxer
git add index.html

# Afegint tots els canvis en el directori actual
git add .

Comanda `commit` de Git

La comanda commit s'utilitza per desar els canvis de l'àrea de preparació al repositori. Cada commit és una instantània del teu projecte en un moment específic.

Conceptes clau

  • Missatge de commit: Una breu descripció dels canvis que s'estan comprometent.
  • Hash SHA-1: Un identificador únic per a cada commit.

Ús

# Compromet canvis amb un missatge
git commit -m "El teu missatge de commit aquí"

Exemple

# Comprometent canvis amb un missatge descriptiu
git commit -m "Afegida nova funcionalitat a la pàgina principal"

Comanda `status` de Git

La comanda status mostra l'estat del directori de treball i de l'àrea de preparació. Mostra quins canvis s'han preparat, quins no, i quins fitxers no estan sent seguits per Git.

Conceptes clau

  • Fitxers no seguits: Fitxers que no estan sent seguits per Git.
  • Fitxers modificats: Fitxers que han estat canviats però encara no s'han preparat.
  • Fitxers preparats: Fitxers que s'han afegit a l'àrea de preparació.

Ús

# Comprova l'estat del directori de treball i de l'àrea de preparació
git status

Exemple

# Mostrant l'estat del repositori
git status

Resum

Entendre les comandes add, commit i status és crucial per a un control de versions efectiu amb Git. Aquí tens un breu resum:

  • Utilitza git add per moure canvis a l'àrea de preparació.
  • Utilitza git commit per desar els canvis preparats al repositori amb un missatge descriptiu.
  • Utilitza git status per comprovar l'estat del teu directori de treball i de l'àrea de preparació.

Dominant aquestes comandes, estaràs ben encaminat per gestionar eficientment el teu codi amb Git.
