El Projecte Sobre nosaltres Contribuir Donacions Llicència

En aquesta secció, cobrim com enviar canvis des del teu repositori Git local a un repositori remot. Aquesta és una habilitat crucial per col·laborar amb altres i assegurar que el teu treball estigui recolzat.

Conceptes Clau

  • Repositori Remot: Una versió del teu projecte allotjada a internet o en una altra xarxa.
  • Repositori Local: La teva versió del projecte emmagatzemada a la teva màquina local.
  • Push: L'acció d'enviar els teus canvis compromesos a un repositori remot.

Configurant un Repositori Remot

Abans de poder enviar canvis, necessites tenir un repositori remot configurat. Això es pot fer en plataformes com GitHub, GitLab o Bitbucket.

Afegint un Repositori Remot

Per afegir un repositori remot, utilitza la comanda git remote add seguida d'un nom per al remot (comunament origin) i la URL del repositori remot.

git remote add origin https://github.com/username/repository.git
  • origin és un nom comú per al repositori remot principal.
  • Substitueix https://github.com/username/repository.git per la URL del teu repositori remot.

Enviant Canvis

Comanda Bàsica de Push

Per enviar canvis al repositori remot, utilitza la comanda git push seguida del nom del remot i la branca que vols enviar.

git push origin main
  • origin és el nom del repositori remot.
  • main és la branca a la qual estàs enviant. Substitueix main pel nom de la teva branca si és diferent.

Exemple de Flux de Treball

  1. Fer Canvis: Modifica fitxers en el teu repositori local.

  2. Etapa de Canvis: Afegeix els canvis a l'àrea d'etapa utilitzant git add.

    git add .

  3. Comprometre Canvis: Compromet els canvis amb un missatge.

    git commit -m "Descriu els teus canvis"

  4. Enviar Canvis: Envia els canvis compromesos al repositori remot.

    git push origin main

Gestionant Conflictes de Push

De vegades, pots trobar-te amb conflictes quan envies canvis. Això sol passar si algú altre ha enviat canvis a la mateixa branca.

  • Obtenir Canvis: Obteniu els últims canvis del repositori remot.

    git fetch origin

  • Fusionar Canvis: Fusiona els canvis obtinguts a la teva branca local.

    git merge origin/main

  • Resoldre Conflictes: Si hi ha conflictes, Git et demanarà que els resolguis.

  • Enviar de Nou: Després de resoldre els conflictes, compromet els canvis i envia de nou.

    git commit -m "Conflictes de fusió resolts"
git push origin main

Resum

Enviar canvis a un repositori remot és una part fonamental d'utilitzar Git per al control de versions. Et permet compartir el teu treball amb altres i mantenir el teu projecte recolzat. Els passos bàsics impliquen afegir un repositori remot, fer canvis, comprometre aquests canvis i després enviar-los al repositori remot. Gestionar conflictes també és una habilitat important per assegurar una col·laboració fluida. Dominant aquests passos, pots gestionar efectivament el teu codi i col·laborar amb altres.
