Introducció

El model d'objectes de Git és fonamental per entendre com Git emmagatzema i gestiona les dades. Aquesta secció cobrirà els quatre tipus principals d'objectes en Git: blobs, arbres, commits i etiquetes. Al final d'aquesta secció, tindràs una comprensió clara de com aquests objectes interactuen i com formen l'esquelet del sistema de control de versions de Git.

Blobs

Els blobs (Binary Large Objects) són el tipus d'objecte més simple en Git. Emmagatzemen dades de fitxers però no contenen cap metadada sobre el fitxer, com ara el seu nom o ubicació en l'estructura de directoris.

  • Conceptes Clau:

    • Els blobs emmagatzemen el contingut dels fitxers.
    • S'identifiquen per un hash SHA-1.
    • Els blobs no emmagatzemen noms de fitxers ni informació de directoris.

  • Exemple:

    echo "Hello, Git!" > hello.txt
git add hello.txt
git cat-file -p $(git hash-object hello.txt)

    Aquesta seqüència de comandes crea un fitxer hello.txt, l'afegeix a l'àrea d'staging i després mostra el contingut de l'objecte blob creat per a hello.txt.

Arbres

Els arbres són objectes que emmagatzemen informació de directoris. Mapejen noms de fitxers a objectes blob i també poden referenciar altres objectes arbre, permetent a Git representar una jerarquia de directoris.

  • Conceptes Clau:

    • Els arbres emmagatzemen estructures de directoris.
    • Mapejen noms de fitxers a objectes blob.
    • Els arbres poden referenciar altres arbres per representar directoris niats.

  • Exemple:

    mkdir project
echo "print('Hello, Git!')" > project/main.py
git add project
git write-tree

    Aquesta seqüència de comandes crea un directori project, afegeix un fitxer main.py a aquest, fa staging del directori i després escriu l'objecte arbre que representa l'estructura del directori.

Commits

Els commits són objectes que emmagatzemen l'estat del repositori en un moment donat. Referencien un objecte arbre (que representa l'estructura del directori) i inclouen metadades com l'autor, el committer i el missatge del commit.

  • Conceptes Clau:

    • Els commits emmagatzemen instantànies del repositori.
    • Referencien un objecte arbre.
    • Els commits inclouen metadades com l'autor, el committer i el missatge.
    • Els commits poden referenciar commits pares, formant una història.

  • Exemple:

    git commit -m "Initial commit"
git cat-file -p HEAD

    Aquesta seqüència de comandes fa commit dels canvis staged amb un missatge "Initial commit" i després mostra l'objecte commit.

Etiquetes

Les etiquetes són referències a commits específics. Sovint s'utilitzen per marcar punts de llançament (per exemple, v1.0, v2.0) en la història del repositori. Les etiquetes poden ser lleugeres (només una referència) o anotades (incloent metadades i un missatge).

  • Conceptes Clau:

    • Les etiquetes referencien commits específics.
    • Les etiquetes lleugeres són referències simples.
    • Les etiquetes anotades inclouen metadades i un missatge.

  • Exemple:

    git tag -a v1.0 -m "Version 1.0"
git show v1.0

    Aquesta seqüència de comandes crea una etiqueta anotada v1.0 amb un missatge "Version 1.0" i després mostra l'objecte etiqueta.

Resum

Entendre el model d'objectes de Git és crucial per dominar Git. Els blobs emmagatzemen continguts de fitxers, els arbres representen estructures de directoris, els commits capturen instantànies del repositori i les etiquetes marquen punts específics en la història. Junts, aquests objectes formen la base de les potents capacitats de control de versions de Git. Comprenent aquests conceptes, estaràs millor equipat per utilitzar Git de manera efectiva i resoldre problemes que puguin sorgir.
