Introducció

Extreure canvis d'un repositori remot és una operació fonamental a Git que et permet actualitzar el teu repositori local amb els últims canvis d'una font remota. Això assegura que la teva còpia local estigui sincronitzada amb el repositori remot, cosa que és crucial per al desenvolupament col·laboratiu.

Conceptes clau

  • Repositori remot: Una versió del teu projecte allotjada a internet o a una altra xarxa.
  • Repositori local: La teva versió del projecte emmagatzemada a la teva màquina local.
  • Operació de pull: Combina la recuperació de canvis del repositori remot i la seva fusió a la teva branca local.

La comanda `git pull`

La comanda git pull s'utilitza per recuperar i integrar canvis d'un repositori remot a la teva branca local.

Sintaxi bàsica

git pull <remote> <branch>
  • <remote>: El nom del repositori remot (per exemple, origin).
  • <branch>: El nom de la branca de la qual vols extreure els canvis (per exemple, main).

Exemple

git pull origin main

Aquesta comanda recupera els últims canvis de la branca main del repositori remot origin i els fusiona a la teva branca actual.

Guia pas a pas

Pas 1: Comprova la branca actual

Abans d'extreure canvis, assegura't que estàs a la branca correcta.

git branch

Aquesta comanda llista totes les branques i destaca la branca actual amb un asterisc (*).

Pas 2: Extreu canvis

Utilitza la comanda git pull per recuperar i fusionar canvis.

git pull origin main

Aquesta comanda farà:

  1. Recuperar els últims canvis de la branca main del remot origin.
  2. Fusionar aquests canvis a la teva branca actual.

Gestionar conflictes de fusió

De vegades, extreure canvis pot resultar en conflictes de fusió. Git t'avisarà si hi ha conflictes que necessiten ser resolts manualment.

Exemple d'un conflicte de fusió

Auto-merging file.txt
CONFLICT (content): Merge conflict in file.txt
Automatic merge failed; fix conflicts and then commit the result.

Per resoldre conflictes:

  1. Obre el(s) fitxer(s) conflictiu(s) en un editor de text.
  2. Busca els marcadors de conflicte (<<<<<<<, =======, >>>>>>>).
  3. Edita el fitxer per resoldre els conflictes.
  4. Afegeix el fitxer resolt a l'àrea de preparació: 
    git add file.txt
  5. Fes el commit de la fusió: 
    git commit

Ús avançat

Extreure amb rebase

En lloc de fusionar, pots rebasejar els teus canvis a sobre dels canvis recuperats.

git pull --rebase origin main

Aquesta comanda farà:

  1. Recuperar els últims canvis de la branca main del remot origin.
  2. Rebasejar la teva branca actual a sobre d'aquests canvis, creant una història lineal.

Especificar un remot diferent

Si tens múltiples remots, pots especificar de quin vols extreure.

git pull upstream develop

Aquesta comanda extreu canvis de la branca develop del remot upstream.

Resum

Extreure canvis d'un repositori remot és essencial per mantenir el teu repositori local actualitzat amb els últims canvis. La comanda git pull simplifica aquest procés combinant la recuperació i la fusió. Entendre com gestionar conflictes de fusió i utilitzar opcions avançades com el rebase pot millorar encara més el teu flux de treball.

Dominant la comanda git pull, assegures una col·laboració fluida i mantens la sincronització amb la feina del teu equip.
