Introducció a CI/CD

  • Integració Contínua (CI): La pràctica d'integrar automàticament els canvis de codi de múltiples contribuïdors en un repositori compartit diverses vegades al dia.
  • Entrega Contínua (CD): La pràctica de preparar automàticament els canvis de codi per a una versió a producció.
  • Desplegament Continu: Extén CD desplegant automàticament cada canvi que passa totes les etapes del pipeline de producció als clients.

Importància d'Integrar Git amb CI/CD

  • Automatització: Redueix la intervenció manual, minimitzant l'error humà.
  • Consistència: Assegura que el codi sempre estigui en un estat desplegable.
  • Retroalimentació: Proporciona retroalimentació immediata sobre la qualitat del codi i problemes d'integració.
  • Col·laboració: Facilita una millor col·laboració entre els membres de l'equip.

Configuració d'un Pipeline de CI/CD amb Git

Triar una Eina de CI/CD

  • Eines Populars:
    • Jenkins
    • GitLab CI/CD
    • CircleCI
    • Travis CI
    • GitHub Actions

Exemple: Utilitzant GitHub Actions per a CI/CD

  1. Crear un Repositori de GitHub:

    • Inicialitza un nou repositori a GitHub o utilitza'n un d'existent.
    • Assegura't que el teu repositori tingui un fitxer .gitignore i un README.md.

  2. Afegir un Fitxer de Workflow:

    • Crea un directori anomenat .github/workflows a l'arrel del teu repositori.
    • Afegeix un nou fitxer anomenat ci.yml dins d'aquest directori.

  3. Definir el Workflow:

    name: CI Pipeline

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

  4. Explicació del Workflow:

    • name: El nom del workflow.
    • on: Especifica els esdeveniments que desencadenen el workflow. Aquí, es desencadena en esdeveniments de push i pull_request a la branca main.
    • jobs: Defineix les tasques a executar. En aquest exemple, hi ha una única tasca anomenada build.
    • runs-on: Especifica el tipus de màquina per executar la tasca. ubuntu-latest és un entorn comunament utilitzat.
    • steps: Llista els passos a executar en la tasca.
      • Checkout code: Utilitza l'acció actions/checkout per clonar el repositori.
      • Set up Node.js: Utilitza l'acció actions/setup-node per configurar Node.js.
      • Install dependencies: Executa npm install per instal·lar les dependències del projecte.
      • Run tests: Executa npm test per executar la suite de proves.

Exemple: Utilitzant Jenkins per a CI/CD

  1. Instal·lar Jenkins:

    • Segueix la guia oficial d'instal·lació de Jenkins per al teu sistema operatiu.

  2. Crear una Nova Tasca de Jenkins:

    • Obre Jenkins i crea un nou projecte Freestyle.

  3. Configurar la Gestió del Codi Font:

    • A la secció "Source Code Management", selecciona "Git".
    • Introdueix l'URL del repositori i les credencials si és necessari.

  4. Afegir Passos de Construcció:

    • A la secció "Build", afegeix passos de construcció com "Execute shell" per executar comandes com:
    npm install
npm test

  5. Configurar Accions Post-Construcció:

    • Afegeix accions post-construcció com "Archive the artifacts" o "Publish JUnit test result report" per gestionar els resultats de la construcció.

Millors Pràctiques per a Integrar Git amb CI/CD

  • Estratègia de Branques: Utilitza una estratègia de branques consistent com GitFlow o GitHub Flow.
  • Missatges de Commit: Escriu missatges de commit clars i descriptius.
  • Proves Automatitzades: Assegura't que el teu pipeline de CI inclogui proves automatitzades.
  • Paritat d'Entorns: Mantingues els entorns de desenvolupament, staging i producció tan similars com sigui possible.
  • Seguretat: Assegura el teu pipeline de CI/CD gestionant correctament secrets i credencials.

Conclusió

Integrar Git amb pipelines de CI/CD és crucial per al desenvolupament de programari modern. Automatitza el procés d'integració, prova i desplegament de codi, assegurant que la teva aplicació estigui sempre en un estat desplegable. Seguint les millors pràctiques i utilitzant eines com GitHub Actions o Jenkins, pots optimitzar el teu flux de treball de desenvolupament i millorar la col·laboració dins del teu equip.
