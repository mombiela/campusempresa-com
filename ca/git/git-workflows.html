Introducció als fluxos de treball de Git

Els fluxos de treball de Git són un conjunt de directrius o models que dicten com els desenvolupadors haurien d'utilitzar Git per gestionar el seu codi. Entendre els diferents fluxos de treball és crucial per a una col·laboració i gestió de projectes efectiva.

Git Flow

Git Flow és un model de branques robust que proporciona un enfocament estructurat per gestionar versions i característiques.

Conceptes clau de Git Flow

  • Branques principals:
    • master: Conté codi llest per a producció.
    • develop: Integra característiques per a la propera versió.
  • Branques de suport:
    • feature/*: Per desenvolupar noves característiques.
    • release/*: Prepara per a una nova versió de producció.
    • hotfix/*: Per a correccions ràpides en producció.

Exemple de Git Flow

  1. Inicialitzar Git Flow:

    git flow init

    Aquesta comanda configura les branques necessàries.

  2. Iniciar una branca de característiques:

    git flow feature start feature-name

    Això crea una nova branca des de develop.

  3. Finalitzar una branca de característiques:

    git flow feature finish feature-name

    Això fusiona la branca de característiques de nou a develop.

  4. Iniciar una branca de versió:

    git flow release start release-name

    Això crea una nova branca des de develop.

  5. Finalitzar una branca de versió:

    git flow release finish release-name

    Això fusiona la branca de versió tant a master com a develop.

  6. Iniciar una branca de correcció ràpida:

    git flow hotfix start hotfix-name

    Això crea una nova branca des de master.

  7. Finalitzar una branca de correcció ràpida:

    git flow hotfix finish hotfix-name

    Això fusiona la branca de correcció ràpida tant a master com a develop.

GitHub Flow

GitHub Flow és un flux de treball més senzill dissenyat per a desplegament i integració contínua.

Conceptes clau de GitHub Flow

  • Branca principal:
    • main: Conté codi llest per a producció.
  • Branques de característiques:
    • Creats des de main per a noves característiques o correccions.
  • Pull Requests:
    • Utilitzats per revisar i fusionar canvis de nou a main.

Exemple de GitHub Flow

  1. Crear una branca de característiques:

    git checkout -b feature-name

    Això crea i canvia a una nova branca des de main.

  2. Cometre canvis:

    git add .
git commit -m "Descripció dels canvis"

  3. Empènyer la branca a GitHub:

    git push origin feature-name

  4. Crear una Pull Request:

    • Ves al repositori a GitHub.
    • Fes clic a "Compare & pull request".
    • Proporciona una descripció i envia la pull request.

  5. Fusionar la Pull Request:

    • Després de la revisió, fes clic a "Merge pull request" a GitHub.

Comparació entre Git Flow i GitHub Flow

| Característica | Git Flow | GitHub Flow | |------------------|-----------------------------------|---------------------------------| | Branques | Múltiples branques principals i de suport | Branca principal única amb branques de característiques | | Complexitat | Més complex, adequat per a projectes més grans | Més senzill, adequat per a desplegament continu | | Gestió de versions | Branques estructurades de versió i correcció ràpida | Integració contínua amb pull requests |

Altres fluxos de treball

GitLab Flow

GitLab Flow combina idees tant de Git Flow com de GitHub Flow, afegint més flexibilitat.

Conceptes clau de GitLab Flow

  • Branques d'entorn:
    • production, staging, development: Representen diferents entorns.
  • Branques de característiques:
    • Creats des de development per a noves característiques.
  • Merge Requests:
    • Similar a les pull requests en GitHub Flow.

Exemple de GitLab Flow

  1. Crear una branca de característiques:

    git checkout -b feature-name development

  2. Cometre i empènyer canvis:

    git add .
git commit -m "Descripció dels canvis"
git push origin feature-name

  3. Crear una Merge Request:

    • Ves al repositori a GitLab.
    • Fes clic a "New merge request".
    • Selecciona les branques d'origen i destinació i envia la merge request.

  4. Fusionar la sol·licitud:

    • Després de la revisió, fusiona la sol·licitud a la branca de destinació.

Conclusió

Entendre els diferents fluxos de treball de Git com Git Flow, GitHub Flow i GitLab Flow és essencial per a una gestió de projectes i col·laboració efectiva. Cada flux de treball té els seus punts forts i és adequat per a diferents necessitats de projecte. En dominar aquests fluxos de treball, pots assegurar un procés de desenvolupament suau i eficient.
