El Projecte Sobre nosaltres Contribuir Donacions Llicència

El rebase és una característica potent de Git que et permet reescriure la història dels commits. Es pot utilitzar per simplificar una sèrie de commits, actualitzar branques i mantenir una història del projecte més neta. Aquest tema cobrirà els conceptes bàsics del rebase, com realitzar un rebase i tècniques avançades de rebase.

Què és el Rebase?

El rebase és el procés de moure o combinar una seqüència de commits a un nou commit base. És una alternativa a la fusió i es pot utilitzar per mantenir una història del projecte lineal.

Conceptes Clau

  • Commit Base: El commit en el qual una branca divergeix d'una altra branca.
  • Branca Upstream: La branca sobre la qual estàs fent el rebase dels teus canvis.
  • Rebase vs. Merge: Mentre que la fusió crea un nou commit que combina els canvis de dues branques, el rebase mou tota la branca per començar des d'un nou commit base.

Rebase Bàsic

Com Fer un Rebase

Per fer un rebase d'una branca, utilitzes el comandament git rebase. Aquí tens un exemple senzill:

# Canvia a la branca que vols fer el rebase
git checkout feature-branch

# Fes el rebase sobre la branca principal
git rebase main

Exemple Explicat

  1. Canvia a la branca: Primer canvies a la branca que vols fer el rebase.
  2. Fes el rebase sobre la branca principal: Aquest comandament mourà els commits de feature-branch per començar des de l'últim commit a la branca main.

Gestionar Conflictes

Durant un rebase, podries trobar-te amb conflictes. Git pausarà el procés de rebase i et permetrà resoldre aquests conflictes.

# Resol els conflictes en el teu editor
# Després de resoldre els conflictes, afegeix els canvis
git add .

# Continua el procés de rebase
git rebase --continue

Si vols avortar el procés de rebase, pots utilitzar:

git rebase --abort

Rebase Interactiu

El rebase interactiu et permet editar, reordenar, combinar o eliminar commits. Això és útil per netejar la història dels commits abans de fusionar-los a la branca principal.

Com Realitzar un Rebase Interactiu

# Inicia un rebase interactiu
git rebase -i HEAD~n

Substitueix n pel nombre de commits que vols fer el rebase. Això obrirà un editor amb una llista de commits.

Exemple Explicat

  1. Pick: Utilitza això per mantenir un commit tal com està.
  2. Reword: Utilitza això per canviar el missatge del commit.
  3. Edit: Utilitza això per aturar i editar el commit.
  4. Squash: Utilitza això per combinar aquest commit amb l'anterior.
  5. Fixup: Com squash, però descarta el missatge del commit.
  6. Drop: Utilitza això per eliminar el commit.

Exemple

pick 1234567 Missatge del commit 1
squash 89abcde Missatge del commit 2
reword fghijkl Missatge del commit 3

En aquest exemple:

  • El segon commit es combinarà amb el primer.
  • El missatge del tercer commit serà editat.

Tècniques Avançades de Rebase

Rebase Sobre una Altra Branca

Pots fer el rebase de la teva branca sobre qualsevol altra branca, no només la branca principal.

git rebase another-branch

Utilitzant `--onto`

L'opció --onto et permet fer el rebase d'un rang de commits sobre una altra branca.

# Fes el rebase dels commits de feature-branch sobre another-branch
git rebase --onto another-branch main feature-branch

Exemple Explicat

  1. main: La branca de la qual es prenen els commits.
  2. feature-branch: La branca que conté els commits que es faran el rebase.
  3. another-branch: La branca sobre la qual es faran el rebase dels commits.

Resum

El rebase és una eina potent a Git que ajuda a mantenir una història del projecte neta i lineal. Es pot utilitzar per:

  • Moure una branca a un nou commit base.
  • Netejar la història dels commits amb el rebase interactiu.
  • Resoldre conflictes de manera controlada.

Tot i que el rebase pot ser complex, entendre els seus conceptes bàsics i practicar amb exemples senzills t'ajudarà a dominar aquesta característica essencial de Git. Recorda sempre comunicar-te amb el teu equip quan facis rebase en branques compartides per evitar interrompre el flux de treball.
