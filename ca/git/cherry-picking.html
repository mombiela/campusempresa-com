El cherry-picking a Git et permet aplicar els canvis introduïts per alguns commits existents a la teva branca actual. Això pot ser particularment útil quan vols incorporar canvis específics d'una branca a una altra sense fusionar tota la branca.

Què és el Cherry-Picking?

  • Definició: El cherry-picking és el procés de seleccionar commits específics d'una branca i aplicar-los a una altra.
  • Casos d'ús:
    • Aplicar correccions d'errors d'una branca a una altra.
    • Incorporar funcions o canvis específics sense fusionar tota la branca.
    • Retroportar canvis a versions més antigues del codi.

Com fer Cherry-Picking

Cherry-Picking bàsic

Per fer cherry-pick d'un commit, necessites el hash del commit (SHA) del commit que vols aplicar.

  1. Troba el hash del commit:

    git log

    Aquesta comanda mostrarà l'historial de commits. Copia el hash del commit que vols fer cherry-pick.

  2. Fes Cherry-Pick del Commit:

    git cherry-pick <commit-hash>

    Substitueix <commit-hash> pel hash real del commit.

Exemple

Suposem que tens el següent historial de commits:

commit 1a2b3c4d5e6f7g8h9i0j
Author: John Doe <john@example.com>
Date:   Dl Oct 4 12:34:56 2021 -0700

    Correcció d'error en l'autenticació d'usuaris

commit 0a9b8c7d6e5f4g3h2i1j
Author: Jane Smith <jane@example.com>
Date:   Dg Oct 3 11:23:45 2021 -0700

    Afegeix nova funció al tauler de control

Si vols aplicar la correcció d'error del commit 1a2b3c4d5e6f7g8h9i0j a la teva branca actual, executaries:

git cherry-pick 1a2b3c4d5e6f7g8h9i0j

Resolució de Conflictes

De vegades, el cherry-picking pot resultar en conflictes si els canvis en el commit que estàs fent cherry-pick entren en conflicte amb els canvis en la teva branca actual.

  1. Identifica els Conflictes: Git et notificarà dels conflictes i marcarà els fitxers conflictius.

  2. Resol els Conflictes: Obre els fitxers conflictius i resol manualment els conflictes. Busca els marcadors de conflicte (<<<<<<<, =======, >>>>>>>).

  3. Marca els Conflictes com a Resolts: Després de resoldre els conflictes, marca'ls com a resolts:

    git add <resolved-file>

  4. Continua el Cherry-Picking:

    git cherry-pick --continue

Cherry-Picking de Múltiples Commits

Pots fer cherry-pick d'un rang de commits especificant els hashes dels commits:

git cherry-pick <start-commit-hash>^..<end-commit-hash>

Per exemple, per fer cherry-pick dels commits des de 0a9b8c7d6e5f4g3h2i1j fins a 1a2b3c4d5e6f7g8h9i0j, executaries:

git cherry-pick 0a9b8c7d6e5f4g3h2i1j^..1a2b3c4d5e6f7g8h9i0j

Millors Pràctiques

  • Evita fer Cherry-Pick de Canvis Grans: El cherry-picking és més adequat per a canvis petits i aïllats. Per a canvis més grans, considera fer una fusió o un rebase.
  • Documenta els Cherry-Picks: Mantingues un registre dels commits que has fet cherry-pick per evitar confusions i possibles conflictes en el futur.
  • Prova a Fons: Sempre prova els canvis després de fer cherry-pick per assegurar-te que s'integren bé amb la branca actual.

Conclusió

El cherry-picking és una eina poderosa a Git que et permet aplicar selectivament canvis d'una branca a una altra. Entenent com fer cherry-pick de commits, resoldre conflictes i seguir les millors pràctiques, pots gestionar i incorporar efectivament canvis específics en el teu codi. Aquesta tècnica és particularment útil per aplicar correccions d'errors, retroportar canvis i integrar funcions específiques sense fusionar branques senceres.
