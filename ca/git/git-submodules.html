Introducció als Submòduls de Git

Els submòduls de Git són una manera de mantenir un repositori de Git com a subdirectori d'un altre repositori de Git. Això és útil quan vols incloure i gestionar repositoris externs dins del teu projecte.

Conceptes Clau

  • Submòdul: Un repositori incrustat dins d'un altre repositori.
  • Superprojecte: El repositori principal que conté el submòdul.
  • URL del Submòdul: L'URL del repositori que s'utilitzarà com a submòdul.
  • Commit del Submòdul: El commit específic del submòdul al qual el superprojecte està apuntant.

Afegir un Submòdul

Per afegir un submòdul al teu repositori de Git, utilitza la comanda git submodule add seguida de l'URL del repositori i el directori on vols col·locar-lo.

git submodule add https://github.com/example/repo.git path/to/submodule

Exemple

git submodule add https://github.com/example/library.git libs/library

Aquesta comanda afegirà el repositori https://github.com/example/library.git com a submòdul en el directori libs/library.

Clonar un Repositori amb Submòduls

Quan clones un repositori que conté submòduls, necessites inicialitzar-los i actualitzar-los.

git clone https://github.com/example/superproject.git
cd superproject
git submodule update --init --recursive

Explicació

  • git clone: Clona el repositori del superprojecte.
  • git submodule update --init --recursive: Inicialitza i actualitza tots els submòduls de manera recursiva.

Treballar amb Submòduls

Actualitzar Submòduls

Per actualitzar un submòdul al commit més recent del seu repositori remot, navega al directori del submòdul i fes un pull dels canvis.

cd path/to/submodule
git pull origin main

Cometre Canvis en Submòduls

Quan fas canvis en un submòdul, necessites cometre aquests canvis dins del submòdul i després actualitzar el superprojecte per apuntar al nou commit.

cd path/to/submodule
git add .
git commit -m "Updated submodule"
cd ../..
git add path/to/submodule
git commit -m "Updated submodule reference"

Eliminar un Submòdul

Per eliminar un submòdul, necessites seguir diversos passos per netejar la configuració i el directori del submòdul.

git submodule deinit -f path/to/submodule
rm -rf .git/modules/path/to/submodule
git rm -f path/to/submodule

Explicació

  • git submodule deinit -f path/to/submodule: Desinicialitza el submòdul.
  • rm -rf .git/modules/path/to/submodule: Elimina el directori del submòdul de les metadades de Git.
  • git rm -f path/to/submodule: Elimina el directori del submòdul del directori de treball i prepara el canvi per al commit.

Resum

Els submòduls de Git són una característica poderosa per gestionar dependències i incloure repositoris externs dins del teu projecte. Et permeten:

  • Afegir i gestionar submòduls dins d'un superprojecte.
  • Clonar repositoris amb submòduls i inicialitzar-los.
  • Actualitzar i cometre canvis en submòduls.
  • Eliminar submòduls de manera neta.

Entendre com utilitzar els submòduls de manera efectiva pot ajudar-te a mantenir un codi modular i organitzat, especialment en projectes complexos amb múltiples dependències.
