Fer un fork d'un repositori és un concepte fonamental en Git i GitHub que et permet crear una còpia personal del projecte d'una altra persona. Això és especialment útil per contribuir a projectes de codi obert o per experimentar amb canvis sense afectar el repositori original.

Què és fer un fork?

  • Definició: Fer un fork d'un repositori significa crear una còpia del repositori sota el teu propi compte de GitHub.
  • Propòsit: Et permet experimentar lliurement amb canvis sense afectar el projecte original. També pots proposar canvis al projecte original creant pull requests.

Passos per fer un fork d'un Repositori

  1. Navega al Repositori: Ves a la pàgina de GitHub del repositori que vols fer un fork.
  2. Fes clic al Botó de Fork: Situat a la cantonada superior dreta de la pàgina del repositori.
  3. Selecciona el Teu Compte: Tria el teu compte de GitHub o organització on vols fer el fork del repositori.

Clonar el Teu Fork

Un cop hagis fet el fork del repositori, necessites clonar-lo a la teva màquina local per començar a treballar-hi.

# Substitueix 'your-username' pel teu nom d'usuari de GitHub i 'repository-name' pel nom del repositori
git clone https://github.com/your-username/repository-name.git

Configurar Remots

Després de clonar el teu fork, és una bona pràctica configurar remots per mantenir el teu fork sincronitzat amb el repositori original.

  • origin: Apunta al teu fork.
  • upstream: Apunta al repositori original.
# Navega al teu repositori clonat
cd repository-name

# Afegeix el repositori original com a upstream
git remote add upstream https://github.com/original-owner/repository-name.git

# Verifica el nou remot anomenat 'upstream'
git remote -v

Mantenir el Teu Fork Actualitzat

Per mantenir el teu fork actualitzat amb els últims canvis del repositori original, necessites obtenir i fusionar els canvis del repositori upstream.

# Obten els últims canvis de l'upstream
git fetch upstream

# Fusiona els canvis a la teva branca local
git merge upstream/main

Fer Canvis i Contribuir

  1. Crea una Nova Branca: Sempre crea una nova branca per als teus canvis per mantenir el teu treball organitzat.

    git checkout -b feature-branch

  2. Fes Canvis: Edita fitxers, afegeix noves funcionalitats o corregeix errors.

  3. Fes Commit dels Teus Canvis: Estableix i fes commit dels teus canvis amb un missatge significatiu.

    git add .
git commit -m "Descripció dels canvis"

  4. Fes Push al Teu Fork: Fes push dels teus canvis al teu repositori forkejat a GitHub.

    git push origin feature-branch

  5. Crea un Pull Request: Ves al teu repositori forkejat a GitHub i fes clic al botó "Compare & pull request" per proposar els teus canvis al repositori original.

Resum

Fer un fork d'un repositori és una característica poderosa que et permet contribuir a projectes, experimentar amb canvis i col·laborar amb altres sense afectar el codi original. Seguint els passos descrits anteriorment, pots fer un fork d'un repositori, clonar-lo, configurar remots, mantenir el teu fork actualitzat i contribuir els teus canvis al projecte original. Entendre aquests conceptes és essencial per a una col·laboració efectiva en la comunitat de codi obert i per gestionar els teus propis projectes.
