Introducció

Comprendre els interns de Git pot millorar significativament la teva capacitat per utilitzar-lo de manera efectiva i resoldre problemes. Aquesta secció aprofundirà en els conceptes i estructures bàsiques que fan de Git un sistema de control de versions potent.

Conceptes Clau

Objectes de Git

Git emmagatzema dades en una estructura coneguda com el model d'objectes de Git. Hi ha quatre tipus principals d'objectes:

  • Blob: Representa el contingut d'un fitxer.
  • Tree: Representa un directori i conté punters a blobs i altres arbres.
  • Commit: Representa una instantània del repositori en un moment determinat.
  • Tag: Apunta a un altre objecte de Git (normalment un commit) i pot emmagatzemar metadades.

Emmagatzematge d'Objectes

Els objectes de Git s'emmagatzemen en un directori anomenat .git/objects. Cada objecte s'identifica per un hash SHA-1.

Referències de Git

Les referències (refs) són punters a commits. Les referències comunes inclouen:

  • Branques: Punters al commit més recent en una línia de desenvolupament.
  • Tags: Punters amb nom a commits específics.
  • HEAD: Una referència especial que apunta a la branca actual.

Explorant Objectes de Git

Creació i Inspecció de Blobs

Quan afegeixes un fitxer a Git, es crea un objecte blob.

echo "Hola, Git!" > hola.txt
git add hola.txt

Per inspeccionar el blob:

git cat-file -p <blob_hash>

Creació i Inspecció d'Arbres

Els arbres es creen quan fas un commit dels canvis.

git commit -m "Afegeix hola.txt"

Per inspeccionar l'arbre:

git cat-file -p <tree_hash>

Creació i Inspecció de Commits

Els commits enllacen arbres i proporcionen un historial de canvis.

git commit -m "Commit inicial"

Per inspeccionar el commit:

git cat-file -p <commit_hash>

Referències de Git i HEAD

Branques

Les branques són simplement punters a commits.

git branch nova-funcionalitat
git checkout nova-funcionalitat

Tags

Els tags s'utilitzen per marcar punts específics en la història.

git tag -a v1.0 -m "Versió 1.0"

HEAD

HEAD apunta a la branca o commit actual.

git checkout master

Exercicis Pràctics

Exercici 1: Inspeccionant Objectes de Git

  1. Crea un nou fitxer i afegeix-lo a Git.
  2. Fes un commit dels canvis.
  3. Utilitza git cat-file per inspeccionar els objectes blob, arbre i commit.

Exercici 2: Treballant amb Branques i Tags

  1. Crea una nova branca i canvia a ella.
  2. Fes alguns canvis i fes un commit.
  3. Crea un tag per al commit actual.
  4. Torna a la branca master i inspecciona els canvis.

Conclusió

Comprendre els interns de Git proporciona una visió més profunda de com Git gestiona les teves dades, la qual cosa pot ser inestimable per depurar i optimitzar el teu flux de treball. Dominant els objectes de Git, les referències i el punter HEAD, pots aprofitar tot el potencial de Git i convertir-te en un desenvolupador més competent.
