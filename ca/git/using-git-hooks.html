Forking un Repositori ►

Els hooks de Git són scripts que Git executa abans o després d'esdeveniments com commit, push i receive. Et permeten automatitzar tasques i aplicar polítiques en el teu flux de treball de desenvolupament.

Què són els Git Hooks?

  • Definició: Els hooks de Git són scripts personalitzats que s'executen automàticament en punts específics del flux de treball de Git.
  • Tipus: Hi ha dos tipus de hooks de Git:
    • Hooks del costat del client: S'executen en operacions com commit i merge.
    • Hooks del costat del servidor: S'executen en operacions de xarxa com rebre commits pujats.

Configuració dels Git Hooks

Localització del Directori de Hooks

  • Els hooks de Git es guarden al directori .git/hooks del teu repositori.
  • Per defecte, aquest directori conté scripts de mostra que estan desactivats (els seus noms de fitxer acaben en .sample).

Creació d'un Hook

  1. Navega al directori de hooks: 
    cd .git/hooks
  2. Crea un nou script de hook:
    • Per exemple, per crear un hook pre-commit: 
      touch pre-commit
  3. Fes l'script executable: 
    chmod +x pre-commit

Exemple: Hook Pre-commit

Un hook pre-commit es pot utilitzar per comprovar problemes d'estil de codi abans de permetre un commit.

#!/bin/sh
# Exemple d'script de hook pre-commit

# Executa un linter (per exemple, ESLint per JavaScript)
eslint .

if [ $? -ne 0 ]; then
  echo "La comprovació de l'estil de codi ha fallat. Commit abortat."
  exit 1
fi

echo "La comprovació de l'estil de codi ha passat. Commit permès."
exit 0
  • Explicació:
    • #!/bin/sh: Especifica l'intèrpret de l'script.
    • eslint .: Executa el linter ESLint al directori actual.
    • if [ $? -ne 0 ]; then ... fi: Comprova l'estat de sortida del linter. Si és diferent de zero (indicant un error), el commit s'aborta.

Hooks de Git Comuns

| Nom del Hook | Esdeveniment que el desencadena | Cas d'ús típic | |-----------------|----------------------------------------|-----------------------------------------------| | pre-commit | Abans de fer un commit | Comprovació d'estil de codi, execució de tests| | commit-msg | Després d'introduir el missatge de commit | Aplicació del format del missatge de commit | | post-commit | Després de fer un commit | Notificacions, registre | | pre-push | Abans de fer un push a un repositori remot | Execució de tests, comprovació de polítiques de branques | | post-receive | Després de rebre un push per part d'un remot | Desplegament de codi, tasques d'integració contínua |

Ús Avançat

Compartir Hooks Entre Projectes

  • Git no suporta nativament compartir hooks entre repositoris. No obstant això, pots:
    • Utilitzar un directori versionat per als hooks i enllaçar-los simbòlicament a .git/hooks.
    • Utilitzar una eina com husky (per a projectes JavaScript) per gestionar els hooks.

Exemple: Ús de Husky

  1. Instal·la Husky: 
    npm install husky --save-dev
  2. Afegeix un hook: 
    npx husky add .husky/pre-commit "npm test"
  3. Fes executable el directori de Husky: 
    chmod +x .husky/pre-commit
  • Explicació:
    • Husky simplifica el procés d'afegir i gestionar hooks en projectes JavaScript.
    • El hook pre-commit executa npm test abans de permetre un commit.

Conclusió

Els hooks de Git són eines poderoses que poden ajudar a automatitzar i aplicar polítiques en el teu flux de treball de desenvolupament. Entenent com configurar i utilitzar els hooks de Git, pots millorar la qualitat del codi i agilitzar els processos. Tant si estàs comprovant l'estil de codi, executant tests o aplicant formats de missatges de commit, els hooks de Git proporcionen una manera flexible d'integrar aquestes tasques en el teu flux de treball de Git.
