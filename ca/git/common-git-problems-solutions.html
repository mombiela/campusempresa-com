Introducció

Git és un sistema de control de versions potent, però pot ser desafiant, especialment per als principiants. Aquesta secció cobreix problemes comuns de Git i les seves solucions, ajudant-te a navegar pels problemes que podries trobar.

Problema 1: Oblidar Estagiar Canvis Abans de Cometre

Solució

Si has oblidat estagiar els teus canvis abans de cometre, pots esmenar el commit per incloure els canvis.

# Estagiar els canvis
git add .

# Esmenar el commit anterior
git commit --amend --no-edit
  • git add . estagia tots els canvis.
  • git commit --amend --no-edit esmena l'últim commit sense canviar el seu missatge.

Problema 2: Cometre a la Branca Incorrecta

Solució

Si has comès accidentalment a la branca incorrecta, pots moure el commit a la branca correcta.

# Crear una nova branca des de l'estat actual
git branch correct-branch

# Canviar a la branca correcta
git checkout correct-branch

# Reiniciar la branca original al commit anterior
git checkout previous-branch
git reset HEAD~1
  • git branch correct-branch crea una nova branca des de l'estat actual.
  • git checkout correct-branch canvia a la branca correcta.
  • git reset HEAD~1 reinicia la branca original al commit anterior.

Problema 3: Conflictes de Fusió

Solució

Els conflictes de fusió ocorren quan els canvis en diferents branques entren en conflicte. Resol-los manualment.

  1. Identifica els fitxers en conflicte.
  2. Obre els fitxers i busca els marcadors de conflicte (<<<<<<<, =======, >>>>>>>).
  3. Edita els fitxers per resoldre els conflictes.
  4. Estagia els fitxers resolts i cometa.
# Estagiar els fitxers resolts
git add resolved-file

# Cometre la resolució
git commit -m "Resolved merge conflict"

Problema 4: Esborrar Accidentalment una Branca

Solució

Si has esborrat accidentalment una branca, pots recuperar-la si no ha estat recollida per escombraries.

# Llistar totes les branques, incloent les esborrades
git reflog

# Trobar el hash del commit de la branca esborrada
# Crear una nova branca des d'aquest commit
git checkout -b recovered-branch <commit-hash>
  • git reflog llista totes les accions, incloent les branques esborrades.
  • git checkout -b recovered-branch <commit-hash> crea una nova branca des del hash del commit.

Problema 5: Empènyer Dades Sensibles

Solució

Si has empès accidentalment dades sensibles, necessites eliminar-les de l'historial.

  1. Elimina les dades sensibles dels teus fitxers.
  2. Utilitza git filter-branch per reescriure l'historial.
# Eliminar les dades sensibles de l'historial
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch path/to/sensitive/file' \
  --prune-empty --tag-name-filter cat -- --all

# Empènyer els canvis
git push origin --force --all
git push origin --force --tags
  • git filter-branch reescriu l'historial per eliminar el fitxer sensible.
  • git push --force actualitza el repositori remot.

Problema 6: Estat de HEAD Desconnectat

Solució

Un estat de HEAD desconnectat ocorre quan fas checkout d'un commit específic en lloc d'una branca.

# Crear una nova branca des de l'estat de HEAD desconnectat
git checkout -b new-branch

# Continuar treballant a la nova branca
  • git checkout -b new-branch crea una nova branca des de l'estat actual.

Conclusió

Entendre els problemes comuns de Git i les seves solucions pot millorar significativament el teu flux de treball. En dominar aquestes solucions, estaràs millor equipat per manejar problemes i mantenir un procés de desenvolupament fluid. Practica aquestes solucions per convertir-te en més proficient amb Git.
