Introducció

Git és un potent sistema de control de versions que ajuda els desenvolupadors a gestionar els canvis en el seu codi. Entendre la terminologia bàsica de Git és essencial per utilitzar Git de manera efectiva. Aquesta secció cobrirà termes i conceptes clau que formen la base de Git.

Conceptes clau

Repositori (Repo)

  • Definició: Un repositori és un espai d'emmagatzematge on viu el teu projecte. Pot ser local en una carpeta del teu ordinador o allotjat en un servidor remot com GitHub.
  • Exemple: 
    git init my-repo
    Aquesta comanda inicialitza un nou repositori Git al directori my-repo.

Commit

  • Definició: Un commit és una instantània del teu repositori en un moment específic. Registra els canvis als fitxers i directoris.
  • Exemple: 
    git commit -m "Commit inicial"
    Aquesta comanda crea un nou commit amb el missatge "Commit inicial".

Branch

  • Definició: Una branca és una versió paral·lela del teu repositori. Et permet treballar en diferents característiques o correccions de manera independent.
  • Exemple: 
    git branch new-feature
    Aquesta comanda crea una nova branca anomenada new-feature.

Merge

  • Definició: Fusionar és el procés de combinar canvis de diferents branques en una sola.
  • Exemple: 
    git merge new-feature
    Aquesta comanda fusiona la branca new-feature en la branca actual.

Clone

  • Definició: Clonar és el procés de crear una còpia d'un repositori remot al teu ordinador local.
  • Exemple: 
    git clone https://github.com/user/repo.git
    Aquesta comanda clona el repositori situat a l'URL donat al teu ordinador local.

Pull

  • Definició: Fer un pull és el procés de recuperar canvis d'un repositori remot i fusionar-los en el teu repositori local.
  • Exemple: 
    git pull origin main
    Aquesta comanda recupera i fusiona canvis de la branca main del repositori remot anomenat origin.

Push

  • Definició: Fer un push és el procés d'enviar els teus commits locals a un repositori remot.
  • Exemple: 
    git push origin main
    Aquesta comanda envia els teus commits locals a la branca main del repositori remot anomenat origin.

Àrea d'Indexació (Index)

  • Definició: L'àrea d'indexació és un lloc on pots agrupar canvis abans de fer un commit.
  • Exemple: 
    git add file.txt
    Aquesta comanda afegeix file.txt a l'àrea d'indexació.

Remote

  • Definició: Un remote és una versió del teu repositori allotjada a internet o en una altra xarxa.
  • Exemple: 
    git remote add origin https://github.com/user/repo.git
    Aquesta comanda afegeix un repositori remot anomenat origin amb l'URL donat.

Resum

Entendre aquests termes bàsics de Git és crucial per utilitzar Git de manera efectiva en el teu flux de treball de desenvolupament. Aquí tens un resum ràpid:

  • Repositori: Espai d'emmagatzematge per al teu projecte.
  • Commit: Instantània del teu repositori.
  • Branca: Versió paral·lela del teu repositori.
  • Fusionar: Combinar canvis de diferents branques.
  • Clonar: Copiar un repositori remot localment.
  • Pull: Recuperar i fusionar canvis d'un repositori remot.
  • Push: Enviar commits locals a un repositori remot.
  • Àrea d'Indexació: Agrupar canvis abans de fer un commit.
  • Remote: Una versió del teu repositori allotjada en una xarxa.

En dominar aquests termes, estaràs ben equipat per navegar i utilitzar Git de manera efectiva en els teus projectes.
