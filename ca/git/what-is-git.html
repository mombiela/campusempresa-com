Introducció

Git és un sistema de control de versions distribuït (VCS) que permet a múltiples persones treballar en un projecte simultàniament sense sobrescriure els canvis dels altres. Ajuda a fer un seguiment dels canvis, revertir a etapes anteriors i col·laborar de manera eficient. Creat per Linus Torvalds el 2005, Git s'ha convertit en el sistema de control de versions més utilitzat al món.

Conceptes Clau

  • Sistema de Control de Versions (VCS): Una eina que ajuda a gestionar els canvis en el codi font al llarg del temps.
  • Distribuït: A diferència dels VCS centralitzats, Git permet que cada usuari tingui una còpia completa del repositori.
  • Repositori (Repo): Una ubicació d'emmagatzematge per a paquets de programari, que conté tots els fitxers del projecte i la història dels canvis.
  • Commit: Una instantània del teu repositori en un moment específic.
  • Branca: Una línia de desenvolupament separada, que et permet treballar en diferents característiques o correccions de manera independent.
  • Merge: El procés de combinar canvis de diferents branques.
  • Clone: Crear una còpia d'un repositori des d'una font remota.
  • Pull: Obtenir canvis d'un repositori remot i fusionar-los en el teu repositori local.
  • Push: Enviar els teus canvis locals a un repositori remot.

Flux de Treball Bàsic de Git

  1. Inicialitzar un Repositori: Crear un nou repositori Git o clonar-ne un d'existent.
  2. Fer Canvis: Modificar fitxers en el teu directori de treball.
  3. Estagiar Canvis: Afegir canvis a l'àrea d'estagi.
  4. Commetre Canvis: Guardar una instantània dels canvis estagiats.
  5. Empènyer Canvis: Enviar els teus commits a un repositori remot.

Comandes Bàsiques de Git

Inicialitzar un Repositori

Per començar un nou repositori Git, navega al teu directori de projecte i executa:

git init

Aquesta comanda crea un nou directori .git en el teu projecte, que conté totes les metadades necessàries per al repositori.

Clonar un Repositori

Per clonar un repositori existent, utilitza:

git clone <repository_url>

Aquesta comanda copia tot el repositori, incloent-hi la seva història, a la teva màquina local.

Estagiar Canvis

Per afegir canvis a l'àrea d'estagi, utilitza:

git add <file_name>

També pots afegir tots els canvis alhora:

git add .

Commetre Canvis

Per commetre els canvis estagiats, utilitza:

git commit -m "El teu missatge de commit"

El flag -m et permet afegir un missatge de commit en línia.

Empènyer Canvis

Per empènyer els teus commits a un repositori remot, utilitza:

git push origin <branch_name>

Aquesta comanda envia els teus commits locals a la branca especificada en el repositori remot.

Obtenir Canvis

Per obtenir i fusionar canvis d'un repositori remot, utilitza:

git pull origin <branch_name>

Aquesta comanda actualitza el teu repositori local amb els últims canvis de la branca remota.

Exemple de Flux de Treball

  1. Inicialitzar un Repositori:

    git init

  2. Crear un Fitxer:

    echo "Hello, Git!" > hello.txt

  3. Estagiar el Fitxer:

    git add hello.txt

  4. Commetre el Fitxer:

    git commit -m "Add hello.txt"

  5. Crear un Repositori Remot (per exemple, a GitHub) i enllaçar-lo:

    git remote add origin <repository_url>

  6. Empènyer el Commit:

    git push -u origin master

Resum

Git és un sistema de control de versions potent i flexible que suporta el desenvolupament distribuït. Entendre els seus conceptes bàsics i comandes és essencial per a una col·laboració i gestió de projectes eficient. En dominar Git, pots fer un seguiment dels canvis, col·laborar amb altres i mantenir un historial del desenvolupament del teu projecte.
