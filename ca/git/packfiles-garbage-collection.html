Introducció

En aquesta secció, explorarem dos conceptes importants en Git: Fitxers de paquet i Recollida d'escombraries. Entendre aquests conceptes és crucial per gestionar la mida i el rendiment del repositori, especialment a mesura que el teu projecte creix.

Fitxers de paquet

Què són els fitxers de paquet?

  • Definició: Els fitxers de paquet són col·leccions comprimides d'objectes en Git. Ajuden a reduir l'espai de disc utilitzat pel repositori i milloren el rendiment.
  • Propòsit: Emmagatzemen múltiples objectes en un sol fitxer, fent eficient la transferència i l'emmagatzematge de dades.

Com funcionen els fitxers de paquet

  • Compressió d'objectes: Git comprimeix objectes (commits, arbres, blobs) i els emmagatzema en fitxers de paquet.
  • Compressió delta: Git utilitza la compressió delta per emmagatzemar objectes com a diferències (deltas) d'altres objectes, reduint encara més la mida.

Creació de fitxers de paquet

  • Empaquetament automàtic: Git crea automàticament fitxers de paquet durant certes operacions com clonar, obtenir i empènyer.
  • Empaquetament manual: Pots crear manualment fitxers de paquet utilitzant la comanda git gc (recollida d'escombraries).
# Crear manualment fitxers de paquet
git gc

Visualització de fitxers de paquet

  • Llistar fitxers de paquet: Pots llistar fitxers de paquet al directori .git/objects/pack.
  • Inspeccionar fitxers de paquet: Utilitza la comanda git verify-pack per inspeccionar el contingut d'un fitxer de paquet.
# Llistar fitxers de paquet
ls .git/objects/pack

# Inspeccionar un fitxer de paquet
git verify-pack -v .git/objects/pack/pack-<hash>.idx

Recollida d'escombraries

Què és la recollida d'escombraries?

  • Definició: La Recollida d'escombraries (GC) en Git és el procés de netejar fitxers innecessaris i optimitzar el repositori.
  • Propòsit: Ajuda a recuperar espai de disc i millorar el rendiment eliminant objectes inaccessibles.

Com funciona la recollida d'escombraries

  • Objectes inaccessibles: Els objectes que ja no són accessibles des de cap branca o etiqueta es consideren escombraries.
  • Poda: Git poda aquests objectes inaccessibles durant la recollida d'escombraries.

Executar la recollida d'escombraries

  • GC automàtic: Git executa la recollida d'escombraries automàticament durant certes operacions, com fer commits o canviar de branques.
  • GC manual: Pots activar manualment la recollida d'escombraries utilitzant la comanda git gc.
# Executar la recollida d'escombraries manualment
git gc

Configuració de la recollida d'escombraries

  • Opcions de configuració: Pots configurar com i quan Git executa la recollida d'escombraries utilitzant diverses configuracions al fitxer .gitconfig.
# Exemple de configuració a .gitconfig
[gc]
    auto = 256  # Executar GC després de 256 objectes solts
    pruneExpire = "2.weeks.ago"  # Podar objectes més antics de 2 setmanes

Problemes comuns i solucions

  • Repositoris grans: Per a repositoris molt grans, executar git gc pot ser intensiu en recursos. Considera utilitzar git gc --aggressive per a una neteja més exhaustiva.
  • GC interromput: Si la recollida d'escombraries és interrompuda, pot deixar el repositori en un estat inconsistent. Executar git fsck pot ajudar a identificar i solucionar problemes.
# Executar recollida d'escombraries agressiva
git gc --aggressive

# Comprovar la integritat del repositori
git fsck

Resum

Entendre els fitxers de paquet i la recollida d'escombraries és essencial per mantenir la salut i el rendiment dels teus repositoris Git. Els fitxers de paquet ajuden a reduir l'espai d'emmagatzematge i millorar l'eficiència de la transferència de dades, mentre que la recollida d'escombraries neteja objectes innecessaris per recuperar espai de disc i optimitzar el rendiment. Executar i configurar regularment aquests processos assegura que el teu repositori es mantingui eficient i manejable a mesura que creix.
