El Projecte Sobre nosaltres Contribuir Donacions Llicència

Obtenir canvis d'un repositori remot és una habilitat fonamental en Git que et permet mantenir el teu repositori local actualitzat amb els últims canvis d'una font remota. Aquest tema cobrirà els conceptes bàsics d'obtenir, com es diferencia de fer un pull, i alguns escenaris d'ús avançat.

Què és Obtenir?

Obtenir en Git és el procés de descarregar objectes i referències d'un altre repositori. Actualitza les teves branques de seguiment remot però no fusiona cap canvi en el teu directori de treball.

  • Branques de seguiment remot: Aquestes són referències a l'estat de les branques en els teus repositoris remots.
  • Directori de treball: El directori on es troben els fitxers del teu projecte.

Obtenir Bàsic

Per obtenir canvis d'un repositori remot, utilitzes la comanda git fetch. Aquesta comanda descarrega commits, fitxers i referències d'un repositori remot al teu repositori local.

Exemple

git fetch origin
  • origin és el nom per defecte del teu repositori remot. Pots substituir-lo pel nom de qualsevol altre repositori remot que hagis configurat.

Explicació

  • git fetch: La comanda per obtenir canvis.
  • origin: El nom del repositori remot del qual vols obtenir canvis.

Obtenir vs. Fer un Pull

És important entendre la diferència entre obtenir i fer un pull.

  • Obtenir: Descarrega canvis del repositori remot però no els aplica al teu directori de treball.
  • Fer un Pull: Combina obtenir amb fusionar. Descarrega canvis i immediatament intenta fusionar-los en la teva branca actual.

Taula de Comparació

| Comanda | Acció | Cas d'Ús | |--------------|-----------------------------------------------------------------------|-----------------------------------------------| | git fetch | Descarrega canvis del repositori remot sense fusionar-los. | Quan vols revisar els canvis abans de fusionar-los.| | git pull | Descarrega i fusiona canvis del repositori remot. | Quan vols actualitzar la teva branca directament. |

Obtenir Branques Específiques

Pots obtenir canvis per a una branca específica si no vols obtenir tot del repositori remot.

Exemple

git fetch origin feature-branch
  • feature-branch és el nom de la branca que vols obtenir.

Explicació

  • git fetch: La comanda per obtenir canvis.
  • origin: El nom del repositori remot.
  • feature-branch: La branca específica que vols obtenir.

Obtenir Tots els Remots

Si tens múltiples remots configurats, pots obtenir canvis de tots ells alhora.

Exemple

git fetch --all

Explicació

  • --all: Una opció que diu a Git que obtingui canvis de tots els remots configurats.

Obtenir Avançat

Obtenir amb Prune

Amb el temps, les branques en el repositori remot poden ser eliminades. L'opció --prune elimina qualsevol branca de seguiment remot que ja no existeixi en el remot.

Exemple

git fetch --prune

Explicació

  • --prune: Una opció que elimina les branques de seguiment remot que han estat eliminades del repositori remot.

Conclusió

Obtenir canvis d'un repositori remot és una part crucial de treballar amb Git, permetent-te mantenir el teu repositori local actualitzat sense fusionar immediatament els canvis. Entendre la diferència entre obtenir i fer un pull, i saber com obtenir branques específiques o tots els remots, et donarà un major control sobre el teu flux de treball. Utilitza l'opció --prune per mantenir les teves branques de seguiment remot netes i actualitzades.
