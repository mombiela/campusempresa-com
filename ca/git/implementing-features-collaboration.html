Introducció

En aquesta secció, explorarem com implementar noves funcionalitats i col·laborar eficaçment utilitzant Git. Això inclou crear branques, fusionar canvis, resoldre conflictes i utilitzar pull requests. Al final d'aquesta secció, seràs capaç de gestionar el teu codi de manera col·laborativa i eficient.

Creació de Branques en Git

Què és una Branca?

Una branca en Git és un punter mòbil lleuger a un commit. Les branques permeten divergir de la línia principal de desenvolupament i continuar treballant sense afectar la línia principal.

Creació d'una Branca

Per crear una nova branca, utilitza el comandament git branch seguit del nom de la branca.

git branch feature-branch

Canviar a una Branca

Per canviar a la nova branca creada, utilitza el comandament git checkout.

git checkout feature-branch

Alternativament, pots crear i canviar a una nova branca en un sol comandament:

git checkout -b feature-branch

Llistar Branques

Per llistar totes les branques del teu repositori, utilitza:

git branch

Fusionar Branques

Què és la Fusió?

La fusió és el procés de combinar els canvis d'una branca en una altra. Això s'utilitza comunament per integrar branques de funcionalitats de nou a la branca principal.

Fusionar una Branca

Per fusionar una branca en la teva branca actual, utilitza el comandament git merge seguit del nom de la branca.

git checkout main
git merge feature-branch

Gestionar Conflictes de Fusió

De vegades, els canvis en diferents branques entren en conflicte entre si. Git et demanarà que resolguis aquests conflictes manualment.

Exemple d'un Conflicte de Fusió

Suposem que tens el següent fitxer en ambdues branques:

branca principal (file.txt):

Hello, World!

feature-branch (file.txt):

Hello, Git!

Quan intentes fusionar feature-branch en main, Git produirà un conflicte:

file.txt:

<<<<<<< HEAD
Hello, World!
=======
Hello, Git!
>>>>>>> feature-branch

Per resoldre el conflicte, edita el fitxer a l'estat desitjat i després afegeix i commita els canvis:

git add file.txt
git commit -m "Resolved merge conflict in file.txt"

Col·laboració amb Pull Requests

Què és un Pull Request?

Un pull request (PR) és una manera de proposar canvis a un repositori. Permet als membres de l'equip revisar i discutir els canvis abans que siguin fusionats.

Crear un Pull Request

  1. Puja la teva branca al repositori remot:

    git push origin feature-branch

  2. Ves al repositori a GitHub (o un altre servei d'allotjament de Git) i navega a la pestanya "Pull requests".

  3. Fes clic a "New pull request" i selecciona la teva branca.

  4. Afegeix un títol i una descripció per al teu pull request, després fes clic a "Create pull request."

Revisar i Fusionar un Pull Request

Els membres de l'equip poden revisar el pull request, deixar comentaris i suggerir canvis. Un cop el pull request és aprovat, es pot fusionar a la branca principal.

Resum

En aquesta secció, hem cobert:

  • Branques: Crear, canviar i llistar branques.
  • Fusió: Combinar canvis de diferents branques i resoldre conflictes.
  • Pull Requests: Proposar, revisar i fusionar canvis col·laborativament.

Dominant aquests conceptes, podràs gestionar i col·laborar efectivament en el codi en un repositori Git, assegurant un flux de treball de desenvolupament suau i eficient.
