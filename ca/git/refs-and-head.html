Introducció

En Git, entendre els conceptes de referències (refs) i HEAD és crucial per navegar i gestionar el teu repositori de manera efectiva. Aquesta secció cobrirà què són les referències i HEAD, com funcionen i com utilitzar-les en les teves operacions diàries amb Git.

Què són les referències?

Les referències, abreujades com a refs, són punters a commits en un repositori Git. S'utilitzen per fer un seguiment de diversos estats en el repositori, com ara branques, etiquetes i branques de seguiment remot.

Tipus de referències

  • Branques: Pointers al commit més recent en una seqüència de commits. Per exemple, main o feature-branch.
  • Etiquetes: Pointers a commits específics, sovint utilitzats per marcar punts de llançament com v1.0 o v2.0.
  • Branques de seguiment remot: Pointers a l'estat de les branques en repositoris remots, com ara origin/main.

Veure les referències

Pots veure les referències en el teu repositori utilitzant les següents comandes:

# Llistar totes les branques
git branch

# Llistar totes les etiquetes
git tag

# Llistar totes les branques de seguiment remot
git branch -r

Què és HEAD?

HEAD és una referència especial que apunta al commit actual en què es basa el teu directori de treball. És essencialment un punter a la referència de la branca actual.

Estat de HEAD desenganxat

Quan HEAD apunta directament a un commit en lloc d'una branca, es diu que està en un estat de "HEAD desenganxat". Això pot passar quan fas checkout d'un commit específic:

# Fer checkout d'un commit específic
git checkout <commit-hash>

En aquest estat, qualsevol nou commit no estarà associat amb cap branca.

Moure HEAD

HEAD es mou automàticament quan canvies de branca o fas nous commits. També pots moure HEAD manualment:

# Fer checkout d'una branca
git checkout <branch-name>

# Reiniciar HEAD a un commit específic
git reset --hard <commit-hash>

Exemples pràctics

Exemple 1: Crear i canviar de branques

# Crear una nova branca
git branch feature-branch

# Canviar a la nova branca
git checkout feature-branch

En aquest exemple, HEAD es mou per apuntar a feature-branch.

Exemple 2: Etiquetar un commit

# Etiquetar el commit actual
git tag v1.0

# Llistar totes les etiquetes
git tag

Les etiquetes són útils per marcar punts específics en la història del teu projecte, com ara llançaments.

Exemple 3: Estat de HEAD desenganxat

# Fer checkout d'un commit específic
git checkout <commit-hash>

# Fer un nou commit en estat de HEAD desenganxat
echo "New change" >> file.txt
git add file.txt
git commit -m "Commit in detached HEAD state"

En aquest estat, el nou commit no està associat amb cap branca.

Resum

  • Referències són punters a commits i inclouen branques, etiquetes i branques de seguiment remot.
  • HEAD és una referència especial que apunta al commit actual en què es basa el teu directori de treball.
  • Entendre com veure, crear i manipular les referències i HEAD és essencial per a un ús efectiu de Git.
  • Sigues cautelós amb l'estat de HEAD desenganxat, ja que els commits fets en aquest estat no estan associats amb cap branca a menys que es moguin explícitament.

En dominar les referències i HEAD, tindràs una base sòlida per a operacions i fluxos de treball més avançats amb Git.
