Visió general

En aquesta secció final, revisarem els conceptes i comandaments clau tractats al llarg del curs. Això assegurarà que tingueu una comprensió sòlida de Git des de nivells bàsics fins a avançats. També tractarem les millors pràctiques per enviar el vostre projecte final o repositori de codi.

Revisió de conceptes clau

Comandaments bàsics de Git

  • git init: Inicialitzar un nou repositori Git.
  • git clone: Clonar un repositori existent.
  • git status: Comprovar l'estat del vostre directori de treball i àrea d'etapa.
  • git add: Establir canvis per al següent commit.
  • git commit: Commetre els canvis establerts al repositori.
  • git push: Empènyer commits locals a un repositori remot.
  • git pull: Recuperar i fusionar canvis d'un repositori remot.
# Inicialitzar un nou repositori Git
git init

# Clonar un repositori existent
git clone https://github.com/user/repo.git

# Comprovar l'estat del vostre directori de treball
git status

# Establir un fitxer per al commit
git add filename.txt

# Commetre els canvis establerts amb un missatge
git commit -m "Commit inicial"

# Empènyer commits locals al repositori remot
git push origin main

# Recuperar i fusionar canvis del repositori remot
git pull origin main

Branching i Merging

  • git branch: Llistar, crear o eliminar branques.
  • git checkout: Canviar de branca o restaurar fitxers del directori de treball.
  • git merge: Fusionar branques a la branca actual.
# Llistar totes les branques
git branch

# Crear una nova branca
git branch feature-branch

# Canviar a la nova branca
git checkout feature-branch

# Fusionar feature-branch a main
git checkout main
git merge feature-branch

Comandaments avançats de Git

  • git rebase: Reaplicar commits a sobre d'una altra punta base.
  • git stash: Guardar canvis en un directori de treball brut.
  • git cherry-pick: Aplicar els canvis introduïts per alguns commits existents.
# Reaplicar la branca actual a sobre de main
git checkout feature-branch
git rebase main

# Guardar canvis
git stash

# Aplicar canvis guardats
git stash apply

# Aplicar un commit
git cherry-pick commit_hash

Col·laboració i flux de treball

  • Forking: Crear una còpia personal del projecte d'una altra persona.
  • Pull Requests: Proposar canvis a un repositori.
  • Revisions de codi: Revisar i discutir canvis de codi.

Millors pràctiques per a l'enviament

  • Historial de commits net: Assegureu-vos que el vostre historial de commits sigui net i significatiu. Utilitzeu git rebase per combinar commits innecessaris.
  • Missatges de commit descriptius: Escriviu missatges de commit clars i descriptius.
  • Convencions de noms de branques: Utilitzeu noms de branques consistents i descriptius.
  • Qualitat del codi: Assegureu-vos que el vostre codi segueixi els estàndards i les directrius del projecte.
  • Documentació: Incloeu la documentació necessària i comentaris en el vostre codi.

Exemple de flux de treball d'enviament

  1. Fork del repositori: Creeu una còpia personal del repositori al qual voleu contribuir.
  2. Clonar el fork: Cloneu el vostre repositori forkejat a la vostra màquina local.
  3. Crear una branca: Creeu una nova branca per a la vostra característica o correcció d'errors.
  4. Fer canvis: Implementar els vostres canvis i commetre'ls amb missatges descriptius.
  5. Empènyer a remot: Empènyer la vostra branca al vostre repositori forkejat.
  6. Crear una Pull Request: Obriu una pull request al repositori original.
# Fork del repositori a GitHub

# Clonar el vostre repositori forkejat
git clone https://github.com/your-username/repo.git

# Navegar al directori del repositori
cd repo

# Crear una nova branca
git checkout -b feature-branch

# Fer canvis i commetre'ls
git add .
git commit -m "Afegir nova característica"

# Empènyer la branca al vostre repositori forkejat
git push origin feature-branch

# Obrir una pull request a GitHub
