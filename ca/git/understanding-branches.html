Introducció a les branques

Les branques a Git són una característica poderosa que et permeten divergir de la línia principal de desenvolupament i continuar treballant sense afectar aquesta línia principal. Això és particularment útil per treballar en noves característiques, correccions d'errors o experiments en aïllament.

Conceptes clau

  • Branca: Una branca a Git és simplement un punter mòbil lleuger a un d'aquests commits.
  • Master/Main: La branca per defecte en la majoria de repositoris. Sovint és la línia principal de desenvolupament.
  • HEAD: Un punter que representa el teu directori de treball actual. Normalment apunta al commit més recent de la teva branca actual.

Creació i gestió de branques

Creació d'una branca

Per crear una nova branca, utilitza el comandament git branch seguit del nom de la branca que vols crear.

git branch new-feature

Aquest comandament crea una nova branca anomenada new-feature però no hi canvia. Per canviar a la nova branca, utilitza el comandament git checkout:

git checkout new-feature

Alternativament, pots crear i canviar a una nova branca en un sol comandament:

git checkout -b new-feature

Llistat de branques

Per llistar totes les branques del teu repositori, utilitza el comandament git branch sense cap argument:

git branch

Això mostrarà totes les branques i destacarà la branca actual amb un asterisc (*).

Eliminació d'una branca

Un cop hagis acabat de treballar en una branca, potser voldràs eliminar-la. Per eliminar una branca, utilitza l'opció -d amb el comandament git branch:

git branch -d new-feature

Si la branca no ha estat fusionada, Git t'impedirà eliminar-la. Per forçar l'eliminació d'una branca, utilitza l'opció -D:

git branch -D new-feature

Fusió de branques

Fusió Fast-Forward

Una fusió fast-forward es produeix quan hi ha un camí directe des de la branca actual fins a la branca objectiu. Git simplement mou el punter cap endavant.

git checkout main
git merge new-feature

Fusió de tres vies

Quan les branques han divergit, Git realitza una fusió de tres vies. Això implica crear un nou commit que combina els canvis de les dues branques.

git checkout main
git merge new-feature

Si hi ha conflictes, Git et demanarà que els resolguis manualment.

Estratègies de branques

Flux de treball de branques de característiques

  • Crea una nova branca per a cada característica o correcció d'errors.
  • Fusiona la branca de característiques de nou a la branca principal un cop el treball estigui complet.

Flux de treball Gitflow

  • Utilitza múltiples branques per a diferents etapes de desenvolupament (per exemple, característica, desenvolupament, llançament, correcció ràpida).
  • Proporciona un marc robust per gestionar projectes més grans.

Flux de treball de forquilla

  • Cada desenvolupador forquilla el repositori principal i treballa en la seva pròpia còpia.
  • Els canvis s'integren de nou al repositori principal mitjançant sol·licituds de pull.

Exercicis pràctics

Exercici 1: Creació i canvi de branques

  1. Crea una nova branca anomenada experiment.
  2. Canvia a la branca experiment.
  3. Fes alguns canvis i fes un commit.
  4. Torna a la branca main.

Exercici 2: Fusió de branques

  1. Crea una nova branca anomenada feature-1.
  2. Fes alguns canvis i fes un commit.
  3. Torna a la branca main.
  4. Fusiona feature-1 a main.

Exercici 3: Resolució de conflictes de fusió

  1. Crea una nova branca anomenada conflict-branch.
  2. Fes canvis conflictius a main i conflict-branch.
  3. Intenta fusionar conflict-branch a main i resol els conflictes.

Resum

Comprendre les branques a Git és crucial per a un control de versions efectiu. Les branques et permeten treballar en diferents característiques, correccions d'errors o experiments en aïllament. Pots crear, canviar, llistar i eliminar branques fàcilment. La fusió de branques pot ser senzilla o complexa, depenent dels canvis realitzats. Dominant les estratègies de branques i els fluxos de treball, pots gestionar els teus projectes de manera més eficient i col·laborar amb altres sense problemes.
