Introducció

Els conflictes de fusió es produeixen quan Git no pot reconciliar automàticament les diferències en el codi entre dues branques. Aquesta secció et guiarà a través de la comprensió, identificació i resolució de conflictes de fusió.

Comprensió dels Conflictes de Fusió

  • Definició: Un conflicte de fusió es produeix quan els canvis de diferents branques no poden ser fusionats automàticament per Git.
  • Causes Comunes:
    • Canvis simultanis a la mateixa línia d'un fitxer.
    • Una branca elimina un fitxer mentre una altra branca el modifica.
    • Canvis en els permisos dels fitxers o canvi de nom dels fitxers.

Identificació dels Conflictes de Fusió

  • Durant una Fusió: Git et notificarà dels conflictes quan intentis fusionar branques.
  • Marcadors de Conflicte: Git utilitza marcadors de conflicte per ressaltar les seccions en conflicte en els fitxers.

Exemple de marcadors de conflicte:

<<<<<<< HEAD
// Canvis de la branca actual
=======
# Canvis de la branca que s'està fusionant
>>>>>>> branch-name

Resolució de Conflictes de Fusió

Procés Pas a Pas

  1. Identificar Fitxers en Conflicte:

    • Git llistarà els fitxers en conflicte després d'un intent de fusió.
    • Utilitza git status per veure quins fitxers estan en conflicte.

  2. Obrir Fitxers en Conflicte:

    • Obre cada fitxer en conflicte en un editor de text o un IDE.

  3. Resoldre Conflictes:

    • Edita manualment el fitxer per reconciliar les diferències.
    • Elimina els marcadors de conflicte (<<<<<<<, =======, >>>>>>>).
    • Assegura't que la versió final del fitxer sigui correcta i consistent.

  4. Marcar Conflictes com a Resolts:

    • Després de resoldre els conflictes, marca el fitxer com a resolt utilitzant git add <file>.

  5. Cometre la Fusió:

    • Completa el procés de fusió comprometent els canvis.
    git commit -m "Resolved merge conflicts"

Exemple

Suposem que tens un fitxer example.txt amb el següent conflicte:

<<<<<<< HEAD
This is some content from the current branch.
=======
This is some content from the branch being merged.
>>>>>>> feature-branch

Per resoldre aquest conflicte, podries editar el fitxer a:

This is some content from the current branch.
This is some content from the branch being merged.

Després, marca el fitxer com a resolt i compromet els canvis:

git add example.txt
git commit -m "Resolved merge conflict in example.txt"

Eines per Resoldre Conflictes

  • Línia de Comandes: La resolució bàsica de conflictes es pot fer directament a la línia de comandes.
  • Editors de Text: Editors com VSCode, Sublime Text i Atom proporcionen ajudes visuals per resoldre conflictes.
  • Eines de Fusió: Eines especialitzades com KDiff3, Meld i Beyond Compare ofereixen característiques avançades per resoldre conflictes.

Millors Pràctiques

  • Extraccions Freqüents: Extreu canvis de la branca principal regularment per minimitzar conflictes.
  • Commits Petits: Fes commits petits i freqüents per facilitar la resolució de conflictes.
  • Comunicació Clara: Coordina't amb els membres de l'equip per evitar canvis superposats.

Conclusió

Els conflictes de fusió són una part comuna del desenvolupament col·laboratiu. Comprendre com identificar i resoldre'ls eficientment és crucial per mantenir un flux de treball suau. Seguint els passos descrits i utilitzant les eines disponibles, pots gestionar i resoldre efectivament els conflictes de fusió en els teus projectes.
