Introducció

Els Pull Requests (PRs) són una part fonamental del desenvolupament de programari col·laboratiu utilitzant Git. Permeten als desenvolupadors proposar canvis a una base de codi, discutir aquests canvis amb els membres de l'equip i fusionar-los a la branca principal després de la revisió. Aquesta secció et guiarà a través del procés de creació i gestió de pull requests, des dels conceptes bàsics fins a pràctiques avançades.

Què és un Pull Request?

  • Un pull request és una sol·licitud per fusionar canvis de codi d'una branca a una altra.
  • Facilita la revisió de codi i la discussió abans que els canvis s'integrin a la base de codi principal.
  • Els PRs s'utilitzen comunament en entorns col·laboratius, especialment en projectes de codi obert.

Creació d'un Pull Request

Guia Pas a Pas

  1. Fes un Fork del Repositori (si és necessari)

    • Fer un fork és crear una còpia personal del projecte d'una altra persona.
    • Exemple: A GitHub, fes clic al botó "Fork" a la pàgina del repositori.

  2. Clona el Repositori

    git clone https://github.com/el-teu-usuari/nom-del-repositori.git
cd nom-del-repositori

  3. Crea una Nova Branca

    • Les branques et permeten treballar en canvis sense afectar la base de codi principal.
    git checkout -b branca-de-característiques

  4. Fes els Teus Canvis

    • Edita fitxers, afegeix noves característiques, corregeix errors, etc.
    echo "print('Hola, Món!')" > hola.py

  5. Fes un Commit dels Teus Canvis

    • Els missatges de commit han de ser clars i descriptius.
    git add hola.py
git commit -m "Afegeix script hola món"

  6. Puja els Canvis al Teu Fork

    git push origin branca-de-característiques

  7. Crea el Pull Request

    • Navega al repositori original a GitHub.
    • Fes clic al botó "New pull request".
    • Selecciona la teva branca i crea el pull request.

Exemple de Descripció de Pull Request

  • Títol: Afegeix Script Hola Món
  • Descripció: Aquest PR afegeix un simple script en Python que imprimeix "Hola, Món!" a la consola. Aquest és un exemple bàsic per demostrar la creació d'un pull request.

Gestió de Pull Requests

Revisió d'un Pull Request

  • Revisió de Codi:

    • Comprova la qualitat del codi, l'adhesió als estàndards de codificació i els possibles errors.
    • Exemple de comentari: "Considera utilitzar un nom de variable més descriptiu aquí."

  • Proves:

    • Assegura't que els canvis no trenquin la funcionalitat existent.
    • Executa proves automatitzades si estan disponibles.

Fusió d'un Pull Request

  • Squash and Merge:

    • Combina tots els commits en un sol commit.
    • Útil per mantenir l'historial de commits net.

  • Rebase and Merge:

    • Reaplica els commits a sobre de la branca base.
    • Manté un historial lineal.

  • Merge Commit:

    • Crea un commit de fusió.
    • Útil per preservar el context de les branques de característiques.

Gestió de Conflictes de Fusió

  • Identifica Conflictes:

    • Git et notificarà dels conflictes quan intentis fusionar.

  • Resol Conflictes:

    • Edita manualment els fitxers en conflicte per resoldre les diferències.
    git merge main
# Resol conflictes als fitxers
git add fitxer-resolt.py
git commit

  • Puja els Canvis Resolts:

    git push origin branca-de-característiques

Millors Pràctiques

  • Missatges de Commit Descriptius:

    • Ajuda a entendre la història dels canvis.

  • Pull Requests Petits i Focalitzats:

    • Més fàcils de revisar i menys propensos a causar conflictes.

  • Comunicació Regular:

    • Interactua amb els revisors i aborda els seus comentaris ràpidament.

Conclusió

Crear i gestionar pull requests és una habilitat crucial per al desenvolupament col·laboratiu. Seguint els passos descrits en aquesta secció, podràs contribuir efectivament a projectes, assegurar la qualitat del codi a través de revisions i integrar canvis de manera suau. Recorda seguir les millors pràctiques per mantenir una base de codi neta i manejable.
