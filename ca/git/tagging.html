Introducció a l'Etiquetatge

L'etiquetatge és una característica de Git que et permet marcar punts específics en la història del teu repositori com a importants. Normalment, la gent utilitza aquesta funcionalitat per marcar punts de llançament (per exemple, v1.0.0, v2.0.0).

Conceptes Clau

  • Etiquetes Lleugeres: Punteres simples a un commit específic.
  • Etiquetes Anotades: Objectes complets a la base de dades de Git, que contenen un nom de l'etiquetador, correu electrònic, data i un missatge.

Creació d'Etiquetes

Etiquetes Lleugeres

Les etiquetes lleugeres són ràpides i senzilles de crear. Són només punteres a un commit específic.

git tag v1.0.0

Aquesta comanda etiqueta el commit actual amb el nom v1.0.0.

Etiquetes Anotades

Les etiquetes anotades són recomanades perquè emmagatzemen metadades addicionals sobre l'etiqueta.

git tag -a v1.0.0 -m "Release version 1.0.0"
  • -a: Crea una etiqueta anotada.
  • -m: Afegeix un missatge a l'etiqueta.

Visualització d'Etiquetes

Per llistar totes les etiquetes al teu repositori, utilitza:

git tag

Per veure detalls sobre una etiqueta específica:

git show v1.0.0

Etiquetatge d'un Commit Específic

Pots etiquetar un commit específic proporcionant el hash del commit:

git tag -a v1.0.0 <commit-hash> -m "Release version 1.0.0"

Compartir Etiquetes

Per defecte, les etiquetes no es pugen als repositoris remots. Per pujar etiquetes, has d'especificar-les explícitament:

Pujar una Sola Etiqueta

git push origin v1.0.0

Pujar Totes les Etiquetes

git push origin --tags

Esborrar Etiquetes

Esborrar una Etiqueta Local

git tag -d v1.0.0

Esborrar una Etiqueta Remota

Primer, esborra l'etiqueta localment:

git tag -d v1.0.0

Després, esborra l'etiqueta del repositori remot:

git push origin :refs/tags/v1.0.0

Exemples Pràctics

Exemple 1: Crear i Pujar una Etiqueta Anotada

  1. Crear una etiqueta anotada: 
    git tag -a v1.1.0 -m "Release version 1.1.0"
  2. Pujar l'etiqueta al repositori remot: 
    git push origin v1.1.0

Exemple 2: Etiquetar un Commit Específic i Pujar Totes les Etiquetes

  1. Etiquetar un commit específic: 
    git tag -a v2.0.0 9fceb02 -m "Release version 2.0.0"
  2. Pujar totes les etiquetes al repositori remot: 
    git push origin --tags

Resum

L'etiquetatge a Git és una manera poderosa de marcar punts importants en la història del teu projecte. Tant si utilitzes etiquetes lleugeres com anotades, entendre com crear, veure, compartir i esborrar etiquetes és essencial per a un control de versions efectiu. Dominant aquests conceptes, podràs gestionar millor els llançaments i fites del teu projecte.
