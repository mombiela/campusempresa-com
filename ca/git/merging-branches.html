Fusionar branques és un concepte fonamental en Git que et permet combinar canvis de diferents branques en una sola branca. Això és essencial per al desenvolupament col·laboratiu i la gestió de diferents característiques o versions d'un projecte.

Conceptes Clau

  • Branca: Una versió paral·lela d'un repositori. Les branques et permeten treballar en diferents característiques o correccions de manera independent.
  • Fusió: El procés d'integrar canvis d'una branca a una altra.
  • Conflicte: Es produeix quan els canvis en diferents branques interfereixen entre si i Git no pot resoldre'ls automàticament.

Tipus de Fusions

  1. Fusió Fast-Forward:

    • Es produeix quan la branca que es fusiona no ha divergit de la branca objectiu.
    • Git simplement mou el punter cap endavant.

  2. Fusió de Tres Vies:

    • S'utilitza quan les branques han divergit.
    • Git utilitza l'ancestre comú i les dues puntes de les branques per crear un nou commit de fusió.

Fusió Fast-Forward

Exemple

Suposem que tens una branca feature que vols fusionar amb main.

  1. Canvia a la branca main:

    git checkout main

  2. Fusiona la branca feature:

    git merge feature

Si main no ha divergit de feature, Git realitzarà una fusió fast-forward.

Explicació

En una fusió fast-forward, Git simplement mou el punter de la branca main cap endavant fins al commit més recent de la branca feature. Aquest és el tipus de fusió més senzill i no crea un nou commit.

Fusió de Tres Vies

Exemple

Suposem que tens una branca feature que ha divergit de main.

  1. Canvia a la branca main:

    git checkout main

  2. Fusiona la branca feature:

    git merge feature

Si main ha divergit de feature, Git realitzarà una fusió de tres vies.

Explicació

En una fusió de tres vies, Git utilitza l'ancestre comú de les dues branques i les puntes de les dues branques per crear un nou commit de fusió. Aquest nou commit representa els canvis combinats de les dues branques.

Gestió de Conflictes de Fusió

Exemple

Suposem que tens un conflicte en fusionar feature amb main.

  1. Git et notificarà del conflicte:

    Auto-merging file.txt
CONFLICT (content): Merge conflict in file.txt
Automatic merge failed; fix conflicts and then commit the result.

  2. Obre el fitxer conflictiu i resol el conflicte:

    <<<<<<< HEAD
Aquest és un contingut de la branca main.
=======
Aquest és un contingut de la branca feature.
>>>>>>> feature

  3. Després de resoldre el conflicte, afegeix el fitxer resolt:

    git add file.txt

  4. Completa la fusió fent un commit:

    git commit

Explicació

Quan es produeix un conflicte, Git marca les àrees conflictuoses en el fitxer. Has d'editar manualment el fitxer per resoldre el conflicte, després afegir i fer commit del fitxer resolt per completar la fusió.

Resum

Fusionar branques en Git és una habilitat crucial per gestionar canvis en un entorn col·laboratiu. Entendre la diferència entre fusions fast-forward i de tres vies, així com com gestionar conflictes de fusió, t'ajudarà a mantenir una base de codi neta i funcional. Practica la fusió de branques i la resolució de conflictes per convertir-te en un expert en aquestes operacions essencials de Git.
