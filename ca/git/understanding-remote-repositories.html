Introducció als Repositoris Remots

Els repositoris remots són versions del teu projecte que estan allotjades a internet o en una altra xarxa. Són essencials per a la col·laboració, la còpia de seguretat i els propòsits de desplegament.

  • Definició: Un repositori remot és un repositori Git que està allotjat en un servidor remot.
  • Propòsit: Permet que múltiples usuaris col·laborin en el mateix projecte, comparteixin canvis i mantinguin una font central de veritat.

Configuració d'un Repositori Remot

Per començar a treballar amb repositoris remots, necessites saber com configurar-los i connectar el teu repositori local a un de remot.

Crear un Repositori Remot

  1. GitHub/GitLab/Bitbucket: Plataformes on pots crear repositoris remots.
  2. Passos per Crear:
    • Inicia sessió al teu compte a la plataforma.
    • Fes clic al botó "New Repository".
    • Omple el nom del repositori, la descripció i la configuració de visibilitat.
    • Fes clic a "Create Repository".

Connectar-se a un Repositori Remot

Un cop tens un repositori remot, necessites enllaçar-lo al teu repositori local.

  • Comanda: git remote add <remote_name> <remote_url>
  • Exemple: 
    git remote add origin https://github.com/username/repository.git
    Aquesta comanda afegeix un repositori remot anomenat origin amb l'URL especificada.

Treballar amb Repositoris Remots

Després de configurar-ho, pots realitzar diverses operacions per interactuar amb el repositori remot.

Empènyer Canvis

Per enviar els teus commits locals al repositori remot:

  • Comanda: git push <remote_name> <branch_name>
  • Exemple: 
    git push origin main
    Aquesta comanda empeny la branca main al remot origin.

Estirar Canvis

Per obtenir i integrar els canvis del repositori remot al teu repositori local:

  • Comanda: git pull <remote_name> <branch_name>
  • Exemple: 
    git pull origin main
    Aquesta comanda estira els últims canvis de la branca main del remot origin.

Obtenir Canvis

Per obtenir canvis del repositori remot sense fusionar-los:

  • Comanda: git fetch <remote_name>
  • Exemple: 
    git fetch origin
    Aquesta comanda obté totes les branques del remot origin.

Gestionar Repositoris Remots

Pots gestionar els teus repositoris remots visualitzant-los, canviant-los el nom o eliminant-los.

Visualitzar Repositoris Remots

Per llistar tots els repositoris remots enllaçats al teu repositori local:

  • Comanda: git remote -v
  • Exemple: 
    git remote -v
    Aquesta comanda mostra les URLs de tots els remots.

Canviar el Nom d'un Repositori Remot

Per canviar el nom d'un repositori remot:

  • Comanda: git remote rename <old_name> <new_name>
  • Exemple: 
    git remote rename origin upstream
    Aquesta comanda canvia el nom del remot origin a upstream.

Eliminar un Repositori Remot

Per eliminar un repositori remot:

  • Comanda: git remote remove <remote_name>
  • Exemple: 
    git remote remove origin
    Aquesta comanda elimina el remot anomenat origin.

Resum

Comprendre els repositoris remots és crucial per a una col·laboració i gestió de projectes efectiva en Git. Configurant, connectant i gestionant repositoris remots, pots assegurar-te que la teva feina estigui sincronitzada i accessible per al teu equip. Les operacions clau inclouen empènyer, estirar i obtenir canvis, que ajuden a mantenir la consistència entre els repositoris locals i remots.
