Introducció a les Proves i la Depuració

Les proves i la depuració són habilitats crucials en qualsevol llenguatge de programació, incloent Prolog. Aquesta secció cobrirà les bases d'escriure proves, tècniques de depuració i eines disponibles per a Prolog.

Escriure Proves en Prolog

Provar en Prolog implica escriure casos de prova que comprovin la correcció dels teus predicats. Hi ha diverses maneres d'escriure proves en Prolog:

  • Proves Manuals: Escriure consultes i comprovar els seus resultats manualment.
  • Proves Automatitzades: Utilitzar biblioteques com plunit per automatitzar el procés de prova.

Proves Manuals

Les proves manuals impliquen executar consultes en l'intèrpret de Prolog i verificar els resultats a mà. Aquest mètode és senzill però pot ser llarg per a programes grans.

% Exemple de predicat per provar
factorial(0, 1).
factorial(N, F) :-
    N > 0,
    N1 is N - 1,
    factorial(N1, F1),
    F is N * F1.

% Casos de prova manuals
?- factorial(0, F).
% Esperat: F = 1

?- factorial(5, F).
% Esperat: F = 120

Proves Automatitzades amb `plunit`

plunit és un marc de proves unitàries per a Prolog que et permet escriure i executar proves automàticament.

:- begin_tests(factorial_tests).

test(factorial_0) :-
    factorial(0, F),
    assertion(F == 1).

test(factorial_5) :-
    factorial(5, F),
    assertion(F == 120).

:- end_tests(factorial_tests).

% Per executar les proves, utilitza la següent consulta:
% ?- run_tests.

Tècniques de Depuració

Depurar en Prolog pot ser un repte degut a la seva naturalesa declarativa. Aquí hi ha algunes tècniques comunes:

  • Traçat: Utilitzar el traçador integrat per seguir l'execució del teu programa.
  • Sentències de Print: Inserir sentències de print per observar els valors de les variables i el flux del programa.
  • Depurador: Utilitzar el depurador de Prolog per a una depuració més avançada.

Traçat

El traçador et permet seguir el teu codi i veure com Prolog avalua les teves consultes.

% Activar el traçat
?- trace.

% Executar una consulta
?- factorial(3, F).
% Veureu una execució pas a pas del predicat factorial.

Sentències de Print

Les sentències de print poden ajudar-te a entendre què està fent el teu programa en cada pas.

factorial(0, 1) :- writeln('Cas base: 0! = 1').
factorial(N, F) :-
    N > 0,
    N1 is N - 1,
    factorial(N1, F1),
    F is N * F1,
    format('Calculat ~w! = ~w~n', [N, F]).

Utilitzar el Depurador

El depurador de Prolog proporciona una manera més sofisticada de depurar els teus programes. Pots establir punts de ruptura, inspeccionar valors de variables i controlar el flux d'execució.

% Iniciar el depurador
?- gtrace.

% Executar una consulta
?- factorial(3, F).
% El depurador gràfic s'obrirà, permetent-te seguir el codi pas a pas.

Escenaris Comuns de Depuració

Aquí hi ha alguns problemes comuns que podries trobar i com depurar-los:

  • Bucles Infinits: Utilitza el traçat per identificar on es produeix el bucle.
  • Resultats Incorrectes: Utilitza sentències de print per comprovar valors intermedis.
  • Fallades Inesperades: Utilitza el depurador per inspeccionar l'estat del teu programa en el punt de fallada.

Conclusió

Les proves i la depuració són habilitats essencials per escriure programes Prolog fiables. Utilitzant tècniques de proves manuals i automatitzades, així com aprofitant les eines de depuració de Prolog, pots assegurar-te que el teu codi és correcte i eficient. Practica aquestes tècniques regularment per convertir-te en un expert en identificar i solucionar problemes en els teus programes Prolog.
