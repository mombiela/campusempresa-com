Introducció a la Recursió

La recursió és un concepte fonamental en Prolog i moltes altres llenguatges de programació. Implica una funció que es crida a si mateixa per resoldre instàncies més petites del mateix problema. En Prolog, la recursió s'utilitza sovint per recórrer llistes, arbres i altres estructures de dades.

Conceptes Clau

  • Cas Base: La condició sota la qual les crides recursives s'aturen.
  • Cas Recursiu: La part de la funció que inclou la crida recursiva.

Recursió Bàsica

Comencem amb un exemple senzill: calcular el factorial d'un nombre.

Exemple de Factorial

El factorial d'un nombre \( n \) es defineix com:

  • \( 0! = 1 \) (Cas Base)
  • \( n! = n \times (n-1)! \) (Cas Recursiu)

Aquí teniu com podeu implementar això en Prolog:

% Cas Base
factorial(0, 1).

% Cas Recursiu
factorial(N, Result) :-
    N > 0,
    N1 is N - 1,
    factorial(N1, Result1),
    Result is N * Result1.

Explicació

  • El cas base factorial(0, 1) indica que el factorial de 0 és 1.
  • El cas recursiu factorial(N, Result) comprova si \( N \) és més gran que 0, després calcula \( N-1 \) i crida recursivament factorial amb \( N-1 \). Finalment, multiplica \( N \) pel resultat de la crida recursiva.

Recursió amb Llistes

Prolog destaca en la gestió de llistes mitjançant la recursió. Vegem un exemple de sumar tots els elements d'una llista.

Suma dels Elements d'una Llista

Per sumar els elements d'una llista, definim:

  • La suma d'una llista buida és 0 (Cas Base).
  • La suma d'una llista és el cap de la llista més la suma de la cua (Cas Recursiu).
% Cas Base
sum_list([], 0).

% Cas Recursiu
sum_list([Head|Tail], Sum) :-
    sum_list(Tail, SumTail),
    Sum is Head + SumTail.

Explicació

  • El cas base sum_list([], 0) indica que la suma d'una llista buida és 0.
  • El cas recursiu sum_list([Head|Tail], Sum) divideix la llista en el seu cap i cua, suma recursivament la cua, i afegeix el cap a aquesta suma.

Recursió Avançada

Seqüència de Fibonacci

La seqüència de Fibonacci és un exemple clàssic de recursió, on cada nombre és la suma dels dos precedents.

% Casos Base
fibonacci(0, 0).
fibonacci(1, 1).

% Cas Recursiu
fibonacci(N, Result) :-
    N > 1,
    N1 is N - 1,
    N2 is N - 2,
    fibonacci(N1, Result1),
    fibonacci(N2, Result2),
    Result is Result1 + Result2.

Explicació

  • Els casos base fibonacci(0, 0) i fibonacci(1, 1) defineixen els dos primers nombres de la seqüència.
  • El cas recursiu fibonacci(N, Result) calcula els nombres de Fibonacci \( N-1 \) i \( N-2 \) i els suma.

Optimització de la Recursió amb Acumuladors

La recursió es pot optimitzar utilitzant acumuladors per evitar càlculs redundants. Optimitzem l'exemple de Fibonacci.

% Predicat auxiliar amb acumulador
fibonacci_acc(N, Result) :-
    fibonacci_acc(N, 0, 1, Result).

% Cas Base
fibonacci_acc(0, A, _, A).

% Cas Recursiu
fibonacci_acc(N, A, B, Result) :-
    N > 0,
    N1 is N - 1,
    Sum is A + B,
    fibonacci_acc(N1, B, Sum, Result).

Explicació

  • El predicat auxiliar fibonacci_acc/2 inicialitza els acumuladors.
  • El cas base fibonacci_acc(0, A, _, A) retorna l'acumulador quan \( N \) és 0.
  • El cas recursiu fibonacci_acc(N, A, B, Result) actualitza els acumuladors i es crida a si mateix amb \( N-1 \).

Conclusió

La recursió és una eina poderosa en Prolog per resoldre problemes que impliquen estructures repetitives com llistes i arbres. Comprenent els casos base i recursius, podeu implementar solucions recursives eficients. Optimitzar la recursió amb tècniques com els acumuladors pot millorar encara més el rendiment. Practiqueu aquests conceptes amb diversos problemes per dominar la recursió en Prolog.
