Introducció

Comprendre l'estructura d'un projecte Django és crucial tant per a principiants com per a usuaris avançats. Aquesta secció et guiarà a través dels diversos components d'un projecte Django, explicant-ne els propòsits i com interactuen entre ells.

Components Clau d'un Projecte Django

Quan crees un nou projecte Django, es generen diversos fitxers i directoris. Aquests són els components clau:

  • manage.py: Una utilitat de línia de comandes que et permet interactuar amb el teu projecte Django.
  • project_name/: El directori principal del projecte que conté fitxers de configuració i ajustaments.
  • app_name/: Un directori per a cada aplicació Django que crees dins del teu projecte.

manage.py

El fitxer manage.py és una utilitat de línia de comandes que et permet interactuar amb el teu projecte Django. Pots utilitzar-lo per executar el servidor de desenvolupament, crear noves aplicacions i realitzar migracions de bases de dades.

# manage.py
#!/usr/bin/env python
import os
import sys

if __name__ == "__main__":
    os.environ.setdefault("DJANGO_SETTINGS_MODULE", "project_name.settings")
    try:
        from django.core.management import execute_from_command_line
    except ImportError as exc:
        raise ImportError(
            "Couldn't import Django. Are you sure it's installed and "
            "available on your PYTHONPATH environment variable? Did you "
            "forget to activate a virtual environment?"
        ) from exc
    execute_from_command_line(sys.argv)

Directori project_name

Aquest directori conté els fitxers de configuració principals del teu projecte Django:

  • init.py: Un fitxer buit que indica que aquest directori ha de ser tractat com un paquet de Python.
  • settings.py: Conté tots els ajustaments i configuracions per al teu projecte.
  • urls.py: Defineix el ruteig de les URL per al teu projecte.
  • wsgi.py: Un punt d'entrada per a servidors web compatibles amb WSGI per servir el teu projecte.
  • asgi.py: Un punt d'entrada per a servidors web compatibles amb ASGI per servir el teu projecte (per suport asincrònic).

Directori app_name

Cada aplicació Django que crees tindrà el seu propi directori que conté els següents fitxers:

  • init.py: Un fitxer buit que indica que aquest directori ha de ser tractat com un paquet de Python.
  • admin.py: Utilitzat per registrar models amb el lloc d'administració de Django.
  • apps.py: Conté la configuració per a l'aplicació.
  • models.py: Defineix els models de dades per a l'aplicació.
  • tests.py: Conté proves per a l'aplicació.
  • views.py: Conté les vistes per a l'aplicació.
  • migrations/: Un directori que emmagatzema fitxers de migració de bases de dades.
# app_name/models.py
from django.db import models

class ExampleModel(models.Model):
    name = models.CharField(max_length=100)
    description = models.TextField()

    def __str__(self):
        return self.name
# app_name/views.py
from django.shortcuts import render
from .models import ExampleModel

def example_view(request):
    examples = ExampleModel.objects.all()
    return render(request, 'app_name/example_template.html', {'examples': examples})

Resum

Comprendre l'estructura d'un projecte Django és fonamental per a un desenvolupament eficient. Els components clau inclouen el fitxer manage.py, el directori principal del projecte (project_name) i els directoris d'aplicacions individuals (app_name). Cadascun d'aquests components té rols i responsabilitats específiques, des de la configuració i ajustaments fins a la definició de models i vistes.

En dominar l'estructura del projecte, estaràs millor equipat per desenvolupar, mantenir i escalar les teves aplicacions Django de manera efectiva.
