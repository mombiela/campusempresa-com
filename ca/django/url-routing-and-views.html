Introducció

En aquesta secció, explorarem com Django gestiona l'enrutament d'URL i les vistes. L'enrutament d'URL és el mecanisme pel qual Django associa una URL sol·licitada amb la funció de vista adequada. Les vistes són funcions de Python que prenen sol·licituds web i retornen respostes web. Entendre aquests conceptes és crucial per construir aplicacions web amb Django.

Enrutament d'URL

Conceptes Clau

  • URLconf: Un mòdul que associa patrons d'URL amb vistes.
  • Funció Path: Una funció utilitzada per definir patrons d'URL.
  • Expressions Regulars: Utilitzades per a una coincidència d'URL més complexa.

Enrutament d'URL Bàsic

En Django, l'enrutament d'URL es defineix en un fitxer urls.py. Aquí teniu un exemple senzill:

# mysite/urls.py
from django.contrib import admin
from django.urls import path
from myapp import views

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', views.home, name='home'),
    path('about/', views.about, name='about'),
]

Explicació

  • path('admin/', admin.site.urls): Enruta la URL /admin/ al lloc d'administració de Django.
  • path('', views.home, name='home'): Enruta la URL arrel (/) a la vista home a myapp/views.py.
  • path('about/', views.about, name='about'): Enruta la URL /about/ a la vista about a myapp/views.py.

Enrutament d'URL Avançat

Per a patrons d'URL més complexos, podeu utilitzar expressions regulars amb la funció re_path:

# mysite/urls.py
from django.urls import re_path
from myapp import views

urlpatterns = [
    re_path(r'^articles/(?P<year>[0-9]{4})/$', views.year_archive),
]

Explicació

  • re_path(r'^articles/(?P<year>[0-9]{4})/$', views.year_archive): Enruta URL com /articles/2023/ a la vista year_archive, capturant l'any com a paràmetre.

Vistes

Conceptes Clau

  • Vistes Basades en Funcions (FBVs): Funcions senzilles de Python que gestionen sol·licituds.
  • Vistes Basades en Classes (CBVs): Vistes que es representen com a classes, proporcionant més estructura i reutilització.
  • HttpResponse: L'objecte de resposta retornat per les vistes.

Vistes Basades en Funcions

Aquí teniu una vista basada en funcions senzilla:

# myapp/views.py
from django.http import HttpResponse

def home(request):
    return HttpResponse("Benvingut a la Pàgina Principal!")

Explicació

  • home(request): Una funció que pren una sol·licitud HTTP i retorna una resposta HTTP amb el text "Benvingut a la Pàgina Principal!".

Vistes Basades en Classes

Les vistes basades en classes proporcionen més funcionalitat i són més reutilitzables. Aquí teniu un exemple:

# myapp/views.py
from django.views import View
from django.http import HttpResponse

class HomeView(View):
    def get(self, request):
        return HttpResponse("Benvingut a la Pàgina Principal!")

Explicació

  • HomeView(View): Una classe que hereta de la classe View de Django.
  • get(self, request): Un mètode que gestiona sol·licituds GET i retorna una resposta HTTP.

Utilitzant Plantilles en Vistes

Les vistes sovint renderitzen plantilles per generar HTML. Aquí teniu com fer-ho:

# myapp/views.py
from django.shortcuts import render

def home(request):
    return render(request, 'home.html')

Explicació

  • render(request, 'home.html'): Renderitza la plantilla home.html i retorna l'HTML generat com a resposta.

Resum

En aquesta secció, hem cobert els conceptes bàsics de l'enrutament d'URL i les vistes en Django. Hem après com definir patrons d'URL utilitzant les funcions path i re_path, i com crear tant vistes basades en funcions com vistes basades en classes. També hem vist com renderitzar plantilles en vistes. Entendre aquests conceptes és essencial per construir aplicacions web dinàmiques amb Django.
