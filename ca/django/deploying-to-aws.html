Desplegar a AWS
Optimització del Rendiment ►

Desplegar una aplicació Django a AWS (Amazon Web Services) pot semblar intimidant al principi, però amb la guia adequada, es converteix en una tasca manejable i gratificant. Aquesta secció et guiarà a través del procés de desplegar la teva aplicació Django a AWS, cobrint tot, des de la configuració del teu compte AWS fins a la configuració de la teva aplicació Django per a producció.

Configurar el teu Compte AWS

Abans de poder desplegar la teva aplicació Django a AWS, necessites configurar un compte AWS.

  1. Registra't a AWS: Ves al lloc web d'AWS i registra't per obtenir un compte.
  2. Crea un Usuari IAM: Utilitza el servei de Gestió d'Identitat i Accés (IAM) per crear un usuari amb permisos administratius.
  3. Configura Alertes de Facturació: Configura alertes de facturació per monitoritzar el teu ús d'AWS i evitar càrrecs inesperats.

Triar els Serveis AWS Adequats

AWS ofereix una varietat de serveis que es poden utilitzar per desplegar una aplicació Django. Els serveis més utilitzats per a aquest propòsit són:

  • EC2 (Elastic Compute Cloud): Servidors virtuals al núvol.
  • RDS (Relational Database Service): Servei de bases de dades relacionals gestionat.
  • S3 (Simple Storage Service): Emmagatzematge escalable per a fitxers estàtics.
  • Elastic Beanstalk: Plataforma com a servei (PaaS) per desplegar i gestionar aplicacions.

Desplegar amb Elastic Beanstalk

Elastic Beanstalk és una de les maneres més fàcils de desplegar una aplicació Django a AWS. Gestiona el desplegament, des de la provisió de capacitat, l'equilibri de càrrega i l'autoescalat fins al monitoratge de la salut de l'aplicació.

Guia Pas a Pas per Desplegar amb Elastic Beanstalk

  1. Instal·la l'EB CLI: La Interfície de Línia de Comandes d'Elastic Beanstalk (EB CLI) simplifica el procés de crear, actualitzar i gestionar entorns d'Elastic Beanstalk.

    pip install awsebcli

  2. Inicialitza la teva Aplicació: Navega al directori del teu projecte Django i inicialitza Elastic Beanstalk.

    eb init

  3. Crea un Entorn i Desplega: Crea un entorn i desplega la teva aplicació.

    eb create my-django-env
eb deploy

  4. Configura Variables d'Entorn: Configura variables d'entorn com DJANGO_SETTINGS_MODULE i SECRET_KEY.

    eb setenv DJANGO_SETTINGS_MODULE=myproject.settings.production SECRET_KEY=your_secret_key

Exemple de Configuració

Aquí tens un exemple de com podries configurar el teu settings.py per a producció:

import os

DEBUG = False
ALLOWED_HOSTS = ['your-elasticbeanstalk-url.amazonaws.com']

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': os.environ.get('RDS_DB_NAME'),
        'USER': os.environ.get('RDS_USERNAME'),
        'PASSWORD': os.environ.get('RDS_PASSWORD'),
        'HOST': os.environ.get('RDS_HOSTNAME'),
        'PORT': os.environ.get('RDS_PORT'),
    }
}

STATIC_URL = '/static/'
STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')

Utilitzar EC2 i RDS

Per tenir més control sobre el teu entorn de desplegament, pots utilitzar EC2 i RDS.

Configurar EC2

  1. Llança una Instància EC2: Tria una Imatge de Màquina d'Amazon (AMI) i el tipus d'instància.

  2. SSH a la teva Instància: Utilitza SSH per connectar-te a la teva instància.

    ssh -i your-key-pair.pem ec2-user@your-ec2-instance-public-dns

  3. Instal·la Dependències: Instal·la Python, Django i altres dependències.

    sudo yum update -y
sudo yum install python3-pip
pip3 install django

  4. Desplega la teva Aplicació: Clona el teu projecte Django des d'un sistema de control de versions com Git i configura'l.

Configurar RDS

  1. Crea una Instància RDS: Tria el motor de base de dades i configura la instància.
  2. Configura Django per Utilitzar RDS: Actualitza el teu settings.py per utilitzar la base de dades RDS.
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'your-db-name',
        'USER': 'your-db-user',
        'PASSWORD': 'your-db-password',
        'HOST': 'your-db-host',
        'PORT': 'your-db-port',
    }
}

Emmagatzemar Fitxers Estàtics a S3

Per servir fitxers estàtics, pots utilitzar AWS S3.

  1. Crea un Bucket S3: Ves al servei S3 i crea un nou bucket.
  2. Configura Django per Utilitzar S3: Utilitza el paquet django-storages per configurar Django per utilitzar S3 per als fitxers estàtics.
pip install django-storages boto3

Actualitza el teu settings.py:

INSTALLED_APPS += ['storages']

AWS_ACCESS_KEY_ID = 'your-access-key-id'
AWS_SECRET_ACCESS_KEY = 'your-secret-access-key'
AWS_STORAGE_BUCKET_NAME = 'your-bucket-name'

STATICFILES_STORAGE = 'storages.backends.s3boto3.S3Boto3Storage'
STATIC_URL = f'https://{AWS_STORAGE_BUCKET_NAME}.s3.amazonaws.com/'

Conclusió

Desplegar una aplicació Django a AWS implica diversos passos, des de configurar el teu compte AWS fins a configurar la teva aplicació per a producció. Tant si tries utilitzar Elastic Beanstalk per a un procés de desplegament simplificat com EC2 i RDS per a més control, AWS proporciona les eines i serveis que necessites per desplegar i escalar la teva aplicació. Seguint aquesta guia, hauràs de poder desplegar la teva aplicació Django a AWS amb confiança.
