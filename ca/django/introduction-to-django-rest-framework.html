Introducció al Django REST Framework
Serialitzadors ►

Visió general

En aquesta secció, introduirem el Django REST Framework (DRF), un conjunt d'eines potent i flexible per construir APIs web en Django. Al final d'aquesta secció, entendràs els conceptes bàsics del DRF, els seus components clau i com crear una API senzilla.

Conceptes clau

  • API web: Una interfície que permet que diferents sistemes de programari es comuniquin entre ells a través de la web.
  • Django REST Framework (DRF): Un conjunt d'eines per construir APIs web en Django, que proporciona característiques com la serialització, l'autenticació i els viewsets.

Configuració del Django REST Framework

Per començar amb el DRF, cal instal·lar-lo en el teu projecte Django. Segueix aquests passos:

  1. Instal·la el DRF:

    pip install djangorestframework

  2. Afegeix el DRF a les Aplicacions Instal·lades: Obre el teu fitxer settings.py i afegeix 'rest_framework' a la llista INSTALLED_APPS:

    INSTALLED_APPS = [
    ...
    'rest_framework',
]

Creació d'una API senzilla

Creem una API senzilla per gestionar una llista de llibres.

Pas 1: Crea una Aplicació Django

Primer, crea una nova aplicació Django anomenada books:

python manage.py startapp books

Pas 2: Defineix el Model Book

En books/models.py, defineix un model Book senzill:

from django.db import models

class Book(models.Model):
    title = models.CharField(max_length=100)
    author = models.CharField(max_length=100)
    published_date = models.DateField()

    def __str__(self):
        return self.title

Pas 3: Crea un Serializer

Els serializers en DRF s'utilitzen per convertir tipus de dades complexos, com els models de Django, en JSON i viceversa. Crea un serializer per al model Book en books/serializers.py:

from rest_framework import serializers
from .models import Book

class BookSerializer(serializers.ModelSerializer):
    class Meta:
        model = Book
        fields = '__all__'

Pas 4: Crea Vistes

En books/views.py, crea vistes per gestionar les sol·licituds de l'API. El DRF proporciona diverses classes de vistes, però començarem amb APIView:

from rest_framework.views import APIView
from rest_framework.response import Response
from rest_framework import status
from .models import Book
from .serializers import BookSerializer

class BookList(APIView):
    def get(self, request):
        books = Book.objects.all()
        serializer = BookSerializer(books, many=True)
        return Response(serializer.data)

    def post(self, request):
        serializer = BookSerializer(data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

Pas 5: Configura les URLs

En books/urls.py, defineix els patrons d'URL per a l'API:

from django.urls import path
from .views import BookList

urlpatterns = [
    path('books/', BookList.as_view(), name='book-list'),
]

Inclou les URLs de books en el urls.py principal del projecte:

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('api/', include('books.urls')),
]

Prova de l'API

Executa el servidor de desenvolupament de Django:

python manage.py runserver

Ara pots provar els punts finals de l'API utilitzant eines com Postman o curl:

  • Sol·licitud GET a /api/books/ per recuperar la llista de llibres.
  • Sol·licitud POST a /api/books/ per afegir un nou llibre.

Conclusió

En aquesta secció, hem cobert els conceptes bàsics del Django REST Framework, incloent la seva configuració, la creació d'una API senzilla i la seva prova. El DRF simplifica el procés de construir APIs robustes i escalables en Django, convertint-lo en una eina essencial per al desenvolupament web modern. En les properes seccions, aprofundirem en característiques més avançades del DRF, com els viewsets, els routers i l'autenticació.
