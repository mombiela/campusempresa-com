Introducció

La seguretat és un aspecte crític del desenvolupament web, i Django proporciona diverses característiques integrades per ajudar els desenvolupadors a crear aplicacions segures. Aquesta secció cobrirà les millors pràctiques per assegurar la teva aplicació Django, des de nivells bàsics fins a avançats.

Pràctiques Bàsiques de Seguretat

Utilitza l'Última Versió de Django

  • Utilitza sempre la darrera versió estable de Django per assegurar-te que tens els últims pegats de seguretat i característiques.
  • Revisa regularment les notes de llançament de Django per a qualsevol actualització de seguretat.

Configuracions Segures

  • SECRET_KEY: Mantingues el teu SECRET_KEY secret i no l'exposis mai en el teu repositori de codi font. 
    import os
SECRET_KEY = os.environ.get('DJANGO_SECRET_KEY', 'your-default-secret-key')
  • DEBUG: Configura DEBUG a False en producció per evitar exposar informació sensible. 
    DEBUG = False

Hosts Permesos

  • Defineix ALLOWED_HOSTS per prevenir atacs de capçalera HTTP Host. 
    ALLOWED_HOSTS = ['yourdomain.com', 'www.yourdomain.com']

Configuració Segura de la Base de Dades

  • Utilitza variables d'entorn per emmagatzemar les credencials de la base de dades. 
    DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': os.environ.get('DB_NAME'),
        'USER': os.environ.get('DB_USER'),
        'PASSWORD': os.environ.get('DB_PASSWORD'),
        'HOST': os.environ.get('DB_HOST'),
        'PORT': os.environ.get('DB_PORT'),
    }
}

Pràctiques Intermèdies de Seguretat

HTTPS

  • Utilitza HTTPS per xifrar les dades en trànsit.
  • Configura SECURE_SSL_REDIRECT a True per redirigir totes les sol·licituds HTTP a HTTPS. 
    SECURE_SSL_REDIRECT = True

Política de Seguretat de Contingut (CSP)

  • Implementa CSP per prevenir atacs de Cross-Site Scripting (XSS). 
    CSP_DEFAULT_SRC = ("'self'",)
CSP_STYLE_SRC = ("'self'", 'https://fonts.googleapis.com')
CSP_SCRIPT_SRC = ("'self'", 'https://ajax.googleapis.com')

Protecció contra Cross-Site Request Forgery (CSRF)

  • Django té protecció CSRF integrada per defecte. Assegura't d'incloure {% csrf_token %} en els teus formularis. 
    <form method="post">
    {% csrf_token %}
    <!-- camps del formulari -->
</form>

Cookies Segures

  • Configura SESSION_COOKIE_SECURE i CSRF_COOKIE_SECURE a True per assegurar que les cookies només s'envien per HTTPS. 
    SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True

Pràctiques Avançades de Seguretat

Autenticació i Autorització d'Usuaris

  • Utilitza el sistema d'autenticació integrat de Django i considera utilitzar paquets de tercers com django-allauth per a característiques més avançades.
  • Implementa l'Autenticació de Doble Factor (2FA) per a més seguretat.

Limitació de Taxa

  • Implementa la limitació de taxa per protegir contra atacs de força bruta.
  • Utilitza paquets de tercers com django-ratelimit.

Capçaleres de Seguretat

  • Afegeix capçaleres de seguretat a les teves respostes. 
    SECURE_BROWSER_XSS_FILTER = True
SECURE_CONTENT_TYPE_NOSNIFF = True
X_FRAME_OPTIONS = 'DENY'

Auditories Regulars de Seguretat

  • Audita regularment el teu codi i dependències per vulnerabilitats de seguretat.
  • Utilitza eines com Bandit per a l'anàlisi estàtica de codi i Safety per comprovar vulnerabilitats conegudes en les dependències.

Conclusió

Assegurar una aplicació Django requereix un enfocament de múltiples capes, des de configuracions bàsiques fins a pràctiques avançades. Seguint aquestes millors pràctiques, pots reduir significativament el risc de vulnerabilitats de seguretat en la teva aplicació. Mantingues-te sempre actualitzat amb les últimes tendències de seguretat i monitora i audita contínuament la teva aplicació per a possibles amenaces.
