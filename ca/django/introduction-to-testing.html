Proves Unitàries ►

Les proves són una part crucial del desenvolupament de programari, assegurant que el teu codi funcioni com s'espera i ajudant a detectar errors aviat. En aquesta secció, cobrirem els conceptes bàsics de les proves en Django, des d'escriure la teva primera prova fins a entendre conceptes de proves més avançats.

Per què són importants les proves

  • Detectar errors aviat: Les proves ajuden a identificar errors abans que arribin a producció.
  • Assegurar la qualitat del codi: Les proves regulars asseguren que la teva base de codi es mantingui neta i fàcil de mantenir.
  • Facilitar la refactorització: Les proves proporcionen una xarxa de seguretat quan es fan canvis al codi.
  • Millorar la col·laboració: Les proves faciliten el treball en equip, sabent que els canvis tenen menys probabilitats de trencar la funcionalitat existent.

Tipus de proves

  • Proves unitàries: Prova unitats individuals de codi, com ara funcions o mètodes.
  • Proves d'integració: Prova la interacció entre diferents parts de l'aplicació.
  • Proves funcionals: Prova l'aplicació des de la perspectiva de l'usuari, assegurant que es comporti com s'espera.
  • Proves de punta a punta: Prova el flux complet de l'aplicació, des del frontend fins al backend.

Configuració de les proves en Django

Django ve amb un marc de proves integrat basat en el mòdul unittest de Python. Per començar, necessites crear casos de prova subclassificant django.test.TestCase.

Escriure la teva primera prova

Escrivim una prova senzilla per a un model de Django.

  1. Crear un model de Django:

    # models.py
from django.db import models

class Item(models.Model):
    name = models.CharField(max_length=100)
    description = models.TextField()

    def __str__(self):
        return self.name

  2. Escriure un cas de prova:

    # tests.py
from django.test import TestCase
from .models import Item

class ItemModelTest(TestCase):
    def test_string_representation(self):
        item = Item(name="Test Item")
        self.assertEqual(str(item), item.name)

  3. Executar la prova:

    python manage.py test

Proves de vistes

Provar vistes implica simular sol·licituds HTTP i comprovar les respostes.

Exemple: Provar una vista senzilla

  1. Crear una vista:

    # views.py
from django.http import HttpResponse

def home(request):
    return HttpResponse("Hola, Món!")

  2. Escriure un cas de prova per a la vista:

    # tests.py
from django.test import TestCase
from django.urls import reverse

class HomeViewTest(TestCase):
    def test_home_view(self):
        response = self.client.get(reverse('home'))
        self.assertEqual(response.status_code, 200)
        self.assertContains(response, "Hola, Món!")

  3. Actualitzar urls.py:

    # urls.py
from django.urls import path
from .views import home

urlpatterns = [
    path('', home, name='home'),
]

  4. Executar la prova:

    python manage.py test

Proves de formularis

Els formularis són una part essencial de moltes aplicacions Django. Provar formularis assegura que validen les dades correctament i gestionen l'entrada de l'usuari com s'espera.

Exemple: Provar un formulari

  1. Crear un formulari:

    # forms.py
from django import forms

class ItemForm(forms.Form):
    name = forms.CharField(max_length=100)
    description = forms.CharField(widget=forms.Textarea)

  2. Escriure un cas de prova per al formulari:

    # tests.py
from django.test import TestCase
from .forms import ItemForm

class ItemFormTest(TestCase):
    def test_valid_form(self):
        form_data = {'name': 'Test Item', 'description': 'Test Description'}
        form = ItemForm(data=form_data)
        self.assertTrue(form.is_valid())

    def test_invalid_form(self):
        form_data = {'name': '', 'description': 'Test Description'}
        form = ItemForm(data=form_data)
        self.assertFalse(form.is_valid())

  3. Executar la prova:

    python manage.py test

Conceptes avançats de proves

A mesura que et sentis més còmode amb les proves bàsiques, pots explorar temes més avançats com ara:

  • Mocking: Utilitzar el mòdul unittest.mock per simular objectes i funcions.
  • Proves amb bases de dades: Utilitzar la base de dades de proves de Django per provar interaccions amb la base de dades.
  • Integració contínua: Configurar pipelines de CI/CD per automatitzar les proves.

Conclusió

Les proves són una habilitat essencial per a qualsevol desenvolupador de Django. Escriure proves per als teus models, vistes i formularis pot assegurar que la teva aplicació funcioni com s'espera i sigui fàcil de mantenir. A mesura que progressis, podràs explorar tècniques de proves més avançades per millorar encara més la teva estratègia de proves. Bones proves!
