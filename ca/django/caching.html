Introducció a la Cache

La cache és una tècnica utilitzada per emmagatzemar còpies de fitxers o dades en una ubicació d'emmagatzematge temporal per reduir el temps que es triga a accedir a les dades i millorar el rendiment de les aplicacions web. En Django, la cache pot accelerar significativament el teu lloc reduint la càrrega a la teva base de dades i servidor.

Per què utilitzar la Cache?

  • Millora del Rendiment: Redueix el temps per servir respostes.
  • Reducció de la Càrrega de la Base de Dades: Disminueix el nombre de consultes a la base de dades.
  • Escalabilitat: Ajuda a gestionar més trànsit reduint la càrrega del servidor.

Tipus de Cache en Django

Django admet diversos tipus de cache:

  1. Cache en Memòria: El tipus més ràpid, emmagatzema dades en memòria.
  2. Cache Basada en Fitxers: Emmagatzema dades de cache en fitxers al disc.
  3. Cache en Base de Dades: Emmagatzema dades de cache a la base de dades.
  4. Memcached: Sistema de cache d'objectes de memòria distribuïda.
  5. Redis: Magatzem avançat de claus-valor, sovint utilitzat per a la cache.

Configuració de la Cache

Cache en Memòria

Per configurar la cache en memòria, afegeix el següent al teu settings.py:

CACHES = {
    'default': {
        'BACKEND': 'django.core.cache.backends.locmem.LocMemCache',
        'LOCATION': 'unique-snowflake',
    }
}

Cache Basada en Fitxers

Per a la cache basada en fitxers, configura settings.py de la següent manera:

CACHES = {
    'default': {
        'BACKEND': 'django.core.cache.backends.filebased.FileBasedCache',
        'LOCATION': '/var/tmp/django_cache',
    }
}

Memcached

Per utilitzar Memcached, necessites instal·lar la biblioteca pylibmc o python-memcached:

pip install python-memcached

Després configura settings.py:

CACHES = {
    'default': {
        'BACKEND': 'django.core.cache.backends.memcached.MemcachedCache',
        'LOCATION': '127.0.0.1:11211',
    }
}

Redis

Per utilitzar Redis, necessites instal·lar la biblioteca django-redis:

pip install django-redis

Després configura settings.py:

CACHES = {
    'default': {
        'BACKEND': 'django_redis.cache.RedisCache',
        'LOCATION': 'redis://127.0.0.1:6379/1',
        'OPTIONS': {
            'CLIENT_CLASS': 'django_redis.client.DefaultClient',
        }
    }
}

Ús de la Cache

Operacions Bàsiques de Cache

  • Establir un Valor de Cache:
from django.core.cache import cache

cache.set('my_key', 'my_value', timeout=60)  # Cache per 60 segons
  • Obtenir un Valor de Cache:
value = cache.get('my_key')
  • Eliminar un Valor de Cache:
cache.delete('my_key')

Cache de Vistes

Django proporciona decoradors per cachejar vistes senceres.

  • Cache Simple de Vistes:
from django.views.decorators.cache import cache_page

@cache_page(60 * 15)  # Cache per 15 minuts
def my_view(request):
    # La teva lògica de vista aquí
    return HttpResponse('Hello, World!')

Cache de Fragments de Plantilla

Cacheja parts específiques d'una plantilla utilitzant l'etiqueta de plantilla {% cache %}.

{% load cache %}
{% cache 500 sidebar %}
    <!-- Lògica costosa de renderització de la barra lateral -->
{% endcache %}

Tècniques Avançades de Cache

API de Cache de Baix Nivell

Per tenir més control sobre la cache, utilitza l'API de cache de baix nivell.

  • Incrementar un Valor de Cache:
cache.set('num', 1)
cache.incr('num')
  • Decrementar un Valor de Cache:
cache.decr('num')

Cache Condicional de Vistes

Cacheja vistes de manera condicional basant-se en paràmetres de la sol·licitud o autenticació d'usuaris.

from django.views.decorators.cache import cache_page
from django.utils.decorators import method_decorator

class MyView(View):
    @method_decorator(cache_page(60 * 15))
    def dispatch(self, *args, **kwargs):
        return super().dispatch(*args, **kwargs)

Conclusió

La cache és una tècnica poderosa per optimitzar el rendiment de les teves aplicacions Django. Entenent i implementant diferents estratègies de cache, pots reduir significativament la càrrega del servidor i millorar els temps de resposta. Comença amb la cache de vistes simples i progressivament avança cap a tècniques més avançades segons sigui necessari. Sempre monitora i prova l'impacte de la cache a la teva aplicació per assegurar-te que proporciona els beneficis de rendiment desitjats.
