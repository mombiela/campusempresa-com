Les Vistes Basades en Classes (CBVs) en Django proporcionen una manera d'organitzar la lògica de les teves vistes d'una manera més orientada a objectes. Ofereixen un enfocament més reutilitzable i modular en comparació amb les vistes basades en funcions (FBVs). Aquest tema et guiarà des dels conceptes bàsics fins a l'ús avançat de les CBVs en Django.

Introducció a les Vistes Basades en Classes

  • Definició: Les CBVs són vistes que es representen com a classes en lloc de funcions.
  • Avantatges:
    • Reutilització: Fàcil de reutilitzar patrons comuns.
    • Modularitat: Descompon les vistes complexes en peces més petites i manejables.
    • Extensibilitat: Més fàcil d'estendre i personalitzar.

Vistes Bàsiques Basades en Classes

Exemple Simple

Comencem amb un exemple simple d'una CBV que renderitza una plantilla.

from django.views.generic import TemplateView

class HomePageView(TemplateView):
    template_name = 'home.html'
  • TemplateView: Una CBV integrada que renderitza una plantilla especificada.
  • template_name: La plantilla que es renderitzarà.

Configuració de l'URL

Per utilitzar aquesta vista, cal afegir-la a la configuració de l'URL.

from django.urls import path
from .views import HomePageView

urlpatterns = [
    path('', HomePageView.as_view(), name='home'),
]
  • as_view(): Converteix la classe en una funció de vista que es pot utilitzar en patrons d'URL.

Vistes Basades en Classes Més Utilitzades

ListView

  • Propòsit: Mostrar una llista d'objectes.
  • Exemple:
from django.views.generic import ListView
from .models import Article

class ArticleListView(ListView):
    model = Article
    template_name = 'article_list.html'
    context_object_name = 'articles'
  • model: El model que es consultarà.
  • template_name: La plantilla que es renderitzarà.
  • context_object_name: El nom de la variable de context a utilitzar a la plantilla.

DetailView

  • Propòsit: Mostrar un únic objecte.
  • Exemple:
from django.views.generic import DetailView
from .models import Article

class ArticleDetailView(DetailView):
    model = Article
    template_name = 'article_detail.html'
    context_object_name = 'article'

Vistes Basades en Classes Avançades

FormView

  • Propòsit: Gestionar formularis.
  • Exemple:
from django.views.generic import FormView
from .forms import ContactForm

class ContactFormView(FormView):
    template_name = 'contact.html'
    form_class = ContactForm
    success_url = '/thanks/'

    def form_valid(self, form):
        # Processar les dades del formulari
        form.send_email()
        return super().form_valid(form)
  • form_class: La classe de formulari a utilitzar.
  • success_url: L'URL a redirigir després de l'enviament correcte del formulari.
  • form_valid(): Mètode per gestionar les dades vàlides del formulari.

CreateView, UpdateView, i DeleteView

  • Propòsit: Gestionar la creació, actualització i eliminació d'objectes.
  • Exemples:
from django.views.generic import CreateView, UpdateView, DeleteView
from .models import Article
from django.urls import reverse_lazy

class ArticleCreateView(CreateView):
    model = Article
    template_name = 'article_form.html'
    fields = ['title', 'content']
    success_url = reverse_lazy('article_list')

class ArticleUpdateView(UpdateView):
    model = Article
    template_name = 'article_form.html'
    fields = ['title', 'content']
    success_url = reverse_lazy('article_list')

class ArticleDeleteView(DeleteView):
    model = Article
    template_name = 'article_confirm_delete.html'
    success_url = reverse_lazy('article_list')
  • fields: Els camps a incloure en el formulari.
  • reverse_lazy: Utilitzat per retardar l'avaluació de l'URL fins que sigui necessari.

Personalització de les Vistes Basades en Classes

Sobreescriure Mètodes

Pots sobreescriure mètodes per personalitzar el comportament de les CBVs.

class CustomListView(ListView):
    model = Article
    template_name = 'custom_article_list.html'

    def get_queryset(self):
        return Article.objects.filter(published=True)
  • get_queryset(): Personalitza el queryset retornat per la vista.

Afegir Dades de Context

Pots afegir dades de context extra a les teves plantilles.

class CustomDetailView(DetailView):
    model = Article
    template_name = 'custom_article_detail.html'

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        context['extra_data'] = 'Això són dades extra'
        return context
  • get_context_data(): Afegeix dades de context extra a la plantilla.

Conclusió

Les Vistes Basades en Classes en Django ofereixen una manera poderosa d'organitzar la lògica de les teves vistes d'una manera més modular i reutilitzable. Des de vistes bàsiques com TemplateView fins a vistes més avançades com FormView, CreateView, UpdateView, i DeleteView, les CBVs proporcionen un enfocament estructurat per gestionar tasques comunes de desenvolupament web. Entenent i utilitzant les CBVs, pots escriure codi més net, més mantenible i aprofitar les funcionalitats integrades de Django per construir aplicacions web robustes.
