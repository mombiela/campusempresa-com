El Projecte Sobre nosaltres Contribuir Donacions Llicència

Desplegar la teva aplicació Django a Heroku és un procés senzill que et permet fer accessible la teva aplicació web al món. Aquesta secció et guiarà a través dels passos necessaris per desplegar el teu projecte Django a Heroku, des de configurar el teu entorn fins a pujar el teu codi en viu.

Prerequisits

Abans de començar, assegura't de tenir el següent:

  • Un compte a Heroku: Registra't a Heroku.
  • Heroku CLI instal·lat: Segueix les instruccions d'instal·lació a Heroku CLI.
  • Git instal·lat: Pots descarregar-lo des de Git.

Pas 1: Prepara el Teu Projecte Django

Instal·la els Paquets Necessaris

Heroku requereix alguns paquets addicionals per executar la teva aplicació Django. Instal·la aquests paquets utilitzant pip:

pip install gunicorn dj-database-url psycopg2-binary
  • gunicorn: Un servidor HTTP WSGI per a UNIX.
  • dj-database-url: Et permet utilitzar l'URL de la base de dades proporcionada per Heroku.
  • psycopg2-binary: Adaptador de base de dades PostgreSQL per a Python.

Actualitza `requirements.txt`

Assegura't que totes les teves dependències estiguin llistades a requirements.txt:

pip freeze > requirements.txt

Crea un `Procfile`

Heroku utilitza un Procfile per determinar com executar la teva aplicació. Crea un fitxer anomenat Procfile al directori arrel del teu projecte amb el següent contingut:

web: gunicorn your_project_name.wsgi

Substitueix your_project_name pel nom del teu projecte Django.

Configura els Paràmetres de la Base de Dades

Modifica el teu settings.py per utilitzar dj-database-url per a la configuració de la base de dades. Afegeix el següent codi:

import dj_database_url

DATABASES = {
    'default': dj_database_url.config(default='postgres://localhost/your_db_name')
}

Substitueix your_db_name pel nom de la teva base de dades local.

Configuració d'Arxius Estàtics

Heroku requereix alguns paràmetres addicionals per servir arxius estàtics. Afegeix el següent al teu settings.py:

import os

BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))

STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')
STATIC_URL = '/static/'

# Llocs addicionals per a collectstatic per trobar arxius estàtics.
STATICFILES_DIRS = (
    os.path.join(BASE_DIR, 'static'),
)

# Servei simplificat d'arxius estàtics.
# https://warehouse.python.org/project/whitenoise/
MIDDLEWARE = [
    'whitenoise.middleware.WhiteNoiseMiddleware',
    # ... altres middleware ...
]

STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'

Instal·la whitenoise:

pip install whitenoise

Actualitza requirements.txt de nou:

pip freeze > requirements.txt

Pas 2: Desplega a Heroku

Inicialitza un Repositori Git

Si encara no ho has fet, inicialitza un repositori Git al directori del teu projecte:

git init

Afegeix tots els teus arxius i fes un commit:

git add .
git commit -m "Initial commit"

Crea una Aplicació Heroku

Inicia sessió a Heroku utilitzant el CLI:

heroku login

Crea una nova aplicació Heroku:

heroku create your-app-name

Substitueix your-app-name per un nom únic per a la teva aplicació.

Configura les Variables d'Entorn

Configura les variables d'entorn SECRET_KEY i DEBUG:

heroku config:set SECRET_KEY='your_secret_key'
heroku config:set DEBUG=False

Substitueix your_secret_key per una clau segura per a la teva aplicació.

Puja el Teu Codi a Heroku

Desplega el teu codi a Heroku:

git push heroku master

Executa les Migracions

Executa les migracions de la base de dades a Heroku:

heroku run python manage.py migrate

Recull els Arxius Estàtics

Recull els arxius estàtics:

heroku run python manage.py collectstatic --noinput

Conclusió

Felicitats! Has desplegat amb èxit la teva aplicació Django a Heroku. La teva aplicació hauria d'estar en viu i accessible a través de l'URL proporcionada per Heroku. Aquest procés implica configurar el teu projecte Django per a producció, configurar els paquets necessaris i utilitzar Git i Heroku CLI per desplegar el teu codi. Amb aquests passos, pots desplegar i gestionar fàcilment les teves aplicacions Django a Heroku.
