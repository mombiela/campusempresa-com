Introducció

Les proves són una part crucial del desenvolupament d'aplicacions web robustes i fiables. En Django, les vistes i les plantilles són components centrals que gestionen la lògica de presentació i la interfície d'usuari. Aquesta secció et guiarà a través del procés de provar vistes i plantilles en Django, des dels conceptes bàsics fins a tècniques avançades.

Conceptes Clau

  • Vistes: Funcions o classes que gestionen sol·licituds HTTP i retornen respostes HTTP.
  • Plantilles: Fitxers HTML que defineixen l'estructura de les pàgines web.
  • Proves: El procés de verificar que el teu codi funciona com s'espera.

Configuració de l'Entorn de Proves

Abans d'endinsar-nos en les proves de vistes i plantilles, assegura't de tenir la següent configuració:

  • Django instal·lat.
  • Un projecte i una aplicació Django creats.
  • Comprensió bàsica de les vistes i plantilles de Django.

Proves de Vistes

Provar vistes implica comprovar que les plantilles correctes es renderitzen i que les dades de context adequades es passen.

Prova Bàsica de Vista

Comencem amb un exemple senzill. Suposem que tens una vista que renderitza una plantilla anomenada home.html.

Exemple de Vista

# views.py
from django.shortcuts import render

def home_view(request):
    return render(request, 'home.html', {'message': 'Hello, World!'})

Escriure la Prova

# tests.py
from django.test import TestCase
from django.urls import reverse

class HomeViewTest(TestCase):
    def test_home_view_status_code(self):
        response = self.client.get(reverse('home'))
        self.assertEqual(response.status_code, 200)

    def test_home_view_template(self):
        response = self.client.get(reverse('home'))
        self.assertTemplateUsed(response, 'home.html')

    def test_home_view_context(self):
        response = self.client.get(reverse('home'))
        self.assertEqual(response.context['message'], 'Hello, World!')

Explicació

  • test_home_view_status_code: Comprova si la vista retorna un codi d'estat 200.
  • test_home_view_template: Assegura que la vista utilitza la plantilla correcta.
  • test_home_view_context: Verifica que les dades de context passades a la plantilla són correctes.

Proves de Plantilles

Provar plantilles implica assegurar-se que la sortida HTML és com s'espera.

Prova Bàsica de Plantilla

Ampliem el nostre exemple per provar el contingut de home.html.

Exemple de Plantilla

<!-- home.html -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Home</title>
</head>
<body>
    <h1>{{ message }}</h1>
</body>
</html>

Escriure la Prova

# tests.py
from django.test import TestCase
from django.urls import reverse

class HomeTemplateTest(TestCase):
    def test_home_template_content(self):
        response = self.client.get(reverse('home'))
        self.assertContains(response, '<h1>Hello, World!</h1>')

Explicació

  • test_home_template_content: Comprova si l'HTML renderitzat conté el contingut esperat.

Tècniques Avançades de Proves

Per a vistes i plantilles més complexes, potser necessitaràs provar:

  • Enviament de formularis: Assegura't que els formularis es renderitzen correctament i gestionen les enviaments adequadament.
  • Autenticació: Verifica que les vistes es comporten correctament per a usuaris autenticats i no autenticats.
  • Permisos: Comprova que els usuaris amb diferents permisos veuen el contingut adequat.

Exemple: Provar l'Enviament de Formularis

Suposem que tens una vista que gestiona l'enviament d'un formulari.

Exemple de Vista

# views.py
from django.shortcuts import render, redirect
from .forms import MyForm

def form_view(request):
    if request.method == 'POST':
        form = MyForm(request.POST)
        if form.is_valid():
            # Processar dades del formulari
            return redirect('success')
    else:
        form = MyForm()
    return render(request, 'form.html', {'form': form})

Escriure la Prova

# tests.py
from django.test import TestCase
from django.urls import reverse

class FormViewTest(TestCase):
    def test_form_view_get(self):
        response = self.client.get(reverse('form'))
        self.assertEqual(response.status_code, 200)
        self.assertTemplateUsed(response, 'form.html')

    def test_form_view_post_valid(self):
        response = self.client.post(reverse('form'), {'field1': 'value1', 'field2': 'value2'})
        self.assertRedirects(response, reverse('success'))

    def test_form_view_post_invalid(self):
        response = self.client.post(reverse('form'), {'field1': '', 'field2': 'value2'})
        self.assertEqual(response.status_code, 200)
        self.assertTemplateUsed(response, 'form.html')
        self.assertFormError(response, 'form', 'field1', 'This field is required.')

Explicació

  • test_form_view_get: Prova la sol·licitud GET per assegurar-se que el formulari es renderitza correctament.
  • test_form_view_post_valid: Prova un enviament de formulari vàlid i comprova que hi ha una redirecció.
  • test_form_view_post_invalid: Prova un enviament de formulari invàlid i comprova els errors del formulari.

Conclusió

Provar vistes i plantilles en Django és essencial per assegurar que la teva aplicació es comporta com s'espera. Escriure proves per a codis d'estat, plantilles, dades de context i enviaments de formularis et permet detectar errors aviat i mantenir un codi de qualitat. A mesura que avancis, considera provar escenaris més complexos que impliquin autenticació i permisos per millorar encara més la fiabilitat de la teva aplicació.
