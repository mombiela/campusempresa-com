El Projecte Sobre nosaltres Contribuir Donacions Llicència

Introducció

Entendre l'estructura d'un projecte Django i com gestionar les aplicacions Django és crucial per desenvolupar aplicacions web escalables i mantenibles. Aquesta secció cobrirà els conceptes bàsics de l'estructura de projectes i aplicacions Django, com crear i gestionar aplicacions, i les millors pràctiques per organitzar el teu codi.

Estructura del Projecte Django

Quan crees un nou projecte Django, Django configura una estructura de directoris per defecte. Comencem creant un nou projecte Django per veure aquesta estructura en acció.

Creant un Nou Projecte Django

Per crear un nou projecte Django, utilitza la següent comanda:

django-admin startproject myproject

Aquesta comanda crea un directori anomenat myproject amb la següent estructura:

myproject/
    manage.py
    myproject/
        __init__.py
        settings.py
        urls.py
        asgi.py
        wsgi.py

Fitxers i Directoris Clau

  • manage.py: Una utilitat de línia de comandes que et permet interactuar amb el teu projecte Django.
  • myproject/: El directori intern amb el mateix nom que el teu projecte.
    • init.py: Un fitxer buit que indica a Python que aquest directori ha de ser considerat un paquet.
    • settings.py: Configuració del teu projecte Django.
    • urls.py: Declaracions d'URL per al teu projecte Django; una "taula de continguts" del teu lloc web impulsat per Django.
    • asgi.py: Punt d'entrada per a servidors web compatibles amb ASGI per servir el teu projecte.
    • wsgi.py: Punt d'entrada per a servidors web compatibles amb WSGI per servir el teu projecte.

Aplicacions Django

Un projecte Django pot contenir múltiples aplicacions. Cada aplicació hauria de fer una cosa i fer-la bé. Per exemple, podries tenir una aplicació per a l'autenticació d'usuaris, una altra per a un blog, i una altra per a un fòrum.

Creant una Nova Aplicació Django

Per crear una nova aplicació dins del teu projecte, utilitza la següent comanda:

python manage.py startapp myapp

Aquesta comanda crea un directori anomenat myapp amb la següent estructura:

myapp/
    __init__.py
    admin.py
    apps.py
    models.py
    tests.py
    views.py
    migrations/
        __init__.py

Fitxers i Directoris Clau en una Aplicació

  • admin.py: Registra els teus models amb el lloc d'administració de Django.
  • apps.py: Configuració per a l'aplicació.
  • models.py: Defineix els teus models de base de dades aquí.
  • tests.py: Escriu les teves proves aquí.
  • views.py: Defineix les teves vistes aquí.
  • migrations/: Emmagatzema les migracions de la base de dades aquí.

Integrant Aplicacions en un Projecte

Un cop hagis creat una aplicació, has de dir-li al teu projecte. Això es fa afegint l'aplicació a la llista INSTALLED_APPS en settings.py.

Exemple

# myproject/settings.py

INSTALLED_APPS = [
    ...
    'myapp',
]

Millors Pràctiques per a l'Estructura de Projectes i Aplicacions

  • Mantingues les aplicacions petites i enfocades: Cada aplicació hauria de tenir una única responsabilitat.
  • Utilitza noms significatius: Anomena les teves aplicacions i projectes de manera clara per reflectir el seu propòsit.
  • Organitza plantilles i fitxers estàtics: Utilitza directoris dins de cada aplicació per mantenir les plantilles i els fitxers estàtics organitzats.

Exemple d'Estructura de Directori

myproject/
    manage.py
    myproject/
        __init__.py
        settings.py
        urls.py
        asgi.py
        wsgi.py
    myapp/
        __init__.py
        admin.py
        apps.py
        models.py
        tests.py
        views.py
        migrations/
            __init__.py
        templates/
            myapp/
                template1.html
        static/
            myapp/
                style.css

Conclusió

Entendre l'estructura dels projectes i aplicacions Django és fonamental per desenvolupar aplicacions web eficients i mantenibles. Seguint les millors pràctiques i organitzant el teu codi adequadament, pots crear aplicacions escalables i modulars. En les següents seccions, aprofundirem en cada component de les aplicacions i projectes Django, explorant models, vistes, plantilles, i més.
