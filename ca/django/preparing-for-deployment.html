El Projecte Sobre nosaltres Contribuir Donacions Llicència

Desplegar una aplicació Django implica diversos passos per assegurar que la teva aplicació sigui segura, escalable i funcioni bé en un entorn de producció. Aquesta secció et guiarà a través dels passos essencials necessaris per preparar el teu projecte Django per al desplegament.

Conceptes Clau

  • Configuració de l'Entorn: Configurar diferents configuracions per al desenvolupament i la producció.
  • Gestió d'Arxius Estàtics: Gestionar arxius estàtics com CSS, JavaScript i imatges.
  • Configuració de la Base de Dades: Configurar una base de dades preparada per a la producció.
  • Millores de Seguretat: Implementar les millors pràctiques de seguretat.
  • Configuració del Servidor Web: Configurar un servidor web per servir la teva aplicació Django.
  • Monitorització i Registre: Configurar la monitorització i el registre per a la teva aplicació.

Configuració de l'Entorn

Gestió de Configuracions

Les configuracions de Django s'han de configurar de manera diferent per als entorns de desenvolupament i producció. Això es pot aconseguir utilitzant variables d'entorn i arxius de configuració separats.

Exemple: Utilitzant Variables d'Entorn

# settings.py
import os

SECRET_KEY = os.getenv('DJANGO_SECRET_KEY', 'default-secret-key')
DEBUG = os.getenv('DJANGO_DEBUG', 'False') == 'True'
ALLOWED_HOSTS = os.getenv('DJANGO_ALLOWED_HOSTS', '').split(',')

Exemple: Arxius de Configuració Separats

# settings/base.py
import os

SECRET_KEY = os.getenv('DJANGO_SECRET_KEY', 'default-secret-key')
ALLOWED_HOSTS = os.getenv('DJANGO_ALLOWED_HOSTS', '').split(',')

# settings/development.py
from .base import *
DEBUG = True

# settings/production.py
from .base import *
DEBUG = False

Gestió d'Arxius Estàtics

En un entorn de producció, els arxius estàtics han de ser recollits i servits de manera eficient.

Recollint Arxius Estàtics

python manage.py collectstatic

Servint Arxius Estàtics

  • Utilitzant un CDN: Descarregar els arxius estàtics a una Xarxa de Distribució de Continguts.
  • Utilitzant un Servidor Web: Configurar el teu servidor web (per exemple, Nginx) per servir arxius estàtics.

Exemple: Configuració de Nginx

server {
    location /static/ {
        alias /path/to/staticfiles/;
    }
}

Configuració de la Base de Dades

Per a la producció, es recomana utilitzar una base de dades robusta com PostgreSQL.

Exemple: Configuració de PostgreSQL

# settings.py
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': os.getenv('DB_NAME'),
        'USER': os.getenv('DB_USER'),
        'PASSWORD': os.getenv('DB_PASSWORD'),
        'HOST': os.getenv('DB_HOST'),
        'PORT': os.getenv('DB_PORT', '5432'),
    }
}

Millores de Seguretat

Configuracions Clau de Seguretat

  • SECRET_KEY: Assegura't que es mantingui en secret i no estigui codificat.
  • DEBUG: Configura a False en producció.
  • ALLOWED_HOSTS: Especifica els hosts permesos.
  • HTTPS: Utilitza HTTPS per xifrar les dades en trànsit.

Exemple: Configuracions de Seguretat

# settings.py
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True

Configuració del Servidor Web

Utilitzant Gunicorn

Gunicorn és un servidor HTTP WSGI popular per servir aplicacions Django.

Exemple: Executant Gunicorn

gunicorn myproject.wsgi:application --bind 0.0.0.0:8000

Utilitzant Nginx com a Proxy Invers

Nginx es pot utilitzar per reenviar sol·licituds a Gunicorn.

Exemple: Configuració de Nginx

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Monitorització i Registre

Configurant el Registre

Configura Django per registrar errors i esdeveniments importants.

Exemple: Configuració del Registre

# settings.py
LOGGING = {
    'version': 1,
    'disable_existing_loggers': False,
    'handlers': {
        'file': {
            'level': 'DEBUG',
            'class': 'logging.FileHandler',
            'filename': '/path/to/django/debug.log',
        },
    },
    'loggers': {
        'django': {
            'handlers': ['file'],
            'level': 'DEBUG',
            'propagate': True,
        },
    },
}

Eines de Monitorització

  • Sentry: Per al seguiment d'errors.
  • Prometheus i Grafana: Per a la monitorització del rendiment.

Conclusió

Preparar una aplicació Django per al desplegament implica configurar les configuracions de l'entorn, gestionar els arxius estàtics, configurar una base de dades preparada per a la producció, millorar la seguretat, configurar un servidor web i configurar la monitorització i el registre. Seguint aquests passos, pots assegurar-te que la teva aplicació Django està preparada per a un entorn de producció, proporcionant un servei segur, escalable i eficient als teus usuaris.
