El Projecte Sobre nosaltres Contribuir Donacions Llicència

Les proves unitàries són una part crucial del desenvolupament d'aplicacions Django robustes i fiables. Aquesta secció et guiarà des dels conceptes bàsics de les proves unitàries a Django fins a tècniques més avançades.

Introducció a les Proves Unitàries

  • Definició: Les proves unitàries impliquen provar components individuals de la teva aplicació per assegurar-se que funcionen com s'espera.
  • Importància: Ajuda a detectar errors aviat, assegura la fiabilitat del codi i facilita la refactorització.

Configuració de Proves Unitàries a Django

  • Marc de Proves de Django: Django ve amb un marc de proves integrat basat en el mòdul unittest de Python.
  • Creació d'Arxius de Proves: Per convenció, els arxius de proves es nomenen test_*.py i es col·loquen al mateix directori que el mòdul que estan provant.

Exemple: Cas de Prova Bàsic

# tests.py
from django.test import TestCase
from .models import MyModel

class MyModelTestCase(TestCase):
    def setUp(self):
        MyModel.objects.create(name="test1", value=10)
        MyModel.objects.create(name="test2", value=20)

    def test_my_model_values(self):
        """Els valors de MyModel es reconeixen correctament"""
        test1 = MyModel.objects.get(name="test1")
        test2 = MyModel.objects.get(name="test2")
        self.assertEqual(test1.value, 10)
        self.assertEqual(test2.value, 20)
  • Explicació:
    • TestCase és una classe de django.test que proporciona un marc per escriure proves.
    • El mètode setUp s'utilitza per crear dades inicials.
    • test_my_model_values és un mètode de prova que comprova si els valors s'assignen correctament.

Executar Proves

  • Comanda: Utilitza python manage.py test per executar totes les proves del teu projecte Django.
  • Sortida: La comanda mostrarà els resultats de les proves, indicant quines proves han passat i quines han fallat.

Escriure Proves Més Complexes

  • Provar Vistes: Pots provar vistes simulant sol·licituds HTTP.
  • Provar Formularis: Assegura't que els formularis validen i processen les dades correctament.
  • Provar Models: Comprova els mètodes i propietats del model.

Exemple: Provar una Vista

# tests.py
from django.test import TestCase, Client
from django.urls import reverse

class MyViewTestCase(TestCase):
    def setUp(self):
        self.client = Client()

    def test_my_view(self):
        response = self.client.get(reverse('my_view'))
        self.assertEqual(response.status_code, 200)
        self.assertContains(response, "Expected Content")
  • Explicació:
    • Client s'utilitza per simular un navegador web.
    • reverse s'utilitza per obtenir l'URL d'una vista pel seu nom.
    • assertContains comprova si la resposta conté el contingut esperat.

Tècniques Avançades de Proves

  • Mocking: Utilitza la biblioteca unittest.mock per simular dependències externes.
  • Fixtures: Conjunts de dades predefinits que es poden carregar a la base de dades per provar.
  • Cobertura de Proves: Mesura quant del teu codi està provat utilitzant eines com coverage.py.

Exemple: Utilitzar Mocking

# tests.py
from django.test import TestCase
from unittest.mock import patch
from .external_service import ExternalService

class ExternalServiceTestCase(TestCase):
    @patch('myapp.external_service.ExternalService.get_data')
    def test_external_service(self, mock_get_data):
        mock_get_data.return_value = {'key': 'value'}
        response = ExternalService().get_data()
        self.assertEqual(response, {'key': 'value'})
  • Explicació:
    • patch s'utilitza per substituir el mètode get_data de ExternalService amb un mock.
    • mock_get_data.return_value estableix el valor de retorn del mètode simulat.

Conclusió

Les proves unitàries són una pràctica essencial per mantenir la qualitat i la fiabilitat de les teves aplicacions Django. Començant amb proves bàsiques i incorporant gradualment escenaris més complexos, pots assegurar-te que la teva base de codi es mantingui robusta i fàcil de mantenir. Recorda executar les teves proves freqüentment i apunta a una alta cobertura de proves per detectar problemes aviat en el procés de desenvolupament.
