Inici de sessió i tancament de sessió a Django
Registre d'Usuaris ►

Introducció

En aquesta secció, tractarem com implementar la funcionalitat d'inici de sessió i tancament de sessió d'usuari en una aplicació Django. Això inclou configurar les vistes, formularis i plantilles necessàries, així com configurar el sistema d'autenticació integrat de Django.

Configuració de l'autenticació

Instal·lació de Django

Abans de començar, assegura't de tenir Django instal·lat. Si no, pots instal·lar-lo utilitzant pip:

pip install django

Creació d'un projecte i una aplicació Django

Primer, crea un nou projecte Django i una aplicació dins d'aquest:

django-admin startproject myproject
cd myproject
django-admin startapp myapp

Configuració del projecte

Afegeix la nova aplicació a la llista INSTALLED_APPS a myproject/settings.py:

INSTALLED_APPS = [
    ...
    'myapp',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    ...
]

Implementació de l'inici de sessió

Creació de la vista d'inici de sessió

Crea una vista per gestionar l'inici de sessió d'usuari a myapp/views.py:

from django.shortcuts import render, redirect
from django.contrib.auth import authenticate, login
from django.contrib import messages

def login_view(request):
    if request.method == 'POST':
        username = request.POST['username']
        password = request.POST['password']
        user = authenticate(request, username=username, password=password)
        if user is not None:
            login(request, user)
            return redirect('home')
        else:
            messages.error(request, 'Invalid username or password')
    return render(request, 'myapp/login.html')

Creació de la plantilla d'inici de sessió

Crea una plantilla per a la pàgina d'inici de sessió a myapp/templates/myapp/login.html:

<!DOCTYPE html>
<html>
<head>
    <title>Login</title>
</head>
<body>
    <h2>Login</h2>
    {% if messages %}
        <ul>
            {% for message in messages %}
                <li>{{ message }}</li>
            {% endfor %}
        </ul>
    {% endif %}
    <form method="post">
        {% csrf_token %}
        <label for="username">Username:</label>
        <input type="text" name="username" id="username" required><br>
        <label for="password">Password:</label>
        <input type="password" name="password" id="password" required><br>
        <button type="submit">Login</button>
    </form>
</body>
</html>

Actualització dels URLs

Afegeix un patró d'URL per a la vista d'inici de sessió a myapp/urls.py:

from django.urls import path
from .views import login_view

urlpatterns = [
    path('login/', login_view, name='login'),
]

Implementació del tancament de sessió

Creació de la vista de tancament de sessió

Crea una vista per gestionar el tancament de sessió d'usuari a myapp/views.py:

from django.contrib.auth import logout

def logout_view(request):
    logout(request)
    return redirect('login')

Actualització dels URLs

Afegeix un patró d'URL per a la vista de tancament de sessió a myapp/urls.py:

from .views import logout_view

urlpatterns = [
    ...
    path('logout/', logout_view, name='logout'),
]

Protecció de vistes amb Login Required

Per assegurar que certes vistes només siguin accessibles per usuaris autenticats, utilitza el decorador login_required. Per exemple, a myapp/views.py:

from django.contrib.auth.decorators import login_required

@login_required
def home_view(request):
    return render(request, 'myapp/home.html')

Actualitza el patró d'URL per a la vista home a myapp/urls.py:

from .views import home_view

urlpatterns = [
    ...
    path('', home_view, name='home'),
]

Conclusió

En aquesta secció, hem tractat els conceptes bàsics per implementar la funcionalitat d'inici de sessió i tancament de sessió en una aplicació Django. Hem creat vistes, plantilles i patrons d'URL per gestionar l'autenticació d'usuaris. A més, hem après com protegir vistes utilitzant el decorador login_required. Amb aquests fonaments, ara pots gestionar sessions d'usuari i assegurar parts de la teva aplicació de manera efectiva.
