El middleware és un concepte fonamental a Django que et permet processar sol·licituds globalment abans que arribin a la vista o després que la vista les hagi processat. Aquesta secció et guiarà des dels conceptes bàsics fins a l'ús avançat del middleware a Django.

Què és el Middleware?

El middleware és una manera de processar sol·licituds globalment abans que arribin a la vista o després que la vista les hagi processat. Els components del middleware s'executen en un ordre específic, i cadascun pot modificar la sol·licitud o la resposta.

Conceptes Clau

  • Request Middleware: Codi que s'executa abans que es cridi la vista.
  • View Middleware: Codi que s'executa després que es cridi la vista.
  • Response Middleware: Codi que s'executa després que s'hagi construït la resposta.
  • Exception Middleware: Codi que s'executa si es produeix una excepció.

Middleware Integrat

Django ve amb diverses classes de middleware integrades que gestionen tasques comunes:

  • SecurityMiddleware
  • SessionMiddleware
  • CommonMiddleware
  • CsrfViewMiddleware
  • AuthenticationMiddleware
  • MessageMiddleware

Exemple de Middleware Integrat

MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.common.CommonMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
]

Middleware Personalitzat

Crear middleware personalitzat et permet afegir la teva pròpia funcionalitat al processament de sol·licituds/respostes.

Passos per Crear Middleware Personalitzat

  1. Defineix la Classe de Middleware: Crea una nova classe Python amb mètodes per processar sol·licituds i respostes.
  2. Afegeix el Middleware a la Configuració: Afegeix la teva classe de middleware a la llista MIDDLEWARE a settings.py.

Exemple de Middleware Personalitzat

# myapp/middleware.py

class SimpleMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        # Codi que s'executa per a cada sol·licitud abans
        # que es cridi la vista (i el middleware posterior).
        print("Abans de la vista")

        response = self.get_response(request)

        # Codi que s'executa per a cada sol·licitud/resposta després
        # que es cridi la vista.
        print("Després de la vista")

        return response

Afegir Middleware Personalitzat a la Configuració

# settings.py

MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.common.CommonMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
    'myapp.middleware.SimpleMiddleware',  # Afegeix el teu middleware personalitzat aquí
]

Conceptes Avançats de Middleware

Ordre del Middleware

L'ordre en què el middleware està llistat a la configuració MIDDLEWARE és important. El middleware s'executa en l'ordre definit a la llista per a les sol·licituds i en ordre invers per a les respostes.

Mètodes del Middleware

  • __init__(self, get_response): Es crida una vegada quan el servidor web s'inicia.
  • __call__(self, request): Es crida en cada sol·licitud.
  • process_view(self, request, view_func, view_args, view_kwargs): Es crida just abans que Django cridi la vista.
  • process_exception(self, request, exception): Es crida si la vista genera una excepció.
  • process_template_response(self, request, response): Es crida just després que la vista hagi acabat d'executar-se si la resposta conté un mètode render().

Exemple amb Mètodes Addicionals

# myapp/middleware.py

class AdvancedMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        print("Abans de la vista")
        response = self.get_response(request)
        print("Després de la vista")
        return response

    def process_view(self, request, view_func, view_args, view_kwargs):
        print("Abans de la funció de vista")
        return None

    def process_exception(self, request, exception):
        print("S'ha produït una excepció")
        return None

    def process_template_response(self, request, response):
        print("Abans de renderitzar la plantilla")
        return response

Conclusió

El middleware a Django és una eina poderosa que et permet processar sol·licituds i respostes globalment. Entenent i utilitzant tant el middleware integrat com el personalitzat, pots afegir una àmplia gamma de funcionalitats a les teves aplicacions Django. Recorda gestionar acuradament l'ordre del middleware i utilitzar els mètodes adequats per gestionar diferents etapes del processament de sol·licituds i respostes.
