Proves d'Integració
Proves de Vistes i Plantilles ►

Les proves d'integració són essencials per assegurar que les diferents parts de la teva aplicació Django funcionen conjuntament com s'espera. A diferència de les proves unitàries, que proven components individuals de manera aïllada, les proves d'integració se centren en les interaccions entre components.

Què són les Proves d'Integració?

  • Definició: Les proves d'integració verifiquen que múltiples components de la teva aplicació funcionen correctament conjuntament.
  • Propòsit: Ajuden a detectar problemes que sorgeixen de la interacció entre diferents parts del sistema.
  • Abast: Normalment més ampli que les proves unitàries, cobrint múltiples mòduls o fins i tot tota l'aplicació.

Per què Utilitzar Proves d'Integració en Django?

  • Cobertura Integral: Assegura que tota l'aplicació funciona com s'espera.
  • Detecció Primerenca d'Errors: Identifica problemes que les proves unitàries poden passar per alt.
  • Confiança en el Codi: Proporciona una major seguretat que els canvis no trencaran l'aplicació.

Configuració de Proves d'Integració en Django

Prerequisits

  • Comprensió bàsica dels models, vistes i plantilles de Django.
  • Familiaritat amb el marc de proves de Django.

Creació d'un Cas de Prova

Django utilitza la classe TestCase de django.test per crear proves d'integració. Aquí tens un exemple senzill:

from django.test import TestCase
from django.urls import reverse
from myapp.models import MyModel

class MyModelIntegrationTest(TestCase):
    def setUp(self):
        # Configurar dades inicials per a la prova
        self.my_model_instance = MyModel.objects.create(field1='value1', field2='value2')

    def test_model_view_integration(self):
        # Provar la interacció entre el model i la vista
        response = self.client.get(reverse('my_model_detail', args=[self.my_model_instance.id]))
        self.assertEqual(response.status_code, 200)
        self.assertContains(response, 'value1')

Explicació

  • Mètode setUp: Inicialitza les dades necessàries per a la prova.
  • Mètode test_model_view_integration: Prova la interacció entre el model i la vista fent una sol·licitud GET a la vista i comprovant la resposta.

Millors Pràctiques per a Proves d'Integració

  • Aïllar Proves: Assegura't que cada prova sigui independent i no depengui de l'estat deixat per altres proves.
  • Utilitzar Fixtures: Utilitza les fixtures de Django per configurar dades inicials.
  • Simular Serveis Externs: Utilitza simulacions per simular interaccions amb serveis externs.

Exemple d'Ús de Fixtures

from django.test import TestCase
from django.urls import reverse
from myapp.models import MyModel

class MyModelIntegrationTest(TestCase):
    fixtures = ['my_model_fixture.json']

    def test_model_view_integration(self):
        my_model_instance = MyModel.objects.get(pk=1)
        response = self.client.get(reverse('my_model_detail', args=[my_model_instance.id]))
        self.assertEqual(response.status_code, 200)
        self.assertContains(response, 'value1')

Simulació de Serveis Externs

from django.test import TestCase
from unittest.mock import patch
from myapp.services import external_service

class ExternalServiceIntegrationTest(TestCase):
    @patch('myapp.services.external_service.call')
    def test_external_service_integration(self, mock_call):
        mock_call.return_value = {'status': 'success'}
        response = self.client.get('/external-service-endpoint/')
        self.assertEqual(response.status_code, 200)
        self.assertContains(response, 'success')

Executar Proves d'Integració

  • Utilitza el comandament manage.py test de Django per executar les teves proves d'integració.
  • Exemple: python manage.py test myapp.tests.test_integration

Conclusió

Les proves d'integració són una part crucial de la teva estratègia de proves en Django. Ajuden a assegurar que les diferents parts de la teva aplicació funcionen conjuntament sense problemes. Seguint les millors pràctiques i utilitzant eficaçment el marc de proves de Django, pots escriure proves d'integració robustes que proporcionen confiança en la funcionalitat de la teva aplicació.

Al final d'aquesta secció, hauràs de ser capaç de:

  • Entendre el propòsit i l'abast de les proves d'integració.
  • Configurar i escriure proves d'integració en Django.
  • Aplicar les millors pràctiques per assegurar que les teves proves siguin efectives i mantenibles.
