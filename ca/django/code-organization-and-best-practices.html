Introducció

En aquesta secció, tractarem com organitzar el teu projecte Django i seguir les millors pràctiques per assegurar que el teu codi sigui mantenible, escalable i fàcil d'entendre. Una organització adequada del codi i l'adherència a les millors pràctiques són crucials per desenvolupar aplicacions Django robustes.

Estructura del Projecte

Una estructura de projecte ben organitzada és la base d'una aplicació Django mantenible.

  • Projecte Django vs. Aplicació Django:
    • Projecte: La configuració completa de Django, incloent-hi configuracions, URLs i configuracions.
    • Aplicació: Una aplicació web que fa alguna cosa, per exemple, un blog, un fòrum o una enquesta.

Exemple d'Estructura de Projecte

myproject/
    manage.py
    myproject/
        __init__.py
        settings.py
        urls.py
        wsgi.py
    myapp/
        __init__.py
        admin.py
        apps.py
        models.py
        tests.py
        views.py
        migrations/
            __init__.py
        templates/
            myapp/
                index.html
        static/
            myapp/
                style.css

Explicació

  • manage.py: Una utilitat de línia de comandes que et permet interactuar amb el teu projecte Django.
  • myproject/: El directori del projecte que conté configuracions i configuració.
  • myapp/: Un directori d'aplicació que conté models, vistes, plantilles i fitxers estàtics.

Millors Pràctiques

Separació del Codi

  • Models: Defineixen la teva estructura de dades.
  • Vistes: Gestionen la lògica per a les teves pàgines web.
  • Plantilles: Defineixen l'estructura HTML.
  • URLs: Dirigeixen les sol·licituds a la vista adequada.

Exemple

# models.py
from django.db import models

class Article(models.Model):
    title = models.CharField(maxlength=100)
    content = models.TextField()

# views.py
from django.shortcuts import render
from .models import Article

def article_list(request):
    articles = Article.objects.all()
    return render(request, 'myapp/article_list.html', {'articles': articles})

# urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('articles/', views.article_list, name='article_list'),
]

# templates/myapp/article_list.html
<!DOCTYPE html>
<html>
<head>
    <title>Llista d'Articles</title>
</head>
<body>
    <h1>Articles</h1>
    <ul>
        {% for article in articles %}
            <li>{{ article.title }}</li>
        {% endfor %}
    </ul>
</body>
</html>

Principi DRY (Don't Repeat Yourself)

  • Reutilitza el codi sempre que sigui possible.
  • Utilitza la herència de plantilles de Django per evitar duplicar HTML.

Exemple

<!-- templates/base.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}El Meu Lloc{% endblock %}</title>
</head>
<body>
    {% block content %}{% endblock %}
</body>
</html>

<!-- templates/myapp/article_list.html -->
{% extends "base.html" %}

{% block title %}Llista d'Articles{% endblock %}

{% block content %}
<h1>Articles</h1>
<ul>
    {% for article in articles %}
        <li>{{ article.title }}</li>
    {% endfor %}
</ul>
{% endblock %}

Ús d'Entorns Virtuals

  • Aïlla les dependències per a cada projecte.
  • Utilitza venv o virtualenv.

Exemple

# Crea un entorn virtual
python -m venv myenv

# Activa l'entorn virtual
# A Windows
myenv\Scripts\activate
# A Unix o MacOS
source myenv/bin/activate

# Instal·la les dependències
pip install django

Control de Versions

  • Utilitza Git per al control de versions.
  • Fes commits sovint amb missatges significatius.

Exemple

# Inicialitza un repositori Git
git init

# Afegeix fitxers al repositori
git add .

# Fes un commit dels canvis
git commit -m "Commit inicial"

Documentació

  • Documenta el teu codi amb comentaris i docstrings.
  • Mantén un fitxer README amb instruccions de configuració i una visió general del projecte.

Exemple

def article_list(request):
    """
    Mostra una llista d'articles.
    
    Args:
        request (HttpRequest): L'objecte de la sol·licitud.
    
    Returns:
        HttpResponse: L'objecte de la resposta que conté la plantilla renderitzada.
    """
    articles = Article.objects.all()
    return render(request, 'myapp/article_list.html', {'articles': articles})

Conclusió

Seguint aquestes millors pràctiques i organitzant el teu projecte Django de manera efectiva, pots crear aplicacions que siguin més fàcils de mantenir, escalar i entendre. Recorda mantenir el teu codi DRY, utilitzar entorns virtuals, control de versions i documentar el teu codi a fons. Aquestes pràctiques t'ajudaran a convertir-te en un desenvolupador Django més eficient i efectiu.
