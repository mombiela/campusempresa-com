Introducció a Models i Bases de Dades

A Django, els models són classes de Python que representen taules de bases de dades. Cada atribut del model correspon a un camp de la base de dades. El Mapeig Objecte-Relacional (ORM) de Django et permet interactuar amb la base de dades utilitzant codi Python en lloc d'escriure consultes SQL.

Creació de Models

Per definir un model, crees una classe que hereta de django.db.models.Model. Cada atribut de la classe representa un camp de la base de dades.

from django.db import models

class Author(models.Model):
    name = models.CharField(max_length=100)
    birth_date = models.DateField()

Conceptes Clau

  • Classe Model: Una classe de Python que representa una taula de la base de dades.
  • Tipus de Camps: Django proporciona diversos tipus de camps com CharField, DateField, IntegerField, etc.
  • Classe Meta: S'utilitza per definir metadades per al model, com ara l'ordenació i els noms descriptius.

Tipus de Camps

Django ofereix una varietat de tipus de camps per adaptar-se a diferents tipus de dades.

| Tipus de Camp | Descripció | |------------------|---------------------------------------| | CharField | Per a cadenes de text petites o grans | | TextField | Per a camps de text grans | | IntegerField | Per a valors enters | | FloatField | Per a nombres de punt flotant | | DateField | Per a valors de data | | DateTimeField | Per a valors de data i hora | | BooleanField | Per a valors vertader/fals | | ForeignKey | Per a crear relacions molts-a-un |

Exemple

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(Author, on_delete=models.CASCADE)
    published_date = models.DateField()
    price = models.FloatField()
    in_stock = models.BooleanField(default=True)

Migracions

Les migracions són la manera de Django de propagar els canvis que fas als teus models en l'esquema de la base de dades.

Crear i Aplicar Migracions

  1. Crear Migracions: Generar fitxers de migració basats en els canvis en els teus models. 
    python manage.py makemigrations
  2. Aplicar Migracions: Aplicar les migracions generades a la base de dades. 
    python manage.py migrate

Exemple

# Crear una nova migració basada en els canvis detectats als teus models
python manage.py makemigrations

# Aplicar la migració a la base de dades
python manage.py migrate

Consultar la Base de Dades

L'ORM de Django proporciona una API potent per consultar la base de dades.

Consultes Bàsiques

  • Tots els Objectes: Recuperar tots els objectes d'un model. 
    books = Book.objects.all()
  • Filtrar Objectes: Recuperar objectes que compleixen certs criteris. 
    books = Book.objects.filter(author__name='John Doe')
  • Obtenir un Sol Objecte: Recuperar un sol objecte que compleixi els criteris. 
    book = Book.objects.get(id=1)

Consultes Avançades

  • Encadenar Filtres: Combinar múltiples filtres. 
    books = Book.objects.filter(author__name='John Doe').filter(in_stock=True)
  • Excloure Objectes: Excloure objectes que compleixen certs criteris. 
    books = Book.objects.exclude(price__gt=20.00)

Exemple

# Recuperar tots els llibres
all_books = Book.objects.all()

# Recuperar llibres d'un autor específic
john_doe_books = Book.objects.filter(author__name='John Doe')

# Recuperar un sol llibre pel seu ID
book = Book.objects.get(id=1)

# Recuperar llibres que estan en estoc i són de John Doe
in_stock_books = Book.objects.filter(author__name='John Doe', in_stock=True)

# Excloure llibres que tenen un preu superior a $20
affordable_books = Book.objects.exclude(price__gt=20.00)

Conclusió

En aquesta secció, hem cobert els conceptes bàsics dels models i bases de dades de Django. Has après com crear models, definir camps i realitzar consultes bàsiques i avançades utilitzant l'ORM de Django. Entendre aquests conceptes és crucial per construir aplicacions web robustes i escalables amb Django.
