En aquesta secció, explorarem com definir models a Django. Els models són una part crucial de qualsevol aplicació Django ja que defineixen l'estructura de la teva base de dades. Cobrirem els conceptes bàsics de la creació de models, l'afegiment de camps i la comprensió dels mètodes del model.

Què és un Model?

Un model a Django és una classe de Python que representa una taula a la teva base de dades. Cada atribut de la classe correspon a un camp de la base de dades. Els models de Django proporcionen una capa d'abstracció que et permet interactuar amb la teva base de dades utilitzant codi Python en lloc de SQL cru.

Creació d'un Model Bàsic

Per crear un model, necessites definir una classe que hereti de django.db.models.Model. Comencem amb un exemple senzill:

from django.db import models

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.CharField(max_length=100)
    published_date = models.DateField()
    isbn = models.CharField(max_length=13)

Explicació:

  • title: Un CharField per emmagatzemar el títol del llibre. El paràmetre max_length especifica la longitud màxima de la cadena.
  • author: Un altre CharField per al nom de l'autor.
  • published_date: Un DateField per emmagatzemar la data de publicació.
  • isbn: Un CharField per al número ISBN del llibre.

Tipus de Camps

Django proporciona diversos tipus de camps per gestionar diferents tipus de dades. Aquí hi ha alguns dels tipus de camps més utilitzats:

| Tipus de Camp | Descripció | |-------------------|---------------------------------------------------| | CharField | Un camp de cadena, per a cadenes de mida petita a gran.| | TextField | Un camp de text gran. | | IntegerField | Un camp enter. | | FloatField | Un camp de número de punt flotant. | | BooleanField | Un camp de cert/fals. | | DateField | Un camp de data. | | DateTimeField | Un camp de data i hora. | | EmailField | Un camp per a adreces de correu electrònic. | | URLField | Un camp per a URLs. |

Afegir Mètodes als Models

Pots afegir mètodes als teus models per encapsular la lògica de negoci. Per exemple, podries voler afegir un mètode per obtenir el nom complet d'un autor:

class Author(models.Model):
    first_name = models.CharField(max_length=50)
    last_name = models.CharField(max_length=50)

    def full_name(self):
        return f"{self.first_name} {self.last_name}"

Explicació:

  • full_name: Aquest mètode concatena els camps first_name i last_name per retornar el nom complet de l'autor.

Opcions Meta

Els models de Django vénen amb una classe Meta que et permet especificar diverses opcions, com ara l'ordenació i els noms descriptius. Aquí tens un exemple:

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.CharField(max_length=100)
    published_date = models.DateField()
    isbn = models.CharField(max_length=13)

    class Meta:
        ordering = ['published_date']
        verbose_name = 'Llibre'
        verbose_name_plural = 'Llibres'

Explicació:

  • ordering: Especifica l'ordenació per defecte per al model. En aquest cas, els llibres s'ordenaran per la seva published_date.
  • verbose_name: Un nom llegible per a humans per al model.
  • verbose_name_plural: La forma plural del nom llegible per a humans.

Conclusió

En aquesta secció, hem cobert els conceptes bàsics de la definició de models a Django. Hem après com crear un model, afegir camps i utilitzar diversos tipus de camps. També hem explorat com afegir mètodes als models i utilitzar la classe Meta per a opcions addicionals. Entendre com definir models és essencial per treballar amb Django, ja que forma la base de l'estructura de dades de la teva aplicació.
