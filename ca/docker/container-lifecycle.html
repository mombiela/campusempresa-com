Introducció

En aquesta secció, explorarem el cicle de vida d'un contenidor Docker. Entendre el cicle de vida del contenidor és crucial per gestionar i orquestrar contenidors de manera efectiva. Cobrirà les diverses etapes per les quals passa un contenidor des de la creació fins a l'eliminació, juntament amb exemples pràctics i exercicis.

Conceptes clau

  • Imatge del contenidor: Un paquet executable lleuger, autònom, que inclou tot el necessari per executar un programa.
  • Contenidor: Una instància d'execució d'una imatge.
  • Docker Daemon: El servei en segon pla que s'executa a l'amfitrió i gestiona els contenidors Docker.
  • Docker CLI: Interfície de línia de comandes utilitzada per interactuar amb Docker Daemon.

Etapes del cicle de vida del contenidor

El cicle de vida d'un contenidor Docker es pot dividir en diverses etapes:

  1. Crear
  2. Iniciar
  3. Executar
  4. Pausar/Despausar
  5. Aturar
  6. Reiniciar
  7. Matar
  8. Eliminar

Crear

  • Descripció: El contenidor es crea a partir d'una imatge Docker però no s'inicia.
  • Comanda: docker create [OPTIONS] IMAGE [COMMAND] [ARG...]
  • Exemple: 
    docker create --name my_container ubuntu
  • Explicació: Aquesta comanda crea un contenidor anomenat my_container a partir de la imatge ubuntu.

Iniciar

  • Descripció: El contenidor s'inicia i comença a executar-se.
  • Comanda: docker start [OPTIONS] CONTAINER
  • Exemple: 
    docker start my_container
  • Explicació: Aquesta comanda inicia el my_container creat prèviament.

Executar

  • Descripció: Combina crear i iniciar en un sol pas.
  • Comanda: docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
  • Exemple: 
    docker run --name my_container -d ubuntu
  • Explicació: Aquesta comanda crea i inicia un contenidor anomenat my_container a partir de la imatge ubuntu en mode desconnectat (-d).

Pausar/Despausar

  • Descripció: Suspèn temporalment tots els processos dins del contenidor.
  • Comandes:
    • Pausar: docker pause CONTAINER
    • Despausar: docker unpause CONTAINER
  • Exemple: 
    docker pause my_container
docker unpause my_container
  • Explicació: Aquestes comandes pausen i despausen el my_container, respectivament.

Aturar

  • Descripció: Atura el contenidor de manera ordenada.
  • Comanda: docker stop [OPTIONS] CONTAINER
  • Exemple: 
    docker stop my_container
  • Explicació: Aquesta comanda atura el my_container de manera ordenada.

Reiniciar

  • Descripció: Atura i després inicia el contenidor.
  • Comanda: docker restart [OPTIONS] CONTAINER
  • Exemple: 
    docker restart my_container
  • Explicació: Aquesta comanda reinicia el my_container.

Matar

  • Descripció: Atura el contenidor de manera forçada.
  • Comanda: docker kill [OPTIONS] CONTAINER
  • Exemple: 
    docker kill my_container
  • Explicació: Aquesta comanda atura forçadament el my_container.

Eliminar

  • Descripció: Elimina el contenidor.
  • Comanda: docker rm [OPTIONS] CONTAINER
  • Exemple: 
    docker rm my_container
  • Explicació: Aquesta comanda elimina el my_container de l'amfitrió Docker.

Resum

Entendre el cicle de vida d'un contenidor Docker és essencial per a una gestió efectiva dels contenidors. Les etapes del cicle de vida inclouen crear, iniciar, executar, pausar, aturar, reiniciar, matar i eliminar contenidors. Cada etapa té comandes Docker específiques associades, que permeten controlar l'estat dels teus contenidors. En dominar aquestes comandes, podràs gestionar eficientment les teves aplicacions contenidoritzades.

Exercicis

  1. Crear i iniciar un contenidor:

    • Crea un contenidor a partir de la imatge nginx i inicia'l.
    docker create --name web_server nginx
docker start web_server

  2. Pausar i despausar un contenidor:

    • Pausa i després despausa el contenidor web_server.
    docker pause web_server
docker unpause web_server

  3. Aturar i eliminar un contenidor:

    • Atura i després elimina el contenidor web_server.
    docker stop web_server
docker rm web_server

En practicar aquests exercicis, obtindràs experiència pràctica amb el cicle de vida dels contenidors Docker, reforçant la teva comprensió de cada etapa i les seves comandes associades.
