Introducció a Dockerfile

Un Dockerfile és un document de text que conté totes les ordres que un usuari podria cridar a la línia de comandes per muntar una imatge. Utilitzant docker build, els usuaris poden crear una construcció automatitzada que executa diverses instruccions de la línia de comandes successivament.

Conceptes clau de Dockerfile

  • FROM: Estableix la imatge base per a les instruccions següents.
  • RUN: Executa una ordre a la shell.
  • CMD: Proporciona l'ordre per defecte per al contenidor.
  • LABEL: Afegeix metadades a una imatge.
  • EXPOSE: Informa Docker que el contenidor escolta als ports de xarxa especificats en temps d'execució.
  • ENV: Estableix variables d'entorn.
  • ADD: Copia nous fitxers, directoris o URLs de fitxers remots des de src i els afegeix al sistema de fitxers del contenidor al camí dest.
  • COPY: Copia nous fitxers o directoris des de src i els afegeix al sistema de fitxers del contenidor al camí dest.
  • ENTRYPOINT: Configura un contenidor que s'executarà com un executable.
  • VOLUME: Crea un punt de muntatge amb el camí especificat i el marca com a contenidor de volums muntats externament des de l'amfitrió natiu o altres contenidors.
  • USER: Estableix l'UID (o nom d'usuari) a utilitzar quan s'executa la imatge i per a qualsevol instrucció RUN, CMD i ENTRYPOINT que la segueixi al Dockerfile.
  • WORKDIR: Estableix el directori de treball per a qualsevol instrucció RUN, CMD, ENTRYPOINT, COPY i ADD que la segueixi al Dockerfile.

Exemple bàsic de Dockerfile

Creem un Dockerfile senzill per a una aplicació Node.js.

# Utilitza una imatge oficial de Node.js com a imatge base
FROM node:14

# Estableix el directori de treball dins del contenidor
WORKDIR /usr/src/app

# Copia els fitxers package.json i package-lock.json al directori de treball
COPY package*.json ./

# Instal·la els paquets necessaris especificats a package.json
RUN npm install

# Copia la resta del codi de l'aplicació al directori de treball
COPY . .

# Fa que el port 8080 estigui disponible per al món fora d'aquest contenidor
EXPOSE 8080

# Defineix la variable d'entorn
ENV NODE_ENV=production

# Executa l'aplicació
CMD ["node", "app.js"]

Explicació del Dockerfile

  • FROM node:14: Aquesta línia especifica la imatge base. En aquest cas, és la imatge oficial de Node.js amb l'etiqueta 14.
  • WORKDIR /usr/src/app: Això estableix el directori de treball dins del contenidor a /usr/src/app.
  • COPY package.json ./*: Això copia els fitxers package.json i package-lock.json al directori de treball.
  • RUN npm install: Això executa l'ordre npm install per instal·lar les dependències.
  • COPY . .: Això copia la resta del codi de l'aplicació al directori de treball.
  • EXPOSE 8080: Això exposa el port 8080 al món exterior.
  • ENV NODE_ENV=production: Això estableix una variable d'entorn NODE_ENV a production.
  • CMD ["node", "app.js"]: Això especifica l'ordre per executar l'aplicació.

Instruccions comunes de Dockerfile

Aquí hi ha una taula que resumeix algunes instruccions comunes de Dockerfile i els seus propòsits:

| Instrucció | Descripció | |-------------|-------------| | FROM | Especifica la imatge base. | | RUN | Executa una ordre a la shell. | | CMD | Proporciona l'ordre per defecte per al contenidor. | | LABEL | Afegeix metadades a una imatge. | | EXPOSE | Informa Docker que el contenidor escolta als ports de xarxa especificats. | | ENV | Estableix variables d'entorn. | | ADD | Copia nous fitxers, directoris o URLs de fitxers remots al contenidor. | | COPY | Copia nous fitxers o directoris al contenidor. | | ENTRYPOINT| Configura un contenidor que s'executarà com un executable. | | VOLUME | Crea un punt de muntatge amb el camí especificat. | | USER | Estableix l'UID (o nom d'usuari) a utilitzar quan s'executa la imatge. | | WORKDIR | Estableix el directori de treball per a les instruccions següents. |

Conclusió

Entendre els conceptes bàsics de Dockerfile és crucial per crear imatges Docker eficients i efectives. Dominant les instruccions clau i els seus propòsits, pots automatitzar el procés de construcció i assegurar-te que les teves aplicacions s'executin de manera consistent en diferents entorns. Practica escrivint Dockerfiles per a diferents aplicacions per obtenir una millor comprensió d'aquests conceptes.
