Introducció a Docker Swarm

Docker Swarm és una eina nativa de clustering i orquestració per a contenidors Docker. Et permet gestionar un clúster de nodes Docker com un sistema virtual únic.

Conceptes Clau

  • Swarm: Un grup de màquines que executen Docker i s'han unit en un clúster.
  • Node: Una màquina individual en un Swarm.
    • Node Manager: Responsable de gestionar el Swarm i els seus recursos.
    • Node Worker: Executa tasques assignades pel Manager.
  • Servei: Una definició de com executar un contenidor, incloent la imatge, ports i rèpliques.
  • Tasca: Una instància única d'un contenidor en execució que forma part d'un servei.
  • Xarxa Overlay: Una xarxa que s'estén per tots els nodes del Swarm, permetent que els contenidors es comuniquin de manera segura.

Configuració d'un Docker Swarm

Per configurar un Docker Swarm, necessites almenys una màquina per actuar com a Node Manager i una o més Nodes Worker.

Guia Pas a Pas

  1. Inicialitza el Swarm al Node Manager:

    docker swarm init --advertise-addr <MANAGER-IP>

    Substitueix <MANAGER-IP> per l'adreça IP del teu Node Manager.

  2. Uneix Nodes Worker al Swarm: En cada Node Worker, executa:

    docker swarm join --token <SWARM-TOKEN> <MANAGER-IP>:2377

    Substitueix <SWARM-TOKEN> pel token proporcionat per la comanda docker swarm init i <MANAGER-IP> per l'adreça IP del Node Manager.

Gestió de Serveis en Docker Swarm

Un cop el teu Swarm està configurat, pots començar a crear i gestionar serveis.

Creació d'un Servei

Per crear un servei, utilitza la comanda docker service create:

docker service create --name my-service --replicas 3 -p 80:80 nginx

Aquesta comanda crea un servei anomenat my-service amb 3 rèpliques del contenidor nginx, exposant el port 80.

Llistat de Serveis

Per llistar tots els serveis en el Swarm:

docker service ls

Escalat d'un Servei

Per escalar un servei cap amunt o cap avall:

docker service scale my-service=5

Aquesta comanda escala my-service a 5 rèpliques.

Actualització d'un Servei

Per actualitzar un servei (per exemple, canviar la versió de la imatge):

docker service update --image nginx:latest my-service

Monitorització i Resolució de Problemes

La monitorització i resolució de problemes són crucials per mantenir un Swarm saludable.

Visualització de Registres de Servei

Per veure els registres d'un servei específic:

docker service logs my-service

Inspecció de Nodes

Per inspeccionar els detalls d'un node:

docker node inspect <NODE-ID>

Eliminació d'un Node

Per eliminar un node del Swarm:

docker node rm <NODE-ID>

Conclusió

Docker Swarm proporciona una manera potent i senzilla de gestionar aplicacions contenidoritzades a escala. Entenent els conceptes clau i les comandes, pots orquestrar i gestionar efectivament els teus contenidors Docker a través de múltiples nodes. Tant si estàs desplegant un servei web senzill com una arquitectura complexa de microserveis, Docker Swarm ofereix les eines que necessites per assegurar que les teves aplicacions funcionin de manera suau i eficient.
