Persistència de Dades amb Volums
Introducció a Docker Compose ►

En aquesta secció, explorarem com Docker gestiona la persistència de dades utilitzant volums. Això és crucial per mantenir les dades entre reinicis de contenidors i compartir dades entre contenidors.

Introducció a la Persistència de Dades

La persistència de dades es refereix a la capacitat de guardar dades més enllà del cicle de vida d'un contenidor. Per defecte, les dades creades dins d'un contenidor Docker són efímeres, és a dir, es perden quan el contenidor s'atura o s'elimina. Per superar això, Docker proporciona volums com a solució.

Tipus d'Emmagatzematge Docker

Docker ofereix diverses maneres de gestionar la persistència de dades:

  • Volums: Gestionats per Docker i emmagatzemats en una part del sistema de fitxers de l'amfitrió.
  • Bind Mounts: Munten directament un directori o fitxer de l'amfitrió en un contenidor.
  • tmpfs Mounts: Emmagatzemen dades en la memòria del sistema de l'amfitrió.

En aquesta secció, ens centrarem en els volums, que són el mètode més flexible i preferit per a la persistència de dades.

Creació i Ús de Volums

Creació d'un Volum

Per crear un volum, utilitza la comanda docker volume create:

docker volume create my_volume

Aquesta comanda crea un volum anomenat my_volume que Docker gestiona.

Adjuntar un Volum a un Contenidor

Per adjuntar un volum a un contenidor, utilitza el flag -v o --mount amb la comanda docker run:

docker run -d --name my_container -v my_volume:/data busybox

En aquest exemple:

  • -d executa el contenidor en mode desconnectat.
  • --name my_container dóna un nom al contenidor.
  • -v my_volume:/data munta el volum my_volume al directori /data dins del contenidor.
  • busybox és la imatge utilitzada per crear el contenidor.

Inspecció de Volums

Per inspeccionar un volum i veure els seus detalls, utilitza la comanda docker volume inspect:

docker volume inspect my_volume

Aquesta comanda proporciona informació sobre el volum, com ara el seu punt de muntatge en l'amfitrió.

Compartir Dades Entre Contenidors

Els volums es poden compartir entre diversos contenidors, permetent-los llegir i escriure a les mateixes dades.

docker run -d --name container1 -v shared_volume:/data busybox
docker run -d --name container2 -v shared_volume:/data busybox

Tant container1 com container2 poden ara accedir a les dades emmagatzemades en shared_volume.

Fer Còpies de Seguretat i Restaurar Volums

Fer una Còpia de Seguretat d'un Volum

Per fer una còpia de seguretat d'un volum, pots utilitzar un contenidor temporal per copiar les dades a un arxiu tar:

docker run --rm -v my_volume:/data -v $(pwd):/backup busybox tar cvf /backup/backup.tar /data

Restaurar un Volum

Per restaurar un volum des d'una còpia de seguretat, utilitza un enfocament similar:

docker run --rm -v my_volume:/data -v $(pwd):/backup busybox tar xvf /backup/backup.tar -C /

Millors Pràctiques per Utilitzar Volums

  • Utilitza Volums Nomenats: Prefereix sempre els volums nomenats sobre els volums anònims per a una millor gestió.
  • Fes Còpies de Seguretat Regularment: Fes còpies de seguretat regularment dels teus volums per evitar la pèrdua de dades.
  • Monitoritza l'Ús del Volum: Mantén un ull sobre l'espai de disc utilitzat pels volums per evitar quedar-te sense espai.

Conclusió

Els volums són una característica poderosa de Docker que proporcionen una manera fiable de persistir dades més enllà del cicle de vida d'un contenidor. Entenent com crear, utilitzar i gestionar volums, pots assegurar-te que les teves dades romanguin segures i accessibles, fins i tot quan els contenidors s'aturen i es reinicien. Recorda seguir les millors pràctiques per a la gestió de volums per mantenir un entorn Docker robust i eficient.
