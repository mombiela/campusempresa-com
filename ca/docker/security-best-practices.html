Introducció

Docker és una eina poderosa per a la contenidorització, però amb un gran poder ve una gran responsabilitat. Assegurar la seguretat del teu entorn Docker és crucial per protegir les teves aplicacions i dades. Aquesta secció cobreix les millors pràctiques per assegurar Docker des de nivells bàsics fins a avançats.

Pràctiques Bàsiques de Seguretat

Utilitza Imatges Oficials

  • Imatges Oficials: Sempre utilitza imatges oficials de Docker Hub o altres fonts de confiança per minimitzar el risc de vulnerabilitats.
  • Verificació: Verifica l'autenticitat de les imatges utilitzant Docker Content Trust (DCT).
# Activa Docker Content Trust
export DOCKER_CONTENT_TRUST=1

# Descarrega una imatge amb DCT activat
docker pull ubuntu:latest

Mantén Docker Actualitzat

  • Actualitzacions: Actualitza Docker regularment a la versió més recent per beneficiar-te dels pegats de seguretat i millores.
  • Notes de Lliurament: Revisa les notes de lliurament de Docker per a qualsevol actualització relacionada amb la seguretat.
# Actualitza Docker en un sistema basat en Debian
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io

Limita els Privilegis del Contenidor

  • Menys Privilegis: Executa els contenidors amb els privilegis mínims necessaris.
  • Usuari No Root: Evita executar contenidors com a usuari root.
# Exemple de Dockerfile per executar com a usuari no root
FROM ubuntu:latest
RUN useradd -m myuser
USER myuser

Pràctiques Intermèdies de Seguretat

Seguretat de Xarxa

  • Aïlla els Contenidors: Utilitza xarxes Docker per aïllar els contenidors i controlar la comunicació.
  • Regles de Tallafocs: Implementa regles de tallafocs per restringir l'accés al dimoni Docker i als contenidors.
# Crea una xarxa aïllada
docker network create --driver bridge isolated_network

# Executa un contenidor a la xarxa aïllada
docker run -d --network isolated_network myapp

Assegura el Dimoni Docker

  • Xifratge TLS: Utilitza TLS per xifrar la comunicació entre el client Docker i el dimoni.
  • Configuració del Dimoni: Configura el dimoni Docker per escoltar en una IP i port específics.
// Exemple de configuració de daemon.json
{
  "tls": true,
  "tlscacert": "/path/to/ca.pem",
  "tlscert": "/path/to/server-cert.pem",
  "tlskey": "/path/to/server-key.pem",
  "hosts": ["tcp://192.168.1.100:2376", "unix:///var/run/docker.sock"]
}

Escaneig d'Imatges

  • Escaneig de Vulnerabilitats: Utilitza eines com Docker Scan o serveis de tercers per escanejar imatges en cerca de vulnerabilitats.
# Escaneja una imatge per vulnerabilitats
docker scan myimage:latest

Pràctiques Avançades de Seguretat

Seguretat en Temps d'Execució

  • Perfils de Seguretat: Utilitza perfils AppArmor o SELinux per fer complir polítiques de seguretat en els contenidors.
  • Seccomp: Utilitza seccomp per limitar les crides al sistema que un contenidor pot fer.
# Executa un contenidor amb un perfil seccomp
docker run --security-opt seccomp=/path/to/seccomp-profile.json myapp

Gestió de Secrets

  • Secrets de Docker: Utilitza secrets de Docker per gestionar dades sensibles com contrasenyes i claus API de manera segura.
# Crea un secret de Docker
echo "mysecretpassword" | docker secret create db_password -

# Utilitza el secret en un servei
docker service create --name myservice --secret db_password myimage

Auditoria i Monitorització

  • Registres d'Auditoria: Activa i revisa els registres d'auditoria de Docker per monitoritzar activitats.
  • Eines de Monitorització: Utilitza eines de monitorització com Prometheus i Grafana per vigilar les mètriques dels contenidors i esdeveniments de seguretat.
# Exemple d'activació de registres d'auditoria en daemon.json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}

Resum

Assegurar Docker implica un enfocament de múltiples capes, des d'utilitzar imatges oficials i mantenir Docker actualitzat fins a implementar mesures avançades de seguretat en temps d'execució i monitorització. Seguint aquestes millors pràctiques, pots reduir significativament el risc de violacions de seguretat i assegurar un entorn Docker robust i segur.
