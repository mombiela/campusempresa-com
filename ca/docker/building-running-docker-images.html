Introducció

En aquesta secció, cobrirem els conceptes fonamentals de construir i executar imatges Docker. Les imatges Docker són la base dels contenidors, i entendre com crear-les i gestionar-les és crucial per a qualsevol usuari de Docker.

Conceptes Clau

  • Dockerfile: Un document de text que conté totes les comandes per a muntar una imatge.
  • Imatge: Un paquet executable lleuger, autònom, que inclou tot el necessari per executar una peça de programari.
  • Contenidor: Una instància d'execució d'una imatge.

Construir Imatges Docker

Escriure un Dockerfile

Un Dockerfile és un script que conté una sèrie d'instruccions sobre com construir una imatge Docker. Aquí hi ha un exemple senzill:

# Utilitza una imatge oficial de Python com a imatge pare
FROM python:3.8-slim

# Estableix el directori de treball en el contenidor
WORKDIR /app

# Copia el contingut del directori actual al contenidor a /app
COPY . /app

# Instal·la qualsevol paquet necessari especificat a requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Fa que el port 80 estigui disponible per al món fora d'aquest contenidor
EXPOSE 80

# Defineix una variable d'entorn
ENV NAME World

# Executa app.py quan el contenidor es llanci
CMD ["python", "app.py"]

Construir la Imatge

Per construir una imatge a partir d'un Dockerfile, utilitza la comanda docker build. Aquí tens la sintaxi:

docker build -t my-python-app .
  • -t my-python-app: Etiqueta la imatge amb el nom my-python-app.
  • .: Especifica el context de construcció, que és el directori actual.

Entendre el Procés de Construcció

Quan executes docker build, Docker realitza els següents passos:

  1. Llegeix el Dockerfile.
  2. Executa cada instrucció del Dockerfile.
  3. Crea una nova capa d'imatge per a cada instrucció.
  4. Etiqueta la imatge final amb el nom especificat.

Executar Imatges Docker

Executar un Contenidor

Per executar un contenidor a partir d'una imatge, utilitza la comanda docker run. Aquí tens un exemple:

docker run -p 4000:80 my-python-app
  • -p 4000:80: Assigna el port 4000 de l'amfitrió al port 80 del contenidor.
  • my-python-app: El nom de la imatge a executar.

Gestionar Contenidors

  • Llistar contenidors en execució: docker ps
  • Aturar un contenidor: docker stop <container_id>
  • Eliminar un contenidor: docker rm <container_id>

Exemple: Executar una Aplicació Web

Suposem que tens una aplicació web Flask senzilla. Aquí tens com pots construir-la i executar-la utilitzant Docker:

  1. Crear un Dockerfile:

    FROM python:3.8-slim
WORKDIR /app
COPY . /app
RUN pip install --no-cache-dir -r requirements.txt
EXPOSE 5000
CMD ["python", "app.py"]

  2. Construir la imatge:

    docker build -t flask-app .

  3. Executar el contenidor:

    docker run -p 5000:5000 flask-app

Conclusió

En aquesta secció, hem cobert els conceptes bàsics de construir i executar imatges Docker. Hem après com escriure un Dockerfile, construir una imatge i executar un contenidor. Aquestes habilitats són fonamentals per treballar amb Docker i desplegar aplicacions en un entorn de contenidors. A mesura que progressis, aprendràs tècniques més avançades per optimitzar i gestionar imatges i contenidors Docker.
