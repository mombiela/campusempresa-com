Introducció

En aquest projecte, construirem una pipeline d'Integració Contínua i Desplegament Continu (CI/CD) utilitzant Docker. Aquest projecte t'ajudarà a entendre com automatitzar la construcció, prova i desplegament d'aplicacions utilitzant contenidors Docker. Al final d'aquest projecte, tindràs una pipeline CI/CD funcional que es pot integrar amb qualsevol aplicació.

Conceptes Clau

  • Integració Contínua (CI): La pràctica d'integrar automàticament els canvis de codi de múltiples contribuïdors en un repositori compartit diverses vegades al dia.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa les proves automatitzades a producció.
  • Docker: Una plataforma per desenvolupar, enviar i executar aplicacions en contenidors.
  • Dockerfile: Un document de text que conté totes les comandes per muntar una imatge.
  • Docker Compose: Una eina per definir i executar aplicacions Docker de múltiples contenidors.

Prerequisits

  • Comprensió bàsica de Docker i Docker Compose.
  • Familiaritat amb sistemes de control de versions com Git.
  • Coneixement bàsic de scripting de shell.

Configuració de l'Entorn

  1. Instal·lar Docker: Segueix la guia d'instal·lació oficial de Docker per al teu sistema operatiu.

  2. Instal·lar Docker Compose: Segueix la guia d'instal·lació oficial de Docker Compose per al teu sistema operatiu.

  3. Configurar un Repositori Git: Crea un nou repositori a GitHub o qualsevol altre servei d'allotjament Git.

Creació d'una Aplicació Simple

Creem una aplicació simple de Node.js que utilitzarem per a la nostra pipeline CI/CD.

  1. Inicialitzar un Projecte Node.js:

    mkdir my-node-app
cd my-node-app
npm init -y

  2. Crear un Fitxer index.js:

    const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(port, () => {
  console.log(`App listening at http://localhost:${port}`);
});

  3. Instal·lar Express:

    npm install express

Escriure un Dockerfile

Crea un Dockerfile a l'arrel del teu directori de projecte.

# Utilitza la imatge oficial de Node.js.
FROM node:14

# Crea i canvia al directori de l'aplicació.
WORKDIR /usr/src/app

# Copia els manifests de dependències de l'aplicació a la imatge del contenidor.
COPY package*.json ./

# Instal·la les dependències.
RUN npm install

# Copia el codi local a la imatge del contenidor.
COPY . .

# Executa el servei web a l'inici del contenidor.
CMD [ "node", "index.js" ]

# Exposa el port en què s'executa l'aplicació.
EXPOSE 3000

Crear un Fitxer Docker Compose

Crea un fitxer docker-compose.yml a l'arrel del teu directori de projecte.

version: '3.8'

services:
  web:
    build: .
    ports:
      - "3000:3000"

Configuració de CI amb GitHub Actions

Crea un fitxer .github/workflows/ci.yml al teu repositori.

name: CI

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v1

      - name: Build the Docker image
        run: docker build . --file Dockerfile --tag my-node-app:latest

      - name: Run tests
        run: docker run my-node-app:latest npm test

Configuració de CD amb GitHub Actions

Amplia el fitxer .github/workflows/ci.yml per incloure els passos de desplegament.

name: CI/CD

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v1

      - name: Build the Docker image
        run: docker build . --file Dockerfile --tag my-node-app:latest

      - name: Run tests
        run: docker run my-node-app:latest npm test

  deploy:
    runs-on: ubuntu-latest
    needs: build

    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Deploy to Docker Hub
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin
          docker tag my-node-app:latest my-dockerhub-username/my-node-app:latest
          docker push my-dockerhub-username/my-node-app:latest

Conclusió

En aquest projecte, hem configurat amb èxit una pipeline CI/CD utilitzant Docker i GitHub Actions. Hem creat una aplicació simple de Node.js, hem escrit un Dockerfile per conteniritzar l'aplicació i hem utilitzat Docker Compose per gestionar el contenidor. Després, hem configurat GitHub Actions per automatitzar la construcció, prova i desplegament del nostre contenidor Docker. Aquesta pipeline CI/CD es pot adaptar per ajustar-se a qualsevol aplicació, proporcionant un flux de treball robust i automatitzat per a la integració i desplegament continus.
