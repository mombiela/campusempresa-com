Introducció

La còpia de seguretat i la recuperació són aspectes crucials de la gestió de contenidors Docker i les seves dades. Aquesta secció et guiarà a través dels conceptes, eines i millors pràctiques per fer còpies de seguretat i recuperar efectivament els contenidors Docker i les seves dades associades.

Conceptes clau

  • Còpia de seguretat: El procés de crear còpies de dades per protegir-les contra la pèrdua.
  • Recuperació: El procés de restaurar dades des de les còpies de seguretat al seu estat original o a un estat funcional.
  • Volums: El mecanisme preferit de Docker per persistir dades generades i utilitzades pels contenidors Docker.
  • Bind Mounts: Una manera de muntar un directori o fitxer de la màquina host dins d'un contenidor.

Estratègies de còpia de seguretat

Còpies de seguretat de volums

Els volums són el mètode més comú per persistir dades a Docker. Fer còpies de seguretat dels volums assegura que pots restaurar l'estat de les teves dades fins i tot si el contenidor és eliminat.

Exemple: Fer una còpia de seguretat d'un volum

  1. Identificar el volum: Utilitza el comandament docker volume ls per llistar tots els volums.

    docker volume ls

  2. Crear una còpia de seguretat: Utilitza el comandament docker run per crear un tarball del contingut del volum.

    docker run --rm -v my_volume:/volume -v $(pwd):/backup busybox tar cvf /backup/backup.tar /volume
    • my_volume: El nom del volum a fer la còpia de seguretat.
    • $(pwd): El directori actual a l'host on es guardarà la còpia de seguretat.
    • busybox: Una imatge Docker lleugera utilitzada per crear el tarball.

Còpies de seguretat de contenidors

Fer una còpia de seguretat de tot el contenidor inclou tant el sistema de fitxers del contenidor com la seva configuració.

Exemple: Fer una còpia de seguretat d'un contenidor

  1. Cometre el contenidor: Crear una nova imatge des de l'estat actual del contenidor.

    docker commit my_container my_backup_image
    • my_container: El nom o ID del contenidor.
    • my_backup_image: El nom de la nova imatge.

  2. Guardar la imatge: Exportar la imatge a un tarball.

    docker save -o my_backup_image.tar my_backup_image
    • my_backup_image.tar: El nom del fitxer tarball.
    • my_backup_image: El nom de la imatge a guardar.

Estratègies de recuperació

Recuperació de volums

Restaurar dades des d'una còpia de seguretat de volum implica extreure el tarball de nou en un volum Docker.

Exemple: Restaurar un volum

  1. Crear un nou volum: Si el volum original es perd, crea'n un de nou.

    docker volume create my_volume

  2. Restaurar la còpia de seguretat: Utilitza el comandament docker run per extreure el tarball en el volum.

    docker run --rm -v my_volume:/volume -v $(pwd):/backup busybox tar xvf /backup/backup.tar -C /
    • my_volume: El nom del volum a restaurar.
    • $(pwd): El directori a l'host on es troba el tarball de la còpia de seguretat.
    • busybox: Una imatge Docker lleugera utilitzada per extreure el tarball.

Recuperació de contenidors

Restaurar un contenidor des d'una còpia de seguretat implica carregar la imatge guardada i crear un nou contenidor a partir d'ella.

Exemple: Restaurar un contenidor

  1. Carregar la imatge: Importar el tarball de la imatge guardada.

    docker load -i my_backup_image.tar
    • my_backup_image.tar: El nom del fitxer tarball.

  2. Executar un nou contenidor: Crear un nou contenidor a partir de la imatge restaurada.

    docker run -d --name my_restored_container my_backup_image
    • my_restored_container: El nom del nou contenidor.
    • my_backup_image: El nom de la imatge restaurada.

Millors pràctiques

  • Còpies de seguretat regulars: Programa còpies de seguretat regulars per minimitzar la pèrdua de dades.
  • Automatitzar les còpies de seguretat: Utilitza scripts o eines per automatitzar el procés de còpia de seguretat.
  • Provar les restauracions: Prova regularment el teu procés de recuperació per assegurar-te que les còpies de seguretat són vàlides.
  • Emmagatzemar les còpies de seguretat de manera segura: Guarda les còpies de seguretat en un lloc segur i redundant.

Conclusió

La còpia de seguretat i la recuperació són pràctiques essencials per mantenir la integritat i disponibilitat dels teus entorns Docker. Entenent i implementant les estratègies discutides en aquesta secció, pots assegurar-te que les teves dades estan protegides i poden ser ràpidament restaurades en cas de fallada. Prova regularment els teus processos de còpia de seguretat i recuperació per assegurar-te que funcionen com s'espera i mantenen les teves dades segures.
