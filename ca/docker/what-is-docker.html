Introducció

Docker és una plataforma de codi obert que automatitza el desplegament, l'escalat i la gestió d'aplicacions. Utilitza la tecnologia de contenidors per empaquetar una aplicació i les seves dependències en una unitat estandarditzada anomenada contenidor. Això assegura que l'aplicació s'executi de manera consistent en diferents entorns.

Conceptes Clau

  • Contenidors: Paquets lleugers, autònoms i executables que inclouen tot el necessari per executar un programari, incloent el codi, el temps d'execució, les biblioteques i les eines del sistema.
  • Imatges: Plantilles de només lectura utilitzades per crear contenidors. Una imatge inclou el codi de l'aplicació, les biblioteques i les dependències.
  • Docker Engine: La part central de Docker, que és responsable de crear i executar contenidors Docker.
  • Docker Hub: Un repositori basat en el núvol on els usuaris de Docker poden crear, provar, emmagatzemar i distribuir imatges de contenidors.

Per què utilitzar Docker?

  • Consistència: Assegura que les aplicacions s'executin de la mateixa manera en entorns de desenvolupament, prova i producció.
  • Aïllament: Cada contenidor s'executa en el seu propi entorn aïllat, cosa que ajuda a evitar conflictes entre diferents aplicacions.
  • Eficiència: Els contenidors comparteixen el nucli i els recursos del sistema hoste, fent-los més lleugers i ràpids d'iniciar en comparació amb les màquines virtuals.
  • Portabilitat: Els contenidors poden executar-se en qualsevol sistema que suporti Docker, facilitant el moviment d'aplicacions entre diferents entorns.

Flux de treball bàsic de Docker

  1. Escriure un Dockerfile: Un fitxer de text que conté instruccions sobre com construir una imatge Docker.
  2. Construir la Imatge: Utilitzar el comandament docker build per crear una imatge a partir del Dockerfile.
  3. Executar el Contenidor: Utilitzar el comandament docker run per crear i iniciar un contenidor a partir de la imatge.
  4. Gestionar Contenidors: Utilitzar diversos comandaments Docker per gestionar contenidors en execució.

Exemple: Creació d'un Contenidor Docker Simple

Pas 1: Escriure un Dockerfile

Crear un fitxer anomenat Dockerfile amb el següent contingut:

# Utilitzar una imatge oficial de Python com a imatge pare
FROM python:3.8-slim

# Establir el directori de treball en el contenidor
WORKDIR /app

# Copiar el contingut del directori actual al contenidor a /app
COPY . /app

# Instal·lar qualsevol paquet necessari especificat a requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Fer que el port 80 estigui disponible per al món fora d'aquest contenidor
EXPOSE 80

# Definir una variable d'entorn
ENV NAME World

# Executar app.py quan el contenidor s'iniciï
CMD ["python", "app.py"]

Pas 2: Construir la Imatge Docker

Executar el següent comandament al terminal:

docker build -t my-python-app .

Pas 3: Executar el Contenidor Docker

Executar el següent comandament per iniciar un contenidor a partir de la imatge:

docker run -p 4000:80 my-python-app

Pas 4: Accedir a l'Aplicació

Obrir un navegador web i navegar a http://localhost:4000 per veure la vostra aplicació en execució.

Conclusió

Docker és una eina poderosa que simplifica el procés de desenvolupament, prova i desplegament d'aplicacions mitjançant la contenedorització. Assegura consistència en diferents entorns, proporciona aïllament i millora l'eficiència i la portabilitat. Entenent els conceptes bàsics i el flux de treball, podeu començar a aprofitar Docker per agilitzar els vostres processos de desenvolupament i desplegament.
