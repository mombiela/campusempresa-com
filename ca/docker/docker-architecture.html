Introducció

Docker és una plataforma que permet als desenvolupadors crear, desplegar i executar aplicacions en contenidors. Entendre l'arquitectura de Docker és crucial per utilitzar Docker de manera efectiva en els teus entorns de desenvolupament i producció. Aquesta secció cobrirà els components i conceptes clau de l'arquitectura de Docker.

Components Clau de l'Arquitectura de Docker

Client Docker

  • Definició: El client Docker és la interfície principal de Docker. Accepta ordres de l'usuari i es comunica amb el dimoni Docker.
  • Ordres: Les ordres comunes inclouen docker build, docker pull i docker run.
  • Exemple: 
    docker run hello-world
    Aquesta ordre diu al client Docker que executi el contenidor hello-world.

Dimoni Docker

  • Definició: El dimoni Docker (dockerd) escolta les sol·licituds de l'API de Docker i gestiona objectes Docker com imatges, contenidors, xarxes i volums.
  • Funció: Realitza la feina pesada de construir, executar i distribuir contenidors Docker.
  • Exemple: El dimoni es pot iniciar amb: 
    sudo dockerd

Imatges Docker

  • Definició: Les imatges Docker són plantilles de només lectura utilitzades per crear contenidors. Contenen el codi de l'aplicació, biblioteques, dependències i altres fitxers necessaris per executar l'aplicació.
  • Capes: Les imatges es construeixen en capes, amb cada capa representant un conjunt de canvis de fitxers.
  • Exemple: 
    FROM ubuntu:latest
RUN apt-get update && apt-get install -y python3
    Aquest Dockerfile crea una imatge amb Python 3 instal·lat a sobre de la imatge més recent d'Ubuntu.

Contenidors Docker

  • Definició: Els contenidors són instàncies executables d'imatges Docker. Són entorns aïllats que executen aplicacions.
  • Cicle de Vida: Els contenidors es poden crear, iniciar, aturar, moure i eliminar.
  • Exemple: 
    docker run -it ubuntu:latest /bin/bash
    Aquesta ordre executa un contenidor a partir de la imatge més recent d'Ubuntu i obre una shell bash dins d'ell.

Registre Docker

  • Definició: Un registre Docker és un sistema d'emmagatzematge i distribució d'imatges Docker. El registre per defecte és Docker Hub.
  • Registres Privats: Les organitzacions també poden configurar registres privats per emmagatzemar i gestionar les seves imatges.
  • Exemple: 
    docker pull ubuntu:latest
    Aquesta ordre descarrega la imatge més recent d'Ubuntu des de Docker Hub.

Conclusió

Entendre l'arquitectura de Docker és fonamental per utilitzar Docker de manera efectiva. El client Docker, el dimoni, les imatges, els contenidors i el registre juguen cadascun un paper crucial en l'ecosistema Docker. Dominant aquests components, podràs construir, desplegar i gestionar aplicacions contenidoritzades de manera eficient.

En les següents seccions, aprofundirem en cadascun d'aquests components, proporcionant explicacions més detallades, exemples i exercicis per solidificar la teva comprensió.
