Què és Docker Compose?

Docker Compose és una eina per definir i executar aplicacions Docker de múltiples contenidors. Et permet gestionar múltiples contenidors com un únic servei, facilitant l'orquestració d'aplicacions complexes.

Conceptes Clau

  • Servei: Un servei defineix un contenidor en el context de Docker Compose. Especifica la imatge a utilitzar, els ports a exposar i altres configuracions.
  • Volum: Els volums s'utilitzen per persistir dades generades i utilitzades pels contenidors Docker.
  • Xarxa: Les xarxes permeten la comunicació entre contenidors Docker. Docker Compose crea automàticament una xarxa per defecte perquè els serveis es comuniquin.

Per què utilitzar Docker Compose?

  • Configuració Simplificada: Defineix aplicacions de múltiples contenidors amb un únic fitxer YAML.
  • Gestió Fàcil: Inicia, atura i reconstrueix serveis amb comandes senzilles.
  • Consistència d'Entorn: Assegura que els teus entorns de desenvolupament, preproducció i producció siguin consistents.

Instal·lant Docker Compose

Docker Compose està inclòs a Docker Desktop per a Windows i Mac. Per a Linux, pots instal·lar-lo utilitzant les següents comandes:

sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

Verifica la instal·lació:

docker-compose --version

Creant un Fitxer Docker Compose

Un fitxer Docker Compose està escrit en format YAML i típicament es diu docker-compose.yml.

Exemple: Aplicació Web Simple

Creem una aplicació web simple utilitzant Docker Compose.

  1. Crea un directori de projecte:

    mkdir my-web-app
cd my-web-app

  2. Crea un fitxer docker-compose.yml:

    version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"
  redis:
    image: redis:latest

Explicació

  • version: Especifica la versió del format del fitxer Docker Compose.
  • services: Defineix els serveis que componen la teva aplicació.
    • web: Utilitza la imatge nginx:latest i mapatge el port 8080 de l'amfitrió al port 80 del contenidor.
    • redis: Utilitza la imatge redis:latest.

Executant Docker Compose

Per iniciar els serveis definits al fitxer docker-compose.yml, utilitza la següent comanda:

docker-compose up

Aquesta comanda:

  • Descarregarà les imatges especificades si no estan disponibles localment.
  • Crearà i iniciarà els contenidors.
  • S'adjuntarà als registres dels contenidors.

Per executar els serveis en segon pla (mode desconnectat), utilitza:

docker-compose up -d

Gestionant Serveis Docker Compose

Aturant Serveis

Per aturar els serveis, utilitza:

docker-compose down

Aquesta comanda atura i elimina els contenidors, xarxes i volums definits al fitxer docker-compose.yml.

Veient Registres

Per veure els registres d'un servei específic, utilitza:

docker-compose logs <nom_servei>

Per exemple, per veure els registres del servei web:

docker-compose logs web

Escalant Serveis

Docker Compose et permet escalar serveis. Per exemple, per escalar el servei web a 3 instàncies:

docker-compose up --scale web=3 -d

Conclusió

Docker Compose simplifica la gestió d'aplicacions Docker de múltiples contenidors. Definint serveis, xarxes i volums en un únic fitxer YAML, pots orquestrar fàcilment aplicacions complexes. Amb comandes com docker-compose up i docker-compose down, gestionar el cicle de vida de la teva aplicació esdevé senzill. A mesura que progressis, podràs explorar funcions més avançades com variables d'entorn, xarxes personalitzades i gestió de volums per millorar encara més les teves configuracions de Docker Compose.
