Introducció

En aquest projecte, construirem una aplicació multi-contenidor utilitzant Docker. Aquest projecte t'ajudarà a entendre com orquestrar múltiples serveis, gestionar dependències i assegurar una comunicació fluida entre contenidors.

Conceptes Clau

  • Docker Compose: Una eina per definir i executar aplicacions Docker multi-contenidor.
  • Servei: Un contenidor en un fitxer Docker Compose.
  • Xarxa: Una xarxa Docker que permet als contenidors comunicar-se entre ells.
  • Volum: Un mecanisme d'emmagatzematge persistent per als contenidors Docker.

Configuració del Projecte

Per començar, crearem una aplicació simple multi-contenidor que consisteix en un servidor web i una base de dades.

Pas 1: Crear el Directori del Projecte

Primer, crea un directori per al teu projecte:

mkdir multi-container-app
cd multi-container-app

Pas 2: Definir el Fitxer Docker Compose

Crea un fitxer docker-compose.yml per definir els serveis:

version: '3.8'

services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    networks:
      - app-network

  db:
    image: postgres:latest
    environment:
      POSTGRES_USER: example
      POSTGRES_PASSWORD: example
    volumes:
      - db-data:/var/lib/postgresql/data
    networks:
      - app-network

networks:
  app-network:
    driver: bridge

volumes:
  db-data:

Explicació

  • version: Especifica la versió de Docker Compose.
  • services: Defineix els contenidors a executar.
    • web: El servei de servidor web utilitzant la imatge Nginx.
      • ports: Assigna el port 80 de l'amfitrió al port 80 del contenidor.
      • networks: Connecta el contenidor a la app-network.
    • db: El servei de base de dades utilitzant la imatge PostgreSQL.
      • environment: Estableix variables d'entorn per a l'usuari i la contrasenya de la base de dades.
      • volumes: Munta un volum per a l'emmagatzematge persistent.
      • networks: Connecta el contenidor a la app-network.
  • networks: Defineix una xarxa personalitzada anomenada app-network.
  • volumes: Defineix un volum anomenat db-data per a l'emmagatzematge persistent.

Executar l'Aplicació

Per executar l'aplicació, utilitza la següent comanda:

docker-compose up -d
  • El flag -d executa els contenidors en mode desconnectat.

Comprovar els Serveis

Pots comprovar els serveis en execució utilitzant:

docker-compose ps

Interactuar amb els Contenidors

Accedir al Servidor Web

Obre el teu navegador i navega a http://localhost. Hauries de veure la pàgina de benvinguda predeterminada de Nginx.

Accedir a la Base de Dades

Pots interactuar amb la base de dades PostgreSQL utilitzant un client PostgreSQL o l'eina de línia de comandes psql.

Aturar l'Aplicació

Per aturar l'aplicació, utilitza:

docker-compose down
  • Aquesta comanda atura i elimina els contenidors, xarxes i volums definits en el fitxer docker-compose.yml.

Conclusió

En aquest projecte, has après a:

  • Definir i executar una aplicació multi-contenidor utilitzant Docker Compose.
  • Configurar serveis amb configuracions específiques.
  • Gestionar xarxes i volums per a la comunicació entre contenidors i l'emmagatzematge persistent.

En dominar aquests conceptes, ara pots construir i gestionar aplicacions multi-contenidor més complexes, millorant les teves habilitats amb Docker des de nivell principiant fins a avançat.
