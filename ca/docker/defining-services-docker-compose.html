Definició de Serveis en Docker Compose
Construint i executant aplicacions multi-contenidor ►

Docker Compose és una eina per definir i executar aplicacions Docker amb múltiples contenidors. Amb Compose, utilitzes un fitxer YAML per configurar els serveis de la teva aplicació. Després, amb una sola comanda, crees i inicies tots els serveis de la teva configuració.

Introducció a Docker Compose

Docker Compose et permet definir i gestionar múltiples contenidors com un sol servei. Això és particularment útil per a aplicacions que requereixen diversos serveis interconnectats, com ara un servidor web, una base de dades i una capa de memòria cau.

Conceptes Clau

  • Servei: Un servei és un contenidor en la teva aplicació. Es defineix en el fitxer docker-compose.yml.
  • Xarxa: Docker Compose crea automàticament una xarxa perquè els teus serveis es puguin comunicar.
  • Volum: Els volums s'utilitzen per persistir dades generades i utilitzades pels contenidors Docker.

Definició de Serveis en `docker-compose.yml`

El fitxer docker-compose.yml és on defineixes els teus serveis. Cada servei és un contenidor que executa una imatge específica.

Estructura Bàsica de `docker-compose.yml`

Aquí tens un exemple senzill d'un fitxer docker-compose.yml:

version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
  db:
    image: postgres:latest
    environment:
      POSTGRES_PASSWORD: example

Explicació

  • version: Especifica la versió del format del fitxer Docker Compose.
  • services: Defineix els serveis que componen la teva aplicació.
    • web: Aquest és el nom del servei.
      • image: Especifica la imatge Docker a utilitzar.
      • ports: Mapeja el port de l'amfitrió al port del contenidor.
    • db: Un altre servei.
      • environment: Estableix variables d'entorn per al contenidor.

Afegint Més Configuració

Pots afegir més configuracions als teus serveis, com ara volums, xarxes i dependències.

version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    depends_on:
      - db
  db:
    image: postgres:latest
    environment:
      POSTGRES_PASSWORD: example
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:

Explicació

  • volumes: Munta un directori de l'amfitrió o un volum amb nom dins del contenidor.
  • depends_on: Especifica dependències entre serveis. En aquest cas, el servei web depèn del servei db.
  • volumes (a la part inferior): Defineix volums amb nom que poden ser utilitzats pels serveis.

Exemple: Una Aplicació Web amb una Base de Dades

Creem un exemple més complet amb una aplicació web que utilitza una base de dades.

version: '3'
services:
  app:
    image: myapp:latest
    build: .
    ports:
      - "5000:5000"
    environment:
      - DATABASE_URL=postgres://user:password@db:5432/mydatabase
    depends_on:
      - db
  db:
    image: postgres:latest
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mydatabase
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:

Explicació

  • app: El servei de l'aplicació web.
    • build: Construeix la imatge Docker des del directori actual.
    • environment: Estableix variables d'entorn, inclosa la URL de la base de dades.
  • db: El servei de la base de dades.
    • environment: Estableix variables d'entorn per a la base de dades PostgreSQL.
    • volumes: Utilitza un volum amb nom per persistir les dades de la base de dades.

Conclusió

Definir serveis en Docker Compose et permet gestionar aplicacions amb múltiples contenidors de manera eficient. Utilitzant un fitxer docker-compose.yml, pots especificar la configuració per a cada servei, incloent imatges, ports, volums i dependències. Això facilita la configuració i manteniment d'aplicacions complexes amb múltiples serveis interconnectats.
