La monitorització i el registre són aspectes crucials per gestionar els contenidors Docker de manera efectiva. Aquest tema et guiarà des dels conceptes bàsics fins a tècniques avançades per a la monitorització i el registre en Docker.

Introducció a la monitorització i el registre

La monitorització i el registre són essencials per a:

  • Assegurar la salut i el rendiment de les teves aplicacions.
  • Depurar i solucionar problemes.
  • Mantenir la seguretat i el compliment.

Conceptes bàsics

Monitorització

  • Definició: El procés de recollir, analitzar i utilitzar informació per fer un seguiment del rendiment i la salut de les teves aplicacions i infraestructura.
  • Mètriques clau: Ús de la CPU, ús de la memòria, I/O de disc, I/O de xarxa, etc.

Registre

  • Definició: El procés de registrar esdeveniments d'aplicacions i sistemes per proporcionar un historial d'activitat.
  • Tipus de registres: Registres d'aplicacions, registres de sistemes, registres del dimoni Docker, etc.

Configuració de la monitorització bàsica

Utilitzant Docker Stats

Docker proporciona una comanda senzilla per monitoritzar l'ús de recursos dels contenidors.

docker stats

Aquesta comanda mostra un flux en viu de les estadístiques d'ús de recursos dels contenidors.

Exemple de sortida

CONTAINER ID   NAME           CPU %     MEM USAGE / LIMIT     MEM %     NET I/O           BLOCK I/O         PIDS
d1f2c3b4e5f6   my_container   0.07%     12.34MiB / 1.952GiB   0.62%     1.23kB / 2.34kB   0B / 0B           2

Configuració del registre bàsic

Utilitzant Docker Logs

Docker proporciona una comanda per obtenir registres d'un contenidor.

docker logs <container_id>

Exemple d'ús

docker logs d1f2c3b4e5f6

Aquesta comanda mostrarà els registres del contenidor amb ID d1f2c3b4e5f6.

Monitorització avançada

Utilitzant Prometheus i Grafana

Prometheus és un sistema de monitorització de codi obert, i Grafana és una solució d'analítica i monitorització de codi obert.

Configuració de Prometheus

  1. Crea un fitxer de configuració de Prometheus (prometheus.yml):

    global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'docker'
    static_configs:
      - targets: ['<docker_host>:<docker_port>']

  2. Executa Prometheus com un contenidor Docker:

    docker run -d -p 9090:9090 -v /path/to/prometheus.yml:/etc/prometheus/prometheus.yml prom/prometheus

Configuració de Grafana

  1. Executa Grafana com un contenidor Docker:

    docker run -d -p 3000:3000 grafana/grafana

  2. Configura Grafana:

    • Accedeix a Grafana a http://localhost:3000.
    • Afegeix Prometheus com a font de dades.
    • Crea quadres de comandament per visualitzar les mètriques.

Utilitzant cAdvisor

cAdvisor (Container Advisor) proporciona als usuaris de contenidors una comprensió de l'ús de recursos i les característiques de rendiment dels seus contenidors en execució.

Executant cAdvisor

docker run \
  --volume=/:/rootfs:ro \
  --volume=/var/run:/var/run:ro \
  --volume=/sys:/sys:ro \
  --volume=/var/lib/docker/:/var/lib/docker:ro \
  --publish=8080:8080 \
  --detach=true \
  --name=cadvisor \
  google/cadvisor:latest

Accedeix a cAdvisor a http://localhost:8080.

Registre avançat

Utilitzant ELK Stack (Elasticsearch, Logstash, Kibana)

La pila ELK és un conjunt d'eines poderoses per gestionar i analitzar registres.

Configuració d'Elasticsearch

docker run -d -p 9200:9200 -p 9300:9300 -e "discovery.type=single-node" elasticsearch:7.10.1

Configuració de Logstash

  1. Crea un fitxer de configuració de Logstash (logstash.conf):

    input {
  docker {
    host => "unix:///var/run/docker.sock"
  }
}
output {
  elasticsearch {
    hosts => ["localhost:9200"]
  }
}

  2. Executa Logstash com un contenidor Docker:

    docker run -d -v /path/to/logstash.conf:/usr/share/logstash/pipeline/logstash.conf logstash:7.10.1

Configuració de Kibana

docker run -d -p 5601:5601 kibana:7.10.1

Accedeix a Kibana a http://localhost:5601.

Conclusió

La monitorització i el registre són crítics per mantenir la salut, el rendiment i la seguretat de les teves aplicacions Dockeritzades. Començant amb comandes bàsiques com docker stats i docker logs, pots progressar cap a configuracions més avançades utilitzant eines com Prometheus, Grafana, cAdvisor i la pila ELK. Implementant aquestes pràctiques, pots assegurar-te que les teves aplicacions funcionin de manera suau i eficient, i pots diagnosticar i resoldre ràpidament qualsevol problema que sorgeixi.
