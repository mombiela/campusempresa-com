Introducció

En aquesta secció, explorarem com construir i executar aplicacions multi-contenidor utilitzant Docker. Això és essencial per crear aplicacions escalables i mantenibles que consisteixen en múltiples serveis treballant junts.

Conceptes Clau

  • Arquitectura de Microserveis: Descomposició d'aplicacions en serveis més petits i independents.
  • Docker Compose: Una eina per definir i executar aplicacions Docker multi-contenidor.
  • Dependències de Serveis: Gestió de com els serveis interactuen i depenen els uns dels altres.
  • Xarxes: Assegurar que els contenidors puguin comunicar-se entre ells.

Conceptes Bàsics de Docker Compose

Docker Compose és una eina que et permet definir i gestionar aplicacions Docker multi-contenidor. Utilitza un fitxer YAML per configurar els serveis, xarxes i volums de l'aplicació.

Instal·lació de Docker Compose

Docker Compose està inclòs a Docker Desktop per a Windows i Mac. Per a Linux, pots instal·lar-lo utilitzant la següent comanda:

sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

Creació d'un Fitxer `docker-compose.yml`

Un fitxer docker-compose.yml defineix els serveis que componen la teva aplicació. Aquí tens un exemple per a una aplicació web senzilla amb un servei de frontend i un de backend:

version: '3'
services:
  frontend:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./frontend:/usr/share/nginx/html
  backend:
    image: node:14
    working_dir: /app
    volumes:
      - ./backend:/app
    command: npm start

Execució de Docker Compose

Per iniciar l'aplicació, navega al directori que conté el teu fitxer docker-compose.yml i executa:

docker-compose up

Aquesta comanda iniciarà tots els serveis definits al fitxer docker-compose.yml.

Dependències de Serveis

En una aplicació multi-contenidor, els serveis sovint depenen els uns dels altres. Docker Compose et permet especificar dependències utilitzant l'opció depends_on.

Exemple amb Dependències

Aquí tens un exemple on el servei de backend depèn d'un servei de base de dades:

version: '3'
services:
  db:
    image: postgres:latest
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mydb
  backend:
    image: node:14
    working_dir: /app
    volumes:
      - ./backend:/app
    command: npm start
    depends_on:
      - db

Xarxes

Docker Compose crea automàticament una xarxa per a la teva aplicació, permetent que els serveis es comuniquin entre ells pels seus noms de servei.

Exemple amb Xarxes

En l'exemple anterior, el servei de backend pot connectar-se al servei de base de dades utilitzant el nom de servei db:

const { Client } = require('pg');
const client = new Client({
  host: 'db',
  user: 'user',
  password: 'password',
  database: 'mydb'
});
client.connect();

Funcions Avançades de Docker Compose

Docker Compose ofereix diverses funcions avançades per gestionar aplicacions complexes.

Variables d'Entorn

Pots utilitzar variables d'entorn per configurar els teus serveis dinàmicament.

version: '3'
services:
  backend:
    image: node:14
    working_dir: /app
    volumes:
      - ./backend:/app
    command: npm start
    environment:
      - NODE_ENV=production

Volums

Els volums et permeten persistir dades i compartir-les entre serveis.

version: '3'
services:
  db:
    image: postgres:latest
    volumes:
      - db-data:/var/lib/postgresql/data
volumes:
  db-data:

Escalar Serveis

Pots escalar serveis per gestionar més càrrega especificant el nombre d'instàncies.

docker-compose up --scale backend=3

Conclusió

Construir i executar aplicacions multi-contenidor amb Docker Compose simplifica la gestió d'aplicacions complexes. Descomponent les aplicacions en serveis més petits, pots aconseguir millor escalabilitat i mantenibilitat. Docker Compose proporciona funcions potents per gestionar dependències de serveis, xarxes i escalabilitat, fent-lo una eina essencial per al desenvolupament d'aplicacions modernes.
