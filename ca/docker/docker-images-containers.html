Introducció a les Imatges i Contenidors de Docker

Docker és una plataforma potent per desenvolupar, enviar i executar aplicacions dins de contenidors. Per utilitzar Docker de manera efectiva, és crucial entendre els conceptes d'imatges i contenidors de Docker.

Què és una Imatge de Docker?

Una imatge de Docker és un paquet de programari lleuger, autònom i executable que inclou tot el necessari per executar una peça de programari, incloent-hi el codi, el temps d'execució, les biblioteques, les variables d'entorn i els fitxers de configuració.

Conceptes Clau de les Imatges de Docker

  • Capes: Les imatges de Docker es construeixen en capes, on cada capa representa una instrucció en el Dockerfile de la imatge.
  • Imatge Base: El punt de partida per construir una imatge de Docker, sovint una imatge oficial com ubuntu o alpine.
  • Dockerfile: Un fitxer de text que conté una sèrie d'instruccions sobre com construir una imatge de Docker.

Exemple d'un Dockerfile

# Utilitza un temps d'execució oficial de Python com a imatge pare
FROM python:3.8-slim

# Estableix el directori de treball en el contenidor
WORKDIR /app

# Copia el contingut del directori actual al contenidor a /app
COPY . /app

# Instal·la qualsevol paquet necessari especificat a requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Fa que el port 80 estigui disponible per al món fora d'aquest contenidor
EXPOSE 80

# Defineix una variable d'entorn
ENV NAME World

# Executa app.py quan el contenidor es llanci
CMD ["python", "app.py"]

Explicació:

  • FROM: Especifica la imatge base.
  • WORKDIR: Estableix el directori de treball dins del contenidor.
  • COPY: Copia fitxers de l'amfitrió al contenidor.
  • RUN: Executa comandes en el contenidor.
  • EXPOSE: Informa Docker que el contenidor escolta en els ports de xarxa especificats en temps d'execució.
  • ENV: Estableix variables d'entorn.
  • CMD: Proporciona la comanda per executar dins del contenidor.

Què és un Contenidor de Docker?

Un contenidor de Docker és una instància executable d'una imatge de Docker. Els contenidors estan aïllats entre si i del sistema amfitrió, però poden interactuar entre ells a través de canals ben definits.

Conceptes Clau dels Contenidors de Docker

  • Aïllament: Els contenidors s'executen en entorns aïllats, assegurant que les aplicacions no interfereixin entre elles.
  • Portabilitat: Els contenidors poden executar-se en qualsevol sistema que suporti Docker, fent-los altament portables.
  • Efímer: Els contenidors estan dissenyats per ser transitoris i poden ser aturats i eliminats fàcilment.

Executar un Contenidor de Docker

# Descarrega la imatge de Docker Hub
docker pull python:3.8-slim

# Executa un contenidor des de la imatge
docker run -d -p 4000:80 --name my-python-app python:3.8-slim

Explicació:

  • docker pull: Descarrega la imatge especificada de Docker Hub.
  • docker run: Crea i inicia un nou contenidor des de la imatge especificada.
    • -d: Executa el contenidor en mode desconnectat.
    • -p 4000:80: Assigna el port 4000 de l'amfitrió al port 80 del contenidor.
    • --name: Assigna un nom al contenidor.

Comparant Imatges i Contenidors de Docker

| Característica | Imatge de Docker | Contenidor de Docker | |---------------|--------------------------------------------------|--------------------------------------------------| | Definició | Una plantilla amb instruccions per crear un contenidor | Una instància en execució d'una imatge de Docker | | Estat | Estàtic (només lectura) | Dinàmic (lectura-escriptura) | | Emmagatzematge| Emmagatzemat en disc com una sèrie de capes | S'executa en memòria, amb canvis emmagatzemats en una capa d'escriptura | | Creació | Construït utilitzant un Dockerfile | Creat a partir d'una imatge de Docker utilitzant docker run | | Portabilitat | Pot ser compartida i reutilitzada | Específica del sistema amfitrió on s'està executant |

Conclusió

Entendre les imatges i contenidors de Docker és fonamental per dominar Docker. Les imatges són els plànols per als teus contenidors, contenint tots els components necessaris per executar les teves aplicacions. Els contenidors, d'altra banda, són les instàncies vives d'aquestes imatges, proporcionant entorns aïllats per executar les teves aplicacions.

Comprenent aquests conceptes, pots construir, compartir i executar aplicacions de manera eficient en una manera consistent i reproduïble, aprofitant tot el poder de Docker.
