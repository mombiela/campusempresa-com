Introducció

Gestionar imatges Docker és una habilitat fonamental per a qualsevol persona que treballi amb Docker. Aquest tema cobrirà els conceptes bàsics de les imatges Docker, com crear-les, gestionar-les i optimitzar-les, i les millors pràctiques per utilitzar imatges Docker en els teus projectes.

Conceptes Clau

  • Imatge Docker: Una plantilla de només lectura que conté un conjunt d'instruccions per crear un contenidor que pot executar-se a la plataforma Docker.
  • Contenidor Docker: Una instància executable d'una imatge Docker.
  • Dockerfile: Un fitxer de text que conté una sèrie d'instruccions sobre com construir una imatge Docker.

Creació d'Imatges Docker

Per crear una imatge Docker, cal escriure un Dockerfile. Aquí tens un exemple senzill:

# Utilitza una imatge oficial de Python com a imatge base
FROM python:3.8-slim

# Estableix el directori de treball dins del contenidor
WORKDIR /app

# Copia el contingut del directori actual al contenidor a /app
COPY . /app

# Instal·la qualsevol paquet necessari especificat a requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Fa que el port 80 estigui disponible per al món fora d'aquest contenidor
EXPOSE 80

# Defineix una variable d'entorn
ENV NAME World

# Executa app.py quan el contenidor es llanci
CMD ["python", "app.py"]

Explicació

  • FROM: Especifica la imatge base a utilitzar.
  • WORKDIR: Estableix el directori de treball dins del contenidor.
  • COPY: Copia fitxers de l'amfitrió al contenidor.
  • RUN: Executa comandes dins del contenidor.
  • EXPOSE: Informa Docker que el contenidor escolta als ports de xarxa especificats en temps d'execució.
  • ENV: Estableix variables d'entorn.
  • CMD: Proporciona la comanda per executar dins del contenidor.

Construcció d'Imatges Docker

Per construir una imatge Docker a partir d'un Dockerfile, utilitza la comanda docker build:

docker build -t my-python-app .
  • -t my-python-app: Etiqueta la imatge amb el nom my-python-app.
  • .: Especifica el context de construcció, que és el directori actual.

Gestió d'Imatges Docker

Llistar Imatges Docker

Per llistar totes les imatges Docker al teu sistema, utilitza la comanda docker images:

docker images

Eliminar Imatges Docker

Per eliminar una imatge Docker, utilitza la comanda docker rmi seguida de l'ID o el nom de la imatge:

docker rmi my-python-app

Etiquetar Imatges Docker

Etiquetar és útil per versionar les teves imatges. Utilitza la comanda docker tag:

docker tag my-python-app my-python-app:v1.0

Pujar Imatges Docker a un Registre

Per compartir les teves imatges Docker, pots pujar-les a un registre Docker com Docker Hub. Primer, inicia sessió a Docker Hub:

docker login

Després, puja la teva imatge:

docker push my-python-app:v1.0

Millors Pràctiques

  • Minimitzar la Mida de la Imatge: Utilitza imatges base més petites i neteja fitxers innecessaris per reduir la mida de la imatge.
  • Caché de Capes: Aprofita el caché de capes de Docker ordenant les instruccions del Dockerfile de menys a més canviants.
  • Seguretat: Actualitza regularment les teves imatges base i dependències per incloure els últims pegats de seguretat.

Conclusió

Gestionar imatges Docker implica crear, construir, etiquetar i pujar imatges a un registre. Seguint les millors pràctiques, pots optimitzar les teves imatges Docker per a rendiment i seguretat. Entendre aquests conceptes és crucial per a un ús eficient de Docker en qualsevol entorn de desenvolupament o producció.
