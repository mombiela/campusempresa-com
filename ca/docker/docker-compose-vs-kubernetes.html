Docker Compose vs Kubernetes
Introducció a Kubernetes ►

En aquesta secció, explorarem les diferències i similituds entre Docker Compose i Kubernetes. Ambdues eines són essencials per gestionar aplicacions contenidoritzades, però serveixen per a propòsits diferents i són adequades per a diferents casos d'ús.

Introducció a Docker Compose

Docker Compose és una eina per definir i executar aplicacions Docker de múltiples contenidors. Amb Docker Compose, pots utilitzar un fitxer YAML per configurar els serveis, xarxes i volums de la teva aplicació.

Conceptes Clau de Docker Compose

  • Serveis: Defineixen els diferents contenidors que componen la teva aplicació.
  • Xarxes: Permeten que els contenidors es comuniquin entre ells.
  • Volums: Persisten les dades generades i utilitzades pels contenidors Docker.

Exemple d'un Fitxer Docker Compose

Aquí tens un exemple senzill d'un fitxer docker-compose.yml:

version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
  database:
    image: postgres:latest
    environment:
      POSTGRES_PASSWORD: example

En aquest exemple:

  • Definim dos serveis: web i database.
  • El servei web utilitza la imatge nginx i mapatge el port 80 de l'amfitrió al port 80 del contenidor.
  • El servei database utilitza la imatge postgres i estableix una variable d'entorn per a la contrasenya de PostgreSQL.

Introducció a Kubernetes

Kubernetes és una plataforma de codi obert dissenyada per automatitzar el desplegament, l'escalat i l'operació de contenidors d'aplicacions. Proporciona un marc robust per executar sistemes distribuïts de manera resilient.

Conceptes Clau de Kubernetes

  • Pods: Les unitats desplegables més petites que es poden crear i gestionar a Kubernetes.
  • Serveis: Defineixen un conjunt lògic de Pods i una política per accedir-hi.
  • Desplegaments: Proporcionen actualitzacions declaratives a les aplicacions.
  • ConfigMaps i Secrets: Gestionen dades de configuració i informació sensible.

Exemple d'un Desplegament de Kubernetes

Aquí tens un exemple senzill d'un fitxer YAML de desplegament de Kubernetes:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

En aquest exemple:

  • Definim un Desplegament anomenat nginx-deployment.
  • El Desplegament crearà 3 rèpliques del contenidor nginx.
  • Cada contenidor executarà la imatge nginx:1.14.2 i exposarà el port 80.

Comparació de Docker Compose i Kubernetes

| Característica | Docker Compose | Kubernetes | |------------------------|----------------------------------------|-----------------------------------------| | Format de Configuració | YAML | YAML | | Complexitat | Simple, fàcil de configurar | Complex, corba d'aprenentatge pronunciada| | Cas d'Ús | Desenvolupament local, aplicacions petites| Producció, aplicacions distribuïdes a gran escala| | Escalat | Escalat manual | Escalat automàtic | | Xarxes | Configuració de xarxa senzilla | Capacitats avançades de xarxa | | Emmagatzematge | Suport bàsic de volums | Volums Persistents, Classes d'Emmagatzematge| | Orquestració | Orquestració limitada | Orquestració i programació avançades |

Exemples de Codi

Exemple de Docker Compose

Per executar l'exemple de Docker Compose, guarda el següent contingut en un fitxer anomenat docker-compose.yml:

version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
  database:
    image: postgres:latest
    environment:
      POSTGRES_PASSWORD: example

Després, executa la següent comanda:

docker-compose up

Aquesta comanda iniciarà els serveis web i database definits en el fitxer docker-compose.yml.

Exemple de Kubernetes

Per executar l'exemple de Kubernetes, guarda el següent contingut en un fitxer anomenat nginx-deployment.yaml:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

Després, executa la següent comanda:

kubectl apply -f nginx-deployment.yaml

Aquesta comanda crearà un Desplegament a Kubernetes que gestionarà 3 rèpliques del contenidor nginx.

Conclusió

Docker Compose i Kubernetes són ambdues eines poderoses per gestionar aplicacions contenidoritzades, però són adequades per a diferents casos d'ús. Docker Compose és ideal per al desenvolupament local i aplicacions de petita escala, mentre que Kubernetes està dissenyat per a entorns de producció i aplicacions distribuïdes a gran escala. Entendre els punts forts i les limitacions de cada eina t'ajudarà a triar la més adequada per a les teves necessitats específiques.
