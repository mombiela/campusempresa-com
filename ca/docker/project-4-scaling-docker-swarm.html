Introducció

En aquest projecte, explorarem Docker Swarm, una eina nativa de clustering i orquestració per a Docker. Docker Swarm et permet gestionar un clúster de nodes Docker com un sistema virtual únic. Aquest projecte et guiarà a través de la configuració d'un Docker Swarm, el desplegament de serveis i l'escalat eficient d'aquests.

Conceptes Clau

  • Docker Swarm: Una eina de clustering i orquestració per a contenidors Docker.
  • Node: Una màquina individual en el clúster Docker Swarm.
  • Node Manager: Un node que gestiona el swarm i pren decisions sobre el clúster.
  • Node Worker: Un node que rep i executa tasques del node manager.
  • Servei: Una definició de com ha de comportar-se un contenidor en el swarm.
  • Tasca: Un únic contenidor que s'executa en el swarm.

Configuració de Docker Swarm

Inicialització del Swarm

Per inicialitzar un Docker Swarm, has d'executar la següent comanda a la màquina que actuarà com a node manager:

docker swarm init --advertise-addr <MANAGER-IP>

Substitueix <MANAGER-IP> per l'adreça IP del teu node manager.

Afegir Nodes Worker

Després d'inicialitzar el swarm, rebràs una comanda per afegir nodes worker. Executa aquesta comanda a cada node worker:

docker swarm join --token <SWARM-TOKEN> <MANAGER-IP>:2377

Substitueix <SWARM-TOKEN> pel token proporcionat per la comanda docker swarm init i <MANAGER-IP> per l'adreça IP del node manager.

Desplegament de Serveis

Creació d'un Servei

Per crear un servei en Docker Swarm, utilitza la comanda docker service create. Per exemple, per crear un servei simple de Nginx, executa:

docker service create --name my-nginx --replicas 3 -p 80:80 nginx

Aquesta comanda:

  • Crearà un servei anomenat my-nginx.
  • Desplegarà 3 rèpliques del contenidor Nginx.
  • Maparà el port 80 de l'amfitrió al port 80 del contenidor.

Llistar Serveis

Per llistar tots els serveis que s'executen en el swarm, utilitza:

docker service ls

Inspeccionar un Servei

Per obtenir informació detallada sobre un servei, utilitza:

docker service inspect <SERVICE-ID>

Substitueix <SERVICE-ID> per l'ID del servei que vols inspeccionar.

Escalat de Serveis

Escalar Cap Amunt

Per escalar un servei cap amunt, utilitza la comanda docker service scale. Per exemple, per escalar el servei my-nginx a 5 rèpliques, executa:

docker service scale my-nginx=5

Escalar Cap Avall

Per escalar un servei cap avall, utilitza la mateixa comanda docker service scale. Per exemple, per escalar el servei my-nginx a 2 rèpliques, executa:

docker service scale my-nginx=2

Monitorització i Gestió del Swarm

Comprovar l'Estat dels Nodes

Per comprovar l'estat de tots els nodes en el swarm, utilitza:

docker node ls

Draining d'un Node

Per drenar un node (aturar-lo de rebre noves tasques), utilitza:

docker node update --availability drain <NODE-ID>

Substitueix <NODE-ID> per l'ID del node que vols drenar.

Promoció i Degradació de Nodes

Per promocionar un node worker a node manager, utilitza:

docker node promote <NODE-ID>

Per degradar un node manager a node worker, utilitza:

docker node demote <NODE-ID>

Conclusió

En aquest projecte, has après com configurar un Docker Swarm, desplegar i escalar serveis, i gestionar el swarm de manera efectiva. Docker Swarm proporciona una manera potent però senzilla d'orquestrar contenidors, facilitant la gestió d'aplicacions a gran escala. En dominar aquests conceptes, pots assegurar-te que les teves aplicacions siguin escalables, resilients i fàcils de gestionar.
