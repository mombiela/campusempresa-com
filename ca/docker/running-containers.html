En aquesta secció, explorarem com executar contenidors utilitzant Docker. Aquesta és una habilitat fonamental per a qualsevol persona que treballi amb Docker, ja que els contenidors són la unitat bàsica de desplegament de programari en Docker.

Conceptes Clau

  • Contenidor: Un paquet executable lleuger, autònom, que inclou tot el necessari per executar una peça de programari, incloent el codi, el temps d'execució, les biblioteques i les eines del sistema.
  • Imatge: Una plantilla de només lectura utilitzada per crear contenidors. Les imatges sovint es basen en altres imatges, amb alguna personalització addicional.
  • Docker Hub: Un repositori basat en el núvol on els usuaris de Docker poden crear, provar, emmagatzemar i distribuir imatges de contenidors.

Executar el Teu Primer Contenidor

Per executar un contenidor, necessites una imatge Docker. Docker Hub proporciona un vast repositori d'imatges preconstruïdes que pots utilitzar.

Exemple: Executar un Contenidor Hello World

  1. Descarregar la Imatge: Primer, has de descarregar la imatge des de Docker Hub.

    docker pull hello-world

  2. Executar el Contenidor: Utilitza el comandament docker run per crear i iniciar un contenidor a partir de la imatge.

    docker run hello-world

    Quan executis aquest comandament, Docker:

    • Comprovarà si la imatge hello-world està disponible localment. Si no, la descarregarà des de Docker Hub.
    • Crearà un nou contenidor a partir de la imatge.
    • Iniciarà el contenidor.
    • Executarà el comandament especificat a la imatge (en aquest cas, imprimir "Hello from Docker!").

Entendre el Comandament `docker run`

El comandament docker run és versàtil i es pot utilitzar amb diverses opcions per personalitzar el comportament del contenidor.

  • Sintaxi Bàsica:

    docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

  • Opcions Comunes:

    • -d: Executar el contenidor en mode desconnectat (en segon pla).
    • -p: Publicar el(s) port(s) d'un contenidor a l'amfitrió.
    • -v: Muntar un volum.
    • --name: Assignar un nom al contenidor.
    • -e: Establir variables d'entorn.

Exemple: Executar un Servidor Web

Anem a executar un servidor web simple utilitzant la imatge nginx.

  1. Descarregar la Imatge:

    docker pull nginx

  2. Executar el Contenidor:

    docker run -d -p 8080:80 --name mynginx nginx

    Explicació:

    • -d: Executa el contenidor en mode desconnectat.
    • -p 8080:80: Mapeja el port 8080 de l'amfitrió al port 80 del contenidor.
    • --name mynginx: Assigna el nom mynginx al contenidor.
    • nginx: La imatge a utilitzar.

  3. Accedir al Servidor Web: Obre un navegador web i navega a http://localhost:8080. Hauries de veure la pàgina de benvinguda predeterminada de Nginx.

Gestionar Contenidors

Un cop tinguis contenidors en execució, necessitaràs gestionar-los. Docker proporciona diversos comandaments per ajudar-te amb això.

Llistar Contenidors

  • Llistar Contenidors en Execució:

    docker ps

  • Llistar Tots els Contenidors (incloent els aturats):

    docker ps -a

Aturar i Eliminar Contenidors

  • Aturar un Contenidor en Execució:

    docker stop <container_id_or_name>

  • Eliminar un Contenidor Aturat:

    docker rm <container_id_or_name>

Exemple: Aturar i Eliminar el Contenidor Nginx

  1. Aturar el Contenidor:

    docker stop mynginx

  2. Eliminar el Contenidor:

    docker rm mynginx

Conclusió

Executar contenidors és una habilitat fonamental en Docker. Entenent com descarregar imatges, executar contenidors i gestionar-los, pots començar a aprofitar el poder de Docker per desplegar i gestionar aplicacions de manera eficient. Practica executant diferents contenidors i utilitzant diverses opcions amb el comandament docker run per sentir-te més còmode amb Docker.
