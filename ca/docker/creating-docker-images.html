Conceptes bàsics de Dockerfile ►

Crear imatges Docker és una habilitat fonamental per a qualsevol persona que treballi amb Docker. Aquest tema et guiarà a través del procés des d'un nivell principiant fins a un nivell avançat, cobrint conceptes clau, exemples pràctics i exercicis per consolidar la teva comprensió.

Introducció a les Imatges Docker

Les imatges Docker són els plànols per crear contenidors Docker. Contenen tot el necessari per executar una aplicació, incloent el codi, el temps d'execució, les biblioteques, les variables d'entorn i els fitxers de configuració.

Conceptes Clau

  • Dockerfile: Un fitxer de text que conté una sèrie d'instruccions sobre com construir una imatge Docker.
  • Imatge Base: El punt de partida per construir la teva imatge Docker, sovint una imatge oficial de Docker Hub.
  • Capes: Cada instrucció en un Dockerfile crea una nova capa en la imatge, cosa que ajuda en la memòria cau i la reutilització.
  • Etiquetatge d'Imatges: Assignar una versió o identificador a una imatge, típicament en el format repository:tag.

Escriure un Dockerfile

Un Dockerfile és el nucli de la creació d'imatges Docker. Defineix els passos necessaris per muntar la imatge.

Estructura Bàsica del Dockerfile

Aquí tens un exemple senzill d'un Dockerfile per a una aplicació Node.js:

# Utilitza una imatge oficial de Node.js com a imatge pare
FROM node:14

# Estableix el directori de treball en el contenidor
WORKDIR /usr/src/app

# Copia els fitxers package.json i package-lock.json
COPY package*.json ./

# Instal·la les dependències de l'aplicació
RUN npm install

# Copia la resta del codi de l'aplicació
COPY . .

# Exposa el port de l'aplicació
EXPOSE 8080

# Defineix la comanda per executar l'aplicació
CMD ["node", "app.js"]

Explicació

  • FROM node:14: Especifica la imatge base.
  • WORKDIR /usr/src/app: Estableix el directori de treball dins del contenidor.
  • COPY package*.json ./: Copia els fitxers de paquets al directori de treball.
  • RUN npm install: Instal·la les dependències.
  • COPY . .: Copia la resta del codi de l'aplicació.
  • EXPOSE 8080: Exposa el port 8080 per a l'aplicació.
  • CMD ["node", "app.js"]: Especifica la comanda per executar l'aplicació.

Construir Imatges Docker

Un cop tens un Dockerfile, pots construir una imatge utilitzant la comanda docker build.

Construir una Imatge

Per construir una imatge a partir del Dockerfile en el directori actual, utilitza la següent comanda:

docker build -t my-node-app:1.0 .

Explicació

  • docker build: La comanda per construir una imatge Docker.
  • -t my-node-app:1.0: Etiqueta la imatge amb el nom my-node-app i la versió 1.0.
  • .: Especifica el context de construcció, que és el directori actual.

Gestionar Imatges Docker

Després de construir imatges, necessites gestionar-les de manera efectiva.

Llistar Imatges

Per llistar totes les imatges Docker en el teu sistema, utilitza:

docker images

Eliminar Imatges

Per eliminar una imatge, utilitza:

docker rmi my-node-app:1.0

Etiquetar Imatges

Per etiquetar una imatge existent amb una nova etiqueta, utilitza:

docker tag my-node-app:1.0 my-node-app:latest

Instruccions Avançades del Dockerfile

A mesura que et sentis més còmode amb Docker, pots utilitzar instruccions més avançades per optimitzar les teves imatges.

Construccions Multi-Etapa

Les construccions multi-etapa ajuden a crear imatges més petites i eficients separant l'entorn de construcció de l'entorn d'execució.

# Etapa 1: Construcció
FROM node:14 AS build
WORKDIR /usr/src/app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# Etapa 2: Producció
FROM node:14-alpine
WORKDIR /usr/src/app
COPY --from=build /usr/src/app/dist ./dist
EXPOSE 8080
CMD ["node", "dist/app.js"]

Explicació

  • AS build: Nomena la primera etapa build.
  • COPY --from=build /usr/src/app/dist ./dist: Copia l'aplicació construïda des de l'etapa build a l'etapa de producció.

Conclusió

Crear imatges Docker és una habilitat crucial per a la contenedorització d'aplicacions. Comprenent i practicant l'ús de Dockerfiles, construint imatges i gestionant-les, pots desplegar i escalar les teves aplicacions de manera eficient. A mesura que avances, aprofitar tècniques com les construccions multi-etapa pot optimitzar encara més les teves imatges Docker per a l'ús en producció.
