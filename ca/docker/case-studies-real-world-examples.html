Introducció

En aquesta secció, explorarem diversos estudis de cas i exemples del món real per entendre com s'utilitza Docker en diferents escenaris. Aquests exemples t'ajudaran a comprendre les aplicacions pràctiques de Docker, des de casos d'ús senzills fins a implementacions més complexes i avançades.

Estudi de cas 1: Simplificació dels entorns de desenvolupament

Problema

Un equip de desenvolupament de programari està tenint problemes amb entorns de desenvolupament inconsistents. Diferents desenvolupadors tenen diferents versions de dependències, cosa que porta a problemes de "funciona a la meva màquina".

Solució

  • Contenidors Docker: Utilitzar Docker per crear un entorn de desenvolupament consistent.
  • Docker Compose: Utilitzar Docker Compose per gestionar aplicacions multi-contenidor.

Implementació

  1. Crear un Dockerfile:

    # Utilitzar una imatge oficial de Python com a imatge pare
FROM python:3.8-slim

# Establir el directori de treball dins del contenidor
WORKDIR /app

# Copiar el contingut del directori actual al contenidor a /app
COPY . /app

# Instal·lar qualsevol paquet necessari especificat a requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Fer que el port 80 estigui disponible per al món fora d'aquest contenidor
EXPOSE 80

# Definir una variable d'entorn
ENV NAME World

# Executar app.py quan es llanci el contenidor
CMD ["python", "app.py"]

  2. Crear un fitxer docker-compose.yml:

    version: '3'
services:
  web:
    build: .
    ports:
      - "5000:80"

  3. Executar l'aplicació:

    docker-compose up

Beneficis

  • Entorn de desenvolupament consistent per a tots els membres de l'equip.
  • Fàcil de configurar i desmuntar l'entorn de desenvolupament.
  • Simplifica la gestió de dependències.

Estudi de cas 2: Arquitectura de microserveis

Problema

Una empresa vol descompondre la seva aplicació monolítica en microserveis per millorar l'escalabilitat i la mantenibilitat.

Solució

  • Contenidors Docker: Utilitzar Docker per conteniritzar cada microservei.
  • Docker Compose: Utilitzar Docker Compose per gestionar l'aplicació multi-contenidor.

Implementació

  1. Crear Dockerfiles per a cada microservei:

    • Servei d'usuari: 
      FROM node:14
WORKDIR /usr/src/app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 8080
CMD ["node", "user-service.js"]
    • Servei de comandes: 
      FROM node:14
WORKDIR /usr/src/app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 8081
CMD ["node", "order-service.js"]

  2. Crear un fitxer docker-compose.yml:

    version: '3'
services:
  user-service:
    build: ./user-service
    ports:
      - "8080:8080"
  order-service:
    build: ./order-service
    ports:
      - "8081:8081"

  3. Executar l'aplicació:

    docker-compose up

Beneficis

  • Millora de l'escalabilitat i la mantenibilitat.
  • Cada microservei es pot desenvolupar, desplegar i escalar de manera independent.
  • Més fàcil de gestionar les dependències de cada microservei.

Estudi de cas 3: Integració i desplegament continus (CI/CD)

Problema

Una empresa vol automatitzar el procés de desplegament per assegurar-se que els últims canvis de codi sempre es despleguen en producció.

Solució

  • Contenidors Docker: Utilitzar Docker per crear entorns consistents per a proves i desplegaments.
  • Eines CI/CD: Integrar Docker amb eines CI/CD com Jenkins, GitLab CI o GitHub Actions.

Implementació

  1. Crear un pipeline de Jenkins:

    pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                script {
                    docker.build('my-app')
                }
            }
        }
        stage('Test') {
            steps {
                script {
                    docker.image('my-app').inside {
                        sh 'pytest'
                    }
                }
            }
        }
        stage('Deploy') {
            steps {
                script {
                    docker.image('my-app').push('my-repo/my-app:latest')
                }
            }
        }
    }
}

  2. Crear un workflow de GitHub Actions:

    name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Build Docker image
        run: docker build -t my-app .

      - name: Run tests
        run: docker run my-app pytest

      - name: Push Docker image
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin
          docker tag my-app my-repo/my-app:latest
          docker push my-repo/my-app:latest

Beneficis

  • Procés de desplegament automatitzat i consistent.
  • Cicle de retroalimentació més ràpid amb proves automatitzades.
  • Reducció d'errors humans en el procés de desplegament.

Conclusió

Examinant aquests estudis de cas, hem vist com es pot aplicar Docker per resoldre problemes del món real en el desenvolupament de programari, l'arquitectura de microserveis i els pipelines de CI/CD. Docker proporciona una manera consistent i eficient de gestionar entorns, dependències i desplegaments, convertint-se en una eina inestimable per a les pràctiques modernes de desenvolupament de programari.
