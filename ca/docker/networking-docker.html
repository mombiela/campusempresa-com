Introducció a les Xarxes de Docker

Les xarxes de Docker són un aspecte crucial de les aplicacions contenidoritzades, permetent que els contenidors es comuniquin entre ells i amb sistemes externs. Entendre les xarxes de Docker és essencial per desplegar i gestionar aplicacions de manera efectiva.

Conceptes Clau

  • Xarxes de Contenidors: Com es comuniquen els contenidors dins d'un host Docker.
  • Controladors de Xarxa: Diferents tipus de controladors de xarxa proporcionats per Docker.
  • Xarxes Bridge: El controlador de xarxa predeterminat per als contenidors Docker.
  • Xarxes Host: Els contenidors comparteixen l'espai de noms de xarxa de l'host.
  • Xarxes Overlay: Utilitzades per a xarxes multi-host.
  • Xarxes Macvlan: Assigna una adreça MAC a cada contenidor per a l'accés directe a la xarxa.
  • Configuració de Xarxa: Personalització dels paràmetres de xarxa per als contenidors.

Xarxes Bridge

Les xarxes bridge són el tipus predeterminat de xarxa creat per Docker. Permeten que els contenidors es comuniquin entre ells en el mateix host.

Creació d'una Xarxa Bridge

docker network create my_bridge_network

Executar Contenidors en una Xarxa Bridge

docker run -d --name container1 --network my_bridge_network nginx
docker run -d --name container2 --network my_bridge_network nginx

Inspeccionar la Xarxa

docker network inspect my_bridge_network

Xarxes Host

Les xarxes host permeten que els contenidors utilitzin directament l'stack de xarxa de l'host. Això pot ser útil per a aplicacions sensibles al rendiment.

Executar un Contenidor en la Xarxa Host

docker run -d --name container3 --network host nginx

Xarxes Overlay

Les xarxes overlay permeten la comunicació entre contenidors a través de diferents hosts Docker, cosa que és essencial per a aplicacions distribuïdes.

Creació d'una Xarxa Overlay

docker network create -d overlay my_overlay_network

Executar Contenidors en una Xarxa Overlay

docker run -d --name container4 --network my_overlay_network nginx
docker run -d --name container5 --network my_overlay_network nginx

Xarxes Macvlan

Les xarxes macvlan assignen una adreça MAC única a cada contenidor, permetent que apareguin com a dispositius físics a la xarxa.

Creació d'una Xarxa Macvlan

docker network create -d macvlan \
  --subnet=192.168.1.0/24 \
  --gateway=192.168.1.1 \
  -o parent=eth0 my_macvlan_network

Executar un Contenidor en una Xarxa Macvlan

docker run -d --name container6 --network my_macvlan_network nginx

Configuració de Xarxa

Docker permet la personalització dels paràmetres de xarxa per adaptar-se a requisits específics.

Servidors DNS Personalitzats

docker run -d --name container7 --network my_bridge_network --dns 8.8.8.8 nginx

Exposar Ports

docker run -d --name container8 -p 8080:80 nginx

Resum

Entendre les xarxes de Docker és essencial per desplegar i gestionar aplicacions contenidoritzades. Docker proporciona diversos controladors de xarxa, incloent bridge, host, overlay i macvlan, cadascun adequat per a diferents casos d'ús. En dominar aquests conceptes, pots assegurar una comunicació eficient i segura entre els teus contenidors i sistemes externs.
