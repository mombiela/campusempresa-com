Project 5: Desplegament a Kubernetes
Documentació oficial de Docker ►

Introducció

En aquest projecte, explorarem com desplegar contenidors Docker a un clúster de Kubernetes. Kubernetes és una eina d'orquestració potent que automatitza el desplegament, l'escalat i la gestió d'aplicacions contenidoritzades. Al final d'aquest projecte, tindràs una comprensió sòlida de com desplegar i gestionar contenidors Docker utilitzant Kubernetes.

Conceptes Clau

  • Clúster de Kubernetes: Un conjunt de nodes (màquines) que executen aplicacions contenidoritzades gestionades per Kubernetes.
  • Pods: Les unitats desplegables més petites a Kubernetes, que poden contenir un o més contenidors.
  • Serveis: Abstraccions que defineixen un conjunt lògic de Pods i una política per accedir-hi.
  • Desplegaments: Proporcionen actualitzacions declaratives a les aplicacions i gestionen el desplegament de Pods.
  • ConfigMaps i Secrets: Mecanismes per injectar dades de configuració als Pods.

Prerequisits

Abans de començar aquest projecte, assegura't de tenir:

  • Una comprensió bàsica de Docker i la contenidorització.
  • Docker instal·lat a la teva màquina local.
  • Minikube o un clúster de Kubernetes configurat.

Guia Pas a Pas

Pas 1: Configurar Minikube

Minikube és una eina que et permet executar Kubernetes localment. Crea un clúster de Kubernetes d'un sol node a la teva màquina local.

  1. Instal·lar Minikube:

    • Segueix les instruccions d'instal·lació per al teu sistema operatiu a la documentació oficial de Minikube.

  2. Iniciar Minikube:

    minikube start

  3. Verificar l'Estat de Minikube:

    minikube status

Pas 2: Crear una Imatge Docker

Crea una imatge Docker senzilla per desplegar a Kubernetes.

  1. Dockerfile:

    FROM nginx:alpine
COPY index.html /usr/share/nginx/html

  2. Construir la Imatge Docker:

    docker build -t my-nginx .

  3. Verificar la Imatge:

    docker images

Pas 3: Desplegar a Kubernetes

Desplega la imatge Docker al clúster de Kubernetes.

  1. Crear un Desplegament:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: my-nginx:latest
        ports:
        - containerPort: 80

  2. Aplicar el Desplegament:

    kubectl apply -f deployment.yaml

  3. Verificar el Desplegament:

    kubectl get deployments
kubectl get pods

Pas 4: Exposar el Desplegament

Exposa el desplegament per fer-lo accessible des de fora del clúster.

  1. Crear un Servei:

    apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer

  2. Aplicar el Servei:

    kubectl apply -f service.yaml

  3. Obtenir l'URL del Servei:

    minikube service nginx-service --url

Pas 5: Gestionar Configuracions

Utilitza ConfigMaps i Secrets per gestionar configuracions.

  1. Crear un ConfigMap:

    kubectl create configmap nginx-config --from-file=nginx.conf

  2. Crear un Secret:

    kubectl create secret generic my-secret --from-literal=password=my-password

  3. Utilitzar ConfigMap i Secret en el Desplegament:

    spec:
  containers:
  - name: nginx
    image: my-nginx:latest
    ports:
    - containerPort: 80
    volumeMounts:
    - name: config-volume
      mountPath: /etc/nginx/nginx.conf
      subPath: nginx.conf
  volumes:
  - name: config-volume
    configMap:
      name: nginx-config

Conclusió

En aquest projecte, hem cobert els passos essencials per desplegar contenidors Docker a un clúster de Kubernetes. Hem començat configurant Minikube, creat una imatge Docker, desplegat-la a Kubernetes, exposat-la mitjançant un servei i gestionat configuracions utilitzant ConfigMaps i Secrets. Seguint aquests passos, ara tens els coneixements fonamentals per desplegar i gestionar aplicacions contenidoritzades utilitzant Kubernetes.
