Introducció

Integrar Docker amb proveïdors de núvol et permet desplegar, gestionar i escalar aplicacions contenidoritzades sense problemes. Aquesta secció cobrirà els conceptes bàsics d'integrar Docker amb proveïdors de núvol populars com AWS, Google Cloud Platform (GCP) i Microsoft Azure.

Conceptes Clau

  • Orquestració de Contenidors: Eines com Kubernetes i Docker Swarm ajuden a gestionar contenidors en un entorn de núvol.
  • Serveis de Núvol: Cada proveïdor de núvol ofereix serveis per executar contenidors Docker, com AWS ECS, GCP GKE i Azure AKS.
  • Pipelines CI/CD: Les pipelines d'Integració Contínua i Desplegament Continu es poden integrar amb Docker per automatitzar el procés de desplegament.

AWS (Amazon Web Services)

Configuració de Docker a AWS ECS (Elastic Container Service)

  1. Crear un Clúster ECS:

    • Ves al tauler de control d'ECS.
    • Fes clic a "Create Cluster".
    • Tria la plantilla de clúster adequada (per exemple, EC2 Linux + Networking).
    • Configura els paràmetres del clúster i crea'l.

  2. Definir una Definició de Tasca:

    • Ves a la secció de Definicions de Tasques.
    • Fes clic a "Create new Task Definition".
    • Tria el tipus de llançament (EC2 o Fargate).
    • Defineix els paràmetres del contenidor, incloent la imatge Docker, la memòria i els requisits de CPU.

  3. Executar una Tasca:

    • Ves al clúster ECS que has creat.
    • Fes clic a "Tasks" i després a "Run new Task".
    • Selecciona la definició de tasca i configura els paràmetres de la tasca.
    • Fes clic a "Run Task".
{
  "family": "my-task",
  "containerDefinitions": [
    {
      "name": "my-container",
      "image": "my-docker-image",
      "memory": 512,
      "cpu": 256,
      "essential": true
    }
  ]
}

Ús d'AWS Fargate

AWS Fargate et permet executar contenidors sense gestionar servidors.

  • Crear un Clúster Fargate: Similar a crear un clúster ECS però tria el tipus de llançament Fargate.
  • Definir una Tasca Fargate: Similar a definir una tasca ECS però amb paràmetres específics de Fargate.

Google Cloud Platform (GCP)

Configuració de Docker a GKE (Google Kubernetes Engine)

  1. Crear un Clúster GKE:

    • Ves al tauler de control de GKE.
    • Fes clic a "Create Cluster".
    • Configura els paràmetres del clúster i crea'l.

  2. Desplegar un Contenidor Docker:

    • Utilitza kubectl per interactuar amb el clúster GKE.
    • Crea un fitxer YAML de desplegament per al teu contenidor Docker.
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-container
        image: my-docker-image
        ports:
        - containerPort: 80
  1. Desplegar l'Aplicació:
    • Aplica el desplegament utilitzant kubectl apply -f deployment.yaml.
    • Exposa el desplegament utilitzant kubectl expose deployment my-deployment --type=LoadBalancer --port=80.

Microsoft Azure

Configuració de Docker a Azure AKS (Azure Kubernetes Service)

  1. Crear un Clúster AKS:

    • Ves al tauler de control d'AKS.
    • Fes clic a "Add" per crear un nou clúster.
    • Configura els paràmetres del clúster i crea'l.

  2. Desplegar un Contenidor Docker:

    • Utilitza az aks get-credentials per configurar kubectl per utilitzar el clúster AKS.
    • Crea un fitxer YAML de desplegament similar a l'exemple de GKE.

  3. Desplegar l'Aplicació:

    • Aplica el desplegament utilitzant kubectl apply -f deployment.yaml.
    • Exposa el desplegament utilitzant kubectl expose deployment my-deployment --type=LoadBalancer --port=80.

Integració CI/CD

Ús de Jenkins amb Docker i Proveïdors de Núvol

  1. Instal·lar Jenkins: Configura Jenkins en un servidor o utilitza un servei gestionat de Jenkins.
  2. Configurar la Pipeline de Jenkins:
    • Crea un Jenkinsfile al teu repositori.
    • Defineix etapes per construir, provar i desplegar contenidors Docker.
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                script {
                    docker.build('my-docker-image')
                }
            }
        }
        stage('Deploy') {
            steps {
                script {
                    // Exemple per AWS ECS
                    sh 'aws ecs update-service --cluster my-cluster --service my-service --force-new-deployment'
                }
            }
        }
    }
}

Conclusió

Integrar Docker amb proveïdors de núvol com AWS, GCP i Azure et permet aprofitar l'escalabilitat i flexibilitat del núvol per gestionar les teves aplicacions contenidoritzades de manera efectiva. En entendre els serveis i eines específiques que ofereix cada proveïdor, pots desplegar, gestionar i escalar les teves aplicacions amb facilitat. A més, integrar Docker amb pipelines CI/CD pot automatitzar el procés de desplegament, assegurant que les teves aplicacions estiguin sempre actualitzades i funcionant sense problemes.
