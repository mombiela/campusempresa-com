Introducció al CI/CD

  • Integració Contínua (CI): La pràctica d'integrar automàticament els canvis de codi de múltiples contribuïdors en un repositori compartit diverses vegades al dia.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa totes les etapes de la teva pipeline de producció a producció.

Beneficis d'Utilitzar Docker en CI/CD

  • Consistència: Docker assegura que l'aplicació s'executi de la mateixa manera independentment d'on es desplegui.
  • Aïllament: Cada aplicació s'executa en el seu propi contenidor, evitant conflictes.
  • Escalabilitat: Els contenidors Docker es poden escalar fàcilment cap amunt o cap avall.

Configuració d'una Pipeline de CI/CD amb Docker

Prerequisits

  • Coneixements bàsics de Docker.
  • Un sistema de control de versions (per exemple, Git).
  • Una eina de CI/CD (per exemple, Jenkins, GitLab CI, GitHub Actions).

Guia Pas a Pas

1. Dockeritzar la Teva Aplicació

  • Crea un Dockerfile per definir l'entorn de la teva aplicació.
# Utilitza una imatge oficial de Python com a imatge pare
FROM python:3.8-slim

# Estableix el directori de treball al contenidor
WORKDIR /app

# Copia el contingut del directori actual al contenidor a /app
COPY . /app

# Instal·la qualsevol paquet necessari especificat a requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Fes que el port 80 estigui disponible per al món fora d'aquest contenidor
EXPOSE 80

# Defineix una variable d'entorn
ENV NAME World

# Executa app.py quan es llanci el contenidor
CMD ["python", "app.py"]

2. Construir i Provar la Imatge Docker

  • Construeix la imatge Docker localment per assegurar-te que funciona com s'espera.
docker build -t my-python-app .
  • Executa el contenidor Docker per provar-lo.
docker run -p 4000:80 my-python-app

3. Configurar una Eina de CI/CD

  • Tria una eina de CI/CD i configura-la per utilitzar el teu Dockerfile.
Exemple: GitHub Actions
  • Crea un fitxer .github/workflows/main.yml al teu repositori.
name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v1

    - name: Build Docker image
      run: docker build -t my-python-app .

    - name: Run tests
      run: docker run my-python-app pytest

4. Desplegar el Contenidor Docker

  • Puja la imatge Docker a un registre de contenidors (per exemple, Docker Hub).
docker tag my-python-app mydockerhubusername/my-python-app
docker push mydockerhubusername/my-python-app
  • Configura la teva eina de CI/CD per desplegar la imatge al teu entorn de producció.
Exemple: GitHub Actions (continuació)
    - name: Log in to Docker Hub
      run: echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin

    - name: Push Docker image
      run: docker push mydockerhubusername/my-python-app

    - name: Deploy to production
      run: |
        ssh user@yourserver "docker pull mydockerhubusername/my-python-app && docker run -d -p 80:80 mydockerhubusername/my-python-app"

Conclusió

En aquest tema, hem cobert els aspectes essencials de la configuració d'una pipeline de CI/CD utilitzant Docker. Hem començat amb una introducció als conceptes de CI/CD i els beneficis d'utilitzar Docker. Després, hem seguit els passos per Dockeritzar una aplicació, construir i provar la imatge Docker, configurar una eina de CI/CD i desplegar el contenidor Docker. Seguint aquests passos, pots assegurar un procés de desplegament consistent, aïllat i escalable per a les teves aplicacions.
