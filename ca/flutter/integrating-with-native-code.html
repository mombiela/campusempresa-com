Introducció

Integrar-se amb codi natiu és un aspecte crucial del desenvolupament de Flutter, ja que et permet aprofitar la funcionalitat específica de la plataforma i les biblioteques que no estan disponibles en Dart. Aquesta secció et guiarà a través del procés d'integració de Flutter amb codi natiu tant en plataformes Android com iOS.

Per què integrar-se amb codi natiu?

  • Accedir a funcions específiques de la plataforma: Utilitza funcions com la càmera, GPS i sensors que són específics d'Android o iOS.
  • Aprofitar biblioteques existents: Utilitza biblioteques natives i SDKs que no estan disponibles en Dart.
  • Optimització del rendiment: Executa codi crític per al rendiment de manera nativa per obtenir una millor eficiència.

Canals de mètodes

Els canals de mètodes proporcionen una manera perquè el teu codi Dart es comuniqui amb codi específic de la plataforma utilitzant el pas de missatges asíncrons.

Configuració dels canals de mètodes

  1. Defineix un canal de mètodes en Dart:

    import 'package:flutter/services.dart';

class NativeIntegration {
  static const platform = MethodChannel('com.example/native');

  Future<String> getNativeMessage() async {
    try {
      final String result = await platform.invokeMethod('getNativeMessage');
      return result;
    } on PlatformException catch (e) {
      return "Failed to get native message: '${e.message}'.";
    }
  }
}

  2. Gestiona el canal de mètodes en Android (Kotlin):

    import io.flutter.embedding.engine.FlutterEngine
import io.flutter.plugin.common.MethodChannel

class MainActivity: FlutterActivity() {
    private val CHANNEL = "com.example/native"

    override fun configureFlutterEngine(flutterEngine: FlutterEngine) {
        super.configureFlutterEngine(flutterEngine)
        MethodChannel(flutterEngine.dartExecutor.binaryMessenger, CHANNEL).setMethodCallHandler {
            call, result ->
            if (call.method == "getNativeMessage") {
                val message = getNativeMessage()
                if (message != null) {
                    result.success(message)
                } else {
                    result.error("UNAVAILABLE", "Message not available.", null)
                }
            } else {
                result.notImplemented()
            }
        }
    }

    private fun getNativeMessage(): String {
        return "Hello from Android Native Code!"
    }
}

  3. Gestiona el canal de mètodes en iOS (Swift):

    import Flutter
import UIKit

@UIApplicationMain
@objc class AppDelegate: FlutterAppDelegate {
    override func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
        let controller : FlutterViewController = window?.rootViewController as! FlutterViewController
        let nativeChannel = FlutterMethodChannel(name: "com.example/native",
                                                  binaryMessenger: controller.binaryMessenger)
        nativeChannel.setMethodCallHandler({
            (call: FlutterMethodCall, result: @escaping FlutterResult) -> Void in
            if call.method == "getNativeMessage" {
                result("Hello from iOS Native Code!")
            } else {
                result(FlutterMethodNotImplemented)
            }
        })

        return super.application(application, didFinishLaunchingWithOptions: launchOptions)
    }
}

Canals de plataforma

Els canals de plataforma són una altra manera de comunicar-se entre Dart i codi natiu, però són més flexibles i poden gestionar tipus de dades més complexos.

Configuració dels canals de plataforma

  1. Defineix un canal de plataforma en Dart:

    import 'package:flutter/services.dart';

class NativeIntegration {
  static const platform = BasicMessageChannel<String>('com.example/native', StringCodec());

  Future<String> getNativeMessage() async {
    final String result = await platform.send('getNativeMessage');
    return result;
  }
}

  2. Gestiona el canal de plataforma en Android (Kotlin):

    import io.flutter.embedding.engine.FlutterEngine
import io.flutter.plugin.common.BasicMessageChannel
import io.flutter.plugin.common.StringCodec

class MainActivity: FlutterActivity() {
    private val CHANNEL = "com.example/native"

    override fun configureFlutterEngine(flutterEngine: FlutterEngine) {
        super.configureFlutterEngine(flutterEngine)
        BasicMessageChannel<String>(flutterEngine.dartExecutor.binaryMessenger, CHANNEL, StringCodec.INSTANCE)
            .setMessageHandler { message, reply ->
                if (message == "getNativeMessage") {
                    reply.reply("Hello from Android Native Code!")
                } else {
                    reply.reply(null)
                }
            }
    }
}

  3. Gestiona el canal de plataforma en iOS (Swift):

    import Flutter
import UIKit

@UIApplicationMain
@objc class AppDelegate: FlutterAppDelegate {
    override func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
        let controller : FlutterViewController = window?.rootViewController as! FlutterViewController
        let nativeChannel = FlutterBasicMessageChannel(name: "com.example/native",
                                                       binaryMessenger: controller.binaryMessenger,
                                                       codec: FlutterStringCodec.sharedInstance())
        nativeChannel.setMessageHandler { (message, reply) in
            if message == "getNativeMessage" {
                reply("Hello from iOS Native Code!")
            } else {
                reply(nil)
            }
        }

        return super.application(application, didFinishLaunchingWithOptions: launchOptions)
    }
}

Conclusió

Integrar-se amb codi natiu en Flutter et permet estendre la funcionalitat de la teva aplicació aprofitant funcions i biblioteques específiques de la plataforma. Utilitzant canals de mètodes i canals de plataforma, pots comunicar-te de manera eficient entre Dart i codi natiu, assegurant que la teva aplicació pugui utilitzar totes les capacitats de la plataforma subjacent.

Entendre aquestes tècniques d'integració és essencial per al desenvolupament avançat de Flutter, permetent-te crear aplicacions més potents i riques en funcions.
