Introducció als Widgets

Els widgets són els blocs de construcció d'una aplicació Flutter. Descriuen com hauria de ser la seva vista donada la seva configuració i estat actuals. Els widgets són immutables i s'utilitzen per construir la interfície d'usuari.

Tipus de Widgets

Flutter proporciona una àmplia gamma de widgets, però es poden classificar àmpliament en dos tipus:

  • Widgets sense estat: Aquests widgets no tenen cap estat intern. Són immutables i es reconstrueixen quan la seva configuració canvia.
  • Widgets amb estat: Aquests widgets mantenen un estat intern que pot canviar amb el temps. Són mutables i poden reconstruir-se a si mateixos quan el seu estat canvia.

Creació d'un Widget sense estat

Un widget sense estat és un widget que no requereix estat mutable. Aquí teniu un exemple d'un widget senzill sense estat:

import 'package:flutter/material.dart';

class MyStatelessWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Exemple de Widget sense estat'),
      ),
      body: Center(
        child: Text('Hola, Flutter!'),
      ),
    );
  }
}

void main() => runApp(MaterialApp(
  home: MyStatelessWidget(),
));

Explicació

  • MyStatelessWidget extén StatelessWidget.
  • El mètode build descriu la part de la interfície d'usuari representada pel widget.
  • Scaffold proporciona un marc per a l'estructura visual bàsica del disseny material.
  • AppBar és una barra d'aplicació de disseny material.
  • Center centra el seu widget fill.
  • Text mostra una cadena de text.

Creació d'un Widget amb estat

Un widget amb estat és un widget que té estat mutable. Aquí teniu un exemple d'un widget senzill amb estat:

import 'package:flutter/material.dart';

class MyStatefulWidget extends StatefulWidget {
  @override
  _MyStatefulWidgetState createState() => _MyStatefulWidgetState();
}

class _MyStatefulWidgetState extends State<MyStatefulWidget> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Exemple de Widget amb estat'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text('Has premut el botó tantes vegades:'),
            Text(
              '$_counter',
              style: Theme.of(context).textTheme.headline4,
            ),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _incrementCounter,
        tooltip: 'Incrementar',
        child: Icon(Icons.add),
      ),
    );
  }
}

void main() => runApp(MaterialApp(
  home: MyStatefulWidget(),
));

Explicació

  • MyStatefulWidget extén StatefulWidget.
  • _MyStatefulWidgetState és la classe d'estat que extén State<MyStatefulWidget>.
  • _counter és una variable enter privada que manté el recompte actual.
  • _incrementCounter és un mètode que incrementa _counter i crida setState per actualitzar la interfície d'usuari.
  • El mètode build descriu la part de la interfície d'usuari representada pel widget.
  • Column disposa els seus fills en una matriu vertical.
  • FloatingActionButton és un botó circular que flota sobre el contingut.

Widgets Utilitzats Comúment

Aquí teniu alguns widgets utilitzats comunament a Flutter:

  • Container: Un widget de conveniència que combina widgets comuns de pintura, posicionament i mida.
  • Row: Disposa els seus fills en una matriu horitzontal.
  • Column: Disposa els seus fills en una matriu vertical.
  • Stack: Permet la superposició de widgets un sobre l'altre.
  • ListView: Una llista desplaçable de widgets disposats linealment.
  • GridView: Una matriu 2D desplaçable de widgets.

Exemple d'un Widget Container

Container(
  height: 100.0,
  width: 100.0,
  color: Colors.blue,
  child: Center(
    child: Text('Hola'),
  ),
)

Exemple d'un Widget Row

Row(
  mainAxisAlignment: MainAxisAlignment.spaceEvenly,
  children: <Widget>[
    Icon(Icons.star, color: Colors.red),
    Icon(Icons.star, color: Colors.green),
    Icon(Icons.star, color: Colors.blue),
  ],
)

Cicle de Vida del Widget

Entendre el cicle de vida d'un widget és crucial per gestionar l'estat i els recursos de manera efectiva.

Cicle de Vida del Widget amb Estat

  1. createState: Es crida quan el framework crea un widget amb estat.
  2. initState: Es crida quan aquest objecte s'insereix a l'arbre.
  3. didChangeDependencies: Es crida quan una dependència d'aquest objecte State canvia.
  4. build: Descriu la part de la interfície d'usuari representada per aquest widget.
  5. didUpdateWidget: Es crida sempre que la configuració del widget canvia.
  6. setState: Notifica al framework que l'estat intern ha canviat.
  7. deactivate: Es crida quan aquest objecte es retira de l'arbre.
  8. dispose: Es crida quan aquest objecte es retira permanentment de l'arbre.

Conclusió

Els widgets són els blocs de construcció principals d'una aplicació Flutter. Entendre la diferència entre widgets sense estat i amb estat, així com el cicle de vida dels widgets amb estat, és essencial per construir aplicacions Flutter eficients i efectives. Dominant aquests conceptes, podeu crear interfícies d'usuari dinàmiques i responsives amb facilitat.
