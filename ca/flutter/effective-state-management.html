La gestió de l'estat és un aspecte crucial per construir aplicacions robustes i escalables a Flutter. Aquest tema et guiarà des dels conceptes bàsics fins a tècniques avançades per gestionar l'estat a les teves aplicacions Flutter.

Introducció a la Gestió de l'Estat

La gestió de l'estat es refereix a la manipulació de l'estat d'una aplicació, que inclou les dades i la resposta de la interfície d'usuari a aquestes dades. A Flutter, la gestió de l'estat és essencial per crear aplicacions responsives i interactives.

Conceptes Clau

  • Estat: La informació que pot canviar amb el temps a la teva aplicació.
  • Estat del Widget: L'estat associat amb un widget, que pot ser efímer (local) o a nivell d'aplicació (global).
  • StatefulWidget: Un widget que té estat mutable.
  • StatelessWidget: Un widget que no té estat mutable.

Gestió de l'Estat Efímer

L'estat efímer és l'estat que pots gestionar fàcilment dins d'un sol widget. Això és adequat per a casos d'ús senzills on l'estat no necessita ser compartit entre múltiples widgets.

Utilitzant StatefulWidget

import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: CounterScreen(),
    );
  }
}

class CounterScreen extends StatefulWidget {
  @override
  _CounterScreenState createState() => _CounterScreenState();
}

class _CounterScreenState extends State<CounterScreen> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Counter App'),
      ),
      body: Center(
        child: Text(
          'Counter: $_counter',
          style: TextStyle(fontSize: 24),
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _incrementCounter,
        child: Icon(Icons.add),
      ),
    );
  }
}
  • Explicació: Aquest exemple demostra una aplicació de comptador simple utilitzant StatefulWidget. La variable _counter conté l'estat, i setState s'utilitza per actualitzar la interfície d'usuari quan l'estat canvia.

Gestió de l'Estat Global

Per a aplicacions més complexes, necessites gestionar l'estat a través de múltiples widgets. Flutter proporciona diverses aproximacions per a la gestió de l'estat global.

Paquet Provider

Provider és un paquet popular per a la gestió de l'estat a Flutter. És fàcil d'utilitzar i s'integra bé amb el framework de Flutter.

Configurant Provider

  1. Afegeix el paquet provider al teu fitxer pubspec.yaml:

    dependencies:
  flutter:
    sdk: flutter
  provider: ^6.0.0

  2. Crea una classe d'estat:

    import 'package:flutter/foundation.dart';

class Counter with ChangeNotifier {
  int _count = 0;

  int get count => _count;

  void increment() {
    _count++;
    notifyListeners();
  }
}

  3. Utilitza el Provider a la teva aplicació:

    import 'package:flutter/material.dart';
import 'package:provider/provider.dart';
import 'counter.dart'; // Importa la classe Counter

void main() {
  runApp(
    ChangeNotifierProvider(
      create: (context) => Counter(),
      child: MyApp(),
    ),
  );
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: CounterScreen(),
    );
  }
}

class CounterScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final counter = Provider.of<Counter>(context);

    return Scaffold(
      appBar: AppBar(
        title: Text('Provider Counter App'),
      ),
      body: Center(
        child: Text(
          'Counter: ${counter.count}',
          style: TextStyle(fontSize: 24),
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: counter.increment,
        child: Icon(Icons.add),
      ),
    );
  }
}
  • Explicació: Aquest exemple mostra com utilitzar el paquet Provider per a la gestió de l'estat global. La classe Counter extén ChangeNotifier, i ChangeNotifierProvider s'utilitza per proporcionar l'estat a l'arbre de widgets.

Gestió Avançada de l'Estat

Per a aplicacions molt complexes, podries necessitar solucions més avançades de gestió de l'estat com Riverpod, Bloc o Redux.

Riverpod

Riverpod és una biblioteca moderna de gestió de l'estat que ofereix una manera més senzilla i robusta de gestionar l'estat.

Configurant Riverpod

  1. Afegeix el paquet riverpod al teu fitxer pubspec.yaml:

    dependencies:
  flutter:
    sdk: flutter
  flutter_riverpod: ^1.0.0

  2. Crea un provider:

    import 'package:flutter_riverpod/flutter_riverpod.dart';

final counterProvider = StateProvider<int>((ref) => 0);

  3. Utilitza el provider a la teva aplicació:

    import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

void main() {
  runApp(
    ProviderScope(
      child: MyApp(),
    ),
  );
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: CounterScreen(),
    );
  }
}

class CounterScreen extends ConsumerWidget {
  @override
  Widget build(BuildContext context, ScopedReader watch) {
    final counter = watch(counterProvider).state;

    return Scaffold(
      appBar: AppBar(
        title: Text('Riverpod Counter App'),
      ),
      body: Center(
        child: Text(
          'Counter: $counter',
          style: TextStyle(fontSize: 24),
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () => context.read(counterProvider).state++,
        child: Icon(Icons.add),
      ),
    );
  }
}
  • Explicació: Aquest exemple demostra com utilitzar Riverpod per a la gestió de l'estat. El counterProvider és un StateProvider que conté un estat enter, i ConsumerWidget s'utilitza per llegir i actualitzar l'estat.

Conclusió

Una gestió efectiva de l'estat és essencial per construir aplicacions Flutter escalables i mantenibles. Començant amb una gestió d'estat simple utilitzant StatefulWidget, pots progressar a tècniques més avançades com Provider i Riverpod per gestionar l'estat global. Entendre i escollir l'aproximació correcta de gestió de l'estat per a la teva aplicació t'ajudarà a crear aplicacions més eficients i responsives.
