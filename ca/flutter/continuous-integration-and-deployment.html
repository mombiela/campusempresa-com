La Integració Contínua (CI) i el Desplegament Continu (CD) són pràctiques essencials en el desenvolupament modern de programari. Ajuden a garantir que el teu codi estigui sempre en un estat desplegable i que les noves funcionalitats i correccions d'errors es puguin lliurar ràpidament i de manera fiable. En aquesta secció, explorarem com configurar CI/CD per a un projecte Flutter, des de nivells bàsics fins a avançats.

Introducció a CI/CD

  • Integració Contínua (CI): La pràctica d'integrar automàticament els canvis de codi de múltiples contribuïdors en un repositori compartit diverses vegades al dia.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa totes les etapes de la teva pipeline de producció als teus usuaris.

Configuració de CI per a Flutter

Triar un Servei de CI

Hi ha diversos serveis de CI disponibles que suporten Flutter, incloent:

  • GitHub Actions
  • Travis CI
  • CircleCI
  • Bitrise

Per a aquest curs, utilitzarem GitHub Actions ja que és gratuït per a repositoris públics i s'integra perfectament amb GitHub.

Flux de Treball Bàsic de CI amb GitHub Actions

  1. Crea un directori .github/workflows en el teu projecte Flutter.
  2. Afegeix un nou fitxer de flux de treball, per exemple, ci.yml.
name: Flutter CI

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Flutter
      uses: subosito/flutter-action@v1
      with:
        flutter-version: '2.5.0'
    - name: Install dependencies
      run: flutter pub get
    - name: Run tests
      run: flutter test

Explicació

  • name: El nom del flux de treball.
  • on: Especifica els esdeveniments que desencadenen el flux de treball. Aquí, es desencadena en empentes i sol·licituds d'extracció a la branca main.
  • jobs: Defineix les tasques que s'executaran com a part del flux de treball.
  • steps: Els passos individuals a executar en la tasca.
    • actions/checkout@v2: Comprova el repositori.
    • subosito/flutter-action@v1: Configura Flutter.
    • flutter pub get: Instal·la les dependències.
    • flutter test: Executa les proves.

Configuració de CD per a Flutter

Desplegament a Firebase App Distribution

Firebase App Distribution és una gran eina per distribuir les teves aplicacions Flutter als testers.

  1. Afegeix Firebase al teu projecte Flutter.
  2. Crea un nou fitxer de flux de treball, per exemple, cd.yml.
name: Flutter CD

on:
  push:
    tags:
      - 'v*.*.*'

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Flutter
      uses: subosito/flutter-action@v1
      with:
        flutter-version: '2.5.0'
    - name: Install dependencies
      run: flutter pub get
    - name: Build APK
      run: flutter build apk --release
    - name: Upload to Firebase App Distribution
      uses: wzieba/Firebase-Distribution-Github-Action@v1
      with:
        appId: ${{ secrets.FIREBASE_APP_ID }}
        token: ${{ secrets.FIREBASE_TOKEN }}
        groups: testers
        file: build/app/outputs/flutter-apk/app-release.apk

Explicació

  • on: El flux de treball es desencadena en empentes a etiquetes que segueixen el patró v*.*.* (per exemple, v1.0.0).
  • steps:
    • flutter build apk --release: Construeix l'APK de producció.
    • wzieba/Firebase-Distribution-Github-Action@v1: Carrega l'APK a Firebase App Distribution.

Gestió de Secrets

  • FIREBASE_APP_ID: L'ID de l'aplicació per al teu projecte Firebase.
  • FIREBASE_TOKEN: El token per a l'autenticació de Firebase CLI.

Pots afegir aquests secrets en la configuració del teu repositori GitHub sota Settings > Secrets.

Pràctiques Avançades de CI/CD

Executar Proves d'Integració

Les proves d'integració es poden executar en dispositius reals utilitzant serveis com Firebase Test Lab.

- name: Run integration tests
  run: flutter drive --target=test_driver/app.dart

Cobertura de Codi

Pots generar informes de cobertura de codi i carregar-los a serveis com Codecov.

- name: Generate code coverage
  run: flutter test --coverage
- name: Upload coverage to Codecov
  uses: codecov/codecov-action@v1
  with:
    file: coverage/lcov.info

Linting i Formatació

Assegura la qualitat del codi afegint passos de linting i formatació.

- name: Analyze code
  run: flutter analyze
- name: Format code
  run: flutter format --set-exit-if-changed .

Conclusió

Configurar CI/CD per al teu projecte Flutter pot millorar significativament el teu flux de treball de desenvolupament automatitzant tasques repetitives i assegurant que el teu codi estigui sempre en un estat desplegable. Seguint els passos descrits en aquesta secció, pots configurar una pipeline robusta de CI/CD que inclou construir, provar i desplegar les teves aplicacions Flutter.

En dominar aquestes pràctiques, podràs lliurar aplicacions d'alta qualitat més ràpidament i de manera més eficient, fent-te un desenvolupador més efectiu i productiu.
