Introducció a l'Arquitectura de Flutter

Flutter és un kit d'eines de desenvolupament de programari d'interfície d'usuari de codi obert creat per Google. S'utilitza per desenvolupar aplicacions multiplataforma per a Android, iOS, Linux, macOS, Windows i la web des d'una única base de codi. Comprendre l'arquitectura de Flutter és crucial per construir aplicacions eficients i escalables.

Components Clau de l'Arquitectura de Flutter

  • Motor de Flutter: El nucli de Flutter, escrit en C++, responsable del renderitzat, la gestió d'entrades i la interfície amb la plataforma.
  • Framework Dart: Una col·lecció de biblioteques escrites en Dart, que proporcionen les eines necessàries per construir aplicacions Flutter.
  • Widgets: Els blocs de construcció d'una aplicació Flutter, utilitzats per crear la interfície d'usuari.
  • Renderitzat: El procés de convertir widgets en píxels a la pantalla.
  • Canals de Plataforma: Mecanismes per comunicar-se entre Dart i la plataforma (iOS/Android).

Motor de Flutter

El Motor de Flutter és responsable de:

  • Renderitzar la interfície d'usuari.
  • Gestionar esdeveniments d'entrada.
  • Gestionar el temps d'execució de Dart i la recollida d'escombraries.
  • Comunicar-se amb les API específiques de la plataforma.

Exemple: Estructura Bàsica d'una Aplicació Flutter

import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Exemple d’Arquitectura Flutter'),
        ),
        body: Center(
          child: Text('Hola, Flutter!'),
        ),
      ),
    );
  }
}

Explicació:

  • main(): El punt d'entrada de l'aplicació.
  • MyApp: Un widget sense estat que construeix l'estructura principal de l'aplicació.
  • MaterialApp: Un widget de conveniència que embolcalla diversos widgets comunament requerits per a aplicacions de disseny material.
  • Scaffold: Implementa l'estructura visual bàsica del disseny material.
  • AppBar: Una barra d'aplicació de disseny material.
  • Center: Centra el seu widget fill.
  • Text: Un widget que mostra una cadena de text.

Framework Dart

El framework Dart proporciona:

  • Biblioteca de Fonaments: Classes i funcions bàsiques.
  • Animació i Moviment: Eines per crear animacions.
  • Pintura i Efectes: Eines per dibuixar i aplicar efectes visuals.
  • Gestos: Gestió d'esdeveniments tàctils i de punter.
  • Widgets: Un conjunt complet de components d'interfície d'usuari preconstruïts.

Widgets

Els widgets són els blocs de construcció principals d'una aplicació Flutter. Descriuen com hauria de ser la seva vista donada la seva configuració i estat actuals.

Exemple: Widgets amb Estat vs Sense Estat

class MyStatelessWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text('Sóc un widget sense estat');
  }
}

class MyStatefulWidget extends StatefulWidget {
  @override
  _MyStatefulWidgetState createState() => _MyStatefulWidgetState();
}

class _MyStatefulWidgetState extends State<MyStatefulWidget> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: <Widget>[
        Text('Comptador: $_counter'),
        ElevatedButton(
          onPressed: _incrementCounter,
          child: Text('Incrementar'),
        ),
      ],
    );
  }
}

Explicació:

  • StatelessWidget: Un widget que no requereix estat mutable.
  • StatefulWidget: Un widget que té estat mutable.
  • setState(): Un mètode per actualitzar l'estat i desencadenar una reconstrucció del widget.

Renderitzat

El renderitzat en Flutter implica convertir l'arbre de widgets en un arbre de renderitzat, que després es pinta a la pantalla.

Procés de Renderitzat

  1. Fase de Construcció: Els widgets es construeixen i formen un arbre de widgets.
  2. Fase de Disseny: L'arbre de widgets es transforma en un arbre de renderitzat, calculant la mida i la posició de cada widget.
  3. Fase de Pintura: L'arbre de renderitzat es pinta a la pantalla.

Canals de Plataforma

Els canals de plataforma permeten que Flutter es comuniqui amb codi específic de la plataforma utilitzant el pas de missatges.

Exemple: Canal de Plataforma Bàsic

import 'package:flutter/services.dart';

class PlatformChannelExample {
  static const platform = MethodChannel('com.example/platform_channel');

  Future<void> getBatteryLevel() async {
    try {
      final int result = await platform.invokeMethod('getBatteryLevel');
      print('Nivell de bateria: $result%');
    } on PlatformException catch (e) {
      print("No s'ha pogut obtenir el nivell de bateria: '${e.message}'.");
    }
  }
}

Explicació:

  • MethodChannel: Un canal per comunicar-se amb codi específic de la plataforma.
  • invokeMethod(): Crida un mètode al costat de la plataforma.

Conclusió

Comprendre l'arquitectura de Flutter és essencial per construir aplicacions robustes i eficients. Els components clau inclouen el Motor de Flutter, el Framework Dart, els Widgets, el Renderitzat i els Canals de Plataforma. Dominant aquests elements, els desenvolupadors poden crear aplicacions multiplataforma d'alt rendiment amb Flutter.
