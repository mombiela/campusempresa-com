Introducció

La navegació i l'enrutament són conceptes fonamentals en el desenvolupament d'aplicacions mòbils. En Flutter, aquests conceptes et permeten moure't entre diferents pantalles (o pàgines) a la teva aplicació. Aquesta secció cobrirà les bases de la navegació i l'enrutament, i gradualment es mourà cap a temes més avançats.

Navegació Bàsica

Utilitzant Navigator

El widget Navigator és la manera principal de gestionar una pila de rutes en Flutter. Cada ruta és una pantalla o pàgina a la teva aplicació.

  • Empènyer una Ruta: Afegir una nova ruta a la pila.
  • Treure una Ruta: Eliminar la ruta actual de la pila.

Exemple: Empènyer i Treure Rutes

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: FirstScreen(),
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(builder: (context) => SecondScreen()),
            );
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            Navigator.pop(context);
          },
        ),
      ),
    );
  }
}

En aquest exemple:

  • Navigator.push s'utilitza per navegar a la SecondScreen.
  • Navigator.pop s'utilitza per tornar a la FirstScreen.

Rutes Nomenades

Les rutes nomenades proporcionen una manera de navegar utilitzant identificadors de cadena. Això pot fer que el teu codi sigui més net i fàcil de gestionar, especialment en aplicacions més grans.

Definint Rutes Nomenades

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      initialRoute: '/',
      routes: {
        '/': (context) => FirstScreen(),
        '/second': (context) => SecondScreen(),
      },
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.pushNamed(context, '/second');
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            Navigator.pop(context);
          },
        ),
      ),
    );
  }
}

En aquest exemple:

  • Les rutes nomenades es defineixen en el widget MaterialApp.
  • Navigator.pushNamed s'utilitza per navegar a la SecondScreen.

Passar Dades Entre Pantalles

Sovint, necessites passar dades entre pantalles. Això es pot fer utilitzant els mètodes de Navigator.

Exemple: Passar Dades

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: FirstScreen(),
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(
                builder: (context) => SecondScreen(data: 'Hello from FirstScreen'),
              ),
            );
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  final String data;

  SecondScreen({required this.data});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: Text(data),
      ),
    );
  }
}

En aquest exemple:

  • Les dades es passen de FirstScreen a SecondScreen a través del constructor.

Navegació Avançada

Utilitzant Navigator 2.0

El Navigator 2.0 de Flutter proporciona més control sobre la pila de navegació i és adequat per a escenaris de navegació complexos.

  • API basada en pàgines: Permet una navegació més flexible i declarativa.
  • Router: Gestiona l'estat de navegació de l'aplicació.

Exemple: Configuració Bàsica de Navigator 2.0

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp.router(
      routerDelegate: MyRouterDelegate(),
      routeInformationParser: MyRouteInformationParser(),
    );
  }
}

class MyRouterDelegate extends RouterDelegate with ChangeNotifier, PopNavigatorRouterDelegateMixin {
  final GlobalKey<NavigatorState> navigatorKey;

  MyRouterDelegate() : navigatorKey = GlobalKey<NavigatorState>();

  @override
  Widget build(BuildContext context) {
    return Navigator(
      key: navigatorKey,
      pages: [
        MaterialPage(child: FirstScreen()),
        if (showSecondScreen) MaterialPage(child: SecondScreen()),
      ],
      onPopPage: (route, result) {
        if (!route.didPop(result)) {
          return false;
        }
        showSecondScreen = false;
        notifyListeners();
        return true;
      },
    );
  }

  bool showSecondScreen = false;

  @override
  Future<void> setNewRoutePath(configuration) async {
    if (configuration == '/second') {
      showSecondScreen = true;
    } else {
      showSecondScreen = false;
    }
  }
}

class MyRouteInformationParser extends RouteInformationParser<String> {
  @override
  Future<String> parseRouteInformation(RouteInformation routeInformation) async {
    return routeInformation.location ?? '/';
  }

  @override
  RouteInformation restoreRouteInformation(String configuration) {
    return RouteInformation(location: configuration);
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            (Router.of(context).routerDelegate as MyRouterDelegate).showSecondScreen = true;
            (Router.of(context).routerDelegate as MyRouterDelegate).notifyListeners();
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            (Router.of(context).routerDelegate as MyRouterDelegate).showSecondScreen = false;
            (Router.of(context).routerDelegate as MyRouterDelegate).notifyListeners();
          },
        ),
      ),
    );
  }
}

En aquest exemple:

  • RouterDelegate i RouteInformationParser s'utilitzen per gestionar l'estat de navegació.
  • El widget Navigator s'utilitza per construir la pila de navegació basada en l'estat.

Conclusió

La navegació i l'enrutament són essencials per construir aplicacions multi-pantalla en Flutter. Aquest tema ha cobert:

  • Navegació bàsica utilitzant Navigator.
  • Rutes nomenades per a una navegació més neta.
  • Passar dades entre pantalles.
  • Navegació avançada utilitzant Navigator 2.0.

Entendre aquests conceptes t'ajudarà a construir aplicacions Flutter robustes i fàcils d'usar. Practica implementant diferents escenaris de navegació en els teus projectes per consolidar la teva comprensió.
