Les proves són una part crucial del cicle de vida del desenvolupament de programari, assegurant que les teves aplicacions Flutter siguin fiables, rendibles i lliures d'errors. Aquesta secció et guiarà a través dels diferents tipus de proves en Flutter, des de proves unitàries fins a proves d'integració, i com implementar-les de manera efectiva.

Tipus de Proves en Flutter

Flutter admet diversos tipus de proves per assegurar que la teva aplicació funcioni com s'espera:

  • Proves Unitàries: Proves d'unitats individuals de codi, com ara funcions o mètodes.
  • Proves de Widget: Proves dels components de la interfície d'usuari i les seves interaccions.
  • Proves d'Integració: Proves de l'aplicació completa o de grans parts d'aquesta per assegurar que tot funcioni conjuntament.

Proves Unitàries

Les proves unitàries se centren en verificar el comportament de funcions o mètodes individuals. Són ràpides i ajuden a detectar errors a l'inici del procés de desenvolupament.

Configuració de les Proves Unitàries

Per configurar proves unitàries en Flutter, has d'afegir el paquet test al teu fitxer pubspec.yaml:

dev_dependencies:
  test: ^1.16.0

Escriure Proves Unitàries

Aquí tens un exemple senzill d'una prova unitària per a una funció Dart:

// lib/calculator.dart
int add(int a, int b) {
  return a + b;
}

// test/calculator_test.dart
import 'package:test/test.dart';
import '../lib/calculator.dart';

void main() {
  test('adds two numbers', () {
    expect(add(2, 3), 5);
  });
}

En aquest exemple:

  • La funció add està definida a lib/calculator.dart.
  • La prova per a la funció add està escrita a test/calculator_test.dart.
  • La funció test del paquet test s'utilitza per definir un cas de prova.
  • La funció expect comprova si el resultat de add(2, 3) és 5.

Proves de Widget

Les proves de widget (o proves de components) proven els components de la interfície d'usuari i les seves interaccions. Són més completes que les proves unitàries i poden detectar problemes relacionats amb la interfície d'usuari.

Configuració de les Proves de Widget

Les proves de widget també utilitzen el paquet test, però has d'afegir el paquet flutter_test també:

dev_dependencies:
  flutter_test:
    sdk: flutter

Escriure Proves de Widget

Aquí tens un exemple d'una prova de widget:

// lib/my_widget.dart
import 'package:flutter/material.dart';

class MyWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        body: Center(
          child: Text('Hello, World!'),
        ),
      ),
    );
  }
}

// test/my_widget_test.dart
import 'package:flutter_test/flutter_test.dart';
import '../lib/my_widget.dart';

void main() {
  testWidgets('MyWidget has a title and message', (WidgetTester tester) async {
    await tester.pumpWidget(MyWidget());

    final titleFinder = find.text('Hello, World!');

    expect(titleFinder, findsOneWidget);
  });
}

En aquest exemple:

  • La classe MyWidget està definida a lib/my_widget.dart.
  • La prova de widget per a MyWidget està escrita a test/my_widget_test.dart.
  • La funció testWidgets s'utilitza per definir una prova de widget.
  • El WidgetTester s'utilitza per construir i interactuar amb widgets en l'entorn de prova.
  • La funció find.text localitza el widget de text, i expect comprova si es troba.

Proves d'Integració

Les proves d'integració verifiquen que totes les parts de l'aplicació funcionin conjuntament com s'espera. Són el tipus de proves més completes i poden simular interaccions reals d'usuaris.

Configuració de les Proves d'Integració

Les proves d'integració requereixen el paquet integration_test. Afegeix-lo al teu fitxer pubspec.yaml:

dev_dependencies:
  integration_test:
    sdk: flutter

Escriure Proves d'Integració

Aquí tens un exemple d'una prova d'integració:

// test_driver/app.dart
import 'package:integration_test/integration_test_driver.dart';

Future<void> main() => integrationDriver();

// test/app_test.dart
import 'package:flutter_test/flutter_test.dart';
import 'package:integration_test/integration_test.dart';
import 'package:your_app/main.dart' as app;

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();

  testWidgets('tap on the floating action button; verify counter', (WidgetTester tester) async {
    app.main();
    await tester.pumpAndSettle();

    final Finder fab = find.byTooltip('Increment');
    await tester.tap(fab);
    await tester.pumpAndSettle();

    expect(find.text('1'), findsOneWidget);
  });
}

En aquest exemple:

  • El paquet integration_test s'utilitza per configurar i executar proves d'integració.
  • El fitxer test_driver/app.dart inicialitza el controlador de proves d'integració.
  • El fitxer test/app_test.dart conté la prova d'integració real.
  • El mètode IntegrationTestWidgetsFlutterBinding.ensureInitialized configura l'entorn de prova.
  • La funció testWidgets defineix la prova d'integració.
  • El WidgetTester s'utilitza per interactuar amb l'aplicació, com ara prémer botons i verificar canvis en la interfície d'usuari.

Conclusió

Les proves són una part essencial del desenvolupament en Flutter, assegurant que la teva aplicació sigui fiable i lliure d'errors. Entenent i implementant proves unitàries, proves de widget i proves d'integració, pots detectar problemes a l'inici i mantenir una base de codi d'alta qualitat. Recorda:

  • Comença amb proves unitàries per a funcions individuals.
  • Utilitza proves de widget per verificar components de la interfície d'usuari.
  • Implementa proves d'integració per assegurar que tota l'aplicació funcioni com s'espera.

Bones proves!
