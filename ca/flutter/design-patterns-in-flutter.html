Introducció als Patrons de Disseny

Els patrons de disseny són solucions reutilitzables a problemes comuns en el disseny de programari. Proporcionen una plantilla per a com resoldre un problema que es pot utilitzar en moltes situacions diferents. En Flutter, els patrons de disseny ajuden a construir aplicacions escalables, mantenibles i eficients.

Per què Utilitzar Patrons de Disseny en Flutter?

  • Reutilització de Codi: Els patrons de disseny promouen la reutilització del codi, fent que la teva base de codi sigui més eficient.
  • Mantenibilitat: Fan que el codi sigui més fàcil de mantenir i estendre.
  • Escalabilitat: Els patrons de disseny ajuden a construir aplicacions escalables proporcionant una estructura robusta.
  • Millors Pràctiques: Encapsulen les millors pràctiques i els estàndards de la indústria.

Patrons de Disseny Comuns en Flutter

Patró Singleton

El patró Singleton assegura que una classe tingui només una instància i proporciona un punt d'accés global a aquesta.

Exemple

class Singleton {
  static final Singleton _instance = Singleton._internal();

  factory Singleton() {
    return _instance;
  }

  Singleton._internal() {
    // Codi d'inicialització aquí
  }
}

void main() {
  var singleton1 = Singleton();
  var singleton2 = Singleton();

  print(singleton1 == singleton2); // true
}
  • Explicació: La classe Singleton té un constructor privat _internal i un constructor de fàbrica que retorna la mateixa instància cada vegada.

Patró Provider

El patró Provider s'utilitza per a la gestió d'estat en Flutter. Permet gestionar i propagar els canvis d'estat de manera eficient.

Exemple

import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

void main() {
  runApp(
    ChangeNotifierProvider(
      create: (context) => Counter(),
      child: MyApp(),
    ),
  );
}

class Counter with ChangeNotifier {
  int _count = 0;

  int get count => _count;

  void increment() {
    _count++;
    notifyListeners();
  }
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Exemple de Patró Provider')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: <Widget>[
              Text('Has premut el botó aquest nombre de vegades:'),
              Consumer<Counter>(
                builder: (context, counter, child) {
                  return Text('${counter.count}', style: Theme.of(context).textTheme.headline4);
                },
              ),
            ],
          ),
        ),
        floatingActionButton: FloatingActionButton(
          onPressed: () => context.read<Counter>().increment(),
          tooltip: 'Incrementar',
          child: Icon(Icons.add),
        ),
      ),
    );
  }
}
  • Explicació: La classe Counter estèn ChangeNotifier i utilitza el mètode notifyListeners per notificar els oients dels canvis d'estat. Els widgets ChangeNotifierProvider i Consumer s'utilitzen per gestionar i accedir a l'estat.

Patró Factory

El patró Factory s'utilitza per crear objectes sense especificar la classe exacta de l'objecte que es crearà.

Exemple

abstract class Shape {
  void draw();
}

class Circle implements Shape {
  @override
  void draw() {
    print("Dibuixant Cercle");
  }
}

class Square implements Shape {
  @override
  void draw() {
    print("Dibuixant Quadrat");
  }
}

class ShapeFactory {
  static Shape getShape(String shapeType) {
    if (shapeType == 'Circle') {
      return Circle();
    } else if (shapeType == 'Square') {
      return Square();
    }
    throw Exception('Forma no trobada');
  }
}

void main() {
  Shape shape1 = ShapeFactory.getShape('Circle');
  shape1.draw(); // Dibuixant Cercle

  Shape shape2 = ShapeFactory.getShape('Square');
  shape2.draw(); // Dibuixant Quadrat
}
  • Explicació: La classe ShapeFactory té un mètode estàtic getShape que retorna una instància de Shape basada en la cadena d'entrada.

Patró Observer

El patró Observer s'utilitza per notificar múltiples objectes sobre qualsevol canvi en l'estat d'un altre objecte.

Exemple

import 'dart:async';

class Subject {
  final _controller = StreamController<int>();

  Stream<int> get stream => _controller.stream;

  void update(int value) {
    _controller.add(value);
  }

  void dispose() {
    _controller.close();
  }
}

class Observer {
  final String name;

  Observer(this.name);

  void listen(Stream<int> stream) {
    stream.listen((value) {
      print('$name ha rebut: $value');
    });
  }
}

void main() {
  final subject = Subject();
  final observer1 = Observer('Observer 1');
  final observer2 = Observer('Observer 2');

  observer1.listen(subject.stream);
  observer2.listen(subject.stream);

  subject.update(1); // Observer 1 ha rebut: 1
                     // Observer 2 ha rebut: 1
  subject.update(2); // Observer 1 ha rebut: 2
                     // Observer 2 ha rebut: 2

  subject.dispose();
}
  • Explicació: La classe Subject utilitza un StreamController per gestionar els canvis d'estat i notificar els observadors. La classe Observer escolta el flux i reacciona a les actualitzacions.

Conclusió

Els patrons de disseny són essencials per construir aplicacions Flutter robustes, mantenibles i escalables. Entenent i implementant aquests patrons, pots millorar la qualitat del teu codi i adherir-te a les millors pràctiques. Els patrons Singleton, Provider, Factory i Observer són només alguns exemples que poden millorar significativament les teves habilitats de desenvolupament en Flutter.
