La fase de desenvolupament en Flutter implica escriure, provar i refinar el codi per construir una aplicació mòbil funcional i eficient. Aquesta fase és crucial ja que estableix les bases per al rendiment, la usabilitat i l'experiència general de l'usuari de l'aplicació. En aquesta secció, cobrirem els conceptes clau, eines i pràctiques essencials per desenvolupar una aplicació Flutter des de zero.

Conceptes Clau

  • Widgets: Els blocs de construcció d'una aplicació Flutter. Tot en Flutter és un widget, des d'estructures de disseny fins a elements de la interfície d'usuari.
  • Gestió de l'Estat: Tècniques per gestionar l'estat de la teva aplicació, assegurant que la interfície d'usuari reflecteixi l'estat actual de l'aplicació.
  • Navegació i Enrutament: Mètodes per navegar entre diferents pantalles o pàgines de la teva aplicació.
  • Programació Asíncrona: Gestionar operacions asíncrones utilitzant Future, async i await.
  • Proves: Escriure proves per assegurar que la teva aplicació funcioni com s'espera.

Widgets

Els widgets són els components principals d'una aplicació Flutter. Descriuen com hauria de ser la seva vista donada la seva configuració i estat actuals.

Exemple: Estructura Bàsica de Widget

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Fase de Desenvolupament en Flutter'),
        ),
        body: Center(
          child: Text('Hola, Flutter!'),
        ),
      ),
    );
  }
}

En aquest exemple:

  • MyApp és un widget sense estat.
  • MaterialApp és un widget que envolta diversos widgets comunament requerits per a aplicacions de disseny material.
  • Scaffold proporciona un marc per a l'estructura visual bàsica del disseny material.
  • AppBar i Text són widgets utilitzats per crear una interfície d'usuari simple.

Gestió de l'Estat

Gestionar l'estat és crucial per construir aplicacions interactives. Flutter proporciona diverses maneres de gestionar l'estat, incloent setState, InheritedWidget, Provider, i més.

Exemple: Utilitzant `setState`

class Counter extends StatefulWidget {
  @override
  _CounterState createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: <Widget>[
        Text('Has premut el botó tantes vegades:'),
        Text(
          '$_counter',
          style: Theme.of(context).textTheme.headline4,
        ),
        ElevatedButton(
          onPressed: _incrementCounter,
          child: Text('Incrementar'),
        ),
      ],
    );
  }
}

En aquest exemple:

  • Counter és un widget amb estat.
  • _CounterState gestiona l'estat del widget Counter.
  • _incrementCounter actualitza l'estat utilitzant setState.

Navegació i Enrutament

Flutter proporciona un sistema de navegació robust per moure's entre pantalles. El widget Navigator gestiona una pila d'objectes de ruta.

Exemple: Navegació Bàsica

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: FirstScreen(),
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Primera Pantalla'),
      ),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(builder: (context) => SecondScreen()),
            );
          },
          child: Text('Anar a la Segona Pantalla'),
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Segona Pantalla'),
      ),
      body: Center(
        child: ElevatedButton(
          onPressed: () {
            Navigator.pop(context);
          },
          child: Text('Tornar a la Primera Pantalla'),
        ),
      ),
    );
  }
}

En aquest exemple:

  • Navigator.push s'utilitza per navegar a la SecondScreen.
  • Navigator.pop s'utilitza per tornar a la FirstScreen.

Programació Asíncrona

Flutter utilitza les característiques asíncrones de Dart per realitzar tasques com obtenir dades d'internet.

Exemple: Obtenint Dades

import 'package:flutter/material.dart';
import 'dart:async';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: DataScreen(),
    );
  }
}

class DataScreen extends StatefulWidget {
  @override
  _DataScreenState createState() => _DataScreenState();
}

class _DataScreenState extends State<DataScreen> {
  Future<String> fetchData() async {
    await Future.delayed(Duration(seconds: 2));
    return 'Dades carregades';
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Obtenció Asíncrona de Dades'),
      ),
      body: Center(
        child: FutureBuilder<String>(
          future: fetchData(),
          builder: (context, snapshot) {
            if (snapshot.connectionState == ConnectionState.waiting) {
              return CircularProgressIndicator();
            } else if (snapshot.hasError) {
              return Text('Error: ${snapshot.error}');
            } else {
              return Text('Resultat: ${snapshot.data}');
            }
          },
        ),
      ),
    );
  }
}

En aquest exemple:

  • fetchData simula una trucada de xarxa.
  • FutureBuilder s'utilitza per construir la interfície d'usuari basada en l'estat del futur.

Proves

Les proves asseguren que la teva aplicació funcioni com s'espera. Flutter suporta proves unitàries, proves de widgets i proves d'integració.

Exemple: Prova Unitària Simple

import 'package:flutter_test/flutter_test.dart';

void main() {
  test('Prova de fum d'increment del comptador', () {
    int counter = 0;

    counter++;

    expect(counter, 1);
  });
}

En aquest exemple:

  • Una prova unitària simple comprova si el comptador incrementa correctament.

Conclusió

La fase de desenvolupament en Flutter implica entendre i utilitzar widgets, gestionar l'estat, navegar entre pantalles, gestionar operacions asíncrones i escriure proves. Dominar aquests conceptes et permetrà construir aplicacions Flutter robustes i eficients. A mesura que progressis, trobaràs que aquestes habilitats fonamentals són crucials per abordar temes més avançats i crear aplicacions complexes.
