Introducció

Abans d'entrar en el codi, és crucial planificar i configurar correctament el teu projecte JavaScript. Això assegura que el teu projecte estigui ben organitzat, sigui mantenible i escalable. En aquesta secció, cobrirem els passos essencials per planificar i configurar un projecte JavaScript, des de la creació d'un directori de projecte fins a la configuració d'eines de desenvolupament.

Conceptes Clau

  • Estructura del Directori del Projecte: Organització de fitxers i carpetes.
  • Control de Versions: Ús de Git per al seguiment de canvis.
  • Gestió de Paquets: Gestió de dependències amb npm o yarn.
  • Eines de Desenvolupament: Configuració de linters, formatadors i eines de construcció.
  • Configuració d'Entorns: Gestió de diferents entorns (desenvolupament, proves, producció).

Estructura del Directori del Projecte

Una estructura de directori ben organitzada ajuda a mantenir el projecte de manera eficient. Aquí tens una estructura bàsica per a un projecte JavaScript:

my-project/
├── src/
│   ├── index.js
│   ├── components/
│   └── utils/
├── tests/
│   └── index.test.js
├── .gitignore
├── package.json
└── README.md
  • src/: Conté el codi font.
    • index.js: Punt d'entrada de l'aplicació.
    • components/: Directori per a components reutilitzables.
    • utils/: Directori per a funcions d'utilitat.
  • tests/: Conté els fitxers de proves.
  • .gitignore: Especifica els fitxers que Git ha d'ignorar.
  • package.json: Gestiona les dependències i els scripts del projecte.
  • README.md: Proporciona documentació del projecte.

Control de Versions

Utilitzar Git per al control de versions et permet fer un seguiment dels canvis i col·laborar amb altres persones. Aquí tens com inicialitzar un repositori Git:

git init
git add .
git commit -m "Initial commit"
  • Inicialitzar Git: git init crea un nou repositori Git.
  • Afegir Fitxers: git add . prepara tots els fitxers per al primer commit.
  • Fer un Commit: git commit -m "Initial commit" fa un commit dels fitxers preparats amb un missatge.

Gestió de Paquets

Els projectes JavaScript sovint depenen de biblioteques externes. npm (Node Package Manager) o yarn es poden utilitzar per gestionar aquestes dependències.

Ús de npm

  1. Inicialitzar npm:

    npm init -y

    Això crea un fitxer package.json amb la configuració per defecte.

  2. Instal·lar Dependències:

    npm install express

    Això instal·la la biblioteca express i l'afegeix a package.json.

Ús de yarn

  1. Inicialitzar yarn:

    yarn init -y

    Això crea un fitxer package.json amb la configuració per defecte.

  2. Instal·lar Dependències:

    yarn add express

    Això instal·la la biblioteca express i l'afegeix a package.json.

Eines de Desenvolupament

Configurar eines de desenvolupament com linters i formatadors assegura la qualitat i consistència del codi.

ESLint

ESLint és un linter popular per a JavaScript.

  1. Instal·lar ESLint:

    npm install eslint --save-dev

  2. Inicialitzar ESLint:

    npx eslint --init

  3. Exemple de Configuració (.eslintrc.json):

    {
  "env": {
    "browser": true,
    "es2021": true
  },
  "extends": "eslint:recommended",
  "parserOptions": {
    "ecmaVersion": 12,
    "sourceType": "module"
  },
  "rules": {
    "indent": ["error", 2],
    "quotes": ["error", "double"],
    "semi": ["error", "always"]
  }
}

Prettier

Prettier és un formatador de codi.

  1. Instal·lar Prettier:

    npm install prettier --save-dev

  2. Exemple de Configuració (.prettierrc):

    {
  "singleQuote": true,
  "trailingComma": "es5"
}

Configuració d'Entorns

Gestionar diferents entorns assegura que la teva aplicació es comporti correctament en desenvolupament, proves i producció.

Ús de dotenv

dotenv és un mòdul que carrega variables d'entorn des d'un fitxer .env a process.env.

  1. Instal·lar dotenv:

    npm install dotenv

  2. Crear un fitxer .env:

    PORT=3000
DATABASE_URL=mongodb://localhost:27017/mydatabase

  3. Carregar Variables d'Entorn:

    require('dotenv').config();

const port = process.env.PORT;
const dbUrl = process.env.DATABASE_URL;

console.log(`Server running on port ${port}`);
console.log(`Database URL: ${dbUrl}`);

Conclusió

Una planificació i configuració adequades del projecte són essencials per a l'èxit de qualsevol projecte JavaScript. Organitzant el teu directori de projecte, utilitzant control de versions, gestionant dependències, configurant eines de desenvolupament i configurant entorns, pots assegurar-te que el teu projecte sigui mantenible i escalable. Aquests passos fonamentals t'ajudaran a tu i al teu equip a treballar de manera més eficient i efectiva mentre desenvolupeu les vostres aplicacions JavaScript.
