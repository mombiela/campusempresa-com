La Integració Contínua (CI) i el Desplegament Continu (CD) són pràctiques essencials en el desenvolupament de programari modern. Ajuden a assegurar que els canvis en el codi es proven i es despleguen automàticament, cosa que condueix a un programari més fiable i mantenible. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de CI/CD en el context del desenvolupament en JavaScript.

Introducció a la Integració i el Desplegament Continu

  • Integració Contínua (CI): La pràctica d'integrar freqüentment els canvis de codi en un repositori compartit, on es fan compilacions i proves automàtiques.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa les proves automàtiques a producció.

Configuració d'una Canalització CI/CD

Triar una Eina CI/CD

Hi ha diverses eines CI/CD disponibles, cadascuna amb els seus punts forts. Algunes de les més populars inclouen:

  • Jenkins
  • Travis CI
  • CircleCI
  • GitHub Actions

Exemple: Configuració de GitHub Actions per a un Projecte JavaScript

  1. Crear un Repositori GitHub: Inicialitza un nou repositori a GitHub per al teu projecte JavaScript.
  2. Afegir un Fitxer de Flux de Treball: Crea un directori .github/workflows a l'arrel del teu projecte i afegeix un fitxer de flux de treball, per exemple, ci.yml.
name: CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

Explicació del Fitxer de Flux de Treball

  • name: El nom del flux de treball.
  • on: Especifica els esdeveniments que desencadenen el flux de treball (per exemple, push, pull_request).
  • jobs: Defineix les tasques a executar.
    • runs-on: Especifica l'entorn (per exemple, ubuntu-latest).
    • steps: Llista els passos a executar en la tasca.
      • Checkout code: Utilitza l'acció actions/checkout per fer el checkout del repositori.
      • Set up Node.js: Utilitza l'acció actions/setup-node per configurar l'entorn Node.js.
      • Install dependencies: Executa npm install per instal·lar les dependències del projecte.
      • Run tests: Executa npm test per executar el conjunt de proves.

Escriure Proves per a la Integració Contínua

Proves Unitàries amb Jest

Jest és un marc de proves popular per a JavaScript. Aquí tens com configurar-lo i escriure una prova simple.

  1. Instal·lar Jest:

    npm install --save-dev jest

  2. Afegir un Script de Prova a package.json:

    {
  "scripts": {
    "test": "jest"
  }
}

  3. Escriure una Prova Simple: Crea un fitxer sum.js:

    function sum(a, b) {
  return a + b;
}
module.exports = sum;

    Crea un fitxer de prova sum.test.js:

    const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

  4. Executar les Proves:

    npm test

Integrar les Proves en el Flux de Treball CI

Assegura't que el teu fitxer de flux de treball CI inclou un pas per executar les proves, com es mostra en l'exemple de GitHub Actions anterior.

Desplegament Continu

Desplegar a un Servei d'Allotjament

Per a aplicacions JavaScript, els serveis d'allotjament populars inclouen:

  • Netlify
  • Vercel
  • Heroku

Exemple: Desplegar a Netlify

  1. Crear un Compte a Netlify: Registra't a Netlify.
  2. Connectar el Teu Repositori: Enllaça el teu repositori GitHub a Netlify.
  3. Configurar la Configuració de Compilació: Configura la comanda de compilació (per exemple, npm run build) i el directori de publicació (per exemple, build).
  4. Desplegar: Netlify desplegarà automàticament la teva aplicació cada vegada que facis un push de canvis al repositori.

Conclusió

La Integració i el Desplegament Continu són crucials per mantenir aplicacions JavaScript d'alta qualitat. Automatitzant els processos de prova i desplegament, pots assegurar-te que el teu codi està sempre en un estat desplegable i que les noves funcionalitats i correccions es lliuren ràpidament i de manera fiable. Comença configurant una canalització CI simple, escriu proves completes i avança gradualment cap al desplegament continu complet per optimitzar el teu flux de treball de desenvolupament.
