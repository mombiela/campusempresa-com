La gestió de memòria és un aspecte crucial de la programació que implica l'assignació, ús i alliberament de memòria de manera eficient. En JavaScript, la gestió de memòria és principalment gestionada pel motor de JavaScript, però entendre com funciona pot ajudar-te a escriure codi més eficient i efectiu.

Conceptes Clau

  • Assignació de Memòria: El procés de reservar una porció de memòria per a ser utilitzada per variables, objectes i altres estructures de dades.
  • Recollida d'escombraries: El procés automàtic d'identificar i alliberar memòria que ja no s'utilitza.
  • Heap i Stack: Dos tipus de memòria utilitzats en JavaScript per a diferents propòsits.

Assignació de Memòria

En JavaScript, la memòria s'assigna quan es creen variables i objectes. Aquesta memòria s'utilitza per emmagatzemar dades i és gestionada pel motor de JavaScript.

Exemple: Assignació de Memòria

let number = 42; // Es reserva memòria per a un número
let name = "John"; // Es reserva memòria per a una cadena
let person = { name: "John", age: 30 }; // Es reserva memòria per a un objecte

En l'exemple anterior:

  • Es reserva memòria per al número 42.
  • Es reserva memòria per a la cadena "John".
  • Es reserva memòria per a l'objecte { name: "John", age: 30 }.

Recollida d'escombraries

JavaScript utilitza un mecanisme automàtic de recollida d'escombraries per gestionar la memòria. El recollidor d'escombraries escaneja periòdicament la memòria per identificar i alliberar memòria que ja no s'utilitza.

Algorisme de Marcat i Escombrat

L'algorisme de recollida d'escombraries més comú utilitzat en JavaScript és l'algorisme de Marcat i Escombrat. Aquest algorisme funciona en dues fases:

  1. Fase de Marcat: El recollidor d'escombraries recorre el gràfic d'objectes, començant pels objectes arrel, i marca tots els objectes accessibles.
  2. Fase d'Escombrat: El recollidor d'escombraries escaneja la memòria i allibera la memòria ocupada pels objectes que no estan marcats com a accessibles.

Exemple: Recollida d'escombraries

function createPerson() {
    let person = { name: "Alice", age: 25 };
    return person;
}

let person1 = createPerson();
let person2 = createPerson();

// En aquest punt, la memòria per als objectes creats dins createPerson encara s'utilitza
person1 = null; // L'objecte referenciat per person1 ara és elegible per a la recollida d'escombraries

En l'exemple anterior:

  • Els objectes creats dins createPerson s'utilitzen inicialment.
  • Quan person1 es posa a null, l'objecte que referenciava esdevé elegible per a la recollida d'escombraries.

Heap i Stack

JavaScript utilitza dos tipus de memòria: el heap i el stack.

  • Stack: Utilitzat per a l'assignació de memòria estàtica. Emmagatzema valors primitius i referències a objectes.
  • Heap: Utilitzat per a l'assignació de memòria dinàmica. Emmagatzema objectes i funcions.

Exemple: Stack i Heap

function example() {
    let number = 10; // Emmagatzemat al stack
    let obj = { value: 20 }; // Emmagatzemat al heap
}

example();

En l'exemple anterior:

  • La variable number s'emmagatzema al stack.
  • L'objecte obj s'emmagatzema al heap, mentre que la referència a obj s'emmagatzema al stack.

Millors Pràctiques per a la Gestió de Memòria

  • Evita Variables Globals: Les variables globals romanen en memòria durant tota la vida de l'aplicació, cosa que pot portar a fuites de memòria.
  • Utilitza Variables Locals: Les variables locals es recullen automàticament quan surten de l'àmbit.
  • Nul·la les Referències: Posa explícitament les variables a null quan ja no es necessiten per fer-les elegibles per a la recollida d'escombraries.
  • Utilitza les Clausures amb Cura: Sigues cautelós amb les clausures ja que poden retenir referències a variables, impedint que siguin recollides per l'escombraries.

Exemple: Millors Pràctiques

function processData() {
    let data = { name: "Bob", age: 40 };
    // Processa les dades
    data = null; // Nul·la explícitament la referència
}

processData();

En l'exemple anterior:

  • La variable data es posa explícitament a null després de no ser necessària, fent-la elegible per a la recollida d'escombraries.

Conclusió

Entendre la gestió de memòria en JavaScript és essencial per escriure codi eficient i efectiu. Sent conscient de com s'assigna, s'utilitza i s'allibera la memòria, pots evitar errors comuns com les fuites de memòria i assegurar-te que les teves aplicacions funcionin sense problemes. Recorda seguir les millors pràctiques com evitar variables globals, utilitzar variables locals, nul·lar referències i utilitzar clausures amb cura per gestionar la memòria de manera efectiva.
