Introducció al Bucle d'Esdeveniments

El bucle d'esdeveniments és un concepte fonamental en JavaScript que permet la programació asíncrona. Permet a JavaScript realitzar operacions no bloquejants, tot i ser monofil.

  • Monofil: JavaScript executa el codi en un sol fil, és a dir, una comanda s'executa a la vegada.
  • Asíncron: Operacions que poden ocórrer independentment del flux principal del programa, com ara operacions d'E/S, temporitzadors i sol·licituds de xarxa.

Com Funciona el Bucle d'Esdeveniments

El bucle d'esdeveniments comprova contínuament la pila de crides i la cua de tasques per determinar què executar a continuació.

  1. Pila de Crides: Una estructura de pila que fa el seguiment de les crides de funció.
  2. Cua de Tasques: Una cua on es col·loquen les tasques asíncrones per ser executades després que la pila actual estigui buida.

Exemple de Pila de Crides i Cua de Tasques

console.log('Start');

setTimeout(() => {
  console.log('Timeout');
}, 0);

console.log('End');
  • Pila de Crides:
    • console.log('Start')
    • console.log('End')
  • Cua de Tasques:
    • setTimeout callback

Flux d'Execució

  1. console.log('Start') es posa a la pila de crides i s'executa.
  2. setTimeout es crida, i el seu callback es col·loca a la cua de tasques.
  3. console.log('End') es posa a la pila de crides i s'executa.
  4. La pila de crides ara està buida, així que el bucle d'esdeveniments comprova la cua de tasques.
  5. El callback de setTimeout es mou a la pila de crides i s'executa.

Microtasques i Macrotasques

JavaScript distingeix entre microtasques i macrotasques, que tenen diferents prioritats en el bucle d'esdeveniments.

  • Microtasques: Tasques de major prioritat, com ara Promises.
  • Macrotasques: Tasques de menor prioritat, com ara setTimeout i setInterval.

Exemple amb Promises

console.log('Start');

setTimeout(() => {
  console.log('Timeout');
}, 0);

Promise.resolve().then(() => {
  console.log('Promise');
});

console.log('End');

Flux d'Execució amb Microtasques

  1. console.log('Start') s'executa.
  2. El callback de setTimeout es col·loca a la cua de macrotasques.
  3. El callback de Promise.resolve().then() es col·loca a la cua de microtasques.
  4. console.log('End') s'executa.
  5. La pila de crides està buida, així que el bucle d'esdeveniments comprova la cua de microtasques.
  6. El callback de Promise s'executa.
  7. La pila de crides està buida de nou, així que el bucle d'esdeveniments comprova la cua de macrotasques.
  8. El callback de setTimeout s'executa.

Aplicacions Pràctiques

Entendre el bucle d'esdeveniments és crucial per escriure codi JavaScript eficient i no bloquejant.

  • Gestió d'operacions d'E/S: Les operacions d'E/S no bloquejants permeten que el fil principal continuï executant-se mentre espera que les operacions d'E/S es completin.
  • Temporitzadors: Utilitzar setTimeout i setInterval per programar tasques.
  • Promises i async/await: Gestionar l'execució de codi asíncron.

Exemple amb async/await

async function fetchData() {
  console.log('Fetching data...');
  let data = await fetch('https://api.example.com/data');
  console.log('Data fetched');
}

console.log('Start');
fetchData();
console.log('End');

Flux d'Execució amb async/await

  1. console.log('Start') s'executa.
  2. fetchData() es crida, i console.log('Fetching data...') s'executa.
  3. L'await fetch pausa la funció fetchData, i el control torna al fil principal.
  4. console.log('End') s'executa.
  5. Un cop l'operació de fetch es completa, console.log('Data fetched') s'executa.

Conclusió

El bucle d'esdeveniments és un concepte clau que permet a JavaScript gestionar operacions asíncrones de manera eficient. Entenent com la pila de crides, la cua de tasques, les microtasques i les macrotasques treballen conjuntament, pots escriure codi JavaScript més eficient i responsiu. Aquest coneixement és essencial tant per a desenvolupadors de JavaScript principiants com avançats per gestionar operacions asíncrones de manera efectiva.
