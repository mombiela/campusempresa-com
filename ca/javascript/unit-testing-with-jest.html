Introducció a les Proves Unitàries

  • Definició: Les proves unitàries impliquen provar unitats o components individuals d'un programari per assegurar-se que funcionen com es preveu.
  • Importància: Ajuda a detectar errors d'hora, assegura la fiabilitat del codi i facilita la refactorització.

Introducció a Jest

  • Què és Jest?: Jest és un marc de proves per a JavaScript mantingut per Facebook, dissenyat per assegurar la correcció de qualsevol base de codi JavaScript.
  • Característiques:
    • No requereix configuració
    • Proves de snapshot
    • Cobertura de codi integrada
    • Mocking fàcil

Configuració de Jest

  • Instal·lació: 
    npm install --save-dev jest
  • Configuració:
    • Afegeix un script de prova a package.json: 
      {
  "scripts": {
    "test": "jest"
  }
}

Escriure la Teva Primera Prova

  • Crear un Fitxer de Prova:

    • Els fitxers de prova haurien de tenir una extensió .test.js o .spec.js.
    • Exemple: sum.test.js

  • Exemple Bàsic de Prova:

    // sum.js
function sum(a, b) {
  return a + b;
}
module.exports = sum;

// sum.test.js
const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Matchers de Jest

  • Matchers Comuns:

    • toBe: Igualtat exacta
    • toEqual: Igualtat profunda
    • toBeNull: Només coincideix amb null
    • toBeUndefined: Només coincideix amb undefined
    • toBeTruthy: Coincideix amb qualsevol cosa que una instrucció if tracta com a veritat
    • toBeFalsy: Coincideix amb qualsevol cosa que una instrucció if tracta com a fals

  • Exemple:

    test('object assignment', () => {
  const data = {one: 1};
  data['two'] = 2;
  expect(data).toEqual({one: 1, two: 2});
});

Provar Codi Asíncron

  • Callbacks:

    // fetchData.js
function fetchData(callback) {
  setTimeout(() => {
    callback('peanut butter');
  }, 1000);
}
module.exports = fetchData;

// fetchData.test.js
const fetchData = require('./fetchData');

test('the data is peanut butter', done => {
  function callback(data) {
    expect(data).toBe('peanut butter');
    done();
  }

  fetchData(callback);
});

  • Promises:

    // fetchDataPromise.js
function fetchData() {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve('peanut butter');
    }, 1000);
  });
}
module.exports = fetchData;

// fetchDataPromise.test.js
const fetchData = require('./fetchDataPromise');

test('the data is peanut butter', () => {
  return fetchData().then(data => {
    expect(data).toBe('peanut butter');
  });
});

  • Async/Await:

    test('the data is peanut butter', async () => {
  const data = await fetchData();
  expect(data).toBe('peanut butter');
});

Mocking a Jest

  • Mocks Manuals: 
    // user.js
const fetchData = require('./fetchData');

function getUser() {
  return fetchData().then(data => data.user);
}
module.exports = getUser;

// __mocks__/fetchData.js
module.exports = jest.fn(() => Promise.resolve({user: 'John Doe'}));

// user.test.js
jest.mock('./fetchData');
const getUser = require('./user');
const fetchData = require('./fetchData');

test('fetches user', async () => {
  const user = await getUser();
  expect(user).toBe('John Doe');
  expect(fetchData).toHaveBeenCalledTimes(1);
});

Proves de Snapshot

  • Definició: Les proves de snapshot són una manera de provar la interfície d'usuari comparant l'estat actual de la interfície amb una snapshot guardada.
  • Exemple: 
    // Link.js
const React = require('react');

function Link({page, children}) {
  return <a href={page}>{children}</a>;
}
module.exports = Link;

// Link.test.js
const React = require('react');
const renderer = require('react-test-renderer');
const Link = require('./Link');

test('Link changes the class when hovered', () => {
  const component = renderer.create(
    <Link page="http://www.facebook.com">Facebook</Link>,
  );
  let tree = component.toJSON();
  expect(tree).toMatchSnapshot();
});

Cobertura de Codi

  • Generar Informes de Cobertura:

    • Executa Jest amb el flag --coverage: 
      npm test -- --coverage

  • Interpretar Informes de Cobertura:

    • Els informes de cobertura es generaran al directori coverage.
    • Les mètriques clau inclouen:
      • Statements: Percentatge d'instruccions executades.
      • Branches: Percentatge de branques executades en instruccions condicionals.
      • Functions: Percentatge de funcions executades.
      • Lines: Percentatge de línies executades.

Conclusió

Les proves unitàries amb Jest són una manera poderosa d'assegurar que el teu codi JavaScript és fiable i lliure d'errors. Entenent els conceptes bàsics de Jest, escrivint proves, utilitzant matchers, gestionant codi asíncron, fent mocks, proves de snapshot i generant informes de cobertura de codi, pots millorar significativament la qualitat de la teva base de codi. Bones proves!
