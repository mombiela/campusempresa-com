Introducció

JavaScript és un llenguatge potent i flexible, però amb gran poder ve gran responsabilitat. Assegurar que el teu codi JavaScript és segur és crucial per protegir les teves aplicacions i usuaris de diverses amenaces de seguretat. Aquesta secció et guiarà a través de les pràctiques essencials de seguretat, des de nivells bàsics fins a avançats.

Validació i Sanitització d'Entrades

  • Valida les Entrades: Sempre valida les entrades d'usuari tant al costat del client com del servidor per evitar que es processin dades malicioses.
  • Sanititza les Entrades: Elimina o escapa qualsevol caràcter potencialment nociu de les entrades d'usuari per prevenir atacs d'injecció.

Exemple: Validació Bàsica d'Entrades

function validateInput(input) {
    const regex = /^[a-zA-Z0-9]+$/; // Només permet caràcters alfanumèrics
    return regex.test(input);
}

const userInput = "test123";
if (validateInput(userInput)) {
    console.log("Entrada vàlida");
} else {
    console.log("Entrada no vàlida");
}

Evitar Variables Globals

  • Minimitza l'Àmbit Global: Utilitza variables i funcions locals per evitar contaminar l'àmbit global, cosa que pot portar a conflictes i vulnerabilitats de seguretat.

Exemple: Ús d'Àmbit Local

(function() {
    var localVariable = "Això és local";
    console.log(localVariable);
})();

console.log(typeof localVariable); // undefined

Emmagatzematge Segur de Dades

  • Evita Emmagatzemar Dades Sensibles en Local Storage: El local storage és accessible via JavaScript i pot ser fàcilment explotat per scripts maliciosos.
  • Utilitza Cookies Segures: Emmagatzema dades sensibles en cookies amb els flags HttpOnly i Secure per evitar l'accés del costat del client i assegurar que les dades es transmeten per HTTPS.

Exemple: Configuració de Cookies Segures

document.cookie = "sessionId=abc123; Secure; HttpOnly";

Prevenció de Cross-Site Scripting (XSS)

  • Escapa la Sortida: Sempre escapa les dades abans de renderitzar-les al navegador per prevenir atacs XSS.
  • Utilitza Content Security Policy (CSP): Implementa headers CSP per restringir les fonts des de les quals es poden carregar scripts.

Exemple: Escapant la Sortida

function escapeHTML(str) {
    return str.replace(/&/g, "&amp;")
              .replace(/</g, "&lt;")
              .replace(/>/g, "&gt;")
              .replace(/"/g, "&quot;")
              .replace(/'/g, "&#039;");
}

const userInput = "<script>alert('XSS');</script>";
const safeOutput = escapeHTML(userInput);
document.getElementById("output").innerHTML = safeOutput;

Protecció contra Cross-Site Request Forgery (CSRF)

  • Utilitza Tokens Anti-CSRF: Inclou tokens únics en formularis i verifica'ls al costat del servidor per assegurar que les sol·licituds són legítimes.
  • Cookies SameSite: Configura l'atribut SameSite a les cookies per evitar que s'enviïn amb sol·licituds cross-site.

Exemple: Configuració de Cookies SameSite

document.cookie = "sessionId=abc123; SameSite=Strict";

Comunicació Segura

  • Utilitza HTTPS: Sempre utilitza HTTPS per encriptar les dades transmeses entre el client i el servidor.
  • Evita Contingut Mixt: Assegura't que tots els recursos (scripts, estils, imatges) es carreguen per HTTPS per evitar problemes de contingut mixt.

Biblioteques i Dependències JavaScript Segures

  • Actualitza Regularment les Dependències: Mantingues totes les biblioteques i dependències actualitzades per corregir vulnerabilitats conegudes.
  • Utilitza Fonts de Confiança: Només utilitza biblioteques de fonts reputades i verifica la seva integritat.

Exemple: Comprovar Vulnerabilitats amb npm

npm audit

Conclusió

La seguretat és un aspecte crític del desenvolupament en JavaScript. Seguint aquestes millors pràctiques, pots reduir significativament el risc de vulnerabilitats de seguretat en les teves aplicacions. Mantén-te sempre informat sobre les últimes amenaces de seguretat i millora contínuament les teves mesures de seguretat per protegir els teus usuaris i dades.
