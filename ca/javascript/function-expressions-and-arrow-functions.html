Introducció

En JavaScript, les funcions es poden definir de diverses maneres. Dos mètodes comuns són les expressions de funcions i les funcions fletxa. Entendre aquests conceptes és crucial per escriure codi JavaScript modern i eficient.

Expressions de funcions

Una expressió de funció és una manera de definir una funció utilitzant una expressió. Això significa que la funció s'assigna a una variable, que després es pot utilitzar per cridar la funció.

Conceptes clau

  • Funció anònima: Una funció sense nom.
  • Expressió de funció amb nom: Una expressió de funció amb un nom.
  • Elevació: Les expressions de funcions no s'eleven, és a dir, no es poden cridar abans de ser definides.

Exemple: Expressió de funció anònima

const greet = function() {
    console.log("Hello, World!");
};

greet(); // Sortida: Hello, World!

Exemple: Expressió de funció amb nom

const factorial = function fact(n) {
    return n <= 1 ? 1 : n * fact(n - 1);
};

console.log(factorial(5)); // Sortida: 120

Comparació amb declaracions de funcions

| Característica | Declaració de funció | Expressió de funció | |------------------------|-------------------------------|-------------------------------| | Sintaxi | function name() {} | const name = function() {} | | Elevació | Sí | No | | Cas d'ús | Quan es defineixen funcions globalment | Quan es defineixen funcions localment o condicionalment |

Funcions fletxa

Les funcions fletxa proporcionen una sintaxi més curta per escriure expressions de funcions i tenen algunes característiques úniques.

Conceptes clau

  • Sintaxi: Utilitza => per definir la funció.
  • Retorn implícit: Si el cos de la funció conté només una expressió, es pot retornar implícitament.
  • No hi ha vinculació de this: Les funcions fletxa no tenen el seu propi context this; hereten this del context pare.

Exemple: Funció fletxa bàsica

const add = (a, b) => {
    return a + b;
};

console.log(add(2, 3)); // Sortida: 5

Exemple: Retorn implícit

const multiply = (a, b) => a * b;

console.log(multiply(2, 3)); // Sortida: 6

Exemple: No hi ha vinculació de `this`

function Person() {
    this.age = 0;

    setInterval(() => {
        this.age++; // `this` es refereix a l'objecte Person
        console.log(this.age);
    }, 1000);
}

const person = new Person();
// Sortida: 1, 2, 3, 4, ... (incrementa cada segon)

Comparació amb funcions regulars

| Característica | Funció regular | Funció fletxa | |------------------------|-------------------------------|-------------------------------| | Sintaxi | function() {} | () => {} | | Vinculació de this | Dinàmica | Lèxica | | Objecte Arguments | Disponible | No disponible | | Cas d'ús | Quan es necessita el context this | Quan es prefereix una sintaxi més curta i no es necessita this |

Conclusió

Les expressions de funcions i les funcions fletxa són eines poderoses en JavaScript. Les expressions de funcions proporcionen flexibilitat en la definició de funcions, mentre que les funcions fletxa ofereixen una sintaxi concisa i una vinculació lèxica de this. Entendre quan i com utilitzar aquests diferents tipus de funcions t'ajudarà a escriure codi JavaScript més eficient i llegible.
