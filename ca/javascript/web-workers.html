Web Workers
Construcció d'una Aplicació de Llista de Tasques ►

Els Web Workers proporcionen una manera d'executar scripts en fils de fons. Poden executar codi sense interferir amb la interfície d'usuari, permetent-te realitzar tasques com càlculs o processament de dades sense causar que la pàgina web es torni irresponsive.

Introducció als Web Workers

  • Definició: Els Web Workers són scripts de JavaScript que s'executen en segon pla, separats del fil d'execució principal d'una aplicació web.
  • Propòsit: S'utilitzen per realitzar tasques que són intensives en càlcul o requereixen esperar recursos externs, sense bloquejar el fil principal.
  • Tipus de Web Workers:
    • Workers Dedicats: Utilitzats per un únic script.
    • Workers Compartits: Poden ser accessibles per múltiples scripts, fins i tot des de diferents finestres o iframes.

Creació i Ús d'un Web Worker

Creació d'un Worker

Per crear un Web Worker, necessites escriure un fitxer JavaScript separat per al worker i després instanciar-lo en el teu script principal.

Exemple:

  1. worker.js (Script del Worker)

    // worker.js
self.onmessage = function(event) {
    const data = event.data;
    // Realitzar algun càlcul
    const result = data * 2;
    // Enviar el resultat de tornada al fil principal
    self.postMessage(result);
};

  2. main.js (Script Principal)

    // main.js
if (window.Worker) {
    const myWorker = new Worker('worker.js');

    myWorker.onmessage = function(event) {
        console.log('Rebut del worker:', event.data);
    };

    myWorker.postMessage(10); // Enviar dades al worker
} else {
    console.log('Els Web Workers no són compatibles amb el teu navegador.');
}

Comunicació amb un Worker

  • Enviar Dades al Worker: Utilitza el mètode postMessage.
  • Rebre Dades del Worker: Escolta l'esdeveniment message.

Exemple:

// Enviar dades al worker
myWorker.postMessage('Hola, Worker!');

// Rebre dades del worker
myWorker.onmessage = function(event) {
    console.log('Missatge del worker:', event.data);
};

Terminació d'un Worker

Els workers poden ser terminats des del script principal utilitzant el mètode terminate.

Exemple:

myWorker.terminate();
console.log('Worker terminat.');

Gestió d'Errors en Workers

Els errors en workers poden ser capturats utilitzant el gestor d'esdeveniments onerror.

Exemple:

myWorker.onerror = function(event) {
    console.error('Error en el worker:', event.message, 'a', event.filename, 'línia', event.lineno);
};

Ús Avançat

Workers Compartits

Els Workers Compartits poden ser utilitzats per múltiples scripts, fins i tot des de diferents finestres o iframes.

Exemple:

  1. sharedWorker.js (Script del Worker Compartit)

    // sharedWorker.js
self.onconnect = function(event) {
    const port = event.ports[0];

    port.onmessage = function(event) {
        const data = event.data;
        // Realitzar algun càlcul
        const result = data * 2;
        // Enviar el resultat de tornada al fil principal
        port.postMessage(result);
    };
};

  2. main.js (Script Principal)

    // main.js
if (window.SharedWorker) {
    const mySharedWorker = new SharedWorker('sharedWorker.js');

    mySharedWorker.port.onmessage = function(event) {
        console.log('Rebut del worker compartit:', event.data);
    };

    mySharedWorker.port.postMessage(10); // Enviar dades al worker compartit
} else {
    console.log('Els Workers Compartits no són compatibles amb el teu navegador.');
}

Resum

Els Web Workers són una característica poderosa en JavaScript que et permeten executar scripts en segon pla, millorant el rendiment i la resposta de les teves aplicacions web. Descarregant càlculs intensius o tasques de llarga durada als Web Workers, pots assegurar-te que el fil principal roman lliure per manejar les interaccions de l'usuari de manera suau. Tant si utilitzes Workers Dedicats per a tasques de fil únic com Workers Compartits per a tasques de múltiples fils, entendre com crear, comunicar-te amb, i gestionar els Web Workers és essencial per construir aplicacions web eficients i responsives.
