Introducció a la concurrència en JavaScript

JavaScript és un llenguatge de fil únic, és a dir, té una pila de trucades i un heap de memòria. No obstant això, pot gestionar operacions concurrents utilitzant el seu model de concurrència basat en esdeveniments i no bloquejant. Aquesta secció us introduirà als conceptes bàsics de la concurrència en JavaScript.

  • Naturalesa de fil únic: JavaScript executa una peça de codi a la vegada.
  • Bucle d'esdeveniments: Gestiona l'execució de múltiples peces de codi al llarg del temps, manejant operacions asíncrones.
  • Cua de callbacks: Emmagatzema callbacks que estan llestos per ser executats.
  • APIs web: APIs proporcionades pel navegador que gestionen tasques asíncrones com sol·licituds HTTP, temporitzadors, etc.

El bucle d'esdeveniments

El bucle d'esdeveniments és el nucli del model de concurrència de JavaScript. Permet a JavaScript realitzar operacions no bloquejants delegant tasques al navegador o a l'entorn Node.js.

  • Pila de trucades: On s'executa el codi.
  • Cua de callbacks: Conté missatges (callbacks) per ser processats.
  • Bucle d'esdeveniments: Comprova contínuament la pila de trucades i la cua de callbacks, empènyer callbacks a la pila de trucades quan està buida.

Exemple: Bucle d'esdeveniments bàsic

console.log('Inici');

setTimeout(() => {
  console.log('Callback de temporitzador');
}, 1000);

console.log('Final');

Explicació:

  1. console.log('Inici') s'executa i es registra immediatament.
  2. setTimeout programa un callback per ser executat després d'1 segon.
  3. console.log('Final') s'executa i es registra immediatament.
  4. Després d'1 segon, el callback de setTimeout es posa a la cua de callbacks i eventualment s'executa.

Programació asíncrona

La programació asíncrona permet a JavaScript realitzar tasques de llarga durada sense bloquejar el fil principal.

  • Callbacks: Funcions passades com a arguments a altres funcions, executades després de la finalització d'una tasca asíncrona.
  • Promises: Objectes que representen la finalització (o fallada) eventual d'una operació asíncrona.
  • Async/Await: Sucre sintàctic sobre les promeses, fent que el codi asíncron sembli síncron.

Exemple: Callback

function fetchData(callback) {
  setTimeout(() => {
    callback('Dades rebudes');
  }, 1000);
}

fetchData((data) => {
  console.log(data);
});

Explicació:

  1. fetchData es crida amb una funció de callback.
  2. setTimeout simula una operació asíncrona.
  3. Després d'1 segon, el callback s'executa, registrant 'Dades rebudes'.

Exemple: Promeses

function fetchData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('Dades rebudes');
    }, 1000);
  });
}

fetchData().then((data) => {
  console.log(data);
});

Explicació:

  1. fetchData retorna una promesa.
  2. setTimeout simula una operació asíncrona.
  3. Després d'1 segon, la promesa es resol, i then es crida, registrant 'Dades rebudes'.

Exemple: Async/Await

function fetchData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('Dades rebudes');
    }, 1000);
  });
}

async function getData() {
  const data = await fetchData();
  console.log(data);
}

getData();

Explicació:

  1. fetchData retorna una promesa.
  2. getData és una funció asíncrona que espera que la promesa es resolgui.
  3. Després d'1 segon, await obté el valor resolt, registrant 'Dades rebudes'.

Conceptes avançats de concurrència

A mesura que avanceu, entendre patrons de concurrència més complexos i eines esdevé essencial.

  • Treballadors web: Permeten executar scripts en fils de fons.
  • Canals de missatges: Faciliten la comunicació entre diferents contextos (per exemple, fil principal i treballadors web).
  • Atomics i SharedArrayBuffer: Proporcionen primitives de concurrència de baix nivell per a memòria compartida.

Exemple: Treballadors web

// worker.js
self.onmessage = function(e) {
  const result = e.data * 2;
  self.postMessage(result);
};

// main.js
const worker = new Worker('worker.js');
worker.onmessage = function(e) {
  console.log('Resultat del treballador:', e.data);
};
worker.postMessage(10);

Explicació:

  1. worker.js escolta missatges i processa dades.
  2. main.js crea un treballador i li envia dades.
  3. El treballador processa les dades i envia el resultat de nou al fil principal.

Conclusió

El model de concurrència de JavaScript, centrat al voltant del bucle d'esdeveniments, li permet gestionar operacions asíncrones de manera eficient malgrat ser de fil únic. Entendre el bucle d'esdeveniments, callbacks, promeses i async/await és crucial per escriure codi eficient i no bloquejant. A mesura que avanceu, explorar conceptes avançats com Treballadors web i memòria compartida pot millorar encara més la vostra capacitat per gestionar la concurrència en aplicacions JavaScript.
