L'API d'Emmagatzematge Web proporciona mecanismes pels quals els navegadors poden emmagatzemar parells clau/valor, d'una manera molt més intuïtiva que utilitzant cookies. Aquesta API és essencial per construir aplicacions web modernes que requereixen emmagatzematge de dades persistent al costat del client.

Conceptes Clau

  • Emmagatzematge Local: Emmagatzema dades sense data de caducitat. Les dades no es borren quan es tanca el navegador i es poden accedir més tard.
  • Emmagatzematge de Sessió: Emmagatzema dades durant la durada de la sessió de la pàgina. Les dades es borren quan la sessió de la pàgina acaba (és a dir, quan es tanca el navegador).

Emmagatzematge Local

L'Emmagatzematge Local està dissenyat per emmagatzemar dades que necessiten persistir fins i tot després que l'usuari tanqui el navegador. És ideal per emmagatzemar preferències de l'usuari, configuracions i altres dades que haurien d'estar disponibles a través de sessions.

Operacions Bàsiques

Establir Dades

Per emmagatzemar dades a l'Emmagatzematge Local, utilitza el mètode setItem.

localStorage.setItem('username', 'JohnDoe');

Obtenir Dades

Per recuperar dades de l'Emmagatzematge Local, utilitza el mètode getItem.

let username = localStorage.getItem('username');
console.log(username); // Mostra: JohnDoe

Eliminar Dades

Per eliminar un element específic de l'Emmagatzematge Local, utilitza el mètode removeItem.

localStorage.removeItem('username');

Esborrar Totes les Dades

Per esborrar totes les dades de l'Emmagatzematge Local, utilitza el mètode clear.

localStorage.clear();

Exemple: Emmagatzemar Preferències de l'Usuari

// Emmagatzemar preferències de l'usuari
localStorage.setItem('theme', 'dark');
localStorage.setItem('fontSize', '16px');

// Recuperar preferències de l'usuari
let theme = localStorage.getItem('theme');
let fontSize = localStorage.getItem('fontSize');

console.log(`Theme: ${theme}, Font Size: ${fontSize}`); // Mostra: Theme: dark, Font Size: 16px

Emmagatzematge de Sessió

L'Emmagatzematge de Sessió està dissenyat per emmagatzemar dades que només haurien d'estar disponibles durant la durada de la sessió de la pàgina. És útil per emmagatzemar dades temporals que no necessiten persistir a través de sessions.

Operacions Bàsiques

Establir Dades

Per emmagatzemar dades a l'Emmagatzematge de Sessió, utilitza el mètode setItem.

sessionStorage.setItem('sessionID', '123456');

Obtenir Dades

Per recuperar dades de l'Emmagatzematge de Sessió, utilitza el mètode getItem.

let sessionID = sessionStorage.getItem('sessionID');
console.log(sessionID); // Mostra: 123456

Eliminar Dades

Per eliminar un element específic de l'Emmagatzematge de Sessió, utilitza el mètode removeItem.

sessionStorage.removeItem('sessionID');

Esborrar Totes les Dades

Per esborrar totes les dades de l'Emmagatzematge de Sessió, utilitza el mètode clear.

sessionStorage.clear();

Exemple: Emmagatzemar Dades Temporals

// Emmagatzemar dades temporals
sessionStorage.setItem('currentPage', 'home');
sessionStorage.setItem('cartItems', JSON.stringify(['item1', 'item2', 'item3']));

// Recuperar dades temporals
let currentPage = sessionStorage.getItem('currentPage');
let cartItems = JSON.parse(sessionStorage.getItem('cartItems'));

console.log(`Current Page: ${currentPage}, Cart Items: ${cartItems}`); // Mostra: Current Page: home, Cart Items: item1,item2,item3

Comparació entre Emmagatzematge Local i Emmagatzematge de Sessió

| Característica | Emmagatzematge Local | Emmagatzematge de Sessió | |---------------------|---------------------------------------|---------------------------------------| | Persistència de Dades | Persistent a través de sessions | Només durant la sessió | | Límits d'Emmagatzematge | Normalment 5-10 MB per origen | Normalment 5-10 MB per origen | | Casos d'Ús | Preferències de l'usuari, configuracions | Dades temporals, informació específica de la sessió | | Àmbit d'Accés | Qualsevol finestra/pestanya del mateix origen | Només la finestra/pestanya que la va crear |

Conclusió

L'API d'Emmagatzematge Web és una eina poderosa per a l'emmagatzematge de dades al costat del client en aplicacions web. Comprenent les diferències entre l'Emmagatzematge Local i l'Emmagatzematge de Sessió, els desenvolupadors poden prendre decisions informades sobre on emmagatzemar les dades segons els requisits de persistència. L'Emmagatzematge Local és ideal per a dades que necessiten persistir a través de sessions, mentre que l'Emmagatzematge de Sessió és adequat per a dades temporals que haurien de ser esborrades quan la sessió acaba.
