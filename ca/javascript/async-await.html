Introducció a la Programació Asíncrona

  • Programació Asíncrona: Permet que el programa comenci una tasca potencialment llarga i continuï amb altres tasques abans que la primera acabi.
  • Callbacks: Funcions passades com a arguments a altres funcions per ser executades més tard.
  • Promises: Objectes que representen la finalització eventual (o fallida) d'una operació asíncrona i el seu valor resultant.

Entenent les Promises

  • Estats d'una Promise:
    • Pending: Estat inicial, ni complerta ni rebutjada.
    • Fulfilled: Operació completada amb èxit.
    • Rejected: Operació fallida.
  • Creant una Promise: 
    const myPromise = new Promise((resolve, reject) => {
  // Operació asíncrona
  if (/* operació exitosa */) {
    resolve('Èxit!');
  } else {
    reject('Fracàs!');
  }
});

Introducció a Async/Await

  • Funcions Async: Funcions declarades amb la paraula clau async, que retornen una Promise.
  • Paraula clau Await: Utilitzada dins d'una funció async per pausar l'execució fins que una Promise es resolgui o es rebutgi.

Utilitzant Async/Await

  • Sintaxi Bàsica: 
    async function fetchData() {
  try {
    let response = await fetch('https://api.example.com/data');
    let data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error:', error);
  }
}
fetchData();
    • async function fetchData() {}: Declara una funció async.
    • await fetch('https://api.example.com/data'): Pausa l'execució fins que la Promise del fetch es resolgui.
    • try...catch: Gestiona qualsevol error que ocorri durant les operacions asíncrones.

Gestió d'Errors amb Async/Await

  • Utilitzant try...catch: 
    async function getData() {
  try {
    let response = await fetch('https://api.example.com/data');
    if (!response.ok) {
      throw new Error('La resposta de la xarxa no va ser correcta');
    }
    let data = await response.json();
    return data;
  } catch (error) {
    console.error('Error de fetch:', error);
  }
}

Enllaçant Funcions Async

  • Execució Seqüencial: 
    async function processData() {
  let data1 = await fetchData1();
  let data2 = await fetchData2(data1);
  let result = await processData(data2);
  return result;
}

Execució Paral·lela amb Async/Await

  • Utilitzant Promise.all: 
    async function fetchMultipleData() {
  try {
    let [data1, data2] = await Promise.all([
      fetch('https://api.example.com/data1').then(response => response.json()),
      fetch('https://api.example.com/data2').then(response => response.json())
    ]);
    console.log(data1, data2);
  } catch (error) {
    console.error('Error obtenint dades:', error);
  }
}
fetchMultipleData();

Patrons Avançats

  • Reintentant Peticions Fallides: 
    async function fetchWithRetry(url, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      let response = await fetch(url);
      if (!response.ok) throw new Error('La resposta de la xarxa no va ser correcta');
      return await response.json();
    } catch (error) {
      if (i === retries - 1) throw error;
    }
  }
}

Conclusió

  • Async/Await simplifica el treball amb Promises, fent que el codi asíncron sigui més fàcil de llegir i escriure.
  • Gestió d'Errors és senzilla amb blocs try...catch.
  • Execució Seqüencial i Paral·lela es poden gestionar efectivament utilitzant async/await i Promise.all.
  • Patrons Avançats com reintentar peticions fallides es poden implementar per gestionar escenaris més complexos.

Dominant async/await, pots escriure codi JavaScript asíncron més net i mantenible.
