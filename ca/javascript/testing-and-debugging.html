Introducció

Les proves i la depuració són habilitats crucials per a qualsevol desenvolupador de JavaScript. Aquesta secció cobrirà els conceptes bàsics de les proves i la depuració, progressant des de nivells principiants fins a avançats. Al final d'aquesta secció, seràs capaç d'escriure proves per al teu codi i depurar problemes de manera efectiva.

Proves en JavaScript

Què són les Proves?

  • Definició: Les proves són el procés d'avaluar un sistema o els seus components per trobar si satisfà els requisits especificats.
  • Propòsit: Identificar defectes, assegurar la qualitat i verificar que el codi funcioni com s'espera.

Tipus de Proves

  • Proves Unitàries: Proves d'unitats individuals o components d'un programari.
  • Proves d'Integració: Proves de la interacció entre unitats o components integrats.
  • Proves de Cap a Cap: Proves del flux complet d'una aplicació de principi a fi.

Eines per a les Proves

  • Jest: Un marc de proves de JavaScript mantingut per Facebook.
  • Mocha: Un marc de proves de JavaScript ric en funcionalitats que s'executa en Node.js.
  • Chai: Una biblioteca d'assertions BDD / TDD per a Node.js i el navegador.

Escriure la Teva Primera Prova amb Jest

  1. Instal·lar Jest:

    npm install --save-dev jest

  2. Crear un Fitxer de Prova:

    // sum.js
function sum(a, b) {
    return a + b;
}
module.exports = sum;

    // sum.test.js
const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
    expect(sum(1, 2)).toBe(3);
});

  3. Executar la Prova:

    npx jest

Explicació

  • sum.js: Una funció simple que suma dos números.
  • sum.test.js: Un fitxer de prova que importa la funció sum i prova si sum(1, 2) és igual a 3.
  • test(): Una funció proporcionada per Jest per definir un cas de prova.
  • expect(): Una funció d'assertion per comprovar si el resultat coincideix amb el valor esperat.

Depuració en JavaScript

Què és la Depuració?

  • Definició: La depuració és el procés d'identificar, analitzar i eliminar errors o bugs d'un programari.
  • Propòsit: Assegurar que el programari s'executi de manera suau i correcta.

Tècniques Comunes de Depuració

  • Registre a la Consola: Utilitzar console.log() per imprimir variables i seguir el flux del programa.
  • Punts de Trencament: Utilitzar punts de trencament en un depurador per pausar l'execució i inspeccionar l'estat de l'aplicació.
  • Gestió d'Errors: Utilitzar blocs try...catch per gestionar errors de manera elegant.

Utilitzar el Registre a la Consola

  • Exemple: 
    function greet(name) {
    console.log('greet function called with:', name);
    return `Hello, ${name}!`;
}

greet('Alice');

Utilitzar Punts de Trencament a les DevTools de Chrome

  1. Obrir DevTools: Clic dret a la pàgina web i seleccionar "Inspect" o prémer Ctrl+Shift+I.
  2. Establir un Punt de Trencament: Navegar a la pestanya "Sources", trobar el teu fitxer JavaScript i fer clic al número de línia on vols establir un punt de trencament.
  3. Executar el Codi: Refrescar la pàgina o activar la funció per arribar al punt de trencament i inspeccionar variables.

Gestió d'Errors amb try...catch

  • Exemple: 
    function parseJSON(jsonString) {
    try {
        const result = JSON.parse(jsonString);
        console.log('Parsing successful:', result);
        return result;
    } catch (error) {
        console.error('Parsing error:', error.message);
        return null;
    }
}

parseJSON('{"name": "Alice"}'); // JSON vàlid
parseJSON('Invalid JSON'); // JSON invàlid

Conclusió

Les proves i la depuració són pràctiques essencials per mantenir un codi JavaScript d'alta qualitat. Escriure proves permet assegurar que el teu codi es comporta com s'espera, i depurar de manera efectiva permet identificar i solucionar problemes ràpidament. Dominar aquestes habilitats et farà un desenvolupador més competent i fiable.
