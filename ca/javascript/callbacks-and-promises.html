Introducció

En JavaScript, gestionar operacions asíncrones és crucial per construir aplicacions responsives i eficients. Dos mètodes principals per gestionar codi asíncron són els callbacks i les promeses. Aquesta secció cobrirà ambdós conceptes, començant pels fonaments i progressant cap a un ús més avançat.

Callbacks

Què és un Callback?

  • Un callback és una funció passada a una altra funció com a argument, que després és invocada dins de la funció externa per completar algun tipus de rutina o acció.

Exemple Bàsic d'un Callback

function fetchData(callback) {
    setTimeout(() => {
        const data = { name: "John", age: 30 };
        callback(data);
    }, 1000);
}

function displayData(data) {
    console.log(`Name: ${data.name}, Age: ${data.age}`);
}

fetchData(displayData);
  • fetchData és una funció asíncrona que pren un callback.
  • setTimeout simula una operació asíncrona.
  • displayData és la funció callback que processa les dades.

Gestionar Errors amb Callbacks

function fetchData(callback) {
    setTimeout(() => {
        const error = false; // Canvia a true per simular un error
        if (error) {
            callback('Error occurred', null);
        } else {
            const data = { name: "John", age: 30 };
            callback(null, data);
        }
    }, 1000);
}

function displayData(error, data) {
    if (error) {
        console.log(error);
    } else {
        console.log(`Name: ${data.name}, Age: ${data.age}`);
    }
}

fetchData(displayData);
  • Patró de callback amb error primer: el primer argument és un error, i el segon és el resultat.

Promeses

Què és una Promesa?

  • Una promesa és un objecte que representa la finalització eventual o el fracàs d'una operació asíncrona.

Exemple Bàsic d'una Promesa

function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const data = { name: "John", age: 30 };
            resolve(data);
        }, 1000);
    });
}

fetchData().then(data => {
    console.log(`Name: ${data.name}, Age: ${data.age}`);
});
  • fetchData retorna una promesa.
  • resolve es crida quan l'operació és exitosa.
  • then s'utilitza per gestionar el valor resolt.

Gestionar Errors amb Promeses

function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const error = false; // Canvia a true per simular un error
            if (error) {
                reject('Error occurred');
            } else {
                const data = { name: "John", age: 30 };
                resolve(data);
            }
        }, 1000);
    });
}

fetchData()
    .then(data => {
        console.log(`Name: ${data.name}, Age: ${data.age}`);
    })
    .catch(error => {
        console.log(error);
    });
  • reject es crida quan l'operació falla.
  • catch s'utilitza per gestionar el valor rebutjat.

Comparació: Callbacks vs Promeses

| Característica | Callbacks | Promeses | |-------------------|------------------------------------|------------------------------------| | Sintaxi | Funció passada com a argument | Objecte amb then i catch | | Gestió d'Errors | Patró de callback amb error primer | Mètode catch | | Llegibilitat | Pot conduir a "callback hell" | Més llegible i mantenible | | Enllaçament | Difícil d'enllaçar múltiples crides| Fàcil enllaçament amb then |

Promeses Avançades

Enllaçar Promeses

function fetchData() {
    return new Promise((resolve) => {
        setTimeout(() => {
            const data = { name: "John", age: 30 };
            resolve(data);
        }, 1000);
    });
}

function processData(data) {
    return new Promise((resolve) => {
        setTimeout(() => {
            data.processed = true;
            resolve(data);
        }, 1000);
    });
}

fetchData()
    .then(data => processData(data))
    .then(processedData => {
        console.log(processedData);
    });
  • Les promeses es poden enllaçar per realitzar operacions asíncrones seqüencials.

Utilitzar `Promise.all`

function fetchData1() {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve("Data from API 1");
        }, 1000);
    });
}

function fetchData2() {
    return new Promise((resolve) => {
        setTimeout(() => {
            resolve("Data from API 2");
        }, 1500);
    });
}

Promise.all([fetchData1(), fetchData2()])
    .then(results => {
        console.log(results); // ["Data from API 1", "Data from API 2"]
    });
  • Promise.all executa múltiples promeses en paral·lel i es resol quan totes les promeses es resolen.

Conclusió

Els callbacks i les promeses són conceptes fonamentals per gestionar operacions asíncrones en JavaScript. Tot i que els callbacks són senzills, poden conduir a codi complex i difícil de mantenir. Les promeses ofereixen una manera més robusta i llegible de gestionar tasques asíncrones, amb millor gestió d'errors i capacitats d'enllaçament. Entendre ambdós mètodes és crucial per escriure codi JavaScript eficient i mantenible.
