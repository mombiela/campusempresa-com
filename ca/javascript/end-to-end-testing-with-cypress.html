Introducció a les Proves End-to-End

Les proves End-to-End (E2E) són una metodologia utilitzada per provar si el flux d'una aplicació funciona tal com està dissenyat de principi a fi. L'objectiu de les proves E2E és identificar les dependències del sistema i assegurar-se que la integritat de les dades es manté entre diversos components i sistemes del sistema.

  • Definició: Provar el flux complet d'una aplicació de principi a fi.
  • Objectiu: Assegurar-se que l'aplicació es comporta com s'espera en un escenari del món real.

Introducció a Cypress

Cypress és un marc de proves End-to-End potent i modern que facilita l'escriptura i l'execució de proves per a les teves aplicacions web.

  • Característiques:
    • Viatge en el temps: Cypress pren captures de pantalla mentre les proves s'executen.
    • Depurabilitat: Missatges d'error detallats i traços de pila.
    • Espera automàtica: No cal afegir esperes o pauses a les teves proves.
    • Recàrregues en temps real: Es recarrega automàticament cada vegada que fas canvis a les teves proves.
    • Control del trànsit de xarxa: Simula i falsifica sol·licituds de xarxa.

Configuració de Cypress

Abans de poder començar a escriure proves amb Cypress, cal configurar-lo al teu projecte.

Instal·lació

Per instal·lar Cypress, necessites tenir Node.js i npm instal·lats al teu ordinador.

npm install cypress --save-dev

Obrir Cypress

Després de la instal·lació, pots obrir Cypress utilitzant la següent comanda:

npx cypress open

Aquesta comanda obrirà el Cypress Test Runner, on podràs veure i executar les teves proves.

Escriure la Teva Primera Prova

Escrivim una prova senzilla per comprovar si una pàgina web es carrega correctament.

Exemple de Prova

Crea un nou fitxer a la carpeta cypress/integration, per exemple, example_spec.js.

describe('My First Test', () => {
  it('Visits the Kitchen Sink', () => {
    cy.visit('https://example.cypress.io')
    cy.contains('type').click()
    cy.url().should('include', '/commands/actions')
    cy.get('.action-email').type('fake@email.com')
      .should('have.value', 'fake@email.com')
  })
})

Explicació

  • describe: Agrupa proves relacionades.
  • it: Defineix una prova única.
  • cy.visit: Navega a una URL.
  • cy.contains: Troba un element que conté text específic.
  • cy.url: Fa una afirmació sobre la URL.
  • cy.get: Obté un element basat en un selector.
  • type: Escriu en un camp d'entrada.
  • should: Fa una afirmació.

Tècniques de Prova Avançades

A mesura que et familiaritzis més amb Cypress, pots començar a utilitzar característiques més avançades.

Comandes Personalitzades

Pots crear comandes personalitzades per reutilitzar codi i fer les teves proves més netes.

Cypress.Commands.add('login', (email, password) => {
  cy.visit('/login')
  cy.get('input[name=email]').type(email)
  cy.get('input[name=password]').type(password)
  cy.get('button[type=submit]').click()
})

Fixtures

Els fixtures s'utilitzen per emmagatzemar i gestionar dades de prova.

// cypress/fixtures/user.json
{
  "email": "user@example.com",
  "password": "password123"
}
// Utilitzant el fixture en una prova
describe('Login Test', () => {
  beforeEach(() => {
    cy.fixture('user').as('userData')
  })

  it('Logs in with valid credentials', function() {
    cy.visit('/login')
    cy.get('input[name=email]').type(this.userData.email)
    cy.get('input[name=password]').type(this.userData.password)
    cy.get('button[type=submit]').click()
    cy.url().should('include', '/dashboard')
  })
})

Millors Pràctiques

  • Mantingues les proves aïllades: Cada prova ha de ser independent.
  • Utilitza noms significatius: Anomena les teves proves i comandes de manera clara.
  • Evita codificar valors: Utilitza fixtures i variables d'entorn.
  • Neteja l'estat: Assegura't que les proves no deixin dades residuals.

Conclusió

Les proves End-to-End amb Cypress et permeten assegurar-te que la teva aplicació web funciona com s'espera des de la perspectiva de l'usuari. Configurant Cypress, escrivint proves bàsiques i avançades, i seguint les millors pràctiques, pots crear proves robustes i fiables que ajuden a mantenir la qualitat de la teva aplicació.

Cypress proporciona un conjunt ric de característiques que fan que escriure i executar proves sigui senzill i eficient, convertint-lo en una excel·lent opció per als desenvolupadors que busquen implementar proves E2E en els seus projectes.
