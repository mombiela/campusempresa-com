Introducció

En aquesta secció del curs, els estudiants aplicaran tot el que han après per desenvolupar un producte complet des de la concepció fins a la implementació. Aquest projecte final permetrà als estudiants integrar coneixements de disseny, desenvolupament, proves i desplegament de programari.

Objectius del Projecte

  • Integrar coneixements adquirits: Aplicar conceptes de disseny, desenvolupament, proves i desplegament.
  • Desenvolupar habilitats pràctiques: Treballar en un projecte realista que simuli un entorn professional.
  • Fomentar el treball en equip: Col·laborar amb companys per simular un entorn de desenvolupament real.

Fases del Projecte

1. Concepció i Planificació

  • Definició del problema: Identificar una necessitat o problema que el producte resoldrà.
  • Anàlisi de requisits: Documentar els requisits funcionals i no funcionals.
  • Planificació del projecte: Crear un pla de projecte amb fites i lliurables.

Exemple de Document de Requisits

# Requisits Funcionals
1. El sistema ha de permetre als usuaris registrar-se i autenticar-se.
2. Els usuaris han de poder crear, llegir, actualitzar i eliminar publicacions.

# Requisits No Funcionals
1. El sistema ha de ser escalable per suportar fins a 10,000 usuaris concurrents.
2. El temps de resposta del sistema ha de ser inferior a 2 segons.

2. Disseny

  • Arquitectura del sistema: Seleccionar l'arquitectura adequada (monolítica, microserveis, etc.).
  • Disseny de la base de dades: Crear l'esquema de la base de dades.
  • Disseny de la interfície d'usuari: Crear wireframes i mockups.

Exemple d'Esquema de Base de Dades

| Taula Usuaris | Camp | Tipus | |---------------|-------------|-------------| | | id | INT | | | nom | VARCHAR(50) | | | email | VARCHAR(50) | | | contrasenya | VARCHAR(50) |

3. Desenvolupament

  • Implementació del backend: Desenvolupar les APIs i lògica del servidor.
  • Implementació del frontend: Desenvolupar la interfície d'usuari.
  • Integració: Assegurar que el frontend i backend funcionin junts correctament.

Exemple de Codi Backend (Node.js)

const express = require('express');
const app = express();
const port = 3000;

app.use(express.json());

let users = [];

app.post('/register', (req, res) => {
    const { name, email, password } = req.body;
    users.push({ name, email, password });
    res.status(201).send('User registered');
});

app.listen(port, () => {
    console.log(`Server running at http://localhost:${port}`);
});

4. Proves

  • Proves unitàries: Escriure i executar proves unitàries per al codi.
  • Proves d'integració: Assegurar que els diferents mòduls del sistema funcionin junts.
  • Proves d'acceptació: Validar que el sistema compleix amb els requisits de l'usuari.

Exemple de Prova Unitària (Jest)

const request = require('supertest');
const app = require('./app'); // Suposant que el servidor està en app.js

describe('POST /register', () => {
    it('should register a new user', async () => {
        const response = await request(app)
            .post('/register')
            .send({ name: 'John Doe', email: 'john@example.com', password: '123456' });
        expect(response.statusCode).toBe(201);
        expect(response.text).toBe('User registered');
    });
});

5. Desplegament

  • Configuració de l'entorn: Preparar l'entorn de producció.
  • Desplegament del sistema: Implementar el sistema en un servidor o servei de núvol.
  • Monitoratge i manteniment: Configurar eines de monitoratge i planificar el manteniment.

Conclusió

El projecte final és una oportunitat perquè els estudiants demostrin la seva capacitat per desenvolupar un producte complet. En finalitzar aquest projecte, els estudiants hauran adquirit una experiència valuosa que els prepararà per afrontar desafiaments reals en el desenvolupament de programari.
