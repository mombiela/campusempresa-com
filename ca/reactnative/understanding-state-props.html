Introducció

A React Native, state i props són conceptes fonamentals que et permeten crear aplicacions dinàmiques i interactives. Entendre com utilitzar-los de manera efectiva és crucial per construir aplicacions mòbils robustes i mantenibles.

Què són les Propietats?

Les propietats (props, abreviatura de properties) són atributs de només lectura que es passen d'un component pare a un component fill. Et permeten passar dades i controladors d'esdeveniments a través de l'arbre de components.

Conceptes Clau de les Propietats

  • Immutabilitat: Les propietats són immutables, és a dir, no poden ser canviades pel component fill.
  • Flux de Dades: Les propietats faciliten un flux de dades unidireccional de pare a fill.
  • Reutilitzabilitat: Passant diferents propietats, pots reutilitzar el mateix component en diversos contextos.

Exemple de Propietats

import React from 'react';
import { Text, View } from 'react-native';

const Greeting = (props) => {
  return (
    <View>
      <Text>Hello, {props.name}!</Text>
    </View>
  );
};

const App = () => {
  return (
    <View>
      <Greeting name="Alice" />
      <Greeting name="Bob" />
    </View>
  );
};

export default App;

En aquest exemple:

  • El component Greeting rep una propietat name.
  • El component App passa diferents valors de name al component Greeting.

Què és l'Estat?

L'estat és un objecte integrat que permet als components crear i gestionar les seves pròpies dades. A diferència de les propietats, l'estat és mutable i pot ser canviat dins del component.

Conceptes Clau de l'Estat

  • Mutabilitat: L'estat pot ser canviat utilitzant la funció setState.
  • Abast Local: L'estat és local al component i no pot ser accedit o modificat per altres components.
  • Reactivitat: Quan l'estat canvia, el component es torna a renderitzar per reflectir el nou estat.

Exemple de l'Estat

import React, { useState } from 'react';
import { Text, View, Button } from 'react-native';

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <View>
      <Text>Count: {count}</Text>
      <Button title="Increment" onPress={() => setCount(count + 1)} />
    </View>
  );
};

const App = () => {
  return (
    <View>
      <Counter />
    </View>
  );
};

export default App;

En aquest exemple:

  • El component Counter utilitza el hook useState per gestionar l'estat count.
  • La funció setCount actualitza l'estat, causant que el component es torni a renderitzar amb el nou compte.

Comparant Propietats i Estat

| Característica | Propietats | Estat | |----------------|-------------------------------|-------------------------------| | Mutabilitat | Immutable | Mutable | | Abast | Passat de pare a fill | Local al component | | Ús | Passar dades i controladors d'esdeveniments | Gestionar dades específiques del component | | Reutilitzabilitat | Alta | Baixa |

Millors Pràctiques

  • Utilitza Propietats per a Dades Estàtiques: Utilitza propietats per passar dades que no canvien amb el temps o que són gestionades pel component pare.
  • Utilitza Estat per a Dades Dinàmiques: Utilitza estat per a dades que canvien amb el temps i que necessiten ser gestionades dins del component.
  • Mantén l'Estat Localitzat: Evita elevar l'estat a menys que sigui necessari. Mantén la gestió de l'estat tan a prop d'on s'utilitza com sigui possible.
  • Evita l'Ús Excessiu de l'Estat: Massa estat pot fer que el teu component sigui complex i difícil de gestionar. Utilitza estat només quan sigui necessari.

Conclusió

Entendre les diferències entre estat i propietats és essencial per construir aplicacions React Native eficients i mantenibles. Les propietats et permeten passar dades i controladors d'esdeveniments a través de l'arbre de components, mentre que l'estat permet als components gestionar les seves pròpies dades. Seguint les millors pràctiques i sabent quan utilitzar cadascun, pots crear aplicacions mòbils dinàmiques i interactives amb facilitat.
