Introducció

En aquesta secció, explorarem dos components essencials en React Native per gestionar contingut desplaçable: ScrollView i FlatList. Aquests components són crucials per crear interfícies d'usuari dinàmiques i responsives, especialment quan es tracta de llistes de dades.

ScrollView

ScrollView és un component senzill i versàtil que permet el desplaçament dins d'una sola vista. És adequat per a llistes més petites o contingut que no requereix optimitzacions complexes.

Conceptes Clau

  • Ús Bàsic: Envoltar contingut dins d'un ScrollView per habilitar el desplaçament.
  • Desplaçament Horitzontal: Habilitar el desplaçament horitzontal establint la propietat horizontal.
  • Consideracions de Rendiment: Entendre quan ScrollView pot no ser la millor opció a causa de problemes de rendiment amb grans conjunts de dades.

Exemple

import React from 'react';
import { ScrollView, Text, StyleSheet } from 'react-native';

const ScrollViewExample = () => {
  return (
    <ScrollView style={styles.container}>
      {Array.from({ length: 20 }, (_, i) => (
        <Text style={styles.item} key={i}>Item {i + 1}</Text>
      ))}
    </ScrollView>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
  },
  item: {
    marginVertical: 10,
    fontSize: 18,
  },
});

export default ScrollViewExample;
  • Explicació: Aquest exemple demostra un ScrollView senzill que conté 20 elements de text. El component ScrollView envolta els elements, permetent el desplaçament vertical.

FlatList

FlatList és un component més avançat i optimitzat per renderitzar grans llistes de dades. Proporciona millor rendiment i característiques addicionals en comparació amb ScrollView.

Conceptes Clau

  • Ús Bàsic: Renderitzar una llista d'elements utilitzant FlatList.
  • KeyExtractor: Proporcionar una clau única per a cada element per optimitzar la renderització.
  • ItemSeparatorComponent: Afegir separadors entre elements.
  • Optimització de Rendiment: Aprofitar propietats com initialNumToRender i windowSize per a un millor rendiment.

Exemple

import React from 'react';
import { FlatList, Text, StyleSheet, View } from 'react-native';

const data = Array.from({ length: 20 }, (_, i) => ({ id: i.toString(), title: `Item ${i + 1}` }));

const FlatListExample = () => {
  const renderItem = ({ item }) => (
    <View style={styles.item}>
      <Text style={styles.title}>{item.title}</Text>
    </View>
  );

  return (
    <FlatList
      data={data}
      renderItem={renderItem}
      keyExtractor={item => item.id}
      ItemSeparatorComponent={() => <View style={styles.separator} />}
    />
  );
};

const styles = StyleSheet.create({
  item: {
    padding: 20,
    backgroundColor: '#f9c2ff',
    marginVertical: 8,
  },
  title: {
    fontSize: 18,
  },
  separator: {
    height: 1,
    backgroundColor: '#ccc',
  },
});

export default FlatListExample;
  • Explicació: Aquest exemple mostra com utilitzar FlatList per renderitzar una llista de 20 elements. Cada element té una clau única, i s'afegeix un separador entre els elements per a una millor separació visual.

Comparació de ScrollView i FlatList

| Característica | ScrollView | FlatList | |---------------------------|-----------------------|---------------------------| | Adequat per a | Llistes petites | Llistes grans | | Rendiment | Menys optimitzat | Altament optimitzat | | Gestió de Claus | No requerida | Requereix claus úniques | | Separació d'Elements | Manual | Suport integrat | | Desplaçament Horitzontal | Suportat | Suportat |

Conclusió

Entendre quan utilitzar ScrollView versus FlatList és crucial per construir aplicacions React Native eficients i responsives. ScrollView és ideal per a llistes més petites i senzilles, mentre que FlatList excel·leix amb grans conjunts de dades gràcies a les seves característiques d'optimització. Dominant aquests components, podràs crear interfícies d'usuari suaus i de bon rendiment en els teus projectes React Native.
