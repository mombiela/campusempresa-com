Construcció d'una aplicació de tasques ►

La Integració Contínua (CI) i el Desplegament Continu (CD) són pràctiques essencials en el desenvolupament modern de programari. Ajuden a assegurar que els canvis de codi es proven i es despleguen automàticament, reduint el risc d'errors i millorant la velocitat de lliurament. Aquesta secció et guiarà a través de la configuració de CI/CD per a un projecte de React Native, des de nivells bàsics fins a avançats.

Introducció a CI/CD

  • Integració Contínua (CI): La pràctica d'integrar automàticament els canvis de codi de múltiples contribuïdors en un repositori compartit diverses vegades al dia.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa totes les etapes del pipeline de producció als usuaris finals.

Configuració d'un Pipeline de CI/CD

Triar un Servei de CI/CD

Hi ha diversos serveis de CI/CD disponibles, cadascun amb els seus propis punts forts. Algunes opcions populars inclouen:

  • GitHub Actions
  • CircleCI
  • Travis CI
  • Bitrise

Per a aquest curs, utilitzarem GitHub Actions per la seva integració perfecta amb els repositoris de GitHub.

Pipeline Bàsic de CI/CD amb GitHub Actions

  1. Crear un Repositori de GitHub: Comença creant un nou repositori a GitHub per al teu projecte de React Native.

  2. Afegir un Fitxer de Flux de Treball: Al teu repositori, crea un directori .github/workflows i afegeix un nou fitxer anomenat ci.yml.

name: CI

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

Explicació del Fitxer de Flux de Treball

  • name: El nom del flux de treball.
  • on: Especifica els esdeveniments que desencadenen el flux de treball. Aquí, s'executa en empentes i sol·licituds d'extracció a la branca main.
  • jobs: Defineix les tasques a executar. En aquest cas, tenim una única tasca anomenada build.
  • runs-on: Especifica el tipus de màquina per executar la tasca. ubuntu-latest és una opció comunament utilitzada.
  • steps: Els passos individuals a executar en la tasca.
    • Checkout repository: Utilitza l'acció actions/checkout per clonar el repositori.
    • Set up Node.js: Utilitza l'acció actions/setup-node per configurar Node.js.
    • Install dependencies: Executa npm install per instal·lar les dependències del projecte.
    • Run tests: Executa npm test per executar el conjunt de proves.

Pipeline Avançat de CI/CD

Afegir Linting i Formatació

Per assegurar la qualitat del codi, pots afegir passos per a linting i formatació.

    - name: Run ESLint
      run: npm run lint

    - name: Run Prettier
      run: npm run format

Construir i Desplegar l'Aplicació

Per desplegar una aplicació de React Native, potser voldràs construir l'aplicació i desplegar-la a un servei com Firebase App Distribution o l'App Store.

Construir l'Aplicació

    - name: Build Android
      run: |
        cd android
        ./gradlew assembleRelease

    - name: Build iOS
      run: |
        cd ios
        xcodebuild -workspace YourApp.xcworkspace -scheme YourApp -sdk iphoneos -configuration AppStoreDistribution archive -archivePath $PWD/build/YourApp.xcarchive

Desplegar l'Aplicació

Per desplegar a Firebase App Distribution:

    - name: Deploy to Firebase App Distribution
      uses: wzieba/Firebase-Distribution-Github-Action@v1
      with:
        appId: ${{ secrets.FIREBASE_APP_ID }}
        token: ${{ secrets.FIREBASE_TOKEN }}
        groups: testers
        file: android/app/build/outputs/apk/release/app-release.apk

Conclusió

Configurar un pipeline de CI/CD per al teu projecte de React Native pot millorar significativament el teu flux de treball de desenvolupament automatitzant els processos de prova i desplegament. Començar amb una configuració bàsica de CI i afegir gradualment funcions més avançades com linting, formatació, construcció i desplegament assegura que el teu codi es mantingui d'alta qualitat i que la teva aplicació arribi als usuaris de manera ràpida i fiable.

Integrant CI/CD als teus projectes de React Native, pots:

  • Detectar errors aviat amb proves automatitzades.
  • Assegurar una qualitat de codi consistent amb linting i formatació.
  • Desplegar actualitzacions més ràpidament i de manera més fiable.

Recorda, la clau per a un CI/CD efectiu és començar de manera senzilla i iterar, afegint més complexitat a mesura que el teu projecte creix i les teves necessitats evolucionen.
