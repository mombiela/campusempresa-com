Introducció a les Proves Unitàries

Les proves unitàries són un mètode de prova de programari on les unitats o components individuals d'un programari es proven de manera aïllada. En el context de React Native, les proves unitàries ajuden a assegurar que els teus components i funcions funcionen com s'espera.

Per què Utilitzar Jest per a les Proves Unitàries?

Jest és un marc de proves de JavaScript mantingut per Facebook, dissenyat per assegurar la correcció de qualsevol base de codi JavaScript. És especialment adequat per provar aplicacions React i React Native a causa de la seva simplicitat i característiques potents.

  • Configuració zero: Jest funciona directament per a la majoria de projectes JavaScript.
  • Proves de snapshot: Jest pot capturar snapshots d'arbres React per assegurar la consistència de la interfície d'usuari.
  • Mocking: Jest proporciona capacitats de mocking potents per aïllar components i funcions.

Configuració de Jest en un Projecte React Native

Instal·lació

Per començar amb Jest en un projecte React Native, has d'instal·lar Jest i les seves dependències.

npm install --save-dev jest @testing-library/react-native @testing-library/jest-native

Configuració

Crea un fitxer jest.config.js a l'arrel del teu projecte per configurar Jest.

module.exports = {
  preset: 'react-native',
  setupFilesAfterEnv: ['@testing-library/jest-native/extend-expect'],
  transformIgnorePatterns: [
    'node_modules/(?!(jest-)?react-native|@react-native|@react-navigation)',
  ],
};

Escriure la Teva Primera Prova

Provar una Funció Simple

Comencem escrivint una prova per a una funció simple.

// sum.js
export const sum = (a, b) => a + b;
// sum.test.js
import { sum } from './sum';

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Executar la Prova

Pots executar les teves proves utilitzant la següent comanda:

npm test

Provar Components React Native

Component d'Exemple

Considera un component React Native simple:

// Greeting.js
import React from 'react';
import { Text } from 'react-native';

const Greeting = ({ name }) => {
  return <Text>Hello, {name}!</Text>;
};

export default Greeting;

Escriure una Prova per al Component

Per provar aquest component, pots utilitzar @testing-library/react-native.

// Greeting.test.js
import React from 'react';
import { render } from '@testing-library/react-native';
import Greeting from './Greeting';

test('renders the correct greeting message', () => {
  const { getByText } = render(<Greeting name="John" />);
  expect(getByText('Hello, John!')).toBeTruthy();
});

Proves de Snapshot

Les proves de snapshot són una característica de Jest que et permeten prendre una "instantània" del teu component renderitzat i comparar-la amb un fitxer de snapshot de referència emmagatzemat al costat de les teves proves.

Crear una Prova de Snapshot

// Greeting.test.js
import React from 'react';
import renderer from 'react-test-renderer';
import Greeting from './Greeting';

test('renders correctly', () => {
  const tree = renderer.create(<Greeting name="John" />).toJSON();
  expect(tree).toMatchSnapshot();
});

Quan executis aquesta prova per primera vegada, Jest crearà un fitxer de snapshot. En execucions posteriors, Jest compararà la sortida renderitzada amb el snapshot existent i t'alertarà si hi ha diferències.

Mocking en Jest

El mocking és útil per aïllar el component o funció que estàs provant substituint les dependències amb implementacions simulades.

Mocking d'un Mòdul

Suposa que tens un mòdul que obté dades d'una API:

// api.js
export const fetchData = async () => {
  const response = await fetch('https://api.example.com/data');
  return response.json();
};

Pots simular aquest mòdul en les teves proves:

// api.test.js
import { fetchData } from './api';

jest.mock('./api');

test('fetchData returns mock data', async () => {
  fetchData.mockResolvedValue({ data: 'mock data' });
  const data = await fetchData();
  expect(data).toEqual({ data: 'mock data' });
});

Conclusió

Les proves unitàries amb Jest en un projecte React Native són una manera poderosa d'assegurar que els teus components i funcions funcionen com s'espera. Configurant Jest, escrivint proves per a funcions i components, utilitzant proves de snapshot i aprofitant les capacitats de mocking de Jest, pots crear una suite de proves robusta que ajudi a mantenir la qualitat i fiabilitat de la teva base de codi.

Recorda, la clau per a proves unitàries efectives és escriure proves que siguin aïllades, repetibles i fàcils d'entendre. Bones proves!
