Introducció a Axios

Axios és una biblioteca de JavaScript popular utilitzada per fer peticions HTTP des del navegador i Node.js. És àmpliament utilitzada en aplicacions React Native per la seva simplicitat i facilitat d'ús.

Característiques Clau d'Axios

  • Basat en Promeses: Axios utilitza Promeses, fent que sigui fàcil treballar amb peticions asíncrones.
  • Interceptors: Et permet interceptar peticions o respostes abans que siguin gestionades.
  • Transformació automàtica de dades JSON: Transforma automàticament les dades JSON.
  • Gestió d'errors: Proporciona una manera senzilla de gestionar errors.
  • Cancel·lació de peticions: Suporta la cancel·lació de peticions.

Instal·lant Axios en un Projecte React Native

Per utilitzar Axios en un projecte React Native, cal instal·lar-lo via npm o yarn.

npm install axios
# o
yarn add axios

Fent una Petició GET Bàsica

Per fer una petició GET utilitzant Axios, pots utilitzar el mètode axios.get. A continuació es mostra un exemple senzill de com obtenir dades d'una API.

import axios from 'axios';
import React, { useEffect, useState } from 'react';
import { View, Text } from 'react-native';

const App = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    axios.get('https://jsonplaceholder.typicode.com/posts/1')
      .then(response => {
        setData(response.data);
        setLoading(false);
      })
      .catch(error => {
        console.error('Error fetching data:', error);
        setLoading(false);
      });
  }, []);

  if (loading) {
    return <Text>Loading...</Text>;
  }

  return (
    <View>
      <Text>{data.title}</Text>
      <Text>{data.body}</Text>
    </View>
  );
};

export default App;

Explicació

  • Importació d'Axios: import axios from 'axios';
  • Gestió d'Estat: Utilitzant useState per gestionar les dades obtingudes i l'estat de càrrega.
  • Hook d'Efecte: Utilitzant useEffect per obtenir dades quan el component es munta.
  • Petició GET: axios.get per obtenir dades de l'API.
  • Gestió de la Resposta: Actualitzant l'estat amb les dades obtingudes.
  • Gestió d'Errors: Registrant errors a la consola.

Fent una Petició POST

Per enviar dades a una API, pots utilitzar el mètode axios.post. A continuació es mostra un exemple de com enviar dades a una API.

import axios from 'axios';
import React, { useState } from 'react';
import { View, Text, Button, TextInput } from 'react-native';

const App = () => {
  const [title, setTitle] = useState('');
  const [body, setBody] = useState('');
  const [response, setResponse] = useState(null);

  const handleSubmit = () => {
    axios.post('https://jsonplaceholder.typicode.com/posts', {
      title,
      body,
      userId: 1,
    })
    .then(response => {
      setResponse(response.data);
    })
    .catch(error => {
      console.error('Error posting data:', error);
    });
  };

  return (
    <View>
      <TextInput
        placeholder="Title"
        value={title}
        onChangeText={setTitle}
      />
      <TextInput
        placeholder="Body"
        value={body}
        onChangeText={setBody}
      />
      <Button title="Submit" onPress={handleSubmit} />
      {response && (
        <View>
          <Text>Response:</Text>
          <Text>{response.title}</Text>
          <Text>{response.body}</Text>
        </View>
      )}
    </View>
  );
};

export default App;

Explicació

  • Gestió d'Estat: Utilitzant useState per gestionar les entrades del formulari i les dades de resposta.
  • Petició POST: axios.post per enviar dades a l'API.
  • Gestió de la Resposta: Actualitzant l'estat amb les dades de resposta.
  • Gestió d'Errors: Registrant errors a la consola.

Utilitzant Interceptors d'Axios

Els interceptors et permeten executar el teu codi o modificar la petició/resposta abans que sigui gestionada per then o catch.

Afegint un Interceptor de Petició

axios.interceptors.request.use(
  config => {
    // Fer alguna cosa abans que la petició sigui enviada
    console.log('Request Interceptor:', config);
    return config;
  },
  error => {
    // Fer alguna cosa amb l'error de la petició
    return Promise.reject(error);
  }
);

Afegint un Interceptor de Resposta

axios.interceptors.response.use(
  response => {
    // Fer alguna cosa amb les dades de la resposta
    console.log('Response Interceptor:', response);
    return response;
  },
  error => {
    // Fer alguna cosa amb l'error de la resposta
    return Promise.reject(error);
  }
);

Explicació

  • Interceptor de Petició: Et permet modificar la petició abans que sigui enviada.
  • Interceptor de Resposta: Et permet modificar la resposta abans que sigui gestionada.

Gestió d'Errors en Axios

La gestió d'errors és crucial per a una aplicació robusta. Axios proporciona una manera senzilla de gestionar errors.

Exemple de Gestió d'Errors

axios.get('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => {
    console.log('Data:', response.data);
  })
  .catch(error => {
    if (error.response) {
      // El servidor va respondre amb un estat diferent del rang 200
      console.error('Response Error:', error.response.data);
    } else if (error.request) {
      // La petició es va fer però no es va rebre cap resposta
      console.error('Request Error:', error.request);
    } else {
      // Alguna altra cosa va passar
      console.error('Error:', error.message);
    }
  });

Explicació

  • Tipus d'Errors:
    • Error de Resposta: Error de resposta del servidor.
    • Error de Petició: No es va rebre cap resposta del servidor.
    • Altres Errors: Qualsevol altre error.

Conclusió

Axios és una biblioteca potent i fàcil d'utilitzar per fer peticions HTTP en aplicacions React Native. Simplifica el procés d'obtenir i enviar dades, gestionar errors i administrar interceptors de petició/resposta. Entenent i utilitzant Axios, pots millorar les capacitats de xarxa de les teves aplicacions React Native, fent-les més robustes i eficients.
