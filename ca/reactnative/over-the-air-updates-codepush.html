Introducció a CodePush

CodePush és un servei al núvol que permet als desenvolupadors de React Native desplegar actualitzacions d'aplicacions mòbils directament als dispositius dels seus usuaris. Això permet solucionar errors ràpidament i afegir millores de funcionalitat sense necessitat de passar pel procés de revisió de la botiga d'aplicacions.

Conceptes Clau

  • Actualitzacions Over-the-Air (OTA): Actualitzacions que es lliuren directament al dispositiu de l'usuari sense requerir una nova versió a la botiga d'aplicacions.
  • CodePush: Un servei proporcionat per Microsoft que permet actualitzacions OTA per a aplicacions React Native.
  • App Center: Un conjunt de serveis proporcionats per Microsoft que inclou CodePush, entre altres eines per al desenvolupament i desplegament d'aplicacions.

Configuració de CodePush

Prerequisits

  • Un projecte React Native.
  • Node.js i npm instal·lats.
  • Un compte a App Center.

Instal·lació

  1. Instal·la el CLI de CodePush:

    npm install -g code-push-cli

  2. Enllaça CodePush al teu projecte React Native:

    npm install --save react-native-code-push

  3. Enllaça el plugin de CodePush:

    npx react-native link react-native-code-push

Configuració

  1. Crea una aplicació CodePush:

    appcenter apps create -d MyApp -o React-Native

  2. Obté les claus de desplegament:

    appcenter codepush deployment list -a <ownerName>/<appName> -k

  3. Configura CodePush a la teva aplicació:

    • Per a iOS: Afegeix la clau de desplegament al teu fitxer AppDelegate.m.
    • Per a Android: Afegeix la clau de desplegament al teu fitxer MainApplication.java.

Exemple de Configuració

iOS (AppDelegate.m):

#import <CodePush/CodePush.h>

- (NSURL *)sourceURLForBridge:(RCTBridge *)bridge
{
  #if DEBUG
    return [[RCTBundleURLProvider sharedSettings] jsBundleURLForBundleRoot:@"index" fallbackResource:nil];
  #else
    return [CodePush bundleURL];
  #endif
}

Android (MainApplication.java):

import com.microsoft.codepush.react.CodePush;

@Override
public String getJSBundleFile() {
  return CodePush.getJSBundleFile();
}

Desplegament d'Actualitzacions

Creació d'una Versió

  1. Agrupa el teu JavaScript:

    npx react-native bundle --platform ios --entry-file index.js --bundle-output ./ios/main.jsbundle --assets-dest ./ios

  2. Llança l'Actualització:

    appcenter codepush release-react -a <ownerName>/<appName> -d <deploymentName>

Comprovació de l'Estat del Desplegament

  • Veure l'Historial de Desplegaments: 
    appcenter codepush deployment history -a <ownerName>/<appName> -d <deploymentName>

Gestió d'Actualitzacions a la teva Aplicació

Exemple de Codi

App.js:

import React from 'react';
import { View, Text } from 'react-native';
import CodePush from 'react-native-code-push';

class App extends React.Component {
  render() {
    return (
      <View>
        <Text>Benvingut a la Demo de CodePush!</Text>
      </View>
    );
  }
}

let codePushOptions = { checkFrequency: CodePush.CheckFrequency.ON_APP_START };

App = CodePush(codePushOptions)(App);

export default App;

Explicació

  • Opcions de CodePush: L'opció checkFrequency determina amb quina freqüència l'aplicació comprova si hi ha actualitzacions. ON_APP_START significa que l'aplicació comprovarà si hi ha actualitzacions cada vegada que s'inicia.
  • Embolcallant el Component de l'Aplicació: La funció CodePush embolcalla el component principal App per habilitar les actualitzacions OTA.

Conclusió

Utilitzar CodePush per a actualitzacions over-the-air en React Native permet un desplegament ràpid de correccions d'errors i noves funcionalitats sense la necessitat d'aprovacions de la botiga d'aplicacions. Seguint els passos de configuració i desplegament, pots assegurar-te que els teus usuaris sempre tinguin la versió més recent de la teva aplicació. Això no només millora l'experiència de l'usuari, sinó que també redueix el temps i l'esforç necessaris per mantenir i actualitzar la teva aplicació.
