Introducció a l'API de Context

L'API de Context a React Native és una característica poderosa per gestionar l'estat a través de la teva aplicació sense la necessitat de passar propietats manualment a cada nivell. Et permet crear variables globals que es poden passar, facilitant la gestió i compartició de l'estat.

  • API de Context: Una manera de passar dades a través de l'arbre de components sense haver de passar propietats manualment a cada nivell.
  • Prop Drilling: El procés de passar dades d'un component pare a un component fill profundament niat a través de components intermedis.

Creant un Context

Per utilitzar l'API de Context, primer has de crear un context. Això es fa utilitzant la funció createContext.

import React, { createContext } from 'react';

// Crear un Context
const MyContext = createContext();

Proveint Context

Un cop has creat un context, necessites proveir-lo al teu arbre de components. Això es fa utilitzant el component Provider que ve amb el context.

import React, { createContext } from 'react';

// Crear un Context
const MyContext = createContext();

const App = () => {
  return (
    <MyContext.Provider value={{ data: 'Hola, Món!' }}>
      <ChildComponent />
    </MyContext.Provider>
  );
};

const ChildComponent = () => {
  return (
    <MyContext.Consumer>
      {context => <Text>{context.data}</Text>}
    </MyContext.Consumer>
  );
};

Explicació

  • MyContext.Provider: Aquest component proveeix el context als seus fills.
  • value: La propietat value del provider és el que serà accessible als components consumidors.
  • MyContext.Consumer: Aquest component consumeix el context proveït pel provider més proper.

Utilitzant el Hook useContext

El hook useContext és una manera més senzilla de consumir context en components funcionals.

import React, { createContext, useContext } from 'react';

// Crear un Context
const MyContext = createContext();

const App = () => {
  return (
    <MyContext.Provider value={{ data: 'Hola, Món!' }}>
      <ChildComponent />
    </MyContext.Provider>
  );
};

const ChildComponent = () => {
  const context = useContext(MyContext);
  return <Text>{context.data}</Text>;
};

Explicació

  • useContext(MyContext): Aquest hook et permet consumir el context directament sense necessitat d'un component consumidor.

Actualitzant el Context

Per actualitzar el context, pots passar una funció a través del valor del context.

import React, { createContext, useState, useContext } from 'react';

// Crear un Context
const MyContext = createContext();

const App = () => {
  const [data, setData] = useState('Hola, Món!');

  return (
    <MyContext.Provider value={{ data, setData }}>
      <ChildComponent />
    </MyContext.Provider>
  );
};

const ChildComponent = () => {
  const { data, setData } = useContext(MyContext);

  return (
    <View>
      <Text>{data}</Text>
      <Button title="Actualitzar" onPress={() => setData('Hola, React Native!')} />
    </View>
  );
};

Explicació

  • useState: Utilitzat per crear estat en el component pare.
  • setData: La funció per actualitzar l'estat, passada a través del valor del context.

Millors Pràctiques

  • Evita l'ús excessiu del Context: Utilitza el context només quan sigui necessari. Per a una gestió d'estat simple, considera utilitzar estat local o altres biblioteques de gestió d'estat com Redux.
  • Modularitza el Context: Crea fitxers de context separats per a diferents parts de la teva aplicació per mantenir el teu codi organitzat.
  • Consideracions de Rendiment: Tingues en compte que actualitzar el context pot causar re-renderitzacions. Optimitza dividint el context en peces més petites si és necessari.

Conclusió

L'API de Context és una eina poderosa per gestionar l'estat en aplicacions React Native. Simplifica el procés de passar dades a través de l'arbre de components i et pot ajudar a evitar el prop drilling. Entenent com crear, proveir i consumir context, així com com actualitzar-lo, pots construir aplicacions més eficients i mantenibles. Recorda utilitzar el context amb prudència i seguir les millors pràctiques per assegurar un rendiment òptim.
