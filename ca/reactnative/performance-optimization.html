L'optimització del rendiment és crucial per construir aplicacions React Native eficients i responsives. Aquest tema et guiarà a través de diverses tècniques i millors pràctiques per millorar el rendiment de les teves aplicacions React Native, des de nivell principiant fins a avançat.

Entendre el Rendiment en React Native

Abans d'endinsar-nos en tècniques d'optimització, és important entendre què significa el rendiment en el context de les aplicacions React Native.

  • Taxa de Fotogrames: Apunta a una taxa consistent de 60 fotogrames per segon (FPS) per assegurar animacions i transicions suaus.
  • Ús de Memòria: Gestió eficient de la memòria per prevenir fuites i bloquejos.
  • Temps de Càrrega: Minimitza el temps que triga la teva aplicació a iniciar-se i a ser interactiva.

Optimització de Nivell Principiant

Utilitza PureComponent i memo

PureComponent i memo de React poden ajudar a prevenir re-renderitzacions innecessàries.

import React, { PureComponent, memo } from 'react';

// Utilitzant PureComponent
class MyComponent extends PureComponent {
  render() {
    return <Text>{this.props.text}</Text>;
  }
}

// Utilitzant memo
const MyFunctionalComponent = memo(({ text }) => {
  return <Text>{text}</Text>;
});

Optimitza Imatges

  • Utilitza mides d'imatge apropiades: Evita utilitzar imatges grans quan les més petites seran suficients.
  • Utilitza la memòria cau: Utilitza biblioteques com react-native-fast-image per a una memòria cau d'imatges eficient.
import FastImage from 'react-native-fast-image';

<FastImage
  style={{ width: 200, height: 200 }}
  source={{
    uri: 'https://example.com/image.jpg',
    priority: FastImage.priority.normal,
  }}
  resizeMode={FastImage.resizeMode.contain}
/>

Optimització de Nivell Intermedi

Utilitza FlatList i SectionList

Per renderitzar llistes grans, utilitza FlatList o SectionList en lloc de ScrollView.

import React from 'react';
import { FlatList, Text, View } from 'react-native';

const DATA = [
  { id: '1', title: 'Item 1' },
  { id: '2', title: 'Item 2' },
  // Més elements...
];

const MyList = () => {
  const renderItem = ({ item }) => (
    <View>
      <Text>{item.title}</Text>
    </View>
  );

  return (
    <FlatList
      data={DATA}
      renderItem={renderItem}
      keyExtractor={item => item.id}
    />
  );
};

Evita Funcions Anònimes en Render

Passar funcions anònimes directament en el mètode render pot causar problemes de rendiment.

// Mala pràctica
<Button onPress={() => this.handlePress()} />

// Bona pràctica
<Button onPress={this.handlePress} />

Optimització de Nivell Avançat

Utilitza InteractionManager

Posposa càlculs o animacions pesades fins que les interaccions estiguin completes.

import { InteractionManager } from 'react-native';

InteractionManager.runAfterInteractions(() => {
  // Tasca costosa
});

Optimitza el Rendiment de Redux

  • Utilitza reselect: Memoitza selectors per prevenir recalculacions innecessàries.
  • Normalitza la forma de l'estat: Mantingues l'estat pla per evitar nius profunds.
import { createSelector } from 'reselect';

const getItems = state => state.items;

const getVisibleItems = createSelector(
  [getItems],
  items => items.filter(item => item.visible)
);

Utilitza el Motor Hermes

Hermes és un motor JavaScript de codi obert optimitzat per executar React Native.

  • Habilita Hermes: Afegeix el següent al teu fitxer android/app/build.gradle.
project.ext.react = [
  enableHermes: true  // Habilita Hermes
]

Conclusió

L'optimització del rendiment en React Native implica una combinació de millors pràctiques, tècniques de codificació eficients i l'ús de les eines adequades. Entenent i aplicant aquestes estratègies d'optimització, pots millorar significativament la responsivitat i l'eficiència de les teves aplicacions React Native. Recorda fer perfils i provar la teva aplicació regularment per identificar i abordar colls d'ampolla de rendiment.
