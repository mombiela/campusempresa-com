Introducció a la Gestió d'Excepcions

La gestió d'excepcions és un aspecte crucial del desenvolupament de programari robust. Permet a un programa gestionar situacions inesperades (excepcions) de manera elegant, en lloc de bloquejar-se o produir resultats incorrectes.

Conceptes Clau

  • Excepció: Un esdeveniment que interromp el flux normal del programa.
  • Bloc Try: Un bloc de codi que es prova per detectar excepcions.
  • Bloc Catch: Un bloc de codi que gestiona l'excepció.
  • Instrucció Throw: Utilitzada per senyalitzar la ocurrència d'una excepció.

Gestió Bàsica d'Excepcions

Blocs Try i Catch

En C++, les excepcions es gestionen utilitzant blocs try i catch. El bloc try conté el codi que podria llençar una excepció, i el bloc catch conté el codi per gestionar l'excepció.

#include <iostream>
using namespace std;

int main() {
    try {
        int num1 = 10;
        int num2 = 0;
        if (num2 == 0) {
            throw "Error de divisió per zero!";
        }
        cout << num1 / num2 << endl;
    } catch (const char* e) {
        cout << "Excepció: " << e << endl;
    }
    return 0;
}

En aquest exemple:

  • El bloc try conté codi que podria llençar una excepció.
  • La instrucció throw llença una excepció si num2 és zero.
  • El bloc catch captura l'excepció i la gestiona imprimint un missatge d'error.

Llençar Excepcions

Pots llençar excepcions de qualsevol tipus, incloent-hi tipus integrats i tipus definits per l'usuari.

Llençar Tipus Integrats

#include <iostream>
using namespace std;

int main() {
    try {
        throw 404;
    } catch (int e) {
        cout << "Excepció: Codi d'error " << e << endl;
    }
    return 0;
}

Llençar Tipus Definits per l'Usuari

#include <iostream>
using namespace std;

class MyException {
public:
    const char* what() const {
        return "S'ha produït una excepció personalitzada!";
    }
};

int main() {
    try {
        throw MyException();
    } catch (MyException& e) {
        cout << "Excepció: " << e.what() << endl;
    }
    return 0;
}

Blocs Catch Múltiples

Pots tenir múltiples blocs catch per gestionar diferents tipus d'excepcions.

#include <iostream>
using namespace std;

int main() {
    try {
        throw 3.14;
    } catch (int e) {
        cout << "S'ha capturat un enter: " << e << endl;
    } catch (double e) {
        cout << "S'ha capturat un doble: " << e << endl;
    } catch (...) {
        cout << "S'ha capturat una excepció desconeguda." << endl;
    }
    return 0;
}

En aquest exemple:

  • Els blocs catch gestionen diferents tipus d'excepcions.
  • El bloc catch(...) és un gestor de captura general per a qualsevol tipus d'excepció no capturada pels gestors anteriors.

Classes d'Excepció Estàndard

C++ proporciona un conjunt de classes d'excepció estàndard definides a la capçalera <stdexcept>.

Excepcions Estàndard Comunes

| Classe d'Excepció | Descripció | |-------------------|-------------| | std::exception | Classe base per a totes les excepcions estàndard | | std::runtime_error | Representa errors en temps d'execució | | std::logic_error | Representa errors lògics | | std::out_of_range | Es llença quan es produeix un error de fora de rang | | std::invalid_argument | Es llença quan es passa un argument invàlid |

Utilitzant Excepcions Estàndard

#include <iostream>
#include <stdexcept>
using namespace std;

int main() {
    try {
        throw runtime_error("S'ha produït un error en temps d'execució");
    } catch (runtime_error& e) {
        cout << "Excepció: " << e.what() << endl;
    }
    return 0;
}

Gestió Avançada d'Excepcions

Classes d'Excepció Personalitzades

Pots crear les teves pròpies classes d'excepció heretant de std::exception.

#include <iostream>
#include <exception>
using namespace std;

class MyException : public exception {
public:
    const char* what() const noexcept override {
        return "La meva excepció personalitzada";
    }
};

int main() {
    try {
        throw MyException();
    } catch (MyException& e) {
        cout << "Excepció: " << e.what() << endl;
    }
    return 0;
}

Especificacions d'Excepció (Desaconsellat)

Les especificacions d'excepció s'utilitzaven per especificar quines excepcions podria llençar una funció, però estan desaconsellades en C++11 i eliminades en C++17.

Especificador noexcept

L'especificador noexcept indica que una funció no llença cap excepció.

#include <iostream>
using namespace std;

void safeFunction() noexcept {
    cout << "Aquesta funció garanteix no llençar excepcions." << endl;
}

int main() {
    safeFunction();
    return 0;
}

Conclusió

La gestió d'excepcions en C++ és una característica poderosa que ajuda a crear programes robustos i resistents als errors. Utilitzant try, catch i throw, pots gestionar errors de manera elegant. Entendre les classes d'excepció estàndard i crear excepcions personalitzades millora encara més la teva capacitat per gestionar errors de manera efectiva. Recorda utilitzar l'especificador noexcept per a funcions que garanteixen no llençar excepcions, millorant la fiabilitat i el rendiment del codi.
