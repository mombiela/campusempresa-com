Introducció

L'encapsulació i l'abstracció són conceptes fonamentals en la programació orientada a objectes (OOP) que ajuden a construir programari robust i mantenible. En aquesta secció, explorarem aquests conceptes en el context de C++.

Encapsulació

L'encapsulació és l'agrupació de dades i mètodes que operen sobre aquestes dades dins d'una única unitat, típicament una classe. Restringeix l'accés directe a alguns dels components de l'objecte, la qual cosa és un mitjà per prevenir interferències no desitjades i l'ús inadequat de les dades.

Conceptes Clau

  • Classe: Un pla per crear objectes (una estructura de dades particular), que conté mètodes i propietats.
  • Especificadors d'Accés: Paraules clau que estableixen l'accessibilitat de classes, mètodes i altres membres. Els principals especificadors d'accés en C++ són public, private i protected.

Exemple

#include <iostream>
using namespace std;

class Rectangle {
private:
    double length;
    double width;

public:
    // Mètodes setters
    void setLength(double len) {
        if (len > 0) length = len;
    }

    void setWidth(double wid) {
        if (wid > 0) width = wid;
    }

    // Mètodes getters
    double getLength() {
        return length;
    }

    double getWidth() {
        return width;
    }

    // Mètode per calcular l'àrea
    double calculateArea() {
        return length * width;
    }
};

int main() {
    Rectangle rect;
    rect.setLength(5.0);
    rect.setWidth(3.0);

    cout << "Àrea del rectangle: " << rect.calculateArea() << endl;

    return 0;
}

En aquest exemple:

  • La classe Rectangle encapsula les propietats length i width.
  • L'accés a aquestes propietats es controla a través de mètodes setters i getters públics.
  • El mètode calculateArea opera sobre les dades encapsulades.

Abstracció

L'abstracció és el concepte d'ocultar els detalls complexos de la implementació i mostrar només les característiques necessàries d'un objecte. Ajuda a reduir la complexitat i l'esforç de programació.

Conceptes Clau

  • Classe Abstracta: Una classe que no es pot instanciar i està dissenyada per ser subclassificada. Sovint conté almenys una funció virtual pura.
  • Funció Virtual Pura: Una funció sense implementació en la classe base, que obliga les classes derivades a proporcionar una implementació.

Exemple

#include <iostream>
using namespace std;

class Shape {
public:
    // Funció virtual pura
    virtual double area() const = 0;
};

class Circle : public Shape {
private:
    double radius;

public:
    Circle(double r) : radius(r) {}

    double area() const override {
        return 3.14159 * radius * radius;
    }
};

class Rectangle : public Shape {
private:
    double length;
    double width;

public:
    Rectangle(double l, double w) : length(l), width(w) {}

    double area() const override {
        return length * width;
    }
};

int main() {
    Shape* shape1 = new Circle(5.0);
    Shape* shape2 = new Rectangle(4.0, 6.0);

    cout << "Àrea del cercle: " << shape1->area() << endl;
    cout << "Àrea del rectangle: " << shape2->area() << endl;

    delete shape1;
    delete shape2;

    return 0;
}

En aquest exemple:

  • Shape és una classe abstracta amb una funció virtual pura area.
  • Circle i Rectangle són classes concretes que hereten de Shape i proporcionen implementacions per a la funció area.
  • Aquesta abstracció ens permet utilitzar una interfície comuna (Shape) per interactuar amb diferents formes sense conèixer els seus detalls específics.

Conclusió

L'encapsulació i l'abstracció són poderosos principis de la OOP que ajuden a crear codi ben estructurat i mantenible. L'encapsulació protegeix l'estat intern d'un objecte, mentre que l'abstracció simplifica la interacció amb sistemes complexos exposant només els detalls necessaris. Entendre i aplicar aquests conceptes en C++ millorarà significativament les teves habilitats de programació i et permetrà construir aplicacions més eficients i escalables.
