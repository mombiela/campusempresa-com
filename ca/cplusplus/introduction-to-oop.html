Visió general

La Programació Orientada a Objectes (POO) és un paradigma de programació que utilitza objectes i classes per estructurar el programari de manera modular, reutilitzable i fàcil de mantenir. En aquesta secció, cobrirem els conceptes fonamentals de la POO en C++, incloent classes, objectes, herència, polimorfisme, encapsulació i abstracció.

Conceptes clau

Classes i Objectes

  • Classe: Un plànol per crear objectes. Defineix un tipus de dada agrupant dades i mètodes que treballen sobre aquestes dades.
  • Objecte: Una instància d'una classe. És un component autònom que conté propietats i mètodes necessaris per fer útil un cert tipus de dades.

Exemple:

#include <iostream>
using namespace std;

class Car {
public:
    string brand;
    string model;
    int year;

    void displayInfo() {
        cout << "Brand: " << brand << ", Model: " << model << ", Year: " << year << endl;
    }
};

int main() {
    Car car1;
    car1.brand = "Toyota";
    car1.model = "Corolla";
    car1.year = 2020;
    car1.displayInfo();

    return 0;
}

Explicació:

  • Definim una classe Car amb tres atributs públics: brand, model i year.
  • També definim un mètode displayInfo per imprimir els detalls del cotxe.
  • En la funció main, creem un objecte car1 de la classe Car i establim els seus atributs.

Encapsulació

  • Encapsulació: El concepte d'embolcallar dades i mètodes que operen sobre les dades dins d'una unitat única (classe). Restringeix l'accés directe a alguns dels components de l'objecte, cosa que és un mitjà per prevenir interferències accidentals i ús indegut de les dades.

Exemple:

#include <iostream>
using namespace std;

class Person {
private:
    string name;
    int age;

public:
    void setName(string n) {
        name = n;
    }

    string getName() {
        return name;
    }

    void setAge(int a) {
        if (a > 0) {
            age = a;
        }
    }

    int getAge() {
        return age;
    }
};

int main() {
    Person person1;
    person1.setName("Alice");
    person1.setAge(30);
    cout << "Name: " << person1.getName() << ", Age: " << person1.getAge() << endl;

    return 0;
}

Explicació:

  • La classe Person té atributs privats name i age.
  • Els mètodes públics setName, getName, setAge i getAge s'utilitzen per accedir i modificar els atributs privats.

Herència

  • Herència: Un mecanisme pel qual una classe pot heretar propietats i mètodes d'una altra classe. Promou la reutilització del codi.

Exemple:

#include <iostream>
using namespace std;

class Animal {
public:
    void eat() {
        cout << "This animal is eating." << endl;
    }
};

class Dog : public Animal {
public:
    void bark() {
        cout << "The dog is barking." << endl;
    }
};

int main() {
    Dog dog1;
    dog1.eat();  // Mètode heretat
    dog1.bark(); // Mètode propi del gos

    return 0;
}

Explicació:

  • La classe Animal té un mètode eat.
  • La classe Dog hereta de Animal i afegeix el seu propi mètode bark.
  • L'objecte Dog dog1 pot utilitzar tant el mètode heretat eat com el seu propi mètode bark.

Polimorfisme

  • Polimorfisme: La capacitat de diferents classes de ser tractades com a instàncies de la mateixa classe a través de l'herència. Permet que els mètodes facin coses diferents segons l'objecte sobre el qual actuen.

Exemple:

#include <iostream>
using namespace std;

class Animal {
public:
    virtual void makeSound() {
        cout << "Some generic animal sound." << endl;
    }
};

class Dog : public Animal {
public:
    void makeSound() override {
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() override {
        cout << "Meow!" << endl;
    }
};

int main() {
    Animal* animal1 = new Dog();
    Animal* animal2 = new Cat();

    animal1->makeSound(); // Imprimeix: Woof!
    animal2->makeSound(); // Imprimeix: Meow!

    delete animal1;
    delete animal2;

    return 0;
}

Explicació:

  • La classe base Animal té un mètode virtual makeSound.
  • Les classes derivades Dog i Cat sobreescriuen el mètode makeSound.
  • Els punters a Animal poden apuntar a objectes Dog o Cat, i el mètode makeSound correcte es crida segons el tipus real de l'objecte.

Abstracció

  • Abstracció: El concepte d'amagar els detalls complexos d'implementació i mostrar només les característiques necessàries d'un objecte. Ajuda a reduir la complexitat i l'esforç de programació.

Exemple:

#include <iostream>
using namespace std;

class AbstractDevice {
public:
    virtual void turnOn() = 0; // Funció virtual pura
    virtual void turnOff() = 0; // Funció virtual pura
};

class Fan : public AbstractDevice {
public:
    void turnOn() override {
        cout << "Fan is now ON." << endl;
    }

    void turnOff() override {
        cout << "Fan is now OFF." << endl;
    }
};

int main() {
    Fan fan1;
    fan1.turnOn();
    fan1.turnOff();

    return 0;
}

Explicació:

  • AbstractDevice és una classe abstracta amb funcions virtuals pures turnOn i turnOff.
  • La classe Fan hereta de AbstractDevice i proporciona implementacions per a les funcions virtuals pures.

Resum

En aquesta secció, hem introduït els conceptes fonamentals de la Programació Orientada a Objectes (POO) en C++. Hem cobert:

  • Classes i Objectes: Els blocs de construcció bàsics de la POO.
  • Encapsulació: Protegir les dades restringint l'accés a elles.
  • Herència: Reutilitzar codi heretant propietats i mètodes de classes existents.
  • Polimorfisme: Permetre que els mètodes es comportin de manera diferent segons l'objecte sobre el qual actuen.
  • Abstracció: Simplificar sistemes complexos mostrant només els detalls necessaris.

Entendre aquests conceptes és crucial per escriure codi modular, reutilitzable i mantenible en C++. A mesura que avancis en el curs, veuràs com s'apliquen aquests principis en escenaris més complexos.
