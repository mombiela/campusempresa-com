Introducció a la Sobrecàrrega d'Operadors

La sobrecàrrega d'operadors et permet redefinir la manera com els operadors funcionen per a tipus definits per l'usuari (classes). Això pot fer que les teves classes siguin més intuïtives d'utilitzar i pot ajudar a integrar-les perfectament amb la resta del llenguatge C++.

Conceptes Clau

  • Operadors: Símbols que realitzen operacions sobre variables i valors. Exemples inclouen +, -, *, /, ==, etc.
  • Sobrecàrrega: Proporcionar una nova definició per a un operador existent per treballar amb tipus definits per l'usuari.
  • Sintaxi: La manera específica en què es implementa la sobrecàrrega d'operadors en C++.

Sintaxi Bàsica de la Sobrecàrrega d'Operadors

Per sobrecarregar un operador, defineixes una funció amb un nom especial: operatorX, on X és l'operador que vols sobrecarregar.

Exemple: Sobrecàrrega de l'Operador `+`

Comencem amb un exemple senzill de sobrecàrrega de l'operador + per a una classe de nombres Complex.

#include <iostream>

class Complex {
public:
    double real, imag;

    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    // Sobrecàrrega de l'operador +
    Complex operator + (const Complex& other) {
        return Complex(real + other.real, imag + other.imag);
    }
};

int main() {
    Complex c1(3.0, 4.0);
    Complex c2(1.0, 2.0);
    Complex c3 = c1 + c2;

    std::cout << "c3 = " << c3.real << " + " << c3.imag << "i" << std::endl;
    return 0;
}

Explicació

  • Definició de la Classe: La classe Complex té dues variables membres: real i imag.
  • Constructor: Inicialitza les parts real i imaginària.
  • Funció de l'Operador: La funció operator + pren un altre objecte Complex com a paràmetre i retorna un nou objecte Complex amb la suma de les parts real i imaginària.
  • Ús: A la funció main, es sumen dos objectes Complex utilitzant l'operador + sobrecarregat.

Sobrecàrrega d'Altres Operadors

Pots sobrecarregar la majoria dels operadors de C++. Aquí tens alguns dels més comuns:

Sobrecàrrega de l'Operador `==`

class Complex {
public:
    double real, imag;

    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    bool operator == (const Complex& other) const {
        return (real == other.real) && (imag == other.imag);
    }
};

Sobrecàrrega de l'Operador `<<` per a la Sortida

Per sobrecarregar l'operador << per a la sortida, necessites definir-lo com una funció amiga.

#include <iostream>

class Complex {
public:
    double real, imag;

    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    friend std::ostream& operator << (std::ostream& out, const Complex& c);
};

std::ostream& operator << (std::ostream& out, const Complex& c) {
    out << c.real << " + " << c.imag << "i";
    return out;
}

int main() {
    Complex c1(3.0, 4.0);
    std::cout << "c1 = " << c1 << std::endl;
    return 0;
}

Explicació

  • Funció Amiga: La funció operator << es declara com a amiga de la classe Complex per poder accedir als membres privats.
  • Definició de la Funció: La funció pren un objecte ostream i un objecte Complex, i mostra el nombre Complex en un format llegible.

Sobrecàrrega d'Operadors Avançada

Sobrecàrrega de l'Operador `[]`

L'operador [] es pot sobrecarregar per proporcionar accés tipus array als membres de la classe.

class Array {
private:
    int arr[10];
public:
    int& operator[](int index) {
        if (index >= 0 && index < 10) {
            return arr[index];
        } else {
            throw std::out_of_range("Index out of range");
        }
    }
};

Explicació

  • Array Privat: La classe conté un array privat d'enters.
  • Funció de l'Operador: La funció operator[] retorna una referència a l'element a l'índex especificat, amb comprovació de límits.

Resum

  • Sobrecàrrega d'Operadors: Permet un comportament personalitzat per als operadors amb tipus definits per l'usuari.
  • Sintaxi: Defineix una funció amb el nom operatorX, on X és l'operador.
  • Operadors Comuns: +, ==, <<, [], etc.
  • Funcions Amigues: Necessàries per sobrecarregar operadors com << i >>.

La sobrecàrrega d'operadors pot fer que les teves classes siguin més intuïtives i fàcils d'utilitzar. No obstant això, s'ha d'utilitzar amb prudència per mantenir la llegibilitat del codi i evitar comportaments inesperats.
