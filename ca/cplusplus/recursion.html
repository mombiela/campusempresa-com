Introducció a la Recursió

La recursió és un concepte fonamental en ciències de la computació i programació on una funció es crida a si mateixa directament o indirectament. És una eina poderosa per resoldre problemes que es poden descompondre en subproblemes més petits i similars.

Conceptes Clau

  • Cas Base: La condició sota la qual la funció recursiva deixa de cridar-se a si mateixa.
  • Cas Recursiu: La part de la funció on la funció es crida a si mateixa amb arguments modificats.
  • Desbordament de Pila: Un problema potencial on massa crides recursives esgoten la memòria de la pila del programa.

Exemple Bàsic de Recursió

Comencem amb un exemple senzill: calcular el factorial d'un nombre.

Funció Factorial

El factorial d'un enter no negatiu n és el producte de tots els enters positius menors o iguals a n. Es denota per n!.

Definició Matemàtica

  • Cas Base: 0! = 1
  • Cas Recursiu: n! = n * (n-1)!

Exemple de Codi en C++

#include <iostream>

int factorial(int n) {
    if (n == 0) {
        return 1; // Cas base
    } else {
        return n * factorial(n - 1); // Cas recursiu
    }
}

int main() {
    int number = 5;
    std::cout << "Factorial de " << number << " és " << factorial(number) << std::endl;
    return 0;
}

Explicació:

  • La funció factorial es crida a si mateixa amb n-1 fins que n és 0.
  • Quan n és 0, la funció retorna 1, acabant la recursió.

Recursió Intermèdia: Seqüència de Fibonacci

La seqüència de Fibonacci és un altre exemple clàssic de recursió. Cada nombre en la seqüència és la suma dels dos precedents.

Funció Fibonacci

Definició Matemàtica

  • Cas Base: F(0) = 0, F(1) = 1
  • Cas Recursiu: F(n) = F(n-1) + F(n-2)

Exemple de Codi en C++

#include <iostream>

int fibonacci(int n) {
    if (n <= 1) {
        return n; // Casos base
    } else {
        return fibonacci(n - 1) + fibonacci(n - 2); // Cas recursiu
    }
}

int main() {
    int number = 10;
    std::cout << "Fibonacci de " << number << " és " << fibonacci(number) << std::endl;
    return 0;
}

Explicació:

  • La funció fibonacci es crida a si mateixa dues vegades, reduint la mida del problema en 1 i 2 respectivament.
  • Això resulta en un nombre exponencial de crides, que pot ser ineficient per a n grans.

Recursió Avançada: Recursió de Cua

La recursió de cua és un cas especial de recursió on la crida recursiva és l'última operació en la funció. Pot ser optimitzada pel compilador per evitar el desbordament de pila.

Factorial Recursiu de Cua

Exemple de Codi en C++

#include <iostream>

int tailFactorial(int n, int accumulator = 1) {
    if (n == 0) {
        return accumulator; // Cas base
    } else {
        return tailFactorial(n - 1, n * accumulator); // Cas recursiu
    }
}

int main() {
    int number = 5;
    std::cout << "Factorial Recursiu de Cua de " << number << " és " << tailFactorial(number) << std::endl;
    return 0;
}

Explicació:

  • La funció tailFactorial porta un paràmetre addicional accumulator per emmagatzemar resultats intermedis.
  • La crida recursiva és l'última operació, permetent al compilador optimitzar-la.

Comparant Recursió i Iteració

La recursió i la iteració sovint poden resoldre els mateixos problemes, però tenen diferents avantatges i inconvenients.

| Aspecte | Recursió | Iteració | |-----------------|-----------------------------------------|-----------------------------------------| | Llegibilitat | Sovint més llegible i concisa | Pot ser menys intuïtiva | | Ús de Memòria | Utilitza més memòria de pila | Utilitza menys memòria | | Rendiment | Pot ser més lenta degut al cost de les crides de funció | Generalment més ràpida | | Complexitat | Més fàcil d'implementar per problemes de divideix i venceràs | Millor per a bucles i iteracions simples|

Exercicis

  1. Bàsic: Escriu una funció recursiva per calcular la suma dels primers n nombres naturals.
  2. Intermedi: Implementa una funció recursiva per invertir una cadena.
  3. Avançat: Escriu una funció recursiva de cua per calcular el màxim comú divisor (MCD) de dos nombres.

Conclusió

La recursió és una tècnica poderosa en C++ que permet solucions elegants a problemes complexos. Entendre els casos base i recursius és crucial per escriure funcions recursives correctes. Tot i que la recursió pot ser menys eficient que la iteració en alguns casos, és inestimable per a problemes que encaixen naturalment en un patró recursiu. L'optimització de la recursió de cua pot ajudar a mitigar alguns dels problemes de rendiment associats amb la recursió.
