◄ Paràmetres de Funció i Tipus de Retorn
Recursió ►

La sobrecàrrega de funcions és una característica en C++ que et permet tenir més d'una funció amb el mateix nom en el mateix àmbit. Aquestes funcions es diferencien per les seves llistes de paràmetres. Aquesta característica és particularment útil quan necessites realitzar operacions similars en diferents tipus o quantitats d'entrades.

Conceptes Clau

  • Signatura de la Funció: La combinació del nom de la funció i la seva llista de paràmetres.
  • Funcions Sobrecàrregades: Funcions que comparteixen el mateix nom però tenen diferents signatures.
  • Polimorfisme en Temps de Compilació: La sobrecàrrega de funcions és un tipus de polimorfisme en temps de compilació, on la funció a cridar es determina en temps de compilació.

Exemple Bàsic de Sobrecàrrega de Funcions

Comencem amb un exemple senzill per il·lustrar la sobrecàrrega de funcions.

#include <iostream>

// Funció per sumar dos enters
int add(int a, int b) {
    return a + b;
}

// Funció per sumar dos nombres de punt flotant
double add(double a, double b) {
    return a + b;
}

int main() {
    int intResult = add(5, 3); // Crida la versió d'enters
    double doubleResult = add(5.5, 3.3); // Crida la versió de dobles

    std::cout << "Suma d'enters: " << intResult << std::endl;
    std::cout << "Suma de dobles: " << doubleResult << std::endl;

    return 0;
}

Explicació

  • Signatures de la Funció: int add(int a, int b) i double add(double a, double b) tenen el mateix nom però diferents tipus de paràmetres.
  • Crides de Funció: El compilador determina quina funció cridar basant-se en els tipus d'arguments.

Sobrecàrrega amb Diferents Quantitats de Paràmetres

També pots sobrecarregar funcions canviant la quantitat de paràmetres.

#include <iostream>

// Funció per imprimir un sol enter
void print(int a) {
    std::cout << "Enter: " << a << std::endl;
}

// Funció per imprimir dos enters
void print(int a, int b) {
    std::cout << "Enters: " << a << ", " << b << std::endl;
}

int main() {
    print(5); // Crida la versió d'un paràmetre
    print(5, 10); // Crida la versió de dos paràmetres

    return 0;
}

Explicació

  • Signatures de la Funció: void print(int a) i void print(int a, int b) tenen el mateix nom però diferents quantitats de paràmetres.
  • Crides de Funció: El compilador determina quina funció cridar basant-se en la quantitat d'arguments.

Sobrecàrrega amb Diferents Tipus de Paràmetres i Quantitats

Pots combinar diferents tipus de paràmetres i quantitats per crear funcions sobrecarregades més complexes.

#include <iostream>

// Funció per multiplicar dos enters
int multiply(int a, int b) {
    return a * b;
}

// Funció per multiplicar tres enters
int multiply(int a, int b, int c) {
    return a * b * c;
}

// Funció per multiplicar dos nombres de punt flotant
double multiply(double a, double b) {
    return a * b;
}

int main() {
    int intResult1 = multiply(2, 3); // Crida la versió de dos enters
    int intResult2 = multiply(2, 3, 4); // Crida la versió de tres enters
    double doubleResult = multiply(2.5, 3.5); // Crida la versió de dobles

    std::cout << "Multiplicació de dos enters: " << intResult1 << std::endl;
    std::cout << "Multiplicació de tres enters: " << intResult2 << std::endl;
    std::cout << "Multiplicació de dos dobles: " << doubleResult << std::endl;

    return 0;
}

Explicació

  • Signatures de la Funció: int multiply(int a, int b), int multiply(int a, int b, int c), i double multiply(double a, double b) tenen el mateix nom però diferents llistes de paràmetres.
  • Crides de Funció: El compilador determina quina funció cridar basant-se en els tipus i quantitats d'arguments.

Trampes Comunes

  • Ambigüitat: La sobrecàrrega pot portar a crides ambigües si el compilador no pot determinar quina funció cridar.
  • Arguments per Defecte: Sigues cautelós quan utilitzis arguments per defecte amb funcions sobrecarregades, ja que pot portar a ambigüitat.

Exemple d'Ambigüitat

#include <iostream>

// Funció per imprimir un enter
void print(int a) {
    std::cout << "Enter: " << a << std::endl;
}

// Funció per imprimir un doble
void print(double a) {
    std::cout << "Doble: " << a << std::endl;
}

// Crida de funció ambigua
int main() {
    print(5.0); // Ambigu: podria coincidir amb print(int) o print(double)

    return 0;
}

Resolució d'Ambigüitat

Pots resoldre l'ambigüitat convertint explícitament l'argument al tipus desitjat.

#include <iostream>

// Funció per imprimir un enter
void print(int a) {
    std::cout << "Enter: " << a << std::endl;
}

// Funció per imprimir un doble
void print(double a) {
    std::cout << "Doble: " << a << std::endl;
}

// Crida de funció resolta
int main() {
    print(static_cast<double>(5)); // Crida explícitament print(double)

    return 0;
}

Conclusió

La sobrecàrrega de funcions és una característica poderosa en C++ que et permet definir múltiples funcions amb el mateix nom però diferents llistes de paràmetres. Això pot fer que el teu codi sigui més llegible i fàcil de mantenir. No obstant això, sigues cautelós amb les possibles ambigüitats i assegura't que les teves funcions sobrecarregades siguin clarament distingibles per les seves llistes de paràmetres. Entenent i aplicant la sobrecàrrega de funcions de manera efectiva, pots escriure codi més flexible i reutilitzable.
