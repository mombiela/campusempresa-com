La gestió d'errors és un aspecte crucial de la programació que assegura que el teu codi pugui gestionar situacions inesperades de manera elegant. En aquesta secció, explorarem com es implementa la gestió d'errors en ALGOL, des de conceptes bàsics fins a tècniques avançades.

Introducció a la Gestió d'Errors

La gestió d'errors en ALGOL implica detectar i gestionar errors que es produeixen durant l'execució d'un programa. Això assegura que el programa pugui continuar executant-se o finalitzar de manera elegant sense causar comportaments inesperats.

Conceptes Clau

  • Detecció d'Errors: Identificar quan s'ha produït un error.
  • Informació d'Errors: Informar l'usuari o el sistema sobre l'error.
  • Recuperació d'Errors: Prendre mesures per recuperar-se de l'error i continuar l'execució si és possible.

Gestió Bàsica d'Errors

En ALGOL, la gestió bàsica d'errors es pot aconseguir utilitzant declaracions condicionals per comprovar possibles errors i gestionar-los adequadament.

Exemple: Divisió per Zero

Un error comú és la divisió per zero. Aquí tens com gestionar-ho en ALGOL:

begin
    real numerator, denominator, result;
    numerator := 10;
    denominator := 0;

    if denominator = 0 then
        print("Error: Divisió per zero")
    else
        result := numerator / denominator;
        print("Resultat: ", result)
    fi
end

En aquest exemple:

  • Comprovem si el denominator és zero.
  • Si ho és, imprimim un missatge d'error.
  • En cas contrari, realitzem la divisió i imprimim el resultat.

Gestió Intermèdia d'Errors

A mesura que et familiaritzis amb la gestió bàsica d'errors, pots començar a implementar tècniques més sofisticades, com ara utilitzar procediments per encapsular la lògica de gestió d'errors.

Exemple: Procediment de Gestió d'Errors

procedure safe_divide(a, b) real a, b;
begin
    if b = 0 then
        print("Error: Divisió per zero");
        return 0
    else
        return a / b
    fi
end;

begin
    real x, y, result;
    x := 10;
    y := 0;
    result := safe_divide(x, y);
    print("Resultat: ", result)
end

En aquest exemple:

  • Definim un procediment safe_divide que gestiona la divisió i comprova la divisió per zero.
  • El programa principal crida aquest procediment i gestiona el resultat.

Gestió Avançada d'Errors

La gestió avançada d'errors en ALGOL pot implicar estructures i tècniques més complexes, com ara utilitzar arrays per registrar errors o implementar codis d'error personalitzats.

Exemple: Registre d'Errors

begin
    integer i, error_log[1:10];
    real numerator, denominator, result;
    integer error_count;

    error_count := 0;

    for i := 1 step 1 until 10 do
        numerator := i;
        denominator := i - 5;  // Això causarà divisió per zero quan i = 5

        if denominator = 0 then
            error_log[error_count + 1] := i;
            error_count := error_count + 1;
            print("Error: Divisió per zero en la iteració ", i)
        else
            result := numerator / denominator;
            print("Resultat: ", result)
        fi
    od;

    print("Total d'errors: ", error_count);
    print("Registre d'errors: ");
    for i := 1 step 1 until error_count do
        print("Error en la iteració: ", error_log[i])
    od
end

En aquest exemple:

  • Utilitzem un array error_log per fer un seguiment de les iteracions on s'ha produït un error.
  • Comptem el nombre total d'errors i imprimim el registre d'errors al final.

Conclusió

La gestió d'errors en ALGOL és essencial per escriure programes robustos i fiables. Començant amb tècniques bàsiques i avançant gradualment cap a mètodes més avançats, pots assegurar-te que els teus programes puguin gestionar situacions inesperades de manera elegant. Recorda sempre comprovar possibles errors i gestionar-los adequadament per mantenir la integritat del teu programa.
