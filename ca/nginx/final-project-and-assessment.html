Visió General

El projecte final i l'avaluació d'aquest curs de NGINX estan dissenyats per consolidar el teu aprenentatge des de nivells bàsics fins a avançats. Aquest projecte posarà a prova la teva comprensió dels conceptes, configuracions i millors pràctiques de NGINX. Se't requerirà configurar un entorn complet de servidor NGINX que inclogui diverses característiques i optimitzacions cobertes al llarg del curs.

Requisits del Projecte

El teu projecte final implicarà configurar un servidor NGINX que compleixi els següents requisits:

  • Configuració Bàsica:

    • Instal·lar NGINX en un servidor Linux.
    • Configurar un bloc de servidor HTTP bàsic per servir contingut estàtic.

  • Proxy Invers:

    • Configurar NGINX com a proxy invers per a un servidor d'aplicacions backend (per exemple, Node.js, Python Flask).

  • Balanceig de Càrrega:

    • Implementar balanceig de càrrega amb múltiples servidors backend.

  • SSL/TLS:

    • Configurar SSL/TLS per a connexions segures utilitzant un certificat autogenerat o un certificat d'una CA de confiança.

  • Caché:

    • Implementar caché per millorar el rendiment.

  • Seguretat:

    • Configurar mesures de seguretat bàsiques, com ara limitació de velocitat i control d'accés.

  • Registre i Monitorització:

    • Configurar registres i configurar una monitorització bàsica utilitzant eines com nginx_status.

Instruccions Pas a Pas

1. Configuració Bàsica

  • Instal·lar NGINX:

    sudo apt update
sudo apt install nginx

  • Configurar Bloc de Servidor HTTP:

    server {
    listen 80;
    server_name example.com;
    root /var/www/html;

    location / {
        try_files $uri $uri/ =404;
    }
}

2. Proxy Invers

  • Configurar Proxy Invers: 
    server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

3. Balanceig de Càrrega

  • Implementar Balanceig de Càrrega: 
    upstream backend {
    server backend1.example.com;
    server backend2.example.com;
}

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

4. SSL/TLS

  • Configurar SSL/TLS: 
    server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/nginx.crt;
    ssl_certificate_key /etc/nginx/ssl/nginx.key;

    location / {
        try_files $uri $uri/ =404;
    }
}

5. Caché

  • Implementar Caché: 
    server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend;
        proxy_cache my_cache;
        proxy_cache_valid 200 1h;
    }
}

proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=1g;

6. Seguretat

  • Configurar Limitació de Velocitat: 
    http {
    limit_req_zone $binary_remote_addr zone=mylimit:10m rate=1r/s;

    server {
        listen 80;
        server_name example.com;

        location / {
            limit_req zone=mylimit burst=5;
            proxy_pass http://backend;
        }
    }
}

7. Registre i Monitorització

  • Configurar Registre:

    server {
    listen 80;
    server_name example.com;

    access_log /var/log/nginx/access.log;
    error_log /var/log/nginx/error.log;

    location / {
        try_files $uri $uri/ =404;
    }
}

  • Configurar Monitorització:

    server {
    listen 80;
    server_name example.com;

    location /nginx_status {
        stub_status;
        allow 127.0.0.1;
        deny all;
    }
}
