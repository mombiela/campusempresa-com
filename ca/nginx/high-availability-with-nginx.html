Introducció a l'Alta Disponibilitat

L'alta disponibilitat (HA) assegura que un sistema romangui operatiu i accessible fins i tot en cas de fallades. En el context dels servidors web, l'HA té com a objectiu minimitzar el temps d'inactivitat i garantir un servei continu.

  • Conceptes Clau:
    • Redundància: Tenir múltiples instàncies d'un servei per evitar un únic punt de fallada.
    • Balanceig de Càrrega: Distribuir el trànsit entrant entre múltiples servidors.
    • Failover: Canviar automàticament a un servidor de reserva si el servidor principal falla.
    • Controls de Salut: Monitoritzar l'estat dels servidors per assegurar-se que estan operatius.

Configuració de NGINX per a Alta Disponibilitat

Balanceig de Càrrega amb NGINX

El balanceig de càrrega és un component crític de l'alta disponibilitat. NGINX pot distribuir el trànsit entrant entre múltiples servidors de backend.

  • Balanceig de Càrrega Round Robin: Aquest és el mètode per defecte on les sol·licituds es distribueixen uniformement entre tots els servidors.

    http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

  • Balanceig de Càrrega per Menys Connexions: Aquest mètode dirigeix el trànsit al servidor amb menys connexions actives.

    http {
    upstream backend {
        least_conn;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

  • Balanceig de Càrrega per Hash d'IP: Aquest mètode assegura que les sol·licituds del mateix IP de client sempre es dirigeixin al mateix servidor.

    http {
    upstream backend {
        ip_hash;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

Controls de Salut

Els controls de salut s'utilitzen per monitoritzar l'estat dels servidors de backend i assegurar-se que són capaços de gestionar sol·licituds.

  • Control de Salut Bàsic: NGINX Plus suporta controls de salut actius, però per a la versió de codi obert, es poden utilitzar mòduls de tercers com nginx_upstream_check_module.

    http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
        check interval=3000 rise=2 fall=5 timeout=1000;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

Failover

El failover assegura que si un servidor cau, el trànsit es redirigeix automàticament a un altre servidor.

  • Configuració Simple de Failover:

    http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com backup;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

    En aquesta configuració, backend2.example.com està marcat com a servidor de reserva i només s'utilitzarà si backend1.example.com no està disponible.

Tècniques Avançades d'Alta Disponibilitat

Persistència de Sessió

La persistència de sessió (o sessions enganxoses) assegura que la sessió d'un usuari sempre es dirigeixi al mateix servidor de backend.

  • Sessions Enganxoses amb NGINX:

    http {
    upstream backend {
        sticky;
        server backend1.example.com;
        server backend2.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

Terminació SSL

La terminació SSL descarrega el procés de desxifrat/encriptació SSL al balancejador de càrrega, reduint la càrrega en els servidors de backend.

  • Configuració de Terminació SSL:

    http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
    }

    server {
        listen 443 ssl;
        ssl_certificate /path/to/cert.pem;
        ssl_certificate_key /path/to/key.pem;

        location / {
            proxy_pass http://backend;
        }
    }
}

Conclusió

L'alta disponibilitat amb NGINX implica configurar el balanceig de càrrega, els controls de salut i els mecanismes de failover per assegurar un servei continu. Entenent i implementant aquestes tècniques, es pot crear una infraestructura de servidor web robusta i resilient.

  • Resum:
    • El balanceig de càrrega distribueix el trànsit entre múltiples servidors.
    • Els controls de salut monitoritzen l'estat dels servidors.
    • El failover redirigeix el trànsit en cas de fallada del servidor.
    • Tècniques avançades com la persistència de sessió i la terminació SSL milloren la configuració d'HA.

Dominant aquests conceptes, es pot assegurar que les aplicacions web romanguin altament disponibles i amb un bon rendiment.
