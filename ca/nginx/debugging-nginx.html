La depuració de NGINX és una habilitat crucial per mantenir i optimitzar el rendiment del servidor web. Aquest tema et guiarà a través de diverses tècniques i eines per depurar de manera efectiva les configuracions i problemes de NGINX.

Entendre els registres de NGINX

Els registres són el primer lloc on mirar quan es depura NGINX. NGINX manté dos registres principals:

  • Registre d'Accés: Registra totes les sol·licituds processades pel servidor.
  • Registre d'Errors: Registra qualsevol problema trobat pel servidor.

Registre d'Accés

El registre d'accés proporciona informació detallada sobre cada sol·licitud feta al servidor. Per defecte, es troba a /var/log/nginx/access.log.

Exemple d'una entrada del registre d'accés:

127.0.0.1 - - [10/Oct/2023:13:55:36 +0000] "GET /index.html HTTP/1.1" 200 612 "-" "Mozilla/5.0"

Components clau:

  • 127.0.0.1: Adreça IP del client.
  • 10/Oct/2023:13:55:36 +0000: Data i hora de la sol·licitud.
  • "GET /index.html HTTP/1.1": Mètode de sol·licitud, URI i protocol.
  • 200: Codi d'estat HTTP.
  • 612: Mida de la resposta en bytes.
  • "Mozilla/5.0": Cadena de l'agent d'usuari.

Registre d'Errors

El registre d'errors captura qualsevol problema que es produeixi mentre es processen les sol·licituds. Per defecte, es troba a /var/log/nginx/error.log.

Exemple d'una entrada del registre d'errors:

2023/10/10 13:55:36 [error] 12345#0: *1 open() "/usr/share/nginx/html/404.html" failed (2: No such file or directory), client: 127.0.0.1, server: localhost, request: "GET /404.html HTTP/1.1", host: "localhost"

Components clau:

  • 2023/10/10 13:55:36: Data i hora de l'error.
  • [error]: Nivell del registre.
  • 12345#0: ID del procés i ID del fil.
  • open() "/usr/share/nginx/html/404.html" failed (2: No such file or directory): Missatge d'error.
  • client: 127.0.0.1: Adreça IP del client.
  • server: localhost: Nom del servidor.
  • request: "GET /404.html HTTP/1.1": Mètode de sol·licitud, URI i protocol.
  • host: "localhost": Capçalera de l'host.

Depuració de Problemes de Configuració

Els problemes de configuració són comuns quan es configura NGINX. Aquí hi ha alguns passos per depurar-los:

Comprovar la Sintaxi de la Configuració

Abans de reiniciar NGINX, sempre comprova la sintaxi de la configuració:

sudo nginx -t

Aquesta comanda validarà els fitxers de configuració i informarà de qualsevol error de sintaxi.

Recarregar la Configuració

Després de fer canvis a la configuració, recarrega NGINX per aplicar els canvis sense aturar el servidor:

sudo systemctl reload nginx

Errors Comuns de Configuració

  • Camins de Fitxers Incorrectes: Assegura't que tots els camins de fitxers a la configuració són correctes.
  • Errors de Sintaxi: Busca punts i comes, claus o directives incorrectes.
  • Problemes de Permisos: Verifica que NGINX tingui els permisos necessaris per accedir als fitxers i directoris.

Ús del Mode de Depuració

NGINX es pot compilar amb suport de depuració per proporcionar registres més detallats. Això és útil per diagnosticar problemes complexos.

Habilitar el Registre de Depuració

  1. Compilar NGINX amb Suport de Depuració:

    ./configure --with-debug
make
sudo make install

  2. Establir el Nivell de Registre de Depuració: Edita el fitxer nginx.conf per establir el nivell del registre d'errors a debug:

    error_log /var/log/nginx/error.log debug;

  3. Recarregar NGINX:

    sudo systemctl reload nginx

Eines per a la Depuració

Diverses eines poden ajudar a depurar NGINX:

  • curl: Prova sol·licituds i respostes HTTP.

    curl -I http://localhost

  • ngrep: Analitzador de paquets de xarxa per al trànsit HTTP.

    sudo ngrep -d any -W byline port 80

  • strace: Traça les crides del sistema i els senyals.

    sudo strace -p $(pgrep nginx)

Conclusió

La depuració de NGINX implica entendre i utilitzar els registres, validar les configuracions i utilitzar diverses eines per diagnosticar problemes. En dominar aquestes tècniques, pots assegurar-te que el teu servidor NGINX funcioni de manera suau i eficient. Sempre comença pels conceptes bàsics—comprovar els registres i la sintaxi de la configuració—abans d'endinsar-te en mètodes de depuració més avançats.
