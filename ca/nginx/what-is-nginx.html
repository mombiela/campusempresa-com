NGINX (pronunciat "engine-x") és un servidor web d'alt rendiment, servidor intermediari invers i equilibrador de càrrega. És conegut per la seva estabilitat, conjunt ric de característiques, configuració senzilla i baix consum de recursos. Aquest tema t'introduirà a NGINX, les seves funcionalitats bàsiques i per què és àmpliament utilitzat en la indústria.

Conceptes Clau de NGINX

  • Servidor Web: NGINX pot servir contingut estàtic com HTML, CSS, JavaScript i imatges directament als clients.
  • Intermediari Invers: NGINX pot actuar com a intermediari per a les sol·licituds dels clients que busquen recursos d'altres servidors.
  • Equilibrador de Càrrega: NGINX pot distribuir el trànsit de xarxa entrant entre diversos servidors per assegurar que cap servidor individual es vegi aclaparat.
  • Caché HTTP: NGINX pot emmagatzemar en caché respostes dels servidors de backend per millorar el rendiment i reduir la càrrega.
  • Terminació SSL/TLS: NGINX pot gestionar el xifrat i desxifrat SSL/TLS, descarregant aquesta feina dels servidors de backend.

Per què Utilitzar NGINX?

  • Rendiment: NGINX està dissenyat per gestionar un gran nombre de connexions simultànies amb un ús mínim de recursos.
  • Escalabilitat: Es pot escalar fàcilment afegint més servidors i distribuint la càrrega.
  • Flexibilitat: NGINX suporta una àmplia gamma de protocols i es pot configurar per a diversos casos d'ús.
  • Seguretat: Proporciona característiques de seguretat robustes, incloent suport SSL/TLS i la capacitat de mitigar vulnerabilitats web comunes.

Configuració Bàsica de NGINX

Per entendre com funciona NGINX, comencem amb un exemple de configuració simple. A continuació es mostra un fitxer de configuració bàsic de NGINX (nginx.conf):

worker_processes 1;

events {
    worker_connections 1024;
}

http {
    server {
        listen 80;
        server_name example.com;

        location / {
            root /usr/share/nginx/html;
            index index.html index.htm;
        }
    }
}

Explicació

  • worker_processes 1;: Especifica el nombre de processos de treball. Per a una configuració bàsica, un procés de treball és suficient.
  • events { worker_connections 1024; }: Defineix el nombre màxim de connexions simultànies que pot gestionar cada procés de treball.
  • http { ... }: El bloc HTTP conté la configuració per gestionar sol·licituds HTTP.
    • server { ... }: Defineix un servidor virtual.
      • listen 80;: Especifica que el servidor escolta al port 80 (port HTTP per defecte).
      • server_name example.com;: Defineix el nom de domini per al servidor.
      • location / { ... }: Configura com gestionar les sol·licituds a l'URL arrel (/).
        • root /usr/share/nginx/html;: Especifica el directori des del qual servir fitxers estàtics.
        • index index.html index.htm;: Defineix els fitxers per defecte a servir si es sol·licita l'URL arrel.

Conclusió

NGINX és una eina versàtil i poderosa per servir webs, actuar com a intermediari invers i equilibrar càrregues. La seva gestió eficient de connexions i baix consum de recursos el fan una excel·lent elecció per a llocs web i aplicacions amb molt trànsit. Entendre els conceptes bàsics de la configuració de NGINX és el primer pas per aprofitar tot el seu potencial en els teus projectes. A mesura que avancis en aquest curs, aprendràs configuracions més avançades i casos d'ús per a NGINX.
