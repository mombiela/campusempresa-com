Introducció a NGINX

NGINX és un servidor HTTP d'alt rendiment, proxy invers i servidor proxy IMAP/POP3. És conegut per la seva estabilitat, conjunt ric de característiques, configuració senzilla i baix consum de recursos.

Conceptes clau

  • Bloc de servidor: Defineix un servidor virtual utilitzat per gestionar sol·licituds.
  • Bloc de localització: Defineix com processar conjunts específics de sol·licituds.
  • Directives: Instruccions que controlen el comportament de NGINX.

Instal·lant NGINX

Abans de configurar NGINX, cal instal·lar-lo. Aquí tens com instal·lar NGINX en un sistema basat en Debian:

sudo apt update
sudo apt install nginx

Estructura bàsica del fitxer de configuració

El fitxer de configuració principal de NGINX es troba típicament a /etc/nginx/nginx.conf. Aquí tens una estructura bàsica:

user www-data;
worker_processes auto;
pid /run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    include /etc/nginx/conf.d/*.conf;
    include /etc/nginx/sites-enabled/*;
}

Explicació:

  • user: Defineix l'usuari i el grup amb els quals s'executaran els processos de treball de NGINX.
  • worker_processes: Especifica el nombre de processos de treball.
  • events: Conté directives que afecten el processament de connexions.
  • http: Context principal per a la configuració HTTP.

Configurant un bloc de servidor

Un bloc de servidor defineix un servidor virtual. Aquí tens un exemple de configuració per a un bloc de servidor bàsic:

server {
    listen 80;
    server_name example.com www.example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }

    error_page 404 /404.html;
    location = /404.html {
        internal;
    }
}

Explicació:

  • listen: Especifica el port en què NGINX escoltarà.
  • server_name: Defineix els noms de domini als quals respondrà aquest bloc de servidor.
  • location: Defineix com processar les sol·licituds per a l'URL arrel (/).
  • root: Especifica el directori arrel per al servidor.
  • index: Defineix els fitxers d'índex a buscar quan es sol·licita un directori.
  • error_page: Personalitza la pàgina d'error per a un codi d'estat HTTP específic.

Configurant un bloc de localització

Els blocs de localització permeten definir com NGINX ha de processar conjunts específics de sol·licituds. Aquí tens un exemple:

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }

    location /images/ {
        root /var/www/media;
    }

    location /api/ {
        proxy_pass http://localhost:3000;
    }
}

Explicació:

  • location /images/: Les sol·licituds a /images/ es serviran des de /var/www/media.
  • location /api/: Les sol·licituds a /api/ es redirigiran a http://localhost:3000.

Directives comunes

Aquí tens algunes directives comunes que podries utilitzar en la teva configuració de NGINX:

| Directiva | Descripció | |-----------------|-------------------------------------------------| | listen | Especifica el port i l'adreça per escoltar. | | server_name | Defineix els noms de domini per al bloc de servidor. | | root | Estableix el directori arrel per a les sol·licituds. | | index | Especifica els fitxers d'índex per als directoris. | | error_page | Personalitza les pàgines d'error. | | proxy_pass | Redirigeix les sol·licituds a un altre servidor. | | location | Defineix com processar URI de sol·licitud específiques. |

Conclusió

Entendre la configuració bàsica de NGINX és crucial per configurar un servidor web estable i eficient. Dominant els blocs de servidor, blocs de localització i directives comunes, pots gestionar i servir contingut web de manera efectiva. En les següents seccions, aprofundirem en configuracions més avançades i optimitzacions.
