Introducció

En aquesta secció, explorarem com es pot utilitzar NGINX com a equilibrador de càrrega. L'equilibrament de càrrega és un aspecte crític de les aplicacions web modernes, assegurant una alta disponibilitat i fiabilitat distribuint el trànsit entrant a través de múltiples servidors.

Conceptes clau

  • Equilibrament de càrrega: El procés de distribuir el trànsit de xarxa o aplicació a través de múltiples servidors.
  • NGINX: Un servidor HTTP d'alt rendiment i proxy invers, així com un servidor proxy IMAP/POP3.
  • Proxy invers: Un tipus de servidor proxy que recupera recursos en nom d'un client des d'un o més servidors.

Tipus d'equilibrament de càrrega en NGINX

NGINX admet diversos mètodes d'equilibrament de càrrega:

  • Round Robin: Distribueix les sol·licituds uniformement a tots els servidors.
  • Least Connections: Envia sol·licituds al servidor amb menys connexions actives.
  • IP Hash: Distribueix les sol·licituds basant-se en l'adreça IP del client.

Exemple de configuració: Equilibrament de càrrega Round Robin

Comencem amb un exemple bàsic de configuració de NGINX com a equilibrador de càrrega utilitzant el mètode Round Robin.

Pas 1: Instal·lar NGINX

Primer, assegureu-vos que NGINX està instal·lat al vostre sistema. Podeu instal·lar-lo utilitzant les següents comandes:

# Per a Debian/Ubuntu
sudo apt update
sudo apt install nginx

# Per a CentOS/RHEL
sudo yum install epel-release
sudo yum install nginx

Pas 2: Configurar NGINX

Editeu el fitxer de configuració de NGINX per configurar l'equilibrament de càrrega. El fitxer de configuració es troba típicament a /etc/nginx/nginx.conf.

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

Explicació

  • upstream backend: Defineix un grup de servidors (backend1, backend2, backend3) als quals NGINX distribuirà les sol·licituds.
  • proxy_pass: Redirigeix les sol·licituds al grup upstream definit.
  • proxy_set_header: Configura diverses capçaleres per passar juntament amb la sol·licitud.

Pas 3: Reiniciar NGINX

Després de fer els canvis de configuració, reinicieu NGINX per aplicar-los.

sudo systemctl restart nginx

Configuració avançada: Equilibrament de càrrega Least Connections

Per a un equilibrament de càrrega més avançat, podeu utilitzar el mètode Least Connections. Aquest mètode envia sol·licituds al servidor amb menys connexions actives.

Exemple de configuració

Modifiqueu el bloc upstream per utilitzar la directiva least_conn.

http {
    upstream backend {
        least_conn;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

Explicació

  • least_conn: Instruccions a NGINX per utilitzar el mètode d'equilibrament de càrrega Least Connections.

Monitoratge i comprovacions de salut

Per assegurar-vos que el vostre equilibrador de càrrega distribueix el trànsit a servidors saludables, podeu configurar comprovacions de salut.

Exemple de configuració

Afegiu la directiva health_check dins del bloc upstream.

http {
    upstream backend {
        least_conn;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
        health_check;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

Explicació

  • health_check: Habilita les comprovacions de salut per als servidors upstream.

Conclusió

En aquest estudi de cas, hem cobert els conceptes bàsics de la configuració de NGINX com a equilibrador de càrrega utilitzant diferents mètodes com Round Robin i Least Connections. També hem tocat la importància del monitoratge i les comprovacions de salut per assegurar la fiabilitat del vostre equilibrador de càrrega. Entenent aquests conceptes i configuracions, podeu distribuir el trànsit de manera efectiva i mantenir una alta disponibilitat per a les vostres aplicacions web.
