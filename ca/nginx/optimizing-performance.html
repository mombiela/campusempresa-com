Introducció a l'Optimització del Rendiment

Optimitzar el rendiment del teu servidor NGINX és crucial per assegurar una entrega de serveis web ràpida, fiable i eficient. Aquesta secció et guiarà a través de diverses tècniques i millors pràctiques per millorar el rendiment del teu servidor NGINX, des de nivells bàsics fins a avançats.

Conceptes Clau

  • Emmagatzematge en memòria cau: Emmagatzemar dades accedides freqüentment en memòria per reduir els temps de càrrega.
  • Equilibri de càrrega: Distribuir el trànsit entrant entre diversos servidors per assegurar que cap servidor estigui sobrecarregat.
  • Compressió: Reduir la mida dels fitxers enviats del servidor al client per accelerar els temps de càrrega.
  • Gestió de connexions: Gestionar eficientment les connexions dels clients per maximitzar el rendiment del servidor.
  • Límits de recursos: Establir límits adequats als recursos per prevenir l'ús excessiu i assegurar l'estabilitat.

Emmagatzematge en memòria cau

L'emmagatzematge en memòria cau pot reduir significativament la càrrega del teu servidor i millorar els temps de resposta.

Principiant: Emmagatzematge en memòria cau bàsic

Per habilitar l'emmagatzematge en memòria cau bàsic en NGINX, pots utilitzar la directiva proxy_cache.

http {
    proxy_cache_path /data/nginx/cache levels=1:2 keys_zone=my_cache:10m max_size=10g inactive=60m use_temp_path=off;
    
    server {
        location / {
            proxy_cache my_cache;
            proxy_pass http://backend;
            proxy_cache_valid 200 302 10m;
            proxy_cache_valid 404 1m;
        }
    }
}
  • proxy_cache_path: Defineix el camí i els paràmetres per a la memòria cau.
  • proxy_cache: Habilita l'emmagatzematge en memòria cau per a una ubicació específica.
  • proxy_pass: Redirigeix les sol·licituds al servidor de backend.
  • proxy_cache_valid: Estableix la validesa de la memòria cau per a diferents codis de resposta.

Avançat: Ajust fi de la memòria cau

Per a un emmagatzematge en memòria cau més avançat, pots ajustar els paràmetres de la memòria cau per adaptar-los millor a les teves necessitats.

http {
    proxy_cache_path /data/nginx/cache levels=1:2 keys_zone=my_cache:10m max_size=10g inactive=60m use_temp_path=off;
    
    server {
        location / {
            proxy_cache my_cache;
            proxy_pass http://backend;
            proxy_cache_valid 200 302 10m;
            proxy_cache_valid 404 1m;
            add_header X-Cache-Status $upstream_cache_status;
        }
    }
}
  • add_header X-Cache-Status $upstream_cache_status: Afegeix una capçalera a la resposta per indicar l'estat de la memòria cau.

Equilibri de càrrega

L'equilibri de càrrega ajuda a distribuir el trànsit de manera uniforme entre diversos servidors.

Principiant: Equilibri de càrrega Round Robin

La forma més senzilla d'equilibri de càrrega és el round-robin.

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
    }

    server {
        location / {
            proxy_pass http://backend;
        }
    }
}
  • upstream: Defineix un grup de servidors.
  • proxy_pass: Redirigeix les sol·licituds al grup upstream.

Avançat: Equilibri de càrrega per menys connexions

Per a una distribució de càrrega més eficient, utilitza el mètode de menys connexions.

http {
    upstream backend {
        least_conn;
        server backend1.example.com;
        server backend2.example.com;
    }

    server {
        location / {
            proxy_pass http://backend;
        }
    }
}
  • least_conn: Distribueix les sol·licituds al servidor amb menys connexions actives.

Compressió

La compressió redueix la mida dels fitxers enviats des del servidor, accelerant els temps de càrrega.

Principiant: Habilitar la compressió Gzip

Habilita la compressió gzip per reduir les mides de les respostes.

http {
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
}
  • gzip on: Habilita la compressió gzip.
  • gzip_types: Especifica els tipus MIME a comprimir.

Avançat: Ajust fi de la compressió Gzip

Ajusta els paràmetres de gzip per obtenir un millor rendiment.

http {
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_comp_level 5;
    gzip_min_length 256;
    gzip_proxied any;
}
  • gzip_comp_level: Estableix el nivell de compressió (1-9).
  • gzip_min_length: Comprimeix les respostes només si superen aquesta longitud.
  • gzip_proxied: Habilita la compressió per a sol·licituds proxy.

Gestió de connexions

Una gestió eficient de les connexions és crucial per maximitzar el rendiment del servidor.

Principiant: Gestió bàsica de connexions

Estableix límits bàsics de connexions per prevenir sobrecàrregues.

http {
    server {
        listen 80;
        server_name example.com;

        location / {
            limit_conn_zone $binary_remote_addr zone=addr:10m;
            limit_conn addr 10;
        }
    }
}
  • limit_conn_zone: Defineix una zona de memòria compartida per als límits de connexions.
  • limit_conn: Estableix el nombre màxim de connexions per IP.

Avançat: Optimització de la gestió de connexions

Optimitza la gestió de connexions per obtenir un millor rendiment.

http {
    server {
        listen 80;
        server_name example.com;

        location / {
            limit_conn_zone $binary_remote_addr zone=addr:10m;
            limit_conn addr 10;
            keepalive_timeout 65;
            sendfile on;
            tcp_nopush on;
        }
    }
}
  • keepalive_timeout: Estableix el temps d'espera per a les connexions keep-alive.
  • sendfile: Habilita la transferència eficient de fitxers.
  • tcp_nopush: Redueix el nombre de paquets enviats.

Límits de recursos

Establir límits adequats als recursos assegura l'estabilitat i prevé l'ús excessiu.

Principiant: Límits bàsics de recursos

Estableix límits bàsics de recursos per prevenir la sobrecàrrega del servidor.

http {
    server {
        listen 80;
        server_name example.com;

        location / {
            client_max_body_size 1m;
        }
    }
}
  • client_max_body_size: Limita la mida dels cossos de sol·licitud dels clients.

Avançat: Ajust fi dels límits de recursos

Ajusta els límits de recursos per obtenir un millor control.

http {
    server {
        listen 80;
        server_name example.com;

        location / {
            client_max_body_size 1m;
            client_body_timeout 12;
            client_header_timeout 12;
            large_client_header_buffers 4 16k;
        }
    }
}
  • client_body_timeout: Estableix el temps d'espera per llegir els cossos de sol·licitud dels clients.
  • client_header_timeout: Estableix el temps d'espera per llegir les capçaleres de sol·licitud dels clients.
  • large_client_header_buffers: Assigna memòria intermèdia per a capçaleres grans dels clients.

Conclusió

Optimitzar el rendiment del teu servidor NGINX implica una combinació d'emmagatzematge en memòria cau, equilibri de càrrega, compressió, gestió eficient de connexions i establiment de límits adequats de recursos. Implementant aquestes tècniques, pots assegurar que el teu servidor funcioni de manera eficient, proporcionant un servei ràpid i fiable als teus usuaris.
