Monitorització de NGINX
Registre i Gestió de Registres ►

Monitoritzar NGINX és crucial per assegurar el rendiment, la fiabilitat i la seguretat del teu servidor web. Aquesta secció et guiarà a través de diversos mètodes i eines per monitoritzar NGINX, des de l'anàlisi bàsica de registres fins a solucions de monitorització avançades.

Conceptes Clau de la Monitorització de NGINX

  • Registres d'Accés: Seguiment de les sol·licituds fetes al teu servidor.
  • Registres d'Errors: Registre de qualsevol problema o error trobat per NGINX.
  • Mòdul d'Estat: Proporciona mètriques en temps real sobre el rendiment de NGINX.
  • Eines de Monitorització de Tercers: Eines com Prometheus, Grafana i Datadog per a una monitorització avançada.

Registres d'Accés

Els registres d'accés registren totes les sol·licituds processades per NGINX. Aquests registres són inavaluables per entendre els patrons de trànsit, identificar problemes potencials i per a finalitats d'auditoria.

Configuració dels Registres d'Accés

Per defecte, NGINX registra totes les sol·licituds al registre d'accés. Pots configurar el format i la ubicació d'aquests registres al fitxer de configuració de NGINX.

http {
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;
}
  • log_format: Defineix el format de les entrades del registre.
  • access_log: Especifica el fitxer on es guardaran els registres i el format a utilitzar.

Anàlisi dels Registres d'Accés

Pots utilitzar eines com awk, grep o eines especialitzades d'anàlisi de registres com GoAccess per analitzar els registres d'accés.

# Exemple: Comptar el nombre de sol·licituds per IP
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr

Registres d'Errors

Els registres d'errors capturen qualsevol problema o error que es produeixi mentre NGINX processa les sol·licituds. Això inclou errors de configuració, errors d'execució i més.

Configuració dels Registres d'Errors

Els registres d'errors es poden configurar per capturar diferents nivells de detall.

http {
    error_log /var/log/nginx/error.log warn;
}
  • error_log: Especifica el fitxer on es guardaran els registres d'errors i el nivell de detall (per exemple, debug, info, notice, warn, error, crit, alert, emerg).

Anàlisi dels Registres d'Errors

Els registres d'errors es poden analitzar utilitzant eines com grep per filtrar tipus específics d'errors.

# Exemple: Trobar tots els errors "404 Not Found"
grep "404" /var/log/nginx/error.log

Mòdul d'Estat de NGINX

El mòdul d'estat de NGINX proporciona mètriques en temps real sobre el rendiment del servidor, incloent connexions actives, sol·licituds per segon i més.

Activació del Mòdul d'Estat

Per activar el mòdul d'estat, has de configurar un bloc de localització al teu fitxer de configuració de NGINX.

http {
    server {
        location /nginx_status {
            stub_status;
            allow 127.0.0.1; # Només permetre accés local
            deny all; # Denegar tot altre accés
        }
    }
}

Accés a la Pàgina d'Estat

Un cop configurat, pots accedir a la pàgina d'estat navegant a http://your_server_ip/nginx_status.

Mètriques Proporcionades

  • Connexions actives: El nombre actual de connexions de clients actives.
  • Connexions acceptades: El nombre total de connexions de clients acceptades.
  • Connexions gestionades: El nombre total de connexions gestionades.
  • Sol·licituds: El nombre total de sol·licituds de clients.

Eines de Monitorització de Tercers

Per a una monitorització més avançada, pots integrar NGINX amb eines de tercers com Prometheus, Grafana i Datadog.

Prometheus i Grafana

Prometheus és un sistema de monitorització de codi obert que recull mètriques, i Grafana és una eina de visualització que pot mostrar aquestes mètriques.

Configuració de Prometheus amb NGINX

  1. Instal·lar l'Exportador de Prometheus per a NGINX:

    docker run -d -p 9113:9113 nginx/nginx-prometheus-exporter:latest -nginx.scrape-uri http://localhost:8080/nginx_status

  2. Configurar Prometheus:

    scrape_configs:
  - job_name: 'nginx'
    static_configs:
      - targets: ['localhost:9113']

  3. Visualitzar amb Grafana:

    • Afegeix Prometheus com a font de dades a Grafana.
    • Importa quadres de comandament de NGINX des del repositori de quadres de comandament de Grafana.

Datadog

Datadog és una plataforma de monitorització i anàlisi que proporciona una monitorització completa per a NGINX.

Configuració de Datadog amb NGINX

  1. Instal·lar l'Agent de Datadog:

    DD_API_KEY=your_api_key bash -c "$(curl -L https://s3.amazonaws.com/dd-agent/scripts/install_script.sh)"

  2. Activar la Integració de NGINX:

    # /etc/datadog-agent/conf.d/nginx.d/conf.yaml
init_config:

instances:
  - nginx_status_url: http://localhost/nginx_status

  3. Veure Mètriques a Datadog:

    • Navega al quadre de comandament de Datadog per veure les mètriques de NGINX.

Conclusió

Monitoritzar NGINX és essencial per mantenir la salut i el rendiment del teu servidor web. Utilitzant registres d'accés, registres d'errors, el mòdul d'estat de NGINX i eines de monitorització de tercers, pots obtenir informació valuosa sobre el funcionament del teu servidor i identificar i resoldre problemes ràpidament. Tant si ets un principiant com un usuari avançat, aquestes eines i tècniques t'ajudaran a assegurar que el teu servidor NGINX funcioni de manera suau i eficient.
