Introducció a l'Equilibrament de Càrrega

L'equilibrament de càrrega és una tècnica crítica utilitzada per distribuir el trànsit de xarxa o aplicació entre diversos servidors. Això assegura que cap servidor individual suporti massa demanda, millorant la fiabilitat i el rendiment.

  • Definició: L'equilibrament de càrrega distribueix el trànsit de xarxa entrant entre diversos servidors.
  • Beneficis:
    • Millora de la fiabilitat i disponibilitat.
    • Rendiment millorat.
    • Escalabilitat.

Tipus d'Equilibrament de Càrrega

NGINX admet diversos mètodes d'equilibrament de càrrega:

  • Round Robin: Distribueix les sol·licituds seqüencialment.
  • Least Connections: Envia sol·licituds al servidor amb menys connexions actives.
  • IP Hash: Dirigeix les sol·licituds basant-se en l'adreça IP del client.

Configuració de NGINX per a l'Equilibrament de Càrrega

Instal·lació de NGINX

Primer, assegureu-vos que NGINX està instal·lat al vostre servidor. Per a la majoria de distribucions Linux, podeu utilitzar el gestor de paquets:

sudo apt update
sudo apt install nginx

Configuració Bàsica d'Equilibrament de Càrrega

Per configurar l'equilibrament de càrrega, heu d'editar el fitxer de configuració de NGINX, normalment situat a /etc/nginx/nginx.conf o /etc/nginx/conf.d/default.conf.

Exemple: Equilibrament de Càrrega Round Robin

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}
  • Explicació:
    • upstream backend { ... }: Defineix un grup de servidors backend.
    • server backend1.example.com;: Especifica cada servidor backend.
    • proxy_pass http://backend;: Redirigeix les sol·licituds al grup backend.

Exemple: Equilibrament de Càrrega Least Connections

http {
    upstream backend {
        least_conn;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}
  • Explicació:
    • least_conn;: Activa l'equilibrament de càrrega per menys connexions.

Exemple: Equilibrament de Càrrega IP Hash

http {
    upstream backend {
        ip_hash;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}
  • Explicació:
    • ip_hash;: Activa l'equilibrament de càrrega per hash d'IP.

Tècniques Avançades d'Equilibrament de Càrrega

Verificacions de Salut

Les verificacions de salut asseguren que NGINX només enviï sol·licituds a servidors backend saludables.

Exemple: Configuració de Verificacions de Salut

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;

        # Configuració de verificació de salut
        server backend1.example.com max_fails=3 fail_timeout=30s;
        server backend2.example.com max_fails=3 fail_timeout=30s;
        server backend3.example.com max_fails=3 fail_timeout=30s;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}
  • Explicació:
    • max_fails=3: Marca un servidor com a inactiu després de 3 intents fallits.
    • fail_timeout=30s: El servidor es considerarà inactiu durant 30 segons després de fallar.

Persistència de Sessió

La persistència de sessió (o sessions enganxoses) assegura que un client sempre sigui dirigit al mateix servidor backend.

Exemple: Configuració de Persistència de Sessió

http {
    upstream backend {
        ip_hash;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}
  • Explicació:
    • ip_hash;: Assegura que les sol·licituds del mateix IP del client sempre siguin enviades al mateix servidor.

Monitorització i Registre

La monitorització i el registre són essencials per mantenir un entorn equilibrat de càrrega saludable.

  • Registres d'Accés: Feu un seguiment de les sol·licituds als vostres servidors.
  • Registres d'Errors: Registreu problemes i errors.

Exemple: Activació de Registres d'Accés

http {
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}
  • Explicació:
    • log_format main ...: Defineix un format de registre personalitzat.
    • access_log /var/log/nginx/access.log main;: Especifica el fitxer de registre i el format.

Conclusió

L'equilibrament de càrrega amb NGINX és una manera poderosa de millorar el rendiment, la fiabilitat i l'escalabilitat de les vostres aplicacions web. Comprenent i implementant diferents mètodes d'equilibrament de càrrega, verificacions de salut, persistència de sessió i monitorització, podeu assegurar-vos que les vostres aplicacions funcionin de manera suau i eficient.

Recordeu:

  • Trieu el mètode d'equilibrament de càrrega adequat segons les vostres necessitats.
  • Implementar verificacions de salut per mantenir una piscina de servidors saludable.
  • Utilitzeu la persistència de sessió si la vostra aplicació ho requereix.
  • Monitoritzeu i registreu el trànsit per fer un seguiment del rendiment del vostre sistema.

Dominant aquestes tècniques, estareu ben preparats per gestionar escenaris de trànsit elevat i assegurar una experiència sense interrupcions per als vostres usuaris.
