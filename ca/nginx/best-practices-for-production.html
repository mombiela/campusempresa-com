Introducció

Quan desplegueu NGINX en un entorn de producció, és crucial seguir les millors pràctiques per assegurar el rendiment, la seguretat i la mantenibilitat. Aquesta secció cobrirà conceptes clau i configuracions que us ajudaran a optimitzar la vostra configuració de NGINX per a l'ús en producció.

Gestió de Configuració

  • Utilitzeu Control de Versions: Emmagatzemeu els vostres fitxers de configuració de NGINX en un sistema de control de versions com Git. Això us permetrà fer un seguiment dels canvis, tornar a versions anteriors i col·laborar amb els membres de l'equip.

  • Configuració Modular: Desglosseu la vostra configuració de NGINX en fitxers més petits i manejables. Utilitzeu la directiva include per incloure aquests fitxers en la vostra configuració principal.

    # Fitxer de configuració principal (nginx.conf)
include /etc/nginx/conf.d/*.conf;
include /etc/nginx/sites-enabled/*;

Optimització del Rendiment

  • Processos i Connexions de Treballadors: Configureu el nombre de processos i connexions de treballadors per adaptar-se a les capacitats del vostre servidor.

    worker_processes auto;  # Configurat automàticament al nombre de nuclis de CPU disponibles
events {
    worker_connections 1024;  # Ajusteu segons la capacitat del vostre servidor
}

  • Compressió Gzip: Activeu la compressió gzip per reduir la mida de les respostes i millorar els temps de càrrega.

    gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

  • Caché: Utilitzeu la caché per reduir la càrrega en els vostres servidors de backend i millorar els temps de resposta.

    proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=10g inactive=60m use_temp_path=off;
server {
    location / {
        proxy_cache my_cache;
        proxy_pass http://backend_server;
    }
}

Seguretat

  • SSL/TLS: Utilitzeu sempre SSL/TLS per xifrar el trànsit entre els clients i el vostre servidor. Obteniu certificats d'una Autoritat de Certificació (CA) de confiança o utilitzeu Let's Encrypt per a certificats gratuïts.

    server {
    listen 443 ssl;
    server_name example.com;
    ssl_certificate /etc/ssl/certs/example.com.crt;
    ssl_certificate_key /etc/ssl/private/example.com.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
}

  • Capçaleres de Seguretat HTTP: Afegiu capçaleres de seguretat per protegir-vos contra vulnerabilitats web comunes.

    add_header X-Content-Type-Options nosniff;
add_header X-Frame-Options DENY;
add_header X-XSS-Protection "1; mode=block";

  • Limitació de Ràtio: Implementeu la limitació de ràtio per protegir-vos contra atacs de força bruta i abusos.

    http {
    limit_req_zone $binary_remote_addr zone=one:10m rate=1r/s;
    server {
        location /login {
            limit_req zone=one burst=5;
        }
    }
}

Monitoratge i Registre

  • Registres d'Accés i Errors: Configureu registres d'accés i errors per monitoritzar el trànsit i diagnosticar problemes.

    http {
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';
    access_log /var/log/nginx/access.log main;
    error_log /var/log/nginx/error.log warn;
}

  • Mètriques i Alertes: Utilitzeu eines com Prometheus i Grafana per recollir mètriques i configurar alertes per a problemes crítics.

Alta Disponibilitat

  • Equilibri de Càrrega: Distribuïu el trànsit entre diversos servidors de backend per assegurar una alta disponibilitat i fiabilitat.

    upstream backend {
    server backend1.example.com;
    server backend2.example.com;
}
server {
    location / {
        proxy_pass http://backend;
    }
}

  • Comprovacions de Salut: Configureu comprovacions de salut per eliminar automàticament els servidors no saludables del grup d'equilibri de càrrega.

    upstream backend {
    server backend1.example.com;
    server backend2.example.com;
    server backend3.example.com down;  # Marcar un servidor com a inactiu
}

Conclusió

Seguir aquestes millors pràctiques us ajudarà a desplegar NGINX en un entorn de producció que sigui segur, eficient i fàcil de mantenir. Gestionant les configuracions de manera efectiva, optimitzant el rendiment, assegurant el vostre servidor, monitoritzant els registres i assegurant una alta disponibilitat, podreu proporcionar un servei robust i fiable als vostres usuaris.
