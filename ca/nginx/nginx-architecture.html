Introducció a NGINX

NGINX és un servidor HTTP d'alt rendiment, proxy invers i servidor proxy IMAP/POP3. És conegut per la seva estabilitat, conjunt ric de funcionalitats, configuració senzilla i baix consum de recursos.

Components Clau de l'Arquitectura de NGINX

Processos Master i Worker

  • Procés Master: Gestiona els processos worker, llegeix i avalua els fitxers de configuració, i manté el procés principal.
  • Processos Worker: Gestionen el processament real de les sol·licituds. Cada procés worker és monofil i gestiona múltiples connexions utilitzant E/S asíncrona i no bloquejant.

Arquitectura Basada en Esdeveniments

  • Bucle d'Esdeveniments: NGINX utilitza un model basat en esdeveniments on cada procés worker executa un bucle que espera esdeveniments (com connexions entrants) i els processa.
  • E/S No Bloquejant: Permet a NGINX gestionar moltes connexions simultàniament sense crear un nou fil per a cada connexió.

Mòduls

  • Mòduls Bàsics: Proporcionen funcionalitats essencials com el processament HTTP, de correu i de flux.
  • Mòduls Opcional: Amplien les capacitats de NGINX, com SSL, compressió Gzip i reescriptura d'URL.

Fitxers de Configuració

  • nginx.conf: El fitxer de configuració principal on es defineixen els paràmetres del servidor, els processos worker i altres directives.
  • Directives Include: Permet dividir la configuració en múltiples fitxers per a una millor organització.

Exemple de Configuració de NGINX

A continuació es mostra un exemple senzill d'un fitxer nginx.conf:

worker_processes  1;

events {
    worker_connections  1024;
}

http {
    server {
        listen       80;
        server_name  localhost;

        location / {
            root   html;
            index  index.html index.htm;
        }

        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }
    }
}

Explicació

  • worker_processes 1;: Defineix el nombre de processos worker.
  • events: Bloc on es configuren els paràmetres relacionats amb esdeveniments.
    • worker_connections 1024;: Nombre màxim de connexions simultànies per worker.
  • http: Bloc on es defineixen els paràmetres del servidor HTTP.
    • server: Defineix un servidor virtual.
      • listen 80;: El servidor escolta al port 80.
      • server_name localhost;: Nom del servidor.
      • location /: Defineix com processar les sol·licituds a l'URL arrel.
        • root html;: Directori arrel per als fitxers.
        • index index.html index.htm;: Fitxers per defecte a servir.
      • error_page 500 502 503 504 /50x.html;: Pàgines d'error personalitzades.
      • location = /50x.html: Defineix com servir la pàgina d'error personalitzada.

Conceptes Avançats

Balanceig de Càrrega

NGINX pot distribuir el trànsit entrant entre múltiples servidors per assegurar alta disponibilitat i fiabilitat.

Exemple de Configuració

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

Explicació

  • upstream backend: Defineix un grup de servidors backend.
    • server backend1.example.com;: Primer servidor backend.
    • server backend2.example.com;: Segon servidor backend.
  • proxy_pass http://backend;: Redirigeix les sol·licituds al grup backend definit.

Emmagatzematge en Memòria Cau

NGINX pot emmagatzemar en memòria cau les respostes per millorar el rendiment i reduir la càrrega als servidors backend.

Exemple de Configuració

http {
    proxy_cache_path /data/nginx/cache levels=1:2 keys_zone=my_cache:10m max_size=10g inactive=60m use_temp_path=off;

    server {
        listen 80;

        location / {
            proxy_cache my_cache;
            proxy_pass http://backend;
        }
    }
}

Explicació

  • proxy_cache_path: Defineix el camí d'emmagatzematge de la memòria cau i els paràmetres.
    • /data/nginx/cache: Directori per a l'emmagatzematge de la memòria cau.
    • levels=1:2: Estructura de directoris per als fitxers de la memòria cau.
    • keys_zone=my_cache:10m: Defineix una zona de memòria compartida per a les claus de la memòria cau.
    • max_size=10g: Mida màxima de la memòria cau.
    • inactive=60m: Temps després del qual els elements inactius de la memòria cau són eliminats.
    • use_temp_path=off: Desactiva l'emmagatzematge temporal de fitxers.
  • proxy_cache my_cache;: Activa l'emmagatzematge en memòria cau per a la ubicació.

Resum

Comprendre l'arquitectura de NGINX és crucial per configurar i optimitzar el teu servidor de manera efectiva. Els components clau inclouen els processos master i worker, l'arquitectura basada en esdeveniments i el disseny modular. Dominant els fitxers de configuració i les funcionalitats avançades com el balanceig de càrrega i l'emmagatzematge en memòria cau, pots aprofitar NGINX per construir aplicacions web d'alt rendiment i escalables.
