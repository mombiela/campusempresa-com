Visió general

NGINX és un servidor web potent que també es pot utilitzar com a proxy invers, equilibrador de càrrega i memòria cau HTTP. Entendre com NGINX gestiona les sol·licituds HTTP és fonamental per configurar i optimitzar el teu servidor web.

Conceptes clau

  • Protocol HTTP: La base de la comunicació de dades a la web.
  • Fitxer de configuració de NGINX: El fitxer principal on defineixes com NGINX gestiona les sol·licituds HTTP.
  • Directives: Instruccions en el fitxer de configuració que controlen el comportament de NGINX.
  • Contextos: Seccions en el fitxer de configuració que agrupen directives relacionades.

Estructura del fitxer de configuració de NGINX

El fitxer de configuració de NGINX es troba típicament a /etc/nginx/nginx.conf. Consisteix en directives i contextos.

Exemple de fitxer de configuració

user www-data;
worker_processes auto;
pid /run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    sendfile on;
    keepalive_timeout 65;

    server {
        listen 80;
        server_name example.com;

        location / {
            root /var/www/html;
            index index.html index.htm;
        }
    }
}

Explicació

  • Directives globals: user, worker_processes, i pid són configuracions globals.
  • Context events: Conté directives relacionades amb connexions de xarxa.
  • Context http: Conté directives per gestionar el tràfic HTTP.
  • Bloc server: Defineix un servidor virtual.
  • Bloc location: Especifica com processar sol·licituds per a una ubicació específica.

Gestió de sol·licituds HTTP

NGINX processa les sol·licituds HTTP en una sèrie de fases, incloent:

  1. Fase de connexió: Establiment d'una connexió amb el client.
  2. Fase de sol·licitud: Recepció i anàlisi de la sol·licitud HTTP.
  3. Fase de resposta: Generació i enviament de la resposta HTTP.

Exemple: Configuració bàsica d'un servidor HTTP

server {
    listen 80;
    server_name mywebsite.com;

    location / {
        root /usr/share/nginx/html;
        index index.html;
    }

    location /images/ {
        root /data;
    }
}

Explicació

  • listen: Especifica el port en què NGINX escolta.
  • server_name: Defineix el nom de domini.
  • location /: Gestiona les sol·licituds a l'URL arrel.
  • location /images/: Gestiona les sol·licituds a la ruta d'URL /images/.

Directives comunes

  • listen: Especifica l'adreça IP i el port.
  • server_name: Defineix els noms de domini.
  • root: Estableix el directori arrel per a les sol·licituds.
  • index: Especifica el fitxer d'índex.

Exemple: Ús de directives

server {
    listen 8080;
    server_name example.org;

    location / {
        root /var/www/example;
        index index.html index.htm;
    }
}

Explicació

  • listen 8080: NGINX escolta al port 8080.
  • server_name example.org: El servidor respon a les sol·licituds per example.org.
  • root /var/www/example: El directori arrel per a les sol·licituds és /var/www/example.
  • index index.html index.htm: Els fitxers d'índex són index.html i index.htm.

Conclusió

Entendre com NGINX gestiona les sol·licituds HTTP i com configurar-lo utilitzant directives i contextos és crucial per configurar un servidor web fiable i eficient. Dominant aquests conceptes bàsics, podràs avançar cap a configuracions i optimitzacions més avançades.
