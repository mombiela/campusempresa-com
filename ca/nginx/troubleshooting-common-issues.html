Introducció

NGINX és un potent servidor web i servidor intermediari invers, però com qualsevol programari, pot trobar problemes. Aquesta secció et guiarà a través de la resolució de problemes comuns que pots enfrontar quan treballes amb NGINX, des de nivells principiants fins a avançats.

Problemes Comuns i Solucions

1. NGINX No S'inicia

  • Comprova la Sintaxi de la Configuració: Abans d'iniciar NGINX, assegura't que el teu fitxer de configuració sigui sintàcticament correcte.

    sudo nginx -t

    Si hi ha errors, NGINX proporcionarà detalls sobre què cal corregir.

  • Conflictes de Port: Assegura't que el port al qual NGINX intenta vincular-se (normalment el port 80 o 443) no estigui ja en ús per un altre servei.

    sudo lsof -i :80

    Si un altre servei està utilitzant el port, potser hauràs d'aturar aquest servei o canviar el port que utilitza NGINX.

  • Problemes de Permisos: Assegura't que NGINX tingui els permisos necessaris per accedir als seus fitxers de configuració i als directoris que serveix.

    sudo chown -R www-data:www-data /var/www/html

2. Error 403 Forbidden

  • Permisos de Fitxers: Assegura't que els fitxers i directoris servits per NGINX tinguin els permisos correctes.

    sudo chmod -R 755 /var/www/html

  • Configuració de Nginx: Comprova el fitxer nginx.conf o els fitxers de configuració específics del lloc per assegurar-te que la directiva root apunta al directori correcte.

    server {
    listen 80;
    server_name example.com;
    root /var/www/html;
    index index.html;
}

3. Error 404 Not Found

  • Comprova les Rutes dels Fitxers: Assegura't que les rutes dels fitxers especificades a la teva configuració coincideixin amb les ubicacions reals dels fitxers.
  • Directiva Root Correcta: Verifica que la directiva root al teu bloc de servidor apunta al directori correcte.

4. Error 502 Bad Gateway

  • Servei de Backend Aturat: Assegura't que el servei de backend (per exemple, un servidor PHP-FPM o Node.js) està en funcionament.

    sudo systemctl status php7.4-fpm

  • Configuració d'Upstream Incorrecta: Verifica que la configuració del servidor upstream a NGINX sigui correcta.

    upstream backend {
    server 127.0.0.1:3000;
}

server {
    listen 80;
    location / {
        proxy_pass http://backend;
    }
}

5. Error 504 Gateway Timeout

  • Augmenta els Paràmetres de Timeout: Si el servei de backend triga més a respondre, potser hauràs d'augmentar els paràmetres de timeout a la teva configuració de NGINX. 
    server {
    listen 80;
    location / {
        proxy_connect_timeout 600;
        proxy_send_timeout 600;
        proxy_read_timeout 600;
        send_timeout 600;
        proxy_pass http://backend;
    }
}

6. Problemes SSL/TLS

  • Comprova els Fitxers de Certificat: Assegura't que el certificat SSL i els fitxers de clau estan especificats correctament i són accessibles.

    server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;
}

  • Permisos Correctes: Assegura't que els fitxers de certificat tenen els permisos correctes.

    sudo chmod 600 /etc/nginx/ssl/example.com.key

7. Alt Ús de CPU/Memòria

  • Optimitza la Configuració: Revisa i optimitza els paràmetres de configuració de NGINX, com ara els processos de treball i les connexions.

    worker_processes auto;
events {
    worker_connections 1024;
}

  • Caché: Implementa la caché per reduir la càrrega al servidor.

    location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 1h;
    proxy_pass http://backend;
}

Conclusió

Resoldre problemes amb NGINX pot ser un repte, però entendre els problemes comuns i les seves solucions pot facilitar significativament el procés. Comença sempre comprovant la sintaxi de la configuració, verificant les rutes i permisos dels fitxers, i assegurant-te que els serveis de backend estan funcionant correctament. Amb aquests passos, podràs resoldre la majoria dels problemes comuns de NGINX de manera eficient.
