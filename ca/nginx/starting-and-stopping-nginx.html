En aquesta secció, cobrirem les operacions essencials d'iniciar i aturar el servidor web NGINX. Aquestes són tasques fonamentals que tot administrador de NGINX hauria de conèixer.

Conceptes Clau

  • Iniciar NGINX: Iniciar el servei NGINX per començar a gestionar el trànsit web.
  • Aturar NGINX: Aturar el servei NGINX per deixar de gestionar el trànsit web.
  • Reiniciar NGINX: Aturar i després iniciar el servei NGINX, sovint utilitzat per aplicar canvis de configuració.
  • Recarregar NGINX: Aplicar canvis de configuració sense aturar el servei.

Iniciar NGINX

Per iniciar NGINX, cal utilitzar la línia de comandes. La comanda pot variar lleugerament segons el teu sistema operatiu.

A Linux (Systemd)

sudo systemctl start nginx

A Linux (SysVinit)

sudo service nginx start

A macOS (Homebrew)

brew services start nginx

Explicació

  • sudo: Executa la comanda amb privilegis de superusuari.
  • systemctl start nginx: Utilitza systemd per iniciar el servei NGINX.
  • service nginx start: Utilitza SysVinit per iniciar el servei NGINX.
  • brew services start nginx: Utilitza Homebrew per iniciar el servei NGINX a macOS.

Aturar NGINX

Per aturar NGINX, pots utilitzar comandes similars a les d'inici, però amb la paraula clau stop.

A Linux (Systemd)

sudo systemctl stop nginx

A Linux (SysVinit)

sudo service nginx stop

A macOS (Homebrew)

brew services stop nginx

Explicació

  • systemctl stop nginx: Utilitza systemd per aturar el servei NGINX.
  • service nginx stop: Utilitza SysVinit per aturar el servei NGINX.
  • brew services stop nginx: Utilitza Homebrew per aturar el servei NGINX a macOS.

Reiniciar NGINX

Reiniciar NGINX és útil quan necessites aplicar canvis de configuració. Aquesta operació atura i després inicia el servei NGINX.

A Linux (Systemd)

sudo systemctl restart nginx

A Linux (SysVinit)

sudo service nginx restart

A macOS (Homebrew)

brew services restart nginx

Explicació

  • systemctl restart nginx: Utilitza systemd per reiniciar el servei NGINX.
  • service nginx restart: Utilitza SysVinit per reiniciar el servei NGINX.
  • brew services restart nginx: Utilitza Homebrew per reiniciar el servei NGINX a macOS.

Recarregar NGINX

Recarregar NGINX és una manera més suau d'aplicar canvis de configuració sense aturar el servei. Sovint es prefereix en entorns de producció per evitar temps d'inactivitat.

A Linux (Systemd)

sudo systemctl reload nginx

A Linux (SysVinit)

sudo service nginx reload

A macOS (Homebrew)

nginx -s reload

Explicació

  • systemctl reload nginx: Utilitza systemd per recarregar el servei NGINX.
  • service nginx reload: Utilitza SysVinit per recarregar el servei NGINX.
  • nginx -s reload: Envia directament un senyal al procés NGINX per recarregar la configuració.

Resum

En aquesta secció, hem cobert les comandes bàsiques per iniciar, aturar, reiniciar i recarregar el servei NGINX. Aquestes operacions són essencials per gestionar el servidor web NGINX i assegurar que els canvis de configuració s'apliquin correctament. Entendre aquestes comandes t'ajudarà a mantenir un entorn de servidor web estable i responsiu.
