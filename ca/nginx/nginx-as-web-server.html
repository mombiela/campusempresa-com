Introducció a NGINX

NGINX és un servidor web potent i d'alt rendiment que també es pot utilitzar com a proxy invers, equilibrador de càrrega i memòria cau HTTP. És conegut per la seva estabilitat, conjunt ric de funcions, configuració senzilla i baix consum de recursos.

Instal·lació de NGINX

Per començar a utilitzar NGINX, primer cal instal·lar-lo al teu sistema. El procés d'instal·lació varia segons el teu sistema operatiu.

A Ubuntu/Debian

sudo apt update
sudo apt install nginx

A CentOS/RHEL

sudo yum install epel-release
sudo yum install nginx

A Windows

Per a Windows, pots descarregar els binaris de NGINX des del lloc web oficial de NGINX i seguir les instruccions d'instal·lació proporcionades.

Configuració bàsica

Un cop instal·lat, cal configurar NGINX per servir el teu contingut web.

Fitxer de configuració per defecte

El fitxer de configuració principal de NGINX normalment es troba a /etc/nginx/nginx.conf.

Exemple de configuració bàsica

A continuació es mostra un exemple senzill d'un fitxer de configuració de NGINX per servir contingut estàtic des de /var/www/html:

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • listen 80; - Indica a NGINX que escolti al port 80.
  • server_name example.com; - Especifica el nom de domini.
  • location / { ... } - Defineix com gestionar les sol·licituds a l'URL arrel.

Iniciar i aturar NGINX

Pots controlar el servei NGINX utilitzant les següents comandes:

A Ubuntu/Debian

sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx

A CentOS/RHEL

sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx

Servir contingut estàtic

NGINX sovint s'utilitza per servir contingut estàtic com fitxers HTML, CSS i JavaScript.

Exemple de configuració

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html;
    }

    location /images/ {
        root /var/www/html;
    }
}
  • El bloc location /images/ especifica que les sol·licituds a /images/ s'han de servir des de /var/www/html/images/.

Gestionar contingut dinàmic

NGINX també es pot configurar per gestionar contingut dinàmic passant les sol·licituds a un servidor d'aplicacions com PHP-FPM, Node.js o Python.

Exemple amb PHP-FPM

server {
    listen 80;
    server_name example.com;

    root /var/www/html;
    index index.php index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
    }
}
  • try_files $uri $uri/ =404; - Intenta servir el fitxer sol·licitat, si no es troba, retorna un error 404.
  • location ~ \.php$ { ... } - Gestiona fitxers PHP passant-los al servei PHP-FPM.

Millors pràctiques de seguretat

Per assegurar que el teu servidor NGINX sigui segur, segueix aquestes millors pràctiques:

  • Desactivar tokens del servidor: Evita que NGINX mostri el seu número de versió.

    server_tokens off;

  • Limitar la mida de les sol·licituds: Evita sol·licituds grans que podrien danyar el teu servidor.

    client_max_body_size 1M;

  • Utilitzar HTTPS: Utilitza sempre HTTPS per xifrar les dades entre el client i el servidor.

    server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;

    location / {
        root /var/www/html;
        index index.html;
    }
}

Conclusió

Utilitzar NGINX com a servidor web implica instal·lar el programari, configurar-lo per servir contingut estàtic i dinàmic, i aplicar les millors pràctiques de seguretat. Dominant aquests conceptes bàsics, pots aprofitar les potents funcions de NGINX per construir aplicacions web robustes i eficients.
