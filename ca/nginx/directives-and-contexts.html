Introducció

A NGINX, les directives i els contextos són conceptes fonamentals que dicten com es comporta el servidor. Les directives són instruccions que configuren el servidor, mentre que els contextos defineixen on es poden aplicar aquestes directives. Entendre aquests conceptes és crucial per configurar i gestionar un servidor NGINX de manera efectiva.

Directives

Les directives són declaracions de configuració que controlen el comportament de NGINX. Poden ser parells clau-valor simples o estructures més complexes.

  • Sintaxi: Les directives normalment segueixen el format nom_directiva valor;.
  • Tipus:
    • Directives Simples: Consisteixen en un nom i un sol valor. 
      worker_processes 4;
    • Directives de Bloc: Envolten múltiples directives dins de claus {}. 
      http {
    server {
        listen 80;
        server_name example.com;
    }
}

Contextos

Els contextos defineixen l'abast dins del qual les directives són vàlides. Són jeràrquics i es poden niar.

  • Context Principal: El context de nivell superior, que inclou configuracions globals. 
    user nginx;
worker_processes auto;
  • Context d'Esdeveniments: Defineix configuracions per a connexions de xarxa. 
    events {
    worker_connections 1024;
}
  • Context HTTP: Conté directives per gestionar el trànsit HTTP. 
    http {
    include       mime.types;
    default_type  application/octet-stream;
}
  • Context del Servidor: Defineix configuracions per a un servidor virtual específic. 
    server {
    listen 80;
    server_name example.com;
}
  • Context de Localització: Especifica com processar les sol·licituds per a URI específics. 
    location / {
    root   /usr/share/nginx/html;
    index  index.html index.htm;
}

Exemple de Configuració

Vegem un exemple complet per entendre com treballen conjuntament les directives i els contextos.

# Context Principal
user nginx;
worker_processes auto;

# Context d'Esdeveniments
events {
    worker_connections 1024;
}

# Context HTTP
http {
    include       mime.types;
    default_type  application/octet-stream;

    # Context del Servidor
    server {
        listen 80;
        server_name example.com;

        # Context de Localització
        location / {
            root   /usr/share/nginx/html;
            index  index.html index.htm;
        }

        # Un altre Context de Localització
        location /images/ {
            root /data;
        }
    }
}

Explicació

  • Context Principal: Estableix l'usuari i el nombre de processos de treball.
  • Context d'Esdeveniments: Configura el nombre màxim de connexions simultànies.
  • Context HTTP: Inclou tipus MIME i estableix un tipus de contingut per defecte.
  • Context del Servidor: Configura un servidor virtual que escolta al port 80 per a example.com.
  • Context de Localització: Defineix com servir les sol·licituds per a l'URI arrel / i l'URI /images/.

Conclusió

Entendre les directives i els contextos és essencial per configurar NGINX de manera efectiva. Les directives són els blocs de construcció que defineixen el comportament del servidor, mentre que els contextos determinen on s'apliquen aquestes directives. En dominar aquests conceptes, pots crear configuracions NGINX robustes i eficients adaptades a les teves necessitats específiques.
