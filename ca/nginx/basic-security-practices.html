Introducció

En aquesta secció, cobrirem pràctiques essencials de seguretat per configurar i gestionar NGINX. Aquestes pràctiques són crucials per protegir el teu servidor web i les aplicacions que serveix de les amenaces i vulnerabilitats comunes.

Conceptes Clau

  • Capçaleres de Seguretat: Capçaleres HTTP que ajuden a protegir contra atacs.
  • SSL/TLS: Xifrat de dades en trànsit per prevenir escoltes.
  • Control d'Accés: Restringir l'accés a certes parts del teu servidor.
  • Limitació de Ràtio: Prevenir abusos limitant el nombre de sol·licituds.
  • Registre i Monitorització: Fer un seguiment de l'activitat del servidor per a l'anàlisi de seguretat.

Capçaleres de Seguretat

Les capçaleres de seguretat poden ajudar a protegir el teu servidor de diversos tipus d'atacs, com XSS, clickjacking i sniffing MIME.

Exemple: Afegir Capçaleres de Seguretat

Afegeix les següents línies al teu fitxer de configuració de NGINX per establir capçaleres de seguretat:

server {
    listen 80;
    server_name example.com;

    add_header X-Content-Type-Options nosniff;
    add_header X-Frame-Options SAMEORIGIN;
    add_header X-XSS-Protection "1; mode=block";
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • X-Content-Type-Options: Prevé el sniffing de tipus MIME.
  • X-Frame-Options: Prevé el clickjacking.
  • X-XSS-Protection: Activa el filtratge XSS.
  • Strict-Transport-Security: Enforça HTTPS.

SSL/TLS

SSL/TLS xifra les dades entre el client i el servidor, assegurant la privacitat i la integritat de les dades.

Exemple: Activar SSL/TLS

Primer, obtén un certificat SSL d'una Autoritat de Certificació (CA) de confiança. Després, configura NGINX per utilitzar-lo:

server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • ssl_certificate: Ruta al certificat SSL.
  • ssl_certificate_key: Ruta a la clau del certificat SSL.
  • ssl_protocols: Especifica els protocols SSL/TLS a utilitzar.
  • ssl_ciphers: Defineix les suites de xifrat a utilitzar.

Control d'Accés

Restringeix l'accés a certes parts del teu servidor només a usuaris autoritzats.

Exemple: Restringir l'Accés per IP

Pots restringir l'accés a una adreça IP específica o rang:

server {
    listen 80;
    server_name example.com;

    location /admin {
        allow 192.168.1.0/24;
        deny all;
        root /var/www/html/admin;
        index index.html index.htm;
    }
}
  • allow: Permet l'accés des de l'adreça IP o rang especificat.
  • deny: Denega l'accés a totes les altres adreces IP.

Limitació de Ràtio

La limitació de ràtio ajuda a prevenir abusos limitant el nombre de sol·licituds que un client pot fer en un període de temps determinat.

Exemple: Implementar la Limitació de Ràtio

Configura la limitació de ràtio al fitxer de configuració de NGINX:

http {
    limit_req_zone $binary_remote_addr zone=mylimit:10m rate=1r/s;

    server {
        listen 80;
        server_name example.com;

        location / {
            limit_req zone=mylimit burst=5;
            root /var/www/html;
            index index.html index.htm;
        }
    }
}
  • limit_req_zone: Defineix la zona de limitació de ràtio.
  • limit_req: Aplica la limitació de ràtio a una ubicació específica.

Registre i Monitorització

El registre i la monitorització són essencials per detectar i respondre a incidents de seguretat.

Exemple: Configurar Registres d'Accés

Activa el registre d'accés a la teva configuració de NGINX:

server {
    listen 80;
    server_name example.com;

    access_log /var/log/nginx/access.log main;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • access_log: Especifica la ruta al fitxer de registre d'accés.

Conclusió

Implementant aquestes pràctiques bàsiques de seguretat, pots millorar significativament la seguretat del teu servidor NGINX. Mantén-te sempre actualitzat amb els últims pegats de seguretat i les millors pràctiques per mantenir el teu servidor segur.
