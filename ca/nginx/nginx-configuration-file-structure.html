Estructura del fitxer de configuració de NGINX
Directives i Contextos ►

Introducció

NGINX és un servidor web i servidor intermediari invers potent que s'utilitza àmpliament per la seva rendiment i escalabilitat. Entendre l'estructura del fitxer de configuració de NGINX és crucial per gestionar i optimitzar el teu servidor de manera efectiva. Aquesta secció et guiarà a través dels conceptes bàsics de l'estructura del fitxer de configuració de NGINX, des de conceptes per a principiants fins a avançats.

Conceptes Clau

  • Directives: Instruccions que defineixen el comportament de NGINX.
  • Contextos: Àmbits dins del fitxer de configuració on les directives són vàlides.
  • Blocs: Agrupacions de directives tancades entre claus {}.
  • Herència: Com es relacionen les directives en diferents contextos entre si.

Estructura Bàsica

El fitxer de configuració de NGINX es troba típicament a /etc/nginx/nginx.conf. L'estructura bàsica inclou els següents contextos:

  • Context Principal: L'àmbit global per a directives que afecten tot el servidor.
  • Context d'Esdeveniments: Defineix configuracions relacionades amb el processament de connexions.
  • Context HTTP: Conté configuracions per gestionar sol·licituds HTTP.
  • Context del Servidor: Defineix configuracions per a un servidor virtual específic.
  • Context de Localització: Especifica com processar sol·licituds per a URI específiques.

Exemple de Configuració

Aquí tens un exemple senzill d'un fitxer de configuració de NGINX:

# Context Principal
user www-data;
worker_processes auto;
pid /run/nginx.pid;

# Context d'Esdeveniments
events {
    worker_connections 1024;
}

# Context HTTP
http {
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    # Context del Servidor
    server {
        listen 80;
        server_name example.com;

        # Context de Localització
        location / {
            root /var/www/html;
            index index.html index.htm;
        }

        location /images/ {
            root /data;
        }
    }
}

Explicació Detallada

Context Principal

  • user: Defineix l'usuari i grup amb què s'executaran els processos de treball de NGINX.
  • worker_processes: Especifica el nombre de processos de treball.
  • pid: Estableix el fitxer on es guarda l'ID del procés mestre.

Context d'Esdeveniments

  • worker_connections: Limita el nombre de connexions simultànies que pot gestionar un procés de treball.

Context HTTP

  • sendfile: Activa o desactiva l'ús de sendfile() per a transferències de fitxers.
  • tcp_nopush: Controla l'ús de TCP_CORK.
  • tcp_nodelay: Controla l'ús de l'opció TCP_NODELAY.
  • keepalive_timeout: Estableix el temps d'espera per a connexions keep-alive.
  • types_hash_max_size: Estableix la mida màxima de les taules hash de tipus.
  • include: Inclou fitxers de configuració addicionals.
  • default_type: Estableix el tipus MIME per defecte per a fitxers.

Context del Servidor

  • listen: Especifica el port i l'adreça en què escoltarà el servidor.
  • server_name: Defineix els noms de domini als quals respondrà el servidor.

Context de Localització

  • root: Estableix el directori arrel per a les sol·licituds.
  • index: Especifica els fitxers d'índex per a directoris.

Conceptes Avançats

Herència

Les directives en contextos niats hereten valors dels seus contextos pares a menys que es sobreescriguin explícitament. Per exemple, un bloc location heretarà configuracions del seu bloc server pare.

Configuració Condicional

Pots utilitzar declaracions condicionals per aplicar configuracions basades en condicions específiques, com el valor d'una variable o la presència d'un fitxer.

Exemple de Configuració Condicional

server {
    listen 80;
    server_name example.com;

    location / {
        if ($request_method = POST) {
            return 405;
        }
        root /var/www/html;
        index index.html index.htm;
    }
}

Conclusió

Entendre l'estructura del fitxer de configuració de NGINX és essencial per gestionar el teu servidor web de manera efectiva. Dominant l'ús de contextos, directives i herència, pots crear configuracions potents i flexibles que s'adaptin a les teves necessitats específiques. A mesura que progressis des de conceptes bàsics fins a avançats, podràs optimitzar el teu servidor per a rendiment, seguretat i escalabilitat.
