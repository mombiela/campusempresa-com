Introducció

Servir contingut estàtic és un dels usos més comuns i fonamentals de NGINX. El contingut estàtic inclou fitxers com HTML, CSS, JavaScript, imatges i vídeos que no canvien dinàmicament. Aquesta secció et guiarà des dels conceptes bàsics fins a tècniques avançades per servir contingut estàtic utilitzant NGINX.

Conceptes Clau

  • Contingut Estàtic: Fitxers que no canvien dinàmicament i es serveixen tal com són.
  • Directiva Root: Especifica el directori arrel des del qual es serveixen els fitxers.
  • Directiva Index: Defineix el fitxer per defecte a servir quan es sol·licita un directori.
  • Bloc Location: S'utilitza per coincidir amb URL específiques i aplicar configuracions.

Configuració Bàsica

Configurar el Directori Arrel

Per servir contingut estàtic, cal especificar el directori arrel on es troben els teus fitxers. Això es fa utilitzant la directiva root dins d'un bloc server.

server {
    listen 80;
    server_name example.com;

    root /var/www/html;

    location / {
        try_files $uri $uri/ =404;
    }
}
  • listen 80;: NGINX escolta a la porta 80 per a sol·licituds HTTP.
  • server_name example.com;: Especifica el nom de domini.
  • root /var/www/html;: Estableix el directori arrel a /var/www/html.
  • location / { ... }: Coincideix amb totes les sol·licituds i intenta servir el fitxer corresponent.

Fitxer Index per Defecte

Pots especificar un fitxer per defecte a servir quan es sol·licita un directori utilitzant la directiva index.

server {
    listen 80;
    server_name example.com;

    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
  • index index.html;: Estableix index.html com el fitxer per defecte a servir.

Configuració Avançada

Servir Diferents Tipus de Contingut Estàtic

Pots crear múltiples blocs location per servir diferents tipus de contingut estàtic des de diferents directoris.

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        try_files $uri $uri/ =404;
    }

    location /images/ {
        root /var/www/media;
    }

    location /css/ {
        root /var/www/styles;
    }
}
  • location /images/ { root /var/www/media; }: Serveix imatges des de /var/www/media.
  • location /css/ { root /var/www/styles; }: Serveix fitxers CSS des de /var/www/styles.

Emmagatzematge en Memòria Cau del Contingut Estàtic

Per millorar el rendiment, pots habilitar l'emmagatzematge en memòria cau per al contingut estàtic.

server {
    listen 80;
    server_name example.com;

    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 30d;
        add_header Cache-Control "public, no-transform";
    }
}
  • location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ { ... }: Coincideix amb sol·licituds per a imatges, CSS i fitxers JavaScript.
  • expires 30d;: Estableix l'expiració de la memòria cau a 30 dies.
  • add_header Cache-Control "public, no-transform";: Afegeix una capçalera Cache-Control a la resposta.

Compressió

Pots habilitar la compressió gzip per reduir la mida dels fitxers que es serveixen.

http {
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 256;
}
  • gzip on;: Habilita la compressió gzip.
  • gzip_types ...;: Especifica els tipus MIME a comprimir.
  • gzip_min_length 256;: Només comprimeix fitxers més grans de 256 bytes.

Conclusió

Servir contingut estàtic amb NGINX és senzill però potent. Comprenent i utilitzant les directives root i index, configurant blocs location, habilitant l'emmagatzematge en memòria cau i la compressió, pots servir fitxers estàtics de manera eficient als teus usuaris. Aquestes tècniques no només milloren el rendiment, sinó que també milloren l'experiència de l'usuari en lliurar contingut de manera ràpida i fiable.
