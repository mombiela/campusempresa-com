Introducció al Proxy Invers

Un proxy invers és un servidor que es troba entre els dispositius clients i els servidors de backend, reenviant les sol·licituds dels clients al servidor de backend adequat i retornant la resposta del servidor al client. NGINX és una opció popular per configurar un proxy invers a causa del seu alt rendiment i flexibilitat.

Conceptes Clau

  • Proxy Invers: Actua com a intermediari per a les sol·licituds dels clients que busquen recursos dels servidors.
  • Servidor de Backend: El servidor que processa les sol·licituds dels clients reenviades pel proxy invers.
  • Balanceig de Càrrega: Distribuir les sol·licituds dels clients entre diversos servidors de backend per assegurar que cap servidor es sobrecarregui.
  • Terminació SSL: Desxifrar el tràfic xifrat SSL al servidor proxy abans de passar-lo al servidor de backend.

Configuració Bàsica d'un Proxy Invers

Per configurar un proxy invers bàsic amb NGINX, segueix aquests passos:

  1. Instal·la NGINX:

    sudo apt update
sudo apt install nginx

  2. Edita el Fitxer de Configuració de NGINX: Obre el fitxer de configuració per defecte situat a /etc/nginx/sites-available/default:

    sudo nano /etc/nginx/sites-available/default

  3. Configura el Proxy Invers: Afegeix la següent configuració al bloc del servidor:

    server {
    listen 80;

    server_name your_domain.com;

    location / {
        proxy_pass http://backend_server_ip:backend_server_port;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

  4. Reinicia NGINX:

    sudo systemctl restart nginx

Explicació de la Configuració

  • listen 80;: NGINX escolta al port 80 per a sol·licituds HTTP entrants.
  • server_name your_domain.com;: Especifica el nom de domini per al servidor.
  • location / { ... }: Defineix el bloc de localització per gestionar les sol·licituds.
    • proxy_pass http://backend_server_ip:backend_server_port;: Reenvia les sol·licituds al servidor de backend especificat.
    • proxy_set_header: Configura diverses capçaleres per passar juntament amb la sol·licitud al servidor de backend.

Opcions de Configuració Avançades

Balanceig de Càrrega

Per distribuir el tràfic entre diversos servidors de backend, utilitza la següent configuració:

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;
        server_name your_domain.com;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

Terminació SSL

Per gestionar la terminació SSL al servidor NGINX, utilitza la següent configuració:

server {
    listen 443 ssl;
    server_name your_domain.com;

    ssl_certificate /etc/nginx/ssl/your_domain.com.crt;
    ssl_certificate_key /etc/nginx/ssl/your_domain.com.key;

    location / {
        proxy_pass http://backend_server_ip:backend_server_port;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name your_domain.com;

    return 301 https://$host$request_uri;
}

Explicació de la Configuració SSL

  • listen 443 ssl;: NGINX escolta al port 443 per a sol·licituds HTTPS.
  • ssl_certificate i ssl_certificate_key: Rutes als fitxers del certificat SSL i la clau.
  • El segon bloc del servidor escolta al port 80 i redirigeix el tràfic HTTP a HTTPS.

Conclusió

Configurar un proxy invers amb NGINX implica configurar NGINX per reenviar les sol·licituds dels clients als servidors de backend. Aquesta configuració es pot millorar amb balanceig de càrrega i terminació SSL per millorar el rendiment i la seguretat. Comprenent i aplicant aquestes configuracions, pots gestionar efectivament el tràfic i assegurar un entorn robust de servidor web.
