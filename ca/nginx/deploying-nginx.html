Desplegar NGINX és un pas crucial en la configuració d'un servidor web o un proxy invers. Aquesta secció et guiarà a través del procés de desplegar NGINX, des de la instal·lació fins a la configuració i, finalment, executar-lo en un entorn de producció.

Instal·lació

Instal·lant NGINX en Diferents Sistemes Operatius

  • Ubuntu/Debian:

    sudo apt update
sudo apt install nginx

  • CentOS/RHEL:

    sudo yum install epel-release
sudo yum install nginx

  • Windows:

    1. Descarrega el fitxer zip de NGINX des del lloc web oficial.
    2. Extreu el fitxer zip a un directori de la teva elecció.
    3. Obre el Command Prompt i navega fins al directori extret.
    4. Executa nginx.exe per iniciar el servidor.

Verificant la Instal·lació

Després de la instal·lació, verifica que NGINX està en funcionament visitant http://localhost al teu navegador web. Hauries de veure la pàgina de benvinguda per defecte de NGINX.

Configuració Bàsica

Entenent el Fitxer de Configuració

El fitxer de configuració principal de NGINX es troba típicament a /etc/nginx/nginx.conf en sistemes Linux. Aquí tens un exemple bàsic de com podria ser aquest fitxer:

worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    sendfile        on;
    keepalive_timeout  65;

    server {
        listen       80;
        server_name  localhost;

        location / {
            root   /usr/share/nginx/html;
            index  index.html index.htm;
        }

        error_page  500 502 503 504  /50x.html;
        location = /50x.html {
            root   /usr/share/nginx/html;
        }
    }
}

Conceptes Clau

  • worker_processes: Defineix el nombre de processos de treball.
  • events: Conté directives que afecten el processament de connexions.
  • http: Context principal per gestionar el tràfic HTTP.
  • server: Defineix un servidor virtual.
  • location: Especifica com processar les sol·licituds per a una ubicació específica.

Configuració Avançada

Configurant un Proxy Invers

Es pot configurar un proxy invers per reenviar les sol·licituds dels clients a un altre servidor. Aquí tens un exemple de configuració:

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend_server;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Balanceig de Càrrega

NGINX també es pot utilitzar per al balanceig de càrrega. Aquí tens un exemple de configuració:

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
    }

    server {
        listen 80;
        server_name example.com;

        location / {
            proxy_pass http://backend;
        }
    }
}

Configuració SSL/TLS

Per assegurar el teu servidor NGINX amb SSL/TLS, necessites obtenir un certificat i configurar NGINX per utilitzar-lo:

server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/nginx.crt;
    ssl_certificate_key /etc/nginx/ssl/nginx.key;

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
    }
}

Executant NGINX en Producció

Iniciant i Aturant NGINX

  • Iniciar NGINX:

    sudo systemctl start nginx

  • Aturar NGINX:

    sudo systemctl stop nginx

  • Reiniciar NGINX:

    sudo systemctl restart nginx

Activant NGINX per Iniciar-se en l'Arrencada

  • Activar:

    sudo systemctl enable nginx

  • Desactivar:

    sudo systemctl disable nginx

Conclusió

Desplegar NGINX implica diversos passos, des de la instal·lació fins a la configuració i executar-lo en un entorn de producció. Entenent les configuracions bàsiques i avançades, pots utilitzar efectivament NGINX com a servidor web, proxy invers o balancejador de càrrega. Practica aquests passos i configuracions per convertir-te en un expert en desplegar NGINX en diverses situacions.
