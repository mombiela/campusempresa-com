Les pàgines d'error personalitzades són una part essencial del desenvolupament web, proporcionant una millor experiència d'usuari quan alguna cosa va malament. En aquesta secció, explorarem com configurar pàgines d'error personalitzades a NGINX, des de configuracions bàsiques fins a configuracions més avançades.

Introducció a les Pàgines d'Error Personalitzades

Les pàgines d'error personalitzades permeten:

  • Proporcionar un missatge amigable per a l'usuari quan es produeix un error.
  • Mantenir una aparença i sensació consistents a tot el vostre lloc web.
  • Oferir informació útil o opcions de navegació als usuaris.

Configuració Bàsica

Per començar amb les pàgines d'error personalitzades a NGINX, heu de modificar el fitxer de configuració de NGINX. Els errors més comuns a gestionar són el 404 (No Trobat) i el 500 (Error Intern del Servidor).

Exemple: Pàgina d'Error 404 Bàsica

  1. Crea una Pàgina d'Error Personalitzada: Crea un fitxer HTML per a la teva pàgina d'error personalitzada, per exemple, 404.html.
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Pàgina No Trobada</title>
</head>
<body>
    <h1>404 - Pàgina No Trobada</h1>
    <p>Ho sentim, la pàgina que estàs buscant no existeix.</p>
</body>
</html>
  1. Modifica la Configuració de NGINX: Edita el teu fitxer de configuració de NGINX (normalment situat a /etc/nginx/nginx.conf o /etc/nginx/sites-available/default).
server {
    listen 80;
    server_name tudomini.com;

    location / {
        try_files $uri $uri/ =404;
    }

    error_page 404 /404.html;
    location = /404.html {
        root /usr/share/nginx/html;
        internal;
    }
}
  • error_page 404 /404.html;: Aquesta directiva indica a NGINX que utilitzi 404.html per als errors 404.
  • location = /404.html { ... }: Aquest bloc especifica on es troba el fitxer 404.html i el fa intern, de manera que no pot ser accedit directament pels usuaris.

Gestió de Múltiples Codis d'Error

Pots gestionar múltiples codis d'error especificant-los a la directiva error_page.

Exemple: Gestió d'Errors 404 i 500

  1. Crea Pàgines d'Error Personalitzades: Crea 404.html i 500.html.
<!-- 404.html -->
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Pàgina No Trobada</title>
</head>
<body>
    <h1>404 - Pàgina No Trobada</h1>
    <p>Ho sentim, la pàgina que estàs buscant no existeix.</p>
</body>
</html>
<!-- 500.html -->
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Error Intern del Servidor</title>
</head>
<body>
    <h1>500 - Error Intern del Servidor</h1>
    <p>Ho sentim, alguna cosa ha anat malament al nostre costat.</p>
</body>
</html>
  1. Modifica la Configuració de NGINX:
server {
    listen 80;
    server_name tudomini.com;

    location / {
        try_files $uri $uri/ =404;
    }

    error_page 404 /404.html;
    error_page 500 502 503 504 /500.html;

    location = /404.html {
        root /usr/share/nginx/html;
        internal;
    }

    location = /500.html {
        root /usr/share/nginx/html;
        internal;
    }
}
  • error_page 500 502 503 504 /500.html;: Aquesta directiva gestiona múltiples codis d'error del servidor amb una sola pàgina d'error personalitzada.

Pàgines d'Error Personalitzades Avançades

Per a configuracions més avançades, potser voldràs incloure contingut dinàmic o utilitzar scripts del servidor.

Exemple: Utilització d'un Script PHP per a Pàgines d'Error Personalitzades

  1. Crea un Script PHP: Crea error.php per gestionar errors de manera dinàmica.
<?php
$error_code = $_GET['code'];
?>
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Error <?php echo $error_code; ?></title>
</head>
<body>
    <h1>Error <?php echo $error_code; ?></h1>
    <p>Ho sentim, s'ha produït un error: <?php echo $error_code; ?></p>
</body>
</html>
  1. Modifica la Configuració de NGINX:
server {
    listen 80;
    server_name tudomini.com;

    location / {
        try_files $uri $uri/ =404;
    }

    error_page 404 = /error.php?code=404;
    error_page 500 502 503 504 = /error.php?code=500;

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
    }
}
  • error_page 404 = /error.php?code=404;: Aquesta directiva utilitza un script PHP per gestionar l'error, passant el codi d'error com a paràmetre.

Conclusió

Les pàgines d'error personalitzades a NGINX milloren l'experiència d'usuari proporcionant missatges d'error informatius i estèticament agradables. Seguint els passos descrits en aquesta secció, pots crear pàgines d'error personalitzades des de bàsiques fins a avançades adaptades a les necessitats del teu lloc web. Recorda provar les teves configuracions a fons per assegurar-te que funcionen com s'espera.
