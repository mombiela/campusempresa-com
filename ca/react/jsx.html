Introducció al JSX

JSX significa JavaScript XML. És una extensió de sintaxi per a JavaScript que s'assembla a XML o HTML. JSX s'utilitza amb React per descriure com hauria de ser la interfície d'usuari. Pot recordar-te un llenguatge de plantilles, però ve amb tot el poder de JavaScript.

  • Sintaxi JSX: Combina el poder de JavaScript amb una sintaxi semblant a HTML.
  • Integració amb React: JSX s'utilitza per crear elements de React.
  • Compilació: JSX es compila a JavaScript amb eines com Babel.

Per què utilitzar JSX?

JSX facilita escriure i afegir HTML en React. També ajuda a mantenir el codi llegible i mantenible.

  • Llegibilitat: La sintaxi JSX és més llegible i fàcil d'entendre.
  • Mantenibilitat: Més fàcil de mantenir i depurar gràcies a la seva estructura.
  • Poder de JavaScript: Pots utilitzar expressions de JavaScript dins de JSX.

Sintaxi bàsica de JSX

JSX et permet escriure elements HTML en JavaScript i col·locar-los en el DOM sense utilitzar funcions com createElement() o appendChild().

Exemple

const element = <h1>Hello, world!</h1>;

Explicació

  • Element: La variable element conté un element JSX.
  • Sintaxi semblant a HTML: La sintaxi s'assembla a HTML però està escrita en JavaScript.

Inserir expressions en JSX

Pots inserir qualsevol expressió de JavaScript en JSX embolicant-la en claus {}.

Exemple

const name = 'John';
const element = <h1>Hello, {name}!</h1>;

Explicació

  • Expressió de JavaScript: {name} és una expressió de JavaScript inserida en JSX.
  • Contingut dinàmic: El contingut dins de les claus es valora i es renderitza dinàmicament.

JSX també és una expressió

Després de la compilació, les expressions JSX es converteixen en crides de funcions de JavaScript regulars i es valoren com a objectes de JavaScript.

Exemple

const element = (
  <h1 className="greeting">
    Hello, world!
  </h1>
);

Explicació

  • Atribut de classe: className s'utilitza en lloc de class perquè class és una paraula reservada en JavaScript.
  • Elements niats: JSX pot contenir elements niats.

Especificar atributs amb JSX

Pots utilitzar cometes per especificar literals de cadena com a atributs i claus per inserir una expressió de JavaScript.

Exemple

const element = <img src={user.avatarUrl} alt="User Avatar" />;

Explicació

  • Literal de cadena: alt="User Avatar" és un literal de cadena.
  • Expressió de JavaScript: src={user.avatarUrl} és una expressió de JavaScript.

JSX representa objectes

Babel compila JSX a crides de React.createElement().

Exemple

const element = (
  <h1 className="greeting">
    Hello, world!
  </h1>
);

Versió compilada

const element = React.createElement(
  'h1',
  { className: 'greeting' },
  'Hello, world!'
);

Explicació

  • React.createElement(): El JSX es compila a crides de React.createElement().
  • Representació d'objectes: L'element és un objecte de JavaScript que descriu el node del DOM.

Millors pràctiques per a JSX

  • Etiquetes auto-tancades: Utilitza etiquetes auto-tancades per a elements sense fills.
  • Nomenclatura de propietats en camelCase: Utilitza camelCase per nomenar propietats en JSX.
  • Fragments: Utilitza fragments de React per agrupar múltiples elements sense afegir nodes extra al DOM.

Exemple

const element = (
  <>
    <h1>Hello, world!</h1>
    <p>Welcome to learning JSX.</p>
  </>
);

Explicació

  • Fragment: <> i </> s'utilitzen per agrupar elements sense afegir nodes extra.

Conclusió

JSX és una potent extensió de sintaxi per a JavaScript que et permet escriure codi semblant a HTML dins de JavaScript. Millora la llegibilitat i mantenibilitat del teu codi mentre aprofita tot el poder de JavaScript. Entendre JSX és crucial per treballar eficaçment amb React, ja que forma la base de com descrius i estructures els teus components d'interfície d'usuari.
