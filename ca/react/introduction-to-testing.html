Introducció a les Proves
Proves Unitàries amb Jest ►

Les proves són una part crucial del desenvolupament de programari, assegurant que la teva aplicació funcioni com s'espera i estigui lliure d'errors. En aquesta secció, et presentarem les proves en el context de les aplicacions React, cobrint els conceptes bàsics i avançant gradualment cap a conceptes més avançats.

Per què són importants les proves

  • Fiabilitat: Assegura que la teva aplicació es comporti com s'espera.
  • Mantenibilitat: Facilita la refactorització del codi sense trencar la funcionalitat.
  • Documentació: Serveix com a documentació de com ha de funcionar el teu codi.
  • Confiança: Proporciona confiança als desenvolupadors i parts interessades que l'aplicació és estable.

Tipus de proves

  • Proves unitàries: Prova components o funcions individuals en aïllament.
  • Proves d'integració: Prova com funcionen junts diferents parts de l'aplicació.
  • Proves de punta a punta (E2E): Prova tota l'aplicació des de la perspectiva de l'usuari.

Eines de proves per a React

  • Jest: Un framework de proves de JavaScript mantingut per Facebook, comunament utilitzat amb React.
  • React Testing Library: Una biblioteca per provar components React, centrada en provar interaccions d'usuari.
  • Cypress: Un framework de proves de punta a punta.

Configuració de l'entorn de proves

Per començar amb les proves en una aplicació React, necessites configurar el teu entorn de proves. La majoria de projectes React utilitzen Jest i React Testing Library.

Instal·lació de Jest i React Testing Library

Primer, assegura't de tenir un projecte React configurat. Després, instal·la els paquets necessaris:

npm install --save-dev jest @testing-library/react @testing-library/jest-dom

Configuració de Jest

Jest normalment requereix una configuració mínima. Crea un fitxer jest.config.js a l'arrel del teu projecte:

module.exports = {
  setupFilesAfterEnv: ['<rootDir>/setupTests.js'],
  testEnvironment: 'jsdom',
};

Crea un fitxer setupTests.js per configurar React Testing Library:

import '@testing-library/jest-dom/extend-expect';

Escriure la teva primera prova

Escrivim una prova simple per a un component React.

Crear un component simple

Crea un component HelloWorld.js:

import React from 'react';

const HelloWorld = () => {
  return <h1>Hello, World!</h1>;
};

export default HelloWorld;

Escriure una prova per al component

Crea un fitxer HelloWorld.test.js:

import React from 'react';
import { render, screen } from '@testing-library/react';
import HelloWorld from './HelloWorld';

test('renders Hello, World! text', () => {
  render(<HelloWorld />);
  const helloWorldElement = screen.getByText(/Hello, World!/i);
  expect(helloWorldElement).toBeInTheDocument();
});

Explicació

  • render: Renderitza el component en un DOM virtual per a proves.
  • screen: Proporciona utilitats per consultar el DOM renderitzat.
  • getByText: Troba un element pel seu contingut de text.
  • expect: Fa afirmacions sobre els resultats de la prova.
  • toBeInTheDocument: Comprova si l'element està present en el DOM.

Conceptes avançats de proves

Simulació de funcions i mòduls

De vegades, necessites simular funcions o mòduls per aïllar el component que s'està provant.

Simulació d'una funció

const mockFunction = jest.fn();

mockFunction();
expect(mockFunction).toHaveBeenCalled();

Simulació d'un mòdul

jest.mock('axios');

import axios from 'axios';
import { fetchData } from './api';

test('fetches data from API', async () => {
  axios.get.mockResolvedValue({ data: 'mockData' });
  const data = await fetchData();
  expect(data).toBe('mockData');
});

Proves de snapshot

Les proves de snapshot capturen la sortida renderitzada d'un component i la comparen amb una snapshot guardada.

Crear una prova de snapshot

import React from 'react';
import renderer from 'react-test-renderer';
import HelloWorld from './HelloWorld';

test('matches snapshot', () => {
  const tree = renderer.create(<HelloWorld />).toJSON();
  expect(tree).toMatchSnapshot();
});

Conclusió

Les proves són una pràctica essencial en el desenvolupament de React, assegurant que la teva aplicació sigui fiable i mantenible. Entenent els conceptes bàsics de les proves unitàries, d'integració i de punta a punta, i utilitzant eines com Jest i React Testing Library, pots escriure proves efectives per als teus components React. A mesura que avancis, aprendràs a simular funcions i mòduls, i a utilitzar proves de snapshot per millorar encara més la cobertura de les teves proves. Bones proves!
