Introducció

En aquesta secció, cobrirem les millors pràctiques i estàndards de codificació per desenvolupar aplicacions React. Adherir-se a aquestes pràctiques assegura que el teu codi sigui net, mantenible i escalable.

Millors Pràctiques Generals

Utilitza Components Funcionals

  • Per què: Els components funcionals són més simples i fàcils de llegir.
  • Exemple: 
    // Component Funcional
const Greeting = ({ name }) => <h1>Hello, {name}!</h1>;

// Component de Classe
class Greeting extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

Utilitza Hooks per a l'Estat i els Efectes Secundaris

  • Per què: Els hooks et permeten utilitzar l'estat i altres característiques de React sense escriure una classe.
  • Exemple: 
    import React, { useState, useEffect } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `Count: ${count}`;
  }, [count]);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
};

Mantingues els Components Petits i Enfocats

  • Per què: Els components més petits són més fàcils de provar i reutilitzar.
  • Exemple: 
    // Bé: Component petit i enfocat
const Button = ({ onClick, label }) => (
  <button onClick={onClick}>{label}</button>
);

// Malament: Component gran i desenfocat
const App = () => (
  <div>
    <button onClick={() => alert('Clicked!')}>Click me</button>
    <p>Some unrelated content</p>
  </div>
);

Estàndards de Codificació

Convencions de Nomenclatura

  • Components: Utilitza PascalCase. 
    // Correcte
const MyComponent = () => <div>My Component</div>;

// Incorrecte
const myComponent = () => <div>My Component</div>;
  • Props i Estat: Utilitza camelCase. 
    const MyComponent = ({ userName }) => <div>{userName}</div>;

Estructura de Fitxers

  • Per què: Una estructura de fitxers ben organitzada fa que el teu projecte sigui més fàcil de navegar.
  • Exemple: 
    src/
├── components/
│   ├── Button.js
│   ├── Header.js
├── pages/
│   ├── HomePage.js
│   ├── AboutPage.js
├── App.js
├── index.js

Format Consistent

  • Per què: Un format consistent millora la llegibilitat.
  • Eines: Utilitza Prettier o ESLint. 
    // .prettierrc
{
  "singleQuote": true,
  "trailingComma": "es5"
}

Evita els Estils en Línia

  • Per què: Els estils en línia poden fer que els teus components siguin més difícils de mantenir.
  • Exemple: 
    // Bé: Utilitzant classes CSS
const MyComponent = () => <div className="my-component">Content</div>;

// Malament: Utilitzant estils en línia
const MyComponent = () => <div style={{ color: 'red' }}>Content</div>;

Optimització del Rendiment

Utilitza React.memo

  • Per què: Prevé renders innecessaris.
  • Exemple: 
    import React from 'react';

const MyComponent = React.memo(({ value }) => {
  return <div>{value}</div>;
});

Utilitza useCallback i useMemo

  • Per què: Memoitza funcions i valors per evitar càlculs innecessaris.
  • Exemple: 
    import React, { useCallback, useMemo } from 'react';

const MyComponent = ({ items }) => {
  const computeExpensiveValue = useCallback(() => {
    // càlcul costós
  }, [items]);

  const memoizedValue = useMemo(() => computeExpensiveValue(), [computeExpensiveValue]);

  return <div>{memoizedValue}</div>;
};

Conclusió

Seguint aquestes millors pràctiques i estàndards de codificació, pots crear aplicacions React que no només siguin funcionals sinó també netes, mantenibles i eficients. La consistència en la teva base de codi farà que sigui més fàcil treballar en projectes col·laborativament i assegurarà l'escalabilitat a llarg termini.
