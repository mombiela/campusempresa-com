Introducció a les Aplicacions Web Progressives (PWA)

  • Definició: Les Aplicacions Web Progressives són aplicacions web que utilitzen capacitats web modernes per oferir una experiència semblant a una aplicació als usuaris.
  • Característiques Clau:
    • Responsives: S'adapten a qualsevol factor de forma (escriptori, mòbil, tauleta).
    • Capacitats Offline: Funcionen sense connexió o amb condicions de xarxa pobres.
    • Com una Aplicació: Se sent com una aplicació nativa amb interaccions i navegació d'estil aplicació.
    • Instal·lables: Es poden instal·lar a la pantalla d'inici de l'usuari sense una botiga d'aplicacions.
    • Segures: Servides via HTTPS per prevenir espionatge i assegurar la integritat del contingut.

Configuració d'un Projecte React per a PWA

  • Create React App: La manera més fàcil de configurar un projecte React amb capacitats PWA. 
    npx create-react-app my-pwa-app --template cra-template-pwa
cd my-pwa-app
npm start
  • Explicació:
    • npx create-react-app my-pwa-app --template cra-template-pwa: Aquest comandament configura un nou projecte React amb configuració PWA.
    • cd my-pwa-app: Navega al directori del projecte.
    • npm start: Inicia el servidor de desenvolupament.

Service Workers

  • Què és un Service Worker?
    • Un script que el navegador executa en segon pla, separat d'una pàgina web, permetent funcions com capacitats offline i notificacions push.
  • Registrar un Service Worker en React: 
    // src/index.js
import React from 'react';
import ReactDOM from 'react-dom';
import './index.css';
import App from './App';
import * as serviceWorkerRegistration from './serviceWorkerRegistration';

ReactDOM.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
  document.getElementById('root')
);

// Registrar el service worker
serviceWorkerRegistration.register();
  • Explicació:
    • La funció serviceWorkerRegistration.register() registra el service worker, permetent capacitats offline.

Estratègies de Caché

  • Tipus d'Estratègies de Caché:
    • Cache First: Intenta servir contingut des de la caché primer, recorrent a la xarxa si no està disponible.
    • Network First: Intenta obtenir contingut de la xarxa primer, recorrent a la caché si la xarxa no està disponible.
    • Stale-While-Revalidate: Serveix contingut des de la caché mentre simultàniament obté una actualització de la xarxa.
  • Implementació de l'Estratègia Cache First: 
    // src/service-worker.js
const CACHE_NAME = 'my-pwa-cache-v1';
const urlsToCache = [
  '/',
  '/index.html',
  '/static/js/bundle.js',
  '/static/css/main.css',
  // Afegeix altres recursos que vulguis cachejar
];

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then((cache) => {
        return cache.addAll(urlsToCache);
      })
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then((response) => {
        if (response) {
          return response; // Cache hit - retorna la resposta
        }
        return fetch(event.request); // Cache miss - obté de la xarxa
      })
  );
});
  • Explicació:
    • Esdeveniment d'Instal·lació: Cacheja els recursos especificats durant la instal·lació del service worker.
    • Esdeveniment de Fetch: Intercepta les sol·licituds de xarxa i serveix recursos cachejats si estan disponibles.

Afegir un Manifest d'Aplicació Web

  • Què és un Manifest d'Aplicació Web?
    • Un fitxer JSON que proporciona metadades sobre l'aplicació web, com ara nom, icones i colors de tema.
  • Creació d'un Fitxer de Manifest: 
    // public/manifest.json
{
  "short_name": "PWA",
  "name": "My Progressive Web App",
  "icons": [
    {
      "src": "icon-192x192.png",
      "type": "image/png",
      "sizes": "192x192"
    },
    {
      "src": "icon-512x512.png",
      "type": "image/png",
      "sizes": "512x512"
    }
  ],
  "start_url": ".",
  "display": "standalone",
  "theme_color": "#000000",
  "background_color": "#ffffff"
}
  • Explicació:
    • short_name: Nom curt de l'aplicació.
    • name: Nom complet de l'aplicació.
    • icons: Array d'objectes d'icones per a diferents mides.
    • start_url: URL per iniciar l'aplicació.
    • display: Com s'ha de mostrar l'aplicació (standalone, pantalla completa, etc.).
    • theme_color: Color de tema de l'aplicació.
    • background_color: Color de fons de l'aplicació.

Prova i Depuració de les PWA

  • Prova de Capacitats Offline:
    • Utilitza les DevTools de Chrome per simular el mode offline.
    • Navega a la pestanya "Application", després "Service Workers", i marca "Offline".
  • Depuració de Service Workers:
    • Utilitza les pestanyes "Console" i "Network" a les DevTools de Chrome per depurar problemes de service worker.
    • Busca errors i sol·licituds de xarxa per assegurar-te que les estratègies de caché funcionen com s'espera.

Conclusió

Les Aplicacions Web Progressives (PWA) milloren l'experiència de l'usuari combinant el millor de les aplicacions web i mòbils. Amb React, crear una PWA és senzill gràcies a eines com Create React App. Entenent i implementant service workers, estratègies de caché i manifests d'aplicació web, els desenvolupadors poden construir aplicacions robustes, amb capacitats offline que proporcionen una experiència d'usuari fluida a través de diferents dispositius i condicions de xarxa.
