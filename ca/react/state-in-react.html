Introducció a l'Estat

L'estat és un concepte fonamental a React que permet als components crear i gestionar les seves pròpies dades. Entendre l'estat és crucial per construir aplicacions dinàmiques i interactives.

  • Estat: Un objecte que conté informació que pot canviar al llarg de la vida del component.
  • setState: Un mètode per actualitzar l'estat i tornar a renderitzar el component.

Ús de l'Estat en Components Funcionals

Amb la introducció dels React Hooks a la versió 16.8, gestionar l'estat en components funcionals s'ha tornat senzill.

Hook useState

El hook useState s'utilitza per afegir estat als components funcionals.

import React, { useState } from 'react';

function Counter() {
  // Declara una variable d'estat anomenada "count" amb un valor inicial de 0
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Has clicat {count} vegades</p>
      <button onClick={() => setCount(count + 1)}>
        Clica'm
      </button>
    </div>
  );
}

export default Counter;
  • useState: Inicialitza l'estat i retorna una matriu amb l'estat actual i una funció per actualitzar-lo.
  • count: El valor actual de l'estat.
  • setCount: Funció per actualitzar l'estat.

Ús de l'Estat en Components de Classe

Abans dels hooks, l'estat es gestionava en components de classe utilitzant this.state i this.setState.

import React, { Component } from 'react';

class Counter extends Component {
  constructor(props) {
    super(props);
    // Inicialitza l'estat
    this.state = { count: 0 };
  }

  render() {
    return (
      <div>
        <p>Has clicat {this.state.count} vegades</p>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          Clica'm
        </button>
      </div>
    );
  }
}

export default Counter;
  • this.state: Inicialitza l'estat al constructor.
  • this.setState: Mètode per actualitzar l'estat i desencadenar una nova renderització.

Estat vs. Props

Entendre la diferència entre estat i props és essencial.

| Característica | Estat | Props | |----------------|-------|-------| | Definició | Conté dades que poden canviar | Conté dades passades des del component pare | | Mutabilitat | Mutable | Immutable | | Ús | Gestionat dins del component | Passat al component |

Millors Pràctiques per Utilitzar l'Estat

  • Mantingues l'estat mínim: Només emmagatzema el que és necessari.
  • Eleva l'estat: Si múltiples components necessiten el mateix estat, eleva'l al seu ancestre comú.
  • Evita la mutació directa de l'estat: Sempre utilitza setState o la funció d'actualització de useState.

Patrons Comuns

Elevar l'Estat

Quan múltiples components necessiten compartir l'estat, eleva l'estat al seu ancestre comú més proper.

import React, { useState } from 'react';

function Parent() {
  const [sharedState, setSharedState] = useState(0);

  return (
    <div>
      <Child1 sharedState={sharedState} />
      <Child2 setSharedState={setSharedState} />
    </div>
  );
}

function Child1({ sharedState }) {
  return <p>Estat Compartit: {sharedState}</p>;
}

function Child2({ setSharedState }) {
  return (
    <button onClick={() => setSharedState(prevState => prevState + 1)}>
      Incrementa l'Estat Compartit
    </button>
  );
}

export default Parent;

Renderització Condicional Basada en l'Estat

Renderitza components o elements de manera condicional basada en l'estat.

import React, { useState } from 'react';

function Toggle() {
  const [isToggled, setIsToggled] = useState(false);

  return (
    <div>
      <button onClick={() => setIsToggled(!isToggled)}>
        {isToggled ? 'Amaga' : 'Mostra'} Missatge
      </button>
      {isToggled && <p>Aquest és un missatge alternat!</p>}
    </div>
  );
}

export default Toggle;

Conclusió

La gestió de l'estat és un aspecte clau de React que permet que els components siguin dinàmics i interactius. Ja sigui utilitzant components funcionals amb hooks o components de classe, entendre com gestionar i actualitzar correctament l'estat és essencial per construir aplicacions robustes amb React. Recorda mantenir l'estat mínim, elevar l'estat quan sigui necessari i evitar mutacions directes per una experiència de desenvolupament més fluida.
