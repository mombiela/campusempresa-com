Introducció als Components d'Ordre Superior

Els Components d'Ordre Superior (HOCs) són una tècnica avançada a React per reutilitzar la lògica dels components. No formen part de l'API de React, sinó que són un patró que emergeix de la naturalesa composicional de React.

Què és un Component d'Ordre Superior?

  • Un Component d'Ordre Superior és una funció que pren un component i retorna un nou component.
  • Els HOCs s'utilitzen per afegir funcionalitat addicional als components existents sense modificar-los directament.
const withExtraInfo = (WrappedComponent) => {
  return class extends React.Component {
    render() {
      return <WrappedComponent extraInfo="This is extra info" {...this.props} />;
    }
  };
};

En aquest exemple, withExtraInfo és un Component d'Ordre Superior que afegeix una prop extraInfo al WrappedComponent.

Per què utilitzar Components d'Ordre Superior?

  • Reutilització de Codi: Els HOCs permeten reutilitzar la lògica dels components a través de múltiples components.
  • Separació de Responsabilitats: Ajuden a separar les responsabilitats encapsulant la lògica en un sol lloc.
  • Components Millorats: Els HOCs poden millorar els components afegint props, estat o comportament addicional.

Crear un Component d'Ordre Superior

Estructura Bàsica

Per crear un HOC, necessites:

  1. Definir una funció que prengui un component com a argument.
  2. Retornar un nou component que renderitzi el component original amb props o comportament addicional.
const withLoading = (WrappedComponent) => {
  return class extends React.Component {
    render() {
      if (this.props.isLoading) {
        return <div>Loading...</div>;
      }
      return <WrappedComponent {...this.props} />;
    }
  };
};

Exemple: Afegir Estat de Càrrega

Creem un HOC que afegeix un estat de càrrega a qualsevol component.

const withLoading = (WrappedComponent) => {
  return class extends React.Component {
    render() {
      if (this.props.isLoading) {
        return <div>Loading...</div>;
      }
      return <WrappedComponent {...this.props} />;
    }
  };
};

// Ús
const DataComponent = (props) => <div>Data: {props.data}</div>;

const DataComponentWithLoading = withLoading(DataComponent);

// En el teu mètode render
<DataComponentWithLoading isLoading={true} data="Some data" />

En aquest exemple, DataComponentWithLoading mostrarà "Loading..." quan isLoading sigui true i les dades reals quan isLoading sigui false.

Usos Comuns

  • Autenticació: Envoltar components per comprovar si un usuari està autenticat.
  • Registre: Afegir funcionalitat de registre als components.
  • Obtenció de Dades: Gestionar l'obtenció de dades i passar les dades com a props al component embolicat.

Millors Pràctiques

  • No Mutar el Component Original: Sempre retorna un nou component.
  • Passar Totes les Props: Assegura't que totes les props es passen al component embolicat.
  • Utilitzar Noms de Visualització: Estableix un nom de visualització per al HOC per facilitar el depuratge.
const withExtraInfo = (WrappedComponent) => {
  class WithExtraInfo extends React.Component {
    render() {
      return <WrappedComponent extraInfo="This is extra info" {...this.props} />;
    }
  }
  WithExtraInfo.displayName = `WithExtraInfo(${getDisplayName(WrappedComponent)})`;
  return WithExtraInfo;
};

const getDisplayName = (WrappedComponent) => {
  return WrappedComponent.displayName || WrappedComponent.name || 'Component';
};

Conclusió

Els Components d'Ordre Superior són un patró poderós a React que permeten reutilitzar la lògica i millorar els components sense modificar-los directament. Entenent i utilitzant els HOCs, pots crear codi més modular, mantenible i reutilitzable.

  • Punts Clau:
    • Els HOCs són funcions que prenen un component i retornen un nou component.
    • Ajuden a reutilitzar codi i separar responsabilitats.
    • Sempre passa totes les props i evita mutar el component original.

Dominant els HOCs, pots millorar significativament l'escalabilitat i mantenibilitat de les teves aplicacions React.
