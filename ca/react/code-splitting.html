La divisió de codi és una característica poderosa en React que et permet dividir el teu codi en trossos més petits, que es poden carregar sota demanda. Això pot millorar significativament el rendiment de la teva aplicació reduint el temps de càrrega inicial.

Què és la Divisió de Codi?

  • Definició: La divisió de codi és el procés de descompondre el codi de la teva aplicació en peces més petites i manejables que es poden carregar de manera asíncrona.
  • Propòsit: Ajuda a optimitzar el rendiment de la teva aplicació React carregant només el codi necessari quan es necessita.
  • Beneficis:
    • Redueix el temps de càrrega inicial.
    • Millora l'experiència de l'usuari carregant parts de l'aplicació sota demanda.
    • Ajuda a una millor gestió de grans bases de codi.

Com Implementar la Divisió de Codi en React

Utilitzant React.lazy i Suspense

React proporciona suport integrat per a la divisió de codi a través de React.lazy i Suspense.

React.lazy

  • Propòsit: Et permet renderitzar una importació dinàmica com un component regular.
  • Sintaxi: 
    const OtherComponent = React.lazy(() => import('./OtherComponent'));

Suspense

  • Propòsit: Et permet especificar un estat de càrrega mentre el component lazy s'està carregant.
  • Sintaxi: 
    <Suspense fallback={<div>Loading...</div>}>
  <OtherComponent />
</Suspense>

Exemple

import React, { Suspense } from 'react';

const OtherComponent = React.lazy(() => import('./OtherComponent'));

function App() {
  return (
    <div>
      <h1>My App</h1>
      <Suspense fallback={<div>Loading...</div>}>
        <OtherComponent />
      </Suspense>
    </div>
  );
}

export default App;

Importacions Dinàmiques amb Webpack

Webpack és un empaquetador de mòduls popular que suporta importacions dinàmiques, que es poden utilitzar per a la divisió de codi.

Exemple

import('./math').then(math => {
  console.log(math.add(16, 26));
});
  • Explicació: La funció import() retorna una promesa que es resol al mòdul. Això et permet carregar el mòdul només quan es necessita.

Tècniques Avançades de Divisió de Codi

Divisió de Codi Basada en Rutes

  • Propòsit: Dividir el codi basat en diferents rutes de la teva aplicació.
  • Eines: React Router es pot utilitzar en combinació amb React.lazy i Suspense.

Exemple

import React, { Suspense } from 'react';
import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';

const Home = React.lazy(() => import('./Home'));
const About = React.lazy(() => import('./About'));

function App() {
  return (
    <Router>
      <Suspense fallback={<div>Loading...</div>}>
        <Switch>
          <Route exact path="/" component={Home} />
          <Route path="/about" component={About} />
        </Switch>
      </Suspense>
    </Router>
  );
}

export default App;

Divisió de Codi a Nivell de Component

  • Propòsit: Dividir el codi a nivell de component per carregar components només quan es necessiten.
  • Exemple: Similar a l'exemple bàsic proporcionat anteriorment, però aplicat a components individuals dins d'una aplicació més gran.

Millors Pràctiques per a la Divisió de Codi

  • Identificar Camins Crítics: Centra't en dividir el codi en parts de l'aplicació que no són crítiques per a la càrrega inicial.
  • Equilibri: Evita dividir en excés, el que pot portar a massa trossos petits i a un augment de les sol·licituds HTTP.
  • Monitoritzar el Rendiment: Utilitza eines com Lighthouse i Webpack Bundle Analyzer per monitoritzar i optimitzar el rendiment de la teva aplicació.

Conclusió

La divisió de codi és una tècnica essencial per optimitzar el rendiment de les aplicacions React. Utilitzant React.lazy i Suspense, juntament amb importacions dinàmiques i la divisió basada en rutes, pots millorar significativament l'experiència de l'usuari reduint els temps de càrrega inicials i carregant el codi sota demanda. Recorda seguir les millors pràctiques per assegurar-te que la teva estratègia de divisió de codi sigui efectiva i eficient.
