Introducció a la renderització del costat del servidor (SSR)

La renderització del costat del servidor (SSR) és una tècnica utilitzada per renderitzar pàgines web al servidor en lloc del client. Això pot millorar el rendiment i el SEO en lliurar HTML completament renderitzat al client.

Per què utilitzar SSR?

  • Rendiment millorat: Els temps de càrrega inicials de la pàgina poden ser més ràpids perquè el servidor envia una pàgina completament renderitzada.
  • Millor SEO: Els motors de cerca poden rastrejar HTML completament renderitzat, millorant la visibilitat del teu lloc.
  • Experiència d'usuari millorada: Els usuaris veuen el contingut més ràpidament, cosa que condueix a una millor experiència general.

Introducció a Next.js

Next.js és un framework de React que permet diverses funcions avançades, incloent SSR, generació de llocs estàtics i més.

Funcions clau de Next.js

  • Enrutament basat en fitxers: Simplifica la creació de rutes.
  • Rutes d'API: Et permet crear punts finals d'API dins del mateix projecte.
  • Suport integrat per a CSS i Sass: Facilita l'estilització.
  • Optimització d'imatges: Optimitza automàticament les imatges per a temps de càrrega més ràpids.

Configuració d'un projecte Next.js

Per començar amb Next.js, necessites configurar un nou projecte.

npx create-next-app@latest my-next-app
cd my-next-app
npm run dev

Això crearà un nou projecte Next.js i iniciarà el servidor de desenvolupament.

Implementació de SSR a Next.js

Next.js fa que sigui fàcil implementar SSR amb la funció getServerSideProps.

Ús de `getServerSideProps`

La funció getServerSideProps et permet obtenir dades al costat del servidor i passar-les com a props al teu component de pàgina.

// pages/index.js
import React from 'react';

export async function getServerSideProps() {
  // Obtenir dades d'una API o base de dades
  const res = await fetch('https://api.example.com/data');
  const data = await res.json();

  // Passar dades a la pàgina via props
  return { props: { data } };
}

const HomePage = ({ data }) => {
  return (
    <div>
      <h1>Pàgina renderitzada al costat del servidor</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
};

export default HomePage;

Explicació

  • getServerSideProps: Aquesta funció s'executa al servidor en el moment de la sol·licitud. Obté dades i les passa al component de la pàgina com a props.
  • Component HomePage: Rep les dades com a props i les renderitza.

Comparació de SSR amb la renderització del costat del client (CSR)

| Característica | SSR | CSR | |--------------------------|------------------------------------------|------------------------------------------| | Temps de càrrega inicial | Més ràpid, ja que l'HTML està pre-renderitzat | Més lent, ja que el JavaScript necessita executar-se | | SEO | Millor, ja que els motors de cerca poden rastrejar HTML | Pitjor, ja que el contingut es carrega dinàmicament | | Experiència d'usuari | Millor experiència inicial | Pot tenir un estat de càrrega inicial | | Complexitat | Més complexa degut a la lògica del costat del servidor | Més simple, ja que tot és del costat del client |

Tècniques avançades de SSR

Emmagatzematge en memòria cau amb SSR

Per millorar el rendiment, pots implementar estratègies d'emmagatzematge en memòria cau per emmagatzemar l'HTML renderitzat i servir-lo per a sol·licituds posteriors.

Autenticació amb SSR

Gestionar l'autenticació en SSR pot ser complicat. Necessites gestionar cookies o tokens al costat del servidor per assegurar una obtenció segura de dades.

Conclusió

La renderització del costat del servidor (SSR) amb Next.js proporciona avantatges significatius en termes de rendiment, SEO i experiència d'usuari. En aprofitar la funció getServerSideProps, pots obtenir fàcilment dades al servidor i renderitzar pàgines HTML completament poblades. Tot i que SSR pot afegir complexitat al teu projecte, els beneficis sovint superen els inconvenients, especialment per a aplicacions amb molt de contingut o crítiques per al SEO.
