Què són els Hooks?

Els Hooks són una característica introduïda a React 16.8 que et permeten utilitzar l'estat i altres característiques de React sense escriure una classe. Et permeten gestionar l'estat i els efectes secundaris en components funcionals, fent que el teu codi sigui més concís i fàcil d'entendre.

Conceptes Clau

  • State Hook (useState): Et permet afegir estat als components funcionals.
  • Effect Hook (useEffect): Et permet realitzar efectes secundaris en components funcionals.
  • Custom Hooks: Et permeten extreure la lògica del component en funcions reutilitzables.

Per què utilitzar Hooks?

Els Hooks simplifiquen la manera d'escriure components:

  • Eliminant la necessitat de components de classe.
  • Fent més fàcil compartir la lògica entre components.
  • Millorant la llegibilitat i mantenibilitat del codi.

State Hook (`useState`)

El hook useState et permet afegir estat als teus components funcionals. Aquí tens un exemple bàsic:

import React, { useState } from 'react';

function Counter() {
  // Declara una variable d'estat anomenada "count" amb un valor inicial de 0
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Has fet clic {count} vegades</p>
      <button onClick={() => setCount(count + 1)}>
        Fes clic
      </button>
    </div>
  );
}

export default Counter;

Explicació

  • useState retorna una matriu amb dos elements: el valor actual de l'estat i una funció per actualitzar-lo.
  • Utilitzem la desestructuració de matrius per assignar aquests valors a count i setCount.
  • La funció setCount es crida quan es fa clic al botó, actualitzant l'estat.

Effect Hook (`useEffect`)

El hook useEffect et permet realitzar efectes secundaris en els teus components, com ara obtenir dades, subscripcions o canviar manualment el DOM.

import React, { useState, useEffect } from 'react';

function DataFetcher() {
  const [data, setData] = useState(null);

  useEffect(() => {
    // Obtenir dades d'una API
    fetch('https://api.example.com/data')
      .then(response => response.json())
      .then(data => setData(data));
  }, []); // La matriu buida significa que aquest efecte només s'executa una vegada, similar a componentDidMount

  return (
    <div>
      {data ? <p>Dades: {data}</p> : <p>Carregant...</p>}
    </div>
  );
}

export default DataFetcher;

Explicació

  • useEffect pren una funció que conté el codi de l'efecte secundari.
  • El segon argument és una matriu de dependències. Si està buida, l'efecte només s'executa una vegada després del renderitzat inicial.

Custom Hooks

Els hooks personalitzats et permeten extreure i reutilitzar la lògica a través de múltiples components. Són simplement funcions de JavaScript els noms de les quals comencen amb "use" i que poden cridar altres hooks.

import { useState, useEffect } from 'react';

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(url)
      .then(response => response.json())
      .then(data => {
        setData(data);
        setLoading(false);
      });
  }, [url]);

  return { data, loading };
}

export default useFetch;

Exemple d'ús

import React from 'react';
import useFetch from './useFetch';

function App() {
  const { data, loading } = useFetch('https://api.example.com/data');

  return (
    <div>
      {loading ? <p>Carregant...</p> : <p>Dades: {data}</p>}
    </div>
  );
}

export default App;

Explicació

  • useFetch és un hook personalitzat que obté dades d'una URL donada.
  • Retorna un objecte que conté les dades obtingudes i un estat de càrrega.
  • El component App utilitza aquest hook personalitzat per obtenir i mostrar dades.

Conclusió

Els Hooks són una addició poderosa a React que simplifiquen la gestió de l'estat i els efectes secundaris en components funcionals. Entenent i utilitzant hooks com useState, useEffect i hooks personalitzats, pots escriure codi React més net i mantenible. A mesura que continuïs explorant els hooks, descobriràs que ofereixen una manera flexible i eficient de gestionar la lògica dels components.
