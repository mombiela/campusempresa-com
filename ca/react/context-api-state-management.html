Introducció a l'API de Context

L'API de Context és una característica de React que et permet compartir l'estat a través de tota l'aplicació (o part d'ella) sense haver de passar les propietats manualment a cada nivell. Això és particularment útil per a la gestió de l'estat global.

Conceptes Clau

  • Context: Una manera de passar dades a través de l'arbre de components sense haver de passar propietats manualment a cada nivell.
  • Provider: Un component que proporciona l'estat als seus fills.
  • Consumer: Un component que consumeix l'estat proporcionat pel Provider.

Creant un Context

Per crear un context, utilitzes la funció React.createContext. Aquesta funció retorna un objecte de Context.

import React from 'react';

const MyContext = React.createContext();

Utilitzant el Provider

El component Provider s'utilitza per envoltar la part de la teva aplicació on vols que el context estigui disponible. Accepta una propietat value, que són les dades que vols compartir.

import React, { useState } from 'react';
import MyContext from './MyContext';

const App = () => {
  const [state, setState] = useState('Hola, Món!');

  return (
    <MyContext.Provider value={{ state, setState }}>
      <MyComponent />
    </MyContext.Provider>
  );
};

Consumint el Context

Hi ha dues maneres principals de consumir el context: utilitzant el hook useContext o el component Consumer.

Utilitzant el Hook `useContext`

El hook useContext és la manera més senzilla de consumir el context en components funcionals.

import React, { useContext } from 'react';
import MyContext from './MyContext';

const MyComponent = () => {
  const { state, setState } = useContext(MyContext);

  return (
    <div>
      <p>{state}</p>
      <button onClick={() => setState('Hola, React!')}>Canviar Estat</button>
    </div>
  );
};

Utilitzant el Component `Consumer`

El component Consumer és una manera alternativa de consumir el context, útil en components de classe.

import React from 'react';
import MyContext from './MyContext';

class MyComponent extends React.Component {
  render() {
    return (
      <MyContext.Consumer>
        {({ state, setState }) => (
          <div>
            <p>{state}</p>
            <button onClick={() => setState('Hola, React!')}>Canviar Estat</button>
          </div>
        )}
      </MyContext.Consumer>
    );
  }
}

Ús Avançat

Contexts Niats

Pots niar múltiples contexts per gestionar diferents peces d'estat de manera independent.

const ThemeContext = React.createContext();
const UserContext = React.createContext();

const App = () => {
  return (
    <ThemeContext.Provider value={{ theme: 'dark' }}>
      <UserContext.Provider value={{ user: 'John Doe' }}>
        <MyComponent />
      </UserContext.Provider>
    </ThemeContext.Provider>
  );
};

Actualitzant el Context des de Components Niats

Per actualitzar el context des de components profundament niats, pots passar funcions a través del context.

const MyContext = React.createContext();

const App = () => {
  const [state, setState] = useState('Hola, Món!');

  return (
    <MyContext.Provider value={{ state, setState }}>
      <NestedComponent />
    </MyContext.Provider>
  );
};

const NestedComponent = () => {
  const { state, setState } = useContext(MyContext);

  return (
    <div>
      <p>{state}</p>
      <button onClick={() => setState('Hola, Món Niat!')}>Canviar Estat</button>
    </div>
  );
};

Conclusió

L'API de Context és una eina poderosa per a la gestió de l'estat en React, especialment per a casos d'ús més senzills i aplicacions més petites. Et permet evitar el prop drilling i fa que el teu codi sigui més net i mantenible. Per a necessitats de gestió d'estat més complexes, podries considerar altres solucions com Redux o MobX.
