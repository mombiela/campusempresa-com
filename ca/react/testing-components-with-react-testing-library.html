Introducció a React Testing Library

React Testing Library (RTL) és una utilitat de proves popular per a aplicacions React. Es centra en provar components des de la perspectiva de l'usuari, assegurant que les teves proves siguin més fiables i mantenibles.

  • Conceptes clau:
    • Proves centrades en l'usuari: RTL fomenta provar components tal com un usuari interactuaria amb ells.
    • Consultes: RTL proporciona diversos mètodes de consulta per trobar elements en el DOM.
    • Afirmacions: Utilitzant afirmacions per verificar els resultats esperats.

Configuració de React Testing Library

Per començar a utilitzar RTL, cal instal·lar-lo i configurar el teu entorn de proves.

  • Instal·lació:

    npm install --save-dev @testing-library/react @testing-library/jest-dom

  • Configuració: Assegura't de tenir Jest configurat, ja que RTL funciona perfectament amb Jest. Crea un fitxer setupTests.js per incloure configuracions personalitzades si cal.

    // setupTests.js
import '@testing-library/jest-dom/extend-expect';

Escriure la teva primera prova

Escrivim una prova senzilla per a un component React utilitzant RTL.

  • Component d'exemple:

    // Greeting.js
import React from 'react';

const Greeting = ({ name }) => {
  return <h1>Hello, {name}!</h1>;
};

export default Greeting;

  • Prova per al component:

    // Greeting.test.js
import React from 'react';
import { render, screen } from '@testing-library/react';
import Greeting from './Greeting';

test('renders greeting message', () => {
  render(<Greeting name="John" />);
  const greetingElement = screen.getByText(/hello, john!/i);
  expect(greetingElement).toBeInTheDocument();
});

    Explicació:

    • render: Renderitza el component en el DOM virtual.
    • screen.getByText: Troba un element pel seu contingut de text.
    • expect: Afirma que l'element està en el document.

Consultes comunes en React Testing Library

RTL proporciona diversos mètodes de consulta per trobar elements en el DOM.

  • Tipus de consultes:

    • getBy: Llença un error si no es troba cap element.
    • queryBy: Retorna null si no es troba cap element.
    • findBy: Retorna una promesa i espera que l'element aparegui.

  • Exemples:

    // ExampleComponent.js
import React from 'react';

const ExampleComponent = () => (
  <div>
    <button>Click me</button>
    <input placeholder="Enter text" />
  </div>
);

export default ExampleComponent;

    // ExampleComponent.test.js
import React from 'react';
import { render, screen } from '@testing-library/react';
import ExampleComponent from './ExampleComponent';

test('finds elements by text, role, and placeholder', () => {
  render(<ExampleComponent />);

  const buttonElement = screen.getByText(/click me/i);
  expect(buttonElement).toBeInTheDocument();

  const inputElement = screen.getByPlaceholderText(/enter text/i);
  expect(inputElement).toBeInTheDocument();
});

Tècniques avançades de proves

A mesura que avancis, potser necessitaràs provar interaccions més complexes i canvis d'estat.

  • Proves d'esdeveniments d'usuari:

    // Counter.js
import React, { useState } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
};

export default Counter;

    // Counter.test.js
import React from 'react';
import { render, screen, fireEvent } from '@testing-library/react';
import Counter from './Counter';

test('increments counter', () => {
  render(<Counter />);

  const buttonElement = screen.getByText(/increment/i);
  fireEvent.click(buttonElement);

  const countElement = screen.getByText(/count: 1/i);
  expect(countElement).toBeInTheDocument();
});

    Explicació:

    • fireEvent.click: Simula un esdeveniment de clic en el botó.

Simulació de crides API

De vegades, necessitaràs provar components que fan crides API. Pots simular aquestes crides utilitzant Jest.

  • Exemple amb simulació:

    // FetchData.js
import React, { useEffect, useState } from 'react';

const FetchData = () => {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch('/api/data')
      .then(response => response.json())
      .then(data => setData(data));
  }, []);

  if (!data) return <div>Loading...</div>;
  return <div>Data: {data}</div>;
};

export default FetchData;

    // FetchData.test.js
import React from 'react';
import { render, screen, waitFor } from '@testing-library/react';
import FetchData from './FetchData';

beforeAll(() => {
  global.fetch = jest.fn(() =>
    Promise.resolve({
      json: () => Promise.resolve('mock data'),
    })
  );
});

test('fetches and displays data', async () => {
  render(<FetchData />);

  await waitFor(() => screen.getByText(/data: mock data/i));
  expect(screen.getByText(/data: mock data/i)).toBeInTheDocument();
});

    Explicació:

    • global.fetch: Simula la funció global fetch.
    • waitFor: Espera que l'operació asíncrona es completi.

Conclusió

React Testing Library és una eina poderosa per provar components React. En centrar-se en les interaccions i resultats de l'usuari, ajuda a crear proves més fiables i mantenibles. A mesura que et sentis més còmode amb RTL, pots explorar tècniques avançades com la simulació de crides API i la prova d'interaccions complexes per assegurar-te que la teva aplicació funcioni com s'espera.
