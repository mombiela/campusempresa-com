En aquesta secció, cobrim com preparar la teva aplicació React per a producció. Això implica optimitzar la teva aplicació per al rendiment, assegurar-se que sigui segura i desplegar-la en un servidor en viu.

Conceptes Clau

  • Optimització: Tècniques per millorar el rendiment de la teva aplicació React.
  • Seguretat: Mesures per protegir la teva aplicació de vulnerabilitats.
  • Desplegament: Passos per desplegar la teva aplicació React en un servidor de producció.

Optimització

Optimitzar la teva aplicació React és crucial per proporcionar una experiència d'usuari fluida. Aquí hi ha algunes tècniques clau:

  • Divisió de Codi: Descompon el teu codi en trossos més petits per carregar només el que és necessari.
  • Carrega Peresosa: Carrega components només quan siguin necessaris.
  • Minificació: Redueix la mida dels teus fitxers JavaScript.
  • Sacsejada d'Arbres: Elimina codi no utilitzat del teu paquet final.

Divisió de Codi

La divisió de codi es pot aconseguir utilitzant declaracions dinàmiques import(). Això et permet dividir el teu codi en paquets separats que es poden carregar sota demanda.

import React, { Suspense, lazy } from 'react';

const LazyComponent = lazy(() => import('./LazyComponent'));

function App() {
  return (
    <div>
      <Suspense fallback={<div>Carregant...</div>}>
        <LazyComponent />
      </Suspense>
    </div>
  );
}

export default App;

Carrega Peresosa

La carrega peresosa és una tècnica per ajornar la càrrega de recursos no crítics en el moment de la càrrega de la pàgina. Això es pot fer utilitzant React.lazy i Suspense de React.

const LazyComponent = React.lazy(() => import('./LazyComponent'));

function App() {
  return (
    <div>
      <React.Suspense fallback={<div>Carregant...</div>}>
        <LazyComponent />
      </React.Suspense>
    </div>
  );
}

Minificació

La minificació es pot gestionar amb eines de construcció com Webpack. Quan crees una construcció de producció utilitzant create-react-app, automàticament minifica el teu codi.

npm run build

Sacsejada d'Arbres

La sacsejada d'arbres és una forma d'eliminació de codi mort. Webpack realitza la sacsejada d'arbres quan crees una construcció de producció.

npm run build

Seguretat

Assegurar que la teva aplicació React sigui segura és essencial. Aquí hi ha algunes pràctiques recomanades:

  • Evita Scripts Inline: Utilitza scripts externs per prevenir atacs de Cross-Site Scripting (XSS).
  • Política de Seguretat de Contingut (CSP): Implementa capçaleres CSP per controlar els recursos que l'agent d'usuari està permès carregar.
  • Variables d'Entorn: Utilitza variables d'entorn per gestionar informació sensible.

Evita Scripts Inline

Evita utilitzar scripts i estils inline per reduir el risc d'atacs XSS.

<!-- Evita això -->
<script>alert('Atac XSS!');</script>

<!-- Utilitza scripts externs en canvi -->
<script src="external-script.js"></script>

Política de Seguretat de Contingut (CSP)

Implementa capçaleres CSP per controlar els recursos que el navegador està permès carregar.

<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' https://apis.google.com">

Variables d'Entorn

Utilitza variables d'entorn per gestionar informació sensible com claus d'API.

// .env
REACT_APP_API_KEY=la_teva_clau_api_aquí

// Accedint a la teva aplicació React
const apiKey = process.env.REACT_APP_API_KEY;

Desplegament

Desplegar la teva aplicació React implica construir l'aplicació i després pujar-la a un servidor. Aquí hi ha els passos:

  • Construeix l'Aplicació: Crea una construcció de producció.
  • Tria un Servei d'Allotjament: Selecciona un servei com Netlify, Vercel o GitHub Pages.
  • Desplega l'Aplicació: Puja els teus fitxers de construcció al servei d'allotjament.

Construeix l'Aplicació

Crea una construcció de producció de la teva aplicació React.

npm run build

Tria un Servei d'Allotjament

Selecciona un servei d'allotjament. Aquí hi ha algunes opcions populars:

| Servei d'Allotjament | Característiques | |----------------------|------------------| | Netlify | Desplegament continu, dominis personalitzats, HTTPS | | Vercel | Funcions sense servidor, dominis personalitzats, HTTPS | | GitHub Pages | Allotjament gratuït per a llocs estàtics, dominis personalitzats|

Desplega l'Aplicació

Segueix les instruccions per al servei d'allotjament que hagis triat per desplegar la teva aplicació. Per exemple, desplegant a Netlify:

  1. Instal·la el CLI de Netlify:

    npm install -g netlify-cli

  2. Desplega la teva aplicació:

    netlify deploy --prod

Conclusió

Construir la teva aplicació React per a producció implica optimitzar el rendiment, assegurar la seguretat i desplegar-la en un servidor en viu. Seguint les millors pràctiques per a la divisió de codi, la carrega peresosa, la minificació i la sacsejada d'arbres, pots crear una aplicació rendible. Implementant mesures de seguretat com evitar scripts inline, utilitzar CSP i gestionar variables d'entorn ajudarà a protegir la teva aplicació. Finalment, desplegant la teva aplicació a un servei d'allotjament com Netlify, Vercel o GitHub Pages la farà accessible als usuaris.
