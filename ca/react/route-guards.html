Introducció a les Guàrdies de Ruta

Les guàrdies de ruta són mecanismes utilitzats per controlar l'accés a certes rutes en una aplicació web. En React, les guàrdies de ruta ajuden a assegurar que els usuaris només puguin accedir a rutes específiques si compleixen certes condicions, com ara estar autenticats o tenir permisos específics.

Per què Utilitzar Guàrdies de Ruta?

  • Seguretat: Protegir rutes sensibles de l'accés no autoritzat.
  • Experiència d'Usuari: Redirigir els usuaris a pàgines apropiades segons el seu estat (p. ex., pàgina d'inici de sessió si no estan autenticats).
  • Control: Gestionar l'accés a diferents parts de l'aplicació basant-se en els rols dels usuaris o altres criteris.

Implementació de Guàrdies de Ruta en React

Prerequisits

Abans d'implementar guàrdies de ruta, assegura't de tenir el següent:

  • Comprensió bàsica de React i React Router.
  • Una aplicació React configurada amb react-router-dom instal·lat.

Exemple Bàsic de Guàrdia de Ruta

Comencem amb un exemple senzill on protegim una ruta basant-nos en l'autenticació de l'usuari.

  1. Instal·la react-router-dom:

    npm install react-router-dom

  2. Crea un context d'autenticació simple:

    // AuthContext.js
import React, { createContext, useState, useContext } from 'react';

const AuthContext = createContext();

export const AuthProvider = ({ children }) => {
    const [isAuthenticated, setIsAuthenticated] = useState(false);

    const login = () => setIsAuthenticated(true);
    const logout = () => setIsAuthenticated(false);

    return (
        <AuthContext.Provider value={{ isAuthenticated, login, logout }}>
            {children}
        </AuthContext.Provider>
    );
};

export const useAuth = () => useContext(AuthContext);

  3. Crea un component PrivateRoute:

    // PrivateRoute.js
import React from 'react';
import { Route, Redirect } from 'react-router-dom';
import { useAuth } from './AuthContext';

const PrivateRoute = ({ component: Component, ...rest }) => {
    const { isAuthenticated } = useAuth();

    return (
        <Route
            {...rest}
            render={props =>
                isAuthenticated ? (
                    <Component {...props} />
                ) : (
                    <Redirect to="/login" />
                )
            }
        />
    );
};

export default PrivateRoute;

  4. Configura les rutes en la teva aplicació:

    // App.js
import React from 'react';
import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';
import { AuthProvider } from './AuthContext';
import PrivateRoute from './PrivateRoute';
import HomePage from './HomePage';
import LoginPage from './LoginPage';
import Dashboard from './Dashboard';

const App = () => {
    return (
        <AuthProvider>
            <Router>
                <Switch>
                    <Route path="/login" component={LoginPage} />
                    <PrivateRoute path="/dashboard" component={Dashboard} />
                    <Route path="/" component={HomePage} />
                </Switch>
            </Router>
        </AuthProvider>
    );
};

export default App;

Guàrdies de Ruta Avançades

Per a escenaris més complexos, potser necessitis comprovar els rols o permisos dels usuaris.

  1. Amplia el AuthContext per incloure rols:

    // AuthContext.js
const AuthProvider = ({ children }) => {
    const [isAuthenticated, setIsAuthenticated] = useState(false);
    const [userRole, setUserRole] = useState(null);

    const login = (role) => {
        setIsAuthenticated(true);
        setUserRole(role);
    };
    const logout = () => {
        setIsAuthenticated(false);
        setUserRole(null);
    };

    return (
        <AuthContext.Provider value={{ isAuthenticated, userRole, login, logout }}>
            {children}
        </AuthContext.Provider>
    );
};

  2. Crea un component RoleBasedRoute:

    // RoleBasedRoute.js
import React from 'react';
import { Route, Redirect } from 'react-router-dom';
import { useAuth } from './AuthContext';

const RoleBasedRoute = ({ component: Component, requiredRole, ...rest }) => {
    const { isAuthenticated, userRole } = useAuth();

    return (
        <Route
            {...rest}
            render={props =>
                isAuthenticated && userRole === requiredRole ? (
                    <Component {...props} />
                ) : (
                    <Redirect to="/unauthorized" />
                )
            }
        />
    );
};

export default RoleBasedRoute;

  3. Configura les rutes basades en rols en la teva aplicació:

    // App.js
import React from 'react';
import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';
import { AuthProvider } from './AuthContext';
import RoleBasedRoute from './RoleBasedRoute';
import HomePage from './HomePage';
import LoginPage from './LoginPage';
import AdminDashboard from './AdminDashboard';
import UserDashboard from './UserDashboard';
import UnauthorizedPage from './UnauthorizedPage';

const App = () => {
    return (
        <AuthProvider>
            <Router>
                <Switch>
                    <Route path="/login" component={LoginPage} />
                    <RoleBasedRoute path="/admin" component={AdminDashboard} requiredRole="admin" />
                    <RoleBasedRoute path="/user" component={UserDashboard} requiredRole="user" />
                    <Route path="/unauthorized" component={UnauthorizedPage} />
                    <Route path="/" component={HomePage} />
                </Switch>
            </Router>
        </AuthProvider>
    );
};

export default App;

Conclusió

Les guàrdies de ruta són essencials per controlar l'accés a diferents parts de la teva aplicació React. Utilitzant guàrdies de ruta, pots assegurar-te que només els usuaris autoritzats puguin accedir a certes rutes, millorant tant la seguretat com l'experiència d'usuari. Tant si necessites comprovacions d'autenticació simples com un control d'accés més complex basat en rols, React i react-router-dom proporcionen les eines necessàries per implementar guàrdies de ruta efectives.
