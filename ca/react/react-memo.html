Introducció a React Memo

React Memo és un component d'ordre superior (HOC) que optimitza el rendiment dels components funcionals evitant re-renderitzacions innecessàries. És particularment útil quan tens components que rendeixen la mateixa sortida donades les mateixes props.

Conceptes Clau

  • Component d'Ordre Superior (HOC): Una funció que pren un component i retorna un nou component amb capacitats addicionals.
  • Re-renderitzacions: Quan un component actualitza la seva sortida a causa de canvis en l'estat o les props.
  • Memoització: Una tècnica d'optimització per emmagatzemar els resultats de crides de funcions costoses i retornar el resultat emmagatzemat quan es produeixen els mateixos inputs de nou.

Quan Utilitzar React Memo

  • Quan tens components funcionals que reben les mateixes props freqüentment.
  • Quan vols evitar re-renderitzacions innecessàries per millorar el rendiment.
  • Quan la lògica de renderització del component és complexa o consumeix molts recursos.

Com Utilitzar React Memo

Per utilitzar React Memo, embolica el teu component funcional amb React.memo:

import React from 'react';

const MyComponent = (props) => {
  console.log('Rendering MyComponent');
  return <div>{props.value}</div>;
};

export default React.memo(MyComponent);

Exemple Explicat

  • Importar React: Importa la biblioteca necessària de React.
  • Definir Component Funcional: Crea un component funcional MyComponent que pren props i retorna un div amb el props.value.
  • Embolicar amb React.memo: Exporta el component embolicat en React.memo per habilitar la memoització.

Comparant React Memo amb PureComponent

| Característica | React.memo (Components Funcionals) | PureComponent (Components de Classe) | |------------------------|-------------------------------------|--------------------------------------| | Ús | Components Funcionals | Components de Classe | | Memoització | Sí | Sí | | Comparació Superficial de Props | Sí | Sí | | Gestió d'Estat | No | Sí |

Ús Avançat

Funció de Comparació Personalitzada

Per defecte, React.memo fa una comparació superficial de les props. Pots proporcionar una funció de comparació personalitzada per controlar el comportament de la re-renderització de manera més precisa.

import React from 'react';

const MyComponent = (props) => {
  console.log('Rendering MyComponent');
  return <div>{props.value}</div>;
};

const areEqual = (prevProps, nextProps) => {
  return prevProps.value === nextProps.value;
};

export default React.memo(MyComponent, areEqual);

Exemple Explicat

  • Funció de Comparació Personalitzada: Defineix una funció areEqual que pren prevProps i nextProps i retorna true si són iguals, evitant la re-renderització.
  • Passar Funció Personalitzada: Passa la funció de comparació personalitzada com a segon argument a React.memo.

Trampes Comunes

  • Sobreutilitzar React Memo: No tots els components es beneficien de la memoització. Utilitza-ho només quan sigui necessari.
  • Props Complexes: Objectes o arrays profundament niats poden no beneficiar-se de la comparació superficial.
  • Canvis d'Estat: React Memo no evita re-renderitzacions a causa de canvis d'estat dins del component.

Conclusió

React Memo és una eina poderosa per optimitzar el rendiment dels components funcionals evitant re-renderitzacions innecessàries. Entenent quan i com utilitzar-la, pots fer que les teves aplicacions React siguin més eficients. Recorda utilitzar-la amb prudència i considera la complexitat de les teves props i la gestió de l'estat.
