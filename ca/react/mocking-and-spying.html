Introducció

La simulació i l'espionatge són tècniques essencials en les proves d'aplicacions React. Ajuden a aïllar components i funcions per assegurar que les proves siguin fiables i enfocades. Aquesta secció cobrirà els conceptes bàsics de la simulació i l'espionatge, la seva importància i com implementar-los utilitzant biblioteques de proves populars com Jest i Enzyme.

Conceptes Clau

  • Simulació: Crear una versió falsa d'una funció o mòdul per controlar el seu comportament durant les proves.
  • Espionatge: Monitoritzar les crides a una funció per verificar com s'ha utilitzat durant les proves.

Per què la Simulació i l'Espionatge són Importants

  • Aïllament: Assegura que les proves estiguin enfocades en el component o funció que s'està provant, no en les seves dependències.
  • Control: Permet simular diferents escenaris i casos límit.
  • Rendiment: Redueix la complexitat i el temps d'execució de les proves evitant implementacions reals.

Simulació en React

La simulació s'utilitza sovint per substituir dependències amb versions controlades. Jest proporciona funcions integrades per crear simulacions.

Simulació de Funcions

Per simular una funció, pots utilitzar jest.fn().

// Funció original
const fetchData = () => {
  return fetch('/api/data').then(response => response.json());
};

// Funció simulada
const fetchDataMock = jest.fn().mockResolvedValue({ data: 'mocked data' });

// Utilitzant la simulació en una prova
test('fetchData retorna dades simulades', async () => {
  const data = await fetchDataMock();
  expect(data).toEqual({ data: 'mocked data' });
});

Simulació de Mòduls

Pots simular mòduls sencers utilitzant jest.mock().

// Mòdul original
// api.js
export const fetchData = () => {
  return fetch('/api/data').then(response => response.json());
};

// Fitxer de prova
import { fetchData } from './api';

jest.mock('./api', () => ({
  fetchData: jest.fn().mockResolvedValue({ data: 'mocked data' })
}));

test('fetchData retorna dades simulades', async () => {
  const data = await fetchData();
  expect(data).toEqual({ data: 'mocked data' });
});

Espionatge en React

L'espionatge permet fer un seguiment de les crides a una funció i inspeccionar com s'ha utilitzat. Jest proporciona jest.spyOn() per a aquest propòsit.

Espionatge de Mètodes

Pots espiar els mètodes dels objectes per verificar el seu ús.

class User {
  getName() {
    return 'John Doe';
  }
}

test('el mètode getName és cridat', () => {
  const user = new User();
  const spy = jest.spyOn(user, 'getName');
  
  user.getName();
  
  expect(spy).toHaveBeenCalled();
});

Espionatge de Mètodes de Components React

L'espionatge també es pot utilitzar per monitoritzar mètodes de components React.

import React from 'react';
import { shallow } from 'enzyme';

class MyComponent extends React.Component {
  componentDidMount() {
    this.fetchData();
  }

  fetchData() {
    // lògica de fetch data
  }

  render() {
    return <div>My Component</div>;
  }
}

test('fetchData és cridat en componentDidMount', () => {
  const spy = jest.spyOn(MyComponent.prototype, 'fetchData');
  shallow(<MyComponent />);
  
  expect(spy).toHaveBeenCalled();
});

Conclusió

La simulació i l'espionatge són tècniques poderoses que milloren la fiabilitat i l'eficiència de les teves proves. Aïllant components i controlant les seves dependències, pots crear proves més enfocades i ràpides. Ja sigui simulant funcions, mòduls o espiant mètodes, aquestes eines són inavaluables per mantenir aplicacions React d'alta qualitat.
