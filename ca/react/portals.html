Introducció als Portals

Els portals proporcionen una manera de renderitzar fills en un node del DOM que existeix fora de la jerarquia del component pare. Això és particularment útil per a escenaris on necessites sortir de l'estructura típica pare-fill del DOM, com ara modals, tooltips o qualsevol element de superposició.

Conceptes Clau

  • Portals: Mecanisme per renderitzar components fora de la seva jerarquia DOM pare.
  • ReactDOM.createPortal: El mètode utilitzat per crear un portal.

Creant un Portal

Per crear un portal, utilitzes el mètode ReactDOM.createPortal. Aquest mètode pren dos arguments:

  1. El JSX que vols renderitzar.
  2. El node del DOM on vols renderitzar el JSX.

Exemple

import React from 'react';
import ReactDOM from 'react-dom';

class Modal extends React.Component {
  render() {
    // Crear un portal
    return ReactDOM.createPortal(
      this.props.children,
      document.getElementById('modal-root')
    );
  }
}

class App extends React.Component {
  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <Modal>
          <div className="modal">
            <p>This is a modal!</p>
          </div>
        </Modal>
      </div>
    );
  }
}

// Suposant que tens un div amb id 'modal-root' en el teu HTML
// <div id="modal-root"></div>

En aquest exemple:

  • El component Modal utilitza ReactDOM.createPortal per renderitzar els seus fills en el node del DOM amb l'id modal-root.
  • El component App renderitza un component Modal, que al seu torn renderitza els seus fills (el contingut del modal) fora de la jerarquia normal del DOM.

Usos dels Portals

  • Modals/Diàlegs: Mostrar contingut en un modal que superposa la resta de l'aplicació.
  • Tooltips: Mostrar tooltips que necessiten escapar de les restriccions d'overflow o z-index dels seus elements pares.
  • Dropdowns: Renderitzar menús desplegables que necessiten aparèixer sobre altres continguts.

Maneig d'Esdeveniments en Portals

Els esdeveniments que ocorren dins d'un portal encara es propaguen als avantpassats en l'arbre de React, fins i tot si aquests elements no són avantpassats en l'arbre del DOM. Això significa que pots manejar esdeveniments en el component pare com de costum.

Exemple

import React from 'react';
import ReactDOM from 'react-dom';

class Modal extends React.Component {
  render() {
    return ReactDOM.createPortal(
      <div className="modal">
        <button onClick={this.props.onClose}>Close Modal</button>
      </div>,
      document.getElementById('modal-root')
    );
  }
}

class App extends React.Component {
  handleClose = () => {
    console.log('Modal closed');
  };

  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <Modal onClose={this.handleClose} />
      </div>
    );
  }
}

En aquest exemple:

  • El component Modal renderitza un botó que, quan es fa clic, crida el mètode onClose passat des del component App.
  • El mètode handleClose en el component App registra un missatge a la consola.

Conclusió

Els portals en React proporcionen una manera poderosa de renderitzar components fora de la seva jerarquia DOM pare, fent-los ideals per a casos d'ús com modals, tooltips i dropdowns. Utilitzant ReactDOM.createPortal, pots assegurar-te que els teus components es renderitzen en el lloc correcte del DOM mentre mantens l'estructura de l'arbre de components de React i el maneig d'esdeveniments.

Entendre i utilitzar portals pot millorar enormement la flexibilitat i usabilitat de les teves aplicacions React, especialment quan tractes amb elements d'interfície complexos que necessiten sortir dels seus contenidors pares.
