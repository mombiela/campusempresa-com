Introducció a les proves d'extrem a extrem

Les proves d'extrem a extrem (E2E) són una metodologia utilitzada per provar si el flux d'una aplicació funciona segons el disseny des del principi fins al final. L'objectiu és simular escenaris reals d'usuari per validar el sistema en el seu conjunt.

  • Propòsit de les proves E2E: Assegurar que els components integrats d'una aplicació funcionen com s'espera.
  • Beneficis:
    • Detecta problemes en la interfície d'usuari.
    • Valida el flux general del sistema.
    • Assegura que l'aplicació funciona en escenaris del món real.

Introducció a Cypress

Cypress és un marc de proves E2E modern i de codi obert construït per a la web. Proporciona una experiència de prova robusta, ràpida i fiable per a qualsevol cosa que s'executi en un navegador.

  • Característiques clau:
    • Viatge en el temps: Depuració fàcil visualitzant instantànies de les teves proves.
    • Recarregues en temps real: Recarrega automàticament les proves en desar.
    • Espera automàtica: No cal afegir esperes o pauses a les teves proves.
    • Espies, simulacions i rellotges: Controla i inspecciona el comportament de funcions, respostes del servidor o temporitzadors.

Configuració de Cypress en un projecte React

Per començar a utilitzar Cypress en el teu projecte React, segueix aquests passos:

  1. Instal·la Cypress:

    npm install cypress --save-dev

  2. Obre Cypress:

    npx cypress open

  3. Estructura del projecte: Cypress crearà un directori cypress a l'arrel del teu projecte amb la següent estructura:

    cypress/
  ├── fixtures/
  ├── integration/
  ├── plugins/
  └── support/

Escriure la teva primera prova

Escrivim una prova senzilla per comprovar si la nostra aplicació React es carrega correctament.

  1. Crea un fitxer de prova: Crea un nou fitxer al directori cypress/integration, per exemple, app_spec.js.

  2. Escriu la prova:

    describe('My First Test', () => {
  it('Visits the app root url', () => {
    cy.visit('http://localhost:3000');
    cy.contains('Welcome to React');
  });
});

  3. Executa la prova: Obre Cypress i executa la prova. Hauries de veure la prova passant si la teva aplicació mostra "Welcome to React" a la pàgina d'inici.

Tècniques avançades de prova

Interactuar amb elements

Aprèn a interactuar amb diversos elements de la pàgina.

  • Fer clic a un botó:

    cy.get('button').click();

  • Escriure en un camp d'entrada:

    cy.get('input[name="username"]').type('myusername');

Afirmacions

Les afirmacions s'utilitzen per validar l'estat de la teva aplicació.

  • Comprovar la visibilitat d'un element:

    cy.get('button').should('be.visible');

  • Comprovar el text d'un element:

    cy.get('h1').should('have.text', 'Welcome to React');

Simulació de sol·licituds API

Cypress et permet simular i falsificar sol·licituds API per provar diferents escenaris.

  • Simular una sol·licitud GET: 
    cy.server();
cy.route('GET', '/api/users', 'fixture:users.json').as('getUsers');
cy.visit('/users');
cy.wait('@getUsers');

Millors pràctiques

  • Mantingues les proves independents: Cada prova hauria de poder executar-se independentment de les altres.
  • Utilitza fixtures per a dades de prova: Emmagatzema i reutilitza dades de prova utilitzant fixtures.
  • Neteja l'estat entre proves: Assegura't que l'estat de l'aplicació es restableix entre proves per evitar proves inestables.

Conclusió

Les proves d'extrem a extrem amb Cypress proporcionen una manera poderosa d'assegurar que la teva aplicació React funcioni com s'espera en escenaris del món real. Configurant Cypress, escrivint proves bàsiques i aprofitant tècniques avançades, pots crear un conjunt de proves robust que ajudi a mantenir la qualitat i la fiabilitat de la teva aplicació.

Recorda seguir les millors pràctiques per mantenir les teves proves mantenibles i fiables. Bones proves!
