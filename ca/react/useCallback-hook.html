Introducció a useCallback

El hook useCallback és una part fonamental per optimitzar el rendiment en aplicacions React. Et permet memoitzar funcions perquè no es recreïn en cada renderització, cosa que pot ajudar a prevenir renderitzacions innecessàries de components fills.

Conceptes Clau

  • Memoització: Emmagatzemar el resultat de crides a funcions costoses i retornar el resultat emmagatzemat en memòria cau quan es produeixen els mateixos inputs.
  • Dependències: Una matriu de valors dels quals depèn el hook. La funció memoitzada només canviarà si una d'aquestes dependències canvia.

Ús Bàsic de useCallback

El hook useCallback pren dos arguments: una funció i una matriu de dependències. Retorna una versió memoitzada de la funció que només canvia si una de les dependències ha canviat.

Sintaxi

const memoizedCallback = useCallback(() => {
  // La teva lògica de funció aquí
}, [dependency1, dependency2, ...]);

Exemple

Comencem amb un exemple senzill per entendre com funciona useCallback.

import React, { useState, useCallback } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  const increment = useCallback(() => {
    setCount(count + 1);
  }, [count]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={increment}>Increment</button>
    </div>
  );
}

export default Counter;

En aquest exemple:

  • increment és una funció memoitzada que només canviarà si count canvia.
  • Això prevé que la funció increment es recreï en cada renderització.

Ús Avançat de useCallback

Optimització de Components Fills

Un dels casos d'ús principals per useCallback és optimitzar components fills que depenen de funcions passades com a props.

Exemple amb Component Fill

import React, { useState, useCallback } from 'react';

function Child({ onClick }) {
  console.log('Child rendered');
  return <button onClick={onClick}>Click me</button>;
}

function Parent() {
  const [count, setCount] = useState(0);

  const handleClick = useCallback(() => {
    console.log('Button clicked');
  }, []);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <Child onClick={handleClick} />
    </div>
  );
}

export default Parent;

En aquest exemple:

  • La funció handleClick està memoitzada i passada al component Child.
  • El component Child no es tornarà a renderitzar a menys que handleClick canviï, cosa que només passa si les seves dependències canvien.

Comparació entre useCallback i useMemo

Tant useCallback com useMemo s'utilitzen per a l'optimització del rendiment, però tenen propòsits diferents.

| Hook | Propòsit | Valor de Retorn | |------------|----------------------------------------------|-------------------------| | useCallback | Memoitza una funció | Funció memoitzada | | useMemo | Memoitza el resultat d'un càlcul | Valor memoitzat (resultat) |

Exemple amb useMemo

import React, { useState, useMemo } from 'react';

function ExpensiveComponent({ num }) {
  const compute = useMemo(() => {
    console.log('Computing...');
    return num * 2;
  }, [num]);

  return <div>Computed Value: {compute}</div>;
}

function App() {
  const [number, setNumber] = useState(1);

  return (
    <div>
      <input 
        type="number" 
        value={number} 
        onChange={(e) => setNumber(parseInt(e.target.value))} 
      />
      <ExpensiveComponent num={number} />
    </div>
  );
}

export default App;

En aquest exemple:

  • useMemo s'utilitza per memoitzar el resultat del càlcul (num * 2), de manera que només es recalcula quan num canvia.

Errors Comuns

  • Ús excessiu de useCallback: No totes les funcions necessiten ser memoitzades. L'ús excessiu de useCallback pot conduir a una complexitat innecessària.
  • Dependències Incorrectes: Assegura't que totes les dependències estan especificades correctament. Les dependències mancants poden conduir a tancaments obsolets i errors.

Conclusió

El hook useCallback és una eina poderosa per optimitzar aplicacions React memoitzant funcions. Ajuda a prevenir renderitzacions innecessàries de components fills i pot millorar significativament el rendiment. No obstant això, s'ha d'utilitzar amb prudència per evitar afegir complexitat innecessària al teu codi.

Entenent i implementant correctament useCallback, pots crear aplicacions React més eficients i amb millor rendiment.
