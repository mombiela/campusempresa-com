Introducció

A React, hi ha dues maneres principals de gestionar les dades dels formularis: components controlats i components no controlats. Aquesta secció se centrarà en els components no controlats, que et permeten gestionar les dades dels formularis utilitzant el mateix DOM en lloc de l'estat de React.

Conceptes clau

  • Components no controlats: Aquests components depenen del DOM per gestionar les dades dels formularis.
  • Refs: Un atribut especial que et permet accedir directament als nodes del DOM dins de React.

Per què utilitzar components no controlats?

  • Simplicitat: Per a formularis senzills, els components no controlats poden ser més fàcils de configurar.
  • Rendiment: Poden ser més eficients per a formularis grans, ja que no requereixen actualitzacions d'estat a cada canvi d'entrada.

Utilitzant Refs en components no controlats

Les refs proporcionen una manera d'accedir als nodes del DOM o als elements de React creats en el mètode de renderització. Aquí tens com pots utilitzar les refs per gestionar les dades dels formularis en un component no controlat.

Exemple: Formulari bàsic no controlat

import React, { useRef } from 'react';

function UncontrolledForm() {
  const inputRef = useRef(null);

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Input Value: ${inputRef.current.value}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" ref={inputRef} />
      </label>
      <button type="submit">Submit</button>
    </form>
  );
}

export default UncontrolledForm;

Explicació

  • Hook useRef: El hook useRef s'utilitza per crear una referència a l'element d'entrada.
  • Accedint al valor: Quan es presenta el formulari, inputRef.current.value s'utilitza per accedir al valor actual del camp d'entrada.

Gestionant múltiples entrades

Pots gestionar múltiples entrades en un component no controlat creant múltiples refs.

Exemple: Múltiples entrades

import React, { useRef } from 'react';

function UncontrolledFormMultiple() {
  const nameRef = useRef(null);
  const emailRef = useRef(null);

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Name: ${nameRef.current.value}, Email: ${emailRef.current.value}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" ref={nameRef} />
      </label>
      <br />
      <label>
        Email:
        <input type="email" ref={emailRef} />
      </label>
      <br />
      <button type="submit">Submit</button>
    </form>
  );
}

export default UncontrolledFormMultiple;

Explicació

  • Múltiples refs: Es creen refs separades per a cada camp d'entrada.
  • Accedint als valors: Cada ref s'utilitza per accedir al valor corresponent de l'entrada quan es presenta el formulari.

Comparació: Components controlats vs. components no controlats

| Característica | Components controlats | Components no controlats | |------------------------------|----------------------------------------|----------------------------------------| | Gestió de l'estat | Gestionat per l'estat de React | Gestionat pel DOM | | Rendiment | Pot ser més lent per a formularis grans| Generalment més ràpid per a formularis grans | | Complexitat | Més complex a causa de la gestió de l'estat | Més senzill per a formularis bàsics | | Validació en temps real | Més fàcil d'implementar | Més difícil d'implementar | | Valors inicials | Establerts a través de l'estat de React| Establerts a través de defaultValue o defaultChecked|

Conclusió

Els components no controlats a React ofereixen una manera més senzilla i, de vegades, més eficient de gestionar les dades dels formularis aprofitant el DOM. Són particularment útils per a formularis bàsics on la validació en temps real i la gestió de l'estat no són crítiques. Entendre tant els components controlats com els no controlats et donarà la flexibilitat per triar l'enfocament més adequat per al teu cas d'ús específic.
