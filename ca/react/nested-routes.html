Introducció a les rutes niades

Les rutes niades et permeten definir rutes dins d'altres rutes, permetent estructures de navegació més complexes i jeràrquiques en les teves aplicacions React. Això és particularment útil per crear menús de navegació de diversos nivells o quan necessites renderitzar diferents components basats en el camí de l'URL.

Conceptes clau

  • Jerarquia de rutes: Organitzar rutes en una relació pare-fill.
  • Component Outlet: Un component marcador de posició utilitzat per renderitzar rutes filles.
  • Configuració de rutes: Definir rutes niades en la teva configuració de rutes.

Configuració de rutes niades

Configuració bàsica

Primer, assegura't que tens react-router-dom instal·lat en el teu projecte:

npm install react-router-dom

Estructura d'exemple

Considera una aplicació senzilla amb la següent estructura:

  • Home
  • About
  • Dashboard
    • Profile
    • Settings

Definint rutes

Aquí tens com pots definir aquestes rutes utilitzant react-router-dom:

import React from 'react';
import { BrowserRouter as Router, Routes, Route, Link, Outlet } from 'react-router-dom';

function App() {
  return (
    <Router>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
        <Link to="/dashboard">Dashboard</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/dashboard" element={<Dashboard />}>
          <Route path="profile" element={<Profile />} />
          <Route path="settings" element={<Settings />} />
        </Route>
      </Routes>
    </Router>
  );
}

function Home() {
  return <h2>Home</h2>;
}

function About() {
  return <h2>About</h2>;
}

function Dashboard() {
  return (
    <div>
      <h2>Dashboard</h2>
      <nav>
        <Link to="profile">Profile</Link>
        <Link to="settings">Settings</Link>
      </nav>
      <Outlet />
    </div>
  );
}

function Profile() {
  return <h2>Profile</h2>;
}

function Settings() {
  return <h2>Settings</h2>;
}

export default App;

Explicació

  • Router: Envolta tota l'aplicació per habilitar el routing.
  • Routes: Defineix la configuració de rutes.
  • Route: Especifica rutes individuals. L'atribut path defineix el camí de l'URL, i l'atribut element especifica el component a renderitzar.
  • Outlet: Utilitzat en el component Dashboard per renderitzar rutes niades (Profile i Settings).

Conceptes avançats

Rutes niades dinàmiques

També pots crear rutes niades dinàmiques on el camí inclou paràmetres. Per exemple:

<Route path="/dashboard/:userId" element={<Dashboard />}>
  <Route path="profile" element={<Profile />} />
  <Route path="settings" element={<Settings />} />
</Route>

Utilitzant el hook `useParams`

Per accedir als paràmetres de la ruta, utilitza el hook useParams de react-router-dom:

import { useParams } from 'react-router-dom';

function Dashboard() {
  let { userId } = useParams();
  return (
    <div>
      <h2>Dashboard for User {userId}</h2>
      <nav>
        <Link to="profile">Profile</Link>
        <Link to="settings">Settings</Link>
      </nav>
      <Outlet />
    </div>
  );
}

Protegint rutes niades

Pots protegir rutes niades utilitzant un component d'ordre superior o un component embolcall que comprova l'autenticació:

function ProtectedRoute({ children }) {
  const isAuthenticated = // lògica per comprovar si l'usuari està autenticat
  return isAuthenticated ? children : <Navigate to="/login" />;
}

// Ús
<Route path="/dashboard" element={<ProtectedRoute><Dashboard /></ProtectedRoute>}>
  <Route path="profile" element={<Profile />} />
  <Route path="settings" element={<Settings />} />
</Route>

Conclusió

Les rutes niades a React et permeten crear estructures de navegació complexes i jeràrquiques, fent que la teva aplicació sigui més organitzada i fàcil de navegar. Entenent i implementant rutes niades, pots millorar l'experiència de l'usuari i mantenir una arquitectura de routing neta. Recorda utilitzar el component Outlet per renderitzar rutes filles i considera tècniques avançades com rutes dinàmiques i protecció de rutes per aplicacions més robustes.
