Introducció a useEffect

El hook useEffect és una part fonamental dels components funcionals de React. Et permet realitzar efectes secundaris en els teus components, com ara obtenir dades, interactuar directament amb el DOM i configurar subscripcions.

Conceptes Clau

  • Efectes Secundaris: Operacions que afecten alguna cosa fora de l'abast de la funció que s'està executant.
  • Dependències: Variables o props de les quals depèn l'efecte. Quan aquestes canvien, l'efecte es torna a executar.
  • Neteja: Una funció que es pot retornar des de l'efecte per netejar recursos quan el component es desmunta o abans que l'efecte es torni a executar.

Ús Bàsic de useEffect

El hook useEffect pren dos arguments: una funció que conté l'efecte secundari i una matriu opcional de dependències.

import React, { useEffect, useState } from 'react';

function ExampleComponent() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `You clicked ${count} times`;
  }, [count]); // Només torna a executar l'efecte si count canvia

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

Explicació

  • Funció d'Efecte: La funció passada a useEffect s'executa després que el renderitzat es comprometi a la pantalla.
  • Matriu de Dependències: [count] assegura que l'efecte s'executi només quan count canviï.

useEffect sense Dependències

Si passes una matriu buida [] com a segon argument, l'efecte només s'executarà una vegada, similar a componentDidMount.

useEffect(() => {
  console.log('Component mounted');
}, []);

Explicació

  • Matriu Buida: L'efecte s'executa només una vegada després del renderitzat inicial.

useEffect amb Neteja

Per netejar recursos, retorna una funció des de l'efecte. Això és útil per a subscripcions o temporitzadors.

useEffect(() => {
  const timer = setInterval(() => {
    console.log('Tick');
  }, 1000);

  return () => clearInterval(timer); // Neteja el temporitzador
}, []);

Explicació

  • Funció de Neteja: La funció retornada s'executa quan el component es desmunta o abans que l'efecte es torni a executar.

Patrons Avançats de useEffect

Múltiples Hooks useEffect

Pots utilitzar múltiples hooks useEffect per separar preocupacions dins d'un component.

useEffect(() => {
  console.log('Effect 1');
}, [dependency1]);

useEffect(() => {
  console.log('Effect 2');
}, [dependency2]);

Efectes Condicionals

De vegades necessites executar un efecte condicionalment basat en certes condicions.

useEffect(() => {
  if (someCondition) {
    // Realitza l'efecte
  }
}, [someCondition]);

Errors Comuns i Millors Pràctiques

  • Evita Bucles Infinits: Assegura't que les dependències estiguin especificades correctament per evitar re-renderitzacions infinites.
  • Matrius de Dependències: Inclou sempre totes les dependències utilitzades dins de l'efecte.
  • Neteja: Neteja sempre les subscripcions i temporitzadors per evitar fuites de memòria.

Resum

El hook useEffect és una eina poderosa per gestionar efectes secundaris en components funcionals de React. Entenent el seu ús bàsic, la gestió de dependències i els mecanismes de neteja, pots controlar efectivament el cicle de vida dels teus components i assegurar-te que es comportin com s'espera.

Punts Clau

  • useEffect s'executa després de cada renderitzat per defecte.
  • Utilitza la matriu de dependències per controlar quan s'executa l'efecte.
  • Neteja sempre els recursos per evitar fuites de memòria.
  • Separa preocupacions utilitzant múltiples hooks useEffect quan sigui necessari.

Dominant useEffect, estaràs ben preparat per gestionar efectes secundaris en les teves aplicacions React, portant a un codi més robust i mantenible.
