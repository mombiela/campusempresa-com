Introducció

El procés ETL (Extract, Transform, Load) és fonamental en les arquitectures de dades modernes. Aquest procés permet la integració de dades provinents de diverses fonts, la seva transformació per complir amb els requisits del negoci i la seva càrrega en un sistema d'emmagatzematge, com un data warehouse. A continuació, es desglossen cadascuna de les fases del procés ETL.

Fase d'Extracció (Extract)

La fase d'extracció se centra en l'obtenció de dades des de diverses fonts. Aquestes fonts poden ser bases de dades relacionals, arxius plans, APIs, entre d'altres.

  • Objectiu: Obtenir dades de diferents fonts.
  • Tipus de fonts:
    • Bases de dades relacionals (SQL).
    • Arxius plans (CSV, JSON, XML).
    • APIs i serveis web.
    • Sistemes legacy.

Exemple de codi: Extracció des d'una base de dades SQL

import pandas as pd
import sqlalchemy

# Connexió a la base de dades
engine = sqlalchemy.create_engine('mysql+pymysql://user:password@host/dbname')

# Consulta SQL per extreure dades
query = "SELECT * FROM tabla_origen"
df = pd.read_sql(query, engine)

print(df.head())

Fase de Transformació (Transform)

La fase de transformació implica la neteja, normalització i estructuració de les dades perquè siguin útils i coherents.

  • Objectiu: Transformar les dades per complir amb els requisits del negoci.
  • Operacions comunes:
    • Neteja de dades (eliminació de duplicats, maneig de valors nuls).
    • Transformacions de format (conversió de tipus de dades).
    • Agregacions i càlculs.
    • Enriquiment de dades (unió de dades de diferents fonts).

Exemple de codi: Transformació de dades

# Eliminació de duplicats
df = df.drop_duplicates()

# Maneig de valors nuls
df['columna'] = df['columna'].fillna('valor_per_defecte')

# Conversió de tipus de dades
df['data'] = pd.to_datetime(df['data'])

# Agregació de dades
df_aggregated = df.groupby('categoria').agg({'vendes': 'sum'}).reset_index()

print(df_aggregated.head())

Fase de Càrrega (Load)

La fase de càrrega s'encarrega d'inserir les dades transformades en el sistema d'emmagatzematge final, com un data warehouse o un data lake.

  • Objectiu: Carregar les dades transformades en la destinació final.
  • Destinacions comunes:
    • Data warehouses (Redshift, BigQuery, Snowflake).
    • Data lakes (Amazon S3, Azure Data Lake).
    • Bases de dades relacionals.

Exemple de codi: Càrrega en un data warehouse

# Connexió al data warehouse
engine_dw = sqlalchemy.create_engine('redshift+psycopg2://user:password@host/dbname')

# Càrrega de dades en la taula destinació
df_aggregated.to_sql('tabla_destino', engine_dw, index=False, if_exists='replace')

print("Dades carregades exitosament.")

Conclusió

El procés ETL és essencial per a la integració i preparació de dades en les arquitectures de dades modernes. Comprendre cadascuna de les seves fases i com implementar-les permet als professionals de dades assegurar que la informació sigui precisa, coherent i útil per a l'anàlisi i la presa de decisions. A través d'exemples pràctics, hem vist com extreure dades de diverses fonts, transformar-les segons les necessitats del negoci i carregar-les en un sistema d'emmagatzematge final.
