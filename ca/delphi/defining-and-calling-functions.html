Definició i Crida de Funcions en Delphi/Object Pascal
Passatge de Paràmetres ►

Introducció

En Delphi/Object Pascal, les funcions són blocs fonamentals que et permeten encapsular codi en unitats reutilitzables. Aquest tema et guiarà a través de la definició i crida de funcions, començant pels conceptes bàsics i avançant cap a escenaris més complexos.

Definició i Crida Bàsica de Funcions

Definició d'una Funció Simple

En Delphi/Object Pascal, una funció es defineix utilitzant la paraula clau function. Aquí tens un exemple simple:

function AddNumbers(a, b: Integer): Integer;
begin
  Result := a + b;
end;
  • Paraula clau function: Indica l'inici de la definició d'una funció.
  • AddNumbers: El nom de la funció.
  • (a, b: Integer): Paràmetres de la funció, ambdós de tipus Integer.
  • : Integer: El tipus de retorn de la funció.
  • begin...end: El bloc on s'implementa la lògica de la funció.
  • Result: Un identificador especial utilitzat per emmagatzemar el valor de retorn de la funció.

Crida d'una Funció

Per cridar una funció, simplement utilitzes el seu nom seguit dels arguments entre parèntesis:

var
  sum: Integer;
begin
  sum := AddNumbers(5, 3);
  WriteLn('The sum is: ', sum);
end;
  • sum := AddNumbers(5, 3);: Crida la funció AddNumbers amb 5 i 3 com a arguments i emmagatzema el resultat en sum.

Funcions amb Diferents Tipus de Paràmetres

Passar Paràmetres per Valor

Per defecte, els paràmetres en Delphi/Object Pascal es passen per valor, és a dir, la funció rep una còpia dels arguments.

function Multiply(a, b: Integer): Integer;
begin
  Result := a * b;
end;

Passar Paràmetres per Referència

Per permetre que una funció modifiqui els arguments, pots passar els paràmetres per referència utilitzant la paraula clau var:

procedure Swap(var x, y: Integer);
var
  temp: Integer;
begin
  temp := x;
  x := y;
  y := temp;
end;
  • procedure: Utilitzat en lloc de function quan no es necessita cap valor de retorn.
  • var x, y: Integer: Paràmetres passats per referència.

Paràmetres per Defecte

Pots proporcionar valors per defecte per als paràmetres, fent-los opcionals quan es crida la funció:

function Greet(name: String = 'Guest'): String;
begin
  Result := 'Hello, ' + name + '!';
end;
  • name: String = 'Guest': name té un valor per defecte de 'Guest'.

Característiques Avançades de les Funcions

Sobrecàrrega de Funcions

Delphi/Object Pascal suporta la sobrecàrrega de funcions, permetent múltiples funcions amb el mateix nom però amb diferents paràmetres:

function Max(a, b: Integer): Integer; overload;
begin
  if a > b then
    Result := a
  else
    Result := b;
end;

function Max(a, b, c: Integer): Integer; overload;
begin
  Result := Max(Max(a, b), c);
end;
  • overload: Indica que la funció està sobrecarregada.

Funcions Recursives

Una funció pot cridar-se a si mateixa, el que es coneix com a recursió. Això és útil per a problemes que es poden descompondre en problemes més petits i similars:

function Factorial(n: Integer): Integer;
begin
  if n = 0 then
    Result := 1
  else
    Result := n * Factorial(n - 1);
end;
  • Factorial(n - 1): La funció es crida a si mateixa amb un valor decrementat de n.

Exercicis

  1. Creació Bàsica de Funcions: Defineix una funció SubtractNumbers que prengui dos enters i retorni la seva diferència. Crida aquesta funció i mostra el resultat.
  2. Paràmetres per Referència: Escriu un procediment Increment que prengui un enter per referència i incrementi el seu valor en 1. Prova aquest procediment.
  3. Paràmetres per Defecte: Crea una funció WelcomeMessage que prengui un paràmetre de cadena amb un valor per defecte de 'User' i retorni un missatge de benvinguda. Crida aquesta funció amb i sense un argument.
  4. Sobrecàrrega de Funcions: Implementa funcions sobrecarregades Min que trobin el mínim de dos i tres enters respectivament. Prova aquestes funcions.
  5. Funció Recursiva: Escriu una funció recursiva Fibonacci que retorni el n-èssim nombre de Fibonacci. Prova aquesta funció amb diferents valors de n.

Conclusió

Entendre com definir i cridar funcions en Delphi/Object Pascal és crucial per escriure codi modular i mantenible. Les funcions t'ajuden a encapsular la lògica, fent que els teus programes siguin més fàcils d'entendre i depurar. En dominar les característiques bàsiques i avançades de les funcions, podràs escriure codi més eficient i flexible.
