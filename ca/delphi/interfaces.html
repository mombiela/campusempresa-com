Introducció a les Interfícies

Les interfícies en Delphi/Object Pascal proporcionen una manera de definir mètodes que han de ser implementats per qualsevol classe que suporti la interfície. Són similars a les classes abstractes però són més flexibles i es poden utilitzar per aconseguir herència múltiple.

Conceptes Clau

  • Definició: Una interfície és un tipus de referència que defineix un conjunt de mètodes sense implementar-los.
  • Implementació: Les classes que implementen una interfície han de proporcionar implementacions concretes per a tots els seus mètodes.
  • Herència Múltiple: Una classe pot implementar múltiples interfícies, permetent una forma d'herència múltiple.

Definició d'Interfícies

Per definir una interfície en Delphi/Object Pascal, utilitza la paraula clau interface seguida de les signatures dels mètodes.

type
  IShape = interface
    ['{8D5B3E1A-2B9D-4B6A-8D3E-1A2B3C4D5E6F}']
    function GetArea: Double;
    function GetPerimeter: Double;
  end;

Explicació

  • GUID: La cadena entre claudàtors és un identificador únic global (GUID) que identifica de manera única la interfície.
  • Signatures de Mètodes: La interfície declara els mètodes GetArea i GetPerimeter sense proporcionar implementacions.

Implementació d'Interfícies

Una classe que implementa una interfície ha de proporcionar implementacions concretes per a tots els mètodes declarats en la interfície.

type
  TCircle = class(TInterfacedObject, IShape)
  private
    FRadius: Double;
  public
    constructor Create(ARadius: Double);
    function GetArea: Double;
    function GetPerimeter: Double;
  end;

constructor TCircle.Create(ARadius: Double);
begin
  FRadius := ARadius;
end;

function TCircle.GetArea: Double;
begin
  Result := Pi * FRadius * FRadius;
end;

function TCircle.GetPerimeter: Double;
begin
  Result := 2 * Pi * FRadius;
end;

Explicació

  • TInterfacedObject: Aquesta és una classe base que proporciona comptatge de referències i altres característiques necessàries per a la gestió d'interfícies.
  • Constructor: Inicialitza el radi del cercle.
  • GetArea i GetPerimeter: Proporcionen implementacions concretes per als mètodes declarats en la interfície IShape.

Ús d'Interfícies

Un cop una interfície està implementada, es pot utilitzar per interactuar amb objectes de manera polimòrfica.

var
  Shape: IShape;
  Circle: TCircle;
begin
  Circle := TCircle.Create(5.0);
  Shape := Circle;
  Writeln('Àrea: ', Shape.GetArea:0:2);
  Writeln('Perímetre: ', Shape.GetPerimeter:0:2);
end;

Explicació

  • Polimorfisme: La variable Shape de tipus IShape pot contenir qualsevol objecte que implementi la interfície IShape.
  • Crides a Mètodes: Els mètodes GetArea i GetPerimeter es criden a la referència de la interfície Shape.

Temes Avançats

Herència d'Interfícies

Les interfícies poden heretar d'altres interfícies, permetent jerarquies més complexes.

type
  IColoredShape = interface(IShape)
    function GetColor: TColor;
  end;

Explicació

  • Herència: IColoredShape hereta de IShape i afegeix un mètode addicional GetColor.

Comptatge de Referències i Gestió de Memòria

Delphi utilitza el comptatge de referències per gestionar automàticament la vida útil de les instàncies d'interfícies.

  • Comptatge de Referències: Cada referència d'interfície augmenta el comptatge de referències, i quan el comptatge arriba a zero, l'objecte es destrueix automàticament.
  • TInterfacedObject: Proporciona la implementació necessària per al comptatge de referències.

Conclusió

Les interfícies en Delphi/Object Pascal són una característica potent que permet un codi flexible i reutilitzable. Permeten polimorfisme, herència múltiple i una separació neta de les definicions de mètodes de les implementacions. Entenent i utilitzant les interfícies, pots crear aplicacions més modulars i mantenibles.
