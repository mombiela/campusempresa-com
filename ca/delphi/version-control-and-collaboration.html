Introducció al Control de Versions

El control de versions és un sistema que registra els canvis en un fitxer o conjunt de fitxers al llarg del temps, de manera que puguis recuperar versions específiques més tard. És essencial per al desenvolupament col·laboratiu de programari i per gestionar els canvis de codi de manera eficient.

Conceptes Clau

  • Repositori: Una ubicació d'emmagatzematge per a paquets de programari, sovint allotjada en plataformes com GitHub, GitLab o Bitbucket.
  • Commit: Una instantània del teu repositori en un moment específic.
  • Branca: Una versió paral·lela del repositori. Permet treballar en diferents característiques o correccions de manera independent.
  • Merge: Combinació de canvis de diferents branques.
  • Conflicte: Quan els canvis de diferents branques es contradiuen, requerint una resolució manual.

Configuració del Control de Versions amb Git

Git és el sistema de control de versions més utilitzat. Aquí tens com configurar-lo per a un projecte Delphi/Object Pascal.

Instal·lació de Git

  1. Descarrega Git des de git-scm.com.
  2. Segueix les instruccions d'instal·lació per al teu sistema operatiu.

Inicialització d'un Repositori

  1. Obre la teva interfície de línia de comandes (CLI).
  2. Navega al directori del teu projecte Delphi.
  3. Inicialitza un repositori Git: 
    git init

Comandes Bàsiques de Git

  • Afegir Fitxers: Preparar fitxers per al commit. 
    git add <filename>
  • Fer Commit dels Canvis: Guardar una instantània de l'estat actual. 
    git commit -m "El teu missatge de commit"
  • Veure l'Estat: Comprovar l'estat del teu repositori. 
    git status
  • Veure l'Historial: Veure l'historial de commits. 
    git log

Branques i Fusió

Les branques permeten treballar en diferents parts d'un projecte simultàniament.

Crear i Canviar de Branca

  • Crear una Branca: 
    git branch <branch-name>
  • Canviar a una Branca: 
    git checkout <branch-name>

Fusionar Branques

  • Fusionar una Branca a la Branca Actual: 
    git merge <branch-name>

Resoldre Conflictes

Quan es fusionen branques, poden sorgir conflictes. Git marcarà les àrees conflictives en els fitxers, i hauràs de resoldre-les manualment.

Col·laboració amb Repositoris Remots

Els repositoris remots permeten que múltiples desenvolupadors col·laborin en el mateix projecte.

Afegir un Repositori Remot

  1. Crea un repositori en una plataforma com GitHub.
  2. Afegeix el repositori remot: 
    git remote add origin <repository-url>

Empènyer i Estirar Canvis

  • Empènyer Canvis al Remot: 
    git push origin <branch-name>
  • Estirar Canvis del Remot: 
    git pull origin <branch-name>

Ús del Control de Versions en Delphi IDE

Delphi IDE suporta la integració del control de versions, facilitant la gestió dels teus repositoris.

Configuració del Control de Versions en Delphi

  1. Obre Delphi IDE.
  2. Ves a Eines > Opcions > Control de Versions.
  3. Configura els paràmetres per a Git (o altres sistemes de control de versions).

Fer Commit dels Canvis des de Delphi IDE

  1. Fes clic dret sobre el teu projecte al Gestor de Projectes.
  2. Selecciona Control de Versions > Commit.
  3. Introdueix un missatge de commit i selecciona els fitxers per fer commit.
  4. Fes clic a Commit.

Millors Pràctiques per al Control de Versions

  • Fes Commit Sovint: Fes commits petits i freqüents per mantenir un seguiment dels canvis.
  • Escriu Missatges de Commit Significatius: Descriu quins canvis s'han fet i per què.
  • Utilitza Branques: Aïlla noves característiques o correccions d'errors en branques separades.
  • Revisa el Codi: Utilitza sol·licituds de pull per revisar el codi abans de fusionar-lo.

Conclusió

El control de versions és una habilitat crítica per a qualsevol desenvolupador, permetent una col·laboració eficient i la gestió dels canvis de codi. Comprenent i utilitzant Git, les branques, les fusions i els repositoris remots, pots millorar significativament el teu flux de treball de desenvolupament en projectes Delphi/Object Pascal. Integrar el control de versions en el Delphi IDE simplifica encara més el procés, facilitant la gestió del teu codi directament dins del teu entorn de desenvolupament.
