Organització i Modularització del Codi en Delphi/Object Pascal

Introducció

L'organització i la modularització del codi són crucials per escriure codi mantenible, escalable i llegible. En Delphi/Object Pascal, aquests conceptes ajuden a gestionar la complexitat descomponent programes grans en unitats més petites i manejables.

Conceptes Clau

  • Unitats: Els blocs fonamentals per a la modularització en Delphi.
  • Espais de noms: S'utilitzen per evitar conflictes de noms i organitzar les unitats lògicament.
  • Seccions d'interfície i implementació: Separen la declaració de tipus, variables, constants i procediments de la seva implementació.
  • Clàusula Uses: Especifica les dependències entre unitats.
  • Encapsulació: Amaga l'estat intern i la funcionalitat d'una unitat, exposant només el que és necessari.

Unitats

Les unitats són la manera principal de modularitzar el codi en Delphi. Cada unitat normalment correspon a un únic fitxer font amb una extensió .pas.

Estructura d'una Unitat

Una unitat es divideix en dues seccions principals: la secció d'interfície i la secció d'implementació.

unit MyUnit;

interface

uses
  SysUtils, Classes;

type
  TMyClass = class
  private
    FValue: Integer;
  public
    procedure SetValue(AValue: Integer);
    function GetValue: Integer;
  end;

implementation

procedure TMyClass.SetValue(AValue: Integer);
begin
  FValue := AValue;
end;

function TMyClass.GetValue: Integer;
begin
  Result := FValue;
end;

end.
  • Secció d'Interfície: Conté declaracions de tipus, variables, constants i procediments que són accessibles des d'altres unitats.
  • Secció d'Implementació: Conté el codi real per als procediments i funcions declarats a la secció d'interfície.

Espais de Noms

Els espais de noms ajuden a organitzar les unitats i evitar conflictes de noms. Són particularment útils en projectes grans.

unit MyProject.Utilities.StringUtils;

interface

// Secció d'interfície

implementation

// Secció d'implementació

end.

Clàusula Uses

La clàusula uses s'utilitza per incloure altres unitats i fer disponibles les seves declaracions públiques.

unit MainUnit;

interface

uses
  MyUnit, SysUtils;

implementation

procedure DoSomething;
var
  MyClassInstance: TMyClass;
begin
  MyClassInstance := TMyClass.Create;
  try
    MyClassInstance.SetValue(10);
    Writeln(MyClassInstance.GetValue);
  finally
    MyClassInstance.Free;
  end;
end;

end.

Encapsulació

L'encapsulació és la pràctica d'amagar l'estat intern i la funcionalitat d'una unitat, exposant només el que és necessari. Això s'aconsegueix utilitzant especificadors d'accés com private, protected, public i published.

type
  TMyClass = class
  private
    FValue: Integer;
  public
    procedure SetValue(AValue: Integer);
    function GetValue: Integer;
  end;
  • Private: Accessible només dins de la mateixa unitat.
  • Protected: Accessible dins de la mateixa unitat i per classes descendents.
  • Public: Accessible des de qualsevol unitat.
  • Published: Similar a public, però també disponible per a RTTI (Informació de Tipus en Temps d'Execució).

Exemple Pràctic

Creem una aplicació senzilla que utilitza la modularització per separar les preocupacions.

Unitat: MathOperations

unit MathOperations;

interface

type
  TMathOperations = class
  public
    function Add(A, B: Integer): Integer;
    function Subtract(A, B: Integer): Integer;
  end;

implementation

function TMathOperations.Add(A, B: Integer): Integer;
begin
  Result := A + B;
end;

function TMathOperations.Subtract(A, B: Integer): Integer;
begin
  Result := A - B;
end;

end.

Unitat: MainApp

unit MainApp;

interface

uses
  MathOperations, SysUtils;

procedure RunApp;

implementation

procedure RunApp;
var
  MathOps: TMathOperations;
begin
  MathOps := TMathOperations.Create;
  try
    Writeln('5 + 3 = ', MathOps.Add(5, 3));
    Writeln('5 - 3 = ', MathOps.Subtract(5, 3));
  finally
    MathOps.Free;
  end;
end;

end.

Programa: Project1

program Project1;

uses
  MainApp;

begin
  RunApp;
end.

Conclusió

Organitzant el codi en unitats, utilitzant espais de noms i aplicant l'encapsulació, es poden crear aplicacions Delphi modulars i mantenibles. Aquest enfocament no només fa que el codi sigui més fàcil d'entendre i gestionar, sinó que també millora la reutilització i l'escalabilitat.
