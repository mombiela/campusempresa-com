Encapsulació en Delphi/Object Pascal
Interfícies ►

L'encapsulació és un concepte fonamental en la Programació Orientada a Objectes (POO) que ajuda a agrupar les dades (atributs) i els mètodes (funcions o procediments) que operen sobre les dades en una sola unitat, coneguda com a classe. També restringeix l'accés directe a alguns dels components de l'objecte, cosa que és un mitjà per prevenir interferències accidentals i l'ús indegut de les dades.

Conceptes Clau de l'Encapsulació

  • Classe: Un plànol per crear objectes. Encapsula dades per a l'objecte i mètodes per manipular aquestes dades.
  • Objecte: Una instància d'una classe.
  • Especificadors d'Accés: Paraules clau que estableixen l'accessibilitat de classes, mètodes i altres membres. Els especificadors d'accés comuns en Delphi/Object Pascal són private, protected, public i published.

Especificadors d'Accés

  • Private: Els membres declarats com a privats són accessibles només dins de la mateixa unitat.
  • Protected: Els membres declarats com a protegits són accessibles dins de la mateixa unitat i per classes descendents.
  • Public: Els membres declarats com a públics són accessibles des de qualsevol codi que pugui veure la classe.
  • Published: Similar a public, però també permet que les propietats siguin emmagatzemades i inspeccionades en temps d'execució.

Exemple d'Encapsulació en Delphi/Object Pascal

Creem una classe simple per demostrar l'encapsulació:

unit UPerson;

interface

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
    procedure SetName(const Value: string);
    procedure SetAge(Value: Integer);
  public
    constructor Create(Name: string; Age: Integer);
    function GetDetails: string;
    property Name: string read FName write SetName;
    property Age: Integer read FAge write SetAge;
  end;

implementation

{ TPerson }

constructor TPerson.Create(Name: string; Age: Integer);
begin
  FName := Name;
  FAge := Age;
end;

procedure TPerson.SetName(const Value: string);
begin
  if Value <> '' then
    FName := Value;
end;

procedure TPerson.SetAge(Value: Integer);
begin
  if (Value > 0) and (Value < 150) then
    FAge := Value;
end;

function TPerson.GetDetails: string;
begin
  Result := Format('Name: %s, Age: %d', [FName, FAge]);
end;

end.

Explicació

  • Declaració de la Classe: La classe TPerson es declara amb camps privats FName i FAge per emmagatzemar el nom i l'edat d'una persona.
  • Constructor: El constructor Create inicialitza els camps.
  • Mètodes de Propietat: SetName i SetAge són mètodes privats per establir els valors de FName i FAge amb validació.
  • Propietats: Les propietats Name i Age proporcionen accés controlat als camps privats.
  • Mètode Públic: GetDetails és un mètode públic per obtenir els detalls de la persona.

Exercicis

  1. Crear una Nova Classe: Crear una classe TCar amb camps privats per Make, Model i Year. Implementar propietats i mètodes per encapsular aquests camps.
  2. Modificar la Classe: Afegir validació al mètode SetYear per assegurar que l'any estigui dins d'un rang raonable (per exemple, 1886 fins a l'any actual).
  3. Utilitzar la Classe: Escriure un programa que creï una instància de TCar, estableixi les seves propietats i imprimeixi els detalls.

Conclusió

L'encapsulació és una característica poderosa en Delphi/Object Pascal que ajuda a organitzar el codi, protegir les dades i millorar la mantenibilitat. Utilitzant classes i especificadors d'accés, pots controlar com s'accedeixen i modifiquen les dades, assegurant que els teus objectes s'utilitzin correctament i de manera segura.
