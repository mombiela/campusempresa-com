Introducció

La injecció és una de les vulnerabilitats més crítiques i comunes en aplicacions web. Ocorre quan un atacant pot enviar dades no fiables a un intèrpret com a part d'un comandament o consulta. Les dades malicioses poden enganyar l'intèrpret per executar comandaments no intencionats o accedir a dades sense autorització.

Tipus Comuns d'Injecció

  • SQL Injection (SQLi): Injecció de comandaments SQL en consultes a bases de dades.
  • Command Injection: Injecció de comandaments del sistema operatiu.
  • LDAP Injection: Injecció de comandaments en consultes LDAP.
  • XPath Injection: Injecció en consultes XPath per a XML.
  • NoSQL Injection: Injecció en bases de dades NoSQL.

Exemple de SQL Injection

Codi Vulnerable

# Exemple en Python amb una consulta SQL vulnerable
import sqlite3

def get_user_info(user_id):
    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()
    query = f"SELECT * FROM users WHERE id = {user_id}"
    cursor.execute(query)
    return cursor.fetchall()

Explicació

En l'exemple anterior, la variable user_id s'insereix directament en la consulta SQL sense cap validació o sanitització. Un atacant podria explotar aquesta vulnerabilitat passant un valor maliciós com 1 OR 1=1, el que resultaria en l'execució de la consulta:

SELECT * FROM users WHERE id = 1 OR 1=1

Això retornaria tots els registres de la taula users.

Prevenció d'Injecció

Ús de Consultes Preparades

Les consultes preparades (també conegudes com a consultes parametritzades) són una tècnica efectiva per prevenir injeccions SQL. Aquí hi ha un exemple de com modificar el codi anterior per utilitzar consultes preparades:

# Exemple en Python utilitzant consultes preparades
import sqlite3

def get_user_info(user_id):
    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()
    query = "SELECT * FROM users WHERE id = ?"
    cursor.execute(query, (user_id,))
    return cursor.fetchall()

Validació i Sanitització d'Entrades

  • Validació: Assegura't que les dades d'entrada compleixin amb els formats esperats abans de processar-les.
  • Sanitització: Neteja les dades d'entrada per eliminar caràcters perillosos.

Ús d'ORM (Object-Relational Mapping)

Els ORM poden ajudar a prevenir injeccions SQL en abstraure les consultes SQL i manejar automàticament la sanitització d'entrades.

# Exemple en Python utilitzant SQLAlchemy ORM
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from models import User

engine = create_engine('sqlite:///example.db')
Session = sessionmaker(bind=engine)
session = Session()

def get_user_info(user_id):
    user = session.query(User).filter(User.id == user_id).first()
    return user

Exercicis Pràctics

  1. Identificació d'Injecció: Donat un fragment de codi, identifica si és vulnerable a injecció i explica per què.
  2. Correcció de Vulnerabilitats: Modifica un codi vulnerable perquè sigui segur utilitzant consultes preparades.
  3. Implementació de Validació: Implementa funcions de validació i sanitització per a entrades d'usuari en un formulari web.

Conclusió

La injecció és una vulnerabilitat crítica que pot tenir conseqüències devastadores per a la seguretat d'una aplicació. És essencial que els desenvolupadors comprenguin com identificar i mitigar aquestes vulnerabilitats mitjançant l'ús de consultes preparades, validació i sanitització d'entrades, i l'ús d'ORM. La pràctica constant i la revisió de codi són claus per mantenir aplicacions segures.
