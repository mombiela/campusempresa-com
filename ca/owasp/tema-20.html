Introducció

OWASP Dependency-Check és una eina d'anàlisi de composició de programari (SCA) que identifica les vulnerabilitats conegudes en les dependències d'un projecte. És crucial per mantenir la seguretat de les aplicacions, ja que moltes vulnerabilitats poden residir en biblioteques i components de tercers.

Objectius del Tema

  • Entendre què és OWASP Dependency-Check.
  • Aprendre a instal·lar i configurar OWASP Dependency-Check.
  • Executar anàlisis de dependències i comprendre els resultats.
  • Integrar OWASP Dependency-Check en un flux de treball CI/CD.

Què és OWASP Dependency-Check?

OWASP Dependency-Check és una eina de codi obert que ajuda a identificar vulnerabilitats conegudes en les dependències d'un projecte. Funciona analitzant els fitxers de configuració de dependències (com pom.xml per a Maven o package.json per a Node.js) i comparant les versions de les biblioteques amb una base de dades de vulnerabilitats conegudes.

Característiques Principals

  • Anàlisi de dependències: Escaneja fitxers de configuració per identificar biblioteques i versions.
  • Base de dades de vulnerabilitats: Utilitza bases de dades com NVD (National Vulnerability Database) per identificar problemes coneguts.
  • Informes detallats: Genera informes que detallen les vulnerabilitats trobades i la seva gravetat.
  • Integració CI/CD: Es pot integrar fàcilment en pipelines de CI/CD per a anàlisis automatitzats.

Instal·lació i Configuració

Requisits Previs

  • Java JDK 8 o superior.
  • Eina de construcció com Maven, Gradle, o Node.js.

Instal·lació

Usant Maven

Per a projectes basats en Maven, afegeix el següent plugin en el fitxer pom.xml:

<build>
    <plugins>
        <plugin>
            <groupId>org.owasp</groupId>
            <artifactId>dependency-check-maven</artifactId>
            <version>6.5.0</version>
            <executions>
                <execution>
                    <goals>
                        <goal>check</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

Usant Gradle

Per a projectes basats en Gradle, afegeix el següent plugin en el fitxer build.gradle:

plugins {
    id "org.owasp.dependencycheck" version "6.5.0"
}

dependencyCheck {
    failBuildOnCVSS = 7
}

Usant Node.js

Per a projectes basats en Node.js, instal·la el paquet dependency-check:

npm install -g dependency-check

Executar Anàlisi de Dependències

Maven

Per executar l'anàlisi en un projecte Maven, utilitza el següent comandament:

mvn dependency-check:check

Gradle

Per executar l'anàlisi en un projecte Gradle, utilitza el següent comandament:

./gradlew dependencyCheckAnalyze

Node.js

Per executar l'anàlisi en un projecte Node.js, utilitza el següent comandament:

dependency-check --project <project-name> --scan <path-to-project>

Comprendre els Resultats

Després d'executar l'anàlisi, OWASP Dependency-Check generarà un informe detallat. Aquest informe generalment inclou:

  • Llista de dependències: Totes les dependències identificades en el projecte.
  • Vulnerabilitats trobades: Detalls de les vulnerabilitats, incloent CVE (Common Vulnerabilities and Exposures) ID, descripció, i severitat.
  • Recomanacions: Suggeriments per mitigar les vulnerabilitats, com actualitzar a una versió més segura de la biblioteca.

Exemple d'Informe

Dependency: com.fasterxml.jackson.core:jackson-databind:2.9.8
Vulnerability: CVE-2018-19360
Severity: HIGH
Description: FasterXML jackson-databind 2.x before 2.9.9 might allow remote attackers to conduct server-side request forgery (SSRF) attacks.
Recommendation: Update to version 2.9.9 or later.

Integració en CI/CD

Integrar OWASP Dependency-Check en un pipeline de CI/CD permet automatitzar l'anàlisi de dependències i garantir que les vulnerabilitats siguin detectades i abordades de manera contínua.

Jenkins

Per integrar en Jenkins, afegeix un pas d'execució de shell en el teu pipeline:

mvn dependency-check:check

GitLab CI

Per integrar en GitLab CI, afegeix el següent script en el teu fitxer .gitlab-ci.yml:

dependency-check:
  image: maven:3.6.3-jdk-8
  script:
    - mvn dependency-check:check
  artifacts:
    paths:
      - target/dependency-check-report.html

Conclusió

OWASP Dependency-Check és una eina essencial per a qualsevol equip de desenvolupament que utilitzi dependències de tercers. En integrar aquesta eina en el flux de treball de desenvolupament, es poden identificar i mitigar vulnerabilitats conegudes de manera proactiva, millorant així la seguretat general de les aplicacions.

Recorda sempre mantenir les teves dependències actualitzades i realitzar anàlisis regulars per assegurar que el teu projecte estigui protegit contra vulnerabilitats conegudes.
