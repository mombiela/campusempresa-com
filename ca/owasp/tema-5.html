Introducció

En el context de la seguretat web, l'autenticació i l'autorització són dos conceptes fonamentals que garanteixen que només els usuaris legítims puguin accedir als recursos i que aquests usuaris només puguin realitzar accions permeses. Aquest tema és crucial en el marc d'OWASP (Open Web Application Security Project), que se centra a millorar la seguretat del programari.

Autenticació

L'autenticació és el procés de verificar la identitat d'un usuari o sistema. En altres paraules, és el mecanisme que assegura que algú és qui diu ser.

Conceptes Clau d'Autenticació

  • Factors d'Autenticació:

    • Alguna cosa que saps: Contrasenyes, PINs.
    • Alguna cosa que tens: Targetes intel·ligents, tokens.
    • Alguna cosa que ets: Empremtes dactilars, reconeixement facial.

  • Mètodes d'Autenticació:

    • Autenticació Bàsica: Ús de noms d'usuari i contrasenyes.
    • Autenticació Multifactor (MFA): Combinació de dos o més factors d'autenticació.
    • OAuth: Protocol d'autorització que permet a les aplicacions obtenir accés limitat a les dades dels usuaris.

Exemple de Codi: Autenticació Bàsica en Python

def authenticate(username, password):
    stored_username = "user1"
    stored_password = "password123"
    
    if username == stored_username and password == stored_password:
        return "Autenticació exitosa"
    else:
        return "Autenticació fallida"

# Exemple d'ús
print(authenticate("user1", "password123"))

Autorització

L'autorització és el procés de determinar si un usuari autenticat té permís per accedir a un recurs o realitzar una acció específica.

Conceptes Clau d'Autorització

  • Rols i Permisos:

    • Rols: Conjunt de permisos assignats a un usuari o grup d'usuaris.
    • Permisos: Accions específiques que un usuari pot realitzar.

  • Models de Control d'Accés:

    • Control d'Accés Basat en Rols (RBAC): Permisos assignats a rols i rols assignats a usuaris.
    • Control d'Accés Basat en Atributs (ABAC): Permisos basats en atributs d'usuari, recurs i entorn.

Exemple de Codi: Autorització Basada en Rols en Python

roles_permissions = {
    "admin": ["read", "write", "delete"],
    "user": ["read"]
}

def authorize(role, action):
    if action in roles_permissions.get(role, []):
        return "Autorització concedida"
    else:
        return "Autorització denegada"

# Exemple d'ús
print(authorize("admin", "delete"))
print(authorize("user", "delete"))

Comparació d'Autenticació i Autorització

| Característica | Autenticació | Autorització | |----------------------|----------------------------------------|----------------------------------------| | Propòsit | Verificar la identitat de l'usuari | Determinar els permisos de l'usuari | | Quan ocorre | Abans de l'autorització | Després de l'autenticació | | Exemple de mètode | Contrasenya, MFA | RBAC, ABAC | | Resultat | Usuari autenticat | Accés permès o denegat |

Conclusió

L'autenticació i l'autorització són components essencials de la seguretat en aplicacions web. L'autenticació assegura que els usuaris són qui diuen ser, mentre que l'autorització determina quines accions poden realitzar aquests usuaris. Comprendre i aplicar correctament aquests conceptes és crucial per protegir els recursos i dades d'una aplicació.

En el context d'OWASP, implementar pràctiques sòlides d'autenticació i autorització pot ajudar a mitigar molts riscos de seguretat comuns, com l'accés no autoritzat i l'exposició de dades sensibles.
