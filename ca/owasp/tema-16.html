Introducció

Els errors en la integritat del programari i les dades es refereixen a problemes que sorgeixen quan les aplicacions no protegeixen adequadament la integritat del codi i les dades crítiques. Això pot portar a l'execució de codi no autoritzat, la manipulació de dades i altres comportaments no desitjats.

Conceptes Clau

  • Integritat del Programari: Garantir que el programari no ha estat alterat de manera no autoritzada.
  • Integritat de les Dades: Assegurar que les dades no han estat modificades o corrompudes de manera no autoritzada.
  • Signatures Digitals i Hashing: Mètodes per verificar la integritat del programari i les dades.
  • Control de Versions: Ús de sistemes de control de versions per rastrejar canvis i mantenir la integritat del codi.
  • Validació d'Entrades: Assegurar que les dades ingressades siguin vàlides i segures.

Exemples d'Errors Comuns

  • Actualitzacions de Programari No Segures: Descàrrega i instal·lació d'actualitzacions de programari sense verificar la seva autenticitat.
  • Manipulació de Dades en Trànsit: Interceptació i modificació de dades mentre es transmeten entre sistemes.
  • Dependències No Confiables: Ús de biblioteques o components de tercers sense verificar la seva integritat.

Exemples de Codi

Verificació de Signatures Digitals

A continuació, es mostra un exemple en Python de com verificar la signatura digital d'un fitxer utilitzant la biblioteca cryptography.

from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.asymmetric import padding
from cryptography.hazmat.primitives.serialization import load_pem_public_key

def verify_signature(public_key_pem, signature, data):
    public_key = load_pem_public_key(public_key_pem)
    try:
        public_key.verify(
            signature,
            data,
            padding.PKCS1v15(),
            hashes.SHA256()
        )
        print("La signatura és vàlida.")
    except:
        print("La signatura no és vàlida.")

# Exemple d'ús
public_key_pem = b"""-----BEGIN PUBLIC KEY-----
...
-----END PUBLIC KEY-----"""
signature = b"..."
data = b"Dades importants"

verify_signature(public_key_pem, signature, data)

Validació d'Entrades

Un exemple en JavaScript per validar entrades d'usuari i prevenir la injecció de codi.

function validateInput(input) {
    const regex = /^[a-zA-Z0-9_]+$/;
    if (regex.test(input)) {
        console.log("Entrada vàlida.");
    } else {
        console.log("Entrada no vàlida.");
    }
}

// Exemple d'ús
validateInput("usuari123"); // Entrada vàlida.
validateInput("usuari<script>alert('hack');</script>"); // Entrada no vàlida.

Bones Pràctiques

  • Signar i Verificar Codi: Utilitzar signatures digitals per assegurar que el codi no ha estat alterat.
  • Control de Versions: Implementar sistemes de control de versions per rastrejar canvis en el codi.
  • Validació d'Entrades: Validar totes les entrades d'usuari per prevenir injeccions i altres manipulacions.
  • Monitoratge i Auditoria: Implementar sistemes de monitoratge i auditoria per detectar i respondre a canvis no autoritzats.

Conclusió

Els errors en la integritat del programari i les dades poden tenir conseqüències greus, incloent l'execució de codi no autoritzat i la manipulació de dades crítiques. És essencial implementar pràctiques robustes de seguretat, com la verificació de signatures digitals, l'ús de sistemes de control de versions i la validació d'entrades, per protegir la integritat del programari i les dades.
