Introducció

El control d'accés és un mecanisme fonamental en la seguretat d'aplicacions web. Es refereix a les polítiques i procediments que determinen qui pot accedir a quins recursos en una aplicació. El problema de "Control d'Accés Trencat" ocorre quan aquestes polítiques no s'implementen correctament, permetent a usuaris no autoritzats accedir a recursos restringits.

Conceptes Clau

  • Control d'Accés: Conjunt de mecanismes que restringeixen l'accés a recursos a usuaris autoritzats.
  • Autenticació vs. Autorització:
    • Autenticació: Verificació de la identitat de l'usuari.
    • Autorització: Determinació dels permisos de l'usuari autenticat.
  • Principi de Menor Privilegi: Els usuaris han de tenir només els permisos necessaris per realitzar les seves tasques.

Tipus Comuns de Fallades en el Control d'Accés

  • Bypass d'Autorització: Accés a recursos sense passar pels controls d'autorització.
  • Escalada de Privilegis: Un usuari obté permisos més alts dels que hauria de tenir.
  • Accés Directe a Objectes: Accés a recursos mitjançant la manipulació d'URLs, paràmetres o formularis.

Exemples de Control d'Accés Trencat

Exemple 1: Accés Directe a Objectes

Un usuari pot accedir a les dades d'un altre usuari manipulant la URL.

# URL original per accedir a les dades de l'usuari
http://example.com/user/123

# Manipulació de la URL per accedir a les dades d'un altre usuari
http://example.com/user/124

Exemple 2: Escalada de Privilegis

Un usuari regular intenta accedir a una pàgina d'administració.

# URL per a usuaris regulars
http://example.com/user/dashboard

# URL per a administradors
http://example.com/admin/dashboard

Si el control d'accés no està ben implementat, un usuari regular podria accedir a la pàgina d'administració.

Millors Pràctiques per Prevenir el Control d'Accés Trencat

  • Implementar controls d'accés en el servidor: No confiar en controls d'accés implementats només en el client.
  • Utilitzar llistes de control d'accés (ACL): Definir clarament quins usuaris tenen accés a quins recursos.
  • Revisar i auditar permisos regularment: Assegurar-se que els permisos assignats siguin els correctes.
  • Principi de Menor Privilegi: Assignar només els permisos necessaris a cada usuari.
  • Validar l'accés en cada sol·licitud: No assumir que un usuari té accés a un recurs basat en una sol·licitud anterior.

Exercici Pràctic

Exercici 1: Identificació de Fallades en el Control d'Accés

Donat el següent codi, identifiqui i corregeixi els problemes de control d'accés.

# Codi original
def get_user_data(user_id):
    user = db.get_user(user_id)
    return user.data

# Codi corregit
def get_user_data(current_user, user_id):
    if current_user.id != user_id i not current_user.is_admin:
        raise PermissionError("Access Denied")
    user = db.get_user(user_id)
    return user.data

Exercici 2: Implementació de Control d'Accés

Implementi un control d'accés bàsic per a una aplicació web en Python usant Flask.

from flask import Flask, request, jsonify

app = Flask(__name__)

# Simulació de base de dades d'usuaris
users_db = {
    1: {"id": 1, "name": "Alice", "role": "user"},
    2: {"id": 2, "name": "Bob", "role": "admin"}
}

def get_current_user():
    # Simulació d'autenticació
    user_id = int(request.headers.get('X-User-ID'))
    return users_db.get(user_id)

@app.route('/admin', methods=['GET'])
def admin_dashboard():
    current_user = get_current_user()
    if current_user['role'] != 'admin':
        return jsonify({"error": "Access Denied"}), 403
    return jsonify({"message": "Welcome to the admin dashboard"})

if __name__ == '__main__':
    app.run(debug=True)

Conclusió

El control d'accés és una part crítica de la seguretat de les aplicacions web. Les fallades en el control d'accés poden permetre a usuaris no autoritzats accedir a recursos restringits, la qual cosa pot tenir conseqüències greus. Implementar controls d'accés robustos i seguir les millors pràctiques pot ajudar a prevenir aquests problemes i protegir la integritat i confidencialitat de les dades.
