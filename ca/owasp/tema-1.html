Introducció a OWASP

OWASP, que significa Open Web Application Security Project, és una organització sense ànim de lucre dedicada a millorar la seguretat del programari. Fundada el 2001, OWASP proporciona recursos gratuïts i oberts que ajuden a individus i organitzacions a desenvolupar, adquirir i mantenir aplicacions segures.

Objectius d'OWASP

OWASP té diversos objectius clau que guien les seves activitats i projectes:

  • Conscienciació sobre la seguretat: Educar desenvolupadors, arquitectes i professionals de TI sobre les millors pràctiques en seguretat d'aplicacions.
  • Eines i recursos: Proveir eines, documentació i estàndards que facilitin la implementació de pràctiques de seguretat.
  • Investigació i desenvolupament: Fomentar la investigació en noves tècniques i metodologies per millorar la seguretat de les aplicacions.

Principals Projectes d'OWASP

OWASP és conegut per una sèrie de projectes que han tingut un impacte significatiu en la comunitat de seguretat d'aplicacions:

  • OWASP Top Ten: Una llista de les deu vulnerabilitats més crítiques en aplicacions web. És una referència essencial per a qualsevol professional de seguretat.
  • OWASP ZAP (Zed Attack Proxy): Una eina de prova de penetració per trobar vulnerabilitats en aplicacions web.
  • OWASP ASVS (Application Security Verification Standard): Un marc per verificar la seguretat de les aplicacions.

OWASP Top Ten

L'OWASP Top Ten és un dels projectes més influents d'OWASP. A continuació, es presenta una llista de les vulnerabilitats incloses en la versió més recent:

  1. Injecció: Inclou SQL, NoSQL, OS i LDAP injeccions.
  2. Autenticació Trencada: Problemes amb l'autenticació i gestió de sessions.
  3. Exposició de Dades Sensibles: Errors en la protecció de dades sensibles.
  4. Entitats Externes XML (XXE): Vulnerabilitats relacionades amb la inclusió d'entitats externes en XML.
  5. Control d'Accés Trencat: Errors en la implementació de controls d'accés.
  6. Configuració de Seguretat Incorrecta: Configuracions de seguretat mal implementades.
  7. Cross-Site Scripting (XSS): Injecció de scripts en llocs web.
  8. Deserialització Insegura: Problemes amb la deserialització de dades no fiables.
  9. Ús de Components amb Vulnerabilitats Conegudes: Dependències amb vulnerabilitats conegudes.
  10. Registre i Monitoratge Insuficients: Falta de registre i monitoratge adequats.

Exemple de Vulnerabilitat: Injecció SQL

Per il·lustrar una de les vulnerabilitats de l'OWASP Top Ten, vegem un exemple d'injecció SQL:

-- Consulta SQL vulnerable
SELECT * FROM usuaris WHERE nom = 'admin' AND contrasenya = '1234';

En aquest cas, si un atacant introdueix ' OR '1'='1 com a nom d'usuari i qualsevol contrasenya, la consulta es converteix en:

SELECT * FROM usuaris WHERE nom = '' OR '1'='1' AND contrasenya = 'qualsevol_contrasenya';

Això permet a l'atacant accedir a la base de dades sense conèixer les credencials correctes.

Prevenció d'Injecció SQL

Per prevenir la injecció SQL, es poden utilitzar consultes preparades o procediments emmagatzemats. Aquí hi ha un exemple de com fer-ho en Java usant JDBC:

String query = "SELECT * FROM usuaris WHERE nom = ? AND contrasenya = ?";
PreparedStatement pstmt = connection.prepareStatement(query);
pstmt.setString(1, nomUsuari);
pstmt.setString(2, contrasenya);
ResultSet rs = pstmt.executeQuery();

Conclusió

OWASP és una organització fonamental en el camp de la seguretat d'aplicacions, proporcionant recursos essencials com l'OWASP Top Ten i eines com OWASP ZAP. Comprendre i aplicar els principis i pràctiques recomanades per OWASP és crucial per a qualsevol professional de TI que desitgi assegurar les seves aplicacions contra les amenaces més comunes i perilloses.
