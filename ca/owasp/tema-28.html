Introducció

En aquesta secció del curs sobre OWASP, ens enfocarem en exercicis pràctics de seguretat. Aquests exercicis estan dissenyats per ajudar els professionals de TI a identificar, entendre i mitigar vulnerabilitats comunes en aplicacions web. A través d'exemples i exercicis pràctics, els participants podran aplicar els conceptes apresos en situacions del món real.

Exercici 1: Injecció SQL

Descripció

La injecció SQL és una de les vulnerabilitats més comunes i perilloses en aplicacions web. Aquest exercici t'ajudarà a identificar i mitigar aquesta vulnerabilitat.

Objectius

  • Comprendre com funciona la injecció SQL.
  • Aprendre a identificar punts vulnerables en una aplicació web.
  • Implementar mesures de mitigació per prevenir injeccions SQL.

Exemple de Codi

# Codi vulnerable a injecció SQL
import sqlite3

def get_user_data(user_id):
    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()
    query = f"SELECT * FROM users WHERE id = {user_id}"
    cursor.execute(query)
    return cursor.fetchall()

# Ús del codi vulnerable
user_data = get_user_data("1 OR 1=1")
print(user_data)

Exercici

  1. Identifica el problema en el codi anterior.
  2. Modifica el codi per usar consultes parametritzades i prevenir la injecció SQL.

Solució

# Codi segur usant consultes parametritzades
import sqlite3

def get_user_data(user_id):
    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()
    query = "SELECT * FROM users WHERE id = ?"
    cursor.execute(query, (user_id,))
    return cursor.fetchall()

# Ús del codi segur
user_data = get_user_data(1)
print(user_data)

Exercici 2: Cross-Site Scripting (XSS)

Descripció

El Cross-Site Scripting (XSS) permet als atacants injectar scripts maliciosos en pàgines web vistes per altres usuaris. Aquest exercici t'ajudarà a identificar i mitigar aquesta vulnerabilitat.

Objectius

  • Comprendre com funciona el XSS.
  • Aprendre a identificar punts vulnerables en una aplicació web.
  • Implementar mesures de mitigació per prevenir XSS.

Exemple de Codi

<!-- Codi vulnerable a XSS -->
<html>
<body>
    <form method="GET">
        <input type="text" name="user_input">
        <input type="submit" value="Submit">
    </form>
    <div>
        <?php echo $_GET['user_input']; ?>
    </div>
</body>
</html>

Exercici

  1. Identifica el problema en el codi anterior.
  2. Modifica el codi per escapar adequadament l'input de l'usuari i prevenir XSS.

Solució

<!-- Codi segur usant escape de caràcters -->
<html>
<body>
    <form method="GET">
        <input type="text" name="user_input">
        <input type="submit" value="Submit">
    </form>
    <div>
        <?php echo htmlspecialchars($_GET['user_input'], ENT_QUOTES, 'UTF-8'); ?>
    </div>
</body>
</html>

Exercici 3: Autenticació i Gestió de Sessions

Descripció

L'autenticació i la gestió de sessions són crucials per a la seguretat d'una aplicació web. Aquest exercici t'ajudarà a implementar pràctiques segures per a l'autenticació i la gestió de sessions.

Objectius

  • Comprendre els principis de l'autenticació segura.
  • Aprendre a gestionar sessions de manera segura.
  • Implementar mesures de mitigació per prevenir atacs relacionats amb l'autenticació i les sessions.

Exemple de Codi

# Codi vulnerable a atacs de sessió
from flask import Flask, request, session

app = Flask(__name__)
app.secret_key = 'super_secret_key'

@app.route('/login', methods=['POST'])
def login():
    username = request.form['username']
    password = request.form['password']
    if authenticate(username, password):
        session['username'] = username
        return "Logged in"
    return "Login failed"

def authenticate(username, password):
    # Autenticació simple (no segura)
    return username == "admin" and password == "password"

if __name__ == '__main__':
    app.run()

Exercici

  1. Identifica els problemes en el codi anterior.
  2. Modifica el codi per usar pràctiques segures d'autenticació i gestió de sessions.

Solució

# Codi segur usant pràctiques segures d'autenticació i gestió de sessions
from flask import Flask, request, session, redirect, url_for
from werkzeug.security import check_password_hash, generate_password_hash

app = Flask(__name__)
app.secret_key = 'super_secret_key'

# Simulació d'una base de dades d'usuaris
users = {
    "admin": generate_password_hash("password")
}

@app.route('/login', methods=['POST'])
def login():
    username = request.form['username']
    password = request.form['password']
    if username in users and check_password_hash(users[username], password):
        session['username'] = username
        return redirect(url_for('dashboard'))
    return "Login failed"

@app.route('/dashboard')
def dashboard():
    if 'username' in session:
        return f"Welcome {session['username']}!"
    return redirect(url_for('login'))

if __name__ == '__main__':
    app.run()

Conclusió

En aquesta secció, hem cobert tres exercicis pràctics de seguretat centrats en injecció SQL, Cross-Site Scripting (XSS) i autenticació i gestió de sessions. Practicar aquests exercicis t'ajudarà a aplicar els conceptes de OWASP en situacions del món real, millorant la seguretat de les teves aplicacions.
