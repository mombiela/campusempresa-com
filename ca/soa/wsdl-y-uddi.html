Introducció a WSDL i UDDI

En el context de l'Arquitectura Orientada a Serveis (SOA), WSDL (Web Services Description Language) i UDDI (Universal Description, Discovery, and Integration) són components fonamentals per a la descripció i descobriment de serveis web. Aquest tema cobreix els conceptes essencials, l'estructura, i l'ús de WSDL i UDDI, proporcionant exemples clars i exercicis pràctics.

WSDL (Web Services Description Language)

WSDL és un llenguatge basat en XML que s'utilitza per descriure els serveis web i com accedir-hi. Proporciona una forma estàndard de descriure els detalls dels serveis web, incloent-hi els mètodes disponibles, els paràmetres requerits i els tipus de dades utilitzats.

Conceptes Clau de WSDL

  • Definició del Servei: Descriu el servei web en la seva totalitat.
  • Tipus: Defineix els tipus de dades utilitzats pel servei.
  • Missatges: Descriu les dades que s'intercanvien entre el client i el servei.
  • Operacions: Defineix les accions que es poden realitzar en el servei.
  • Enllaços (Bindings): Descriu el protocol i el format de dades per a cada operació.
  • Port: Defineix una adreça de xarxa per al servei.

Estructura d'un Document WSDL

Un document WSDL típic inclou les següents seccions:

  1. : Defineix els tipus de dades complexos utilitzats pel servei.
  2. : Defineix els missatges que s'intercanvien.
  3. : Defineix les operacions del servei.
  4. : Defineix el protocol i el format de dades.
  5. : Defineix la URL del servei.

Exemple de Document WSDL

<definitions xmlns="http://schemas.xmlsoap.org/wsdl/" 
             xmlns:soap="http://schemas.xmlsoap.org/wsdl/soap/" 
             xmlns:tns="http://example.com/wsdl" 
             targetNamespace="http://example.com/wsdl">
  <types>
    <xsd:schema targetNamespace="http://example.com/wsdl">
      <xsd:element name="getGreeting" type="xsd:string"/>
      <xsd:element name="getGreetingResponse" type="xsd:string"/>
    </xsd:schema>
  </types>
  <message name="getGreetingRequest">
    <part name="name" element="tns:getGreeting"/>
  </message>
  <message name="getGreetingResponse">
    <part name="greeting" element="tns:getGreetingResponse"/>
  </message>
  <portType name="GreetingPortType">
    <operation name="getGreeting">
      <input message="tns:getGreetingRequest"/>
      <output message="tns:getGreetingResponse"/>
    </operation>
  </portType>
  <binding name="GreetingBinding" type="tns:GreetingPortType">
    <soap:binding style="rpc" transport="http://schemas.xmlsoap.org/soap/http"/>
    <operation name="getGreeting">
      <soap:operation soapAction="getGreeting"/>
      <input>
        <soap:body use="literal"/>
      </input>
      <output>
        <soap:body use="literal"/>
      </output>
    </operation>
  </binding>
  <service name="GreetingService">
    <port name="GreetingPort" binding="tns:GreetingBinding">
      <soap:address location="http://example.com/greeting"/>
    </port>
  </service>
</definitions>

UDDI (Universal Description, Discovery, and Integration)

UDDI és un estàndard per a la publicació i descobriment de serveis web. Actua com una "pàgina groga" per als serveis web, permetent a les empreses registrar i cercar serveis disponibles.

Conceptes Clau de UDDI

  • Registre de Negocis: Informació sobre l'empresa que ofereix el servei.
  • Registre de Serveis: Informació sobre els serveis web que ofereix l'empresa.
  • Models de Negoci: Descripcions de les categories i classificacions dels serveis.
  • Interfície de Programació d'Aplicacions (API): Mètodes per interactuar amb el registre UDDI.

Estructura de UDDI

UDDI utilitza una estructura jeràrquica per organitzar la informació:

  • BusinessEntity: Representa una empresa.
  • BusinessService: Representa un servei ofert per l'empresa.
  • BindingTemplate: Detalls tècnics sobre com accedir al servei.
  • tModel: Models tècnics que descriuen les especificacions del servei.

Exemple de Registre UDDI

<businessEntity businessKey="uuid:12345">
  <name>Example Company</name>
  <description>Provider of web services</description>
  <businessServices>
    <businessService serviceKey="uuid:67890">
      <name>Greeting Service</name>
      <bindingTemplates>
        <bindingTemplate bindingKey="uuid:abcde">
          <accessPoint URLType="http">http://example.com/greeting</accessPoint>
        </bindingTemplate>
      </bindingTemplates>
    </businessService>
  </businessServices>
</businessEntity>

Comparació entre WSDL i UDDI

| Característica | WSDL | UDDI | |-------------------------|-------------------------------------------|-------------------------------------------| | Propòsit | Descriure serveis web | Registrar i descobrir serveis web | | Format | XML | XML | | Components Principals | Definició del servei, tipus, missatges | BusinessEntity, BusinessService, tModel | | Ús | Especificació tècnica del servei | Registre i cerca de serveis |

Conclusió

WSDL i UDDI són components essencials en l'Arquitectura Orientada a Serveis (SOA). Mentre que WSDL proporciona una descripció detallada i tècnica dels serveis web, UDDI facilita el registre i descobriment d'aquests serveis. Comprendre ambdós estàndards és crucial per dissenyar, implementar i gestionar serveis web eficientment.

Exercici Pràctic

  1. Crear un Document WSDL: Escrigui un document WSDL per a un servei web que proporcioni informació meteorològica.
  2. Registrar un Servei en UDDI: Creï un registre UDDI per al servei meteorològic descrit en l'exercici anterior.

Aquests exercicis ajudaran a consolidar el coneixement adquirit i a aplicar els conceptes en escenaris pràctics.
