Les proves de punta a punta (E2E) són una part crucial del cicle de vida del desenvolupament de programari, assegurant que tot el flux de l'aplicació funcioni com s'espera. En Angular 2+, les proves E2E es fan típicament utilitzant Protractor, un marc de proves dissenyat específicament per a aplicacions Angular.

Introducció a les proves de punta a punta

  • Definició: Les proves E2E impliquen provar el flux complet d'una aplicació de principi a fi.
  • Propòsit: Validar que els components integrats d'una aplicació funcionin com s'espera.
  • Eines: En Angular, Protractor és l'eina principal utilitzada per a les proves E2E.

Configuració de Protractor

  • Instal·lació: Protractor es pot instal·lar via npm. 
    npm install -g protractor
  • Actualització de WebDriver: Assegura't que WebDriver estigui actualitzat. 
    webdriver-manager update
  • Configuració: Protractor requereix un fitxer de configuració (protractor.conf.js).

Exemple de fitxer de configuració

exports.config = {
  framework: 'jasmine',
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['spec.js'],
  capabilities: {
    'browserName': 'chrome'
  }
};
  • Explicació:
    • framework: Especifica el marc de proves (Jasmine en aquest cas).
    • seleniumAddress: L'adreça del servidor Selenium en execució.
    • specs: Una matriu de fitxers de proves.
    • capabilities: Defineix el navegador que s'utilitzarà per a les proves.

Escriure proves E2E

  • Estructura bàsica: Les proves E2E en Protractor es redacten utilitzant Jasmine.
  • Exemple de prova: 
    describe('Aplicació Angular', function() {
  it('hauria de tenir un títol', function() {
    browser.get('http://localhost:4200');
    expect(browser.getTitle()).toEqual('My Angular App');
  });
});
  • Explicació:
    • describe: Defineix un conjunt de proves.
    • it: Defineix un cas de prova.
    • browser.get: Navega a la URL especificada.
    • expect: Assert l'esperat resultat.

Escenaris comuns de proves E2E

  • Navegació de pàgines: 
    it('hauria de navegar a la pàgina about', function() {
  element(by.linkText('About')).click();
  expect(browser.getCurrentUrl()).toContain('/about');
});
  • Interaccions amb formularis: 
    it('hauria d'enviar un formulari', function() {
  element(by.css('input[name="username"]')).sendKeys('testuser');
  element(by.css('input[name="password"]')).sendKeys('password');
  element(by.buttonText('Submit')).click();
  expect(element(by.css('.success-message')).isPresent()).toBe(true);
});

Tècniques avançades de proves E2E

  • Localitzadors personalitzats: Crear localitzadors personalitzats per a elements més complexos. 
    by.addLocator('buttonText', function(buttonText, opt_parentElement) {
  var using = opt_parentElement || document;
  return using.querySelectorAll('button:contains("' + buttonText + '")');
});
  • Gestió d'operacions asíncrones: Utilitzar async i await per gestionar operacions asíncrones. 
    it('hauria d'esperar que un element sigui visible', async function() {
  await browser.get('http://localhost:4200');
  let myElement = element(by.css('.async-element'));
  await browser.wait(protractor.ExpectedConditions.visibilityOf(myElement), 5000);
  expect(myElement.isPresent()).toBe(true);
});

Millors pràctiques per a proves E2E

  • Mantenir les proves independents: Cada prova hauria de poder executar-se independentment de les altres.
  • Utilitzar objectes de pàgina: Encapsular la lògica específica de la pàgina en objectes de pàgina per millorar la mantenibilitat.
  • Executar proves freqüentment: Integrar les proves E2E en la teva pipeline de CI/CD per detectar problemes aviat.

Conclusió

Les proves de punta a punta són una pràctica essencial per assegurar la fiabilitat i correcció de les teves aplicacions Angular. Utilitzant Protractor, pots simular interaccions reals dels usuaris i validar tot el flux de l'aplicació. Recorda seguir les millors pràctiques per mantenir les teves proves mantenibles i eficients.
