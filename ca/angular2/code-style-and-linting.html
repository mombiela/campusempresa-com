Introducció

L'estil de codi i el linting són pràctiques essencials en el desenvolupament d'Angular que ajuden a mantenir la qualitat del codi, la llegibilitat i la consistència en tot el projecte. Aquesta secció cobrirà la importància de l'estil de codi, com configurar el linting en un projecte Angular i les millors pràctiques per escriure codi net i mantenible.

Importància de l'Estil de Codi

  • Llegibilitat: Un estil de codi consistent facilita que els desenvolupadors llegeixin i entenguin el codi.
  • Mantenibilitat: El codi ben estructurat és més fàcil de mantenir i actualitzar.
  • Col·laboració: Un estil de codi uniforme assegura que tots els membres de l'equip segueixin les mateixes directrius, reduint la fricció en entorns col·laboratius.
  • Prevenció d'Errors: Les eines de linting poden detectar errors potencials a l'inici del procés de desenvolupament.

Configuració del Linting en Angular

Els projectes Angular solen utilitzar TSLint per fer el linting del codi TypeScript. No obstant això, TSLint ha estat depreciat en favor d'ESLint. Aquesta secció et guiarà a través de la configuració d'ESLint en un projecte Angular.

Pas 1: Instal·lar ESLint

Primer, cal instal·lar ESLint i els plugins necessaris per a Angular:

npm install eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin eslint-plugin-import eslint-plugin-jsdoc eslint-plugin-prefer-arrow --save-dev

Pas 2: Crear la Configuració d'ESLint

Crea un fitxer .eslintrc.json a l'arrel del teu projecte amb el següent contingut:

{
  "root": true,
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "project": ["./tsconfig.json"]
  },
  "plugins": ["@typescript-eslint", "import", "jsdoc", "prefer-arrow"],
  "extends": [
    "eslint:recommended",
    "plugin:@typescript-eslint/recommended",
    "plugin:import/errors",
    "plugin:import/warnings",
    "plugin:import/typescript",
    "plugin:jsdoc/recommended",
    "plugin:prefer-arrow/recommended"
  ],
  "rules": {
    "@typescript-eslint/explicit-function-return-type": "off",
    "@typescript-eslint/no-explicit-any": "off",
    "import/order": "error",
    "prefer-arrow/prefer-arrow-functions": [
      "warn",
      {
        "disallowPrototype": true,
        "singleReturnOnly": false,
        "classPropertiesAllowed": false
      }
    ]
  }
}

Pas 3: Actualitzar la Configuració d'Angular

Actualitza el fitxer angular.json per utilitzar ESLint en lloc de TSLint:

"architect": {
  "lint": {
    "builder": "@angular-eslint/builder:lint",
    "options": {
      "eslintConfig": ".eslintrc.json",
      "lintFilePatterns": ["src/**/*.ts"]
    }
  }
}

Pas 4: Executar ESLint

Executa ESLint per comprovar el teu codi per problemes d'estil:

npx eslint src/**/*.ts

Millors Pràctiques per a l'Estil de Codi

  • Convencions de Nomenclatura Consistents: Utilitza camelCase per a variables i funcions, PascalCase per a classes i interfícies.
  • Indentació: Utilitza 2 o 4 espais per a la indentació, i sigues consistent en tot el projecte.
  • Longitud de Línia: Mantingues les línies per sota de 80 caràcters quan sigui possible.
  • Comentaris: Utilitza comentaris per explicar la lògica complexa, però evita comentaris redundants.
  • Estructura de Fitxers: Organitza els fitxers per funció o mòdul per mantenir l'estructura del projecte neta.

Exemple de Bon Estil de Codi

// Import statements
import { Component, OnInit } from '@angular/core';

// Component decorator
@Component({
  selector: 'app-example',
  templateUrl: './example.component.html',
  styleUrls: ['./example.component.css']
})
export class ExampleComponent implements OnInit {
  // Class properties
  title: string;

  // Constructor
  constructor() {
    this.title = 'Hello, Angular!';
  }

  // Lifecycle hook
  ngOnInit(): void {
    console.log(this.title);
  }
}

Exemple de Mal Estil de Codi

export class ExampleComponent implements OnInit{title:string;constructor(){this.title='Hello, Angular!';}ngOnInit():void{console.log(this.title);}}

Conclusió

Adherir-se a un estil de codi consistent i utilitzar eines de linting són pràctiques crucials per mantenir aplicacions Angular d'alta qualitat. Configurant ESLint i seguint les millors pràctiques, pots assegurar-te que el teu codi sigui net, llegible i mantenible, cosa que finalment condueix a un desenvolupament més eficient i una col·laboració més fàcil dins del teu equip.
