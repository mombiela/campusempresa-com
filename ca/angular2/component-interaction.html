El Projecte Sobre nosaltres Contribuir Donacions Llicència

La interacció entre components és un concepte fonamental en Angular que permet que diferents components es comuniquin i comparteixin dades. Aquest tema cobrirà diversos mètodes d'interacció entre components, des de tècniques bàsiques fins a avançades.

Comunicació de Pare a Fill

La comunicació de pare a fill és una de les formes més comunes d'interacció entre components. Implica passar dades d'un component pare a un component fill utilitzant l'enllaç de propietats.

Decorador Input

El decorador @Input s'utilitza per definir propietats d'entrada en el component fill que poden rebre dades del component pare.

Exemple

Component Pare (app.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<app-child [childProperty]="parentProperty"></app-child>`,
})
export class AppComponent {
  parentProperty = 'Hola des del Pare!';
}

Component Fill (child.component.ts):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `<p>{{ childProperty }}</p>`,
})
export class ChildComponent {
  @Input() childProperty: string;
}

En aquest exemple, la parentProperty del component pare es passa a la childProperty en el component fill utilitzant el decorador @Input.

Comunicació de Fill a Pare

La comunicació de fill a pare implica enviar dades d'un component fill al seu component pare. Això es pot aconseguir utilitzant el decorador @Output i EventEmitter.

Decorador Output i EventEmitter

El decorador @Output i EventEmitter s'utilitzen per crear esdeveniments personalitzats que el component pare pot escoltar.

Exemple

Component Fill (child.component.ts):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `<button (click)="sendData()">Enviar Dades</button>`,
})
export class ChildComponent {
  @Output() dataEmitter = new EventEmitter<string>();

  sendData() {
    this.dataEmitter.emit('Hola des del Fill!');
  }
}

Component Pare (app.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<app-child (dataEmitter)="receiveData($event)"></app-child>
             <p>{{ receivedData }}</p>`,
})
export class AppComponent {
  receivedData: string;

  receiveData(data: string) {
    this.receivedData = data;
  }
}

En aquest exemple, el component fill emet un esdeveniment amb dades, que després són rebudes i gestionades pel component pare.

Comunicació entre Germans

Els components germans poden comunicar-se a través d'un component pare compartit o d'un servei compartit.

Servei Compartit

Un servei compartit es pot utilitzar per facilitar la comunicació entre components germans.

Exemple

Servei Compartit (data.service.ts):

import { Injectable } from '@angular/core';
import { Subject } from 'rxjs';

@Injectable({
  providedIn: 'root',
})
export class DataService {
  private dataSubject = new Subject<string>();
  data$ = this.dataSubject.asObservable();

  sendData(data: string) {
    this.dataSubject.next(data);
  }
}

Component Germà 1 (sibling1.component.ts):

import { Component } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-sibling1',
  template: `<button (click)="sendData()">Enviar Dades al Germà 2</button>`,
})
export class Sibling1Component {
  constructor(private dataService: DataService) {}

  sendData() {
    this.dataService.sendData('Hola des del Germà 1!');
  }
}

Component Germà 2 (sibling2.component.ts):

import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-sibling2',
  template: `<p>{{ receivedData }}</p>`,
})
export class Sibling2Component implements OnInit {
  receivedData: string;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.dataService.data$.subscribe((data) => {
      this.receivedData = data;
    });
  }
}

En aquest exemple, Sibling1Component envia dades a Sibling2Component a través del DataService.

Conclusió

La interacció entre components és un aspecte crucial per construir aplicacions Angular. Entendre com comunicar-se eficaçment entre components utilitzant @Input, @Output i serveis compartits et permetrà crear aplicacions més dinàmiques i responsives. En dominar aquestes tècniques, pots assegurar-te que els teus components treballin junts sense problemes, conduint a una base de codi més mantenible i escalable.
