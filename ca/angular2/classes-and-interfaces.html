Introducció

En Angular, entendre les classes i les interfícies és crucial per construir aplicacions robustes i escalables. Aquesta secció cobrirà els conceptes bàsics de les classes i les interfícies, el seu ús i com s'integren en el marc de treball d'Angular.

Classes en Angular

Què és una Classe?

  • Una classe en TypeScript (i JavaScript) és un plànol per crear objectes.
  • Encapsula dades i comportament relacionat amb aquestes dades.

Crear una Classe

Per crear una classe en TypeScript, utilitzes la paraula clau class seguida del nom de la classe.

class Person {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
  }
}

const john = new Person('John', 30);
john.greet(); // Output: Hello, my name is John and I am 30 years old.

Conceptes Clau

  • Constructor: Un mètode especial per crear i inicialitzar un objecte creat amb una classe.
  • Mètodes: Funcions definides dins d'una classe.
  • Propietats: Variables que pertanyen a la classe.

Ús en Angular

Les classes s'utilitzen extensivament en Angular per crear components, serveis, directives i més.

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<h1>{{ title }}</h1>`,
})
export class AppComponent {
  title = 'Hello Angular';
}

Interfícies en Angular

Què és una Interfície?

  • Una interfície en TypeScript és una manera de definir l'estructura d'un objecte.
  • Assegura que un objecte s'adhereixi a una forma específica.

Crear una Interfície

Per crear una interfície, utilitza la paraula clau interface seguida del nom de la interfície.

interface Person {
  name: string;
  age: number;
}

const john: Person = {
  name: 'John',
  age: 30,
};

Conceptes Clau

  • Seguretat de Tipus: Les interfícies proporcionen seguretat de tipus assegurant que els objectes compleixin una estructura específica.
  • Propietats Opcional: Les propietats es poden marcar com a opcionals utilitzant el símbol ?.

Ús en Angular

Les interfícies s'utilitzen sovint per definir la forma dels models de dades, facilitant la gestió i comprensió de les estructures de dades.

interface User {
  id: number;
  name: string;
  email: string;
}

class UserService {
  getUser(): User {
    return { id: 1, name: 'John Doe', email: 'john.doe@example.com' };
  }
}

Classes vs. Interfícies

| Característica | Classes | Interfícies | |----------------|-------------------------------------------|---------------------------------------------| | Definició | Plànol per crear objectes | Defineix l'estructura d'un objecte | | Instanciació | Es pot instanciar utilitzant la paraula clau new | No es pot instanciar | | Membres | Pot tenir propietats i mètodes | Només defineix propietats i signatures de mètodes | | Herència | Suporta herència | Suporta herència múltiple |

Conclusió

Entendre les classes i les interfícies és fonamental per dominar Angular. Les classes proporcionen una manera d'encapsular dades i comportament, mentre que les interfícies asseguren que els objectes compleixin una estructura específica. Ambdues són eines essencials per construir aplicacions Angular escalables i mantenibles. Aprofitant aquests conceptes, pots crear codi més robust i segur, el que condueix a menys errors i a un manteniment més fàcil.
