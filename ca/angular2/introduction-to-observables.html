Què són els Observables?

Els Observables són una part clau de la programació reactiva a Angular. Proporcionen una manera de gestionar fluxos de dades i esdeveniments asíncrons. Entendre els Observables és crucial per treballar amb el client HTTP d'Angular, formularis i altres operacions asíncrones.

  • Fluxos de Dades Asíncrons: Els Observables poden emetre múltiples valors al llarg del temps.
  • Programació Reactiva: Els Observables et permeten reaccionar als canvis de dades i esdeveniments.
  • Integració amb Angular: S'utilitzen extensament en serveis d'Angular, sol·licituds HTTP i gestió d'esdeveniments.

Conceptes Clau dels Observables

  • Observable: Representa un flux de dades o esdeveniments.
  • Observer: Consumeix les dades emeses per l'Observable.
  • Subscription: Representa l'execució d'un Observable. Et permet començar i aturar l'escolta de l'Observable.
  • Operators: Funcions que et permeten manipular les dades emeses per un Observable.
  • Subject: Un tipus especial d'Observable que permet multicast a múltiples Observers.

Crear i Utilitzar Observables

Crear un Observable

Pots crear un Observable utilitzant el constructor Observable o funcions auxiliars com of, from, interval, etc.

import { Observable } from 'rxjs';

// Utilitzant el constructor Observable
const observable = new Observable(subscriber => {
  subscriber.next('Hola');
  subscriber.next('Món');
  subscriber.complete();
});

// Utilitzant una funció auxiliar
import { of } from 'rxjs';
const observableOf = of('Hola', 'Món');

Subscriure's a un Observable

Per consumir les dades emeses per un Observable, necessites subscriure't a ell.

observable.subscribe({
  next(value) { console.log(value); },
  error(err) { console.error('Alguna cosa ha anat malament: ' + err); },
  complete() { console.log('Fet'); }
});

Utilitzar Operators

Els Operators et permeten transformar i manipular les dades emeses per un Observable.

import { from } from 'rxjs';
import { map, filter } from 'rxjs/operators';

const numbers = from([1, 2, 3, 4, 5]);

const squaredEvenNumbers = numbers.pipe(
  filter(n => n % 2 === 0),
  map(n => n * n)
);

squaredEvenNumbers.subscribe(value => console.log(value)); // Sortida: 4, 16

Operators Més Utilitzats

| Operator | Descripció | Exemple d'Ús | |-----------|-------------------------------------------------|---------------------------------------| | map | Transforma cada valor emès per l'Observable | map(x => x * 2) | | filter | Emès només aquells valors que passen una condició| filter(x => x > 10) | | mergeMap| Projecta cada valor font a un Observable | mergeMap(x => of(x * 2)) | | catchError| Captura errors en l'Observable font | catchError(err => of('S'ha produït un error')) |

Exemple Pràctic: Sol·licituds HTTP amb Observables

El servei HttpClient d'Angular utilitza Observables per gestionar sol·licituds HTTP.

import { HttpClient } from '@angular/common/http';
import { Injectable } from '@angular/core';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  constructor(private http: HttpClient) {}

  getData(): Observable<any> {
    return this.http.get('https://api.example.com/data');
  }
}

// Component que utilitza el DataService
import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  template: `<div *ngFor="let item of data">{{ item }}</div>`
})
export class DataComponent implements OnInit {
  data: any[];

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.dataService.getData().subscribe(
      data => this.data = data,
      error => console.error('Error: ', error)
    );
  }
}

Conclusió

Els Observables són una eina poderosa per gestionar dades i esdeveniments asíncrons a Angular. Entenent com crear, subscriure's i manipular Observables, pots gestionar fluxos de dades complexos i millorar la capacitat de resposta de les teves aplicacions. Recorda utilitzar operators per transformar fluxos de dades i gestionar errors de manera elegant. A mesura que avancis, trobaràs els Observables integrals en molts aspectes del desenvolupament amb Angular.
