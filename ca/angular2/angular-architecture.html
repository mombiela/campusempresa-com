Introducció a l'Arquitectura d'Angular

Angular és una plataforma i un framework per construir aplicacions client d'una sola pàgina utilitzant HTML i TypeScript. Angular està escrit en TypeScript i proporciona un conjunt d'eines i biblioteques per construir i gestionar aplicacions Angular.

Conceptes Clau de l'Arquitectura d'Angular

  • Mòduls: Les aplicacions Angular són modulars i utilitzen NgModules per organitzar una aplicació en blocs cohesius de funcionalitat.
  • Components: Els blocs bàsics d'una aplicació Angular. Cada component consta d'una plantilla HTML, una fulla d'estil CSS i una classe TypeScript.
  • Plantilles: Defineixen la vista per a un component utilitzant HTML.
  • Serveis i Injecció de Dependències: Els serveis s'utilitzen per compartir dades i lògica entre components. Angular utilitza la injecció de dependències per proporcionar serveis als components.
  • Ruting: El router d'Angular permet la navegació d'una vista a la següent mentre els usuaris realitzen tasques de l'aplicació.

Mòduls d'Angular

Els mòduls són una manera d'agrupament de components, directives, pipes i serveis que estan relacionats, de tal manera que es poden combinar amb altres mòduls per crear una aplicació.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }
  • declarations: Els components, directives i pipes que pertanyen a aquest mòdul.
  • imports: Altres mòduls les classes exportades dels quals són necessàries per a les plantilles de components declarades en aquest mòdul.
  • providers: Creadors de serveis que aquest mòdul contribueix a la col·lecció global de serveis.
  • bootstrap: La vista principal de l'aplicació, anomenada el component arrel, que allotja totes les altres vistes de l'aplicació.

Components

Els components són el bloc de construcció més bàsic d'una aplicació Angular. Controlen una part de la pantalla anomenada vista.

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'my-angular-app';
}
  • selector: El selector d'element CSS del component.
  • templateUrl: La ubicació del fitxer de plantilla del component.
  • styleUrls: La ubicació dels estils CSS privats del component.

Plantilles

Les plantilles s'utilitzen per definir la vista per a un component. Utilitzen HTML juntament amb la sintaxi de plantilla d'Angular.

<h1>{{ title }}</h1>
<p>Welcome to {{ title }}!</p>
  • Interpolació: {{ title }} és un exemple d'interpolació, que permet incrustar expressions a la plantilla.

Serveis i Injecció de Dependències

Els serveis s'utilitzen per compartir dades i lògica entre components. Angular utilitza la injecció de dependències per proporcionar serveis als components.

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class DataService {
  getData() {
    return ['Data1', 'Data2', 'Data3'];
  }
}
  • @Injectable: Marca una classe com a disponible per ser proporcionada i injectada com a dependència.

Ruting

El router d'Angular permet la navegació d'una vista a la següent mentre els usuaris realitzen tasques de l'aplicació.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }
  • Routes: Defineixen els camins i els components que s'han de mostrar per a cada camí.

Resum

Entendre l'arquitectura d'Angular és crucial per construir aplicacions eficients i escalables. Els components clau de l'arquitectura d'Angular inclouen mòduls, components, plantilles, serveis i ruting. Cada un d'aquests elements juga un paper vital en l'estructura i funcionalitat d'una aplicació Angular. En dominar aquests conceptes, pots crear aplicacions robustes i mantenibles.
