Ús de serveis per a la gestió d'estat

La gestió d'estat és un aspecte crucial per construir aplicacions escalables i mantenibles en Angular. Aquest tema et guiarà a través de l'ús de serveis per a la gestió d'estat, des de conceptes bàsics fins a tècniques avançades.

Introducció a la gestió d'estat

La gestió d'estat es refereix a la pràctica de gestionar l'estat d'una aplicació. En Angular, els serveis juguen un paper clau en la gestió efectiva de l'estat.

  • Estat: Les dades que impulsen el comportament i el renderitzat de la teva aplicació.
  • Gestió d'estat: Tècniques per gestionar i sincronitzar l'estat a través de diferents parts de l'aplicació.

Per què utilitzar serveis per a la gestió d'estat?

Els serveis en Angular són objectes singleton que es poden injectar en components i altres serveis. Proporcionen una manera centralitzada de gestionar l'estat i compartir dades a través de l'aplicació.

  • Naturalesa singleton: Els serveis s'instancien una vegada i es comparteixen a través de l'aplicació.
  • Injecció de dependències: El sistema DI d'Angular facilita la injecció de serveis on sigui necessari.
  • Separació de preocupacions: Els serveis ajuden a mantenir els components lleugers movent la lògica de negoci i la gestió d'estat fora dels components.

Creació d'un servei bàsic

Comencem creant un servei senzill per gestionar una llista d'elements.

Pas 1: Generar el servei

Utilitza Angular CLI per generar un servei:

ng generate service item

Aquesta comanda crea un nou fitxer de servei item.service.ts.

Pas 2: Definir el servei

Obre item.service.ts i defineix el servei:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class ItemService {
  private items: string[] = [];

  constructor() { }

  getItems(): string[] {
    return this.items;
  }

  addItem(item: string): void {
    this.items.push(item);
  }

  removeItem(item: string): void {
    this.items = this.items.filter(i => i !== item);
  }
}
  • @Injectable: Marca la classe com un servei que es pot injectar.
  • providedIn: 'root': Assegura que el servei és un singleton i està disponible a tota l'aplicació.
  • Mètodes: getItems, addItem, i removeItem gestionen l'estat dels elements.

Ús del servei en un component

Ara, utilitzem el ItemService en un component per gestionar l'estat dels elements.

Pas 1: Injectar el servei

Obre un fitxer de component, per exemple, app.component.ts, i injecta el servei:

import { Component } from '@angular/core';
import { ItemService } from './item.service';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  items: string[] = [];

  constructor(private itemService: ItemService) {
    this.items = this.itemService.getItems();
  }

  addItem(item: string): void {
    this.itemService.addItem(item);
    this.items = this.itemService.getItems();
  }

  removeItem(item: string): void {
    this.itemService.removeItem(item);
    this.items = this.itemService.getItems();
  }
}
  • Injecció en el constructor: El servei s'injecta en el constructor del component.
  • Sincronització d'estat: Els mètodes del component criden els mètodes del servei per actualitzar i recuperar l'estat.

Pas 2: Actualitzar la plantilla

Actualitza la plantilla del component, per exemple, app.component.html, per interactuar amb l'estat:

<div>
  <input #newItem type="text" placeholder="Add new item">
  <button (click)="addItem(newItem.value); newItem.value=''">Add Item</button>
</div>
<ul>
  <li *ngFor="let item of items">
    {{ item }} <button (click)="removeItem(item)">Remove</button>
  </li>
</ul>
  • Enllaç d'entrada: Enllaça el valor de l'entrada amb el mètode addItem.
  • Enllaç d'esdeveniments: Enllaça els clics dels botons amb els mètodes addItem i removeItem.
  • Directiva ngFor: Itera sobre l'array d'elements per mostrar cada element.

Gestió avançada d'estat amb RxJS

Per a una gestió d'estat més complexa, pots utilitzar RxJS per gestionar fluxos de dades asíncrons i canvis d'estat.

Pas 1: Modificar el servei per utilitzar RxJS

Actualitza item.service.ts per utilitzar BehaviorSubject de RxJS:

import { Injectable } from '@angular/core';
import { BehaviorSubject, Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class ItemService {
  private itemsSubject: BehaviorSubject<string[]> = new BehaviorSubject<string[]>([]);
  items$: Observable<string[]> = this.itemsSubject.asObservable();

  constructor() { }

  getItems(): Observable<string[]> {
    return this.items$;
  }

  addItem(item: string): void {
    const currentItems = this.itemsSubject.value;
    this.itemsSubject.next([...currentItems, item]);
  }

  removeItem(item: string): void {
    const currentItems = this.itemsSubject.value;
    this.itemsSubject.next(currentItems.filter(i => i !== item));
  }
}
  • BehaviorSubject: Manté l'estat actual i emet nous valors quan l'estat canvia.
  • Observable: Exposa l'estat com un flux observable.

Pas 2: Actualitzar el component per subscriure's a l'observable

Modifica app.component.ts per subscriure's a l'observable:

import { Component, OnInit } from '@angular/core';
import { ItemService } from './item.service';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent implements OnInit {
  items: string[] = [];

  constructor(private itemService: ItemService) {}

  ngOnInit(): void {
    this.itemService.getItems().subscribe(items => {
      this.items = items;
    });
  }

  addItem(item: string): void {
    this.itemService.addItem(item);
  }

  removeItem(item: string): void {
    this.itemService.removeItem(item);
  }
}
  • ngOnInit: Es subscriu a l'observable d'elements per obtenir l'estat més recent.
  • Actualitzacions automàtiques: El component s'actualitza automàticament quan l'estat canvia.

Conclusió

Utilitzar serveis per a la gestió d'estat en Angular ajuda a crear una arquitectura d'aplicació neta, mantenible i escalable. Mitjançant l'ús de la injecció de dependències d'Angular i RxJS, pots gestionar l'estat de manera efectiva i assegurar-te que els teus components es mantinguin centrats en la lògica de presentació.

  • Serveis bàsics: Utilitza serveis per encapsular la lògica de gestió d'estat.
  • RxJS: Utilitza RxJS per a la gestió avançada d'estat i el maneig de dades asíncrones.
  • Separació de preocupacions: Mantingues els components lleugers movent la gestió d'estat als serveis.

En dominar aquestes tècniques, estaràs ben preparat per gestionar l'estat en qualsevol aplicació Angular.
