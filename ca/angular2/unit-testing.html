Les proves unitàries són una part crítica del desenvolupament d'aplicacions robustes i mantenibles. En aquesta secció, cobrirem els conceptes bàsics de les proves unitàries en Angular 2+ i progressarem cap a temes més avançats.

Introducció a les proves unitàries

  • Definició: Les proves unitàries impliquen provar components individuals o peces de codi per assegurar-se que funcionen com s'espera.
  • Importància: Ajuda a identificar errors aviat, facilita la refactorització i assegura la fiabilitat del codi.

Configuració de les proves unitàries en Angular

Angular utilitza Jasmine per escriure proves i Karma com a executor de proves. Aquestes eines estan integrades en Angular CLI.

Instal·lació d'Angular CLI

npm install -g @angular/cli

Creació d'un nou projecte Angular

ng new my-angular-app
cd my-angular-app

Execució de proves

ng test

Aquesta comanda iniciarà l'executor de proves Karma i executarà totes les proves del teu projecte.

Escriure la teva primera prova

Estructura bàsica de la prova

Una prova bàsica en Angular es veu així:

import { TestBed } from '@angular/core/testing';
import { AppComponent } from './app.component';

describe('AppComponent', () => {
  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [
        AppComponent
      ],
    }).compileComponents();
  });

  it('hauria de crear l\'aplicació', () => {
    const fixture = TestBed.createComponent(AppComponent);
    const app = fixture.componentInstance;
    expect(app).toBeTruthy();
  });
});
  • describe: Defineix un conjunt de proves.
  • beforeEach: S'executa abans de cada prova del conjunt.
  • it: Defineix una prova individual.
  • expect: Fa una afirmació sobre el codi.

Proves de components

Exemple de prova de component

Provem un component simple:

import { Component } from '@angular/core';

@Component({
  selector: 'app-hello',
  template: '<h1>Hello, {{name}}!</h1>'
})
export class HelloComponent {
  name: string = 'World';
}

Escriure proves per al component

import { TestBed, ComponentFixture } from '@angular/core/testing';
import { HelloComponent } from './hello.component';

describe('HelloComponent', () => {
  let component: HelloComponent;
  let fixture: ComponentFixture<HelloComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ HelloComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(HelloComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('hauria de crear-se', () => {
    expect(component).toBeTruthy();
  });

  it('hauria de tenir el nom per defecte com "World"', () => {
    expect(component.name).toBe('World');
  });

  it('hauria de renderitzar el nom en la plantilla', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('h1').textContent).toContain('Hello, World!');
  });
});

Proves de serveis

Exemple de prova de servei

Provem un servei simple:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  getData() {
    return 'data';
  }
}

Escriure proves per al servei

import { TestBed } from '@angular/core/testing';
import { DataService } from './data.service';

describe('DataService', () => {
  let service: DataService;

  beforeEach(() => {
    TestBed.configureTestingModule({});
    service = TestBed.inject(DataService);
  });

  it('hauria de crear-se', () => {
    expect(service).toBeTruthy();
  });

  it('hauria de retornar "data"', () => {
    expect(service.getData()).toBe('data');
  });
});

Temes avançats

Simulació de dependències

Quan es proven components o serveis que tenen dependències, pot ser necessari simular aquestes dependències.

Simulació d'un servei

import { TestBed } from '@angular/core/testing';
import { DataService } from './data.service';
import { AnotherService } from './another.service';

class MockAnotherService {
  getValue() {
    return 'mock value';
  }
}

describe('DataService with Mock', () => {
  let service: DataService;

  beforeEach(() => {
    TestBed.configureTestingModule({
      providers: [
        DataService,
        { provide: AnotherService, useClass: MockAnotherService }
      ]
    });
    service = TestBed.inject(DataService);
  });

  it('hauria d\'utilitzar el servei simulat', () => {
    const anotherService = TestBed.inject(AnotherService);
    expect(anotherService.getValue()).toBe('mock value');
  });
});

Proves de codi asíncron

Angular proporciona utilitats per provar codi asíncron, com fakeAsync i tick.

Exemple

import { fakeAsync, tick } from '@angular/core/testing';

it('hauria de gestionar el codi asíncron', fakeAsync(() => {
  let value = false;
  setTimeout(() => {
    value = true;
  }, 1000);
  tick(1000);
  expect(value).toBe(true);
}));

Resum

Les proves unitàries són una pràctica essencial en el desenvolupament d'Angular. Escriure proves per a components, serveis i gestionar codi asíncron assegura que la teva aplicació sigui fiable i mantenible. Comença amb proves bàsiques i incorpora gradualment tècniques més avançades com la simulació de dependències i les proves de codi asíncron per cobrir tots els aspectes de la teva aplicació.
