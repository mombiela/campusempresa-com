La gestió de l'estat és un aspecte crucial per construir aplicacions robustes i escalables en Angular. Aquest tema et guiarà a través dels fonaments de la gestió de l'estat, la seva importància i com implementar-la de manera efectiva en aplicacions Angular.

Què és la Gestió de l'Estat?

La gestió de l'estat es refereix a la pràctica de gestionar l'estat d'una aplicació. L'estat inclou totes les dades que l'aplicació necessita per funcionar correctament, com ara les entrades de l'usuari, les configuracions de l'aplicació i les dades recuperades de les API.

Conceptes Clau

  • Estat: La representació de l'estat actual d'una aplicació.
  • Magatzem: Un lloc centralitzat per gestionar l'estat.
  • Accions: Esdeveniments que desencadenen canvis d'estat.
  • Reductors: Funcions que determinen com canvia l'estat en resposta a les accions.
  • Selectors: Funcions que extreuen peces específiques de l'estat.

Per què és Important la Gestió de l'Estat?

Una gestió adequada de l'estat ajuda en:

  • Predictibilitat: Assegura que l'aplicació es comporti de manera consistent.
  • Depuració: Facilita el seguiment i la correcció d'errors.
  • Escalabilitat: Facilita el desenvolupament d'aplicacions a gran escala.
  • Mantenibilitat: Simplifica el procés d'actualització i manteniment del codi.

Gestió de l'Estat en Angular

Angular proporciona diverses maneres de gestionar l'estat, incloent:

  • Estat del Component: Estat local gestionat dins d'un component.
  • Estat del Servei: Estat compartit gestionat mitjançant serveis Angular.
  • NgRx: Una potent biblioteca de gestió de l'estat inspirada en Redux.

Estat del Component

L'estat del component és la forma més senzilla de gestió de l'estat, on l'estat es gestiona dins de components individuals.

import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  template: `
    <div>
      <p>Count: {{ count }}</p>
      <button (click)="increment()">Increment</button>
      <button (click)="decrement()">Decrement</button>
    </div>
  `
})
export class CounterComponent {
  count = 0;

  increment() {
    this.count++;
  }

  decrement() {
    this.count--;
  }
}

Estat del Servei

L'estat del servei permet compartir l'estat entre múltiples components mitjançant serveis Angular.

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class CounterService {
  private count = 0;

  getCount() {
    return this.count;
  }

  increment() {
    this.count++;
  }

  decrement() {
    this.count--;
  }
}
import { Component } from '@angular/core';
import { CounterService } from './counter.service';

@Component({
  selector: 'app-counter',
  template: `
    <div>
      <p>Count: {{ counterService.getCount() }}</p>
      <button (click)="counterService.increment()">Increment</button>
      <button (click)="counterService.decrement()">Decrement</button>
    </div>
  `
})
export class CounterComponent {
  constructor(public counterService: CounterService) {}
}

NgRx

NgRx és una biblioteca per a la gestió reactiva de l'estat en aplicacions Angular. Proporciona una manera potent de gestionar l'estat utilitzant un flux de dades unidireccional.

Configuració de NgRx

  1. Instal·lar NgRx:
npm install @ngrx/store @ngrx/effects @ngrx/store-devtools
  1. Definir Accions:
import { createAction } from '@ngrx/store';

export const increment = createAction('[Counter] Increment');
export const decrement = createAction('[Counter] Decrement');
export const reset = createAction('[Counter] Reset');
  1. Crear Reductor:
import { createReducer, on } from '@ngrx/store';
import { increment, decrement, reset } from './counter.actions';

export const initialState = 0;

const _counterReducer = createReducer(initialState,
  on(increment, state => state + 1),
  on(decrement, state => state - 1),
  on(reset, state => 0)
);

export function counterReducer(state, action) {
  return _counterReducer(state, action);
}
  1. Registrar Reductor:
import { NgModule } from '@angular/core';
import { StoreModule } from '@ngrx/store';
import { counterReducer } from './counter.reducer';

@NgModule({
  imports: [
    StoreModule.forRoot({ count: counterReducer })
  ]
})
export class AppModule {}
  1. Utilitzar Magatzem en el Component:
import { Component } from '@angular/core';
import { Store } from '@ngrx/store';
import { increment, decrement, reset } from './counter.actions';

@Component({
  selector: 'app-counter',
  template: `
    <div>
      <p>Count: {{ count$ | async }}</p>
      <button (click)="increment()">Increment</button>
      <button (click)="decrement()">Decrement</button>
      <button (click)="reset()">Reset</button>
    </div>
  `
})
export class CounterComponent {
  count$ = this.store.select('count');

  constructor(private store: Store<{ count: number }>) {}

  increment() {
    this.store.dispatch(increment());
  }

  decrement() {
    this.store.dispatch(decrement());
  }

  reset() {
    this.store.dispatch(reset());
  }
}

Conclusió

La gestió de l'estat és un concepte fonamental en les aplicacions Angular. Tant si utilitzes l'estat del component, l'estat del servei o una biblioteca de gestió de l'estat com NgRx, entendre com gestionar l'estat de manera efectiva t'ajudarà a construir aplicacions més robustes, mantenibles i escalables. Comença amb tècniques senzilles de gestió de l'estat i gradualment passa a solucions més avançades a mesura que la teva aplicació creixi.
