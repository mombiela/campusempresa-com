Introducció a la Internacionalització (i18n)

La internacionalització (i18n) és el procés de dissenyar i preparar la teva aplicació per ser adaptable a diversos idiomes i regions sense requerir canvis d'enginyeria en el codi font. Angular proporciona un suport robust per a i18n, facilitant la creació d'aplicacions que es poden traduir a diferents idiomes.

Conceptes Clau de i18n

  • Localització (l10n): El procés de traduir la teva aplicació internacionalitzada a un idioma i regió específics.
  • Fitxers de Traducció: Fitxers que contenen el text traduït per a diferents idiomes.
  • Eines i18n d'Angular: Eines proporcionades per Angular per extreure, gestionar i aplicar traduccions.

Configuració de i18n en Angular

Pas 1: Marcar Text per a Traducció

Angular proporciona l'atribut i18n per marcar text per a traducció en les teves plantilles.

<h1 i18n="@@homeTitle">Welcome to our application!</h1>
<p i18n="@@homeDescription">This is a sample application to demonstrate i18n in Angular.</p>
  • i18n="@@homeTitle": L'atribut i18n marca el text per a traducció.
  • @@homeTitle: Un ID personalitzat opcional per a la traducció.

Pas 2: Extreure Missatges de Traducció

Utilitza Angular CLI per extreure el text marcat en un fitxer de traducció.

ng extract-i18n --output-path src/locale

Aquesta comanda genera un fitxer messages.xlf en el camí de sortida especificat.

Pas 3: Traduir Missatges

Traduir els missatges extrets als idiomes desitjats. Per exemple, crea messages.fr.xlf per a traduccions al francès.

<trans-unit id="homeTitle" datatype="html">
  <source>Welcome to our application!</source>
  <target>Bienvenue dans notre application!</target>
</trans-unit>
<trans-unit id="homeDescription" datatype="html">
  <source>This is a sample application to demonstrate i18n in Angular.</source>
  <target>Ceci est une application d'exemple pour démontrer i18n dans Angular.</target>
</trans-unit>

Pas 4: Configurar Angular per Utilitzar Traduccions

Modifica angular.json per incloure els fitxers de traducció i configurar les opcions de compilació.

"projects": {
  "your-app-name": {
    "i18n": {
      "sourceLocale": "en",
      "locales": {
        "fr": "src/locale/messages.fr.xlf"
      }
    },
    ...
  }
}

Pas 5: Compilar i Servir l'Aplicació

Compila l'aplicació per a una localització específica.

ng build --localize

Serveix l'aplicació per a una localització específica.

ng serve --configuration=fr

Funcions Avançades de i18n

Pluralització

Gestiona les formes plurals en les traduccions utilitzant la sintaxi de missatges ICU d'Angular.

<p i18n="@@itemCount">
  {itemCount, plural, 
    =0 {No items}
    =1 {One item}
    other {# items}
  }
</p>

Paràmetres Dinàmics

Inclou valors dinàmics en les traduccions.

<p i18n="@@welcomeUser">Welcome, {{ userName }}!</p>

Dades de Localització Personalitzades

Afegeix dades de localització personalitzades per a idiomes o regions específiques.

import { registerLocaleData } from '@angular/common';
import localeFr from '@angular/common/locales/fr';

registerLocaleData(localeFr, 'fr');

Conclusió

La internacionalització en Angular 2+ és una característica poderosa que et permet crear aplicacions que es poden adaptar fàcilment a diferents idiomes i regions. Seguint els passos descrits en aquest tema, pots marcar text per a traducció, extreure i traduir missatges, i configurar la teva aplicació per utilitzar aquestes traduccions. Les funcions avançades com la pluralització i els paràmetres dinàmics milloren encara més la flexibilitat de la teva aplicació internacionalitzada.
