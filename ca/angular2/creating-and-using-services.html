Introducció als serveis d'Angular

Els serveis d'Angular són objectes singleton que s'instancien només una vegada durant la vida d'una aplicació. S'utilitzen per compartir dades, lògica i funcions entre diferents components d'una aplicació Angular.

Conceptes clau

  • Servei: Una classe amb un propòsit específic, sovint utilitzada per gestionar operacions de dades, lògica de negoci o altres funcionalitats reutilitzables.
  • Injecció de dependències (DI): Un patró de disseny utilitzat per implementar IoC (Inversió de Control), permetent que una classe rebi les seves dependències d'una font externa en lloc de crear-les ella mateixa.
  • Singleton: Un patró de disseny que restringeix la instanciació d'una classe a una "única" instància.

Creació d'un servei bàsic

Per crear un servei a Angular, pots utilitzar l'Angular CLI o crear manualment el fitxer del servei.

Utilitzant Angular CLI

ng generate service my-service

Creació manual d'un servei

  1. Crea un nou fitxer my-service.service.ts.
  2. Defineix la classe del servei i decora-la amb el decorador @Injectable.
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class MyService {
  constructor() { }

  getGreeting(): string {
    return 'Hello, World!';
  }
}

Explicació

  • @Injectable({ providedIn: 'root' }): Aquest decorador marca la classe com un servei que es pot injectar. La sintaxi providedIn: 'root' assegura que el servei estigui disponible a tota l'aplicació com un singleton.

Ús d'un servei en un component

Per utilitzar el servei en un component, cal injectar-lo al constructor del component.

Exemple de component

  1. Crea un nou component my-component.component.ts.
  2. Injecta el servei al component.
import { Component, OnInit } from '@angular/core';
import { MyService } from './my-service.service';

@Component({
  selector: 'app-my-component',
  template: `<h1>{{ greeting }}</h1>`,
})
export class MyComponent implements OnInit {
  greeting: string;

  constructor(private myService: MyService) { }

  ngOnInit(): void {
    this.greeting = this.myService.getGreeting();
  }
}

Explicació

  • constructor(private myService: MyService): Aquesta sintaxi injecta el MyService al component.
  • ngOnInit(): Hook del cicle de vida on cridem el mètode del servei per obtenir el missatge de salutació.

Funcionalitats avançades del servei

Gestió de sol·licituds HTTP

Els serveis sovint s'utilitzen per gestionar sol·licituds HTTP per interactuar amb APIs de backend.

Exemple: Sol·licitud HTTP GET

  1. Importa HttpClientModule a app.module.ts.
import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [
    HttpClientModule,
    // altres imports
  ],
})
export class AppModule { }
  1. Actualitza el servei per utilitzar HttpClient.
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root',
})
export class MyService {
  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) { }

  getData(): Observable<any> {
    return this.http.get<any>(this.apiUrl);
  }
}
  1. Utilitza el servei en un component per obtenir dades.
import { Component, OnInit } from '@angular/core';
import { MyService } from './my-service.service';

@Component({
  selector: 'app-my-component',
  template: `<div *ngIf="data">{{ data | json }}</div>`,
})
export class MyComponent implements OnInit {
  data: any;

  constructor(private myService: MyService) { }

  ngOnInit(): void {
    this.myService.getData().subscribe(response => {
      this.data = response;
    });
  }
}

Explicació

  • HttpClientModule: Mòdul necessari per fer sol·licituds HTTP.
  • HttpClient: Servei integrat d'Angular per fer sol·licituds HTTP.
  • Observable: Utilitzat per gestionar fluxos de dades asíncrons.

Conclusió

Els serveis a Angular són eines poderoses per compartir dades i lògica entre components. Aprofitant el sistema d'injecció de dependències d'Angular, els serveis es poden crear i utilitzar fàcilment a tota l'aplicació. Ja sigui gestionant operacions de dades simples o sol·licituds HTTP complexes, els serveis ajuden a mantenir el teu codi modular i mantenible.
