Introducció al Routing d'Angular

El routing en Angular et permet navegar entre diferents vistes o components dins de la teva aplicació. És un aspecte crucial de les aplicacions d'una sola pàgina (SPAs) ja que permet a l'usuari moure's entre diferents parts de l'aplicació sense recarregar tota la pàgina.

Conceptes Clau del Routing d'Angular

  • Router Module: El mòdul d'Angular que proporciona els serveis i directives necessaris per al routing.
  • Routes: Objectes de configuració que defineixen la correspondència entre els camins URL i els components.
  • Router Outlet: Una directiva que actua com a marcador de posició per al component rutejat.
  • Router Link: Una directiva per enllaçar a diferents rutes.

Configuració de Rutes Bàsiques

Per configurar el routing en una aplicació Angular, segueix aquests passos:

  1. Instal·la Angular Router: Si no està inclòs, instal·la el paquet Angular Router.

    npm install @angular/router

  2. Importa RouterModule: Importa RouterModule i Routes en el teu mòdul principal de l'aplicació.

    import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { RouterModule, Routes } from '@angular/router';

import { AppComponent } from './app.component';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  declarations: [
    AppComponent,
    HomeComponent,
    AboutComponent
  ],
  imports: [
    BrowserModule,
    RouterModule.forRoot(routes)
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

  3. Afegeix Router Outlet: Afegeix la directiva <router-outlet> en la plantilla del teu component principal.

    <!-- app.component.html -->
<nav>
  <a routerLink="/">Home</a>
  <a routerLink="/about">About</a>
</nav>
<router-outlet></router-outlet>

  4. Crea Components: Assegura't que els components als quals vols rutejar estan creats.

    // home.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-home',
  template: `<h1>Home</h1>`
})
export class HomeComponent { }

// about.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-about',
  template: `<h1>About</h1>`
})
export class AboutComponent { }

Tècniques Avançades de Routing

Rutes Niuades

Les rutes niuades et permeten definir rutes dins d'altres rutes, creant una jerarquia.

const routes: Routes = [
  { path: '', component: HomeComponent },
  { 
    path: 'about', 
    component: AboutComponent,
    children: [
      { path: 'team', component: TeamComponent },
      { path: 'history', component: HistoryComponent }
    ]
  }
];

Guàrdies de Ruta

Les guàrdies de ruta s'utilitzen per controlar l'accés a certes rutes basant-se en condicions.

import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, Router } from '@angular/router';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  constructor(private router: Router) {}

  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): boolean {
    const isAuthenticated = // lògica per comprovar si l'usuari està autenticat
    if (!isAuthenticated) {
      this.router.navigate(['/login']);
      return false;
    }
    return true;
  }
}

// Aplicant la guàrdia a una ruta
const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'protected', component: ProtectedComponent, canActivate: [AuthGuard] }
];

Càrrega Per Mandra

La càrrega per mandra ajuda a carregar mòduls de funcions només quan són necessaris, millorant el rendiment de l'aplicació.

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'feature', loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) }
];

Resum

Configurar rutes en Angular implica configurar el RouterModule amb definicions de rutes, utilitzar la directiva <router-outlet> per mostrar components rutejats, i emprar tècniques avançades com rutes niuades, guàrdies de ruta i càrrega per mandra per millorar les capacitats de routing de l'aplicació. Dominant aquests conceptes, pots crear una navegació robusta i eficient dins de les teves aplicacions Angular.
