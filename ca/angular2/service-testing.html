Les proves de serveis són un aspecte crucial del desenvolupament en Angular. Asseguren que els serveis, que són responsables de gestionar la lògica de negoci i la recuperació de dades, funcionin correctament. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de les proves de serveis en Angular 2+.

Introducció a les Proves de Serveis

  • Què són les Proves de Serveis?

    • Les proves de serveis impliquen provar els serveis en la teva aplicació Angular per assegurar-se que funcionen com s'espera.
    • Es concentren en els aspectes de la lògica de negoci i la gestió de dades de la teva aplicació.

  • Per què són Importants les Proves de Serveis?

    • Asseguren la fiabilitat i correcció de la lògica de negoci.
    • Ajuden a identificar errors en les primeres etapes del cicle de desenvolupament.
    • Faciliten el manteniment i la refactorització.

Configuració de l'Entorn de Proves

Abans de començar a escriure proves, configurem l'entorn de proves.

  • Instal·lació de Paquets Necessaris
    • Angular CLI ve amb Jasmine i Karma preconfigurats per a proves.
    • Assegura't de tenir instal·lat Angular CLI: 
      npm install -g @angular/cli
    • Crea un nou projecte Angular si no en tens cap: 
      ng new my-angular-app
cd my-angular-app

Escriure Proves Bàsiques de Serveis

  • Crear un Servei

    • Genera un nou servei utilitzant Angular CLI: 
      ng generate service my-service
    • Exemple de servei (my-service.service.ts): 
      import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MyService {
  constructor() { }

  getValue(): string {
    return 'Hello, World!';
  }
}

  • Escriure una Prova per al Servei

    • Exemple de prova (my-service.service.spec.ts): 
      import { TestBed } from '@angular/core/testing';
import { MyService } from './my-service.service';

describe('MyService', () => {
  let service: MyService;

  beforeEach(() => {
    TestBed.configureTestingModule({});
    service = TestBed.inject(MyService);
  });

  it('should be created', () => {
    expect(service).toBeTruthy();
  });

  it('should return "Hello, World!"', () => {
    expect(service.getValue()).toBe('Hello, World!');
  });
});

Proves de Serveis amb Dependències

  • Servei amb Dependència

    • Exemple de servei amb dependència (data.service.ts): 
      import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  constructor(private http: HttpClient) { }

  fetchData(): Observable<any> {
    return this.http.get('https://api.example.com/data');
  }
}

  • Simulació de Dependències

    • Utilitza HttpClientTestingModule per simular sol·licituds HTTP: 
      import { TestBed } from '@angular/core/testing';
import { HttpClientTestingModule, HttpTestingController } from '@angular/common/http/testing';
import { DataService } from './data.service';

describe('DataService', () => {
  let service: DataService;
  let httpMock: HttpTestingController;

  beforeEach(() => {
    TestBed.configureTestingModule({
      imports: [HttpClientTestingModule],
      providers: [DataService]
    });
    service = TestBed.inject(DataService);
    httpMock = TestBed.inject(HttpTestingController);
  });

  it('should fetch data', () => {
    const dummyData = [{ id: 1, name: 'John' }];

    service.fetchData().subscribe(data => {
      expect(data.length).toBe(1);
      expect(data).toEqual(dummyData);
    });

    const req = httpMock.expectOne('https://api.example.com/data');
    expect(req.request.method).toBe('GET');
    req.flush(dummyData);
  });

  afterEach(() => {
    httpMock.verify();
  });
});

Tècniques Avançades de Proves de Serveis

  • Proves d'Escenaris d'Error

    • Exemple de prova de gestió d'errors: 
      it('should handle error', () => {
  service.fetchData().subscribe(
    () => fail('should have failed with the 500 error'),
    (error: any) => {
      expect(error.status).toBe(500);
    }
  );

  const req = httpMock.expectOne('https://api.example.com/data');
  req.flush('Something went wrong', { status: 500, statusText: 'Server Error' });
});

  • Utilitzar Espies per Simular Mètodes

    • Exemple d'ús d'espies: 
      it('should call getValue method', () => {
  const spy = spyOn(service, 'getValue').and.returnValue('Mock Value');
  expect(service.getValue()).toBe('Mock Value');
  expect(spy).toHaveBeenCalled();
});

Conclusió

Les proves de serveis en Angular 2+ són essencials per assegurar la fiabilitat i correcció de la lògica de negoci de la teva aplicació. Configurant un entorn de proves adequat, escrivint proves bàsiques i avançades, i utilitzant tècniques com la simulació de dependències i la gestió d'errors, pots crear aplicacions Angular robustes i fàcils de mantenir. Recorda provar de manera exhaustiva i regular per detectar problemes aviat i mantenir una alta qualitat del codi.
