Les proves de components són un aspecte crucial per desenvolupar aplicacions Angular robustes. Aquest tema et guiarà a través dels fonaments de les proves de components, des de la configuració del teu entorn fins a escriure i executar proves per als teus components Angular.

Introducció a les proves de components

Les proves de components impliquen provar components individuals de la teva aplicació Angular per assegurar-se que funcionen com s'espera. Aquest tipus de proves ajuda a detectar errors aviat i assegura que cada component funcioni correctament en aïllament.

Conceptes clau

  • Proves unitàries: Provar unitats o components individuals en aïllament.
  • Test Bed: L'API principal d'Angular per provar components.
  • Fixtures: Un embolcall al voltant d'un component i la seva plantilla.
  • Espies: Funcions simulades que fan un seguiment de les trucades i els arguments.

Configuració de l'entorn de proves

Abans de poder començar a escriure proves, has de configurar el teu entorn de proves.

Instal·lació de dependències

Angular CLI ve amb suport integrat per a proves utilitzant Jasmine i Karma. Assegura't de tenir els paquets necessaris instal·lats:

ng new my-angular-app
cd my-angular-app
ng add @angular/cli

Configuració dels fitxers de prova

Angular CLI genera un fitxer src/test.ts, que és el punt d'entrada principal per executar proves. Assegura't que el teu fitxer angular.json inclogui la següent configuració:

"test": {
  "builder": "@angular-devkit/build-angular:karma",
  "options": {
    "main": "src/test.ts",
    "polyfills": "src/polyfills.ts",
    "tsConfig": "src/tsconfig.spec.json",
    "karmaConfig": "src/karma.conf.js"
  }
}

Escriure la teva primera prova de component

Escrivim una prova senzilla per a un component.

Component d'exemple

Considera un component bàsic d'Angular:

// src/app/hello.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-hello',
  template: '<h1>Hello, {{name}}!</h1>'
})
export class HelloComponent {
  name: string = 'World';
}

Fitxer de prova

Crea un fitxer de prova per al component:

// src/app/hello.component.spec.ts
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { HelloComponent } from './hello.component';

describe('HelloComponent', () => {
  let component: HelloComponent;
  let fixture: ComponentFixture<HelloComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ HelloComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(HelloComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should display the correct name', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('h1').textContent).toContain('Hello, World!');
  });
});

Explicació

  • Configuració de TestBed: Configura l'entorn de proves per al component.
  • Fixture: Proporciona accés a la instància del component i la seva plantilla.
  • Proves bàsiques: Comprova si el component es crea i si mostra el nom correcte.

Proves avançades de components

A mesura que et sentis més còmode amb les proves bàsiques, pots començar a explorar escenaris més avançats.

Provar interaccions de components

Els components sovint interactuen amb serveis. Pots utilitzar espies per simular aquestes interaccions.

Servei d'exemple

// src/app/data.service.ts
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  getData() {
    return 'Real Data';
  }
}

Component que utilitza el servei

// src/app/data.component.ts
import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  template: '<p>{{data}}</p>'
})
export class DataComponent implements OnInit {
  data: string;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.data = this.dataService.getData();
  }
}

Fitxer de prova amb espies

// src/app/data.component.spec.ts
import { ComponentFixture, TestBed } from '@angular/core/testing';
import { DataComponent } from './data.component';
import { DataService } from './data.service';

class MockDataService {
  getData() {
    return 'Mock Data';
  }
}

describe('DataComponent', () => {
  let component: DataComponent;
  let fixture: ComponentFixture<DataComponent>;
  let dataService: DataService;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ DataComponent ],
      providers: [{ provide: DataService, useClass: MockDataService }]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(DataComponent);
    component = fixture.componentInstance;
    dataService = TestBed.inject(DataService);
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should display mock data', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('p').textContent).toContain('Mock Data');
  });
});

Explicació

  • Servei simulat: Una implementació simulada del DataService.
  • Sobreescriptura del proveïdor: Reemplaça el servei real amb el servei simulat en el test bed.
  • Prova: Verifica que el component mostri dades del servei simulat.

Conclusió

Les proves de components en Angular són essencials per assegurar la fiabilitat i correcció de la teva aplicació. Configurant un entorn de proves adequat, escrivint proves bàsiques i avançant cap a escenaris més complexos, pots detectar errors aviat i mantenir una alta qualitat del codi. Recorda:

  • Comença amb proves senzilles per verificar la creació del component i la funcionalitat bàsica.
  • Utilitza simulacions i espies per provar interaccions amb serveis.
  • Expandeix contínuament la teva cobertura de proves a mesura que la teva aplicació creixi.

Amb aquestes pràctiques, estaràs ben equipat per escriure proves efectives de components en Angular.
