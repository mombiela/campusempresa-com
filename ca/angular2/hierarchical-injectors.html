El Projecte Sobre nosaltres Contribuir Donacions Llicència

En Angular, la injecció de dependències (DI) és un patró poderós utilitzat per gestionar com es creen i interactuen els components i serveis entre si. Els injectors jeràrquics són un concepte clau en el sistema de DI d'Angular, permetent una gestió flexible i eficient de les dependències.

Conceptes Clau

  • Injecció de Dependències (DI): Un patró de disseny en el qual una classe sol·licita dependències de fonts externes en lloc de crear-les.
  • Injector: Un localitzador de serveis que és responsable de proporcionar instàncies de dependències.
  • Injectors Jeràrquics: El sistema de DI d'Angular és jeràrquic, el que significa que els injectors poden estar niats, i els injectors fills poden sobreescriure les dependències proporcionades pels injectors pares.

Com Funcionen els Injectors Jeràrquics

Els injectors d'Angular formen una estructura d'arbre que paral·lela l'arbre de components. Cada component té el seu propi injector, i aquests injectors estan disposats jeràrquicament.

  • Injector Arrel: L'injector de nivell superior que es crea quan l'aplicació comença. Proporciona serveis singleton que estan disponibles a tota l'aplicació.
  • Injectors Fills: Injectors que es creen per a cada component. Aquests poden sobreescriure els serveis proporcionats pels injectors pares.

Exemple: Injector Jeràrquic Bàsic

Comencem amb un exemple senzill per il·lustrar com funcionen els injectors jeràrquics.

app.module.ts

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { ParentComponent } from './parent/parent.component';
import { ChildComponent } from './child/child.component';
import { MyService } from './my.service';

@NgModule({
  declarations: [
    AppComponent,
    ParentComponent,
    ChildComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [MyService],
  bootstrap: [AppComponent]
})
export class AppModule { }

my.service.ts

import { Injectable } from '@angular/core';

@Injectable()
export class MyService {
  getValue(): string {
    return 'Valor del Servei des de l'Injector Arrel';
  }
}

parent.component.ts

import { Component } from '@angular/core';
import { MyService } from '../my.service';

@Component({
  selector: 'app-parent',
  template: `
    <div>
      Component Pare: {{ value }}
      <app-child></app-child>
    </div>
  `,
  providers: [MyService] // Això crearà una nova instància de MyService per a aquest component i els seus fills
})
export class ParentComponent {
  value: string;

  constructor(private myService: MyService) {
    this.value = this.myService.getValue();
  }
}

child.component.ts

import { Component } from '@angular/core';
import { MyService } from '../.my.service';

@Component({
  selector: 'app-child',
  template: `
    <div>
      Component Fill: {{ value }}
    </div>
  `
})
export class ChildComponent {
  value: string;

  constructor(private myService: MyService) {
    this.value = this.myService.getValue();
  }
}

Explicació

  • Injector Arrel: El MyService proporcionat en AppModule està disponible a tota l'aplicació.
  • Injector Pare: El MyService proporcionat en ParentComponent sobreescriu el MyService de l'injector arrel per al ParentComponent i els seus fills.
  • Injector Fill: El ChildComponent utilitza el MyService proporcionat pel seu pare, ParentComponent.

En aquest exemple, el ParentComponent i el ChildComponent utilitzaran la instància de MyService proporcionada per l'injector de ParentComponent, no l'injector arrel.

Ús Avançat

Proporcionar Diferents Instàncies

Pots proporcionar diferents instàncies d'un servei a diferents nivells de la jerarquia d'injectors.

Exemple: Diferents Instàncies

// parent.component.ts
@Component({
  selector: 'app-parent',
  template: `
    <div>
      Component Pare: {{ value }}
      <app-child></app-child>
    </div>
  `,
  providers: [{ provide: MyService, useClass: ParentService }]
})
export class ParentComponent {
  value: string;

  constructor(private myService: MyService) {
    this.value = this.myService.getValue();
  }
}

// parent.service.ts
@Injectable()
export class ParentService extends MyService {
  getValue(): string {
    return 'Valor del Servei des de l'Injector del Pare';
  }
}

En aquest exemple, ParentComponent utilitza ParentService, mentre que ChildComponent utilitza la mateixa instància proporcionada per ParentComponent.

Resum

Els injectors jeràrquics en Angular permeten una gestió flexible i eficient de les dependències creant un arbre d'injectors que paral·lela l'arbre de components. Això permet:

  • Serveis Escopats: Els serveis poden estar escopats a components o mòduls específics.
  • Sobreescriure Serveis: Els injectors fills poden sobreescriure els serveis proporcionats pels injectors pares.
  • Gestió Eficient de Recursos: Els serveis singleton poden ser compartits a tota l'aplicació, mentre que altres serveis poden ser instanciats segons sigui necessari.

Entendre els injectors jeràrquics és crucial per construir aplicacions Angular escalables i mantenibles. Aprofitant aquesta característica poderosa, pots assegurar-te que els teus serveis es proporcionen de la manera més eficient i lògica.
