Els interceptors en Angular són una característica poderosa que et permet interceptar i modificar les sol·licituds i respostes HTTP. Són útils per a tasques com afegir tokens d'autenticació, registre, maneig d'errors i més.

Què són els Interceptors?

  • Els interceptors formen part del mòdul HttpClient d'Angular.
  • Et permeten interceptar i modificar les sol·licituds i respostes HTTP globalment.
  • S'implementen com serveis que implementen la interfície HttpInterceptor.

Per què Utilitzar Interceptors?

  • Autenticació: Afegir automàticament tokens d'autenticació a les sol·licituds.
  • Registre: Registrar sol·licituds i respostes HTTP per a propòsits de depuració.
  • Maneig d'Errors: Manejar errors globalment i proporcionar missatges amigables per a l'usuari.
  • Modificar Sol·licituds/Respostes: Modificar capçaleres de sol·licitud, cos o dades de resposta.

Crear un Interceptor

Per crear un interceptor, necessites:

  1. Implementar la interfície HttpInterceptor.
  2. Proveir l'interceptor en el teu mòdul d'Angular.

Exemple Pas a Pas

  1. Crear el Servei d'Interceptor
import { Injectable } from '@angular/core';
import { HttpInterceptor, HttpRequest, HttpHandler, HttpEvent } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable()
export class AuthInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    // Clonar la sol·licitud per afegir la nova capçalera
    const authReq = req.clone({
      headers: req.headers.set('Authorization', 'Bearer YOUR_TOKEN_HERE')
    });

    // Passar la sol·licitud clonada en lloc de la sol·licitud original
    return next.handle(authReq);
  }
}
  1. Proveir l'Interceptor en el Teu Mòdul
import { HTTP_INTERCEPTORS } from '@angular/common/http';
import { AuthInterceptor } from './auth.interceptor';

@NgModule({
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthInterceptor, multi: true }
  ]
})
export class AppModule { }

Casos d'Ús Comuns

Afegir Tokens d'Autenticació

  • Afegir automàticament JWT o altres tokens a cada sol·licitud HTTP.
intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
  const authToken = this.authService.getToken();
  const authReq = req.clone({
    headers: req.headers.set('Authorization', `Bearer ${authToken}`)
  });
  return next.handle(authReq);
}

Registrar Sol·licituds i Respostes

  • Registrar detalls de les sol·licituds i respostes HTTP per a depuració.
intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
  console.log('Request:', req);
  return next.handle(req).pipe(
    tap(event => {
      if (event instanceof HttpResponse) {
        console.log('Response:', event);
      }
    })
  );
}

Maneig Global d'Errors

  • Manejar errors HTTP globalment i mostrar missatges amigables per a l'usuari.
import { catchError } from 'rxjs/operators';
import { throwError } from 'rxjs';

intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
  return next.handle(req).pipe(
    catchError((error: HttpErrorResponse) => {
      let errorMessage = '';
      if (error.error instanceof ErrorEvent) {
        // Error del costat del client
        errorMessage = `Error: ${error.error.message}`;
      } else {
        // Error del costat del servidor
        errorMessage = `Error Code: ${error.status}\nMessage: ${error.message}`;
      }
      window.alert(errorMessage);
      return throwError(errorMessage);
    })
  );
}

Conclusió

Els interceptors en Angular proporcionen una manera poderosa de manejar les sol·licituds i respostes HTTP globalment. Es poden utilitzar per a una varietat de tasques com afegir tokens d'autenticació, registre i maneig d'errors. Implementant la interfície HttpInterceptor i proveint l'interceptor en el teu mòdul d'Angular, pots estendre fàcilment la funcionalitat de les teves sol·licituds i respostes HTTP.

Entendre i utilitzar interceptors pot millorar enormement la robustesa i mantenibilitat de les teves aplicacions Angular.
