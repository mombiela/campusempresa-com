Introducció a les Proves Unitàries

Les proves unitàries són una part essencial del desenvolupament de programari i un component clau en qualsevol pipeline de CI/CD (Integració Contínua/Desplegament Continu). Aquestes proves permeten verificar que cada unitat individual de codi (generalment una funció o mètode) funciona correctament de manera aïllada.

Conceptes Clau

  • Unitat de Codi: La part més petita del codi que pot ser provada de manera independent, com una funció o mètode.
  • Framework de Proves Unitàries: Eines que faciliten la creació i execució de proves unitàries. Exemples inclouen JUnit per a Java, NUnit per a .NET, i pytest per a Python.
  • Mocking: Tècnica per simular el comportament d'objectes complexos o externs que no es desitgen provar directament.
  • Cobertura de Codi: Mètrica que indica el percentatge de codi que és executat durant les proves.

Importància de les Proves Unitàries en CI/CD

Les proves unitàries són fonamentals en un pipeline de CI/CD per diverses raons:

  • Detecció Primerenca d'Errors: Permeten identificar errors en el codi de manera primerenca, abans que s'integrin en el codi base.
  • Faciliten Refactorització: Proporcionen una xarxa de seguretat que permet refactoritzar el codi amb confiança.
  • Documentació del Comportament: Actuen com a documentació viva del comportament esperat del codi.

Exemples de Proves Unitàries

Exemple en Python amb pytest

# math_operations.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

# test_math_operations.py
import pytest
from math_operations import add, subtract

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0

def test_subtract():
    assert subtract(2, 1) == 1
    assert subtract(2, 2) == 0

if __name__ == "__main__":
    pytest.main()

Exemple en Java amb JUnit

// MathOperations.java
public class MathOperations {
    public int add(int a, int b) {
        return a + b;
    }

    public int subtract(int a, int b) {
        return a - b;
    }
}

// MathOperationsTest.java
import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class MathOperationsTest {
    @Test
    public void testAdd() {
        MathOperations math = new MathOperations();
        assertEquals(3, math.add(1, 2));
        assertEquals(0, math.add(-1, 1));
    }

    @Test
    public void testSubtract() {
        MathOperations math = new MathOperations();
        assertEquals(1, math.subtract(2, 1));
        assertEquals(0, math.subtract(2, 2));
    }
}

Exercici Pràctic

  1. Configura un Entorn de Proves: Tria un llenguatge de programació i configura un entorn de proves utilitzant un dels frameworks esmentats.
  2. Escriu Proves Unitàries: Crea una sèrie de funcions simples i escriu proves unitàries per a cadascuna d'elles.
  3. Executa i Analitza: Executa les proves i analitza els resultats. Assegura't que totes les proves passin i revisa la cobertura de codi.

Conclusió

Les proves unitàries són una eina poderosa per assegurar la qualitat del codi i són essencials en qualsevol pipeline de CI/CD. En escriure i executar proves unitàries, els desenvolupadors poden detectar errors primerencament, facilitar la refactorització i documentar el comportament del codi. Utilitzar frameworks de proves unitàries adequats i seguir bones pràctiques en l'escriptura de proves pot millorar significativament l'eficiència i la fiabilitat del desenvolupament de programari.
