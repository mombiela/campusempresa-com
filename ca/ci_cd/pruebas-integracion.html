Introducció

Les proves d'integració són una fase crucial en el cicle de vida del desenvolupament de programari, especialment en un entorn d'Integració Contínua/Desplegament Continu (CI/CD). Aquest tipus de proves es concentren en verificar la interacció entre diferents mòduls o serveis d'una aplicació per assegurar que funcionin junts correctament.

Objectius de les Proves d'Integració

  • Verificar la interacció entre mòduls: Assegurar-se que els diferents components del sistema es comuniquin i funcionin junts com s'espera.
  • Detectar errors en la interfície: Identificar problemes en les interfícies entre mòduls que podrien no ser evidents en proves unitàries.
  • Validar fluxos de treball complets: Provar escenaris d'ús que involucrin múltiples mòduls per garantir que el sistema complet funcioni correctament.

Tipus de Proves d'Integració

  • Proves d'Integració de Big Bang: Tots els mòduls s'integren i proven junts en un sol pas.
  • Proves d'Integració Incremental: Els mòduls s'integren i proven en petits grups. Pot ser:
    • Ascendent: S'integren i proven primer els mòduls de baix nivell i després els d'alt nivell.
    • Descendent: S'integren i proven primer els mòduls d'alt nivell i després els de baix nivell.
    • Sandwich: Una combinació dels enfocaments ascendent i descendent.

Estratègies de Proves d'Integració

  • Proves Basades en la Interfície: Es concentren en les interfícies entre els mòduls.
  • Proves Basades en el Flux de Dades: Es concentren en el flux de dades entre els mòduls.
  • Proves Basades en la Configuració: Es concentren en la configuració del sistema i com afecta la integració dels mòduls.

Eines per a Proves d'Integració

  • JUnit: Popular en l'ecosistema Java per a proves unitàries i integració.
  • TestNG: Similar a JUnit, però amb més funcionalitats per a proves d'integració.
  • Postman: Útil per provar APIs i serveis web.
  • Selenium: Per a proves d'integració en aplicacions web.

Exemple de Prova d'Integració

A continuació, es mostra un exemple d'una prova d'integració utilitzant JUnit i Spring Boot en una aplicació Java.

Configuració del Projecte

Primer, assegura't de tenir les dependències necessàries en el teu arxiu pom.xml:

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>

Exemple de Codi

Suposem que tenim dos serveis: UserService i OrderService. Volem provar la integració entre aquests dos serveis.

@RunWith(SpringRunner.class)
@SpringBootTest
public class IntegrationTest {

    @Autowired
    private UserService userService;

    @Autowired
    private OrderService orderService;

    @Test
    public void testUserOrderIntegration() {
        // Crear un nou usuari
        User user = new User("John Doe", "john.doe@example.com");
        userService.saveUser(user);

        // Crear una nova comanda per a l'usuari
        Order order = new Order(user.getId(), "Laptop", 1);
        orderService.saveOrder(order);

        // Verificar que la comanda s'ha guardat correctament
        List<Order> orders = orderService.getOrdersByUserId(user.getId());
        assertEquals(1, orders.size());
        assertEquals("Laptop", orders.get(0).getProductName());
    }
}

Explicació del Codi

  1. Configuració de l'Entorn de Proves: Utilitzem @SpringBootTest per carregar el context de Spring Boot.
  2. Injecció de Dependències: Utilitzem @Autowired per injectar UserService i OrderService.
  3. Prova d'Integració: Creem un usuari i una comanda, i després verifiquem que la comanda s'ha guardat correctament.

Conclusió

Les proves d'integració són essencials per assegurar que els diferents mòduls d'una aplicació funcionin junts sense problemes. Utilitzant eines i estratègies adequades, podem detectar i resoldre problemes d'integració abans que arribin a producció. En un entorn CI/CD, aquestes proves ajuden a mantenir la qualitat i l'estabilitat del programari al llarg del cicle de desenvolupament.
