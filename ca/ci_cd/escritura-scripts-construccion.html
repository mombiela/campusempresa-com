Introducció

En el context de CI/CD (Integració Contínua/Desplegament Continu), els scripts de construcció són fonamentals per automatitzar el procés de compilació, prova i desplegament d'aplicacions. Aquests scripts permeten als equips de desenvolupament assegurar que el codi es construeixi de manera consistent i eficient en diferents entorns.

Conceptes Clau

  • Automatització: Els scripts de construcció automatitzen tasques repetitives, reduint errors humans i augmentant l'eficiència.
  • Reproduïbilitat: Garantitzen que el procés de construcció sigui el mateix en qualsevol entorn, cosa que facilita la detecció i resolució de problemes.
  • Integració Contínua: Permeten la integració contínua automatitzant la construcció i prova del codi cada vegada que es realitza un canvi.
  • Desplegament Continu: Faciliten el desplegament continu automatitzant el procés de lliurament del programari als entorns de producció.

Eines Comunes

  • Maven: Una eina de gestió i comprensió de projectes principalment per a projectes Java.
  • Gradle: Una eina d'automatització de construcció que suporta múltiples llenguatges de programació.
  • Make: Una eina de construcció que controla la generació d'executables i altres fitxers no font d'un programa.
  • Ant: Una eina de construcció basada en Java que utilitza fitxers XML per definir el procés de construcció.

Comparació d'Eines

| Eina | Llenguatges Suportats | Avantatges | Desavantatges | |-------------|-----------------------|------------|---------------| | Maven | Principalment Java | Gestió de dependències, estandardització | Verbositat en configuració | | Gradle | Múltiples llenguatges | Flexibilitat, rendiment | Corba d'aprenentatge | | Make | C/C++ i altres | Simplicitat, àmpliament usat | Menys adequat per a projectes grans | | Ant | Java | Flexibilitat, extensibilitat | Verbositat en configuració |

Exemples de Scripts de Construcció

Exemple amb Maven

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0-SNAPSHOT</version>
    <dependencies>
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>

Explicació:

  • <groupId>: Identifica el grup o l'organització del projecte.
  • <artifactId>: Identifica el projecte específic.
  • <version>: Versió del projecte.
  • <dependencies>: Llista de dependències necessàries per al projecte.

Exemple amb Gradle

plugins {
    id 'java'
}

group 'com.example'
version '1.0-SNAPSHOT'

repositories {
    mavenCentral()
}

dependencies {
    testImplementation 'junit:junit:4.12'
}

test {
    useJUnitPlatform()
}

Explicació:

  • plugins: Defineix els plugins utilitzats en el projecte.
  • group i version: Identifiquen el grup i la versió del projecte.
  • repositories: Defineix els repositoris d'on s'obtindran les dependències.
  • dependencies: Llista de dependències necessàries per al projecte.
  • test: Configuració específica per a les proves.

Exercici Pràctic

  1. Configurar un Projecte Maven:

    • Crear un fitxer pom.xml per a un projecte Java simple.
    • Afegir una dependència de JUnit per a proves.

  2. Configurar un Projecte Gradle:

    • Crear un fitxer build.gradle per a un projecte Java simple.
    • Afegir una dependència de JUnit per a proves.

Conclusió

L'escriptura de scripts de construcció és una habilitat essencial per a qualsevol professional de TI involucrat en CI/CD. Aquests scripts no només automatitzen i estandarditzen el procés de construcció, sinó que també faciliten la integració i el desplegament continu. Amb eines com Maven i Gradle, els desenvolupadors poden gestionar dependències i configurar projectes de manera eficient, assegurant que el programari es construeixi i desplegui de manera consistent en tots els entorns.
