Proves de Seguretat en CI/CD

Introducció

Les proves de seguretat són una part crucial del procés d'Integració Contínua i Entrega Contínua (CI/CD). Garanteixen que les aplicacions siguin segures i estiguin protegides contra vulnerabilitats abans de ser desplegades en producció. Aquest tema abordarà els conceptes clau, eines i pràctiques recomanades per implementar proves de seguretat en un pipeline de CI/CD.

Conceptes Clau

  • Integració Contínua (CI): Procés d'automatització que permet als desenvolupadors integrar codi en un repositori compartit diverses vegades al dia.
  • Entrega Contínua (CD): Procés que automatitza l'entrega d'aplicacions a entorns de prova i producció.
  • Proves de Seguretat: Conjunt de pràctiques i eines dissenyades per identificar i mitigar vulnerabilitats en el programari.

Tipus de Proves de Seguretat

  1. Anàlisi de Codi Estàtic (SAST):

    • Analitza el codi font sense executar-lo.
    • Detecta vulnerabilitats com injeccions SQL, XSS, etc.
    • Eines: SonarQube, Checkmarx.

  2. Anàlisi de Codi Dinàmic (DAST):

    • Analitza l'aplicació en execució.
    • Detecta vulnerabilitats en temps d'execució.
    • Eines: OWASP ZAP, Burp Suite.

  3. Anàlisi de Dependències:

    • Escaneja biblioteques i dependències per vulnerabilitats conegudes.
    • Eines: OWASP Dependency-Check, Snyk.

  4. Proves de Penetració (Pentesting):

    • Simula atacs reals per identificar vulnerabilitats.
    • Pot ser manual o automatitzat.
    • Eines: Metasploit, Kali Linux.

Integració de Proves de Seguretat en CI/CD

Pipeline de CI/CD amb Proves de Seguretat

Un pipeline de CI/CD típic amb proves de seguretat pot incloure les següents etapes:

  1. Compilació:

    • Compilar el codi font.
    • Executar proves unitàries.

  2. Anàlisi de Codi Estàtic (SAST):

    • Executar eines de SAST per analitzar el codi font.

  3. Anàlisi de Dependències:

    • Escanejar dependències per vulnerabilitats conegudes.

  4. Desplegament a Entorn de Proves:

    • Desplegar l'aplicació a un entorn de proves.

  5. Anàlisi de Codi Dinàmic (DAST):

    • Executar eines de DAST per analitzar l'aplicació en execució.

  6. Proves de Penetració:

    • Realitzar proves de penetració automatitzades.

  7. Desplegament a Producció:

    • Desplegar l'aplicació a producció si totes les proves són satisfactòries.

Exemple de Pipeline en Jenkins

pipeline {
    agent any

    stages {
        stage('Compilació') {
            steps {
                sh 'mvn clean compile'
            }
        }
        stage('Proves Unitàries') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Anàlisi de Codi Estàtic (SAST)') {
            steps {
                sh 'sonar-scanner'
            }
        }
        stage('Anàlisi de Dependències') {
            steps {
                sh 'dependency-check.sh --project my-app --scan .'
            }
        }
        stage('Desplegament a Entorn de Proves') {
            steps {
                sh 'kubectl apply -f deployment.yaml'
            }
        }
        stage('Anàlisi de Codi Dinàmic (DAST)') {
            steps {
                sh 'zap-cli start'
                sh 'zap-cli spider http://my-app-test'
                sh 'zap-cli active-scan http://my-app-test'
                sh 'zap-cli report -o zap_report.html -f html'
            }
        }
        stage('Proves de Penetració') {
            steps {
                sh 'run-pentests.sh'
            }
        }
        stage('Desplegament a Producció') {
            steps {
                input 'Desplegar a producció?'
                sh 'kubectl apply -f deployment-prod.yaml'
            }
        }
    }
}

Conclusió

Les proves de seguretat són essencials per garantir la integritat i protecció de les aplicacions en un entorn CI/CD. Integrar eines i pràctiques de seguretat en el pipeline de CI/CD ajuda a identificar i mitigar vulnerabilitats de manera primerenca, reduint el risc d'atacs i millorant la qualitat del programari. Implementar un enfocament de seguretat des de l'inici del desenvolupament fins al desplegament en producció és fonamental per mantenir aplicacions segures i fiables.
