Introducció

En aquesta secció del curs sobre CI/CD, aprendràs com configurar un repositori de codi, una tasca fonamental per a qualsevol flux de treball d'integració i lliurament continu. Un repositori ben configurat facilita la col·laboració, el control de versions i l'automatització de processos.

Conceptes Clau

  • Repositori de Codi: Un lloc on s'emmagatzema el codi font d'un projecte. Pot ser local o remot.
  • Control de Versions: Sistema que registra els canvis en un fitxer o conjunt de fitxers al llarg del temps.
  • Git: Sistema de control de versions distribuït més utilitzat.
  • GitHub/GitLab/Bitbucket: Serveis d'allotjament de repositoris Git que ofereixen característiques addicionals com integració contínua, gestió de projectes, etc.

Passos per Configurar un Repositori de Codi

1. Instal·lació de Git

Abans de configurar un repositori, necessites tenir Git instal·lat a la teva màquina.

En Windows:

  1. Descarrega l'instal·lador des de git-scm.com.
  2. Executa l'instal·lador i segueix les instruccions.

En macOS:

brew install git

En Linux:

sudo apt-get install git

2. Configuració Inicial de Git

Configura el teu nom d'usuari i correu electrònic, que s'associaran amb els teus commits.

git config --global user.name "El Teu Nom"
git config --global user.email "elteucorreu@example.com"

3. Creació d'un Nou Repositori

Pots crear un repositori nou localment o clonar-ne un d'existent.

Crear un Repositori Local

mkdir el-meu-projecte
cd el-meu-projecte
git init

Clonar un Repositori Existent

git clone https://github.com/usuari/repositori.git

4. Afegir Fitxers al Repositori

Afegeix fitxers a l'àrea de preparació (staging area) i realitza un commit.

echo "# El Meu Projecte" >> README.md
git add README.md
git commit -m "Primer commit"

5. Connectar el Repositori Local amb un Repositori Remot

Per connectar el teu repositori local amb un repositori remot, usa el següent comandament:

git remote add origin https://github.com/usuari/repositori.git
git push -u origin master

Exemple Complet

A continuació, es mostra un exemple complet de configuració d'un repositori de codi:

# Crear un nou directori per al projecte
mkdir el-meu-projecte
cd el-meu-projecte

# Inicialitzar un repositori Git
git init

# Crear un fitxer README
echo "# El Meu Projecte" >> README.md

# Afegir el fitxer README a l'àrea de preparació
git add README.md

# Realitzar el primer commit
git commit -m "Primer commit"

# Connectar el repositori local amb un repositori remot
git remote add origin https://github.com/usuari/repositori.git

# Pujar els canvis al repositori remot
git push -u origin master

Bones Pràctiques

  • Commits Freqüents: Realitza commits petits i freqüents per facilitar el seguiment de canvis.
  • Missatges de Commit Descriptius: Usa missatges clars i descriptius per als teus commits.
  • Branques: Utilitza branques per desenvolupar noves característiques o corregir errors sense afectar la branca principal.

Conclusió

Configurar un repositori de codi és un pas essencial en qualsevol flux de treball de CI/CD. Un repositori ben configurat no només facilita la col·laboració i el control de versions, sinó que també és fonamental per a l'automatització de processos. Seguint els passos i bones pràctiques esmentades, estaràs en el camí correcte per gestionar el teu codi de manera eficient i efectiva.
