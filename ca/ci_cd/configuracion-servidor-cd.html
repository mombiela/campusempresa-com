Introducció

En aquesta secció del curs sobre CI/CD, aprendràs a configurar un servidor de desplegament continu. Aquest procés és essencial per automatitzar la integració i el desplegament d'aplicacions, millorant l'eficiència i reduint els errors humans. Utilitzarem eines populars com Jenkins, GitLab CI/CD, i CircleCI per il·lustrar els conceptes.

Requisits Previs

Abans de començar, assegura't de tenir els següents requisits:

  • Coneixements bàsics de CI/CD.
  • Familiaritat amb sistemes operatius basats en Unix/Linux.
  • Accés a un servidor o màquina virtual amb permisos d'administrador.
  • Instal·lació de Docker (opcional però recomanat).

Selecció de l'Eina de CI/CD

Hi ha diverses eines per configurar un servidor de desplegament continu. Aquí comparem algunes de les més populars:

| Eina | Avantatges | Desavantatges | |---------------|-------------------------------------------------|----------------------------------------------| | Jenkins | Open-source, altament configurable | Requereix configuració manual extensa | | GitLab CI/CD | Integrat amb GitLab, fàcil d'usar | Pot ser costós en versions premium | | CircleCI | Fàcil de configurar, bona integració amb GitHub | Limitacions en el pla gratuït |

Instal·lació de Jenkins

Pas 1: Instal·lació de Java

Jenkins requereix Java per funcionar. Instal·la Java amb els següents comandaments:

sudo apt update
sudo apt install openjdk-11-jdk -y

Pas 2: Instal·lació de Jenkins

Afegeix el repositori de Jenkins i després instal·la Jenkins:

wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt update
sudo apt install jenkins -y

Pas 3: Iniciar Jenkins

Inicia el servei de Jenkins i verifica el seu estat:

sudo systemctl start jenkins
sudo systemctl status jenkins

Pas 4: Configuració Inicial

Accedeix a Jenkins en el teu navegador web a http://<el-teu-servidor>:8080. Segueix les instruccions en pantalla per completar la configuració inicial.

Configuració d'un Pipeline Bàsic en Jenkins

Pas 1: Crear un Nou Job

  1. Al panell de Jenkins, selecciona "Nou Element".
  2. Tria "Pipeline" i proporciona un nom per al teu job.

Pas 2: Definir el Pipeline

A la secció de configuració del job, defineix el teu pipeline utilitzant el següent script d'exemple:

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Comandaments de construcció
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Comandaments de prova
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Comandaments de desplegament
            }
        }
    }
}

Pas 3: Executar el Pipeline

Guarda la configuració i executa el pipeline per verificar que tot funcioni correctament.

Configuració de GitLab CI/CD

Pas 1: Crear un Fitxer `.gitlab-ci.yml`

Al directori arrel del teu projecte, crea un fitxer anomenat .gitlab-ci.yml amb el següent contingut:

stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - echo "Building the project..."

test:
  stage: test
  script:
    - echo "Running tests..."

deploy:
  stage: deploy
  script:
    - echo "Deploying the project..."

Pas 2: Configurar Runners

Assegura't de tenir runners configurats a la teva instància de GitLab. Pots fer-ho des de la secció d'administració de GitLab.

Pas 3: Verificar el Pipeline

Fes un commit i push del teu fitxer .gitlab-ci.yml al teu repositori. GitLab automàticament executarà el pipeline definit.

Conclusió

Configurar un servidor de desplegament continu és un pas crucial per a qualsevol equip de desenvolupament que busqui automatitzar i millorar el seu flux de treball. En aquest tema, hem cobert la instal·lació i configuració bàsica de Jenkins i GitLab CI/CD, dues de les eines més populars en l'àmbit de CI/CD. Amb aquesta base, estaràs preparat per explorar configuracions més avançades i personalitzades segons les necessitats del teu projecte.

Segueix practicant i experimentant amb diferents configuracions per dominar l'art del desplegament continu!
