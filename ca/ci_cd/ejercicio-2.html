Introducció

En aquest exercici, aprendràs a automatitzar proves dins d'un pipeline de CI/CD. L'automatització de proves és crucial per garantir que el codi nou no trenqui funcionalitats existents i que el programari es mantingui estable i d'alta qualitat.

Objectius

  • Comprendre la importància de l'automatització de proves en CI/CD.
  • Configurar un entorn de proves automatitzades.
  • Integrar proves automatitzades en un pipeline de CI/CD.

Conceptes Clau

  • Proves Unitàries: Proves que verifiquen el funcionament de components individuals del codi.
  • Proves d'Integració: Proves que verifiquen la interacció entre diferents components del sistema.
  • Proves de Funcionalitat: Proves que asseguren que el sistema compleix amb els requisits funcionals.
  • Pipeline de CI/CD: Un conjunt de processos automatitzats que permeten la integració contínua i el lliurament continu del programari.

Configuració de l'Entorn de Proves

Eines Necessàries

  • Framework de Proves: JUnit (per a Java), pytest (per a Python), etc.
  • CI/CD Tool: Jenkins, GitLab CI, Travis CI, etc.

Exemple de Configuració amb pytest i GitLab CI

Pas 1: Configurar pytest

  1. Instal·la pytest:

    pip install pytest

  2. Crea un fitxer de prova test_sample.py:

    def test_example():
    assert 1 + 1 == 2

Pas 2: Configurar GitLab CI

  1. Crea un fitxer .gitlab-ci.yml a l'arrel del teu projecte:

    stages:
  - test

test:
  stage: test
  script:
    - pip install pytest
    - pytest

  2. Aquest fitxer defineix una etapa de prova que instal·la pytest i executa les proves.

Integració de Proves en el Pipeline de CI/CD

Exemple de Pipeline en Jenkins

  1. Instal·la el plugin de Git en Jenkins.
  2. Configura un nou projecte Freestyle en Jenkins.
  3. A la secció "Source Code Management", selecciona Git i proporciona la URL del teu repositori.
  4. A la secció "Build Triggers", selecciona "GitHub hook trigger for GITScm polling".
  5. A la secció "Build", afegeix un pas d'execució de shell: 
    pip install pytest
pytest

Exemple de Pipeline en GitLab CI

  1. Assegura't que el teu fitxer .gitlab-ci.yml estigui correctament configurat com es mostra a la secció anterior.
  2. Cada vegada que facis un push al teu repositori, GitLab CI executarà les proves automàticament.

Exercici Pràctic

  1. Configura pytest al teu projecte: Segueix els passos esmentats per instal·lar i configurar pytest.
  2. Escriu Proves Unitàries: Crea almenys tres proves unitàries per a diferents funcions en el teu codi.
  3. Configura GitLab CI: Crea un fitxer .gitlab-ci.yml i assegura't que les proves s'executin automàticament en cada push.
  4. Verifica els Resultats: Assegura't que les proves s'executin correctament i verifica els resultats a la interfície de GitLab CI.

Conclusió

L'automatització de proves és un component essencial en qualsevol pipeline de CI/CD. En integrar proves automatitzades, pots detectar errors de manera primerenca i assegurar la qualitat del programari. En aquest exercici, has après a configurar un entorn de proves, escriure proves unitàries i automatitzar la seva execució en un pipeline de CI/CD.
