Introducció

En aquest exercici, aprendràs a configurar un pipeline bàsic de CI/CD (Integració Contínua i Entrega Contínua) utilitzant una eina popular com GitHub Actions. Aquest exercici està dissenyat per ajudar-te a comprendre els conceptes fonamentals i a posar en pràctica una configuració bàsica que pots expandir en projectes futurs.

Objectius de l'Exercici

  • Configurar un repositori de GitHub per a CI/CD.
  • Crear un fitxer de configuració de GitHub Actions.
  • Executar proves automàtiques en cada push al repositori.
  • Desplegar automàticament a un entorn de staging.

Requisits Previs

  • Coneixement bàsic de Git i GitHub.
  • Familiaritat amb YAML.
  • Un projecte simple en qualsevol llenguatge de programació (utilitzarem un projecte en Python per a aquest exemple).

Pas 1: Crear un Repositori a GitHub

  1. Ves a GitHub i crea un nou compte si no en tens una.
  2. Crea un nou repositori:
    • Nom: ci-cd-demo
    • Descripció: Repositori de demostració per a configuració bàsica de CI/CD
    • Visibilitat: Pública o privada, segons la teva preferència.

Pas 2: Configurar GitHub Actions

  1. Navega a la pestanya "Actions" en el teu nou repositori.
  2. Selecciona "Set up a workflow yourself" per crear un fitxer de configuració des de zero.
  3. Crea un fitxer anomenat ci.yml al directori .github/workflows/.

Exemple de Fitxer de Configuració

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.8'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run tests
      run: |
        pytest

Explicació del Fitxer

  • name: Nom del workflow.
  • on: Defineix l'esdeveniment que desencadena el workflow (en aquest cas, un push a la branca main).
  • jobs: Defineix els treballs que s'executaran.
    • build: Nom del treball.
    • runs-on: Especifica el sistema operatiu del runner.
    • steps: Llista de passos a executar.
      • Checkout code: Utilitza l'acció actions/checkout@v2 per clonar el repositori.
      • Set up Python: Utilitza l'acció actions/setup-python@v2 per configurar Python.
      • Install dependencies: Instal·la les dependències del projecte.
      • Run tests: Executa les proves utilitzant pytest.

Pas 3: Executar el Pipeline

  1. Realitza un commit i push del teu fitxer ci.yml al repositori.
  2. Navega a la pestanya "Actions" en el teu repositori per veure el progrés del pipeline.
  3. Verifica que totes les etapes (checkout, setup, install, test) s'executin correctament.

Conclusió

En aquest exercici, has configurat un pipeline bàsic de CI/CD utilitzant GitHub Actions. Has après a:

  • Configurar un repositori de GitHub per a CI/CD.
  • Crear i entendre un fitxer de configuració de GitHub Actions.
  • Executar proves automàtiques en cada push al repositori.

Aquest és només el començament. A mesura que et familiaritzis amb aquests conceptes, podràs expandir el teu pipeline per incloure desplegaments a producció, anàlisi de codi estàtic, i molt més.
