Introducció

En aquest projecte final, aplicarem tots els conceptes i tècniques apreses al llarg del curs per implementar una solució completa d'Integració Contínua (CI) i Desplegament Continu (CD). Aquest projecte servirà com una demostració pràctica de les teves habilitats i coneixements en CI/CD.

Objectius del Projecte

  • Implementar un pipeline de CI/CD des de zero.
  • Integrar eines de CI/CD com Jenkins, GitHub Actions, GitLab CI, o CircleCI.
  • Configurar proves automatitzades i anàlisi de qualitat de codi.
  • Desplegar una aplicació en un entorn de producció.

Requisits del Projecte

  • Repositori de Codi: Utilitzar un repositori en GitHub, GitLab, o Bitbucket.
  • Aplicació: Pot ser una aplicació web senzilla en qualsevol llenguatge de programació (per exemple, Node.js, Python, Java).
  • Pipeline de CI/CD: Configurar un pipeline que inclogui:
    • Construcció del projecte.
    • Execució de proves unitàries i d'integració.
    • Anàlisi de qualitat de codi.
    • Desplegament automatitzat a un entorn de producció (pot ser un servidor al núvol com AWS, Azure, o un contenidor Docker).

Passos per a la Implementació

1. Configuració del Repositori

  • Crear un nou repositori a la plataforma de la teva elecció (GitHub, GitLab, Bitbucket).
  • Clonar el repositori a la teva màquina local.
  • Afegir el codi de la teva aplicació al repositori.
  • Configurar un fitxer README.md amb una descripció del projecte i les instruccions per executar l'aplicació localment.

2. Configuració del Pipeline de CI/CD

Jenkins

  1. Instal·lació i Configuració de Jenkins:

    • Instal·lar Jenkins a la teva màquina local o en un servidor.
    • Configurar Jenkins perquè tingui accés al teu repositori de codi.

  2. Creació del Pipeline:

    • Crear un nou pipeline a Jenkins.
    • Configurar el pipeline perquè s'executi automàticament en cada commit.
    pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'npm install'
            }
        }
        stage('Test') {
            steps {
                sh 'npm test'
            }
        }
        stage('Deploy') {
            steps {
                sh 'npm run deploy'
            }
        }
    }
}

GitHub Actions

  1. Configuració del Workflow:

    • Crear un fitxer .github/workflows/ci-cd.yml al teu repositori.
    name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

    - name: Deploy
      run: npm run deploy

3. Proves Automatitzades i Anàlisi de Qualitat de Codi

  • Proves Unitàries: Assegura't que la teva aplicació tingui proves unitàries configurades.
  • Proves d'Integració: Configura proves d'integració si és necessari.
  • Anàlisi de Qualitat de Codi: Utilitza eines com ESLint, SonarQube, o CodeClimate per analitzar la qualitat del codi.

4. Desplegament Automatitzat

  • Entorn de Producció: Configura un entorn de producció per a la teva aplicació. Pot ser un servidor al núvol, un contenidor Docker, o una plataforma com Heroku.
  • Automatització del Desplegament: Configura el pipeline perquè desplegui automàticament l'aplicació a l'entorn de producció després de passar totes les proves.

Exemple Complet

A continuació, es mostra un exemple complet d'un pipeline de CI/CD utilitzant GitHub Actions per a una aplicació Node.js:

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

    - name: Lint code
      run: npm run lint

    - name: Deploy
      run: npm run deploy

Conclusió

Aquest projecte final et permetrà consolidar els teus coneixements en CI/CD i demostrar la teva capacitat per implementar una solució completa des de zero. Assegura't de documentar cada pas i de provar exhaustivament el teu pipeline per garantir que funcioni correctament en tots els escenaris possibles. Bona sort!
