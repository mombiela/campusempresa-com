Introducció als Servlets

Els servlets són programes Java que s'executen en un servidor web, gestionant les sol·licituds dels clients i generant contingut web dinàmic. Són un component clau de Java EE (Enterprise Edition) i s'utilitzen per crear aplicacions web.

Conceptes Clau

  • Contenidor de Servlets: L'entorn on s'executen els servlets, gestionant el seu cicle de vida i proporcionant els serveis necessaris.
  • Protocol HTTP: El protocol utilitzat pels servlets per comunicar-se amb els clients.
  • Sol·licitud i Resposta: El mecanisme pel qual els clients envien sol·licituds al servidor i reben respostes.

Configuració del Teu Entorn

Abans d'endinsar-te en els servlets, assegura't de tenir el següent configurat:

  • Java Development Kit (JDK): Necessari per compilar i executar programes Java.
  • Apache Tomcat: Un contenidor de servlets popular.
  • IDE: Entorn de Desenvolupament Integrat com Eclipse o IntelliJ IDEA per facilitar el desenvolupament.

Creant el Teu Primer Servlet

Guia Pas a Pas

  1. Crea un Projecte Web Dinàmic al teu IDE.
  2. Afegeix una Classe Servlet: 
    import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/hello")
public class HelloWorldServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        response.setContentType("text/html");
        response.getWriter().println("<h1>Hello, World!</h1>");
    }
}
  3. Desplega i Executa: Desplega el teu projecte al contenidor de servlets (per exemple, Tomcat) i executa'l. Accedeix al servlet via http://localhost:8080/your_project_name/hello.

Explicació

  • Anotacions: @WebServlet("/hello") mapatge del servlet al patró d'URL /hello.
  • Mètode doGet: Gestiona les sol·licituds GET del client. response.setContentType("text/html") estableix el tipus de resposta, i response.getWriter().println("<h1>Hello, World!</h1>") envia una resposta HTML.

Gestionant Sol·licituds HTTP

Els servlets poden gestionar diversos tipus de sol·licituds HTTP, com GET, POST, PUT, DELETE, etc.

Gestionant Sol·licituds POST

protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
    String name = request.getParameter("name");
    response.setContentType("text/html");
    response.getWriter().println("<h1>Hello, " + name + "!</h1>");
}
  • Mètode doPost: Gestiona les sol·licituds POST. request.getParameter("name") recupera les dades del formulari enviades pel client.

Gestió de Sessions

Les sessions permeten als servlets mantenir l'estat a través de múltiples sol·licituds del mateix client.

Utilitzant HttpSession

protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
    HttpSession session = request.getSession();
    session.setAttribute("username", "JohnDoe");
    response.getWriter().println("Sessió creada amb nom d'usuari: JohnDoe");
}
  • HttpSession: request.getSession() recupera la sessió actual o en crea una de nova. session.setAttribute("username", "JohnDoe") emmagatzema dades a la sessió.

Temes Avançats

Filtres

Els filtres s'utilitzen per preprocessar o postprocessar sol·licituds i respostes.

import java.io.IOException;
import javax.servlet.Filter;
import javax.servlet.FilterChain;
import javax.servlet.FilterConfig;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebFilter;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebFilter("/hello")
public class LoggingFilter implements Filter {
    public void doFilter(HttpServletRequest request, HttpServletResponse response, FilterChain chain) throws IOException, ServletException {
        System.out.println("Sol·licitud rebuda a " + new java.util.Date());
        chain.doFilter(request, response);
    }
}
  • Mètode doFilter: Intercepta la sol·licitud i la resposta. chain.doFilter(request, response) passa el control al següent filtre o servlet.

Listeners

Els listeners s'utilitzen per monitoritzar esdeveniments en una aplicació web.

import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;
import javax.servlet.annotation.WebListener;

@WebListener
public class AppContextListener implements ServletContextListener {
    public void contextInitialized(ServletContextEvent sce) {
        System.out.println("Aplicació iniciada");
    }

    public void contextDestroyed(ServletContextEvent sce) {
        System.out.println("Aplicació aturada");
    }
}
  • ServletContextListener: Els mètodes contextInitialized i contextDestroyed gestionen els esdeveniments d'inici i aturada de l'aplicació.

Conclusió

Els servlets són una eina poderosa per construir aplicacions web dinàmiques en Java. Entenent els conceptes bàsics dels servlets, gestionant sol·licituds HTTP, gestionant sessions i utilitzant característiques avançades com filtres i listeners, pots crear aplicacions web robustes i escalables. Practica creant diversos servlets i experimentant amb diferents característiques per aprofundir en el teu coneixement.
