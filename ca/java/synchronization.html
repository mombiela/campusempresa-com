Utilitats de Concurrència ►

La sincronització és un concepte fonamental en Java que et permet controlar l'accés de múltiples fils a recursos compartits. Això és crucial en un entorn multi-fil per prevenir inconsistències de dades i assegurar la seguretat dels fils.

Conceptes Clau

  • Seguretat dels Fils: Assegurar que les dades compartides siguin accedides per només un fil a la vegada.
  • Secció Crítica: Una secció de codi que accedeix a recursos compartits i no ha de ser executada per més d'un fil a la vegada.
  • Monitor: Un mecanisme de sincronització que permet als fils tenir accés mutu exclusiu a recursos compartits.
  • Bloquejos Intrínsecs: Bloquejos integrats que cada objecte en Java té, utilitzats per gestionar l'accés sincronitzat als mètodes i blocs de l'objecte.

Mètodes Sincronitzats

Un mètode sincronitzat assegura que només un fil pot executar-lo a la vegada per a una instància donada.

public class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }
}

Explicació

  • La paraula clau synchronized assegura que els mètodes increment i getCount siguin accedits per només un fil a la vegada.
  • Això prevé que múltiples fils modifiquin la variable count simultàniament, assegurant la consistència de les dades.

Blocs Sincronitzats

Els blocs sincronitzats permeten un control més detallat sobre el procés de sincronització bloquejant només una secció específica de codi.

public class Counter {
    private int count = 0;
    private final Object lock = new Object();

    public void increment() {
        synchronized (lock) {
            count++;
        }
    }

    public int getCount() {
        synchronized (lock) {
            return count;
        }
    }
}

Explicació

  • El bloc synchronized bloqueja l'objecte especificat (lock en aquest cas) en lloc de tot el mètode.
  • Això pot millorar el rendiment reduint l'abast de la sincronització.

Sincronització Estàtica

Els mètodes estàtics també poden ser sincronitzats, cosa que bloqueja l'objecte de la classe en lloc d'una instància de la classe.

public class StaticCounter {
    private static int count = 0;

    public static synchronized void increment() {
        count++;
    }

    public static synchronized int getCount() {
        return count;
    }
}

Explicació

  • La paraula clau synchronized en un mètode estàtic assegura que només un fil pot executar qualsevol mètode estàtic sincronitzat de la classe a la vegada.
  • Això és útil quan necessites sincronitzar l'accés a variables estàtiques.

Interbloqueig

L'interbloqueig ocorre quan dos o més fils estan bloquejats per sempre, esperant que l'altre alliberi els bloquejos que necessiten.

Exemple d'Interbloqueig

public class DeadlockExample {
    private final Object lock1 = new Object();
    private final Object lock2 = new Object();

    public void method1() {
        synchronized (lock1) {
            System.out.println("Thread 1: Holding lock 1...");
            try { Thread.sleep(10); } catch (InterruptedException e) {}
            synchronized (lock2) {
                System.out.println("Thread 1: Holding lock 1 & 2...");
            }
        }
    }

    public void method2() {
        synchronized (lock2) {
            System.out.println("Thread 2: Holding lock 2...");
            try { Thread.sleep(10); } catch (InterruptedException e) {}
            synchronized (lock1) {
                System.out.println("Thread 2: Holding lock 2 & 1...");
            }
        }
    }
}

Explicació

  • method1 bloqueja lock1 i després intenta bloquejar lock2.
  • method2 bloqueja lock2 i després intenta bloquejar lock1.
  • Si method1 i method2 són cridats per fils diferents simultàniament, pot ocórrer un interbloqueig.

Evitar l'Interbloqueig

  • Ordre de Bloqueig: Adquireix sempre els bloquejos en un ordre consistent.
  • Temps d'Espera: Utilitza temps d'espera quan intentis adquirir bloquejos.
  • Detecció d'Interbloqueig: Implementa algoritmes de detecció d'interbloqueig.

Conclusió

La sincronització en Java és essencial per assegurar la seguretat dels fils en un entorn multi-fil. Utilitzant mètodes sincronitzats, blocs sincronitzats, i entenent el potencial d'interbloqueig, pots gestionar efectivament l'accés concurrent a recursos compartits. Sempre tingues en compte les implicacions de rendiment i esforça't per escriure codi eficient i lliure d'interbloquejos.
