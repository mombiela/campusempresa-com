Introducció a l'API de Fluxos

L'API de Fluxos a Java proporciona una manera moderna i eficient de processar seqüències d'elements. Permet operacions d'estil funcional en col·leccions d'objectes, fent que el processament de dades sigui més llegible i concís.

Conceptes Clau

  • Flux: Una seqüència d'elements que suporta operacions agregades seqüencials i paral·leles.
  • Operacions Intermèdies: Operacions que transformen un flux en un altre flux (per exemple, filter, map).
  • Operacions Terminals: Operacions que produeixen un resultat o un efecte secundari (per exemple, collect, forEach).

Creació de Fluxos

Els fluxos es poden crear a partir de diverses fonts de dades, com ara col·leccions, arrays o canals d'E/S.

A partir de Col·leccions

List<String> list = Arrays.asList("a", "b", "c");
Stream<String> stream = list.stream();

A partir d'Arrays

String[] array = {"a", "b", "c"};
Stream<String> stream = Arrays.stream(array);

A partir de Valors

Stream<String> stream = Stream.of("a", "b", "c");

Operacions Intermèdies

Les operacions intermèdies retornen un nou flux i són mandroses, és a dir, no s'executen fins que s'invoca una operació terminal.

Operacions Intermèdies Comunes

  • filter: Filtra elements basats en un predicat.
  • map: Transforma elements utilitzant una funció.
  • sorted: Ordena elements basats en un comparador.

Exemple: Filtratge i Mapeig

List<String> list = Arrays.asList("apple", "banana", "cherry");
List<String> result = list.stream()
    .filter(s -> s.startsWith("a"))
    .map(String::toUpperCase)
    .collect(Collectors.toList());

System.out.println(result); // Sortida: [APPLE]

Operacions Terminals

Les operacions terminals produeixen un resultat o un efecte secundari i marquen el final del processament del flux.

Operacions Terminals Comunes

  • collect: Converteix el flux en una col·lecció o una altra estructura de dades.
  • forEach: Realitza una acció per a cada element.
  • reduce: Combina elements per produir un únic resultat.

Exemple: Recollida i Reducció

List<String> list = Arrays.asList("apple", "banana", "cherry");
String result = list.stream()
    .map(String::toUpperCase)
    .collect(Collectors.joining(", "));

System.out.println(result); // Sortida: APPLE, BANANA, CHERRY

Fluxos Paral·lels

Els fluxos paral·lels permeten el processament paral·lel de dades, aprofitant els processadors multi-nucli per obtenir un millor rendiment.

Creació de Fluxos Paral·lels

List<String> list = Arrays.asList("apple", "banana", "cherry");
Stream<String> parallelStream = list.parallelStream();

Exemple: Processament Paral·lel

List<String> list = Arrays.asList("apple", "banana", "cherry");
list.parallelStream()
    .forEach(s -> System.out.println(s + " " + Thread.currentThread().getName()));

Millors Pràctiques

  • Utilitza fluxos paral·lels amb prudència, ja que poden introduir complexitat i sobrecàrrega.
  • Prefereix referències a mètodes (per exemple, String::toUpperCase) per a un codi més net i llegible.
  • Tingues en compte els efectes secundaris en les operacions terminals, especialment quan utilitzis fluxos paral·lels.

Conclusió

L'API de Fluxos a Java proporciona una manera potent i expressiva de processar col·leccions de dades. Entenent i utilitzant fluxos, pots escriure codi més concís, llegible i eficient. Recorda equilibrar l'ús de fluxos paral·lels amb la complexitat que introdueixen, i sempre apunta a un codi clar i mantenible.
