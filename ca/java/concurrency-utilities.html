Introducció a la Concurrència en Java

La concurrència en Java permet que múltiples fils s'executin simultàniament, fent que els programes siguin més eficients i responsius. Java proporciona un conjunt ric d'utilitats de concurrència en el paquet java.util.concurrent per ajudar els desenvolupadors a gestionar fils i sincronització.

Conceptes Clau

  • Thread: La unitat més petita de processament. Java proporciona la classe Thread i la interfície Runnable per crear i gestionar fils.
  • Executor Framework: Un reemplaçament de nivell superior per treballar directament amb fils. Proporciona un grup de fils per executar tasques.
  • Sincronització: Mecanismes per controlar l'accés de múltiples fils a recursos compartits.
  • Locks: Més flexibles i potents que els blocs sincronitzats per controlar l'accés a recursos compartits.
  • Variables Atòmiques: Variables que suporten programació segura per fils sense bloquejos en variables individuals.
  • Col·leccions Concurrents: Versions segures per fils de col·leccions estàndard com List, Map i Queue.

Creació i Gestió de Fils

Creació Bàsica de Fils

public class BasicThreadExample {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("Hola des d'un fil!");
        });
        thread.start();
    }
}
  • Explicació: Aquest exemple crea un nou fil utilitzant una expressió lambda i l'inicia. El mètode start() comença l'execució del fil.

Ús de la Interfície Runnable

public class RunnableExample implements Runnable {
    @Override
    public void run() {
        System.out.println("Hola des d'un Runnable!");
    }

    public static void main(String[] args) {
        Thread thread = new Thread(new RunnableExample());
        thread.start();
    }
}
  • Explicació: Aquest exemple implementa la interfície Runnable i passa una instància de RunnableExample a un nou objecte Thread.

Executor Framework

Ús d'Executors

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ExecutorExample {
    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        executor.submit(() -> System.out.println("Tasca 1"));
        executor.submit(() -> System.out.println("Tasca 2"));
        executor.shutdown();
    }
}
  • Explicació: Aquest exemple utilitza el ExecutorService per gestionar un grup de fils. El mètode submit s'utilitza per executar tasques, i shutdown es crida per aturar l'executor d'acceptar noves tasques.

Sincronització

Mètodes i Blocs Sincronitzats

public class SynchronizedExample {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public static void main(String[] args) {
        SynchronizedExample example = new SynchronizedExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explicació: El mètode increment està sincronitzat, assegurant que només un fil pot executar-lo alhora.

Locks

Ús de ReentrantLock

import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class LockExample {
    private final Lock lock = new ReentrantLock();
    private int count = 0;

    public void increment() {
        lock.lock();
        try {
            count++;
        } finally {
            lock.unlock();
        }
    }

    public static void main(String[] args) {
        LockExample example = new LockExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explicació: Aquest exemple utilitza ReentrantLock per controlar l'accés al mètode increment. El mètode lock es crida abans de la secció crítica, i unlock es crida en un bloc finally per assegurar-se que sempre s'executi.

Variables Atòmiques

Ús de AtomicInteger

import java.util.concurrent.atomic.AtomicInteger;

public class AtomicExample {
    private final AtomicInteger count = new AtomicInteger(0);

    public void increment() {
        count.incrementAndGet();
    }

    public static void main(String[] args) {
        AtomicExample example = new AtomicExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explicació: Aquest exemple utilitza AtomicInteger per realitzar operacions atòmiques en una variable enter sense utilitzar sincronització explícita.

Col·leccions Concurrents

Ús de ConcurrentHashMap

import java.util.concurrent.ConcurrentHashMap;
import java.util.Map;

public class ConcurrentMapExample {
    public static void main(String[] args) {
        Map<String, Integer> map = new ConcurrentHashMap<>();
        map.put("key1", 1);
        map.put("key2", 2);

        Thread t1 = new Thread(() -> map.put("key3", 3));
        Thread t2 = new Thread(() -> map.put("key4", 4));
        t1.start();
        t2.start();
    }
}
  • Explicació: Aquest exemple utilitza ConcurrentHashMap per realitzar operacions segures per fils en un mapa.

Conclusió

La concurrència en Java és una característica poderosa que permet als desenvolupadors escriure aplicacions eficients i responsives. Entenent i utilitzant les utilitats de concurrència proporcionades per Java, com fils, el framework Executor, sincronització, locks, variables atòmiques i col·leccions concurrents, els desenvolupadors poden gestionar efectivament tasques concurrents i assegurar la seguretat dels fils en les seves aplicacions.
