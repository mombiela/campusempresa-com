Introducció a les Interfícies

Les interfícies en Java són un concepte fonamental que permet la definició de mètodes que una o més classes han d'implementar. Són una manera d'aconseguir l'abstracció i la herència múltiple en Java.

Conceptes Clau

  • Definició: Una interfície és un tipus de referència en Java, similar a una classe, que només pot contenir constants, signatures de mètodes, mètodes per defecte, mètodes estàtics i tipus imbricats.
  • Propòsit: Les interfícies s'utilitzen per especificar un conjunt de mètodes que una classe ha d'implementar, promovent un disseny basat en contractes.
  • Sintaxi: La paraula clau interface s'utilitza per declarar una interfície.

Exemple

// Definir una interfície
public interface Animal {
    void eat();
    void sleep();
}

// Implementar la interfície en una classe
public class Dog implements Animal {
    @Override
    public void eat() {
        System.out.println("Dog is eating");
    }

    @Override
    public void sleep() {
        System.out.println("Dog is sleeping");
    }
}

Implementació d'Interfícies

Quan una classe implementa una interfície, ha de proporcionar implementacions concretes per a tots els mètodes de la interfície.

Conceptes Clau

  • Implementació: Utilitza la paraula clau implements.
  • Interfícies Múltiples: Una classe pot implementar múltiples interfícies, permetent la herència múltiple de tipus.

Exemple

public interface Animal {
    void eat();
    void sleep();
}

public interface Pet {
    void play();
}

public class Dog implements Animal, Pet {
    @Override
    public void eat() {
        System.out.println("Dog is eating");
    }

    @Override
    public void sleep() {
        System.out.println("Dog is sleeping");
    }

    @Override
    public void play() {
        System.out.println("Dog is playing");
    }
}

Mètodes per Defecte i Estàtics

Java 8 va introduir mètodes per defecte i estàtics en les interfícies, permetent implementacions de mètodes dins de les interfícies.

Conceptes Clau

  • Mètodes per Defecte: Mètodes amb una implementació per defecte que poden ser sobreescrits per les classes que els implementen.
  • Mètodes Estàtics: Mètodes que pertanyen a la classe de la interfície i es poden cridar independentment de qualsevol objecte.

Exemple

public interface Animal {
    void eat();
    void sleep();

    // Mètode per defecte
    default void run() {
        System.out.println("Animal is running");
    }

    // Mètode estàtic
    static void breathe() {
        System.out.println("Animal is breathing");
    }
}

public class Dog implements Animal {
    @Override
    public void eat() {
        System.out.println("Dog is eating");
    }

    @Override
    public void sleep() {
        System.out.println("Dog is sleeping");
    }

    @Override
    public void run() {
        System.out.println("Dog is running");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.eat();
        dog.sleep();
        dog.run();
        Animal.breathe(); // Crida al mètode estàtic
    }
}

Interfícies Funcionals i Expressions Lambda

Les interfícies funcionals són interfícies amb un únic mètode abstracte, i són la base de les expressions lambda en Java.

Conceptes Clau

  • Interfície Funcional: Una interfície amb exactament un mètode abstracte, anotada amb @FunctionalInterface.
  • Expressions Lambda: Proporcionen una manera clara i concisa d'implementar l'únic mètode d'una interfície funcional.

Exemple

@FunctionalInterface
public interface Greeting {
    void sayHello(String name);
}

public class Main {
    public static void main(String[] args) {
        // Utilitzant una expressió lambda per implementar la interfície Greeting
        Greeting greeting = (name) -> System.out.println("Hello, " + name);
        greeting.sayHello("World");
    }
}

Temes Avançats

Herència d'Interfícies

Les interfícies poden estendre altres interfícies, permetent un disseny més complex i jeràrquic.

Conceptes Clau

  • Estendre Interfícies: Utilitza la paraula clau extends per heretar mètodes d'una altra interfície.

Exemple

public interface Animal {
    void eat();
    void sleep();
}

public interface Mammal extends Animal {
    void giveBirth();
}

public class Dog implements Mammal {
    @Override
    public void eat() {
        System.out.println("Dog is eating");
    }

    @Override
    public void sleep() {
        System.out.println("Dog is sleeping");
    }

    @Override
    public void giveBirth() {
        System.out.println("Dog is giving birth");
    }
}

Conclusió

Les interfícies en Java proporcionen una manera poderosa de definir contractes per al que una classe pot fer, promovent un disseny més modular i flexible. Entenent i utilitzant les interfícies, pots crear aplicacions més mantenibles i escalables. Des de la implementació bàsica fins a conceptes avançats com les interfícies funcionals i la herència d'interfícies, dominar les interfícies és essencial per a qualsevol programador Java.
