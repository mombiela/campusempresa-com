Introducció a l'Encapsulació

L'encapsulació és un dels quatre conceptes fonamentals de la Programació Orientada a Objectes (POO). És el mecanisme d'embolicar les dades (variables) i el codi (mètodes) que actuen sobre les dades en una única unitat, coneguda com a classe. L'encapsulació ajuda a protegir les dades d'accés i modificació no autoritzats.

Conceptes Clau de l'Encapsulació

  • Ocultació de Dades: Restringir l'accés directe a alguns components d'un objecte.
  • Modificadors d'Accés: Paraules clau que estableixen l'accessibilitat de classes, mètodes i altres membres.
  • Getters i Setters: Mètodes utilitzats per accedir i actualitzar el valor de variables privades.

Modificadors d'Accés

Java proporciona quatre tipus de modificadors d'accés per controlar la visibilitat dels membres de la classe:

  • private: Accessible només dins de la mateixa classe.
  • default (sense modificador): Accessible només dins del mateix paquet.
  • protected: Accessible dins del mateix paquet i subclasses.
  • public: Accessible des de qualsevol altra classe.

Exemple de Modificadors d'Accés

public class Example {
    private int privateVar; // accés privat
    int defaultVar;         // accés per defecte
    protected int protectedVar; // accés protegit
    public int publicVar;   // accés públic

    // Getter i Setter per a privateVar
    public int getPrivateVar() {
        return privateVar;
    }

    public void setPrivateVar(int privateVar) {
        this.privateVar = privateVar;
    }
}

Getters i Setters

Els getters i setters són mètodes que proporcionen accés controlat a les variables privades d'una classe.

Exemple de Getters i Setters

public class Person {
    private String name;
    private int age;

    // Getter per a name
    public String getName() {
        return name;
    }

    // Setter per a name
    public void setName(String name) {
        this.name = name;
    }

    // Getter per a age
    public int getAge() {
        return age;
    }

    // Setter per a age
    public void setAge(int age) {
        if (age > 0) { // Afegint una validació simple
            this.age = age;
        }
    }
}

Beneficis de l'Encapsulació

  • Control sobre les Dades: Utilitzant getters i setters, pots afegir lògica per controlar com s'accedeix o es modifica la dada.
  • Major Flexibilitat: Pots canviar la implementació interna sense afectar el codi extern.
  • Millor Mantenibilitat: El codi encapsulat és més fàcil de mantenir i entendre.
  • Millora de la Seguretat: L'encapsulació ajuda a protegir les dades d'accés i modificació no autoritzats.

Exemple Pràctic

Creem una classe simple BankAccount que demostri l'encapsulació.

Exemple de la Classe BankAccount

public class BankAccount {
    private String accountNumber;
    private double balance;

    // Constructor
    public BankAccount(String accountNumber, double initialBalance) {
        this.accountNumber = accountNumber;
        this.balance = initialBalance;
    }

    // Getter per a accountNumber
    public String getAccountNumber() {
        return accountNumber;
    }

    // Getter per a balance
    public double getBalance() {
        return balance;
    }

    // Mètode per dipositar diners
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    // Mètode per retirar diners
    public boolean withdraw(double amount) {
        if (amount > 0 i amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }
}

Utilitzant la Classe BankAccount

public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount("123456789", 1000.0);

        // Dipositar diners
        account.deposit(500.0);
        System.out.println("Saldo després del dipòsit: " + account.getBalance());

        // Retirar diners
        if (account.withdraw(200.0)) {
            System.out.println("Retirada exitosa. Saldo restant: " + account.getBalance());
        } else {
            System.out.println("Retirada fallida. Saldo insuficient.");
        }
    }
}

Conclusió

L'encapsulació és un poderós concepte de la POO que ajuda a protegir les dades i mantenir la integritat del codi. Utilitzant modificadors d'accés i mètodes getter/setter, pots controlar com s'accedeix i es modifica la dada, conduint a un codi més segur i mantenible. Entendre i aplicar l'encapsulació és essencial per escriure aplicacions Java robustes.
