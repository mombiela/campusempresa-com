Introducció a les Expressions Lambda

Les expressions lambda, introduïdes a Java 8, proporcionen una manera clara i concisa de representar una interfície d'un sol mètode utilitzant una expressió. Són una característica significativa que aporta capacitats de programació funcional a Java.

Conceptes Clau

  • Interfície Funcional: Una interfície amb només un mètode abstracte. Les expressions lambda es poden utilitzar per instanciar aquestes interfícies.
  • Sintaxi: La sintaxi bàsica d'una expressió lambda és (parameters) -> expression o (parameters) -> { statements; }.

Sintaxi Bàsica i Ús

Les expressions lambda es poden utilitzar principalment per definir el mètode d'una interfície funcional.

Exemple: Sintaxi Bàsica

// Manera tradicional utilitzant una classe anònima
Runnable runnable = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello, World!");
    }
};

// Utilitzant una expressió lambda
Runnable lambdaRunnable = () -> System.out.println("Hello, World!");

Explicació

  • Interfície Runnable: Una interfície funcional amb un únic mètode abstracte run().
  • Classe Anònima: La manera tradicional de proporcionar una implementació per al mètode run().
  • Expressió Lambda: Una manera més concisa d'aconseguir el mateix resultat.

Interfícies Funcionals

Java proporciona diverses interfícies funcionals integrades en el paquet java.util.function.

Interfícies Funcionals Comunes

  • Predicate: Representa una funció que retorna un valor booleà d'un argument.
  • Function<T, R>: Representa una funció que accepta un argument i produeix un resultat.
  • Consumer: Representa una operació que accepta un únic argument d'entrada i no retorna cap resultat.
  • Supplier: Representa un proveïdor de resultats.

Exemple: Utilitzant Predicate

import java.util.function.Predicate;

public class LambdaExample {
    public static void main(String[] args) {
        Predicate<String> isEmpty = s -> s.isEmpty();
        
        System.out.println(isEmpty.test("")); // true
        System.out.println(isEmpty.test("Hello")); // false
    }
}

Explicació

  • Interfície Predicate: El mètode test s'implementa utilitzant una expressió lambda s -> s.isEmpty().
  • Ús: El mètode test comprova si una cadena està buida.

Ús Avançat

Les expressions lambda es poden utilitzar en escenaris més complexos, com ara fluxos i referències a mètodes.

Exemple: Utilitzant Fluxos

import java.util.Arrays;
import java.util.List;

public class StreamExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "David");
        
        names.stream()
             .filter(name -> name.startsWith("A"))
             .forEach(System.out::println);
    }
}

Explicació

  • API de Fluxos: Permet operacions d'estil funcional en col·leccions.
  • Mètode Filter: Utilitza una expressió lambda per filtrar noms que comencen amb "A".
  • Referència a Mètode: System.out::println és una forma abreujada de name -> System.out.println(name).

Conclusió

Les expressions lambda simplifiquen la sintaxi i milloren la llegibilitat del codi Java, especialment quan es tracta d'interfícies funcionals. Són una característica potent que permet paradigmes de programació funcional a Java, fent el codi més concís i expressiu.

Resum

  • Les expressions lambda proporcionen una manera concisa d'implementar interfícies funcionals.
  • Són particularment útils en operacions amb col·leccions i fluxos.
  • Entendre les expressions lambda és crucial per a la programació moderna en Java, especialment quan es treballa amb l'API de Fluxos i interfícies funcionals.

En dominar les expressions lambda, pots escriure codi Java més eficient i llegible, aprofitant tot el poder de la programació funcional en Java.
