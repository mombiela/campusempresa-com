Introducció a NIO

Java NIO (New I/O) és una API d'I/O alternativa per a Java que ofereix una manera diferent de treballar amb I/O que el paquet d'I/O estàndard (java.io). Es va introduir a Java 1.4 i proporciona una manera més eficient i escalable de gestionar operacions d'I/O.

Conceptes Clau de NIO

  • Buffers: Contenidors per a dades que es poden llegir o escriure.
  • Canals: Una nova abstracció primitiva d'I/O que suporta la lectura i escriptura en buffers.
  • Selectors: Permeten que un únic fil gestioni múltiples canals per a operacions d'I/O no bloquejants.

Buffers

Els buffers són fonamentals per a NIO. S'utilitzen per a contenir dades que es transfereixen entre un canal i una aplicació.

  • Tipus de Buffers: ByteBuffer, CharBuffer, ShortBuffer, IntBuffer, LongBuffer, FloatBuffer, DoubleBuffer.
  • Operacions de Buffer:
    • put(): Escriu dades al buffer.
    • get(): Llegeix dades del buffer.
    • flip(): Prepara el buffer per a la lectura després de l'escriptura.
    • clear(): Neteja el buffer per a l'escriptura.
    • rewind(): Rebobina el buffer per a tornar a llegir les dades.

Exemple d'Ús de Buffers

import java.nio.ByteBuffer;

public class BufferExample {
    public static void main(String[] args) {
        ByteBuffer buffer = ByteBuffer.allocate(10);

        // Escriure dades al buffer
        for (int i = 0; i < buffer.capacity(); i++) {
            buffer.put((byte) i);
        }

        // Girar el buffer per preparar-lo per a la lectura
        buffer.flip();

        // Llegir dades del buffer
        while (buffer.hasRemaining()) {
            System.out.print(buffer.get() + " ");
        }
    }
}

Canals

Els canals s'utilitzen per a la transferència de dades entre buffers i entitats d'I/O com fitxers o sockets.

  • Tipus de Canals: FileChannel, DatagramChannel, SocketChannel, ServerSocketChannel.
  • Operacions de Canal:
    • read(): Llegeix dades del canal en un buffer.
    • write(): Escriu dades d'un buffer al canal.
    • close(): Tanca el canal.

Exemple d'Ús de Canals

import java.io.RandomAccessFile;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;

public class ChannelExample {
    public static void main(String[] args) throws Exception {
        RandomAccessFile file = new RandomAccessFile("example.txt", "rw");
        FileChannel channel = file.getChannel();

        ByteBuffer buffer = ByteBuffer.allocate(48);
        int bytesRead = channel.read(buffer);

        while (bytesRead != -1) {
            buffer.flip();
            while (buffer.hasRemaining()) {
                System.out.print((char) buffer.get());
            }
            buffer.clear();
            bytesRead = channel.read(buffer);
        }
        channel.close();
        file.close();
    }
}

Selectors

Els selectors permeten que un únic fil gestioni múltiples canals, cosa que és útil per a operacions d'I/O no bloquejants.

  • Operacions de Selector:
    • select(): Bloqueja fins que almenys un canal estigui llest per a les operacions que t'interessen.
    • selectedKeys(): Retorna el conjunt de claus que representen els canals que estan llestos per a operacions d'I/O.
    • register(): Registra un canal amb el selector.

Exemple d'Ús de Selectors

import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.SelectionKey;
import java.nio.channels.Selector;
import java.nio.channels.ServerSocketChannel;
import java.nio.channels.SocketChannel;
import java.util.Iterator;

public class SelectorExample {
    public static void main(String[] args) throws IOException {
        Selector selector = Selector.open();
        ServerSocketChannel serverChannel = ServerSocketChannel.open();
        serverChannel.bind(new InetSocketAddress("localhost", 8080));
        serverChannel.configureBlocking(false);
        serverChannel.register(selector, SelectionKey.OP_ACCEPT);

        while (true) {
            selector.select();
            Iterator<SelectionKey> keys = selector.selectedKeys().iterator();

            while (keys.hasNext()) {
                SelectionKey key = keys.next();
                keys.remove();

                if (key.isAcceptable()) {
                    ServerSocketChannel server = (ServerSocketChannel) key.channel();
                    SocketChannel client = server.accept();
                    client.configureBlocking(false);
                    client.register(selector, SelectionKey.OP_READ);
                } else if (key.isReadable()) {
                    SocketChannel client = (SocketChannel) key.channel();
                    ByteBuffer buffer = ByteBuffer.allocate(256);
                    client.read(buffer);
                    buffer.flip();
                    System.out.println(new String(buffer.array()).trim());
                    buffer.clear();
                }
            }
        }
    }
}

Conclusió

Java NIO proporciona una manera potent i flexible de gestionar operacions d'I/O, oferint millor rendiment i escalabilitat en comparació amb l'I/O tradicional. Entendre buffers, canals i selectors és crucial per aprofitar tot el potencial de NIO. Dominant aquests conceptes, pots construir aplicacions eficients i responsives que gestionin operacions d'I/O de manera efectiva.
