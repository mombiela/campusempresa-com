Introducció a les Transaccions

Les transaccions són un concepte fonamental en la gestió de bases de dades i són crucials per assegurar la integritat i consistència de les dades. En Java, les transaccions sovint es gestionen a través de JDBC (Java Database Connectivity) o JPA (Java Persistence API). Aquesta secció cobrirà els conceptes bàsics de les transaccions, les seves propietats i com implementar-les en Java.

Conceptes Clau

  • Propietats ACID: Les transaccions han d'adherir-se a les propietats ACID per assegurar la fiabilitat.
    • Atomicitat: Assegura que totes les operacions dins d'una transacció es completen amb èxit. Si alguna operació falla, la transacció es desfà.
    • Consistència: Assegura que una transacció porta la base de dades d'un estat vàlid a un altre.
    • Aïllament: Assegura que les transaccions s'executen de manera aïllada les unes de les altres.
    • Durabilitat: Assegura que una vegada una transacció es compromet, es manté així, fins i tot en cas de fallada del sistema.

Gestionar Transaccions amb JDBC

JDBC proporciona una manera senzilla de gestionar transaccions en Java. Per defecte, JDBC opera en mode d'auto-compromís, el que significa que cada instrucció SQL es compromet a la base de dades tan aviat com s'executa. Per gestionar les transaccions manualment, l'auto-compromís s'ha de desactivar.

Exemple: Gestió de Transaccions amb JDBC

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;
import java.sql.Statement;

public class JdbcTransactionExample {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/mydatabase";
        String user = "root";
        String password = "password";

        Connection conn = null;
        Statement stmt = null;

        try {
            conn = DriverManager.getConnection(url, user, password);
            conn.setAutoCommit(false); // Desactivar auto-compromís

            stmt = conn.createStatement();
            stmt.executeUpdate("INSERT INTO accounts (name, balance) VALUES ('Alice', 1000)");
            stmt.executeUpdate("INSERT INTO accounts (name, balance) VALUES ('Bob', 1500)");

            conn.commit(); // Comprometre la transacció
            System.out.println("Transacció compromesa amb èxit.");
        } catch (SQLException e) {
            if (conn != null) {
                try {
                    conn.rollback(); // Desfer la transacció en cas d'error
                    System.out.println("Transacció desfeta.");
                } catch (SQLException ex) {
                    ex.printStackTrace();
                }
            }
            e.printStackTrace();
        } finally {
            try {
                if (stmt != null) stmt.close();
                if (conn != null) conn.close();
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }
    }
}

Explicació

  • Desactivar Auto-Compromís: conn.setAutoCommit(false); desactiva el mode d'auto-compromís.
  • Executar Instruccions SQL: Es realitzen múltiples instruccions SQL dins de la transacció.
  • Comprometre Transacció: conn.commit(); compromet la transacció.
  • Desfer Transacció: conn.rollback(); desfà la transacció si es produeix un error.

Gestionar Transaccions amb JPA

JPA proporciona una abstracció de més alt nivell per gestionar transaccions en comparació amb JDBC. Les transaccions en JPA es gestionen típicament utilitzant les interfícies EntityManager i EntityTransaction.

Exemple: Gestió de Transaccions amb JPA

import javax.persistence.EntityManager;
import javax.persistence.EntityManagerFactory;
import javax.persistence.EntityTransaction;
import javax.persistence.Persistence;

public class JpaTransactionExample {
    public static void main(String[] args) {
        EntityManagerFactory emf = Persistence.createEntityManagerFactory("my-persistence-unit");
        EntityManager em = emf.createEntityManager();
        EntityTransaction tx = em.getTransaction();

        try {
            tx.begin(); // Iniciar transacció

            Account alice = new Account("Alice", 1000);
            Account bob = new Account("Bob", 1500);

            em.persist(alice);
            em.persist(bob);

            tx.commit(); // Comprometre transacció
            System.out.println("Transacció compromesa amb èxit.");
        } catch (Exception e) {
            if (tx.isActive()) {
                tx.rollback(); // Desfer transacció en cas d'error
                System.out.println("Transacció desfeta.");
            }
            e.printStackTrace();
        } finally {
            em.close();
            emf.close();
        }
    }
}

Explicació

  • Iniciar Transacció: tx.begin(); inicia la transacció.
  • Persistir Entitats: em.persist(entity); persisteix les entitats dins de la transacció.
  • Comprometre Transacció: tx.commit(); compromet la transacció.
  • Desfer Transacció: tx.rollback(); desfà la transacció si es produeix un error.

Conclusió

Les transaccions són essencials per mantenir la integritat i consistència de les dades en les operacions de bases de dades. Java proporciona mecanismes robustos per gestionar transaccions a través de JDBC i JPA. Entendre com implementar i gestionar transaccions de manera efectiva és crucial per desenvolupar aplicacions fiables i eficients. Adherint-se a les propietats ACID, es pot assegurar que les transaccions siguin fiables i que les dades es mantinguin consistents.
