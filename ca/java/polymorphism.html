Polimorfisme en Java
Encapsulació ►

El polimorfisme és un dels conceptes clau de la Programació Orientada a Objectes (OOP) en Java. Permet que els objectes es tractin com a instàncies de la seva classe pare en lloc de la seva classe real. Això permet que una única interfície representi diferents formes subjacents (tipus de dades).

Què és el Polimorfisme?

El polimorfisme en Java es pot categoritzar en dos tipus:

  • Polimorfisme en temps de compilació (Sobrecàrrega de mètodes)
  • Polimorfisme en temps d'execució (Sobreescriptura de mètodes)

Polimorfisme en temps de compilació (Sobrecàrrega de mètodes)

La sobrecàrrega de mètodes es produeix quan múltiples mètodes en la mateixa classe tenen el mateix nom però diferents paràmetres (diferent tipus, nombre, o ambdós).

Exemple:

class MathOperations {
    // Mètode per sumar dos enters
    int add(int a, int b) {
        return a + b;
    }

    // Mètode per sumar tres enters
    int add(int a, int b, int c) {
        return a + b + c;
    }

    // Mètode per sumar dos valors dobles
    double add(double a, double b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        MathOperations math = new MathOperations();
        System.out.println(math.add(2, 3)); // Sortida: 5
        System.out.println(math.add(2, 3, 4)); // Sortida: 9
        System.out.println(math.add(2.5, 3.5)); // Sortida: 6.0
    }
}

Polimorfisme en temps d'execució (Sobreescriptura de mètodes)

La sobreescriptura de mètodes es produeix quan una subclasse proporciona una implementació específica per a un mètode que ja està definit en la seva superclasse. El mètode en la subclasse ha de tenir el mateix nom, tipus de retorn i paràmetres que el mètode en la superclasse.

Exemple:

class Animal {
    void sound() {
        System.out.println("Animal fa un so");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("El gos borda");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("El gat miola");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Animal(); // Referència i objecte Animal
        Animal myDog = new Dog(); // Referència Animal però objecte Dog
        Animal myCat = new Cat(); // Referència Animal però objecte Cat

        myAnimal.sound(); // Sortida: Animal fa un so
        myDog.sound(); // Sortida: El gos borda
        myCat.sound(); // Sortida: El gat miola
    }
}

Conceptes Clau del Polimorfisme

  • Herència: El polimorfisme es basa en l'herència. Una subclasse hereta mètodes de la seva superclasse i els pot sobreescriure.
  • Sobrecàrrega de mètodes: Múltiples mètodes amb el mateix nom però diferents paràmetres dins de la mateixa classe.
  • Sobreescriptura de mètodes: Una subclasse proporciona una implementació específica d'un mètode que ja està definit en la seva superclasse.
  • Despatx dinàmic de mètodes: El procés pel qual una crida a un mètode sobreescrit es resol en temps d'execució en lloc de temps de compilació.

Beneficis del Polimorfisme

  • Reutilització de codi: El polimorfisme permet la reutilització de codi a través de la sobreescriptura i sobrecàrrega de mètodes.
  • Flexibilitat i mantenibilitat: Fa que el codi sigui més flexible i fàcil de mantenir.
  • Extensibilitat: Es pot afegir nova funcionalitat amb canvis mínims al codi existent.

Conclusió

El polimorfisme és una característica poderosa en Java que millora la flexibilitat i mantenibilitat del codi. Comprenent i utilitzant la sobrecàrrega i sobreescriptura de mètodes, els programadors poden escriure aplicacions més eficients i escalables. Tant si ets un principiant com un desenvolupador avançat de Java, dominar el polimorfisme és essencial per a una programació orientada a objectes efectiva.
