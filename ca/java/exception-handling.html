Introducció a la Gestió d'Excepcions

La gestió d'excepcions és un mecanisme potent a Java per gestionar errors en temps d'execució, assegurant el flux normal de l'aplicació. Et permet gestionar errors de manera elegant i mantenir la robustesa del teu codi.

Conceptes Clau

  • Excepció: Un esdeveniment que interromp el flux normal del programa.
  • Gestió d'Excepcions: El procés de respondre a excepcions quan ocorren.
  • Bloc Try-Catch: Un bloc de codi utilitzat per gestionar excepcions.
  • Bloc Finally: Un bloc que s'executa independentment de si es llança una excepció o no.
  • Throw: Utilitzat per llançar explícitament una excepció.
  • Throws: Indica quines excepcions pot llançar un mètode.

Tipus d'Excepcions

Les excepcions a Java es divideixen en dues categories principals:

  1. Excepcions Comprovades: Excepcions que es comproven en temps de compilació.
  2. Excepcions No Comprovades: Excepcions que es comproven en temps d'execució.

Exemples

  • Excepció Comprovada: IOException, SQLException
  • Excepció No Comprovada: ArithmeticException, NullPointerException

Gestió Bàsica d'Excepcions

Bloc Try-Catch

El bloc try conté codi que podria llançar una excepció, i el bloc catch gestiona l'excepció.

public class ExceptionExample {
    public static void main(String[] args) {
        try {
            int divideByZero = 5 / 0;
        } catch (ArithmeticException e) {
            System.out.println("ArithmeticException caught: " + e.getMessage());
        }
    }
}

En aquest exemple, dividir per zero llança una ArithmeticException, que és capturada pel bloc catch.

Bloc Finally

El bloc finally sempre s'executa, independentment de si es llança una excepció o no.

public class FinallyExample {
    public static void main(String[] args) {
        try {
            int divideByZero = 5 / 0;
        } catch (ArithmeticException e) {
            System.out.println("ArithmeticException caught: " + e.getMessage());
        } finally {
            System.out.println("This is the finally block.");
        }
    }
}

Aquí, el bloc finally s'executarà fins i tot després que l'excepció sigui capturada.

Gestió Avançada d'Excepcions

Llançament d'Excepcions

Pots llançar una excepció utilitzant la paraula clau throw.

public class ThrowExample {
    public static void main(String[] args) {
        try {
            checkAge(15);
        } catch (Exception e) {
            System.out.println("Exception caught: " + e.getMessage());
        }
    }

    static void checkAge(int age) throws Exception {
        if (age < 18) {
            throw new Exception("Age is less than 18");
        }
    }
}

En aquest exemple, si l'edat és menor de 18, es llança una excepció.

Paraula Clau Throws

La paraula clau throws s'utilitza en les signatures de mètodes per declarar que un mètode pot llançar excepcions.

public class ThrowsExample {
    public static void main(String[] args) {
        try {
            checkFile();
        } catch (IOException e) {
            System.out.println("IOException caught: " + e.getMessage());
        }
    }

    static void checkFile() throws IOException {
        FileReader file = new FileReader("nonexistentfile.txt");
    }
}

Aquí, el mètode checkFile declara que pot llançar una IOException.

Excepcions Personalitzades

Pots crear les teves pròpies excepcions estenent la classe Exception.

class CustomException extends Exception {
    public CustomException(String message) {
        super(message);
    }
}

public class CustomExceptionExample {
    public static void main(String[] args) {
        try {
            validate(15);
        } catch (CustomException e) {
            System.out.println("CustomException caught: " + e.getMessage());
        }
    }

    static void validate(int age) throws CustomException {
        if (age < 18) {
            throw new CustomException("Age is less than 18");
        }
    }
}

En aquest exemple, es crea i es llança una excepció personalitzada CustomException si l'edat és menor de 18.

Millors Pràctiques

  • Sempre captura l'excepció més específica primer.
  • Evita capturar Exception tret que sigui necessari.
  • Neteja els recursos al bloc finally.
  • Documenta les excepcions que els teus mètodes poden llançar utilitzant la paraula clau throws.

Conclusió

La gestió d'excepcions és un aspecte crucial de la programació a Java. Et permet gestionar errors de manera elegant i assegurar la robustesa de les teves aplicacions. Entenent i aplicant els conceptes de blocs try-catch, blocs finally, llançament d'excepcions i creació d'excepcions personalitzades, pots escriure codi més fiable i mantenible.
