Introducció a les classes abstractes

Les classes abstractes en Java són un concepte fonamental que et permet definir classes que no es poden instanciar per si soles. S'utilitzen per proporcionar una base sobre la qual les subclasses poden construir, assegurant una estructura i comportament comuns.

Conceptes clau

  • Classe abstracta: Una classe declarada amb la paraula clau abstract. No es pot instanciar directament.
  • Mètode abstracte: Un mètode declarat sense una implementació, utilitzant la paraula clau abstract. Les subclasses han de proporcionar una implementació per a aquests mètodes.
  • Mètode concret: Un mètode en una classe abstracta que té una implementació. Les subclasses poden heretar o sobreescriure aquests mètodes.

Definició de classes abstractes

Una classe abstracta es defineix utilitzant la paraula clau abstract. Pot contenir tant mètodes abstractes (sense cos) com mètodes concrets (amb cos).

abstract class Animal {
    // Mètode abstracte (no té cos)
    abstract void makeSound();

    // Mètode concret (té cos)
    void sleep() {
        System.out.println("Zzz...");
    }
}

Exemple explicat

  • Animal és una classe abstracta.
  • makeSound() és un mètode abstracte, el que significa que les subclasses han d'implementar aquest mètode.
  • sleep() és un mètode concret, el que significa que proporciona una implementació per defecte que pot ser heretada per les subclasses.

Implementació de classes abstractes

Per utilitzar una classe abstracta, necessites crear una subclasse que ampliï la classe abstracta i proporcioni implementacions per a tots els mètodes abstractes.

class Dog extends Animal {
    // Proporcionant implementació per al mètode abstracte
    void makeSound() {
        System.out.println("Woof");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.makeSound(); // Mostra: Woof
        myDog.sleep();     // Mostra: Zzz...
    }
}

Exemple explicat

  • Dog és una subclasse de Animal.
  • Dog proporciona una implementació per al mètode makeSound().
  • En el mètode main, creem una instància de Dog i cridem els seus mètodes.

Classes abstractes vs Interfícies

Tant les classes abstractes com les interfícies s'utilitzen per aconseguir l'abstracció en Java, però tenen algunes diferències.

| Característica | Classe abstracta | Interfície | |------------------------|----------------------------------------|----------------------------------------| | Mètodes | Pot tenir tant mètodes abstractes com concrets | Només pot tenir mètodes abstractes (Java 7 i anteriors) o mètodes per defecte/estàtics (Java 8 i posteriors) | | Camps | Pot tenir variables d'instància | Només pot tenir variables finals estàtiques | | Herència múltiple | No suporta herència múltiple | Suporta herència múltiple | | Constructor | Pot tenir constructors | No pot tenir constructors |

Exemple d'interfície per a comparació

interface AnimalInterface {
    void makeSound();
    default void sleep() {
        System.out.println("Zzz...");
    }
}

class Cat implements AnimalInterface {
    public void makeSound() {
        System.out.println("Meow");
    }
}

public class Main {
    public static void main(String[] args) {
        Cat myCat = new Cat();
        myCat.makeSound(); // Mostra: Meow
        myCat.sleep();     // Mostra: Zzz...
    }
}

Quan utilitzar classes abstractes

  • Quan vols compartir codi entre diverses classes estretament relacionades.
  • Quan esperes que les classes que ampliïn la teva classe abstracta tinguin molts mètodes o camps comuns.
  • Quan vols definir camps no estàtics o no finals.

Conclusió

Les classes abstractes en Java proporcionen una manera poderosa de definir una plantilla comuna per a un grup de classes relacionades. Permeten fer complir un contracte per a les subclasses mentre també proporcionen codi compartit. Entendre quan i com utilitzar classes abstractes és crucial per dissenyar sistemes orientats a objectes robustos i mantenibles.
