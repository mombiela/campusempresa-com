El multithreading és una característica poderosa en Java que permet l'execució concurrent de dos o més fils per a una utilització màxima de la CPU. Aquest tema et guiarà a través dels conceptes bàsics del multithreading, els seus beneficis i com implementar-lo en Java, progressant des de conceptes per a principiants fins a conceptes avançats.

Què és el Multithreading?

  • Definició: El multithreading és la capacitat d'una CPU, o d'un sol nucli en un processador multi-nucli, de proporcionar múltiples fils d'execució simultàniament, suportats pel sistema operatiu.
  • Fil: Un fil és un procés lleuger, i el multithreading és el procés d'executar múltiples fils simultàniament.

Beneficis del Multithreading

  • Millora del Rendiment: Permetent que múltiples fils s'executin simultàniament, els programes poden realitzar múltiples operacions al mateix temps.
  • Compartició de Recursos: Els fils dins del mateix procés comparteixen recursos com la memòria, cosa que pot conduir a un ús més eficient dels recursos.
  • Responsivitat: El multithreading pot fer que les aplicacions siguin més responsives, especialment en interfícies d'usuari on un fil pot gestionar l'entrada de l'usuari mentre un altre realitza tasques en segon pla.

Creació de Fils en Java

Extenent la Classe Thread

Una manera de crear un fil en Java és estenent la classe Thread.

class MyThread extends Thread {
    public void run() {
        System.out.println("El fil està funcionant.");
    }

    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        t1.start(); // Inicia el fil
    }
}
  • Explicació:
    • La classe MyThread estèn la classe Thread.
    • El mètode run conté el codi que constitueix el nou fil.
    • El mètode start es crida per començar l'execució del fil.

Implementant la Interfície Runnable

Una altra manera de crear un fil és implementant la interfície Runnable.

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("El fil està funcionant.");
    }

    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        Thread t1 = new Thread(myRunnable);
        t1.start(); // Inicia el fil
    }
}
  • Explicació:
    • La classe MyRunnable implementa la interfície Runnable.
    • El mètode run conté el codi que constitueix el nou fil.
    • Es crea un objecte Thread amb MyRunnable com a objectiu, i es crida el mètode start per començar l'execució.

Cicle de Vida del Fil

Un fil en Java pot estar en un dels següents estats:

  • Nou: Un fil que ha estat creat però encara no ha començat.
  • Runnable: Un fil que està llest per córrer i està esperant temps de CPU.
  • Bloquejat: Un fil que està bloquejat esperant un monitor de bloqueig.
  • Esperant: Un fil que està esperant indefinidament que un altre fil realitzi una acció particular.
  • Esperant amb Temps: Un fil que està esperant que un altre fil realitzi una acció durant un temps d'espera especificat.
  • Terminat: Un fil que ha sortit.

Sincronització

Quan múltiples fils accedeixen a recursos compartits, poden sorgir problemes com la inconsistència de dades. La sincronització s'utilitza per controlar l'accés de múltiples fils als recursos compartits.

Mètode Sincronitzat

class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public int getCount() {
        return count;
    }
}

class MyThread extends Thread {
    Counter counter;

    MyThread(Counter counter) {
        this.counter = counter;
    }

    public void run() {
        for (int i = 0; i < 1000; i++) {
            counter.increment();
        }
    }

    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();
        MyThread t1 = new MyThread(counter);
        MyThread t2 = new MyThread(counter);
        t1.start();
        t2.start();
        t1.join();
        t2.join();
        System.out.println("Count: " + counter.getCount());
    }
}
  • Explicació:
    • El mètode increment està sincronitzat, assegurant que només un fil pot executar-lo a la vegada.
    • Dos fils (t1 i t2) incrementen el comptador simultàniament.
    • El mètode join assegura que el fil principal espera que t1 i t2 acabin abans d'imprimir el comptador.

Conceptes Avançats

Pools de Fils

Els pools de fils gestionen un pool de fils de treball, reduint la sobrecàrrega de crear i destruir fils.

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("El fil està funcionant.");
    }

    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(5);
        for (int i = 0; i < 10; i++) {
            executor.execute(new MyRunnable());
        }
        executor.shutdown();
    }
}
  • Explicació:
    • ExecutorService s'utilitza per crear un pool de fils.
    • Executors.newFixedThreadPool(5) crea un pool de 5 fils.
    • El mètode execute s'utilitza per enviar tasques per a l'execució.

Callable i Future

Callable és similar a Runnable però pot retornar un resultat i llençar una excepció verificada.

import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.Future;

class MyCallable implements Callable<Integer> {
    public Integer call() throws Exception {
        return 123;
    }

    public static void main(String[] args) throws InterruptedException, ExecutionException {
        ExecutorService executor = Executors.newFixedThreadPool(5);
        Future<Integer> future = executor.submit(new MyCallable());
        System.out.println("Resultat: " + future.get());
        executor.shutdown();
    }
}
  • Explicació:
    • MyCallable implementa Callable i retorna un enter.
    • El mètode submit s'utilitza per enviar una tasca Callable.
    • El mètode get de Future s'utilitza per recuperar el resultat.

Conclusió

El multithreading en Java és una característica complexa però poderosa que pot millorar enormement el rendiment i la responsivitat de les aplicacions. Entenent els conceptes bàsics de la creació de fils, la sincronització i conceptes avançats com els pools de fils i Callable, pots utilitzar efectivament el multithreading en els teus programes Java. Practica amb els exemples i exercicis proporcionats per solidificar la teva comprensió i convertir-te en un expert en multithreading.
