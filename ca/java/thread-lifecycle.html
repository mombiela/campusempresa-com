Introducció

En Java, un fil és un procés lleuger que permet l'execució concurrent de codi. Entendre el cicle de vida d'un fil és crucial per escriure aplicacions multithreaded eficients i efectives. Aquesta secció cobrirà els diferents estats en què pot estar un fil, com transitar entre aquests estats, i proporcionarà exemples de codi per il·lustrar aquests conceptes.

Estats dels fils

Un fil en Java pot estar en un dels següents estats:

  1. Nou
  2. Runnable
  3. Bloquejat
  4. Esperant
  5. Esperant amb temps limitat
  6. Terminat

Nou

  • Un fil està en l'estat "Nou" quan es crea però encara no s'ha iniciat.
  • Roman en aquest estat fins que es crida el mètode start().
Thread thread = new Thread(() -> {
    System.out.println("El fil s'està executant");
});
// El fil està en estat Nou

Runnable

  • Un fil entra en l'estat "Runnable" quan es crida el mètode start().
  • En aquest estat, el fil està llest per executar-se i està esperant temps de CPU.
thread.start(); // El fil ara està en estat Runnable

Bloquejat

  • Un fil entra en l'estat "Bloquejat" quan està esperant un bloqueig de monitor per entrar en un bloc/mètode sincronitzat.
  • Transita de nou a l'estat "Runnable" un cop adquireix el bloqueig.
synchronized (someObject) {
    // El fil està en estat Bloquejat si no pot adquirir el bloqueig
}

Esperant

  • Un fil entra en l'estat "Esperant" quan està esperant indefinidament que un altre fil realitzi una acció particular.
  • Els mètodes que poden causar que un fil entri en aquest estat inclouen Object.wait(), Thread.join(), i LockSupport.park().
synchronized (someObject) {
    someObject.wait(); // El fil està en estat Esperant
}

Esperant amb temps limitat

  • Un fil entra en l'estat "Esperant amb temps limitat" quan està esperant que un altre fil realitzi una acció durant un temps d'espera especificat.
  • Els mètodes que poden causar que un fil entri en aquest estat inclouen Thread.sleep(), Object.wait(long timeout), Thread.join(long millis), LockSupport.parkNanos(), i LockSupport.parkUntil().
Thread.sleep(1000); // El fil està en estat Esperant amb temps limitat durant 1 segon

Terminat

  • Un fil entra en l'estat "Terminat" quan ha completat la seva execució o ha estat terminat explícitament.
  • Un cop un fil està en aquest estat, no es pot reiniciar.
// El fil està en estat Terminat després que el mètode run() es completi

Transicions del cicle de vida del fil

La següent taula resumeix les transicions entre els diferents estats del fil:

| Estat actual | Mètode de transició | Estat següent | |---------------|---------------------|------------------| | Nou | start() | Runnable | | Runnable | Adquireix temps de CPU | Executant | | Executant | yield() | Runnable | | Executant | wait() | Esperant | | Executant | sleep() | Esperant amb temps limitat | | Executant | join() | Esperant/Esperant amb temps limitat | | Executant | Completa run() | Terminat | | Esperant | notify()/notifyAll() | Runnable | | Esperant amb temps limitat | El temps expira | Runnable | | Bloquejat | Adquireix el bloqueig | Runnable |

Exemple de codi

A continuació es mostra un exemple complet que demostra els diferents estats d'un fil:

public class ThreadLifecycleDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            try {
                System.out.println("El fil està en estat Runnable");
                Thread.sleep(1000); // Estat Esperant amb temps limitat
                synchronized (ThreadLifecycleDemo.class) {
                    ThreadLifecycleDemo.class.wait(); // Estat Esperant
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("El fil està en estat Terminat");
        });

        System.out.println("El fil està en estat Nou");
        thread.start(); // Estat Runnable

        try {
            Thread.sleep(500); // El fil principal dorm per permetre que el fil fill comenci
            synchronized (ThreadLifecycleDemo.class) {
                ThreadLifecycleDemo.class.notify(); // Notifica el fil esperant
            }
            thread.join(); // El fil principal espera que el fil fill es termini
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

Conclusió

Entendre el cicle de vida dels fils és essencial per gestionar els estats i les transicions dels fils de manera efectiva en una aplicació multithreaded en Java. En dominar aquests conceptes, pots escriure programes més eficients i robustos que facin un millor ús dels recursos del sistema.
