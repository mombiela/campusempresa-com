Introducció al Disseny i Implementació

El disseny i la implementació són etapes crucials en el desenvolupament de programari. En aquesta secció, tractarem els principis de dissenyar aplicacions robustes en Java i implementar-les de manera efectiva. Això inclou entendre els patrons de disseny, escriure codi net i seguir les millors pràctiques.

Conceptes Clau en Disseny i Implementació

  • Principis de Disseny de Programari: SOLID, DRY, KISS, YAGNI
  • Patrons de Disseny: Singleton, Factory, Observer, Strategy, etc.
  • Llegibilitat i Mantenibilitat del Codi: Convencions de nomenclatura, comentaris i documentació
  • Proves i Depuració: Proves unitàries, proves d'integració i tècniques de depuració
  • Control de Versions: Ús de Git per al control de versions

Principis de Disseny de Programari

Principis SOLID

  • Principi de Responsabilitat Única (SRP): Una classe hauria de tenir només una raó per canviar.
  • Principi Obert/Tancat (OCP): Les entitats de programari haurien d'estar obertes a l'extensió però tancades a la modificació.
  • Principi de Substitució de Liskov (LSP): Els subtipus han de ser substituïbles pels seus tipus base.
  • Principi de Segregació d'Interfícies (ISP): Moltes interfícies específiques per a clients són millors que una interfície de propòsit general.
  • Principi d'Inversió de Dependències (DIP): Depèn de les abstraccions, no de les implementacions concretes.

Exemple: Principi de Responsabilitat Única

// Abans d'aplicar SRP
class User {
    public void login(String username, String password) {
        // lògica de login
    }

    public void register(String username, String password, String email) {
        // lògica de registre
    }

    public void sendEmail(String email, String message) {
        // lògica d'enviament de correu electrònic
    }
}

// Després d'aplicar SRP
class User {
    public void login(String username, String password) {
        // lògica de login
    }

    public void register(String username, String password, String email) {
        // lògica de registre
    }
}

class EmailService {
    public void sendEmail(String email, String message) {
        // lògica d'enviament de correu electrònic
    }
}

En aquest exemple, la classe User inicialment tenia múltiples responsabilitats. Aplicant el SRP, vam separar la funcionalitat de correu electrònic en la seva pròpia classe EmailService.

Patrons de Disseny

Patró Singleton

El patró Singleton assegura que una classe tingui només una instància i proporciona un punt d'accés global a aquesta.

public class Singleton {
    private static Singleton instance;

    private Singleton() {
        // constructor privat
    }

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

En aquest exemple, la classe Singleton té un constructor privat i un mètode estàtic per obtenir la instància única de la classe.

Patró Factory

El patró Factory proporciona una manera de crear objectes sense especificar la classe exacta de l'objecte que es crearà.

public interface Shape {
    void draw();
}

public class Circle implements Shape {
    public void draw() {
        System.out.println("Dibuixant Cercle");
    }
}

public class Square implements Shape {
    public void draw() {
        System.out.println("Dibuixant Quadrat");
    }
}

public class ShapeFactory {
    public Shape getShape(String shapeType) {
        if (shapeType == null) {
            return null;
        }
        if (shapeType.equalsIgnoreCase("CIRCLE")) {
            return new Circle();
        } else if (shapeType.equalsIgnoreCase("SQUARE")) {
            return new Square();
        }
        return null;
    }
}

En aquest exemple, la classe ShapeFactory s'utilitza per obtenir objectes de diferents formes sense exposar la lògica de creació al client.

Llegibilitat i Mantenibilitat del Codi

  • Convencions de Nomenclatura: Utilitza noms significatius i descriptius per a variables, mètodes i classes.
  • Comentaris i Documentació: Escriu comentaris per explicar la lògica complexa i utilitza Javadoc per a la documentació de l'API.
  • Format Consistent: Segueix un estil de codi i regles de formatació consistents.

Exemple: Convencions de Nomenclatura

// Mala nomenclatura
int a = 10;
String b = "John";

// Bona nomenclatura
int age = 10;
String name = "John";

Proves i Depuració

  • Proves Unitàries: Escriure proves per a unitats individuals de codi utilitzant frameworks com JUnit.
  • Proves d'Integració: Provar la integració de diferents mòduls.
  • Tècniques de Depuració: Utilitzar punts de ruptura, registre i eines de depuració.

Exemple: Prova Unitària amb JUnit

import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {
    @Test
    public void testAdd() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
}

En aquest exemple, utilitzem JUnit per escriure una prova unitària per al mètode add de la classe Calculator.

Control de Versions

  • Ús de Git: Aprèn a utilitzar Git per al control de versions, incloent comandes com clone, commit, push, pull i merge.

Exemple: Comandes Bàsiques de Git

# Clonar un repositori
git clone https://github.com/username/repository.git

# Afegir canvis a l'àrea d'staging
git add .

# Fer un commit dels canvis
git commit -m "Missatge del commit"

# Empènyer els canvis al repositori remot
git push origin main

Conclusió

El disseny i la implementació són aspectes fonamentals del desenvolupament de programari. Entenent i aplicant els principis de disseny, patrons de disseny i millors pràctiques, pots crear aplicacions en Java robustes, mantenibles i escalables. Les proves i el control de versions també són essencials per assegurar la qualitat i consistència del teu codi. Practica aquests conceptes regularment per convertir-te en un expert en dissenyar i implementar aplicacions en Java.
