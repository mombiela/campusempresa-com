Introducció a les eines de construcció

Les eines de construcció són essencials en el desenvolupament de programari modern per automatitzar tasques com compilar codi, gestionar dependències, executar proves i empaquetar aplicacions. En aquesta secció, cobrirem dues eines de construcció populars per a Java: Maven i Gradle.

Maven

Maven és una eina d'automatització de construcció utilitzada principalment per a projectes Java. Utilitza un fitxer XML per descriure l'estructura del projecte i les dependències.

Conceptes clau

  • POM (Project Object Model): La unitat fonamental de treball en Maven, definida en un fitxer pom.xml.
  • Dependències: Biblioteques externes que el teu projecte necessita per funcionar.
  • Plugins: Extensions que proporcionen funcionalitats addicionals, com compilar codi o executar proves.
  • Fases del cicle de vida: Etapes predefinides en el procés de construcció, com validate, compile, test, package, verify, install, i deploy.

Estructura bàsica d'un projecte Maven

my-app
|-- src
|   |-- main
|   |   |-- java
|   |   |-- resources
|   |-- test
|       |-- java
|       |-- resources
|-- pom.xml

Exemple de `pom.xml`

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0-SNAPSHOT</version>

    <dependencies>
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>

Comandes comunes de Maven

  • mvn compile: Compila el codi font.
  • mvn test: Executa les proves.
  • mvn package: Empaqueta el codi compilat en un fitxer JAR.
  • mvn install: Instal·la el paquet al repositori local.

Gradle

Gradle és una eina d'automatització de construcció flexible que utilitza un DSL (Domain Specific Language) basat en Groovy o Kotlin per als seus scripts de construcció.

Conceptes clau

  • Script de construcció: El fitxer d'script (build.gradle o build.gradle.kts) que defineix la configuració de la construcció.
  • Tasques: Unitats de treball que Gradle executa, com compilar codi o executar proves.
  • Dependències: Biblioteques externes necessàries pel projecte.
  • Plugins: Extensions que afegeixen funcionalitat al procés de construcció.

Estructura bàsica d'un projecte Gradle

my-app
|-- src
|   |-- main
|   |   |-- java
|   |   |-- resources
|   |-- test
|       |-- java
|       |-- resources
|-- build.gradle

Exemple de `build.gradle`

plugins {
    id 'java'
}

group 'com.example'
version '1.0-SNAPSHOT'

repositories {
    mavenCentral()
}

dependencies {
    testImplementation 'junit:junit:4.12'
}

test {
    useJUnitPlatform()
}

Comandes comunes de Gradle

  • gradle build: Compila i prova el codi, després l'empaqueta.
  • gradle test: Executa les proves.
  • gradle clean: Esborra el directori de construcció.
  • gradle assemble: Compila el codi i l'empaqueta sense executar proves.

Comparació de Maven i Gradle

| Característica | Maven | Gradle | |----------------------|--------------------------------|-------------------------------| | Configuració | XML (pom.xml) | Groovy/Kotlin DSL (build.gradle) | | Rendiment | Més lent degut a l'anàlisi XML | Més ràpid amb construccions incrementals | | Flexibilitat | Convenció sobre configuració | Altament personalitzable | | Gestió de dependències | Centralitzada en pom.xml | Centralitzada en build.gradle | | Suport de la comunitat | Gran, comunitat madura | Comunitat creixent i moderna |

Conclusió

Tant Maven com Gradle són eines de construcció poderoses que poden agilitzar significativament el procés de desenvolupament. Maven és conegut per la seva simplicitat i enfocament de convenció sobre configuració, fent-lo una bona elecció per a projectes estàndard. Gradle, per altra banda, ofereix una major flexibilitat i rendiment, fent-lo adequat per a construccions més complexes. Entendre ambdues eines et proporcionarà el coneixement per triar l'eina adequada per a les necessitats del teu projecte.
