Introducció a les Proves Unitàries

Les proves unitàries són un mètode de prova de programari on es proven unitats o components individuals d'un programari. L'objectiu és validar que cada unitat del programari funcioni com s'espera.

  • Unitat: La part més petita testejable de qualsevol programari.
  • Prova Unitària: Un fragment de codi escrit per un desenvolupador que executa una funció específica en el codi per provar el seu comportament.

Què és JUnit?

JUnit és un marc de proves popular per a Java que permet als desenvolupadors escriure i executar proves repetibles. És un marc de codi obert utilitzat per escriure i executar proves, proporcionant anotacions per identificar mètodes de prova i afirmacions per provar resultats esperats.

Configuració de JUnit

Abans de començar a escriure proves amb JUnit, cal configurar l'entorn de desenvolupament.

Afegir JUnit al Teu Projecte

Per a un projecte Maven, afegeix la següent dependència al teu fitxer pom.xml:

<dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.13.2</version>
    <scope>test</scope>
</dependency>

Per a un projecte Gradle, afegeix el següent al teu fitxer build.gradle:

testImplementation 'junit:junit:4.13.2'

Escriure la Teva Primera Prova

Escrivim un cas de prova senzill utilitzant JUnit.

Exemple: Provar una Classe Calculadora

Calculator.java

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

CalculatorTest.java

import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {
    @Test
    public void testAdd() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
}

Explicació

  • @Test: Aquesta anotació identifica el mètode testAdd com un mètode de prova.
  • assertEquals: Aquesta afirmació comprova si el resultat esperat (5) coincideix amb el resultat real retornat pel mètode add.

Funcions Avançades de JUnit

Proves Parametritzades

Les proves parametritzades permeten a un desenvolupador executar la mateixa prova una i altra vegada utilitzant diferents valors.

Exemple: Prova Parametritzada per a la Calculadora

import org.junit.Test;
import org.junit.runner.RunWith;
import org.junit.runners.Parameterized;
import java.util.Arrays;
import java.util.Collection;
import static org.junit.Assert.assertEquals;

@RunWith(Parameterized.class)
public class CalculatorParameterizedTest {
    private int a;
    private int b;
    private int expected;

    public CalculatorParameterizedTest(int a, int b, int expected) {
        this.a = a;
        this.b = b;
        this.expected = expected;
    }

    @Parameterized.Parameters
    public static Collection<Object[]> data() {
        return Arrays.asList(new Object[][] {
            { 1, 1, 2 },
            { 2, 3, 5 },
            { 3, 3, 6 }
        });
    }

    @Test
    public void testAdd() {
        Calculator calculator = new Calculator();
        assertEquals(expected, calculator.add(a, b));
    }
}

Explicació

  • @RunWith(Parameterized.class): Aquesta anotació indica a JUnit que executi la classe de prova utilitzant el runner Parameterized.
  • @Parameterized.Parameters: Aquesta anotació proporciona els paràmetres per a la prova.

Suites de Proves

Una suite de proves agrupa uns quants casos de proves unitàries i els executa junts.

Exemple: Suite de Proves

import org.junit.runner.RunWith;
import org.junit.runners.Suite;

@RunWith(Suite.class)
@Suite.SuiteClasses({
    CalculatorTest.class,
    CalculatorParameterizedTest.class
})
public class CalculatorTestSuite {
}

Explicació

  • @RunWith(Suite.class): Aquesta anotació indica a JUnit que executi la classe de prova utilitzant el runner Suite.
  • @Suite.SuiteClasses: Aquesta anotació especifica les classes a incloure a la suite.

Millors Pràctiques per a les Proves Unitàries

  • Escriu Proves per a Cada Funció: Assegura't que cada funció del teu codi tingui proves unitàries corresponents.
  • Mantingues les Proves Independents: Les proves no haurien de dependre les unes de les altres.
  • Utilitza Noms de Prova Significatius: Els noms de les proves haurien de descriure clarament què estan provant.
  • Prova Casos Extrems: Assegura't de provar casos extrems i no només el camí feliç.

Conclusió

Les proves unitàries són una part crucial del procés de desenvolupament de programari, assegurant que els components individuals funcionin com s'espera. JUnit proporciona un marc robust per escriure i executar proves en Java. Seguint les millors pràctiques i utilitzant funcions avançades com les proves parametritzades i les suites de proves, pots crear casos de prova complets i mantenibles per a les teves aplicacions.
