Els genèrics a Java permeten crear classes, interfícies i mètodes amb un marcador de posició per al tipus de dades amb què operen. Això proporciona comprovacions de tipus més fortes en temps de compilació i elimina la necessitat de fer conversions de tipus.

Introducció als Genèrics

  • Definició: Els genèrics permeten que els tipus (classes i interfícies) siguin paràmetres quan es defineixen classes, interfícies i mètodes.
  • Propòsit: Proporcionen una manera de reutilitzar el mateix codi amb diferents entrades i proporcionen comprovacions de tipus més fortes en temps de compilació.

Sintaxi Bàsica

// Un exemple de classe genèrica
public class Box<T> {
    private T t;

    public void set(T t) {
        this.t = t;
    }

    public T get() {
        return t;
    }
}

En aquest exemple, T és un paràmetre de tipus que serà substituït per un tipus concret quan es creï una instància de Box.

Beneficis d'Usar Genèrics

  • Seguretat de Tipus: Assegura que obtens comprovacions en temps de compilació i evites ClassCastException.
  • Reutilització de Codi: Escriu un mètode o classe una vegada i utilitza'l per a qualsevol tipus.
  • Eliminació de la Conversió de Tipus: Redueix la necessitat de fer conversions de tipus explícites.

Exemple Sense Genèrics

List list = new ArrayList();
list.add("Hello");
String s = (String) list.get(0); // Necessita conversió

Exemple Amb Genèrics

List<String> list = new ArrayList<>();
list.add("Hello");
String s = list.get(0); // No necessita conversió

Mètodes Genèrics

També pots definir mètodes genèrics a Java.

Sintaxi

public static <T> void printArray(T[] array) {
    for (T element : array) {
        System.out.println(element);
    }
}

Aquí, <T> abans del tipus de retorn void indica que el mètode és genèric.

Exemple d'Ús

Integer[] intArray = {1, 2, 3, 4, 5};
String[] stringArray = {"Hello", "World"};

printArray(intArray); // Imprimeix enters
printArray(stringArray); // Imprimeix cadenes

Paràmetres de Tipus Acotats

De vegades vols restringir els tipus que es poden utilitzar com a arguments de tipus. Això es fa utilitzant paràmetres de tipus acotats.

Límits Superiors

public <T extends Number> void printNumbers(T[] numbers) {
    for (T number : numbers) {
        System.out.println(number);
    }
}

En aquest exemple, T només pot ser un tipus que extengui Number.

Límits Inferiors

public static void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}

Aquí, ? super Integer significa qualsevol tipus que sigui un supertipus de Integer.

Comodins en Genèrics

Els comodins s'utilitzen per especificar un tipus desconegut.

Comodí Sense Acotar

public void printList(List<?> list) {
    for (Object element : list) {
        System.out.println(element);
    }
}

<?> representa un tipus desconegut.

Comodins Acotats

  • Comodí Acotat Superior: <? extends Number>
  • Comodí Acotat Inferior: <? super Integer>

Exemple

public void processElements(List<? extends Number> list) {
    for (Number number : list) {
        System.out.println(number);
    }
}

Classes Genèriques

Pots crear classes que poden treballar amb qualsevol tipus.

Exemple

public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() {
        return key;
    }

    public V getValue() {
        return value;
    }
}

Aquí, K i V són paràmetres de tipus.

Conclusió

Els genèrics a Java proporcionen una manera potent i flexible d'escriure codi segur i reutilitzable. Entenent i utilitzant els genèrics, pots crear aplicacions més robustes i mantenibles. Recorda utilitzar paràmetres de tipus acotats i comodins per millorar encara més la flexibilitat i seguretat del teu codi genèric.
