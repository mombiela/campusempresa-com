Introducció

Les proves i la depuració són habilitats crucials per a qualsevol programador de Java. Aquesta secció et guiarà a través dels fonaments de les proves i la depuració, des de conceptes bàsics fins a tècniques avançades.

Proves Unitàries amb JUnit

Les proves unitàries impliquen provar components individuals del teu codi per assegurar-te que funcionen com s'espera.

Conceptes Clau

  • Prova Unitària: Una prova que verifica el comportament d'una petita part de l'aplicació, típicament un sol mètode.
  • JUnit: Un marc popular per escriure i executar proves unitàries en Java.

Escriure la Teva Primera Prova Unitària

Per començar amb JUnit, necessites afegir la biblioteca JUnit al teu projecte. Aquí tens un exemple senzill:

import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
}

Explicació

  • import org.junit.Test: Importa l'anotació Test de JUnit.
  • import static org.junit.Assert.assertEquals: Importa el mètode assertEquals per a les afirmacions.
  • @Test: Marca el mètode testAddition com un mètode de prova.
  • assertEquals(5, result): Comprova si el resultat és igual a 5.

Executar Proves

Pots executar les teves proves utilitzant el teu IDE o una eina de construcció com Maven o Gradle. La majoria dels IDEs tenen suport integrat per executar proves JUnit.

Desenvolupament Guiat per Proves (TDD)

El Desenvolupament Guiat per Proves és un procés de desenvolupament de programari on escrius proves abans d'escriure el codi real.

Conceptes Clau

  • Red-Green-Refactor: El cicle TDD:
    1. Red: Escriu una prova que falla.
    2. Green: Escriu codi per fer que la prova passi.
    3. Refactor: Millora el codi mantenint la prova passant.

Exemple

  1. Red: Escriu una prova que falla. 
    @Test
public void testSubtraction() {
    Calculator calculator = new Calculator();
    int result = calculator.subtract(5, 3);
    assertEquals(2, result);
}
  2. Green: Escriu codi per fer que la prova passi. 
    public int subtract(int a, int b) {
    return a - b;
}
  3. Refactor: Millora el codi si és necessari.

Tècniques de Depuració

La depuració és el procés d'identificar i corregir errors en el teu codi.

Conceptes Clau

  • Punts de Trencament: Punts en el teu codi on l'execució es pausarà, permetent-te inspeccionar l'estat de l'aplicació.
  • Step Over: Executa la línia de codi actual i pausa a la següent línia.
  • Step Into: Entra en el mètode que s'està cridant i pausa a la primera línia d'aquest mètode.
  • Step Out: Completa el mètode actual i pausa a la línia on es va cridar el mètode.

Utilitzar un IDE per a la Depuració

La majoria dels IDEs moderns proporcionen eines robustes de depuració. Aquí tens com utilitzar-les:

  1. Estableix un Punt de Trencament: Fes clic al marge esquerre al costat del número de línia on vols pausar l'execució.
  2. Inicia la Depuració: Executa la teva aplicació en mode de depuració.
  3. Inspecciona Variables: Passa el cursor sobre les variables per veure els seus valors actuals.
  4. Utilitza els Controls de Depuració: Utilitza els botons de step over, step into, i step out per navegar pel teu codi.

Exemple

Considera el següent codi amb un error:

public class Calculator {
    public int divide(int a, int b) {
        return a / b;
    }
}

Per depurar un error de divisió per zero:

  1. Estableix un punt de trencament a return a / b;.
  2. Executa l'aplicació en mode de depuració.
  3. Quan l'execució es pausa, inspecciona els valors de a i b.
  4. Identifica que b és zero i corregeix el codi per manejar aquest cas.

Tècniques Avançades de Proves

A mesura que et sentis més còmode amb les proves i la depuració bàsiques, pots explorar tècniques més avançades.

Mocking amb Mockito

Mockito és un marc que et permet crear objectes simulats per a fins de prova.

Exemple

import static org.mockito.Mockito.*;
import org.junit.Test;

public class ServiceTest {

    @Test
    public void testService() {
        // Crea un objecte simulat
        Database mockDatabase = mock(Database.class);
        
        // Defineix el comportament per a l'objecte simulat
        when(mockDatabase.getData()).thenReturn("Mock Data");
        
        // Utilitza l'objecte simulat en la teva prova
        Service service = new Service(mockDatabase);
        String result = service.getData();
        
        assertEquals("Mock Data", result);
    }
}

Explicació

  • mock(Database.class): Crea un objecte simulat de la classe Database.
  • when(mockDatabase.getData()).thenReturn("Mock Data"): Defineix el comportament de l'objecte simulat.
  • Service service = new Service(mockDatabase): Injecta l'objecte simulat en el servei.

Conclusió

Les proves i la depuració són habilitats essencials per a qualsevol programador de Java. Dominant les proves unitàries amb JUnit, adoptant el Desenvolupament Guiat per Proves, i utilitzant eines de depuració en el teu IDE, pots escriure codi més fiable i mantenible. A mesura que avancis, explorar marcs com Mockito per a la simulació pot millorar encara més les teves capacitats de prova. Practica aquestes tècniques regularment per convertir-te en un expert en identificar i corregir problemes en el teu codi.
