Què són els Globals?

Els globals en MUMPS (M) són un concepte fonamental que permet l'emmagatzematge i la recuperació de dades persistents. A diferència de les variables locals, els globals s'emmagatzemen en disc i poden ser accessibles per diferents processos, fent-los essencials per a les operacions de base de dades.

  • Emmagatzematge Persistent: Els globals s'emmagatzemen en disc, assegurant la persistència de les dades entre sessions.
  • Accés Multi-Proces: Els globals poden ser accessibles i modificats per múltiples processos, permetent operacions de dades concurrents.
  • Estructura Jeràrquica: Els globals estan organitzats en una estructura en forma d'arbre, permetent relacions de dades complexes.

Sintaxi i Estructura dels Globals

Sintaxi Bàsica

Els globals s'identifiquen per un accent circumflex (^) seguit d'un nom. Poden tenir subíndexs per representar dades jeràrquiques.

^GlobalName(subscript1, subscript2, ...)

Exemple

; Establint un valor global
SET ^Patient(12345, "Name") = "John Doe"

; Recuperant un valor global
WRITE ^Patient(12345, "Name")

En aquest exemple:

  • ^Patient és el nom del global.
  • 12345 és el primer subíndex, representant un ID de pacient.
  • "Name" és el segon subíndex, representant el nom del pacient.
  • El valor "John Doe" s'emmagatzema en aquesta ubicació.

Conceptes Clau

  • Nodes Globals: Cada combinació de subíndexs en un global representa un node.
  • Arrays Globals: Els globals es poden considerar com arrays multidimensionals.
  • Recorregut de Globals: Pots recórrer els globals utilitzant comandes especials com ORDER i NEXT.

Exemple de Recorregut

; Establint múltiples valors globals
SET ^Patient(12345, "Name") = "John Doe"
SET ^Patient(12345, "Age") = 30
SET ^Patient(67890, "Name") = "Jane Smith"
SET ^Patient(67890, "Age") = 25

; Recorregut de nodes globals
SET patientID = ""
FOR  SET patientID = $ORDER(^Patient(patientID)) QUIT:patientID=""  DO
. WRITE "Patient ID: ", patientID, !
. WRITE "Name: ", ^Patient(patientID, "Name"), !
. WRITE "Age: ", ^Patient(patientID, "Age"), !

Ús Avançat

Bloqueig de Globals

Per assegurar la integritat de les dades quan múltiples processos accedeixen als globals, MUMPS proporciona mecanismes de bloqueig.

; Bloquejant un node global
LOCK ^Patient(12345)

; Realitzar operacions
SET ^Patient(12345, "Name") = "John Doe Updated"

; Desbloquejant el node global
UNLOCK ^Patient(12345)

Arrays Globals i Indirecció

Els globals es poden manipular utilitzant la indirecció, permetent l'accés dinàmic als nodes globals.

; Utilitzant la indirecció per establir un valor global
SET globalName = "^Patient"
SET @globalName@(12345, "Address") = "123 Main St"

; Utilitzant la indirecció per recuperar un valor global
WRITE @globalName@(12345, "Address")

Resum

Els globals són una característica poderosa en MUMPS que proporcionen emmagatzematge persistent, jeràrquic i accessible per múltiples processos. Comprendre com utilitzar efectivament els globals, incloent establir, recuperar, recórrer i bloquejar-los, és crucial per desenvolupar aplicacions robustes en MUMPS.

  • Emmagatzematge Persistent: Els globals asseguren que les dades es guarden entre sessions.
  • Estructura Jeràrquica: Permet relacions de dades complexes.
  • Concurrència: Els globals poden ser accessibles per múltiples processos, amb mecanismes de bloqueig per assegurar la integritat de les dades.

En dominar els globals, seràs capaç de gestionar tasques complexes d'emmagatzematge i recuperació de dades de manera eficient en MUMPS.
