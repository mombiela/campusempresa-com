En aquesta secció, et guiarem a través del procés de construir una aplicació simple utilitzant MUMPS (M). Això t'ajudarà a entendre els conceptes bàsics i les aplicacions pràctiques del llenguatge, des de nivells principiants fins a avançats.

Introducció a MUMPS

MUMPS, també conegut com a M, és un llenguatge de programació dissenyat per a la indústria sanitària però és prou versàtil per a altres aplicacions. És conegut per la seva base de dades integrada i capacitats de programació procedimental.

Configurant el Teu Entorn

Abans de començar a construir la nostra aplicació, assegura't de tenir un entorn MUMPS configurat. Pots utilitzar GT.M o qualsevol altra implementació de MUMPS.

Instal·lant GT.M

  1. Descarrega GT.M: Visita el lloc web oficial de GT.M i descarrega la versió adequada per al teu sistema operatiu.
  2. Instal·la GT.M: Segueix les instruccions d'instal·lació proporcionades al lloc web.
  3. Verifica la Instal·lació: Obre el teu terminal i escriu gtm per assegurar-te que GT.M està instal·lat correctament.

Creant una Aplicació Simple: Sistema de Gestió de Pacients

Construirem un simple Sistema de Gestió de Pacients per demostrar les capacitats de MUMPS.

Pas 1: Definir l'Estructura de Dades

Primer, necessitem definir l'estructura de dades per a la nostra aplicació. Emmagatzemarem informació dels pacients com ID, nom, edat i historial mèdic.

; Defineix l'estructura de dades
SET ^PATIENTS(1,"NAME")="John Doe"
SET ^PATIENTS(1,"AGE")=30
SET ^PATIENTS(1,"HISTORY")="No known allergies"

Pas 2: Crear Operacions Bàsiques CRUD

CRUD significa Crear, Llegir, Actualitzar i Esborrar. Aquestes operacions són fonamentals per a qualsevol aplicació.

Operació de Creació

CREATE(ID, NAME, AGE, HISTORY)
    SET ^PATIENTS(ID,"NAME")=NAME
    SET ^PATIENTS(ID,"AGE")=AGE
    SET ^PATIENTS(ID,"HISTORY")=HISTORY
    WRITE "Registre de pacient creat amb èxit!",!
    QUIT

Operació de Lectura

READ(ID)
    WRITE "Nom: ",^PATIENTS(ID,"NAME"),!
    WRITE "Edat: ",^PATIENTS(ID,"AGE"),!
    WRITE "Historial Mèdic: ",^PATIENTS(ID,"HISTORY"),!
    QUIT

Operació d'Actualització

UPDATE(ID, NAME, AGE, HISTORY)
    SET ^PATIENTS(ID,"NAME")=NAME
    SET ^PATIENTS(ID,"AGE")=AGE
    SET ^PATIENTS(ID,"HISTORY")=HISTORY
    WRITE "Registre de pacient actualitzat amb èxit!",!
    QUIT

Operació d'Esborrat

DELETE(ID)
    KILL ^PATIENTS(ID)
    WRITE "Registre de pacient esborrat amb èxit!",!
    QUIT

Pas 3: Implementant la Interacció amb l'Usuari

Ara crearem un menú simple per interactuar amb el nostre Sistema de Gestió de Pacients.

MENU
    WRITE "1. Crear Registre de Pacient",!
    WRITE "2. Llegir Registre de Pacient",!
    WRITE "3. Actualitzar Registre de Pacient",!
    WRITE "4. Esborrar Registre de Pacient",!
    WRITE "5. Sortir",!
    READ "Tria una opció: ", OPTION
    IF OPTION=1 DO
    . READ "Introdueix ID: ", ID
    . READ "Introdueix Nom: ", NAME
    . READ "Introdueix Edat: ", AGE
    . READ "Introdueix Historial Mèdic: ", HISTORY
    . DO CREATE(ID, NAME, AGE, HISTORY)
    ELSE  IF OPTION=2 DO
    . READ "Introdueix ID: ", ID
    . DO READ(ID)
    ELSE  IF OPTION=3 DO
    . READ "Introdueix ID: ", ID
    . READ "Introdueix Nom: ", NAME
    . READ "Introdueix Edat: ", AGE
    . READ "Introdueix Historial Mèdic: ", HISTORY
    . DO UPDATE(ID, NAME, AGE, HISTORY)
    ELSE  IF OPTION=4 DO
    . READ "Introdueix ID: ", ID
    . DO DELETE(ID)
    ELSE  IF OPTION=5 QUIT
    ELSE  WRITE "Opció no vàlida!",!
    GOTO MENU

Característiques Avançades

Gestió d'Errors

En una aplicació del món real, la gestió d'errors és crucial. MUMPS proporciona mecanismes per gestionar errors de manera elegant.

TRY
    NEW $ETRAP SET $ETRAP="DO ERROR^%ZTER"
    ; El teu codi aquí
    QUIT

ERROR
    WRITE "S'ha produït un error: ",$ZERROR,!
    QUIT

Validació de Dades

Assegura't que les dades introduïdes per l'usuari siguin vàlides.

VALIDATE(NAME, AGE, HISTORY)
    IF NAME="" WRITE "El nom no pot estar buit!",! QUIT 0
    IF AGE'?1N.N WRITE "L'edat ha de ser un número!",! QUIT 0
    IF HISTORY="" WRITE "L'historial mèdic no pot estar buit!",! QUIT 0
    QUIT 1

Implementant la Validació en les Operacions CRUD

CREATE(ID, NAME, AGE, HISTORY)
    IF '$$VALIDATE(NAME, AGE, HISTORY) QUIT
    SET ^PATIENTS(ID,"NAME")=NAME
    SET ^PATIENTS(ID,"AGE")=AGE
    SET ^PATIENTS(ID,"HISTORY")=HISTORY
    WRITE "Registre de pacient creat amb èxit!",!
    QUIT

Conclusió

En aquest tema, hem cobert els conceptes bàsics de construir una aplicació simple utilitzant MUMPS. Hem començat amb la configuració de l'entorn, definint l'estructura de dades, implementant operacions CRUD i creant un menú d'interacció amb l'usuari. També hem tocat característiques avançades com la gestió d'errors i la validació de dades. Amb aquests fonaments, ara pots explorar aplicacions i característiques més complexes en MUMPS.
