Introducció

Les transaccions i el bloqueig són conceptes fonamentals en la gestió de bases de dades, assegurant la integritat i la consistència de les dades. En aquesta secció, explorarem com aquests conceptes s'implementen en MUMPS (M), un llenguatge conegut per les seves potents capacitats de bases de dades.

Transaccions

Una transacció en MUMPS és una seqüència d'operacions realitzades com una única unitat lògica de treball. Les transaccions asseguren que o bé totes les operacions es completen amb èxit, o bé cap d'elles, mantenint la integritat de les dades.

Conceptes Clau

  • Atomicitat: Assegura que totes les operacions dins d'una transacció es completen; si no, la transacció s'aborta.
  • Consistència: Assegura que la base de dades es manté en un estat consistent abans i després de la transacció.
  • Aïllament: Assegura que les transaccions estan aïllades entre si fins que es completen.
  • Durabilitat: Assegura que una vegada una transacció es compromet, es manté així, fins i tot en cas de fallada del sistema.

Comandes bàsiques de transacció

  • TSTART: Comença una transacció.
  • TCOMMIT: Compromet la transacció.
  • TROLLBACK: Desfà la transacció.

Exemple

; Comença una transacció
TSTART

; Realitza algunes operacions
SET ^Patient(1, "Name") = "John Doe"
SET ^Patient(1, "Age") = 30

; Compromet la transacció
TCOMMIT

En aquest exemple, la transacció comença amb TSTART, actualitza el nom i l'edat del pacient, i després compromet la transacció amb TCOMMIT.

Gestió d'errors

TSTART
SET $ETRAP="DO ERROR^%ZTER"
SET ^Patient(1, "Name") = "John Doe"
SET ^Patient(1, "Age") = 30
TCOMMIT
QUIT

ERROR
TROLLBACK
WRITE "La transacció ha fallat i s'ha desfet",!
QUIT

Aquí, si es produeix un error, la transacció es desfà utilitzant TROLLBACK, assegurant que no es facin actualitzacions parcials.

Bloqueig

El bloqueig és un mecanisme per controlar l'accés concurrent a les dades, prevenint conflictes i assegurant la integritat de les dades.

Conceptes Clau

  • Bloqueig Exclusiu: Impedeix que altres processos accedeixin a les dades bloquejades.
  • Bloqueig Compartit: Permet que múltiples processos llegeixin però no modifiquin les dades bloquejades.

Comandes bàsiques de bloqueig

  • LOCK: Adquireix un bloqueig.
  • UNLOCK: Allibera un bloqueig.

Exemple

; Adquireix un bloqueig en un node global
LOCK ^Patient(1)

; Realitza operacions
SET ^Patient(1, "Name") = "Jane Doe"

; Allibera el bloqueig
UNLOCK ^Patient(1)

En aquest exemple, la comanda LOCK adquireix un bloqueig exclusiu en el node ^Patient(1), assegurant que cap altre procés el pugui modificar fins que el bloqueig es alliberi amb UNLOCK.

Temps d'espera del bloqueig

; Intenta adquirir un bloqueig amb un temps d'espera
LOCK +^Patient(1):5
IF '$TEST WRITE "No s'ha pogut adquirir el bloqueig",! QUIT

; Realitza operacions
SET ^Patient(1, "Name") = "Jane Doe"

; Allibera el bloqueig
UNLOCK ^Patient(1)

Aquí, la comanda LOCK intenta adquirir un bloqueig amb un temps d'espera de 5 segons. Si el bloqueig no es pot adquirir dins d'aquest temps, l'operació falla.

Conclusió

Entendre les transaccions i el bloqueig en MUMPS és crucial per mantenir la integritat i la consistència de les dades en les teves aplicacions. Les transaccions asseguren que les operacions es completen totalment o no es fan, mentre que el bloqueig controla l'accés concurrent a les dades, prevenint conflictes. En dominar aquests conceptes, pots construir aplicacions MUMPS robustes i fiables.
