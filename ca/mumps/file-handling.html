Introducció a la Gestió de Fitxers

La gestió de fitxers en MUMPS (també conegut com M) implica llegir i escriure fitxers. Això és essencial per a l'emmagatzematge, recuperació i manipulació de dades. En aquesta secció, cobrirem els conceptes bàsics de la gestió de fitxers, incloent obrir, llegir, escriure i tancar fitxers.

Operacions Bàsiques amb Fitxers

Obrir un Fitxer

Per treballar amb un fitxer, primer cal obrir-lo. En MUMPS, s'utilitza la comanda OPEN per obrir un fitxer.

OPEN "nom_fitxer":("R"):1
IF '$TEST WRITE "Error obrint el fitxer",! QUIT
  • nom_fitxer: El nom del fitxer que vols obrir.
  • ("R"): El mode en què s'obre el fitxer. "R" significa mode de lectura.
  • 1: El número de dispositiu. És opcional però recomanat per claredat.

Tancar un Fitxer

Després d'acabar amb un fitxer, s'ha de tancar utilitzant la comanda CLOSE.

CLOSE "nom_fitxer"

Llegir d'un Fitxer

Per llegir d'un fitxer, s'utilitza la comanda READ. Primer s'ha d'obrir el fitxer en mode de lectura.

OPEN "nom_fitxer":("R"):1
IF '$TEST WRITE "Error obrint el fitxer",! QUIT

USE "nom_fitxer"
READ linia
WRITE "Línia llegida: ", linia,!

CLOSE "nom_fitxer"
  • USE "nom_fitxer": Dirigeix les operacions d'E/S posteriors al fitxer especificat.
  • READ linia: Llegeix una línia del fitxer a la variable linia.

Escriure a un Fitxer

Per escriure a un fitxer, s'utilitza la comanda WRITE. Primer s'ha d'obrir el fitxer en mode d'escriptura.

OPEN "nom_fitxer":("W"):1
IF '$TEST WRITE "Error obrint el fitxer",! QUIT

USE "nom_fitxer"
WRITE "Aquesta és una línia de text",!

CLOSE "nom_fitxer"
  • ("W"): El mode en què s'obre el fitxer. "W" significa mode d'escriptura.
  • WRITE "Aquesta és una línia de text": Escriu el text especificat al fitxer.

Operacions Avançades amb Fitxers

Afegir a un Fitxer

Per afegir dades a un fitxer existent, s'obre el fitxer en mode d'afegir utilitzant ("A").

OPEN "nom_fitxer":("A"):1
IF '$TEST WRITE "Error obrint el fitxer",! QUIT

USE "nom_fitxer"
WRITE "Afegint aquesta línia",!

CLOSE "nom_fitxer"
  • ("A"): El mode en què s'obre el fitxer. "A" significa mode d'afegir.

Gestió d'Errors

La gestió adequada d'errors és crucial per a operacions robustes amb fitxers. Utilitza la variable $TEST per comprovar si les operacions amb fitxers tenen èxit.

OPEN "nom_fitxer":("R"):1
IF '$TEST DO  QUIT
. WRITE "Error obrint el fitxer",!
. ; Codi addicional de gestió d'errors

USE "nom_fitxer"
READ linia
IF '$TEST DO  QUIT
. WRITE "Error llegint el fitxer",!
. ; Codi addicional de gestió d'errors

CLOSE "nom_fitxer"

Treballar amb Múltiples Fitxers

Es pot treballar amb múltiples fitxers simultàniament utilitzant diferents números de dispositiu.

OPEN "fitxer1":("R"):1
IF '$TEST WRITE "Error obrint fitxer1",! QUIT

OPEN "fitxer2":("W"):2
IF '$TEST WRITE "Error obrint fitxer2",! QUIT

USE "fitxer1"
READ linia
USE "fitxer2"
WRITE linia,!

CLOSE "fitxer1"
CLOSE "fitxer2"

Resum

En aquest tema, hem cobert els conceptes essencials de la gestió de fitxers en MUMPS, incloent:

  • Obrir i tancar fitxers.
  • Llegir i escriure fitxers.
  • Afegir dades a fitxers.
  • Gestionar errors durant les operacions amb fitxers.
  • Treballar amb múltiples fitxers simultàniament.

Entendre aquests conceptes és crucial per a una gestió efectiva de dades en MUMPS. Practica aquestes operacions amb diferents fitxers per esdevenir proficient en la gestió de fitxers.
