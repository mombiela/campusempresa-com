Introducció a la Base de Dades MUMPS

MUMPS (Massachusetts General Hospital Utility Multi-Programming System), també conegut com M, és un llenguatge potent dissenyat per a operacions de base de dades. És particularment fort en la gestió de dades jeràrquiques i multidimensionals.

Conceptes Clau

  • Variables Globals: Emmagatzematge persistent en MUMPS, utilitzat per emmagatzemar dades entre sessions.
  • Variables Locals: Emmagatzematge temporal, utilitzat dins de la sessió o rutina actual.
  • Nodes: Elements de les variables globals, formant una estructura jeràrquica.
  • Subscripts: Índexs utilitzats per accedir als nodes en les variables globals.

Operacions Bàsiques de Base de Dades

Creació i Emmagatzematge de Dades

Per emmagatzemar dades en MUMPS, utilitzes variables globals. Aquí tens un exemple senzill:

SET ^Patient(1, "Name") = "John Doe"
SET ^Patient(1, "Age") = 30
SET ^Patient(1, "Condition") = "Healthy"

Explicació:

  • ^Patient és la variable global.
  • 1 és el primer subscript, representant l'ID del pacient.
  • "Name", "Age", i "Condition" són subscripts addicionals que representen diferents atributs del pacient.

Recuperació de Dades

Per recuperar dades, utilitzes la comanda WRITE:

WRITE ^Patient(1, "Name")

Això mostrarà:

John Doe

Eliminació de Dades

Per eliminar un node o una branca sencera, utilitza la comanda KILL:

KILL ^Patient(1, "Condition")

Això eliminarà el node "Condition" per al pacient 1.

Operacions Intermèdies de Base de Dades

Iteració sobre Dades

Pots iterar sobre nodes utilitzant el bucle FOR i la comanda ORDER:

SET patientID = ""
FOR  SET patientID = $ORDER(^Patient(patientID)) QUIT:patientID=""  DO
. WRITE "Patient ID: ", patientID, " Name: ", ^Patient(patientID, "Name"), !

Explicació:

  • $ORDER(^Patient(patientID)) recupera el següent subscript en la global ^Patient.
  • QUIT:patientID="" surt del bucle quan no hi ha més pacients.

Recuperació Condicional

Pots recuperar dades condicionalment utilitzant declaracions IF:

SET patientID = 1
IF ^Patient(patientID, "Age") > 25 WRITE "Patient is older than 25"

Operacions Avançades de Base de Dades

Estructures de Dades Multinivell

MUMPS suporta estructures de dades complexes i multinivell. Aquí tens un exemple d'un registre de pacient més complex:

SET ^Patient(1, "Name") = "John Doe"
SET ^Patient(1, "Age") = 30
SET ^Patient(1, "Condition", "Primary") = "Healthy"
SET ^Patient(1, "Condition", "Secondary") = "None"

Transaccions

MUMPS suporta transaccions per assegurar la integritat de les dades. Utilitza TSTART i TCOMMIT per gestionar transaccions:

TSTART
SET ^Patient(2, "Name") = "Jane Smith"
SET ^Patient(2, "Age") = 28
TCOMMIT

Si es produeix un error, pots utilitzar TROLLBACK per desfer els canvis:

TSTART
SET ^Patient(3, "Name") = "Alice Brown"
IF $GET(^Patient(3, "Name"))="" DO
. TROLLBACK
. WRITE "Transaction failed", !
ELSE  DO
. TCOMMIT
. WRITE "Transaction successful", !

Resum

En aquest tema, hem cobert els aspectes essencials de les operacions de base de dades en MUMPS, des de l'emmagatzematge i recuperació bàsics de dades fins a conceptes més avançats com les estructures de dades multinivell i les transaccions. Entendre aquestes operacions és crucial per gestionar i manipular dades de manera efectiva en MUMPS.

Punts Clau

  • MUMPS utilitza variables globals per a l'emmagatzematge persistent de dades.
  • Les operacions bàsiques inclouen establir, recuperar i eliminar dades.
  • Les operacions intermèdies impliquen iterar sobre dades i recuperació condicional.
  • Les operacions avançades inclouen la gestió d'estructures de dades multinivell i la gestió de transaccions per a la integritat de les dades.

En dominar aquestes operacions de base de dades, estaràs ben preparat per gestionar tasques complexes de gestió de dades en MUMPS.
