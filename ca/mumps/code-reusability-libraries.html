Introducció

La reutilització de codi és un concepte fonamental en la programació que permet als desenvolupadors escriure codi una vegada i utilitzar-lo múltiples vegades. Això no només estalvia temps, sinó que també assegura consistència i redueix errors. En MUMPS (també conegut com a M), la reutilització de codi es pot aconseguir mitjançant l'ús de biblioteques i tècniques de programació modular.

Conceptes Clau

  • Programació Modular: Dividir un programa en peces més petites, manejables i reutilitzables.
  • Biblioteques: Col·leccions de codi preescrit que poden ser utilitzades per múltiples programes.
  • Rutines: En MUMPS, les rutines són la manera principal d'emmagatzemar i reutilitzar codi.

Creant Codi Reutilitzable

Escriure Codi Modular

El codi modular està dissenyat per ser autònom i independent. Això fa que sigui més fàcil de reutilitzar i mantenir.

Exemple

; Aquesta és una rutina simple per sumar dos números
ADDNUMS(A, B)
    QUIT A+B

Utilitzant Rutines

Les rutines en MUMPS són fitxers que contenen un o més procediments o funcions. Aquestes rutines poden ser cridades des d'altres rutines o programes.

Exemple

; Rutina per calcular la suma de dos números
SUM
    READ "Introdueix el primer número: ", A
    READ "Introdueix el segon número: ", B
    SET RESULT=$$ADDNUMS^MYLIB(A, B)
    WRITE "La suma és: ", RESULT, !
    QUIT

En aquest exemple, ADDNUMS és una funció emmagatzemada en una rutina de biblioteca MYLIB.

Creant Biblioteques

Les biblioteques en MUMPS són simplement rutines que estan dissenyades per ser reutilitzades per altres rutines. Aquestes poden ser emmagatzemades en fitxers separats i incloses segons sigui necessari.

Exemple

; MYLIB.m - Una biblioteca de funcions útils

; Funció per sumar dos números
ADDNUMS(A, B)
    QUIT A+B

; Funció per restar dos números
SUBNUMS(A, B)
    QUIT A-B

Cridant Funcions de Biblioteca

Per utilitzar una funció d'una biblioteca, utilitzes la sintaxi $$ seguida del nom de la funció i el nom de la rutina.

Exemple

; Rutina per calcular la diferència de dos números
DIFF
    READ "Introdueix el primer número: ", A
    READ "Introdueix el segon número: ", B
    SET RESULT=$$SUBNUMS^MYLIB(A, B)
    WRITE "La diferència és: ", RESULT, !
    QUIT

Tècniques Avançades

Espais de Noms

Utilitzar espais de noms pot ajudar a organitzar les teves rutines i evitar conflictes de noms.

Exemple

; MYLIB.ADD - Una biblioteca de funcions d'addició

; Funció per sumar dos números
ADDNUMS(A, B)
    QUIT A+B

Passatge de Paràmetres

Passar paràmetres a rutines i funcions permet un codi més flexible i reutilitzable.

Exemple

; Rutina per calcular el producte de dos números
PRODUCT
    READ "Introdueix el primer número: ", A
    READ "Introdueix el segon número: ", B
    SET RESULT=$$MULTIPLY^MYLIB(A, B)
    WRITE "El producte és: ", RESULT, !
    QUIT

; MYLIB.m - Una biblioteca de funcions útils

; Funció per multiplicar dos números
MULTIPLY(A, B)
    QUIT A*B

Gestió d'Errors

Implementar la gestió d'errors en el teu codi reutilitzable assegura que pugui manejar situacions inesperades de manera elegant.

Exemple

; MYLIB.m - Una biblioteca de funcions útils amb gestió d'errors

; Funció per dividir dos números amb gestió d'errors
DIVIDE(A, B)
    IF B=0 QUIT "Error: Divisió per zero"
    QUIT A/B

Conclusió

La reutilització de codi en MUMPS pot millorar significativament la productivitat i la mantenibilitat. Escriure codi modular, crear biblioteques i utilitzar tècniques avançades com espais de noms i gestió d'errors, et permetrà crear codi robust i reutilitzable que es pot aprofitar en múltiples projectes. Recorda sempre provar el teu codi reutilitzable a fons per assegurar-te que funciona correctament en diferents contextos.
