La gestió d'errors és un aspecte crucial de la programació en qualsevol llenguatge, incloent-hi MUMPS. Una gestió d'errors adequada assegura que els teus programes puguin manejar situacions inesperades de manera elegant i continuar operant o fallar de manera segura. Aquesta secció cobrirà la gestió d'errors des de nivells bàsics fins a avançats.

Introducció a la Gestió d'Errors

La gestió d'errors en MUMPS implica detectar i gestionar errors que es produeixen durant l'execució d'un programa. MUMPS proporciona mecanismes per atrapar errors i prendre les accions apropiades.

Conceptes Clau

  • Captura d'Errors: El procés de capturar errors quan es produeixen.
  • $ZTRAP: Una variable especial utilitzada per especificar la rutina de gestió d'errors.
  • $ECODE: Una variable especial que conté el codi d'error quan es produeix un error.
  • $ETRAP: Un mecanisme de gestió d'errors més nou i flexible en comparació amb $ZTRAP.

Gestió Bàsica d'Errors amb $ZTRAP

Configuració de $ZTRAP

La variable $ZTRAP s'utilitza per especificar una etiqueta a la qual s'ha de transferir el control si es produeix un error.

SET $ZTRAP="ERROR^MYPROG"

En aquest exemple, si es produeix un error, el control es transfereix a l'etiqueta ERROR en la rutina MYPROG.

Exemple

MYPROG ;
    NEW X,Y
    SET $ZTRAP="ERROR^MYPROG"
    SET X=10,Y=0
    WRITE "Resultat: ",X/Y,!
    QUIT

ERROR ;
    WRITE "S'ha produït un error: ",$ZERROR,!
    QUIT

En aquest exemple:

  • El programa intenta dividir X per Y, que és zero, causant un error de divisió per zero.
  • El control es transfereix a l'etiqueta ERROR, on es mostra el missatge d'error.

Gestió Intermèdia d'Errors amb $ECODE

Ús de $ECODE

La variable $ECODE conté el codi d'error quan es produeix un error. Pots utilitzar-la per determinar el tipus d'error i prendre l'acció apropiada.

SET $ECODE=",<DIVIDE>,"

Exemple

MYPROG ;
    NEW X,Y
    SET $ZTRAP="ERROR^MYPROG"
    SET X=10,Y=0
    WRITE "Resultat: ",X/Y,!
    QUIT

ERROR ;
    IF $ECODE[",<DIVIDE>," DO
    . WRITE "Error de divisió per zero",!
    ELSE  DO
    . WRITE "S'ha produït un error inesperat: ",$ZERROR,!
    SET $ECODE=""
    QUIT

En aquest exemple:

  • El programa comprova si el codi d'error conté <DIVIDE>, indicant un error de divisió per zero.
  • Es mostren missatges apropiats basats en el tipus d'error.

Gestió Avançada d'Errors amb $ETRAP

Configuració de $ETRAP

La variable $ETRAP permet una gestió d'errors més flexible. Pot contenir codi executable de MUMPS que s'executa quan es produeix un error.

SET $ETRAP="DO ERROR^MYPROG"

Exemple

MYPROG ;
    NEW X,Y
    SET $ETRAP="DO ERROR^MYPROG"
    SET X=10,Y=0
    WRITE "Resultat: ",X/Y,!
    QUIT

ERROR ;
    WRITE "S'ha produït un error: ",$ZERROR,!
    SET $ECODE=""
    QUIT

En aquest exemple:

  • $ETRAP està configurat per executar la rutina ERROR quan es produeix un error.
  • La rutina de gestió d'errors s'executa i es mostra el missatge d'error.

Resum

La gestió d'errors en MUMPS és essencial per crear programes robustos i fiables. Aquesta secció ha cobert:

  • Gestió bàsica d'errors utilitzant $ZTRAP.
  • Gestió intermèdia d'errors utilitzant $ECODE.
  • Gestió avançada d'errors utilitzant $ETRAP.

Entenent i implementant aquests mecanismes de gestió d'errors, pots assegurar-te que els teus programes MUMPS puguin manejar situacions inesperades de manera elegant i continuar operant o fallar de manera segura.
