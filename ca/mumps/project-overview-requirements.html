Introducció

En aquesta secció, proporcionarem una visió general del projecte en el qual treballaràs al llarg d'aquest curs. Aquest projecte t'ajudarà a aplicar els conceptes i tècniques que aprenguis en la programació MUMPS (M), des de nivells principiants fins a avançats. Al final d'aquest curs, tindràs una comprensió completa de MUMPS i seràs capaç de desenvolupar aplicacions robustes utilitzant aquest llenguatge.

Descripció del projecte

El projecte en el qual treballarem és un Sistema de Gestió de Pacients (SGP) senzill però complet. Aquest sistema permetrà als usuaris:

  • Afegir nous pacients.
  • Actualitzar la informació dels pacients.
  • Recuperar els detalls dels pacients.
  • Eliminar registres de pacients.

El projecte es dividirà en diversos mòduls, cadascun centrat en diferents aspectes de la programació MUMPS. A mesura que avancis en el curs, aniràs construint i millorant el SGP.

Requisits del projecte

Per completar amb èxit aquest projecte, hauràs de complir els següents requisits:

Requisits tècnics

  • Entorn MUMPS: Assegura't de tenir un entorn MUMPS funcional. Pots utilitzar GT.M o qualsevol altra implementació de MUMPS.
  • Editor de text: Utilitza un editor de text que admeti ressaltat de sintaxi MUMPS. Exemples inclouen VS Code amb extensions adequades o qualsevol IDE específic per a MUMPS.
  • Coneixements bàsics de MUMPS: Familiaritat amb la sintaxi i les comandes bàsiques de MUMPS. Cobrirem aquests aspectes en les seccions per a principiants del curs.

Requisits funcionals

  • Gestió de dades de pacients: El sistema ha de ser capaç de gestionar operacions CRUD (Crear, Llegir, Actualitzar, Eliminar) per als registres de pacients.
  • Validació de dades: Implementar una validació bàsica per assegurar la integritat de les dades (per exemple, que no hi hagi camps buits, formats de data vàlids).
  • Interfície d'usuari: Una interfície senzilla basada en text per interactuar amb el sistema.

Conceptes clau

Aquí tens alguns conceptes clau que hauràs d'entendre i implementar en el teu projecte:

  • Variables globals: MUMPS utilitza variables globals per emmagatzemar dades de manera persistent. Aprendràs a utilitzar-les per emmagatzemar registres de pacients.
  • Subrutines i funcions: Modularitza el teu codi utilitzant subrutines i funcions per a una millor mantenibilitat.
  • Gestió d'errors: Implementa la gestió d'errors per gestionar situacions inesperades de manera elegant.
  • I/O de fitxers: Aprèn a llegir i escriure fitxers per a la persistència de dades.

Exemple de codi

Aquí tens un exemple senzill per il·lustrar com podries començar afegint un nou registre de pacient:

AddPatient
    WRITE "Enter Patient ID: "
    READ PatientID
    WRITE "Enter Patient Name: "
    READ PatientName
    WRITE "Enter Patient DOB (YYYYMMDD): "
    READ PatientDOB
    
    ; Emmagatzema les dades en variables globals
    SET ^Patient(PatientID, "Name") = PatientName
    SET ^Patient(PatientID, "DOB") = PatientDOB
    
    WRITE "Patient record added successfully!", !
    QUIT

Explicació

  • WRITE i READ: Aquestes comandes s'utilitzen per a operacions d'entrada i sortida.
  • Variables globals: ^Patient(PatientID, "Name") i ^Patient(PatientID, "DOB") s'utilitzen per emmagatzemar informació del pacient de manera persistent.

Resum

En aquesta secció, hem esbossat el projecte en el qual treballaràs al llarg del curs. El Sistema de Gestió de Pacients t'ajudarà a aplicar els conceptes de programació MUMPS en un context pràctic. També hem discutit els requisits tècnics i funcionals, els conceptes clau, i hem proporcionat un exemple de codi senzill per començar. A mesura que avancis en el curs, construiràs sobre aquesta base per crear una aplicació completament funcional.
