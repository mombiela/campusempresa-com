Les proves i la validació són aspectes crucials del desenvolupament de programari, assegurant que el teu codi MUMPS (M) sigui fiable, eficient i lliure d'errors. Aquesta secció et guiarà a través dels conceptes bàsics de les proves i la validació, progressant cap a tècniques més avançades.

Introducció a les Proves en MUMPS

Les proves són el procés d'executar un programa amb la intenció de trobar errors. Ajuda a assegurar que el codi es comporti com s'espera en diverses condicions.

Conceptes Clau

  • Proves Unitàries: Proves d'unitats o components individuals d'un programa.
  • Proves d'Integració: Proves de la interacció entre unitats o components integrats.
  • Proves de Sistema: Proves del sistema de programari complet i integrat.
  • Proves de Regressió: Reexecutar proves per assegurar que els canvis no han introduït nous errors.
  • Validació: Assegurar que el programari compleix els requisits i les expectatives dels usuaris finals.

Proves Unitàries en MUMPS

Les proves unitàries se centren en verificar les parts més petites de l'aplicació, com ara funcions o procediments, per assegurar que funcionen correctament.

Exemple: Prova Unitària Simple

Creem una funció simple i una prova unitària corresponent.

; Funció per sumar dos números
ADD(a, b)
    Q a + b

; Prova Unitària per la funció ADD
TESTADD
    N result
    S result=$$ADD(2,3)
    I result=5 W "Prova Superada",! Q
    W "Prova Fallida",!
    Q
  • Explicació:
    • ADD(a, b): Una funció que pren dos arguments i retorna la seva suma.
    • TESTADD: Una prova unitària que comprova si ADD(2, 3) retorna 5.

Executar la Prova

Per executar la prova, simplement crida el procediment TESTADD des del prompt de MUMPS.

D TESTADD

Proves d'Integració en MUMPS

Les proves d'integració asseguren que diferents mòduls o serveis utilitzats per la teva aplicació funcionin bé junts.

Exemple: Prova d'Integració

Suposem que tenim dues funcions, ADD i MULTIPLY, i volem provar la seva integració.

; Funció per multiplicar dos números
MULTIPLY(a, b)
    Q a * b

; Prova d'Integració per les funcions ADD i MULTIPLY
TESTINTEGRATION
    N sum, product
    S sum=$$ADD(2,3)
    S product=$$MULTIPLY(sum,4)
    I product=20 W "Prova d'Integració Superada",! Q
    W "Prova d'Integració Fallida",!
    Q
  • Explicació:
    • MULTIPLY(a, b): Una funció que pren dos arguments i retorna el seu producte.
    • TESTINTEGRATION: Una prova d'integració que comprova si MULTIPLY($$ADD(2, 3), 4) retorna 20.

Executar la Prova

Per executar la prova d'integració, crida el procediment TESTINTEGRATION des del prompt de MUMPS.

D TESTINTEGRATION

Proves de Sistema en MUMPS

Les proves de sistema impliquen provar tot el sistema en conjunt per assegurar que compleix els requisits especificats.

Exemple: Prova de Sistema

Considera un sistema simple que calcula el preu total dels articles en un carret de la compra.

; Funció per calcular el preu total
TOTALPRICE(cart)
    N total, i
    S total=0
    F i=1:1:$L(cart,",") S total=total+$P(cart,",",i)
    Q total

; Prova de Sistema per la funció TOTALPRICE
TESTSYSTEM
    N cart, total
    S cart="10,20,30"
    S total=$$TOTALPRICE(cart)
    I total=60 W "Prova de Sistema Superada",! Q
    W "Prova de Sistema Fallida",!
    Q
  • Explicació:
    • TOTALPRICE(cart): Una funció que pren una llista de preus separats per comes i retorna el seu total.
    • TESTSYSTEM: Una prova de sistema que comprova si TOTALPRICE("10,20,30") retorna 60.

Executar la Prova

Per executar la prova de sistema, crida el procediment TESTSYSTEM des del prompt de MUMPS.

D TESTSYSTEM

Proves de Regressió en MUMPS

Les proves de regressió asseguren que els nous canvis no afectin negativament la funcionalitat existent.

Exemple: Prova de Regressió

Després de modificar la funció ADD per manejar números negatius, necessitem assegurar-nos que encara funciona per números positius.

; Funció ADD modificada per manejar números negatius
ADD(a, b)
    Q a + b

; Prova de Regressió per la funció ADD
TESTREGRESSION
    N result
    S result=$$ADD(2,3)
    I result=5 W "Prova de Regressió Superada",! Q
    W "Prova de Regressió Fallida",!
    Q
  • Explicació:
    • TESTREGRESSION: Una prova de regressió que comprova si ADD(2, 3) encara retorna 5 després de les modificacions.

Executar la Prova

Per executar la prova de regressió, crida el procediment TESTREGRESSION des del prompt de MUMPS.

D TESTREGRESSION

Conclusió

Les proves i la validació són pràctiques essencials en la programació MUMPS per assegurar la qualitat i la fiabilitat del codi. Implementant proves unitàries, proves d'integració, proves de sistema i proves de regressió, pots verificar sistemàticament que el teu codi funciona com s'espera i compleix els requisits dels usuaris. Les proves regulars ajuden a identificar problemes aviat, fent més fàcil mantenir i millorar les teves aplicacions MUMPS.
