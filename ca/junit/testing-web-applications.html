Introducció a les proves d'aplicacions web

Provar aplicacions web és crucial per assegurar-se que funcionen correctament i proporcionen una bona experiència d'usuari. JUnit és un marc de proves popular per a Java que permet als desenvolupadors escriure i executar proves repetibles. Aquesta secció cobrirà els conceptes bàsics d'ús de JUnit per provar aplicacions web, des de la configuració del vostre entorn fins a escriure i executar proves.

Configuració del vostre entorn

Abans de començar a provar aplicacions web amb JUnit, heu de configurar el vostre entorn de desenvolupament.

Prerequisits

  • Kit de desenvolupament de Java (JDK) instal·lat
  • Entorn de desenvolupament integrat (IDE) com IntelliJ IDEA o Eclipse
  • Maven o Gradle per a la gestió de dependències

Afegir JUnit al vostre projecte

Per utilitzar JUnit en el vostre projecte, heu d'afegir-lo com a dependència al vostre fitxer de configuració de compilació.

Maven

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-engine</artifactId>
    <version>5.8.1</version>
    <scope>test</scope>
</dependency>

Gradle

testImplementation 'org.junit.jupiter:junit-jupiter-engine:5.8.1'

Escriure la vostra primera prova

Comencem escrivint una prova senzilla per entendre els conceptes bàsics de JUnit.

Exemple: Provar un servei web senzill

Suposem que teniu un servei web que retorna un missatge de salutació. Aquí teniu com podeu escriure una prova per a això.

Codi del servei web

public class GreetingService {
    public String getGreeting() {
        return "Hello, World!";
    }
}

Codi de la prova

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class GreetingServiceTest {

    @Test
    public void testGetGreeting() {
        GreetingService service = new GreetingService();
        String greeting = service.getGreeting();
        assertEquals("Hello, World!", greeting);
    }
}

Explicació

  • @Test: Aquesta anotació marca el mètode com a mètode de prova.
  • assertEquals: Aquesta asserció comprova si el valor esperat coincideix amb el valor real.

Tècniques avançades de prova

A mesura que us familiaritzeu amb JUnit, podeu començar a explorar tècniques de prova més avançades.

Simulació de dependències

En aplicacions del món real, les vostres classes sovint depenen d'altres classes. Podeu utilitzar marcs de simulació com Mockito per crear objectes simulats per a aquestes dependències.

Exemple: Simulació d'una dependència

Suposem que el vostre GreetingService depèn d'un TimeService per proporcionar una salutació basada en el temps.

Codi de TimeService
public class TimeService {
    public String getCurrentTime() {
        return "Morning";
    }
}
Codi de GreetingService
public class GreetingService {
    private TimeService timeService;

    public GreetingService(TimeService timeService) {
        this.timeService = timeService;
    }

    public String getGreeting() {
        String timeOfDay = timeService.getCurrentTime();
        return "Good " + timeOfDay + "!";
    }
}
Codi de la prova amb Mockito
import org.junit.jupiter.api.Test;
import org.mockito.Mockito;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class GreetingServiceTest {

    @Test
    public void testGetGreeting() {
        TimeService mockTimeService = Mockito.mock(TimeService.class);
        Mockito.when(mockTimeService.getCurrentTime()).thenReturn("Evening");

        GreetingService service = new GreetingService(mockTimeService);
        String greeting = service.getGreeting();
        assertEquals("Good Evening!", greeting);
    }
}

Proves parametritzades

JUnit us permet executar la mateixa prova amb diferents paràmetres utilitzant proves parametritzades.

Exemple: Prova parametritzada

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;
import static org.junit.jupiter.api.Assertions.assertTrue;

public class ParameterizedTestExample {

    @ParameterizedTest
    @ValueSource(strings = {"racecar", "radar", "level"})
    public void testIsPalindrome(String word) {
        assertTrue(isPalindrome(word));
    }

    public boolean isPalindrome(String word) {
        return new StringBuilder(word).reverse().toString().equals(word);
    }
}

Proves de capes web

Provar capes web implica provar controladors, serveis i repositoris. Spring Boot proporciona un excel·lent suport per provar capes web amb JUnit.

Exemple: Provar un controlador de Spring Boot

Suposem que teniu un controlador de Spring Boot que gestiona sol·licituds HTTP.

Codi del controlador

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class GreetingController {

    @GetMapping("/greeting")
    public String getGreeting() {
        return "Hello, World!";
    }
}

Codi de la prova

import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.web.servlet.WebMvcTest;
import org.springframework.test.web.servlet.MockMvc;
import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.get;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.content;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.status;

@WebMvcTest(GreetingController.class)
public class GreetingControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    public void testGetGreeting() throws Exception {
        mockMvc.perform(get("/greeting"))
                .andExpect(status().isOk())
                .andExpect(content().string("Hello, World!"));
    }
}

Explicació

  • @WebMvcTest: Aquesta anotació s'utilitza per provar controladors de Spring MVC.
  • MockMvc: Aquesta classe proporciona suport per provar controladors de Spring MVC realitzant sol·licituds HTTP i verificant respostes.

Conclusió

Provar aplicacions web amb JUnit implica escriure i executar proves per assegurar-se que la vostra aplicació es comporta com s'espera. Començant amb proves senzilles, podeu avançar gradualment cap a tècniques més avançades com la simulació de dependències, proves parametritzades i proves de capes web. Dominant aquestes tècniques, podeu millorar la fiabilitat i mantenibilitat de les vostres aplicacions web.
