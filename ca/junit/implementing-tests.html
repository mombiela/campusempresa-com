Introducció a JUnit

JUnit és un popular marc de proves per a Java que permet als desenvolupadors escriure i executar proves repetibles. És una eina essencial per assegurar la qualitat i la fiabilitat del codi.

Conceptes Clau

  • Cas de Prova: Una unitat única de prova.
  • Suite de Proves: Una col·lecció de casos de prova.
  • Afirmacions: Declaracions que comproven si una condició és certa.
  • Anotacions: Marcadors especials en el codi que proporcionen metadades al marc JUnit.

Configuració de JUnit

Abans de començar a escriure proves, cal configurar JUnit en el teu projecte.

Afegir JUnit al Teu Projecte

Per a un projecte Maven, afegeix la següent dependència al teu fitxer pom.xml:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-engine</artifactId>
    <version>5.7.0</version>
    <scope>test</scope>
</dependency>

Per a un projecte Gradle, afegeix el següent al teu fitxer build.gradle:

testImplementation 'org.junit.jupiter:junit-jupiter-engine:5.7.0'

Escriure la Teva Primera Prova

Comencem amb un cas de prova senzill.

Exemple: Provar un Mètode de Suma

Suposem que tenim un mètode senzill que suma dos enters:

public class Calculator {
    public int sum(int a, int b) {
        return a + b;
    }
}

Ara, escrivim una prova per a aquest mètode:

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    public void testSum() {
        Calculator calculator = new Calculator();
        int result = calculator.sum(2, 3);
        assertEquals(5, result, "2 + 3 hauria de ser igual a 5");
    }
}

Explicació

  • @Test: Marca el mètode com un cas de prova.
  • assertEquals(expected, actual, message): Comprova si el valor actual coincideix amb el valor esperat.

Tècniques Avançades de Prova

Proves Parametritzades

JUnit et permet executar la mateixa prova amb diferents paràmetres.

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

public class CalculatorTest {

    @ParameterizedTest
    @CsvSource({
        "1, 1, 2",
        "2, 3, 5",
        "3, 7, 10"
    })
    public void testSum(int a, int b, int expected) {
        Calculator calculator = new Calculator();
        assertEquals(expected, calculator.sum(a, b));
    }
}

Explicació

  • @ParameterizedTest: Marca el mètode com una prova parametritzada.
  • @CsvSource: Proporciona els paràmetres per a la prova.

Provar Excepcions

També pots provar si el teu codi llança correctament excepcions.

import static org.junit.jupiter.api.Assertions.assertThrows;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    public void testDivideByZero() {
        Calculator calculator = new Calculator();
        assertThrows(ArithmeticException.class, () -> calculator.divide(1, 0));
    }
}

Explicació

  • assertThrows(expectedType, executable): Comprova si l'executable llança l'excepció esperada.

Millors Pràctiques

  • Escriu Proves Clares i Concises: Cada prova hauria de centrar-se en un únic comportament.
  • Utilitza Noms Significatius: Els noms dels mètodes de prova haurien de descriure clarament què estan provant.
  • Mantingues les Proves Independents: Les proves no haurien de dependre les unes de les altres.
  • Executa les Proves Freqüentment: Integra les proves en el teu flux de treball de desenvolupament.

Resum

JUnit és un marc potent que t'ajuda a escriure i executar proves per al teu codi Java. Entenent els conceptes bàsics dels casos de prova, afirmacions i anotacions, pots començar a escriure proves efectives. A mesura que et sentis més còmode, pots explorar tècniques avançades com les proves parametritzades i la prova d'excepcions. Seguint les millors pràctiques, asseguraràs que les teves proves siguin mantenibles i fiables.
