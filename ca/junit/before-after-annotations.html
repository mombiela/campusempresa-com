Anotacions @Before i @After a JUnit
Anotacions @BeforeClass i @AfterClass ►

JUnit és un marc de proves popular per a Java, i entendre les seves anotacions és crucial per escriure proves efectives. En aquesta secció, cobrirem les anotacions @Before i @After, que s'utilitzen per configurar i desmuntar entorns de prova.

Introducció a @Before i @After

JUnit proporciona diverses anotacions per controlar el flux d'execució de les proves. Dues de les anotacions més utilitzades són @Before i @After.

  • @Before: Aquesta anotació s'utilitza per especificar un mètode que s'ha de executar abans de cada mètode de prova.
  • @After: Aquesta anotació s'utilitza per especificar un mètode que s'ha de executar després de cada mètode de prova.

Aquestes anotacions ajuden a configurar l'entorn de prova i a netejar després de les proves, assegurant que cada prova s'executi de manera aïllada.

Ús de @Before

L'anotació @Before s'utilitza per executar un mètode abans de cada mètode de prova a la classe. Això és útil per inicialitzar objectes, configurar connexions a bases de dades o qualsevol altra tasca de configuració que necessiti ser realitzada abans de cada prova.

Exemple

import org.junit.Before;
import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {
    private Calculator calculator;

    @Before
    public void setUp() {
        calculator = new Calculator();
    }

    @Test
    public void testAddition() {
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }

    @Test
    public void testSubtraction() {
        int result = calculator.subtract(5, 3);
        assertEquals(2, result);
    }
}

En aquest exemple:

  • El mètode setUp està anotat amb @Before, de manera que s'executa abans de cada mètode de prova.
  • L'objecte calculator s'inicialitza abans de cada prova, assegurant una instància nova per a cada mètode de prova.

Ús de @After

L'anotació @After s'utilitza per executar un mètode després de cada mètode de prova a la classe. Això és útil per netejar recursos, tancar connexions a bases de dades o qualsevol altra tasca de desmuntatge que necessiti ser realitzada després de cada prova.

Exemple

import org.junit.After;
import org.junit.Before;
import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {
    private Calculator calculator;

    @Before
    public void setUp() {
        calculator = new Calculator();
    }

    @After
    public void tearDown() {
        calculator = null;
    }

    @Test
    public void testAddition() {
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }

    @Test
    public void testSubtraction() {
        int result = calculator.subtract(5, 3);
        assertEquals(2, result);
    }
}

En aquest exemple:

  • El mètode tearDown està anotat amb @After, de manera que s'executa després de cada mètode de prova.
  • L'objecte calculator es posa a null després de cada prova, assegurant que els recursos es netegin.

Conceptes Clau

  • Aïllament: Cada mètode de prova hauria d'executar-se independentment dels altres. Utilitzar @Before i @After ajuda a aconseguir això configurant i desmuntant l'entorn de prova per a cada prova.
  • Gestió de Recursos: Gestionar correctament recursos com connexions a bases de dades, manejadors de fitxers o connexions de xarxa és crucial. @Before i @After ajuden a assegurar que els recursos es inicialitzin i es netegin correctament.
  • Llegibilitat: Utilitzar aquestes anotacions fa que el codi de prova sigui més llegible i mantenible separant clarament la configuració, l'execució de la prova i la lògica de desmuntatge.

Conclusió

Les anotacions @Before i @After a JUnit són eines essencials per gestionar el cicle de vida de les proves. Ajuden a assegurar que cada prova s'executi en un entorn net, cosa que és crucial per a proves fiables i mantenibles. Utilitzant aquestes anotacions, pots escriure proves que siguin més fàcils d'entendre i mantenir, conduint a una millor qualitat del programari.

En les properes seccions, explorarem funcions i anotacions més avançades de JUnit, continuant a construir sobre aquesta base.
