JUnit 5 és l'última versió del popular framework de proves de Java JUnit. Introdueix diverses noves característiques i millores respecte als seus predecessors, fent-lo més potent i flexible per escriure i executar proves. Aquesta secció proporcionarà una visió general de JUnit 5, la seva arquitectura i l'ús bàsic.

Què és JUnit 5?

JUnit 5 és un framework per escriure i executar proves en Java. S'utilitza per assegurar que el teu codi es comporta com s'espera i ajuda a identificar errors en les primeres fases del procés de desenvolupament. JUnit 5 està compost per tres mòduls principals:

  • JUnit Platform: La base per llançar frameworks de proves a la JVM.
  • JUnit Jupiter: El nou model de programació i model d'extensió per escriure proves i extensions en JUnit 5.
  • JUnit Vintage: Proporciona suport per executar proves de JUnit 3 i JUnit 4 a la plataforma JUnit 5.

Característiques Clau de JUnit 5

  • Arquitectura Modular: JUnit 5 està dividit en diversos mòduls, permetent més flexibilitat i extensibilitat.
  • Noves Anotacions: Introdueix noves anotacions per a un millor control sobre l'execució de les proves.
  • Model d'Extensió: Proporciona un potent model d'extensió per personalitzar l'execució de les proves.
  • Proves Dinàmiques: Suporta proves dinàmiques que es poden generar en temps d'execució.
  • Millores en les Assertions: Mètodes d'assert millorats per a proves més expressives.

Configuració de JUnit 5

Per començar a utilitzar JUnit 5, necessites afegir les dependències necessàries al teu projecte. Aquí tens un exemple de com configurar JUnit 5 amb Maven:

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.junit</groupId>
            <artifactId>junit-bom</artifactId>
            <version>5.8.1</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<dependencies>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter-api</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter-engine</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>

Per Gradle, pots afegir les següents dependències:

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter-api:5.8.1'
    testRuntimeOnly 'org.junit.jupiter:junit-jupiter-engine:5.8.1'
}

Escriure la Teva Primera Prova

Escrivim una prova simple utilitzant JUnit 5. Considera una classe Calculator amb un mètode add:

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

Ara, escrivim una prova per al mètode add:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    void testAdd() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 hauria de ser igual a 5");
    }
}

Explicació

  • @Test: Marca el mètode com a mètode de prova.
  • assertEquals: Assegura que el valor esperat (5) és igual al valor real (result).

Anotacions Comunes en JUnit 5

JUnit 5 introdueix diverses noves anotacions per controlar el cicle de vida i el comportament de les proves:

  • @Test: Marca un mètode com a mètode de prova.
  • @BeforeEach: S'executa abans de cada mètode de prova.
  • @AfterEach: S'executa després de cada mètode de prova.
  • @BeforeAll: S'executa una vegada abans de tots els mètodes de prova de la classe.
  • @AfterAll: S'executa una vegada després de tots els mètodes de prova de la classe.
  • @Disabled: Desactiva un mètode de prova o una classe.

Exemple

import org.junit.jupiter.api.*;

public class LifecycleTest {

    @BeforeAll
    static void initAll() {
        System.out.println("Abans de totes les proves");
    }

    @BeforeEach
    void init() {
        System.out.println("Abans de cada prova");
    }

    @Test
    void testOne() {
        System.out.println("Prova un");
    }

    @Test
    void testTwo() {
        System.out.println("Prova dos");
    }

    @AfterEach
    void tearDown() {
        System.out.println("Després de cada prova");
    }

    @AfterAll
    static void tearDownAll() {
        System.out.println("Després de totes les proves");
    }
}

Conclusió

JUnit 5 és un framework de proves potent i flexible que proporciona un enfocament modern per escriure i executar proves en Java. La seva arquitectura modular, noves anotacions i característiques millorades el converteixen en una eina essencial per als desenvolupadors de Java. En aquesta introducció, hem cobert els conceptes bàsics de JUnit 5, incloent les seves característiques clau, configuració i escriure proves simples.
