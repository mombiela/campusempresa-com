El Desenvolupament Guiat pel Comportament (BDD) és una extensió del Desenvolupament Guiat per Proves (TDD) que posa èmfasi en la col·laboració entre desenvolupadors, QA i participants no tècnics o de negoci en un projecte de programari. El BDD fomenta l'escriptura de proves en un llenguatge natural que els no programadors poden llegir.

Introducció al BDD

  • Definició: El BDD és un procés de desenvolupament de programari que implica exemples escrits en un llenguatge específic del domini per descriure el comportament d'un sistema.
  • Propòsit: Millorar la comunicació entre els interessats del projecte i assegurar que el programari compleixi els requisits de negoci.

Conceptes Clau del BDD

  • Històries d'Usuari: Descripcions de les característiques del programari des de la perspectiva de l'usuari final.
  • Escenaris: Exemples específics de com una característica hauria de comportar-se en diferents situacions.
  • Sintaxi Given-When-Then: Una manera estructurada d'escriure escenaris.

Sintaxi Given-When-Then

  • Given: Descriu el context inicial o l'estat del sistema.
  • When: Descriu l'acció o esdeveniment.
  • Then: Descriu el resultat o l'efecte esperat.

BDD amb JUnit

JUnit és un marc de proves popular per a Java, i es pot utilitzar per implementar BDD escrivint proves que segueixin l'estructura Given-When-Then.

Configuració de JUnit per a BDD

  1. Afegir la Dependència de JUnit: Assegura't que JUnit està inclòs en el teu projecte. 
    <!-- Per a Maven -->
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.7.0</version>
    <scope>test</scope>
</dependency>
  2. Crear Classes de Prova: Organitza les teves classes de prova per reflectir les característiques i escenaris.

Escriure Proves BDD amb JUnit

Considerem un exemple d'una aplicació bancària simple on volem provar el comportament d'una característica de dipòsit.

Exemple d'Escenari

Història d'Usuari: Com a client bancari, vull dipositar diners al meu compte per tal d'augmentar el meu saldo.

Escenari: Dipòsit exitós

  • Given: El saldo del compte és de 100 $.
  • When: El client diposita 50 $.
  • Then: El saldo del compte hauria de ser de 150 $.

Exemple de Codi

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;

class BankAccountTest {
    private BankAccount account;

    @BeforeEach
    void setUp() {
        account = new BankAccount(100); // Given: El saldo del compte és de 100 $
    }

    @Test
    void testDeposit() {
        account.deposit(50); // When: El client diposita 50 $
        assertEquals(150, account.getBalance()); // Then: El saldo del compte hauria de ser de 150 $
    }
}

class BankAccount {
    private int balance;

    public BankAccount(int initialBalance) {
        this.balance = initialBalance;
    }

    public void deposit(int amount) {
        this.balance += amount;
    }

    public int getBalance() {
        return this.balance;
    }
}

Pràctiques Avançades de BDD amb JUnit

  • Proves Parametritzades: Utilitza les proves parametritzades de JUnit per executar la mateixa prova amb diferents entrades.
  • Afirmacions Personalitzades: Crea mètodes d'afirmació personalitzats per millorar la llegibilitat i reutilització.
  • Simulació de Dependències: Utilitza marcs de simulació com Mockito per aïllar la unitat sota prova.

Exemple de Prova Parametritzada

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

class BankAccountParameterizedTest {
    @ParameterizedTest
    @CsvSource({
        "100, 50, 150",
        "200, 100, 300",
        "50, 25, 75"
    })
    void testDeposit(int initialBalance, int depositAmount, int expectedBalance) {
        BankAccount account = new BankAccount(initialBalance);
        account.deposit(depositAmount);
        assertEquals(expectedBalance, account.getBalance());
    }
}

Conclusió

El Desenvolupament Guiat pel Comportament (BDD) amb JUnit ajuda a tancar la bretxa entre els interessats tècnics i no tècnics escrivint proves en un format clar i comprensible. Seguint l'estructura Given-When-Then, pots assegurar-te que les teves proves siguin fàcils de llegir i mantenir, cosa que condueix a una millor qualitat del programari i col·laboració.
