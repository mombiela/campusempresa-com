Introducció a les Proves de Temps d'Espera

Les proves de temps d'espera són un aspecte essencial de les proves unitàries, especialment quan es tracta de mètodes que poden trigar més a executar-se. Ajuda a assegurar que el teu codi s'executa dins d'un marc de temps acceptable i pot ser crucial per a aplicacions crítiques pel que fa al rendiment.

Per què són importants les proves de temps d'espera

  • Assegurança de Rendiment: Assegura que els mètodes es completen dins d'un límit de temps especificat.
  • Gestió de Recursos: Prevé l'acaparament de recursos per part de proves de llarga durada.
  • Fiabilitat: Ajuda a identificar possibles colls d'ampolla de rendiment.

Configuració de les Proves de Temps d'Espera a JUnit

JUnit proporciona suport integrat per a les proves de temps d'espera. Pots especificar un temps d'espera per a un mètode de prova utilitzant l'anotació @Test.

Exemple: Prova Bàsica de Temps d'Espera

Aquí tens un exemple senzill per demostrar com utilitzar la funció de temps d'espera a JUnit.

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertTimeout;
import java.time.Duration;

public class TimeoutTest {

    @Test
    void testWithTimeout() {
        assertTimeout(Duration.ofSeconds(2), () -> {
            // Simula una tasca que triga 1 segon
            Thread.sleep(1000);
        });
    }
}
  • Explicació:
    • assertTimeout(Duration.ofSeconds(2), () -> {...}): Aquest mètode assegura que el bloc de codi dins de l'expressió lambda es completi dins de 2 segons.
    • Thread.sleep(1000): Simula una tasca que triga 1 segon a completar-se.

Proves de Temps d'Espera Avançades

Per a escenaris més complexos, potser necessitis gestionar els temps d'espera de manera diferent, com aplicar temps d'espera a totes les proves d'una classe o utilitzar diferents estratègies de temps d'espera.

Aplicar Temps d'Espera a Totes les Proves d'una Classe

Pots aplicar un temps d'espera a totes les proves d'una classe utilitzant l'anotació @Timeout a nivell de classe.

import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.Timeout;
import java.util.concurrent.TimeUnit;

@Timeout(value = 2, unit = TimeUnit.SECONDS)
public class ClassLevelTimeoutTest {

    @Test
    void testMethodOne() throws InterruptedException {
        // Simula una tasca que triga 1 segon
        Thread.sleep(1000);
    }

    @Test
    void testMethodTwo() throws InterruptedException {
        // Simula una tasca que triga 1.5 segons
        Thread.sleep(1500);
    }
}
  • Explicació:
    • @Timeout(value = 2, unit = TimeUnit.SECONDS): Aquesta anotació assegura que cada mètode de prova a la classe s'ha de completar dins de 2 segons.

Gestió d'Excepcions de Temps d'Espera

Quan una prova supera el temps d'espera especificat, JUnit llança una TimeoutException. Pots gestionar aquesta excepció per realitzar accions addicionals, com ara registrar o netejar.

import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.function.Executable;
import static org.junit.jupiter.api.Assertions.assertThrows;
import java.time.Duration;

public class TimeoutExceptionTest {

    @Test
    void testTimeoutException() {
        Executable executable = () -> {
            // Simula una tasca que triga 3 segons
            Thread.sleep(3000);
        };

        assertThrows(java.util.concurrent.TimeoutException.class, () -> {
            assertTimeout(Duration.ofSeconds(2), executable);
        });
    }
}
  • Explicació:
    • assertThrows(java.util.concurrent.TimeoutException.class, () -> {...}): Aquest mètode assegura que el bloc de codi llança una TimeoutException quan supera el temps d'espera especificat.

Conclusió

Les proves de temps d'espera a JUnit són una característica poderosa que ajuda a assegurar que el teu codi s'executa de manera eficient i dins dels límits de temps acceptables. Utilitzant l'anotació @Test amb paràmetres de temps d'espera, aplicant temps d'espera a nivell de classe i gestionant excepcions de temps d'espera, pots crear proves unitàries robustes i conscients del rendiment.

Resum

  • Proves Bàsiques de Temps d'Espera: Utilitza assertTimeout per especificar límits de temps per a proves individuals.
  • Temps d'Espera a Nivell de Classe: Aplica l'anotació @Timeout per fer complir els límits de temps a totes les proves d'una classe.
  • Gestió d'Excepcions de Temps d'Espera: Utilitza assertThrows per gestionar i provar excepcions de temps d'espera.

Incorporant les proves de temps d'espera a les teves proves de JUnit, pots millorar el rendiment i la fiabilitat de les teves aplicacions.
