Introducció

L'anotació @Test és una part fonamental de JUnit, un popular marc de proves per a Java. Marca un mètode com a mètode de prova, permetent que JUnit el reconegui i l'executi com a part del conjunt de proves. Entendre com utilitzar l'anotació @Test és crucial per escriure proves unitàries efectives.

Conceptes Clau

  • Anotació: Una forma de metadades que proporciona dades sobre un programa però que no forma part del programa en si.
  • Mètode de Prova: Un mètode que conté codi per provar una funcionalitat específica de l'aplicació.
  • JUnit: Un marc de proves utilitzat per escriure i executar proves automatitzades repetibles en Java.

Ús Bàsic

Per utilitzar l'anotació @Test, cal importar-la des de la biblioteca JUnit. Aquí hi ha un exemple senzill:

import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
}

Explicació:

  • Declaracions d'Importació: import org.junit.Test; i import static org.junit.Assert.assertEquals; són necessàries per utilitzar l'anotació @Test i els mètodes d'assert.
  • Mètode de Prova: testAddition està marcat amb @Test, indicant que és un mètode de prova.
  • Assert: assertEquals(5, result); comprova si el resultat de la suma és l'esperat.

Ús Avançat

Excepcions Esperades

Pots especificar que un mètode de prova hauria de llençar una excepció utilitzant el paràmetre expected.

@Test(expected = ArithmeticException.class)
public void testDivisionByZero() {
    Calculator calculator = new Calculator();
    calculator.divide(1, 0);
}

Explicació:

  • Excepció Esperada: La prova passarà si es llença una ArithmeticException durant l'execució de testDivisionByZero.

Temps d'Espera

Pots especificar un temps d'espera per a un mètode de prova per assegurar-te que es completi dins d'un període determinat.

@Test(timeout = 1000)
public void testPerformance() {
    Calculator calculator = new Calculator();
    for (int i = 0; i < 1000000; i++) {
        calculator.add(i, i);
    }
}

Explicació:

  • Temps d'Espera: La prova fallarà si triga més de 1000 mil·lisegons en executar-se.

Combinació d'Excepcions Esperades i Temps d'Espera

Pots combinar els paràmetres expected i timeout en un sol mètode de prova.

@Test(expected = IllegalArgumentException.class, timeout = 500)
public void testInvalidInput() {
    Calculator calculator = new Calculator();
    calculator.add(null, 5);
}

Explicació:

  • Paràmetres Combinats: La prova passarà si es llença una IllegalArgumentException i el mètode es completa dins de 500 mil·lisegons.

Resum

L'anotació @Test és una eina poderosa en JUnit que et permet definir mètodes de prova fàcilment. Entenent el seu ús bàsic i avançat, pots escriure proves més efectives i eficients. Les característiques clau inclouen:

  • Marcar mètodes com a mètodes de prova.
  • Especificar excepcions esperades.
  • Establir temps d'espera per als mètodes de prova.

Dominar l'anotació @Test millorarà significativament la teva capacitat per escriure proves unitàries robustes, assegurant que el teu codi sigui fiable i mantenible.
