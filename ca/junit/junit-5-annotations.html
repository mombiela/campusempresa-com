JUnit 5 és un potent marc de proves per a aplicacions Java. Introdueix diverses anotacions que ajuden a escriure i organitzar proves. Aquest tema cobrirà les anotacions essencials de JUnit 5, els seus propòsits i com utilitzar-les de manera efectiva.

Anotacions clau a JUnit 5

JUnit 5 proporciona una varietat d'anotacions per controlar el cicle de vida de les proves i especificar comportaments de prova. Aquí teniu les anotacions més utilitzades:

  • @Test
  • @BeforeEach
  • @AfterEach
  • @BeforeAll
  • @AfterAll
  • @DisplayName
  • @Disabled
  • @RepeatedTest
  • @ParameterizedTest
  • @ExtendWith

@Test

L'anotació @Test s'utilitza per marcar un mètode com a mètode de prova.

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@BeforeEach

L'anotació @BeforeEach s'utilitza per especificar un mètode que s'ha d'executar abans de cada mètode de prova.

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @Test
    void addition() {
        assertEquals(2, calculator.add(1, 1));
    }
}

@AfterEach

L'anotació @AfterEach s'utilitza per especificar un mètode que s'ha d'executar després de cada mètode de prova.

import org.junit.jupiter.api.AfterEach;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @AfterEach
    void tearDown() {
        calculator = null;
    }

    @Test
    void addition() {
        assertEquals(2, calculator.add(1, 1));
    }
}

@BeforeAll

L'anotació @BeforeAll s'utilitza per especificar un mètode que s'ha d'executar una vegada abans de tots els mètodes de prova de la classe. Aquest mètode ha de ser estàtic.

import org.junit.jupiter.api.BeforeAll;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @BeforeAll
    static void initAll() {
        // Codi d'inicialització
    }

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@AfterAll

L'anotació @AfterAll s'utilitza per especificar un mètode que s'ha d'executar una vegada després de tots els mètodes de prova de la classe. Aquest mètode ha de ser estàtic.

import org.junit.jupiter.api.AfterAll;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @BeforeAll
    static void initAll() {
        // Codi d'inicialització
    }

    @AfterAll
    static void tearDownAll() {
        // Codi de neteja
    }

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@DisplayName

L'anotació @DisplayName s'utilitza per proporcionar un nom personalitzat per al mètode de prova.

import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    @DisplayName("Addició de dos números")
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@Disabled

L'anotació @Disabled s'utilitza per desactivar un mètode de prova o una classe de prova.

import org.junit.jupiter.api.Disabled;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    @Disabled("Desactivat fins que es resolgui l'error #42")
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@RepeatedTest

L'anotació @RepeatedTest s'utilitza per repetir una prova un nombre especificat de vegades.

import org.junit.jupiter.api.RepeatedTest;

public class CalculatorTest {

    @RepeatedTest(5)
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@ParameterizedTest

L'anotació @ParameterizedTest s'utilitza per executar la mateixa prova diverses vegades amb diferents arguments.

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

public class CalculatorTest {

    @ParameterizedTest
    @ValueSource(ints = {1, 2, 3, 4, 5})
    void isPositive(int number) {
        assertTrue(number > 0);
    }
}

@ExtendWith

L'anotació @ExtendWith s'utilitza per registrar extensions per a una classe de prova o un mètode de prova.

import org.junit.jupiter.api.extension.ExtendWith;
import org.junit.jupiter.api.Test;

@ExtendWith(MyExtension.class)
public class CalculatorTest {

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

Resum

Les anotacions de JUnit 5 proporcionen una manera potent de controlar el cicle de vida i el comportament de les vostres proves. Entendre i utilitzar aquestes anotacions de manera efectiva us pot ajudar a escriure proves més organitzades, llegibles i mantenibles. Aquí teniu un breu resum de les anotacions clau cobertes:

  • @Test: Marca un mètode com a mètode de prova.
  • @BeforeEach: Executa un mètode abans de cada mètode de prova.
  • @AfterEach: Executa un mètode després de cada mètode de prova.
  • @BeforeAll: Executa un mètode una vegada abans de tots els mètodes de prova (ha de ser estàtic).
  • @AfterAll: Executa un mètode una vegada després de tots els mètodes de prova (ha de ser estàtic).
  • @DisplayName: Proporciona un nom personalitzat per al mètode de prova.
  • @Disabled: Desactiva un mètode o classe de prova.
  • @RepeatedTest: Repeteix una prova un nombre especificat de vegades.
  • @ParameterizedTest: Executa la mateixa prova diverses vegades amb diferents arguments.
  • @ExtendWith: Registra extensions per a una classe o mètode de prova.

En dominar aquestes anotacions, podeu aprofitar al màxim les capacitats de JUnit 5 per crear suites de proves robustes i eficients.
