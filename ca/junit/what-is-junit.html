JUnit és un marc de proves àmpliament utilitzat per a la programació en Java. Proporciona una manera simple i eficient d'escriure i executar proves repetibles, assegurant que el teu codi es comporti com s'espera. JUnit és una eina essencial per als desenvolupadors que practiquen el desenvolupament guiat per proves (TDD) i per mantenir un codi d'alta qualitat.

Conceptes Clau de JUnit

  • Proves Unitàries: Provar unitats o components individuals d'un programari.
  • Fixtures de Prova: Un estat fix d'un conjunt d'objectes utilitzat com a base per executar proves.
  • Afirmacions: Declaracions que comproven si una condició és certa.
  • Anotacions: Marcadors especials en el codi que proporcionen metadades sobre els mètodes de prova.

Per què Utilitzar JUnit?

  • Automatització: Automatitza el procés d'executar proves.
  • Repetibilitat: Assegura que les proves es puguin executar múltiples vegades amb els mateixos resultats.
  • Integració: S'integra fàcilment amb eines de construcció com Maven i Gradle.
  • Suport Comunitari: Gran comunitat i documentació extensa.

Estructura Bàsica d'una Prova JUnit

Una prova JUnit típicament consisteix en els següents components:

  1. Classe de Prova: Una classe que conté mètodes de prova.
  2. Mètodes de Prova: Mètodes anotats amb @Test que contenen el codi per provar funcionalitats específiques.
  3. Afirmacions: Declaracions dins dels mètodes de prova que comproven si el resultat esperat coincideix amb el resultat real.

Exemple d'una Prova JUnit Simple

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 hauria de ser igual a 5");
    }
}

En aquest exemple:

  • L'anotació @Test marca el mètode testAddition com un mètode de prova.
  • assertEquals comprova si el resultat de calculator.add(2, 3) és igual a 5.

Anotacions Comunes en JUnit

  • @Test: Marca un mètode com un mètode de prova.
  • @BeforeEach: S'executa abans de cada mètode de prova.
  • @AfterEach: S'executa després de cada mètode de prova.
  • @BeforeAll: S'executa una vegada abans de tots els mètodes de prova de la classe.
  • @AfterAll: S'executa una vegada després de tots els mètodes de prova de la classe.

Exemple Utilitzant Múltiples Anotacions

import org.junit.jupiter.api.*;

public class LifecycleTest {

    @BeforeAll
    static void setupAll() {
        System.out.println("Abans de totes les proves");
    }

    @BeforeEach
    void setup() {
        System.out.println("Abans de cada prova");
    }

    @Test
    void testOne() {
        System.out.println("Prova Un");
    }

    @Test
    void testTwo() {
        System.out.println("Prova Dos");
    }

    @AfterEach
    void tearDown() {
        System.out.println("Després de cada prova");
    }

    @AfterAll
    static void tearDownAll() {
        System.out.println("Després de totes les proves");
    }
}

Afirmacions en JUnit

Les afirmacions s'utilitzen per validar els resultats esperats. Aquí hi ha algunes afirmacions comunes:

| Mètode d'Afirmació | Descripció | |----------------------------|------------------------------------------------| | assertEquals(expected, actual) | Comprova si dos valors són iguals. | | assertTrue(condition) | Comprova si una condició és certa. | | assertFalse(condition) | Comprova si una condició és falsa. | | assertNull(object) | Comprova si un objecte és nul. | | assertNotNull(object) | Comprova si un objecte no és nul. | | assertThrows(exception.class, executable) | Comprova si es llança una excepció. |

Exemple d'Afirmacions

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class AssertionTest {

    @Test
    public void testAssertions() {
        assertEquals(4, 2 + 2);
        assertTrue(5 > 3);
        assertFalse(3 > 5);
        assertNull(null);
        assertNotNull(new Object());
        assertThrows(ArithmeticException.class, () -> {
            int result = 1 / 0;
        });
    }
}

Conclusió

JUnit és un marc poderós i essencial per als desenvolupadors de Java, permetent-los escriure i executar proves de manera eficient. Entenent l'estructura bàsica, les anotacions i les afirmacions, pots començar a escriure les teves pròpies proves per assegurar que el teu codi sigui fiable i mantenible. A mesura que avancis, exploraràs característiques més complexes i millors pràctiques per millorar encara més les teves habilitats de prova.
