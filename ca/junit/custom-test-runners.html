En aquesta secció, explorarem el concepte de runners de proves personalitzats a JUnit. Els runners de proves són una part integral de JUnit, responsables d'executar les proves i informar dels seus resultats. Tot i que JUnit proporciona diversos runners de proves integrats, hi ha escenaris en què potser necessiteu crear el vostre propi runner de proves personalitzat per complir requisits específics.

Què és un Runner de Proves?

Un runner de proves és un component de JUnit que orquestra l'execució de les proves. És responsable de:

  • Descobrir els mètodes de prova.
  • Executar els mètodes de prova.
  • Informar dels resultats de les proves.

JUnit proporciona diversos runners de proves integrats, com ara:

  • BlockJUnit4ClassRunner: El runner per defecte per a proves JUnit 4.
  • Parameterized: Un runner per a proves parametritzades.
  • Suite: Un runner per executar diverses classes de proves juntes.

Per què Crear un Runner de Proves Personalitzat?

Crear un runner de proves personalitzat pot ser útil en diversos escenaris, com ara:

  • Personalitzar el procés d'execució de les proves.
  • Integrar-se amb altres eines o marcs.
  • Afegir registre o informes personalitzats.
  • Implementar polítiques específiques d'execució de proves.

Crear un Runner de Proves Personalitzat

Per crear un runner de proves personalitzat, heu d'extendre la classe org.junit.runner.Runner i sobreescriure els seus mètodes. Aquí teniu una guia pas a pas:

  1. Extendre la Classe Runner:

    import org.junit.runner.Runner;
import org.junit.runner.notification.RunNotifier;
import org.junit.runners.model.InitializationError;

public class CustomRunner extends Runner {
    private final Class<?> testClass;

    public CustomRunner(Class<?> testClass) throws InitializationError {
        this.testClass = testClass;
    }

    @Override
    public Description getDescription() {
        return Description.createTestDescription(testClass, "Custom Test Runner");
    }

    @Override
    public void run(RunNotifier notifier) {
        // Lògica d'execució de proves personalitzada
    }
}

  2. Sobreescriure el Mètode getDescription:

    • Aquest mètode proporciona una descripció de les proves que s'estan executant.
    • S'utilitza per JUnit per mostrar els noms i l'estructura de les proves.

  3. Sobreescriure el Mètode run:

    • Aquest mètode conté la lògica per executar les proves.
    • Podeu afegir lògica personalitzada aquí per controlar com s'executen i s'informen les proves.

Exemple: Runner Personalitzat amb Registre

Creem un runner personalitzat que registra l'inici i el final de cada mètode de prova:

import org.junit.runner.Description;
import org.junit.runner.Runner;
import org.junit.runner.notification.RunNotifier;
import org.junit.runner.notification.RunListener;
import org.junit.runners.model.InitializationError;
import org.junit.runners.model.Statement;

public class LoggingRunner extends Runner {
    private final Class<?> testClass;

    public LoggingRunner(Class<?> testClass) throws InitializationError {
        this.testClass = testClass;
    }

    @Override
    public Description getDescription() {
        return Description.createTestDescription(testClass, "Logging Test Runner");
    }

    @Override
    public void run(RunNotifier notifier) {
        notifier.addListener(new RunListener() {
            @Override
            public void testStarted(Description description) throws Exception {
                System.out.println("Starting: " + description.getMethodName());
            }

            @Override
            public void testFinished(Description description) throws Exception {
                System.out.println("Finished: " + description.getMethodName());
            }
        });

        try {
            Object testInstance = testClass.getDeclaredConstructor().newInstance();
            for (java.lang.reflect.Method method : testClass.getMethods()) {
                if (method.isAnnotationPresent(org.junit.Test.class)) {
                    Statement statement = new Statement() {
                        @Override
                        public void evaluate() throws Throwable {
                            method.invoke(testInstance);
                        }
                    };
                    statement.evaluate();
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Utilitzar el Runner Personalitzat

Per utilitzar el runner personalitzat, anoteu la vostra classe de prova amb @RunWith:

import org.junit.Test;
import org.junit.runner.RunWith;

@RunWith(LoggingRunner.class)
public class ExampleTest {
    @Test
    public void testMethod1() {
        System.out.println("Executing testMethod1");
    }

    @Test
    public void testMethod2() {
        System.out.println("Executing testMethod2");
    }
}

Conclusió

Els runners de proves personalitzats a JUnit proporcionen una manera poderosa de controlar i estendre el procés d'execució de les proves. Creant un runner personalitzat, podeu integrar-vos amb altres eines, afegir registre personalitzat o implementar polítiques específiques d'execució de proves. Entendre com crear i utilitzar runners de proves personalitzats pot millorar significativament les vostres capacitats de prova i permetre-vos adaptar el procés de prova a les vostres necessitats específiques.
