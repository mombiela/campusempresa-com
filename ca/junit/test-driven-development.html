Introducció al TDD

El Desenvolupament Basat en Proves (TDD) és un procés de desenvolupament de programari on escrius proves abans d'escriure el codi real. Aquest enfocament ajuda a assegurar que el teu codi sigui fiable, mantenible i lliure d'errors.

  • Cicle Red-Green-Refactor:
    1. Red: Escriu una prova que falli.
    2. Green: Escriu el codi mínim necessari per passar la prova.
    3. Refactor: Millora el codi mantenint la prova passant.

Beneficis del TDD

  • Millora de la Qualitat del Codi: Escriure proves primer t'obliga a pensar en els requisits i el disseny abans de la implementació.
  • Reducció d'Errors: Les proves detecten errors a l'inici del procés de desenvolupament.
  • Documentació: Les proves serveixen com a documentació de com ha de funcionar el codi.
  • Confiança en la Refactorització: Amb un conjunt de proves, pots refactoritzar el codi amb la confiança que no trencaràs la funcionalitat existent.

Configuració de JUnit

Abans d'entrar en el TDD, necessites configurar JUnit en el teu projecte.

Configuració de Maven

Afegeix la següent dependència al teu fitxer pom.xml:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.7.0</version>
    <scope>test</scope>
</dependency>

Configuració de Gradle

Afegeix el següent al teu fitxer build.gradle:

testImplementation 'org.junit.jupiter:junit-jupiter-api:5.7.0'
testRuntimeOnly 'org.junit.jupiter:junit-jupiter-engine:5.7.0'

Escriure la Teva Primera Prova

Comencem amb un exemple senzill per entendre el procés de TDD.

Exemple: Afecció en una Calculadora

  1. Red: Escriu una prova que falli.
import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {
    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        assertEquals(5, calculator.add(2, 3));
    }
}
  • Aquesta prova fallarà perquè la classe Calculator i el mètode add encara no existeixen.
  1. Green: Escriu el codi mínim necessari per passar la prova.
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}
  • Ara, la prova hauria de passar perquè el mètode add retorna la suma correcta.
  1. Refactor: Millora el codi mantenint la prova passant.
  • En aquest exemple senzill, potser no hi ha gaire a refactoritzar, però sempre busca oportunitats per millorar la llegibilitat i mantenibilitat del codi.

Conceptes Avançats de TDD

Mocking i Stubbing

Quan escrius proves, potser necessites aïllar la unitat de codi que estàs provant. Aquí és on entren en joc el mocking i el stubbing.

  • Mocking: Crear una versió falsa d'un objecte per controlar el seu comportament.
  • Stubbing: Proporcionar respostes predefinides a les crides de mètodes.

Exemple: Utilitzant Mockito

Afegeix la dependència de Mockito al teu projecte:

<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>3.6.0</version>
    <scope>test</scope>
</dependency>

Exemple de mocking d'un servei:

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {
    @Test
    public void testGetUser() {
        UserRepository mockRepo = mock(UserRepository.class);
        when(mockRepo.findById(1)).thenReturn(new User(1, "John Doe"));

        UserService userService = new UserService(mockRepo);
        User user = userService.getUser(1);

        assertEquals("John Doe", user.getName());
    }
}

Proves Parametritzades

JUnit et permet executar la mateixa prova amb diferents paràmetres.

Exemple:

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;
import static org.junit.jupiter.api.Assertions.assertTrue;

public class ParameterizedTestExample {
    @ParameterizedTest
    @ValueSource(strings = {"racecar", "radar", "level"})
    public void testIsPalindrome(String word) {
        assertTrue(isPalindrome(word));
    }

    public boolean isPalindrome(String word) {
        return new StringBuilder(word).reverse().toString().equals(word);
    }
}

Conclusió

El Desenvolupament Basat en Proves (TDD) és una metodologia poderosa que pot millorar significativament la qualitat i mantenibilitat del teu codi. Escriure proves primer assegura que el teu codi compleix els requisits i està lliure d'errors. Utilitzant JUnit, pots implementar fàcilment el TDD en els teus projectes Java, i amb tècniques avançades com el mocking i les proves parametritzades, pots manejar escenaris més complexos. Recorda el cicle Red-Green-Refactor i aplica'l consistentment per obtenir els beneficis del TDD.
