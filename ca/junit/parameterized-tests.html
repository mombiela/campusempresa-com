Les proves parametritzades permeten executar la mateixa prova diverses vegades amb diferents entrades. Això és particularment útil per provar funcions que haurien de comportar-se de la mateixa manera per a una varietat d'entrades.

Introducció a les Proves Parametritzades

  • Definició: Les proves parametritzades permeten a un desenvolupador executar la mateixa prova amb diferents conjunts de paràmetres.
  • Propòsit: Ajuden a reduir la duplicació de codi i a millorar la cobertura de proves.

Configuració de Proves Parametritzades a JUnit

Per utilitzar proves parametritzades a JUnit, cal incloure la dependència junit-jupiter-params al teu projecte. Aquí tens un exemple per a un projecte Maven:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-params</artifactId>
    <version>5.8.1</version>
    <scope>test</scope>
</dependency>

Escriure una Prova Parametritzada

Pas 1: Importar les Anotacions Necessàries

Cal importar les següents anotacions:

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

Pas 2: Crear un Mètode de Prova Parametritzada

Aquí tens un exemple senzill que prova si un número és parell:

import static org.junit.jupiter.api.Assertions.assertTrue;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

public class EvenNumberTest {

    @ParameterizedTest
    @ValueSource(ints = {2, 4, 6, 8, 10})
    void testIsEven(int number) {
        assertTrue(number % 2 == 0);
    }
}
  • Explicació:
    • @ParameterizedTest: Marca el mètode com una prova parametritzada.
    • @ValueSource(ints = {2, 4, 6, 8, 10}): Proporciona els paràmetres per al mètode de prova. En aquest cas, proporciona una matriu d'enters.

Proves Parametritzades Avançades

Utilitzant `@CsvSource`

Pots utilitzar @CsvSource per passar múltiples arguments al teu mètode de prova:

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

public class StringUtilsTest {

    @ParameterizedTest
    @CsvSource({
        "hello, HELLO",
        "world, WORLD",
        "junit, JUNIT"
    })
    void testToUpperCase(String input, String expected) {
        assertEquals(expected, input.toUpperCase());
    }
}
  • Explicació:
    • @CsvSource: Et permet passar múltiples conjunts d'arguments al mètode de prova.
    • El mètode de prova testToUpperCase pren dos paràmetres: input i expected.

Utilitzant `@MethodSource`

Per a escenaris més complexos, pots utilitzar @MethodSource per proporcionar paràmetres des d'un mètode:

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.MethodSource;
import java.util.stream.Stream;

public class MathUtilsTest {

    @ParameterizedTest
    @MethodSource("provideNumbersForAddition")
    void testAddition(int a, int b, int expected) {
        assertEquals(expected, a + b);
    }

    static Stream<Arguments> provideNumbersForAddition() {
        return Stream.of(
            Arguments.of(1, 1, 2),
            Arguments.of(2, 3, 5),
            Arguments.of(3, 5, 8)
        );
    }
}
  • Explicació:
    • @MethodSource("provideNumbersForAddition"): Especifica el mètode que proporciona els paràmetres.
    • El mètode provideNumbersForAddition retorna un Stream de Arguments.

Conclusió

Les proves parametritzades a JUnit són una característica poderosa que pot ajudar-te a escriure proves més eficients i completes. Utilitzant anotacions com @ValueSource, @CsvSource i @MethodSource, pots executar fàcilment la mateixa prova amb diferents entrades, reduint la duplicació de codi i millorant la cobertura de proves.

  • Punts Clau:
    • Les proves parametritzades permeten executar la mateixa prova amb diferents entrades.
    • Ajuden a reduir la duplicació de codi i a millorar la cobertura de proves.
    • JUnit proporciona diverses anotacions (@ValueSource, @CsvSource, @MethodSource) per facilitar les proves parametritzades.

Dominant les proves parametritzades, pots millorar significativament la robustesa i mantenibilitat dels teus conjunts de proves.
