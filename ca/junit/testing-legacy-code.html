Introducció

Provar codi heretat pot ser un repte, però és crucial per mantenir i millorar la qualitat del teu programari. El codi heretat sovint manca de proves, cosa que fa difícil refactoritzar o estendre sense introduir errors. Aquesta secció et guiarà a través del procés de provar codi heretat utilitzant JUnit, des d'entendre els conceptes bàsics fins a implementar tècniques avançades.

Conceptes Clau

  • Codi Heretat: Codi heretat d'una versió anterior del programari que pot no tenir proves.
  • Refactorització: El procés de reestructurar codi existent sense canviar el seu comportament extern.
  • Cobertura de Proves: Una mesura de quant del codi està provat per proves automatitzades.
  • Mocking: Crear objectes que simulen el comportament d'objectes reals de manera controlada.

Reptes de Provar Codi Heretat

  • Manca de proves existents.
  • Alta dependència i couplatge.
  • Documentació deficient.
  • Risc d'introduir errors durant la refactorització.

Estratègies per Provar Codi Heretat

  1. Proves de Caracterització: Escriu proves per entendre el comportament actual del codi.
  2. Refactorització Incremental: Refactoritza el codi gradualment mentre afegeixes proves.
  3. Injecció de Dependències: Utilitza la injecció de dependències per fer el codi més provable.
  4. Mocking i Stubbing: Utilitza mocks i stubs per aïllar el codi sota prova.

Escriure Proves de Caracterització

Les proves de caracterització t'ajuden a entendre què fa el codi heretat. Capturen el comportament actual, que pot ser útil quan es refactoritza.

Exemple

public class LegacyCalculator {
    public int add(int a, int b) {
        return a + b;
    }
}

// Prova de Caracterització
import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class LegacyCalculatorTest {
    @Test
    public void testAdd() {
        LegacyCalculator calculator = new LegacyCalculator();
        assertEquals(5, calculator.add(2, 3));
    }
}

Explicació:

  • Creem una prova simple per al mètode add per capturar el seu comportament actual.
  • Aquesta prova assegura que qualsevol canvi futur al mètode add no alteri el seu comportament esperat.

Refactorització Incremental

Refactoritza el codi en petits passos mentre assegures que les proves existents passin. Aquest enfocament minimitza el risc d'introduir errors.

Exemple

Refactoritza un mètode altament couplat introduint la injecció de dependències.

Abans de la Refactorització:

public class OrderService {
    private Database database = new Database();

    public void placeOrder(Order order) {
        database.save(order);
    }
}

Després de la Refactorització:

public class OrderService {
    private Database database;

    public OrderService(Database database) {
        this.database = database;
    }

    public void placeOrder(Order order) {
        database.save(order);
    }
}

Prova per al Codi Refactoritzat

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class OrderServiceTest {
    @Test
    public void testPlaceOrder() {
        Database mockDatabase = mock(Database.class);
        OrderService orderService = new OrderService(mockDatabase);
        Order order = new Order();

        orderService.placeOrder(order);

        verify(mockDatabase).save(order);
    }
}

Explicació:

  • Refactoritzem OrderService per utilitzar la injecció de dependències.
  • Creem un objecte Database mock per provar OrderService en aïllament.

Mocking i Stubbing

Utilitza frameworks de mocking com Mockito per crear mocks i stubs per a les dependències. Això et permet provar el codi en aïllament.

Exemple amb Mockito

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class PaymentServiceTest {
    @Test
    public void testProcessPayment() {
        PaymentGateway mockGateway = mock(PaymentGateway.class);
        PaymentService paymentService = new PaymentService(mockGateway);
        Payment payment = new Payment();

        when(mockGateway.process(payment)).thenReturn(true);

        boolean result = paymentService.processPayment(payment);

        assertTrue(result);
        verify(mockGateway).process(payment);
    }
}

Explicació:

  • Creem un objecte PaymentGateway mock.
  • Stubem el mètode process per retornar true.
  • Verifiquem que el mètode process es va cridar amb l'objecte de pagament esperat.

Conclusió

Provar codi heretat és una habilitat crítica per mantenir i millorar la qualitat del programari. Escriure proves de caracterització, refactoritzar incrementalment i utilitzar frameworks de mocking et permetrà provar i refactoritzar codi heretat de manera efectiva. Aquestes tècniques t'ajudaran a assegurar que el teu codi es mantingui fiable i mantenible al llarg del temps.
