Introducció a les Regles de JUnit

Les regles de JUnit són una característica potent que et permet afegir comportament addicional als teus tests. Proporcionen una manera d'encapsular la lògica de test reutilitzable i es poden utilitzar per gestionar recursos, manejar excepcions o modificar el comportament dels mètodes de test.

Conceptes Clau de les Regles de JUnit

  • Regla: Un objecte que pot interceptar les crides als mètodes de test i afegir comportament addicional.
  • Anotació @Rule: Utilitzada per declarar una regla en una classe de test.
  • Interfície TestRule: La interfície que s'ha d'implementar per crear una regla personalitzada.
  • ExternalResource: Una classe base per a regles que configuren i desmunten recursos externs.

Regles de JUnit Integrades

JUnit proporciona diverses regles integrades que es poden utilitzar per simplificar tasques de test comuns.

Regla TemporaryFolder

La regla TemporaryFolder et permet crear fitxers i carpetes que s'eliminen automàticament després que el mètode de test acabi.

public class TemporaryFolderTest {
    @Rule
    public TemporaryFolder folder = new TemporaryFolder();

    @Test
    public void testUsingTempFolder() throws IOException {
        File createdFile = folder.newFile("myfile.txt");
        File createdFolder = folder.newFolder("subfolder");
        
        assertTrue(createdFile.exists());
        assertTrue(createdFolder.exists());
    }
}

Regla ExpectedException

La regla ExpectedException et permet verificar que el teu codi llança una excepció específica.

public class ExpectedExceptionTest {
    @Rule
    public ExpectedException thrown = ExpectedException.none();

    @Test
    public void testException() {
        thrown.expect(NullPointerException.class);
        thrown.expectMessage("Null pointer exception");

        throw new NullPointerException("Null pointer exception");
    }
}

Regla Timeout

La regla Timeout es pot utilitzar per fallar un test si triga més del temps especificat.

public class TimeoutTest {
    @Rule
    public Timeout globalTimeout = Timeout.seconds(2);

    @Test
    public void testWithTimeout() throws InterruptedException {
        Thread.sleep(1000); // Això passarà
    }

    @Test
    public void testWithTimeoutExceeded() throws InterruptedException {
        Thread.sleep(3000); // Això fallarà
    }
}

Creació de Regles Personalitzades

Pots crear les teves pròpies regles personalitzades implementant la interfície TestRule.

Exemple: Regla de Registre Personalitzada

Aquesta regla personalitzada registra l'inici i el final de cada mètode de test.

public class LoggingRule implements TestRule {
    @Override
    public Statement apply(Statement base, Description description) {
        return new Statement() {
            @Override
            public void evaluate() throws Throwable {
                System.out.println("Starting test: " + description.getMethodName());
                try {
                    base.evaluate();
                } finally {
                    System.out.println("Finished test: " + description.getMethodName());
                }
            }
        };
    }
}

public class CustomRuleTest {
    @Rule
    public LoggingRule loggingRule = new LoggingRule();

    @Test
    public void testMethod1() {
        System.out.println("Executing testMethod1");
    }

    @Test
    public void testMethod2() {
        System.out.println("Executing testMethod2");
    }
}

Conclusió

Les regles de JUnit proporcionen una manera flexible i potent d'afegir comportament addicional als teus tests. Utilitzant regles integrades com TemporaryFolder, ExpectedException i Timeout, pots simplificar tasques de test comuns. A més, crear regles personalitzades et permet encapsular la lògica de test reutilitzable, fent que els teus tests siguin més nets i mantenibles. Entendre i aprofitar les regles de JUnit pot millorar significativament les teves capacitats de test i la qualitat del teu conjunt de tests.
