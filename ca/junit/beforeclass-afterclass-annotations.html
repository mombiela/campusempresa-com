Introducció

JUnit és un marc de proves popular per a Java que ajuda els desenvolupadors a escriure i executar proves repetibles. Entre les seves moltes característiques, JUnit proporciona anotacions per gestionar el cicle de vida de les classes i mètodes de prova. Dues anotacions importants per gestionar la configuració i el desmantellament a nivell de classe són @BeforeClass i @AfterClass.

Conceptes Clau

  • @BeforeClass: Aquesta anotació s'utilitza per especificar un mètode que s'hauria d'executar una vegada abans de qualsevol dels mètodes de prova de la classe. Normalment s'utilitza per a operacions de configuració costoses que només cal realitzar una vegada.
  • @AfterClass: Aquesta anotació s'utilitza per especificar un mètode que s'hauria d'executar una vegada després que tots els mètodes de prova de la classe hagin estat executats. Normalment s'utilitza per a operacions de neteja que només cal realitzar una vegada.

Ús de @BeforeClass i @AfterClass

@BeforeClass

El mètode anotat amb @BeforeClass s'executa una vegada abans de qualsevol dels mètodes de prova de la classe. Aquest mètode ha de ser public i static.

import org.junit.BeforeClass;
import org.junit.Test;

public class ExampleTest {

    @BeforeClass
    public static void setUpBeforeClass() {
        System.out.println("Executat abans de qualsevol mètode de prova en aquesta classe");
        // Realitza operacions de configuració costoses aquí
    }

    @Test
    public void testMethod1() {
        System.out.println("Mètode de Prova 1");
        // Lògica de prova aquí
    }

    @Test
    public void testMethod2() {
        System.out.println("Mètode de Prova 2");
        // Lògica de prova aquí
    }
}

@AfterClass

El mètode anotat amb @AfterClass s'executa una vegada després que tots els mètodes de prova de la classe hagin estat executats. Aquest mètode també ha de ser public i static.

import org.junit.AfterClass;
import org.junit.Test;

public class ExampleTest {

    @BeforeClass
    public static void setUpBeforeClass() {
        System.out.println("Executat abans de qualsevol mètode de prova en aquesta classe");
        // Realitza operacions de configuració costoses aquí
    }

    @AfterClass
    public static void tearDownAfterClass() {
        System.out.println("Executat després de tots els mètodes de prova en aquesta classe");
        // Realitza operacions de neteja aquí
    }

    @Test
    public void testMethod1() {
        System.out.println("Mètode de Prova 1");
        // Lògica de prova aquí
    }

    @Test
    public void testMethod2() {
        System.out.println("Mètode de Prova 2");
        // Lògica de prova aquí
    }
}

Escenari d'Exemple

Considera un escenari on necessites provar una connexió a una base de dades. Potser voldràs establir la connexió una vegada abans que es realitzin les proves i tancar-la una vegada després que totes les proves hagin finalitzat.

import org.junit.BeforeClass;
import org.junit.AfterClass;
import org.junit.Test;

public class DatabaseTest {

    private static DatabaseConnection connection;

    @BeforeClass
    public static void setUpBeforeClass() {
        System.out.println("Configurant la connexió a la base de dades");
        connection = new DatabaseConnection();
        connection.connect();
    }

    @AfterClass
    public static void tearDownAfterClass() {
        System.out.println("Tancant la connexió a la base de dades");
        connection.disconnect();
    }

    @Test
    public void testDatabaseQuery1() {
        System.out.println("Executant testDatabaseQuery1");
        // Lògica de prova utilitzant la connexió
    }

    @Test
    public void testDatabaseQuery2() {
        System.out.println("Executant testDatabaseQuery2");
        // Lògica de prova utilitzant la connexió
    }
}

Resum

  • Les anotacions @BeforeClass i @AfterClass s'utilitzen per definir mètodes que s'executen una vegada abans i després de tots els mètodes de prova en una classe, respectivament.
  • Aquests mètodes han de ser public i static.
  • Són útils per realitzar operacions de configuració i neteja costoses que només cal fer una vegada per a tota la classe de prova.

Entendre i utilitzar les anotacions @BeforeClass i @AfterClass pot ajudar-te a escriure proves més eficients i organitzades, especialment quan es tracta d'operacions de configuració i desmantellament intensives en recursos.
