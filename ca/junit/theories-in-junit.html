Introducció a les Teories en JUnit

Les teories en JUnit són una característica poderosa que et permet provar el teu codi amb una àmplia gamma de valors d'entrada. A diferència de les proves unitàries tradicionals que proven un mètode amb un conjunt específic d'entrades, les teories proven el mètode amb múltiples conjunts d'entrades per assegurar-se que es comporta correctament sota diverses condicions.

Conceptes Clau

  • Teoria: Un tipus especial de prova que està parametritzada i es pot executar amb múltiples conjunts de dades.
  • Punts de Dades: Els conjunts de dades que s'utilitzen per executar la teoria.
  • Suposicions: Condicions que han de ser certes perquè la teoria s'executi.

Configuració de les Teories

Per utilitzar teories en JUnit, necessites incloure la biblioteca junit-theories en el teu projecte. Si estàs utilitzant Maven, afegeix la següent dependència al teu pom.xml:

<dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.13.2</version>
</dependency>

Escriure una Teoria Simple

Aquí tens un exemple simple d'una teoria en JUnit:

import org.junit.experimental.theories.Theories;
import org.junit.experimental.theories.Theory;
import org.junit.experimental.theories.DataPoint;
import org.junit.runner.RunWith;
import static org.junit.Assume.assumeTrue;
import static org.junit.Assert.assertTrue;

@RunWith(Theories.class)
public class SimpleTheoryTest {

    @DataPoint
    public static int positiveNumber = 1;

    @DataPoint
    public static int negativeNumber = -1;

    @Theory
    public void testAbsoluteValue(int number) {
        assumeTrue(number != 0); // Salta la prova si el número és zero
        assertTrue(Math.abs(number) >= 0);
    }
}

Explicació

  • @RunWith(Theories.class): Aquesta anotació indica a JUnit que executi la classe com una prova de teoria.
  • @DataPoint: Aquesta anotació marca els punts de dades que s'utilitzaran en la teoria.
  • @Theory: Aquesta anotació marca el mètode com una teoria.
  • assumeTrue: Aquest mètode s'utilitza per saltar la prova si la suposició és falsa.

Utilitzant Múltiples Punts de Dades

Pots definir múltiples punts de dades per provar la teva teoria amb una gamma més àmplia d'entrades:

@RunWith(Theories.class)
public class MultipleDataPointsTest {

    @DataPoint
    public static int positiveNumber = 1;

    @DataPoint
    public static int negativeNumber = -1;

    @DataPoint
    public static int zero = 0;

    @Theory
    public void testNonNegativeAbsoluteValue(int number) {
        assertTrue(Math.abs(number) >= 0);
    }
}

Combinant Punts de Dades

També pots combinar múltiples punts de dades per provar escenaris més complexos:

@RunWith(Theories.class)
public class CombinedDataPointsTest {

    @DataPoint
    public static int positiveNumber = 1;

    @DataPoint
    public static int negativeNumber = -1;

    @DataPoint
    public static int zero = 0;

    @DataPoint
    public static String nonEmptyString = "JUnit";

    @DataPoint
    public static String emptyString = "";

    @Theory
    public void testStringLength(String str, int number) {
        assumeTrue(str != null);
        assertTrue(str.length() >= 0);
    }
}

Suposicions en les Teories

Les suposicions s'utilitzen per filtrar punts de dades que no compleixen certs criteris. Si una suposició falla, la teoria es salta per aquell conjunt de punts de dades:

@RunWith(Theories.class)
public class AssumptionsTest {

    @DataPoint
    public static int positiveNumber = 1;

    @DataPoint
    public static int negativeNumber = -1;

    @DataPoint
    public static int zero = 0;

    @Theory
    public void testPositiveNumbers(int number) {
        assumeTrue(number > 0); // Només prova números positius
        assertTrue(number > 0);
    }
}

Conclusió

Les teories en JUnit proporcionen una manera flexible de provar el teu codi amb una varietat d'entrades, assegurant que es comporta correctament sota diferents condicions. Utilitzant punts de dades i suposicions, pots crear proves completes que cobreixin una àmplia gamma d'escenaris. Aquest enfocament ajuda a identificar casos límit i possibles errors que podrien no ser detectats amb les proves unitàries tradicionals.

A la següent secció, explorarem característiques més avançades de JUnit, incloent executors personalitzats i regles, per millorar encara més les teves capacitats de prova.
