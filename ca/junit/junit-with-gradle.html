Introducció a JUnit i Gradle

JUnit és un popular framework de proves per a Java, mentre que Gradle és una potent eina d'automatització de compilació. Combinar JUnit amb Gradle permet processos de proves eficients i automatitzats.

Configuració de JUnit amb Gradle

Per començar amb JUnit en un projecte Gradle, cal configurar el fitxer build.gradle.

Configuració Pas a Pas

  1. Crear un Projecte Gradle:

    • Si no tens un projecte Gradle, crea'n un utilitzant la següent comanda: 
      gradle init --type java-application

  2. Afegir la Dependència de JUnit:

    • Obre el teu fitxer build.gradle i afegeix la dependència de JUnit: 
      dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.8.1'
}

  3. Aplicar el Plugin de Java:

    • Assegura't que el plugin de Java està aplicat en el teu build.gradle: 
      plugins {
    id 'java'
}

  4. Configurar la Tasca de Proves:

    • Configura la tasca de proves per utilitzar la Plataforma JUnit: 
      test {
    useJUnitPlatform()
}

Exemple de Fitxer `build.gradle`

plugins {
    id 'java'
}

repositories {
    mavenCentral()
}

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.8.1'
}

test {
    useJUnitPlatform()
}

Escriure i Executar Proves amb JUnit i Gradle

Un cop el teu projecte està configurat, pots començar a escriure i executar proves.

Escriure una Prova Simple

Crea una classe de proves en el directori src/test/java:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

Executar Proves

  • Línia de Comandes:
    • Executa les proves utilitzant la següent comanda: 
      gradle test
  • Integració amb IDE:
    • La majoria d'IDEs com IntelliJ IDEA o Eclipse poden executar tasques de Gradle directament. Fes clic dret sobre la classe de proves o el mètode i selecciona "Run".

Configuració Avançada

Per a projectes més complexos, potser necessitis configuracions avançades.

Configuracions Personalitzades de Proves

  • Execució Paral·lela de Proves: 
    test {
    maxParallelForks = 4
}
  • Filtratge de Proves: 
    test {
    include '**/CalculatorTest.class'
}

Compilacions Multi-Projecte

Si tens una compilació multi-projecte, configura cada subprojecte:

subprojects {
    apply plugin: 'java'

    repositories {
        mavenCentral()
    }

    dependencies {
        testImplementation 'org.junit.jupiter:junit-jupiter:5.8.1'
    }

    test {
        useJUnitPlatform()
    }
}

Conclusió

Integrar JUnit amb Gradle agilita el procés de proves, fent més fàcil escriure, executar i gestionar proves. Seguint els passos de configuració, pots aprofitar tot el potencial de les dues eines per mantenir una alta qualitat i fiabilitat del codi en els teus projectes Java.
