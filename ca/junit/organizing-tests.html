Organitzant Proves
Introducció al Mocking ►

En aquesta secció, explorarem com organitzar efectivament les teves proves quan utilitzes JUnit. Una bona organització de les proves és crucial per mantenir una base de codi neta, comprensible i escalable. Cobrirem les millors pràctiques, estructures de directoris, convencions de nomenclatura i l'ús de les característiques de JUnit per mantenir les teves proves ben organitzades.

Millors Pràctiques per Organitzar Proves

  • Separació de Responsabilitats: Mantingues el teu codi de prova separat del teu codi de producció.
  • Convencions de Nomenclatura Consistents: Utilitza noms clars i consistents per a les classes i mètodes de prova.
  • Proves Modulars: Escriu proves petites i enfocades que provin una cosa a la vegada.
  • Ús d'Anotacions: Aprofita les anotacions de JUnit per gestionar el cicle de vida i l'organització de les proves.

Estructura de Directoris

Una estructura de directoris ben organitzada ajuda a localitzar i gestionar fàcilment els fitxers de prova. Una estructura comuna és:

arrel-projecte/
├── src/
│   ├── main/
│   │   └── java/
│   │       └── com/
│   │           └── exemple/
│   │               └── MyClass.java
│   └── test/
│       └── java/
│           └── com/
│               └── exemple/
│                   └── MyClassTest.java
  • src/main/java: Conté el teu codi de producció.
  • src/test/java: Conté el teu codi de prova.

Convencions de Nomenclatura

  • Noms de Classes de Prova: Haurien de reflectir la classe que estan provant. Per exemple, MyClass hauria de tenir una classe de prova corresponent anomenada MyClassTest.
  • Noms de Mètodes de Prova: Haurien de descriure el comportament que s'està provant. Utilitza un patró com nomMètode_condició_resultatEsperat.

Exemple

public class MyClassTest {

    @Test
    public void add_twoPositiveNumbers_returnsPositiveSum() {
        MyClass myClass = new MyClass();
        int result = myClass.add(2, 3);
        assertEquals(5, result);
    }

    @Test
    public void add_negativeAndPositiveNumber_returnsCorrectSum() {
        MyClass myClass = new MyClass();
        int result = myClass.add(-2, 3);
        assertEquals(1, result);
    }
}

Ús d'Anotacions de JUnit

JUnit proporciona diverses anotacions per ajudar a organitzar i gestionar les teves proves:

  • @Test: Marca un mètode com a mètode de prova.
  • @BeforeEach: S'executa abans de cada mètode de prova.
  • @AfterEach: S'executa després de cada mètode de prova.
  • @BeforeAll: S'executa una vegada abans de tots els mètodes de prova de la classe.
  • @AfterAll: S'executa una vegada després de tots els mètodes de prova de la classe.
  • @Disabled: Desactiva temporalment un mètode o classe de prova.

Exemple

public class MyClassTest {

    private MyClass myClass;

    @BeforeAll
    public static void initAll() {
        System.out.println("Abans de totes les proves");
    }

    @BeforeEach
    public void init() {
        myClass = new MyClass();
    }

    @Test
    public void add_twoPositiveNumbers_returnsPositiveSum() {
        int result = myClass.add(2, 3);
        assertEquals(5, result);
    }

    @Test
    public void add_negativeAndPositiveNumber_returnsCorrectSum() {
        int result = myClass.add(-2, 3);
        assertEquals(1, result);
    }

    @AfterEach
    public void tearDown() {
        System.out.println("Després de cada prova");
    }

    @AfterAll
    public static void tearDownAll() {
        System.out.println("Després de totes les proves");
    }
}

Agrupant Proves

JUnit 5 et permet agrupar proves utilitzant classes niades i etiquetes:

  • Classes Niades: Organitza proves relacionades dins de classes niades.
  • Etiquetes: Utilitza etiquetes per categoritzar proves i executar grups específics de proves.

Exemple

@Tag("fast")
public class MyClassTest {

    @Nested
    @Tag("addition")
    class AdditionTests {

        @Test
        public void add_twoPositiveNumbers_returnsPositiveSum() {
            MyClass myClass = new MyClass();
            int result = myClass.add(2, 3);
            assertEquals(5, result);
        }

        @Test
        public void add_negativeAndPositiveNumber_returnsCorrectSum() {
            MyClass myClass = new MyClass();
            int result = myClass.add(-2, 3);
            assertEquals(1, result);
        }
    }

    @Nested
    @Tag("subtraction")
    class SubtractionTests {

        @Test
        public void subtract_twoPositiveNumbers_returnsPositiveDifference() {
            MyClass myClass = new MyClass();
            int result = myClass.subtract(5, 3);
            assertEquals(2, result);
        }
    }
}

Conclusió

Organitzar proves en JUnit és essencial per mantenir una base de codi neta i eficient. Seguint les millors pràctiques, utilitzant una estructura de directoris consistent i aprofitant les anotacions i característiques de JUnit, pots crear un conjunt de proves ben organitzat que sigui fàcil d'entendre i mantenir. Recorda mantenir les teves proves petites, enfocades i modulars per assegurar-te que siguin efectives i fiables.
