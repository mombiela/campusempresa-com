En aquesta secció, explorarem com crear i utilitzar mocks en proves JUnit. Els mocks són essencials per aïllar la unitat de treball i assegurar que les proves siguin fiables i ràpides. Cobrirem els conceptes bàsics del mocking, com crear mocks utilitzant biblioteques populars com Mockito, i tècniques avançades per a escenaris més complexos.

Què són els Mocks?

Els mocks són objectes que simulen el comportament d'objectes reals de manera controlada. S'utilitzen en proves unitàries per aïllar el codi que s'està provant de les seves dependències.

  • Aïllament: Els mocks ajuden aïllar la unitat de treball de les dependències externes.
  • Control: Els mocks permeten controlar el comportament de les dependències.
  • Verificació: Els mocks poden verificar les interaccions entre la unitat de treball i les seves dependències.

Per què utilitzar Mocks?

  • Velocitat: Els mocks són més ràpids que els objectes reals, especialment quan es tracta de sistemes externs com bases de dades o serveis web.
  • Fiabilitat: Els mocks proporcionen un comportament consistent, fent que les proves siguin més fiables.
  • Focalització: Els mocks permeten centrar-se en la unitat de treball sense preocupar-se pel comportament de les seves dependències.

Creació de Mocks amb Mockito

Mockito és una biblioteca popular per crear mocks en Java. S'integra perfectament amb JUnit i proporciona una API senzilla per crear i utilitzar mocks.

Afegir Mockito al teu Projecte

Primer, necessites afegir Mockito al teu projecte. Si utilitzes Maven, afegeix la següent dependència al teu pom.xml:

<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>4.0.0</version>
    <scope>test</scope>
</dependency>

Creació Bàsica de Mocks

Aquí tens un exemple senzill de creació i ús d'un mock amb Mockito:

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        // Crear una instància mock de UserRepository
        UserRepository mockRepository = mock(UserRepository.class);

        // Definir el comportament del mock
        when(mockRepository.findUserById(1)).thenReturn(new User(1, "John Doe"));

        // Utilitzar el mock en el servei
        UserService userService = new UserService(mockRepository);
        User user = userService.getUser(1);

        // Verificar el resultat
        assertEquals("John Doe", user.getName());
    }
}

Verificació d'Interaccions

Mockito et permet verificar les interaccions amb el mock per assegurar-te que els mètodes esperats han estat cridats:

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        UserRepository mockRepository = mock(UserRepository.class);
        when(mockRepository.findUserById(1)).thenReturn(new User(1, "John Doe"));

        UserService userService = new UserService(mockRepository);
        userService.getUser(1);

        // Verificar que findUserById ha estat cridat amb l'argument correcte
        verify(mockRepository).findUserById(1);
    }
}

Tècniques Avançades de Mocking

Argument Matchers

Mockito proporciona argument matchers per permetre una verificació i estubatge més flexibles:

import static org.mockito.Mockito.*;
import static org.mockito.ArgumentMatchers.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        UserRepository mockRepository = mock(UserRepository.class);
        when(mockRepository.findUserById(anyInt())).thenReturn(new User(1, "John Doe"));

        UserService userService = new UserService(mockRepository);
        User user = userService.getUser(1);

        assertEquals("John Doe", user.getName());
    }
}

Captura d'Arguments

De vegades necessites capturar els arguments passats a un mètode mock per a fer més afirmacions:

import static org.mockito.Mockito.*;
import static org.mockito.ArgumentCaptor.*;
import org.junit.jupiter.api.Test;
import org.mockito.ArgumentCaptor;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        UserRepository mockRepository = mock(UserRepository.class);
        UserService userService = new UserService(mockRepository);

        userService.getUser(1);

        ArgumentCaptor<Integer> captor = ArgumentCaptor.forClass(Integer.class);
        verify(mockRepository).findUserById(captor.capture());

        assertEquals(1, captor.getValue().intValue());
    }
}

Resum

En aquesta secció, hem cobert els conceptes bàsics de creació i ús de mocks en proves JUnit amb Mockito. Els mocks són eines poderoses per aïllar la unitat de treball, controlar el comportament de les dependències i verificar les interaccions. En dominar aquestes tècniques, pots escriure proves unitàries més fiables i focalitzades.

  • Mocks: Simulen el comportament d'objectes reals.
  • Mockito: Una biblioteca popular per crear mocks.
  • Tècniques Bàsiques: Creació de mocks, definició de comportament i verificació d'interaccions.
  • Tècniques Avançades: Ús d'argument matchers i captura d'arguments.

Incorporant aquestes pràctiques a la teva estratègia de proves, pots millorar la qualitat i mantenibilitat del teu codi.
