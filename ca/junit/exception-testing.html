Introducció a la Prova d'Excepcions

La prova d'excepcions és un aspecte crucial de les proves unitàries que assegura que el teu codi gestiona les condicions d'error de manera elegant. A JUnit, pots provar si el teu codi llança les excepcions esperades sota condicions específiques. Això ajuda a verificar la robustesa i la fiabilitat de la teva aplicació.

Conceptes Clau

  • Gestió d'Excepcions: Mecanisme per gestionar errors en temps d'execució, assegurant el flux normal de l'aplicació.
  • Excepcions Esperades: Excepcions específiques que s'espera que el teu codi llanci sota certes condicions.
  • Anotacions de JUnit: Marcadors especials a JUnit per especificar les excepcions esperades.

Prova d'Excepcions amb JUnit 4

A JUnit 4, pots utilitzar l'anotació @Test amb el paràmetre expected per especificar l'excepció que esperes que una prova llanci.

Exemple

import org.junit.Test;
import static org.junit.Assert.*;

public class ExceptionTest {

    @Test(expected = IllegalArgumentException.class)
    public void testExceptionIsThrown() {
        MyClass myClass = new MyClass();
        myClass.methodThatThrowsException();
    }
}

En aquest exemple:

  • @Test(expected = IllegalArgumentException.class): Aquesta anotació diu a JUnit que esperi una IllegalArgumentException quan es cridi methodThatThrowsException.
  • Si l'excepció es llança, la prova passa. Si no, la prova falla.

Prova d'Excepcions amb JUnit 5

JUnit 5 introdueix el mètode assertThrows, que proporciona més flexibilitat i control sobre la prova d'excepcions.

Exemple

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertThrows;

public class ExceptionTest {

    @Test
    public void testExceptionIsThrown() {
        MyClass myClass = new MyClass();
        assertThrows(IllegalArgumentException.class, () -> {
            myClass.methodThatThrowsException();
        });
    }
}

En aquest exemple:

  • assertThrows(IllegalArgumentException.class, () -> { ... }): Aquest mètode assegura que l'expressió lambda proporcionada llança una IllegalArgumentException.
  • Et permet capturar i inspeccionar l'excepció si cal.

Prova d'Excepcions Avançada

Per a escenaris més avançats, potser necessitis verificar el missatge de l'excepció o altres propietats.

Exemple

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class AdvancedExceptionTest {

    @Test
    public void testExceptionMessage() {
        MyClass myClass = new MyClass();
        Exception exception = assertThrows(IllegalArgumentException.class, () -> {
            myClass.methodThatThrowsException();
        });

        String expectedMessage = "Invalid argument";
        String actualMessage = exception.getMessage();

        assertTrue(actualMessage.contains(expectedMessage));
    }
}

En aquest exemple:

  • El mètode assertThrows captura l'excepció.
  • La prova després verifica que el missatge de l'excepció conté el text esperat.

Resum

La prova d'excepcions és una part essencial de les proves unitàries que assegura que la teva aplicació gestiona els errors com s'espera. JUnit proporciona diferents maneres de provar excepcions:

  • JUnit 4 utilitza l'anotació @Test(expected = Exception.class).
  • JUnit 5 utilitza el mètode més flexible assertThrows.

Dominant la prova d'excepcions, pots escriure codi més robust i fiable, assegurant que la teva aplicació es comporti correctament sota condicions d'error.
