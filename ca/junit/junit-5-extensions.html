Introducció a les Extensions de JUnit 5

Les Extensions de JUnit 5 et permeten ampliar la funcionalitat de JUnit 5 creant comportaments personalitzats que es poden reutilitzar en diverses classes de prova. Aquesta característica és potent per crear lògica de prova reutilitzable, anotacions personalitzades i més.

Conceptes Clau

  • Punts d'Extensió: JUnit 5 proporciona diversos punts d'extensió que et permeten connectar-te al cicle de vida de la prova.
  • Model d'Extensió: Les extensions s'implementen creant classes que implementen una o més de les interfícies d'extensió proporcionades per JUnit 5.
  • Registre: Les extensions es poden registrar de manera declarativa mitjançant anotacions o programàticament.

Punts d'Extensió Comuns

  • Post-processament de la Instància de Prova: Modificar instàncies de prova després de ser creades.
  • Condició d'Execució de la Prova: Controlar si les proves s'han d'executar o saltar.
  • Resolució de Paràmetres: Proporcionar paràmetres personalitzats als mètodes de prova.
  • Callbacks de Cicle de Vida: Connectar-se a diverses etapes del cicle de vida de la prova (per exemple, abans de totes les proves, abans de cada prova).

Creació d'una Extensió Simple

Creem una extensió simple que registri missatges abans i després de cada mètode de prova.

Exemple Pas a Pas

  1. Definir la Classe d'Extensió:

    import org.junit.jupiter.api.extension.BeforeEachCallback;
import org.junit.jupiter.api.extension.AfterEachCallback;
import org.junit.jupiter.api.extension.ExtensionContext;

public class LoggingExtension implements BeforeEachCallback, AfterEachCallback {
    @Override
    public void beforeEach(ExtensionContext context) {
        System.out.println("Before test method: " + context.getDisplayName());
    }

    @Override
    public void afterEach(ExtensionContext context) {
        System.out.println("After test method: " + context.getDisplayName());
    }
}

  2. Registrar l'Extensió:

    • Declarativament: Utilitza l'anotació @ExtendWith.

      import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;

@ExtendWith(LoggingExtension.class)
public class MyTests {
    @Test
    void testExample() {
        System.out.println("Executing testExample");
    }
}

    • Programàticament: Utilitza l'anotació @RegisterExtension.

      import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.RegisterExtension;

public class MyTests {
    @RegisterExtension
    static LoggingExtension loggingExtension = new LoggingExtension();

    @Test
    void testExample() {
        System.out.println("Executing testExample");
    }
}

Explicació

  • Classe LoggingExtension: Implementa BeforeEachCallback i AfterEachCallback per registrar missatges abans i després de cada mètode de prova.
  • Anotació @ExtendWith: Registra l'extensió declarativament a nivell de classe.
  • Anotació @RegisterExtension: Registra l'extensió programàticament a nivell de camp.

Extensions Avançades

Execució Condicional de Proves

Pots crear extensions per executar condicionalment proves basades en lògica personalitzada.

import org.junit.jupiter.api.extension.ExecutionCondition;
import org.junit.jupiter.api.extension.ExtensionContext;
import org.junit.jupiter.api.extension.ConditionEvaluationResult;

public class CustomCondition implements ExecutionCondition {
    @Override
    public ConditionEvaluationResult evaluateExecutionCondition(ExtensionContext context) {
        boolean condition = // your custom condition logic
        return condition ? ConditionEvaluationResult.enabled("Condition met") :
                           ConditionEvaluationResult.disabled("Condition not met");
    }
}

Resolució de Paràmetres

Pots crear extensions per proporcionar paràmetres personalitzats als mètodes de prova.

import org.junit.jupiter.api.extension.ParameterContext;
import org.junit.jupiter.api.extension.ParameterResolver;
import org.junit.jupiter.api.extension.ExtensionContext;

public class CustomParameterResolver implements ParameterResolver {
    @Override
    public boolean supportsParameter(ParameterContext parameterContext, ExtensionContext extensionContext) {
        return parameterContext.getParameter().getType() == String.class;
    }

    @Override
    public Object resolveParameter(ParameterContext parameterContext, ExtensionContext extensionContext) {
        return "Custom Parameter";
    }
}

Resum

Les Extensions de JUnit 5 proporcionen una manera potent d'ampliar la funcionalitat de les teves proves. En implementar diversos punts d'extensió, pots crear lògica de prova reutilitzable i modular. Les extensions es poden registrar de manera declarativa o programàtica, donant-te flexibilitat en com les apliques. Tant si necessites registrar missatges, executar proves condicionalment o proporcionar paràmetres personalitzats, les Extensions de JUnit 5 ofereixen una solució robusta per millorar el teu marc de proves.
