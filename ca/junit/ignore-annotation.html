JUnit és un marc de proves popular per a Java, i entendre les seves anotacions és crucial per escriure proves efectives. Una d'aquestes anotacions és @Ignore, que s'utilitza per saltar l'execució d'un mètode de prova o d'una classe de prova sencera.

Què és l'anotació @Ignore?

  • L'anotació @Ignore s'utilitza per indicar que un mètode de prova o una classe no s'ha d'executar.
  • Es pot aplicar tant a mètodes de prova individuals com a classes de prova senceres.
  • Això és útil quan una prova no està llesta per ser executada o si està temporalment trencada i no vols que afecti els resultats generals de les proves.

Com utilitzar @Ignore

Ignorar un mètode de prova individual

Per ignorar un mètode de prova individual, simplement col·loca l'anotació @Ignore sobre el mètode. Aquí tens un exemple:

import org.junit.Ignore;
import org.junit.Test;

public class ExampleTest {

    @Test
    public void testMethod1() {
        // Aquesta prova s'executarà
        System.out.println("testMethod1 executat");
    }

    @Ignore
    @Test
    public void testMethod2() {
        // Aquesta prova serà ignorada
        System.out.println("testMethod2 executat");
    }
}

En aquest exemple:

  • testMethod1 s'executarà.
  • testMethod2 serà ignorat i no s'executarà.

Ignorar una classe de prova sencera

Per ignorar totes les proves dins d'una classe, col·loca l'anotació @Ignore sobre la declaració de la classe:

import org.junit.Ignore;
import org.junit.Test;

@Ignore
public class IgnoredClassTest {

    @Test
    public void testMethod1() {
        // Aquesta prova serà ignorada
        System.out.println("testMethod1 executat");
    }

    @Test
    public void testMethod2() {
        // Aquesta prova també serà ignorada
        System.out.println("testMethod2 executat");
    }
}

En aquest exemple:

  • Tant testMethod1 com testMethod2 seran ignorats perquè tota la classe està anotada amb @Ignore.

Utilitzar @Ignore amb una raó

Pots proporcionar una raó per ignorar una prova passant una cadena a l'anotació @Ignore. Això pot ser útil per a fins de documentació:

import org.junit.Ignore;
import org.junit.Test;

public class ExampleTest {

    @Ignore("Aquesta prova s'ignora perquè encara no està implementada")
    @Test
    public void testMethod1() {
        // Aquesta prova serà ignorada amb una raó
        System.out.println("testMethod1 executat");
    }
}

En aquest exemple:

  • testMethod1 serà ignorat, i la raó "Aquesta prova s'ignora perquè encara no està implementada" serà documentada.

Resum

  • L'anotació @Ignore a JUnit s'utilitza per saltar l'execució de mètodes de prova o classes de prova senceres.
  • Es pot aplicar a mètodes de prova individuals o a classes senceres.
  • Proporcionar una raó per ignorar una prova pot ser útil per a la documentació i referència futura.

Utilitzant l'anotació @Ignore de manera efectiva, pots gestionar la teva suite de proves de manera més eficient, assegurant que les proves incompletes o trencades no interfereixin amb el teu procés de proves.
