El Projecte Sobre nosaltres Contribuir Donacions Llicència

La Integració Contínua (CI) és una pràctica de desenvolupament on els desenvolupadors integren codi en un repositori compartit freqüentment, idealment diverses vegades al dia. Cada integració pot ser verificada per una compilació automatitzada i proves automatitzades, permetent als equips detectar problemes aviat.

En aquesta secció, explorarem com integrar proves de JUnit en una pipeline de CI.

Conceptes Clau de la Integració Contínua

  • Compilacions Automatitzades: Compilar i construir automàticament el projecte sempre que es facin canvis.
  • Proves Automatitzades: Executar proves automàticament per assegurar que els nous canvis de codi no trenquin la funcionalitat existent.
  • Sistemes de Control de Versions (VCS): Eines com Git per gestionar els canvis de codi.
  • Servidors de CI: Eines com Jenkins, Travis CI, o GitHub Actions per automatitzar el procés de compilació i prova.

Configurant una Pipeline de CI amb JUnit

Pas 1: Sistema de Control de Versions

Assegura't que el teu projecte està sota control de versions. Per a aquest exemple, utilitzarem Git.

  1. Inicialitza un repositori de Git:

    git init

  2. Afegeix els fitxers del teu projecte i fes un commit:

    git add .
git commit -m "Commit inicial"

Pas 2: Escriure Proves de JUnit

Assegura't que tens proves de JUnit en el teu projecte. Aquí tens un exemple senzill:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        assertEquals(5, calculator.add(2, 3));
    }
}

Pas 3: Triar un Servidor de CI

Hi ha diversos servidors de CI disponibles. Utilitzarem GitHub Actions per a aquest exemple.

Pas 4: Configurar GitHub Actions

  1. Crea un directori .github/workflows en el teu projecte.

  2. Crea un nou fitxer de workflow, per exemple, ci.yml:

    name: Java CI with Maven

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:

    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up JDK 11
      uses: actions/setup-java@v2
      with:
        java-version: '11'

    - name: Build with Maven
      run: mvn clean install

    - name: Run tests
      run: mvn test

Pas 5: Pujar Canvis per a Desencadenar la CI

  1. Afegeix el fitxer de workflow a Git:

    git add .github/workflows/ci.yml
git commit -m "Afegeix workflow de CI"

  2. Puja els canvis al repositori:

    git push origin main

GitHub Actions ara executarà automàticament la pipeline de CI sempre que es pugin canvis a la branca main o es faci una pull request.

Conclusió

Integrar proves de JUnit en una pipeline de CI assegura que el teu codi es prova contínuament, proporcionant retroalimentació immediata sobre qualsevol problema introduït per nous canvis. Aquesta pràctica ajuda a mantenir la qualitat del codi i redueix el risc d'errors en producció.

Seguint els passos descrits anteriorment, pots configurar una pipeline de CI utilitzant GitHub Actions per construir i provar automàticament el teu projecte Java amb JUnit. Aquesta configuració es pot adaptar a altres servidors de CI com Jenkins o Travis CI amb modificacions menors.
