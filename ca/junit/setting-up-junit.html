Introducció

JUnit és un popular framework de proves per a Java que permet als desenvolupadors escriure i executar proves repetibles. Aquesta secció et guiarà a través del procés de configuració de JUnit en el teu entorn de desenvolupament, ja sigui que estiguis utilitzant un Entorn de Desenvolupament Integrat (IDE) o una eina d'automatització de construcció.

Prerequisits

Abans de configurar JUnit, assegura't de tenir el següent:

  • Kit de Desenvolupament de Java (JDK) instal·lat.
  • Un IDE com IntelliJ IDEA, Eclipse, o NetBeans, o una eina de construcció com Maven o Gradle.

Configuració de JUnit en un IDE

IntelliJ IDEA

  1. Crear un Nou Projecte:

    • Obre IntelliJ IDEA.
    • Selecciona "New Project" i tria "Java".
    • Segueix les indicacions per crear el teu projecte.

  2. Afegir JUnit al Projecte:

    • Fes clic dret sobre la carpeta del projecte en el Project Explorer.
    • Selecciona "Add Framework Support".
    • Tria "JUnit" i fes clic a "OK".

  3. Verificar la Dependència de JUnit:

    • Obre el fitxer pom.xml (si utilitzes Maven) o el fitxer build.gradle (si utilitzes Gradle) per assegurar-te que JUnit està llistat com a dependència.

Eclipse

  1. Crear un Nou Projecte Java:

    • Obre Eclipse.
    • Ves a File > New > Java Project.
    • Segueix les indicacions per crear el teu projecte.

  2. Afegir la Biblioteca JUnit:

    • Fes clic dret sobre la carpeta del projecte en el Project Explorer.
    • Selecciona Build Path > Add Libraries.
    • Tria "JUnit" i fes clic a "Next".
    • Selecciona la versió de JUnit (es recomana JUnit 5) i fes clic a "Finish".

  3. Verificar la Dependència de JUnit:

    • Assegura't que la biblioteca JUnit està llistada sota la secció Referenced Libraries en el Project Explorer.

Configuració de JUnit amb Eines de Construcció

Maven

  1. Crear un Projecte Maven:

    • Obre el teu terminal o símbol del sistema.
    • Executa la següent comanda per crear un nou projecte Maven: 
      mvn archetype:generate -DgroupId=com.example -DartifactId=my-app -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false

  2. Afegir la Dependència de JUnit:

    • Obre el fitxer pom.xml en el directori del teu projecte.
    • Afegeix la següent dependència dins de l'etiqueta <dependencies>: 
      <dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-engine</artifactId>
    <version>5.8.1</version>
    <scope>test</scope>
</dependency>

  3. Verificar la Dependència:

    • Executa mvn clean install per assegurar-te que les dependències es descarreguen i instal·len correctament.

Gradle

  1. Crear un Projecte Gradle:

    • Obre el teu terminal o símbol del sistema.
    • Executa la següent comanda per crear un nou projecte Gradle: 
      gradle init --type java-application

  2. Afegir la Dependència de JUnit:

    • Obre el fitxer build.gradle en el directori del teu projecte.
    • Afegeix la següent dependència dins del bloc dependencies: 
      testImplementation 'org.junit.jupiter:junit-jupiter-engine:5.8.1'

  3. Verificar la Dependència:

    • Executa gradle build per assegurar-te que les dependències es descarreguen i instal·len correctament.

Escriure la Teva Primera Prova

Un cop JUnit està configurat, pots escriure la teva primera prova. Aquí tens un exemple senzill:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 should equal 5");
    }
}

Explicació

  • Importar Declaracions: Importa les classes necessàries de JUnit.
  • Anotació de Prova: Utilitza @Test per indicar que el mètode és un mètode de prova.
  • Afirmacions: Utilitza assertEquals per comprovar si el resultat esperat coincideix amb el resultat real.

Conclusió

Configurar JUnit és un procés senzill, ja sigui que estiguis utilitzant un IDE o una eina de construcció com Maven o Gradle. Un cop configurat, pots començar a escriure i executar proves per assegurar-te que el teu codi funciona com s'espera. Aquest coneixement fonamental serà crucial a mesura que avancis a temes més avançats de JUnit.
