Introducció als Supòsits

En JUnit, els supòsits s'utilitzen per desactivar condicionalment les proves basades en certs criteris. Si un supòsit falla, la prova es salta en lloc de marcar-se com a fallida. Això és particularment útil per a proves que només s'han d'executar sota condicions específiques, com ara quan es configuren certes propietats del sistema o quan s'executa en un sistema operatiu particular.

Conceptes Clau

  • Supòsits: Condicions que s'han de complir perquè una prova s'executi.
  • Classe Assume: JUnit proporciona la classe Assume per fer supòsits en les proves.
  • Execució Condicional de Proves: Les proves es salten si els supòsits no es compleixen.

Ús dels Supòsits en JUnit

JUnit proporciona diversos mètodes estàtics en la classe Assume per fer supòsits. Els mètodes més utilitzats són:

  • assumeTrue(boolean condition): Salta la prova si la condició és falsa.
  • assumeFalse(boolean condition): Salta la prova si la condició és certa.
  • assumeThat(T actual, Matcher<T> matcher): Salta la prova si el valor actual no coincideix amb el matcher.

Exemple: Supòsits Bàsics

Vegem un exemple senzill on utilitzem assumeTrue per saltar una prova si no es compleix una certa condició.

import static org.junit.jupiter.api.Assumptions.assumeTrue;
import org.junit.jupiter.api.Test;

public class AssumptionTest {

    @Test
    void testOnlyOnCiServer() {
        assumeTrue("CI".equals(System.getenv("ENV")));
        // Lògica de prova que només s'ha d'executar en un servidor CI
        System.out.println("Aquesta prova només s'executa en un servidor CI.");
    }
}

En aquest exemple:

  • La prova testOnlyOnCiServer només s'executarà si la variable d'entorn ENV està configurada a CI.
  • Si el supòsit falla, la prova es salta.

Exemple: Ús de assumeThat

El mètode assumeThat és útil quan vols fer supòsits més complexos utilitzant matchers.

import static org.hamcrest.CoreMatchers.is;
import static org.junit.jupiter.api.Assumptions.assumeThat;
import org.junit.jupiter.api.Test;

public class AssumptionTest {

    @Test
    void testOnlyOnSpecificJavaVersion() {
        assumeThat(System.getProperty("java.version"), is("11"));
        // Lògica de prova que només s'ha d'executar en Java 11
        System.out.println("Aquesta prova només s'executa en Java 11.");
    }
}

En aquest exemple:

  • La prova testOnlyOnSpecificJavaVersion només s'executarà si la versió de Java és 11.
  • Si el supòsit falla, la prova es salta.

Casos Pràctics

  • Proves Específiques d'Entorn: Executar proves només en certs entorns (p. ex., servidors CI, sistema operatiu específic).
  • Proves Específiques de Configuració: Executar proves només quan es configuren certes configuracions o propietats.
  • Proves Dependents de Recursos: Saltar proves si els recursos necessaris (p. ex., bases de dades, serveis externs) no estan disponibles.

Conclusió

Els supòsits en JUnit proporcionen una manera poderosa de saltar condicionalment les proves basades en criteris específics. Utilitzant la classe Assume, pots assegurar-te que les proves només s'executin quan es compleixin les condicions necessàries, fent que el teu conjunt de proves sigui més robust i adaptable a diferents entorns i configuracions. Entendre i utilitzar efectivament els supòsits pot ajudar-te a escriure proves més flexibles i mantenibles.
