Introducció a les Assercions

Les assercions són una part fonamental de les proves en JUnit. Et permeten verificar que el teu codi es comporta com s'espera. Si una asserció falla, significa que hi ha una discrepància entre els resultats esperats i els reals, indicant un possible error en el codi.

Conceptes Clau de les Assercions

  • Assercions: Declaracions que comproven si una condició és certa. Si la condició és falsa, la prova falla.
  • Esperat vs. Real: Les assercions normalment comparen un valor esperat amb un valor real.
  • Assercions Comunes: JUnit proporciona una varietat d'asserccions per manejar diferents tipus de comprovacions.

Assercions Comunes en JUnit

Aquí hi ha algunes de les assercions més utilitzades en JUnit:

| Mètode d'Asserció | Descripció | |-------------------------|------------------------------------------------------------| | assertEquals | Comprova si dos valors són iguals | | assertNotEquals | Comprova si dos valors no són iguals | | assertTrue | Comprova si una condició és certa | | assertFalse | Comprova si una condició és falsa | | assertNull | Comprova si un objecte és nul | | assertNotNull | Comprova si un objecte no és nul | | assertArrayEquals | Comprova si dos arrays són iguals | | assertSame | Comprova si dues referències apunten al mateix objecte | | assertNotSame | Comprova si dues referències no apunten al mateix objecte | | assertThrows | Comprova si es llança una excepció específica |

Ús Bàsic de les Assercions

Exemple: Ús de `assertEquals`

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 hauria de ser igual a 5");
    }
}

En aquest exemple, assertEquals comprova si el resultat de calculator.add(2, 3) és igual a 5. Si no ho és, la prova fallarà amb el missatge "2 + 3 hauria de ser igual a 5".

Exemple: Ús de `assertTrue` i `assertFalse`

import static org.junit.jupiter.api.Assertions.assertTrue;
import static org.junit.jupiter.api.Assertions.assertFalse;
import org.junit.jupiter.api.Test;

public class NumberTest {

    @Test
    public void testIsEven() {
        NumberUtils numberUtils = new NumberUtils();
        assertTrue(numberUtils.isEven(4), "4 hauria de ser parell");
        assertFalse(numberUtils.isEven(5), "5 no hauria de ser parell");
    }
}

Aquí, assertTrue comprova si numberUtils.isEven(4) retorna cert, i assertFalse comprova si numberUtils.isEven(5) retorna fals.

Ús Avançat de les Assercions

Exemple: Ús de `assertThrows`

import static org.junit.jupiter.api.Assertions.assertThrows;
import org.junit.jupiter.api.Test;

public class ExceptionTest {

    @Test
    public void testException() {
        ExceptionThrower thrower = new ExceptionThrower();
        assertThrows(IllegalArgumentException.class, () -> {
            thrower.throwException();
        }, "S'esperava que es llancés IllegalArgumentException");
    }
}

En aquest exemple, assertThrows comprova si thrower.throwException() llança una IllegalArgumentException. Si l'excepció no es llança, la prova falla.

Exemple: Ús de `assertArrayEquals`

import static org.junit.jupiter.api.Assertions.assertArrayEquals;
import org.junit.jupiter.api.Test;

public class ArrayTest {

    @Test
    public void testArrayEquality() {
        int[] expected = {1, 2, 3};
        int[] actual = {1, 2, 3};
        assertArrayEquals(expected, actual, "Els arrays haurien de ser iguals");
    }
}

assertArrayEquals comprova si els arrays expected i actual són iguals. Si no ho són, la prova falla.

Conclusió

Les assercions són una eina poderosa en JUnit per verificar que el teu codi es comporta com s'espera. Utilitzant assercions, pots detectar errors a l'inici del procés de desenvolupament. Entendre i utilitzar efectivament les assercions és crucial per escriure proves robustes i fiables.
