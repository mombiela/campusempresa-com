Introducció

Mockito és un popular framework de simulació per a Java que s'utilitza conjuntament amb JUnit per crear proves unitàries. Permet als desenvolupadors crear objectes simulats i definir el seu comportament, facilitant l'aïllament del codi que s'està provant. Aquesta secció cobrirà els conceptes bàsics d'utilitzar Mockito amb JUnit, incloent la configuració, la creació de simulacions i la verificació d'interaccions.

Configurant Mockito amb JUnit

Per utilitzar Mockito amb JUnit, cal afegir la biblioteca Mockito al teu projecte. Això es pot fer utilitzant una eina de construcció com Maven o Gradle.

Dependència Maven

<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>4.0.0</version>
    <scope>test</scope>
</dependency>

Dependència Gradle

testImplementation 'org.mockito:mockito-core:4.0.0'

Creant Simulacions

Les simulacions són objectes que simulen el comportament d'objectes reals. A Mockito, pots crear simulacions utilitzant l'anotació @Mock o el mètode Mockito.mock().

Utilitzant l'Anotació `@Mock`

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
import org.mockito.Mock;
import org.mockito.MockitoAnnotations;

public class ExampleTest {

    @Mock
    private MyService myService;

    @BeforeEach
    public void setUp() {
        MockitoAnnotations.openMocks(this);
    }

    @Test
    public void testMyService() {
        // Defineix el comportament i prova
    }
}

Utilitzant el Mètode `Mockito.mock()`

import org.junit.jupiter.api.Test;
import static org.mockito.Mockito.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        // Defineix el comportament i prova
    }
}

Definint el Comportament

Un cop hagis creat una simulació, pots definir el seu comportament utilitzant els mètodes when i thenReturn.

Exemple

import static org.mockito.Mockito.*;
import static org.junit.jupiter.api.Assertions.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        when(myService.performAction()).thenReturn("Success");

        String result = myService.performAction();
        assertEquals("Success", result);
    }
}

Verificant Interaccions

Mockito et permet verificar que certs mètodes han estat cridats en els teus objectes simulats. Això és útil per assegurar-te que el teu codi interactua amb les dependències tal com s'espera.

Exemple

import static org.mockito.Mockito.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        myService.performAction();

        verify(myService).performAction();
    }
}

Funcions Avançades

Mockito proporciona diverses funcions avançades que poden ser útils en escenaris de prova més complexos.

Coincidències d'Arguments

Les coincidències d'arguments et permeten especificar criteris flexibles per als arguments dels mètodes.

import static org.mockito.Mockito.*;
import static org.mockito.ArgumentMatchers.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        when(myService.performAction(anyString())).thenReturn("Success");

        String result = myService.performAction("Test");
        assertEquals("Success", result);
    }
}

Simulant Mètodes Void

Pots simular mètodes void per llençar excepcions o realitzar altres accions.

import static org.mockito.Mockito.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        doThrow(new RuntimeException()).when(myService).performVoidAction();

        assertThrows(RuntimeException.class, () -> {
            myService.performVoidAction();
        });
    }
}

Conclusió

Utilitzar Mockito amb JUnit et permet crear proves unitàries robustes aïllant el codi sota prova i simulant el comportament de les seves dependències. Comprenent com crear simulacions, definir el seu comportament i verificar interaccions, pots escriure proves més efectives i fiables. A mesura que et familiaritzis amb aquests conceptes bàsics, pots explorar funcions avançades com les coincidències d'arguments i la simulació de mètodes void per gestionar escenaris de prova més complexos.
