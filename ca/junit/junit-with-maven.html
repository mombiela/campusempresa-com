Introducció a JUnit i Maven

JUnit és un popular framework de proves per a aplicacions Java, mentre que Maven és una eina d'automatització de construcció utilitzada principalment per a projectes Java. Combinar JUnit amb Maven permet una gestió eficient de les dependències i proves automatitzades.

Configuració de Maven per a JUnit

Per començar a utilitzar JUnit amb Maven, cal configurar un projecte Maven i incloure JUnit com a dependència.

Crear un Projecte Maven

  1. Utilitzant la Línia de Comandes:

    mvn archetype:generate -DgroupId=com.example -DartifactId=JUnitWithMaven -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false

    Aquesta comanda crea un nou projecte Maven amb el group ID i artifact ID especificats.

  2. Utilitzant un IDE (p. ex., IntelliJ IDEA, Eclipse):

    • Obre el teu IDE i selecciona "Crear Nou Projecte".
    • Tria "Maven" com a tipus de projecte.
    • Omple els detalls necessaris (Group ID, Artifact ID, etc.) i finalitza la configuració.

Afegir la Dependència de JUnit

Obre el fitxer pom.xml en el teu projecte Maven i afegeix la dependència de JUnit:

<dependencies>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter-engine</artifactId>
        <version>5.8.1</version>
        <scope>test</scope>
    </dependency>
</dependencies>

Aquesta configuració afegeix JUnit 5 al teu projecte, que és la versió més recent de JUnit.

Escriure la Teva Primera Prova

Crea una classe Java simple i escriu una prova per a ella utilitzant JUnit.

Exemple de Classe Java

package com.example;

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

Exemple de Prova JUnit

Crea una classe de prova en el directori src/test/java:

package com.example;

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {
    @Test
    public void testAdd() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 hauria de ser igual a 5");
    }
}

Executar Proves amb Maven

Per executar les teves proves, utilitza la següent comanda Maven:

mvn test

Aquesta comanda compila el teu codi i executa totes les proves en el directori src/test/java.

Funcions Avançades de JUnit amb Maven

Explora funcions més avançades de JUnit i com gestionar-les amb Maven.

Proves Parametritzades

Les proves parametritzades permeten executar la mateixa prova amb diferents paràmetres.

Exemple

package com.example;

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

public class CalculatorTest {
    @ParameterizedTest
    @CsvSource({
        "1, 1, 2",
        "2, 3, 5",
        "3, 7, 10"
    })
    public void testAdd(int a, int b, int expected) {
        Calculator calculator = new Calculator();
        assertEquals(expected, calculator.add(a, b));
    }
}

Suites de Proves

Les suites de proves permeten agrupar múltiples classes de prova i executar-les juntes.

Exemple

package com.example;

import org.junit.platform.suite.api.SelectClasses;
import org.junit.platform.suite.api.Suite;

@Suite
@SelectClasses({CalculatorTest.class, AnotherTest.class})
public class AllTests {
}

Integració Contínua amb Maven

Integra el teu projecte Maven amb una eina de CI com Jenkins per automatitzar el procés de proves.

Configuració de Jenkins

  1. Instal·lar Jenkins: Segueix la guia oficial d'instal·lació de Jenkins.
  2. Crear una Nova Tasca:
    • Selecciona "Nou Element" i tria "Projecte Freestyle".
    • Configura la gestió del codi font (p. ex., Git).
    • Afegeix un pas de construcció per executar els objectius de Maven (clean test).

Conclusió

Combinar JUnit amb Maven agilitza el procés d'escriure, gestionar i executar proves en projectes Java. Configurant un projecte Maven, afegint dependències de JUnit i utilitzant funcions avançades de JUnit, pots assegurar-te que el teu codi és robust i fiable. Integrar amb eines de CI com Jenkins millora encara més el teu flux de treball de desenvolupament automatitzant el procés de proves.
