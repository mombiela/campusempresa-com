Introducció a les Excepcions

En Python, les excepcions són errors que es produeixen durant l'execució d'un programa. Quan es produeix un error, Python atura el programa i genera un missatge d'error. Això es coneix com una excepció.

Conceptes Clau

  • Excepció: Un error que es produeix durant l'execució d'un programa.
  • Gestió: El procés de respondre a les excepcions d'una manera controlada.
  • Bloc Try: El codi que podria generar una excepció es col·loca dins d'aquest bloc.
  • Bloc Except: El codi que gestiona l'excepció es col·loca dins d'aquest bloc.
  • Bloc Finally: El codi que sempre s'executarà, independentment de si s'ha produït una excepció o no.

Gestió Bàsica d'Excepcions

Per gestionar excepcions, utilitzes els blocs try i except.

try:
    # Codi que pot causar una excepció
    result = 10 / 0
except ZeroDivisionError:
    # Codi que s'executa si es produeix una excepció
    print("No pots dividir per zero!")

En aquest exemple:

  • El codi dins del bloc try intenta dividir per zero, cosa que genera una ZeroDivisionError.
  • El bloc except captura l'excepció i imprimeix un missatge.

Gestió de Múltiples Excepcions

Pots gestionar múltiples excepcions especificant múltiples blocs except.

try:
    value = int(input("Introdueix un número: "))
    result = 10 / value
except ValueError:
    print("Entrada no vàlida! Si us plau, introdueix un número vàlid.")
except ZeroDivisionError:
    print("No pots dividir per zero!")

En aquest exemple:

  • Si l'usuari introdueix un valor no enter, es genera una ValueError.
  • Si l'usuari introdueix zero, es genera una ZeroDivisionError.
  • Cada excepció es gestiona pel seu respectiu bloc except.

Ús del Bloc Finally

El bloc finally s'utilitza per executar codi independentment de si es produeix una excepció o no.

try:
    file = open("example.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("Fitxer no trobat!")
finally:
    file.close()
    print("Fitxer tancat.")

En aquest exemple:

  • El bloc finally assegura que el fitxer es tanqui tant si es produeix una excepció com si no.

Generació d'Excepcions

Pots generar excepcions utilitzant la paraula clau raise.

def check_age(age):
    if age < 0:
        raise ValueError("L'edat no pot ser negativa!")
    return age

try:
    age = check_age(-1)
except ValueError as e:
    print(e)

En aquest exemple:

  • La funció check_age genera una ValueError si l'edat és negativa.
  • El bloc except captura l'excepció i imprimeix el missatge d'error.

Excepcions Personalitzades

Pots crear excepcions personalitzades definint una nova classe que hereti de la classe Exception.

class CustomError(Exception):
    pass

try:
    raise CustomError("Aquest és un error personalitzat!")
except CustomError as e:
    print(e)

En aquest exemple:

  • Es defineix una excepció personalitzada CustomError.
  • L'excepció personalitzada es genera i es captura en el bloc except.

Conclusió

La gestió d'excepcions és una part crucial de l'escriptura de programes robustos en Python. Utilitzant els blocs try, except i finally, pots gestionar els errors de manera elegant i assegurar-te que el teu programa continuï funcionant sense problemes. A més, generar i crear excepcions personalitzades permet una gestió d'errors més precisa adaptada a les teves necessitats específiques.
