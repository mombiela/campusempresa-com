El Projecte Sobre nosaltres Contribuir Donacions Llicència

El polimorfisme és un concepte central en la programació orientada a objectes (OOP) que permet que objectes de diferents classes siguin tractats com a objectes d'una superclasse comuna. És una característica poderosa que permet flexibilitat i integració en el codi.

Què és el Polimorfisme?

  • Definició: El polimorfisme significa "moltes formes" i permet que els mètodes facin coses diferents segons l'objecte sobre el qual estan actuant.
  • Propòsit: Ajuda a dissenyar sistemes que són més modulars i més fàcils d'ampliar.

Tipus de Polimorfisme

  1. Polimorfisme en temps de compilació (Sobrecàrrega de mètodes):

    • No està directament suportat en Python.
    • S'aconsegueix utilitzant arguments per defecte o arguments de longitud variable.

  2. Polimorfisme en temps d'execució (Sobreescriptura de mètodes):

    • Suportat en Python.
    • Permet que una subclasse proporcioni una implementació específica d'un mètode que ja està definit en la seva superclasse.

Sobrecàrrega de mètodes en Python

Python no suporta la sobrecàrrega de mètodes tradicional com es veu en altres llenguatges com Java o C++. No obstant això, es pot aconseguir una funcionalitat similar utilitzant arguments per defecte o arguments de longitud variable.

Exemple de Sobrecàrrega de mètodes Utilitzant Arguments per Defecte

class MathOperations:
    def add(self, a, b=0, c=0):
        return a + b + c

math_op = MathOperations()
print(math_op.add(1))       # Output: 1
print(math_op.add(1, 2))    # Output: 3
print(math_op.add(1, 2, 3)) # Output: 6

Exemple de Sobrecàrrega de mètodes Utilitzant Arguments de Longitud Variable

class MathOperations:
    def add(self, *args):
        return sum(args)

math_op = MathOperations()
print(math_op.add(1))       # Output: 1
print(math_op.add(1, 2))    # Output: 3
print(math_op.add(1, 2, 3)) # Output: 6

Sobreescriptura de mètodes en Python

La sobreescriptura de mètodes permet que una subclasse proporcioni una implementació específica d'un mètode que ja està definit en la seva superclasse.

Exemple de Sobreescriptura de mètodes

class Animal:
    def sound(self):
        return "Some generic sound"

class Dog(Animal):
    def sound(self):
        return "Bark"

class Cat(Animal):
    def sound(self):
        return "Meow"

animals = [Dog(), Cat(), Animal()]

for animal in animals:
    print(animal.sound())

Output:

Bark
Meow
Some generic sound

Polimorfisme amb Herència

El polimorfisme sovint s'utilitza amb herència per permetre que una sola interfície s'utilitzi per a una classe general d'accions.

Exemple de Polimorfisme amb Herència

class Shape:
    def area(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius * self.radius

shapes = [Rectangle(2, 3), Circle(5)]

for shape in shapes:
    print(shape.area())

Output:

6
78.5

Polimorfisme amb Funcions i Objectes

El polimorfisme també es pot aconseguir utilitzant funcions que poden prendre objectes de diferents classes i realitzar accions basades en el seu tipus de classe.

Exemple de Polimorfisme amb Funcions

class Bird:
    def fly(self):
        return "Flying high"

class Airplane:
    def fly(self):
        return "Flying at 30,000 feet"

def let_it_fly(flying_object):
    print(flying_object.fly())

bird = Bird()
airplane = Airplane()

let_it_fly(bird)       # Output: Flying high
let_it_fly(airplane)   # Output: Flying at 30,000 feet

Resum

El polimorfisme és un concepte fonamental en OOP que permet flexibilitat i integració en el codi. Es pot aconseguir en Python a través de la sobrecàrrega de mètodes (utilitzant arguments per defecte o arguments de longitud variable) i la sobreescriptura de mètodes. Utilitzant el polimorfisme, es poden dissenyar sistemes que són més modulars i més fàcils d'ampliar, fent que el codi sigui més mantenible i escalable.
