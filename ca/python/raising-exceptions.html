Introducció

En Python, les excepcions són una manera de gestionar errors que es produeixen durant l'execució d'un programa. Llençar excepcions et permet senyalar que s'ha produït un error i proporcionar informació sobre la naturalesa de l'error. Aquesta secció cobrirà els conceptes bàsics de llençar excepcions, com crear excepcions personalitzades i tècniques avançades per gestionar excepcions.

Conceptes Clau

  • Excepció: Un error que es produeix durant l'execució d'un programa.
  • Llençar una Excepció: L'acte de senyalar que s'ha produït un error utilitzant la instrucció raise.
  • Excepcions Personalitzades: Excepcions definides per l'usuari que estenen la classe base Exception.

Llençar Excepcions

Sintaxi Bàsica

Per llençar una excepció en Python, utilitzes la instrucció raise seguida d'una instància d'una classe d'excepció.

# Exemple: Llençar una excepció bàsica
raise Exception("Aquesta és una excepció genèrica")

Excepcions Integrades Comunes

Python proporciona diverses excepcions integrades que pots utilitzar per senyalar diferents tipus d'errors:

  • ValueError: Es llença quan una funció rep un argument del tipus correcte però amb un valor inapropiat.
  • TypeError: Es llença quan una operació o funció s'aplica a un objecte de tipus inapropiat.
  • IndexError: Es llença quan un subíndex d'una seqüència està fora de rang.
  • KeyError: Es llença quan una clau d'un diccionari no es troba.
# Exemple: Llençar un ValueError
def divide(a, b):
    if b == 0:
        raise ValueError("No es pot dividir per zero")
    return a / b

try:
    result = divide(10, 0)
except ValueError as e:
    print(e)

Crear Excepcions Personalitzades

Pots crear les teves pròpies excepcions definint una nova classe que hereti de la classe Exception.

# Exemple: Crear una excepció personalitzada
class CustomError(Exception):
    def __init__(self, message):
        self.message = message

# Llençar l'excepció personalitzada
raise CustomError("Aquesta és una errada personalitzada")

Utilitzar Excepcions Personalitzades

Les excepcions personalitzades es poden utilitzar per proporcionar missatges d'error més específics i per gestionar errors d'una manera més granular.

# Exemple: Utilitzar una excepció personalitzada en una funció
class NegativeNumberError(Exception):
    pass

def check_positive(number):
    if number < 0:
        raise NegativeNumberError("No es permeten nombres negatius")

try:
    check_positive(-5)
except NegativeNumberError as e:
    print(e)

Tècniques Avançades

Enllaçar Excepcions

Pots enllaçar excepcions per proporcionar més context sobre l'error. Això es fa utilitzant la paraula clau from.

# Exemple: Enllaçar excepcions
try:
    raise ValueError("Error inicial")
except ValueError as e:
    raise CustomError("S'ha produït un error personalitzat") from e

Re-llençar Excepcions

De vegades potser vulguis capturar una excepció, realitzar algunes accions i després re-llençar la mateixa excepció.

# Exemple: Re-llençar una excepció
try:
    raise ValueError("S'ha produït un error")
except ValueError as e:
    print("Gestionant l'error")
    raise  # Re-llençar la mateixa excepció

Conclusió

Llençar excepcions és una part fonamental de la gestió d'errors en Python. Entenent com llençar excepcions integrades, crear excepcions personalitzades i utilitzar tècniques avançades com l'enllaç i la re-llençada, pots escriure codi més robust i mantenible. Recorda sempre proporcionar missatges d'error significatius per ajudar amb la depuració i gestionar les excepcions de manera elegant per assegurar una bona experiència d'usuari.
