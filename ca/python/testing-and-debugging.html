Introducció

Les proves i la depuració són habilitats crucials per a qualsevol programador. Asseguren que el teu codi funcioni com s'espera i t'ajuden a identificar i corregir errors. Aquesta secció cobrirà els conceptes bàsics de les proves i la depuració en Python, progressant des de nivells principiants fins a avançats.

Proves Unitàries amb `unittest`

Les proves unitàries impliquen provar components individuals del teu codi per assegurar-se que funcionin correctament.

Conceptes Bàsics de `unittest`

  • Què és unittest?
    • unittest és un mòdul integrat de Python per crear i executar proves.
  • Crear un Cas de Prova
    • Un cas de prova és una unitat única de prova. Comprova una resposta específica a un conjunt particular d'entrades.
import unittest

def add(a, b):
    return a + b

class TestMathOperations(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(1, 2), 3)
        self.assertEqual(add(-1, 1), 0)

if __name__ == '__main__':
    unittest.main()

Executar Proves

  • Línia de Comandes
    • Executa les teves proves des de la línia de comandes utilitzant: 
      python -m unittest test_module.py
  • Descobriment de Proves
    • Descobreix i executa automàticament les proves: 
      python -m unittest discover

Proves Avançades amb `pytest`

pytest és un marc de proves més potent que simplifica l'escriptura i l'execució de proves.

Conceptes Bàsics de `pytest`

  • Instal·lació
    • Instal·la pytest utilitzant pip: 
      pip install pytest
  • Escriure Proves
    • Les proves s'escriuen com a funcions prefixades amb test_.
def add(a, b):
    return a + b

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0

Executar Proves

  • Línia de Comandes
    • Executa les teves proves utilitzant: 
      pytest

Fixtures

  • Què són els Fixtures?
    • Els fixtures s'utilitzen per configurar un estat conegut abans que s'executin les proves i netejar després.
import pytest

@pytest.fixture
def sample_data():
    return [1, 2, 3, 4, 5]

def test_sum(sample_data):
    assert sum(sample_data) == 15

Depuració amb `pdb`

pdb és el depurador integrat de Python, que et permet avançar línia per línia pel codi, inspeccionar variables i avaluar expressions.

Conceptes Bàsics de `pdb`

  • Iniciar el Depurador
    • Insereix pdb.set_trace() al punt on vols començar a depurar.
import pdb

def add(a, b):
    pdb.set_trace()
    return a + b

print(add(1, 2))

Comandes Comunes

  • n (next)
    • Avança a la següent línia dins de la mateixa funció.
  • s (step)
    • Entra en una crida de funció.
  • c (continue)
    • Continua l'execució fins al següent punt de ruptura.
  • p (print)
    • Imprimeix el valor d'una expressió.

Tècniques Avançades de Depuració

Utilitzar `logging`

  • Per què Utilitzar Logging?
    • El logging proporciona una manera de fer un seguiment dels esdeveniments que succeeixen quan s'executa un programari. Pot ser més flexible i informatiu que les instruccions print.
import logging

logging.basicConfig(level=logging.DEBUG)

def add(a, b):
    logging.debug(f'Adding {a} and {b}')
    return a + b

print(add(1, 2))

Perfilat del Codi

  • Per què Perfilar?
    • El perfilat t'ajuda a entendre on el teu programa passa el seu temps i identificar colls d'ampolla.
import cProfile

def slow_function():
    total = 0
    for i in range(10000):
        total suma i
    return total

cProfile.run('slow_function()')

Conclusió

Les proves i la depuració són pràctiques essencials per escriure codi Python fiable i mantenible. En dominar eines com unittest, pytest, pdb i logging, pots assegurar-te que el teu codi sigui robust i eficient. Recorda, l'objectiu no és només trobar errors, sinó entendre per què ocorren i com prevenir-los en el futur.
