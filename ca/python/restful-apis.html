Introducció a les APIs RESTful

  • Definició: REST (Representational State Transfer) és un estil arquitectònic per dissenyar aplicacions en xarxa. Es basa en un protocol de comunicacions sense estat, client-servidor i emmagatzemable en memòria cau: l'HTTP.
  • Conceptes Clau:
    • Recursos: Objectes o representacions de dades que es poden accedir a través d'una URL.
    • Mètodes HTTP: Els mètodes comuns inclouen GET, POST, PUT, DELETE.
    • Sense estat: Cada sol·licitud d'un client al servidor ha de contenir tota la informació necessària per entendre i processar la sol·licitud.
    • Operacions CRUD: Operacions de Crear, Llegir, Actualitzar, Eliminar que es mapegen als mètodes HTTP.

Configuració del Teu Entorn

  • Instal·lació de Python: Assegura't que Python està instal·lat al teu sistema.
  • Biblioteques Necessàries: Flask per crear APIs, requests per consumir APIs. 
    pip install Flask requests

Creació d'una API RESTful Simple amb Flask

Configuració de Flask

  • Aplicació Bàsica de Flask: 
    from flask import Flask, jsonify, request

app = Flask(__name__)

@app.route('/')
def home():
    return "Benvingut a l'API RESTful amb Flask!"

if __name__ == '__main__':
    app.run(debug=True)

Definició de Punts Finals

  • Sol·licitud GET:

    @app.route('/api/v1/resource', methods=['GET'])
def get_resource():
    data = {"message": "Aquesta és una sol·licitud GET"}
    return jsonify(data)

  • Sol·licitud POST:

    @app.route('/api/v1/resource', methods=['POST'])
def create_resource():
    new_data = request.get_json()
    response = {"message": "Recurs creat", "data": new_data}
    return jsonify(response), 201

  • Sol·licitud PUT:

    @app.route('/api/v1/resource/<int:id>', methods=['PUT'])
def update_resource(id):
    updated_data = request.get_json()
    response = {"message": f"Recurs {id} actualitzat", "data": updated_data}
    return jsonify(response)

  • Sol·licitud DELETE:

    @app.route('/api/v1/resource/<int:id>', methods=['DELETE'])
def delete_resource(id):
    response = {"message": f"Recurs {id} eliminat"}
    return jsonify(response), 204

Consumir APIs RESTful amb `requests`

  • Sol·licitud GET:

    import requests

response = requests.get('http://localhost:5000/api/v1/resource')
print(response.json())

  • Sol·licitud POST:

    import requests

data = {"name": "Nou Recurs"}
response = requests.post('http://localhost:5000/api/v1/resource', json=data)
print(response.json())

  • Sol·licitud PUT:

    import requests

data = {"name": "Recurs Actualitzat"}
response = requests.put('http://localhost:5000/api/v1/resource/1', json=data)
print(response.json())

  • Sol·licitud DELETE:

    import requests

response = requests.delete('http://localhost:5000/api/v1/resource/1')
print(response.status_code)

Temes Avançats

Autenticació i Autorització

  • Autenticació Bàsica: 
    from flask import request, jsonify

@app.route('/api/v1/secure-resource', methods=['GET'])
def secure_resource():
    auth = request.authorization
    if auth and auth.username == 'user' and auth.password == 'pass':
        return jsonify({"message": "Autenticat"})
    return jsonify({"message": "Autenticació fallida"}), 401

Ús de Flask-RESTful

  • Instal·lació:

    pip install Flask-RESTful

  • Creació d'una API RESTful amb Flask-RESTful:

    from flask import Flask
from flask_restful import Resource, Api

app = Flask(__name__)
api = Api(app)

class HelloWorld(Resource):
    def get(self):
        return {'hello': 'world'}

api.add_resource(HelloWorld, '/')

if __name__ == '__main__':
    app.run(debug=True)

Conclusió

  • Resum:
    • Les APIs RESTful són una manera poderosa de permetre la comunicació entre diferents parts d'una aplicació web o entre diferents aplicacions.
    • Flask és un framework lleuger i fàcil d'utilitzar per crear APIs RESTful en Python.
    • Entendre com crear i consumir APIs RESTful és essencial per al desenvolupament web modern.
