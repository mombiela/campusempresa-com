La Integració Contínua (CI) i el Desplegament Continu (CD) són pràctiques essencials en el desenvolupament de programari modern. Ajuden a garantir que els canvis de codi es proven i es despleguen automàticament, cosa que condueix a cicles de desenvolupament més fiables i ràpids. Aquesta secció et guiarà a través dels conceptes bàsics de CI/CD, centrant-se en projectes Python.

Què és la Integració Contínua?

La Integració Contínua és la pràctica de fusionar totes les còpies de treball dels desenvolupadors en una línia principal compartida diverses vegades al dia. Els objectius principals són detectar problemes d'integració aviat i millorar la qualitat del programari.

Conceptes Clau de la Integració Contínua

  • Sistema de Control de Versions (VCS): Una eina com Git per gestionar els canvis de codi.
  • Automatització de la Construcció: Compilar i construir el projecte automàticament.
  • Proves Automatitzades: Executar proves automàticament per detectar problemes aviat.
  • Integració: Fusionar els canvis de codi freqüentment per evitar problemes d'integració.

Exemple: Configurar CI amb GitHub Actions

GitHub Actions és una eina de CI popular que s'integra perfectament amb els repositoris de GitHub.

  1. Crea un Repositori a GitHub: Inicialitza un nou repositori a GitHub.
  2. Afegeix un Projecte Python: Afegeix els fitxers del teu projecte Python al repositori.
  3. Crea un Fitxer de Flux de Treball: Afegeix un fitxer .github/workflows/ci.yml per definir el pipeline de CI.
name: Python CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.x'
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
    - name: Run tests
      run: |
        pytest

Explicació

  • name: El nom del flux de treball.
  • on: Especifica els esdeveniments que desencadenen el flux de treball (push i pull_request).
  • jobs: Defineix les tasques a executar.
  • runs-on: Especifica el sistema operatiu per a la tasca.
  • steps: La seqüència de passos a executar en la tasca.
    • uses: Especifica una acció a utilitzar.
    • run: Executa una comanda.

Què és el Desplegament Continu?

El Desplegament Continu és la pràctica de desplegar automàticament cada canvi que passa les proves automatitzades a producció. Això garanteix que la versió més recent de l'aplicació estigui sempre disponible per als usuaris.

Conceptes Clau del Desplegament Continu

  • Desplegament Automatitzat: Desplegar codi automàticament a producció.
  • Configuració d'Entorns: Gestionar diferents entorns (desenvolupament, staging, producció).
  • Mecanisme de Reversió: Capacitat de revertir a una versió anterior si alguna cosa va malament.

Exemple: Configurar CD amb GitHub Actions

Amplia el flux de treball de CI anterior per incloure passos de desplegament.

  1. Actualitza el Fitxer de Flux de Treball: Modifica .github/workflows/ci.yml per incloure el desplegament.
name: Python CI/CD

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.x'
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
    - name: Run tests
      run: |
        pytest

  deploy:
    needs: build
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Deploy to Production
      run: |
        echo "Deploying to production..."
        # Afegeix el teu script de desplegament aquí

Explicació

  • deploy: Una nova tasca que depèn de la tasca build.
  • needs: Especifica que la tasca deploy només s'ha d'executar si la tasca build té èxit.
  • run: Executa l'script de desplegament.

Resum

La Integració i el Desplegament Continu són pràctiques crucials per al desenvolupament de programari modern. Ajuden a:

  • Detectar problemes aviat mitjançant la integració freqüent i les proves automatitzades.
  • Garantir que el codi més recent estigui sempre desplegat a producció, cosa que condueix a llançaments més ràpids i fiables.

Configurant pipelines de CI/CD amb eines com GitHub Actions, pots automatitzar tot el procés des de la integració de codi fins al desplegament, fent que el teu flux de treball de desenvolupament sigui més eficient i robust.
