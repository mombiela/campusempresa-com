Introducció a les Proves Unitàries

Les proves unitàries són una tècnica de proves de programari on es proven unitats o components individuals d'un programari de manera aïllada. L'objectiu és validar que cada unitat del programari funcioni com s'espera. En Python, el mòdul unittest proporciona un marc per crear i executar proves unitàries.

Per què Proves Unitàries?

  • Detecció Primerenca d'Errors: Identifica problemes a l'inici del cicle de desenvolupament.
  • Qualitat del Codi: Assegura que el codi funcioni com s'espera.
  • Confiança en la Refactorització: Permet als desenvolupadors refactoritzar el codi sense por d'introduir nous errors.
  • Documentació: Serveix com a documentació per a la base de codi.

Començar amb unittest

El mòdul unittest forma part de la biblioteca estàndard de Python, per tant, no es requereix cap instal·lació addicional. A continuació es mostra un exemple senzill per començar.

Exemple Bàsic

import unittest

def add(a, b):
    return a + b

class TestAddFunction(unittest.TestCase):
    def test_add_integers(self):
        self.assertEqual(add(1, 2), 3)
    
    def test_add_floats(self):
        self.assertEqual(add(1.1, 2.2), 3.3)
    
    def test_add_strings(self):
        self.assertEqual(add('hello', ' world'), 'hello world')

if __name__ == '__main__':
    unittest.main()

Explicació

  • import unittest: Importa el mòdul unittest.
  • funció add: Una funció senzilla per sumar dos números o concatenar dues cadenes.
  • classe TestAddFunction: Hereta de unittest.TestCase i conté mètodes de prova.
  • test_add_integers, test_add_floats, test_add_strings: Mètodes de prova que utilitzen self.assertEqual per comprovar si la funció add funciona correctament.
  • unittest.main(): Executa les proves quan s'executa l'script.

Conceptes Clau

  • Cas de Prova: La unitat més petita de prova. Comprova una resposta específica a un conjunt particular d'entrades.
  • Suite de Proves: Una col·lecció de casos de prova, suites de prova, o ambdues.
  • Executor de Proves: Un component que orquestra l'execució de les proves i proporciona el resultat a l'usuari.

Funcions Avançades

Configuració i Desmuntatge

De vegades, cal configurar un entorn particular abans d'executar les proves i netejar després. unittest proporciona els mètodes setUp i tearDown per a aquest propòsit.

import unittest

class TestExample(unittest.TestCase):
    def setUp(self):
        self.value = 42

    def tearDown(self):
        del self.value

    def test_value(self):
        self.assertEqual(self.value, 42)

if __name__ == '__main__':
    unittest.main()

Mocking

El mocking és útil per aïllar la unitat de treball que s'està provant substituint les dependències amb objectes mock.

from unittest.mock import MagicMock
import unittest

class Database:
    def fetch_data(self):
        pass  # Imaginem que aquest mètode obté dades d'una base de dades

class DataProcessor:
    def __init__(self, db):
        self.db = db

    def process(self):
        data = self.db.fetch_data()
        return data * 2

class TestDataProcessor(unittest.TestCase):
    def test_process(self):
        mock_db = MagicMock()
        mock_db.fetch_data.return_value = 21
        processor = DataProcessor(mock_db)
        self.assertEqual(processor.process(), 42)

if __name__ == '__main__':
    unittest.main()

Proves Parametritzades

Podeu executar la mateixa prova amb diferents paràmetres utilitzant la funció subTest del mòdul unittest.

import unittest

class TestParameterized(unittest.TestCase):
    def test_add(self):
        casos_de_prova = [
            (1, 2, 3),
            (1.1, 2.2, 3.3),
            ('hello', ' world', 'hello world')
        ]
        for a, b, expected in casos_de_prova:
            with self.subTest(a=a, b=b, expected=expected):
                self.assertEqual(add(a, b), expected)

if __name__ == '__main__':
    unittest.main()

Conclusió

Les proves unitàries són una pràctica essencial per mantenir un codi d'alta qualitat. El mòdul unittest en Python proporciona un marc robust per escriure i executar proves. Entenent i utilitzant les seves funcions, podeu assegurar-vos que el vostre codi sigui fiable, mantenible i lliure d'errors. Bones proves!
