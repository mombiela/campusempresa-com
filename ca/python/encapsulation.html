L'encapsulació és un dels principis fonamentals de la Programació Orientada a Objectes (OOP). Es refereix a l'agrupació de dades (atributs) i mètodes (funcions) que operen sobre les dades en una sola unitat, normalment una classe. L'encapsulació també implica restringir l'accés directe a alguns dels components d'un objecte, cosa que és un mitjà per prevenir interferències accidentals i ús indegut de les dades.

Conceptes Clau de l'Encapsulació

  • Classe: Un plànol per crear objectes (una estructura de dades particular).
  • Objecte: Una instància d'una classe.
  • Atributs: Variables que pertanyen a un objecte.
  • Mètodes: Funcions que pertanyen a un objecte.
  • Membres Privats: Atributs o mètodes que no poden ser accedits directament des de fora de la classe.
  • Membres Públics: Atributs o mètodes que poden ser accedits des de fora de la classe.

Encapsulació en Python

En Python, l'encapsulació es implementa utilitzant classes i objectes. Python no té modificadors d'accés estrictes com alguns altres llenguatges (per exemple, private, protected, public en Java), però utilitza convencions de nomenclatura per indicar el nivell d'accés previst.

Membres Públics

Els membres públics són accessibles des de fora de la classe. Per defecte, tots els membres d'una classe són públics en Python.

class Car:
    def __init__(self, make, model):
        self.make = make  # Atribut públic
        self.model = model  # Atribut públic

    def display_info(self):
        return f"Car make: {self.make}, Model: {self.model}"  # Mètode públic

# Creant un objecte de la classe Car
my_car = Car("Toyota", "Corolla")
print(my_car.display_info())  # Accedint al mètode públic
print(my_car.make)  # Accedint a l'atribut públic

Membres Privats

Els membres privats estan destinats a ser accedits només dins de la classe. En Python, els membres privats s'indiquen prefixant el nom del membre amb dobles guions baixos (__).

class Car:
    def __init__(self, make, model):
        self.__make = make  # Atribut privat
        self.__model = model  # Atribut privat

    def __display_info(self):  # Mètode privat
        return f"Car make: {self.__make}, Model: {self.__model}"

    def get_info(self):  # Mètode públic per accedir al mètode privat
        return self.__display_info()

# Creant un objecte de la classe Car
my_car = Car("Toyota", "Corolla")
print(my_car.get_info())  # Accedint al mètode privat a través d'un mètode públic
# print(my_car.__make)  # Això generarà un AttributeError

Ofuscació de Noms

Python realitza l'ofuscació de noms dels membres privats per evitar l'accés i modificació accidental. L'intèrpret canvia el nom de les variables privades d'una manera que les fa més difícils de crear i accedir des de fora de la classe.

class Car:
    def __init__(self, make, model):
        self.__make = make  # Atribut privat
        self.__model = model  # Atribut privat

    def get_make(self):
        return self.__make

# Creant un objecte de la classe Car
my_car = Car("Toyota", "Corolla")
print(my_car.get_make())  # Accedint a l'atribut privat a través d'un mètode públic
print(my_car._Car__make)  # Accedint a l'atribut privat utilitzant l'ofuscació de noms (no recomanat)

Beneficis de l'Encapsulació

  • Control: L'encapsulació permet controlar com es accedeixen i modifiquen les dades.
  • Seguretat: Ajuda a protegir l'estat intern d'un objecte de canvis no intencionats o perjudicials.
  • Flexibilitat: La implementació interna d'una classe es pot canviar sense afectar el codi que utilitza la classe.
  • Mantenibilitat: L'encapsulació fa que el codi sigui més fàcil de mantenir i entendre.

Conclusió

L'encapsulació és un concepte crucial en OOP que ajuda a agrupar dades i mètodes en una sola unitat i controlar l'accés a les dades. En Python, l'encapsulació s'aconsegueix utilitzant classes i objectes, amb membres públics i privats indicats per convencions de nomenclatura. Entendre i aplicar l'encapsulació pot conduir a un codi més segur, mantenible i flexible.
