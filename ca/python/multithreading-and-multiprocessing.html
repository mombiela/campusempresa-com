Asyncio per a Programació Asíncrona ►

Introducció

En aquesta secció, explorarem els conceptes de multithreading i multiprocessing en Python. Aquestes tècniques són essencials per millorar el rendiment dels teus programes permetent-los realitzar múltiples tasques de manera concurrent.

Multithreading

El multithreading permet que un programa executi múltiples fils (unitats més petites d'un procés) simultàniament. Això és particularment útil per a tasques de tipus I/O.

Conceptes Clau

  • Fil: La unitat més petita d'un procés que es pot programar per a l'execució.
  • GIL (Global Interpreter Lock): Un mutex que protegeix l'accés als objectes de Python, evitant que múltiples fils executin bytecodes de Python alhora.
  • Seguretat de Fil: Assegurar que les dades compartides siguin accedides per només un fil a la vegada.

Exemple Bàsic

import threading

def print_numbers():
    for i in range(5):
        print(i)

def print_letters():
    per letter in 'abcde':
        print(letter)

# Creació de fils
thread1 = threading.Thread(target=print_numbers)
thread2 = threading.Thread(target=print_letters)

# Inici dels fils
thread1.start()
thread2.start()

# Espera que els fils completin
thread1.join()
thread2.join()

print("Fet!")

Explicació:

  • Definim dues funcions: print_numbers i print_letters.
  • Creem dos fils, cadascun apuntant a una de les funcions.
  • Iniciem ambdós fils i esperem que completin utilitzant join().

Exemple de Seguretat de Fil

import threading

counter = 0
lock = threading.Lock()

def increment_counter():
    global counter
    per _ in range(1000):
        with lock:
            counter += 1

threads = []
per _ in range(10):
    thread = threading.Thread(target=increment_counter)
    threads.append(thread)
    thread.start()

per thread in threads:
    thread.join()

print(f"Valor final del comptador: {counter}")

Explicació:

  • Utilitzem un Lock per assegurar que només un fil pugui incrementar el comptador a la vegada, evitant condicions de carrera.

Multiprocessing

El multiprocessing implica utilitzar múltiples processos, cadascun amb el seu propi intèrpret de Python i espai de memòria. Això és útil per a tasques de tipus CPU.

Conceptes Clau

  • Procés: Un programa independent en execució.
  • IPC (Inter-Process Communication): Mecanismes perquè els processos es comuniquin i sincronitzin les seves accions.
  • Memòria Compartida: Memòria que pot ser accedida per múltiples processos.

Exemple Bàsic

import multiprocessing

def print_numbers():
    per i in range(5):
        print(i)

def print_letters():
    per letter in 'abcde':
        print(letter)

# Creació de processos
process1 = multiprocessing.Process(target=print_numbers)
process2 = multiprocessing.Process(target=print_letters)

# Inici dels processos
process1.start()
process2.start()

# Espera que els processos completin
process1.join()
process2.join()

print("Fet!")

Explicació:

  • Similar a l'exemple de fils, però utilitzant multiprocessing.Process en lloc de threading.Thread.

Exemple de Memòria Compartida

import multiprocessing

def increment_counter(counter):
    per _ in range(1000):
        counter.value += 1

if __name__ == "__main__":
    counter = multiprocessing.Value('i', 0)
    processes = []
    
    per _ in range(10):
        process = multiprocessing.Process(target=increment_counter, args=(counter,))
        processes.append(process)
        process.start()
    
    per process in processes:
        process.join()
    
    print(f"Valor final del comptador: {counter.value}")

Explicació:

  • Utilitzem multiprocessing.Value per crear un enter compartit ('i' significa enter) que pot ser modificat de manera segura per múltiples processos.

Taula Comparativa

| Característica | Multithreading | Multiprocessing | |------------------|-----------------------------------------|-----------------------------------------| | Tipus de Concurrència | A nivell de fil | A nivell de procés | | Espai de Memòria | Compartit | Separat | | Adequat Per | Tasques de tipus I/O | Tasques de tipus CPU | | Impacte del GIL | Afectat pel GIL | No afectat pel GIL | | Comunicació | Més fàcil (memòria compartida) | Més complexa (mecanismes IPC) |

Conclusió

El multithreading i el multiprocessing són eines poderoses per millorar el rendiment dels programes en Python. El multithreading és adequat per a tasques de tipus I/O, mentre que el multiprocessing és millor per a tasques de tipus CPU. Entendre les diferències i els casos d'ús adequats per a cadascun pot ajudar-te a escriure codi Python més eficient i efectiu.
