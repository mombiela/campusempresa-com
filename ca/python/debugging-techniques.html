La depuració és una habilitat essencial per a qualsevol programador. Aquest tema cobrirà diverses tècniques de depuració en Python, des de nivells bàsics fins a avançats. Al final d'aquesta secció, estaràs equipat amb el coneixement per identificar, diagnosticar i corregir errors en el teu codi Python de manera eficient.

Introducció a la depuració

  • Definició: La depuració és el procés d'identificar i eliminar errors del maquinari o programari de l'ordinador.
  • Importància: La depuració ajuda a assegurar que el teu codi s'executi correctament i eficientment.

Tècniques bàsiques de depuració

Declaracions de print

Una de les tècniques de depuració més simples i comunes és utilitzar declaracions de print per mostrar els valors de les variables en diferents punts del teu codi.

def add(a, b):
    print(f"a: {a}, b: {b}")  # Declaració de print per a depuració
    return a + b

result = add(3, 4)
print(f"Result: {result}")
  • Pros: Fàcil d'implementar i entendre.
  • Cons: Pot omplir el teu codi i no és adequat per a projectes grans.

Ús d'assertions

Les assertions són declaracions que comproven si una condició és certa. Si la condició és falsa, el programa generarà un AssertionError.

def divide(a, b):
    assert b != 0, "b no pot ser zero"
    return a / b

result = divide(10, 2)
print(f"Result: {result}")
  • Pros: Ajuda a detectar errors a l'inici del procés de desenvolupament.
  • Cons: No s'haurien d'utilitzar per gestionar errors en temps d'execució en codi de producció.

Tècniques de depuració intermèdies

Ús del mòdul `logging`

El mòdul logging proporciona un marc flexible per emetre missatges de registre des de programes Python. És més avançat que les declaracions de print i es pot configurar per enviar missatges a diferents destinacions.

import logging

logging.basicConfig(level=logging.DEBUG)

def multiply(a, b):
    logging.debug(f"multiply called with a={a}, b={b}")
    return a * b

result = multiply(3, 4)
logging.info(f"Result: {result}")
  • Pros: Més flexible i potent que les declaracions de print.
  • Cons: Requereix una configuració inicial i comprensió dels nivells de registre.

Ús del mòdul `pdb`

El depurador de Python (pdb) et permet establir punts de ruptura, avançar pas a pas pel codi i inspeccionar variables.

import pdb

def subtract(a, b):
    pdb.set_trace()  # Establir un punt de ruptura
    return a - b

result = subtract(10, 5)
print(f"Result: {result}")
  • Pros: Eina potent per a la depuració interactiva.
  • Cons: Pot ser complex d'utilitzar de manera efectiva per a principiants.

Tècniques de depuració avançades

Ús d'entorns de desenvolupament integrats (IDEs)

IDEs com PyCharm, VSCode i altres vénen amb eines de depuració integrades que proporcionen una interfície gràfica per establir punts de ruptura, avançar pas a pas pel codi i inspeccionar variables.

  • Pros: Fàcil d'utilitzar i rica en funcionalitats.
  • Cons: Pot ser aclaparador per a principiants i requerir una corba d'aprenentatge.

Perfilat i optimització del rendiment

El perfilat t'ajuda a entendre on el teu programa passa la major part del temps i identificar colls d'ampolla.

import cProfile

def slow_function():
    total = 0
    for i in range(10000):
        total += i
    return total

cProfile.run('slow_function()')
  • Pros: Ajuda a optimitzar el rendiment del teu codi.
  • Cons: Requereix comprensió de les mètriques de rendiment i eines de perfilat.

Conclusió

La depuració és una habilitat crítica per a qualsevol programador. Començant amb tècniques bàsiques com les declaracions de print i les assertions, pots avançar gradualment cap a mètodes més avançats com l'ús dels mòduls logging i pdb, i aprofitar les potents eines proporcionades pels IDEs moderns. El perfilat i l'optimització del rendiment també són essencials per optimitzar el teu codi.

En dominar aquestes tècniques de depuració, estaràs millor equipat per escriure codi Python eficient i lliure d'errors.
