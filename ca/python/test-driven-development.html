Introducció al Desenvolupament Basat en Proves

El Desenvolupament Basat en Proves (TDD) és un procés de desenvolupament de programari on escrius proves per al teu codi abans d'escriure el codi real. Aquest enfocament ajuda a assegurar que el teu codi sigui fiable, mantenible i lliure d'errors.

Conceptes Clau del TDD

  • Cicle Red-Green-Refactor: La pràctica central del TDD implica tres passos:
    1. Red: Escriu una prova que falli.
    2. Green: Escriu la quantitat mínima de codi necessària per fer que la prova passi.
    3. Refactor: Millora el codi mentre assegures que totes les proves continuen passant.
  • Proves Unitàries: Proves petites que verifiquen el comportament d'una sola funció o mètode.
  • Afirmacions: Declaracions en les proves que comproven si una condició és certa.

Configuració del Teu Entorn

Abans de començar amb TDD, necessites configurar el teu entorn Python amb les eines necessàries.

Instal·lació de pytest

pytest és un marc de proves popular per a Python. Pots instal·lar-lo utilitzant pip:

pip install pytest

Escriure la Teva Primera Prova

Comencem amb un exemple senzill per il·lustrar el procés TDD.

Exemple: Funció de Suma

  1. Red: Escriu una prova que falli perquè la funció encara no existeix.
  2. Green: Implementa la funció per fer que la prova passi.
  3. Refactor: Neteja el codi si és necessari.

Pas 1: Escriure una Prova que Falli

Crea un fitxer anomenat test_sum.py:

def test_sum():
    assert sum([1, 2, 3]) == 6

Executa la prova utilitzant pytest:

pytest test_sum.py

Hauries de veure un error perquè la funció sum no està definida.

Pas 2: Implementar la Funció

Crea un fitxer anomenat sum.py:

def sum(numbers):
    return sum(numbers)

Actualitza test_sum.py per importar la funció sum:

from sum import sum

def test_sum():
    assert sum([1, 2, 3]) == 6

Executa la prova de nou:

pytest test_sum.py

La prova hauria de passar ara.

Pas 3: Refactoritzar

En aquest exemple senzill, potser no hi hagi molt a refactoritzar, però sempre comprova si el codi es pot millorar sense trencar les proves.

Conceptes Avançats de TDD

A mesura que et sentis més còmode amb TDD, pots explorar temes més avançats.

Mocking i Patching

De vegades, necessites provar codi que interactua amb sistemes externs (per exemple, bases de dades, serveis web). Pots utilitzar mocking per simular aquestes interaccions.

Exemple: Mocking d'una Crida a la Base de Dades

from unittest.mock import patch

def get_user_from_db(user_id):
    # Simula una crida a la base de dades
    pass

@patch('your_module.get_user_from_db')
def test_get_user(mock_get_user):
    mock_get_user.return_value = {'id': 1, 'name': 'John Doe'}
    user = get_user_from_db(1)
    assert user['name'] == 'John Doe'

Proves Parametritzades

Pots executar la mateixa prova amb diferents entrades utilitzant proves parametritzades.

Exemple: Prova Parametritzada

import pytest

@pytest.mark.parametrize("input,expected", [
    ([1, 2, 3], 6),
    ([4, 5, 6], 15),
    ([7, 8, 9], 24),
])
def test_sum(input, expected):
    assert sum(input) == expected

Conclusió

El Desenvolupament Basat en Proves és una metodologia poderosa que pot ajudar-te a escriure codi millor i més fiable. Seguint el cicle Red-Green-Refactor, assegures que el teu codi compleix els requisits i està lliure d'errors. A mesura que avancis, pots incorporar tècniques més sofisticades com el mocking i les proves parametritzades per manejar escenaris complexos. Feliç codificació!
