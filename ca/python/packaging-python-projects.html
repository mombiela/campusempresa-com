Introducció

Empaquetar projectes Python és una habilitat essencial per als desenvolupadors que volen distribuir el seu codi a altres persones. Aquesta secció et guiarà a través del procés d'empaquetar els teus projectes Python, des dels conceptes bàsics fins a tècniques més avançades.

Per què empaquetar els teus projectes Python?

  • Distribució: Comparteix fàcilment el teu codi amb altres persones.
  • Gestió de dependències: Assegura't que totes les biblioteques i dependències necessàries estiguin incloses.
  • Control de versions: Gestiona diferents versions del teu projecte.
  • Instal·lació: Simplifica el procés d'instal·lació per als usuaris.

Conceptes bàsics

  • Paquet: Una col·lecció de mòduls Python.
  • Mòdul: Un únic fitxer Python que conté codi.
  • Distribució: Un arxiu d'arxiu versionat que es pot instal·lar.

Configuració del teu projecte

  1. Estructura de directoris: Organitza els fitxers del teu projecte. 
    my_project/
├── my_project/
│   ├── __init__.py
│   ├── module1.py
│   └── module2.py
├── tests/
│   ├── __init__.py
│   └── test_module1.py
├── README.md
├── setup.py
└── requirements.txt
  2. README.md: Proporciona una descripció del teu projecte.
  3. requirements.txt: Llista totes les dependències. 
    numpy==1.21.0
pandas==1.3.0

Creació de setup.py

El fitxer setup.py és el nucli de cada projecte Python. Conté metadades sobre el teu projecte i instruccions sobre com instal·lar-lo.

Exemple de setup.py

from setuptools import setup, find_packages

setup(
    name='my_project',
    version='0.1.0',
    packages=find_packages(),
    install_requires=[
        'numpy==1.21.0',
        'pandas==1.3.0',
    ],
    entry_points={
        'console_scripts': [
            'my_project=my_project.module1:main',
        ],
    },
    author='Your Name',
    author_email='your.email@example.com',
    description='A brief description of your project',
    long_description=open('README.md').read(),
    long_description_content_type='text/markdown',
    url='https://github.com/yourusername/my_project',
    classifiers=[
        'Programming Language :: Python :: 3',
        'License :: OSI Approved :: MIT License',
        'Operating System :: OS Independent',
    ],
    python_requires='>=3.6',
)

Explicació

  • name: El nom del teu projecte.
  • version: La versió actual del teu projecte.
  • packages: Una llista de tots els paquets d'importació Python que s'han d'incloure en el paquet de distribució.
  • install_requires: Una llista de dependències que són necessàries per al teu projecte.
  • entry_points: Defineix punts d'entrada per a scripts de línia de comandes.
  • author: El teu nom.
  • author_email: La teva adreça de correu electrònic.
  • description: Un resum breu del teu projecte.
  • long_description: Una descripció detallada, normalment el contingut del teu fitxer README.
  • url: L'URL de la pàgina principal del teu projecte.
  • classifiers: Una llista de classificadors que descriuen el teu projecte.
  • python_requires: La versió de Python requerida per al teu projecte.

Construcció i distribució del teu paquet

  1. Construeix el teu paquet:

    python setup.py sdist bdist_wheel
    • sdist: Distribució de codi font.
    • bdist_wheel: Distribució construïda.

  2. Carrega a PyPI:

    pip install twine
twine upload dist/*
    • twine: Una utilitat per publicar paquets Python a PyPI.

Instal·lació del teu paquet

Els usuaris poden instal·lar el teu paquet utilitzant pip:

pip install my_project

Conclusió

Empaquetar els teus projectes Python et permet compartir el teu treball amb altres persones, gestionar dependències i simplificar el procés d'instal·lació. Seguint els passos descrits en aquesta secció, pots crear, construir i distribuir els teus paquets Python de manera eficient. Recorda mantenir el teu fitxer setup.py actualitzat amb les metadades i dependències correctes per assegurar una experiència d'instal·lació fluida per als teus usuaris.
