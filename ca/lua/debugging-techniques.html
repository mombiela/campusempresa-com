Tècniques de Depuració en Lua
Utilitzar la Biblioteca de Depuració ►

La depuració és una habilitat essencial per a qualsevol programador. En aquesta secció, explorarem diverses tècniques de depuració en Lua, des de nivells bàsics fins a avançats. Al final d'aquest tema, hauríeu de ser capaços d'identificar i corregir errors en el vostre codi Lua de manera eficient.

Tècniques Bàsiques de Depuració

Declaracions Print

Una de les tècniques de depuració més simples i efectives és utilitzar declaracions print per mostrar els valors de les variables i el flux d'execució.

local function add(a, b)
    print("a:", a, "b:", b)  -- Declaració print de depuració
    return a + b
end

print(add(5, 3))
  • Pros: Fàcil d'implementar i entendre.
  • Cons: Pot omplir el codi i no és adequat per a projectes grans.

Funció Error

Lua proporciona una funció error per acabar el programa i mostrar un missatge d'error.

local function divide(a, b)
    if b == 0 then
        error("La divisió per zero no està permesa")
    end
    return a / b
end

print(divide(10, 2))
print(divide(10, 0))  -- Això provocarà un error
  • Pros: Indica clarament on ha ocorregut l'error.
  • Cons: Atura l'execució del programa.

Tècniques Intermèdies de Depuració

Ús de `assert`

La funció assert s'utilitza per comprovar condicions i llançar un error si la condició és falsa.

local function sqrt(x)
    assert(x >= 0, "x ha de ser no negatiu")
    return math.sqrt(x)
end

print(sqrt(9))
print(sqrt(-1))  -- Això provocarà un error d'assert
  • Pros: Ajuda a validar supòsits.
  • Cons: Atura l'execució del programa si la condició falla.

Biblioteca de Depuració

Lua té una biblioteca de depuració integrada que proporciona diverses funcions per inspeccionar i manipular l'execució dels programes Lua.

debug.traceback

La funció debug.traceback retorna una cadena amb la pila de crides.

local function faultyFunction()
    error("Ha ocorregut un error")
end

local function wrapperFunction()
    faultyFunction()
end

local status, err = pcall(wrapperFunction)
if not status then
    print(debug.traceback(err))
end
  • Pros: Proporciona informació detallada sobre la pila de crides.
  • Cons: Requereix entendre la pila de crides.

Tècniques Avançades de Depuració

Ús de `debug.sethook`

La funció debug.sethook permet establir una funció hook que es cridarà durant certs esdeveniments (per exemple, crides de funció, retorns i execucions de línia).

local function hook(event, line)
    local info = debug.getinfo(2)
    print(event, info.name, line)
end

debug.sethook(hook, "crl")

local function testFunction()
    local x = 10
    x = x + 1
    return x
end

print(testFunction())
  • Pros: Eina poderosa per traçar i fer perfilatge.
  • Cons: Pot ser complex i pot afectar el rendiment.

Ús de Depuradors Externs

Per a projectes grans, utilitzar un depurador extern pot ser molt útil. Alguns depuradors Lua populars inclouen:

  • ZeroBrane Studio: Un IDE amb suport de depuració integrat per a Lua.
  • MobDebug: Un depurador remot per a Lua.

Exemple amb ZeroBrane Studio

  1. Instal·leu ZeroBrane Studio.
  2. Obriu el vostre script Lua a ZeroBrane Studio.
  3. Establiu punts de ruptura fent clic als números de línia.
  4. Executeu el script en mode de depuració per passar pel codi, inspeccionar variables i avaluar expressions.
  • Pros: Proporciona una interfície gràfica i funcions avançades de depuració.
  • Cons: Requereix instal·lació i configuració.

Conclusió

La depuració és una part crucial del procés de desenvolupament. En dominar diverses tècniques de depuració, podeu identificar i corregir errors en el vostre codi Lua de manera eficient. Comenceu amb tècniques bàsiques com les declaracions print i gradualment passeu a mètodes més avançats com l'ús de la biblioteca de depuració i depuradors externs. Bona depuració!
