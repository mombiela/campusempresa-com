Introducció

En aquesta secció del curs sobre Algoritmes, explorarem les estratègies de disseny d'algoritmes. Aquestes estratègies són fonamentals per resoldre problemes de manera eficient i efectiva. Aprendrem sobre diferents enfocaments i tècniques que es poden aplicar per dissenyar algoritmes robustos i optimitzats.

Estratègies Comunes de Disseny d'Algoritmes

1. Divideix i Venceràs

  • Descripció: Aquesta estratègia implica dividir un problema en subproblemes més petits, resoldre cada subproblema de manera recursiva i després combinar les solucions dels subproblemes per resoldre el problema original.
  • Exemple Clàssic: Algoritme d'ordenació Merge Sort.
def merge_sort(arr):
    if len(arr) > 1:
        mid = len(arr) // 2
        L = arr[:mid]
        R = arr[mid:]

        merge_sort(L)
        merge_sort(R)

        i = j = k = 0

        while i < len(L) and j < len(R):
            if L[i] < R[j]:
                arr[k] = L[i]
                i += 1
            else:
                arr[k] = R[j]
                j += 1
            k += 1

        while i < len(L):
            arr[k] = L[i]
            i += 1
            k += 1

        while j < len(R):
            arr[k] = R[j]
            j += 1
            k += 1

arr = [12, 11, 13, 5, 6, 7]
merge_sort(arr)
print("Sorted array is:", arr)

2. Programació Dinàmica

  • Descripció: Aquesta tècnica s'utilitza per resoldre problemes complexos dividint-los en subproblemes més simples i emmagatzemant els resultats d'aquests subproblemes per evitar càlculs redundants.
  • Exemple Clàssic: Problema de la motxilla (Knapsack Problem).
def knapsack(W, wt, val, n):
    K = [[0 for x in range(W + 1)] for x in range(n + 1)]

    for i in range(n + 1):
        for w in range(W + 1):
            if i == 0 o w == 0:
                K[i][w] = 0
            elif wt[i-1] <= w:
                K[i][w] = max(val[i-1] + K[i-1][w-wt[i-1]], K[i-1][w])
            else:
                K[i][w] = K[i-1][w]

    return K[n][W]

val = [60, 100, 120]
wt = [10, 20, 30]
W = 50
n = len(val)
print(knapsack(W, wt, val, n))

3. Algoritmes Greedy

  • Descripció: Aquesta estratègia construeix una solució pas a pas, triant en cada pas l'opció que sembla ser la millor en aquell moment.
  • Exemple Clàssic: Problema de la moneda (Coin Change Problem).
def coin_change(coins, amount):
    coins.sort(reverse=True)
    count = 0
    for coin in coins:
        while amount >= coin:
            amount -= coin
            count += 1
    return count

coins = [1, 2, 5, 10, 20, 50]
amount = 93
print(coin_change(coins, amount))

4. Backtracking

  • Descripció: Aquesta tècnica implica explorar totes les possibles solucions i retrocedir quan es detecta que una solució no és vàlida.
  • Exemple Clàssic: Problema de les N-Reines.
def is_safe(board, row, col):
    for i in range(col):
        if board[row][i] == 1:
            return False
    for i, j in zip(range(row, -1, -1), range(col, -1, -1)):
        if board[i][j] == 1:
            return False
    for i, j in zip(range(row, len(board), 1), range(col, -1, -1)):
        if board[i][j] == 1:
            return False
    return True

def solve_n_queens(board, col):
    if col >= len(board):
        return True
    for i in range(len(board)):
        if is_safe(board, i, col):
            board[i][col] = 1
            if solve_n_queens(board, col + 1):
                return True
            board[i][col] = 0
    return False

n = 4
board = [[0] * n for _ in range(n)]
if solve_n_queens(board, 0):
    for row in board:
        print(row)
else:
    print("No solution found")

Comparació d'Estratègies

| Estratègia | Avantatges | Desavantatges | |----------------------|-------------------------------------------------|------------------------------------------------| | Divideix i Venceràs | Eficient per a problemes grans i complexos | Pot tenir sobrecàrrega de recursió | | Programació Dinàmica | Evita càlculs redundants | Requereix espai addicional per emmagatzematge | | Algoritmes Greedy | Simple i fàcil d'implementar | No sempre produeix la solució òptima | | Backtracking | Troba totes les solucions possibles | Pot ser molt lent per a problemes grans |

Conclusió

Les estratègies de disseny d'algoritmes són eines poderoses que permeten abordar una àmplia varietat de problemes de manera eficient. Cada estratègia té els seus propis avantatges i desavantatges, i l'elecció de l'estratègia adequada depèn del problema específic que s'estigui tractant de resoldre. A través de la pràctica i l'experiència, els professionals de TI poden aprendre a seleccionar i aplicar l'estratègia més adequada per a cada situació.
