Introducció

En el desenvolupament d'algoritmes, l'elecció d'estructures de dades adequades és crucial per optimitzar el rendiment i l'eficiència del programari. Aquest tema abordarà les estructures de dades més comunes i eficients, explicant quan i com utilitzar-les.

Conceptes Clau

  • Estructures de Dades: Formes d'organitzar i emmagatzemar dades perquè puguin ser utilitzades de manera eficient.
  • Eficiència: Mesura de rendiment que pot incloure temps d'execució (complexitat temporal) i ús de memòria (complexitat espacial).
  • Algoritmes: Conjunt d'instruccions que s'executen en una estructura de dades per realitzar una tasca específica.

Tipus d'Estructures de Dades

Arrays (Arranjaments)

  • Descripció: Col·lecció d'elements, tots del mateix tipus, emmagatzemats en ubicacions contigües de memòria.
  • Avantatges:
    • Accés ràpid a elements mitjançant índexs.
    • Eficient en termes d'ús de memòria.
  • Desavantatges:
    • Mida fixa.
    • Costós en termes de temps per a operacions d'inserció i eliminació.
# Exemple d'ús d'un array en Python
array = [1, 2, 3, 4, 5]
print(array[2])  # Accés ràpid al tercer element

Llistes Enllaçades

  • Descripció: Col·lecció de nodes on cada node conté una dada i una referència (enllaç) al següent node.
  • Avantatges:
    • Mida dinàmica.
    • Insercions i eliminacions eficients.
  • Desavantatges:
    • Accés seqüencial a elements (no indexat).
    • Major ús de memòria a causa dels enllaços addicionals.
# Exemple d'una llista enllaçada en Python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        last = self.head
        while last.next:
            last = last.next
        last.next = new_node

# Ús de la llista enllaçada
ll = LinkedList()
ll.append(1)
ll.append(2)
ll.append(3)

Piles (Stacks)

  • Descripció: Col·lecció d'elements amb accés LIFO (Last In, First Out).
  • Avantatges:
    • Operacions d'inserció i eliminació ràpides.
  • Desavantatges:
    • Accés limitat a només l'element superior.
# Exemple d'una pila en Python usant llistes
stack = []
stack.append(1)  # Push
stack.append(2)
stack.append(3)
print(stack.pop())  # Pop, retorna 3

Cues (Queues)

  • Descripció: Col·lecció d'elements amb accés FIFO (First In, First Out).
  • Avantatges:
    • Operacions d'inserció i eliminació ràpides.
  • Desavantatges:
    • Accés limitat a només el primer i últim element.
# Exemple d'una cua en Python usant collections.deque
from collections import deque
queue = deque()
queue.append(1)  # Enqueue
queue.append(2)
queue.append(3)
print(queue.popleft())  # Dequeue, retorna 1

Arbres

  • Descripció: Estructura jeràrquica amb nodes connectats per arestes, on cada node té un valor i referències a nodes fills.
  • Avantatges:
    • Eficient per a operacions de cerca, inserció i eliminació.
  • Desavantatges:
    • Complexitat en la implementació i manteniment del balanç.
# Exemple d'un arbre binari en Python
class TreeNode:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None

# Creació d'un arbre binari
root = TreeNode(10)
root.left = TreeNode(5)
root.right = TreeNode(15)

Taules Hash

  • Descripció: Estructura que associa claus amb valors mitjançant una funció hash.
  • Avantatges:
    • Accés, inserció i eliminació molt ràpids.
  • Desavantatges:
    • Possibles col·lisions que requereixen maneig addicional.
# Exemple d'una taula hash en Python usant diccionaris
hash_table = {}
hash_table["key1"] = "value1"
hash_table["key2"] = "value2"
print(hash_table["key1"])  # Accés ràpid al valor associat a "key1"

Comparació d'Estructures de Dades

| Estructura de Dades | Accés | Inserció | Eliminació | Ús de Memòria | |---------------------|-------|----------|------------|---------------| | Array | O(1) | O(n) | O(n) | Baix | | Llista Enllaçada | O(n) | O(1) | O(1) | Alt | | Pila | O(1) | O(1) | O(1) | Baix | | Cua | O(1) | O(1) | O(1) | Baix | | Arbre | O(log n) | O(log n) | O(log n) | Variable | | Taula Hash | O(1) | O(1) | O(1) | Variable |

Conclusió

L'elecció de l'estructura de dades adequada és essencial per a l'eficiència dels algoritmes. Cada estructura té els seus propis avantatges i desavantatges, i és important comprendre'ls per prendre decisions informades en el disseny de programari. Practicar amb exemples i exercicis ajudarà a consolidar aquests conceptes i a aplicar-los efectivament en situacions del món real.
