El Projecte Sobre nosaltres Contribuir Donacions Llicència

La paral·lelització d'algorismes és una tècnica utilitzada per millorar el rendiment dels programes dividint les tasques en sub-tasques que poden ser executades simultàniament en múltiples processadors. Aquest enfocament és essencial en l'era de la computació d'alt rendiment i és àmpliament utilitzat en diverses aplicacions, des de la simulació científica fins al processament de grans volums de dades.

Conceptes Clau

  • Paral·lelisme: Execució simultània de múltiples tasques.
  • Fils (Threads): Unitats més petites d'un procés que poden executar-se en paral·lel.
  • Processos: Programes en execució que poden contenir múltiples fils.
  • Granularitat: Mida de les tasques en les quals es divideix un algorisme per a la seva paral·lelització.
  • Sincronització: Coordinació dels fils o processos per assegurar resultats correctes.
  • Acceleració (Speedup): Mesura del rendiment millorat mitjançant la paral·lelització.

Tipus de Paral·lelisme

  • Paral·lelisme de Dades: Divideix les dades en parts que es processen simultàniament.
  • Paral·lelisme de Tasques: Divideix les tasques en sub-tasques independents que s'executen en paral·lel.

Exemple de Paral·lelització d'un Algorisme

Algorisme Seqüencial: Suma d'un Array

def suma_sequencial(arr):
    suma = 0
    for num in arr:
        suma += num
    return suma

arr = [1, 2, 3, 4, 5]
resultat = suma_sequencial(arr)
print(f"Suma seqüencial: {resultat}")

Algorisme Paral·lelitzat: Suma d'un Array

Usarem la biblioteca multiprocessing en Python per paral·lelitzar la suma d'un array.

import multiprocessing

def suma_parcial(arr):
    return sum(arr)

def suma_paral·lela(arr):
    num_processos = multiprocessing.cpu_count()
    pool = multiprocessing.Pool(processes=num_processos)
    
    # Dividir l'array en parts iguals
    mida_part = len(arr) // num_processos
    parts = [arr[i * mida_part:(i + 1) * mida_part] for i in range(num_processos)]
    
    # Executar la suma parcial en paral·lel
    resultats = pool.map(suma_parcial, parts)
    
    # Sumar els resultats parcials
    suma_total = sum(resultats)
    return suma_total

arr = [1, 2, 3, 4, 5]
resultat = suma_paral·lela(arr)
print(f"Suma paral·lela: {resultat}")

Explicació del Codi

  1. Importar la Biblioteca: multiprocessing és una biblioteca estàndard de Python que permet la creació de processos.
  2. Definir la Funció de Suma Parcial: suma_parcial pren una part de l'array i retorna la suma d'aquesta part.
  3. Dividir l'Array: L'array es divideix en parts iguals segons el nombre de processadors disponibles.
  4. Crear un Pool de Processos: multiprocessing.Pool crea un pool de processos que executen les tasques en paral·lel.
  5. Executar la Suma Parcial en Paral·lel: pool.map distribueix les parts de l'array als processos en el pool.
  6. Combinar els Resultats: Els resultats parcials es sumen per obtenir el resultat final.

Comparació: Seqüencial vs Paral·lel

| Aspecte | Algorisme Seqüencial | Algorisme Paral·lel | |------------------|----------------------|----------------------------| | Complexitat | Baixa | Alta | | Rendiment | Limitat per CPU | Millorat amb múltiples CPU | | Ús de Recursos | Baix | Alt | | Escalabilitat | Limitada | Alta |

Conclusió

La paral·lelització d'algorismes és una tècnica poderosa per millorar el rendiment dels programes, especialment en sistemes amb múltiples processadors. Encara que pot augmentar la complexitat del codi, els beneficis en termes de velocitat i eficiència poden ser significatius. És crucial entendre els conceptes clau i les tècniques de paral·lelització per aplicar aquesta estratègia de manera efectiva en projectes reals.
