Introducció

En l'anàlisi d'algoritmes, és crucial entendre com es comporten en diferents escenaris. Això s'avalua mitjançant els casos de complexitat: millor cas, pitjor cas i cas promig. Aquest coneixement permet als professionals de TI seleccionar i optimitzar algoritmes de manera més efectiva.

Conceptes Clau

  • Millor Cas: Descriu l'escenari en el qual l'algoritme realitza la menor quantitat de treball possible.
  • Pitjor Cas: Descriu l'escenari en el qual l'algoritme realitza la major quantitat de treball possible.
  • Cas Promig: Descriu el comportament esperat de l'algoritme en un escenari típic o promig.

Millor Cas

El millor cas es refereix a la situació en la qual l'algoritme té el rendiment més eficient. Aquest escenari és útil per entendre el potencial màxim d'eficiència d'un algoritme.

Exemple de Codi: Cerca Lineal

def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

# Millor cas: l'element està a la primera posició
arr = [1, 2, 3, 4, 5]
target = 1
print(linear_search(arr, target))  # Output: 0

En aquest exemple, el millor cas ocorre quan el target està a la primera posició de l'array, resultant en una complexitat de O(1).

Pitjor Cas

El pitjor cas es refereix a la situació en la qual l'algoritme té el rendiment menys eficient. Aquest escenari és crucial per entendre les limitacions i el comportament en situacions extremes.

Exemple de Codi: Cerca Lineal

def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

# Pitjor cas: l'element no està a l'array
arr = [1, 2, 3, 4, 5]
target = 6
print(linear_search(arr, target))  # Output: -1

En aquest exemple, el pitjor cas ocorre quan el target no està a l'array, resultant en una complexitat de O(n).

Cas Promig

El cas promig es refereix a la situació en la qual l'algoritme té un rendiment mitjà, considerant totes les possibles entrades. Aquest escenari és útil per entendre el comportament general de l'algoritme.

Exemple de Codi: Cerca Lineal

def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

# Cas promig: l'element està en una posició intermèdia
arr = [1, 2, 3, 4, 5]
target = 3
print(linear_search(arr, target))  # Output: 2

En aquest exemple, el cas promig ocorre quan el target està en una posició intermèdia de l'array, resultant en una complexitat de O(n/2), que es simplifica a O(n).

Comparació de Casos

| Cas | Descripció | Complexitat | |--------------|--------------------------------------------------|-------------| | Millor Cas | L'algoritme realitza la menor quantitat de treball possible | O(1) | | Pitjor Cas | L'algoritme realitza la major quantitat de treball possible | O(n) | | Cas Promig | L'algoritme realitza una quantitat de treball mitjana | O(n) |

Conclusió

Comprendre els diferents casos de complexitat és essencial per avaluar el rendiment d'un algoritme en diverses situacions. El millor cas mostra el potencial màxim d'eficiència, el pitjor cas revela les limitacions i el cas promig proporciona una visió general del comportament esperat. Aquest coneixement és fonamental per a la selecció i optimització d'algoritmes en el desenvolupament de programari.
