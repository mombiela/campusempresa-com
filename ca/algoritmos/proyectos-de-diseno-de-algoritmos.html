Introducció

El disseny d'algoritmes és una habilitat fonamental per a qualsevol professional de TI. Aquest tema se centra en els principis i tècniques essencials per dissenyar algoritmes eficients i efectius. Al llarg d'aquesta secció, explorarem diferents enfocaments i estratègies per resoldre problemes mitjançant algoritmes, proporcionant exemples clars i exercicis pràctics.

Conceptes Clau

  • Algoritme: Una seqüència finita d'instruccions ben definides i ordenades que resolen un problema específic.
  • Eficiència: Mesura dels recursos computacionals utilitzats per un algoritme, típicament temps i espai.
  • Complexitat Temporal: Temps que triga un algoritme a completar-se, generalment expressat en notació Big O.
  • Complexitat Espacial: Quantitat de memòria que utilitza un algoritme durant la seva execució.
  • Recursió: Tècnica on una funció es crida a si mateixa per resoldre subproblemes més petits.
  • Dividir i Vèncer: Estratègia que divideix un problema en subproblemes més petits, els resol de manera recursiva i combina les solucions.

Estratègies de Disseny d'Algoritmes

Divideix i Vènceràs

Aquesta estratègia implica dividir el problema en subproblemes més petits, resoldre cada subproblema de manera recursiva i després combinar les solucions dels subproblemes per resoldre el problema original.

Exemple: Algoritme d'Ordenació Ràpida (Quicksort)

def quicksort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr si x == pivot]
    right = [x for x in arr if x > pivot]
    return quicksort(left) + middle + quicksort(right)

# Exemple d'ús
arr = [3, 6, 8, 10, 1, 2, 1]
print(quicksort(arr))
  • Explicació:
    • Si la llista té un sol element o està buida, es retorna tal qual.
    • Es selecciona un pivot (en aquest cas, l'element central).
    • Es creen tres llistes: left amb elements menors que el pivot, middle amb elements iguals al pivot i right amb elements majors que el pivot.
    • S'aplica recursivament quicksort a les llistes left i right i es combinen els resultats.

Programació Dinàmica

Aquesta tècnica s'utilitza per resoldre problemes complexos dividint-los en subproblemes més simples i emmagatzemant els resultats d'aquests subproblemes per evitar càlculs redundants.

Exemple: Problema de la Motxilla (Knapsack Problem)

def knapsack(weights, values, W):
    n = len(weights)
    dp = [[0 for _ in range(W + 1)] for _ in range(n + 1)]

    for i in range(n + 1):
        for w in range(W + 1):
            if i == 0 or w == 0:
                dp[i][w] = 0
            elif weights[i-1] <= w:
                dp[i][w] = max(values[i-1] + dp[i-1][w-weights[i-1]], dp[i-1][w])
            else:
                dp[i][w] = dp[i-1][w]

    return dp[n][W]

# Exemple d'ús
weights = [1, 2, 3]
values = [10, 15, 40]
W = 6
print(knapsack(weights, values, W))
  • Explicació:
    • dp[i][w] representa el valor màxim que es pot obtenir amb els primers i elements i una capacitat de w.
    • Si no s'inclou l'element i, el valor és el mateix que dp[i-1][w].
    • Si s'inclou l'element i, el valor és values[i-1] + dp[i-1][w-weights[i-1]].
    • Es pren el màxim de les dues opcions anteriors.

Comparació d'Estratègies

| Estratègia | Avantatges | Desavantatges | |-----------------------|-----------------------------------------------|---------------------------------------------| | Divideix i Vènceràs | Fàcil d'entendre i aplicar | Pot ser ineficient si no es tria bé el pivot | | Programació Dinàmica | Evita càlculs redundants, eficient | Pot consumir molta memòria |

Conclusió

El disseny d'algoritmes és una habilitat essencial per resoldre problemes de manera eficient. Les estratègies com "Divideix i Vènceràs" i "Programació Dinàmica" són eines poderoses que, quan s'apliquen correctament, poden millorar significativament el rendiment dels algoritmes. Practicar amb exemples i exercicis és crucial per dominar aquestes tècniques i aplicar-les efectivament en situacions del món real.
