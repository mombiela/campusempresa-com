Introducció a les ACLs

Les Llistes de Control d'Accés (ACLs) a Redis s'utilitzen per gestionar permisos i controlar l'accés a diferents comandes i claus dins d'una instància de Redis. Les ACLs ajuden a assegurar Redis definint quines accions poden realitzar els usuaris.

Conceptes Clau

  • Usuaris: Entitats que interactuen amb la instància de Redis.
  • Regles: Permisos assignats als usuaris, definint quines comandes i claus poden accedir.
  • Usuari per Defecte: L'usuari que es crea per defecte quan s'inicialitza Redis.

Configuració de les ACLs

Per començar a utilitzar les ACLs, cal configurar els usuaris i els seus permisos. Això es pot fer utilitzant la comanda ACL SETUSER.

Creació i Configuració d'Usuaris

Per crear un nou usuari i establir permisos, utilitza la següent comanda:

ACL SETUSER alice on >password ~* +@all
  • alice: El nom d'usuari.
  • on: Habilita l'usuari.
  • >password: Estableix la contrasenya per a l'usuari.
  • ~*: Permet l'accés a totes les claus.
  • +@all: Concedeix tots els permisos de comanda.

Exemple: Creació d'un Usuari Només de Lectura

Per crear un usuari que només pugui llegir dades:

ACL SETUSER bob on >password ~* +@read
  • bob: El nom d'usuari.
  • on: Habilita l'usuari.
  • >password: Estableix la contrasenya per a l'usuari.
  • ~*: Permet l'accés a totes les claus.
  • +@read: Concedeix permisos de comandes només de lectura.

Gestió de Permisos d'Usuari

Els permisos es poden ajustar especificant comandes exactes i patrons de claus.

Concessió de Permisos de Comandes Específiques

Per permetre que un usuari executi només comandes específiques:

ACL SETUSER charlie on >password ~* +get +set
  • +get: Permet la comanda GET.
  • +set: Permet la comanda SET.

Restricció d'Accés a Claus Específiques

Per restringir un usuari a accedir només a certes claus:

ACL SETUSER dave on >password ~key1:* +@all
  • ~key1:*: Permet l'accés a claus que coincideixin amb el patró key1:*.

Visualització i Modificació de les ACLs

Pots veure la configuració actual de les ACLs i modificar-la segons sigui necessari.

Llistat de Tots els Usuaris

Per llistar tots els usuaris:

ACL LIST

Obtenció de Detalls d'un Usuari

Per obtenir els detalls d'un usuari específic:

ACL GETUSER alice

Eliminació d'un Usuari

Per eliminar un usuari:

ACL DELUSER alice

Exemple: Configuració Completa d'una ACL

Creem un usuari amb un conjunt complet de permisos:

ACL SETUSER eve on >password ~data:* +get +set +del
  • eve: El nom d'usuari.
  • on: Habilita l'usuari.
  • >password: Estableix la contrasenya per a l'usuari.
  • ~data:*: Permet l'accés a claus que coincideixin amb el patró data:*.
  • +get: Permet la comanda GET.
  • +set: Permet la comanda SET.
  • +del: Permet la comanda DEL.

Conclusió

Les Llistes de Control d'Accés (ACLs) a Redis proporcionen un mecanisme robust per gestionar els permisos dels usuaris i assegurar la teva instància de Redis. Comprenent com crear i configurar usuaris, establir permisos específics i gestionar l'accés a comandes i claus, pots assegurar que el teu entorn Redis sigui tant segur com eficient.

Resum

  • Les ACLs ajuden a gestionar i assegurar l'accés a Redis.
  • Es poden crear i configurar usuaris amb permisos específics.
  • Les comandes i patrons de claus es poden controlar finament.
  • Utilitza ACL LIST, ACL GETUSER i ACL DELUSER per gestionar usuaris.

Dominant les ACLs, pots millorar la seguretat i el control de les teves instàncies de Redis, assegurant que només els usuaris autoritzats puguin realitzar accions específiques.
