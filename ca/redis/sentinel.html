Redis Sentinel és un sistema dissenyat per ajudar a gestionar instàncies de Redis, proporcionant alta disponibilitat i capacitats de monitorització. Aquest tema cobrirà els conceptes bàsics de Sentinel, la seva configuració i com utilitzar-lo per assegurar que les teves instàncies de Redis siguin resilients i altament disponibles.

Introducció a Redis Sentinel

Redis Sentinel proporciona diverses característiques clau:

  • Monitorització: Sentinel comprova constantment si les teves instàncies mestre i rèplica funcionen com s'espera.
  • Notificació: Sentinel pot notificar els administradors del sistema o altres sistemes via una API quan alguna cosa no va bé amb una de les instàncies de Redis monitoritzades.
  • Failover Automàtic: Si un mestre no funciona com s'espera, Sentinel pot iniciar un procés de failover on una rèplica és promoguda a mestre.
  • Proveïdor de Configuració: Sentinel actua com una font d'autoritat perquè els clients descobreixin el mestre i les rèpliques actuals.

Configuració de Redis Sentinel

Per configurar Redis Sentinel, necessites configurar instàncies de Sentinel i assegurar-te que poden comunicar-se entre elles i amb les instàncies de Redis que estan monitoritzant.

Fitxer de Configuració de Sentinel

Un fitxer de configuració típic de Sentinel (sentinel.conf) inclou els següents paràmetres clau:

# Exemple de sentinel.conf

# Nom del mestre monitoritzat
sentinel monitor mymaster 127.0.0.1 6379 2

# Autenticació per al mestre
sentinel auth-pass mymaster your_password

# Com de sovint fer ping al mestre i a les rèpliques
sentinel down-after-milliseconds mymaster 5000

# Quant de temps esperar abans de fer failover
sentinel failover-timeout mymaster 60000

# Nombre de rèpliques a reconfigurar després d'un failover
sentinel parallel-syncs mymaster 1
  • sentinel monitor <master-name> <ip> <port> <quorum>: Defineix el mestre a monitoritzar. El quòrum és el nombre de Sentinels que han d'estar d'acord abans d'iniciar un failover.
  • sentinel auth-pass <master-name> <password>: Estableix la contrasenya per autenticar-se amb el mestre.
  • sentinel down-after-milliseconds <master-name> <milliseconds>: Temps d'espera abans de considerar el mestre com a caigut.
  • sentinel failover-timeout <master-name> <milliseconds>: Temps d'espera abans d'iniciar un nou failover.
  • sentinel parallel-syncs <master-name> <number>: Nombre de rèpliques a reconfigurar en paral·lel durant un failover.

Executar Sentinel

Per iniciar una instància de Sentinel, utilitza la següent comanda:

redis-server /path/to/sentinel.conf --sentinel

Monitorització i Notificació

Sentinel monitoritza contínuament la salut de les instàncies de Redis. Si una instància mestre es considera caiguda, Sentinel notificarà els administradors i altres sistemes configurats per rebre notificacions.

Exemple de Monitorització

# Sortida del log de Sentinel
[12345] 01 Jan 01:00:00.000 * +monitor master mymaster 127.0.0.1 6379 quorum 2
[12345] 01 Jan 01:00:05.000 * +sdown master mymaster 127.0.0.1 6379
[12345] 01 Jan 01:00:10.000 * +odown master mymaster 127.0.0.1 6379 #quorum 2
[12345] 01 Jan 01:00:15.000 * +new-epoch 1
[12345] 01 Jan 01:00:20.000 * +try-failover master mymaster 127.0.0.1 6379

Failover Automàtic

Quan un mestre es considera caigut, Sentinel iniciarà un procés de failover. Això implica promoure una de les rèpliques a mestre i reconfigurar les altres rèpliques per replicar des del nou mestre.

Exemple de Failover

# Sortida del log de Sentinel durant el failover
[12345] 01 Jan 01:00:20.000 * +failover-state-select-slave master mymaster 127.0.0.1 6379
[12345] 01 Jan 01:00:25.000 * +selected-slave slave 127.0.0.1:6380
[12345] 01 Jan 01:00:30.000 * +failover-state-send-slaveof-noone slave 127.0.0.1:6380
[12345] 01 Jan 01:00:35.000 * +failover-end master mymaster 127.0.0.1 6379
[12345] 01 Jan 01:00:40.000 * +switch-master mymaster 127.0.0.1 6379 127.0.0.1 6380

Proveïdor de Configuració

Els clients poden utilitzar Sentinel per descobrir el mestre i les rèpliques actuals. Això és particularment útil per a aplicacions que necessiten connectar-se al mestre per a operacions d'escriptura i a les rèpliques per a operacions de lectura.

Exemple d'Ús de Sentinel per Descobrir el Mestre

import redis
from redis.sentinel import Sentinel

sentinel = Sentinel([('localhost', 26379)], socket_timeout=0.1)
master = sentinel.master_for('mymaster', socket_timeout=0.1)
slave = sentinel.slave_for('mymaster', socket_timeout=0.1)

# Escriure al mestre
master.set('foo', 'bar')

# Llegir de la rèplica
print(slave.get('foo'))

Conclusió

Redis Sentinel és una eina poderosa per assegurar alta disponibilitat i monitorització d'instàncies de Redis. Entenent la seva configuració i capacitats, pots configurar un entorn Redis resilient que pugui gestionar fallades de manera elegant. La capacitat de Sentinel per monitoritzar, notificar i fer failover automàticament el converteix en un component essencial per a qualsevol desplegament de Redis en producció.
