Introducció

En aquesta secció, tractarem com presentar les teves tasques i projectes per a revisió en el curs de Redis. També discutirem el procés de revisió i què pots esperar en termes de retroalimentació i qualificacions.

Directrius de Presentació

Per assegurar un procés de presentació fluid, segueix aquestes directrius:

  • Formats de Fitxer: Presenta els teus fitxers de codi en format .py (Python) o .txt llevat que s'especifiqui el contrari.
  • Convencions de Nomenclatura: Nomena els teus fitxers de manera clara, per exemple, assignment1.py o project_final.txt.
  • Documentació: Inclou comentaris en el teu codi per explicar la teva lògica i enfocament.
  • Plataforma de Presentació: Utilitza el sistema de gestió d'aprenentatge (LMS) designat per a totes les presentacions.

Exemple d'una Presentació de Codi Ben Documentada

# assignment1.py

import redis

# Connectar-se al servidor Redis
client = redis.StrictRedis(host='localhost', port=6379, db=0)

# Establir una parella clau-valor
client.set('name', 'Alice')

# Recuperar el valor per a la clau 'name'
name = client.get('name')
print(name)  # El resultat hauria de ser b'Alice'

Procés de Revisió

Un cop hagis presentat la teva tasca o projecte, passarà pel següent procés de revisió:

  1. Cribratge Inicial: Es comprova que la presentació sigui completa i que segueixi les directrius.
  2. Revisió de Codi: Es revisa el codi per a la seva correcció, eficiència i llegibilitat.
  3. Retroalimentació: Es proporciona una retroalimentació detallada, destacant punts forts i àrees de millora.
  4. Qualificació: Es dóna una qualificació basada en criteris predefinits.

Criteris Clau de Revisió

  • Correcció: El codi produeix els resultats esperats?
  • Eficiència: El codi està optimitzat per al rendiment?
  • Llegibilitat: El codi és fàcil d'entendre i està ben documentat?
  • Adherència a les Directrius: La presentació segueix les directrius proporcionades?

Exemple de Retroalimentació

Aquí tens un exemple del tipus de retroalimentació que podries rebre:

  • Punts Forts:

    • El codi es connecta correctament al servidor Redis i realitza les operacions requerides.
    • Bon ús dels comentaris per explicar cada pas.

  • Àrees de Millora:

    • Considera manejar excepcions per fer el teu codi més robust.
    • Utilitza noms de variables més descriptius per a una millor llegibilitat.

Exemple de Codi Millorat Basat en la Retroalimentació

# assignment1_improved.py

import redis

try:
    # Connectar-se al servidor Redis
    client = redis.StrictRedis(host='localhost', port=6379, db=0)

    # Establir una parella clau-valor
    client.set('user_name', 'Alice')

    # Recuperar el valor per a la clau 'user_name'
    user_name = client.get('user_name')
    print(user_name)  # El resultat hauria de ser b'Alice'

except redis.ConnectionError as e:
    print(f"No s'ha pogut connectar a Redis: {e}")

Conclusió

Presentar i revisar tasques és una part crucial del procés d'aprenentatge. Seguint les directrius de presentació i prestant atenció a la retroalimentació, pots millorar la teva comprensió i habilitats en Redis. Recorda, l'objectiu no és només completar les tasques sinó aprendre i créixer com a programador.
