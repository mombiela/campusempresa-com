Introducció

En aquesta secció, explorarem diversos estudis de cas per entendre com s'utilitza Redis en aplicacions del món real. Examinant aquests exemples, obtindreu informació sobre implementacions pràctiques i millors pràctiques per utilitzar Redis de manera efectiva.

Estudi de Cas 1: Emmagatzematge en Memòria Cau en una Aplicació Web

Visió General

L'emmagatzematge en memòria cau és un dels casos d'ús més comuns per a Redis. En aquest estudi de cas, veurem com es pot utilitzar Redis per emmagatzemar en memòria cau dades accedides freqüentment en una aplicació web per millorar el rendiment.

Conceptes Clau

  • Cache Hit: Quan les dades sol·licitades es troben a la memòria cau.
  • Cache Miss: Quan les dades sol·licitades no es troben a la memòria cau i s'han de recuperar de l'emmagatzematge de dades principal.
  • TTL (Time to Live): La durada durant la qual un element emmagatzemat en memòria cau roman a la memòria cau.

Implementació

Considerem una aplicació web senzilla que recupera perfils d'usuari d'una base de dades. Utilitzarem Redis per emmagatzemar en memòria cau aquests perfils.

Exemple Pas a Pas

  1. Connexió a Redis:

    import redis

r = redis.Redis(host='localhost', port=6379, db=0)

  2. Recuperació de Dades amb Emmagatzematge en Memòria Cau:

    def get_user_profile(user_id):
    # Intentar obtenir el perfil d'usuari de la memòria cau de Redis
    profile = r.get(f"user:{user_id}")

    if profile:
        print("Cache hit")
        return profile
    else:
        print("Cache miss")
        # Simular la recuperació de la base de dades
        profile = fetch_from_database(user_id)
        # Emmagatzemar el perfil a Redis amb un TTL d'1 hora
        r.setex(f"user:{user_id}", 3600, profile)
        return profile

def fetch_from_database(user_id):
    # Simular una trucada a la base de dades
    return f"Dades del perfil d'usuari per a {user_id}"

  3. Prova de la Funció:

    user_id = 1
print(get_user_profile(user_id))  # Cache miss, recupera de la BD
print(get_user_profile(user_id))  # Cache hit, recupera de Redis

Resum

En aquest estudi de cas, hem demostrat com es pot utilitzar Redis per emmagatzemar en memòria cau perfils d'usuari en una aplicació web, reduint la càrrega a la base de dades principal i millorant els temps de resposta.

Estudi de Cas 2: Analítica en Temps Real

Visió General

Redis també s'utilitza àmpliament per a l'analítica en temps real gràcies al seu accés ràpid a les dades i el suport per a diverses estructures de dades com llistes, conjunts i conjunts ordenats.

Conceptes Clau

  • Conjunts Ordenats: Una estructura de dades de Redis que manté els elements en un ordre ordenat.
  • Processament de Dades en Temps Real: La capacitat de processar dades a mesura que arriben.

Implementació

Construirem un sistema d'analítica en temps real senzill per fer un seguiment de les visualitzacions de pàgines en un lloc web.

Exemple Pas a Pas

  1. Connexió a Redis:

    import redis

r = redis.Redis(host='localhost', port=6379, db=0)

  2. Seguiment de Visualitzacions de Pàgines:

    def track_page_view(page):
    # Incrementar el recompte de visualitzacions de pàgina
    r.zincrby('page_views', 1, page)

def get_top_pages(n):
    # Obtenir les n pàgines principals per recompte de visualitzacions
    return r.zrevrange('page_views', 0, n-1, withscores=True)

  3. Prova de les Funcions:

    track_page_view('home')
track_page_view('about')
track_page_view('home')

print(get_top_pages(2))  # [('home', 2.0), ('about', 1.0)]

Resum

En aquest estudi de cas, hem utilitzat conjunts ordenats de Redis per fer un seguiment i recuperar les pàgines més vistes en un lloc web en temps real, demostrant la capacitat de Redis per a l'analítica en temps real.

Estudi de Cas 3: Gestió de Sessions

Visió General

Redis s'utilitza sovint per a la gestió de sessions en aplicacions web gràcies a la seva velocitat i suport per a TTL, que ajuda a gestionar l'expiració de sessions.

Conceptes Clau

  • Emmagatzematge de Sessions: Emmagatzemar dades de sessió d'usuari.
  • Expiració de Sessions: Eliminar automàticament les sessions després d'un cert període d'inactivitat.

Implementació

Crearem un sistema de gestió de sessions senzill utilitzant Redis.

Exemple Pas a Pas

  1. Connexió a Redis:

    import redis

r = redis.Redis(host='localhost', port=6379, db=0)

  2. Gestió de Sessions:

    def create_session(user_id, session_data):
    # Emmagatzemar dades de sessió amb un TTL de 30 minuts
    r.setex(f"session:{user_id}", 1800, session_data)

def get_session(user_id):
    # Recuperar dades de sessió
    return r.get(f"session:{user_id}")

def delete_session(user_id):
    # Eliminar dades de sessió
    r.delete(f"session:{user_id}")

  3. Prova de les Funcions:

    user_id = 1
session_data = "Dades de sessió d'usuari"

create_session(user_id, session_data)
print(get_session(user_id))  # Hauria d'imprimir les dades de sessió
delete_session(user_id)
print(get_session(user_id))  # Hauria d'imprimir None

Resum

En aquest estudi de cas, hem demostrat com es pot utilitzar Redis per a la gestió de sessions en una aplicació web, proporcionant una manera ràpida i eficient de gestionar sessions d'usuari amb expiració automàtica.

Conclusió

A través d'aquests estudis de cas, hem explorat diverses aplicacions pràctiques de Redis, incloent l'emmagatzematge en memòria cau, l'analítica en temps real i la gestió de sessions. Aquests exemples destaquen la versatilitat i els beneficis de rendiment d'utilitzar Redis en diferents escenaris. Entenent aquests casos d'ús, podeu aprofitar millor Redis en els vostres propis projectes per aconseguir resultats òptims.
