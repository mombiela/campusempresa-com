Redis Streams
RedisGraph ►

Redis Streams és una estructura de dades potent introduïda a Redis 5.0, dissenyada per gestionar i processar fluxos de dades. Aquesta secció cobrirà des dels conceptes bàsics fins a l'ús avançat de Redis Streams, incloent conceptes clau, comandes i exemples pràctics.

Introducció a Redis Streams

Redis Streams és una estructura de dades de registre que et permet emmagatzemar múltiples camps i valors de cadena amb una seqüència automàtica basada en el temps. És particularment útil per al processament de dades en temps real, l'origen d'esdeveniments i els sistemes de missatgeria.

Conceptes Clau

  • Stream: Una seqüència d'entrades, cadascuna amb una ID única.
  • Entry: Un registre únic en el flux, que consisteix en una ID única i un o més parells camp-valor.
  • Consumer Group: Un grup de consumidors que poden llegir del flux de manera coordinada.
  • Consumer: Un client únic que llegeix entrades del flux.

Comandes Bàsiques

Afegir Entrades a un Flux

Per afegir una entrada a un flux, utilitza la comanda XADD. La sintaxi bàsica és:

XADD stream_name * field1 value1 field2 value2 ...

Exemple:

XADD mystream * sensor-id 1234 temperature 19.8

Llegir Entrades d'un Flux

Per llegir entrades d'un flux, utilitza la comanda XRANGE. La sintaxi bàsica és:

XRANGE stream_name start end [COUNT count]

Exemple:

XRANGE mystream - +

Aquesta comanda llegeix totes les entrades del flux mystream.

Llegir Noves Entrades

Per llegir noves entrades afegides al flux, utilitza la comanda XREAD. La sintaxi bàsica és:

XREAD COUNT count STREAMS stream_name last_id

Exemple:

XREAD COUNT 2 STREAMS mystream $

Aquesta comanda llegeix les dues següents noves entrades de mystream.

Ús Avançat

Grups de Consumidors

Els grups de consumidors permeten que múltiples consumidors llegeixin del mateix flux sense duplicar la feina. Utilitza la comanda XGROUP per gestionar grups de consumidors.

Crear un Grup de Consumidors

XGROUP CREATE stream_name group_name $ MKSTREAM

Exemple:

XGROUP CREATE mystream mygroup $ MKSTREAM

Llegir Entrades amb Grups de Consumidors

Utilitza la comanda XREADGROUP per llegir entrades com a part d'un grup de consumidors.

XREADGROUP GROUP group_name consumer_name COUNT count STREAMS stream_name last_id

Exemple:

XREADGROUP GROUP mygroup consumer1 COUNT 2 STREAMS mystream >

Acknowledging Entrades

Els consumidors han d'acknowledging les entrades que han processat utilitzant la comanda XACK.

XACK stream_name group_name entry_id

Exemple:

XACK mystream mygroup 1526569495631-0

Gestionar Entrades Pendents

Per veure les entrades pendents, utilitza la comanda XPENDING.

XPENDING stream_name group_name [start end count [consumer]]

Exemple:

XPENDING mystream mygroup

Exemples Pràctics

Exemple: Productor-Consumidor Simple

Productor:

XADD mystream * sensor-id 1234 temperature 19.8
XADD mystream * sensor-id 1235 temperature 20.1

Consumidor:

XREAD COUNT 2 STREAMS mystream 0

Exemple: Utilitzant Grups de Consumidors

Crear un grup de consumidors:

XGROUP CREATE mystream mygroup $ MKSTREAM

Productor:

XADD mystream * sensor-id 1234 temperature 19.8
XADD mystream * sensor-id 1235 temperature 20.1

Consumidor 1:

XREADGROUP GROUP mygroup consumer1 COUNT 2 STREAMS mystream >

Consumidor 2:

XREADGROUP GROUP mygroup consumer2 COUNT 2 STREAMS mystream >

Acknowledge entrades processades:

XACK mystream mygroup 1526569495631-0

Resum

Redis Streams és una estructura de dades versàtil per gestionar fluxos de dades en temps real. Suporta afegir i llegir entrades, gestionar grups de consumidors i acknowledging entrades processades. Entenent i utilitzant aquestes comandes, pots gestionar i processar eficientment fluxos de dades en les teves aplicacions.
