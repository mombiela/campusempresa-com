Introducció a les Transaccions

Les transaccions de Redis permeten executar una sèrie de comandes en una única operació atòmica. Això assegura que o bé totes les comandes s'executen, o cap, mantenint la integritat de les dades.

  • Atomicitat: Totes les comandes d'una transacció s'executen com una única unitat.
  • Aïllament: Cap altra comanda s'executa simultàniament amb una transacció.

Comandes Bàsiques per a Transaccions

Redis proporciona algunes comandes clau per gestionar transaccions:

  • MULTI: Marca l'inici d'un bloc de transacció.
  • EXEC: Executa totes les comandes emeses després de MULTI.
  • DISCARD: Cancel·la la transacció.
  • WATCH: Monitoritza una o més claus per determinar si la transacció s'ha d'avortar.

Exemple: Transacció Bàsica

MULTI
SET key1 "value1"
SET key2 "value2"
EXEC

En aquest exemple:

  1. MULTI inicia la transacció.
  2. SET key1 "value1" i SET key2 "value2" es posen en cua.
  3. EXEC executa ambdues comandes atòmicament.

Gestió d'Errors en Transaccions

Les transaccions de Redis gestionen errors de dues maneres:

  • Errors de Posada en Cua de Comandes: Si una comanda és sintàcticament incorrecta, es detectarà i es reportarà quan es posi en cua.
  • Errors d'Execució: Els errors que es produeixen durant l'execució de les comandes es reporten, però la transacció encara executarà totes les comandes.

Exemple: Gestió d'Errors

MULTI
SET key1 "value1"
INCR key1  # Això causarà un error perquè key1 no és un enter
EXEC

En aquest exemple:

  • La comanda INCR key1 causarà un error d'execució.
  • Malgrat l'error, SET key1 "value1" encara s'executarà.

Bloqueig Optimista amb WATCH

La comanda WATCH s'utilitza per implementar el bloqueig optimista. Monitoritza les claus especificades i avorta la transacció si alguna d'elles es modifica abans que la transacció s'executi.

Exemple: Bloqueig Optimista

WATCH key1
MULTI
SET key1 "new_value"
EXEC

En aquest exemple:

  1. WATCH key1 monitoritza key1.
  2. Si key1 és modificada per un altre client abans de EXEC, la transacció s'avortarà.

Usos Pràctics

  • Operacions Atòmiques: Assegurar que una sèrie de comandes s'executin com una única unitat.
  • Comptadors i Seqüències: Incrementar comptadors de manera segura.
  • Actualitzacions Condicionals: Utilitzar WATCH per assegurar la consistència de les dades.

Exemple: Increment de Comptador

WATCH counter
val = GET counter
val = val + 1
MULTI
SET counter val
EXEC

En aquest exemple:

  1. WATCH counter assegura que el comptador no sigui modificat per un altre client.
  2. El comptador es llegeix, s'incrementa i es configura dins d'una transacció.

Conclusió

Les transaccions de Redis proporcionen un mecanisme potent per assegurar l'atomicitat i la consistència en les teves operacions de dades. Entenent i utilitzant comandes com MULTI, EXEC, DISCARD i WATCH, pots gestionar efectivament manipulacions de dades complexes i mantenir la integritat de les dades en les teves aplicacions Redis.
