Introducció al Mode Clúster de Redis

El Mode Clúster de Redis és una manera d'executar una instal·lació de Redis on les dades es distribueixen automàticament entre múltiples nodes de Redis. Això permet que Redis escali horitzontalment i gestioni conjunts de dades més grans i un major rendiment.

Conceptes Clau

  • Sharding: Dividir les dades entre múltiples nodes.
  • Node Mestre: Un node que conté les dades reals i gestiona les operacions de lectura/escriptura.
  • Node Replica: Un node que conté una còpia de les dades d'un node mestre per a redundància i recuperació.
  • Slots: Redis Cluster utilitza 16.384 slots de hash per distribuir les claus a través del clúster.

Configuració d'un Clúster de Redis

Configurar un Clúster de Redis implica configurar múltiples nodes de Redis per treballar junts. Aquí teniu una guia pas a pas:

Prerequisits

  • Redis instal·lat en múltiples servidors o instàncies.
  • Comprensió bàsica de les comandes i la configuració de Redis.

Fitxers de Configuració

Cada node de Redis necessita un fitxer de configuració. Aquí teniu un exemple de configuració per a un node de Redis:

port 7000
cluster-enabled yes
cluster-config-file nodes.conf
cluster-node-timeout 5000
appendonly yes

Iniciar Nodes de Redis

Inicieu cada instància de Redis amb el seu fitxer de configuració:

redis-server /path/to/redis-7000.conf
redis-server /path/to/redis-7001.conf
redis-server /path/to/redis-7002.conf

Crear el Clúster

Utilitzeu el redis-cli per crear el clúster. Necessiteu almenys tres nodes mestres:

redis-cli --cluster create 127.0.0.1:7000 127.0.0.1:7001 127.0.0.1:7002 --cluster-replicas 1

Aquesta comanda crearà un clúster amb tres nodes mestres i una rèplica per a cada mestre.

Gestió d'un Clúster de Redis

Un cop el clúster està configurat, necessiteu gestionar-lo. Això inclou afegir/eliminar nodes, gestionar recuperacions i monitoritzar el clúster.

Afegir un Node

Per afegir un nou node al clúster:

  1. Inicieu la nova instància de Redis.
  2. Afegiu el node al clúster utilitzant redis-cli:
redis-cli --cluster add-node 127.0.0.1:7003 127.0.0.1:7000

Eliminar un Node

Per eliminar un node del clúster:

  1. Assegureu-vos que el node està buit o migreu les seves dades.
  2. Utilitzeu redis-cli per eliminar el node:
redis-cli --cluster del-node 127.0.0.1:7000 <node-id>

Gestionar la Recuperació

Redis Cluster gestiona la recuperació automàticament. Si un node mestre falla, una de les seves rèpliques es promou a mestre. També podeu desencadenar una recuperació manualment:

redis-cli -c -p 7001 cluster failover

Exemples de Codi

Aquí teniu alguns exemples de com interactuar amb un Clúster de Redis utilitzant Python i la biblioteca redis-py:

Connectar-se a un Clúster de Redis

import rediscluster

startup_nodes = [{"host": "127.0.0.1", "port": "7000"}]
rc = rediscluster.RedisCluster(startup_nodes=startup_nodes, decode_responses=True)

# Establir una clau
rc.set("foo", "bar")

# Obtenir una clau
value = rc.get("foo")
print(value)  # Sortida: bar

Gestionar la Recuperació en el Codi

La biblioteca redis-py gestiona la recuperació automàticament. Si un node mestre cau, la biblioteca redirigirà la sol·licitud al nou mestre.

Resum

El Mode Clúster de Redis permet que Redis escali horitzontalment distribuint les dades entre múltiples nodes. Configurar un Clúster de Redis implica configurar múltiples instàncies de Redis, crear el clúster i gestionar-lo mitjançant l'afegit/eliminació de nodes i la gestió de recuperacions. Utilitzant biblioteques com redis-py, podeu interactuar fàcilment amb un Clúster de Redis en les vostres aplicacions.

Comprenent i implementant el Mode Clúster de Redis, podeu assegurar alta disponibilitat, tolerància a fallades i escalabilitat per a les vostres aplicacions basades en Redis.
