Introducció a l'Autenticació en Redis

L'autenticació és un aspecte crucial per assegurar la teva instància de Redis. Per defecte, Redis no requereix autenticació, però habilitar-la pot ajudar a protegir les teves dades d'accés no autoritzat. En aquesta secció, cobrirem els conceptes bàsics per configurar i utilitzar l'autenticació en Redis.

Configuració de l'Autenticació

Configuració de Redis amb una Contrasenya

Per habilitar l'autenticació en Redis, has de configurar una contrasenya en el fitxer de configuració de Redis (redis.conf). Aquí tens com fer-ho:

  1. Obre el fitxer redis.conf en un editor de text.
  2. Troba la línia que comença amb # requirepass i descomenta-la eliminant el #.
  3. Configura una contrasenya afegint-la després de requirepass.
# redis.conf
requirepass yourpassword
  1. Desa el fitxer i reinicia el servidor Redis perquè els canvis tinguin efecte.

Connexió a Redis amb una Contrasenya

Un cop l'autenticació està habilitada, els clients han de proporcionar la contrasenya per connectar-se al servidor Redis. Aquí tens com fer-ho utilitzant el Redis CLI:

redis-cli -a yourpassword

O, si ja estàs connectat al servidor Redis, pots autenticar-te utilitzant el comandament AUTH:

AUTH yourpassword

Exemple: Client Python amb Autenticació

Aquí tens un exemple de com connectar-se a un servidor Redis amb autenticació utilitzant la biblioteca redis-py en Python:

import redis

# Connectar-se al servidor Redis amb autenticació
client = redis.StrictRedis(host='localhost', port=6379, password='yourpassword')

# Provar la connexió
try:
    client.ping()
    print("Connectat a Redis!")
except redis.AuthenticationError:
    print("Autenticació fallida!")

Tècniques Avançades d'Autenticació

Ús de ACLs (Llistes de Control d'Accés)

Redis 6.0 va introduir les Llistes de Control d'Accés (ACLs), que proporcionen un control més granular sobre els permisos dels usuaris. Amb les ACLs, pots crear múltiples usuaris amb diferents nivells d'accés.

Creació d'un Usuari amb ACLs

Per crear un nou usuari amb permisos específics, utilitza el comandament ACL SETUSER:

ACL SETUSER alice on >alicepassword ~* +@all

Aquest comandament crea un usuari anomenat alice amb la contrasenya alicepassword, accés a totes les claus (~*), i tots els comandaments (+@all).

Exemple: Client Python amb ACLs

Aquí tens un exemple de com connectar-se a un servidor Redis utilitzant un usuari creat amb ACLs:

import redis

# Connectar-se al servidor Redis amb usuari ACL
client = redis.StrictRedis(host='localhost', port=6379, username='alice', password='alicepassword')

# Provar la connexió
try:
    client.ping()
    print("Connectat a Redis com alice!")
except redis.AuthenticationError:
    print("Autenticació fallida!")

Gestió de ACLs

Pots llistar tots els usuaris i els seus permisos utilitzant el comandament ACL LIST:

ACL LIST

Per eliminar un usuari, utilitza el comandament ACL DELUSER:

ACL DELUSER alice

Resum

L'autenticació en Redis és essencial per assegurar les teves dades. Començant amb una configuració simple de contrasenya, pots protegir la teva instància de Redis d'accés no autoritzat. Per a una seguretat més avançada, Redis 6.0 i versions posteriors ofereixen ACLs, permetent-te crear múltiples usuaris amb diferents permisos. Comprenent i implementant aquestes tècniques d'autenticació, pots assegurar-te que el teu servidor Redis romangui segur i accessible només per a usuaris autoritzats.
