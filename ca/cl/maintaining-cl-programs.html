Mantenir programes CL (Control Language) és una habilitat crucial per a qualsevol programador que treballi amb sistemes IBM i. Aquest tema et guiarà a través de les millors pràctiques, eines i tècniques per mantenir i optimitzar els teus programes CL, des de nivells bàsics fins a avançats.

Introducció al Manteniment de Programes CL

Mantenir programes CL implica assegurar-se que el teu codi sigui eficient, llegible i actualitzat amb els estàndards actuals. Aquesta secció cobrirà els conceptes bàsics de per què el manteniment és important i introduirà alguns conceptes clau.

  • Llegibilitat del Codi: Assegurar que el teu codi sigui fàcil de llegir i entendre.
  • Eficiència del Codi: Optimitzar el teu codi per obtenir un millor rendiment.
  • Documentació: Mantenir una documentació exhaustiva per a futures referències.
  • Control de Versions: Utilitzar eines per gestionar diferents versions del teu codi.

Millors Pràctiques per Mantenir Programes CL

Llegibilitat del Codi

El codi llegible és més fàcil de mantenir i depurar. Aquí tens alguns consells per millorar la llegibilitat dels teus programes CL:

  • Ús de Noms Significatius: Utilitza noms descriptius per a variables, etiquetes i procediments.
  • Format Consistent: Segueix un estil consistent per a la indentació i l'espaiat.
  • Comentaris: Afegeix comentaris per explicar la lògica complexa o seccions importants del codi.

Exemple

/* Aquest programa CL comprova si un fitxer existeix i després el processa */
PGM

DCL VAR(&FILE) TYPE(*CHAR) LEN(10) VALUE('MYFILE')
DCL VAR(&EXISTS) TYPE(*LGL)

CHKOBJ OBJ(&FILE) OBJTYPE(*FILE)
MONMSG MSGID(CPF9801) EXEC(DO)
    CHGVAR VAR(&EXISTS) VALUE('0')
ENDDO

IF COND(&EXISTS) THEN(DO)
    /* Processa el fitxer */
    CALL PGM(PROCESSFILE) PARM(&FILE)
ENDDO

ENDPGM

Eficiència del Codi

El codi eficient s'executa més ràpid i utilitza menys recursos. Aquí tens alguns consells per optimitzar els teus programes CL:

  • Minimitza Operacions d'I/O: Redueix el nombre de lectures i escriptures de fitxers.
  • Utilitza Funcions Incorporades: Aprofita les funcions i comandes incorporades sempre que sigui possible.
  • Evita Bucles Innecessaris: Assegura't que els bucles siguin necessaris i estiguin optimitzats.

Exemple

/* Comprovant eficientment múltiples fitxers */
PGM

DCL VAR(&FILES) TYPE(*CHAR) LEN(10) VALUE('FILE1 FILE2 FILE3')
DCL VAR(&FILE) TYPE(*CHAR) LEN(10)
DCL VAR(&INDEX) TYPE(*INT) LEN(2) VALUE(1)

DOFOR VAR(&INDEX) FROM(1) TO(3)
    CHGVAR VAR(&FILE) VALUE(%SST(&FILES &INDEX 10))
    CHKOBJ OBJ(&FILE) OBJTYPE(*FILE)
    MONMSG MSGID(CPF9801) EXEC(DO)
        /* Gestiona fitxer no trobat */
    ENDDO
ENDDO

ENDPGM

Eines per Mantenir Programes CL

Sistemes de Control de Fonts

Utilitzar sistemes de control de fonts com Git pot ajudar-te a gestionar diferents versions dels teus programes CL i col·laborar amb altres desenvolupadors.

  • Seguiment de Versions: Mantén un seguiment dels canvis i torna a versions anteriors si és necessari.
  • Col·laboració: Treballa amb altres desenvolupadors sense sobreescriure els canvis dels altres.
  • Branques i Fusió: Experimenta amb noves característiques sense afectar la base de codi principal.

Eines de Depuració

IBM i proporciona diverses eines per ajudar-te a depurar els teus programes CL:

  • STRDBG (Iniciar Depuració): Inicia una sessió de depuració per seguir el teu codi pas a pas.
  • DSPJOBLOG (Mostrar Registre de Treball): Visualitza el registre de treball per veure missatges i errors.
  • WRKACTJOB (Treballar amb Treballs Actius): Monitoritza els treballs actius i el seu rendiment.

Exemple

/* Iniciant una sessió de depuració */
STRDBG PGM(MYLIB/MYPGM)

Tècniques Avançades per al Manteniment de Programes CL

Programació Modular

Dividir els teus programes CL en mòduls més petits i reutilitzables pot fer-los més fàcils de mantenir i provar.

  • Procediments: Defineix procediments reutilitzables per a tasques comunes.
  • Programes de Servei: Agrupa procediments relacionats en programes de servei.

Exemple

/* Programa principal que crida un procediment */
PGM

DCL VAR(&RESULT) TYPE(*DEC) LEN(5 2)

CALLPRC PRC(CALCULATE) PARM(5 10) RTNVAL(&RESULT)

ENDPGM

/* Definició del procediment */
PRC CALCULATE
    DCL VAR(&A) TYPE(*DEC) LEN(5 2)
    DCL VAR(&B) TYPE(*DEC) LEN(5 2)
    DCL VAR(&RESULT) TYPE(*DEC) LEN(5 2)

    CHGVAR VAR(&RESULT) VALUE(&A + &B)

    RETURN VALUE(&RESULT)
ENDPRC

Optimització del Rendiment

Optimitzar el rendiment dels teus programes CL pot portar a millores significatives en l'eficiència del sistema.

  • Programació de Treballs: Programa treballs durant hores de poca activitat per equilibrar la càrrega del sistema.
  • Assignació de Recursos: Assigna recursos suficients als treballs crítics.
  • Monitorització del Rendiment: Monitoritza i analitza regularment el rendiment dels treballs.

Exemple

/* Programant un treball */
SBMJOB CMD(CALL PGM(MYLIB/MYPGM)) JOB(MYJOB) JOBQ(QBATCH) SCDDATE('2023-10-01') SCDTIME('010000')

Conclusió

Mantenir programes CL és un procés continu que implica assegurar la llegibilitat del codi, optimitzar el rendiment i utilitzar les eines adequades. Seguint les millors pràctiques i aprofitant les tècniques avançades, pots mantenir els teus programes CL eficients, fàcils de mantenir i actualitzats. El manteniment regular no només millora el rendiment dels teus programes, sinó que també els fa més fàcils d'entendre i modificar en el futur.
