Introducció

A Linux, els comodins i les expressions regulars són eines poderoses per cercar, coincidir i manipular text. Aquesta secció et guiarà des dels conceptes bàsics fins a l'ús avançat d'aquestes eines, proporcionant explicacions clares, exemples i exercicis.

Comodins

Els comodins són caràcters especials utilitzats per coincidir amb patrons en noms de fitxers i directoris. S'utilitzen habitualment en ordres de shell per realitzar operacions en múltiples fitxers alhora.

Comodins bàsics

  • * (asterisc): Coincideix amb zero o més caràcters.
  • ? (interrogant): Coincideix exactament amb un caràcter.
  • [] (claus): Coincideix amb qualsevol dels caràcters inclosos.

Exemples

# Llistar tots els fitxers al directori actual
ls *

# Llistar tots els fitxers que comencen amb 'a'
ls a*

# Llistar tots els fitxers amb un sol caràcter de nom
ls ?

# Llistar tots els fitxers que comencen amb 'a', 'b' o 'c'
ls [abc]*

Comodins avançats

  • [!abc]: Coincideix amb qualsevol caràcter excepte 'a', 'b' o 'c'.
  • {} (claus): Coincideix amb una llista de patrons separats per comes.

Exemples

# Llistar tots els fitxers excepte aquells que comencen amb 'a', 'b' o 'c'
ls [!abc]*

# Llistar fitxers amb extensions .txt o .md
ls *.{txt,md}

Expressions regulars

Les expressions regulars (regex) són seqüències de caràcters que defineixen patrons de cerca. Són més poderoses i flexibles que els comodins i s'utilitzen en moltes eines de Linux com grep, sed i awk.

Expressions regulars bàsiques

  • . (punt): Coincideix amb qualsevol caràcter únic.
  • * (asterisc): Coincideix amb zero o més de l'element anterior.
  • ^ (caret): Coincideix amb el començament d'una línia.
  • $ (signe de dòlar): Coincideix amb el final d'una línia.
  • [] (claus): Coincideix amb qualsevol dels caràcters inclosos.

Exemples

# Coincideix amb línies que contenen 'cat'
grep 'cat' file.txt

# Coincideix amb línies que comencen amb 'cat'
grep '^cat' file.txt

# Coincideix amb línies que acaben amb 'cat'
grep 'cat$' file.txt

# Coincideix amb línies que contenen 'c', 'a' o 't'
grep '[cat]' file.txt

Expressions regulars avançades

  • + (més): Coincideix amb un o més de l'element anterior.
  • ? (interrogant): Coincideix amb zero o un de l'element anterior.
  • {n,m}: Coincideix amb entre n i m ocurrències de l'element anterior.
  • () (parèntesis): Agrupa patrons.
  • | (tub): Coincideix amb el patró abans o després del tub.

Exemples

# Coincideix amb línies que contenen 'cat' una o més vegades
grep 'cat+' file.txt

# Coincideix amb línies que contenen 'cat' zero o una vegada
grep 'cat?' file.txt

# Coincideix amb línies que contenen 'cat' entre 2 i 4 vegades
grep 'cat{2,4}' file.txt

# Coincideix amb línies que contenen 'cat' o 'dog'
grep 'cat|dog' file.txt

# Coincideix amb línies que contenen 'cat' seguit de 'dog'
grep '(cat)(dog)' file.txt

Exercicis

Comodins

  1. Llista tots els fitxers al teu directori personal que comencen amb 'd'.
  2. Llista tots els fitxers al teu directori personal que tenen una extensió '.sh'.
  3. Llista tots els fitxers al teu directori personal que no comencen amb 'a', 'b' o 'c'.

Expressions regulars

  1. Troba totes les línies en un fitxer que comencen amb 'Hello'.
  2. Troba totes les línies en un fitxer que acaben amb 'world'.
  3. Troba totes les línies en un fitxer que contenen 'foo' o 'bar'.
  4. Troba totes les línies en un fitxer que contenen un dígit.

Conclusió

Els comodins i les expressions regulars són eines essencials per gestionar fitxers i processar text de manera eficient a Linux. Dominar aquestes eines millorarà significativament la teva capacitat per realitzar cerques complexes i manipulacions de text. Practica amb els exemples i exercicis proporcionats per convertir-te en un expert en l'ús de comodins i expressions regulars.
