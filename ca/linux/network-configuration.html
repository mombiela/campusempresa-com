Introducció

La configuració de xarxa en Linux implica configurar i gestionar interfícies de xarxa, adreces IP, rutes i configuracions DNS. Aquest tema et guiarà des de tasques bàsiques fins a avançades de configuració de xarxa en Linux.

Configuració Bàsica de Xarxa

Veure les Interfícies de Xarxa

Per veure les interfícies de xarxa actuals i els seus estats, pots utilitzar la comanda ip.

ip a
  • Explicació: La comanda ip a llista totes les interfícies de xarxa i els seus detalls, com ara adreces IP i estat (actiu o inactiu).

Configurar una Adreça IP Estàtica

Per configurar una adreça IP estàtica, pots utilitzar la comanda ip.

sudo ip addr add 192.168.1.100/24 dev eth0
sudo ip link set dev eth0 up
  • Explicació:
    • ip addr add 192.168.1.100/24 dev eth0 assigna l'adreça IP 192.168.1.100 amb una màscara de subxarxa de 255.255.255.0 a la interfície eth0.
    • ip link set dev eth0 up activa la interfície eth0.

Configurar DNS

Les configuracions DNS es configuren típicament al fitxer /etc/resolv.conf.

sudo nano /etc/resolv.conf

Afegeix les següents línies per configurar els servidors DNS:

nameserver 8.8.8.8
nameserver 8.8.4.4
  • Explicació: Les línies nameserver especifiquen les adreces IP dels servidors DNS a utilitzar.

Configuració Intermèdia de Xarxa

Utilitzar NetworkManager

NetworkManager és una eina per gestionar configuracions de xarxa en Linux.

Comprovar l'Estat de NetworkManager

sudo systemctl status NetworkManager
  • Explicació: Aquesta comanda comprova si NetworkManager està en funcionament.

Configurar una Interfície de Xarxa amb NetworkManager

nmcli con add type ethernet ifname eth0 con-name my-eth0 ip4 192.168.1.100/24 gw4 192.168.1.1
nmcli con up my-eth0
  • Explicació:
    • nmcli con add afegeix una nova connexió anomenada my-eth0 per a la interfície eth0 amb l'adreça IP i la passarel·la especificades.
    • nmcli con up my-eth0 activa la connexió my-eth0.

Configurar Rutes

Les rutes determinen com es reenvien els paquets entre xarxes.

Afegir una Ruta Estàtica

sudo ip route add 192.168.2.0/24 via 192.168.1.1
  • Explicació: Aquesta comanda afegeix una ruta a la xarxa 192.168.2.0/24 a través de la passarel·la 192.168.1.1.

Configuració Avançada de Xarxa

Utilitzar `netplan` per a la Configuració de Xarxa

netplan és una utilitat per configurar configuracions de xarxa en Ubuntu i altres distribucions que utilitzen netplan.

Configurar netplan

Edita el fitxer de configuració a /etc/netplan/.

sudo nano /etc/netplan/01-netcfg.yaml

Afegeix la següent configuració:

network:
  version: 2
  ethernets:
    eth0:
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

Aplica la configuració:

sudo netplan apply
  • Explicació: Aquesta configuració YAML estableix una adreça IP estàtica, una passarel·la i servidors DNS per a la interfície eth0.

Configurar el Tallafocs amb `iptables`

iptables és una eina poderosa per configurar regles de filtratge de paquets de xarxa.

Comandes Bàsiques de iptables

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
sudo iptables -A INPUT -j DROP
  • Explicació:
    • iptables -A INPUT -p tcp --dport 22 -j ACCEPT permet connexions SSH entrants.
    • iptables -A INPUT -j DROP descarta tot el tràfic entrant restant.

Utilitzar `firewalld` per a la Gestió del Tallafocs

firewalld és una eina de gestió de tallafocs dinàmica amb iptables com a backend.

Configurar firewalld

sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload
  • Explicació:
    • firewall-cmd --permanent --add-service=ssh permet el servei SSH de manera permanent.
    • firewall-cmd --reload recarrega la configuració del tallafocs.

Conclusió

La configuració de xarxa en Linux abasta des de tasques bàsiques com establir adreces IP i servidors DNS fins a tasques avançades com configurar tallafocs i rutes. Dominar aquestes habilitats és essencial per gestionar sistemes Linux de manera efectiva. Comprenent i practicant aquestes configuracions, pots assegurar-te que els teus sistemes Linux estiguin correctament connectats i segurs.
