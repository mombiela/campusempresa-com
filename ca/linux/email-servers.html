Introducció als servidors de correu electrònic

Els servidors de correu electrònic són components crítics dels sistemes de comunicació moderns, permetent l'enviament, la recepció i l'emmagatzematge de missatges de correu electrònic. En un entorn Linux, entendre com configurar i gestionar servidors de correu electrònic és essencial per als administradors de sistemes i desenvolupadors.

Conceptes clau

  • SMTP (Simple Mail Transfer Protocol): Protocol utilitzat per enviar correus electrònics.
  • IMAP (Internet Message Access Protocol): Protocol utilitzat per recuperar correus electrònics d'un servidor.
  • POP3 (Post Office Protocol 3): Un altre protocol utilitzat per recuperar correus electrònics, normalment descarregant-los al client.
  • MTA (Mail Transfer Agent): Programari que transfereix correus electrònics d'un servidor a un altre (per exemple, Postfix, Sendmail).
  • MDA (Mail Delivery Agent): Programari que lliura correus electrònics a la bústia del destinatari (per exemple, Dovecot, Procmail).
  • MUA (Mail User Agent): Programari client de correu electrònic utilitzat pels usuaris per llegir i enviar correus electrònics (per exemple, Thunderbird, Outlook).

Configuració d'un servidor de correu electrònic

En aquesta secció, cobrim els passos per configurar un servidor de correu electrònic bàsic utilitzant Postfix (MTA) i Dovecot (MDA) en un sistema Linux.

Instal·lació de Postfix

Postfix és un MTA popular que és fàcil de configurar i segur. Seguiu aquests passos per instal·lar i configurar Postfix:

  1. Instal·lar Postfix:

    sudo apt-get update
sudo apt-get install postfix

  2. Configurar Postfix: Durant la instal·lació, se us demanarà que configureu Postfix. Trieu "Internet Site" i establiu el nom del sistema de correu al vostre nom de domini (per exemple, example.com).

  3. Editar la configuració de Postfix: Obriu el fitxer de configuració principal de Postfix:

    sudo nano /etc/postfix/main.cf

    Assegureu-vos que els següents paràmetres estiguin configurats:

    myhostname = mail.example.com
mydomain = example.com
myorigin = /etc/mailname
inet_interfaces = all
mydestination = $myhostname, localhost.$mydomain, localhost, $mydomain
relayhost =

  4. Reiniciar Postfix:

    sudo systemctl restart postfix

Instal·lació de Dovecot

Dovecot és un MDA àmpliament utilitzat que suporta tant els protocols IMAP com POP3.

  1. Instal·lar Dovecot:

    sudo apt-get install dovecot-imapd dovecot-pop3d

  2. Configurar Dovecot: Obriu el fitxer de configuració principal de Dovecot:

    sudo nano /etc/dovecot/dovecot.conf

    Assegureu-vos que els següents paràmetres estiguin configurats:

    protocols = imap pop3
mail_location = maildir:~/Maildir

  3. Reiniciar Dovecot:

    sudo systemctl restart dovecot

Assegurar el vostre servidor de correu electrònic

La seguretat és primordial quan es gestiona un servidor de correu electrònic. Aquí teniu alguns passos per assegurar la vostra configuració de Postfix i Dovecot:

Activació de TLS

TLS (Transport Layer Security) xifra el trànsit de correu electrònic, fent-lo més segur.

  1. Generar certificats SSL:

    sudo openssl req -new -x509 -days 365 -nodes -out /etc/ssl/certs/mailserver.crt -keyout /etc/ssl/private/mailserver.key

  2. Configurar Postfix per a TLS: Editeu el fitxer de configuració principal de Postfix:

    sudo nano /etc/postfix/main.cf

    Afegiu les següents línies:

    smtpd_tls_cert_file=/etc/ssl/certs/mailserver.crt
smtpd_tls_key_file=/etc/ssl/private/mailserver.key
smtpd_use_tls=yes

  3. Configurar Dovecot per a TLS: Editeu el fitxer de configuració SSL de Dovecot:

    sudo nano /etc/dovecot/conf.d/10-ssl.conf

    Assegureu-vos que els següents paràmetres estiguin configurats:

    ssl = required
ssl_cert = </etc/ssl/certs/mailserver.crt
ssl_key = </etc/ssl/private/mailserver.key

  4. Reiniciar serveis:

    sudo systemctl restart postfix
sudo systemctl restart dovecot

Configuració avançada

Per a configuracions més avançades, potser necessiteu integrar funcions addicionals com el filtratge de correu brossa, l'escaneig de virus i l'autenticació d'usuaris.

Filtratge de correu brossa amb SpamAssassin

SpamAssassin és una eina potent per filtrar correus brossa.

  1. Instal·lar SpamAssassin:

    sudo apt-get install spamassassin spamc

  2. Activar SpamAssassin:

    sudo systemctl enable spamassassin
sudo systemctl start spamassassin

  3. Configurar Postfix per utilitzar SpamAssassin: Editeu el fitxer de configuració principal de Postfix:

    sudo nano /etc/postfix/master.cf

    Afegiu les següents línies:

    smtp      inet  n       -       y       -       -       smtpd
  -o content_filter=spamassassin
spamassassin unix -     n       n       -       -       pipe
  user=spamd argv=/usr/bin/spamc -f -e
  /usr/sbin/sendmail -oi -f ${sender} ${recipient}

  4. Reiniciar Postfix:

    sudo systemctl restart postfix

Conclusió

Configurar i gestionar un servidor de correu electrònic en Linux implica entendre diversos protocols i components com SMTP, IMAP, POP3, MTA i MDA. Seguint els passos descrits en aquesta guia, podeu configurar un servidor de correu electrònic bàsic utilitzant Postfix i Dovecot, assegurar-lo amb TLS i millorar-lo amb el filtratge de correu brossa utilitzant SpamAssassin. A mesura que adquiriu més experiència, podeu explorar funcions i configuracions addicionals per satisfer les vostres necessitats específiques.
