Introducció a SSH

Secure Shell (SSH) és un protocol utilitzat per connectar-se de manera segura a sistemes remots a través d'una xarxa. Proporciona un canal segur sobre una xarxa no segura mitjançant l'ús de xifratge.

  • Què és SSH?

    • SSH significa Secure Shell.
    • S'utilitza per a l'inici de sessió remot segur i altres serveis de xarxa segurs sobre una xarxa insegura.
    • SSH utilitza criptografia de clau pública per autenticar l'ordinador remot i permetre que aquest autentiqui l'usuari.

  • Per què utilitzar SSH?

    • Comunicació segura entre client i servidor.
    • Transferència de dades xifrades.
    • Autenticació d'usuaris i dispositius.
    • Transferència segura de fitxers (SCP, SFTP).

Configuració de SSH

Per utilitzar SSH, necessites tenir un client SSH a la teva màquina local i un servidor SSH a la màquina remota.

Instal·lació de SSH

  • A Ubuntu/Debian:

    sudo apt update
sudo apt install openssh-server

  • A CentOS/RHEL:

    sudo yum install openssh-server
sudo systemctl start sshd
sudo systemctl enable sshd

  • A Windows:

    • Utilitza un client SSH com PuTTY o habilita el client OpenSSH integrat a Windows 10.

Iniciar i habilitar el servei SSH

  • A Ubuntu/Debian:

    sudo systemctl start ssh
sudo systemctl enable ssh

  • A CentOS/RHEL:

    sudo systemctl start sshd
sudo systemctl enable sshd

Connexió a un servidor remot

Per connectar-te a un servidor remot utilitzant SSH, necessites l'adreça IP o el nom de l'amfitrió del servidor i el nom d'usuari.

Comanda bàsica de SSH

ssh username@hostname_or_ip
  • Exemple: 
    ssh user@192.168.1.10

Utilitzar claus SSH per a l'autenticació

Les claus SSH proporcionen una manera més segura d'iniciar sessió en un servidor SSH.

Generar claus SSH

ssh-keygen -t rsa -b 2048
  • Aquesta comanda genera un parell de claus: una clau privada i una clau pública.

Copiar la clau pública al servidor

ssh-copy-id username@hostname_or_ip
  • Exemple: 
    ssh-copy-id user@192.168.1.10

Ús avançat de SSH

Reenviament de ports

SSH es pot utilitzar per reenviar ports, permetent accedir de manera segura a serveis en una xarxa remota.

  • Reenviament de port local:

    ssh -L local_port:remote_address:remote_port username@hostname_or_ip
    • Exemple: 
      ssh -L 8080:localhost:80 user@192.168.1.10

  • Reenviament de port remot:

    ssh -R remote_port:local_address:local_port username@hostname_or_ip
    • Exemple: 
      ssh -R 8080:localhost:80 user@192.168.1.10

Túnel SSH

El túnel SSH permet crear una connexió segura entre la teva màquina local i una màquina remota a través d'un servidor SSH.

  • Reenviament de port dinàmic (Proxy SOCKS): 
    ssh -D local_port username@hostname_or_ip
    • Exemple: 
      ssh -D 8080 user@192.168.1.10

Fitxer de configuració SSH

Pots simplificar les connexions SSH utilitzant el fitxer de configuració SSH (~/.ssh/config).

  • Exemple de fitxer de configuració:

    Host myserver
    HostName 192.168.1.10
    User user
    Port 22
    IdentityFile ~/.ssh/id_rsa

  • Connexió utilitzant el fitxer de configuració:

    ssh myserver

Resum

SSH és una eina poderosa per a l'accés remot segur i la gestió de servidors. Entenent els conceptes bàsics de SSH, configurant claus SSH i utilitzant funcions avançades com el reenviament de ports i el túnel, pots millorar la seguretat i l'eficiència de les teves operacions remotes. Assegura't sempre que el teu servidor SSH estigui configurat i protegit adequadament per evitar accessos no autoritzats.
