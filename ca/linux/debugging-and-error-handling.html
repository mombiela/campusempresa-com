Introducció

La depuració i la gestió d'errors són habilitats crítiques per a qualsevol programador que treballi en un entorn Linux. Aquesta secció et guiarà a través dels conceptes bàsics per identificar, diagnosticar i resoldre errors en el teu codi, així com implementar mecanismes robustos de gestió d'errors.

Comprendre els Errors

  • Errors de Sintaxi: Errors en el codi que impedeixen que sigui analitzat correctament.
  • Errors d'Execució: Errors que es produeixen durant l'execució del programa.
  • Errors Lògics: Errors que fan que el programa funcioni incorrectament però no es bloquegi.

Eines Bàsiques de Depuració

Declaracions Print

Una de les maneres més senzilles de depurar és utilitzar declaracions print per mostrar els valors de les variables i els estats del programa.

#!/bin/bash
# Exemple de script amb declaracions print per a depuració

echo "Iniciant script..."
var1="Hola"
var2="Món"
echo "var1: $var1"
echo "var2: $var2"
resultat="$var1 $var2"
echo "Resultat: $resultat"

Ús de `gdb` (GNU Debugger)

gdb és una eina poderosa per a la depuració de programes escrits en C i C++.

# Compilar amb informació de depuració
gcc -g -o myprogram myprogram.c

# Iniciar gdb amb el programa compilat
gdb ./myprogram

# Comandes comunes de gdb
break main       # Establir un punt de ruptura a la funció principal
run              # Iniciar el programa
next             # Executar la següent línia de codi
print var        # Mostrar el valor d'una variable
continue         # Continuar executant el programa fins al següent punt de ruptura

Ús de `strace`

strace és una utilitat d'espai d'usuari per a diagnòstic, depuració i instrucció en Linux. S'utilitza per monitoritzar les crides al sistema utilitzades per un programa i els senyals que rep.

# Traçar l'execució d'un programa
strace ./myprogram

# Traçar un procés en execució per PID
strace -p <pid>

Gestió d'Errors en Scripts de Shell

Ús de l'Estat de Sortida

Cada comanda en Linux retorna un estat de sortida. Un estat de 0 significa èxit, mentre que qualsevol valor diferent de zero indica un error.

#!/bin/bash
# Exemple de gestió d'estat de sortida

cp source.txt destination.txt
if [ $? -eq 0 ]; then
    echo "Fitxer copiat amb èxit."
else
    echo "S'ha produït un error en copiar el fitxer."
fi

Ús de `trap` per a Neteja

La comanda trap et permet especificar comandes per executar quan el script rep senyals específics.

#!/bin/bash
# Exemple d'ús de trap per a neteja

trap 'echo "Netejant..."; rm -f temp.txt; exit' INT TERM EXIT

# Simular treball del script
echo "Treballant..."
sleep 10

Tècniques Avançades de Depuració

Ús de `valgrind`

valgrind és una eina de programació per a la depuració de memòria, detecció de fuites de memòria i perfilat.

# Executar un programa amb valgrind
valgrind --leak-check=yes ./myprogram

Ús de `lsof`

lsof (List Open Files) s'utilitza per informar d'una llista de tots els fitxers oberts i els processos que els han obert.

# Llistar tots els fitxers oberts
lsof

# Llistar fitxers oberts per un procés específic
lsof -p <pid>

Ús de `dmesg`

dmesg s'utilitza per examinar o controlar el buffer d'anell del nucli. Pot ser útil per diagnosticar problemes de maquinari i controladors.

# Mostrar el buffer d'anell del nucli
dmesg

# Seguir nous missatges en el buffer d'anell del nucli
dmesg -w

Conclusió

La depuració i la gestió d'errors són habilitats essencials per a qualsevol programador de Linux. Comprenent els diferents tipus d'errors, utilitzant eines bàsiques i avançades de depuració, i implementant mecanismes robustos de gestió d'errors, pots millorar significativament la fiabilitat i mantenibilitat del teu codi. Practica aquestes tècniques regularment per convertir-te en un expert en diagnosticar i resoldre problemes en els teus programes.
