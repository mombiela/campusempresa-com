Servidors de Bases de Dades ►

Configurar un servidor web en un sistema Linux és una habilitat fonamental per a qualsevol desenvolupador web o administrador de sistemes. Aquest tema et guiarà a través del procés des d'un nivell principiant fins a un nivell avançat, cobrint diversos programes de servidor web, configuracions i optimitzacions.

Introducció als Servidors Web

  • Definició: Un servidor web és un programari que serveix pàgines web als usuaris a través d'internet o una intranet.
  • Servidors Web Comuns:
    • Apache
    • Nginx
    • Lighttpd

Instal·lació d'un Servidor Web

Apache

Apache és un dels servidors web més populars i àmpliament utilitzats. És conegut per la seva robustesa i flexibilitat.

Instal·lació

sudo apt update
sudo apt install apache2
  • Explicació:
    • sudo apt update: Actualitza la llista de paquets.
    • sudo apt install apache2: Instal·la el servidor web Apache.

Iniciant i Activant Apache

sudo systemctl start apache2
sudo systemctl enable apache2
  • Explicació:
    • sudo systemctl start apache2: Inicia el servei Apache.
    • sudo systemctl enable apache2: Activa Apache per iniciar-se en l'arrencada.

Nginx

Nginx és conegut pel seu alt rendiment i baix ús de recursos, cosa que el fa ideal per servir contingut estàtic i gestionar alt trànsit.

Instal·lació

sudo apt update
sudo apt install nginx
  • Explicació:
    • sudo apt update: Actualitza la llista de paquets.
    • sudo apt install nginx: Instal·la el servidor web Nginx.

Iniciant i Activant Nginx

sudo systemctl start nginx
sudo systemctl enable nginx
  • Explicació:
    • sudo systemctl start nginx: Inicia el servei Nginx.
    • sudo systemctl enable nginx: Activa Nginx per iniciar-se en l'arrencada.

Configuració Bàsica

Configuració d'Apache

El fitxer de configuració principal d'Apache es troba a /etc/apache2/apache2.conf.

Editant el Fitxer de Configuració

sudo nano /etc/apache2/apache2.conf
  • Explicació:
    • sudo nano /etc/apache2/apache2.conf: Obre el fitxer de configuració d'Apache en l'editor de text nano.

Hosts Virtuals

Els hosts virtuals et permeten allotjar múltiples llocs web en un sol servidor.

sudo nano /etc/apache2/sites-available/example.com.conf
<VirtualHost *:80>
    ServerAdmin webmaster@example.com
    ServerName example.com
    DocumentRoot /var/www/example.com
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>
  • Explicació:
    • ServerAdmin: Adreça de correu electrònic de l'administrador del servidor.
    • ServerName: Nom de domini del lloc web.
    • DocumentRoot: Directori on es guarden els fitxers del lloc web.

Activant el Host Virtual

sudo a2ensite example.com.conf
sudo systemctl reload apache2
  • Explicació:
    • sudo a2ensite example.com.conf: Activa la configuració del host virtual.
    • sudo systemctl reload apache2: Recarrega Apache per aplicar els canvis.

Configuració de Nginx

El fitxer de configuració principal de Nginx es troba a /etc/nginx/nginx.conf.

Editant el Fitxer de Configuració

sudo nano /etc/nginx/nginx.conf
  • Explicació:
    • sudo nano /etc/nginx/nginx.conf: Obre el fitxer de configuració de Nginx en l'editor de text nano.

Blocs de Servidor

Els blocs de servidor en Nginx són similars als hosts virtuals en Apache.

sudo nano /etc/nginx/sites-available/example.com
server {
    listen 80;
    server_name example.com;
    root /var/www/example.com;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}
  • Explicació:
    • listen 80: Escolta al port 80.
    • server_name: Nom de domini del lloc web.
    • root: Directori on es guarden els fitxers del lloc web.
    • index: Fitxer per defecte a servir.

Activant el Bloc de Servidor

sudo ln -s /etc/nginx/sites-available/example.com /etc/nginx/sites-enabled/
sudo systemctl reload nginx
  • Explicació:
    • sudo ln -s /etc/nginx/sites-available/example.com /etc/nginx/sites-enabled/: Crea un enllaç simbòlic per activar el bloc de servidor.
    • sudo systemctl reload nginx: Recarrega Nginx per aplicar els canvis.

Configuració Avançada

Configuració SSL/TLS

Assegurar el teu servidor web amb SSL/TLS és crucial per protegir les dades transmeses entre el servidor i els clients.

Configuració SSL d'Apache

sudo a2enmod ssl
sudo systemctl restart apache2
  • Explicació:
    • sudo a2enmod ssl: Activa el mòdul SSL en Apache.
    • sudo systemctl restart apache2: Reinicia Apache per aplicar els canvis.

Configuració SSL de Nginx

server {
    listen 443 ssl;
    server_name example.com;
    
    ssl_certificate /etc/ssl/certs/example.com.crt;
    ssl_certificate_key /etc/ssl/private/example.com.key;
    
    root /var/www/example.com;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}
  • Explicació:
    • listen 443 ssl: Escolta al port 443 amb SSL.
    • ssl_certificate: Ruta al certificat SSL.
    • ssl_certificate_key: Ruta a la clau del certificat SSL.

Optimització del Rendiment

  • Apache:

    • Utilitza el mòdul mpm_event per a un millor rendiment amb moltes connexions simultànies.
    • Optimitza els paràmetres de KeepAlive per reduir la latència.

  • Nginx:

    • Utilitza la memòria cau per millorar el rendiment.
    • Optimitza els processos i connexions dels treballadors.

Conclusió

Configurar un servidor web en Linux implica seleccionar el programari adequat, instal·lar-lo i configurar-lo per servir els teus llocs web. Apache i Nginx són dos dels servidors web més populars, cadascun amb els seus punts forts. Entendre els conceptes bàsics d'instal·lació, configuració i configuracions avançades com SSL/TLS i optimització del rendiment t'ajudarà a gestionar servidors web de manera efectiva.

Seguint aquesta guia, ara hauries de tenir una base sòlida per configurar i configurar servidors web en un sistema Linux. Continua explorant i experimentant amb diferents configuracions per optimitzar el teu servidor web per a les teves necessitats específiques.
