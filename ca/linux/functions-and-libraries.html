Introducció

En aquesta secció, explorarem els conceptes de funcions i biblioteques en el context de la programació en Linux. Entendre aquests conceptes és crucial per escriure codi modular, reutilitzable i eficient. Cobrirem les bases de la definició i ús de funcions, i després passarem a crear i utilitzar biblioteques.

Funcions

Què és una Funció?

  • Una funció és un bloc de codi dissenyat per realitzar una tasca específica.
  • Les funcions ajuden a descompondre problemes complexos en peces més petites i manejables.
  • Promouen la reutilització del codi i milloren la llegibilitat.

Definició d'una Funció

En la programació en Linux, particularment en C, una funció es defineix utilitzant la següent sintaxi:

return_type function_name(parameters) {
    // cos de la funció
}

Exemple:

#include <stdio.h>

// Declaració de la funció
int add(int a, int b);

int main() {
    int result = add(5, 3);
    printf("Resultat: %d\n", result);
    return 0;
}

// Definició de la funció
int add(int a, int b) {
    return a + b;
}
  • int add(int a, int b); és la declaració de la funció.
  • int add(int a, int b) { return a + b; } és la definició de la funció.

Crida a una Funció

  • Una funció es crida pel seu nom seguit de parèntesis que contenen arguments.
  • Exemple: result = add(5, 3);

Paràmetres i Valors de Retorn de la Funció

  • Les funcions poden prendre paràmetres i retornar valors.
  • Els paràmetres es especifiquen en els parèntesis en la definició de la funció.
  • El tipus de retorn es especifica abans del nom de la funció.

Àmbit i Durada de les Variables

  • Les variables definides dins d'una funció són locals a aquesta funció.
  • Es creen quan es crida la funció i es destrueixen quan la funció surt.

Biblioteques

Què és una Biblioteca?

  • Una biblioteca és una col·lecció de funcions precompilades que es poden utilitzar en programes.
  • Les biblioteques promouen la reutilització del codi i la modularitat.

Tipus de Biblioteques

  • Biblioteques Estàtiques: Enllaçades a l'executable en temps de compilació.
  • Biblioteques Dinàmiques (Compartides): Enllaçades en temps d'execució.

Creació d'una Biblioteca Estàtica

  1. Escriure les Funcions: Crear un fitxer .c amb les funcions que es volen incloure a la biblioteca.
  2. Compilar les Funcions: Compilar el fitxer .c en fitxers objecte (.o).
  3. Crear la Biblioteca: Utilitzar la comanda ar per crear un fitxer d'arxiu (.a).

Exemple:

// math_functions.c
int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}
# Compilar les funcions en fitxers objecte
gcc -c math_functions.c

# Crear la biblioteca estàtica
ar rcs libmath.a math_functions.o

Ús d'una Biblioteca Estàtica

  • Enllaçar la biblioteca durant la compilació del programa.

Exemple:

gcc main.c -L. -lmath -o main

Creació d'una Biblioteca Dinàmica

  1. Escriure les Funcions: Crear un fitxer .c amb les funcions que es volen incloure a la biblioteca.
  2. Compilar les Funcions: Compilar el fitxer .c en codi independent de posició (.o).
  3. Crear la Biblioteca: Utilitzar la comanda gcc per crear una biblioteca compartida (.so).

Exemple:

// math_functions.c
int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}
# Compilar les funcions en codi independent de posició
gcc -fPIC -c math_functions.c

# Crear la biblioteca dinàmica
gcc -shared -o libmath.so math_functions.o

Ús d'una Biblioteca Dinàmica

  • Enllaçar la biblioteca durant la compilació del programa.
  • Assegurar-se que la biblioteca està en el camí de biblioteques en temps d'execució.

Exemple:

gcc main.c -L. -lmath -o main
export LD_LIBRARY_PATH=.:$LD_LIBRARY_PATH
./main

Conclusió

Entendre les funcions i les biblioteques és fonamental per a una programació eficient en Linux. Les funcions permeten escriure codi modular i reutilitzable, mentre que les biblioteques permeten compartir i reutilitzar codi entre múltiples programes. En dominar aquests conceptes, es pot escriure codi més organitzat, mantenible i eficient.
