Introducció

L'evitació d'obstacles és una tècnica crucial en el desenvolupament d'intel·ligència artificial per a videojocs. Permet als personatges no jugadors (NPCs) navegar de manera eficient i realista en entorns complexos, evitant col·lisions amb objectes i altres personatges. Aquest tema és fonamental per millorar la jugabilitat i la immersió del jugador.

Conceptes Clau

  • Detecció d'Obstacles: Identificació d'objectes i personatges en l'entorn de l'NPC.
  • Algoritmes d'Evitació: Mètodes utilitzats per calcular rutes que evitin col·lisions.
  • Sensors Virtuals: Eines que simulen la percepció de l'entorn per part de l'NPC.
  • Espai de Navegació: Representació de l'entorn en el qual l'NPC pot moure's.

Detecció d'Obstacles

La detecció d'obstacles és el primer pas en l'evitació de col·lisions. Els NPCs han de ser capaços d'identificar objectes i altres personatges en el seu entorn.

  • Raycasting: Tècnica que utilitza raigs projectats des de l'NPC per detectar col·lisions.
  • Bounding Boxes: Ús de caixes delimitadores al voltant d'objectes per simplificar la detecció de col·lisions.
  • Mapes de Navegació: Representació de l'entorn que inclou informació sobre àrees navegables i obstacles.

Exemple de Raycasting en Unity

void Update() {
    RaycastHit hit;
    if (Physics.Raycast(transform.position, transform.forward, out hit, 10)) {
        Debug.Log("Obstacle detected: " + hit.collider.name);
    }
}

En aquest exemple, es projecta un raig des de la posició de l'NPC cap endavant. Si el raig col·lisiona amb un objecte en un rang de 10 unitats, es registra el nom de l'objecte.

Algoritmes d'Evitació

Un cop detectats els obstacles, el següent pas és calcular una ruta que els eviti. Existeixen diversos algoritmes per aconseguir això:

  • Algoritme de A (A-Star)*: Troba la ruta més curta evitant obstacles.
  • Algoritme de Dijkstra: Similar al A*, però més adequat per a gràfics densos.
  • Steering Behaviors: Comportaments de direcció que permeten als NPCs moure's de manera més natural.

Exemple d'Algoritme de A* en Pseudocodi

function A*(start, goal)
    openSet := {start}
    cameFrom := empty map
    gScore := map with default value of Infinity
    gScore[start] := 0
    fScore := map with default value of Infinity
    fScore[start] := heuristic(start, goal)

    while openSet is not empty
        current := node in openSet with lowest fScore[current]
        if current = goal
            return reconstruct_path(cameFrom, current)

        openSet.remove(current)
        for each neighbor of current
            tentative_gScore := gScore[current] + dist_between(current, neighbor)
            if tentative_gScore < gScore[neighbor]
                cameFrom[neighbor] := current
                gScore[neighbor] := tentative_gScore
                fScore[neighbor] := gScore[neighbor] + heuristic(neighbor, goal)
                if neighbor not in openSet
                    openSet.add(neighbor)

    return failure

Aquest pseudocodi mostra com l'algoritme A* troba la ruta més curta des d'un punt d'inici fins a un objectiu, evitant obstacles en el camí.

Sensors Virtuals

Els sensors virtuals permeten als NPCs percebre el seu entorn de manera similar a com ho farien els éssers humans o animals.

  • Sensors de Proximitat: Detecten objectes propers.
  • Sensors de Visió: Simulen la visió de l'NPC, detectant objectes en el seu camp de visió.
  • Sensors de So: Detecten sons en l'entorn, permetent als NPCs reaccionar a esdeveniments no visuals.

Exemple de Sensor de Proximitat en Unity

void OnTriggerEnter(Collider other) {
    if (other.gameObject.CompareTag("Obstacle")) {
        Debug.Log("Obstacle detected: " + other.gameObject.name);
    }
}

En aquest exemple, s'utilitza un sensor de proximitat que detecta quan un objecte amb l'etiqueta "Obstacle" entra a l'àrea de detecció de l'NPC.

Espai de Navegació

L'espai de navegació és una representació de l'entorn que defineix les àrees on els NPCs poden moure's i les àrees que han d'evitar.

  • NavMesh (Malla de Navegació): Una malla que defineix les àrees navegables en un entorn 3D.
  • Gràfics de Navegació: Representació de l'entorn com un gràfic de nodes i arestes.
  • Mapes de Calor: Representació visual de les àrees més transitades pels NPCs.

Exemple de Configuració de NavMesh en Unity

using UnityEngine;
using UnityEngine.AI;

public class NavMeshSetup : MonoBehaviour {
    void Start() {
        NavMeshSurface navMeshSurface = GetComponent<NavMeshSurface>();
        navMeshSurface.BuildNavMesh();
    }
}

En aquest exemple, es configura una malla de navegació en Unity, permetent als NPCs moure's de manera eficient en l'entorn.

Conclusió

L'evitació d'obstacles és una tècnica essencial en la intel·ligència artificial per a videojocs. A través de la detecció d'obstacles, l'ús d'algoritmes d'evitació, sensors virtuals i la configuració de l'espai de navegació, els desenvolupadors poden crear NPCs que es moguin de manera realista i eficient, millorant l'experiència de joc.
