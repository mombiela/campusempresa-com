Introducció

La planificació de tasques és un component crucial en el desenvolupament d'intel·ligència artificial (IA) per a videojocs. Permet que els personatges no jugadors (NPCs) prenguin decisions complexes i realitzin accions de manera organitzada i eficient. En aquesta secció, explorarem els conceptes clau, tècniques i exemples pràctics per implementar la planificació de tasques en videojocs.

Conceptes Clau

  • Tasques i Subtasques: Les tasques són accions que un NPC ha de realitzar, i poden descompondre's en subtasques més petites.
  • Arbres de Comportament (Behavior Trees): Estructures jeràrquiques que organitzen tasques i subtasques de manera lògica.
  • Planificació Basada en Objectius (Goal-Oriented Action Planning, GOAP): Tècnica que permet als NPCs seleccionar i planificar accions basades en objectius específics.
  • Màquines d'Estats Finits (Finite State Machines, FSM): Models que representen estats i transicions entre ells, útils per a tasques seqüencials.

Arbres de Comportament

Els arbres de comportament són una eina poderosa per organitzar i gestionar les tasques dels NPCs. Es componen de nodes que representen accions i condicions.

Exemple d'Arbre de Comportament

class Node:
    def __init__(self, name):
        self.name = name
        self.children = []

    def add_child(self, child):
        self.children.append(child)

class BehaviorTree:
    def __init__(self, root):
        self.root = root

    def execute(self):
        return self.root.execute()

class TaskNode(Node):
    def execute(self):
        print(f"Executing task: {self.name}")
        for child in self.children:
            child.execute()

# Crear nodes de tasques
root = TaskNode("Root Task")
subtask1 = TaskNode("Subtask 1")
subtask2 = TaskNode("Subtask 2")

# Construir l'arbre de comportament
root.add_child(subtask1)
root.add_child(subtask2)

# Executar l'arbre de comportament
tree = BehaviorTree(root)
tree.execute()

En aquest exemple, hem creat un arbre de comportament simple amb una tasca arrel i dues subtasques. En executar l'arbre, s'imprimeixen les tasques en ordre jeràrquic.

Planificació Basada en Objectius (GOAP)

GOAP és una tècnica avançada que permet als NPCs planificar les seves accions basant-se en objectius específics. Els NPCs avaluen els seus objectius i seleccionen les accions més adequades per assolir-los.

Exemple de GOAP

class Action:
    def __init__(self, name, cost):
        self.name = name
        self.cost = cost

    def execute(self):
        print(f"Executing action: {self.name}")

class Goal:
    def __init__(self, name):
        self.name = name
        self.actions = []

    def add_action(self, action):
        self.actions.append(action)

    def plan(self):
        # Ordenar accions per cost
        self.actions.sort(key=lambda x: x.cost)
        for action in self.actions:
            action.execute()

# Crear accions
action1 = Action("Action 1", 10)
action2 = Action("Action 2", 5)

# Crear objectiu i afegir accions
goal = Goal("Reach Goal")
goal.add_action(action1)
goal.add_action(action2)

# Planificar i executar accions
goal.plan()

En aquest exemple, hem creat un objectiu amb dues accions. Les accions s'ordenen per cost i s'executen en conseqüència.

Màquines d'Estats Finits (FSM)

Les FSM són útils per modelar comportaments seqüencials i estats dels NPCs. Cada estat representa una tasca o acció específica, i les transicions defineixen com moure's entre estats.

Exemple de FSM

class State:
    def __init__(self, name):
        self.name = name

    def execute(self):
        print(f"Executing state: {self.name}")

class FSM:
    def __init__(self):
        self.states = {}
        self.current_state = None

    def add_state(self, state):
        self.states[state.name] = state

    def set_state(self, state_name):
        self.current_state = self.states[state_name]

    def execute(self):
        if self.current_state:
            self.current_state.execute()

# Crear estats
state1 = State("State 1")
state2 = State("State 2")

# Crear FSM i afegir estats
fsm = FSM()
fsm.add_state(state1)
fsm.add_state(state2)

# Establir estat inicial i executar
fsm.set_state("State 1")
fsm.execute()
fsm.set_state("State 2")
fsm.execute()

En aquest exemple, hem creat una FSM amb dos estats i hem demostrat com canviar i executar estats.

Conclusió

La planificació de tasques és essencial per crear NPCs intel·ligents i realistes en videojocs. Les tècniques com els arbres de comportament, GOAP i FSM proporcionen diferents enfocaments per organitzar i gestionar les accions dels NPCs. Amb una comprensió sòlida d'aquestes tècniques, els desenvolupadors poden crear experiències de joc més immersives i desafiants.

Recorda practicar amb els exemples proporcionats i experimentar amb diferents configuracions per comprendre millor com funcionen aquestes tècniques en la pràctica.
