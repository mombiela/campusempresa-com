Introducció

En el desenvolupament de videojocs, la intel·ligència artificial (IA) juga un paper crucial per crear experiències immersives i desafiants. La implementació de comportaments complexos en NPCs (Non-Player Characters) és essencial per aconseguir un entorn de joc dinàmic i realista. Aquest tema abordarà les tècniques i algoritmes utilitzats per dissenyar i programar comportaments avançats en IA per a videojocs.

Conceptes Clau

  • FSM (Finite State Machines): Màquines d'estats finits que permeten als NPCs canviar entre diferents estats de comportament.
  • BT (Behavior Trees): Arbres de comportament que estructuren les decisions dels NPCs de manera jeràrquica.
  • Pathfinding: Algoritmes que permeten als NPCs trobar rutes òptimes en l'entorn del joc.
  • Steering Behaviors: Comportaments de direcció que permeten als NPCs moure's de manera realista.
  • Utility AI: Sistemes que avaluen múltiples factors per prendre decisions complexes.

Finite State Machines (FSM)

Les FSM són una de les tècniques més comunes per implementar comportaments en NPCs. Una FSM consisteix en:

  • Estats: Representen diferents comportaments o accions.
  • Transicions: Condicions que permeten canviar d'un estat a un altre.
  • Esdeveniments: Successos que desencadenen transicions.

Exemple de Codi

class NPC:
    def __init__(self):
        self.state = "idle"

    def update(self, event):
        if self.state == "idle":
            if event == "see_enemy":
                self.state = "attack"
        elif self.state == "attack":
            if event == "enemy_dead":
                self.state = "idle"

npc = NPC()
npc.update("see_enemy")
print(npc.state)  # Output: attack
npc.update("enemy_dead")
print(npc.state)  # Output: idle

Behavior Trees (BT)

Els BT són una alternativa a les FSM que permeten una major modularitat i reutilització de comportaments. Un BT es compon de:

  • Nodes: Representen accions o decisions.
  • Seqüències: Executen nodes fills en ordre fins que un falla.
  • Selectors: Executen nodes fills en ordre fins que un té èxit.

Exemple de Codi

class Node:
    def run(self):
        pass

class Sequence(Node):
    def __init__(self, nodes):
        self.nodes = nodes

    def run(self):
        for node in self.nodes:
            if not node.run():
                return False
        return True

class Selector(Node):
    def __init__(self, nodes):
        self.nodes = nodes

    def run(self):
        for node in self.nodes:
            if node.run():
                return True
        return False

class Action(Node):
    def __init__(self, action):
        self.action = action

    def run(self):
        return self.action()

def see_enemy():
    return True

def attack_enemy():
    print("Attacking enemy")
    return True

bt = Selector([Sequence([Action(see_enemy), Action(attack_enemy)])])
bt.run()  # Output: Attacking enemy

Pathfinding

El pathfinding és crucial perquè els NPCs naveguin per l'entorn del joc de manera eficient. L'algoritme A* (A-star) és un dels més utilitzats.

Exemple de Codi

from heapq import heappop, heappush

def heuristic(a, b):
    return abs(a[0] - b[0]) + abs(a[1] - b[1])

def a_star_search(graph, start, goal):
    frontier = []
    heappush(frontier, (0, start))
    came_from = {}
    cost_so_far = {}
    came_from[start] = None
    cost_so_far[start] = 0

    while frontier:
        current = heappop(frontier)[1]

        if current == goal:
            break

        for next in graph.neighbors(current):
            new_cost = cost_so_far[current] + graph.cost(current, next)
            if next not in cost_so_far or new_cost < cost_so_far[next]:
                cost_so_far[next] = new_cost
                priority = new_cost + heuristic(goal, next)
                heappush(frontier, (priority, next))
                came_from[next] = current

    return came_from, cost_so_far

Steering Behaviors

Els steering behaviors permeten als NPCs moure's de manera realista i fluida. Alguns comportaments comuns inclouen:

  • Seek: Moure's cap a un objectiu.
  • Flee: Allunyar-se d'una amenaça.
  • Arrive: Arribar a un objectiu i aturar-se suaument.

Exemple de Codi

class Vector2:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __sub__(self, other):
        return Vector2(self.x - other.x, self.y - other.y)

    def length(self):
        return (self.x**2 + self.y**2)**0.5

    def normalize(self):
        l = self.length()
        return Vector2(self.x / l, self.y / l)

class NPC:
    def __init__(self, position):
        self.position = position
        self.velocity = Vector2(0, 0)

    def seek(self, target):
        desired_velocity = (target - self.position).normalize()
        self.velocity = desired_velocity

npc = NPC(Vector2(0, 0))
target = Vector2(10, 10)
npc.seek(target)
print(npc.velocity.x, npc.velocity.y)  # Output: 0.707 0.707

Utility AI

La Utility AI permet als NPCs prendre decisions basades en múltiples factors, assignant una "utilitat" a cada possible acció.

Exemple de Codi

class UtilityAI:
    def __init__(self, actions):
        self.actions = actions

    def evaluate(self):
        best_action = None
        highest_utility = -float('inf')
        for action, utility in self.actions.items():
            if utility() > highest_utility:
                highest_utility = utility()
                best_action = action
        return best_action

def attack_utility():
    return 0.8

def flee_utility():
    return 0.5

actions = {
    "attack": attack_utility,
    "flee": flee_utility
}

ai = UtilityAI(actions)
print(ai.evaluate())  # Output: attack

Conclusió

La implementació de comportaments complexos en IA per a videojocs és un camp vast i en constant evolució. Des de les FSM i BT fins al pathfinding i els steering behaviors, cada tècnica ofereix diferents avantatges i desafiaments. L'elecció de la tècnica adequada depèn del context del joc i dels comportaments específics que es desitgen implementar. Amb una comprensió sòlida d'aquests conceptes, els desenvolupadors poden crear NPCs que millorin significativament l'experiència de joc.
