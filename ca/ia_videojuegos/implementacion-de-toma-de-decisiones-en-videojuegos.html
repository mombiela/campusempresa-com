Implementació de Presa de Decisions en Videojocs

Introducció

La presa de decisions és una part crucial de la intel·ligència artificial (IA) en videojocs. Permet que els personatges no jugables (NPCs) i altres elements del joc actuïn de manera autònoma i responguin a les accions del jugador i a l'entorn del joc. En aquesta secció, explorarem diferents tècniques i algorismes per implementar la presa de decisions en videojocs.

Tècniques de Presa de Decisions

Màquines d'Estats Finits (FSM)

Les Màquines d'Estats Finits són una de les tècniques més comunes i senzilles per a la presa de decisions en videojocs.

  • Definició: Una FSM és un model matemàtic que consisteix en un nombre finit d'estats, transicions entre aquests estats i accions.
  • Avantatges:
    • Fàcil d'implementar i entendre.
    • Adequada per a comportaments simples i predictibles.
  • Desavantatges:
    • Pot tornar-se complexa i difícil de mantenir amb comportaments més avançats.

Arbres de Decisió

Els arbres de decisió són una altra tècnica popular que permet una presa de decisions més complexa i estructurada.

  • Definició: Un arbre de decisió és una estructura d'arbre on cada node representa una decisió basada en una condició.
  • Avantatges:
    • Més flexible que les FSM.
    • Fàcil de visualitzar i entendre.
  • Desavantatges:
    • Pot tornar-se molt gran i complex.

Exemple de Codi

class DecisionTreeNode:
    def __init__(self, condition, true_branch=None, false_branch=None):
        self.condition = condition
        self.true_branch = true_branch
        self.false_branch = false_branch

    def evaluate(self, context):
        if self.condition(context):
            if self.true_branch:
                return self.true_branch.evaluate(context)
            else:
                return True
        else:
            if self.false_branch:
                return self.false_branch.evaluate(context)
            else:
                return False

# Exemple d'ús
def is_enemy_near(context):
    return context["enemy_distance"] < 10

root = DecisionTreeNode(is_enemy_near, true_branch=DecisionTreeNode(lambda x: True), false_branch=DecisionTreeNode(lambda x: False))
context = {"enemy_distance": 5}
decision = root.evaluate(context)

Arbres de Comportament (Behavior Trees)

Els arbres de comportament són una evolució dels arbres de decisió i s'utilitzen per modelar comportaments més complexos i jeràrquics.

  • Definició: Un arbre de comportament és una estructura jeràrquica de nodes que representen tasques i condicions.
  • Avantatges:
    • Modular i reutilitzable.
    • Facilita la creació de comportaments complexos.
  • Desavantatges:
    • Requereix més temps per aprendre i dominar.

Exemple de Codi

class BehaviorTreeNode:
    def run(self):
        raise NotImplementedError

class Selector(BehaviorTreeNode):
    def __init__(self, children):
        self.children = children

    def run(self):
        for child in self.children:
            if child.run():
                return True
        return False

class Sequence(BehaviorTreeNode):
    def __init__(self, children):
        self.children = children

    def run(self):
        for child in self.children:
            if not child.run():
                return False
        return True

class Condition(BehaviorTreeNode):
    def __init__(self, condition):
        self.condition = condition

    def run(self):
        return self.condition()

class Action(BehaviorTreeNode):
    def __init__(self, action):
        self.action = action

    def run(self):
        self.action()
        return True

def attack_enemy():

root = Selector([
    Sequence([

root.run()

Conclusió

La implementació de la presa de decisions en videojocs és essencial per crear experiències de joc immersives i desafiants. Les tècniques com les Màquines d'Estats Finits, els Arbres de Decisió i els Arbres de Comportament ofereixen diferents nivells de complexitat i flexibilitat, permetent als desenvolupadors triar la millor eina per a les seves necessitats específiques. En dominar aquestes tècniques, els desenvolupadors poden crear NPCs i altres elements del joc que responguin de manera intel·ligent i dinàmica a les accions del jugador i a l'entorn del joc.
