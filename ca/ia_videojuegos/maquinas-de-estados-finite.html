Introducció a les Màquines d'Estats Finits

Les Màquines d'Estats Finits (FSM, per les seves sigles en anglès) són un model matemàtic utilitzat per dissenyar sistemes lògics i de control. En el context dels videojocs, les FSM són una eina poderosa per gestionar el comportament dels personatges no jugables (NPCs), permetent que aquests reaccionin de manera coherent a diferents situacions del joc.

Conceptes Clau

  • Estat: Representa una condició o situació específica en què pot trobar-se el sistema.
  • Transició: El canvi d'un estat a un altre, generalment desencadenat per un esdeveniment o condició.
  • Esdeveniment: Una acció o condició que provoca una transició entre estats.
  • Acció: Una operació que s'executa quan s'entra, es surt o es roman en un estat.

Estructura d'una FSM

Una FSM es compon de:

  1. Conjunt d'Estats: Tots els estats possibles en què pot estar el sistema.
  2. Estat Inicial: L'estat en què comença el sistema.
  3. Esdeveniments/Condicions: Factors que provoquen transicions entre estats.
  4. Transicions: Les regles que defineixen com i quan canviar d'un estat a un altre.
  5. Accions: Les operacions que s'executen durant les transicions o mentre s'està en un estat.

Exemple de FSM en Videojocs

Considerem un NPC en un videojoc que pot estar en un dels següents estats: Patrullant, Perseguint, Atacant, Fugint.

Definició d'Estats i Transicions

| Estat | Esdeveniment/Condició | Estat Següent | Acció | |--------------|------------------------------|------------------|--------------------------------| | Patrullant | Veu el jugador | Perseguint | Començar a seguir el jugador | | Perseguint | Jugador fora de vista | Patrullant | Tornar a patrullar | | Perseguint | A prop del jugador | Atacant | Iniciar atac | | Atacant | Jugador fora d'abast | Perseguint | Tornar a seguir el jugador | | Atacant | Salut baixa | Fugint | Escapar del jugador | | Fugint | Salut recuperada | Patrullant | Tornar a patrullar |

Exemple de Codi en Python

A continuació, es presenta un exemple bàsic de com implementar una FSM per a un NPC en Python:

class NPC:
    def __init__(self):
        self.state = 'Patrullant'
    
    def see_player(self):
        if self.state == 'Patrullant':
            self.state = 'Perseguint'
            print("Començar a seguir el jugador")
    
    def lose_sight_of_player(self):
        if self.state == 'Perseguint':
            self.state = 'Patrullant'
            print("Tornar a patrullar")
    
    def reach_player(self):
        if self.state == 'Perseguint':
            self.state = 'Atacant'
            print("Iniciar atac")
    
    def player_out_of_reach(self):
        if self.state == 'Atacant':
            self.state = 'Perseguint'
            print("Tornar a seguir el jugador")
    
    def low_health(self):
        if self.state == 'Atacant':
            self.state = 'Fugint'
            print("Escapar del jugador")
    
    def health_recovered(self):
        if self.state == 'Fugint':
            self.state = 'Patrullant'
            print("Tornar a patrullar")

# Exemple d'ús
npc = NPC()
npc.see_player()  # Començar a seguir el jugador
npc.reach_player()  # Iniciar atac
npc.low_health()  # Escapar del jugador
npc.health_recovered()  # Tornar a patrullar

Avantatges i Desavantatges de les FSM

Avantatges

  • Simplicitat: Fàcil d'entendre i d'implementar.
  • Claredat: Els estats i transicions són explícits, cosa que facilita el manteniment i la depuració.
  • Determinisme: El comportament del sistema és previsible i reproduïble.

Desavantatges

  • Escalabilitat: Pot tornar-se complex i difícil de manejar amb un gran nombre d'estats i transicions.
  • Flexibilitat: Menys flexible per a comportaments molt dinàmics o adaptatius.

Conclusió

Les Màquines d'Estats Finits són una eina fonamental en el desenvolupament d'IA per a videojocs, proporcionant una estructura clara i manejable per definir el comportament dels NPCs. Tot i que tenen les seves limitacions, la seva simplicitat i claredat les fan ideals per a molts escenaris comuns en el desenvolupament de videojocs. Amb una comprensió sòlida de les FSM, els desenvolupadors poden crear personatges més realistes i reactius, millorant l'experiència de joc per als usuaris.
