Introducció

En aquest estudi de cas, explorarem com dissenyar i implementar una base de dades de comerç electrònic utilitzant PostgreSQL. Això cobrirà tot, des de la configuració de la base de dades fins a tècniques avançades de consulta. Al final d'aquest tema, hauríeu de tenir una comprensió completa de com gestionar una base de dades de comerç electrònic de manera efectiva.

Configuració de la Base de Dades

Creació de la Base de Dades

Primer, necessitem crear una nova base de dades per a la nostra aplicació de comerç electrònic.

CREATE DATABASE ecommerce_db;

Connexió a la Base de Dades

A continuació, connecteu-vos a la base de dades acabada de crear.

\c ecommerce_db;

Disseny de l'Esquema

Taules i Relacions

Una base de dades de comerç electrònic típicament inclou diverses taules clau:

  • Users: Informació sobre els clients.
  • Products: Detalls dels articles disponibles per a la venda.
  • Orders: Registres de les compres dels clients.
  • OrderItems: Articles inclosos en cada comanda.
  • Categories: Categories de productes.

Creació de Taules

Creem aquestes taules amb les relacions adequades.

Taula d'Usuaris

CREATE TABLE Users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Taula de Productes

CREATE TABLE Products (
    product_id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    price DECIMAL(10, 2) NOT NULL,
    category_id INT,
    FOREIGN KEY (category_id) REFERENCES Categories(category_id)
);

Taula de Comandes

CREATE TABLE Orders (
    order_id SERIAL PRIMARY KEY,
    user_id INT NOT NULL,
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(50) NOT NULL,
    FOREIGN KEY (user_id) REFERENCES Users(user_id)
);

Taula d'Articles de Comanda

CREATE TABLE OrderItems (
    order_item_id SERIAL PRIMARY KEY,
    order_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (order_id) REFERENCES Orders(order_id),
    FOREIGN KEY (product_id) REFERENCES Products(product_id)
);

Taula de Categories

CREATE TABLE Categories (
    category_id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

Població de la Base de Dades

Inserció de Dades

Inserim algunes dades de mostra a les nostres taules.

Inserir Categories

INSERT INTO Categories (name) VALUES ('Electronics'), ('Books'), ('Clothing');

Inserir Comandes

INSERT INTO OrderItems (order_id, product_id, quantity, price) VALUES
(1, 1, 1, 999.99),

Consulta de la Base de Dades

Consultes Bàsiques

Recuperar tots els usuaris:

SELECT * FROM Users;

Recuperar tots els productes de la categoria 'Electronics':

SELECT * FROM Products WHERE category_id = 1;

Consultes Avançades

Recuperar totes les comandes juntament amb la informació de l'usuari:

SELECT Orders.order_id, Users.username, Orders.order_date, Orders.status
FROM Orders
JOIN Users ON Orders.user_id = Users.user_id;

Recuperar les vendes totals per a cada producte:

SELECT Products.name, SUM(OrderItems.quantity * OrderItems.price) AS total_sales
FROM OrderItems
JOIN Products ON OrderItems.product_id = Products.product_id
GROUP BY Products.name;
