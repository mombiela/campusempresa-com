Optimització de consultes ►

Els índexs són una part crucial de l'optimització de bases de dades. Ajuden a accelerar la recuperació de dades proporcionant camins d'accés ràpids a les files d'una taula. En aquesta secció, cobrirem els conceptes bàsics de la creació i gestió d'índexs en PostgreSQL, progressant des de conceptes per a principiants fins a conceptes avançats.

Introducció als Índexs

  • Què és un Índex?

    • Un índex és un objecte de base de dades que millora la velocitat de les operacions de recuperació de dades en una taula.
    • És similar a l'índex d'un llibre, que t'ajuda a localitzar ràpidament la informació sense llegir tot el llibre.

  • Tipus d'Índexs en PostgreSQL

    • Índexs B-tree: El tipus per defecte i més utilitzat.
    • Índexs Hash: Útils per a comparacions d'igualtat.
    • GIN (Generalized Inverted Index): Adequat per a la cerca de text complet.
    • GiST (Generalized Search Tree): Útil per a tipus de dades complexes com dades geomètriques.
    • SP-GiST (Space-partitioned Generalized Search Tree): Optimitzat per a certs tipus de dades.
    • BRIN (Block Range INdexes): Eficient per a taules grans amb dades ordenades naturalment.

Creació d'Índexs

  • Sintaxi Bàsica

    CREATE INDEX index_name ON table_name (column_name);
    • index_name: El nom que vols donar al teu índex.
    • table_name: El nom de la taula en la qual estàs creant l'índex.
    • column_name: La columna sobre la qual es crea l'índex.

  • Exemple

    CREATE INDEX idx_employee_name ON employees (name);

    Aquesta comanda crea un índex anomenat idx_employee_name a la columna name de la taula employees.

  • Índexs Únics

    CREATE UNIQUE INDEX index_name ON table_name (column_name);
    • Assegura que tots els valors a la columna indexada siguin únics.
    CREATE UNIQUE INDEX idx_employee_email ON employees (email);

Gestió d'Índexs

  • Visualització d'Índexs

    • Per llistar tots els índexs d'una taula: 
      \d table_name
    • Exemple: 
      \d employees

  • Eliminació d'Índexs

    DROP INDEX index_name;
    • Exemple: 
      DROP INDEX idx_employee_name;

  • Canvi de Nom d'Índexs

    ALTER INDEX old_index_name RENAME TO new_index_name;
    • Exemple: 
      ALTER INDEX idx_employee_name RENAME TO idx_emp_name;

Tècniques Avançades d'Indexació

  • Índexs Parcials

    • Índexs que cobreixen només un subconjunt de files d'una taula.
    CREATE INDEX index_name ON table_name (column_name) WHERE condition;
    • Exemple: 
      CREATE INDEX idx_active_employees ON employees (name) WHERE status = 'active';

  • Índexs d'Expressió

    • Índexs basats en expressions en lloc de només columnes.
    CREATE INDEX index_name ON table_name (expression);
    • Exemple: 
      CREATE INDEX idx_lower_name ON employees (LOWER(name));

  • Creació Concurrent d'Índexs

    • Permet la creació d'índexs sense bloquejar la taula.
    CREATE INDEX CONCURRENTLY index_name ON table_name (column_name);
    • Exemple: 
      CREATE INDEX CONCURRENTLY idx_employee_name ON employees (name);

Consideracions de Rendiment

  • Quan Utilitzar Índexs

    • Els índexs són beneficiosos per a columnes utilitzades freqüentment en clàusules WHERE, JOIN, ORDER BY i GROUP BY.
    • Evita indexar columnes amb un gran nombre de valors únics si no es consulten freqüentment.

  • Quan No Utilitzar Índexs

    • Evita indexar columnes que rarament s'utilitzen en consultes.
    • Els índexs poden alentir les operacions INSERT, UPDATE i DELETE perquè l'índex també s'ha d'actualitzar.

Conclusió

Els índexs són eines poderoses per optimitzar el rendiment de la base de dades, però s'han d'utilitzar amb prudència. Entendre els diferents tipus d'índexs i els seus casos d'ús adequats pot millorar significativament l'eficiència de la teva base de dades PostgreSQL. Considera sempre els compromisos entre el rendiment de lectura i escriptura quan decideixis crear un índex.
