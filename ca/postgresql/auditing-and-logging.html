Auditoria i Registre en PostgreSQL
Extensions i mòduls ►

Introducció

L'auditoria i el registre són components crucials en la gestió de bases de dades, proporcionant informació sobre les activitats de la base de dades i ajudant en la resolució de problemes, l'optimització del rendiment i el monitoratge de la seguretat. Aquesta secció et guiarà des de les tècniques bàsiques fins a les avançades per a l'auditoria i el registre en PostgreSQL.

Conceptes Clau

  • Registre: Enregistrament d'esdeveniments de la base de dades, errors i altres activitats significatives.
  • Auditoria: Seguiment d'accions específiques realitzades a la base de dades, sovint per motius de seguretat i compliment.

Registre en PostgreSQL

Configuració del Registre

PostgreSQL utilitza el fitxer postgresql.conf per configurar el registre. Els paràmetres clau inclouen:

  • log_destination: Defineix on enviar la sortida del registre (per exemple, stderr, csvlog, syslog).
  • logging_collector: Habilita la recollida de missatges de registre en fitxers de registre.
  • log_directory: Especifica el directori on es guardaran els fitxers de registre.
  • log_filename: Defineix el patró de nom per als fitxers de registre.
  • log_statement: Controla quines instruccions SQL es registren (none, ddl, mod, all).

Exemple de configuració en postgresql.conf:

log_destination = 'stderr'
logging_collector = on
log_directory = 'pg_log'
log_filename = 'postgresql-%Y-%m-%d_%H%M%S.log'
log_statement = 'all'

Nivells de Registre

PostgreSQL admet diversos nivells de registre:

  • DEBUG
  • INFO
  • NOTICE
  • WARNING
  • ERROR
  • LOG
  • FATAL
  • PANIC

Exemple de configuració del nivell de registre:

log_min_messages = 'WARNING'

Rotació de Registres

La rotació de registres ajuda a gestionar la mida dels fitxers de registre i assegura que els registres es desin arxivadament.

  • log_rotation_age: Edat màxima d'un fitxer de registre abans de ser rotat.
  • log_rotation_size: Mida màxima d'un fitxer de registre abans de ser rotat.

Exemple de configuració:

log_rotation_age = 1d
log_rotation_size = 10MB

Auditoria en PostgreSQL

Ús de `pgAudit`

pgAudit (PostgreSQL Audit) és una extensió que proporciona un registre detallat de sessions i objectes d'auditoria.

  • Instal·lació: Assegura't que pgAudit està instal·lat i configurat.
  • Configuració: Habilita pgAudit en postgresql.conf.

Exemple de configuració:

shared_preload_libraries = 'pgaudit'
pgaudit.log = 'all'

Auditoria d'Accions Específiques

Pots configurar pgAudit per registrar tipus específics d'accions:

  • READ: Registra instruccions SELECT.
  • WRITE: Registra instruccions INSERT, UPDATE, DELETE i TRUNCATE.
  • FUNCTION: Registra crides a funcions.
  • ROLE: Registra canvis de rol i permisos.

Exemple de configuració:

pgaudit.log = 'read, write'

Exemple: Auditoria d'Accés a Taules

Per auditar l'accés a una taula específica:

  1. Crea la taula:

    CREATE TABLE sensitive_data (
    id SERIAL PRIMARY KEY,
    info TEXT
);

  2. Configura pgAudit per registrar l'accés:

    pgaudit.log = 'read, write'

  3. Realitza operacions i comprova els registres:

    INSERT INTO sensitive_data (info) VALUES ('Confidential');
SELECT * FROM sensitive_data;

Els registres capturaran aquestes operacions, proporcionant una pista d'auditoria.

Conclusió

L'auditoria i el registre en PostgreSQL són essencials per mantenir la seguretat, el rendiment i el compliment de la base de dades. Configurant els paràmetres de registre i utilitzant eines com pgAudit, pots obtenir informació valuosa sobre les activitats de la base de dades i assegurar-te que totes les accions crítiques es registren i es controlen. Aquest coneixement és fonamental tant per als administradors de bases de dades com per als desenvolupadors per gestionar i assegurar eficaçment les bases de dades PostgreSQL.
