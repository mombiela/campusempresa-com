Introducció

En aquest estudi de cas, explorarem com gestionar transaccions financeres utilitzant PostgreSQL. Això inclourà la creació de taules, la inserció de dades, la realització de consultes i l'assegurament de la integritat de les dades. Al final d'aquesta secció, tindràs una comprensió sòlida de com manejar dades financeres en una base de dades relacional.

Configuració de la Base de Dades

Creació de la Base de Dades

Primer, creem una nova base de dades per a les nostres transaccions financeres.

CREATE DATABASE financial_transactions;

Connexió a la Base de Dades

Connecta't a la base de dades acabada de crear.

\c financial_transactions;

Creació de Taules

Taula de Comptes

Necessitem una taula per emmagatzemar la informació dels comptes.

CREATE TABLE accounts (
    account_id SERIAL PRIMARY KEY,
    account_name VARCHAR(100) NOT NULL,
    balance NUMERIC(15, 2) NOT NULL DEFAULT 0.00
);
  • account_id: Identificador únic per a cada compte.
  • account_name: Nom del titular del compte.
  • balance: Saldo actual del compte.

Taula de Transaccions

A continuació, creem una taula per emmagatzemar els detalls de les transaccions.

CREATE TABLE transactions (
    transaction_id SERIAL PRIMARY KEY,
    account_id INT REFERENCES accounts(account_id),
    transaction_date TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    amount NUMERIC(15, 2) NOT NULL,
    transaction_type VARCHAR(10) CHECK (transaction_type IN ('credit', 'debit'))
);
  • transaction_id: Identificador únic per a cada transacció.
  • account_id: Clau forana que fa referència a la taula accounts.
  • transaction_date: Data i hora de la transacció.
  • amount: Import de la transacció.
  • transaction_type: Tipus de transacció, ja sigui 'credit' o 'debit'.

Inserció de Dades

Afegir Comptes

Afegim alguns comptes a la nostra taula accounts.

INSERT INTO accounts (account_name, balance) VALUES
('Alice', 1000.00),
('Bob', 1500.00),
('Charlie', 2000.00);

Afegir Transaccions

Ara, afegirem algunes transaccions.

INSERT INTO transactions (account_id, amount, transaction_type) VALUES
(1, 200.00, 'credit'),
(2, 300.00, 'debit'),
(3, 150.00, 'credit'),
(1, 50.00, 'debit');

Consultes de Dades

Veure Saldo dels Comptes

Per veure el saldo actual de tots els comptes:

SELECT * FROM accounts;

Veure Transaccions

Per veure totes les transaccions:

SELECT * FROM transactions;

Unir Taules

Per obtenir una vista detallada de les transaccions juntament amb els noms dels comptes:

SELECT t.transaction_id, a.account_name, t.transaction_date, t.amount, t.transaction_type
FROM transactions t
JOIN accounts a ON t.account_id = a.account_id;

Assegurar la Integritat de les Dades

Utilitzar Transaccions

Per assegurar la integritat de les dades durant operacions complexes, utilitza transaccions.

BEGIN;

UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
INSERT INTO transactions (account_id, amount, transaction_type) VALUES (1, 100.00, 'debit');

COMMIT;

Exemple de Rollback

Si alguna cosa va malament, pots fer un rollback de la transacció.

BEGIN;

UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
-- Simular un error
-- INSERT INTO transactions (account_id, amount, transaction_type) VALUES (1, 'invalid_amount', 'debit');

ROLLBACK;

Conclusió

En aquest estudi de cas, hem cobert els conceptes bàsics de la gestió de transaccions financeres en PostgreSQL. Hem creat taules per a comptes i transaccions, hem inserit dades, hem realitzat consultes i hem assegurat la integritat de les dades utilitzant transaccions. Aquest coneixement fonamental t'ajudarà a gestionar dades financeres de manera efectiva en una base de dades relacional.
