Introducció

Dissenyar una base de dades és un pas crític en la creació d'una aplicació robusta i eficient. En aquesta secció, cobrirem els conceptes fonamentals i les millors pràctiques per dissenyar una base de dades en PostgreSQL. Això inclourà la comprensió del modelatge de dades, la normalització i la creació de taules amb tipus de dades i restriccions adequades.

Conceptes Clau

Modelatge de Dades

  • Entitats i Relacions: Identificar les principals entitats (taules) i les seves relacions.
  • Diagrames ER: Utilitzar diagrames Entitat-Relació per visualitzar l'estructura de la base de dades.

Normalització

  • Primera Forma Normal (1NF): Assegurar que cada columna contingui valors atòmics i cada registre sigui únic.
  • Segona Forma Normal (2NF): Assegurar que tots els atributs no clau depenguin funcionalment de la clau primària.
  • Tercera Forma Normal (3NF): Assegurar que tots els atributs només depenguin de la clau primària.

Tipus de Dades

  • Tipus de Dades Primitives: Enter, Booleà, Text, etc.
  • Tipus de Dades Compostes: Arrays, JSON, etc.
  • Tipus de Dades Personalitzades: Creació de tipus definits per l'usuari.

Restriccions

  • Clau Primària: Identifica de manera única cada registre en una taula.
  • Clau Forana: Assegura la integritat referencial entre taules.
  • Únic: Assegura que tots els valors en una columna siguin únics.
  • No Nul: Assegura que una columna no pugui tenir valors NULL.
  • Comprovació: Assegura que tots els valors en una columna compleixin una condició específica.

Creació de Taules

Creació Bàsica de Taules

Per crear una taula en PostgreSQL, s'utilitza la instrucció CREATE TABLE. Aquí teniu un exemple:

CREATE TABLE users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
  • user_id SERIAL PRIMARY KEY: Crea una clau primària auto-incremental.
  • username VARCHAR(50) UNIQUE NOT NULL: Assegura que el nom d'usuari sigui únic i no nul.
  • email VARCHAR(100) UNIQUE NOT NULL: Assegura que el correu electrònic sigui únic i no nul.
  • created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP: Estableix un segell de temps per defecte per quan es crea el registre.

Afegir Restriccions

Les restriccions es poden afegir en el moment de la creació de la taula o posteriorment utilitzant la instrucció ALTER TABLE.

ALTER TABLE users
ADD CONSTRAINT email_check CHECK (email LIKE '%@%');

Això assegura que la columna de correu electrònic contingui un caràcter '@'.

Claus Foranes

Les claus foranes s'utilitzen per enllaçar dues taules. Aquí teniu un exemple:

CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
  • user_id INT REFERENCES users(user_id): Crea una clau forana que fa referència a la columna user_id en la taula users.

Exemple: Disseny d'una Base de Dades Simple de Comerç Electrònic

Diagrama ER

Abans de crear taules, és útil dibuixar un diagrama ER. Per a una base de dades simple de comerç electrònic, podríem tenir les següents entitats:

  • Usuaris
  • Productes
  • Comandes
  • Articles de Comanda

Creació de Taules

CREATE TABLE products (
    product_id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    price NUMERIC(10, 2) NOT NULL,
    stock INT NOT NULL
);

CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE order_items (
    order_item_id SERIAL PRIMARY KEY,
    order_id INT REFERENCES orders(order_id),
    product_id INT REFERENCES products(product_id),
    quantity INT NOT NULL,
    price NUMERIC(10, 2) NOT NULL
);

Explicació

  • Taula de Productes: Conté detalls sobre els productes.
  • Taula de Comandes: Conté detalls sobre les comandes fetes pels usuaris.
  • Taula d'Articles de Comanda: Conté detalls sobre els articles en cada comanda, enllaçant comandes i productes.

Conclusió

Dissenyar una base de dades implica una planificació acurada i la comprensió de les dades i les seves relacions. Seguint les millors pràctiques com la normalització i utilitzant tipus de dades i restriccions adequades, es pot crear una base de dades robusta i eficient. En PostgreSQL, les instruccions CREATE TABLE i ALTER TABLE són eines fonamentals per definir i modificar l'esquema de la base de dades.
