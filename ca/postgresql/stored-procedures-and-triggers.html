Introducció

Els procediments emmagatzemats i els triggers són components essencials en PostgreSQL que et permeten automatitzar i encapsular la lògica de negoci a nivell de base de dades. Aquesta secció et guiarà des dels conceptes bàsics fins a l'ús avançat dels procediments emmagatzemats i els triggers.

Procediments Emmagatzemats

Què són els Procediments Emmagatzemats?

  • Els procediments emmagatzemats són un conjunt de sentències SQL que es poden emmagatzemar a la base de dades i executar-se com una unitat única.
  • Ajuden a encapsular lògica de negoci complexa, millorar el rendiment i assegurar la reutilització del codi.

Crear un Procediment Emmagatzemat Simple

Per crear un procediment emmagatzemat en PostgreSQL, utilitzes la sentència CREATE PROCEDURE.

CREATE PROCEDURE add_numbers(IN num1 INT, IN num2 INT, OUT sum INT)
LANGUAGE plpgsql
AS $$
BEGIN
    sum := num1 + num2;
END;
$$;
  • Els paràmetres IN són entrades per al procediment.
  • Els paràmetres OUT són sortides del procediment.
  • LANGUAGE plpgsql especifica que el procediment està escrit en PL/pgSQL.

Executar un Procediment Emmagatzemat

Per executar un procediment emmagatzemat, utilitzes la sentència CALL.

CALL add_numbers(5, 10, sum);

Avantatges dels Procediments Emmagatzemats

  • Rendiment: Reducció de la comunicació client-servidor.
  • Seguretat: Encapsulació de la lògica de negoci.
  • Mantenibilitat: Gestió centralitzada del codi.

Procediments Emmagatzemats Avançats

Els procediments emmagatzemats també poden incloure estructures de control com bucles i sentències condicionals.

CREATE PROCEDURE calculate_factorial(IN num INT, OUT result INT)
LANGUAGE plpgsql
AS $$
DECLARE
    i INT := 1;
BEGIN
    result := 1;
    WHILE i <= num LOOP
        result := result * i;
        i := i + 1;
    END LOOP;
END;
$$;

Triggers

Què són els Triggers?

  • Els triggers són tipus especials de procediments emmagatzemats que s'executen automàticament o "es disparen" quan es produeixen certs esdeveniments a la base de dades.
  • Els esdeveniments poden ser INSERT, UPDATE o DELETE.

Crear un Trigger Simple

Per crear un trigger, necessites definir una funció de trigger i després vincular-la a un esdeveniment de taula.

Pas 1: Crear una Funció de Trigger

CREATE FUNCTION log_changes() RETURNS trigger AS $$
BEGIN
    IF TG_OP = 'INSERT' THEN
        INSERT INTO audit_log(table_name, operation, new_data)
        VALUES (TG_TABLE_NAME, TG_OP, row_to_json(NEW));
    ELSIF TG_OP = 'UPDATE' THEN
        INSERT INTO audit_log(table_name, operation, old_data, new_data)
        VALUES (TG_TABLE_NAME, TG_OP, row_to_json(OLD), row_to_json(NEW));
    ELSIF TG_OP = 'DELETE' THEN
        INSERT INTO audit_log(table_name, operation, old_data)
        VALUES (TG_TABLE_NAME, TG_OP, row_to_json(OLD));
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

Pas 2: Vincular la Funció de Trigger a un Esdeveniment de Taula

CREATE TRIGGER audit_trigger
AFTER INSERT OR UPDATE OR DELETE ON your_table
FOR EACH ROW EXECUTE FUNCTION log_changes();

Avantatges dels Triggers

  • Automatització: Aplicació automàtica de regles de negoci.
  • Consistència: Assegurament de la integritat de les dades.
  • Auditabilitat: Seguiment dels canvis en les dades.

Triggers Avançats

Els triggers es poden utilitzar per a operacions complexes com actualitzacions en cascada o manteniment de taules resum.

CREATE FUNCTION update_summary() RETURNS trigger AS $$
BEGIN
    UPDATE summary_table
    SET total = total + NEW.amount
    WHERE id = NEW.id;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_summary_trigger
AFTER INSERT ON transactions
FOR EACH ROW EXECUTE FUNCTION update_summary();

Conclusió

Els procediments emmagatzemats i els triggers són eines poderoses en PostgreSQL que ajuden a automatitzar i encapsular lògica de negoci complexa. Milloren el rendiment, la seguretat i la mantenibilitat de les teves aplicacions de base de dades. En dominar aquestes eines, pots crear sistemes de bases de dades més eficients i robustos.
