Introducció

La replicació i l'alta disponibilitat són conceptes crítics en la gestió de bases de dades, assegurant que la teva base de dades PostgreSQL romangui accessible i resilient davant fallades de maquinari, errors de programari o altres interrupcions. Aquesta secció et guiarà a través dels conceptes bàsics de la replicació, diferents mètodes de replicació i estratègies per aconseguir alta disponibilitat en PostgreSQL.

Conceptes Clau

  • Replicació: El procés de copiar dades d'un servidor de bases de dades (primari) a un altre (rèplica).
  • Alta Disponibilitat (HA): Assegurar que el sistema de bases de dades estigui contínuament operatiu amb un temps d'inactivitat mínim.
  • Servidor Primari: El servidor principal on es realitzen totes les operacions d'escriptura.
  • Servidor Rèplica: Un servidor que manté una còpia de les dades del servidor primari, generalment per a operacions de lectura o per a finalitats de failover.
  • Failover: El procés de canviar a un servidor rèplica quan el servidor primari falla.

Tipus de Replicació

PostgreSQL admet diversos mètodes de replicació, cadascun amb els seus propis casos d'ús i configuracions.

Replicació en Streaming

La replicació en streaming és un mètode on el servidor primari envia contínuament canvis al servidor rèplica en temps real.

  • Avantatges:

    • Replicació de dades en temps real.
    • Baixa latència.
    • Adequat per a configuracions d'alta disponibilitat.

  • Desavantatges:

    • Requereix una connexió de xarxa estable.
    • La rèplica pot estar lleugerament darrere del servidor primari.

Exemple de Configuració

  1. Configuració del Servidor Primari:

    # postgresql.conf
wal_level = replica
max_wal_senders = 3
wal_keep_segments = 64
hot_standby = on

# pg_hba.conf
host replication replicator 192.168.1.0/24 md5

  2. Configuració del Servidor Rèplica:

    pg_basebackup -h primary_ip -D /var/lib/postgresql/12/main -U replicator -P -v --wal-method=stream

    # recovery.conf
standby_mode = 'on'
primary_conninfo = 'host=primary_ip port=5432 user=replicator password=yourpassword'
trigger_file = '/tmp/trigger'

Replicació Lògica

La replicació lògica permet més flexibilitat replicant canvis de dades basats en canvis lògics en lloc de canvis físics.

  • Avantatges:

    • Permet la replicació selectiva de taules.
    • Pot replicar entre diferents versions de PostgreSQL.
    • Adequat per a processos de magatzem de dades i ETL.

  • Desavantatges:

    • Configuració més complexa.
    • Potencialment major latència en comparació amb la replicació en streaming.

Exemple de Configuració

  1. Configuració del Servidor Primari:

    # postgresql.conf
wal_level = logical
max_replication_slots = 4
max_wal_senders = 4

# pg_hba.conf
host replication replicator 192.168.1.0/24 md5

  2. Creació d'una Publicació:

    CREATE PUBLICATION my_publication FOR TABLE my_table;

  3. Configuració del Servidor Rèplica:

    CREATE SUBSCRIPTION my_subscription CONNECTION 'host=primary_ip dbname=mydb user=replicator password=yourpassword' PUBLICATION my_publication;

Estratègies d'Alta Disponibilitat

L'alta disponibilitat en PostgreSQL implica més que només replicació. Inclou monitoratge, failover automàtic i equilibrament de càrrega.

Monitoratge

Les eines de monitoratge ajuden a detectar fallades i problemes de rendiment. Eines comunes inclouen:

  • pg_stat_replication: Proporciona informació sobre l'estat de la replicació.
  • pg_stat_activity: Mostra l'activitat actual de la base de dades.
  • Eines de tercers: Eines com Nagios, Zabbix i Prometheus poden ser utilitzades per a un monitoratge complet.

Failover Automàtic

El failover automàtic assegura que un servidor rèplica prengui el relleu quan el servidor primari falla.

  • Eines:
    • Patroni: Una solució d'alta disponibilitat per a PostgreSQL.
    • repmgr: Un conjunt d'eines de codi obert per gestionar la replicació i el failover.

Exemple amb Patroni

  1. Instal·lar Patroni:

    pip install patroni[etcd]

  2. Fitxer de Configuració (patroni.yml):

    scope: my_cluster
namespace: /db/
name: postgresql0

etcd:
  host: 127.0.0.1:2379

postgresql:
  listen: 127.0.0.1:5432
  connect_address: 127.0.0.1:5432
  data_dir: /var/lib/postgresql/12/main
  config_dir: /etc/postgresql/12/main
  bin_dir: /usr/lib/postgresql/12/bin
  authentication:
    replication:
      username: replicator
      password: yourpassword
    superuser:
      username: postgres
      password: yourpassword

Equilibrament de Càrrega

L'equilibrament de càrrega distribueix les consultes de lectura entre diversos servidors rèplica per millorar el rendiment.

  • pgpool-II: Un middleware que proporciona agrupació de connexions, equilibrament de càrrega i gestió de replicació.

Exemple amb pgpool-II

  1. Instal·lar pgpool-II:

    sudo apt-get install pgpool2

  2. Fitxer de Configuració (pgpool.conf):

    listen_addresses = '*'
port = 9999
backend_hostname0 = 'primary_ip'
backend_port0 = 5432
backend_weight0 = 1
backend_data_directory0 = '/var/lib/postgresql/12/main'
backend_flag0 = 'ALLOW_TO_FAILOVER'

backend_hostname1 = 'replica_ip'
backend_port1 = 5432
backend_weight1 = 1
backend_data_directory1 = '/var/lib/postgresql/12/main'
backend_flag1 = 'ALLOW_TO_FAILOVER'

Conclusió

La replicació i l'alta disponibilitat són essencials per mantenir un sistema de bases de dades PostgreSQL resilient i amb bon rendiment. Comprenent i implementant els diferents mètodes de replicació i estratègies d'alta disponibilitat, pots assegurar que la teva base de dades romangui operativa i pugui gestionar fallades de manera elegant. Tant si tries la replicació en streaming per a la consistència de dades en temps real com la replicació lògica per a més flexibilitat, combinant aquestes amb mecanismes robustos de monitoratge i failover t'ajudarà a aconseguir un entorn PostgreSQL altament disponible.
