Els índexs són un component crucial en els sistemes de gestió de bases de dades, incloent PostgreSQL. Milloren la velocitat de les operacions de recuperació de dades proporcionant accés ràpid a les files d'una taula. Aquesta secció cobrirà els conceptes bàsics dels índexs, els seus tipus i com utilitzar-los de manera efectiva en PostgreSQL.

Què és un índex?

Un índex és un objecte de base de dades que millora la velocitat de les operacions de recuperació de dades en una taula a costa d'un emmagatzematge addicional i una sobrecàrrega de manteniment. Pensa en un índex com un mapa de carreteres que t'ajuda a localitzar ràpidament les dades que necessites sense escanejar tota la taula.

Conceptes Clau

  • Indexació: El procés de crear un índex en una columna de taula per accelerar la recuperació de dades.
  • Índex B-tree: El tipus d'índex per defecte i més utilitzat en PostgreSQL.
  • Índex únic: Assegura que les columnes indexades continguin valors únics.
  • Índex compost: Un índex en múltiples columnes.
  • Índex parcial: Un índex construït en un subconjunt d'una taula, definit per una condició.
  • Índex d'expressió: Un índex basat en una expressió en lloc d'un valor de columna.

Tipus d'índexs en PostgreSQL

PostgreSQL admet diversos tipus d'índexs, cadascun adequat per a diferents casos d'ús.

Índex B-tree

El tipus d'índex per defecte en PostgreSQL, adequat per a la majoria de consultes.

CREATE INDEX idx_column_name ON table_name(column_name);

Índex únic

Assegura que tots els valors en la columna indexada siguin únics.

CREATE UNIQUE INDEX idx_unique_column_name ON table_name(column_name);

Índex compost

Un índex en múltiples columnes, útil per a consultes que filtren en més d'una columna.

CREATE INDEX idx_composite_columns ON table_name(column1, column2);

Índex parcial

Un índex en un subconjunt de files, definit per una condició.

CREATE INDEX idx_partial ON table_name(column_name) WHERE condition;

Índex d'expressió

Un índex basat en una expressió en lloc d'un valor de columna directe.

CREATE INDEX idx_expression ON table_name((lower(column_name)));

Creació i Gestió d'Índexs

Creació d'un Índex

Per crear un índex, utilitza la instrucció CREATE INDEX. Aquí tens un exemple de creació d'un índex B-tree en la columna name d'una taula users:

CREATE INDEX idx_users_name ON users(name);

Eliminació d'un Índex

Per eliminar un índex, utilitza la instrucció DROP INDEX:

DROP INDEX idx_users_name;

Visualització d'Índexs

Per veure els índexs en una taula, pots consultar el catàleg de sistema pg_indexes:

SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'users';

Quan Utilitzar Índexs

Els índexs són beneficiosos per a:

  • Accelerar consultes SELECT: Especialment aquelles amb claus WHERE, JOINs i claus ORDER BY.
  • Enforçar la unicitat: Utilitzant índexs únics per assegurar la integritat de les dades.

No obstant això, els índexs tenen desavantatges:

  • Sobrecàrrega d'emmagatzematge: Els índexs consumeixen espai addicional en disc.
  • Sobrecàrrega de manteniment: Els índexs necessiten ser actualitzats cada vegada que les columnes indexades són modificades, la qual cosa pot alentir les operacions INSERT, UPDATE i DELETE.

Millors Pràctiques

  • Indexar selectivament: Només indexa columnes que s'utilitzen freqüentment en consultes.
  • Monitoritzar el rendiment: Utilitza la comanda EXPLAIN de PostgreSQL per analitzar el rendiment de les consultes i determinar si els índexs s'estan utilitzant de manera efectiva.
  • Evitar la sobreindexació: Massa índexs poden degradar el rendiment a causa de l'augment de la sobrecàrrega de manteniment.

Conclusió

Els índexs són eines poderoses per optimitzar el rendiment de les consultes en PostgreSQL. Comprenent els diferents tipus d'índexs i els seus casos d'ús, pots prendre decisions informades sobre quan i com utilitzar-los. Recorda equilibrar els beneficis d'una recuperació de dades més ràpida amb els costos d'emmagatzematge addicional i manteniment. Utilitza els índexs amb prudència per mantenir un sistema de bases de dades eficient i amb bon rendiment.
