Particionament de Taules
Replicació i alta disponibilitat ►

El particionament de taules a PostgreSQL és una característica potent que et permet dividir taules grans en peces més petites i manejables, anomenades particions. Això pot millorar el rendiment, la gestió i l'escalabilitat. En aquest tema, tractarem els conceptes bàsics del particionament de taules, diferents mètodes de particionament i com implementar-los a PostgreSQL.

Introducció al Particionament de Taules

  • Definició: El particionament és el procés de dividir una taula gran en peces més petites i manejables, anomenades particions, que poden millorar el rendiment de les consultes i simplificar la gestió de dades.
  • Beneficis:
    • Millora del rendiment de les consultes escanejant només les particions rellevants.
    • Manteniment més fàcil permetent operacions en particions individuals.
    • Millor organització i gestió de dades.

Tipus de Particionament

PostgreSQL admet diversos tipus de particionament:

  1. Particionament per Rang: Divideix les dades basant-se en un rang de valors.
  2. Particionament per Llista: Divideix les dades basant-se en una llista de valors.
  3. Particionament per Hash: Divideix les dades basant-se en una funció hash.
  4. Particionament Compost: Combina múltiples mètodes de particionament.

Particionament per Rang

El particionament per rang és útil quan vols dividir les dades basant-te en un rang continu de valors, com ara dates o rangs numèrics.

Exemple

CREATE TABLE sales (
    id SERIAL PRIMARY KEY,
    sale_date DATE NOT NULL,
    amount NUMERIC
) PARTITION BY RANGE (sale_date);

CREATE TABLE sales_2022 PARTITION OF sales
    FOR VALUES FROM ('2022-01-01') TO ('2023-01-01');

CREATE TABLE sales_2023 PARTITION OF sales
    FOR VALUES FROM ('2023-01-01') TO ('2024-01-01');

En aquest exemple:

  • La taula sales està particionada per la columna sale_date.
  • Es creen dues particions: sales_2022 i sales_2023, cadascuna cobrint un rang específic de dates.

Particionament per Llista

El particionament per llista és útil quan vols dividir les dades basant-te en un conjunt discret de valors, com ara categories o regions.

Exemple

CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    department TEXT NOT NULL,
    name TEXT
) PARTITION BY LIST (department);

CREATE TABLE employees_sales PARTITION OF employees
    FOR VALUES IN ('Sales');

CREATE TABLE employees_hr PARTITION OF employees
    FOR VALUES IN ('HR');

En aquest exemple:

  • La taula employees està particionada per la columna department.
  • Es creen dues particions: employees_sales i employees_hr, cadascuna cobrint departaments específics.

Particionament per Hash

El particionament per hash és útil quan vols distribuir uniformement les dades a través d'un nombre de particions utilitzant una funció hash.

Exemple

CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    customer_id INT NOT NULL,
    order_date DATE
) PARTITION BY HASH (customer_id);

CREATE TABLE orders_p0 PARTITION OF orders
    FOR VALUES WITH (MODULUS 4, REMAINDER 0);

CREATE TABLE orders_p1 PARTITION OF orders
    FOR VALUES WITH (MODULUS 4, REMAINDER 1);

CREATE TABLE orders_p2 PARTITION OF orders
    FOR VALUES WITH (MODULUS 4, REMAINDER 2);

CREATE TABLE orders_p3 PARTITION OF orders
    FOR VALUES WITH (MODULUS 4, REMAINDER 3);

En aquest exemple:

  • La taula orders està particionada per la columna customer_id utilitzant una funció hash.
  • Es creen quatre particions, cadascuna gestionant un residu diferent de la funció hash.

Particionament Compost

El particionament compost combina múltiples mètodes de particionament per aconseguir una organització de dades més complexa.

Exemple

CREATE TABLE logs (
    id SERIAL PRIMARY KEY,
    log_date DATE NOT NULL,
    region TEXT NOT NULL,
    message TEXT
) PARTITION BY RANGE (log_date);

CREATE TABLE logs_2022 PARTITION OF logs
    FOR VALUES FROM ('2022-01-01') TO ('2023-01-01')
    PARTITION BY LIST (region);

CREATE TABLE logs_2022_us PARTITION OF logs_2022
    FOR VALUES IN ('US');

CREATE TABLE logs_2022_eu PARTITION OF logs_2022
    FOR VALUES IN ('EU');

En aquest exemple:

  • La taula logs està primer particionada per la columna log_date utilitzant particionament per rang.
  • La partició logs_2022 està ulteriorment particionada per la columna region utilitzant particionament per llista.

Gestió de Particions

  • Afegir Particions: Pots afegir noves particions a mesura que les teves dades creixen.
  • Eliminar Particions: Pots eliminar particions que ja no són necessàries.
  • Adjuntar/Desadjuntar Particions: Pots adjuntar o desadjuntar particions a/de una taula particionada.

Exemple: Afegir una Partició

CREATE TABLE sales_2024 PARTITION OF sales
    FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');

Exemple: Eliminar una Partició

DROP TABLE sales_2022;

Conclusió

El particionament de taules a PostgreSQL és una característica potent que pot millorar significativament el rendiment i la gestió de grans conjunts de dades. Entenent i implementant diferents mètodes de particionament—rang, llista, hash i compost—pots optimitzar la teva base de dades per a un millor rendiment de les consultes i un manteniment més fàcil. Recorda planificar acuradament la teva estratègia de particionament basant-te en el teu cas d'ús específic i la distribució de dades.
