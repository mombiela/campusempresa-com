Introducció

La gestió d'usuaris i rols són components essencials de la seguretat i administració de bases de dades a PostgreSQL. Aquesta secció et guiarà a través dels conceptes bàsics de creació i gestió d'usuaris, assignació de rols i configuració de permisos des de nivells bàsics fins a avançats.

Creació d'Usuaris

Crear usuaris a PostgreSQL implica utilitzar la comanda CREATE USER. Els usuaris són entitats que poden connectar-se a la base de dades i realitzar operacions.

Creació Bàsica d'Usuaris

Per crear un nou usuari, pots utilitzar la següent sintaxi:

CREATE USER username WITH PASSWORD 'password';

Exemple:

CREATE USER john_doe WITH PASSWORD 'securepassword123';
  • username: El nom de l'usuari que vols crear.
  • password: La contrasenya per a l'usuari.

Visualització d'Usuaris

Per llistar tots els usuaris a la base de dades PostgreSQL, pots consultar la taula pg_user:

SELECT * FROM pg_user;

Gestió de Rols

Els rols a PostgreSQL són una manera d'agrupament d'usuaris i gestió de permisos de manera més eficient. Els rols poden ser assignats a usuaris, i els permisos poden ser atorgats als rols.

Creació de Rols

Per crear un nou rol, utilitza la comanda CREATE ROLE:

CREATE ROLE role_name;

Exemple:

CREATE ROLE read_only;

Assignació de Rols a Usuaris

Per assignar un rol a un usuari, utilitza la comanda GRANT:

GRANT role_name TO username;

Exemple:

GRANT read_only TO john_doe;

Visualització de Rols

Per llistar tots els rols a la base de dades PostgreSQL, pots consultar la taula pg_roles:

SELECT * FROM pg_roles;

Configuració de Permisos

Els permisos determinen quines accions un usuari o rol pot realitzar sobre objectes de la base de dades com taules, vistes i seqüències.

Atorgament de Permisos

Per atorgar permisos, utilitza la comanda GRANT. Aquí tens alguns permisos comuns:

  • SELECT: Permet llegir dades d'una taula.
  • INSERT: Permet inserir dades en una taula.
  • UPDATE: Permet actualitzar dades en una taula.
  • DELETE: Permet eliminar dades d'una taula.

Exemple:

GRANT SELECT, INSERT ON table_name TO role_name;

Revocació de Permisos

Per revocar permisos, utilitza la comanda REVOKE:

REVOKE permission ON table_name FROM role_name;

Exemple:

REVOKE INSERT ON employees FROM read_only;

Gestió Avançada d'Usuaris

La gestió avançada d'usuaris implica escenaris més complexos com la creació de superusuaris, la gestió d'atributs d'usuaris i l'ús d'esquemes per a una millor organització.

Creació de Superusuaris

Els superusuaris tenen tots els privilegis i poden realitzar qualsevol acció a la base de dades. Utilitza la comanda CREATE USER amb l'atribut SUPERUSER:

CREATE USER admin WITH PASSWORD 'adminpassword' SUPERUSER;

Gestió d'Atributs d'Usuaris

Pots configurar diversos atributs per als usuaris, com límits de connexió i dates de caducitat.

Exemple:

ALTER USER john_doe WITH CONNECTION LIMIT 5 VALID UNTIL '2023-12-31';

Ús d'Esquemes

Els esquemes ajuden a organitzar objectes de la base de dades i gestionar permisos a un nivell superior.

Exemple:

CREATE SCHEMA marketing;
GRANT USAGE ON SCHEMA marketing TO read_only;

Resum

En aquesta secció, hem cobert els conceptes essencials de la gestió d'usuaris i rols a PostgreSQL, des de la creació d'usuaris i rols fins a la configuració i revocació de permisos. Entendre aquests conceptes és crucial per mantenir un entorn de base de dades segur i ben organitzat. En dominar la gestió d'usuaris i rols, pots assegurar-te que la teva base de dades PostgreSQL sigui tant segura com eficient.
