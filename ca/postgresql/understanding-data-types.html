Introducció

Els tipus de dades són fonamentals per a qualsevol sistema de bases de dades, incloent PostgreSQL. Defineixen el tipus de dades que es poden emmagatzemar en una columna de taula. Comprendre els tipus de dades és crucial per dissenyar bases de dades eficients i escriure consultes efectives.

Conceptes clau

  • Tipus de dades: Una classificació que especifica quin tipus de valor pot contenir una columna.
  • Categories de tipus: PostgreSQL admet diverses categories de tipus de dades, incloent numèric, de caràcter, data/hora i més.
  • Restriccions de tipus: Regles que limiten el tipus de dades que es poden emmagatzemar en una columna.

Tipus numèrics

Els tipus de dades numèrics s'utilitzen per emmagatzemar números. PostgreSQL admet diversos tipus numèrics:

| Tipus de dades | Descripció | Mida d'emmagatzematge | |----------------|------------|-----------------------| | smallint | Enter de rang petit | 2 bytes | | integer o int | Elecció típica per a enters | 4 bytes | | bigint | Enter de rang gran | 8 bytes | | decimal o numeric | Precisió especificada per l'usuari | Variable | | real | Punt flotant de precisió simple | 4 bytes | | double precision | Punt flotant de doble precisió | 8 bytes | | serial | Enter auto-incremental | 4 bytes | | bigserial | Enter auto-incremental gran | 8 bytes |

Exemple

CREATE TABLE numbers_example (
    small_num smallint,
    medium_num integer,
    large_num bigint,
    precise_num numeric(10, 2),
    float_num real,
    double_num double precision
);

Explicació:

  • small_num pot emmagatzemar enters petits.
  • medium_num és adequat per a valors enters típics.
  • large_num pot emmagatzemar enters molt grans.
  • precise_num emmagatzema números amb fins a 10 dígits, 2 dels quals són després del punt decimal.
  • float_num i double_num emmagatzemen números de punt flotant amb precisió simple i doble, respectivament.

Tipus de caràcter

Els tipus de dades de caràcter s'utilitzen per emmagatzemar text. PostgreSQL proporciona diverses opcions:

| Tipus de dades | Descripció | Mida d'emmagatzematge | |----------------|------------|-----------------------| | char(n) | Tipus de caràcter de longitud fixa | n bytes | | varchar(n) | Tipus de caràcter de longitud variable | Fins a n bytes | | text | Tipus de caràcter de longitud variable sense límit | Variable |

Exemple

CREATE TABLE text_example (
    fixed_char char(10),
    variable_char varchar(50),
    unlimited_text text
);

Explicació:

  • fixed_char emmagatzema una cadena de longitud fixa de 10 caràcters.
  • variable_char pot emmagatzemar una cadena de fins a 50 caràcters.
  • unlimited_text pot emmagatzemar una cadena de qualsevol longitud.

Tipus de data/hora

Els tipus de dades de data i hora s'utilitzen per emmagatzemar dates, hores i marques de temps.

| Tipus de dades | Descripció | Mida d'emmagatzematge | |----------------|------------|-----------------------| | date | Data del calendari (any, mes, dia) | 4 bytes | | time | Hora del dia (sense fus horari) | 8 bytes | | timestamp | Data i hora (sense fus horari) | 8 bytes | | timestamptz | Data i hora amb fus horari | 8 bytes | | interval | Interval de temps | 16 bytes |

Exemple

CREATE TABLE datetime_example (
    event_date date,
    event_time time,
    event_timestamp timestamp,
    event_timestamptz timestamptz,
    duration interval
);

Explicació:

  • event_date emmagatzema una data.
  • event_time emmagatzema una hora del dia.
  • event_timestamp emmagatzema una data i hora.
  • event_timestamptz emmagatzema una data i hora amb informació de fus horari.
  • duration emmagatzema un interval de temps.

Tipus booleà

El tipus de dades booleà s'utilitza per emmagatzemar valors vertader/fals.

| Tipus de dades | Descripció | Mida d'emmagatzematge | |----------------|------------|-----------------------| | boolean | Booleà lògic (vertader/fals) | 1 byte |

Exemple

CREATE TABLE boolean_example (
    is_active boolean
);

Explicació:

  • is_active pot emmagatzemar un valor vertader o fals.

Tipus d'array

PostgreSQL permet definir columnes com a arrays de longitud variable.

Exemple

CREATE TABLE array_example (
    integer_array integer[],
    text_array text[]
);

Explicació:

  • integer_array pot emmagatzemar un array d'enters.
  • text_array pot emmagatzemar un array de cadenes de text.

Tipus JSON

PostgreSQL admet tipus de dades JSON per emmagatzemar dades JSON.

| Tipus de dades | Descripció | Mida d'emmagatzematge | |----------------|------------|-----------------------| | json | Dades JSON textuals | Variable | | jsonb | Dades JSON binàries, més eficients per a l'indexació | Variable |

Exemple

CREATE TABLE json_example (
    json_data json,
    jsonb_data jsonb
);

Explicació:

  • json_data emmagatzema dades JSON en format de text.
  • jsonb_data emmagatzema dades JSON en format binari, que és més eficient per a l'indexació i les consultes.

Conclusió

Comprendre els tipus de dades en PostgreSQL és essencial per dissenyar bases de dades eficients i efectives. Triant el tipus de dades adequat per a cada columna, pots assegurar la integritat de les dades, optimitzar l'emmagatzematge i millorar el rendiment de les consultes. Aquest tema ha cobert els tipus de dades més utilitzats, incloent numèrics, de caràcter, data/hora, booleans, arrays i tipus JSON. A mesura que progressis en el teu viatge amb PostgreSQL, trobaràs tipus de dades més especialitzats i aprendràs a utilitzar-los de manera efectiva.
