Assegurar les connexions a la teva base de dades PostgreSQL és crucial per protegir dades sensibles i assegurar que només els usuaris autoritzats puguin accedir a la base de dades. Aquesta secció et guiarà a través de diversos mètodes i millors pràctiques per assegurar les connexions de PostgreSQL, des de tècniques bàsiques fins a avançades.

Introducció a la Seguretat de Connexió

  • Per què assegurar les connexions?

    • Protegir dades sensibles d'accés no autoritzat.
    • Prevenir violacions de dades i possibles problemes legals.
    • Assegurar la integritat i confidencialitat de les dades.

  • Amenaces comunes:

    • Atacs de l'home al mig.
    • Accés no autoritzat.
    • Intercepció de dades.

Mesures de Seguretat Bàsiques

Utilitzant Autenticació per Contrasenya

PostgreSQL admet diversos mètodes d'autenticació, però l'autenticació per contrasenya és un dels mètodes més comuns i senzills.

  • Configuració de pg_hba.conf: El fitxer pg_hba.conf controla l'autenticació del client. Aquí tens com configurar-lo per a l'autenticació per contrasenya:

    # TYPE  DATABASE        USER            ADDRESS                 METHOD
host    all             all             0.0.0.0/0               md5
    • TYPE: Especifica el tipus de connexió (per exemple, host per a connexions TCP/IP).
    • DATABASE: Especifica el nom de la base de dades (all per a totes les bases de dades).
    • USER: Especifica el nom de l'usuari (all per a tots els usuaris).
    • ADDRESS: Especifica el rang d'adreces IP del client (0.0.0.0/0 per a totes les adreces).
    • METHOD: Especifica el mètode d'autenticació (md5 per a l'autenticació per contrasenya).

Enforçant Contrasenyes Fortes

  • Utilitzant l'extensió pgcrypto: L'extensió pgcrypto de PostgreSQL es pot utilitzar per enfortir les contrasenyes.

    CREATE EXTENSION pgcrypto;

    • Exemple de hash d'una contrasenya:

      SELECT crypt('my_secure_password', gen_salt('bf'));

Mesures de Seguretat Intermèdies

Activant Connexions SSL

Secure Sockets Layer (SSL) xifra les dades transmeses entre el client i el servidor.

  • Configuració de SSL en postgresql.conf:

    ssl = on
ssl_cert_file = 'server.crt'
ssl_key_file = 'server.key'

  • Generant certificats SSL:

    openssl req -new -text -out server.req
openssl rsa -in privkey.pem -out server.key
openssl req -x509 -in server.req -text -key server.key -out server.crt

  • Actualitzant pg_hba.conf per a SSL:

    hostssl all all 0.0.0.0/0 md5

Utilitzant Certificats de Client

Els certificats de client afegeixen una capa extra de seguretat requerint que els clients proporcionin un certificat per autenticar-se.

  • Generant certificats de client:

    openssl req -new -text -out client.req
openssl rsa -in privkey.pem -out client.key
openssl req -x509 -in client.req -text -key client.key -out client.crt

  • Configuració de pg_hba.conf per a certificats de client:

    hostssl all all 0.0.0.0/0 cert

Mesures de Seguretat Avançades

Utilitzant Autenticació LDAP

Lightweight Directory Access Protocol (LDAP) es pot utilitzar per a l'autenticació centralitzada.

  • Configuració de LDAP en pg_hba.conf:

    host all all 0.0.0.0/0 ldap ldapserver=ldap.example.com ldapbasedn="dc=example,dc=com"

Implementant Seguretat a Nivell de Fila (RLS)

La Seguretat a Nivell de Fila (RLS) et permet controlar l'accés a files específiques en una taula.

  • Activant RLS:

    ALTER TABLE employees ENABLE ROW LEVEL SECURITY;

  • Creant una política:

    CREATE POLICY employee_policy ON employees
USING (department = current_setting('myapp.current_department'));

  • Establint el departament actual:

    SET myapp.current_department = 'Sales';

Resum

Assegurar les connexions a la teva base de dades PostgreSQL implica múltiples capes de mesures de seguretat, des de l'autenticació bàsica per contrasenya fins a tècniques avançades com SSL, certificats de client, autenticació LDAP i Seguretat a Nivell de Fila. Implementant aquestes pràctiques, pots millorar significativament la seguretat de la teva base de dades PostgreSQL i protegir les teves dades d'accés no autoritzat i possibles amenaces.
