Crear taules és un aspecte fonamental de treballar amb PostgreSQL. Aquest tema et guiarà a través del procés des d'una perspectiva de principiant fins a tècniques més avançades.

Introducció a les Taules

Les taules són l'estructura principal utilitzada per emmagatzemar dades en una base de dades relacional. Cada taula consisteix en files i columnes, on cada columna representa un atribut específic de les dades, i cada fila representa un registre.

Creació Bàsica de Taules

Per crear una taula a PostgreSQL, utilitzes la instrucció CREATE TABLE. Aquí tens un exemple senzill:

CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    email VARCHAR(100),
    hire_date DATE
);

Explicació:

  • CREATE TABLE employees: Aquesta comanda crea una nova taula anomenada employees.
  • id SERIAL PRIMARY KEY: Crea una columna id que s'auto-incrementa amb cada nou registre i serveix com a clau primària.
  • first_name VARCHAR(50): Crea una columna first_name que pot emmagatzemar fins a 50 caràcters.
  • last_name VARCHAR(50): Crea una columna last_name que pot emmagatzemar fins a 50 caràcters.
  • email VARCHAR(100): Crea una columna email que pot emmagatzemar fins a 100 caràcters.
  • hire_date DATE: Crea una columna hire_date per emmagatzemar valors de data.

Tipus de Dades

Entendre els tipus de dades és crucial quan es creen taules. Aquí tens alguns tipus de dades comuns utilitzats a PostgreSQL:

| Tipus de Dada | Descripció | |---------------|-------------| | INTEGER | Nombres enters | | SERIAL | Enter auto-incremental | | VARCHAR(n) | Cadena de longitud variable amb un límit de n caràcters | | TEXT | Cadena de longitud variable sense límit | | DATE | Data del calendari (any, mes, dia) | | BOOLEAN | Valors vertader o fals |

Restriccions

Les restriccions són regles aplicades a les columnes per fer complir la integritat de les dades. Aquí tens algunes restriccions comunes:

  • PRIMARY KEY: Identifica de manera única cada registre.
  • NOT NULL: Assegura que una columna no pot tenir un valor NULL.
  • UNIQUE: Assegura que tots els valors en una columna siguin únics.
  • CHECK: Assegura que tots els valors en una columna compleixin una condició específica.
  • FOREIGN KEY: Assegura la integritat referencial entre dues taules.

Exemple amb Restriccions:

CREATE TABLE departments (
    department_id SERIAL PRIMARY KEY,
    department_name VARCHAR(100) NOT NULL UNIQUE,
    manager_id INTEGER,
    CHECK (department_name <> '')
);

Explicació:

  • department_id SERIAL PRIMARY KEY: Clau primària auto-incremental.
  • department_name VARCHAR(100) NOT NULL UNIQUE: El nom del departament ha de ser únic i no pot ser NULL.
  • manager_id INTEGER: ID del gerent, que pot estar enllaçat a una altra taula.
  • CHECK (department_name <> ''): Assegura que el nom del departament no sigui una cadena buida.

Creació Avançada de Taules

Ús de Claus Foranes

Les claus foranes s'utilitzen per enllaçar dues taules. Aquí tens un exemple:

CREATE TABLE projects (
    project_id SERIAL PRIMARY KEY,
    project_name VARCHAR(100) NOT NULL,
    start_date DATE,
    end_date DATE,
    department_id INTEGER,
    FOREIGN KEY (department_id) REFERENCES departments(department_id)
);

Explicació:

  • department_id INTEGER: Aquesta columna emmagatzemarà l'ID del departament.
  • FOREIGN KEY (department_id) REFERENCES departments(department_id): Això fa complir una relació entre la taula projects i la taula departments.

Claus Primàries Compostes

De vegades, pots necessitar una clau primària que consisteixi en múltiples columnes:

CREATE TABLE project_assignments (
    project_id INTEGER,
    employee_id INTEGER,
    assignment_date DATE,
    PRIMARY KEY (project_id, employee_id)
);

Explicació:

  • PRIMARY KEY (project_id, employee_id): Això crea una clau primària composta utilitzant tant project_id com employee_id.

Conclusió

Crear taules a PostgreSQL implica entendre l'estructura de les teves dades i aplicar els tipus de dades i restriccions adequades per assegurar la integritat de les dades. Des de la creació bàsica de taules fins a tècniques avançades com claus foranes i claus primàries compostes, dominar aquests conceptes és essencial per a una gestió efectiva de bases de dades.

Practicar aquests exemples i entendre els principis subjacents et permetrà dissenyar i implementar esquemes de bases de dades robustos a PostgreSQL.
