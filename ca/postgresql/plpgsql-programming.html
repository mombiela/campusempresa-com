Introducció a PL/pgSQL

PL/pgSQL (Procedural Language/PostgreSQL) és un llenguatge procedimental suportat per PostgreSQL. Permet escriure funcions i triggers complexos en PostgreSQL, millorant les capacitats de SQL.

  • Conceptes Clau:
    • Llenguatge Procedimental: Extén SQL amb construccions procedimentals.
    • Funcions: Blocs de codi reutilitzables que poden prendre paràmetres i retornar un valor.
    • Triggers: Tipus especial de procediment emmagatzemat que s'executa automàticament quan es produeixen certs esdeveniments.

Configuració de PL/pgSQL

Abans de començar a escriure codi PL/pgSQL, assegura't que el llenguatge està habilitat a la teva base de dades PostgreSQL.

CREATE EXTENSION IF NOT EXISTS plpgsql;

Escriure Funcions PL/pgSQL

Les funcions en PL/pgSQL es defineixen utilitzant la sentència CREATE FUNCTION. Poden prendre paràmetres, executar comandes SQL i retornar resultats.

Exemple Bàsic de Funció

Aquí tens una funció simple que pren dos enters i retorna la seva suma:

CREATE OR REPLACE FUNCTION add_numbers(a INTEGER, b INTEGER)
RETURNS INTEGER AS $$
BEGIN
    RETURN a + b;
END;
$$ LANGUAGE plpgsql;
  • Explicació:
    • CREATE OR REPLACE FUNCTION: Defineix una nova funció o reemplaça una existent.
    • RETURNS INTEGER: Especifica el tipus de retorn de la funció.
    • $$ ... $$: Delimitadors per al cos de la funció.
    • BEGIN ... END;: Bloc on s'escriu la lògica de la funció.
    • RETURN a + b;: Retorna la suma dels dos paràmetres d'entrada.

Cridar una Funció

Pots cridar la funció utilitzant una sentència SELECT:

SELECT add_numbers(5, 3);

Estructures de Control

PL/pgSQL suporta diverses estructures de control com ara sentències condicionals i bucles.

Sentències Condicionals

Utilitza IF, ELSIF i ELSE per executar codi de manera condicional.

CREATE OR REPLACE FUNCTION check_number(n INTEGER)
RETURNS TEXT AS $$
BEGIN
    IF n > 0 THEN
        RETURN 'Positive';
    ELSIF n < 0 THEN
        RETURN 'Negative';
    ELSE
        RETURN 'Zero';
    END IF;
END;
$$ LANGUAGE plpgsql;

Bucles

PL/pgSQL proporciona diversos tipus de bucles: LOOP, WHILE, FOR.

Exemple: Bucle FOR

CREATE OR REPLACE FUNCTION sum_series(n INTEGER)
RETURNS INTEGER AS $$
DECLARE
    total INTEGER := 0;
BEGIN
    FOR i IN 1..n LOOP
        total := total + i;
    END LOOP;
    RETURN total;
END;
$$ LANGUAGE plpgsql;
  • Explicació:
    • DECLARE: Secció per declarar variables.
    • total INTEGER := 0;: Inicialitza la variable total.
    • FOR i IN 1..n LOOP: Bucla de l'1 a n.
    • total := total + i;: Afegeix el valor actual de i a total.

Gestió d'Errors

PL/pgSQL proporciona mecanismes per gestionar errors utilitzant blocs EXCEPTION.

Exemple: Gestió d'Errors

CREATE OR REPLACE FUNCTION divide_numbers(a INTEGER, b INTEGER)
RETURNS FLOAT AS $$
BEGIN
    RETURN a / b;
EXCEPTION
    WHEN division_by_zero THEN
        RETURN NULL;
END;
$$ LANGUAGE plpgsql;
  • Explicació:
    • EXCEPTION: Bloc per capturar excepcions.
    • WHEN division_by_zero THEN: Captura errors de divisió per zero.
    • RETURN NULL;: Retorna NULL si es produeix un error.

Triggers

Els triggers són procediments especials que s'executen automàticament en resposta a certs esdeveniments en una taula.

Exemple: Trigger per Registrar Canvis

CREATE OR REPLACE FUNCTION log_changes()
RETURNS TRIGGER AS $$
BEGIN
    INSERT INTO changes_log(table_name, operation, changed_at)
    VALUES (TG_TABLE_NAME, TG_OP, NOW());
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER after_update
AFTER UPDATE ON your_table
FOR EACH ROW
EXECUTE FUNCTION log_changes();
  • Explicació:
    • TG_TABLE_NAME: Variable especial que conté el nom de la taula.
    • TG_OP: Variable especial que conté el tipus d'operació (INSERT, UPDATE, DELETE).
    • AFTER UPDATE ON your_table: Especifica que el trigger s'executa després d'una actualització en your_table.

Conclusió

PL/pgSQL extén les capacitats de PostgreSQL permetent escriure codi procedimental. Això inclou definir funcions, utilitzar estructures de control, gestionar errors i crear triggers. Dominar PL/pgSQL pot millorar significativament la teva capacitat per gestionar i manipular dades dins de PostgreSQL.

Entenent i practicant els conceptes coberts en aquest tema, estaràs ben preparat per escriure codi PL/pgSQL eficient i robust per a diverses aplicacions.
