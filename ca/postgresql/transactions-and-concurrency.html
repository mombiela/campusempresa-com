Introducció a les Transaccions

Les transaccions són un concepte fonamental en les bases de dades, assegurant que una sèrie d'operacions es completin amb èxit conjuntament o fallin conjuntament, mantenint la integritat de les dades.

  • Atomicitat: Assegura que totes les operacions dins d'una transacció es completin; si no, la transacció s'aborta.
  • Consistència: Assegura que una transacció porta la base de dades d'un estat vàlid a un altre.
  • Aïllament: Assegura que les transaccions es processen de manera segura i independent al mateix temps sense interferències.
  • Durabilitat: Assegura que una vegada una transacció ha estat compromesa, romandrà així, fins i tot en cas de fallada del sistema.

Comandes Bàsiques de Transacció

  • BEGIN: Comença una nova transacció.
  • COMMIT: Desa tots els canvis fets durant la transacció.
  • ROLLBACK: Reverteix tots els canvis fets durant la transacció.

Exemple

BEGIN;

INSERT INTO accounts (user_id, balance) VALUES (1, 1000);
UPDATE accounts SET balance = balance - 100 WHERE user_id = 1;

COMMIT;

En aquest exemple:

  1. Es comença una transacció amb BEGIN.
  2. S'insereix un nou compte.
  3. Es modifica el saldo del compte.
  4. La transacció es compromet, fent tots els canvis permanents.

Control de Concurrència

El control de concurrència és essencial per assegurar que les transaccions de la base de dades es realitzin simultàniament sense violar la integritat de les dades.

Nivells d'Aïllament

PostgreSQL suporta diversos nivells d'aïllament que controlen la visibilitat dels canvis fets per transaccions concurrents.

  • Read Uncommitted: Les transaccions poden veure canvis no compromesos fets per altres transaccions.
  • Read Committed: Una transacció només veu canvis compromesos fets per altres transaccions.
  • Repeatable Read: Assegura que si una transacció llegeix un valor, veurà el mateix valor si el torna a llegir.
  • Serializable: Assegura un aïllament complet d'altres transaccions, proporcionant el nivell més alt d'aïllament.

Exemple

SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;

BEGIN;

SELECT balance FROM accounts WHERE user_id = 1;

-- Realitzar algunes operacions

COMMIT;

En aquest exemple:

  1. El nivell d'aïllament de la transacció es configura a REPEATABLE READ.
  2. Es comença una transacció.
  3. Es llegeix el saldo de la taula accounts.
  4. La transacció es compromet.

Mecanismes de Bloqueig

PostgreSQL utilitza bloquejos per gestionar l'accés concurrent a les dades.

  • Bloquejos a nivell de fila: Bloquegen files individuals.
  • Bloquejos a nivell de taula: Bloquegen taules senceres.

Exemple

BEGIN;

SELECT * FROM accounts WHERE user_id = 1 FOR UPDATE;

-- Realitzar algunes operacions

COMMIT;

En aquest exemple:

  1. Es comença una transacció.
  2. S'adquireix un bloqueig a nivell de fila en la fila seleccionada utilitzant FOR UPDATE.
  3. Es realitzen operacions.
  4. La transacció es compromet.

Interbloquejos

Els interbloquejos es produeixen quan dues o més transaccions estan esperant que l'altra alliberi bloquejos, causant un cicle de dependències.

Exemple

-- Transacció 1
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE user_id = 1;

-- Transacció 2
BEGIN;
UPDATE accounts SET balance = balance + 100 WHERE user_id = 2;

-- Transacció 1
UPDATE accounts SET balance = balance + 100 WHERE user_id = 2; -- espera per Transacció 2

-- Transacció 2
UPDATE accounts SET balance = balance - 100 WHERE user_id = 1; -- espera per Transacció 1

En aquest exemple:

  1. La Transacció 1 bloqueja una fila.
  2. La Transacció 2 bloqueja una altra fila.
  3. Ambdues transaccions esperen que l'altra alliberi els seus bloquejos, causant un interbloqueig.

Conclusió

Entendre les transaccions i el control de concurrència és crucial per mantenir la integritat de les dades i el rendiment en PostgreSQL. Dominant les comandes de transacció, els nivells d'aïllament, els mecanismes de bloqueig i la gestió d'interbloquejos, pots assegurar que les teves operacions de base de dades siguin fiables i eficients.
