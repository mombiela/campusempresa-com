Introducció a les Expressions de Taula Comunes (CTEs)

Les Expressions de Taula Comunes (CTEs) són una característica poderosa en PostgreSQL que et permeten crear conjunts de resultats temporals que poden ser referenciats dins d'una instrucció SELECT, INSERT, UPDATE o DELETE. Les CTEs poden simplificar consultes complexes, millorar la llegibilitat i fer possibles les consultes recursives.

Conceptes Clau

  • Definició de CTE: Una CTE es defineix utilitzant la paraula clau WITH seguida d'una consulta que genera un conjunt de resultats temporals.
  • Àmbit Temporal: El conjunt de resultats d'una CTE només està disponible dins de l'àmbit d'execució de la instrucció en què es defineix.
  • CTEs Recursives: Aquestes et permeten realitzar consultes recursives, que són útils per a estructures de dades jeràrquiques com organigrames o arbres genealògics.

Sintaxi Bàsica de les CTEs

Per definir una CTE, utilitzes la clàusula WITH seguida del nom de la CTE i la consulta que genera el conjunt de resultats.

WITH cte_name AS (
    SELECT column1, column2
    FROM table_name
    WHERE condition
)
SELECT *
FROM cte_name;

Exemple: CTE Simple

Comencem amb un exemple senzill. Suposem que tenim una taula employees amb les columnes id, name i department.

WITH dept_sales AS (
    SELECT id, name
    FROM employees
    WHERE department = 'Sales'
)
SELECT *
FROM dept_sales;

En aquest exemple:

  • Definim una CTE anomenada dept_sales.
  • La CTE selecciona id i name de la taula employees on el department és 'Sales'.
  • La consulta principal selecciona totes les columnes de la CTE dept_sales.

Utilitzant Múltiples CTEs

Pots definir múltiples CTEs en una sola consulta separant-les amb comes.

WITH dept_sales AS (
    SELECT id, name
    FROM employees
    WHERE department = 'Sales'
),
dept_marketing AS (
    SELECT id, name
    FROM employees
    WHERE department = 'Marketing'
)
SELECT *
FROM dept_sales
UNION ALL
SELECT *
FROM dept_marketing;

Exemple: Combinant Resultats de Múltiples CTEs

En aquest exemple:

  • Definim dues CTEs: dept_sales i dept_marketing.
  • La consulta principal combina els resultats de les dues CTEs utilitzant l'operador UNION ALL.

CTEs Recursives

Les CTEs recursives són útils per a consultar dades jeràrquiques. Consten de dues parts: un membre ancla i un membre recursiu.

Sintaxi per a CTEs Recursives

WITH RECURSIVE cte_name AS (
    -- Membre ancla
    SELECT column1, column2
    FROM table_name
    WHERE condition
    
    UNION ALL
    
    -- Membre recursiu
    SELECT column1, column2
    FROM table_name
    JOIN cte_name ON table_name.column = cte_name.column
)
SELECT *
FROM cte_name;

Exemple: CTE Recursiva

Suposem que tenim una taula employees amb les columnes id, name i manager_id per representar una jerarquia organitzativa.

WITH RECURSIVE employee_hierarchy AS (
    -- Membre ancla: Selecciona el gerent de nivell superior
    SELECT id, name, manager_id
    FROM employees
    WHERE manager_id IS NULL
    
    UNION ALL
    
    -- Membre recursiu: Selecciona empleats que reporten al nivell actual
    SELECT e.id, e.name, e.manager_id
    FROM employees e
    JOIN employee_hierarchy eh ON e.manager_id = eh.id
)
SELECT *
FROM employee_hierarchy;

En aquest exemple:

  • El membre ancla selecciona el gerent de nivell superior (on manager_id és NULL).
  • El membre recursiu selecciona empleats que reporten al nivell actual de la jerarquia.
  • La consulta principal selecciona totes les columnes de la CTE employee_hierarchy.

Conclusió

Les Expressions de Taula Comunes (CTEs) en PostgreSQL proporcionen una manera poderosa de simplificar consultes complexes i gestionar dades jeràrquiques. Utilitzant la clàusula WITH, pots crear conjunts de resultats temporals que milloren la llegibilitat i mantenibilitat de les consultes. Les CTEs recursives amplien aquesta funcionalitat per manejar estructures de dades recursives de manera efectiva.

Resum

  • CTEs: Conjunts de resultats temporals definits utilitzant la clàusula WITH.
  • Múltiples CTEs: Es poden definir en una sola consulta i combinar utilitzant operadors com UNION ALL.
  • CTEs Recursives: Permeten consultar dades jeràrquiques a través de consultes recursives.

Practica utilitzant CTEs en les teves consultes per veure com poden simplificar el teu codi SQL i fer-lo més llegible.
