Introducció

Consultar dades és una de les operacions fonamentals en qualsevol sistema de gestió de bases de dades. En PostgreSQL, es fa servir el llenguatge SQL per recuperar, manipular i gestionar dades. Aquesta secció cobrirà els conceptes bàsics de la consulta de dades, des d'instruccions SELECT senzilles fins a tècniques de consulta més avançades.

Instruccions SELECT Bàsiques

La instrucció SELECT s'utilitza per consultar dades d'una base de dades. Les dades retornades es guarden en una taula de resultats, de vegades anomenada conjunt de resultats.

Sintaxi

SELECT column1, column2, ...
FROM table_name;

Exemple

SELECT first_name, last_name
FROM employees;

Aquesta consulta recupera les columnes first_name i last_name de la taula employees.

Filtrar Dades amb WHERE

La clàusula WHERE s'utilitza per filtrar registres que compleixen una certa condició.

Sintaxi

SELECT column1, column2, ...
FROM table_name
WHERE condition;

Exemple

SELECT first_name, last_name
FROM employees
WHERE department = 'Sales';

Aquesta consulta recupera els first_name i last_name dels empleats que treballen al departament de Vendes.

Ordenar Dades amb ORDER BY

La clàusula ORDER BY s'utilitza per ordenar el conjunt de resultats en ordre ascendent o descendent.

Sintaxi

SELECT column1, column2, ...
FROM table_name
ORDER BY column1 [ASC|DESC], column2 [ASC|DESC], ...;

Exemple

SELECT first_name, last_name
FROM employees
ORDER BY last_name ASC;

Aquesta consulta recupera els first_name i last_name dels empleats i els ordena per last_name en ordre ascendent.

Limitar Resultats amb LIMIT

La clàusula LIMIT s'utilitza per especificar el nombre de registres a retornar.

Sintaxi

SELECT column1, column2, ...
FROM table_name
LIMIT number;

Exemple

SELECT first_name, last_name
FROM employees
ORDER BY last_name ASC
LIMIT 10;

Aquesta consulta recupera els primers 10 empleats ordenats per last_name en ordre ascendent.

Combinar Condicions amb AND, OR, i NOT

Podeu combinar múltiples condicions en una clàusula WHERE utilitzant AND, OR i NOT.

Exemple

SELECT first_name, last_name
FROM employees
WHERE department = 'Sales' AND salary > 50000;

Aquesta consulta recupera els first_name i last_name dels empleats que treballen al departament de Vendes i tenen un salari superior a 50.000.

Utilitzar Funcions Agregades

Les funcions agregades realitzen un càlcul sobre un conjunt de valors i retornen un únic valor. Les funcions agregades comunes inclouen COUNT, SUM, AVG, MAX i MIN.

Exemple

SELECT COUNT(*)
FROM employees
WHERE department = 'Sales';

Aquesta consulta retorna el nombre d'empleats al departament de Vendes.

Agrupar Dades amb GROUP BY

La instrucció GROUP BY agrupa files que tenen els mateixos valors en files de resum.

Sintaxi

SELECT column1, aggregate_function(column2)
FROM table_name
GROUP BY column1;

Exemple

SELECT department, COUNT(*)
FROM employees
GROUP BY department;

Aquesta consulta retorna el nombre d'empleats en cada departament.

Filtrar Grups amb HAVING

La clàusula HAVING s'utilitza per filtrar grups basats en una condició.

Exemple

SELECT department, COUNT(*)
FROM employees
GROUP BY department
HAVING COUNT(*) > 10;

Aquesta consulta retorna departaments amb més de 10 empleats.

Unir Taules

Les unions s'utilitzen per combinar files de dues o més taules basades en una columna relacionada.

Tipus d'Unions

  • INNER JOIN: Retorna registres que tenen valors coincidents en ambdues taules.
  • LEFT JOIN (o LEFT OUTER JOIN): Retorna tots els registres de la taula esquerra, i els registres coincidents de la taula dreta.
  • RIGHT JOIN (o RIGHT OUTER JOIN): Retorna tots els registres de la taula dreta, i els registres coincidents de la taula esquerra.
  • FULL JOIN (o FULL OUTER JOIN): Retorna tots els registres quan hi ha una coincidència en qualsevol de les taules esquerra o dreta.

Exemple

SELECT employees.first_name, employees.last_name, departments.department_name
FROM employees
INNER JOIN departments ON employees.department_id = departments.department_id;

Aquesta consulta recupera els first_name i last_name dels empleats juntament amb el seu department_name.

Subconsultes

Una subconsulta és una consulta inserida dins d'una altra consulta.

Exemple

SELECT first_name, last_name
FROM employees
WHERE department_id = (SELECT department_id FROM departments WHERE department_name = 'Sales');

Aquesta consulta recupera els first_name i last_name dels empleats que treballen al departament de Vendes.

Conclusió

Consultar dades en PostgreSQL implica utilitzar diverses clàusules i funcions SQL per recuperar i manipular dades de manera efectiva. Des d'instruccions SELECT bàsiques fins a unions i subconsultes complexes, dominar aquestes tècniques és essencial per a una gestió eficient de bases de dades i anàlisi de dades. Practiqueu aquestes consultes per esdevenir proficient en extreure informació significativa de les vostres bases de dades PostgreSQL.
