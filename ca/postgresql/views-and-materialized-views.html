Introducció

A PostgreSQL, les vistes i les vistes materialitzades són eines poderoses que et permeten simplificar consultes complexes, millorar la seguretat i augmentar el rendiment. Aquesta secció cobrirà els conceptes bàsics de les vistes i les vistes materialitzades, les seves diferències i com utilitzar-les de manera efectiva.

Vistes

Què és una Vista?

  • Una vista és una taula virtual basada en el conjunt de resultats d'una consulta SQL.
  • No emmagatzema dades físicament; en canvi, proporciona una manera de veure dades d'una o més taules.
  • Les vistes poden simplificar consultes complexes encapsulant-les en un sol objecte.

Crear una Vista

Per crear una vista, utilitza la instrucció CREATE VIEW seguida del nom de la vista i la consulta que defineix la vista.

CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

Exemple

CREATE VIEW employee_view AS
SELECT id, first_name, last_name, department
FROM employees
WHERE active = true;

Aquesta vista, employee_view, mostra només els empleats actius.

Utilitzar una Vista

Pots consultar una vista igual que una taula regular.

SELECT * FROM employee_view;

Actualitzar Dades a través d'una Vista

  • Les vistes poden ser actualitzables si compleixen certs criteris (per exemple, referencien una sola taula base).
  • Utilitza les instruccions INSERT, UPDATE o DELETE sobre les vistes per modificar la taula subjacent.

Exemple

UPDATE employee_view
SET department = 'HR'
WHERE id = 1;

Avantatges de les Vistes

  • Simplifiquen consultes complexes.
  • Milloren la seguretat restringint l'accés a dades específiques.
  • Proporcionen un nivell d'abstracció.

Limitacions de les Vistes

  • El rendiment pot ser un problema per a vistes complexes.
  • No totes les vistes són actualitzables.

Vistes Materialitzades

Què és una Vista Materialitzada?

  • Una vista materialitzada emmagatzema el resultat d'una consulta físicament.
  • Es pot actualitzar per reflectir les dades.
  • Útil per millorar el rendiment en consultes complexes.

Crear una Vista Materialitzada

Per crear una vista materialitzada, utilitza la instrucció CREATE MATERIALIZED VIEW.

CREATE MATERIALIZED VIEW mat_view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

Exemple

CREATE MATERIALIZED VIEW active_employees AS
SELECT id, first_name, last_name, department
FROM employees
WHERE active = true;

Aquesta vista materialitzada, active_employees, emmagatzema les dades dels empleats actius.

Actualitzar una Vista Materialitzada

  • Les vistes materialitzades necessiten ser actualitzades per reflectir els canvis en les taules subjacents.
  • Utilitza la instrucció REFRESH MATERIALIZED VIEW.
REFRESH MATERIALIZED VIEW active_employees;

Utilitzar una Vista Materialitzada

Consulta una vista materialitzada igual que una taula regular.

SELECT * FROM active_employees;

Avantatges de les Vistes Materialitzades

  • Millora del rendiment per a consultes complexes.
  • Les dades s'emmagatzemen físicament, reduint la necessitat de càlculs repetits.

Limitacions de les Vistes Materialitzades

  • Requereix espai d'emmagatzematge.
  • Necessita ser actualitzada per mantenir-se al dia.

Comparació de Vistes i Vistes Materialitzades

| Característica | Vista | Vista Materialitzada | |-------------------------|--------------------------------|---------------------------------| | Emmagatzematge | Virtual, sense emmagatzematge físic | Emmagatzematge físic de dades | | Rendiment | Pot ser més lent per a consultes complexes | Més ràpid per a consultes complexes | | Actualització de Dades | Sempre al dia | Necessita ser actualitzada | | Actualitzabilitat | Pot ser actualitzable | No actualitzable directament | | Cas d'Ús | Simplificar consultes, millorar la seguretat | Millorar el rendiment en consultes complexes |

Conclusió

Les vistes i les vistes materialitzades són eines essencials a PostgreSQL per gestionar i optimitzar l'accés a les dades. Les vistes proporcionen una manera de simplificar consultes complexes i millorar la seguretat, mentre que les vistes materialitzades ofereixen beneficis de rendiment per a consultes complexes a costa d'emmagatzematge i la necessitat d'actualització. Entendre quan i com utilitzar cadascuna pot millorar significativament l'eficiència i mantenibilitat de les teves aplicacions de base de dades.
