Introducció a COBOL

COBOL (Common Business-Oriented Language) és un llenguatge de programació d'alt nivell dissenyat per a aplicacions empresarials. Ha estat àmpliament utilitzat en diverses indústries, especialment en institucions financeres i agències governamentals.

  • Història de COBOL: Desenvolupat el 1959 per CODASYL (Conference on Data Systems Languages).
  • Característiques Clau:
    • Sintaxi semblant a l'anglès
    • Fortes capacitats de processament de dades
    • Portabilitat entre diferents sistemes

Configuració del Teu Entorn COBOL

Abans d'endinsar-te en la programació COBOL, necessites configurar el teu entorn de desenvolupament.

  • Compiladors COBOL:

    • GnuCOBOL (codi obert)
    • Micro Focus COBOL
    • IBM COBOL per a z/OS

  • Instal·lació:

    • Segueix les instruccions específiques per al compilador escollit.
    • Assegura't de tenir accés a un mainframe o a un emulador de mainframe si és necessari.

Sintaxi i Estructura Bàsica de COBOL

Els programes COBOL es divideixen en quatre divisions principals:

  1. Identification Division: Conté metadades sobre el programa.
  2. Environment Division: Especifica l'entorn en el qual s'executarà el programa.
  3. Data Division: Defineix les variables i estructures de dades utilitzades.
  4. Procedure Division: Conté el codi executable.

Exemple de Programa

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.

ENVIRONMENT DIVISION.

DATA DIVISION.

PROCEDURE DIVISION.
    DISPLAY 'Hello, World!'.
    STOP RUN.
  • Explicació:
    • IDENTIFICATION DIVISION: Declara el nom del programa.
    • PROCEDURE DIVISION: Conté la instrucció DISPLAY per imprimir "Hello, World!" i STOP RUN per acabar el programa.

Tipus de Dades i Variables

COBOL suporta diversos tipus de dades, principalment enfocats en dades numèriques i alfanumèriques.

  • Tipus de Dades Numèriques:

    • PIC 9(n): Dades numèriques amb n dígits.
    • PIC S9(n): Dades numèriques amb signe.

  • Tipus de Dades Alfanumèriques:

    • PIC X(n): Dades alfanumèriques amb n caràcters.

Exemple

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM-VAR PIC 9(5).
01 ALPHA-VAR PIC X(10).

PROCEDURE DIVISION.
    MOVE 12345 TO NUM-VAR.
    MOVE 'Hello' TO ALPHA-VAR.
    DISPLAY NUM-VAR.
    DISPLAY ALPHA-VAR.
    STOP RUN.
  • Explicació:
    • NUM-VAR és una variable numèrica amb 5 dígits.
    • ALPHA-VAR és una variable alfanumèrica amb 10 caràcters.
    • Les instruccions MOVE assignen valors a les variables.
    • Les instruccions DISPLAY imprimeixen els valors.

Estructures de Control

COBOL proporciona diverses estructures de control per al control de flux, incloent-hi instruccions condicionals i bucles.

  • Instruccions IF:

    IF condition
    THEN
        statements
    ELSE
        statements
END-IF.

  • Bucles PERFORM:

    PERFORM UNTIL condition
    statements
END-PERFORM.

Exemple

DATA DIVISION.
WORKING-STORAGE SECTION.
01 COUNTER PIC 9(2) VALUE 0.

PROCEDURE DIVISION.
    PERFORM UNTIL COUNTER > 10
        ADD 1 TO COUNTER
        DISPLAY 'Counter: ' COUNTER
    END-PERFORM.
    STOP RUN.
  • Explicació:
    • COUNTER és una variable numèrica inicialitzada a 0.
    • El bucle PERFORM UNTIL incrementa COUNTER fins que supera 10.
    • DISPLAY imprimeix el valor de COUNTER.

Gestió d'Arxius

COBOL és conegut per les seves robustes capacitats de gestió d'arxius, essencials per a aplicacions empresarials.

  • Control d'Arxius:

    • Instrucció SELECT per associar un arxiu amb un descriptor d'arxiu.
    • Clàusula ASSIGN per especificar l'arxiu físic.

  • Operacions d'Arxius:

    • OPEN, READ, WRITE, CLOSE.

Exemple

IDENTIFICATION DIVISION.
PROGRAM-ID. FileExample.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT IN-FILE ASSIGN TO 'input.txt'
    ORGANIZATION IS LINE SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD IN-FILE.
01 IN-RECORD PIC X(100).

WORKING-STORAGE SECTION.

PROCEDURE DIVISION.
    OPEN INPUT IN-FILE.
    READ IN-FILE INTO IN-RECORD
        AT END
            DISPLAY 'End of file reached.'
    END-READ.
    DISPLAY IN-RECORD.
    CLOSE IN-FILE.
    STOP RUN.
  • Explicació:
    • La secció FILE-CONTROL associa IN-FILE amb input.txt.
    • FD (File Description) defineix l'estructura de l'arxiu.
    • Les operacions OPEN, READ i CLOSE gestionen l'accés a l'arxiu.

Temes Avançats

A mesura que avancis, trobaràs conceptes i característiques més complexes en COBOL.

  • Subprogrames: Modularitza el codi utilitzant les instruccions CALL i PERFORM.
  • Gestió d'Errors: Utilitza les clàusules INVALID KEY i AT END.
  • Accés a Bases de Dades: Integra amb bases de dades utilitzant SQL incrustat (EXEC SQL).

Exemple d'un Subprograma

IDENTIFICATION DIVISION.
PROGRAM-ID. MainProgram.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 RESULT PIC 9(2).

PROCEDURE DIVISION.
    CALL 'SubProgram' USING RESULT.
    DISPLAY 'Result from SubProgram: ' RESULT.
    STOP RUN.

IDENTIFICATION DIVISION.
PROGRAM-ID. SubProgram.

DATA DIVISION.
LINKAGE SECTION.
01 RESULT PIC 9(2).

PROCEDURE DIVISION USING RESULT.
    MOVE 42 TO RESULT.
    EXIT PROGRAM.
  • Explicació:
    • MainProgram crida SubProgram i passa RESULT.
    • SubProgram estableix RESULT a 42 i retorna el control a MainProgram.

Conclusió

COBOL continua sent un llenguatge vital en el món empresarial, especialment per a sistemes heretats en mainframes. Entendre la seva sintaxi, estructures de control i capacitats de gestió d'arxius és crucial per mantenir i desenvolupar aplicacions empresarials. A mesura que progressis, dominar temes avançats com subprogrames i accés a bases de dades et permetrà gestionar tasques més complexes de manera eficient.
