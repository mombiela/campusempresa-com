◄ Estructura del Programa COBOL
Seccions i Paràgrafs del COBOL ►

COBOL (Common Business-Oriented Language) està estructurat en quatre divisions principals. Cada divisió té un propòsit específic i ajuda a organitzar el codi d'una manera lògica i llegible. Entendre aquestes divisions és crucial per escriure i mantenir programes COBOL.

1. Identification Division

La Identification Division és la primera divisió en un programa COBOL. Proporciona metadades sobre el programa, com ara el nom del programa i els detalls de l'autor.

Conceptes Clau:

  • PROGRAM-ID: Especifica el nom del programa.
  • AUTHOR: Proporciona el nom del programador.
  • INSTALLATION: Descriu el lloc d'instal·lació.
  • DATE-WRITTEN: Indica quan es va escriure el programa.
  • DATE-COMPILED: Mostra quan es va compilar el programa.

Exemple:

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.
AUTHOR. John Doe.
INSTALLATION. XYZ Corporation.
DATE-WRITTEN. 2023-10-01.
DATE-COMPILED. 2023-10-02.

2. Environment Division

La Environment Division especifica l'entorn informàtic en què s'executarà el programa. Inclou informació sobre la configuració i els fitxers utilitzats pel programa.

Conceptes Clau:

  • CONFIGURATION SECTION: Descriu la configuració del sistema.
  • INPUT-OUTPUT SECTION: Defineix els fitxers i dispositius utilitzats pel programa.

Exemple:

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
SOURCE-COMPUTER. IBM-370.
OBJECT-COMPUTER. IBM-370.

INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'.
    SELECT OutputFile ASSIGN TO 'output.dat'.

3. Data Division

La Data Division és on es defineixen totes les variables i estructures de dades utilitzades en el programa. Es divideix en diverses seccions, però les més utilitzades són la Working-Storage Section i la File Section.

Conceptes Clau:

  • FILE SECTION: Defineix l'estructura dels fitxers.
  • WORKING-STORAGE SECTION: Declara variables i constants.
  • LINKAGE SECTION: S'utilitza per a paràmetres passats entre programes.

Exemple:

DATA DIVISION.
FILE SECTION.
FD  InputFile.
01  InputRecord.
    05  InputField1  PIC X(10).
    05  InputField2  PIC 9(5).

FD  OutputFile.
01  OutputRecord.
    05  OutputField1  PIC X(10).
    05  OutputField2  PIC 9(5).

WORKING-STORAGE SECTION.
01  WS-Variable1  PIC X(20) VALUE 'Hello, COBOL!'.
01  WS-Variable2  PIC 9(4) VALUE 2023.

4. Procedure Division

La Procedure Division conté les instruccions executables del programa. Aquí és on es implementa la lògica del programa a través de diverses instruccions COBOL.

Conceptes Clau:

  • Paragraphs: Blocs lògics de codi.
  • Sections: Grup de paràgrafs.
  • Statements: Instruccions individuals.

Exemple:

PROCEDURE DIVISION.
Main-Procedure.
    OPEN INPUT InputFile.
    OPEN OUTPUT OutputFile.
    PERFORM Read-Process-Write UNTIL End-Of-File.
    CLOSE InputFile.
    CLOSE OutputFile.
    STOP RUN.

Read-Process-Write.
    READ InputFile INTO InputRecord
        AT END
            SET End-Of-File TO TRUE
        NOT AT END
            MOVE InputField1 TO OutputField1
            MOVE InputField2 TO OutputField2
            WRITE OutputRecord.

Resum

Entendre les quatre divisions principals en COBOL és essencial per escriure programes estructurats i mantenibles. Cada divisió té un paper específic:

  • Identification Division: Proporciona metadades sobre el programa.
  • Environment Division: Especifica l'entorn del sistema i l'ús dels fitxers.
  • Data Division: Defineix les estructures de dades i les variables.
  • Procedure Division: Conté la lògica executable del programa.

En dominar aquestes divisions, podràs crear programes COBOL ben organitzats que siguin més fàcils de llegir, depurar i mantenir.
