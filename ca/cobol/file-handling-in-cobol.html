Gestió de Fitxers en COBOL
Processament d'Arxius Seqüencials ►

La gestió de fitxers és un aspecte crucial de la programació en COBOL, ja que sovint s'utilitza per a aplicacions empresarials que requereixen un processament extensiu de dades. Aquest tema et guiarà a través dels conceptes bàsics de la gestió de fitxers en COBOL, des de la comprensió dels tipus de fitxers fins a operacions avançades amb fitxers.

Introducció a la Gestió de Fitxers

En COBOL, els fitxers s'utilitzen per emmagatzemar dades que poden ser llegides i escrites pel programa. Comprendre com gestionar fitxers és essencial per a qualsevol programador de COBOL.

Conceptes Clau

  • Tipus de Fitxers: Seqüencials, Indexats i Relatius.
  • Organització de Fitxers: Com es estructuren les dades dins del fitxer.
  • Modes d'Accés a Fitxers: Input, Output, I-O i Extend.
  • Control de Fitxers: Utilitzant les instruccions SELECT i ASSIGN.
  • Descripció de Fitxers: Utilitzant l'entrada FD (File Description) a la Divisió de Dades.

Tipus de Fitxers

COBOL suporta tres tipus principals de fitxers:

  1. Fitxers Seqüencials: Els registres s'emmagatzemen un darrere l'altre.
  2. Fitxers Indexats: Els registres s'emmagatzemen amb un índex, permetent un accés més ràpid.
  3. Fitxers Relatius: Els registres s'emmagatzemen en posicions relatives, permetent un accés directe.

Fitxers Seqüencials

Els fitxers seqüencials són el tipus més simple de fitxer en COBOL. Els registres es llegeixen i escriuen de manera seqüencial.

Exemple: Llegint un Fitxer Seqüencial

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadSeqFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD InputFile.
01 InputRecord PIC X(80).

WORKING-STORAGE SECTION.
01 WS-EOF PIC X VALUE 'N'.

PROCEDURE DIVISION.
    OPEN INPUT InputFile
    PERFORM UNTIL WS-EOF = 'Y'
        READ InputFile INTO InputRecord
            AT END
                MOVE 'Y' TO WS-EOF
            NOT AT END
                DISPLAY InputRecord
        END-READ
    END-PERFORM
    CLOSE InputFile
    STOP RUN.

Fitxers Indexats

Els fitxers indexats permeten un accés més ràpid als registres utilitzant un índex.

Exemple: Llegint un Fitxer Indexat

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadIdxFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT IndexedFile ASSIGN TO 'indexed.dat'
    ORGANIZATION IS INDEXED
    ACCESS MODE IS RANDOM
    RECORD KEY IS KeyField.

DATA DIVISION.
FILE SECTION.
FD IndexedFile.
01 IndexedRecord.
   05 KeyField PIC X(10).
   05 DataField PIC X(70).

WORKING-STORAGE SECTION.
01 WS-Key PIC X(10).

PROCEDURE DIVISION.
    OPEN I-O IndexedFile
    MOVE 'somekey' TO WS-Key
    READ IndexedFile
        INVALID KEY
            DISPLAY 'Record not found'
        NOT INVALID KEY
            DISPLAY IndexedRecord
    END-READ
    CLOSE IndexedFile
    STOP RUN.

Fitxers Relatius

Els fitxers relatius emmagatzemen registres en posicions relatives específiques.

Exemple: Llegint un Fitxer Relatiu

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadRelFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT RelativeFile ASSIGN TO 'relative.dat'
    ORGANIZATION IS RELATIVE
    ACCESS MODE IS RANDOM
    RELATIVE KEY IS RelKey.

DATA DIVISION.
FILE SECTION.
FD RelativeFile.
01 RelativeRecord.
   05 RelKey PIC 9(4).
   05 DataField PIC X(76).

WORKING-STORAGE SECTION.
01 WS-RelKey PIC 9(4).

PROCEDURE DIVISION.
    OPEN I-O RelativeFile
    MOVE 1234 TO WS-RelKey
    READ RelativeFile
        INVALID KEY
            DISPLAY 'Record not found'
        NOT INVALID KEY
            DISPLAY RelativeRecord
    END-READ
    CLOSE RelativeFile
    STOP RUN.

Modes d'Accés a Fitxers

COBOL proporciona diferents modes per accedir als fitxers:

  • INPUT: Accés només de lectura.
  • OUTPUT: Accés només d'escriptura, crea un nou fitxer.
  • I-O: Accés de lectura i escriptura.
  • EXTEND: Afegeix registres al final del fitxer.

Exemple: Escriure en un Fitxer Seqüencial

IDENTIFICATION DIVISION.
PROGRAM-ID. WriteSeqFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT OutputFile ASSIGN TO 'output.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD OutputFile.
01 OutputRecord PIC X(80).

WORKING-STORAGE SECTION.
01 WS-Data PIC X(80) VALUE 'This is a test record'.

PROCEDURE DIVISION.
    OPEN OUTPUT OutputFile
    MOVE WS-Data TO OutputRecord
    WRITE OutputRecord
    CLOSE OutputFile
    STOP RUN.

Control i Descripció de Fitxers

Control de Fitxers

Les instruccions SELECT i ASSIGN a la secció de Control de Fitxers de la Divisió d'Ambient s'utilitzen per associar un fitxer amb un mitjà d'emmagatzematge físic.

Descripció de Fitxers

L'entrada FD (File Description) a la Divisió de Dades descriu l'estructura dels registres del fitxer.

Exemple: Control i Descripció de Fitxers

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT MyFile ASSIGN TO 'myfile.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD MyFile.
01 MyRecord.
   05 Field1 PIC X(10).
   05 Field2 PIC 9(5).

Conclusió

La gestió de fitxers en COBOL és una habilitat fonamental per a qualsevol programador de COBOL. Comprenent els diferents tipus de fitxers, modes d'accés i com controlar i descriure fitxers, pots gestionar eficientment les dades dins dels teus programes COBOL. Practica amb els exemples proporcionats i experimenta amb diferents operacions de fitxers per aprofundir en la teva comprensió.
