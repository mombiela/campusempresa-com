Introducció

En aquest estudi de cas, explorarem com desenvolupar un Sistema de Gestió d'Inventari utilitzant COBOL. Aquest sistema permetrà als usuaris gestionar els articles d'inventari, incloent-hi afegir nous articles, actualitzar articles existents i generar informes. Al final d'aquest estudi de cas, tindràs una comprensió completa de com aplicar COBOL per resoldre problemes del món real.

Conceptes Clau

  • Divisió de Dades: Comprendre com definir i estructurar les dades.
  • Gestió d'Arxius: Llegir i escriure arxius.
  • Divisió Procedimental: Implementar la lògica de negoci.
  • Generació d'Informes: Crear sortides formatades.

Divisió de Dades

La Divisió de Dades és crucial per definir l'estructura de les dades amb les quals treballarem. En el nostre Sistema de Gestió d'Inventari, definirem els articles d'inventari.

Exemple

DATA DIVISION.
FILE SECTION.
FD  INVENTORY-FILE.
01  INVENTORY-RECORD.
    05  ITEM-ID            PIC X(10).
    05  ITEM-NAME          PIC X(30).
    05  ITEM-QUANTITY      PIC 9(5).
    05  ITEM-PRICE         PIC 9(5)V99.

Explicació

  • FILE SECTION: Defineix els arxius utilitzats en el programa.
  • FD INVENTORY-FILE: Descriptor d'arxiu per a l'arxiu d'inventari.
  • 01 INVENTORY-RECORD: Estructura de registre per a cada article d'inventari.
  • ITEM-ID: Identificador únic per a l'article.
  • ITEM-NAME: Nom de l'article.
  • ITEM-QUANTITY: Quantitat de l'article en estoc.
  • ITEM-PRICE: Preu de l'article.

Gestió d'Arxius

La gestió d'arxius en COBOL implica obrir, llegir, escriure i tancar arxius. Utilitzarem aquestes operacions per gestionar les nostres dades d'inventari.

Exemple

PROCEDURE DIVISION.
OPEN-INPUT-FILE.
    OPEN INPUT INVENTORY-FILE.
    IF FILE-STATUS NOT = "00"
        DISPLAY "Error opening file."
        STOP RUN.
    END-IF.

Explicació

  • OPEN INPUT INVENTORY-FILE: Obre l'arxiu d'inventari per a lectura.
  • FILE-STATUS: Comprova si l'arxiu s'ha obert correctament.

Divisió Procedimental

La Divisió Procedimental conté la lògica de negoci del programa. Aquí, implementarem funcionalitats com afegir nous articles i actualitzar articles existents.

Afegir un Nou Article

ADD-NEW-ITEM.
    DISPLAY "Enter Item ID: ".
    ACCEPT ITEM-ID.
    DISPLAY "Enter Item Name: ".
    ACCEPT ITEM-NAME.
    DISPLAY "Enter Item Quantity: ".
    ACCEPT ITEM-QUANTITY.
    DISPLAY "Enter Item Price: ".
    ACCEPT ITEM-PRICE.
    
    WRITE INVENTORY-RECORD.
    IF FILE-STATUS = "00"
        DISPLAY "Item added successfully."
    ELSE
        DISPLAY "Error adding item."
    END-IF.

Explicació

  • DISPLAY: Mostra un missatge a l'usuari.
  • ACCEPT: Pren entrada de l'usuari.
  • WRITE INVENTORY-RECORD: Escriu el nou article a l'arxiu d'inventari.
  • FILE-STATUS: Comprova si l'operació d'escriptura ha estat exitosa.

Generació d'Informes

Generar informes implica llegir les dades d'inventari i formatar-les per a la sortida.

Exemple

GENERATE-REPORT.
    OPEN OUTPUT REPORT-FILE.
    IF FILE-STATUS NOT = "00"
        DISPLAY "Error opening report file."
        STOP RUN.
    END-IF.
    
    PERFORM UNTIL END-OF-FILE
        READ INVENTORY-FILE INTO INVENTORY-RECORD
        AT END
            SET END-OF-FILE TO TRUE
        NOT AT END
            WRITE REPORT-RECORD FROM INVENTORY-RECORD
        END-READ
    END-PERFORM.
    
    CLOSE REPORT-FILE.

Explicació

  • OPEN OUTPUT REPORT-FILE: Obre l'arxiu d'informes per a escriptura.
  • PERFORM UNTIL END-OF-FILE: Bucla fins al final de l'arxiu d'inventari.
  • READ INVENTORY-FILE INTO INVENTORY-RECORD: Llegeix cada registre de l'arxiu d'inventari.
  • WRITE REPORT-RECORD FROM INVENTORY-RECORD: Escriu cada registre a l'arxiu d'informes.
  • CLOSE REPORT-FILE: Tanca l'arxiu d'informes.

Conclusió

En aquest estudi de cas, hem cobert els components essencials d'un Sistema de Gestió d'Inventari utilitzant COBOL. Hem explorat com definir estructures de dades, gestionar arxius, implementar la lògica de negoci i generar informes. Comprenent aquests conceptes i practicant amb els exemples proporcionats, estaràs ben preparat per desenvolupar sistemes similars en COBOL.
