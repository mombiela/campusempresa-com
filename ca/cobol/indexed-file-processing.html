El processament de fitxers indexats és un concepte crucial en COBOL, especialment per a aplicacions que requereixen una recuperació i manipulació eficient de dades. Aquest tema et guiarà des dels conceptes bàsics fins a tècniques avançades per gestionar fitxers indexats en COBOL.

Introducció als Fitxers Indexats

Els fitxers indexats en COBOL permeten un accés ràpid als registres utilitzant claus. Cada registre en un fitxer indexat té una o més claus que es poden utilitzar per localitzar ràpidament el registre.

  • Clau Primària: Identificador únic per a cada registre.
  • Clau Alternativa: Claus addicionals que es poden utilitzar per accedir als registres.

Definició de Fitxers Indexats

Per definir un fitxer indexat en COBOL, cal especificar-lo en el FILE-CONTROL i FILE SECTION.

FILE-CONTROL

El paràgraf FILE-CONTROL especifica l'organització del fitxer i el mode d'accés.

SELECT EMPLOYEE-FILE ASSIGN TO 'employee.dat'
    ORGANIZATION IS INDEXED
    ACCESS MODE IS DYNAMIC
    RECORD KEY IS EMP-ID
    ALTERNATE RECORD KEY IS EMP-NAME WITH DUPLICATES.

FILE SECTION

La FILE SECTION descriu l'estructura dels registres en el fitxer.

FD EMPLOYEE-FILE.
01 EMPLOYEE-RECORD.
   05 EMP-ID      PIC X(5).
   05 EMP-NAME    PIC X(20).
   05 EMP-ADDRESS PIC X(50).
   05 EMP-SALARY  PIC 9(7)V99.

Obrir i Tancar Fitxers Indexats

Abans de processar un fitxer indexat, cal obrir-lo. De la mateixa manera, s'ha de tancar després de processar-lo.

Obrir un Fitxer

OPEN INPUT EMPLOYEE-FILE.

Tancar un Fitxer

CLOSE EMPLOYEE-FILE.

Llegir Fitxers Indexats

Llegir registres d'un fitxer indexat es pot fer utilitzant la instrucció READ. Es pot llegir seqüencialment o aleatòriament basant-se en la clau.

Lectura Seqüencial

READ EMPLOYEE-FILE NEXT
    AT END
        DISPLAY 'End of File Reached'
    NOT AT END
        DISPLAY EMPLOYEE-RECORD.

Lectura Aleatòria

MOVE '12345' TO EMP-ID.
READ EMPLOYEE-FILE
    INVALID KEY
        DISPLAY 'Record Not Found'
    NOT INVALID KEY
        DISPLAY EMPLOYEE-RECORD.

Escriure a Fitxers Indexats

Per afegir nous registres a un fitxer indexat, utilitza la instrucció WRITE.

MOVE '67890' TO EMP-ID.
MOVE 'John Doe' TO EMP-NAME.
MOVE '123 Elm Street' TO EMP-ADDRESS.
MOVE 50000.00 TO EMP-SALARY.
WRITE EMPLOYEE-RECORD.

Actualitzar Fitxers Indexats

Actualitzar registres implica llegir el registre, modificar-lo i després reescriure'l.

MOVE '12345' TO EMP-ID.
READ EMPLOYEE-FILE
    INVALID KEY
        DISPLAY 'Record Not Found'
    NOT INVALID KEY
        MOVE 'Jane Doe' TO EMP-NAME
        REWRITE EMPLOYEE-RECORD.

Eliminar Registres

Per eliminar un registre, utilitza la instrucció DELETE.

MOVE '12345' TO EMP-ID.
READ EMPLOYEE-FILE
    INVALID KEY
        DISPLAY 'Record Not Found'
    NOT INVALID KEY
        DELETE EMPLOYEE-RECORD.

Gestió d'Errors

La gestió adequada d'errors és crucial per a un processament robust de fitxers.

  • INVALID KEY: S'utilitza quan no es troba una clau.
  • AT END: S'utilitza quan s'arriba al final del fitxer durant el processament seqüencial.
READ EMPLOYEE-FILE NEXT
    AT END
        DISPLAY 'End of File Reached'
    NOT AT END
        DISPLAY EMPLOYEE-RECORD.

Resum

El processament de fitxers indexats en COBOL permet una recuperació i manipulació eficient de dades utilitzant claus. Entendre com definir, obrir, llegir, escriure, actualitzar i eliminar registres en fitxers indexats és essencial per desenvolupar aplicacions robustes en COBOL. Una gestió adequada d'errors assegura que la teva aplicació pugui gestionar situacions inesperades de manera elegant.

Dominant aquests conceptes, estaràs ben equipat per gestionar tasques complexes de processament de dades en COBOL.
