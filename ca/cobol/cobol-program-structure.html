Introducció

COBOL (Common Business-Oriented Language) és un dels llenguatges de programació d'alt nivell més antics, utilitzat principalment en sistemes de negocis, finances i administració per a empreses i governs. Entendre l'estructura d'un programa COBOL és fonamental per escriure codi eficient i sense errors. Aquesta secció et guiarà a través de l'estructura bàsica d'un programa COBOL, incloent-hi les seves divisions, seccions i paràgrafs.

Estructura Bàsica d'un Programa COBOL

Un programa COBOL es divideix en quatre divisions principals:

  1. Identification Division
  2. Environment Division
  3. Data Division
  4. Procedure Division

Cada divisió té un propòsit específic i conté diverses seccions i paràgrafs.

Identification Division

La Identification Division és la primera divisió en un programa COBOL. Proporciona metadades sobre el programa.

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.
AUTHOR. John Doe.
  • PROGRAM-ID: Especifica el nom del programa.
  • AUTHOR: Indica l'autor del programa.

Environment Division

La Environment Division especifica l'entorn en què s'executarà el programa. Conté dues seccions: Configuration Section i Input-Output Section.

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
SOURCE-COMPUTER. IBM-370.
OBJECT-COMPUTER. IBM-370.

INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT INPUT-FILE ASSIGN TO 'input.txt'.
    SELECT OUTPUT-FILE ASSIGN TO 'output.txt'.
  • SOURCE-COMPUTER: Especifica l'ordinador en què es compila el programa font.
  • OBJECT-COMPUTER: Especifica l'ordinador en què s'executarà el programa.
  • FILE-CONTROL: Gestiona les assignacions de fitxers.

Data Division

La Data Division s'utilitza per definir les variables i estructures de dades. Conté diverses seccions, però les més utilitzades són la File Section i la Working-Storage Section.

DATA DIVISION.
FILE SECTION.
FD  INPUT-FILE.
01  INPUT-RECORD PIC X(100).

FD  OUTPUT-FILE.
01  OUTPUT-RECORD PIC X(100).

WORKING-STORAGE SECTION.
01  WS-NAME PIC X(20).
01  WS-AGE  PIC 99.
  • FILE SECTION: Defineix l'estructura dels fitxers.
  • WORKING-STORAGE SECTION: Defineix variables i constants utilitzades en el programa.

Procedure Division

La Procedure Division conté la lògica del programa. Consisteix en una sèrie d'instruccions que manipulen les dades definides a la Data Division.

PROCEDURE DIVISION.
MAIN-PARA.
    OPEN INPUT INPUT-FILE.
    OPEN OUTPUT OUTPUT-FILE.
    PERFORM READ-INPUT UNTIL END-OF-FILE.
    CLOSE INPUT-FILE.
    CLOSE OUTPUT-FILE.
    STOP RUN.

READ-INPUT.
    READ INPUT-FILE INTO INPUT-RECORD
        AT END
            MOVE 'YES' TO END-OF-FILE
        NOT AT END
            MOVE INPUT-RECORD TO OUTPUT-RECORD
            WRITE OUTPUT-RECORD.
  • MAIN-PARA: El paràgraf principal on comença l'execució.
  • READ-INPUT: Un paràgraf que llegeix registres del fitxer d'entrada i els escriu al fitxer de sortida.

Conclusió

Entendre l'estructura d'un programa COBOL és essencial per escriure codi efectiu i mantenible. Les quatre divisions principals—Identification, Environment, Data i Procedure—cadascuna té un propòsit específic i han d'estar incloses en cada programa COBOL. Dominant aquestes divisions i les seves respectives seccions, estaràs ben preparat per abordar tasques de programació COBOL més complexes.
