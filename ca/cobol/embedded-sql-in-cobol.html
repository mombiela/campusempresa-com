Introducció al SQL incrustat

El SQL incrustat permet que els programes COBOL executin sentències SQL directament dins del codi COBOL. Aquesta integració permet que les aplicacions COBOL interactuïn amb bases de dades relacionals sense problemes.

Conceptes clau

  • SQL incrustat: Sentències SQL incrustades dins d'un programa en un llenguatge hoste (COBOL).
  • Variables hoste: Variables COBOL utilitzades per passar dades entre el programa COBOL i les sentències SQL.
  • EXEC SQL: La paraula clau utilitzada per iniciar una sentència SQL incrustada.
  • END-EXEC: La paraula clau utilitzada per acabar una sentència SQL incrustada.

Configuració del SQL incrustat en COBOL

Per utilitzar SQL incrustat en COBOL, cal:

  1. Incloure l'SQLCA (Àrea de Comunicació SQL): Aquesta és una estructura que proporciona retroalimentació sobre l'execució de les sentències SQL.
  2. Declarar Variables Hoste: Aquestes són variables COBOL que s'utilitzaran en les sentències SQL.
  3. Utilitzar EXEC SQL i END-EXEC: Per incrustar sentències SQL dins del codi COBOL.

Exemple: Configuració bàsica

IDENTIFICATION DIVISION.
PROGRAM-ID. EmbeddedSQLExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
EXEC SQL INCLUDE SQLCA END-EXEC.

01 EMP-ID PIC 9(4).
01 EMP-NAME PIC X(20).

PROCEDURE DIVISION.
    EXEC SQL
        SELECT EMP_NAME
        INTO :EMP-NAME
        FROM EMPLOYEES
        WHERE EMP_ID = :EMP-ID
    END-EXEC.

    DISPLAY 'Employee Name: ' EMP-NAME.
    STOP RUN.
  • SQLCA: Inclòs per gestionar la retroalimentació de l'execució SQL.
  • Variables Hoste: EMP-ID i EMP-NAME s'utilitzen per passar dades entre COBOL i SQL.

Operacions SQL comunes en COBOL

Sentència SELECT

La sentència SELECT recupera dades de la base de dades.

Exemple: Sentència SELECT

EXEC SQL
    SELECT EMP_NAME
    INTO :EMP-NAME
    FROM EMPLOYEES
    WHERE EMP_ID = :EMP-ID
END-EXEC.
  • Clàusula INTO: Especifica la variable hoste per emmagatzemar el resultat.
  • Clàusula WHERE: Filtra les dades basant-se en la condició.

Sentència INSERT

La sentència INSERT afegeix nous registres a la base de dades.

Exemple: Sentència INSERT

EXEC SQL
    INSERT INTO EMPLOYEES (EMP_ID, EMP_NAME)
    VALUES (:EMP-ID, :EMP-NAME)
END-EXEC.
  • Clàusula VALUES: Especifica els valors que s'inseriran a la taula.

Sentència UPDATE

La sentència UPDATE modifica registres existents a la base de dades.

Exemple: Sentència UPDATE

EXEC SQL
    UPDATE EMPLOYEES
    SET EMP_NAME = :EMP-NAME
    WHERE EMP_ID = :EMP-ID
END-EXEC.
  • Clàusula SET: Especifica la columna a actualitzar i el nou valor.

Sentència DELETE

La sentència DELETE elimina registres de la base de dades.

Exemple: Sentència DELETE

EXEC SQL
    DELETE FROM EMPLOYEES
    WHERE EMP_ID = :EMP-ID
END-EXEC.
  • Clàusula WHERE: Especifica la condició per identificar quins registres eliminar.

Gestió d'errors SQL

Els errors SQL es poden gestionar utilitzant l'estructura SQLCA. El camp SQLCODE en SQLCA indica el resultat de l'operació SQL.

Exemple: Gestió d'errors

EXEC SQL
    SELECT EMP_NAME
    INTO :EMP-NAME
    FROM EMPLOYEES
    WHERE EMP_ID = :EMP-ID
END-EXEC.

IF SQLCODE NOT = 0
    DISPLAY 'Error: ' SQLCODE
ELSE
    DISPLAY 'Employee Name: ' EMP-NAME.
  • SQLCODE: Un camp en SQLCA que conté el resultat de l'operació SQL. Un valor de 0 indica èxit, mentre que altres valors indiquen diferents tipus d'errors.

Conclusió

El SQL incrustat en COBOL permet interaccions potents amb bases de dades directament dins dels programes COBOL. Entenent la configuració bàsica, les operacions SQL comunes i la gestió d'errors, pots integrar efectivament SQL amb COBOL per gestionar i manipular dades en bases de dades relacionals. Practica aquests conceptes amb diverses operacions SQL per convertir-te en un expert en l'ús del SQL incrustat en COBOL.
