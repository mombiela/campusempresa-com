Introducció

En aquest estudi de cas, explorarem com dissenyar i implementar un sistema bancari utilitzant COBOL. Això cobrirà diversos aspectes des de la gestió bàsica de comptes fins a funcions més avançades com el processament de transaccions i la generació d'informes. Al final d'aquest estudi de cas, tindràs una comprensió completa de com es pot utilitzar COBOL per construir aplicacions bancàries robustes.

Conceptes clau

  • Divisió de dades: Comprendre com definir i estructurar les dades.
  • Divisió de procediments: Escriure la lògica per manipular les dades.
  • Gestió d'arxius: Llegir i escriure arxius.
  • Estructures de control: Utilitzar instruccions IF, bucles PERFORM, etc.
  • Subprogrames: Modularitzar el codi per a un millor manteniment i llegibilitat.

Divisió de dades

La Divisió de dades és on definim les estructures de dades que s'utilitzaran en el nostre sistema bancari.

Exemple: Definició de registres de clients

DATA DIVISION.
FILE SECTION.
FD  CUSTOMER-FILE.
01  CUSTOMER-RECORD.
    05  CUSTOMER-ID          PIC 9(5).
    05  CUSTOMER-NAME        PIC X(30).
    05  CUSTOMER-ADDRESS     PIC X(50).
    05  CUSTOMER-BALANCE     PIC 9(7)V99.
  • CUSTOMER-ID: Un identificador numèric de 5 dígits per al client.
  • CUSTOMER-NAME: Un camp de 30 caràcters per al nom del client.
  • CUSTOMER-ADDRESS: Un camp de 50 caràcters per a l'adreça del client.
  • CUSTOMER-BALANCE: Un camp numèric amb 7 dígits abans del decimal i 2 dígits després del decimal per al saldo del client.

Divisió de procediments

La Divisió de procediments conté la lògica per processar les dades definides a la Divisió de dades.

Exemple: Afegir un nou client

PROCEDURE DIVISION.
ADD-CUSTOMER.
    DISPLAY "Enter Customer ID: ".
    ACCEPT CUSTOMER-ID.
    DISPLAY "Enter Customer Name: ".
    ACCEPT CUSTOMER-NAME.
    DISPLAY "Enter Customer Address: ".
    ACCEPT CUSTOMER-ADDRESS.
    MOVE 0 TO CUSTOMER-BALANCE.
    WRITE CUSTOMER-RECORD.
    DISPLAY "Customer added successfully.".
  • DISPLAY: Mostra un missatge a la pantalla.
  • ACCEPT: Pren entrada de l'usuari.
  • MOVE: Assigna un valor a una variable.
  • WRITE: Escriu un registre a l'arxiu.

Gestió d'arxius

La gestió d'arxius és crucial per a un sistema bancari per emmagatzemar i recuperar dades de clients.

Exemple: Llegir registres de clients

PROCEDURE DIVISION.
READ-CUSTOMER.
    OPEN INPUT CUSTOMER-FILE.
    READ CUSTOMER-FILE INTO CUSTOMER-RECORD
        AT END
            DISPLAY "No more records."
            GO TO END-READ.
    DISPLAY "Customer ID: " CUSTOMER-ID.
    DISPLAY "Customer Name: " CUSTOMER-NAME.
    DISPLAY "Customer Address: " CUSTOMER-ADDRESS.
    DISPLAY "Customer Balance: " CUSTOMER-BALANCE.
END-READ.
    CLOSE CUSTOMER-FILE.
  • OPEN INPUT: Obre l'arxiu per a lectura.
  • READ: Llegeix un registre de l'arxiu.
  • AT END: Comprova si s'ha arribat al final de l'arxiu.
  • CLOSE: Tanca l'arxiu.

Estructures de control

Les estructures de control ajuden a prendre decisions i repetir tasques.

Exemple: Actualitzar el saldo del client

PROCEDURE DIVISION.
UPDATE-BALANCE.
    DISPLAY "Enter Customer ID: ".
    ACCEPT CUSTOMER-ID.
    OPEN I-O CUSTOMER-FILE.
    READ CUSTOMER-FILE INTO CUSTOMER-RECORD
        INVALID KEY
            DISPLAY "Customer not found."
            GO TO END-UPDATE.
    DISPLAY "Enter new balance: ".
    ACCEPT CUSTOMER-BALANCE.
    REWRITE CUSTOMER-RECORD.
    DISPLAY "Balance updated successfully.".
END-UPDATE.
    CLOSE CUSTOMER-FILE.
  • INVALID KEY: Comprova si no s'ha trobat el registre.
  • REWRITE: Actualitza un registre existent a l'arxiu.

Subprogrames

Els subprogrames permeten modularitzar el codi per a una millor organització i reutilització.

Exemple: Subprograma per mostrar els detalls del client

IDENTIFICATION DIVISION.
PROGRAM-ID. DISPLAY-CUSTOMER.
DATA DIVISION.
LINKAGE SECTION.
01  CUSTOMER-RECORD.
    05  CUSTOMER-ID          PIC 9(5).
    05  CUSTOMER-NAME        PIC X(30).
    05  CUSTOMER-ADDRESS     PIC X(50).
    05  CUSTOMER-BALANCE     PIC 9(7)V99.
PROCEDURE DIVISION USING CUSTOMER-RECORD.
    DISPLAY "Customer ID: " CUSTOMER-ID.
    DISPLAY "Customer Name: " CUSTOMER-NAME.
    DISPLAY "Customer Address: " CUSTOMER-ADDRESS.
    DISPLAY "Customer Balance: " CUSTOMER-BALANCE.
    EXIT PROGRAM.
  • LINKAGE SECTION: Defineix les dades passades al subprograma.
  • USING: Especifica els paràmetres passats al subprograma.
  • EXIT PROGRAM: Finalitza el subprograma.

Conclusió

En aquest estudi de cas, hem cobert els components essencials per construir un sistema bancari utilitzant COBOL. Hem començat definint estructures de dades, hem passat a escriure la lògica per afegir i actualitzar registres de clients, i hem explorat la gestió d'arxius i les estructures de control. Finalment, hem vist com modularitzar el codi utilitzant subprogrames. Comprenent aquests conceptes, pots construir un sistema bancari robust i fàcil de mantenir en COBOL.
