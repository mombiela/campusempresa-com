Introducció a NoSQL

Les bases de dades NoSQL (Not Only SQL) són una alternativa a les bases de dades relacionals tradicionals. Estan dissenyades per manejar grans volums de dades, alta velocitat de processament i estructures de dades flexibles. Són especialment útils en aplicacions que requereixen escalabilitat horitzontal i alta disponibilitat.

Característiques Principals de NoSQL

  • Escalabilitat Horitzontal: Capacitat de distribuir dades i càrregues de treball a través de múltiples servidors.
  • Flexibilitat d'Esquema: Permeten emmagatzemar dades sense un esquema fix, cosa que facilita l'adaptació a canvis en les dades.
  • Alt Rendiment: Optimitzades per a operacions ràpides de lectura i escriptura.
  • Alta Disponibilitat: Dissenyades per minimitzar el temps d'inactivitat i garantir la disponibilitat contínua de les dades.

Tipus de Bases de Dades NoSQL

Hi ha diversos tipus de bases de dades NoSQL, cadascuna optimitzada per a diferents tipus d'aplicacions i casos d'ús.

Bases de Dades de Documents

  • Descripció: Emmagatzemen dades en documents similars a JSON.
  • Exemples: MongoDB, CouchDB.
  • Ús Comú: Aplicacions web, sistemes de gestió de contingut.
{
  "nom": "Joan",
  "edat": 30,
  "adreça": {
    "carrer": "Carrer Fals 123",
    "ciutat": "Springfield"
  }
}

Bases de Dades de Clau-Valor

  • Descripció: Emmagatzemen dades com a parells clau-valor.
  • Exemples: Redis, DynamoDB.
  • Ús Comú: Memòries cau, sessions d'usuari.
# Exemple en Redis
SET "nom" "Joan"
GET "nom"

Bases de Dades de Columnes

  • Descripció: Emmagatzemen dades en columnes en lloc de files.
  • Exemples: Apache Cassandra, HBase.
  • Ús Comú: Anàlisi de grans volums de dades, emmagatzematge de registres.
-- Exemple en Cassandra CQL
CREATE TABLE usuaris (
  id UUID PRIMARY KEY,
  nom TEXT,
  edat INT
);

Bases de Dades de Grafs

  • Descripció: Emmagatzemen dades en nodes i relacions.
  • Exemples: Neo4j, ArangoDB.
  • Ús Comú: Xarxes socials, motors de recomanació.
// Exemple en Neo4j
CREATE (joan:Persona {nom: "Joan", edat: 30})
CREATE (maria:Persona {nom: "Maria", edat: 25})
CREATE (joan)-[:AMIC_DE]->(maria)

Comparació entre NoSQL i SQL

| Característica | NoSQL | SQL | |------------------------|--------------------------------|---------------------------------| | Esquema | Flexible | Fix | | Escalabilitat | Horitzontal | Vertical | | Consistència | Eventual (en molts casos) | Forta | | Transaccions | Limitades | Completes | | Tipus de Dades | Diversos (documents, grafs) | Taules |

Exemples Pràctics

Exemple amb MongoDB

// Connexió a MongoDB i creació d'una col·lecció
const { MongoClient } = require('mongodb');
const uri = "mongodb+srv://<username>:<password>@cluster0.mongodb.net/test?retryWrites=true&w=majority";
const client = new MongoClient(uri, { useNewUrlParser: true, useUnifiedTopology: true });

async function run() {
  try {
    await client.connect();
    const database = client.db('testdb');
    const collection = database.collection('usuaris');

    // Inserir un document
    const result = await collection.insertOne({ nom: "Joan", edat: 30 });
    console.log(`Document inserit amb el _id: ${result.insertedId}`);

    // Consultar un document
    const usuari = await collection.findOne({ nom: "Joan" });
    console.log(usuari);
  } finally {
    await client.close();
  }
}

run().catch(console.dir);

Exemple amb Redis

import redis

# Connexió a Redis
r = redis.Redis(host='localhost', port=6379, db=0)

# Inserir un valor
r.set('nom', 'Joan')

# Obtenir un valor
nom = r.get('nom')
print(nom.decode('utf-8'))

Conclusió

Les bases de dades NoSQL ofereixen una solució poderosa i flexible per al processament de dades massives. La seva capacitat per manejar grans volums de dades, juntament amb la seva escalabilitat i flexibilitat d'esquema, les fa ideals per a una varietat d'aplicacions modernes. Comprendre els diferents tipus de bases de dades NoSQL i els seus casos d'ús específics és crucial per seleccionar la millor eina per a cada projecte.
