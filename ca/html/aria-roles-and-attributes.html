Introducció a ARIA

ARIA (Accessible Rich Internet Applications) és un conjunt d'atributs que defineixen maneres de fer que el contingut web i les aplicacions web siguin més accessibles per a persones amb discapacitats. ARIA és especialment útil per a contingut dinàmic i controls avançats d'interfície d'usuari desenvolupats amb HTML, JavaScript i tecnologies relacionades.

Per què ARIA és Important

  • Millora l'Accessibilitat: ARIA ajuda a fer que les aplicacions web siguin accessibles per a usuaris amb discapacitats, incloent-hi aquells que depenen de lectors de pantalla.
  • Millora la Usabilitat: Proporcionant context i informació addicional, ARIA pot millorar l'experiència general de l'usuari.
  • Compliment: Utilitzar ARIA pot ajudar a assegurar que les teves aplicacions web compleixin amb els estàndards d'accessibilitat i els requisits legals.

Rols ARIA

Els rols ARIA defineixen què és o què fa un element. Ajuden les tecnologies assistives a entendre la finalitat d'un element.

Rols ARIA Comuns

  • role="button": Indica que un element és un botó.
  • role="checkbox": Indica que un element és una casella de verificació.
  • role="dialog": Indica que un element és una caixa de diàleg.
  • role="alert": Indica que un element és un missatge d'alerta.
  • role="navigation": Indica que un element és una secció de navegació.

Exemple de Rols ARIA

<button role="button">Clica'm</button>
<div role="alert">Aquest és un missatge d'alerta.</div>
<nav role="navigation">
  <ul>
    <li><a href="#home">Inici</a></li>
    <li><a href="#about">Sobre nosaltres</a></li>
    <li><a href="#contact">Contacte</a></li>
  </ul>
</nav>

Atributs ARIA

Els atributs ARIA proporcionen informació addicional sobre els elements, com ara el seu estat o propietats.

Atributs ARIA Comuns

  • aria-label: Proporciona una etiqueta per a un element.
  • aria-labelledby: Identifica l'element (o elements) que etiqueta l'element actual.
  • aria-hidden: Indica si un element és visible o amagat.
  • aria-expanded: Indica si un element, com ara un menú o un node d'arbre, està expandit o col·lapsat.
  • aria-checked: Indica l'estat actual de "verificat" de les caselles de verificació, botons de ràdio i altres ginys.

Exemple d'Atributs ARIA

<button aria-label="Tancar">X</button>
<div id="menu" aria-expanded="false">
  <button aria-controls="menu" aria-expanded="false">Menú</button>
  <ul>
    <li><a href="#item1">Element 1</a></li>
    <li><a href="#item2">Element 2</a></li>
    <li><a href="#item3">Element 3</a></li>
  </ul>
</div>

Combinació de Rols i Atributs ARIA

Combinant rols i atributs ARIA es pot proporcionar una experiència més rica i accessible per als usuaris.

Exemple de Combinació de Rols i Atributs ARIA

<div role="dialog" aria-labelledby="dialogTitle" aria-hidden="true">
  <h2 id="dialogTitle">Títol del Diàleg</h2>
  <p>Aquesta és una caixa de diàleg.</p>
  <button aria-label="Tancar Diàleg">Tancar</button>
</div>

Millors Pràctiques per Utilitzar ARIA

  • Utilitza Elements HTML Nadius Primer: Sempre que sigui possible, utilitza elements HTML nadius que tinguin funcions d'accessibilitat integrades.
  • No Abusis d'ARIA: Afegir massa atributs ARIA pot fer que el teu codi sigui més difícil de mantenir i entendre.
  • Prova amb Tecnologies Assistives: Prova regularment les teves aplicacions web amb lectors de pantalla i altres tecnologies assistives per assegurar-te que funcionen com s'espera.

Conclusió

Els rols i atributs ARIA són eines poderoses per millorar l'accessibilitat de les aplicacions web. Entenent i implementant correctament ARIA, pots crear experiències web més inclusives i amigables per als usuaris. Recorda utilitzar primer els elements HTML nadius, evitar l'abús d'ARIA i provar les teves aplicacions amb tecnologies assistives per assegurar-te que compleixen els estàndards d'accessibilitat.
