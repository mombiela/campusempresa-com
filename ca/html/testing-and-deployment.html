Introducció

En aquesta secció, tractarem els aspectes essencials de les proves i el desplegament de projectes HTML. Les proves asseguren que el teu codi HTML funcioni com s'espera en diferents navegadors i dispositius, mentre que el desplegament implica fer que el teu projecte sigui accessible als usuaris a la web.

Proves HTML

Per què són importants les proves

  • Asseguren la compatibilitat entre navegadors.
  • Identifiquen i solucionen errors abans del desplegament.
  • Milloren l'experiència de l'usuari assegurant la funcionalitat.

Eines per provar HTML

  • Eines de desenvolupador del navegador: Eines integrades en navegadors com Chrome, Firefox i Edge.
  • Validators: Eines com el W3C Markup Validation Service per comprovar errors de sintaxi HTML.
  • Eines de proves entre navegadors: Serveis com BrowserStack i Sauce Labs per provar en diferents navegadors i dispositius.

Ús de les eines de desenvolupador del navegador

La majoria dels navegadors moderns venen amb eines de desenvolupador integrades que et permeten inspeccionar i depurar el teu codi HTML.

Exemple: Inspeccionar un element a Chrome

  1. Fes clic dret sobre l'element que vols inspeccionar.
  2. Selecciona "Inspecciona" del menú contextual.
  3. El panell d'eines de desenvolupador s'obrirà, ressaltant el codi HTML per a l'element seleccionat.
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Pàgina de mostra</title>
</head>
<body>
    <h1>Benvingut al meu lloc web</h1>
    <p>Aquest és un paràgraf de mostra.</p>
</body>
</html>

Validació HTML

Utilitzar un validator ajuda a assegurar que el teu codi HTML compleixi amb els estàndards web.

Exemple: Ús del W3C Markup Validation Service

  1. Ves a W3C Markup Validation Service.
  2. Introdueix l'URL de la teva pàgina HTML o carrega el teu fitxer HTML.
  3. Fes clic a "Check" per veure els resultats de la validació.

Proves entre navegadors

Provar el teu codi HTML en diferents navegadors assegura una experiència d'usuari consistent.

Exemple: Ús de BrowserStack

  1. Registra't per a un compte de BrowserStack.
  2. Tria els navegadors i dispositius que vols provar.
  3. Introdueix l'URL de la teva pàgina HTML.
  4. BrowserStack carregarà la teva pàgina en els entorns seleccionats, permetent-te interactuar amb ella i identificar qualsevol problema.

Desplegament

Preparació per al desplegament

  • Minimitzar HTML: Redueix la mida del fitxer eliminant espais en blanc i comentaris innecessaris.
  • Optimitzar recursos: Comprimeix imatges i altres fitxers multimèdia per millorar els temps de càrrega.
  • Comprovar enllaços: Assegura't que tots els enllaços interns i externs funcionin correctament.

Mètodes de desplegament

  • FTP/SFTP: Carrega fitxers a un servidor web utilitzant el Protocol de Transferència de Fitxers.
  • GitHub Pages: Allotja llocs web estàtics directament des d'un repositori de GitHub.
  • Serveis d'allotjament web: Utilitza serveis com Netlify, Vercel o proveïdors tradicionals d'allotjament web.

Exemple: Desplegament amb GitHub Pages

  1. Crea un repositori de GitHub per al teu projecte.
  2. Puja els teus fitxers HTML al repositori.
  3. Ves a la configuració del repositori.
  4. Desplaça't fins a la secció "GitHub Pages".
  5. Selecciona la branca des de la qual vols desplegar (normalment main o master).
  6. El teu lloc estarà disponible a https://<nom-usuari>.github.io/<nom-repositori>/.

Exemple: Desplegament amb Netlify

  1. Registra't per a un compte de Netlify.
  2. Connecta el teu repositori de GitHub a Netlify.
  3. Configura els paràmetres de construcció si és necessari.
  4. Fes clic a "Deploy Site."
  5. El teu lloc estarà disponible a una URL generada per Netlify, que pots personalitzar.

Conclusió

Les proves i el desplegament són passos crucials en el procés de desenvolupament web. Provar a fons el teu codi HTML et permet assegurar una experiència d'usuari consistent i lliure d'errors en diferents navegadors i dispositius. Desplegar el teu projecte el fa accessible als usuaris a la web, i hi ha diversos mètodes per triar segons les teves necessitats. Seguir les millors pràctiques tant en les proves com en el desplegament t'ajudarà a crear projectes web d'alta qualitat i fiables.
