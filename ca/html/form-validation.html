La validació de formularis és un aspecte crucial del desenvolupament web que assegura que les dades introduïdes pels usuaris són precises i compleixen els criteris requerits abans de ser processades o emmagatzemades. Aquest tema cobrirà els conceptes bàsics de la validació de formularis en HTML, progressant cap a tècniques més avançades.

Introducció a la Validació de Formularis

La validació de formularis es pot categoritzar en dos tipus principals:

  • Validació del costat del client: Es realitza al navegador de l'usuari abans que les dades del formulari s'enviïn al servidor.
  • Validació del costat del servidor: Es realitza al servidor després que les dades del formulari hagin estat enviades.

En aquesta secció, ens centrarem en la validació del costat del client utilitzant HTML.

Validació Bàsica de Formularis en HTML

HTML5 va introduir diversos nous tipus d'entrada i atributs que fan que la validació de formularis sigui més fàcil i intuïtiva.

Tipus d'Entrada

HTML5 proporciona diversos tipus d'entrada que automàticament validen la introducció de l'usuari:

  • type="email": Assegura que la introducció és una adreça de correu electrònic vàlida.
  • type="url": Assegura que la introducció és una URL vàlida.
  • type="number": Assegura que la introducció és un número.
  • type="tel": Assegura que la introducció és un número de telèfon vàlid.
  • type="date": Assegura que la introducció és una data vàlida.

Exemple

<form>
  <label for="email">Correu electrònic:</label>
  <input type="email" id="email" name="email" required>
  
  <label for="age">Edat:</label>
  <input type="number" id="age" name="age" min="18" max="99" required>
  
  <button type="submit">Enviar</button>
</form>

En aquest exemple:

  • L'entrada email requereix una adreça de correu electrònic vàlida.
  • L'entrada age requereix un número entre 18 i 99.

Atributs per a la Validació

HTML5 també proporciona diversos atributs per millorar la validació de formularis:

  • required: Assegura que el camp no es deixi buit.
  • min i max: Defineixen els valors mínims i màxims per a les entrades numèriques.
  • pattern: Defineix una expressió regular que la introducció ha de coincidir.
  • maxlength: Especifica el nombre màxim de caràcters permesos.

Exemple

<form>
  <label for="username">Nom d'usuari:</label>
  <input type="text" id="username" name="username" required maxlength="15">
  
  <label for="password">Contrasenya:</label>
  <input type="password" id="password" name="password" required pattern=".{8,}">
  
  <button type="submit">Registrar</button>
</form>

En aquest exemple:

  • L'entrada username és requerida i no pot excedir els 15 caràcters.
  • L'entrada password és requerida i ha de tenir almenys 8 caràcters.

Validació Avançada de Formularis en HTML

Per a una validació més complexa, es poden utilitzar missatges de validació personalitzats i JavaScript.

Missatges de Validació Personalitzats

Pots personalitzar els missatges de validació utilitzant el mètode setCustomValidity en JavaScript.

Exemple

<form id="customForm">
  <label for="customEmail">Correu electrònic:</label>
  <input type="email" id="customEmail" name="customEmail" required>
  
  <button type="submit">Enviar</button>
</form>

<script>
  document.getElementById('customForm').addEventListener('submit', function(event) {
    var emailInput = document.getElementById('customEmail');
    if (!emailInput.validity.valid) {
      emailInput.setCustomValidity('Si us plau, introdueix una adreça de correu electrònic vàlida.');
    } else {
      emailInput.setCustomValidity('');
    }
  });
</script>

En aquest exemple:

  • El formulari mostrarà un missatge personalitzat si el correu electrònic no és vàlid.

JavaScript per a Validació Complexa

Per a validacions que no es poden gestionar només amb atributs HTML, es pot utilitzar JavaScript.

Exemple

<form id="complexForm">
  <label for="password1">Contrasenya:</label>
  <input type="password" id="password1" name="password1" required>
  
  <label for="password2">Confirma la Contrasenya:</label>
  <input type="password" id="password2" name="password2" required>
  
  <button type="submit">Registrar</button>
</form>

<script>
  document.getElementById('complexForm').addEventListener('submit', function(event) {
    var password1 = document.getElementById('password1').value;
    var password2 = document.getElementById('password2').value;
    
    if (password1 !== password2) {
      event.preventDefault();
      alert('Les contrasenyes no coincideixen.');
    }
  });
</script>

En aquest exemple:

  • JavaScript comprova si les contrasenyes coincideixen abans de permetre que el formulari sigui enviat.

Conclusió

La validació de formularis és una part essencial del desenvolupament web que assegura la integritat de les dades i millora l'experiència de l'usuari. HTML5 proporciona funcions de validació integrades que són fàcils d'utilitzar i cobreixen la majoria d'escenaris comuns. Per a necessitats de validació més complexes, es pot utilitzar JavaScript per crear una lògica de validació personalitzada. Comprenent i utilitzant aquestes tècniques, pots crear formularis robustos i fàcils d'utilitzar.
