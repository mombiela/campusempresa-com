Introducció

En el desenvolupament d'una REST API, la definició de rutes i controladors és fonamental per organitzar i gestionar les sol·licituds HTTP de manera eficient. Les rutes determinen com s'accedeix als recursos, mentre que els controladors gestionen la lògica associada a aquestes rutes.

Rutes

Les rutes són els punts finals (endpoints) de l'API que responen a les sol·licituds HTTP. Cada ruta està associada a una URL específica i a un mètode HTTP (GET, POST, PUT, DELETE, etc.).

Conceptes Clau de les Rutes

  • Endpoint: La URL específica que s'utilitza per accedir a un recurs.
  • Mètode HTTP: Defineix l'acció que es realitzarà en el recurs (GET, POST, PUT, DELETE).
  • Paràmetres de Ruta: Variables que s'inclouen en la URL per identificar recursos específics.

Exemple de Rutes

GET /users          # Obtenir una llista d'usuaris
GET /users/{id}     # Obtenir un usuari específic per ID
POST /users         # Crear un nou usuari
PUT /users/{id}     # Actualitzar un usuari existent per ID
DELETE /users/{id}  # Eliminar un usuari per ID

Controladors

Els controladors són responsables de gestionar la lògica de negoci associada a cada ruta. Cada controlador generalment conté mètodes que corresponen a les accions CRUD (Crear, Llegir, Actualitzar, Eliminar).

Conceptes Clau dels Controladors

  • Mètode del Controlador: Una funció que gestiona una sol·licitud HTTP específica.
  • Separació de Responsabilitats: Mantenir la lògica de negoci separada de la definició de rutes per millorar la mantenibilitat del codi.

Exemple de Controlador en Node.js amb Express

const express = require('express');
const router = express.Router();

// Controlador per gestionar les sol·licituds d'usuaris
const userController = {
    getAllUsers: (req, res) => {
        // Lògica per obtenir tots els usuaris
        res.send('Llista d\'usuaris');
    },
    getUserById: (req, res) => {
        const userId = req.params.id;
        // Lògica per obtenir un usuari per ID
        res.send(`Usuari amb ID: ${userId}`);
    },
    createUser: (req, res) => {
        const newUser = req.body;
        // Lògica per crear un nou usuari
        res.send('Usuari creat');
    },
    updateUser: (req, res) => {
        const userId = req.params.id;
        const updatedUser = req.body;
        // Lògica per actualitzar un usuari existent
        res.send(`Usuari amb ID: ${userId} actualitzat`);
    },
    deleteUser: (req, res) => {
        const userId = req.params.id;
        // Lògica per eliminar un usuari
        res.send(`Usuari amb ID: ${userId} eliminat`);
    }
};

// Definició de rutes i assignació de controladors
router.get('/users', userController.getAllUsers);
router.get('/users/:id', userController.getUserById);
router.post('/users', userController.createUser);
router.put('/users/:id', userController.updateUser);
router.delete('/users/:id', userController.deleteUser);

module.exports = router;

Comparació de Mètodes HTTP

| Mètode HTTP | Descripció | Exemple d'Ús | |-------------|-------------------------------------|-----------------------------| | GET | Obtenir dades d'un servidor | GET /users | | POST | Enviar dades al servidor per crear | POST /users | | PUT | Actualitzar dades existents al servidor | PUT /users/{id} | | DELETE | Eliminar dades del servidor | DELETE /users/{id} |

Conclusió

La definició de rutes i controladors és essencial per al desenvolupament d'una REST API ben estructurada i mantenible. Les rutes determinen com s'accedeix als recursos, mentre que els controladors gestionen la lògica de negoci associada a aquestes rutes. Comprendre aquests conceptes i la seva implementació pràctica és crucial per a qualsevol desenvolupador d'APIs.

En el següent tema, aprofundirem en la validació de dades i gestió d'errors per assegurar que la nostra API sigui robusta i segura.
