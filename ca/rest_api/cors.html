Introducció a CORS

Cross-Origin Resource Sharing (CORS) és un mecanisme que utilitza capçaleres HTTP addicionals per permetre que un servidor indiqui qualsevol origen (domini, esquema o port) des del qual un navegador hauria de permetre la càrrega de recursos. CORS és una especificació del W3C que permet sol·licituds HTTP entre llocs web de diferents orígens.

Per què és important CORS?

  • Seguretat: CORS és crucial per a la seguretat web, ja que prevé atacs com Cross-Site Request Forgery (CSRF).
  • Interoperabilitat: Permet que les aplicacions web realitzin sol·licituds a dominis diferents al d'origen, facilitant la integració de serveis i APIs de tercers.

Funcionament de CORS

Capçaleres HTTP de CORS

  • Access-Control-Allow-Origin: Especifica els orígens permesos per accedir al recurs.
  • Access-Control-Allow-Methods: Indica els mètodes HTTP permesos (GET, POST, PUT, DELETE, etc.).
  • Access-Control-Allow-Headers: Llista de capçaleres permeses en la sol·licitud.
  • Access-Control-Allow-Credentials: Indica si les credencials (cookies, capçaleres d'autenticació) estan permeses.
  • Access-Control-Expose-Headers: Llista de capçaleres que poden ser exposades al client.
  • Access-Control-Max-Age: Temps en segons que el resultat d'una sol·licitud preflight pot ser emmagatzemat en memòria cau.

Sol·licituds Preflight

Les sol·licituds preflight són sol·licituds de verificació que el navegador envia abans de la sol·licitud real per assegurar-se que el servidor permetrà la sol·licitud d'origen creuat.

Exemple de sol·licitud preflight:

OPTIONS /api/data HTTP/1.1
Host: api.example.com
Origin: http://example.com
Access-Control-Request-Method: POST
Access-Control-Request-Headers: X-Custom-Header

Exemple de resposta preflight:

HTTP/1.1 200 OK
Access-Control-Allow-Origin: http://example.com
Access-Control-Allow-Methods: POST, GET, OPTIONS
Access-Control-Allow-Headers: X-Custom-Header
Access-Control-Max-Age: 86400

Implementació de CORS en una REST API

Exemple en Node.js amb Express

Per habilitar CORS en una aplicació Node.js usant Express, pots utilitzar el middleware cors.

  1. Instal·lar el paquet cors:
npm install cors
  1. Configurar CORS en la teva aplicació Express:
const express = require('express');
const cors = require('cors');
const app = express();

// Configuració bàsica de CORS
app.use(cors());

// Configuració avançada de CORS
const corsOptions = {
  origin: 'http://example.com',
  methods: 'GET,POST,PUT,DELETE',
  allowedHeaders: 'Content-Type,Authorization',
  credentials: true,
  optionsSuccessStatus: 200 // Per a navegadors antics
};

app.use(cors(corsOptions));

app.get('/api/data', (req, res) => {
  res.json({ message: 'CORS enabled!' });
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});

Exemple en Python amb Flask

Per habilitar CORS en una aplicació Flask, pots utilitzar l'extensió flask-cors.

  1. Instal·lar el paquet flask-cors:
pip install flask-cors
  1. Configurar CORS en la teva aplicació Flask:
from flask import Flask, jsonify
from flask_cors import CORS

app = Flask(__name__)
CORS(app)

@app.route('/api/data')
def data():
    return jsonify(message='CORS enabled!')

if __name__ == '__main__':
    app.run(port=3000)

Conclusió

CORS és una part essencial del desenvolupament d'aplicacions web modernes que interactuen amb APIs de diferents orígens. Comprendre com funciona i com implementar-lo correctament és crucial per garantir la seguretat i funcionalitat de les teves aplicacions. Assegura't de configurar adequadament les capçaleres CORS en el teu servidor per permetre només els orígens i mètodes necessaris, minimitzant així els riscos de seguretat.
