Introducció

En aquesta secció, abordarem la implementació d'una REST API, cobrint des de la configuració inicial fins a la creació d'endpoints funcionals. Utilitzarem exemples pràctics i explicacions detallades per assegurar que els conceptes siguin clars i aplicables.

Configuració Inicial

Per començar amb la implementació d'una REST API, és crucial configurar l'entorn de desenvolupament. En aquest curs, utilitzarem Node.js amb Express.js com a framework.

Instal·lació de Node.js i Express.js

  1. Instal·lar Node.js: Descarrega i instal·la Node.js des de nodejs.org.
  2. Crear un projecte nou: 
    mkdir my-rest-api
cd my-rest-api
npm init -y
  3. Instal·lar Express.js: 
    npm install express

Creació del Servidor Bàsic

Un cop tenim el nostre entorn configurat, el següent pas és crear un servidor bàsic que pugui manejar sol·licituds HTTP.

Codi del Servidor Bàsic

const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(port, () => {
  console.log(`Server is running on http://localhost:${port}`);
});
  • Explicació del Codi:
    • express: Importa el mòdul Express.
    • app: Crea una instància d'una aplicació Express.
    • app.get('/'): Defineix un endpoint GET a la ruta arrel ('/').
    • res.send('Hello World!'): Envia una resposta de text simple.
    • app.listen(port): Inicia el servidor al port especificat.

Creació d'Endpoints

Una REST API es compon de múltiples endpoints que permeten realitzar operacions CRUD (Crear, Llegir, Actualitzar, Eliminar).

Exemple d'Endpoints CRUD

Crear (POST)

app.post('/items', (req, res) => {
  // Lògica per crear un nou ítem
  res.status(201).send('Item created');
});

Llegir (GET)

app.get('/items', (req, res) => {
  // Lògica per obtenir tots els ítems
  res.status(200).json({ items: [] });
});

app.get('/items/:id', (req, res) => {
  const itemId = req.params.id;
  // Lògica per obtenir un ítem per ID
  res.status(200).json({ item: { id: itemId } });
});

Actualitzar (PUT)

app.put('/items/:id', (req, res) => {
  const itemId = req.params.id;
  // Lògica per actualitzar un ítem per ID
  res.status(200).send(`Item ${itemId} updated`);
});

Eliminar (DELETE)

app.delete('/items/:id', (req, res) => {
  const itemId = req.params.id;
  // Lògica per eliminar un ítem per ID
  res.status(200).send(`Item ${itemId} deleted`);
});

Maneig d'Errors

És fonamental manejar els errors de manera adequada per proporcionar respostes clares i útils als usuaris de l'API.

Exemple de Maneig d'Errors

app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something broke!');
});
  • Explicació del Codi:
    • app.use((err, req, res, next) => {...}): Middleware per manejar errors.
    • console.error(err.stack): Imprimeix el stack trace de l'error a la consola.
    • res.status(500).send('Something broke!'): Envia una resposta d'error 500 al client.

Conclusió

En aquesta secció, hem cobert els passos essencials per implementar una REST API utilitzant Node.js i Express.js. Des de la configuració inicial fins a la creació d'endpoints CRUD i el maneig d'errors, aquests fonaments et permetran construir APIs robustes i escalables.

Recorda que la pràctica és clau per dominar aquests conceptes. Et recomanem que experimentis amb diferents configuracions i casos d'ús per enfortir la teva comprensió i habilitats.
