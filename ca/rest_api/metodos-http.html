Introducció

En el context de les REST APIs, els mètodes HTTP són fonamentals per definir les operacions que es poden realitzar sobre els recursos. Cada mètode HTTP té un propòsit específic i s'utilitza per realitzar diferents tipus d'accions en els recursos d'una API. Els mètodes més comuns són GET, POST, PUT, DELETE, PATCH, OPTIONS i HEAD.

Mètodes HTTP Comuns

GET

  • Propòsit: Recuperar dades d'un servidor.
  • Idempotent: Sí (múltiples sol·licituds no canvien l'estat del recurs).
  • Segur: Sí (no modifica el recurs).

Exemple d'ús:

GET /api/users/123 HTTP/1.1
Host: example.com

Explicació: Aquest exemple sol·licita informació sobre l'usuari amb ID 123.

POST

  • Propòsit: Enviar dades al servidor per crear un nou recurs.
  • Idempotent: No (múltiples sol·licituds poden crear múltiples recursos).
  • Segur: No (modifica l'estat del servidor).

Exemple d'ús:

POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john.doe@example.com"
}

Explicació: Aquest exemple crea un nou usuari amb el nom "John Doe" i el correu "john.doe@example.com".

PUT

  • Propòsit: Actualitzar un recurs existent o crear-ne un si no existeix.
  • Idempotent: Sí (múltiples sol·licituds tenen el mateix efecte que una sola).
  • Segur: No (modifica l'estat del servidor).

Exemple d'ús:

PUT /api/users/123 HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john.doe@example.com"
}

Explicació: Aquest exemple actualitza la informació de l'usuari amb ID 123.

DELETE

  • Propòsit: Eliminar un recurs existent.
  • Idempotent: Sí (múltiples sol·licituds tenen el mateix efecte que una sola).
  • Segur: No (modifica l'estat del servidor).

Exemple d'ús:

DELETE /api/users/123 HTTP/1.1
Host: example.com

Explicació: Aquest exemple elimina l'usuari amb ID 123.

PATCH

  • Propòsit: Aplicar modificacions parcials a un recurs.
  • Idempotent: No necessàriament (depèn de la implementació).
  • Segur: No (modifica l'estat del servidor).

Exemple d'ús:

PATCH /api/users/123 HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "email": "new.email@example.com"
}

Explicació: Aquest exemple actualitza només el correu de l'usuari amb ID 123.

OPTIONS

  • Propòsit: Descriure les opcions de comunicació per al recurs de destinació.
  • Idempotent: Sí.
  • Segur: Sí.

Exemple d'ús:

OPTIONS /api/users HTTP/1.1
Host: example.com

Explicació: Aquest exemple sol·licita les opcions de comunicació disponibles per al recurs /api/users.

HEAD

  • Propòsit: Recuperar els encapçalaments de resposta idèntics als d'una sol·licitud GET, però sense el cos de la resposta.
  • Idempotent: Sí.
  • Segur: Sí.

Exemple d'ús:

HEAD /api/users/123 HTTP/1.1
Host: example.com

Explicació: Aquest exemple sol·licita els encapçalaments de la resposta per a l'usuari amb ID 123 sense el cos de la resposta.

Comparació de Mètodes HTTP

| Mètode | Propòsit | Idempotent | Segur | |---------|-----------------------------------|------------|-------| | GET | Recuperar dades | Sí | Sí | | POST | Crear un nou recurs | No | No | | PUT | Actualitzar o crear un recurs | Sí | No | | DELETE | Eliminar un recurs | Sí | No | | PATCH | Modificar parcialment un recurs | No | No | | OPTIONS | Descriure opcions de comunicació | Sí | Sí | | HEAD | Recuperar encapçalaments | Sí | Sí |

Conclusió

Comprendre els mètodes HTTP és essencial per treballar amb REST APIs de manera efectiva. Cada mètode té un propòsit específic i s'utilitza en diferents situacions per interactuar amb els recursos d'una API. Assegura't d'utilitzar el mètode adequat per a cada operació per mantenir la integritat i l'eficiència de la teva API.
