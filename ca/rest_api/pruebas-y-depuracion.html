Introducció

En aquesta secció, aprendràs com realitzar proves i depuració d'una REST API. Les proves i la depuració són essencials per garantir que la teva API funcioni correctament i sigui robusta davant errors i excepcions.

Tipus de proves

Proves Unitàries

Les proves unitàries se centren a provar components individuals de l'API, com funcions o mètodes. Aquestes proves són crucials per assegurar que cada part de la teva API funcioni com s'espera.

  • Objectiu: Verificar que cada unitat de codi funcioni correctament.
  • Eines comunes: Jest, Mocha, Chai (per a Node.js), JUnit (per a Java).
// Exemple de prova unitària amb Jest
const request = require('supertest');
const app = require('../app');

describe('GET /api/resource', () => {
  it('should return a list of resources', async () => {
    const res = await request(app).get('/api/resource');
    expect(res.statusCode).toEqual(200);
    expect(res.body).toHaveProperty('data');
  });
});

Proves d'Integració

Les proves d'integració verifiquen que diferents mòduls o serveis de la teva API funcionin junts correctament.

  • Objectiu: Assegurar que els components integrats funcionin com un tot.
  • Eines comunes: Supertest, Postman, Newman.
// Exemple de prova d'integració amb Supertest
const request = require('supertest');
const app = require('../app');

describe('POST /api/resource', () => {
  it('should create a new resource', async () => {
    const res = await request(app)
      .post('/api/resource')
      .send({
        name: 'New Resource',
        description: 'This is a new resource'
      });
    expect(res.statusCode).toEqual(201);
    expect(res.body).toHaveProperty('id');
  });
});

Proves Funcionals

Les proves funcionals se centren a verificar que l'API compleixi amb els requisits funcionals especificats.

  • Objectiu: Validar que l'API faci el que se suposa que ha de fer.
  • Eines comunes: Postman, Newman, SoapUI.
// Exemple de prova funcional amb Postman
{
  "info": {
    "name": "API Functional Test",
    "schema": "https://schema.getpostman.com/json/collection/v2.1.0/collection.json"
  },
  "item": [
    {
      "name": "Get Resources",
      "request": {
        "method": "GET",
        "header": [],

          "raw": "http://localhost:3000/api/resource",
          "protocol": "http",
          "host": ["localhost"],
          "port": "3000",
          "path": ["api", "resource"]
        }
      },
      "response": []
    }
  ]
}

Eines de depuració

Logs

Els logs són una eina fonamental per a la depuració. Permeten registrar esdeveniments i errors que ocorren durant l'execució de l'API.

  • Llibreries comunes: Winston, Morgan (per a Node.js), Log4j (per a Java).
// Exemple d'ús de Winston per a logging
const winston = require('winston');

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' })
  ]
});

logger.info('This is an informational message');
logger.error('This is an error message');

Depuradors

Els depuradors permeten pausar l'execució del codi i examinar l'estat de l'aplicació en temps real.

  • Eines comunes: Node.js Debugger, Chrome DevTools, Visual Studio Code Debugger.
// Exemple d'ús del depurador en Node.js
const http = require('http');

const server = http.createServer((req, res) => {
  debugger; // Punt d'interrupció
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(3000, '127.0.0.1', () => {
  console.log('Server running at http://127.0.0.1:3000/');
});

Exercicis pràctics

  1. Prova Unitària: Escriu una prova unitària per a una funció que sumi dos números.
  2. Prova d'Integració: Crea una prova d'integració per a un endpoint que creï un nou usuari.
  3. Prova Funcional: Usa Postman per crear una col·lecció de proves funcionals per a la teva API.
  4. Depuració: Afegeix logs a la teva API i usa un depurador per identificar i corregir un error en un endpoint específic.

Conclusió

Les proves i la depuració són components essencials en el desenvolupament d'una REST API robusta i fiable. En implementar proves unitàries, d'integració i funcionals, i en utilitzar eines de depuració efectives, pots assegurar que la teva API funcioni correctament i estigui preparada per manejar errors i excepcions.
