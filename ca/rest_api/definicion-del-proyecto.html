Introducció

En aquesta secció, definirem el projecte d'una REST API que desenvoluparem al llarg del curs. Aquest projecte servirà com a exemple pràctic per entendre els conceptes i tècniques necessàries per construir una API RESTful robusta i eficient.

Objectius del Projecte

  • Comprendre els fonaments de les API REST: Aprendrem què és una API REST, els seus principis i com es diferencia d'altres tipus d'APIs.
  • Dissenyar una API RESTful: Cobrirem les millors pràctiques per dissenyar una API que sigui fàcil d'usar i mantenir.
  • Implementar la API: Utilitzarem un llenguatge de programació i un framework específic per construir la nostra API.
  • Provar i Documentar la API: Aprendrem a provar la nostra API per assegurar-nos que funciona correctament i a documentar-la perquè altres desenvolupadors puguin usar-la fàcilment.

Descripció del Projecte

El projecte consisteix a desenvolupar una API RESTful per a una aplicació de gestió de tasques. La API permetrà als usuaris:

  • Crear tasques: Els usuaris podran afegir noves tasques a la seva llista.
  • Llegir tasques: Els usuaris podran veure totes les seves tasques o una tasca específica.
  • Actualitzar tasques: Els usuaris podran modificar els detalls d'una tasca existent.
  • Eliminar tasques: Els usuaris podran eliminar tasques que ja no necessitin.

Requisits del Projecte

Funcionals

  • CRUD de Tasques: La API ha de suportar operacions de Crear, Llegir, Actualitzar i Eliminar (CRUD) per a les tasques.
  • Autenticació i Autorització: La API ha d'assegurar que només els usuaris autenticats puguin accedir a les seves tasques.
  • Validació de Dades: La API ha de validar les dades d'entrada per assegurar que siguin correctes i completes.

No Funcionals

  • Escalabilitat: La API ha de ser capaç de manejar un gran nombre de sol·licituds concurrents.
  • Mantenibilitat: El codi ha de ser fàcil d'entendre i mantenir.
  • Documentació: La API ha d'estar ben documentada per facilitar el seu ús per altres desenvolupadors.

Estructura de la API

Endpoints

| Mètode HTTP | Endpoint | Descripció | |-------------|-----------------|---------------------------------------| | POST | /tasks | Crear una nova tasca | | GET | /tasks | Obtenir totes les tasques | | GET | /tasks/{id} | Obtenir una tasca específica per ID | | PUT | /tasks/{id} | Actualitzar una tasca específica per ID| | DELETE | /tasks/{id} | Eliminar una tasca específica per ID |

Exemple de Codi

A continuació, es mostra un exemple de com podria veure's l'endpoint per crear una nova tasca utilitzant Node.js i Express:

const express = require('express');
const app = express();
app.use(express.json());

let tasks = [];

app.post('/tasks', (req, res) => {
    const task = {
        id: tasks.length + 1,
        title: req.body.title,
        description: req.body.description,
        completed: false
    };
    tasks.push(task);
    res.status(201).send(task);
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});

En aquest exemple, hem definit un endpoint /tasks que permet als usuaris crear una nova tasca. La tasca s'afegeix a un array tasks i es retorna al client amb un codi d'estat 201 (Created).

Conclusió

En aquesta secció, hem definit el projecte d'una API RESTful per a una aplicació de gestió de tasques. Hem cobert els objectius del projecte, els requisits funcionals i no funcionals, i l'estructura bàsica de la API. A mesura que avancem en el curs, aprofundirem en cadascun d'aquests aspectes i construirem una API completa i funcional.
