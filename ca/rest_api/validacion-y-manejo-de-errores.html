Introducció

En aquest tema, aprendrem sobre la importància de la validació i el maneig d'errors en el desenvolupament de REST APIs. Aquests aspectes són crucials per garantir la robustesa, seguretat i usabilitat de les APIs. Cobrirem conceptes clau, pràctiques recomanades i exemples d'implementació.

Validació de Dades

Què és la validació de dades?

La validació de dades és el procés d'assegurar que les dades enviades a l'API compleixen amb els requisits esperats abans de ser processades. Això inclou verificar tipus de dades, formats, rangs i altres restriccions.

Importància de la validació

  • Seguretat: Prevé atacs com la injecció de SQL.
  • Integritat de dades: Assegura que les dades emmagatzemades siguin correctes i consistents.
  • Experiència de l'usuari: Proporciona retroalimentació immediata sobre errors en les dades enviades.

Tipus de validació

  • Validació del costat del client: Realitzada al navegador abans d'enviar les dades al servidor.
  • Validació del costat del servidor: Realitzada al servidor per assegurar que les dades rebudes són vàlides.

Exemple de validació en Node.js amb Express

const express = require('express');
const app = express();
app.use(express.json());

app.post('/api/users', (req, res) => {
    const { name, email, age } = req.body;

    // Validació bàsica
    if (!name || !email || !age) {
        return res.status(400).json({ error: 'Tots els camps són obligatoris' });
    }

    // Validació de tipus de dades
    if (typeof name !== 'string' || typeof email !== 'string' || typeof age !== 'number') {
        return res.status(400).json({ error: 'Tipus de dades incorrectes' });
    }

    // Validació de format de correu electrònic
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (!emailRegex.test(email)) {
        return res.status(400).json({ error: 'Format de correu electrònic invàlid' });
    }

    res.status(201).json({ message: 'Usuari creat exitosament' });
});

app.listen(3000, () => console.log('Server running on port 3000'));

Maneig d'Errors

Què és el maneig d'errors?

El maneig d'errors és el procés de capturar, gestionar i respondre a errors que ocorren durant l'execució d'una API. Això inclou errors de validació, errors del servidor i errors de xarxa.

Importància del maneig d'errors

  • Robustesa: Assegura que l'API pot gestionar situacions inesperades sense fallar.
  • Seguretat: Evita l'exposició d'informació sensible.
  • Usabilitat: Proporciona missatges d'error clars i útils als usuaris.

Tipus d'errors comuns

  • Errors de client (4xx): Errors causats per sol·licituds incorrectes del client.
  • Errors del servidor (5xx): Errors causats per problemes al servidor.

Exemple de maneig d'errors en Node.js amb Express

const express = require('express');
const app = express();
app.use(express.json());

app.get('/api/resource', (req, res, next) => {
    try {
        // Simulació d'un error del servidor
        throw new Error('Error del servidor');
    } catch (error) {
        next(error);
    }
});

// Middleware de maneig d'errors
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).json({ error: 'Va ocórrer un error al servidor' });
});

app.listen(3000, () => console.log('Server running on port 3000'));

Pràctiques recomanades

Validació

  • Usar llibreries de validació: Llibreries com Joi, Yup o Validator.js poden simplificar la validació.
  • Validar tant al client com al servidor: Encara que la validació del costat del client millora l'experiència de l'usuari, la validació del costat del servidor és essencial per a la seguretat.

Maneig d'errors

  • Usar codis d'estat HTTP apropiats: Assegura't de retornar el codi d'estat HTTP correcte per a cada tipus d'error.
  • No exposar detalls interns: Evita incloure informació sensible en els missatges d'error.
  • Registrar errors: Implementa un sistema de logging per registrar i monitoritzar errors.

Conclusió

La validació i el maneig d'errors són components essencials en el desenvolupament de REST APIs. Implementar una validació robusta i un maneig d'errors adequat no només millora la seguretat i la integritat de les dades, sinó que també proporciona una millor experiència a l'usuari. Assegura't de seguir les pràctiques recomanades i utilitzar eines adequades per simplificar aquests processos.
