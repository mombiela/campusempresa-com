Recursos i URIs
Mètodes HTTP ►

En aquest tema, explorarem els conceptes fonamentals de recursos i URIs (Uniform Resource Identifiers) en el context de les APIs REST. Comprendre aquests conceptes és crucial per dissenyar i consumir APIs de manera efectiva.

Què és un Recurs?

En REST, un recurs és qualsevol tipus d'objecte, dada o servei que pot ser accedit i manipulat a través de l'API. Els recursos són les entitats principals que s'exposen a través de l'API i s'identifiquen mitjançant URIs.

  • Exemples de recursos:
    • Usuaris
    • Productes
    • Comandes
    • Comentaris

Cada recurs té una representació que pot ser en format JSON, XML, HTML, etc.

Què és un URI?

Un URI (Uniform Resource Identifier) és una cadena de caràcters que identifica de manera única un recurs a la web. En el context de REST, els URIs són utilitzats per accedir als recursos.

  • Components d'un URI:
    • Esquema: Indica el protocol a utilitzar (e.g., http, https).
    • Host: El domini on resideix el recurs (e.g., api.exemple.com).
    • Ruta: La ubicació específica del recurs (e.g., /usuaris/123).
    • Query Parameters: Paràmetres addicionals per filtrar o modificar la sol·licitud (e.g., ?page=2&limit=10).

Exemple de URI

https://api.exemple.com/usuaris/123?include=detalls
  • Esquema: https
  • Host: api.exemple.com
  • Ruta: /usuaris/123
  • Query Parameters: include=detalls

Disseny de URIs

El disseny de URIs és fonamental per a la claredat i la usabilitat d'una API. Aquí hi ha algunes millors pràctiques:

  • Utilitzar substantius en plural: Els URIs han de representar col·leccions de recursos.
    • Correcte: /usuaris
    • Incorrecte: /usuari
  • Utilitzar jerarquies clares: Organitzar els recursos de manera jeràrquica.
    • Correcte: /usuaris/123/comandes
    • Incorrecte: /comandes/usuari/123
  • Evitar verbs: Els URIs han de centrar-se en els recursos, no en les accions.
    • Correcte: /productes/456
    • Incorrecte: /obtenirProducte/456
  • Utilitzar guions en lloc de guions baixos: Els guions són més llegibles en URIs.
    • Correcte: /productes-nous
    • Incorrecte: /productes_nous

Exemple de Codi

A continuació, es mostra un exemple de com es pot consumir una API REST utilitzant JavaScript i la funció fetch per obtenir un recurs específic.

// URL del recurs
const url = 'https://api.exemple.com/usuaris/123';

// Realitzar una sol·licitud GET per obtenir el recurs
fetch(url)
  .then(response => {
    if (!response.ok) {
      throw new Error('Error en la sol·licitud');
    }
    return response.json();
  })
  .then(data => {
    console.log('Dades de l'usuari:', data);
  })
  .catch(error => {
    console.error('Error:', error);
  });

Conclusió

Comprendre els conceptes de recursos i URIs és essencial per treballar amb APIs REST. Els recursos són les entitats que es manipulen a través de l'API, i els URIs són les adreces que permeten accedir a aquests recursos. Un bon disseny de URIs millora la claredat i la usabilitat de l'API, facilitant el seu consum per part dels desenvolupadors.

En el proper tema, aprofundirem en els mètodes HTTP i com s'utilitzen per interactuar amb els recursos en una API REST.
