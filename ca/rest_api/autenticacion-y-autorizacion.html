Introducció

En el desenvolupament de REST APIs, la seguretat és un aspecte crucial. Dos conceptes fonamentals per garantir la seguretat són l'autenticació i l'autorització. Encara que sovint s'utilitzen de manera intercanviable, tenen significats diferents:

  • Autenticació: Procés de verificar la identitat d'un usuari o sistema.
  • Autorització: Procés de determinar si un usuari o sistema autenticat té permís per realitzar una acció específica.

Autenticació

L'autenticació és el primer pas per assegurar una API. Hi ha diversos mètodes per autenticar usuaris en una REST API:

Mètodes d'Autenticació

  1. Autenticació Bàsica (Basic Authentication)

    • Envia el nom d'usuari i la contrasenya en cada sol·licitud HTTP.
    • Codifica les credencials en Base64.
    • No és segura per si sola; ha d'utilitzar-se amb HTTPS.
    GET /api/resource HTTP/1.1
Host: example.com
Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=

  2. Tokens d'API (API Tokens)

    • Utilitza un token únic per a cada usuari.
    • El token s'envia en la capçalera de la sol·licitud.
    GET /api/resource HTTP/1.1
Host: example.com
Authorization: Bearer your_api_token

  3. OAuth 2.0

    • Protocol d'autorització que permet a les aplicacions obtenir accés limitat als recursos d'usuari.
    • Utilitza tokens d'accés i tokens d'actualització.
    GET /api/resource HTTP/1.1
Host: example.com
Authorization: Bearer your_oauth_token

  4. JWT (JSON Web Tokens)

    • Token compacte i autònom que s'utilitza per transmetre informació entre dues parts.
    • Inclou una signatura per verificar la integritat i autenticitat del token.
    GET /api/resource HTTP/1.1
Host: example.com
Authorization: Bearer your_jwt_token

Autorització

Un cop un usuari està autenticat, el següent pas és l'autorització. Això implica verificar si l'usuari té permisos per accedir a certs recursos o realitzar certes accions.

Mètodes d'Autorització

  1. Rols i Permisos

    • Assigna rols als usuaris (per exemple, administrador, usuari, convidat).
    • Defineix permisos específics per a cada rol.
    {
  "user": {
    "role": "admin",
    "permissions": ["read", "write", "delete"]
  }
}

  2. Llistes de Control d'Accés (ACL)

    • Defineix permisos a nivell de recurs.
    • Cada recurs té una llista d'usuaris i els seus permisos associats.
    {
  "resource": "file123",
  "acl": [
    {"user": "user1", "permissions": ["read"]},
    {"user": "user2", "permissions": ["read", "write"]}
  ]
}

  3. Polítiques Basades en Atributs (ABAC)

    • Utilitza atributs de l'usuari, del recurs i de l'entorn per prendre decisions d'autorització.
    • Més flexible i granular que els mètodes basats en rols.
    {
  "user": {
    "attributes": {"department": "sales"}
  },
  "resource": {
    "attributes": {"type": "report"}
  },
  "environment": {
    "attributes": {"time": "business_hours"}
  },
  "policy": "allow if user.department == 'sales' and resource.type == 'report' and environment.time == 'business_hours'"
}

Exemples d'Implementació

Exemple d'Autenticació amb JWT

  1. Generació del Token

    import jwt
import datetime

def generate_token(user_id):
    payload = {
        'user_id': user_id,
        'exp': datetime.datetime.utcnow() + datetime.timedelta(hours=1)
    }
    token = jwt.encode(payload, 'secret_key', algorithm='HS256')
    return token

  2. Verificació del Token

    def verify_token(token):
    try:
        payload = jwt.decode(token, 'secret_key', algorithms=['HS256'])
        return payload['user_id']
    except jwt.ExpiredSignatureError:
        return 'Token has expired'
    except jwt.InvalidTokenError:
        return 'Invalid token'

Exemple d'Autorització Basada en Rols

  1. Definició de Rols i Permisos

    roles_permissions = {
    'admin': ['read', 'write', 'delete'],
    'user': ['read', 'write'],
    'guest': ['read']
}

  2. Verificació de Permisos

    def check_permission(user_role, permission):
    if permission in roles_permissions.get(user_role, []):
        return True
    return False

Conclusió

L'autenticació i l'autorització són components essencials per a la seguretat d'una REST API. L'autenticació assegura que només els usuaris legítims puguin accedir a l'API, mentre que l'autorització garanteix que aquests usuaris només puguin realitzar accions per a les quals tenen permís. En implementar aquests mecanismes, és crucial triar el mètode adequat segons les necessitats i el context de l'aplicació, i sempre utilitzar HTTPS per protegir les credencials i els tokens en trànsit.
