En aquest tema, explorarem les estratègies i millors pràctiques per protegir les teves REST APIs contra atacs comuns. És fonamental per a qualsevol professional de TI assegurar-se que les seves APIs siguin segures i resistents a les amenaces.

Tipus d'atacs comuns

A continuació, es descriuen alguns dels atacs més comuns que poden afectar les REST APIs:

  • Injecció SQL: Ocorre quan un atacant insereix codi SQL maliciós en una consulta.
  • Cross-Site Scripting (XSS): Permet als atacants injectar scripts en les pàgines web vistes per altres usuaris.
  • Cross-Site Request Forgery (CSRF): Enganya els usuaris perquè executin accions no desitjades en una aplicació en la qual estan autenticats.
  • Atacs de força bruta: Intentos repetits d'endevinar contrasenyes o tokens d'autenticació.
  • Denegació de Servei (DoS): Sobrecàrrega el servidor amb sol·licituds per fer-lo inaccessible.

Estratègies de protecció

Validació i sanitització d'entrades

  • Validació d'entrades: Assegura't que totes les dades rebudes per l'API siguin vàlides i estiguin en el format esperat.
  • Sanitització d'entrades: Neteja les dades rebudes per eliminar qualsevol contingut maliciós.
from flask import request, jsonify
from werkzeug.security import safe_str_cmp

@app.route('/api/data', methods=['POST'])
def process_data():
    data = request.json
    if not isinstance(data, dict):
        return jsonify({"error": "Invalid input"}), 400
    # Sanitització
    sanitized_data = {k: str(v).replace("<", "&lt;").replace(">", "&gt;") for k, v in data.items()}
    # Processar dades
    return jsonify(sanitized_data), 200

Ús de tokens i autenticació

  • Tokens JWT: Utilitza JSON Web Tokens (JWT) per autenticar i autoritzar usuaris.
  • Expiració de tokens: Assegura't que els tokens tinguin un temps d'expiració per minimitzar el risc d'ús indegut.
import jwt
import datetime

def generate_token(user_id):
    payload = {
        'user_id': user_id,
        'exp': datetime.datetime.utcnow() + datetime.timedelta(hours=1)
    }
    token = jwt.encode(payload, 'secret_key', algorithm='HS256')
    return token

Protecció contra CSRF

  • Tokens CSRF: Utilitza tokens CSRF per verificar que les sol·licituds provenen d'usuaris autenticats.
  • Capçaleres personalitzades: Requereix que les sol·licituds incloguin capçaleres específiques que els navegadors no envien per defecte.
from flask_wtf.csrf import CSRFProtect

csrf = CSRFProtect(app)

@app.route('/api/secure', methods=['POST'])
@csrf.exempt
def secure_endpoint():
    # Processar sol·licitud segura
    return jsonify({"message": "Secure data"}), 200

Limitació de taxa i monitoratge

  • Limitació de taxa: Implementa límits de taxa per prevenir atacs de força bruta i DoS.
  • Monitoratge i alertes: Configura sistemes de monitoratge per detectar i alertar sobre activitats sospitoses.
from flask_limiter import Limiter

limiter = Limiter(app, key_func=get_remote_address)

@app.route('/api/limited', methods=['GET'])
@limiter.limit("5 per minute")
def limited_endpoint():
    return jsonify({"message": "This is a rate-limited endpoint"}), 200

Conclusió

Protegir les teves REST APIs contra atacs comuns és essencial per mantenir la integritat i seguretat de les teves aplicacions. En implementar validació i sanitització d'entrades, usar tokens i autenticació, protegir contra CSRF, i aplicar limitació de taxa i monitoratge, pots reduir significativament el risc d'atacs. Assegura't de mantenir-te actualitzat amb les millors pràctiques de seguretat i revisar regularment els teus sistemes per detectar possibles vulnerabilitats.
