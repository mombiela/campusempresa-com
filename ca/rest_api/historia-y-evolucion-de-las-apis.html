Introducció a les APIs

Les APIs (Application Programming Interfaces) són un conjunt de regles i protocols que permeten que diferents aplicacions es comuniquin entre si. Són fonamentals en el desenvolupament de programari modern, facilitant la integració i la interoperabilitat entre sistemes.

Primers dies de les APIs

  • Dècada de 1960-1970: Les primeres formes d'APIs van sorgir amb la necessitat de compartir dades entre diferents sistemes informàtics. Aquestes primeres APIs eren principalment biblioteques i funcions que permetien la comunicació entre diferents parts d'un mateix programa.
  • Dècada de 1980: Amb l'auge de les xarxes i la computació distribuïda, les APIs van començar a evolucionar per permetre la comunicació entre diferents sistemes. Els protocols com RPC (Remote Procedure Call) i CORBA (Common Object Request Broker Architecture) van ser alguns dels primers intents d'estandarditzar aquestes comunicacions.

Evolució cap a les APIs web

  • Dècada de 1990: L'arribada de la web va portar amb si la necessitat de noves formes de comunicació entre aplicacions. Les APIs basades en HTTP van començar a guanyar popularitat. SOAP (Simple Object Access Protocol) va ser un dels primers estàndards àmpliament adoptats per a APIs web.
  • Dècada de 2000: La simplicitat i flexibilitat de REST (Representational State Transfer) van començar a guanyar tracció. REST es basa en els principis de la web i utilitza HTTP per realitzar operacions CRUD (Create, Read, Update, Delete) sobre recursos.

REST APIs

  • Principis de REST:

    • Stateless: Cada sol·licitud del client al servidor ha de contenir tota la informació necessària per entendre i processar la sol·licitud.
    • Client-Server: L'arquitectura REST separa les preocupacions del client i del servidor.
    • Cacheable: Les respostes han de ser explícitament marcades com a cacheables o no cacheables.
    • Uniform Interface: Una interfície uniforme simplifica i desacobla l'arquitectura, permetent que cada part evolucioni independentment.

  • Components clau de REST:

    • Recursos: Representats per URLs.
    • Mètodes HTTP: GET, POST, PUT, DELETE, etc.
    • Representacions: JSON, XML, etc.

Exemple d'una REST API

A continuació, es mostra un exemple bàsic d'una REST API per gestionar una llista de tasques:

// GET /tasks
[
  {
    "id": 1,
    "title": "Aprendre sobre REST APIs",
    "completed": false
  },
  {
    "id": 2,
    "title": "Escriure un exemple de codi",
    "completed": true
  }
]

// POST /tasks
{
  "title": "Nova tasca",
  "completed": false
}

// PUT /tasks/1
{
  "title": "Aprendre sobre REST APIs",
  "completed": true
}

// DELETE /tasks/1
// No es necessita cos en la sol·licitud

Comparació entre SOAP i REST

| Característica | SOAP | REST | |----------------|------|------| | Protocol | Basat en XML | Basat en HTTP | | Estat | Stateful o Stateless | Stateless | | Complexitat | Alta (requereix WS-* standards) | Baixa (simple ús d'HTTP) | | Flexibilitat | Menor (estrictament definit) | Major (més flexible) | | Performance | Menor (més pesat) | Major (més lleuger) |

Conclusió

Les APIs han evolucionat significativament des dels seus inicis, adaptant-se a les necessitats canviants de la tecnologia i la indústria. Des de les primeres biblioteques i funcions fins a les modernes REST APIs, han jugat un paper crucial en la integració i la interoperabilitat de sistemes. REST, en particular, ha demostrat ser una solució simple i efectiva per a la comunicació entre aplicacions a la web, i continua sent una elecció popular per al desenvolupament d'APIs avui en dia.
