Introducció

En aquest curs, aprendrem a dissenyar, desenvolupar i consumir una REST API. La API que construirem permetrà gestionar una col·lecció de llibres en una biblioteca digital. Aquest projecte ens permetrà explorar els conceptes fonamentals de REST, així com les millors pràctiques per construir APIs robustes i escalables.

Objectius del Projecte

  • Comprendre els principis de REST: Aprendrem què és REST i per què és important en el desenvolupament d'APIs.
  • Dissenyar una REST API: Definirem els endpoints, mètodes HTTP i estructures de dades necessàries per a la nostra API.
  • Implementar la API: Utilitzarem un framework de desenvolupament (com Flask per a Python o Express per a Node.js) per construir la nostra API.
  • Provar la API: Verificarem que la nostra API funciona correctament utilitzant eines com Postman o Insomnia.
  • Documentar la API: Crearem documentació clara i útil perquè altres desenvolupadors puguin utilitzar la nostra API.

Estructura del Projecte

Endpoints

La nostra API tindrà els següents endpoints:

| Mètode HTTP | Endpoint | Descripció | |-------------|-------------------|-----------------------------------| | GET | /books | Obtenir una llista de tots els llibres | | POST | /books | Afegir un nou llibre | | GET | /books/{id} | Obtenir detalls d'un llibre específic | | PUT | /books/{id} | Actualitzar un llibre específic | | DELETE | /books/{id} | Eliminar un llibre específic |

Exemple de Codi

A continuació, es mostra un exemple de com podríem definir l'endpoint per obtenir tots els llibres utilitzant Flask en Python:

from flask import Flask, jsonify

app = Flask(__name__)

# Dades d'exemple
books = [
    {"id": 1, "title": "1984", "author": "George Orwell"},
    {"id": 2, "title": "To Kill a Mockingbird", "author": "Harper Lee"}
]

@app.route('/books', methods=['GET'])
def get_books():
    return jsonify(books)

if __name__ == '__main__':
    app.run(debug=True)

Proves

Per provar la nostra API, utilitzarem Postman. Aquí hi ha un exemple de com provar l'endpoint GET /books:

  1. Obrir Postman.
  2. Crear una nova sol·licitud.
  3. Seleccionar el mètode GET.
  4. Introduir la URL http://localhost:5000/books.
  5. Fer clic a "Send".
  6. Verificar que la resposta contingui la llista de llibres.

Documentació

La documentació és crucial perquè altres desenvolupadors puguin entendre i utilitzar la nostra API. Utilitzarem Swagger per generar documentació automàtica. Aquí hi ha un exemple de com integrar Swagger en la nostra aplicació Flask:

from flask_swagger_ui import get_swaggerui_blueprint

SWAGGER_URL = '/swagger'
API_URL = '/static/swagger.json'

swaggerui_blueprint = get_swaggerui_blueprint(
    SWAGGER_URL,
    API_URL,
    config={
        'app_name': "Library API"
    }
)

app.register_blueprint(swaggerui_blueprint, url_prefix=SWAGGER_URL)

Conclusió

En aquest projecte, hem cobert els aspectes fonamentals del disseny, implementació, prova i documentació d'una REST API. Al llarg del curs, aprofundirem en cadascun d'aquests temes, proporcionant exemples pràctics i exercicis per reforçar l'aprenentatge. Al final del curs, estaràs equipat amb les habilitats necessàries per construir i mantenir les teves pròpies APIs RESTful.
