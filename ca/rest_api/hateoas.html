Introducció a HATEOAS

HATEOAS (Hypermedia As The Engine Of Application State) és un component clau de l'estil arquitectònic REST (Representational State Transfer). Permet que les interaccions amb una API siguin auto-descriptives, proporcionant enllaços (hypermedia) dins de les respostes que guien els clients sobre les possibles accions següents.

Conceptes Clau de HATEOAS

  • Hypermedia: És qualsevol contingut que conté enllaços a altres recursos. En el context de HATEOAS, aquests enllaços permeten navegar a través de l'API.
  • Auto-descriptiu: Les respostes de l'API contenen tota la informació necessària perquè el client sàpiga quines accions pot realitzar a continuació.
  • Desacoblament: Els clients no necessiten conèixer l'estructura interna de l'API, ja que poden descobrir els recursos i les accions disponibles a través dels enllaços proporcionats.

Implementació de HATEOAS

Estructura d'una Resposta HATEOAS

Una resposta HATEOAS típica inclou enllaços que descriuen les accions possibles. Aquí hi ha un exemple de com podria veure's una resposta JSON que segueix el principi HATEOAS:

{
  "id": 1,
  "name": "John Doe",
  "email": "john.doe@example.com",
  "links": [
    {
      "rel": "self",
      "href": "/users/1"
    },
    {
      "rel": "update",
      "href": "/users/1/update"
    },
    {
      "rel": "delete",
      "href": "/users/1/delete"
    }
  ]
}

Explicació de l'Exemple

  • rel: Descriu la relació de l'enllaç amb el recurs actual. En aquest cas, self indica l'enllaç al propi recurs, update indica l'enllaç per actualitzar el recurs i delete per eliminar-lo.
  • href: És la URL del recurs o acció corresponent.

Exemple d'Implementació en un Servidor RESTful

Suposem que estem utilitzant un framework com Spring Boot en Java per implementar una API RESTful. Aquí hi ha un exemple de com podríem afegir enllaços HATEOAS a una resposta:

import org.springframework.hateoas.EntityModel;
import org.springframework.hateoas.server.mvc.WebMvcLinkBuilder;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class UserController {

    @GetMapping("/users/{id}")
    public EntityModel<User> getUser(@PathVariable Long id) {
        User user = userService.findById(id);
        EntityModel<User> resource = EntityModel.of(user);
        
        resource.add(WebMvcLinkBuilder.linkTo(
            WebMvcLinkBuilder.methodOn(UserController.class).getUser(id)).withSelfRel());
        resource.add(WebMvcLinkBuilder.linkTo(
            WebMvcLinkBuilder.methodOn(UserController.class).updateUser(id)).withRel("update"));
        resource.add(WebMvcLinkBuilder.linkTo(
            WebMvcLinkBuilder.methodOn(UserController.class).deleteUser(id)).withRel("delete"));
        
        return resource;
    }
}

Explicació del Codi

  • EntityModel: És una classe de Spring HATEOAS que embolcalla el recurs i afegeix enllaços.
  • WebMvcLinkBuilder: Utilitzat per construir enllaços als mètodes del controlador.
  • withSelfRel: Afegeix un enllaç amb la relació self.
  • withRel: Afegeix un enllaç amb una relació específica, com update o delete.

Beneficis d'Usar HATEOAS

  • Navegabilitat: Els clients poden descobrir recursos i accions disponibles sense necessitat de documentació externa.
  • Desacoblament: Els canvis en l'API no afecten els clients, sempre que els enllaços continuïn sent vàlids.
  • Auto-descriptiu: Les respostes contenen tota la informació necessària per interactuar amb l'API.

Conclusió

HATEOAS és una pràctica poderosa en el disseny d'APIs RESTful que millora la navegabilitat i el desacoblament entre el client i el servidor. En proporcionar enllaços dins de les respostes, permet que els clients descobreixin i realitzin accions sense necessitat de conèixer l'estructura interna de l'API. Implementar HATEOAS pot semblar complex al principi, però els beneficis a llarg termini en termes de mantenibilitat i flexibilitat són significatius.
