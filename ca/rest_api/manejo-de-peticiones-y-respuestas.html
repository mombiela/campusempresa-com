Introducció

En aquest tema, aprendrem sobre la gestió de peticions i respostes en el context d'una REST API. Aquest coneixement és fonamental per a qualsevol professional de TI que treballi amb serveis web, ja que les APIs RESTful són una de les formes més comunes de comunicació entre sistemes.

Conceptes Clau

  • Peticions HTTP: Són sol·licituds enviades pel client al servidor per realitzar alguna acció.
  • Respostes HTTP: Són les respostes enviades pel servidor al client després de processar una petició.
  • Mètodes HTTP: GET, POST, PUT, DELETE, PATCH, etc., que defineixen l'acció a realitzar.
  • Codis d'Estat HTTP: Indiquen el resultat de la petició (200 OK, 404 Not Found, 500 Internal Server Error, etc.).
  • Headers HTTP: Informació addicional enviada en les peticions i respostes (Content-Type, Authorization, etc.).
  • Cos de la Petició/Resposta: Dades enviades en el cos de la petició o resposta, generalment en format JSON.

Peticions HTTP

Estructura d'una Petició HTTP

Una petició HTTP típica inclou:

  • Línia de sol·licitud: Conté el mètode HTTP, la URL i la versió del protocol.
  • Headers: Informació addicional sobre la petició.
  • Cos: Dades enviades amb la petició (opcional, depèn del mètode HTTP).

Exemple d'una petició HTTP GET:

GET /api/usuaris HTTP/1.1
Host: exemple.com
Authorization: Bearer token123
Accept: application/json

Exemple d'una petició HTTP POST:

POST /api/usuaris HTTP/1.1
Host: exemple.com
Content-Type: application/json
Authorization: Bearer token123

{
  "nom": "Joan Perez",
  "email": "joan.perez@exemple.com"
}

Respostes HTTP

Estructura d'una Resposta HTTP

Una resposta HTTP típica inclou:

  • Línia d'estat: Conté la versió del protocol, el codi d'estat i el missatge d'estat.
  • Headers: Informació addicional sobre la resposta.
  • Cos: Dades enviades amb la resposta (opcional, depèn del codi d'estat).

Exemple d'una resposta HTTP 200 OK:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "nom": "Joan Perez",
  "email": "joan.perez@exemple.com"
}

Exemple d'una resposta HTTP 404 Not Found:

HTTP/1.1 404 Not Found
Content-Type: application/json

{
  "error": "Usuari no trobat"
}

Mètodes HTTP

Principals Mètodes HTTP

  • GET: Recupera informació del servidor.
  • POST: Envia dades al servidor per crear un nou recurs.
  • PUT: Actualitza un recurs existent en el servidor.
  • DELETE: Elimina un recurs del servidor.
  • PATCH: Aplica modificacions parcials a un recurs.

| Mètode | Descripció | Idempotent | Segur | |--------|----------------------------------|------------|--------| | GET | Recupera informació | Sí | Sí | | POST | Crea un nou recurs | No | No | | PUT | Actualitza un recurs existent | Sí | No | | DELETE | Elimina un recurs | Sí | No | | PATCH | Modifica parcialment un recurs | No | No |

Codis d'Estat HTTP

Principals Codis d'Estat

  • 2xx (Èxit): La petició va ser rebuda, entesa i acceptada.
    • 200 OK: La petició va ser exitosa.
    • 201 Created: Un nou recurs va ser creat.
  • 4xx (Errors del Client): La petició conté sintaxi incorrecta o no pot ser processada.
    • 400 Bad Request: La petició és invàlida.
    • 401 Unauthorized: L'autenticació és requerida.
    • 404 Not Found: El recurs no va ser trobat.
  • 5xx (Errors del Servidor): El servidor va fallar en complir una petició aparentment vàlida.
    • 500 Internal Server Error: Error genèric del servidor.
    • 503 Service Unavailable: El servei no està disponible.

Headers HTTP

Principals Headers

  • Content-Type: Indica el tipus de contingut del cos de la petició/resposta.
  • Authorization: Conté credencials per autenticar el client.
  • Accept: Indica quins tipus de contingut pot processar el client.

Exemple d'ús de headers:

GET /api/usuaris HTTP/1.1
Host: exemple.com
Authorization: Bearer token123
Accept: application/json

Exemples Pràctics

Exemple d'una Petició i Resposta Completa

Petició POST per crear un usuari:

POST /api/usuaris HTTP/1.1
Host: exemple.com
Content-Type: application/json
Authorization: Bearer token123

{
  "nom": "Joan Perez",
  "email": "joan.perez@exemple.com"
}

Resposta del servidor:

HTTP/1.1 201 Created
Content-Type: application/json

{
  "id": 1,
  "nom": "Joan Perez",
  "email": "joan.perez@exemple.com"
}

Conclusió

La gestió de peticions i respostes és un aspecte crucial en el desenvolupament i consum d'APIs RESTful. Comprendre l'estructura de les peticions i respostes, els mètodes HTTP, els codis d'estat i els headers és essencial per interactuar eficaçment amb qualsevol API. Amb aquesta base, estaràs millor preparat per desenvolupar i consumir serveis web de manera eficient i efectiva.
