Automatització i Personalització Avançada

Introducció

En aquesta secció del curs, explorarem com portar l'automatització a un nivell superior mitjançant la personalització avançada. Aprendrem a utilitzar eines i tècniques que permeten adaptar els processos automatitzats a les necessitats específiques de cada projecte o client.

Conceptes Clau

  • Automatització: Ús de tecnologia per realitzar tasques repetitives sense intervenció humana.
  • Personalització: Adaptació de processos i eines per satisfer necessitats específiques.
  • Scripts: Programes petits que automatitzen tasques específiques.
  • APIs: Interfícies de programació que permeten la interacció entre diferents sistemes.
  • Workflows: Seqüències de tasques automatitzades que s'executen en un ordre específic.

Eines d'Automatització

Scripts

Els scripts són una forma poderosa de personalitzar l'automatització. Poden ser escrits en diversos llenguatges de programació com Python, Bash, PowerShell, entre altres.

Exemple de Script en Python

import os

def backup_files(source_dir, backup_dir):
    if not os.path.exists(backup_dir):
        os.makedirs(backup_dir)
    for filename in os.listdir(source_dir):
        full_file_name = os.path.join(source_dir, filename)
        if os.path.isfile(full_file_name):
            os.path.copy(full_file_name, backup_dir)

source_directory = '/path/to/source'
backup_directory = '/path/to/backup'

backup_files(source_directory, backup_directory)

Explicació:

  • import os: Importa el mòdul os per interactuar amb el sistema operatiu.
  • backup_files: Funció que copia arxius d'un directori a un altre.
  • os.makedirs: Crea el directori de còpia de seguretat si no existeix.
  • os.listdir: Llista els arxius en el directori d'origen.
  • os.path.isfile: Verifica si l'arxiu és un arxiu regular.
  • os.path.copy: Copia l'arxiu al directori de còpia de seguretat.

APIs

Les APIs permeten la integració i automatització entre diferents sistemes i aplicacions.

Exemple d'Ús d'una API en Python

import requests

def get_weather(city):
    api_key = 'your_api_key'
    base_url = 'http://api.openweathermap.org/data/2.5/weather'
    complete_url = f"{base_url}?q={city}&appid={api_key}"
    response = requests.get(complete_url)
    data = response.json()
    if data['cod'] == 200:
        main = data['main']
        temperature = main['temp']
        print(f"Temperature in {city}: {temperature}K")
    else:
        print(f"City {city} not found.")

city_name = 'London'
get_weather(city_name)

Explicació:

  • import requests: Importa el mòdul requests per fer sol·licituds HTTP.
  • get_weather: Funció que obté el clima d'una ciutat usant l'API de OpenWeatherMap.
  • requests.get: Fa una sol·licitud GET a l'API.
  • response.json(): Converteix la resposta en format JSON.
  • data['main']: Accedeix a la secció principal de les dades del clima.
  • data['cod']: Verifica el codi de resposta per assegurar-se que la ciutat ha estat trobada.

Workflows

Els workflows permeten l'execució d'una sèrie de tasques automatitzades en un ordre específic. Eines com Jenkins, GitHub Actions i Zapier són populars per crear workflows.

Exemple de Workflow en GitHub Actions

name: CI

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.x'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run tests
      run: |
        pytest

Explicació:

  • name: Nom del workflow.
  • on: Esdeveniment que desencadena el workflow (en aquest cas, un push a la branca main).
  • jobs: Defineix els treballs a realitzar.
  • runs-on: Especifica el sistema operatiu en el qual s'executarà el treball.
  • steps: Llista de passos a seguir en el treball.
  • actions/checkout@v2: Acció per fer checkout del codi.
  • actions/setup-python@v2: Acció per configurar Python.
  • pip install -r requirements.txt: Instal·la les dependències del projecte.
  • pytest: Executa els tests del projecte.

Conclusió

L'automatització i personalització avançada permeten optimitzar i adaptar els processos a les necessitats específiques de cada projecte. Mitjançant l'ús de scripts, APIs i workflows, és possible crear solucions automatitzades eficients i personalitzades. La clau està en entendre les eines disponibles i com combinar-les per assolir els objectius desitjats.
