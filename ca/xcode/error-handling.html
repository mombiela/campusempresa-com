Gestió d'Errors a Xcode
Generics ►

La gestió d'errors és un aspecte crucial de la programació que assegura que la teva aplicació pugui gestionar situacions inesperades de manera elegant. En aquesta secció, tractarem la gestió d'errors a Xcode, des de conceptes bàsics fins a tècniques avançades.

Introducció a la Gestió d'Errors

La gestió d'errors permet que la teva aplicació respongui a condicions inesperades (errors) de manera controlada. En Swift, la gestió d'errors es fa utilitzant el protocol Error i els blocs do-catch.

Conceptes Clau

  • Protocol Error: Un tipus que representa un error.
  • Funcions que Llancen Errors: Funcions que poden llançar un error.
  • Blocs Do-Catch: Utilitzats per gestionar errors llançats per funcions.
  • Try, Try?, i Try!: Diferents maneres de cridar funcions que llancen errors.

Gestió Bàsica d'Errors

Definició d'Errors

En Swift, defineixes errors creant una enumeració que conforma el protocol Error.

enum FileError: Error {
    case fileNotFound
    case unreadable
    case encodingFailed
}

Llançament d'Errors

Una funció que pot llançar un error ha d'estar marcada amb la paraula clau throws.

func readFile(at path: String) throws -> String {
    // Simula un error de fitxer no trobat
    throw FileError.fileNotFound
}

Gestió d'Errors amb Do-Catch

Gestionar errors utilitzant un bloc do-catch.

do {
    let content = try readFile(at: "path/to/file")
    print(content)
} catch FileError.fileNotFound {
    print("Fitxer no trobat.")
} catch FileError.unreadable {
    print("El fitxer no es pot llegir.")
} catch {
    print("S'ha produït un error desconegut.")
}

Ús de Try, Try?, i Try!

  • try: Utilitzat quan esperes que la funció llanci un error.
  • try?: Converteix l'error en un valor opcional.
  • try!: Força la funció a tenir èxit o fa que el programa es bloquegi.
let content = try? readFile(at: "path/to/file") // content és nil si es llança un error
let forcedContent = try! readFile(at: "path/to/file") // es bloqueja si es llança un error

Gestió Avançada d'Errors

Propagació d'Errors

Funcions que criden altres funcions que llancen errors poden propagar errors utilitzant la paraula clau throws.

func processFile(at path: String) throws {
    let content = try readFile(at: path)
    // Processa el contingut
}

Tipus d'Errors Personalitzats

Pots crear tipus d'errors personalitzats per proporcionar més context sobre l'error.

struct CustomError: Error {
    let message: String
    let code: Int
}

func performTask() throws {
    throw CustomError(message: "Alguna cosa ha anat malament", code: 404)
}

Gestió d'Errors en Codi Asíncron

Gestionar errors en codi asíncron requereix un enfocament diferent, sovint utilitzant manejadors de finalització o async/await.

func fetchData(completion: @escaping (Result<Data, Error>) -> Void) {
    // Simula una operació asíncrona
    DispatchQueue.global().async {
        // Simula un error
        completion(.failure(FileError.fileNotFound))
    }
}

fetchData { result in
    switch result {
    case .success(let data):
        print("Dades rebudes: \(data)")
    case .failure(let error):
        print("Error: \(error)")
    }
}

Conclusió

La gestió d'errors és una habilitat essencial per a qualsevol desenvolupador. Entenent i implementant tècniques adequades de gestió d'errors, pots fer que les teves aplicacions siguin més robustes i fàcils d'utilitzar. En aquesta secció, hem cobert:

  • Conceptes i sintaxi bàsica de la gestió d'errors.
  • Tècniques avançades com la propagació d'errors i tipus d'errors personalitzats.
  • Gestió d'errors en codi asíncron.

Amb aquestes eines, pots assegurar-te que les teves aplicacions gestionin situacions inesperades de manera elegant, proporcionant una millor experiència per als teus usuaris.
