Introducció a LLDB

LLDB (Low-Level Debugger) és un depurador potent que ve integrat amb Xcode. Permet als desenvolupadors inspeccionar i controlar l'execució dels seus programes, facilitant la identificació i correcció d'errors.

  • Què és LLDB?

    • LLDB és part del projecte LLVM.
    • S'utilitza per depurar programes en C, C++, Objective-C i Swift.
    • Integrat a Xcode, proporcionant una experiència de depuració sense interrupcions.

  • Per què utilitzar LLDB?

    • Proporciona informació detallada sobre l'execució del programa.
    • Permet punts de ruptura, punts de vigilància i avançar pas a pas pel codi.
    • Suporta la inspecció i manipulació de dades complexes.

Comandes bàsiques de LLDB

Entendre les comandes bàsiques de LLDB és crucial per a una depuració efectiva.

  • Iniciant LLDB

    • S'inicia automàticament quan executes la teva aplicació a Xcode.
    • També es pot iniciar manualment a través del terminal utilitzant lldb <executable>.

  • Comandes comunes

    • run o r: Inicia el programa.
    • breakpoint set o b: Estableix un punt de ruptura.
    • continue o c: Continua l'execució fins al següent punt de ruptura.
    • step o s: Entra a la següent línia de codi.
    • next o n: Salta a la següent línia de codi.
    • print o p: Imprimeix el valor d'una variable.

Exemple: Establint un punt de ruptura

func greet(name: String) {
    print("Hello, \(name)!")
}

greet(name: "World")

Per establir un punt de ruptura a la instrucció print:

(lldb) breakpoint set --file main.swift --line 2

Tècniques avançades de LLDB

Un cop et sentis còmode amb els conceptes bàsics, pots explorar funcions més avançades de LLDB.

  • Punts de ruptura condicionals

    • Punts de ruptura que només es disparen quan es compleix una condició específica.
    (lldb) breakpoint set --file main.swift --line 2 --condition 'name == "World"'

  • Punts de vigilància

    • Monitoritzen els canvis en una variable específica.
    (lldb) watchpoint set variable name

  • Examinant la memòria

    • Inspecciona adreces de memòria i continguts.
    (lldb) memory read <address>

  • Comandes personalitzades de LLDB

    • Crea comandes personalitzades per automatitzar tasques repetitives.
    (lldb) command alias myprint print (int)myVariable

Depurant Swift amb LLDB

Swift té algunes característiques i sintaxi úniques que requereixen comandes específiques de LLDB.

  • Imprimint variables de Swift

    • Utilitza po (print object) per a una millor llegibilitat.
    (lldb) po mySwiftVariable

  • Inspeccionant opcionals

    • Utilitza frame variable per inspeccionar valors opcionals.
    (lldb) frame variable myOptional

  • Gestionant errors de Swift

    • Utilitza catchpoint per capturar errors de Swift.
    (lldb) catchpoint throw

Consells i millors pràctiques

  • Utilitza punts de ruptura descriptius

    • Nomena els teus punts de ruptura per a una millor organització.
    (lldb) breakpoint set --name myBreakpoint

  • Actualitza regularment Xcode i LLDB

    • Assegura't de tenir les últimes funcions i correccions d'errors.

  • Practica regularment

    • Com més utilitzis LLDB, més proficient seràs.

Conclusió

Depurar amb LLDB a Xcode és una habilitat essencial per a qualsevol desenvolupador. Dominant tant les comandes bàsiques com les avançades de LLDB, pots millorar significativament la teva eficiència i efectivitat en la depuració. Recorda practicar regularment i explorar les àmplies capacitats de LLDB per convertir-te en un depurador proficient.
