Introducció als Protocols

Els protocols a Swift defineixen un pla de mètodes, propietats i altres requisits que s'adapten a una tasca o funcionalitat particular. Qualsevol classe, estructura o enumeració pot adoptar protocols per proporcionar una implementació real d'aquests requisits.

Conceptes Clau

  • Declaració de Protocol: Com declarar un protocol.
  • Adoptar un Protocol: Com fer que una classe, estructura o enumeració compleixi un protocol.
  • Herència de Protocol: Com els protocols poden heretar d'altres protocols.
  • Requisits de Protocol Opcional: Com definir requisits opcionals en protocols.

Declaració de Protocol

Per declarar un protocol, utilitza la paraula clau protocol seguida del nom del protocol i un conjunt de requisits.

protocol Describable {
    var description: String { get }
    func describe() -> String
}

Adoptar un Protocol

Un tipus adopta un protocol llistant el nom del protocol després del nom del tipus, separat per un dos punts.

struct Car: Describable {
    var description: String
    var make: String
    var model: String

    func describe() -> String {
        return "Aquest cotxe és un \(make) \(model)."
    }
}

Herència de Protocol

Els protocols poden heretar d'un o més altres protocols, i un protocol pot requerir que els tipus conformants implementin propietats o mètodes de múltiples protocols.

protocol Identifiable {
    var id: String { get }
}

protocol Nameable {
    var name: String { get }
}

protocol Person: Identifiable, Nameable {
    var age: Int { get }
}

Requisits de Protocol Opcional

Els protocols poden definir requisits opcionals utilitzant l'atribut @objc i la paraula clau optional. Això només està disponible per a protocols que estan marcats amb l'atribut @objc.

@objc protocol OptionalProtocol {
    @objc optional func optionalMethod()
}

Introducció a les Extensions

Les extensions afegeixen nova funcionalitat a una classe, estructura, enumeració o tipus de protocol existent. Això inclou la capacitat d'estendre tipus per als quals no tens accés al codi font original.

Conceptes Clau

  • Declaració d'Extensió: Com declarar una extensió.
  • Afegir Propietats Calculades: Com afegir propietats calculades utilitzant extensions.
  • Afegir Mètodes: Com afegir nous mètodes utilitzant extensions.
  • Conformitat de Protocol amb Extensions: Com utilitzar extensions per conformar-se a protocols.

Declaració d'Extensió

Per declarar una extensió, utilitza la paraula clau extension seguida del nom del tipus que vols estendre.

extension Int {
    var isEven: Bool {
        return self % 2 == 0
    }
}

Afegir Propietats Calculades

Les extensions poden afegir propietats calculades però no poden afegir propietats emmagatzemades.

extension Double {
    var km: Double { return self * 1_000.0 }
    var m: Double { return self }
    var cm: Double { return self / 100.0 }
    var mm: Double { return self / 1_000.0 }
}

Afegir Mètodes

Les extensions poden afegir nous mètodes a tipus existents.

extension String {
    func reversedString() -> String {
        return String(self.reversed())
    }
}

Conformitat de Protocol amb Extensions

Les extensions poden ser utilitzades per fer que un tipus existent compleixi un protocol.

protocol Greetable {
    func greet()
}

extension String: Greetable {
    func greet() {
        print("Hola, \(self)!")
    }
}

Resum

En aquesta secció, hem explorat els conceptes de protocols i extensions a Swift. Els protocols defineixen un pla de mètodes i propietats que poden ser adoptats per classes, estructures i enumeracions. Les extensions ens permeten afegir nova funcionalitat a tipus existents, incloent la capacitat de conformar-se a protocols. Entendre aquests conceptes és crucial per escriure codi flexible i reutilitzable a Swift.
