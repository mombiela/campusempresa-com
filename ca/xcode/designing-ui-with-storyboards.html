Introducció als Storyboards

Els storyboards són una característica potent a Xcode que permet als desenvolupadors dissenyar i organitzar la interfície d'usuari (UI) de les seves aplicacions iOS de manera visual. Aquesta secció cobrirà els conceptes bàsics dels storyboards, els seus avantatges i com començar a utilitzar-los.

  • Què és un Storyboard?

    • Una representació visual de la UI d'una aplicació iOS.
    • Permet dissenyar múltiples pantalles i les transicions entre elles.

  • Avantatges d'Utilitzar Storyboards

    • El disseny visual facilita la comprensió del flux de l'aplicació.
    • Simplifica el procés de disseny de UIs complexes.
    • Facilita la col·laboració entre els membres de l'equip.

Començant amb els Storyboards

En aquesta secció, recorrerem el procés de creació d'un nou storyboard i l'afegiment d'elements bàsics de la UI.

  • Creació d'un Nou Storyboard

    • Obre Xcode i crea un nou projecte.
    • Selecciona la plantilla "Single View App".
    • Al navegador de projectes, localitza el fitxer Main.storyboard.

  • Afegint Elements de la UI

    • Arrossega i deixa anar elements de la UI des de la Biblioteca d'Objectes al storyboard.
    • Els elements comuns inclouen etiquetes, botons, camps de text i imatges.

// 3. Arrossega un Botó des de la Biblioteca d'Objectes al View Controller.
// 4. Estableix el títol del botó a "Prem-me".

Auto Layout i Restriccions

L'Auto Layout és essencial per crear UIs responsives que funcionin en diferents mides de pantalla i orientacions. Aquesta secció cobrirà els conceptes bàsics de l'Auto Layout i com utilitzar les restriccions en els storyboards.

  • Comprenent l'Auto Layout

    • Un sistema que permet als desenvolupadors definir la posició i la mida dels elements de la UI en relació amb altres elements.
    • Assegura que la UI s'adapti a diferents mides de pantalla i orientacions.

  • Afegint Restriccions

    • Selecciona un element de la UI i fes clic al botó "Afegir Noves Restriccions" a la cantonada inferior dreta del storyboard.
    • Estableix restriccions per a l'amplada, l'alçada, el marge esquerre, el marge dret, la part superior i la part inferior.
// Exemple: Afegint restriccions a un botó
// 1. Selecciona el botó al storyboard.
// 2. Fes clic al botó "Afegir Noves Restriccions".
// 3. Estableix les restriccions de marge esquerre i dret a 20 punts des de les vores.
// 4. Estableix la restricció inferior a 50 punts des de la part inferior de la vista.

Segues i Navegació

Els segues s'utilitzen per definir les transicions entre diferents view controllers en un storyboard. Aquesta secció cobrirà els diferents tipus de segues i com implementar la navegació a la teva aplicació.

  • Tipus de Segues

    • Show (Push): Utilitzat per a controladors de navegació per empènyer un nou view controller a la pila.
    • Modal: Presenta un view controller de manera modal.
    • Custom: Permet animacions de transició personalitzades.

  • Creant un Segue

    • Control-arrossega des d'un element de la UI (per exemple, un botó) a un altre view controller.
    • Selecciona el tipus de segue des del menú emergent.

Tècniques Avançades de Storyboard

Aquesta secció cobrirà tècniques més avançades, com ara l'ús de referències de storyboard, view controllers personalitzats i contingut dinàmic.

  • Referències de Storyboard

    • Permet dividir un storyboard gran en peces més petites i manejables.
    • Millora el rendiment i la col·laboració.

  • View Controllers Personalitzats

    • Crea subclasses personalitzades de UIViewController per gestionar la lògica complexa de la UI.
    • Utilitza l'Inspector d'Identitat per establir la classe personalitzada per a un view controller al storyboard.
// Exemple: Creant un view controller personalitzat
import UIKit

class CustomViewController: UIViewController {
    override func viewDidLoad() {
        super.viewDidLoad()
        // Codi d'inicialització personalitzat
    }
}

// Al storyboard, estableix la classe personalitzada per al view controller a CustomViewController
  • Contingut Dinàmic
    • Utilitza UITableView i UICollectionView per mostrar contingut dinàmic i desplaçable.
    • Implementa els mètodes del data source i del delegate per gestionar el contingut.
// Exemple: Implementant una taula simple
import UIKit

class TableViewController: UITableViewController {
    let items = ["Item 1", "Item 2", "Item 3"]

    override func tableView(_ tableView: UITableView, numberOfRowsInSection section: Int) -> Int {
        return items.count
    }

    override func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
        let cell = tableView.dequeueReusableCell(withIdentifier: "Cell", for: indexPath)
        cell.textLabel?.text = items[indexPath.row]
        return cell
    }
}

// Al storyboard, estableix la classe del table view controller a TableViewController
// i estableix l'identificador de la cel·la a "Cell"

Conclusió

Els storyboards són una eina potent per dissenyar i gestionar la UI de les aplicacions iOS. Comprenent els conceptes bàsics dels storyboards, l'Auto Layout, els segues i les tècniques avançades, els desenvolupadors poden crear interfícies d'usuari responsives i dinàmiques. Practica aquests conceptes construint projectes de mostra i experimentant amb diferents elements de la UI i dissenys.
