Introducció al control de versions

  • Definició: El control de versions és un sistema que registra els canvis en un fitxer o conjunt de fitxers al llarg del temps perquè puguis recordar versions específiques més tard.
  • Importància:
    • Manté un registre de cada modificació.
    • Facilita la col·laboració.
    • Ajuda a gestionar versions i llançaments del projecte.

Començant amb Git

  • Què és Git?: Git és un sistema de control de versions distribuït dissenyat per gestionar tot, des de projectes petits fins a molt grans amb velocitat i eficiència.
  • Terminologia bàsica de Git:
    • Repositori (Repo): Un directori que conté el treball del teu projecte, incloent-hi tots els canvis fets als fitxers.
    • Commit: Una instantània del teu repositori en un moment específic.
    • Branca: Una versió paral·lela del teu repositori. Et permet treballar en diferents característiques o correccions de bugs de manera independent.
    • Merge: Combinació de canvis de diferents branques en una sola.

Configurant Git a Xcode

  • Prerequisits: Assegura't que Git està instal·lat al teu sistema. Xcode normalment ve amb Git preinstal·lat.

  • Creant un nou projecte amb Git:

    1. Obre Xcode i crea un nou projecte.
    2. En l'assistent de creació de projectes, assegura't que la casella "Create Git repository on my Mac" està seleccionada.

  • Clonant un repositori existent:

    1. Obre Xcode.
    2. Ves a Source Control > Clone.
    3. Introdueix l'URL del repositori i tria la carpeta de destinació.

Operacions bàsiques de Git a Xcode

  • Cometent canvis:

    1. Fes canvis als fitxers del teu projecte.
    2. Ves a Source Control > Commit.
    3. Selecciona els fitxers que vols cometre, introdueix un missatge de commit i fes clic a Commit.
    // Exemple de canvi en codi Swift
func greet() {
    print("Hello, World!")
}

  • Creant i canviant de branques:

    1. Ves a Source Control > New Branch.
    2. Introdueix el nom de la branca i fes clic a Create.
    3. Per canviar de branca, ves a Source Control > Switch Branch i selecciona la branca desitjada.

  • Fusionant branques:

    1. Ves a Source Control > Merge.
    2. Selecciona la branca que vols fusionar amb la teva branca actual i fes clic a Merge.

Funcions avançades de Git a Xcode

  • Guardant canvis temporalment (Stashing): Guarda temporalment els canvis que no estan llestos per ser comesos.

    1. Ves a Source Control > Stash Changes.
    2. Per aplicar els canvis guardats, ves a Source Control > Apply Stash.

  • Resolent conflictes de fusió:

    1. Quan es produeix un conflicte, Xcode destacarà els fitxers conflictius.
    2. Obre el fitxer conflictiu i resol manualment els conflictes.
    3. Després de resoldre'ls, marca el fitxer com a resolt a Source Control > Mark as Resolved.

  • Veient l'historial i la culpa (Blame):

    1. Per veure l'historial de commits, ves a Source Control > History.
    2. Per veure qui ha fet canvis en una línia específica, utilitza Source Control > Show Blame for Line.

Resum

  • Punts clau:
    • La integració de Git a Xcode simplifica el control de versions per a projectes iOS/macOS.
    • Operacions bàsiques com cometre, crear branques i fusionar es poden realitzar fàcilment dins de Xcode.
    • Funcions avançades com guardar canvis temporalment i resoldre conflictes també estan suportades, millorant el teu flux de treball.

Dominant Git a Xcode, pots gestionar eficientment les versions del teu projecte, col·laborar amb altres i mantenir una base de codi neta i organitzada.
