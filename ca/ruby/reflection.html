DSLs (Llenguatges Específics de Domini) ►

La reflexió és una característica poderosa en Ruby que permet a un programa inspeccionar i modificar la seva pròpia estructura i comportament en temps d'execució. Aquesta capacitat és essencial per a la metaprogramació, on el codi pot definir dinàmicament mètodes, classes i altres estructures.

Conceptes Clau

  • Introspecció: La capacitat d'examinar les propietats dels objectes, com ara la seva classe, mètodes i variables d'instància.
  • Invocació Dinàmica de Mètodes: Cridar mètodes els noms dels quals no es coneixen fins al temps d'execució.
  • Modificació de Classes i Mòduls: Afegir o alterar mètodes i propietats de classes i mòduls durant l'execució.

Introspecció

La introspecció és el procés d'examinar les propietats dels objectes en temps d'execució. Ruby proporciona diversos mètodes per facilitar la introspecció.

Exemple: Inspeccionant la Classe i els Mètodes d'un Objecte

class Person
  attr_accessor :name, :age

  def initialize(name, age)
    @name = name
    @age = age
  end

  def greet
    "Hello, my name is #{@name}."
  end
end

person = Person.new("Alice", 30)

# Inspeccionant la classe de l'objecte
puts person.class # Output: Person

# Llistant tots els mètodes de l'objecte
puts person.methods.sort

# Llistant variables d'instància
puts person.instance_variables # Output: [:@name, :@age]

# Obtenint el valor d'una variable d'instància
puts person.instance_variable_get(:@name) # Output: Alice

Invocació Dinàmica de Mètodes

La invocació dinàmica de mètodes et permet cridar mètodes els noms dels quals es determinen en temps d'execució. Això es pot fer utilitzant el mètode send.

Exemple: Utilitzant `send` per Invocar Mètodes Dinàmicament

class Calculator
  def add(a, b)
    a + b
  end

  def subtract(a, b)
    a - b
  end
end

calc = Calculator.new

# Noms de mètodes com a símbols
method_name = :add
puts calc.send(method_name, 5, 3) # Output: 8

method_name = :subtract
puts calc.send(method_name, 5, 3) # Output: 2

Modificació de Classes i Mòduls

Ruby et permet modificar classes i mòduls existents en temps d'execució. Això pot ser útil per afegir nous mètodes o alterar els existents.

Exemple: Afegint Mètodes a una Classe

class String
  def shout
    self.upcase + "!!!"
  end
end

puts "hello".shout # Output: HELLO!!!

Exemple: Utilitzant `define_method` per Crear Mètodes Dinàmicament

class Person
  [:name, :age, :gender].each do |attr|
    define_method(attr) do
      instance_variable_get("@#{attr}")
    end

    define_method("#{attr}=") do |value|
      instance_variable_set("@#{attr}", value)
    end
  end
end

person = Person.new
person.name = "Bob"
person.age = 25
person.gender = "Male"

puts person.name   # Output: Bob
puts person.age    # Output: 25
puts person.gender # Output: Male

Conclusió

La reflexió en Ruby proporciona eines poderoses per a la introspecció, la invocació dinàmica de mètodes i la modificació de classes i mòduls en temps d'execució. Aquestes capacitats són essencials per a la metaprogramació i poden millorar enormement la flexibilitat i dinamisme del teu codi. Comprenent i utilitzant la reflexió, pots escriure programes Ruby més adaptables i intel·ligents.
