Introducció a l'Arquitectura MVC

Model-View-Controller (MVC) és un patró de disseny utilitzat per separar les preocupacions en aplicacions de programari. Divideix una aplicació en tres components interconnectats:

  • Model: Gestiona les dades i la lògica de negoci.
  • Vista: S'encarrega de la visualització i la interfície d'usuari.
  • Controlador: Gestiona l'entrada de l'usuari i interactua amb el Model i la Vista.

Comprensió dels Components

Model

El Model representa les dades i les regles que governen l'accés i les actualitzacions d'aquestes dades. En Ruby, això sovint implica classes que interactuen amb la base de dades.

# app/models/article.rb
class Article < ApplicationRecord
  validates :title, presence: true
  validates :body, presence: true
end
  • Explicació: Aquesta classe Article hereta de ApplicationRecord, la qual cosa significa que està connectada a la base de dades. Els mètodes validates asseguren que tant title com body han d'estar presents perquè un objecte Article sigui vàlid.

Vista

La Vista és responsable de renderitzar la interfície d'usuari. En Ruby on Rails, les vistes es solen escriure en Ruby Embedit (ERB) o altres llenguatges de plantilles.

<!-- app/views/articles/show.html.erb -->
<h1><%= @article.title %></h1>
<p><%= @article.body %></p>
  • Explicació: Aquesta plantilla ERB mostra el títol i el cos d'un article. Les etiquetes <%= %> s'utilitzen per incrustar codi Ruby dins de l'HTML.

Controlador

El Controlador gestiona l'entrada de l'usuari, interactua amb el Model i renderitza la Vista adequada.

# app/controllers/articles_controller.rb
class ArticlesController < ApplicationController
  def show
    @article = Article.find(params[:id])
  end
end
  • Explicació: L'acció show en el ArticlesController troba un article pel seu ID (passat en els paràmetres de l'URL) i l'assigna a una variable d'instància @article, que després és accessible a la vista.

Com Interactuen els Components MVC

  1. Interacció de l'Usuari: L'usuari interactua amb la Vista (per exemple, fent clic en un enllaç).
  2. Acció del Controlador: El Controlador rep l'entrada, la processa (per exemple, obtenint dades del Model) i determina la resposta adequada.
  3. Interacció amb el Model: El Controlador pot interactuar amb el Model per recuperar o actualitzar dades.
  4. Renderització de la Vista: El Controlador passa les dades a la Vista, que renderitza la sortida final per a l'usuari.

Flux d'Exemple

  • Usuari: Fa clic en un enllaç per veure un article.
  • Controlador: Es desencadena l'acció show del ArticlesController.
  • Model: El model Article obté l'article de la base de dades.
  • Vista: Es renderitza la plantilla show.html.erb amb les dades de l'article.

Conceptes Avançats

Enrutament

L'enrutament en Rails mapatge URLs a accions del controlador.

# config/routes.rb
Rails.application.routes.draw do
  resources :articles, only: [:show]
end
  • Explicació: Aquesta declaració de ruta crea una ruta per a l'acció show del ArticlesController.

Vistes Parcials

Les vistes parcials permeten reutilitzar codi de vista.

<!-- app/views/articles/_article.html.erb -->
<div class="article">
  <h2><%= article.title %></h2>
  <p><%= article.body %></p>
</div>
  • Explicació: Aquesta parcial es pot renderitzar en altres vistes per mostrar un article.
<!-- app/views/articles/index.html.erb -->
<% @articles.each do |article| %>
  <%= render partial: "article", locals: { article: article } %>
<% end %>
  • Explicació: La vista index renderitza la parcial _article per a cada article en la col·lecció @articles.

Paràmetres Forts

Els paràmetres forts proporcionen una manera de llistar atributs que es poden assignar massivament.

# app/controllers/articles_controller.rb
class ArticlesController < ApplicationController
  def create
    @article = Article.new(article_params)
    if @article.save
      redirect_to @article
    else
      render 'new'
    end
  end

  private

  def article_params
    params.require(:article).permit(:title, :body)
  end
end
  • Explicació: El mètode article_params assegura que només els atributs title i body es poden assignar massivament.

Conclusió

L'arquitectura MVC és un patró de disseny potent que ajuda a organitzar el codi i separar les preocupacions en una aplicació Ruby on Rails. Comprenent i implementant MVC, pots crear aplicacions escalables i mantenibles. Aquest tema ha cobert els conceptes bàsics de MVC, com interactuen els components i alguns conceptes avançats per aprofundir en la teva comprensió.
