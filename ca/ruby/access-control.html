El control d'accés és un concepte fonamental en la programació orientada a objectes que ajuda a gestionar la visibilitat i l'accessibilitat dels mètodes i atributs en una classe. A Ruby, el control d'accés es gestiona utilitzant tres paraules clau principals: public, protected i private.

Mètodes Públics

  • Definició: Els mètodes públics poden ser cridats per qualsevol—no hi ha restriccions en la seva visibilitat.
  • Per Defecte: Per defecte, tots els mètodes en una classe Ruby són públics tret que s'especifiqui el contrari.
class Greeter
  def greet
    puts "Hello!"
  end
end

greeter = Greeter.new
greeter.greet  # Output: Hello!

Mètodes Protegits

  • Definició: Els mètodes protegits poden ser cridats per qualsevol instància de la classe definidora o les seves subclasses. No obstant això, no poden ser cridats amb un receptor explícit fora de la jerarquia de classes.
  • Cas d'Ús: Normalment s'utilitzen per a mètodes que haurien de ser accessibles dins de la classe i les seves subclasses però no des de fora.
class Person
  def initialize(name)
    @name = name
  end

  def compare_name(other_person)
    self.name == other_person.name
  end

  protected

  def name
    @name
  end
end

alice = Person.new("Alice")
bob = Person.new("Bob")
puts alice.compare_name(bob)  # Output: false

Mètodes Privats

  • Definició: Els mètodes privats només poden ser cridats dins del context de l'objecte actual. No poden ser cridats amb un receptor explícit, fins i tot si aquest receptor és self.
  • Cas d'Ús: S'utilitzen per a mètodes d'utilitat interna que no haurien de ser exposats fora de la classe.
class BankAccount
  def initialize(balance)
    @balance = balance
  end

  def display_balance
    puts "Your balance is $#{balance}"
  end

  private

  def balance
    @balance
  end
end

account = BankAccount.new(1000)
account.display_balance  # Output: Your balance is $1000
# account.balance  # Això generarà un error: private method `balance' called

Paraules Clau de Control d'Accés

  • public: Els mètodes que segueixen aquesta paraula clau són accessibles des de qualsevol lloc.
  • protected: Els mètodes que segueixen aquesta paraula clau són accessibles dins de la classe i les seves subclasses.
  • private: Els mètodes que segueixen aquesta paraula clau només són accessibles dins del context de l'objecte actual.

Exemple d'Ús de les Paraules Clau de Control d'Accés

class Example
  def public_method
    puts "This is a public method"
  end

  protected

  def protected_method
    puts "This is a protected method"
  end

  private

  def private_method
    puts "This is a private method"
  end
end

example = Example.new
example.public_method  # Output: This is a public method
# example.protected_method  # Genera un error: protected method `protected_method' called
# example.private_method  # Genera un error: private method `private_method' called

Resum

El control d'accés a Ruby és un concepte crucial per gestionar la visibilitat i l'accessibilitat dels mètodes dins d'una classe. Utilitzant les paraules clau public, protected i private, pots controlar com es criden els mètodes i assegurar-te que els mecanismes interns de la teva classe estiguin adequadament encapsulats. Entendre i aplicar aquests mecanismes de control d'accés t'ajudarà a escriure codi més segur i mantenible.
