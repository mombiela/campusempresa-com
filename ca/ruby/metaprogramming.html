Metaprogramació en Ruby
Reflexió ►

La metaprogramació és una característica poderosa en Ruby que et permet escriure codi que escriu codi. Això pot fer que els teus programes siguin més flexibles i redueixin la redundància. En aquesta secció, explorarem les bases de la metaprogramació i gradualment passarem a conceptes més avançats.

Introducció a la Metaprogramació

La metaprogramació implica escriure codi que manipula altre codi. En Ruby, això sovint significa definir mètodes dinàmicament, alterar classes o fins i tot modificar el comportament de mètodes existents.

Conceptes Clau

  • Definició Dinàmica de Mètodes: Crear mètodes en temps d'execució.
  • Method Missing: Gestionar trucades a mètodes no definits.
  • Manipulació de Classes i Mòduls: Canviar l'estructura de classes i mòduls dinàmicament.
  • Reflexió: Inspeccionar objectes, classes i mètodes en temps d'execució.

Definició Dinàmica de Mètodes

Ruby et permet definir mètodes dinàmicament utilitzant define_method.

Exemple

class DynamicGreeter
  def initialize(name)
    @name = name
  end

  define_method(:greet) do
    "Hola, #{@name}!"
  end
end

greeter = DynamicGreeter.new("Alice")
puts greeter.greet  # Sortida: Hola, Alice!

En aquest exemple, el mètode greet es defineix dinàmicament dins de la classe.

Method Missing

El mètode method_missing és una manera poderosa de gestionar trucades a mètodes que no existeixen.

Exemple

class GhostMethod
  def method_missing(name, *args)
    "Has intentat trucar a #{name} amb arguments: #{args.join(', ')}"
  end
end

ghost = GhostMethod.new
puts ghost.any_method(1, 2, 3)  # Sortida: Has intentat trucar a any_method amb arguments: 1, 2, 3

Aquí, method_missing captura qualsevol trucada a mètodes no definits i proporciona una resposta personalitzada.

Manipulació de Classes i Mòduls

Pots afegir mètodes dinàmicament a classes i mòduls.

Exemple

module DynamicMethods
  def dynamic_method
    "Sóc un mètode afegit dinàmicament!"
  end
end

class MyClass
  include DynamicMethods
end

obj = MyClass.new
puts obj.dynamic_method  # Sortida: Sóc un mètode afegit dinàmicament!

En aquest exemple, el mètode dynamic_method s'afegeix a MyClass a través del mòdul DynamicMethods.

Reflexió

La reflexió et permet inspeccionar i manipular les propietats dels objectes, classes i mètodes en temps d'execució.

Exemple

class Reflective
  def initialize; end
  def example_method; end
end

reflective = Reflective.new
puts reflective.class  # Sortida: Reflective
puts Reflective.instance_methods(false)  # Sortida: [:example_method]

Aquest exemple mostra com inspeccionar la classe d'un objecte i llistar els seus mètodes d'instància.

Metaprogramació Avançada

Mètodes Singleton

Els mètodes singleton són mètodes que es defineixen per a un sol objecte en lloc de per a totes les instàncies d'una classe.

Exemple

obj = Object.new

def obj.singleton_method
  "Sóc un mètode singleton!"
end

puts obj.singleton_method  # Sortida: Sóc un mètode singleton!

Utilitzant `send` i `define_singleton_method`

El mètode send et permet trucar a mètodes dinàmicament, i define_singleton_method et permet definir mètodes singleton.

Exemple

class DynamicCaller
  def initialize(name)
    @name = name
  end

  def call_method(method_name)
    send(method_name)
  end

  def greet
    "Hola, #{@name}!"
  end
end

caller = DynamicCaller.new("Bob")
puts caller.call_method(:greet)  # Sortida: Hola, Bob!

En aquest exemple, send s'utilitza per trucar al mètode greet dinàmicament.

Conclusió

La metaprogramació en Ruby és una eina poderosa que pot fer que el teu codi sigui més flexible i redueixi la redundància. Entenent i utilitzant la definició dinàmica de mètodes, method_missing, la manipulació de classes i mòduls, la reflexió i tècniques avançades com els mètodes singleton i send, pots escriure programes Ruby més dinàmics i adaptables. Recorda utilitzar aquestes tècniques amb prudència, ja que poden fer que el teu codi sigui més difícil d'entendre i mantenir si s'abusen.
