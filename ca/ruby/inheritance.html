El Projecte Sobre nosaltres Contribuir Donacions Llicència

L'herència és un concepte fonamental en la programació orientada a objectes (OOP) que permet a una classe heretar atributs i mètodes d'una altra classe. Això ajuda en la reutilització del codi i a establir una jerarquia natural entre classes.

Què és l'herència?

  • Herència permet que una nova classe (subclasse o classe derivada) hereti propietats i comportaments (mètodes) d'una classe existent (superclasse o classe base).
  • La subclasse pot afegir les seves pròpies propietats i mètodes o sobreescriure els heretats.

Sintaxi bàsica

En Ruby, l'herència s'implementa utilitzant el símbol <.

class Animal
  def speak
    puts "So generico d'animal"
  end
end

class Dog < Animal
  def speak
    puts "Bup!"
  end
end

dog = Dog.new
dog.speak  # Mostra: Bup!

En aquest exemple:

  • Animal és la superclasse.
  • Dog és la subclasse que hereta d'Animal.
  • La classe Dog sobreescriu el mètode speak.

Conceptes clau

  • Superclasse i Subclasse: La superclasse és la classe de la qual s'hereta, i la subclasse és la classe que hereta.
  • Sobreescriptura de mètodes: Les subclasses poden proporcionar una implementació específica d'un mètode que ja està definit en la seva superclasse.
  • Paraula clau super: Utilitzada per cridar mètodes de la superclasse dins de la subclasse.

Sobreescriptura de mètodes

Quan una subclasse defineix un mètode amb el mateix nom que un mètode en la superclasse, el mètode de la subclasse sobreescriu el mètode de la superclasse.

class Animal
  def speak
    puts "So generico d'animal"
  end
end

class Cat < Animal
  def speak
    puts "Mèu!"
  end
end

cat = Cat.new
cat.speak  # Mostra: Mèu!

Utilitzant `super`

La paraula clau super s'utilitza per cridar el mateix mètode de la superclasse.

class Animal
  def speak
    puts "So generico d'animal"
  end
end

class Dog < Animal
  def speak
    super
    puts "Bup!"
  end
end

dog = Dog.new
dog.speak
# Mostra:
# So generico d'animal
# Bup!

Herència i inicialització

Quan una subclasse s'inicialitza, pot cridar l'inicialitzador de la superclasse utilitzant super.

class Animal
  def initialize(name)
    @name = name
  end

  def display_name
    puts "Nom: #{@name}"
  end
end

class Dog < Animal
  def initialize(name, breed)
    super(name)
    @breed = breed
  end

  def display_breed
    puts "Raça: #{@breed}"
  end
end

dog = Dog.new("Buddy", "Golden Retriever")
dog.display_name   # Mostra: Nom: Buddy
dog.display_breed  # Mostra: Raça: Golden Retriever

Herència i control d'accés

Ruby proporciona tres nivells de control d'accés per als mètodes:

  • Públic: Els mètodes són accessibles des de qualsevol lloc.
  • Protegit: Els mètodes són accessibles dins de la classe i les seves subclasses.
  • Privat: Els mètodes són accessibles només dins de la classe.
class Animal
  def public_method
    puts "Mètode públic"
  end

  protected
  def protected_method
    puts "Mètode protegit"
  end

  private
  def private_method
    puts "Mètode privat"
  end
end

class Dog < Animal
  def access_methods
    public_method
    protected_method
    # private_method  # Això generarà un error
  end
end

dog = Dog.new
dog.access_methods
# Mostra:
# Mètode públic
# Mètode protegit

Conclusió

L'herència és una característica poderosa en Ruby que promou la reutilització del codi i una jerarquia lògica de classes. Entenent i utilitzant l'herència, pots crear codi més organitzat i mantenible. Recorda utilitzar la sobreescriptura de mètodes i la paraula clau super de manera efectiva per estendre i personalitzar el comportament dels mètodes heretats. El control d'accés ajuda a encapsular i protegir l'estat intern dels objectes, assegurant que els mètodes s'utilitzin adequadament dins de la jerarquia de classes.
