Introducció

Les proves i la depuració són habilitats essencials per a qualsevol programador. En aquesta secció, cobrim els conceptes bàsics de les proves i la depuració en Ruby, progressant des de nivells principiants fins a avançats. Al final d'aquesta secció, seràs capaç d'escriure proves per al teu codi Ruby i depurar eficaçment qualsevol problema que sorgeixi.

Proves en Ruby

Per què són importants les proves

  • Asseguren la correcció del codi.
  • Faciliten el manteniment del codi.
  • Ajuden a detectar errors aviat.
  • Proporcionen documentació sobre el comportament del codi.

Tipus de proves

  • Proves unitàries: Prova unitats o components individuals.
  • Proves d'integració: Prova la interacció entre diferents components.
  • Proves de sistema: Prova tot el sistema en conjunt.
  • Proves d'acceptació: Valida la funcionalitat de punta a punta segons els requisits.

Eines de proves en Ruby

  • Minitest: Una biblioteca de proves lleugera i ràpida.
  • RSpec: Un marc de proves més expressiu i ric en funcionalitats.

Escriure la teva primera prova amb Minitest

# Instal·la Minitest si encara no ho has fet
# gem install minitest

require 'minitest/autorun'

class TestExample < Minitest::Test
  def test_addition
    assert_equal 4, 2 + 2
  end
end
  • Explicació: Aquesta prova simple comprova si la suma de 2 i 2 és igual a 4 utilitzant Minitest.

Escriure la teva primera prova amb RSpec

# Instal·la RSpec si encara no ho has fet
# gem install rspec

# Crea un fitxer anomenat example_spec.rb
require 'rspec'

RSpec.describe 'Addition' do
  it 'adds 2 and 2 to get 4' do
    expect(2 + 2).to eq(4)
  end
end

# Executa la prova amb la comanda: rspec example_spec.rb
  • Explicació: Aquesta prova de RSpec comprova si la suma de 2 i 2 és igual a 4.

Proves avançades amb RSpec

  • Matchers: Utilitza diferents matchers per provar diverses condicions.
  • Hooks: Configura i desmunta el codi utilitzant hooks before i after.
  • Exemples compartits: Reutilitza el codi de prova per a múltiples contextos.
RSpec.describe 'Advanced Testing' do
  before(:each) do
    @array = [1, 2, 3]
  end

  it 'checks array length' do
    expect(@array.length).to eq(3)
  end

  it 'includes a specific element' do
    expect(@array).to include(2)
  end
end
  • Explicació: Aquest exemple demostra l'ús de hooks before i matchers en RSpec.

Depuració en Ruby

Tècniques comunes de depuració

  • Declaracions de print: Utilitza puts o p per imprimir els valors de les variables.
  • IRB (Interactive Ruby): Utilitza IRB per provar fragments de codi de manera interactiva.
  • Depurador: Utilitza el depurador integrat de Ruby o eines com pry.

Utilitzant declaracions de print

def add(a, b)
  result = a + b
  puts "Adding #{a} and #{b} to get #{result}"
  result
end

add(2, 3)
  • Explicació: Aquest codi utilitza puts per imprimir els valors de les variables durant l'execució.

Utilitzant IRB

  • Inicia IRB escrivint irb al teu terminal.
  • Prova fragments de codi de manera interactiva.
irb(main):001:0> 2 + 2
=> 4
irb(main):002:0> def greet(name)
irb(main):003:1>   "Hello, #{name}!"
irb(main):004:1> end
=> :greet
irb(main):005:0> greet("Alice")
=> "Hello, Alice!"
  • Explicació: Aquest exemple mostra com utilitzar IRB per provar codi Ruby de manera interactiva.

Utilitzant el depurador

  • Byebug: Un depurador popular per a Ruby.
  • Pry: Una alternativa a Byebug amb més funcionalitats.

Utilitzant Byebug

# Instal·la Byebug si encara no ho has fet
# gem install byebug

require 'byebug'

def add(a, b)
  byebug
  a + b
end

add(2, 3)
  • Explicació: Aquest codi utilitza Byebug per pausar l'execució i iniciar una sessió de depuració.

Utilitzant Pry

# Instal·la Pry si encara no ho has fet
# gem install pry

require 'pry'

def add(a, b)
  binding.pry
  a + b
end

add(2, 3)
  • Explicació: Aquest codi utilitza Pry per pausar l'execució i iniciar una sessió interactiva.

Conclusió

Les proves i la depuració són crucials per escriure codi Ruby fiable i fàcil de mantenir. En dominar aquestes habilitats, pots assegurar-te que el teu codi funcioni com s'espera i identificar i solucionar ràpidament qualsevol problema que sorgeixi. Tant si utilitzes Minitest, RSpec, declaracions de print, IRB, Byebug o Pry, la clau és integrar aquestes pràctiques en el teu flux de treball de desenvolupament per millorar la qualitat del teu codi.
