En aquesta secció, cobrirem com escriure fitxers en Ruby, començant pels conceptes bàsics i avançant cap a tècniques més avançades. Escriure fitxers és una habilitat essencial per a qualsevol programador, ja que et permet emmagatzemar dades de manera persistent.

Escriptura Bàsica de Fitxers

Obrir un Fitxer per Escriure

Per escriure en un fitxer en Ruby, primer cal obrir-lo. Pots fer-ho utilitzant el mètode File.open.

File.open('example.txt', 'w') do |file|
  file.puts 'Hello, world!'
end
  • 'example.txt': El nom del fitxer en el qual vols escriure.
  • 'w': El mode en què vols obrir el fitxer. 'w' significa mode d'escriptura. Si el fitxer no existeix, es crearà. Si ja existeix, es truncarà (és a dir, tot el contingut existent serà eliminat).

Escriure Dades en un Fitxer

Pots escriure dades en un fitxer utilitzant mètodes com puts, print i write.

File.open('example.txt', 'w') do |file|
  file.puts 'Hello, world!'  # Afegeix una nova línia al final
  file.print 'This is a test.'  # No afegeix una nova línia
  file.write 'Writing to a file in Ruby.'  # No afegeix una nova línia
end

Modes de Fitxer

Ruby suporta diversos modes de fitxer, cadascun amb un propòsit diferent. Aquí tens alguns dels més comuns:

| Mode | Descripció | |------|-------------| | 'r' | Mode de només lectura. El fitxer ha d'existir. | | 'w' | Mode de només escriptura. Crea un fitxer nou o trunca un d'existent. | | 'a' | Mode d'addició. Escriu dades al final del fitxer. | | 'r+' | Mode de lectura i escriptura. El fitxer ha d'existir. | | 'w+' | Mode de lectura i escriptura. Crea un fitxer nou o trunca un d'existent. | | 'a+' | Mode de lectura i escriptura. Escriu dades al final del fitxer. |

Escriptura Avançada de Fitxers

Escriure Dades Binàries

De vegades pot ser necessari escriure dades binàries en un fitxer. Pots fer-ho obrint el fitxer en mode binari.

File.open('binary_file.bin', 'wb') do |file|
  file.write [255, 0, 127].pack('C*')
end
  • 'wb': Mode d'escriptura binària.
  • [255, 0, 127].pack('C*'): Converteix una matriu d'enters en una cadena binària.

Gestió d'Excepcions

Quan treballes amb fitxers, és important gestionar les excepcions per evitar possibles errors.

begin
  File.open('example.txt', 'w') do |file|
    file.puts 'Hello, world!'
  end
rescue Errno::ENOENT => e
  puts "Fitxer no trobat: #{e.message}"
rescue IOError => e
  puts "Error d'IO: #{e.message}"
ensure
  file.close unless file.nil?
end
  • begin...rescue...ensure: Un bloc per gestionar excepcions.
  • Errno::ENOENT: S'aixeca quan el fitxer no es troba.
  • IOError: S'aixeca per errors generals d'IO.
  • ensure: Assegura que el fitxer es tanqui fins i tot si es produeix un error.

Resum

Escriure fitxers en Ruby és senzill però potent. Entenent els diferents modes de fitxer i mètodes disponibles, pots escriure dades en fitxers de manera eficient. Recorda gestionar les excepcions per fer que el teu codi sigui robust i lliure d'errors.

  • Utilitza File.open amb el mode adequat per escriure fitxers.
  • Tria el mètode correcte (puts, print, write) segons les teves necessitats.
  • Entén i utilitza diferents modes de fitxer ('w', 'a', 'r+', etc.).
  • Gestiona les excepcions per assegurar que el teu programa pugui fer front a situacions inesperades de manera elegant.

Dominant aquests conceptes, estaràs ben preparat per gestionar tasques d'escriptura de fitxers en Ruby, des de fitxers de text simples fins a dades binàries complexes.
