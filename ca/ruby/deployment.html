Introducció al Desplegament

El desplegament és el procés de fer que la teva aplicació Ruby estigui disponible per a l'ús. Això implica transferir el teu codi a un servidor, configurar el servidor i assegurar-te que la teva aplicació funcioni sense problemes en un entorn de producció.

Conceptes Clau

  • Servidor: Un ordinador o sistema que proporciona recursos, dades, serveis o programes a altres ordinadors, coneguts com a clients, a través d'una xarxa.
  • Entorn de Producció: L'entorn on la teva aplicació està en viu i accessible per als usuaris.
  • Entorn de Staging: Una rèplica de l'entorn de producció utilitzada per a proves abans del desplegament.
  • Integració Contínua/Desplegament Continu (CI/CD): Pràctiques que impliquen provar i desplegar automàticament els canvis de codi.

Configuració del Teu Entorn

Abans de desplegar la teva aplicació Ruby, necessites configurar els teus entorns de desenvolupament i producció.

Entorn de Desenvolupament

  • Instal·la Ruby i Rails (si utilitzes Rails).
  • Utilitza un gestor de versions com RVM o rbenv per gestionar les versions de Ruby.
  • Configura una base de dades local (SQLite, PostgreSQL, etc.).

Entorn de Producció

  • Tria un proveïdor d'allotjament (Heroku, AWS, DigitalOcean, etc.).
  • Configura una base de dades de producció (PostgreSQL, MySQL, etc.).
  • Configura variables d'entorn per a dades sensibles (claus API, credencials de la base de dades).

Desplegant a Heroku

Heroku és una plataforma com a servei (PaaS) popular que simplifica el procés de desplegament.

Prerequisits

  • Un compte de Heroku.
  • Heroku CLI instal·lat a la teva màquina local.

Passos per Desplegar

  1. Inicia sessió a Heroku:

    heroku login

  2. Crea una Nova Aplicació a Heroku:

    heroku create your-app-name

  3. Afegeix el Complement PostgreSQL:

    heroku addons:create heroku-postgresql:hobby-dev

  4. Desplega el Teu Codi:

    git push heroku main

  5. Executa les Migracions de la Base de Dades:

    heroku run rails db:migrate

  6. Obre la Teva Aplicació:

    heroku open

Codi d'Exemple

Aquí tens un exemple d'una aplicació Rails simple llesta per al desplegament:

# Gemfile
source 'https://rubygems.org'
gem 'rails', '~> 6.1.0'
gem 'pg', '>= 0.18', '< 2.0'

# config/database.yml
default: &default
  adapter: postgresql
  encoding: unicode
  pool: 5

development:
  <<: *default
  database: myapp_development

production:
  <<: *default
  database: myapp_production
  username: myapp
  password: <%= ENV['MYAPP_DATABASE_PASSWORD'] %>

Integració Contínua i Desplegament (CI/CD)

CI/CD automatitza el procés de provar i desplegar la teva aplicació. Eines populars de CI/CD inclouen GitHub Actions, CircleCI i Travis CI.

Exemple amb GitHub Actions

  1. Crea un Fitxer de Workflow:

    # .github/workflows/deploy.yml
name: Deploy to Heroku

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v2

      - name: Set up Ruby
        uses: ruby/setup-ruby@v1
        with:
          ruby-version: '2.7'

      - name: Install dependencies
        run: |
          gem install bundler
          bundle install --jobs 4 --retry 3

      - name: Deploy to Heroku
        env:
          HEROKU_API_KEY: ${{ secrets.HEROKU_API_KEY }}
        run: |
          git remote add heroku https://git.heroku.com/your-app-name.git
          git push heroku main

  2. Afegeix Secrets a GitHub:

    • Ves a la configuració del teu repositori.
    • Afegeix HEROKU_API_KEY amb la teva clau API de Heroku.

Conclusió

Desplegar una aplicació Ruby implica diversos passos, des de configurar el teu entorn fins a utilitzar plataformes com Heroku per al desplegament. Entendre aquests passos assegura que la teva aplicació funcioni sense problemes en un entorn de producció. A més, implementar pràctiques de CI/CD pot agilitzar el procés de desplegament, fent-lo més eficient i fiable.
