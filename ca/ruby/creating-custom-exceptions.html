Introducció

A Ruby, les excepcions s'utilitzen per gestionar errors i altres esdeveniments excepcionals. Tot i que Ruby proporciona un conjunt ric d'excepcions integrades, hi ha moments en què potser necessiteu crear les vostres pròpies excepcions personalitzades per gestionar situacions específiques a la vostra aplicació. Aquesta secció us guiarà a través del procés de creació i ús d'excepcions personalitzades a Ruby.

Per què Crear Excepcions Personalitzades?

  • Especificitat: Les excepcions personalitzades us permeten gestionar condicions d'error específiques amb més precisió.
  • Llegibilitat: Fan que el vostre codi sigui més llegible i mantenible indicant clarament quin tipus d'error ha ocorregut.
  • Separació de Responsabilitats: Les excepcions personalitzades ajuden a separar la lògica de gestió d'errors de la lògica principal del codi.

Creació d'una Excepció Personalitzada

Crear una excepció personalitzada a Ruby és senzill. Només cal definir una nova classe que hereti de la classe StandardError o qualsevol altra classe d'excepció existent.

Exemple Bàsic

Aquí teniu un exemple bàsic de creació i ús d'una excepció personalitzada:

class MyCustomError < StandardError
end

def risky_method
  raise MyCustomError, "Something went wrong!"
end

begin
  risky_method
rescue MyCustomError => e
  puts e.message
end

Explicació

  • Definició de l'Excepció: class MyCustomError < StandardError crea una nova classe d'excepció anomenada MyCustomError que hereta de StandardError.
  • Llançament de l'Excepció: raise MyCustomError, "Something went wrong!" llança l'excepció personalitzada amb un missatge d'error específic.
  • Gestió de l'Excepció: El bloc begin...rescue s'utilitza per gestionar l'excepció. Quan es llança MyCustomError, el bloc rescue l'atrapa i imprimeix el missatge d'error.

Afegir Comportament Personalitzat

Podeu afegir comportament personalitzat a les vostres classes d'excepció definint mètodes dins d'elles.

Exemple amb Comportament Personalitzat

class InvalidAgeError < StandardError
  attr_reader :age

  def initialize(age)
    @age = age
    super("Invalid age: #{age}")
  end
end

def validate_age(age)
  raise InvalidAgeError.new(age) if age < 0 || age > 150
  puts "Age is valid."
end

begin
  validate_age(-5)
rescue InvalidAgeError => e
  puts e.message
  puts "Invalid age provided: #{e.age}"
end

Explicació

  • Inicialització Personalitzada: El mètode initialize es sobreescriu per acceptar un paràmetre age i establir una variable d'instància.
  • Missatge Personalitzat: super("Invalid age: #{age}") crida el mètode initialize de la classe pare amb un missatge d'error personalitzat.
  • Accés a Dades Personalitzades: La línia attr_reader :age crea un mètode getter per a la variable d'instància age, permetent que sigui accessible en el bloc rescue.

Millors Pràctiques

  • Heretar de StandardError: Sempre hereteu de StandardError o les seves subclasses per assegurar-vos que les vostres excepcions personalitzades es gestionin correctament pel mecanisme de rescue per defecte de Ruby.
  • Noms Significatius: Doneu noms significatius a les vostres excepcions personalitzades que indiquin clarament el tipus d'error.
  • Missatges Personalitzats: Proporcioneu missatges d'error informatius per facilitar la depuració.

Conclusió

Crear excepcions personalitzades a Ruby us permet gestionar condicions d'error específiques de manera més efectiva i fer que el vostre codi sigui més llegible i mantenible. Seguint les millors pràctiques i afegint comportament personalitzat quan sigui necessari, podeu crear mecanismes de gestió d'errors robustos adaptats a les necessitats de la vostra aplicació.
