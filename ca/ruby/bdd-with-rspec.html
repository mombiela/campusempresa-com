Introducció al Desenvolupament Guiat pel Comportament (BDD)

  • Definició: El BDD és un procés de desenvolupament de programari que fomenta la col·laboració entre desenvolupadors, QA i participants no tècnics o de negoci en un projecte de programari.
  • Propòsit: Assegurar que tots els interessats tinguin una comprensió clara del comportament del programari abans que comenci el desenvolupament.
  • Conceptes Clau:
    • Històries d'Usuari: Descripcions de les característiques del programari des de la perspectiva de l'usuari final.
    • Escenaris: Exemples específics de com una característica hauria de comportar-se en diferents situacions.
    • Given-When-Then: Una plantilla per escriure escenaris.

Configuració de RSpec

  • Instal·lació: 
    gem install rspec
  • Inicialització: 
    rspec --init
    Aquesta comanda crea el directori spec necessari i els fitxers de configuració.

Escriure la Teva Primera Prova amb RSpec

  • Crear un Fitxer de Prova: 
    # spec/calculator_spec.rb
require 'calculator'

RSpec.describe Calculator do
  it 'afegeix dos números' do
    calculator = Calculator.new
    result = calculator.add(5, 10)
    expect(result).to eq(15)
  end
end
  • Executar Proves: 
    rspec

Comprendre la Sintaxi de RSpec

  • Blocs Describe: S'utilitzen per agrupar proves relacionades. 
    RSpec.describe 'Un grup de proves' do
  # les proves van aquí
end
  • Blocs It: Defineixen casos de prova individuals. 
    it 'fa alguna cosa' do
  # codi de prova aquí
end
  • Expectatives: Afirmacions per comprovar si el codi es comporta com s'espera. 
    expect(actual).to eq(expected)

Característiques Avançades de RSpec

  • Hooks: Codi que s'executa en certs punts del cicle de vida de la prova.
    • before: S'executa abans de cada prova. 
      before(:each) do
  # codi de configuració aquí
end
    • after: S'executa després de cada prova. 
      after(:each) do
  # codi de desmuntatge aquí
end
  • Exemples Compartits: Casos de prova reutilitzables. 
    RSpec.shared_examples 'un exemple compartit' do
  it 'fa alguna cosa' do
    expect(subject).to be_something
  end
end

RSpec.describe MyClass do
  it_behaves_like 'un exemple compartit'
end

Mocking i Stubbing

  • Mocks: Objectes que simulen el comportament d'objectes reals. 
    let(:mock_object) { double('MockObject') }
  • Stubs: Mètodes que retornen respostes predefinides. 
    allow(mock_object).to receive(:method_name).and_return('value')

Integració de RSpec amb Rails

  • Generar Fitxers RSpec: 
    rails generate rspec:install
  • Proves de Model: 
    RSpec.describe User, type: :model do
  it 'és vàlid amb atributs vàlids' do
    user = User.new(name: 'John Doe', email: 'john@example.com')
    expect(user).to be_valid
  end
end
  • Proves de Controlador: 
    RSpec.describe UsersController, type: :controller do
  describe 'GET #index' do
    it 'retorna una resposta satisfactòria' do
      get :index
      expect(response).to be_successful
    end
  end
end

Conclusió

El Desenvolupament Guiat pel Comportament amb RSpec permet un enfocament més col·laboratiu i clar per al desenvolupament de programari. Escriure proves que descriguin el comportament desitjat de l'aplicació permet als desenvolupadors assegurar-se que el programari compleix les necessitats de tots els interessats. RSpec proporciona un marc potent i flexible per escriure aquestes proves, convertint-lo en una eina essencial per als desenvolupadors de Ruby.
