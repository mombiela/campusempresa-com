Introducció a les Excepcions

A Ruby, les excepcions són una manera de gestionar els errors que es produeixen durant l'execució d'un programa. En lloc de fer que el programa es bloquegi, Ruby proporciona mecanismes per capturar i gestionar aquests errors de manera elegant.

  • Excepció: Un esdeveniment que interromp el flux normal d'un programa.
  • Gestió d'Excepcions: El procés de respondre a les excepcions quan es produeixen.

Gestió Bàsica d'Excepcions

Ruby utilitza les paraules clau begin, rescue i end per gestionar les excepcions.

Sintaxi

begin
  # Codi que podria generar una excepció
rescue
  # Codi que s'executa si es produeix una excepció
end

Exemple

begin
  puts "Introdueix un número:"
  number = gets.to_i
  result = 10 / number
  puts "El resultat és #{result}"
rescue
  puts "S'ha produït un error: No es permet la divisió per zero."
end

En aquest exemple:

  • El bloc begin conté codi que podria generar una excepció.
  • El bloc rescue conté codi que s'executa si es produeix una excepció.

Gestió d'Excepcions Específiques

Pots capturar tipus específics d'excepcions especificant la classe de l'excepció.

Sintaxi

begin
  # Codi que podria generar una excepció
rescue SpecificExceptionClass
  # Codi que s'executa si es produeix una excepció específica
end

Exemple

begin
  puts "Introdueix un número:"
  number = gets.to_i
  result = 10 / number
  puts "El resultat és #{result}"
rescue ZeroDivisionError
  puts "S'ha produït un error: No es permet la divisió per zero."
end

En aquest exemple:

  • El bloc rescue ZeroDivisionError només captura excepcions de tipus ZeroDivisionError.

Assegurar la Neteja amb `ensure`

El bloc ensure s'executa tant si es produeix una excepció com si no, fent-lo útil per a tasques de neteja.

Sintaxi

begin
  # Codi que podria generar una excepció
rescue
  # Codi que s'executa si es produeix una excepció
ensure
  # Codi que sempre s'executa
end

Exemple

begin
  file = File.open("example.txt", "r")
  # Realitzar operacions amb el fitxer
rescue
  puts "S'ha produït un error en obrir el fitxer."
ensure
  file.close if file
  puts "Fitxer tancat."
end

En aquest exemple:

  • El bloc ensure assegura que el fitxer es tanqui tant si es produeix una excepció com si no.

Generar Excepcions

Pots generar les teves pròpies excepcions utilitzant la paraula clau raise.

Sintaxi

raise "Missatge d'error"

Exemple

def divide(a, b)
  raise "No es permet la divisió per zero" si b == 0
  a / b
end

begin
  result = divide(10, 0)
  puts "El resultat és #{result}"
rescue => e
  puts "S'ha produït un error: #{e.message}"
end

En aquest exemple:

  • La paraula clau raise s'utilitza per generar una excepció si b és zero.
  • El bloc rescue captura l'excepció i imprimeix el missatge d'error.

Classes d'Excepcions Personalitzades

Pots definir les teves pròpies classes d'excepcions heretant de StandardError.

Exemple

class MyCustomError < StandardError; end

def risky_method
  raise MyCustomError, "Alguna cosa ha anat malament!"
end

begin
  risky_method
rescue MyCustomError => e
  puts "S'ha capturat un error personalitzat: #{e.message}"
end

En aquest exemple:

  • MyCustomError és una classe d'excepció personalitzada.
  • El bloc rescue MyCustomError captura excepcions de tipus MyCustomError.

Conclusió

La gestió d'excepcions és una part crucial per escriure programes Ruby robustos i resistents als errors. Utilitzant begin, rescue, ensure i classes d'excepcions personalitzades, pots gestionar els errors de manera elegant i assegurar-te que el teu programa pugui manejar situacions inesperades sense bloquejar-se. Practica aquests conceptes amb diversos escenaris per convertir-te en un expert en la gestió d'excepcions a Ruby.
