Què són les excepcions?

Les excepcions són esdeveniments que poden alterar el flux normal d'un programa. S'utilitzen per gestionar errors i altres condicions excepcionals de manera controlada. En Ruby, les excepcions són objectes que representen aquestes condicions d'error.

Conceptes clau

  • Gestió d'excepcions: El procés de respondre a l'ocurrència d'excepcions.
  • Raise: L'acció de crear una excepció.
  • Rescue: L'acció de gestionar una excepció.
  • Ensure: Un bloc que sempre s'executarà, independentment de si s'ha produït una excepció o no.

Crear excepcions

En Ruby, pots crear una excepció utilitzant la paraula clau raise. Això es pot fer amb o sense un argument.

Exemple: Crear una excepció

# Crear una excepció sense un argument
raise "S'ha produït un error!"

# Crear una excepció amb un tipus específic
raise ArgumentError, "Argument invàlid!"

Gestionar excepcions

Per gestionar excepcions, Ruby proporciona el bloc begin-rescue. Això et permet capturar i gestionar excepcions de manera elegant.

Exemple: Gestionar una excepció

begin
  # Codi que pot crear una excepció
  result = 10 / 0
rescue ZeroDivisionError => e
  # Codi per gestionar l'excepció
  puts "Error: #{e.message}"
end

Explicació

  • begin: Inicia el bloc de codi on poden ocórrer excepcions.
  • rescue: Defineix el bloc de codi a executar si es crea una excepció.
  • ZeroDivisionError: El tipus específic d'excepció que es gestiona.
  • e: La variable que conté l'objecte de l'excepció.

Garantir l'execució

El bloc ensure s'utilitza per garantir que cert codi sempre s'executarà, independentment de si s'ha produït una excepció o no.

Exemple: Utilitzar Ensure

begin
  # Codi que pot crear una excepció
  file = File.open("example.txt", "r")
  # Realitzar operacions amb el fitxer
rescue Errno::ENOENT => e
  # Gestionar l'excepció de fitxer no trobat
  puts "Fitxer no trobat: #{e.message}"
ensure
  # Garantir que el fitxer es tanqui
  file.close if file
end

Tipus comuns d'excepcions

Aquí hi ha alguns tipus comuns d'excepcions que podries trobar en Ruby:

| Tipus d'excepció | Descripció | |----------------------|----------------------------------------| | StandardError | El predeterminat per a la majoria d'errors | | ArgumentError | Creat quan els arguments són incorrectes | | NoMethodError | Creat quan no es troba un mètode | | RuntimeError | Un error genèric creat per raise | | ZeroDivisionError | Creat quan es divideix per zero | | TypeError | Creat quan un objecte és del tipus incorrecte |

Excepcions personalitzades

Pots definir les teves pròpies excepcions creant una nova classe que hereti de StandardError.

Exemple: Excepció personalitzada

class MyCustomError < StandardError
end

begin
  # Codi que pot crear una excepció personalitzada
  raise MyCustomError, "Alguna cosa ha anat malament!"
rescue MyCustomError => e
  # Gestionar l'excepció personalitzada
  puts "Error personalitzat: #{e.message}"
end

Resum

  • Les excepcions s'utilitzen per gestionar errors i condicions excepcionals en Ruby.
  • Utilitza raise per crear excepcions i rescue per gestionar-les.
  • El bloc ensure garanteix que cert codi sempre s'executarà.
  • Ruby proporciona una varietat de tipus d'excepcions integrades, i també pots crear excepcions personalitzades.

Entenent i utilitzant les excepcions, pots fer que els teus programes Ruby siguin més robustos i resistents als errors.
