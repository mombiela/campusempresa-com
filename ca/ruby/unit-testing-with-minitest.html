Introducció a les Proves Unitàries

  • Definició: Les proves unitàries impliquen provar unitats o components individuals d'un programari per assegurar-se que funcionen com es preveu.
  • Importància: Ajuda a identificar errors aviat, assegura la fiabilitat del codi i facilita la refactorització.

Començar amb Minitest

  • Què és Minitest?: Una biblioteca de proves per a Ruby que proporciona un conjunt complet de facilitats de prova que suporten TDD, BDD, simulació i benchmarking.
  • Instal·lació: 
    gem install minitest

Escriure la Teva Primera Prova

  • Crear un Fitxer de Prova: Normalment, els fitxers de prova es col·loquen en un directori test.
  • Estructura Bàsica: 
    require 'minitest/autorun'

class ExampleTest < Minitest::Test
  def test_example
    assert_equal 4, 2 + 2
  end
end
    • require 'minitest/autorun': Carrega Minitest i executa les proves automàticament.
    • class ExampleTest < Minitest::Test: Defineix una classe de prova que hereta de Minitest::Test.
    • def test_example: Defineix un mètode de prova. Els mètodes de prova han de començar amb test_.
    • assert_equal 4, 2 + 2: Una afirmació que comprova si l'expressió 2 + 2 és igual a 4.

Afirmacions Comunes

  • assert_equal: Comprova si dos valors són iguals. 
    assert_equal expected, actual
  • assert: Comprova si una condició és certa. 
    assert condition
  • assert_nil: Comprova si un valor és nil. 
    assert_nil value
  • assert_raises: Comprova si es llança una excepció específica. 
    assert_raises(ExpectedException) { code }

Tècniques Avançades de Prova

  • Configuració i Desmuntatge: Mètodes per executar codi abans i després de cada prova. 
    class AdvancedTest < Minitest::Test
  def setup
    @value = 2
  end

  def teardown
    # Codi de neteja
  end

  def test_value
    assert_equal 2, @value
  end
end
  • Simulació i Substitució: Simulant objectes o mètodes per aïllar proves. 
    require 'minitest/mock'

def test_mock
  mock = Minitest::Mock.new
  mock.expect(:foo, true)

  assert mock.foo
  mock.verify
end

Organitzar les Proves

  • Estructura del Directori de Proves: 
    project_root/
├── lib/
│   └── example.rb
└── test/
    ├── example_test.rb
    └── test_helper.rb
  • Ajuda de Prova: Un fitxer per requerir dependències comunes i configurar configuracions. 
    # test/test_helper.rb
require 'minitest/autorun'

Executar Proves

  • Línia de Comandes: Executar totes les proves amb: 
    ruby -Ilib:test test/example_test.rb
  • Rake: Utilitzant Rake per automatitzar l'execució de proves. 
    # Rakefile
require 'rake/testtask'

Rake::TestTask.new do |t|
  t.libs << 'test'
  t.test_files = FileList['test/**/*_test.rb']
end

task default: :test

Conclusió

Les proves unitàries amb Minitest són una manera poderosa d'assegurar que el teu codi Ruby sigui fiable i mantenible. Escriure proves et permet detectar errors aviat, refactoritzar amb confiança i construir una base de codi robusta. Comença amb afirmacions senzilles i incorpora gradualment tècniques més avançades com la simulació i els mètodes de configuració/desmuntatge per millorar el teu conjunt de proves.
