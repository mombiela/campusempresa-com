Introducció als Patrons de Disseny

Els patrons de disseny són solucions típiques a problemes comuns en el disseny de programari. Són com plànols que pots personalitzar per resoldre un problema de disseny particular en el teu codi. En aquesta secció, explorarem com implementar alguns dels patrons de disseny més comuns en F#.

Patrons Creacionals

Patró Singleton

El patró Singleton assegura que una classe només tingui una instància i proporciona un punt d'accés global a aquesta.

Exemple:

type Singleton private () =
    static let instance = Singleton()
    static member Instance = instance

// Ús
let singletonInstance = Singleton.Instance

Patró Mètode Fàbrica

El patró Mètode Fàbrica defineix una interfície per crear un objecte però permet que les subclasses alterin el tipus d'objectes que es crearan.

Exemple:

type IProduct =
    abstract member Operation: unit -> string

type ConcreteProductA() =
    interface IProduct with
        member this.Operation() = "Resultat de ConcreteProductA"

type ConcreteProductB() =
    interface IProduct with
        member this.Operation() = "Resultat de ConcreteProductB"

type Creator() =
    member this.FactoryMethod(productType: string) : IProduct =
        match productType with
        | "A" -> ConcreteProductA() :> IProduct
        | "B" -> ConcreteProductB() :> IProduct
        | _ -> failwith "Tipus de producte desconegut"

// Ús
let creator = Creator()
let productA = creator.FactoryMethod("A")
printfn "%s" (productA.Operation())

Patrons Estructurals

Patró Adaptador

El patró Adaptador permet que interfícies incompatibles treballin juntes. Actua com un pont entre dues interfícies incompatibles.

Exemple:

type ITarget =
    abstract member Request: unit -> string

type Adaptee() =
    member this.SpecificRequest() = "Petició específica."

type Adapter(adaptee: Adaptee) =
    interface ITarget with
        member this.Request() = adaptee.SpecificRequest()

// Ús
let adaptee = Adaptee()
let adapter = Adapter(adaptee)
printfn "%s" (adapter :> ITarget).Request()

Patró Decorador

El patró Decorador permet afegir comportament a un objecte individual, dinàmicament, sense afectar el comportament d'altres objectes de la mateixa classe.

Exemple:

type IComponent =
    abstract member Operation: unit -> string

type ConcreteComponent() =
    interface IComponent with
        member this.Operation() = "ConcreteComponent"

type Decorator(component: IComponent) =
    interface IComponent with
        member this.Operation() = sprintf "Decorator(%s)" (component.Operation())

// Ús
let component = ConcreteComponent()
let decorator = Decorator(component)
printfn "%s" (decorator :> IComponent).Operation()

Patrons de Comportament

Patró Observador

El patró Observador defineix una dependència de un-a-molts entre objectes de manera que quan un objecte canvia d'estat, tots els seus dependents són notificats i actualitzats automàticament.

Exemple:

type IObserver =
    abstract member Update: string -> unit

type Subject() =
    let observers = System.Collections.Generic.List<IObserver>()
    member this.Attach(observer: IObserver) = observers.Add(observer)
    member this.Detach(observer: IObserver) = observers.Remove(observer)
    member this.Notify(message: string) =
        for observer in observers do
            observer.Update(message)

type ConcreteObserver(name: string) =
    interface IObserver with
        member this.Update(message: string) =
            printfn "Observador %s ha rebut el missatge: %s" name message

// Ús
let subject = Subject()
let observer1 = ConcreteObserver("A")
let observer2 = ConcreteObserver("B")

subject.Attach(observer1)
subject.Attach(observer2)

subject.Notify("Hola Observadors!")

Patró Estratègia

El patró Estratègia defineix una família d'algoritmes, encapsula cadascun d'ells, i els fa intercanviables. Estratègia permet que l'algoritme variï independentment dels clients que l'utilitzen.

Exemple:

type IStrategy =
    abstract member Execute: int -> int -> int

type ConcreteStrategyAdd() =
    interface IStrategy with
        member this.Execute(a: int, b: int) = a + b

type ConcreteStrategySubtract() =
    interface IStrategy with
        member this.Execute(a: int, b: int) = a - b

type Context(strategy: IStrategy) =
    member this.ExecuteStrategy(a: int, b: int) = strategy.Execute(a, b)

// Ús
let contextAdd = Context(ConcreteStrategyAdd())
let contextSubtract = Context(ConcreteStrategySubtract())

printfn "Suma: %d" (contextAdd.ExecuteStrategy(3, 4))
printfn "Resta: %d" (contextSubtract.ExecuteStrategy(3, 4))

Conclusió

Els patrons de disseny són eines poderoses en la caixa d'eines d'un programador. Proporcionen solucions provades a problemes comuns i poden millorar significativament l'estructura i mantenibilitat del teu codi. Entenent i aplicant aquests patrons en F#, pots escriure aplicacions més robustes i escalables.
