Introducció

En aquesta secció, explorarem dos conceptes fonamentals de la programació orientada a objectes (OOP) en F#: Encapsulació i Polimorfisme. Aquests conceptes ajuden a crear programari robust, mantenible i escalable.

Encapsulació

L'encapsulació és el concepte d'agrupar dades i mètodes que operen sobre aquestes dades dins d'una unitat única, típicament una classe. Restringeix l'accés directe a alguns dels components d'un objecte, cosa que pot prevenir la modificació accidental de dades.

Conceptes Clau de l'Encapsulació

  • Ocultació de Dades: Restringir l'accés a l'estat intern de l'objecte.
  • Interfície Pública: Proporcionar mètodes per interactuar amb les dades de l'objecte.

Exemple d'Encapsulació en F#

type Person(name: string, age: int) =
    // Camps privats
    let mutable privateName = name
    let mutable privateAge = age

    // Mètodes públics
    member this.GetName() = privateName
    member this.GetAge() = privateAge
    member this.SetAge(newAge: int) =
        if newAge > 0 then
            privateAge <- newAge
        else
            printfn "L'edat ha de ser positiva"

// Ús
let person = Person("John", 30)
printfn "Nom: %s" (person.GetName())
printfn "Edat: %d" (person.GetAge())
person.SetAge(35)
printfn "Edat Actualitzada: %d" (person.GetAge())

En aquest exemple:

  • La classe Person encapsula els camps name i age.
  • Els camps són privats i només es poden accedir o modificar a través de mètodes públics (GetName, GetAge, SetAge).

Polimorfisme

El polimorfisme permet que objectes de diferents classes siguin tractats com a objectes d'una superclasse comuna. És una manera de realitzar una única acció en diferents formes.

Conceptes Clau del Polimorfisme

  • Sobreescriptura de Mètodes: Una subclasse proporciona una implementació específica d'un mètode que ja està definit a la seva superclasse.
  • Implementació d'Interfícies: Diferents classes poden implementar la mateixa interfície de maneres diferents.

Exemple de Polimorfisme en F#

Sobreescriptura de Mètodes

type Animal() =
    abstract member Speak: unit -> unit

type Dog() =
    inherit Animal()
    override this.Speak() = printfn "Woof!"

type Cat() =
    inherit Animal()
    override this.Speak() = printfn "Meow!"

// Ús
let animals: Animal list = [ Dog(); Cat() ]
animals |> List.iter (fun animal -> animal.Speak())

En aquest exemple:

  • Animal és una classe abstracta amb un mètode abstracte Speak.
  • Les classes Dog i Cat hereten de Animal i proporcionen la seva pròpia implementació del mètode Speak.
  • Es crea una llista d'objectes Animal, que conté tant objectes Dog com Cat. Es crida el mètode Speak en cada objecte, demostrant el polimorfisme.

Implementació d'Interfícies

type IShape =
    abstract member Area: unit -> float

type Circle(radius: float) =
    interface IShape with
        member this.Area() = System.Math.PI * radius * radius

type Rectangle(width: float, height: float) =
    interface IShape with
        member this.Area() = width * height

// Ús
let shapes: IShape list = [ Circle(5.0); Rectangle(4.0, 6.0) ]
shapes |> List.iter (fun shape -> printfn "Àrea: %f" (shape.Area()))

En aquest exemple:

  • IShape és una interfície amb un mètode Area.
  • Les classes Circle i Rectangle implementen la interfície IShape.
  • Es crea una llista d'objectes IShape, que conté tant objectes Circle com Rectangle. Es crida el mètode Area en cada objecte, demostrant el polimorfisme.

Conclusió

L'encapsulació i el polimorfisme són conceptes poderosos en F# que ajuden a crear codi ben estructurat i flexible. L'encapsulació assegura que l'estat intern d'un objecte estigui protegit, mentre que el polimorfisme permet codi flexible i reutilitzable permetent que els objectes siguin tractats com a instàncies de la seva classe pare o interfície. Entendre i aplicar aquests conceptes millorarà significativament la teva capacitat per escriure aplicacions F# robustes i mantenibles.
