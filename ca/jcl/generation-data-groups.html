Introducció als GDGs

Els Grups de Dades de Generació (GDGs) són un mètode en JCL (Job Control Language) per gestionar conjunts de dades que estan lògicament relacionats i necessiten ser processats en una seqüència específica. Els GDGs són particularment útils per manejar conjunts de dades que es creen de manera regular, com ara registres diaris o còpies de seguretat.

Conceptes Clau

  • Grup de Dades de Generació (GDG): Una col·lecció de conjunts de dades que estan relacionats i es poden referenciar amb un nom comú.
  • Generació: Cada conjunt de dades dins d'un GDG es coneix com una generació i s'identifica per un número de generació.
  • Base: El nom comú utilitzat per referenciar tot el GDG.
  • Número de Generació Relativa: Una manera de referenciar conjunts de dades dins d'un GDG en relació amb la generació actual.

Creació i Gestió de GDGs

Definició d'una Base GDG

Per crear un GDG, primer cal definir una base GDG. Això es fa utilitzant la utilitat IDCAMS.

//DEFINEGDG EXEC PGM=IDCAMS
//SYSPRINT DD SYSOUT=*
//SYSIN    DD *
  DEFINE GDG (NAME(MY.GDG.BASE) LIMIT(5) SCRATCH)
/*
  • NAME: Especifica el nom de la base GDG.
  • LIMIT: Especifica el nombre màxim de generacions que es poden mantenir.
  • SCRATCH: Indica que les generacions antigues s'han d'eliminar quan se superi el límit.

Creació d'una Nova Generació

Per crear una nova generació dins d'un GDG, s'utilitza el nom de la base GDG seguit de (0).

//NEWGEN   EXEC PGM=IEFBR14
//NEWFILE  DD DSN=MY.GDG.BASE(+1),
//            DISP=(NEW,CATLG,DELETE),
//            SPACE=(TRK,(1,1)),
//            DCB=(LRECL=80,BLKSIZE=800,RECFM=FB)
  • MY.GDG.BASE(+1): Crea una nova generació.
  • DISP: Especifica la disposició del conjunt de dades.
  • SPACE: Assigna espai per al conjunt de dades.
  • DCB: Defineix els paràmetres del bloc de control del conjunt de dades.

Accés a les Generacions GDG

Es poden accedir a generacions específiques utilitzant números de generació relativa.

//ACCESSGEN EXEC PGM=IEFBR14
//OLDGEN   DD DSN=MY.GDG.BASE(-1),DISP=SHR
//CURGEN   DD DSN=MY.GDG.BASE(0),DISP=SHR
  • MY.GDG.BASE(-1): Es refereix a la generació anterior.
  • MY.GDG.BASE(0): Es refereix a la generació actual.

Gestió Avançada de GDGs

Eliminació de Generacions

Quan s'assoleix el límit de generacions, la generació més antiga s'elimina automàticament si s'utilitza l'opció SCRATCH. Si es especifica NOSCRATCH, la generació més antiga es descataloga però no s'elimina.

Ús de GDGs en Procediments

Els GDGs es poden utilitzar en procediments JCL per automatitzar la creació i el processament de conjunts de dades.

//PROC1    PROC
//STEP1    EXEC PGM=MYPROG
//INPUT    DD DSN=MY.GDG.BASE(0),DISP=SHR
//OUTPUT   DD DSN=MY.GDG.BASE(+1),DISP=(NEW,CATLG,DELETE),
//            SPACE=(TRK,(1,1)),
//            DCB=(LRECL=80,BLKSIZE=800,RECFM=FB)
/*
//PROC1    PEND
  • PROC: Defineix l'inici d'un procediment.
  • PEND: Marca el final del procediment.

Comparació d'Estructures GDG

| Característica | Descripció | |------------------|-----------------------------------------------------------------------------| | Nom Base | Nom comú per a totes les generacions. | | Generació | Conjunt de dades individual dins del GDG. | | Generació Relativa | Número utilitzat per referenciar conjunts de dades en relació amb la generació actual. | | Límit | Nombre màxim de generacions mantingudes. | | SCRATCH | Opció per eliminar generacions antigues quan se supera el límit. | | NOSCRATCH | Opció per descatalogar però no eliminar generacions antigues quan se supera el límit. |

Conclusió

Els Grups de Dades de Generació (GDGs) en JCL proporcionen un mecanisme poderós per gestionar conjunts de dades que es creen i processen en una seqüència. Entenent com definir, crear i accedir als GDGs, es poden gestionar efectivament conjunts de dades que necessiten ser manejats en un ordre lògic. Les característiques avançades com l'eliminació de generacions i l'ús de GDGs en procediments milloren encara més la seva utilitat en fluxos de treball JCL complexos.
