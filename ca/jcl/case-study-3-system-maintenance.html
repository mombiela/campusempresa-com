Introducció

En aquest estudi de cas, explorarem com es pot utilitzar JCL (Job Control Language) per a tasques de manteniment del sistema. El manteniment del sistema és crucial per assegurar que l'entorn mainframe funcioni de manera fluida i eficient. Cobrirem diverses tasques de manteniment, incloent còpies de seguretat, neteja de dades i monitorització del sistema.

Conceptes Clau

  • Manteniment del Sistema: El procés de mantenir un sistema informàtic funcionant eficientment mitjançant la realització de comprovacions regulars, actualitzacions i reparacions.
  • JCL: Un llenguatge de script utilitzat en sistemes operatius mainframe d'IBM per instruir el sistema sobre com executar un treball per lots o iniciar un subsistema.

Procediments de Còpia de Seguretat

Fer còpies de seguretat de les dades és una tasca de manteniment crítica per prevenir la pèrdua de dades. JCL es pot utilitzar per automatitzar els processos de còpia de seguretat.

Exemple: Treball de Còpia de Seguretat

//BACKUP   JOB (ACCT),'BACKUP JOB',CLASS=A,MSGCLASS=X
//STEP1    EXEC PGM=ADRDSSU
//SYSPRINT DD SYSOUT=*
//SYSIN    DD *
  DUMP DATASET(INCLUDE(**))
  OUTDDNAME(TAPE1)
  COMPRESS
/*
//TAPE1    DD DSN=BACKUP.DATASET,DISP=(NEW,KEEP),UNIT=TAPE,
//         VOL=SER=000001,LABEL=(1,SL)
//
  • Explicació:
    • Declaració JOB: Defineix el nom del treball i la informació comptable.
    • Declaració EXEC: Especifica el programa a executar (ADRDSSU per a la utilitat DFSMSdss).
    • Declaració SYSPRINT DD: Dirigeix els missatges de sortida a la sortida del sistema.
    • Declaració SYSIN DD: Conté les declaracions de control per a la utilitat DFSMSdss.
    • DUMP DATASET(INCLUDE())**: Especifica que tots els datasets han de ser inclosos en el dump.
    • OUTDDNAME(TAPE1): Nomena la declaració DD de sortida.
    • COMPRESS: Indica que les dades han de ser comprimides.
    • Declaració TAPE1 DD: Defineix el dataset de sortida en cinta.

Neteja de Dades

La neteja regular de dades ajuda a mantenir el rendiment del sistema eliminant fitxers i datasets innecessaris.

Exemple: Treball de Neteja de Dades

//CLEANUP  JOB (ACCT),'CLEANUP JOB',CLASS=A,MSGCLASS=X
//STEP1    EXEC PGM=IDCAMS
//SYSPRINT DD SYSOUT=*
//SYSIN    DD *
  DELETE USER.DATASET1
  DELETE USER.DATASET2
  DELETE USER.DATASET3
/*
//
  • Explicació:
    • Declaració JOB: Defineix el nom del treball i la informació comptable.
    • Declaració EXEC: Especifica el programa a executar (IDCAMS per a Access Method Services).
    • Declaració SYSPRINT DD: Dirigeix els missatges de sortida a la sortida del sistema.
    • Declaració SYSIN DD: Conté les declaracions de control per a IDCAMS.
    • Declaracions DELETE: Especifica els datasets a eliminar.

Monitorització del Sistema

Monitoritzar el sistema és essencial per detectar i resoldre problemes abans que afectin el rendiment.

Exemple: Treball de Monitorització del Sistema

//MONITOR  JOB (ACCT),'MONITOR JOB',CLASS=A,MSGCLASS=X
//STEP1    EXEC PGM=RMF
//SYSPRINT DD SYSOUT=*
//SYSIN    DD *
  REPORT TYPE(SYSTEM)
  INTERVAL(15)
/*
//
  • Explicació:
    • Declaració JOB: Defineix el nom del treball i la informació comptable.
    • Declaració EXEC: Especifica el programa a executar (RMF per a Resource Measurement Facility).
    • Declaració SYSPRINT DD: Dirigeix els missatges de sortida a la sortida del sistema.
    • Declaració SYSIN DD: Conté les declaracions de control per a RMF.
    • REPORT TYPE(SYSTEM): Especifica el tipus d'informe a generar.
    • INTERVAL(15): Estableix l'interval de reportatge a 15 minuts.

Conclusió

En aquest estudi de cas, hem demostrat com es pot utilitzar JCL per a diverses tasques de manteniment del sistema com còpies de seguretat, neteja de dades i monitorització del sistema. Automatitzant aquestes tasques amb JCL, els administradors de sistemes poden assegurar que l'entorn mainframe es mantingui eficient i fiable. El manteniment regular és essencial per prevenir problemes i mantenir un rendiment òptim.
