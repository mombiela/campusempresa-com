Introducció

El Job Control Language (JCL) s'utilitza en sistemes mainframe d'IBM per instruir el sistema sobre com executar treballs batch o iniciar subsistemes. Seguir les millors pràctiques en JCL assegura scripts de treball eficients, fiables i mantenibles.

Millors Pràctiques Generals

Utilitza Noms Significatius

  • Noms de Treball: Utilitza noms de treball descriptius per identificar fàcilment la finalitat del treball.
  • Noms de Pas: Cada pas dins d'un treball hauria de tenir un nom significatiu per descriure la seva funció.
//BACKUP01 JOB (ACCT),'Backup Job',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP01   EXEC PGM=IEBGENER

Comenta el Teu Codi

  • Comentaris: Utilitza comentaris per explicar la finalitat del treball, cada pas i qualsevol lògica complexa.
  • Sintaxi: Els comentaris en JCL comencen amb //*.
//* Aquest treball realitza una còpia de seguretat diària de la base de dades
//BACKUP01 JOB (ACCT),'Backup Job',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//* Pas per copiar els fitxers de la base de dades a la ubicació de còpia de seguretat
//STEP01   EXEC PGM=IEBGENER

Format Consistent

  • Indentació: Utilitza una indentació consistent per a la llegibilitat.
  • Alineació: Alinea els paràmetres i valors per a la claredat.
//BACKUP01 JOB (ACCT),'Backup Job',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP01   EXEC PGM=IEBGENER
//SYSPRINT DD SYSOUT=*
//SYSIN    DD DUMMY
//SYSUT1   DD DSN=DATABASE.FILE,DISP=SHR
//SYSUT2   DD DSN=BACKUP.FILE,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(CYL,(50,10),RLSE)

Gestió Eficient de Recursos

Utilitza Paràmetres DISP Adequats

  • DISP: El paràmetre DISP controla la disposició dels datasets. Utilitza'l correctament per evitar pèrdua de dades o retenció innecessària de datasets.
//SYSUT2   DD DSN=BACKUP.FILE,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(CYL,(50,10),RLSE)

Optimitza l'Assignació de Datasets

  • Assignació d'Espai: Assigna només l'espai necessari per als datasets per evitar malbaratament de recursos.
  • Allibera Espai No Utilitzat: Utilitza el paràmetre RLSE per alliberar espai no utilitzat.
//SYSUT2   DD DSN=BACKUP.FILE,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(CYL,(50,10),RLSE)

Gestió d'Errors i Depuració

Utilitza Codis de Condició

  • Codis de Condició: Comprova els codis de condició per gestionar errors i controlar el flux del treball.
//STEP01   EXEC PGM=IEBGENER
//STEP02   EXEC PGM=ANOTHERPGM,COND=(0,NE,STEP01)

Inclou Passos de Diagnòstic

  • Passos de Diagnòstic: Afegeix passos per capturar informació de diagnòstic en cas de fallades.
//STEP01   EXEC PGM=IEBGENER
//SYSPRINT DD SYSOUT=*
//SYSIN    DD DUMMY
//SYSUT1   DD DSN=DATABASE.FILE,DISP=SHR
//SYSUT2   DD DSN=BACKUP.FILE,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(CYL,(50,10),RLSE)
//*
//DIAGSTEP EXEC PGM=IEBGENER,COND=(0,NE,STEP01)
//SYSPRINT DD SYSOUT=*
//SYSIN    DD DUMMY
//SYSUT1   DD DSN=DATABASE.FILE,DISP=SHR
//SYSUT2   DD SYSOUT=*

Consideracions de Seguretat

Protegeix Dades Sensibles

  • Mascara Informació Sensible: Evita codificar informació sensible com contrasenyes en els scripts JCL.
  • Utilitza Funcions de Seguretat: Utilitza les funcions de seguretat del sistema per protegir datasets i l'execució de treballs.
//STEP01   EXEC PGM=IEBGENER
//SYSPRINT DD SYSOUT=*
//SYSIN    DD DUMMY
//SYSUT1   DD DSN=DATABASE.FILE,DISP=SHR,PROTECT
//SYSUT2   DD DSN=BACKUP.FILE,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(CYL,(50,10),RLSE),PROTECT

Documentació i Manteniment

Manté la Documentació

  • Documentació del Treball: Mantén la documentació actualitzada amb els canvis en els scripts de treball.
  • Control de Versions: Utilitza sistemes de control de versions per fer un seguiment dels canvis i mantenir l'historial.

Revisió Regular

  • Revisions de Codi: Realitza revisions de codi regulars per assegurar l'adherència a les millors pràctiques.
  • Revisions de Rendiment: Revisa periòdicament el rendiment del treball i optimitza segons sigui necessari.

Conclusió

Seguir les millors pràctiques en JCL assegura que els teus scripts de treball siguin eficients, mantenibles i segurs. Utilitzant noms significatius, comentant el teu codi, gestionant els recursos eficientment, gestionant els errors adequadament, protegint dades sensibles i mantenint una documentació exhaustiva, pots crear scripts JCL robustos i fiables. Les revisions i actualitzacions regulars ajudaran a mantenir els teus scripts JCL en condicions òptimes.
