La Infraestructura com a Codi (IaC) és un concepte clau en les pràctiques modernes de DevOps, que permet l'automatització i la gestió de la infraestructura a través del codi. Aquesta secció et guiarà pels fonaments de la IaC, centrant-se en Terraform, una eina popular de IaC.

Què és la Infraestructura com a Codi (IaC)?

La Infraestructura com a Codi és el procés de gestionar i proporcionar infraestructura informàtica a través d'arxius de definició llegibles per màquina, en lloc de mitjançant la configuració de maquinari físic o eines de configuració interactiva.

Conceptes Clau de IaC

  • Automatització: Automatitza la provisió i gestió de la infraestructura.
  • Control de Versions: Les configuracions d'infraestructura es poden versionar i emmagatzemar en repositoris.
  • Consistència: Assegura que els entorns siguin consistents i reproduïbles.
  • Escalabilitat: Facilita l'escalat de la infraestructura segons sigui necessari.
  • Documentació: El codi serveix com a documentació per a la configuració de la infraestructura.

Introducció a Terraform

Terraform és una eina de IaC de codi obert creada per HashiCorp. Et permet definir i proporcionar infraestructura de centres de dades utilitzant un llenguatge de configuració d'alt nivell.

Característiques Clau de Terraform

  • Llenguatge Declaratiu: Utilitza el HashiCorp Configuration Language (HCL) per descriure l'estat desitjat de la infraestructura.
  • Suport de Proveïdors: Admet una àmplia gamma de proveïdors de núvol com AWS, Azure, GCP, i més.
  • Gestió d'Estat: Manté un seguiment de l'estat de la teva infraestructura.
  • Planificar i Aplicar: Et permet previsualitzar els canvis abans d'aplicar-los.

Començar amb Terraform

Instal·lar Terraform

Per instal·lar Terraform, segueix aquests passos:

  1. Descarrega el paquet adequat per al teu sistema operatiu des del lloc web de Terraform.
  2. Descomprimeix el paquet i mou el binari terraform a un directori inclòs en el PATH del teu sistema.

Escriure la Teva Primera Configuració de Terraform

Creem una configuració simple de Terraform per proporcionar una instància EC2 d'AWS.

  1. Crea un nou directori per al teu projecte de Terraform:

    mkdir my-terraform-project
cd my-terraform-project

  2. Crea un arxiu de configuració anomenat main.tf:

    provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "example-instance"
  }
}

  3. Inicialitza el projecte:

    terraform init

  4. Previsualitza els canvis:

    terraform plan

  5. Aplica la configuració:

    terraform apply

Conceptes Avançats de Terraform

Mòduls

Els mòduls són configuracions reutilitzables que es poden compartir i utilitzar en diferents projectes.

  • Crear un Mòdul: Organitza els teus arxius de configuració en una estructura de directoris.
  • Utilitzar un Mòdul: Referencia el mòdul en el teu arxiu de configuració principal.

Exemple:

module "vpc" {
  source = "./modules/vpc"

  cidr_block = "10.0.0.0/16"
}

Gestió d'Estat

Terraform utilitza un arxiu d'estat per mantenir un seguiment dels recursos que gestiona.

  • Arxiu d'Estat: Un arxiu JSON que mapatge els recursos del món real a la teva configuració.
  • Estat Remot: Emmagatzema el teu arxiu d'estat remotament per a col·laboració i seguretat.

Exemple de configuració d'estat remot:

terraform {
  backend "s3" {
    bucket = "my-terraform-state"
    key    = "state/terraform.tfstate"
    region = "us-west-2"
  }
}

Espais de Treball

Els espais de treball et permeten gestionar múltiples entorns (per exemple, desenvolupament, preproducció, producció) dins de la mateixa configuració.

  • Crear un Espai de Treball:

    terraform workspace new dev

  • Canviar d'Espai de Treball:

    terraform workspace select dev
