Introducció

El monitoratge i el registre són components essencials de la gestió d'infraestructura. Ajuden a fer un seguiment del rendiment, la disponibilitat i la salut dels teus recursos. Terraform, com a eina d'Infraestructura com a Codi (IaC), et permet automatitzar la configuració del monitoratge i el registre per a la teva infraestructura.

Conceptes clau

  • Monitoratge: El procés de recollir, analitzar i utilitzar informació per fer un seguiment del rendiment i la salut de la teva infraestructura.
  • Registre: La pràctica de registrar esdeveniments que succeeixen dins de la teva infraestructura per ajudar a diagnosticar problemes i entendre el comportament del sistema.
  • Proveïdors de Terraform: Plugins que permeten a Terraform interactuar amb proveïdors de núvol i altres serveis.
  • Recursos de Terraform: Els blocs bàsics de les configuracions de Terraform, que representen components d'infraestructura.

Configuració del monitoratge amb Terraform

Utilitzant AWS CloudWatch

AWS CloudWatch és un servei de monitoratge i observabilitat que proporciona dades i informació accionable per als recursos d'AWS.

Exemple: Creació d'una alarma de CloudWatch

provider "aws" {
  region = "us-west-2"
}

resource "aws_cloudwatch_metric_alarm" "cpu_alarm" {
  alarm_name          = "high_cpu_usage"
  comparison_operator = "GreaterThanThreshold"
  evaluation_periods  = "2"
  metric_name         = "CPUUtilization"
  namespace           = "AWS/EC2"
  period              = "120"
  statistic           = "Average"
  threshold           = "80"
  alarm_description   = "This metric monitors high CPU usage"
  dimensions = {
    InstanceId = "i-1234567890abcdef0"
  }
  alarm_actions = [
    "arn:aws:sns:us-west-2:123456789012:my-sns-topic"
  ]
}
  • Explicació:
    • provider "aws": Configura el proveïdor d'AWS.
    • resource "aws_cloudwatch_metric_alarm": Defineix un recurs d'alarma de CloudWatch.
    • alarm_name: El nom de l'alarma.
    • comparison_operator: La condició per activar l'alarma.
    • evaluation_periods: El nombre de períodes durant els quals es compara la dada.
    • metric_name: El nom de la mètrica a monitoritzar.
    • namespace: L'espai de noms de la mètrica.
    • period: El període en segons durant el qual s'aplica l'estadística especificada.
    • statistic: L'estadística que s'aplica a la mètrica associada a l'alarma.
    • threshold: El valor contra el qual es compara l'estadística especificada.
    • dimensions: Parells clau-valor que defineixen les dimensions per a la mètrica.
    • alarm_actions: Les accions a executar quan aquesta alarma transita a un estat d'ALARMA.

Configuració del registre amb Terraform

Utilitzant AWS CloudWatch Logs

AWS CloudWatch Logs et permet monitoritzar, emmagatzemar i accedir a fitxers de registre dels recursos d'AWS.

Exemple: Creació d'un grup de registres de CloudWatch i un flux de registres

provider "aws" {
  region = "us-west-2"
}

resource "aws_cloudwatch_log_group" "example" {
  name              = "example-log-group"
  retention_in_days = 14
}

resource "aws_cloudwatch_log_stream" "example" {
  name           = "example-log-stream"
  log_group_name = aws_cloudwatch_log_group.example.name
}
  • Explicació:
    • resource "aws_cloudwatch_log_group": Defineix un grup de registres de CloudWatch.
    • name: El nom del grup de registres.
    • retention_in_days: El nombre de dies per retenir els esdeveniments de registre en el grup de registres especificat.
    • resource "aws_cloudwatch_log_stream": Defineix un flux de registres de CloudWatch.
    • log_group_name: El nom del grup de registres per associar amb el flux de registres.

Monitoratge i registre avançats

Combinació de monitoratge i registre

La combinació de monitoratge i registre pot proporcionar una visió completa de la salut i el rendiment de la teva infraestructura.

Exemple: Creació d'un tema SNS per a notificacions d'alarma

resource "aws_sns_topic" "example" {
  name = "example-sns-topic"
}

resource "aws_sns_topic_subscription" "example" {
  topic_arn = aws_sns_topic.example.arn
  protocol  = "email"
  endpoint  = "example@example.com"
}

resource "aws_cloudwatch_metric_alarm" "cpu_alarm" {
  alarm_name          = "high_cpu_usage"
  comparison_operator = "GreaterThanThreshold"
  evaluation_periods  = "2"
  metric_name         = "CPUUtilization"
  namespace           = "AWS/EC2"
  period              = "120"
  statistic           = "Average"
  threshold           = "80"
  alarm_description   = "This metric monitors high CPU usage"
  dimensions = {
    InstanceId = "i-1234567890abcdef0"
  }
  alarm_actions = [
    aws_sns_topic.example.arn
  ]
}
  • Explicació:
    • resource "aws_sns_topic": Defineix un tema SNS.
    • resource "aws_sns_topic_subscription": Subscriu un punt final al tema SNS.
    • alarm_actions: Especifica l'ARN del tema SNS per notificar quan es desencadena l'estat de l'alarma.

Conclusió

El monitoratge i el registre són crítics per mantenir la salut i el rendiment de la teva infraestructura. Terraform simplifica el procés de configuració d'aquests serveis permetent-te definir-los com a codi. Integrant el monitoratge i el registre a les teves configuracions de Terraform, pots assegurar-te que la teva infraestructura està ben monitoritzada i que els problemes es tracten ràpidament.
