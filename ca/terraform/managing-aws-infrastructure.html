Introducció

Terraform és una eina de codi obert d'infraestructura com a codi (IaC) que et permet definir i provisionar la infraestructura del centre de dades utilitzant un llenguatge de configuració d'alt nivell. Aquest tema et guiarà a través de la gestió de la infraestructura d'AWS utilitzant Terraform, des de nivells principiants fins a avançats.

Començar amb Terraform i AWS

Prerequisits

  • Comprensió bàsica dels serveis d'AWS.
  • Compte d'AWS amb els permisos adequats.
  • Terraform instal·lat a la teva màquina local.

Configurar Terraform

  1. Instal·lar Terraform: Segueix les instruccions al lloc web de Terraform per instal·lar Terraform al teu sistema operatiu.
  2. Configurar AWS CLI: Assegura't de tenir l'AWS CLI instal·lat i configurat amb les teves credencials d'AWS. 
    aws configure

La teva primera configuració de Terraform

  1. Crear un directori: Crea un nou directori per als teus fitxers de configuració de Terraform. 
    mkdir terraform-aws-demo
cd terraform-aws-demo
  2. Escriure una configuració bàsica: Crea un fitxer anomenat main.tf i afegeix el següent contingut per provisionar una instància EC2 d'AWS. 
    provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "TerraformExample"
  }
}
  3. Inicialitzar Terraform: Inicialitza el directori per descarregar els connectors necessaris del proveïdor. 
    terraform init
  4. Aplicar la configuració: Aplica la configuració per crear els recursos. 
    terraform apply

Conceptes intermedis

Gestionar l'estat

  • Fitxer d'estat: Terraform utilitza un fitxer d'estat per fer un seguiment dels recursos que gestiona.
  • Estat remot: Emmagatzema el fitxer d'estat remotament per a la col·laboració i la seguretat. 
    terraform {
  backend "s3" {
    bucket = "my-terraform-state"
    key    = "path/to/my/key"
    region = "us-west-2"
  }
}

Variables i sortides

  • Variables: Utilitza variables per fer les teves configuracions més flexibles. 
    variable "instance_type" {
  description = "Tipus d'instància a crear"
  default     = "t2.micro"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = var.instance_type

  tags = {
    Name = "TerraformExample"
  }
}
  • Sortides: Defineix sortides per extreure informació sobre els teus recursos. 
    output "instance_id" {
  value = aws_instance.example.id
}

Mòduls

  • Crear mòduls: Organitza la teva configuració en mòduls reutilitzables. 
    module "ec2_instance" {
  source       = "./modules/ec2"
  instance_type = "t2.micro"
  ami           = "ami-0c55b159cbfafe1f0"
}

Conceptes avançats

Gestionar infraestructura complexa

  • VPC i subxarxes: Crea una VPC amb subxarxes. 
    resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"
}

resource "aws_subnet" "subnet1" {
  vpc_id     = aws_vpc.main.id
  cidr_block = "10.0.1.0/24"
}

Espais de treball de Terraform

  • Espais de treball: Utilitza espais de treball per gestionar diferents entorns. 
    terraform workspace new dev
terraform workspace select dev

Millors pràctiques de Terraform

  • Control de versions: Mantingues les teves configuracions de Terraform en sistemes de control de versions com Git.
  • Gestió de l'estat: Fes còpies de seguretat regulars dels teus fitxers d'estat i utilitza emmagatzematge d'estat remot.
  • Revisions de codi: Implementa revisions de codi per a les configuracions de Terraform per assegurar la qualitat i la seguretat.

Conclusió

Gestionar la infraestructura d'AWS amb Terraform permet una gestió eficient, repetible i escalable de la infraestructura. Progressant des de configuracions bàsiques fins a conceptes avançats com mòduls i espais de treball, pots gestionar entorns complexos amb facilitat. Segueix sempre les millors pràctiques per assegurar que la teva infraestructura sigui segura i mantenible.
