Introducció a Terraform Cloud
Configurant Terraform Cloud ►

Terraform Cloud és un servei gestionat proporcionat per HashiCorp que ofereix un entorn col·laboratiu per a equips per gestionar la seva infraestructura com a codi. Aquesta secció us introduirà a Terraform Cloud, les seves característiques i com començar.

Conceptes Clau

  • Terraform Cloud: Un servei gestionat que proporciona un entorn col·laboratiu per a equips per utilitzar Terraform.
  • Workspaces: Entorns aïllats dins de Terraform Cloud on podeu gestionar una col·lecció de recursos d'infraestructura.
  • Gestió d'Estat Remot: Terraform Cloud emmagatzema els vostres fitxers d'estat remotament, assegurant la consistència i la col·laboració.
  • Integració amb VCS: Integració amb Sistemes de Control de Versions (VCS) com GitHub, GitLab i Bitbucket per a execucions automàtiques de Terraform.
  • Triggers d'Execució: Activar automàticament execucions de Terraform basades en canvis en el vostre VCS.
  • Polítiques Sentinel: Marc de polítiques com a codi per fer complir la conformitat i la governança.

Característiques de Terraform Cloud

  • Col·laboració: Diversos membres de l'equip poden treballar en la mateixa infraestructura.
  • Operacions Remotes: Les operacions de Terraform s'executen remotament, reduint la necessitat d'execució local.
  • Gestió d'Estat: Emmagatzematge d'estat segur i consistent.
  • Aplicació de Polítiques: Utilitzeu Sentinel per fer complir polítiques i la conformitat.
  • Estimació de Costos: Estimeu el cost dels vostres canvis d'infraestructura abans d'aplicar-los.
  • Notificacions: Rebeu notificacions sobre l'estat de les vostres execucions.

Començar amb Terraform Cloud

Pas 1: Registrar-se a Terraform Cloud

  1. Visiteu el lloc web de Terraform Cloud.
  2. Registreu-vos per obtenir un compte gratuït o inicieu sessió si ja en teniu un.

Pas 2: Crear una Organització

  1. Després d'iniciar sessió, creeu una nova organització.
  2. Proporcioneu un nom per a la vostra organització i feu clic a "Crear Organització".

Pas 3: Crear un Workspace

  1. Dins de la vostra organització, creeu un nou workspace.
  2. Trieu un nom per al vostre workspace i seleccioneu el repositori VCS que conté els vostres fitxers de configuració de Terraform.

Pas 4: Configurar la Integració amb VCS

  1. Connecteu el vostre proveïdor de VCS (per exemple, GitHub, GitLab).
  2. Autoritzeu Terraform Cloud per accedir al vostre repositori.
  3. Seleccioneu el repositori i la branca que conté la vostra configuració de Terraform.

Pas 5: Executar Terraform

  1. Empenyeu canvis al vostre repositori VCS.
  2. Terraform Cloud detectarà automàticament els canvis i iniciarà una execució.
  3. Reviseu el pla i apliqueu els canvis si tot sembla correcte.

Exemple: Configuració Bàsica de Terraform Cloud

Aquí teniu un exemple senzill d'una configuració de Terraform que aprovisiona una instància EC2 d'AWS. Aquesta configuració s'emmagatzemarà al vostre repositori VCS i serà gestionada per Terraform Cloud.

provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "TerraformExample"
  }
}

Explicació

  • provider "aws": Configura el proveïdor d'AWS amb la regió especificada.
  • resource "aws_instance" "example": Defineix un recurs d'instància EC2 d'AWS.
    • ami: La ID de la Imatge de Màquina d'Amazon (AMI) a utilitzar per a la instància.
    • instance_type: El tipus d'instància a llançar.
    • tags: Etiquetes per assignar a la instància.

Conclusió

Terraform Cloud proporciona una plataforma potent per gestionar la vostra infraestructura com a codi amb col·laboració millorada, operacions remotes i aplicació de polítiques. Integrant-se amb el vostre VCS, Terraform Cloud automatitza el procés d'aplicar canvis d'infraestructura, assegurant la consistència i reduint el risc d'errors. A les properes seccions, aprofundirem en les característiques avançades i les millors pràctiques per utilitzar Terraform Cloud de manera efectiva.
