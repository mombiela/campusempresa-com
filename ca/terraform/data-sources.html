Fonts de dades
Espais de Treball de Terraform ►

En aquesta secció, explorarem el concepte de fonts de dades a Terraform, com es poden utilitzar per obtenir informació de sistemes externs i com integrar-les a les teves configuracions de Terraform. Les fonts de dades són una característica poderosa que et permet consultar i utilitzar dades que són externes a la teva configuració de Terraform.

Què són les fonts de dades?

Les fonts de dades a Terraform et permeten obtenir dades de sistemes externs i utilitzar-les a les teves configuracions de Terraform. Això pot incloure informació de proveïdors de núvol, bases de dades o altres serveis. Les fonts de dades són només de lectura i no creen ni modifiquen infraestructura.

Conceptes clau

  • Només de lectura: Les fonts de dades s'utilitzen per llegir informació i no es poden utilitzar per crear o modificar recursos.
  • Sistemes externs: Les fonts de dades poden obtenir dades de diversos sistemes externs com AWS, Azure, Google Cloud, i més.
  • Integració: Les dades obtingudes de les fonts de dades es poden utilitzar per configurar altres recursos a la teva configuració de Terraform.

Ús de fonts de dades

Per utilitzar una font de dades a Terraform, cal definir-la al teu fitxer de configuració. Aquí tens un exemple bàsic de com utilitzar una font de dades per obtenir informació sobre una VPC d'AWS.

Exemple: Obtenint informació d'una VPC d'AWS

provider "aws" {
  region = "us-west-2"
}

data "aws_vpc" "example" {
  filter {
    name   = "tag:Name"
    values = ["my-vpc"]
  }
}

output "vpc_id" {
  value = data.aws_vpc.example.id
}

En aquest exemple:

  • Definim el proveïdor d'AWS i especifiquem la regió.
  • Utilitzem la font de dades aws_vpc per obtenir informació sobre una VPC amb l'etiqueta Name establerta a my-vpc.
  • Mostrem l'ID de la VPC utilitzant el bloc output.

Fonts de dades comunes

Aquí tens algunes fonts de dades comunes que podries utilitzar a Terraform:

  • AWS: aws_instance, aws_vpc, aws_subnet, etc.
  • Azure: azurerm_resource_group, azurerm_virtual_network, etc.
  • Google Cloud: google_compute_instance, google_compute_network, etc.

Exemple: Obtenint informació d'un grup de recursos d'Azure

provider "azurerm" {
  features {}
}

data "azurerm_resource_group" "example" {
  name = "my-resource-group"
}

output "resource_group_location" {
  value = data.azurerm_resource_group.example.location
}

En aquest exemple:

  • Definim el proveïdor d'Azure.
  • Utilitzem la font de dades azurerm_resource_group per obtenir informació sobre un grup de recursos anomenat my-resource-group.
  • Mostrem la ubicació del grup de recursos.

Ús avançat

Les fonts de dades també es poden utilitzar en escenaris més complexos, com ara enllaçar múltiples fonts de dades o utilitzar-les en expressions condicionals.

Exemple: Enllaçant fonts de dades

provider "aws" {
  region = "us-west-2"
}

data "aws_vpc" "example" {
  filter {
    name   = "tag:Name"
    values = ["my-vpc"]
  }
}

data "aws_subnet_ids" "example" {
  vpc_id = data.aws_vpc.example.id
}

output "subnet_ids" {
  value = data.aws_subnet_ids.example.ids
}

En aquest exemple:

  • Primer obtenim la informació de la VPC utilitzant la font de dades aws_vpc.
  • Després utilitzem l'ID de la VPC per obtenir els IDs de les subxarxes dins d'aquesta VPC utilitzant la font de dades aws_subnet_ids.
  • Finalment, mostrem la llista d'IDs de les subxarxes.

Conclusió

Les fonts de dades a Terraform són una característica poderosa que et permet obtenir i utilitzar dades de sistemes externs a les teves configuracions. Són només de lectura i es poden utilitzar per integrar informació de diversos proveïdors de núvol i serveis. Entenent com utilitzar les fonts de dades, pots crear configuracions de Terraform més dinàmiques i flexibles.

En aquesta secció, hem cobert:

  • El concepte de fonts de dades.
  • Com utilitzar fonts de dades amb exemples d'AWS i Azure.
  • Escenaris d'ús avançat com enllaçar fonts de dades.

Amb aquest coneixement, pots començar a incorporar fonts de dades als teus projectes de Terraform per fer-los més robustos i adaptables.
