Introducció

Terraform és una eina de codi obert d'infraestructura com a codi (IaC) que et permet definir i provisionar la infraestructura del centre de dades utilitzant un llenguatge de configuració declaratiu. Aquesta secció cobrirà quatre comandes essencials de Terraform: init, plan, apply, i destroy. Aquestes comandes són fonamentals per gestionar el cicle de vida de la teva infraestructura amb Terraform.

Terraform Init

La comanda terraform init inicialitza un directori de treball de Terraform. Aquesta és la primera comanda que s'ha de executar després d'escriure una nova configuració de Terraform o clonar-ne una d'existent des del control de versions.

Conceptes Clau

  • Inicialització: Prepara el directori de treball per a altres comandes de Terraform.
  • Configuració del Backend: Configura el backend on es guarda l'estat de Terraform.
  • Plugins de Proveïdors: Descarrega els plugins de proveïdors necessaris.

Exemple

$ terraform init

Sortida:

Initializing the backend...

Initializing provider plugins...
- Finding latest version of hashicorp/aws...
- Installing hashicorp/aws v3.27.0...
- Installed hashicorp/aws v3.27.0 (signed by HashiCorp)

Terraform has been successfully initialized!

Terraform Plan

La comanda terraform plan crea un pla d'execució, mostrant quines accions prendrà Terraform per aconseguir l'estat desitjat definit als teus fitxers de configuració.

Conceptes Clau

  • Pla d'Execució: Una vista prèvia dels canvis que farà Terraform.
  • Sense Canvis Aplicats: La comanda no fa cap canvi; només mostra què passarà.

Exemple

$ terraform plan

Sortida:

An execution plan has been generated and is shown below.
Resource actions are indicated with the following symbols:
  + create

Terraform will perform the following actions:

  # aws_instance.example will be created
  + resource "aws_instance" "example" {
      + ami           = "ami-0c55b159cbfafe1f0"
      + instance_type = "t2.micro"
    }

Plan: 1 to add, 0 to change, 0 to destroy.

Terraform Apply

La comanda terraform apply aplica els canvis necessaris per aconseguir l'estat desitjat de la configuració. Executa les accions proposades en el pla d'execució.

Conceptes Clau

  • Execució: Aplica els canvis a la infraestructura.
  • Confirmació: Pot requerir confirmació de l'usuari abans d'aplicar els canvis.

Exemple

$ terraform apply

Sortida:

An execution plan has been generated and is shown below.
Resource actions are indicated with the following symbols:
  + create

Terraform will perform the following actions:

  # aws_instance.example will be created
  + resource "aws_instance" "example" {
      + ami           = "ami-0c55b159cbfafe1f0"
      + instance_type = "t2.micro"
    }

Plan: 1 to add, 0 to change, 0 to destroy.

Do you want to perform these actions?
  Terraform will perform the actions described above.
  Only 'yes' will be accepted to approve.

  Enter a value: yes

aws_instance.example: Creating...
aws_instance.example: Creation complete after 10s [id=i-0abcd1234efgh5678]

Apply complete! Resources: 1 added, 0 changed, 0 destroyed.

Terraform Destroy

La comanda terraform destroy s'utilitza per destruir la infraestructura gestionada per Terraform. És l'oposat de terraform apply.

Conceptes Clau

  • Destrucció: Elimina tots els recursos definits a la configuració.
  • Confirmació: Requereix confirmació de l'usuari abans de destruir els recursos.

Exemple

$ terraform destroy

Sortida:

aws_instance.example: Refreshing state... [id=i-0abcd1234efgh5678]

An execution plan has been generated and is shown below.
Resource actions are indicated with the following symbols:
  - destroy

Terraform will perform the following actions:

  # aws_instance.example will be destroyed
  - resource "aws_instance" "example" {
      - ami           = "ami-0c55b159cbfafe1f0"
      - instance_type = "t2.micro"
    }

Plan: 0 to add, 0 to change, 1 to destroy.

Do you really want to destroy all resources?
  Terraform will destroy all your managed infrastructure, as shown above.
  There is no undo. Only 'yes' will be accepted to confirm.

  Enter a value: yes

aws_instance.example: Destroying... [id=i-0abcd1234efgh5678]
aws_instance.example: Destruction complete after 5s

Destroy complete! Resources: 1 destroyed.

Resum

Entendre les comandes bàsiques de Terraform init, plan, apply, i destroy és crucial per gestionar el cicle de vida de la teva infraestructura. Aquestes comandes t'ajuden a inicialitzar el teu directori de treball, previsualitzar canvis, aplicar configuracions i destruir recursos quan ja no són necessaris. La mestria d'aquestes comandes et permetrà gestionar i automatitzar efectivament la teva infraestructura utilitzant Terraform.
