La gestió de secrets és un aspecte crític de la infraestructura com a codi (IaC) per assegurar que la informació sensible com ara claus API, contrasenyes i tokens es manegin de manera segura. Aquesta secció et guiarà a través de les millors pràctiques i eines per gestionar secrets a Terraform des d'un nivell principiant fins a un nivell avançat.

Introducció a la Gestió de Secrets

  • Què són els Secrets?

    • Els secrets inclouen informació sensible com claus API, contrasenyes, tokens i altres credencials.

  • Per què és Important la Gestió de Secrets?

    • Prevenir l'accés no autoritzat.
    • Assegurar el compliment de les polítiques de seguretat.
    • Protegir la informació sensible d'estar exposada.

Emmagatzemar Secrets de Manera Segura

Variables d'Entorn

  • Pros:

    • Fàcil d'utilitzar i integrar.
    • No es requereixen canvis en el codi de Terraform.

  • Contres:

    • Poden ser exposades si no es manegen adequadament.
    • No són adequades per a la gestió de secrets a gran escala.

Exemple:

export TF_VAR_db_password="mysecretpassword"
terraform apply

Variables de Terraform

  • Variables Sensibles:
    • Marca les variables com a sensibles per evitar que es mostrin als registres.

Exemple:

variable "db_password" {
  description = "La contrasenya per a la base de dades"
  type        = string
  sensitive   = true
}

resource "aws_db_instance" "example" {
  allocated_storage = 20
  engine            = "mysql"
  instance_class    = "db.t2.micro"
  name              = "example"
  username          = "admin"
  password          = var.db_password
}

Ús d'Eines de Gestió de Secrets

HashiCorp Vault

  • Característiques:
    • Emmagatzematge segur per a secrets.
    • Generació dinàmica de secrets.
    • Polítiques de control d'accés.

Exemple:

  1. Emmagatzemar un Secret a Vault:
vault kv put secret/db_password value="mysecretpassword"
  1. Recuperar un Secret a Terraform:
provider "vault" {
  address = "http://127.0.0.1:8200"
}

data "vault_generic_secret" "db_password" {
  path = "secret/db_password"
}

resource "aws_db_instance" "example" {
  allocated_storage = 20
  engine            = "mysql"
  instance_class    = "db.t2.micro"
  name              = "example"
  username          = "admin"
  password          = data.vault_generic_secret.db_password.data["value"]
}

AWS Secrets Manager

  • Característiques:
    • Gestió centralitzada de secrets.
    • Rotació automàtica de secrets.
    • Integrat amb AWS IAM per al control d'accés.

Exemple:

  1. Emmagatzemar un Secret a AWS Secrets Manager:
aws secretsmanager create-secret --name MySecret --secret-string "mysecretpassword"
  1. Recuperar un Secret a Terraform:
provider "aws" {
  region = "us-west-2"
}

data "aws_secretsmanager_secret" "example" {
  name = "MySecret"
}

data "aws_secretsmanager_secret_version" "example" {
  secret_id = data.aws_secretsmanager_secret.example.id
}

resource "aws_db_instance" "example" {
  allocated_storage = 20
  engine            = "mysql"
  instance_class    = "db.t2.micro"
  name              = "example"
  username          = "admin"
  password          = data.aws_secretsmanager_secret_version.example.secret_string
}

Millors Pràctiques per a la Gestió de Secrets

  • Evitar Codificar Secrets:

    • Mai codifiquis secrets directament als teus fitxers de configuració de Terraform.

  • Ús de Control de Versions Ignorant:

    • Afegeix fitxers sensibles a .gitignore per evitar que es comprometin al control de versions.

  • Rotar Secrets Regularment:

    • Actualitza i rota regularment els secrets per minimitzar el risc d'exposició.

  • Control d'Accés:

    • Utilitza polítiques IAM i rols per controlar l'accés als secrets.

Conclusió

Gestionar secrets a Terraform és essencial per mantenir la seguretat i integritat de la teva infraestructura. Utilitzant variables d'entorn, variables de Terraform i eines de gestió de secrets com HashiCorp Vault i AWS Secrets Manager, pots manejar informació sensible de manera segura. Segueix sempre les millors pràctiques per assegurar que els teus secrets estiguin protegits i revisats regularment.

Entenent i implementant aquestes tècniques, pots millorar la seguretat de la teva infraestructura gestionada per Terraform i reduir el risc d'accés no autoritzat a dades sensibles.
