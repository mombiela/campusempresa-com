L'auditoria i el registre són aspectes crucials de la gestió de la infraestructura com a codi (IaC) amb Terraform. Ajuden a fer el seguiment dels canvis, entendre l'estat de la teva infraestructura i diagnosticar problemes. Aquest tema et guiarà des dels conceptes bàsics fins als avançats d'auditoria i registre en Terraform.

Introducció a l'auditoria i el registre

L'auditoria i el registre en Terraform impliquen:

  • Seguiment de canvis: Monitoritzar qui ha fet canvis, quins canvis s'han fet i quan.
  • Entendre l'estat: Mantenir un registre de l'estat actual de la teva infraestructura.
  • Diagnosticar problemes: Identificar i resoldre problemes analitzant els registres.

Estat de Terraform i fitxers d'estat

Terraform utilitza fitxers d'estat per fer el seguiment de la infraestructura que gestiona. Aquests fitxers d'estat són essencials per a l'auditoria i el registre.

  • Fitxer d'estat: Un fitxer JSON que registra l'estat actual de la teva infraestructura.
  • Backend: Una ubicació d'emmagatzematge per als teus fitxers d'estat (per exemple, local, S3, Azure Blob Storage).

Exemple: Fitxer d'estat local

terraform {
  backend "local" {
    path = "terraform.tfstate"
  }
}

Exemple: Fitxer d'estat remot (S3 Backend)

terraform {
  backend "s3" {
    bucket = "my-terraform-state"
    key    = "path/to/my/key"
    region = "us-west-2"
  }
}

Activació del registre detallat

Terraform proporciona diversos nivells de registre per ajudar-te a entendre què està passant durant l'execució de les teves configuracions.

  • TRACE: Registres molt detallats, incloent operacions internes.
  • DEBUG: Registres detallats útils per a la depuració.
  • INFO: Informació general sobre l'execució.
  • WARN: Advertències sobre possibles problemes.
  • ERROR: Errors que han ocorregut durant l'execució.

Configuració dels nivells de registre

Pots configurar el nivell de registre utilitzant la variable d'entorn TF_LOG.

export TF_LOG=DEBUG
terraform apply

Traçabilitat d'auditoria amb Terraform Cloud/Enterprise

Terraform Cloud i Terraform Enterprise ofereixen capacitats d'auditoria millorades, incloent registres detallats i historial de canvis.

  • Historial d'execucions: Registres detallats de totes les execucions, incloent qui les ha iniciat i els seus resultats.
  • Versions d'estat: Historial de versions del fitxer d'estat, permetent veure els canvis al llarg del temps.
  • Polítiques Sentinel: Polítiques que imposen regles i proporcionen registres d'auditoria quan es violen les regles.

Exemple: Veure l'historial d'execucions

A Terraform Cloud, navega al teu espai de treball i selecciona la pestanya "Runs" per veure l'historial d'execucions.

Millors pràctiques per a l'auditoria i el registre

  • Utilitza estat remot: Emmagatzema els fitxers d'estat en un backend remot per assegurar que són segurs i accessibles.
  • Activa el registre detallat: Utilitza nivells de registre adequats durant el desenvolupament i la depuració.
  • Revisa regularment els registres: Revisa periòdicament els registres i els canvis d'estat per assegurar el compliment i detectar problemes aviat.
  • Implementa polítiques Sentinel: Utilitza polítiques Sentinel a Terraform Cloud/Enterprise per imposar regles de compliment i seguretat.

Conclusió

L'auditoria i el registre són essencials per mantenir la integritat i la seguretat de la teva infraestructura gestionada per Terraform. Entenent i implementant els conceptes discutits, pots assegurar que els canvis a la teva infraestructura es facin seguiment, els problemes es diagnostiquin eficientment i es mantingui el compliment.

En resum:

  • Utilitza fitxers d'estat i backends per fer el seguiment de l'estat de la infraestructura.
  • Activa i configura el registre detallat per obtenir millor visibilitat.
  • Aprofita les capacitats d'auditoria avançades de Terraform Cloud/Enterprise.
  • Segueix les millors pràctiques per mantenir un sistema robust d'auditoria i registre.
