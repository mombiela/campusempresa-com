Introducció

En aquesta secció, explorarem com utilitzar variables i sortides a Terraform. Les variables permeten parametritzar les teves configuracions de Terraform, fent-les més flexibles i reutilitzables. Les sortides et permeten extreure i mostrar informació del teu estat de Terraform.

Variables

Què són les variables?

Les variables a Terraform s'utilitzen per parametritzar les teves configuracions. Et permeten definir valors que es poden reutilitzar a través dels teus fitxers de configuració.

Definint variables

Les variables es defineixen en un fitxer .tf utilitzant la paraula clau variable. Aquí tens un exemple:

variable "instance_type" {
  description = "Tipus d'instància EC2"
  type        = string
  default     = "t2.micro"
}
  • description: Una breu explicació de per a què serveix la variable.
  • type: El tipus de la variable (per exemple, string, number, list, map).
  • default: Un valor per defecte per a la variable si no se'n proporciona cap.

Utilitzant variables

Un cop definides, les variables es poden utilitzar en els teus fitxers de configuració de Terraform. Aquí tens un exemple de com utilitzar la variable instance_type:

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = var.instance_type
}

Tipus de variables

Terraform suporta diversos tipus de variables:

  • string: Un valor de cadena únic.
  • number: Un valor numèric.
  • bool: Un valor booleà (true o false).
  • list: Una llista de valors.
  • map: Un mapa de parells clau-valor.

Proporcionant valors per a les variables

Els valors per a les variables es poden proporcionar de diverses maneres:

  1. Flags de línia de comandes:

    terraform apply -var="instance_type=t2.large"

  2. Fitxer de definicions de variables (.tfvars):

    instance_type = "t2.large"

  3. Variables d'entorn:

    export TF_VAR_instance_type="t2.large"

Exemple: Utilitzant una variable de llista

Aquí tens un exemple de definició i ús d'una variable de llista:

variable "availability_zones" {
  description = "Llista de zones de disponibilitat"
  type        = list(string)
  default     = ["us-west-1a", "us-west-1b"]
}

resource "aws_instance" "example" {
  ami               = "ami-0c55b159cbfafe1f0"
  instance_type     = "t2.micro"
  availability_zone = var.availability_zones[0]
}

Sortides

Què són les sortides?

Les sortides a Terraform s'utilitzen per mostrar informació del teu estat de Terraform. Poden ser útils per a depuració, o per proporcionar informació a altres configuracions o usuaris.

Definint sortides

Les sortides es defineixen en un fitxer .tf utilitzant la paraula clau output. Aquí tens un exemple:

output "instance_id" {
  description = "L'ID de la instància EC2"
  value       = aws_instance.example.id
}
  • description: Una breu explicació de per a què serveix la sortida.
  • value: El valor que es mostrarà, que pot referenciar recursos i variables.

Utilitzant sortides

Les sortides es poden veure després d'executar terraform apply utilitzant la comanda terraform output:

terraform output instance_id

Exemple: Múltiples sortides

Aquí tens un exemple de definició de múltiples sortides:

output "instance_id" {
  description = "L'ID de la instància EC2"
  value       = aws_instance.example.id
}

output "instance_public_ip" {
  description = "La IP pública de la instància EC2"
  value       = aws_instance.example.public_ip
}

Resum

En aquesta secció, hem cobert els conceptes bàsics de les variables i les sortides a Terraform. Les variables permeten parametritzar les teves configuracions, fent-les més flexibles i reutilitzables. Les sortides et permeten extreure i mostrar informació del teu estat de Terraform. Entenent i utilitzant aquestes característiques, pots crear configuracions de Terraform més dinàmiques i informatives.

Punts clau

  • Les variables es poden definir, utilitzar i proporcionar valors de diverses maneres.
  • Les sortides ajuden a extreure i mostrar informació útil de l'estat de Terraform.
  • L'ús adequat de variables i sortides pot millorar significativament la flexibilitat i llegibilitat de les teves configuracions de Terraform.
