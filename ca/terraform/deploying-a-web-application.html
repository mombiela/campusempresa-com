Introducció

Desplegar una aplicació web implica diversos passos, des de la provisió d'infraestructura fins a la configuració de serveis i el desplegament de codi. Terraform és una eina potent que et permet gestionar la teva infraestructura com a codi, fent que els desplegaments siguin repetibles i gestionables. En aquest tema, cobrirem com desplegar una aplicació web utilitzant Terraform, començant pels conceptes bàsics i avançant cap a conceptes més avançats.

Conceptes Clau

  • Infraestructura com a Codi (IaC): Gestionar i proveir infraestructura informàtica a través d'arxius de configuració llegibles per màquines.
  • Proveïdors de Terraform: Plugins que permeten a Terraform interactuar amb proveïdors de núvol, proveïdors SaaS i altres APIs.
  • Estat de Terraform: Un arxiu que fa un seguiment dels recursos gestionats per Terraform.
  • Mòduls: Configuracions reutilitzables que es poden compartir i versionar.

Configuració de Terraform

Abans de desplegar una aplicació web, necessites configurar Terraform a la teva màquina local.

  1. Instal·lar Terraform:

    • Descarrega Terraform des del lloc web oficial.
    • Segueix les instruccions d'instal·lació per al teu sistema operatiu.

  2. Configurar un Proveïdor:

    • Crea un nou directori per als teus arxius de configuració de Terraform.
    • Crea un arxiu anomenat main.tf i configura el teu proveïdor de núvol (per exemple, AWS, Azure, GCP).
# main.tf
provider "aws" {
  region = "us-west-2"
}

Provisió d'Infraestructura

Per desplegar una aplicació web, necessites proveir la infraestructura necessària, com ara servidors, bases de dades i components de xarxa.

  1. Definir Recursos:
    • Utilitza Terraform per definir els recursos necessaris per a la teva aplicació web.
# main.tf
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "WebServer"
  }
}
  1. Inicialitzar i Aplicar:
    • Inicialitza Terraform per descarregar els proveïdors i mòduls necessaris.
    • Aplica la configuració per crear els recursos.
terraform init
terraform apply

Configuració de Serveis

Un cop la infraestructura està proveïda, necessites configurar els serveis necessaris per a la teva aplicació web.

  1. Grups de Seguretat:
    • Defineix grups de seguretat per controlar el trànsit entrant i sortint.
# main.tf
resource "aws_security_group" "web_sg" {
  name_prefix = "web-sg-"

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}
  1. Configuració de la Base de Dades:
    • Proveir i configurar una base de dades per a la teva aplicació web.
# main.tf
resource "aws_db_instance" "default" {
  allocated_storage    = 20
  engine               = "mysql"
  engine_version       = "5.7"
  instance_class       = "db.t2.micro"
  name                 = "mydb"
  username             = "foo"
  password             = "bar"
  parameter_group_name = "default.mysql5.7"
}

Desplegament del Codi de l'Aplicació

Després de configurar la infraestructura i els serveis, el següent pas és desplegar el codi de la teva aplicació.

  1. Provisió d'un Servidor Web:
    • Utilitza un provisioner per desplegar el codi de la teva aplicació al servidor web.
# main.tf
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  provisioner "file" {
    source      = "app.zip"
    destination = "/tmp/app.zip"
  }

  provisioner "remote-exec" {
    inline = [
      "unzip /tmp/app.zip -d /var/www/html",
      "sudo systemctl restart apache2"
    ]
  }

  tags = {
    Name = "WebServer"
  }
}

Conceptes Avançats

Per a desplegaments més complexos, pots utilitzar característiques avançades de Terraform com ara mòduls, espais de treball i estat remot.

  1. Mòduls:
    • Crea mòduls reutilitzables per organitzar la teva configuració de Terraform.
# modules/web_server/main.tf
resource "aws_instance" "web" {
  ami           = var.ami
  instance_type = var.instance_type

  tags = {
    Name = var.name
  }
}

# main.tf
module "web_server" {
  source        = "./modules/web_server"
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  name          = "WebServer"
}
  1. Espais de Treball:
    • Utilitza espais de treball per gestionar diferents entorns (per exemple, desenvolupament, preproducció, producció).
terraform workspace new development
terraform workspace select development
terraform apply
  1. Estat Remot:
    • Emmagatzema l'arxiu d'estat de Terraform remotament per permetre la col·laboració i el bloqueig d'estat.
# main.tf
terraform {
  backend "s3" {
    bucket = "my-terraform-state"
    key    = "state/terraform.tfstate"
    region = "us-west-2"
  }
}

Conclusió

Desplegar una aplicació web utilitzant Terraform implica diversos passos, des de la configuració de la infraestructura fins a la configuració de serveis i el desplegament de codi. Seguint l'enfocament estructurat descrit en aquest tema, pots gestionar els teus desplegaments de manera efectiva i aprofitar les característiques potents de Terraform. A mesura que et familiaritzis més amb Terraform, pots explorar conceptes avançats com ara mòduls, espais de treball i estat remot per optimitzar encara més el teu procés de desplegament.
