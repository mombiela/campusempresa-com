Proveïdors i Blocs de Connexió
Fonts de Dades ►

Els proveïdors i els blocs de connexió són components essencials a Terraform que et permeten executar scripts o comandes en els teus recursos i gestionar com Terraform es connecta a ells. Aquest tema cobrirà els conceptes bàsics dels proveïdors, com utilitzar els blocs de connexió i escenaris d'ús avançat.

Proveïdors

Els proveïdors a Terraform s'utilitzen per executar scripts o comandes en una màquina local o remota com a part del procés de creació o destrucció del recurs. Són útils per a l'arrencada, la gestió de configuració i altres tasques posteriors al desplegament.

Conceptes Clau

  • Tipus de Proveïdors:

    • local-exec: Executa una comanda a la màquina on s'executa Terraform.
    • remote-exec: Executa una comanda en un recurs remot.
    • file: Copia fitxers o directoris a un recurs remot.

  • Gallets del Cicle de Vida:

    • creation-time: S'executa durant la creació del recurs.
    • destroy-time: S'executa durant la destrucció del recurs.

Exemples

Proveïdor local-exec

El proveïdor local-exec executa una comanda localment a la màquina on s'executa Terraform.

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  provisioner "local-exec" {
    command = "echo ${aws_instance.example.public_ip} > ip_address.txt"
  }
}

En aquest exemple, després de crear la instància AWS, l'adreça IP pública de la instància s'escriu en un fitxer anomenat ip_address.txt a la màquina local.

Proveïdor remote-exec

El proveïdor remote-exec executa comandes en el recurs remot.

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  connection {
    type     = "ssh"
    user     = "ubuntu"
    private_key = file("~/.ssh/id_rsa")
    host     = self.public_ip
  }

  provisioner "remote-exec" {
    inline = [
      "sudo apt-get update",
      "sudo apt-get install -y nginx"
    ]
  }
}

En aquest exemple, després de crear la instància AWS, el proveïdor remote-exec es connecta a la instància via SSH i instal·la Nginx.

Blocs de Connexió

Els blocs de connexió defineixen com Terraform es connecta al recurs remot. S'utilitzen conjuntament amb els proveïdors per especificar detalls de connexió com el tipus de connexió, les credencials d'usuari i la informació de l'amfitrió.

Conceptes Clau

  • Tipus de Connexió:

    • ssh: Es connecta via SSH.
    • winrm: Es connecta via Windows Remote Management.

  • Atributs de Connexió:

    • type: El tipus de connexió (per exemple, ssh o winrm).
    • user: El nom d'usuari per a la connexió.
    • password: La contrasenya per a la connexió (si escau).
    • private_key: La clau privada per a connexions SSH.
    • host: El nom de l'amfitrió o l'adreça IP del recurs remot.

Exemple

Bloc de Connexió SSH

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  connection {
    type     = "ssh"
    user     = "ubuntu"
    private_key = file("~/.ssh/id_rsa")
    host     = self.public_ip
  }

  provisioner "remote-exec" {
    inline = [
      "sudo apt-get update",
      "sudo apt-get install -y nginx"
    ]
  }
}

En aquest exemple, el bloc de connexió especifica que Terraform s'ha de connectar a la instància AWS utilitzant SSH amb l'usuari i la clau privada proporcionats.

Ús Avançat

Múltiples Proveïdors

Pots utilitzar múltiples proveïdors dins d'un sol recurs per realitzar una sèrie de tasques.

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  connection {
    type     = "ssh"
    user     = "ubuntu"
    private_key = file("~/.ssh/id_rsa")
    host     = self.public_ip
  }

  provisioner "remote-exec" {
    inline = [
      "sudo apt-get update",
      "sudo apt-get install -y nginx"
    ]
  }

  provisioner "file" {
    source      = "index.html"
    destination = "/var/www/html/index.html"
  }
}

En aquest exemple, el primer proveïdor instal·la Nginx, i el segon proveïdor copia un fitxer index.html al directori arrel del servidor web.

Gestió d'Errors

Pots controlar el comportament dels proveïdors en cas d'errors utilitzant l'atribut on_failure.

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  connection {
    type     = "ssh"
    user     = "ubuntu"
    private_key = file("~/.ssh/id_rsa")
    host     = self.public_ip
  }

  provisioner "remote-exec" {
    inline = [
      "sudo apt-get update",
      "sudo apt-get install -y nginx"
    ]
    on_failure = "continue"
  }
}

En aquest exemple, si el proveïdor remote-exec falla, Terraform continuarà amb els següents passos en lloc d'aturar-se.

Resum

Els proveïdors i els blocs de connexió són eines poderoses a Terraform que et permeten executar scripts i comandes en els teus recursos, tant localment com remotament. Entenent com utilitzar aquests components de manera efectiva, pots automatitzar tasques complexes de desplegament i configuració, assegurant que la teva infraestructura estigui configurada exactament com necessites. Recorda gestionar els errors de manera elegant i utilitzar múltiples proveïdors quan sigui necessari per realitzar una sèrie de tasques.
