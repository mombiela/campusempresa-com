Introducció al Control de Versions

El control de versions és un sistema que registra els canvis en un fitxer o conjunt de fitxers al llarg del temps, de manera que puguis recordar versions específiques més tard. Git és un sistema de control de versions distribuït que permet a múltiples desenvolupadors treballar en un projecte simultàniament sense sobreescriure els canvis dels altres.

Configuració de Git

Abans de poder utilitzar Git, cal instal·lar-lo i configurar la teva informació d'usuari.

Instal·lació de Git

  • A Linux: Utilitza el teu gestor de paquets (per exemple, sudo apt-get install git per a sistemes basats en Debian).
  • A macOS: Utilitza Homebrew (brew install git).
  • A Windows: Descarrega l'instal·lador des del lloc web oficial de Git.

Configuració de Git

Després de la instal·lació, configura el teu nom d'usuari i correu electrònic de Git:

git config --global user.name "El teu Nom"
git config --global user.email "el.teu.email@example.com"

Comandes Bàsiques de Git

Aquí tens algunes comandes fonamentals de Git per començar:

Inicialització d'un Repositori

Per iniciar un nou repositori de Git, navega al directori del teu projecte i executa:

git init

Clonació d'un Repositori

Per clonar un repositori existent:

git clone https://github.com/username/repository.git

Comprovació de l'Estat del Repositori

Per veure l'estat dels teus fitxers:

git status

Afegir Canvis

Per afegir canvis a l'àrea de preparació:

git add filename
# o per afegir tots els canvis
git add .

Confirmació de Canvis

Per confirmar els teus canvis amb un missatge:

git commit -m "El teu missatge de confirmació"

Visualització de l'Historial de Confirmacions

Per veure l'historial de confirmacions:

git log

Ramificació i Fusió

Les branques et permeten treballar en diferents parts d'un projecte simultàniament.

Creació d'una Branca

Per crear una nova branca:

git branch branch_name

Canvi de Branca

Per canviar a una branca diferent:

git checkout branch_name

Fusió de Branques

Per fusionar una branca a la teva branca actual:

git merge branch_name

Repositoris Remots

Els repositoris remots són versions del teu projecte que estan allotjades a internet o a la xarxa.

Afegir un Repositori Remot

Per afegir un repositori remot:

git remote add origin https://github.com/username/repository.git

Empènyer Canvis

Per empènyer els teus canvis al repositori remot:

git push origin branch_name

Estirar Canvis

Per estirar canvis del repositori remot:

git pull origin branch_name

Gestió de Conflictes

Els conflictes es produeixen quan els canvis en diferents branques es contradiuen entre si.

Resolució de Conflictes

  1. Obre els fitxers en conflicte i resol els conflictes manualment.
  2. Després de resoldre'ls, afegeix els fitxers resolts:
git add filename
  1. Confirma els canvis:
git commit -m "Conflicte de fusió resolt"

Comandes Avançades de Git

Per a un ús més avançat, aquí tens algunes comandes addicionals:

Emmagatzematge de Canvis

Per desar temporalment els canvis sense confirmar:

git stash

Per aplicar els canvis emmagatzemats:

git stash apply

Rebase

Per rebase la teva branca actual sobre una altra branca:

git rebase branch_name

Resum

En aquesta secció, hem cobert els conceptes bàsics d'ús de Git per al control de versions, incloent la configuració de Git, comandes bàsiques, ramificació i fusió, treball amb repositoris remots, gestió de conflictes i algunes comandes avançades. Entendre aquests conceptes t'ajudarà a gestionar el teu codi de manera més efectiva i a col·laborar amb altres sense problemes.
