Introducció a cron

cron és un programador de tasques basat en el temps en sistemes operatius tipus Unix. Els usuaris poden programar tasques (comandes o scripts) per executar-se periòdicament a hores, dates o intervals fixos. Això és especialment útil per automatitzar tasques repetitives com còpies de seguretat, manteniment del sistema i altres tasques rutinàries.

Entendre la Sintaxi de cron

Una tasca cron es defineix utilitzant una sintaxi específica en un fitxer anomenat crontab (taula cron). Cada línia en un fitxer crontab representa una tasca programada i segueix aquest format:

* * * * * command_to_execute
- - - - -
| | | | |
| | | | +---- Dia de la setmana (0 - 7) (Diumenge és tant 0 com 7)
| | | +------ Mes (1 - 12)
| | +-------- Dia del mes (1 - 31)
| +---------- Hora (0 - 23)
+------------ Minut (0 - 59)

Exemple

Per executar un script cada dia a les 2:30 AM, escriuries:

30 2 * * * /path/to/script.sh

Gestionar Fitxers crontab

Veure el crontab Actual

Per veure el crontab de l'usuari actual, utilitza:

crontab -l

Editar el crontab

Per editar el crontab, utilitza:

crontab -e

Aquesta comanda obre el fitxer crontab en l'editor de text per defecte. Pots afegir, modificar o eliminar tasques cron aquí.

Eliminar el crontab

Per eliminar el crontab de l'usuari actual, utilitza:

crontab -r

Patrons de Programació Comuns

Aquí tens alguns patrons de programació comuns que podries utilitzar:

| Programació | Sintaxi | Descripció | |----------------------------------|---------------------|------------------------------------------| | Cada minut | * * * * * | S'executa cada minut | | Cada hora | 0 * * * * | S'executa a l'inici de cada hora | | Cada dia a mitjanit | 0 0 * * * | S'executa cada dia a mitjanit | | Cada diumenge a mitjanit | 0 0 * * 0 | S'executa cada diumenge a mitjanit | | Primer dia de cada mes a les 3 AM| 0 3 1 * * | S'executa a les 3 AM el primer dia del mes|

Ús Avançat de cron

Redirigir la Sortida

Per capturar la sortida d'una tasca cron, pots redirigir-la a un fitxer:

30 2 * * * /path/to/script.sh >> /path/to/logfile.log 2>&1

Aquest exemple afegeix tant la sortida estàndard com l'error estàndard a logfile.log.

Utilitzar Variables d'Entorn

Pots establir variables d'entorn en el fitxer crontab:

SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin

30 2 * * * /path/to/script.sh

Establir SHELL i PATH assegura que la tasca cron s'executi en l'entorn correcte.

Cadenes Especials

cron també suporta cadenes especials per substituir els cinc camps de temps i data:

  • @reboot : S'executa una vegada, en arrencar.
  • @yearly o @annually : S'executa una vegada a l'any, "0 0 1 1 *".
  • @monthly : S'executa una vegada al mes, "0 0 1 * *".
  • @weekly : S'executa una vegada a la setmana, "0 0 * * 0".
  • @daily o @midnight : S'executa una vegada al dia, "0 0 * * *".
  • @hourly : S'executa una vegada a l'hora, "0 * * * *".

Exemple:

@daily /path/to/daily_script.sh

Depurar Tasques cron

Comprovar els Registres de cron

Els registres de cron es poden trobar a /var/log/syslog o /var/log/cron depenent de la configuració del teu sistema. Pots utilitzar grep per filtrar les entrades relacionades amb cron:

grep cron /var/log/syslog

Problemes Comuns

  • Permisos: Assegura't que l'script tingui permisos d'execució.
  • Variables d'Entorn: L'entorn cron és diferent del teu entorn de shell. Estableix explícitament les variables d'entorn necessàries.
  • Problemes de Ruta: Utilitza rutes absolutes per a les comandes i scripts.

Conclusió

Programar tasques amb cron és una manera poderosa d'automatitzar tasques repetitives en sistemes tipus Unix. Entendre la sintaxi i com gestionar fitxers crontab et permet programar tasques de manera eficient. Amb funcions avançades com la redirecció de sortida, variables d'entorn i cadenes especials, cron proporciona flexibilitat i control sobre la programació de tasques. Recorda sempre comprovar els registres i els permisos per solucionar qualsevol problema amb les teves tasques cron.
