Introducció a SSH

Secure Shell (SSH) és un protocol utilitzat per connectar-se de manera segura a sistemes remots a través d'una xarxa. Proporciona un canal segur sobre una xarxa no segura mitjançant l'ús de xifratge.

  • Què és SSH?

    • SSH significa Secure Shell.
    • S'utilitza per a l'inici de sessió remot segur i altres serveis de xarxa segurs sobre una xarxa insegura.
    • SSH utilitza criptografia de clau pública per autenticar l'ordinador remot i permetre que aquest autentiqui l'usuari.

  • Per què utilitzar SSH?

    • Comunicació segura: Xifra les dades transferides entre el client i el servidor.
    • Autenticació: Utilitza criptografia de clau pública per a una autenticació segura.
    • Redirecció de ports: Pot tunelitzar altres protocols a través de SSH.

Configuració de SSH

Per utilitzar SSH, necessites tant un client com un servidor. El client inicia la connexió, i el servidor respon.

Instal·lació de SSH

  • A Ubuntu/Debian: 
    sudo apt-get update
sudo apt-get install openssh-server
  • A CentOS/RHEL: 
    sudo yum install openssh-server
sudo systemctl start sshd
sudo systemctl enable sshd

Iniciar i habilitar el servei SSH

  • A Ubuntu/Debian: 
    sudo systemctl start ssh
sudo systemctl enable ssh
  • A CentOS/RHEL: 
    sudo systemctl start sshd
sudo systemctl enable sshd

Connexió a un servidor remot

Per connectar-te a un servidor remot utilitzant SSH, necessites l'adreça IP o el nom de l'host del servidor i un compte d'usuari vàlid al servidor.

Comanda bàsica de SSH

ssh username@hostname
  • username: El compte d'usuari al servidor remot.
  • hostname: L'adreça IP o el nom de domini del servidor remot.

Exemple

ssh john@192.168.1.10

Aquesta comanda es connectarà al servidor a 192.168.1.10 utilitzant l'usuari john.

Autenticació basada en claus SSH

L'autenticació basada en claus SSH és més segura que l'autenticació basada en contrasenyes. Utilitza un parell de claus criptogràfiques: una clau privada i una clau pública.

Generació de claus SSH

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
  • -t rsa: Especifica el tipus de clau a crear (RSA).
  • -b 4096: Especifica el nombre de bits a la clau (4096 bits).
  • -C "your_email@example.com": Afegeix una etiqueta a la clau.

Copiar la clau pública al servidor

ssh-copy-id username@hostname
  • username: El compte d'usuari al servidor remot.
  • hostname: L'adreça IP o el nom de domini del servidor remot.

Exemple

ssh-copy-id john@192.168.1.10

Aquesta comanda copiarà la teva clau pública al servidor a 192.168.1.10 per a l'usuari john.

Gestió de configuracions SSH

Pots gestionar múltiples connexions SSH utilitzant el fitxer de configuració SSH situat a ~/.ssh/config.

Exemple de configuració

Host myserver
    HostName 192.168.1.10
    User john
    IdentityFile ~/.ssh/id_rsa
  • Host: Un sobrenom per al teu servidor.
  • HostName: L'adreça IP o el nom de domini del servidor.
  • User: El compte d'usuari al servidor.
  • IdentityFile: El camí a la teva clau privada.

Connexió utilitzant la configuració

ssh myserver

Aquesta comanda utilitzarà la configuració definida per a myserver.

Redirecció de ports amb SSH

SSH pot redirigir ports des de la màquina local a la màquina remota, o viceversa.

Redirecció de ports locals

ssh -L local_port:remote_address:remote_port username@hostname
  • local_port: El port local a redirigir.
  • remote_address: L'adreça de la màquina remota.
  • remote_port: El port a la màquina remota.

Exemple

ssh -L 8080:localhost:80 john@192.168.1.10

Aquesta comanda redirigeix el port local 8080 al port 80 a 192.168.1.10.

Conclusió

SSH és una eina poderosa per a l'accés remot segur i la gestió de servidors. Entenent com configurar i utilitzar SSH, pots connectar-te i gestionar sistemes remots de manera segura. L'autenticació basada en claus i les configuracions SSH poden simplificar i assegurar les teves connexions, mentre que la redirecció de ports pot estendre la funcionalitat de SSH a altres serveis.
