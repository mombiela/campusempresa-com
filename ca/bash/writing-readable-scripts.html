Introducció

Escriure scripts llegibles és crucial per mantenir i depurar el teu codi. Els scripts llegibles són més fàcils d'entendre, modificar i compartir amb altres. Aquesta secció cobrirà les millors pràctiques per escriure scripts Bash clars i mantenibles.

Conceptes Clau

  • Convencions de Nomenclatura Consistents: Utilitza noms significatius i consistents per a variables, funcions i fitxers.
  • Comentaris: Afegeix comentaris per explicar el propòsit del script, la lògica complexa i qualsevol codi no evident.
  • Indentació i Espaiat: Utilitza una indentació i espaiat consistents per millorar la llegibilitat.
  • Modularització: Divideix el teu script en funcions més petites i reutilitzables.
  • Gestió d'Errors: Implementa la gestió d'errors per gestionar situacions inesperades de manera elegant.
  • Documentació: Proporciona documentació per als teus scripts, incloent instruccions d'ús i exemples.

Convencions de Nomenclatura Consistents

Utilitzar noms consistents i descriptius ajuda als altres (i al teu futur jo) a entendre què fa el teu script.

Exemple

# Noms de variables dolents
a=10
b=20

# Noms de variables bons
number_of_files=10
max_retry_attempts=20

Comentaris

Els comentaris són essencials per explicar el propòsit i la lògica del teu script. Utilitza comentaris per descriure què fa el teu script, per què es van prendre certes decisions i com utilitzar el script.

Exemple

#!/bin/bash

# Aquest script fa una còpia de seguretat d'un directori a una ubicació especificada

# Comprova si s'ha proporcionat el nombre correcte d'arguments
if [ "$#" -ne 2 ]; then
    echo "Ús: $0 directori_origen directori_còpia_de_seguretat"
    exit 1
fi

source_directory=$1
backup_directory=$2

# Crea el directori de còpia de seguretat si no existeix
mkdir -p "$backup_directory"

# Copia els fitxers
cp -r "$source_directory"/* "$backup_directory"

echo "Còpia de seguretat completada amb èxit."

Indentació i Espaiat

Una indentació i espaiat consistents fan que el teu script sigui més fàcil de llegir i entendre. Utilitza 2 o 4 espais per a la indentació i evita barrejar tabuladors i espais.

Exemple

#!/bin/bash

# Funció per comprovar si un directori existeix
check_directory() {
    local dir=$1
    if [ -d "$dir" ]; then
        echo "El directori $dir existeix."
    else
        echo "El directori $dir no existeix."
    fi
}

# Script principal
check_directory "/etc"
check_directory "/inexistent"

Modularització

Dividir el teu script en funcions més petites i reutilitzables fa que sigui més fàcil de gestionar i entendre. Cada funció hauria de realitzar una única tasca.

Exemple

#!/bin/bash

# Funció per crear una còpia de seguretat
create_backup() {
    local source=$1
    local destination=$2
    cp -r "$source" "$destination"
    echo "Còpia de seguretat de $source completada."
}

# Funció per comprovar l'espai en disc
check_disk_space() {
    df -h | grep "$1"
}

# Script principal
source_directory="/home/user/data"
backup_directory="/home/user/backup"

create_backup "$source_directory" "$backup_directory"
check_disk_space "/home"

Gestió d'Errors

Implementa la gestió d'errors per gestionar situacions inesperades de manera elegant. Utilitza codis de sortida i missatges d'error per indicar problemes.

Exemple

#!/bin/bash

# Funció per crear una còpia de seguretat amb gestió d'errors
create_backup() {
    local source=$1
    local destination=$2

    if [ ! -d "$source" ]; then
        echo "Error: El directori d'origen $source no existeix."
        exit 1
    fi

    cp -r "$source" "$destination"
    if [ $? -ne 0 ]; then
        echo "Error: No s'ha pogut crear la còpia de seguretat."
        exit 1
    fi

    echo "Còpia de seguretat de $source completada."
}

# Script principal
source_directory="/home/user/data"
backup_directory="/home/user/backup"

create_backup "$source_directory" "$backup_directory"

Documentació

Proporciona documentació per als teus scripts, incloent instruccions d'ús i exemples. Això ajuda als altres a entendre com utilitzar el teu script i què fa.

Exemple

#!/bin/bash

# backup.sh - Un script per crear una còpia de seguretat d'un directori
#
# Ús:
#   ./backup.sh directori_origen directori_còpia_de_seguretat
#
# Exemple:
#   ./backup.sh /home/user/data /home/user/backup
#
# Aquest script crea una còpia de seguretat del directori_origen al directori_còpia_de_seguretat.

# Script principal
source_directory=$1
backup_directory=$2

# Funció per crear una còpia de seguretat
create_backup() {
    local source=$1
    local destination=$2
    cp -r "$source" "$destination"
    echo "Còpia de seguretat de $source completada."
}

create_backup "$source_directory" "$backup_directory"

Conclusió

Escriure scripts llegibles és una habilitat essencial per a qualsevol programador. Seguint les millors pràctiques com utilitzar convencions de nomenclatura consistents, afegir comentaris, mantenir una correcta indentació i espaiat, modularitzar el teu codi, implementar la gestió d'errors i proporcionar documentació, pots crear scripts que siguin fàcils d'entendre, mantenir i compartir. Aquestes pràctiques no només milloren la qualitat del teu codi sinó que també fan que la col·laboració amb altres sigui més eficient.
