Introducció a les expressions regulars

Les expressions regulars (regex) són seqüències de caràcters que defineixen un patró de cerca. S'utilitzen per a la coincidència de cadenes dins de textos i són increïblement poderoses per a tasques de processament de text. En Bash, les expressions regulars s'utilitzen comunament amb comandes com grep, sed i awk.

Conceptes clau

  • Caràcters literals: Caràcters que coincideixen amb ells mateixos.
  • Metacaràcters: Caràcters especials que tenen un significat únic en regex (per exemple, . * ^ $).
  • Classes de caràcters: Conjunts de caràcters tancats entre claudàtors [] que coincideixen amb qualsevol caràcter dins del conjunt.
  • Quantificadors: Símbols que especifiquen quantes vegades un caràcter o grup ha de coincidir (per exemple, * + ? {n} {n,} {n,m}).
  • Ancoratges: Caràcters que afirmen una posició en la cadena (per exemple, ^ per al començament, $ per al final).

Expressions regulars bàsiques

Caràcters literals

Els caràcters literals coincideixen exactament amb ells mateixos. Per exemple, la regex cat coincideix amb la cadena "cat".

Metacaràcters

  • .: Coincideix amb qualsevol caràcter excepte una nova línia.
  • *: Coincideix amb 0 o més de l'element precedent.
  • ^: Coincideix amb el començament d'una línia.
  • $: Coincideix amb el final d'una línia.

Exemples

# Coincideix amb qualsevol línia que contingui 'cat'
grep 'cat' filename.txt

# Coincideix amb qualsevol línia que comenci amb 'cat'
grep '^cat' filename.txt

# Coincideix amb qualsevol línia que acabi amb 'cat'
grep 'cat$' filename.txt

# Coincideix amb qualsevol línia que contingui 'c.t' (on . pot ser qualsevol caràcter)
grep 'c.t' filename.txt

Classes de caràcters

Les classes de caràcters permeten coincidir amb qualsevol dels caràcters d'un conjunt.

Exemples

  • [abc]: Coincideix amb qualsevol de 'a', 'b' o 'c'.
  • [a-z]: Coincideix amb qualsevol lletra minúscula.
  • [A-Z]: Coincideix amb qualsevol lletra majúscula.
  • [0-9]: Coincideix amb qualsevol dígit.
# Coincideix amb qualsevol línia que contingui un dígit
grep '[0-9]' filename.txt

# Coincideix amb qualsevol línia que contingui una lletra minúscula
grep '[a-z]' filename.txt

# Coincideix amb qualsevol línia que contingui 'a', 'b' o 'c'
grep '[abc]' filename.txt

Quantificadors

Els quantificadors especifiquen el nombre d'ocurrències d'un caràcter o grup.

Exemples

  • *: 0 o més vegades.
  • +: 1 o més vegades.
  • ?: 0 o 1 vegada.
  • {n}: Exactament n vegades.
  • {n,}: n o més vegades.
  • {n,m}: Entre n i m vegades.
# Coincideix amb qualsevol línia que contingui 'a' seguit de zero o més 'b's
grep 'ab*' filename.txt

# Coincideix amb qualsevol línia que contingui 'a' seguit d'una o més 'b's
grep 'ab+' filename.txt

# Coincideix amb qualsevol línia que contingui 'a' seguit de zero o una 'b'
grep 'ab?' filename.txt

# Coincideix amb qualsevol línia que contingui 'a' seguit exactament de 3 'b's
grep 'ab{3}' filename.txt

# Coincideix amb qualsevol línia que contingui 'a' seguit de 2 o més 'b's
grep 'ab{2,}' filename.txt

# Coincideix amb qualsevol línia que contingui 'a' seguit entre 2 i 4 'b's
grep 'ab{2,4}' filename.txt

Expressions regulars avançades

Les expressions regulars avançades (AREs) inclouen característiques addicionals com el agrupament i l'alternança.

Agrupament i alternança

  • Agrupament: Els parèntesis () s'utilitzen per agrupar parts de la regex.
  • Alternança: La barra vertical | s'utilitza per especificar alternatives.

Exemples

# Coincideix amb qualsevol línia que contingui 'cat' o 'dog'
grep 'cat|dog' filename.txt

# Coincideix amb qualsevol línia que contingui 'cat' seguit de 'dog'
grep 'cat(dog)' filename.txt

# Coincideix amb qualsevol línia que contingui 'cat' seguit de 'dog' o 'mouse'
grep 'cat(dog|mouse)' filename.txt

Aplicacions pràctiques

Utilitzant `grep`

grep és una utilitat de línia de comandes per cercar dades de text pla per línies que coincideixin amb una expressió regular.

# Cerca línies que continguin 'error' en un fitxer de registre
grep 'error' /var/log/syslog

# Cerca línies que comencin amb 'error'
grep '^error' /var/log/syslog

# Cerca línies que acabin amb 'error'
grep 'error$' /var/log/syslog

Utilitzant `sed`

sed és un editor de flux per filtrar i transformar text.

# Substitueix 'foo' per 'bar' en un fitxer
sed 's/foo/bar/g' filename.txt

# Elimina línies que continguin 'error'
sed '/error/d' filename.txt

Utilitzant `awk`

awk és un llenguatge de programació per a l'escaneig i processament de patrons.

# Imprimeix línies que continguin 'error'
awk '/error/' filename.txt

# Imprimeix la segona columna de línies que continguin 'error'
awk '/error/ {print $2}' filename.txt

Conclusió

Les expressions regulars són una eina poderosa per al processament de text en Bash. En dominar els conceptes bàsics i avançats de regex, pots cercar, substituir i manipular dades de text de manera eficient. Practica utilitzant grep, sed i awk amb expressions regulars per millorar les teves habilitats de processament de text en Bash.
