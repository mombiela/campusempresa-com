Introducció a PyTorch Lightning

PyTorch Lightning és un embolcall lleuger de PyTorch per a la investigació d'IA d'alt rendiment. Ajuda a organitzar el codi de PyTorch, fent-lo més llegible i escalable mentre redueix el codi redundant.

Conceptes Clau

  • LightningModule: Una subclasse de torch.nn.Module que organitza el teu codi de PyTorch en seccions específiques.
  • Trainer: Una classe que gestiona el bucle d'entrenament, validació i proves.
  • DataModule: Una classe que encapsula totes les operacions relacionades amb les dades, com ara la càrrega, el preprocessament i la divisió de conjunts de dades.

Configuració de PyTorch Lightning

Per començar amb PyTorch Lightning, cal instal·lar-lo. Pots fer-ho utilitzant pip:

pip install pytorch-lightning

LightningModule

El LightningModule és el component principal on defineixes el teu model, els passos d'entrenament, validació i proves.

Exemple: Creació d'un Simple LightningModule

import pytorch_lightning as pl
import torch
from torch import nn
from torch.optim import Adam

class SimpleModel(pl.LightningModule):
    def __init__(self):
        super(SimpleModel, self).__init__()
        self.layer = nn.Linear(28 * 28, 10)

    def forward(self, x):
        return self.layer(x.view(x.size(0), -1))

    def training_step(self, batch, batch_idx):
        x, y = batch
        logits = self.forward(x)
        loss = nn.functional.cross_entropy(logits, y)
        return loss

    def configure_optimizers(self):
        return Adam(self.parameters(), lr=1e-3)

Explicació

  • __init__: Inicialitza les capes del model.
  • forward: Defineix el pas endavant.
  • training_step: Defineix la lògica del bucle d'entrenament.
  • configure_optimizers: Defineix l'optimitzador.

Trainer

La classe Trainer automatitza el bucle d'entrenament, validació i proves.

Exemple: Utilitzant el Trainer

from pytorch_lightning import Trainer
from torch.utils.data import DataLoader, random_split
from torchvision.datasets import MNIST
from torchvision.transforms import ToTensor

# Carregar dades
dataset = MNIST('', train=True, download=True, transform=ToTensor())
train, val = random_split(dataset, [55000, 5000])
train_loader = DataLoader(train, batch_size=32)
val_loader = DataLoader(val, batch_size=32)

# Inicialitzar model
model = SimpleModel()

# Inicialitzar trainer
trainer = Trainer(max_epochs=5)

# Entrenar el model
trainer.fit(model, train_loader, val_loader)

Explicació

  • DataLoader: Utilitzat per carregar i preprocessar el conjunt de dades.
  • Trainer: Gestiona el procés d'entrenament.
  • trainer.fit: Entrena el model utilitzant els carregadors de dades proporcionats.

DataModule

La classe DataModule encapsula totes les operacions relacionades amb les dades, facilitant la gestió de la càrrega i el preprocessament de dades.

Exemple: Creació d'un DataModule

from pytorch_lightning import LightningDataModule
from torchvision.datasets import MNIST
from torchvision.transforms import ToTensor
from torch.utils.data import DataLoader, random_split

class MNISTDataModule(LightningDataModule):
    def __init__(self, batch_size=32):
        super().__init__()
        self.batch_size = batch_size

    def prepare_data(self):
        MNIST('', train=True, download=True)

    def setup(self, stage=None):
        dataset = MNIST('', train=True, transform=ToTensor())
        self.train, self.val = random_split(dataset, [55000, 5000])

    def train_dataloader(self):
        return DataLoader(self.train, batch_size=self.batch_size)

    def val_dataloader(self):
        return DataLoader(self.val, batch_size=self.batch_size)

Explicació

  • prepare_data: Descarregar o preparar dades.
  • setup: Dividir el conjunt de dades en conjunts d'entrenament i validació.
  • train_dataloader: Retorna el carregador de dades d'entrenament.
  • val_dataloader: Retorna el carregador de dades de validació.

Funcions Avançades

PyTorch Lightning també proporciona funcions avançades com:

  • Callbacks: Galletes personalitzades que es poden utilitzar per executar codi en punts específics durant l'entrenament.
  • Loggers: Integració amb marcs de registre com TensorBoard, MLflow, etc.
  • Entrenament Distribuït: Escalar fàcilment el teu entrenament a múltiples GPUs o nodes.

Exemple: Utilitzant Callbacks

from pytorch_lightning.callbacks import ModelCheckpoint

checkpoint_callback = ModelCheckpoint(
    monitor='val_loss',
    dirpath='my/path/',
    filename='sample-{epoch:02d}-{val_loss:.2f}',
    save_top_k=3,
    mode='min',
)

trainer = Trainer(callbacks=[checkpoint_callback])

Explicació

  • ModelCheckpoint: Desa els punts de control del model basats en la pèrdua de validació.
  • callbacks: Passa la galleta al Trainer.

Conclusió

PyTorch Lightning simplifica el procés de construir i entrenar models de PyTorch organitzant el codi en components reutilitzables. Redueix el codi redundant i proporciona funcions avançades com callbacks, loggers i entrenament distribuït, fent-lo una eina poderosa tant per a principiants com per a usuaris avançats.
