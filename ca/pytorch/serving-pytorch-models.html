En aquesta secció, cobrirem com servir models PyTorch perquè es puguin utilitzar en entorns de producció. Servir un model implica fer-lo disponible per a inferència a través d'una API o una altra interfície. Aquest és un pas crucial en el desplegament de models d'aprenentatge automàtic a aplicacions del món real.

Conceptes Clau

  • Serialització del Model: Guardar un model entrenat al disc.
  • Deserialització del Model: Carregar un model guardat des del disc.
  • Inferència: Executar el model per fer prediccions sobre noves dades.
  • API REST: Una manera comuna de servir models a través de la web.
  • Eines de Desplegament: Eines i marcs que ajuden en el desplegament de models.

Serialització i Deserialització del Model

Abans de servir un model, cal guardar-lo després de l'entrenament. PyTorch proporciona mètodes senzills per a això.

Guardar un Model

Pots guardar un model utilitzant torch.save. Aquesta funció serialitza el model i el guarda al disc.

import torch

# Suposant que `model` és el teu model entrenat
torch.save(model.state_dict(), 'model.pth')

Carregar un Model

Per carregar un model guardat, utilitzes torch.load i load_state_dict.

import torch
from your_model_file import YourModelClass

model = YourModelClass()
model.load_state_dict(torch.load('model.pth'))
model.eval()  # Posar el model en mode d'avaluació

Servir Models amb una API REST

Una de les maneres més comunes de servir un model és a través d'una API REST. Això permet als clients enviar sol·licituds HTTP per obtenir prediccions del model.

Utilitzant Flask

Flask és un marc web lleuger per a Python que es fa servir comunament per servir models d'aprenentatge automàtic.

Configurar Flask

Primer, instal·la Flask si no ho has fet ja:

pip install Flask

Crear una Aplicació Flask

Aquí tens un exemple bàsic d'una aplicació Flask que serveix un model PyTorch:

from flask import Flask, request, jsonify
import torch
from your_model_file import YourModelClass

app = Flask(__name__)

# Carregar el model
model = YourModelClass()
model.load_state_dict(torch.load('model.pth'))
model.eval()

@app.route('/predict', methods=['POST'])
def predict():
    data = request.get_json(force=True)
    inputs = torch.tensor(data['inputs'])
    with torch.no_grad():
        outputs = model(inputs)
    return jsonify({'outputs': outputs.tolist()})

if __name__ == '__main__':
    app.run(debug=True)

Provar l'API

Pots provar l'API utilitzant eines com curl o Postman.

curl -X POST -H "Content-Type: application/json" -d '{"inputs": [1.0, 2.0, 3.0]}' http://127.0.0.1:5000/predict

Eines de Desplegament

Diverses eines poden ajudar-te a desplegar el teu model de manera més eficient:

  • TorchServe: Una eina flexible i fàcil d'utilitzar per servir models PyTorch.
  • Docker: Eina de contenidorització que pot empaquetar la teva aplicació i les seves dependències.
  • Kubernetes: Eina d'orquestració per desplegar, escalar i gestionar aplicacions contenidoritzades.

Utilitzant TorchServe

TorchServe és una eina dissenyada específicament per servir models PyTorch. Simplifica el procés i proporciona característiques com registre, mètriques i servei de múltiples models.

Instal·lar TorchServe

pip install torchserve torch-model-archiver

Arxivar el Model

Necessites crear un arxiu d'arxiu de model (.mar) que TorchServe pugui utilitzar.

torch-model-archiver --model-name my_model --version 1.0 --serialized-file model.pth --handler handler.py --extra-files "your_model_file.py"

Iniciar TorchServe

torchserve --start --ncs --model-store model_store --models my_model=my_model.mar

Conclusió

Servir models PyTorch és un pas crucial en el desplegament d'aplicacions d'aprenentatge automàtic. Entenent com serialitzar i deserialitzar models, crear APIs REST amb Flask i utilitzar eines de desplegament com TorchServe, pots fer que els teus models estiguin disponibles per a ús en el món real. Aquesta secció t'ha proporcionat el coneixement fonamental i exemples pràctics per començar a servir els teus models PyTorch.
