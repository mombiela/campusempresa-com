Introducció

Els algorismes de cerca són fonamentals en el camp de la Intel·ligència Artificial (IA). Aquests algorismes permeten trobar solucions a problemes específics mitjançant l'exploració de possibles estats o solucions. En aquest tema, explorarem els conceptes clau, els tipus d'algorismes de cerca i exemples pràctics per entendre el seu funcionament.

Conceptes Clau

  • Espai d'Estats: Conjunt de tots els estats possibles que es poden assolir des de l'estat inicial.
  • Node: Representació d'un estat en l'espai d'estats.
  • Camí: Seqüència de nodes que porta des de l'estat inicial fins a l'estat objectiu.
  • Cost del Camí: Suma dels costos associats amb els passos en el camí.
  • Funció d'Avaluació: Funció que assigna un valor a cada node, utilitzat per decidir el següent node a explorar.

Tipus d'Algorismes de Cerca

Cerca No Informada

  1. Cerca en Amplada (Breadth-First Search, BFS)

    • Explora tots els nodes a un nivell de profunditat abans de passar al següent nivell.
    • Utilitza una cua (FIFO) per emmagatzemar els nodes per explorar.
    • Avantatges: Garanteix trobar la solució més curta si tots els costos dels passos són iguals.
    • Desavantatges: Pot ser ineficient en termes de memòria i temps.
    from collections import deque

def bfs(graph, start, goal):
    queue = deque([start])
    visited = set()
    while queue:
        node = queue.popleft()
        if node == goal:
            return True
        if node not in visited:
            visited.add(node)
            queue.extend(graph[node] - visited)
    return False

graph = {
    'A': {'B', 'C'},
    'B': {'A', 'D', 'E'},
    'C': {'A', 'F'},
    'D': {'B'},
    'E': {'B', 'F'},
    'F': {'C', 'E'}
}

print(bfs(graph, 'A', 'F'))  # Output: True

  2. Cerca en Profunditat (Depth-First Search, DFS)

    • Explora tan profund com sigui possible al llarg de cada branca abans de retrocedir.
    • Utilitza una pila (LIFO) per emmagatzemar els nodes per explorar.
    • Avantatges: Requereix menys memòria que BFS.
    • Desavantatges: No garanteix trobar la solució més curta i pot quedar atrapat en cicles.
    def dfs(graph, start, goal, visited=None):
    if visited is None:
        visited = set()
    visited.add(start)
    if start == goal:
        return True
    for next_node in graph[start] - visited:
        if dfs(graph, next_node, goal, visited):
            return True
    return False

print(dfs(graph, 'A', 'F'))  # Output: True

Cerca Informada

  1. Cerca A (A-Star)*

    • Utilitza una funció d'avaluació f(n) = g(n) + h(n), on g(n) és el cost del camí des de l'inici fins al node n, i h(n) és una heurística que estima el cost des de n fins a l'objectiu.
    • Avantatges: És òptima i completa si l'heurística és admissible (mai sobreestima el cost real).
    • Desavantatges: Pot ser costosa en termes de memòria i temps.
    import heapq

def a_star(graph, start, goal, h):
    open_set = []
    heapq.heappush(open_set, (0, start))
    came_from = {}
    g_score = {start: 0}
    f_score = {start: h(start)}

    while open_set:
        _, current = heapq.heappop(open_set)
        if current == goal:
            return reconstruct_path(came_from, current)
        for neighbor, cost in graph[current].items():
            tentative_g_score = g_score[current] + cost
            if tentative_g_score < g_score.get(neighbor, float('inf')):
                came_from[neighbor] = current
                g_score[neighbor] = tentative_g_score
                f_score[neighbor] = tentative_g_score + h(neighbor)
                heapq.heappush(open_set, (f_score[neighbor], neighbor))
    return None

def reconstruct_path(came_from, current):
    total_path = [current]
    while current in came_from:
        current = came_from[current]
        total_path.append(current)
    return total_path[::-1]

graph = {
    'A': {'B': 1, 'C': 3},
    'B': {'A': 1, 'D': 1, 'E': 5},
    'C': {'A': 3, 'F': 2},
    'D': {'B': 1},
    'E': {'B': 5, 'F': 2},
    'F': {'C': 2, 'E': 2}
}

h = lambda x: {'A': 6, 'B': 5, 'C': 4, 'D': 3, 'E': 2, 'F': 1}[x]

print(a_star(graph, 'A', 'F', h))  # Output: ['A', 'C', 'F']

Comparació d'Algorismes

| Algorisme | Complet | Òptim | Complexitat de Temps | Complexitat d'Espai | |-----------|---------|-------|----------------------|---------------------| | BFS | Sí | Sí | O(b^d) | O(b^d) | | DFS | No | No | O(b^m) | O(bm) | | A* | Sí | Sí | O(b^d) | O(b^d) |

  • Complet: Troba una solució si existeix.
  • Òptim: Troba la solució més curta.
  • Complexitat de Temps: Temps requerit per trobar la solució.
  • Complexitat d'Espai: Memòria requerida per emmagatzemar els nodes.

Conclusió

Els algorismes de cerca són eines essencials en la Intel·ligència Artificial per resoldre problemes de manera eficient. L'elecció de l'algorisme adequat depèn de les característiques del problema, com la mida de l'espai d'estats i la necessitat de trobar la solució més curta. La comprensió d'aquests algorismes i la seva implementació pràctica és crucial per desenvolupar solucions efectives en IA.
