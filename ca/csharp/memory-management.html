Introducció a la Gestió de Memòria

La gestió de memòria és un aspecte crític de la programació en C#. Implica l'assignació, ús i alliberament de memòria de manera eficient. Entendre com es gestiona la memòria en C# ajuda a escriure codi optimitzat i lliure d'errors.

Conceptes Clau

  • Pila i Munt: Dues àrees principals de memòria on es guarden les dades.
    • Pila: Emmagatzema tipus de valor i informació de crides a mètodes.
    • Munt: Emmagatzema tipus de referència i objectes.
  • Tipus de Valor vs. Tipus de Referència: Determina com es guarden i accedeixen les dades.
    • Tipus de Valor: Emmagatzemats a la pila (per exemple, int, float, bool).
    • Tipus de Referència: Emmagatzemats al munt (per exemple, objectes, arrays).

Exemple de Codi: Tipus de Valor i Tipus de Referència

int valueType = 10; // Emmagatzemat a la pila
string referenceType = "Hello, World!"; // Emmagatzemat al munt

Recollida d'Escombraries

La Recollida d'Escombraries (GC) és una característica automàtica de gestió de memòria en C#. Ajuda a recuperar memòria ocupada per objectes que ja no s'utilitzen, prevenint així les fuites de memòria.

Conceptes Clau

  • Generacions: GC utilitza generacions per optimitzar el procés de recollida.
    • Generació 0: Objectes de curta durada.
    • Generació 1: Objectes de durada mitjana.
    • Generació 2: Objectes de llarga durada.
  • Arrels de GC: Referències que sempre són accessibles i utilitzades pel GC per identificar objectes vius.
  • Finalització: Procés de neteja per a objectes abans de ser recollits.

Exemple de Codi: Recollida d'Escombraries Simple

class Program
{
    static void Main()
    {
        CreateObjects();
        GC.Collect(); // Força la recollida d'escombraries
    }

    static void CreateObjects()
    {
        for (int i = 0; i < 1000; i++)
        {
            var obj = new object(); // Objectes creats en un bucle
        }
    }
}

Tècniques Avançades de Recollida d'Escombraries

A mesura que avances, entendre i controlar el procés de GC pot conduir a aplicacions més eficients.

Conceptes Clau

  • Patró Dispose: Implementació de IDisposable per a la neteja determinista.
  • Referències Febles: Permeten que el GC reculli objectes mentre encara permeten accedir-hi.
  • Munt d'Objectes Grans (LOH): Munt especial per a objectes grans, gestionat de manera diferent pel GC.

Exemple de Codi: Implementació de IDisposable

class ResourceHolder : IDisposable
{
    private bool disposed = false;

    public void Dispose()
    {
        Dispose(true);
        GC.SuppressFinalize(this);
    }

    protected virtual void Dispose(bool disposing)
    {
        if (!disposed)
        {
            if (disposing)
            {
                // Alliberar recursos gestionats
            }
            // Alliberar recursos no gestionats
            disposed = true;
        }
    }

    ~ResourceHolder()
    {
        Dispose(false);
    }
}

Millors Pràctiques per a la Gestió de Memòria

  • Minimitzar la Creació d'Objectes: Reutilitzar objectes quan sigui possible.
  • Utilitzar Structs per a Dades Petites: Els structs són tipus de valor i poden ser més eficients per a dades petites.
  • Evitar Finalitzadors Innecessaris: Els finalitzadors poden retardar la recollida d'escombraries.
  • Perfil i Optimitza: Utilitzar eines de perfilatge per identificar problemes de memòria i optimitzar en conseqüència.

Resum

La gestió de memòria i la recollida d'escombraries són essencials per escriure aplicacions eficients i fiables en C#. Entenent la pila i el munt, els tipus de valor i referència, i el funcionament del recollidor d'escombraries, pots gestionar la memòria de manera efectiva. Les tècniques avançades com la implementació de IDisposable i l'ús de referències febles poden optimitzar encara més les teves aplicacions. Segueix sempre les millors pràctiques i utilitza eines de perfilatge per assegurar un ús òptim de la memòria.
