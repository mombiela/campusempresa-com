Introducció a les Interfícies

Les interfícies en C# són un concepte fonamental que et permet definir un contracte que les classes o structs poden implementar. S'utilitzen per especificar quins mètodes i propietats ha de tenir una classe, sense proporcionar els detalls de la implementació.

Conceptes Clau

  • Definició: Una interfície es defineix utilitzant la paraula clau interface.
  • Implementació: Una classe o struct implementa una interfície proporcionant el codi per a cada mètode i propietat definits en la interfície.
  • Herència Múltiple: C# admet l'herència múltiple d'interfícies, és a dir, una classe pot implementar múltiples interfícies.
  • Polimorfisme: Les interfícies permeten un comportament polimòrfic, permetent-te utilitzar diferents classes de manera intercanviable si implementen la mateixa interfície.

Definició d'una Interfície

Per definir una interfície en C#, utilitza la paraula clau interface seguida del nom de la interfície. Els noms de les interfícies típicament comencen amb una 'I' majúscula.

public interface IAnimal
{
    void MakeSound();
    void Move();
}

Implementació d'una Interfície

Una classe o struct implementa una interfície utilitzant el símbol : seguit del nom de la interfície. La classe ha de proporcionar implementacions concretes per a tots els mètodes i propietats definits en la interfície.

public class Dog : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Bark");
    }

    public void Move()
    {
        Console.WriteLine("Run");
    }
}

Ús de les Interfícies

Les interfícies es poden utilitzar per aconseguir polimorfisme. Pots escriure codi que funcioni amb objectes de qualsevol classe que implementi una interfície particular.

public class Program
{
    public static void Main()
    {
        IAnimal myDog = new Dog();
        myDog.MakeSound(); // Sortida: Bark
        myDog.Move();      // Sortida: Run
    }
}

Herència d'Interfícies

Les interfícies poden heretar d'altres interfícies. Això et permet crear interfícies més específiques basades en altres més generals.

public interface IAnimal
{
    void MakeSound();
    void Move();
}

public interface IPet : IAnimal
{
    void Play();
}

public class Cat : IPet
{
    public void MakeSound()
    {
        Console.WriteLine("Meow");
    }

    public void Move()
    {
        Console.WriteLine("Jump");
    }

    public void Play()
    {
        Console.WriteLine("Chase laser pointer");
    }
}

Implementació Explícita d'Interfícies

De vegades, una classe pot implementar múltiples interfícies que tenen mètodes amb el mateix nom. En aquests casos, pots utilitzar la implementació explícita d'interfícies per evitar ambigüitats.

public interface IPrinter
{
    void Print();
}

public interface IScanner
{
    void Print();
}

public class MultiFunctionDevice : IPrinter, IScanner
{
    void IPrinter.Print()
    {
        Console.WriteLine("Printing document");
    }

    void IScanner.Print()
    {
        Console.WriteLine("Scanning document");
    }
}

Conceptes Avançats

Mètodes per Defecte d'Interfície (C# 8.0 i posteriors)

C# 8.0 va introduir els mètodes per defecte d'interfície, permetent-te proporcionar una implementació per defecte per als mètodes en una interfície.

public interface IAnimal
{
    void MakeSound();
    void Move()
    {
        Console.WriteLine("Moving...");
    }
}

public class Bird : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Chirp");
    }
}

Interfície vs Classe Abstracta

| Característica | Interfície | Classe Abstracta | |----------------------------|------------------------------------|-------------------------------------| | Herència Múltiple | Sí | No | | Implementació per Defecte | C# 8.0 i posteriors | Sí | | Modificadors d'Accés | No permesos | Permesos | | Camps | No permesos | Permesos | | Constructors | No permesos | Permesos |

Conclusió

Les interfícies en C# són una eina poderosa per definir contractes i aconseguir polimorfisme. Et permeten escriure codi flexible i reutilitzable especificant quins mètodes i propietats ha de implementar una classe sense dictar com s'han d'implementar. Entendre les interfícies i les seves característiques avançades, com els mètodes per defecte i la implementació explícita, és crucial per escriure aplicacions C# robustes i mantenibles.
