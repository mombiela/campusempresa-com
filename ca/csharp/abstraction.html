Introducció a l'Abstracció

L'abstracció és un dels quatre principis fonamentals de la Programació Orientada a Objectes (OOP), juntament amb l'Encapsulació, l'Herència i el Polimorfisme. Permet als programadors centrar-se en què fa un objecte en lloc de com ho fa.

  • Definició: L'abstracció és el concepte d'amagar els detalls complexos d'implementació i mostrar només les característiques essencials d'un objecte.
  • Propòsit: Simplifica la gestió del codi, millora la llegibilitat del codi i redueix la complexitat.

Conceptes Bàsics de l'Abstracció

En C#, l'abstracció es pot aconseguir utilitzant classes abstractes i interfícies.

Classes Abstractes

Una classe abstracta és una classe que no es pot instanciar i està dissenyada per ser subclassificada. Pot contenir tant mètodes abstractes (sense implementació) com mètodes concrets (amb implementació).

  • Sintaxi:

    public abstract class Animal
{
    public abstract void MakeSound(); // Mètode abstracte
    public void Sleep() // Mètode concret
    {
        Console.WriteLine("Sleeping...");
    }
}

  • Exemple:

    public abstract class Animal
{
    public abstract void MakeSound();
    public void Sleep()
    {
        Console.WriteLine("Sleeping...");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Bark");
    }
}

public class Program
{
    public static void Main()
    {
        Dog myDog = new Dog();
        myDog.MakeSound(); // Output: Bark
        myDog.Sleep(); // Output: Sleeping...
    }
}

Interfícies

Una interfície és un contracte que defineix un conjunt de mètodes i propietats que una classe ha d'implementar. A diferència de les classes abstractes, les interfícies no poden contenir cap implementació.

  • Sintaxi:

    public interface IAnimal
{
    void MakeSound();
    void Sleep();
}

  • Exemple:

    public interface IAnimal
{
    void MakeSound();
    void Sleep();
}

public class Dog : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Bark");
    }

    public void Sleep()
    {
        Console.WriteLine("Sleeping...");
    }
}

public class Program
{
    public static void Main()
    {
        IAnimal myDog = new Dog();
        myDog.MakeSound(); // Output: Bark
        myDog.Sleep(); // Output: Sleeping...
    }
}

Conceptes Avançats de l'Abstracció

A mesura que avances, pots aprofitar l'abstracció per dissenyar sistemes més complexos.

Classes Abstractes vs Interfícies

Entendre quan utilitzar classes abstractes en lloc d'interfícies és crucial.

| Característica | Classe Abstracta | Interfície | |------------------------|-----------------------------------------|-----------------------------------------| | Herència Múltiple | No (Herència Simple) | Sí (Múltiples Interfícies) | | Implementació Per Defecte | Sí | No | | Modificadors d'Accés | Sí (public, protected, etc.) | No (implícitament públic) | | Camps | Sí | No |

Exemple del Món Real

Considera un sistema de pagaments on diferents mètodes de pagament (CreditCard, PayPal, etc.) comparteixen comportaments comuns però tenen implementacions específiques.

  • Exemple de Classe Abstracta:

    public abstract class Payment
{
    public abstract void ProcessPayment();
    public void GenerateReceipt()
    {
        Console.WriteLine("Receipt Generated.");
    }
}

public class CreditCardPayment : Payment
{
    public override void ProcessPayment()
    {
        Console.WriteLine("Processing Credit Card Payment.");
    }
}

public class PayPalPayment : Payment
{
    public override void ProcessPayment()
    {
        Console.WriteLine("Processing PayPal Payment.");
    }
}

  • Exemple d'Interfície:

    public interface IPayment
{
    void ProcessPayment();
    void GenerateReceipt();
}

public class CreditCardPayment : IPayment
{
    public void ProcessPayment()
    {
        Console.WriteLine("Processing Credit Card Payment.");
    }

    public void GenerateReceipt()
    {
        Console.WriteLine("Receipt Generated.");
    }
}

public class PayPalPayment : IPayment
{
    public void ProcessPayment()
    {
        Console.WriteLine("Processing PayPal Payment.");
    }

    public void GenerateReceipt()
    {
        Console.WriteLine("Receipt Generated.");
    }
}

Exercicis

  1. Exercici Bàsic: Crea una classe abstracta Shape amb un mètode abstracte CalculateArea(). Implementa aquest mètode en classes derivades Circle i Rectangle.
  2. Exercici Intermedi: Defineix una interfície IVehicle amb mètodes StartEngine() i StopEngine(). Implementa aquesta interfície en classes Car i Bike.
  3. Exercici Avançat: Dissenya un sistema utilitzant tant classes abstractes com interfícies per a un sistema de gestió de biblioteques on diferents tipus d'usuaris (Student, Teacher) poden demanar llibres.

Conclusió

L'abstracció és un concepte poderós en C# que ajuda a gestionar la complexitat amagant detalls innecessaris. Utilitzant classes abstractes i interfícies, pots crear codi flexible i fàcil de mantenir. Entendre quan i com utilitzar aquestes eines és essencial per a un disseny de programari efectiu.
