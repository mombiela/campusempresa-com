L'encapsulació és un dels principis fonamentals de la Programació Orientada a Objectes (OOP). Es refereix a l'agrupació de dades (variables) i mètodes (funcions) que operen sobre les dades en una sola unitat, coneguda com a classe. L'encapsulació també implica restringir l'accés directe a alguns components d'un objecte, la qual cosa és un mitjà per prevenir interferències no desitjades i l'ús indegut de les dades.

Conceptes Clau de l'Encapsulació

  • Classe: Un plànol per crear objectes. Encapsula dades per a l'objecte.
  • Objecte: Una instància d'una classe.
  • Modificadors d'Accés: Paraules clau que estableixen l'accessibilitat de classes, mètodes i altres membres. Els modificadors d'accés comuns en C# inclouen public, private, protected i internal.
  • Propietats: Mètodes especials anomenats accessors (getters i setters) que proporcionen una manera de llegir, escriure o calcular els valors dels camps privats.

Modificadors d'Accés

Els modificadors d'accés controlen la visibilitat dels membres de la classe. Aquí hi ha una comparació dels modificadors d'accés més utilitzats en C#:

| Modificador d'Accés | Descripció | |---------------------|-------------| | public | El membre és accessible des de qualsevol altre codi. | | private | El membre és accessible només dins de la classe o estructura en què es declara. | | protected | El membre és accessible dins de la seva classe i per instàncies de classes derivades. | | internal | El membre és accessible només dins dels fitxers del mateix assembly. |

Implementació de l'Encapsulació

Exemple 1: Encapsulació Bàsica

Creem una classe simple per demostrar l'encapsulació:

public class Person
{
    // Camps privats
    private string name;
    private int age;

    // Propietats públiques
    public string Name
    {
        get { return name; }
        set { name = value; }
    }

    public int Age
    {
        get { return age; }
        set 
        { 
            if (value >= 0)
            {
                age = value;
            }
        }
    }
}

En aquest exemple:

  • Els camps name i age són privats, el que significa que no es poden accedir directament des de fora de la classe Person.
  • Les propietats Name i Age proporcionen accés controlat a aquests camps. La propietat Age inclou validació per assegurar-se que l'edat no es configuri a un valor negatiu.

Exemple 2: Encapsulació Avançada amb Mètodes

L'encapsulació també pot implicar mètodes que operen sobre les dades:

public class BankAccount
{
    private decimal balance;

    public decimal Balance
    {
        get { return balance; }
    }

    public void Deposit(decimal amount)
    {
        if (amount > 0)
        {
            balance += amount;
        }
    }

    public bool Withdraw(decimal amount)
    {
        if (amount > 0 && amount <= balance)
        {
            balance -= amount;
            return true;
        }
        return false;
    }
}

En aquest exemple:

  • El camp balance és privat i només es pot accedir a través de la propietat Balance.
  • Els mètodes Deposit i Withdraw proporcionen maneres controlades de modificar el saldo, assegurant que només es realitzin operacions vàlides.

Beneficis de l'Encapsulació

  • Millora de la Seguretat: En restringir l'accés a l'estat intern d'un objecte, l'encapsulació ajuda a protegir la integritat de les dades.
  • Modularitat: L'encapsulació permet agrupar dades i mètodes relacionats, fent que el codi sigui més modular i fàcil de gestionar.
  • Mantenibilitat: Els canvis en la implementació interna d'una classe es poden fer sense afectar altres parts del codi que utilitzen la classe.
  • Flexibilitat: L'encapsulació permet canviar la implementació interna d'una classe sense canviar la seva interfície externa.

Conclusió

L'encapsulació és un concepte clau en C# i OOP que ajuda a protegir les dades, millorar la modularitat del codi i augmentar la mantenibilitat. Mitjançant l'ús de modificadors d'accés, propietats i mètodes, es pot controlar com s'accedeixen i es modifiquen les dades, assegurant que les classes s'utilitzin correctament i de manera segura. Entendre i aplicar l'encapsulació de manera efectiva és crucial per escriure aplicacions robustes i mantenibles en C#.
