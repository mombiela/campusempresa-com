Introducció

La sobrecàrrega de mètodes és un concepte fonamental en C# que et permet definir múltiples mètodes amb el mateix nom però amb diferents paràmetres. Aquesta característica millora la llegibilitat i usabilitat del codi permetent que els mètodes gestionin diferents tipus d'entrada.

Conceptes Clau

  • Signatura del Mètode: La combinació del nom del mètode i la seva llista de paràmetres.
  • Criteris de Sobrecàrrega: Els mètodes han de diferir en el nombre o tipus de paràmetres.
  • Polimorfisme en Temps de Compilació: La sobrecàrrega de mètodes és una forma de polimorfisme en temps de compilació, on el mètode a executar es determina en temps de compilació.

Per què Utilitzar la Sobrecàrrega de Mètodes?

  • Millora de la Llegibilitat del Codi: Utilitzar el mateix nom de mètode per a operacions similars fa que el codi sigui més fàcil de llegir i entendre.
  • Reutilització del Codi: Redueix la necessitat de crear múltiples mètodes amb noms diferents per a operacions similars.
  • Flexibilitat: Permet que els mètodes gestionin diferents tipus i quantitats d'entrades.

Exemple Bàsic de Sobrecàrrega de Mètodes

using System;

class Program
{
    // Mètode per sumar dos enters
    public int Add(int a, int b)
    {
        return a + b;
    }

    // Mètode sobrecarregat per sumar tres enters
    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }

    // Mètode sobrecarregat per sumar dos dobles
    public double Add(double a, double b)
    {
        return a + b;
    }

    static void Main()
    {
        Program p = new Program();

        Console.WriteLine(p.Add(1, 2)); // Crida a Add(int, int)
        Console.WriteLine(p.Add(1, 2, 3)); // Crida a Add(int, int, int)
        Console.WriteLine(p.Add(1.5, 2.5)); // Crida a Add(double, double)
    }
}

Explicació

  • El mètode Add està sobrecarregat tres vegades:
    • Add(int a, int b): Suma dos enters.
    • Add(int a, int b, int c): Suma tres enters.
    • Add(double a, double b): Suma dos dobles.
  • El mètode apropiat es crida basant-se en els arguments passats.

Conceptes Avançats

Sobrecàrrega amb Diferents Tipus de Paràmetres

using System;

class Program
{
    // Mètode per concatenar dues cadenes
    public string Concatenate(string a, string b)
    {
        return a + b;
    }

    // Mètode sobrecarregat per concatenar una cadena i un enter
    public string Concatenate(string a, int b)
    {
        return a + b.ToString();
    }

    static void Main()
    {
        Program p = new Program();

        Console.WriteLine(p.Concatenate("Hola, ", "Món!")); // Crida a Concatenate(string, string)
        Console.WriteLine(p.Concatenate("Número: ", 42)); // Crida a Concatenate(string, int)
    }
}

Ús de Paràmetres Opcional i Arguments Només

Els paràmetres opcionals i els arguments només es poden utilitzar a vegades com una alternativa a la sobrecàrrega de mètodes.

using System;

class Program
{
    // Mètode amb paràmetres opcionals
    public void DisplayInfo(string name, int age = 0, string city = "Desconegut")
    {
        Console.WriteLine($"Nom: {name}, Edat: {age}, Ciutat: {city}");
    }

    static void Main()
    {
        Program p = new Program();

        p.DisplayInfo("Alice"); // Utilitza valors per defecte per a edat i ciutat
        p.DisplayInfo("Bob", 25); // Utilitza valor per defecte per a ciutat
        p.DisplayInfo("Charlie", 30, "Nova York"); // Utilitza valors proporcionats per a tots els paràmetres
    }
}

Consideracions de Rendiment

  • La sobrecàrrega pot conduir a un codi més llegible i mantenible, però una sobrecàrrega excessiva pot fer que el codi sigui més difícil d'entendre.
  • Assegura't que els mètodes sobrecarregats estiguin lògicament relacionats per evitar confusions.

Resum

La sobrecàrrega de mètodes és una característica poderosa en C# que et permet definir múltiples mètodes amb el mateix nom però amb diferents paràmetres. Millora la llegibilitat, reutilització i flexibilitat del codi. Entenent i aplicant la sobrecàrrega de mètodes, pots escriure codi més eficient i mantenible. Recorda utilitzar la sobrecàrrega amb prudència per mantenir la claredat del codi i evitar confusions.
