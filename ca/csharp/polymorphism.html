El polimorfisme és un concepte clau en la programació orientada a objectes (OOP) que permet que objectes de diferents classes siguin tractats com a objectes d'una superclasse comuna. Permet que una única interfície representi diferents formes subjacents (tipus de dades).

Conceptes Clau del Polimorfisme

  • Definició: El polimorfisme significa "moltes formes" i permet que els mètodes facin coses diferents segons l'objecte sobre el qual actuen.
  • Tipus de Polimorfisme:
    • Polimorfisme en temps de compilació (Sobrecàrrega de Mètodes)
    • Polimorfisme en temps d'execució (Sobreescriptura de Mètodes)
  • Beneficis:
    • Reutilització de codi
    • Flexibilitat i mantenibilitat
    • Simplifica la llegibilitat del codi

Polimorfisme en temps de compilació (Sobrecàrrega de Mètodes)

La Sobrecàrrega de Mètodes permet que múltiples mètodes en la mateixa classe tinguin el mateix nom però diferents paràmetres.

Exemple de Sobrecàrrega de Mètodes

public class MathOperations
{
    // Mètode per sumar dos enters
    public int Add(int a, int b)
    {
        return a + b;
    }

    // Mètode per sumar tres enters
    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }

    // Mètode per sumar dos dobles
    public double Add(double a, double b)
    {
        return a + b;
    }
}

class Program
{
    static void Main()
    {
        MathOperations math = new MathOperations();
        Console.WriteLine(math.Add(2, 3)); // Sortida: 5
        Console.WriteLine(math.Add(2, 3, 4)); // Sortida: 9
        Console.WriteLine(math.Add(2.5, 3.5)); // Sortida: 6.0
    }
}

En aquest exemple, el mètode Add està sobrecarregat amb diferents llistes de paràmetres.

Polimorfisme en temps d'execució (Sobreescriptura de Mètodes)

La Sobreescriptura de Mètodes permet que una subclasse proporcioni una implementació específica d'un mètode que ja està definit en la seva superclasse.

Exemple de Sobreescriptura de Mètodes

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Animal fa un so");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("El gos borda");
    }
}

public class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("El gat miola");
    }
}

class Program
{
    static void Main()
    {
        Animal myDog = new Dog();
        Animal myCat = new Cat();

        myDog.MakeSound(); // Sortida: El gos borda
        myCat.MakeSound(); // Sortida: El gat miola
    }
}

En aquest exemple, el mètode MakeSound està sobreescrit en les classes Dog i Cat per proporcionar implementacions específiques.

Polimorfisme amb Interfícies

Les interfícies també es poden utilitzar per aconseguir polimorfisme. Una classe pot implementar múltiples interfícies, i una interfície pot ser implementada per múltiples classes.

Exemple de Polimorfisme amb Interfícies

public interface IShape
{
    void Draw();
}

public class Circle : IShape
{
    public void Draw()
    {
        Console.WriteLine("Dibuixant un Cercle");
    }
}

public class Square : IShape
{
    public void Draw()
    {
        Console.WriteLine("Dibuixant un Quadrat");
    }
}

class Program
{
    static void Main()
    {
        IShape shape1 = new Circle();
        IShape shape2 = new Square();

        shape1.Draw(); // Sortida: Dibuixant un Cercle
        shape2.Draw(); // Sortida: Dibuixant un Quadrat
    }
}

En aquest exemple, les classes Circle i Square implementen la interfície IShape, i el mètode Draw es crida de manera polimòrfica.

Resum

El polimorfisme en C# permet flexibilitat i reutilització en el codi permetent que els objectes siguin tractats com a instàncies de la seva classe pare o interfície. Es pot aconseguir mitjançant la sobrecàrrega de mètodes (polimorfisme en temps de compilació) i la sobreescriptura de mètodes (polimorfisme en temps d'execució). Entendre i utilitzar el polimorfisme és essencial per escriure codi orientat a objectes eficient i mantenible.
