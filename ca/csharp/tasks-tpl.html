Introducció a les Tasques i TPL

La Biblioteca de Tasques Paral·leles (TPL) és un conjunt de tipus públics i APIs en l'espai de noms System.Threading.Tasks que simplifica el procés d'afegir paral·lelisme i concurrència a les aplicacions. És un component clau del .NET Framework i està dissenyada per facilitar l'escriptura de codi paral·lel eficient, escalable i mantenible.

Conceptes Clau

  • Task: Representa una operació asíncrona.
  • Task Parallel Library (TPL): Proporciona una abstracció de nivell superior per a la programació paral·lela en comparació amb el threading tradicional.
  • Programació Asíncrona: Permet que les tasques s'executin independentment del flux principal del programa, millorant la capacitat de resposta i el rendiment.

Creació i Execució de Tasques

Crear i executar tasques en C# és senzill amb la TPL.

Exemple: Creació d'una Tasca Simple

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task myTask = new Task(() => Console.WriteLine("Hola des de la Tasca!"));
        myTask.Start();
        myTask.Wait(); // Espera que la tasca es completi
    }
}
  • Explicació:
    • Task myTask = new Task(...): Crea una nova tasca.
    • myTask.Start(): Inicia la tasca.
    • myTask.Wait(): Espera que la tasca es completi abans de continuar.

Exemple: Utilitzant Task.Run

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task.Run(() => Console.WriteLine("Hola des de Task.Run!")).Wait();
    }
}
  • Explicació:
    • Task.Run(...): Una forma abreujada de crear i iniciar una tasca en un sol pas.
    • .Wait(): Assegura que el fil principal espera que la tasca es completi.

Continuacions de Tasques

Les tasques es poden enllaçar utilitzant continuacions, permetent especificar accions que es realitzaran després que una tasca es completi.

Exemple: Continuació de Tasques

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task firstTask = Task.Run(() => Console.WriteLine("Primera Tasca"));
        Task continuation = firstTask.ContinueWith(t => Console.WriteLine("Tasca de Continuació"));
        continuation.Wait();
    }
}
  • Explicació:
    • firstTask.ContinueWith(...): Especifica una tasca de continuació que s'executa després que firstTask es completi.

Gestió d'Exceptions en Tasques

Les tasques poden llençar excepcions, i gestionar-les correctament és crucial per a aplicacions robustes.

Exemple: Gestió d'Exceptions

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task task = Task.Run(() => { throw new InvalidOperationException("Error en la Tasca!"); });

        try
        {
            task.Wait();
        }
        catch (AggregateException ex)
        {
            foreach (var innerEx in ex.InnerExceptions)
            {
                Console.WriteLine(innerEx.Message);
            }
        }
    }
}
  • Explicació:
    • AggregateException: Quan una tasca llença una excepció, es troba encapsulada en una AggregateException.
    • ex.InnerExceptions: Conté totes les excepcions llençades per la tasca.

Programació Paral·lela amb TPL

La TPL també proporciona mètodes per a bucles paral·lels i altres construccions paral·leles.

Exemple: Parallel.For

using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Parallel.For(0, 10, i =>
        {
            Console.WriteLine($"Processant {i}");
        });
    }
}
  • Explicació:
    • Parallel.For(...): Executa un bucle for en el qual les iteracions poden executar-se en paral·lel.

Exemple: Parallel.ForEach

using System;
using System.Collections.Generic;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        List<int> numbers = new List<int> { 1, 2, 3, 4, 5 };

        Parallel.ForEach(numbers, number =>
        {
            Console.WriteLine($"Processant {number}");
        });
    }
}
  • Explicació:
    • Parallel.ForEach(...): Executa un bucle foreach en el qual les iteracions poden executar-se en paral·lel.

Característiques Avançades de les Tasques

Per a escenaris avançats, la TPL proporciona característiques més sofisticades com la cancel·lació de tasques i la notificació de progrés.

Exemple: Cancel·lació de Tasques

using System;
using System.Threading;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        CancellationTokenSource cts = new CancellationTokenSource();

        Task task = Task.Run(() =>
        {
            for (int i = 0; i < 10; i++)
            {
                if (cts.Token.IsCancellationRequested)
                {
                    Console.WriteLine("La tasca ha estat cancel·lada");
                    return;
                }
                Console.WriteLine($"Processant {i}");
                Thread.Sleep(500);
            }
        }, cts.Token);

        Thread.Sleep(2000); // Simula una mica de treball
        cts.Cancel(); // Sol·licita la cancel·lació

        try
        {
            task.Wait();
        }
        catch (AggregateException ex)
        {
            foreach (var innerEx in ex.InnerExceptions)
            {
                Console.WriteLine(innerEx.Message);
            }
        }
    }
}
  • Explicació:
    • CancellationTokenSource: Proporciona una manera de senyalitzar la cancel·lació.
    • cts.Token.IsCancellationRequested: Comprova si s'ha sol·licitat la cancel·lació.

Conclusió

La Biblioteca de Tasques Paral·leles (TPL) en C# proporciona una manera potent i flexible d'escriure codi paral·lel i asíncron. Comprenent les tasques, les continuacions, la gestió d'excepcions i les característiques avançades com la cancel·lació, pots crear aplicacions eficients i responsives. Tant si ets un programador principiant com avançat, dominar la TPL millorarà significativament la teva capacitat per escriure codi C# d'alt rendiment.
