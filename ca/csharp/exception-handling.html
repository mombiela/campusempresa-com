Gestió d'Excepcions en C#
Definició i Crida de Mètodes ►

La gestió d'excepcions és un aspecte crucial del desenvolupament de programari robust. Permet als desenvolupadors gestionar i respondre als errors en temps d'execució de manera elegant, assegurant que l'aplicació pugui manejar situacions inesperades sense bloquejar-se.

Introducció a la Gestió d'Excepcions

  • Definició: La gestió d'excepcions és un mecanisme per gestionar errors en temps d'execució, permetent que el programa continuï la seva execució o finalitzi de manera elegant.
  • Importància: Ajuda a mantenir el flux normal de l'aplicació fins i tot quan es produeixen errors inesperats.

Conceptes Bàsics

Què és una Excepció?

  • Excepció: Un esdeveniment que es produeix durant l'execució d'un programa que interromp el flux normal d'instruccions.
  • Excepcions Comunes: NullReferenceException, IndexOutOfRangeException, InvalidOperationException, etc.

El Bloc try-catch

  • Bloc try: Conté el codi que podria llançar una excepció.
  • Bloc catch: Conté el codi que gestiona l'excepció.
try
{
    // Codi que pot llançar una excepció
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[5]);
}
catch (IndexOutOfRangeException ex)
{
    // Codi per gestionar l'excepció
    Console.WriteLine("S'ha produït un error: " + ex.Message);
}

El Bloc finally

  • Bloc finally: Conté codi que sempre s'executa, independentment de si s'ha llançat una excepció o no. Normalment s'utilitza per a activitats de neteja.
try
{
    // Codi que pot llançar una excepció
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[5]);
}
catch (IndexOutOfRangeException ex)
{
    // Codi per gestionar l'excepció
    Console.WriteLine("S'ha produït un error: " + ex.Message);
}
finally
{
    // Codi que sempre s'executarà
    Console.WriteLine("Execució completada.");
}

Conceptes Avançats

Llançar Excepcions

  • Instrucció throw: S'utilitza per llançar explícitament una excepció.
public void CheckAge(int age)
{
    if (age < 18)
    {
        throw new ArgumentException("L'edat ha de ser 18 o més.");
    }
}

Excepcions Personalitzades

  • Excepció Personalitzada: Crear excepcions definides per l'usuari ampliant la classe Exception.
public class InvalidAgeException : Exception
{
    public InvalidAgeException(string message) : base(message)
    {
    }
}

public void CheckAge(int age)
{
    if (age < 18)
    {
        throw new InvalidAgeException("L'edat ha de ser 18 o més.");
    }
}

Propagació d'Excepcions

  • Propagació: Si una excepció no es captura en el mètode actual, es propaga al mètode que l'ha cridat.
public void Method1()
{
    Method2();
}

public void Method2()
{
    throw new InvalidOperationException("S'ha produït un error a Method2.");
}

try
{
    Method1();
}
catch (InvalidOperationException ex)
{
    Console.WriteLine("S'ha capturat una excepció: " + ex.Message);
}

Filtratge d'Excepcions

  • Filtratge: Utilitzar múltiples blocs catch per gestionar diferents tipus d'excepcions.
try
{
    // Codi que pot llançar múltiples tipus d'excepcions
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[5]);
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine("Índex fora de rang: " + ex.Message);
}
catch (Exception ex)
{
    Console.WriteLine("S'ha produït un error: " + ex.Message);
}

Millors Pràctiques

  • Blocs Catch Específics: Captura excepcions específiques en lloc d'utilitzar un bloc catch general.
  • Evita Blocs Catch Buits: Sempre gestiona les excepcions o registra-les; no deixis blocs catch buits.
  • Utilitza finally per a la Neteja: Utilitza el bloc finally per alliberar recursos com ara manejadors de fitxers, connexions a bases de dades, etc.
  • Excepcions Personalitzades: Crea excepcions personalitzades per a escenaris d'error específics per fer el codi més llegible i mantenible.

Conclusió

La gestió d'excepcions és una part fonamental per escriure aplicacions C# robustes i fiables. Entenent i implementant tècniques adequades de gestió d'excepcions, els desenvolupadors poden assegurar-se que les seves aplicacions poden gestionar errors inesperats de manera elegant i mantenir una experiència d'usuari fluida.

  • Punts Clau:
    • Utilitza blocs try-catch per gestionar excepcions.
    • Utilitza blocs finally per a la neteja.
    • Crea i llança excepcions personalitzades per a escenaris d'error específics.
    • Segueix les millors pràctiques per escriure codi de gestió d'excepcions net i mantenible.

Dominant aquests conceptes, estaràs ben preparat per gestionar excepcions de manera efectiva en les teves aplicacions C#.
