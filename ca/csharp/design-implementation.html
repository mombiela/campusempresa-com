Introducció

El disseny i la implementació són fases crucials en el desenvolupament de programari. En aquesta secció, cobrim els principis de dissenyar aplicacions robustes i implementar-les de manera efectiva utilitzant C#. Aquest tema et guiarà des dels conceptes bàsics del disseny orientat a objectes fins a patrons de disseny avançats i millors pràctiques.

Conceptes Clau

  • Disseny Orientat a Objectes (OOD)
  • Principis SOLID
  • Patrons de Disseny
  • Refactorització de Codi
  • Proves Unitàries

Disseny Orientat a Objectes (OOD)

El disseny orientat a objectes és un paradigma de programació basat en el concepte d'"objectes", que poden contenir dades i codi per manipular aquestes dades.

Conceptes Clau en OOD

  • Classes i Objectes: Blocs fonamentals del OOD.
  • Herència: Mecanisme per crear una nova classe utilitzant propietats i mètodes d'una classe existent.
  • Polimorfisme: Capacitat de processar objectes de manera diferent segons el seu tipus de dada o classe.
  • Encapsulació: Agrupació de les dades i els mètodes que operen sobre les dades dins d'una unitat.
  • Abstracció: Amagar detalls complexos d'implementació i mostrar només les característiques necessàries.

Exemple

public class Animal
{
    public string Name { get; set; }
    public void Speak()
    {
        Console.WriteLine($"{Name} fa un so.");
    }
}

public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} borda.");
    }
}

public class Program
{
    public static void Main()
    {
        Dog myDog = new Dog();
        myDog.Name = "Buddy";
        myDog.Speak(); // Sortida: Buddy fa un so.
        myDog.Bark();  // Sortida: Buddy borda.
    }
}

Principis SOLID

SOLID és un acrònim de cinc principis de disseny destinats a fer els dissenys de programari més comprensibles, flexibles i mantenibles.

Principis

  • Principi de Responsabilitat Única (SRP): Una classe hauria de tenir només una raó per canviar.
  • Principi Obert/Tancat (OCP): Les entitats de programari haurien d'estar obertes a l'extensió però tancades a la modificació.
  • Principi de Substitució de Liskov (LSP): Els objectes d'una superclasse haurien de ser reemplaçables amb objectes d'una subclasse sense afectar la correcció del programa.
  • Principi de Segregació d'Interfícies (ISP): Els clients no haurien de ser forçats a dependre d'interfícies que no utilitzen.
  • Principi d'Inversió de Dependència (DIP): Els mòduls d'alt nivell no haurien de dependre dels mòduls de baix nivell. Ambdós haurien de dependre d'abstraccions.

Exemple

// Exemple SRP
public class Invoice
{
    public void CalculateTotal() { /*...*/ }
}

public class InvoicePrinter
{
    public void Print(Invoice invoice) { /*...*/ }
}

Patrons de Disseny

Els patrons de disseny són solucions típiques a problemes comuns en el disseny de programari. Són com plantilles que es poden aplicar a situacions de programació del món real.

Patrons de Disseny Comuns

  • Patrons Creacionals: Singleton, Factory, Abstract Factory
  • Patrons Estructurals: Adapter, Composite, Proxy
  • Patrons de Comportament: Observer, Strategy, Command

Exemple: Patró Singleton

public class Singleton
{
    private static Singleton instance = null;
    private Singleton() { }
    
    public static Singleton Instance
    {
        get
        {
            if (instance == null)
            {
                instance = new Singleton();
            }
            return instance;
        }
    }
}

Refactorització de Codi

La refactorització és el procés de reestructurar el codi existent sense canviar el seu comportament extern. Millora els atributs no funcionals del programari.

Tècniques

  • Extreure Mètode: Crear un nou mètode extreient codi d'un mètode existent.
  • Canviar Nom de Variable: Canviar el nom d'una variable per fer-la més descriptiva.
  • Incrustar Mètode: Reemplaçar una crida a un mètode amb el contingut del mètode.

Exemple

// Abans de la Refactorització
public void PrintOwing()
{
    PrintBanner();
    double outstanding = GetOutstanding();
    PrintDetails(outstanding);
}

// Després de la Refactorització
public void PrintOwing()
{
    PrintBanner();
    double outstanding = CalculateOutstanding();
    PrintDetails(outstanding);
}

private double CalculateOutstanding()
{
    double outstanding = 0.0;
    // Calcular l'import pendent
    return outstanding;
}

Proves Unitàries

Les proves unitàries impliquen provar components individuals del programari per assegurar-se que funcionen com s'espera.

Frameworks

  • NUnit
  • xUnit
  • MSTest

Exemple

using NUnit.Framework;

[TestFixture]
public class CalculatorTests
{
    [Test]
    public void Add_WhenCalled_ReturnsSumOfArguments()
    {
        var calculator = new Calculator();
        var result = calculator.Add(1, 2);
        Assert.AreEqual(3, result);
    }
}

public class Calculator
{
    public int Add(int a, int b)
    {
        return a + b;
    }
}

Conclusió

El disseny i la implementació en C# impliquen entendre i aplicar els principis orientats a objectes, adherir-se als principis SOLID, utilitzar patrons de disseny, refactoritzar el codi per a una millor mantenibilitat i assegurar la qualitat del codi mitjançant proves unitàries. Dominar aquests conceptes et permetrà construir aplicacions robustes, escalables i mantenibles.
