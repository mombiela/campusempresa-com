Gestió d'Excepcions en Codi Asíncron

Introducció

La gestió d'excepcions en codi asíncron pot ser un repte tant per a principiants com per a programadors avançats. Aquesta secció et guiarà a través dels conceptes bàsics de la gestió d'excepcions en codi asíncron, progressant cap a tècniques més avançades i millors pràctiques.

Conceptes Clau

  • Programació Asíncrona: Executar tasques en segon pla sense bloquejar el fil principal.
  • Excepcions: Errors que ocorren durant l'execució d'un programa.
  • Gestió d'Excepcions: Mecanismes per capturar i gestionar errors de manera elegant.

Gestió Bàsica d'Excepcions en Codi Asíncron

Ús de `try-catch` amb `async` i `await`

Quan treballes amb mètodes asíncrons, pots utilitzar blocs try-catch per gestionar excepcions igual que en codi síncron.

public async Task ExampleMethodAsync()
{
    try
    {
        await Task.Run(() => { throw new InvalidOperationException("An error occurred."); });
    }
    catch (InvalidOperationException ex)
    {
        Console.WriteLine($"Caught an exception: {ex.Message}");
    }
}

En aquest exemple:

  • Task.Run s'utilitza per simular una operació asíncrona que llança una excepció.
  • El bloc try-catch captura la InvalidOperationException i la gestiona imprimint un missatge a la consola.

Gestió d'Excepcions en Múltiples Mètodes Asíncrons

Propagació d'Excepcions

Les excepcions en mètodes asíncrons poden propagar-se cap amunt en la pila de crides. Pots capturar-les a un nivell superior si cal.

public async Task Method1Async()
{
    await Method2Async();
}

public async Task Method2Async()
{
    throw new InvalidOperationException("An error occurred in Method2Async.");
}

public async Task MainMethodAsync()
{
    try
    {
        await Method1Async();
    }
    catch (InvalidOperationException ex)
    {
        Console.WriteLine($"Caught an exception: {ex.Message}");
    }
}

En aquest exemple:

  • Method2Async llança una excepció.
  • Method1Async crida Method2Async però no gestiona l'excepció.
  • MainMethodAsync crida Method1Async i captura l'excepció propagada.

Gestió Avançada d'Excepcions

Ús de `Task.WhenAll` amb Gestió d'Excepcions

Quan executes múltiples tasques simultàniament, pots utilitzar Task.WhenAll per esperar que totes les tasques es completin i gestionar les excepcions.

public async Task RunMultipleTasksAsync()
{
    var task1 = Task.Run(() => { throw new InvalidOperationException("Task1 failed."); });
    var task2 = Task.Run(() => { throw new InvalidOperationException("Task2 failed."); });

    try
    {
        await Task.WhenAll(task1, task2);
    }
    catch (Exception ex)
    {
        foreach (var innerException in ex.InnerExceptions)
        {
            Console.WriteLine($"Caught an exception: {innerException.Message}");
        }
    }
}

En aquest exemple:

  • Es creen dues tasques, ambdues llancen excepcions.
  • Task.WhenAll espera que ambdues tasques es completin.
  • El bloc catch itera a través de InnerExceptions per gestionar cada excepció individualment.

Ús de la Propietat `Task.Exception`

També pots accedir a la propietat Exception d'una Task per gestionar excepcions després que la tasca s'hagi completat.

public async Task HandleTaskExceptionAsync()
{
    var task = Task.Run(() => { throw new InvalidOperationException("Task failed."); });

    try
    {
        await task;
    }
    catch
    {
        if (task.Exception != null)
        {
            foreach (var innerException in task.Exception.InnerExceptions)
            {
                Console.WriteLine($"Caught an exception: {innerException.Message}");
            }
        }
    }
}

En aquest exemple:

  • Es crea una tasca que llança una excepció.
  • El bloc catch comprova la propietat Exception de la tasca i gestiona cada excepció interna.

Conclusió

La gestió d'excepcions en codi asíncron és crucial per construir aplicacions robustes i fiables. Entenent i aplicant les tècniques discutides en aquesta secció, pots gestionar efectivament els errors en els teus mètodes asíncrons. Recorda:

  • Utilitza blocs try-catch per gestionar excepcions en mètodes asíncrons.
  • Propaga excepcions a nivells superiors quan sigui necessari.
  • Utilitza Task.WhenAll i la propietat Exception de les tasques per a escenaris avançats de gestió d'excepcions.

Amb aquestes eines i tècniques, estaràs ben preparat per gestionar excepcions en el teu codi asíncron en C#, assegurant que les teves aplicacions funcionin sense problemes fins i tot quan es produeixin errors inesperats.
