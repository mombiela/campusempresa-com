El Projecte Sobre nosaltres Contribuir Donacions Llicència

L'herència és un concepte fonamental en la programació orientada a objectes (OOP) que permet a una classe heretar propietats i mètodes d'una altra classe. Això promou la reutilització del codi i estableix una jerarquia natural entre les classes.

Conceptes Clau

  • Classe Base (Classe Pare): La classe les propietats i mètodes de la qual són heretats.
  • Classe Derivada (Classe Filla): La classe que hereta de la classe base.
  • Modificador d'Accés protected: Permet que un membre sigui accessible dins de la seva pròpia classe i per instàncies de la classe derivada.
  • Paraula Clau virtual: Indica que un mètode o propietat pot ser sobreescrit en una classe derivada.
  • Paraula Clau override: S'utilitza per estendre o modificar la implementació virtual/abstracta d'un mètode, propietat, indexador o esdeveniment heretat.
  • Paraula Clau sealed: Impedeix que una classe sigui heretada o que un mètode sigui sobreescrit.

Herència Bàsica

Exemple

// Classe base
public class Animal
{
    public string Name { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }
}

// Classe derivada
public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} is barking.");
    }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.Name = "Buddy";
        myDog.Eat();  // Mètode heretat
        myDog.Bark(); // Mètode de la classe derivada
    }
}

Explicació

  • La classe Animal és la classe base amb una propietat Name i un mètode Eat.
  • La classe Dog és la classe derivada que hereta de Animal i afegeix un nou mètode Bark.
  • En el mètode Main, es crea una instància de Dog, i es criden tant els mètodes heretats com els nous.

Modificadors d'Accés en l'Herència

Exemple

public class Animal
{
    protected string Name { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }
}

public class Dog : Animal
{
    public void SetName(string name)
    {
        Name = name; // Accedint al membre protegit
    }

    public void Bark()
    {
        Console.WriteLine($"{Name} is barking.");
    }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.SetName("Buddy");
        myDog.Eat();
        myDog.Bark();
    }
}

Explicació

  • La propietat Name en la classe Animal està marcada com protected, fent-la accessible dins de la classe Dog.
  • El mètode SetName en la classe Dog estableix la propietat Name, demostrant l'ús del modificador d'accés protected.

Sobreescriptura de Mètodes

Exemple

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Some generic animal sound.");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Bark!");
    }
}

class Program
{
    static void Main()
    {
        Animal myAnimal = new Animal();
        myAnimal.MakeSound(); // Sortida: Some generic animal sound.

        Dog myDog = new Dog();
        myDog.MakeSound(); // Sortida: Bark!
    }
}

Explicació

  • El mètode MakeSound en la classe Animal està marcat com virtual, permetent que sigui sobreescrit.
  • La classe Dog sobreescriu el mètode MakeSound per proporcionar una implementació específica.

Classes i Mètodes Segellats

Exemple

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Some generic animal sound.");
    }
}

public class Dog : Animal
{
    public sealed override void MakeSound()
    {
        Console.WriteLine("Bark!");
    }
}

// Això causarà un error en temps de compilació
public class Bulldog : Dog
{
    // No es pot sobreescriure el mètode segellat
    // public override void MakeSound()
    // {
    //     Console.WriteLine("Bulldog bark!");
    // }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.MakeSound(); // Sortida: Bark!
    }
}

Explicació

  • El mètode MakeSound en la classe Dog està marcat com sealed, impedint una posterior sobreescriptura en classes derivades com Bulldog.

Conclusió

L'herència en C# permet la creació d'una jerarquia de classes on les classes derivades hereten i amplien la funcionalitat de les classes base. Els conceptes clau inclouen l'ús de modificadors d'accés, les paraules clau virtual i override, i la paraula clau sealed per controlar l'herència i la sobreescriptura de mètodes. Entendre aquests conceptes és crucial per dissenyar programari orientat a objectes robust i reutilitzable.
