Introducció

Les Futures i les Promeses són construccions poderoses en Scala que et permeten treballar amb computacions concurrents i asíncrones. Són essencials per escriure codi no bloquejant i gestionar tasques que poden completar-se en algun moment del futur.

Futures

Una Future representa un valor que pot no estar disponible encara. És un marcador de posició per a un resultat que eventualment serà calculat.

Creació de Futures

Pots crear una Future utilitzant l'objecte company Future i un ExecutionContext implícit.

import scala.concurrent._
import ExecutionContext.Implicits.global

val future: Future[Int] = Future {
  // Simula una computació de llarga durada
  Thread.sleep(1000)
  42
}

Treballant amb Futures

Les Futures proporcionen diversos mètodes per treballar amb els seus resultats un cop estan disponibles.

onComplete

El mètode onComplete et permet especificar una funció de retorn que s'executarà quan la future es completi.

future.onComplete {
  case Success(value) => println(s"El resultat és $value")
  case Failure(exception) => println(s"S'ha produït un error: ${exception.getMessage}")
}

map i flatMap

Pots transformar el resultat d'una future utilitzant map i flatMap.

val transformedFuture: Future[String] = future.map(value => s"La resposta és $value")

for-comprehensions

Les for-comprehensions proporcionen una manera més llegible de concatenar múltiples futures.

val future1 = Future { 10 }
val future2 = Future { 20 }

val combinedFuture: Future[Int] = for {
  value1 <- future1
  value2 <- future2
} yield value1 + value2

Promeses

Una Promise és un contenidor d'assignació única que completa una Future. Pots pensar en una Promise com una manera de controlar manualment la finalització d'una Future.

Creació de Promeses

Pots crear una Promise utilitzant l'objecte company Promise.

val promise: Promise[Int] = Promise[Int]()
val futureFromPromise: Future[Int] = promise.future

Completant Promeses

Pots completar una Promise cridant success o failure.

promise.success(42)
// o
promise.failure(new Exception("Alguna cosa ha anat malament"))

Exemple: Utilitzant Promeses per Completar Futures

Aquí tens un exemple que demostra com utilitzar una Promise per completar una Future.

import scala.concurrent._
import ExecutionContext.Implicits.global

val promise = Promise[Int]()
val future = promise.future

future.onComplete {
  case Success(value) => println(s"El resultat és $value")
  case Failure(exception) => println(s"S'ha produït un error: ${exception.getMessage}")
}

// Simula alguna computació asíncrona
Future {
  Thread.sleep(1000)
  promise.success(42)
}

Comparació: Futures vs Promeses

| Característica | Future | Promise | |------------------|---------------------------------|---------------------------------| | Creació | Creat amb l'objecte Future | Creat amb l'objecte Promise | | Finalització | Completat per la computació | Completat manualment per l'usuari| | Llegibilitat | Només lectura | Lectura-escriptura | | Cas d'ús | Resultat de computació asíncrona| Control manual sobre la finalització|

Conclusió

Les Futures i les Promeses són eines essencials per gestionar la programació asíncrona i concurrent en Scala. Les Futures et permeten treballar amb valors que estaran disponibles en el futur, mentre que les Promeses et donen control sobre quan i com es proporcionen aquests valors. Entenent i utilitzant aquestes construccions, pots escriure codi més eficient i no bloquejant.
