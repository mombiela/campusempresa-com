Exemple: Configuració d'un nou projecte Play

sbt new playframework/play-scala-seed.g8
cd play-scala-seed
sbt run
  • Explicació: Aquesta seqüència de comandes inicialitza un nou projecte Play utilitzant la plantilla Play Scala seed i executa el servidor de desenvolupament.

Conceptes bàsics

  • Comprensió de l'arquitectura MVC

    • Model: Dades i lògica de negoci
    • Vista: Interfície d'usuari
    • Controlador: Gestió de sol·licituds i generació de respostes

  • Enrutament

    • Definició de rutes a conf/routes
    • Exemple: 
      GET     /                           controllers.HomeController.index
POST    /submit                     controllers.HomeController.submit
    • Explicació: Aquest fitxer de rutes mapatge les sol·licituds HTTP a les accions del controlador.

  • Controladors

    • Creació d'un controlador bàsic
    • Exemple: 
      package controllers

import javax.inject._
import play.api.mvc._

@Singleton
class HomeController @Inject()(val controllerComponents: ControllerComponents) extends BaseController {

  def index() = Action { implicit request: Request[AnyContent] =>
    Ok("Hola, Play Framework!")
  }
}
    • Explicació: Aquest controlador gestiona una sol·licitud GET a l'URL arrel i respon amb un missatge de text simple.

Treballant amb Vistes

  • Utilització de plantilles Twirl

    • Creació d'una vista HTML bàsica
    • Exemple: 
      @(message: String)

<!DOCTYPE html>
<html>
  <head>
    <title>Benvingut a Play</title>
  </head>
  <body>
    <h1>@message</h1>
  </body>
</html>
    • Explicació: Aquesta plantilla Twirl pren un paràmetre message i el mostra dins d'una estructura HTML.

  • Integració de Vistes amb Controladors

    • Exemple: 
      def index() = Action { implicit request: Request[AnyContent] =>
  Ok(views.html.index("Benvingut a Play Framework!"))
}
    • Explicació: Aquesta acció del controlador renderitza la vista index.scala.html amb un missatge.

Temes avançats

  • Gestió de Formularis

    • Creació i processament de formularis
    • Exemple: 
      import play.api.data._
import play.api.data.Forms._

case class UserData(name: String, age: Int)

val userForm = Form(
  mapping(
    "name" -> nonEmptyText,
    "age" -> number(min = 0, max = 100)
  )(UserData.apply)(UserData.unapply)
)

  • Integració de Bases de Dades

    • Utilització de Slick per a operacions de bases de dades
    • Exemple: 
      import slick.jdbc.H2Profile.api._

case class User(id: Long, name: String, age: Int)

class Users(tag: Tag) extends Table[User](tag, "USERS") {
  def id = column[Long]("ID", O.PrimaryKey, O.AutoInc)
  def name = column[String]("NAME")
  def age = column[Int]("AGE")
  def * = (id, name, age) <> (User.tupled, User.unapply)
}

val users = TableQuery[Users]
    • Explicació: Això defineix un esquema de taula User utilitzant Slick.

  • Programació Asíncrona

    • Utilització de Futures en Play
    • Exemple: 
      import scala.concurrent.Future
import scala.concurrent.ExecutionContext.Implicits.global

def asyncAction() = Action.async {
  val futureResult: Future[String] = Future {
    // Simular un càlcul de llarga durada
    Thread.sleep(1000)
    "Resultat Asíncron"
  }
  futureResult.map(result => Ok(result))
}
    • Explicació: Aquesta acció del controlador demostra com gestionar operacions asíncrones utilitzant Futures de Scala.
