Introducció

En aquesta secció, esbossarem el projecte en el qual treballaràs al llarg d'aquest curs de Scala. Aquest projecte t'ajudarà a aplicar els conceptes que aprens d'una manera pràctica i pràctica. Al final del curs, tindràs una aplicació totalment funcional que mostrarà les teves habilitats de programació en Scala.

Descripció del projecte

El projecte que desenvoluparàs és una Aplicació de Gestió de Tasques. Aquesta aplicació permetrà als usuaris crear, actualitzar, eliminar i veure tasques. També inclourà funcions com la categorització de tasques, dates de venciment i nivells de prioritat.

Funcions clau

  • Autenticació d'Usuaris: Els usuaris poden registrar-se, iniciar sessió i tancar sessió.
  • Operacions CRUD de Tasques: Els usuaris poden crear, llegir, actualitzar i eliminar tasques.
  • Categorització de Tasques: Les tasques es poden categoritzar en diferents grups.
  • Dates de Venciment i Nivells de Prioritat: Les tasques poden tenir dates de venciment i nivells de prioritat assignats.
  • Cerca i Filtrat: Els usuaris poden cercar i filtrar tasques segons diversos criteris.

Requisits

Per completar amb èxit aquest projecte, hauràs de complir els següents requisits:

Requisits Tècnics

  • Versió de Scala: Utilitza Scala 2.13 o posterior.
  • Eina de Construcció: Utilitza SBT (Simple Build Tool) per gestionar les dependències i construir el projecte.
  • Base de Dades: Utilitza una base de dades en memòria com H2 per al desenvolupament i les proves.
  • Frameworks i Llibreries:
    • Play Framework: Per construir l'aplicació web.
    • Slick: Per a la interacció amb la base de dades.
    • ScalaTest: Per escriure i executar proves.

Requisits Funcionals

  • Autenticació d'Usuaris:
    • Els usuaris haurien de poder registrar-se amb un nom d'usuari i una contrasenya.
    • Els usuaris haurien de poder iniciar sessió i tancar sessió.
  • Gestió de Tasques:
    • Els usuaris haurien de poder crear noves tasques amb un títol, descripció, data de venciment i nivell de prioritat.
    • Els usuaris haurien de poder veure una llista de totes les seves tasques.
    • Els usuaris haurien de poder actualitzar els detalls d'una tasca.
    • Els usuaris haurien de poder eliminar una tasca.
  • Categorització i Filtrat:
    • Els usuaris haurien de poder categoritzar les tasques en diferents grups.
    • Els usuaris haurien de poder filtrar les tasques segons la categoria, la data de venciment i el nivell de prioritat.

Exemple de Codi

Aquí tens un exemple senzill de com podries definir una classe de cas Task en Scala:

case class Task(
  id: Long,
  title: String,
  description: String,
  dueDate: Option[java.time.LocalDate],
  priority: PriorityLevel,
  category: Option[String]
)

sealed trait PriorityLevel
case object High extends PriorityLevel
case object Medium extends PriorityLevel
case object Low extends PriorityLevel

Explicació

  • Classe de Cas: Task és una classe de cas que representa una tasca amb diversos atributs.
  • Tipus Option: dueDate i category són camps opcionals, representats pel tipus Option.
  • Trait Segellat: PriorityLevel és un trait segellat amb tres valors possibles: High, Medium i Low.
