Introducció a SBT

SBT (Simple Build Tool) és una eina de construcció popular per a projectes Scala. S'utilitza per compilar, executar i provar codi Scala, així com per gestionar dependències i automatitzar tasques.

Conceptes Clau

  • Definició de Construcció: La definició de construcció s'escriu en un fitxer anomenat build.sbt.
  • Tasques: Accions que SBT pot realitzar, com compilar codi o executar proves.
  • Configuracions: Opcions de configuració per a la teva construcció, com la versió de Scala o les dependències de la biblioteca.
  • Plugins: Extensions que afegeixen funcionalitat a SBT.

Configuració de SBT

Per començar amb SBT, necessites instal·lar-lo i configurar una estructura bàsica de projecte.

Instal·lació

  1. Descarregar i Instal·lar SBT: Segueix les instruccions a la pàgina oficial de SBT.
  2. Verificar la Instal·lació: Obre una terminal i executa: 
    sbt sbtVersion
    Això hauria d'imprimir la versió de SBT instal·lada.

Creació d'un Nou Projecte

  1. Crear el Directori del Projecte: 
    mkdir my-scala-project
cd my-scala-project
  2. Crear el Fitxer de Definició de Construcció: Crea un fitxer anomenat build.sbt amb el següent contingut: 
    name := "MyScalaProject"

version := "0.1"

scalaVersion := "2.13.6"
  3. Crear el Directori de Fonts: 
    mkdir -p src/main/scala
mkdir -p src/test/scala

Gestió de Dependències

Les dependències són biblioteques externes que el teu projecte necessita. Pots afegir-les al teu fitxer build.sbt.

Afegir Dependències

  1. Trobar la Dependència: Cerca la biblioteca a Maven Central.
  2. Afegir a la Definició de Construcció: Afegeix la dependència al teu fitxer build.sbt: 
    libraryDependencies += "org.scalatest" %% "scalatest" % "3.2.9" % Test

Compilar i Executar Codi

SBT proporciona comandes per compilar i executar el teu codi Scala.

Compilar Codi

  1. Comanda de Compilació: A la terminal, executa: 
    sbt compile
    Això compilarà el codi al directori src/main/scala.

Executar Codi

  1. Crear Objecte Principal: Crea un fitxer src/main/scala/Main.scala: 
    object Main extends App {
  println("Hello, Scala!")
}
  2. Comanda d'Execució: A la terminal, executa: 
    sbt run

Proves amb SBT

Les proves són una part essencial del desenvolupament de programari. SBT s'integra bé amb marcs de proves com ScalaTest.

Escriure Proves

  1. Crear Fitxer de Prova: Crea un fitxer src/test/scala/MainSpec.scala: 
    import org.scalatest.flatspec.AnyFlatSpec

class MainSpec extends AnyFlatSpec {
  "The Main object" should "print Hello, Scala!" in {
    assert(Main.main(Array()) == "Hello, Scala!")
  }
}

Executar Proves

  1. Comanda de Prova: A la terminal, executa: 
    sbt test

Automatització de Tasques

SBT et permet automatitzar tasques repetitives utilitzant tasques personalitzades i plugins.

Tasques Personalitzades

  1. Definir Tasca Personalitzada: Afegeix el següent al teu build.sbt: 
    lazy val hello = taskKey[Unit]("Prints 'Hello, World!'")

hello := {
  println("Hello, World!")
}
  2. Executar Tasca Personalitzada: A la terminal, executa: 
    sbt hello

Conclusió

SBT és una eina poderosa per gestionar projectes Scala. Simplifica el procés de compilar, executar i provar el teu codi, així com gestionar dependències i automatitzar tasques. Dominant SBT, pots millorar significativament la teva productivitat i optimitzar el teu flux de treball de desenvolupament.
