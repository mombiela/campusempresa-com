La concurrència és un concepte fonamental en la programació moderna, que permet que múltiples computacions es duguin a terme simultàniament. En Scala, la concurrència és una característica poderosa que et pot ajudar a escriure aplicacions eficients i responsives. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de la concurrència en Scala.

Què és la Concurrència?

La concurrència implica múltiples seqüències d'operacions que succeeixen al mateix temps. És essencial per a:

  • Millorar el rendiment utilitzant múltiples nuclis de CPU.
  • Millorar la responsivitat en aplicacions, especialment en interfícies d'usuari.
  • Gestionar múltiples tasques de manera eficient, com ara manejar múltiples sol·licituds de clients en un servidor.

Concurrència en Scala

Scala proporciona diverses eines i biblioteques per manejar la concurrència, incloent:

  • Fils
  • Futures i Promises
  • Actors (Akka)

Fils

Els fils són la unitat bàsica de la concurrència. Permeten que múltiples parts d'un programa s'executin simultàniament.

Exemple: Crear i Executar un Fil

object ThreadExample extends App {
  val thread = new Thread(new Runnable {
    def run(): Unit = {
      println("Hola des d'un fil!")
    }
  })
  thread.start()
  thread.join() // Espera que el fil acabi
}
  • new Thread(new Runnable { ... }): Crea un nou fil.
  • thread.start(): Inicia el fil.
  • thread.join(): Espera que el fil acabi.

Futures i Promises

Futures i Promises proporcionen una abstracció de més alt nivell per a la concurrència, fent més fàcil treballar amb computacions asíncrones.

Exemple: Utilitzant Futures

import scala.concurrent.{Future, ExecutionContext}
import scala.util.{Success, Failure}
import ExecutionContext.Implicits.global

val future = Future {
  // Simula una computació de llarga durada
  Thread.sleep(1000)
  42
}

future.onComplete {
  case Success(value) => println(s"La resposta és $value")
  case Failure(e) => println(s"S'ha produït un error: ${e.getMessage}")
}
  • Future { ... }: Crea un future que es completarà amb el resultat del bloc.
  • future.onComplete { ... }: Registra una callback per ser executada quan el future es completi.

Actors (Akka)

Els actors són un model de concurrència més avançat proporcionat per la biblioteca Akka. Encapsulen estat i comportament, i es comuniquen mitjançant el pas de missatges.

Exemple: Actor Bàsic

import akka.actor.{Actor, ActorSystem, Props}

class HelloActor extends Actor {
  def receive = {
    case "hola" => println("Hola de tornada!")
    case _ => println("Missatge desconegut")
  }
}

object ActorExample extends App {
  val system = ActorSystem("HelloSystem")
  val helloActor = system.actorOf(Props[HelloActor], name = "helloactor")

  helloActor ! "hola"
  helloActor ! "adéu"

  system.terminate()
}
  • ActorSystem: Gestiona el cicle de vida dels actors.
  • Props[HelloActor]: Objecte de configuració per crear un actor.
  • helloActor ! "hola": Envia un missatge a l'actor.

Conclusió

La concurrència és una eina poderosa en Scala que et pot ajudar a escriure aplicacions eficients i responsives. Entenent i utilitzant fils, futures i actors, pots gestionar múltiples tasques simultàniament i millorar el rendiment dels teus programes. A mesura que progressis, trobaràs que aquestes eines es poden combinar i estendre per manejar escenaris de concurrència encara més complexos.
