Introducció

L'estil i les convencions de codi són crucials per escriure codi net, llegible i mantenible. En aquesta secció, cobrirem les directrius d'estil essencials i les convencions per a la programació en Scala. Seguir aquestes pràctiques t'ajudarà a escriure codi que sigui consistent i fàcil d'entendre, tant per a tu com per a altres persones que puguin treballar amb el teu codi en el futur.

Convencions de Nomenclatura

Les convencions de nomenclatura són essencials per a la llegibilitat i la mantenibilitat. Aquí tens algunes directrius clau:

  • Classes i Traits: Utilitza PascalCase.

    class MyClass
trait MyTrait

  • Objectes i Paquets: Utilitza camelCase.

    object myObject
package myPackage

  • Variables i Mètodes: Utilitza camelCase.

    val myVariable = 10
def myMethod(): Unit = {}

  • Constants: Utilitza UPPER_SNAKE_CASE.

    val MAX_SIZE = 100

Indentació i Espaiat

Una correcta indentació i espaiat fan que el teu codi sigui més llegible.

  • Indentació: Utilitza 2 espais per nivell d'indentació.

    def myMethod(): Unit = {
  val x = 10
  if (x > 5) {
    println("x és més gran que 5")
  }
}

  • Espaiat: Utilitza espais al voltant dels operadors i després de les comes.

    val sum = a + b
val list = List(1, 2, 3)

Longitud de Línia

Mantingues les línies de codi a una longitud raonable, idealment no més de 80 caràcters. Això fa que el teu codi sigui més fàcil de llegir en diversos dispositius i mides de pantalla.

Comentaris

Els comentaris són essencials per explicar el propòsit i la funcionalitat del teu codi.

  • Comentaris d'una línia: Utilitza // per a comentaris d'una línia.

    // Aquest és un comentari d'una línia
val x = 10

  • Comentaris de múltiples línies: Utilitza /* ... */ per a comentaris de múltiples línies.

    /*
 * Aquest és un comentari de múltiples línies
 * que s'estén per diverses línies.
 */
val y = 20

  • Comentaris de documentació: Utilitza /** ... */ per a comentaris de documentació.

    /**
 * Aquest mètode suma dos números.
 * @param a El primer número
 * @param b El segon número
 * @return La suma de a i b
 */
def add(a: Int, b: Int): Int = a + b

Organització del Codi

Organitza el teu codi lògicament per fer-lo més fàcil de navegar.

  • Paquets: Agrupa classes i objectes relacionats en paquets.

    package com.example.myapp

  • Imports: Importa només el que necessites.

    import scala.collection.mutable.ListBuffer

Convencions de Programació Funcional

Scala fomenta la programació funcional. Aquí tens algunes convencions a seguir:

  • Immutabilitat: Prefereix estructures de dades immutables.

    val immutableList = List(1, 2, 3)

  • Funcions d'ordre superior: Utilitza funcions d'ordre superior quan sigui apropiat.

    val numbers = List(1, 2, 3, 4)
val doubled = numbers.map(_ * 2)

  • Coincidència de patrons: Utilitza la coincidència de patrons per a estructures de dades complexes.

    val result = x match {
  case 1 => "un"
  case 2 => "dos"
  case _ => "altre"
}

Gestió d'Errors

Gestiona els errors de manera elegant utilitzant les potents característiques de gestió d'errors de Scala.

  • Try, Success, i Failure: Utilitza aquests per gestionar excepcions.

    import scala.util.{Try, Success, Failure}

val result = Try(10 / 0) match {
  case Success(value) => s"Èxit: $value"
  case Failure(exception) => s"Error: ${exception.getMessage}"
}

  • Option: Utilitza Option per a valors que poden o no estar presents.

    def findUser(id: Int): Option[String] = {
  val users = Map(1 -> "Alice", 2 -> "Bob")
  users.get(id)
}

val user = findUser(1) match {
  case Some(name) => s"Usuari trobat: $name"
  case None => "Usuari no trobat"
}

Conclusió

Seguir aquestes convencions d'estil i codi t'ajudarà a escriure codi Scala net, llegible i mantenible. La consistència en la nomenclatura, la indentació, l'espaiat i l'organització fa que el teu codi sigui més fàcil d'entendre i col·laborar-hi. Adopta les característiques de programació funcional de Scala i gestiona els errors de manera elegant per crear aplicacions robustes. Bona programació!
