Col·leccions Paral·leles
Macros i Metaprogramació ►

Les col·leccions paral·leles a Scala proporcionen una manera d'aprofitar els processadors multi-nucli realitzant operacions en col·leccions en paral·lel. Això pot millorar significativament el rendiment per a certs tipus de tasques. Aquest tema cobrirà els conceptes bàsics de les col·leccions paral·leles, com utilitzar-les i algunes millors pràctiques.

Introducció a les Col·leccions Paral·leles

Les col·leccions paral·leles són una part de la biblioteca estàndard de Scala i estan dissenyades per facilitar la paral·lelització d'operacions en col·leccions. Proporcionen una implementació paral·lela de les operacions estàndard de col·leccions com map, filter, reduce, etc.

Conceptes Clau

  • Paral·lelisme: El procés d'executar múltiples càlculs simultàniament.
  • Concurrència: La capacitat de gestionar múltiples tasques al mateix temps.
  • Seguretat de Fil: Assegurar que les dades compartides siguin accedides per múltiples fils de manera segura.

Ús de les Col·leccions Paral·leles

Convertir a Col·leccions Paral·leles

Per convertir una col·lecció estàndard a una col·lecció paral·lela, pots utilitzar el mètode .par.

val list = List(1, 2, 3, 4, 5)
val parList = list.par

Operacions Bàsiques

Les col·leccions paral·leles suporten les mateixes operacions que les col·leccions estàndard, però s'executen en paral·lel.

val list = List(1, 2, 3, 4, 5).par
val doubled = list.map(_ * 2)
println(doubled) // La sortida pot variar en ordre: ParVector(2, 4, 6, 8, 10)

Exemple: Map Paral·lel

val numbers = (1 a 1000000).toList.par
val squares = numbers.map(n => n * n)
println(squares.take(10)) // La sortida pot variar en ordre

Exemple: Reduce Paral·lel

val numbers = (1 a 1000000).toList.par
val sum = numbers.reduce(_ + _)
println(sum) // Sortida: 500000500000

Consideracions de Rendiment

Tot i que les col·leccions paral·leles poden millorar el rendiment, no sempre són la millor opció. Aquí hi ha algunes consideracions:

  • Sobrecàrrega: Les col·leccions paral·leles introdueixen una certa sobrecàrrega a causa de la gestió de fils.
  • Granularitat de Tasques: Les tasques haurien de ser prou grans per justificar la sobrecàrrega del paral·lelisme.
  • Immutabilitat: Assegura't que les operacions en col·leccions siguin segures per a fils i no causin efectes secundaris.

Exemple: Comparant Rendiment

import scala.util.Random
import scala.collection.parallel.CollectionConverters._

val size = 1000000
val list = List.fill(size)(Random.nextInt(100))

val startTime = System.nanoTime()
val sumSeq = list.sum
val endTime = System.nanoTime()
println(s"Sumatori seqüencial: $sumSeq, Temps: ${(endTime - startTime) / 1e6} ms")

val parList = list.par
val startTimePar = System.nanoTime()
val sumPar = parList.sum
val endTimePar = System.nanoTime()
println(s"Sumatori paral·lel: $sumPar, Temps: ${(endTimePar - startTimePar) / 1e6} ms")

Millors Pràctiques

  • Utilitza Immutabilitat: Prefereix col·leccions immutables per evitar problemes de concurrència.
  • Mesura el Rendiment: Mesura sempre el rendiment per assegurar-te que el paral·lelisme proporciona un benefici.
  • Evita Efectes Secundaris: Assegura't que les operacions en col·leccions no tinguin efectes secundaris.

Conclusió

Les col·leccions paral·leles a Scala proporcionen una manera poderosa d'aprofitar els processadors multi-nucli per a un millor rendiment. Convertint col·leccions estàndard a col·leccions paral·leles, pots realitzar operacions com map, filter i reduce en paral·lel. No obstant això, és important considerar la sobrecàrrega i assegurar-se que les tasques siguin prou grans per beneficiar-se del paral·lelisme. Mesura sempre el rendiment i segueix les millors pràctiques per evitar problemes comuns.

Entenent i utilitzant col·leccions paral·leles, pots escriure programes Scala més eficients i amb millor rendiment.
