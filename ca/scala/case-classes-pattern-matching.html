Introducció

En aquesta secció, explorarem dues característiques potents de Scala: les classes de cas i la coincidència de patrons. Aquestes característiques són essencials per escriure codi concís i llegible, especialment quan es treballa amb estructures de dades immutables i manipulacions de dades complexes.

Classes de cas

Què són les classes de cas?

Les classes de cas són un tipus especial de classe a Scala que estan optimitzades per a l'ús en la coincidència de patrons. Són immutables per defecte i vénen amb diverses característiques integrades que les fan molt convenients d'utilitzar.

Característiques clau de les classes de cas

  • Immutabilitat: Els camps són immutables per defecte.
  • toString, equals i hashCode automàtics: Aquests mètodes es generen automàticament.
  • Coincidència de patrons: Les classes de cas es poden descompondre utilitzant la coincidència de patrons.
  • Mètode copy: Proporciona una manera convenient de crear còpies modificades de les instàncies.
  • Mètode apply per defecte: Permet la instanciació sense la paraula clau new.

Definició d'una classe de cas

case class Person(name: String, age: Int)

En aquest exemple, Person és una classe de cas amb dos camps: name i age.

Ús de les classes de cas

val alice = Person("Alice", 25)
val bob = Person("Bob", 30)

// Accés als camps
println(alice.name)  // Sortida: Alice
println(bob.age)     // Sortida: 30

// Copiar amb modificacions
val olderAlice = alice.copy(age = 26)
println(olderAlice)  // Sortida: Person(Alice,26)

Coincidència de patrons

Què és la coincidència de patrons?

La coincidència de patrons és un mecanisme per comprovar un valor contra un patró. És una versió més potent de l'instrucció switch que es troba en molts altres llenguatges. Permet descompondre estructures de dades i executar codi basat en la seva forma i contingut.

Sintaxi bàsica

value match {
  case pattern1 => result1
  case pattern2 => result2
  // més casos
  case _ => defaultResult
}

Exemple amb classes de cas

def greet(person: Person): String = person match {
  case Person("Alice", _) => "Hello, Alice!"
  case Person(_, age) if age < 18 => "Hi there, young one!"
  case Person(name, age) => s"Hello, $name! You are $age years old."
}

val alice = Person("Alice", 25)
val bob = Person("Bob", 17)
val charlie = Person("Charlie", 30)

println(greet(alice))    // Sortida: Hello, Alice!
println(greet(bob))      // Sortida: Hi there, young one!
println(greet(charlie))  // Sortida: Hello, Charlie! You are 30 years old.

Coincidència de patrons amb trets segellats

Els trets segellats asseguren que totes les implementacions es coneixen en temps de compilació, cosa que fa que la coincidència de patrons sigui exhaustiva i segura.

sealed trait Shape
case class Circle(radius: Double) extends Shape
case class Rectangle(width: Double, height: Double) extends Shape

def area(shape: Shape): Double = shape match {
  case Circle(radius) => Math.PI * radius * radius
  case Rectangle(width, height) => width * height
}

val circle = Circle(5.0)
val rectangle = Rectangle(4.0, 6.0)

println(area(circle))      // Sortida: 78.53981633974483
println(area(rectangle))   // Sortida: 24.0

Exercicis

Exercici 1: Definir i utilitzar una classe de cas

  1. Defineix una classe de cas Book amb els camps title (String) i author (String).
  2. Crea una instància de Book.
  3. Imprimeix el title i l'author del llibre.

Exercici 2: Coincidència de patrons amb classes de cas

  1. Defineix un tret segellat Animal i les classes de cas Dog i Cat que l'extenguin.
  2. Escriu una funció describe que prengui un Animal i retorni una descripció basada en el tipus d'animal.
  3. Prova la funció amb instàncies de Dog i Cat.

Conclusió

Les classes de cas i la coincidència de patrons són característiques fonamentals a Scala que permeten escriure codi concís i expressiu. Les classes de cas proporcionen una manera convenient de definir estructures de dades immutables, mentre que la coincidència de patrons permet una descomposició elegant de dades i un flux de control. En dominar aquestes característiques, pots escriure codi Scala més llegible i mantenible.
