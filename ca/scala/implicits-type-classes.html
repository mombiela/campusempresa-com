Introducció als Implícits

Els implícits a Scala són una característica poderosa que permet un codi més concís i llegible en permetre que el compilador subministri automàticament certs valors o conversions. Es poden utilitzar per a:

  • Paràmetres implícits
  • Conversions implícites
  • Classes implícites

Paràmetres Implícits

Els paràmetres implícits permeten ometre certs arguments quan es crida a una funció, sempre que el compilador pugui trobar un valor implícit del tipus correcte en l'àmbit.

def greet(name: String)(implicit greeting: String): String = s"$greeting, $name!"

implicit val defaultGreeting: String = "Hello"

println(greet("Alice"))  // Output: Hello, Alice!

Conversions Implícites

Les conversions implícites converteixen automàticament un tipus a un altre quan és necessari.

implicit def intToString(x: Int): String = x.toString

val str: String = 42  // Converteix implícitament Int a String
println(str)  // Output: 42

Classes Implícites

Les classes implícites proporcionen una manera d'afegir mètodes a tipus existents sense modificar el seu codi font.

implicit class RichInt(val x: Int) extends AnyVal {
  def square: Int = x * x
}

println(4.square)  // Output: 16

Introducció a les Classes de Tipus

Les classes de tipus a Scala proporcionen una manera de definir comportament per a tipus sense modificar-los. Són un patró per a polimorfisme ad-hoc i sovint s'utilitzen en programació funcional.

Definició d'una Classe de Tipus

Una classe de tipus es defineix creant un trait que especifica el comportament.

trait Show[A] {
  def show(a: A): String
}

Proporcionar Instàncies per a Classes de Tipus

Les instàncies de la classe de tipus es proporcionen per a tipus específics.

implicit val intShow: Show[Int] = new Show[Int] {
  def show(a: Int): String = a.toString
}

implicit val stringShow: Show[String] = new Show[String] {
  def show(a: String): String = a
}

Ús de Classes de Tipus

Per utilitzar una classe de tipus, es pot definir un mètode que requereixi una instància implícita de la classe de tipus.

def show[A](a: A)(implicit s: Show[A]): String = s.show(a)

println(show(123))  // Output: 123
println(show("Hello"))  // Output: Hello

Combinació d'Implícits i Classes de Tipus

Els implícits i les classes de tipus es poden combinar per crear codi flexible i reutilitzable.

Exemple: Classe de Tipus d'Ordenació

Considerem una classe de tipus per ordenar elements.

trait Ordering[A] {
  def compare(x: A, y: A): Int
}

implicit val intOrdering: Ordering[Int] = new Ordering[Int] {
  def compare(x: Int, y: Int): Int = x - y
}

def sort[A](list: List[A])(implicit ord: Ordering[A]): List[A] = {
  list.sorted((x, y) => ord.compare(x, y))
}

println(sort(List(3, 1, 2)))  // Output: List(1, 2, 3)

Ús Avançat

Límits de Context

Els límits de context proporcionen una forma abreujada de requerir un paràmetre implícit d'una classe de tipus.

def showAll[A: Show](list: List[A]): String = list.map(show).mkString(", ")

println(showAll(List(1, 2, 3)))  // Output: 1, 2, 3

Regles de Resolució Implícita

Entendre com Scala resol els implícits és crucial per a l'ús avançat:

  • Àmbit local
  • Àmbit importat
  • Objectes companys del tipus

Prioritats Implícites

Es pot controlar quin implícit és escollit pel compilador utilitzant prioritats implícites.

object LowPriorityImplicits {
  implicit val intShow: Show[Int] = new Show[Int] {
    def show(a: Int): String = s"Low priority: $a"
  }
}

object HighPriorityImplicits extends LowPriorityImplicits {
  implicit override val intShow: Show[Int] = new Show[Int] {
    def show(a: Int): String = s"High priority: $a"
  }
}

import HighPriorityImplicits._

println(show(42))  // Output: High priority: 42

Conclusió

Els implícits i les classes de tipus són característiques poderoses a Scala que permeten codi concís, flexible i reutilitzable. En entendre i aprofitar aquestes característiques, es poden escriure programes Scala més expressius i mantenibles. Recorda utilitzar els implícits amb prudència per mantenir la claredat del codi i evitar comportaments inesperats.
