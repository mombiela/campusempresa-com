Les funcions d'ordre superior són un concepte fonamental en la programació funcional i s'utilitzen extensament en Scala. Permeten escriure codi més abstracte i reutilitzable. Aquest tema et guiarà des dels conceptes bàsics fins a l'ús avançat de les funcions d'ordre superior en Scala.

Què són les Funcions d'Ordre Superior?

Les funcions d'ordre superior són funcions que poden prendre altres funcions com a paràmetres, retornar funcions com a resultats, o fer ambdues coses. Això permet un alt nivell d'abstracció i reutilització de codi.

Conceptes Clau

  • Funció com a Paràmetre: Una funció que pren una altra funció com a argument.
  • Funció com a Valor de Retorn: Una funció que retorna una altra funció.
  • Funcions Anònimes: Funcions definides sense un nom, sovint utilitzades com a arguments per a funcions d'ordre superior.
  • Currying: Transformar una funció que pren múltiples arguments en una sèrie de funcions que cadascuna pren un sol argument.

Funció com a Paràmetre

Un ús comú de les funcions d'ordre superior és passar funcions com a paràmetres a altres funcions. Això és útil per a operacions com mapar, filtrar i reduir col·leccions.

Exemple: Utilitzant `map` amb un Paràmetre de Funció

val numbers = List(1, 2, 3, 4, 5)

// Defineix una funció per duplicar un número
def double(x: Int): Int = x * 2

// Utilitza la funció `map` per aplicar `double` a cada element de la llista
val doubledNumbers = numbers.map(double)

println(doubledNumbers) // Sortida: List(2, 4, 6, 8, 10)

En aquest exemple, la funció map pren una altra funció double com a paràmetre i l'aplica a cada element de la llista.

Funció com a Valor de Retorn

Les funcions d'ordre superior també poden retornar funcions. Això és útil per crear codi més flexible i reutilitzable.

Exemple: Retornant una Funció

def createMultiplier(factor: Int): Int => Int = {
  (x: Int) => x * factor
}

val triple = createMultiplier(3)
println(triple(4)) // Sortida: 12

Aquí, createMultiplier retorna una funció que multiplica la seva entrada pel factor donat.

Funcions Anònimes

Les funcions anònimes (també conegudes com a expressions lambda) són funcions definides sense un nom. Sovint s'utilitzen com a arguments per a funcions d'ordre superior.

Exemple: Utilitzant Funcions Anònimes

val numbers = List(1, 2, 3, 4, 5)

// Utilitza una funció anònima per duplicar cada element de la llista
val doubledNumbers = numbers.map(x => x * 2)

println(doubledNumbers) // Sortida: List(2, 4, 6, 8, 10)

En aquest exemple, la funció anònima x => x * 2 es passa directament a la funció map.

Currying

El currying és el procés de transformar una funció que pren múltiples arguments en una sèrie de funcions que cadascuna pren un sol argument. Això pot fer que les funcions siguin més flexibles i fàcils d'utilitzar en certs contextos.

Exemple: Currying

def add(x: Int)(y: Int): Int = x + y

val addFive = add(5) _
println(addFive(3)) // Sortida: 8

En aquest exemple, add és una funció curried. L'expressió add(5) _ crea una nova funció que afegeix 5 al seu argument.

Conclusió

Les funcions d'ordre superior són una característica poderosa de Scala que permeten un codi més abstracte i reutilitzable. Entenent com passar funcions com a paràmetres, retornar funcions, utilitzar funcions anònimes i aplicar currying, pots escriure programes Scala més flexibles i concisos. Practica aquests conceptes amb diversos exemples i exercicis per esdevenir proficient en l'ús de funcions d'ordre superior en els teus projectes Scala.
