Introducció als Actors i Akka

Els actors són un concepte fonamental en la programació concurrent i distribuïda. Akka és una potent caixa d'eines i entorn d'execució per construir aplicacions altament concurrents, distribuïdes i resilients impulsades per missatges a la JVM, utilitzant el model d'actors.

Conceptes Clau

  • Model d'Actors: Un model matemàtic de computació concurrent que tracta els "actors" com les primitives universals de la computació concurrent.
  • Akka: Una caixa d'eines i entorn d'execució per construir aplicacions concurrents, distribuïdes i tolerants a fallades en Scala i Java.
  • Concurrència: La capacitat de diferents parts o unitats d'un programa per executar-se fora d'ordre o en ordre parcial, sense afectar el resultat final.
  • Impulsat per Missatges: La comunicació entre actors es fa mitjançant el pas de missatges asíncrons.

Configurant Akka

Per començar a utilitzar Akka en el teu projecte Scala, necessites afegir les dependències d'Akka al teu fitxer build.sbt:

libraryDependencies += "com.typesafe.akka" %% "akka-actor-typed" % "2.6.16"

Creant el teu Primer Actor

Creem un actor senzill que rep un missatge i el mostra.

Exemple: Actor Hola Món

import akka.actor.typed.ActorSystem
import akka.actor.typed.scaladsl.Behaviors

object HelloWorld {
  def apply(): Behaviors.Receive[String] = Behaviors.receive { (context, message) =>
    context.log.info("Missatge rebut: {}", message)
    Behaviors.same
  }
}

object HelloWorldMain {
  def main(args: Array[String]): Unit = {
    val system: ActorSystem[String] = ActorSystem(HelloWorld(), "hello")
    system ! "Hola, Akka!"
  }
}

Explicació

  • Behaviors.receive: Defineix el comportament de l'actor quan rep un missatge.
  • context.log.info: Mostra el missatge rebut.
  • Behaviors.same: Indica que el comportament de l'actor no canvia després de processar el missatge.
  • ActorSystem: El punt d'entrada per Akka, crea i gestiona actors.

Comunicació entre Actors

Els actors es comuniquen enviant-se missatges entre ells. Els missatges són immutables i haurien de ser serialitzables.

Exemple: Comunicació entre Actors

import akka.actor.typed.ActorRef
import akka.actor.typed.ActorSystem
import akka.actor.typed.scaladsl.Behaviors

object Greeter {
  final case class Greet(whom: String, replyTo: ActorRef[Greeted])
  final case class Greeted(whom: String, from: ActorRef[Greet])

  def apply(): Behaviors.Receive[Greet] = Behaviors.receive { (context, message) =>
    context.log.info("Hola {}!", message.whom)
    message.replyTo ! Greeted(message.whom, context.self)
    Behaviors.same
  }
}

object GreeterBot {
  def apply(max: Int): Behaviors.Receive[Greeter.Greeted] = {
    def bot(greetingCounter: Int): Behaviors.Receive[Greeter.Greeted] =
      Behaviors.receive { (context, message) =>
        val n = greetingCounter + 1
        context.log.info("Salutació {} per {}", n, message.whom)
        if (n == max) {
          Behaviors.stopped
        } else {
          message.from ! Greeter.Greet(message.whom, context.self)
          bot(n)
        }
      }

    bot(0)
  }
}

object GreeterMain {
  def main(args: Array[String]): Unit = {
    val greeterMain: ActorSystem[Greeter.Greet] = ActorSystem(Greeter(), "greeter")
    val greeterBot: ActorRef[Greeter.Greeted] = greeterMain.systemActorOf(GreeterBot(3), "greeterBot")

    greeterMain ! Greeter.Greet("Akka", greeterBot)
  }
}

Explicació

  • Greet i Greeted: Classes de cas utilitzades com a missatges.
  • Greeter: Un actor que envia un missatge de salutació i espera una resposta.
  • GreeterBot: Un actor que compta el nombre de salutacions i s'atura després d'un cert nombre.
  • ActorRef: Una referència a un actor, utilitzada per enviar-li missatges.

Supervisió i Tolerància a Fallades

Els actors poden supervisar altres actors, gestionant les seves fallades i decidint una estratègia per gestionar-les.

Exemple: Estratègia de Supervisió

import akka.actor.typed.SupervisorStrategy
import akka.actor.typed.scaladsl.Behaviors

object Supervisor {
  def apply(): Behaviors.Receive[String] = Behaviors.supervise(Behaviors.receive[String] { (context, message) =>
    if (message == "fail") throw new RuntimeException("Fallada!")
    else {
      context.log.info("Missatge rebut: {}", message)
      Behaviors.same
    }
  }).onFailure[RuntimeException](SupervisorStrategy.restart)
}

Explicació

  • SupervisorStrategy.restart: Reinicia l'actor si es llança una RuntimeException.

Conclusió

Els actors i Akka proporcionen un marc robust per construir aplicacions concurrents i distribuïdes en Scala. Entenent els conceptes bàsics del model d'actors, configurant Akka i creant actors, pots construir sistemes escalables i resilients. Els temes avançats com la supervisió i la tolerància a fallades milloren encara més la fiabilitat de les teves aplicacions.
