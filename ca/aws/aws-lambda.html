Introducció a AWS Lambda

AWS Lambda és un servei de computació sense servidor que et permet executar codi sense aprovisionar ni gestionar servidors. Amb Lambda, pots executar codi en resposta a esdeveniments com canvis en les dades d'un bucket d'Amazon S3 o actualitzacions en una taula de DynamoDB.

Conceptes Clau

  • Computació sense servidor: No cal gestionar la infraestructura.
  • Basat en esdeveniments: Les funcions es desencadenen per esdeveniments.
  • Autoescalat: S'escala automàticament segons el nombre de sol·licituds entrants.
  • Pagament per ús: Els càrrecs es basen en el nombre de sol·licituds i la durada de l'execució del codi.

Configuració d'AWS Lambda

Prerequisits

  • Un compte d'AWS.
  • Comprensió bàsica dels serveis d'AWS com S3 i IAM.

Creació de la teva primera funció Lambda

  1. Navega a la consola d'AWS Lambda:

    • Ves a la consola de gestió d'AWS.
    • Cerca i selecciona "Lambda".

  2. Crea una funció:

    • Fes clic a "Create function".
    • Tria "Author from scratch".
    • Proporciona un nom per a la funció.
    • Tria un runtime (per exemple, Python 3.8, Node.js 14.x).
    • Configura el rol d'execució (crea un nou rol amb permisos bàsics de Lambda).

  3. Escriu el teu codi:

    • Utilitza l'editor de codi en línia o carrega un fitxer .zip.
    • Exemple de codi (Python):
    def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': 'Hello, World!'
    }

  4. Configura la funció:

    • Configura la memòria i els paràmetres de temps d'execució.
    • Afegeix variables d'entorn si cal.

  5. Prova la teva funció:

    • Crea un esdeveniment de prova.
    • Executa la prova i comprova el resultat.

Integració d'AWS Lambda amb altres serveis

Exemple de desencadenador S3

  1. Crea un bucket S3:

    • Ves a la consola de S3.
    • Crea un nou bucket.

  2. Afegeix un desencadenador S3 a Lambda:

    • Ves a la teva funció Lambda.
    • Selecciona "Add trigger".
    • Tria "S3" i configura el bucket i el tipus d'esdeveniment (per exemple, ObjectCreated).

  3. Modifica el codi de Lambda:

    • Exemple de codi per processar un esdeveniment S3:
    import json

def lambda_handler(event, context):
    # Registra l'esdeveniment
    print("Received event: " + json.dumps(event, indent=2))

    # Processa l'esdeveniment S3
    for record in event['Records']:
        s3 = record['s3']
        bucket = s3['bucket']['name']
        key = s3['object']['key']
        print(f"Bucket: {bucket}, Key: {key}")

    return {
        'statusCode': 200,
        'body': 'Event processed'
    }

Exemple de desencadenador DynamoDB

  1. Crea una taula DynamoDB:

    • Ves a la consola de DynamoDB.
    • Crea una nova taula.

  2. Afegeix un desencadenador DynamoDB a Lambda:

    • Ves a la teva funció Lambda.
    • Selecciona "Add trigger".
    • Tria "DynamoDB" i configura la taula i el tipus d'esdeveniment (per exemple, INSERT).

  3. Modifica el codi de Lambda:

    • Exemple de codi per processar un esdeveniment DynamoDB:
    import json

def lambda_handler(event, context):
    # Registra l'esdeveniment
    print("Received event: " + json.dumps(event, indent=2))

    # Processa l'esdeveniment DynamoDB
    for record in event['Records']:
        if record['eventName'] == 'INSERT':
            new_image = record['dynamodb']['NewImage']
            print(f"New item added: {new_image}")

    return {
        'statusCode': 200,
        'body': 'Event processed'
    }

Temes avançats en AWS Lambda

Gestió d'errors i reintents

  • Blocs Try-Catch: Utilitza blocs try-catch en el teu codi per gestionar excepcions.
  • Dead Letter Queues (DLQ): Configura DLQs per capturar esdeveniments fallits.
  • Reintents: Configura el comportament de reintents en la configuració de la funció Lambda.

Monitoratge i registre

  • CloudWatch Logs: Registra automàticament totes les invocacions de funcions Lambda.
  • Mètriques personalitzades: Utilitza CloudWatch per crear mètriques personalitzades per a les teves funcions Lambda.

Optimització del rendiment

  • Assignació de memòria: Ajusta els paràmetres de memòria per optimitzar el rendiment.
  • Inicis en fred: Comprèn i mitiga la latència dels inicis en fred.
  • Límits de concurrència: Gestiona els paràmetres de concurrència per controlar el nombre d'execucions simultànies.

Conclusió

AWS Lambda és una eina poderosa per construir aplicacions sense servidor. Comprenent els seus conceptes bàsics, configurant funcions bàsiques, integrant amb altres serveis d'AWS i explorant temes avançats, pots aprofitar Lambda per construir solucions escalables i rendibles. Recorda monitoritzar i optimitzar les teves funcions per assegurar-te que funcionen de manera eficient.
