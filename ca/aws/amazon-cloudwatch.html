Amazon CloudWatch és un servei de monitoratge i observabilitat dissenyat per a enginyers DevOps, desenvolupadors, enginyers de fiabilitat del lloc (SREs) i gestors de TI. Proporciona dades i informació accionable per monitorar aplicacions, respondre a canvis de rendiment a nivell de sistema, optimitzar la utilització de recursos i obtenir una visió unificada de la salut operativa.

Introducció a Amazon CloudWatch

  • Què és Amazon CloudWatch?

    • Un servei de monitoratge i observabilitat.
    • Recull dades de monitoratge i operacionals en forma de registres, mètriques i esdeveniments.
    • Proporciona una visió unificada dels recursos, aplicacions i serveis d'AWS.

  • Característiques clau:

    • Mètriques: Recull i fa el seguiment de mètriques dels recursos d'AWS i aplicacions personalitzades.
    • Registres: Recull, monitoritza i analitza fitxers de registre.
    • Alarmes: Configura alarmes per reaccionar automàticament als canvis en les teves mètriques.
    • Quadres de comandament: Crea quadres de comandament personalitzats per visualitzar mètriques i registres.
    • Esdeveniments: Respon als canvis en el teu entorn AWS.

Configuració d'Amazon CloudWatch

  • Prerequisits:

    • Compte d'AWS.
    • Comprensió bàsica dels serveis d'AWS.

  • Passos per habilitar CloudWatch:

    1. Navega a CloudWatch a la Consola de Gestió d'AWS:

      • Ves a la Consola de Gestió d'AWS.
      • Cerca i selecciona "CloudWatch."

    2. Crea un Quadre de Comandament de CloudWatch:

      • Fes clic a "Dashboards" al menú de l'esquerra.
      • Fes clic a "Create dashboard."
      • Introdueix un nom per al teu quadre de comandament i fes clic a "Create dashboard."

    3. Afegeix Widgets al teu Quadre de Comandament:

      • Tria el tipus de widget (per exemple, Línia, Àrea Apilada, Número).
      • Selecciona les mètriques que vols mostrar.
      • Configura els paràmetres del widget i fes clic a "Add to dashboard."

Monitoratge amb Mètriques de CloudWatch

  • Comprensió de les Mètriques:

    • Les mètriques són punts de dades recollits al llarg del temps.
    • Cada mètrica té un espai de noms, un nom i una o més dimensions.

  • Mètriques Comunes d'AWS:

    • EC2: CPUUtilization, DiskReadOps, NetworkIn.
    • RDS: CPUUtilization, DatabaseConnections, FreeStorageSpace.
    • S3: BucketSizeBytes, NumberOfObjects.

  • Creació de Mètriques Personalitzades:

    import boto3
import time

# Crear client de CloudWatch
cloudwatch = boto3.client('cloudwatch')

# Posa dades de mètrica personalitzada
response = cloudwatch.put_metric_data(
    Namespace='MyCustomNamespace',
    MetricData=[
        {
            'MetricName': 'MyCustomMetric',
            'Dimensions': [
                {
                    'Name': 'InstanceType',
                    'Value': 'm5.large'
                },
            ],
            'Value': 1.0,
            'Unit': 'Count',
            'Timestamp': time.time()
        },
    ]
)
print(response)

Treballant amb Registres de CloudWatch

  • Grups de Registres i Fluxos de Registres:

    • Grups de Registres: Un grup de fluxos de registres que comparteixen la mateixa retenció, monitoratge i configuració de control d'accés.
    • Fluxos de Registres: Una seqüència d'esdeveniments de registre de la mateixa font.

  • Creació d'un Grup de Registres:

    import boto3

# Crear client de CloudWatch Logs
logs = boto3.client('logs')

# Crear un grup de registres
response = logs.create_log_group(
    logGroupName='MyLogGroup'
)
print(response)

  • Enviar Registres a CloudWatch:

    import boto3

# Crear client de CloudWatch Logs
logs = boto3.client('logs')

# Crear un flux de registres
response = logs.create_log_stream(
    logGroupName='MyLogGroup',
    logStreamName='MyLogStream'
)

# Posa esdeveniments de registre
response = logs.put_log_events(
    logGroupName='MyLogGroup',
    logStreamName='MyLogStream',
    logEvents=[
        {
            'timestamp': int(time.time() * 1000),
            'message': 'This is a log message'
        },
    ]
)
print(response)

Configuració d'Alarmes de CloudWatch

  • Creació d'una Alarma:

    1. Navega a Alarms a la Consola de CloudWatch:

      • Fes clic a "Alarms" al menú de l'esquerra.
      • Fes clic a "Create alarm."

    2. Selecciona una Mètrica:

      • Tria una mètrica de la llista o cerca una mètrica específica.
      • Fes clic a "Select metric."

    3. Configura l'Alarma:

      • Configura les condicions per a l'alarma (per exemple, llindar, període).
      • Configura accions (per exemple, enviar notificació, executar una política d'Auto Scaling).
      • Fes clic a "Create alarm."

Quadres de Comandament de CloudWatch

  • Creació i Personalització de Quadres de Comandament:

    • Els quadres de comandament proporcionen una pàgina d'inici personalitzable a la consola de CloudWatch.
    • Pots afegir widgets per mostrar mètriques, registres i alarmes.

  • Exemple de Configuració d'un Quadre de Comandament:

    {
    "widgets": [
        {
            "type": "metric",
            "x": 0,
            "y": 0,
            "width": 6,
            "height": 6,
            "properties": {
                "metrics": [
                    [ "AWS/EC2", "CPUUtilization", "InstanceId", "i-1234567890abcdef0" ]
                ],
                "period": 300,
                "stat": "Average",
                "region": "us-east-1",
                "title": "EC2 Instance CPU"
            }
        }
    ]
}

Esdeveniments de CloudWatch

  • Comprensió dels Esdeveniments:

    • Els esdeveniments es generen quan es produeixen canvis en el teu entorn AWS.
    • Pots crear regles per coincidir amb esdeveniments i dirigir-los a destinacions com funcions Lambda, temes SNS o cues SQS.

  • Creació d'una Regla d'Esdeveniments:

    import boto3

# Crear client de CloudWatch Events
events = boto3.client('events')

# Crear una regla
response = events.put_rule(
    Name='MyRule',
    EventPattern='{"source": ["aws.ec2"]}',
    State='ENABLED'
)
print(response)

Conclusió

Amazon CloudWatch és una eina potent per monitorar i gestionar els teus recursos d'AWS. Comprenent les seves característiques clau i com configurar i utilitzar mètriques, registres, alarmes, quadres de comandament i esdeveniments, pots obtenir informació valuosa sobre les teves aplicacions i infraestructura. Aquest coneixement et permet optimitzar el rendiment, assegurar la fiabilitat i respondre ràpidament a qualsevol problema que sorgeixi.
