Introducció a AWS Config

AWS Config és un servei que et permet avaluar, auditar i avaluar les configuracions dels teus recursos AWS. T'ajuda a monitoritzar i registrar les configuracions dels recursos AWS i automatitzar l'avaluació de les configuracions registrades en comparació amb les configuracions desitjades.

Conceptes Clau

  • Element de Configuració (CI): Un registre de la configuració d'un recurs en un moment determinat.
  • Registrador de Configuració: Registra les configuracions dels recursos suportats en el teu compte.
  • Canal de Lliurament: Especifica on AWS Config lliura les instantànies de configuració i els arxius d'historial de configuració.
  • Instantània de Configuració: Una col·lecció d'elements de configuració per als recursos suportats en el teu compte.
  • Historial de Configuració: Una col·lecció d'elements de configuració per a un recurs durant un període de temps.
  • Regles: Les regles d'AWS Config representen les teves configuracions desitjades per als teus recursos AWS.

Configuració d'AWS Config

Per començar a utilitzar AWS Config, necessites configurar el registrador de configuració i el canal de lliurament.

Guia Pas a Pas

  1. Obre la Consola de Gestió d'AWS.
  2. Navega a AWS Config.
  3. Configura el Registrador de Configuració:
    • Tria els recursos que vols registrar.
    • Especifica la freqüència de registre (per exemple, cada 1 hora).
  4. Configura el Canal de Lliurament:
    • Tria un bucket S3 per emmagatzemar les instantànies de configuració i l'historial.
    • Opcionalment, configura un tema SNS per a notificacions.

Exemple

import boto3

# Crear un client de Config
config = boto3.client('config')

# Configurar el registrador de configuració
config.put_configuration_recorder(
    ConfigurationRecorder={
        'name': 'default',
        'roleARN': 'arn:aws:iam::123456789012:role/config-role',
        'recordingGroup': {
            'allSupported': True,
            'includeGlobalResourceTypes': True
        }
    }
)

# Configurar el canal de lliurament
config.put_delivery_channel(
    DeliveryChannel={
        'name': 'default',
        's3BucketName': 'my-config-bucket',
        'snsTopicARN': 'arn:aws:sns:us-east-1:123456789012:config-topic'
    }
)

# Iniciar el registrador de configuració
config.start_configuration_recorder(
    ConfigurationRecorderName='default'
)

Regles d'AWS Config

Les regles d'AWS Config et permeten definir directrius de compliment per als teus recursos. Pots utilitzar regles gestionades per AWS o crear regles personalitzades.

Regles Gestionades

AWS proporciona un conjunt de regles predefinides que pots utilitzar per avaluar si els teus recursos AWS compleixen amb les millors pràctiques comunes.

Regles Personalitzades

Pots crear regles personalitzades utilitzant funcions AWS Lambda per avaluar els teus recursos basant-te en criteris específics.

Exemple: Regla Gestionada

config.put_config_rule(
    ConfigRule={
        'ConfigRuleName': 's3-bucket-public-read-prohibited',
        'Description': 'Checks that your S3 buckets do not allow public read access.',
        'Scope': {
            'ComplianceResourceTypes': [
                'AWS::S3::Bucket',
            ]
        },
        'Source': {
            'Owner': 'AWS',
            'SourceIdentifier': 'S3_BUCKET_PUBLIC_READ_PROHIBITED'
        }
    }
)

Exemple: Regla Personalitzada

config.put_config_rule(
    ConfigRule={
        'ConfigRuleName': 'custom-rule',
        'Description': 'Custom rule to check specific compliance.',
        'Scope': {
            'ComplianceResourceTypes': [
                'AWS::EC2::Instance',
            ]
        },
        'Source': {
            'Owner': 'CUSTOM_LAMBDA',
            'SourceIdentifier': 'arn:aws:lambda:us-east-1:123456789012:function:custom-rule-function'
        }
    }
)

Monitorització i Notificacions

AWS Config proporciona diverses maneres de monitoritzar i rebre notificacions sobre l'estat de compliment dels teus recursos.

Monitorització

  • Consola de Gestió d'AWS: Visualitza l'estat de compliment dels teus recursos.
  • AWS CLI/SDK: Utilitza comandes i crides API per recuperar informació de compliment.

Notificacions

  • Notificacions SNS: Configura temes SNS per rebre notificacions sobre canvis de configuració i estat de compliment.

Exemple: Recuperar l'Estat de Compliment

response = config.get_compliance_details_by_config_rule(
    ConfigRuleName='s3-bucket-public-read-prohibited'
)

for result in response['EvaluationResults']:
    print(result['ComplianceType'], result['EvaluationResultIdentifier']['EvaluationResultQualifier']['ResourceId'])

Conclusió

AWS Config és una eina poderosa per gestionar i auditar les configuracions dels teus recursos AWS. Configurant registradors de configuració, canals de lliurament i regles, pots assegurar-te que els teus recursos compleixen amb les polítiques i millors pràctiques de la teva organització. La monitorització regular i les notificacions t'ajuden a mantenir-te informat sobre l'estat de compliment dels teus recursos, permetent-te prendre accions correctives quan sigui necessari.
