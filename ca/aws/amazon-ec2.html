Introducció a Amazon EC2

Amazon Elastic Compute Cloud (EC2) és un servei web que proporciona capacitat de càlcul redimensionable al núvol. Està dissenyat per facilitar la computació al núvol a escala web per als desenvolupadors.

Conceptes Clau

  • Instàncies: Servidors virtuals per executar aplicacions a la infraestructura d'AWS.
  • AMI (Amazon Machine Image): Una plantilla que conté la configuració del programari (sistema operatiu, servidor d'aplicacions i aplicacions) necessària per llançar una instància.
  • Tipus d'Instàncies: Diverses configuracions de CPU, memòria, emmagatzematge i capacitat de xarxa per a les teves instàncies.
  • Regions i Zones de Disponibilitat: Ubicacions físiques arreu del món on es troben els centres de dades d'AWS.
  • Grups de Seguretat: Tallafocs virtuals que controlen el trànsit per a una o més instàncies.
  • Adreces IP Elàstiques: Adreces IP estàtiques dissenyades per a la computació dinàmica al núvol.
  • Parells de Claus: Informació d'inici de sessió segura per a les teves instàncies.

Començant amb EC2

Llançament d'una Instància EC2

  1. Inicia sessió a la Consola de Gestió d'AWS.
  2. Navega al Tauler de Control d'EC2.
  3. Fes clic a "Llançar Instància".
  4. Tria una Amazon Machine Image (AMI):
    • Selecciona una AMI de la llista (per exemple, Amazon Linux 2 AMI).
  5. Tria un Tipus d'Instància:
    • Selecciona el tipus d'instància que s'adapti a les teves necessitats (per exemple, t2.micro per a la capa gratuïta).
  6. Configura els Detalls de la Instància:
    • Estableix el nombre d'instàncies, la configuració de la xarxa i altres configuracions.
  7. Afegeix Emmagatzematge:
    • Especifica la mida i el tipus d'emmagatzematge (per exemple, 8 GiB SSD d'Ús General).
  8. Afegeix Etiquetes:
    • Afegeix parells clau-valor per ajudar a gestionar les teves instàncies.
  9. Configura el Grup de Seguretat:
    • Crea un nou grup de seguretat o selecciona'n un d'existent per definir les regles del tallafocs.
  10. Revisa i Llança:
    • Revisa la teva configuració i fes clic a "Llançar".
  11. Selecciona un Parell de Claus:
    • Tria un parell de claus existent o crea'n un de nou per SSH a la teva instància.

Exemple: Llançament d'una Instància EC2 utilitzant AWS CLI

aws ec2 run-instances \
    --image-id ami-0abcdef1234567890 \
    --count 1 \
    --instance-type t2.micro \
    --key-name MyKeyPair \
    --security-group-ids sg-0123456789abcdef0 \
    --subnet-id subnet-6e7f829e

Explicació:

  • --image-id: La ID de l'AMI.
  • --count: Nombre d'instàncies a llançar.
  • --instance-type: El tipus d'instància.
  • --key-name: El nom del parell de claus.
  • --security-group-ids: La ID del grup de seguretat.
  • --subnet-id: La ID de la subxarxa.

Gestió d'Instàncies EC2

Monitorització i Mètriques

  • Amazon CloudWatch: Monitoritza les teves instàncies EC2 i estableix alarmes.
  • Verificacions d'Estat de la Instància: Realitzades automàticament per AWS per comprovar la salut de les teves instàncies.

Escalabilitat de les Instàncies EC2

  • Elastic Load Balancing (ELB): Distribueix automàticament el trànsit d'aplicacions entrant entre diverses instàncies.
  • Auto Scaling: Ajusta automàticament el nombre d'instàncies EC2 en resposta a la demanda canviant.

Exemple: Configuració d'un Grup d'Auto Scaling

aws autoscaling create-auto-scaling-group \
    --auto-scaling-group-name my-asg \
    --launch-configuration-name my-launch-config \
    --min-size 1 \
    --max-size 5 \
    --desired-capacity 2 \
    --vpc-zone-identifier subnet-6e7f829e

Explicació:

  • --auto-scaling-group-name: Nom del grup d'Auto Scaling.
  • --launch-configuration-name: Nom de la configuració de llançament.
  • --min-size: Nombre mínim d'instàncies.
  • --max-size: Nombre màxim d'instàncies.
  • --desired-capacity: Nombre desitjat d'instàncies.
  • --vpc-zone-identifier: ID de la subxarxa.

Funcions Avançades d'EC2

Instàncies Spot

  • Instàncies Spot: Compra capacitat EC2 no utilitzada a un preu reduït.
  • Flota Spot: Col·lecció d'Instàncies Spot i opcionalment Instàncies On-Demand.

Instàncies Reservades

  • Instàncies Reservades: Compromís d'utilitzar EC2 durant un període d'1 o 3 anys per rebre un descompte significatiu.

Exemple: Compra d'una Instància Reservada

aws ec2 purchase-reserved-instances-offering \
    --instance-count 1 \
    --reserved-instances-offering-id e-12345678

Explicació:

  • --instance-count: Nombre d'instàncies a reservar.
  • --reserved-instances-offering-id: ID de l'oferta d'instància reservada.

Conclusió

Amazon EC2 proporciona una manera escalable, flexible i rendible d'executar aplicacions al núvol. Comprenent els conceptes bàsics de llançament i gestió d'instàncies, així com funcions avançades com Auto Scaling i Instàncies Spot, pots optimitzar la teva infraestructura al núvol per satisfer les teves necessitats específiques. Tant si ets un principiant com un usuari avançat, dominar EC2 és una habilitat crucial per aprofitar tot el potencial d'AWS.
