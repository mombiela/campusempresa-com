Introducció a AWS Direct Connect

AWS Direct Connect és una solució de servei al núvol que facilita l'establiment d'una connexió de xarxa dedicada des de les teves instal·lacions fins a AWS. Utilitzant AWS Direct Connect, pots establir una connectivitat privada entre AWS i el teu centre de dades, oficina o entorn de col·locació, la qual cosa pot reduir els costos de xarxa, augmentar el rendiment de l'ample de banda i proporcionar una experiència de xarxa més consistent que les connexions basades en internet.

Conceptes Clau

  • Connexió de Xarxa Dedicada: Una connexió privada que evita l'internet públic.
  • Interfícies Virtuals (VIFs): Interfícies lògiques que pots crear per connectar-te als serveis d'AWS.
  • Direct Connect Gateway: Et permet connectar-te a múltiples VPCs en diferents regions.
  • Grups d'Agrupació d'Enllaços (LAGs): Combinen múltiples connexions per augmentar l'ample de banda i la redundància.
  • Redundància i Failover: Assegurar alta disponibilitat mitjançant múltiples connexions.

Configuració d'AWS Direct Connect

Pas 1: Sol·licitud d'una Connexió Direct Connect

  1. Inicia sessió a la Consola de Gestió d'AWS.
  2. Navega al servei Direct Connect.
  3. Fes clic a Create a Connection.
  4. Omple els detalls requerits com ara ubicació, ample de banda i nom de la connexió.
  5. Envia la sol·licitud i espera que AWS provisioni la connexió.

Pas 2: Configuració d'Interfícies Virtuals

  1. Un cop la connexió estigui provisionada, navega a la secció Virtual Interfaces.
  2. Fes clic a Create Virtual Interface.
  3. Tria el tipus de VIF (Pública, Privada o Transit).
  4. Omple els detalls necessaris com ara VLAN ID, BGP ASN i adreces IP.
  5. Crea el VIF i configura el teu router local per establir el peering BGP.
# Exemple de configuració BGP per a un router Cisco
router bgp 65000
  neighbor 192.168.1.1 remote-as 7224
  neighbor 192.168.1.1 description AWS Direct Connect
  neighbor 192.168.1.1 ebgp-multihop 3
  neighbor 192.168.1.1 update-source Loopback0
  !
  address-family ipv4
    neighbor 192.168.1.1 activate
    neighbor 192.168.1.1 next-hop-self
  exit-address-family

Pas 3: Utilització de Direct Connect Gateway

  1. Navega a la secció Direct Connect Gateway.
  2. Fes clic a Create Direct Connect Gateway.
  3. Proporciona un nom i ASN per al gateway.
  4. Associa el Direct Connect Gateway amb les teves VPCs en diferents regions.

Funcions Avançades

Grups d'Agrupació d'Enllaços (LAGs)

  • Propòsit: Combinar múltiples connexions Direct Connect en una sola connexió lògica per augmentar l'ample de banda i la redundància.
  • Configuració:
    1. Navega a la secció Link Aggregation Groups.
    2. Fes clic a Create LAG.
    3. Selecciona les connexions que vols agrupar.
    4. Configura els paràmetres del LAG i crea'l.

Redundància i Failover

  • Alta Disponibilitat: Assegura't de tenir múltiples connexions Direct Connect en diferents ubicacions.
  • Mecanismes de Failover: Utilitza BGP per configurar el failover automàtic entre connexions.
# Exemple de configuració BGP per a redundància
router bgp 65000
  neighbor 192.168.1.1 remote-as 7224
  neighbor 192.168.1.2 remote-as 7224
  !
  address-family ipv4
    neighbor 192.168.1.1 activate
    neighbor 192.168.1.2 activate
    neighbor 192.168.1.1 route-map PREFER_PRIMARY in
    neighbor 192.168.1.2 route-map PREFER_SECONDARY in
  exit-address-family

Resum

AWS Direct Connect proporciona una manera robusta i fiable de connectar la teva infraestructura local a AWS. Entenent els conceptes clau i seguint els passos de configuració, pots aprofitar Direct Connect per millorar el rendiment i la fiabilitat de la teva xarxa. Les funcions avançades com els Grups d'Agrupació d'Enllaços i les configuracions de redundància asseguren que la teva connexió sigui tant escalable com altament disponible.
