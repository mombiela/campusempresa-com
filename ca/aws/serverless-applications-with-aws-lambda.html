Introducció a la computació sense servidor

  • Definició de la computació sense servidor.
  • Beneficis de l'arquitectura sense servidor.
  • Visió general d'AWS Lambda i el seu paper en la computació sense servidor.

Començant amb AWS Lambda

  • Configuració d'un compte d'AWS.
  • Introducció a la Consola de Gestió d'AWS.
  • Creació de la teva primera funció Lambda. 
    import json

def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': json.dumps('Hello from Lambda!')
    }
    • Explicació:
      • lambda_handler: La funció principal que AWS Lambda crida per iniciar l'execució.
      • event: Conté les dades de l'esdeveniment.
      • context: Proporciona informació d'execució al controlador.

Conceptes d'AWS Lambda

  • Funcions: La unitat bàsica de desplegament a AWS Lambda.
  • Fonts d'esdeveniments: Serveis que poden desencadenar funcions Lambda (per exemple, S3, DynamoDB, API Gateway).
  • Rol d'execució: Rol IAM que atorga a la funció permisos per accedir als serveis d'AWS.
  • Variables d'entorn: Parelles clau-valor que es poden utilitzar per passar configuracions a la teva funció.

Creació i desplegament de funcions Lambda

  • Utilitzant la Consola de Gestió d'AWS.
  • Utilitzant l'AWS CLI. 
    aws lambda create-function --function-name my-function 
        --runtime python3.8 --role arn:aws:iam::account-id:role/execution_role 
        --handler lambda_function.lambda_handler --zip-file fileb://function.zip
    • Explicació:
      • --function-name: El nom de la funció Lambda.
      • --runtime: L'entorn d'execució per a la funció Lambda.
      • --role: L'ARN del rol IAM que Lambda assumeix quan executa la funció.
      • --handler: La funció dins del teu codi que Lambda crida per iniciar l'execució.
      • --zip-file: La ruta al paquet de desplegament.

Fonts d'esdeveniments i desencadenants

  • Amazon S3: Desencadenant funcions Lambda en la creació d'objectes. 
    import json

def lambda_handler(event, context):
    # Registra l'esdeveniment
    print(json.dumps(event))
    return {
        'statusCode': 200,
        'body': json.dumps('Event processed')
    }
  • Amazon DynamoDB: Desencadenant funcions Lambda en actualitzacions de taules.
  • Amazon API Gateway: Creant APIs RESTful amb Lambda.

Funcions avançades de Lambda

  • Concurrència i escalat: Comprendre com Lambda escala amb les sol·licituds entrants.
  • Gestió d'errors i reintents: Configurant reintents i gestionant errors.
  • Monitorització i registre: Utilitzant CloudWatch per a registres i mètriques. 
    import logging
logger = logging.getLogger()
logger.setLevel(logging.INFO)

def lambda_handler(event, context):
    logger.info('Event: %s', event)
    return {
        'statusCode': 200,
        'body': json.dumps('Logged event')
    }

Integració amb altres serveis d'AWS

  • AWS Step Functions: Orquestrant múltiples funcions Lambda.
  • Amazon SNS i SQS: Utilitzant Lambda amb serveis de missatgeria.
  • AWS RDS i DynamoDB: Accedint a bases de dades des de funcions Lambda.

Millors pràctiques de seguretat

  • Rols i polítiques IAM: Atorgant accés de menor privilegi.
  • Variables d'entorn: Emmagatzemant informació sensible de manera segura.
  • Integració VPC: Executant funcions Lambda dins d'una VPC.

Gestió de costos

  • Comprendre el model de preus d'AWS Lambda.
  • Millors pràctiques per a l'optimització de costos.
  • Utilitzant AWS Cost Explorer per monitoritzar els costos de Lambda.

Conclusió

  • Resum dels conceptes clau: computació sense servidor, funcions Lambda, fonts d'esdeveniments i integracions.
  • Encoratjament a explorar temes més avançats i serveis d'AWS.
