Introducció a la Infraestructura Global d'AWS

Amazon Web Services (AWS) proporciona una plataforma de computació en núvol robusta i escalable. Entendre la infraestructura global d'AWS és crucial per aprofitar tot el seu potencial. Aquesta secció cobrirà els components i conceptes clau de la infraestructura global d'AWS.

Components Clau de la Infraestructura Global d'AWS

Regions

  • Definició: Una Regió és una ubicació física arreu del món on AWS agrupa centres de dades.
  • Propòsit: Les regions permeten desplegar aplicacions i serveis més a prop dels teus usuaris finals per reduir la latència.
  • Exemple: us-east-1 (Virginia del Nord), eu-west-1 (Irlanda).

Zones de Disponibilitat (AZs)

  • Definició: Les Zones de Disponibilitat són ubicacions aïllades dins d'una Regió.
  • Propòsit: Les AZs proporcionen redundància i tolerància a fallades aïllant les fallades a una sola zona.
  • Exemple: us-east-1a, us-east-1b.

Ubicacions d'Edge

  • Definició: Les Ubicacions d'Edge són punts finals per als serveis d'AWS utilitzats per a la memòria cau de continguts.
  • Propòsit: Milloren la latència emmagatzemant continguts més a prop dels usuaris.
  • Exemple: Utilitzades per Amazon CloudFront per lliurar continguts.

Regions i Zones de Disponibilitat

Triar una Regió

  • Factors a Considerar:
    • Latència: Tria una regió a prop dels teus usuaris.
    • Compliment: Algunes dades han de residir en ubicacions geogràfiques específiques.
    • Cost: Els preus poden variar entre regions.

Alta Disponibilitat i Tolerància a Fallades

  • Desplegaments Multi-AZ: Distribueix aplicacions a través de múltiples AZs per assegurar alta disponibilitat.
  • Recuperació de Desastres: Utilitza múltiples regions per a estratègies de recuperació de desastres.

Exemple: Desplegar una Instància EC2 en una Regió i AZ Específica

aws ec2 run-instances --image-id ami-0abcdef1234567890 --count 1 --instance-type t2.micro --key-name MyKeyPair --region us-west-2 --placement AvailabilityZone=us-west-2b
  • Explicació: Aquesta comanda llança una instància EC2 a la regió us-west-2 i a l'AZ us-west-2b.

Ubicacions d'Edge i Lliurament de Continguts

Amazon CloudFront

  • Definició: Un servei de xarxa de lliurament de continguts (CDN) que utilitza ubicacions d'edge.
  • Propòsit: Distribueix continguts als usuaris amb baixa latència.
  • Exemple: Servir continguts web estàtics i dinàmics.

Exemple: Crear una Distribució CloudFront

{
  "CallerReference": "unique-string",
  "Aliases": {
    "Quantity": 1,
    "Items": ["example.com"]
  },
  "DefaultRootObject": "index.html",
  "Origins": {
    "Quantity": 1,
    "Items": [
      {
        "Id": "S3-example-bucket",
        "DomainName": "example-bucket.s3.amazonaws.com",
        "S3OriginConfig": {
          "OriginAccessIdentity": ""
        }
      }
    ]
  },
  "DefaultCacheBehavior": {
    "TargetOriginId": "S3-example-bucket",
    "ViewerProtocolPolicy": "redirect-to-https",
    "AllowedMethods": {
      "Quantity": 2,
      "Items": ["GET", "HEAD"]
    },
    "ForwardedValues": {
      "QueryString": false,
      "Cookies": {
        "Forward": "none"
      }
    },
    "MinTTL": 0
  },
  "Comment": "Example CloudFront Distribution",
  "Enabled": true
}
  • Explicació: Aquesta configuració JSON crea una distribució CloudFront que serveix continguts des d'un bucket S3.

Resum

Entendre la infraestructura global d'AWS és fonamental per dissenyar aplicacions escalables, fiables i eficients. Els components clau inclouen Regions, Zones de Disponibilitat i Ubicacions d'Edge, cadascun servint propòsits específics com reduir la latència, assegurar alta disponibilitat i lliurar continguts de manera eficient. Triant estratègicament les regions i aprofitant els desplegaments multi-AZ, pots optimitzar el rendiment i la fiabilitat. A més, utilitzar serveis com Amazon CloudFront pot millorar significativament el lliurament de continguts als usuaris finals.
