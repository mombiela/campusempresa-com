Introducció a AWS CloudFormation

AWS CloudFormation és un servei que t'ajuda a modelar i configurar els teus recursos d'Amazon Web Services perquè puguis passar menys temps gestionant aquests recursos i més temps centrant-te en les teves aplicacions. Crees una plantilla que descriu tots els recursos d'AWS que vols (com instàncies d'Amazon EC2 o instàncies de bases de dades d'Amazon RDS), i AWS CloudFormation s'encarrega de provisionar i configurar aquests recursos per a tu.

Conceptes Clau

  • Plantilles: Fitxers de text en format JSON o YAML que descriuen els recursos d'AWS que vols crear.
  • Stack: Una col·lecció de recursos d'AWS que pots gestionar com una unitat única. Crees, actualitzes i elimines una col·lecció de recursos creant, actualitzant i eliminant stacks.
  • Change Sets: Una vista prèvia dels canvis que AWS CloudFormation farà al teu stack quan l'actualitzis.
  • Recursos: Els components d'AWS que formen el teu stack, com instàncies EC2, cubs S3 i bases de dades RDS.
  • Paràmetres: Valors d'entrada que pots passar a la teva plantilla en temps d'execució per personalitzar el teu stack.
  • Sortides: Valors que es retornen sempre que visualitzes les propietats del teu stack.

Creant la Teva Primera Plantilla de CloudFormation

Per començar amb AWS CloudFormation, necessites crear una plantilla. A continuació es mostra un exemple senzill d'una plantilla de CloudFormation escrita en YAML que crea un cub S3.

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  MyS3Bucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      BucketName: my-first-cloudformation-bucket

Explicació

  • AWSTemplateFormatVersion: Especifica la versió de la plantilla d'AWS CloudFormation que estàs utilitzant.
  • Resources: Defineix els recursos d'AWS que vols crear.
  • MyS3Bucket: Nom lògic per al cub S3.
  • Type: Especifica el tipus de recurs d'AWS (en aquest cas, un cub S3).
  • Properties: Defineix les propietats per al recurs (en aquest cas, el nom del cub).

Desplegant un Stack

Un cop tens la teva plantilla, pots desplegar-la com un stack utilitzant la Consola de Gestió d'AWS, AWS CLI o AWS SDKs.

Utilitzant la Consola de Gestió d'AWS

  1. Obre la Consola de Gestió d'AWS.
  2. Navega al servei de CloudFormation.
  3. Fes clic a "Create stack".
  4. Puja el teu fitxer de plantilla.
  5. Segueix les indicacions per crear el stack.

Utilitzant AWS CLI

aws cloudformation create-stack --stack-name my-first-stack --template-body file://template.yaml

Utilitzant AWS SDK (Exemple en Python)

import boto3

client = boto3.client('cloudformation')

with open('template.yaml', 'r') as template_file:
    template_body = template_file.read()

response = client.create_stack(
    StackName='my-first-stack',
    TemplateBody=template_body
)

print(response)

Actualitzant un Stack

Per actualitzar un stack, modifiques la plantilla i després apliques els canvis.

Exemple: Afegint una Taula DynamoDB

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  MyS3Bucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      BucketName: my-first-cloudformation-bucket
  MyDynamoDBTable:
    Type: 'AWS::DynamoDB::Table'
    Properties:
      TableName: my-first-dynamodb-table
      AttributeDefinitions:
        - AttributeName: Id
          AttributeType: S
      KeySchema:
        - AttributeName: Id
          KeyType: HASH
      ProvisionedThroughput:
        ReadCapacityUnits: 5
        WriteCapacityUnits: 5

Aplicant l'Actualització

aws cloudformation update-stack --stack-name my-first-stack --template-body file://updated-template.yaml

Funcions Avançades

Stacks Niats

Els stacks niats et permeten crear stacks com a part d'altres stacks, permetent-te reutilitzar plantilles de CloudFormation.

StackSets

Els StackSets et permeten crear, actualitzar o eliminar stacks a través de múltiples comptes i regions d'AWS amb una sola operació.

Funcions Intrínseques

Les funcions intrínseques en les plantilles de CloudFormation et permeten assignar valors a propietats que no estan disponibles fins al temps d'execució.

Exemple: Utilitzant Ref i GetAtt

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  MyS3Bucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      BucketName: !Ref BucketNameParameter
  MyEC2Instance:
    Type: 'AWS::EC2::Instance'
    Properties:
      ImageId: ami-0ff8a91507f77f867
      InstanceType: t2.micro
      SecurityGroups:
        - !GetAtt MySecurityGroup.GroupId

Conclusió

AWS CloudFormation és una eina poderosa per automatitzar el desplegament i la gestió de recursos d'AWS. Utilitzant plantilles, pots definir la teva infraestructura com a codi, fent més fàcil replicar i gestionar. Tant si ets un principiant com un usuari avançat, entendre CloudFormation pot millorar significativament la teva capacitat per gestionar entorns d'AWS de manera eficient.
