Introducció

Construir una aplicació web escalable implica dissenyar i implementar un sistema que pugui gestionar càrregues creixents de manera eficient. AWS (Amazon Web Services) proporciona un conjunt d'eines i serveis que poden ajudar els desenvolupadors a crear aplicacions web escalables, fiables i d'alt rendiment.

Conceptes Clau

  • Escalabilitat: La capacitat d'un sistema per gestionar una càrrega augmentada afegint recursos.
  • Elasticitat: La capacitat d'ajustar automàticament els recursos per satisfer la demanda actual.
  • Alta Disponibilitat: Assegurar que l'aplicació estigui operativa i accessible en tot moment.
  • Tolerància a Fallades: La capacitat de continuar operant correctament en cas de fallada d'alguns dels seus components.

Serveis d'AWS per a l'Escalabilitat

  • Amazon EC2 (Elastic Compute Cloud): Proporciona capacitat de càlcul redimensionable.
  • Amazon S3 (Simple Storage Service): Emmagatzematge d'objectes escalable.
  • Amazon RDS (Relational Database Service): Servei de base de dades relacional gestionat.
  • Amazon DynamoDB: Servei de base de dades NoSQL totalment gestionat.
  • Amazon Elastic Load Balancing (ELB): Distribueix el trànsit d'aplicacions entrant entre múltiples objectius.
  • Amazon Auto Scaling: Ajusta automàticament el nombre d'instàncies EC2.
  • Amazon CloudFront: Xarxa de lliurament de contingut (CDN) per a una entrega ràpida de contingut.

Configuració d'una Aplicació Web Escalable

Pas 1: Llançament d'una Instància EC2

  1. Inicia sessió a la Consola de Gestió d'AWS.
  2. Navega al Tauler de Control d'EC2.
  3. Fes clic a "Llançar Instància".
  4. Tria una Imatge de Màquina d'Amazon (AMI).
  5. Selecciona un Tipus d'Instància (per exemple, t2.micro per a principiants).
  6. Configura els Detalls de la Instància:
    • Xarxa: VPC per defecte.
    • Subxarxa: Tria una subxarxa per defecte.
    • Assignació Automàtica d'IP Pública: Habilita.
  7. Afegeix Emmagatzematge: Els paràmetres per defecte solen ser suficients.
  8. Afegeix Etiquetes: Opcional, però útil per a l'organització.
  9. Configura el Grup de Seguretat:
    • Crea un nou grup de seguretat.
    • Afegeix regles per permetre l'accés HTTP (port 80) i SSH (port 22).
  10. Revisa i Llança.

Pas 2: Configuració d'un Balancejador de Càrrega

  1. Navega al Tauler de Control d'EC2.
  2. Fes clic a "Balancejadors de Càrrega" sota "Balanceig de Càrrega".
  3. Fes clic a "Crear Balancejador de Càrrega".
  4. Selecciona "Balancejador de Càrrega d'Aplicacions".
  5. Configura el Balancejador de Càrrega:
    • Nom: Introdueix un nom per al balancejador de càrrega.
    • Esquema: Orientat a Internet.
    • Oients: HTTP (port 80).
    • Zones de Disponibilitat: Selecciona almenys dues.
  6. Configura els Paràmetres de Seguretat: Omet per a HTTP.
  7. Configura els Grups de Seguretat: Selecciona el grup de seguretat creat anteriorment.
  8. Configura el Ruting:
    • Grup d'Objectius: Crea un nou grup d'objectius.
    • Tipus d'Objectiu: Instància.
    • Protocol: HTTP.
    • Port: 80.
  9. Registra Objectius: Selecciona les instàncies EC2 per incloure en el grup d'objectius.
  10. Revisa i Crea.

Pas 3: Configuració de l'Auto Scaling

  1. Navega al Tauler de Control d'EC2.
  2. Fes clic a "Grups d'Auto Scaling" sota "Auto Scaling".
  3. Fes clic a "Crear Grup d'Auto Scaling".
  4. Tria la Configuració de Llançament:
    • Utilitza la configuració de llançament existent o crea'n una de nova.
  5. Configura el Grup d'Auto Scaling:
    • Nom del Grup: Introdueix un nom.
    • Mida del Grup: Estableix el nombre desitjat d'instàncies.
    • Xarxa: Selecciona la VPC.
    • Subxarxa: Selecciona les subxarxes.
  6. Configura les Polítiques d'Escalabilitat:
    • Política d'Escalabilitat de Seguiment d'Objectius: Manté la utilització mitjana de la CPU al 50%.
  7. Afegeix Notificacions: Opcional, però útil per al monitoratge.
  8. Afegeix Etiquetes: Opcional, però útil per a l'organització.
  9. Revisa i Crea.

Exemple de Codi: Desplegament d'una Aplicació Web Simple

# Exemple d'aplicació Flask
from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hello, AWS!"

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=80)
  • Explicació:
    • Aquesta és una aplicació Flask simple que retorna "Hello, AWS!" quan s'accedeix.
    • L'aplicació escolta en totes les adreces IP (0.0.0.0) i el port 80.

Conclusió

Construir una aplicació web escalable a AWS implica entendre i utilitzar diversos serveis d'AWS com EC2, S3, RDS, DynamoDB, ELB, Auto Scaling i CloudFront. Seguint els passos descrits anteriorment, pots configurar una aplicació web bàsica escalable que pugui gestionar càrregues augmentades de manera eficient. A mesura que avancis, podràs explorar arquitectures més complexes i serveis d'AWS per millorar encara més l'escalabilitat, la fiabilitat i el rendiment de les teves aplicacions web.
