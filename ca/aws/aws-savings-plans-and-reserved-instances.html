Introducció

AWS Savings Plans i Instàncies Reservades (RIs) són opcions d'estalvi de costos proporcionades per Amazon Web Services (AWS) per ajudar els usuaris a reduir les seves despeses de computació en el núvol. Aquesta secció cobrirà els conceptes bàsics de totes dues opcions, les seves diferències i com triar la més adequada per a les teves necessitats.

Conceptes Clau

AWS Savings Plans

  • Definició: Els Savings Plans són models de preus flexibles que ofereixen estalvis significatius en l'ús d'AWS, a canvi d'un compromís amb una quantitat consistent d'ús (mesurada en $/hora) per un termini d'1 o 3 anys.
  • Tipus de Savings Plans:
    • Compute Savings Plans: Proporcionen la major flexibilitat i s'apliquen a qualsevol ús d'instàncies EC2 independentment de la regió, la família d'instàncies, el sistema operatiu o la tenència.
    • EC2 Instance Savings Plans: Ofereixen preus més baixos però són menys flexibles, aplicant-se només a famílies d'instàncies específiques dins d'una regió.

Instàncies Reservades (RIs)

  • Definició: Les Instàncies Reservades proporcionen un descompte en comparació amb els preus On-Demand a canvi de comprometre's a utilitzar un tipus d'instància específic en una regió específica per un termini d'1 o 3 anys.
  • Tipus de RIs:
    • RIs Estàndard: Ofereixen el descompte més alt però són menys flexibles. Es poden modificar per canviar la Zona de Disponibilitat, la mida de l'instància (dins de la mateixa família) i el tipus de xarxa.
    • RIs Convertibles: Ofereixen un descompte més baix però permeten canviar el tipus d'instància, el sistema operatiu i la tenència.

Comparació de Savings Plans i Instàncies Reservades

| Característica | Savings Plans | Instàncies Reservades | |----------------------------|----------------------------------------|--------------------------------------| | Flexibilitat | Alta (especialment Compute Savings Plans) | Mitjana (RIs Convertibles) a Baixa (RIs Estàndard) | | Nivell de Descompte | Moderat a Alt | Alt (RIs Estàndard) a Moderat (RIs Convertibles) | | Termini de Compromís | 1 o 3 anys | 1 o 3 anys | | Abast | A nivell de compte | Específic per tipus d'instància i regió | | Aplicabilitat | A través d'EC2, Fargate, Lambda | Només instàncies EC2 |

Com Tria Entre Savings Plans i Instàncies Reservades

Factors a Considerar

  • Previsibilitat de la Càrrega de Treball: Si la teva càrrega de treball és previsible i estable, les RIs poden ser més beneficioses a causa dels seus descomptes més alts.
  • Necessitats de Flexibilitat: Si necessites més flexibilitat en termes de tipus d'instàncies i regions, els Savings Plans, especialment els Compute Savings Plans, són una millor opció.
  • Pressupost i Gestió de Costos: Avalua el teu pressupost i quant estàs disposat a comprometre't per endavant. Ambdues opcions ofereixen estalvis significatius, però el nivell de compromís i el pagament inicial poden variar.

Escenaris d'Exemple

  1. Escenari 1: Una startup amb una càrrega de treball creixent i canviant podria preferir els Compute Savings Plans per la seva flexibilitat.
  2. Escenari 2: Una gran empresa amb càrregues de treball estables i previsibles podria beneficiar-se més de les RIs Estàndard a causa dels descomptes més alts.

Exemples Pràctics

Exemple 1: Comprar un Compute Savings Plan

import boto3

client = boto3.client('savingsplans')

response = client.create_savings_plan(
    savingsPlanOfferingId='string',  # ID de l'oferta del Savings Plan
    commitment='0.1',  # Quantitat de compromís en $/hora
    upfrontPaymentAmount='0',  # Quantitat de pagament inicial
    purchaseTime='2023-10-01T00:00:00Z',  # Hora de compra
    tags={
        'Project': 'MyProject'
    }
)

print(response)

Explicació:

  • Aquest codi utilitza el SDK d'AWS per a Python (Boto3) per comprar un Compute Savings Plan.
  • El paràmetre commitment especifica el compromís horari en dòlars.
  • El paràmetre upfrontPaymentAmount especifica qualsevol pagament inicial.

Exemple 2: Modificar una Instància Reservada

import boto3

client = boto3.client('ec2')

response = client.modify_reserved_instances(
    ReservedInstancesIds=[
        'ri-12345678',  # ID de la Instància Reservada
    ],
    TargetConfigurations=[
        {
            'AvailabilityZone': 'us-west-2a',
            'InstanceType': 'm5.large',
            'Platform': 'Linux/UNIX',
        },
    ]
)

print(response)

Explicació:

  • Aquest codi modifica una Instància Reservada existent per canviar la seva Zona de Disponibilitat i el tipus d'instància.
  • El paràmetre ReservedInstancesIds especifica l'ID de la RI a modificar.
  • El paràmetre TargetConfigurations especifica la nova configuració per a la RI.

Conclusió

AWS Savings Plans i Instàncies Reservades són eines poderoses per gestionar i reduir els teus costos d'AWS. Comprenent les seves diferències i avaluant les teves necessitats específiques, pots triar l'opció adequada per maximitzar els teus estalvis. Els Savings Plans ofereixen més flexibilitat, mentre que les Instàncies Reservades proporcionen descomptes més alts per a càrregues de treball predictibles. Utilitza els exemples proporcionats per començar a comprar i gestionar aquestes opcions d'estalvi de costos en el teu entorn AWS.
