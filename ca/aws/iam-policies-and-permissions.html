Introducció a les Polítiques IAM

Les polítiques d'Identity and Access Management (IAM) d'AWS són documents JSON que defineixen permisos per a accions sobre recursos d'AWS. Entendre les polítiques IAM és crucial per gestionar el control d'accés a AWS.

  • Polítiques IAM: Documents JSON que defineixen permisos.
  • Permisos: Permeten o deneguen accions sobre recursos d'AWS.
  • Principals: Usuaris, grups o rols als quals s'assignen les polítiques.

Estructura Bàsica d'una Política IAM

Una política IAM consta de diversos elements:

  • Versió: Especifica la versió del llenguatge de la política.
  • Declaració: Conté els elements principals de la política.
    • Efecte: Pot ser "Allow" o "Deny".
    • Acció: Llista d'accions que es permeten o es deneguen.
    • Recurs: Especifica els recursos d'AWS als quals s'apliquen les accions.
    • Condició: (Opcional) Especifica les condicions sota les quals la política està en efecte.

Exemple d'una Política IAM Simple

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:ListBucket",
      "Resource": "arn:aws:s3:::example-bucket"
    }
  ]
}
  • Explicació:
    • Aquesta política permet l'acció s3:ListBucket sobre el example-bucket.

Tipus de Polítiques IAM

Hi ha diversos tipus de polítiques IAM:

  • Polítiques Gestionades: Polítiques predefinides creades i gestionades per AWS o la teva organització.
  • Polítiques Inline: Polítiques que crees i incrustes directament en un sol usuari, grup o rol.
  • Polítiques Gestionades pel Client: Polítiques que crees i gestiones al teu compte d'AWS.

Polítiques Gestionades vs. Polítiques Inline

| Característica | Polítiques Gestionades | Polítiques Inline | |----------------------|----------------------------------|--------------------------------| | Creació | Creades per AWS o per tu | Creades per tu | | Gestió | Més fàcils de gestionar i reutilitzar | Incrustades directament en un usuari, grup o rol | | Reutilització | Es poden assignar a múltiples entitats | Específiques per a una sola entitat |

Creació i Assignació de Polítiques IAM

Creació d'una Política Gestionada

  1. Navega a la consola IAM.
  2. Selecciona "Policies" a la barra lateral.
  3. Fes clic a "Create policy".
  4. Utilitza l'editor visual o l'editor JSON per definir la política.
  5. Revisa i crea la política.

Assignació d'una Política a un Usuari, Grup o Rol

  1. Navega a la consola IAM.
  2. Selecciona "Users", "Groups" o "Roles" a la barra lateral.
  3. Tria l'usuari, grup o rol específic.
  4. Fes clic a la pestanya "Permissions".
  5. Fes clic a "Add permissions" i assigna la política desitjada.

Conceptes Avançats de Polítiques IAM

Condicions de la Política

Les condicions permeten especificar quan una política està en efecte. Utilitzen parells clau-valor per definir les condicions.

Exemple amb Condicions

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::example-bucket/*",
      "Condition": {
        "IpAddress": {
          "aws:SourceIp": "192.0.2.0/24"
        }
      }
    }
  ]
}
  • Explicació:
    • Aquesta política permet l'acció s3:GetObject sobre objectes en example-bucket només si la sol·licitud prové del rang d'IP 192.0.2.0/24.

Lògica d'Avaluació de Polítiques

IAM avalua les polítiques basant-se en la següent lògica:

  1. Denegació Explícita: Si alguna política denega l'acció, la sol·licitud és denegada.
  2. Permís Explícit: Si no hi ha cap denegació explícita i almenys una política permet l'acció, la sol·licitud és permesa.
  3. Denegació per Defecte: Si cap política permet l'acció, la sol·licitud és denegada per defecte.

Millors Pràctiques per a Polítiques IAM

  • Principi del Menor Privilegi: Concedeix només els permisos necessaris per a la tasca.
  • Utilitza Polítiques Gestionades: Prefereix les polítiques gestionades per AWS per a casos d'ús comuns.
  • Auditories Regulars: Revisa i audita regularment les polítiques i permisos IAM.
  • Utilitza Condicions: Aplica condicions per afegir una capa extra de seguretat.

Conclusió

Les polítiques i permisos IAM són fonamentals per assegurar el teu entorn AWS. Entenent l'estructura, els tipus i les millors pràctiques per a les polítiques IAM, pots gestionar efectivament l'accés als teus recursos AWS. Segueix sempre el principi del menor privilegi i revisa regularment les teves polítiques per assegurar-te que compleixen amb els teus requisits de seguretat.
