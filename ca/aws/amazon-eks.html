Introducció a Amazon EKS

Amazon Elastic Kubernetes Service (EKS) és un servei Kubernetes gestionat que facilita l'execució de Kubernetes a AWS sense necessitat d'instal·lar i operar el teu propi pla de control de Kubernetes o nodes. EKS és conforme amb Kubernetes certificat, de manera que pots utilitzar les eines i connectors existents de la comunitat de Kubernetes.

Conceptes Clau

  • Kubernetes: Un sistema de codi obert per automatitzar el desplegament, l'escalat i la gestió d'aplicacions contenidoritzades.
  • Pla de Control d'EKS: Gestionat per AWS, inclou el servidor API de Kubernetes i la base de dades etcd.
  • Nodes de Treball: Instàncies EC2 que executen les teves aplicacions contenidoritzades.
  • Clúster EKS: Una col·lecció d'instàncies EC2 gestionades per EKS per executar càrregues de treball de Kubernetes.

Configuració d'un Clúster EKS

Configurar un clúster EKS implica diversos passos, incloent la creació del clúster EKS en si, la configuració dels nodes de treball i la configuració dels rols i polítiques IAM necessaris.

Guia Pas a Pas

  1. Crear un Clúster EKS:

    aws eks create-cluster --name my-cluster --role-arn arn:aws:iam::123456789012:role/EKS-Role --resources-vpc-config subnetIds=subnet-6782e71e,subnet-e7e761ac,securityGroupIds=sg-6979fe18

  2. Crear un Grup de Nodes:

    aws eks create-nodegroup --cluster-name my-cluster --nodegroup-name my-nodegroup --subnets subnet-6782e71e subnet-e7e761ac --node-role arn:aws:iam::123456789012:role/EKS-Node-Role --scaling-config minSize=1,maxSize=3,desiredSize=2

  3. Actualitzar kubeconfig:

    aws eks update-kubeconfig --name my-cluster

  4. Desplegar una Aplicació de Mostra:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

    Aplicar el desplegament:

    kubectl apply -f nginx-deployment.yaml

Gestió de Clústers EKS

Gestionar un clúster EKS implica escalar, actualitzar i monitoritzar el clúster i les seves càrregues de treball.

Escalar el Clúster

  • Autoscalador Horitzontal de Pods: Escala automàticament el nombre de pods en un desplegament basat en la utilització observada de la CPU.

    apiVersion: autoscaling/v1
kind: HorizontalPodAutoscaler
metadata:
  name: nginx-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: nginx-deployment
  minReplicas: 1
  maxReplicas: 10
  targetCPUUtilizationPercentage: 50

    Aplicar l'autoscalador:

    kubectl apply -f nginx-hpa.yaml

Actualitzar el Clúster

  • Actualitzacions Progressives: Kubernetes suporta actualitzacions progressives per actualitzar la versió de l'aplicació sense temps d'inactivitat. 
    kubectl set image deployment/nginx-deployment nginx=nginx:1.16.1

Monitoritzar el Clúster

  • CloudWatch: Utilitza Amazon CloudWatch per monitoritzar el teu clúster EKS.
  • Prometheus i Grafana: Eines de codi obert populars per monitoritzar clústers de Kubernetes.

Funcions Avançades d'EKS

Integració amb Fargate

EKS suporta AWS Fargate, que et permet executar pods de Kubernetes sense gestionar les instàncies EC2 subjacents.

Xarxa de Serveis amb AWS App Mesh

AWS App Mesh proporciona xarxes a nivell d'aplicació per facilitar la comunicació entre els teus serveis a través de múltiples tipus d'infraestructura de computació.

Millors Pràctiques de Seguretat

  • Rols IAM per a Comptes de Servei (IRSA): Permisos de gran precisió per a comptes de servei de Kubernetes.
  • Polítiques de Xarxa: Controla la comunicació entre pods.

Conclusió

Amazon EKS simplifica l'execució de Kubernetes a AWS gestionant el pla de control i integrant-se amb altres serveis d'AWS. Comprenent com configurar, gestionar i utilitzar funcions avançades d'EKS, pots desplegar i escalar efectivament aplicacions contenidoritzades al núvol. Tant si ets un usuari principiant com avançat, EKS proporciona les eines i la flexibilitat necessàries per executar càrregues de treball de Kubernetes de grau de producció.
