Introducció

El Model de Responsabilitat Compartida d'AWS és un concepte fonamental que defineix les responsabilitats de seguretat i compliment d'AWS i dels seus clients. Entendre aquest model és crucial per a qualsevol persona que utilitzi serveis d'AWS, ja que delimita què és responsabilitat d'AWS i què ha de gestionar el client.

Conceptes Clau

  • Responsabilitat d'AWS: AWS és responsable de la seguretat del núvol.
  • Responsabilitat del Client: Els clients són responsables de la seguretat dins del núvol.

Responsabilitat d'AWS

AWS gestiona i controla els components des del sistema operatiu host i la capa de virtualització fins a la seguretat física de les instal·lacions on operen els serveis. Això inclou:

  • Seguretat Física: Protegir la infraestructura que executa tots els serveis oferts al Núvol d'AWS.
  • Seguretat de Xarxa: Assegurar la seguretat de la infraestructura de xarxa.
  • Seguretat del Maquinari: Assegurar els servidors físics i els dispositius d'emmagatzematge.
  • Seguretat del Programari: Gestionar la seguretat del programari que executa els serveis d'AWS.

Responsabilitat del Client

Els clients són responsables de gestionar la seguretat de les seves dades i aplicacions dins de l'entorn d'AWS. Això inclou:

  • Xifrat de Dades: Xifrar dades en repòs i en trànsit.
  • Gestió d'Identitat i Accés (IAM): Gestionar l'accés d'usuaris i permisos.
  • Seguretat d'Aplicacions: Assegurar la seguretat de les aplicacions que s'executen a AWS.
  • Actualització del Sistema Operatiu: Mantenir els sistemes operatius actualitzats.
  • Configuració de Xarxa: Configurar els paràmetres de seguretat de xarxa, com ara grups de seguretat i ACLs de xarxa.

Responsabilitats Compartides

Algunes responsabilitats són compartides entre AWS i el client. Aquestes inclouen:

  • Gestió de Configuració: AWS proporciona eines i serveis, però els clients han de configurar-los correctament.
  • Gestió de Parches: AWS gestiona la infraestructura, però els clients han de parchejar els seus sistemes operatius i aplicacions.
  • Conscienciació i Formació: AWS proporciona programes de formació i conscienciació en seguretat, però els clients han d'assegurar-se que els seus empleats estiguin formats.

Exemples i Millors Pràctiques

Exemple 1: Xifrat de Dades

AWS proporciona serveis com AWS Key Management Service (KMS) per ajudar els clients a xifrar les seves dades. No obstant això, els clients han de:

  • Definir i gestionar les seves claus de xifrat.
  • Assegurar-se que les dades estiguin xifrades abans de ser pujades a serveis d'AWS com S3.
import boto3

# Crear un client de KMS
kms_client = boto3.client('kms')

# Xifrar dades
response = kms_client.encrypt(
    KeyId='alias/my-key',
    Plaintext=b'Hola, Món!'
)

ciphertext = response['CiphertextBlob']

# Desxifrar dades
response = kms_client.decrypt(
    CiphertextBlob=ciphertext
)

plaintext = response['Plaintext']
print(plaintext.decode('utf-8'))

Exemple 2: Gestió d'Identitat i Accés (IAM)

AWS proporciona IAM per gestionar l'accés als serveis i recursos d'AWS de manera segura. Els clients han de:

  • Crear polítiques d'IAM que concedeixin el mínim privilegi necessari.
  • Revisar i actualitzar regularment les polítiques i rols d'IAM.
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "s3:ListBucket",
            "Resource": "arn:aws:s3:::example_bucket"
        },
        {
            "Effect": "Allow",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::example_bucket/*"
        }
    ]
}

Resum

El Model de Responsabilitat Compartida d'AWS és un marc crític per entendre la divisió de les tasques de seguretat i compliment entre AWS i els seus clients. AWS assegura la infraestructura, mentre que els clients asseguren les seves dades i aplicacions. Seguint les millors pràctiques i aprofitant les eines d'AWS, els clients poden gestionar efectivament les seves responsabilitats i mantenir un entorn de núvol segur.
