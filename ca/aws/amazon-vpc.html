Introducció a Amazon VPC

Amazon Virtual Private Cloud (VPC) et permet llançar recursos d'AWS en una xarxa virtual lògicament aïllada que tu defineixes. Aquesta secció cobrirà els conceptes bàsics de VPC, els seus components i com s'integra amb altres serveis d'AWS.

Conceptes Clau

  • VPC: Una xarxa virtual dedicada al teu compte d'AWS.
  • Subxarxes: Segments del rang d'adreces IP d'una VPC on pots col·locar grups de recursos aïllats.
  • Taules de rutes: Regles que determinen on es dirigeix el tràfic de xarxa.
  • Passarel·la d'Internet: Una passarel·la que permet la comunicació entre instàncies a la teva VPC i Internet.
  • Passarel·la NAT: Una passarel·la que permet que les instàncies en una subxarxa privada es connectin a Internet o altres serveis d'AWS, però impedeix que Internet iniciï una connexió amb aquestes instàncies.
  • Grups de seguretat: Tallafocs virtuals que controlen el tràfic entrant i sortint per a les teves instàncies.
  • ACLs de xarxa: Capa opcional de seguretat per a la teva VPC que actua com un tallafocs per controlar el tràfic entrant i sortint d'una o més subxarxes.

Creació de la teva primera VPC

En aquesta secció, recorrerem els passos per crear una VPC utilitzant la Consola de Gestió d'AWS.

Passos per crear una VPC

  1. Obre el Tauler de VPC: Navega al Tauler de VPC a la Consola de Gestió d'AWS.

  2. Crea una VPC:

    • Fes clic a "Create VPC".
    • Introdueix un nom per a la teva VPC.
    • Especifica un bloc CIDR IPv4 (per exemple, 10.0.0.0/16).
    • Tria un bloc CIDR IPv6 si cal.
    • Selecciona la tenència (per defecte o dedicada).
    • Fes clic a "Create VPC".

  3. Crea subxarxes:

    • Ves a la secció de Subxarxes.
    • Fes clic a "Create Subnet".
    • Selecciona la teva VPC.
    • Introdueix un nom per a la teva subxarxa.
    • Especifica la Zona de Disponibilitat.
    • Introdueix un bloc CIDR IPv4 (per exemple, 10.0.1.0/24).
    • Fes clic a "Create".

  4. Crea una Passarel·la d'Internet:

    • Ves a la secció de Passarel·les d'Internet.
    • Fes clic a "Create Internet Gateway".
    • Introdueix un nom per a la passarel·la.
    • Fes clic a "Create".
    • Adjunta la Passarel·la d'Internet a la teva VPC.

  5. Actualitza les Taules de Rutes:

    • Ves a la secció de Taules de Rutes.
    • Selecciona la taula de rutes associada amb la teva VPC.
    • Fes clic a la pestanya "Routes".
    • Fes clic a "Edit routes".
    • Afegeix una ruta amb destinació 0.0.0.0/0 i objectiu com la teva Passarel·la d'Internet.
    • Fes clic a "Save routes".

  6. Configura Grups de Seguretat:

    • Ves a la secció de Grups de Seguretat.
    • Fes clic a "Create Security Group".
    • Introdueix un nom i descripció.
    • Selecciona la teva VPC.
    • Afegeix regles entrants i sortints segons calgui.
    • Fes clic a "Create".

Exemple: Creació d'una VPC utilitzant AWS CLI

# Crea una VPC
aws ec2 create-vpc --cidr-block 10.0.0.0/16

# Crea una subxarxa
aws ec2 create-subnet --vpc-id vpc-12345678 --cidr-block 10.0.1.0/24

# Crea una Passarel·la d'Internet
aws ec2 create-internet-gateway

# Adjunta la Passarel·la d'Internet a la VPC
aws ec2 attach-internet-gateway --vpc-id vpc-12345678 --internet-gateway-id igw-12345678

# Crea una taula de rutes
aws ec2 create-route-table --vpc-id vpc-12345678

# Crea una ruta a la Passarel·la d'Internet
aws ec2 create-route --route-table-id rtb-12345678 --destination-cidr-block 0.0.0.0/0 --gateway-id igw-12345678

# Associa la taula de rutes amb la subxarxa
aws ec2 associate-route-table --subnet-id subnet-12345678 --route-table-id rtb-12345678

# Crea un grup de seguretat
aws ec2 create-security-group --group-name my-security-group --description "My security group" --vpc-id vpc-12345678

# Afegeix una regla per permetre l'accés SSH
aws ec2 authorize-security-group-ingress --group-id sg-12345678 --protocol tcp --port 22 --cidr 0.0.0.0/0

Configuracions Avançades de VPC

Aquesta secció cobrirà temes més avançats com el Peering de VPC, Connexions VPN i Punts Finals de VPC.

Peering de VPC

  • Definició: Una connexió de xarxa entre dues VPCs que et permet redirigir el tràfic entre elles utilitzant adreces IP privades.
  • Casos d'ús: Compartir recursos entre VPCs, accés entre comptes.

Connexions VPN

  • Definició: Una connexió segura entre la teva xarxa local i la teva VPC d'AWS.
  • Casos d'ús: Arquitectures de núvol híbrid, transferència segura de dades.

Punts Finals de VPC

  • Definició: Et permet connectar privadament la teva VPC amb serveis d'AWS suportats i serveis de punt final de VPC impulsats per AWS PrivateLink.
  • Tipus: Punts Finals d'Interfície, Punts Finals de Passarel·la.

Millors Pràctiques per a VPC

  • Disseny de Subxarxes: Utilitza múltiples Zones de Disponibilitat per a alta disponibilitat.
  • Seguretat: Implementa accés de mínim privilegi utilitzant Grups de Seguretat i ACLs de Xarxa.
  • Monitorització: Utilitza VPC Flow Logs per monitoritzar el tràfic.
  • Gestió de Costos: Optimitza l'ús de la Passarel·la NAT per reduir costos.

Conclusió

Amazon VPC és un servei potent que et permet crear una xarxa segura i escalable per als teus recursos d'AWS. Entenent els components bàsics i les configuracions, pots dissenyar i implementar una arquitectura de xarxa robusta adaptada a les teves necessitats. Les configuracions avançades com el Peering de VPC, Connexions VPN i Punts Finals de VPC amplien encara més les capacitats de la teva VPC, permetent escenaris de xarxa complexos i segurs. Sempre segueix les millors pràctiques per assegurar la seguretat, disponibilitat i eficiència de costos.
