Introducció a AWS KMS

AWS Key Management Service (KMS) és un servei gestionat que facilita la creació i el control de les claus de xifratge utilitzades per xifrar les teves dades. KMS està integrat amb altres serveis d'AWS, cosa que fa senzill xifrar les dades que emmagatzemes en aquests serveis i controlar l'accés a les claus que les desxifren.

Conceptes Clau

  • Customer Master Keys (CMKs): Els recursos principals a AWS KMS. Les CMKs es poden utilitzar per xifrar i desxifrar fins a 4 KB de dades directament o per generar claus de dades que poden xifrar quantitats més grans de dades.
  • Claus de Dades: Claus generades per les CMKs que s'utilitzen per xifrar dades. Les claus de dades es poden utilitzar fora d'AWS KMS.
  • Polítiques de Claus: Polítiques basades en JSON que defineixen qui pot utilitzar i gestionar les CMKs.
  • Concessions: Permeten permisos temporals per utilitzar les CMKs.
  • Context de Xifratge: Dades opcionals i no secretes que es poden associar amb dades xifrades per proporcionar seguretat addicional.

Creació i Gestió de CMKs

Creació d'una CMK

Per crear una CMK, pots utilitzar la Consola de Gestió d'AWS, AWS CLI o els SDKs d'AWS.

Utilitzant la Consola de Gestió d'AWS

  1. Obre la consola d'AWS KMS.
  2. Tria Crear clau.
  3. Selecciona el tipus de clau (simètrica o asimètrica).
  4. Configura els paràmetres de la clau, incloent-hi els permisos d'ús de la clau.
  5. Revisa i crea la clau.

Utilitzant AWS CLI

aws kms create-key --description "My CMK" --key-usage ENCRYPT_DECRYPT --origin AWS_KMS

Gestió de CMKs

  • Activació/Desactivació de CMKs: Pots activar o desactivar les CMKs per controlar el seu ús.
  • Rotació de CMKs: AWS KMS pot rotar automàticament les CMKs cada any.
  • Eliminació de CMKs: Programa una CMK per a la seva eliminació si ja no és necessària.

Xifratge i Desxifratge de Dades

Xifratge de Dades

Pots xifrar dades utilitzant CMKs directament o mitjançant claus de dades.

Xifratge amb AWS CLI

aws kms encrypt --key-id <CMK_ID> --plaintext fileb://example.txt --output text --query CiphertextBlob

Desxifratge de Dades

Per desxifrar dades, necessites tenir els permisos necessaris per utilitzar la CMK.

Desxifratge amb AWS CLI

aws kms decrypt --ciphertext-blob fileb://encrypted_file --output text --query Plaintext

Polítiques de Claus i Control d'Accés

Polítiques de Claus

Les polítiques de claus són documents JSON que especifiquen qui pot utilitzar i gestionar una CMK. Són similars a les polítiques IAM però són específiques per a KMS.

Exemple de Política de Clau

{
  "Version": "2012-10-17",
  "Id": "key-default-1",
  "Statement": [
    {
      "Sid": "Enable IAM User Permissions",
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::123456789012:root"
      },
      "Action": "kms:*",
      "Resource": "*"
    }
  ]
}

Concessions

Les concessions proporcionen permisos temporals per utilitzar una CMK. Són útils per delegar accés sense modificar la política de la clau.

Creació d'una Concessió

aws kms create-grant --key-id <CMK_ID> --grantee-principal <IAM_ROLE_ARN> --operations Encrypt Decrypt

Temes Avançats

Utilitzant Context de Xifratge

Un context de xifratge és informació addicional i no secreta que pots associar amb dades xifrades per proporcionar seguretat addicional.

Exemple amb Context de Xifratge

aws kms encrypt --key-id <CMK_ID> --plaintext fileb://example.txt --encryption-context Purpose=Test --output text --query CiphertextBlob

Claus Asimètriques

AWS KMS suporta claus asimètriques per a xifratge/desxifratge i signatura/verificació.

Creació d'una CMK Asimètrica

aws kms create-key --description "My Asymmetric CMK" --key-usage ENCRYPT_DECRYPT --key-spec RSA_2048

Conclusió

AWS KMS és un servei potent i flexible per gestionar claus de xifratge al núvol. Comprenent els conceptes clau, com crear i gestionar CMKs, i com xifrar i desxifrar dades, pots assegurar efectivament les teves dades a AWS. Funcions avançades com el context de xifratge i les claus asimètriques proporcionen capacitats addicionals per a casos d'ús més complexos.
