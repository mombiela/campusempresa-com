Amazon Route 53 és un servei web de Sistema de Noms de Domini (DNS) escalable i altament disponible dissenyat per dirigir els usuaris finals a aplicacions d'Internet traduint noms de domini en adreces IP. Aquest tema et guiarà des dels conceptes bàsics fins a les característiques avançades d'Amazon Route 53.

Introducció a Amazon Route 53

  • Què és Amazon Route 53?

    • Un servei web DNS escalable.
    • Dissenyat per a alta disponibilitat i fiabilitat.
    • Es pot utilitzar per dirigir usuaris a serveis d'AWS i recursos externs.

  • Característiques clau:

    • Registre de dominis.
    • Direcció DNS.
    • Verificació de salut i monitoratge.
    • Gestió del tràfic.

Configuració d'Amazon Route 53

Registrar un Domini

  • Passos per Registrar un Domini:
    1. Obre la consola de Route 53.
    2. Fes clic a "Registered Domains" al panell de navegació.
    3. Fes clic a "Register Domain."
    4. Introdueix el nom de domini que vols registrar i comprova la seva disponibilitat.
    5. Segueix les instruccions per completar el procés de registre.
# Exemple: Registrar un domini utilitzant AWS CLI
aws route53domains register-domain --domain-name example.com --duration-in-years 1 --admin-contact file://admin-contact.json --registrant-contact file://registrant-contact.json --tech-contact file://tech-contact.json

Crear una Zona Allotjada

  • Passos per Crear una Zona Allotjada:
    1. Obre la consola de Route 53.
    2. Fes clic a "Hosted Zones" al panell de navegació.
    3. Fes clic a "Create Hosted Zone."
    4. Introdueix el nom de domini i un comentari opcional.
    5. Fes clic a "Create."
# Exemple: Crear una zona allotjada utilitzant AWS CLI
aws route53 create-hosted-zone --name example.com --caller-reference unique-string

Polítiques de Direcció DNS

Direcció Simple

  • Política de Direcció Simple:
    • Assigna un domini a un únic recurs.
    • Adequat per a consultes DNS senzilles.
{
  "Comment": "Simple routing policy example",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ]
      }
    }
  ]
}

Direcció Ponderada

  • Política de Direcció Ponderada:
    • Distribueix el tràfic entre múltiples recursos basant-se en pesos assignats.
    • Útil per a equilibrar la càrrega i provar noves versions d'aplicacions.
{
  "Comment": "Weighted routing policy example",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Primary",
        "Weight": 70,
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ]
      }
    },
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Secondary",
        "Weight": 30,
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.45"
          }
        ]
      }
    }
  ]
}

Direcció Basada en Latència

  • Política de Direcció Basada en Latència:
    • Dirigeix el tràfic al recurs amb la latència més baixa per a l'usuari.
    • Millora el rendiment de l'aplicació reduint la latència.
{
  "Comment": "Latency-based routing policy example",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "US-East",
        "Region": "us-east-1",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ]
      }
    },
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "EU-West",
        "Region": "eu-west-1",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.45"
          }
        ]
      }
    }
  ]
}

Verificacions de Salut i Monitoratge

  • Verificacions de Salut:

    • Monitoritzen la salut dels recursos.
    • Dirigeixen automàticament el tràfic lluny dels recursos no saludables.

  • Crear una Verificació de Salut:

    1. Obre la consola de Route 53.
    2. Fes clic a "Health Checks" al panell de navegació.
    3. Fes clic a "Create Health Check."
    4. Configura els paràmetres de la verificació de salut.
    5. Fes clic a "Create."
# Exemple: Crear una verificació de salut utilitzant AWS CLI
aws route53 create-health-check --caller-reference unique-string --health-check-config file://health-check-config.json

Gestió del Tràfic

Direcció per Geolocalització

  • Política de Direcció per Geolocalització:
    • Dirigeix el tràfic basant-se en la ubicació geogràfica de l'usuari.
    • Útil per servir contingut localitzat.
{
  "Comment": "Geolocation routing policy example",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "US",
        "GeoLocation": {
          "CountryCode": "US"
        },
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ]
      }
    },
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "EU",
        "GeoLocation": {
          "CountryCode": "EU"
        },
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.45"
          }
        ]
      }
    }
  ]
}

Direcció per Fallada

  • Política de Direcció per Fallada:
    • Dirigeix el tràfic a un recurs primari tret que estigui no saludable, llavors dirigeix al recurs secundari.
    • Assegura alta disponibilitat i recuperació de desastres.
{
  "Comment": "Failover routing policy example",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Primary",
        "Failover": "PRIMARY",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ],
        "HealthCheckId": "health-check-id"
      }
    },
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Secondary",
        "Failover": "SECONDARY",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.45"
          }
        ]
      }
    }
  ]
}

Conclusió

Amazon Route 53 és un potent servei web DNS que proporciona una varietat de polítiques de direcció i capacitats de verificació de salut per assegurar alta disponibilitat i rendiment per a les teves aplicacions. Entenent i utilitzant aquestes característiques, pots gestionar i optimitzar efectivament el tràfic cap als teus recursos. Tant si estàs començant amb el registre de dominis com si estàs implementant estratègies avançades de gestió del tràfic, Amazon Route 53 ofereix les eines que necessites per tenir èxit.
