Introducció

Les solucions de recuperació de desastres i còpies de seguretat són components crítics de qualsevol infraestructura IT robusta. En aquest mòdul, explorarem com implementar aquestes solucions utilitzant Amazon Web Services (AWS). Aquest tema cobrirà els conceptes bàsics per a principiants i gradualment es mourà cap a conceptes i implementacions avançades.

Conceptes Clau

  • Recuperació de Desastres (DR): El procés de restaurar dades i aplicacions a un estat funcional després d'un desastre.
  • Còpia de Seguretat: El procés de crear còpies de dades per protegir-se contra la pèrdua de dades.
  • Objectiu de Temps de Recuperació (RTO): La quantitat màxima de temps acceptable per restaurar un sistema després d'un desastre.
  • Objectiu de Punt de Recuperació (RPO): La quantitat màxima de pèrdua de dades acceptable mesurada en temps.

Serveis d'AWS per a la Recuperació de Desastres i Còpies de Seguretat

AWS proporciona una varietat de serveis per ajudar a implementar solucions de recuperació de desastres i còpies de seguretat:

  • Amazon S3 (Simple Storage Service): Servei d'emmagatzematge d'objectes per emmagatzemar i recuperar qualsevol quantitat de dades.
  • Amazon RDS (Relational Database Service): Servei de base de dades relacional gestionat amb còpies de seguretat automàtiques.
  • AWS Backup: Servei centralitzat de còpies de seguretat per automatitzar i gestionar còpies de seguretat a través dels serveis d'AWS.
  • Amazon EC2 (Elastic Compute Cloud): Servidors virtuals escalables al núvol.
  • AWS Lambda: Servei de computació sense servidor que es pot utilitzar per a tasques de còpia de seguretat automatitzades.

Solucions de Còpia de Seguretat

Amazon S3

Amazon S3 és un servei d'emmagatzematge d'objectes altament durador i escalable. S'utilitza comunament per emmagatzemar dades de còpia de seguretat.

Exemple: Fer una Còpia de Seguretat de Dades a S3

import boto3

# Inicialitzar una sessió utilitzant Amazon S3
s3 = boto3.client('s3')

# Pujar un fitxer a S3
s3.upload_file('local_file.txt', 'my-bucket', 'backup/local_file.txt')

print("Fitxer pujat amb èxit.")

Explicació:

  • boto3 és el SDK d'AWS per a Python.
  • El mètode upload_file puja un fitxer del sistema local a un bucket de S3.

AWS Backup

AWS Backup proporciona una manera centralitzada de gestionar còpies de seguretat a través dels serveis d'AWS.

Exemple: Crear un Pla de Còpia de Seguretat

{
  "BackupPlanName": "MyBackupPlan",
  "Rules": [
    {
      "RuleName": "DailyBackup",
      "TargetBackupVaultName": "Default",
      "ScheduleExpression": "cron(0 12 * * ? *)",
      "StartWindowMinutes": 60,
      "CompletionWindowMinutes": 180,
      "Lifecycle": {
        "MoveToColdStorageAfterDays": 30,
        "DeleteAfterDays": 365
      }
    }
  ]
}

Explicació:

  • BackupPlanName especifica el nom del pla de còpia de seguretat.
  • Rules defineixen el calendari de còpia de seguretat i les polítiques de cicle de vida.

Estratègies de Recuperació de Desastres

Pilot Light

Una versió mínima d'un entorn sempre està en funcionament al núvol.

Warm Standby

Una versió reduïda d'un entorn completament funcional sempre està en funcionament.

Multi-Site

Un entorn de plena capacitat està en funcionament en paral·lel al núvol.

Exemple: Configurar un Warm Standby

  1. Crear un Entorn Reduït:
    • Utilitzar Amazon EC2 per llançar instàncies amb recursos mínims.
    • Utilitzar Amazon RDS per a una instància de base de dades reduïda.
  2. Automatitzar el Failover:
    • Utilitzar Route 53 per al failover DNS.
    • Utilitzar AWS Lambda per automatitzar l'escalat de recursos.

Exemple: Automatitzar el Failover amb Route 53

{
  "Comment": "Configuració de failover",
  "Changes": [
    {
      "Action": "UPSERT",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Primary",
        "Failover": "PRIMARY",
        "TTL": 60,
        "ResourceRecords": [{ "Value": "192.0.2.44" }]
      }
    },
    {
      "Action": "UPSERT",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Secondary",
        "Failover": "SECONDARY",
        "TTL": 60,
        "ResourceRecords": [{ "Value": "198.51.100.2" }]
      }
    }
  ]
}

Explicació:

  • Failover especifica si el registre és primari o secundari.
  • ResourceRecords conté les adreces IP per al failover.

Conclusió

Les solucions de recuperació de desastres i còpies de seguretat són essencials per mantenir la integritat i disponibilitat de les dades. AWS ofereix una suite completa de serveis per implementar aquestes solucions de manera efectiva. Entenent i utilitzant aquests serveis, pots assegurar-te que les teves aplicacions i dades estan protegides contra diversos tipus de fallades i desastres.
