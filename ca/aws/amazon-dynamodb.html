Introducció a Amazon DynamoDB

Amazon DynamoDB és un servei de base de dades NoSQL totalment gestionat proporcionat per AWS que ofereix un rendiment ràpid i previsible amb una escalabilitat sense problemes. Està dissenyat per manejar grans quantitats de dades i altes taxes de sol·licituds, fent-lo ideal per a aplicacions que requereixen una latència consistent de mil·lisegons d'una sola xifra a qualsevol escala.

Característiques Clau

  • Totalment Gestionat: AWS s'encarrega de les tasques administratives com la provisió de maquinari, la configuració i la configuració.
  • Escalable: Escala automàticament cap amunt i cap avall per ajustar-se a la capacitat i mantenir el rendiment.
  • Alt Rendiment: Proporciona un rendiment consistent i de baixa latència.
  • Model de Dades Flexible: Suporta tant models de document com de magatzem de claus-valor.
  • Seguretat Integrada: Ofereix xifratge en repòs, control d'accés granular i monitorització completa.

Començant amb DynamoDB

Configurant DynamoDB

  1. Crea un Compte AWS: Si encara no en tens un, registra't per obtenir un compte AWS.
  2. Accedeix a la Consola DynamoDB: Navega al servei DynamoDB des de la Consola de Gestió AWS.
  3. Crea una Taula: Defineix una taula amb una clau primària.

Creant una Taula

Una taula a DynamoDB és una col·lecció d'elements, i cada element és una col·lecció d'atributs. Cada taula ha de tenir una clau primària, que pot ser un sol atribut (clau de partició) o una clau composta (clau de partició i clau de classificació).

import boto3

# Inicialitza una sessió utilitzant Amazon DynamoDB
dynamodb = boto3.resource('dynamodb', region_name='us-west-2')

# Crea la taula DynamoDB
table = dynamodb.create_table(
    TableName='Movies',
    KeySchema=[
        {
            'AttributeName': 'year',
            'KeyType': 'HASH'  # Clau de partició
        },
        {
            'AttributeName': 'title',
            'KeyType': 'RANGE'  # Clau de classificació
        }
    ],
    AttributeDefinitions=[
        {
            'AttributeName': 'year',
            'AttributeType': 'N'
        },
        {
            'AttributeName': 'title',
            'AttributeType': 'S'
        }
    ],
    ProvisionedThroughput={
        'ReadCapacityUnits': 10,
        'WriteCapacityUnits': 10
    }
)

# Espera fins que la taula existeixi.
table.meta.client.get_waiter('table_exists').wait(TableName='Movies')

print("Estat de la taula:", table.table_status)

Operacions Bàsiques

  • PutItem: Afegeix un nou element a la taula.
  • GetItem: Recupera un element de la taula.
  • UpdateItem: Modifica un element existent.
  • DeleteItem: Elimina un element de la taula.
# Exemple de PutItem
table.put_item(
   Item={
        'year': 2015,
        'title': 'The Big New Movie',
        'info': {
            'plot': 'Nothing happens at all.',
            'rating': 0
        }
    }
)

# Exemple de GetItem
response = table.get_item(
    Key={
        'year': 2015,
        'title': 'The Big New Movie'
    }
)
item = response['Item']
print(item)

Conceptes Avançats de DynamoDB

Índexs Secundaris

Els índexs secundaris permeten consultar les dades de la taula utilitzant una clau alternativa. DynamoDB suporta dos tipus d'índexs secundaris:

  • Índex Secundari Global (GSI): Es pot crear amb una clau de partició i una clau de classificació que són diferents de les de la taula base.
  • Índex Secundari Local (LSI): Es pot crear amb la mateixa clau de partició que la taula base però una clau de classificació diferent.

Fluxos

Els fluxos de DynamoDB capturen una seqüència ordenada en el temps de modificacions a nivell d'element en una taula i emmagatzemen aquesta informació durant fins a 24 hores. Això es pot utilitzar per:

  • Replicar dades a través de múltiples taules.
  • Activar funcions AWS Lambda per al processament en temps real.

Transaccions

Les transaccions de DynamoDB proporcionen atomicitat, consistència, aïllament i durabilitat (ACID) a través d'una o més taules dins d'un sol compte AWS i regió.

# Exemple de transacció
dynamodb.transact_write_items(
    TransactItems=[
        {
            'Put': {
                'TableName': 'Movies',
                'Item': {
                    'year': {'N': '2015'},
                    'title': {'S': 'The Big New Movie'},
                    'info': {'M': {'plot': {'S': 'Nothing happens at all.'}, 'rating': {'N': '0'}}}
                }
            }
        },
        {
            'Update': {
                'TableName': 'Movies',
                'Key': {
                    'year': {'N': '2013'},
                    'title': {'S': 'The Old Movie'}
                },
                'UpdateExpression': 'SET info.rating = :r',
                'ExpressionAttributeValues': {
                    ':r': {'N': '5'}
                }
            }
        }
    ]
)

Millors Pràctiques

Modelatge de Dades

  • Entendre els Patrons d'Accés: Dissenya l'esquema de la taula basant-te en com planejes accedir a les dades.
  • Utilitza Claus Compostes: Aprofita les claus de partició i de classificació per organitzar les dades de manera eficient.
  • Evita Particions Calentes: Distribueix les operacions de lectura i escriptura de manera uniforme a través de les particions.

Optimització del Rendiment

  • Rendiment Proveït: Monitoritza i ajusta les unitats de capacitat de lectura/escriptura segons sigui necessari.
  • Utilitza Índexs amb Saviesa: Crea índexs secundaris per optimitzar el rendiment de les consultes.
  • Operacions en Lot: Utilitza operacions en lot per reduir el nombre de crides a l'API.

Conclusió

Amazon DynamoDB és un servei de base de dades NoSQL potent que ofereix alt rendiment, escalabilitat i flexibilitat. Entenent els seus conceptes bàsics, configurant taules, realitzant operacions bàsiques i avançades, i seguint les millors pràctiques, pots aprofitar efectivament DynamoDB per a una àmplia gamma d'aplicacions. Tant si estàs construint una aplicació web senzilla com un sistema complex i de gran trànsit, DynamoDB proporciona les eines i característiques necessàries per satisfer les teves necessitats.
