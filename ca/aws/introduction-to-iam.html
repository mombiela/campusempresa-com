Què és IAM?

AWS Identity and Access Management (IAM) és un servei web que t'ajuda a controlar de manera segura l'accés als serveis i recursos d'AWS per als teus usuaris. IAM et permet gestionar permisos per a usuaris i grups per assegurar-te que només els usuaris autoritzats puguin accedir a recursos específics.

Conceptes Clau

  • Usuaris: Comptes individuals que representen una persona o servei.
  • Grups: Col·leccions d'usuaris que comparteixen permisos comuns.
  • Rols: Credencials temporals que poden ser assumides per usuaris o serveis.
  • Polítiques: Documents que defineixen permisos i s'assignen a usuaris, grups o rols.

Per què Utilitzar IAM?

  • Seguretat: Controla qui pot accedir als teus recursos.
  • Permisos Granulars: Defineix permisos específics per a diferents usuaris i serveis.
  • Gestió Centralitzada: Gestiona l'accés des d'una única ubicació.

Crear un Usuari IAM

Per crear un usuari IAM, segueix aquests passos:

  1. Inicia sessió a la Consola de Gestió d'AWS.
  2. Obre la consola IAM a https://console.aws.amazon.com/iam/.
  3. Al panell de navegació, tria "Usuaris" i després "Afegeix usuari".
  4. Introdueix els detalls de l'usuari i selecciona el tipus d'accés (Accés programàtic, Accés a la Consola de Gestió d'AWS).
  5. Adjunta polítiques a l'usuari per concedir permisos.
  6. Revisa i crea l'usuari.

Exemple: Crear un Usuari IAM amb AWS CLI

aws iam create-user --user-name JohnDoe
aws iam attach-user-policy --user-name JohnDoe --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

En aquest exemple:

  • La primera comanda crea un usuari anomenat JohnDoe.
  • La segona comanda adjunta la política AmazonS3ReadOnlyAccess a JohnDoe, concedint accés de només lectura a Amazon S3.

Gestionar Polítiques IAM

Les polítiques IAM són documents JSON que defineixen permisos. Aquí tens una estructura bàsica d'una política:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:ListBucket",
      "Resource": "arn:aws:s3:::example_bucket"
    }
  ]
}

Elements Clau d'una Política

  • Version: La versió del llenguatge de la política.
  • Statement: Una o més declaracions individuals.
    • Effect: Pot ser Allow o Deny.
    • Action: L'acció específica (per exemple, s3:ListBucket).
    • Resource: L'ARN del recurs (per exemple, arn:aws:s3:::example_bucket).

Rols IAM

Els rols IAM et permeten delegar accés amb credencials temporals. Els rols són útils per a:

  • Concedir permisos a serveis d'AWS.
  • Permetre que usuaris d'un altre compte d'AWS accedeixin als teus recursos.
  • Permetre que aplicacions que s'executen en instàncies EC2 accedeixin a recursos d'AWS.

Exemple: Crear un Rol IAM per a EC2

  1. Obre la consola IAM.
  2. Al panell de navegació, tria "Rols" i després "Crea rol".
  3. Selecciona "Servei d'AWS" i després "EC2".
  4. Adjunta les polítiques necessàries.
  5. Completa el procés de creació del rol.

Exemple: Crear un Rol IAM amb AWS CLI

aws iam create-role --role-name EC2S3AccessRole --assume-role-policy-document file://trust-policy.json
aws iam attach-role-policy --role-name EC2S3AccessRole --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

En aquest exemple:

  • La primera comanda crea un rol anomenat EC2S3AccessRole amb una política de confiança definida a trust-policy.json.
  • La segona comanda adjunta la política AmazonS3ReadOnlyAccess al rol.

Millors Pràctiques per a IAM

  • Menys Privilegis: Concedeix només els permisos necessaris perquè els usuaris puguin realitzar les seves tasques.
  • Utilitza Grups: Gestiona permisos per a múltiples usuaris assignant-los a grups.
  • Activa MFA: Utilitza Autenticació Multifactor per a una seguretat addicional.
  • Auditories Regulars: Revisa i actualitza periòdicament els permisos.

Conclusió

IAM és un servei fonamental a AWS que t'ajuda a gestionar l'accés als teus recursos de manera segura. Entenent i utilitzant les característiques d'IAM, com ara usuaris, grups, rols i polítiques, pots assegurar-te que el teu entorn AWS sigui tant segur com eficient. Recorda seguir les millors pràctiques per mantenir una postura de seguretat robusta.
