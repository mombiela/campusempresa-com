Introducció a l'Equilibri de Càrrega Elàstic (ELB)

L'Equilibri de Càrrega Elàstic (ELB) és un servei proporcionat per AWS que distribueix automàticament el trànsit d'aplicacions entrant entre múltiples destinacions, com ara instàncies EC2, contenidors i adreces IP. ELB ajuda a garantir un alt nivell de disponibilitat i tolerància a fallades per a les teves aplicacions.

Conceptes Clau

  • Equilibrador de Càrrega: Un dispositiu que distribueix el trànsit de xarxa o aplicació entre diversos servidors.
  • Destinació: La destinació del trànsit, com ara instàncies EC2, contenidors o adreces IP.
  • Listener: Un procés que comprova les sol·licituds de connexió utilitzant el protocol i el port que configures.
  • Grup de Destinacions: Un grup de destinacions a les quals un equilibrador de càrrega dirigeix les sol·licituds.

Tipus d'Equilibradors de Càrrega

AWS ofereix tres tipus d'equilibradors de càrrega:

  1. Equilibrador de Càrrega d'Aplicacions (ALB)

    • Més adequat per al trànsit HTTP i HTTPS.
    • Opera a la capa d'aplicació (Capa 7).
    • Suporta enrutament avançat, com ara enrutament basat en camins i enrutament basat en host.

  2. Equilibrador de Càrrega de Xarxa (NLB)

    • Més adequat per al trànsit TCP, UDP i TLS.
    • Opera a la capa de transport (Capa 4).
    • Capable de gestionar milions de sol·licituds per segon mantenint latències ultra-baixes.

  3. Equilibrador de Càrrega Clàssic (CLB)

    • Opera tant a la capa d'aplicació (Capa 7) com a la capa de transport (Capa 4).
    • Adequat per a aplicacions construïdes dins de la xarxa EC2-Classic.

Taula Comparativa

| Característica | Equilibrador de Càrrega d'Aplicacions (ALB) | Equilibrador de Càrrega de Xarxa (NLB) | Equilibrador de Càrrega Clàssic (CLB) | |-----------------------------|--------------------------------------------|----------------------------------------|---------------------------------------| | Protocols Suportats | HTTP, HTTPS | TCP, UDP, TLS | HTTP, HTTPS, TCP, SSL | | Capa | 7 | 4 | 4 i 7 | | Enrutament Avançat | Sí | No | No | | Suport WebSocket | Sí | No | No | | Suport IP Estàtica | No | Sí | No | | Verificacions de Salut | HTTP, HTTPS | TCP, HTTP, HTTPS | TCP, HTTP, HTTPS |

Configuració d'un Equilibrador de Càrrega d'Aplicacions (ALB)

Guia Pas a Pas

  1. Crear un Equilibrador de Càrrega

    • Navega al Tauler de Control d'EC2.
    • Selecciona "Equilibradors de Càrrega" des del menú de l'esquerra.
    • Fes clic a "Crear Equilibrador de Càrrega" i tria "Equilibrador de Càrrega d'Aplicacions".

  2. Configurar l'Equilibrador de Càrrega

    • Proporciona un nom per al teu equilibrador de càrrega.
    • Selecciona l'esquema (Exposat a Internet o Intern).
    • Tria el tipus d'adreça IP (IPv4 o Dualstack).
    • Configura els listeners (per defecte és HTTP al port 80).

  3. Configurar la Seguretat

    • Si utilitzes HTTPS, configura els certificats SSL.

  4. Configurar Grups de Seguretat

    • Selecciona o crea un grup de seguretat que permeti el trànsit entrant als ports dels listeners.

  5. Configurar l'Enrutament

    • Crea un grup de destinacions.
    • Tria el tipus de destinació (Instàncies, adreces IP o funcions Lambda).
    • Configura les verificacions de salut (protocol, camí i port).

  6. Registrar Destinacions

    • Afegeix les instàncies o adreces IP a les quals vols dirigir el trànsit.

  7. Revisar i Crear

    • Revisa la teva configuració i fes clic a "Crear".

Exemple de Codi: Crear un ALB utilitzant AWS CLI

aws elbv2 create-load-balancer \
    --name my-load-balancer \
    --subnets subnet-12345678 subnet-87654321 \
    --security-groups sg-12345678 \
    --scheme internet-facing \
    --type application

Exemple de Codi: Registrar Destinacions utilitzant AWS CLI

aws elbv2 register-targets \
    --target-group-arn arn:aws:elasticloadbalancing:region:account-id:targetgroup/my-targets/1234567890123456 \
    --targets Id=i-1234567890abcdef0 Id=i-0abcdef1234567890

Funcions Avançades d'ELB

Sessions Adhesives

Les sessions adhesives (afinitat de sessió) et permeten vincular la sessió d'un usuari a una destinació específica dins del grup de destinacions. Això assegura que totes les sol·licituds d'un usuari durant una sessió es dirigeixin a la mateixa destinació.

Equilibri de Càrrega entre Zones

L'equilibri de càrrega entre zones distribueix les sol·licituds entrants uniformement entre totes les destinacions en totes les Zones de Disponibilitat habilitades. Això millora la tolerància a fallades i la disponibilitat de la teva aplicació.

Terminació SSL

La terminació SSL desxifra les sol·licituds SSL entrants a l'equilibrador de càrrega abans d'enviar-les a les destinacions. Això redueix la càrrega sobre les teves destinacions i simplifica la gestió dels certificats SSL.

Integració amb Auto Scaling

ELB s'integra perfectament amb Auto Scaling, permetent-te afegir o eliminar destinacions automàticament segons la demanda. Això assegura que la teva aplicació pugui gestionar nivells variables de trànsit sense intervenció manual.

Monitorització i Registre

Mètriques de CloudWatch

ELB proporciona diverses mètriques de CloudWatch per monitoritzar el rendiment i la salut del teu equilibrador de càrrega, com ara:

  • RequestCount
  • HealthyHostCount
  • UnHealthyHostCount
  • Latency

Registres d'Accés

Els registres d'accés capturen informació detallada sobre les sol·licituds enviades al teu equilibrador de càrrega, incloent-hi el temps en què es va rebre la sol·licitud, l'adreça IP del client i el camí de la sol·licitud. Aquests registres es poden emmagatzemar a S3 per a l'anàlisi.

Exemple de Codi: Habilitar Registres d'Accés utilitzant AWS CLI

aws elbv2 modify-load-balancer-attributes \
    --load-balancer-arn arn:aws:elasticloadbalancing:region:account-id:loadbalancer/app/my-load-balancer/50dc6c495c0c9188 \
    --attributes Key=access_logs.s3.enabled,Value=true Key=access_logs.s3.bucket,Value=my-logs-bucket Key=access_logs.s3.prefix,Value=my-app

Conclusió

L'Equilibri de Càrrega Elàstic (ELB) és un servei potent que millora la disponibilitat, la tolerància a fallades i l'escalabilitat de les teves aplicacions. Comprenent els diferents tipus d'equilibradors de càrrega, configurant i configurant un ALB, i aprofitant les funcions avançades, pots assegurar-te que les teves aplicacions siguin robustes i puguin gestionar nivells variables de trànsit de manera eficient. Les capacitats de monitorització i registre ajuden encara més a mantenir la salut i el rendiment de les teves aplicacions.
