Introducció als RDDs

Resilient Distributed Datasets (RDDs) són una estructura de dades fonamental d'Apache Spark. Són col·leccions d'objectes immutables i distribuïdes que es poden processar en paral·lel. Els RDDs proporcionen tolerància a fallades, permetent que Spark es recuperi de fallades de nodes.

Conceptes Clau

  • Immutabilitat: Un cop creats, els RDDs no es poden canviar. Les transformacions en RDDs produeixen nous RDDs.
  • Distribuït: Els RDDs estan distribuïts a través de múltiples nodes en un clúster.
  • Tolerància a Fallades: Els RDDs poden recuperar-se de fallades de nodes utilitzant informació de línia.
  • Avaluació Mandrosa: Les transformacions en RDDs no s'executen fins que es realitza una acció.

Creació de RDDs

Els RDDs es poden crear de diverses maneres:

  1. Paral·lelitzant una col·lecció existent.
  2. Carregant un conjunt de dades extern.

Exemple: Paral·lelitzant una Col·lecció

from pyspark import SparkContext

sc = SparkContext("local", "RDD Example")
data = [1, 2, 3, 4, 5]
rdd = sc.parallelize(data)
print(rdd.collect())

Explicació:

  • SparkContext és el punt d'entrada per a la funcionalitat de Spark.
  • El mètode parallelize crea un RDD a partir d'una col·lecció existent.
  • L'acció collect recupera tot el RDD al programa controlador.

Exemple: Carregant un Conjunt de Dades Extern

rdd = sc.textFile("path/to/file.txt")
print(rdd.collect())

Explicació:

  • El mètode textFile llegeix un fitxer de text i crea un RDD de cadenes.

Transformacions de RDD

Les transformacions creen un nou RDD a partir d'un existent. Són mandroses i només s'executen quan es crida a una acció.

Transformacions Comunes

  • map: Aplica una funció a cada element.
  • filter: Selecciona els elements que compleixen una condició.
  • flatMap: Similar a map, però cada element d'entrada es pot mapar a zero o més elements de sortida.
  • union: Combina dos RDDs.
  • distinct: Elimina duplicats.

Exemple: Utilitzant `map` i `filter`

rdd = sc.parallelize([1, 2, 3, 4, 5])
mapped_rdd = rdd.map(lambda x: x * 2)
filtered_rdd = mapped_rdd.filter(lambda x: x > 5)
print(filtered_rdd.collect())

Explicació:

  • La transformació map multiplica cada element per 2.
  • La transformació filter manté els elements majors de 5.

Accions de RDD

Les accions desencadenen l'execució de transformacions i retornen un resultat al programa controlador.

Accions Comunes

  • collect: Retorna tots els elements del RDD.
  • count: Retorna el nombre d'elements.
  • take: Retorna els primers n elements.
  • reduce: Agrega elements utilitzant una funció.
  • saveAsTextFile: Desa el RDD en un fitxer de text.

Exemple: Utilitzant `reduce`

rdd = sc.parallelize([1, 2, 3, 4, 5])
sum = rdd.reduce(lambda x, y: x + y)
print(sum)

Explicació:

  • L'acció reduce suma tots els elements del RDD.

Operacions Avançades de RDD

RDDs de Parells Clau-Valor

Els RDDs poden contenir parells clau-valor, permetent operacions com reduceByKey i groupByKey.

Exemple: Utilitzant `reduceByKey`

rdd = sc.parallelize([("a", 1), ("b", 1), ("a", 1)])
reduced_rdd = rdd.reduceByKey(lambda x, y: x + y)
print(reduced_rdd.collect())

Explicació:

  • reduceByKey agrega valors amb la mateixa clau.

Emmagatzematge en Memòria i Persistència

Els RDDs es poden emmagatzemar en memòria o persistir per optimitzar el rendiment per a algorismes iteratius.

Exemple: Emmagatzemant en Memòria un RDD

rdd = sc.parallelize([1, 2, 3, 4, 5])
rdd.cache()
print(rdd.count())

Explicació:

  • El mètode cache emmagatzema el RDD en memòria per a un accés més ràpid.

Conclusió

Els RDDs són una abstracció poderosa en Apache Spark, permetent el processament de dades distribuïdes amb tolerància a fallades i escalabilitat. Entendre com crear, transformar i realitzar accions en RDDs és essencial per aprofitar les capacitats de Spark. A mesura que avancis, exploraràs operacions i optimitzacions més complexes, com els RDDs de parells clau-valor i les estratègies d'emmagatzematge en memòria.
