Apache Spark és un potent motor de processament de codi obert dissenyat per al processament i l'anàlisi de dades a gran escala. Entendre la seva arquitectura és crucial per utilitzar eficaçment les seves capacitats. Aquest tema cobrirà els components i conceptes bàsics de l'arquitectura de Spark, des dels fonaments fins als aspectes més avançats.

Components Clau de l'Arquitectura de Spark

Programa Driver

  • Definició: El programa driver és el procés principal de control que coordina l'execució de tasques en el clúster de Spark.
  • Responsabilitats:
    • Converteix el codi de l'usuari en un graf acíclic dirigit (DAG) d'etapes.
    • Programa tasques en el clúster.
    • Recull i mostra els resultats.

Gestor de Clúster

  • Definició: El gestor de clúster és responsable de gestionar els recursos a través del clúster.
  • Tipus:
    • Standalone: El gestor de clúster integrat de Spark.
    • Apache Mesos: Un gestor de clúster general que pot gestionar recursos per a múltiples marcs.
    • Hadoop YARN: El gestor de recursos per a clústers Hadoop.
    • Kubernetes: Un sistema d'orquestració de contenidors que també pot gestionar recursos de Spark.

Executors

  • Definició: Els executors són agents distribuïts responsables d'executar tasques i emmagatzemar dades.
  • Responsabilitats:
    • Executar el codi assignat pel driver.
    • Emmagatzemar dades en memòria o en disc.
    • Informar l'estat i els resultats al driver.

Tasques

  • Definició: Les tasques són les unitats més petites de treball a Spark, representant una única operació en una partició de dades.
  • Execució: Les tasques són executades pels executors.

Model d'Execució de Spark

Treballs i Etapes

  • Treball: Un treball és desencadenat per una acció (per exemple, count(), collect()) i consta de múltiples etapes.
  • Etapa: Una etapa és un conjunt de tasques que es poden executar en paral·lel, dividides per límits de shuffle.

Graf Acíclic Dirigit (DAG)

  • Definició: Un DAG és una representació de la seqüència de càlculs realitzats sobre les dades.
  • Importància: Ajuda a optimitzar el pla d'execució dividint-lo en etapes.

Exemple: Comptador de Paraules Simple

from pyspark import SparkContext

# Inicialitzar SparkContext
sc = SparkContext("local", "WordCount")

# Llegir fitxer d'entrada
text_file = sc.textFile("hdfs://path/to/input.txt")

# Realitzar comptador de paraules
counts = text_file.flatMap(lambda line: line.split(" ")) \
                  .map(lambda word: (word, 1)) \
                  .reduceByKey(lambda a, b: a + b)

# Guardar el resultat
counts.saveAsTextFile("hdfs://path/to/output")
  • Explicació:
    • flatMap: Divideix cada línia en paraules.
    • map: Mapeja cada paraula a una parella (paraula, 1).
    • reduceByKey: Agrega els comptadors per a cada paraula.
    • saveAsTextFile: Guarda el resultat a HDFS.

Conceptes Avançats

Conjunts de Dades Distribuïts Resilients (RDDs)

  • Definició: Els RDDs són l'estructura de dades fonamental de Spark, representant una col·lecció distribuïda immutable d'objectes.
  • Propietats:
    • Immutabilitat: Un cop creats, els RDDs no es poden canviar.
    • Tolerància a Fallades: Els RDDs es poden recomputar en cas de fallades de nodes.
    • Avaluació Mandrosa: Les transformacions en RDDs no s'executen fins que es crida a una acció.

DataFrames i Datasets

  • DataFrame: Una col·lecció distribuïda de dades organitzada en columnes amb nom, similar a una taula en una base de dades relacional.
  • Dataset: Una col·lecció fortament tipada d'objectes, proporcionant els beneficis dels RDDs amb les optimitzacions dels DataFrames.

Spark SQL

  • Definició: Spark SQL és un mòdul per al processament de dades estructurades, permetent la consulta de dades mitjançant SQL així com l'API de DataFrame.
  • Exemple:
from pyspark.sql import SparkSession

# Inicialitzar SparkSession
spark = SparkSession.builder.appName("SparkSQLExample").getOrCreate()

# Carregar dades en DataFrame
df = spark.read.json("hdfs://path/to/input.json")

# Registrar DataFrame com a vista temporal SQL
df.createOrReplaceTempView("people")

# Executar consulta SQL
sqlDF = spark.sql("SELECT name FROM people WHERE age > 21")

# Mostrar el resultat
sqlDF.show()

Conclusió

Entendre l'arquitectura de Spark és essencial per aprofitar tot el seu potencial en el processament de grans volums de dades. El programa driver, el gestor de clúster, els executors i les tasques formen els components clau, mentre que el model d'execució DAG assegura una programació i execució eficient de les tasques. Conceptes avançats com els RDDs, DataFrames i Spark SQL proporcionen abstraccions poderoses per manejar tasques complexes de processament de dades. En dominar aquests components i conceptes, pots construir aplicacions robustes i escalables de processament de dades amb Apache Spark.
