Introducció a HTTP i APIs REST

En aquesta secció, cobrirem els conceptes bàsics de HTTP i APIs REST, que són essencials per comunicar-se amb serveis web en aplicacions Android.

  • HTTP (HyperText Transfer Protocol): La base de qualsevol intercanvi de dades a la Web i un protocol utilitzat per transmetre sol·licituds d'hipertext i informació entre servidors i clients.
  • REST (Representational State Transfer): Un estil arquitectònic per dissenyar aplicacions en xarxa. Les APIs RESTful utilitzen sol·licituds HTTP per realitzar operacions CRUD (Crear, Llegir, Actualitzar, Eliminar).

Entenent HTTP

HTTP és un protocol utilitzat per enviar i rebre dades a través de la web. Aquí hi ha alguns conceptes clau:

  • Mètodes HTTP:

    • GET: Recuperar dades d'un servidor.
    • POST: Enviar dades a un servidor per crear un recurs.
    • PUT: Actualitzar un recurs existent al servidor.
    • DELETE: Eliminar un recurs del servidor.

  • Codis d'Estat HTTP:

    • 200 OK: La sol·licitud ha estat exitosa.
    • 201 Created: La sol·licitud ha estat exitosa i s'ha creat un recurs.
    • 400 Bad Request: La sol·licitud no s'ha pogut entendre o faltaven paràmetres necessaris.
    • 401 Unauthorized: L'autenticació ha fallat o l'usuari no té permisos per a l'acció desitjada.
    • 404 Not Found: El recurs sol·licitat no s'ha pogut trobar.
    • 500 Internal Server Error: S'ha produït un error al servidor.

Introducció a les APIs REST

Les APIs REST et permeten interactuar amb serveis web utilitzant mètodes HTTP. Són sense estat i es poden utilitzar per realitzar operacions CRUD.

  • Recursos i URIs: En REST, els recursos s'identifiquen per URIs (Identificadors Uniformes de Recursos).

    • Exemple: https://api.example.com/users podria representar una col·lecció d'usuaris.

  • JSON (JavaScript Object Notation): Un format lleuger d'intercanvi de dades que és fàcil de llegir i escriure per als humans i fàcil de parsejar i generar per a les màquines.

Realitzant Sol·licituds HTTP en Android

Per realitzar sol·licituds HTTP en Android, pots utilitzar biblioteques com HttpURLConnection, OkHttp o Retrofit. Aquí, ens centrarem en Retrofit per la seva simplicitat i facilitat d'ús.

Configurant Retrofit

Primer, afegeix la dependència de Retrofit al teu fitxer build.gradle:

dependencies {
    implementation 'com.squareup.retrofit2:retrofit:2.9.0'
    implementation 'com.squareup.retrofit2:converter-gson:2.9.0'
}

Creant una Instància de Retrofit

Crea una instància de Retrofit per definir la URL base i la fàbrica de convertidors:

Retrofit retrofit = new Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .addConverterFactory(GsonConverterFactory.create())
    .build();

Definint Punts Finals de l'API

Defineix els punts finals utilitzant una interfície:

public interface ApiService {
    @GET("users")
    Call<List<User>> getUsers();

    @POST("users")
    Call<User> createUser(@Body User user);
}

Realitzant una Sol·licitud

Crea una instància del servei API i realitza una sol·licitud:

Conclusió

En aquest tema, hem cobert els conceptes bàsics de HTTP i APIs REST, incloent mètodes HTTP, codis d'estat, i com realitzar sol·licituds HTTP en Android utilitzant Retrofit. Entendre aquests conceptes és crucial per integrar serveis web a les teves aplicacions Android. Practicant els exemples i exercicis proporcionats, seràs capaç de comunicar-te efectivament amb APIs RESTful i gestionar dades JSON a les teves aplicacions.
