Proves Unitàries a Android Studio

Les proves unitàries són una part crucial del procés de desenvolupament de programari, assegurant que unitats individuals de codi funcionin com s'espera. En el desenvolupament d'Android, les proves unitàries ajuden a mantenir la qualitat i la fiabilitat del codi. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de les proves unitàries a Android Studio.

Introducció a les Proves Unitàries

  • Definició: Les proves unitàries impliquen provar components o funcions individuals d'una aplicació de manera aïllada.
  • Propòsit: Verificar que cada unitat del programari funcioni com està dissenyada.
  • Beneficis:
    • Detecta errors aviat.
    • Simplifica la depuració.
    • Facilita la refactorització del codi.
    • Millora la qualitat i la fiabilitat del codi.

Configuració de les Proves Unitàries a Android Studio

  1. Crear un Nou Projecte:

    • Obre Android Studio i crea un nou projecte.
    • Assegura't de seleccionar el SDK i la configuració del projecte adequats.

  2. Afegir Dependències de Prova:

    • Obre el fitxer build.gradle (Mòdul: app) i afegeix les següents dependències: 
      dependencies {
    testImplementation 'junit:junit:4.13.2'
    testImplementation 'org.mockito:mockito-core:3.9.0'
}

  3. Sincronitzar el Projecte:

    • Fes clic a "Sync Now" per descarregar i integrar les dependències.

Escriure la Teva Primera Prova Unitària

  1. Crear una Classe de Prova:

    • Navega al directori src/test/java.
    • Fes clic dret i selecciona New > Java Class.
    • Anomena la classe ExampleUnitTest.

  2. Escriure una Prova Simple:

    • Afegeix un mètode de prova simple a la classe: 
      import org.junit.Test;
import static org.junit.Assert.*;

public class ExampleUnitTest {
    @Test
    public void addition_isCorrect() {
        assertEquals(4, 2 + 2);
    }
}

  3. Executar la Prova:

    • Fes clic dret a la classe de prova i selecciona Run 'ExampleUnitTest'.
    • Revisa els resultats a la finestra de Run.

Conceptes Avançats de Proves Unitàries

Mocking amb Mockito

  • Definició: El mocking és una tècnica per crear una versió simulada d'un objecte.
  • Propòsit: Aïllar la unitat de codi que s'està provant i controlar les seves dependències.

Exemple: Mocking d'una Dependència

  1. Afegir la Dependència de Mockito (ja afegida en el pas de configuració):

    testImplementation 'org.mockito:mockito-core:3.9.0'

  2. Crear una Classe amb Dependències:

    public class UserService {
    private UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public String getUserFullName(int userId) {
        User user = userRepository.getUserById(userId);
        return user.getFirstName() + " " + user.getLastName();
    }
}

public interface UserRepository {
    User getUserById(int userId);
}

public class User {
    private String firstName;
    private String lastName;

    // Constructor, getters, and setters
}

  3. Escriure una Prova amb Mockito:

    import org.junit.Before;
import org.junit.Test;
import org.mockito.Mock;
import org.mockito.MockitoAnnotations;

import static org.mockito.Mockito.*;
import static org.junit.Assert.*;

public class UserServiceTest {
    @Mock
    private UserRepository userRepository;

    private UserService userService;

    @Before
    public void setUp() {
        MockitoAnnotations.initMocks(this);
        userService = new UserService(userRepository);
    }

    @Test
    public void getUserFullName_returnsCorrectFullName() {
        User user = new User("John", "Doe");
        when(userRepository.getUserById(1)).thenReturn(user);

        String fullName = userService.getUserFullName(1);
        assertEquals("John Doe", fullName);
    }
}

Millors Pràctiques per a les Proves Unitàries

  • Escriure Proves Independents: Assegura't que les proves no depenguin les unes de les altres.
  • Utilitzar Noms Descriptius: Anomena les proves de manera clara per descriure el seu propòsit.
  • Provar Casos Extrems: Inclou proves per a condicions límit i casos extrems.
  • Mantenir les Proves Ràpides: Les proves unitàries han de ser ràpides d'executar.
  • Mantenir les Proves: Actualitza regularment les proves a mesura que el codi evoluciona.

Conclusió

Les proves unitàries són una pràctica essencial en el desenvolupament d'Android, ajudant a assegurar que el teu codi sigui fiable i mantenible. Configurant proves unitàries a Android Studio, escrivint proves bàsiques i avançades, i seguint les millors pràctiques, pots millorar significativament la qualitat de les teves aplicacions. Recorda, la clau per a unes proves unitàries efectives és la consistència i la minuciositat. Bones proves!
