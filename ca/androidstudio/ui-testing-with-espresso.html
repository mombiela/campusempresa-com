Introducció a les proves d'interfície d'usuari

  • Què són les proves d'interfície d'usuari?

    • Les proves d'interfície d'usuari asseguren que la interfície d'usuari d'una aplicació funcioni com s'espera.
    • Implica provar els elements visuals i les interaccions de l'usuari.

  • Per què utilitzar Espresso per a les proves d'interfície d'usuari?

    • Espresso és un marc de proves per a Android que facilita l'escriptura de proves d'interfície d'usuari fiables.
    • Forma part de la Biblioteca de Suport de Proves d'Android.
    • Les proves d'Espresso s'executen en dispositius reals i emuladors.

Configuració d'Espresso a Android Studio

  • Afegir dependències

    • Obre el teu fitxer build.gradle i afegeix les següents dependències: 
      androidTestImplementation 'androidx.test.espresso:espresso-core:3.4.0'
androidTestImplementation 'androidx.test.ext:junit:1.1.3'
androidTestImplementation 'androidx.test:rules:1.4.0'

  • Sincronitzar el projecte

    • Després d'afegir les dependències, sincronitza el teu projecte per descarregar les biblioteques necessàries.

Escriure la teva primera prova d'Espresso

  • Crear una classe de prova

    • Crea una nova classe de prova al directori androidTest: 
      package com.example.myapp;

import androidx.test.ext.junit.runners.AndroidJUnit4;
import androidx.test.rule.ActivityTestRule;

import org.junit.Rule;
import org.junit.Test;
import org.junit.runner.RunWith;

import static androidx.test.espresso.Espresso.onView;
import static androidx.test.espresso.action.ViewActions.click;
import static androidx.test.espresso.matcher.ViewMatchers.withId;

@RunWith(AndroidJUnit4.class)
public class MainActivityTest {

    @Rule
    public ActivityTestRule<MainActivity> activityRule =
            new ActivityTestRule<>(MainActivity.class);

    @Test
    public void testButtonClick() {
        onView(withId(R.id.my_button)).perform(click());
    }
}

  • Explicació

    • @RunWith(AndroidJUnit4.class): Especifica que la prova s'ha d'executar utilitzant el corredor de proves AndroidJUnit4.
    • ActivityTestRule: Llença l'activitat especificada abans de cada prova i la tanca després de cada prova.
    • onView(withId(R.id.my_button)): Troba la vista amb l'ID especificat.
    • perform(click()): Simula una acció de clic a la vista trobada.

Conceptes avançats d'Espresso

  • Matchers

    • Els matchers s'utilitzen per trobar vistes a la interfície d'usuari.
    • Els matchers comuns inclouen:
      • withId(int id): Troba una vista pel seu ID.
      • withText(String text): Troba una vista pel seu text.
      • isDisplayed(): Troba vistes que estan actualment mostrades.

  • Accions de vista

    • Les accions són operacions que es poden realitzar a les vistes.
    • Les accions comunes inclouen:
      • click(): Fa clic a una vista.
      • typeText(String text): Escriu text en una vista.
      • scrollTo(): Desplaça fins a una vista.

  • Afirmacions de vista

    • Les afirmacions s'utilitzen per verificar l'estat de les vistes.
    • Les afirmacions comunes inclouen:
      • matches(Matcher<View> matcher): Comprova si una vista coincideix amb una condició donada.
      • doesNotExist(): Comprova si una vista no existeix.

Exemple: Provar una pantalla de login

  • Classe de prova

    package com.example.myapp;

import androidx.test.ext.junit.runners.AndroidJUnit4;
import androidx.test.rule.ActivityTestRule;

import org.junit.Rule;
import org.junit.Test;
import org.junit.runner.RunWith;

import static androidx.test.espresso.Espresso.onView;
import static androidx.test.espresso.action.ViewActions.click;
import static androidx.test.espresso.action.ViewActions.typeText;
import static androidx.test.espresso.matcher.ViewMatchers.withId;
import static androidx.test.espresso.matcher.ViewMatchers.withText;
import static androidx.test.espresso.assertion.ViewAssertions.matches;

@RunWith(AndroidJUnit4.class)
public class LoginActivityTest {

    @Rule
    public ActivityTestRule<LoginActivity> activityRule =
            new ActivityTestRule<>(LoginActivity.class);

    @Test
    public void testLoginSuccess() {
        onView(withId(R.id.username)).perform(typeText("user"));
        onView(withId(R.id.password)).perform(typeText("password"));
        onView(withId(R.id.login_button)).perform(click());
        onView(withText("Login Successful")).check(matches(isDisplayed()));
    }

    @Test
    public void testLoginFailure() {
        onView(withId(R.id.username)).perform(typeText("user"));
        onView(withId(R.id.password)).perform(typeText("wrongpassword"));
        onView(withId(R.id.login_button)).perform(click());
        onView(withText("Login Failed")).check(matches(isDisplayed()));
    }
}

  • Explicació

    • testLoginSuccess(): Prova un escenari d'inici de sessió amb èxit.
    • testLoginFailure(): Prova un escenari d'inici de sessió fallit.
    • typeText("text"): Escriu text a la vista especificada.
    • check(matches(isDisplayed())): Afirma que una vista està mostrada.

Conclusió

  • Resum

    • Espresso és una eina potent per a les proves d'interfície d'usuari a Android.
    • Proporciona una API senzilla per interactuar amb i verificar l'estat dels components de la interfície d'usuari.
    • Dominant Espresso, pots assegurar-te que la interfície d'usuari de la teva aplicació es comporti com s'espera, proporcionant una millor experiència d'usuari.

  • Pròxims passos
