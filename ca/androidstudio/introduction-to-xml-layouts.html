Visió general

XML (Extensible Markup Language) és un component clau en el desenvolupament d'Android, utilitzat per definir la interfície d'usuari (UI) d'una aplicació. Aquesta secció t'introduirà als dissenys XML a Android Studio, cobrint els conceptes bàsics i avançant gradualment cap a conceptes més avançats.

Conceptes clau

  • Conceptes bàsics d'XML: Comprendre l'estructura i la sintaxi de l'XML.
  • Fitxers de disseny: Com crear i gestionar fitxers de disseny a Android Studio.
  • Elements XML comuns: Visió general dels elements XML més utilitzats en els dissenys d'Android.
  • Atributs: Com utilitzar atributs per personalitzar els components de la UI.
  • Jerarquia de vistes: Comprendre la relació pare-fill en els dissenys XML.
  • Tipus de disseny: Diferents tipus de dissenys disponibles a Android (LinearLayout, RelativeLayout, ConstraintLayout, etc.).

Conceptes bàsics d'XML

XML és un llenguatge de marques similar a HTML però més flexible. S'utilitza per definir l'estructura de les dades.

Exemple d'un disseny XML senzill

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello, World!" />

</LinearLayout>
  • LinearLayout: Un disseny que organitza els seus fills en una única columna o fila.
  • TextView: Un element de la UI que mostra text a l'usuari.

Fitxers de disseny

A Android Studio, els fitxers de disseny es guarden al directori res/layout. Cada fitxer de disseny és un fitxer XML que defineix la UI per a una pantalla específica o part d'una pantalla.

Creació d'un nou fitxer de disseny

  1. Fes clic amb el botó dret sobre el directori res/layout.
  2. Selecciona New > Layout resource file.
  3. Introdueix el nom del fitxer i tria l'element arrel (per exemple, LinearLayout, RelativeLayout).

Elements XML comuns

  • View: La classe base per a tots els components de la UI.
  • ViewGroup: Una vista especial que pot contenir altres vistes (per exemple, LinearLayout, RelativeLayout).
  • TextView: Mostra text.
  • EditText: Un camp de text per a l'entrada de l'usuari.
  • Button: Un botó clicable.

Exemple amb múltiples elements

<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <TextView
        android:id="@+id/textView"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Enter your name:" />

    <EditText
        android:id="@+id/editText"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_below="@id/textView"
        android:hint="Name" />

    <Button
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_below="@id/editText"
        android:text="Submit" />

</RelativeLayout>
  • RelativeLayout: Un disseny on la posició de cada vista fill es defineix en relació amb altres vistes o amb el disseny pare.

Atributs

Els atributs s'utilitzen per personalitzar l'aparença i el comportament dels components de la UI.

Atributs comuns

  • android:layout_width: Especifica l'amplada de la vista.
  • android:layout_height: Especifica l'alçada de la vista.
  • android:text: Defineix el text per a TextView o Button.
  • android:hint: Defineix la pista per a EditText.

Exemple amb atributs

<Button
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Click Me"
    android:background="#FF0000"
    android:textColor="#FFFFFF" />
  • android:background: Defineix el color de fons.
  • android:textColor: Defineix el color del text.

Jerarquia de vistes

En els dissenys XML, les vistes s'organitzen en una estructura jeràrquica. Un ViewGroup pot contenir múltiples vistes fill, i cada vista fill pot tenir els seus propis fills.

Exemple de jerarquia de vistes

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <LinearLayout
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:orientation="horizontal">

        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Label:" />

        <EditText
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:hint="Enter text" />
    </LinearLayout>

    <Button
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Submit" />
</LinearLayout>
  • El LinearLayout extern conté un LinearLayout intern i un Button.
  • El LinearLayout intern conté un TextView i un EditText.

Tipus de disseny

Diferents tipus de dissenys s'utilitzen per organitzar els components de la UI de diverses maneres.

Comparació de tipus de disseny

| Tipus de disseny | Descripció | Exemple d'ús | |------------------|-------------------------------------------------|-------------------------------------------| | LinearLayout | Organitza els fills en una única fila o columna | Formularis senzills, llistes verticals o horitzontals | | RelativeLayout | Posiciona els fills en relació amb altres o amb el pare | UIs complexes amb elements superposats | | ConstraintLayout | Disseny flexible amb restriccions per a la posició | UIs responsives, dissenys complexos | | FrameLayout | Apila els fills un sobre l'altre | Vista única amb superposicions |

Conclusió

Els dissenys XML són un aspecte fonamental del desenvolupament d'Android, proporcionant una manera estructurada de definir la UI de la teva aplicació. Comprenent els conceptes bàsics de la sintaxi XML, els elements comuns, els atributs, la jerarquia de vistes i els diferents tipus de dissenys, pots crear interfícies d'usuari sofisticades i responsives a Android Studio. A mesura que progressis, podràs aprofitar aquests conceptes per construir dissenys més complexos i dinàmics per a les teves aplicacions.
