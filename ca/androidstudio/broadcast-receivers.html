Introducció als Broadcast Receivers

Els Broadcast Receivers són un component fonamental del desenvolupament d'Android. Permeten que la teva aplicació escolti anuncis de difusió a nivell de sistema o intents d'altres aplicacions.

  • Definició: Un Broadcast Receiver és un component d'Android que respon a missatges de difusió d'altres aplicacions o del mateix sistema.
  • Casos d'ús:
    • Respondre a esdeveniments del sistema com bateria baixa, canvis en la connectivitat de xarxa.
    • Comunicar-se entre diferents parts de la teva aplicació.
    • Escoltar esdeveniments personalitzats difosos per altres aplicacions.

Tipus de Difusions

Hi ha dos tipus principals de difusions a Android:

  1. Difusions Normals:

    • Són asíncrones i tots els receptors reben la difusió al mateix temps.
    • Menys intensives en recursos.
    • Exemple: Intent.ACTION_BATTERY_LOW.

  2. Difusions Ordenades:

    • Són síncrones i es lliuren a un receptor a la vegada.
    • Els receptors poden propagar la difusió al següent receptor o aturar la difusió.
    • Exemple: Intent.ACTION_BOOT_COMPLETED.

Registrar Broadcast Receivers

Els Broadcast Receivers es poden registrar de dues maneres:

Registre Estàtic

El registre estàtic es fa al fitxer AndroidManifest.xml. Aquest mètode s'utilitza per a difusions a nivell de sistema que la teva aplicació necessita escoltar fins i tot quan no està en execució.

<receiver android:name=".MyBroadcastReceiver">
    <intent-filter>
        <action android:name="android.intent.action.BOOT_COMPLETED" />
    </intent-filter>
</receiver>

Registre Dinàmic

El registre dinàmic es fa en el codi, típicament dins d'una Activity o Service. Aquest mètode s'utilitza per a difusions que la teva aplicació necessita escoltar només quan està en execució.

@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

    IntentFilter filter = new IntentFilter(Intent.ACTION_BATTERY_LOW);
    MyBroadcastReceiver receiver = new MyBroadcastReceiver();
    registerReceiver(receiver, filter);
}

@Override
protected void onDestroy() {
    super.onDestroy();
    unregisterReceiver(receiver);
}

Implementar un Broadcast Receiver

Per implementar un Broadcast Receiver, necessites estendre la classe BroadcastReceiver i sobreescriure el mètode onReceive.

public class MyBroadcastReceiver extends BroadcastReceiver {
    @Override
    public void onReceive(Context context, Intent intent) {
        String action = intent.getAction();
        if (Intent.ACTION_BATTERY_LOW.equals(action)) {
            // Gestionar l'esdeveniment de bateria baixa
            Toast.makeText(context, "La bateria està baixa!", Toast.LENGTH_SHORT).show();
        }
    }
}

Millors Pràctiques

  • Desregistrar Receptors: Sempre desregistra els receptors dinàmics per evitar fuites de memòria.
  • Minimitzar el Treball a onReceive: El mètode onReceive s'executa en el fil principal, així que evita operacions de llarga durada.
  • Consideracions de Seguretat: Sigues cautelós amb les dades que exposes a través de difusions. Utilitza permisos per restringir qui pot enviar o rebre les teves difusions.

Conceptes Avançats

LocalBroadcastManager

Per a la comunicació dins de la teva aplicació, utilitza LocalBroadcastManager per evitar riscos de seguretat i millorar el rendiment.

LocalBroadcastManager localBroadcastManager = LocalBroadcastManager.getInstance(this);
Intent intent = new Intent("custom-event-name");
localBroadcastManager.sendBroadcast(intent);

BroadcastReceiver amb PendingIntent

Pots utilitzar PendingIntent per activar un BroadcastReceiver en un moment posterior, sovint utilitzat amb AlarmManager.

Intent intent = new Intent(this, MyBroadcastReceiver.class);
PendingIntent pendingIntent = PendingIntent.getBroadcast(this, 0, intent, 0);

AlarmManager alarmManager = (AlarmManager) getSystemService(ALARM_SERVICE);
alarmManager.set(AlarmManager.RTC_WAKEUP, System.currentTimeMillis() + 60000, pendingIntent); // Activar després d'1 minut

Conclusió

Els Broadcast Receivers són una eina poderosa en el desenvolupament d'Android, permetent que les aplicacions responguin a esdeveniments a nivell de sistema i es comuniquin entre elles. Entendre com registrar, implementar i gestionar correctament els Broadcast Receivers és crucial per construir aplicacions d'Android responsives i eficients. Sempre segueix les millors pràctiques per assegurar que la teva aplicació es mantingui segura i funcioni bé.
