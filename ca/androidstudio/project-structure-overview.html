El Projecte Sobre nosaltres Contribuir Donacions Llicència

Entendre l'estructura del projecte a Android Studio és crucial tant per a principiants com per a desenvolupadors avançats. Aquesta secció et guiarà a través dels components essencials d'un projecte Android, explicant els seus rols i com encaixen entre si.

Components Clau d'un Projecte Android

Un projecte Android a Android Studio consta de diversos components clau. Aquí tens un desglossament dels més importants:

  • app: Aquest directori conté tot el codi i recursos específics de l'aplicació.
  • build: Aquest directori conté les sortides de la construcció.
  • gradle: Aquest directori conté els fitxers de configuració per al sistema de construcció Gradle.
  • .idea: Aquest directori conté la configuració específica del projecte per a Android Studio.
  • build.gradle (Nivell de Projecte): Aquest fitxer conté la configuració per a tot el projecte.
  • build.gradle (Nivell de Mòdul): Aquest fitxer conté la configuració per al mòdul específic.
  • settings.gradle: Aquest fitxer especifica els mòduls a incloure en la construcció.

Desglossament Detallat del Directori `app`

El directori app és la part més crítica del teu projecte Android. Conté diversos subdirectoris i fitxers:

  • manifests: Conté el fitxer AndroidManifest.xml, que defineix informació essencial sobre la teva aplicació.
  • java: Conté els fitxers de codi font Java o Kotlin.
  • res: Conté tots els recursos no de codi, com dissenys XML, cadenes de text i imatges.

`AndroidManifest.xml`

El fitxer AndroidManifest.xml és una part crucial del teu projecte. Proporciona informació essencial sobre la teva aplicació al sistema Android, com ara:

  • El nom del paquet de l'aplicació.
  • Els components de l'aplicació (activitats, serveis, receptors de difusió i proveïdors de contingut).
  • Els permisos que requereix l'aplicació.

Exemple:

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.example.myapp">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
        
    </application>

</manifest>

Directori `java`

El directori java conté el teu codi font. Està organitzat per noms de paquet. Per exemple:

com
└── example
    └── myapp
        ├── MainActivity.java
        └── ExampleUnitTest.java

Directori `res`

El directori res conté tots els recursos no de codi. Està dividit en diversos subdirectoris:

  • drawable: Conté recursos dibuixables, com imatges.
  • layout: Conté fitxers XML que defineixen el disseny de la interfície d'usuari.
  • values: Conté fitxers XML per emmagatzemar recursos com cadenes de text, colors i dimensions.

Exemple d'un fitxer de disseny (activity_main.xml):

<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <TextView
        android:id="@+id/textView"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello, World!" />

</RelativeLayout>

Sistema de Construcció Gradle

Gradle és el sistema de construcció utilitzat a Android Studio. Automatitza el procés de construir, provar i desplegar la teva aplicació.

`build.gradle` a nivell de Projecte

Aquest fitxer conté opcions de configuració que s'apliquen a tots els mòduls del teu projecte.

Exemple:

// Fitxer de construcció de nivell superior on pots afegir opcions de configuració comunes a tots els subprojectes/mòduls.
buildscript {
    repositories {
        google()
        mavenCentral()
    }
    dependencies {
        classpath "com.android.tools.build:gradle:7.0.2"
    }
}

allprojects {
    repositories {
        google()
        mavenCentral()
    }
}

`build.gradle` a nivell de Mòdul

Aquest fitxer conté opcions de configuració específiques per al mòdul.

Exemple:

plugins {
    id 'com.android.application'
}

android {
    compileSdk 30

    defaultConfig {
        applicationId "com.example.myapp"
        minSdk 16
        targetSdk 30
        versionCode 1
        versionName "1.0"
    }

    buildTypes {
        release {
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}

dependencies {
    implementation 'androidx.appcompat:appcompat:1.3.1'
    implementation 'com.google.android.material:material:1.4.0'
    implementation 'androidx.constraintlayout:constraintlayout:2.1.0'
    testImplementation 'junit:junit:4.13.2'
    androidTestImplementation 'androidx.test.ext:junit:1.1.3'
    androidTestImplementation 'androidx.test.espresso:espresso-core:3.4.0'
}

Conclusió

Entendre l'estructura del projecte a Android Studio és fonamental per a un desenvolupament eficient. Cada component té un paper específic, des del fitxer AndroidManifest.xml que defineix la informació essencial de l'aplicació fins al directori res que conté tots els recursos no de codi. Familiaritzar-te amb aquests components t'ajudarà a navegar i gestionar el teu projecte de manera més efectiva. A mesura que progressis des de nivells de principiant a avançat, aquest coneixement fonamental serà inestimable per crear aplicacions Android robustes i escalables.
