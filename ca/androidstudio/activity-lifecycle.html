Introducció

El Cicle de Vida de l'Activitat és un concepte fonamental en el desenvolupament d'Android. Entendre com les activitats transiten per diferents estats és crucial per crear aplicacions robustes i eficients. Aquesta secció et guiarà des dels conceptes bàsics fins als avançats del Cicle de Vida de l'Activitat a Android Studio.

Conceptes Clau

  • Activitat: Una cosa única i enfocada que l'usuari pot fer. Gairebé totes les activitats interactuen amb l'usuari.
  • Estats del Cicle de Vida: Diferents estats en què una activitat pot estar, com ara creat, iniciat, reprès, pausat, aturat i destruït.
  • Callbacks del Cicle de Vida: Mètodes que són cridats pel sistema en diferents etapes del cicle de vida de l'activitat.

Estats del Cicle de Vida i Callbacks

Creat

  • onCreate(): Cridat quan l'activitat es crea per primera vegada. Aquí és on hauríeu de fer tota la configuració estàtica normal: crear vistes, vincular dades a llistes, etc.
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    // Inicialitza la teva activitat aquí
}

Iniciat

  • onStart(): Cridat quan l'activitat esdevé visible per a l'usuari.
@Override
protected void onStart() {
    super.onStart();
    // L'activitat està a punt de ser visible
}

Reprès

  • onResume(): Cridat quan l'activitat comença a interactuar amb l'usuari.
@Override
protected void onResume() {
    super.onResume();
    // L'activitat ha esdevingut visible (ara està "reprès")
}

Pausat

  • onPause(): Cridat quan el sistema està a punt de començar a reprendre una altra activitat.
@Override
protected void onPause() {
    super.onPause();
    // Una altra activitat està prenent el focus (aquesta activitat està a punt de ser "pausada")
}

Aturat

  • onStop(): Cridat quan l'activitat ja no és visible per a l'usuari.
@Override
protected void onStop() {
    super.onStop();
    // L'activitat ja no és visible (ara està "aturada")
}

Destruït

  • onDestroy(): Cridat abans que l'activitat sigui destruïda.
@Override
protected void onDestroy() {
    super.onDestroy();
    // L'activitat està a punt de ser destruïda
}

Conceptes Avançats

Gestió de Canvis de Configuració

  • onConfigurationChanged(): Cridat quan la configuració del dispositiu canvia mentre la teva activitat està en execució.
@Override
public void onConfigurationChanged(Configuration newConfig) {
    super.onConfigurationChanged(newConfig);
    // Gestiona els canvis de configuració aquí
}

Guardar i Restaurar Estat

  • onSaveInstanceState(): Cridat per guardar l'estat actual abans que l'activitat sigui destruïda.
@Override
protected void onSaveInstanceState(Bundle outState) {
    super.onSaveInstanceState(outState);
    // Guarda la informació d'estat amb outState.putXXX()
}
  • onRestoreInstanceState(): Cridat per restaurar l'estat després que l'activitat sigui recreada.
@Override
protected void onRestoreInstanceState(Bundle savedInstanceState) {
    super.onRestoreInstanceState(savedInstanceState);
    // Restaura la informació d'estat des de savedInstanceState.getXXX()
}

Comparació de Mètodes del Cicle de Vida

| Mètode | Quan es crida | Propòsit | |-----------------------|------------------------------------------------------|-------------------------------------------------------------------------| | onCreate() | Quan l'activitat es crea per primera vegada | Inicialitzar l'activitat | | onStart() | Quan l'activitat esdevé visible | Preparar l'activitat per esdevenir visible | | onResume() | Quan l'activitat comença a interactuar amb l'usuari | Reprendre qualsevol actualització de la UI, fils o processos pausats | | onPause() | Quan una altra activitat està prenent el focus | Pausar accions en curs que no haurien de continuar mentre l'activitat no està en primer pla | | onStop() | Quan l'activitat ja no és visible | Alliberar recursos que no són necessaris mentre l'activitat no és visible | | onDestroy() | Abans que l'activitat sigui destruïda | Netejar recursos | | onSaveInstanceState() | Abans que l'activitat sigui destruïda per guardar l'estat | Guardar canvis d'estat de la UI en un bundle | | onRestoreInstanceState() | Després que l'activitat sigui recreada per restaurar l'estat | Restaurar l'estat de la UI des del bundle guardat |

Conclusió

Entendre el Cicle de Vida de l'Activitat és essencial per desenvolupar aplicacions d'Android que siguin responsives, eficients i mantenibles. En dominar els callbacks del cicle de vida i saber quan utilitzar-los, pots assegurar-te que la teva aplicació es comporti correctament sota diverses condicions, com ara canvis de configuració, multitasking i gestió de recursos. Practica implementant aquests mètodes en els teus projectes per solidificar la teva comprensió i millorar les teves habilitats com a desenvolupador d'Android.
