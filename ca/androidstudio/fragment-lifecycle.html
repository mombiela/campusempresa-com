Introducció

Els fragments són una part crucial del desenvolupament d'Android, permetent dissenys d'interfície d'usuari més flexibles i modulars. Entendre el cicle de vida d'un fragment és essencial per crear aplicacions d'Android robustes i eficients. Aquest tema et guiarà a través del cicle de vida del fragment des d'un nivell principiant fins a un nivell avançat.

Conceptes clau

  • Fragment: Una porció reutilitzable de la interfície d'usuari de la teva aplicació que es pot gestionar de manera independent.
  • Cicle de vida: La sèrie d'estats pels quals passa un fragment des de la seva creació fins a la seva destrucció.

Estats del cicle de vida del Fragment

El cicle de vida del fragment consta de diversos estats, cadascun representant una etapa diferent en l'existència del fragment. Aquí tens els estats clau:

  1. onAttach(): Es crida quan el fragment s'enganxa per primera vegada al seu context.
  2. onCreate(): Es crida per inicialitzar el fragment.
  3. onCreateView(): Es crida per crear la jerarquia de vistes associada amb el fragment.
  4. onActivityCreated(): Es crida quan el mètode onCreate() de l'activitat ha retornat.
  5. onStart(): Es crida quan el fragment es torna visible.
  6. onResume(): Es crida quan el fragment és visible i actiu.
  7. onPause(): Es crida quan el fragment ja no està interactuant amb l'usuari.
  8. onStop(): Es crida quan el fragment ja no és visible.
  9. onDestroyView(): Es crida per netejar recursos associats amb la vista.
  10. onDestroy(): Es crida per fer la neteja final.
  11. onDetach(): Es crida quan el fragment es desenganxa del seu context.

Mètodes del cicle de vida explicats

onAttach()

@Override
public void onAttach(Context context) {
    super.onAttach(context);
    // Inicialització que requereix un context
}
  • Propòsit: Realitzar la configuració inicial que requereix un context.
  • Exemple d'ús: Accedir a recursos o serveis que necessiten un context.

onCreate()

@Override
public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    // Inicialitzar variables a nivell de fragment
}
  • Propòsit: Inicialitzar el fragment.
  • Exemple d'ús: Configurar components no relacionats amb la interfície d'usuari com ViewModels.

onCreateView()

@Override
public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
    // Inflar el disseny per a aquest fragment
    return inflater.inflate(R.layout.fragment_example, container, false);
}
  • Propòsit: Inflar la interfície d'usuari del fragment.
  • Exemple d'ús: Retornar la vista arrel del disseny del fragment.

onActivityCreated()

@Override
public void onActivityCreated(Bundle savedInstanceState) {
    super.onActivityCreated(savedInstanceState);
    // Realitzar la inicialització final un cop el mètode onCreate() de l'activitat ha completat
}
  • Propòsit: Realitzar la inicialització final.
  • Exemple d'ús: Accedir als elements de la interfície d'usuari de l'activitat.

onStart()

@Override
public void onStart() {
    super.onStart();
    // El fragment ara és visible
}
  • Propòsit: Fer visible el fragment.
  • Exemple d'ús: Iniciar animacions o actualitzar elements de la interfície d'usuari.

onResume()

@Override
public void onResume() {
    super.onResume();
    // El fragment ara és actiu
}
  • Propòsit: Fer actiu el fragment.
  • Exemple d'ús: Reprendre tasques pausades o actualitzar la interfície d'usuari.

onPause()

@Override
public void onPause() {
    super.onPause();
    // El fragment ja no està interactuant amb l'usuari
}
  • Propòsit: Pausar tasques en curs.
  • Exemple d'ús: Pausar animacions o desar dades.

onStop()

@Override
public void onStop() {
    super.onStop();
    // El fragment ja no és visible
}
  • Propòsit: Aturar tasques que no necessiten executar-se quan el fragment no és visible.
  • Exemple d'ús: Aturar fils en segon pla.

onDestroyView()

@Override
public void onDestroyView() {
    super.onDestroyView();
    // Netejar recursos relacionats amb la vista
}
  • Propòsit: Netejar recursos relacionats amb la vista.
  • Exemple d'ús: Nul·lificar referències de vista per evitar fuites de memòria.

onDestroy()

@Override
public void onDestroy() {
    super.onDestroy();
    // Neteja final
}
  • Propòsit: Realitzar la neteja final.
  • Exemple d'ús: Alliberar recursos que no estan lligats a la vista.

onDetach()

@Override
public void onDetach() {
    super.onDetach();
    // El fragment es desenganxa del seu context
}
  • Propòsit: Netejar referències al context.
  • Exemple d'ús: Nul·lificar referències al context per evitar fuites de memòria.

Taula de comparació del cicle de vida

Conclusió

Entendre el cicle de vida del fragment és essencial per crear aplicacions d'Android eficients i responsives. Cada mètode del cicle de vida serveix a un propòsit específic, permetent-te gestionar recursos i components de la interfície d'usuari de manera efectiva. En dominar aquests conceptes, podràs construir aplicacions més modulars i mantenibles.
