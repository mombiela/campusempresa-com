La depuració és una habilitat crucial per a qualsevol desenvolupador, i Android Studio proporciona un conjunt robust d'eines per ajudar-te a identificar i solucionar problemes en el teu codi. Aquesta secció et guiarà a través del procés de depuració a Android Studio, des de nivells principiants fins a avançats.

Introducció a la Depuració

La depuració és el procés d'identificar i eliminar errors del teu codi. A Android Studio, les eines de depuració t'ajuden a inspeccionar el teu codi, monitoritzar els valors de les variables i controlar el flux d'execució per trobar i solucionar errors.

Conceptes Clau

  • Punts de trencament: Marcadors que pots establir en el teu codi per pausar l'execució en una línia específica.
  • Punts de vigilància: Punts de trencament especials que pausen l'execució quan el valor d'una variable canvia.
  • Logcat: Una eina que proporciona missatges del sistema, incloent traços de pila quan l'aplicació llença un error.
  • Depurador: Una eina que et permet executar la teva aplicació pas a pas per inspeccionar el seu comportament.

Configuració de Punts de Trencament

Els punts de trencament són essencials per pausar l'execució de la teva aplicació per inspeccionar l'estat de la teva aplicació en un punt específic.

Com Establir un Punt de Trencament

  1. Obre el teu projecte a Android Studio.
  2. Navega fins a la línia de codi on vols establir el punt de trencament.
  3. Fes clic al marge esquerre al costat del número de línia, o prem Ctrl+F8 (Windows/Linux) o Cmd+F8 (Mac).
public void onClick(View view) {
    int a = 5;
    int b = 10;
    int sum = a + b; // Estableix un punt de trencament aquí
    Log.d("Debugging", "Sum: " + sum);
}

Ús de Logcat

Logcat és una eina poderosa per veure missatges del sistema, incloent registres d'errors i traços de pila.

Filtratge de Sortida de Logcat

  1. Obre Logcat fent clic a View > Tool Windows > Logcat.
  2. Utilitza la barra de cerca per filtrar missatges per etiqueta o paraula clau.
  3. També pots utilitzar els menús desplegables per filtrar per nivell de registre (per exemple, Debug, Info, Warn, Error).
Log.d("MainActivity", "Aquest és un missatge de depuració");
Log.e("MainActivity", "Aquest és un missatge d'error");

Depuració amb el Depurador

El depurador et permet controlar el flux d'execució de la teva aplicació i inspeccionar variables.

Iniciant una Sessió de Depuració

  1. Estableix punts de trencament en el teu codi.
  2. Fes clic al botó Debug (una icona de bug) o prem Shift+F9.
  3. L'aplicació començarà en mode de depuració i es pausarà en el primer punt de trencament.

Inspecció de Variables

  • Quan l'aplicació es pausa en un punt de trencament, pots passar el cursor sobre les variables per veure els seus valors.
  • Utilitza el panell de Variables per veure i modificar els valors de les variables.
public void calculateSum() {
    int x = 3;
    int y = 7;
    int result = x + y; // Inspecciona les variables aquí
    Log.d("Debugging", "Result: " + result);
}

Tècniques Avançades de Depuració

Punts de Trencament Condicionals

Els punts de trencament condicionals et permeten pausar l'execució només quan es compleix una condició específica.

  1. Fes clic dret en un punt de trencament.
  2. Selecciona Breakpoint Properties.
  3. Introdueix una condició (per exemple, x > 5).

Punts de Vigilància

Els punts de vigilància pausen l'execució quan el valor d'una variable canvia.

  1. Fes clic dret en una variable al panell de Variables.
  2. Selecciona Add Watchpoint.

Anàlisi de Traços de Pila

Els traços de pila proporcionen una instantània de la pila de crides en el moment en què es produeix una excepció.

  • Utilitza Logcat per veure els traços de pila.
  • Fes clic en un traç de pila per navegar fins a la línia de codi corresponent.
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    Log.e("MainActivity", "Error: " + e.getMessage());
}

Conclusió

La depuració és una habilitat essencial per a qualsevol desenvolupador d'Android. Dominant les eines i tècniques proporcionades per Android Studio, pots identificar i solucionar problemes en el teu codi de manera eficient. Comença amb punts de trencament bàsics i Logcat, i gradualment explora funcions avançades com els punts de trencament condicionals i els punts de vigilància per millorar les teves capacitats de depuració.
