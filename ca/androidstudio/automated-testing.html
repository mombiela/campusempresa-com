Les proves automàtiques són un aspecte crucial del desenvolupament d'Android, assegurant que la teva aplicació funcioni com s'espera i es mantingui estable al llarg del temps. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de les proves automàtiques a Android Studio.

Introducció a les proves automàtiques

Les proves automàtiques impliquen l'ús d'eines de programari per executar proves a la teva aplicació automàticament. Això ajuda a:

  • Reduir l'esforç de proves manuals.
  • Assegurar una execució consistent de les proves.
  • Identificar ràpidament errors i problemes.

Tipus de proves automàtiques

Hi ha tres tipus principals de proves automàtiques en el desenvolupament d'Android:

  1. Proves unitàries: Prova components o funcions individuals.
  2. Proves d'integració: Prova la interacció entre diferents components.
  3. Proves d'interfície d'usuari (UI): Prova la interfície d'usuari i les interaccions de l'usuari.

Configuració de les proves a Android Studio

Abans d'escriure proves, has de configurar el teu projecte d'Android Studio per a les proves.

Afegir dependències de proves

Afegeix les següents dependències al teu fitxer build.gradle:

dependencies {
    // Dependències de proves unitàries
    testImplementation 'junit:junit:4.13.2'
    testImplementation 'org.mockito:mockito-core:3.9.0'

    // Dependències de proves d'instrumentació d'Android
    androidTestImplementation 'androidx.test.ext:junit:1.1.3'
    androidTestImplementation 'androidx.test.espresso:espresso-core:3.4.0'
}

Escriure proves unitàries

Les proves unitàries es concentren en provar funcions o mètodes individuals.

Exemple: Provar una funció d'utilitat

Considera una simple funció d'utilitat que suma dos números:

// Fitxer: Utils.kt
fun add(a: Int, b: Int): Int {
    return a + b
}

Per provar aquesta funció, crea una classe de proves:

// Fitxer: UtilsTest.kt
import org.junit.Assert.*
import org.junit.Test

class UtilsTest {

    @Test
    fun testAdd() {
        val result = add(2, 3)
        assertEquals(5, result)
    }
}

Executar proves unitàries

Per executar les teves proves unitàries:

  • Fes clic dret a la classe de proves o al mètode.
  • Selecciona "Run 'testAdd'".

Escriure proves d'integració

Les proves d'integració asseguren que diferents parts de la teva aplicació funcionin correctament juntes.

Exemple: Provar un repositori

Considera un repositori que obté dades d'una xarxa:

// Fitxer: UserRepository.kt
class UserRepository(private val apiService: ApiService) {

    fun getUser(userId: String): User {
        return apiService.getUser(userId)
    }
}

Per provar això, pots utilitzar un marc de simulació com Mockito:

// Fitxer: UserRepositoryTest.kt
import org.junit.Assert.*
import org.junit.Test
import org.mockito.Mockito.*

class UserRepositoryTest {

    @Test
    fun testGetUser() {
        val apiService = mock(ApiService::class.java)
        val user = User("1", "John Doe")
        `when`(apiService.getUser("1")).thenReturn(user)

        val userRepository = UserRepository(apiService)
        val result = userRepository.getUser("1")

        assertEquals(user, result)
    }
}

Escriure proves d'interfície d'usuari (UI)

Les proves d'interfície d'usuari interactuen amb la interfície d'usuari per assegurar que es comporti com s'espera.

Exemple: Provar una pantalla de login

Considera una simple pantalla de login amb un camp de nom d'usuari i contrasenya:

<!-- Fitxer: activity_login.xml -->
<LinearLayout ...>
    <EditText
        android:id="@+id/username"
        ... />
    <EditText
        android:id="@+id/password"
        ... />
    <Button
        android:id="@+id/loginButton"
        ... />
</LinearLayout>

Per provar aquesta pantalla, utilitza Espresso:

// Fitxer: LoginActivityTest.kt
import androidx.test.ext.junit.runners.AndroidJUnit4
import androidx.test.rule.ActivityTestRule
import androidx.test.espresso.Espresso.*
import androidx.test.espresso.assertion.ViewAssertions.*
import androidx.test.espresso.matcher.ViewMatchers.*
import org.junit.Rule
import org.junit.Test
import org.junit.runner.RunWith

@RunWith(AndroidJUnit4::class)
class LoginActivityTest {

    @Rule @JvmField
    val activityRule = ActivityTestRule(LoginActivity::class.java)

    @Test
    fun testLoginButton() {
        onView(withId(R.id.username)).perform(typeText("user"))
        onView(withId(R.id.password)).perform(typeText("password"))
        onView(withId(R.id.loginButton)).perform(click())

        // Afegeix afirmacions per verificar el resultat
    }
}

Conclusió

Les proves automàtiques són una pràctica essencial en el desenvolupament d'Android que ajuda a mantenir la qualitat del codi i l'estabilitat de l'aplicació. Escriure proves unitàries, proves d'integració i proves d'interfície d'usuari pot assegurar que la teva aplicació es comporti com s'espera sota diverses condicions. Comença amb proves simples i gradualment passa a escenaris més complexos a mesura que et familiaritzis amb els marcs de proves i les eines disponibles a Android Studio.
