Introducció a JobScheduler

JobScheduler és una API potent introduïda a Android 5.0 (Lollipop) que et permet programar tasques per executar-se en moments específics o sota certes condicions. És particularment útil per a tasques en segon pla que necessiten ser executades periòdicament o quan es compleixen certes condicions, com la disponibilitat de xarxa o la càrrega del dispositiu.

Conceptes Clau

  • JobService: Un component que executa la tasca programada.
  • JobInfo: Conté els criteris per activar la tasca.
  • JobScheduler: El servei del sistema que programa i gestiona les tasques.

Configuració de JobScheduler

Afegir Permisos

Per utilitzar JobScheduler, necessites afegir els permisos adequats al teu fitxer AndroidManifest.xml:

<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED"/>

Crear un JobService

Un JobService és un servei que s'executa en segon pla quan es desencadena la tasca. Aquí tens un exemple d'un JobService senzill:

import android.app.job.JobParameters;
import android.app.job.JobService;
import android.util.Log;

public class MyJobService extends JobService {
    private static final String TAG = "MyJobService";

    @Override
    public boolean onStartJob(JobParameters params) {
        Log.d(TAG, "Job started");
        // Realitza la teva tasca en segon pla aquí
        jobFinished(params, false);
        return true; // Retorna true si la tasca necessita continuar executant-se
    }

    @Override
    public boolean onStopJob(JobParameters params) {
        Log.d(TAG, "Job stopped");
        return false; // Retorna true per reprogramar la tasca
    }
}

Programar una Tasca

Per programar una tasca, necessites crear una instància de JobInfo i passar-la al JobScheduler. Aquí tens un exemple:

import android.app.job.JobInfo;
import android.app.job.JobScheduler;
import android.content.ComponentName;
import android.content.Context;
import android.os.Build;

public class JobSchedulerUtil {
    public static void scheduleJob(Context context) {
        ComponentName componentName = new ComponentName(context, MyJobService.class);
        JobInfo.Builder builder = new JobInfo.Builder(1, componentName)
                .setRequiredNetworkType(JobInfo.NETWORK_TYPE_ANY)
                .setPersisted(true)
                .setPeriodic(15 * 60 * 1000); // 15 minuts

        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            builder.setRequiresBatteryNotLow(true);
        }

        JobScheduler jobScheduler = (JobScheduler) context.getSystemService(Context.JOB_SCHEDULER_SERVICE);
        jobScheduler.schedule(builder.build());
    }
}

Cancel·lar una Tasca

Per cancel·lar una tasca, pots utilitzar el mètode cancel de JobScheduler:

public class JobSchedulerUtil {
    public static void cancelJob(Context context, int jobId) {
        JobScheduler jobScheduler = (JobScheduler) context.getSystemService(Context.JOB_SCHEDULER_SERVICE);
        jobScheduler.cancel(jobId);
    }
}

Funcions Avançades de JobScheduler

Restriccions de la Tasca

JobScheduler et permet establir diverses restriccions per a les teves tasques:

  • Tipus de Xarxa: Requereix qualsevol xarxa o una xarxa no mesurada.
  • Càrrega del Dispositiu: Requereix que el dispositiu estigui carregant.
  • Dispositiu en Repòs: Requereix que el dispositiu estigui en repòs.
  • Bateria No Baixa: Requereix que la bateria no estigui baixa.

Criteris de Reintents

Pots especificar criteris de reintents per controlar com s'ha de tornar a intentar la tasca si falla:

builder.setBackoffCriteria(3000, JobInfo.BACKOFF_POLICY_LINEAR);

Tasques Persistents

Les tasques persistents romanen programades fins i tot després de reiniciar el dispositiu:

builder.setPersisted(true);

Exemple: Sincronització de Dades Periòdica

Aquí tens un exemple complet d'una tasca de sincronització de dades periòdica que s'executa cada 15 minuts:

import android.app.job.JobInfo;
import android.app.job.JobParameters;
import android.app.job.JobScheduler;
import android.app.job.JobService;
import android.content.ComponentName;
import android.content.Context;
import android.os.Build;
import android.util.Log;

public class DataSyncJobService extends JobService {
    private static final String TAG = "DataSyncJobService";

    @Override
    public boolean onStartJob(JobParameters params) {
        Log.d(TAG, "Data sync job started");
        // Realitza la sincronització de dades aquí
        jobFinished(params, false);
        return true;
    }

    @Override
    public boolean onStopJob(JobParameters params) {
        Log.d(TAG, "Data sync job stopped");
        return false;
    }

    public static void scheduleJob(Context context) {
        ComponentName componentName = new ComponentName(context, DataSyncJobService.class);
        JobInfo.Builder builder = new JobInfo.Builder(1, componentName)
                .setRequiredNetworkType(JobInfo.NETWORK_TYPE_ANY)
                .setPersisted(true)
                .setPeriodic(15 * 60 * 1000);

        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            builder.setRequiresBatteryNotLow(true);
        }

        JobScheduler jobScheduler = (JobScheduler) context.getSystemService(Context.JOB_SCHEDULER_SERVICE);
        jobScheduler.schedule(builder.build());
    }
}

Conclusió

JobScheduler és una API robusta per programar tasques en segon pla a Android. Entenent com crear i gestionar tasques, pots assegurar-te que la teva aplicació realitzi les operacions necessàries en segon pla de manera eficient i efectiva. Recorda considerar les restriccions i els criteris de reintents per optimitzar el rendiment i la fiabilitat de les teves tasques programades.
