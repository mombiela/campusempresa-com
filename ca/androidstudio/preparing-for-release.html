El Projecte Sobre nosaltres Contribuir Donacions Llicència

Preparar la teva aplicació Android per a la publicació és un pas crític en el procés de desenvolupament. Això implica diverses tasques per assegurar que la teva aplicació està llesta per a la distribució a Google Play Store o altres plataformes. Aquesta secció et guiarà a través dels passos necessaris des de nivells principiants fins a avançats.

Conceptes Clau

  • Tipus de Build: Diferents configuracions per construir la teva aplicació (per exemple, debug, release).
  • Signatura de l'App: Assegurar que la teva aplicació és segura i identificable.
  • ProGuard: Optimitzar i ofuscar el teu codi.
  • Versionat: Gestionar versions i actualitzacions de l'aplicació.
  • Proves: Assegurar que la teva aplicació està lliure d'errors i funciona bé.
  • Publicació: Pujar la teva aplicació a Google Play Store.

Tipus de Build

Android Studio utilitza tipus de build per definir diferents configuracions per a la teva aplicació. Els dos principals tipus de build són debug i release.

  • Debug Build: Utilitzat durant el desenvolupament. Inclou informació de depuració i no està optimitzat.
  • Release Build: Utilitzat per a la distribució. Està optimitzat i no inclou informació de depuració.

Configurant Tipus de Build

En el teu fitxer build.gradle, pots definir tipus de build de la següent manera:

android {
    ...
    buildTypes {
        release {
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
  • minifyEnabled true: Habilita la reducció de codi, ofuscació i optimització.
  • proguardFiles: Especifica els fitxers de configuració de ProGuard.

Signatura de l'App

Per publicar la teva aplicació, has de signar-la amb un certificat digital. Això assegura la integritat i autenticitat de la teva aplicació.

Generant un APK Signat

  1. Generar un Key Store:

    • Ves a Build > Generate Signed Bundle / APK.
    • Selecciona APK i fes clic a Next.
    • Fes clic a Create new... per generar un nou key store.

  2. Omplir la Informació del Key Store:

    • Ruta del key store
    • Contrasenya
    • Àlies de la clau
    • Contrasenya de la clau
    • Validesa (anys)
    • Detalls del certificat (Nom, Organització, etc.)

  3. Construir l'APK Signat:

    • Selecciona el tipus de build release.
    • Fes clic a Finish.

ProGuard

ProGuard és una eina que ajuda a optimitzar i ofuscar el teu codi, fent-lo més difícil de descompilar.

Configurant ProGuard

En el teu fitxer proguard-rules.pro, pots afegir regles per mantenir certes classes o mètodes sense ofuscar:

# Manté totes les classes en el paquet especificat
-keep class com.example.myapp.** { *; }

# Manté tots els mètodes públics en una classe
-keepclassmembers class * {
    public *;
}

Versionat

Un versionat adequat és crucial per gestionar actualitzacions i assegurar que els usuaris tenen les últimes característiques i correccions d'errors.

Configurant la Informació de la Versió

En el teu fitxer build.gradle, estableix el codi de versió i el nom de la versió:

android {
    ...
    defaultConfig {
        ...
        versionCode 1
        versionName "1.0"
    }
}
  • versionCode: Un valor enter que representa la versió de l'aplicació. Incrementa aquest valor amb cada publicació.
  • versionName: Un valor de cadena que representa la versió de l'aplicació.

Proves

Abans de publicar la teva aplicació, és essencial fer proves exhaustives per assegurar que està lliure d'errors i funciona bé.

Tipus de Proves

  • Proves Unitàries: Prova components individuals.
  • Proves d'Integració: Prova la interacció entre components.
  • Proves d'Interfície d'Usuari: Prova la interfície d'usuari.

Executant Proves

Pots executar proves des d'Android Studio:

  • Proves Unitàries: Fes clic dret sobre la classe o mètode de prova i selecciona Run.
  • Proves d'Interfície d'Usuari: Utilitza eines com Espresso per automatitzar les proves d'interfície d'usuari.

Exemple d'una prova unitària simple:

public class ExampleUnitTest {
    @Test
    public void addition_isCorrect() {
        assertEquals(4, 2 + 2);
    }
}

Publicació

Un cop la teva aplicació està llesta, pots publicar-la a Google Play Store.

Passos per Publicar

  1. Preparar la Llista de la Botiga:

    • Títol
    • Descripció
    • Captures de pantalla
    • Icona

  2. Pujar l'APK:

    • Ves a Google Play Console.
    • Crea una nova aplicació.
    • Puja l'APK signat.

  3. Establir Preus i Distribució:

    • Tria els països on la teva aplicació estarà disponible.
    • Estableix el preu (si és aplicable).

  4. Enviar per Revisió:

    • Un cop tot està configurat, envia la teva aplicació per revisió.

Conclusió

Preparar la teva aplicació Android per a la publicació implica diversos passos importants, des de configurar tipus de build i signar la teva aplicació fins a optimitzar amb ProGuard, gestionar versions, fer proves exhaustives i finalment publicar. Seguint aquests passos, pots assegurar que la teva aplicació està llesta per a la distribució i proporciona una gran experiència d'usuari.
