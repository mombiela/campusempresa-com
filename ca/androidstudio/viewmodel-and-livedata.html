Introducció

En aquest mòdul, explorarem els conceptes de ViewModel i LiveData en el desenvolupament d'Android. Aquests components formen part dels components d'arquitectura d'Android, dissenyats per ajudar-te a construir aplicacions robustes, mantenibles i testejables.

Què és ViewModel?

Un ViewModel és una classe que és responsable de preparar i gestionar les dades per a una Activitat o un Fragment. També gestiona la comunicació de l'Activitat/Fragment amb la resta de l'aplicació (per exemple, cridant classes de lògica de negoci).

Conceptes Clau

  • Consciència del Cicle de Vida: Els objectes ViewModel estan dissenyats per sobreviure als canvis de configuració com les rotacions de pantalla.
  • Separació de Responsabilitats: ViewModel ajuda a separar les dades de la interfície d'usuari dels controladors de la interfície d'usuari com Activitats i Fragments.

Exemple

class MyViewModel : ViewModel() {
    private val _data = MutableLiveData<String>()
    val data: LiveData<String> get() = _data

    fun loadData() {
        // Simular la càrrega de dades des d'un repositori
        _data.value = "Hola, ViewModel!"
    }
}

Què és LiveData?

LiveData és una classe contenidora de dades observable. A diferència d'un observable regular, LiveData és conscient del cicle de vida, el que significa que respecta el cicle de vida d'altres components de l'aplicació, com Activitats, Fragments o Serveis.

Conceptes Clau

  • Consciència del Cicle de Vida: LiveData només actualitza els observadors dels components de l'aplicació que estan en un estat de cicle de vida actiu.
  • Neteja Automàtica: LiveData elimina automàticament els observadors quan el seu cicle de vida associat és destruït.

Exemple

class MyActivity : AppCompatActivity() {
    private lateinit var viewModel: MyViewModel

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        viewModel = ViewModelProvider(this).get(MyViewModel::class.java)

        viewModel.data.observe(this, Observer { data ->
            // Actualitzar la interfície d'usuari
            findViewById<TextView>(R.id.textView).text = data
        })

        viewModel.loadData()
    }
}

Combinació de ViewModel i LiveData

La combinació de ViewModel i LiveData et permet crear una arquitectura robusta per a les teves aplicacions Android. ViewModel manté i gestiona les dades relacionades amb la interfície d'usuari, mentre que LiveData proporciona observació de dades conscient del cicle de vida.

Exemple

class MyViewModel : ViewModel() {
    private val _data = MutableLiveData<String>()
    val data: LiveData<String> get() = _data

    fun loadData() {
        // Simular la càrrega de dades des d'un repositori
        _data.value = "Hola, ViewModel i LiveData!"
    }
}

class MyActivity : AppCompatActivity() {
    private lateinit var viewModel: MyViewModel

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        viewModel = ViewModelProvider(this).get(MyViewModel::class.java)

        viewModel.data.observe(this, Observer { data ->
            // Actualitzar la interfície d'usuari
            findViewById<TextView>(R.id.textView).text = data
        })

        viewModel.loadData()
    }
}

Temes Avançats

Transformacions

LiveData proporciona mètodes de transformació com map i switchMap per manipular les dades.

val transformedData: LiveData<String> = Transformations.map(viewModel.data) { data ->
    "Transformat: $data"
}

MediatorLiveData

MediatorLiveData pot observar altres objectes LiveData i reaccionar als seus canvis.

val mediatorLiveData = MediatorLiveData<String>()
mediatorLiveData.addSource(viewModel.data) { data ->
    mediatorLiveData.value = "Mediador: $data"
}

Conclusió

ViewModel i LiveData són eines poderoses en el desenvolupament d'Android que ajuden a gestionar les dades relacionades amb la interfície d'usuari de manera conscient del cicle de vida. Utilitzant aquests components, pots crear aplicacions més robustes, mantenibles i testejables. Entendre i implementar aquests components de manera efectiva millorarà significativament les teves habilitats de desenvolupament d'Android.
