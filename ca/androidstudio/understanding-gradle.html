Introducció a Gradle

Gradle és una eina poderosa d'automatització de compilació utilitzada principalment per a projectes Java, i és el sistema de compilació predeterminat per a Android Studio. Comprendre Gradle és essencial per gestionar dependències, compilar, provar i desplegar aplicacions Android de manera eficient.

Conceptes Clau

  • Automatització de Compilació: Automatitza el procés de compilar codi, empaquetar binaris, executar proves i desplegar aplicacions.
  • Dependències: Llibreries o mòduls externs que el teu projecte necessita per funcionar.
  • Scripts de Compilació: Escrits en Groovy o Kotlin, aquests scripts defineixen com s'ha d'executar el procés de compilació.

Scripts de Compilació de Gradle

Els scripts de compilació de Gradle es solen escriure en Groovy o Kotlin DSL (Domain Specific Language). Aquests scripts s'utilitzen per configurar el procés de compilació.

Estructura Bàsica d'un Script de Gradle

Un fitxer build.gradle típic per a un projecte Android inclou:

// Fitxer de compilació de nivell superior on pots afegir opcions de configuració comunes a tots els sub-projectes/mòduls.
buildscript {
    ext.kotlin_version = '1.5.10'
    repositories {
        google()
        mavenCentral()
    }
    dependencies {
        classpath 'com.android.tools.build:gradle:4.2.2'
        classpath "org.jetbrains.kotlin:kotlin-gradle-plugin:$kotlin_version"
    }
}

allprojects {
    repositories {
        google()
        mavenCentral()
    }
}

task clean(type: Delete) {
    delete rootProject.buildDir
}

Explicació

  • buildscript: Configura els repositoris i les dependències per al mateix script de compilació.
  • ext: Defineix propietats extres, com la versió de Kotlin.
  • repositories: Especifica on trobar les dependències.
  • dependencies: Llista les dependències necessàries per a l'script de compilació.
  • allprojects: Aplica configuracions a tots els sub-projectes/mòduls.
  • task clean: Defineix una tasca personalitzada per eliminar el directori de compilació.

Gestió de Dependències

Les dependències són crucials per a qualsevol projecte. Gradle facilita la seva gestió.

Afegir Dependències

Les dependències s'afegeixen al bloc dependencies del teu fitxer build.gradle:

dependencies {
    implementation 'com.android.support:appcompat-v7:28.0.0'
    implementation 'com.android.support.constraint:constraint-layout:1.1.3'
    testImplementation 'junit:junit:4.12'
}

Explicació

  • implementation: Afegeix una dependència al conjunt de fonts principal.
  • testImplementation: Afegeix una dependència al conjunt de fonts de prova.

Tasques de Gradle

Les tasques de Gradle són els blocs de construcció del procés de compilació. Cada tasca representa una peça de treball que Gradle pot executar.

Tasques Comunes de Gradle

  • assemble: Compila i empaqueta l'aplicació.
  • clean: Elimina el directori de compilació.
  • build: Compila i prova el projecte.

Tasques Personalitzades

Pots definir tasques personalitzades al teu fitxer build.gradle:

task hello {
    doLast {
        println 'Hello, Gradle!'
    }
}

Explicació

  • task hello: Defineix una nova tasca anomenada hello.
  • doLast: Especifica l'acció a realitzar quan s'executa la tasca.

Funcions Avançades de Gradle

A mesura que et familiaritzis més amb Gradle, podràs aprofitar les seves funcions avançades per optimitzar el teu procés de compilació.

Variants de Compilació

Les variants de compilació et permeten crear diferents versions de la teva aplicació des del mateix projecte.

android {
    ...
    buildTypes {
        release {
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
        debug {
            applicationIdSuffix ".debug"
            debuggable true
        }
    }
}

Explicació

  • buildTypes: Defineix diferents configuracions de compilació.
  • release: Configuració per a la compilació de llançament.
  • debug: Configuració per a la compilació de depuració.

Plugins de Gradle

Els plugins de Gradle amplien la funcionalitat del teu script de compilació.

plugins {
    id 'com.android.application'
    id 'kotlin-android'
}

Explicació

  • plugins: Bloc per aplicar plugins de Gradle.
  • id: Especifica el plugin a aplicar.

Conclusió

Comprendre Gradle és fonamental per al desenvolupament d'Android. Et permet automatitzar el procés de compilació, gestionar dependències i personalitzar les teves configuracions de compilació. En dominar Gradle, pots optimitzar el teu flux de treball de desenvolupament i centrar-te més en escriure bon codi.

Resum

  • Gradle és el sistema de compilació predeterminat per a Android Studio.
  • Els scripts de compilació es redacten en Groovy o Kotlin DSL.
  • Les dependències es gestionen al bloc dependencies.
  • Les tasques de Gradle automatitzen diversos processos de compilació.
  • Les funcions avançades com les variants de compilació i els plugins milloren el procés de compilació.

Seguint aquesta guia, hauríes de tenir una base sòlida en Gradle i estar ben preparat per manejar escenaris de compilació més complexos en els teus projectes Android.
