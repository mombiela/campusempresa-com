Introducció als Microserveis

  • Definició: L'arquitectura de microserveis és un mètode de desenvolupament de sistemes de programari que intenta centrar-se en la construcció de mòduls de funció única amb interfícies i operacions ben definides.
  • Beneficis:
    • Escalabilitat
    • Flexibilitat
    • Resiliència
    • Desplegament independent

Configuració de Spring Boot per a Microserveis

  • Visió general de Spring Boot: Spring Boot simplifica el desenvolupament de noves aplicacions Spring proporcionant valors per defecte per a la configuració del codi i les anotacions.
  • Creació d'un projecte Spring Boot:
    • Utilitza Spring Initializr (https://start.spring.io/) per generar un nou projecte Spring Boot.
    • Inclou dependències com Spring Web, Spring Data JPA i H2 Database.
@SpringBootApplication
public class MicroserviceApplication {
    public static void main(String[] args) {
        SpringApplication.run(MicroserviceApplication.class, args);
    }
}

Construcció d'un Microservei Simple

  • Creació d'un REST Controller:
    • Defineix un endpoint REST simple per gestionar les sol·licituds HTTP.
@RestController
@RequestMapping("/api")
public class SimpleController {
    
    @GetMapping("/hello")
    public String sayHello() {
        return "Hola, Món!";
    }
}
  • Execució de l'Aplicació:
    • Utilitza mvn spring-boot:run per iniciar l'aplicació.
    • Accedeix a l'endpoint a http://localhost:8080/api/hello.

Comunicació entre Serveis

  • REST Template: Un client sincrònic per realitzar sol·licituds HTTP.
  • Feign Client: Un client de serveis web declaratiu que facilita l'escriptura de clients de serveis web.

Ús de RestTemplate

@RestController
@RequestMapping("/api")
public class CommunicationController {

    @Autowired
    private RestTemplate restTemplate;

    @GetMapping("/fetch-data")
    public String fetchData() {
        String response = restTemplate.getForObject("http://other-service/api/data", String.class);
        return response;
    }
}

Ús de Feign Client

@FeignClient(name = "other-service")
public interface OtherServiceClient {

    @GetMapping("/api/data")
    String getData();
}

@RestController
@RequestMapping("/api")
public class CommunicationController {

    @Autowired
    private OtherServiceClient otherServiceClient;

    @GetMapping("/fetch-data")
    public String fetchData() {
        return otherServiceClient.getData();
    }
}

Descobriment de Serveis amb Eureka

  • Eureka Server: Actua com a registre de serveis.
  • Eureka Client: Es registra amb el servidor Eureka.

Configuració del Servidor Eureka

@SpringBootApplication
@EnableEurekaServer
public class EurekaServerApplication {
    public static void main(String[] args) {
        SpringApplication.run(EurekaServerApplication.class, args);
    }
}

Configuració del Client Eureka

# application.yml
eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka/

API Gateway amb Spring Cloud Gateway

  • Propòsit: Actua com a punt d'entrada únic per a tots els microserveis.
  • Configuració:
    • Defineix rutes per a diferents microserveis.
# application.yml
spring:
  cloud:
    gateway:
      routes:
        - id: simple-service
          uri: http://localhost:8081
          predicates:
            - Path=/api/simple/**

Circuit Breaker amb Resilience4j

  • Propòsit: Gestionar les fallades de manera elegant i proporcionar mecanismes de fallback.
  • Implementació:
    • Afegeix la dependència de Resilience4j.
    • Anota els mètodes amb @CircuitBreaker.
@RestController
@RequestMapping("/api")
public class CircuitBreakerController {

    @Autowired
    private RestTemplate restTemplate;

    @GetMapping("/fetch-data")
    @CircuitBreaker(name = "otherService", fallbackMethod = "fallback")
    public String fetchData() {
        return restTemplate.getForObject("http://other-service/api/data", String.class);
    }

    public String fallback(Throwable t) {
        return "Resposta de fallback";
    }
}

Seguretat en Microserveis

  • OAuth2 i JWT: Utilitza OAuth2 per a l'autenticació i JWT per a la seguretat basada en tokens.
  • Spring Security: Integra Spring Security per assegurar els endpoints.
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
            .antMatchers("/api/public/**").permitAll()
            .anyRequest().authenticated()
            .and()
            .oauth2Login();
    }
}
