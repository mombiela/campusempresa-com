Introducció

La gestió d'excepcions és un aspecte crític de qualsevol aplicació, i Spring Boot proporciona diverses maneres de gestionar les excepcions de manera neta i eficient. Aquesta secció cobrirà els conceptes bàsics de la gestió d'excepcions a Spring Boot, progressant cap a tècniques més avançades.

Conceptes Clau

  • Conceptes Bàsics de la Gestió d'Excepcions: Entendre què són les excepcions i per què cal gestionar-les.
  • Gestió d'Excepcions a Spring Boot: Utilitzar els mecanismes integrats de Spring Boot per gestionar excepcions.
  • Gestió d'Excepcions Personalitzades: Crear excepcions personalitzades i gestionar-les.
  • Gestió Global d'Excepcions: Gestionar excepcions globalment a tota l'aplicació.
  • Millors Pràctiques: Seguir les millors pràctiques per a una gestió d'excepcions efectiva.

Conceptes Bàsics de la Gestió d'Excepcions

Abans d'endinsar-nos en les tècniques específiques de Spring Boot, revisem els conceptes bàsics de la gestió d'excepcions a Java.

Exemple: Gestió Bàsica d'Excepcions a Java

public class BasicExceptionHandling {
    public static void main(String[] args) {
        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("No es pot dividir per zero!");
        }
    }
}

En aquest exemple, es captura una ArithmeticException i es gestiona imprimint un missatge a la consola.

Gestió d'Excepcions a Spring Boot

Spring Boot proporciona diverses anotacions i classes per gestionar excepcions de manera efectiva.

Ús de `@ExceptionHandler`

L'anotació @ExceptionHandler s'utilitza per definir un mètode que gestionarà una excepció específica.

@RestController
public class MyController {

    @GetMapping("/example")
    public String example() {
        if (true) {
            throw new RuntimeException("Exemple d'excepció");
        }
        return "Hola, món!";
    }

    @ExceptionHandler(RuntimeException.class)
    public ResponseEntity<String> handleRuntimeException(RuntimeException ex) {
        return new ResponseEntity<>("RuntimeException gestionada: " + ex.getMessage(), HttpStatus.INTERNAL_SERVER_ERROR);
    }
}

En aquest exemple, qualsevol RuntimeException llançada dins de MyController serà capturada pel mètode handleRuntimeException.

Ús de `@ControllerAdvice`

@ControllerAdvice és una característica més avançada que permet gestionar excepcions globalment a tots els controladors.

@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(RuntimeException.class)
    public ResponseEntity<String> handleRuntimeException(RuntimeException ex) {
        return new ResponseEntity<>("Gestor Global: " + ex.getMessage(), HttpStatus.INTERNAL_SERVER_ERROR);
    }
}

Aquest exemple demostra com gestionar excepcions globalment utilitzant @ControllerAdvice.

Gestió d'Excepcions Personalitzades

Crear excepcions personalitzades pot fer que el teu codi sigui més llegible i mantenible.

Exemple: Excepció Personalitzada

public class CustomException extends RuntimeException {
    public CustomException(String message) {
        super(message);
    }
}

Exemple: Gestió d'Excepció Personalitzada

@RestController
public class MyController {

    @GetMapping("/custom")
    public String customExample() {
        if (true) {
            throw new CustomException("Aquesta és una excepció personalitzada");
        }
        return "Hola, món!";
    }

    @ExceptionHandler(CustomException.class)
    public ResponseEntity<String> handleCustomException(CustomException ex) {
        return new ResponseEntity<>("CustomException gestionada: " + ex.getMessage(), HttpStatus.BAD_REQUEST);
    }
}

En aquest exemple, es crea una excepció personalitzada CustomException i es gestiona específicament.

Gestió Global d'Excepcions

Utilitzar @ControllerAdvice per gestionar excepcions globalment pot simplificar la teva lògica de gestió d'excepcions.

Exemple: Gestor Global d'Excepcions

@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(Exception.class)
    public ResponseEntity<String> handleAllExceptions(Exception ex) {
        return new ResponseEntity<>("Gestor Global d'Excepcions: " + ex.getMessage(), HttpStatus.INTERNAL_SERVER_ERROR);
    }

    @ExceptionHandler(CustomException.class)
    public ResponseEntity<String> handleCustomException(CustomException ex) {
        return new ResponseEntity<>("Gestor Global de CustomException: " + ex.getMessage(), HttpStatus.BAD_REQUEST);
    }
}

Aquest exemple mostra com gestionar totes les excepcions i les excepcions personalitzades globalment.

Millors Pràctiques

  • Utilitza Excepcions Específiques: Prefereix sempre excepcions específiques en lloc de genèriques.
  • Centralitza la Gestió d'Excepcions: Utilitza @ControllerAdvice per gestionar excepcions globalment.
  • Proporciona Missatges Significatius: Assegura't que els missatges d'excepció siguin clars i informatius.
  • Registra les Excepcions: Registra sempre les excepcions per a finalitats de depuració.
  • Evita Empassar Excepcions: No capturis excepcions sense gestionar-les adequadament.

Conclusió

La gestió d'excepcions a Spring Boot és una característica poderosa que et permet gestionar errors de manera elegant. Utilitzant anotacions com @ExceptionHandler i @ControllerAdvice, pots gestionar excepcions tant localment com globalment. Les excepcions personalitzades milloren encara més la llegibilitat i mantenibilitat del teu codi. Seguir les millors pràctiques assegura que la teva aplicació es mantingui robusta i fàcil d'usar.
