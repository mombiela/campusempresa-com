Introducció

Spring Boot és un potent framework que simplifica el desenvolupament d'aplicacions Java. No obstant això, hi ha errors comuns que els desenvolupadors, tant principiants com avançats, sovint troben. Aquesta secció cobrirà aquests errors i proporcionarà estratègies per evitar-los.

Mala configuració

Propietats d'aplicació incorrectes

  • Error: Configurar malament els fitxers application.properties o application.yml pot portar a comportaments inesperats.
  • Solució: Revisa sempre els noms i valors de les propietats. Utilitza la validació de propietats de configuració de Spring Boot.
# application.properties
server.port=8080
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=secret

Dependències mancants

  • Error: Oblidar incloure les dependències necessàries en pom.xml o build.gradle.
  • Solució: Utilitza Spring Initializr per generar un projecte amb les dependències correctes.
<!-- pom.xml -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

Problemes de rendiment

Consultes de base de dades ineficients

  • Error: Escriure consultes ineficients que condueixen a colls d'ampolla de rendiment.
  • Solució: Utilitza Spring Data JPA i escriu consultes JPQL o Criteria eficients. Utilitza la paginació i l'indexació adequada.
// Exemple d'una consulta paginada
Page<User> findByLastName(String lastName, Pageable pageable);

Operacions de I/O bloquejants

  • Error: Realitzar operacions de I/O bloquejants en el fil principal.
  • Solució: Utilitza el processament asíncron amb @Async i CompletableFuture.
@Async
public CompletableFuture<String> asyncMethod() {
    // Realitzar operació de I/O
    return CompletableFuture.completedFuture("Resultat");
}

Errors de seguretat

Exposar dades sensibles

  • Error: Exposar dades sensibles a través d'endpoints.
  • Solució: Utilitza Spring Security per assegurar els endpoints i gestionar adequadament les dades sensibles.
// Assegurant un endpoint
@GetMapping("/secure-data")
@PreAuthorize("hasRole('ROLE_USER')")
public String getSecureData() {
    return "Dades Sensibles";
}

Credencials codificades

  • Error: Codificar credencials en el codi font.
  • Solució: Utilitza variables d'entorn o l'anotació @Value de Spring Boot per injectar credencials.
@Value("${db.password}")
private String dbPassword;

Reptes de proves

Saltar proves

  • Error: Saltar proves per estalviar temps.
  • Solució: Escriu proves unitàries i d'integració completes utilitzant el suport de proves de Spring Boot.
@RunWith(SpringRunner.class)
@SpringBootTest
public class MyServiceTests {

    @Autowired
    private MyService myService;

    @Test
    public void testServiceMethod() {
        assertEquals("Resultat Esperat", myService.method());
    }
}

Configuració de proves incorrecta

  • Error: Configurar malament els entorns de prova, la qual cosa condueix a falsos positius/negatius.
  • Solució: Utilitza @TestConfiguration i perfils per aïllar les configuracions de prova.
@TestConfiguration
public class TestConfig {

    @Bean
    public MyService myService() {
        return new MyService();
    }
}

Gestió de dependències

Conflictes de versions

  • Error: Conflictes de versions de dependències que causen errors en temps d'execució.
  • Solució: Utilitza eines de gestió de dependències com dependencyManagement de Maven o resolutionStrategy de Gradle.
<!-- Gestió de dependències de Maven -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>2.5.4</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

Conclusió

Entendre i evitar els errors comuns en el desenvolupament de Spring Boot pot estalviar temps i evitar maldecaps. Seguint les millors pràctiques per a la configuració, rendiment, seguretat, proves i gestió de dependències, pots construir aplicacions Spring Boot robustes i eficients. Mantingues-te sempre actualitzat amb la documentació més recent de Spring Boot i les millors pràctiques de la comunitat per continuar millorant les teves habilitats.
