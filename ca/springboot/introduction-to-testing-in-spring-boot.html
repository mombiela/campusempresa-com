Visió General

Les proves són una part crucial del desenvolupament de programari, assegurant que la teva aplicació es comporti com s'espera. Spring Boot proporciona un suport complet per a les proves, facilitant l'escriptura de proves unitàries, proves d'integració i proves de punta a punta.

Conceptes Clau

  • Proves Unitàries: Provar components individuals en aïllament.
  • Proves d'Integració: Provar múltiples components junts per assegurar-se que funcionen com s'espera.
  • Proves de Punta a Punta: Provar tot el flux de l'aplicació de principi a fi.
  • Annotations de Prova de Spring Boot: Anotacions especials proporcionades per Spring Boot per facilitar les proves.

Configuració de les Dependències de Prova

Per començar a provar en Spring Boot, necessites incloure les dependències necessàries en el teu pom.xml (per a Maven) o build.gradle (per a Gradle).

Maven

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>

Gradle

testImplementation 'org.springframework.boot:spring-boot-starter-test'

Escriure Proves Unitàries

Les proves unitàries se centren en provar components individuals en aïllament. Spring Boot proporciona l'anotació @SpringBootTest per facilitar això.

Exemple: Provar una Capa de Servei

Creem un servei senzill i escrivim una prova unitària per a ell.

Classe de Servei

@Service
public class CalculatorService {
    public int add(int a, int b) {
        return a + b;
    }
}

Prova Unitària

@RunWith(SpringRunner.class)
@SpringBootTest
public class CalculatorServiceTest {

    @Autowired
    private CalculatorService calculatorService;

    @Test
    public void testAdd() {
        int result = calculatorService.add(2, 3);
        assertEquals(5, result);
    }
}
  • Explicació:
    • @RunWith(SpringRunner.class): Integra el Spring TestContext Framework amb JUnit.
    • @SpringBootTest: Indica que el context sota prova és una aplicació Spring Boot.
    • @Autowired: Injecta el bean CalculatorService.
    • @Test: Marca el mètode com a mètode de prova.

Escriure Proves d'Integració

Les proves d'integració asseguren que diferents parts de l'aplicació funcionin juntes com s'espera.

Exemple: Provar un Controlador REST

Creem un controlador REST senzill i escrivim una prova d'integració per a ell.

Controlador REST

@RestController
@RequestMapping("/api")
public class CalculatorController {

    @Autowired
    private CalculatorService calculatorService;

    @GetMapping("/add")
    public ResponseEntity<Integer> add(@RequestParam int a, @RequestParam int b) {
        int result = calculatorService.add(a, b);
        return new ResponseEntity<>(result, HttpStatus.OK);
    }
}

Prova d'Integració

@RunWith(SpringRunner.class)
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
public class CalculatorControllerTest {

    @Autowired
    private TestRestTemplate restTemplate;

    @Test
    public void testAdd() {
        ResponseEntity<Integer> response = restTemplate.getForEntity("/api/add?a=2&b=3", Integer.class);
        assertEquals(HttpStatus.OK, response.getStatusCode());
        assertEquals(5, response.getBody().intValue());
    }
}
  • Explicació:
    • @SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT): Inicia el servidor en un port aleatori.
    • TestRestTemplate: Una alternativa convenient a RestTemplate que és útil per a proves d'integració.
    • getForEntity: Envia una sol·licitud GET i retorna la resposta com a ResponseEntity.

Conclusió

Les proves en Spring Boot es simplifiquen amb l'ajuda de diverses anotacions i utilitats. Entenent i utilitzant proves unitàries i proves d'integració, pots assegurar-te que els components de la teva aplicació funcionin correctament tant en aïllament com junts. Aquest coneixement fonamental et prepara per a escenaris de prova més avançats, incloent proves de punta a punta i proves de rendiment.
