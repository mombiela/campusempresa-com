Introducció a la Injecció de Dependències (DI)

La Injecció de Dependències (DI) és un patró de disseny utilitzat per implementar IoC, permetent la creació d'objectes dependents fora d'una classe i proporcionant aquests objectes a una classe de diverses maneres.

  • Definició: La Injecció de Dependències és una tècnica on un objecte rep altres objectes dels quals depèn.
  • Propòsit: Desacoblar la creació d'una dependència de la classe que la utilitza, fent el codi més modular i testable.

Tipus de Injecció de Dependències

  1. Injecció de Constructor: Les dependències es proporcionen a través d'un constructor de classe.
  2. Injecció de Setter: Les dependències es proporcionen a través de mètodes setter.
  3. Injecció de Camp: Les dependències s'injecten directament en els camps.

Exemple: Injecció de Constructor

public class Service {
    private final Repository repository;

    // Injecció de Constructor
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Inversió de Control (IoC)

La Inversió de Control és un principi en l'enginyeria de programari on el control dels objectes o parts d'un programa es transfereix a un contenidor o marc.

  • Definició: IoC és un principi de disseny en el qual el flux de control d'un programa s'inverteix en comparació amb la programació tradicional.
  • Propòsit: Augmentar la modularitat i fer el codi més fàcil de gestionar i provar.

IoC en Spring Boot

Spring Boot utilitza el principi IoC per gestionar el cicle de vida i les dependències dels beans.

  • Bean: Un objecte que és instanciat, assemblat i gestionat per un contenidor IoC de Spring.
  • Contenidor IoC: El nucli del marc Spring, responsable d'instanciar, configurar i assemblar beans.

Exemple: Contenidor IoC en Spring Boot

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ApplicationContext;

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(Application.class, args);
        Service service = context.getBean(Service.class);
        service.performAction();
    }
}

Combinació de DI i IoC en Spring Boot

Spring Boot aprofita tant DI com IoC per gestionar les dependències i el cicle de vida dels beans.

Exemple: Implementació Completa

  1. Definir una Interfície de Repository i Implementació:

    public interface Repository {
    void action();
}

@Service
public class RepositoryImpl implements Repository {
    @Override
    public void action() {
        System.out.println("Acció realitzada!");
    }
}

  2. Definir una Classe de Servei amb Injecció de Constructor:

    @Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

  3. Executar l'Aplicació:

    @SpringBootApplication
public class Application {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(Application.class, args);
        Service service = context.getBean(Service.class);
        service.performAction();
    }
}

Temes Avançats

Qualificadors i Beans Primaris

Quan existeixen múltiples beans del mateix tipus, pots utilitzar @Qualifier o @Primary per especificar quin bean s'ha d'injectar.

  • @Qualifier: Utilitzat per especificar quin bean s'ha d'injectar quan existeixen múltiples beans del mateix tipus.
  • @Primary: Utilitzat per designar un bean com a candidat principal per a la injecció.

Exemple: Ús de @Qualifier

@Service
@Qualifier("repository1")
public class RepositoryImpl1 implements Repository {
    @Override
    public void action() {
        System.out.println("Acció del Repository1 realitzada!");
    }
}

@Service
@Qualifier("repository2")
public class RepositoryImpl2 implements Repository {
    @Override
    public void action() {
        System.out.println("Acció del Repository2 realitzada!");
    }
}

@Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(@Qualifier("repository1") Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Exemple: Ús de @Primary

@Service
@Primary
public class PrimaryRepositoryImpl implements Repository {
    @Override
    public void action() {
        System.out.println("Acció del repositori principal realitzada!");
    }
}

@Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Conclusió

La Injecció de Dependències i la Inversió de Control són conceptes fonamentals en Spring Boot que ajuden a crear codi desacoblat, modular i testable. Entenent i aplicant aquests principis, pots aprofitar tot el potencial del marc Spring per construir aplicacions robustes i mantenibles.
