El Projecte Sobre nosaltres Contribuir Donacions Llicència

En aquesta secció, explorarem l'estructura d'un projecte Spring Boot. Comprendre la disposició i la finalitat de cada component és crucial per desenvolupar, mantenir i escalar les teves aplicacions de manera efectiva.

Components Clau d'un Projecte Spring Boot

Un projecte típic de Spring Boot consta de diversos components i directoris clau. Anem a desglossar cadascun d'aquests:

1. `src/main/java`

Aquest directori conté el codi font principal de la teva aplicació.

  • Classe d'Aplicació: Aquest és el punt d'entrada de la teva aplicació Spring Boot. Normalment està anotada amb @SpringBootApplication.

    package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

  • Classes de Controlador: Aquestes gestionen les sol·licituds HTTP entrants i retornen respostes. Normalment estan anotades amb @RestController.

    package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
    @GetMapping("/hello")
    public String sayHello() {
        return "Hello, World!";
    }
}

  • Classes de Servei: Aquestes contenen la lògica de negoci i estan anotades amb @Service.

    package com.example.demo.service;

import org.springframework.stereotype.Service;

@Service
public class HelloService {
    public String getGreeting() {
        return "Hello, World!";
    }
}

  • Classes de Repositori: Aquestes interactuen amb la base de dades i estan anotades amb @Repository.

    package com.example.demo.repository;

import org.springframework.data.jpa.repository.JpaRepository;
import com.example.demo.model.User;

public interface UserRepository extends JpaRepository<User, Long> {
}

2. `src/main/resources`

Aquest directori conté fitxers de configuració i recursos estàtics.

  • application.properties o application.yml: Configuració de la teva aplicació.

    server.port=8081
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=secret

  • Recursos Estàtics: Fitxers com HTML, CSS i JavaScript que es serveixen directament al client.

    src/main/resources/static/
├── css/
├── js/
└── images/

  • Plantilles: Plantilles HTML utilitzades per Thymeleaf o altres motors de plantilles.

    src/main/resources/templates/
├── index.html
└── about.html

3. `src/test/java`

Aquest directori conté casos de prova per a la teva aplicació.

  • Proves Unitàries: Prova components individuals com serveis i controladors.

    package com.example.demo;

import static org.assertj.core.api.Assertions.assertThat;

import com.example.demo.controller.HelloController;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;

@SpringBootTest
public class DemoApplicationTests {

    @Autowired
    private HelloController helloController;

    @Test
    public void contextLoads() throws Exception {
        assertThat(helloController).isNotNull();
    }
}

  • Proves d'Integració: Prova la interacció entre múltiples components.

    package com.example.demo;

import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.get;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.status;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.content;

import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.web.servlet.AutoConfigureMockMvc;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.test.web.servlet.MockMvc;

@SpringBootTest
@AutoConfigureMockMvc
public class HelloControllerIntegrationTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    public void shouldReturnDefaultMessage() throws Exception {
        this.mockMvc.perform(get("/hello"))
            .andExpect(status().isOk())
            .andExpect(content().string("Hello, World!"));
    }
}

4. `pom.xml` o `build.gradle`

Aquest fitxer s'utilitza per gestionar les dependències del projecte i les configuracions de construcció.

  • Maven (pom.xml):

    <project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>demo</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <packaging>jar</packaging>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        <!-- Afegeix altres dependències aquí -->
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>

  • Gradle (build.gradle):

    plugins {
    id 'org.springframework.boot' version '2.5.4'
    id 'io.spring.dependency-management' version '1.0.11.RELEASE'
    id 'java'
}

group = 'com.example'
version = '0.0.1-SNAPSHOT'
sourceCompatibility = '11'

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    // Afegeix altres dependències aquí
}

test {
    useJUnitPlatform()
}

Resum

Comprendre l'estructura d'un projecte Spring Boot és fonamental per a un desenvolupament eficient. Aquí tens un breu resum del que hem cobert:

  • src/main/java: Conté el codi principal de l'aplicació, incloent controladors, serveis i repositoris.
  • src/main/resources: Conté fitxers de configuració i recursos estàtics.
  • src/test/java: Conté proves unitàries i d'integració.
  • Fitxers de Construcció: pom.xml per a Maven o build.gradle per a Gradle, utilitzats per gestionar dependències i configuracions de construcció.

Familiaritzant-te amb aquests components, podràs navegar i gestionar els teus projectes Spring Boot de manera més efectiva.
