Introducció

Les anotacions de Spring Boot són una manera poderosa de simplificar la configuració i el desenvolupament d'aplicacions Spring. Proporcionen metadades que poden ser utilitzades pel framework Spring per realitzar diverses tasques com la injecció de dependències, la gestió de transaccions i més. Aquesta secció cobrirà les anotacions de Spring Boot més comunament utilitzades, els seus propòsits i com utilitzar-les de manera efectiva.

Anotacions clau

@SpringBootApplication

  • Propòsit: Aquesta és una anotació de conveniència que combina @Configuration, @EnableAutoConfiguration i @ComponentScan.
  • Ús: Normalment es col·loca a la classe principal de la teva aplicació Spring Boot.
@SpringBootApplication
public class MySpringBootApplication {
    public static void main(String[] args) {
        SpringApplication.run(MySpringBootApplication.class, args);
    }
}

@RestController

  • Propòsit: Aquesta anotació s'utilitza per crear serveis web RESTful utilitzant Spring MVC.
  • Ús: Combina @Controller i @ResponseBody.
@RestController
public class MyRestController {
    @GetMapping("/hello")
    public String sayHello() {
        return "Hola, Món!";
    }
}

@RequestMapping

  • Propòsit: Aquesta anotació s'utilitza per mapar sol·licituds web a mètodes de controlador específics.
  • Ús: Es pot aplicar tant a nivell de classe com de mètode.
@RestController
@RequestMapping("/api")
public class ApiController {
    @GetMapping("/users")
    public List<User> getUsers() {
        return userService.getAllUsers();
    }
}

@Autowired

  • Propòsit: Aquesta anotació s'utilitza per a la injecció automàtica de dependències.
  • Ús: Es pot aplicar a constructors, mètodes i camps.
@Service
public class UserService {
    private final UserRepository userRepository;

    @Autowired
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

@Component

  • Propòsit: Aquesta anotació indica que una classe és un component de Spring.
  • Ús: És un estereotip genèric per a qualsevol component gestionat per Spring.
@Component
public class MyComponent {
    public void doSomething() {
        System.out.println("Fent alguna cosa...");
    }
}

@Service

  • Propòsit: Aquesta anotació és una especialització de @Component i s'utilitza per indicar que una classe és un servei.
  • Ús: Normalment s'utilitza a la capa de servei.
@Service
public class MyService {
    public void performService() {
        System.out.println("El servei s'està realitzant...");
    }
}

@Repository

  • Propòsit: Aquesta anotació és una especialització de @Component i s'utilitza per indicar que una classe és un repositori.
  • Ús: Normalment s'utilitza a la capa d'accés a dades.
@Repository
public class UserRepository {
    // Mètodes d'accés a dades
}

@Configuration

  • Propòsit: Aquesta anotació indica que una classe declara un o més mètodes @Bean.
  • Ús: S'utilitza per definir beans i les seves dependències.
@Configuration
public class AppConfig {
    @Bean
    public MyBean myBean() {
        return new MyBean();
    }
}

@Bean

  • Propòsit: Aquesta anotació indica que un mètode produeix un bean que serà gestionat pel contenidor de Spring.
  • Ús: S'utilitza dins de classes @Configuration.
@Configuration
public class AppConfig {
    @Bean
    public MyService myService() {
        return new MyService();
    }
}

@Value

  • Propòsit: Aquesta anotació s'utilitza per injectar valors en camps des d'arxius de propietats.
  • Ús: Es pot aplicar a camps, mètodes i paràmetres de mètode.
@Component
public class MyComponent {
    @Value("${my.property}")
    private String myProperty;

    public void printProperty() {
        System.out.println(myProperty);
    }
}

Conclusió

Les anotacions de Spring Boot són eines essencials per simplificar el desenvolupament i la configuració d'aplicacions Spring. Entenent i utilitzant efectivament aquestes anotacions, pots crear aplicacions robustes i mantenibles amb una configuració mínima. Aquesta visió general ha cobert les anotacions més comunament utilitzades, proporcionant-te una base sòlida per construir mentre continues explorant Spring Boot.
