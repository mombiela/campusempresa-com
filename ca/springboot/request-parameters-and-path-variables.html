Paràmetres de Sol·licitud i Variables de Camí
Gestió d'Excepcions a Spring Boot ►

En aquesta secció, explorarem com gestionar els paràmetres de sol·licitud i les variables de camí a Spring Boot. Aquests són conceptes essencials per construir serveis web RESTful i gestionar les sol·licituds dels clients de manera efectiva.

Paràmetres de Sol·licitud

Els paràmetres de sol·licitud s'utilitzen per enviar dades del client al servidor com a part de la cadena de consulta de l'URL. Normalment s'utilitzen per filtrar, ordenar o especificar opcions addicionals per a una sol·licitud.

Conceptes Clau

  • Cadena de Consulta: La part de l'URL després del símbol ?, que conté parells clau-valor.
  • Anotació @RequestParam: S'utilitza per vincular els paràmetres de sol·licitud als paràmetres del mètode a Spring Boot.

Exemple

Creem una aplicació simple de Spring Boot per demostrar com utilitzar els paràmetres de sol·licitud.

  1. Crear un Projecte de Spring Boot: Utilitza Spring Initializr per generar un nou projecte de Spring Boot amb les dependències necessàries (Spring Web).

  2. Classe Controladora: Crea una classe controladora per gestionar les sol·licituds HTTP.

package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class GreetingController {

    @GetMapping("/greet")
    public String greet(@RequestParam(value = "name", defaultValue = "World") String name) {
        return "Hello, " + name + "!";
    }
}

Explicació

  • @RestController: Indica que aquesta classe és un controlador REST.
  • @GetMapping("/greet"): Assigna les sol·licituds HTTP GET al punt final /greet.
  • @RequestParam: Vincula el paràmetre de sol·licitud name al paràmetre del mètode name. Si no es proporciona el paràmetre name, el valor per defecte és "World".

Provar el Punt Final

  • Accedeix al punt final mitjançant http://localhost:8080/greet?name=John.
  • La resposta serà Hello, John!.
  • Si s'ometeix el paràmetre name, la resposta serà Hello, World!.

Variables de Camí

Les variables de camí s'utilitzen per capturar valors del camí de l'URL. Normalment s'utilitzen per identificar recursos específics.

Conceptes Clau

  • Camí de l'URL: La part de l'URL que especifica la ubicació del recurs.
  • Anotació @PathVariable: S'utilitza per vincular les variables de camí als paràmetres del mètode a Spring Boot.

Exemple

Ampliem la nostra aplicació de Spring Boot per demostrar com utilitzar les variables de camí.

  1. Classe Controladora: Actualitza la classe controladora per gestionar les variables de camí.
package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class GreetingController {

    @GetMapping("/greet")
    public String greet(@RequestParam(value = "name", defaultValue = "World") String name) {
        return "Hello, " + name + "!";
    }

    @GetMapping("/greet/{name}")
    public String greetWithPathVariable(@PathVariable String name) {
        return "Hello, " + name + "!";
    }
}

Explicació

  • @GetMapping("/greet/{name}"): Assigna les sol·licituds HTTP GET al punt final /greet/{name}.
  • @PathVariable: Vincula la variable de camí name al paràmetre del mètode name.

Provar el Punt Final

  • Accedeix al punt final mitjançant http://localhost:8080/greet/John.
  • La resposta serà Hello, John!.

Taula Comparativa

| Característica | Paràmetres de Sol·licitud | Variables de Camí | |-----------------|------------------------------------------|---------------------------------------| | Ús | Filtrar, ordenar, opcions addicionals | Identificar recursos específics | | Anotació | @RequestParam | @PathVariable | | Estructura URL | http://localhost:8080/greet?name=John | http://localhost:8080/greet/John | | Valors per Defecte | Suportats | No suportats |

Conclusió

Entendre com gestionar els paràmetres de sol·licitud i les variables de camí és crucial per construir serveis web RESTful robustos i flexibles a Spring Boot. Els paràmetres de sol·licitud són ideals per a dades opcionals i filtratge, mentre que les variables de camí són perfectes per identificar recursos específics. En dominar aquests conceptes, pots crear APIs més dinàmiques i fàcils d'utilitzar.
