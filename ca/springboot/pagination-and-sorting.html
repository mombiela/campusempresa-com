Paginació i Ordenació en Spring Boot
Introducció a Spring Security ►

La paginació i l'ordenació són característiques essencials per gestionar grans conjunts de dades en aplicacions web. Ajuden a dividir grans dades en fragments manejables i a ordenar les dades d'una manera significativa. En aquesta secció, cobrirem com implementar la paginació i l'ordenació en una aplicació Spring Boot.

Introducció a la Paginació i l'Ordenació

  • Paginació: El procés de dividir un conjunt de dades en pàgines discretes.
  • Ordenació: Disposar les dades en un ordre particular (ascendent o descendent) basat en un o més camps.

Configurant el Projecte

Abans d'endinsar-nos en la paginació i l'ordenació, assegureu-vos que teniu un projecte Spring Boot configurat. Podeu crear un nou projecte Spring Boot utilitzant Spring Initializr o el vostre IDE preferit.

Dependències

Afegiu les següents dependències al vostre fitxer pom.xml:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>

Configurant el Model de Dades

Creem una classe d'entitat simple Product per treballar.

@Entity
public class Product {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private Double price;

    // Getters and Setters
}

Creant el Repositori

Spring Data JPA proporciona suport integrat per a la paginació i l'ordenació. Creeu una interfície de repositori per a l'entitat Product.

public interface ProductRepository extends JpaRepository<Product, Long> {
}

Implementant la Paginació

Per implementar la paginació, heu d'utilitzar la interfície Pageable proporcionada per Spring Data JPA.

Exemple: Obtenint una Pàgina de Productes

@RestController
@RequestMapping("/products")
public class ProductController {

    @Autowired
    private ProductRepository productRepository;

    @GetMapping
    public Page<Product> getProducts(@RequestParam int page, @RequestParam int size) {
        Pageable pageable = PageRequest.of(page, size);
        return productRepository.findAll(pageable);
    }
}

Explicació

  • @RequestParam int page: El número de pàgina a obtenir.
  • @RequestParam int size: La mida de la pàgina.
  • PageRequest.of(page, size): Crea un objecte Pageable amb el número de pàgina i la mida especificats.
  • productRepository.findAll(pageable): Obté la pàgina de productes.

Implementant l'Ordenació

L'ordenació es pot afegir fàcilment a la lògica de paginació utilitzant la classe Sort.

Exemple: Obtenint Productes Ordenats

@GetMapping("/sorted")
public Page<Product> getSortedProducts(@RequestParam int page, @RequestParam int size, @RequestParam String sortBy) {
    Pageable pageable = PageRequest.of(page, size, Sort.by(sortBy));
    return productRepository.findAll(pageable);
}

Explicació

  • @RequestParam String sortBy: El camp per ordenar.
  • Sort.by(sortBy): Crea un objecte Sort per especificar el camp d'ordenació.

Combinant Paginació i Ordenació

Podeu combinar la paginació i l'ordenació en una sola sol·licitud.

Exemple: Obtenint una Pàgina de Productes Ordenats

@GetMapping("/paged-sorted")
public Page<Product> getPagedSortedProducts(@RequestParam int page, @RequestParam int size, @RequestParam String sortBy, @RequestParam String direction) {
    Sort.Direction sortDirection = Sort.Direction.fromString(direction);
    Pageable pageable = PageRequest.of(page, size, Sort.by(sortDirection, sortBy));
    return productRepository.findAll(pageable);
}

Explicació

  • @RequestParam String direction: La direcció de l'ordenació (ASC o DESC).
  • Sort.Direction.fromString(direction): Converteix la cadena de direcció en un objecte Sort.Direction.
  • Sort.by(sortDirection, sortBy): Crea un objecte Sort amb la direcció i el camp especificats.

Conclusió

En aquesta secció, hem cobert els conceptes bàsics d'implementar la paginació i l'ordenació en una aplicació Spring Boot utilitzant Spring Data JPA. Hem après com:

  • Configurar el projecte i les dependències.
  • Crear un model de dades i un repositori.
  • Implementar la paginació i l'ordenació.
  • Combinar la paginació i l'ordenació en una sola sol·licitud.

Aquestes característiques són crucials per construir aplicacions web eficients i fàcils d'usar que gestionin grans conjunts de dades. En dominar la paginació i l'ordenació, podeu millorar significativament el rendiment i la usabilitat de les vostres aplicacions.
