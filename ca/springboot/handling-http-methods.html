Introducció

En aquesta secció, tractarem com gestionar els principals mètodes HTTP (GET, POST, PUT, DELETE) utilitzant Spring Boot. Entendre aquests mètodes és crucial per construir serveis web RESTful. Començarem amb els conceptes bàsics i gradualment passarem a conceptes més avançats.

Mètode GET

El mètode GET s'utilitza per recuperar informació del servidor. És idempotent, el que significa que múltiples sol·licituds idèntiques haurien de tenir el mateix efecte que una sola sol·licitud.

Conceptes Clau

  • Utilitzat per obtenir dades.
  • No hauria d'alterar l'estat del servidor.
  • Pot ser emmagatzemat en memòria cau.

Exemple

@RestController
@RequestMapping("/api")
public class ExampleController {

    @GetMapping("/items")
    public List<Item> getAllItems() {
        // Lògica per recuperar ítems
        return itemService.getAllItems();
    }
}

En aquest exemple, l'anotació @GetMapping mapatge les sol·licituds HTTP GET al mètode getAllItems.

Mètode POST

El mètode POST s'utilitza per enviar dades a un recurs especificat per ser processades. Sovint resulta en un canvi en l'estat del servidor o efectes secundaris.

Conceptes Clau

  • Utilitzat per crear nous recursos.
  • Pot alterar l'estat del servidor.
  • No és idempotent.

Exemple

@RestController
@RequestMapping("/api")
public class ExampleController {

    @PostMapping("/items")
    public Item createItem(@RequestBody Item item) {
        // Lògica per crear un ítem
        return itemService.createItem(item);
    }
}

Aquí, l'anotació @PostMapping mapatge les sol·licituds HTTP POST al mètode createItem. L'anotació @RequestBody s'utilitza per vincular el cos de la sol·licitud al paràmetre del mètode.

Mètode PUT

El mètode PUT s'utilitza per actualitzar un recurs existent o crear un nou recurs si no existeix.

Conceptes Clau

  • Utilitzat per actualitzar recursos existents.
  • També pot crear recursos si no existeixen.
  • Idempotent.

Exemple

@RestController
@RequestMapping("/api")
public class ExampleController {

    @PutMapping("/items/{id}")
    public Item updateItem(@PathVariable Long id, @RequestBody Item item) {
        // Lògica per actualitzar un ítem
        return itemService.updateItem(id, item);
    }
}

En aquest exemple, l'anotació @PutMapping mapatge les sol·licituds HTTP PUT al mètode updateItem. L'anotació @PathVariable s'utilitza per extreure l'id de l'URL.

Mètode DELETE

El mètode DELETE s'utilitza per eliminar un recurs especificat.

Conceptes Clau

  • Utilitzat per eliminar recursos.
  • Idempotent.

Exemple

@RestController
@RequestMapping("/api")
public class ExampleController {

    @DeleteMapping("/items/{id}")
    public void deleteItem(@PathVariable Long id) {
        // Lògica per eliminar un ítem
        itemService.deleteItem(id);
    }
}

Aquí, l'anotació @DeleteMapping mapatge les sol·licituds HTTP DELETE al mètode deleteItem.

Resum

En aquesta secció, hem tractat els quatre principals mètodes HTTP (GET, POST, PUT, DELETE) i com gestionar-los utilitzant Spring Boot. Cada mètode serveix per a un propòsit específic en serveis web RESTful:

  • GET: Recuperar dades.
  • POST: Crear nous recursos.
  • PUT: Actualitzar recursos existents o crear-ne de nous.
  • DELETE: Eliminar recursos.

Entendre aquests mètodes i els seus casos d'ús apropiats és essencial per construir serveis web robustos i eficients.
