Visió general

En aquest tema, explorarem els serveis web RESTful i com implementar-los utilitzant Spring Boot. Cobrirarem els conceptes bàsics de REST, els principis darrere de l'arquitectura RESTful i com crear, llegir, actualitzar i eliminar recursos utilitzant Spring Boot.

Què és REST?

  • REST (Transferència d'Estat Representacional): Un estil arquitectònic per dissenyar aplicacions en xarxa.
  • Recursos: Abstracció clau en REST, típicament representada per URIs.
  • Mètodes HTTP: Utilitzats per realitzar operacions sobre recursos.
    • GET: Recuperar un recurs.
    • POST: Crear un nou recurs.
    • PUT: Actualitzar un recurs existent.
    • DELETE: Eliminar un recurs.

Principis de l'arquitectura RESTful

  • Sense estat: Cada sol·licitud d'un client al servidor ha de contenir tota la informació necessària per entendre i processar la sol·licitud.
  • Client-Servidor: Separació de les preocupacions del client i del servidor.
  • Emmagatzemable: Les respostes han de definir-se com emmagatzemables o no per evitar que els clients reutilitzin dades obsoletes.
  • Interfície uniforme: Simplifica i desacobla l'arquitectura, permetent que cada part evolucioni independentment.
  • Sistema en capes: Permet que una arquitectura es composi de capes jeràrquiques restringint el comportament dels components.

Configuració de Spring Boot

Creació d'un projecte Spring Boot

  1. Utilitzant Spring Initializr:

    • Aneu a Spring Initializr.
    • Seleccioneu les opcions del projecte (per exemple, Maven, Java, versió de Spring Boot).
    • Afegiu dependències: Spring Web.
    • Genereu i descarregueu el projecte.

  2. Importació del projecte:

    • Importeu el projecte descarregat al vostre IDE (per exemple, IntelliJ IDEA, Eclipse).

Estructura del projecte

  • src/main/java: Conté el codi font Java.
  • src/main/resources: Conté fitxers de configuració.
  • pom.xml: Fitxer de configuració de Maven.

Creació d'un controlador

  • EmployeeController: Gestiona les sol·licituds HTTP. 
    @RestController
@RequestMapping("/employees")
public class EmployeeController {

    private List<Employee> employees = new ArrayList<>();

    @GetMapping
    public List<Employee> getAllEmployees() {
        return employees;
    }

    @PostMapping
    public Employee createEmployee(@RequestBody Employee employee) {
        employees.add(employee);
        return employee;
    }

    @GetMapping("/{id}")
    public Employee getEmployeeById(@PathVariable Long id) {
        return employees.stream()
                .filter(emp -> emp.getId().equals(id))
                .findFirst()
                .orElseThrow(() -> new ResourceNotFoundException("Employee not found"));
    }

    @PutMapping("/{id}")
    public Employee updateEmployee(@PathVariable Long id, @RequestBody Employee employeeDetails) {
        Employee employee = employees.stream()
                .filter(emp -> emp.getId().equals(id))
                .findFirst()
                .orElseThrow(() -> new ResourceNotFoundException("Employee not found"));

        employee.setName(employeeDetails.getName());
        employee.setRole(employeeDetails.getRole());
        return employee;
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<?> deleteEmployee(@PathVariable Long id) {
        Employee employee = employees.stream()
                .filter(emp -> emp.getId().equals(id))
                .findFirst()
                .orElseThrow(() -> new ResourceNotFoundException("Employee not found"));

        employees.remove(employee);
        return ResponseEntity.ok().build();
    }
}

Gestió d'excepcions

  • ResourceNotFoundException: Excepció personalitzada per recurs no trobat. 
    @ResponseStatus(HttpStatus.NOT_FOUND)
public class ResourceNotFoundException extends RuntimeException {
    public ResourceNotFoundException(String message) {
        super(message);
    }
}

Prova del servei web RESTful

Utilitzant Postman

  • GET /employees: Recuperar tots els empleats.
  • POST /employees: Crear un nou empleat.
  • GET /employees/{id}: Recuperar un empleat per ID.
  • PUT /employees/{id}: Actualitzar un empleat.
  • DELETE /employees/{id}: Eliminar un empleat.
