Introducció als Perfils

Els perfils de Spring Boot proporcionen una manera de separar parts de la configuració de la teva aplicació i fer-la disponible només en certs entorns. Això és particularment útil quan necessites executar la teva aplicació amb diferents configuracions per a entorns de desenvolupament, proves i producció.

Conceptes Clau

  • Perfil: Un grup lògic i amb nom de configuracions.
  • Configuració Específica de l'Entorn: Diferents configuracions per a diferents entorns (per exemple, application-dev.properties, application-prod.properties).
  • Activació: Mecanisme per activar un perfil específic.

Creació i Ús de Perfils

Definició de Perfils

Els perfils es poden definir en fitxers de propietats o fitxers YAML. Per exemple:

  • application-dev.properties
  • application-prod.properties

Exemple: Fitxers de Propietats

Crea diferents fitxers de propietats per a diferents entorns.

application-dev.properties

server.port=8081
spring.datasource.url=jdbc:h2:mem:devdb
spring.datasource.username=sa
spring.datasource.password=password

application-prod.properties

server.port=8080
spring.datasource.url=jdbc:mysql://prod-db-server/mydb
spring.datasource.username=produser
spring.datasource.password=prodpassword

Activació de Perfils

Els perfils es poden activar de diverses maneres:

  • Línia de Comandes: Utilitzant -Dspring.profiles.active=dev
  • Variable d'Entorn: Configurant SPRING_PROFILES_ACTIVE=dev
  • Propietats de l'Aplicació: Afegint spring.profiles.active=dev a application.properties

Exemple: Activació de Perfil via Línia de Comandes

java -jar myapp.jar --spring.profiles.active=dev

Exemple: Activació de Perfil via Variable d'Entorn

export SPRING_PROFILES_ACTIVE=dev

Ús de Perfils en el Codi

També pots utilitzar perfils directament en el teu codi de Spring Boot per carregar condicionalment beans o configuracions.

Exemple: Creació Condicional de Beans

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Profile;

@Configuration
public class DataSourceConfig {

    @Bean
    @Profile("dev")
    public DataSource devDataSource() {
        return new H2DataSource("jdbc:h2:mem:devdb", "sa", "password");
    }

    @Bean
    @Profile("prod")
    public DataSource prodDataSource() {
        return new MySQLDataSource("jdbc:mysql://prod-db-server/mydb", "produser", "prodpassword");
    }
}

Gestió Avançada de Perfils

Perfils Múltiples

Pots activar múltiples perfils simultàniament separant-los amb comes.

Exemple: Activació de Perfils Múltiples

java -jar myapp.jar --spring.profiles.active=dev,debug

Beans Específics de Perfil

Pots definir beans que només haurien d'estar disponibles en certs perfils utilitzant l'anotació @Profile.

Exemple: Beans Específics de Perfil

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Profile;

@Configuration
public class AppConfig {

    @Bean
    @Profile("test")
    public TestService testService() {
        return new TestService();
    }

    @Bean
    @Profile("prod")
    public ProdService prodService() {
        return new ProdService();
    }
}

Resum

Els perfils de Spring Boot et permeten gestionar fàcilment diferents configuracions per a diferents entorns. Definint fitxers de propietats específics per a cada entorn i activant perfils a través de diverses maneres, pots assegurar-te que la teva aplicació es comporti correctament en entorns de desenvolupament, proves i producció. Els perfils també es poden utilitzar directament en el teu codi per carregar condicionalment beans i configuracions, proporcionant una manera flexible i potent de gestionar les configuracions de l'aplicació.

Dominant els perfils, pots crear aplicacions de Spring Boot més robustes i adaptables que són més fàcils de gestionar i desplegar en diferents entorns.
