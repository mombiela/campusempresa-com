Introducció

En aquesta secció, explorarem com crear i utilitzar repositoris en Spring Boot. Els repositoris són una part crucial del mòdul Spring Data JPA, que simplifica la implementació de capes d'accés a dades proporcionant un conjunt d'interfícies i anotacions.

Conceptes Clau

  • Spring Data JPA: Una part de la família més gran de Spring Data, Spring Data JPA facilita la implementació de repositoris basats en JPA.
  • Interfície de Repositori: Una interfície que extén una de les interfícies de repositori de Spring Data (per exemple, CrudRepository, JpaRepository).
  • Entitat: Una classe anotada amb @Entity que representa una taula a la base de dades.
  • Operacions CRUD: Operacions bàsiques (Crear, Llegir, Actualitzar, Eliminar) proporcionades pels repositoris de Spring Data.

Configuració del Projecte

Abans d'endinsar-nos en la creació de repositoris, assegureu-vos que teniu un projecte Spring Boot configurat amb les dependències necessàries.

Dependències

Afegiu les següents dependències al vostre pom.xml si esteu utilitzant Maven:

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>
    <dependency>
        <groupId>com.h2database</groupId>
        <artifactId>h2</artifactId>
        <scope>runtime</scope>
    </dependency>
</dependencies>

Creació d'una Entitat

Primer, creem una classe d'entitat que es mapegi a una taula de la base de dades.

import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;

@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private String email;

    // Getters and Setters
}
  • @Entity: Especifica que la classe és una entitat i es mapeja a una taula de la base de dades.
  • @Id: Especifica la clau primària d'una entitat.
  • @GeneratedValue: Proporciona l'estratègia de generació per a la clau primària.

Creació d'una Interfície de Repositori

A continuació, creem una interfície de repositori per a l'entitat User.

import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository<User, Long> {
}
  • JpaRepository<User, Long>: Extén la interfície JpaRepository, proporcionant operacions CRUD per a l'entitat User. El primer paràmetre és el tipus d'entitat, i el segon és el tipus de la clau primària.

Ús del Repositori

Ara, utilitzem el UserRepository en una classe de servei per realitzar operacions CRUD.

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Optional;

@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;

    public User createUser(User user) {
        return userRepository.save(user);
    }

    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    public Optional<User> getUserById(Long id) {
        return userRepository.findById(id);
    }

    public User updateUser(Long id, User userDetails) {
        User user = userRepository.findById(id).orElseThrow(() -> new RuntimeException("User not found"));
        user.setName(userDetails.getName());
        user.setEmail(userDetails.getEmail());
        return userRepository.save(user);
    }

    public void deleteUser(Long id) {
        userRepository.deleteById(id);
    }
}
  • @Autowired: Injecta el bean UserRepository.
  • save(): Desa una entitat donada.
  • findAll(): Recupera totes les entitats.
  • findById(): Recupera una entitat pel seu ID.
  • deleteById(): Elimina una entitat pel seu ID.

Funcionalitats Avançades del Repositori

Spring Data JPA proporciona diverses funcionalitats avançades per millorar la funcionalitat del repositori.

Mètodes de Consulta Personalitzats

Podeu definir mètodes de consulta personalitzats a la vostra interfície de repositori.

import java.util.List;

public interface UserRepository extends JpaRepository<User, Long> {
    List<User> findByName(String name);
}
  • findByName(String name): Troba usuaris pel seu nom. Spring Data JPA genera automàticament la consulta basada en el nom del mètode.

JPQL i Consultes Natives

També podeu utilitzar JPQL (Java Persistence Query Language) o consultes SQL natives.

import org.springframework.data.jpa.repository.Query;

public interface UserRepository extends JpaRepository<User, Long> {
    @Query("SELECT u FROM User u WHERE u.email = ?1")
    User findByEmail(String email);

    @Query(value = "SELECT * FROM User WHERE email = ?1", nativeQuery = true)
    User findByEmailNative(String email);
}
  • @Query: Defineix una consulta JPQL o SQL nativa.

Conclusió

En aquesta secció, hem cobert els conceptes bàsics de la creació i ús de repositoris en Spring Boot. Hem començat amb la configuració del projecte, la creació d'una entitat, i després la creació d'una interfície de repositori. També hem explorat com utilitzar el repositori en una classe de servei i hem vist algunes funcionalitats avançades com els mètodes de consulta personalitzats i les consultes JPQL/natives. Entendre aquests conceptes és crucial per construir capes d'accés a dades robustes i mantenibles en les vostres aplicacions Spring Boot.
