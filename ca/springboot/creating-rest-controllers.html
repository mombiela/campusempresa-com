En aquesta secció, explorarem com crear controladors REST a Spring Boot. Començarem amb els conceptes bàsics i gradualment passarem a conceptes més avançats. Al final d'aquesta secció, hauríeu de ser capaços de crear i gestionar serveis web RESTful utilitzant Spring Boot.

Introducció als Controladors REST

REST (Representational State Transfer) és un estil arquitectònic per dissenyar aplicacions en xarxa. Les aplicacions RESTful utilitzen sol·licituds HTTP per realitzar operacions CRUD (Create, Read, Update, Delete).

Conceptes Clau

  • Mètodes HTTP: GET, POST, PUT, DELETE, PATCH
  • Endpoints: URLs que representen recursos
  • Codis d'Estat: Indiquen el resultat d'una sol·licitud HTTP (per exemple, 200 OK, 404 Not Found)
  • Cos de la Sol·licitud: Dades enviades amb sol·licituds POST, PUT i PATCH
  • Cos de la Resposta: Dades retornades pel servidor

Configuració de Spring Boot

Abans de crear controladors REST, assegureu-vos que teniu un projecte Spring Boot configurat. Podeu crear un nou projecte Spring Boot utilitzant Spring Initializr o el vostre IDE preferit.

Dependències

Assegureu-vos que teniu les següents dependències al vostre fitxer pom.xml o build.gradle:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

Creació d'un Controlador REST Simple

Comencem creant un controlador REST simple que gestioni sol·licituds HTTP GET bàsiques.

Exemple: Controlador Hello World

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/api")
public class HelloWorldController {

    @GetMapping("/hello")
    public String sayHello() {
        return "Hello, World!";
    }
}

Explicació

  • @RestController: Indica que aquesta classe és un controlador REST.
  • @RequestMapping("/api"): URL base per a tots els endpoints en aquest controlador.
  • @GetMapping("/hello"): Assigna sol·licituds HTTP GET a /api/hello al mètode sayHello.

Gestió de Diferents Mètodes HTTP

Spring Boot facilita la gestió de diferents mètodes HTTP utilitzant anotacions.

Exemple: Operacions CRUD per a un Recurs

import org.springframework.web.bind.annotation.*;

import java.util.ArrayList;
import java.util.List;

@RestController
@RequestMapping("/api/items")
public class ItemController {

    private List<String> items = new ArrayList<>();

    @GetMapping
    public List<String> getAllItems() {
        return items;
    }

    @PostMapping
    public void addItem(@RequestBody String item) {
        items.add(item);
    }

    @PutMapping("/{index}")
    public void updateItem(@PathVariable int index, @RequestBody String item) {
        items.set(index, item);
    }

    @DeleteMapping("/{index}")
    public void deleteItem(@PathVariable int index) {
        items.remove(index);
    }
}

Explicació

  • @GetMapping: Recupera tots els ítems.
  • @PostMapping: Afegeix un nou ítem.
  • @PutMapping("/{index}"): Actualitza un ítem a l'índex especificat.
  • @DeleteMapping("/{index}"): Elimina un ítem a l'índex especificat.
  • @RequestBody: Assigna el cos de la sol·licitud HTTP a un paràmetre del mètode.
  • @PathVariable: Extreu valors de l'URI.

Gestió d'Excepcions

Gestionar excepcions de manera elegant és crucial per a una API REST robusta.

Exemple: Gestor d'Excepcions Personalitzat

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(IndexOutOfBoundsException.class)
    public ResponseEntity<String> handleIndexOutOfBounds(IndexOutOfBoundsException ex) {
        return new ResponseEntity<>("Item not found", HttpStatus.NOT_FOUND);
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<String> handleGeneralException(Exception ex) {
        return new ResponseEntity<>("An error occurred", HttpStatus.INTERNAL_SERVER_ERROR);
    }
}

Explicació

  • @RestControllerAdvice: Permet gestionar excepcions globalment.
  • @ExceptionHandler: Especifica el tipus d'excepció a gestionar.

Validació de Sol·licituds

La validació assegura que les dades enviades pel client compleixin certs criteris.

Exemple: Validació d'un Cos de Sol·licitud

import javax.validation.Valid;
import javax.validation.constraints.NotEmpty;

import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/validated-items")
public class ValidatedItemController {

    @PostMapping
    public void addItem(@Valid @RequestBody ValidatedItem item) {
        // Processa l'ítem
    }

    public static class ValidatedItem {
        @NotEmpty(message = "Name cannot be empty")
        private String name;

        // Getters i setters
    }
}

Explicació

  • @Valid: Activa la validació sobre el cos de la sol·licitud.
  • @NotEmpty: Assegura que el camp no estigui buit.
