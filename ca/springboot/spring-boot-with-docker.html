Introducció a Docker

  • Què és Docker?
    • Docker és una plataforma que permet als desenvolupadors automatitzar el desplegament d'aplicacions dins de contenidors lleugers i portàtils.
  • Per què utilitzar Docker amb Spring Boot?
    • Simplifica el desplegament.
    • Assegura la consistència a través de diferents entorns.
    • Facilita l'escalat i la gestió d'aplicacions.

Configuració de Docker

  • Instal·lació de Docker
    • Segueix la guia oficial d'instal·lació de Docker per al teu sistema operatiu: Docker Installation
  • Comandes bàsiques de Docker
    • docker --version: Comprovar la versió de Docker.
    • docker run hello-world: Executar un contenidor de prova per verificar la instal·lació.

Creació d'una aplicació Spring Boot

  • Inicialitzar un projecte Spring Boot
    • Utilitza Spring Initializr per crear un nou projecte Spring Boot amb les dependències necessàries.
    • Exemple de dependències: Spring Web, Spring Data JPA, H2 Database.
// Exemple d'aplicació Spring Boot
package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

Dockeritzar una aplicació Spring Boot

  • Creació d'un Dockerfile
    • Un Dockerfile és un script que conté una sèrie d'instruccions sobre com construir una imatge Docker per a la teva aplicació.
# Utilitza una imatge oficial de JDK com a imatge base
FROM openjdk:11-jdk-slim

# Estableix el directori de treball dins del contenidor
WORKDIR /app

# Copia l'arxiu JAR executable dins del contenidor
COPY target/demo-0.0.1-SNAPSHOT.jar app.jar

# Executa l'arxiu JAR
ENTRYPOINT ["java", "-jar", "app.jar"]

  • Construcció de la imatge Docker

    • Navega al directori que conté el Dockerfile i executa la següent comanda: 
      docker build -t demo-app .

  • Execució del contenidor Docker

    • Executa el contenidor Docker utilitzant la imatge que acabes de construir: 
      docker run -p 8080:8080 demo-app

Gestió de contenidors Docker

  • Comandes comunes de Docker
    • docker ps: Llistar els contenidors en execució.
    • docker stop <container_id>: Aturar un contenidor en execució.
    • docker rm <container_id>: Eliminar un contenidor aturat.
    • docker images: Llistar totes les imatges Docker.
    • docker rmi <image_id>: Eliminar una imatge Docker.

Conceptes avançats de Docker

  • Docker Compose
    • Docker Compose és una eina per definir i executar aplicacions Docker multi-contenidor.
    • Exemple de docker-compose.yml per a una aplicació Spring Boot amb una base de dades PostgreSQL:
version: '3.8'
services:
  app:
    image: demo-app
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8080:8080"
    depends_on:
      - db
  db:
    image: postgres:13
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: demo
    ports:
      - "5432:5432"
  • Escalat de serveis
    • Docker Compose et permet escalar serveis utilitzant el flag --scale: 
      docker-compose up --scale app=3

Conclusió

  • Resum

    • Docker simplifica el desplegament i la gestió d'aplicacions Spring Boot.
    • En conteniritzar la teva aplicació, assegures la consistència a través de diferents entorns.
    • Docker Compose facilita la gestió d'aplicacions multi-contenidor, fent més fàcil escalar i gestionar dependències.

  • Pròxims passos

    • Explora funcions més avançades de Docker com ara xarxes, volums i Docker Swarm.
    • Integra Docker amb pipelines de CI/CD per a un desplegament automatitzat.
    • Experimenta amb Kubernetes per orquestrar contenidors Docker a gran escala.
