Spring Security és un marc d'autenticació i control d'accés potent i altament personalitzable per a aplicacions Java. És l'estàndard de facto per assegurar aplicacions basades en Spring. Aquesta secció et guiarà a través dels conceptes bàsics de Spring Security, des de la configuració fins a configuracions avançades.

Què és Spring Security?

Spring Security és un marc que se centra en proporcionar tant autenticació com autorització a aplicacions Java. És part de l'ecosistema més gran de Spring Framework i s'integra perfectament amb Spring Boot.

Conceptes Clau

  • Autenticació: Verificar la identitat d'un usuari o sistema.
  • Autorització: Determinar si un usuari autenticat té permís per realitzar una acció específica.
  • Principal: L'usuari actualment autenticat.
  • Autoritat Concedida: Un permís o rol assignat a un principal.
  • Context de Seguretat: Conté la informació d'autenticació de l'usuari actual.

Configuració de Spring Security

Per començar amb Spring Security en una aplicació Spring Boot, necessites afegir la dependència de Spring Security al teu projecte.

Afegint Dependències

Afegeix la següent dependència al teu pom.xml si estàs utilitzant Maven:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>

Per a Gradle, afegeix la següent línia al teu build.gradle:

implementation 'org.springframework.boot:spring-boot-starter-security'

Configuració Bàsica

Per defecte, Spring Security assegura tots els punts finals de la teva aplicació. Per personalitzar la configuració de seguretat, necessites crear una classe de configuració.

Creant una Classe de Configuració de Seguretat

Crea una classe que estengui WebSecurityConfigurerAdapter i sobreescriu els mètodes configure per personalitzar la configuració de seguretat.

import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;

@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/public/**").permitAll() // Permet l'accés als punts finals públics
                .anyRequest().authenticated() // Requereix autenticació per a tots els altres punts finals
                .and()
            .formLogin() // Habilita l'inici de sessió basat en formulari
                .loginPage("/login") // Pàgina d'inici de sessió personalitzada
                .permitAll()
                .and()
            .logout() // Habilita la funcionalitat de tancament de sessió
                .permitAll();
    }
}

Explicació

  • @Configuration i @EnableWebSecurity: Aquestes anotacions indiquen que aquesta classe és una classe de configuració i habiliten Spring Security.
  • authorizeRequests(): Configura l'autorització basada en URL.
    • antMatchers("/public/**").permitAll(): Permet l'accés a totes les URL sota /public/ sense autenticació.
    • anyRequest().authenticated(): Requereix autenticació per a qualsevol altra sol·licitud.
  • formLogin(): Habilita l'inici de sessió basat en formulari.
    • loginPage("/login"): Especifica una pàgina d'inici de sessió personalitzada.
    • permitAll(): Permet a tothom accedir a la pàgina d'inici de sessió.
  • logout(): Habilita la funcionalitat de tancament de sessió.

Configuració Avançada

Spring Security proporciona diverses funcions avançades com ara servei de detalls d'usuari personalitzat, seguretat a nivell de mètode, i més.

Custom UserDetailsService

Per utilitzar un servei de detalls d'usuari personalitzat, necessites implementar la interfície UserDetailsService i sobreescriure el mètode loadUserByUsername.

import org.springframework.security.core.userdetails.UserDetails;
import org.springframework.security.core.userdetails.UserDetailsService;
import org.springframework.security.core.userdetails.UsernameNotFoundException;
import org.springframework.stereotype.Service;

@Service
public class CustomUserDetailsService implements UserDetailsService {

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        // Recupera l'usuari de la base de dades o qualsevol altra font
        // Per simplicitat, estem utilitzant un usuari codificat
        if ("user".equals(username)) {
            return org.springframework.security.core.userdetails.User
                .withUsername("user")
                .password("{noop}password") // {noop} indica que no hi ha codificació de contrasenya
                .roles("USER")
                .build();
        } else {
            throw new UsernameNotFoundException("Usuari no trobat");
        }
    }
}

Seguretat a Nivell de Mètode

Pots assegurar mètodes individuals utilitzant anotacions com @PreAuthorize i @Secured.

import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.stereotype.Service;

@Service
public class SampleService {

    @PreAuthorize("hasRole('ROLE_USER')")
    public void securedMethod() {
        // Lògica del mètode
    }
}

Explicació

  • @PreAuthorize("hasRole('ROLE_USER')"): Assegura que el mètode només pot ser accedit per usuaris amb l'autoritat ROLE_USER.
