Spring Data JPA simplifica la capa d'accés a dades proporcionant una abstracció de repositori sobre JPA. Una de les seves característiques més potents és la capacitat de definir mètodes de consulta directament a les interfícies de repositori. Aquest tema cobrirà els conceptes bàsics fins als avançats dels mètodes de consulta a Spring Data JPA.

Introducció a Spring Data JPA

  • Spring Data JPA: Una part de la família més gran de Spring Data, facilita la implementació de repositoris basats en JPA.
  • Abstracció de Repositori: Proporciona una abstracció de nivell superior per a les capes d'accés a dades.

Definició de Mètodes de Consulta

Spring Data JPA permet definir mètodes de consulta a les teves interfícies de repositori. Aquests mètodes poden ser implementats automàticament per Spring Data JPA basant-se en els seus noms.

Creació de Mètodes de Consulta

  • Convencions de Nomenclatura: Spring Data JPA deriva la consulta del nom del mètode.
  • Paraules Clau: Les paraules clau comunes inclouen find, read, query, count, get.

Exemple

public interface UserRepository extends JpaRepository<User, Long> {
    List<User> findByLastName(String lastName);
    User findByEmail(String email);
}

En aquest exemple:

  • findByLastName generarà una consulta per trobar usuaris pel seu cognom.
  • findByEmail generarà una consulta per trobar un usuari pel seu correu electrònic.

Paràmetres de Consulta

  • Paràmetres Posicionals: Utilitza ?1, ?2, etc.
  • Paràmetres Nominals: Utilitza :paramName.

Exemple

@Query("SELECT u FROM User u WHERE u.lastName = ?1")
List<User> findByLastName(String lastName);

@Query("SELECT u FROM User u WHERE u.email = :email")
User findByEmail(@Param("email") String email);

Operadors de Comparació

  • Paraules Clau: Is, Equals, Between, LessThan, GreaterThan, etc.

Exemple

List<User> findByAgeGreaterThan(int age);
List<User> findByAgeBetween(int startAge, int endAge);

Mètodes de Consulta Avançats

JPQL i Consultes Natives

  • JPQL: Java Persistence Query Language, similar a SQL però opera sobre el model d'entitat.
  • Consultes Natives: Consultes SQL directes.

Exemple

@Query("SELECT u FROM User u WHERE u.status = 'ACTIVE'")
List<User> findActiveUsers();

@Query(value = "SELECT * FROM users WHERE status = 'ACTIVE'", nativeQuery = true)
List<User> findActiveUsersNative();

Paginació i Ordenació

  • Paginació: Utilitza la interfície Pageable.
  • Ordenació: Utilitza la interfície Sort.

Exemple

Page<User> findByLastName(String lastName, Pageable pageable);

List<User> findByLastName(String lastName, Sort sort);

Consultes Dinàmiques

  • Especificacions: Utilitza la interfície Specification per a consultes complexes.

Exemple

public interface UserRepository extends JpaRepository<User, Long>, JpaSpecificationExecutor<User> {
}

public class UserSpecifications {
    public static Specification<User> hasLastName(String lastName) {
        return (root, query, criteriaBuilder) -> criteriaBuilder.equal(root.get("lastName"), lastName);
    }
}

Conclusió

Spring Data JPA proporciona una manera potent i flexible de definir mètodes de consulta. Entenent els conceptes bàsics de la creació de mètodes de consulta, els paràmetres de consulta, els operadors de comparació, i conceptes avançats com JPQL, consultes natives, paginació, ordenació i consultes dinàmiques, pots gestionar eficientment l'accés a dades en les teves aplicacions Spring Boot.

Resum

  • Creació de Mètodes de Consulta: Defineix consultes basades en noms de mètodes.
  • Paràmetres de Consulta: Utilitza paràmetres posicionals o nominals.
  • Operadors de Comparació: Utilitza paraules clau per a diverses condicions.
  • JPQL i Consultes Natives: Escriu consultes personalitzades utilitzant JPQL o SQL.
  • Paginació i Ordenació: Gestiona grans conjunts de dades eficientment.
  • Consultes Dinàmiques: Utilitza especificacions per a consultes complexes.

Dominant aquests conceptes, pots aprofitar tot el potencial de Spring Data JPA en les teves aplicacions.
