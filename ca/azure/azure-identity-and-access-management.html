Introducció a l'IAM d'Azure

L'Identitat i Gestió d'Accés d'Azure (IAM) és un component crític d'Azure que t'ajuda a gestionar l'accés als recursos d'Azure de manera segura. Et permet controlar qui pot fer què dins del teu entorn d'Azure.

  • Identitat: Es refereix a la representació d'un usuari, aplicació o altra entitat que pot autenticar-se i ser autoritzada per accedir a recursos.
  • Gestió d'Accés: Implica definir i aplicar polítiques que determinen què poden fer les identitats autenticades dins del teu entorn d'Azure.

Conceptes Clau de l'IAM d'Azure

Azure Active Directory (Azure AD)

Azure AD és el servei de gestió d'identitat i accés al núvol de Microsoft. És l'eix vertebrador de l'IAM d'Azure.

  • Usuaris: Individus que tenen accés als teus recursos d'Azure.
  • Grups: Col·leccions d'usuaris que es poden gestionar com una sola entitat.
  • Principals de Servei: Identitats utilitzades per aplicacions o serveis per accedir a recursos específics d'Azure.
  • Identitats Gestionades: Identitats gestionades automàticament per serveis d'Azure per accedir a altres recursos d'Azure.

Control d'Accés Basat en Rols (RBAC)

RBAC és un sistema que proporciona una gestió d'accés granular dels recursos d'Azure.

  • Rols: Defineixen un conjunt de permisos. Exemples inclouen Propietari, Col·laborador i Lector.
  • Assignacions de Rols: Concedeixen accés associant un rol amb un usuari, grup o principal de servei.
  • Abast: Defineix el nivell en què s'aplica l'assignació de rol (per exemple, subscripció, grup de recursos o recurs).

Accés Condicional

Les polítiques d'Accés Condicional t'ajuden a aplicar controls d'accés basats en condicions específiques.

  • Condicions: Criteris que s'han de complir perquè es concedeixi l'accés (per exemple, ubicació de l'usuari, estat del dispositiu).
  • Controls: Accions que es prenen quan es compleixen les condicions (per exemple, requerir autenticació multifactor).

Configuració de l'IAM d'Azure

Creació d'Usuaris i Grups a Azure AD

Per crear un usuari a Azure AD:

az ad user create --display-name "John Doe" --user-principal-name johndoe@example.com --password "StrongPassword123!"

Per crear un grup a Azure AD:

az ad group create --display-name "Developers" --mail-nickname "developers"

Assignació de Rols Utilitzant RBAC

Per assignar el rol de Col·laborador a un usuari a nivell de subscripció:

az role assignment create --assignee johndoe@example.com --role Contributor --scope /subscriptions/{subscription-id}

Implementació de Polítiques d'Accés Condicional

Per crear una política d'accés condicional que requereixi MFA per a tots els usuaris:

  1. Navega a Azure AD > Seguretat > Accés Condicional.
  2. Crea una nova política.
  3. Defineix les condicions (per exemple, Tots els usuaris).
  4. Defineix els controls (per exemple, Requerir autenticació multifactor).
  5. Activa la política.

Funcions Avançades de l'IAM

Gestió d'Identitat Privilegiada (PIM)

PIM t'ajuda a gestionar, controlar i monitoritzar l'accés a recursos importants dins de la teva organització.

  • Accés Just-In-Time (JIT): Proporciona accés temporal als recursos.
  • Revisions d'Accés: Revisa i certifica regularment els drets d'accés.

Protecció d'Identitat

La Protecció d'Identitat utilitza l'aprenentatge automàtic per detectar i respondre a possibles riscos d'identitat.

  • Detecció de Riscos: Identifica activitats sospitoses relacionades amb identitats d'usuaris.
  • Polítiques de Risc: Respon automàticament als riscos detectats (per exemple, requerir canvi de contrasenya).

Identitats Gestionades per a Recursos d'Azure

Les identitats gestionades eliminen la necessitat que els desenvolupadors gestionin credencials.

  • Identitat Gestionada Assignada pel Sistema: Creada i gestionada automàticament per Azure.
  • Identitat Gestionada Assignada per l'Usuari: Creada com un recurs autònom d'Azure i es pot assignar a múltiples recursos.

Resum

L'IAM d'Azure és essencial per assegurar i gestionar l'accés als teus recursos d'Azure. Comprenent i implementant Azure AD, RBAC, Accés Condicional i funcions avançades com PIM i Protecció d'Identitat, pots assegurar-te que el teu entorn d'Azure sigui segur i eficient.

  • Azure AD: Gestió centralitzada d'identitats.
  • RBAC: Control d'accés granular.
  • Accés Condicional: Aplicar polítiques d'accés basades en condicions.
  • Funcions Avançades: Millorar la seguretat amb PIM, Protecció d'Identitat i Identitats Gestionades.

Dominant aquests conceptes, pots gestionar efectivament les identitats i l'accés dins del teu entorn d'Azure, assegurant-te que només els usuaris autoritzats tinguin el nivell d'accés adequat als teus recursos.
