Introducció a la Informàtica Serverless

  • Definició: La informàtica serverless et permet construir i executar aplicacions i serveis sense gestionar la infraestructura. El proveïdor de núvol provisiona, escala i gestiona automàticament la infraestructura necessària per executar el codi.
  • Beneficis:
    • Eficiència de Costos: Paga només pel temps de computació que consumeixes.
    • Escalabilitat: Escala automàticament cap amunt o cap avall segons la demanda.
    • Reducció de la Sobrecàrrega Operativa: No cal gestionar servidors o infraestructura.

Què són les Azure Functions?

  • Definició: Azure Functions és un servei de computació serverless que et permet executar codi impulsat per esdeveniments sense haver de provisionar o gestionar explícitament la infraestructura.
  • Característiques Clau:
    • Impulsat per Esdeveniments: Activat per esdeveniments com sol·licituds HTTP, temporitzadors o missatges d'altres serveis d'Azure.
    • Desenvolupament Flexible: Suporta múltiples llenguatges de programació, incloent C#, JavaScript, Python i més.
    • Seguretat Integrada: Funcions de seguretat integrades com autenticació i autorització.

Començant amb Azure Functions

  • Creant la Teva Primera Funció:
    1. Configura el Teu Entorn:
      • Instal·la les Azure Functions Core Tools.
      • Instal·la l'Azure CLI.
    2. Crea una Aplicació de Funcions: 
      az functionapp create --resource-group <ResourceGroupName> --consumption-plan-location <Region> --runtime <Runtime> --functions-version 3 --name <FunctionAppName> --storage-account <StorageAccountName>
    3. Crea una Nova Funció: 
      func init MyFunctionProj --dotnet
cd MyFunctionProj
func new --name MyHttpTrigger --template "HTTP trigger" --authlevel "anonymous"
    4. Executa la Funció Localment: 
      func start
    5. Desplega la Funció a Azure: 
      func azure functionapp publish <FunctionAppName>

Triggers i Bindings

  • Triggers: Defineixen com s'invoca una funció.
    • HTTP Trigger: Invocat per sol·licituds HTTP.
    • Timer Trigger: Invocat segons un horari.
    • Queue Trigger: Invocat quan s'afegeix un missatge a una cua d'emmagatzematge d'Azure.
  • Bindings: Connecten funcions amb altres serveis.
    • Input Bindings: Recuperen dades de fonts externes.
    • Output Bindings: Envien dades a fonts externes.

Exemple: Funció HTTP Trigger

using System.IO;
using Microsoft.AspNetCore.Mvc;
using Microsoft.Azure.WebJobs;
using Microsoft.Azure.WebJobs.Extensions.Http;
using Microsoft.AspNetCore.Http;
using Microsoft.Extensions.Logging;
using Newtonsoft.Json;

public static class MyHttpTrigger
{
    [FunctionName("MyHttpTrigger")]
    public static async Task<IActionResult> Run(
        [HttpTrigger(AuthorizationLevel.Function, "get", "post", Route = null)] HttpRequest req,
        ILogger log)
    {
        log.LogInformation("C# HTTP trigger function processed a request.");

        string name = req.Query["name"];

        string requestBody = await new StreamReader(req.Body).ReadToEndAsync();
        dynamic data = JsonConvert.DeserializeObject(requestBody);
        name = name ?? data?.name;

        return name != null
            ? (ActionResult)new OkObjectResult($"Hello, {name}")
            : new BadRequestObjectResult("Please pass a name on the query string or in the request body");
    }
}
  • Explicació:
    • La funció és activada per una sol·licitud HTTP.
    • Llegeix el paràmetre name de la cadena de consulta o del cos de la sol·licitud.
    • Retorna un missatge de salutació si es proporciona name, en cas contrari retorna un missatge de sol·licitud incorrecta.

Conceptes Avançats

  • Durable Functions: Extén Azure Functions amb fluxos de treball amb estat.
    • Orchestrator Functions: Defineixen fluxos de treball utilitzant codi.
    • Activity Functions: Realitzen tasques dins del flux de treball.
  • Escalabilitat i Rendiment:
    • Consumption Plan: Escala automàticament segons la demanda.
    • Premium Plan: Proporciona opcions millorades de rendiment i escalabilitat.
  • Seguretat:
    • Autenticació i Autorització: Integra amb Azure AD, proveïdors OAuth, etc.
    • Identitats Gestionades: Accedeix de manera segura a recursos d'Azure sense emmagatzemar credencials.

Monitorització i Resolució de Problemes

  • Application Insights: Monitoritza i diagnostica problemes amb les teves funcions.
    • Registre de Logs: Captura registres per depuració.
    • Mètriques: Fes un seguiment del rendiment i les mètriques d'ús.
    • Alertes: Configura alertes per a condicions específiques.

Conclusió

Azure Functions i la informàtica serverless ofereixen una manera poderosa de construir aplicacions escalables i impulsades per esdeveniments sense la sobrecàrrega de gestionar la infraestructura. Entenent els conceptes bàsics de crear i desplegar funcions, treballant amb triggers i bindings, i explorant conceptes avançats com Durable Functions i seguretat, pots aprofitar tot el potencial d'Azure Functions per construir solucions eficients i rendibles.
