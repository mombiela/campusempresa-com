Introducció

Desplegar una aplicació multi-capa implica configurar diferents capes (tiers) d'una aplicació, com ara la capa de presentació, la capa de lògica de negoci i la capa de dades, a Azure. Aquesta guia et portarà des dels conceptes bàsics fins a estratègies avançades de desplegament.

Conceptes Clau

  • Arquitectura Multi-Capa: Una arquitectura d'aplicació que separa diferents funcionalitats en capes.
    • Capa de Presentació: Interfície d'usuari.
    • Capa de Lògica de Negoci: Processament de l'aplicació.
    • Capa de Dades: Gestió de bases de dades.
  • Serveis d'Azure: Diversos serveis proporcionats per Azure per donar suport a aplicacions multi-capa.
    • Azure App Service: Per allotjar aplicacions web.
    • Azure SQL Database: Servei de base de dades relacional gestionat.
    • Azure Virtual Machines: Per allotjar aplicacions i serveis personalitzats.
    • Azure Load Balancer: Distribueix el tràfic de xarxa entrant entre diversos servidors.

Configuració de l'Entorn

Creació de Grups de Recursos

Els grups de recursos a Azure són contenidors lògics per als recursos desplegats a Azure.

az group create --name MyResourceGroup --location eastus

Desplegament de la Capa de Presentació

Utilitza Azure App Service per allotjar l'aplicació web.

  1. Crear un Pla de Servei d'Aplicacions:

    az appservice plan create --name MyAppServicePlan --resource-group MyResourceGroup --sku B1 --is-linux

  2. Crear una Aplicació Web:

    az webapp create --resource-group MyResourceGroup --plan MyAppServicePlan --name MyWebApp --runtime "NODE|14-lts"

Desplegament de la Capa de Lògica de Negoci

Utilitza Azure Virtual Machines o Azure Functions per a la lògica de negoci.

  1. Crear una Màquina Virtual:

    az vm create --resource-group MyResourceGroup --name MyVM --image UbuntuLTS --admin-username azureuser --generate-ssh-keys

  2. Instal·lar el Programari Necessari:

    az vm run-command invoke --command-id RunShellScript --name MyVM --resource-group MyResourceGroup --scripts "sudo apt-get update && sudo apt-get install -y nodejs"

Desplegament de la Capa de Dades

Utilitza Azure SQL Database per a la capa de dades.

  1. Crear un Servidor SQL:

    az sql server create --name myserver --resource-group MyResourceGroup --location eastus --admin-user myadmin --admin-password MyPassword123

  2. Crear una Base de Dades SQL:

    az sql db create --resource-group MyResourceGroup --server myserver --name mydatabase --service-objective S0

Connexió de les Capaes

Configuració de l'Aplicació Web per Connectar-se a la Capa de Lògica de Negoci

Actualitza la configuració de l'aplicació web per apuntar a la capa de lògica de negoci.

az webapp config appsettings set --resource-group MyResourceGroup --name MyWebApp --settings BUSINESS_LOGIC_URL=http://<VM_IP_ADDRESS>

Configuració de la Capa de Lògica de Negoci per Connectar-se a la Capa de Dades

Actualitza la capa de lògica de negoci per connectar-se a la base de dades SQL.

const sql = require('mssql');

const config = {
    user: 'myadmin',
    password: 'MyPassword123',
    server: 'myserver.database.windows.net',
    database: 'mydatabase',
    options: {
        encrypt: true
    }
};

sql.connect(config).then(pool => {
    return pool.request().query('SELECT * FROM mytable');
}).then(result => {
    console.dir(result);
}).catch(err => {
    console.error(err);
});

Estratègies Avançades de Desplegament

Ús d'Azure DevOps per a CI/CD

Configura pipelines d'integració contínua i desplegament continu utilitzant Azure DevOps.

  1. Crear un Nou Pipeline:

    • Navega a Azure DevOps.
    • Crea un nou pipeline i connecta'l al teu repositori.
    • Defineix les teves etapes de build i release.

  2. Exemple de Pipeline YAML:

    trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseNode@2
  inputs:
    version: '14.x'
- script: |
    npm install
    npm run build
  displayName: 'Build project'
- task: AzureWebApp@1
  inputs:
    azureSubscription: '<Azure Service Connection>'
    appName: 'MyWebApp'
    package: '$(Build.ArtifactStagingDirectory)/**/*.zip'

Escalabilitat de l'Aplicació

Utilitza Azure Load Balancer per distribuir el tràfic i escalar l'aplicació.

  1. Crear un Load Balancer:

    az network lb create --resource-group MyResourceGroup --name MyLoadBalancer --sku Basic --frontend-ip-name MyFrontEnd --backend-pool-name MyBackEndPool

  2. Afegir Màquines Virtuals al Load Balancer:

    az network lb address-pool address add --resource-group MyResourceGroup --lb-name MyLoadBalancer --pool-name MyBackEndPool --vnet-name MyVNet --nic-name MyVMNic

Conclusió

Desplegar una aplicació multi-capa a Azure implica configurar diferents capes de l'aplicació utilitzant diversos serveis d'Azure. Seguint els passos descrits en aquesta guia, pots desplegar una aplicació multi-capa robusta i escalable. Estratègies avançades com l'ús d'Azure DevOps per a CI/CD i Azure Load Balancer per a l'escalabilitat poden millorar encara més el teu procés de desplegament.
