Introducció a Azure Kubernetes Service (AKS)

Azure Kubernetes Service (AKS) és un servei d'orquestració de contenidors gestionat basat en el sistema Kubernetes de codi obert. AKS simplifica el desplegament d'un clúster Kubernetes gestionat a Azure descarregant la càrrega operativa a Azure. Aquesta secció cobrirà els conceptes bàsics d'AKS, els seus beneficis i conceptes clau.

  • Què és Kubernetes?

    • Una plataforma de codi obert per automatitzar el desplegament, l'escalat i les operacions de contenidors d'aplicacions.
    • Gestiona aplicacions contenidoritzades a través de múltiples hosts.
    • Proporciona una infraestructura centrada en contenidors.

  • Què és AKS?

    • Un servei Kubernetes gestionat proporcionat per Azure.
    • Simplifica la gestió del clúster Kubernetes.
    • S'integra amb serveis d'Azure per a la monitorització, l'escalat i la seguretat.

  • Beneficis d'AKS:

    • Servei Gestionat: Azure gestiona el pla de control de Kubernetes.
    • Escalabilitat: Escala fàcilment les aplicacions cap amunt o cap avall.
    • Integració: Integració perfecta amb altres serveis d'Azure.
    • Seguretat: Funcions de seguretat integrades i compliment amb els estàndards d'Azure.

Configuració d'AKS

Aquesta secció et guiarà a través de la configuració d'un clúster AKS des de zero.

Prerequisits

  • Una subscripció activa d'Azure.
  • Azure CLI instal·lat a la teva màquina local.
  • Comprensió bàsica dels conceptes de Kubernetes.

Passos per crear un clúster AKS

  1. Instal·lar Azure CLI:

    curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

  2. Iniciar sessió a Azure:

    az login

  3. Crear un Grup de Recursos:

    az group create --name myResourceGroup --location eastus

  4. Crear un clúster AKS:

    az aks create --resource-group myResourceGroup --name myAKSCluster --node-count 1 --enable-addons monitoring --generate-ssh-keys

  5. Obtenir les credencials d'AKS:

    az aks get-credentials --resource-group myResourceGroup --name myAKSCluster

  6. Verificar el clúster:

    kubectl get nodes

Desplegament d'Aplicacions a AKS

Aquesta secció cobrirà com desplegar aplicacions al teu clúster AKS utilitzant manifests de Kubernetes.

Creació d'un Desplegament

  • Exemple de YAML de Desplegament:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

  • Aplicar el Desplegament:

    kubectl apply -f nginx-deployment.yaml

  • Verificar el Desplegament:

    kubectl get deployments
kubectl get pods

Exposar l'Aplicació

  • YAML del Servei:

    apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: LoadBalancer
  selector:
    app: nginx
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

  • Aplicar el Servei:

    kubectl apply -f nginx-service.yaml

  • Obtenir l'IP Extern:

    kubectl get services

Funcionalitats Avançades d'AKS

Aquesta secció aprofundirà en funcionalitats més avançades d'AKS, com l'escalat, la monitorització i la seguretat.

Escalat d'Aplicacions

  • Autoscalador Horitzontal de Pods (HPA):

    apiVersion: autoscaling/v1
kind: HorizontalPodAutoscaler
metadata:
  name: nginx-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: nginx-deployment
  minReplicas: 1
  maxReplicas: 10
  targetCPUUtilizationPercentage: 50

  • Aplicar HPA:

    kubectl apply -f nginx-hpa.yaml

Monitorització i Registre

  • Azure Monitor per a Contenidors:

    • S'integra amb AKS per proporcionar monitorització i registre.
    • Automàticament habilitat si es especifica durant la creació del clúster.

  • Veure Mètriques:

    • Utilitza el Portal d'Azure per veure mètriques i registres.
    • Utilitza kubectl top per veure l'ús de recursos.

Millors Pràctiques de Seguretat

  • Control d'Accés Basat en Rols (RBAC):

    • Utilitza RBAC per controlar l'accés a l'API de Kubernetes.
    • Defineix rols i vinculacions de rols.

  • Polítiques de Xarxa:

    • Utilitza polítiques de xarxa per controlar el trànsit entre pods.
    • Exemple de YAML de Política de Xarxa: 
      apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-nginx
spec:
  podSelector:
    matchLabels:
      app: nginx
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: nginx
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: nginx

Conclusió

Azure Kubernetes Service (AKS) és una eina poderosa per gestionar aplicacions contenidoritzades a escala. Aprofitant AKS, pots simplificar el desplegament, la gestió i l'escalat de clústers Kubernetes. Aquest curs ha cobert els conceptes bàsics d'AKS, la configuració d'un clúster, el desplegament d'aplicacions i funcionalitats avançades com l'escalat, la monitorització i la seguretat. Amb aquest coneixement, estàs ben equipat per començar a utilitzar AKS per a les teves necessitats d'orquestració de contenidors.
