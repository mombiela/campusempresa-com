Introducció

En aquest mòdul, explorarem com construir una API sense servidor utilitzant Azure Functions. Començarem amb els conceptes bàsics i gradualment passarem a temes més avançats. Al final d'aquest mòdul, tindràs una comprensió sòlida de com crear, desplegar i gestionar APIs sense servidor a Azure.

Què és la computació sense servidor?

  • Definició: La computació sense servidor et permet construir i executar aplicacions i serveis sense pensar en els servidors. El proveïdor de núvol provisiona, escala i gestiona automàticament la infraestructura necessària per executar el codi.
  • Beneficis:
    • Sense gestió de servidors
    • Escalat automàtic
    • Preus per ús
    • Augment de la productivitat dels desenvolupadors

Començant amb Azure Functions

Prerequisits

  • Un compte actiu d'Azure
  • Comprensió bàsica de programació (preferiblement en C#, JavaScript o Python)
  • Azure CLI instal·lat

Configurant el teu entorn

  1. Instal·la Azure Functions Core Tools: 
    npm install -g azure-functions-core-tools@3
  2. Crea una nova aplicació de funcions d'Azure: 
    func init MyFunctionApp --worker-runtime node
  3. Crea una nova funció: 
    cd MyFunctionApp
func new
    Segueix les instruccions per crear una funció de desencadenador HTTP.

Exemple: Creació d'una funció de desencadenador HTTP simple

module.exports = async function (context, req) {
    context.log('HTTP trigger function processed a request.');

    const name = (req.query.name || (req.body && req.body.name));
    const responseMessage = name
        ? "Hello, " + name + ". This HTTP triggered function executed successfully."
        : "This HTTP triggered function executed successfully. Pass a name in the query string or in the request body for a personalized response.";

    context.res = {
        // status: 200, /* Per defecte és 200 */
        body: responseMessage
    };
}
  • Explicació:
    • context: Proporciona mètodes i propietats per interactuar amb el temps d'execució.
    • req: Representa la sol·licitud HTTP.
    • context.res: Representa la resposta HTTP.

Desplegant la teva funció a Azure

Passos per desplegar

  1. Inicia sessió a Azure: 
    az login
  2. Crea un grup de recursos: 
    az group create --name myResourceGroup --location eastus
  3. Crea una aplicació de funcions a Azure: 
    az functionapp create --resource-group myResourceGroup --consumption-plan-location eastus --runtime node --functions-version 3 --name myFunctionApp --storage-account <STORAGE_ACCOUNT_NAME>
  4. Desplega la teva funció: 
    func azure functionapp publish myFunctionApp

Temes avançats

Autenticació i autorització

  • Azure Active Directory (AAD): Utilitza AAD per assegurar la teva API.
  • Claus d'API: Mètode simple per assegurar la teva API.
  • Identitats gestionades: Accedeix de manera segura a altres recursos d'Azure.

Integració amb altres serveis d'Azure

  • Azure Cosmos DB: Emmagatzema i recupera dades.
  • Azure Storage: Emmagatzema fitxers i blobs.
  • Azure Event Grid: Arquitectura basada en esdeveniments.

Monitorització i registre

  • Application Insights: Monitoritza el rendiment i l'ús.
  • Azure Monitor: Solució de monitorització integral.
  • Registre: Utilitza context.log per registrar informació.

Escalat i optimització del rendiment

  • Escalat: Azure Functions escala automàticament segons la demanda.
  • Inicis en fred: Mitiga els problemes d'inici en fred utilitzant plans premium o instàncies preescalfades.
  • Optimització del rendiment: Optimitza el codi de la funció i la configuració per obtenir un millor rendiment.

Conclusió

Construir una API sense servidor a Azure és una manera poderosa de crear aplicacions escalables i rendibles. Des de la configuració del teu entorn fins al desplegament i la gestió de les teves funcions, Azure proporciona un conjunt complet d'eines i serveis per donar suport al teu procés de desenvolupament. A mesura que avancis, integrar-te amb altres serveis d'Azure i optimitzar el rendiment t'ajudarà a construir aplicacions sense servidor robustes i eficients.
