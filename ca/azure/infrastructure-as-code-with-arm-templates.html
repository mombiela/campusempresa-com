Introducció a la Infraestructura com a Codi (IaC)

  • Definició: La Infraestructura com a Codi (IaC) és el procés de gestionar i aprovisionar la infraestructura informàtica a través de fitxers de definició llegibles per màquina, en lloc de la configuració de maquinari físic o eines de configuració interactives.
  • Beneficis:
    • Consistència i repetibilitat
    • Control de versions
    • Automatització i eficiència
    • Reducció del risc d'error humà

Introducció a les Plantilles ARM

  • Definició: Les Plantilles Azure Resource Manager (ARM) són fitxers JSON que defineixen la infraestructura i configuració per a la teva solució Azure.
  • Conceptes Clau:
    • Recursos: Els components individuals que vols desplegar (per exemple, màquines virtuals, comptes d'emmagatzematge).
    • Paràmetres: Valors que pots passar a la plantilla per personalitzar el desplegament.
    • Variables: Valors que s'utilitzen dins de la plantilla per simplificar expressions complexes.
    • Sortides: Valors que es retornen després del desplegament.
    • Funcions: Mètodes predefinits per simplificar expressions de plantilla.

Estructura Bàsica d'una Plantilla ARM

  • Estructura de la Plantilla: 
    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {},
  "variables": {},
  "resources": [],
  "outputs": {}
}

Creant la Teva Primera Plantilla ARM

  • Exemple: Desplegant un Compte d'Emmagatzematge Simple 
    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Storage/storageAccounts",
      "apiVersion": "2021-04-01",
      "name": "mystorageaccount",
      "location": "[resourceGroup().location]",
      "sku": {
        "name": "Standard_LRS"
      },
      "kind": "StorageV2",
      "properties": {}
    }
  ]
}
  • Explicació:
    • type: El tipus de recurs.
    • apiVersion: La versió de l'API a utilitzar per a aquest recurs.
    • name: El nom del compte d'emmagatzematge.
    • location: La ubicació on es desplegarà el recurs.
    • sku: L'SKU del compte d'emmagatzematge.
    • kind: El tipus de compte d'emmagatzematge.
    • properties: Propietats addicionals per al recurs.

Paràmetres i Variables

  • Utilitzant Paràmetres: 
    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {
    "storageAccountType": {
      "type": "string",
      "defaultValue": "Standard_LRS",
      "allowedValues": [
        "Standard_LRS",
        "Standard_GRS",
        "Standard_ZRS",
        "Premium_LRS"
      ],
      "metadata": {
        "description": "El tipus de compte d'emmagatzematge a crear."
      }
    }
  },
  "resources": [
    {
      "type": "Microsoft.Storage/storageAccounts",
      "apiVersion": "2021-04-01",
      "name": "mystorageaccount",
      "location": "[resourceGroup().location]",
      "sku": {
        "name": "[parameters('storageAccountType')]"
      },
      "kind": "StorageV2",
      "properties": {}
    }
  ]
}
  • Explicació:
    • parameters: Defineix els paràmetres que es poden passar a la plantilla.
    • defaultValue: El valor per defecte si no se'n proporciona cap.
    • allowedValues: El conjunt de valors permesos per al paràmetre.
    • metadata: Informació addicional sobre el paràmetre.

Funcions Avançades de les Plantilles ARM

  • Plantilles Imbricades: Utilitzant plantilles enllaçades per modularitzar el teu desplegament.
  • Condicions: Desplegar recursos condicionalment basant-se en valors de paràmetres.
  • Bucles: Desplegar múltiples instàncies d'un recurs utilitzant bucles.

Exemple: Plantilles Imbricades

  • Plantilla Principal: 
    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Resources/deployments",
      "apiVersion": "2021-04-01",
      "name": "nestedTemplate",
      "properties": {
        "mode": "Incremental",
        "templateLink": {
          "uri": "https://example.com/nestedTemplate.json",
          "contentVersion": "1.0.0.0"
        }
      }
    }
  ]
}
  • Explicació:
    • templateLink: Especifica l'URI de la plantilla imbricada.

Millors Pràctiques per a les Plantilles ARM

  • Modularització: Divideix plantilles grans en plantilles més petites i reutilitzables.
  • Parametrització: Utilitza paràmetres per fer les plantilles flexibles i reutilitzables.
  • Control de Versions: Emmagatzema les plantilles en un sistema de control de versions com Git.
  • Documentació: Documenta les teves plantilles per a un manteniment i comprensió més fàcils.

Conclusió

Les Plantilles ARM proporcionen una manera potent de gestionar i automatitzar la teva infraestructura Azure. Entenent l'estructura bàsica, utilitzant paràmetres i variables, i aprofitant funcions avançades com les plantilles imbricades, pots crear desplegaments d'infraestructura robustos i mantenibles. Segueix sempre les millors pràctiques per assegurar que les teves plantilles siguin eficients, reutilitzables i fàcils de gestionar.
