Introducció a l'Emmagatzematge de Taules d'Azure

L'Emmagatzematge de Taules d'Azure és un servei que emmagatzema dades NoSQL estructurades al núvol, proporcionant un emmagatzematge de claus/atributs amb un disseny sense esquema. És ideal per emmagatzemar grans quantitats de dades estructurades i no relacionals.

Conceptes Clau

  • Base de Dades NoSQL: Una base de dades no relacional que permet models de dades flexibles.
  • Disseny sense Esquema: Sense esquema predefinit, permetent estructures de dades dinàmiques.
  • Emmagatzematge de Claus/Atributs: Les dades s'emmagatzemen com a parells clau-valor.
  • Particionament: Les dades es particionen per millorar l'escalabilitat i el rendiment.

Configuració de l'Emmagatzematge de Taules d'Azure

Per començar a utilitzar l'Emmagatzematge de Taules d'Azure, necessites un compte d'Azure i un compte d'emmagatzematge.

Passos per Crear un Compte d'Emmagatzematge

  1. Inicia sessió al Portal d'Azure: Navega a https://portal.azure.com.
  2. Crea un Compte d'Emmagatzematge:
    • Fes clic a "Crear un recurs".
    • Selecciona "Compte d'emmagatzematge".
    • Omple els detalls requerits (Subscripció, Grup de recursos, Nom del compte d'emmagatzematge, etc.).
    • Fes clic a "Revisar + crear" i després a "Crear".

Creació d'una Taula

  1. Navega al Compte d'Emmagatzematge: Ves al compte d'emmagatzematge que has creat.
  2. Crea una Taula:
    • Sota "Servei de taules", fes clic a "Taules".
    • Fes clic a "+ Taula".
    • Introdueix un nom per la teva taula i fes clic a "OK".

Treballant amb l'Emmagatzematge de Taules d'Azure

Operacions Bàsiques

L'Emmagatzematge de Taules d'Azure suporta operacions bàsiques CRUD (Crear, Llegir, Actualitzar, Eliminar).

Inserint Dades

Per inserir dades en una taula, necessites crear una entitat. Una entitat és un conjunt de propietats, similar a una fila en una base de dades relacional.

// Exemple en C#: Inserint una entitat a l'Emmagatzematge de Taules d'Azure
CloudStorageAccount storageAccount = CloudStorageAccount.Parse(connectionString);
CloudTableClient tableClient = storageAccount.CreateCloudTableClient(new TableClientConfiguration());
CloudTable table = tableClient.GetTableReference("MyTable");

CustomerEntity customer = new CustomerEntity("Smith", "John")
{
    Email = "john.smith@example.com",
    PhoneNumber = "123-456-7890"
};

TableOperation insertOperation = TableOperation.Insert(customer);
table.Execute(insertOperation);

Consultar Dades

Pots consultar dades utilitzant la classe TableQuery.

// Exemple en C#: Consultar entitats de l'Emmagatzematge de Taules d'Azure
TableQuery<CustomerEntity> query = new TableQuery<CustomerEntity>().Where(
    TableQuery.GenerateFilterCondition("PartitionKey", QueryComparisons.Equal, "Smith"));

foreach (CustomerEntity entity in table.ExecuteQuery(query))
{
    Console.WriteLine($"{entity.PartitionKey} {entity.RowKey} {entity.Email} {entity.PhoneNumber}");
}

Actualitzar Dades

Per actualitzar una entitat, la recuperes, modifiques les seves propietats, i després executes una operació d'actualització.

// Exemple en C#: Actualitzar una entitat a l'Emmagatzematge de Taules d'Azure
TableOperation retrieveOperation = TableOperation.Retrieve<CustomerEntity>("Smith", "John");
TableResult retrievedResult = table.Execute(retrieveOperation);
CustomerEntity updateEntity = (CustomerEntity)retrievedResult.Result;

if (updateEntity != null)
{
    updateEntity.Email = "new.email@example.com";
    TableOperation updateOperation = TableOperation.Replace(updateEntity);
    table.Execute(updateOperation);
}

Eliminar Dades

Per eliminar una entitat, necessites la seva clau de partició i clau de fila.

// Exemple en C#: Eliminar una entitat de l'Emmagatzematge de Taules d'Azure
TableOperation retrieveOperation = TableOperation.Retrieve<CustomerEntity>("Smith", "John");
TableResult retrievedResult = table.Execute(retrieveOperation);
CustomerEntity deleteEntity = (CustomerEntity)retrievedResult.Result;

if (deleteEntity != null)
{
    TableOperation deleteOperation = TableOperation.Delete(deleteEntity);
    table.Execute(deleteOperation);
}

Temes Avançats

Particionament i Escalabilitat

El particionament és crucial per escalar la teva aplicació. Cada entitat en una taula està identificada de manera única per una combinació de PartitionKey i RowKey.

  • PartitionKey: S'utilitza per distribuir entitats a través de múltiples nodes d'emmagatzematge.
  • RowKey: Identifica de manera única una entitat dins d'una partició.

Optimització del Rendiment

  • Operacions per Lots: Realitza múltiples operacions en una sola sol·licitud per reduir la latència.
  • Indexació: Utilitza PartitionKey i RowKey de manera efectiva per optimitzar les consultes.
  • Patrons de Disseny: Utilitza patrons com el sharding i la desnormalització per millorar el rendiment.

Seguretat

  • Signatures d'Accés Compartit (SAS): Proporciona accés limitat a les teves taules sense exposar les claus del teu compte.
  • Xifratge: Les dades poden ser xifrades en repòs i en trànsit.

Conclusió

L'Emmagatzematge de Taules d'Azure és un servei de base de dades NoSQL potent i flexible que et permet emmagatzemar grans quantitats de dades estructurades. Entenent els seus conceptes clau, configurant el teu compte d'emmagatzematge, i dominant les operacions bàsiques i avançades, pots aprofitar efectivament l'Emmagatzematge de Taules d'Azure per les teves aplicacions. Recorda considerar el particionament, l'optimització del rendiment i la seguretat per assegurar que les teves dades siguin escalables, ràpides i segures.
