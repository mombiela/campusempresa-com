Introducció

La Integració Contínua (CI) i el Desplegament Continu (CD) són pràctiques essencials en el desenvolupament de programari modern. Ajuden a automatitzar el procés d'integració de canvis de codi, proves i desplegament d'aplicacions. En aquesta secció, explorarem com implementar un pipeline CI/CD utilitzant Azure DevOps.

Conceptes Clau

  • Integració Contínua (CI): La pràctica d'integrar automàticament canvis de codi de múltiples contribuïdors en un repositori compartit diverses vegades al dia.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa les proves automatitzades a producció.
  • Azure DevOps: Un conjunt d'eines de desenvolupament proporcionades per Microsoft per donar suport a tot el cicle de vida del desenvolupament de programari.

Configuració d'Azure DevOps

  1. Crear una organització d'Azure DevOps:

    • Aneu al portal d'Azure DevOps.
    • Inicieu sessió amb el vostre compte de Microsoft.
    • Feu clic a "Create new organization" i seguiu les indicacions.

  2. Crear un projecte:

    • Un cop la vostra organització estigui configurada, feu clic a "New Project."
    • Proporcioneu un nom i una descripció per al vostre projecte.
    • Trieu la visibilitat (pública o privada) i feu clic a "Create."

Configuració de Repositoris

  • Crear un Repositori:

    • Navegueu a Repos > Files.
    • Feu clic a "New Repository."
    • Proporcioneu un nom i inicialitzeu-lo amb un fitxer README.

  • Clonar el Repositori:

    git clone https://dev.azure.com/yourorganization/yourproject/_git/yourrepository
cd yourrepository

Configuració de la Integració Contínua (CI)

  1. Crear un pipeline de build:

    • Navegueu a Pipelines > Pipelines.
    • Feu clic a "New Pipeline."
    • Seleccioneu el repositori que voleu construir.
    • Trieu la plantilla que s'adapti al vostre projecte (per exemple, ASP.NET Core, Node.js).

  2. Configurar el fitxer YAML:

    • Azure DevOps utilitza fitxers YAML per definir el pipeline de build.
    • Exemple per a una aplicació Node.js: 
      trigger:
  - main

pool:
  vmImage: 'ubuntu-latest'

steps:
  - task: UseNode@2
    inputs:
      version: '14.x'
  - script: |
      npm install
      npm test
    displayName: 'Install and Test'

  3. Executar el pipeline:

    • Deseu i executeu el pipeline.
    • Superviseu el procés de build i assegureu-vos que es completi amb èxit.

Configuració del Desplegament Continu (CD)

  1. Crear un pipeline de release:

    • Navegueu a Pipelines > Releases.
    • Feu clic a "New Pipeline."
    • Seleccioneu l'artifact de build del pipeline de CI.
    • Afegiu etapes per al desplegament (per exemple, Dev, QA, Producció).

  2. Configurar tasques de desplegament:

    • Exemple per desplegar a un Azure App Service: 
      stages:
  - stage: Deploy
    jobs:
      - job: DeployJob
        pool:
          vmImage: 'ubuntu-latest'
        steps:
          - task: AzureWebApp@1
            inputs:
              azureSubscription: 'your-azure-subscription'
              appName: 'your-app-service-name'
              package: '$(System.DefaultWorkingDirectory)/**/*.zip'

  3. Activar el release:

    • Definiu triggers per crear automàticament un release quan un build es completi.
    • Superviseu el procés de desplegament i assegureu-vos que es completi amb èxit.

Monitorització i Manteniment

  • Supervisar pipelines:

    • Utilitzeu els dashboards d'Azure DevOps per supervisar l'estat dels vostres pipelines.
    • Configureu alertes i notificacions per a fallades de build i release.

  • Mantenir pipelines:

    • Actualitzeu regularment les configuracions del vostre pipeline per adaptar-vos als canvis en el vostre projecte.
    • Optimitzeu el rendiment del pipeline emmagatzemant en memòria cau les dependències i paral·lelitzant tasques.

Conclusió

Implementar un pipeline CI/CD a Azure DevOps implica configurar repositoris, configurar pipelines de build i release, i supervisar i mantenir contínuament aquests pipelines. Automatitzant els processos d'integració i desplegament, podeu assegurar una entrega més ràpida i fiable de les vostres aplicacions de programari.
