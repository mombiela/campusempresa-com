Introducció a CI/CD

La Integració Contínua (CI) i el Desplegament Continu (CD) són pràctiques essencials en el desenvolupament de programari modern. Les pipelines de CI/CD automatitzen el procés d'integració de canvis de codi, execució de proves i desplegament d'aplicacions, assegurant una entrega de programari més ràpida i fiable.

Conceptes Clau de CI/CD

  • Integració Contínua (CI): La pràctica d'integrar automàticament els canvis de codi de múltiples contribuïdors en un repositori compartit diverses vegades al dia.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi de codi que passa les proves automatitzades a producció.
  • Pipeline: Un conjunt de processos automatitzats que impulsen el desenvolupament de programari a través de diverses etapes, des de la integració de codi fins al desplegament.
  • Build: El procés de compilar codi font en codi executable.
  • Test: Proves automatitzades per verificar la correcció del codi.
  • Deploy: El procés de lliurar l'aplicació a un entorn de producció.

Configuració de CI/CD a Azure

Azure proporciona diverses eines i serveis per implementar pipelines de CI/CD, incloent Azure DevOps i GitHub Actions.

Azure DevOps

Azure DevOps és un conjunt d'eines de desenvolupament que suporten tot el cicle de vida del desenvolupament de programari.

Creació d'una Pipeline de CI

  1. Crea un nou projecte a Azure DevOps.
  2. Configura un repositori:
    • Navega a Repos > Files.
    • Inicialitza el repositori amb el teu codi.
  3. Defineix una pipeline de build:
    • Navega a Pipelines > Builds.
    • Fes clic a "New pipeline" i selecciona el teu repositori.
    • Tria una plantilla o comença amb una feina buida.
    • Defineix els passos de build al fitxer YAML.
# azure-pipelines.yml
trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- script: dotnet build --configuration Release
  displayName: 'Build project'

- script: dotnet test --no-build --verbosity normal
  displayName: 'Run tests'

Creació d'una Pipeline de CD

  1. Defineix una pipeline de release:
    • Navega a Pipelines > Releases.
    • Fes clic a "New pipeline" i selecciona una plantilla.
    • Defineix les etapes (per exemple, Dev, QA, Producció).
  2. Afegeix artefactes:
    • Enllaça la pipeline de build com a font d'artefactes.
  3. Defineix tasques de desplegament:
    • Afegeix tasques per desplegar l'aplicació a l'entorn desitjat (per exemple, Azure App Service).
# Exemple de tasca de desplegament en YAML
- task: AzureRmWebAppDeployment@4
  inputs:
    azureSubscription: 'your-azure-subscription'
    appType: 'webApp'
    WebAppName: 'your-app-service-name'
    packageForLinux: '$(System.DefaultWorkingDirectory)/**/*.zip'

GitHub Actions

GitHub Actions és una eina d'automatització integrada amb GitHub que permet crear workflows de CI/CD.

Creació d'un Workflow de CI

  1. Crea un directori .github/workflows al teu repositori.
  2. Afegeix un fitxer de workflow:
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up .NET
      uses: actions/setup-dotnet@v1
      with:
        dotnet-version: '5.x'
    - name: Build with dotnet
      run: dotnet build --configuration Release
    - name: Test with dotnet
      run: dotnet test --no-build --verbosity normal

Creació d'un Workflow de CD

  1. Afegeix un fitxer de workflow de desplegament:
# .github/workflows/cd.yml
name: CD

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up .NET
      uses: actions/setup-dotnet@v1
      with:
        dotnet-version: '5.x'
    - name: Deploy to Azure Web App
      uses: azure/webapps-deploy@v2
      with:
        app-name: 'your-app-service-name'
        publish-profile: ${{ secrets.AZURE_WEBAPP_PUBLISH_PROFILE }}
        package: .

Pràctiques Avançades de CI/CD

  • Variables d'Entorn i Secrets: Gestiona de manera segura la informació sensible utilitzant Azure Key Vault o GitHub Secrets.
  • Pipelines Multi-Etapa: Implementa pipelines amb múltiples etapes per a diferents entorns (per exemple, Dev, QA, Producció).
  • Monitorització i Alertes: Integra eines de monitorització com Azure Monitor per seguir la salut dels teus desplegaments i configurar alertes per a fallades.
  • Estratègies de Rollback: Implementa estratègies per revertir desplegaments en cas de fallades.

Conclusió

Implementar pipelines de CI/CD a Azure utilitzant Azure DevOps o GitHub Actions pot millorar significativament el teu procés de desenvolupament de programari automatitzant la integració de codi, les proves i el desplegament. Seguint les millors pràctiques i aprofitant el conjunt d'eines robustes d'Azure, pots aconseguir una entrega de programari més ràpida i fiable.
