Introducció

La Infraestructura Global d'Azure és l'eix vertebrador dels serveis al núvol de Microsoft, proporcionant els recursos físics i virtuals necessaris per executar aplicacions i serveis a nivell mundial. Comprendre la infraestructura d'Azure és crucial per dissenyar, desplegar i gestionar aplicacions de manera efectiva.

Conceptes Clau

  • Regions: Àrees geogràfiques que contenen múltiples centres de dades.
  • Zones de Disponibilitat: Ubicacions físicament separades dins d'una regió.
  • Centres de Dades: Instal·lacions físiques que allotgen servidors i equips de xarxa.
  • Ubicacions Edge: Llocs que proporcionen accés de baixa latència als serveis d'Azure.
  • Grups de Recursos: Contenidors lògics per gestionar recursos.

Regions

Les regions d'Azure són la base de la seva infraestructura global. Cada regió és un conjunt de centres de dades desplegats dins d'una ubicació geogràfica específica.

  • Exemples de Regions d'Azure:
    • Est dels EUA
    • Oest d'Europa
    • Sud-est d'Àsia

Triar una Regió

Quan seleccioneu una regió, considereu els següents factors:

  • Proximitat als Usuaris: Trieu una regió propera a la vostra base d'usuaris per reduir la latència.
  • Requisits de Compliment: Algunes regions compleixen amb requisits reguladors específics.
  • Disponibilitat de Serveis: No tots els serveis estan disponibles a totes les regions.

Zones de Disponibilitat

Les Zones de Disponibilitat són ubicacions físiques úniques dins d'una regió d'Azure. Cada zona està composta per un o més centres de dades equipats amb energia, refrigeració i xarxa independents.

  • Beneficis de les Zones de Disponibilitat:
    • Alta Disponibilitat: Protegeix aplicacions i dades de fallades de centres de dades.
    • Aïllament de Fallades: Cada zona està aïllada per prevenir fallades en cascada.

Exemple: Desplegar una Màquina Virtual en una Zona de Disponibilitat

# Crear un grup de recursos
az group create --name myResourceGroup --location eastus

# Crear una xarxa virtual
az network vnet create --resource-group myResourceGroup --name myVnet --subnet-name mySubnet

# Crear una adreça IP pública
az network public-ip create --resource-group myResourceGroup --name myPublicIP

# Crear un grup de seguretat de xarxa
az network nsg create --resource-group myResourceGroup --name myNetworkSecurityGroup

# Crear una màquina virtual en una zona de disponibilitat específica
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --vnet-name myVnet \
  --subnet mySubnet \
  --public-ip-address myPublicIP \
  --nsg myNetworkSecurityGroup \
  --zone 1

Centres de Dades

Els centres de dades són les instal·lacions físiques que allotgen els servidors i equips de xarxa. Estan dissenyats per proporcionar alta disponibilitat i seguretat.

  • Característiques Clau:
    • Subministraments d'energia redundants
    • Sistemes de refrigeració avançats
    • Mesures de seguretat física robustes

Ubicacions Edge

Les ubicacions Edge són llocs que emmagatzemen en memòria cau contingut més a prop dels usuaris, reduint la latència i millorant el rendiment. Formen part de la Xarxa de Distribució de Continguts (CDN) d'Azure.

  • Casos d'Ús:
    • Transmissió de mitjans
    • Lliurament de contingut web
    • Acceleració d'API

Grups de Recursos

Els grups de recursos són contenidors lògics que ajuden a gestionar i organitzar els recursos d'Azure. Proporcionen una manera d'agrupament de recursos relacionats per a una gestió i facturació més fàcil.

  • Millors Pràctiques:
    • Agrupar recursos per cicle de vida i permisos.
    • Utilitzar noms significatius per als grups de recursos.
    • Aplicar etiquetes per a una millor organització i gestió de costos.

Exemple: Crear un Grup de Recursos

# Crear un grup de recursos
az group create --name myResourceGroup --location eastus

Resum

La Infraestructura Global d'Azure és un sistema complex i robust dissenyat per proporcionar alta disponibilitat, escalabilitat i seguretat per a aplicacions al núvol. Comprenent les regions, zones de disponibilitat, centres de dades, ubicacions edge i grups de recursos, podeu prendre decisions informades sobre on i com desplegar les vostres aplicacions a Azure. Aquest coneixement és essencial per optimitzar el rendiment, assegurar el compliment i gestionar els costos de manera efectiva.
