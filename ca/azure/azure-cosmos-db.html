Introducció a Azure Cosmos DB

Azure Cosmos DB és un servei de base de dades distribuïda globalment i multi-model dissenyat per proporcionar alta disponibilitat, baixa latència i escalabilitat. Admet múltiples models de dades com ara document, clau-valor, gràfic i família de columnes.

Característiques Clau

  • Distribució Global: Distribueix dades a través de múltiples regions d'Azure amb un sol clic.
  • Suport Multi-Model: Treballa amb models de dades de document, clau-valor, gràfic i família de columnes.
  • Escalabilitat Elàstica: Escala el rendiment i l'emmagatzematge de manera independent i sota demanda.
  • Baixa Latència: Garanteix temps de resposta de mil·lisegons d'una sola xifra.
  • SLA Completes: Ofereix SLA per a rendiment, latència, disponibilitat i consistència.

Configuració d'Azure Cosmos DB

Creació d'un Compte de Cosmos DB

  1. Navega al Portal d'Azure: Ves al portal d'Azure a portal.azure.com.
  2. Crea un Recurs: Fes clic a "Crea un recurs" i cerca "Azure Cosmos DB".
  3. Configura el Compte: Tria l'API (Core (SQL), MongoDB, Cassandra, Gremlin, Table), estableix el nom del compte, el grup de recursos i la ubicació.
  4. Revisa i Crea: Revisa la configuració i fes clic a "Crea".

Exemple: Creació d'un Compte de Cosmos DB utilitzant Azure CLI

# Crea un grup de recursos
az group create --name myResourceGroup --location eastus

# Crea un compte de Cosmos DB amb l'API SQL
az cosmosdb create --name myCosmosDBAccount --resource-group myResourceGroup --kind GlobalDocumentDB

Treballant amb Models de Dades

Model de Document (API SQL)

El model de document emmagatzema dades en format JSON i s'accedeix utilitzant consultes similars a SQL.

Exemple: Inserció d'un Document

{
  "id": "1",
  "name": "John Doe",
  "email": "john.doe@example.com",
  "age": 30
}

Exemple: Consulta de Documents

SELECT * FROM c WHERE c.age > 25

Model Clau-Valor (API Table)

El model clau-valor emmagatzema dades com a parells clau-valor, similar a Azure Table Storage.

Exemple: Inserció d'un Parell Clau-Valor

{
  "PartitionKey": "Users",
  "RowKey": "1",
  "Name": "John Doe",
  "Email": "john.doe@example.com"
}

Model de Gràfic (API Gremlin)

El model de gràfic emmagatzema dades com a vèrtexs i arestes, adequat per a consultes basades en gràfics.

Exemple: Afegir un Vèrtex

g.addV('person').property('id', '1').property('name', 'John Doe')

Exemple: Afegir una Aresta

g.V('1').addE('knows').to(g.V('2'))

Nivells de Consistència

Azure Cosmos DB ofereix cinc nivells de consistència per equilibrar entre consistència i rendiment:

| Nivell de Consistència | Descripció | |------------------------|------------| | Fort | Garanteix la màxima consistència assegurant que les lectures sempre retornin l'escriptura més recent. | | Estancament Limitat | Garanteix que les lectures es retarden respecte a les escriptures per un interval de temps o nombre de versions especificat. | | Sessió | Garanteix consistència dins d'una sola sessió. | | Prefix Consistent | Garanteix que les lectures mai vegin escriptures fora de l'ordre. | | Eventual | Garanteix consistència eventual amb la latència més baixa. |

Característiques Avançades

Particionament

El particionament ajuda a distribuir dades a través de múltiples servidors per assegurar escalabilitat i rendiment.

Exemple: Establir una Clau de Partició

{
  "id": "1",
  "name": "John Doe",
  "email": "john.doe@example.com",
  "age": 30,
  "partitionKey": "email"
}

Feed de Canvis

El Feed de Canvis et permet escoltar els canvis (insercions i actualitzacions) en el teu contenidor de Cosmos DB.

Exemple: Utilitzar Feed de Canvis amb Azure Functions

public static class CosmosDBTrigger
{
    [FunctionName("CosmosDBTrigger")]
    public static void Run([CosmosDBTrigger(
        databaseName: "myDatabase",
        collectionName: "myCollection",
        ConnectionStringSetting = "AzureWebJobsCosmosDBConnectionString",
        LeaseCollectionName = "leases")] IReadOnlyList<Document> input, ILogger log)
    {
        if (input != null && input.Count > 0)
        {
            log.LogInformation("Documents modified " + input.Count);
            log.LogInformation("First document Id " + input[0].Id);
        }
    }
}

Conclusió

Azure Cosmos DB és un servei de base de dades potent, distribuït globalment, que admet múltiples models de dades i ofereix alta disponibilitat, baixa latència i escalabilitat elàstica. Comprenent les seves característiques clau, configurant un compte, treballant amb diferents models de dades i aprofitant característiques avançades com el particionament i el feed de canvis, pots construir aplicacions robustes i escalables a Azure Cosmos DB.
