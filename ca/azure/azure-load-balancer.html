Introducció a Azure Load Balancer

Azure Load Balancer és un equilibrador de càrrega de capa 4 (TCP, UDP) que distribueix el trànsit de xarxa entrant entre diverses màquines virtuals (VMs) o serveis. Assegura una alta disponibilitat i fiabilitat distribuint el trànsit de manera eficient.

Conceptes Clau

  • Equilibrament de Càrrega: Distribuir el trànsit entrant entre diversos servidors per assegurar que cap servidor individual es vegi aclaparat.
  • Alta Disponibilitat: Assegurar que els serveis estiguin disponibles fins i tot si un o més servidors fallen.
  • Escalabilitat: Capacitat per gestionar una càrrega augmentada afegint més servidors.

Tipus d'Azure Load Balancers

Azure ofereix dos tipus principals d'equilibradors de càrrega:

Equilibrador de Càrrega Bàsic

  • Cas d'Ús: Adequat per a aplicacions a petita escala o entorns de desenvolupament/prova.
  • Característiques:
    • Limitat a un únic conjunt de disponibilitat o conjunt d'escala de màquines virtuals.
    • Sense SLA (Acord de Nivell de Servei) per a l'Equilibrador de Càrrega Bàsic.
    • Limitat a 300 punts finals.

Equilibrador de Càrrega Estàndard

  • Cas d'Ús: Adequat per a entorns de producció que requereixen alta disponibilitat i escalabilitat.
  • Característiques:
    • Suporta múltiples zones de disponibilitat.
    • Proporciona un SLA més alt (99,99%).
    • Suporta fins a 1000 punts finals.
    • Diagnòstics i mètriques avançades.

| Característica | Equilibrador de Càrrega Bàsic | Equilibrador de Càrrega Estàndard | |----------------------------|-------------------------------|-----------------------------------| | SLA | Cap | 99,99% | | Zones de Disponibilitat | No | Sí | | Punts Finals Màxims | 300 | 1000 | | Diagnòstics i Mètriques | Bàsic | Avançat |

Configuració d'un Azure Load Balancer

Prerequisits

  • Un compte d'Azure.
  • Una xarxa virtual amb almenys dues màquines virtuals.

Guia Pas a Pas

  1. Crear un Equilibrador de Càrrega:

    • Navega al portal d'Azure.
    • Selecciona "Crear un recurs" > "Xarxes" > "Equilibrador de Càrrega".
    • Tria la subscripció, grup de recursos i regió.
    • Selecciona el tipus (Bàsic o Estàndard) i SKU (Públic o Intern).

  2. Configurar IP Frontal:

    • A la configuració de l'Equilibrador de Càrrega, selecciona "Configuració d'IP Frontal".
    • Afegeix una nova configuració d'IP frontal i assigna una adreça IP pública o privada.

  3. Crear un Grup de Backend:

    • Selecciona "Grups de backend" i afegeix un nou grup de backend.
    • Afegeix les màquines virtuals que rebran el trànsit.

  4. Configurar Probes de Salut:

    • Selecciona "Probes de salut" i afegeix un nou probe.
    • Configura el probe amb protocol (TCP/HTTP), port i configuracions d'interval.

  5. Crear Regles d'Equilibrament de Càrrega:

    • Selecciona "Regles d'equilibrament de càrrega" i afegeix una nova regla.
    • Configura la regla amb IP frontal, grup de backend, protocol, port i configuracions de persistència de sessió.

Exemple: Crear un Equilibrador de Càrrega amb Azure CLI

# Crear un grup de recursos
az group create --name myResourceGroup --location eastus

# Crear una adreça IP pública
az network public-ip create --resource-group myResourceGroup --name myPublicIP

# Crear un equilibrador de càrrega
az network lb create --resource-group myResourceGroup --name myLoadBalancer --public-ip-address myPublicIP --frontend-ip-name myFrontEnd --backend-pool-name myBackEndPool

# Crear un probe de salut
az network lb probe create --resource-group myResourceGroup --lb-name myLoadBalancer --name myHealthProbe --protocol tcp --port 80

# Crear una regla d'equilibrament de càrrega
az network lb rule create --resource-group myResourceGroup --lb-name myLoadBalancer --name myLoadBalancingRule --protocol tcp --frontend-port 80 --backend-port 80 --frontend-ip-name myFrontEnd --backend-pool-name myBackEndPool --probe-name myHealthProbe

Configuració Avançada

Autoscaling amb Conjunts d'Escala de Màquines Virtuals

  • Conjunts d'Escala de Màquines Virtuals: Augmenta o disminueix automàticament el nombre d'instàncies de VM segons la demanda.
  • Integració amb l'Equilibrador de Càrrega: Afegeix o elimina automàticament VMs del grup de backend.

Diagnòstics i Monitoratge

  • Azure Monitor: Proporciona mètriques i registres per als equilibradors de càrrega.
  • Network Watcher: Ajuda a diagnosticar i solucionar problemes de xarxa.

Consideracions de Seguretat

  • Grups de Seguretat de Xarxa (NSGs): Controlen el trànsit entrant i sortint cap a les VMs.
  • Protecció DDoS: Protegeix contra atacs de Denegació de Servei Distribuïda.

Conclusió

Azure Load Balancer és una eina poderosa per assegurar l'alta disponibilitat, fiabilitat i escalabilitat de les teves aplicacions. Comprenent els diferents tipus d'equilibradors de càrrega, configurant-los correctament i aprofitant les característiques avançades com l'autoscaling i els diagnòstics, pots optimitzar el rendiment i la resiliència dels teus serveis a Azure.
