Introducció a l'Emmagatzematge de Blobs

Azure Blob Storage és la solució d'emmagatzematge d'objectes de Microsoft per al núvol. L'emmagatzematge de blobs està optimitzat per emmagatzemar grans quantitats de dades no estructurades, com ara dades de text o binàries.

Conceptes Clau

  • Blob: Un objecte binari gran que pot emmagatzemar dades de text o binàries.
  • Contenidor: Un contenidor organitza un conjunt de blobs, similar a un directori en un sistema de fitxers.
  • Compte d'emmagatzematge: Un compte d'emmagatzematge proporciona un espai de noms únic a Azure per a les teves dades.

Tipus de Blobs

  • Block Blobs: Utilitzats per emmagatzemar dades de text i binàries, ideals per a fitxers com documents i fitxers multimèdia.
  • Append Blobs: Optimitzats per a operacions d'addició, comunament utilitzats per a registres.
  • Page Blobs: Utilitzats per a operacions de lectura/escriptura aleatòries, adequats per a fitxers de disc dur virtual (VHD).

Configuració de l'Emmagatzematge de Blobs

Creació d'un Compte d'Emmagatzematge

  1. Navega al Portal d'Azure: Ves a Azure Portal.
  2. Crea un nou recurs: Fes clic a "Create a resource" i selecciona "Storage account".
  3. Configura el compte d'emmagatzematge:
    • Subscripció: Tria la teva subscripció.
    • Grup de recursos: Crea un nou grup de recursos o utilitza'n un d'existent.
    • Nom del compte d'emmagatzematge: Proporciona un nom únic.
    • Regió: Selecciona la regió més propera als teus usuaris.
    • Rendiment: Tria entre Standard i Premium.
    • Replicació: Selecciona l'estratègia de replicació (LRS, GRS, RA-GRS, etc.).

Creació d'un Contenidor

  1. Navega al teu compte d'emmagatzematge: Al Portal d'Azure, ves al teu compte d'emmagatzematge.
  2. Crea un contenidor: A la secció "Blob service", fes clic a "Containers" i després a "+ Container".
  3. Configura el contenidor:
    • Nom: Proporciona un nom per al teu contenidor.
    • Nivell d'accés públic: Tria el nivell d'accés (Privat, Blob, Contenidor).

Treballant amb Blobs

Pujant Blobs

Pots pujar blobs utilitzant el Portal d'Azure, Azure CLI o SDKs.

Utilitzant el Portal d'Azure

  1. Navega al teu contenidor: Al Portal d'Azure, ves al teu contenidor.
  2. Puja un blob: Fes clic a "Upload" i selecciona el fitxer que vols pujar.

Utilitzant Azure CLI

# Puja un fitxer a un contenidor
az storage blob upload --account-name <storage_account_name> --container-name <container_name> --name <blob_name> --file <file_path>

Utilitzant Azure SDK per a Python

from azure.storage.blob import BlobServiceClient

# Inicialitza una connexió al servei de Blobs
blob_service_client = BlobServiceClient.from_connection_string("<your_connection_string>")

# Obté una referència a un contenidor
container_client = blob_service_client.get_container_client("<container_name>")

# Puja un blob
with open("<file_path>", "rb") as data:
    container_client.upload_blob(name="<blob_name>", data=data)

Descarregant Blobs

Utilitzant Azure CLI

# Descarrega un blob d'un contenidor
az storage blob download --account-name <storage_account_name> --container-name <container_name> --name <blob_name> --file <download_path>

Utilitzant Azure SDK per a Python

# Descarrega un blob
blob_client = container_client.get_blob_client("<blob_name>")
with open("<download_path>", "wb") as download_file:
    download_file.write(blob_client.download_blob().readall())

Funcions Avançades de l'Emmagatzematge de Blobs

Nivells d'Emmagatzematge de Blobs

Azure Blob Storage ofereix diferents nivells d'accés per optimitzar els costos basats en els patrons d'accés a les dades:

  • Hot: Optimitzat per a dades que s'accedeixen freqüentment.
  • Cool: Optimitzat per a dades que s'accedeixen infreqüentment i s'emmagatzemen durant almenys 30 dies.
  • Archive: Optimitzat per a dades que s'accedeixen rarament i s'emmagatzemen durant almenys 180 dies.

Gestió del Cicle de Vida

Azure Blob Storage admet polítiques de gestió del cicle de vida per automatitzar la transició de blobs entre nivells d'accés i l'eliminació de blobs.

Exemple de Política

{
  "rules": [
    {
      "name": "move-to-cool",
      "enabled": true,
      "definition": {
        "actions": {
          "baseBlob": {
            "tierToCool": {
              "daysAfterModificationGreaterThan": 30
            }
          }
        },
        "filters": {
          "blobTypes": ["blockBlob"]
        }
      }
    }
  ]
}

Funcions de Seguretat

  • Signatures d'Accés Compartit (SAS): Proporciona accés limitat als recursos del teu compte d'emmagatzematge sense exposar la teva clau de compte.
  • Integració amb Azure Active Directory (AAD): Et permet utilitzar AAD per autoritzar sol·licituds a l'emmagatzematge de blobs.

Conclusió

Azure Blob Storage és una solució versàtil i escalable per emmagatzemar dades no estructurades al núvol. Comprenent els seus conceptes clau, configurant comptes d'emmagatzematge i contenidors, i aprofitant funcions avançades com els nivells d'emmagatzematge i la gestió del cicle de vida, pots gestionar eficientment les teves necessitats d'emmagatzematge de dades. Tant si ets un usuari principiant com avançat, Azure Blob Storage ofereix les eines i la flexibilitat per satisfer els teus requisits.
