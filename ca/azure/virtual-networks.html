Introducció a les Xarxes Virtuals (VNet)

Les Xarxes Virtuals (VNets) són blocs de construcció fonamentals per a la teva xarxa privada a Azure. Permeten que molts tipus de recursos d'Azure, com ara Màquines Virtuals d'Azure (VMs), es comuniquin de manera segura entre si, amb Internet i amb xarxes locals.

Conceptes Clau

  • Aïllament i Segmentació: Les VNets proporcionen aïllament i segmentació dins de l'entorn d'Azure.
  • Subxarxes: Les VNets es poden dividir en subxarxes per segmentar encara més la xarxa.
  • Adreçament IP: Les VNets utilitzen adreçament IP privat.
  • Grups de Seguretat de Xarxa (NSGs): Controlen el trànsit entrant i sortint cap a les interfícies de xarxa (NIC), VMs i subxarxes.
  • Peering: Connecta VNets entre si.
  • Passarel·la VPN: Connecta VNets amb xarxes locals.

Creació d'una Xarxa Virtual

Crear una VNet és el primer pas per configurar la teva infraestructura de xarxa a Azure.

Passos per Crear una VNet

  1. Navega al Portal d'Azure: Ves al portal d'Azure a https://portal.azure.com.

  2. Crea un Recurs: Fes clic a "Crear un recurs" i cerca "Xarxa Virtual".

  3. Configura els Paràmetres Bàsics:

    • Nom: Proporciona un nom per a la teva VNet.
    • Espai d'Adreces: Defineix el rang d'adreces IP (per exemple, 10.0.0.0/16).
    • Subscripció: Selecciona la teva subscripció d'Azure.
    • Grup de Recursos: Tria un grup de recursos existent o crea'n un de nou.
    • Ubicació: Selecciona la regió d'Azure.

  4. Configura les Subxarxes:

    • Nom de la Subxarxa: Proporciona un nom per a la teva subxarxa.
    • Rang d'Adreces de la Subxarxa: Defineix el rang d'adreces IP per a la subxarxa (per exemple, 10.0.1.0/24).

  5. Revisa i Crea: Revisa els teus paràmetres i fes clic a "Crear".

Exemple de Codi: Creació d'una VNet utilitzant Azure CLI

# Crea un grup de recursos
az group create --name MyResourceGroup --location eastus

# Crea una xarxa virtual
az network vnet create \
  --name MyVNet \
  --resource-group MyResourceGroup \
  --address-prefix 10.0.0.0/16 \
  --subnet-name MySubnet \
  --subnet-prefix 10.0.1.0/24

Subxarxes

Les subxarxes et permeten segmentar la VNet en seccions més petites i manejables.

Conceptes Clau

  • Rang d'Adreces de la Subxarxa: Cada subxarxa ha de tenir un rang d'adreces únic dins de la VNet.
  • Grups de Seguretat de Xarxa (NSGs): Aplica NSGs a les subxarxes per controlar el trànsit.

Exemple de Codi: Afegir una Subxarxa utilitzant Azure CLI

# Afegeix una subxarxa a una VNet existent
az network vnet subnet create \
  --address-prefix 10.0.2.0/24 \
  --name MySecondSubnet \
  --resource-group MyResourceGroup \
  --vnet-name MyVNet

Grups de Seguretat de Xarxa (NSGs)

Els NSGs s'utilitzen per filtrar el trànsit de xarxa cap a i des de recursos d'Azure en una xarxa virtual d'Azure.

Conceptes Clau

  • Regles Entrants i Sortints: Defineix regles per permetre o denegar el trànsit.
  • Associacions: Els NSGs es poden associar amb subxarxes o interfícies de xarxa individuals.

Exemple de Codi: Creació d'un NSG i Afegir Regles

# Crea un NSG
az network nsg create \
  --resource-group MyResourceGroup \
  --name MyNSG

# Crea una regla de seguretat entrant
az network nsg rule create \
  --resource-group MyResourceGroup \
  --nsg-name MyNSG \
  --name AllowHTTP \
  --protocol tcp \
  --priority 1000 \
  --destination-port-range 80 \
  --access allow

Peering de VNet

El peering de VNet connecta dues VNets, permetent que els recursos de cada VNet es comuniquin entre si.

Conceptes Clau

  • Baixa Latència: Proporciona connectivitat de baixa latència i ample de banda alt.
  • Peering Transitiu: No és compatible; cada VNet ha de ser peering directament.

Exemple de Codi: Creació d'un Peering de VNet

# Peering de VNet1 amb VNet2
az network vnet peering create \
  --name VNet1ToVNet2 \
  --resource-group MyResourceGroup \
  --vnet-name VNet1 \
  --remote-vnet /subscriptions/{subscription-id}/resourceGroups/MyResourceGroup/providers/Microsoft.Network/virtualNetworks/VNet2 \
  --allow-vnet-access

Passarel·la VPN

La Passarel·la VPN s'utilitza per enviar trànsit xifrat entre una VNet d'Azure i una ubicació local a través d'Internet públic.

Conceptes Clau

  • Tipus de Passarel·la: Basada en polítiques i basada en rutes.
  • Tipus de Connexió: Site-to-Site, Point-to-Site i VNet-to-VNet.

Exemple de Codi: Creació d'una Passarel·la VPN

# Crea una passarel·la VPN
az network vnet-gateway create \
  --resource-group MyResourceGroup \
  --name MyVPNGateway \
  --public-ip-address MyGatewayPublicIP \
  --vnet MyVNet \
  --gateway-type Vpn \
  --vpn-type RouteBased \
  --sku VpnGw1 \
  --no-wait

Conclusió

Les Xarxes Virtuals (VNets) són essencials per crear una infraestructura de xarxa segura i escalable a Azure. Entenent i utilitzant VNets, subxarxes, NSGs, peering de VNet i passarel·les VPN, pots gestionar i assegurar efectivament els teus recursos d'Azure. Aquest coneixement fonamental és crucial tant per a principiants com per a usuaris avançats per construir solucions robustes al núvol.
