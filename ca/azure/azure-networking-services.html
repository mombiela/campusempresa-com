Introducció a les Xarxes d'Azure

Els Serveis de Xarxa d'Azure proporcionen la infraestructura i les eines necessàries per connectar i gestionar els teus recursos al núvol. Entendre aquests serveis és crucial per construir solucions al núvol segures, escalables i eficients.

Serveis Clau de Xarxa d'Azure

  • Virtual Network (VNet)
  • Azure Load Balancer
  • Azure Application Gateway
  • Azure VPN Gateway
  • Azure ExpressRoute
  • Azure DNS
  • Azure Traffic Manager

Virtual Network (VNet)

Azure Virtual Network (VNet) és el bloc fonamental per a la teva xarxa privada a Azure. VNet permet que molts tipus de recursos d'Azure, com ara Azure Virtual Machines (VMs), es comuniquin de manera segura entre ells, amb internet i amb xarxes locals.

Conceptes Clau

  • Subxarxes: Divideix el teu VNet en múltiples subxarxes per a una millor organització i seguretat.
  • Grups de Seguretat de Xarxa (NSGs): Controlen el trànsit entrant i sortint a les interfícies de xarxa (NIC), VMs i subxarxes.
  • Peering: Connecta VNets entre ells dins de la mateixa regió o entre regions.

Exemple: Creació d'un VNet

az network vnet create \
  --name MyVNet \
  --resource-group MyResourceGroup \
  --address-prefix 10.0.0.0/16 \
  --subnet-name MySubnet \
  --subnet-prefix 10.0.1.0/24

Aquesta comanda crea un VNet anomenat MyVNet amb un espai d'adreces de 10.0.0.0/16 i una subxarxa anomenada MySubnet amb un rang d'adreces de 10.0.1.0/24.

Azure Load Balancer

Azure Load Balancer distribueix el trànsit de xarxa entrant entre múltiples VMs per assegurar que cap VM es vegi sobrecarregada.

Conceptes Clau

  • Public Load Balancer: Equilibra el trànsit d'internet cap a les VMs.
  • Internal Load Balancer: Equilibra el trànsit dins d'un VNet.
  • Health Probes: Monitoritzen l'estat de les VMs per assegurar que el trànsit només es dirigeix a instàncies saludables.

Exemple: Creació d'un Load Balancer

az network lb create \
  --resource-group MyResourceGroup \
  --name MyLoadBalancer \
  --frontend-ip-name MyFrontEndPool \
  --backend-pool-name MyBackEndPool \
  --sku Basic

Aquesta comanda crea un load balancer bàsic anomenat MyLoadBalancer amb pools de frontend i backend.

Azure Application Gateway

Azure Application Gateway és un equilibrador de càrrega de trànsit web que et permet gestionar el trànsit cap a les teves aplicacions web.

Conceptes Clau

  • Ruting basat en URL: Dirigeix el trànsit basat en els camins d'URL.
  • Terminació SSL: Desxifra el trànsit SSL al gateway.
  • Tallafocs d'Aplicacions Web (WAF): Protegeix les teves aplicacions web de les amenaces comunes.

Exemple: Creació d'un Application Gateway

az network application-gateway create \
  --name MyAppGateway \
  --resource-group MyResourceGroup \
  --vnet-name MyVNet \
  --subnet MySubnet \
  --capacity 2 \
  --sku WAF_v2 \
  --http-settings-cookie-based-affinity Enabled \
  --frontend-port 80 \
  --routing-rule-type Basic

Aquesta comanda crea un application gateway anomenat MyAppGateway amb WAF habilitat i regles de ruting bàsiques.

Azure VPN Gateway

Azure VPN Gateway connecta les teves xarxes locals a Azure a través de VPNs de lloc a lloc.

Conceptes Clau

  • VPN de Lloc a Lloc: Connecta la teva xarxa local a Azure.
  • VPN de Punt a Lloc: Connecta dispositius individuals a la teva xarxa d'Azure.
  • ExpressRoute: Proporciona una connexió privada a Azure.

Exemple: Creació d'un VPN Gateway

az network vnet-gateway create \
  --resource-group MyResourceGroup \
  --name MyVPNGateway \
  --public-ip-address MyGatewayIP \
  --vnet MyVNet \
  --gateway-type Vpn \
  --vpn-type RouteBased \
  --sku VpnGw1 \
  --no-wait

Aquesta comanda crea un VPN gateway anomenat MyVPNGateway amb un tipus de VPN basat en rutes.

Azure ExpressRoute

Azure ExpressRoute proporciona una connexió privada entre la teva infraestructura local i els centres de dades d'Azure.

Conceptes Clau

  • Peering Privat: Connecta als serveis d'Azure utilitzant adreces IP privades.
  • Peering Públic: Connecta als serveis d'Azure utilitzant adreces IP públiques.
  • Peering de Microsoft: Connecta als serveis de Microsoft com Office 365.

Exemple: Creació d'un Circuit ExpressRoute

az network express-route create \
  --name MyExpressRoute \
  --resource-group MyResourceGroup \
  --bandwidth 200 \
  --peering-location "Silicon Valley" \
  --provider "Equinix" \
  --sku-family MeteredData \
  --sku-tier Standard

Aquesta comanda crea un circuit ExpressRoute anomenat MyExpressRoute amb una amplada de banda de 200 Mbps.

Azure DNS

Azure DNS és un servei d'allotjament per a dominis DNS, proporcionant resolució de noms utilitzant la infraestructura de Microsoft Azure.

Conceptes Clau

  • Zones DNS: Contenidors per a registres DNS.
  • Registres DNS: Mapegen noms de domini a adreces IP.

Exemple: Creació d'una Zona DNS

az network dns zone create \
  --resource-group MyResourceGroup \
  --name mydomain.com

Aquesta comanda crea una zona DNS anomenada mydomain.com.

Azure Traffic Manager

Azure Traffic Manager és un equilibrador de càrrega de trànsit basat en DNS que et permet distribuir el trànsit de manera òptima als serveis a través de regions globals d'Azure.

Conceptes Clau

  • Mètodes de Ruting de Trànsit: Rendiment, Prioritat, Ponderat, Geogràfic, Multivalor, Subxarxa.
  • Endpoints: Representen les destinacions per al trànsit.

Exemple: Creació d'un Perfil de Traffic Manager

az network traffic-manager profile create \
  --name MyTrafficManagerProfile \
  --resource-group MyResourceGroup \
  --routing-method Performance \
  --unique-dns-name myapptrafficmanager \
  --ttl 30 \
  --protocol HTTP \
  --port 80 \
  --path "/"

Aquesta comanda crea un perfil de Traffic Manager anomenat MyTrafficManagerProfile amb ruting basat en rendiment.

Conclusió

Els Serveis de Xarxa d'Azure proporcionen una suite completa d'eines per connectar, gestionar i assegurar els teus recursos al núvol. Des de Xarxes Virtuals i Equilibradors de Càrrega fins a VPN Gateways i Traffic Managers, entendre aquests serveis és essencial per construir solucions al núvol robustes i escalables. En dominar aquestes eines, pots assegurar que les teves aplicacions siguin altament disponibles, segures i rendibles.
