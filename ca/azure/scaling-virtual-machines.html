Introducció

Escalar màquines virtuals (VMs) a Azure és una habilitat crucial per gestionar càrregues de treball de manera eficient. Aquest tema cobrirà els conceptes bàsics d'escalar VMs, diferents estratègies d'escalat i tècniques avançades per assegurar un rendiment òptim i una gestió de costos adequada.

Conceptes Clau

  • Escalar Cap Amunt vs. Escalar Cap a Fora: Entendre la diferència entre augmentar la mida d'una VM (escalar cap amunt) i augmentar el nombre de VMs (escalar cap a fora).
  • Azure VM Scale Sets: Un servei que et permet crear i gestionar un grup de VMs equilibrades en càrrega.
  • Autoscaling: Ajustar automàticament el nombre d'instàncies de VM basant-se en la demanda.
  • Escalat Manual: Ajustar manualment el nombre d'instàncies de VM.

Escalar Cap Amunt vs. Escalar Cap a Fora

Escalar Cap Amunt

  • Definició: Augmentar la mida (CPU, memòria) d'una VM existent.
  • Cas d'Ús: Adequat per a aplicacions que requereixen més recursos però no es poden distribuir entre múltiples VMs.
  • Exemple: 
    Suposem que tens una VM amb 2 CPUs i 4GB de RAM. Escalar cap amunt podria implicar actualitzar a una VM amb 4 CPUs i 8GB de RAM.

Escalar Cap a Fora

  • Definició: Augmentar el nombre d'instàncies de VM.
  • Cas d'Ús: Ideal per a aplicacions que es poden distribuir entre múltiples VMs, com ara servidors web.
  • Exemple: 
    Si tens una VM gestionant el trànsit web, escalar cap a fora podria implicar afegir més VMs per distribuir la càrrega.

Azure VM Scale Sets

  • Definició: Els Azure VM Scale Sets et permeten crear i gestionar un grup de VMs idèntiques i equilibrades en càrrega.
  • Beneficis:
    • Fàcil de gestionar i escalar.
    • Integrat amb Azure Load Balancer.
    • Suporta autoscaling.
  • Exemple: 
    Crear un VM Scale Set utilitzant Azure CLI:
```bash
az vmss create --resource-group myResourceGroup --name myScaleSet --image UbuntuLTS --upgrade-policy-mode automatic --admin-username azureuser --generate-ssh-keys

Autoscaling

  • Definició: Ajusta automàticament el nombre d'instàncies de VM basant-se en regles i mètriques predefinides.
  • Beneficis:
    • Cost-eficient: Només pagues pel que utilitzes.
    • Rendiment: Assegura que la teva aplicació pugui gestionar càrregues variables.
  • Exemple: 
    Configurar autoscaling per a un VM Scale Set:
```bash
az monitor autoscale create --resource-group myResourceGroup --resource myScaleSet --resource-type Microsoft.Compute/virtualMachineScaleSets --name myAutoscale --min-count 1 --max-count 10 --count 1
az monitor autoscale rule create --resource-group myResourceGroup --autoscale-name myAutoscale --condition "Percentage CPU > 75 avg 5m" --scale out 1
az monitor autoscale rule create --resource-group myResourceGroup --autoscale-name myAutoscale --condition "Percentage CPU < 25 avg 5m" --scale in 1

Escalat Manual

  • Definició: Ajustar manualment el nombre d'instàncies de VM.
  • Beneficis:
    • Control total sobre les decisions d'escalat.
    • Útil per a càrregues de treball predictibles.
  • Exemple: 
    Escalar manualment un VM Scale Set utilitzant Azure CLI:
```bash
az vmss scale --resource-group myResourceGroup --name myScaleSet --new-capacity 5

Resum

Escalar màquines virtuals a Azure implica entendre la diferència entre escalar cap amunt i escalar cap a fora, utilitzar Azure VM Scale Sets i implementar autoscaling per a l'eficiència. L'escalat manual proporciona control per a càrregues de treball predictibles. Dominar aquestes tècniques assegura que les teves aplicacions puguin gestionar càrregues variables de manera eficient mentre optimitzes els costos.
