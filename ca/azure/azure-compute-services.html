Serveis de Còmput d'Azure
Serveis d'Emmagatzematge d'Azure ►

Els Serveis de Còmput d'Azure proporcionen la infraestructura i les eines necessàries per executar aplicacions, gestionar càrregues de treball i escalar recursos al núvol. Aquest tema cobrirà els diversos serveis de còmput oferts per Azure, des de màquines virtuals bàsiques fins a orquestració avançada de contenidors i computació sense servidor.

Introducció als Serveis de Còmput d'Azure

Els Serveis de Còmput d'Azure inclouen una gamma d'ofertes que permeten executar aplicacions i gestionar càrregues de treball al núvol. Aquests serveis inclouen:

  • Màquines Virtuals (VMs)
  • Serveis d'Aplicacions
  • Servei de Kubernetes d'Azure (AKS)
  • Funcions d'Azure
  • Azure Batch

Màquines Virtuals (VMs)

Les Màquines Virtuals d'Azure proporcionen recursos de computació escalables sota demanda. Són ideals per a escenaris on necessiteu control complet sobre el sistema operatiu i l'entorn.

Conceptes Clau

  • Mides de VM: Diferents mides ofereixen quantitats variables de CPU, memòria i emmagatzematge.
  • Imatges: Plantilles preconfigurades per a VMs, incloent-hi distribucions de Windows i Linux.
  • Discos: Opcions d'emmagatzematge persistent per a VMs, incloent-hi HDD Estàndard, SSD Estàndard i SSD Premium.
  • Xarxes: Xarxes virtuals, adreces IP públiques i grups de seguretat de xarxa.

Exemple: Creació d'una Màquina Virtual

# Crear un grup de recursos
az group create --name myResourceGroup --location eastus

# Crear una màquina virtual
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Gestió de Màquines Virtuals

  • Iniciar/Aturar: Controlar l'estat de la vostra VM.
  • Redimensionar: Canviar la mida de la vostra VM per escalar cap amunt o cap avall.
  • Instantànies: Crear còpies de seguretat puntuals dels discos de la vostra VM.

Serveis d'Aplicacions

Els Serveis d'Aplicacions d'Azure són una plataforma totalment gestionada per construir, desplegar i escalar aplicacions web i APIs.

Conceptes Clau

  • Aplicacions Web: Allotjar aplicacions web utilitzant diversos frameworks com .NET, Java, PHP, Node.js i Python.
  • API Apps: Allotjar APIs RESTful.
  • Aplicacions Mòbils: Serveis de backend per a aplicacions mòbils.
  • Ranures de Desplegament: Entorns de prova abans del desplegament en producció.

Exemple: Desplegament d'una Aplicació Web

# Crear un pla de servei d'aplicacions
az appservice plan create --name myAppServicePlan --resource-group myResourceGroup --sku B1 --is-linux

# Crear una aplicació web
az webapp create --resource-group myResourceGroup --plan myAppServicePlan --name myUniqueAppName --runtime "NODE|14-lts"

Escalat i Monitorització

  • Escalat: Escalar cap amunt (augmentar recursos) o escalar cap a fora (augmentar instàncies).
  • Monitorització: Utilitzar Azure Monitor i Application Insights per a mètriques de rendiment i ús.

Servei de Kubernetes d'Azure (AKS)

El Servei de Kubernetes d'Azure simplifica el desplegament, la gestió i l'escalat d'aplicacions contenidoritzades utilitzant Kubernetes.

Conceptes Clau

  • Clústers: Un conjunt de VMs que executen aplicacions contenidoritzades.
  • Nodes: VMs individuals dins d'un clúster.
  • Pods: Les unitats desplegables més petites a Kubernetes, que poden contenir un o més contenidors.
  • Serveis: Definir com accedir als pods.

Exemple: Creació d'un Clúster AKS

# Crear un grup de recursos
az group create --name myResourceGroup --location eastus

# Crear un clúster AKS
az aks create --resource-group myResourceGroup --name myAKSCluster --node-count 1 --enable-addons monitoring --generate-ssh-keys

Gestió d'AKS

  • Escalat: Ajustar el nombre de nodes en el vostre clúster.
  • Actualitzacions: Actualitzar la versió de Kubernetes.
  • Monitorització: Utilitzar Azure Monitor per a contenidors per fer un seguiment del rendiment i la salut.

Funcions d'Azure

Les Funcions d'Azure són un servei de computació sense servidor que permet executar codi sota demanda sense gestionar la infraestructura.

Conceptes Clau

  • Desencadenants: Esdeveniments que fan que la funció s'executi (per exemple, sol·licituds HTTP, temporitzadors, cues).
  • Enllaços: Connectar funcions a altres serveis d'Azure (per exemple, emmagatzematge, bases de dades).
  • Pla de Consum: Escala automàticament i es cobra segons el nombre d'execucions.

Exemple: Creació d'una Funció Desencadenada per HTTP

using System.IO;
using Microsoft.AspNetCore.Mvc;
using Microsoft.Azure.WebJobs;
using Microsoft.Azure.WebJobs.Extensions.Http;
using Microsoft.AspNetCore.Http;
using Microsoft.Extensions.Logging;
using Newtonsoft.Json;

public static class HttpTriggerFunction
{
    [FunctionName("HttpTriggerFunction")]
    public static async Task<IActionResult> Run(
        [HttpTrigger(AuthorizationLevel.Function, "get", "post", Route = null)] HttpRequest req,
        ILogger log)
    {
        log.LogInformation("Funció de desencadenament HTTP de C# ha processat una sol·licitud.");

        string name = req.Query["name"];

        string requestBody = await new StreamReader(req.Body).ReadToEndAsync();
        dynamic data = JsonConvert.DeserializeObject(requestBody);
        name = name ?? data?.name;

        return name != null
            ? (ActionResult)new OkObjectResult($"Hola, {name}")
            : new BadRequestObjectResult("Si us plau, passeu un nom a la cadena de consulta o al cos de la sol·licitud");
    }
}

Escalat i Monitorització

  • Escalat: Escala automàticament segons la demanda.
  • Monitorització: Utilitzar Application Insights per a registres detallats i mètriques de rendiment.

Azure Batch

Azure Batch és un servei per executar aplicacions de computació paral·lela a gran escala i d'alt rendiment (HPC) de manera eficient al núvol.

Conceptes Clau

  • Treballs: Una col·lecció de tasques que s'executen als nodes de computació.
  • Tasques: Unitats individuals de treball dins d'un treball.
  • Pools: Col·leccions de nodes de computació que executen les tasques.

Exemple: Creació d'un Treball Batch

# Crear un compte Batch
az batch account create --name mybatchaccount --resource-group myResourceGroup --location eastus

# Crear un pool Batch
az batch pool create --id mypool --vm-size Standard_A1_v2 --target-dedicated-nodes 2 --image canonical:ubuntuserver:18.04-LTS

# Crear un treball Batch
az batch job create --id myjob --pool-id mypool

# Afegir una tasca al treball
az batch task create --job-id myjob --task-id mytask --command-line "echo Hello, Batch!"

Gestió de Treballs Batch

  • Escalat: Ajustar el nombre de nodes al pool.
  • Monitorització: Utilitzar Azure Monitor per fer un seguiment del rendiment dels treballs i les tasques.

Conclusió

Els Serveis de Còmput d'Azure proporcionen una suite completa d'eines per executar i gestionar aplicacions al núvol. Des de màquines virtuals bàsiques fins a funcions sense servidor avançades i orquestració de contenidors, Azure ofereix solucions per a una àmplia gamma d'escenaris. Entendre aquests serveis i com utilitzar-los de manera efectiva us pot ajudar a construir aplicacions escalables, resilients i eficients.
