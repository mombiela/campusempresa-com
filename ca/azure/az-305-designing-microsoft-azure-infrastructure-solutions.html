Introducció

Benvingut al curs AZ-305 sobre Disseny de Solucions d'Infraestructura de Microsoft Azure. Aquest curs està dissenyat per portar-te des d'un nivell principiant fins a un nivell avançat en la comprensió i implementació de solucions d'infraestructura d'Azure. Tant si ets nou a Azure com si vols aprofundir els teus coneixements, aquest curs et proporcionarà les habilitats necessàries per dissenyar i gestionar la infraestructura d'Azure de manera efectiva.

1. Introducció a Azure

Conceptes Clau

  • Visió General d'Azure: Comprendre els conceptes bàsics de Microsoft Azure, incloent la seva infraestructura global, serveis i models de preus.
  • Portal d'Azure: Navegar pel Portal d'Azure i utilitzar-lo per gestionar recursos.
  • Grups de Recursos: Organitzar recursos utilitzant grups de recursos.
  • Azure CLI i PowerShell: Gestionar recursos d'Azure utilitzant eines de línia de comandes.

Exemple: Crear una Màquina Virtual

# Utilitzant Azure CLI per crear una màquina virtual
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Explicació:

  • az vm create: Comanda per crear una màquina virtual.
  • --resource-group: Especifica el grup de recursos.
  • --name: Nom de la màquina virtual.
  • --image: Especifica la imatge del sistema operatiu.
  • --admin-username: Nom d'usuari administrador per a la màquina virtual.
  • --generate-ssh-keys: Genera claus SSH per a un accés segur.

2. Disseny per a Seguretat i Compliment

Conceptes Clau

  • Gestió d'Identitat i Accés (IAM): Utilitzar Azure Active Directory (AAD) per a la gestió d'identitat.
  • Seguretat de Xarxa: Implementar Grups de Seguretat de Xarxa (NSGs) i Firewall d'Azure.
  • Protecció de Dades: Xifrar dades en repòs i en trànsit.
  • Compliment: Assegurar el compliment amb estàndards i regulacions de la indústria.

Exemple: Configurar un Grup de Seguretat de Xarxa

# Crear un Grup de Seguretat de Xarxa
az network nsg create \
  --resource-group myResourceGroup \
  --name myNetworkSecurityGroup

# Crear una regla de seguretat
az network nsg rule create \
  --resource-group myResourceGroup \
  --nsg-name myNetworkSecurityGroup \
  --name AllowSSH \
  --protocol tcp \
  --priority 1000 \
  --destination-port-range 22 \
  --access allow

Explicació:

  • az network nsg create: Comanda per crear un Grup de Seguretat de Xarxa.
  • az network nsg rule create: Comanda per crear una regla de seguretat dins del NSG.
  • --protocol: Especifica el protocol (TCP).
  • --priority: Prioritat de la regla.
  • --destination-port-range: Rang de ports (22 per SSH).
  • --access: Permetre o denegar el trànsit.

3. Disseny per a Rendiment i Escalabilitat

Conceptes Clau

  • Escalabilitat: Comprendre l'escalabilitat vertical i horitzontal.
  • Equilibri de Càrrega: Implementar Azure Load Balancer i Application Gateway.
  • Caché: Utilitzar Azure Cache per a Redis.
  • Xarxa de Distribució de Contingut (CDN): Distribuir contingut globalment amb Azure CDN.

Exemple: Configurar un Equilibrador de Càrrega

# Crear una adreça IP pública
az network public-ip create \
  --resource-group myResourceGroup \
  --name myPublicIP

# Crear un equilibrador de càrrega
az network lb create \
  --resource-group myResourceGroup \
  --name myLoadBalancer \
  --public-ip-address myPublicIP \
  --frontend-ip-name myFrontEndPool \
  --backend-pool-name myBackEndPool

Explicació:

  • az network public-ip create: Comanda per crear una adreça IP pública.
  • az network lb create: Comanda per crear un equilibrador de càrrega.
  • --frontend-ip-name: Nom del pool d'IP de frontend.
  • --backend-pool-name: Nom del pool d'IP de backend.

4. Disseny per a Optimització de Costos

Conceptes Clau

  • Gestió de Costos: Utilitzar Azure Cost Management i Billing.
  • Instàncies Reservades: Estalviar costos amb instàncies reservades.
  • Calculadora de Preus d'Azure: Estimar costos amb la Calculadora de Preus d'Azure.
  • Monitoratge i Alertes: Configurar alertes de costos.

Exemple: Utilitzar la Calculadora de Preus d'Azure

  • Visita la Calculadora de Preus d'Azure.
  • Selecciona els serveis que planejes utilitzar.
  • Configura les opcions per a cada servei.
  • Revisa el cost estimat i ajusta segons sigui necessari.

5. Disseny per a Operacions i Monitoratge

Conceptes Clau

  • Azure Monitor: Monitoritzar recursos amb Azure Monitor.
  • Log Analytics: Recollir i analitzar registres.
  • Automatització d'Azure: Automatitzar tasques amb Azure Automation.
  • Còpia de Seguretat i Recuperació de Desastres: Implementar Azure Backup i Site Recovery.

Exemple: Configurar Azure Monitor

# Crear un espai de treball de Log Analytics
az monitor log-analytics workspace create \
  --resource-group myResourceGroup \
  --workspace-name myWorkspace

# Activar el monitoratge per a una màquina virtual
az monitor diagnostic-settings create \
  --resource myVM \
  --resource-group myResourceGroup \
  --workspace myWorkspace \
  --name myDiagnosticSetting \
  --metrics '[{"category": "AllMetrics", "enabled": true}]' \
  --logs '[{"category": "Syslog", "enabled": true}]'

Explicació:

  • az monitor log-analytics workspace create: Comanda per crear un espai de treball de Log Analytics.
  • az monitor diagnostic-settings create: Comanda per activar el monitoratge per a una màquina virtual.
  • --metrics: Especifica les mètriques a recollir.
  • --logs: Especifica els registres a recollir.

6. Temes Avançats i Millors Pràctiques

Conceptes Clau

  • Solucions de Núvol Híbrid: Integrar recursos locals i del núvol.
  • DevOps i CI/CD: Implementar pràctiques de DevOps amb Azure DevOps.
  • Xarxes Avançades: Utilitzar Azure Virtual WAN i ExpressRoute.
  • Millors Pràctiques de Seguretat: Implementar mesures de seguretat avançades.

Exemple: Configurar una Canalització CI/CD amb Azure DevOps

# azure-pipelines.yml
trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- script: dotnet build
  displayName: 'Build project'

- script: dotnet test
  displayName: 'Run tests'

Explicació:

  • trigger: Especifica la branca per activar la canalització.
  • pool: Especifica la imatge de la màquina virtual a utilitzar.
  • steps: Defineix els passos en la canalització.
  • UseDotNet@2: Tasca per instal·lar el SDK de .NET.
  • dotnet build: Comanda per compilar el projecte.
  • dotnet test: Comanda per executar les proves.
