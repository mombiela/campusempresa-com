Azure Policy és un servei a Azure que s'utilitza per crear, assignar i gestionar polítiques. Aquestes polítiques imposen diferents regles i efectes sobre els teus recursos, de manera que aquests recursos es mantinguin conformes amb els teus estàndards corporatius i acords de nivell de servei. Azure Policy fa això executant avaluacions dels teus recursos i escanejant aquells que no compleixen amb les polítiques que has creat.

Introducció a Azure Policy

  • Definició: Azure Policy és una eina de governança que et permet imposar estàndards organitzacionals i avaluar el compliment a gran escala.
  • Propòsit: Assegurar que els recursos compleixin amb els estàndards corporatius i els requisits reguladors.
  • Característiques clau:
    • Definicions de Polítiques
    • Iniciatives
    • Assignacions
    • Compliment

Definicions de Polítiques

Una definició de política expressa què avaluar i quina acció prendre. Aquestes definicions es poden crear des de zero o utilitzant polítiques integrades proporcionades per Azure.

  • Estructura:
    • Mode: Especifica l'abast de la política (per exemple, Tot, Indexat).
    • Paràmetres: Defineixen valors que es poden utilitzar en la regla de la política.
    • Regla: L'avaluació lògica que determina si un recurs és conforme.

Exemple de Definició de Política

{
  "properties": {
    "displayName": "Ubicacions permeses",
    "description": "Aquesta política et permet restringir les ubicacions que la teva organització pot especificar quan desplega recursos.",
    "parameters": {
      "allowedLocations": {
        "type": "Array",
        "metadata": {
          "description": "La llista d'ubicacions permeses per als recursos.",
          "displayName": "Ubicacions permeses",
          "strongType": "location"
        }
      }
    },
    "policyRule": {
      "if": {
        "not": {
          "field": "location",
          "in": "[parameters('allowedLocations')]"
        }
      },
      "then": {
        "effect": "deny"
      }
    }
  }
}
  • Explicació:
    • displayName: Nom de la política.
    • description: Descripció del que fa la política.
    • parameters: Defineix les ubicacions permeses.
    • policyRule: La regla comprova si la ubicació del recurs no està en les ubicacions permeses i denega el desplegament si no ho està.

Iniciatives

Una iniciativa és una col·lecció de definicions de polítiques que es agrupen amb un objectiu o propòsit específic.

  • Propòsit: Simplificar la gestió de múltiples polítiques agrupant-les.
  • Estructura:
    • Definició d'Iniciativa: Conté múltiples definicions de polítiques.
    • Paràmetres: Paràmetres que es poden compartir entre les polítiques de la iniciativa.

Exemple de Definició d'Iniciativa

{
  "properties": {
    "displayName": "Iniciativa de Seguretat",
    "description": "Assegura que els recursos compleixin amb els estàndards de seguretat.",
    "parameters": {},
    "policyDefinitions": [
      {
        "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/allowedLocations",
        "parameters": {
          "allowedLocations": {
            "value": ["eastus", "westus"]
          }
        }
      },
      {
        "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/requireTag",
        "parameters": {
          "tagName": {
            "value": "Environment"
          }
        }
      }
    ]
  }
}
  • Explicació:
    • displayName: Nom de la iniciativa.
    • description: Descripció del propòsit de la iniciativa.
    • policyDefinitions: Llista de definicions de polítiques incloses en la iniciativa.

Assignacions

Les assignacions s'utilitzen per aplicar una política o iniciativa a un abast específic, com una subscripció, grup de recursos o recurs individual.

  • Abast: Defineix on s'aplica la política o iniciativa.
  • Paràmetres: Valors proporcionats per als paràmetres definits en la política o iniciativa.

Exemple d'Assignació

{
  "properties": {
    "displayName": "Imposar ubicacions permeses",
    "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/allowedLocations",
    "scope": "/subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}",
    "parameters": {
      "allowedLocations": {
        "value": ["eastus", "westus"]
      }
    }
  }
}
  • Explicació:
    • displayName: Nom de l'assignació.
    • policyDefinitionId: L'ID de la definició de política que s'està assignant.
    • scope: L'abast on s'aplica la política.
    • parameters: Valors per als paràmetres definits en la política.

Compliment

El compliment a Azure Policy es refereix a l'estat dels recursos en relació amb les polítiques assignades a ells.

  • Estat de Compliment: Indica si els recursos són conformes o no conformes.
  • Informes de Compliment: Proporcionen informació detallada sobre l'estat de compliment dels recursos.

Veure el Compliment

Pots veure els detalls de compliment al portal d'Azure:

  1. Navega al servei Azure Policy.
  2. Selecciona la pestanya "Compliment".
  3. Revisa l'estat de compliment dels teus recursos.

Conclusió

Azure Policy és una eina poderosa per assegurar que els teus recursos d'Azure compleixin amb els estàndards organitzacionals i els requisits reguladors. Entenent i utilitzant les definicions de polítiques, iniciatives, assignacions i informes de compliment, pots gestionar i imposar efectivament la governança a tot el teu entorn d'Azure.
