Introducció

La passarel·la VPN d'Azure és un component crític en la xarxa d'Azure que et permet connectar de manera segura les teves xarxes locals amb els recursos d'Azure. Aquesta secció cobrirà els conceptes bàsics de la passarel·la VPN d'Azure, els seus tipus, configuracions i funcions avançades.

Conceptes Clau

  • Passarel·la de Xarxa Virtual: Una passarel·la de xarxa virtual està composta per dues o més màquines virtuals que es configuren i es despleguen automàticament a una subxarxa específica anomenada subxarxa de passarel·la.
  • Subxarxa de Passarel·la: Una subxarxa específica a la teva xarxa virtual que conté els recursos de la passarel·la de xarxa virtual.
  • Tipus de Passarel·la VPN: Hi ha dos tipus de passarel·les VPN: Basada en Rutes i Basada en Polítiques.
  • Tipus de Connexió: Connexions Site-to-Site (S2S), Point-to-Site (P2S) i VNet-to-VNet.

Configuració d'una Passarel·la VPN

Prerequisits

  • Una subscripció d'Azure.
  • Una xarxa virtual amb una subxarxa de passarel·la.
  • Una adreça IP pública per a la passarel·la VPN.

Passos per Crear una Passarel·la VPN

  1. Crear una Xarxa Virtual:

    az network vnet create --name MyVNet --resource-group MyResourceGroup --address-prefix 10.1.0.0/16 --subnet-name MySubnet --subnet-prefix 10.1.0.0/24

  2. Crear una Subxarxa de Passarel·la:

    az network vnet subnet create --address-prefix 10.1.255.0/27 --name GatewaySubnet --resource-group MyResourceGroup --vnet-name MyVNet

  3. Crear una Adreça IP Pública:

    az network public-ip create --name MyGatewayIP --resource-group MyResourceGroup --allocation-method Dynamic

  4. Crear la Passarel·la VPN:

    az network vnet-gateway create --name MyVPNGateway --public-ip-address MyGatewayIP --resource-group MyResourceGroup --vnet MyVNet --gateway-type Vpn --vpn-type RouteBased --sku VpnGw1 --no-wait

Configuració de Connexions VPN

VPN Site-to-Site (S2S)

  • Definició: Connecta la teva xarxa local amb una xarxa virtual d'Azure a través d'un túnel VPN IPsec/IKE.
  • Cas d'Ús: Ideal per estendre la teva xarxa local a Azure.

VPN Point-to-Site (P2S)

  • Definició: Permet que clients individuals es connectin a la teva xarxa virtual d'Azure des de qualsevol lloc a través d'internet.
  • Cas d'Ús: Adequat per a treballadors remots que necessiten accés segur als recursos d'Azure.

VPN VNet-to-VNet

  • Definició: Connecta dues xarxes virtuals d'Azure.
  • Cas d'Ús: Útil per connectar xarxes virtuals en diferents regions o subscripcions.

Exemple: Configuració d'una VPN Site-to-Site

  1. Crear una Passarel·la de Xarxa Local:

    az network local-gateway create --gateway-ip-address <OnPremisesPublicIP> --name MyLocalGateway --resource-group MyResourceGroup --local-address-prefixes 10.2.0.0/16

  2. Crear la Connexió VPN:

    az network vpn-connection create --name MyS2SConnection --resource-group MyResourceGroup --vnet-gateway1 MyVPNGateway --local-gateway2 MyLocalGateway --shared-key 'MySharedKey'

Funcions Avançades

Alta Disponibilitat

  • Mode Actiu-Actiu: Desplega dues passarel·les VPN en una configuració activa-activa per a alta disponibilitat.
  • Mode Actiu-Passiu: Configuració per defecte on una passarel·la està activa i l'altra està passiva.

BGP (Border Gateway Protocol)

  • Definició: BGP s'utilitza per intercanviar informació de rutes entre xarxes.
  • Cas d'Ús: Útil per a topologies de xarxa complexes i rutes dinàmiques.

Integració amb ExpressRoute

  • Definició: ExpressRoute proporciona una connexió privada a Azure.
  • Cas d'Ús: Per a escenaris que requereixen més amplada de banda i menor latència que les connexions VPN típiques.

Conclusió

La passarel·la VPN d'Azure és una manera versàtil i segura de connectar les teves xarxes locals, clients remots i altres xarxes virtuals d'Azure. Comprenent els diferents tipus de passarel·les VPN i connexions, pots dissenyar una arquitectura de xarxa robusta i escalable a Azure. Tant si estàs configurant una simple VPN site-to-site com una connexió complexa VNet-to-VNet multi-regió, la passarel·la VPN d'Azure proporciona les eines i funcions necessàries per satisfer les teves necessitats de xarxa.
