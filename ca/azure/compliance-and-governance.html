Introducció al Compliment i Governança

El compliment i la governança són aspectes crítics per gestionar els recursos del núvol de manera efectiva. Asseguren que la vostra organització compleixi amb els requisits legals, reguladors i organitzatius mentre manté el control sobre el vostre entorn de núvol.

Conceptes Clau

  • Compliment: Complir amb les lleis, regulacions i polítiques organitzatives.
  • Governança: Establir polítiques, rols, responsabilitats i processos per controlar i gestionar els recursos TI.

Política d'Azure

Azure Policy és un servei a Azure que podeu utilitzar per crear, assignar i gestionar polítiques. Aquestes polítiques imposen diferents regles i efectes sobre els vostres recursos, assegurant que aquests recursos es mantinguin complint amb els vostres estàndards corporatius i acords de nivell de servei.

Creació i Assignació de Polítiques

  1. Crear una Definició de Política: Definir les regles i efectes per a la vostra política.
  2. Assignar la Política: Aplicar la política a un abast específic, com una subscripció o grup de recursos.
  3. Avaluar el Compliment: Comprovar l'estat de compliment dels vostres recursos enfront de la política.

Exemple: Creació d'una Definició de Política

{
  "properties": {
    "displayName": "Ubicacions permeses",
    "policyType": "BuiltIn",
    "mode": "All",
    "parameters": {
      "allowedLocations": {
        "type": "Array",
        "metadata": {
          "description": "La llista d'ubicacions permeses per als recursos.",
          "displayName": "Ubicacions permeses",
          "strongType": "location"
        }
      }
    },
    "policyRule": {
      "if": {
        "field": "location",
        "notIn": "[parameters('allowedLocations')]"
      },
      "then": {
        "effect": "deny"
      }
    }
  }
}

Aquesta política assegura que els recursos només es poden crear en ubicacions especificades.

Blueprints d'Azure

Azure Blueprints us permeten definir un conjunt repetible de recursos d'Azure que implementen i compleixen amb estàndards, patrons i requisits. Els blueprints s'utilitzen per orquestrar el desplegament de diversos plantilles de recursos i altres artefactes, com ara:

  • Assignacions de Rols
  • Assignacions de Polítiques
  • Plantilles d'Azure Resource Manager (ARM)
  • Grups de Recursos

Creació d'un Blueprint

  1. Definir Artefactes: Especificar els recursos i configuracions.
  2. Assignar el Blueprint: Aplicar el blueprint a una subscripció o grup de recursos.
  3. Fer Seguiment i Gestionar: Monitoritzar l'estat del desplegament i el compliment.

Exemple: Creació d'un Blueprint

{
  "properties": {
    "displayName": "Blueprint de Mostra",
    "description": "Un blueprint de mostra per desplegar recursos.",
    "targetScope": "subscription",
    "parameters": {},
    "resourceGroups": {
      "myResourceGroup": {
        "properties": {
          "displayName": "El Meu Grup de Recursos",
          "description": "Grup de recursos per als meus recursos.",
          "location": "West US"
        }
      }
    },
    "artifacts": [
      {
        "type": "policyAssignment",
        "name": "allowedLocations",
        "properties": {
          "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/allowedLocations",
          "parameters": {
            "allowedLocations": {
              "value": ["West US", "East US"]
            }
          }
        }
      }
    ]
  }
}

Aquest blueprint assegura que es creï un grup de recursos i s'assigni una política per restringir les ubicacions dels recursos.

Plantilles d'Azure Resource Manager (ARM)

Les plantilles ARM són fitxers JSON que defineixen la infraestructura i configuració per a la vostra solució d'Azure. Permeten desplegar, gestionar i configurar tots els recursos per a la vostra solució de manera declarativa.

Característiques Clau

  • Sintaxi Declarativa: Definir el que voleu desplegar sense escriure la seqüència de comandes de programació.
  • Desplegaments Repetibles: Assegurar entorns consistents a través de diferents etapes.
  • Modular i Reutilitzable: Dividir les plantilles en components més petits i reutilitzables.

Exemple: Plantilla ARM Bàsica

{
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Storage/storageAccounts",
      "apiVersion": "2019-04-01",
      "name": "mystorageaccount",
      "location": "westus",
      "sku": {
        "name": "Standard_LRS"
      },
      "kind": "StorageV2",
      "properties": {}
    }
  ]
}

Aquesta plantilla desplega un compte d'emmagatzematge a la regió "westus".

Conclusió

El compliment i la governança a Azure són essencials per assegurar que els vostres recursos del núvol compleixin amb els estàndards legals, reguladors i organitzatius. Mitjançant l'ús d'eines com Azure Policy, Azure Blueprints i plantilles ARM, podeu automatitzar i fer complir aquests estàndards a tot el vostre entorn d'Azure. Comprendre i implementar aquestes eines us ajudarà a mantenir el control i el compliment, reduint riscos i millorant la gestió general dels vostres recursos del núvol.
