L'emmagatzematge de cues d'Azure és un servei per emmagatzemar grans quantitats de missatges que es poden accedir des de qualsevol lloc del món mitjançant trucades autenticades utilitzant HTTP o HTTPS. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de l'emmagatzematge de cues d'Azure.

Introducció a l'emmagatzematge de cues d'Azure

L'emmagatzematge de cues d'Azure està dissenyat per permetre la comunicació entre diferents components d'una aplicació al núvol. Et permet emmagatzemar i recuperar missatges, assegurant que la teva aplicació pugui escalar i mantenir-se responsiva.

Conceptes clau

  • Cua: Una cua conté un conjunt de missatges. Cada missatge pot tenir fins a 64 KB de mida.
  • Missatge: Un missatge és la dada que s'emmagatzema en una cua. Els missatges són típicament text, però poden ser de qualsevol tipus de dada.
  • Temps d'invisibilitat: El període de temps durant el qual un missatge és invisible per a altres consumidors després de ser recuperat.
  • Missatges tòxics: Missatges que no es poden processar amb èxit després de múltiples intents.

Configuració de l'emmagatzematge de cues d'Azure

Prerequisits

  • Un compte d'Azure. Si no en tens un, pots crear un compte gratuït a Azure Free Account.

Creació d'un compte d'emmagatzematge de cues

  1. Inicia sessió al portal d'Azure.

  2. Crea un compte d'emmagatzematge:

    • Navega a "Comptes d'emmagatzematge" i fes clic a "Crear".
    • Omple els detalls necessaris com Subscripció, Grup de recursos, Nom del compte d'emmagatzematge i Ubicació.
    • Tria el nivell de rendiment "Estàndard" i "StorageV2 (general-purpose v2)" per al tipus de compte.
    • Fes clic a "Revisar + crear" i després a "Crear".

  3. Crea una cua:

    • Ves al teu compte d'emmagatzematge acabat de crear.
    • Sota "Emmagatzematge de dades", selecciona "Cues".
    • Fes clic a "+ Cua" i proporciona un nom per a la teva cua.
    • Fes clic a "D'acord" per crear la cua.

Treballant amb l'emmagatzematge de cues d'Azure

Afegint missatges a una cua

Per afegir missatges a una cua, pots utilitzar els SDKs d'emmagatzematge d'Azure o l'API REST. A continuació es mostra un exemple utilitzant l'SDK d'Azure per a Python.

from azure.storage.queue import QueueServiceClient

# Crea un QueueServiceClient
queue_service_client = QueueServiceClient.from_connection_string("your_connection_string")

# Obté un QueueClient
queue_client = queue_service_client.get_queue_client("your_queue_name")

# Afegeix un missatge a la cua
queue_client.send_message("Hello, Azure Queue Storage!")

Recuperant missatges d'una cua

Per recuperar missatges, pots utilitzar el mètode receive_messages. Aquí tens un exemple en Python:

# Recupera missatges de la cua
messages = queue_client.receive_messages()

for message in messages:
    print(f"Message: {message.content}")
    # Esborra el missatge després de processar-lo
    queue_client.delete_message(message)

Gestionant el temps d'invisibilitat

Quan es recupera un missatge, es torna invisible per a altres consumidors durant un període especificat (temps d'invisibilitat). Si el missatge no s'esborra dins d'aquest període, es torna visible de nou.

# Recupera un missatge amb un temps d'invisibilitat de 30 segons
messages = queue_client.receive_messages(visibility_timeout=30)

for message in messages:
    print(f"Message: {message.content}")
    # Processa el missatge
    # ...
    # Esborra el missatge després de processar-lo
    queue_client.delete_message(message)

Conceptes avançats

TTL del missatge (temps de vida)

Els missatges en una cua poden tenir un TTL, que especifica quant de temps el missatge ha de romandre a la cua abans de ser esborrat automàticament.

# Envia un missatge amb un TTL de 60 segons
queue_client.send_message("This message will expire in 60 seconds", time_to_live=60)

Gestió de missatges tòxics

Els missatges que no es poden processar amb èxit després de múltiples intents es coneixen com a missatges tòxics. Pots implementar una lògica per moure aquests missatges a una cua separada per a una anàlisi posterior.

# Lògica d'exemple per gestionar missatges tòxics
max_attempts = 5

for message in messages:
    if message.dequeue_count > max_attempts:
        # Mou a la cua de missatges tòxics
        poison_queue_client.send_message(message.content)
        queue_client.delete_message(message)
    else:
        # Processa el missatge
        # ...
        queue_client.delete_message(message)

Conclusió

L'emmagatzematge de cues d'Azure és un servei potent per desacoblar components d'una aplicació al núvol i assegurar un lliurament fiable de missatges. Entenent els conceptes bàsics i les característiques avançades, pots utilitzar efectivament l'emmagatzematge de cues d'Azure per construir aplicacions escalables i resilients.
