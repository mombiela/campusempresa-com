Introducció als Mòduls

Els mòduls són una part fonamental de Node.js, permetent-te organitzar el teu codi en peces reutilitzables. Aquesta secció cobrirà els conceptes bàsics dels mòduls i com utilitzar-los eficaçment en les teves aplicacions Node.js.

Què és un Mòdul?

  • Un mòdul és una peça de codi reutilitzable que es pot exportar i importar en altres parts de la teva aplicació.
  • Node.js utilitza el sistema de mòduls CommonJS, on cada fitxer es tracta com un mòdul separat.

Per què Utilitzar Mòduls?

  • Organització del Codi: Divideix el teu codi en peces més petites i manejables.
  • Reutilització: Utilitza el mateix codi en diverses parts de la teva aplicació.
  • Mantenibilitat: Més fàcil de mantenir i actualitzar el teu codi.

Utilitzant require() per Importar Mòduls

La funció require() s'utilitza per importar mòduls en Node.js.

Ús Bàsic

Per importar un mòdul, utilitzes la funció require() seguida del camí al mòdul.

// Important un mòdul integrat
const http = require('http');

// Important un mòdul local
const myModule = require('./myModule');

Mòduls Integrats

Node.js ve amb diversos mòduls integrats que pots utilitzar sense instal·lar res.

  • http: Utilitzat per crear servidors i clients HTTP.
  • fs: Proporciona una API per interactuar amb el sistema de fitxers.
  • path: Utilitats per treballar amb camins de fitxers i directoris.

Exemple d'ús del mòdul http:

const http = require('http');

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello, World!\n');
});

server.listen(3000, '127.0.0.1', () => {
  console.log('Server running at http://127.0.0.1:3000/');
});

Mòduls Locals

Pots crear els teus propis mòduls i importar-los utilitzant require().

Exemple d'un mòdul local (math.js):

// math.js
function add(a, b) {
  return a + b;
}

function subtract(a, b) {
  return a - b;
}

module.exports = {
  add,
  subtract
};

Important i utilitzant el mòdul local:

// app.js
const math = require('./math');

console.log(math.add(2, 3)); // Sortida: 5
console.log(math.subtract(5, 2)); // Sortida: 3

Exportant Mòduls

Per fer funcions, objectes o valors disponibles per a altres mòduls, necessites exportar-los.

module.exports vs exports

  • module.exports: L'objecte que realment es retorna com a resultat d'una crida a require().
  • exports: Una abreviatura per a module.exports.

Exemple utilitzant module.exports:

// logger.js
module.exports = function(message) {
  console.log(message);
};

Exemple utilitzant exports:

// calculator.js
exports.add = function(a, b) {
  return a + b;
};

exports.subtract = function(a, b) {
  return a - b;
};

Conceptes Avançats de Mòduls

A mesura que et familiaritzis més amb els mòduls, pots explorar conceptes més avançats.

Dependències Circulars

Les dependències circulars ocorren quan dos o més mòduls depenen l'un de l'altre. Node.js maneja això retornant un mòdul parcialment construït.

Exemple:

// a.js
const b = require('./b');
console.log('In a.js, b.done =', b.done);
exports.done = true;

// b.js
const a = require('./a');
console.log('In b.js, a.done =', a.done);
exports.done = true;

Emmagatzematge en Memòria Cau

Els mòduls es guarden en memòria cau després de ser carregats per primera vegada. Això significa que les crides subseqüents a require() retornaran la versió emmagatzemada en memòria cau.

Exemple:

// counter.js
let count = 0;

module.exports = {
  increment: () => count++,
  getCount: () => count
};

// app.js
const counter = require('./counter');
counter.increment();
console.log(counter.getCount()); // Sortida: 1

const counterAgain = require('./counter');
console.log(counterAgain.getCount()); // Sortida: 1 (emmagatzemat en memòria cau)

Conclusió

Entendre els mòduls i la funció require() és crucial per desenvolupar aplicacions Node.js eficients i fàcils de mantenir. Organitzant el teu codi en mòduls, pots millorar la llegibilitat, la reutilització i la mantenibilitat del codi. A mesura que avancis, trobaràs escenaris més complexos com les dependències circulars i l'emmagatzematge en memòria cau, però els conceptes fonamentals romanen iguals. Feliç programació!
