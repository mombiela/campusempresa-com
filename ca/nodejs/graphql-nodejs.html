Introducció a GraphQL

GraphQL és un llenguatge de consulta per a APIs i un entorn d'execució per a executar aquestes consultes utilitzant un sistema de tipus que defineixes per a les teves dades. Proporciona una alternativa més eficient, potent i flexible a REST.

  • Conceptes Clau:
    • Esquema: Defineix l'estructura de la teva API, incloent tipus i relacions.
    • Consulta: Sol·licita dades específiques i modela la resposta.
    • Mutació: Modifica dades al servidor.
    • Resolver: Funcions que gestionen la recuperació de dades per a una consulta o mutació.

Configurant un Projecte Node.js amb GraphQL

Per començar amb GraphQL a Node.js, necessites configurar un projecte bàsic de Node.js i instal·lar els paquets necessaris.

Configuració Pas a Pas

  1. Inicialitza un Projecte Node.js:

    mkdir graphql-nodejs
cd graphql-nodejs
npm init -y

  2. Instal·la els Paquets Necessaris:

    npm install express express-graphql graphql

  3. Crea un Servidor Bàsic:

    const express = require('express');
const { graphqlHTTP } = require('express-graphql');
const { buildSchema } = require('graphql');

// Defineix un esquema
const schema = buildSchema(`
  type Query {
    hello: String
  }
`);

// Defineix un resolver
const root = {
  hello: () => {
    return 'Hello world!';
  },
};

const app = express();
app.use('/graphql', graphqlHTTP({
  schema: schema,
  rootValue: root,
  graphiql: true,
}));

app.listen(4000, () => console.log('Servidor en funcionament a http://localhost:4000/graphql'));

Definint un Esquema GraphQL

L'esquema és el nucli de qualsevol servidor GraphQL. Defineix els tipus i les relacions entre ells.

Exemple d'Esquema

type Query {
  book(id: ID!): Book
  books: [Book]
}

type Book {
  id: ID!
  title: String!
  author: Author!
}

type Author {
  id: ID!
  name: String!
  books: [Book]
}
  • Tipus:
    • Query: Punt d'entrada per a operacions de lectura.
    • Book: Representa un llibre amb un ID, títol i autor.
    • Author: Representa un autor amb un ID, nom i llista de llibres.

Escriure Resolvers

Els resolvers són funcions que gestionen la lògica per a recuperar les dades definides en el teu esquema.

Exemple de Resolvers

const books = [
  { id: '1', title: '1984', authorId: '1' },
  { id: '2', title: 'Brave New World', authorId: '2' },
];

const authors = [
  { id: '1', name: 'George Orwell' },
  { id: '2', name: 'Aldous Huxley' },
];

const resolvers = {
  Query: {
    book: (_, { id }) => books.find(book => book.id === id),
    books: () => books,
  },
  Book: {
    author: (book) => authors.find(author => author.id === book.authorId),
  },
  Author: {
    books: (author) => books.filter(book => book.authorId === author.id),
  },
};

Integració amb una Base de Dades

Per fer la teva API GraphQL més dinàmica, pots integrar-la amb una base de dades.

Exemple amb MongoDB

  1. Instal·la Mongoose:

    npm install mongoose

  2. Connecta't a MongoDB:

    const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/graphql', { useNewUrlParser: true, useUnifiedTopology: true });

const BookSchema = new mongoose.Schema({
  title: String,
  authorId: String,
});

const Book = mongoose.model('Book', BookSchema);

const AuthorSchema = new mongoose.Schema({
  name: String,
});

const Author = mongoose.model('Author', AuthorSchema);

  3. Actualitza els Resolvers per Utilitzar Mongoose:

    const resolvers = {
  Query: {
    book: async (_, { id }) => await Book.findById(id),
    books: async () => await Book.find(),
  },
  Book: {
    author: async (book) => await Author.findById(book.authorId),
  },
  Author: {
    books: async (author) => await Book.find({ authorId: author.id }),
  },
};

Temes Avançats

Autenticació i Autorització

Implementar autenticació i autorització en GraphQL es pot fer utilitzant middleware.

Exemple de Middleware

const jwt = require('jsonwebtoken');

const authMiddleware = (req, res, next) => {
  const token = req.headers.authorization;
  si (token) {
    try {
      const user = jwt.verify(token, 'your_secret_key');
      req.user = user;
    } catch (err) {
      console.error(err);
    }
  }
  next();
};

app.use(authMiddleware);

Subscricions

Les subscripcions de GraphQL et permeten subscriure't a actualitzacions en temps real.

Exemple amb Apollo Server

  1. Instal·la Apollo Server i WebSocket:

    npm install apollo-server apollo-server-express graphql-subscriptions

  2. Configura les Subscricions:

    const { ApolloServer, gql, PubSub } = require('apollo-server');
const pubsub = new PubSub();

const typeDefs = gql`
  type Query {
    _: Boolean
  }

  type Subscription {
    bookAdded: Book
  }

  type Book {
    id: ID!
    title: String!
    author: Author!
  }

  type Author {
    id: ID!
    name: String!
    books: [Book]
  }
`;

const resolvers = {
  Subscription: {
    bookAdded: {
      subscribe: () => pubsub.asyncIterator(['BOOK_ADDED']),
    },
  },
};

const server = new ApolloServer({ typeDefs, resolvers });

server.listen().then(({ url }) => {
  console.log(`🚀 Servidor llest a ${url}`);
});

Conclusió

GraphQL amb Node.js proporciona una manera potent i flexible de construir APIs. Entenent els conceptes clau, configurant un servidor bàsic, definint esquemes, escrivint resolvers i integrant amb una base de dades, pots crear APIs robustes i eficients. Temes avançats com l'autenticació, l'autorització i les subscripcions milloren encara més les capacitats del teu servidor GraphQL.
