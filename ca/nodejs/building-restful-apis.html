Introducció a les APIs RESTful

  • Definició: REST (Representational State Transfer) és un estil arquitectònic per dissenyar aplicacions en xarxa.
  • Principis:
    • Statelessness
    • Arquitectura Client-Servidor
    • Interfície Uniforme
    • Sistema en Capes
    • Cacheabilitat
    • Codi sota Demanda (opcional)

Configuració del Teu Entorn

  • Instal·lació de Node.js: Assegura't que Node.js i npm (Node Package Manager) estan instal·lats.
  • Inicialització del Projecte: 
    mkdir rest-api-example
cd rest-api-example
npm init -y
  • Instal·lació de Dependències:
    • Express.js: Un framework minimalista i flexible per aplicacions web amb Node.js.
    • Nodemon: Una eina que ajuda a desenvolupar aplicacions Node.js reiniciant automàticament l'aplicació quan es detecten canvis en els fitxers.
    npm install express
npm install --save-dev nodemon

Creació d'un Servidor Bàsic amb Express

  • Configuració Bàsica del Servidor: 
    const express = require('express');
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(PORT, () => {
  console.log(`Server is running on http://localhost:${PORT}`);
});
  • Execució del Servidor: 
    node index.js

Rutes RESTful

  • Operacions CRUD:
    • Crear: POST
    • Llegir: GET
    • Actualitzar: PUT
    • Eliminar: DELETE
  • Exemples de Rutes: 
    app.post('/items', (req, res) => {
  // Codi per crear un element
});

app.get('/items', (req, res) => {
  // Codi per obtenir tots els elements
});

app.get('/items/:id', (req, res) => {
  // Codi per obtenir un element per id
});

app.put('/items/:id', (req, res) => {
  // Codi per actualitzar un element per id
});

app.delete('/items/:id', (req, res) => {
  // Codi per eliminar un element per id
});

Middleware i Anàlisi del Cos de la Sol·licitud

  • Middleware: Funcions que tenen accés a l'objecte de sol·licitud, l'objecte de resposta i la següent funció middleware.
  • Body Parser: Middleware per analitzar els cossos de les sol·licituds entrants. 
    npm install body-parser

    const bodyParser = require('body-parser');
app.use(bodyParser.json());

Connexió a una Base de Dades

  • MongoDB i Mongoose:
    • Instal·lació: 
      npm install mongoose
    • Connexió a MongoDB: 
      const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost:27017/rest-api-example', { useNewUrlParser: true, useUnifiedTopology: true });

Definició d'Esquemes i Models de Mongoose

  • Definició de l'Esquema: 
    const itemSchema = new mongoose.Schema({
  name: String,
  quantity: Number,
  price: Number
});

const Item = mongoose.model('Item', itemSchema);

Implementació d'Operacions CRUD amb Mongoose

  • Operació de Creació: 
    app.post('/items', async (req, res) => {
  const newItem = new Item(req.body);
  try {
    const savedItem = await newItem.save();
    res.status(201).send(savedItem);
  } catch (error) {
    res.status(400).send(error);
  }
});
  • Operacions de Lectura: 
    app.get('/items', async (req, res) => {
  try {
    const items = await Item.find();
    res.status(200).send(items);
  } catch (error) {
    res.status(500).send(error);
  }
});

app.get('/items/:id', async (req, res) => {
  try {
    const item = await Item.findById(req.params.id);
    if (!item) {
      return res.status(404).send();
    }
    res.status(200).send(item);
  } catch (error) {
    res.status(500).send(error);
  }
});
  • Operació d'Actualització: 
    app.put('/items/:id', async (req, res) => {
  try {
    const item = await Item.findByIdAndUpdate(req.params.id, req.body, { new: true, runValidators: true });
    if (!item) {
      return res.status(404).send();
    }
    res.status(200).send(item);
  } catch (error) {
    res.status(400).send(error);
  }
});
  • Operació d'Eliminació: 
    app.delete('/items/:id', async (req, res) => {
  try {
    const item = await Item.findByIdAndDelete(req.params.id);
    if (!item) {
      return res.status(404).send();
    }
    res.status(200).send(item);
  } catch (error) {
    res.status(500).send(error);
  }
});

Gestió d'Errors

  • Middleware Centralitzat de Gestió d'Errors: 
    app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something broke!');
});

Autenticació i Autorització

  • JWT (JSON Web Tokens):
    • Instal·lació: 
      npm install jsonwebtoken
    • Generació de Tokens: 
      const jwt = require('jsonwebtoken');
const token = jwt.sign({ _id: user._id }, 'secretKey', { expiresIn: '1h' });
    • Verificació de Tokens: 
      const auth = (req, res, next) => {
  const token = req.header('Authorization').replace('Bearer ', '');
  try {
    const decoded = jwt.verify(token, 'secretKey');
    req.user = decoded;
    next();
  } catch (error) {
    res.status(401).send({ error: 'Please authenticate.' });
  }
};

Temes Avançats

  • Limitació de Ràtio: Prevenir abusos limitant el nombre de sol·licituds que un client pot fer.
  • Validació de Dades: Utilitzar biblioteques com Joi per validar les dades de les sol·licituds.
  • Documentació de l'API: Utilitzar eines com Swagger per documentar la teva API.

Conclusió

Construir APIs RESTful amb Node.js implica entendre els principis de REST, configurar un servidor amb Express, definir rutes, connectar-se a una base de dades i implementar operacions CRUD. Temes avançats com l'autenticació, la gestió d'errors i la limitació de ràtio són crucials per crear APIs robustes i segures. Seguint aquest enfocament estructurat, pots desenvolupar APIs escalables i mantenibles que s'adhereixen als estàndards de la indústria.
