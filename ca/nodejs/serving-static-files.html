El Projecte Sobre nosaltres Contribuir Donacions Llicència

En aquesta secció, explorarem com servir fitxers estàtics utilitzant Node.js. Els fitxers estàtics inclouen HTML, CSS, JavaScript, imatges i altres recursos que no canvien dinàmicament. Servir aquests fitxers de manera eficient és crucial per construir aplicacions web.

Conceptes Clau

  • Fitxers Estàtics: Fitxers que no canvien dinàmicament i es serveixen tal com són al client.
  • Node.js: Un entorn d'execució de JavaScript construït sobre el motor de JavaScript V8 de Chrome, comunament utilitzat per al desenvolupament del costat del servidor.
  • Express.js: Un marc d'aplicacions web minimalista i flexible per a Node.js que proporciona característiques robustes per a aplicacions web i mòbils.

Configuració del Projecte

Abans de començar a servir fitxers estàtics, configurem un projecte bàsic de Node.js.

  1. Inicialitzar un Projecte de Node.js:

    mkdir static-files-example
cd static-files-example
npm init -y

  2. Instal·lar Express.js:

    npm install express

  3. Crear l'Estructura del Projecte:

    static-files-example/
├── public/
│   ├── css/
│   │   └── styles.css
│   ├── js/
│   │   └── script.js
│   └── index.html
├── server.js
└── package.json

Servir Fitxers Estàtics amb Express.js

Express.js fa que sigui senzill servir fitxers estàtics utilitzant el middleware express.static.

Exemple Bàsic

  1. Crear server.js:

    const express = require('express');
const path = require('path');
const app = express();

// Servir fitxers estàtics des del directori 'public'
app.use(express.static(path.join(__dirname, 'public')));

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});

  2. Crear public/index.html:

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Static Files Example</title>
    <link rel="stylesheet" href="css/styles.css">
</head>
<body>
    <h1>Hello, World!</h1>
    <script src="js/script.js"></script>
</body>
</html>

  3. Crear public/css/styles.css:

    body {
    font-family: Arial, sans-serif;
    background-color: #f0f0f0;
    text-align: center;
    padding: 50px;
}

  4. Crear public/js/script.js:

    document.addEventListener('DOMContentLoaded', () => {
    console.log('JavaScript is working!');
});

Explicació

  • express.static: Aquest middleware serveix fitxers estàtics des del directori especificat. En aquest cas, serveix fitxers des del directori public.
  • path.join: Aquest mètode uneix els segments de ruta donats en una sola ruta, assegurant la compatibilitat entre plataformes.

Configuració Avançada

Servir Múltiples Directoris

Pots servir fitxers estàtics des de múltiples directoris cridant express.static diverses vegades.

app.use(express.static(path.join(__dirname, 'public')));
app.use(express.static(path.join(__dirname, 'assets')));

Personalitzar el Control de Caché

Pots personalitzar les capçaleres de control de caché per als fitxers estàtics passant opcions a express.static.

app.use(express.static(path.join(__dirname, 'public'), {
    maxAge: '1d', // Caché dels fitxers per un dia
    etag: false   // Desactivar les capçaleres ETag
}));

Gestionar Errors 404 per a Fitxers Estàtics

Per gestionar errors 404 per a fitxers estàtics, pots afegir una funció middleware després del middleware de fitxers estàtics.

app.use((req, res, next) => {
    res.status(404).send('File not found');
});

Conclusió

Servir fitxers estàtics és un aspecte fonamental de la construcció d'aplicacions web amb Node.js i Express.js. Entenent com configurar i configurar el servei de fitxers estàtics, pots assegurar-te que la teva aplicació web lliura recursos de manera eficient i efectiva. Aquest coneixement forma la base per a temes més avançats en el desenvolupament web, com ara l'optimització del rendiment i la implementació de mesures de seguretat.
