Introducció

Express és un framework minimalista i flexible per a aplicacions web de Node.js que proporciona un conjunt robust de característiques per a aplicacions web i mòbils. En aquesta secció, cobrirem com configurar una aplicació Express des de zero, incloent la instal·lació, la configuració bàsica i la creació de la teva primera ruta.

Prerequisits

Abans de començar, assegura't de tenir instal·lat el següent:

  • Node.js (versió 12 o superior)
  • npm (Node Package Manager)

Guia Pas a Pas

1. Inicialitzar un Nou Projecte Node.js

Primer, crea un nou directori per al teu projecte i navega-hi utilitzant el teu terminal o símbol del sistema.

mkdir my-express-app
cd my-express-app

A continuació, inicialitza un nou projecte Node.js utilitzant npm:

npm init -y

Aquesta comanda crearà un fitxer package.json amb la configuració per defecte.

2. Instal·lar Express

Per instal·lar Express, executa la següent comanda:

npm install express

Això afegirà Express al teu projecte i actualitzarà el fitxer package.json per incloure Express com a dependència.

3. Crear el Fitxer de l'Aplicació

Crea un nou fitxer anomenat app.js al directori del teu projecte. Aquest fitxer serà el punt d'entrada per a la teva aplicació Express.

// app.js
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(port, () => {
  console.log(`Example app listening at http://localhost:${port}`);
});

4. Executar l'Aplicació

Per iniciar la teva aplicació Express, executa la següent comanda al teu terminal:

node app.js

Obre el teu navegador web i navega a http://localhost:3000. Hauries de veure el missatge "Hello World!".

Conceptes Clau

  • Aplicació Express: El component principal que gestiona les sol·licituds i respostes.
  • Middleware: Funcions que s'executen durant el cicle de vida d'una sol·licitud al servidor Express.
  • Routing: Definició de punts finals (URIs) i com l'aplicació respon a les sol·licituds dels clients.

Exemple: Afegir Middleware i Rutes

Afegir Middleware

Les funcions middleware són funcions que tenen accés a l'objecte de sol·licitud (req), l'objecte de resposta (res) i la següent funció middleware en el cicle de sol·licitud-resposta de l'aplicació.

// app.js
const express = require('express');
const app = express();
const port = 3000;

// Funció middleware per registrar sol·licituds
app.use((req, res, next) => {
  console.log(`${req.method} request for '${req.url}'`);
  next();
});

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(port, () => {
  console.log(`Example app listening at http://localhost:${port}`);
});

Definir Rutes

Les rutes defineixen els punts finals de la teva aplicació i especifiquen com ha de respondre a les sol·licituds dels clients.

// app.js
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.get('/about', (req, res) => {
  res.send('About Page');
});

app.get('/contact', (req, res) => {
  res.send('Contact Page');
});

app.listen(port, () => {
  console.log(`Example app listening at http://localhost:${port}`);
});

Resum

En aquesta secció, hem cobert els conceptes bàsics per configurar una aplicació Express. Hem començat inicialitzant un nou projecte Node.js, instal·lant Express i creant un fitxer d'aplicació bàsic. També hem discutit conceptes clau com middleware i routing, i hem proporcionat exemples per il·lustrar com afegir middleware i definir rutes a la teva aplicació.

Seguint aquests passos, ara hauries de tenir una comprensió bàsica de com configurar i executar una aplicació Express. En les següents seccions, aprofundirem en temes més avançats com la gestió d'errors, l'ús de plantilles i la connexió a bases de dades.
