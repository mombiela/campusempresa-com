Gestió d'Errors
Mòdul HTTP ►

La gestió d'errors és un aspecte crític per construir aplicacions robustes i fiables en Node.js. Aquest tema et guiarà a través dels fonaments de la gestió d'errors, des de conceptes bàsics fins a tècniques avançades.

Introducció a la Gestió d'Errors

La gestió d'errors és el procés de respondre i gestionar els errors que es produeixen durant l'execució d'un programa. En Node.js, els errors poden ocórrer per diverses raons com ara entrada d'usuari no vàlida, sol·licituds de xarxa fallides o errors en el codi.

Conceptes Clau

  • Objecte Error: En Node.js, els errors es representen amb l'objecte Error, que conté informació sobre l'error.
  • Errors Sincrònics vs Asincrònics: Els errors poden ocórrer tant en codi sincrònic com asincrònic. Gestionar-los requereix enfocaments diferents.
  • Bloc Try-Catch: Una manera comuna de gestionar errors sincrònics.
  • Callbacks amb Error Primer: Un patró utilitzat per gestionar errors en codi asincrònic.
  • Promises i Async/Await: Maneres modernes de gestionar errors asincrònics.

Gestió d'Errors Sincrònics

Els errors sincrònics ocorren durant l'execució d'una funció i es poden capturar utilitzant blocs try-catch.

Exemple

function divide(a, b) {
    if (b === 0) {
        throw new Error('Divisió per zero');
    }
    return a / b;
}

try {
    const result = divide(4, 0);
    console.log(result);
} catch (error) {
    console.error('Error:', error.message);
}

En aquest exemple, si b és zero, es llança un error, i el bloc catch el gestiona registrant el missatge d'error.

Gestió d'Errors Asincrònics

Els errors asincrònics ocorren en operacions com sol·licituds de xarxa, I/O de fitxers o temporitzadors. Gestionar aquests errors requereix tècniques diferents.

Callbacks amb Error Primer

Node.js utilitza el patró de callback amb error primer per gestionar errors asincrònics.

Exemple

const fs = require('fs');

fs.readFile('nonexistentfile.txt', 'utf8', (err, data) => {
    if (err) {
        console.error('Error:', err.message);
        return;
    }
    console.log(data);
});

En aquest exemple, si el fitxer no existeix, l'objecte err contindrà l'error, i es registrarà el missatge d'error.

Promises

Les promises proporcionen una manera més neta de gestionar operacions i errors asincrònics.

Exemple

const fs = require('fs').promises;

fs.readFile('nonexistentfile.txt', 'utf8')
    .then(data => {
        console.log(data);
    })
    .catch(err => {
        console.error('Error:', err.message);
    });

Async/Await

Async/Await és sucre sintàctic sobre les promises, fent que el codi asincrònic sembli sincrònic.

Exemple

const fs = require('fs').promises;

async function readFile() {
    try {
        const data = await fs.readFile('nonexistentfile.txt', 'utf8');
        console.log(data);
    } catch (err) {
        console.error('Error:', err.message);
    }
}

readFile();

Gestió d'Errors Avançada

Classes d'Errors Personalitzades

Crear classes d'errors personalitzades pot ajudar a categoritzar i gestionar diferents tipus d'errors de manera més efectiva.

Exemple

class ValidationError extends Error {
    constructor(message) {
        super(message);
        this.name = 'ValidationError';
    }
}

function validateUser(user) {
    if (!user.name) {
        throw new ValidationError('El nom és obligatori');
    }
    // Altres lògiques de validació
}

try {
    validateUser({});
} catch (error) {
    if (error instanceof ValidationError) {
        console.error('Error de Validació:', error.message);
    } else {
        console.error('Error:', error.message);
    }
}

Middleware de Gestió d'Errors en Express

En una aplicació Express, pots utilitzar middleware per gestionar errors globalment.

Exemple

const express = require('express');
const app = express();

app.get('/', (req, res) => {
    throw new Error('Alguna cosa ha anat malament');
});

// Middleware de gestió d'errors
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).send('Alguna cosa s'ha trencat!');
});

app.listen(3000, () => {
    console.log('El servidor està funcionant al port 3000');
});

Conclusió

Gestionar errors de manera efectiva és crucial per construir aplicacions fiables en Node.js. Entenent i implementant les diverses tècniques de gestió d'errors discutides en aquest tema, pots assegurar-te que la teva aplicació pugui gestionar situacions inesperades de manera elegant i proporcionar una millor experiència d'usuari.

  • Errors Sincrònics: Utilitza blocs try-catch.
  • Errors Asincrònics: Utilitza callbacks amb error primer, promises o async/await.
  • Errors Personalitzats: Crea classes d'errors personalitzades per a una millor categorització d'errors.
  • Middleware d'Express: Utilitza middleware per a la gestió global d'errors en aplicacions Express.

Dominant aquestes tècniques, estaràs ben preparat per gestionar errors en les teves aplicacions Node.js, des de scripts simples fins a servidors web complexos.
