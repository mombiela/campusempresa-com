El middleware és un concepte fonamental a Node.js, especialment quan es treballa amb frameworks com Express.js. Les funcions de middleware són funcions que tenen accés a l'objecte de sol·licitud (req), l'objecte de resposta (res) i la següent funció de middleware en el cicle de sol·licitud-resposta de l'aplicació. El middleware pot executar codi, modificar els objectes de sol·licitud i resposta, acabar el cicle de sol·licitud-resposta i cridar la següent funció de middleware a la pila.

Què és el Middleware?

  • Les funcions de middleware són funcions que tenen accés a l'objecte de sol·licitud (req), l'objecte de resposta (res) i la següent funció de middleware en el cicle de sol·licitud-resposta de l'aplicació.
  • El middleware pot realitzar les següents tasques:
    • Executar qualsevol codi.
    • Fer canvis als objectes de sol·licitud i resposta.
    • Acabar el cicle de sol·licitud-resposta.
    • Cridar la següent funció de middleware a la pila.

Tipus de Middleware

  1. Middleware a nivell d'aplicació
  2. Middleware a nivell de router
  3. Middleware de gestió d'errors
  4. Middleware integrat
  5. Middleware de tercers

Middleware a nivell d'aplicació

El middleware a nivell d'aplicació està vinculat a una instància de l'objecte app utilitzant app.use() i app.METHOD().

const express = require('express');
const app = express();

// Middleware a nivell d'aplicació
app.use((req, res, next) => {
  console.log('Hora:', Date.now());
  next();
});

app.get('/', (req, res) => {
  res.send('Hola Món!');
});

app.listen(3000, () => {
  console.log('El servidor està funcionant al port 3000');
});

Middleware a nivell de router

El middleware a nivell de router funciona de la mateixa manera que el middleware a nivell d'aplicació, excepte que està vinculat a una instància de express.Router().

const express = require('express');
const app = express();
const router = express.Router();

// Middleware a nivell de router
router.use((req, res, next) => {
  console.log('URL de la sol·licitud:', req.originalUrl);
  next();
});

router.get('/', (req, res) => {
  res.send('Hola des del Router!');
});

app.use('/router', router);

app.listen(3000, () => {
  console.log('El servidor està funcionant al port 3000');
});

Middleware de gestió d'errors

El middleware de gestió d'errors es defineix de la mateixa manera que altres funcions de middleware, excepte que té quatre arguments en lloc de tres: (err, req, res, next).

const express = require('express');
const app = express();

// Middleware de gestió d'errors
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Alguna cosa ha fallat!');
});

app.get('/', (req, res) => {
  throw new Error('TRENCAT'); // Express ho capturarà per si sol.
});

app.listen(3000, () => {
  console.log('El servidor està funcionant al port 3000');
});

Middleware integrat

Express ve amb algunes funcions de middleware integrades, com ara express.static, express.json i express.urlencoded.

const express = require('express');
const app = express();

// Middleware integrat
app.use(express.json()); // Per analitzar application/json
app.use(express.urlencoded({ extended: true })); // Per analitzar application/x-www-form-urlencoded

app.post('/data', (req, res) => {
  res.send(req.body);
});

app.listen(3000, () => {
  console.log('El servidor està funcionant al port 3000');
});

Middleware de tercers

El middleware de tercers es pot instal·lar a través de npm i després requerir a la teva aplicació. El middleware de tercers popular inclou morgan, body-parser i cookie-parser.

const express = require('express');
const morgan = require('morgan');
const app = express();

// Middleware de tercers
app.use(morgan('tiny'));

app.get('/', (req, res) => {
  res.send('Hola Món!');
});

app.listen(3000, () => {
  console.log('El servidor està funcionant al port 3000');
});

Conclusió

El middleware és una característica potent i flexible a Node.js que et permet gestionar diversos aspectes del processament de sol·licituds i respostes. Entendre com utilitzar diferents tipus de middleware de manera efectiva pot millorar significativament la funcionalitat i mantenibilitat de les teves aplicacions Node.js. Ja sigui per registrar sol·licituds, analitzar dades entrants, gestionar errors o servir fitxers estàtics, el middleware proporciona una manera estructurada de gestionar aquestes tasques.
