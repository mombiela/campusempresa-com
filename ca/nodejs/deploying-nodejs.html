Introducció al Desplegament

Desplegar una aplicació Node.js implica moure la teva aplicació d'un entorn de desenvolupament a un entorn de producció on pot ser accessible pels usuaris. Aquesta secció cobrirà els conceptes bàsics del desplegament, incloent els passos necessaris i les eines.

Conceptes Clau

  • Entorn de Desenvolupament: La configuració local on escrius i proves el teu codi.
  • Entorn de Producció: L'entorn en viu on la teva aplicació s'executa i és accessible pels usuaris.
  • Integració Contínua/Desplegament Continu (CI/CD): Pràctiques que automatitzen el procés de desplegament.

Preparant la teva Aplicació Node.js per al Desplegament

Abans de desplegar, has d'assegurar-te que la teva aplicació està llesta per a la producció.

Passos per Preparar

  1. Variables d'Entorn: Utilitza variables d'entorn per gestionar la configuració. 
    // config.js
module.exports = {
  port: process.env.PORT || 3000,
  dbUrl: process.env.DB_URL,
};
  2. Gestió d'Errors: Implementa una gestió d'errors robusta per gestionar problemes inesperats. 
    // app.js
process.on('uncaughtException', (err) => {
  console.error('Uncaught Exception:', err);
  process.exit(1);
});
  3. Registre: Utilitza biblioteques de registre com winston o morgan per fer un seguiment del comportament de l'aplicació. 
    const winston = require('winston');
const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' }),
  ],
});

Opcions de Desplegament

Hi ha diverses maneres de desplegar una aplicació Node.js. Aquí, cobrirem algunes opcions populars.

Desplegant a un Servidor Privat Virtual (VPS)

  1. Tria un Proveïdor de VPS: Exemples inclouen DigitalOcean, AWS EC2 i Linode.
  2. Configura el Servidor: Instal·la Node.js, npm i altres dependències. 
    sudo apt update
sudo apt install nodejs npm
  3. Transfereix la teva Aplicació: Utilitza scp o un sistema de control de versions com Git. 
    scp -r your-app-folder user@your-vps-ip:/path/to/destination
  4. Instal·la les Dependències: Navega al directori de la teva aplicació i instal·la les dependències. 
    cd /path/to/destination
npm install
  5. Inicia l'Aplicació: Utilitza un gestor de processos com pm2 per mantenir la teva aplicació en funcionament. 
    npm install -g pm2
pm2 start app.js

Desplegant a una Plataforma com a Servei (PaaS)

  1. Heroku: Una PaaS popular per desplegar aplicacions Node.js.

    • Instal·la Heroku CLI: Segueix les instruccions al lloc web de Heroku.
    • Crea una Aplicació Heroku: 
      heroku create your-app-name
    • Desplega el teu Codi: 
      git push heroku main
    • Gestiona les Variables d'Entorn: 
      heroku config:set NODE_ENV=production

  2. Altres Opcions PaaS: Exemples inclouen Vercel, Netlify i Google App Engine.

Integració Contínua/Desplegament Continu (CI/CD)

Automatitzar el procés de desplegament assegura que la teva aplicació estigui sempre actualitzada amb els últims canvis.

Configurant CI/CD amb GitHub Actions

  1. Crea un Fitxer de Flux de Treball: Afegeix un fitxer .github/workflows/deploy.yml al teu repositori. 
    name: Deploy Node.js App

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

    - name: Deploy to Heroku
      env:
        HEROKU_API_KEY: ${{ secrets.HEROKU_API_KEY }}
      run: |
        git remote add heroku https://git.heroku.com/your-app-name.git
        git push heroku main

Conclusió

Desplegar una aplicació Node.js implica diversos passos, des de preparar la teva aplicació per a la producció fins a triar la plataforma de desplegament adequada. Tant si optes per un VPS, una PaaS o una pipeline de CI/CD, entendre aquests conceptes t'ajudarà a assegurar-te que la teva aplicació sigui fiable i escalable. Seguint les millors pràctiques i utilitzant les eines adequades, pots agilitzar el procés de desplegament i centrar-te en construir grans aplicacions.
