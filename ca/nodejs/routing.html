Enrutament a Node.js
Servint Fitxers Estàtics ►

L'enrutament és un concepte fonamental en el desenvolupament web, que et permet definir com la teva aplicació respon a diferents sol·licituds HTTP. A Node.js, l'enrutament es maneja típicament utilitzant frameworks com Express.js. Aquesta secció et guiarà des dels conceptes bàsics de l'enrutament fins a tècniques més avançades.

Introducció a l'Enrutament

L'enrutament és el procés de determinar com una aplicació respon a una sol·licitud del client a un punt final particular, que és un URI (o camí) i un mètode de sol·licitud HTTP específic (GET, POST, etc.).

Conceptes Clau

  • Punt Final: Una combinació específica de camí URL i mètode HTTP.
  • Mètodes HTTP: Mètodes comuns inclouen GET, POST, PUT, DELETE, etc.
  • Gestor de Rutes: Una funció que s'executa quan es correspon amb una ruta específica.

Configurant Express.js

Per manejar l'enrutament a Node.js, utilitzarem el framework Express.js. Primer, configurem una aplicació bàsica d'Express.

Instal·lació

npm install express

Configuració Bàsica

const express = require('express');
const app = express();
const port = 3000;

app.listen(port, () => {
  console.log(`El servidor està funcionant a http://localhost:${port}`);
});

Enrutament Bàsic

Definint Rutes

Les rutes es defineixen utilitzant mètodes de l'objecte app que corresponen als mètodes HTTP. Per exemple:

app.get('/', (req, res) => {
  res.send('Hola, Món!');
});

app.post('/submit', (req, res) => {
  res.send('Formulari enviat!');
});

Paràmetres de Ruta

Els paràmetres de ruta són segments de l'URL amb nom que s'utilitzen per capturar valors en posicions específiques de l'URL.

app.get('/user/:id', (req, res) => {
  const userId = req.params.id;
  res.send(`ID d'usuari: ${userId}`);
});

Paràmetres de Consulta

Els paràmetres de consulta s'utilitzen per passar dades al servidor a través de l'URL.

app.get('/search', (req, res) => {
  const query = req.query.q;
  res.send(`Consulta de cerca: ${query}`);
});

Enrutament Avançat

Middleware

Les funcions middleware són funcions que tenen accés a l'objecte de sol·licitud (req), l'objecte de resposta (res), i la següent funció middleware en el cicle de sol·licitud-resposta de l'aplicació.

const logger = (req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

app.use(logger);

Grups de Rutes

Els grups de rutes et permeten agrupar rutes que comparteixen un prefix de camí comú.

const router = express.Router();

router.get('/profile', (req, res) => {
  res.send('Perfil d'usuari');
});

router.get('/settings', (req, res) => {
  res.send('Configuració d'usuari');
});

app.use('/user', router);

Maneig d'Errors

Les funcions middleware de maneig d'errors es defineixen de la mateixa manera que altres funcions middleware, excepte que tenen quatre arguments en lloc de tres: err, req, res, i next.

app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Alguna cosa ha fallat!');
});

Resum

L'enrutament és una part crucial de qualsevol aplicació web, definint com la teva aplicació respon a diverses sol·licituds HTTP. A Node.js, Express.js proporciona una manera senzilla i potent de manejar l'enrutament. Hem cobert:

  • Enrutament bàsic amb diferents mètodes HTTP.
  • Utilització de paràmetres de ruta i de consulta.
  • Tècniques avançades com middleware, grups de rutes, i maneig d'errors.

En dominar aquests conceptes, pots construir aplicacions web robustes i escalables amb Node.js i Express.js.
