Introducció a les Proves Unitàries

  • Definició: Les proves unitàries impliquen provar unitats o components individuals d'un programari per assegurar-se que funcionen com es preveu.
  • Importància: Ajuda a identificar errors aviat, simplifica la integració i assegura la fiabilitat del codi.

Configuració de Mocha i Chai

  • Mocha: Un marc de proves JavaScript ric en funcions que s'executa en Node.js, fent que les proves asíncrones siguin senzilles i divertides.
  • Chai: Una biblioteca d'asserts BDD / TDD per a Node.js que es pot combinar amb qualsevol marc de proves JavaScript.

Instal·lació de Mocha i Chai

Per començar, cal instal·lar Mocha i Chai utilitzant npm:

npm install --save-dev mocha chai

Configuració de Mocha

Crea un directori test a l'arrel del teu projecte i afegeix una configuració al teu package.json:

{
  "scripts": {
    "test": "mocha"
  }
}

Escriure la Teva Primera Prova

Estructura Bàsica de la Prova

Crea un fitxer anomenat test.js dins del directori test:

const assert = require('chai').assert;

describe('Array', function() {
  it('hauria de començar buit', function() {
    let arr = [];
    assert.equal(arr.length, 0);
  });
});
  • describe(): Agrupa proves relacionades.
  • it(): Defineix un cas de prova únic.
  • assert: Proporciona mètodes d'assert per verificar els resultats de les proves.

Executar les Proves

Executa les teves proves utilitzant la següent comanda:

npm test

Proves Avançades amb Mocha i Chai

Proves Asíncrones

Mocha admet proves asíncrones utilitzant callbacks, promeses o async/await.

Utilitzant Callbacks

describe('Prova Asíncrona', function() {
  it('hauria de completar-se asíncronament', function(done) {
    setTimeout(function() {
      assert.isTrue(true);
      done();
    }, 1000);
  });
});

Utilitzant Promeses

describe('Prova Asíncrona', function() {
  it('hauria de completar-se amb una promesa', function() {
    return new Promise((resolve) => {
      setTimeout(function() {
        assert.isTrue(true);
        resolve();
      }, 1000);
    });
  });
});

Utilitzant Async/Await

describe('Prova Asíncrona', function() {
  it('hauria de completar-se amb async/await', async function() {
    await new Promise((resolve) => setTimeout(resolve, 1000));
    assert.isTrue(true);
  });
});

Utilitzant Asserts de Chai

Chai proporciona tres estils d'asserts: Assert, Expect i Should.

Estil Assert

assert.equal(3, 3, '3 és igual a 3');

Estil Expect

const expect = require('chai').expect;
expect(3).to.equal(3);

Estil Should

const should = require('chai').should();
(3).should.equal(3);

Proves d'APIs HTTP amb Chai HTTP

Instal·la chai-http per provar punts finals HTTP:

npm install --save-dev chai-http

Exemple de Prova

const chai = require('chai');
const chaiHttp = require('chai-http');
const app = require('../app'); // La teva aplicació Express
chai.use(chaiHttp);

describe('GET /api', function() {
  it('hauria de retornar l'estat 200', function(done) {
    chai.request(app)
      .get('/api')
      .end(function(err, res) {
        chai.assert.equal(res.status, 200);
        done();
      });
  });
});

Mocking i Stubbing amb Sinon

Introducció a Sinon

Sinon és una biblioteca utilitzada per crear espies, mocks i stubs.

Instal·lació de Sinon

npm install --save-dev sinon

Exemple de Stubbing

const sinon = require('sinon');
const myModule = require('../myModule');

describe('myModule', function() {
  it('hauria de cridar el callback', function() {
    const callback = sinon.stub();
    myModule.doSomething(callback);
    sinon.assert.calledOnce(callback);
  });
});

Conclusió

Les proves unitàries són una part crucial del procés de desenvolupament, assegurant que els components individuals de la teva aplicació funcionin com s'espera. Mocha i Chai proporcionen una combinació poderosa per escriure i executar proves en Node.js. Dominant aquestes eines, pots escriure codi fiable, mantenible i robust. Recorda:

  • Comença amb proves senzilles i gradualment passa a escenaris més complexos.
  • Utilitza tècniques de proves asíncrones segons sigui necessari.
  • Aprofita els diversos estils d'asserts de Chai per fer les teves proves més llegibles.
  • Utilitza Sinon per a mocking i stubbing avançats.
