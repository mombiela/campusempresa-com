Introducció a MongoDB i Mongoose

  • MongoDB: Una base de dades NoSQL que emmagatzema dades en documents semblants a JSON.
  • Mongoose: Una biblioteca de Modelatge de Dades d'Objectes (ODM) per a MongoDB i Node.js, que proporciona una solució basada en esquemes per modelar les dades de la teva aplicació.

Per què utilitzar Mongoose?

  • Simplifica la validació de dades.
  • Proporciona un esquema per estructurar les teves dades.
  • Ofereix conversió de tipus integrada, construcció de consultes i ganxos de lògica empresarial.

Configurant MongoDB i Mongoose

Instal·lant MongoDB

  1. Descarrega i instal·la MongoDB des del lloc web oficial.
  2. Inicia el servidor MongoDB: 
    mongod

Instal·lant Mongoose

  1. Inicialitza un projecte Node.js si encara no ho has fet: 
    npm init -y
  2. Instal·la Mongoose: 
    npm install mongoose

Connectant-se a MongoDB amb Mongoose

Connexió bàsica

const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/mydatabase', {
  useNewUrlParser: true,
  useUnifiedTopology: true
}).then(() => {
  console.log('Connectat a MongoDB');
}).catch(err => {
  console.error('Error de connexió', err);
});
  • Explicació:
    • mongoose.connect: Connecta al servidor MongoDB.
    • useNewUrlParser i useUnifiedTopology: Opcions per evitar advertències de desaprovació.

Definint Esquemes i Models

Creant un Esquema

const { Schema } = mongoose;

const userSchema = new Schema({
  name: String,
  age: Number,
  email: String
});
  • Explicació:
    • Schema: Defineix l'estructura del document.
    • name, age, email: Camps del document amb els seus respectius tipus de dades.

Creant un Model

const User = mongoose.model('User', userSchema);
  • Explicació:
    • model: Compila l'esquema en un model, una classe amb la qual construïm documents.

Operacions CRUD

Crear

const newUser = new User({
  name: 'John Doe',
  age: 30,
  email: 'john.doe@example.com'
});

newUser.save().then(() => console.log('Usuari guardat'));
  • Explicació:
    • new User: Crea una nova instància del model User.
    • save: Guarda el document a la base de dades.

Llegir

User.find({ age: { $gte: 18 } }).then(users => {
  console.log(users);
});
  • Explicació:
    • find: Recupera documents que coincideixen amb la consulta.

Actualitzar

User.updateOne({ name: 'John Doe' }, { age: 31 }).then(() => {
  console.log('Usuari actualitzat');
});
  • Explicació:
    • updateOne: Actualitza un sol document que coincideix amb la consulta.

Esborrar

User.deleteOne({ name: 'John Doe' }).then(() => {
  console.log('Usuari esborrat');
});
  • Explicació:
    • deleteOne: Esborra un sol document que coincideix amb la consulta.

Temes Avançats

Middleware

  • Pre Middleware: S'executa abans d'un cert esdeveniment.

    userSchema.pre('save', function(next) {
  console.log('Abans de guardar');
  next();
});

  • Post Middleware: S'executa després d'un cert esdeveniment.

    userSchema.post('save', function(doc) {
  console.log('Després de guardar', doc);
});

Virtuals

  • Propietats Virtuals: Camps que no s'emmagatzemen a MongoDB però que es poden utilitzar com qualsevol altre camp. 
    userSchema.virtual('fullName').get(function() {
  return `${this.firstName} ${this.lastName}`;
});

Població

  • Poblant Referències: Reemplaça els camins especificats en el document amb document(s) d'una altra col·lecció. 
    const postSchema = new Schema({
  title: String,
  author: { type: Schema.Types.ObjectId, ref: 'User' }
});

const Post = mongoose.model('Post', postSchema);

Post.find().populate('author').then(posts => {
  console.log(posts);
});

Conclusió

  • Resum:
    • Mongoose simplifica les interaccions amb MongoDB proporcionant una solució basada en esquemes.
    • Les operacions CRUD bàsiques són senzilles amb Mongoose.
    • Funcionalitats avançades com middleware, virtuals i població milloren la funcionalitat i flexibilitat de la teva aplicació.

Dominant Mongoose, pots gestionar eficientment les teves dades de MongoDB i construir aplicacions robustes amb Node.js.
