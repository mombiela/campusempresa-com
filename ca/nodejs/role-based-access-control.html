El Control d'Accés Basat en Rols (RBAC) és un mètode per regular l'accés als recursos basat en els rols dels usuaris individuals dins d'una organització. Aquest tema et guiarà a través de la implementació de RBAC en una aplicació Node.js, des de conceptes bàsics fins a tècniques avançades.

Introducció al Control d'Accés Basat en Rols

  • Definició: RBAC és un mecanisme de control d'accés neutral en termes de polítiques, definit al voltant de rols i privilegis.
  • Propòsit: Simplificar la gestió de permisos assignant rols als usuaris i associant permisos amb aquests rols.
  • Beneficis:
    • Simplifica la gestió dels permisos d'usuari.
    • Millora la seguretat assegurant que els usuaris només tinguin els permisos que necessiten.
    • Escala bé amb el creixement de l'aplicació.

Configuració d'un Projecte Node.js

Abans d'entrar en RBAC, configurem un projecte bàsic de Node.js.

  1. Inicialitza un nou projecte Node.js:

    mkdir rbac-example
cd rbac-example
npm init -y

  2. Instal·la els paquets necessaris:

    npm install express jsonwebtoken bcryptjs

  3. Crea l'estructura del projecte:

    mkdir src
touch src/app.js src/routes.js src/controllers.js src/middleware.js

Conceptes Bàsics de RBAC

Rols i Permisos

  • Rols: Un rol és una col·lecció de permisos. Exemples inclouen admin, editor i viewer.
  • Permisos: Els permisos són drets d'accés específics als recursos, com ara read, write i delete.

Exemple de Mapeig de Rols i Permisos

| Rol | Permisos | |--------|----------------------| | Admin | read, write, delete | | Editor | read, write | | Viewer | read |

Implementació de RBAC en Node.js

Pas 1: Definir Rols i Permisos

Crea un fitxer src/roles.js per definir els rols i els seus permisos associats.

const roles = {
    admin: ['read', 'write', 'delete'],
    editor: ['read', 'write'],
    viewer: ['read']
};

module.exports = roles;

Pas 2: Middleware per a la Verificació de Rols

Crea un middleware per verificar si un usuari té el rol requerit per accedir a un recurs.

// src/middleware.js
const roles = require('./roles');

function authorize(requiredRole) {
    return (req, res, next) => {
        const userRole = req.user.role; // Suposant que req.user és omplert per un middleware d'autenticació
        if (roles[userRole] && roles[userRole].includes(requiredRole)) {
            next();
        } else {
            res.status(403).json({ message: 'Forbidden' });
        }
    };
}

module.exports = { authorize };

Pas 3: Aplicar Middleware a les Rutes

Utilitza el middleware a les teves rutes per protegir els endpoints basats en rols.

// src/routes.js
const express = require('express');
const { authorize } = require('./middleware');
const router = express.Router();

router.get('/admin', authorize('admin'), (req, res) => {
    res.send('Admin content');
});

router.get('/editor', authorize('editor'), (req, res) => {
    res.send('Editor content');
});

router.get('/viewer', authorize('viewer'), (req, res) => {
    res.send('Viewer content');
});

module.exports = router;

Pas 4: Integrar les Rutes a l'Aplicació

Integra les rutes al teu fitxer principal de l'aplicació.

// src/app.js
const express = require('express');
const app = express();
const routes = require('./routes');

// Middleware per simular l'autenticació d'usuaris
app.use((req, res, next) => {
    req.user = { role: 'admin' }; // Simula un usuari autenticat amb rol 'admin'
    next();
});

app.use('/api', routes);

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});

Tècniques Avançades de RBAC

Assignació Dinàmica de Rols

Els rols es poden assignar dinàmicament basant-se en dades d'usuari d'una base de dades.

// src/middleware.js
const roles = require('./roles');
const User = require('./models/User'); // Suposant que tens un model User

async function authorize(requiredRole) {
    return async (req, res, next) => {
        const user = await User.findById(req.user.id); // Recupera l'usuari de la base de dades
        const userRole = user.role;
        if (roles[userRole] && roles[userRole].includes(requiredRole)) {
            next();
        } else {
            res.status(403).json({ message: 'Forbidden' });
        }
    };
}

module.exports = { authorize };

Jerarquies de Rols

Implementa jerarquies de rols on els rols superiors hereten permisos dels rols inferiors.

const roles = {
    admin: ['read', 'write', 'delete'],
    editor: ['read', 'write'],
    viewer: ['read']
};

function getPermissions(role) {
    const roleHierarchy = {
        admin: ['editor', 'viewer'],
        editor: ['viewer'],
        viewer: []
    };

    let permissions = new Set(roles[role]);
    (roleHierarchy[role] || []).forEach(subRole => {
        roles[subRole].forEach(permission => permissions.add(permission));
    });

    return Array.from(permissions);
}

module.exports = { getPermissions };

Conclusió

El Control d'Accés Basat en Rols és un mètode poderós per gestionar els permisos d'usuari en una aplicació Node.js. Definint rols i permisos, creant middleware per a l'autorització, i aplicant aquests conceptes a les teves rutes, pots construir un sistema de control d'accés segur i escalable. Les tècniques avançades com l'assignació dinàmica de rols i les jerarquies de rols milloren encara més la flexibilitat i robustesa de la teva implementació de RBAC.
