Introducció a les Promeses

Les promeses són una manera moderna de gestionar operacions asíncrones a JavaScript, proporcionant un enfocament més llegible i manejable en comparació amb els callbacks tradicionals.

Conceptes Clau

  • Operacions Asíncrones: Operacions que no bloquegen l'execució del programa.
  • Estats de la Promesa:
    • Pending: Estat inicial, ni complerta ni rebutjada.
    • Fulfilled: Operació completada amb èxit.
    • Rejected: Operació fallida.
  • Mètodes de la Promesa:
    • .then(): Gestiona el compliment.
    • .catch(): Gestiona el rebuig.
    • .finally(): S'executa independentment del resultat.

Exemple Bàsic

const myPromise = new Promise((resolve, reject) => {
  let success = true; // Simula èxit o fracàs
  if (success) {
    resolve('L’operació ha estat exitosa!');
  } else {
    reject('L’operació ha fallat.');
  }
});

myPromise
  .then((message) => {
    console.log(message); // Sortida: L’operació ha estat exitosa!
  })
  .catch((error) => {
    console.error(error);
  });

Creació i Ús de Promeses

Creació d'una Promesa

Una promesa es crea utilitzant el constructor Promise, que pren una funció amb dos paràmetres: resolve i reject.

const promise = new Promise((resolve, reject) => {
  // Operació asíncrona
  setTimeout(() => {
    resolve('Dades obtingudes amb èxit!');
  }, 2000);
});

Gestió de Promeses amb `.then()`, `.catch()`, i `.finally()`

  • .then(): Utilitzat per gestionar el valor resolt de la promesa.
  • .catch(): Utilitzat per gestionar qualsevol error o rebuig.
  • .finally(): Executa codi després que la promesa s'hagi resolt, independentment del seu resultat.
promise
  .then((data) => {
    console.log(data); // Sortida: Dades obtingudes amb èxit!
  })
  .catch((error) => {
    console.error(error);
  })
  .finally(() => {
    console.log('Operació completada.');
  });

Enllaçar Promeses

Les promeses es poden enllaçar per gestionar múltiples operacions asíncrones en seqüència.

Exemple d'Enllaçament

const fetchData = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Dades obtingudes');
  }, 1000);
});

fetchData
  .then((data) => {
    console.log(data); // Sortida: Dades obtingudes
    return 'Processant dades';
  })
  .then((processMessage) => {
    console.log(processMessage); // Sortida: Processant dades
    return 'Dades processades';
  })
  .then((finalMessage) => {
    console.log(finalMessage); // Sortida: Dades processades
  })
  .catch((error) => {
    console.error(error);
  });

Patrons Avançats de Promeses

Promise.all

Promise.all s'utilitza per gestionar múltiples promeses simultàniament. Es resol quan totes les promeses de l'array s'han resolt.

const promise1 = Promise.resolve('Promesa 1 resolta');
const promise2 = Promise.resolve('Promesa 2 resolta');

Promise.all([promise1, promise2])
  .then((results) => {
    console.log(results); // Sortida: ['Promesa 1 resolta', 'Promesa 2 resolta']
  })
  .catch((error) => {
    console.error(error);
  });

Promise.race

Promise.race es resol o es rebutja tan aviat com una de les promeses de l'array es resol o es rebutja.

const promise1 = new Promise((resolve) => setTimeout(resolve, 500, 'Primer'));
const promise2 = new Promise((resolve) => setTimeout(resolve, 100, 'Segon'));

Promise.race([promise1, promise2])
  .then((result) => {
    console.log(result); // Sortida: Segon
  })
  .catch((error) => {
    console.error(error);
  });

Gestió d'Errors en Promeses

Una gestió adequada dels errors és crucial quan es treballa amb promeses per assegurar que els errors es capturen i es gestionen adequadament.

Exemple de Gestió d'Errors

const faultyPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    reject('Alguna cosa ha anat malament!');
  }, 1000);
});

faultyPromise
  .then((data) => {
    console.log(data);
  })
  .catch((error) => {
    console.error(error); // Sortida: Alguna cosa ha anat malament!
  });

Conclusió

Les promeses proporcionen una manera potent i flexible de gestionar operacions asíncrones a Node.js. Entenent i utilitzant les promeses, els desenvolupadors poden escriure codi més net, llegible i mantenible. Els punts clau inclouen:

  • Entendre els estats i mètodes de les promeses.
  • Crear i utilitzar promeses de manera efectiva.
  • Enllaçar promeses per a operacions seqüencials.
  • Utilitzar patrons avançats com Promise.all i Promise.race.
  • Gestionar adequadament els errors en les promeses.

Amb aquests conceptes, pots gestionar amb confiança operacions asíncrones en les teves aplicacions Node.js.
