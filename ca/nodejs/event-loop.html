Introducció

El bucle d'esdeveniments de Node.js és un concepte fonamental que permet a Node.js realitzar operacions d'E/S no bloquejants. Entendre com funciona el bucle d'esdeveniments és crucial per escriure aplicacions Node.js eficients i amb bon rendiment.

Conceptes clau

  • Naturalesa de fil únic: Node.js opera en un únic fil, però pot gestionar múltiples operacions simultàniament utilitzant el bucle d'esdeveniments.
  • E/S no bloquejant: Node.js utilitza operacions d'E/S no bloquejants per gestionar múltiples sol·licituds sense esperar que cap d'elles es completi.
  • Fases del bucle d'esdeveniments: El bucle d'esdeveniments té diverses fases, cadascuna responsable de diferents tipus d'operacions.

Fases del bucle d'esdeveniments

El bucle d'esdeveniments en Node.js consta de les següents fases:

  1. Timers: Aquesta fase executa les devolucions de trucada programades per setTimeout() i setInterval().
  2. Devolucions de trucada pendents: Executa les devolucions de trucada d'E/S ajornades a la següent iteració del bucle.
  3. Idle, Prepare: Ús intern només.
  4. Poll: Recupera nous esdeveniments d'E/S; executa devolucions de trucada relacionades amb l'E/S (excloent devolucions de trucada de tancament, temporitzadors i setImmediate()).
  5. Check: Executa devolucions de trucada programades per setImmediate().
  6. Devolucions de trucada de tancament: Executa devolucions de trucada de tancament, per exemple, socket.on('close', ...).

Exemple de codi: Bucle d'esdeveniments bàsic

Aquí teniu un exemple senzill per il·lustrar com funciona el bucle d'esdeveniments en Node.js:

const fs = require('fs');

console.log('Start');

setTimeout(() => {
  console.log('Timeout 1');
}, 0);

setImmediate(() => {
  console.log('Immediate 1');
});

fs.readFile(__filename, () => {
  console.log('File Read');
});

console.log('End');

Explicació

  1. Codi sincrònic: console.log('Start') i console.log('End') s'executen primer.
  2. Fase de temporitzadors: setTimeout(() => { console.log('Timeout 1'); }, 0); està programat per executar-se després de la fase de poll.
  3. Fase de check: setImmediate(() => { console.log('Immediate 1'); }); està programat per executar-se després de la fase de poll.
  4. Fase de poll: fs.readFile(__filename, () => { console.log('File Read'); }); llegeix el fitxer de manera asíncrona i programa la devolució de trucada.

Conceptes avançats

  • Microtasques: Les microtasques (per exemple, process.nextTick() i Promeses) s'executen immediatament després del script que s'està executant i abans que el bucle d'esdeveniments continuï.
  • Operacions bloquejants: Eviteu operacions bloquejants (per exemple, lectures de fitxers sincròniques) ja que poden aturar el bucle d'esdeveniments i degradar el rendiment.

Exemple de codi: Microtasques

console.log('Start');

setTimeout(() => {
  console.log('Timeout');
}, 0);

Promise.resolve().then(() => {
  console.log('Promise');
});

process.nextTick(() => {
  console.log('Next Tick');
});

console.log('End');

Explicació

  1. Codi sincrònic: console.log('Start') i console.log('End') s'executen primer.
  2. Microtasques: process.nextTick() i les devolucions de trucada de la Promesa s'executen abans que el bucle d'esdeveniments continuï.
  3. Fase de temporitzadors: setTimeout(() => { console.log('Timeout'); }, 0); està programat per executar-se després de la fase de poll.

Resum

Entendre el bucle d'esdeveniments de Node.js és essencial per escriure aplicacions eficients i amb bon rendiment. Punts clau a recordar:

  • Node.js és de fil únic però pot gestionar múltiples operacions simultàniament utilitzant el bucle d'esdeveniments.
  • El bucle d'esdeveniments té diverses fases, cadascuna responsable de diferents tipus d'operacions.
  • Les microtasques s'executen abans que el bucle d'esdeveniments continuï a la següent fase.
  • Eviteu operacions bloquejants per assegurar que el bucle d'esdeveniments funcioni sense problemes.

En dominar el bucle d'esdeveniments, podeu escriure aplicacions Node.js més eficients i responsives.
