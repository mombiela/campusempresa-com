La gestió d'errors és una part crucial per construir aplicacions robustes. En aquesta secció, tractarem com gestionar errors a Express, un popular framework web per a Node.js. Començarem amb els conceptes bàsics i avançarem cap a tècniques més avançades.

Introducció a la Gestió d'Errors

La gestió d'errors a Express implica capturar i gestionar errors que es produeixen durant l'execució de la teva aplicació. Això assegura que la teva aplicació pugui manejar situacions inesperades de manera elegant i proporcionar retroalimentació significativa als usuaris.

Conceptes Clau

  • Errors Sincrònics: Errors que es produeixen durant l'execució de codi sincrònic.
  • Errors Asincrònics: Errors que es produeixen en codi asincrònic, com ara callbacks, promeses o async/await.
  • Middleware: Funcions que tenen accés als objectes de sol·licitud i resposta i poden modificar-los o acabar el cicle de sol·licitud-resposta.

Gestió Bàsica d'Errors

Errors Sincrònics

A Express, els errors sincrònics es poden capturar utilitzant blocs try-catch. Aquí tens un exemple:

const express = require('express');
const app = express();

app.get('/', (req, res) => {
  try {
    // Simula un error
    throw new Error('Alguna cosa ha anat malament!');
  } catch (err) {
    res.status(500).send('Error Intern del Servidor');
  }
});

app.listen(3000, () => {
  console.log('El servidor està funcionant al port 3000');
});

Errors Asincrònics

Per al codi asincrònic, pots utilitzar promeses o async/await juntament amb blocs try-catch:

app.get('/async', async (req, res) => {
  try {
    // Simula un error asincrònic
    await Promise.reject(new Error('Error asincrònic!'));
  } catch (err) {
    res.status(500).send('Error Intern del Servidor');
  }
});

Middleware de Gestió d'Errors

Express proporciona un tipus especial de middleware per a la gestió d'errors. Aquest middleware té quatre paràmetres: err, req, res i next.

Exemple

const express = require('express');
const app = express();

// Ruta normal
app.get('/', (req, res) => {
  throw new Error('Ups!');
});

// Middleware de gestió d'errors
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Alguna cosa s'ha trencat!');
});

app.listen(3000, () => {
  console.log('El servidor està funcionant al port 3000');
});

Explicació

  • err: L'objecte d'error.
  • req: L'objecte de sol·licitud.
  • res: L'objecte de resposta.
  • next: La següent funció middleware en la pila.

Gestió Avançada d'Errors

Classes d'Errors Personalitzats

Crear classes d'errors personalitzats pot ajudar-te a gestionar diferents tipus d'errors de manera més efectiva.

class NotFoundError extends Error {
  constructor(message) {
    super(message);
    this.name = 'NotFoundError';
    this.statusCode = 404;
  }
}

app.get('/custom-error', (req, res, next) => {
  next(new NotFoundError('Aquest recurs no s'ha trobat'));
});

app.use((err, req, res, next) => {
  res.status(err.statusCode || 500).send(err.message);
});

Gestió Centralitzada d'Errors

Centralitzar la gestió d'errors pot fer que el teu codi sigui més net i fàcil de mantenir.

const errorHandler = (err, req, res, next) => {
  console.error(err.stack);
  res.status(err.statusCode || 500).send(err.message);
};

app.use(errorHandler);

Conclusió

La gestió d'errors a Express és essencial per construir aplicacions fiables. Entenent i implementant tant tècniques bàsiques com avançades de gestió d'errors, pots assegurar-te que la teva aplicació pugui manejar situacions inesperades de manera elegant i proporcionar retroalimentació significativa als usuaris.

Resum

  • Errors Sincrònics: Utilitza blocs try-catch.
  • Errors Asincrònics: Utilitza promeses o async/await amb blocs try-catch.
  • Middleware de Gestió d'Errors: Utilitza middleware amb quatre paràmetres (err, req, res, next).
  • Classes d'Errors Personalitzats: Crea classes d'errors personalitzats per a tipus d'errors específics.
  • Gestió Centralitzada d'Errors: Utilitza un gestor d'errors centralitzat per gestionar errors en un sol lloc.

Dominant aquestes tècniques, estaràs ben preparat per gestionar errors de manera efectiva a les teves aplicacions Express.
