Introducció als Microserveis

  • Definició: L'arquitectura de microserveis és un patró de disseny on una aplicació està composta per serveis petits i independents que es comuniquen a través d'APIs ben definides.
  • Beneficis:
    • Escalabilitat
    • Flexibilitat
    • Millora en l'aïllament de fallades
    • Desplegament independent
  • Reptes:
    • Complexitat en la gestió
    • Comunicació entre serveis
    • Consistència de dades

Configuració d'un Projecte Node.js per a Microserveis

  • Prerequisits:
    • Coneixements bàsics de Node.js i JavaScript
    • Node.js instal·lat a la teva màquina
    • npm (Node Package Manager) instal·lat

Inicialització d'un Projecte Node.js

mkdir microservices-demo
cd microservices-demo
npm init -y

Instal·lació de Paquets Essencials

npm install express body-parser axios

Creació d'un Microservei Simple

  • Estructura del Servei:
    • user-service/
      • index.js
      • package.json
    • order-service/
      • index.js
      • package.json

Servei d'Usuaris

user-service/index.js

const express = require('express');
const bodyParser = require('body-parser');

const app = express();
app.use(bodyParser.json());

let users = [
  { id: 1, name: 'John Doe' },
  { id: 2, name: 'Jane Doe' }
];

app.get('/users', (req, res) => {
  res.json(users);
});

app.listen(3001, () => {
  console.log('User service listening on port 3001');
});

Servei de Comandes

order-service/index.js

const express = require('express');
const bodyParser = require('body-parser');
const axios = require('axios');

const app = express();
app.use(bodyParser.json());

let orders = [
  { id: 1, userId: 1, product: 'Book' },
  { id: 2, userId: 2, product: 'Pen' }
];

app.get('/orders', (req, res) => {
  res.json(orders);
});

app.get('/orders/:id/user', async (req, res) => {
  const orderId = parseInt(req.params.id);
  const order = orders.find(o => o.id === orderId);
  if (order) {
    const user = await axios.get(`http://localhost:3001/users/${order.userId}`);
    res.json({ ...order, user: user.data });
  } else {
    res.status(404).send('Order not found');
  }
});

app.listen(3002, () => {
  console.log('Order service listening on port 3002');
});

Comunicació entre Serveis

  • Peticions HTTP: Utilitzant axios per comunicar-se entre serveis.
  • Descobriment de Serveis: Utilitzant eines com Consul, Eureka, o implementacions personalitzades.
  • API Gateway: Punt d'entrada centralitzat per gestionar i dirigir peticions.

Exemple de Comunicació entre Serveis

app.get('/orders/:id/user', async (req, res) => {
  const orderId = parseInt(req.params.id);
  const order = orders.find(o => o.id === orderId);
  if (order) {

    res.json({ ...order, user: user.data });
  } else {
    res.status(404).send('Order not found');
  }
});

Gestió de Dades en Microserveis

  • Base de Dades per Servei: Cada microservei gestiona la seva pròpia base de dades.
  • Consistència de Dades: Utilitzant consistència eventual i transaccions distribuïdes.
  • CQRS (Command Query Responsibility Segregation): Separant operacions de lectura i escriptura per optimitzar el rendiment.

Exemple de Base de Dades per Servei

  • Servei d'Usuaris: Gestiona dades relacionades amb els usuaris.
  • Servei de Comandes: Gestiona dades relacionades amb les comandes.

Temes Avançats

  • Service Mesh: Gestionant la comunicació de microserveis amb eines com Istio o Linkerd.
  • Arquitectura Basada en Esdeveniments: Utilitzant brokers de missatges com RabbitMQ o Kafka per a la comunicació asíncrona.
  • Seguretat: Implementant autenticació i autorització a través dels microserveis.

Exemple de Comunicació Basada en Esdeveniments

const amqp = require('amqplib/callback_api');

amqp.connect('amqp://localhost', (err, conn) => {
  conn.createChannel((err, ch) => {
    const queue = 'order_created';
    ch.assertQueue(queue, { durable: false });
    ch.sendToQueue(queue, Buffer.from(JSON.stringify({ orderId: 1, userId: 1 })));
    console.log(" [x] Sent 'Order Created'");
  });
});

Conclusió

  • Resum:
    • L'arquitectura de microserveis permet un desenvolupament d'aplicacions escalables i flexibles.
    • Node.js és adequat per construir microserveis gràcies al seu I/O no bloquejant i la seva naturalesa lleugera.
    • Els conceptes clau inclouen la comunicació entre serveis, la gestió de dades i temes avançats com el service mesh i l'arquitectura basada en esdeveniments.
  • Pròxims Passos:
    • Experimentar amb escenaris més complexos.
    • Explorar eines i frameworks addicionals per a microserveis.
    • Implementar millors pràctiques de seguretat per protegir els teus microserveis.

Entenent i aplicant aquests conceptes, pots construir i gestionar efectivament microserveis utilitzant Node.js, permetent-te crear aplicacions robustes, escalables i mantenibles.
