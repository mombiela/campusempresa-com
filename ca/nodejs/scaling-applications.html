Escalar Aplicacions
Monitorització i Optimització del Rendiment ►

Escalar aplicacions és un aspecte crucial del desenvolupament d'aplicacions Node.js robustes i d'alt rendiment. Aquest tema et guiarà a través dels fonaments de l'escalat, des de la comprensió dels conceptes bàsics fins a la implementació de tècniques avançades.

Introducció a l'Escalat

Escalar es refereix a la capacitat d'una aplicació per manejar una càrrega augmentada afegint recursos. Hi ha dos tipus principals d'escalat:

  • Escalat Vertical: Augmentar la capacitat d'un sol servidor (per exemple, afegint més CPU, RAM).
  • Escalat Horitzontal: Afegir més servidors per distribuir la càrrega.

Escalat Vertical

L'escalat vertical implica millorar les capacitats d'un sol servidor. Aquest mètode és senzill però té limitacions, com ara restriccions de maquinari i possibles punts únics de fallada.

Exemple: Augmentar els Recursos del Servidor

// Exemple d'augmentar els recursos del servidor en un entorn de núvol
const server = require('http').createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hola Món\n');
});

server.listen(3000, '127.0.0.1');
console.log('Servidor en funcionament a http://127.0.0.1:3000/');

// Per escalar verticalment, podries actualitzar la instància del servidor al tauler de control del teu proveïdor de núvol

Escalat Horitzontal

L'escalat horitzontal implica afegir més servidors per manejar la càrrega. Aquest mètode és més escalable i tolerant a fallades.

Balanceig de Càrrega

El balanceig de càrrega és una tècnica per distribuir el trànsit de xarxa entrant entre múltiples servidors. Això assegura que cap servidor individual suporti massa càrrega.

Exemple: Utilitzar Nginx com a Balancejador de Càrrega

  1. Instal·lar Nginx:

    sudo apt-get update
sudo apt-get install nginx

  2. Configurar Nginx: Edita el fitxer de configuració de Nginx (normalment situat a /etc/nginx/nginx.conf):

    http {
    upstream node_app {
        server 127.0.0.1:3000;
        server 127.0.0.1:3001;
        server 127.0.0.1:3002;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://node_app;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

  3. Iniciar Múltiples Instàncies de Node.js:

    const http = require('http');

const requestHandler = (req, res) => {
  res.end('Hola des del servidor Node.js');
};

const servers = [3000, 3001, 3002].map(port => {
  const server = http.createServer(requestHandler);
  server.listen(port, () => {
    console.log(`Servidor en funcionament al port ${port}`);
  });
  return server;
});

Clustering

Node.js proporciona un mòdul integrat anomenat cluster per aprofitar els sistemes multi-nucli creant processos fills (treballadors) que comparteixen el mateix port del servidor.

Exemple: Utilitzar el Mòdul Cluster

const cluster = require('cluster');
const http = require('http');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  console.log(`El mestre ${process.pid} està en funcionament`);

  // Crear treballadors
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }

  cluster.on('exit', (worker, code, signal) => {
    console.log(`El treballador ${worker.process.pid} ha mort`);
  });
} else {
  // Els treballadors poden compartir qualsevol connexió TCP
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('Hola Món\n');
  }).listen(8000);

  console.log(`El treballador ${process.pid} ha començat`);
}

Tècniques Avançades d'Escalat

Microserveis

L'arquitectura de microserveis implica dividir una aplicació en serveis més petits i independents. Cada servei es pot desenvolupar, desplegar i escalar de manera independent.

Exemple: Configuració Bàsica de Microserveis

  1. Servei A (Servei d'Usuaris):

    const express = require('express');
const app = express();

app.get('/users', (req, res) => {
  res.json([{ id: 1, name: 'John Doe' }]);
});

app.listen(3000, () => {
  console.log('Servei d\'usuaris en funcionament al port 3000');
});

  2. Servei B (Servei de Comandes):

    const express = require('express');
const app = express();

app.get('/orders', (req, res) => {
  res.json([{ id: 1, user_id: 1, product: 'Llibre' }]);
});

app.listen(3001, () => {
  console.log('Servei de comandes en funcionament al port 3001');
});

Contenidorització

La contenidorització implica empaquetar una aplicació i les seves dependències en un contenidor, que pot funcionar de manera consistent en diferents entorns.

Exemple: Dockeritzar una Aplicació Node.js

  1. Crear un Dockerfile:

    FROM node:14

WORKDIR /usr/src/app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 8080
CMD [ "node", "server.js" ]

  2. Construir i Executar el Contenidor Docker:

    docker build -t my-node-app .
docker run -p 8080:8080 -d my-node-app

Conclusió

Escalar aplicacions Node.js és essencial per manejar càrregues augmentades i assegurar una alta disponibilitat. Comprenent i implementant l'escalat vertical, l'escalat horitzontal, el clustering, els microserveis i la contenidorització, pots construir aplicacions escalables i resilients. Comença amb tècniques bàsiques i gradualment passa a mètodes més avançats a mesura que creixin les necessitats de la teva aplicació.
