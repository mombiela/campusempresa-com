Introducció

Les operacions del sistema de fitxers són una part crucial de moltes aplicacions, permetent-te llegir, escriure i manipular fitxers i directoris. Node.js proporciona un mòdul integrat anomenat fs per gestionar aquestes operacions de manera eficient.

Operacions Bàsiques del Sistema de Fitxers

Llegir Fitxers

Llegir fitxers és una de les operacions més comunes. Node.js proporciona mètodes sincrònics i asincrònics per llegir fitxers.

Lectura Asincrònica

Els mètodes asincrònics són no bloquejants, és a dir, el programa pot continuar executant-se mentre el fitxer s'està llegint.

const fs = require('fs');

fs.readFile('example.txt', 'utf8', (err, data) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(data);
});

Lectura Sincrònica

Els mètodes sincrònics bloquegen l'execució fins que l'operació es completa.

const fs = require('fs');

try {
  const data = fs.readFileSync('example.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

Escriure Fitxers

Escriure fitxers també es pot fer de manera asincrònica o sincrònica.

Escriptura Asincrònica

const fs = require('fs');

const content = 'Aquest és un contingut per escriure al fitxer.';

fs.writeFile('example.txt', content, 'utf8', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('El fitxer ha estat escrit');
});

Escriptura Sincrònica

const fs = require('fs');

const content = 'Aquest és un contingut per escriure al fitxer.';

try {
  fs.writeFileSync('example.txt', content, 'utf8');
  console.log('El fitxer ha estat escrit');
} catch (err) {
  console.error(err);
}

Afegir a Fitxers

Afegir dades a un fitxer existent es pot fer utilitzant fs.appendFile i fs.appendFileSync.

Afegir Asincrònicament

const fs = require('fs');

const content = '\nAquest és un contingut addicional.';

fs.appendFile('example.txt', content, 'utf8', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('El contingut ha estat afegit');
});

Afegir Sincrònicament

const fs = require('fs');

const content = '\nAquest és un contingut addicional.';

try {
  fs.appendFileSync('example.txt', content, 'utf8');
  console.log('El contingut ha estat afegit');
} catch (err) {
  console.error(err);
}

Esborrar Fitxers

Esborrar fitxers es pot fer utilitzant fs.unlink i fs.unlinkSync.

Esborrar Asincrònicament

const fs = require('fs');

fs.unlink('example.txt', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('El fitxer ha estat esborrat');
});

Esborrar Sincrònicament

const fs = require('fs');

try {
  fs.unlinkSync('example.txt');
  console.log('El fitxer ha estat esborrat');
} catch (err) {
  console.error(err);
}

Operacions Avançades del Sistema de Fitxers

Treballar amb Directoris

Node.js et permet crear, llegir i esborrar directoris.

Crear Directoris

const fs = require('fs');

// Asincrònicament
fs.mkdir('newDir', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('Directori creat');
});

// Sincrònicament
try {
  fs.mkdirSync('newDir');
  console.log('Directori creat');
} catch (err) {
  console.error(err);
}

Llegir Directoris

const fs = require('fs');

// Asincrònicament
fs.readdir('.', (err, files) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(files);
});

// Sincrònicament
try {
  const files = fs.readdirSync('.');
  console.log(files);
} catch (err) {
  console.error(err);
}

Esborrar Directoris

const fs = require('fs');

// Asincrònicament
fs.rmdir('newDir', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('Directori esborrat');
});

// Sincrònicament
try {
  fs.rmdirSync('newDir');
  console.log('Directori esborrat');
} catch (err) {
  console.error(err);
}

Vigilar Fitxers i Directoris

Node.js et permet vigilar canvis en fitxers i directoris utilitzant fs.watch.

const fs = require('fs');

fs.watch('example.txt', (eventType, filename) => {
  if (filename) {
    console.log(`${filename} fitxer canviat`);
  }
});

Conclusió

Entendre les operacions del sistema de fitxers en Node.js és essencial per desenvolupar aplicacions robustes. Tant si estàs llegint, escrivint, afegint o esborrant fitxers, Node.js proporciona mètodes sincrònics i asincrònics per adaptar-se a les teves necessitats. A més, treballar amb directoris i vigilar canvis pot ajudar-te a construir aplicacions més dinàmiques i responsives.
