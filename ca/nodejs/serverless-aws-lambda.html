Introducció a la Computació sense servidor

  • Definició: La computació sense servidor et permet construir i executar aplicacions i serveis sense pensar en servidors. AWS Lambda és un servei de computació sense servidor que et permet executar codi sense aprovisionar ni gestionar servidors.
  • Beneficis:
    • Sense gestió de servidors
    • Escalabilitat flexible
    • Alta disponibilitat
    • Rendible

Configuració d'AWS Lambda

  • Prerequisits:

    • Compte d'AWS
    • Coneixements bàsics de la Consola de Gestió d'AWS
    • Node.js instal·lat a la teva màquina local

  • Passos per configurar:

    1. Crear un compte d'AWS: Si encara no en tens un, registra't a AWS.
    2. Instal·lar AWS CLI: Segueix les instruccions aquí.
    3. Configurar AWS CLI: Executa aws configure i introdueix el teu AWS Access Key ID, Secret Access Key, regió i format de sortida.

Creació de la teva primera funció Lambda

  • Creació d'una funció Lambda mitjançant la consola d'AWS:

    1. Ves a la Consola de Gestió d'AWS.
    2. Navega al servei Lambda.
    3. Fes clic a "Create function".
    4. Tria "Author from scratch".
    5. Introdueix un nom per a la funció.
    6. Tria "Node.js" com a runtime.
    7. Fes clic a "Create function".

  • Codi d'exemple:

    exports.handler = async (event) => {
    const response = {
        statusCode: 200,
        body: JSON.stringify('Hello from Lambda!'),
    };
    return response;
};

  • Desplegament de la funció:

    1. A l'editor de funcions, enganxa el codi d'exemple.
    2. Fes clic a "Deploy".

Invocació de funcions Lambda

  • Invocació manual:

    • Utilitza el botó "Test" a la consola d'AWS per invocar manualment la funció.

  • Invocació automàtica:

    • Configura triggers com API Gateway, esdeveniments S3, streams de DynamoDB, etc.

  • Exemple amb API Gateway:

    1. Crea una API a API Gateway.
    2. Crea un recurs i un mètode.
    3. Integra el mètode amb la funció Lambda.

Conceptes avançats de Lambda

  • Variables d'entorn:

    • Emmagatzema configuracions i secrets.
    • Exemple: 
      const apiKey = process.env.API_KEY;

  • Capes:

    • Llibreries de codi reutilitzables que es poden compartir entre diverses funcions Lambda.
    • Exemple: Empaquetar funcions utilitàries comunes.

  • Concurrència i escalabilitat:

    • Configura la concurrència reservada per controlar el nombre d'instàncies.
    • Entén el comportament i els límits d'escalabilitat.

  • Gestió d'errors:

    • Utilitza blocs try-catch en el teu codi.
    • Configura Dead Letter Queues (DLQ) per a invocacions fallides.

Monitorització i registre

  • CloudWatch Logs:

    • Captura automàticament registres de les funcions Lambda.
    • Exemple de registre en Lambda: 
      console.log('This is a log message');

  • Mètriques:

    • Monitoritza el recompte d'invocacions, durada, recompte d'errors, etc.
    • Configura CloudWatch Alarms per a mètriques crítiques.

Millors pràctiques

  • Optimització del codi:

    • Minimitza la mida del paquet.
    • Utilitza pràctiques de codificació eficients per reduir el temps d'execució.

  • Seguretat:

    • Utilitza rols IAM amb el menor privilegi.
    • Xifra variables d'entorn i dades sensibles.

  • Gestió de costos:

    • Monitoritza l'ús i configura alertes de facturació.
    • Optimitza el temps d'execució de les funcions per reduir costos.

Conclusió

La computació sense servidor amb AWS Lambda proporciona una manera potent i flexible de construir i desplegar aplicacions sense la càrrega de gestionar servidors. Aprofitant AWS Lambda amb Node.js, pots crear solucions escalables i rendibles que són fàcils de mantenir i desplegar. Aquest curs ha cobert els conceptes bàsics de configuració i ús d'AWS Lambda, conceptes avançats i millors pràctiques per assegurar-te que pots utilitzar efectivament la computació sense servidor en els teus projectes.
