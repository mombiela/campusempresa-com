Introducció

En aquesta secció, revisarem i resumirem els conceptes clau, eines i tècniques cobertes al llarg del curs d'Ansible. Això ajudarà a reforçar la teva comprensió i assegurar-te que estàs ben preparat per aplicar Ansible en escenaris del món real.

Conceptes Clau

Conceptes Bàsics d'Ansible

  • Què és Ansible?
    • Ansible és una eina d'automatització de codi obert utilitzada per a la gestió de configuracions, desplegament d'aplicacions i automatització de tasques.
  • Característiques Clau:
    • Arquitectura sense agent
    • Sintaxi YAML senzilla
    • Idempotència

Inventari

  • Fitxers d'Inventari:
    • Defineixen els hosts i grups de hosts sobre els quals operen les comandes, mòduls i playbooks d'Ansible.
    • Exemple: 
      [webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com
  • Inventari Dinàmic:
    • Permet la generació dinàmica d'inventari basat en fonts externes com proveïdors de núvol.

Mòduls

  • Mòduls Bàsics:
    • Preinstal·lats amb Ansible i cobreixen una àmplia gamma de funcionalitats.
    • Exemple: 
      - name: Instal·lar Nginx
  ansible.builtin.yum:
    name: nginx
    state: present
  • Mòduls Personalitzats:
    • Mòduls definits per l'usuari per estendre les capacitats d'Ansible.

Playbooks

  • Estructura:
    • Escrits en YAML i defineixen una sèrie de tasques a executar en un conjunt de hosts.
    • Exemple: 
      - hosts: webservers
  tasks:
    - name: Instal·lar Nginx
      ansible.builtin.yum:
        name: nginx
        state: present
  • Handlers:
    • Tasques especials que s'executen només quan són notificades per altres tasques.
  • Variables:
    • Utilitzades per emmagatzemar valors que es poden reutilitzar al llarg dels playbooks.

Rols

  • Propòsit:
    • Organitzar els playbooks en components reutilitzables.
  • Estructura:
    • Normalment inclouen directoris com tasks, handlers, files, templates i vars.

Ansible Galaxy

  • Què és Ansible Galaxy?
    • Un repositori per a rols d'Ansible, permetent als usuaris compartir i descarregar rols.
  • Ús d'Ansible Galaxy:
    • Exemple: 
      ansible-galaxy install username.role_name

Temes Avançats

Ansible Vault

  • Propòsit:
    • Emmagatzemar de manera segura dades sensibles com contrasenyes i claus.
  • Ús:
    • Xifrar un fitxer: 
      ansible-vault encrypt secrets.yml
    • Desxifrar un fitxer: 
      ansible-vault decrypt secrets.yml

Ansible Tower

  • Què és Ansible Tower?
    • Una solució basada en web que fa Ansible encara més potent proporcionant una interfície gràfica, API REST i altres característiques empresarials.
  • Característiques Clau:
    • Control d'accés basat en rols
    • Programació de tasques
    • Actualitzacions d'estat de tasques en temps real

Plugins Personalitzats

  • Tipus de Plugins:
    • Plugins d'Acció, Callback, Connexió, Lookup i Filtre.
  • Creació d'un Plugin Personalitzat:
    • Exemple d'un simple plugin de filtre: 
      # filter_plugins/my_filter.py
def to_uppercase(value):
    return value.upper()

class FilterModule(object):
    def filters(self):
        return {
            'to_uppercase': to_uppercase
        }

Resum

En aquest curs, hem cobert una àmplia gamma de temes des dels conceptes bàsics d'Ansible fins a característiques avançades i millors pràctiques. Aquí tens un resum ràpid:

  • Conceptes Bàsics d'Ansible: Comprendre què és Ansible i les seves característiques clau.
  • Gestió d'Inventari: Com definir i gestionar inventaris.
  • Mòduls i Playbooks: Escriure i executar tasques utilitzant mòduls i playbooks.
  • Rols i Ansible Galaxy: Organitzar playbooks i utilitzar rols compartits.
  • Característiques Avançades: Utilitzar Ansible Vault per a la seguretat, Ansible Tower per a característiques empresarials i crear plugins personalitzats.

En dominar aquests temes, ara estàs equipat per utilitzar Ansible de manera efectiva per automatitzar tasques de TI i gestionar desplegaments complexos. Continua practicant i explorant casos d'ús més avançats per millorar encara més les teves habilitats.
