Introducció a CI/CD

La Integració Contínua (CI) i el Desplegament Continu (CD) són pràctiques essencials en el desenvolupament de programari modern. El seu objectiu és automatitzar el procés d'integració de canvis de codi, proves i desplegament d'aplicacions en entorns de producció. Ansible, una eina d'automatització potent, es pot utilitzar per agilitzar aquests processos.

Conceptes Clau

  • Integració Contínua (CI): La pràctica d'integrar freqüentment canvis de codi en un repositori compartit, seguida de compilacions i proves automàtiques.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa les proves automàtiques a producció.
  • Ansible: Una eina d'automatització de codi obert utilitzada per a la gestió de configuracions, desplegament d'aplicacions i automatització de tasques.

Configuració d'Ansible per a CI/CD

Prerequisits

  • Ansible instal·lat a la teva màquina local o servidor CI.
  • Un sistema de control de versions (per exemple, Git) per gestionar la teva base de codi.
  • Una plataforma CI/CD (per exemple, Jenkins, GitLab CI, GitHub Actions).

Estructura Bàsica d'un Playbook d'Ansible

Un playbook d'Ansible és un fitxer YAML que defineix una sèrie de tasques a executar en hosts remots.

---
- name: Desplegar aplicació
  hosts: webservers
  become: yes
  tasks:
    - name: Assegurar que es descarrega la darrera versió de l'aplicació
      git:
        repo: 'https://github.com/your-repo/your-app.git'
        dest: /var/www/your-app
        version: master

    - name: Instal·lar dependències
      command: npm install
      args:
        chdir: /var/www/your-app

    - name: Reiniciar el servidor web
      service:
        name: apache2
        state: restarted

Integració d'Ansible amb Eines de CI

Exemple: Jenkins

  1. Instal·lar el Plugin d'Ansible: Assegura't que el plugin d'Ansible està instal·lat a Jenkins.
  2. Crear un Pipeline de Jenkins: Defineix un Jenkinsfile al teu repositori.
pipeline {
    agent any

    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/your-repo/your-app.git'
            }
        }
        stage('Install Ansible') {
            steps {
                sh 'sudo apt-get update && sudo apt-get install -y ansible'
            }
        }
        stage('Run Ansible Playbook') {
            steps {
                ansiblePlaybook(
                    playbook: 'deploy.yml',
                    inventory: 'inventory/hosts'
                )
            }
        }
    }
}

Exemple: GitLab CI

  1. Definir .gitlab-ci.yml: Crea un fitxer .gitlab-ci.yml al teu repositori.
stages:
  - deploy

deploy:
  stage: deploy
  script:
    - apt-get update -qq && apt-get install -qq -y ansible
    - ansible-playbook -i inventory/hosts deploy.yml
  only:
    - master

Tècniques Avançades d'Ansible per a CI/CD

  • Rols: Organitza els teus playbooks en rols reutilitzables.
  • Vault: Protegeix dades sensibles utilitzant Ansible Vault.
  • Inventari Dinàmic: Utilitza scripts d'inventari dinàmic per gestionar hosts en entorns de núvol.

Exemple: Utilitzant Rols

---
- name: Desplegar aplicació
  hosts: webservers
  become: yes
  roles:
    - { role: git }
    - { role: npm }
    - { role: apache }

Exemple: Utilitzant Ansible Vault

ansible-vault create secret.yml
---
- name: Desplegar aplicació
  hosts: webservers
  become: yes
  vars_files:
    - secret.yml
  tasks:
    - name: Assegurar que es descarrega la darrera versió de l'aplicació
      git:
        repo: 'https://github.com/your-repo/your-app.git'
        dest: /var/www/your-app
        version: master
        key_file: "{{ vault_ssh_key }}"

Conclusió

Ansible és una eina versàtil que pot millorar significativament els teus pipelines de CI/CD automatitzant el procés de desplegament. Integrant Ansible amb plataformes CI/CD populars com Jenkins i GitLab CI, pots assegurar-te que les teves aplicacions es despleguen de manera consistent i fiable. A mesura que avances, aprofitar les funcions avançades d'Ansible com rols, vault i inventari dinàmic agilitzarà i protegirà encara més els teus processos de desplegament.
