Introducció

Els exàmens de pràctica i les preguntes de mostra són eines essencials per dominar Ansible. Ajuden a reforçar l'aprenentatge, identificar àrees de millora i construir confiança. Aquesta secció proporciona una varietat de preguntes de pràctica i exàmens simulats, que van des de nivells per a principiants fins a avançats, per ajudar-te a consolidar la teva comprensió d'Ansible.

Nivell per a principiants

Conceptes clau

  • Comandes bàsiques d'Ansible
  • Fitxers d'inventari
  • Playbooks i sintaxi YAML
  • Mòduls i tasques

Preguntes de mostra

  1. Quina comanda s'utilitza per comprovar la versió d'Ansible instal·lada al teu sistema?

    ansible --version

  2. Escriu un fitxer d'inventari simple per a dos hosts: webserver i dbserver.

    [web]
webserver

[db]
dbserver

  3. Crea un playbook bàsic per instal·lar nginx a webserver.

    ---
- name: Install nginx on webserver
  hosts: web
  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present

  4. Explica la finalitat del fitxer ansible.cfg.

    • El fitxer ansible.cfg s'utilitza per configurar els paràmetres d'Ansible com la ubicació de l'inventari, l'usuari remot i la configuració SSH.

Nivell intermedi

Conceptes clau

  • Rols i estructura de rols
  • Variables i fets
  • Gestors i notificacions
  • Condicionals i bucles

Preguntes de mostra

  1. Quina és la finalitat dels rols a Ansible?

    • Els rols ajuden a organitzar els playbooks agrupant tasques, variables, fitxers, plantilles i gestors en components reutilitzables.

  2. Escriu un playbook que utilitzi una variable per instal·lar un paquet especificat per l'usuari.

    ---
- name: Install a package
  hosts: all
  vars:
    package_name: "default_package"
  tasks:
    - name: Install the specified package
      apt:
        name: "{{ package_name }}"
        state: present

  3. Crea un gestor per reiniciar nginx i notifica'l en una tasca.

    ---
- name: Install and configure nginx
  hosts: web
  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present
      notify:
        - Restart nginx

  handlers:
    - name: Restart nginx
      service:
        name: nginx
        state: restarted

  4. Explica com utilitzar condicionals en les tasques d'Ansible. Proporciona un exemple.

    • Els condicionals a Ansible s'utilitzen per executar tasques basades en certes condicions. Es defineixen utilitzant la paraula clau when.
    - name: Install nginx only on Ubuntu
  apt:
    name: nginx
    state: present
  when: ansible_os_family == "Debian"

Nivell avançat

Conceptes clau

  • Mòduls personalitzats
  • Ansible Vault
  • Inventari dinàmic
  • Estratègies avançades de playbook

Preguntes de mostra

  1. Descriu com crear un mòdul personalitzat d'Ansible.

    • Els mòduls personalitzats es poden crear utilitzant Python o qualsevol altre llenguatge de script. Han de retornar un objecte JSON amb claus específiques com changed i msg.

  2. Escriu un playbook que utilitzi Ansible Vault per xifrar dades sensibles.

    ---
- name: Decrypt and use sensitive data
  hosts: all
  vars_files:
    - vault.yml
  tasks:
    - name: Use sensitive data
      debug:
        msg: "The secret is {{ secret_value }}"
    • Per xifrar el fitxer vault.yml: 
      ansible-vault encrypt vault.yml

  3. Explica la finalitat de l'inventari dinàmic i proporciona un exemple de script.

    • L'inventari dinàmic permet a Ansible consultar fonts de dades externes per generar inventari dinàmicament.
    # dynamic_inventory.py
import json

inventory = {
    "web": {
        "hosts": ["webserver1", "webserver2"]
    },
    "db": {
        "hosts": ["dbserver1"]
    }
}

print(json.dumps(inventory))
    • Per utilitzar l'script: 
      ansible-playbook -i dynamic_inventory.py playbook.yml

  4. Discuteix estratègies avançades de playbook com serial, max_fail_percentage i delegate_to.

    • serial: Controla el nombre d'hosts que es gestionaran alhora.
    • max_fail_percentage: Defineix el percentatge màxim d'hosts que poden fallar abans que es cancel·li el play.
    • delegate_to: Executa una tasca en un host diferent del que s'ha especificat a la directiva hosts.

Conclusió

Els exàmens de pràctica i les preguntes de mostra són invaluables per dominar Ansible. Treballant aquestes preguntes, pots identificar els teus punts forts i àrees de millora, assegurant-te que estàs ben preparat per a escenaris reals i exàmens de certificació. Continua practicant i explorant les vastes capacitats d'Ansible per convertir-te en un expert en automatització i gestió de configuracions.
