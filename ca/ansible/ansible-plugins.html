Els plugins d'Ansible són una característica poderosa que amplia la funcionalitat d'Ansible. Permeten personalitzar i millorar el comportament d'Ansible de diverses maneres. Aquesta secció cobrirà els diferents tipus de plugins d'Ansible, com utilitzar-los i com crear els teus propis plugins.

Tipus de Plugins d'Ansible

Ansible suporta diversos tipus de plugins, cadascun amb una finalitat diferent:

  • Plugins d'Acció: Modifiquen el comportament dels mòduls d'Ansible.
  • Plugins de Consulta: Recuperen dades de fonts externes.
  • Plugins de Filtre: Transformen dades dins dels playbooks d'Ansible.
  • Plugins de Callback: Personalitzen la sortida de les execucions d'Ansible.
  • Plugins de Connexió: Gestionen la connexió amb hosts remots.
  • Plugins d'Inventari: Generen dinàmicament inventaris a partir de fonts externes.
  • Plugins de Variables: Carreguen variables de fonts externes.
  • Plugins de Test: Proporcionen proves addicionals de Jinja2.

Ús dels Plugins d'Ansible

Els plugins d'Ansible es poden utilitzar directament en els teus playbooks o rols. Aquí tens alguns exemples de com utilitzar diferents tipus de plugins:

Plugins d'Acció

Els plugins d'acció permeten modificar el comportament dels mòduls. Per exemple, pots utilitzar un plugin d'acció per afegir registre addicional o per canviar com un mòdul maneja els errors.

---
- name: Exemple de playbook utilitzant un plugin d'acció
  hosts: localhost
  tasks:
    - name: Utilitza el plugin d'acció personalitzat
      my_custom_action:
        param1: value1
        param2: value2

Plugins de Consulta

Els plugins de consulta s'utilitzen per recuperar dades de fonts externes. Per exemple, pots utilitzar un plugin de consulta per llegir dades d'un fitxer o una base de dades.

---
- name: Exemple de playbook utilitzant un plugin de consulta
  hosts: localhost
  tasks:
    - name: Recupera dades utilitzant un plugin de consulta
      debug:
        msg: "{{ lookup('file', '/path/to/file.txt') }}"

Plugins de Filtre

Els plugins de filtre s'utilitzen per transformar dades dins dels playbooks d'Ansible. Per exemple, pots utilitzar un plugin de filtre per formatar una data o per manipular cadenes de text.

---
- name: Exemple de playbook utilitzant un plugin de filtre
  hosts: localhost
  tasks:
    - name: Transforma dades utilitzant un plugin de filtre
      debug:
        msg: "{{ '2023-10-01' | to_datetime | date('long') }}"

Plugins de Callback

Els plugins de callback permeten personalitzar la sortida de les execucions d'Ansible. Per exemple, pots utilitzar un plugin de callback per enviar notificacions o per registrar resultats en un fitxer.

---
- name: Exemple de playbook utilitzant un plugin de callback
  hosts: localhost
  tasks:
    - name: Executa una tasca amb un callback personalitzat
      command: echo "Hello, World!"

Plugins de Connexió

Els plugins de connexió gestionen la connexió amb hosts remots. Per exemple, pots utilitzar un plugin de connexió per connectar-te a un host utilitzant SSH o per gestionar connexions a serveis en el núvol.

---
- name: Exemple de playbook utilitzant un plugin de connexió
  hosts: remote_host
  connection: local
  tasks:
    - name: Executa una tasca en el host remot
      command: echo "Running on remote host"

Plugins d'Inventari

Els plugins d'inventari generen dinàmicament inventaris a partir de fonts externes. Per exemple, pots utilitzar un plugin d'inventari per generar inventaris a partir de l'API d'un proveïdor de núvol.

---
plugin: aws_ec2
regions:
  - us-east-1
filters:
  instance-state-name: running

Plugins de Variables

Els plugins de variables carreguen variables de fonts externes. Per exemple, pots utilitzar un plugin de variables per carregar variables d'un fitxer o una base de dades.

---
- name: Exemple de playbook utilitzant un plugin de variables
  hosts: localhost
  vars_files:
    - /path/to/vars.yml
  tasks:
    - name: Utilitza les variables carregades
      debug:
        msg: "{{ my_variable }}"

Plugins de Test

Els plugins de test proporcionen proves addicionals de Jinja2. Per exemple, pots utilitzar un plugin de test per comprovar si una variable és una adreça IP vàlida.

---
- name: Exemple de playbook utilitzant un plugin de test
  hosts: localhost
  tasks:
    - name: Comprova si una variable és una adreça IP vàlida
      debug:
        msg: "La variable és una adreça IP vàlida"
      when: my_variable is ipaddr

Creació de Plugins d'Ansible Personalitzats

Crear plugins personalitzats et permet ampliar la funcionalitat d'Ansible per satisfer les teves necessitats específiques. Aquí tens un exemple bàsic de com crear un plugin de filtre personalitzat:

  1. Crea una estructura de directoris per al teu plugin:
my_plugins/
└── filter_plugins/
    └── my_custom_filter.py
  1. Implementa el plugin de filtre personalitzat a my_custom_filter.py:
def to_uppercase(value):
    return value.upper()

class FilterModule(object):
    def filters(self):
        return {
            'to_uppercase': to_uppercase
        }
  1. Utilitza el plugin de filtre personalitzat en el teu playbook:
---
- name: Exemple de playbook utilitzant un plugin de filtre personalitzat
  hosts: localhost
  tasks:
    - name: Transforma dades utilitzant el plugin de filtre personalitzat
      debug:
        msg: "{{ 'hello, world!' | to_uppercase }}"

Conclusió

Els plugins d'Ansible són una característica versàtil i poderosa que et permet ampliar i personalitzar el comportament d'Ansible. Comprenent els diferents tipus de plugins i com utilitzar-los, pots millorar els teus fluxos de treball d'automatització i adaptar Ansible a les teves necessitats específiques. Tant si utilitzes plugins integrats com si crees els teus propis, els plugins proporcionen una manera flexible d'integrar Ansible amb sistemes externs i de realitzar transformacions de dades complexes.
