Introducció

La seguretat és un aspecte crític de qualsevol infraestructura informàtica, i Ansible, com a eina d'automatització potent, s'ha d'utilitzar tenint en compte les millors pràctiques de seguretat. Aquesta secció et guiarà a través de les pràctiques de seguretat essencials per utilitzar Ansible, des de nivells bàsics fins a avançats.

Pràctiques de Seguretat Bàsiques

Utilitza Claus SSH en Lloc de Contrasenyes

  • Per què: Les claus SSH proporcionen una manera més segura i convenient d'autenticar-se que les contrasenyes.
  • Com: Genera claus SSH i distribueix la clau pública als nodes gestionats.
# Generar parell de claus SSH
ssh-keygen -t rsa -b 4096

# Copiar la clau pública al node gestionat
ssh-copy-id user@managed_node

Limita l'Accés SSH

  • Per què: Restringir l'accés SSH redueix la superfície d'atac.
  • Com: Utilitza tallafocs i grups de seguretat per limitar l'accés SSH a adreces IP de confiança.

Utilitza Ansible Vault per a Dades Sensibles

  • Per què: Emmagatzemar dades sensibles com contrasenyes i claus API en text pla és un risc de seguretat.
  • Com: Xifra les dades sensibles utilitzant Ansible Vault.
# Crear un fitxer xifrat
ansible-vault create secrets.yml

# Editar un fitxer xifrat
ansible-vault edit secrets.yml

# Utilitzar el fitxer xifrat en un playbook
ansible-playbook playbook.yml --ask-vault-pass

Pràctiques de Seguretat Intermèdies

Principi del Menor Privilegi

  • Per què: Minimitza els permisos concedits a usuaris i processos per reduir el potencial dany de les bretxes de seguretat.
  • Com: Configura el fitxer sudoers per concedir només els permisos necessaris.
# Exemple d'entrada sudoers per a privilegis limitats
user ALL=(ALL) NOPASSWD: /usr/bin/ansible-playbook

Configuració Segura d'Ansible

  • Per què: Assegurar que la configuració d'Ansible sigui segura evita canvis no autoritzats.
  • Com: Estableix permisos de fitxer adequats i utilitza control de versions amb controls d'accés.
# Establir permisos de fitxer
chmod 600 ansible.cfg
chmod 700 /etc/ansible

Utilitza Rols i Playbooks de Fonts de Confiança

  • Per què: Utilitzar rols i playbooks no confiables pot introduir vulnerabilitats.
  • Com: Verifica la font i revisa el codi abans d'utilitzar-lo.

Pràctiques de Seguretat Avançades

Implementa Ansible Tower o AWX

  • Per què: Proporciona una manera centralitzada, segura i escalable de gestionar les implementacions d'Ansible.
  • Com: Instal·la i configura Ansible Tower o AWX, i utilitza les seves característiques per al control d'accés basat en rols (RBAC), registre i auditoria.

Utilitza Mòduls Personalitzats d'Ansible amb Cura

  • Per què: Els mòduls personalitzats poden introduir riscos de seguretat si no es revisen adequadament.
  • Com: Revisa i prova els mòduls personalitzats a fons abans de la seva implementació.

Auditories de Seguretat Regulars i Verificacions de Compliment

  • Per què: Les auditories regulars ajuden a identificar i mitigar vulnerabilitats de seguretat.
  • Com: Utilitza Ansible per automatitzar les verificacions de compliment i generar informes d'auditoria.
# Exemple de playbook per a verificació de compliment
- name: Comprovar usuaris no autoritzats
  hosts: all
  tasks:
    - name: Assegurar que no existeixin usuaris no autoritzats
      shell: cat /etc/passwd | grep unauthorized_user
      register: result
      failed_when: result.stdout != ""

Conclusió

La seguretat en Ansible és un procés continu que implica utilitzar les millors pràctiques a cada nivell del teu flux de treball d'automatització. Seguint aquestes directrius, pots reduir significativament el risc de bretxes de seguretat i assegurar-te que la teva infraestructura es mantingui segura. Mantén-te sempre actualitzat amb les últimes pràctiques de seguretat i monitora i audita contínuament els teus sistemes.
