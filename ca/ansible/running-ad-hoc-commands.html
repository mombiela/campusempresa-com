Introducció

Les comandes ad-hoc d'Ansible et permeten realitzar tasques ràpides i puntuals sense la necessitat d'escriure un playbook. Són útils per a operacions senzilles, proves i resolució de problemes. Aquesta secció et guiarà a través dels conceptes bàsics de l'execució de comandes ad-hoc a Ansible, des de nivells bàsics fins a avançats.

Conceptes Clau

  • Comanda Ad-Hoc: Una comanda puntual executada directament des de la línia de comandes.
  • Mòduls: Scripts reutilitzables i autònoms que Ansible executa en el teu nom. Alguns exemples inclouen ping, shell, copy i yum.
  • Inventari: Una llista d'hostes que Ansible gestiona.

Comandes Ad-Hoc Bàsiques

Mòdul Ping

El mòdul ping s'utilitza per comprovar la connectivitat dels teus hostes.

ansible all -m ping
  • all: Destina tots els hostes de l'inventari.
  • -m ping: Utilitza el mòdul ping.

Mòdul Shell

El mòdul shell et permet executar comandes de shell als teus hostes.

ansible all -m shell -a 'uptime'
  • -a 'uptime': L'argument per passar al mòdul shell, que en aquest cas és la comanda uptime.

Mòdul Copy

El mòdul copy s'utilitza per copiar fitxers als hostes remots.

ansible all -m copy -a 'src=/path/to/local/file dest=/path/to/remote/file'
  • src: Ruta del fitxer font a la màquina local.
  • dest: Ruta del fitxer de destinació a la màquina remota.

Comandes Ad-Hoc Intermèdies

Gestió de Paquets

Pots gestionar paquets als teus hostes utilitzant mòduls com yum per a sistemes basats en Red Hat o apt per a sistemes basats en Debian.

Mòdul Yum

ansible all -m yum -a 'name=httpd state=present'
  • name: El nom del paquet.
  • state: L'estat desitjat del paquet (present per instal·lar, absent per eliminar).

Mòdul Apt

ansible all -m apt -a 'name=nginx state=latest'
  • state=latest: Assegura que el paquet és la versió més recent.

Gestió d'Usuaris

Pots gestionar usuaris als teus hostes utilitzant el mòdul user.

ansible all -m user -a 'name=johndoe state=present'
  • name: El nom d'usuari.
  • state=present: Assegura que l'usuari existeix.

Comandes Ad-Hoc Avançades

Ús de Variables

Pots passar variables a les comandes ad-hoc per a operacions més dinàmiques.

ansible all -m shell -a 'echo {{ my_var }}' -e 'my_var=HelloWorld'
  • -e 'my_var=HelloWorld': Defineix la variable my_var.

Execució de Comandes en Hostes Específics

Pots destinar hostes o grups específics en el teu inventari.

ansible webservers -m shell -a 'systemctl restart nginx'
  • webservers: Destina el grup webservers definit en el teu inventari.

Ús d'Etiquetes

Les etiquetes es poden utilitzar per executar parts específiques dels teus playbooks, però també es poden aplicar en comandes ad-hoc per a un control més granular.

ansible all -m shell -a 'systemctl restart nginx' --tags "web"
  • --tags "web": Només executa la comanda en hostes etiquetats amb web.

Resum

Les comandes ad-hoc a Ansible són eines poderoses per realitzar tasques ràpides i puntuals a diversos hostes. Són útils per a proves, resolució de problemes i operacions senzilles sense la necessitat d'escriure un playbook complet. Comprenent els usos bàsics, intermedis i avançats de les comandes ad-hoc, pots gestionar eficientment la teva infraestructura amb Ansible.

En aquesta secció, hem cobert:

  • Comandes ad-hoc bàsiques com ping, shell i copy.
  • Comandes intermèdies per a la gestió de paquets i usuaris.
  • Comandes avançades utilitzant variables, destinant hostes específics i aplicant etiquetes.

Dominant aquestes comandes, pots aprofitar tot el potencial d'Ansible per a operacions ad-hoc.
