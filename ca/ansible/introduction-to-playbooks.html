Què és un Playbook?

Els Playbooks d'Ansible són fitxers YAML que defineixen una sèrie de tasques a executar en hosts remots. Són el component central de les capacitats de configuració, desplegament i orquestració d'Ansible. Els Playbooks permeten descriure l'estat desitjat dels teus sistemes en un format llegible per a humans.

Conceptes Clau

  • Sintaxi YAML: Els Playbooks estan escrits en YAML, un estàndard de serialització de dades llegible per a humans.
  • Tasques: Accions individuals que Ansible executa, com ara instal·lar un paquet o copiar un fitxer.
  • Mòduls: Scripts reutilitzables i autònoms que Ansible executa en nom teu. Cada tasca utilitza un mòdul.
  • Gestors: Tasques especials que es desencadenen per altres tasques.
  • Variables: S'utilitzen per emmagatzemar valors que es poden reutilitzar al llarg del playbook.
  • Plantilles: Fitxers que contenen variables i són processats per Ansible per produir fitxers de configuració.

Estructura Bàsica d'un Playbook

Un playbook senzill consisteix en una o més jugades. Cada jugada apunta a un grup d'hosts i defineix tasques a executar en aquests hosts.

---
- name: Ensure Apache is installed
  hosts: webservers
  become: yes

  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present

    - name: Start Apache service
      service:
        name: apache2
        state: started

Explicació

  • name: Una descripció de la jugada.
  • hosts: El grup d'hosts als quals apunta la jugada.
  • become: Indica si s'ha d'utilitzar l'escalat de privilegis (per exemple, sudo).
  • tasks: Una llista de tasques a executar.

Tasques i Mòduls

Les tasques són els blocs de construcció d'un playbook. Cada tasca utilitza un mòdul per realitzar una acció específica.

tasks:
  - name: Install Apache
    apt:
      name: apache2
      state: present

En aquest exemple:

  • name: Una descripció de la tasca.
  • apt: El mòdul utilitzat per gestionar paquets en sistemes basats en Debian.
  • name: El paquet a gestionar.
  • state: L'estat desitjat del paquet (per exemple, present, absent).

Ús de Variables

Les variables permeten fer els teus playbooks més flexibles i reutilitzables.

---
- name: Install software
  hosts: all
  become: yes

  vars:
    package_name: apache2

  tasks:
    - name: Install package
      apt:
        name: "{{ package_name }}"
        state: present

Explicació

  • vars: Defineix variables que es poden utilitzar al llarg del playbook.
  • {{ package_name }}: La variable es referencia utilitzant doble claudàtor.

Gestors

Els gestors són tasques que només s'executen quan són notificades per altres tasques. Normalment s'utilitzen per reiniciar serveis després de canvis de configuració.

---
- name: Configure web server
  hosts: webservers
  become: yes

  tasks:
    - name: Copy configuration file
      copy:
        src: /path/to/httpd.conf
        dest: /etc/httpd/conf/httpd.conf
      notify: Restart Apache

  handlers:
    - name: Restart Apache
      service:
        name: apache2
        state: restarted

Explicació

  • notify: Especifica un gestor que es desencadenarà si la tasca canvia alguna cosa.
  • handlers: Defineix tasques que s'executen quan són notificades.

Plantilles

Les plantilles permeten crear fitxers de configuració dinàmics utilitzant variables.

---
- name: Configure web server
  hosts: webservers
  become: yes

  vars:
    server_port: 8080

  tasks:
    - name: Deploy configuration file from template
      template:
        src: /path/to/httpd.conf.j2
        dest: /etc/httpd/conf/httpd.conf

Explicació

  • template: El mòdul utilitzat per processar plantilles.
  • src: El fitxer de plantilla font.
  • dest: La ruta de destinació en l'host remot.

Conclusió

Els Playbooks d'Ansible són una manera poderosa d'automatitzar la configuració i gestió dels teus sistemes. Entenent l'estructura bàsica i conceptes clau com tasques, mòduls, variables, gestors i plantilles, pots començar a escriure playbooks efectius per gestionar la teva infraestructura. A mesura que avancis, aprendràs a crear playbooks més complexos que poden manejar una àmplia gamma de tasques i escenaris.
