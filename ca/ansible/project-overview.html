Introducció a Ansible

Ansible és una eina d'automatització de codi obert utilitzada per a la gestió de configuracions, el desplegament d'aplicacions i l'automatització de tasques. Simplifica tasques complexes i garanteix la coherència a tota la teva infraestructura. Aquest curs et portarà des d'un nivell principiant fins a un nivell avançat, proporcionant-te les habilitats necessàries per utilitzar Ansible de manera efectiva en escenaris del món real.

Conceptes clau

  • Gestió de configuracions: Automatitza la configuració i el manteniment de la teva infraestructura.
  • Playbooks: Fitxers YAML que defineixen una sèrie de tasques a executar en els teus nodes gestionats.
  • Mòduls: Scripts reutilitzables i autònoms que Ansible executa en els teus nodes.
  • Inventari: Una llista de nodes (servidors) que Ansible gestiona.
  • Rols: Una manera d'organitzar playbooks i altres fitxers per facilitar la reutilització i el compartiment.
  • Variables: Utilitzades per emmagatzemar valors que es poden reutilitzar a tot el teu playbook.
  • Plantilles: Fitxers que poden generar fitxers de configuració dinàmicament basats en variables.

Estructura del curs

Nivell principiant

  • Introducció a YAML: Aprèn els conceptes bàsics de la sintaxi YAML, que és essencial per escriure playbooks d'Ansible.
  • Configuració d'Ansible: Guia pas a pas per instal·lar Ansible a la teva màquina local.
  • Playbooks bàsics: Escriu el teu primer playbook per automatitzar tasques simples.
  • Ús de mòduls: Introducció als mòduls d'Ansible més utilitzats.

Nivell intermedi

  • Playbooks avançats: Escriu playbooks més complexos amb condicionals i bucles.
  • Rols i inclusions: Organitza els teus playbooks per a una millor mantenibilitat.
  • Variables i fets: Utilitza variables i recull informació sobre els teus nodes.
  • Plantilles i fitxers: Utilitza plantilles Jinja2 per crear fitxers de configuració dinàmics.

Nivell avançat

  • Mòduls personalitzats: Escriu els teus propis mòduls d'Ansible per ampliar la funcionalitat.
  • Gestió d'errors: Implementa la gestió d'errors i el depurament en els teus playbooks.
  • Optimització del rendiment: Optimitza els teus playbooks d'Ansible per obtenir un millor rendiment.
  • Ansible Tower: Introducció a Ansible Tower per a la gestió a nivell empresarial.

Codi d'exemple

Exemple per a principiants: Playbook simple

---
- name: Install and start Apache
  hosts: webservers
  become: yes
  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present

    - name: Start Apache
      service:
        name: apache2
        state: started

Explicació:

  • hosts: webservers: Especifica el grup de servidors en què s'executarà aquest playbook.
  • become: yes: Executa tasques amb privilegis elevats.
  • tasks: Una llista de tasques a executar.
    • apt: Mòdul per gestionar paquets en sistemes basats en Debian.
    • service: Mòdul per gestionar serveis.

Exemple intermedi: Ús de variables i plantilles

---
- name: Deploy Nginx with custom configuration
  hosts: webservers
  become: yes
  vars:
    nginx_port: 8080
  tasks:
    - name: Install Nginx
      apt:
        name: nginx
        state: present

    - name: Deploy Nginx configuration
      template:
        src: nginx.conf.j2
        dest: /etc/nginx/nginx.conf

    - name: Start Nginx
      service:
        name: nginx
        state: started

Explicació:

  • vars: Defineix una variable nginx_port que es pot utilitzar en el playbook.
  • template: Utilitza una plantilla Jinja2 per crear el fitxer de configuració de Nginx.

Exemple avançat: Mòdul personalitzat

#!/usr/bin/python

from ansible.module_utils.basic import AnsibleModule

def run_module():
    module_args = dict(
        name=dict(type='str', required=True)
    )

    result = dict(
        changed=False,
        original_message='',
        message=''
    )

    module = AnsibleModule(
        argument_spec=module_args,
        supports_check_mode=True
    )

    result['original_message'] = module.params['name']
    result['message'] = 'Hello, {}'.format(module.params['name'])

    if module.check_mode:
        module.exit_json(**result)

    result['changed'] = True
    module.exit_json(**result)

def main():
    run_module()

if __name__ == '__main__':
    main()

Explicació:

  • Aquest és un mòdul personalitzat simple escrit en Python.
  • AnsibleModule: Una classe auxiliar per gestionar els paràmetres d'entrada i els resultats de sortida.
  • module_args: Defineix els paràmetres d'entrada per al mòdul.
  • result: Un diccionari per emmagatzemar els resultats de l'execució del mòdul.
