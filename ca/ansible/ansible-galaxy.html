Introducció a Ansible Galaxy

Ansible Galaxy és un centre per trobar, compartir i reutilitzar contingut d'Ansible. Permet als usuaris descarregar rols i col·leccions que es poden utilitzar per automatitzar diverses tasques. Aquesta secció cobrirà els conceptes bàsics d'Ansible Galaxy, com utilitzar-lo i com contribuir-hi.

Conceptes Clau

  • Rols: Unitats preempaquetades de contingut d'Ansible que es poden reutilitzar en múltiples playbooks.
  • Col·leccions: Un format de distribució per al contingut d'Ansible que pot incloure rols, mòduls, connectors i més.
  • Galaxy CLI: Interfície de línia de comandes per interactuar amb Ansible Galaxy.

Utilitzant Ansible Galaxy

Instal·lant Rols

Els rols es poden instal·lar fàcilment des d'Ansible Galaxy utilitzant la comanda ansible-galaxy.

ansible-galaxy install username.role_name
  • username: El nom d'usuari de Galaxy de l'autor del rol.
  • role_name: El nom del rol que vols instal·lar.

Exemple

Per instal·lar un rol per gestionar Nginx:

ansible-galaxy install geerlingguy.nginx

Utilitzant Rols Instal·lats en Playbooks

Un cop un rol està instal·lat, es pot utilitzar en els teus playbooks.

---
- hosts: webservers
  roles:
    - geerlingguy.nginx

Gestionant Rols

Pots llistar els rols instal·lats utilitzant:

ansible-galaxy list

Per eliminar un rol:

ansible-galaxy remove username.role_name

Creant i Compartint Rols

Creant un Rol

Pots crear un nou rol utilitzant la comanda ansible-galaxy:

ansible-galaxy init my_new_role

Aquesta comanda crea una estructura de directoris per al teu nou rol.

Estructura de Directoris

El rol creat tindrà la següent estructura:

my_new_role/
├── defaults
│   └── main.yml
├── files
├── handlers
│   └── main.yml
├── meta
│   └── main.yml
├── tasks
│   └── main.yml
├── templates
├── tests
│   ├── inventory
│   └── test.yml
└── vars
    └── main.yml

Publicant un Rol

Per compartir el teu rol a Ansible Galaxy, necessites:

  1. Crear un compte a Galaxy.
  2. Enllaçar el teu compte de GitHub a Galaxy.
  3. Pujar el teu rol a un repositori de GitHub.
  4. Importar el rol a Galaxy utilitzant la interfície web de Galaxy.

Col·leccions d'Ansible

Instal·lant Col·leccions

Les col·leccions es poden instal·lar de manera similar als rols:

ansible-galaxy collection install namespace.collection_name

Exemple

Per instal·lar una col·lecció per AWS:

ansible-galaxy collection install amazon.aws

Utilitzant Col·leccions en Playbooks

Les col·leccions es poden utilitzar en playbooks especificant el nom complet de la col·lecció.

---
- hosts: localhost
  tasks:
    - name: Crear un bucket S3
      amazon.aws.s3_bucket:
        name: my-bucket
        state: present

Creant una Col·lecció

Pots crear una nova col·lecció utilitzant la comanda ansible-galaxy:

ansible-galaxy collection init my_namespace.my_collection

Aquesta comanda crea una estructura de directoris per a la teva nova col·lecció.

Estructura de Directoris

La col·lecció creada tindrà la següent estructura:

my_namespace-my_collection/
├── docs
├── plugins
│   ├── modules
│   └── ...
├── roles
├── playbooks
└── tests

Publicant una Col·lecció

Per compartir la teva col·lecció a Ansible Galaxy, necessites:

  1. Crear un compte a Galaxy.
  2. Enllaçar el teu compte de GitHub a Galaxy.
  3. Pujar la teva col·lecció a un repositori de GitHub.
  4. Importar la col·lecció a Galaxy utilitzant la interfície web de Galaxy.

Conclusió

Ansible Galaxy és una eina poderosa per trobar, compartir i reutilitzar contingut d'Ansible. Entenent com utilitzar rols i col·leccions, pots agilitzar significativament les teves tasques d'automatització. Tant si estàs instal·lant rols i col·leccions existents com si estàs creant i compartint els teus propis, Ansible Galaxy proporciona una plataforma robusta per millorar la teva experiència amb Ansible.
