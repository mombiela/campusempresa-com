Introducció als rols en Ansible

Els rols en Ansible són una manera d'agrupament de tasques, variables, fitxers, plantilles i gestors relacionats en unitats reutilitzables. Ajuden a organitzar els playbooks i a fer-los més mantenibles i escalables.

Conceptes clau

  • Estructura del directori de rols: Una disposició estàndard per organitzar fitxers dins d'un rol.
  • Reutilitzabilitat: Els rols es poden reutilitzar en diferents playbooks i projectes.
  • Modularitat: Descomposició de playbooks complexos en rols més petits i manejables.

Creació i ús de rols

Estructura del directori de rols

Un rol d'Ansible té una estructura de directori específica. Aquí teniu un exemple:

my_role/
├── defaults/
│   └── main.yml
├── files/
├── handlers/
│   └── main.yml
├── meta/
│   └── main.yml
├── tasks/
│   └── main.yml
├── templates/
├── tests/
│   ├── inventory
│   └── test.yml
└── vars/
    └── main.yml
  • defaults/main.yml: Variables per defecte per al rol.
  • files/: Fitxers estàtics per ser utilitzats en tasques.
  • handlers/main.yml: Gestors activats per tasques.
  • meta/main.yml: Metadades del rol, incloent-hi dependències.
  • tasks/main.yml: Llista principal de tasques a executar pel rol.
  • templates/: Plantilles Jinja2 per ser utilitzades en tasques.
  • tests/: Playbooks de prova i inventari per al rol.
  • vars/main.yml: Variables específiques del rol.

Creació d'un rol senzill

Creem un rol senzill per instal·lar i configurar Nginx.

  1. Crea l'estructura del directori del rol:

    ansible-galaxy init nginx_role

  2. Defineix les tasques a tasks/main.yml:

    ---
- name: Install Nginx
  apt:
    name: nginx
    state: present

- name: Start Nginx service
  service:
    name: nginx
    state: started
    enabled: true

  3. Defineix els gestors a handlers/main.yml:

    ---
- name: Restart Nginx
  service:
    name: nginx
    state: restarted

  4. Utilitza el rol en un playbook:

    ---
- hosts: webservers
  roles:
    - nginx_role

Ús de variables en rols

Les variables es poden definir a defaults/main.yml o vars/main.yml. Aquí teniu un exemple d'ús de variables en un rol:

  1. Defineix les variables a defaults/main.yml:

    ---
nginx_port: 80

  2. Utilitza les variables a tasks/main.yml:

    ---
- name: Install Nginx
  apt:
    name: nginx
    state: present

- name: Configure Nginx
  template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
  notify: Restart Nginx

- name: Start Nginx service
  service:
    name: nginx
    state: started
    enabled: true

  3. Crea una plantilla a templates/nginx.conf.j2:

    server {
    listen {{ nginx_port }};
    server_name localhost;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}

Característiques avançades dels rols

Dependències de rols

Els rols poden dependre d'altres rols. Les dependències es defineixen a meta/main.yml.

---
dependencies:
  - { role: common, some_var: 42 }
  - { role: apache, apache_port: 8080 }

Prova de rols

Provar els rols assegura que funcionen com s'espera. Utilitza el directori tests/ per crear playbooks de prova i fitxers d'inventari.

  1. Crea un playbook de prova a tests/test.yml:

    ---
- hosts: localhost
  roles:
    - nginx_role

  2. Executa el playbook de prova:

    ansible-playbook tests/test.yml

Conclusió

Els rols en Ansible proporcionen una manera poderosa d'organitzar i reutilitzar codi, fent els playbooks més modulars i mantenibles. Entenent l'estructura i l'ús dels rols, pots crear scripts d'automatització escalables i eficients. Recorda aprofitar les variables, gestors i dependències per maximitzar la flexibilitat i funcionalitat dels teus rols.
