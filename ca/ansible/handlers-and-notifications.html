En aquesta secció, explorarem els Gestors i les Notificacions a Ansible. Aquests són conceptes essencials per gestionar els canvis d'estat i assegurar que les tasques s'executin només quan sigui necessari.

Què són els Gestors?

Els gestors són tasques especials a Ansible que es desencadenen per notificacions. S'utilitzen típicament per reiniciar serveis, recarregar configuracions o realitzar altres accions que només haurien de tenir lloc quan s'ha fet un canvi.

Conceptes Clau

  • Gestors: Tasques que s'executen només quan són notificades.
  • Notificacions: Desencadenants que indiquen als gestors que s'executin.

Exemple d'un Gestor

Vegem un exemple bàsic d'un gestor:

# playbook.yml
- name: Instal·lar i configurar Apache
  hosts: webservers
  tasks:
    - name: Instal·lar Apache
      apt:
        name: apache2
        state: present
      notify:
        - Reiniciar Apache

  handlers:
    - name: Reiniciar Apache
      service:
        name: apache2
        state: restarted

En aquest exemple:

  • La tasca apt instal·la Apache i notifica el gestor Reiniciar Apache.
  • El gestor Reiniciar Apache reiniciarà el servei Apache, però només si la tasca apt fa un canvi (és a dir, instal·la Apache).

Notificacions

Les notificacions s'utilitzen per desencadenar gestors. Es defineixen dins de les tasques i especifiquen quins gestors s'han d'executar si la tasca resulta en un canvi.

Exemple de Notificacions

Continuant amb l'exemple anterior, la paraula clau notify s'utilitza per especificar el gestor:

- name: Instal·lar Apache
  apt:
    name: apache2
    state: present
  notify:
    - Reiniciar Apache

Múltiples Notificacions

Una tasca pot notificar múltiples gestors. Aquí teniu un exemple:

- name: Instal·lar Apache
  apt:
    name: apache2
    state: present
  notify:
    - Reiniciar Apache
    - Recarregar Tallafocs

handlers:
  - name: Reiniciar Apache
    service:
      name: apache2
      state: restarted

  - name: Recarregar Tallafocs
    command: ufw reload

En aquest exemple:

  • La tasca apt notifica tant els gestors Reiniciar Apache com Recarregar Tallafocs.
  • Ambdós gestors s'executaran si la tasca apt fa un canvi.

Usos Pràctics

Reiniciar Serveis

Un dels usos més comuns dels gestors és reiniciar serveis quan es canvien els fitxers de configuració.

- name: Copiar fitxer de configuració d'Apache
  copy:
    src: /path/to/httpd.conf
    dest: /etc/httpd/conf/httpd.conf
  notify:
    - Reiniciar Apache

handlers:
  - name: Reiniciar Apache
    service:
      name: httpd
      state: restarted

Recarregar Configuracions

Els gestors també es poden utilitzar per recarregar configuracions sense reiniciar serveis.

- name: Actualitzar configuració de Nginx
  template:
    src: /path/to/nginx.conf.j2
    dest: /etc/nginx/nginx.conf
  notify:
    - Recarregar Nginx

handlers:
  - name: Recarregar Nginx
    service:
      name: nginx
      state: reloaded

Millors Pràctiques

  • Idempotència: Assegureu-vos que les tasques i els gestors siguin idempotents, és a dir, que es puguin executar múltiples vegades sense canviar el sistema després de la primera execució.
  • Noms Descriptius: Utilitzeu noms descriptius per als gestors per fer que els playbooks siguin més fàcils d'entendre.
  • Canvis Mínims: Només notifiqueu els gestors quan sigui necessari per evitar reinicis o recàrregues de serveis innecessaris.

Conclusió

Els gestors i les notificacions són característiques poderoses a Ansible que ajuden a gestionar els canvis d'estat de manera eficient. Utilitzant gestors, podeu assegurar-vos que certes tasques només s'executin quan sigui necessari, cosa que pot ajudar a mantenir l'estabilitat i el rendiment dels vostres sistemes. Recordeu seguir les millors pràctiques per crear playbooks clars i mantenibles.
