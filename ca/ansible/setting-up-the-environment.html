Configurar l'entorn és el primer pas per començar amb Ansible. Aquesta secció et guiarà a través dels passos necessaris per instal·lar i configurar Ansible al teu sistema, assegurant que tinguis una base sòlida sobre la qual construir a mesura que avancis en el curs.

Prerequisits

Abans d'instal·lar Ansible, assegura't de tenir els següents prerequisits:

  • Un sistema operatiu tipus Unix (Linux, macOS o WSL a Windows).
  • Python 3.6 o posterior instal·lat al teu sistema.
  • Coneixements bàsics d'operacions de línia de comandes.

Instal·lant Ansible

Utilitzant el gestor de paquets

A Ubuntu/Debian

Per instal·lar Ansible a Ubuntu o Debian, utilitza les següents comandes:

sudo apt update
sudo apt install ansible -y

A CentOS/RHEL

Per a CentOS o RHEL, utilitza les següents comandes:

sudo yum install epel-release -y
sudo yum install ansible -y

A macOS

Per als usuaris de macOS, Homebrew és la manera més fàcil d'instal·lar Ansible:

brew install ansible

Utilitzant Pip

Alternativament, pots instal·lar Ansible utilitzant el gestor de paquets de Python, pip:

pip install ansible

Verificant la instal·lació

Després d'instal·lar Ansible, verifica la instal·lació comprovant la versió:

ansible --version

Hauries de veure una sortida similar a:

ansible [core 2.10.7]
  config file = /etc/ansible/ansible.cfg
  configured module search path = ['/home/user/.ansible/plugins/modules', '/usr/share/ansible/plugins/modules']
  ansible python module location = /usr/local/lib/python3.8/dist-packages/ansible
  executable location = /usr/local/bin/ansible
  python version = 3.8.5 (default, Jul 28 2020, 12:59:40) [GCC 9.3.0]

Configurant Ansible

Fitxer de configuració d'Ansible

El comportament d'Ansible es pot personalitzar utilitzant el fitxer de configuració ansible.cfg. Per defecte, aquest fitxer es troba a /etc/ansible/ansible.cfg. També pots crear un fitxer de configuració personalitzat al teu directori de projecte.

Aquí tens un exemple d'un fitxer bàsic ansible.cfg:

[defaults]
inventory = ./hosts
remote_user = your_username
host_key_checking = False

Fitxer d'inventari

El fitxer d'inventari llista els hosts i grups de hosts que Ansible gestiona. Aquí tens un exemple d'un fitxer d'inventari simple anomenat hosts:

[webservers]
webserver1 ansible_host=192.168.1.10 ansible_user=ubuntu
webserver2 ansible_host=192.168.1.11 ansible_user=ubuntu

[databases]
dbserver1 ansible_host=192.168.1.20 ansible_user=ubuntu

Provant la configuració

Per assegurar-te que tot està configurat correctament, executa una comanda simple d'Ansible per fer un ping a tots els hosts del teu inventari:

ansible all -m ping

Hauries de veure una sortida similar a:

webserver1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
webserver2 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
dbserver1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}

Conclusió

En aquesta secció, hem cobert els passos essencials per configurar l'entorn d'Ansible, incloent la instal·lació, configuració i verificació. Amb el teu entorn preparat, ara estàs llest per aprofundir en Ansible i començar a automatitzar les teves tasques.
