Implementant el Projecte
Proves i Validació ►

En aquesta secció, cobrirem com implementar un projecte utilitzant Ansible de principi a fi. Això inclourà configurar el teu entorn, escriure playbooks i executar-los per aconseguir la configuració desitjada. Al final d'aquesta secció, hauràs de ser capaç d'implementar un projecte amb confiança utilitzant Ansible.

Configurant el Teu Entorn

Abans de començar a escriure playbooks d'Ansible, necessites configurar el teu entorn. Això inclou instal·lar Ansible i configurar el teu inventari.

Instal·lant Ansible

Per instal·lar Ansible, pots utilitzar el gestor de paquets del teu sistema operatiu. Aquí tens les comandes per a alguns sistemes comuns:

  • Ubuntu/Debian:

    sudo apt update
sudo apt install ansible

  • CentOS/RHEL:

    sudo yum install epel-release
sudo yum install ansible

  • macOS:

    brew install ansible

Configurant el Teu Inventari

Ansible utilitza un fitxer d'inventari per gestionar els hosts que configurarà. El fitxer d'inventari per defecte es troba a /etc/ansible/hosts, però pots especificar un fitxer diferent utilitzant l'opció -i.

Exemple d'un fitxer d'inventari:

[webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com

Escriure el Teu Primer Playbook

Un playbook és un fitxer YAML que descriu les tasques a executar en els teus hosts. Anem a escriure un playbook senzill per instal·lar i iniciar el servidor web Apache en els nostres servidors web.

Exemple de Playbook

---
- name: Instal·lar i iniciar Apache
  hosts: webservers
  become: yes
  tasks:
    - name: Instal·lar Apache
      apt:
        name: apache2
        state: present
      when: ansible_os_family == "Debian"

    - name: Instal·lar Apache
      yum:
        name: httpd
        state: present
      when: ansible_os_family == "RedHat"

    - name: Iniciar Apache
      service:
        name: "{{ ansible_os_family == 'Debian' | ternary('apache2', 'httpd') }}"
        state: started

Explicació

  • name: Una descripció del playbook o tasca.
  • hosts: El grup de hosts del fitxer d'inventari en què s'executarà aquest playbook.
  • become: Si s'ha d'utilitzar sudo per executar les tasques.
  • tasks: Una llista de tasques a executar.

Cada tasca té:

  • name: Una descripció de la tasca.
  • module: El mòdul d'Ansible a utilitzar (per exemple, apt, yum, service).
  • arguments: Els arguments per al mòdul.

Executant el Teu Playbook

Per executar el teu playbook, utilitza la comanda ansible-playbook seguida del camí al teu fitxer de playbook.

ansible-playbook -i inventory.ini install_apache.yml

Opcions Comunes

  • -i: Especificar el fitxer d'inventari.
  • -u: Especificar l'usuari remot.
  • --ask-become-pass: Sol·licitar la contrasenya de sudo.

Característiques Avançades del Playbook

A mesura que et sentis més còmode amb Ansible, pots començar a utilitzar característiques més avançades en els teus playbooks.

Variables

Les variables permeten fer els teus playbooks més flexibles i reutilitzables.

Exemple:

---
- name: Instal·lar i iniciar Apache
  hosts: webservers
  become: yes
  vars:
    apache_package: "{{ ansible_os_family == 'Debian' | ternary('apache2', 'httpd') }}"
  tasks:
    - name: Instal·lar Apache
      package:
        name: "{{ apache_package }}"
        state: present

    - name: Iniciar Apache
      service:
        name: "{{ apache_package }}"
        state: started

Handlers

Els handlers són tasques que només s'executen quan són notificades per altres tasques. Són útils per a accions com reiniciar un servei després d'un canvi de configuració.

Exemple:

---
- name: Instal·lar i configurar Apache
  hosts: webservers
  become: yes
  tasks:
    - name: Instal·lar Apache
      package:
        name: apache2
        state: present
      notify: Reiniciar Apache

    - name: Configurar Apache
      template:
        src: apache.conf.j2
        dest: /etc/apache2/sites-available/000-default.conf
      notify: Reiniciar Apache

  handlers:
    - name: Reiniciar Apache
      service:
        name: apache2
        state: restarted

Conclusió

En aquesta secció, hem cobert els passos essencials per implementar un projecte utilitzant Ansible. Hem començat amb la configuració de l'entorn, escrivint un playbook bàsic i executant-lo. També hem tocat característiques avançades com les variables i els handlers per fer els teus playbooks més potents i flexibles. Amb aquestes habilitats, ara estàs preparat per començar a implementar els teus propis projectes utilitzant Ansible.
