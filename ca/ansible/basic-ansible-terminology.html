En aquesta secció, tractarem la terminologia fonamental utilitzada en Ansible. Entendre aquests termes és crucial per utilitzar Ansible de manera efectiva per a l'automatització i la gestió de configuracions.

Conceptes clau

  • Node de control: La màquina on s'instal·la Ansible i des de la qual s'executen les ordres. Aquest és el punt central des del qual es gestionen totes les tasques d'automatització.

  • Nodes gestionats: Les màquines que són gestionades per Ansible. També es coneixen com a hosts. Ansible es comunica amb aquests nodes per executar tasques.

  • Inventari: Un fitxer que llista tots els nodes gestionats. Pot ser un fitxer de text simple amb una llista d'adreces IP o noms de host, o pot ser un fitxer més complex amb grups i variables.

    [webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com

  • Playbook: Un fitxer YAML que conté una sèrie de tasques a executar en els nodes gestionats. Els playbooks són el nucli del llenguatge de configuració, desplegament i orquestració d'Ansible.

    - name: Instal·lar i iniciar Apache
  hosts: webservers
  tasks:
    - name: Instal·lar Apache
      apt:
        name: apache2
        state: present

    - name: Iniciar Apache
      service:
        name: apache2
        state: started

  • Tasca: Una unitat de treball única en un playbook. Cada tasca crida un mòdul d'Ansible per realitzar una acció específica.

  • Mòdul: Les unitats de treball que Ansible executa. Els mòduls poden gestionar recursos del sistema, instal·lar programari i realitzar diverses altres tasques. Exemples inclouen apt, yum, service i copy.

  • Rol: Una manera d'agrupament de múltiples tasques, variables, fitxers, plantilles i manejadors en una sola unitat. Els rols ajuden a organitzar els playbooks i reutilitzar codi.

  • Manejador: Un tipus especial de tasca que només s'executa quan és notificat per una altra tasca. Els manejadors s'utilitzen típicament per reiniciar serveis.

    - name: Instal·lar Apache
  apt:
    name: apache2
    state: present
  notify:
    - Restart Apache

handlers:
  - name: Restart Apache
    service:
      name: apache2
      state: restarted

  • Variable: Utilitzada per emmagatzemar valors que es poden reutilitzar al llarg dels playbooks i rols. Les variables es poden definir en molts llocs, incloent fitxers d'inventari, playbooks i rols.

    - name: Instal·lar Apache
  hosts: webservers
  vars:
    apache_package: apache2
  tasks:
    - name: Instal·lar Apache
      apt:
        name: "{{ apache_package }}"
        state: present

  • Fet: Informació sobre els nodes gestionats recollida per Ansible. Els fets es recullen automàticament i es poden utilitzar en els playbooks.

  • Plantilla: Un fitxer que conté variables i és processat pel motor de plantilles Jinja2 per produir un fitxer de configuració final.

    # template.j2
ServerName {{ ansible_hostname }}

    - name: Configurar Apache
  hosts: webservers
  tasks:
    - name: Desplegar configuració d'Apache
      template:
        src: template.j2
        dest: /etc/apache2/sites-available/000-default.conf

Resum

Entendre aquests termes bàsics d'Ansible és essencial per a qualsevol persona que comenci amb Ansible. Aquests conceptes formen la base sobre la qual es construeixen temes i pràctiques més avançades. Familiaritzant-te amb aquests termes, estaràs millor preparat per escriure i entendre playbooks d'Ansible, gestionar configuracions i automatitzar tasques de manera efectiva.
