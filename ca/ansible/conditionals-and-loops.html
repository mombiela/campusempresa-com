Introducció

En aquesta secció, explorarem com utilitzar condicionals i bucles a Ansible per crear playbooks més dinàmics i flexibles. Entendre aquests conceptes és crucial per escriure scripts d'automatització eficients.

Condicionals a Ansible

Els condicionals et permeten executar tasques basades en certes condicions. Això pot ser útil per prendre decisions dins dels teus playbooks.

Conceptes Clau

  • When Statement: S'utilitza per executar una tasca només si es compleix una certa condició.
  • Facts: Variables que Ansible recull sobre el sistema, les quals es poden utilitzar en condicionals.

When Statement

La declaració when s'utilitza per controlar si una tasca o un conjunt de tasques s'haurien d'executar basant-se en una condició.

Exemple

- name: Instal·lar Apache en sistemes basats en Debian
  apt:
    name: apache2
    state: present
  when: ansible_os_family == "Debian"

En aquest exemple, la tasca per instal·lar Apache només s'executarà si la família del sistema operatiu és Debian.

Utilitzant Facts en Condicionals

Els facts d'Ansible són variables que Ansible descobreix automàticament quan es connecta a un host. Aquests facts es poden utilitzar en condicionals.

Exemple

- name: Reiniciar el servidor si està executant Ubuntu
  command: /sbin/reboot
  when: ansible_distribution == "Ubuntu"

Aquí, el servidor només es reiniciarà si el sistema operatiu és Ubuntu.

Bucles a Ansible

Els bucles et permeten repetir una tasca múltiples vegades amb diferents elements. Això és útil per a tasques com instal·lar múltiples paquets o crear múltiples usuaris.

Conceptes Clau

  • with_items: Una directiva de bucle comuna utilitzada per iterar sobre una llista d'elements.
  • loop: Una directiva més flexible introduïda a Ansible 2.5, que pot substituir with_items.

with_items

La directiva with_items s'utilitza per iterar sobre una llista d'elements.

Exemple

- name: Instal·lar múltiples paquets
  apt:
    name: "{{ item }}"
    state: present
  with_items:
    - git
    - curl
    - vim

Aquesta tasca instal·larà git, curl i vim al sistema objectiu.

loop

La directiva loop proporciona més flexibilitat i es pot utilitzar en lloc de with_items.

Exemple

- name: Afegir múltiples usuaris
  user:
    name: "{{ item.name }}"
    state: present
    groups: "{{ item.groups }}"
  loop:
    - { name: 'alice', groups: 'admin' }
    - { name: 'bob', groups: 'users' }

Aquesta tasca crearà dos usuaris, alice i bob, amb els seus respectius grups.

Combinant Condicionals i Bucles

Pots combinar condicionals i bucles per crear una lògica més complexa en els teus playbooks.

Exemple

- name: Instal·lar paquets només si no estan ja instal·lats
  apt:
    name: "{{ item }}"
    state: present
  loop:
    - git
    - curl
    - vim
  when: ansible_facts.packages[item] is not defined

Aquesta tasca instal·larà git, curl i vim només si no estan ja instal·lats al sistema.

Conclusió

Entendre i utilitzar condicionals i bucles a Ansible et permet crear playbooks més dinàmics i eficients. Aprofitant aquestes característiques, pots automatitzar tasques complexes i gestionar diverses situacions amb facilitat. Practica utilitzant aquests conceptes en els teus playbooks per convertir-te en un expert en l'automatització amb Ansible.
