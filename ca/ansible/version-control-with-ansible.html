Introducció

El control de versions és un aspecte crític de la gestió de playbooks i rols d'Ansible. Permet als equips col·laborar de manera efectiva, fer un seguiment dels canvis i mantenir un historial de les seves configuracions. En aquesta secció, explorarem com utilitzar sistemes de control de versions, particularment Git, amb Ansible.

Conceptes Clau

  • Sistemes de Control de Versions (VCS): Eines que ajuden a gestionar els canvis en el codi font al llarg del temps.
  • Git: Un sistema de control de versions distribuït àmpliament utilitzat en el desenvolupament de programari.
  • Repositoris: Llocs d'emmagatzematge per al teu codi, típicament allotjats en plataformes com GitHub, GitLab o Bitbucket.
  • Branques: Línies de desenvolupament separades dins d'un repositori.
  • Commits: Instantànies del teu projecte en punts específics del temps.
  • Pull Requests/Merge Requests: Propostes per fusionar canvis d'una branca a una altra.

Configuració de Git per a Ansible

Instal·lació de Git

Abans de poder utilitzar Git amb Ansible, necessites instal·lar-lo al teu sistema.

Per a Windows:

choco install git

Per a macOS:

brew install git

Per a Linux:

sudo apt-get install git

Inicialització d'un Repositori Git

Navega al directori del teu projecte Ansible i inicialitza un repositori Git.

cd /path/to/your/ansible/project
git init

Afegir Fitxers al Repositori

Afegeix els teus playbooks i rols d'Ansible al repositori.

git add .
git commit -m "Initial commit of Ansible playbooks and roles"

Treballant amb Branques

Les branques et permeten treballar en diferents característiques o correccions simultàniament sense afectar la base de codi principal.

Creació d'una Nova Branca

Crea una nova branca per a una característica o correcció específica.

git checkout -b feature/new-playbook

Canvi de Branques

Canvia entre branques per treballar en diferents parts del teu projecte.

git checkout main

Fusió de Branques

Fusiona canvis d'una branca a una altra.

git checkout main
git merge feature/new-playbook

Col·laborant amb Altres

Clonació d'un Repositori

Clona un repositori a la teva màquina local per començar a treballar-hi.

git clone https://github.com/yourusername/your-ansible-repo.git

Pull Requests/Merge Requests

Utilitza pull requests (GitHub) o merge requests (GitLab) per proposar canvis.

  1. Puja la teva branca al repositori remot. 
    git push origin feature/new-playbook
  2. Crea un pull request a la plataforma (GitHub, GitLab, etc.).
  3. Revisa i fusiona el pull request.

Millors Pràctiques

  • Fes Commits Sovint: Fes commits petits i freqüents amb missatges clars.
  • Utilitza Branques: Aïlla característiques i correccions en branques separades.
  • Revisions de Codi: Utilitza pull requests per revisar el codi abans de fusionar-lo.
  • Documentació: Documenta els teus playbooks i rols dins del repositori.

Exemple de Flux de Treball

  1. Inicialitza el Repositori:

    git init
git add .
git commit -m "Initial commit"

  2. Crea una Branca de Característica:

    git checkout -b feature/add-user-playbook

  3. Desenvolupa la Característica: Crea un nou playbook add_user.yml:

    ---
- name: Add a new user
  hosts: all
  become: yes
  tasks:
    - name: Add user
      user:
        name: john
        state: present

  4. Fes Commit dels Canvis:

    git add add_user.yml
git commit -m "Add playbook to create a new user"

  5. Puja i Crea un Pull Request:

    git push origin feature/add-user-playbook

    Crea un pull request a GitHub.

  6. Fusiona el Pull Request: Un cop revisat, fusiona el pull request a la branca principal.

Conclusió

Utilitzar el control de versions amb Ansible assegura que el teu codi d'infraestructura estigui ben gestionat, sigui col·laboratiu i mantenible. Seguint les millors pràctiques i aprofitant eines com Git, pots agilitzar el teu flux de treball i millorar la qualitat dels teus projectes Ansible.
