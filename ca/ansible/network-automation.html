Automatització de Xarxes amb Ansible
Proveïment al Núvol ►

Introducció a l'Automatització de Xarxes

  • Definició: L'automatització de xarxes implica utilitzar programari per automatitzar la provisió i gestió de xarxes i seguretat per millorar l'eficiència i reduir els errors humans.
  • Importància:
    • Redueix els errors de configuració manuals.
    • Accelera els processos de desplegament.
    • Millora la consistència i el compliment.

Començar amb Ansible

  • Què és Ansible?: Ansible és una eina d'automatització de codi obert utilitzada per a tasques de TI com la gestió de configuracions, el desplegament d'aplicacions i l'automatització de tasques.
  • Característiques Clau:
    • Arquitectura sense agents.
    • Sintaxi YAML simple per a playbooks.
    • Biblioteca de mòduls extensa.

Instal·lar Ansible

  • Prerequisits: Python instal·lat a la màquina de control.
  • Comanda d'Instal·lació: 
    sudo apt update
sudo apt install ansible

Conceptes Bàsics d'Ansible

  • Inventari: Una llista de nodes o hosts que Ansible gestiona.
  • Playbook: Un fitxer YAML que conté una sèrie de tasques per executar als nodes gestionats.
  • Mòdul: Una unitat de codi que Ansible executa, com ping, command o ios_config.

Exemple de Fitxer d'Inventari

[routers]
router1 ansible_host=192.168.1.1
router2 ansible_host=192.168.1.2

[switches]
switch1 ansible_host=192.168.2.1
switch2 ansible_host=192.168.2.2

Exemple de Playbook

---
- name: Configurar routers i switches
  hosts: all
  tasks:
    - name: Assegurar que els dispositius són accessibles
      ping:

Mòduls de Xarxa en Ansible

  • Mòduls de Xarxa Comuns:
    • ios_config: Gestiona la configuració de Cisco IOS.
    • nxos_config: Gestiona la configuració de Cisco NX-OS.
    • eos_config: Gestiona la configuració d'Arista EOS.
    • junos_config: Gestiona la configuració de Juniper JUNOS.

Exemple: Configurar un Router Cisco

---
- name: Configurar Router Cisco
  hosts: routers
  tasks:
    - name: Configurar hostname
      ios_config:
        lines:
          - hostname Router1

Tècniques Avançades d'Automatització de Xarxes

  • Plantilles: Utilitzar plantilles Jinja2 per crear configuracions dinàmiques.
  • Rols: Organitzar playbooks en rols reutilitzables.
  • Ansible Vault: Emmagatzemar de manera segura dades sensibles com contrasenyes.

Exemple: Utilitzar Plantilles Jinja2

  • Fitxer de Plantilla (router_config.j2): 
    hostname {{ hostname }}
interface GigabitEthernet0/1
  ip address {{ ip_address }} 255.255.255.0
  • Playbook Utilitzant Plantilla: 
    ---
- name: Configurar Router Cisco amb Plantilla
  hosts: routers
  vars:
    hostname: Router1
    ip_address: 192.168.1.1
  tasks:
    - name: Aplicar configuració des de la plantilla
      ios_config:
        src: router_config.j2

Resolució de Problemes i Millors Pràctiques

  • Problemes Comuns:
    • Problemes de connectivitat.
    • Errors d'autenticació.
    • Errors de sintaxi en playbooks.
  • Millors Pràctiques:
    • Utilitzar control de versions per a playbooks.
    • Provar playbooks en un entorn de proves.
    • Utilitzar noms descriptius i comentaris.

Conclusió

L'automatització de xarxes amb Ansible simplifica la gestió i configuració de dispositius de xarxa, reduint errors i augmentant l'eficiència. Dominant les característiques bàsiques i avançades d'Ansible, pots automatitzar tasques de xarxa complexes i assegurar operacions de xarxa consistents i fiables.
