Introducció

Ansible és una eina d'automatització de codi obert utilitzada per a la gestió de configuracions, el desplegament d'aplicacions i l'automatització de tasques. Entendre la seva arquitectura i components és crucial per utilitzar Ansible de manera efectiva en escenaris reals. Aquesta secció cobrirà l'arquitectura bàsica i els components clau d'Ansible.

Arquitectura d'Ansible

Conceptes Clau

  • Arquitectura sense Agent: Ansible no requereix cap programari agent instal·lat als nodes gestionats. Utilitza SSH per a la comunicació.
  • Model Basat en Push: Ansible envia configuracions des d'una màquina central (node de control) als nodes gestionats.
  • Idempotència: Ansible assegura que aplicar la mateixa configuració múltiples vegades no canviarà el sistema després de la primera aplicació.

Node de Control i Nodes Gestionats

  • Node de Control: La màquina on Ansible està instal·lat i des de la qual s'executen les comandes.
  • Nodes Gestionats: Les màquines objectiu que són gestionades per Ansible.

Inventari

  • Fitxer d'Inventari: Un fitxer que llista els nodes gestionats. Pot ser un fitxer de text simple o un script d'inventari dinàmic. 
    [webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com

Components d'Ansible

Playbooks

  • Playbooks: Fitxers YAML que defineixen una sèrie de tasques a executar als nodes gestionats. 
    - name: Install and start Apache
  hosts: webservers
  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present
    - name: Start Apache
      service:
        name: apache2
        state: started

Mòduls

  • Mòduls: Scripts reutilitzables i autònoms que Ansible executa en nom teu. Poden gestionar recursos del sistema, instal·lar programari i més.
    • Exemple: apt, yum, service, copy, template.

Rols

  • Rols: Una manera d'organitzar playbooks i altres fitxers per facilitar la reutilització i el compartir.
    • Estructura de directoris: 
      roles/
  common/
    tasks/
      main.yml
    handlers/
      main.yml
    templates/
    files/
    vars/
      main.yml
    defaults/
      main.yml
    meta/
      main.yml

Plugins

  • Plugins: Amplien la funcionalitat bàsica d'Ansible. Tipus inclouen:
    • Plugins d'Acció: Modifiquen el comportament de les tasques d'Ansible.
    • Plugins de Consulta: Recuperen dades de fonts externes.
    • Plugins de Filtre: Transformen dades dins de plantilles.
    • Plugins de Callback: Personalitzen la sortida o el registre.

Variables i Fets

  • Variables: Utilitzades per emmagatzemar valors que es poden reutilitzar al llarg dels playbooks. 
    vars:
  http_port: 80
  • Fets: Informació recollida automàticament sobre els nodes gestionats.
    • Exemple: ansible_facts, ansible_os_family.

Handlers

  • Handlers: Tasques especials que es desencadenen per altres tasques. Normalment s'utilitzen per reiniciar serveis. 
    handlers:
  - name: restart apache
    service:
      name: apache2
      state: restarted

Conclusió

Entendre l'arquitectura i els components d'Ansible és fonamental per aprofitar tot el seu potencial. El model sense agent i basat en push simplifica la gestió de configuracions, mentre que components com playbooks, mòduls, rols i plugins proporcionen eines poderoses per a l'automatització. Dominant aquests elements, pots gestionar i automatitzar eficientment la teva infraestructura IT.
