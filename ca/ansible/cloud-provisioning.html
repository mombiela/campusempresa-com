Introducció al Proveïment de Núvol

El proveïment de núvol és el procés de configurar i gestionar recursos de núvol. Ansible, una eina d'automatització poderosa, simplifica aquest procés permetent-te definir la teva infraestructura com a codi. Aquesta secció et guiarà des dels conceptes bàsics del proveïment de núvol amb Ansible fins a conceptes més avançats.

Conceptes Clau

  • Infraestructura com a Codi (IaC): Gestionar i proveir infraestructura informàtica a través d'arxius de definició llegibles per màquina.
  • Playbooks: Arxius YAML que defineixen les tasques a executar en màquines remotes.
  • Mòduls: Scripts reutilitzables i autònoms que Ansible executa en nom teu.
  • Inventari: Una llista d'amfitrions o nodes que Ansible gestiona.
  • Proveïdors de Núvol: Serveis com AWS, Azure i Google Cloud que ofereixen recursos de computació en núvol.

Configuració d'Ansible per al Proveïment de Núvol

Abans de començar a proveir recursos de núvol, necessites configurar Ansible i configurar-lo per interactuar amb el teu proveïdor de núvol.

Instal·lació d'Ansible

Per instal·lar Ansible, executa la següent comanda:

sudo apt update
sudo apt install ansible

Configuració de Credencials del Proveïdor de Núvol

Necessites configurar les credencials del teu proveïdor de núvol perquè Ansible pugui interactuar amb els teus recursos de núvol. Per exemple, per configurar les credencials d'AWS, pots utilitzar l'AWS CLI:

aws configure

Aquesta comanda et demanarà que introdueixis el teu AWS Access Key ID, Secret Access Key, regió i format de sortida.

Escriure el teu Primer Playbook per al Proveïment de Núvol

Un playbook és un arxiu YAML que descriu les tasques a executar. Aquí tens un exemple senzill d'un playbook que proveeix una instància EC2 a AWS.

Exemple de Playbook

---
- name: Provisionar una instància EC2
  hosts: localhost
  gather_facts: no
  tasks:
    - name: Llançar instància EC2
      ec2:
        key_name: my_key
        instance_type: t2.micro
        image: ami-0c55b159cbfafe1f0
        wait: yes
        region: us-west-2
        count: 1
      register: ec2
    - name: Afegir nova instància al grup d'amfitrions
      add_host:
        hostname: "{{ item.public_ip }}"
        groupname: launched
      with_items: "{{ ec2.instances }}"
    - name: Esperar que SSH estigui disponible
      wait_for:
        host: "{{ item.public_ip }}"
        port: 22
        delay: 60
        timeout: 320
        state: started
      with_items: "{{ ec2.instances }}"

Explicació

  • hosts: Especifica els amfitrions objectiu. Aquí, localhost s'utilitza perquè el proveïment es fa des de la màquina local.
  • gather_facts: Configurat a no per saltar la recopilació de fets sobre el localhost.
  • tasks: Una llista de tasques a executar.
    • ec2: Llança una instància EC2 amb els paràmetres especificats.
    • register: Emmagatzema la sortida del mòdul EC2 en una variable anomenada ec2.
    • add_host: Afegeix la nova instància a un grup d'amfitrions anomenat launched.
    • wait_for: Espera que SSH estigui disponible a la nova instància.

Tècniques Avançades de Proveïment de Núvol

A mesura que et familiaritzis més amb Ansible, pots explorar tècniques més avançades per al proveïment de núvol.

Inventari Dinàmic

L'inventari dinàmic permet a Ansible consultar els proveïdors de núvol per obtenir l'estat actual dels recursos. Això és útil per gestionar entorns on la infraestructura canvia freqüentment.

Exemple: Inventari Dinàmic d'AWS

Per utilitzar l'inventari dinàmic amb AWS, necessites instal·lar les biblioteques de Python boto3 i botocore:

pip install boto3 botocore

Després, configura el plugin aws_ec2 al teu ansible.cfg:

[defaults]
inventory = ./aws_ec2.yml

Crea un arxiu aws_ec2.yml:

plugin: aws_ec2
regions:
  - us-west-2
filters:
  instance-state-name: running
keyed_groups:
  - key: tags.Name
    prefix: tag

Utilitzar Rols per a la Reutilització

Els rols et permeten organitzar els playbooks en components reutilitzables. Un rol típicament conté tasques, manejadors, variables, arxius i plantilles.

Exemple d'Estructura de Rol

my_role/
├── tasks/
│   └── main.yml
├── handlers/
│   └── main.yml
├── templates/
│   └── my_template.j2
├── files/
│   └── my_file
├── vars/
│   └── main.yml
├── defaults/
│   └── main.yml
├── meta/
│   └── main.yml

Exemple d'Ús de Rol

Al teu playbook, pots incloure un rol de la següent manera:

---
- hosts: launched
  roles:
    - my_role

Conclusió

El proveïment de núvol amb Ansible et permet automatitzar la configuració i gestió de recursos de núvol de manera eficient. Des de playbooks bàsics fins a tècniques avançades com l'inventari dinàmic i els rols, Ansible proporciona eines poderoses per gestionar la teva infraestructura com a codi. En dominar aquests conceptes, pots optimitzar les teves operacions de núvol i assegurar la consistència a través dels teus entorns.
