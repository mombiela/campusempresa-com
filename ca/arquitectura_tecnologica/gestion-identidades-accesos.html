Introducció

La Gestió d'Identitats i Accessos (IAM, per les seves sigles en anglès) és un component crític en l'arquitectura tecnològica de qualsevol organització. IAM s'encarrega de garantir que les persones adequades tinguin accés als recursos adequats en el moment adequat i per les raons correctes.

Conceptes Clau

  • Identitat Digital: Representació electrònica d'una entitat (usuari, dispositiu, aplicació).
  • Autenticació: Procés de verificar la identitat d'una entitat.
  • Autorització: Procés de determinar si una entitat té permís per realitzar una acció específica.
  • Single Sign-On (SSO): Sistema que permet als usuaris accedir a múltiples aplicacions amb una sola autenticació.
  • Federació d'Identitats: Mecanisme que permet als usuaris utilitzar les mateixes credencials per accedir a diferents dominis o sistemes.
  • Gestió de Rols: Assignació de permisos i accessos basats en rols predefinits.
  • Multi-Factor Authentication (MFA): Ús de múltiples mètodes de verificació per augmentar la seguretat.

Autenticació

L'autenticació és el primer pas en la gestió d'identitats i accessos. Hi ha diversos mètodes d'autenticació:

  • Contrasenyes: El mètode més comú, encara que no el més segur.
  • Biometria: Ús de característiques físiques com empremtes dactilars o reconeixement facial.
  • Tokens: Dispositius físics o aplicacions que generen codis d'un sol ús.
  • Certificats Digitals: Ús de certificats emesos per una autoritat de certificació (CA).

Exemple d'Autenticació amb Contrasenya en Python

def authenticate(username, password):
    stored_password = get_stored_password(username)
    if stored_password == password:
        return True
    else:
        return False

def get_stored_password(username):
    # Simulació d'una base de dades d'usuaris
    users_db = {
        'user1': 'password123',
        'user2': 'securepassword'
    }
    return users_db.get(username, None)

# Exemple d'ús
username = 'user1'
password = 'password123'
if authenticate(username, password):
    print("Autenticació exitosa")
else:
    print("Autenticació fallida")

Autorització

Un cop autenticat l'usuari, el següent pas és l'autorització. Això implica verificar si l'usuari té els permisos necessaris per accedir a un recurs o realitzar una acció.

  • Llistes de Control d'Accés (ACL): Llistes que especifiquen quins usuaris o grups tenen accés a certs recursos.
  • Rols i Permisos: Assignació de permisos a rols específics, i després assignació de rols a usuaris.

Exemple d'Autorització Basada en Rols en Python

roles_permissions = {
    'admin': ['read', 'write', 'delete'],
    'user': ['read']
}

user_roles = {
    'alice': 'admin',
    'bob': 'user'
}

def authorize(username, action):
    role = user_roles.get(username, None)
    if role and action in roles_permissions.get(role, []):
        return True
    else:
        return False

# Exemple d'ús
username = 'alice'
action = 'delete'
if authorize(username, action):
    print("Autorització exitosa")
else:
    print("Autorització fallida")

Single Sign-On (SSO)

El SSO permet als usuaris autenticar-se una sola vegada i obtenir accés a múltiples sistemes sense necessitat de tornar a autenticar-se. Això s'aconsegueix mitjançant protocols com SAML, OAuth i OpenID Connect.

Comparació de Protocols SSO

| Protocol | Descripció | Ús Comú | |---------------|--------------------------------------------------|----------------------------| | SAML | Protocol basat en XML per intercanvi de dades d'autenticació i autorització | Empreses i aplicacions corporatives | | OAuth | Protocol d'autorització que permet a les aplicacions obtenir accés limitat als recursos d'usuari | APIs i aplicacions web | | OpenID Connect| Extensió d'OAuth 2.0 per autenticació d'usuaris | Aplicacions mòbils i web |

Federació d'Identitats

La federació d'identitats permet als usuaris utilitzar les mateixes credencials per accedir a diferents sistemes o dominis. Això és comú en entorns on múltiples organitzacions necessiten col·laborar.

  • Proveïdors d'Identitat (IdP): Entitats que autentiquen els usuaris i proporcionen tokens d'identitat.
  • Proveïdors de Serveis (SP): Entitats que confien en els tokens d'identitat proporcionats pels IdP.

Multi-Factor Authentication (MFA)

MFA afegeix una capa addicional de seguretat en requerir múltiples mètodes de verificació. Els factors comuns inclouen:

  • Alguna cosa que saps: Contrasenya o PIN.
  • Alguna cosa que tens: Token físic o aplicació d'autenticació.
  • Alguna cosa que ets: Empremta dactilar o reconeixement facial.

Conclusió

La Gestió d'Identitats i Accessos és essencial per a la seguretat i eficiència de qualsevol sistema tecnològic. Comprendre i aplicar correctament els conceptes d'autenticació, autorització, SSO, federació d'identitats i MFA pot ajudar a protegir els recursos i dades d'una organització, alhora que millora l'experiència de l'usuari.

Aquest tema proporciona una base sòlida per entendre com gestionar identitats i accessos en un entorn tecnològic. A mesura que avancis en la teva carrera, aquests conceptes seran fonamentals per dissenyar i mantenir sistemes segurs i eficients.
