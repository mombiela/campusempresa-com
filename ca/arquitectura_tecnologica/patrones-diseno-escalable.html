Introducció

En aquesta secció, explorarem els patrons de disseny escalable, fonamentals per construir sistemes que puguin créixer i manejar majors càrregues de treball sense comprometre el rendiment. Aquests patrons són essencials per a arquitectes de programari, desenvolupadors i altres professionals de TI que busquen crear aplicacions robustes i eficients.

Patrons de Disseny Comuns

1. Patró Singleton

El patró Singleton assegura que una classe tingui només una instància i proporciona un punt d'accés global a aquesta instància.

  • Ús: Ideal per gestionar recursos compartits com connexions a bases de dades o configuracions globals.
  • Exemple de Codi: 
    class Singleton:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super(Singleton, cls).__new__(cls)
        return cls._instance

# Ús del Singleton
singleton1 = Singleton()
singleton2 = Singleton()
print(singleton1 is singleton2)  # True

2. Patró Factory

El patró Factory proporciona una interfície per crear objectes en una superclasse, però permet a les subclasses alterar el tipus d'objectes que es crearan.

  • Ús: Útil quan el procés de creació d'objectes és complex o quan es necessita flexibilitat en la creació d'objectes.
  • Exemple de Codi: 
    class Animal:
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

class AnimalFactory:
    @staticmethod
    def create_animal(animal_type):
        if animal_type == "dog":
            return Dog()
        elif animal_type == "cat":
            return Cat()
        else:
            return None

# Ús del Factory
factory = AnimalFactory()
dog = factory.create_animal("dog")
cat = factory.create_animal("cat")
print(dog.speak())  # Woof!
print(cat.speak())  # Meow!

3. Patró Observer

El patró Observer defineix una dependència un-a-molts entre objectes, de manera que quan un objecte canvia d'estat, tots els seus dependents són notificats i actualitzats automàticament.

  • Ús: Ideal per implementar sistemes d'esdeveniments i notificacions.
  • Exemple de Codi: 
    class Subject:
    def __init__(self):
        self._observers = []

    def attach(self, observer):
        self._observers.append(observer)

    def detach(self, observer):
        self._observers.remove(observer)

    def notify(self):
        for observer in self._observers:
            observer.update(self)

class Observer:
    def update(self, subject):
        pass

class ConcreteObserver(Observer):
    def update(self, subject):
        print("Observer notified")

# Ús del Observer
subject = Subject()
observer = ConcreteObserver()
subject.attach(observer)
subject.notify()  # Observer notified

Comparació de Patrons

| Patró | Ús Principal | Exemple d'Aplicació | |--------------|-------------------------------------------|--------------------------------------------| | Singleton | Gestió de recursos compartits | Connexions a bases de dades | | Factory | Creació flexible d'objectes | Creació d'objectes complexos | | Observer | Implementació de sistemes d'esdeveniments | Sistemes de notificacions i subscripcions |

Conclusió

Els patrons de disseny escalable són eines poderoses que permeten als desenvolupadors crear sistemes robustos i eficients. En comprendre i aplicar aquests patrons, els professionals de TI poden dissenyar arquitectures que no només satisfan les necessitats actuals, sinó que també estan preparades per al creixement futur.
