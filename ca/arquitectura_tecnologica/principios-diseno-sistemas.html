Introducció

El disseny de sistemes és una disciplina fonamental en l'arquitectura tecnològica que se centra en la planificació i estructuració de sistemes informàtics. Aquest tema cobreix els principis essencials que guien el disseny de sistemes robustos, escalables i mantenibles.

Principis de Disseny de Sistemes

1. Modularitat

  • Definició: La modularitat implica dividir un sistema en parts més petites i manejables anomenades mòduls.
  • Beneficis:
    • Facilita la comprensió i el manteniment del sistema.
    • Permet la reutilització de components.
    • Millora l'escalabilitat i la flexibilitat.

2. Abstracció

  • Definició: L'abstracció és el procés de simplificar la complexitat ocultant els detalls d'implementació i mostrant només les funcionalitats essencials.
  • Beneficis:
    • Redueix la complexitat del sistema.
    • Facilita la comunicació entre diferents parts del sistema.
    • Permet la creació d'interfícies clares i concises.

3. Encapsulament

  • Definició: L'encapsulament consisteix a agrupar dades i mètodes que operen sobre aquestes dades dins d'una única unitat, com una classe en programació orientada a objectes.
  • Beneficis:
    • Protegeix les dades d'accessos no autoritzats.
    • Millora la modularitat i la reutilització del codi.
    • Facilita el manteniment i l'evolució del sistema.

4. Separació de Concerns (SoC)

  • Definició: La separació de concerns és el principi de dividir un sistema en parts distintes, cadascuna de les quals aborda una preocupació específica.
  • Beneficis:
    • Millora la claredat i l'organització del codi.
    • Facilita el desenvolupament paral·lel i la col·laboració.
    • Redueix l'acoblament entre components.

5. Principi de Responsabilitat Única (SRP)

  • Definició: Cada mòdul o classe ha de tenir una única responsabilitat o raó per canviar.
  • Beneficis:
    • Facilita la comprensió i el manteniment del codi.
    • Redueix el risc d'errors i conflictes.
    • Millora la cohesió del sistema.

6. Principi de Obert/Tancat (OCP)

  • Definició: Els mòduls han d'estar oberts per a l'extensió però tancats per a la modificació.
  • Beneficis:
    • Facilita l'evolució del sistema sense afectar el codi existent.
    • Promou la reutilització i l'extensibilitat.

7. Principi de Substitució de Liskov (LSP)

  • Definició: Els objectes d'una classe derivada han de poder reemplaçar objectes de la classe base sense alterar el comportament del sistema.
  • Beneficis:
    • Garanteix la interoperabilitat entre classes.
    • Millora la robustesa i la flexibilitat del sistema.

8. Principi d'Inversió de Dependència (DIP)

  • Definició: Els mòduls d'alt nivell no han de dependre de mòduls de baix nivell; ambdós han de dependre d'abstraccions.
  • Beneficis:
    • Redueix l'acoblament entre mòduls.
    • Facilita la prova i el manteniment del sistema.
    • Promou la reutilització de components.

9. Principi de Segregació d'Interfícies (ISP)

  • Definició: Els clients no han d'estar obligats a dependre d'interfícies que no utilitzen.
  • Beneficis:
    • Millora la cohesió i la claredat de les interfícies.
    • Redueix l'acoblament i facilita l'evolució del sistema.

Exemples de Codi

Exemple d'Encapsulament en Java

public class CompteBancari {
    private double saldo;

    public CompteBancari(double saldoInicial) {
        this.saldo = saldoInicial;
    }

    public double getSaldo() {
        return saldo;
    }

    public void dipositar(double quantitat) {
        si (quantitat > 0) {
            saldo += quantitat;
        }
    }

    public void retirar(double quantitat) {
        si (quantitat > 0 && quantitat <= saldo) {
            saldo -= quantitat;
        }
    }
}

Exemple d'Abstracció en Python

from abc import ABC, abstractmethod

class Forma(ABC):
    @abstractmethod
    def area(self):
        pass

class Cercle(Forma):
    def __init__(self, radi):
        self.radi = radi

    def area(self):
        return 3.14 * self.radi ** 2

class Rectangle(Forma):
    def __init__(self, ample, alt):
        self.ample = ample
        self.alt = alt

    def area(self):
        return self.ample * self.alt

Conclusió

Els principis de disseny de sistemes són fonamentals per crear arquitectures tecnològiques robustes, escalables i mantenibles. La comprensió i aplicació d'aquests principis permeten als professionals de TI dissenyar sistemes que no només compleixen amb els requisits actuals, sinó que també poden adaptar-se i evolucionar amb el temps.
