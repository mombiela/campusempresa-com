El Projecte Sobre nosaltres Contribuir Donacions Llicència

Introducció

En aquest projecte, desenvoluparem un sistema de col·lisions per a un videojoc. La detecció i maneig de col·lisions és fonamental per a la física de videojocs, ja que permet que els objectes interactuïn de manera realista. Aprendrem a implementar col·lisions bàsiques entre diferents tipus d'objectes i com respondre a aquestes col·lisions de manera adequada.

Objectius del Projecte

  • Implementar detecció de col·lisions entre diferents tipus d'objectes.
  • Manejar les respostes a les col·lisions de manera realista.
  • Optimitzar el sistema de col·lisions per millorar el rendiment del joc.

Conceptes Clau

  • Col·lisió: Ocorre quan dos o més objectes en el joc es toquen o se superposen.
  • Bounding Box: Un rectangle (o caixa) que envolta un objecte, utilitzat per simplificar la detecció de col·lisions.
  • Bounding Circle: Un cercle que envolta un objecte, una altra forma de simplificar la detecció de col·lisions.
  • Resposta a la Col·lisió: L'acció que es pren quan es detecta una col·lisió, com rebotar, aturar-se o lliscar-se.

Tipus de Col·lisions

Col·lisions AABB (Axis-Aligned Bounding Box)

Les col·lisions AABB són una forma simple i eficient de detectar col·lisions entre objectes rectangulars.

  • Avantatges:

    • Fàcil d'implementar.
    • Ràpid en termes de càlcul.

  • Desavantatges:

    • No és adequat per a objectes rotats o formes complexes.

Exemple de Codi: Detecció de Col·lisions AABB

class BoundingBox:
    def __init__(self, x, y, width, height):
        self.x = x
        self.y = y
        self.width = width
        self.height = height

def check_collision(box1, box2):
    return (box1.x < box2.x + box2.width and
            box1.x + box1.width > box2.x and
            box1.y < box2.y + box2.height and
            box1.y + box1.height > box2.y)

# Exemple d'ús
box1 = BoundingBox(0, 0, 50, 50)
box2 = BoundingBox(25, 25, 50, 50)

if check_collision(box1, box2):
    print("Col·lisió detectada")
else:
    print("No hi ha col·lisió")

Col·lisions de Cercles

Les col·lisions de cercles són útils per a objectes rodons o quan es necessita una detecció de col·lisions més precisa que AABB.

  • Avantatges:

    • Més precís per a objectes rodons.
    • Adequat per a objectes rotats.

  • Desavantatges:

    • Més costós en termes de càlcul comparat amb AABB.

Exemple de Codi: Detecció de Col·lisions de Cercles

import math

class BoundingCircle:
    def __init__(self, x, y, radius):
        self.x = x
        self.y = y
        self.radius = radius

def check_collision(circle1, circle2):
    distance = math.sqrt((circle1.x - circle2.x)**2 + (circle1.y - circle2.y)**2)
    return distance < (circle1.radius + circle2.radius)

# Exemple d'ús
circle1 = BoundingCircle(0, 0, 25)
circle2 = BoundingCircle(30, 30, 25)

if check_collision(circle1, circle2):
    print("Col·lisió detectada")
else:
    print("No hi ha col·lisió")

Resposta a la Col·lisió

Una vegada detectada una col·lisió, és crucial manejar la resposta de manera adequada. Les respostes poden variar depenent del tipus de joc i dels objectes involucrats.

  • Rebot: Els objectes reboten un contra l'altre.
  • Aturada: Els objectes s'aturen en col·lidir.
  • Lliscar-se: Els objectes es llisquen un sobre l'altre.

Exemple de Codi: Resposta de Rebot

def handle_collision_rebound(obj1, obj2):
    # Invertir la velocitat dels objectes
    obj1.vx = -obj1.vx
    obj1.vy = -obj1.vy
    obj2.vx = -obj2.vx
    obj2.vy = -obj2.vy

# Exemple d'ús
class GameObject:
    def __init__(self, x, y, vx, vy):
        self.x = x
        self.y = y
        self.vx = vx
        self.vy = vy

obj1 = GameObject(0, 0, 1, 1)
obj2 = GameObject(25, 25, -1, -1)

if check_collision(box1, box2):
    handle_collision_rebound(obj1, obj2)

Optimització del Sistema de Col·lisions

Per millorar el rendiment del sistema de col·lisions, especialment en jocs amb molts objectes, es poden utilitzar tècniques d'optimització com:

  • Quadrícula Espacial: Dividir l'espai del joc en una quadrícula i només comprovar col·lisions entre objectes en la mateixa cel·la o cel·les adjacents.
  • Arbres de Col·lisió: Utilitzar estructures de dades com Quadtrees o BSP Trees per organitzar els objectes i reduir el nombre de comprovacions de col·lisió.

Conclusió

En aquest projecte, hem après a implementar un sistema de col·lisions bàsic utilitzant Bounding Boxes i Bounding Circles. També hem explorat diferents respostes a les col·lisions i discutit tècniques d'optimització per millorar el rendiment. La detecció i maneig de col·lisions és una habilitat essencial en el desenvolupament de videojocs i proporciona una base sòlida per crear interaccions realistes entre els objectes del joc.
