Introducció

La detecció de col·lisions és un component essencial en la física de videojocs. Permet determinar quan dos o més objectes en un entorn virtual es toquen o interactuen d'alguna manera. Aquest procés és crucial per a la jugabilitat i la física realista en els videojocs.

Conceptes Clau

  • Col·lisió: Passa quan dos objectes ocupen el mateix espai en un moment donat.
  • Bounding Box (Caixa Envolvent): Una caixa rectangular que envolta un objecte, utilitzada per simplificar la detecció de col·lisions.
  • Bounding Sphere (Esfera Envolvent): Una esfera que envolta un objecte, una altra simplificació per a la detecció de col·lisions.
  • Algorisme de Separació d'Eixos (SAT): Un mètode per determinar si dos polígons convexos estan col·lisionant.
  • Broad Phase i Narrow Phase: Dues etapes en la detecció de col·lisions. La fase ampla (Broad Phase) redueix el nombre de possibles col·lisions, i la fase estreta (Narrow Phase) realitza una detecció més precisa.

Tipus de Col·lisions

Col·lisions AABB (Axis-Aligned Bounding Box)

Les col·lisions AABB són una de les formes més simples i ràpides de detecció de col·lisions. S'utilitzen caixes rectangulars alineades amb els eixos del sistema de coordenades.

class AABB:
    def __init__(self, x, y, width, height):
        self.x = x
        self.y = y
        self.width = width
        self.height = height

def check_collision(a, b):
    return (a.x < b.x + b.width and
            a.x + a.width > b.x and
            a.y < b.y + b.height and
            a.y + a.height > b.y)

# Exemple d'ús
a = AABB(0, 0, 10, 10)
b = AABB(5, 5, 10, 10)
print(check_collision(a, b))  # Torna True

Col·lisions d'Esferes

Les col·lisions d'esferes són útils quan els objectes tenen una forma més circular. La detecció es basa en la distància entre els centres de les esferes.

import math

class Sphere:
    def __init__(self, x, y, radius):
        self.x = x
        self.y = y
        self.radius = radius

def check_collision(s1, s2):
    distance = math.sqrt((s1.x - s2.x)**2 + (s1.y - s2.y)**2)
    return distance < (s1.radius + s2.radius)

# Exemple d'ús
s1 = Sphere(0, 0, 5)
s2 = Sphere(7, 0, 5)
print(check_collision(s1, s2))  # Torna True

Algorisme de Separació d'Eixos (SAT)

El SAT és més complex però molt efectiu per a polígons convexos. Es basa en projectar els vèrtexs dels polígons en un eix i comprovar si hi ha una separació.

# Exemple simplificat de SAT en pseudocodi
def sat_collision(poly1, poly2):
    for axis in poly1.get_axes() + poly2.get_axes():
        projection1 = poly1.project(axis)
        projection2 = poly2.project(axis)
        if not projection1.overlaps(projection2):
            return False
    return True

# poly1 i poly2 són objectes que tenen mètodes get_axes() i project(axis)

Broad Phase i Narrow Phase

Broad Phase

La fase ampla redueix el nombre de possibles col·lisions utilitzant mètodes com el barrido i poda (Sweep and Prune) o les quadrícules espacials.

Narrow Phase

La fase estreta realitza una detecció de col·lisions més precisa entre els parells d'objectes que van passar la fase ampla.

Conclusió

La detecció de col·lisions és fonamental per a la física de videojocs, proporcionant la base per a interaccions realistes entre objectes. Des de mètodes simples com les col·lisions AABB i d'esferes fins a tècniques més avançades com el SAT, cada mètode té els seus avantatges i desavantatges. Comprendre i aplicar aquests conceptes és crucial per a qualsevol desenvolupador de videojocs.

Aquest tema proporciona una base sòlida per a la detecció de col·lisions en videojocs. A mesura que avancis en el curs, explorarem més tècniques i optimitzacions per millorar l'eficiència i precisió de la detecció de col·lisions.
