Introducció

En aquest projecte, aplicarem tots els conceptes de física de videojocs que hem après fins ara per implementar una simulació completa en un motor de joc. Utilitzarem un motor de joc popular com Unity o Unreal Engine per dur a terme aquesta tasca. L'objectiu és crear un entorn interactiu on els principis de la física s'apliquin de manera realista i eficient.

Objectius del Projecte

  • Integrar conceptes de física en un motor de joc.
  • Implementar col·lisions i respostes físiques.
  • Aplicar forces i moviments realistes.
  • Optimitzar el rendiment de la simulació física.

Requisits Previs

  • Coneixement bàsic de programació en C# (per a Unity) o C++ (per a Unreal Engine).
  • Familiaritat amb l'entorn de desenvolupament del motor de joc escollit.
  • Comprensió dels conceptes bàsics de física de videojocs.

Configuració del Projecte

Selecció del Motor de Joc

Per a aquest projecte, pots escollir entre Unity i Unreal Engine. Ambdós motors són àmpliament utilitzats en la indústria i ofereixen potents eines per a la simulació física.

  • Unity: Utilitza C# com a llenguatge de programació principal.
  • Unreal Engine: Utilitza C++ com a llenguatge de programació principal.

Creació del Projecte

  1. Unity:

    • Obre Unity Hub i crea un nou projecte 3D.
    • Configura l'escena inicial amb un pla i alguns objectes bàsics (cubos, esferes).

  2. Unreal Engine:

    • Obre Unreal Engine i crea un nou projecte de tipus "Blank" o "Third Person".
    • Configura l'escena inicial amb un terreny i alguns objectes bàsics.

Implementació de la Física

Configuració dels Components Físics

  1. Unity:

    • Afegeix components Rigidbody als objectes per habilitar la física.
    • Configura les propietats del Rigidbody com massa, arrossegament i gravetat.
    // Exemple de configuració d'un Rigidbody a Unity
Rigidbody rb = gameObject.AddComponent<Rigidbody>();
rb.mass = 1.0f;
rb.drag = 0.5f;
rb.useGravity = true;

  2. Unreal Engine:

    • Afegeix components StaticMesh i Physics als objectes.
    • Configura les propietats físiques com massa i fricció.
    // Exemple de configuració d'un component físic a Unreal Engine
UStaticMeshComponent* MeshComponent = CreateDefaultSubobject<UStaticMeshComponent>(TEXT("MeshComponent"));
MeshComponent->SetSimulatePhysics(true);
MeshComponent->SetMassOverrideInKg(NAME_None, 1.0f);

Implementació de Col·lisions

  1. Unity:

    • Utilitza colliders (BoxCollider, SphereCollider) per definir les àrees de col·lisió.
    • Gestiona esdeveniments de col·lisió utilitzant mètodes com OnCollisionEnter.
    void OnCollisionEnter(Collision collision)
{
    Debug.Log("Collision detected with " + collision.gameObject.name);
}

  2. Unreal Engine:

    • Utilitza components de col·lisió (BoxComponent, SphereComponent).
    • Gestiona esdeveniments de col·lisió utilitzant funcions com OnHit.
    void AMyActor::OnHit(UPrimitiveComponent* HitComp, AActor* OtherActor, UPrimitiveComponent* OtherComp, FVector NormalImpulse, const FHitResult& Hit)
{
    UE_LOG(LogTemp, Warning, TEXT("Collision detected with %s"), *OtherActor->GetName());
}

Aplicació de Forces i Moviments

  1. Unity:

    • Aplica forces utilitzant el mètode AddForce del Rigidbody.
    rb.AddForce(Vector3.up * 10.0f, ForceMode.Impulse);

  2. Unreal Engine:

    • Aplica forces utilitzant el mètode AddForce del component físic.
    MeshComponent->AddForce(FVector(0, 0, 1000), NAME_None, true);

Optimització del Rendiment

  • Unity:

    • Utilitza FixedUpdate per a les actualitzacions físiques.
    • Optimitza els colliders i redueix la complexitat geomètrica.

  • Unreal Engine:

    • Utilitza Tick i Physics Sub-Stepping per millorar la precisió.
    • Optimitza les malles i redueix el nombre d'actors físics.

Conclusió

En aquest projecte, hem integrat els conceptes de física de videojocs en un motor de joc, creant una simulació interactiva i realista. Hem après a configurar components físics, gestionar col·lisions i aplicar forces, així com a optimitzar el rendiment de la nostra simulació. Aquest projecte ens proporciona una base sòlida per desenvolupar jocs i simulacions físiques més complexes en el futur.
