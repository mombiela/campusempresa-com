Moviment i Controls del Personatge

Introducció

En aquesta secció, cobrirem els aspectes essencials del moviment i controls del personatge a Unreal Engine. Aquest tema és crucial per crear una jugabilitat interactiva i atractiva. Començarem amb el moviment bàsic i gradualment avançarem cap a mecanismes de control més avançats.

Moviment Bàsic del Personatge

Configuració d'un Personatge

  • Creació d'un Nou Blueprint de Personatge: Comença creant un nou Blueprint de Personatge a Unreal Engine.
  • Afegir Components: Afegeix els components necessaris com un Component de Càpsula per a la col·lisió, un Mesh Esquelètic per al model del personatge, i un Component de Moviment del Personatge per gestionar el moviment.
// Exemple: Configuració d'un personatge bàsic en C++
#include "GameFramework/Character.h"
#include "MyCharacter.generated.h"

UCLASS()
class AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    AMyCharacter();
};

Configuració del Moviment

  • Component de Moviment del Personatge: Aquest component gestiona tota la lògica de moviment.
    • Velocitat de Caminada: Ajusta la propietat MaxWalkSpeed per establir la velocitat de caminada del personatge.
    • Salt: Activa el salt configurant JumpZVelocity.
// Exemple: Configuració del moviment en C++
AMyCharacter::AMyCharacter()
{
    // Establir la velocitat de caminada per defecte
    GetCharacterMovement()->MaxWalkSpeed = 600.0f;

    // Activar el salt
    GetCharacterMovement()->JumpZVelocity = 600.0f;
}

Configuració d'Entrada

  • Configuració del Projecte: Ves a Editar > Configuració del Projecte > Entrada per configurar els mapatges d'entrada.
    • Mapatges d'Eix: Per a entrades contínues com el moviment (per exemple, MoveForward, MoveRight).
    • Mapatges d'Acció: Per a accions discretes com saltar (per exemple, Jump).
// Exemple: Configuració de les vinculacions d'entrada en C++
void AMyCharacter::SetupPlayerInputComponent(UInputComponent* PlayerInputComponent)
{
    Super::SetupPlayerInputComponent(PlayerInputComponent);

    // Vincular les entrades de moviment
    PlayerInputComponent->BindAxis("MoveForward", this, &AMyCharacter::MoveForward);
    PlayerInputComponent->BindAxis("MoveRight", this, &AMyCharacter::MoveRight);

    // Vincular l'entrada de salt
    PlayerInputComponent->BindAction("Jump", IE_Pressed, this, &ACharacter::Jump);
    PlayerInputComponent->BindAction("Jump", IE_Released, this, &ACharacter::StopJumping);
}

Implementació de Funcions de Moviment

  • MoveForward i MoveRight: Implementa aquestes funcions per gestionar el moviment del personatge basat en l'entrada.
// Exemple: Implementació de funcions de moviment en C++
void AMyCharacter::MoveForward(float Value)
{
    if (Value != 0.0f)
    {
        AddMovementInput(GetActorForwardVector(), Value);
    }
}

void AMyCharacter::MoveRight(float Value)
{
    if (Value != 0.0f)
    {
        AddMovementInput(GetActorRightVector(), Value);
    }
}

Controls Intermedis del Personatge

Control de la Càmera

  • Component de Braç de Molla: Adjunta un Component de Braç de Molla al personatge per controlar la posició i rotació de la càmera.
  • Component de Càmera: Adjunta un Component de Càmera al Braç de Molla.
// Exemple: Configuració del control de la càmera en C++
AMyCharacter::AMyCharacter()
{
    // Crear un Component de Braç de Molla
    SpringArm = CreateDefaultSubobject<USpringArmComponent>(TEXT("SpringArm"));
    SpringArm->SetupAttachment(RootComponent);
    SpringArm->TargetArmLength = 300.0f;

    // Crear un Component de Càmera
    Camera = CreateDefaultSubobject<UCameraComponent>(TEXT("Camera"));
    Camera->SetupAttachment(SpringArm);
}

Mecàniques de Moviment Avançades

  • Ajupir-se: Activa l'ajupiment configurant CanCrouch a true i vinculant les entrades d'ajupiment.
  • Esprintar: Implementa l'esprint modificant dinàmicament el MaxWalkSpeed.
// Exemple: Implementació d'ajupiment i esprint en C++
void AMyCharacter::SetupPlayerInputComponent(UInputComponent* PlayerInputComponent)
{
    Super::SetupPlayerInputComponent(PlayerInputComponent);

    // Vincular les entrades d'ajupiment
    PlayerInputComponent->BindAction("Crouch", IE_Pressed, this, &AMyCharacter::StartCrouch);
    PlayerInputComponent->BindAction("Crouch", IE_Released, this, &AMyCharacter::StopCrouch);

    // Vincular les entrades d'esprint
    PlayerInputComponent->BindAction("Sprint", IE_Pressed, this, &AMyCharacter::StartSprint);
    PlayerInputComponent->BindAction("Sprint", IE_Released, this, &AMyCharacter::StopSprint);
}

void AMyCharacter::StartCrouch()
{
    Crouch();
}

void AMyCharacter::StopCrouch()
{
    UnCrouch();
}

void AMyCharacter::StartSprint()
{
    GetCharacterMovement()->MaxWalkSpeed = 1200.0f;
}

void AMyCharacter::StopSprint()
{
    GetCharacterMovement()->MaxWalkSpeed = 600.0f;
}

Controls Avançats del Personatge

Modes de Moviment Personalitzats

  • Vol: Implementa el vol configurant el mode de moviment a MOVE_Flying.
  • Nedar: Gestiona la natació configurant el mode de moviment a MOVE_Swimming.
// Exemple: Implementació de modes de moviment personalitzats en C++
void AMyCharacter::StartFlying()
{
    GetCharacterMovement()->SetMovementMode(MOVE_Flying);
}

void AMyCharacter::StopFlying()
{
    GetCharacterMovement()->SetMovementMode(MOVE_Walking);
}

Integració d'Animacions

  • Blueprints d'Animació: Utilitza Blueprints d'Animació per barrejar animacions basades en els estats de moviment.
  • Màquines d'Estat: Implementa màquines d'estat per gestionar les transicions entre diferents animacions de moviment.
// Exemple: Configuració bàsica de Blueprint d'Animació
// En el Blueprint d'Animació, crea una màquina d'estat amb estats com Idle, Walk, Run, Jump, etc.

Xarxes i Replicació

  • Replicació del Moviment: Assegura't que el moviment del personatge es repliqui a través de la xarxa per a jocs multijugador.
  • Validació del Costat del Servidor: Valida les entrades de moviment al servidor per prevenir trampes.
// Exemple: Activació de la replicació del moviment en C++
AMyCharacter::AMyCharacter()
{
    // Activar la replicació
    bReplicates = true;
    bReplicateMovement = true;
}

Conclusió
