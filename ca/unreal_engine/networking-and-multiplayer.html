Introducció a les Xarxes a Unreal Engine

Les xarxes són un aspecte crucial del desenvolupament de jocs moderns, permetent als jugadors interactuar entre ells en un món de joc compartit. Unreal Engine proporciona un marc robust per crear jocs multijugador, des de configuracions senzilles peer-to-peer fins a arquitectures complexes client-servidor.

Conceptes Clau

  • Model Client-Servidor: Unreal Engine utilitza un model client-servidor on el servidor gestiona l'estat del joc, i els clients envien entrades al servidor.
  • Replicació: El procés de sincronitzar dades entre el servidor i els clients.
  • RPC (Remote Procedure Calls): Funcions que es poden cridar a través de la xarxa.
  • Latència i Amplada de Banda: Comprendre el rendiment de la xarxa i l'optimització.

Configuració d'un Joc Multijugador Bàsic

Creació d'un Nou Projecte

  1. Obre Unreal Engine i crea un nou projecte.
  2. Selecciona la plantilla "Third Person" per simplicitat.
  3. Activa el "Starter Content" per obtenir actius addicionals.

Configuració dels Paràmetres de Xarxa

  1. Ves a Edit > Project Settings > Maps & Modes.
  2. Configura el Number of Players a 2 o més.
  3. Activa Use Single Process per a proves locals.

Generació de Jugadors

// A la teva classe GameMode
void AMyGameMode::PostLogin(APlayerController* NewPlayer)

    // Gestiona la lògica de generació de jugadors aquí

  • Explicació: PostLogin es crida quan un jugador inicia sessió amb èxit. Pots sobreescriure aquesta funció per gestionar la generació de jugadors.

Replicació

Comprensió de la Replicació

La replicació és el procés de sincronitzar dades entre el servidor i els clients. Unreal Engine utilitza el sistema UProperty per marcar variables per a la replicació.

Replicació de Variables

UCLASS()
class MYGAME_API AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    UPROPERTY(Replicated)
    int32 Health;

    virtual void GetLifetimeReplicatedProps(TArray<FLifetimeProperty>& OutLifetimeProps) const override;
};

void AMyCharacter::GetLifetimeReplicatedProps(TArray<FLifetimeProperty>& OutLifetimeProps) const
{
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);
    DOREPLIFETIME(AMyCharacter, Health);
}
  • Explicació: La variable Health està marcada per a la replicació utilitzant UPROPERTY(Replicated). La funció GetLifetimeReplicatedProps es sobreescriu per especificar quines propietats s'han de replicar.

Remote Procedure Calls (RPCs)

Tipus de RPCs

  • Client RPCs: Funcions cridades al client des del servidor.
  • Server RPCs: Funcions cridades al servidor des del client.
  • Multicast RPCs: Funcions cridades a tots els clients des del servidor.

Implementació de RPCs

UCLASS()
class MYGAME_API AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    UFUNCTION(Server, Reliable, WithValidation)
    void ServerDoSomething();

    UFUNCTION(Client, Reliable)
    void ClientDoSomething();

    UFUNCTION(NetMulticast, Reliable)
    void MulticastDoSomething();
};

void AMyCharacter::ServerDoSomething_Implementation()
{
    // Lògica del servidor
}

bool AMyCharacter::ServerDoSomething_Validate()
{
    return true; // Afegeix la lògica de validació aquí
}

void AMyCharacter::ClientDoSomething_Implementation()
{
    // Lògica del client
}

void AMyCharacter::MulticastDoSomething_Implementation()
{
    // Lògica per a tots els clients
}
  • Explicació: ServerDoSomething és un RPC de servidor, ClientDoSomething és un RPC de client, i MulticastDoSomething és un RPC multicast. El sufix _Implementation s'utilitza per definir la lògica de la funció.

Tècniques Avançades de Xarxa

Optimització del Rendiment de la Xarxa

  • Rellevància de la Xarxa: Només replicar dades rellevants als clients.
  • Gestió de l'Amplada de Banda: Limitar la quantitat de dades enviades a través de la xarxa.
  • Predicció i Interpolació: Suavitzar el moviment i les accions per compensar la latència.

Exemple de Rellevància de la Xarxa

void AMyCharacter::GetLifetimeReplicatedProps(TArray<FLifetimeProperty>& OutLifetimeProps) const
{
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);
    DOREPLIFETIME_CONDITION(AMyCharacter, Health, COND_OwnerOnly);
}
  • Explicació: La variable Health només es replica al client propietari utilitzant COND_OwnerOnly.

Conclusió

El desenvolupament de xarxes i multijugador a Unreal Engine pot ser complex, però comprendre els conceptes bàsics del model client-servidor, la replicació i els RPCs és essencial. Seguint les millors pràctiques i optimitzant el rendiment de la xarxa, pots crear experiències multijugador atractives i responsives. Continua experimentant amb aquests conceptes per aprofundir en la teva comprensió i millorar les capacitats multijugador del teu joc.
