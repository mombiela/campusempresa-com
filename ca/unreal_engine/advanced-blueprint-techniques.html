Introducció

En aquesta secció, explorarem tècniques avançades per utilitzar Blueprints a Unreal Engine. Els Blueprints són un potent sistema de scripting visual que et permet crear elements de joc sense escriure codi. Aquesta secció assumeix que tens una comprensió bàsica dels Blueprints i busques ampliar les teves habilitats.

Comunicació entre Blueprints

Comunicació Directa entre Blueprints

La comunicació directa implica cridar funcions o accedir a variables directament d'un Blueprint a un altre.

// Exemple: Cridar una funció en un altre Blueprint
// Suposem que tenim un Blueprint anomenat 'BP_Target' amb una funció 'TargetFunction'

BP_Target* TargetBlueprint = Cast<BP_Target>(TargetActor);
if (TargetBlueprint)
{
    TargetBlueprint->TargetFunction();
}

Dispatchers d'Esdeveniments

Els Dispatchers d'Esdeveniments permeten als Blueprints enviar esdeveniments que altres Blueprints poden escoltar i respondre.

// Exemple: Configurar un Dispatcher d'Esdeveniments
// En el Blueprint que envia l'esdeveniment
EventDispatcherName.Broadcast();

// En el Blueprint que escolta l'esdeveniment
TargetBlueprint->EventDispatcherName.AddDynamic(this, &ThisClass::FunctionToCall);

Interfícies de Blueprints

Les Interfícies de Blueprints defineixen un conjunt de funcions que altres Blueprints poden implementar, permetent una comunicació més flexible.

// Exemple: Utilitzar una Interfície de Blueprint
// Definir la interfície amb les funcions requerides
UINTERFACE(Blueprintable)
class UMyInterface : public UInterface
{
    GENERATED_BODY()
};

class IMyInterface
{
    GENERATED_BODY()

public:
    UFUNCTION(BlueprintNativeEvent, BlueprintCallable, Category = "MyInterface")
    void MyFunction();
};

// Implementar la interfície en un Blueprint
class MyBlueprint : public AActor, public IMyInterface
{
    GENERATED_BODY()

public:
    virtual void MyFunction_Implementation() override
    {
        // Implementació de la funció
    }
};

Variables Avançades i Tipus de Dades

Structs

Els Structs són tipus de dades personalitzats que agrupen múltiples variables.

// Exemple: Definir i utilitzar un Struct
USTRUCT(BlueprintType)
struct FMyStruct
{
    GENERATED_BODY()

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "MyStruct")
    int32 MyInt;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "MyStruct")
    FString MyString;
};

// Utilitzar el Struct en un Blueprint
FMyStruct MyStructInstance;
MyStructInstance.MyInt = 10;
MyStructInstance.MyString = "Hello";

Enums

Els Enums són una manera de definir un conjunt de valors amb nom.

// Exemple: Definir i utilitzar un Enum
UENUM(BlueprintType)
enum class EMyEnum : uint8
{
    VE_Option1 UMETA(DisplayName = "Option 1"),
    VE_Option2 UMETA(DisplayName = "Option 2")
};

// Utilitzar l'Enum en un Blueprint
EMyEnum MyEnumValue = EMyEnum::VE_Option1;

Arrays i Maps

Els Arrays i Maps són col·leccions de variables.

// Exemple: Utilitzar Arrays i Maps
TArray<int32> MyArray;
MyArray.Add(1);
MyArray.Add(2);

TMap<FString, int32> MyMap;
MyMap.Add("Key1", 1);
MyMap.Add("Key2", 2);

Tècniques d'Optimització

Nativització de Blueprints

La Nativització de Blueprints converteix els Blueprints a codi C++, millorant el rendiment.

  • Nativització Inclusiva: Converteix tots els Blueprints.
  • Nativització Exclusiva: Converteix Blueprints seleccionats.

Ús Eficient del Tick

Minimitza l'ús de les funcions Tick per millorar el rendiment.

  • Utilitza Timers i Esdeveniments en lloc de Tick quan sigui possible.
  • Només habilita Tick per actors essencials.

Gestió de Memòria

Una gestió eficient de la memòria és crucial per al rendiment.

  • Utilitza agrupació d'objectes per actors que es generen freqüentment.
  • Gestiona correctament les referències per evitar fuites de memòria.

Gestió Avançada d'Esdeveniments

Esdeveniments Personalitzats

Els Esdeveniments Personalitzats et permeten crear nodes d'esdeveniments reutilitzables en Blueprints.

// Exemple: Crear i utilitzar un Esdeveniment Personalitzat
UFUNCTION(BlueprintCallable, Category = "CustomEvent")
void MyCustomEvent();

Timers

Els Timers et permeten executar funcions a intervals especificats.

// Exemple: Configurar un Timer
GetWorld()->GetTimerManager().SetTimer(TimerHandle, this, &ThisClass::FunctionToCall, 1.0f, true);

Delegats

Els Delegats són una manera de passar funcions com a paràmetres i cridar-les més tard.

// Exemple: Utilitzar Delegats
DECLARE_DELEGATE(FMyDelegate);

FMyDelegate MyDelegate;
MyDelegate.BindUFunction(this, FName("FunctionToCall"));
MyDelegate.ExecuteIfBound();

Blueprints d'Animació

Màquines d'Estats

Les Màquines d'Estats gestionen diferents estats d'animació.

  • Defineix estats i transicions.
  • Utilitza condicions per controlar les transicions.

Espais de Mescla

Els Espais de Mescla permeten transicions suaus entre animacions basades en paràmetres d'entrada.

  • Defineix eixos d'entrada.
  • Afegeix animacions a la graella.

Montatges d'Animació

Els Montatges d'Animació et permeten reproduir seqüències d'animació complexes.

  • Crea un Montatge a partir d'animacions.
  • Utilitza nodes de Montatge per controlar la reproducció.

Conclusió

Les tècniques avançades de Blueprints a Unreal Engine proporcionen eines poderoses per crear sistemes de joc complexos i eficients. En dominar la comunicació entre Blueprints, variables avançades, tècniques d'optimització, gestió d'esdeveniments i Blueprints d'animació, pots millorar significativament les teves habilitats de desenvolupament de jocs. Practica aquestes tècniques per convertir-te en un expert en la creació de jocs d'alt rendiment i amb moltes funcionalitats.
