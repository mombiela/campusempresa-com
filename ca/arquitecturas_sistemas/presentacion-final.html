Objectius de la Presentació Final

La presentació final té com a objectiu que els estudiants demostrin la seva comprensió i aplicació dels conceptes apresos al llarg del curs d'Arquitectures de Sistemes. Els estudiants hauran de:

  • Presentar un projecte d'arquitectura de sistema complet.
  • Explicar les decisions de disseny preses.
  • Mostrar com s'implementen els principis d'arquitectura en el seu projecte.
  • Respondre preguntes i rebre retroalimentació.

Estructura de la Presentació

Cada presentació ha de seguir una estructura clara i lògica per assegurar que tots els aspectes importants siguin coberts. L'estructura recomanada és la següent:

  1. Introducció

    • Breu descripció del projecte.
    • Objectius del sistema.
    • Públic objectiu i necessitats de l'usuari.

  2. Arquitectura del Sistema

    • Diagrama d'arquitectura general.
    • Components principals i les seves interaccions.
    • Justificació de l'elecció de l'arquitectura (e.g., microserveis, monolítica, etc.).

  3. Tecnologies Utilitzades

    • Llenguatges de programació.
    • Frameworks i biblioteques.
    • Eines de desenvolupament i desplegament.

  4. Patrons de Disseny

    • Patrons de disseny aplicats (e.g., Singleton, Factory, Observer).
    • Exemples de codi que demostrin la implementació d'aquests patrons.

  5. Seguretat i Escalabilitat

    • Estratègies de seguretat implementades.
    • Plans d'escalabilitat i com es manegen les càrregues de treball.

  6. Proves i Validació

    • Mètodes de prova utilitzats (e.g., unit tests, integration tests).
    • Resultats de les proves i com es va validar el sistema.

  7. Conclusió

    • Resum dels punts clau.
    • Lliçons apreses.
    • Possibles millores futures.

Exemple de Projecte

Introducció

El projecte presentat és un sistema de gestió d'inventaris per a una botiga en línia. L'objectiu del sistema és permetre als administradors gestionar l'inventari de productes, mentre que els clients poden veure la disponibilitat dels productes en temps real.

Arquitectura del Sistema

  • Diagrama d'Arquitectura General:

  • Components Principals:

    • Frontend: Aplicació web desenvolupada en React.
    • Backend: API RESTful construïda amb Node.js i Express.
    • Base de Dades: MongoDB per a emmagatzematge de dades.

  • Interaccions:

    • El frontend es comunica amb el backend a través de trucades API.
    • El backend interactua amb la base de dades per a operacions CRUD (Create, Read, Update, Delete).

Tecnologies Utilitzades

  • Llenguatges de Programació:

    • JavaScript (Node.js, React)

  • Frameworks i Biblioteques:

    • Express per al backend.
    • Mongoose per a la interacció amb MongoDB.
    • Redux per a la gestió de l'estat en el frontend.

  • Eines de Desenvolupament i Desplegament:

    • Docker per a la contenedorizació.
    • Jenkins per a CI/CD.
    • AWS per al desplegament al núvol.

Patrons de Disseny

  • Singleton:

    class Database {
  constructor() {
    if (!Database.instance) {
      this.connection = this.createConnection();
      Database.instance = this;
    }
    return Database.instance;
  }

  createConnection() {
    // Lògica per crear una connexió a la base de dades
  }
}

const instance = new Database();
Object.freeze(instance);
export default instance;

  • Factory:

    class ProductFactory {
  static createProduct(type, attributes) {
    switch (type) {
      case 'electronic':
        return new ElectronicProduct(attributes);
      case 'furniture':
        return new FurnitureProduct(attributes);
      default:
        throw new Error('Tipus de producte no suportat');
    }
  }
}

Seguretat i Escalabilitat

  • Estratègies de Seguretat:

    • Autenticació i autorització amb JWT.
    • Encriptació de dades sensibles.
    • Validació d'entrades per prevenir injeccions SQL i XSS.

  • Plans d'Escalabilitat:

    • Ús de balancejadors de càrrega.
    • Implementació de microserveis per a components crítics.
    • Escalat horitzontal de la base de dades.

Proves i Validació

  • Mètodes de Prova:

    • Proves unitàries amb Jest.
    • Proves d'integració amb Supertest.
    • Proves de càrrega amb JMeter.

  • Resultats de les Proves:

    • 95% de cobertura de codi.
    • Totes les proves d'integració van passar.
    • El sistema suporta fins a 1000 sol·licituds per segon sense degradació del rendiment.

Conclusió

En la presentació final, els estudiants han de demostrar la seva capacitat per dissenyar i construir una arquitectura de sistema robusta i escalable. Han de ser capaços de justificar les seves decisions de disseny i mostrar com han aplicat els principis i patrons d'arquitectura apresos durant el curs. La retroalimentació rebuda durant la presentació serà crucial per al seu desenvolupament professional i els ajudarà a identificar àrees de millora per a futurs projectes.
