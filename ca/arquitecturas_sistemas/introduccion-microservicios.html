Què són els Microserveis?

Els microserveis són una arquitectura de programari que estructura una aplicació com un conjunt de serveis petits i autònoms, cadascun executant-se en el seu propi procés i comunicant-se entre si a través d'APIs ben definides. Aquesta arquitectura permet una major flexibilitat i escalabilitat en el desenvolupament i manteniment d'aplicacions complexes.

Característiques Principals

  • Descomposició en Serveis Petits: Cada microservei és responsable d'una funcionalitat específica i pot ser desenvolupat, desplegat i escalat de manera independent.
  • Comunicació a través d'APIs: Els microserveis es comuniquen entre si mitjançant APIs, generalment utilitzant protocols HTTP/HTTPS i formats de dades com JSON o XML.
  • Desplegament Independent: Cada microservei pot ser desplegat de manera independent, cosa que facilita l'actualització i el manteniment sense afectar altres serveis.
  • Escalabilitat: Els microserveis permeten escalar individualment els components de l'aplicació segons la demanda, optimitzant l'ús de recursos.
  • Tecnologies Heterogènies: Els equips poden triar diferents tecnologies i llenguatges de programació per a cada microservei, segons el que millor s'adapti a les seves necessitats.

Avantatges i Desavantatges

Avantatges

  • Flexibilitat en el Desenvolupament: Equips petits poden treballar en diferents microserveis de manera simultània.
  • Escalabilitat: Permet escalar només els serveis que ho necessiten, optimitzant recursos.
  • Resiliència: Fallades en un microservei no afecten necessàriament tota l'aplicació.
  • Desplegament Continu: Facilita la integració i el desplegament continu (CI/CD).

Desavantatges

  • Complexitat en la Gestió: Requereix una gestió més complexa de la infraestructura i la comunicació entre serveis.
  • Latència: La comunicació entre microserveis pot introduir latència.
  • Consistència de Dades: Mantenir la consistència de dades entre microserveis pot ser desafiant.
  • Sobrecàrrega Operacional: Necessita eines i pràctiques addicionals per a monitoratge, logging i maneig de fallades.

Comparació amb Arquitectura Monolítica

| Característica | Microserveis | Monolítica | |------------------------|---------------------------------------|----------------------------------------| | Desplegament | Independent per servei | Tot el sistema es desplega junt | | Escalabilitat | Escalabilitat individual per servei | Escalabilitat de tota l'aplicació | | Flexibilitat Tecnològica| Alta, diferents tecnologies per servei| Baixa, una sola tecnologia per a tota l'aplicació | | Complexitat | Alta, requereix gestió de múltiples serveis | Baixa, un sol codi base | | Resiliència | Alta, fallades aïllades | Baixa, fallades poden afectar tot el sistema |

Exemple de Microservei en Node.js

A continuació, es mostra un exemple bàsic d'un microservei en Node.js utilitzant Express per a manejar sol·licituds HTTP.

const express = require('express');
const app = express();
const port = 3000;

// Middleware per parsejar JSON
app.use(express.json());

// Endpoint per obtenir informació d'usuari
app.get('/user/:id', (req, res) => {
    const userId = req.params.id;
    // Lògica per obtenir informació de l'usuari
    res.json({ id: userId, name: 'John Doe' });
});

// Endpoint per crear un nou usuari
app.post('/user', (req, res) => {
    const newUser = req.body;
    // Lògica per crear un nou usuari
    res.status(201).json(newUser);
});

app.listen(port, () => {
    console.log(`Microservei d'usuaris escoltant a http://localhost:${port}`);
});

Explicació del Codi

  • Express: És un framework per a Node.js que facilita la creació d'aplicacions web i APIs.
  • Middleware: express.json() s'utilitza per parsejar el cos de les sol·licituds en format JSON.
  • Endpoints: Es defineixen dos endpoints, un per obtenir informació d'un usuari i un altre per crear un nou usuari.
  • Port: El microservei escolta en el port 3000.

Conclusió

L'arquitectura de microserveis ofereix una forma flexible i escalable de desenvolupar aplicacions complexes. Encara que introdueix certa complexitat en la gestió i comunicació entre serveis, els seus avantatges en termes de desplegament independent, resiliència i escalabilitat la fan una opció atractiva per a moltes organitzacions. És important avaluar acuradament les necessitats i capacitats del teu equip abans d'adoptar aquesta arquitectura.
