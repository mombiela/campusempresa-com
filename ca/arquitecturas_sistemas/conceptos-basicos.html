Introducció

L'arquitectura de sistemes és una disciplina fonamental en el camp de la tecnologia de la informació (TI) que se centra en el disseny i l'estructura de sistemes complexos. Aquest tema proporciona una base sòlida per comprendre com es construeixen i s'organitzen els sistemes de TI, abastant des dels components individuals fins a la integració de sistemes complets.

Components de l'Arquitectura de Sistemes

Maquinari

El maquinari és la base física sobre la qual s'executen els sistemes de TI. Inclou tots els components físics com servidors, ordinadors, dispositius d'emmagatzematge, i xarxes.

  • Servidors: Màquines que proporcionen serveis a altres ordinadors a la xarxa.
  • Dispositius d'emmagatzematge: Unitats de disc dur, SSDs, i sistemes d'emmagatzematge en xarxa (NAS).
  • Xarxes: Routers, switches, i altres dispositius que faciliten la comunicació entre diferents parts del sistema.

Programari

El programari és el conjunt d'instruccions i dades que fan que el maquinari funcioni. Es divideix en diverses categories:

  • Sistemes Operatius: Programari que gestiona el maquinari i proporciona serveis bàsics a altres aplicacions (e.g., Windows, Linux).
  • Aplicacions: Programes que realitzen tasques específiques per als usuaris (e.g., navegadors web, aplicacions d'oficina).
  • Middleware: Programari que connecta diferents aplicacions i permet la seva comunicació (e.g., servidors d'aplicacions, sistemes de missatgeria).

Xarxes

Les xarxes permeten la comunicació entre diferents components del sistema. Inclouen:

  • Xarxes Locals (LAN): Xarxes que connecten ordinadors dins d'una àrea limitada, com una oficina.
  • Xarxes d'Àrea Ampla (WAN): Xarxes que connecten ordinadors a través de grans distàncies, com Internet.
  • Protocols de Comunicació: Regles i estàndards que permeten la transferència de dades (e.g., TCP/IP, HTTP).

Models d'Arquitectura

Arquitectura Monolítica

En una arquitectura monolítica, tots els components del sistema estan integrats en una única aplicació. Això pot simplificar el desenvolupament inicial, però pot dificultar el manteniment i l'escalabilitat.

  • Avantatges:
    • Simplicitat inicial.
    • Facilitat de desplegament.
  • Desavantatges:
    • Dificultat per escalar.
    • Problemes de manteniment.

Arquitectura de Microserveis

L'arquitectura de microserveis divideix el sistema en serveis petits i independents que es comuniquen entre si. Cada servei és responsable d'una funcionalitat específica.

  • Avantatges:
    • Escalabilitat.
    • Mantenibilitat.
  • Desavantatges:
    • Complexitat en la gestió.
    • Necessitat d'una infraestructura robusta.

Comparació d'Arquitectures

| Característica | Monolítica | Microserveis | |------------------------|--------------------------|--------------------------| | Escalabilitat | Limitada | Alta | | Mantenibilitat | Difícil | Fàcil | | Complexitat Inicial | Baixa | Alta | | Desplegament | Senzill | Complex |

Exemple de Codi

A continuació, es mostra un exemple simple de com es podria estructurar una aplicació monolítica enfront d'una aplicació de microserveis.

Monolítica

# app.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Benvingut a l'aplicació monolítica"

@app.route('/user')
def user():
    return "Informació de l'usuari"

if __name__ == '__main__':
    app.run(debug=True)

Microserveis

Servei d'Usuari

# user_service.py
from flask import Flask

app = Flask(__name__)

@app.route('/user')
def user():
    return "Informació de l'usuari"

if __name__ == '__main__':
    app.run(debug=True, port=5001)

Servei Principal

# main_service.py
from flask import Flask
import requests

app = Flask(__name__)

@app.route('/')
def home():
    return "Benvingut a l'aplicació de microserveis"

@app.route('/user')
def user():
    response = requests.get('http://localhost:5001/user')
    return response.text

if __name__ == '__main__':
    app.run(debug=True, port=5000)

Conclusió

Comprendre els conceptes bàsics de l'arquitectura de sistemes és crucial per a qualsevol professional de TI. Des del maquinari i el programari fins a les xarxes i els models d'arquitectura, cada component juga un paper vital en la construcció de sistemes eficients i escalables. L'elecció entre una arquitectura monolítica i una de microserveis dependrà de les necessitats específiques del projecte i dels recursos disponibles.
