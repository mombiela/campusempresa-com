Introducció

La gestió d'errors i la depuració són habilitats crucials per a qualsevol programador. En REXX, entendre com gestionar errors i depurar el teu codi pot millorar significativament la teva eficiència i la fiabilitat dels teus programes. Aquesta secció cobrirà els conceptes bàsics de la gestió d'errors, tècniques de depuració i estratègies avançades per gestionar i solucionar problemes en els teus programes REXX.

Conceptes Bàsics de la Gestió d'Errors

Tipus d'Errors

  • Errors de Sintaxi: Es produeixen quan el codi viola les regles del llenguatge REXX.
  • Errors d'Execució: Succeeixen durant l'execució del programa, com ara divisió per zero o fitxer no trobat.
  • Errors Lògics: Són errors en la lògica del programa que produeixen resultats incorrectes.

Captura d'Errors

REXX proporciona mecanismes per capturar i gestionar errors de manera elegant.

Exemple: Ús de SIGNAL ON ERROR

/* Exemple de captura d'errors en REXX */
signal on error
say "Introdueix un número:"
pull number
result = 10 / number
say "El resultat és:" result
exit

error:
say "S'ha produït un error. Si us plau, introdueix un número diferent de zero."
exit
  • Explicació: La instrucció SIGNAL ON ERROR indica a REXX que salti a l'etiqueta error: si es produeix un error. Això permet que el programa gestioni l'error de manera elegant en lloc de finalitzar abruptament.

Variables d'Error Incorporades

  • RC: Codi de retorn de l'última comanda executada.
  • SIGL: Número de línia on s'ha produït l'error.
  • SYNTAX: Conté informació sobre l'últim error de sintaxi.

Exemple: Ús de RC i SIGL

/* Exemple d'ús de RC i SIGL */
signal on error
say "Introdueix un número:"
pull number
result = 10 / number
say "El resultat és:" result
exit

error:
say "S'ha produït un error a la línia" sigl "amb el codi de retorn" rc
exit
  • Explicació: RC i SIGL proporcionen informació addicional sobre l'error, que pot ser útil per a la depuració.

Tècniques de Depuració

Ús de TRACE

La instrucció TRACE és una eina poderosa per a la depuració de programes REXX.

Exemple: Ús de TRACE

/* Exemple d'ús de TRACE */
trace ?r
say "Introdueix un número:"
pull number
result = 10 / number
say "El resultat és:" result
  • Explicació: TRACE ?R activa la depuració interactiva, mostrant cada línia de codi a mesura que s'executa i els resultats de les expressions.

Depuració Interactiva

  • TRACE I: Mode de depuració interactiva.
  • TRACE R: Mostra cada línia de codi i el resultat de les expressions.
  • TRACE O: Mostra només els resultats de les expressions.

Consells de Depuració

  • Aïllar el Problema: Divideix el teu codi en seccions més petites per aïllar on es produeix l'error.
  • Comprovar Valors de Variables: Utilitza instruccions say o TRACE per comprovar els valors de les variables en diferents punts del teu programa.
  • Revisar la Lògica: Assegura't que la lògica del teu programa sigui sòlida i que es gestionin tots els casos possibles.

Gestió d'Errors Avançada

Gestió Personalitzada d'Errors

Pots crear rutines personalitzades de gestió d'errors per gestionar tipus específics d'errors.

Exemple: Gestió Personalitzada d'Errors

/* Exemple de gestió personalitzada d'errors */
signal on error name myErrorHandler
say "Introdueix un número:"
pull number
result = 10 / number
say "El resultat és:" result
exit

myErrorHandler:
say "Gestor d'Errors Personalitzat: S'ha produït un error a la línia" sigl "amb el codi de retorn" rc
exit
  • Explicació: La paraula clau name et permet especificar una rutina personalitzada de gestió d'errors.

Ús de `CALL ON`

La instrucció CALL ON proporciona una altra manera de gestionar errors.

Exemple: Ús de CALL ON

/* Exemple d'ús de CALL ON */
call on error name myErrorHandler
say "Introdueix un número:"
pull number
result = 10 / number
say "El resultat és:" result
exit

myErrorHandler:
say "Gestor d'Errors CALL ON: S'ha produït un error a la línia" sigl "amb el codi de retorn" rc
exit
  • Explicació: CALL ON funciona de manera similar a SIGNAL ON però utilitza una subrutina per a la gestió d'errors.

Conclusió

La gestió d'errors i la depuració són habilitats essencials per escriure programes REXX robustos i fiables. Entenent els tipus d'errors, utilitzant mecanismes de captura d'errors i emprant tècniques efectives de depuració, pots millorar significativament la teva eficiència en la programació. Practica aquests conceptes amb els exemples proporcionats i incorpora'ls en el teu flux de treball de programació REXX per gestionar errors de manera elegant i depurar el teu codi de manera efectiva.
