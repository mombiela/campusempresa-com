Introducció a REXX

  • Què és REXX?

    • REXX (Restructured Extended Executor) és un llenguatge de programació d'alt nivell dissenyat per a ser fàcil d'aprendre i llegir.
    • És àmpliament utilitzat per a scripting i automatització en el sistema operatiu mainframe z/OS d'IBM.

  • Per què utilitzar REXX a z/OS?

    • Simplifica tasques complexes amb menys línies de codi.
    • S'integra perfectament amb altres utilitats i programes de z/OS.
    • Ideal tant per a principiants com per a programadors experimentats gràcies a la seva sintaxi senzilla.

Començar amb REXX a z/OS

  • Configurar l'entorn

    • Assegura't de tenir accés a un sistema z/OS.
    • Familiaritza't amb TSO (Time Sharing Option) i ISPF (Interactive System Productivity Facility).

  • Escriure el teu primer script REXX

    • Crea un nou membre en un PDS (Partitioned Data Set) amb una extensió .REXX.
    • Exemple: 
      /* Hello World en REXX */
SAY 'Hello, World!'
    • Desa i executa l'script utilitzant la comanda EXEC en TSO.

Sintaxi i comandes bàsiques de REXX

  • Comentaris

    • Comentaris d'una sola línia: /* Aquest és un comentari */

  • Variables

    • No cal declarar les variables explícitament.
    • Exemple: 
      name = 'John Doe'
SAY 'Hello, ' name

  • Operacions bàsiques

    • Aritmètica: +, -, *, /
    • Exemple: 
      a = 5
b = 3
c = a + b
SAY 'La suma de' a 'i' b 'és' c

  • Estructures de control

    • IF-THEN-ELSE 
      age = 18
IF age >= 18 THEN
  SAY 'Ets un adult.'
ELSE
  SAY 'Ets un menor.'
    • Bucles DO 
      DO i = 1 TO 5
  SAY 'Iteració' i
END

Funcions avançades de REXX

  • Funcions integrades

    • Manipulació de cadenes 
      str = 'Hello, World!'
SAY LENGTH(str)  /* Mostra: 13 */
SAY SUBSTR(str, 1, 5)  /* Mostra: Hello */

  • Operacions d'entrada/sortida de fitxers

    • Llegir d'un fitxer: 
      /* Llegir un fitxer línia per línia */
"ALLOCATE FILE(INPUT) DA('your.dataset.name') SHR"
DO WHILE LINES('INPUT') > 0
  LINEIN('INPUT')
  SAY LINE
END
"FREE FILE(INPUT)"
    • Escriure a un fitxer: 
      /* Escriure a un fitxer */
"ALLOCATE FILE(OUTPUT) DA('your.dataset.name') SHR"
CALL LINEOUT 'OUTPUT', 'Aquesta és una línia de text.'
"FREE FILE(OUTPUT)"

  • Interfície amb utilitats de z/OS

    • Comandes TSO

      ADDRESS TSO "LISTCAT ENTRIES('your.dataset.name')"

    • Panells ISPF

      ADDRESS ISPEXEC "DISPLAY PANEL(your.panel.name)"

Gestió d'errors i depuració

  • Captura d'errors

    • Exemple: 
      SIGNAL ON ERROR
/* Codi que podria causar un error */
ERROR:
SAY 'S'ha produït un error.'

  • Tècniques de depuració

    • Utilitza TRACE per seguir el flux d'execució. 
      TRACE R

Millors pràctiques per a la programació en REXX

  • Llegibilitat del codi

    • Utilitza noms de variables significatius.
    • Comenta el teu codi extensament.

  • Programació modular

    • Divideix scripts grans en procediments més petits i reutilitzables.

  • Optimització del rendiment

    • Minimitza l'ús de comandes externes i operacions d'entrada/sortida.

Conclusió

  • Resum

    • REXX és un llenguatge de scripting potent a z/OS, oferint simplicitat i integració amb utilitats de mainframe.
    • Des de la sintaxi bàsica fins a funcions avançades, REXX pot gestionar una varietat de tasques de manera eficient.

  • Pròxims passos

    • Practica escrivint i executant scripts REXX.
