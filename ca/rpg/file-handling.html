Introducció

En aquesta secció, explorarem la gestió de fitxers i les estructures de dades en RPG (Report Program Generator). Entendre aquests conceptes és crucial per gestionar les dades de manera eficient i escriure programes robustos. Començarem amb els conceptes bàsics de la gestió de fitxers i gradualment passarem a estructures de dades més avançades.

Gestió de Fitxers

Conceptes Bàsics de la Gestió de Fitxers

La gestió de fitxers en RPG implica llegir i escriure fitxers. RPG suporta diversos tipus de fitxers, incloent fitxers físics, fitxers lògics i fitxers de pantalla.

  • Fitxers Físics: Aquests són els fitxers reals que emmagatzemen dades.
  • Fitxers Lògics: Aquests proporcionen una vista o ordre diferent de les dades emmagatzemades en fitxers físics.
  • Fitxers de Pantalla: Utilitzats per crear interfícies d'usuari.

Llegir d'un Fitxer

Per llegir dades d'un fitxer, cal definir el fitxer en l'Especificació de Fitxers (F-spec) i utilitzar l'operació READ.

FMYFILE    IF   E           K DISK

DName            S             50A

C     READ      MYFILE
C     IF        %EOF(MYFILE)
C     ELSE
C       EVAL    Name = MYFILE.Name
C     ENDIF
C     *INLR     = *ON
  • F-spec: Defineix el fitxer i les seves atributs.
  • READ: Llegeix un registre del fitxer.
  • %EOF: Comprova si s'ha arribat al final del fitxer.

Escriure a un Fitxer

Escriure dades a un fitxer implica utilitzar l'operació WRITE.

FMYFILE    O   E           K DISK

DName            S             50A

C     EVAL      MYFILE.Name = 'John Doe'
C     WRITE     MYFILE
C     *INLR     = *ON
  • WRITE: Escriu un registre al fitxer.

Actualitzar un Fitxer

Per actualitzar un registre en un fitxer, utilitzeu les operacions CHAIN i UPDATE.

FMYFILE    UF   E           K DISK

DName            S             50A

C     CHAIN     'John Doe' MYFILE
C     IF        %FOUND(MYFILE)
C       EVAL    MYFILE.Name = 'Jane Doe'
C       UPDATE  MYFILE
C     ENDIF
C     *INLR     = *ON
  • CHAIN: Recupera un registre per clau.
  • UPDATE: Actualitza el registre recuperat.

Estructures de Dades

Introducció a les Estructures de Dades

Les estructures de dades en RPG s'utilitzen per agrupar dades relacionades. Poden ser simples o complexes, depenent dels requisits.

Definició d'Estructures de Dades

Una estructura de dades es defineix utilitzant l'especificació D (D-spec).

DNameDS          DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0
  • DS: Indica una estructura de dades.
  • A: Tipus de dada alfanumèrica.
  • P: Tipus de dada decimal empaquetada.

Ús d'Estructures de Dades

Les estructures de dades es poden utilitzar per simplificar la manipulació de dades.

DNameDS          DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0

C     EVAL      NameDS.FirstName = 'John'
C     EVAL      NameDS.LastName = 'Doe'
C     EVAL      NameDS.Age = 30

C     DSPLY     NameDS.FirstName + ' ' + NameDS.LastName
C     *INLR     = *ON

Estructures de Dades Imbricades

Les estructures de dades es poden imbricar per representar dades més complexes.

DAddressDS       DS
D  Street                     50A
D  City                       25A
D  ZipCode                    10A

DPersonDS        DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0
D  Address                     LIKE(AddressDS)
  • LIKE: Utilitzat per definir un subcamp amb els mateixos atributs que un altre camp.

Arrays en Estructures de Dades

Els arrays es poden utilitzar dins d'estructures de dades per gestionar múltiples valors.

DGradesDS        DS
D  StudentName                25A
D  Grades                     5P 0 DIM(5)

C     EVAL      GradesDS.StudentName = 'John Doe'
C     EVAL      GradesDS.Grades(1) = 85
C     EVAL      GradesDS.Grades(2) = 90
C     EVAL      GradesDS.Grades(3) = 88

C     DSPLY     GradesDS.StudentName
C     DSPLY     GradesDS.Grades(1)
C     *INLR     = *ON
