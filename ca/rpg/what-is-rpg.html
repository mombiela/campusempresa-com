Introducció a RPG

RPG, que significa Report Program Generator, és un llenguatge de programació d'alt nivell utilitzat principalment per a aplicacions empresarials. Va ser desenvolupat originalment per IBM als anys 60 i ha evolucionat significativament al llarg dels anys. RPG és conegut per la seva simplicitat i eficiència en la gestió de la lògica empresarial i les tasques de processament de dades.

Conceptes Clau de RPG

  • Llenguatge d'Alt Nivell: RPG està dissenyat per ser fàcil de llegir i escriure, fent-lo accessible per als programadors empresarials.
  • Aplicacions Empresarials: És àmpliament utilitzat en indústries com les finances, la manufactura i el comerç al detall per crear informes, gestionar dades i automatitzar processos empresarials.
  • Sistemes IBM i: RPG s'utilitza predominantment en sistemes IBM i (anteriorment AS/400), coneguts per la seva robustesa i fiabilitat.

Evolució de RPG

RPG ha passat per diverses versions, cadascuna afegint noves característiques i capacitats:

  • RPG I: La versió original, introduïda als anys 60.
  • RPG II: Introduïda als anys 70, va afegir més funcionalitat i va millorar el rendiment.
  • RPG III: Llançada als anys 80, va portar conceptes de programació estructurada.
  • RPG IV (RPGLE): La versió moderna, introduïda als anys 90, suporta la codificació en format lliure i s'integra amb entorns de desenvolupament moderns.

Estructura Bàsica d'un Programa RPG

Un programa RPG típicament consta de les següents seccions:

  1. H-Specs (Especificacions de Capçalera): Defineixen configuracions a nivell de programa.
  2. F-Specs (Especificacions de Fitxer): Defineixen els fitxers utilitzats en el programa.
  3. D-Specs (Especificacions de Dades): Defineixen variables i estructures de dades.
  4. C-Specs (Especificacions de Càlcul): Contenen la lògica i els càlculs.
  5. O-Specs (Especificacions de Sortida): Defineixen el format de sortida.

Exemple d'un Programa RPG Simple

Aquí teniu un exemple bàsic d'un programa RPG IV (RPGLE):

**FREE
// H-Specs
H DFTACTGRP(*NO) ACTGRP(*NEW)

// F-Specs
FMYFILE   IF   E           K DISK

// D-Specs
D Name            S             50A

// C-Specs
C     *ENTRY        PLIST
C                   PARM                    Name

C                   IF        Name = 'John'
C                   EVAL      *INLR = *ON
C                   RETURN
C                   ENDIF

C                   EVAL      *INLR = *ON
C                   RETURN

Explicació del Codi

  • H-Specs: DFTACTGRP(*NO) i ACTGRP(*NEW) especifiquen que el programa no utilitza el grup d'activació per defecte i crea un nou grup d'activació.
  • F-Specs: La línia FMYFILE defineix un fitxer anomenat MYFILE que el programa utilitzarà.
  • D-Specs: La línia D Name defineix una variable Name de tipus caràcter amb una longitud de 50.
  • C-Specs: Les especificacions de càlcul contenen la lògica del programa. En aquest exemple, comprova si el Name és 'John' i estableix l'indicador de l'últim registre (*INLR) a *ON, cosa que finalitza el programa.

Conclusió

RPG és un llenguatge potent i eficient per a aplicacions empresarials, especialment en sistemes IBM i. Entendre la seva estructura i conceptes clau és essencial per aprofitar les seves capacitats. A mesura que avancis en aquest curs, aprendràs característiques i tècniques més avançades per convertir-te en un programador proficient en RPG.
