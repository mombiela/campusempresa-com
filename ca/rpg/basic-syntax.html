Introducció

RPG (Report Program Generator) és un llenguatge de programació d'alt nivell utilitzat principalment per a aplicacions empresarials. Entendre la sintaxi i l'estructura bàsica de l'RPG és crucial per escriure programes eficients i efectius. Aquesta secció cobrirà els elements fonamentals de la sintaxi i l'estructura de l'RPG, proporcionant una base sòlida per a un aprenentatge posterior.

Conceptes Clau

  • Especificacions: Els programes RPG es divideixen en diferents tipus d'especificacions, cadascun amb una finalitat específica.

    • H-Spec: Especificació de capçalera, utilitzada per a configuracions globals.
    • F-Spec: Especificació de fitxer, utilitzada per definir fitxers.
    • D-Spec: Especificació de definició, utilitzada per definir variables, constants i estructures de dades.
    • C-Spec: Especificació de càlcul, utilitzada per a codi executable.
    • P-Spec: Especificació de procediment, utilitzada per definir subrutines i procediments.

  • Format Fix vs. Format Lliure: RPG IV va introduir la codificació en format lliure, que és més flexible i llegible en comparació amb el format fix tradicional.

Especificacions en Detall

H-Spec (Especificació de Capçalera)

L'H-spec s'utilitza per establir paràmetres globals per al programa.

H dftactgrp(*no) actgrp('MYACTGRP')
  • dftactgrp(*no): Especifica que el programa no s'executa en el grup d'activació per defecte.
  • actgrp('MYACTGRP'): Defineix el grup d'activació per al programa.

F-Spec (Especificació de Fitxer)

L'F-spec s'utilitza per definir els fitxers que utilitzarà el programa.

FMYFILE   IF   E           K DISK
  • FMYFILE: El nom del fitxer.
  • I: Fitxer d'entrada.
  • F: Fitxer procedimental complet.
  • E: Fitxer extern.
  • K: Fitxer amb clau.
  • DISK: Tipus de fitxer.

D-Spec (Especificació de Definició)

La D-spec s'utilitza per definir variables, constants i estructures de dades.

D CustomerName   S             50A
D OrderTotal     S              9P 2
  • CustomerName: Una variable de tipus caràcter amb una longitud de 50.
  • OrderTotal: Una variable decimal empaquetada amb 9 dígits i 2 decimals.

C-Spec (Especificació de Càlcul)

La C-spec és on resideix el codi executable.

C     Eval      OrderTotal = OrderTotal + 100.00
C     If        OrderTotal > 500.00
C     Eval      Discount = OrderTotal * 0.1
C     EndIf
  • Eval: Utilitzat per assignar valors a variables.
  • If i EndIf: Utilitzats per a la lògica condicional.

P-Spec (Especificació de Procediment)

La P-spec s'utilitza per definir subrutines i procediments.

P MyProcedure   B
D MyProcedure   PI
D InputValue    10I 0
D OutputValue   10I 0

C     Eval      OutputValue = InputValue * 2
C     Return
P MyProcedure   E
  • B i E: Inici i final del procediment.
  • PI: Interfície de Procediment.
  • Eval: Càlcul dins del procediment.
  • Return: Final de la lògica del procediment.

Format Fix vs. Format Lliure

Exemple de Format Fix

FMYFILE   IF   E           K DISK
D CustomerName   S             50A
C     Eval      CustomerName = 'John Doe'
C     If        CustomerName = 'John Doe'
C     Eval      Greeting = 'Hello, John!'
C     EndIf

Exemple de Format Lliure

dcl-f MYFILE DISK(*EXT) USAGE(*INPUT);

dcl-s CustomerName char(50);
dcl-s Greeting char(50);

CustomerName = 'John Doe';

if CustomerName = 'John Doe';
  Greeting = 'Hello, John!';
endif;
  • Format Lliure: Més llegible i flexible, permetent pràctiques de codificació modernes.

Conclusió

Entendre la sintaxi i l'estructura bàsica de l'RPG és essencial per escriure programes efectius. En dominar les diferents especificacions i les diferències entre el format fix i el format lliure, estaràs ben preparat per abordar tasques de programació més complexes en RPG. Aquest coneixement fonamental servirà com a base per a temes més avançats en la programació RPG.
