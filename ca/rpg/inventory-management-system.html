Introducció

En aquest estudi de cas, explorarem el desenvolupament d'un Sistema de Gestió d'Inventari utilitzant RPG (Report Program Generator). Aquest sistema ajudarà a gestionar els nivells d'inventari, fer el seguiment de l'estoc i generar informes. Cobrirà tot el procés des de conceptes bàsics fins a funcions avançades, assegurant que tant els principiants com els programadors avançats puguin seguir-lo.

Conceptes Clau

  • Gestió d'Inventari: El procés de demanar, emmagatzemar i utilitzar l'inventari d'una empresa.
  • Programació RPG: Un llenguatge de programació d'alt nivell utilitzat per a aplicacions empresarials.
  • Interacció amb Bases de Dades: Com interactuar amb bases de dades per emmagatzemar i recuperar dades d'inventari.
  • Interfície d'Usuari: Crear interfícies fàcils d'usar per interactuar amb el sistema.

Configuració de l'Entorn

Abans de començar a programar, assegura't de tenir el següent:

  • Un sistema IBM i o un compilador RPG.
  • Accés a una base de dades (per exemple, DB2).
  • Comprensió bàsica de la sintaxi i estructura de RPG.

Sistema Bàsic de Gestió d'Inventari

Disseny de la Base de Dades

Primer, necessitem dissenyar les taules de la base de dades per al nostre sistema d'inventari. Crearem una taula anomenada INVENTORY amb els següents camps:

  • ITEM_ID (Clau Primària)
  • ITEM_NAME
  • QUANTITY
  • PRICE
CREATE TABLE INVENTORY (
    ITEM_ID INT PRIMARY KEY,
    ITEM_NAME VARCHAR(100),
    QUANTITY INT,
    PRICE DECIMAL(10, 2)
);

Programa RPG Bàsic

A continuació, escriurem un programa RPG senzill per afegir articles a l'inventari.

H DFTACTGRP(*NO) ACTGRP(*NEW)

D ITEM_ID       S              10I 0
D ITEM_NAME     S             100A
D QUANTITY      S              10I 0
D PRICE         S              10P 2

C     *ENTRY    PLIST
C               PARM                    ITEM_ID
C               PARM                    ITEM_NAME
C               PARM                    QUANTITY
C               PARM                    PRICE

C/EXEC SQL
C+   INSERT INTO INVENTORY (ITEM_ID, ITEM_NAME, QUANTITY, PRICE)
C+   VALUES (:ITEM_ID, :ITEM_NAME, :QUANTITY, :PRICE)
C/END-EXEC

C     *INLR     SETON

Explicació

  • H DFTACTGRP(*NO) ACTGRP(*NEW): Especifica el grup d'activació per al programa.
  • D: Declara variables.
  • *C ENTRY PLIST: Defineix la llista de paràmetres per al programa.
  • C/EXEC SQL: Comença una instrucció SQL incrustada.
  • C+ INSERT INTO INVENTORY: Comanda SQL per inserir un nou registre a la taula INVENTORY.
  • C/END-EXEC: Finalitza la instrucció SQL incrustada.
  • *C INLR SETON: Activa l'indicador de l'últim registre, finalitzant el programa.

Funcions Intermèdies

Actualització de l'Inventari

Ara afegirem funcionalitat per actualitzar la quantitat d'un article existent.

H DFTACTGRP(*NO) ACTGRP(*NEW)

D ITEM_ID       S              10I 0
D NEW_QUANTITY  S              10I 0

C     *ENTRY    PLIST
C               PARM                    ITEM_ID
C               PARM                    NEW_QUANTITY

C/EXEC SQL
C+   UPDATE INVENTORY
C+   SET QUANTITY = :NEW_QUANTITY
C+   WHERE ITEM_ID = :ITEM_ID
C/END-EXEC

C     *INLR     SETON

Explicació

  • C+ UPDATE INVENTORY: Comanda SQL per actualitzar el camp QUANTITY d'un registre existent a la taula INVENTORY.

Funcions Avançades

Generació d'Informes

Crearem un programa per generar un informe de tots els articles de l'inventari.

H DFTACTGRP(*NO) ACTGRP(*NEW)

D ITEM_ID       S              10I 0
D ITEM_NAME     S             100A
D QUANTITY      S              10I 0
D PRICE         S              10P 2

C/EXEC SQL
C+   DECLARE C1 CURSOR FOR
C+   SELECT ITEM_ID, ITEM_NAME, QUANTITY, PRICE
C+   FROM INVENTORY
C/END-EXEC

C/EXEC SQL
C+   OPEN C1
C/END-EXEC

C/EXEC SQL
C+   FETCH C1 INTO :ITEM_ID, :ITEM_NAME, :QUANTITY, :PRICE
C/END-EXEC

C     DOU       SQLCOD <> 0
C     ITEM_ID   DSPLY
C     ITEM_NAME DSPLY
C     QUANTITY  DSPLY
C     PRICE     DSPLY

C/EXEC SQL
C+   FETCH C1 INTO :ITEM_ID, :ITEM_NAME, :QUANTITY, :PRICE
C/END-EXEC
C     ENDDO

C/EXEC SQL
C+   CLOSE C1
C/END-EXEC

C     *INLR     SETON

Explicació

  • DECLARE C1 CURSOR FOR: Declara un cursor per seleccionar tots els registres de la taula INVENTORY.
  • OPEN C1: Obre el cursor.
  • FETCH C1 INTO: Recupera la següent fila del cursor en les variables especificades.
  • DOU SQLCOD <> 0: Bucle fins que el codi SQL no sigui zero (indicant que no hi ha més files).
  • DSPLY: Mostra els valors de les variables.

Conclusió

En aquest estudi de cas, hem construït un sistema bàsic de Gestió d'Inventari utilitzant RPG. Hem començat configurant l'entorn, creant una base de dades i escrivint programes bàsics per afegir i actualitzar articles d'inventari. Després hem passat a funcions més avançades com la generació d'informes. Aquest enfocament integral assegura que tinguis una comprensió sòlida tant dels conceptes bàsics com dels avançats de la programació RPG aplicats a un escenari del món real.
