Introducció

La gestió d'errors i la depuració són habilitats crucials per a qualsevol programador. En RPG (Report Program Generator), entendre com gestionar els errors i depurar el teu codi de manera efectiva pot estalviar-te temps i assegurar que els teus programes funcionin sense problemes. Aquesta secció cobrirà els conceptes bàsics de la gestió d'errors, tècniques comunes de depuració i estratègies avançades per gestionar errors en RPG.

Conceptes Bàsics de la Gestió d'Errors

La gestió d'errors en RPG implica anticipar possibles problemes i escriure codi per gestionar aquestes situacions de manera elegant.

Conceptes Clau

  • Codis d'Error: RPG utilitza codis específics per indicar diferents tipus d'errors.
  • Indicadors: Variables especials que es poden establir per indicar la presència d'un error.
  • Funcions Incorporades: RPG proporciona funcions incorporades per gestionar errors, com ara MONITOR, ON-ERROR i ENDMON.

Exemple: Gestió Bàsica d'Errors

Dcl-S Num1 Int(10);
Dcl-S Num2 Int(10);
Dcl-S Result Int(10);

Num1 = 10;
Num2 = 0;

Monitor;
  Result = Num1 / Num2;
On-Error;
  Dsply 'Error de divisió per zero!';
EndMon;

En aquest exemple:

  • Monitor inicia el bloc de gestió d'errors.
  • On-Error especifica què fer si es produeix un error.
  • EndMon finalitza el bloc de gestió d'errors.

Tècniques de Depuració

La depuració és el procés d'identificar i corregir errors en el teu codi. RPG ofereix diverses eines i tècniques per ajudar amb això.

Conceptes Clau

  • Eines de Depuració: Eines com STRDBG (Start Debug) es poden utilitzar per avançar pas a pas pel teu codi.
  • Punts de Ruptura: Punts en el teu codi on l'execució es pausarà, permetent-te inspeccionar variables i el flux del programa.
  • Traça: Un mètode per registrar l'execució del teu programa per a una anàlisi posterior.

Exemple: Ús de STRDBG

Dcl-S Num1 Int(10);
Dcl-S Num2 Int(10);
Dcl-S Result Int(10);

Num1 = 10;
Num2 = 5;

Result = Num1 / Num2;

Dsply Result;

Per depurar aquest programa:

  1. Utilitza la comanda STRDBG seguida del nom del programa.
  2. Estableix punts de ruptura teclejant F6 a les línies desitjades.
  3. Executa el programa i utilitza F10 per avançar pas a pas pel codi.

Gestió Avançada d'Errors

La gestió avançada d'errors implica tècniques més sofisticades per gestionar errors i assegurar que el teu programa es pugui recuperar de manera elegant.

Conceptes Clau

  • Missatges d'Error Personalitzats: Crear missatges d'error fàcils d'entendre per l'usuari.
  • Registre: Mantenir un registre dels errors per a una anàlisi posterior.
  • Lògica de Reintents: Implementar lògica per reintentar operacions que fallin.

Exemple: Missatges d'Error Personalitzats i Registre

Dcl-S Num1 Int(10);
Dcl-S Num2 Int(10);
Dcl-S Result Int(10);
Dcl-S ErrorMsg Char(50);

Num1 = 10;
Num2 = 0;

Monitor;
  Result = Num1 / Num2;
On-Error;
  ErrorMsg = 'Error: Divisió per zero';
  CallP LogError(ErrorMsg);
EndMon;

Dcl-Proc LogError;
  Dcl-Pi LogError ExtPgm('LOGERR');
    ErrorMsg Char(50);
  End-Pi;

  // Codi per registrar el missatge d'error
  Dsply ErrorMsg;
End-Proc;

En aquest exemple:

  • LogError és un procediment que registra el missatge d'error.
  • ErrorMsg és un missatge d'error personalitzat que proporciona més context.

Conclusió

La gestió d'errors i la depuració són habilitats essencials per als programadors d'RPG. Entenent els conceptes bàsics de la gestió d'errors, utilitzant les eines de depuració de manera efectiva i implementant estratègies avançades de gestió d'errors, pots crear programes robustos i fiables. Practica aquestes tècniques regularment per esdevenir expert en la gestió d'errors i la depuració del teu codi RPG.
