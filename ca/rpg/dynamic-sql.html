L'SQL dinàmic és una característica poderosa en RPG que permet construir i executar sentències SQL dinàmicament en temps d'execució. Això és particularment útil quan la sentència SQL exacta no es pot determinar fins que el programa està en execució.

Conceptes clau

  • SQL estàtic: Les sentències SQL estan codificades en el programa i no canvien.
  • SQL dinàmic: Les sentències SQL es construeixen i s'executen en temps d'execució, permetent més flexibilitat.

Avantatges de l'SQL dinàmic

  • Flexibilitat: Permet la construcció de consultes SQL basades en l'entrada de l'usuari o altres condicions en temps d'execució.
  • Adaptabilitat: Pot gestionar requisits de consulta variables sense canviar el codi font.

Desavantatges de l'SQL dinàmic

  • Complexitat: Més complex d'escriure i depurar en comparació amb l'SQL estàtic.
  • Rendiment: Pot ser més lent a causa de la sobrecàrrega de construir i analitzar les sentències SQL en temps d'execució.
  • Seguretat: Major risc d'atacs d'injecció SQL si no es maneja correctament.

Construcció d'SQL dinàmic en RPG

Exemple bàsic

Aquí hi ha un exemple senzill de construcció i execució d'una sentència SQL dinàmica en RPG:

Dcl-S sqlStmt Varchar(500);
Dcl-S custId Char(10);
Dcl-S custName Char(50);

custId = 'CUST001';

sqlStmt = 'SELECT CUSTNAME FROM CUSTOMERS WHERE CUSTID = ''' + custId + '''';

Exec SQL
  Prepare stmt1 From :sqlStmt;

Exec SQL
  Declare c1 Cursor For stmt1;

Exec SQL
  Open c1;

Exec SQL
  Fetch c1 Into :custName;

Exec SQL
  Close c1;

Dsply custName;

Explicació

  1. Declarar variables: Declarar les variables necessàries per a la sentència SQL i la recuperació de dades.
  2. Construir la sentència SQL: Construir la sentència SQL dinàmicament utilitzant la concatenació de cadenes.
  3. Preparar la sentència: Utilitzar Prepare per compilar la sentència SQL.
  4. Declarar el cursor: Declarar un cursor per a la sentència preparada.
  5. Obrir el cursor: Obrir el cursor per executar la sentència SQL.
  6. Recuperar dades: Recuperar dades del cursor en variables RPG.
  7. Tancar el cursor: Tancar el cursor després de recuperar les dades.

Maneig de l'entrada de l'usuari

Quan es tracta de l'entrada de l'usuari, és crucial prevenir la injecció SQL. Utilitza marcadors de paràmetres per incloure de manera segura l'entrada de l'usuari en les sentències SQL.

Dcl-S sqlStmt Varchar(500);
Dcl-S custId Char(10);
Dcl-S custName Char(50);

custId = 'CUST001';

sqlStmt = 'SELECT CUSTNAME FROM CUSTOMERS WHERE CUSTID = ?';

Exec SQL
  Prepare stmt1 From :sqlStmt;

Exec SQL
  Declare c1 Cursor For stmt1;

Exec SQL
  Open c1 Using :custId;

Exec SQL
  Fetch c1 Into :custName;

Exec SQL
  Close c1;

Dsply custName;

Explicació

  1. Marcador de paràmetres: Utilitza ? com a marcador de posició per a la variable en la sentència SQL.
  2. Obrir el cursor amb paràmetres: Utilitza Using per passar el valor de la variable de manera segura a la sentència SQL.

SQL dinàmic avançat

Utilització d'SQL dinàmic amb múltiples paràmetres

Dcl-S sqlStmt Varchar(500);
Dcl-S custId Char(10);
Dcl-S orderDate Date;
Dcl-S orderAmount Packed(7:2);

custId = 'CUST001';
orderDate = %Date('2023-01-01');

sqlStmt = 'SELECT ORDERAMOUNT FROM ORDERS WHERE CUSTID = ? AND ORDERDATE = ?';

Exec SQL
  Prepare stmt1 From :sqlStmt;

Exec SQL
  Declare c1 Cursor For stmt1;

Exec SQL
  Open c1 Using :custId, :orderDate;

Exec SQL
  Fetch c1 Into :orderAmount;

Exec SQL
  Close c1;

Dsply %Char(orderAmount);

Explicació

  1. Múltiples paràmetres: Utilitza múltiples marcadors ? per a diferents variables.
  2. Passar múltiples variables: Utilitza Using per passar múltiples variables a la sentència SQL.

Conclusió

L'SQL dinàmic en RPG proporciona la flexibilitat per construir i executar sentències SQL en temps d'execució, fent-lo una eina poderosa per gestionar requisits de consulta complexos i variables. No obstant això, comporta desafiaments com una major complexitat, possibles problemes de rendiment i riscos de seguretat. Un maneig adequat de l'entrada de l'usuari i la comprensió dels compromisos són essencials per utilitzar efectivament l'SQL dinàmic en els teus programes RPG.
