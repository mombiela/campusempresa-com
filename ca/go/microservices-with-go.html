Introducció als Microserveis

  • Definició: Els microserveis són un estil arquitectònic que estructura una aplicació com una col·lecció de serveis poc acoblats.
  • Beneficis:
    • Escalabilitat
    • Flexibilitat
    • Desplegament independent
    • Aïllament de fallades
  • Reptes:
    • Complexitat en la gestió
    • Latència de xarxa
    • Consistència de dades

Per què Go per als Microserveis?

  • Rendiment: Go és un llenguatge compilat, oferint alt rendiment.
  • Concurrència: Suport integrat per a la programació concurrent amb goroutines.
  • Simplicitat: Fàcil d'aprendre i escriure, amb una sintaxi neta.
  • Biblioteca Estàndard: Rica biblioteca estàndard que suporta la construcció de servidors web i la gestió de sol·licituds HTTP.

Configuració del Teu Entorn Go

  • Instal·lar Go: Segueix la guia oficial d'instal·lació de Go.
  • Configuració de l'Espai de Treball: 
    mkdir -p ~/go/src/microservices
cd ~/go/src/microservices

Construint el Teu Primer Microservei

  • Estructura del Projecte: 
    microservices/
├── main.go
├── handlers/
│   └── hello.go
└── go.mod
  • main.go: 
    package main

import (
    "log"
    "net/http"
    "microservices/handlers"
)

func main() {
    http.HandleFunc("/hello", handlers.HelloHandler)
    log.Fatal(http.ListenAndServe(":8080", nil))
}
  • handlers/hello.go: 
    package handlers

import (
    "fmt"
    "net/http"
)

func HelloHandler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "Hola, Món!")
}
  • go.mod: 
    module microservices

go 1.17

Gestió de Sol·licituds HTTP

  • Enrutament: Utilitza el paquet net/http per gestionar les rutes.
  • Exemple: 
    http.HandleFunc("/hello", handlers.HelloHandler)

Gestió de Dades

  • Utilitzant JSON: 
    import (
    "encoding/json"
    "net/http"
)

type Response struct {
    Message string `json:"message"`
}

func HelloHandler(w http.ResponseWriter, r *http.Request) {
    response := Response{Message: "Hola, Món!"}
    json.NewEncoder(w).Encode(response)
}

Concurrència amb Goroutines

  • Goroutines: Fils lleugers gestionats pel runtime de Go.
  • Exemple: 
    func main() {
    go func() {
        fmt.Println("Hola des d'una goroutine")
    }()
    fmt.Println("Hola des del main")
}

Comunicació Entre Microserveis

  • APIs RESTful: Utilitza HTTP per a la comunicació.
  • Exemple: 
    resp, err := http.Get("http://localhost:8080/hello")
if err != nil {
    log.Fatal(err)
}
defer resp.Body.Close()
body, err := ioutil.ReadAll(resp.Body)
if err != nil {
    log.Fatal(err)
}
fmt.Println(string(body))

Gestió d'Errors

  • Millors Pràctiques:
    • Comprova els errors després de cada operació.
    • Utilitza log.Fatal per a errors crítics.
    • Retorna codis d'estat HTTP significatius.

Proves de Microserveis

  • Proves Unitàries: 
    import (
    "net/http"
    "net/http/httptest"
    "testing"
)

func TestHelloHandler(t *testing.T) {
    req, err := http.NewRequest("GET", "/hello", nil)
    if err != nil {
        t.Fatal(err)
    }
    rr := httptest.NewRecorder()
    handler := http.HandlerFunc(HelloHandler)
    handler.ServeHTTP(rr, req)

    if status := rr.Code; status != http.StatusOK {
        t.Errorf("handler returned wrong status code: got %v want %v", status, http.StatusOK)
    }

    expected := `{"message":"Hola, Món!"}`
    if rr.Body.String() != expected {
        t.Errorf("handler returned unexpected body: got %v want %v", rr.Body.String(), expected)
    }
}

Conclusió

Els microserveis amb Go ofereixen una combinació poderosa de rendiment, simplicitat i concurrència. Seguint les millors pràctiques i aprofitant la robusta biblioteca estàndard de Go, pots construir microserveis escalables i mantenibles. Aquest curs ha cobert els conceptes bàsics de la configuració d'un entorn Go, la construcció de microserveis, la gestió de sol·licituds HTTP, la gestió de dades i el desplegament dels teus serveis. Continua explorant temes més avançats com el descobriment de serveis, les passarel·les d'API i el traçat distribuït per millorar encara més la teva arquitectura de microserveis.
