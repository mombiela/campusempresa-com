Els canals són una característica poderosa en Go que permeten que les goroutines es comuniquin entre elles i sincronitzin la seva execució. Proporcionen una manera d'enviar i rebre valors entre goroutines, fent que la programació concurrent sigui més manejable i menys propensa a errors.

Introducció als Canals

  • Definició: Els canals són conductes tipats a través dels quals pots enviar i rebre valors amb l'operador de canal, <-.
  • Propòsit: S'utilitzen per sincronitzar goroutines i passar dades entre elles.

Creació de Canals

Els canals es creen utilitzant la funció make. La sintaxi és:

ch := make(chan int)
  • Canals Sense Búfer: Aquests canals no tenen cap capacitat per emmagatzemar valors. Bloquegen l'emissor fins que el receptor rep el valor.
  • Canals Amb Búfer: Aquests canals tenen una capacitat per emmagatzemar valors. Bloquegen l'emissor només quan el búfer està ple.

Exemple:

// Canal sense búfer
ch := make(chan int)

// Canal amb búfer amb capacitat de 2
ch := make(chan int, 2)

Enviament i Recepció de Valors

  • Enviament: Utilitza l'operador <- per enviar un valor a un canal.
  • Recepció: Utilitza l'operador <- per rebre un valor d'un canal.

Exemple:

package main

import "fmt"

func main() {
    ch := make(chan int)

    // Enviament d'un valor al canal
    go func() {
        ch <- 42
    }()

    // Recepció d'un valor del canal
    value := <-ch
    fmt.Println(value) // Sortida: 42
}

Direcció del Canal

Els canals poden ser direccionals, és a dir, poden estar restringits a només enviar o només rebre.

  • Canal Només d'Enviament: chan<- int
  • Canal Només de Recepció: <-chan int

Exemple:

package main

import "fmt"

func send(ch chan<- int, value int) {
    ch <- value
}

func receive(ch <-chan int) int {
    return <-ch
}

func main() {
    ch := make(chan int)

    go send(ch, 42)
    value := receive(ch)
    fmt.Println(value) // Sortida: 42
}

Tancament de Canals

Els canals es poden tancar per indicar que no s'enviaran més valors. Això es fa utilitzant la funció close.

Exemple:

package main

import "fmt"

func main() {
    ch := make(chan int)

    go func() {
        for i := 0; i < 5; i++ {
            ch <- i
        }
        close(ch)
    }()

    for value := range ch {
        fmt.Println(value)
    }
}

Sentència Select

La sentència select permet que una goroutine esperi en múltiples operacions de comunicació.

Exemple:

package main

import "fmt"

func main() {
    ch1 := make(chan int)
    ch2 := make(chan int)

    go func() {
        ch1 <- 42
    }()

    go func() {
        ch2 <- 43
    }()

    select {
    case value1 := <-ch1:
        fmt.Println("Rebut", value1, "de ch1")
    case value2 := <-ch2:
        fmt.Println("Rebut", value2, "de ch2")
    }
}

Conceptes Avançats de Canals

Canals Amb Búfer vs Sense Búfer

| Característica | Canals Amb Búfer | Canals Sense Búfer | |-----------------------|------------------------------------|------------------------------------| | Comportament de Blocatge | Bloquegen només quan el búfer està ple | Bloquegen fins que el receptor està llest | | Cas d'Ús | Quan necessites emmagatzemar valors | Quan necessites sincronització | | Rendiment | Pot ser més rendible en alguns casos | Pot conduir a bloquejos si no es gestiona correctament |

Patrons de Canals

  • Fan-out: Distribuir treball entre múltiples goroutines.
  • Fan-in: Multiplexar resultats de múltiples goroutines en un sol canal.

Exemple de Fan-out:

package main

import (
    "fmt"
    "sync"
)

func worker(id int, jobs <-chan int, results chan<- int) {
    for j := range jobs {
        fmt.Printf("Treballador %d ha començat la feina %d\n", id, j)
        results <- j * 2
    }
}

func main() {
    jobs := make(chan int, 5)
    results := make(chan int, 5)

    for w := 1; w < 3; w++ {
        go worker(w, jobs, results)
    }

    for j := 1; j <= 5; j++ {
        jobs <- j
    }
    close(jobs)

    for a := 1; a <= 5; a++ {
        fmt.Println(<-results)
    }
}

Conclusió

Els canals són una part fonamental del model de concurrència de Go, proporcionant una manera simple i poderosa de comunicar-se entre goroutines. Entenent com crear, utilitzar i gestionar canals, pots escriure programes concurrents més eficients i efectius. Recorda utilitzar canals amb búfer quan necessitis emmagatzemar valors i canals sense búfer per a la sincronització. La sentència select és una eina poderosa per gestionar múltiples operacions de canal, i patrons com fan-out i fan-in poden ajudar-te a gestionar fluxos de treball complexos.
