El maneig d'errors és un aspecte crític de la programació, i Go proporciona mecanismes robustos per manejar errors de manera efectiva. Aquest tema cobrirà les millors pràctiques per al maneig d'errors en Go, des de nivells principiants fins a avançats.

Introducció al Maneig d'Errors en Go

  • Errors com a Valors: En Go, els errors es tracten com a valors. El tipus error és una interfície que representa una condició d'error, amb el mètode Error() string.
  • Retornant Errors: Les funcions en Go sovint retornen un error com a últim valor de retorn.

Maneig Bàsic d'Errors

Comencem amb un exemple senzill de maneig d'errors en Go:

package main

import (
    "errors"
    "fmt"
)

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("cannot divide by zero")
    }
    return a / b, nil
}

func main() {
    result, err := divide(4, 0)
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Result:", result)
    }
}
  • Explicació:
    • La funció divide comprova si el divisor b és zero i retorna un error si és cert.
    • En la funció main, l'error es comprova i es maneja adequadament.

Millors Pràctiques per al Maneig d'Errors

1. Comprova els Errors Immediatament

  • Sempre comprova i maneja els errors immediatament després de la crida a la funció que els retorna.
file, err := os.Open("file.txt")
if err != nil {
    log.Fatal(err)
}
defer file.Close()

2. Utilitza Tipus d'Errors Personalitzats

  • Crea tipus d'errors personalitzats per proporcionar més context sobre l'error.
package main

import (
    "fmt"
)

type DivideError struct {
    Dividend float64
    Divisor  float64
}

func (e *DivideError) Error() string {
    return fmt.Sprintf("cannot divide %v by %v", e.Dividend, e.Divisor)
}

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, &DivideError{a, b}
    }
    return a / b, nil
}

func main() {
    result, err := divide(4, 0)
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Result:", result)
    }
}

3. Envolta Errors amb Context Addicional

  • Utilitza la funció fmt.Errorf per envoltar errors amb context addicional.
package main

import (
    "fmt"
    "os"
)

func readFile(filename string) ([]byte, error) {
    file, err := os.Open(filename)
    if err != nil {
        return nil, fmt.Errorf("failed to open file %s: %w", filename, err)
    }
    defer file.Close()

    data := make([]byte, 100)
    _, err = file.Read(data)
    if err != nil {
        return nil, fmt.Errorf("failed to read file %s: %w", filename, err)
    }

    return data, nil
}

func main() {
    _, err := readFile("nonexistent.txt")
    if err != nil {
        fmt.Println("Error:", err)
    }
}

4. Utilitza Errors Sentinel

  • Defineix i utilitza errors sentinel per a condicions d'error comunes.
package main

import (
    "errors"
    "fmt"
)

var ErrDivideByZero = errors.New("cannot divide by zero")

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, ErrDivideByZero
    }
    return a / b, nil
}

func main() {
    result, err := divide(4, 0)
    if errors.Is(err, ErrDivideByZero) {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Result:", result)
    }
}

5. Utilitza `errors.Is` i `errors.As`

  • Utilitza errors.Is per comprovar errors específics.
  • Utilitza errors.As per comprovar i extreure tipus d'errors específics.
package main

import (
    "errors"
    "fmt"
)

type DivideError struct {
    Dividend float64
    Divisor  float64
}

func (e *DivideError) Error() string {
    return fmt.Sprintf("cannot divide %v by %v", e.Dividend, e.Divisor)
}

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, &DivideError{a, b}
    }
    return a / b, nil
}

func main() {
    _, err := divide(4, 0)
    var divErr *DivideError
    if errors.As(err, &divErr) {
        fmt.Println("Divide error:", divErr)
    } else if err != nil {
        fmt.Println("Error:", err)
    }
}

Conclusió

El maneig d'errors és una part essencial d'escriure programes robustos en Go. Seguint aquestes millors pràctiques, pots assegurar-te que el teu codi maneja errors de manera elegant i proporciona retroalimentació significativa als usuaris i desenvolupadors. Recorda:

  • Comprova els errors immediatament.
  • Utilitza tipus d'errors personalitzats per a més context.
  • Envolta errors amb context addicional.
  • Utilitza errors sentinel per a condicions comunes.
  • Utilitza errors.Is i errors.As per a la inspecció i extracció d'errors.

Dominant aquestes tècniques, estaràs ben equipat per manejar errors de manera efectiva en les teves aplicacions Go.
