Introducció

En Go, la gestió d'errors es fa típicament utilitzant el tipus error. No obstant això, hi ha situacions en què un error és tan greu que el programa no pot continuar funcionant normalment. Per a aquests casos, Go proporciona els mecanismes panic i recover. Aquesta secció cobrirà com utilitzar panic i recover, els seus propòsits i les millors pràctiques.

Pànic

Pànic és una funció integrada que atura el flux ordinari de control i comença a pànicar. Quan una funció entra en pànic, deixa d'executar-se, i qualsevol funció ajornada s'executa en ordre invers. Després que totes les funcions ajornades s'hagin executat, el programa es bloqueja amb un missatge de registre.

Quan Utilitzar Pànic

  • Quan el programa no pot continuar funcionant, com quan troba un error crític.
  • Quan s'arriba a un estat inesperat que no hauria de passar mai si el programa és correcte.

Exemple de Pànic

package main

import "fmt"

func main() {
    fmt.Println("Inici del programa")
    panic("Alguna cosa ha anat malament!")
    fmt.Println("Aquesta línia no s'executarà")
}

En aquest exemple, el programa imprimirà "Inici del programa" i després entrarà en pànic amb el missatge "Alguna cosa ha anat malament!". La línia després del pànic no s'executarà.

Recuperació

Recuperació és una funció integrada que recupera el control d'una goroutine en pànic. Recuperació només és útil dins de funcions ajornades. Quan es crida dins d'una funció ajornada, recuperació atura el pànic i retorna el valor que es va passar a la crida de pànic.

Quan Utilitzar Recuperació

  • Per gestionar un pànic i permetre que el programa continuï funcionant.
  • Per registrar el pànic i fer neteja abans que el programa es bloquegi.

Exemple de Recuperació

package main

import "fmt"

func main() {
    defer func() {
        si r := recover(); r != nil {
            fmt.Println("Recuperat del pànic:", r)
        }
    }()
    fmt.Println("Inici del programa")
    panic("Alguna cosa ha anat malament!")
    fmt.Println("Aquesta línia no s'executarà")
}

En aquest exemple, la funció ajornada utilitza recuperació per atrapar el pànic. El programa imprimirà "Inici del programa" i després "Recuperat del pànic: Alguna cosa ha anat malament!". La línia després del pànic encara no s'executarà, però el programa no es bloquejarà.

Millors Pràctiques

  • Utilitza pànic amb moderació: Pànic s'ha d'utilitzar per a situacions realment excepcionals. Per a la gestió d'errors regular, utilitza el tipus error.
  • Recupera al nivell superior: Utilitza recuperació en una funció de nivell superior per assegurar-te que el teu programa pot gestionar els pànics de manera elegant.
  • Registra els pànics: Sempre registra el missatge de pànic per ajudar en la depuració.
  • Evita utilitzar pànic i recuperació per al flux de control: No s'han d'utilitzar com a substitut de la gestió d'errors regular.

Conclusió

Pànic i recuperació són eines poderoses en Go per gestionar errors greus i estats inesperats. Tot i que s'han d'utilitzar amb moderació, entendre com utilitzar-los de manera efectiva pot ajudar-te a escriure programes més robustos i resistents. Recorda utilitzar pànic només per a errors crítics i recuperar-te a un nivell alt per assegurar-te que el teu programa pot gestionar situacions inesperades de manera elegant.
