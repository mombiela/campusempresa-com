Introducció als errors personalitzats

En Go, la gestió d'errors és una part crucial per escriure codi robust i mantenible. Tot i que el tipus error integrat de Go sovint és suficient, hi ha escenaris on els errors personalitzats poden proporcionar més context i millorar el depurament. Aquesta secció cobrirà com crear i utilitzar errors personalitzats en Go.

Creació d'errors personalitzats

Per crear un error personalitzat en Go, normalment es defineix un nou tipus que implementa la interfície error. La interfície error en Go és simple i consisteix en un únic mètode:

type error interface {
    Error() string
}

Definició d'un tipus d'error personalitzat

Aquí tens com pots definir un tipus d'error personalitzat:

package main

import (
    "fmt"
)

// Defineix un tipus d'error personalitzat
type MyCustomError struct {
    Code    int
    Message string
}

// Implementa el mètode Error() per al tipus d'error personalitzat
func (e *MyCustomError) Error() string {
    return fmt.Sprintf("Error %d: %s", e.Code, e.Message)
}

func main() {
    // Crea una instància de l'error personalitzat
    err := &MyCustomError{
        Code:    404,
        Message: "Resource not found",
    }

    // Imprimeix l'error
    fmt.Println(err)
}

Explicació

  • Definició de l'estructura: MyCustomError és una estructura amb camps Code i Message.
  • Mètode Error: El mètode Error() formata el missatge d'error, fent que MyCustomError compleixi la interfície error.
  • Ús: Es crea i s'imprimeix una instància de MyCustomError, demostrant com es pot utilitzar l'error personalitzat.

Ús d'errors personalitzats

Els errors personalitzats es poden utilitzar igual que qualsevol altre error en Go. Es poden retornar des de funcions i comprovar mitjançant assercions de tipus o commutadors de tipus.

Retorn d'errors personalitzats des de funcions

Aquí tens un exemple d'una funció que retorna un error personalitzat:

package main

import (
    "fmt"
)

// Defineix un tipus d'error personalitzat
type MyCustomError struct {
    Code    int
    Message string
}

func (e *MyCustomError) Error() string {
    return fmt.Sprintf("Error %d: %s", e.Code, e.Message)
}

// Funció que retorna un error personalitzat
func doSomething() error {
    return &MyCustomError{
        Code:    500,
        Message: "Internal Server Error",
    }
}

func main() {
    err := doSomething()
    if err != nil {
        fmt.Println(err)
    }
}

Comprovació d'errors personalitzats específics

Pots utilitzar assercions de tipus o commutadors de tipus per comprovar errors personalitzats específics:

package main

import (
    "fmt"
)

// Defineix un tipus d'error personalitzat
type MyCustomError struct {
    Code    int
    Message string
}

func (e *MyCustomError) Error() string {
    return fmt.Sprintf("Error %d: %s", e.Code, e.Message)
}

func doSomething() error {
    return &MyCustomError{
        Code:    500,
        Message: "Internal Server Error",
    }
}

func main() {
    err := doSomething()
    if err != nil {
        // Asserció de tipus
        if customErr, ok := err.(*MyCustomError); ok {
            fmt.Printf("S'ha produït un error personalitzat: Codi=%d, Missatge=%s\n", customErr.Code, customErr.Message)
        } else {
            fmt.Println("S'ha produït un error:", err)
        }
    }
}

Millors pràctiques per a errors personalitzats

  • Utilitza errors personalitzats amb moderació: Només utilitza errors personalitzats quan sigui necessari un context addicional.
  • Missatges d'error consistents: Assegura't que els missatges d'error siguin consistents i informatius.
  • Embolcall d'errors: Considera utilitzar les característiques d'embolcall d'errors de Go 1.13+ (fmt.Errorf amb %w) per a escenaris de gestió d'errors més complexos.

Conclusió

Els errors personalitzats en Go proporcionen una manera poderosa d'afegir context a la gestió d'errors. Definint tipus d'errors personalitzats i implementant la interfície error, pots crear missatges d'error més informatius i mantenibles. Recorda utilitzar errors personalitzats amb prudència i seguir les millors pràctiques per assegurar que el teu codi es mantingui net i comprensible.
