Introducció a les proves en Go

Les proves són una part crucial del desenvolupament de programari, assegurant que el teu codi es comporti com s'espera. Go proporciona un marc de proves robust com a part de la seva biblioteca estàndard, fent que sigui fàcil escriure i executar proves.

Conceptes clau

  • Funcions de prova: Funcions que proven el teu codi, típicament anomenades amb el prefix Test.
  • Fitxers de prova: Fitxers que contenen funcions de prova, normalment acabant amb _test.go.
  • Paquets de prova: Paquets que inclouen fitxers i funcions de prova.
  • Afirmacions: Declaracions que comproven si una condició és certa, utilitzades per validar els resultats de les proves.
  • Cobertura de proves: Una mesura de quant del teu codi està provat.

Escriure proves bàsiques

En Go, les proves s'escriuen en fitxers que acaben amb _test.go i utilitzen el paquet testing.

Exemple: Funció de prova bàsica

// math_test.go
package math

import "testing"

func TestAdd(t *testing.T) {
    result := Add(2, 3)
    if result != 5 {
        t.Errorf("Add(2, 3) = %d; want 5", result)
    }
}
  • Explicació:
    • La funció de prova TestAdd comprova si la funció Add retorna el resultat correcte.
    • t.Errorf s'utilitza per informar d'un error si el resultat no és l'esperat.

Executar proves

Per executar proves, utilitza la comanda go test al terminal.

Exemple: Executar proves

$ go test
  • Explicació:
    • Aquesta comanda executa totes les proves en el paquet actual.

Taula de funcions de prova comunes

| Funció | Descripció | |-------------------|--------------------------------------------------| | t.Error | Registra un missatge d'error però continua l'execució. | | t.Errorf | Registra un missatge d'error formatat però continua l'execució. | | t.Fail | Marca la prova com fallida però continua l'execució.| | t.FailNow | Marca la prova com fallida i atura l'execució. | | t.Fatal | Registra un missatge d'error i atura l'execució. | | t.Fatalf | Registra un missatge d'error formatat i atura l'execució.|

Tècniques avançades de prova

A mesura que et sentis més còmode amb les proves bàsiques, pots explorar tècniques més avançades.

Proves basades en taules

Les proves basades en taules utilitzen taules (slices) per provar múltiples casos en una sola funció.

Exemple: Prova basada en taules

func TestAdd(t *testing.T) {
    tests := []struct {
        a, b, want int
    }{
        {1, 1, 2},
        {2, 2, 4},
        {2, -2, 0},
    }

    for _, tt := range tests {
        t.Run(fmt.Sprintf("%d+%d", tt.a, tt.b), func(t *testing.T) {
            got := Add(tt.a, tt.b)
            if got != tt.want {
                t.Errorf("Add(%d, %d) = %d; want %d", tt.a, tt.b, got, tt.want)
            }
        })
    }
}
  • Explicació:
    • El slice tests conté múltiples casos de prova.
    • t.Run s'utilitza per executar cada cas de prova com una subprova.

Benchmarking

Go també suporta el benchmarking per mesurar el rendiment del teu codi.

Exemple: Funció de benchmark

func BenchmarkAdd(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Add(1, 2)
    }
}
  • Explicació:
    • La funció de benchmark BenchmarkAdd mesura quantes vegades es pot executar Add en un període determinat.

Mocking i Stubbing

Per a proves més complexes, potser necessitaràs fer mocking o stubbing de dependències.

Exemple: Mocking d'una dependència

type MockDatabase struct{}

func (m *MockDatabase) GetUser(id int) User {
    return User{ID: id, Name: "Mock User"}
}

func TestGetUser(t *testing.T) {
    db := &MockDatabase{}
    user := db.GetUser(1)
    if user.Name != "Mock User" {
        t.Errorf("Expected 'Mock User', got %s", user.Name)
    }
}
  • Explicació:
    • MockDatabase és una implementació mock d'una base de dades.
    • La prova comprova si GetUser retorna l'usuari mock esperat.

Conclusió

Les proves en Go són senzilles i potents, gràcies al seu marc de proves integrat. Dominant les proves bàsiques, les proves basades en taules, el benchmarking i el mocking, pots assegurar-te que el teu codi Go sigui fiable i eficient. Les proves regulars ajuden a detectar errors aviat i milloren la qualitat del codi, fent que sigui una habilitat essencial per a qualsevol desenvolupador Go.
