Introducció

Les proves i la depuració són habilitats crucials per a qualsevol programador. En aquesta secció, cobrirem els conceptes bàsics de les proves i la depuració en Go, progressant des de tècniques per a principiants fins a tècniques avançades. Al final d'aquesta secció, hauries de sentir-te còmode escrivint proves, utilitzant les eines de proves de Go i depurant els teus programes Go de manera efectiva.

Proves en Go

Conceptes Bàsics de les Proves

  • Paquet de Proves de Go: Go té un paquet de proves integrat testing que proporciona suport per a proves automatitzades.
  • Funcions de Prova: Les funcions de prova en Go es identifiquen pel prefix Test seguit del nom de la funció que s'està provant.

Exemple d'una Prova Bàsica

package main

import (
    "testing"
)

func Add(a, b int) int {
    return a + b
}

func TestAdd(t *testing.T) {
    result := Add(2, 3)
    expected := 5
    if result != expected {
        t.Errorf("Add(2, 3) = %d; want %d", result, expected)
    }
}
  • Executar Proves: Utilitza la comanda go test per executar proves en el teu paquet Go.

Proves Basades en Taules

  • Concepte: Les proves basades en taules utilitzen una taula d'entrades i sortides esperades per executar múltiples casos de prova en una sola funció de prova.
  • Avantatges: Faciliten l'afegiment de nous casos de prova i milloren la llegibilitat de les proves.

Exemple de Prova Basada en Taules

package main

import (
    "testing"
)

func Multiply(a, b int) int {
    return a * b
}

func TestMultiply(t *testing.T) {
    tests := []struct {
        a, b, expected int
    }{
        {2, 3, 6},
        {4, 5, 20},
        {0, 10, 0},
    }

    for _, tt := range tests {
        result := Multiply(tt.a, tt.b)
        if result != tt.expected {
            t.Errorf("Multiply(%d, %d) = %d; want %d", tt.a, tt.b, result, tt.expected)
        }
    }
}

Proves de Rendiment

  • Propòsit: Les proves de rendiment ajuden a mesurar el rendiment del teu codi.
  • Funcions de Rendiment: Identificades pel prefix Benchmark i prenen un paràmetre *testing.B.

Exemple de Prova de Rendiment

package main

import (
    "testing"
)

func BenchmarkAdd(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Add(2, 3)
    }
}
  • Executar Proves de Rendiment: Utilitza la comanda go test -bench . per executar proves de rendiment.

Cobertura de Proves

  • Concepte: La cobertura de proves mesura el percentatge del teu codi que està cobert per proves.
  • Comanda: Utilitza go test -cover per veure la cobertura de proves del teu paquet.

Depuració en Go

Utilitzant Sentències de Print

  • Tècnica Bàsica: Inserir sentències de print (fmt.Println) en el teu codi per entendre el flux i l'estat de les variables.
  • Limitacions: Pot ser que consumeixi temps i embruti el codi.

Utilitzant el Depurador Delve

  • Delve: Un depurador potent per a Go que et permet establir punts de ruptura, inspeccionar variables i controlar el flux d'execució.
  • Instal·lació: Instal·la Delve utilitzant go install github.com/go-delve/delve/cmd/dlv@latest.

Comandes Bàsiques en Delve

  • Iniciar Delve: dlv debug per començar a depurar el teu programa.
  • Establir Punts de Ruptura: break main.go:10 per establir un punt de ruptura a la línia 10 de main.go.
  • Executar el Programa: continue per executar el programa fins al següent punt de ruptura.
  • Inspeccionar Variables: print variableName per inspeccionar el valor d'una variable.
  • Passar pel Codi: step per entrar en funcions, next per passar per sobre de funcions, i stepout per sortir de la funció actual.

Conclusió

Les proves i la depuració són pràctiques essencials per escriure codi Go fiable i mantenible. Dominant el paquet de proves de Go, escrivint proves efectives i utilitzant eines com Delve per a la depuració, pots millorar significativament la qualitat del teu codi. Recorda escriure proves per al teu codi regularment i utilitzar eines de depuració per identificar i solucionar ràpidament els problemes.
