Introducció

En aquesta secció, cobrirem els passos essencials i les millors pràctiques per construir i desplegar aplicacions Go. Això inclou compilar el teu codi Go, gestionar dependències, crear binaris executables i desplegar la teva aplicació a diferents entorns.

Construcció d'Aplicacions Go

Compilació de Codi Go

  • Comanda Go Build: La comanda go build compila els paquets i les dependències.

    // main.go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}

    $ go build main.go
    • Aquesta comanda genera un fitxer executable anomenat main (o main.exe a Windows).

  • Comanda Go Install: La comanda go install compila i instal·la el paquet.

    $ go install main.go
    • Aquesta comanda col·loca l'executable al directori $GOPATH/bin.

Gestió de Dependències

  • Mòduls Go: Els mòduls Go són l'estàndard per a la gestió de dependències en Go.
    • Inicialitzar un nou mòdul: 
      $ go mod init example.com/myapp
    • Afegir una dependència: 
      $ go get github.com/gin-gonic/gin
    • Netejar les dependències: 
      $ go mod tidy

Creació de Binaris Executables

  • Compilació Creuada: Go suporta la compilació creuada per a diferents sistemes operatius i arquitectures. 
    $ GOOS=linux GOARCH=amd64 go build -o myapp-linux main.go
$ GOOS=windows GOARCH=amd64 go build -o myapp.exe main.go
    • Aquesta comanda crea binaris per a Linux i Windows.

Desplegament d'Aplicacions Go

Desplegament a un Servidor

  • Copiar Binaris: Simplement copia el binari compilat al servidor de destinació. 
    $ scp myapp-linux user@server:/path/to/deploy
  • Executar l'Aplicació: SSH al servidor i executa el binari. 
    $ ssh user@server
$ ./myapp-linux

Ús de Docker per al Desplegament

  • Dockerfile: Crea un Dockerfile per conteniritzar la teva aplicació Go. 
    # Dockerfile
FROM golang:1.17-alpine AS builder
WORKDIR /app
COPY . .
RUN go build -o myapp

FROM alpine:latest
WORKDIR /root/
COPY --from=builder /app/myapp .
CMD ["./myapp"]
  • Construcció de la Imatge Docker: 
    $ docker build -t myapp:latest .
  • Execució del Contenidor Docker: 
    $ docker run -p 8080:8080 myapp:latest

Desplegament a Serveis al Núvol

  • Google Cloud Run: Desplegament d'una aplicació Go conteniritzada a Google Cloud Run.

    • Construir i pujar la imatge Docker a Google Container Registry: 
      $ docker build -t gcr.io/[PROJECT-ID]/myapp .
$ docker push gcr.io/[PROJECT-ID]/myapp
    • Desplegar a Cloud Run: 
      $ gcloud run deploy --image gcr.io/[PROJECT-ID]/myapp --platform managed

  • Heroku: Desplegament d'una aplicació Go a Heroku.

    • Crear un Procfile: 
      web: ./myapp
    • Desplegar utilitzant Git: 
      $ git init
$ heroku create
$ git add .
$ git commit -m "Initial commit"
$ git push heroku master

Conclusió

Construir i desplegar aplicacions Go implica diversos passos, des de compilar el codi fins a gestionar les dependències i crear binaris executables. El desplegament es pot fer en diverses plataformes, incloent servidors, contenidors Docker i serveis al núvol. Seguint les millors pràctiques i utilitzant les eines adequades, pots assegurar-te que les teves aplicacions Go es construeixen i despleguen de manera eficient i fiable.
