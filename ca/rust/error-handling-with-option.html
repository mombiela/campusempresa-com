Gestió d'Errors amb Option
Pànic i Desenrotllament ►

Introducció

La gestió d'errors és un aspecte crucial de la programació, i Rust proporciona eines poderoses per gestionar els errors de manera efectiva. Una d'aquestes eines és el tipus Option, que s'utilitza per representar un valor que pot estar present (Some) o absent (None). Aquesta secció cobrirà els conceptes bàsics de Option, com utilitzar-lo per a la gestió d'errors, i proporcionarà exemples pràctics i exercicis.

Entenent `Option`

  • Option és un enum proporcionat per la biblioteca estàndard de Rust.
  • Té dues variants:
    • Some(T): Conté un valor de tipus T.
    • None: Representa l'absència d'un valor.

Exemple

fn find_element(vec: Vec<i32>, target: i32) -> Option<usize> {
    for (index, &element) in vec.iter().enumerate() {
        if element == target {
            return Some(index);
        }
    }
    None
}

En aquest exemple, la funció find_element retorna un Option<usize>. Si es troba l'element objectiu, retorna Some(index), en cas contrari, retorna None.

Utilitzant `Option` per a la Gestió d'Errors

  • Option és útil per a funcions que podrien no retornar un valor.
  • Ajuda a evitar excepcions de punter nul gestionant explícitament l'absència d'un valor.

Mètodes Comuns

  • is_some(): Retorna true si l'Option és Some.
  • is_none(): Retorna true si l'Option és None.
  • unwrap(): Retorna el valor dins de Some, pànic si es crida sobre None.
  • unwrap_or(default): Retorna el valor dins de Some, o un valor per defecte si es crida sobre None.
  • map(): Aplica una funció al valor contingut (si n'hi ha) i retorna un nou Option.

Exemple

fn divide(numerator: f64, denominator: f64) -> Option<f64> {
    if denominator == 0.0 {
        None
    } else {
        Some(numerator / denominator)
    }
}

fn main() {
    let result = divide(10.0, 2.0);
    match result {
        Some(value) => println!("Resultat: {}", value),
        None => println!("No es pot dividir per zero"),
    }
}

En aquest exemple, la funció divide retorna un Option<f64>. Si el denominador és zero, retorna None, en cas contrari, retorna el resultat embolicat en Some.

Exemples Pràctics

Exemple 1: Utilitzant `unwrap_or`

fn get_user_name(user_id: u32) -> Option<String> {
    if user_id == 1 {
        Some("Alice".to_string())
    } else {
        None
    }
}

fn main() {
    let user_name = get_user_name(2).unwrap_or("Guest".to_string());
    println!("Hola, {}", user_name);
}

En aquest exemple, si la funció get_user_name retorna None, el mètode unwrap_or proporciona un valor per defecte de "Guest".

Exemple 2: Utilitzant `map`

fn square_even_number(number: Option<i32>) -> Option<i32> {
    number.map(|n| if n % 2 == 0 { n * n } else { n })
}

fn main() {
    let number = Some(4);
    let squared = square_even_number(number);
    println!("Quadrat: {:?}", squared);
}

En aquest exemple, el mètode map s'utilitza per aplicar una funció al valor dins de Some, si existeix.

Exercicis

  1. Ús Bàsic: Escriu una funció find_max que prengui un vector d'enters i retorni el valor màxim com un Option<i32>. Si el vector està buit, retorna None.
  2. Utilitzant unwrap_or: Modifica la funció find_max per retornar un valor per defecte de 0 si el vector està buit.
  3. Utilitzant map: Escriu una funció increment_if_some que prengui un Option<i32> i retorni un Option<i32> amb el valor incrementat en 1 si existeix.

Conclusió

El tipus Option en Rust és una eina poderosa per gestionar casos on un valor podria estar absent. Utilitzant Option, pots escriure codi més segur i expressiu, evitant errors comuns com les excepcions de punter nul. Practica utilitzant Option amb els exercicis proporcionats per solidificar la teva comprensió i millorar les teves habilitats de gestió d'errors en Rust.
