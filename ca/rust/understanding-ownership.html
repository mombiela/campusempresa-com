Comprendre la Propietat
Referències i Préstec ►

La propietat és una de les característiques més úniques i poderoses de Rust. Permet a Rust gestionar la memòria de manera segura i eficient sense un col·lector d'escombraries. Aquest tema et guiarà a través dels conceptes bàsics de la propietat, el préstec i les durades, proporcionant-te una base sòlida per escriure codi Rust segur i eficient.

Conceptes Clau de la Propietat

  • Propietat: Cada valor en Rust té una variable que s'anomena el seu propietari.
  • Préstec: Referències a un valor sense prendre'n la propietat.
  • Durades: Assegurar que les referències siguin vàlides mentre s'utilitzen.

Regles de la Propietat

  1. Cada valor en Rust té un únic propietari.
  2. Quan el propietari surt de l'àmbit, el valor es descarta.
  3. La propietat es pot transferir (moure) a una altra variable.

Exemple: Propietat Bàsica

fn main() {
    let s1 = String::from("hello"); // s1 és propietari de la String
    let s2 = s1; // La propietat de la String es mou a s2
    // println!("{}", s1); // Això causaria un error en temps de compilació perquè s1 ja no és propietari de la String
    println!("{}", s2); // Això és vàlid
}

En aquest exemple:

  • s1 inicialment és propietari de la String.
  • La propietat es transfereix a s2 quan s1 s'assigna a s2.
  • Intentar utilitzar s1 després de la transferència resulta en un error en temps de compilació.

Préstec

El préstec et permet referenciar un valor sense prendre'n la propietat. Això es fa utilitzant referències (&).

Exemple: Préstec

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1); // Préstec de s1
    println!("La longitud de '{}' és {}.", s1, len); // s1 encara es pot utilitzar aquí
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

En aquest exemple:

  • s1 es presta a la funció calculate_length.
  • s1 encara es pot utilitzar després de la crida a la funció perquè la propietat no es transfereix.

Préstec Mutable

Rust imposa que pots tenir o bé una referència mutable o qualsevol nombre de referències immutables, però no ambdues simultàniament.

Exemple: Préstec Mutable

fn main() {
    let mut s = String::from("hello");
    change(&mut s); // Préstec mutable
    println!("{}", s); // s encara es pot utilitzar aquí
}

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}

En aquest exemple:

  • s és mutable i es presta de manera mutable a change.
  • change modifica la String afegint ", world".

Durades

Les durades asseguren que les referències siguin vàlides mentre s'utilitzen. Rust utilitza durades per evitar referències penjants.

Exemple: Durades

fn main() {
    let r;
    {
        let x = 5;
        r = &x;
    }
    // println!("r: {}", r); // Això causaria un error en temps de compilació
}

En aquest exemple:

  • x surt de l'àmbit abans que r s'utilitzi, la qual cosa condueix a una referència penjant.
  • Rust evita això imposant durades.

Resum

  • Propietat: Assegura que cada valor tingui un únic propietari, i els valors es descarten quan el propietari surt de l'àmbit.
  • Préstec: Permet referències a un valor sense prendre'n la propietat, permetent un accés segur.
  • Durades: Asseguren que les referències siguin vàlides mentre s'utilitzen, evitant referències penjants.

Comprendre aquests conceptes és crucial per escriure codi Rust segur i eficient. Practica aquests conceptes amb diversos exemples per consolidar la teva comprensió.
