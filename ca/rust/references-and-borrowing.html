Introducció

En Rust, gestionar la memòria de manera segura i eficient és un aspecte fonamental del llenguatge. Dos conceptes clau que faciliten això són les referències i els préstecs. Entendre aquests conceptes és crucial per escriure codi idiomàtic i segur en Rust.

Referències

Les referències en Rust et permeten referir-te a algun valor sense prendre'n la propietat. Això és útil quan vols llegir o modificar dades sense transferir-ne la propietat.

Creació de referències

Per crear una referència a un valor, utilitzes el símbol &.

fn main() {
    let x = 5;
    let y = &x; // y és una referència a x

    println!("x: {}, y: {}", x, y);
}

En aquest exemple:

  • x és un enter amb un valor de 5.
  • y és una referència a x.

Desreferenciació

Per accedir al valor al qual apunta una referència, utilitzes l'operador *.

fn main() {
    let x = 5;
    let y = &x;

    println!("x: {}, *y: {}", x, *y); // *y desreferencia y per obtenir el valor de x
}

Préstecs

El préstec en Rust es refereix a la pràctica d'utilitzar referències per accedir a dades sense prendre'n la propietat. El préstec pot ser immutable o mutable.

Préstec immutable

Quan prens un valor de manera immutable, no pots modificar el valor a través de la referència.

fn main() {
    let x = 5;
    let y = &x; // Préstec immutable

    println!("x: {}, y: {}", x, y);
    // x i y no es poden modificar aquí
}

Préstec mutable

Quan prens un valor de manera mutable, pots modificar el valor a través de la referència. No obstant això, Rust imposa regles estrictes per garantir la seguretat:

  • Pots tenir una referència mutable o qualsevol nombre de referències immutables, però no ambdues al mateix temps.
fn main() {
    let mut x = 5;
    {
        let y = &mut x; // Préstec mutable
        *y += 1; // Modifica el valor a través de la referència mutable
    } // y surt de l'àmbit aquí, així que el préstec mutable acaba

    println!("x: {}", x); // x ara és 6
}

Verificador de préstecs

El verificador de préstecs de Rust imposa regles en temps de compilació per garantir que les referències s'utilitzin de manera segura. Evita les curses de dades i garanteix la seguretat de la memòria.

Regles del préstec

  • Pots tenir múltiples referències immutables o una referència mutable, però no ambdues.
  • Les referències sempre han de ser vàlides.
fn main() {
    let mut x = 5;
    let y = &x; // Préstec immutable
    // let z = &mut x; // Error: no es pot prendre `x` com mutable perquè també es pren com immutable

    println!("x: {}, y: {}", x, y);
}

Durades de vida

Les durades de vida són una manera perquè Rust faci un seguiment de quant de temps són vàlides les referències. Asseguren que les referències no sobrevisquin a les dades a les quals apunten.

Anotació bàsica de durades de vida

Les durades de vida normalment són inferides pel compilador, però de vegades cal especificar-les explícitament.

fn longest<'a>(s1: &'a str, s2: &'a str) -> &'a str {
    if s1.len() > s2.len() {
        s1
    } else {
        s2
    }
}

fn main() {
    let string1 = String::from("long string is long");
    let string2 = "xyz";

    let result = longest(string1.as_str(), string2);
    println!("The longest string is {}", result);
}

En aquest exemple:

  • 'a és un paràmetre de durada de vida que assegura que la referència retornada sigui vàlida mentre ambdues s1 i s2 siguin vàlides.

Conclusió

Les referències i els préstecs són conceptes fonamentals en Rust que permeten una gestió segura i eficient de la memòria. Entenent i aplicant aquests conceptes, pots escriure programes robustos i eficients en Rust. Recorda:

  • Utilitza referències per accedir a dades sense prendre'n la propietat.
  • Segueix les regles del préstec per evitar curses de dades i garantir la seguretat de la memòria.
  • Utilitza durades de vida per assegurar que les referències siguin vàlides durant el temps necessari.
