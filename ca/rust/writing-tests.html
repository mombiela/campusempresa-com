Escriure Proves a Rust
Executar Proves ►

Provar és una part crucial del desenvolupament de programari. Assegura que el teu codi funcioni com s'espera i ajuda a prevenir errors. Rust té un marc de proves integrat que fa que escriure i executar proves sigui senzill. Aquesta secció et guiarà a través de l'escriptura de proves a Rust, des de nivells bàsics fins a avançats.

Introducció a les Proves a Rust

  • Per què les Proves són Importants: Provar ajuda a detectar errors aviat, assegura la fiabilitat del codi i millora la mantenibilitat.
  • El Marc de Proves de Rust: Rust inclou un marc de proves integrat que s'integra perfectament amb Cargo, el gestor de paquets i sistema de construcció de Rust.

Escriure Proves Bàsiques

Crear una Funció de Prova

A Rust, les funcions de prova s'anoten amb l'atribut #[test]. Aquí tens un exemple senzill:

#[cfg(test)]
mod tests {
    #[test]
    fn it_works() {
        let result = 2 + 2;
        assert_eq!(result, 4);
    }
}
  • Explicació:
    • #[cfg(test)]: Aquest atribut assegura que el mòdul de prova només es compili quan s'executen proves.
    • #[test]: Marca la funció com una funció de prova.
    • assert_eq!: Una macro que comprova si dos valors són iguals. Si no ho són, la prova falla.

Executar Proves

Per executar proves, utilitza la següent comanda:

cargo test

Aquesta comanda compila el teu codi en mode de prova i executa totes les funcions de prova.

Escriure Proves Més Complexes

Provar Panics

De vegades, vols assegurar-te que una funció fa pànic sota certes condicions. Utilitza l'atribut #[should_panic] per això:

#[cfg(test)]
mod tests {
    #[test]
    #[should_panic]
    fn it_panics() {
        panic!("Aquesta funció hauria de fer pànic");
    }
}
  • Explicació:
    • #[should_panic]: Marca la prova com una que hauria de fer pànic. Si la funció no fa pànic, la prova falla.

Utilitzar Result en Proves

Les funcions de prova poden retornar Result<T, E>, permetent l'ús de l'operador ? per a la gestió d'errors:

#[cfg(test)]
mod tests {
    #[test]
    fn it_works_with_result() -> Result<(), String> {
        if 2 + 2 == 4 {
            Ok(())
        } else {
            Err(String::from("Les matemàtiques estan trencades"))
        }
    }
}
  • Explicació:
    • Retornar Result<(), String> permet l'ús de ? per a una gestió d'errors concisa.
    • La prova passa si es retorna Ok(()) i falla si es retorna Err.

Organitzar Proves

Mòduls de Prova

Organitza les proves en mòduls per mantenir el teu codi net i fàcil de mantenir:

#[cfg(test)]
mod tests {
    mod math_tests {
        #[test]
        fn addition_works() {
            assert_eq!(2 + 2, 4);
        }
    }

    mod string_tests {
        #[test]
        fn string_contains() {
            assert!(String::from("hello").contains("ell"));
        }
    }
}
  • Explicació:
    • Agrupa proves relacionades en submòduls per a una millor organització.

Proves d'Integració

Les proves d'integració es col·loquen al directori tests a l'arrel del teu projecte. Aquestes proves són externes a la teva biblioteca i proven l'API pública:

// tests/integration_test.rs
use my_crate;

#[test]
fn it_adds_two() {
    assert_eq!(my_crate::add_two(2), 4);
}
  • Explicació:
    • Les proves d'integració utilitzen l'API pública del teu paquet.
    • Col·loca-les al directori tests.

Tècniques Avançades de Prova

Mocking i Stubbing

Rust no té suport integrat per a mocking, però pots utilitzar paquets com mockall o mockito per a aquest propòsit.

Proves Basades en Propietats

Les proves basades en propietats comproven que una propietat es manté per a una àmplia gamma d'entrades. El paquet proptest és comunament utilitzat per això:

use proptest::prelude::*;

proptest! {
    #[test]
    fn test_addition(a in 0..1000, b in 0..1000) {
        assert_eq!(a + b, b + a);
    }
}
  • Explicació:
    • La macro proptest! genera proves per a una gamma d'entrades.
    • Assegura que la propietat (commutativitat de l'addició) es manté per a totes les entrades generades.

Conclusió

Provar és una part essencial de la programació en Rust. Escriure proves assegura que el teu codi sigui fiable i fàcil de mantenir. Comença amb proves bàsiques utilitzant #[test] i assert_eq!, després passa a escenaris més complexos com provar panics i utilitzar Result<T, E>. Organitza les teves proves en mòduls i utilitza proves d'integració per provar la teva API pública. Per a proves avançades, explora mocking, stubbing i proves basades en propietats.

Incorporant aquestes pràctiques, podràs escriure codi robust i fiable en Rust.
