Introducció als Mòduls

Els mòduls en Rust són una manera d'organitzar el codi en espais de noms separats, fent-lo més fàcil de gestionar i entendre. Permeten agrupar funcions, structs, enums i altres elements relacionats. Això ajuda a mantenir una base de codi neta i organitzada, especialment a mesura que el teu projecte creix.

Conceptes Clau

  • Declaració de Mòduls: Com declarar un mòdul.
  • Fitxers de Mòduls: Com dividir els mòduls en diferents fitxers.
  • Visibilitat: Com controlar la visibilitat dels elements dins d'un mòdul.
  • Ús de Mòduls: Com utilitzar elements d'altres mòduls.

Declaració de Mòduls

En Rust, pots declarar un mòdul utilitzant la paraula clau mod. Això es pot fer dins d'un sol fitxer o a través de múltiples fitxers.

Exemple

mod my_module {
    pub fn say_hello() {
        println!("Hello from my_module!");
    }
}

fn main() {
    my_module::say_hello();
}

En aquest exemple:

  • Declarem un mòdul anomenat my_module.
  • Dins de my_module, definim una funció pública say_hello.
  • A la funció main, cridem my_module::say_hello.

Fitxers de Mòduls

Els mòduls es poden dividir en fitxers separats per mantenir la base de codi organitzada. El sistema de mòduls en Rust et permet crear una estructura de directoris que reflecteixi la jerarquia dels teus mòduls.

Exemple

Suposem la següent estructura de directoris:

src/
├── main.rs
└── my_module/
    └── mod.rs

En main.rs:

mod my_module;

fn main() {
    my_module::say_hello();
}

En my_module/mod.rs:

pub fn say_hello() {
    println!("Hello from my_module!");
}

Aquí:

  • mod my_module; en main.rs diu a Rust que busqui el mòdul en my_module/mod.rs.
  • La funció say_hello està definida en my_module/mod.rs.

Visibilitat

Per defecte, tots els elements dins d'un mòdul són privats. Pots controlar la visibilitat utilitzant la paraula clau pub.

Exemple

mod my_module {
    pub fn public_function() {
        println!("This is a public function.");
    }

    fn private_function() {
        println!("This is a private function.");
    }
}

fn main() {
    my_module::public_function();
    // my_module::private_function(); // Això causarà un error en temps de compilació
}

En aquest exemple:

  • public_function és accessible fora del mòdul perquè està declarada amb pub.
  • private_function no és accessible fora del mòdul.

Ús de Mòduls

Per utilitzar elements d'altres mòduls, pots utilitzar la paraula clau use. Això pot fer que el teu codi sigui més net i fàcil de llegir.

Exemple

mod my_module {
    pub fn say_hello() {
        println!("Hello from my_module!");
    }
}

use my_module::say_hello;

fn main() {
    say_hello();
}

En aquest exemple:

  • Utilitzem use my_module::say_hello; per portar say_hello a l'abast.
  • Llavors podem cridar say_hello directament a la funció main.

Conclusió

Els mòduls en Rust són una característica poderosa que t'ajuda a organitzar el teu codi en peces manejables. Entenent com declarar mòduls, dividir-los en fitxers, controlar la visibilitat i utilitzar elements d'altres mòduls, pots escriure codi Rust més net i mantenible. A mesura que progressis, trobaràs que els mòduls són essencials per construir aplicacions més grans i complexes.
