Durades
Traits ►

Les durades a Rust són una manera d'assegurar que les referències siguin vàlides mentre siguin necessàries, prevenint referències penjants i assegurant la seguretat de la memòria sense necessitat d'un col·lector d'escombraries. Aquest tema et guiarà des dels conceptes bàsics fins a l'ús avançat de les durades a Rust.

Introducció a les Durades

  • Definició: Les durades són anotacions que indiquen al compilador de Rust quant de temps han de ser vàlides les referències.
  • Propòsit: Ajuden a prevenir referències penjants i a assegurar la seguretat de la memòria.
  • Sintaxi: Les durades es denoten amb un apòstrof seguit d'un nom, per exemple, 'a.

Anotacions Bàsiques de Durades

Quan tens una funció que pren referències com a paràmetres, has d'especificar les durades per dir al compilador com es relacionen les durades d'aquestes referències entre elles.

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}
  • Explicació:
    • 'a és un paràmetre de durada.
    • &'a str significa una referència a un slice de cadena que viu almenys tant com 'a.
    • La funció longest pren dos slices de cadena amb la mateixa durada 'a i retorna un slice de cadena amb la mateixa durada 'a.

Eliminació de Durades

Rust té una característica anomenada eliminació de durades, que permet al compilador inferir les durades en certes situacions, reduint la necessitat d'anotacions explícites.

  • Tres regles de l'eliminació de durades:
    1. Cada paràmetre que és una referència obté el seu propi paràmetre de durada.
    2. Si hi ha exactament un paràmetre de durada d'entrada, aquesta durada s'assigna a tots els paràmetres de durada de sortida.
    3. Si hi ha múltiples paràmetres de durada d'entrada, però un d'ells és &self o &mut self, la durada de self s'assigna a tots els paràmetres de durada de sortida.
fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]
}
  • Explicació:
    • La funció first_word pren un slice de cadena i retorna un slice de cadena.
    • El compilador infereix les durades basant-se en les regles d'eliminació.

Durades Avançades

Structs amb Durades

Quan defineixes una struct que conté referències, has d'especificar les durades per a aquestes referències.

struct ImportantExcerpt<'a> {
    part: &'a str,
}

impl<'a> ImportantExcerpt<'a> {
    fn level(&self) -> i32 {
        3
    }

    fn announce_and_return_part(&self, announcement: &str) -> &str {
        println!("Atenció, si us plau: {}", announcement);
        self.part
    }
}
  • Explicació:
    • La struct ImportantExcerpt té un paràmetre de durada 'a.
    • El camp part és una referència que ha de viure almenys tant com 'a.
    • Els mètodes dins del bloc impl poden utilitzar la durada 'a per assegurar-se que les referències siguin vàlides.

Límits de Durades

Pots especificar que un paràmetre de tipus ha de viure almenys tant com una certa durada.

fn longest_with_an_announcement<'a, T>(x: &'a str, y: &'a str, ann: T) -> &'a str
where
    T: std::fmt::Display,
{
    println!("Anunci! {}", ann);
    if x.len() > y.len() {
        x
    } else {
        y
    }
}
  • Explicació:
    • La funció longest_with_an_announcement té un paràmetre de durada 'a i un tipus genèric T.
    • La clàusula where especifica que T ha d'implementar el trait Display.
    • La funció imprimeix un anunci i retorna el més llarg de dos slices de cadena.

Conclusió

Les durades són una característica poderosa a Rust que ajuden a assegurar la seguretat de la memòria prevenint referències penjants. Entenent i utilitzant les durades, pots escriure codi Rust més robust i segur. Recorda:

  • Utilitza anotacions de durades per especificar quant de temps han de ser vàlides les referències.
  • Aprofita l'eliminació de durades per reduir el codi redundant.
  • Utilitza durades en structs i funcions genèriques per assegurar que les referències siguin vàlides a través de diferents àmbits.

Dominant les durades, pots aprofitar el potent sistema de tipus de Rust per escriure programes eficients i segurs.
