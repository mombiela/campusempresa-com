Contribuir al Codi Obert
Comunitat i Ecosistema de Rust ►

Contribuir a projectes de codi obert pot ser una manera gratificant de millorar les teves habilitats en Rust, col·laborar amb altres desenvolupadors i retornar a la comunitat. Aquesta secció et guiarà a través del procés de contribuir a projectes de codi obert en Rust, des de trobar un projecte fins a fer la teva primera sol·licitud d'extracció.

Trobar un Projecte de Codi Obert

Abans de poder contribuir, necessites trobar un projecte que t'interessi i que coincideixi amb el teu nivell d'habilitat.

  • GitHub: La plataforma més popular per a projectes de codi obert. Utilitza la funció de cerca per trobar projectes en Rust.
  • Crates.io: El registre de paquets de Rust. Navega per les crates populars i mira si necessiten ajuda.
  • Rustacean Station: Un lloc impulsat per la comunitat amb una llista de projectes en Rust que busquen col·laboradors.
  • Good First Issues: Molts repositoris etiqueten problemes fàcils per a principiants amb etiquetes com good first issue o beginner.

Exemple: Cercar Projectes en Rust a GitHub

1. Ves a [GitHub](https://github.com).
2. A la barra de cerca, escriu `language:rust` per filtrar projectes escrits en Rust.
3. Explora repositoris i busca aquells amb problemes actius i contribucions.
4. Revisa la pestanya `Issues` per etiquetes com `good first issue`.

Entendre el Projecte

Un cop hagis trobat un projecte, és important entendre la seva estructura i les seves directrius.

  • Llegeix el README: El fitxer README normalment conté una visió general del projecte, instruccions d'instal·lació i exemples d'ús.
  • Revisa el CONTRIBUTING.md: Aquest fitxer proporciona directrius per contribuir, incloent estàndards de codificació, convencions de nomenclatura de branques i com enviar una sol·licitud d'extracció.
  • Explora el Codi: Familiaritza't amb l'estructura del projecte, els mòduls principals i com està organitzat el codi.

Exemple: Explorant l'Estructura d'un Projecte en Rust

1. Clona el repositori:
   ```sh
   git clone https://github.com/username/project.git
   cd project
   ```
2. Obre el projecte al teu editor favorit (per exemple, VSCode):
   ```sh
   code .
   ```
3. Explora el directori `src` per entendre els mòduls principals i les seves responsabilitats.
4. Mira el fitxer `Cargo.toml` per veure les dependències i metadades del projecte.

Fer la Teva Primera Contribució

Després d'entendre el projecte, pots començar a contribuir. Aquests són els passos típics:

  • Fes un Fork del Repositori: Crea una còpia personal del repositori al teu compte de GitHub.
  • Clona el Teu Fork: Descarrega el teu fork a la teva màquina local.
  • Crea una Branca: Sempre crea una nova branca per als teus canvis.
  • Fes Canvis: Implementa els teus canvis, seguint els estàndards de codificació del projecte.
  • Fes un Commit i Push: Fes commit dels teus canvis i puja'ls al teu fork.
  • Crea una Sol·licitud d'Extracció: Envia els teus canvis per a revisió.

Exemple: Fer una Contribució Simple

1. Fes un fork del repositori a GitHub.
2. Clona el teu fork:
   ```sh
   git clone https://github.com/yourusername/project.git
   cd project
   ```
3. Crea una nova branca:
   ```sh
   git checkout -b fix-issue-123
   ```
4. Fes els teus canvis a l'editor de codi.
5. Fes commit dels teus canvis:
   ```sh
   git add .
   git commit -m "Fix issue #123: Corrected typo in README"
   ```
6. Puja els teus canvis al teu fork:
   ```sh
   git push origin fix-issue-123
   ```
7. Crea una sol·licitud d'extracció a GitHub navegant al teu fork i fent clic al botó `New pull request`.

Revisar i Respondre als Comentaris

Després d'enviar una sol·licitud d'extracció, els mantenidors del projecte revisaran els teus canvis. Estigues preparat per:

  • Respondre als Comentaris: Aborda qualsevol comentari o pregunta dels mantenidors.
  • Fer Canvis Addicionals: Potser necessitaràs fer més canvis basats en la revisió.
  • Mantenir-te Compromès: Mantingues un ull a la teva sol·licitud d'extracció i respon ràpidament.

Exemple: Respondre als Comentaris

1. Revisa la pestanya `Pull Requests` al repositori original per a comentaris sobre la teva PR.
2. Si es demanen canvis, fes-los a la teva branca local:
   ```sh
   git checkout fix-issue-123
   git add .
   git commit -m "Addressed feedback: Fixed typo in README"
   git push origin fix-issue-123
   ```
3. Comenta a la sol·licitud d'extracció per notificar als mantenidors que has fet els canvis.

Conclusió

Contribuir a projectes de codi obert en Rust és una excel·lent manera de millorar les teves habilitats, col·laborar amb altres desenvolupadors i contribuir a la comunitat. Seguint els passos descrits en aquesta secció, pots trobar un projecte, entendre la seva estructura, fer contribucions significatives i comprometre't amb els mantenidors i altres col·laboradors. Feliç codificació!
