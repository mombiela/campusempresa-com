Introducció

En Rust, els paquets i crates són conceptes fonamentals per organitzar i gestionar el codi. Entendre aquests conceptes és crucial per construir aplicacions Rust escalables i mantenibles. Aquesta secció cobrirà els conceptes bàsics dels paquets i crates, com crear-los i utilitzar-los, i les millors pràctiques per gestionar dependències.

Què és un Crate?

Un crate és la unitat més petita de distribució de codi en Rust. Pot ser una biblioteca o un binari. Els crates es compilen en fitxers binaris o de biblioteca, i formen els blocs de construcció dels programes Rust.

  • Crates Binàries: Aquests són programes executables. Cada crate binari té una funció main que serveix com a punt d'entrada.
  • Crates de Biblioteca: Aquests proporcionen funcionalitat que es pot compartir entre múltiples projectes. No tenen una funció main.

Crear un Crate

Per crear un nou crate, pots utilitzar la comanda cargo new. Per exemple:

cargo new my_crate --lib

Aquesta comanda crea un nou crate de biblioteca anomenat my_crate. L'estructura del directori serà així:

my_crate/
├── Cargo.toml
└── src
    └── lib.rs
  • Cargo.toml: Aquest fitxer conté metadades sobre el crate, com el seu nom, versió i dependències.
  • src/lib.rs: Aquest és el fitxer de codi font principal per al crate de biblioteca.

Què és un Paquet?

Un paquet és una col·lecció d'un o més crates. Inclou un fitxer Cargo.toml que descriu com construir aquests crates. Un paquet pot contenir zero o un crate de biblioteca i qualsevol nombre de crates binaris.

Crear un Paquet

Per crear un nou paquet, pots utilitzar la comanda cargo new sense el flag --lib:

cargo new my_package

Aquesta comanda crea un nou paquet amb un crate binari. L'estructura del directori serà així:

my_package/
├── Cargo.toml
└── src
    └── main.rs
  • Cargo.toml: Aquest fitxer conté metadades sobre el paquet i les seves dependències.
  • src/main.rs: Aquest és el fitxer de codi font principal per al crate binari.

Gestionar Dependències

Les dependències es especifiquen en el fitxer Cargo.toml. Pots afegir dependències al teu paquet editant aquest fitxer.

Afegir una Dependència

Per afegir una dependència, pots utilitzar la comanda cargo add o editar manualment el fitxer Cargo.toml. Per exemple, per afegir el crate serde per a serialització i deserialització:

cargo add serde

O afegir-lo manualment a Cargo.toml:

[dependencies]
serde = "1.0"

Utilitzar una Dependència

Un cop afegida una dependència, pots utilitzar-la en el teu codi incloent-la amb la directiva extern crate (per a versions antigues de Rust) o utilitzant la declaració use (per a versions més noves de Rust).

// src/main.rs
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize)]
struct MyStruct {
    field1: String,
    field2: i32,
}

fn main() {
    let my_struct = MyStruct {
        field1: String::from("Hello"),
        field2: 42,
    };
    // Codi de serialització i deserialització aquí
}

Millors Pràctiques

  • Disseny Modular: Divideix el teu codi en crates més petits i reutilitzables per millorar la mantenibilitat.
  • Control de Versions: Utilitza la versió semàntica per als teus crates per gestionar els canvis i les dependències de manera efectiva.
  • Documentació: Documenta els teus crates i paquets a fons per fer-los més fàcils d'utilitzar i entendre.

Conclusió

Entendre els paquets i crates és essencial per a una programació efectiva en Rust. Els crates són els blocs de construcció de les aplicacions Rust, mentre que els paquets t'ajuden a gestionar i distribuir aquests crates. Seguint les millors pràctiques i utilitzant eines com Cargo, pots crear aplicacions Rust modulars, mantenibles i escalables.
