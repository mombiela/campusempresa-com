Gestió d'Errors amb Result
Gestió d'Errors amb Option ►

La gestió d'errors és un aspecte crucial de la programació, assegurant que el teu codi pugui manejar situacions inesperades de manera elegant. En Rust, el tipus Result és una eina poderosa per gestionar errors. Aquest tema et guiarà des dels conceptes bàsics fins a l'ús avançat de Result per a la gestió d'errors en Rust.

Introducció a `Result`

El tipus Result és un enum que representa èxit (Ok) o fracàs (Err). Es defineix com:

enum Result<T, E> {
    Ok(T),
    Err(E),
}
  • T: El tipus del valor retornat en cas d'èxit.
  • E: El tipus de l'error retornat en cas de fracàs.

Ús Bàsic

Comencem amb un exemple senzill:

fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("Divisió per zero"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    match divide(4.0, 2.0) {
        Ok(result) => println!("Resultat: {}", result),
        Err(e) => println!("Error: {}", e),
    }

    match divide(4.0, 0.0) {
        Ok(result) => println!("Resultat: {}", result),
        Err(e) => println!("Error: {}", e),
    }
}

En aquest exemple:

  • La funció divide retorna un Result<f64, String>.
  • Si la divisió té èxit, retorna Ok(result).
  • Si hi ha un error (divisió per zero), retorna Err(error_message).

Gestionar `Result` amb `match`

L'instrucció match és una manera comuna de gestionar Result:

fn main() {
    let result = divide(10.0, 2.0);

    match result {
        Ok(value) => println!("Èxit: {}", value),
        Err(error) => println!("Error: {}", error),
    }
}

Ús de `unwrap` i `expect`

Per a prototips ràpids, pots utilitzar unwrap i expect per gestionar Result:

  • unwrap: Retorna el valor dins de Ok o pànic si és Err.
  • expect: Similar a unwrap, però permet proporcionar un missatge de pànic personalitzat.
fn main() {
    let result = divide(10.0, 2.0).unwrap();
    println!("Resultat: {}", result);

    let result = divide(10.0, 0.0).expect("No s'ha pogut dividir");
}

Nota: Utilitzar unwrap i expect no es recomana per a codi de producció ja que poden causar que el teu programa entri en pànic.

Gestió d'Errors Avançada

Enllaçar Mètodes amb `and_then` i `or_else`

Pots enllaçar múltiples operacions que retornen Result utilitzant and_then i or_else:

  • and_then: Enllaça una altra operació si el resultat és Ok.
  • or_else: Enllaça una altra operació si el resultat és Err.
fn add_one(x: f64) -> Result<f64, String> {
    Ok(x + 1.0)
}

fn main() {
    let result = divide(10.0, 2.0)
        .and_then(add_one)
        .or_else(|e| Err(format!("Fracàs: {}", e)));

    match result {
        Ok(value) => println!("Èxit: {}", value),
        Err(error) => println!("Error: {}", error),
    }
}

Ús de l'Operador `?`

L'operador ? és una forma abreujada per propagar errors. Només es pot utilitzar en funcions que retornen Result.

fn divide_and_add(a: f64, b: f64) -> Result<f64, String> {
    let result = divide(a, b)?;
    add_one(result)
}

fn main() {
    match divide_and_add(10.0, 2.0) {
        Ok(value) => println!("Èxit: {}", value),
        Err(error) => println!("Error: {}", error),
    }
}

Tipus d'Errors Personalitzats

Per a aplicacions més complexes, potser voldràs definir tipus d'errors personalitzats:

#[derive(Debug)]
enum MathError {
    DivisionByZero,
    NegativeLogarithm,
}

fn divide(a: f64, b: f64) -> Result<f64, MathError> {
    if b == 0.0 {
        Err(MathError::DivisionByZero)
    } else {
        Ok(a / b)
    }
}

fn main() {
    match divide(10.0, 0.0) {
        Ok(value) => println!("Èxit: {}", value),
        Err(error) => println!("Error: {:?}", error),
    }
}

Resum

  • El tipus Result és essencial per a la gestió d'errors en Rust.
  • Utilitza match per gestionar Result explícitament.
  • unwrap i expect es poden utilitzar per a prototips ràpids però no es recomanen per a producció.
  • L'operador ? simplifica la propagació d'errors.
  • Els tipus d'errors personalitzats proporcionen més flexibilitat i claredat en la gestió d'errors.

En dominar Result, podràs escriure codi en Rust robust i resistent als errors.
