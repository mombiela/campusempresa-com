Introducció als Punteros Crus

Els punteros crus en Rust són similars als punteros en C o C++. S'utilitzen per referenciar ubicacions de memòria directament i es denoten per *const T per a punteros immutables i *mut T per a punteros mutables. A diferència de les referències segures de Rust (&T i &mut T), els punteros crus no imposen regles de préstec o durades de vida, cosa que els fa insegurs d'utilitzar.

  • Característiques Clau dels Punteros Crus:
    • No garanteixen la seguretat de la memòria.
    • Poden ser nuls o penjants.
    • No gestionen automàticament la memòria (no hi ha desal·locació automàtica).
    • Requereixen blocs unsafe per desreferenciar.

Creació de Punteros Crus

Pots crear punteros crus a partir de referències utilitzant la paraula clau as.

fn main() {
    let x: i32 = 42;
    let r1: *const i32 = &x as *const i32; // Puntero cru immutable
    let r2: *mut i32 = &x as *mut i32;     // Puntero cru mutable

    println!("r1 apunta a: {:?}", r1);
    println!("r2 apunta a: {:?}", r2);
}

Desreferenciació de Punteros Crus

Desreferenciar punteros crus és insegur i s'ha de fer dins d'un bloc unsafe.

fn main() {
    let x: i32 = 42;
    let r: *const i32 = &x as *const i32;

    unsafe {
        println!("Valor a r: {}", *r);
    }
}

Usos dels Punteros Crus

Els punteros crus s'utilitzen típicament en escenaris on:

  • Es fa interfície amb codi C o altres llenguatges.
  • Es implementen estructures de dades de baix nivell.
  • Codi crític de rendiment on necessites control detallat sobre la memòria.

Comparació amb Referències Segures

| Característica | Referències Segures (&T, &mut T) | Punteros Crus (*const T, *mut T) | |------------------------|--------------------------------------|--------------------------------------| | Seguretat de Memòria | Enforçada | No garantida | | Nullabilitat | No permesa | Pot ser nul | | Regles de Préstec | Enforçades | No enforçades | | Durades de Vida | Enforçades | No enforçades | | Seguretat de Desreferenciació | Segura | Requereix bloc unsafe |

Exemple: Interfície amb Codi C

Els punteros crus són essencials quan es treballa amb FFI (Foreign Function Interface) per cridar funcions C des de Rust.

extern "C" {
    fn abs(input: i32) -> i32;
}

fn main() {
    let x: i32 = -42;
    unsafe {
        println!("Valor absolut de {}: {}", x, abs(x));
    }
}

Exemple: Implementació d'una Estructura de Dades de Baix Nivell

Aquí hi ha un exemple senzill d'una llista enllaçada utilitzant punteros crus.

struct Node {
    value: i32,
    next: *mut Node,
}

fn main() {
    let node1 = Box::into_raw(Box::new(Node { value: 1, next: std::ptr::null_mut() }));
    let node2 = Box::into_raw(Box::new(Node { value: 2, next: node1 }));

    unsafe {
        println!("Valor del Node 2: {}", (*node2).value);
        println!("Valor del Node 1: {}", (*(*node2).next).value);
    }

    // Neteja de memòria
    unsafe {
        Box::from_raw(node1);
        Box::from_raw(node2);
    }
}

Conclusió

Els punteros crus en Rust proporcionen una manera de treballar amb la memòria directament, oferint més control però a costa de la seguretat. Són útils en escenaris específics com interfícies amb altres llenguatges o implementació d'estructures de dades de baix nivell. No obstant això, degut a la seva naturalesa insegura, s'han d'utilitzar amb moderació i precaució. Entendre els punteros crus és crucial per a la programació avançada en Rust, especialment quan el rendiment i la interoperabilitat són crítics.
