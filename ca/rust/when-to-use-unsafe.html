Quan Utilitzar Unsafe
Punteres Crues ►

Rust és conegut per la seva forta èmfasi en la seguretat, particularment la seguretat de memòria. No obstant això, hi ha situacions on podries necessitar utilitzar codi unsafe per assolir certs objectius. Aquesta secció et guiarà per entendre quan i per què podries necessitar utilitzar unsafe en Rust, i com fer-ho de manera responsable.

Què és el Codi Unsafe?

El codi unsafe en Rust et permet realitzar operacions que el compilador de Rust no pot garantir que siguin segures. Això inclou:

  • Desreferenciar punters crus.
  • Cridar funcions o mètodes unsafe.
  • Accedir o modificar variables estàtiques mutables.
  • Implementar traits unsafe.
  • Realitzar assemblatge en línia.

Quan Utilitzar Codi Unsafe

Optimització de Rendiment

De vegades, podries necessitar utilitzar codi unsafe per optimitzar seccions del teu programa crítiques per al rendiment. Les garanties de seguretat de Rust poden introduir sobrecàrrega, i en alguns casos, evitar aquests controls pot portar a guanys significatius de rendiment.

fn sum_slice(slice: &[i32]) -> i32 {
    let mut sum = 0;
    for &item in slice {
        sum += item;
    }
    sum
}

fn sum_slice_unsafe(slice: &[i32]) -> i32 {
    let mut sum = 0;
    unsafe {
        let ptr = slice.as_ptr();
        for i in 0..slice.len() {
            sum += *ptr.add(i);
        }
    }
    sum
}

En l'exemple anterior, sum_slice_unsafe utilitza punters crus per iterar sobre el slice, cosa que pot ser més ràpid que la iteració segura en sum_slice.

Interfície amb Altres Llenguatges

Quan interacciones amb altres llenguatges de programació, com C, sovint necessites utilitzar codi unsafe. Això és perquè Rust no pot garantir la seguretat de les interfícies de funcions estrangeres (FFI).

extern "C" {
    fn abs(input: i32) -> i32;
}

fn main() {
    let x = -42;
    unsafe {
        println!("El valor absolut de {} és {}", x, abs(x));
    }
}

Aquí, la funció abs de la llibreria estàndard de C es crida dins d'un bloc unsafe.

Manipulació de Memòria de Baix Nivell

En alguns casos, necessites realitzar manipulació de memòria de baix nivell que les abstraccions segures de Rust no permeten.

fn copy_nonoverlapping(src: *const u8, dst: *mut u8, count: usize) {
    unsafe {
        std::ptr::copy_nonoverlapping(src, dst, count);
    }
}

Aquesta funció utilitza std::ptr::copy_nonoverlapping per copiar memòria d'una ubicació a una altra, cosa que és inherentment insegura.

Directrius per Utilitzar Codi Unsafe

  • Minimitza el Codi Unsafe: Utilitza unsafe només quan sigui absolutament necessari. Mantingues el codi unsafe tan petit i aïllat com sigui possible.
  • Revisa i Prova Exhaustivament: El codi unsafe pot introduir errors subtils. Revisa'l amb cura i escriu proves extenses.
  • Documenta Clarament: Documenta clarament per què el codi és unsafe i quines invariants s'han de mantenir.

Conclusió

Utilitzar unsafe en Rust és una eina poderosa que s'ha d'utilitzar amb moderació i precaució. Et permet realitzar operacions que no són possibles en Rust segur, però comporta el risc d'introduir errors i comportament indefinit. Sempre intenta escriure codi segur en Rust primer, i recorre a unsafe només quan tinguis una raó convincent i entenguis els riscos implicats.
