Introducció

En aquest mòdul, explorarem com construir una aplicació d'Interfície de Línia de Comandes (CLI) utilitzant Rust. Les aplicacions CLI són eines poderoses que permeten als usuaris interactuar amb el programari a través de comandes de text. Al final d'aquest mòdul, seràs capaç de crear una aplicació CLI totalment funcional en Rust.

Prerequisits

Abans de començar aquest mòdul, hauries de tenir:

  • Comprensió bàsica de la sintaxi i els conceptes de Rust.
  • Rust instal·lat a la teva màquina.
  • Familiaritat amb l'ús del terminal o la línia de comandes.

Configurant el Teu Projecte

Per començar, necessitem crear un nou projecte Rust. Obre el teu terminal i executa la següent comanda:

cargo new cli_app
cd cli_app

Aquesta comanda crea un nou projecte Rust anomenat cli_app i navega dins del directori del projecte.

Afegint Dependències

Per construir aplicacions CLI, utilitzarem el crate clap, que proporciona una manera senzilla de parsejar arguments de línia de comandes.

Afegeix el següent al teu fitxer Cargo.toml:

[dependencies]
clap = "3.0"

Aplicació CLI Bàsica

Comencem creant una aplicació CLI bàsica que accepti un únic argument i el mostri.

main.rs

use clap::Parser;

/// Programa senzill per saludar una persona
#[derive(Parser)]
struct Cli {
    /// El nom de la persona a saludar
    name: String,
}

fn main() {
    let args = Cli::parse();
    println!("Hola, {}!", args.name);
}

Explicació

  • use clap::Parser;: Importa el trait Parser del crate clap.
  • #[derive(Parser)]: Deriva el trait Parser per a l'estructura Cli, permetent-li parsejar arguments de línia de comandes.
  • struct Cli: Defineix una estructura amb un únic camp name per mantenir l'argument.
  • Cli::parse(): Parseja els arguments de línia de comandes i retorna una instància de Cli.
  • println!: Mostra un missatge de salutació utilitzant el nom proporcionat.

Gestionant Múltiples Arguments

A continuació, ampliem la nostra aplicació per gestionar múltiples arguments i opcions.

main.rs

use clap::{Parser, ArgEnum};

#[derive(Parser)]
struct Cli {
    /// El nom de la persona a saludar
    name: String,
    /// El nombre de vegades a saludar
    #[clap(short, long, default_value_t = 1)]
    count: u8,
    /// L'idioma a utilitzar per a la salutació
    #[clap(arg_enum, short, long, default_value_t = Language::English)]
    language: Language,
}

#[derive(Copy, Clone, PartialEq, Eq, PartialOrd, Ord, ArgEnum)]
enum Language {
    English,
    Spanish,
    French,
}

fn main() {
    let args = Cli::parse();
    for _ in 0..args.count {
        match args.language {
            Language::English => println!("Hello, {}!", args.name),
            Language::Spanish => println!("Hola, {}!", args.name),
            Language::French => println!("Bonjour, {}!", args.name),
        }
    }
}

Explicació

  • #[clap(short, long, default_value_t = 1)]: Especifica que count es pot establir utilitzant -c o --count i per defecte és 1.
  • #[clap(arg_enum, short, long, default_value_t = Language::English)]: Especifica que language es pot establir utilitzant -l o --language i per defecte és English.
  • enum Language: Defineix una enumeració per als idiomes suportats.
  • match args.language: Compara l'idioma proporcionat i mostra la salutació en l'idioma corresponent.

Gestió d'Errors

La gestió adequada d'errors és crucial per a aplicacions CLI robustes. Afegim gestió d'errors a la nostra aplicació.

main.rs

use clap::{Parser, ArgEnum};
use std::error::Error;

#[derive(Parser)]
struct Cli {
    name: String,
    #[clap(short, long, default_value_t = 1)]
    count: u8,
    #[clap(arg_enum, short, long, default_value_t = Language::English)]
    language: Language,
}

#[derive(Copy, Clone, PartialEq, Eq, PartialOrd, Ord, ArgEnum)]
enum Language {
    English,
    Spanish,
    French,
}

fn main() -> Result<(), Box<dyn Error>> {
    let args = Cli::parse();
    for _ in 0..args.count {
        match args.language {
            Language::English => println!("Hello, {}!", args.name),
            Language::Spanish => println!("Hola, {}!", args.name),
            Language::French => println!("Bonjour, {}!", args.name),
        }
    }
    Ok(())
}

Explicació

  • Result<(), Box<dyn Error>>: Canvia el tipus de retorn de main a un Result per gestionar possibles errors.
  • Ok(()): Indica una execució exitosa.

Funcionalitats Avançades

Per a aplicacions CLI avançades, podries necessitar subcomandes, fitxers de configuració o un parsejat d'arguments més complex. El crate clap suporta aquestes funcionalitats.

Exemple de Subcomandes

use clap::{Parser, Subcommand};

#[derive(Parser)]
struct Cli {
    #[clap(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    Greet {
        name: String,
    },
    Farewell {
        name: String,
    },
}

fn main() {
    let args = Cli::parse();
    match args.command {
        Commands::Greet { name } => println!("Hello, {}!", name),
        Commands::Farewell { name } => println!("Goodbye, {}!", name),
    }
}

Explicació

  • #[clap(subcommand)]: Especifica que command és una subcomanda.
  • enum Commands: Defineix les subcomandes Greet i Farewell.

Conclusió

En aquest mòdul, hem cobert els conceptes bàsics de construir una aplicació CLI en Rust utilitzant el crate clap. Hem començat amb una aplicació senzilla, hem afegit múltiples arguments i opcions, hem implementat la gestió d'errors, i hem explorat funcionalitats avançades com les subcomandes. Amb aquestes habilitats, pots crear aplicacions CLI poderoses i fàcils d'utilitzar en Rust.
