Introducció a l'estat compartit

A Rust, gestionar l'estat compartit entre fils o diferents parts d'un programa pot ser un repte a causa de les estrictes regles de propietat i préstec de Rust. No obstant això, Rust proporciona diverses eines per manejar l'estat compartit de manera segura i eficient.

  • Propietat i préstec: El sistema de propietat de Rust garanteix la seguretat de la memòria sense necessitat d'un col·lector d'escombraries.
  • Concurrència: El model de concurrència de Rust aprofita la propietat per evitar carreres de dades en temps de compilació.

Conceptes clau

Resum de propietat i préstec

Abans d'endinsar-nos en l'estat compartit, fem un breu resum de les regles de propietat i préstec de Rust:

  • Propietat: Cada valor a Rust té un únic propietari.
  • Préstec: Es poden prestar referències a un valor, ja sigui de manera mutable o immutable.
  • Durades de vida: Les durades de vida asseguren que les referències siguin vàlides mentre s'utilitzin.

Mecanismes d'estat compartit

Rust proporciona diversos mecanismes per manejar l'estat compartit:

  • Rc<T> i Arc<T>: Punteres intel·ligents amb recompte de referències per a escenaris de fil únic i multi-fil, respectivament.
  • RefCell<T> i Mutex<T>: Tipus de mutabilitat interior que permeten l'accés mutable a dades compartides.
  • RwLock<T>: Un bloqueig de lectura-escriptura que permet múltiples lectors o un únic escriptor.

Ús de `Rc` per a estat compartit de fil únic

Rc<T> significa Reference Counted. Permet la propietat múltiple de dades en un context de fil únic.

use std::rc::Rc;

fn main() {
    let a = Rc::new(5);
    let b = Rc::clone(&a);
    println!("a: {}, b: {}", a, b);
}
  • Explicació: Rc::new crea un nou valor amb recompte de referències. Rc::clone augmenta el recompte de referències, permetent la propietat múltiple.

Ús de `Arc` per a estat compartit multi-fil

Arc<T> significa Atomic Reference Counted. És similar a Rc<T> però segur d'utilitzar entre fils.

use std::sync::Arc;
use std::thread;

fn main() {
    let a = Arc::new(5);
    let b = Arc::clone(&a);

    let handle = thread::spawn(move || {
        println!("b: {}", b);
    });

    println!("a: {}", a);
    handle.join().unwrap();
}
  • Explicació: Arc::new crea un nou valor amb recompte de referències atòmic. Arc::clone funciona de manera similar a Rc::clone però és segur per a fils.

Mutabilitat interior amb `RefCell` i `Mutex`

`RefCell` per a contextos de fil únic

RefCell<T> permet el préstec mutable comprovat en temps d'execució.

use std::cell::RefCell;

fn main() {
    let a = RefCell::new(5);
    *a.borrow_mut() += 1;
    println!("a: {}", a.borrow());
}
  • Explicació: RefCell::new crea un nou RefCell. borrow_mut i borrow permeten el préstec mutable i immutable, respectivament.

`Mutex` per a contextos multi-fil

Mutex<T> proporciona exclusió mútua, assegurant que només un fil pugui accedir a les dades alhora.

use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let a = Arc::new(Mutex::new(5));
    let b = Arc::clone(&a);

    let handle = thread::spawn(move || {
        let mut num = b.lock().unwrap();
        *num += 1;
    });

    handle.join().unwrap();
    println!("a: {:?}", a.lock().unwrap());
}
  • Explicació: Mutex::new crea un nou mutex. lock adquireix el bloqueig, permetent l'accés a les dades.

Bloqueigs de lectura-escriptura amb `RwLock`

RwLock<T> permet múltiples lectors o un únic escriptor.

use std::sync::{Arc, RwLock};
use std::thread;

fn main() {
    let a = Arc::new(RwLock::new(5));
    let b = Arc::clone(&a);

    let handle = thread::spawn(move || {
        let mut num = b.write().unwrap();
        *num += 1;
    });

    handle.join().unwrap();
    println!("a: {:?}", a.read().unwrap());
}
  • Explicació: RwLock::new crea un nou bloqueig de lectura-escriptura. Els mètodes read i write adquireixen bloqueigs de lectura i escriptura, respectivament.

Conclusió

Gestionar l'estat compartit a Rust requereix entendre i aprofitar les seves regles de propietat i préstec. Rust proporciona eines poderoses com Rc<T>, Arc<T>, RefCell<T>, Mutex<T> i RwLock<T> per manejar l'estat compartit de manera segura i eficient. Utilitzant aquestes eines, pots escriure programes concurrents i paral·lels que siguin tant segurs com eficients.
