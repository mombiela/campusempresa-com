Genèrics en Rust
Fils ►

Els genèrics són una característica poderosa en Rust que et permet escriure codi flexible i reutilitzable. Utilitzant genèrics, pots crear funcions, structs, enums i traits que poden operar sobre molts tipus diferents sense sacrificar rendiment o seguretat.

Conceptes Clau

  • Tipus Genèrics: Tipus de marcador de posició que es poden especificar quan s'utilitza la funció, struct o enum.
  • Paràmetres de Tipus: Els tipus reals que substitueixen els tipus genèrics quan es compila el codi.
  • Traits i Límits: Restriccions que especifiquen quines capacitats ha de tenir un tipus per ser utilitzat com a tipus genèric.

Funcions Genèriques

Les funcions genèriques et permeten escriure una funció que pot operar sobre diferents tipus sense duplicar codi.

Exemple

fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list.iter() {
        if item > largest {
            largest = item;
        }
    }
    largest
}

En aquest exemple:

  • T és un tipus genèric.
  • PartialOrd és un trait que assegura que el tipus T es pot comparar utilitzant l'operador >.

Explicació

  • fn largest<T: PartialOrd>(list: &[T]) -> &T declara una funció anomenada largest que pren una slice de tipus T i retorna una referència a un element de tipus T.
  • T: PartialOrd és un límit de trait que assegura que els elements es poden comparar.

Structs Genèrics

Pots definir structs amb tipus genèrics per crear estructures de dades que poden emmagatzemar diferents tipus de dades.

Exemple

struct Point<T> {
    x: T,
    y: T,
}

impl<T> Point<T> {
    fn new(x: T, y: T) -> Self {
        Point { x, y }
    }
}

Explicació

  • struct Point<T> defineix un struct amb un tipus genèric T.
  • impl<T> Point<T> implementa mètodes per al struct Point.
  • fn new(x: T, y: T) -> Self és un mètode constructor que inicialitza un Point amb valors de tipus T.

Enums Genèrics

Els enums també poden ser genèrics, permetent-te definir variants que poden contenir diferents tipus.

Exemple

enum Option<T> {
    Some(T),
    None,
}

Explicació

  • enum Option<T> defineix un enum amb un tipus genèric T.
  • Some(T) i None són les variants de l'enum. Some conté un valor de tipus T, mentre que None no conté cap valor.

Traits i Límits

Els traits defineixen comportaments compartits que els tipus poden implementar. Pots utilitzar límits de trait per especificar que un tipus genèric ha d'implementar un trait particular.

Exemple

fn print_area<T: Area>(shape: &T) {
    println!("The area is {}", shape.area());
}

trait Area {
    fn area(&self) -> f64;
}

struct Circle {
    radius: f64,
}

impl Area for Circle {
    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }
}

Explicació

  • fn print_area<T: Area>(shape: &T) defineix una funció que pren una referència a un tipus genèric T que implementa el trait Area.
  • trait Area defineix un trait amb un mètode area.
  • struct Circle defineix un struct que representa un cercle.
  • impl Area for Circle implementa el trait Area per al struct Circle.

Resum

Els genèrics en Rust et permeten escriure codi flexible, reutilitzable i segur en termes de tipus. Utilitzant tipus genèrics, pots crear funcions, structs, enums i traits que funcionen amb molts tipus diferents mentre mantens el rendiment i la seguretat. Entendre els genèrics és crucial per escriure codi idiomàtic i eficient en Rust.
