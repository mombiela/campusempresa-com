Executar Proves
Depuració ►

Les proves són una part crucial del desenvolupament de programari, assegurant que el teu codi es comporti com s'espera. Rust proporciona un potent marc de proves integrat en el llenguatge. Aquesta secció et guiarà a través dels conceptes bàsics d'escriure i executar proves en Rust, progressant cap a tècniques de proves més avançades.

Introducció a les Proves en Rust

El marc de proves de Rust està integrat en el llenguatge, fent que sigui fàcil escriure i executar proves. Les proves són funcions que verifiquen la correcció del teu codi.

  • Funcions de Prova: Funcions anotades amb l'atribut #[test].
  • Afirmacions de Prova: Macros com assert!, assert_eq! i assert_ne! per comprovar condicions.

Exemple: Funció de Prova Bàsica

Aquí tens un exemple senzill d'una funció de prova:

// src/lib.rs

pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
    }
}
  • #[cfg(test)]: Aquest atribut assegura que el mòdul de prova només es compili quan s'executen proves.
  • #[test]: Marca la funció com a funció de prova.
  • assert_eq!: Comprova si els dos arguments són iguals.

Executar Proves

Per executar proves, utilitza la comanda cargo test. Aquesta comanda compila el teu codi en mode de prova i executa totes les funcions de prova.

$ cargo test

Sortida de `cargo test`

Quan executis cargo test, veuràs una sortida similar a aquesta:

running 1 test
test tests::test_add ... ok

test result: ok. 1 passed; 0 failed; 0 ignored; 0 measured; 0 filtered out
  • Resum de Proves: Mostra el nombre de proves passades, fallades, ignorades, etc.

Escriure Proves Més Complexes

A mesura que el teu codi creixi, necessitaràs proves més complexes. Rust suporta diverses estratègies de prova:

  • Proves Unitàries: Prova unitats individuals de codi (funcions, mètodes).
  • Proves d'Integració: Prova la interacció entre múltiples parts del teu codi.

Proves Unitàries

Les proves unitàries s'escriuen en el mateix fitxer que el codi que proven, típicament dins d'un mòdul tests.

// src/lib.rs

pub fn multiply(a: i32, b: i32) -> i32 {
    a * b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_multiply() {
        assert_eq!(multiply(2, 3), 6);
    }
}

Proves d'Integració

Les proves d'integració es col·loquen en el directori tests a l'arrel del teu projecte. Cada fitxer en aquest directori es compila com un crate separat.

// tests/integration_test.rs

extern crate my_crate;

#[test]
fn test_integration() {
    assert_eq!(my_crate::add(2, 3), 5);
}
  • extern crate: S'utilitza per portar la biblioteca a l'abast.

Tècniques de Prova Avançades

Ignorar Proves

De vegades potser voldràs desactivar temporalment una prova. Utilitza l'atribut #[ignore].

#[test]
#[ignore]
fn expensive_test() {
    // codi que triga molt a executar-se
}

Executar Proves Específiques

Per executar una prova específica, utilitza el flag --test seguit del nom de la prova.

$ cargo test test_add

Fixtures de Prova

Les fixtures de prova són configuracions comunes per a múltiples proves. Rust no té suport integrat per a fixtures, però pots aconseguir-ho utilitzant funcions de configuració.

#[cfg(test)]
mod tests {
    use super::*;

    fn setup() -> i32 {
        // codi de configuració
        42
    }

    #[test]
    fn test_with_setup() {
        let value = setup();
        assert_eq!(value, 42);
    }
}

Conclusió

Les proves són una part integral de la programació en Rust, assegurant que el teu codi sigui fiable i mantenible. El marc de proves integrat de Rust fa que sigui fàcil escriure i executar proves, des de simples proves unitàries fins a complexes proves d'integració. En dominar aquestes tècniques de prova, estaràs ben equipat per desenvolupar aplicacions robustes en Rust.

  • Punts Clau:
    • Utilitza #[test] per marcar funcions de prova.
    • Executa proves amb cargo test.
    • Escriu proves unitàries en el mateix fitxer i proves d'integració en el directori tests.
    • Utilitza tècniques avançades com ignorar proves i configurar fixtures per a escenaris més complexos.
