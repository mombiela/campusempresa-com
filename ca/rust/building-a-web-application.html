Construir una Aplicació Web en Rust
Interfície amb Altres Llenguatges ►

Introducció

Construir una aplicació web en Rust pot ser una experiència gratificant a causa del rendiment, la seguretat i les característiques de concurrència del llenguatge. Aquesta secció et guiarà des dels conceptes bàsics fins a temes avançats, assegurant que tinguis una comprensió sòlida de com crear una aplicació web utilitzant Rust.

Configurant el Teu Entorn

Abans de començar a codificar, necessitem configurar el nostre entorn de desenvolupament.

  • Instal·lar Rust: Segueix les instruccions al lloc web oficial de Rust.
  • Instal·lar Cargo: Cargo és el gestor de paquets i sistema de construcció de Rust, que ve amb Rust.
  • IDE/Editor: Utilitza un IDE com Visual Studio Code amb l'extensió de Rust per a una millor experiència de desenvolupament.

Creant un Nou Projecte Rust

Creem un nou projecte Rust utilitzant Cargo.

cargo new my_web_app
cd my_web_app

Aquesta comanda crea un nou directori anomenat my_web_app amb una estructura bàsica de projecte Rust.

Afegint Dependències

Per construir una aplicació web, necessitem afegir algunes dependències al nostre fitxer Cargo.toml.

[dependencies]
actix-web = "4"
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
  • actix-web: Un marc web potent, pragmàtic i extremadament ràpid per a Rust.
  • serde: Un marc per a la serialització i deserialització d'estructures de dades de Rust.
  • serde_json: Una biblioteca per treballar amb dades JSON.

Servidor Web Bàsic

Comencem creant un servidor web bàsic.

use actix_web::{web, App, HttpServer, Responder};

async fn hello() -> impl Responder {
    "Hola, món!"
}

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    HttpServer::new(|| {
        App::new()
            .route("/", web::get().to(hello))
    })
    .bind("127.0.0.1:8080")?
    .run()
    .await
}
  • HttpServer: Crea un nou servidor HTTP.
  • App: Configura l'aplicació.
  • route: Defineix una ruta per a l'aplicació.
  • hello: Una funció manejadora simple que retorna "Hola, món!".

Gestionant Peticions i Respostes

Gestionem peticions i respostes més complexes.

Analitzant Paràmetres de Consulta

use actix_web::{web, App, HttpServer, Responder};

async fn greet(name: web::Query<String>) -> impl Responder {
    format!("Hola, {}!", name)
}

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    HttpServer::new(|| {
        App::new()
            .route("/greet", web::get().to(greet))
    })
    .bind("127.0.0.1:8080")?
    .run()
    .await
}

Gestionant JSON

use actix_web::{web, App, HttpServer, Responder};
use serde::{Deserialize, Serialize};

#[derive(Serialize, Deserialize)]
struct MyObj {
    name: String,
}

async fn echo(item: web::Json<MyObj>) -> impl Responder {
    web::Json(item.into_inner())
}

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    HttpServer::new(|| {
        App::new()
            .route("/echo", web::post().to(echo))
    })
    .bind("127.0.0.1:8080")?
    .run()
    .await
}

Middleware

El middleware es pot utilitzar per modificar peticions i respostes.

use actix_web::{web, App, HttpServer, Responder, middleware::Logger};

async fn hello() -> impl Responder {
    "Hola, món!"
}

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    env_logger::init();
    HttpServer::new(|| {
        App::new()
            .wrap(Logger::default())
            .route("/", web::get().to(hello))
    })
    .bind("127.0.0.1:8080")?
    .run()
    .await
}

Integració de Base de Dades

Integrar una base de dades és crucial per a la majoria d'aplicacions web. Utilitzarem SQLite per simplicitat.

Afegint Dependències

Afegeix el següent al teu Cargo.toml:

[dependencies]
diesel = { version = "1.4.8", features = ["sqlite"] }
dotenv = "0.15"

Configurant Diesel

Inicialitza Diesel amb SQLite:

cargo install diesel_cli --no-default-features --features sqlite
diesel setup

Creant un Model

Defineix un model i un esquema.

#[macro_use]
extern crate diesel;
extern crate dotenv;

use diesel::prelude::*;
use dotenv::dotenv;
use std::env;

#[derive(Queryable)]
struct User {
    id: i32,
    name: String,
}

table! {
    users (id) {
        id -> Integer,
        name -> Text,
    }
}

fn main() {
    dotenv().ok();
    let database_url = env::var("DATABASE_URL").expect("DATABASE_URL must be set");
    let connection = SqliteConnection::establish(&database_url)
        .expect(&format!("Error connecting to {}", database_url));
}

Temes Avançats

Programació Asíncrona

La sintaxi async/await de Rust permet una programació asíncrona eficient.

use actix_web::{web, App, HttpServer, Responder};

async fn index() -> impl Responder {
    "Hola, món asíncron!"
}

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    HttpServer::new(|| {
        App::new()
            .route("/", web::get().to(index))
    })
    .bind("127.0.0.1:8080")?
    .run()
    .await
}

Gestió d'Errors

Una gestió d'errors adequada és crucial per a aplicacions robustes.

use actix_web::{web, App, HttpServer, Responder, Result, HttpResponse};

async fn index() -> Result<impl Responder> {
    Ok(HttpResponse::Ok().body("Hola, món!"))
}

#[actix_web::main]
async fn main() -> std::io::Result<()> {
    HttpServer::new(|| {
        App::new()
            .route("/", web::get().to(index))
    })
    .bind("127.0.0.1:8080")?
    .run()
    .await
}

Conclusió

Construir una aplicació web en Rust implica configurar el teu entorn, crear un servidor web bàsic, gestionar peticions i respostes, integrar middleware, connectar-se a una base de dades i entendre temes avançats com la programació asíncrona i la gestió d'errors. Amb les potents característiques de Rust, pots construir aplicacions web eficients i segures.
