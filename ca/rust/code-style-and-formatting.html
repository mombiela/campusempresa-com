Introducció

L'estil i el format del codi són aspectes crucials per escriure codi net, llegible i mantenible. En Rust, adherir-se a un estil consistent ajuda a assegurar que el teu codi sigui comprensible no només per a tu, sinó també per a altres desenvolupadors que puguin treballar en el teu codi. Aquesta secció cobrirà els principis clau de l'estil i el format del codi en Rust, incloent convencions de nomenclatura, indentació i l'ús de l'eina de format integrada de Rust, rustfmt.

Convencions de Nomenclatura

Les convencions de nomenclatura en Rust ajuden a fer el codi més llegible i consistent. Aquí tens les convencions clau:

  • Variables i Funcions: Usa snake_case.

    let my_variable = 10;
fn my_function() {
    // cos de la funció
}

  • Constants i Estàtics: Usa SCREAMING_SNAKE_CASE.

    const MAX_SIZE: u32 = 100;
static GLOBAL_COUNT: u32 = 0;

  • Structs, Enums i Traits: Usa CamelCase.

    struct MyStruct {
    field: u32,
}

enum MyEnum {
    VariantOne,
    VariantTwo,
}

trait MyTrait {
    fn do_something(&self);
}

Indentació i Espaiat

Una correcta indentació i espaiat fan que el teu codi sigui més llegible. Rust utilitza 4 espais per nivell d'indentació.

  • Indentació: Usa 4 espais, no tabuladors.

    fn main() {
    let x = 5;
    if x > 0 {
        println!("x és positiu");
    }
}

  • Espaiat: Usa espais al voltant dels operadors i després de les comes.

    let sum = a + b;
let array = [1, 2, 3, 4];

Longitud de la Línia

Mantingues les línies amb un màxim de 100 caràcters. Això ajuda a assegurar que el codi sigui llegible en diverses mides de pantalla i en diferents entorns.

  • Exemple: 
    // Aquesta línia té exactament 100 caràcters de llarg
let long_variable_name = "Aquesta és una cadena molt llarga que s'hauria de dividir en múltiples línies si supera els 100 caràcters.";

Comentaris

Els comentaris són essencials per explicar el propòsit del codi. Rust suporta dos tipus de comentaris:

  • Comentaris de Línia: Usa // per a comentaris d'una sola línia.

    // Aquest és un comentari d'una sola línia
let x = 5; // Aquest és un comentari en línia

  • Comentaris de Bloc: Usa /* ... */ per a comentaris de múltiples línies.

    /*
 * Aquest és un comentari de bloc.
 * Pot abastar múltiples línies.
 */
let y = 10;

Ús de `rustfmt`

Rust proporciona una eina integrada anomenada rustfmt per formatar automàticament el teu codi segons les directrius oficials d'estil de Rust. Aquesta eina ajuda a mantenir la consistència i la llegibilitat a tot el teu codi.

  • Instal·lació:

    rustup component add rustfmt

  • Ús:

    cargo fmt

Exemple de Codi Ben Formatat

Aquí tens un exemple d'un programa en Rust ben formatat:

// Aquest és un programa simple en Rust que demostra un estil i format de codi adequat.

fn main() {
    let numbers = vec![1, 2, 3, 4, 5];

    for number in &numbers {
        if is_even(*number) {
            println!("{} és parell", number);
        } else {
            println!("{} és senar", number);
        }
    }
}

fn is_even(num: i32) -> bool {
    num % 2 == 0
}

Conclusió

Adherir-se a directrius consistents d'estil i format del codi és essencial per escriure codi net, llegible i mantenible en Rust. Seguint les convencions de nomenclatura, les regles d'indentació i utilitzant eines com rustfmt, pots assegurar-te que el teu codi sigui fàcil d'entendre i treballar-hi. Recorda, un codi ben formatat no només et beneficia a tu, sinó també als teus col·laboradors i futurs mantenidors del codi.
