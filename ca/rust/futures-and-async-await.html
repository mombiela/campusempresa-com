Introducció

En aquesta secció, explorarem la programació asíncrona en Rust utilitzant Futures i la sintaxi async/await. La programació asíncrona et permet escriure programes que poden gestionar moltes tasques simultàniament, millorant el rendiment i la capacitat de resposta.

Conceptes Clau

  • Futures: Un future representa un valor que potser encara no està disponible però que es calcularà eventualment.
  • Async/Await: Sintaxi per treballar amb futures d'una manera més llegible i mantenible.
  • Concurrència vs. Paral·lelisme: La concurrència tracta de gestionar moltes tasques alhora, mentre que el paral·lelisme tracta de fer moltes tasques al mateix temps.

Futures

Què és un Future?

Un future és una abstracció per a un valor que potser encara no està disponible. Et permet escriure codi no bloquejant que pot esperar que el valor es calculi.

Exemple d'un Future

use futures::executor::block_on;

async fn hello_world() {
    println!("Hello, world!");
}

fn main() {
    let future = hello_world(); // Això no executa la funció
    block_on(future); // Això executa la funció
}
  • async fn hello_world() defineix una funció asíncrona.
  • block_on(future) executa el future fins a la seva finalització.

Async/Await

Funcions Async

Una funció async retorna un future. Pots utilitzar la paraula clau await per esperar que el future es completi.

Exemple d'Async/Await

use std::time::Duration;
use tokio::time::sleep;

async fn say_hello() {
    sleep(Duration::from_secs(2)).await;
    println!("Hello, world!");
}

#[tokio::main]
async fn main() {
    say_hello().await;
}
  • sleep(Duration::from_secs(2)).await pausa la funció durant 2 segons sense bloquejar el fil.
  • #[tokio::main] és una macro que configura el runtime de Tokio, que és necessari per executar codi asíncron.

Combinació de Futures

Unir Futures

Pots executar múltiples futures simultàniament utilitzant la macro join!.

Exemple d'Unir Futures

use tokio::time::{sleep, Duration};

async fn task_one() {
    sleep(Duration::from_secs(1)).await;
    println!("Task one completed");
}

async fn task_two() {
    sleep(Duration::from_secs(2)).await;
    println!("Task two completed");
}

#[tokio::main]
async fn main() {
    tokio::join!(task_one(), task_two());
    println!("Both tasks completed");
}
  • tokio::join!(task_one(), task_two()) executa ambdues tasques simultàniament.

Gestió d'Errors en Codi Async

Utilitzar `Result` amb Funcions Async

Les funcions asíncrones poden retornar tipus Result per a la gestió d'errors.

Exemple de Gestió d'Errors

use tokio::time::{sleep, Duration};
use std::error::Error;

async fn might_fail() -> Result<(), Box<dyn Error>> {
    sleep(Duration::from_secs(1)).await;
    Err("Something went wrong")?
}

#[tokio::main]
async fn main() {
    match might_fail().await {
        Ok(_) => println!("Success"),
        Err(e) => println!("Error: {}", e),
    }
}
  • Err("Something went wrong")? propaga l'error.
  • La sentència match gestiona el resultat del future.

Conclusió

La programació asíncrona en Rust utilitzant Futures i la sintaxi async/await et permet escriure codi eficient i no bloquejant. Entenent com definir i executar futures, gestionar errors i combinar múltiples futures, pots construir aplicacions responsives i d'alt rendiment.
