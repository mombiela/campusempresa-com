Introducció als Traits

Els traits en Rust són una característica potent que et permet definir comportaments compartits de manera abstracta. Són similars a les interfícies en altres llenguatges com Java o C#. Els traits et permeten definir mètodes que poden ser implementats per múltiples tipus, assegurant que aquests tipus adhereixin a un comportament específic.

Conceptes Clau

  • Definició de Trait: Com definir un trait.
  • Implementació de Trait: Com implementar un trait per a un tipus.
  • Límits de Trait: Com utilitzar traits per restringir tipus genèrics.
  • Implementacions per Defecte: Proporcionar comportament per defecte per als mètodes de trait.
  • Objectes de Trait: Utilitzar traits per a dispatch dinàmic.

Definint Traits

Per definir un trait, utilitzes la paraula clau trait seguida del nom del trait i un bloc que conté les signatures dels mètodes.

trait Summary {
    fn summarize(&self) -> String;
}

En aquest exemple, Summary és un trait amb un únic mètode summarize que pren una referència immutable a self i retorna un String.

Implementant Traits

Per implementar un trait per a un tipus específic, utilitzes la paraula clau impl seguida del nom del trait per al tipus.

struct Article {
    title: String,
    author: String,
    content: String,
}

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{} by {}", self.title, self.author)
    }
}

Aquí, implementem el trait Summary per a l'estructura Article. El mètode summarize retorna una cadena formatada que conté el títol i l'autor de l'article.

Límits de Trait

Els límits de trait et permeten especificar que un tipus genèric ha d'implementar un trait particular. Això és útil per escriure funcions que poden operar sobre qualsevol tipus que implementi un trait específic.

fn notify<T: Summary>(item: &T) {
    println!("Breaking news! {}", item.summarize());
}

En aquesta funció, T és un tipus genèric que ha d'implementar el trait Summary. Això assegura que qualsevol tipus passat a notify tingui un mètode summarize.

Implementacions per Defecte

Els traits poden proporcionar implementacions per defecte per als mètodes. Això permet als tipus utilitzar el comportament per defecte o sobreescriure'l amb la seva pròpia implementació.

trait Summary {
    fn summarize(&self) -> String {
        String::from("(Read more...)")
    }
}

struct Article {
    title: String,
    author: String,
    content: String,
}

impl Summary for Article {}

En aquest exemple, l'estructura Article utilitza la implementació per defecte del mètode summarize proporcionada pel trait Summary.

Objectes de Trait

Els objectes de trait permeten el dispatch dinàmic, permetent-te utilitzar traits d'una manera similar al polimorfisme en llenguatges orientats a objectes. Pots crear un objecte de trait utilitzant una referència a un trait.

fn notify(item: &dyn Summary) {
    println!("Breaking news! {}", item.summarize());
}

Aquí, &dyn Summary és un objecte de trait que pot contenir qualsevol tipus que implementi el trait Summary. Això permet un codi més flexible i dinàmic.

Conclusió

Els traits en Rust proporcionen una manera robusta de definir i compartir comportaments entre diferents tipus. Entenent com definir, implementar i utilitzar traits, pots escriure codi més modular i reutilitzable. Conceptes clau com els límits de trait i els objectes de trait milloren encara més la flexibilitat i el poder dels traits en Rust.

Resum

  • Els traits defineixen comportaments compartits.
  • Implementa traits per a tipus específics.
  • Utilitza límits de trait per restringir tipus genèrics.
  • Proporciona implementacions per defecte per als mètodes de trait.
  • Utilitza objectes de trait per a dispatch dinàmic.

Dominant aquests conceptes, estaràs ben preparat per aprofitar tot el poder dels traits en els teus programes en Rust.
