Introducció

La representació de grafs és fonamental en l'estudi d'algoritmes avançats. Un graf és una estructura matemàtica que modela relacions entre parells d'objectes. Els grafs s'utilitzen en diverses aplicacions, des de xarxes socials fins a sistemes de recomanació i anàlisi de rutes.

Tipus de Grafs

  • Graf Dirigit (Digraf): Els arcs tenen una direcció, és a dir, van d'un node a un altre.
  • Graf No Dirigit: Els arcs no tenen direcció, és a dir, la relació entre els nodes és bidireccional.
  • Graf Ponderat: Els arcs tenen un pes associat, que pot representar distància, cost, etc.
  • Graf No Ponderat: Els arcs no tenen pes associat.

Representacions de Grafs

Matriu d'Adjacència

Una matriu d'adjacència és una matriu quadrada utilitzada per representar un graf. Els elements de la matriu indiquen si parells de vèrtexs estan adjacents o no en el graf.

Avantatges

  • Accés ràpid per verificar si existeix una aresta entre dos nodes.
  • Fàcil d'implementar i entendre.

Desavantatges

  • Consumeix molt espai, especialment per a grafs dispersos.

Exemple

Per a un graf no dirigit amb 4 nodes (A, B, C, D):

| | A | B | C | D | |---|---|---|---|---| | A | 0 | 1 | 0 | 1 | | B | 1 | 0 | 1 | 0 | | C | 0 | 1 | 0 | 1 | | D | 1 | 0 | 1 | 0 |

Llista d'Adjacència

Una llista d'adjacència és una col·lecció de llistes, una per cada vèrtex del graf. Cada llista conté els nodes adjacents a aquest vèrtex.

Avantatges

  • Més eficient en termes d'espai per a grafs dispersos.
  • Fàcil d'iterar sobre els veïns d'un node.

Desavantatges

  • No és tan ràpida com la matriu d'adjacència per verificar l'existència d'una aresta.

Exemple

Per al mateix graf no dirigit amb 4 nodes (A, B, C, D):

A: B, D
B: A, C
C: B, D
D: A, C

Exemple de Codi en Python

Matriu d'Adjacència

# Representació d'un graf no dirigit usant una matriu d'adjacència
graf = [
    [0, 1, 0, 1],  # Node A
    [1, 0, 1, 0],  # Node B
    [0, 1, 0, 1],  # Node C
    [1, 0, 1, 0]   # Node D
]

# Verificar si hi ha una aresta entre A i B
print(graf[0][1])  # Sortida: 1 (Sí hi ha una aresta)

Llista d'Adjacència

# Representació d'un graf no dirigit usant una llista d'adjacència
graf = {
    'A': ['B', 'D'],
    'B': ['A', 'C'],
    'C': ['B', 'D'],
    'D': ['A', 'C']
}

# Obtenir els veïns del node A
print(graf['A'])  # Sortida: ['B', 'D']

Comparació de Representacions

| Característica | Matriu d'Adjacència | Llista d'Adjacència | |-----------------------------|---------------------|---------------------| | Espai | O(V^2) | O(V + E) | | Verificació d'aresta | O(1) | O(V) | | Iteració sobre veïns | O(V) | O(E) | | Implementació | Simple | Moderada |

Conclusió

L'elecció de la representació d'un graf depèn de les característiques del problema i de les operacions que es necessitin realitzar amb més freqüència. La matriu d'adjacència és útil per a grafs densos i quan es necessita accés ràpid a les arestes, mentre que la llista d'adjacència és més eficient en termes d'espai per a grafs dispersos i facilita la iteració sobre els veïns d'un node. Comprendre aquestes representacions i els seus avantatges i desavantatges és crucial per dissenyar algoritmes eficients en el context de grafs.
