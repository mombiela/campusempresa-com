Introducció

Els algorismes d'aproximació són tècniques utilitzades per trobar solucions properes a l'òptima per a problemes d'optimització que són difícils o impossibles de resoldre exactament en un temps raonable. Aquests algorismes són especialment útils en problemes NP-complets, on trobar la solució exacta pot ser computacionalment inviable.

Conceptes Clau

  • Problemes NP-complets: Problemes per als quals no es coneix un algorisme que pugui resoldre'ls en temps polinòmic.
  • Ràtio d'Aproximació: Mesura com de propera està la solució trobada per l'algorisme d'aproximació de la solució òptima.
  • Algorismes Greedy: Algorismes que construeixen una solució pas a pas, triant l'opció que sembla la millor en cada pas.
  • Algorismes de Programació Dinàmica: Algorismes que resolen problemes dividint-los en subproblemes més petits i combinant les seves solucions.

Exemples d'Algorismes d'Aproximació

Problema del Viatjant de Comerç (TSP)

El problema del viatjant de comerç és un problema NP-complet on es busca trobar el camí més curt que visita una sèrie de ciutats i torna al punt de partida.

Algorisme Greedy d'Aproximació

  1. Descripció: Selecciona la ciutat més propera no visitada en cada pas.
  2. Ràtio d'Aproximació: No garanteix una solució òptima, però és fàcil d'implementar.
def tsp_greedy(graph, start):
    n = len(graph)
    visited = [False] * n
    path = [start]
    visited[start] = True
    current = start

    while len(path) < n:
        next_city = None
        min_dist = float('inf')
        for city in range(n):
            if not visited[city] and graph[current][city] < min_dist:
                next_city = city
                min_dist = graph[current][city]
        path.append(next_city)
        visited[next_city] = True
        current = next_city

    path.append(start)  # Tornar al punt de partida
    return path

# Exemple d'ús
graph = [
    [0, 10, 15, 20],
    [10, 0, 35, 25],
    [15, 35, 0, 30],
    [20, 25, 30, 0]
]
start = 0
print(tsp_greedy(graph, start))

Problema de la Motxilla (Knapsack Problem)

El problema de la motxilla és un altre problema NP-complet on es busca maximitzar el valor dels objectes que es poden portar en una motxilla amb una capacitat limitada.

Algorisme d'Aproximació per Fracció

  1. Descripció: Permet fraccionar els objectes i selecciona els objectes en ordre de valor per unitat de pes.
  2. Ràtio d'Aproximació: Pot assolir una solució òptima en el cas de la versió fraccionària del problema.
def fractional_knapsack(weights, values, capacity):
    index = list(range(len(values)))
    ratio = [v/w for v, w in zip(values, weights)]
    index.sort(key=lambda i: ratio[i], reverse=True)

    max_value = 0
    fractions = [0] * len(values)
    for i in index:
        if weights[i] <= capacity:
            fractions[i] = 1
            max_value += values[i]
            capacity -= weights[i]
        else:
            fractions[i] = capacity / weights[i]
            max_value += values[i] * fractions[i]
            break

    return max_value, fractions

# Exemple d'ús
weights = [10, 20, 30]
values = [60, 100, 120]
capacity = 50
print(fractional_knapsack(weights, values, capacity))

Conclusió

Els algorismes d'aproximació són eines poderoses per abordar problemes complexos on les solucions exactes són impracticables. Encara que no sempre garanteixen una solució òptima, proporcionen solucions prou bones en un temps raonable. És crucial entendre les limitacions i les garanties de rendiment d'aquests algorismes per aplicar-los adequadament en problemes del món real.
