Introducció

En aquesta secció, cobrirem els conceptes fonamentals i la notació utilitzada en l'estudi d'algoritmes avançats. És crucial comprendre aquests conceptes bàsics per poder abordar temes més complexos de manera efectiva.

Conceptes Clau

  • Algoritme: Un conjunt d'instruccions ben definides i finites que resolen un problema específic.
  • Eficiència: Mesura dels recursos que un algoritme consumeix, principalment temps i espai.
  • Complexitat Temporal: Temps que triga un algoritme a executar-se, generalment expressat en funció de la mida de l'entrada (n).
  • Complexitat Espacial: Quantitat de memòria que un algoritme necessita durant la seva execució.
  • Notació Big O: Representa el pitjor cas de la complexitat temporal d'un algoritme.
  • Notació Big Ω: Representa el millor cas de la complexitat temporal d'un algoritme.
  • Notació Big Θ: Representa el cas mitjà de la complexitat temporal d'un algoritme.

Notació Asimptòtica

La notació asimptòtica és una eina fonamental per descriure l'eficiència dels algoritmes. A continuació, es presenten les notacions més comunes:

Notació Big O

La notació Big O descriu el límit superior del temps d'execució d'un algoritme. S'utilitza per expressar el pitjor cas.

Exemple:

def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

En aquest cas, la complexitat temporal és O(n), on n és la mida de l'array arr.

Notació Big Ω

La notació Big Ω descriu el límit inferior del temps d'execució d'un algoritme. S'utilitza per expressar el millor cas.

Exemple:

def find_min(arr):
    min_val = arr[0]
    for i in range(1, len(arr)):
        if arr[i] < min_val:
            min_val = arr[i]
    return min_val

En aquest cas, la complexitat temporal és Ω(n), ja que en el millor cas, l'algoritme encara ha de revisar tots els elements de l'array.

Notació Big Θ

La notació Big Θ descriu tant el límit superior com el límit inferior del temps d'execució d'un algoritme. S'utilitza per expressar el cas mitjà.

Exemple:

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

En aquest cas, la complexitat temporal és Θ(log n), on n és la mida de l'array arr.

Comparació de Notacions

| Notació | Descripció | Exemple de Complexitat | |---------|-------------|------------------------| | O(f(n)) | Pitjor cas | O(n^2) | | Ω(f(n)) | Millor cas | Ω(n) | | Θ(f(n)) | Cas mitjà | Θ(log n) |

Exemples Pràctics

Exemple 1: Ordenament de Bombolla

L'algoritme d'ordenament de bombolla té una complexitat temporal de O(n^2) en el pitjor cas.

def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr

Exemple 2: Ordenament Ràpid

L'algoritme d'ordenament ràpid té una complexitat temporal de O(n log n) en el millor cas i O(n^2) en el pitjor cas.

def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr si x == pivot]
    right = [x for x in arr si x > pivot]
    return quick_sort(left) + middle + quick_sort(right)

Conclusió

Comprendre els conceptes bàsics i la notació és essencial per a l'estudi d'algoritmes avançats. La notació asimptòtica ens permet analitzar i comparar l'eficiència de diferents algoritmes de manera formal. A mesura que avancem en el curs, aquests conceptes seran fonamentals per entendre i dissenyar algoritmes més complexos.
