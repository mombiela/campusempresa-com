Introducció

En aquesta secció del curs sobre Algorismes Avançats, explorarem dues tècniques fonamentals per resoldre problemes complexos: la recursió i la programació dinàmica. Ambdues tècniques són essencials per dissenyar algorismes eficients i són àmpliament utilitzades en diverses àrees de la informàtica.

Recursió

La recursió és una tècnica en la qual una funció es crida a si mateixa per resoldre subproblemes més petits del problema original. És especialment útil per a problemes que poden dividir-se en subproblemes similars.

Conceptes Clau

  • Cas Base: La condició que atura la recursió.
  • Cas Recursiu: La part de la funció que inclou la crida recursiva.
  • Pila de Trucades: L'estructura de dades que maneja les crides recursives.

Exemple de Codi: Factorial

def factorial(n):
    if n == 0:
        return 1  # Cas Base
    else:
        return n * factorial(n - 1)  # Cas Recursiu

# Exemple d'ús
print(factorial(5))  # Sortida: 120

Explicació de l'Exemple

  • Cas Base: Si n és 0, la funció retorna 1.
  • Cas Recursiu: La funció es crida a si mateixa amb n - 1 fins que arriba al cas base.

Programació Dinàmica

La programació dinàmica és una tècnica d'optimització que resol problemes emmagatzemant els resultats de subproblemes ja resolts per evitar càlculs redundants. S'utilitza principalment per a problemes d'optimització i de presa de decisions.

Conceptes Clau

  • Memoització: Emmagatzematge de resultats de subproblemes en una estructura de dades (com un diccionari) per reutilitzar-los.
  • Tabulació: Construcció d'una taula (normalment un array) de baix cap a dalt per emmagatzemar els resultats de subproblemes.

Exemple de Codi: Fibonacci amb Memoització

def fibonacci(n, memo={}):
    if n in memo:
        return memo[n]  # Retorna el resultat emmagatzemat
    if n <= 1:
        return n  # Cas Base
    memo[n] = fibonacci(n - 1, memo) + fibonacci(n - 2, memo)  # Cas Recursiu
    return memo[n]

# Exemple d'ús
print(fibonacci(10))  # Sortida: 55

Exemple de Codi: Fibonacci amb Tabulació

def fibonacci_tab(n):
    if n <= 1:
        return n
    fib = [0] * (n + 1)
    fib[1] = 1
    for i in range(2, n + 1):
        fib[i] = fib[i - 1] + fib[i - 2]
    return fib[n]

# Exemple d'ús
print(fibonacci_tab(10))  # Sortida: 55

Comparació: Recursió vs Programació Dinàmica

| Aspecte | Recursió | Programació Dinàmica | |------------------------|---------------------------------|----------------------------------| | Eficiència | Pot ser ineficient | Més eficient degut a la reutilització de subproblemes | | Facilitat d'Implementació | Sol ser més intuïtiva i fàcil d'implementar | Pot ser més complexa degut a la necessitat d'emmagatzematge de subproblemes | | Ús de Memòria | Pot consumir molta memòria degut a la pila de trucades | Consumeix memòria addicional per emmagatzemar resultats de subproblemes |

Exercicis

  1. Implementar l'Algorisme d'Euclides: Escriu una funció recursiva per trobar el màxim comú divisor (MCD) de dos números.
  2. Problema de la Motxilla (Knapsack Problem): Implementa una solució utilitzant programació dinàmica per al problema de la motxilla 0/1.
  3. Seqüència de Longest Common Subsequence (LCS): Escriu una funció que utilitzi programació dinàmica per trobar la longitud de la subseqüència comuna més llarga entre dues cadenes.

Conclusió

La recursió i la programació dinàmica són tècniques poderoses per resoldre problemes complexos de manera eficient. La recursió és intuïtiva i fàcil d'implementar, però pot ser ineficient per a problemes grans. La programació dinàmica, encara que més complexa, optimitza el rendiment evitant càlculs redundants. Comprendre quan i com utilitzar aquestes tècniques és crucial per dissenyar algorismes avançats i eficients.
