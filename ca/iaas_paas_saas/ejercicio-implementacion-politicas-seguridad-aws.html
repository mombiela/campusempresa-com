Introducció

En aquest exercici, aprendrem a implementar polítiques de seguretat a AWS utilitzant IAM (Identity and Access Management). Les polítiques de seguretat són essencials per controlar l'accés als recursos al núvol i garantir que només els usuaris autoritzats puguin realitzar accions específiques.

Objectius de l'Exercici

  • Comprendre el concepte de polítiques de seguretat a AWS.
  • Crear i assignar polítiques d'IAM.
  • Aplicar polítiques a usuaris, grups i rols.
  • Verificar l'efectivitat de les polítiques implementades.

Requisits Previs

  • Compte d'AWS actiu.
  • Coneixements bàsics d'AWS IAM.
  • Accés a la consola d'administració d'AWS.

Passos per Implementar Polítiques de Seguretat a AWS

1. Crear una Política d'IAM

Per crear una política d'IAM, segueix aquests passos:

  1. Inicia sessió a la consola d'administració d'AWS.
  2. Navega a IAM.
  3. Al panell de navegació, selecciona Policies.
  4. Fes clic a Create policy.
  5. Utilitza l'editor visual o JSON per definir la política.

Exemple de Política JSON

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "s3:ListBucket",
            "Resource": "arn:aws:s3:::example_bucket"
        },
        {
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:PutObject"
            ],
            "Resource": "arn:aws:s3:::example_bucket/*"
        }
    ]
}

Aquesta política permet llistar el contingut del bucket example_bucket i obtenir i posar objectes dins d'aquest bucket.

2. Assignar la Política a un Usuari

  1. Al panell de navegació d'IAM, selecciona Users.
  2. Tria l'usuari al qual vols assignar la política.
  3. A la pestanya Permissions, fes clic a Add permissions.
  4. Selecciona Attach policies directly.
  5. Cerca i selecciona la política que vas crear anteriorment.
  6. Fes clic a Next: Review i després a Add permissions.

3. Verificar la Política

Per verificar que la política s'ha aplicat correctament:

  1. Inicia sessió amb l'usuari al qual vas assignar la política.
  2. Intenta realitzar accions permeses i no permeses segons la política.
  3. Verifica que només les accions permeses a la política es poden realitzar.

Exercici Pràctic

Tasca 1: Crear una Política de Només Lectura per a S3

  1. Crea una política que permeti només accions de lectura (s3:GetObject) en un bucket específic.
  2. Assigna aquesta política a un nou usuari.

Tasca 2: Crear una Política d'Administració per a EC2

  1. Crea una política que permeti totes les accions (ec2:*) en els recursos d'EC2.
  2. Assigna aquesta política a un grup d'administradors.

Tasca 3: Verificar Polítiques

  1. Inicia sessió amb els usuaris corresponents i verifica que les polítiques aplicades funcionen com s'espera.
  2. Documenta qualsevol problema trobat i com el vas solucionar.

Conclusió

En aquest exercici, hem après a crear i assignar polítiques de seguretat a AWS utilitzant IAM. Les polítiques de seguretat són fonamentals per protegir els recursos al núvol i garantir que només els usuaris autoritzats puguin accedir-hi. Practicar la creació i assignació de polítiques t'ajudarà a gestionar millor la seguretat en els teus entorns d'AWS.
