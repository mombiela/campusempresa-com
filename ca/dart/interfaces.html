Introducció a les Interfícies

Les interfícies en Dart són una manera de definir un contracte que les classes poden implementar. A diferència d'alguns altres llenguatges, Dart no té una sintaxi especial per a les interfícies. En canvi, qualsevol classe pot ser utilitzada com una interfície.

Conceptes Clau

  • Definició d'Interfície: Qualsevol classe pot actuar com una interfície.
  • Implementació: Les classes poden implementar múltiples interfícies.
  • Mètodes Abstractes: Mètodes en una interfície que han de ser implementats per la classe que implementa.

Definint una Interfície

En Dart, defineixes una interfície simplement creant una classe amb signatures de mètodes (mètodes abstractes) que vols que altres classes implementin.

class Animal {
  void makeSound();
}

En aquest exemple, Animal actua com una interfície amb un únic mètode makeSound.

Implementant una Interfície

Una classe implementa una interfície utilitzant la paraula clau implements. La classe que implementa ha de proporcionar implementacions concretes per a tots els mètodes definits en la interfície.

class Dog implements Animal {
  @override
  void makeSound() {
    print('Bark');
  }
}

class Cat implements Animal {
  @override
  void makeSound() {
    print('Meow');
  }
}

Aquí, les classes Dog i Cat implementen la interfície Animal i proporcionen les seves pròpies versions del mètode makeSound.

Múltiples Interfícies

Una classe en Dart pot implementar múltiples interfícies. Això permet un codi més flexible i reutilitzable.

class Flyable {
  void fly();
}

class Bird implements Animal, Flyable {
  @override
  void makeSound() {
    print('Chirp');
  }

  @override
  void fly() {
    print('Flap Flap');
  }
}

En aquest exemple, la classe Bird implementa tant les interfícies Animal com Flyable.

Classes Abstractes vs Interfícies

Les classes abstractes també poden ser utilitzades per definir interfícies, però poden contenir implementacions de mètodes també. Això és útil quan vols proporcionar algun comportament per defecte.

Taula Comparativa

| Característica | Interfície (Classe) | Classe Abstracta | |------------------------|---------------------|----------------------| | Implementacions de Mètodes | No | Sí | | Herència Múltiple | Sí | No | | Instanciació | No | No |

abstract class Vehicle {
  void startEngine();

  void stopEngine() {
    print('Engine stopped');
  }
}

class Car implements Vehicle {
  @override
  void startEngine() {
    print('Car engine started');
  }
}

void main() {
  Car myCar = Car();
  myCar.startEngine(); // Car engine started
  myCar.stopEngine();  // Engine stopped
}

En aquest exemple, Vehicle és una classe abstracta amb un mètode concret stopEngine i un mètode abstracte startEngine. La classe Car implementa la interfície Vehicle i proporciona una implementació per a startEngine.

Conclusió

Les interfícies en Dart proporcionen una manera poderosa de definir contractes per a les teves classes. Utilitzant interfícies, pots assegurar-te que les teves classes adhereixen a comportaments específics i poden ser fàcilment esteses o modificades. Recorda:

  • Qualsevol classe pot actuar com una interfície.
  • Utilitza la paraula clau implements per implementar una interfície.
  • Les classes poden implementar múltiples interfícies.
  • Les classes abstractes també poden ser utilitzades per definir interfícies amb algun comportament per defecte.

Entendre les interfícies t'ajudarà a escriure codi més modular, reutilitzable i mantenible.
