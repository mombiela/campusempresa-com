Xarxes en Flutter
Proves Unitàries ►

Les xarxes són un aspecte crucial de les aplicacions mòbils modernes, ja que els permeten obtenir dades d'internet, comunicar-se amb servidors i interactuar amb diversos serveis web. En aquesta secció, explorarem com gestionar les xarxes en Flutter utilitzant Dart.

Introducció a les Xarxes en Flutter

  • Sol·licituds HTTP: Aprèn a fer sol·licituds HTTP per obtenir dades d'internet.
  • Anàlisi de JSON: Entén com analitzar dades JSON rebudes de serveis web.
  • Gestió d'Errors: Implementa la gestió d'errors per a les sol·licituds de xarxa.
  • Programació Asíncrona: Utilitza async i await de Dart per gestionar operacions asíncrones.

Fer Sol·licituds HTTP

Per fer sol·licituds HTTP en Flutter, utilitzem el paquet http. Primer, afegeix el paquet http al teu fitxer pubspec.yaml:

dependencies:
  http: ^0.13.3

Sol·licitud GET

Una sol·licitud GET s'utilitza per obtenir dades d'un servidor. Aquí tens un exemple de com fer una sol·licitud GET:

import 'package:http/http.dart' as http;
import 'dart:convert';

void fetchData() async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

  if (response.statusCode == 200) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to load data');
  }
}
  • Explicació:
    • http.get envia una sol·licitud GET a l'URL especificat.
    • response.statusCode comprova l'estat de la resposta.
    • jsonDecode analitza la resposta JSON en un objecte Dart.

Sol·licitud POST

Una sol·licitud POST s'utilitza per enviar dades a un servidor. Aquí tens un exemple:

import 'package:http/http.dart' as http;
import 'dart:convert';

void sendData() async {
  final response = await http.post(
    Uri.parse('https://jsonplaceholder.typicode.com/posts'),
    headers: <String, String>{
      'Content-Type': 'application/json; charset=UTF-8',
    },
    body: jsonEncode(<String, String>{
      'title': 'foo',
      'body': 'bar',
      'userId': '1',
    }),
  );

  if (response.statusCode == 201) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to send data');
  }
}
  • Explicació:
    • http.post envia una sol·licitud POST a l'URL especificat.
    • headers especifiquen el tipus de contingut.
    • jsonEncode converteix l'objecte Dart en una cadena JSON.

Anàlisi de JSON

Quan es treballa amb APIs, les dades sovint es retornen en format JSON. Aquí tens com analitzar dades JSON:

Exemple de Dades JSON

{
  "userId": 1,
  "id": 1,
  "title": "Sample Title",
  "body": "This is a sample body."
}

Analitzar JSON en Dart

import 'dart:convert';

class Post {
  final int userId;
  final int id;
  final String title;
  final String body;

  Post({required this.userId, required this.id, required this.title, required this.body});

  factory Post.fromJson(Map<String, dynamic> json) {
    return Post(
      userId: json['userId'],
      id: json['id'],
      title: json['title'],
      body: json['body'],
    );
  }
}

void parseJson() {
  String jsonString = '{"userId": 1, "id": 1, "title": "Sample Title", "body": "This is a sample body."}';
  Map<String, dynamic> jsonMap = jsonDecode(jsonString);

  Post post = Post.fromJson(jsonMap);
  print('Title: ${post.title}');
}
  • Explicació:
    • La classe Post representa l'estructura JSON.
    • Post.fromJson és un constructor de fàbrica que crea una instància de Post a partir d'un mapa JSON.
    • jsonDecode converteix la cadena JSON en un mapa Dart.

Gestió d'Errors

Gestionar errors és essencial per a una xarxa robusta. Aquí tens com gestionar errors en sol·licituds HTTP:

void fetchDataWithErrorHandling() async {
  try {
    final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

    if (response.statusCode == 200) {
      var data = jsonDecode(response.body);
      print(data);
    } else {
      print('Server error: ${response.statusCode}');
    }
  } catch (e) {
    print('Network error: $e');
  }
}
  • Explicació:
    • El bloc try-catch s'utilitza per gestionar excepcions.
    • response.statusCode es comprova per gestionar errors del servidor.

Programació Asíncrona

Dart utilitza async i await per gestionar operacions asíncrones. Aquí tens una breu visió general:

  • async: Marca una funció com a asíncrona.
  • await: Espera que una operació asíncrona es completi.

Exemple

Future<void> fetchData() async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

  if (response.statusCode == 200) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to load data');
  }
}
  • Explicació:
    • Future<void> indica que la funció retorna un Future.
    • await s'utilitza per esperar que la sol·licitud HTTP es completi.

Conclusió

En aquesta secció, hem cobert els conceptes bàsics de les xarxes en Flutter utilitzant Dart. Hem après a fer sol·licituds GET i POST, analitzar dades JSON, gestionar errors i utilitzar la programació asíncrona. Aquestes habilitats són essencials per construir aplicacions mòbils modernes basades en dades.

En dominar aquests conceptes, seràs capaç de crear aplicacions Flutter que puguin interactuar amb serveis web i proporcionar contingut dinàmic als usuaris.
