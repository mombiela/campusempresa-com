Mixins a Dart
Classes Abstractes ►

Els mixins són una característica poderosa a Dart que et permet reutilitzar el codi d'una classe en múltiples jerarquies de classes. Et permeten afegir funcionalitat a les classes sense utilitzar la herència. Aquest tema cobrirà els conceptes bàsics dels mixins, com utilitzar-los i alguns conceptes avançats.

Què són els Mixins?

  • Definició: Un mixin és una classe que proporciona mètodes a altres classes sense ser una classe pare.
  • Propòsit: S'utilitzen per compartir funcionalitat entre classes sense crear una jerarquia d'herència complexa.

Crear i Utilitzar Mixins

Sintaxi Bàsica

Per crear un mixin, utilitza la paraula clau mixin seguida del nom del mixin. Després, pots afegir mètodes i propietats al mixin.

mixin Logger {
  void log(String message) {
    print('Log: $message');
  }
}

Per utilitzar un mixin en una classe, utilitza la paraula clau with seguida del nom del mixin.

class Printer with Logger {
  void printDocument(String document) {
    log('Printing document: $document');
    // Lògica d'impressió aquí
  }
}

void main() {
  Printer printer = Printer();
  printer.printDocument('MyDocument.txt');
}

Múltiples Mixins

Dart et permet utilitzar múltiples mixins en una sola classe. Simplement separa els noms dels mixins amb comes.

mixin Scanner {
  void scan(String document) {
    print('Scanning document: $document');
  }
}

class MultiFunctionDevice with Logger, Scanner {
  void copy(String document) {
    log('Copying document: $document');
    scan(document);
    // Lògica de còpia aquí
  }
}

void main() {
  MultiFunctionDevice mfd = MultiFunctionDevice();
  mfd.copy('MyDocument.txt');
}

Restriccions en els Mixins

Els mixins poden tenir restriccions, és a dir, només es poden aplicar a classes que extenguin o implementin una classe o interfície específica.

class Machine {
  void start() {
    print('Machine started');
  }
}

mixin Maintenance on Machine {
  void performMaintenance() {
    print('Performing maintenance');
  }
}

class Car extends Machine with Maintenance {
  void drive() {
    print('Driving the car');
  }
}

void main() {
  Car car = Car();
  car.start();
  car.performMaintenance();
  car.drive();
}

En aquest exemple, el mixin Maintenance només es pot utilitzar en classes que extenguin Machine.

Conceptes Avançats

Sobreescriure Mètodes en Mixins

Els mixins poden sobreescriure mètodes d'altres mixins o de la classe en la qual estan barrejats.

mixin Logger {
  void log(String message) {
    print('Log: $message');
  }
}

mixin ErrorLogger on Logger {
  @override
  void log(String message) {
    super.log('Error: $message');
  }
}

class Application with Logger, ErrorLogger {
  void run() {
    log('Application started');
  }
}

void main() {
  Application app = Application();
  app.run();
}

En aquest exemple, ErrorLogger sobreescriu el mètode log de Logger.

Resum

  • Mixins: Permeten la reutilització de codi sense herència.
  • Sintaxi: Utilitza la paraula clau mixin per definir i la paraula clau with per aplicar.
  • Múltiples Mixins: Pots aplicar múltiples mixins a una sola classe.
  • Restriccions: Els mixins poden tenir restriccions per limitar la seva aplicació.
  • Sobreescriure Mètodes: Els mixins poden sobreescriure mètodes d'altres mixins o de la classe en la qual estan barrejats.

Els mixins són una eina versàtil a Dart que et poden ajudar a escriure codi més net i mantenible promovent la reutilització de codi i reduint la complexitat de les jerarquies d'herència.
