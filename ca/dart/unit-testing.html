Proves Unitàries en Dart
Proves de Widgets en Flutter ►

Les proves unitàries són un aspecte crucial del desenvolupament de programari que assegura que les unitats individuals de codi funcionin com s'espera. En Dart, les proves unitàries es faciliten amb el paquet test, que proporciona un marc per escriure i executar proves.

Introducció a les Proves Unitàries

  • Definició: Les proves unitàries impliquen provar components o funcions individuals d'un programa per assegurar-se que funcionen com s'espera.
  • Propòsit: Detectar errors aviat, assegurar la fiabilitat del codi i facilitar la refactorització.

Configuració de les Proves Unitàries en Dart

Per començar a fer proves unitàries en Dart, cal afegir el paquet test al teu projecte.

  1. Afegeix el paquet test:

    dev_dependencies:
  test: ^1.16.0

  2. Instal·la les dependències:

    dart pub get

Escriure la Teva Primera Prova

Escrivim una prova simple per entendre els conceptes bàsics.

  1. Crea un fitxer Dart per a les proves:

    touch test/example_test.dart

  2. Escriu una prova simple:

    // test/example_test.dart
import 'package:test/test.dart';

void main() {
  test('Divisió de cadena', () {
    var string = 'Hola, món!';
    var result = string.split(',');

    expect(result, equals(['Hola', ' món!']));
  });
}

Explicació:

  • Importar el paquet test: Això és necessari per utilitzar les funcions de prova.
  • Definir una prova: La funció test pren una descripció i una funció de retorn que conté la lògica de la prova.
  • Afirmacions: La funció expect comprova si el resultat real coincideix amb el resultat esperat.

Agrupació de Proves

Agrupar proves relacionades pot fer que el teu conjunt de proves sigui més organitzat i llegible.

  1. Utilitzant la funció group: 
    // test/example_test.dart
import 'package:test/test.dart';

void main() {
  group('Proves de cadena', () {
    test('Divisió de cadena', () {
      var string = 'Hola, món!';
      var result = string.split(',');

      expect(result, equals(['Hola', ' món!']));
    });

    test('Retall de cadena', () {
      var string = '  Hola  ';
      var result = string.trim();

      expect(result, equals('Hola'));
    });
  });
}

Explicació:

  • Agrupació: La funció group ajuda a organitzar les proves lògicament.
  • Proves múltiples: Pots afegir múltiples proves dins d'un grup per a una millor estructura.

Executar Proves

Per executar les teves proves, utilitza la següent comanda:

dart test

Tècniques Avançades de Proves

Mocking

El mocking s'utilitza per simular el comportament d'objectes complexos.

  1. Afegeix el paquet mockito:

    dev_dependencies:
  mockito: ^5.0.0

  2. Crea una classe mock:

    // lib/counter.dart
class Counter {
  int value = 0;

  void increment() {
    value++;
  }
}

  3. Escriu una prova amb un mock:

    // test/counter_test.dart
import 'package:test/test.dart';
import 'package:mockito/mockito.dart';
import 'package:your_package/counter.dart';

class MockCounter extends Mock implements Counter {}

void main() {
  test('Increment del Counter', () {
    var counter = MockCounter();

    when(counter.increment()).thenReturn(null);
    counter.increment();

    verify(counter.increment()).called(1);
  });
}

Explicació:

  • Classe mock: MockCounter extén Mock i implementa Counter.
  • Comportament mock: Les funcions when i verify s'utilitzen per definir i comprovar el comportament del mock.

Conclusió

Les proves unitàries en Dart són una manera poderosa d'assegurar que el teu codi funcioni correctament. Començant amb proves simples i incorporant gradualment tècniques més avançades com l'agrupació i el mocking, pots construir un conjunt de proves robust que ajudi a mantenir la qualitat i la fiabilitat del codi.

  • Punts Clau:
    • Les proves unitàries ajuden a detectar errors aviat i asseguren la fiabilitat del codi.
    • El paquet test proporciona un marc per escriure i executar proves.
    • Agrupar proves i utilitzar mocking pot millorar la teva estratègia de proves.

Seguint aquestes directrius, pots implementar efectivament proves unitàries en els teus projectes Dart i assegurar que el teu codi es mantingui mantenible i lliure d'errors.
