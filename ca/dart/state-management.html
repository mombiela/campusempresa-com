Gestió de l'Estat en Dart
Xarxes en Flutter ►

La gestió de l'estat és un aspecte crucial per construir aplicacions responsives i dinàmiques. En Dart, especialment quan es treballa amb Flutter, gestionar l'estat de la teva aplicació de manera eficient pot impactar significativament en el rendiment i l'experiència de l'usuari. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de la gestió de l'estat en Dart.

Introducció a la Gestió de l'Estat

La gestió de l'estat es refereix a la gestió de l'estat d'una aplicació, que inclou les dades i l'estat de la interfície d'usuari. Entendre com gestionar l'estat de manera efectiva és essencial per construir aplicacions escalables i mantenibles.

Conceptes Clau

  • Estat: Les dades que la teva aplicació necessita mantenir.
  • Stateful Widget: Un widget que manté un estat que pot canviar durant la vida del widget.
  • Stateless Widget: Un widget que no manté cap estat.

Widgets Stateful i Stateless

Widgets Stateless

Els widgets stateless són immutables, és a dir, les seves propietats no poden canviar un cop són creats. Són ideals per a contingut estàtic.

import 'package:flutter/material.dart';

class MyStatelessWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Exemple de Widget Stateless'),
      ),
      body: Center(
        child: Text('Aquest és un widget stateless'),
      ),
    );
  }
}

Widgets Stateful

Els widgets stateful mantenen un estat que pot canviar amb el temps. Són útils per a contingut dinàmic.

import 'package:flutter/material.dart';

class MyStatefulWidget extends StatefulWidget {
  @override
  _MyStatefulWidgetState createState() => _MyStatefulWidgetState();
}

class _MyStatefulWidgetState extends State<MyStatefulWidget> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Exemple de Widget Stateful'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text('Has premut el botó tantes vegades:'),
            Text('$_counter', style: Theme.of(context).textTheme.headline4),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _incrementCounter,
        tooltip: 'Incrementar',
        child: Icon(Icons.add),
      ),
    );
  }
}

Tècniques de Gestió de l'Estat

setState

La manera més senzilla de gestionar l'estat en Flutter és utilitzant setState. Aquest mètode s'utilitza dins d'un widget stateful per actualitzar la interfície d'usuari.

void _incrementCounter() {
  setState(() {
    _counter++;
  });
}

InheritedWidget

InheritedWidget és una classe base per a widgets que propaguen informació eficientment cap avall en l'arbre. És útil per compartir estat entre molts widgets.

import 'package:flutter/material.dart';

class MyInheritedWidget extends InheritedWidget {
  final int counter;
  final Widget child;

  MyInheritedWidget({this.counter, this.child}) : super(child: child);

  @override
  bool updateShouldNotify(MyInheritedWidget oldWidget) {
    return oldWidget.counter != counter;
  }

  static MyInheritedWidget of(BuildContext context) {
    return context.dependOnInheritedWidgetOfExactType<MyInheritedWidget>();
  }
}

Provider

Provider és un embolcall al voltant de InheritedWidget per fer-lo més fàcil d'utilitzar i més reutilitzable. És una de les solucions de gestió de l'estat més populars en Flutter.

import 'package:flutter/material.dart';
import 'package:provider/provider.dart';

void main() {
  runApp(
    ChangeNotifierProvider(
      create: (context) => Counter(),
      child: MyApp(),
    ),
  );
}

class Counter with ChangeNotifier {
  int _count = 0;

  int get count => _count;

  void increment() {
    _count++;
    notifyListeners();
  }
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Exemple de Provider'),
        ),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: <Widget>[
              Text('Has premut el botó tantes vegades:'),
              Consumer<Counter>(
                builder: (context, counter, child) {
                  return Text('${counter.count}', style: Theme.of(context).textTheme.headline4);
                },
              ),
            ],
          ),
        ),
        floatingActionButton: FloatingActionButton(
          onPressed: () {
            Provider.of<Counter>(context, listen: false).increment();
          },
          tooltip: 'Incrementar',
          child: Icon(Icons.add),
        ),
      ),
    );
  }
}

Gestió Avançada de l'Estat

Bloc (Business Logic Component)

Bloc és un patró que ajuda a separar la lògica de negoci de la interfície d'usuari. Utilitza fluxos per gestionar l'estat.

import 'package:flutter_bloc/flutter_bloc.dart';

enum CounterEvent { increment }

class CounterBloc extends Bloc<CounterEvent, int> {
  CounterBloc() : super(0);

  @override
  Stream<int> mapEventToState(CounterEvent event) async* {
    switch (event) {
      case CounterEvent.increment:
        yield state + 1;
        break;
    }
  }
}

Riverpod

Riverpod és una solució de gestió de l'estat similar a Provider que ofereix més flexibilitat i robustesa.

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

final counterProvider = StateProvider((ref) => 0);

void main() {
  runApp(ProviderScope(child: MyApp()));
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Exemple de Riverpod'),
        ),
        body: Center(
          child: Consumer(
            builder: (context, watch, child) {
              final count = watch(counterProvider).state;
              return Text('$count', style: Theme.of(context).textTheme.headline4);
            },
          ),
        ),
        floatingActionButton: FloatingActionButton(
          onPressed: () {
            context.read(counterProvider).state++;
          },
          tooltip: 'Incrementar',
          child: Icon(Icons.add),
        ),
      ),
    );
  }
}
