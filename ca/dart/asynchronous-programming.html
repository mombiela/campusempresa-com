Programació Asíncrona en Dart
Streams ►

La programació asíncrona és un concepte clau en el desenvolupament de programari modern, permetent que les aplicacions gestionin múltiples tasques simultàniament sense bloquejar el fil d'execució principal. En Dart, la programació asíncrona es maneja principalment utilitzant Future, async i await.

Introducció a la Programació Asíncrona

  • Sincrònic vs Asíncronic:

    • Sincrònic: Les tasques s'executen una darrere l'altra. Cada tasca espera que la tasca anterior es completi.
    • Asíncronic: Les tasques es poden executar simultàniament, permetent que el programa continuï executant altres tasques mentre espera que algunes operacions es completin.

  • Per què la Programació Asíncrona?:

    • Millora el rendiment en no bloquejar el fil principal.
    • Essencial per a tasques com sol·licituds de xarxa, E/S de fitxers i temporitzadors.

Futures en Dart

Un Future representa un valor potencial o un error que estarà disponible en algun moment en el futur.

  • Creant un Future:

    Future<String> fetchUserOrder() {
  // Imagina que aquesta funció està obtenint la comanda de l'usuari d'una base de dades o una API
  return Future.delayed(Duration(seconds: 2), () => 'Large Latte');
}

  • Gestionant un Future:

    void main() {
  fetchUserOrder().then((order) {
    print('La teva comanda és: $order');
  }).catchError((error) {
    print('Alguna cosa ha anat malament: $error');
  });
}

Async i Await

Les paraules clau async i await fan que el codi asíncron sembli més codi sincrònic, millorant la llegibilitat.

  • Utilitzant async i await:

    Future<void> printOrderMessage() async {
  var order = await fetchUserOrder();
  print('La teva comanda és: $order');
}

void main() {
  printOrderMessage();
}

  • Gestió d'Errors amb async i await:

    Future<void> printOrderMessage() async {
  try {
    var order = await fetchUserOrder();
    print('La teva comanda és: $order');
  } catch (error) {
    print('Alguna cosa ha anat malament: $error');
  }
}

void main() {
  printOrderMessage();
}

Streams en Dart

Els Streams proporcionen una manera de gestionar una seqüència d'esdeveniments asíncrons.

  • Creant un Stream:

    Stream<int> countStream(int max) async* {
  for (int i = 1; i <= max; i++) {
    await Future.delayed(Duration(seconds: 1));
    yield i;
  }
}

  • Escoltant un Stream:

    void main() {
  var stream = countStream(5);
  stream.listen((count) {
    print(count);
  });
}

  • Gestió d'Errors en Streams:

    void main() {
  var stream = countStream(5);
  stream.listen(
    (count) {
      print(count);
    },
    onError: (error) {
      print('Error: $error');
    },
    onDone: () {
      print('El Stream ha acabat');
    },
  );
}

Comparant Futures i Streams

| Característica | Future | Stream | |----------------|---------------------------------|--------------------------------| | Valor Únic | Retorna un valor únic o error | Pot emetre múltiples valors o errors al llarg del temps | | Finalització | Es completa amb un valor o error | Pot continuar emetent valors fins que acabi | | Ús | Ideal per a operacions asíncrones úniques | Ideal per a gestionar seqüències d'esdeveniments asíncrons |

Conclusió

La programació asíncrona en Dart és una eina poderosa que permet als desenvolupadors escriure codi eficient i no bloquejant. Entenent i utilitzant Future, async, await i Stream, pots gestionar tasques asíncrones de manera efectiva, fent que les teves aplicacions siguin més responsives i amb millor rendiment. Practica amb els exemples i exercicis proporcionats per consolidar la teva comprensió d'aquests conceptes.
