Introducció a les classes abstractes

Les classes abstractes són un concepte fonamental en la programació orientada a objectes. S'utilitzen per definir un pla per a altres classes. Una classe abstracta no es pot instanciar directament; en canvi, altres classes han d'extendre-la i implementar els seus mètodes abstractes.

Conceptes clau

  • Classe abstracta: Una classe que no es pot instanciar i està destinada a ser subclassificada.
  • Mètode abstracte: Un mètode que es declara sense una implementació i ha de ser sobreescrit en una subclasse.
  • Herència: El mecanisme pel qual una classe pot heretar propietats i mètodes d'una altra classe.

Definició d'una classe abstracta

En Dart, es defineix una classe abstracta utilitzant la paraula clau abstract. Les classes abstractes poden tenir tant mètodes abstractes (sense implementació) com mètodes concrets (amb implementació).

Exemple

abstract class Animal {
  // Mètode abstracte
  void makeSound();

  // Mètode concret
  void sleep() {
    print("Sleeping...");
  }
}

En aquest exemple:

  • Animal és una classe abstracta.
  • makeSound és un mètode abstracte.
  • sleep és un mètode concret.

Extensió d'una classe abstracta

Per utilitzar una classe abstracta, cal crear una subclasse que extengui la classe abstracta i proporcioni implementacions per als mètodes abstractes.

Exemple

class Dog extends Animal {
  @override
  void makeSound() {
    print("Bark!");
  }
}

void main() {
  Dog dog = Dog();
  dog.makeSound(); // Output: Bark!
  dog.sleep();     // Output: Sleeping...
}

En aquest exemple:

  • Dog és una subclasse de Animal.
  • Dog proporciona una implementació per al mètode makeSound.
  • Dog hereta el mètode sleep de Animal.

Classes abstractes vs Interfícies

En Dart, les classes abstractes també es poden utilitzar com a interfícies. No obstant això, hi ha algunes diferències entre les classes abstractes i les interfícies.

Taula comparativa

| Característica | Classe abstracta | Interfície (Implementada via Classe Abstracta) | |------------------------|------------------------------------------|------------------------------------------------| | Instanciació | No es pot instanciar | No es pot instanciar | | Implementació de mètodes | Pot tenir tant mètodes abstractes com concrets | Només mètodes abstractes (si s'utilitza purament com a interfície) | | Herència múltiple | Herència simple | Pot implementar múltiples interfícies |

Exemple d'ús com a interfície

abstract class Animal {
  void makeSound();
}

abstract class CanFly {
  void fly();
}

class Bird extends Animal implements CanFly {
  @override
  void makeSound() {
    print("Chirp!");
  }

  @override
  void fly() {
    print("Flying...");
  }
}

void main() {
  Bird bird = Bird();
  bird.makeSound(); // Output: Chirp!
  bird.fly();       // Output: Flying...
}

En aquest exemple:

  • Animal i CanFly són classes abstractes utilitzades com a interfícies.
  • Bird implementa tant Animal com CanFly.

Usos pràctics

Les classes abstractes són útils en escenaris on es vol definir una interfície comuna per a un grup de classes relacionades, alhora que es proporciona alguna funcionalitat compartida.

Exemple: Jerarquia de formes

abstract class Shape {
  double area();
}

class Circle extends Shape {
  final double radius;

  Circle(this.radius);

  @override
  double area() {
    return 3.14 * radius * radius;
  }
}

class Rectangle extends Shape {
  final double width;
  final double height;

  Rectangle(this.width, this.height);

  @override
  double area() {
    return width * height;
  }
}

void main() {
  Shape circle = Circle(5);
  Shape rectangle = Rectangle(4, 6);

  print("Circle Area: ${circle.area()}");       // Output: Circle Area: 78.5
  print("Rectangle Area: ${rectangle.area()}"); // Output: Rectangle Area: 24
}

En aquest exemple:

  • Shape és una classe abstracta amb un mètode abstracte area.
  • Circle i Rectangle són subclasses concretes que proporcionen les seves pròpies implementacions del mètode area.

Conclusió

Les classes abstractes en Dart proporcionen una manera poderosa de definir una interfície comuna per a un grup de classes relacionades, alhora que permeten funcionalitat compartida. Són essencials per crear codi robust i mantenible en la programació orientada a objectes. Entenent i utilitzant les classes abstractes, es poden crear estructures de codi més flexibles i reutilitzables.
