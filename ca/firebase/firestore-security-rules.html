Normes de Seguretat de Firestore
Consultes Avançades ►

Les Normes de Seguretat de Firestore són essencials per controlar l'accés a la teva base de dades de Firestore. Et permeten definir com s'ha d'estructurar, consultar i actualitzar les teves dades. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de les Normes de Seguretat de Firestore.

Introducció a les Normes de Seguretat de Firestore

  • Què són les Normes de Seguretat de Firestore?

    • Les Normes de Seguretat de Firestore són expressions que controlen l'accés als documents i col·leccions de la teva base de dades de Firestore.
    • Estan escrites en un llenguatge similar a JavaScript.

  • Per què són importants?

    • Ajuden a protegir les teves dades d'accés no autoritzat i asseguren que només els usuaris autenticats puguin llegir o escriure dades segons les teves especificacions.

Conceptes Bàsics

Estructura de les Normes de Seguretat de Firestore

  • Estructura de les Normes:

    service cloud.firestore {
  match /databases/{database}/documents {
    // Les normes van aquí
  }
}

  • Exemple de Norma Bàsica:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
    • Aquesta norma denega totes les operacions de lectura i escriptura.

Permetre Operacions de Lectura i Escriptura

  • Permetre Operacions de Lectura:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /public/{document=**} {
      allow read: if true;
    }
  }
}
    • Aquesta norma permet operacions de lectura en tots els documents de la col·lecció public.

  • Permetre Operacions d'Escriptura:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /public/{document=**} {
      allow write: if request.auth != null;
    }
  }
}
    • Aquesta norma permet operacions d'escriptura només si l'usuari està autenticat.

Conceptes Intermedis

Control d'Accés Basat en Rols

  • Definició de Rols:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
    }
  }
}
    • Aquesta norma permet als usuaris llegir i escriure el seu propi document d'usuari.

  • Exemple Basat en Rols:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /projects/{projectId} {
      allow read: if resource.data.public == true;
      allow write: if request.auth.uid == resource.data.ownerId;
    }
  }
}
    • Aquesta norma permet l'accés de lectura a projectes públics i l'accés d'escriptura al propietari del projecte.

Condicions Avançades

  • Condicions Complexes: 
    service cloud.firestore {
  match /databases/{database}/documents {
    match /posts/{postId} {
      allow read: if request.time < timestamp.date(2023, 12, 31);
      allow write: if request.auth.token.admin == true;
    }
  }
}
    • Aquesta norma permet l'accés de lectura a publicacions abans d'una data específica i l'accés d'escriptura només a usuaris administradors.

Conceptes Avançats

Funcions Personalitzades

  • Creació de Funcions Personalitzades: 
    service cloud.firestore {
  function isOwner(userId) {
    return request.auth.uid == userId;
  }

  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if isOwner(userId);
    }
  }
}
    • Aquesta norma utilitza una funció personalitzada per comprovar si l'usuari és el propietari del document.

Validació de Dades

  • Validació de Dades: 
    service cloud.firestore {
  match /databases/{database}/documents {
    match /products/{productId} {
      allow write: if request.resource.data.price > 0;
    }
  }
}
    • Aquesta norma assegura que el preu d'un producte sigui superior a zero abans de permetre una operació d'escriptura.

Combinació de Múltiples Condicions

  • Combinació de Condicions: 
    service cloud.firestore {
  match /databases/{database}/documents {
    match /orders/{orderId} {
      allow read: if request.auth != null && request.auth.token.email_verified == true;
      allow write: if request.auth != null && request.auth.token.email_verified == true && request.resource.data.amount > 0;
    }
  }
}
    • Aquesta norma combina múltiples condicions per assegurar que l'usuari estigui autenticat, el seu correu electrònic estigui verificat i l'import de la comanda sigui superior a zero.

Conclusió

Les Normes de Seguretat de Firestore són una eina poderosa per assegurar la teva base de dades de Firestore. Entenent i implementant aquestes normes, pots controlar l'accés a les teves dades i assegurar-te que només siguin accessibles per usuaris autoritzats. Des de permisos bàsics de lectura/escriptura fins a control d'accés avançat basat en rols i validació de dades, dominar les Normes de Seguretat de Firestore és essencial per a qualsevol desenvolupador de Firebase.
