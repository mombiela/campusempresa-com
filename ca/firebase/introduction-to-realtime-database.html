Visió General

Firebase Realtime Database és una base de dades NoSQL allotjada al núvol que et permet emmagatzemar i sincronitzar dades entre els teus usuaris en temps real. Aquesta secció t'introduirà als conceptes bàsics de Firebase Realtime Database, incloent les seves característiques clau, com configurar-la i com realitzar operacions bàsiques.

Característiques Clau

  • Sincronització en Temps Real: Les dades es sincronitzen en temps real a tots els clients.
  • Capacitats Fora de Línia: Les dades es cachegen localment, permetent l'ús fora de línia.
  • Seguretat: Regles de seguretat integrades per controlar l'accés a les dades.
  • Escalabilitat: S'escala automàticament per gestionar grans quantitats de dades i usuaris.

Configuració de Firebase Realtime Database

Per començar a utilitzar Firebase Realtime Database, segueix aquests passos:

  1. Crea un Projecte de Firebase:

    • Ves a la Consola de Firebase.
    • Fes clic a "Afegeix projecte" i segueix les instruccions per crear un nou projecte.

  2. Afegeix Firebase a la Teva Aplicació:

    • A la Consola de Firebase, selecciona el teu projecte.
    • Fes clic a la icona web, iOS o Android per afegir Firebase a la teva aplicació.
    • Segueix les instruccions de configuració per integrar el SDK de Firebase a la teva aplicació.

  3. Habilita Realtime Database:

    • A la Consola de Firebase, navega a "Base de dades" al menú de l'esquerra.
    • Fes clic a "Crea base de dades" i tria la ubicació i les regles de seguretat adequades.

Operacions Bàsiques

Escriure Dades

Per escriure dades a Firebase Realtime Database, pots utilitzar el mètode set. Aquí tens un exemple en JavaScript:

// Inicialitza Firebase
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
  databaseURL: "https://YOUR_PROJECT_ID.firebaseio.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
  appId: "YOUR_APP_ID"
};
firebase.initializeApp(firebaseConfig);

// Referència a la base de dades
const database = firebase.database();

// Escriure dades
database.ref('users/1').set({
  username: "JohnDoe",
  email: "johndoe@example.com"
});

Explicació:

  • firebase.initializeApp: Inicialitza Firebase amb la configuració del teu projecte.
  • firebase.database(): Obté una referència al servei de base de dades.
  • database.ref('users/1').set({...}): Escriu dades a la ruta especificada (users/1).

Llegir Dades

Per llegir dades de Firebase Realtime Database, pots utilitzar el mètode once. Aquí tens un exemple en JavaScript:

// Referència a la base de dades
const database = firebase.database();

// Llegir dades
database.ref('users/1').once('value').then((snapshot) => {
  const userData = snapshot.val();
  console.log(userData);
});

Explicació:

  • database.ref('users/1').once('value'): Llegeix dades a la ruta especificada (users/1) una vegada.
  • snapshot.val(): Recupera les dades de l'instantània.

Actualitzar Dades

Per actualitzar dades, pots utilitzar el mètode update. Aquí tens un exemple en JavaScript:

// Referència a la base de dades
const database = firebase.database();

// Actualitzar dades
database.ref('users/1').update({
  email: "newemail@example.com"
});

Explicació:

  • database.ref('users/1').update({...}): Actualitza els camps especificats a la ruta (users/1).

Esborrar Dades

Per esborrar dades, pots utilitzar el mètode remove. Aquí tens un exemple en JavaScript:

// Referència a la base de dades
const database = firebase.database();

// Esborrar dades
database.ref('users/1').remove();

Explicació:

  • database.ref('users/1').remove(): Esborra les dades a la ruta especificada (users/1).

Regles de Seguretat

Firebase Realtime Database utilitza regles de seguretat per determinar qui pot llegir i escriure a la base de dades. Aquí tens un exemple bàsic de regles de seguretat:

{
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null"
  }
}

Explicació:

  • ".read": "auth != null": Només els usuaris autenticats poden llegir dades.
  • ".write": "auth != null": Només els usuaris autenticats poden escriure dades.

També has tingut una breu introducció a les regles de seguretat.
