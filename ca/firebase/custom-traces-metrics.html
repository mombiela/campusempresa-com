Introducció

Firebase Performance Monitoring és una eina potent que t'ajuda a obtenir informació sobre el rendiment de la teva aplicació. Les traces i mètriques personalitzades et permeten monitoritzar parts específiques del rendiment de la teva aplicació que són crítiques per a l'experiència de l'usuari.

Conceptes Clau

  • Traça Personalitzada: Una traça personalitzada és una manera de mesurar el rendiment d'una secció específica del teu codi. Pots iniciar i aturar traces per mesurar el temps que triguen certes operacions.
  • Mètrica Personalitzada: Les mètriques personalitzades s'utilitzen per comptar el nombre de vegades que es produeixen certs esdeveniments o per mesurar valors que són importants per al rendiment de la teva aplicació.

Configuració de Firebase Performance Monitoring

Abans de poder utilitzar traces i mètriques personalitzades, has de configurar Firebase Performance Monitoring al teu projecte.

Pas 1: Afegeix Firebase al Teu Projecte

Segueix la documentació oficial de Firebase per afegir Firebase al teu projecte Android o iOS.

Pas 2: Afegeix el SDK de Performance Monitoring

Afegeix el SDK de Firebase Performance Monitoring al teu projecte.

Per Android (build.gradle):

dependencies {
    implementation 'com.google.firebase:firebase-perf:20.0.3'
}

Per iOS (Podfile):

pod 'Firebase/Performance'

Ús de Traces Personalitzades

Les traces personalitzades t'ajuden a mesurar el temps que triguen operacions específiques a la teva aplicació.

Creació d'una Traça Personalitzada

Per crear una traça personalitzada, has de definir un objecte traça i iniciar-lo i aturar-lo al voltant del codi que vols mesurar.

Exemple: Mesurant una Petició de Xarxa (Android)

import com.google.firebase.perf.FirebasePerformance;
import com.google.firebase.perf.metrics.Trace;

public void fetchData() {
    // Crea un objecte traça
    Trace myTrace = FirebasePerformance.getInstance().newTrace("fetchDataTrace");
    
    // Inicia la traça
    myTrace.start();

    // Codi a mesurar
    // Simula una petició de xarxa
    try {
        Thread.sleep(2000); // Simula un retard de xarxa
    } catch (InterruptedException e) {
        e.printStackTrace();
    }

    // Atura la traça
    myTrace.stop();
}

Exemple: Mesurant una Petició de Xarxa (iOS)

import FirebasePerformance

func fetchData() {
    // Crea un objecte traça
    let trace = Performance.startTrace(name: "fetchDataTrace")
    
    // Codi a mesurar
    // Simula una petició de xarxa
    DispatchQueue.global().async {
        sleep(2) // Simula un retard de xarxa
        
        // Atura la traça
        trace?.stop()
    }
}

Ús de Mètriques Personalitzades

Les mètriques personalitzades et permeten comptar esdeveniments o mesurar valors que són importants per a la teva aplicació.

Afegint Mètriques Personalitzades a una Traça

Pots afegir mètriques personalitzades a una traça per comptar esdeveniments o mesurar valors durant la traça.

Exemple: Comptant Crides API (Android)

public void fetchData() {
    Trace myTrace = FirebasePerformance.getInstance().newTrace("fetchDataTrace");
    myTrace.start();

    // Simula una petició de xarxa
    try {
        Thread.sleep(2000); // Simula un retard de xarxa
    } catch (InterruptedException e) {
        e.printStackTrace();
    }

    // Incrementa la mètrica personalitzada
    myTrace.incrementMetric("api_calls", 1);

    myTrace.stop();
}

Exemple: Comptant Crides API (iOS)

import FirebasePerformance

func fetchData() {
    let trace = Performance.startTrace(name: "fetchDataTrace")
    
    DispatchQueue.global().async {
        sleep(2) // Simula un retard de xarxa
        
        // Incrementa la mètrica personalitzada
        trace?.incrementMetric("api_calls", by: 1)
        
        trace?.stop()
    }
}

Comparant Traces i Mètriques Personalitzades

| Característica | Traça Personalitzada | Mètrica Personalitzada | |------------------|---------------------------------------------------|----------------------------------------| | Propòsit | Mesurar el temps que triguen seccions específiques del codi | Comptar esdeveniments o mesurar valors | | Inici/Atura | Sí | No | | Exemple d'Ús | Mesurar el temps d'una petició de xarxa | Comptar el nombre de crides API |

Conclusió

Les traces i mètriques personalitzades a Firebase Performance Monitoring proporcionen eines potents per mesurar i optimitzar el rendiment de la teva aplicació. Entenent i implementant aquestes característiques, pots obtenir informació valuosa sobre parts específiques de la teva aplicació i assegurar una experiència d'usuari fluida. Recorda començar configurant Firebase Performance Monitoring, després crea traces personalitzades i afegeix mètriques personalitzades segons sigui necessari per monitoritzar operacions crítiques a la teva aplicació.
