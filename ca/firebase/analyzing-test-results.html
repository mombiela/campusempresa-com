El Projecte Sobre nosaltres Contribuir Donacions Llicència

Analitzar els resultats de les proves és una part crucial del procés de desenvolupament, especialment quan es treballa amb Firebase. Aquest tema et guiarà a través dels passos per analitzar efectivament els resultats de les proves, des de nivells principiants fins a avançats.

Introducció als Resultats de Proves

Abans d'endinsar-nos en l'anàlisi, és essencial entendre què són els resultats de les proves i per què són importants.

  • Resultats de Proves: Sortides generades després d'executar proves en la teva base de codi.
  • Importància: Ajuda a identificar errors, assegurar la qualitat del codi i mantenir l'estabilitat de l'aplicació.

Configurant Firebase Test Lab

Firebase Test Lab et permet provar la teva aplicació en una àmplia gamma de dispositius i configuracions.

Passos per Configurar Firebase Test Lab

  1. Crear un Projecte de Firebase:

    • Ves a la Consola de Firebase.
    • Fes clic a "Afegeix projecte" i segueix les instruccions de configuració.

  2. Afegeix la Teva Aplicació al Projecte:

    • Fes clic al botó "Afegeix aplicació" i segueix les instruccions per registrar la teva aplicació.

  3. Habilita Firebase Test Lab:

    • Navega a la secció de Test Lab a la Consola de Firebase.
    • Segueix les instruccions per habilitar Test Lab per al teu projecte.

Executant Proves

Tipus de Proves

  • Proves Unitàries: Prova components o funcions individuals.
  • Proves d'Integració: Prova la interacció entre diferents components.
  • Proves d'Interfície d'Usuari (UI): Prova la interfície d'usuari i les interaccions dels usuaris.

Executant Proves a Firebase Test Lab

  1. Carrega el Teu APK de Prova:

    • A la secció de Test Lab, fes clic a "Executa una prova".
    • Carrega l'APK de la teva aplicació i l'APK de prova.

  2. Selecciona Dispositius i Configuracions:

    • Tria entre una varietat de dispositius i configuracions per executar les teves proves.

  3. Inicia la Prova:

    • Fes clic a "Inicia prova" per començar el procés de prova.

Analitzant Resultats de Proves

Un cop les proves estan completes, Firebase Test Lab proporciona resultats detallats que pots analitzar.

Entenent el Tauler de Resultats de Proves

  • Visió General: Resum dels resultats de les proves, incloent l'estat de pas/fallada.
  • Registres: Registres detallats de l'execució de la prova.
  • Captures de Pantalla: Representació visual de l'aplicació durant la prova.
  • Vídeos: Vídeos enregistrats de l'execució de la prova.

Mètriques Clau per Analitzar

  • Taxa de Pas/Fallada: Percentatge de proves que han passat o fallat.
  • Registres d'Errors: Missatges d'error detallats i traçats de pila.
  • Mètriques de Rendiment: Ús de CPU, memòria i activitat de xarxa.

Exemple: Analitzant una Prova Fallida

@Test
public void testLoginFunctionality() {
    // Simula l'entrada de l'usuari
    onView(withId(R.id.username)).perform(typeText("testuser"));
    onView(withId(R.id.password)).perform(typeText("password"));
    onView(withId(R.id.login_button)).perform(click());

    // Comprova si l'inici de sessió ha estat exitós
    onView(withId(R.id.welcome_message)).check(matches(isDisplayed()));
}

  • Registre d'Errors:

    java.lang.AssertionError: View is not displayed
at androidx.test.espresso.matcher.ViewMatchers$10.matchesSafely(ViewMatchers.java:1234)

  • Anàlisi:

    • El registre d'errors indica que la vista welcome_message no es mostra.
    • Les possibles raons podrien ser credencials incorrectes, problemes de xarxa o un error en la lògica d'inici de sessió.

Tècniques Avançades d'Anàlisi

Per a usuaris avançats, Firebase Test Lab ofereix eines i integracions addicionals.

Integrant amb Pipelines CI/CD

  • Integració Contínua (CI): Executa automàticament proves en cada commit de codi.
  • Desplegament Continu (CD): Desplega l'aplicació si totes les proves passen.

Utilitzant Firebase Crashlytics

  • Informes de Fallades: Informes detallats de les fallades de l'aplicació.
  • Errors No Fatals: Registres d'errors no fatals que ocorren durant les proves.

Exemple: Integrant Firebase Test Lab amb Jenkins

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                sh 'gradlew assembleDebug assembleAndroidTest'
            }
        }
        stage('Test') {
            steps {
                sh 'gcloud firebase test android run --type instrumentation --app app/build/outputs/apk/debug/app-debug.apk --test app/build/outputs/apk/androidTest/debug/app-debug-androidTest.apk'
            }
        }
    }
}
  • Explicació:
    • El pipeline construeix l'aplicació i els APKs de prova.
    • Després executa les proves utilitzant Firebase Test Lab.

Conclusió

Analitzar els resultats de les proves és una habilitat vital per mantenir aplicacions d'alta qualitat. Aprofitant Firebase Test Lab, pots executar proves completes en diversos dispositius i configuracions, assegurant que la teva aplicació funcioni bé en escenaris del món real. Entendre com interpretar els resultats de les proves, identificar problemes i integrar les proves en el teu pipeline CI/CD millorarà significativament el teu flux de treball de desenvolupament.
