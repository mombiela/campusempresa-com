Introducció a la Seguretat de Firebase

Firebase proporciona un conjunt d'eines per ajudar-te a assegurar les teves aplicacions. Entendre i implementar les millors pràctiques de seguretat és crucial per protegir les dades dels usuaris i mantenir la integritat de la teva aplicació.

Autenticació

L'autenticació és el primer pas per assegurar la teva aplicació Firebase. Firebase Authentication proporciona serveis de backend per ajudar a autenticar els usuaris.

  • Autenticació per Correu Electrònic i Contrasenya: Mètode simple i comú.
  • Proveïdors OAuth: Google, Facebook, Twitter, etc.
  • Autenticació Anònima: Útil per a usuaris convidats.

Exemple: Autenticació per Correu Electrònic i Contrasenya

firebase.auth().createUserWithEmailAndPassword(email, password)
  .then((userCredential) => {
    // Sessió iniciada
    var user = userCredential.user;
    console.log("Usuari creat: ", user);
  })
  .catch((error) => {
    var errorCode = error.code;
    var errorMessage = error.message;
    console.error("Error: ", errorCode, errorMessage);
  });

Explicació:

  • El mètode createUserWithEmailAndPassword crea un nou usuari amb el correu electrònic i la contrasenya proporcionats.
  • Gestiona errors com ara format de correu electrònic invàlid o contrasenya dèbil.

Regles de Seguretat de Firestore

Les Regles de Seguretat de Firestore controlen l'accés a la teva base de dades Firestore. Determinen qui pot llegir o escriure a la teva base de dades.

  • Declaracions Match: Defineixen el camí als documents.
  • Declaracions Allow: Especifiquen les condicions sota les quals es concedeix l'accés.

Exemple: Regla Bàsica de Firestore

service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Explicació:

  • La regla permet l'accés de lectura i escriptura a la col·lecció users només si l'usuari està autenticat i el userId coincideix amb l'ID de l'usuari autenticat.

Regles de Seguretat de la Base de Dades en Temps Real

Similar a Firestore, les Regles de Seguretat de la Base de Dades en Temps Real controlen l'accés a la teva Base de Dades en Temps Real.

  • Regles .read i .write: Defineixen qui pot llegir i escriure dades.
  • Regles de Validació: Asseguren la integritat de les dades.

Exemple: Regla Bàsica de la Base de Dades en Temps Real

{
  "rules": {
    "users": {
      "$user_id": {
        ".read": "auth != null && auth.uid == $user_id",
        ".write": "auth != null && auth.uid == $user_id"
      }
    }
  }
}

Explicació:

  • La regla permet l'accés de lectura i escriptura al node users només si l'usuari està autenticat i el user_id coincideix amb l'ID de l'usuari autenticat.

Regles de Seguretat d'Emmagatzematge

Les Regles de Seguretat d'Emmagatzematge de Firebase determinen qui pot pujar, descarregar i eliminar fitxers al teu emmagatzematge.

  • Declaracions Match: Defineixen el camí als fitxers.
  • Declaracions Allow: Especifiquen les condicions sota les quals es concedeix l'accés.

Exemple: Regla Bàsica d'Emmagatzematge

service firebase.storage {
  match /b/{bucket}/o {
    match /user_files/{userId}/{fileName} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Explicació:

  • La regla permet l'accés de lectura i escriptura al directori user_files només si l'usuari està autenticat i el userId coincideix amb l'ID de l'usuari autenticat.

Pràctiques Avançades de Seguretat

Per a una seguretat avançada, considera les següents pràctiques:

  • Reclamacions i Rols Personalitzats: Utilitza reclamacions personalitzades per gestionar rols i permisos d'usuari.
  • Limitació de Ràtio: Prevé l'abús limitant el nombre de sol·licituds.
  • Xifrat de Dades: Assegura que les dades sensibles estiguin xifrades tant en trànsit com en repòs.
  • Auditories Regulars: Revisa i actualitza regularment les teves regles de seguretat.

Exemple: Reclamacions Personalitzades

// Estableix reclamacions d'usuari personalitzades
admin.auth().setCustomUserClaims(uid, {admin: true})
  .then(() => {
    console.log("Reclamacions personalitzades establertes per a l'usuari: ", uid);
  })
  .catch((error) => {
    console.error("Error establint reclamacions personalitzades: ", error);
  });

Explicació:

  • El mètode setCustomUserClaims assigna reclamacions personalitzades a un usuari, que es poden utilitzar per gestionar rols i permisos.

Conclusió

Assegurar la teva aplicació Firebase implica múltiples capes, des de l'autenticació fins a les regles de base de dades i emmagatzematge. Seguint aquestes millors pràctiques, pots assegurar-te que la teva aplicació romangui segura i que les dades dels teus usuaris estiguin protegides. Revisa i actualitza regularment les teves regles de seguretat per adaptar-te a noves amenaces i canvis en la teva aplicació.
