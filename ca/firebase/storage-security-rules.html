Les Regles de Seguretat d'Emmagatzematge de Firebase són essencials per protegir les teves dades i assegurar que només els usuaris autoritzats puguin accedir-hi o modificar-les. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de les Regles de Seguretat d'Emmagatzematge de Firebase.

Introducció a les Regles de Seguretat d'Emmagatzematge de Firebase

Les Regles de Seguretat d'Emmagatzematge de Firebase et permeten controlar l'accés als teus fitxers emmagatzemats. Es redacten en un llenguatge personalitzat que és similar a JavaScript i s'avaluen cada vegada que es llegeix o s'escriu un fitxer.

Conceptes Clau

  • Llenguatge de Regles: Un llenguatge personalitzat utilitzat per definir les regles de seguretat.
  • Autenticació: Assegurar que els usuaris són qui diuen ser.
  • Autorització: Determinar què poden fer els usuaris autenticats.
  • Condicions: Expressions lògiques que s'avaluen com a vertaderes o falses per fer complir les regles.

Regles Bàsiques

Regles Per Defecte

Quan configures Firebase Storage per primera vegada, ve amb regles per defecte que permeten l'accés de lectura i escriptura als usuaris autenticats.

service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if request.auth != null;
    }
  }
}

Explicació

  • service firebase.storage: Defineix que aquestes regles s'apliquen a Firebase Storage.
  • match /b/{bucket}/o: Coincideix amb tots els fitxers del bucket d'emmagatzematge.
  • match /{allPaths=**}: Coincideix amb tots els camins dins del bucket.
  • allow read, write: if request.auth != null;: Permet l'accés de lectura i escriptura si l'usuari està autenticat.

Regles Intermèdies

Restringir l'Accés a Camins Específics

Pots restringir l'accés a camins específics dins del teu bucket d'emmagatzematge.

service firebase.storage {
  match /b/{bucket}/o {
    match /public/{allPaths=**} {
      allow read;
    }
    match /private/{userId}/{allPaths=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Explicació

  • match /public/{allPaths=**}: Permet l'accés de lectura a tots els fitxers de la carpeta public.
  • match /private/{userId}/{allPaths=**}: Permet l'accés de lectura i escriptura als fitxers de la carpeta private només si l'usuari està autenticat i el seu ID d'usuari coincideix amb el userId del camí.

Regles Avançades

Condicions Personalitzades

Pots crear condicions personalitzades per fer complir regles més complexes.

service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read: if isUserSignedIn() && isEmailVerified();
      allow write: if isUserSignedIn() && isEmailVerified() && isFileSizeValid();
    }
  }
}

function isUserSignedIn() {
  return request.auth != null;
}

function isEmailVerified() {
  return request.auth.token.email_verified == true;
}

function isFileSizeValid() {
  return request.resource.size < 5 * 1024 * 1024; // 5MB
}

Explicació

  • isUserSignedIn(): Comprova si l'usuari està autenticat.
  • isEmailVerified(): Comprova si el correu electrònic de l'usuari està verificat.
  • isFileSizeValid(): Comprova si la mida del fitxer és inferior a 5MB.
  • Les regles allow read i allow write utilitzen aquestes funcions per fer complir condicions més complexes.

Conclusió

Les Regles de Seguretat d'Emmagatzematge de Firebase són una eina poderosa per controlar l'accés a les teves dades. Entenent i aplicant aquestes regles, pots assegurar que les teves dades estiguin segures i només accessibles per usuaris autoritzats. Comença amb les regles bàsiques i gradualment passa a condicions més complexes a mesura que creixin les necessitats de seguretat de la teva aplicació.

Resum

  • Regles Bàsiques: Permetre o denegar l'accés basat en l'autenticació.
  • Regles Intermèdies: Restringir l'accés a camins i usuaris específics.
  • Regles Avançades: Utilitzar condicions personalitzades per a requisits de seguretat més complexos.

Dominant les Regles de Seguretat d'Emmagatzematge de Firebase, pots protegir les teves dades i proporcionar una experiència segura per als teus usuaris.
