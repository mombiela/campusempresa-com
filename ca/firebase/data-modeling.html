Introducció al Modelatge de Dades

El modelatge de dades és el procés de crear una estructura de dades que defineix com es guarden, accedeixen i gestionen les dades. A Firebase, el modelatge de dades és crucial per garantir una recuperació i manipulació eficient de les dades.

Conceptes Clau

  • Firebase Realtime Database: Una base de dades al núvol NoSQL que emmagatzema dades com a JSON.
  • Firestore: Una base de dades flexible i escalable per al desenvolupament mòbil, web i de servidor de Firebase i Google Cloud Platform.
  • Col·leccions i Documents: Firestore organitza les dades en col·leccions i documents.
  • Normalització vs. Desnormalització: Estratègies per estructurar les dades per optimitzar el rendiment i l'escalabilitat.

Firebase Realtime Database

Estructura

La Firebase Realtime Database emmagatzema les dades com un gran arbre JSON. Aquesta estructura és diferent de les bases de dades SQL tradicionals i requereix un enfocament diferent per al modelatge de dades.

Exemple d'Estructura JSON

{
  "users": {
    "user1": {
      "name": "John Doe",
      "email": "john@example.com"
    },
    "user2": {
      "name": "Jane Smith",
      "email": "jane@example.com"
    }
  },
  "posts": {
    "post1": {
      "title": "First Post",
      "content": "This is my first post!",
      "author": "user1"
    },
    "post2": {
      "title": "Second Post",
      "content": "This is my second post!",
      "author": "user2"
    }
  }
}

Millors Pràctiques

  • Aplanar Dades: Evitar estructures profundament niades per millorar el rendiment.
  • Utilitzar Identificadors Únics: Assegurar que cada node tingui una clau única.
  • Desnormalitzar Dades: Duplicar dades per evitar consultes complexes.

Exemple de Dades Aplanades

{
  "users": {
    "user1": {
      "name": "John Doe",
      "email": "john@example.com"
    },
    "user2": {
      "name": "Jane Smith",
      "email": "jane@example.com"
    }
  },
  "posts": {
    "post1": {
      "title": "First Post",
      "content": "This is my first post!",
      "author": "user1",
      "authorName": "John Doe"
    },
    "post2": {
      "title": "Second Post",
      "content": "This is my second post!",
      "author": "user2",
      "authorName": "Jane Smith"
    }
  }
}

Firestore

Estructura

Firestore organitza les dades en col·leccions i documents, que són més flexibles i escalables en comparació amb la Realtime Database.

Exemple d'Estructura

  • Col·lecció: users
    • Document: user1
      • name: "John Doe"
      • email: "john@example.com"
  • Col·lecció: posts
    • Document: post1
      • title: "First Post"
      • content: "This is my first post!"
      • author: Referència a users/user1

Millors Pràctiques

  • Utilitzar Col·leccions i Subcol·leccions: Organitzar les dades jeràrquicament.
  • Utilitzar Referències: Enllaçar documents utilitzant referències per evitar la duplicació de dades.
  • Indexació: Utilitzar les capacitats d'indexació de Firestore per optimitzar les consultes.

Exemple d'Ús de Referències

// Afegir un usuari
db.collection("users").doc("user1").set({
  name: "John Doe",
  email: "john@example.com"
});

// Afegir una publicació amb una referència a l'usuari
db.collection("posts").doc("post1").set({
  title: "First Post",
  content: "This is my first post!",
  author: db.doc("users/user1")
});

Comparació: Realtime Database vs. Firestore

| Característica | Realtime Database | Firestore | |--------------------------|------------------------------|----------------------------------| | Estructura de Dades | Arbre JSON | Col·leccions i Documents | | Consultes | Capacitats de consulta limitades | Capacitats avançades de consulta | | Suport Offline | Bàsic | Avançat | | Escalabilitat | Adequat per a conjunts de dades petits | Altament escalable | | Preus | Basat en la mida de les dades i l'ample de banda | Basat en operacions i emmagatzematge |

Conclusió

El modelatge de dades a Firebase implica entendre l'estructura i les millors pràctiques tant per a la Realtime Database com per a Firestore. Seguint les directrius per aplanar les dades, utilitzar identificadors únics i aprofitar les col·leccions i referències, es poden crear models de dades eficients i escalables. La decisió entre la Realtime Database i Firestore depèn del cas d'ús específic i els requisits.
