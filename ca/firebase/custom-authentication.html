Firebase Authentication proporciona serveis de backend per ajudar a autenticar usuaris a la teva aplicació. Tot i que Firebase ofereix diversos mètodes d'autenticació integrats, hi ha casos en què necessites més control sobre el procés d'autenticació. Aquí és on entra en joc l'autenticació personalitzada.

Introducció a l'Autenticació Personalitzada

L'Autenticació Personalitzada et permet integrar el teu propi sistema d'autenticació amb Firebase. Això significa que pots utilitzar la teva base de dades d'usuaris existent, proveïdors d'autenticació de tercers o qualsevol altre mètode per autenticar usuaris.

Conceptes Clau

  • Token Personalitzat: Un JWT (JSON Web Token) que generes al teu servidor per autenticar usuaris.
  • Firebase Admin SDK: Un conjunt de biblioteques de servidor que t'ajuden a generar tokens personalitzats.
  • Client SDK: Firebase SDK utilitzat al costat del client per autenticar usuaris amb el token personalitzat.

Configuració de Firebase Admin SDK

Per generar tokens personalitzats, necessites configurar el Firebase Admin SDK al teu entorn de servidor.

Passos per Configurar Firebase Admin SDK

  1. Instal·la el Firebase Admin SDK:

    npm install firebase-admin --save

  2. Inicialitza el SDK:

    const admin = require('firebase-admin');
const serviceAccount = require('path/to/serviceAccountKey.json');

admin.initializeApp({
  credential: admin.credential.cert(serviceAccount)
});

  3. Genera un Token Personalitzat:

    const uid = 'some-unique-identifier';

admin.auth().createCustomToken(uid)
  .then((customToken) => {
    // Envia el token de tornada al client
  })
  .catch((error) => {
    console.log('Error creant el token personalitzat:', error);
  });

Autenticació d'Usuaris amb Tokens Personalitzats

Un cop hagis generat un token personalitzat al teu servidor, necessites enviar-lo al client i utilitzar-lo per autenticar amb Firebase.

Passos per Autenticar amb Tokens Personalitzats

  1. Rep el Token Personalitzat al Client:

    // Suposem que has rebut el token personalitzat del teu servidor
const customToken = 'YOUR_CUSTOM_TOKEN';

  2. Inicia Sessió amb el Token Personalitzat:

    firebase.auth().signInWithCustomToken(customToken)
  .then((userCredential) => {
    // L'usuari ha iniciat sessió
    console.log('Usuari ha iniciat sessió:', userCredential.user);
  })
  .catch((error) => {
    console.log('Error iniciant sessió amb el token personalitzat:', error);
  });

Escenaris Avançats d'Autenticació Personalitzada

Ús de Proveïdors d'Autenticació de Tercers

Pots integrar proveïdors d'autenticació de tercers com OAuth, SAML o qualsevol altre proveïdor amb l'Autenticació Personalitzada de Firebase.

Exemple: Integració d'OAuth

  1. Autentica amb el Proveïdor OAuth:

    // Exemple utilitzant OAuth2.0
const oauth2Client = new OAuth2Client(CLIENT_ID, CLIENT_SECRET, REDIRECT_URI);
const { tokens } = await oauth2Client.getToken(code);
oauth2Client.setCredentials(tokens);

  2. Genera un Token Personalitzat amb l'UID d'OAuth:

    const uid = 'oauth-unique-identifier';

admin.auth().createCustomToken(uid)
  .then((customToken) => {
    // Envia el token de tornada al client
  })
  .catch((error) => {
    console.log('Error creant el token personalitzat:', error);
  });

Claims Personalitzades

Les claims personalitzades et permeten afegir informació addicional al token d'ID de l'usuari, que es pot utilitzar per controlar l'accés a diferents parts de la teva aplicació.

Afegint Claims Personalitzades

  1. Estableix Claims Personalitzades:

    const uid = 'some-unique-identifier';
const customClaims = {
  admin: true,
  premiumAccount: true
};

admin.auth().setCustomUserClaims(uid, customClaims)
  .then(() => {
    // Les noves claims personalitzades es propagaran al token d'ID de l'usuari la propera vegada que se'n generi un de nou.
  })
  .catch((error) => {
    console.log('Error establint claims personalitzades:', error);
  });

  2. Accedeix a les Claims Personalitzades al Client:

    firebase.auth().currentUser.getIdTokenResult()
  .then((idTokenResult) => {
    // Confirma que l'usuari és un Admin.
    if (!!idTokenResult.claims.admin) {
      console.log('L'usuari és un admin');
    }
  })
  .catch((error) => {
    console.log('Error obtenint el resultat del token d'ID:', error);
  });

Conclusió

L'Autenticació Personalitzada a Firebase proporciona la flexibilitat per integrar el teu propi sistema d'autenticació, utilitzar proveïdors de tercers i afegir claims personalitzades per controlar l'accés dins de la teva aplicació. Entenent com generar i utilitzar tokens personalitzats, pots aprofitar les potents funcions d'autenticació de Firebase mentre mantens el control sobre el procés d'autenticació dels teus usuaris.
