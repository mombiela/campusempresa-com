Les proves són una part crucial de qualsevol procés de desenvolupament, assegurant que la teva aplicació es comporti com s'espera. Firebase proporciona diverses eines i serveis per ajudar-te a executar proves de manera efectiva. Aquesta secció et guiarà a través dels conceptes bàsics d'executar proves a Firebase, des de la configuració del teu entorn fins a l'execució i el seguiment de les proves.

Configurar el Teu Entorn de Proves

Abans de poder executar proves, necessites configurar el teu entorn de proves. Firebase ofereix diverses eines que poden ajudar-te amb això:

  • Firebase Test Lab: Una infraestructura de proves d'aplicacions basada en el núvol.
  • Firebase Emulator Suite: Et permet executar i automatitzar proves localment.

Firebase Test Lab

Firebase Test Lab proporciona una infraestructura basada en el núvol per provar la teva aplicació en una àmplia gamma de dispositius i configuracions.

  1. Crea un Projecte de Firebase: Si encara no ho has fet, crea un projecte de Firebase a la Consola de Firebase.
  2. Habilita Test Lab: Navega a la secció de Test Lab a la Consola de Firebase i habilita-ho per al teu projecte.
  3. Carrega la Teva Aplicació: Pots carregar la teva aplicació Android o iOS a Test Lab.
  4. Selecciona Dispositius de Prova: Tria entre una varietat de dispositius i configuracions per executar les teves proves.
  5. Executa Proves: Executa les teves proves i visualitza els resultats a la Consola de Firebase.

Firebase Emulator Suite

Firebase Emulator Suite et permet executar i automatitzar proves localment, cosa que pot ser més ràpida i econòmica que utilitzar solucions basades en el núvol.

  1. Instal·la Firebase CLI: Assegura't de tenir instal·lat Firebase CLI. 
    npm install -g firebase-tools
  2. Inicialitza l'Emulator Suite: Al directori del teu projecte, inicialitza l'Emulator Suite. 
    firebase init emulators
  3. Configura Emuladors: Tria quins emuladors configurar (Firestore, Realtime Database, Authentication, etc.).
  4. Inicia els Emuladors: Inicia els emuladors per executar les teves proves localment. 
    firebase emulators:start

Escriure i Executar Proves

Un cop el teu entorn està configurat, pots començar a escriure i executar proves. Firebase admet diversos tipus de proves, incloent proves unitàries, proves d'integració i proves de punta a punta.

Proves Unitàries

Les proves unitàries se centren en provar components o funcions individuals de manera aïllada.

  • Exemple: Provar una funció de Firestore. 
    const assert = require('assert');
const firebase = require('@firebase/testing');

describe('Proves Unitàries de Firestore', () => {
    it('hauria de crear un document a Firestore', async () => {
        const db = firebase.initializeTestApp({ projectId: 'my-project' }).firestore();
        const docRef = db.collection('users').doc('alice');
        await docRef.set({ name: 'Alice' });
        const doc = await docRef.get();
        assert.equal(doc.data().name, 'Alice');
    });
});

Proves d'Integració

Les proves d'integració se centren en provar com diferents parts de la teva aplicació treballen juntes.

  • Exemple: Provar Firestore amb Firebase Authentication. 
    const assert = require('assert');
const firebase = require('@firebase/testing');

describe('Proves d\'Integració de Firestore', () => {
    it('hauria de permetre als usuaris autenticats escriure a Firestore', async () => {
        const db = firebase.initializeTestApp({ projectId: 'my-project', auth: { uid: 'user123' } }).firestore();
        const docRef = db.collection('users').doc('user123');
        await docRef.set({ name: 'Alice' });
        const doc = await docRef.get();
        assert.equal(doc.data().name, 'Alice');
    });
});

Proves de Punta a Punta

Les proves de punta a punta simulen interaccions reals d'usuaris i proven tot el flux de l'aplicació.

  • Exemple: Utilitzar Firebase Test Lab per executar una prova d'Espresso a Android. 
    @RunWith(AndroidJUnit4.class)
public class MainActivityTest {
    @Rule
    public ActivityTestRule<MainActivity> activityRule = new ActivityTestRule<>(MainActivity.class);

    @Test
    public void testUserLogin() {
        onView(withId(R.id.username)).perform(typeText("user123"), closeSoftKeyboard());
        onView(withId(R.id.password)).perform(typeText("password"), closeSoftKeyboard());
        onView(withId(R.id.login_button)).perform(click());
        onView(withId(R.id.welcome_message)).check(matches(withText("Welcome, user123!")));
    }
}

Seguiment dels Resultats de les Proves

Després d'executar les teves proves, és important fer un seguiment dels resultats per assegurar-te que tot funciona com s'espera.

  • Consola de Firebase: Visualitza resultats detallats de les proves, incloent registres, captures de pantalla i vídeos.
  • Informes de Proves: Genera i descarrega informes de proves per a una anàlisi més detallada.
  • Alertes: Configura alertes per notificar-te de fallades de proves o problemes.

Conclusió

Executar proves és una part essencial del desenvolupament d'aplicacions robustes. Firebase proporciona eines potents com Firebase Test Lab i Firebase Emulator Suite per ajudar-te a provar la teva aplicació de manera efectiva. Configurant el teu entorn de proves, escrivint diversos tipus de proves i fent un seguiment dels resultats, pots assegurar-te que la teva aplicació és fiable i funciona bé en diferents escenaris.
