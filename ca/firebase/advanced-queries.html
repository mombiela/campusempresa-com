Consultes Avançades

En aquesta secció, aprofundirem en tècniques avançades de consulta a Firebase. Això inclourà consultes complexes, indexació i optimització del rendiment. Al final d'aquesta secció, hauríeu de ser capaços de recuperar i manipular dades de manera eficient a Firebase utilitzant mètodes de consulta avançats.

Conceptes Clau

  • Consultes Compostes: Combinació de múltiples condicions de consulta.
  • Indexació: Optimització del rendiment de les consultes.
  • Paginació: Gestió eficient de grans conjunts de dades.
  • Actualitzacions en Temps Real: Mantenir les dades sincronitzades en temps real.
  • Regles de Seguretat: Assegurar que les consultes compleixin amb les restriccions de seguretat.

Consultes Compostes

Les consultes compostes us permeten combinar múltiples condicions per filtrar les vostres dades de manera més precisa.

Exemple: Combinació de Múltiples Condicions

const db = firebase.firestore();
const query = db.collection('users')
  .where('age', '>=', 18)
  .where('city', '==', 'New York');

query.get().then((querySnapshot) => {
  querySnapshot.forEach((doc) => {
    console.log(doc.id, ' => ', doc.data());
  });
});

En aquest exemple:

  • Estem consultant la col·lecció users.
  • Filtreu els usuaris que tenen 18 anys o més i viuen a Nova York.

Indexació

La indexació és crucial per optimitzar el rendiment de les vostres consultes. Firebase indexa automàticament alguns camps, però per a consultes compostes, és possible que necessiteu crear índexs personalitzats.

Creació d'un Índex

  1. Realitzeu una consulta que requereixi un índex.
  2. Si es necessita un índex, Firebase proporcionarà un enllaç per crear-lo.
  3. Seguiu l'enllaç i creeu l'índex a la consola de Firebase.

Exemple: Indexació per a Consultes Compostes

const db = firebase.firestore();
const query = db.collection('orders')
  .where('status', '==', 'shipped')
  .where('total', '>', 100);

query.get().then((querySnapshot) => {
  querySnapshot.forEach((doc) => {
    console.log(doc.id, ' => ', doc.data());
  });
});

Per a aquesta consulta, és possible que necessiteu crear un índex en status i total.

Paginació

Gestionar grans conjunts de dades de manera eficient és essencial per al rendiment i l'experiència de l'usuari. La paginació ajuda a carregar dades en blocs.

Exemple: Implementació de la Paginació

const db = firebase.firestore();
const pageSize = 10;
let lastVisible = null;

function getNextPage() {
  let query = db.collection('products')
    .orderBy('name')
    .limit(pageSize);

  if (lastVisible) {
    query = query.startAfter(lastVisible);
  }

  query.get().then((querySnapshot) => {
    lastVisible = querySnapshot.docs[querySnapshot.docs.length - 1];
    querySnapshot.forEach((doc) => {
      console.log(doc.id, ' => ', doc.data());
    });
  });
}

// Truca a getNextPage() per carregar la primera pàgina
getNextPage();

En aquest exemple:

  • Carreguem productes en blocs de 10.
  • lastVisible fa un seguiment de l'últim document de la consulta anterior per obtenir el següent conjunt de documents.

Actualitzacions en Temps Real

Firebase proporciona sincronització en temps real, que és útil per a aplicacions que requereixen actualitzacions en directe.

Exemple: Escoltar Actualitzacions en Temps Real

const db = firebase.firestore();
db.collection('messages')
  .where('chatId', '==', 'chat123')
  .onSnapshot((snapshot) => {
    snapshot.docChanges().forEach((change) => {
      if (change.type === 'added') {
        console.log('Nou missatge: ', change.doc.data());
      }
      if (change.type === 'modified') {
        console.log('Missatge modificat: ', change.doc.data());
      }
      if (change.type === 'removed') {
        console.log('Missatge eliminat: ', change.doc.data());
      }
    });
  });

En aquest exemple:

  • Escoltem els canvis a la col·lecció messages per a un xat específic.
  • El mètode onSnapshot proporciona actualitzacions en temps real per a missatges afegits, modificats i eliminats.

Regles de Seguretat

Assegurar que les vostres consultes compleixin amb les restriccions de seguretat és vital per protegir les vostres dades.

Exemple: Regles de Seguretat per a Consultes

service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read: if request.auth.uid == userId;
      allow write: if request.auth.uid == userId;
    }
  }
}

En aquest exemple:

  • Els usuaris només poden llegir i escriure les seves pròpies dades.
  • request.auth.uid assegura que l'usuari autenticat només pugui accedir als seus documents.

Conclusió

En aquesta secció, hem cobert tècniques avançades de consulta a Firebase, incloent-hi consultes compostes, indexació, paginació, actualitzacions en temps real i regles de seguretat. Dominar aquestes tècniques us ajudarà a construir aplicacions eficients i segures que poden gestionar tasques complexes de recuperació i manipulació de dades. Recordeu sempre considerar el rendiment i la seguretat quan dissenyeu les vostres consultes.
