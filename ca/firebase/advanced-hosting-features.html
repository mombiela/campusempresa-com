Introducció

Firebase Hosting és una eina potent per desplegar aplicacions web. Tot i que les funcions bàsiques d'allotjament són suficients per a molts projectes, les funcions avançades poden millorar significativament el rendiment, la seguretat i l'experiència de l'usuari de la teva aplicació. Aquesta secció cobrirà aquestes funcions avançades, incloent dominis personalitzats, reescriptures i redireccions d'URL, capçaleres personalitzades i desplegament de múltiples llocs.

Dominis personalitzats

Els dominis personalitzats et permeten utilitzar el teu propi nom de domini en lloc del subdomini predeterminat de Firebase.

  • Configuració d'un domini personalitzat:

    1. Ves a la consola de Firebase.
    2. Selecciona el teu projecte i navega a la secció d'Allotjament.
    3. Fes clic a "Afegeix un domini personalitzat".
    4. Segueix les instruccions per verificar la propietat del domini i configurar els registres DNS.

  • Exemple:

    Tipus: CNAME
Nom: www
Valor: your-project-id.firebaseapp.com

Reescriptures i redireccions d'URL

Les reescriptures i redireccions d'URL són essencials per gestionar l'estructura d'URL de la teva aplicació i assegurar una experiència d'usuari fluida.

  • Reescriptures: Mapeja un patró d'URL a un recurs específic, com una funció de Cloud o un fitxer estàtic.

  • Redireccions: Redirigeix automàticament una URL a una altra, útil per mantenir el SEO i els marcadors d'usuari.

  • Exemple de configuració:

    {
  "hosting": {
    "rewrites": [
      {
        "source": "/api/**",
        "function": "api"
      }
    ],
    "redirects": [
      {
        "source": "/old-page",
        "destination": "/new-page",
        "type": 301
      }
    ]
  }
}

Capçaleres personalitzades

Les capçaleres personalitzades es poden utilitzar per millorar la seguretat, controlar la memòria cau i gestionar els tipus de contingut.

  • Capçaleres comunes:

    • Content-Security-Policy: Prevé atacs XSS.
    • Cache-Control: Gestiona el comportament de la memòria cau.
    • Strict-Transport-Security: Enforça HTTPS.

  • Exemple de configuració:

    {
  "hosting": {
    "headers": [
      {
        "source": "**/*.@(js|css)",
        "headers": [
          {
            "key": "Cache-Control",
            "value": "max-age=31536000"
          }
        ]
      },
      {
        "source": "/**",
        "headers": [
          {
            "key": "Strict-Transport-Security",
            "value": "max-age=31536000; includeSubDomains"
          }
        ]
      }
    ]
  }
}

Desplegament de múltiples llocs

Firebase Hosting et permet desplegar múltiples llocs dins d'un sol projecte de Firebase. Això és útil per gestionar diferents entorns (per exemple, staging i producció) o allotjar múltiples aplicacions.

  • Passos per desplegar múltiples llocs:

    1. Defineix cada lloc al teu fitxer firebase.json.
    2. Utilitza la comanda firebase use --add per configurar múltiples configuracions de llocs.
    3. Desplega cada lloc utilitzant la comanda firebase deploy --only hosting:<site-name>.

  • Exemple de configuració:

    {
  "hosting": [
    {
      "target": "site1",
      "public": "public-site1",
      "ignore": ["firebase.json", "**/.*", "**/node_modules/**"]
    },
    {
      "target": "site2",
      "public": "public-site2",
      "ignore": ["firebase.json", "**/.*", "**/node_modules/**"]
    }
  ]
}

Conclusió

Les funcions avançades d'allotjament a Firebase proporcionen eines potents per millorar les teves aplicacions web. Els dominis personalitzats, les reescriptures i redireccions d'URL, les capçaleres personalitzades i la capacitat de desplegar múltiples llocs ofereixen flexibilitat i control sobre el teu entorn d'allotjament. En dominar aquestes funcions, pots optimitzar el rendiment, la seguretat i l'experiència de l'usuari de la teva aplicació.
