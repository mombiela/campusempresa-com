Introducció

Firebase Cloud Messaging (FCM) és una solució de missatgeria multiplataforma que et permet enviar missatges de manera fiable i sense cost. Aquesta secció cobrirà com gestionar les notificacions a la teva aplicació utilitzant Firebase, des de la configuració bàsica fins a la personalització avançada.

Configuració de Firebase Cloud Messaging

Prerequisits

  • Un projecte de Firebase configurat a la Consola de Firebase.
  • SDK de Firebase integrat a la teva aplicació (iOS, Android o Web).

Afegir FCM a la teva Aplicació

  1. iOS:

    • Afegeix l'SDK de Firebase al teu projecte utilitzant CocoaPods.
    • Configura la teva aplicació per a notificacions push.
    • Registra't per a notificacions remotes al teu delegat de l'aplicació.
    import Firebase
import UserNotifications

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate, UNUserNotificationCenterDelegate {
    func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        FirebaseApp.configure()
        UNUserNotificationCenter.current().delegate = self
        application.registerForRemoteNotifications()
        return true
    }
}

  2. Android:

    • Afegeix l'SDK de Firebase al teu projecte via Gradle.
    • Configura la teva aplicació per rebre notificacions modificant l'AndroidManifest.xml.
    <service
    android:name=".MyFirebaseMessagingService">
    <intent-filter>
        <action android:name="com.google.firebase.MESSAGING_EVENT"/>
    </intent-filter>
</service>

  3. Web:

    • Afegeix l'SDK de Firebase al teu projecte.
    • Sol·licita permís per enviar notificacions.
    import { getMessaging, getToken } from "firebase/messaging";

const messaging = getMessaging();
getToken(messaging, { vapidKey: "YOUR_PUBLIC_VAPID_KEY" }).then((currentToken) => {
    if (currentToken) {
        console.log('Token received: ', currentToken);
    } else {
        console.log('No registration token available.');
    }
}).catch((err) => {
    console.log('An error occurred while retrieving token. ', err);
});

Enviament de Notificacions

Utilitzant la Consola de Firebase

  • Navega a la Consola de Firebase.
  • Selecciona el teu projecte i ves a la secció de Cloud Messaging.
  • Fes clic a "Send your first message".
  • Omple els detalls de la notificació i dirigeix-la al teu públic.

Utilitzant l'SDK d'Administrador de Firebase

  • Instal·la l'SDK d'Administrador de Firebase al teu entorn de servidor.
  • Envia una notificació utilitzant l'SDK.
const admin = require('firebase-admin');
admin.initializeApp();

const message = {
    notification: {
        title: 'Hello World',
        body: 'This is a test notification'
    },
    token: 'RECIPIENT_DEVICE_TOKEN'
};

admin.messaging().send(message)
    .then((response) => {
        console.log('Successfully sent message:', response);
    })
    .catch((error) => {
        console.log('Error sending message:', error);
    });

Gestió de Notificacions a l'Aplicació

Notificacions en Primer Pla

  • iOS:

    • Implementa els mètodes de UNUserNotificationCenterDelegate per gestionar les notificacions mentre l'aplicació està en primer pla.
    func userNotificationCenter(_ center: UNUserNotificationCenter, willPresent notification: UNNotification, withCompletionHandler completionHandler: @escaping (UNNotificationPresentationOptions) -> Void) {
    completionHandler([.alert, .sound])
}

  • Android:

    • Sobreescriu el mètode onMessageReceived al teu FirebaseMessagingService.
    public class MyFirebaseMessagingService extends FirebaseMessagingService {
    @Override
    public void onMessageReceived(RemoteMessage remoteMessage) {
        // Handle the message here
    }
}

  • Web:

    • Gestiona els missatges utilitzant el mètode onMessage de la instància de Firebase Messaging.
    import { getMessaging, onMessage } from "firebase/messaging";

const messaging = getMessaging();
onMessage(messaging, (payload) => {
    console.log('Message received. ', payload);
});

Notificacions en Segon Pla

  • iOS & Android:

    • Les notificacions són gestionades pel sistema i es mostren a la safata de notificacions.

  • Web:

    • Utilitza un service worker per gestionar les notificacions en segon pla.
    self.addEventListener('push', function(event) {
    const data = event.data.json();
    const options = {
        body: data.body,
        icon: 'icon.png'
    };
    event.waitUntil(
        self.registration.showNotification(data.title, options)
    );
});

Personalització Avançada

Càrregues de Dades Personalitzades

  • Envia dades personalitzades juntament amb les notificacions per gestionar accions específiques o mostrar elements d'UI personalitzats.
const message = {
    data: {
        key1: 'value1',
        key2: 'value2'
    },
    token: 'RECIPIENT_DEVICE_TOKEN'
};

admin.messaging().send(message)
    .then((response) => {
        console.log('Successfully sent message:', response);
    })
    .catch((error) => {
        console.log('Error sending message:', error);
    });

Canals de Notificació (Android)

  • Crea canals de notificació per categoritzar les notificacions.
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
    NotificationChannel channel = new NotificationChannel("CHANNEL_ID", "Channel Name", NotificationManager.IMPORTANCE_DEFAULT);
    NotificationManager manager = getSystemService(NotificationManager.class);
    manager.createNotificationChannel(channel);
}

Conclusió

Gestionar notificacions amb Firebase Cloud Messaging implica configurar els SDKs necessaris, enviar notificacions a través de la Consola de Firebase o l'SDK d'Administrador, i gestionar les notificacions a la teva aplicació tant en estats de primer pla com de segon pla. La personalització avançada permet experiències d'usuari adaptades i una millor gestió de les notificacions. Seguint aquests passos, pots integrar i gestionar efectivament les notificacions a la teva aplicació.
