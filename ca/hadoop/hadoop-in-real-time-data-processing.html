Introducció al Processament de Dades en Temps Real

  • Definició: El processament de dades en temps real implica processar dades tan aviat com es produeixen o es reben.
  • Importància: Crític per a aplicacions que requereixen informació immediata, com la detecció de fraus, sistemes de recomanació i sistemes de monitoratge.

Visió General de l'Ecosistema Hadoop

  • Components de Hadoop:
    • HDFS (Hadoop Distributed File System): Capa d'emmagatzematge.
    • YARN (Yet Another Resource Negotiator): Capa de gestió de recursos.
    • MapReduce: Capa de processament.
  • Limitacions del Hadoop Tradicional:
    • Naturalesa del processament per lots.
    • Alta latència inadequada per a requisits en temps real.

Processament de Dades en Temps Real amb Hadoop

  • Tecnologies Clau:
    • Apache Kafka: Plataforma de transmissió distribuïda.
    • Apache Storm: Sistema de computació en temps real.
    • Apache Flink: Marc de processament de fluxos.
    • Apache Spark Streaming: Extensió d'Apache Spark per al processament de dades en temps real.

Apache Kafka

  • Visió General: Kafka s'utilitza per construir canals de dades en temps real i aplicacions de transmissió.
  • Conceptes Clau:
    • Productors: Envien dades a temes de Kafka.
    • Consumidors: Llegeixen dades dels temes de Kafka.
    • Brokers: Servidors de Kafka que emmagatzemen dades.
    • Temes: Categories a les quals s'envien dades.
  • Exemple: 
    // Exemple de productor en Java
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

Producer<String, String> producer = new KafkaProducer<>(props);
producer.send(new ProducerRecord<>("my-topic", "key", "value"));
producer.close();

Apache Storm

  • Visió General: Storm processa fluxos de dades en temps real.
  • Conceptes Clau:
    • Spouts: Fonts de fluxos de dades.
    • Bolts: Processen fluxos de dades.
    • Topologies: Xarxes de spouts i bolts.
  • Exemple: 
    // Exemple simple de topologia de Storm
TopologyBuilder builder = new TopologyBuilder();
builder.setSpout("spout", new RandomSpout());
builder.setBolt("bolt", new PrinterBolt()).shuffleGrouping("spout");

Config conf = new Config();
LocalCluster cluster = new LocalCluster();
cluster.submitTopology("test", conf, builder.createTopology());

Apache Flink

  • Visió General: Flink proporciona processament de fluxos amb alta capacitat i baixa latència.
  • Conceptes Clau:
    • API de DataStream: Per al processament de fluxos.
    • API de DataSet: Per al processament per lots.
  • Exemple: 
    // Exemple simple de streaming amb Flink
StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();
DataStream<String> text = env.socketTextStream("localhost", 9999);
DataStream<Tuple2<String, Integer>> counts = text
    .flatMap(new Tokenizer())
    .keyBy(0)
    .sum(1);
counts.print();
env.execute("WordCount");

Apache Spark Streaming

  • Visió General: Spark Streaming extén l'API principal de Spark per processar fluxos de dades en temps real.
  • Conceptes Clau:
    • DStream (Discretized Stream): Abstracció bàsica en Spark Streaming.
    • Transformacions: Operacions sobre DStreams.
    • Accions: Operacions que desencadenen l'execució.
  • Exemple: 
    // Exemple simple de Spark Streaming
val conf = new SparkConf().setMaster("local[2]").setAppName("NetworkWordCount")
val ssc = new StreamingContext(conf, Seconds(1))
val lines = ssc.socketTextStream("localhost", 9999)
val words = lines.flatMap(_.split(" "))
val pairs = words.map(word => (word, 1))
val wordCounts = pairs.reduceByKey(_ + _)
wordCounts.print()
ssc.start()
ssc.awaitTermination()

Conclusió

El processament de dades en temps real és essencial per a aplicacions que requereixen informació immediata. Tot i que el Hadoop tradicional està dissenyat per al processament per lots, integrar-lo amb tecnologies com Apache Kafka, Storm, Flink i Spark Streaming permet capacitats de processament de dades en temps real. Cada tecnologia té els seus punts forts i casos d'ús, i comprendre'ls pot ajudar a seleccionar l'eina adequada per a necessitats específiques de processament de dades en temps real.
