Introducció al Processament de Dades amb Hadoop

Hadoop és un marc de treball de codi obert que permet el processament distribuït de grans conjunts de dades a través de clústers d'ordinadors utilitzant models de programació simples. Aquesta secció introduirà els conceptes fonamentals de Hadoop i el seu ecosistema.

  • Hadoop Distributed File System (HDFS): Un sistema de fitxers distribuït que proporciona accés d'alt rendiment a les dades de les aplicacions.
  • MapReduce: Un model de programació per processar grans conjunts de dades amb un algorisme distribuït en un clúster de Hadoop.
  • YARN (Yet Another Resource Negotiator): Gestiona recursos i la programació de treballs a Hadoop.

HDFS: La Base de Hadoop

HDFS està dissenyat per emmagatzemar conjunts de dades molt grans de manera fiable i per transmetre aquests conjunts de dades a gran ample de banda a les aplicacions dels usuaris.

  • Blocs: HDFS divideix els fitxers en grans blocs (normalment 128 MB) i els distribueix a través dels nodes d'un clúster.
  • Replicació: Cada bloc es replica a través de múltiples nodes per assegurar la tolerància a fallades.
  • NameNode i DataNode: El NameNode gestiona l'espai de noms del sistema de fitxers i regula l'accés als fitxers per part dels clients. Els DataNodes gestionen l'emmagatzematge adjunt als nodes en què s'executen.

Exemple: Escriure i Llegir Fitxers a HDFS

# Pujar un fitxer a HDFS
hdfs dfs -put localfile.txt /user/hadoop/

# Llistar fitxers al directori HDFS
hdfs dfs -ls /user/hadoop/

# Llegir un fitxer de HDFS
hdfs dfs -cat /user/hadoop/localfile.txt

MapReduce: Processament de Dades a Hadoop

MapReduce és un model de programació i una implementació associada per processar i generar grans conjunts de dades.

  • Funció Map: Processa dades d'entrada i produeix un conjunt de parelles clau/valor intermèdies.
  • Funció Reduce: Fusiona tots els valors intermedis associats amb la mateixa clau intermèdia.

Exemple: Programa de Comptatge de Paraules

import java.io.IOException;
import java.util.StringTokenizer;
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.io.IntWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Job;
import org.apache.hadoop.mapreduce.Mapper;
import org.apache.hadoop.mapreduce.Reducer;
import org.apache.hadoop.mapreduce.lib.input.FileInputFormat;
import org.apache.hadoop.mapreduce.lib.output.FileOutputFormat;

public class WordCount {
    public static class TokenizerMapper extends Mapper<Object, Text, Text, IntWritable>{
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();

        public void map(Object key, Text value, Context context) throws IOException, InterruptedException {
            StringTokenizer itr = new StringTokenizer(value.toString());
            while (itr.hasMoreTokens()) {
                word.set(itr.nextToken());
                context.write(word, one);
            }
        }
    }

    public static class IntSumReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
        private IntWritable result = new IntWritable();

        public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            result.set(sum);
            context.write(key, result);
        }
    }

    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "word count");
        job.setJarByClass(WordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}

YARN: Gestió de Recursos a Hadoop

YARN és la capa de gestió de recursos de Hadoop. Permet que múltiples motors de processament de dades com el processament per lots, el processament de fluxos, el processament interactiu i el processament de gràfics s'executin i processin dades emmagatzemades a HDFS.

  • ResourceManager: Gestiona recursos i programa aplicacions.
  • NodeManager: Gestiona recursos en un sol node.
  • ApplicationMaster: Gestiona el cicle de vida de les aplicacions.

Processament de Dades Avançat amb l'Ecosistema Hadoop

L'ecosistema Hadoop inclou diverses eines i tecnologies que milloren les seves capacitats.

  • Hive: Infraestructura de magatzem de dades construïda sobre Hadoop per proporcionar resum de dades, consulta i anàlisi.
  • Pig: Plataforma d'alt nivell per crear programes MapReduce utilitzats amb Hadoop.
  • HBase: Magatzem de dades grans, distribuït i escalable, modelat després del Bigtable de Google.
  • Spark: Motor ràpid i general per al processament de dades a gran escala.

Exemple: Utilitzant Hive per a Consultes de Dades

-- Crear una taula a Hive
CREATE TABLE employee (id INT, name STRING, salary FLOAT)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n';

-- Carregar dades a la taula
LOAD DATA LOCAL INPATH '/path/to/employee.csv' INTO TABLE employee;

-- Consultar la taula
SELECT name, salary FROM employee WHERE salary > 50000;

Conclusió

Hadoop proporciona un marc robust per processar grans conjunts de dades en un entorn de computació distribuïda. Entendre HDFS, MapReduce i YARN és crucial per aprofitar tot el potencial de Hadoop. A més, eines com Hive, Pig, HBase i Spark amplien les capacitats de Hadoop, convertint-lo en una solució versàtil per al processament de grans dades. En dominar aquests components, podràs gestionar i analitzar grans quantitats de dades de manera eficient.
