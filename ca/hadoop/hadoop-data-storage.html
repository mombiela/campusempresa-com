Introducció a l'Emmagatzematge de Dades Hadoop

Hadoop és un marc de codi obert que permet el processament distribuït de grans conjunts de dades a través de clústers d'ordinadors utilitzant models de programació simples. Un dels components principals de Hadoop és el seu sistema d'emmagatzematge de dades, Hadoop Distributed File System (HDFS).

Conceptes Clau

  • HDFS (Hadoop Distributed File System): Un sistema de fitxers distribuït dissenyat per funcionar amb maquinari de mercaderia.
  • Blocs: La quantitat mínima de dades que HDFS pot llegir o escriure.
  • Replicació: HDFS emmagatzema múltiples còpies dels blocs de dades per garantir la fiabilitat i la tolerància a fallades.
  • NameNode: El servidor mestre que gestiona l'espai de noms del sistema de fitxers i controla l'accés als fitxers per part dels clients.
  • DataNode: Els nodes esclaus que emmagatzemen les dades reals.

Arquitectura HDFS

HDFS segueix una arquitectura mestre/esclau on el NameNode és el mestre i els DataNodes són els esclaus.

Components

  • NameNode: Gestiona les metadades i l'espai de noms del sistema de fitxers.
  • DataNode: Emmagatzema els blocs de dades reals.
  • Secondary NameNode: Realitza punts de control periòdics de les metadades del NameNode.

Flux de Dades en HDFS

  1. Operació d'Escriptura:

    • El client sol·licita escriure un fitxer.
    • El NameNode proporciona les ubicacions dels blocs.
    • Les dades s'escriuen als DataNodes en forma de pipeline.

  2. Operació de Lectura:

    • El client sol·licita llegir un fitxer.
    • El NameNode proporciona les ubicacions dels blocs.
    • El client llegeix les dades directament dels DataNodes.

Exemple: Escriure un Fitxer a HDFS

import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import java.io.OutputStream;

public class HDFSWriteExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path path = new Path("/user/hadoop/example.txt");
        OutputStream os = fs.create(path);
        os.write("Hello, HDFS!".getBytes());
        os.close();
        fs.close();
    }
}

Explicació:

  • Configuration conf: Carrega la configuració de Hadoop.
  • FileSystem fs: Obté la instància HDFS.
  • Path path: Especifica la ruta del fitxer a HDFS.
  • OutputStream os: Crea un flux de sortida per escriure dades a HDFS.
  • os.write(): Escriu dades al fitxer.
  • os.close(): Tanca el flux de sortida.
  • fs.close(): Tanca la instància HDFS.

Replicació de Dades i Tolerància a Fallades

HDFS garanteix la fiabilitat de les dades replicant cada bloc de dades a través de múltiples DataNodes.

Factor de Replicació

  • El factor de replicació per defecte és 3.
  • Es pot configurar segons els requisits.

Exemple: Configurar el Factor de Replicació

import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.fs.FSDataOutputStream;

public class HDFSReplicationExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path path = new Path("/user/hadoop/replicated_example.txt");
        FSDataOutputStream os = fs.create(path, true, 4096, (short) 2, 1048576);
        os.write("Replicated data in HDFS".getBytes());
        os.close();
        fs.close();
    }
}

Explicació:

  • (short) 2: Configura el factor de replicació a 2.

HDFS vs Sistemes de Fitxers Tradicionals

| Característica | HDFS | Sistemes de Fitxers Tradicionals | |------------------------|-------------------------------|----------------------------------| | Escalabilitat | Altament escalable | Escalabilitat limitada | | Tolerància a Fallades | Alta (degut a la replicació) | Baixa | | Accés a Dades | Optimitzat per grans conjunts de dades | Optimitzat per petits conjunts de dades | | Cost | Baix (maquinari de mercaderia) | Alt (maquinari especialitzat) |

Conclusió

El sistema d'emmagatzematge de dades de Hadoop, HDFS, està dissenyat per manejar grans volums de dades amb alta fiabilitat i escalabilitat. Entendre l'arquitectura, el flux de dades i els conceptes clau d'HDFS és crucial per treballar eficaçment amb Hadoop. Mitjançant l'ús d'HDFS, les organitzacions poden emmagatzemar i processar grans quantitats de dades de manera eficient, convertint-lo en una pedra angular de les solucions de big data.
