Introducció

Hadoop és un marc que permet el processament distribuït de grans conjunts de dades a través de clústers d'ordinadors utilitzant models de programació simples. Està dissenyat per escalar des de servidors únics fins a milers de màquines, cadascuna oferint computació i emmagatzematge local.

Orígens de Hadoop

  • Influència de Google: Els orígens de Hadoop es poden rastrejar fins al treball de Google sobre computació distribuïda. Google va publicar articles sobre el Google File System (GFS) i MapReduce, que van establir les bases per a Hadoop.
  • Doug Cutting i Mike Cafarella: Inspirats pels articles de Google, Doug Cutting i Mike Cafarella van començar a desenvolupar un projecte de codi obert anomenat Nutch, un motor de cerca web. Més tard van dividir la part de computació distribuïda de Nutch en un projecte separat, que es va convertir en Hadoop.

Fites Clau en l'Evolució de Hadoop

  • 2006: Hadoop va ser creat oficialment com a part del projecte Apache Lucene.
  • 2008: Hadoop es va convertir en un projecte de nivell superior d'Apache.
  • 2011: Es va llançar Hadoop 1.0, amb el Sistema de Fitxers Distribuït de Hadoop (HDFS) i MapReduce.
  • 2013: Es va llançar Hadoop 2.0, introduint YARN (Yet Another Resource Negotiator), que permetia una gestió de recursos i programació de treballs més flexible.
  • 2017: Es va llançar Hadoop 3.0, aportant millores significatives com la codificació d'esborrament HDFS, el servei de línia de temps YARN v.2, i el suport per a més de 2 NameNodes.

Components de Hadoop

Hadoop consta de diversos components clau que han evolucionat amb el temps:

  • HDFS (Hadoop Distributed File System): Un sistema de fitxers distribuït que proporciona accés d'alt rendiment a les dades de l'aplicació.
  • MapReduce: Un model de programació per al processament de dades a gran escala.
  • YARN (Yet Another Resource Negotiator): Una capa de gestió de recursos per a la programació i gestió de recursos del clúster.
  • Hadoop Common: Les utilitats i biblioteques comunes que donen suport als altres mòduls de Hadoop.

Evolució de l'Ecosistema Hadoop

Al llarg dels anys, l'ecosistema Hadoop s'ha expandit per incloure una varietat d'eines i projectes que milloren les seves capacitats:

  • Hive: Una infraestructura de magatzem de dades construïda sobre Hadoop per proporcionar resum de dades, consulta i anàlisi.
  • Pig: Una plataforma d'alt nivell per crear programes MapReduce utilitzats amb Hadoop.
  • HBase: Un magatzem de dades distribuït, escalable i gran modelat després del Bigtable de Google.
  • Spark: Un marc de computació en clúster de propòsit general de codi obert.
  • Flink: Un marc de processament de fluxos que pot processar dades en temps real.

Exemple de Codi: Programa Simple MapReduce

Per il·lustrar la funcionalitat bàsica de Hadoop, vegem un programa simple MapReduce escrit en Java. Aquest programa compta el nombre d'ocurrències de cada paraula en un fitxer de text.

Classe Mapper

import java.io.IOException;
import org.apache.hadoop.io.IntWritable;
import org.apache.hadoop.io.LongWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Mapper;

public class TokenizerMapper extends Mapper<LongWritable, Text, Text, IntWritable> {
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();

    public void map(LongWritable key, Text value, Context context) throws IOException, InterruptedException {
        String[] tokens = value.toString().split("\\s+");
        for (String token : tokens) {
            word.set(token);
            context.write(word, one);
        }
    }
}

Classe Reducer

import java.io.IOException;
import org.apache.hadoop.io.IntWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Reducer;

public class IntSumReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
    public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        context.write(key, new IntWritable(sum));
    }
}

Classe Driver

import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.io.IntWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Job;
import org.apache.hadoop.mapreduce.lib.input.FileInputFormat;
import org.apache.hadoop.mapreduce.lib.output.FileOutputFormat;

public class WordCount {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "word count");
        job.setJarByClass(WordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}

Conclusió

La història i evolució de Hadoop demostren la seva transformació d'un projecte simple inspirat pels articles de recerca de Google a un ecosistema robust i complet per al processament de grans dades. Entendre aquesta evolució ajuda a apreciar les decisions de disseny i les millores fetes al llarg dels anys, fent de Hadoop una pedra angular en el món de les grans dades.
