Introducció a MapReduce

MapReduce és un model de programació i una implementació associada per processar i generar grans conjunts de dades amb un algorisme paral·lel i distribuït en un clúster. És un component central de Hadoop i s'utilitza per processar grans quantitats de dades de manera fiable i tolerant a fallades.

Conceptes clau de MapReduce

  • Funció Map: Processa parells clau-valor d'entrada per generar un conjunt de parells clau-valor intermedis.
  • Funció Reduce: Fusiona tots els valors intermedis associats amb la mateixa clau intermèdia.
  • Treball: Un càlcul complet de MapReduce, que consisteix en múltiples tasques.
  • Tasca: Una unitat de treball individual, ja sigui una tasca de map o una tasca de reduce.
  • Job Tracker: Gestiona els treballs i els recursos en el clúster de Hadoop.
  • Task Tracker: Executa les tasques segons les indicacions del Job Tracker.

Flux d'execució de treballs MapReduce

  1. Divisions d'entrada: Les dades d'entrada es divideixen en peces de mida fixa anomenades divisions d'entrada.
  2. Fase Map: Cada divisió és processada per una tasca de map, que genera parells clau-valor intermedis.
  3. Barreja i ordenació: Les dades intermèdies es barregen i s'ordenen per agrupar tots els valors per clau.
  4. Fase Reduce: Les dades agrupades són processades per tasques de reduce per produir el resultat final.
  5. Sortida: El resultat final s'escriu al sistema de fitxers distribuït.

Passos detallats de l'execució de treballs MapReduce

Divisions d'entrada

  • Les dades d'entrada es divideixen en divisions, que es processen en paral·lel per tasques de map.
  • Exemple: 
    FileInputFormat.addInputPath(job, new Path("input"));

Fase Map

  • Cada tasca de map processa una divisió i genera parells clau-valor intermedis.
  • Exemple: 
    public class TokenizerMapper extends Mapper<Object, Text, Text, IntWritable>{
    private final static IntWritable one = new IntWritable(1);
    private Text word = new Text();

    public void map(Object key, Text value, Context context) throws IOException, InterruptedException {
        StringTokenizer itr = new StringTokenizer(value.toString());
        while (itr.hasMoreTokens()) {
            word.set(itr.nextToken());
            context.write(word, one);
        }
    }
}

Barreja i ordenació

  • Els parells clau-valor intermedis es barregen i s'ordenen per clau.
  • Aquest pas assegura que tots els valors per a una clau donada s'enviïn al mateix reducer.

Fase Reduce

  • Les tasques de reduce processen les dades agrupades i generen el resultat final.
  • Exemple: 
    public class IntSumReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
    private IntWritable result = new IntWritable();

    public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
        int sum = 0;
        for (IntWritable val : values) {
            sum += val.get();
        }
        result.set(sum);
        context.write(key, result);
    }
}

Sortida

  • El resultat final s'escriu al sistema de fitxers distribuït.
  • Exemple: 
    FileOutputFormat.setOutputPath(job, new Path("output"));

Exemple: Programa de recompte de paraules

Aquí teniu un exemple complet d'un programa MapReduce que compta les ocurrències de cada paraula en un fitxer de text.

import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.io.IntWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Job;
import org.apache.hadoop.mapreduce.Mapper;
import org.apache.hadoop.mapreduce.Reducer;
import org.apache.hadoop.mapreduce.lib.input.FileInputFormat;
import org.apache.hadoop.mapreduce.lib.output.FileOutputFormat;

import java.io.IOException;
import java.util.StringTokenizer;

public class WordCount {

    public static class TokenizerMapper extends Mapper<Object, Text, Text, IntWritable>{
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();

        public void map(Object key, Text value, Context context) throws IOException, InterruptedException {
            StringTokenizer itr = new StringTokenizer(value.toString());
            while (itr.hasMoreTokens()) {
                word.set(itr.nextToken());
                context.write(word, one);
            }
        }
    }

    public static class IntSumReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
        private IntWritable result = new IntWritable();

        public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            result.set(sum);
            context.write(key, result);
        }
    }

    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "word count");
        job.setJarByClass(WordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}

Conclusió

MapReduce és una eina poderosa per processar grans conjunts de dades de manera distribuïda. Entendre el flux d'execució dels treballs, des de les divisions d'entrada fins a la sortida final, és crucial per escriure programes MapReduce eficients. En dominar aquests conceptes i practicar amb exemples com el programa de recompte de paraules, podeu aprofitar tot el potencial de Hadoop per afrontar els reptes del big data.
