Introducció

Hadoop és un marc de treball de codi obert que permet el processament distribuït de grans conjunts de dades a través de clústers d'ordinadors utilitzant models de programació simples. Està dissenyat per escalar des de servidors únics fins a milers de màquines, cadascuna oferint computació i emmagatzematge local. Aquesta secció cobrirà els components bàsics de Hadoop, que són essencials per entendre com funciona Hadoop i com utilitzar-lo de manera efectiva.

Components bàsics de Hadoop

1. Sistema de fitxers distribuït de Hadoop (HDFS)

HDFS és el sistema d'emmagatzematge principal utilitzat per les aplicacions de Hadoop. Proporciona accés d'alt rendiment a les dades de l'aplicació i està dissenyat per ser altament tolerant a fallades.

  • Conceptes clau:

    • Blocs: HDFS emmagatzema cada fitxer com una sèrie de blocs. Cada bloc és típicament de 128 MB per defecte.
    • Namenode: El servidor mestre que gestiona l'espai de noms del sistema de fitxers i controla l'accés als fitxers per part dels clients.
    • Datanodes: Aquests són els nodes treballadors que emmagatzemen i recuperen blocs quan se'ls indica (pel Namenode).

  • Exemple:

    # Llistar fitxers a HDFS
hdfs dfs -ls /

# Carregar un fitxer a HDFS
hdfs dfs -put localfile.txt /hdfs/directory/

# Llegir un fitxer de HDFS
hdfs dfs -cat /hdfs/directory/localfile.txt

2. MapReduce

MapReduce és un model de programació i una implementació associada per processar i generar grans conjunts de dades amb un algorisme paral·lel i distribuït en un clúster.

  • Conceptes clau:

    • Funció Map: Processa parells clau/valor d'entrada per generar un conjunt de parells clau/valor intermedis.
    • Funció Reduce: Fusiona tots els valors intermedis associats amb la mateixa clau intermèdia.

  • Exemple:

    public class WordCount {
    public static class TokenizerMapper
         extends Mapper<Object, Text, Text, IntWritable>{
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();

        public void map(Object key, Text value, Context context
                        ) throws IOException, InterruptedException {
            StringTokenizer itr = new StringTokenizer(value.toString());
            while (itr.hasMoreTokens()) {
                word.set(itr.nextToken());
                context.write(word, one);
            }
        }
    }

    public static class IntSumReducer
         extends Reducer<Text,IntWritable,Text,IntWritable> {
        private IntWritable result = new IntWritable();

        public void reduce(Text key, Iterable<IntWritable> values,
                           Context context
                           ) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            result.set(sum);
            context.write(key, result);
        }
    }
}

3. YARN (Yet Another Resource Negotiator)

YARN és el sistema de gestió de recursos del clúster de Hadoop. Permet que múltiples motors de processament de dades com SQL interactiu, streaming en temps real, ciència de dades i processament per lots s'executin i processin dades emmagatzemades a HDFS.

  • Conceptes clau:

    • ResourceManager: El mestre que arbitra tots els recursos disponibles del clúster.
    • NodeManager: L'agent de marc per màquina responsable dels contenidors, monitoritzant el seu ús de recursos (CPU, memòria, disc, xarxa) i informant del mateix al ResourceManager.
    • ApplicationMaster: Gestiona el cicle de vida de les aplicacions que s'executen a YARN.

  • Exemple:

    # Comprovar l'estat de YARN
yarn application -list

# Matar una aplicació de YARN
yarn application -kill application_1234567890123_0001

4. Hadoop Common

Hadoop Common conté biblioteques i utilitats necessàries per altres mòduls de Hadoop. Aquestes biblioteques proporcionen abstraccions de sistema de fitxers i nivell de sistema operatiu i contenen els fitxers Java i scripts necessaris per iniciar Hadoop.

  • Conceptes clau:

    • Configuració: Proporciona una manera de configurar els diferents paràmetres de Hadoop.
    • Serialització: Mecanisme per convertir estructures de dades o estat d'objectes en un format que es pot emmagatzemar o transmetre i reconstruir més tard.

  • Exemple:

    Configuration conf = new Configuration();
conf.set("fs.defaultFS", "hdfs://namenode:8020");
FileSystem fs = FileSystem.get(conf);

Conclusió

Entendre els components bàsics de Hadoop és crucial per utilitzar efectivament el marc per al processament de grans dades. HDFS proporciona la capa d'emmagatzematge, MapReduce ofereix el model de processament, YARN gestiona els recursos, i Hadoop Common inclou utilitats essencials. Junts, aquests components formen l'esquelet de l'ecosistema Hadoop, permetent un processament de dades escalable i eficient en entorns distribuïts.
