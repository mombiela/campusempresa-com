Introducció

En aquesta secció, explorarem diversos estudis de cas del món real on Hadoop ha estat implementat amb èxit. Aquests estudis de cas t'ajudaran a entendre les aplicacions pràctiques de Hadoop i com pot resoldre problemes complexos de processament de dades.

Estudi de Cas 1: Analítica de Venda al Detall

Visió General

Una gran empresa de venda al detall volia analitzar el comportament dels clients per millorar les seves estratègies de màrqueting i la gestió d'inventari. Tenien enormes quantitats de dades transaccionals de diverses fonts, incloent vendes en línia, compres a la botiga i programes de fidelització de clients.

Conceptes Clau

  • Ingestió de Dades: Recollida de dades de múltiples fonts.
  • Emmagatzematge de Dades: Utilització de HDFS (Hadoop Distributed File System) per emmagatzemar grans conjunts de dades.
  • Processament de Dades: Utilització de MapReduce per processar i analitzar les dades.
  • Anàlisi de Dades: Ús d'eines com Hive i Pig per fer consultes i analitzar les dades.

Implementació

  1. Ingestió de Dades:

    • L'empresa va utilitzar Apache Flume per recollir dades dels servidors web i emmagatzemar-les a HDFS.
    • Sqoop es va utilitzar per importar dades de bases de dades relacionals a Hadoop.

  2. Emmagatzematge de Dades:

    • Totes les dades recollides es van emmagatzemar a HDFS, que proporcionava una solució d'emmagatzematge escalable i tolerant a fallades.

  3. Processament de Dades:

    public class TransactionMapper extends Mapper<LongWritable, Text, Text, IntWritable> {
    private Text customerId = new Text();
    private IntWritable amount = new IntWritable();

    public void map(LongWritable key, Text value, Context context) throws IOException, InterruptedException {
        String[] fields = value.toString().split(",");
        customerId.set(fields[0]);
        amount.set(Integer.parseInt(fields[2]));
        context.write(customerId, amount);
    }
}

public class TransactionReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
    public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
        int totalAmount = 0;
        for (IntWritable value : values) {
            totalAmount += value.get();
        }
        context.write(key, new IntWritable(totalAmount));
    }
}
    • La feina de MapReduce anterior es va utilitzar per calcular la quantitat total gastada per cada client.

  4. Anàlisi de Dades:

    • Hive es va utilitzar per executar consultes similars a SQL sobre les dades processades per generar informes i coneixements.
    • Exemple de Consulta Hive:
    SELECT customerId, SUM(amount) as totalSpent
FROM transactions
GROUP BY customerId;

Resultats

  • L'empresa va poder identificar clients d'alt valor i dirigir-los campanyes de màrqueting personalitzades.
  • La gestió d'inventari es va optimitzar basant-se en les tendències de compra, reduint l'excés d'estoc i les ruptures d'estoc.

Estudi de Cas 2: Anàlisi de Dades Sanitàries

Visió General

Un proveïdor de serveis sanitaris necessitava analitzar els registres dels pacients per millorar els resultats dels tractaments i l'eficiència operativa. Tenien una gran quantitat de dades no estructurades, incloent registres mèdics, resultats de laboratoris i dades d'imatges.

Conceptes Clau

  • Integració de Dades: Combinació de dades estructurades i no estructurades.
  • Emmagatzematge de Dades: Utilització de HDFS per a un emmagatzematge escalable.
  • Processament de Dades: Utilització de Spark per a un processament de dades més ràpid.
  • Aprenentatge Automàtic: Aplicació d'algoritmes de ML per predir els resultats dels pacients.

Implementació

  1. Integració de Dades:

    • Apache Nifi es va utilitzar per ingerir dades de diverses fonts, incloent registres electrònics de salut (EHR) i sistemes de laboratori.

  2. Emmagatzematge de Dades:

    • Totes les dades es van emmagatzemar a HDFS, assegurant escalabilitat i fiabilitat.

  3. Processament de Dades:

    from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("HealthcareDataAnalysis").getOrCreate()
df = spark.read.csv("hdfs://path/to/medical_records.csv", header=True, inferSchema=True)

df.createOrReplaceTempView("patients")
result = spark.sql("SELECT age, AVG(blood_pressure) as avg_bp FROM patients GROUP BY age")
result.show()

  4. Aprenentatge Automàtic:

    • Spark MLlib es va utilitzar per construir models predictius.
    from pyspark.ml.classification import LogisticRegression
from pyspark.ml.feature import VectorAssembler

assembler = VectorAssembler(inputCols=["age", "blood_pressure", "cholesterol"], outputCol="features")
data = assembler.transform(df)

lr = LogisticRegression(featuresCol="features", labelCol="outcome")
model = lr.fit(data)
predictions = model.transform(data)
predictions.select("age", "blood_pressure", "cholesterol", "prediction").show()

Resultats

  • El proveïdor de serveis sanitaris va poder predir els resultats dels pacients amb més precisió, conduint a millors plans de tractament.
  • L'eficiència operativa va millorar identificant colls d'ampolla en el procés de tractament.

Conclusió

Aquests estudis de cas demostren la versatilitat i el poder de Hadoop per resoldre desafiaments complexos de processament de dades en diverses indústries. Entenent aquestes aplicacions del món real, pots apreciar millor el potencial de Hadoop i com es pot aplicar a les teves pròpies necessitats de processament de dades.
