Introducció a HDFS

HDFS, o Hadoop Distributed File System, és el sistema d'emmagatzematge principal utilitzat per les aplicacions Hadoop. Està dissenyat per emmagatzemar grans conjunts de dades de manera fiable i per transmetre aquests conjunts de dades a gran amplada de banda a les aplicacions d'usuari.

Conceptes Clau

  • Escalabilitat: HDFS pot escalar fins a milers de nodes i petabytes de dades.
  • Tolerància a Fallades: Les dades es repliquen a través de múltiples nodes per garantir la fiabilitat.
  • Alt Rendiment: Dissenyat per a un alt rendiment de dades, no per a baixa latència.
  • Accés a Dades en Streaming: Optimitzat per a grans conjunts de dades i processament per lots.
  • Maquinari de Commoditat: Funciona en maquinari econòmic i de prestatge.

Arquitectura de HDFS

HDFS segueix una arquitectura mestre-esclau on el mestre s'anomena NameNode i els esclaus s'anomenen DataNodes.

NameNode

  • Rol: Gestiona les metadades i l'espai de noms del sistema de fitxers.
  • Responsabilitats:
    • Fa un seguiment de l'arbre del sistema de fitxers i les metadades de tots els fitxers i directoris.
    • Gestiona els DataNodes i els blocs que emmagatzemen.
    • Gestiona les sol·licituds dels clients per a operacions de fitxers.

DataNode

  • Rol: Emmagatzema els blocs de dades reals.
  • Responsabilitats:
    • Serveix sol·licituds de lectura i escriptura dels clients del sistema de fitxers.
    • Realitza la creació, eliminació i replicació de blocs segons les instruccions del NameNode.

Emmagatzematge de Blocs

  • Mida del Bloc: La mida del bloc per defecte és de 128 MB.
  • Replicació: El factor de replicació per defecte és 3.
  • Avantatges:
    • La mida gran del bloc minimitza el cost de les cerques.
    • La replicació garanteix la fiabilitat i disponibilitat de les dades.

Operacions de HDFS

Lectura i Escriptura de Fitxers

  • Operació d'Escriptura:

    1. El client sol·licita al NameNode crear un nou fitxer.
    2. El NameNode comprova els permisos i crea les metadades.
    3. El client escriu dades als DataNodes en blocs.
    4. Els DataNodes repliquen els blocs a altres DataNodes.

  • Operació de Lectura:

    1. El client sol·licita al NameNode les metadades del fitxer.
    2. El NameNode proporciona la llista de DataNodes que contenen els blocs.
    3. El client llegeix els blocs directament dels DataNodes.

Exemple de Codi: Escriptura i Lectura de Fitxers

// Escriptura d'un fitxer a HDFS
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import java.io.OutputStream;

public class HDFSWriteExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path path = new Path("/user/hadoop/example.txt");
        OutputStream os = fs.create(path);
        os.write("Hello HDFS!".getBytes());
        os.close();
    }
}

// Lectura d'un fitxer de HDFS
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import java.io.InputStream;
import java.io.BufferedReader;
import java.io.InputStreamReader;

public class HDFSReadExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path path = new Path("/user/hadoop/example.txt");
        InputStream is = fs.open(path);
        BufferedReader br = new BufferedReader(new InputStreamReader(is));
        String line;
        while ((line = br.readLine()) != null) {
            System.out.println(line);
        }
        br.close();
    }
}

Funcions Avançades de HDFS

Federació

  • Propòsit: Escalar el NameNode horitzontalment.
  • Com funciona: Múltiples NameNodes gestionen diferents espais de noms, i els DataNodes emmagatzemen blocs per a múltiples NameNodes.

Alta Disponibilitat (HA)

  • Propòsit: Assegurar que el NameNode no sigui un únic punt de fallada.
  • Com funciona: Utilitza una parella de NameNodes en una configuració activa-redundant.

Instantànies

  • Propòsit: Capturar l'estat del sistema de fitxers en un moment específic.
  • Com funciona: Permet instantànies de només lectura o escriptura del sistema de fitxers.

Conclusió

HDFS és un sistema de fitxers robust i escalable dissenyat per gestionar grans conjunts de dades amb alt rendiment. La seva arquitectura garanteix la fiabilitat i disponibilitat de les dades mitjançant la replicació i la tolerància a fallades. Entendre les operacions i funcions de HDFS és crucial per aprofitar tot el potencial de Hadoop en el processament de grans volums de dades.
