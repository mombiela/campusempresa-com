Apache Zookeeper és un servei centralitzat per mantenir informació de configuració, nomenclatura, proporcionar sincronització distribuïda i oferir serveis de grup. És un component crític en l'ecosistema Hadoop, assegurant la coordinació i gestió d'aplicacions distribuïdes.

Introducció a Apache Zookeeper

  • Què és Zookeeper?

    • Un servei de coordinació distribuïda.
    • Proporciona un conjunt de primitives per implementar serveis de sincronització, manteniment de configuració i nomenclatura de nivell superior.

  • Per què Zookeeper?

    • Assegura alta disponibilitat i fiabilitat.
    • Simplifica la complexitat dels sistemes distribuïts.
    • Gestiona la configuració i la sincronització en un entorn distribuït.

Arquitectura de Zookeeper

  • Components Clau:

    • Znodes: La unitat de dades fonamental en Zookeeper, similar a un node de sistema de fitxers.
    • Sessions: Sessions de clients que mantenen una connexió amb l'ensemble de Zookeeper.
    • Watches: Mecanisme per notificar els clients dels canvis en els znodes.
    • Líder i Seguidors: Zookeeper utilitza un model de líder-seguidor per al consens.

  • Ensemble de Zookeeper:

    • Un grup de servidors Zookeeper.
    • Normalment, un nombre imparell de servidors per evitar escenaris de split-brain.
    • Assegura tolerància a fallades i alta disponibilitat.

Configuració de Zookeeper

  • Instal·lació:

    • Descarrega l'última versió estable des del lloc web d'Apache Zookeeper.
    • Extreu el tarball i navega al directori extret.
    tar -xzf zookeeper-3.7.0.tar.gz
cd zookeeper-3.7.0

  • Configuració:

    • Crea un fitxer de configuració zoo.cfg al directori conf.
    cp conf/zoo_sample.cfg conf/zoo.cfg
    • Edita zoo.cfg per establir el directori de dades i els detalls del servidor.
    dataDir=/var/lib/zookeeper
server.1=zookeeper1:2888:3888
server.2=zookeeper2:2888:3888
server.3=zookeeper3:2888:3888

  • Iniciant Zookeeper:

    bin/zkServer.sh start

Operacions de Zookeeper

  • Comandes bàsiques de CLI:

    • Inicia el CLI de Zookeeper:
    bin/zkCli.sh -server localhost:2181
    • Crea un znode:
    create /my_znode "my_data"
    • Llegeix un znode:
    get /my_znode
    • Actualitza un znode:
    set /my_znode "new_data"
    • Elimina un znode:
    delete /my_znode

Conceptes Avançats de Zookeeper

  • Watches de Zookeeper:

    • Els watches són desencadenants únics que notifiquen els clients dels canvis en els znodes.
    • Exemple de configuració d'un watch:
    get /my_znode watch

  • Elecció de Líder:

    • Zookeeper es pot utilitzar per implementar l'elecció de líder en sistemes distribuïts.
    • Exemple de codi per a l'elecció de líder:
    public class LeaderElection implements Watcher {
    private ZooKeeper zk;
    private String znode = "/election";

    public LeaderElection(String hostPort) throws IOException {
        zk = new ZooKeeper(hostPort, 3000, this);
    }

Casos d'Ús de Zookeeper en Hadoop

  • Hadoop YARN:

    • Zookeeper s'utilitza per a la gestió de recursos i la programació de treballs.
    • Assegura alta disponibilitat del ResourceManager.

  • HBase:

    • Utilitza Zookeeper per a la coordinació distribuïda.
    • Gestiona els servidors de regions i assegura la consistència de les dades.

Conclusió

Apache Zookeeper és una eina poderosa per gestionar sistemes distribuïts. Simplifica la complexitat de la coordinació, configuració i sincronització en un entorn distribuït. Entendre Zookeeper és crucial per gestionar i desplegar efectivament ecosistemes Hadoop. Dominant Zookeeper, pots assegurar alta disponibilitat, fiabilitat i una gestió eficient de les teves aplicacions distribuïdes.
