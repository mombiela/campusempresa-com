Introducció

En aquesta secció, explorarem els conceptes d'interrupcions i crides al sistema en llenguatge assemblador. Aquests són crucials per interactuar amb el maquinari i el sistema operatiu, permetent que els programes realitzin tasques com operacions d'entrada/sortida, control de processos i més.

Interrupcions

Les interrupcions són senyals que informen el processador que un esdeveniment necessita atenció immediata. Poden ser desencadenades per maquinari o programari.

Conceptes Clau

  • Interrupcions de Maquinari: Generades per dispositius de maquinari per senyalitzar que necessiten processament.
  • Interrupcions de Programari: Generades per instruccions de programari per sol·licitar serveis del sistema operatiu.
  • Taula de Vectors d'Interrupció (IVT): Una taula que conté les adreces de les rutines de servei d'interrupció (ISRs).

Exemple: Gestionant una Interrupció del Teclat

section .data
    msg db 'Tecla premuda!', 0

section .bss

section .text
    global _start

_start:
    ; Habilitar interrupcions del teclat
    in al, 0x21
    and al, 0xFD
    out 0x21, al

    ; Bucle infinit per mantenir el programa en execució
    jmp $

keyboard_interrupt:
    pusha
    ; Gestionar la interrupció del teclat
    mov edx, msg
    call print_string
    popa
    iret

print_string:
    ; Imprimir la cadena apuntada per EDX
    mov eax, 4
    mov ebx, 1
    mov ecx, edx
    mov edx, 13
    int 0x80
    ret

En aquest exemple, habilitem les interrupcions del teclat i definim una simple rutina de servei d'interrupció (ISR) per imprimir un missatge quan es prem una tecla.

Crides al Sistema

Les crides al sistema proporcionen una interfície controlada perquè els programes sol·licitin serveis del nucli del sistema operatiu.

Conceptes Clau

  • Interfície de Crida al Sistema: El mecanisme a través del qual les aplicacions d'espai d'usuari sol·liciten serveis del nucli.
  • Registres: Es fan servir registres específics per passar arguments a les crides al sistema (per exemple, eax per al número de crida al sistema).

Exemple: Utilitzant una Crida al Sistema per Escriure a la Sortida Estàndard

section .data
    msg db 'Hola, Món!', 0xA

section .text
    global _start

_start:
    ; Escriure el missatge a la sortida estàndard
    mov eax, 4          ; número de crida al sistema sys_write
    mov ebx, 1          ; descriptor de fitxer 1 (stdout)
    mov ecx, msg        ; punter al missatge
    mov edx, 13         ; longitud del missatge
    int 0x80            ; interrupció per invocar la crida al sistema

    ; Sortir del programa
    mov eax, 1          ; número de crida al sistema sys_exit
    xor ebx, ebx        ; codi de sortida 0
    int 0x80            ; interrupció per invocar la crida al sistema

En aquest exemple, utilitzem la crida al sistema sys_write per imprimir "Hola, Món!" a la sortida estàndard i després sortim del programa utilitzant la crida al sistema sys_exit.

Comparació d'Interrupcions i Crides al Sistema

| Característica | Interrupcions | Crides al Sistema | |------------------|-----------------------------------|-----------------------------------| | Desencadenat per | Maquinari o programari | Programari | | Propòsit | Gestionar esdeveniments asíncrons | Sol·licitar serveis del SO | | Transferència de Control | Taula de vectors d'interrupció (IVT) | Interfície de crida al sistema | | Exemple d'Ús | Entrada de teclat, interrupcions de temporitzador | Operacions de fitxers, control de processos |

Conclusió

Entendre les interrupcions i les crides al sistema és essencial per a la programació de baix nivell i per interactuar amb el maquinari i el sistema operatiu. Les interrupcions permeten que el teu programa respongui a esdeveniments asíncrons, mentre que les crides al sistema proporcionen una manera de realitzar operacions privilegiades de manera controlada. Dominar aquests conceptes et permetrà escriure programes en assemblador més eficients i potents.
