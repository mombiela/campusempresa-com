Introducció a l'ARM Assembly

ARM (Advanced RISC Machine) és una família d'arquitectures de computació de conjunt d'instruccions reduïdes (RISC) per a processadors d'ordinador. Els processadors ARM s'utilitzen àmpliament en dispositius mòbils, sistemes incrustats i, cada vegada més, en servidors i ordinadors de sobretaula.

Conceptes Clau

  • Arquitectura RISC: ARM es basa en els principis RISC, que tenen com a objectiu simplificar les instruccions per aconseguir un rendiment més alt.
  • Registres: ARM té 16 registres d'ús general (R0-R15), amb rols específics per a alguns registres (per exemple, R15 és el Program Counter).
  • Conjunt d'Instruccions: Les instruccions ARM solen tenir 32 bits de llarg i estan dissenyades per executar-se en un sol cicle.
  • Endianness: Els processadors ARM es poden configurar per utilitzar formats de memòria little-endian o big-endian.

Conceptes Bàsics de l'ARM Assembly

Registres i Tipus de Dades

Els processadors ARM tenen un conjunt de 16 registres d'ús general, anomenats R0 a R15. Alguns d'aquests registres tenen propòsits especials:

  • R13 (SP): Stack Pointer
  • R14 (LR): Link Register
  • R15 (PC): Program Counter

Instruccions Bàsiques

  • MOV: Mou dades d'un registre a un altre.
  • ADD: Suma dos registres.
  • SUB: Resta un registre d'un altre.
  • CMP: Compara dos registres.
  • B: Salta a una etiqueta.

Exemple: Suma Simple

    MOV R0, #5        ; Carrega el valor immediat 5 a R0
    MOV R1, #10       ; Carrega el valor immediat 10 a R1
    ADD R2, R0, R1    ; Suma R0 i R1, emmagatzema el resultat a R2

Explicació

  • MOV R0, #5: Mou el valor immediat 5 al registre R0.
  • MOV R1, #10: Mou el valor immediat 10 al registre R1.
  • ADD R2, R0, R1: Suma els valors a R0 i R1, i emmagatzema el resultat a R2.

Flux de Control

Execució Condicional

ARM admet l'execució condicional d'instruccions utilitzant codis de condició. Els codis de condició comuns inclouen:

  • EQ: Igual
  • NE: No Igual
  • GT: Més Gran Que
  • LT: Menys Que

Exemple: Salt Condicional

    CMP R0, R1        ; Compara R0 i R1
    BEQ equal_label   ; Salta a equal_label si R0 == R1
    BNE not_equal_label ; Salta a not_equal_label si R0 != R1
equal_label:
    ; Codi per al cas d'igualtat
not_equal_label:
    ; Codi per al cas de no igualtat

Explicació

  • CMP R0, R1: Compara els valors a R0 i R1.
  • BEQ equal_label: Salta a equal_label si els valors són iguals.
  • BNE not_equal_label: Salta a not_equal_label si els valors no són iguals.

Temes Avançats

Subrutines i Gestió de la Pila

Les subrutines s'utilitzen per estructurar el codi en blocs reutilitzables. La pila s'utilitza per gestionar les crides a funcions i les variables locals.

Exemple: Crida a Subrutina

    BL my_subroutine  ; Salta amb enllaç a my_subroutine
    ; Continua l'execució aquí després de la subrutina

my_subroutine:
    PUSH {LR}         ; Desa el Link Register
    ; Codi de la subrutina
    POP {LR}          ; Restaura el Link Register
    BX LR             ; Retorna de la subrutina

Explicació

  • BL my_subroutine: Salta a my_subroutine i desa l'adreça de retorn al Link Register (LR).
  • PUSH {LR}: Desa el Link Register a la pila.
  • POP {LR}: Restaura el Link Register de la pila.
  • BX LR: Salta a l'adreça al Link Register, efectivament retornant de la subrutina.

Assembly Inline en C

Combinar l'ARM assembly amb C pot optimitzar seccions de codi crítiques per al rendiment.

Exemple: Assembly Inline

void add_numbers(int a, int b, int *result) {
    asm("ADD %[res], %[val1], %[val2]"
        : [res] "=r" (*result)
        : [val1] "r" (a), [val2] "r" (b));
}

Explicació

  • asm: Paraula clau d'assembly inline en C.
  • ADD %[res], %[val1], %[val2]: Instrucció d'assembly per sumar val1 i val2, emmagatzemant el resultat a res.
  • : [res] "=r" (*result): Operand de sortida, especificant que res és un registre i es desarà a *result.
  • : [val1] "r" (a), [val2] "r" (b): Operands d'entrada, especificant que val1 i val2 són registres que contenen a i b.

Resum

L'ARM Assembly és un llenguatge potent per programar processadors ARM, oferint un equilibri entre simplicitat i rendiment. Entendre els conceptes bàsics dels registres, instruccions i flux de control és essencial per escriure codi d'assembly ARM eficient. Temes avançats com les subrutines i l'assembly inline en C milloren encara més la capacitat d'optimitzar i estructurar el codi de manera efectiva. Amb pràctica, els programadors poden aprofitar l'ARM assembly per desenvolupar aplicacions d'alt rendiment per a una àmplia gamma de dispositius.
