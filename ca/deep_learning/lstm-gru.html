Introducció

En aquesta secció, explorarem dos tipus populars de xarxes neuronals recurrents (RNN) utilitzades en Deep Learning: Long Short-Term Memory (LSTM) i Gated Recurrent Unit (GRU). Aquestes arquitectures són especialment útils per manejar dades seqüencials i resoldre problemes on la memòria a llarg termini és crucial.

LSTM (Long Short-Term Memory)

Conceptes Clau

  • Cèl·lules de Memòria: Les LSTM utilitzen cèl·lules de memòria per mantenir informació al llarg de seqüències llargues.
  • Portes (Gates): Les LSTM tenen tres tipus de portes que controlen el flux d'informació:
    • Porta d'Entrada (Input Gate): Decideix quina informació s'actualitzarà a la cèl·lula de memòria.
    • Porta d'Oblit (Forget Gate): Decideix quina informació es descartarà de la cèl·lula de memòria.
    • Porta de Sortida (Output Gate): Decideix quina part de la cèl·lula de memòria s'utilitzarà per a la sortida.

Estructura d'una LSTM

| Component | Funció | |------------------|-------------------------------------------------------------------------| | Input Gate | Controla la quantitat d'informació que entra a la cèl·lula de memòria. | | Forget Gate | Controla la quantitat d'informació que s'oblida de la cèl·lula de memòria.| | Output Gate | Controla la quantitat d'informació que surt de la cèl·lula de memòria. | | Cell State | Emmagatzema informació al llarg de la seqüència. | | Hidden State | Sortida de la cèl·lula que es passa a la següent cèl·lula en la seqüència. |

Exemple de Codi

import tensorflow as tf
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import LSTM, Dense

# Crear un model seqüencial
model = Sequential()

# Afegir una capa LSTM
model.add(LSTM(50, input_shape=(100, 1)))

# Afegir una capa densa per a la sortida
model.add(Dense(1))

# Compilar el model
model.compile(optimizer='adam', loss='mean_squared_error')

# Resum del model
model.summary()

GRU (Gated Recurrent Unit)

Conceptes Clau

  • Portes (Gates): Les GRU simplifiquen l'arquitectura de les LSTM combinant les portes d'entrada i oblit en una sola porta d'actualització.
    • Porta d'Actualització (Update Gate): Controla la quantitat d'informació que s'actualitza a la cèl·lula de memòria.
    • Porta de Reinici (Reset Gate): Controla la quantitat d'informació que s'oblida de la cèl·lula de memòria.

Estructura d'una GRU

| Component | Funció | |------------------|-------------------------------------------------------------------------| | Update Gate | Controla la quantitat d'informació que s'actualitza a la cèl·lula de memòria.| | Reset Gate | Controla la quantitat d'informació que s'oblida de la cèl·lula de memòria.| | Hidden State | Sortida de la cèl·lula que es passa a la següent cèl·lula en la seqüència. |

Exemple de Codi

import tensorflow as tf
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import GRU, Dense

# Crear un model seqüencial
model = Sequential()

# Afegir una capa GRU
model.add(GRU(50, input_shape=(100, 1)))

# Afegir una capa densa per a la sortida
model.add(Dense(1))

# Compilar el model
model.compile(optimizer='adam', loss='mean_squared_error')

# Resum del model
model.summary()

Comparació entre LSTM i GRU

| Característica | LSTM | GRU | |-----------------------------|-----------------------------------------|----------------------------------------| | Complexitat | Més complexa degut a tres portes | Menys complexa amb dues portes | | Rendiment en seqüències llargues | Millor en seqüències molt llargues | Rendiment similar amb menor complexitat| | Temps d'entrenament | Més lent degut a la complexitat | Més ràpid degut a menor complexitat |

Conclusió

LSTM i GRU són dues arquitectures poderoses per manejar dades seqüencials en Deep Learning. Mentre que les LSTM són més complexes i poden manejar seqüències més llargues de manera més efectiva, les GRU ofereixen una alternativa més simple i ràpida amb un rendiment similar en molts casos. L'elecció entre LSTM i GRU depèn del problema específic i de les restriccions de temps i recursos computacionals.
