Gestió d'Errors en Objective-C
Multifil i Concurrència ►

La gestió d'errors és un aspecte crucial de la programació que assegura que les teves aplicacions puguin gestionar situacions inesperades de manera elegant. En Objective-C, la gestió d'errors es gestiona típicament utilitzant la classe NSError i excepcions. Aquest tema et guiarà des dels conceptes bàsics fins a tècniques avançades en la gestió d'errors.

Introducció a la Gestió d'Errors

La gestió d'errors permet que el teu programa respongui a condicions inesperades (errors) de manera controlada. En Objective-C, hi ha dos mecanismes principals per a la gestió d'errors:

  • NSError: Utilitzat per a errors recuperables.
  • Excepcions: Utilitzat per a errors no recuperables.

NSError

Què és NSError?

NSError és una classe utilitzada per encapsular informació sobre una condició d'error en Objective-C. Proporciona detalls com el domini de l'error, el codi d'error i el diccionari d'informació de l'usuari.

Crear i Utilitzar NSError

Per crear un objecte NSError, típicament utilitzes el mètode errorWithDomain:code:userInfo:.

NSError *error = [NSError errorWithDomain:@"com.example.MyApp"
                                     code:1001
                                 userInfo:@{NSLocalizedDescriptionKey: @"S'ha produït un error"}];

Gestionar NSError

Quan un mètode pot produir un error, normalment pren un paràmetre NSError ** per retornar informació sobre l'error.

- (BOOL)performTaskWithError:(NSError **)error {
    // Simular una condició d'error
    if (/* alguna condició d'error */) {
        if (error) {
            *error = [NSError errorWithDomain:@"com.example.MyApp"
                                         code:1001
                                     userInfo:@{NSLocalizedDescriptionKey: @"S'ha produït un error"}];
        }
        return NO;
    }
    return YES;
}

Exemple: Utilitzar NSError en un Mètode

NSError *error = nil;
BOOL success = [self performTaskWithError:&error];
if (!success) {
    NSLog(@"Error: %@", error.localizedDescription);
}

Excepcions

Què són les Excepcions?

Les excepcions s'utilitzen per gestionar errors crítics que són típicament no recuperables. En Objective-C, les excepcions estan representades per la classe NSException.

Llançar i Capturar Excepcions

Per llançar una excepció, utilitzes la directiva @throw. Per capturar excepcions, utilitzes els blocs @try, @catch i @finally.

@try {
    // Codi que podria llançar una excepció
    [self riskyMethod];
}
@catch (NSException *exception) {
    // Gestionar l'excepció
    NSLog(@"Excepció: %@", exception.reason);
}
@finally {
    // Codi que s'executa tant si es llança una excepció com si no
    NSLog(@"Bloc finally executat");
}

Exemple: Llançar i Capturar Excepcions

- (void)riskyMethod {
    @throw [NSException exceptionWithName:@"MyException"
                                   reason:@"Alguna cosa ha anat malament"
                                 userInfo:nil];
}

- (void)performRiskyOperation {
    @try {
        [self riskyMethod];
    }
    @catch (NSException *exception) {
        NSLog(@"S'ha capturat una excepció: %@", exception.reason);
    }
    @finally {
        NSLog(@"Codi de neteja executat");
    }
}

Millors Pràctiques

  • Utilitza NSError per a errors recuperables: Reserva les excepcions per a condicions realment excepcionals.
  • Comprova si és nil: Sempre comprova si el punter NSError és no-nil abans d'assignar un error.
  • Proporciona missatges d'error significatius: Utilitza el diccionari userInfo per proporcionar informació detallada sobre l'error.
  • Neteja recursos: Utilitza el bloc @finally per netejar recursos, assegurant que el codi de neteja s'executi independentment de si es llança una excepció.

Resum

La gestió d'errors en Objective-C implica utilitzar NSError per a errors recuperables i excepcions per a errors no recuperables. Seguint les millors pràctiques i utilitzant aquests mecanismes adequadament, pots crear aplicacions robustes que gestionin els errors de manera elegant.

  • NSError: Utilitzat per a errors recuperables, proporciona informació detallada sobre l'error.
  • Excepcions: Utilitzat per a errors no recuperables, gestionat amb blocs @try, @catch i @finally.

Entendre i implementar estratègies efectives de gestió d'errors farà que les teves aplicacions Objective-C siguin més fiables i fàcils d'utilitzar.
