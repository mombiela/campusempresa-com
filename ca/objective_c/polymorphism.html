Polimorfisme en Objective-C
Categories i Extensions ►

El polimorfisme és un concepte fonamental en la programació orientada a objectes que permet que els objectes siguin tractats com a instàncies de la seva classe pare en lloc de la seva classe real. Això permet que una única interfície representi diferents formes subjacents (tipus de dades). En Objective-C, el polimorfisme s'aconsegueix mitjançant la sobrescripció de mètodes i el tipatge dinàmic.

Conceptes Clau del Polimorfisme

  • Sobrescripció de Mètodes: Les subclasses poden proporcionar una implementació específica d'un mètode que ja està definit en la seva superclasse.
  • Tipatge Dinàmic: Objective-C permet enviar missatges a objectes sense conèixer el seu tipus exacte en temps de compilació.
  • Tipus id: Un tipus d'objecte genèric que pot contenir qualsevol objecte.

Sobrescripció de Mètodes

La sobrescripció de mètodes permet que una subclasse proporcioni una implementació específica d'un mètode que ja està definit en la seva superclasse. Això és un aspecte fonamental del polimorfisme.

Exemple

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Animal *myAnimal = [[Animal alloc] init];
        [myAnimal speak]; // Output: Animal speaks
        
        Dog *myDog = [[Dog alloc] init];
        [myDog speak]; // Output: Dog barks
    }
    return 0;
}

En aquest exemple:

  • La classe Animal té un mètode speak.
  • La classe Dog sobrescriu el mètode speak per proporcionar la seva pròpia implementació.
  • Quan es crida speak en un objecte Animal, utilitza la implementació de Animal.
  • Quan es crida speak en un objecte Dog, utilitza la implementació de Dog.

Tipatge Dinàmic

Objective-C suporta el tipatge dinàmic, que permet enviar missatges a objectes sense conèixer el seu tipus exacte en temps de compilació. Això és particularment útil per al polimorfisme.

Exemple

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

@interface Cat : Animal
@end

@implementation Cat
- (void)speak {
    NSLog(@"Cat meows");
}
@end

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        NSArray *animals = @[[[Dog alloc] init], [[Cat alloc] init], [[Animal alloc] init]];
        
        for (Animal *animal in animals) {
            [animal speak];
        }
    }
    return 0;
}

En aquest exemple:

  • Es crea una matriu d'objectes Animal, que conté instàncies de Dog, Cat i Animal.
  • Es crida el mètode speak en cada objecte de la matriu.
  • Es crida la implementació correcta de speak per a cada objecte, demostrant el polimorfisme.

Tipus id

El tipus id és un tipus d'objecte genèric en Objective-C que pot contenir qualsevol objecte. Això és útil per al polimorfisme perquè permet escriure codi que pot treballar amb objectes de qualsevol tipus.

Exemple

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

@interface Cat : Animal
@end

@implementation Cat
- (void)speak {
    NSLog(@"Cat meows");
}
@end

void makeAnimalSpeak(id animal) {
    [animal speak];
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        id myDog = [[Dog alloc] init];
        id myCat = [[Cat alloc] init];
        
        makeAnimalSpeak(myDog); // Output: Dog barks
        makeAnimalSpeak(myCat); // Output: Cat meows
    }
    return 0;
}

En aquest exemple:

  • La funció makeAnimalSpeak pren un paràmetre de tipus id, permetent-li acceptar qualsevol objecte.
  • Es crida la implementació correcta de speak per a cada objecte passat a makeAnimalSpeak.

Conclusió

El polimorfisme en Objective-C permet un codi flexible i reutilitzable permetent que els objectes siguin tractats com a instàncies de la seva classe pare. Mitjançant la sobrescripció de mètodes, el tipatge dinàmic i l'ús del tipus id, Objective-C proporciona eines poderoses per implementar el polimorfisme. Entendre i utilitzar aquests conceptes t'ajudarà a escriure codi més eficient i mantenible.
