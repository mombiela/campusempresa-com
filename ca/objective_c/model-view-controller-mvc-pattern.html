Patró Model-Vista-Controlador (MVC)
Gestió d'Esdeveniments ►

El patró Model-Vista-Controlador (MVC) és un patró de disseny fonamental en l'enginyeria de programari, especialment en el desenvolupament d'interfícies d'usuari. Divideix una aplicació en tres components interconnectats: Model, Vista i Controlador. Aquesta separació ajuda a gestionar la complexitat, millorar la reutilització del codi i facilitar el desenvolupament en paral·lel.

Introducció al MVC

  • Model: Representa les dades i la lògica de negoci de l'aplicació. Gestiona directament les dades, la lògica i les regles de l'aplicació.
  • Vista: Representa els components de la interfície d'usuari i mostra les dades del Model a l'usuari. Envia les ordres de l'usuari al Controlador.
  • Controlador: Actua com a intermediari entre el Model i la Vista. Escolta les entrades de la Vista, les processa (sovint actualitzant el Model) i retorna la sortida a la Vista.

Beneficis d'Utilitzar MVC

  • Separació de Responsabilitats: Cada component té una responsabilitat distintiva, fent que la base de codi sigui més fàcil de gestionar i entendre.
  • Reutilització: Els components es poden reutilitzar en diferents parts de l'aplicació o fins i tot en diferents projectes.
  • Desenvolupament en Paral·lel: Diferents desenvolupadors poden treballar en el Model, la Vista i el Controlador simultàniament sense interferir entre ells.

Implementació de MVC en Objective-C

Model

El Model és responsable de les dades i la lògica de negoci. En Objective-C, això es representa típicament per classes que gestionen les dades.

// Person.h
#import <Foundation/Foundation.h>

@interface Person : NSObject

@property (nonatomic, strong) NSString *name;
@property (nonatomic, assign) NSInteger age;

- (instancetype)initWithName:(NSString *)name age:(NSInteger)age;

@end

// Person.m
#import "Person.h"

@implementation Person

- (instancetype)initWithName:(NSString *)name age:(NSInteger)age {
    self = [super init];
    if (self) {
        _name = name;
        _age = age;
    }
    return self;
}

@end

Vista

La Vista és responsable de la capa de presentació. Mostra les dades del Model i envia les entrades de l'usuari al Controlador.

// PersonView.h
#import <UIKit/UIKit.h>

@interface PersonView : UIView

@property (nonatomic, strong) UILabel *nameLabel;
@property (nonatomic, strong) UILabel *ageLabel;

- (void)updateWithPerson:(Person *)person;

@end

// PersonView.m
#import "PersonView.h"
#import "Person.h"

@implementation PersonView

- (instancetype)initWithFrame:(CGRect)frame {
    self = [super initWithFrame:frame];
    if (self) {
        _nameLabel = [[UILabel alloc] initWithFrame:CGRectMake(10, 10, 300, 40)];
        _ageLabel = [[UILabel alloc] initWithFrame:CGRectMake(10, 60, 300, 40)];
        [self addSubview:_nameLabel];
        [self addSubview:_ageLabel];
    }
    return self;
}

- (void)updateWithPerson:(Person *)person {
    self.nameLabel.text = person.name;
    self.ageLabel.text = [NSString stringWithFormat:@"%ld", (long)person.age];
}

@end

Controlador

El Controlador gestiona les entrades de l'usuari, actualitza el Model i refresca la Vista.

// PersonViewController.h
#import <UIKit/UIKit.h>
#import "Person.h"
#import "PersonView.h"

@interface PersonViewController : UIViewController

@property (nonatomic, strong) Person *person;
@property (nonatomic, strong) PersonView *personView;

@end

// PersonViewController.m
#import "PersonViewController.h"

@implementation PersonViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    
    self.person = [[Person alloc] initWithName:@"John Doe" age:30];
    self.personView = [[PersonView alloc] initWithFrame:self.view.bounds];
    [self.view addSubview:self.personView];
    
    [self.personView updateWithPerson:self.person];
}

@end

Resum

El patró MVC és un patró de disseny poderós que ajuda a organitzar el codi d'una manera neta i mantenible. Separant l'aplicació en components Model, Vista i Controlador, es pot gestionar la complexitat, millorar la reutilització del codi i facilitar el desenvolupament en paral·lel. En Objective-C, implementar MVC implica crear classes per a cada component i assegurar-se que interactuen correctament per proporcionar una experiència d'usuari cohesiva.

Entenent i aplicant el patró MVC, es poden construir aplicacions robustes i escalables que són més fàcils de mantenir i ampliar.
