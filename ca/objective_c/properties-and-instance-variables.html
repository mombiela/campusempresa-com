Introducció

En Objective-C, les propietats i les variables d'instància són conceptes fonamentals que et permeten emmagatzemar i gestionar dades dins de les teves classes. Entendre com utilitzar-les de manera efectiva és crucial per escriure codi robust i mantenible.

Variables d'Instància

Les variables d'instància (sovint referides com a ivars) són variables que es declaren a la interfície o implementació d'una classe. Contenen dades específiques d'una instància de la classe.

Conceptes Clau

  • Declaració: Les variables d'instància es declaren típicament al bloc @interface o @implementation.
  • Àmbit: Són privades per a la instància de la classe i no són directament accessibles des de fora de la classe.
  • Accés: S'accedeix a elles dins dels mètodes de la classe utilitzant la paraula clau self.

Exemple

@interface MyClass : NSObject {
    int _myInstanceVariable;
}
@end

@implementation MyClass
- (void)printInstanceVariable {
    NSLog(@"Instance Variable: %d", _myInstanceVariable);
}
@end

En aquest exemple:

  • _myInstanceVariable és una variable d'instància.
  • Es declara dins del bloc @interface.
  • S'accedeix a ella dins del mètode printInstanceVariable.

Propietats

Les propietats proporcionen una manera de definir l'encapsulació de dades i patrons d'accés per a les variables d'instància. Ofereixen un nivell més alt d'abstracció i són preferides sobre l'accés directe a les variables d'instància.

Conceptes Clau

  • Declaració: Les propietats es declaren utilitzant la paraula clau @property.
  • Atributs: Les propietats poden tenir atributs com nonatomic, strong, weak, copy, readonly, i readwrite.
  • Síntesi: La paraula clau @synthesize s'utilitza per generar automàticament els mètodes getter i setter per a una propietat.
  • Accés: S'accedeix a les propietats utilitzant la notació de punt.

Exemple

@interface MyClass : NSObject
@property (nonatomic, strong) NSString *myProperty;
@end

@implementation MyClass
@synthesize myProperty = _myProperty;

- (void)printProperty {
    NSLog(@"Property: %@", self.myProperty);
}
@end

En aquest exemple:

  • myProperty és una propietat de tipus NSString.
  • La paraula clau @property s'utilitza per declarar-la.
  • La paraula clau @synthesize s'utilitza per generar els mètodes getter i setter.
  • S'accedeix a la propietat utilitzant self.myProperty.

Atributs de Propietat

| Atribut | Descripció | |------------|-------------| | nonatomic | Assegura que la propietat no és segura per a fils, millorant el rendiment. | | strong | Reté una referència forta a l'objecte, evitant que sigui desalocat. | | weak | Manté una referència dèbil a l'objecte, permetent que sigui desalocat si no existeixen referències fortes. | | copy | Crea una còpia de l'objecte quan s'assigna. | | readonly | Només genera un mètode getter, fent la propietat de només lectura. | | readwrite | Genera tant mètodes getter com setter (comportament per defecte). |

Exemple amb Atributs

@interface MyClass : NSObject
@property (nonatomic, copy, readonly) NSString *readOnlyProperty;
@property (nonatomic, weak) id delegate;
@end

@implementation MyClass
@synthesize readOnlyProperty = _readOnlyProperty;
@synthesize delegate = _delegate;

- (void)printProperties {
    NSLog(@"Read-Only Property: %@", self.readOnlyProperty);
    NSLog(@"Delegate: %@", self.delegate);
}
@end

En aquest exemple:

  • readOnlyProperty és una propietat de només lectura amb l'atribut copy.
  • delegate és una referència dèbil per evitar cicles de retenció.

Conclusió

Entendre les propietats i les variables d'instància és essencial per gestionar dades dins de les teves classes Objective-C. Les variables d'instància proporcionen una manera d'emmagatzemar dades específiques d'una instància, mentre que les propietats ofereixen un enfocament més flexible i encapsulat per a la gestió de dades. Utilitzant propietats amb els atributs adequats, pots controlar com s'accedeixen i es modifiquen les dades, portant a un codi més robust i mantenible.
