Introducció

La depuració i les proves són habilitats crucials per a qualsevol programador d'Objective-C. Aquesta secció cobrirà els conceptes bàsics de la depuració i les proves, incloent eines, tècniques i millors pràctiques. Al final d'aquesta secció, seràs capaç d'identificar i corregir errors en el teu codi i escriure proves per assegurar-te que el teu codi funciona com s'espera.

Depuració

Conceptes Clau

  • Punts de trencament: Punts en el teu codi on l'execució es pausarà, permetent-te inspeccionar l'estat de la teva aplicació.
  • Step Over, Step Into, Step Out: Comandes per controlar el flux d'execució mentre es depura.
  • Variables de vigilància: Monitoritzar els valors de les variables mentre el teu codi s'executa.
  • Sortida de consola: Utilitzar NSLog per imprimir missatges a la consola amb finalitats de depuració.

Eines

  • Depurador de Xcode: L'eina principal per a la depuració d'aplicacions Objective-C.
  • LLDB: El depurador de línia de comandes utilitzat per Xcode.

Configuració de Punts de Trencament

Els punts de trencament et permeten pausar l'execució del teu codi en línies específiques per inspeccionar l'estat de la teva aplicació.

- (void)viewDidLoad {
    [super viewDidLoad];
    int a = 5;
    int b = 10;
    int sum = a + b; // Configura un punt de trencament aquí
    NSLog(@"Sum: %d", sum);
}

Ús de Comandes LLDB

Les comandes LLDB es poden utilitzar a la consola de Xcode per inspeccionar i manipular l'estat de la teva aplicació.

(lldb) breakpoint set --file ViewController.m --line 12
(lldb) run
(lldb) print sum

Exemple: Depurant una Aplicació Simple

Anem a depurar una aplicació simple que calcula la suma de dos números.

  1. Configura un punt de trencament a la línia on es calcula la suma.
  2. Executa l'aplicació a Xcode.
  3. Quan es toqui el punt de trencament, utilitza la consola LLDB per inspeccionar els valors de a, b i sum.

Proves

Conceptes Clau

  • Proves unitàries: Provar unitats individuals de codi per assegurar-se que funcionen com s'espera.
  • Proves d'integració: Provar com diferents unitats de codi treballen juntes.
  • Desenvolupament guiat per proves (TDD): Escriure proves abans d'escriure el codi real.

Eines

  • XCTest: El framework proporcionat per Apple per escriure i executar proves a Xcode.

Escriure Proves Unitàries

Les proves unitàries s'escriuen utilitzant el framework XCTest. Aquí tens un exemple d'una prova unitària simple:

#import <XCTest/XCTest.h>
#import "Calculator.h"

@interface CalculatorTests : XCTestCase
@end

@implementation CalculatorTests

- (void)testAddition {
    Calculator *calculator = [[Calculator alloc] init];
    int result = [calculator addNumber:5 toNumber:10];
    XCTAssertEqual(result, 15, @"Expected 5 + 10 to equal 15");
}

@end

Executar Proves a Xcode

  1. Obre el Navigator de Proves fent clic a la icona de diamant a la barra d'eines de Xcode.
  2. Selecciona la prova que vols executar.
  3. Fes clic al botó de reproducció per executar la prova.

Exemple: Escriure i Executar una Prova Unitària

Anem a escriure una prova unitària per a una classe de calculadora simple.

  1. Crea una nova classe anomenada Calculator amb un mètode addNumber:toNumber:.
  2. Escriu una prova unitària per al mètode addNumber:toNumber:.
  3. Executa la prova a Xcode per assegurar-te que passa.

Conclusió

La depuració i les proves són habilitats essencials per a qualsevol programador d'Objective-C. Utilitzant punts de trencament, comandes LLDB i proves unitàries, pots identificar i corregir errors en el teu codi i assegurar-te que el teu codi funciona com s'espera. Practica aquestes tècniques regularment per convertir-te en un expert en la depuració i les proves de les teves aplicacions Objective-C.
