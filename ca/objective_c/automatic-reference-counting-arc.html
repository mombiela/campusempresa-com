Comptatge Automàtic de Referències (ARC)
Retenció i Alliberament Manual ►

El Comptatge Automàtic de Referències (ARC) és una característica de gestió de memòria en Objective-C que ajuda els desenvolupadors a gestionar la vida útil dels objectes. ARC gestiona automàticament el comptatge de referències dels objectes, cosa que ajuda a prevenir fuites de memòria i assegura que els objectes es desallotgin quan ja no es necessiten.

Introducció a ARC

  • Què és ARC?

    • ARC és una característica en temps de compilació que insereix automàticament les crides retain i release per gestionar la memòria dels objectes d'Objective-C.
    • Simplifica la gestió de memòria reduint la necessitat de comptatge de referències manual.

  • Per què utilitzar ARC?

    • Redueix el risc de fuites de memòria i punters penjants.
    • Simplifica el codi eliminant la necessitat de crides explícites a retain, release i autorelease.
    • Millora la llegibilitat i mantenibilitat del codi.

Com Funciona ARC

  • Conceptes Bàsics del Comptatge de Referències

    • Cada objecte té un comptador de referències que fa un seguiment de quantes referències apunten a ell.
    • Quan el comptador de referències d'un objecte baixa a zero, l'objecte es desallotja.

  • ARC en Acció

    • ARC insereix automàticament les crides retain i release en els llocs adequats del codi.
    • Els desenvolupadors ja no necessiten gestionar manualment el comptador de referències.

Exemple: Ús Bàsic d'ARC

#import <Foundation/Foundation.h>

@interface Person : NSObject
@property (nonatomic, strong) NSString *name;
@end

@implementation Person
@end

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Person *person = [[Person alloc] init];
        person.name = @"John Doe";
        NSLog(@"Person's name: %@", person.name);
    }
    return 0;
}
  • Explicació:
    • El bloc @autoreleasepool assegura que els objectes temporals es desallotgin correctament.
    • L'atribut de propietat strong indica que la propietat reté l'objecte assignat.
    • ARC gestiona automàticament la memòria de l'objecte person.

Atributs de Propietat i ARC

  • Strong

    • Reté l'objecte i augmenta el seu comptador de referències.
    • S'utilitza comunament per a propietats que necessiten mantenir una referència forta a un objecte.

  • Weak

    • No reté l'objecte i no augmenta el seu comptador de referències.
    • Automàticament estableix la propietat a nil quan l'objecte referenciat es desallotja.
    • Útil per evitar cicles de retenció, especialment en relacions de delegat.

  • Copy

    • Crea una còpia de l'objecte assignat.
    • Assegura que la propietat tingui la seva pròpia còpia, independent de l'objecte original.

  • Assign

    • Assigna directament el valor sense retenir-lo.
    • Normalment s'utilitza per a tipus de dades primitives (per exemple, int, float).

Exemple: Atributs de Propietat

@interface Car : NSObject
@property (nonatomic, strong) NSString *model;
@property (nonatomic, weak) id delegate;
@property (nonatomic, copy) NSString *licensePlate;
@property (nonatomic, assign) int year;
@end

@implementation Car
@end
  • Explicació:
    • strong per a model assegura que l'objecte cotxe reté la cadena del model.
    • weak per a delegate evita cicles de retenció.
    • copy per a licensePlate assegura que l'objecte cotxe tingui la seva pròpia còpia de la cadena de la matrícula.
    • assign per a year s'utilitza per al tipus de dada primitiva int.

Cicles de Retenció i Referències Weak

  • Cicles de Retenció

    • Es produeixen quan dos o més objectes mantenen referències fortes entre ells, impedint que es desallotgin.
    • Poden conduir a fuites de memòria.

  • Evitant Cicles de Retenció

    • Utilitza referències weak per trencar cicles de retenció.
    • S'utilitza comunament en patrons de delegat i relacions pare-fill.

Exemple: Evitant Cicles de Retenció

@interface Parent : NSObject
@property (nonatomic, strong) Child *child;
@end

@interface Child : NSObject
@property (nonatomic, weak) Parent *parent;
@end

@implementation Parent
@end

@implementation Child
@end
  • Explicació:
    • L'objecte Parent manté una referència forta a l'objecte Child.
    • L'objecte Child manté una referència dèbil a l'objecte Parent, evitant un cicle de retenció.

Característiques Avançades d'ARC

  • Referències Weak que es Nul·len

    • Automàticament estableix les referències dèbils a nil quan l'objecte referenciat es desallotja.
    • Prevé punters penjants.

  • Pools d'Autorelease

    • S'utilitzen per gestionar la vida útil dels objectes temporals.
    • ARC insereix automàticament pools d'autorelease en llocs adequats, però els desenvolupadors també poden crear els seus propis.

Exemple: Pool d'Autorelease Personalitzat

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        // Codi que crea objectes temporals
        NSString *tempString = [NSString stringWithFormat:@"Temporary String"];
        NSLog(@"%@", tempString);
    }
    // tempString es desallotja aquí
    return 0;
}
  • Explicació:
    • El bloc @autoreleasepool assegura que els objectes temporals com tempString es desallotgin quan el bloc finalitza.

Conclusió

El Comptatge Automàtic de Referències (ARC) és una característica poderosa en Objective-C que simplifica la gestió de memòria gestionant automàticament el comptatge de referències. Entenent com funciona ARC i utilitzant els atributs de propietat de manera efectiva, els desenvolupadors poden escriure codi més segur i més fàcil de mantenir. Evitar cicles de retenció i utilitzar referències dèbils adequadament són crucials per prevenir fuites de memòria. Amb ARC, els desenvolupadors poden centrar-se més en construir funcionalitats i menys en gestionar la memòria manualment.
