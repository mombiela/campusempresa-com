Introducció

En aquesta secció, tractarem com implementar diverses funcionalitats en Objective-C. Això inclourà la creació de classes, mètodes, propietats, i la comprensió de protocols i categories. Al final d'aquesta secció, seràs capaç de crear i gestionar funcionalitats complexes en els teus projectes d'Objective-C.

Creació de Classes i Objectes

Les classes són el plànol per als objectes en Objective-C. Aquí tens com pots crear una classe i instanciar un objecte.

Definició d'una Classe

Per definir una classe, necessites crear un fitxer de capçalera (.h) i un fitxer d'implementació (.m).

Fitxer de Capçalera: Person.h

#import <Foundation/Foundation.h>

@interface Person : NSObject

@property (nonatomic, strong) NSString *name;
@property (nonatomic, assign) NSInteger age;

- (void)displayInfo;

@end

Fitxer d'Implementació: Person.m

#import "Person.h"

@implementation Person

- (void)displayInfo {
    NSLog(@"Name: %@, Age: %ld", self.name, (long)self.age);
}

@end

Instanciació d'un Objecte

Pots crear una instància de la classe Person i utilitzar les seves propietats i mètodes.

Person *person = [[Person alloc] init];
person.name = @"John Doe";
person.age = 30;
[person displayInfo]; // Sortida: Name: John Doe, Age: 30

Mètodes

Els mètodes en Objective-C són funcions que pertanyen a una classe. Poden ser mètodes d'instància o mètodes de classe.

Mètodes d'Instància

Els mètodes d'instància operen sobre una instància de la classe.

- (void)displayInfo {
    NSLog(@"Name: %@, Age: %ld", self.name, (long)self.age);
}

Mètodes de Classe

Els mètodes de classe es criden sobre la classe mateixa, no sobre una instància.

+ (void)classMethodExample {
    NSLog(@"This is a class method");
}

Propietats

Les propietats en Objective-C proporcionen una manera de definir els atributs d'una classe. Generen automàticament els mètodes getter i setter.

Atributs de Propietat

  • nonatomic: Accés no atòmic.
  • strong: Referència forta, utilitzada per a objectes.
  • weak: Referència dèbil, utilitzada per evitar cicles de retenció.
  • assign: Utilitzat per a tipus de dades primitives.
@property (nonatomic, strong) NSString *name;
@property (nonatomic, assign) NSInteger age;

Protocols

Els protocols en Objective-C són similars a les interfícies en altres llenguatges. Defineixen una llista de mètodes que una classe pot implementar.

Definició d'un Protocol

@protocol Drivable <NSObject>

- (void)startEngine;
- (void)stopEngine;

@end

Implementació d'un Protocol

@interface Car : NSObject <Drivable>

@end

@implementation Car

- (void)startEngine {
    NSLog(@"Engine started");
}

- (void)stopEngine {
    NSLog(@"Engine stopped");
}

@end

Categories

Les categories et permeten afegir mètodes a classes existents sense modificar la classe original.

Definició d'una Categoria

@interface NSString (ReverseString)

- (NSString *)reverseString;

@end

@implementation NSString (ReverseString)

- (NSString *)reverseString {
    NSUInteger len = [self length];
    NSMutableString *reversedString = [NSMutableString stringWithCapacity:len];
    
    while (len > 0) {
        [reversedString appendString:[NSString stringWithFormat:@"%C", [self characterAtIndex:--len]]];
    }
    
    return reversedString;
}

@end

Ús d'una Categoria

NSString *originalString = @"Objective-C";
NSString *reversedString = [originalString reverseString];
NSLog(@"Reversed String: %@", reversedString); // Sortida: C-evitcejbO

Conclusió

En aquesta secció, hem cobert els conceptes bàsics de la implementació de funcionalitats en Objective-C, incloent la creació de classes, mètodes, propietats, protocols i categories. Aquests conceptes formen la base per construir aplicacions robustes i mantenibles en Objective-C. Practica aquests conceptes creant les teves pròpies classes i mètodes, i intenta implementar protocols i categories per estendre la funcionalitat de les classes existents.
