Introducció

En Objective-C, les categories i extensions són característiques poderoses que et permeten afegir mètodes a classes existents sense modificar el seu codi font. Això pot ser particularment útil per organitzar el codi, ampliar la funcionalitat i mantenir bases de codi netes i modulars.

Categories

Les categories et permeten afegir mètodes a classes existents. Sovint s'utilitzen per:

  • Organitzar mètodes en grups lògics.
  • Afegir mètodes a classes de les quals no tens el codi font.
  • Separar els detalls d'implementació de la interfície principal de la classe.

Crear una Categoria

Per crear una categoria, cal definir-la en un fitxer de capçalera i implementar-la en un fitxer d'implementació corresponent.

Fitxer de Capçalera (.h)

// NSString+ReversedString.h
#import <Foundation/Foundation.h>

@interface NSString (ReversedString)

- (NSString *)reversedString;

@end

Fitxer d'Implementació (.m)

// NSString+ReversedString.m
#import "NSString+ReversedString.h"

@implementation NSString (ReversedString)

- (NSString *)reversedString {
    NSUInteger length = [self length];
    NSMutableString *reversedString = [NSMutableString stringWithCapacity:length];
    
    for (NSInteger i = length - 1; i >= 0; i--) {
        [reversedString appendFormat:@"%C", [self characterAtIndex:i]];
    }
    
    return reversedString;
}

@end

Utilitzar una Categoria

Per utilitzar els mètodes definits en una categoria, simplement importa el fitxer de capçalera de la categoria i crida els mètodes en instàncies de la classe.

#import "NSString+ReversedString.h"

NSString *originalString = @"Objective-C";
NSString *reversedString = [originalString reversedString];
NSLog(@"Reversed String: %@", reversedString); // Sortida: "C-evitcejbO"

Punts Clau

  • Les categories no poden afegir variables d'instància.
  • Els mètodes afegits per categories estan disponibles per a totes les instàncies de la classe.
  • Si un mètode de categoria té el mateix nom que un mètode en la classe original, el mètode de la categoria sobreescriurà el mètode original.

Extensions

Les extensions són similars a les categories però s'utilitzen per afegir mètodes i propietats privades a una classe. Es declaren típicament en el fitxer d'implementació de la classe que amplien.

Crear una Extensió

Les extensions es declaren en el fitxer d'implementació de la classe que amplien i no requereixen un fitxer de capçalera separat.

Fitxer d'Implementació (.m)

// MyClass.m
#import "MyClass.h"

@interface MyClass ()

@property (nonatomic, strong) NSString *privateProperty;

- (void)privateMethod;

@end

@implementation MyClass

- (void)publicMethod {
    NSLog(@"Public method called");
    [self privateMethod];
}

- (void)privateMethod {
    NSLog(@"Private method called");
}

@end

Utilitzar una Extensió

Les extensions s'utilitzen internament dins de la classe que amplien i no són visibles per a altres classes.

MyClass *myObject = [[MyClass alloc] init];
[myObject publicMethod]; // Sortida: "Public method called" seguit de "Private method called"
// [myObject privateMethod]; // Això causarà un error de compilació

Punts Clau

  • Les extensions poden afegir tant mètodes com propietats.
  • Els mètodes i propietats afegits per extensions són privats per a la classe.
  • Les extensions s'utilitzen típicament per amagar detalls d'implementació.

Resum

Les categories i extensions són eines essencials en Objective-C per ampliar i organitzar el teu codi. Les categories et permeten afegir mètodes a classes existents, facilitant la gestió i ampliació de la funcionalitat sense alterar la classe original. Les extensions, d'altra banda, proporcionen una manera d'afegir mètodes i propietats privades, ajudant a encapsular els detalls d'implementació i mantenir una interfície neta.

Entenent i utilitzant categories i extensions, pots escriure codi Objective-C més modular, mantenible i organitzat.
