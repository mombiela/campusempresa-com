Herència en Objective-C
Polimorfisme ►

L'herència és un concepte fonamental en la programació orientada a objectes que permet a una classe heretar propietats i mètodes d'una altra classe. Això ajuda en la reutilització del codi i en l'establiment d'una relació jeràrquica entre classes.

Què és l'herència?

  • Definició: L'herència permet que una nova classe (subclasse) hereti les propietats i mètodes d'una classe existent (superclasse).
  • Propòsit: Promou la reutilització del codi i estableix una jerarquia natural entre classes.

Conceptes bàsics

  • Superclasse: La classe les propietats i mètodes de la qual són heretats.
  • Subclasse: La classe que hereta propietats i mètodes de la superclasse.
  • @interface i @implementation: Paraules clau utilitzades per definir classes en Objective-C.

Creant una Superclasse

Comencem creant una superclasse simple anomenada Person.

// Person.h
#import <Foundation/Foundation.h>

@interface Person : NSObject

@property NSString *name;
@property NSInteger age;

- (void)displayInfo;

@end

// Person.m
#import "Person.h"

@implementation Person

- (void)displayInfo {
    NSLog(@"Name: %@, Age: %ld", self.name, (long)self.age);
}

@end
  • Explicació:
    • @interface Person : NSObject: Declara una nova classe Person que hereta de NSObject.
    • @property: Declara les propietats name i age.
    • - (void)displayInfo;: Declara un mètode per mostrar la informació de la persona.
    • @implementation Person: Implementa la classe Person.
    • NSLog: Mostra el nom i l'edat a la consola.

Creant una Subclasse

Ara, creem una subclasse anomenada Student que hereta de Person.

// Student.h
#import "Person.h"

@interface Student : Person

@property NSString *schoolName;

- (void)displayStudentInfo;

@end

// Student.m
#import "Student.h"

@implementation Student

- (void)displayStudentInfo {
    [self displayInfo];
    NSLog(@"School: %@", self.schoolName);
}

@end
  • Explicació:
    • @interface Student : Person: Declara una nova classe Student que hereta de Person.
    • @property: Declara una nova propietat schoolName.
    • - (void)displayStudentInfo;: Declara un mètode per mostrar la informació de l'estudiant.
    • @implementation Student: Implementa la classe Student.
    • [self displayInfo]: Crida el mètode displayInfo de la superclasse Person.

Utilitzant la Subclasse

Vegem com utilitzar la subclasse Student en un programa.

#import <Foundation/Foundation.h>
#import "Student.h"

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Student *student = [[Student alloc] init];
        student.name = @"John Doe";
        student.age = 20;
        student.schoolName = @"XYZ University";
        
        [student displayStudentInfo];
    }
    return 0;
}
  • Explicació:
    • Student *student = [[Student alloc] init];: Crea una instància de la classe Student.
    • student.name = @"John Doe";: Assigna la propietat name heretada de Person.
    • student.age = 20;: Assigna la propietat age heretada de Person.
    • student.schoolName = @"XYZ University";: Assigna la propietat schoolName.
    • [student displayStudentInfo];: Crida el mètode displayStudentInfo per mostrar tota la informació.

Sobreescrivint Mètodes

Una subclasse pot sobreescriure mètodes de la seva superclasse per proporcionar implementacions específiques.

// Student.m
#import "Student.h"

@implementation Student

- (void)displayInfo {
    NSLog(@"Student Name: %@, Age: %ld", self.name, (long)self.age);
}

- (void)displayStudentInfo {
    [self displayInfo];
    NSLog(@"School: %@", self.schoolName);
}

@end
  • Explicació:
    • - (void)displayInfo: Sobreescriu el mètode displayInfo de Person per proporcionar una implementació específica per a Student.

Resum

  • L'herència permet que una classe hereti propietats i mètodes d'una altra classe.
  • La superclasse proporciona les propietats i mètodes base.
  • La subclasse pot afegir noves propietats i mètodes o sobreescriure els existents.
  • Això promou la reutilització del codi i ajuda a crear una estructura jeràrquica.

Entenent i utilitzant l'herència, pots crear codi més organitzat i mantenible en Objective-C.
