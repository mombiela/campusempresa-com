Introducció

Les proves i la depuració són habilitats crítiques per a qualsevol programador. Aquesta secció et guiarà a través dels conceptes bàsics de les proves i la depuració en Objective-C, des de nivells principiants fins a avançats. Al final d'aquesta secció, seràs capaç d'escriure proves per al teu codi i depurar-lo de manera efectiva.

Proves Unitàries amb XCTest

Què són les Proves Unitàries?

  • Les proves unitàries impliquen provar components individuals del teu codi per assegurar-te que funcionen com s'espera.
  • En Objective-C, el framework XCTest és comunament utilitzat per a les proves unitàries.

Configurant XCTest

  1. Crear un Objectiu de Prova:

    • A Xcode, ves a la configuració del teu projecte.
    • Fes clic al botó "+" sota la secció "Targets".
    • Selecciona "iOS Unit Testing Bundle" o "macOS Unit Testing Bundle" i fes clic a "Next".
    • Nomena el teu objectiu de prova i completa la configuració.

  2. Importar XCTest:

    • En els teus arxius de prova, importa el framework XCTest.
    @import XCTest;

Escriure la Teva Primera Prova

  • Crea una nova classe de cas de prova subclassejant XCTestCase.
  • Escriu mètodes de prova utilitzant la convenció de nomenclatura - (void)testExample.
#import <XCTest/XCTest.h>

@interface MyFirstTests : XCTestCase
@end

@implementation MyFirstTests

- (void)testAddition {
    int a = 5;
    int b = 10;
    int resultat = a + b;
    XCTAssertEqual(resultat, 15, @"El resultat de la suma hauria de ser 15");
}

@end

Executar Proves

  • Pots executar proves seleccionant l'objectiu de prova i prement Cmd+U a Xcode.
  • Els resultats de les proves es mostraran al navegador de proves de Xcode.

Tècniques Avançades de Prova

Mocking i Stubbing

  • Mocking: Crear objectes que simulen el comportament d'objectes reals.
  • Stubbing: Proporcionar respostes predefinides a crides de mètodes.

Exemple amb OCMock

#import <OCMock/OCMock.h>
#import <XCTest/XCTest.h>
#import "MyClass.h"

@interface MyClassTests : XCTestCase
@end

@implementation MyClassTests

- (void)testMethodWithMock {
    id mock = OCMClassMock([MyClass class]);
    OCMStub([mock someMethod]).andReturn(@"Resposta Mocked");

    XCTAssertEqualObjects([mock someMethod], @"Resposta Mocked", @"La resposta hauria de ser mocked");
}

@end

Proves de Rendiment

  • XCTest també et permet mesurar el rendiment del teu codi.
- (void)testPerformanceExample {
    [self measureBlock:^{
        // Codi per mesurar el temps
        for (int i = 0; i < 1000; i++) {
            // Alguna tasca intensiva en rendiment
        }
    }];
}

Depuració en Objective-C

Utilitzant el Depurador de Xcode

  • Punts de Trencament: Configura punts de trencament fent clic al número de línia a Xcode.
  • Pas a Pas: Utilitza els controls del depurador per avançar pas a pas pel teu codi.
  • Vista de Variables: Inspecciona variables i els seus valors al depurador.

Comandes LLDB

  • Imprimir Variables: po variableName
  • Continuar Execució: continue
  • Pas a Pas: next
  • Pas a l'Interior: step

Tècniques Comunes de Depuració

  • Declaracions de Print: Utilitza NSLog per imprimir informació de depuració.
  • Afirmacions: Utilitza NSAssert per fer complir condicions durant el desenvolupament.
NSLog(@"Valor de la variable: %@", variable);
NSAssert(variable != nil, @"La variable no hauria de ser nil");

Conclusió

Les proves i la depuració són pràctiques essencials per escriure codi Objective-C robust i mantenible. Dominant les proves unitàries amb XCTest i utilitzant tècniques avançades com el mocking i les proves de rendiment, pots assegurar-te que el teu codi funciona com s'espera. A més, una depuració efectiva utilitzant les eines de Xcode i les comandes LLDB t'ajudarà a identificar i solucionar problemes ràpidament. Practica aquestes habilitats regularment per convertir-te en un expert en mantenir aplicacions Objective-C d'alta qualitat.
