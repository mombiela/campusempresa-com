Introducció als Protocols

Els protocols en Objective-C són una manera de definir un pla de mètodes que poden ser implementats per qualsevol classe. Són similars a les interfícies en altres llenguatges de programació.

  • Definició: Un protocol declara una llista de mètodes que una classe pot implementar.
  • Propòsit: Assegurar que una classe compleix amb un conjunt específic de mètodes, proporcionant una manera d'aconseguir polimorfisme.

Declaració d'un Protocol

Per declarar un protocol, utilitza la paraula clau @protocol seguida del nom del protocol i les declaracions dels mètodes.

@protocol MyProtocol <NSObject>
- (void)requiredMethod;
@optional
- (void)optionalMethod;
@end
  • Mètodes Requerits: Mètodes que han de ser implementats per qualsevol classe que adopti el protocol.
  • Mètodes Opcional: Mètodes que poden ser implementats opcionalment.

Adopció d'un Protocol

Una classe adopta un protocol llistant el protocol en la seva declaració d'interfície.

@interface MyClass : NSObject <MyProtocol>
@end

@implementation MyClass
- (void)requiredMethod {
    // Implementació del mètode requerit
}
@end

Introducció als Delegats

Els delegats són un patró de disseny utilitzat per gestionar esdeveniments o accions en Objective-C. Permeten que un objecte enviï missatges a un altre objecte quan es produeix un esdeveniment.

  • Propòsit: Desacoblar objectes i permetre un codi més flexible i reutilitzable.
  • Usos Comuns: Gestionar interaccions d'usuari, respondre a esdeveniments asíncrons, etc.

Creació d'un Protocol de Delegat

Per crear un delegat, defineix un protocol amb els mètodes que el delegat implementarà.

@protocol MyDelegate <NSObject>
- (void)eventOccurred;
@end

Configuració d'una Propietat de Delegat

La classe que enviarà missatges al delegat ha de tenir una propietat de delegat.

@interface MyClass : NSObject
@property (nonatomic, weak) id<MyDelegate> delegate;
- (void)triggerEvent;
@end

@implementation MyClass
- (void)triggerEvent {
    [self.delegate eventOccurred];
}
@end

Implementació del Delegat

Una altra classe implementa el protocol de delegat i es configura com a delegat.

@interface AnotherClass : NSObject <MyDelegate>
@end

@implementation AnotherClass
- (void)eventOccurred {
    NSLog(@"Event occurred!");
}
@end

// Ús
MyClass *myClass = [[MyClass alloc] init];
AnotherClass *anotherClass = [[AnotherClass alloc] init];
myClass.delegate = anotherClass;
[myClass triggerEvent]; // Registra "Event occurred!"

Temes Avançats

Combinació de Múltiples Protocols

Una classe pot adoptar múltiples protocols llistant-los en la declaració d'interfície.

@interface MyClass : NSObject <ProtocolOne, ProtocolTwo>
@end

Protocols Informals

Abans de la introducció dels protocols formals, s'utilitzaven protocols informals. Aquests són simplement categories en NSObject.

@interface NSObject (MyInformalProtocol)
- (void)informalMethod;
@end

Protocols amb Propietats

Els protocols també poden declarar propietats que les classes adoptants han d'implementar.

@protocol PropertyProtocol <NSObject>
@property (nonatomic, strong) NSString *name;
@end

@interface MyClass : NSObject <PropertyProtocol>
@property (nonatomic, strong) NSString *name;
@end

Resum

Els protocols i delegats són característiques poderoses en Objective-C que promouen la reutilització i flexibilitat del codi. Els protocols defineixen un conjunt de mètodes que poden ser implementats per qualsevol classe, assegurant una interfície consistent. Els delegats permeten que un objecte es comuniqui amb un altre, permetent la programació basada en esdeveniments.

Entenent i utilitzant protocols i delegats, pots escriure codi Objective-C més modular i fàcil de mantenir.
