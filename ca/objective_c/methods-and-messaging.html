Mètodes i Missatgeria en Objective-C
Herència ►

Introducció

En Objective-C, els mètodes i la missatgeria són conceptes fonamentals que permeten als objectes interactuar entre ells. Entendre aquests conceptes és crucial per desenvolupar aplicacions robustes i eficients en Objective-C.

Mètodes

Els mètodes en Objective-C són similars a les funcions en altres llenguatges de programació però estan associats amb classes o instàncies de classes.

Definició de Mètodes

Els mètodes es defineixen en el fitxer d'implementació (.m) i es declaren en el fitxer de capçalera (.h).

Exemple:

// Fitxer de capçalera: MyClass.h
#import <Foundation/Foundation.h>

@interface MyClass : NSObject

- (void)sayHello;
- (int)addNumber:(int)a toNumber:(int)b;

@end

// Fitxer d'implementació: MyClass.m
#import "MyClass.h"

@implementation MyClass

- (void)sayHello {
    NSLog(@"Hello, World!");
}

- (int)addNumber:(int)a toNumber:(int)b {
    return a + b;
}

@end

Tipus de Mètodes

  • Mètodes d'Instància: Operen sobre una instància d'una classe.
    • Definits amb un prefix - (guió).
  • Mètodes de Classe: Operen sobre la classe mateixa.
    • Definits amb un prefix + (signe més).

Exemple:

// Fitxer de capçalera: MyClass.h
@interface MyClass : NSObject

- (void)instanceMethod;
+ (void)classMethod;

@end

// Fitxer d'implementació: MyClass.m
@implementation MyClass

- (void)instanceMethod {
    NSLog(@"This is an instance method.");
}

+ (void)classMethod {
    NSLog(@"This is a class method.");
}

@end

Missatgeria

La missatgeria en Objective-C és el procés d'enviar missatges als objectes per invocar mètodes.

Enviament de Missatges

Els missatges s'envien utilitzant la sintaxi de claudàtors.

Exemple:

MyClass *myObject = [[MyClass alloc] init];
[myObject sayHello]; // Enviant un missatge a una instància
[MyClass classMethod]; // Enviant un missatge a la classe

Naturalesa Dinàmica

Objective-C utilitza un temps d'execució dinàmic, el que significa que les crides a mètodes es resolen en temps d'execució en lloc de temps de compilació. Això permet un codi més flexible i dinàmic.

Reenviament de Missatges

Si un objecte no reconeix un missatge, pot reenviar el missatge a un altre objecte.

Exemple:

// Fitxer de capçalera: MyClass.h
@interface MyClass : NSObject

@end

// Fitxer d'implementació: MyClass.m
#import "MyClass.h"

@implementation MyClass

- (void)forwardInvocation:(NSInvocation *)anInvocation {
    SEL selector = [anInvocation selector];
    if ([anotherObject respondsToSelector:selector]) {
        [anInvocation invokeWithTarget:anotherObject];
    } else {
        [super forwardInvocation:anInvocation];
    }
}

@end

Conceptes Clau

  • Selectors: Representen el nom d'un mètode.
    • Creats utilitzant la directiva @selector.
    • Exemple: SEL mySelector = @selector(sayHello);
  • Tipatge Dinàmic: Els objectes poden ser tipats dinàmicament utilitzant id.
    • Exemple: id myObject = [[MyClass alloc] init];
  • Introspecció: Capacitat de consultar els objectes sobre les seves propietats i capacitats en temps d'execució.
    • Exemple: [myObject isKindOfClass:[MyClass class]]

Resum

Entendre els mètodes i la missatgeria en Objective-C és essencial per a una programació orientada a objectes efectiva. Els mètodes defineixen el comportament dels objectes, mentre que la missatgeria permet als objectes comunicar-se i interactuar dinàmicament. En dominar aquests conceptes, pots crear aplicacions en Objective-C flexibles i potents.
