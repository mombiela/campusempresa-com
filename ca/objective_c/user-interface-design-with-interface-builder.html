Introducció

El disseny d'interfície d'usuari (UI) és un aspecte crucial en el desenvolupament d'aplicacions. En Objective-C, Interface Builder és una eina potent que permet als desenvolupadors dissenyar i construir interfícies d'usuari de manera visual. Aquesta secció et guiarà des dels conceptes bàsics fins als avançats de l'ús d'Interface Builder en Xcode.

Començant amb Interface Builder

Què és Interface Builder?

  • Interface Builder és una eina gràfica integrada en Xcode.
  • Permet als desenvolupadors dissenyar i provar interfícies d'usuari sense escriure codi.
  • Les interfícies es guarden en arxius .xib o .storyboard.

Creant la Teva Primera Interfície

  1. Obre Xcode i Crea un Nou Projecte:

    • Selecciona "App" sota iOS.
    • Tria "Storyboard" com a opció d'interfície.

  2. Entenent l'Espai de Treball d'Interface Builder:

    • Canvas: L'àrea principal on dissenyes la teva UI.
    • Library: Conté elements de UI (botons, etiquetes, etc.) que pots arrossegar al canvas.
    • Inspector: Et permet configurar propietats dels elements de UI.

  3. Afegint Elements de UI:

    • Arrossega un UIButton des de la Library al Canvas.
    • Arrossega un UILabel des de la Library al Canvas.

Connectant Elements de UI al Codi

  • IBOutlet: Connecta elements de UI al teu codi.
  • IBAction: Connecta accions d'usuari (com taps de botó) al teu codi.

Exemple:

// ViewController.h
#import <UIKit/UIKit.h>

@interface ViewController : UIViewController

@property (weak, nonatomic) IBOutlet UILabel *myLabel;
- (IBAction)buttonTapped:(id)sender;

@end

// ViewController.m
#import "ViewController.h"

@implementation ViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    // Configuració addicional si cal
}

- (IBAction)buttonTapped:(id)sender {
    self.myLabel.text = @"Button was tapped!";
}

@end
  • Passos:
    1. Control-arrossega des del UIButton al header del ViewController per crear un IBAction.
    2. Control-arrossega des del UILabel al header del ViewController per crear un IBOutlet.

Tècniques Avançades d'Interface Builder

Auto Layout

  • Què és Auto Layout?: Un sistema que et permet crear interfícies adaptatives que responen als canvis de mida i orientació de la pantalla.
  • Constraints: Regles que defineixen la posició i mida dels elements de UI.

Exemple:

// No es necessita codi; estableix les constraints en Interface Builder
  • Passos:
    1. Selecciona un element de UI.
    2. Fes clic al botó "Add New Constraints".
    3. Estableix constraints per a amplada, alçada, leading, trailing, top i bottom.

Segues i Navegació

  • Què és un Segue?: Una transició entre dos view controllers.
  • Tipus de Segues: Show, Modal, Popover, Custom.

Exemple:

// Prepara el segue en ViewController.m
- (void)prepareForSegue:(UIStoryboardSegue *)segue sender:(id)sender {
    if ([segue.identifier isEqualToString:@"showDetail"]) {
        // Passa dades al view controller de destinació
        DetailViewController *detailVC = segue.destinationViewController;
        detailVC.data = self.someData;
    }
}
  • Passos:
    1. Control-arrossega des d'un botó a un altre view controller per crear un segue.
    2. Estableix l'identificador del segue en l'Inspector d'Atributs.

Components de UI Personalitzats

  • Creant Vistes Personalitzades: Subclass UIView i dissenya en Interface Builder.
  • Components Reutilitzables: Utilitza arxius .xib per a components de UI reutilitzables.

Exemple:

// CustomView.h
#import <UIKit/UIKit.h>

@interface CustomView : UIView

@property (weak, nonatomic) IBOutlet UILabel *customLabel;

@end

// CustomView.m
#import "CustomView.h"

@implementation CustomView

- (instancetype)initWithCoder:(NSCoder *)coder {
    self = [super initWithCoder:coder];
    if (self) {
        // Carrega l'arxiu .xib
        UIView *view = [[[NSBundle mainBundle] loadNibNamed:@"CustomView" owner:self options:nil] firstObject];
        [self addSubview:view];
        view.frame = self.bounds;
    }
    return self;
}

@end
  • Passos:
    1. Crea un nou arxiu .xib i dissenya la teva vista personalitzada.
    2. Crea una subclasse corresponent de UIView.
    3. Carrega l'arxiu .xib en la subclasse.

Conclusió

Interface Builder és una eina essencial per dissenyar interfícies d'usuari en Objective-C. Dominant els conceptes bàsics d'afegir i connectar elements de UI, utilitzar Auto Layout, implementar segues i crear components personalitzats, pots construir interfícies sofisticades i responsives. Practica aquestes tècniques per convertir-te en un expert en disseny de UI amb Interface Builder.
