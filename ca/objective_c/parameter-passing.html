Passatge de Paràmetres en Objective-C
Tipus de Retorn ►

El passatge de paràmetres és un concepte fonamental en la programació que implica proporcionar entrades a funcions o mètodes. En Objective-C, entendre com es passen els paràmetres als mètodes és crucial per escriure codi eficient i efectiu. Aquest tema cobrirà les bases del passatge de paràmetres, diferents tipus de paràmetres i tècniques avançades.

Bases del Passatge de Paràmetres

En Objective-C, els paràmetres es poden passar als mètodes de dues maneres principals:

  • Per Valor
  • Per Referència

Passatge de Paràmetres per Valor

Quan els paràmetres es passen per valor, es fa una còpia del valor real del paràmetre a la memòria. Això significa que els canvis fets al paràmetre dins del mètode no afecten el valor original.

Exemple:

#import <Foundation/Foundation.h>

void modifyValue(int value) {
    value = 100;
    NSLog(@"Inside method: %d", value);
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        int originalValue = 50;
        NSLog(@"Before method call: %d", originalValue);
        modifyValue(originalValue);
        NSLog(@"After method call: %d", originalValue);
    }
    return 0;
}

Explicació:

  • originalValue es passa a modifyValue per valor.
  • Dins de modifyValue, el paràmetre value es modifica, però això no afecta originalValue.

Passatge de Paràmetres per Referència

Quan els paràmetres es passen per referència, el mètode rep una referència (o punter) al paràmetre real. Això permet que el mètode modifiqui el paràmetre original.

Exemple:

#import <Foundation/Foundation.h>

void modifyValue(int *value) {
    *value = 100;
    NSLog(@"Inside method: %d", *value);
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        int originalValue = 50;
        NSLog(@"Before method call: %d", originalValue);
        modifyValue(&originalValue);
        NSLog(@"After method call: %d", originalValue);
    }
    return 0;
}

Explicació:

  • originalValue es passa a modifyValue per referència utilitzant l'operador d'adreça (&).
  • Dins de modifyValue, el punter desreferenciat (*value) es modifica, cosa que afecta originalValue.

Passatge de Paràmetres Avançat

Passatge d'Objectes

En Objective-C, els objectes es passen típicament per referència. Això significa que quan passes un objecte a un mètode, estàs passant una referència a l'objecte, no una còpia de l'objecte.

Exemple:

#import <Foundation/Foundation.h>

@interface MyClass : NSObject
@property (nonatomic, strong) NSString *name;
@end

@implementation MyClass
@end

void modifyObject(MyClass *obj) {
    obj.name = @"New Name";
    NSLog(@"Inside method: %@", obj.name);
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        MyClass *myObject = [[MyClass alloc] init];
        myObject.name = @"Original Name";
        NSLog(@"Before method call: %@", myObject.name);
        modifyObject(myObject);
        NSLog(@"After method call: %@", myObject.name);
    }
    return 0;
}

Explicació:

  • myObject es passa a modifyObject per referència.
  • Dins de modifyObject, la propietat name de myObject es modifica, cosa que afecta l'objecte original.

Paràmetres Const

Per evitar que un mètode modifiqui un paràmetre, pots utilitzar la paraula clau const. Això és útil per passar paràmetres per referència assegurant-se que no es modifiquin.

Exemple:

#import <Foundation/Foundation.h>

void printValue(const int *value) {
    NSLog(@"Value: %d", *value);
    // *value = 100; // Això causaria un error de compilació
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        int originalValue = 50;
        printValue(&originalValue);
    }
    return 0;
}

Explicació:

  • value és un punter constant a un enter.
  • Intentar modificar *value dins de printValue resultaria en un error de compilació.

Conclusió

Entendre el passatge de paràmetres en Objective-C és essencial per escriure codi robust i mantenible. En dominar tant les tècniques bàsiques com les avançades, pots controlar com es passen les dades als mètodes, assegurant que els teus programes es comportin com s'espera. Tant si passes per valor com per referència, cada mètode té els seus casos d'ús i implicacions, que s'han de considerar acuradament durant el desenvolupament.
