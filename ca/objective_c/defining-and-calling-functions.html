Definició i Crida de Funcions en Objective-C
Sintaxi de Mètodes ►

Introducció

En Objective-C, les funcions són blocs fonamentals que et permeten encapsular codi en unitats reutilitzables. Aquesta secció et guiarà a través dels conceptes bàsics de la definició i crida de funcions, progressant des de conceptes per a principiants fins a conceptes avançats.

Definició de Funcions

Definició Bàsica de Funció

Una funció en Objective-C es defineix utilitzant la següent sintaxi:

returnType functionName(parameterType parameterName, ...) {
    // Cos de la funció
}
  • returnType: El tipus de valor que retorna la funció. Utilitza void si la funció no retorna cap valor.
  • functionName: El nom de la funció.
  • parameterType: El tipus del paràmetre.
  • parameterName: El nom del paràmetre.

Exemple

#import <Foundation/Foundation.h>

void sayHello() {
    NSLog(@"Hello, World!");
}

En aquest exemple:

  • void indica que la funció no retorna cap valor.
  • sayHello és el nom de la funció.
  • El cos de la funció conté una única instrucció que imprimeix "Hello, World!" a la consola.

Funció amb Paràmetres

Les funcions poden acceptar paràmetres per operar sobre dades diferents.

Exemple

#import <Foundation/Foundation.h>

int add(int a, int b) {
    return a + b;
}

En aquest exemple:

  • int és el tipus de retorn.
  • add és el nom de la funció.
  • La funció pren dos paràmetres int, a i b, i retorna la seva suma.

Crida de Funcions

Crida Bàsica de Funció

Per cridar una funció, utilitza el seu nom seguit de parèntesis que continguin els arguments necessaris.

Exemple

int result = add(5, 3);
NSLog(@"The result is %d", result);

En aquest exemple:

  • add(5, 3) crida la funció add amb els arguments 5 i 3.
  • El resultat es guarda a la variable result i després es imprimeix a la consola.

Crida de Funcions sense Paràmetres

Per a funcions que no prenen paràmetres, simplement utilitza parèntesis buits.

Exemple

sayHello();

Conceptes Avançats

Prototips de Funció

En programes més grans, és comú declarar prototips de funció abans de definir-les. Això ajuda al compilador a entendre les signatures de les funcions abans de les seves definicions reals.

Exemple

#import <Foundation/Foundation.h>

// Prototip de funció
int multiply(int a, int b);

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        int result = multiply(4, 5);
        NSLog(@"The result is %d", result);
    }
    return 0;
}

// Definició de funció
int multiply(int a, int b) {
    return a * b;
}

Funcions Estàtiques

Les funcions estàtiques estan limitades en abast al fitxer en què es declaren. Això és útil per encapsular funcionalitats que no haurien de ser accessibles des d'altres fitxers.

Exemple

#import <Foundation/Foundation.h>

static void printMessage() {
    NSLog(@"This is a static function.");
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        printMessage();
    }
    return 0;
}

Funcions Recursives

Una funció que es crida a si mateixa es coneix com una funció recursiva. La recursió és útil per resoldre problemes que es poden descompondre en problemes més petits i similars.

Exemple

#import <Foundation/Foundation.h>

int factorial(int n) {
    if (n == 0) {
        return 1;
    } else {
        return n * factorial(n - 1);
    }
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        int result = factorial(5);
        NSLog(@"Factorial of 5 is %d", result);
    }
    return 0;
}

Resum

  • Les funcions en Objective-C es defineixen utilitzant una sintaxi específica que inclou el tipus de retorn, el nom de la funció i els paràmetres.
  • Les funcions es poden cridar utilitzant el seu nom seguit de parèntesis que continguin els arguments necessaris.
  • Els conceptes avançats inclouen prototips de funció, funcions estàtiques i funcions recursives.
  • Entendre com definir i cridar funcions és crucial per escriure codi modular i reutilitzable en Objective-C.

En dominar aquests conceptes, estaràs ben equipat per escriure programes eficients i organitzats en Objective-C.
