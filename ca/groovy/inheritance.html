Introducció a l'Herència

L'herència és un concepte fonamental en la programació orientada a objectes (OOP) que permet a una classe heretar propietats i mètodes d'una altra classe. Això promou la reutilització del codi i estableix una jerarquia natural entre classes.

  • Classe superior (Classe Pare): La classe les propietats i mètodes de la qual són heretats.
  • Subclasse (Classe Filla): La classe que hereta de la classe superior.

Herència Bàsica en Groovy

En Groovy, l'herència s'implementa utilitzant la paraula clau extends. Comencem amb un exemple senzill per il·lustrar l'herència bàsica.

Exemple: Herència Bàsica

// Classe superior
class Animal {
    String name

    void eat() {
        println("${name} is eating")
    }
}

// Subclasse
class Dog extends Animal {
    void bark() {
        println("${name} is barking")
    }
}

// Creant una instància de Dog
Dog dog = new Dog()
dog.name = "Buddy"
dog.eat()  // Mètode heretat
dog.bark() // Mètode de la subclasse

Explicació

  • La classe Animal és la classe superior amb una propietat name i un mètode eat().
  • La classe Dog és la subclasse que estèn Animal i afegeix un nou mètode bark().
  • Una instància de Dog pot accedir tant al mètode eat() de Animal com al mètode bark() definit en Dog.

Sobreescriptura de Mètodes

Una subclasse pot sobreescriure mètodes de la seva classe superior per proporcionar una implementació específica.

Exemple: Sobreescriptura de Mètodes

class Animal {
    String name

    void eat() {
        println("${name} is eating")
    }
}

class Dog extends Animal {
    @Override
    void eat() {
        println("${name} is eating dog food")
    }
}

Dog dog = new Dog()
dog.name = "Buddy"
dog.eat()  // Imprimeix: Buddy is eating dog food

Explicació

  • La classe Dog sobreescriu el mètode eat() de la classe Animal.
  • L'anotació @Override s'utilitza per indicar que el mètode està sent sobreescrit.

Utilitzant la Paraula Clau `super`

La paraula clau super s'utilitza per cridar els mètodes i constructors de la classe superior des de la subclasse.

Exemple: Utilitzant la Paraula Clau `super`

class Animal {
    String name

    Animal(String name) {
        this.name = name
    }

    void eat() {
        println("${name} is eating")
    }
}

class Dog extends Animal {
    Dog(String name) {
        super(name)  // Cridant el constructor de la classe superior
    }

    @Override
    void eat() {
        super.eat()  // Cridant el mètode de la classe superior
        println("${name} is eating dog food")
    }
}

Dog dog = new Dog("Buddy")
dog.eat()
// Imprimeix:
// Buddy is eating
// Buddy is eating dog food

Explicació

  • El constructor de Dog crida el constructor de Animal utilitzant super(name).
  • El mètode eat() en Dog crida el mètode eat() de Animal utilitzant super.eat() abans d'afegir el seu propi comportament.

Classes Abstractes

Les classes abstractes no poden ser instanciades i estan destinades a ser subclassificades. Poden contenir mètodes abstractes que han de ser implementats per les subclasses.

Exemple: Classes Abstractes

abstract class Animal {
    String name

    Animal(String name) {
        this.name = name
    }

    abstract void makeSound()  // Mètode abstracte
}

class Dog extends Animal {
    Dog(String name) {
        super(name)
    }

    @Override
    void makeSound() {
        println("${name} is barking")
    }
}

Dog dog = new Dog("Buddy")
dog.makeSound()  // Imprimeix: Buddy is barking

Explicació

  • La classe Animal és abstracta i conté un mètode abstracte makeSound().
  • La classe Dog estèn Animal i proporciona una implementació per al mètode makeSound().

Resum

L'herència en Groovy permet a les classes heretar propietats i mètodes d'altres classes, promovent la reutilització del codi i establint una jerarquia. Els conceptes clau inclouen:

  • Herència bàsica utilitzant la paraula clau extends.
  • Sobreescriptura de mètodes per proporcionar implementacions específiques.
  • Utilització de la paraula clau super per cridar mètodes i constructors de la classe superior.
  • Classes abstractes que defineixen mètodes abstractes per ser implementats per les subclasses.

Comprenent i utilitzant l'herència, pots crear codi més modular, mantenible i reutilitzable en Groovy.
