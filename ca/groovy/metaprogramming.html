Metaprogramació en Groovy
Constructors ►

La metaprogramació és una característica poderosa en Groovy que et permet escriure codi que escriu o modifica altre codi en temps d'execució. Això pot ser particularment útil per crear aplicacions més flexibles i dinàmiques. En aquesta secció, explorarem els conceptes bàsics de la metaprogramació en Groovy, movent-nos des de conceptes per a principiants fins a tècniques més avançades.

Introducció a la Metaprogramació

La metaprogramació en Groovy es pot dividir en dues categories principals:

  • Metaprogramació en temps de compilació: Modificar el codi durant el procés de compilació.
  • Metaprogramació en temps d'execució: Modificar el codi mentre el programa s'està executant.

Conceptes Clau

  • Invocació Dinàmica de Mètodes: Afegir mètodes a les classes en temps d'execució.
  • ExpandoMetaClass: Una classe que et permet afegir mètodes, propietats i constructors a classes existents.
  • Categories: Afegir mètodes a classes temporalment dins d'un àmbit específic.
  • Transformacions AST: Modificar l'Arbre de Sintaxi Abstracta (AST) durant la compilació.

Invocació Dinàmica de Mètodes

La invocació dinàmica de mètodes et permet afegir mètodes a les classes en temps d'execució. Això es pot fer utilitzant la propietat metaClass.

Exemple

class Person {
    String name
}

Person.metaClass.sayHello = { ->
    println "Hello, my name is ${delegate.name}"
}

def john = new Person(name: 'John')
john.sayHello()  // Output: Hello, my name is John

En aquest exemple, afegim dinàmicament un mètode sayHello a la classe Person.

ExpandoMetaClass

L'ExpandoMetaClass et permet afegir mètodes, propietats i constructors a classes existents. Està habilitat per defecte en Groovy.

Exemple

class Car {
    String model
}

Car.metaClass.startEngine = { ->
    println "Engine started for ${delegate.model}"
}

def tesla = new Car(model: 'Tesla Model S')
tesla.startEngine()  // Output: Engine started for Tesla Model S

Aquí, afegim un mètode startEngine a la classe Car utilitzant ExpandoMetaClass.

Categories

Les categories et permeten afegir mètodes a classes dins d'un àmbit específic. Això és útil quan vols estendre classes temporalment.

Exemple

class StringCategory {
    static String shout(String str) {
        return str.toUpperCase() + "!!!"
    }
}

use(StringCategory) {
    println "hello".shout()  // Output: HELLO!!!
}

En aquest exemple, definim una classe StringCategory amb un mètode shout i l'utilitzem dins d'un àmbit específic.

Transformacions AST

Les transformacions AST (Arbre de Sintaxi Abstracta) et permeten modificar l'estructura del codi durant el procés de compilació. Groovy proporciona diverses transformacions AST integrades, com ara @Delegate, @Immutable i @Singleton.

Exemple

import groovy.transform.Immutable

@Immutable
class Point {
    int x, y
}

def p = new Point(x: 10, y: 20)
println p  // Output: Point(10, 20)

En aquest exemple, la transformació @Immutable fa que la classe Point sigui immutable, és a dir, que les seves propietats no es poden canviar després de la inicialització.

Conclusió

La metaprogramació en Groovy proporciona eines poderoses per modificar i estendre el teu codi dinàmicament. Entenent i utilitzant la invocació dinàmica de mètodes, ExpandoMetaClass, categories i transformacions AST, pots crear aplicacions més flexibles i dinàmiques. A mesura que et familiaritzis amb aquests conceptes, podràs aprofitar tot el potencial de les capacitats de metaprogramació de Groovy.
