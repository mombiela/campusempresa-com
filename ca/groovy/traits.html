Introducció als Traits

Els traits en Groovy són una característica poderosa que et permet compondre comportaments en les teves classes. Són similars a les interfícies en Java però amb la capacitat afegida d'incloure implementacions de mètodes. Els traits t'ajuden a aconseguir herència múltiple i reutilització de codi d'una manera neta i manejable.

Conceptes Clau

  • Definició: Els traits es defineixen utilitzant la paraula clau trait.
  • Composició: Les classes poden implementar múltiples traits.
  • Mètodes Per Defecte: Els traits poden proporcionar implementacions de mètodes per defecte.
  • Estat: Els traits poden tenir camps i propietats.
  • Resolució de Conflictes: Mecanismes per resoldre conflictes quan múltiples traits proporcionen el mateix mètode.

Definició i Ús dels Traits

Definició d'un Trait

Per definir un trait, utilitza la paraula clau trait seguida del nom del trait i el seu cos.

trait Greeter {
    String greet(String name) {
        return "Hello, $name"
    }
}

Implementació d'un Trait en una Classe

Una classe pot implementar un o més traits utilitzant la paraula clau implements.

class Person implements Greeter {
    String name

    Person(String name) {
        this.name = name
    }
}

def person = new Person("Alice")
println(person.greet(person.name))  // Output: Hello, Alice

Múltiples Traits

Una classe pot implementar múltiples traits, i heretarà tots els mètodes i propietats d'aquests traits.

trait Farewell {
    String sayGoodbye(String name) {
        return "Goodbye, $name"
    }
}

class FriendlyPerson implements Greeter, Farewell {
    String name

    FriendlyPerson(String name) {
        this.name = name
    }
}

def friendlyPerson = new FriendlyPerson("Bob")
println(friendlyPerson.greet(friendlyPerson.name))  // Output: Hello, Bob
println(friendlyPerson.sayGoodbye(friendlyPerson.name))  // Output: Goodbye, Bob

Característiques Avançades dels Traits

Mètodes Per Defecte i Sobreescriptura

Els traits poden proporcionar implementacions per defecte per als mètodes, que poden ser sobreescrits per la classe que els implementa.

trait Speaker {
    String speak() {
        return "I am speaking"
    }
}

class LoudSpeaker implements Speaker {
    @Override
    String speak() {
        return "I am speaking loudly!"
    }
}

def loudSpeaker = new LoudSpeaker()
println(loudSpeaker.speak())  // Output: I am speaking loudly!

Estat en els Traits

Els traits poden tenir camps i propietats, que poden ser accedits i modificats per la classe que els implementa.

trait Counter {
    int count = 0

    void increment() {
        count++
    }
}

class CounterUser implements Counter {}

def counterUser = new CounterUser()
counterUser.increment()
println(counterUser.count)  // Output: 1

Resolució de Conflictes

Quan múltiples traits proporcionen el mateix mètode, has de resoldre el conflicte explícitament.

trait A {
    String message() {
        return "Message from A"
    }
}

trait B {
    String message() {
        return "Message from B"
    }
}

class C implements A, B {
    @Override
    String message() {
        A.super.message() + " and " + B.super.message()
    }
}

def c = new C()
println(c.message())  // Output: Message from A and Message from B

Conclusió

Els traits en Groovy proporcionen una manera flexible i poderosa de compondre comportaments en les teves classes. Et permeten aconseguir herència múltiple, reutilitzar codi i gestionar conflictes de manera efectiva. Entenent i utilitzant els traits, pots escriure codi Groovy més modular, manejable i expressiu.
