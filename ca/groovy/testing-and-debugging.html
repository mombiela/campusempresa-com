Introducció

Les proves i la depuració són aspectes crucials del desenvolupament de programari. Asseguren que el teu codi funcioni com s'espera i ajuden a identificar i solucionar problemes. En aquesta secció, cobrirem els conceptes bàsics de les proves i la depuració en Groovy, progressant des de nivells principiants fins a avançats.

Proves Unitàries amb Groovy

Què són les Proves Unitàries?

  • Les proves unitàries impliquen provar components o funcions individuals del teu codi per assegurar-se que funcionen correctament.
  • Ajuda a detectar errors en les primeres etapes del procés de desenvolupament.

Configuració d'un Entorn de Proves

  • Groovy utilitza JUnit, un marc de proves popular per a Java, que s'integra perfectament amb Groovy.
  • Per començar, has d'incloure JUnit en les dependències del teu projecte.
// build.gradle
dependencies {
    testImplementation 'junit:junit:4.13.2'
}

Escriure la Teva Primera Prova

  • Crea una classe de prova i anota-la amb @Test.
  • Utilitza afirmacions per verificar els resultats esperats.
import org.junit.Test
import static org.junit.Assert.*

class CalculatorTest {
    @Test
    void testAddition() {
        def calculator = new Calculator()
        assertEquals(5, calculator.add(2, 3))
    }
}

class Calculator {
    int add(int a, int b) {
        return a + b
    }
}

Executar Proves

  • Pots executar les teves proves utilitzant Gradle o un IDE com IntelliJ IDEA o Eclipse.
  • En Gradle, utilitza la comanda ./gradlew test.

Tècniques Avançades de Proves

Mocking i Stubbing

  • Mocking i stubbing són tècniques utilitzades per simular el comportament d'objectes complexos.
  • Groovy proporciona el paquet groovy.mock.interceptor per crear mocks.
import groovy.mock.interceptor.*

class UserService {
    def userRepository

    def getUserById(id) {
        return userRepository.findById(id)
    }
}

class UserServiceTest {
    @Test
    void testGetUserById() {
        def userRepository = Mock(UserRepository)
        userRepository.findById(1) >> new User(id: 1, name: 'John Doe')

        def userService = new UserService(userRepository: userRepository)
        def user = userService.getUserById(1)

        assertEquals('John Doe', user.name)
    }
}

Proves Parametritzades

  • Les proves parametritzades et permeten executar la mateixa prova amb diferents entrades.
  • JUnit proporciona les anotacions @RunWith i @Parameters per a aquest propòsit.
import org.junit.Test
import org.junit.runner.RunWith
import org.junit.runners.Parameterized

@RunWith(Parameterized)
class CalculatorParameterizedTest {
    int a, b, expected

    CalculatorParameterizedTest(int a, int b, int expected) {
        this.a = a
        this.b = b
        this.expected = expected
    }

    @Parameterized.Parameters
    static data() {
        return [
            [1, 2, 3],
            [2, 3, 5],
            [3, 3, 6]
        ]
    }

    @Test
    void testAddition() {
        def calculator = new Calculator()
        assertEquals(expected, calculator.add(a, b))
    }
}

Depuració de Codi Groovy

Utilitzant Sentències Print

  • La forma més senzilla de depurar és utilitzar sentències print per mostrar els valors de les variables i el flux del programa.
def add(a, b) {
    println "Adding $a and $b"
    return a + b
}

println add(2, 3)

Utilitzant un Depurador

  • Els IDEs moderns com IntelliJ IDEA i Eclipse vénen amb depuradors integrats.
  • Col·loca punts de ruptura en el teu codi i executa el depurador per seguir el teu codi línia per línia.

Tècniques Comunes de Depuració

  • Punts de Ruptura: Pausa l'execució en línies específiques.
  • Step Over: Executa la línia actual i passa a la següent.
  • Step Into: Entra en les crides de mètode.
  • Watch Variables: Monitoritza els valors de les variables en temps real.

Conclusió

Les proves i la depuració són habilitats essencials per a qualsevol programador. En Groovy, pots aprofitar eines potents com JUnit per a proves i IDEs moderns per a la depuració per assegurar-te que el teu codi és robust i lliure d'errors. Dominant aquestes tècniques, estaràs ben preparat per manejar qualsevol problema que sorgeixi durant el desenvolupament.
