Introducció a la Depuració

La depuració és el procés d'identificar i eliminar errors dels programes informàtics. En Kotlin, com en qualsevol altre llenguatge de programació, una depuració efectiva és crucial per desenvolupar aplicacions robustes i lliures d'errors.

Conceptes Clau

  • Bug: Un error, defecte o fallada en un programa que fa que produeixi resultats incorrectes o inesperats.
  • Depuració: El procés de trobar i corregir errors.
  • Punt de trencament: Un marcador establert pel desenvolupador per pausar l'execució del programa en un punt específic.
  • Traça de pila: Un informe que proporciona informació sobre els marcs de pila actius en un moment determinat durant l'execució d'un programa.

Tècniques Bàsiques de Depuració

Ús de Sentències Print

Una de les maneres més senzilles de depurar és utilitzar sentències print per mostrar els valors de les variables en diferents punts del programa.

fun main() {
    val numbers = listOf(1, 2, 3, 4, 5)
    for (number in numbers) {
        println("Current number: $number")
    }
}
  • Explicació: La sentència println ajuda a seguir el valor de number durant cada iteració del bucle.

Llegir Traces de Pila

Quan es produeix una excepció, Kotlin proporciona una traça de pila que pot ajudar a identificar on ha ocorregut l'error.

fun main() {
    val numbers = listOf(1, 2, 3, 4, 5)
    println(numbers[5]) // Això llançarà una IndexOutOfBoundsException
}
  • Explicació: La traça de pila indicarà que l'error ha ocorregut a println(numbers[5]) perquè l'índex 5 està fora dels límits de la llista.

Tècniques Intermèdies de Depuració

Ús de Punts de Trencament

Els punts de trencament permeten pausar l'execució del programa en punts específics per inspeccionar l'estat de l'aplicació.

  • Establir Punts de Trencament: A IntelliJ IDEA, pots establir un punt de trencament fent clic a la vora al costat del número de línia.
  • Inspeccionar Variables: Un cop el programa està pausat, pots passar el cursor sobre les variables per veure els seus valors actuals.

Punts de Trencament Condicionals

Els punts de trencament condicionals pausen l'execució només quan es compleix una condició especificada.

  • Exemple: Establir un punt de trencament que només es desencadeni quan una variable x sigui més gran que 10.
  • Com Establir: Fes clic dret sobre el punt de trencament i afegeix la condició x > 10.

Tècniques Avançades de Depuració

Ús del Depurador de Kotlin

El Depurador de Kotlin a IntelliJ IDEA proporciona funcions avançades com avaluar expressions, avançar línia a línia pel codi i inspeccionar la pila de crides.

  • Step Over: Executa la línia actual i passa a la següent línia.
  • Step Into: Entra a la crida del mètode a la línia actual.
  • Step Out: Surt del mètode actual i torna al cridant.

Analitzar Fuites de Memòria

Les fuites de memòria poden fer que la teva aplicació consumeixi més memòria amb el temps, provocant bloquejos o un rendiment lent.

  • Eines: Utilitza eines com Android Profiler o VisualVM per monitoritzar l'ús de memòria.
  • Identificar Fuites: Busca objectes que no s'estan recollint per escombraries i investiga per què encara estan referenciats.

Registre

Utilitzar biblioteques de registre com Log4j o el registre integrat de Kotlin pot ajudar-te a seguir el comportament de l'aplicació al llarg del temps.

import java.util.logging.Logger

val logger = Logger.getLogger("MyLogger")

fun main() {
    logger.info("Application started")
    try {
        val result = 10 / 0
    } catch (e: ArithmeticException) {
        logger.severe("Error occurred: ${e.message}")
    }
}
  • Explicació: El logger proporciona diferents nivells de registre (info, warning, severe) per categoritzar la importància dels missatges.

Conclusió

La depuració és una habilitat essencial per a qualsevol desenvolupador de Kotlin. Començant amb tècniques bàsiques com les sentències print i la lectura de traces de pila, pots avançar gradualment cap a mètodes més avançats com l'ús de punts de trencament, l'anàlisi de fuites de memòria i l'ús del registre. Dominar aquestes tècniques t'ajudarà a escriure codi més fiable i mantenible.
