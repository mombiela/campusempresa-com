La delegació és un patró de disseny en Kotlin que permet a un objecte transferir algunes de les seves responsabilitats a un altre objecte. Això pot ser particularment útil per a la reutilització de codi i la composició. Kotlin proporciona suport integrat per a la delegació a través de la paraula clau by, fent més fàcil implementar patrons de delegació.

Què és la Delegació?

  • Definició: La delegació és un patró de disseny on un objecte delega algunes de les seves responsabilitats a un altre objecte.
  • Propòsit: Ajuda en la reutilització de codi i la composició, fent el codi més modular i més fàcil de mantenir.

Tipus de Delegació en Kotlin

  1. Delegació de Classes
  2. Delegació de Propietats

Delegació de Classes

La delegació de classes permet que una classe delegui la seva implementació a una altra classe. Això es fa utilitzant la paraula clau by.

Exemple

interface Printer {
    fun print()
}

class SimplePrinter : Printer {
    override fun print() {
        println("Printing from SimplePrinter")
    }
}

class AdvancedPrinter(printer: Printer) : Printer by printer

fun main() {
    val simplePrinter = SimplePrinter()
    val advancedPrinter = AdvancedPrinter(simplePrinter)
    advancedPrinter.print()  // Output: Printing from SimplePrinter
}
  • Explicació:
    • Printer és una interfície amb un mètode print.
    • SimplePrinter és una classe que implementa la interfície Printer.
    • AdvancedPrinter delega el mètode print a una instància de Printer utilitzant la paraula clau by.

Delegació de Propietats

La delegació de propietats permet delegar el getter i setter d'una propietat a un altre objecte. Kotlin proporciona diversos delegats integrats, com lazy, observable i vetoable.

Inicialització Peresosa

La inicialització peresosa és un cas d'ús comú per a la delegació de propietats. El delegat lazy inicialitza la propietat només quan s'accedeix per primera vegada.

Exemple

class LazyExample {
    val lazyValue: String by lazy {
        println("Computed!")
        "Hello, Kotlin!"
    }
}

fun main() {
    val example = LazyExample()
    println(example.lazyValue)  // Output: Computed! Hello, Kotlin!
    println(example.lazyValue)  // Output: Hello, Kotlin!
}
  • Explicació:
    • lazyValue és una propietat que utilitza el delegat lazy.
    • La primera vegada que s'accedeix a lazyValue, s'executa l'expressió lambda i es mostra "Computed!".
    • Els accessos posteriors a lazyValue no tornen a executar l'expressió lambda.

Propietats Observables

El delegat observable permet fer un seguiment dels canvis d'una propietat.

Exemple

import kotlin.properties.Delegates

class User {
    var name: String by Delegates.observable("<no name>") { prop, old, new ->
        println("Property ${prop.name} changed from $old to $new")
    }
}

fun main() {
    val user = User()
    user.name = "John"  // Output: Property name changed from <no name> to John
    user.name = "Doe"   // Output: Property name changed from John to Doe
}
  • Explicació:
    • name és una propietat que utilitza el delegat observable.
    • L'expressió lambda es crida cada vegada que la propietat canvia, imprimint els valors antics i nous.

Delegats de Propietats Personalitzats

També pots crear els teus propis delegats de propietats personalitzats implementant les interfícies ReadOnlyProperty o ReadWriteProperty.

Exemple

import kotlin.reflect.KProperty

class CustomDelegate {
    operator fun getValue(thisRef: Any?, property: KProperty<*>): String {
        return "Custom Delegate Value"
    }

    operator fun setValue(thisRef: Any?, property: KProperty<*>, value: String) {
        println("Setting value '$value' to '${property.name}'")
    }
}

class Example {
    var customProperty: String by CustomDelegate()
}

fun main() {
    val example = Example()
    println(example.customProperty)  // Output: Custom Delegate Value
    example.customProperty = "New Value"  // Output: Setting value 'New Value' to 'customProperty'
}
  • Explicació:
    • La classe CustomDelegate implementa els operadors getValue i setValue.
    • customProperty a la classe Example utilitza el CustomDelegate.

Conclusió

La delegació en Kotlin és una característica poderosa que permet un codi més net i modular. Utilitzant la delegació de classes i propietats, pots delegar fàcilment responsabilitats i comportaments a altres objectes, promovent la reutilització de codi i reduint el codi redundant. Ja sigui utilitzant delegats integrats com lazy i observable o creant delegats personalitzats, els mecanismes de delegació de Kotlin proporcionen una manera flexible i eficient de gestionar les responsabilitats dels objectes.
