Introducció a les Interfícies

Les interfícies en Kotlin són una manera poderosa de definir contractes que les classes poden implementar. Permeten especificar mètodes i propietats que han de ser implementats per qualsevol classe que implementi la interfície.

  • Definició: Una interfície és un pla d'una classe que conté mètodes i propietats abstractes.
  • Propòsit: Proporcionar una manera d'aconseguir abstracció i herència múltiple en Kotlin.

Conceptes Clau

  • Mètodes Abstractes: Mètodes sense cos que han de ser implementats per la classe.
  • Mètodes per Defecte: Mètodes amb cos que poden ser opcionalment sobreescrits per la classe.
  • Propietats en Interfícies: Les interfícies també poden contenir propietats, que poden ser abstractes o tenir implementacions per defecte.

Definint una Interfície

Per definir una interfície en Kotlin, utilitza la paraula clau interface.

interface Drivable {
    fun drive() // Mètode abstracte
    val maxSpeed: Int // Propietat abstracta
}

Implementant una Interfície

Una classe que implementa una interfície ha de proporcionar implementacions per a tots els mètodes i propietats abstractes.

class Car : Drivable {
    override fun drive() {
        println("El cotxe està conduint")
    }

    override val maxSpeed: Int = 200
}

Mètodes i Propietats per Defecte

Les interfícies també poden contenir mètodes i propietats amb implementacions per defecte.

interface Flyable {
    fun fly() {
        println("Volant")
    }

    val altitude: Int
        get() = 1000
}

class Airplane : Flyable {
    // No cal sobreescriure fly() o altitude tret que es necessiti un comportament personalitzat
}

Interfícies Múltiples

Una classe en Kotlin pot implementar múltiples interfícies, permetent un codi més flexible i reutilitzable.

interface Swimmable {
    fun swim()
}

class AmphibiousCar : Drivable, Swimmable {
    override fun drive() {
        println("El cotxe amfibi està conduint")
    }

    override val maxSpeed: Int = 150

    override fun swim() {
        println("El cotxe amfibi està nedant")
    }
}

Herència d'Interfícies

Les interfícies poden heretar d'altres interfícies, permetent dissenys més complexos i jeràrquics.

interface Vehicle {
    fun start()
}

interface ElectricVehicle : Vehicle {
    fun chargeBattery()
}

class Tesla : ElectricVehicle {
    override fun start() {
        println("El Tesla està arrencant")
    }

    override fun chargeBattery() {
        println("Carregant la bateria del Tesla")
    }
}

Exemples Pràctics

Exemple 1: Implementació Simple d'Interfície

interface Animal {
    fun sound()
}

class Dog : Animal {
    override fun sound() {
        println("Bup")
    }
}

fun main() {
    val dog = Dog()
    dog.sound() // Sortida: Bup
}

Exemple 2: Interfície amb Mètodes per Defecte

interface Printer {
    fun print() {
        println("Imprimint document")
    }
}

class LaserPrinter : Printer

fun main() {
    val printer = LaserPrinter()
    printer.print() // Sortida: Imprimint document
}

Exemple 3: Interfícies Múltiples

interface Scanner {
    fun scan()
}

class MultiFunctionPrinter : Printer, Scanner {
    override fun scan() {
        println("Escanejant document")
    }
}

fun main() {
    val mfp = MultiFunctionPrinter()
    mfp.print() // Sortida: Imprimint document
    mfp.scan()  // Sortida: Escanejant document
}

Conclusió

Les interfícies en Kotlin proporcionen un mecanisme robust per definir contractes que les classes poden implementar. Donen suport a l'abstracció, l'herència múltiple, i poden contenir tant mètodes i propietats abstractes com per defecte. Entenent i utilitzant les interfícies, pots crear codi més modular, reutilitzable i mantenible.
