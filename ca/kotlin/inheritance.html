El Projecte Sobre nosaltres Contribuir Donacions Llicència

L'herència és un concepte fonamental en la programació orientada a objectes que permet a una classe heretar propietats i funcions d'una altra classe. A Kotlin, l'herència és senzilla i proporciona una manera poderosa de reutilitzar codi i crear una estructura jeràrquica de classes.

Conceptes Clau

  • Classe Pare (Superclass): La classe les propietats i funcions de la qual són heretades.
  • Classe Filla (Subclass): La classe que hereta propietats i funcions de la classe pare.
  • Paraula Clau open: Per defecte, les classes a Kotlin són finals i no es poden heretar. La paraula clau open s'utilitza per fer una classe heretable.
  • Paraula Clau override: S'utilitza per sobreescriure una funció o propietat a la classe filla.

Creació d'una Classe Pare

A Kotlin, cal marcar una classe com open per permetre que altres classes l'heretin.

open class Animal {
    open fun sound() {
        println("Animal fa un so")
    }
}

Creació d'una Classe Filla

Una classe filla hereta d'una classe pare utilitzant el símbol :.

class Dog : Animal() {
    override fun sound() {
        println("El gos borda")
    }
}

Exemple: Herència Bàsica

Vegem un exemple complet d'herència a Kotlin.

// Classe Pare
open class Animal {
    open fun sound() {
        println("Animal fa un so")
    }
}

// Classe Filla
class Dog : Animal() {
    override fun sound() {
        println("El gos borda")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()  // Sortida: El gos borda
}

En aquest exemple:

  • Animal és la classe pare amb una funció oberta sound.
  • Dog és la classe filla que sobreescriu la funció sound.

Propietats en l'Herència

També pots heretar propietats de la classe pare.

open class Animal {
    open val type: String = "Desconegut"
}

class Dog : Animal() {
    override val type: String = "Gos"
}

fun main() {
    val myDog = Dog()
    println(myDog.type)  // Sortida: Gos
}

Cridar Funcions de la Classe Pare

Pots cridar la funció de la classe pare utilitzant la paraula clau super.

open class Animal {
    open fun sound() {
        println("Animal fa un so")
    }
}

class Dog : Animal() {
    override fun sound() {
        super.sound()  // Crida la funció de la classe pare
        println("El gos borda")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()
    // Sortida:
    // Animal fa un so
    // El gos borda
}

Classes Abstractes

De vegades, vols crear una classe pare que no s'hauria d'instanciar directament. Per això, pots utilitzar classes abstractes.

abstract class Animal {
    abstract fun sound()
}

class Dog : Animal() {
    override fun sound() {
        println("El gos borda")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()  // Sortida: El gos borda
}

En aquest exemple:

  • Animal és una classe abstracta amb una funció abstracta sound.
  • Dog és una classe filla que proporciona una implementació per a la funció sound.

Resum

  • L'herència permet a una classe heretar propietats i funcions d'una altra classe.
  • Utilitza la paraula clau open per fer una classe heretable.
  • Utilitza la paraula clau override per sobreescriure funcions i propietats a la classe filla.
  • Les classes abstractes es poden utilitzar per definir classes que no s'haurien d'instanciar directament però que es poden heretar.

Entenent i utilitzant l'herència, pots crear codi més modular, reutilitzable i mantenible a Kotlin.
