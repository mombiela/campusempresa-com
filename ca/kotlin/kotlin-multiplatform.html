Introducció a Kotlin Multiplatform

Kotlin Multiplatform és una característica poderosa de Kotlin que permet als desenvolupadors compartir codi entre múltiples plataformes com Android, iOS, JVM, JavaScript i sistemes natius. Això permet la creació d'aplicacions multiplataforma amb una única base de codi, reduint el temps i l'esforç de desenvolupament.

Conceptes Clau

  • Mòdul Comú: Conté codi que es comparteix entre totes les plataformes.
  • Mòduls Específics de Plataforma: Conté codi que és específic per a una plataforma en particular (per exemple, Android, iOS).
  • Mecanisme Expect/Actual: Et permet definir implementacions específiques de plataforma per al codi comú.

Configuració d'un Projecte Kotlin Multiplatform

Per començar amb Kotlin Multiplatform, necessites configurar el teu entorn de desenvolupament i crear un nou projecte.

Passos per Configurar

  1. Instal·lar el Plugin de Kotlin: Assegura't de tenir el plugin de Kotlin instal·lat al teu IDE (IntelliJ IDEA o Android Studio).
  2. Crear un Nou Projecte: Utilitza l'assistent de projectes per crear un nou projecte Kotlin Multiplatform.
  3. Configurar Gradle: Configura el teu fitxer build.gradle.kts per incloure els plugins i dependències necessaris.

Exemple de Configuració de Gradle

plugins {
    kotlin("multiplatform") version "1.5.31"
}

kotlin {
    jvm()
    js {
        browser {
            testTask {
                useKarma {
                    useChromeHeadless()
                }
            }
        }
    }
    ios()

    sourceSets {
        val commonMain by getting {
            dependencies {
                implementation(kotlin("stdlib-common"))
            }
        }
        val jvmMain by getting {
            dependencies {
                implementation(kotlin("stdlib-jdk8"))
            }
        }
        val jsMain by getting {
            dependencies {
                implementation(kotlin("stdlib-js"))
            }
        }
        val iosMain by getting
    }
}

Escriure Codi Compartit

El codi compartit s'escriu en el mòdul comú i es pot utilitzar a totes les plataformes. Això inclou la lògica de negoci, models de dades i funcions d'utilitat.

Exemple de Codi Compartit

// commonMain/src/commonMain/kotlin/com/example/SharedCode.kt
package com.example

expect fun platformName(): String

fun createGreeting(): String {
    return "Hello from ${platformName()}"
}

Implementacions Específiques de Plataforma

El codi específic de plataforma s'escriu en els respectius mòduls de plataforma. Aquí és on proporciones les implementacions reals per a les declaracions esperades en el mòdul comú.

Exemple de Codi Específic de Plataforma

// jvmMain/src/jvmMain/kotlin/com/example/Platform.kt
package com.example

actual fun platformName(): String {
    return "JVM"
}

// jsMain/src/jsMain/kotlin/com/example/Platform.kt
package com.example

actual fun platformName(): String {
    return "JavaScript"
}

// iosMain/src/iosMain/kotlin/com/example/Platform.kt
package com.example

actual fun platformName(): String {
    return "iOS"
}

Executar i Provar

Executar i provar el teu projecte Kotlin Multiplatform implica construir i executar el codi en cada plataforma objectiu.

Executar en JVM

./gradlew jvmMainClasses
java -cp build/classes/kotlin/jvm/main com.example.MainKt

Executar en JavaScript

./gradlew jsBrowserDevelopmentRun

Executar en iOS

Utilitza Xcode per construir i executar l'aplicació iOS.

Conclusió

Kotlin Multiplatform és una manera versàtil i eficient de desenvolupar aplicacions multiplataforma. Compartint codi entre plataformes, pots reduir significativament el temps de desenvolupament i mantenir una única base de codi. Entendre la configuració, el codi compartit, les implementacions específiques de plataforma i els procediments d'execució/prova són crucials per aprofitar tot el potencial de Kotlin Multiplatform.

Amb aquest coneixement, ara estàs preparat per començar a construir les teves pròpies aplicacions multiplataforma utilitzant Kotlin Multiplatform. Feliç programació!
