Els modificadors de visibilitat en Kotlin controlen l'accessibilitat de classes, objectes, interfícies, constructors, funcions, propietats i els seus setters. Entendre aquests modificadors és crucial per dissenyar codi robust i mantenible.

Principals Modificadors de Visibilitat

Kotlin proporciona quatre modificadors de visibilitat:

  • public
  • private
  • protected
  • internal

Públic

  • Modificador de visibilitat per defecte: Si no es especifica cap modificador de visibilitat, el membre és public per defecte.
  • Accessibilitat: Els membres públics són accessibles des de qualsevol lloc del projecte.
class PublicExample {
    public val publicProperty: String = "I am public"
    
    public fun publicFunction() {
        println("This is a public function")
    }
}

fun main() {
    val example = PublicExample()
    println(example.publicProperty) // Accessible
    example.publicFunction() // Accessible
}

Privat

  • Accessibilitat: Els membres privats són accessibles només dins del fitxer o la classe on es declaren.
class PrivateExample {
    private val privateProperty: String = "I am private"
    
    private fun privateFunction() {
        println("This is a private function")
    }
    
    fun accessPrivateMembers() {
        println(privateProperty) // Accessible dins de la classe
        privateFunction() // Accessible dins de la classe
    }
}

fun main() {
    val example = PrivateExample()
    // println(example.privateProperty) // No accessible
    // example.privateFunction() // No accessible
    example.accessPrivateMembers() // Accessible
}

Protegit

  • Accessibilitat: Els membres protegits són accessibles dins de la classe i les seves subclasses.
open class ProtectedExample {
    protected val protectedProperty: String = "I am protected"
    
    protected fun protectedFunction() {
        println("This is a protected function")
    }
}

class SubclassExample : ProtectedExample() {
    fun accessProtectedMembers() {
        println(protectedProperty) // Accessible dins de la subclasse
        protectedFunction() // Accessible dins de la subclasse
    }
}

fun main() {
    val example = SubclassExample()
    example.accessProtectedMembers() // Accessible
    // println(example.protectedProperty) // No accessible
    // example.protectedFunction() // No accessible
}

Intern

  • Accessibilitat: Els membres interns són accessibles dins del mateix mòdul.
internal class InternalExample {
    internal val internalProperty: String = "I am internal"
    
    internal fun internalFunction() {
        println("This is an internal function")
    }
}

fun main() {
    val example = InternalExample()
    println(example.internalProperty) // Accessible dins del mateix mòdul
    example.internalFunction() // Accessible dins del mateix mòdul
}

Conclusió

Els modificadors de visibilitat en Kotlin són essencials per controlar l'accés a diferents parts del teu codi. Utilitzant public, private, protected i internal adequadament, pots assegurar-te que el teu codi sigui segur i mantenible. Entendre aquests modificadors t'ajudarà a dissenyar millors APIs i encapsular el teu codi de manera efectiva.
