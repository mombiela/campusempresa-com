Introducció

En aquesta secció, explorarem els conceptes de ViewModel i LiveData en Kotlin, que són components essencials dels components d'arquitectura d'Android. Aquestes eines ajuden a gestionar les dades relacionades amb la interfície d'usuari de manera conscient del cicle de vida, assegurant que les dades sobrevisquin als canvis de configuració com les rotacions de pantalla.

ViewModel

La classe ViewModel està dissenyada per emmagatzemar i gestionar les dades relacionades amb la interfície d'usuari de manera conscient del cicle de vida. Permet que les dades sobrevisquin als canvis de configuració com les rotacions de pantalla.

Conceptes Clau

  • Consciència del Cicle de Vida: Els objectes ViewModel es retenen automàticament durant els canvis de configuració.
  • Separació de Responsabilitats: Ajuda a separar les dades de la interfície d'usuari dels controladors de la interfície d'usuari com les Activitats i els Fragments.

Exemple

Creem un simple ViewModel per gestionar un comptador.

// CounterViewModel.kt
import androidx.lifecycle.ViewModel

class CounterViewModel : ViewModel() {
    var counter: Int = 0

    fun incrementCounter() {
        counter++
    }
}

A la teva Activitat o Fragment, pots utilitzar aquest ViewModel de la següent manera:

// MainActivity.kt
import android.os.Bundle
import androidx.activity.viewModels
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.ViewModelProvider
import kotlinx.android.synthetic.main.activity_main.*

class MainActivity : AppCompatActivity() {
    private val viewModel: CounterViewModel by viewModels()

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        button_increment.setOnClickListener {
            viewModel.incrementCounter()
            textView_counter.text = viewModel.counter.toString()
        }
    }
}

LiveData

LiveData és una classe contenidora de dades observable. A diferència d'un observable regular, LiveData és conscient del cicle de vida, el que significa que respecta el cicle de vida d'altres components de l'aplicació, com activitats, fragments o serveis.

Conceptes Clau

  • Consciència del Cicle de Vida: LiveData només actualitza els observadors dels components de l'aplicació que estan en un estat de cicle de vida actiu.
  • Neteja Automàtica: Els observadors es retiren automàticament quan el seu cicle de vida associat es destrueix.
  • Sense Fuites de Memòria: Els observadors estan vinculats a objectes de cicle de vida i es netegen quan el cicle de vida es destrueix.

Exemple

Modifiquem el nostre exemple anterior per utilitzar LiveData.

// CounterViewModel.kt
import androidx.lifecycle.LiveData
import androidx.lifecycle.MutableLiveData
import androidx.lifecycle.ViewModel

class CounterViewModel : ViewModel() {
    private val _counter = MutableLiveData<Int>()
    val counter: LiveData<Int> get() = _counter

    init {
        _counter.value = 0
    }

    fun incrementCounter() {
        _counter.value = (_counter.value ?: 0) + 1
    }
}

A la teva Activitat o Fragment, pots observar el LiveData de la següent manera:

// MainActivity.kt
import android.os.Bundle
import androidx.activity.viewModels
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.Observer
import kotlinx.android.synthetic.main.activity_main.*

class MainActivity : AppCompatActivity() {
    private val viewModel: CounterViewModel by viewModels()

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        viewModel.counter.observe(this, Observer { newCounterValue ->
            textView_counter.text = newCounterValue.toString()
        })

        button_increment.setOnClickListener {
            viewModel.incrementCounter()
        }
    }
}

Combinació de ViewModel i LiveData

Combinant ViewModel i LiveData proporciona una manera robusta de gestionar les dades relacionades amb la interfície d'usuari de manera conscient del cicle de vida. El ViewModel manté les dades, i LiveData assegura que la interfície d'usuari s'actualitzi sempre que les dades canviïn.

Exemple

Creem un exemple més complex on obtenim dades d'un repositori.

// UserRepository.kt
class UserRepository {
    fun getUser(): LiveData<User> {
        val user = MutableLiveData<User>()
        // Simula una trucada de xarxa o una consulta a la base de dades
        user.value = User("John Doe", 30)
        return user
    }
}

// UserViewModel.kt
import androidx.lifecycle.LiveData
import androidx.lifecycle.ViewModel

class UserViewModel : ViewModel() {
    private val userRepository = UserRepository()
    val user: LiveData<User> = userRepository.getUser()
}

// User.kt
data class User(val name: String, val age: Int)

A la teva Activitat o Fragment, observa les dades de l'usuari:

// MainActivity.kt
import android.os.Bundle
import androidx.activity.viewModels
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.Observer
import kotlinx.android.synthetic.main.activity_main.*

class MainActivity : AppCompatActivity() {
    private val viewModel: UserViewModel by viewModels()

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        viewModel.user.observe(this, Observer { user ->
            textView_name.text = user.name
            textView_age.text = user.age.toString()
        })
    }
}

Conclusió

ViewModel i LiveData són eines poderoses per gestionar les dades relacionades amb la interfície d'usuari de manera conscient del cicle de vida en aplicacions Android. Utilitzant aquests components, pots assegurar-te que les dades de la teva aplicació sobrevisquin als canvis de configuració i que la teva interfície d'usuari estigui sempre actualitzada amb les últimes dades. Aquesta separació de responsabilitats també fa que el teu codi sigui més modular i fàcil de provar.
