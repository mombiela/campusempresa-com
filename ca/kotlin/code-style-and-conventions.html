Introducció

L'estil i les convencions de codi són essencials per escriure codi net, llegible i mantenible. En Kotlin, adherir-se a aquestes convencions assegura que el teu codi sigui consistent i fàcil d'entendre per altres. Aquesta secció cobrirà els aspectes clau de l'estil i les convencions de codi en Kotlin, incloent convencions de nomenclatura, format i millors pràctiques.

Convencions de Nomenclatura

Les convencions de nomenclatura ajuden a fer el codi més llegible i comprensible. Kotlin segueix convencions de nomenclatura específiques per a diferents tipus d'identificadors.

  • Classes i Objectes: Utilitza PascalCase.

    class MyClass
object MySingleton

  • Funcions i Propietats: Utilitza camelCase.

    fun myFunction() { }
val myProperty = "value"

  • Constants: Utilitza UPPER_SNAKE_CASE.

    const val MAX_COUNT = 100

  • Variables: Utilitza camelCase.

    var myVariable = 10

Format

Un format adequat millora la llegibilitat del codi. Aquí hi ha algunes regles clau de format en Kotlin:

  • Indentació: Utilitza 4 espais per a la indentació.

    fun exampleFunction() {
    val x = 10
    if (x > 5) {
        println("x is greater than 5")
    }
}

  • Longitud de Línia: Mantingues les línies dins dels 100 caràcters.

  • Claudàtors: Utilitza claudàtors per a totes les estructures de control.

    if (condition) {
    // do something
} else {
    // do something else
}

  • Línies en Blanc: Utilitza línies en blanc per separar seccions lògiques del codi.

    fun firstFunction() {
    // code
}

fun secondFunction() {
    // code
}

Millors Pràctiques

Seguir les millors pràctiques assegura que el teu codi Kotlin no només sigui correcte, sinó també eficient i mantenible.

  • Utilitza val en lloc de var: Prefereix val per a variables de només lectura.

    val immutableValue = 10
var mutableValue = 20

  • Utilitza Cossos d'Expressió per a Funcions: Quan una funció consisteix en una sola expressió, utilitza un cos d'expressió.

    fun sum(a: Int, b: Int) = a + b

  • Utilitza Plantilles de Cadena: Utilitza plantilles de cadena en lloc de concatenació.

    val name = "John"
println("Hello, $name!")

  • Evita els Punt i Coma Innecessaris: Kotlin no requereix punt i coma al final de les declaracions.

    val x = 5
val y = 10

  • Utilitza l'Expressió when: Prefereix when en lloc de if-else per a múltiples condicions.

    when (x) {
    1 -> println("x is 1")
    2 -> println("x is 2")
    else -> println("x is neither 1 nor 2")
}

Exemple de Codi

Aquí hi ha un exemple que incorpora les convencions i millors pràctiques discutides:

class Person(val name: String, val age: Int)

fun greet(person: Person) {
    val greeting = if (person.age < 18) {
        "Hello, young ${person.name}!"
    } else {
        "Hello, Mr./Ms. ${person.name}!"
    }
    println(greeting)
}

fun main() {
    val john = Person("John", 25)
    greet(john)
}

Conclusió

Adherir-se a l'estil i les convencions de codi en Kotlin és crucial per escriure codi net, llegible i mantenible. Seguint les convencions de nomenclatura, les regles de format i les millors pràctiques descrites en aquesta secció, pots assegurar-te que el teu codi Kotlin sigui consistent i fàcil d'entendre. Això no només et beneficia a tu, sinó també a altres que puguin treballar amb el teu codi en el futur.
