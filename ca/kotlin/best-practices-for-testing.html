Introducció a les Proves en Kotlin

Les proves són una part crucial del desenvolupament de programari que assegura que el teu codi funcioni com s'espera. En Kotlin, pots utilitzar diversos marcs de proves com JUnit, TestNG i KotlinTest. Aquesta secció et guiarà a través de les millors pràctiques per escriure proves efectives en Kotlin.

Tipus de Proves

  • Proves Unitàries: Prova components o funcions individuals.
  • Proves d'Integració: Prova la interacció entre diferents components.
  • Proves de Cap a Cap: Prova tot el flux de l'aplicació.

Configuració del Teu Entorn de Proves

Per començar amb les proves en Kotlin, necessites configurar el teu entorn. Aquí tens una configuració bàsica utilitzant JUnit:

  1. Afegeix dependències al teu fitxer build.gradle.kts:

    dependencies {
    testImplementation("org.junit.jupiter:junit-jupiter-api:5.7.0")
    testRuntimeOnly("org.junit.jupiter:junit-jupiter-engine:5.7.0")
}

  2. Crea una classe de prova:

    import org.junit.jupiter.api.Assertions.*
import org.junit.jupiter.api.Test

class MyTests {
    @Test
    fun testAddition() {
        assertEquals(4, 2 + 2)
    }
}

Escriure Proves Unitàries Efectives

  • Aïlla les Proves: Assegura't que cada prova sigui independent.
  • Utilitza Assertions: Valida el resultat esperat.
  • Nomena les Proves Clarament: Utilitza noms descriptius per als mètodes de prova.

Exemple

import org.junit.jupiter.api.Assertions.*
import org.junit.jupiter.api.Test

class CalculatorTests {
    @Test
    fun `addition of two positive numbers`() {
        val result = 2 + 3
        assertEquals(5, result)
    }

    @Test
    fun `subtraction of two numbers`() {
        val result = 5 - 3
        assertEquals(2, result)
    }
}

Simulació de Dependències

Quan proves classes que tenen dependències, utilitza marcs de simulació com Mockito per simular el comportament d'aquestes dependències.

Exemple amb Mockito

  1. Afegeix la dependència de Mockito:

    dependencies {
    testImplementation("org.mockito:mockito-core:3.7.7")
}

  2. Simula una dependència:

    import org.junit.jupiter.api.Test
import org.mockito.Mockito.*

class ServiceTests {
    @Test
    fun `test service with mocked repository`() {
        val repository = mock(MyRepository::class.java)
        `when`(repository.getData()).thenReturn("Mocked Data")

        val service = MyService(repository)
        val result = service.fetchData()

        assertEquals("Mocked Data", result)
    }
}

Proves Parametritzades

JUnit 5 suporta proves parametritzades, permetent-te executar la mateixa prova amb diferents entrades.

Exemple

import org.junit.jupiter.params.ParameterizedTest
import org.junit.jupiter.params.provider.ValueSource
import org.junit.jupiter.api.Assertions.*

class ParameterizedTests {
    @ParameterizedTest
    @ValueSource(strings = ["racecar", "radar", "level"])
    fun `test palindromes`(word: String) {
        assertTrue(word == word.reversed())
    }
}

Cobertura de Proves

Assegura't que les teves proves cobreixin tant de codi com sigui possible. Eines com JaCoCo poden ajudar a mesurar la cobertura de proves.

Exemple amb JaCoCo

  1. Afegeix el plugin de JaCoCo a build.gradle.kts:

    plugins {
    id("jacoco")
}

  2. Genera l'informe de cobertura:

    ./gradlew test jacocoTestReport

Integració Contínua

Integra les teves proves en una pipeline de CI/CD per executar automàticament les proves en cada commit. Eines de CI populars inclouen Jenkins, Travis CI i GitHub Actions.

Exemple de Workflow de GitHub Actions

name: Kotlin CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up JDK 11
      uses: actions/setup-java@v1
      with:
        java-version: 11
    - name: Build with Gradle
      run: ./gradlew build
    - name: Run tests
      run: ./gradlew test

Conclusió

Les proves són una part essencial del desenvolupament en Kotlin. Seguint les millors pràctiques com aïllar les proves, utilitzar assertions, simular dependències i integrar les proves en pipelines de CI/CD, pots assegurar-te que el teu codi sigui fiable i mantenible. Recorda mantenir les teves proves clares, concises i comprensives per cobrir tants escenaris com sigui possible.
