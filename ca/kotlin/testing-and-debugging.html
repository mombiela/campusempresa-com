Introducció

Les proves i la depuració són aspectes crucials del desenvolupament de programari. Asseguren que el teu codi funcioni com s'espera i t'ajuden a identificar i solucionar problemes. En aquesta secció, cobrirem els conceptes bàsics de les proves i la depuració en Kotlin, progressant des de nivells principiants fins a avançats.

Proves Unitàries amb JUnit

Què són les Proves Unitàries?

  • Les proves unitàries impliquen provar components individuals del teu programari per assegurar-se que funcionen correctament.
  • JUnit és un marc popular per a proves unitàries en Kotlin.

Configurant JUnit

Per començar a utilitzar JUnit en el teu projecte Kotlin, has d'afegir la dependència de JUnit al teu fitxer build.gradle:

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.7.0'
}

Escriure la Teva Primera Prova

Aquí tens un exemple senzill d'una prova unitària en Kotlin:

import org.junit.jupiter.api.Assertions.assertEquals
import org.junit.jupiter.api.Test

class CalculatorTest {

    @Test
    fun testAddition() {
        val calculator = Calculator()
        val result = calculator.add(2, 3)
        assertEquals(5, result)
    }
}

class Calculator {
    fun add(a: Int, b: Int): Int {
        return a + b
    }
}
  • L'anotació @Test marca el mètode com a mètode de prova.
  • assertEquals comprova si el resultat esperat coincideix amb el resultat real.

Executant Proves

  • Pots executar proves directament des del teu IDE (per exemple, IntelliJ IDEA) o utilitzant eines de línia de comandes com Gradle.

Tècniques Avançades de Proves

Proves Parametritzades

Les proves parametritzades et permeten executar la mateixa prova amb diferents entrades.

import org.junit.jupiter.params.ParameterizedTest
import org.junit.jupiter.params.provider.CsvSource
import org.junit.jupiter.api.Assertions.assertEquals

class CalculatorTest {

    @ParameterizedTest
    @CsvSource("1, 1, 2", "2, 3, 5", "3, 5, 8")
    fun testAddition(a: Int, b: Int, expected: Int) {
        val calculator = Calculator()
        val result = calculator.add(a, b)
        assertEquals(expected, result)
    }
}

Mocking amb Mockito

Mockito és un marc que et permet crear objectes simulats per a proves.

import org.junit.jupiter.api.Test
import org.mockito.Mockito.*
import org.mockito.MockitoAnnotations

class UserServiceTest {

    @Test
    fun testGetUser() {
        MockitoAnnotations.openMocks(this)
        val userRepository = mock(UserRepository::class.java)
        val userService = UserService(userRepository)

        `when`(userRepository.findUserById(1)).thenReturn(User(1, "John Doe"))

        val user = userService.getUser(1)
        assertEquals("John Doe", user.name)
    }
}

Depuració en Kotlin

Utilitzant el Depurador de Kotlin

  • La majoria d'IDEs, com IntelliJ IDEA, vénen amb eines de depuració integrades.
  • Pots establir punts de ruptura en el teu codi per pausar l'execució i inspeccionar variables.

Tècniques Comunes de Depuració

  • Punts de Ruptura: Pausen l'execució en línies específiques.
  • Step Over: Executa la línia actual i passa a la següent.
  • Step Into: Entra en les crides de mètodes.
  • Watch Variables: Monitora els valors de les variables durant l'execució.

Exemple de Sessió de Depuració

fun main() {
    val numbers = listOf(1, 2, 3, 4, 5)
    val result = sum(numbers)
    println("Sum: $result")
}

fun sum(numbers: List<Int>): Int {
    var total = 0
    for (number in numbers) {
        total += number
    }
    return total
}
  • Estableix un punt de ruptura a total += number.
  • Executa el programa en mode de depuració.
  • Inspecciona el valor de total i number en cada iteració.

Conclusió

Les proves i la depuració són habilitats essencials per a qualsevol desenvolupador de Kotlin. Dominant les proves unitàries amb JUnit, tècniques avançades de proves amb proves parametritzades i Mockito, i estratègies efectives de depuració, pots assegurar-te que el teu codi sigui robust i fiable. Practica aquestes tècniques regularment per convertir-te en un expert en identificar i solucionar problemes en el teu codi.
