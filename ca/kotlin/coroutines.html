Introducció a les Coroutines

Les coroutines són una característica potent a Kotlin que permet escriure codi asíncron de manera seqüencial. Ajuden a gestionar tasques de llarga durada, com ara sol·licituds de xarxa o operacions de base de dades, sense bloquejar el fil principal.

Conceptes Clau

  • Funcions de Suspensió: Funcions que es poden pausar i reprendre més tard.
  • CoroutineScope: Defineix el cicle de vida de les coroutines i es pot utilitzar per llançar noves coroutines.
  • Job: Un manejador per gestionar el cicle de vida d'una coroutine.
  • Deferred: Un Job amb un resultat.

Configuració de les Coroutines

Per utilitzar coroutines a Kotlin, cal incloure la biblioteca de coroutines de Kotlin al teu projecte.

Dependència de Gradle

dependencies {
    implementation "org.jetbrains.kotlinx:kotlinx-coroutines-core:1.5.2"
}

Ús Bàsic

Llançament d'una Coroutine

La manera més senzilla de començar una coroutine és utilitzant la funció launch.

import kotlinx.coroutines.*

fun main() = runBlocking {
    launch {
        delay(1000L)
        println("Món!")
    }
    println("Hola,")
}
  • runBlocking: Crea una coroutine bloquejant que bloqueja el fil actual fins a la seva finalització.
  • launch: Comença una nova coroutine sense bloquejar el fil actual.
  • delay: Suspèn la coroutine durant un temps específic sense bloquejar el fil.

Funcions de Suspensió

Les funcions de suspensió es poden pausar i reprendre sense bloquejar el fil.

suspend fun fetchData(): String {
    delay(1000L) // Simula una tasca de llarga durada
    return "Dades obtingudes"
}

fun main() = runBlocking {
    val data = fetchData()
    println(data)
}

Scopes de Coroutine

Els scopes de coroutine ajuden a gestionar el cicle de vida de les coroutines. Els scopes comuns inclouen GlobalScope, CoroutineScope i runBlocking.

Exemple amb CoroutineScope

fun main() = runBlocking {
    val scope = CoroutineScope(Dispatchers.Default)
    scope.launch {
        delay(1000L)
        println("Coroutine en scope personalitzat")
    }
}

Concurrència Estructurada

La concurrència estructurada assegura que les coroutines es llancen de manera estructurada, facilitant la gestió del seu cicle de vida.

Exemple amb Concurrència Estructurada

fun main() = runBlocking {
    launch {
        delay(1000L)
        println("Tasca de runBlocking")
    }
    coroutineScope {
        launch {
            delay(500L)
            println("Tasca de launch niat")
        }
        delay(100L)
        println("Tasca de coroutineScope")
    }
    println("El scope de coroutine ha acabat")
}

Temes Avançats

Context de Coroutine i Dispatchers

Els dispatchers determinen el fil en el qual s'executa una coroutine.

  • Dispatchers.Default: Utilitzat per a tasques intensives de CPU.
  • Dispatchers.IO: Utilitzat per a operacions d'E/S.
  • Dispatchers.Main: Utilitzat per a operacions d'interfície d'usuari.

Exemple amb Dispatchers

fun main() = runBlocking {
    launch(Dispatchers.Default) {
        println("Executant en el dispatcher Default")
    }
    launch(Dispatchers.IO) {
        println("Executant en el dispatcher IO")
    }
    launch(Dispatchers.Main) {
        println("Executant en el dispatcher Main")
    }
}

Gestió d'Excepcions

La gestió d'excepcions en coroutines es pot fer utilitzant blocs try-catch o CoroutineExceptionHandler.

Exemple amb Gestió d'Excepcions

fun main() = runBlocking {
    val handler = CoroutineExceptionHandler { _, exception ->
        println("Capturat $exception")
    }
    val job = GlobalScope.launch(handler) {
        throw AssertionError("El meu error personalitzat")
    }
    job.join()
}

Conclusió

Les coroutines a Kotlin proporcionen una manera robusta de gestionar la programació asíncrona, fent que el codi sigui més fàcil de llegir i mantenir. Entenent els conceptes bàsics de llançament de coroutines, utilització de funcions de suspensió i gestió de scopes de coroutine, pots utilitzar efectivament les coroutines en les teves aplicacions Kotlin. Els temes avançats com els dispatchers i la gestió d'excepcions milloren encara més la teva capacitat per escriure codi asíncron eficient i segur.
