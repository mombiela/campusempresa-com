Introducció a les Proves Unitàries

Les proves unitàries són un mètode de prova de programari on es proven unitats o components individuals d'un programari. L'objectiu és validar que cada unitat del programari funcioni com s'espera. En Kotlin, les proves unitàries es fan típicament utilitzant el framework JUnit.

Conceptes Clau

  • Prova Unitària: Una prova que comprova una sola "unitat" de codi, normalment una funció o mètode.
  • Cas de Prova: Un conjunt de condicions o variables sota les quals un provador determina si una unitat de codi funciona com s'espera.
  • Suite de Proves: Una col·lecció de casos de prova.
  • Mocking: Crear una versió falsa d'un objecte per provar les interaccions entre objectes.

Configuració de JUnit en Kotlin

Per començar a fer proves unitàries en Kotlin, cal configurar JUnit al teu projecte. Això es pot fer afegint les dependències necessàries al teu fitxer build.gradle.

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter-api:5.7.0'
    testRuntimeOnly 'org.junit.jupiter:junit-jupiter-engine:5.7.0'
}

Escriure la Teva Primera Prova Unitària

Escrivim una prova unitària simple per a una funció de Kotlin. Suposem que tenim una funció que suma dos números:

fun add(a: Int, b: Int): Int {
    return a + b
}

Per provar aquesta funció, podem escriure una prova unitària utilitzant JUnit:

import org.junit.jupiter.api.Assertions.assertEquals
import org.junit.jupiter.api.Test

class MathTests {

    @Test
    fun testAdd() {
        val result = add(2, 3)
        assertEquals(5, result)
    }
}

Explicació

  • @Test: Aquesta anotació indica a JUnit que la funció següent és un cas de prova.
  • assertEquals: Aquesta funció comprova si el valor esperat (5) és igual al valor real (result).

Tècniques Avançades de Proves Unitàries

A mesura que et sentis més còmode amb les proves unitàries bàsiques, pots explorar tècniques més avançades com el mocking i les proves parametritzades.

Mocking amb Mockito

Mockito és una biblioteca popular per crear objectes mock en proves unitàries. Per utilitzar Mockito, afegeix la següent dependència al teu fitxer build.gradle:

testImplementation 'org.mockito:mockito-core:3.7.7'

Aquí tens un exemple de com utilitzar Mockito per fer mock d'una dependència:

import org.junit.jupiter.api.Test
import org.mockito.Mockito.*

class UserServiceTest {

    @Test
    fun testGetUser() {
        val mockRepository = mock(UserRepository::class.java)
        `when`(mockRepository.getUser(1)).thenReturn(User(1, "John Doe"))

        val userService = UserService(mockRepository)
        val user = userService.getUser(1)

        assertEquals("John Doe", user.name)
    }
}

Proves Parametritzades

Les proves parametritzades permeten executar la mateixa prova amb diferents entrades. JUnit 5 proporciona suport per a proves parametritzades.

import org.junit.jupiter.params.ParameterizedTest
import org.junit.jupiter.params.provider.CsvSource
import org.junit.jupiter.api.Assertions.assertEquals

class MathTests {

    @ParameterizedTest
    @CsvSource("1, 1, 2", "2, 3, 5", "3, 5, 8")
    fun testAdd(a: Int, b: Int, expected: Int) {
        assertEquals(expected, add(a, b))
    }
}

Millors Pràctiques per a les Proves Unitàries

  • Escriu Proves Independents: Cada prova ha de ser independent de les altres.
  • Utilitza Noms de Proves Significatius: Els noms de les proves han de descriure clarament què estan provant.
  • Prova Casos Límit: Assegura't de provar casos límit i no només el "camí feliç".
  • Mantingues les Proves Ràpides: Les proves unitàries han de ser ràpides d'executar.

Conclusió

Les proves unitàries són una part crucial del desenvolupament de programari que assegura que el teu codi funcioni com s'espera. Utilitzant JUnit amb Kotlin, pots escriure proves unitàries efectives i eficients. Comença amb proves simples i incorpora gradualment tècniques avançades com el mocking i les proves parametritzades. Seguir les millors pràctiques t'ajudarà a mantenir una base de codi robusta i fiable.
