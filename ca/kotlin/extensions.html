Introducció a les Extensions

Les extensions en Kotlin proporcionen una manera d'ampliar la funcionalitat de les classes sense modificar el seu codi font. Això és especialment útil quan vols afegir funcions d'utilitat a classes existents.

Conceptes Clau

  • Funcions d'Extensió: Funcions que proporcionen funcionalitat addicional a classes existents.
  • Propietats d'Extensió: Propietats que afegeixen noves propietats a classes existents.

Funcions d'Extensió

Les funcions d'extensió permeten afegir noves funcions a classes existents. Aquestes funcions es defineixen fora de la classe però es poden cridar com si fossin part de la classe.

Sintaxi

La sintaxi per definir una funció d'extensió és:

fun NomClasse.nomFuncio(paràmetres): TipusRetorn {
    // cos de la funció
}

Exemple

Afegim una funció a la classe String que retorna l'últim caràcter de la cadena:

fun String.lastChar(): Char {
    return this[this.length - 1]
}

fun main() {
    val str = "Kotlin"
    println(str.lastChar())  // Sortida: n
}

En aquest exemple, lastChar és una funció d'extensió per a la classe String.

Explicació

  • fun String.lastChar(): Declara una funció d'extensió per a la classe String.
  • this[this.length - 1]: Utilitza this per referir-se a la instància de la classe sobre la qual es crida la funció.

Propietats d'Extensió

Les propietats d'extensió permeten afegir noves propietats a classes existents. Aquestes propietats es defineixen fora de la classe però es poden accedir com si fossin part de la classe.

Sintaxi

La sintaxi per definir una propietat d'extensió és:

val NomClasse.nomPropietat: TipusPropietat
    get() = // cos del getter

Exemple

Afegim una propietat a la classe String que retorna el primer caràcter de la cadena:

val String.firstChar: Char
    get() = this[0]

fun main() {
    val str = "Kotlin"
    println(str.firstChar)  // Sortida: K
}

En aquest exemple, firstChar és una propietat d'extensió per a la classe String.

Explicació

  • val String.firstChar: Declara una propietat d'extensió per a la classe String.
  • get() = this[0]: Defineix el getter per a la propietat, utilitzant this per referir-se a la instància de la classe.

Limitacions de les Extensions

Tot i que les extensions són poderoses, tenen algunes limitacions:

  • No Accés a Membres Privats: Les extensions no poden accedir a membres privats o protegits de la classe.
  • Resolució Estàtica: Les extensions es resolen de manera estàtica, és a dir, la funció o propietat d'extensió es determina pel tipus de l'expressió en temps de compilació, no en temps d'execució.

Exemple

Considera el següent codi:

open class Shape

class Circle : Shape()

fun Shape.name() = "Shape"
fun Circle.name() = "Circle"

fun printName(shape: Shape) {
    println(shape.name())
}

fun main() {
    val shape: Shape = Circle()
    printName(shape)  // Sortida: Shape
}

En aquest exemple, tot i que shape és una instància de Circle, es crida la funció name per a Shape perquè les extensions es resolen de manera estàtica.

Conclusió

Les extensions en Kotlin proporcionen una manera poderosa d'afegir funcionalitat a classes existents sense modificar el seu codi font. Es poden utilitzar per afegir tant funcions com propietats. Tanmateix, és important ser conscient de les seves limitacions, com la incapacitat d'accedir a membres privats i la resolució estàtica. Entenent i utilitzant les extensions de manera efectiva, pots escriure codi Kotlin més concís i llegible.
