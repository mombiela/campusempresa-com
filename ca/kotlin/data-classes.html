Introducció a les Classes de Dades

Les classes de dades a Kotlin són una manera concisa de crear classes que s'utilitzen principalment per contenir dades. Proporcionen automàticament diversos mètodes útils, com equals(), hashCode(), toString(), i copy(), cosa que les fa molt convenients per gestionar dades.

Conceptes Clau

  • Constructor Primari: Les classes de dades han de tenir almenys un paràmetre de constructor primari.
  • Propietats: Tots els paràmetres del constructor primari han d'estar marcats com val o var.
  • Mètodes Generats: Kotlin genera automàticament els mètodes equals(), hashCode(), toString(), copy(), i componentN() per a les classes de dades.
  • Immutabilitat: Per defecte, les classes de dades són immutables si utilitzes val per a les propietats.

Creació d'una Classe de Dades

Per definir una classe de dades, utilitza la paraula clau data abans de la declaració de la classe.

data class User(val name: String, val age: Int)

Exemple Explicat

  • data: Paraula clau que indica que aquesta és una classe de dades.
  • User: Nom de la classe de dades.
  • val name: String: Propietat immutable name de tipus String.
  • val age: Int: Propietat immutable age de tipus Int.

Mètodes Generats

`toString()`

Proporciona una representació en cadena de l'objecte.

val user = User("Alice", 30)
println(user) // Sortida: User(name=Alice, age=30)

`equals()` i `hashCode()`

S'utilitzen per comparar objectes i per a hashing.

val user1 = User("Alice", 30)
val user2 = User("Alice", 30)
println(user1 == user2) // Sortida: true
println(user1.hashCode() == user2.hashCode()) // Sortida: true

`copy()`

Crea una còpia de l'objecte, modificant opcionalment algunes propietats.

val user1 = User("Alice", 30)
val user2 = user1.copy(age = 31)
println(user2) // Sortida: User(name=Alice, age=31)

`componentN()`

Permet declaracions de desestructuració.

val user = User("Alice", 30)
val (name, age) = user
println("Name: $name, Age: $age") // Sortida: Name: Alice, Age: 30

Funcionalitats Avançades

Valors Per Defecte

Pots proporcionar valors per defecte per a les propietats en el constructor primari.

data class User(val name: String = "Unknown", val age: Int = 0)
val user = User()
println(user) // Sortida: User(name=Unknown, age=0)

Herència de Classes de Dades

Les classes de dades no poden ser abstractes, obertes, segellades o internes. No obstant això, poden implementar interfícies.

interface Printable {
    fun print()
}

data class User(val name: String, val age: Int) : Printable {
    override fun print() {
        println("User(name=$name, age=$age)")
    }
}

Copiar amb Classes de Dades Imbricades

Quan es copien classes de dades imbricades, només l'objecte de nivell superior es copia. Els objectes imbricats no es copien profundament.

data class Address(val city: String, val country: String)
data class User(val name: String, val age: Int, val address: Address)

val address = Address("New York", "USA")
val user1 = User("Alice", 30, address)
val user2 = user1.copy()

println(user1.address === user2.address) // Sortida: true (mateixa referència)

Conclusió

Les classes de dades a Kotlin proporcionen una manera potent i concisa de gestionar classes centrades en dades. Generen automàticament mètodes útils, reduint el codi redundant i millorant la llegibilitat. Entendre i utilitzar les classes de dades de manera efectiva pot millorar significativament les teves habilitats de programació en Kotlin, especialment quan es tracta d'aplicacions amb moltes dades.
