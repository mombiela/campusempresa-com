Introducció a l'API de Xarxes de Unity

L'API de Xarxes de Unity permet als desenvolupadors crear jocs multijugador gestionant objectes en xarxa, connexions de jugadors i sincronització de dades. Aquesta secció introduirà els conceptes bàsics i els components de l'API de Xarxes de Unity.

  • NetworkManager: Gestiona l'estat de la xarxa, incloent l'inici i l'aturada de servidors i clients.
  • NetworkIdentity: Identifica objectes en xarxa i assegura que estiguin sincronitzats entre clients.
  • NetworkBehaviour: Classe base per a scripts en xarxa, proporcionant ganxos per a esdeveniments de xarxa.
  • NetworkTransform: Sincronitza la posició, rotació i escala dels objectes en xarxa.

Configuració d'un Joc Bàsic en Xarxa

Creació d'un NetworkManager

El NetworkManager és el component principal per gestionar connexions de xarxa. Aquí tens com configurar-lo:

  1. Afegeix el Component NetworkManager:
    • Crea un GameObject buit a la teva escena.
    • Afegeix el component NetworkManager al GameObject.
    • Opcionalment, afegeix el component NetworkManagerHUD per a una interfície d'usuari simple per controlar l'estat de la xarxa.
using UnityEngine;
using UnityEngine.Networking;

public class MyNetworkManager : NetworkManager
{
    // Es pot afegir lògica personalitzada per a la gestió de la xarxa aquí
}
  1. Configura el NetworkManager:
    • Configura el Player Prefab a l'objecte jugador que vols generar per a cada client.
    • Configura l'Network Address i el Network Port per al servidor.

Generació d'Objectes en Xarxa

Els objectes en xarxa es generen i es sincronitzen entre tots els clients. Utilitza el component NetworkIdentity per marcar objectes per a la xarxa.

  1. Afegeix el Component NetworkIdentity:
    • Adjunta el component NetworkIdentity al GameObject que vols posar en xarxa.
    • Assegura't que Local Player Authority estigui marcat si l'objecte ha de ser controlat pel client.
using UnityEngine;
using UnityEngine.Networking;

public class PlayerController : NetworkBehaviour
{
    void Update()
    {
        if (!isLocalPlayer)
            return;

        // Lògica de control del jugador aquí
    }
}
  1. Generació de l'Objecte:
    • Utilitza NetworkServer.Spawn per generar objectes al servidor i sincronitzar-los amb els clients.
public class GameManager : NetworkBehaviour
{
    public GameObject playerPrefab;

    public override void OnStartServer()
    {
        GameObject player = Instantiate(playerPrefab);
        NetworkServer.Spawn(player);
    }
}

Gestió de Connexions de Jugadors

Gestionar connexions de jugadors implica gestionar esdeveniments quan els jugadors s'uneixen o abandonen el joc.

  • OnServerConnect: Es crida quan un client es connecta al servidor.
  • OnServerDisconnect: Es crida quan un client es desconnecta del servidor.
  • OnClientConnect: Es crida quan el client es connecta al servidor.
  • OnClientDisconnect: Es crida quan el client es desconnecta del servidor.
public class MyNetworkManager : NetworkManager
{
    public override void OnServerConnect(NetworkConnection conn)
    {
        Debug.Log("Client connectat: " + conn.address);
    }

    public override void OnServerDisconnect(NetworkConnection conn)
    {
        Debug.Log("Client desconnectat: " + conn.address);
        base.OnServerDisconnect(conn);
    }

    public override void OnClientConnect(NetworkConnection conn)
    {
        Debug.Log("Connectat al servidor");
    }

    public override void OnClientDisconnect(NetworkConnection conn)
    {
        Debug.Log("Desconnectat del servidor");
        base.OnClientDisconnect(conn);
    }
}

Sincronització de Dades

La sincronització de dades assegura que tots els clients tinguin el mateix estat per als objectes en xarxa.

SyncVars

Les SyncVars són variables que es sincronitzen automàticament del servidor a tots els clients.

public class Player : NetworkBehaviour
{
    [SyncVar]
    public int health;

    void TakeDamage(int amount)
    {
        if (!isServer)
            return;

        health -= amount;
    }
}

Comandes i RPCs

Les comandes i RPCs s'utilitzen per enviar missatges entre clients i el servidor.

  • Comandes: Mètodes cridats pels clients però executats al servidor.
  • ClientRPCs: Mètodes cridats pel servidor però executats als clients.
public class Player : NetworkBehaviour
{
    [Command]
    void CmdTakeDamage(int amount)
    {
        health -= amount;
    }

    [ClientRpc]
    void RpcUpdateHealth(int newHealth)
    {
        health = newHealth;
    }
}

Conclusió

L'API de Xarxes de Unity proporciona un marc robust per crear jocs multijugador. Entenent i utilitzant components com NetworkManager, NetworkIdentity i NetworkBehaviour, els desenvolupadors poden gestionar connexions de jugadors, sincronitzar dades i gestionar objectes en xarxa de manera efectiva. La pràctica i l'experimentació amb aquests conceptes ajudaran a solidificar la teva comprensió i capacitat per crear experiències multijugador atractives.
