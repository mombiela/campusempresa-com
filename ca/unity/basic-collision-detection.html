La detecció de col·lisions és un concepte fonamental en el desenvolupament de jocs, permetent que els objectes interactuïn entre ells en un entorn virtual. A Unity, la detecció de col·lisions es maneja a través de motors de física i col·liders. Aquesta secció et guiarà a través dels conceptes bàsics de la detecció de col·lisions, des d'entendre els col·liders fins a implementar respostes simples a les col·lisions.

Entenent els col·liders

Els col·liders són components que defineixen la forma d'un objecte amb finalitats de col·lisions físiques. No necessiten coincidir exactament amb l'aparença visual de l'objecte, però haurien d'aproximar la seva forma.

  • Tipus de col·liders:
    • Box Collider: Una forma de caixa rectangular.
    • Sphere Collider: Una forma esfèrica.
    • Capsule Collider: Una forma de càpsula.
    • Mesh Collider: Un col·lider que utilitza la malla de l'objecte per a la detecció de col·lisions.
    • 2D Colliders: Similar als col·liders 3D però utilitzats per a jocs 2D (per exemple, Box Collider 2D, Circle Collider 2D).

Exemple: Afegint un Box Collider

Per afegir un Box Collider a un GameObject a Unity:

  1. Selecciona el GameObject a la Jerarquia.
  2. A la finestra de l'Inspector, fes clic a "Add Component."
  3. Escriu "Box Collider" i selecciona'l de la llista.
// Exemple d'afegir un Box Collider via script
void Start() {
    gameObject.AddComponent<BoxCollider>();
}

Component Rigidbody

Perquè la detecció de col·lisions funcioni efectivament, almenys un dels objectes que col·lideixen hauria de tenir un component Rigidbody. El component Rigidbody fa que el GameObject estigui subjecte a la física, com la gravetat i les col·lisions.

  • Afegint un Rigidbody:
    • Selecciona el GameObject.
    • Fes clic a "Add Component" a l'Inspector.
    • Escriu "Rigidbody" i selecciona'l.
// Exemple d'afegir un Rigidbody via script
void Start() {
    gameObject.AddComponent<Rigidbody>();
}

Detectant col·lisions

Unity proporciona diverses maneres de detectar col·lisions i desencadenar respostes. Les maneres més comunes són OnCollisionEnter, OnCollisionStay i OnCollisionExit.

  • OnCollisionEnter: Es crida quan el col·lider/rigidbody ha començat a tocar un altre rigidbody/col·lider.
  • OnCollisionStay: Es crida una vegada per fotograma per cada col·lider/rigidbody que està tocant un altre rigidbody/col·lider.
  • OnCollisionExit: Es crida quan el col·lider/rigidbody ha deixat de tocar un altre rigidbody/col·lider.

Exemple: Utilitzant OnCollisionEnter

void OnCollisionEnter(Collision collision) {
    // Comprova si l'objecte col·lisionat té una etiqueta específica
    if (collision.gameObject.tag == "Enemy") {
        Debug.Log("Col·lisionat amb un enemic!");
    }
}

Col·liders de Triggers

Els col·liders de triggers s'utilitzen per detectar superposicions sense aplicar forces físiques. Són útils per detectar quan un objecte entra en una àrea específica, com un punt de control o un objecte per recollir.

  • Configurant un col·lider com a Trigger:
    • Selecciona el GameObject amb el col·lider.
    • Al component del col·lider, marca la casella "Is Trigger."

Exemple: Utilitzant OnTriggerEnter

void OnTriggerEnter(Collider other) {
    // Comprova si l'objecte que activa el trigger té una etiqueta específica
    if (other.gameObject.tag == "Pickup") {
        Debug.Log("Objecte recollit!");
        // Realitza una lògica addicional, com augmentar la puntuació
    }
}

Resum

En aquesta secció, hem cobert els conceptes bàsics de la detecció de col·lisions a Unity, incloent:

  • Entendre i afegir diferents tipus de col·liders.
  • La importància del component Rigidbody per a les interaccions físiques.
  • Detectar col·lisions utilitzant OnCollisionEnter, OnCollisionStay i OnCollisionExit.
  • Utilitzar col·liders de triggers per a interaccions no físiques amb OnTriggerEnter.

En dominar aquests conceptes, podràs crear jocs més interactius i dinàmics. Practica afegint col·liders i components Rigidbody als teus GameObjects i implementant la lògica de detecció de col·lisions als teus scripts.
