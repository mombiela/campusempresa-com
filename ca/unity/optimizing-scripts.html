Introducció a l'Optimització de Scripts

Optimitzar scripts a Unity és crucial per millorar el rendiment i l'eficiència del teu joc. Aquesta secció cobrirà els conceptes bàsics de l'optimització de scripts, incloent per què és important i els principis generals que has de seguir.

  • Per què Optimitzar?

    • Millora el rendiment del joc.
    • Redueix el retard i les caigudes de fotogrames.
    • Millora l'experiència del jugador.
    • Fa que el joc funcioni sense problemes en una gamma més àmplia de dispositius.

  • Principis Generals

    • Evita càlculs innecessaris.
    • Utilitza estructures de dades eficients.
    • Minimitza l'ús de memòria.
    • Optimitza els càlculs de renderització i física.

Perfilat i Identificació de Colls d'Ampolla

Abans d'optimitzar, cal identificar les parts del teu codi que estan causant problemes de rendiment.

  • Unity Profiler
    • Utilitza el Unity Profiler per monitoritzar la CPU, GPU, memòria i altres mètriques de rendiment.
    • Identifica quins scripts i funcions estan consumint més recursos.
using UnityEngine;

public class Example : MonoBehaviour
{
    void Update()
    {
        // Aquest és un exemple senzill per mostrar com Update pot ser un coll d'ampolla
        for (int i = 0; i < 1000; i++)
        {
            Debug.Log("Això és un coll d'ampolla de rendiment!");
        }
    }
}
  • Perfilat Profund
    • Activa el perfilat profund per obtenir informació detallada sobre les crides a funcions.
    • Tingues en compte que el perfilat profund pot alentir significativament el teu joc.

Optimització de Bucles Update

El mètode Update es crida una vegada per fotograma, convertint-lo en una font comuna de problemes de rendiment.

  • Evita Càlculs Pesats a Update
    • Mou els càlculs pesats a FixedUpdate o LateUpdate si no necessiten executar-se cada fotograma.
    • Utilitza coroutines per a tasques que es poden repartir en múltiples fotogrames.
using UnityEngine;

public class OptimizedExample : MonoBehaviour
{
    void Start()
    {
        StartCoroutine(HeavyTask());
    }

    IEnumerator HeavyTask()
    {
        for (int i = 0; i < 1000; i++)
        {
            Debug.Log("Això està repartit en múltiples fotogrames!");
            yield return null; // Espera al següent fotograma
        }
    }
}
  • Utilitza Pooling d'Objectes
    • Reutilitza objectes en lloc de crear-los i destruir-los freqüentment.
    • Redueix la sobrecàrrega de la recollida d'escombraries.
using UnityEngine;
using System.Collections.Generic;

public class ObjectPool : MonoBehaviour
{
    public GameObject prefab;
    private List<GameObject> pool = new List<GameObject>();

    public GameObject GetObject()
    {
        foreach (var obj in pool)
        {
            if (!obj.activeInHierarchy)
            {
                obj.SetActive(true);
                return obj;
            }
        }

        GameObject newObj = Instantiate(prefab);
        pool.Add(newObj);
        return newObj;
    }

    public void ReturnObject(GameObject obj)
    {
        obj.SetActive(false);
    }
}

Gestió de Memòria

Una gestió eficient de la memòria és clau per prevenir problemes de rendiment relacionats amb la recollida d'escombraries.

  • Evita Assignacions Freqüents
    • Utilitza StringBuilder per a concatenacions de cadenes.
    • Emmagatzema en memòria cau les referències a components i objectes.
using UnityEngine;
using System.Text;

public class MemoryExample : MonoBehaviour
{
    void Start()
    {
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < 1000; i++)
        {
            sb.Append("Això és eficient!");
        }
        Debug.Log(sb.ToString());
    }
}
  • Utilitza Structs en Lloc de Classes
    • Els structs són tipus de valor i poden ser més eficients que les classes, que són tipus de referència.
public struct Position
{
    public float x, y, z;
}

Optimització de Càlculs de Física

Els càlculs de física poden ser una càrrega significativa de rendiment si no es gestionen adequadament.

  • Utilitza Capes i Matriu de Col·lisions

    • Exclou comprovacions de col·lisió innecessàries configurant la matriu de col·lisions.

  • Optimitza la Configuració de Rigidbody

    • Utilitza Rigidbody2D per a jocs 2D i Rigidbody per a jocs 3D.
    • Ajusta la interpolació i la detecció de col·lisions segons les teves necessitats.
using UnityEngine;

public class PhysicsExample : MonoBehaviour
{
    void Start()
    {
        Rigidbody rb = GetComponent<Rigidbody>();
        rb.interpolation = RigidbodyInterpolation.Interpolate;
        rb.collisionDetectionMode = CollisionDetectionMode.Continuous;
    }
}

Conclusió

Optimitzar scripts a Unity implica una combinació de perfilat, pràctiques de codificació eficients i comprensió dels sistemes subjacents com la gestió de memòria i els càlculs de física. Seguint els principis i tècniques descrits en aquesta secció, pots millorar significativament el rendiment dels teus projectes Unity.

  • Punts Clau
    • Sempre perfila abans d'optimitzar.
    • Evita càlculs pesats a Update.
    • Utilitza pooling d'objectes per gestionar objectes utilitzats freqüentment.
    • Gestiona eficientment la memòria per reduir la sobrecàrrega de la recollida d'escombraries.
    • Optimitza els càlculs de física configurant capes i ajustant la configuració de rigidbody.

Implementant aquestes estratègies, estaràs en bon camí per crear jocs Unity fluids i eficients.
