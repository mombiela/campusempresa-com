Introducció

El perfilatge i l'optimització són crucials per desenvolupar jocs eficients i que funcionin sense problemes a Unity. Aquesta secció et guiarà a través dels conceptes bàsics del perfilatge, la identificació de colls d'ampolla de rendiment i l'aplicació de tècniques d'optimització per millorar el rendiment del teu joc.

Conceptes Bàsics del Perfilatge

El perfilatge és el procés d'analitzar el rendiment del teu joc per identificar àrees que necessiten optimització.

  • Finestra del Profiler: Eina integrada de Unity per a l'anàlisi de rendiment en temps real.
  • Depurador de Fotogrames: T'ajuda a entendre com es renderitza cada fotograma.
  • Perfilatge Profund: Proporciona informació detallada però pot alentir el joc.

Ús de la Finestra del Profiler

La Finestra del Profiler és la teva eina principal per identificar problemes de rendiment.

  1. Obre la Finestra del Profiler: Window > Analysis > Profiler
  2. Selecciona el Dispositiu Objectiu: Tria si vols perfilar l'Editor, un dispositiu connectat o una build.
  3. Grava Dades: Prem el botó de gravar per començar a capturar dades de rendiment.
  4. Analitza les Dades: Mira diferents mòduls com CPU, GPU, Memòria, etc.
// Exemple: Script senzill per simular un problema de rendiment
void Update() {
    for (int i = 0; i < 1000000; i++) {
        // Simula un càlcul pesat
        float x = Mathf.Sqrt(i);
    }
}

Identificació de Colls d'Ampolla

Un cop tinguis dades de perfilatge, el següent pas és identificar colls d'ampolla.

  • Ús de la CPU: Un ús elevat de la CPU pot indicar codi ineficient o massa objectes actius.
  • Ús de la GPU: Un ús elevat de la GPU pot ser degut a shaders complexos o massa crides de dibuix.
  • Ús de la Memòria: Un ús excessiu de la memòria pot provocar pics de recollida d'escombraries.

Colls d'Ampolla Comuns

  • Execució de Scripts: Algoritmes ineficients o càlculs innecessaris.
  • Renderització: Massa crides de dibuix, shaders complexos o textures d'alta resolució.
  • Física: Massa càlculs de física o malles de col·lisió complexes.

Tècniques d'Optimització

Després d'identificar els colls d'ampolla, aplica tècniques d'optimització per millorar el rendiment.

Optimització de Scripts

  • Evita Operacions Costoses: Minimitza l'ús d'operacions com Find, GetComponent i Instantiate a Update.
  • Pooling d'Objectes: Reutilitza objectes en lloc de crear-los i destruir-los freqüentment.
  • Algoritmes Eficients: Utilitza estructures de dades i algoritmes més eficients.
// Exemple: Pooling d'Objectes
public class ObjectPool : MonoBehaviour {
    public GameObject prefab;
    private Queue<GameObject> pool = new Queue<GameObject>();

    public GameObject GetObject() {
        if (pool.Count > 0) {
            GameObject obj = pool.Dequeue();
            obj.SetActive(true);
            return obj;
        } else {
            return Instantiate(prefab);
        }
    }

    public void ReturnObject(GameObject obj) {
        obj.SetActive(false);
        pool.Enqueue(obj);
    }
}

Optimització de la Renderització

  • Batching: Combina múltiples malles en una sola malla per reduir les crides de dibuix.
  • Nivell de Detall (LOD): Utilitza models més senzills per a objectes que estan lluny de la càmera.
  • Occlusion Culling: Desactiva la renderització per a objectes no visibles per la càmera.
// Exemple: Activació del Batching Estàtic
void Start() {
    GameObject[] objects = FindObjectsOfType<GameObject>();
    foreach (GameObject obj in objects) {
        obj.isStatic = true; // Marca els objectes com a estàtics per al batching
    }
    StaticBatchingUtility.Combine(objects, this.gameObject);
}

Optimització de la Física

  • Simplifica els Colliders: Utilitza formes de collider més senzilles com caixes i esferes.
  • Col·lisió Basada en Capes: Utilitza capes de col·lisió per reduir el nombre de comprovacions de col·lisió.
  • Fixed Timestep: Ajusta el fixed timestep per equilibrar la precisió de la física i el rendiment.
// Exemple: Col·lisió Basada en Capes
void Start() {
    Physics.IgnoreLayerCollision(LayerMask.NameToLayer("Player"), LayerMask.NameToLayer("Enemy"), true);
}

Conclusió

El perfilatge i l'optimització són habilitats essencials per a qualsevol desenvolupador de Unity. Utilitzant la Finestra del Profiler per identificar colls d'ampolla i aplicant diverses tècniques d'optimització, pots millorar significativament el rendiment del teu joc. Recorda perfilar regularment i optimitzar iterativament per assegurar una experiència suau i agradable per als teus jugadors.
