Introducció

Les proves i la depuració són habilitats crucials per a qualsevol desenvolupador de Unity. Aquesta secció et guiarà a través dels conceptes bàsics de les proves i la depuració a Unity, progressant cap a tècniques més avançades. Al final d'aquesta secció, estaràs equipat amb el coneixement per identificar, arreglar i prevenir errors en els teus projectes de Unity.

Proves Unitàries a Unity

Les proves unitàries impliquen provar components individuals del teu joc per assegurar-se que funcionen com s'espera.

Conceptes Clau

  • Proves Unitàries: Proves que es concentren en components o funcions individuals.
  • Test Runner: Eina integrada de Unity per executar proves.
  • Afirmacions: Declaracions que comproven si una condició és certa.

Configuració de Proves Unitàries

  1. Instal·la el Marc de Proves:

    • Ves a Window > Package Manager.
    • Cerca Test Framework i instal·la'l.

  2. Crea un Script de Prova:

    • Fes clic dret a la finestra del Projecte i selecciona Create > Testing > Test Assembly Folder.
    • Dins de la carpeta, crea un nou script C# per a les teves proves.

Exemple: Prova Unitària Bàsica

using NUnit.Framework;

public class PlayerTests
{
    [Test]
    public void PlayerHealth_ShouldBe100_WhenGameStarts()
    {
        // Arrange
        var player = new Player();

        // Act
        var health = player.Health;

        // Assert
        Assert.AreEqual(100, health);
    }
}
  • Explicació:
    • Arrange: Configura les condicions per a la prova.
    • Act: Realitza l'acció que vols provar.
    • Assert: Verifica el resultat.

Depuració a Unity

La depuració és el procés d'identificar i arreglar errors en el teu codi.

Conceptes Clau

  • Punts de Trencament: Pausa l'execució del teu codi en una línia específica.
  • Debug.Log: Imprimeix missatges a la Consola.
  • Pila de Trucades: Mostra la seqüència de trucades de funcions que han portat al punt actual en el codi.

Ús de Debug.Log

void Update()
{
    if (Input.GetKeyDown(KeyCode.Space))
    {
        Debug.Log("S'ha premut la tecla espai");
    }
}
  • Explicació: Aquest codi registra un missatge a la Consola cada vegada que es prem la tecla espai.

Configuració de Punts de Trencament

  1. Obre l'Script a Visual Studio:
    • Fes doble clic a l'script a Unity per obrir-lo a Visual Studio.
  2. Configura un Punt de Trencament:
    • Fes clic al marge al costat del número de línia on vols pausar l'execució.
  3. Adjunta a Unity:
    • A Visual Studio, ves a Debug > Attach Unity Debugger.
  4. Executa el Joc:
    • Quan el joc arribi al punt de trencament, es pausarà, permetent-te inspeccionar variables i la pila de trucades.

Tècniques Avançades de Proves

Proves d'Integració

Les proves d'integració comproven si diferents parts del teu joc funcionen juntes com s'espera.

Exemple: Prova d'Integració

using UnityEngine;
using UnityEngine.TestTools;
using NUnit.Framework;
using System.Collections;

public class GameIntegrationTests
{
    [UnityTest]
    public IEnumerator Player_ShouldLoseHealth_WhenHitByEnemy()
    {
        // Arrange
        var player = new GameObject().AddComponent<Player>();
        var enemy = new GameObject().AddComponent<Enemy>();

        // Act
        enemy.Attack(player);
        yield return null; // Espera un fotograma

        // Assert
        Assert.Less(player.Health, 100);
    }
}
  • Explicació:
    • UnityTest: Permet provar durant múltiples fotogrames.
    • yield return null: Espera fins al següent fotograma.

Conclusió

Les proves i la depuració són pràctiques essencials per desenvolupar jocs de Unity robustos i fiables. En dominar les proves unitàries, les proves d'integració i les tècniques de depuració, pots assegurar-te que el teu joc funcioni sense problemes i proporcioni una gran experiència als jugadors. Recorda:

  • Escriu proves per a parts crítiques del teu joc.
  • Utilitza eines de depuració per identificar i arreglar problemes de manera eficient.
  • Prova i depura contínuament mentre desenvolupes el teu joc per detectar problemes aviat.
