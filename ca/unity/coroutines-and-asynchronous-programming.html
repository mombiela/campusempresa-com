Coroutines i programació asíncrona a Unity
Objectes Scriptables ►

Introducció

En aquesta secció, explorarem les coroutines i la programació asíncrona a Unity. Aquests conceptes són crucials per gestionar tasques que requereixen temps per completar-se, com ara carregar recursos, esperar l'entrada de l'usuari o gestionar sol·licituds de xarxa. Al final d'aquest tema, entendràs com implementar i utilitzar coroutines i mètodes asíncrons per crear aplicacions Unity més eficients i responsives.

Què són les Coroutines?

Les coroutines són mètodes especials a Unity que et permeten pausar l'execució i reprendre-la més tard. Això és particularment útil per a tasques que necessiten ser distribuïdes en múltiples fotogrames.

Conceptes Clau

  • Instruccions Yield: Comandes que indiquen a la coroutine quan pausar i reprendre.
  • IEnumerator: El tipus de retorn de les coroutines.
  • StartCoroutine(): Mètode per iniciar una coroutine.
  • StopCoroutine(): Mètode per aturar una coroutine.

Exemple Bàsic de Coroutine

using UnityEngine;

public class CoroutineExample : MonoBehaviour
{
    void Start()
    {
        // Iniciar la coroutine
        StartCoroutine(PrintNumbers());
    }

    IEnumerator PrintNumbers()
    {
        for (int i = 0; i < 5; i++)
        {
            Debug.Log(i);
            // Pausar l'execució durant 1 segon
            yield return new WaitForSeconds(1);
        }
    }
}

En aquest exemple, la coroutine PrintNumbers imprimeix números del 0 al 4, pausant durant un segon entre cada impressió.

Instruccions Yield

Les instruccions yield s'utilitzen dins de les coroutines per especificar quan la coroutine ha de pausar i reprendre.

Instruccions Yield Comunes

  • WaitForSeconds(float seconds): Pausa la coroutine durant un nombre especificat de segons.
  • WaitForEndOfFrame(): Pausa la coroutine fins al final del fotograma.
  • WaitForFixedUpdate(): Pausa la coroutine fins al següent cicle d'actualització fixa.
  • WaitUntil(Func<bool> predicate): Pausa la coroutine fins que la condició especificada sigui certa.
  • WaitWhile(Func<bool> predicate): Pausa la coroutine mentre la condició especificada sigui certa.

Exemple d'Instrucció Yield

using UnityEngine;

public class YieldExample : MonoBehaviour
{
    void Start()
    {
        StartCoroutine(WaitForCondition());
    }

    IEnumerator WaitForCondition()
    {
        Debug.Log("Esperant durant 3 segons...");
        yield return new WaitForSeconds(3);
        Debug.Log("Han passat 3 segons!");

        Debug.Log("Esperant fins que es premi la tecla espai...");
        yield return new WaitUntil(() => Input.GetKeyDown(KeyCode.Space));
        Debug.Log("Tecla espai premuda!");
    }
}

En aquest exemple, la coroutine espera durant 3 segons i després espera fins que es premi la tecla espai.

Programació Asíncrona amb Async/Await

Unity també suporta la programació asíncrona utilitzant les paraules clau async i await, que poden fer que el teu codi sigui més net i fàcil de llegir.

Conceptes Clau

  • Mètodes Asíncrons: Mètodes marcats amb la paraula clau async.
  • Paraula Clau Await: Utilitzada per pausar l'execució d'un mètode asíncron fins que la tasca esperada es completi.
  • Task: Representa una operació asíncrona.

Exemple Bàsic d'Async/Await

using System.Threading.Tasks;
using UnityEngine;

public class AsyncExample : MonoBehaviour
{
    async void Start()
    {
        Debug.Log("Iniciant tasca asíncrona...");
        await WaitAndPrint();
        Debug.Log("Tasca asíncrona completada!");
    }

    async Task WaitAndPrint()
    {
        await Task.Delay(3000); // Esperar durant 3 segons
        Debug.Log("Han passat 3 segons en el mètode asíncron!");
    }
}

En aquest exemple, el mètode WaitAndPrint és un mètode asíncron que espera durant 3 segons abans d'imprimir un missatge.

Comparació: Coroutines vs Async/Await

| Característica | Coroutines | Async/Await | |----------------------|-------------------------------------|--------------------------------------| | Sintaxi | Utilitza IEnumerator i yield | Utilitza async i await | | Rendiment | Lleuger, mínim overhead | Lleugerament més overhead degut a les tasques | | Llegibilitat | Pot esdevenir complex amb yields niats | Generalment més llegible i mantenible | | Casos d'Ús | Bo per a operacions basades en fotogrames | Bo per a operacions I/O-bound |

Conclusió

Les coroutines i la programació asíncrona són eines poderoses a Unity per gestionar tasques que requereixen temps sense congelar el fil principal. Les coroutines són ideals per a operacions basades en fotogrames, mentre que async/await és millor per a tasques I/O-bound. Entendre ambdós enfocaments t'ajudarà a escriure aplicacions Unity més eficients i responsives. Practica utilitzant aquestes tècniques en els teus projectes per esdevenir proficient en la gestió d'operacions asíncrones.
