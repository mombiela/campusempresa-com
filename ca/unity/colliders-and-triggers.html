Els col·liders i triggers són components fonamentals a Unity que permeten que els objectes interactuïn entre si en el món del joc. Entendre com utilitzar-los de manera efectiva és crucial per crear jocs dinàmics i interactius.

Què són els Col·liders?

Els col·liders són components que defineixen la forma d'un objecte amb finalitats de col·lisions físiques. Es poden adjuntar a qualsevol GameObject per fer que interactuï amb altres objectes a l'escena.

Tipus de Col·liders

  • Box Collider: Defineix una forma de caixa rectangular.
  • Sphere Collider: Defineix una forma esfèrica.
  • Capsule Collider: Defineix una forma de càpsula.
  • Mesh Collider: Utilitza la malla de l'objecte per a la detecció de col·lisions.
  • Wheel Collider: Col·lidor especial utilitzat per a vehicles.

Afegir un Col·lidor

Per afegir un col·lidor a un GameObject:

  1. Selecciona el GameObject a la Jerarquia.
  2. A l'Inspector, fes clic a "Add Component".
  3. Cerca el tipus de col·lidor desitjat (per exemple, Box Collider) i selecciona'l.
// Exemple: Afegir un Box Collider via script
void Start() {
    gameObject.AddComponent<BoxCollider>();
}

Propietats del Col·lidor

  • Is Trigger: Quan està marcat, el col·lidor actuarà com un trigger, és a dir, no interactuarà físicament amb altres col·liders però encara detectarà superposicions.
  • Material: Defineix les propietats físiques com la fricció i el rebot.

Què són els Triggers?

Els triggers són tipus especials de col·liders que no causen col·lisions físiques però poden detectar quan altres objectes entren, surten o es mantenen dins dels seus límits.

Utilitzar Triggers

Per utilitzar un col·lidor com a trigger:

  1. Afegeix un col·lidor al GameObject.
  2. Marca la casella "Is Trigger" a les propietats del col·lidor.

Esdeveniments de Trigger

Unity proporciona diversos mètodes per gestionar esdeveniments de trigger:

  • OnTriggerEnter: Es crida quan un altre col·lidor entra al trigger.
  • OnTriggerStay: Es crida cada fotograma que un altre col·lidor es manté dins del trigger.
  • OnTriggerExit: Es crida quan un altre col·lidor surt del trigger.
// Exemple: Utilitzar esdeveniments de trigger
void OnTriggerEnter(Collider other) {
    Debug.Log("Objecte ha entrat al trigger");
}

void OnTriggerStay(Collider other) {
    Debug.Log("Objecte està dins del trigger");
}

void OnTriggerExit(Collider other) {
    Debug.Log("Objecte ha sortit del trigger");
}

Exemples Pràctics

Exemple 1: Crear un Trigger de Porta Simple

  1. Crea un GameObject de porta i afegeix un Box Collider.
  2. Marca la casella "Is Trigger" al Box Collider.
  3. Adjunta el següent script al GameObject de la porta:
public class DoorTrigger : MonoBehaviour {
    void OnTriggerEnter(Collider other) {
        if (other.CompareTag("Player")) {
            // Obre la porta
            Debug.Log("La porta s'obre");
        }
    }

    void OnTriggerExit(Collider other) {
        if (other.CompareTag("Player")) {
            // Tanca la porta
            Debug.Log("La porta es tanca");
        }
    }
}

Exemple 2: Detectar Col·lisions amb un Jugador

  1. Crea un GameObject de jugador i afegeix un Rigidbody i un Box Collider.
  2. Adjunta el següent script al GameObject del jugador:
public class PlayerCollision : MonoBehaviour {
    void OnCollisionEnter(Collision collision) {
        if (collision.gameObject.CompareTag("Enemy")) {
            // Gestiona la col·lisió amb l'enemic
            Debug.Log("El jugador ha col·lisionat amb l'enemic");
        }
    }
}

Conclusió

Els col·liders i triggers són essencials per crear entorns interactius i dinàmics a Unity. Entenent com utilitzar diferents tipus de col·liders i com gestionar esdeveniments de trigger, pots crear experiències de joc més atractives i responsives. Practica afegint i configurant col·liders i triggers en els teus projectes per convertir-te en més proficient en el seu ús.
