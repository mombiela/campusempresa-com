Introducció

En aquesta secció, explorarem diversos estudis de cas i exemples per entendre com es pot aplicar PL/SQL en escenaris del món real. Aquests exemples aniran des de nivells principiants fins a avançats, ajudant-te a comprendre les aplicacions pràctiques de PL/SQL.

Nivell principiant

Exemple 1: Bloc PL/SQL simple

Un bloc PL/SQL simple que imprimeix un missatge.

BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello, PL/SQL!');
END;
/
  • Explicació:
    • BEGIN i END defineixen l'inici i el final del bloc PL/SQL.
    • DBMS_OUTPUT.PUT_LINE és una procedura incorporada per imprimir missatges a la consola.

Exemple 2: Declaració i assignació de variables

Un bloc PL/SQL que declara una variable i li assigna un valor.

DECLARE
    v_message VARCHAR2(50);
BEGIN
    v_message := 'Welcome to PL/SQL!';
    DBMS_OUTPUT.PUT_LINE(v_message);
END;
/
  • Explicació:
    • La secció DECLARE s'utilitza per declarar variables.
    • v_message és una variable de tipus VARCHAR2 amb una longitud màxima de 50 caràcters.
    • La variable s'assigna un valor i després s'imprimeix.

Nivell intermedi

Exemple 3: Ús de cursos

Un bloc PL/SQL que utilitza un cursor explícit per obtenir i mostrar els noms dels empleats.

DECLARE
    CURSOR emp_cursor IS
        SELECT first_name, last_name FROM employees;
    v_first_name employees.first_name%TYPE;
    v_last_name employees.last_name%TYPE;
BEGIN
    OPEN emp_cursor;
    LOOP
        FETCH emp_cursor INTO v_first_name, v_last_name;
        EXIT WHEN emp_cursor%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_first_name || ' ' || v_last_name);
    END LOOP;
    CLOSE emp_cursor;
END;
/
  • Explicació:
    • CURSOR emp_cursor IS declara un cursor per seleccionar els noms i cognoms de la taula employees.
    • OPEN emp_cursor obre el cursor.
    • FETCH obté les dades en variables.
    • EXIT WHEN emp_cursor%NOTFOUND surt del bucle quan no es troben més files.
    • CLOSE emp_cursor tanca el cursor.

Exemple 4: Gestió d'excepcions

Un bloc PL/SQL que demostra la gestió d'excepcions.

DECLARE
    v_num NUMBER;
BEGIN
    v_num := 10 / 0;
EXCEPTION
    WHEN ZERO_DIVIDE THEN
        DBMS_OUTPUT.PUT_LINE('Error: Division by zero');
END;
/
  • Explicació:
    • El bloc BEGIN intenta dividir un nombre per zero.
    • El bloc EXCEPTION captura l'excepció ZERO_DIVIDE i imprimeix un missatge d'error.

Nivell avançat

Exemple 5: Creació i ús de procediments

Un procediment PL/SQL que calcula el salari total dels empleats en un departament.

CREATE OR REPLACE PROCEDURE calculate_total_salary (p_dept_id IN NUMBER, p_total_salary OUT NUMBER) IS
BEGIN
    SELECT SUM(salary) INTO p_total_salary
    FROM employees
    WHERE department_id = p_dept_id;
END;
/
  • Explicació:
    • CREATE OR REPLACE PROCEDURE defineix un nou procediment.
    • p_dept_id és un paràmetre d'entrada per a l'ID del departament.
    • p_total_salary és un paràmetre de sortida per al salari total.
    • La sentència SELECT calcula el salari total per al departament especificat.

Exemple 6: Ús de triggers

Un trigger que registra els canvis a la taula employees.

CREATE OR REPLACE TRIGGER log_employee_changes
AFTER INSERT OR UPDATE OR DELETE ON employees
FOR EACH ROW
BEGIN
    IF INSERTING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'INSERT', :NEW.employee_id);
    ELSIF UPDATING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'UPDATE', :NEW.employee_id);
    ELSIF DELETING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'DELETE', :OLD.employee_id);
    END IF;
END;
/
  • Explicació:
    • CREATE OR REPLACE TRIGGER defineix un nou trigger.
    • El trigger s'activa AFTER INSERT OR UPDATE OR DELETE a la taula employees.
    • FOR EACH ROW especifica que el trigger actua sobre cada fila afectada.
    • Les sentències IF comproven el tipus d'operació DML i registren els canvis en conseqüència.

Conclusió
