Introducció

En PL/SQL, els cursors s'utilitzen per gestionar els resultats de les consultes. Permeten obtenir i manipular dades recuperades per consultes SQL. Hi ha dos tipus de cursors en PL/SQL: implícits i explícits. Entendre aquests cursors és crucial per a una programació eficient de bases de dades.

Cursors Implícits

Els cursors implícits es creen automàticament per Oracle sempre que s'executa una instrucció SQL. Són més senzills d'utilitzar però ofereixen menys control en comparació amb els cursors explícits.

Conceptes Clau

  • Gestió Automàtica: Els cursors implícits són gestionats automàticament per PL/SQL.
  • Abast: Només estan disponibles durant l'execució de la instrucció SQL.
  • Ús: Comunament utilitzats per a consultes d'una sola fila.

Exemple

DECLARE
  v_employee_name employees.first_name%TYPE;
BEGIN
  SELECT first_name INTO v_employee_name FROM employees WHERE employee_id = 101;
  DBMS_OUTPUT.PUT_LINE('Nom de l'empleat: ' || v_employee_name);
END;
/

En aquest exemple:

  • S'utilitza un cursor implícit per a la instrucció SELECT.
  • El resultat es guarda a la variable v_employee_name.

Cursors Explícits

Els cursors explícits són definits i controlats pel programador. Proporcionen més flexibilitat i s'utilitzen per a consultes de múltiples files.

Conceptes Clau

  • Gestió Manual: Cal declarar, obrir, obtenir i tancar els cursors explícits.
  • Control: Ofereix més control sobre l'execució de la consulta i la recuperació de dades.
  • Ús: Adequat per a consultes de múltiples files i manipulació complexa de dades.

Passos per Utilitzar Cursors Explícits

  1. Declarar el Cursor: Definir la consulta SQL.
  2. Obrir el Cursor: Executar la consulta SQL.
  3. Obtenir Dades: Recuperar el conjunt de resultats fila per fila.
  4. Tancar el Cursor: Alliberar els recursos.

Exemple

DECLARE
  CURSOR emp_cursor IS
    SELECT first_name, last_name FROM employees WHERE department_id = 10;
  v_first_name employees.first_name%TYPE;
  v_last_name employees.last_name%TYPE;
BEGIN
  OPEN emp_cursor;
  LOOP
    FETCH emp_cursor INTO v_first_name, v_last_name;
    EXIT WHEN emp_cursor%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE('Empleat: ' || v_first_name || ' ' || v_last_name);
  END LOOP;
  CLOSE emp_cursor;
END;
/

En aquest exemple:

  • El cursor emp_cursor es declara per seleccionar empleats del departament 10.
  • El cursor s'obre, i les dades es recuperen en un bucle fins que totes les files són processades.
  • El cursor es tanca després.

Comparació de Cursors Implícits i Explícits

| Característica | Cursor Implícit | Cursor Explícit | |---------------------|----------------------------------------------|----------------------------------------------| | Declaració | Automàtica | Manual | | Control | Limitat | Complet | | Ús | Consultes d'una sola fila | Consultes de múltiples files | | Gestió de Recursos | Automàtica | Manual (obrir, obtenir, tancar) | | Flexibilitat | Menys flexible | Més flexible |

Conclusió

Els cursors són essencials per gestionar els resultats de les consultes en PL/SQL. Els cursors implícits són fàcils d'utilitzar per a consultes senzilles d'una sola fila, mentre que els cursors explícits proporcionen un major control i flexibilitat per a consultes complexes de múltiples files. Entendre quan i com utilitzar cada tipus de cursor t'ajudarà a escriure codi PL/SQL eficient i efectiu.
