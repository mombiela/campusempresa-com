Introducció

PL/SQL (Procedural Language/Structured Query Language) és l'extensió procedimental de SQL i la base de dades relacional d'Oracle Corporation. Optimitzar el codi PL/SQL és crucial per millorar el rendiment i l'eficiència de les aplicacions de bases de dades. Aquesta secció cobrirà diverses tècniques per optimitzar el codi PL/SQL, adequades tant per a principiants com per a usuaris avançats.

Conceptes clau en l'optimització PL/SQL

Comprensió dels plans d'execució

  • Pla d'execució: Un full de ruta que mostra els passos que la base de dades Oracle seguirà per executar una consulta SQL.
  • EXPLAIN PLAN: Una eina per mostrar el pla d'execució d'una instrucció SQL.
EXPLAIN PLAN FOR
SELECT * FROM employees WHERE department_id = 10;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);

Indexació

  • Índexs: Estructures que milloren la velocitat de recuperació de dades.
  • Tipus d'índexs: B-tree, Bitmap, Basats en funcions, etc.
CREATE INDEX emp_dept_idx ON employees(department_id);

Variables de vinculació

  • Variables de vinculació: Marcadors de posició en instruccions SQL que es substitueixen per valors reals en temps d'execució.
  • Beneficis: Redueixen el temps d'anàlisi i milloren el rendiment.
DECLARE
  v_dept_id NUMBER := 10;
BEGIN
  EXECUTE IMMEDIATE 'SELECT * FROM employees WHERE department_id = :dept_id' 
  USING v_dept_id;
END;

Operacions massives

  • Bulk Collect: Recupera múltiples files en una col·lecció en un únic canvi de context.
  • FORALL: Executa una instrucció DML múltiples vegades amb valors diferents en un únic canvi de context.
DECLARE
  TYPE emp_tab IS TABLE OF employees%ROWTYPE;
  l_emps emp_tab;
BEGIN
  SELECT * BULK COLLECT INTO l_emps FROM employees WHERE department_id = 10;

  FORALL i IN l_emps.FIRST..l_emps.LAST
    INSERT INTO employees_archive VALUES l_emps(i);
END;

Eines de perfilat i ajustament

  • DBMS_PROFILER: Un paquet integrat per perfilar el codi PL/SQL.
  • SQL Trace i TKPROF: Eines per rastrejar i analitzar l'execució de SQL.
BEGIN
  DBMS_PROFILER.START_PROFILER('my_profile');
  -- El teu codi PL/SQL aquí
  DBMS_PROFILER.STOP_PROFILER;
END;

Tècniques avançades d'optimització

Ús de col·leccions

  • Taules imbricades i VARRAYs: Milloren el rendiment reduint els canvis de context entre SQL i PL/SQL.
DECLARE
  TYPE numlist IS TABLE OF NUMBER;
  v_numbers numlist := numlist(1, 2, 3, 4, 5);
BEGIN
  FOR i IN v_numbers.FIRST..v_numbers.LAST LOOP
    DBMS_OUTPUT.PUT_LINE(v_numbers(i));
  END LOOP;
END;

Funcions de taula canalitzades

  • Funcions canalitzades: Retornen files a la consulta que les crida abans que la funció completa.
CREATE OR REPLACE FUNCTION get_employees RETURN emp_tab PIPELINED IS
BEGIN
  FOR r IN (SELECT * FROM employees) LOOP
    PIPE ROW(r);
  END LOOP;
  RETURN;
END;

Emmagatzematge en memòria cau de resultats

  • Memòria cau de resultats de funcions PL/SQL: Emmagatzema en memòria cau els resultats de les crides a funcions per millorar el rendiment.
CREATE OR REPLACE FUNCTION get_department_name(p_dept_id NUMBER) 
RETURN VARCHAR2 RESULT_CACHE IS
  v_dept_name VARCHAR2(50);
BEGIN
  SELECT department_name INTO v_dept_name FROM departments WHERE department_id = p_dept_id;
  RETURN v_dept_name;
END;

Execució paral·lela

  • DML i consulta paral·lela: Distribueix la càrrega de treball entre múltiples processadors.
ALTER SESSION ENABLE PARALLEL DML;

INSERT /*+ PARALLEL(employees, 4) */ INTO employees_archive
SELECT * FROM employees WHERE department_id = 10;

Conclusió

Optimitzar el codi PL/SQL implica comprendre i aplicar diverses tècniques per millorar el rendiment i l'eficiència. Des de l'ús de plans d'execució i indexació fins a mètodes avançats com funcions canalitzades i execució paral·lela, cada tècnica juga un paper crucial en l'optimització del rendiment de les teves aplicacions PL/SQL. En dominar aquestes tècniques, pots assegurar-te que les teves aplicacions de bases de dades funcionin de manera fluida i eficient.
