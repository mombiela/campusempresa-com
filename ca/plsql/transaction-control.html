El control de transaccions és un aspecte crucial de la gestió de bases de dades, assegurant la integritat i la consistència de les dades. En PL/SQL, les instruccions de control de transaccions et permeten gestionar els canvis fets per les instruccions DML (Data Manipulation Language) com INSERT, UPDATE i DELETE.

Conceptes Clau

  • Transacció: Una seqüència d'una o més instruccions SQL que s'executen com una unitat de treball única.
  • Commit: Desa tots els canvis fets durant la transacció actual.
  • Rollback: Desfà tots els canvis fets durant la transacció actual.
  • Savepoint: Estableix un punt dins d'una transacció al qual pots tornar més tard.

Instruccions de Control de Transaccions

COMMIT

La instrucció COMMIT s'utilitza per desar tots els canvis fets durant la transacció actual. Un cop una transacció es compromet, els canvis són permanents i visibles per altres usuaris.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (101, 'John', 'Doe');
    
    COMMIT;
END;

ROLLBACK

La instrucció ROLLBACK desfà tots els canvis fets durant la transacció actual. Això és útil si es produeix un error i necessites revertir la base de dades al seu estat anterior.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (102, 'Jane', 'Smith');
    
    ROLLBACK;
END;

SAVEPOINT

La instrucció SAVEPOINT estableix un punt dins d'una transacció al qual pots tornar més tard. Això permet desfer parcialment dins d'una transacció.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (103, 'Alice', 'Johnson');
    
    SAVEPOINT savepoint1;
    
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (104, 'Bob', 'Brown');
    
    ROLLBACK TO savepoint1;
    
    COMMIT;
END;

En aquest exemple, la inserció d''Alice Johnson' es desa, però la inserció de 'Bob Brown' es desfà.

Exemples Pràctics

Exemple 1: Control Bàsic de Transaccions

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (105, 'Charlie', 'Davis');
    
    UPDATE employees
    SET last_name = 'Davison'
    WHERE employee_id = 105;
    
    COMMIT;
END;

En aquest exemple, tant les instruccions INSERT com UPDATE formen part d'una única transacció. La instrucció COMMIT assegura que ambdós canvis es desin.

Exemple 2: Utilitzant SAVEPOINT

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (106, 'Eve', 'White');
    
    SAVEPOINT savepoint1;
    
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (107, 'Frank', 'Black');
    
    SAVEPOINT savepoint2;
    
    DELETE FROM employees
    WHERE employee_id = 106;
    
    ROLLBACK TO savepoint1;
    
    COMMIT;
END;

En aquest exemple:

  • La inserció d''Eve White' i 'Frank Black' són inicialment part de la transacció.
  • L'eliminació d''Eve White' es desfà fins a savepoint1, així que 'Eve White' roman a la base de dades.
  • La transacció es compromet, desant la inserció de 'Frank Black'.

Conclusió

El control de transaccions en PL/SQL és essencial per mantenir la integritat i la consistència de les dades. Utilitzant COMMIT, ROLLBACK i SAVEPOINT, pots gestionar les transaccions de manera efectiva, assegurant que les teves operacions de base de dades siguin fiables i lliures d'errors. Entendre aquests conceptes i com aplicar-los millorarà significativament la teva capacitat per escriure codi PL/SQL robust.
