Introducció

En aquesta secció, aprofundirem en temes avançats de PL/SQL que t'ajudaran a convertir-te en un desenvolupador de PL/SQL competent. Aquests temes inclouen l'optimització del rendiment, estructures de dades avançades i millors pràctiques per escriure codi eficient i mantenible. Al final d'aquesta secció, tindràs una comprensió més profunda de PL/SQL i estaràs preparat per afrontar tasques complexes de programació de bases de dades.

Optimització del Rendiment

L'optimització del rendiment és crucial per assegurar que el teu codi PL/SQL s'executi de manera eficient. Aquí, explorarem diverses tècniques per optimitzar el teu codi.

Ús de Variables de Vinculació

Les variables de vinculació poden millorar significativament el rendiment del teu codi PL/SQL reduint el temps de parsing i millorant l'eficiència de la memòria cau.

-- Exemple d'ús de variables de vinculació
DECLARE
  v_employee_id NUMBER := :employee_id;
  v_employee_name VARCHAR2(100);
BEGIN
  SELECT first_name || ' ' || last_name
  INTO v_employee_name
  FROM employees
  WHERE employee_id = v_employee_id;
  
  DBMS_OUTPUT.PUT_LINE('Nom de l'empleat: ' || v_employee_name);
END;

Bulk Collect i FORALL

Utilitzar BULK COLLECT i FORALL pot reduir els canvis de context entre els motors PL/SQL i SQL, millorant així el rendiment.

-- Exemple de BULK COLLECT i FORALL
DECLARE
  TYPE t_employee_ids IS TABLE OF employees.employee_id%TYPE;
  v_employee_ids t_employee_ids;
  v_employee_names DBMS_SQL.VARCHAR2_TABLE;
BEGIN
  -- Bulk collect d'IDs d'empleats
  SELECT employee_id BULK COLLECT INTO v_employee_ids FROM employees WHERE department_id = 10;
  
  -- Utilitzar FORALL per actualitzar els noms dels empleats
  FORALL i IN v_employee_ids.FIRST .. v_employee_ids.LAST
    UPDATE employees
    SET last_name = 'Updated'
    WHERE employee_id = v_employee_ids(i);
  
  COMMIT;
END;

Estructures de Dades Avançades

PL/SQL suporta diverses estructures de dades avançades que poden ajudar-te a gestionar dades complexes de manera més efectiva.

Taules Imbricades

Les taules imbricades són un tipus de col·lecció que pot emmagatzemar un nombre arbitrari d'elements.

-- Exemple d'ús de taules imbricades
DECLARE
  TYPE t_numbers IS TABLE OF NUMBER;
  v_numbers t_numbers := t_numbers(1, 2, 3, 4, 5);
BEGIN
  FOR i IN 1 .. v_numbers.COUNT LOOP
    DBMS_OUTPUT.PUT_LINE('Número: ' || v_numbers(i));
  END LOOP;
END;

VARRAYs

Els VARRAYs (matrius de mida variable) són un altre tipus de col·lecció que té una mida màxima fixa.

-- Exemple d'ús de VARRAYs
DECLARE
  TYPE t_varray IS VARRAY(5) OF VARCHAR2(20);
  v_varray t_varray := t_varray('A', 'B', 'C', 'D', 'E');
BEGIN
  FOR i IN 1 .. v_varray.COUNT LOOP
    DBMS_OUTPUT.PUT_LINE('Element: ' || v_varray(i));
  END LOOP;
END;

Millors Pràctiques

Seguir les millors pràctiques pot ajudar-te a escriure codi PL/SQL net, eficient i mantenible.

Gestió d'Excepcions

Una gestió adequada de les excepcions assegura que el teu codi pugui gestionar situacions inesperades de manera elegant.

-- Exemple de gestió d'excepcions
BEGIN
  -- Algunes línies de codi que podrien generar una excepció
  UPDATE employees SET salary = salary * 1.1 WHERE employee_id = 9999;
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    DBMS_OUTPUT.PUT_LINE('No s'ha trobat cap empleat amb l'ID donat.');
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('S'ha produït un error inesperat: ' || SQLERRM);
END;

Modularització del Codi

Dividir el teu codi en mòduls més petits i reutilitzables pot fer que sigui més fàcil de mantenir i entendre.

-- Exemple de modularització del codi utilitzant procediments
CREATE OR REPLACE PROCEDURE update_salary(p_employee_id IN NUMBER, p_percentage IN NUMBER) IS
BEGIN
  UPDATE employees
  SET salary = salary * (1 + p_percentage / 100)
  WHERE employee_id = p_employee_id;
END update_salary;

-- Crida al procediment
BEGIN
  update_salary(1001, 10);
END;
