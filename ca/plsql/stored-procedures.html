Procediments emmagatzemats en PL/SQL
Funcions ►

Els procediments emmagatzemats són una característica poderosa en PL/SQL que et permet encapsular i reutilitzar codi. Aquest tema et guiarà des dels conceptes bàsics fins als avançats dels procediments emmagatzemats en PL/SQL.

Introducció als Procediments Emmagatzemats

Els procediments emmagatzemats són subprogrames emmagatzemats a la base de dades que es poden executar per realitzar una tasca específica. Ajuden a millorar el rendiment, mantenir la consistència i augmentar la seguretat.

Conceptes Clau

  • Definició: Un procediment emmagatzemat és un bloc PL/SQL amb nom que pot acceptar paràmetres i ser invocat per realitzar una tasca específica.
  • Avantatges:
    • Rendiment: Reducció del trànsit de xarxa i millor velocitat d'execució.
    • Reutilització: El codi es pot reutilitzar en múltiples aplicacions.
    • Seguretat: L'accés es pot controlar mitjançant permisos.
    • Mantenibilitat: Més fàcil de gestionar i actualitzar el codi.

Creació de Procediments Emmagatzemats

Per crear un procediment emmagatzemat, utilitzes la instrucció CREATE PROCEDURE. Aquí tens la sintaxi bàsica:

CREATE [OR REPLACE] PROCEDURE procedure_name 
  [ (parameter_name [IN | OUT | IN OUT] data_type [, ...]) ]
IS
  -- Declaracions
BEGIN
  -- Instruccions executables
EXCEPTION
  -- Gestió d'excepcions
END procedure_name;

Exemple: Procediment Emmagatzemat Simple

Creem un procediment emmagatzemat simple que imprimeix "Hola, Món!".

CREATE OR REPLACE PROCEDURE say_hello IS
BEGIN
  DBMS_OUTPUT.PUT_LINE('Hola, Món!');
END say_hello;

Per executar aquest procediment, pots utilitzar:

BEGIN
  say_hello;
END;

Paràmetres en Procediments Emmagatzemats

Els procediments emmagatzemats poden acceptar paràmetres per fer-los més flexibles i dinàmics. Hi ha tres tipus de paràmetres:

  • IN: Passa un valor al procediment.
  • OUT: Retorna un valor del procediment.
  • IN OUT: Passa un valor al procediment i retorna un valor actualitzat.

Exemple: Procediment amb Paràmetres

Aquí tens un exemple d'un procediment emmagatzemat amb paràmetres IN i OUT:

CREATE OR REPLACE PROCEDURE greet_user (
  p_name IN VARCHAR2,
  p_greeting OUT VARCHAR2
) IS
BEGIN
  p_greeting := 'Hola, ' || p_name || '!';
END greet_user;

Per executar aquest procediment:

DECLARE
  v_greeting VARCHAR2(50);
BEGIN
  greet_user('Alice', v_greeting);
  DBMS_OUTPUT.PUT_LINE(v_greeting);
END;

Gestió d'Excepcions en Procediments Emmagatzemats

La gestió d'excepcions és crucial per gestionar errors i assegurar la robustesa dels teus procediments emmagatzemats.

Exemple: Procediment amb Gestió d'Excepcions

CREATE OR REPLACE PROCEDURE divide_numbers (
  p_num1 IN NUMBER,
  p_num2 IN NUMBER,
  p_result OUT NUMBER
) IS
BEGIN
  p_result := p_num1 / p_num2;
EXCEPTION
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('Error: Divisió per zero.');
    p_result := NULL;
END divide_numbers;

Per executar aquest procediment:

DECLARE
  v_result NUMBER;
BEGIN
  divide_numbers(10, 0, v_result);
  DBMS_OUTPUT.PUT_LINE('Resultat: ' || v_result);
END;

Temes Avançats

Ús de Cursors en Procediments Emmagatzemats

Els cursors et permeten obtenir i manipular dades fila per fila.

Exemple: Procediment amb Cursor

CREATE OR REPLACE PROCEDURE list_employees IS
  CURSOR emp_cursor IS
    SELECT first_name, last_name FROM employees;
  v_first_name employees.first_name%TYPE;
  v_last_name employees.last_name%TYPE;
BEGIN
  OPEN emp_cursor;
  LOOP
    FETCH emp_cursor INTO v_first_name, v_last_name;
    EXIT WHEN emp_cursor%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE(v_first_name || ' ' || v_last_name);
  END LOOP;
  CLOSE emp_cursor;
END list_employees;

SQL Dinàmic en Procediments Emmagatzemats

El SQL dinàmic et permet construir i executar instruccions SQL dinàmicament en temps d'execució.

Exemple: Procediment amb SQL Dinàmic

CREATE OR REPLACE PROCEDURE execute_dynamic_sql (
  p_table_name IN VARCHAR2
) IS
  v_sql VARCHAR2(200);
BEGIN
  v_sql := 'SELECT COUNT(*) FROM ' || p_table_name;
  EXECUTE IMMEDIATE v_sql INTO v_count;
  DBMS_OUTPUT.PUT_LINE('Count: ' || v_count);
END execute_dynamic_sql;

Conclusió

Els procediments emmagatzemats en PL/SQL són una eina poderosa per encapsular i reutilitzar codi. Milloren el rendiment, la seguretat i la mantenibilitat. Comprenent com crear, utilitzar paràmetres, gestionar excepcions i implementar funcions avançades com cursors i SQL dinàmic, pots aprofitar els procediments emmagatzemats per construir aplicacions de base de dades robustes i eficients.
