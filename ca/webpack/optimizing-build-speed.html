Introducció

Webpack és un potent empaquetador de mòduls per a aplicacions JavaScript. No obstant això, a mesura que els projectes creixen, els temps de compilació poden convertir-se en un coll d'ampolla. Aquesta secció et guiarà a través de diverses tècniques per optimitzar la velocitat de compilació, des d'estratègies bàsiques fins a avançades.

Conceptes clau

  • Velocitat de compilació: El temps que triga Webpack a processar i empaquetar la teva aplicació.
  • Compilacions incrementals: Compilar només les parts de l'aplicació que han canviat.
  • Caché: Emmagatzemar artefactes de compilació per evitar processaments redundants.
  • Paral·lelització: Executar tasques de manera concurrent per accelerar el procés de compilació.

Tècniques bàsiques

1. Configuració del mode

Webpack 4 i versions superiors introdueixen una opció de configuració mode que estableix diverses optimitzacions segons l'entorn.

module.exports = {
  mode: 'development', // o 'production'
};
  • Mode de desenvolupament: Optimitza per a la velocitat de compilació i depuració.
  • Mode de producció: Optimitza per al rendiment i mides de paquet més petites.

2. Punts d'entrada

Minimitza el nombre de punts d'entrada per reduir la complexitat del procés de compilació.

module.exports = {
  entry: './src/index.js',
};

3. Excloure Node Modules

Utilitza l'opció exclude en els carregadors per saltar el processament de fitxers en node_modules.

module: {
  rules: [
    {
      test: /\.js$/,
      exclude: /node_modules/,
      use: 'babel-loader',
    },
  ],
},

Tècniques intermèdies

1. Caché

Habilita la caché per emmagatzemar resultats de compilació intermedis.

module.exports = {
  cache: {
    type: 'filesystem', // o 'memory'
  },
};

2. Paral·lelització

Utilitza el thread-loader per executar carregadors en un grup de treballadors.

module: {
  rules: [
    {
      test: /\.js$/,
      use: [
        'thread-loader',
        'babel-loader',
      ],
    },
  ],
},

3. DLL Plugin

Utilitza el DllPlugin per dividir els paquets en peces més petites i reutilitzables.

const webpack = require('webpack');

module.exports = {
  entry: {
    vendor: ['react', 'react-dom'],
  },
  output: {
    filename: '[name].dll.js',
    path: path.resolve(__dirname, 'dist'),
    library: '[name]_dll',
  },
  plugins: [
    new webpack.DllPlugin({
      name: '[name]_dll',
      path: path.resolve(__dirname, 'dist/[name]-manifest.json'),
    }),
  ],
};

Tècniques avançades

1. Divisió de codi

Divideix el teu codi en trossos més petits per carregar només les parts necessàries.

module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
    },
  },
};

2. Tree Shaking

Elimina codi no utilitzat per reduir la mida del paquet i millorar la velocitat de compilació.

module.exports = {
  mode: 'production',
  optimization: {
    usedExports: true,
  },
};

3. Caché persistent

Utilitza cache-loader per emmagatzemar en caché els resultats de carregadors costosos.

module: {
  rules: [
    {
      test: /\.js$/,
      use: [
        'cache-loader',
        'babel-loader',
      ],
    },
  ],
},

Resum

Optimitzar la velocitat de compilació de Webpack implica una combinació de tècniques bàsiques, intermèdies i avançades. Configurant el mode, minimitzant els punts d'entrada i excloent fitxers innecessaris, pots aconseguir millores significatives. Es poden fer més optimitzacions utilitzant caché, paral·lelització, plugins DLL, divisió de codi, tree shaking i caché persistent. Implementar aquestes estratègies t'ajudarà a mantenir un procés de compilació ràpid i eficient a mesura que el teu projecte creix.
