Comprendre l'Entrada, la Sortida i la Configuració
Comandes bàsiques de Webpack ►

Webpack és un potent empaquetador de mòduls per a aplicacions JavaScript. Per utilitzar Webpack de manera efectiva, és crucial entendre els seus conceptes bàsics: Entrada, Sortida i Configuració. Aquesta secció et guiarà a través d'aquests conceptes des d'un nivell principiant fins a un nivell avançat.

Entrada

El punt d'entrada és el primer fitxer que Webpack mira per començar a construir el gràfic de dependències. Indica a Webpack on començar i segueix el gràfic de dependències per saber què empaquetar.

Punt d'Entrada Únic

Per a aplicacions senzilles, potser només necessitis un únic punt d'entrada.

// webpack.config.js
module.exports = {
  entry: './src/index.js',
};
  • entry: Especifica el fitxer del punt d'entrada. En aquest cas, és ./src/index.js.

Múltiples Punts d'Entrada

Per a aplicacions més complexes, potser tinguis múltiples punts d'entrada.

// webpack.config.js
module.exports = {
  entry: {
    app: './src/app.js',
    admin: './src/admin.js',
  },
};
  • entry: Un objecte on les claus són els noms dels punts d'entrada i els valors són els camins als fitxers.

Sortida

La propietat de sortida indica a Webpack on emetre els paquets que crea i com anomenar aquests fitxers.

Configuració Bàsica de Sortida

// webpack.config.js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist',
  },
};
  • filename: El nom del fitxer de sortida del paquet.
  • path: El directori on es col·locarà el fitxer de sortida. __dirname és una variable de Node.js que es refereix al directori del mòdul actual.

Configuració Avançada de Sortida

// webpack.config.js
module.exports = {
  entry: {
    app: './src/app.js',
    admin: './src/admin.js',
  },
  output: {
    filename: '[name].bundle.js',
    path: __dirname + '/dist',
    publicPath: '/assets/',
  },
};
  • filename: [name] és un marcador de posició que serà reemplaçat pel nom del punt d'entrada (per exemple, app.bundle.js i admin.bundle.js).
  • publicPath: Especifica l'URL públic del directori de sortida quan es fa referència en un navegador.

Configuració

La configuració de Webpack es fa típicament en un fitxer webpack.config.js. Aquest fitxer és un mòdul JavaScript que exporta un objecte.

Configuració Bàsica

// webpack.config.js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist',
  },
  mode: 'development',
};
  • mode: Especifica el mode en què ha de funcionar Webpack. Les opcions són development, production i none.

Loaders i Plugins

Els loaders i plugins s'utilitzen per transformar i millorar les capacitats de Webpack.

Loaders

Els loaders et permeten preprocessar fitxers mentre els require o import.

// webpack.config.js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist',
  },
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
};
  • module.rules: Una matriu de regles per especificar com s'han de gestionar els diferents tipus de fitxers.
  • test: Una expressió regular per coincidir amb els tipus de fitxers.
  • use: Especifica els loaders a utilitzar per als fitxers coincidents.

Plugins

Els plugins s'utilitzen per realitzar una àmplia gamma de tasques com l'optimització de paquets, la gestió d'actius i la injecció de variables d'entorn.

// webpack.config.js
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist',
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
    }),
  ],
};
  • plugins: Una matriu de plugins utilitzats en el procés de construcció.
  • HtmlWebpackPlugin: Genera un fitxer HTML que inclou els paquets de sortida.

Conclusió

Entendre els conceptes bàsics d'Entrada, Sortida i Configuració en Webpack és essencial per empaquetar i gestionar efectivament les teves aplicacions JavaScript. En dominar aquests conceptes bàsics, pots aprofitar les potents característiques de Webpack per optimitzar i agilitzar el teu flux de treball de desenvolupament. A mesura que avancis, integrar loaders i plugins millorarà encara més les capacitats de la teva aplicació, fent de Webpack una eina indispensable en el teu kit d'eines de desenvolupament.
