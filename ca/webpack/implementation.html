En aquesta secció, cobrirem la implementació pràctica de Webpack, des de la configuració bàsica fins a l'ús més avançat. Al final d'aquesta secció, hauríeu de ser capaços de crear i gestionar una configuració de Webpack per als vostres projectes.

Configurant Webpack

Per començar amb Webpack, necessiteu instal·lar-lo juntament amb les seves dependències.

Instal·lació

  1. Inicialitzar un nou projecte:

    mkdir my-webpack-project
cd my-webpack-project
npm init -y

  2. Instal·lar Webpack i Webpack CLI:

    npm install --save-dev webpack webpack-cli

Configuració Bàsica

Webpack requereix un fitxer de configuració per definir com ha de processar els fitxers del vostre projecte. El fitxer de configuració per defecte és webpack.config.js.

Creant `webpack.config.js`

  1. Crear el fitxer de configuració:

    // webpack.config.js
const path = require('path');

module.exports = {
    entry: './src/index.js',
    output: {
        filename: 'bundle.js',
        path: path.resolve(__dirname, 'dist')
    },
    mode: 'development'
};

  2. Explicació:

    • entry: Especifica el punt d'entrada de la vostra aplicació.
    • output: Defineix el fitxer i el directori de sortida.
    • mode: Estableix el mode a 'development' per a una depuració més fàcil.

Executant Webpack

Per empaquetar el vostre projecte, executeu la següent comanda:

npx webpack

Això generarà un fitxer dist/bundle.js basat en la vostra configuració.

Loaders

Els loaders permeten a Webpack processar diferents tipus de fitxers més enllà de JavaScript, com ara CSS, imatges i més.

Afegint CSS Loader

  1. Instal·lar els loaders necessaris:

    npm install --save-dev style-loader css-loader

  2. Actualitzar webpack.config.js:

    module.exports = {
    // ... configuració anterior
    module: {
        rules: [
            {
                test: /\.css$/,
                use: ['style-loader', 'css-loader']
            }
        ]
    }
};

  3. Explicació:

    • test: Una expressió regular per coincidir amb el tipus de fitxer.
    • use: Especifica els loaders per processar els fitxers coincidents.

Exemple de Fitxer CSS

  1. Crear un fitxer CSS:

    /* src/styles.css */
body {
    background-color: lightblue;
}

  2. Importar el fitxer CSS en el vostre punt d'entrada JavaScript:

    // src/index.js
import './styles.css';

Plugins

Els plugins amplien la funcionalitat de Webpack. Poden realitzar una àmplia gamma de tasques com optimitzar paquets, gestionar actius i més.

Afegint HTML Webpack Plugin

  1. Instal·lar el plugin:

    npm install --save-dev html-webpack-plugin

  2. Actualitzar webpack.config.js:

    const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    // ... configuració anterior
    plugins: [
        new HtmlWebpackPlugin({
            template: './src/index.html'
        })
    ]
};

  3. Explicació:

    • HtmlWebpackPlugin: Genera automàticament un fitxer HTML i injecta el JavaScript empaquetat.

Exemple de Fitxer HTML

  1. Crear un fitxer HTML: 
    <!-- src/index.html -->
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Projecte Webpack</title>
</head>
<body>
    <h1>Hola, Webpack!</h1>
</body>
</html>

Configuració Avançada

A mesura que el vostre projecte creix, podeu necessitar configuracions més avançades.

Dividint la Configuració

Podeu dividir la vostra configuració en fitxers separats per a desenvolupament i producció.

  1. Crear una configuració comuna:

    // webpack.common.js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    entry: './src/index.js',
    output: {
        filename: 'bundle.js',
        path: path.resolve(__dirname, 'dist')
    },
    plugins: [
        new HtmlWebpackPlugin({
            template: './src/index.html'
        })
    ]
};

  2. Crear una configuració de desenvolupament:

    // webpack.dev.js
const { merge } = require('webpack-merge');
const common = require('./webpack.common.js');

module.exports = merge(common, {
    mode: 'development',
    devtool: 'inline-source-map',
    devServer: {
        contentBase: './dist'
    }
});

  3. Crear una configuració de producció:

    // webpack.prod.js
const { merge } = require('webpack-merge');
const common = require('./webpack.common.js');

module.exports = merge(common, {
    mode: 'production',
    optimization: {
        minimize: true
    }
});

  4. Explicació:

    • webpack-merge: Utilitat per fusionar configuracions.
    • devtool: Mapes de fonts per a una depuració més fàcil en desenvolupament.
    • devServer: Configuració per al servidor de desenvolupament de Webpack.
    • optimization: Minimitzar la sortida per a producció.

Executant Diferents Configuracions

Per executar la configuració de desenvolupament:

npx webpack --config webpack.dev.js

Per executar la configuració de producció:

npx webpack --config webpack.prod.js

Conclusió

En aquesta secció, hem cobert els passos essencials per implementar Webpack en el vostre projecte, des de la configuració bàsica fins a configuracions avançades. Heu après a:

  • Instal·lar i configurar Webpack.
  • Crear un fitxer de configuració bàsic.
  • Utilitzar loaders per gestionar diferents tipus de fitxers.
  • Utilitzar plugins per ampliar la funcionalitat de Webpack.
  • Dividir configuracions per a entorns de desenvolupament i producció.

Amb aquest coneixement, ara podeu gestionar i optimitzar efectivament el procés de construcció del vostre projecte utilitzant Webpack.
