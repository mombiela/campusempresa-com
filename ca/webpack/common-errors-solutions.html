Errors comuns i solucions en Webpack
Tècniques de depuració ►

Webpack és un potent empaquetador de mòduls per a aplicacions JavaScript, però de vegades pot ser complicat de configurar i depurar. Aquesta secció cobreix errors comuns que podries trobar mentre treballes amb Webpack i proporciona solucions per ajudar-te a resoldre'ls.

1. Errors de configuració

1.1. Objecte de configuració invàlid

Un dels errors més comuns és un objecte de configuració invàlid. Això sol ocórrer quan el fitxer de configuració de Webpack (webpack.config.js) no està configurat correctament.

Missatge d'error:

Invalid configuration object. Webpack has been initialised using a configuration object that does not match the API schema.

Solució:

  • Assegura't que el teu objecte de configuració coincideixi amb l'esquema de Webpack.
  • Revisa l'ortografia i l'estructura de les teves opcions de configuració.

Exemple:

// Configuració incorrecta
module.exports = {
  entry: 'src/index.js',
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist'
  },
  mode: 'development'
};

// Configuració correcta
module.exports = {
  entry: './src/index.js', // Assegura't que el camí és correcte
  output: {
    filename: 'bundle.js',
    path: __dirname + '/dist'
  },
  mode: 'development'
};

1.2. Falten carregadors

Els carregadors són essencials per processar diferents tipus d'arxius. Un error comú és oblidar incloure els carregadors necessaris en la teva configuració.

Missatge d'error:

Module parse failed: Unexpected token

Solució:

  • Instal·la els carregadors necessaris utilitzant npm o yarn.
  • Afegeix els carregadors a la teva configuració de Webpack.

Exemple:

// Instal·la el carregador
// npm install --save-dev babel-loader @babel/core @babel/preset-env

// Configuració incorrecta
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        use: 'babel-loader'
      }
    ]
  }
};

// Configuració correcta
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/, // Exclou node_modules per evitar processar arxius innecessaris
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      }
    ]
  }
};

2. Errors de dependència

2.1. Mòdul no trobat

Aquest error es produeix quan Webpack no pot trobar un mòdul que s'està important en el teu codi.

Missatge d'error:

Module not found: Error: Can't resolve 'module-name'

Solució:

  • Assegura't que el mòdul està instal·lat.
  • Revisa el camí d'importació per detectar errors tipogràfics o camins incorrectes.

Exemple:

// Importació incorrecta
import myModule from 'my-module';

// Importació correcta
import myModule from './my-module'; // Assegura't que el camí és correcte

2.2. Desajust de versions

De vegades, diferents versions de dependències poden causar conflictes.

Solució:

  • Revisa les versions de les teves dependències en package.json.
  • Utilitza npm ls o yarn list per veure l'arbre de dependències i identificar conflictes.
  • Actualitza o degrada les dependències a versions compatibles.

Exemple:

// package.json
{
  "dependencies": {
    "webpack": "^5.0.0",
    "webpack-cli": "^4.0.0"
  }
}

3. Errors en temps d'execució

3.1. ReferenceError: process is not defined

Aquest error sovint es produeix quan s'utilitzen variables d'entorn en el navegador.

Solució:

  • Utilitza el DefinePlugin per definir variables d'entorn.

Exemple:

const webpack = require('webpack');

module.exports = {
  plugins: [
    new webpack.DefinePlugin({
      'process.env.NODE_ENV': JSON.stringify('production')
    })
  ]
};

3.2. Cannot read property 'call' of undefined

Aquest error pot ocórrer a causa de problemes amb la resolució de mòduls o un ús incorrecte de plugins.

Solució:

  • Assegura't que tots els plugins estan configurats correctament.
  • Revisa si hi ha errors tipogràfics o un ús incorrecte en el teu codi.

Exemple:

// Ús incorrecte del plugin
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};

// Ús correcte del plugin
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};

Conclusió

Entendre i resoldre errors comuns de Webpack pot millorar significativament el teu flux de treball de desenvolupament. Configurant acuradament la teva configuració de Webpack, assegurant-te que totes les dependències i carregadors estan instal·lats correctament, i utilitzant adequadament els plugins, pots evitar moltes de les trampes que els desenvolupadors sovint troben. Mantingues aquesta guia a mà com a referència per solucionar i resoldre problemes a mesura que sorgeixin. Bona programació!
