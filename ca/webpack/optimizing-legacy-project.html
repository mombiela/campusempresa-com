Introducció

Optimitzar un projecte heretat pot millorar significativament el rendiment, la mantenibilitat i l'escalabilitat. Aquesta secció et guiarà a través del procés d'utilitzar Webpack per optimitzar un projecte existent, cobrint tant tècniques bàsiques com avançades.

Entendre el Projecte Heretat

Abans de submergir-te en l'optimització, és crucial entendre l'estat actual del projecte.

  • Analitza el procés de construcció actual: Identifica com es construeix i empaqueta actualment el projecte.
  • Identifica colls d'ampolla: Busca àrees on el rendiment és deficient.
  • Estableix objectius: Defineix què vols aconseguir amb l'optimització (per exemple, temps de construcció més ràpids, mides de paquet més petites).

Configuració de Webpack

Si el teu projecte heretat no utilitza ja Webpack, el primer pas és configurar-lo.

  1. Instal·la Webpack i Webpack CLI:

    npm install --save-dev webpack webpack-cli

  2. Crea un fitxer de configuració bàsic de Webpack (webpack.config.js):

    const path = require('path');

module.exports = {
    entry: './src/index.js',
    output: {
        filename: 'bundle.js',
        path: path.resolve(__dirname, 'dist')
    },
    mode: 'development'
};

Divisió de Codi

La divisió de codi ajuda a descompondre el codi en trossos més petits, que es poden carregar sota demanda.

  • Punts d'entrada: Defineix múltiples punts d'entrada per dividir el codi.

    module.exports = {
    entry: {
        app: './src/app.js',
        vendor: './src/vendor.js'
    },
    output: {
        filename: '[name].bundle.js',
        path: path.resolve(__dirname, 'dist')
    }
};

  • Imports dinàmics: Utilitza imports dinàmics per dividir el codi en punts específics.

    import(/* webpackChunkName: "lodash" */ 'lodash').then(_ => {
    // Utilitza lodash aquí
});

Carregadors i Connectors

Els carregadors i connectors són essencials per transformar i optimitzar el teu codi.

  • Carregadors: Transformen fitxers en mòduls.

    • Babel Loader: Transpila JavaScript modern a versions més antigues per compatibilitat. 
      npm install --save-dev babel-loader @babel/core @babel/preset-env

      module: {
    rules: [
        {
            test: /\.js$/,
            exclude: /node_modules/,
            use: {
                loader: 'babel-loader',
                options: {
                    presets: ['@babel/preset-env']
                }
            }
        }
    ]
}

  • Connectors: Realitzen una gamma més àmplia de tasques com l'optimització de paquets, la gestió d'actius i la injecció de variables d'entorn.

    • HtmlWebpackPlugin: Simplifica la creació de fitxers HTML per servir els teus paquets. 
      npm install --save-dev html-webpack-plugin

      const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    plugins: [
        new HtmlWebpackPlugin({
            template: './src/index.html'
        })
    ]
};

Optimització de la Sortida

Optimitza la sortida per millorar el rendiment i reduir la mida del paquet.

  • Minificació: Minifica JavaScript utilitzant TerserPlugin.

    npm install --save-dev terser-webpack-plugin

    const TerserPlugin = require('terser-webpack-plugin');

module.exports = {
    optimization: {
        minimize: true,
        minimizer: [new TerserPlugin()],
    },
};

  • Tree Shaking: Elimina codi no utilitzat.

    module.exports = {
    mode: 'production',
    optimization: {
        usedExports: true,
    },
};

Tècniques Avançades

Per a una optimització més avançada, considera les següents tècniques:

  • Caché: Millora el rendiment de construcció emmagatzemant en caché els mòduls i trossos generats.

    module.exports = {
    output: {
        filename: '[name].[contenthash].js',
    },
    optimization: {
        moduleIds: 'deterministic',
        runtimeChunk: 'single',
        splitChunks: {
            cacheGroups: {
                vendor: {
                    test: /[\\/]node_modules[\\/]/,
                    name: 'vendors',
                    chunks: 'all',
                },
            },
        },
    },
};

  • Perfilat: Utilitza les eines de perfilat integrades de Webpack per identificar colls d'ampolla de rendiment.

    webpack --profile --json > stats.json

Conclusió

Optimitzar un projecte heretat amb Webpack implica configurar Webpack, utilitzar la divisió de codi, carregadors i connectors, i aplicar tècniques avançades com la caché i el perfilat. Seguint aquests passos, pots millorar significativament el rendiment i la mantenibilitat del teu projecte. Recorda monitoritzar i refinar contínuament el teu procés de construcció per mantenir el teu projecte optimitzat.
