Introducció a Webpack

Webpack és un potent empaquetador de mòduls per a aplicacions JavaScript. Pren mòduls amb dependències i genera actius estàtics que representen aquests mòduls. Entendre com planificar i configurar Webpack és crucial per a fluxos de treball eficients de desenvolupament i producció.

Conceptes Clau

  • Punt d'Entrada: El punt de partida perquè Webpack construeixi el seu gràfic de dependències intern.
  • Sortida: On es generaran els fitxers empaquetats.
  • Loaders: Transformacions aplicades al codi font d'un mòdul.
  • Plugins: Amplien o personalitzen el procés de construcció de Webpack.
  • Mode: Defineix l'entorn (desenvolupament o producció) per optimitzar la construcció en conseqüència.

Configuració Inicial

Instal·lant Webpack

Per començar amb Webpack, cal instal·lar-lo mitjançant npm (Node Package Manager).

npm install --save-dev webpack webpack-cli

Configuració Bàsica

Crea un fitxer webpack.config.js a l'arrel del teu projecte. Aquest fitxer contindrà la configuració per a Webpack.

// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.js', // Punt d'entrada
  output: {
    filename: 'bundle.js', // Fitxer de sortida
    path: path.resolve(__dirname, 'dist') // Directori de sortida
  },
  mode: 'development' // Mode
};

Explicació:

  • entry: Especifica el punt d'entrada de la teva aplicació.
  • output: Defineix el nom del fitxer de sortida i el directori.
  • mode: Estableix el mode a 'development' per a una millor experiència de depuració.

Ús de Loaders

Els loaders permeten preprocessar fitxers mentre els importes o carregues. Per exemple, pots utilitzar Babel per transcompilar JavaScript modern a versions més antigues per a compatibilitat.

Instal·lant Babel Loader

npm install --save-dev babel-loader @babel/core @babel/preset-env

Configurant Babel Loader

Actualitza el teu webpack.config.js per incloure el Babel loader.

// webpack.config.js
module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  mode: 'development',
  module: {
    rules: [
      {
        test: /\.js$/, // Aplica aquesta regla als fitxers .js
        exclude: /node_modules/, // Exclou node_modules
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env'] // Utilitza el preset-env
          }
        }
      }
    ]
  }
};

Ús de Plugins

Els plugins poden realitzar una àmplia gamma de tasques com l'optimització del paquet, la gestió d'actius i la injecció de variables d'entorn.

Instal·lant HTML Webpack Plugin

npm install --save-dev html-webpack-plugin

Configurant HTML Webpack Plugin

Actualitza el teu webpack.config.js per incloure l'HTML Webpack Plugin.

// webpack.config.js
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  mode: 'development',
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html' // Fitxer de plantilla
    })
  ]
};

Desenvolupament vs. Producció

Webpack es pot configurar de manera diferent per a entorns de desenvolupament i producció per optimitzar el procés de construcció.

Configurant Configuracions Específiques per a l'Entorn

Crea fitxers de configuració separats per a desenvolupament i producció.

webpack.dev.js

// webpack.dev.js
const { merge } = require('webpack-merge');
const common = require('./webpack.common.js');

module.exports = merge(common, {
  mode: 'development',
  devtool: 'inline-source-map', // Mapes de fonts per a depuració
  devServer: {
    contentBase: './dist'
  }
});

webpack.prod.js

// webpack.prod.js
const { merge } = require('webpack-merge');
const common = require('./webpack.common.js');

module.exports = merge(common, {
  mode: 'production',
  optimization: {
    minimize: true // Minimitza la sortida
  }
});

webpack.common.js

// webpack.common.js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};

Executant Diferents Configuracions

Utilitza scripts npm per executar diferents configuracions.

// package.json
{
  "scripts": {
    "start": "webpack serve --config webpack.dev.js",
    "build": "webpack --config webpack.prod.js"
  }
}

Conclusió

Configurar Webpack implica entendre els seus conceptes bàsics com punts d'entrada, sortida, loaders, plugins i modes. Seguint els passos descrits en aquesta secció, pots crear una configuració bàsica de Webpack i ampliar-la per a entorns de desenvolupament i producció. Aquest coneixement fonamental t'ajudarà a construir configuracions de Webpack més complexes i optimitzades a mesura que progressis en el curs.
