Introducció a Webpack i TypeScript

  • Webpack: Un potent empaquetador de mòduls per a aplicacions JavaScript.
  • TypeScript: Un superconjunt tipat de JavaScript que es compila a JavaScript pla.

Per què utilitzar Webpack amb TypeScript?

  • Modularització: Gestiona i empaqueta eficientment els mòduls de TypeScript.
  • Optimització: Minimitza i optimitza la sortida per a producció.
  • Compatibilitat: Integra perfectament TypeScript amb les característiques modernes de JavaScript.

Configuració de Webpack amb TypeScript

Prerequisits

  • Node.js i npm instal·lats.
  • Comprensió bàsica de JavaScript i TypeScript.

Configuració inicial del projecte

  1. Crea un nou directori de projecte: 
    mkdir webpack-typescript-demo
cd webpack-typescript-demo
  2. Inicialitza npm: 
    npm init -y
  3. Instal·la Webpack i TypeScript: 
    npm install --save-dev webpack webpack-cli typescript ts-loader

Configuració de TypeScript

  1. Crea un fitxer tsconfig.json: 
    {
  "compilerOptions": {
    "outDir": "./dist",
    "sourceMap": true,
    "strict": true,
    "module": "es6",
    "target": "es5",
    "jsx": "react",
    "moduleResolution": "node",
    "esModuleInterop": true
  },
  "include": ["src/**/*"]
}
  2. Crea el directori de fonts: 
    mkdir src

Configuració de Webpack

  1. Crea un fitxer webpack.config.js: 
    const path = require('path');

module.exports = {
  entry: './src/index.ts',
  module: {
    rules: [
      {
        test: /\.ts$/,
        use: 'ts-loader',
        exclude: /node_modules/
      }
    ]
  },
  resolve: {
    extensions: ['.ts', '.js']
  },
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  devtool: 'source-map'
};

Afegint codi TypeScript

  1. Crea un fitxer index.ts al directori src: 
    const greet = (name: string): string => {
  return `Hello, ${name}!`;
};

console.log(greet('World'));

Construint el projecte

  1. Afegeix un script de construcció a package.json: 
    "scripts": {
  "build": "webpack"
}
  2. Executa l'script de construcció: 
    npm run build

Configuració avançada

Afegint plugins

  • HtmlWebpackPlugin: Simplifica la creació de fitxers HTML per servir els teus paquets. 
    npm install --save-dev html-webpack-plugin

    const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  // ... altres configuracions
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};

Divisió de codi

  • Propòsit: Millorar el temps de càrrega dividint el codi en fragments més petits. 
    module.exports = {
  // ... altres configuracions
  optimization: {
    splitChunks: {
      chunks: 'all'
    }
  }
};

Substitució de mòduls en calent (HMR)

  • Propòsit: Actualitzar els mòduls al navegador en temps d'execució sense una actualització completa. 
    npm install --save-dev webpack-dev-server

    module.exports = {
  // ... altres configuracions
  devServer: {
    contentBase: './dist',
    hot: true
  }
};

Conclusió

  • Integració de Webpack i TypeScript: Proporciona un entorn de desenvolupament robust amb modularització, optimització i compatibilitat.
  • Escalabilitat: Fàcilment extensible amb plugins i configuracions avançades.
  • Eficiència: Agilitza el procés de desenvolupament amb funcions com HMR i divisió de codi.

Seguint aquesta guia, ara hauríeu de tenir una base sòlida per utilitzar Webpack amb TypeScript en els vostres projectes. Continueu explorant funcions i plugins més avançats per millorar encara més el vostre flux de treball de desenvolupament.
