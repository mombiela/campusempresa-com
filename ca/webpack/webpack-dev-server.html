Introducció a Webpack Dev Server

Webpack Dev Server és una eina potent que proporciona un servidor de desenvolupament amb capacitats de recàrrega en viu. Millora significativament l'experiència de desenvolupament refrescant automàticament el navegador cada vegada que fas canvis al teu codi.

Conceptes Clau

  • Recàrrega en Viu: Refresca automàticament el navegador quan es fan canvis.
  • Reemplaçament de Mòduls en Calent (HMR): Actualitza mòduls al navegador sense una recàrrega completa.
  • Proxying: Redirigeix les crides API a un servidor de backend.
  • Servir Fitxers Estàtics: Serveix fitxers estàtics des d'un directori especificat.

Configuració de Webpack Dev Server

Instal·lació

Per utilitzar Webpack Dev Server, cal instal·lar-lo via npm:

npm install webpack-dev-server --save-dev

Configuració Bàsica

Afegeix la següent configuració al teu fitxer webpack.config.js per configurar el servidor de desenvolupament:

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  devServer: {
    contentBase: path.join(__dirname, 'dist'),
    compress: true,
    port: 9000,
  },
};

Explicació

  • entry: Especifica el punt d'entrada de la teva aplicació.
  • output: Defineix el fitxer de sortida i el directori.
  • devServer: Objecte de configuració per a Webpack Dev Server.
    • contentBase: Directori des d'on es serveixen els fitxers estàtics.
    • compress: Habilita la compressió gzip.
    • port: Número de port per al servidor.

Funcions Avançades

Reemplaçament de Mòduls en Calent (HMR)

HMR et permet actualitzar mòduls sense una recàrrega completa del navegador. Per habilitar HMR, modifica el teu webpack.config.js:

const webpack = require('webpack');
const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  devServer: {
    contentBase: path.join(__dirname, 'dist'),
    compress: true,
    port: 9000,
    hot: true,
  },
  plugins: [
    new webpack.HotModuleReplacementPlugin(),
  ],
};

Proxying de Peticions API

Per redirigir les crides API a un servidor de backend, utilitza l'opció proxy:

devServer: {
  contentBase: path.join(__dirname, 'dist'),
  compress: true,
  port: 9000,
  proxy: {
    '/api': 'http://localhost:3000',
  },
}

Servir Fitxers Estàtics

Pots servir fitxers estàtics addicionals especificant múltiples directoris:

devServer: {
  contentBase: [path.join(__dirname, 'dist'), path.join(__dirname, 'public')],
  compress: true,
  port: 9000,
}

Projecte d'Exemple

Aquí tens una estructura de projecte simple:

my-app/
├── dist/
│   └── index.html
├── src/
│   └── index.js
├── package.json
└── webpack.config.js

`index.html`

<!DOCTYPE html>
<html lang="ca">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Exemple de Webpack Dev Server</title>
</head>
<body>
  <div id="app"></div>
  <script src="bundle.js"></script>
</body>
</html>

`index.js`

console.log('Hola, Webpack Dev Server!');

Executar el Servidor de Desenvolupament

Afegeix un script al teu package.json per iniciar el servidor de desenvolupament:

"scripts": {
  "start": "webpack serve --open"
}

Executa la següent comanda per iniciar el servidor:

npm start

Conclusió

Webpack Dev Server és una eina essencial per al desenvolupament web modern, proporcionant funcions com la recàrrega en viu, HMR i proxying. Integrant-lo al teu flux de treball, pots accelerar significativament el desenvolupament i millorar la productivitat.
