En aquest tema, explorarem com crear una aplicació d'una sola pàgina (SPA) complexa utilitzant Webpack. Això cobrirà tot, des de la configuració bàsica de Webpack fins a l'optimització de la construcció per a producció. Al final d'aquest tema, tindràs una comprensió sòlida de com aprofitar Webpack per gestionar i optimitzar la teva SPA.

Introducció a les Aplicacions d'una Sola Pàgina (SPAs)

  • Definició: Les SPAs són aplicacions web que carreguen una sola pàgina HTML i actualitzen dinàmicament el contingut a mesura que l'usuari interactua amb l'aplicació.
  • Avantatges:
    • Temps de càrrega més ràpids després de la càrrega inicial.
    • Millora de l'experiència de l'usuari amb una navegació sense interrupcions.
    • Reducció de la càrrega del servidor.

Configuració de Webpack per a una SPA

Instal·lació de Webpack i Dependències

Per començar, necessites instal·lar Webpack i les seves dependències. Crea un nou directori de projecte i inicialitza'l amb npm:

mkdir my-spa
cd my-spa
npm init -y
npm install webpack webpack-cli --save-dev

Configuració Bàsica de Webpack

Crea un fitxer webpack.config.js a l'arrel del teu projecte:

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  mode: 'development',
};
  • Entry: Especifica el punt d'entrada de la teva aplicació.
  • Output: Defineix el fitxer de sortida i el directori.
  • Mode: Estableix el mode a 'development' per facilitar el depurament.

Afegint Loaders i Plugins

Loaders

Els loaders permeten a Webpack processar diferents tipus de fitxers. Per exemple, per gestionar fitxers CSS, necessites css-loader i style-loader:

npm install css-loader style-loader --save-dev

Actualitza el teu webpack.config.js per incloure aquests loaders:

module.exports = {
  // ... configuració anterior
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
};

Plugins

Els plugins amplien la funcionalitat de Webpack. Per exemple, HtmlWebpackPlugin genera un fitxer HTML que inclou el teu bundle:

npm install html-webpack-plugin --save-dev

Actualitza el teu webpack.config.js per incloure aquest plugin:

const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  // ... configuració anterior
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
    }),
  ],
};

Configuració del Servidor de Desenvolupament

Webpack Dev Server proporciona recàrrega en viu i un servidor de desenvolupament:

npm install webpack-dev-server --save-dev

Actualitza el teu webpack.config.js per configurar el servidor de desenvolupament:

module.exports = {
  // ... configuració anterior
  devServer: {
    contentBase: './dist',
    hot: true,
  },
};

Afegeix un script al teu package.json per iniciar el servidor de desenvolupament:

"scripts": {
  "start": "webpack serve --open"
}

Divisió de Codi i Càrrega Per Mandat

La divisió de codi ajuda a carregar només el codi necessari per a la pàgina actual, millorant el rendiment.

Imports Dinàmics

Utilitza imports dinàmics per dividir el codi:

// src/index.js
import('./module').then(module => {
  module.default();
});

Actualitza el teu webpack.config.js per habilitar la divisió de codi:

module.exports = {
  // ... configuració anterior
  optimization: {
    splitChunks: {
      chunks: 'all',
    },
  },
};

Optimització per a Producció

Minificació i Tree Shaking

La minificació redueix la mida dels teus fitxers JavaScript, i el tree shaking elimina el codi no utilitzat.

Instal·la els plugins necessaris:

npm install terser-webpack-plugin --save-dev

Actualitza el teu webpack.config.js:

const TerserPlugin = require('terser-webpack-plugin');

module.exports = {
  // ... configuració anterior
  mode: 'production',
  optimization: {
    minimize: true,
    minimizer: [new TerserPlugin()],
  },
};

Variables d'Entorn

Utilitza variables d'entorn per gestionar diferents configuracions per a desenvolupament i producció:

npm install dotenv-webpack --save-dev

Actualitza el teu webpack.config.js:

const Dotenv = require('dotenv-webpack');

module.exports = {
  // ... configuració anterior
  plugins: [
    new Dotenv(),
    // altres plugins
  ],
};
