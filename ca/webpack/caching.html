La memòria cau a Webpack
Sacsejada d'arbre ►

La memòria cau és un concepte crucial en el desenvolupament web que ajuda a millorar el rendiment de les teves aplicacions web emmagatzemant dades accedides freqüentment. En aquesta secció, explorarem com Webpack gestiona la memòria cau, des de tècniques bàsiques fins a avançades.

Introducció a la memòria cau

  • Definició: La memòria cau és el procés d'emmagatzemar còpies d'arxius o dades en una memòria cau, o ubicació d'emmagatzematge temporal, perquè es puguin accedir més ràpidament.
  • Importància: Redueix els temps de càrrega, disminueix la càrrega del servidor i millora l'experiència de l'usuari.

Memòria cau bàsica amb Webpack

Noms dels arxius de sortida

Per aprofitar la memòria cau, has d'assegurar-te que els noms dels arxius de sortida canviïn sempre que canviï el contingut. Això es pot fer utilitzant hashes de contingut.

module.exports = {
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
};
  • [name]: El nom del punt d'entrada.
  • [contenthash]: Un hash únic basat en el contingut de l'arxiu.

Exemple

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
};

Aquesta configuració assegura que els noms dels arxius de sortida canviïn sempre que canviï el contingut, ajudant els navegadors a emmagatzemar en memòria cau els arxius de manera efectiva.

Tècniques avançades de memòria cau

SplitChunksPlugin

El SplitChunksPlugin de Webpack es pot utilitzar per dividir el teu codi en fragments més petits, que es poden emmagatzemar en memòria cau per separat.

module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
    },
  },
};

Exemple

const path = require('path');

module.exports = {
  entry: {
    app: './src/index.js',
  },
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
  optimization: {
    splitChunks: {
      chunks: 'all',
    },
  },
};

Grups de memòria cau

Pots personalitzar encara més el comportament del SplitChunksPlugin utilitzant grups de memòria cau.

module.exports = {
  optimization: {
    splitChunks: {
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};

Exemple

const path = require('path');

module.exports = {
  entry: {
    app: './src/index.js',
  },
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
  optimization: {
    splitChunks: {
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
};

Gestió de la memòria cau a llarg termini

Identificadors de mòduls

Webpack utilitza identificadors de mòduls per fer un seguiment dels mòduls. Per a la memòria cau a llarg termini, pots utilitzar el HashedModuleIdsPlugin per assegurar-te que els identificadors de mòduls siguin consistents entre compilacions.

const webpack = require('webpack');

module.exports = {
  plugins: [
    new webpack.HashedModuleIdsPlugin(),
  ],
};

Exemple

const path = require('path');
const webpack = require('webpack');

module.exports = {
  entry: {
    app: './src/index.js',
  },
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve(__dirname, 'dist'),
  },
  optimization: {
    splitChunks: {
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
  plugins: [
    new webpack.HashedModuleIdsPlugin(),
  ],
};

Resum

La memòria cau és una tècnica poderosa per millorar el rendiment de les aplicacions web. Utilitzant les característiques de Webpack com els hashes de contingut, SplitChunksPlugin i HashedModuleIdsPlugin, pots gestionar efectivament la memòria cau i assegurar-te que els teus usuaris tinguin una experiència ràpida i fluida.

  • Memòria cau bàsica: Utilitza hashes de contingut en els noms dels arxius.
  • Memòria cau avançada: Utilitza SplitChunksPlugin i grups de memòria cau.
  • Memòria cau a llarg termini: Utilitza HashedModuleIdsPlugin per a identificadors de mòduls consistents.

Dominant aquestes tècniques, pots millorar significativament el rendiment i la fiabilitat de les teves aplicacions web.
