En aquesta secció, explorarem com carregar fitxers JavaScript utilitzant Webpack. Aquesta és una habilitat fonamental per a qualsevol persona que vulgui empaquetar i optimitzar el seu codi JavaScript per a producció.

Introducció a Webpack

Webpack és un potent empaquetador de mòduls per a aplicacions JavaScript. Pren mòduls amb dependències i genera actius estàtics que representen aquests mòduls. Webpack pot gestionar diversos tipus d'actius, incloent JavaScript, CSS i imatges.

Configurant Webpack

Abans d'endinsar-nos en la càrrega de fitxers JavaScript, configurem una configuració bàsica de Webpack.

  1. Instal·lar Webpack i Webpack CLI:

    npm install --save-dev webpack webpack-cli

  2. Crear un fitxer de configuració bàsic: Crea un fitxer anomenat webpack.config.js a l'arrel del teu projecte:

    const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  mode: 'development',
};

  3. Crear el punt d'entrada: Crea un directori anomenat src i dins d'aquest, crea un fitxer anomenat index.js:

    console.log('Hello, Webpack!');

  4. Executar Webpack: Afegeix un script al teu package.json per executar Webpack:

    "scripts": {
  "build": "webpack"
}

    Llavors executa:

    npm run build

Carregant fitxers JavaScript

Punts d'entrada

El punt d'entrada és el primer fitxer que Webpack mira per construir el gràfic de dependències. En la nostra configuració bàsica, hem utilitzat ./src/index.js com a punt d'entrada.

  • Punt d'entrada únic:

    module.exports = {
  entry: './src/index.js',
};

  • Punts d'entrada múltiples:

    module.exports = {
  entry: {
    app: './src/app.js',
    admin: './src/admin.js',
  },
};

Configuració de sortida

La propietat output diu a Webpack on emetre els paquets que crea i com anomenar-los.

  • Sortida única:

    module.exports = {
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
};

  • Sortides múltiples:

    module.exports = {
  output: {
    filename: '[name].bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
};

Carregadors

Els carregadors permeten a Webpack processar altres tipus de fitxers i convertir-los en mòduls vàlids. Per als fitxers JavaScript, normalment utilitzem el babel-loader per transpilar el codi ES6+ a ES5.

  1. Instal·lar Babel Loader:

    npm install --save-dev babel-loader @babel/core @babel/preset-env

  2. Configurar Babel Loader: Actualitza webpack.config.js per incloure el carregador:

    module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env'],
          },
        },
      },
    ],
  },
};

Plugins

Els plugins amplien les capacitats de Webpack. Per exemple, el HtmlWebpackPlugin pot generar un fitxer HTML que inclou tots els teus paquets.

  1. Instal·lar HtmlWebpackPlugin:

    npm install --save-dev html-webpack-plugin

  2. Configurar HtmlWebpackPlugin: Actualitza webpack.config.js per incloure el plugin:

    const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
    }),
  ],
};

Conclusió

Carregar fitxers JavaScript amb Webpack implica configurar punts d'entrada, configurar la sortida, utilitzar carregadors per processar fitxers i aprofitar plugins per ampliar la funcionalitat. Dominant aquests conceptes, pots empaquetar i optimitzar eficientment el teu codi JavaScript per a qualsevol aplicació web.

En les properes seccions, aprofundirem en funcions avançades de Webpack com la divisió de codi, la càrrega mandrosa i l'optimització del procés de construcció per a producció.
