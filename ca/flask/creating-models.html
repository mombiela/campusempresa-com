En aquesta secció, explorarem com crear models a Flask. Els models són una part crucial de qualsevol aplicació web, ja que representen l'estructura de dades i la lògica de negoci. Començarem amb els conceptes bàsics i gradualment passarem a conceptes més avançats.

Introducció als Models

Els models a Flask es creen típicament utilitzant SQLAlchemy, un potent Object Relational Mapper (ORM) que et permet interactuar amb la base de dades utilitzant objectes de Python.

Conceptes Clau

  • ORM (Object Relational Mapper): Una tècnica que et permet consultar i manipular dades d'una base de dades utilitzant un paradigma orientat a objectes.
  • SQLAlchemy: Un ORM popular per a Python que proporciona un conjunt complet de patrons de persistència a nivell empresarial ben coneguts.

Configuració de SQLAlchemy

Abans de crear models, necessites configurar SQLAlchemy a la teva aplicació Flask.

Instal·lació

Primer, instal·la Flask-SQLAlchemy utilitzant pip:

pip install Flask-SQLAlchemy

Configuració

A continuació, configura SQLAlchemy a la teva aplicació Flask:

from flask import Flask
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///site.db'
db = SQLAlchemy(app)
  • SQLALCHEMY_DATABASE_URI: Aquesta clau de configuració estableix la URI de la base de dades que s'ha d'utilitzar per a la connexió. En aquest exemple, estem utilitzant SQLite.

Creació d'un Model Bàsic

Creem un model simple per a un User:

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(20), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)
    password = db.Column(db.String(60), nullable=False)

    def __repr__(self):
        return f"User('{self.username}', '{self.email}')"

Explicació

  • db.Model: Aquesta és la classe base per a tots els models a SQLAlchemy.
  • db.Column: Defineix una columna a la base de dades.
  • db.Integer, db.String: Especifica el tipus de dades que la columna contindrà.
  • primary_key=True: Marca aquesta columna com la clau primària.
  • unique=True: Assegura que tots els valors en aquesta columna siguin únics.
  • nullable=False: Assegura que aquesta columna no pugui estar buida.

Creació de Relacions

Els models sovint necessiten relacionar-se entre ells. Creem un model Post que es relaciona amb el model User:

class Post(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(100), nullable=False)
    date_posted = db.Column(db.DateTime, nullable=False, default=datetime.utcnow)
    content = db.Column(db.Text, nullable=False)
    user_id = db.Column(db.Integer, db.ForeignKey('user.id'), nullable=False)

    def __repr__(self):
        return f"Post('{self.title}', '{self.date_posted}')"

Explicació

  • db.ForeignKey('user.id'): Això crea una restricció de clau forana, enllaçant la columna user_id en el model Post amb la columna id en el model User.

    def set_password(self, password):
        self.password = generate_password_hash(password)
