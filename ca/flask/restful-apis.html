Introducció a les API RESTful

  • Definició: REST (Representational State Transfer) és un estil arquitectònic per dissenyar aplicacions en xarxa.
  • Conceptes Clau:
    • Recursos: Objectes o representacions de dades que es poden accedir a través d'una URL.
    • Mètodes HTTP: GET, POST, PUT, DELETE, etc., utilitzats per realitzar operacions CRUD.
    • Estatelessness: Cada sol·licitud d'un client al servidor ha de contenir tota la informació que el servidor necessita per complir amb aquesta sol·licitud.
    • JSON: Format comú per enviar i rebre dades en API RESTful.

Configuració de Flask per a API RESTful

  • Instal·lació: 
    pip install Flask
pip install Flask-RESTful
  • Aplicació bàsica de Flask: 
    from flask import Flask
app = Flask(__name__)

@app.route('/')
def home():
    return "Hola, Flask!"

if __name__ == '__main__':
    app.run(debug=True)

Creació d'una API RESTful senzilla

  • Creació de Punts Finals: 
    from flask import Flask, jsonify, request
app = Flask(__name__)

@app.route('/api/v1/resource', methods=['GET'])
def get_resource():
    data = {"message": "Hola, aquest és el teu recurs"}
    return jsonify(data)

if __name__ == '__main__':
    app.run(debug=True)
  • Explicació:
    • @app.route('/api/v1/resource', methods=['GET']): Defineix el punt final i el mètode HTTP.
    • jsonify(data): Converteix el diccionari de Python en una resposta JSON.

Gestió d'Operacions CRUD

  • Exemple amb Dades en Memòria: 
    from flask import Flask, jsonify, request

app = Flask(__name__)

resources = []

@app.route('/api/v1/resources', methods=['GET'])
def get_resources():
    return jsonify(resources)

@app.route('/api/v1/resources', methods=['POST'])
def add_resource():
    new_resource = request.get_json()
    resources.append(new_resource)
    return jsonify(new_resource), 201

@app.route('/api/v1/resources/<int:id>', methods=['PUT'])
def update_resource(id):
    updated_resource = request.get_json()
    resources[id] = updated_resource
    return jsonify(updated_resource)

@app.route('/api/v1/resources/<int:id>', methods=['DELETE'])
def delete_resource(id):
    resources.pop(id)
    return '', 204

if __name__ == '__main__':
    app.run(debug=True)
  • Explicació:
    • request.get_json(): Analitza les dades de la sol·licitud JSON entrant.
    • resources.append(new_resource): Afegeix un nou recurs a la llista.
    • resources[id] = updated_resource: Actualitza un recurs existent.
    • resources.pop(id): Elimina un recurs.

Gestió d'Errors i Validació

  • Missatges d'Error Personalitzats: 
    from flask import Flask, jsonify, request, abort

app = Flask(__name__)

resources = []

@app.route('/api/v1/resources/<int:id>', methods=['GET'])
def get_resource(id):
    if id >= len(resources) or id < 0:
        abort(404, description="Recurs no trobat")
    return jsonify(resources[id])

@app.errorhandler(404)
def resource_not_found(e):
    return jsonify(error=str(e)), 404

if __name__ == '__main__':
    app.run(debug=True)
  • Explicació:
    • abort(404, description="Recurs no trobat"): Aborta la sol·licitud amb un codi d'estat 404 i un missatge personalitzat.
    • @app.errorhandler(404): Gestor d'errors personalitzat per a errors 404.

Temes Avançats

  • Autenticació i Autorització:
    • JWT (JSON Web Tokens): 
      from flask import Flask, jsonify, request
import jwt
import datetime

app = Flask(__name__)
app.config['SECRET_KEY'] = 'your_secret_key'

@app.route('/login', methods=['POST'])
def login():
    auth_data = request.get_json()
    if auth_data['username'] == 'user' and auth_data['password'] == 'pass':
        token = jwt.encode({'user': auth_data['username'], 'exp': datetime.datetime.utcnow() + datetime.timedelta(minutes=30)}, app.config['SECRET_KEY'])
        return jsonify({'token': token})
    return jsonify({'message': 'Credencials invàlides'}), 401

if __name__ == '__main__':
    app.run(debug=True)
    • Explicació:
      • jwt.encode(): Codifica una càrrega útil en un token JWT.
      • app.config['SECRET_KEY']: Clau secreta per codificar i descodificar tokens JWT.

Conclusió

  • Resum:
    • Les API RESTful són una manera poderosa de dissenyar aplicacions en xarxa.
    • Flask proporciona un marc senzill i flexible per construir API RESTful.
    • Els conceptes clau inclouen recursos, mètodes HTTP i estatelessness.
    • La gestió d'operacions CRUD, la gestió d'errors i l'autenticació són essencials per construir API robustes.
