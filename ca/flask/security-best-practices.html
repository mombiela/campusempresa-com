Introducció

Flask és un framework web lleuger i flexible per a Python, però amb gran flexibilitat ve la responsabilitat d'assegurar que la teva aplicació sigui segura. Aquesta secció cobrirà les millors pràctiques essencials de seguretat per a aplicacions Flask, des de nivells principiants fins a avançats.

Pràctiques Bàsiques de Seguretat

Validació d'Entrades

  • Sanititzar Entrades d'Usuari: Sempre valida i sanititza les entrades d'usuari per prevenir atacs d'injecció. 
    from flask import request, escape

@app.route('/submit', methods=['POST'])
def submit():
    user_input = request.form['user_input']
    sanitized_input = escape(user_input)
    return f"Processed input: {sanitized_input}"

Configuració Segura

  • Variables d'Entorn: Emmagatzema informació sensible com claus secretes i credencials de bases de dades en variables d'entorn.

    import os

app.config['SECRET_KEY'] = os.environ.get('SECRET_KEY', 'default_secret_key')

  • Mode Debug: Mai executis la teva aplicació en mode debug en producció.

    if __name__ == '__main__':
    app.run(debug=False)

HTTPS

  • Utilitza HTTPS: Assegura't que la teva aplicació es serveixi sobre HTTPS per encriptar les dades en trànsit. 
    from flask import Flask, redirect, request

app = Flask(__name__)

@app.before_request
def before_request():
    if not request.is_secure:
        return redirect(request.url.replace("http://", "https://"))

Pràctiques Intermèdies de Seguretat

Protecció contra Cross-Site Request Forgery (CSRF)

  • Tokens CSRF: Utilitza tokens CSRF per protegir contra atacs CSRF. 
    from flask_wtf.csrf import CSRFProtect

csrf = CSRFProtect(app)

Gestió de Sessions

  • Cookies Segures: Configura cookies segures per prevenir el segrest de sessions. 
    app.config.update(
    SESSION_COOKIE_HTTPONLY=True,
    SESSION_COOKIE_SECURE=True,
)

Autenticació i Autorització

  • Flask-Login: Utilitza Flask-Login per a la gestió de sessions d'usuari. 
    from flask_login import LoginManager

login_manager = LoginManager()
login_manager.init_app(app)

Pràctiques Avançades de Seguretat

Política de Seguretat de Contingut (CSP)

  • Capçaleres CSP: Implementa capçaleres CSP per prevenir atacs XSS. 
    @app.after_request
def set_csp(response):
    response.headers['Content-Security-Policy'] = "default-src 'self'"
    return response

Limitació de Ràtio

  • Flask-Limiter: Utilitza Flask-Limiter per protegir contra atacs de força bruta. 
    from flask_limiter import Limiter

limiter = Limiter(app, key_func=get_remote_address)

@app.route('/login')
@limiter.limit("5 per minute")
def login():
    return "Login Page"

Capçaleres Segures

  • Flask-Talisman: Utilitza Flask-Talisman per establir diverses capçaleres HTTP per a la seguretat. 
    from flask_talisman import Talisman

Talisman(app)

Conclusió

Assegurar una aplicació Flask implica múltiples capes de defensa, des de la validació bàsica d'entrades i configuracions segures fins a tècniques avançades com CSP i limitació de ràtio. Seguint aquestes millors pràctiques, pots reduir significativament el risc de vulnerabilitats de seguretat en la teva aplicació. Mantén-te sempre actualitzat amb les últimes tendències de seguretat i monitora contínuament la teva aplicació per a possibles amenaces.
