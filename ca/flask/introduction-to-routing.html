El routing és un concepte fonamental en el desenvolupament web que implica mapar URLs a funcions específiques en la teva aplicació. En Flask, el routing és senzill i flexible, fent-lo una excel·lent elecció tant per a principiants com per a desenvolupadors avançats.

Què és el Routing?

El routing és el procés de determinar quin codi executar quan un client sol·licita una URL específica. En Flask, això s'aconsegueix utilitzant decoradors per vincular URLs a funcions.

Conceptes Clau

  • Ruta: Un patró de URL que es mapeja a una funció específica.
  • Funció de Vista: Una funció que s'executa quan es sol·licita la seva ruta associada.
  • Decorador: Una sintaxi especial en Python utilitzada per modificar el comportament d'una funció o classe.

Configuració de Rutes en Flask

Per configurar rutes en Flask, utilitzes el decorador @app.route. Aquest decorador vincula una URL a una funció de vista.

Exemple de Ruta Bàsica

from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Benvingut a la Pàgina Principal!"

if __name__ == '__main__':
    app.run(debug=True)
  • Explicació:
    • @app.route('/'): Aquest decorador vincula la URL arrel (/) a la funció home.
    • def home(): Aquesta és la funció de vista que retorna una cadena simple quan s'accedeix a la URL arrel.
    • app.run(debug=True): Això inicia l'aplicació Flask en mode debug.

Rutes Dinàmiques

Les rutes dinàmiques et permeten capturar valors de la URL i passar-los a la teva funció de vista.

Exemple de Ruta Dinàmica

@app.route('/user/<username>')
def show_user_profile(username):
    return f"Usuari: {username}"
  • Explicació:
    • <username>: Aquesta part de la URL és dinàmica i capturarà qualsevol valor col·locat en aquest segment de la URL.
    • show_user_profile(username): El valor capturat es passa com a argument a la funció de vista.

Mètodes HTTP

Per defecte, les rutes en Flask responen a sol·licituds GET. No obstant això, pots especificar altres mètodes HTTP com POST, PUT, DELETE, etc.

Gestionant Diferents Mètodes HTTP

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        return "Iniciant sessió..."
    else:
        return "Mostra el formulari d'inici de sessió"
  • Explicació:
    • methods=['GET', 'POST']: Això especifica que la ruta login pot gestionar tant sol·licituds GET com POST.
    • request.method: Això comprova el mètode HTTP utilitzat per a la sol·licitud.

Construcció de URLs

Flask proporciona una funció url_for per construir URLs per a la teva aplicació de manera dinàmica. Això és útil per generar URLs sense codificar-les directament.

Exemple de Construcció de URLs

from flask import url_for

@app.route('/user/<username>')
def show_user_profile(username):
    return f"Usuari: {username}"

@app.route('/profile')
def profile():
    return redirect(url_for('show_user_profile', username='JohnDoe'))
  • Explicació:
    • url_for('show_user_profile', username='JohnDoe'): Això genera la URL per a la ruta show_user_profile amb el nom d'usuari 'JohnDoe'.
    • redirect(url_for(...)): Això redirigeix l'usuari a la URL generada.

Resum

En aquest tema, hem cobert els conceptes bàsics del routing en Flask, incloent:

  • Què és el routing i per què és important.
  • Com configurar rutes bàsiques i dinàmiques.
  • Gestionar diferents mètodes HTTP en rutes.
  • Utilitzar url_for per construir URLs de manera dinàmica.

Entendre el routing és crucial per desenvolupar aplicacions web, ja que et permet controlar el flux de la teva aplicació i respondre adequadament a les sol·licituds dels usuaris. Amb aquests fonaments, pots començar a construir aplicacions web més complexes i interactives utilitzant Flask.
