El Projecte Sobre nosaltres Contribuir Donacions Llicència

Les proves unitàries són una part crucial del desenvolupament d'aplicacions web fiables i mantenibles. En aquesta secció, cobrirem com escriure proves unitàries per a una aplicació Flask, assegurant-nos que el teu codi funcioni com s'espera i pugui manejar diversos casos límit.

Introducció a les Proves Unitàries

  • Definició: Les proves unitàries impliquen provar components o funcions individuals d'una aplicació de manera aïllada per assegurar-se que funcionen correctament.
  • Importància: Ajuda a detectar errors aviat, simplifica la depuració i assegura la fiabilitat del codi.
  • Eines: Utilitzarem el mòdul unittest integrat de Python i les utilitats de prova de Flask.

Configuració de l'Entorn de Proves

Abans d'escriure proves, necessitem configurar el nostre entorn de proves.

  1. Instal·lar Dependències:

    pip install Flask
pip install pytest

  2. Crear una Aplicació Flask:

    # app.py
from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/hello')
def hello():
    return jsonify(message="Hello, World!")

if __name__ == '__main__':
    app.run(debug=True)

  3. Crear un Directori de Proves:

    mkdir tests

Escriure la Teva Primera Prova

Escrivim una prova senzilla per comprovar si el nostre punt final /hello retorna el missatge correcte.

  1. Crear un Fitxer de Prova:

    # tests/test_app.py
import unittest
from app import app

class BasicTests(unittest.TestCase):

    def setUp(self):
        self.app = app.test_client()
        self.app.testing = True

    def test_hello(self):
        response = self.app.get('/hello')
        self.assertEqual(response.status_code, 200)
        self.assertEqual(response.json, {"message": "Hello, World!"})

if __name__ == "__main__":
    unittest.main()

  2. Executar la Prova:

    python -m unittest discover -s tests

Proves amb Pytest

Tot i que unittest és potent, molts desenvolupadors prefereixen pytest per la seva simplicitat i característiques avançades.

  1. Convertir la Prova a Pytest:

    # tests/test_app.py
import pytest
from app import app

@pytest.fixture
def client():
    app.testing = True
    return app.test_client()

def test_hello(client):
    response = client.get('/hello')
    assert response.status_code == 200
    assert response.json == {"message": "Hello, World!"}

  2. Executar la Prova amb Pytest:

    pytest

Provar Casos Límit i Maneig d'Errors

És essencial provar no només el camí feliç sinó també casos límit i escenaris d'error.

  1. Afegir una Nova Ruta amb Maneig d'Errors:

    # app.py
@app.route('/divide')
def divide():
    try:
        dividend = int(request.args.get('dividend'))
        divisor = int(request.args.get('divisor'))
        result = dividend / divisor
        return jsonify(result=result)
    except ZeroDivisionError:
        return jsonify(error="Cannot divide by zero"), 400
    except ValueError:
        return jsonify(error="Invalid input"), 400

  2. Escriure Proves per a Casos Límit:

    # tests/test_app.py
def test_divide_success(client):
    response = client.get('/divide?dividend=10&divisor=2')
    assert response.status_code == 200
    assert response.json == {"result": 5.0}

def test_divide_by_zero(client):
    response = client.get('/divide?dividend=10&divisor=0')
    assert response.status_code == 400
    assert response.json == {"error": "Cannot divide by zero"}

def test_divide_invalid_input(client):
    response = client.get('/divide?dividend=ten&divisor=2')
    assert response.status_code == 400
    assert response.json == {"error": "Invalid input"}

Mocking i Aïllament de Dependències

De vegades, necessites aïllar les teves proves de dependències externes com bases de dades o APIs.

  1. Utilitzant unittest.mock: 
    from unittest.mock import patch

@patch('app.some_external_function')
def test_mocked_function(mock_function, client):
    mock_function.return_value = "mocked result"
    response = client.get('/some-endpoint')
    assert response.json == {"result": "mocked result"}

Conclusió

Les proves unitàries són una pràctica essencial per desenvolupar aplicacions Flask robustes. Configurant un entorn de proves adequat, escrivint proves per a diversos escenaris i utilitzant eines com unittest i pytest, pots assegurar-te que la teva aplicació es comporti com s'espera. Recorda provar tant el camí feliç com els casos límit, i utilitza mocking per aïllar dependències quan sigui necessari. Bones proves!
