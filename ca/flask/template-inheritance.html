Herència de Plantilles a Flask
Treballar amb Fitxers Estàtics ►

L'herència de plantilles és una característica poderosa a Flask que et permet crear una plantilla base i estendre-la en altres plantilles. Això ajuda a mantenir una aparença i sensació consistents a tota la teva aplicació web i fa que el teu codi sigui més modular i fàcil de gestionar.

Conceptes Clau

  • Plantilla Base: Una plantilla que conté l'estructura i els elements comuns (com ara capçalera, peu de pàgina, navegació) que es comparteixen a diverses pàgines.
  • Plantilla Filla: Una plantilla que estèn la plantilla base i substitueix blocs específics per proporcionar contingut únic per a diferents pàgines.
  • Blocs: Seccions dins d'una plantilla que poden ser substituïdes per plantilles filles.

Creant una Plantilla Base

Una plantilla base normalment inclou l'estructura HTML i elements comuns com la capçalera i el peu de pàgina. Aquí tens un exemple d'una plantilla base anomenada base.html:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}My Flask App{% endblock %}</title>
</head>
<body>
    <header>
        <h1>Welcome to My Flask App</h1>
    </header>
    <nav>
        <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/about">About</a></li>
        </ul>
    </nav>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 My Flask App</p>
    </footer>
</body>
</html>

Explicació

  • {% block title %}My Flask App{% endblock %}: Aquest bloc permet que les plantilles filles estableixin el seu propi títol.
  • {% block content %}{% endblock %}: Aquest bloc serà substituït per les plantilles filles per proporcionar contingut únic per a diferents pàgines.

Creant Plantilles Filles

Les plantilles filles estenen la plantilla base i substitueixen blocs específics per proporcionar contingut únic. Aquí tens un exemple d'una plantilla filla anomenada home.html:

{% extends "base.html" %}

{% block title %}Home - My Flask App{% endblock %}

{% block content %}
    <h2>Home Page</h2>
    <p>Welcome to the home page of my Flask app!</p>
{% endblock %}

I una altra plantilla filla anomenada about.html:

{% extends "base.html" %}

{% block title %}About - My Flask App{% endblock %}

{% block content %}
    <h2>About Us</h2>
    <p>This is the about page of my Flask app.</p>
{% endblock %}

Explicació

  • {% extends "base.html" %}: Aquesta línia diu a Flask que aquesta plantilla estèn la plantilla base.html.
  • {% block title %}Home - My Flask App{% endblock %}: Aquest bloc substitueix el bloc de títol a la plantilla base.
  • {% block content %}: Aquest bloc substitueix el bloc de contingut a la plantilla base per proporcionar contingut únic per a les pàgines d'inici i sobre.

Utilitzant Plantilles a les Rutes de Flask

Per renderitzar aquestes plantilles a la teva aplicació Flask, necessites configurar rutes al teu fitxer app.py:

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def home():
    return render_template('home.html')

@app.route('/about')
def about():
    return render_template('about.html')

if __name__ == '__main__':
    app.run(debug=True)

Explicació

  • render_template('home.html'): Aquesta funció renderitza la plantilla home.html.
  • render_template('about.html'): Aquesta funció renderitza la plantilla about.html.

Conclusió

L'herència de plantilles a Flask et permet crear una estructura consistent i fàcil de mantenir per a la teva aplicació web. Utilitzant una plantilla base i estenent-la en plantilles filles, pots gestionar fàcilment elements comuns i contingut únic a diferents pàgines. Aquest enfocament no només estalvia temps, sinó que també fa que el teu codi sigui més modular i fàcil de mantenir.
