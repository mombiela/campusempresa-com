En aquesta secció, explorarem com renderitzar plantilles en Flask, un aspecte crucial per construir aplicacions web dinàmiques. Cobrirem els conceptes bàsics de la renderització de plantilles, com passar dades a les plantilles i tècniques avançades per crear plantilles reutilitzables i mantenibles.

Introducció a les Plantilles

Les plantilles permeten separar la capa de presentació de la lògica de negoci de la teva aplicació. Flask utilitza Jinja2 com a motor de plantilles, que proporciona una manera poderosa de generar HTML dinàmicament.

Conceptes Clau

  • Motor de Plantilles: Un component de programari que combina plantilles amb un model de dades per produir documents.
  • Jinja2: El motor de plantilles per defecte de Flask, que permet la generació de contingut dinàmic.
  • Renderització: El procés de generar l'HTML final a partir de plantilles i dades.

Renderització Bàsica de Plantilles

Per renderitzar una plantilla en Flask, utilitzes la funció render_template. Aquesta funció pren el nom del fitxer de la plantilla i qualsevol dada que vulguis passar-li.

Exemple

  1. Crear un Fitxer de Plantilla: Desa el següent codi HTML en un fitxer anomenat index.html dins d'una carpeta anomenada templates.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{{ title }}</title>
</head>
<body>
    <h1>{{ heading }}</h1>
    <p>{{ message }}</p>
</body>
</html>

  2. Renderitzar la Plantilla en Flask: Utilitza la funció render_template en la teva ruta de Flask.

    from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def home():
    return render_template('index.html', title='Home Page', heading='Welcome to Flask', message='This is a basic template rendering example.')

if __name__ == '__main__':
    app.run(debug=True)

Explicació

  • Variables de Plantilla: {{ title }}, {{ heading }} i {{ message }} són espais reservats en la plantilla que seran reemplaçats amb els valors passats des de la ruta de Flask.
  • Funció render_template: Aquesta funció pren el nom del fitxer de la plantilla i arguments de paraula clau per a les dades que es passaran a la plantilla.

Passar Dades a les Plantilles

Pots passar diversos tipus de dades a les plantilles, incloent-hi cadenes de text, llistes, diccionaris i més.

Exemple amb Llista i Diccionari

  1. Fitxer de Plantilla: Desa el següent codi HTML en un fitxer anomenat data.html.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Data Example</title>
</head>
<body>
    <h1>Users</h1>
    <ul>
        {% for user in users %}
            <li>{{ user.name }} - {{ user.email }}</li>
        {% endfor %}
    </ul>
</body>
</html>

  2. Renderitzar la Plantilla amb Dades:

    @app.route('/users')
def users():
    users = [
        {'name': 'Alice', 'email': 'alice@example.com'},
        {'name': 'Bob', 'email': 'bob@example.com'},
        {'name': 'Charlie', 'email': 'charlie@example.com'}
    ]
    return render_template('data.html', users=users)

Explicació

  • Iteració en Plantilles: {% for user in users %} és un bucle de Jinja2 que itera sobre la llista d'usuaris i renderitza el nom i el correu electrònic de cada usuari.
  • Context de Plantilla: La variable users es passa a la plantilla i es pot accedir dins de la plantilla.

Funcions Avançades de Plantilles

Flask i Jinja2 ofereixen funcions avançades per fer les plantilles més poderoses i mantenibles.

Herència de Plantilles

L'herència de plantilles permet crear una plantilla base que altres plantilles poden estendre. Això és útil per mantenir un disseny consistent a través de múltiples pàgines.

  1. Plantilla Base: Desa el següent codi HTML en un fitxer anomenat base.html.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}My Website{% endblock %}</title>
</head>
<body>
    <header>
        <h1>My Website</h1>
    </header>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 My Website</p>
    </footer>
</body>
</html>

  2. Plantilla Filla: Desa el següent codi HTML en un fitxer anomenat child.html.

    {% extends "base.html" %}

{% block title %}Home Page{% endblock %}

{% block content %}
    <h2>Welcome to the Home Page</h2>
    <p>This is the content of the home page.</p>
{% endblock %}

  3. Renderitzar la Plantilla Filla:

    @app.route('/home')
def home():
    return render_template('child.html')

Explicació

  • Blocs de Plantilla: {% block title %} i {% block content %} defineixen seccions de la plantilla que poden ser sobreescrites per plantilles filles.
  • Extensió de Plantilles: {% extends "base.html" %} indica que child.html estèn base.html.

Conclusió

Renderitzar plantilles en Flask és una manera poderosa de crear pàgines web dinàmiques. Entenent els conceptes bàsics de la renderització de plantilles, passant dades a les plantilles i utilitzant funcions avançades com l'herència de plantilles, pots construir aplicacions web mantenibles i escalables. Practica aquests conceptes amb diferents tipus de dades i plantilles per convertir-te en un expert en la renderització de plantilles amb Flask.
