Introducció als Microserveis

  • Definició: Els microserveis són un estil arquitectònic que estructura una aplicació com una col·lecció de serveis poc acoblats.
  • Beneficis:
    • Escalabilitat
    • Flexibilitat
    • Desplegament independent
    • Millora de l'aïllament de fallades

Configuració de Flask per a Microserveis

  • Instal·lació: 
    pip install Flask
  • Aplicació bàsica de Flask: 
    from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hola, Microserveis!"

if __name__ == '__main__':
    app.run(debug=True)

Disseny de Microserveis

  • Límits del servei: Defineix límits clars per a cada microservei.
  • Comunicació: Utilitza APIs RESTful per a la comunicació entre serveis.
  • Gestió de dades: Cada microservei hauria de gestionar la seva pròpia base de dades.

Construcció d'un Microservei Simple

  • Creació d'un servei d'usuaris: 
    from flask import Flask, jsonify, request

app = Flask(__name__)

users = []

@app.route('/users', methods=['GET'])
def get_users():
    return jsonify(users)

@app.route('/users', methods=['POST'])
def add_user():
    user = request.json
    users.append(user)
    return jsonify(user), 201

if __name__ == '__main__':
    app.run(debug=True)
  • Explicació:
    • @app.route('/users', methods=['GET']): Defineix una ruta per obtenir tots els usuaris.
    • @app.route('/users', methods=['POST']): Defineix una ruta per afegir un nou usuari.
    • request.json: Recupera dades JSON del cos de la sol·licitud.

Comunicació entre Serveis

  • APIs RESTful: Utilitza mètodes HTTP (GET, POST, PUT, DELETE) per a la comunicació.
  • Exemple: 
    import requests

response = requests.get('http://user-service/users')
users = response.json()
print(users)

Gestió d'Errors i Registre

  • Gestió d'Errors: 
    @app.errorhandler(404)
def not_found(error):
    return jsonify({'error': 'No trobat'}), 404
  • Registre: 
    import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

@app.route('/users', methods=['POST'])
def add_user():
    user = request.json
    users.append(user)
    logger.info(f'Usuari afegit: {user}')
    return jsonify(user), 201

Proves de Microserveis

  • Proves Unitàries: 
    import unittest

class UserServiceTest(unittest.TestCase):
    def test_add_user(self):
        with app.test_client() as client:
            response = client.post('/users', json={'name': 'John'})
            self.assertEqual(response.status_code, 201)
            self.assertEqual(response.json, {'name': 'John'})

if __name__ == '__main__':
    unittest.main()
  • Proves d'Integració: Prova la interacció entre múltiples serveis.

Estratègies de Desplegament

  • Docker: Contenitza els teus microserveis. 
    FROM python:3.8-slim

WORKDIR /app

COPY requirements.txt requirements.txt
RUN pip install -r requirements.txt

COPY . .

CMD ["python", "app.py"]
  • Kubernetes: Orquestra els teus microserveis.

Conclusió

Els microserveis amb Flask permeten serveis escalables, flexibles i desplegables de manera independent. Entenent les bases de Flask, dissenyant límits clars de servei, gestionant la comunicació entre serveis i implementant una correcta gestió d'errors i registre, pots construir microserveis robustos. Les estratègies de prova i desplegament asseguren encara més la fiabilitat i escalabilitat de les teves aplicacions.
