Introducció a les Proves i la Garantia de Qualitat

Les proves i la Garantia de Qualitat (QA) són components crítics del desenvolupament de programari, assegurant que el teu codi VBA sigui fiable, eficient i lliure d'errors. Aquesta secció et guiarà a través dels fonaments de les proves i la QA en VBA, des de conceptes bàsics fins a tècniques avançades.

Conceptes Clau en Proves i QA

  • Proves Unitàries: Provar unitats o components individuals del codi per assegurar-se que funcionen com s'espera.
  • Proves d'Integració: Provar la interacció entre diferents unitats o components.
  • Proves de Regressió: Assegurar-se que els nous canvis de codi no afectin negativament la funcionalitat existent.
  • Depuració: Identificar i corregir errors o fallades en el codi.
  • Revisió de Codi: Examen sistemàtic del codi per part dels companys per identificar possibles problemes i millores.

Configuració del Teu Entorn per a les Proves

Abans d'entrar en les proves, és essencial configurar correctament el teu entorn VBA.

  1. Activa la Pestanya Desenvolupador:

    • Obre Excel.
    • Ves a Fitxer > Opcions.
    • Selecciona Personalitza la Cinta.
    • Marca l'opció Desenvolupador i fes clic a D'acord.

  2. Accedeix a l'Editor VBA:

    • Prem Alt + F11 per obrir l'Editor VBA.

  3. Crea un Mòdul per a les Proves:

    • A l'Editor VBA, ves a Insereix > Mòdul.

Escriure Proves Unitàries en VBA

Les proves unitàries són proves petites i aïllades que comproven la funcionalitat de parts específiques del teu codi.

Exemple: Provar una Funció Simple

Escrivim una prova unitària per a una funció simple que suma dos números.

' Funció a provar
Function AddNumbers(a As Double, b As Double) As Double
    AddNumbers = a + b
End Function

' Prova unitària per a la funció AddNumbers
Sub TestAddNumbers()
    Dim result As Double
    result = AddNumbers(2, 3)
    
    If result = 5 Then
        Debug.Print "TestAddNumbers Passed"
    Else
        Debug.Print "TestAddNumbers Failed"
    End If
End Sub
  • Explicació:
    • AddNumbers és una funció que pren dos números i retorna la seva suma.
    • TestAddNumbers és una subrutina que prova la funció AddNumbers. Comprova si el resultat és l'esperat (5 en aquest cas) i imprimeix el resultat de la prova.

Tècniques de Depuració

La depuració és una habilitat essencial per identificar i corregir problemes en el teu codi.

Ús de Punts de Trencament

  • Configura punts de trencament fent clic al marge al costat de la línia de codi a l'Editor VBA.
  • Executa el teu codi, i es pararà al punt de trencament, permetent-te inspeccionar variables i avançar pas a pas pel codi.

Finestra Immediata

  • Utilitza la Finestra Immediata (Ctrl + G) per executar fragments de codi i inspeccionar variables durant la depuració.

Exemple: Depurar amb Punts de Trencament

Sub DebugExample()
    Dim x As Integer
    Dim y As Integer
    Dim result As Integer
    
    x = 5
    y = 10
    result = AddNumbers(x, y) ' Configura un punt de trencament aquí
    
    Debug.Print result
End Sub

Millors Pràctiques de Revisió de Codi

Les revisions de codi ajuden a millorar la qualitat del codi detectant problemes aviat i compartint coneixements entre els membres de l'equip.

  • Revisió per Companys: Fes que un altre desenvolupador revisi el teu codi.
  • Eines Automatitzades: Utilitza eines com Rubberduck VBA per a l'anàlisi estàtica del codi.
  • Llistes de Verificació: Utilitza una llista de verificació per assegurar la consistència i la completesa en les revisions.

Tècniques Avançades de Proves

Per a projectes més complexos, considera utilitzar tècniques avançades de proves.

Mocking i Stubbing

  • Mocking: Crear objectes simulats per imitar el comportament d'objectes reals.
  • Stubbing: Substituir parts del codi amb implementacions simplificades per a proves.

Exemple: Mocking d'una Connexió a Base de Dades

' Connexió a base de dades simulada per a proves
Function MockDatabaseConnection() As Object
    Dim mockConn As Object
    Set mockConn = CreateObject("Scripting.Dictionary")
    mockConn.Add "Connected", True
    Set MockDatabaseConnection = mockConn
End Function

' Prova utilitzant la connexió a base de dades simulada
Sub TestDatabaseConnection()
    Dim dbConn As Object
    Set dbConn = MockDatabaseConnection()
    
    If dbConn("Connected") = True Then
        Debug.Print "TestDatabaseConnection Passed"
    Else
        Debug.Print "TestDatabaseConnection Failed"
    End If
End Sub

Conclusió

Les proves i la Garantia de Qualitat són vitals per desenvolupar aplicacions VBA robustes i fiables. Incorporant proves unitàries, tècniques de depuració, revisions de codi i mètodes avançats de proves, pots assegurar-te que el teu codi és d'alta qualitat i fàcil de mantenir. Les pràctiques regulars de proves i QA t'estalviaran temps i esforç a llarg termini, conduint a projectes més reeixits.
