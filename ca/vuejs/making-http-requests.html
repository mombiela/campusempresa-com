En aquesta secció, tractarem com fer sol·licituds HTTP a Vue.js. Aquesta és una habilitat crucial per interactuar amb APIs i obtenir dades per a les teves aplicacions. Començarem amb els conceptes bàsics i avançarem cap a conceptes més avançats.

Introducció a les sol·licituds HTTP a Vue.js

Les sol·licituds HTTP s'utilitzen per comunicar-se amb servidors per obtenir o enviar dades. A Vue.js, sovint utilitzem biblioteques com Axios o l'API Fetch per gestionar aquestes sol·licituds.

Conceptes clau

  • Mètodes HTTP: GET, POST, PUT, DELETE, etc.
  • Endpoints: URLs on s'envien les sol·licituds.
  • Headers: Informació addicional enviada amb la sol·licitud.
  • Body: Dades enviades amb sol·licituds POST, PUT, DELETE.

Utilitzant Axios per a sol·licituds HTTP

Axios és un client HTTP basat en promeses per a JavaScript, que funciona perfectament amb Vue.js.

Instal·lant Axios

Primer, has d'instal·lar Axios en el teu projecte Vue.js:

npm install axios

Fent una sol·licitud GET

Aquí tens com fer una sol·licitud GET senzilla per obtenir dades d'una API:

<template>
  <div>
    <h1>Dades de l'API</h1>
    <ul>
      <li v-for="item in items" :key="item.id">{{ item.name }}</li>
    </ul>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  data() {
    return {
      items: []
    };
  },
  created() {
    axios.get('https://api.example.com/items')
      .then(response => {
        this.items = response.data;
      })
      .catch(error => {
        console.error('Hi ha hagut un error!', error);
      });
  }
};
</script>

Fent una sol·licitud POST

Per enviar dades a una API, pots utilitzar una sol·licitud POST:

<template>
  <div>
    <h1>Afegeix un nou element</h1>
    <form @submit.prevent="addItem">
      <input v-model="newItem" placeholder="Introdueix el nom de l'element" />
      <button type="submit">Afegeix</button>
    </form>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  data() {
    return {
      newItem: ''
    };
  },
  methods: {
    addItem() {
      axios.post('https://api.example.com/items', { name: this.newItem })
        .then(response => {
          console.log('Element afegit:', response.data);
        })
        .catch(error => {
          console.error('Hi ha hagut un error!', error);
        });
    }
  }
};
</script>

Gestionant errors

És important gestionar els errors de manera adequada en la teva aplicació. Axios proporciona una manera de capturar errors utilitzant .catch().

axios.get('https://api.example.com/items')
  .then(response => {
    this.items = response.data;
  })
  .catch(error => {
    console.error('Hi ha hagut un error!', error);
    this.errorMessage = 'No s\'han pogut carregar les dades';
  });

Utilitzant l'API Fetch

L'API Fetch és una API de JavaScript integrada per fer sol·licituds HTTP. Retorna una promesa que es resol amb l'objecte Response.

Fent una sol·licitud GET amb Fetch

<template>
  <div>
    <h1>Dades de l'API</h1>
    <ul>
      <li v-for="item in items" :key="item.id">{{ item.name }}</li>
    </ul>
  </div>
</template>

<script>
export default {
  data() {
    return {
      items: []
    };
  },
  created() {
    fetch('https://api.example.com/items')
      .then(response => response.json())
      .then(data => {
        this.items = data;
      })
      .catch(error => {
        console.error('Hi ha hagut un error!', error);
      });
  }
};
</script>

Fent una sol·licitud POST amb Fetch

<template>
  <div>
    <h1>Afegeix un nou element</h1>
    <form @submit.prevent="addItem">
      <input v-model="newItem" placeholder="Introdueix el nom de l'element" />
      <button type="submit">Afegeix</button>
    </form>
  </div>
</template>

<script>
export default {
  data() {
    return {
      newItem: ''
    };
  },
  methods: {
    addItem() {
      fetch('https://api.example.com/items', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({ name: this.newItem })
      })
      .then(response => response.json())
      .then(data => {
        console.log('Element afegit:', data);
      })
      .catch(error => {
        console.error('Hi ha hagut un error!', error);
      });
    }
  }
};
</script>

Conceptes avançats

Interceptors a Axios

Els interceptors et permeten executar el teu codi o modificar la sol·licitud/resposta abans que sigui gestionada per then o catch.

axios.interceptors.request.use(config => {
  // Fes alguna cosa abans que la sol·licitud sigui enviada
  config.headers.Authorization = 'Bearer token';
  return config;
}, error => {
  // Fes alguna cosa amb l'error de la sol·licitud
  return Promise.reject(error);
});

axios.interceptors.response.use(response => {
  // Fes alguna cosa amb les dades de la resposta
  return response;
}, error => {
  // Fes alguna cosa amb l'error de la resposta
  return Promise.reject(error);
});

Utilitzant Async/Await

La sintaxi async/await pot fer que el teu codi sigui més net i fàcil de llegir.

<template>
  <div>
    <h1>Dades de l'API</h1>
    <ul>
      <li v-for="item in items" :key="item.id">{{ item.name }}</li>
    </ul>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  data() {
    return {
      items: []
    };
  },
  async created() {
    try {
      const response = await axios.get('https://api.example.com/items');
      this.items = response.data;
    } catch (error) {
      console.error('Hi ha hagut un error!', error);
    }
  }
};
</script>
