Introducció a les Guàrdies de Navegació

Les Guàrdies de Navegació són una característica essencial en Vue.js que et permeten controlar la navegació de la teva aplicació. Proporcionen hooks que es poden utilitzar per protegir rutes, realitzar accions abans o després de la navegació, i fins i tot cancel·lar la navegació basant-se en certes condicions.

Tipus de Guàrdies de Navegació

Vue.js ofereix diversos tipus de guàrdies de navegació:

  • Guàrdies Globals: S'apliquen a totes les rutes de l'aplicació.
  • Guàrdies per Ruta: S'apliquen a rutes específiques.
  • Guàrdies en Components: S'apliquen a components específics.

Guàrdies Globals

Les guàrdies globals es defineixen a nivell del router i s'apliquen a totes les rutes.

Guàrdia `beforeEach`

La guàrdia beforeEach es crida abans de cada navegació. Es pot utilitzar per realitzar comprovacions o obtenir dades abans de permetre que la navegació continuï.

const router = new VueRouter({
  routes: [
    // les teves rutes aquí
  ]
});

router.beforeEach((to, from, next) => {
  // Realitza alguna lògica
  if (to.meta.requiresAuth && !isAuthenticated()) {
    next('/login');
  } else {
    next();
  }
});
  • to: L'objecte Ruta de destinació al qual s'està navegant.
  • from: La ruta actual de la qual s'està navegant.
  • next: Una funció que s'ha de cridar per resoldre el hook.

Guàrdia `beforeResolve`

La guàrdia beforeResolve és similar a beforeEach però es crida just abans que la navegació es confirmi, després de totes les guàrdies beforeEach i que els components de ruta asíncrons es resolguin.

router.beforeResolve((to, from, next) => {
  // Realitza alguna lògica
  next();
});

Guàrdia `afterEach`

La guàrdia afterEach es crida després de cada navegació. No afecta la navegació en si mateixa i normalment s'utilitza per a tasques d'analítica o neteja.

router.afterEach((to, from) => {
  // Realitza alguna lògica
  console.log(`Navegat a ${to.path} des de ${from.path}`);
});

Guàrdies per Ruta

Les guàrdies per ruta es defineixen directament en la configuració de la ruta i s'apliquen només a rutes específiques.

const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    beforeEnter: (to, from, next) => {
      // Realitza alguna lògica
      si no (isAuthenticated()) {
        next('/login');
      } else {
        next();
      }
    }
  }
];

Guàrdies en Components

Les guàrdies en components es defineixen dins del mateix component i s'apliquen només quan el component està sent navegant cap a o des de.

`beforeRouteEnter`

Es crida abans que la ruta que renderitza aquest component es confirmi. No té accés a this perquè la instància del component encara no s'ha creat.

export default {
  beforeRouteEnter(to, from, next) {
    // Realitza alguna lògica
    next(vm => {
      // Accedeix a la instància del component a través de `vm`
    });
  }
};

`beforeRouteUpdate`

Es crida quan la ruta que renderitza aquest component ha canviat però el component es reutilitza en la nova ruta.

export default {
  beforeRouteUpdate(to, from, next) {
    // Realitza alguna lògica
    next();
  }
};

`beforeRouteLeave`

Es crida quan la ruta que renderitza aquest component està a punt de ser navegada.

export default {
  beforeRouteLeave(to, from, next) {
    // Realitza alguna lògica
    next();
  }
};

Resum

Les Guàrdies de Navegació en Vue.js proporcionen una manera poderosa de controlar el flux de la teva aplicació. Utilitzant guàrdies globals, per ruta i en components, pots assegurar-te que la teva aplicació es comporti com s'espera durant la navegació. Ja sigui que necessitis comprovar l'autenticació, obtenir dades o realitzar tasques de neteja, les guàrdies de navegació ofereixen una solució flexible per manejar aquests escenaris de manera efectiva.
