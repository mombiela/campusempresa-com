En aquesta secció, cobrirem com preparar la teva aplicació Vue.js per a producció. Això inclou optimitzar la teva aplicació per al rendiment, configurar les teves eines de construcció i desplegar la teva aplicació a un servidor en viu.

Conceptes Clau

  • Minificació i Agrupació: Reduir la mida dels teus fitxers JavaScript i CSS.
  • Variables d'Entorn: Configurar diferents paràmetres per a desenvolupament i producció.
  • Divisió de Codi: Carregar només el codi necessari per a cada pàgina.
  • Caché: Utilitzar la caché del navegador per millorar els temps de càrrega.
  • Desplegament: Moure la teva aplicació a un servidor en viu.

Minificació i Agrupació

La minificació i l'agrupació són passos essencials per optimitzar la teva aplicació Vue.js per a producció. La minificació redueix la mida dels teus fitxers eliminant caràcters innecessaris, mentre que l'agrupació combina múltiples fitxers en un per reduir el nombre de sol·licituds HTTP.

Exemple: Minificació i Agrupació amb Vue CLI

Vue CLI gestiona automàticament la minificació i l'agrupació quan construeixes el teu projecte per a producció.

# Construeix el teu projecte per a producció
npm run build

Aquesta comanda generarà una carpeta dist que conté els teus fitxers minificats i agrupats.

Variables d'Entorn

Les variables d'entorn et permeten configurar diferents paràmetres per a desenvolupament i producció. Vue CLI utilitza un fitxer .env per gestionar aquestes variables.

Exemple: Configurar Variables d'Entorn

Crea un fitxer .env a l'arrel del teu projecte:

VUE_APP_API_URL=https://api.example.com

Accedeix a la variable d'entorn en els teus components Vue:

export default {
  data() {
    return {
      apiUrl: process.env.VUE_APP_API_URL
    };
  }
};

Divisió de Codi

La divisió de codi et permet carregar només el codi necessari per a cada pàgina, millorant el temps de càrrega inicial de la teva aplicació.

Exemple: Divisió de Codi amb Vue Router

Utilitza importacions dinàmiques en la teva configuració de Vue Router per habilitar la divisió de codi:

import Vue from 'vue';
import Router from 'vue-router';

Vue.use(Router);

const Home = () => import('./views/Home.vue');
const About = () => import('./views/About.vue');

export default new Router({
  routes: [
    {
      path: '/',
      name: 'home',
      component: Home
    },
    {
      path: '/about',
      name: 'about',
      component: About
    }
  ]
});

Caché

La caché pot millorar significativament el rendiment de la teva aplicació emmagatzemant actius estàtics en el navegador.

Exemple: Configurar la Caché amb Vue CLI

Vue CLI genera automàticament noms de fitxers amb hash per als teus actius estàtics, cosa que ajuda amb la caché. Pots configurar encara més la caché modificant el fitxer vue.config.js:

module.exports = {
  chainWebpack: config => {
    config.output
      .filename('[name].[hash].js')
      .chunkFilename('[name].[hash].js');
  }
};

Desplegament

Desplegar la teva aplicació Vue.js implica moure els teus fitxers construïts a un servidor en viu. Hi ha diverses maneres de desplegar la teva aplicació, incloent serveis com Netlify, Vercel, o allotjament web tradicional.

Exemple: Desplegar a Netlify

  1. Construeix el teu projecte:

    npm run build

  2. Crea un fitxer netlify.toml a l'arrel del teu projecte:

    [build]
  publish = "dist"
  command = "npm run build"

  3. Puja el teu projecte a un repositori Git.

  4. Inicia sessió a Netlify i crea un nou lloc des del teu repositori Git.

Netlify detectarà automàticament el teu fitxer netlify.toml i desplegarà la teva aplicació.

Conclusió

Construir la teva aplicació Vue.js per a producció implica diversos passos per optimitzar el rendiment i assegurar un procés de desplegament suau. Minificant i agrupant els teus fitxers, configurant variables d'entorn, implementant la divisió de codi, aprofitant la caché i desplegant a un servidor en viu, pots assegurar-te que la teva aplicació sigui ràpida, eficient i estigui llesta per als usuaris.
