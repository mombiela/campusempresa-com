Vuex és un patró de gestió d'estat + biblioteca per a aplicacions Vue.js. Serveix com a magatzem centralitzat per a tots els components d'una aplicació, amb regles que asseguren que l'estat només es pot mutar de manera previsible. Vuex també s'integra profundament amb el sistema de reactivitat de Vue per fer que la gestió de l'estat sigui senzilla i eficient.

Conceptes Clau

  • Estat: La font única de veritat que conté les dades de l'aplicació.
  • Getters: Propietats computades per a l'estat del magatzem, permetent als components accedir a l'estat.
  • Mutacions: L'única manera de canviar l'estat en Vuex, i han de ser síncrones.
  • Accions: Similars a les mutacions, però poden ser asíncrones. Cometen mutacions.
  • Mòduls: Permeten que el magatzem es divideixi en peces més petites i manejables.

Configuració de Vuex

Per començar a utilitzar Vuex en el teu projecte Vue.js, primer l'has d'instal·lar:

npm install vuex --save

Després, pots crear un magatzem i integrar-lo en la teva instància de Vue:

import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment(context) {
      context.commit('increment');
    }
  },
  getters: {
    count: state => state.count
  }
});

new Vue({
  el: '#app',
  store,
  render: h => h(App)
});

Estat

L'estat és la font de dades de la teva aplicació. És un objecte únic que conté tot l'estat a nivell d'aplicació i serveix com la "font única de veritat."

const store = new Vuex.Store({
  state: {
    count: 0
  }
});

Accedir a l'Estat en Components

Pots accedir a l'estat en els teus components utilitzant this.$store.state:

export default {
  computed: {
    count() {
      return this.$store.state.count;
    }
  }
};

Getters

Els getters són com propietats computades per als magatzems. Et permeten definir propietats que depenen de l'estat del magatzem.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
    doubleCount: state => state.count * 2
  }
});

Utilitzar Getters en Components

Pots utilitzar getters en els teus components de manera similar a com utilitzes l'estat:

export default {
  computed: {
    doubleCount() {
      return this.$store.getters.doubleCount;
    }
  }
};

Mutacions

Les mutacions són l'única manera de canviar l'estat en Vuex. Han de ser síncrones.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  }
});

Cometre Mutacions

Pots cometre mutacions utilitzant this.$store.commit:

export default {
  methods: {
    increment() {
      this.$store.commit('increment');
    }
  }
};

Accions

Les accions són similars a les mutacions, però poden ser asíncrones. Cometen mutacions.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment');
      }, 1000);
    }
  }
});

Despatxar Accions

Pots despatxar accions utilitzant this.$store.dispatch:

export default {
  methods: {
    incrementAsync() {
      this.$store.dispatch('incrementAsync');
    }
  }
};

Mòduls

Els mòduls permeten dividir el teu magatzem en peces més petites i manejables. Cada mòdul pot tenir el seu propi estat, mutacions, accions i getters.

const moduleA = {
  state: () => ({ count: 0 }),
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment');
    }
  },
  getters: {
    doubleCount: state => state.count * 2
  }
};

const store = new Vuex.Store({
  modules: {
    a: moduleA
  }
});

Conclusió

Vuex és una eina poderosa de gestió d'estat que s'integra perfectament amb Vue.js. Entenent i utilitzant els seus conceptes bàsics—estat, getters, mutacions, accions i mòduls—pots gestionar l'estat de la teva aplicació d'una manera més estructurada i previsible. Aquesta introducció hauria de donar-te una base sòlida per començar a utilitzar Vuex en els teus projectes Vue.js.
