Vuex és un patró de gestió d'estat + biblioteca per a aplicacions Vue.js. Serveix com a magatzem centralitzat per a tots els components d'una aplicació, amb regles que asseguren que l'estat només es pot mutar de manera previsible. A mesura que la teva aplicació creix, pots trobar que el magatzem es torna difícil de gestionar. Aquí és on els mòduls de Vuex són útils.

Què són els mòduls de Vuex?

Els mòduls de Vuex et permeten dividir el teu magatzem en peces més petites i manejables. Cada mòdul pot contenir el seu propi estat, mutacions, accions, getters i fins i tot mòduls niats. Aquest enfocament modular ajuda a organitzar el magatzem d'una manera que és més fàcil de mantenir i escalar.

Conceptes clau dels mòduls de Vuex

  • Estat: Cada mòdul pot tenir el seu propi estat, que està aïllat de l'estat arrel i d'altres mòduls.
  • Mutacions: Cada mòdul pot definir les seves pròpies mutacions per modificar el seu estat.
  • Accions: Cada mòdul pot tenir accions per cometre mutacions o despatxar altres accions.
  • Getters: Cada mòdul pot tenir getters per calcular l'estat derivat basat en el seu estat.
  • Espai de noms: Els mòduls poden tenir un espai de noms per evitar conflictes de noms i fer que el magatzem sigui més modular.

Creant un mòdul de Vuex

Creem un mòdul de Vuex senzill per gestionar l'autenticació d'usuaris.

Definint el mòdul

Primer, defineix el mòdul en un fitxer separat, auth.js:

// store/modules/auth.js
const state = {
  isAuthenticated: false,
  user: null,
};

const mutations = {
  SET_AUTH(state, payload) {
    state.isAuthenticated = payload.isAuthenticated;
    state.user = payload.user;
  },
};

const actions = {
  login({ commit }, user) {
    // Simula una crida a l'API
    setTimeout(() => {
      commit('SET_AUTH', { isAuthenticated: true, user });
    }, 1000);
  },
  logout({ commit }) {
    commit('SET_AUTH', { isAuthenticated: false, user: null });
  },
};

const getters = {
  isAuthenticated: (state) => state.isAuthenticated,
  user: (state) => state.user,
};

export default {
  state,
  mutations,
  actions,
  getters,
};

Registrant el mòdul

A continuació, registra el mòdul en el teu magatzem de Vuex:

// store/index.js
import Vue from 'vue';
import Vuex from 'vuex';
import auth from './modules/auth';

Vue.use(Vuex);

export default new Vuex.Store({
  modules: {
    auth,
  },
});

Utilitzant el mòdul en un component

Ara, pots utilitzar l'estat, getters, accions i mutacions del mòdul en els teus components Vue:

<template>
  <div>
    <p v-if="isAuthenticated">Benvingut, {{ user.name }}</p>
    <button @click="login">Iniciar sessió</button>
    <button @click="logout">Tancar sessió</button>
  </div>
</template>

<script>
import { mapState, mapGetters, mapActions } from 'vuex';

export default {
  computed: {
    ...mapState('auth', ['isAuthenticated', 'user']),
    ...mapGetters('auth', ['isAuthenticated', 'user']),
  },
  methods: {
    ...mapActions('auth', ['login', 'logout']),
  },
};
</script>

Espai de noms dels mòduls

Per defecte, tots els mòduls es registren sota l'espai de noms global. Això pot portar a conflictes de noms, especialment en aplicacions grans. Per evitar això, pots donar un espai de noms als teus mòduls.

Habilitant l'espai de noms

Per habilitar l'espai de noms, estableix la propietat namespaced a true en el teu mòdul:

// store/modules/auth.js
export default {
  namespaced: true,
  state,
  mutations,
  actions,
  getters,
};

Accedint als mòduls amb espai de noms

Quan utilitzis mòduls amb espai de noms, has d'especificar el nom del mòdul quan accedeixis al seu estat, getters, accions i mutacions:

<script>
import { mapState, mapGetters, mapActions } from 'vuex';

export default {
  computed: {
    ...mapState('auth', ['isAuthenticated', 'user']),
    ...mapGetters('auth', ['isAuthenticated', 'user']),
  },
  methods: {
    ...mapActions('auth', ['login', 'logout']),
  },
};
</script>

Conclusió

Els mòduls de Vuex són una característica poderosa que ajuda a organitzar i gestionar l'estat en aplicacions grans de Vue.js. Dividint el magatzem en mòduls més petits i autònoms, pots fer que el teu codi sigui més modular, mantenible i escalable. Recorda utilitzar l'espai de noms per evitar conflictes i mantenir el teu magatzem net i organitzat.
