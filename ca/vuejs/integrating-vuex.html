Introducció a Vuex

Vuex és un patró de gestió d'estat + biblioteca per a aplicacions Vue.js. Serveix com a magatzem centralitzat per a tots els components d'una aplicació, amb regles que asseguren que l'estat només es pot mutar de manera previsible.

Conceptes Clau

  • Estat: La font única de veritat.
  • Getters: L'equivalent de les propietats computades per als magatzems.
  • Mutacions: L'única manera de canviar realment l'estat en un magatzem Vuex.
  • Accions: Similar a les mutacions, però en lloc de mutar l'estat, les accions cometen mutacions.
  • Mòduls: Permeten dividir el magatzem en múltiples mòduls, cadascun amb el seu propi estat, mutacions, accions i getters.

Configuració de Vuex

Per integrar Vuex al teu projecte Vue.js, primer l'has d'instal·lar.

npm install vuex --save

Després, crea un magatzem i integra'l a la teva instància de Vue.

// store.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment(context) {
      context.commit('increment');
    }
  },
  getters: {
    count: state => state.count
  }
});
// main.js
import Vue from 'vue';
import App from './App.vue';
import store from './store';

new Vue({
  render: h => h(App),
  store
}).$mount('#app');

Gestió d'Estat

L'estat en Vuex és un objecte únic que conté tot l'estat a nivell d'aplicació.

const store = new Vuex.Store({
  state: {
    count: 0
  }
});

Accedir a l'Estat

Pots accedir a l'estat en els teus components utilitzant this.$store.state.

<template>
  <div>{{ count }}</div>
</template>

<script>
export default {
  computed: {
    count() {
      return this.$store.state.count;
    }
  }
};
</script>

Mutacions

Les mutacions són l'única manera de canviar realment l'estat en un magatzem Vuex. Cada mutació té un tipus de cadena i un controlador.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  }
});

Cometre Mutacions

Pots cometre mutacions utilitzant store.commit.

<template>
  <button @click="increment">Incrementar</button>
</template>

<script>
export default {
  methods: {
    increment() {
      this.$store.commit('increment');
    }
  }
};
</script>

Accions

Les accions són similars a les mutacions, però en lloc de mutar l'estat, les accions cometen mutacions. Les accions poden contenir operacions asíncrones arbitràries.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment');
      }, 1000);
    }
  }
});

Despatxar Accions

Pots despatxar accions utilitzant store.dispatch.

<template>
  <button @click="incrementAsync">Incrementar Asíncronament</button>
</template>

<script>
export default {
  methods: {
    incrementAsync() {
      this.$store.dispatch('incrementAsync');
    }
  }
};
</script>

Getters

Els getters són l'equivalent de les propietats computades per als magatzems. S'utilitzen per calcular l'estat derivat basat en l'estat del magatzem.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
    doubleCount: state => state.count * 2
  }
});

Accedir als Getters

Pots accedir als getters en els teus components utilitzant this.$store.getters.

<template>
  <div>{{ doubleCount }}</div>
</template>

<script>
export default {
  computed: {
    doubleCount() {
      return this.$store.getters.doubleCount;
    }
  }
};
</script>

Mòduls

Els mòduls permeten dividir el teu magatzem en múltiples mòduls, cadascun amb el seu propi estat, mutacions, accions i getters.

const moduleA = {
  state: { count: 0 },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment');
    }
  },
  getters: {
    doubleCount: state => state.count * 2
  }
};

const store = new Vuex.Store({
  modules: {
    a: moduleA
  }
});

Accedir a l'Estat del Mòdul

Pots accedir a l'estat del mòdul en els teus components utilitzant this.$store.state.moduleName.

<template>
  <div>{{ count }}</div>
</template>

<script>
export default {
  computed: {
    count() {
      return this.$store.state.a.count;
    }
  }
};
</script>

Conclusió

Integrar Vuex a la teva aplicació Vue.js proporciona una solució robusta per gestionar l'estat a tota la teva aplicació. Entenent i utilitzant l'estat, les mutacions, les accions, els getters i els mòduls, pots crear aplicacions escalables i mantenibles. Practica construint petites aplicacions i incorporant gradualment una gestió d'estat més complexa per convertir-te en un expert en l'ús de Vuex.
