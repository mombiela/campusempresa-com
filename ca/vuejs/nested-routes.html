Introducció a les rutes niades

Les rutes niades a Vue.js et permeten crear estructures de navegació complexes i jeràrquiques dins de la teva aplicació. Això és particularment útil per a aplicacions amb múltiples capes de vistes, com ara panells de control, perfils d'usuari o llocs de comerç electrònic amb categories i subcategories de productes.

Conceptes clau

  • Rutes niades: Rutes dins de rutes, permetent una estructura jeràrquica.
  • Vista del Router: Un component que renderitza el component corresponent per a una ruta determinada.
  • Rutes fill: Rutes que estan niades dins d'una ruta pare.

Configuració de les rutes niades

Per configurar rutes niades, has de definir-les a la configuració del teu Vue Router. Aquí tens una guia pas a pas:

  1. Instal·la Vue Router: Si encara no ho has fet, instal·la Vue Router al teu projecte Vue.js.

    npm install vue-router

  2. Defineix les rutes: Crea un fitxer router.js (o modifica la teva configuració de router existent) per definir les teves rutes niades.

Exemple: Rutes niades bàsiques

Considera una aplicació senzilla amb un component User que té rutes niades per a Profile i Posts.

// router.js
import Vue from 'vue';
import Router from 'vue-router';
import Home from './components/Home.vue';
import User from './components/User.vue';
import UserProfile from './components/UserProfile.vue';
import UserPosts from './components/UserPosts.vue';

Vue.use(Router);

const routes = [
  { path: '/', component: Home },
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      }
    ]
  }
];

const router = new Router({
  routes
});

export default router;

Explicació

  • Ruta pare: /user/:id - Aquesta ruta renderitzarà el component User.
  • Rutes fill: /user/:id/profile i /user/:id/posts - Aquestes rutes renderitzaran els components UserProfile i UserPosts respectivament, dins del component User.

Vista del Router en el component pare

Per renderitzar els components fill, has d'incloure un <router-view> en el component pare.

<!-- User.vue -->
<template>
  <div>
    <h2>User {{ $route.params.id }}</h2>
    <router-view></router-view> <!-- Aquí es renderitzaran els components fill -->
  </div>
</template>

<script>
export default {
  name: 'User'
};
</script>

Rutes niades avançades

Per a aplicacions més complexes, potser necessitaràs rutes niades més profundes o rutes niades dinàmiques.

Exemple: Rutes niades més profundes

Imagina una aplicació on un usuari té múltiples projectes, i cada projecte té múltiples tasques.

// router.js
import Vue from 'vue';
import Router from 'vue-router';
import Home from './components/Home.vue';
import User from './components/User.vue';
import UserProfile from './components/UserProfile.vue';
import UserPosts from './components/UserPosts.vue';
import Project from './components/Project.vue';
import Task from './components/Task.vue';

Vue.use(Router);

const routes = [
  { path: '/', component: Home },
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      },
      {
        path: 'project/:projectId',
        component: Project,
        children: [
          {
            path: 'task/:taskId',
            component: Task
          }
        ]
      }
    ]
  }
];

const router = new Router({
  routes
});

export default router;

Explicació

  • Rutes niades més profundes: El component Project està niat dins del component User, i el component Task està niat dins del component Project.
  • Segments dinàmics: Tant projectId com taskId són segments dinàmics, permetent una navegació dinàmica basada en els IDs de projecte i tasca.

Vista del Router en components niats

Assegura't que cada component pare inclogui un <router-view> per renderitzar els seus components fill.

<!-- Project.vue -->
<template>
  <div>
    <h3>Project {{ $route.params.projectId }}</h3>
    <router-view></router-view> <!-- Aquí es renderitzarà el component Task -->
  </div>
</template>

<script>
export default {
  name: 'Project'
};
</script>

Resum

Les rutes niades a Vue.js proporcionen una manera potent d'estructurar la navegació de la teva aplicació. Definint rutes dins de rutes, pots crear vistes complexes i jeràrquiques que són fàcils de gestionar i navegar. Recorda:

  • Defineix rutes niades a la configuració del teu router.
  • Utilitza <router-view> en els components pare per renderitzar els components fill.
  • Utilitza segments dinàmics per a una navegació flexible i dinàmica.

Dominant les rutes niades, pots construir aplicacions sofisticades i fàcils d'utilitzar amb Vue.js.
