Introducció

En aquesta secció, aprofundirem en els conceptes bàsics de la gestió d'estat en Vue.js utilitzant Vuex. Cobrirem l'estat, els getters, les mutacions i les accions, explicant els seus rols i com interactuen entre ells per gestionar l'estat de la teva aplicació de manera efectiva.

Estat

L'estat en Vuex és una única font de veritat que conté les dades de la teva aplicació. És un objecte JavaScript senzill que emmagatzema l'estat.

Exemple

const store = new Vuex.Store({
  state: {
    count: 0
  }
});
  • Concepte Clau: L'estat és reactiu, el que significa que qualsevol canvi en l'estat actualitzarà automàticament els components que en depenen.

Getters

Els getters són similars a les propietats computades en els components de Vue. Et permeten calcular l'estat derivat basat en l'estat de la botiga.

Exemple

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
    doubleCount: state => state.count * 2
  }
});
  • Concepte Clau: Els getters es poden utilitzar per filtrar o calcular dades a partir de l'estat.

Mutacions

Les mutacions són l'única manera de canviar l'estat en Vuex. Han de ser síncrones i es cometen utilitzant store.commit.

Exemple

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  }
});

// Cometre una mutació
store.commit('increment');
  • Concepte Clau: Les mutacions són síncrones i han de ser l'única manera de canviar directament l'estat.

Accions

Les accions són similars a les mutacions, però poden ser asíncrones. Es despatxen utilitzant store.dispatch i poden cometre mutacions.

Exemple

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment');
      }, 1000);
    }
  }
});

// Despatxar una acció
store.dispatch('incrementAsync');
  • Concepte Clau: Les accions poden contenir operacions asíncrones i poden cometre múltiples mutacions.

Resum

En aquesta secció, hem cobert els conceptes fonamentals de la gestió d'estat en Vue.js utilitzant Vuex:

  • Estat: La única font de veritat per a les dades de la teva aplicació.
  • Getters: Propietats computades per a l'estat de la teva botiga.
  • Mutacions: L'única manera de canviar directament l'estat, i han de ser síncrones.
  • Accions: Poden ser asíncrones i s'utilitzen per cometre mutacions.

Entendre aquests conceptes és crucial per gestionar l'estat complex en aplicacions Vue.js. Dominant l'estat, els getters, les mutacions i les accions, pots construir aplicacions més predictibles i mantenibles.
