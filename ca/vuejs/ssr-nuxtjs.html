Introducció a la renderització del costat del servidor (SSR)

La renderització del costat del servidor (SSR) és una tècnica utilitzada per renderitzar pàgines web al servidor en lloc del client. Això pot millorar el rendiment, el SEO i l'experiència de l'usuari.

Conceptes clau

  • Renderització del costat del client (CSR): El navegador descarrega una pàgina HTML mínima i renderitza el contingut utilitzant JavaScript.
  • Renderització del costat del servidor (SSR): El servidor genera l'HTML complet per a una pàgina i l'envia al client.
  • Hidratació: El procés de fer que una pàgina renderitzada pel servidor sigui interactiva adjuntant oients d'esdeveniments i altres comportaments dinàmics.

Per què utilitzar SSR amb Nuxt.js?

Nuxt.js és un framework construït sobre Vue.js que simplifica la implementació de SSR. Proporciona un conjunt potent de funcions per construir aplicacions amb bon rendiment i amigables amb el SEO.

Beneficis de SSR amb Nuxt.js

  • Millora del SEO: Els motors de cerca poden indexar l'HTML completament renderitzat.
  • Temps més ràpid per al contingut: Els usuaris veuen el contingut més ràpidament perquè el servidor envia una pàgina completament renderitzada.
  • Millor rendiment: Els temps de càrrega inicials es poden reduir, especialment en dispositius més lents.

Configuració d'un projecte Nuxt.js

Per començar amb SSR a Nuxt.js, necessites configurar un projecte Nuxt.js.

Instal·lació

Primer, instal·la el CLI de Nuxt.js si encara no ho has fet:

npm install -g create-nuxt-app

Crea un nou projecte Nuxt.js:

npx create-nuxt-app my-nuxt-app

Segueix les indicacions per configurar el teu projecte. Assegura't de seleccionar "Universal (SSR / Static)" quan se't pregunti sobre el mode de renderització.

Estructura del projecte

Un projecte típic de Nuxt.js té la següent estructura:

my-nuxt-app/
├── assets/
├── components/
├── layouts/
├── middleware/
├── pages/
├── plugins/
├── static/
├── store/
├── nuxt.config.js
  • assets/: Actius no compilats com Sass, imatges o fonts.
  • components/: Components Vue.
  • layouts/: Dissenys per a les teves pàgines.
  • middleware/: Funcions personalitzades per executar abans de renderitzar una pàgina.
  • pages/: Components Vue per a cada ruta.
  • plugins/: Plugins JavaScript per executar abans de muntar l'aplicació principal de Vue.js.
  • static/: Fitxers estàtics accessibles directament a través de l'URL arrel.
  • store/: Fitxers de la botiga Vuex.

Creació de pàgines i rutes

Nuxt.js utilitza el directori pages/ per generar automàticament rutes.

Pàgina d'exemple

Crea un fitxer pages/index.vue:

<template>
  <div>
    <h1>Benvingut a la meva aplicació Nuxt.js</h1>
    <p>Aquesta pàgina està renderitzada pel servidor.</p>
  </div>
</template>

<script>
export default {
  asyncData() {
    return {
      message: 'Hola des de SSR!'
    }
  }
}
</script>

Explicació

  • Plantilla: Defineix l'estructura HTML.
  • Script: El mètode asyncData obté dades abans de renderitzar la pàgina al servidor.

Obtenció de dades amb SSR

Nuxt.js proporciona diverses maneres d'obtenir dades per a SSR, incloent asyncData i fetch.

Utilitzant `asyncData`

El mètode asyncData es crida abans de renderitzar la pàgina, permetent-te obtenir dades i fer-les disponibles per al component.

<template>
  <div>
    <h1>{{ post.title }}</h1>
    <p>{{ post.body }}</p>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  async asyncData({ params }) {
    const { data } = await axios.get(`https://jsonplaceholder.typicode.com/posts/${params.id}`);
    return { post: data };
  }
}
</script>

Utilitzant `fetch`

El mètode fetch es crida al costat del servidor abans de renderitzar i al costat del client quan es navega.

<template>
  <div>
    <h1>{{ post.title }}</h1>
    <p>{{ post.body }}</p>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  data() {
    return {
      post: {}
    }
  },
  async fetch() {
    const { data } = await axios.get(`https://jsonplaceholder.typicode.com/posts/${this.$route.params.id}`);
    this.post = data;
  }
}
</script>

Middleware i autenticació

Nuxt.js et permet definir middleware per executar lògica personalitzada abans de renderitzar una pàgina.

Middleware d'exemple

Crea un fitxer middleware/auth.js:

export default function ({ store, redirect }) {
  if (!store.state.authenticated) {
    return redirect('/login');
  }
}

Aplica el middleware a una pàgina:

<template>
  <div>
    <h1>Pàgina protegida</h1>
  </div>
</template>

<script>
export default {
  middleware: 'auth'
}
</script>
