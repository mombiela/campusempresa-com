Introducció

En aquesta secció, explorarem patrons avançats en Vuex, la biblioteca de gestió d'estat per a Vue.js. Aquests patrons t'ajudaran a gestionar la lògica d'estat complexa, millorar el manteniment del codi i augmentar l'escalabilitat de les teves aplicacions Vue.js.

Conceptes clau

  • Mòduls: Organitzar l'estat, les mutacions, les accions i els getters en mòduls separats.
  • Espais de noms: Utilitzar espais de noms per evitar conflictes de noms en aplicacions grans.
  • Mòduls dinàmics: Registrar i desregistrar mòduls dinàmicament.
  • API de composició amb Vuex: Integrar Vuex amb l'API de composició per a una millor reutilització.
  • Plugins: Estendre la funcionalitat de Vuex amb plugins.
  • Proves de Vuex: Escriure proves unitàries per a la botiga Vuex.

Mòduls

Els mòduls et permeten dividir la teva botiga en múltiples botigues més petites, cadascuna gestionant el seu propi estat, mutacions, accions i getters.

Exemple

// store/modules/user.js
const state = {
  name: '',
  age: null,
};

const mutations = {
  SET_USER(state, user) {
    state.name = user.name;
    state.age = user.age;
  },
};

const actions = {
  updateUser({ commit }, user) {
    commit('SET_USER', user);
  },
};

const getters = {
  userName: (state) => state.name,
  userAge: (state) => state.age,
};

export default {
  state,
  mutations,
  actions,
  getters,
};

Registrar mòduls

// store/index.js
import Vue from 'vue';
import Vuex from 'vuex';
import user from './modules/user';

Vue.use(Vuex);

export default new Vuex.Store({
  modules: {
    user,
  },
});

Espais de noms

Els espais de noms ajuden a evitar conflictes de noms encapsulant l'estat del mòdul, les mutacions, les accions i els getters.

Exemple

// store/modules/user.js
export default {
  namespaced: true,
  state: { ... },
  mutations: { ... },
  actions: { ... },
  getters: { ... },
};

Accedir a mòduls amb espais de noms

// En un component Vue
this.$store.dispatch('user/updateUser', user);
this.$store.commit('user/SET_USER', user);

Mòduls dinàmics

Els mòduls dinàmics es poden registrar i desregistrar en temps d'execució, cosa que és útil per a aplicacions grans o quan els mòduls només es necessiten sota certes condicions.

Exemple

// Registrar un mòdul dinàmic
this.$store.registerModule('dynamicModule', {
  state: { ... },
  mutations: { ... },
  actions: { ... },
  getters: { ... },
});

// Desregistrar un mòdul dinàmic
this.$store.unregisterModule('dynamicModule');

API de composició amb Vuex

Utilitzar Vuex amb l'API de composició permet una millor organització del codi i reutilització.

Exemple

// store/modules/user.js
import { reactive, toRefs } from 'vue';

export default {
  namespaced: true,
  state: () => reactive({
    name: '',
    age: null,
  }),
  mutations: { ... },
  actions: { ... },
  getters: { ... },
};

// En un component Vue
import { useStore } from 'vuex';
import { computed } from 'vue';

export default {
  setup() {
    const store = useStore();
    const userName = computed(() => store.state.user.name);
    const userAge = computed(() => store.state.user.age);

    return { userName, userAge };
  },
};

Plugins

Els plugins de Vuex es poden utilitzar per estendre la funcionalitat de la botiga.

Exemple

// Plugin de registre
const logger = (store) => {
  store.subscribe((mutation, state) => {
    console.log(mutation.type);
    console.log(mutation.payload);
  });
};

export default new Vuex.Store({
  plugins: [logger],
});

Proves de Vuex

Escriure proves unitàries per a la botiga Vuex assegura la fiabilitat de la teva lògica de gestió d'estat.

Exemple

// store/modules/user.js
import { mutations } from './user';
const { SET_USER } = mutations;

describe('mutations', () => {
  it('SET_USER', () => {
    const state = { name: '', age: null };
    SET_USER(state, { name: 'John', age: 30 });
    expect(state.name).toBe('John');
    expect(state.age).toBe(30);
  });
});

Conclusió

Els patrons avançats de Vuex com ara mòduls, espais de noms, mòduls dinàmics, l'API de composició, plugins i proves poden millorar significativament l'estructura i el manteniment de les teves aplicacions Vue.js. En dominar aquests patrons, pots construir aplicacions escalables i robustes amb facilitat.
