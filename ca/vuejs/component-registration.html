Registre de Components
Props i esdeveniments personalitzats ►

En Vue.js, els components són els blocs de construcció de la teva aplicació. Entendre com registrar i utilitzar components és crucial per crear codi modular i mantenible. Aquesta secció cobrirà les diferents maneres de registrar components en Vue.js, des de tècniques bàsiques fins a avançades.

Registre Local

El registre local és útil quan vols utilitzar un component en un únic component pare. Aquest mètode manté els teus components encapsulats i evita contaminar l'espai de noms global.

Exemple

// ChildComponent.vue
<template>
  <div>
    <p>Aquest és un component fill</p>
  </div>
</template>

<script>
export default {
  name: 'ChildComponent'
}
</script>
// ParentComponent.vue
<template>
  <div>
    <h1>Component Pare</h1>
    <ChildComponent />
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';

export default {
  name: 'ParentComponent',
  components: {
    ChildComponent
  }
}
</script>

Conceptes Clau

  • Encapsulació: El registre local manté els components encapsulats dins dels seus components pare.
  • Àmbit: Els components registrats localment només estan disponibles dins del component pare.

Registre Global

El registre global fa que un component estigui disponible a tota la teva aplicació. Això és útil per a components que es reutilitzen freqüentment.

Exemple

// main.js
import Vue from 'vue';
import App from './App.vue';
import GlobalComponent from './components/GlobalComponent.vue';

Vue.component('GlobalComponent', GlobalComponent);

new Vue({
  render: h => h(App),
}).$mount('#app');
// GlobalComponent.vue
<template>
  <div>
    <p>Aquest és un component global</p>
  </div>
</template>

<script>
export default {
  name: 'GlobalComponent'
}
</script>

Conceptes Clau

  • Reutilització: Els components globals es poden utilitzar en qualsevol lloc de l'aplicació.
  • Contaminació de l'Espai de Noms: L'ús excessiu de components globals pot portar a un espai de noms global desordenat.

Registre Dinàmic i Asíncron

El registre dinàmic i asíncron de components pot ser útil per optimitzar el rendiment i carregar components només quan sigui necessari.

Exemple

// ParentComponent.vue
<template>
  <div>
    <h1>Component Pare</h1>
    <component :is="currentComponent" />
  </div>
</template>

<script>
export default {
  name: 'ParentComponent',
  data() {
    return {
      currentComponent: null
    };
  },
  created() {
    import('./DynamicComponent.vue').then(module => {
      this.currentComponent = module.default;
    });
  }
}
</script>
// DynamicComponent.vue
<template>
  <div>
    <p>Aquest és un component carregat dinàmicament</p>
  </div>
</template>

<script>
export default {
  name: 'DynamicComponent'
}
</script>

Conceptes Clau

  • Carrega Per Mandat: Carrega components només quan siguin necessaris per millorar el rendiment.
  • Components Dinàmics: Utilitza l'element component amb l'atribut :is per canviar dinàmicament entre components.

Resum

El registre de components en Vue.js es pot fer localment, globalment o dinàmicament. Cada mètode té els seus propis casos d'ús i beneficis:

  • Registre Local: Manté els components encapsulats i àmbit al seu pare.
  • Registre Global: Fa que els components siguin reutilitzables a tota l'aplicació però pot portar a una contaminació de l'espai de noms.
  • Registre Dinàmic i Asíncron: Optimitza el rendiment carregant components només quan sigui necessari.

Entendre aquests diferents mètodes t'ajudarà a crear aplicacions Vue.js més eficients i mantenibles.
