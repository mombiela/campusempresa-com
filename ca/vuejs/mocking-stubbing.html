La simulació i la substitució són tècniques essencials en les proves unitàries que ajuden a aïllar el component o la funció que s'està provant substituint les dependències per substituts controlats. Això assegura que les proves siguin fiables i es centrin en la funcionalitat específica que es prova.

Què és la Simulació?

La simulació implica crear objectes que simulen el comportament dels objectes reals. Aquests objectes simulats es poden programar per retornar valors específics o realitzar certes accions quan es criden durant una prova.

Conceptes Clau de la Simulació

  • Objectes Simulats: Objectes simulats que imiten el comportament dels objectes reals.
  • Verificació de Comportament: Assegurar-se que els objectes simulats s'utilitzen com s'espera.
  • Aïllament: Provar components en aïllament substituint les dependències per simulacions.

Què és la Substitució?

La substitució és una tècnica utilitzada per substituir una funció o mètode amb un "stub" que retorna resultats predefinits. Això és particularment útil quan es vol controlar la sortida d'una funció durant una prova.

Conceptes Clau de la Substitució

  • Funcions Stub: Funcions que retornen valors predefinits.
  • Entorn Controlat: Crear un entorn previsible per a les proves.
  • Simplificació: Simplificar dependències complexes utilitzant stubs.

Simulació i Substitució en Vue.js

En Vue.js, la simulació i la substitució s'utilitzen comunament en proves unitàries per aïllar components i assegurar-se que les proves siguin enfocades i fiables. La biblioteca vue-test-utils, juntament amb marcs de proves com Jest, proporciona eines per simular i substituir fàcilment les dependències.

Simulació amb `vue-test-utils` i Jest

Exemple: Simulació d'un Servei

Suposem que tenim un servei que obté dades d'usuari:

// userService.js
export const getUser = async (id) => {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
};

Podem simular aquest servei en les nostres proves:

// UserComponent.vue
<template>
  <div>{{ user.name }}</div>
</template>

<script>
import { getUser } from './userService';

export default {
  data() {
    return {
      user: {},
    };
  },
  async created() {
    this.user = await getUser(1);
  },
};
</script>
// UserComponent.test.js
import { shallowMount } from '@vue/test-utils';
import UserComponent from './UserComponent';
import * as userService from './userService';

jest.mock('./userService');

describe('UserComponent', () => {
  it('fetches and displays user data', async () => {
    userService.getUser.mockResolvedValue({ name: 'John Doe' });

    const wrapper = shallowMount(UserComponent);
    await wrapper.vm.$nextTick();

    expect(wrapper.text()).toContain('John Doe');
  });
});

Substitució amb `vue-test-utils`

Exemple: Substitució d'un Component Fill

Suposem que tenim un component pare que utilitza un component fill:

// ChildComponent.vue
<template>
  <div>Child Component</div>
</template>
// ParentComponent.vue
<template>
  <div>
    <ChildComponent />
  </div>
</template>

<script>
import ChildComponent from './ChildComponent';

export default {
  components: {
    ChildComponent,
  },
};
</script>

Podem substituir el component fill en les nostres proves:

// ParentComponent.test.js
import { shallowMount } from '@vue/test-utils';
import ParentComponent from './ParentComponent';

describe('ParentComponent', () => {
  it('renders the child component', () => {
    const wrapper = shallowMount(ParentComponent, {
      stubs: {
        ChildComponent: true,
      },
    });

    expect(wrapper.html()).toContain('<childcomponent-stub></childcomponent-stub>');
  });
});

Conclusió

La simulació i la substitució són tècniques poderoses en les proves de Vue.js que permeten als desenvolupadors aïllar components i controlar les dependències. Utilitzant eines com vue-test-utils i Jest, pots crear proves unitàries fiables i enfocades que assegurin que els teus components funcionin com s'espera. Entendre i aplicar aquestes tècniques t'ajudarà a escriure millors proves i mantenir un alt nivell de qualitat del codi en les teves aplicacions Vue.js.
