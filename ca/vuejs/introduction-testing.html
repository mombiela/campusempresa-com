Visió General

Les proves són una part crucial del desenvolupament de programari que assegura que la teva aplicació funcioni com s'espera. En aquesta secció, t'introduirem a les proves en Vue.js, cobrint els conceptes bàsics i avançant gradualment cap a conceptes més avançats.

Per què són importants les proves

  • Assegura la Qualitat del Codi: Ajuda a detectar errors en les primeres fases del procés de desenvolupament.
  • Facilita la Refactorització: Fa més fàcil canviar el codi sense trencar la funcionalitat existent.
  • Millora la Fiabilitat: Assegura que la teva aplicació es comporti com s'espera en diferents condicions.
  • Documentació: Les proves poden servir com a documentació de com ha de funcionar el teu codi.

Tipus de Proves

  • Proves Unitàries: Prova components o funcions individuals de manera aïllada.
  • Proves d'Integració: Prova com treballen junts diferents parts de l'aplicació.
  • Proves de Cap a Cap (E2E): Prova tota l'aplicació des de la perspectiva de l'usuari.

Configuració de les Proves en Vue.js

Per començar amb les proves en Vue.js, necessites configurar el teu entorn. Vue CLI proporciona suport integrat per a les proves.

Instal·lació de Vue CLI

Primer, assegura't de tenir Vue CLI instal·lat:

npm install -g @vue/cli

Creació d'un Nou Projecte amb Suport per a Proves

Crea un nou projecte Vue amb suport per a proves:

vue create my-vue-app

Durant la configuració, selecciona l'opció per incloure Proves Unitàries i Proves E2E.

Instal·lació de Llibreries de Proves

Per a les proves unitàries, utilitzarem Jest, i per a les proves E2E, utilitzarem Cypress.

Instal·la Jest:

npm install --save-dev @vue/test-utils jest

Instal·la Cypress:

npm install --save-dev cypress

Escriure Proves Unitàries

Les proves unitàries se centren en provar components individuals. Anem a escriure una prova unitària simple per a un component Vue.

Component Exemple

Aquí tens un component Vue simple HelloWorld.vue:

<template>
  <div>
    <p>{{ message }}</p>
  </div>
</template>

<script>
export default {
  name: 'HelloWorld',
  props: {
    message: {
      type: String,
      required: true
    }
  }
}
</script>

Escriure una Prova Unitària

Crea un fitxer de prova HelloWorld.spec.js:

import { shallowMount } from '@vue/test-utils';
import HelloWorld from '@/components/HelloWorld.vue';

describe('HelloWorld.vue', () => {
  it('renders props.message when passed', () => {
    const message = 'Hello, Vue!';
    const wrapper = shallowMount(HelloWorld, {
      propsData: { message }
    });
    expect(wrapper.text()).toMatch(message);
  });
});

Executar Proves Unitàries

Per executar les teves proves unitàries, utilitza la següent comanda:

npm run test:unit

Escriure Proves E2E

Les proves de Cap a Cap simulen interaccions d'usuari amb la teva aplicació. Anem a escriure una prova E2E simple utilitzant Cypress.

Exemple de Prova E2E

Crea un fitxer de prova helloWorld.spec.js a la carpeta cypress/integration:

describe('HelloWorld Component', () => {
  it('displays the correct message', () => {
    cy.visit('/');
    cy.contains('Hello, Vue!');
  });
});

Executar Proves E2E

Per executar les teves proves E2E, utilitza la següent comanda:

npm run test:e2e

Conclusió

Les proves són una pràctica essencial en el desenvolupament de Vue.js que assegura que la teva aplicació sigui fiable, mantenible i lliure d'errors. Configurant proves unitàries i E2E, pots detectar problemes aviat i fer que la refactorització sigui més fàcil. A mesura que progressis, podràs explorar tècniques i eines de prova més avançades per millorar encara més la teva estratègia de proves.
