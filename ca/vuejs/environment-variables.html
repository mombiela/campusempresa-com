Les variables d'entorn són un aspecte crucial del desenvolupament web modern, ja que permeten configurar la teva aplicació per a diferents entorns (desenvolupament, proves, producció) sense canviar el codi. En aquesta secció, explorarem com utilitzar les variables d'entorn en una aplicació Vue.js, des de conceptes bàsics fins a usos avançats.

Què són les Variables d'Entorn?

Les variables d'entorn són parells clau-valor utilitzats per configurar aplicacions. Permeten:

  • Emmagatzemar informació sensible com les claus d'API de manera segura.
  • Canviar la configuració sense modificar el codi base.
  • Gestionar diferents configuracions per a diversos entorns (desenvolupament, staging, producció).

Configuració de Variables d'Entorn a Vue.js

Vue.js utilitza un fitxer .env per gestionar les variables d'entorn. El Vue CLI proporciona suport integrat per a les variables d'entorn, fent que sigui fàcil configurar-les i utilitzar-les.

Creació de Fitxers d'Entorn

Pots crear diferents fitxers .env per a diferents entorns:

  • .env - Per defecte.
  • .env.development - Entorn de desenvolupament.
  • .env.production - Entorn de producció.

Exemple de Fitxers d'Entorn

.env

VUE_APP_API_URL=https://api.example.com
VUE_APP_FEATURE_FLAG=true

.env.development

VUE_APP_API_URL=https://dev.api.example.com
VUE_APP_FEATURE_FLAG=false

.env.production

VUE_APP_API_URL=https://api.example.com
VUE_APP_FEATURE_FLAG=true

Regles Importants

  • Els noms de les variables d'entorn han de començar amb VUE_APP_.
  • No comprometis informació sensible al control de versions.

Ús de Variables d'Entorn a Vue.js

Per utilitzar les variables d'entorn a la teva aplicació Vue.js, pots accedir-hi a través de process.env.

Accés a les Variables d'Entorn

export default {
  data() {
    return {
      apiUrl: process.env.VUE_APP_API_URL,
      featureFlag: process.env.VUE_APP_FEATURE_FLAG === 'true'
    };
  },
  created() {
    console.log('API URL:', this.apiUrl);
    console.log('Feature Flag:', this.featureFlag);
  }
};

Exemple: Activació Condicional de Funcionalitats

<template>
  <div>
    <h1>Benvingut a la Meva App Vue</h1>
    <div v-if="featureFlag">
      <p>La Nova Funcionalitat està Activada!</p>
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      featureFlag: process.env.VUE_APP_FEATURE_FLAG === 'true'
    };
  }
};
</script>

Ús Avançat

Ús de Variables d'Entorn a Vuex

Si estàs utilitzant Vuex per a la gestió d'estat, també pots accedir a les variables d'entorn dins de la teva botiga.

const store = new Vuex.Store({
  state: {
    apiUrl: process.env.VUE_APP_API_URL
  },
  getters: {
    getApiUrl: state => state.apiUrl
  }
});

Variables d'Entorn en el Procés de Construcció

També pots utilitzar variables d'entorn per controlar el procés de construcció. Per exemple, podries voler incloure diferents scripts d'analítica segons l'entorn.

// vue.config.js
module.exports = {
  configureWebpack: config => {
    if (process.env.NODE_ENV === 'production') {
      // Configuracions específiques per a producció
    } else {
      // Configuracions específiques per a desenvolupament
    }
  }
};

Conclusió

Les variables d'entorn són una eina poderosa per configurar la teva aplicació Vue.js en diferents entorns. Utilitzant fitxers .env i l'objecte process.env, pots gestionar fàcilment configuracions i informació sensible sense codificar-les directament a la teva aplicació. Això no només millora la seguretat, sinó que també fa que la teva aplicació sigui més flexible i fàcil de mantenir.

En aquesta secció, hem cobert:

  • Què són les variables d'entorn.
  • Com configurar i utilitzar les variables d'entorn a Vue.js.
  • Escenaris d'ús avançat, incloent Vuex i configuracions del procés de construcció.

Entendre i utilitzar efectivament les variables d'entorn t'ajudarà a construir aplicacions Vue.js més robustes i adaptables.
