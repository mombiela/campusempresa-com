Introducció

Apache Kafka és una plataforma de streaming distribuïda que et permet publicar i subscriure't a fluxos de registres. En aquesta secció, ens centrarem en consumir missatges de temes de Kafka. Cobrirem els conceptes bàsics dels consumidors de Kafka, com configurar-los i tècniques avançades per a un consum eficient de missatges.

Conceptes Clau

  • Consumer: Un client que llegeix registres d'un tema de Kafka.
  • Consumer Group: Un grup de consumidors que treballen junts per consumir missatges d'un tema.
  • Offset: Un identificador únic per a cada registre dins d'una partició, utilitzat per fer un seguiment de la posició del consumidor.

Configurant un Consumer de Kafka

Per consumir missatges de Kafka, necessites configurar un consumer de Kafka. Això implica configurar les propietats del consumer i escriure codi per llegir missatges d'un tema de Kafka.

Configuració Bàsica del Consumer

Aquí tens les propietats essencials que necessites configurar per a un consumer de Kafka:

  • bootstrap.servers: Una llista de parells host/port per utilitzar per establir la connexió inicial amb el clúster de Kafka.
  • group.id: Una cadena única que identifica el grup de consumidors al qual pertany aquest consumer.
  • key.deserializer: El nom de la classe del deserialitzador per a la clau.
  • value.deserializer: El nom de la classe del deserialitzador per al valor.

Exemple: Configuració Bàsica del Consumer en Java

import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.clients.consumer.KafkaConsumer;
import org.apache.kafka.clients.consumer.ConsumerRecords;
import org.apache.kafka.clients.consumer.ConsumerRecord;

import java.util.Properties;
import java.util.Collections;

public class BasicConsumer {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "test-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringDeserializer");
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringDeserializer");

        KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
        consumer.subscribe(Collections.singletonList("test-topic"));

        while (true) {
            ConsumerRecords<String, String> records = consumer.poll(100);
            for (ConsumerRecord<String, String> record : records) {
                System.out.printf("offset = %d, key = %s, value = %s%n", record.offset(), record.key(), record.value());
            }
        }
    }
}

Funcions Avançades del Consumer

Grups de Consumidors i Paral·lelisme

  • Consumer Groups: Múltiples consumidors poden unir-se a un grup de consumidors per llegir d'un tema en paral·lel. Cada partició en el tema s'assigna només a un consumidor dins d'un grup, assegurant que cada missatge es processi només una vegada.
  • Rebalancing: Quan un nou consumidor s'uneix o deixa el grup, Kafka redistribueix automàticament les particions entre els consumidors.

Exemple: Grup de Consumidors en Acció

// Configuració similar al consumer bàsic, però executa múltiples instàncies d'aquest consumer per veure el rebalancing en acció.

Cometre Offsets

  • Cometre Offsets Automàticament: Kafka pot cometre offsets automàticament a intervals regulars.
  • Cometre Offsets Manualment: Pots cometre offsets manualment per tenir més control sobre el processament de missatges.

Exemple: Cometre Offsets Manualment

import org.apache.kafka.clients.consumer.CommitFailedException;

// Dins del bucle while
try {
    consumer.commitSync();
} catch (CommitFailedException e) {
    System.err.printf("Commit failed: %s%n", e.getMessage());
}

Gestió d'Errors i Reintents

  • Gestió d'Errors de Deserialització: Implementa deserialitzadors personalitzats per gestionar errors de manera elegant.
  • Reintents: Configura polítiques de reintents per a errors transitoris.

Exemple: Deserialitzador Personalitzat

import org.apache.kafka.common.serialization.Deserializer;

public class CustomDeserializer implements Deserializer<MyObject> {
    @Override
    public MyObject deserialize(String topic, byte[] data) {
        try {
            // Lògica de deserialització personalitzada
        } catch (Exception e) {
            // Gestiona l'error de deserialització
        }
        return null;
    }
}

Conclusió

Consumir missatges de Kafka és un aspecte fonamental de treballar amb aquesta potent plataforma de streaming. Entenent els conceptes bàsics dels consumidors de Kafka, configurant-los correctament i aprofitant funcions avançades com els grups de consumidors i el cometre offsets manualment, pots construir aplicacions de consum de missatges robustes i eficients. Considera sempre la gestió d'errors i els reintents per assegurar que les teves aplicacions de consumidors siguin resilients i fiables.
