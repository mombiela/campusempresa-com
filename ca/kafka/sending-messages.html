En aquesta secció, tractarem com enviar missatges utilitzant Apache Kafka. Començarem amb els conceptes bàsics i gradualment passarem a conceptes més avançats. Al final d'aquesta secció, hauríeu de ser capaços d'enviar missatges a temes de Kafka de manera eficient i entendre els mecanismes subjacents.

Introducció als Productors de Kafka

Un Productor de Kafka és un client que envia registres a un clúster de Kafka. Els productors són responsables de triar quin registre assignar a quina partició dins d'un tema.

Conceptes Clau

  • Productor: L'aplicació client que envia registres a Kafka.
  • Registre: Les dades que s'envien a Kafka, consistents en una clau, valor i metadades opcionals.
  • Tema: Una categoria o nom de feed al qual s'envien els registres.
  • Partició: Una divisió d'un tema, que permet el processament en paral·lel.

Configuració d'un Productor de Kafka

Abans d'enviar missatges, cal configurar un productor de Kafka. Això implica configurar les propietats del productor i inicialitzar la instància del productor.

Configuració Bàsica

Aquí teniu algunes propietats essencials que cal configurar:

  • bootstrap.servers: Una llista de parells host/port per utilitzar per establir la connexió inicial amb el clúster de Kafka.
  • key.serializer: Classe de serialitzador per a la clau que implementa la interfície org.apache.kafka.common.serialization.Serializer.
  • value.serializer: Classe de serialitzador per al valor que implementa la interfície org.apache.kafka.common.serialization.Serializer.

Exemple de Configuració

import org.apache.kafka.clients.producer.KafkaProducer;
import org.apache.kafka.clients.producer.ProducerConfig;
import org.apache.kafka.common.serialization.StringSerializer;

import java.util.Properties;

public class ProducerConfigExample {
    public static void main(String[] args) {
        Properties properties = new Properties();
        properties.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        properties.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class.getName());
        properties.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class.getName());

        KafkaProducer<String, String> producer = new KafkaProducer<>(properties);
        // Utilitzar el productor per enviar missatges
    }
}

Enviament de Missatges

Un cop el productor està configurat, podeu començar a enviar missatges als temes de Kafka.

Enviament d'un Missatge Simple

Per enviar un missatge, cal crear un ProducerRecord i utilitzar el mètode send del productor.

import org.apache.kafka.clients.producer.ProducerRecord;

public class SimpleMessageProducer {
    public static void main(String[] args) {
        // Suposant que el productor ja està configurat com es mostra anteriorment
        KafkaProducer<String, String> producer = new KafkaProducer<>(properties);

        String topic = "my-topic";
        String key = "my-key";
        String value = "Hello, Kafka!";

        ProducerRecord<String, String> record = new ProducerRecord<>(topic, key, value);
        producer.send(record);

        producer.close();
    }
}

Gestió de Callbacks

També podeu gestionar callbacks per ser notificats quan un missatge s'envia correctament o si es produeix un error.

import org.apache.kafka.clients.producer.Callback;
import org.apache.kafka.clients.producer.RecordMetadata;
import org.apache.kafka.clients.producer.ProducerRecord;

public class CallbackProducer {
    public static void main(String[] args) {
        KafkaProducer<String, String> producer = new KafkaProducer<>(properties);

        String topic = "my-topic";
        String key = "my-key";
        String value = "Hello, Kafka with Callback!";

        ProducerRecord<String, String> record = new ProducerRecord<>(topic, key, value);
        producer.send(record, new Callback() {
            @Override
            public void onCompletion(RecordMetadata metadata, Exception exception) {
                if (exception == null) {
                    System.out.println("Missatge enviat correctament a " + metadata.topic() + " partició " + metadata.partition());
                } else {
                    exception.printStackTrace();
                }
            }
        });

        producer.close();
    }
}

Configuracions Avançades

Configuració d'Acknowledgment

Kafka permet configurar el nivell d'acknowledgment per assegurar la fiabilitat de l'entrega de missatges.

  • acks=0: No es requereix acknowledgment.
  • acks=1: Es requereix acknowledgment del líder.
  • acks=all: Totes les rèpliques sincronitzades han de reconèixer.

Exemple amb Acknowledgment

properties.put(ProducerConfig.ACKS_CONFIG, "all");

Mida del Lot i Temps de Retard

Podeu configurar la mida del lot i el temps de retard per optimitzar el rendiment.

  • batch.size: La mida del lot en bytes.
  • linger.ms: El temps d'espera abans d'enviar un lot.

Exemple amb Mida del Lot i Temps de Retard

properties.put(ProducerConfig.BATCH_SIZE_CONFIG, 16384); // 16 KB
properties.put(ProducerConfig.LINGER_MS_CONFIG, 10); // 10 ms
