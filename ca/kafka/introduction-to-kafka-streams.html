Kafka Streams és una biblioteca potent per construir aplicacions de processament de fluxos i microserveis. Et permet processar dades en temps real amb facilitat, aprofitant la potència d'Apache Kafka. Aquest tema t'introduirà als conceptes bàsics de Kafka Streams, la seva arquitectura i com crear una aplicació senzilla de Kafka Streams.

Què és Kafka Streams?

Kafka Streams és una biblioteca client per construir aplicacions i microserveis, on les dades d'entrada i sortida s'emmagatzemen en clústers de Kafka. Combina la simplicitat d'escriure i desplegar aplicacions estàndard de Java i Scala al costat del client amb els beneficis de la tecnologia de clúster del servidor de Kafka.

Conceptes Clau

  • Flux: Una seqüència il·limitada i contínuament actualitzada de registres.
  • Taula: Una vista d'un flux com un registre de canvis, que representa l'estat més recent de cada clau.
  • Topologia: Un gràfic de nodes de processament de fluxos que representa la lògica de càlcul.
  • Processador: Un node en la topologia que processa registres.
  • Magatzem d'Estat: Una base de dades local que emmagatzema l'estat d'una aplicació de processament de fluxos.

Arquitectura de Kafka Streams

Les aplicacions de Kafka Streams es construeixen al voltant d'uns quants conceptes bàsics:

  • Fluxos i Taules: Els fluxos són l'abstracció principal a Kafka Streams, representant una seqüència il·limitada de registres. Les taules es deriven dels fluxos i representen l'estat més recent de cada clau.
  • Topologia: La topologia defineix la lògica de processament de fluxos, que consisteix en fonts, processadors i sortides.
  • Processament amb Estat: Kafka Streams admet el processament amb estat, permetent-te mantenir i consultar l'estat dins de la teva aplicació.

Exemple d'Arquitectura

| Component | Descripció | |-----------------|-----------------------------------------------------------------------------| | Processador de Font| Llegeix dades dels temes de Kafka. | | Processador de Flux| Processa les dades, aplicant transformacions, filtres i agregacions. | | Processador de Sortida | Escriu les dades processades de nou als temes de Kafka o a sistemes externs. | | Magatzem d'Estat | Manté l'estat per al processament amb estat, com ara agregacions o unions. |

Creant una Aplicació Senzilla de Kafka Streams

Vegem els passos per crear una aplicació senzilla de Kafka Streams que llegeixi dades d'un tema de Kafka, les processi i escrigui els resultats a un altre tema de Kafka.

Pas 1: Configura el Teu Projecte

Primer, crea un nou projecte Maven o Gradle i afegeix la dependència de Kafka Streams al teu fitxer pom.xml o build.gradle.

Maven:

<dependency>
    <groupId>org.apache.kafka</groupId>
    <artifactId>kafka-streams</artifactId>
    <version>3.0.0</version>
</dependency>

Gradle:

implementation 'org.apache.kafka:kafka-streams:3.0.0'

Pas 2: Defineix la Topologia de Processament de Fluxos

Crea una classe per definir la teva topologia de processament de fluxos. En aquest exemple, llegirem d'un tema d'entrada, convertirem els valors a majúscules i escriurem els resultats a un tema de sortida.

import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.StreamsConfig;
import org.apache.kafka.streams.kstream.KStream;

import java.util.Properties;

public class SimpleKafkaStreamsApp {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(StreamsConfig.APPLICATION_ID_CONFIG, "streams-example");
        props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, "org.apache.kafka.common.serialization.Serdes$StringSerde");
        props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, "org.apache.kafka.common.serialization.Serdes$StringSerde");

        StreamsBuilder builder = new StreamsBuilder();
        KStream<String, String> sourceStream = builder.stream("input-topic");
        KStream<String, String> uppercasedStream = sourceStream.mapValues(value -> value.toUpperCase());
        uppercasedStream.to("output-topic");

        KafkaStreams streams = new KafkaStreams(builder.build(), props);
        streams.start();

        Runtime.getRuntime().addShutdownHook(new Thread(streams::close));
    }
}

Pas 3: Executa la Teva Aplicació

Compila i executa la teva aplicació. Assegura't que el teu clúster de Kafka estigui en funcionament i que els temes d'entrada i sortida existeixin.

mvn clean package
java -cp target/your-app-jar-with-dependencies.jar com.yourpackage.SimpleKafkaStreamsApp

Conclusió
