Apache Kafka és una eina potent per construir canals de dades en temps real i aplicacions de streaming. Aquesta secció cobrirà diversos casos d'ús de Kafka, des de nivells principiants fins a avançats, per ajudar-te a entendre la seva versatilitat i aplicacions pràctiques.

1. Streaming de Dades en Temps Real

El streaming de dades en temps real és un dels casos d'ús més comuns per Kafka. Et permet processar i analitzar dades a mesura que es generen.

  • Agregació de Logs: Recollir logs de diversos serveis i aplicacions en temps real.
  • Recollida de Mètriques: Recollir mètriques de diferents sistemes per monitoritzar el rendiment.
  • Sourcing d'Esdeveniments: Capturar canvis en l'estat d'una aplicació com una seqüència d'esdeveniments.

Exemple: Agregació de Logs

import org.apache.kafka.clients.producer.KafkaProducer;
import org.apache.kafka.clients.producer.ProducerRecord;
import java.util.Properties;

public class LogProducer {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

        KafkaProducer<String, String> producer = new KafkaProducer<>(props);

        for (int i = 0; i < 100; i++) {
            producer.send(new ProducerRecord<>("logs", "logKey", "Missatge de log " + i));
        }

        producer.close();
    }
}

En aquest exemple, un productor de Kafka senzill envia missatges de log a un tema de Kafka anomenat "logs".

2. Missatgeria

Kafka es pot utilitzar com un sistema de missatgeria per desacoblar la producció i el consum de missatges.

  • Desacoblament de Sistemes: Permet l'escalat independent de productors i consumidors.
  • Durabilitat: Els missatges es guarden en disc i es repliquen per tolerància a fallades.
  • Alt Rendiment: Capacitat per gestionar grans volums de missatges.

Exemple: Sistema de Missatgeria Simple

import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.apache.kafka.clients.consumer.ConsumerRecords;
import org.apache.kafka.clients.consumer.KafkaConsumer;
import java.util.Collections;
import java.util.Properties;

public class MessageConsumer {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("group.id", "test-group");
        props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
        props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

        KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
        consumer.subscribe(Collections.singletonList("messages"));

        while (true) {
            ConsumerRecords<String, String> records = consumer.poll(100);
            for (ConsumerRecord<String, String> record : records) {
                System.out.printf("Missatge consumit: %s%n", record.value());
            }
        }
    }
}

Aquest exemple demostra un consumidor de Kafka que llegeix missatges d'un tema anomenat "messages".

3. Integració de Dades

Kafka es pot utilitzar per integrar dades de diverses fonts i fer-les disponibles per a diferents sistemes.

  • ETL (Extracció, Transformació, Càrrega): Kafka es pot utilitzar per extreure dades de diverses fonts, transformar-les i carregar-les en sistemes de destinació.
  • Replicació de Dades: Replicar dades entre diferents bases de dades o centres de dades.
  • Comunicació entre Microserveis: Facilitar la comunicació entre microserveis.

Exemple: Canal ETL

import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.apache.kafka.clients.consumer.ConsumerRecords;
import org.apache.kafka.clients.consumer.KafkaConsumer;
import org.apache.kafka.clients.producer.KafkaProducer;
import org.apache.kafka.clients.producer.ProducerRecord;
import java.util.Collections;
import java.util.Properties;

public class ETLProcessor {
    public static void main(String[] args) {
        // Propietats del consumidor
        Properties consumerProps = new Properties();
        consumerProps.put("bootstrap.servers", "localhost:9092");
        consumerProps.put("group.id", "etl-group");
        consumerProps.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
        consumerProps.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

        // Propietats del productor
        Properties producerProps = new Properties();
        producerProps.put("bootstrap.servers", "localhost:9092");
        producerProps.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        producerProps.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

        KafkaConsumer<String, String> consumer = new KafkaConsumer<>(consumerProps);
        KafkaProducer<String, String> producer = new KafkaProducer<>(producerProps);

        consumer.subscribe(Collections.singletonList("source-topic"));

        while (true) {
            ConsumerRecords<String, String> records = consumer.poll(100);
            for (ConsumerRecord<String, String> record : records) {
                String transformedValue = transform(record.value());
                producer.send(new ProducerRecord<>("target-topic", record.key(), transformedValue));
            }
        }
    }

    private static String transform(String value) {
        // Lògica de transformació simple
        return value.toUpperCase();
    }
}

En aquest exemple, un consumidor de Kafka llegeix dades d'un "source-topic", les transforma, i després un productor de Kafka envia les dades transformades a un "target-topic".

4. Processament de Fluxos

L'API de Kafka Streams et permet construir aplicacions que processen dades en temps real.

  • Anàlisi en Temps Real: Analitzar dades a mesura que arriben.
  • Monitorització i Alertes: Monitoritzar mètriques del sistema i generar alertes.
  • Enriquiment de Dades: Enriquir fluxos de dades amb informació addicional.

Exemple: Anàlisi en Temps Real

import org.apache.kafka.common.serialization.Serdes;
import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.KStream;
import org.apache.kafka.streams.kstream.KTable;
import org.apache.kafka.streams.kstream.Materialized;
import org.apache.kafka.streams.kstream.Produced;

public class RealTimeAnalytics {
    public static void main(String[] args) {
        StreamsBuilder builder = new StreamsBuilder();

        KStream<String, String> input = builder.stream("input-topic");
        KTable<String, Long> wordCounts = input
                .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
                .groupBy((key, value) -> value)
                .count(Materialized.as("counts-store"));

        wordCounts.toStream().to("output-topic", Produced.with(Serdes.String(), Serdes.Long()));

        KafkaStreams streams = new KafkaStreams(builder.build(), getStreamsConfig());
        streams.start();
    }

    private static Properties getStreamsConfig() {
        Properties props = new Properties();
        props.put("application.id", "real-time-analytics");
        props.put("bootstrap.servers", "localhost:9092");
        props.put("default.key.serde", Serdes.String().getClass().getName());
        props.put("default.value.serde", Serdes.String().getClass().getName());
        return props;
    }
}

Aquest exemple demostra una aplicació de Kafka Streams que realitza un recompte de paraules en missatges d'un "input-topic" i escriu els resultats en un "output-topic".

Conclusió

Kafka és una eina versàtil que es pot utilitzar per a una àmplia gamma de casos d'ús, des del streaming de dades en temps real i la missatgeria fins a la integració de dades i el processament de fluxos. Entenent aquests casos d'ús i explorant els exemples proporcionats, pots aprofitar Kafka per construir aplicacions basades en dades robustes i escalables.
