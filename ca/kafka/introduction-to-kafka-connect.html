Introducció a Kafka Connect
Connectors de font i destí ►

Visió general

Kafka Connect és un marc per connectar Kafka amb sistemes externs com bases de dades, magatzems de claus-valor, índexs de cerca i sistemes de fitxers. Simplifica el procés d'integració de Kafka amb altres sistemes, permetent pipelines de dades escalables i fiables.

Conceptes clau

  • Connector: Un connector és un component reutilitzable que captura dades d'un sistema font o envia dades a un sistema objectiu.
    • Source Connector: Captura dades d'un sistema extern i les escriu a temes de Kafka.
    • Sink Connector: Llegeix dades de temes de Kafka i les escriu a un sistema extern.
  • Task: Una tasca és una unitat de treball que un connector realitza. Els connectors es poden configurar per executar múltiples tasques en paral·lel per a l'escalabilitat.
  • Worker: Un worker és un procés JVM que executa connectors i tasques. Els workers es poden executar en mode autònom o distribuït.
  • Standalone Mode: Adequat per a configuracions senzilles d'un sol node.
  • Distributed Mode: Adequat per a configuracions a gran escala i tolerants a fallades.

Configuració de Kafka Connect

Prerequisits

  • Apache Kafka instal·lat i en funcionament.
  • Java Development Kit (JDK) instal·lat.

Exemple en mode autònom

  1. Crea un fitxer de propietats per al connector:

    name=local-file-source
connector.class=FileStreamSource
tasks.max=1
file=/path/to/input/file.txt
topic=connect-test

  2. Executa Kafka Connect en mode autònom:

    bin/connect-standalone.sh config/connect-standalone.properties config/connect-file-source.properties

Exemple en mode distribuït

  1. Crea un fitxer de propietats per al worker:

    bootstrap.servers=localhost:9092
group.id=connect-cluster
key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter
config.storage.topic=connect-configs
offset.storage.topic=connect-offsets
status.storage.topic=connect-status

  2. Inicia el worker de Kafka Connect:

    bin/connect-distributed.sh config/connect-distributed.properties

  3. Envia una configuració de connector:

    curl -X POST -H "Content-Type: application/json" --data '{
    "name": "local-file-source",
    "config": {
        "connector.class": "FileStreamSource",
        "tasks.max": "1",
        "file": "/path/to/input/file.txt",
        "topic": "connect-test"
    }
}' http://localhost:8083/connectors

Connectors comuns

Source Connectors

  • JDBC Source Connector: Captura dades de bases de dades relacionals.
  • File Source Connector: Captura dades de fitxers.

Sink Connectors

  • JDBC Sink Connector: Escriu dades a bases de dades relacionals.
  • Elasticsearch Sink Connector: Escriu dades a Elasticsearch.

Exemple: JDBC Source Connector

Configuració

  1. Crea un fitxer de propietats per al JDBC Source Connector:

    name=my-jdbc-source
connector.class=io.confluent.connect.jdbc.JdbcSourceConnector
tasks.max=1
connection.url=jdbc:mysql://localhost:3306/mydb
connection.user=myuser
connection.password=mypassword
table.whitelist=mytable
mode=incrementing
incrementing.column.name=id
topic.prefix=mydb-

  2. Inicia el connector:

    bin/connect-standalone.sh config/connect-standalone.properties config/my-jdbc-source.properties

Monitorització i gestió

  • REST API: Kafka Connect proporciona una API REST per gestionar connectors, tasques i workers.
  • Mètriques: Kafka Connect exposa mètriques via JMX per monitoritzar el rendiment i la salut.

Conclusió

Kafka Connect és una eina poderosa per integrar Kafka amb diversos sistemes externs. Entenent els seus conceptes clau i configuracions, pots construir pipelines de dades escalables i fiables. Tant si treballes en mode autònom com en mode distribuït, Kafka Connect simplifica el procés d'ingestió i exportació de dades, fent-lo un component essencial en l'ecosistema de Kafka.
