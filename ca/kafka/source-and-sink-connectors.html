Apache Kafka Connect és una eina poderosa per transmetre dades entre Apache Kafka i altres sistemes. Proporciona un marc per definir connectors que mouen grans col·leccions de dades dins i fora de Kafka. En aquesta secció, ens centrarem en els Connectors d'Origen i Destí, que són components fonamentals de Kafka Connect.

Què són els Connectors d'Origen i Destí?

  • Connectors d'Origen: Aquests connectors s'utilitzen per extreure dades de sistemes externs cap a temes de Kafka.
  • Connectors de Destí: Aquests connectors s'utilitzen per enviar dades des de temes de Kafka a sistemes externs.

Connectors d'Origen

Els connectors d'origen estan dissenyats per importar dades de diverses fonts de dades a Kafka. Això podria ser bases de dades, cues de missatges, fitxers o qualsevol altra font de dades.

Conceptes Clau

  • Polling: Els connectors d'origen solen fer polling a la font de dades a intervals regulars per obtenir noves dades.
  • Offsets: Kafka Connect fa un seguiment dels offsets de les dades que ha llegit per assegurar-se que no es perdi ni es dupliqui cap dada.

Exemple: Connector d'Origen JDBC

El Connector d'Origen JDBC et permet importar dades de qualsevol base de dades relacional amb un controlador JDBC.

{
  "name": "jdbc-source-connector",
  "config": {
    "connector.class": "io.confluent.connect.jdbc.JdbcSourceConnector",
    "tasks.max": "1",
    "connection.url": "jdbc:mysql://localhost:3306/mydatabase",
    "connection.user": "myuser",
    "connection.password": "mypassword",
    "table.whitelist": "mytable",
    "mode": "incrementing",
    "incrementing.column.name": "id",
    "topic.prefix": "jdbc-"
  }
}
  • connector.class: Especifica la classe per al Connector d'Origen JDBC.
  • tasks.max: El nombre màxim de tasques a utilitzar per aquest connector.
  • connection.url: La URL de connexió JDBC.
  • connection.user: L'usuari de la base de dades.
  • connection.password: La contrasenya de la base de dades.
  • table.whitelist: Les taules a incloure.
  • mode: El mode per a la recuperació incremental (per exemple, incrementing, timestamp, etc.).
  • incrementing.column.name: La columna a utilitzar per al mode incremental.
  • topic.prefix: Prefix per als noms dels temes de Kafka.

Connectors de Destí

Els connectors de destí estan dissenyats per exportar dades des de temes de Kafka a diversos sistemes externs. Això podria ser bases de dades, índexs de cerca, sistemes de fitxers o qualsevol altre destí de dades.

Conceptes Clau

  • Semàntica de Lliurament: Els connectors de destí es poden configurar per assegurar una semàntica de lliurament almenys una vegada o exactament una vegada.
  • Agrupació: Els connectors de destí sovint agrupen registres per millorar el rendiment i reduir la càrrega en el sistema de destinació.

Exemple: Connector de Destí Elasticsearch

El Connector de Destí Elasticsearch et permet exportar dades des de temes de Kafka a un índex d'Elasticsearch.

{
  "name": "elasticsearch-sink-connector",
  "config": {
    "connector.class": "io.confluent.connect.elasticsearch.ElasticsearchSinkConnector",
    "tasks.max": "1",
    "topics": "my-topic",
    "connection.url": "http://localhost:9200",
    "type.name": "_doc",
    "key.ignore": "true",
    "schema.ignore": "true",
    "batch.size": "2000"
  }
}
  • connector.class: Especifica la classe per al Connector de Destí Elasticsearch.
  • tasks.max: El nombre màxim de tasques a utilitzar per aquest connector.
  • topics: Els temes de Kafka dels quals llegir.
  • connection.url: La URL de connexió d'Elasticsearch.
  • type.name: El tipus de document d'Elasticsearch.
  • key.ignore: Si s'ha d'ignorar la clau del registre.
  • schema.ignore: Si s'ha d'ignorar l'esquema.
  • batch.size: El nombre de registres a agrupar abans d'enviar a Elasticsearch.

Comparació de Connectors d'Origen i Destí

| Característica | Connectors d'Origen | Connectors de Destí | |------------------------|--------------------------------------------|--------------------------------------------| | Direcció | Sistema Extern -> Kafka | Kafka -> Sistema Extern | | Usos Comuns | Importar dades de bases de dades, fitxers, etc. | Exportar dades a bases de dades, índexs de cerca, etc. | | Configuració Clau | Intervals de polling, offsets | Semàntica de lliurament, agrupació | | Connectors Exemple | Connector d'Origen JDBC, Connector d'Origen de Fitxers | Connector de Destí Elasticsearch, Connector de Destí JDBC |

Conclusió

Els Connectors d'Origen i Destí són components essencials de Kafka Connect, permetent una integració de dades sense problemes entre Kafka i diversos sistemes externs. Els connectors d'origen extreuen dades cap a Kafka, mentre que els connectors de destí envien dades fora de Kafka. Entendre com configurar i utilitzar aquests connectors és crucial per construir canals de dades robustos.

En dominar els Connectors d'Origen i Destí, pots aprofitar Kafka Connect per optimitzar el moviment de dades i assegurar la consistència de dades a través dels teus sistemes.
