Introducció

Apache Kafka és una plataforma de streaming distribuïda que et permet publicar i subscriure't a fluxos de registres. L'API del consumidor és una part crucial de Kafka, permetent a les aplicacions llegir registres dels temes de Kafka. Aquesta secció proporcionarà una visió general de l'API del consumidor, els seus conceptes clau i com utilitzar-la de manera efectiva.

Conceptes clau

  • Grup de consumidors: Un grup de consumidors que treballen junts per consumir registres d'un conjunt de temes. Cada consumidor del grup llegeix un subconjunt de les particions.
  • Offset: Un identificador únic per a cada registre dins d'una partició. Els consumidors utilitzen offsets per fer un seguiment dels registres que s'han consumit.
  • Reequilibri: El procés de redistribuir particions entre consumidors en un grup quan canvia la membresia del grup.
  • Commit: L'acció de guardar l'offset actual d'un consumidor, assegurant que el consumidor pugui reprendre des d'aquest punt en cas de fallada.

Configuració d'un consumidor de Kafka

Per crear un consumidor de Kafka, cal configurar diverses propietats i utilitzar la classe KafkaConsumer. A continuació es mostra un exemple bàsic en Java.

Exemple: Consumidor bàsic de Kafka en Java

import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.apache.kafka.clients.consumer.ConsumerRecords;
import org.apache.kafka.clients.consumer.KafkaConsumer;

import java.time.Duration;
import java.util.Collections;
import java.util.Properties;

public class BasicKafkaConsumer {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("group.id", "test-group");
        props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
        props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

        KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
        consumer.subscribe(Collections.singletonList("test-topic"));

        try {
            while (true) {
                ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
                for (ConsumerRecord<String, String> record : records) {
                    System.out.printf("offset = %d, key = %s, value = %s%n", record.offset(), record.key(), record.value());
                }
            }
        } finally {
            consumer.close();
        }
    }
}

Explicació

  • Configuració de propietats: L'objecte Properties s'utilitza per configurar el consumidor. Les propietats clau inclouen bootstrap.servers, group.id, key.deserializer i value.deserializer.
  • Inicialització de KafkaConsumer: La classe KafkaConsumer s'instancia amb les propietats configurades.
  • Subscripció: El consumidor es subscriu a un tema utilitzant el mètode subscribe.
  • Polling: El mètode poll s'utilitza per obtenir registres del tema. L'argument Duration.ofMillis(100) especifica el temps d'espera per a l'operació de polling.
  • Processament de registres: Els registres es processen en un bucle, imprimint l'offset, la clau i el valor de cada registre.
  • Tancament ordenat: El consumidor es tanca en el bloc finally per assegurar que els recursos es tanquin correctament.

Funcions avançades del consumidor

Control manual d'offsets

Per defecte, Kafka fa commits d'offsets automàticament. No obstant això, pots controlar els offsets manualment per tenir un control més precís sobre el processament de registres.

consumer.commitSync();

Oients de reequilibri

Pots implementar un ConsumerRebalanceListener per gestionar les reassigancions de particions durant el reequilibri.

consumer.subscribe(Collections.singletonList("test-topic"), new ConsumerRebalanceListener() {
    @Override
    public void onPartitionsRevoked(Collection<TopicPartition> partitions) {
        // Gestionar la revocació de particions
    }

    @Override
    public void onPartitionsAssigned(Collection<TopicPartition> partitions) {
        // Gestionar l'assignació de particions
    }
});

Gestió d'errors

La gestió adequada d'errors és crucial per a aplicacions de consumidors robustes. Utilitza blocs try-catch i gestiona excepcions específiques com WakeupException.

try {
    while (true) {
        ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
        // Processar registres
    }
} catch (WakeupException e) {
    // Gestionar el tancament
} finally {
    consumer.close();
}

Conclusió

L'API del consumidor de Kafka és una eina poderosa per llegir registres dels temes de Kafka. Entendre els seus conceptes clau, com els grups de consumidors, els offsets i el reequilibri, és essencial per construir aplicacions de consumidors eficients i fiables. Dominant tant les funcions bàsiques com les avançades, pots assegurar que els teus consumidors de Kafka siguin robustos i rendibles.
