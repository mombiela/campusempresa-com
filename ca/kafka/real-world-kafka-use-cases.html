Apache Kafka és una eina poderosa per construir canals de dades en temps real i aplicacions de streaming. Aquesta secció explora diversos casos d'ús reals de Kafka, demostrant la seva versatilitat i robustesa en diferents indústries.

1. Agregació de Logs

L'agregació de logs és un dels casos d'ús més comuns per Kafka. Implica recollir dades de logs de diversos serveis i aplicacions, centralitzant-les per a l'anàlisi i el monitoratge.

  • Conceptes Clau:

    • Productors: Aplicacions que generen dades de logs.
    • Temes: Canals on es publiquen les dades de logs.
    • Consumidors: Sistemes que llegeixen i processen les dades de logs.

  • Exemple:

    // Exemple de productor per a l'agregació de logs
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

Producer<String, String> producer = new KafkaProducer<>(props);
for (int i = 0; i < 100; i++) {
    producer.send(new ProducerRecord<>("logs", Integer.toString(i), "Missatge de log " + i));
}
producer.close();

2. Analítica en Temps Real

Kafka és àmpliament utilitzat per a l'analítica en temps real, on les dades es processen i analitzen a mesura que es generen.

  • Conceptes Clau:

    • Processament de Fluxos: Processament continu de fluxos de dades.
    • API de Kafka Streams: Una biblioteca client per construir aplicacions i microserveis.

  • Exemple:

    // Exemple de Kafka Streams per a l'analítica en temps real
Properties props = new Properties();
props.put(StreamsConfig.APPLICATION_ID_CONFIG, "streams-analytics");
props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, Serdes.String().getClass().getName());
props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, Serdes.String().getClass().getName());

StreamsBuilder builder = new StreamsBuilder();
KStream<String, String> source = builder.stream("input-topic");
KStream<String, String> transformed = source.mapValues(value -> "Processat " + value);
transformed.to("output-topic");

KafkaStreams streams = new KafkaStreams(builder.build(), props);
streams.start();

3. Sourcing d'Esdeveniments

El sourcing d'esdeveniments és un patró de disseny on els canvis d'estat es registren com una seqüència d'esdeveniments. Kafka és una excel·lent opció per a aquest patró a causa de la seva durabilitat i escalabilitat.

  • Conceptes Clau:

    • Esdeveniments: Registres immutables de canvis d'estat.
    • Magatzem d'Esdeveniments: Temes de Kafka que emmagatzemen esdeveniments.

  • Exemple:

    // Exemple de productor per al sourcing d'esdeveniments
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

Producer<String, String> producer = new KafkaProducer<>(props);
producer.send(new ProducerRecord<>("events", "user-1", "UserCreated"));
producer.send(new ProducerRecord<>("events", "user-1", "UserUpdated"));
producer.close();

4. Missatgeria

Kafka es pot utilitzar com a intermediari de missatges, similar als sistemes de missatgeria tradicionals com RabbitMQ o ActiveMQ.

  • Conceptes Clau:

    • Intermediari de Missatges: Middleware que tradueix missatges entre diferents serveis.
    • Model de Publicació/Subscripció: Els productors publiquen missatges en temes, i els consumidors es subscriuen a aquests temes.

  • Exemple:

    // Exemple de consumidor per a la missatgeria
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "test-group");
props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Collections.singletonList("messages"));

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
    for (ConsumerRecord<String, String> record : records) {
        System.out.printf("offset = %d, key = %s, value = %s%n", record.offset(), record.key(), record.value());
    }
}

5. Integració de Dades

Kafka s'utilitza sovint per integrar diverses fonts i destinacions de dades, permetent un flux de dades sense problemes entre diferents sistemes.

  • Conceptes Clau:

    • Connectors: Components preconstruïts per integrar-se amb sistemes externs.
    • Kafka Connect: Un marc per connectar Kafka amb sistemes externs.

  • Exemple:

    // Configuració de Kafka Connect per a un connector de font JDBC
{
  "name": "jdbc-source",
  "config": {
    "connector.class": "io.confluent.connect.jdbc.JdbcSourceConnector",
    "tasks.max": "1",
    "connection.url": "jdbc:mysql://localhost:3306/mydb",
    "connection.user": "user",
    "connection.password": "password",
    "table.whitelist": "my_table",
    "mode": "incrementing",
    "incrementing.column.name": "id",
    "topic.prefix": "jdbc-"
  }
}

Conclusió

La versatilitat de Kafka el fa adequat per a una àmplia gamma de casos d'ús, des de l'agregació de logs i l'analítica en temps real fins al sourcing d'esdeveniments, la missatgeria i la integració de dades. Entendre aquestes aplicacions reals pot ajudar-te a aprofitar Kafka de manera efectiva en els teus propis projectes. Explorant aquests exemples i conceptes clau, pots veure com Kafka es pot aplicar per resoldre desafiaments complexos de dades en diverses indústries.
