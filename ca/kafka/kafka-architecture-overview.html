Introducció

Apache Kafka és una plataforma distribuïda de transmissió d'esdeveniments capaç de gestionar trilions d'esdeveniments al dia. S'utilitza per construir canals de dades en temps real i aplicacions de transmissió. Entendre l'arquitectura de Kafka és crucial per aprofitar efectivament les seves capacitats.

Components clau de l'arquitectura de Kafka

1. Temes i particions

  • Tema: Una categoria o nom de feed on s'emmagatzemen i es publiquen registres.
  • Partició: Un tema es divideix en particions, que són la unitat bàsica de paral·lelisme a Kafka.
Tema: Comandes
Particions: [Partició 0, Partició 1, Partició 2]

2. Productors

  • Els productors són clients que publiquen (escriuen) registres als temes de Kafka.
  • Els productors poden triar a quina partició enviar registres, sovint basant-se en una clau.
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

KafkaProducer<String, String> producer = new KafkaProducer<>(props);
ProducerRecord<String, String> record = new ProducerRecord<>("Comandes", "comanda123", "Detalls de la comanda");
producer.send(record);
producer.close();

3. Consumidors

  • Els consumidors són clients que llegeixen registres dels temes de Kafka.
  • Els consumidors pertanyen a grups de consumidors, cosa que permet l'equilibri de càrrega i la tolerància a fallades.
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "consumidors-comandes");
props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Arrays.asList("Comandes"));

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
    for (ConsumerRecord<String, String> record : records) {
        System.out.printf("Registre consumit amb clau %s i valor %s%n", record.key(), record.value());
    }
}

4. Intermediaris

  • Els intermediaris són servidors de Kafka que emmagatzemen dades i serveixen clients.
  • Un clúster de Kafka està compost per múltiples intermediaris.

5. Zookeeper

  • Zookeeper gestiona i coordina els intermediaris de Kafka.
  • Gestiona tasques com l'elecció de líder per a les particions i la gestió de configuració.

6. Replicació

  • Kafka assegura la tolerància a fallades replicant particions a través de múltiples intermediaris.
  • Cada partició té un líder i múltiples seguidors.

| Concepte | Descripció | |---------------|-----------------------------------------------------------------------------| | Líder | L'intermediari responsable de totes les lectures i escriptures per a la partició. | | Seguidor | Replia les dades del líder i pren el relleu si el líder falla. | | Factor de replicació | El nombre de còpies d'una partició a través del clúster de Kafka. |

7. Desplaçament

  • Un desplaçament és un identificador únic per a cada registre dins d'una partició.
  • Els consumidors utilitzen desplaçaments per fer un seguiment de la seva posició en el registre.

Resum

L'arquitectura de Kafka està dissenyada per a un alt rendiment, tolerància a fallades i escalabilitat. Els components clau inclouen temes, particions, productors, consumidors, intermediaris, Zookeeper, replicació i desplaçaments. Entendre aquests components i les seves interaccions és essencial per construir aplicacions robustes basades en Kafka.

Dominant l'arquitectura de Kafka, pots dissenyar i implementar eficientment solucions de transmissió de dades que satisfacin les necessitats de la teva organització.
