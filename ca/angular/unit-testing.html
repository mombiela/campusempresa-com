Introducció a les Proves Unitàries

Les proves unitàries són un mètode de prova de programari on les unitats o components individuals d'un programari es proven de manera aïllada de la resta de l'aplicació. A Angular, les proves unitàries es solen escriure amb Jasmine i s'executen amb Karma.

  • Jasmine: Un marc de desenvolupament dirigit pel comportament per provar codi JavaScript.
  • Karma: Un executor de proves que permet executar les teves proves en diversos navegadors.

Configuració de les Proves Unitàries a Angular

Els projectes Angular vénen preconfigurats amb Jasmine i Karma. Quan crees un nou projecte Angular utilitzant l'Angular CLI, els fitxers de configuració necessaris i les dependències s'inclouen automàticament.

ng new my-angular-app
cd my-angular-app
ng test

Escriure la Teva Primera Prova Unitària

Comencem escrivint una prova unitària simple per a un servei Angular.

Crear un Servei

Primer, crea un nou servei utilitzant l'Angular CLI:

ng generate service my-service

Això generarà un fitxer de servei (my-service.service.ts) i un fitxer de proves corresponent (my-service.service.spec.ts).

Escriure una Prova per al Servei

Obre el fitxer my-service.service.spec.ts i escriu una prova simple:

import { TestBed } from '@angular/core/testing';
import { MyService } from './my-service.service';

describe('MyService', () => {
  let service: MyService;

  beforeEach(() => {
    TestBed.configureTestingModule({});
    service = TestBed.inject(MyService);
  });

  it('should be created', () => {
    expect(service).toBeTruthy();
  });

  it('should return correct value from getValue', () => {
    expect(service.getValue()).toBe('some value');
  });
});

Explicació

  • TestBed: L'API principal d'Angular per configurar i inicialitzar l'entorn per a les proves unitàries.
  • describe: Una funció de Jasmine per agrupar proves relacionades.
  • it: Una funció de Jasmine per definir casos de prova individuals.
  • expect: Una funció de Jasmine per fer afirmacions.

Proves de Components

Les proves unitàries de components impliquen provar la classe del component i la plantilla.

Crear un Component

Genera un nou component utilitzant l'Angular CLI:

ng generate component my-component

Escriure una Prova per al Component

Obre el fitxer my-component.component.spec.ts i escriu una prova:

import { ComponentFixture, TestBed } from '@angular/core/testing';
import { MyComponent } from './my-component.component';

describe('MyComponent', () => {
  let component: MyComponent;
  let fixture: ComponentFixture<MyComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ MyComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(MyComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should render title', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('h1').textContent).toContain('My Component Title');
  });
});

Explicació

  • ComponentFixture: Un embolcall al voltant d'un component i la seva plantilla.
  • compileComponents: Compila el component i la seva plantilla.
  • fixture.detectChanges: Activa la detecció de canvis i actualitza la vista.

Tècniques Avançades de Prova

Simulació de Dependències

En aplicacions del món real, els components i serveis sovint depenen d'altres serveis. Simular aquestes dependències és crucial per aïllar la unitat sota prova.

import { TestBed } from '@angular/core/testing';
import { MyService } from './my-service.service';
import { MyComponent } from './my-component.component';

class MockMyService {
  getValue() {
    return 'mock value';
  }
}

describe('MyComponent', () => {
  let component: MyComponent;
  let service: MyService;

  beforeEach(() => {
    TestBed.configureTestingModule({
      declarations: [ MyComponent ],
      providers: [
        { provide: MyService, useClass: MockMyService }
      ]
    });

    service = TestBed.inject(MyService);
    component = TestBed.createComponent(MyComponent).componentInstance;
  });

  it('should use the mock service', () => {
    expect(service.getValue()).toBe('mock value');
  });
});

Provar Codi Asíncron

Provar codi asíncron es pot fer utilitzant les utilitats async i fakeAsync proporcionades per Angular.

import { fakeAsync, tick } from '@angular/core/testing';

it('should handle async code', fakeAsync(() => {
  let flag = false;
  setTimeout(() => {
    flag = true;
  }, 1000);

  tick(1000);
  expect(flag).toBe(true);
}));

Conclusió

Les proves unitàries són una part essencial del desenvolupament d'aplicacions Angular robustes. Escriure proves per a serveis i components, i utilitzar tècniques avançades com la simulació i la prova de codi asíncron, et permet assegurar que la teva aplicació es comporta com s'espera. Comença amb proves simples i gradualment passa a escenaris més complexos a mesura que et familiaritzis amb les eines i metodologies de prova.
