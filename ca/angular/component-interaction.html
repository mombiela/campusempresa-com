Interacció entre components
Directives integrades ►

La interacció entre components és un concepte fonamental en Angular que permet que diferents components es comuniquin i comparteixin dades. Aquest tema cobrirà diversos mètodes d'interacció entre components, des de la comunicació simple entre pare i fill fins a tècniques més avançades com l'ús de serveis i la gestió d'estat.

Comunicació de pare a fill

La comunicació de pare a fill és una de les formes més comunes d'interacció entre components en Angular. Això s'aconsegueix típicament utilitzant propietats d'entrada.

Propietats d'entrada

  • Definició: Les propietats d'entrada permeten que un component pare passi dades a un component fill.
  • Ús: Utilitza el decorador @Input per definir una propietat d'entrada en el component fill.

Exemple

Plantilla del component pare (parent.component.html):

<app-child [childProperty]="parentData"></app-child>

Classe del component pare (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  parentData = 'Dades del pare';
}

Classe del component fill (child.component.ts):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-child',
  template: '<p>{{ childProperty }}</p>',
})
export class ChildComponent {
  @Input() childProperty: string;
}

En aquest exemple, el component pare passa les parentData al component fill utilitzant la propietat d'entrada childProperty.

Comunicació de fill a pare

La comunicació de fill a pare sovint s'aconsegueix utilitzant propietats de sortida i emissors d'esdeveniments.

Propietats de sortida i emissors d'esdeveniments

  • Definició: Les propietats de sortida permeten que un component fill emeti esdeveniments que un component pare pot escoltar.
  • Ús: Utilitza el decorador @Output juntament amb un EventEmitter per definir una propietat de sortida en el component fill.

Exemple

Classe del component fill (child.component.ts):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-child',
  template: '<button (click)="sendData()">Enviar dades</button>',
})
export class ChildComponent {
  @Output() dataEmitter = new EventEmitter<string>();

  sendData() {
    this.dataEmitter.emit('Dades del fill');
  }
}

Plantilla del component pare (parent.component.html):

<app-child (dataEmitter)="receiveData($event)"></app-child>

Classe del component pare (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  receiveData(data: string) {
    console.log(data);
  }
}

En aquest exemple, el component fill emet un esdeveniment amb dades, que el component pare escolta i gestiona.

Comunicació entre germans

Els components germans poden comunicar-se a través d'un component pare compartit o un servei comú.

Utilitzant un component pare compartit

  • Definició: Els components germans poden comunicar-se passant dades a través del seu component pare compartit.
  • Ús: El component pare actua com a mediador, rebent dades d'un fill i passant-les a un altre.

Exemple

Plantilla del component pare (parent.component.html):

<app-sibling-one (dataEmitter)="receiveData($event)"></app-sibling-one>
<app-sibling-two [siblingData]="sharedData"></app-sibling-two>

Classe del component pare (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  sharedData: string;

  receiveData(data: string) {
    this.sharedData = data;
  }
}

Classe del component germà un (sibling-one.component.ts):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-sibling-one',
  template: '<button (click)="sendData()">Enviar dades</button>',
})
export class SiblingOneComponent {
  @Output() dataEmitter = new EventEmitter<string>();

  sendData() {
    this.dataEmitter.emit('Dades del germà un');
  }
}

Classe del component germà dos (sibling-two.component.ts):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-sibling-two',
  template: '<p>{{ siblingData }}</p>',
})
export class SiblingTwoComponent {
  @Input() siblingData: string;
}

Utilitzant un servei

  • Definició: Un servei pot ser utilitzat per compartir dades i facilitar la comunicació entre components germans.
  • Ús: Crea un servei amb dades compartides i mètodes per actualitzar i recuperar aquestes dades.

Exemple

Servei compartit (shared.service.ts):

import { Injectable } from '@angular/core';
import { Subject } from 'rxjs';

@Injectable({
  providedIn: 'root',
})
export class SharedService {
  private dataSubject = new Subject<string>();
  data$ = this.dataSubject.asObservable();

  sendData(data: string) {
    this.dataSubject.next(data);
  }
}

Classe del component germà un (sibling-one.component.ts):

import { Component } from '@angular/core';
import { SharedService } from './shared.service';

@Component({
  selector: 'app-sibling-one',
  template: '<button (click)="sendData()">Enviar dades</button>',
})
export class SiblingOneComponent {
  constructor(private sharedService: SharedService) {}

  sendData() {
    this.sharedService.sendData('Dades del germà un');
  }
}

Classe del component germà dos (sibling-two.component.ts):

import { Component, OnInit } from '@angular/core';
import { SharedService } from './shared.service';

@Component({
  selector: 'app-sibling-two',
  template: '<p>{{ siblingData }}</p>',
})
export class SiblingTwoComponent implements OnInit {
  siblingData: string;

  constructor(private sharedService: SharedService) {}

  ngOnInit() {
    this.sharedService.data$.subscribe(data => {
      this.siblingData = data;
    });
  }
}

En aquest exemple, el SharedService facilita la comunicació entre SiblingOneComponent i SiblingTwoComponent.

Conclusió

La interacció entre components és un aspecte crucial per construir aplicacions Angular. Entendre com comunicar-se efectivament entre components utilitzant propietats d'entrada i sortida, components pares compartits i serveis et permetrà crear aplicacions més dinàmiques i responsives. A mesura que progressis des de nivells principiants fins a avançats, dominar aquestes tècniques millorarà significativament la teva capacitat per gestionar i estructurar els teus projectes Angular.
