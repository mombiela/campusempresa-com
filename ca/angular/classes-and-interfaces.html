Introducció

A Angular, entendre les classes i les interfícies és crucial per construir aplicacions robustes i mantenibles. Aquesta secció cobrirà els conceptes bàsics de les classes i les interfícies, les seves diferències i com utilitzar-les de manera efectiva en projectes Angular.

Classes a Angular

Les classes a Angular són plànols per crear objectes. Encapsulen dades i comportament, facilitant la gestió i manipulació de dades dins de la teva aplicació.

Conceptes clau

  • Declaració de classe: Defineix una classe utilitzant la paraula clau class.
  • Constructor: Un mètode especial per crear i inicialitzar un objecte creat amb una classe.
  • Propietats: Variables que pertanyen a la classe.
  • Mètodes: Funcions que pertanyen a la classe.

Exemple

class User {
  // Propietats
  name: string;
  age: number;

  // Constructor
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  // Mètode
  greet() {
    return `Hello, my name is ${this.name} and I am ${this.age} years old.`;
  }
}

// Creant una instància de la classe User
const user = new User('Alice', 30);
console.log(user.greet()); // Output: Hello, my name is Alice and I am 30 years old.

Explicació

  • Declaració de classe: class User defineix una nova classe anomenada User.
  • Propietats: name i age són propietats de la classe User.
  • Constructor: El constructor inicialitza les propietats quan es crea una nova instància de la classe.
  • Mètode: greet és un mètode que retorna un missatge de salutació.

Interfícies a Angular

Les interfícies a Angular defineixen l'estructura d'un objecte. S'utilitzen per imposar una forma específica als objectes, assegurant que tinguin les propietats i mètodes requerits.

Conceptes clau

  • Declaració d'interfície: Defineix una interfície utilitzant la paraula clau interface.
  • Propietats: Variables que l'interfície espera.
  • Mètodes: Funcions que l'interfície espera.

Exemple

interface IUser {
  name: string;
  age: number;
  greet(): string;
}

class User implements IUser {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet() {
    return `Hello, my name is ${this.name} and I am ${this.age} years old.`;
  }
}

// Creant una instància de la classe User
const user = new User('Bob', 25);
console.log(user.greet()); // Output: Hello, my name is Bob and I am 25 years old.

Explicació

  • Declaració d'interfície: interface IUser defineix una nova interfície anomenada IUser.
  • Propietats i mètodes: La interfície especifica que qualsevol objecte de tipus IUser ha de tenir les propietats/mètodes name, age i greet.
  • Implementació de classe: class User implements IUser assegura que la classe User s'adhereix a la interfície IUser.

Diferències entre classes i interfícies

| Característica | Classe | Interfície | |-----------------|------------------------------------------|------------------------------------------| | Instanciació | Es pot instanciar utilitzant la paraula clau new | No es pot instanciar | | Implementació | Pot implementar múltiples interfícies | No pot implementar altres interfícies | | Mètodes | Pot tenir implementacions de mètodes | Només signatures de mètodes, sense implementacions | | Ús | S'utilitza per crear objectes amb comportament | S'utilitza per definir la forma dels objectes |

Conclusió

Entendre les classes i les interfícies a Angular és essencial per crear codi ben estructurat i mantenible. Les classes et permeten crear objectes amb comportaments específics, mentre que les interfícies asseguren que els objectes s'adhereixin a una estructura definida. Dominant aquests conceptes, podràs construir aplicacions Angular més robustes i escalables.
