Introducció

Els formularis basats en plantilles són una manera simple i poderosa de gestionar l'entrada de l'usuari en aplicacions Angular. Es basen en l'enllaç de dades bidireccional d'Angular i directives per crear i gestionar formularis. Aquest enfocament és adequat per a formularis més simples i és més fàcil de configurar en comparació amb els formularis reactius.

Conceptes clau

  • Directiva NgModel: Enllaça els controls del formulari a les propietats del model.
  • FormsModule: Mòdul d'Angular necessari per utilitzar formularis basats en plantilles.
  • Validació de formularis: Validadors integrats i validadors personalitzats.
  • Enviament de formularis: Gestió de les dades del formulari en enviar-lo.

Configuració dels formularis basats en plantilles

Importació de FormsModule

Per utilitzar formularis basats en plantilles, cal importar el FormsModule en el vostre mòdul Angular.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { FormsModule } from '@angular/forms';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, FormsModule],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Creació d'un formulari bàsic

Aquí teniu un exemple simple d'un formulari basat en plantilles:

<!-- app.component.html -->
<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Nom:</label>
    <input type="text" id="name" name="name" ngModel required>
  </div>
  <div>
    <label for="email">Correu electrònic:</label>
    <input type="email" id="email" name="email" ngModel required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>

Gestió de l'enviament del formulari

En el vostre component, podeu gestionar l'enviament del formulari i accedir a les dades del formulari:

// app.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  onSubmit(form: any) {
    console.log('Dades del formulari:', form.value);
  }
}

Validació de formularis

Validadors integrats

Angular proporciona diversos validadors integrats com required, minlength, maxlength, email, etc.

<div>
  <label for="password">Contrasenya:</label>
  <input type="password" id="password" name="password" ngModel required minlength="8">
  <div *ngIf="userForm.controls.password?.invalid && userForm.controls.password?.touched">
    <small *ngIf="userForm.controls.password?.errors?.required">La contrasenya és obligatòria.</small>
    <small *ngIf="userForm.controls.password?.errors?.minlength">La contrasenya ha de tenir almenys 8 caràcters.</small>
  </div>
</div>

Validadors personalitzats

També podeu crear validadors personalitzats per a una lògica de validació més complexa.

// app.component.ts
import { Component } from '@angular/core';
import { NgForm } from '@angular/forms';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  onSubmit(form: NgForm) {
    console.log('Dades del formulari:', form.value);
  }

  validateUsername(control: any): { [key: string]: boolean } | null {
    const forbiddenUsernames = ['admin', 'user'];
    if (forbiddenUsernames.includes(control.value)) {
      return { 'forbiddenUsername': true };
    }
    return null;
  }
}
<div>
  <label for="username">Nom d'usuari:</label>
  <input type="text" id="username" name="username" ngModel #username="ngModel" [ngModelOptions]="{ updateOn: 'blur' }" [ngModel]="validateUsername">
  <div *ngIf="username.invalid && username.touched">
    <small *ngIf="username.errors?.forbiddenUsername">Aquest nom d'usuari no està permès.</small>
  </div>
</div>

Temes avançats

Enllaç de dades bidireccional

Els formularis basats en plantilles aprofiten l'enllaç de dades bidireccional d'Angular per sincronitzar els controls del formulari amb les propietats del model.

<!-- app.component.html -->
<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Nom:</label>
    <input type="text" id="name" name="name" [(ngModel)]="user.name" required>
  </div>
  <div>
    <label for="email">Correu electrònic:</label>
    <input type="email" id="email" name="email" [(ngModel)]="user.email" required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>
// app.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  user = {
    name: '',
    email: ''
  };

  onSubmit(form: any) {
    console.log('Dades del formulari:', form.value);
  }
}

Estat del formulari i classes CSS

Angular afegeix automàticament classes CSS als controls del formulari segons el seu estat (per exemple, ng-touched, ng-dirty, ng-valid, ng-invalid).

<div>
  <label for="name">Nom:</label>
  <input type="text" id="name" name="name" ngModel required>
  <div *ngIf="userForm.controls.name?.invalid && userForm.controls.name?.touched">
    <small *ngIf="userForm.controls.name?.errors?.required">El nom és obligatori.</small>
  </div>
</div>

Conclusió

Els formularis basats en plantilles en Angular proporcionen una manera senzilla de crear i gestionar formularis utilitzant les poderoses directives i l'enllaç de dades bidireccional d'Angular. Són ideals per a formularis més simples i ofereixen validació integrada, validació personalitzada i una fàcil gestió de l'estat del formulari. En entendre i utilitzar aquests conceptes, podeu gestionar eficientment l'entrada de l'usuari i la validació de formularis en les vostres aplicacions Angular.
