Introducció

A Angular, la injecció de dependències (DI) és un patró de disseny potent que permet que les classes rebin dependències d'una font externa en lloc de crear-les. Una de les característiques avançades del sistema DI d'Angular és el concepte d'injectors jeràrquics. Aquest tema et guiarà a través dels conceptes bàsics i avançats dels injectors jeràrquics a Angular.

Conceptes Clau

  • Injector: Un injector és un localitzador de serveis que és responsable d'instanciar dependències i injectar-les a les classes.
  • Injectors Jeràrquics: Els injectors d'Angular són jeràrquics, el que significa que formen una estructura d'arbre. Això permet que diferents injectors proporcionin diferents instàncies d'una dependència.

Com Funcionen els Injectors Jeràrquics

Les aplicacions Angular tenen un injector arrel i una jerarquia d'injectors fills. Cada component té el seu propi injector, que pot heretar dependències del seu injector pare o proporcionar les seves pròpies dependències.

Exemple: Injector Jeràrquic Bàsic

Considera el següent exemple on tenim un injector arrel i un injector fill:

import { Injectable, Component, NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';

@Injectable()
class RootService {
  constructor() {
    console.log('RootService instance created');
  }
}

@Injectable()
class ChildService {
  constructor() {
    console.log('ChildService instance created');
  }
}

@Component({
  selector: 'child-component',
  template: `<p>Child Component</p>`,
  providers: [ChildService]
})
class ChildComponent {
  constructor(private childService: ChildService) {}
}

@Component({
  selector: 'app-root',
  template: `<child-component></child-component>`,
  providers: [RootService]
})
class AppComponent {
  constructor(private rootService: RootService) {}
}

@NgModule({
  declarations: [AppComponent, ChildComponent],
  imports: [BrowserModule],
  bootstrap: [AppComponent]
})
class AppModule {}

En aquest exemple:

  • RootService es proporciona a nivell arrel.
  • ChildService es proporciona a nivell de ChildComponent.

Quan es creen AppComponent i ChildComponent, es creen i s'injecten els seus respectius serveis.

Proveïdors i Injectors Jeràrquics

Ús de Proveïdors `@Injectable` i `@Component`

Angular permet definir proveïdors a diferents nivells:

  • Nivell Arrel: Usant @Injectable({ providedIn: 'root' }).
  • Nivell de Component: Usant l'array providers en el decorador @Component.

Exemple: Proveïdor a Nivell Arrel

@Injectable({
  providedIn: 'root'
})
class GlobalService {
  constructor() {
    console.log('GlobalService instance created');
  }
}

@Component({
  selector: 'app-root',
  template: `<p>Root Component</p>`
})
class AppComponent {
  constructor(private globalService: GlobalService) {}
}

En aquest exemple, GlobalService es proporciona a nivell arrel, el que significa que serà un singleton i es compartirà a tota l'aplicació.

Exemple: Proveïdor a Nivell de Component

@Component({
  selector: 'child-component',
  template: `<p>Child Component</p>`,
  providers: [ChildService]
})
class ChildComponent {
  constructor(private childService: ChildService) {}
}

En aquest exemple, ChildService es proporciona a nivell de component, el que significa que cada instància de ChildComponent tindrà la seva pròpia instància de ChildService.

Comparació entre Proveïdors a Nivell Arrel i Nivell de Component

| Característica | Proveïdor a Nivell Arrel | Proveïdor a Nivell de Component | |--------------------------|---------------------------------------|---------------------------------------| | Abast | A tota l'aplicació | Limitat al component i els seus fills | | Creació d'Instància | Singleton | Nova instància per instància de component | | Cas d'Ús | Serveis compartits com logging, config| Serveis específics de component |

Exercici Pràctic

Exercici: Crear un Injector Jeràrquic

  1. Crea un nou servei Angular anomenat LoggingService i proporciona'l a nivell arrel.
  2. Crea un altre servei anomenat FeatureService i proporciona'l a un nivell de component específic.
  3. Injecta aquests serveis als seus respectius components i registra missatges a la consola per observar la creació d'instàncies.
// logging.service.ts
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class LoggingService {
  constructor() {
    console.log('LoggingService instance created');
  }
}

// feature.service.ts
import { Injectable } from '@angular/core';

@Injectable()
export class FeatureService {
  constructor() {
    console.log('FeatureService instance created');
  }
}

// feature.component.ts
import { Component } from '@angular/core';
import { FeatureService } from './feature.service';

@Component({
  selector: 'app-feature',
  template: `<p>Feature Component</p>`,
  providers: [FeatureService]
})
export class FeatureComponent {
  constructor(private featureService: FeatureService) {}
}

// app.component.ts
import { Component } from '@angular/core';
import { LoggingService } from './logging.service';

@Component({
  selector: 'app-root',
  template: `<app-feature></app-feature>`
})
export class AppComponent {
  constructor(private loggingService: LoggingService) {}
}

Conclusió

Els injectors jeràrquics a Angular proporcionen una manera flexible i potent de gestionar les dependències a la teva aplicació. Entenent com utilitzar proveïdors a nivell arrel i a nivell de component, pots controlar l'abast i el cicle de vida dels teus serveis de manera efectiva. Això permet una millor organització i modularització del teu codi, fent-lo més fàcil de mantenir i estendre.
