Introducció

La seguretat és un aspecte crític del desenvolupament web, i Angular proporciona diverses funcions integrades per ajudar els desenvolupadors a crear aplicacions segures. Aquesta secció cobrirà les millors pràctiques per assegurar les teves aplicacions Angular contra vulnerabilitats i atacs comuns.

Política de Seguretat de Contingut (CSP)

  • Definició: CSP és una funció de seguretat que ajuda a prevenir diversos tipus d'atacs, com ara Cross-Site Scripting (XSS) i atacs d'injecció de dades.
  • Implementació: Defineix una CSP en les teves capçaleres HTTP per restringir les fonts des de les quals es pot carregar contingut.
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' https://apis.google.com">

Prevenció de Cross-Site Scripting (XSS)

  • Sanititzar l'Entrada de l'Usuari: Sempre sanititza les entrades dels usuaris per evitar que s'executin scripts maliciosos.
  • Sanitització Integrada d'Angular: Angular sanititza automàticament l'enllaç de dades per prevenir atacs XSS.
<!-- Angular escapa automàticament el contingut -->
<div [innerHTML]="userInput"></div>
  • Ús de DomSanitizer: Per als casos en què necessitis evitar la sanitització d'Angular, utilitza el servei DomSanitizer amb precaució.
import { DomSanitizer, SafeHtml } from '@angular/platform-browser';

constructor(private sanitizer: DomSanitizer) {}

public getSanitizedHtml(html: string): SafeHtml {
  return this.sanitizer.bypassSecurityTrustHtml(html);
}

Protecció contra Cross-Site Request Forgery (CSRF)

  • Definició: Els atacs CSRF enganyen l'usuari perquè executi accions no desitjades en una aplicació web en la qual està autenticat.
  • HttpClient d'Angular: Utilitza el HttpClient d'Angular que inclou automàticament tokens CSRF en les sol·licituds HTTP.
import { HttpClient } from '@angular/common/http';

constructor(private http: HttpClient) {}

this.http.post('/api/data', { data: 'example' }).subscribe(response => {
  console.log(response);
});

Autenticació i Autorització

  • JWT (JSON Web Tokens): Utilitza JWT per a una autenticació i autorització segura.
  • Serveis d'Autenticació d'Angular: Implementa serveis d'autenticació per gestionar les sessions dels usuaris de manera segura.
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class AuthService {
  private authUrl = 'https://example.com/api/auth';

  constructor(private http: HttpClient) {}

  login(credentials: { username: string, password: string }): Observable<any> {
    return this.http.post(this.authUrl, credentials);
  }

  logout(): void {
    // Implementa la lògica de tancament de sessió
  }
}

Capçaleres HTTP Segures

  • HTTP Strict Transport Security (HSTS): Enforça HTTPS per protegir les dades en trànsit.
  • X-Content-Type-Options: Prevé el sniffing de tipus MIME.
  • X-Frame-Options: Protegeix contra el clickjacking.
<meta http-equiv="Strict-Transport-Security" content="max-age=31536000; includeSubDomains">
<meta http-equiv="X-Content-Type-Options" content="nosniff">
<meta http-equiv="X-Frame-Options" content="DENY">

Gestió de Dependències

  • Mantingues les Dependències Actualitzades: Actualitza regularment Angular i les biblioteques de tercers per corregir vulnerabilitats de seguretat.
  • Audita les Dependències: Utilitza eines com npm audit per comprovar vulnerabilitats conegudes.
npm audit

Emmagatzematge Segur de Dades

  • Evita Emmagatzemar Dades Sensibles en Local Storage: Utilitza mecanismes d'emmagatzematge segur com les cookies amb els flags HttpOnly i Secure.
  • Encripta les Dades Sensibles: Encripta qualsevol dada sensible abans d'emmagatzemar-la.
// Exemple de configuració d'una cookie segura
document.cookie = "token=encryptedToken; Secure; HttpOnly";

Conclusió

Assegurar una aplicació Angular implica una combinació de millors pràctiques, funcions integrades d'Angular i mesures de seguretat addicionals. Seguint aquestes directrius, pots reduir significativament el risc de vulnerabilitats web comunes i garantir una experiència d'usuari més segura. Mantingues-te sempre actualitzat amb les últimes pràctiques de seguretat i revisa regularment la postura de seguretat de la teva aplicació.
