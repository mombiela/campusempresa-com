Introducció

En aquesta secció, cobrirem les millors pràctiques per organitzar i estilitzar el teu codi Angular. Una organització adequada del codi i l'adherència a una guia d'estil són crucials per mantenir la llegibilitat, escalabilitat i facilitat de manteniment en els teus projectes Angular.

Conceptes Clau

  • Estructura Modular: Organitza el teu codi en mòduls per separar preocupacions i millorar la mantenibilitat.
  • Arquitectura Basada en Components: Utilitza l'arquitectura basada en components d'Angular per crear components d'UI reutilitzables i testables.
  • Convencions de Nomenclatura Consistents: Segueix convencions de nomenclatura consistents per a fitxers, classes i variables.
  • Formatació del Codi: Utilitza un estil de formatació del codi consistent per millorar la llegibilitat.
  • Documentació i Comentaris: Escriu documentació i comentaris clars per explicar el propòsit i la funcionalitat del teu codi.

Estructura Modular

Les aplicacions Angular es construeixen utilitzant mòduls. Cada mòdul encapsula una característica o funcionalitat específica de l'aplicació. Aquest enfocament modular ajuda a organitzar el codi i fa que sigui més fàcil de gestionar i escalar.

Exemple

// app.module.ts
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { UserModule } from './user/user.module';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule,
    UserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

En aquest exemple, l'AppModule importa l'UserModule, que encapsula tota la funcionalitat relacionada amb l'usuari.

Arquitectura Basada en Components

Angular promou una arquitectura basada en components on la UI es divideix en petits components reutilitzables. Cada component té la seva pròpia plantilla, estils i lògica.

Exemple

// user.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-user',
  templateUrl: './user.component.html',
  styleUrls: ['./user.component.css']
})
export class UserComponent {
  userName: string = 'John Doe';
}
<!-- user.component.html -->
<div>
  <h1>{{ userName }}</h1>
</div>

En aquest exemple, el UserComponent mostra el nom de l'usuari.

Convencions de Nomenclatura Consistents

Les convencions de nomenclatura consistents fan que el teu codi sigui més previsible i fàcil d'entendre. Aquí tens algunes directrius:

  • Fitxers: Utilitza kebab-case per als noms dels fitxers (per exemple, user.component.ts).
  • Classes: Utilitza PascalCase per als noms de les classes (per exemple, UserComponent).
  • Variables: Utilitza camelCase per als noms de les variables (per exemple, userName).

Formatació del Codi

Una formatació del codi consistent millora la llegibilitat i ajuda a mantenir una base de codi neta. Utilitza eines com Prettier o TSLint per fer complir un estil consistent.

Exemple

// Abans de la formatació
let userName:string='John Doe';

// Després de la formatació
let userName: string = 'John Doe';

Documentació i Comentaris

Una documentació i comentaris clars ajuden altres desenvolupadors a entendre el teu codi. Utilitza comentaris JSDoc per documentar funcions, classes i mètodes.

Exemple

/**
 * Representa un usuari.
 */
class User {
  /**
   * El nom de l'usuari.
   */
  name: string;

  /**
   * Crea un nou usuari.
   * @param name - El nom de l'usuari.
   */
  constructor(name: string) {
    this.name = name;
  }
}

Conclusió

En aquesta secció, hem cobert les millors pràctiques per organitzar i estilitzar el teu codi Angular. Seguint una estructura modular, utilitzant una arquitectura basada en components, adherint-se a convencions de nomenclatura consistents, mantenint una formatació del codi consistent i escrivint documentació i comentaris clars, pots crear aplicacions Angular netes, mantenibles i escalables.
