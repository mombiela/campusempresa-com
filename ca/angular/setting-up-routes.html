El routing és una característica fonamental en Angular que et permet navegar entre diferents vistes o components de la teva aplicació. Aquesta secció et guiarà a través de la configuració de rutes des d'un nivell principiant fins a un nivell avançat.

Introducció al Routing en Angular

El routing en Angular permet la navegació d'una vista a una altra mentre els usuaris realitzen tasques a la teva aplicació. Ajuda a crear Aplicacions d'una Sola Pàgina (SPAs) on la pàgina no es recarrega però la vista canvia.

Conceptes Clau

  • Router Module: Mòdul d'Angular per al routing.
  • Routes: Objectes de configuració que defineixen el camí i el component a mostrar.
  • Router Outlet: Una directiva que actua com a marcador de posició per al component rutejat.
  • Router Link: Una directiva per enllaçar rutes.

Configuració de Rutes Bàsiques

Per començar amb el routing, necessites importar el RouterModule i definir les teves rutes.

Guia Pas a Pas

  1. Instal·la Angular Router (si no està ja instal·lat):

    npm install @angular/router

  2. Importa RouterModule en el teu mòdul principal (app.module.ts):

    import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  { path: 'home', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppModule { }

  3. Afegeix Router Outlet en la teva plantilla principal (app.component.html):

    <nav>
  <a routerLink="/home">Home</a>
  <a routerLink="/about">About</a>
</nav>
<router-outlet></router-outlet>

Explicació

  • RouterModule.forRoot(routes): Configura el router al nivell arrel de l'aplicació.
  • routerLink: Directiva per enllaçar l'etiqueta d'ancoratge a una ruta.
  • router-outlet: Marcador de posició per al component rutejat.

Tècniques Avançades de Routing

A mesura que la teva aplicació creix, pots necessitar característiques de routing més avançades com rutes niades, guàrdies de ruta i càrrega per demanda.

Rutes Niades

Les rutes niades et permeten definir rutes dins d'altres rutes.

const routes: Routes = [
  {
    path: 'dashboard',
    component: DashboardComponent,
    children: [
      { path: 'stats', component: StatsComponent },
      { path: 'reports', component: ReportsComponent }
    ]
  }
];

Guàrdies de Ruta

Les guàrdies de ruta s'utilitzen per controlar l'accés a les rutes basant-se en certes condicions.

  1. Crea una Guàrdia:

    import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot } from '@angular/router';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): boolean {
    // Afegeix la teva lògica d'autenticació aquí
    return true; // o false basat en la condició
  }
}

  2. Aplica la Guàrdia a una Ruta:

    const routes: Routes = [
  { path: 'admin', component: AdminComponent, canActivate: [AuthGuard] }
];

Càrrega per Demanda

La càrrega per demanda ajuda a carregar mòduls de característiques només quan són necessaris, millorant el rendiment de l'aplicació.

  1. Crea un Mòdul de Característiques:

    @NgModule({
  declarations: [FeatureComponent],
  imports: [CommonModule, RouterModule.forChild([{ path: '', component: FeatureComponent }])]
})
export class FeatureModule { }

  2. Configura la Càrrega per Demanda en les Rutes:

    const routes: Routes = [
  { path: 'feature', loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) }
];

Resum

Configurar rutes en Angular implica configurar el RouterModule amb definicions de rutes i utilitzar directives com routerLink i router-outlet per navegar i mostrar components. A mesura que avances, pots implementar rutes niades, guàrdies de ruta i càrrega per demanda per optimitzar i assegurar la teva aplicació. Entendre aquests conceptes t'ajudarà a construir aplicacions Angular robustes i escalables.
