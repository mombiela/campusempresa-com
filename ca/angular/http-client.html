Introducció

En aquesta secció, explorarem com utilitzar el mòdul HTTP Client a Angular per fer sol·licituds HTTP a un servidor. Aquesta és una habilitat crucial per a qualsevol desenvolupador d'Angular, ja que permet que la teva aplicació interactuï amb serveis de backend i APIs.

Configuració del Client HTTP

Per utilitzar el mòdul HTTP Client, primer cal importar-lo a la teva aplicació Angular.

  1. Instal·lar Angular HTTP Client: El mòdul HTTP Client forma part del paquet @angular/common/http, que s'inclou per defecte en els nous projectes d'Angular.
  2. Importar el Mòdul HTTP Client: Afegeix el HttpClientModule al mòdul arrel de la teva aplicació.
// app.module.ts
import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { HttpClientModule } from '@angular/common/http';

import { AppComponent } from './app.component';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule,
    HttpClientModule // Importar HttpClientModule aquí
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Fer Sol·licituds HTTP

El servei HttpClient s'utilitza per fer sol·licituds HTTP. Proporciona mètodes per a diverses operacions HTTP com GET, POST, PUT, DELETE, etc.

Sol·licitud GET

Una sol·licitud GET s'utilitza per obtenir dades d'un servidor.

// data.service.ts
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {

  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) { }

  getData(): Observable<any> {
    return this.http.get<any>(this.apiUrl);
  }
}

Sol·licitud POST

Una sol·licitud POST s'utilitza per enviar dades a un servidor.

// data.service.ts
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {

  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) { }

  postData(data: any): Observable<any> {
    return this.http.post<any>(this.apiUrl, data);
  }
}

Sol·licitud PUT

Una sol·licitud PUT s'utilitza per actualitzar dades existents en un servidor.

// data.service.ts
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {

  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) { }

  updateData(id: string, data: any): Observable<any> {
    return this.http.put<any>(`${this.apiUrl}/${id}`, data);
  }
}

Sol·licitud DELETE

Una sol·licitud DELETE s'utilitza per eliminar dades d'un servidor.

// data.service.ts
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'

export class DataService {

  deleteData(id: string): Observable<any> {

  }
}

Gestió de Respostes HTTP

Gestionar respostes i errors és crucial per a una aplicació robusta.

Gestió d'Èxit

Pots gestionar respostes exitoses utilitzant el mètode subscribe.

// app.component.ts
import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-root',
  template: `
    <div *ngIf="data">
      <pre>{{ data | json }}</pre>
    </div>
  `
})
export class AppComponent implements OnInit {
  data: any;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.dataService.getData().subscribe(
      (response) => {
        this.data = response;
      }
    );
  }
}

Gestió d'Errors

Pots gestionar errors utilitzant l'operador catchError de RxJS.

// data.service.ts
import { Injectable } from '@angular/core';
import { HttpClient, HttpErrorResponse } from '@angular/common/http';
import { Observable, throwError } from 'rxjs';
import { catchError } from 'rxjs/operators';

@Injectable({
  providedIn: 'root'
})
export class DataService {

  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) { }

  getData(): Observable<any> {
    return this.http.get<any>(this.apiUrl).pipe(
      catchError(this.handleError)
    );
  }

  private handleError(error: HttpErrorResponse) {
    // Gestionar l'error aquí
    return throwError('Alguna cosa ha anat malament; si us plau, torna-ho a intentar més tard.');
  }
}

Interceptors

Els interceptors et permeten modificar sol·licituds i respostes HTTP globalment.

Crear un Interceptor

Crea un interceptor implementant la interfície HttpInterceptor.

// auth.interceptor.ts
import { Injectable } from '@angular/core';
import { HttpInterceptor, HttpRequest, HttpHandler, HttpEvent } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable()
export class AuthInterceptor implements HttpInterceptor {

  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    const clonedRequest = req.clone({
      headers: req.headers.set('Authorization', 'Bearer your-token')
    });

    return next.handle(clonedRequest);
  }
}

Proveir l'Interceptor

Registra l'interceptor al mòdul de la teva aplicació.

// app.module.ts
import { HTTP_INTERCEPTORS } from '@angular/common/http';
import { AuthInterceptor } from './auth.interceptor';

@NgModule({
  // ... altres imports i declaracions
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthInterceptor, multi: true }
  ]
})
export class AppModule { }

Conclusió

El mòdul HTTP Client a Angular és una eina poderosa per fer sol·licituds HTTP i interactuar amb serveis de backend. Entenent com fer sol·licituds GET, POST, PUT i DELETE, gestionar respostes i errors, i utilitzar interceptors, pots construir aplicacions Angular robustes i eficients.
