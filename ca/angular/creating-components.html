Introducció als Components

Els components són els blocs de construcció de qualsevol aplicació Angular. Encapsulen la vista, la lògica i els estils, fent que sigui més fàcil gestionar i reutilitzar el codi.

Conceptes Clau

  • Component: Una classe que interactua amb la vista (HTML) i defineix el comportament d'una part de la interfície d'usuari.
  • Plantilla: La part HTML del component que defineix la vista.
  • Estils: CSS o SCSS que defineixen l'aspecte del component.
  • Metadades: Informació sobre el component, com ara el seu selector, plantilla i estils.

Creació d'un Component Bàsic

Per crear un nou component en Angular, pots utilitzar l'Angular CLI.

Utilitzant Angular CLI

Executa la següent comanda al teu terminal:

ng generate component my-component

Aquesta comanda crearà una nova carpeta anomenada my-component amb els següents fitxers:

  • my-component.component.ts: El fitxer TypeScript que conté la classe del component.
  • my-component.component.html: La plantilla HTML per al component.
  • my-component.component.css: Els estils CSS per al component.
  • my-component.component.spec.ts: El fitxer de proves per al component.

Estructura del Component

Desglossem el fitxer generat my-component.component.ts:

import { Component } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  // La lògica del component va aquí
}
  • Decorador @Component: Aquest decorador proporciona metadades sobre el component.
    • selector: L'etiqueta HTML personalitzada que representa el component.
    • templateUrl: El camí al fitxer de la plantilla HTML.
    • styleUrls: El camí al fitxer(s) CSS.

Utilitzant el Component

Per utilitzar el component acabat de crear, afegeix el seu selector a la plantilla d'un altre component, com ara app.component.html:

<app-my-component></app-my-component>

Interacció de Components

Els components sovint necessiten interactuar entre ells. Això es pot fer utilitzant:

  • Input: Passar dades d'un component pare a un component fill.
  • Output: Emetre esdeveniments d'un component fill a un component pare.

Exemple d'Input

En el component fill (my-component):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  @Input() data: string;
}

En el component pare (app.component.html):

<app-my-component [data]="parentData"></app-my-component>

En la classe del component pare (app.component.ts):

export class AppComponent {
  parentData = 'Hello from Parent';
}

Exemple d'Output

En el component fill (my-component):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  @Output() notify: EventEmitter<string> = new EventEmitter<string>();

  sendNotification() {
    this.notify.emit('Notification from Child');
  }
}

En la plantilla del component fill (my-component.component.html):

<button (click)="sendNotification()">Notify Parent</button>

En el component pare (app.component.html):

<app-my-component (notify)="onNotify($event)"></app-my-component>

En la classe del component pare (app.component.ts):

export class AppComponent {
  onNotify(message: string) {
    console.log(message);
  }
}

Funcionalitats Avançades del Component

Hooks de Cicle de Vida

Angular proporciona hooks de cicle de vida que et permeten aprofitar esdeveniments clau en el cicle de vida d'un component:

  • ngOnInit(): Cridat una vegada que el component està inicialitzat.
  • ngOnChanges(): Cridat quan una propietat d'entrada canvia.
  • ngOnDestroy(): Cridat just abans que el component sigui destruït.

Exemple:

import { Component, OnInit, OnDestroy } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent implements OnInit, OnDestroy {
  ngOnInit() {
    console.log('Component Initialized');
  }

  ngOnDestroy() {
    console.log('Component Destroyed');
  }
}

ViewEncapsulation

Angular proporciona tres estratègies d'encapsulació:

  • Emulated: Per defecte, emula el comportament natiu del shadow DOM.
  • None: Sense encapsulació, els estils són globals.
  • ShadowDom: Utilitza el shadow DOM natiu.

Exemple:

import { Component, ViewEncapsulation } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css'],
  encapsulation: ViewEncapsulation.Emulated // Per defecte
})
export class MyComponent {}

Conclusió

Els components són fonamentals per a les aplicacions Angular. Encapsulen la vista, la lògica i els estils, fent que sigui més fàcil gestionar i reutilitzar el codi. Entenent com crear i interactuar amb components, pots construir aplicacions Angular complexes i escalables. Recorda aprofitar l'Angular CLI per a una creació eficient de components i utilitzar els hooks de cicle de vida per gestionar el comportament dels components de manera efectiva.
