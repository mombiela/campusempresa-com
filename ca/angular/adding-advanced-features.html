Carregament per Demanda

El carregament per demanda és un patró de disseny que retarda la càrrega de recursos fins que realment es necessiten. Això pot millorar significativament el rendiment de la vostra aplicació Angular reduint el temps de càrrega inicial.

Conceptes Clau

  • Mòduls: Les aplicacions Angular es divideixen en mòduls. El carregament per demanda us permet carregar aquests mòduls només quan es necessiten.
  • Rutes: El carregament per demanda sovint s'implementa utilitzant el sistema de routing d'Angular.

Exemple

Primer, creeu un nou mòdul que voleu carregar per demanda:

ng generate module lazy --route lazy --module app.module

Aquesta comanda crearà un nou mòdul anomenat LazyModule i el configurarà per ser carregat per demanda.

A continuació, configureu les rutes a app-routing.module.ts:

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: 'lazy',
    loadChildren: () => import('./lazy/lazy.module').then(m => m.LazyModule)
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Ara, el LazyModule només es carregarà quan l'usuari navegui a la ruta /lazy.

Gestió d'Estat amb NgRx

NgRx és una biblioteca de gestió d'estat per a aplicacions Angular, inspirada en Redux. Ajuda a gestionar l'estat de la vostra aplicació d'una manera previsible.

Conceptes Clau

  • Store: La font única de veritat per a l'estat de la vostra aplicació.
  • Actions: Esdeveniments que descriuen alguna cosa que ha passat a l'aplicació.
  • Reducers: Funcions que gestionen les accions i actualitzen l'estat.
  • Selectors: Funcions que seleccionen una part de l'estat.

Exemple

Primer, instal·leu NgRx:

ng add @ngrx/store

Creeu una acció a src/app/state/counter.actions.ts:

import { createAction } from '@ngrx/store';

export const increment = createAction('[Counter Component] Increment');
export const decrement = createAction('[Counter Component] Decrement');
export const reset = createAction('[Counter Component] Reset');

Creeu un reducer a src/app/state/counter.reducer.ts:

import { createReducer, on } from '@ngrx/store';
import { increment, decrement, reset } from './counter.actions';

export const initialState = 0;

const _counterReducer = createReducer(initialState,
  on(increment, state => state + 1),
  on(decrement, state => state - 1),
  on(reset, state => 0),
);

export function counterReducer(state, action) {
  return _counterReducer(state, action);
}

Registreu el reducer a app.module.ts:

import { StoreModule } from '@ngrx/store';
import { counterReducer } from './state/counter.reducer';

@NgModule({
  imports: [
    // altres imports
    StoreModule.forRoot({ count: counterReducer })
  ],
  // altres configuracions
})
export class AppModule { }

Utilitzeu la store en un component:

import { Component } from '@angular/core';
import { Store } from '@ngrx/store';
import { increment, decrement, reset } from './state/counter.actions';

@Component({
  selector: 'app-counter',
  template: `
    <button (click)="increment()">Increment</button>
    <div>Current Count: {{ count$ | async }}</div>
    <button (click)="decrement()">Decrement</button>
    <button (click)="reset()">Reset</button>
  `
})
export class CounterComponent {
  count$ = this.store.select('count');

  constructor(private store: Store<{ count: number }>) {}

  increment() {
    this.store.dispatch(increment());
  }

  decrement() {
    this.store.dispatch(decrement());
  }

  reset() {
    this.store.dispatch(reset());
  }
}

Tècniques Avançades de Routing

Les tècniques avançades de routing poden ajudar-vos a crear aplicacions Angular més dinàmiques i flexibles.

Conceptes Clau

  • Guards: S'utilitzen per controlar l'accés a les rutes.
  • Resolvers: S'utilitzen per pre-carregar dades abans de navegar a una ruta.
  • Child Routes: Rutes niades que permeten estructures de routing més complexes.

Exemple

Creeu un guard a src/app/auth.guard.ts:

import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, UrlTree } from '@angular/router';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): Observable<boolean | UrlTree> | Promise<boolean | UrlTree> | boolean | UrlTree {
    // Implementeu la vostra lògica d'autenticació aquí
    return true; // o false segons l'autenticació
  }
}

Configureu el guard a app-routing.module.ts:

import { AuthGuard } from './auth.guard';

const routes: Routes = [
  {
    path: 'protected',
    component: ProtectedComponent,
    canActivate: [AuthGuard]
  }
];
