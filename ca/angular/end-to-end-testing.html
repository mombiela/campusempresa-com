Les proves de punta a punta (E2E) són una part crucial del cicle de vida del desenvolupament de programari, assegurant que tot el flux de l'aplicació funcioni com s'espera. A Angular, les proves E2E es fan típicament amb Protractor, un marc de proves dissenyat específicament per a aplicacions Angular.

Introducció a les proves de punta a punta

Les proves de punta a punta simulen escenaris reals d'usuari per validar la funcionalitat de l'aplicació de principi a fi. Prova tota la pila de l'aplicació, incloent el frontend, backend i la base de dades.

Conceptes clau

  • Escenaris d'usuari: Simulant interaccions reals d'usuari.
  • Automatització: Automatitzant el procés de proves per assegurar la consistència i l'eficiència.
  • Protractor: Un marc de proves per a aplicacions Angular.
  • WebDriver: Una eina per automatitzar les proves d'aplicacions web.

Configuració de Protractor

Per començar amb les proves E2E a Angular, necessites configurar Protractor.

Instal·lació

Primer, instal·la Protractor globalment utilitzant npm:

npm install -g protractor

A continuació, actualitza el gestor de WebDriver:

webdriver-manager update

Configuració

Crea un fitxer protractor.conf.js al directori arrel del teu projecte:

exports.config = {
  framework: 'jasmine',
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['e2e/**/*.e2e-spec.js'],
  capabilities: {
    browserName: 'chrome'
  },
  onPrepare: function() {
    browser.ignoreSynchronization = true;
  }
};

Escriure la teva primera prova E2E

Escrivim una prova E2E senzilla per comprovar si la pàgina d'inici es carrega correctament.

Exemple de prova

Crea un nou fitxer e2e/homepage.e2e-spec.js:

describe('Homepage', function() {
  it('should have a title', function() {
    browser.get('http://localhost:4200');
    expect(browser.getTitle()).toEqual('My Angular App');
  });
});

Explicació

  • describe: Defineix un conjunt de proves.
  • it: Defineix un cas de prova.
  • browser.get: Navega a l'URL especificat.
  • expect: Asserta que l'aplicació es comporta com s'espera.

Tècniques avançades de proves E2E

A mesura que et familiaritzis amb les proves E2E, pots començar a utilitzar tècniques més avançades.

Objectes de pàgina

Els objectes de pàgina ajuden a organitzar les teves proves separant el codi de prova del codi específic de la pàgina.

Exemple

Crea un nou fitxer e2e/pages/homepage.po.js:

class HomePage {
  constructor() {
    this.title = element(by.css('app-root h1'));
  }

  getTitle() {
    return this.title.getText();
  }
}

module.exports = new HomePage();

Actualitza la teva prova per utilitzar l'objecte de pàgina:

const HomePage = require('./pages/homepage.po');

describe('Homepage', function() {
  it('should have a title', function() {
    browser.get('http://localhost:4200');
    expect(HomePage.getTitle()).toEqual('Welcome to My Angular App!');
  });
});

Gestió d'operacions asíncrones

Les aplicacions Angular sovint impliquen operacions asíncrones. Protractor proporciona maneres de gestionar-les.

Exemple

it('should wait for an element to be visible', function() {
  const EC = protractor.ExpectedConditions;
  const button = element(by.css('.my-button'));
  browser.wait(EC.visibilityOf(button), 5000);
  button.click();
});

Executar proves E2E

Per executar les teves proves E2E, inicia el gestor de WebDriver i executa Protractor:

webdriver-manager start
protractor protractor.conf.js

Conclusió

Les proves de punta a punta són una pràctica essencial per assegurar la fiabilitat i funcionalitat de les teves aplicacions Angular. Utilitzant Protractor, pots automatitzar escenaris d'usuari, gestionar operacions asíncrones i organitzar les teves proves utilitzant objectes de pàgina. A mesura que avancis, pots integrar les proves E2E al teu pipeline de CI/CD per mantenir estàndards d'alta qualitat durant tot el cicle de vida del desenvolupament.
