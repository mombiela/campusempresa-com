El Projecte Sobre nosaltres Contribuir Donacions Llicència

La internacionalització (i18n) és el procés de dissenyar i preparar la teva aplicació perquè sigui utilitzable en diferents idiomes i regions sense requerir canvis d'enginyeria en el codi font. Angular proporciona un suport robust per a i18n, facilitant la creació d'aplicacions multilingües.

Introducció a la Internacionalització (i18n)

  • Definició: La internacionalització és el procés de dissenyar una aplicació perquè es pugui adaptar a diversos idiomes i regions sense requerir canvis en el codi.
  • Importància: Permet que la teva aplicació arribi a una audiència més àmplia donant suport a múltiples idiomes i formats regionals.

Configuració d'Angular per a i18n

Per començar amb i18n en Angular, has de configurar el teu projecte per donar suport a les traduccions.

Pas 1: Instal·lar Angular CLI

Assegura't de tenir instal·lat Angular CLI:

npm install -g @angular/cli

Pas 2: Crear un Nou Projecte Angular

Crea un nou projecte Angular si encara no en tens un:

ng new my-i18n-app
cd my-i18n-app

Pas 3: Afegir Marcadors i18n en Plantilles

Utilitza els atributs i18n d'Angular per marcar el text per a la traducció en les teves plantilles:

<h1 i18n="@@homeTitle">Welcome to My App</h1>
<p i18n="@@homeDescription">This is a sample application for demonstrating i18n in Angular.</p>
  • Atribut i18n: Marca el text per a la traducció.
  • @@id: Identificador personalitzat opcional per a la unitat de traducció.

Extracció de Cadena de Traducció

Després de marcar el text per a la traducció, has d'extreure aquestes cadenes en un fitxer de traducció.

Pas 1: Extreure Cadenes

Utilitza Angular CLI per extreure les cadenes marcades:

ng extract-i18n

Aquesta comanda genera un fitxer messages.xlf al directori arrel del teu projecte.

Pas 2: Traduir Cadenes

Traduir les cadenes extretes en el fitxer messages.xlf:

<trans-unit id="homeTitle" datatype="html">
  <source>Welcome to My App</source>
  <target>Benvingut a la Meva Aplicació</target>
</trans-unit>
<trans-unit id="homeDescription" datatype="html">
  <source>This is a sample application for demonstrating i18n in Angular.</source>
  <target>Aquesta és una aplicació de mostra per demostrar i18n en Angular.</target>
</trans-unit>

Configuració d'Angular per a Múltiples Localitzacions

Per donar suport a múltiples localitzacions, has de configurar Angular per utilitzar diferents fitxers de traducció segons la localització de l'usuari.

Pas 1: Actualitzar `angular.json`

Modifica el fitxer angular.json per incloure diferents localitzacions:

"projects": {
  "my-i18n-app": {
    ...
    "i18n": {
      "sourceLocale": "en",
      "locales": {
        "es": "src/locale/messages.es.xlf"
      }
    }
  }
}

Pas 2: Compilar per a Diferents Localitzacions

Compila la teva aplicació per a diferents localitzacions:

ng build --localize

Aquesta comanda genera compilacions separades per a cada localització especificada en el fitxer angular.json.

Canvi Dinàmic de Localització

Per a aplicacions més avançades, potser voldràs canviar les localitzacions dinàmicament en temps d'execució.

Pas 1: Instal·lar Dependències

Instal·la els paquets necessaris d'Angular:

npm install @ngx-translate/core @ngx-translate/http-loader

Pas 2: Configurar `app.module.ts`

Configura el carregador de traduccions en el teu AppModule:

import { HttpClient } from '@angular/common/http';
import { TranslateLoader, TranslateModule } from '@ngx-translate/core';
import { TranslateHttpLoader } from '@ngx-translate/http-loader';

export function HttpLoaderFactory(http: HttpClient) {
  return new TranslateHttpLoader(http);
}

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,
    HttpClientModule,
    TranslateModule.forRoot({
      loader: {
        provide: TranslateLoader,
        useFactory: HttpLoaderFactory,
        deps: [HttpClient]
      }
    })
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Pas 3: Carregar Traduccions en Components

Carrega i utilitza les traduccions en els teus components:

import { Component } from '@angular/core';
import { TranslateService } from '@ngx-translate/core';

@Component({
  selector: 'app-root',
  template: `
    <div>
      <h1>{{ 'homeTitle' | translate }}</h1>
      <p>{{ 'homeDescription' | translate }}</p>
      <button (click)="switchLanguage('en')">English</button>
      <button (click)="switchLanguage('es')">Español</button>
    </div>
  `
})
export class AppComponent {
  constructor(private translate: TranslateService) {
    this.translate.setDefaultLang('en');
  }

  switchLanguage(language: string) {
    this.translate.use(language);
  }
}

Conclusió

La internacionalització (i18n) en Angular és una característica potent que et permet crear aplicacions que es poden adaptar fàcilment a diferents idiomes i regions. Seguint els passos descrits en aquesta guia, pots configurar la teva aplicació Angular per a i18n, extreure i traduir cadenes, configurar múltiples localitzacions i fins i tot canviar localitzacions dinàmicament en temps d'execució. Això assegura que la teva aplicació pugui arribar a una audiència global amb un esforç mínim.
