Introducció a les Rutes Filles

A Angular, les rutes filles s'utilitzen per crear rutes niades dins d'una ruta pare. Això permet escenaris de routing més complexos i ajuda a organitzar l'aplicació en parts més petites i manejables. Les rutes filles són especialment útils quan es creen aplicacions amb múltiples vistes on es mostren diferents vistes dins d'un component pare.

Conceptes Clau

  • Ruta Pare: La ruta principal que conté una o més rutes filles.
  • Ruta Filla: Una ruta que està niada dins d'una ruta pare.
  • Mòdul de Router: Mòdul d'Angular que proporciona els serveis i directives necessaris per al routing.

Configuració de les Rutes Filles

Per configurar les rutes filles, cal definir-les dins de la propietat children d'una ruta pare en el teu mòdul de routing.

Exemple

Creem una aplicació Angular simple amb una ruta pare i dues rutes filles.

  1. Crear Components: Primer, crea els components necessaris per a les rutes pare i filles.

    ng generate component parent
ng generate component child1
ng generate component child2

  2. Definir Rutes: A continuació, defineix les rutes en el teu fitxer app-routing.module.ts.

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { ParentComponent } from './parent/parent.component';
import { Child1Component } from './child1/child1.component';
import { Child2Component } from './child2/child2.component';

const routes: Routes = [
  {
    path: 'parent',
    component: ParentComponent,
    children: [
      { path: 'child1', component: Child1Component },
      { path: 'child2', component: Child2Component }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

  3. Actualitzar la Plantilla del Component Pare: En la plantilla del component pare, afegeix un <router-outlet> per mostrar els components fills.

    <!-- parent.component.html -->
<h2>Component Pare</h2>
<nav>
  <a routerLink="child1">Fill 1</a>
  <a routerLink="child2">Fill 2</a>
</nav>
<router-outlet></router-outlet>

  4. Executar l'Aplicació: Inicia la teva aplicació Angular i navega a /parent. Hauries de veure el component pare amb enllaços als components fills. En fer clic als enllaços es mostraran els components fills respectius dins del component pare.

Càrrega Per Mandat de les Rutes Filles

La càrrega per mandat és una tècnica per carregar les rutes filles només quan són necessàries, cosa que pot millorar el rendiment de la teva aplicació.

Exemple

  1. Crear un Mòdul per a les Rutes Filles:

    ng generate module child --route child --module app.module

  2. Definir Rutes Filles en el Mòdul:

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { Child1Component } from './child1/child1.component';
import { Child2Component } from './child2/child2.component';

const routes: Routes = [
  {
    path: '',
    component: Child1Component,
    children: [
      { path: 'child2', component: Child2Component }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forChild(routes)],
  exports: [RouterModule]
})
export class ChildModule { }

  3. Actualitzar el Mòdul de Routing Principal:

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: 'parent',
    loadChildren: () => import('./child/child.module').then(m => m.ChildModule)
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Conclusió

Les rutes filles a Angular permeten un enfocament més organitzat i modular per al routing dins d'una aplicació. En niar rutes, pots crear estructures de routing complexes que són més fàcils de gestionar i mantenir. A més, aprofitar la càrrega per mandat per a les rutes filles pot millorar significativament el rendiment de la teva aplicació carregant només les parts necessàries quan es necessiten. Entendre i implementar rutes filles és una habilitat crucial per construir aplicacions Angular escalables i eficients.
