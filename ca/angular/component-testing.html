Les proves de components són una part crucial del desenvolupament d'aplicacions Angular robustes. Aquesta secció et guiarà des dels conceptes bàsics fins als avançats de les proves de components Angular.

Introducció a les proves de components

Les proves de components impliquen provar components individuals d'una aplicació Angular per assegurar-se que funcionen com s'espera. Això inclou provar la plantilla del component, la classe i les interaccions amb altres components i serveis.

Conceptes clau

  • Proves unitàries: Provar unitats o components individuals en aïllament.
  • TestBed: L'API principal d'Angular per configurar i inicialitzar l'entorn per a les proves unitàries.
  • Fixtures: S'utilitzen per crear una instància d'un component i accedir a les seves propietats i mètodes.
  • Spies: Funcions que registren arguments, valors de retorn i altra informació per a finalitats de prova.

Configuració de l'entorn de proves

Abans d'entrar en les proves de components, assegura't que el teu projecte Angular està configurat per a les proves.

Passos per configurar

  1. Instal·la Angular CLI: Assegura't que tens Angular CLI instal·lat. 
    npm install -g @angular/cli
  2. Crea un nou projecte Angular: Si no tens un projecte, crea'n un. 
    ng new my-angular-app
  3. Navega al directori del projecte: 
    cd my-angular-app
  4. Executa les proves: Utilitza Angular CLI per executar les proves. 
    ng test

Escriure la teva primera prova de component

Escrivim una prova senzilla per a un component. Utilitzarem Jasmine i Karma, que estan configurats per defecte en els projectes Angular.

Exemple de component

Considera un simple CounterComponent que incrementa un comptador.

counter.component.ts

import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  template: `
    <div>
      <p>Counter: {{ counter }}</p>
      <button (click)="increment()">Increment</button>
    </div>
  `
})
export class CounterComponent {
  counter = 0;

  increment() {
    this.counter++;
  }
}

Escriure la prova

counter.component.spec.ts

import { ComponentFixture, TestBed } from '@angular/core/testing';
import { CounterComponent } from './counter.component';

describe('CounterComponent', () => {
  let component: CounterComponent;
  let fixture: ComponentFixture<CounterComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ CounterComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(CounterComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should have initial counter value as 0', () => {
    expect(component.counter).toBe(0);
  });

  it('should increment counter value by 1 when increment method is called', () => {
    component.increment();
    expect(component.counter).toBe(1);
  });

  it('should increment counter value when button is clicked', () => {
    const button = fixture.nativeElement.querySelector('button');
    button.click();
    fixture.detectChanges();
    expect(component.counter).toBe(1);
  });
});

Explicació

  • TestBed: Configura el mòdul de proves.
  • ComponentFixture: Proporciona accés a la instància del component i la seva plantilla.
  • beforeEach: Configura l'entorn de proves abans de cada prova.
  • it: Defineix casos de prova individuals.

Proves avançades de components

Proves amb dependències

Els components sovint depenen de serveis. Provem un component que utilitza un servei.

data.service.ts

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  getData() {
    return 'Hello World';
  }
}

data.component.ts

import { Component } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  template: `<p>{{ data }}</p>`
})
export class DataComponent {
  data: string;

  constructor(private dataService: DataService) {
    this.data = this.dataService.getData();
  }
}

Escriure la prova amb servei simulat

data.component.spec.ts

import { ComponentFixture, TestBed } from '@angular/core/testing';
import { DataComponent } from './data.component';
import { DataService } from './data.service';

class MockDataService {
  getData() {
    return 'Mock Data';
  }
}

describe('DataComponent', () => {
  let component: DataComponent;
  let fixture: ComponentFixture<DataComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ DataComponent ],
      providers: [
        { provide: DataService, useClass: MockDataService }
      ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(DataComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should display data from the service', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('p').textContent).toContain('Mock Data');
  });
});

Explicació

  • Servei simulat: S'utilitza una classe simulada per proporcionar una implementació falsa del servei.
  • providers: Configura el mòdul de proves per utilitzar el servei simulat en lloc del real.

Conclusió

Les proves de components en Angular asseguren que els components individuals funcionin correctament en aïllament. Configurant l'entorn de proves, escrivint proves bàsiques i gestionant les dependències amb serveis simulats, pots crear aplicacions Angular robustes i fiables. Recorda:

  • Utilitza TestBed per configurar l'entorn de proves.
  • Utilitza ComponentFixture per interactuar amb el component.
  • Simula les dependències per aïllar el component sota prova.

Amb aquestes pràctiques, pots provar i mantenir amb confiança els teus components Angular.
