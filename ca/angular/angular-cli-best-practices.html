Introducció

La Interfície de Línia de Comandes d'Angular (CLI) és una eina poderosa que ajuda els desenvolupadors a crear, construir i gestionar aplicacions Angular. Aquesta secció cobrirà les millors pràctiques per utilitzar l'Angular CLI de manera efectiva, des de comandes bàsiques fins a configuracions avançades.

Començant amb Angular CLI

  • Instal·lació: Assegura't de tenir Node.js instal·lat, després instal·la Angular CLI globalment utilitzant: 
    npm install -g @angular/cli
  • Creació d'un Nou Projecte: Utilitza la comanda ng new per crear un nou projecte Angular: 
    ng new my-angular-app
    • Segueix les indicacions per configurar el teu projecte amb enrutament, fulls d'estil i més.

Estructura i Organització del Projecte

  • Convencions de Nomenclatura Consistents: Utilitza noms consistents i descriptius per a fitxers i directoris.
    • Exemple: Utilitza kebab-case per als noms dels fitxers (my-component.component.ts).
  • Arquitectura Modular: Organitza la teva aplicació en mòduls per millorar la mantenibilitat i escalabilitat.
    • Exemple: Crea mòduls de funcionalitat per a diferents parts de la teva aplicació (user.module.ts, admin.module.ts).

Ús de les Comandes d'Angular CLI

  • Generació de Components, Serveis i Mòduls: Utilitza la comanda ng generate (o ng g) per crear nous components, serveis i mòduls. 
    ng generate component my-component
ng generate service my-service
ng generate module my-module
  • Servir l'Aplicació: Utilitza la comanda ng serve per executar la teva aplicació localment. 
    ng serve
    • Utilitza el flag --open per obrir automàticament l'aplicació en el teu navegador per defecte.
    ng serve --open

Gestió de Configuració i Entorns

  • Fitxers d'Entorn: Utilitza fitxers d'entorn per gestionar diferents configuracions per a desenvolupament, staging i producció.
    • Exemple: environment.ts per a desenvolupament i environment.prod.ts per a producció.
  • Configuració Personalitzada: Modifica el fitxer angular.json per personalitzar les configuracions de construcció i servei.
    • Exemple: Afegir una nova configuració de construcció per a staging.
    "configurations": {
  "staging": {
    "fileReplacements": [
      {
        "replace": "src/environments/environment.ts",
        "with": "src/environments/environment.staging.ts"
      }
    ]
  }
}

Optimització de Construccions

  • Construccions de Producció: Utilitza sempre el flag --prod quan construeixis per a producció per habilitar optimitzacions com la compilació Ahead-of-Time (AOT) i la minificació. 
    ng build --prod
  • Càrrega Per Vagues: Implementa la càrrega per vagues per als mòduls de funcionalitat per millorar el temps de càrrega inicial.
    • Exemple: Defineix rutes amb càrrega per vagues.
    const routes: Routes = [
  { path: 'feature', loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) }
];

Proves i Linting

  • Proves Unitàries: Utilitza l'Angular CLI per executar proves unitàries amb Karma i Jasmine. 
    ng test
  • Proves de Cap a Cap: Utilitza Protractor per a proves de cap a cap. 
    ng e2e
  • Linting: Utilitza la comanda ng lint per executar comprovacions de linting i assegurar la qualitat del codi. 
    ng lint

Desplegament

  • Desplegament a Firebase: Utilitza l'Angular CLI per desplegar la teva aplicació a Firebase Hosting. 
    ng add @angular/fire
ng deploy
  • Altres Opcions de Desplegament: Configura l'Angular CLI per al desplegament a altres plataformes com AWS, Azure o Netlify.

Conclusió

Utilitzar l'Angular CLI de manera efectiva pot millorar significativament el teu flux de treball de desenvolupament, millorar la qualitat del codi i agilitzar el procés de desplegament. Seguint aquestes millors pràctiques, pots assegurar-te que les teves aplicacions Angular estiguin ben estructurades, siguin mantenibles i estiguin optimitzades per al rendiment.
