Introducció a l'Arquitectura d'Angular

Angular és una plataforma i un framework per construir aplicacions client d'una sola pàgina utilitzant HTML i TypeScript. Angular està escrit en TypeScript i proporciona un conjunt d'eines i biblioteques per construir aplicacions robustes. Entendre l'arquitectura d'Angular és crucial per desenvolupar aplicacions eficients i mantenibles.

Conceptes Clau de l'Arquitectura d'Angular

  • Mòduls
  • Components
  • Plantilles
  • Serveis
  • Injecció de Dependències
  • Enrutament

Mòduls

Els mòduls són els blocs fonamentals d'una aplicació Angular. Ajuden a organitzar una aplicació en blocs cohesius de funcionalitat.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }
  • NgModule: Decorador que marca una classe com un mòdul d'Angular.
  • declarations: Array de components, directives i pipes que pertanyen a aquest mòdul.
  • imports: Array de mòduls per importar en aquest mòdul.
  • providers: Array de serveis que el mòdul contribueix a la col·lecció global de serveis.
  • bootstrap: Array de components per inicialitzar quan aquest mòdul s'inicialitza.

Components

Els components són els principals blocs de construcció d'una aplicació Angular. Cada component consisteix en una plantilla HTML i una classe TypeScript.

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'my-angular-app';
}
  • @Component: Decorador que marca una classe com un component d'Angular.
  • selector: Selector CSS que identifica aquest component en una plantilla.
  • templateUrl: URL a un fitxer extern que conté la plantilla del component.
  • styleUrls: Array d'URLs a fulls d'estil per a aquest component.

Plantilles

Les plantilles defineixen la vista per a un component. Utilitzen la sintaxi de plantilles d'Angular per vincular dades i gestionar esdeveniments d'usuari.

<h1>{{ title }}</h1>
<button (click)="onClick()">Click me</button>
  • Interpolació: {{ title }} vincula la propietat title del component a l'element <h1>.
  • Vinculació d'Esdeveniments: (click)="onClick()" vincula l'esdeveniment de clic del botó al mètode onClick en la classe del component.

Serveis

Els serveis s'utilitzen per encapsular la lògica de negoci i l'accés a dades. Normalment s'injecten en components i altres serveis utilitzant el sistema d'injecció de dependències d'Angular.

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class DataService {
  getData() {
    return ['Data1', 'Data2', 'Data3'];
  }
}
  • @Injectable: Decorador que marca una classe com disponible per ser injectada com a dependència.
  • providedIn: Especifica que el servei ha de ser proporcionat en l'injector arrel.

Injecció de Dependències

La Injecció de Dependències (DI) és un patró de disseny utilitzat per implementar IoC (Inversió de Control). Permet que una classe rebi dependències d'una font externa en lloc de crear-les ella mateixa.

import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  templateUrl: './data.component.html',
})
export class DataComponent implements OnInit {
  data: string[];

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.data = this.dataService.getData();
  }
}
  • constructor(private dataService: DataService): El DataService s'injecta en el DataComponent.

Enrutament

L'enrutament permet la navegació entre diferents vistes o components en una aplicació Angular.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent },
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }
  • RouterModule.forRoot(routes): Configura el router al nivell arrel de l'aplicació.
  • routes: Array de definicions de rutes.

Conclusió

Entendre l'arquitectura d'Angular és essencial per construir aplicacions escalables i mantenibles. Dominant els mòduls, components, plantilles, serveis, injecció de dependències i enrutament, pots crear aplicacions Angular robustes. Cadascun d'aquests conceptes juga un paper crític en l'estructura i funcionalitat general d'una aplicació Angular.
