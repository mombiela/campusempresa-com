Configurar i planificar un projecte Angular és crucial per assegurar un procés de desenvolupament fluid. Aquesta secció et guiarà a través dels passos inicials necessaris per configurar un projecte Angular i planificar la seva estructura de manera efectiva.

Introducció a Angular

  • Què és Angular?

    • Angular és una plataforma i un framework per construir aplicacions client de pàgina única utilitzant HTML i TypeScript.
    • Desenvolupat i mantingut per Google.

  • Característiques Clau d'Angular:

    • Arquitectura basada en components.
    • Enllaç de dades bidireccional.
    • Injecció de dependències.
    • Eines completes i suport CLI.
    • Comunitat forta i documentació extensa.

Configuració de l'Entorn de Desenvolupament

Prerequisits

  • Node.js i npm:

    • Angular requereix Node.js i npm (Node Package Manager) per gestionar les dependències.
    • Descarrega i instal·la des del lloc web oficial de Node.js.

  • Angular CLI:

    • La Interfície de Línia de Comandes d'Angular (CLI) simplifica el procés de desenvolupament.
    • Instal·la Angular CLI globalment utilitzant npm: 
      npm install -g @angular/cli

Creació d'un Nou Projecte Angular

  • Utilitzant Angular CLI:

    • Crea un nou projecte Angular executant: 
      ng new my-angular-app
    • Segueix les indicacions per configurar el teu projecte (p. ex., routing, fulls d'estil).

  • Estructura del Projecte:

    • El projecte generat tindrà una estructura predefinida: 
      my-angular-app/
├── e2e/                # Proves de punta a punta
├── node_modules/       # Dependències del projecte
├── src/                # Fitxers font
│   ├── app/            # Codi de l'aplicació
│   ├── assets/         # Actius estàtics
│   ├── environments/   # Configuracions d'entorn
│   ├── index.html      # Fitxer HTML principal
│   ├── main.ts         # Punt d'entrada principal
│   ├── styles.css      # Estils globals
│   └── ...             # Altres fitxers de configuració
├── angular.json        # Configuració de l'Angular CLI
├── package.json        # Configuració de npm
├── tsconfig.json       # Configuració de TypeScript
└── ...                 # Altres fitxers de configuració

Planificació del Teu Projecte Angular

Definició de l'Abast del Projecte

  • Recopilació de Requisits:

    • Identifica les característiques i funcionalitats clau necessàries.
    • Crea una llista d'històries d'usuari o casos d'ús.

  • Objectius del Projecte:

    • Defineix objectius clars i assolibles per al projecte.
    • Prioritza les característiques basant-te en la importància i la complexitat.

Disseny de l'Arquitectura de l'Aplicació

  • Jerarquia de Components:

    • Divideix l'aplicació en components reutilitzables.
    • Defineix una jerarquia clara i un flux de comunicació entre components.

  • Gestió de l'Estat:

    • Decideix una estratègia de gestió de l'estat (p. ex., utilitzant serveis, NgRx).
    • Assegura un enfocament consistent per gestionar l'estat de l'aplicació.

Configuració del Control de Versions

  • Utilitzant Git:

    • Inicialitza un repositori Git al directori del teu projecte: 
      git init
    • Crea un fitxer .gitignore per excloure fitxers innecessaris: 
      node_modules/
dist/
.env

  • Repositori Remot:

    • Crea un repositori remot en plataformes com GitHub, GitLab o Bitbucket.
    • Enllaça el teu repositori local al repositori remot: 
      git remote add origin <repository-url>

Configuració d'Eines de Desenvolupament

  • Editor de Codi:

    • Utilitza un editor de codi com Visual Studio Code, que té un excel·lent suport per Angular.
    • Instal·la extensions rellevants (p. ex., Angular Language Service, TSLint).

  • Linting i Formatació:

    • Configura eines de linting i formatació per mantenir la qualitat del codi.
    • Exemple de configuració per TSLint: 
      {
  "extends": "tslint:recommended",
  "rules": {
    "quotemark": [true, "single"],
    "semicolon": [true, "always"]
  }
}

Conclusió

Configurar i planificar un projecte Angular implica diversos passos importants, des d'instal·lar les eines necessàries fins a definir l'abast i l'arquitectura del projecte. Seguint aquestes directrius, pots assegurar un procés de desenvolupament ben organitzat i eficient. Una bona planificació i configuració estableixen les bases per a una aplicació Angular exitosa, facilitant la seva gestió i escalabilitat a mesura que el projecte creix.
