Què és el Model OSI?

El Model d'Interconnexió de Sistemes Oberts (OSI) és un marc conceptual que estandarditza les funcions d'un sistema de telecomunicacions o de computació sense tenir en compte la seva estructura interna i tecnologia subjacent. Va ser desenvolupat per l'Organització Internacional de Normalització (ISO) el 1984.

Objectius del Model OSI

  • Estandardització: Proporcionar un estàndard perquè diferents sistemes de xarxes puguin comunicar-se entre si.
  • Interoperabilitat: Facilitar la interoperabilitat entre productes de diferents fabricants.
  • Modularitat: Dividir les funcions de xarxa en capes separades per simplificar el disseny i la implementació.

Les Set Capes del Model OSI

El Model OSI es compon de set capes, cadascuna de les quals té funcions específiques i es comunica amb les capes adjacents.

  1. Capa Física
  2. Capa d'Enllaç de Dades
  3. Capa de Xarxa
  4. Capa de Transport
  5. Capa de Sessió
  6. Capa de Presentació
  7. Capa d'Aplicació

Descripció de Cada Capa

1. Capa Física

  • Funció: Transmissió de bits a través d'un mitjà físic.
  • Exemples: Cables, connectors, voltatges.
  • Dispositius: Hubs, repetidors.

2. Capa d'Enllaç de Dades

  • Funció: Proporcionar transferència de dades lliure d'errors entre dos nodes adjacents.
  • Exemples: Ethernet, Wi-Fi.
  • Dispositius: Switches, ponts.

3. Capa de Xarxa

  • Funció: Determinar la ruta que les dades han de prendre per arribar al seu destí.
  • Exemples: IP (Protocol d'Internet).
  • Dispositius: Routers.

4. Capa de Transport

  • Funció: Proporcionar transferència de dades fiable i control de flux.
  • Exemples: TCP (Protocol de Control de Transmissió), UDP (Protocol de Datagram de Usuari).

5. Capa de Sessió

  • Funció: Establir, gestionar i finalitzar sessions entre aplicacions.
  • Exemples: RPC (Crida a Procediment Remot), NetBIOS.

6. Capa de Presentació

  • Funció: Traduir dades entre el format de la xarxa i el format comprensible per a l'aplicació.
  • Exemples: SSL/TLS (Seguretat de la Capa de Transport), xifrat, compressió.

7. Capa d'Aplicació

  • Funció: Proporcionar serveis de xarxa directament a les aplicacions de l'usuari.
  • Exemples: HTTP, FTP, SMTP.

Comparació de Capes

| Capa | Funció Principal | Exemples de Protocols/Dispositius | |------------------|--------------------------------------------------|-------------------------------------| | Física | Transmissió de bits | Cables, Hubs | | Enllaç de Dades | Transferència de dades lliure d'errors | Ethernet, Switches | | Xarxa | Determinació de rutes | IP, Routers | | Transport | Transferència de dades fiable | TCP, UDP | | Sessió | Gestió de sessions | RPC, NetBIOS | | Presentació | Traducció de dades | SSL/TLS, xifrat | | Aplicació | Serveis de xarxa per a aplicacions | HTTP, FTP, SMTP |

Exemple de Comunicació en el Model OSI

Suposem que un usuari vol accedir a una pàgina web. Aquí està com les capes del Model OSI interactuen:

  1. Capa d'Aplicació: El navegador web (HTTP) sol·licita una pàgina web.
  2. Capa de Presentació: Les dades de la sol·licitud HTTP poden ser xifrades amb SSL/TLS.
  3. Capa de Sessió: S'estableix una sessió entre el navegador i el servidor web.
  4. Capa de Transport: TCP divideix la sol·licitud en segments i assegura el lliurament correcte.
  5. Capa de Xarxa: IP determina la millor ruta per enviar els segments al servidor.
  6. Capa d'Enllaç de Dades: Ethernet encapsula les dades en trames i maneja la transmissió a través de la xarxa local.
  7. Capa Física: Els bits es transmeten a través del cable de xarxa al servidor.

Conclusió

El Model OSI és fonamental per entendre com es estructuren i funcionen les xarxes de comunicació. Cada capa té un rol específic i treballa conjuntament amb les altres per assegurar una comunicació eficient i fiable. Conèixer aquest model és essencial per a qualsevol professional de TI que treballi amb xarxes, ja que proporciona una base sòlida per diagnosticar i resoldre problemes de xarxa.
