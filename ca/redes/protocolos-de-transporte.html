Protocols de Transport

Introducció

Els protocols de transport són fonamentals en l'arquitectura de xarxes, ja que s'encarreguen de gestionar la transmissió de dades entre sistemes finals. En aquesta secció, explorarem els principals protocols de transport, les seves característiques i com s'utilitzen en la pràctica.

Principals Protocols de Transport

TCP (Transmission Control Protocol)

TCP és un dels protocols de transport més utilitzats en les xarxes de dades. Proporciona una connexió fiable i orientada a la connexió entre dos sistemes finals.

  • Característiques de TCP:

    • Fiabilitat: Garanteix el lliurament de les dades sense errors i en l'ordre correcte.
    • Control de flux: Gestiona la quantitat de dades que s'envien per evitar la congestió de la xarxa.
    • Control de congestió: Ajusta la velocitat de transmissió en funció de la capacitat de la xarxa.
    • Orientat a la connexió: Estableix una connexió abans de transmetre dades.

  • Estructura del Segment TCP: | Camp | Descripció | |--------------------|------------------------------------| | Port d'origen | Port de l'emissor | | Port de destí | Port del receptor | | Número de seqüència| Número de seqüència del segment | | Número d'acusament | Número d'acusament de rebut | | Longitud de capçalera| Longitud de la capçalera TCP | | Reservat | Reservat per a ús futur | | Flags | Indicadors de control | | Mida de finestra | Mida de la finestra de recepció | | Checksum | Verificació d'errors | | Punter urgent | Indica dades urgents | | Opcions | Opcions addicionals | | Dades | Dades de l'aplicació |

  • Exemple de Codi TCP en Python:

    import socket

# Crear un socket TCP
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# Connectar al servidor
s.connect(('www.example.com', 80))

# Enviar una sol·licitud HTTP GET
request = "GET / HTTP/1.1\r\nHost: www.example.com\r\n\r\n"
s.sendall(request.encode('utf-8'))

# Rebre la resposta del servidor
response = s.recv(4096)
print(response.decode('utf-8'))

# Tancar la connexió
s.close()

UDP (User Datagram Protocol)

UDP és un protocol de transport que proporciona un servei de transmissió de dades sense connexió i sense garantia de lliurament.

  • Característiques de UDP:

    • No fiable: No garanteix el lliurament de les dades ni l'ordre dels mateixos.
    • Sense connexió: No estableix una connexió abans de transmetre dades.
    • Baixa sobrecàrrega: Té una capçalera més petita i menys control que TCP.
    • Ideal per a aplicacions que requereixen velocitat i poden tolerar pèrdua de dades (e.g., streaming de vídeo, jocs en línia).

  • Estructura del Datagram UDP: | Camp | Descripció | |--------------------|------------------------------------| | Port d'origen | Port de l'emissor | | Port de destí | Port del receptor | | Longitud | Longitud del datagrama UDP | | Checksum | Verificació d'errors | | Dades | Dades de l'aplicació |

  • Exemple de Codi UDP en Python:

    import socket

# Crear un socket UDP
s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# Adreça del servidor
server_address = ('localhost', 10000)

# Missatge a enviar
message = 'Aquest és un missatge UDP'

try:
    # Enviar el missatge
    print(f'Enviant: {message}')
    sent = s.sendto(message.encode('utf-8'), server_address)

    # Rebre resposta
    print('Esperant resposta...')
    data, server = s.recvfrom(4096)
    print(f'Rebut: {data.decode("utf-8")}')
finally:
    # Tancar el socket
    s.close()

Comparació entre TCP i UDP

| Característica | TCP | UDP | |----------------------|------------------------------------|------------------------------------| | Fiabilitat | Alta (garanteix lliurament i ordre)| Baixa (no garanteix lliurament ni ordre)| | Control de flux | Sí | No | | Control de congestió | Sí | No | | Orientat a connexió | Sí | No | | Sobrecàrrega | Alta (capçalera més gran) | Baixa (capçalera més petita) | | Ús típic | Transferència d'arxius, correus electrònics | Streaming de vídeo, jocs en línia |

Conclusió

En aquesta secció, hem explorat els dos principals protocols de transport: TCP i UDP. Tots dos tenen els seus propis avantatges i desavantatges, i s'utilitzen en diferents escenaris segons les necessitats de l'aplicació. TCP és ideal per a aplicacions que requereixen fiabilitat i ordre, mentre que UDP és més adequat per a aplicacions que necessiten velocitat i poden tolerar la pèrdua de dades. Entendre aquestes diferències és crucial per dissenyar i gestionar xarxes de manera efectiva.
