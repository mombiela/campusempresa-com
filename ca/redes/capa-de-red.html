Introducció a la Capa de Xarxa

La Capa de Xarxa és una de les capes fonamentals en el model OSI (Open Systems Interconnection) i en el model TCP/IP. La seva principal funció és determinar com les dades es transfereixen d'un dispositiu a un altre a través de múltiples xarxes. En aquesta secció, explorarem els conceptes clau, protocols i mecanismes que operen en la Capa de Xarxa.

Funcions Principals de la Capa de Xarxa

  • Encaminament (Routing): Determina la ruta òptima per enviar paquets des de l'origen fins al destí.
  • Adreçament (Addressing): Assigna adreces IP als dispositius per identificar-los de manera única a la xarxa.
  • Fragmentació i Reassemblatge: Divideix els paquets grans en fragments més petits per a la seva transmissió i els reassembla al destí.
  • Control de Congestió: Gestiona el trànsit de xarxa per evitar la congestió i assegurar una transmissió eficient.

Protocols de la Capa de Xarxa

Protocol IP (Internet Protocol)

El Protocol IP és el protocol principal de la Capa de Xarxa en el model TCP/IP. Hi ha dues versions principals:

  • IPv4 (Internet Protocol version 4):

    • Utilitza adreces de 32 bits.
    • Format d'adreça: 192.168.1.1
    • Espai d'adreces limitat a aproximadament 4.3 mil milions d'adreces.

  • IPv6 (Internet Protocol version 6):

    • Utilitza adreces de 128 bits.
    • Format d'adreça: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
    • Espai d'adreces pràcticament il·limitat.

Protocol ICMP (Internet Control Message Protocol)

L'ICMP s'utilitza per enviar missatges de control i error entre dispositius de xarxa. Alguns dels seus usos inclouen:

  • Ping: Verifica la connectivitat entre dos dispositius.
  • Traceroute: Rastreja la ruta que segueix un paquet fins al seu destí.

Protocol ARP (Address Resolution Protocol)

L'ARP s'utilitza per mapar adreces IP a adreces MAC (Media Access Control) en una xarxa local.

Exemple de Configuració d'Adreces IP

A continuació, es mostra un exemple de com configurar una adreça IP en un sistema Linux utilitzant la línia de comandes:

# Configurar una adreça IP estàtica en una interfície de xarxa
sudo ifconfig eth0 192.168.1.10 netmask 255.255.255.0

# Configurar la porta d'enllaç predeterminada
sudo route add default gw 192.168.1.1

En aquest exemple:

  • eth0 és la interfície de xarxa.
  • 192.168.1.10 és l'adreça IP assignada.
  • 255.255.255.0 és la màscara de subxarxa.
  • 192.168.1.1 és la porta d'enllaç predeterminada.

Comparació entre IPv4 i IPv6

| Característica | IPv4 | IPv6 | |----------------------|----------------------------|--------------------------------------| | Longitud d'Adreça | 32 bits | 128 bits | | Format d'Adreça | Decimal (e.g., 192.168.1.1)| Hexadecimal (e.g., 2001:0db8::1) | | Espai d'Adreces | ~4.3 mil milions | Pràcticament il·limitat | | Fragmentació | Realitzada per routers i hosts| Realitzada només per l'host emissor | | Seguretat | Opcional (IPSec) | Integrada (IPSec obligatori) |

Conclusió

La Capa de Xarxa és essencial per a la comunicació eficient i efectiva en xarxes de computadors. A través de les seves funcions d'encaminament, adreçament, fragmentació i control de congestió, assegura que les dades arribin al seu destí de manera fiable. Comprendre els protocols i mecanismes d'aquesta capa és crucial per a qualsevol professional de TI que treballi amb xarxes.

En les següents seccions del curs, aprofundirem en cadascun d'aquests conceptes i explorarem com s'implementen en xarxes reals.
