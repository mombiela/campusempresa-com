Introducció

La detecció de col·lisions i la física són aspectes fonamentals del desenvolupament de jocs. Asseguren que els objectes en el món del joc interactuïn de manera realista. Aquest tema cobrirà els conceptes bàsics de la detecció de col·lisions, diferents algoritmes i com implementar la física utilitzant DirectX.

Conceptes Clau

  • Detecció de Col·lisions: El procés de detectar quan dos o més objectes en un món de joc s'intersecten o entren en contacte.
  • Motor de Física: Un component de programari que simula sistemes físics aplicant les lleis de la física.
  • Volums de Delimitació: Formes simplificades (per exemple, esferes, caixes) utilitzades per aproximar la forma d'objectes complexos per a la detecció de col·lisions.
  • Resposta: L'acció que es pren quan es detecta una col·lisió, com ara rebotar, aturar-se o lliscar.

Conceptes Bàsics de la Detecció de Col·lisions

Volums de Delimitació

Els volums de delimitació s'utilitzen per simplificar la detecció de col·lisions. Els tipus comuns inclouen:

  • Caixa de Delimitació Alineada amb els Eixos (AABB): Una caixa alineada amb els eixos de coordenades.
  • Esfera de Delimitació: Una esfera que engloba l'objecte.

Detecció de Col·lisions AABB

Per comprovar si dues AABB s'intersecten, pots utilitzar el següent codi:

struct AABB {
    DirectX::XMFLOAT3 min; // Punt mínim (x, y, z)
    DirectX::XMFLOAT3 max; // Punt màxim (x, y, z)
};

bool CheckAABBCollision(const AABB& box1, const AABB& box2) {
    return (box1.min.x <= box2.max.x && box1.max.x >= box2.min.x) &&
           (box1.min.y <= box2.max.y && box1.max.y >= box2.min.y) &&
           (box1.min.z <= box2.max.z && box1.max.z >= box2.min.z);
}

Detecció de Col·lisions d'Esferes de Delimitació

Per comprovar si dues esferes de delimitació s'intersecten, pots utilitzar el següent codi:

struct BoundingSphere {
    DirectX::XMFLOAT3 center; // Punt central (x, y, z)
    float radius;             // Radi de l'esfera
};

bool CheckSphereCollision(const BoundingSphere& sphere1, const BoundingSphere& sphere2) {
    float distanceSquared = (sphere1.center.x - sphere2.center.x) * (sphere1.center.x - sphere2.center.x) +
                            (sphere1.center.y - sphere2.center.y) * (sphere1.center.y - sphere2.center.y) +
                            (sphere1.center.z - sphere2.center.z) * (sphere1.center.z - sphere2.center.z);
    float radiusSum = sphere1.radius + sphere2.radius;
    return distanceSquared <= (radiusSum * radiusSum);
}

Detecció de Col·lisions Avançada

Teorema de l'Eix Separador (SAT)

El Teorema de l'Eix Separador s'utilitza per a formes més complexes. Afirma que dues formes convexes no s'intersecten si existeix una línia (eix) al llarg de la qual les seves projeccions no se superposen.

Implementació del SAT

Implementar el SAT implica projectar els vèrtexs de les formes sobre eixos separadors potencials i comprovar si hi ha superposicions. Això és més complex i requereix una bona comprensió de les matemàtiques vectorials.

Simulació de Física

Lleis del Moviment de Newton

Els motors de física sovint simulen el moviment utilitzant les lleis de Newton:

  1. Primera Llei: Un objecte roman en repòs o en moviment uniforme a menys que una força actuï sobre ell.
  2. Segona Llei: L'acceleració d'un objecte és proporcional a la força neta que actua sobre ell i inversament proporcional a la seva massa (F = ma).
  3. Tercera Llei: Per a cada acció, hi ha una reacció igual i oposada.

Simulació Bàsica de Física

Per simular física bàsica, pots actualitzar la posició i la velocitat d'un objecte basant-te en les forces que actuen sobre ell:

struct PhysicsObject {
    DirectX::XMFLOAT3 position;
    DirectX::XMFLOAT3 velocity;
    DirectX::XMFLOAT3 acceleration;
    float mass;
};

void UpdatePhysics(PhysicsObject& obj, float deltaTime) {
    // Actualitzar la velocitat
    obj.velocity.x += obj.acceleration.x * deltaTime;
    obj.velocity.y += obj.acceleration.y * deltaTime;
    obj.velocity.z += obj.acceleration.z * deltaTime;

    // Actualitzar la posició
    obj.position.x += obj.velocity.x * deltaTime;
    obj.position.y += obj.velocity.y * deltaTime;
    obj.position.z += obj.velocity.z * deltaTime;
}

Resposta a la Col·lisió

Quan es detecta una col·lisió, cal respondre adequadament. Per exemple, pots invertir la velocitat d'un objecte per simular un rebot:

void HandleCollision(PhysicsObject& obj1, PhysicsObject& obj2) {
    // Resposta simple de col·lisió elàstica
    DirectX::XMFLOAT3 temp = obj1.velocity;
    obj1.velocity = obj2.velocity;
    obj2.velocity = temp;
}

Conclusió

La detecció de col·lisions i la física són crucials per crear jocs realistes i atractius. Entenent i implementant algoritmes bàsics de detecció de col·lisions i simulacions de física, pots crear mons de joc més dinàmics i interactius. A mesura que avancis, podràs explorar algoritmes més complexos i motors de física per millorar encara més els teus jocs.
