En aquesta secció, explorarem com passar dades entre pàgines en una aplicació Ionic. Aquesta és una habilitat crucial per crear aplicacions mòbils dinàmiques i interactives. Cobrirem diversos mètodes, incloent l'ús del Router d'Angular, el NavController d'Ionic i serveis compartits.

Conceptes Clau

  • Routing i Navegació: Entendre com Ionic gestiona la navegació entre pàgines.
  • Passar Dades via Paràmetres d'URL: Utilitzar el Router d'Angular per passar dades.
  • Utilitzar el NavController d'Ionic: Aprofitar el controlador de navegació integrat d'Ionic.
  • Serveis Compartits: Utilitzar serveis d'Angular per compartir dades entre components.

Routing i Navegació

Ionic utilitza el sistema de routing d'Angular per navegar entre pàgines. Entendre com configurar rutes i navegar entre elles és el primer pas per passar dades entre pàgines.

Configurar Rutes

A Ionic, les rutes es defineixen al fitxer app-routing.module.ts. Aquí teniu un exemple de com configurar rutes:

import { NgModule } from '@angular/core';
import { PreloadAllModules, RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: '',
    redirectTo: 'home',
    pathMatch: 'full'
  },
  {
    path: 'home',
    loadChildren: () => import('./home/home.module').then(m => m.HomePageModule)
  },
  {
    path: 'details/:id',
    loadChildren: () => import('./details/details.module').then(m => m.DetailsPageModule)
  }
];

@NgModule({
  imports: [
    RouterModule.forRoot(routes, { preloadingStrategy: PreloadAllModules })
  ],
  exports: [RouterModule]
})
export class AppRoutingModule { }

En aquest exemple, tenim dues rutes: home i details/:id. La part :id és un paràmetre de ruta que podem utilitzar per passar dades a la pàgina details.

Passar Dades via Paràmetres d'URL

Un mètode comú per passar dades entre pàgines és utilitzar paràmetres d'URL. Això es fa utilitzant el Router d'Angular.

Enviar Dades

Per navegar a la pàgina details i passar un paràmetre id, podeu utilitzar el servei Router:

import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private router: Router) {}

  goToDetails(id: number) {
    this.router.navigate(['/details', id]);
  }
}

Rebre Dades

A la pàgina details, podeu recuperar el paràmetre id utilitzant el servei ActivatedRoute:

import { Component, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  id: number;

  constructor(private route: ActivatedRoute) {}

  ngOnInit() {
    this.id = +this.route.snapshot.paramMap.get('id');
  }
}

Utilitzar el NavController d'Ionic

Ionic proporciona el seu propi controlador de navegació, NavController, que també es pot utilitzar per passar dades entre pàgines.

Enviar Dades

Per navegar i passar dades utilitzant NavController, podeu utilitzar el mètode navigateForward amb un objecte d'estat:

import { Component } from '@angular/core';
import { NavController } from '@ionic/angular';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private navCtrl: NavController) {}

  goToDetails(id: number) {
    this.navCtrl.navigateForward('/details', {
      state: {
        id: id
      }
    });
  }
}

Rebre Dades

A la pàgina details, podeu recuperar les dades de l'estat de navegació:

import { Component, OnInit } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  id: number;

  constructor(private router: Router) {}

  ngOnInit() {
    if (this.router.getCurrentNavigation().extras.state) {
      this.id = this.router.getCurrentNavigation().extras.state.id;
    }
  }
}

Serveis Compartits

Un altre mètode per passar dades entre pàgines és utilitzar serveis d'Angular. Això és particularment útil per compartir dades o estat complexos.

Crear un Servei

Primer, creeu un servei per mantenir les dades compartides:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  private data: any;

  setData(data: any) {
    this.data = data;
  }

  getData() {
    return this.data;
  }
}

Utilitzar el Servei

Podeu utilitzar aquest servei als vostres components per establir i obtenir dades:

import { Component } from '@angular/core';
import { Router } from '@angular/router';
import { DataService } from '../services/data.service';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private router: Router, private dataService: DataService) {}

  goToDetails(data: any) {
    this.dataService.setData(data);
    this.router.navigate(['/details']);
  }
}
import { Component, OnInit } from '@angular/core';
import { DataService } from '../services/data.service';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  data: any;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.data = this.dataService.getData();
  }
}

Conclusió

Passar dades entre pàgines en una aplicació Ionic es pot aconseguir a través de diversos mètodes, cadascun amb els seus propis casos d'ús i avantatges. Tant si trieu utilitzar paràmetres d'URL, el NavController d'Ionic o serveis compartits, entendre aquestes tècniques us ajudarà a crear aplicacions més dinàmiques i interactives. Experimenteu amb cada mètode per veure quin s'adapta millor a les necessitats de la vostra aplicació.
