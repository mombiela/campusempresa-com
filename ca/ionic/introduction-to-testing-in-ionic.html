Introducció a les Proves en Ionic
Proves Unitàries amb Jasmine i Karma ►

Visió General

Les proves són una part crucial del procés de desenvolupament, assegurant que la teva aplicació funcioni com s'espera i estigui lliure d'errors. En aquesta secció, t'introduirem als conceptes bàsics de les proves en Ionic, cobrint diferents tipus de proves, eines i millors pràctiques.

Conceptes Clau

  • Proves Unitàries: Provar components o serveis individuals de manera aïllada.
  • Proves d'Integració: Provar com diferents parts de l'aplicació funcionen juntes.
  • Proves de Cap a Cap (E2E): Provar tot el flux de l'aplicació de principi a fi.
  • Desenvolupament Guiat per Proves (TDD): Escriure proves abans d'escriure el codi real.

Eines per a les Proves en Ionic

  • Jasmine: Un marc de desenvolupament guiat pel comportament per provar codi JavaScript.
  • Karma: Un executor de proves que et permet executar codi JavaScript en múltiples navegadors reals.
  • Protractor: Un marc de proves de cap a cap per a aplicacions Angular i AngularJS.
  • Ionic CLI: Proporciona comandes per executar proves fàcilment.

Configuració de l'Entorn de Proves

Abans de començar a escriure proves, has de configurar el teu entorn de proves. Segueix aquests passos:

  1. Instal·la Jasmine i Karma:

    npm install jasmine-core karma karma-jasmine karma-chrome-launcher --save-dev

  2. Configura Karma: Crea un fitxer karma.conf.js a l'arrel del teu projecte:

    module.exports = function(config) {
  config.set({
    basePath: '',
    frameworks: ['jasmine'],
    files: [
      'src/**/*.spec.ts'
    ],
    preprocessors: {
      'src/**/*.spec.ts': ['webpack']
    },
    webpack: {
      // Configuració de Webpack
    },
    browsers: ['Chrome'],
    singleRun: false
  });
};

  3. Instal·la Protractor:

    npm install protractor --save-dev

  4. Configura Protractor: Crea un fitxer protractor.conf.js a l'arrel del teu projecte:

    exports.config = {
  framework: 'jasmine',
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['e2e/**/*.e2e-spec.ts'],
  capabilities: {
    browserName: 'chrome'
  }
};

Escriure Proves Unitàries

Les proves unitàries es concentren en provar components o serveis individuals. Aquí tens un exemple d'una prova unitària simple per a un servei d'Ionic:

  1. Crea un Servei:

    // src/app/services/data.service.ts
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  getData() {
    return 'Hola, Món!';
  }
}

  2. Escriu una Prova Unitària:

    // src/app/services/data.service.spec.ts
import { TestBed } from '@angular/core/testing';
import { DataService } from './data.service';

describe('DataService', () => {
  let service: DataService;

  beforeEach(() => {
    TestBed.configureTestingModule({});
    service = TestBed.inject(DataService);
  });

  it('hauria de retornar "Hola, Món!"', () => {
    expect(service.getData()).toBe('Hola, Món!');
  });
});

Escriure Proves de Cap a Cap

Les proves de cap a cap simulen interaccions reals d'usuari. Aquí tens un exemple d'una prova E2E per a una pàgina d'Ionic:

  1. Crea una Pàgina:

    // src/app/pages/home/home.page.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-home',
  templateUrl: './home.page.html',
  styleUrls: ['./home.page.scss'],
})
export class HomePage {
  title = 'Home Page';
}

  2. Escriu una Prova E2E:

    // e2e/home.e2e-spec.ts
import { browser, by, element } from 'protractor';

describe('Home Page', () => {
  it('hauria de mostrar el títol', () => {
    browser.get('/');
    expect(element(by.css('app-home h1')).getText()).toEqual('Home Page');
  });
});

Executar Proves

  • Executar Proves Unitàries:

    npx karma start

  • Executar Proves E2E:

    npx protractor protractor.conf.js

Millors Pràctiques

  • Escriu proves per a totes les parts crítiques de la teva aplicació.
  • Utilitza mocks i stubs per aïllar components i serveis.
  • Mantingues les teves proves simples i centrades en una funcionalitat.
  • Executa les teves proves regularment per detectar problemes aviat.

Conclusió

Les proves són una part essencial del desenvolupament d'aplicacions robustes en Ionic. Entenent i implementant proves unitàries, proves d'integració i proves de cap a cap, pots assegurar-te que la teva aplicació és fiable i funciona com s'espera. Utilitza les eines i les millors pràctiques discutides en aquesta secció per crear una estratègia de proves sòlida per als teus projectes en Ionic.
