Introducció a les Proves de Cap a Cap

Les proves de cap a cap (E2E) són una metodologia utilitzada per provar si el flux d'una aplicació funciona com està dissenyat de principi a fi. L'objectiu de les proves E2E és identificar les dependències del sistema i assegurar-se que la integritat de les dades es manté entre els diversos components i sistemes del sistema.

  • Conceptes Clau:
    • Flux d'Usuari: Provar el viatge complet de l'usuari.
    • Dependències del Sistema: Assegurar-se que totes les parts del sistema funcionen juntes.
    • Integritat de les Dades: Verificar la consistència de les dades a través de l'aplicació.

Què és Protractor?

Protractor és un marc de proves de cap a cap per a aplicacions Angular i AngularJS. Està construït sobre WebDriverJS, que utilitza esdeveniments natius i controladors específics del navegador per interactuar amb la teva aplicació com ho faria un usuari.

  • Característiques Clau:
    • Consciència d'Angular: Protractor entén els elements específics d'Angular.
    • Espera Automàtica: Protractor espera automàticament que l'aplicació sigui estable.
    • Integració Sense Fissures: Funciona bé amb Jasmine, Mocha i altres marcs de proves.

Configurar Protractor en un Projecte Ionic

Per començar amb Protractor en un projecte Ionic, cal instal·lar Protractor i configurar-lo.

Pas 1: Instal·lar Protractor

Primer, instal·la Protractor globalment utilitzant npm:

npm install -g protractor

Pas 2: Actualitzar WebDriver

Actualitza el gestor de WebDriver per assegurar-te que tens la versió més recent:

webdriver-manager update

Pas 3: Configurar Protractor

Crea un fitxer protractor.conf.js a l'arrel del teu projecte:

exports.config = {
  framework: 'jasmine',
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['e2e/**/*.spec.js'],
  capabilities: {
    browserName: 'chrome'
  },
  onPrepare: function() {
    browser.ignoreSynchronization = true;
  }
};

Pas 4: Iniciar WebDriver

Inicia el servidor WebDriver:

webdriver-manager start

Escriure la Teva Primera Prova amb Protractor

Escrivim una prova senzilla per comprovar si la pàgina d'inici de la teva aplicació Ionic es carrega correctament.

Exemple de Prova

Crea un fitxer e2e/homepage.spec.js:

describe('Ionic App Homepage', function() {
  it('should have a title', function() {
    browser.get('http://localhost:8100');
    expect(browser.getTitle()).toEqual('My Ionic App');
  });
});

Explicació

  • describe: Un conjunt de proves.
  • it: Una prova única.
  • browser.get: Navega a la URL especificada.
  • expect: Una afirmació per comprovar si el títol és com s'espera.

Executar les Teves Proves amb Protractor

Per executar les teves proves, utilitza la següent comanda:

protractor protractor.conf.js

Característiques Avançades de Protractor

A mesura que et familiaritzis amb Protractor, pots aprofitar les seves característiques avançades per escriure proves més completes.

Gestionar Operacions Asíncrones

Protractor espera automàticament que Angular acabi de renderitzar i fer trucades HTTP. No obstant això, per a aplicacions no Angular o escenaris específics, potser necessitaràs gestionar operacions asíncrones manualment.

Exemple amb Operacions Asíncrones

it('should wait for an element to be visible', function() {
  var EC = protractor.ExpectedConditions;
  var button = element(by.id('myButton'));
  browser.wait(EC.visibilityOf(button), 5000);
  button.click();
});

Objectes de Pàgina

Els Objectes de Pàgina són un patró de disseny que t'ajuda a escriure proves més netes encapsulant l'estructura i les operacions de la pàgina.

Exemple d'Objecte de Pàgina

Crea un fitxer e2e/pages/homepage.js:

var HomePage = function() {
  this.title = element(by.css('h1'));
  this.get = function() {
    browser.get('http://localhost:8100');
  };
};

module.exports = new HomePage();

Utilitza l'Objecte de Pàgina a la teva prova:

var HomePage = require('./pages/homepage');

describe('Ionic App Homepage', function() {
  it('should have a title', function() {
    HomePage.get();
    expect(HomePage.title.getText()).toEqual('Welcome to My Ionic App');
  });
});

Conclusió

Les proves de cap a cap amb Protractor en un projecte Ionic asseguren que la teva aplicació funciona com s'espera des de la perspectiva de l'usuari. Configurant Protractor, escrivint proves i aprofitant característiques avançades com les operacions asíncrones i els Objectes de Pàgina, pots crear suites de proves robustes i mantenibles. Això no només millora la qualitat de la teva aplicació, sinó que també millora el flux de treball de desenvolupament detectant problemes a l'inici del procés.
