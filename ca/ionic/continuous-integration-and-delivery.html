El Projecte Sobre nosaltres Contribuir Donacions Llicència

La Integració Contínua (CI) i el Lliurament Continu (CD) són pràctiques essencials en el desenvolupament de programari modern. Ajuden a assegurar que el teu codi estigui sempre en un estat desplegable, redueixen els problemes d'integració i permeten una entrega més ràpida de característiques i correccions. En aquest tema, explorarem com implementar CI/CD per a una aplicació Ionic.

Introducció a la Integració i Lliurament Continu

  • Integració Contínua (CI): La pràctica d'integrar automàticament els canvis de codi de múltiples contribuïdors en un repositori compartit diverses vegades al dia.
  • Lliurament Continu (CD): La pràctica de desplegar automàticament cada canvi de codi a un entorn de staging o producció després de passar per la pipeline de CI.

Configuració de la Integració Contínua per a Ionic

Triar una Eina de CI

Hi ha diverses eines de CI disponibles, cadascuna amb els seus propis punts forts. Algunes opcions populars inclouen:

  • GitHub Actions
  • Travis CI
  • CircleCI
  • Jenkins

Per a aquest curs, utilitzarem GitHub Actions degut a la seva integració perfecta amb els repositoris de GitHub.

Configuració de GitHub Actions

  1. Crear un Repositori de GitHub: Si encara no ho has fet, crea un repositori per al teu projecte Ionic a GitHub.
  2. Afegir un Fitxer de Workflow: Al teu repositori, crea un directori anomenat .github/workflows i afegeix un fitxer YAML (per exemple, ci.yml).
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Build the app
      run: npm run build

    - name: Run tests
      run: npm test

Explicació del Fitxer de Workflow

  • name: El nom del workflow.
  • on: Especifica els esdeveniments que desencadenen el workflow (per exemple, push, pull_request).
  • jobs: Defineix les tasques a executar en el workflow.
    • runs-on: Especifica el sistema operatiu per a la tasca.
    • steps: Llista els passos a executar en la tasca.
      • Checkout repository: Utilitza l'acció actions/checkout per clonar el repositori.
      • Set up Node.js: Utilitza l'acció actions/setup-node per configurar l'entorn Node.js.
      • Install dependencies: Executa npm install per instal·lar les dependències del projecte.
      • Build the app: Executa npm run build per construir l'aplicació Ionic.
      • Run tests: Executa npm test per executar la suite de proves.

Configuració del Lliurament Continu per a Ionic

Desplegar a Firebase Hosting

Firebase Hosting és una opció popular per desplegar aplicacions web. Per desplegar la teva aplicació Ionic a Firebase Hosting, segueix aquests passos:

  1. Instal·lar Firebase CLI: Si encara no ho has fet, instal·la Firebase CLI.
npm install -g firebase-tools
  1. Inicialitzar Firebase en el Teu Projecte: Executa la següent comanda i segueix les indicacions per configurar Firebase Hosting.
firebase init
  1. Afegir Passos de Desplegament a GitHub Actions: Actualitza el teu fitxer de workflow per incloure passos per desplegar a Firebase Hosting.
name: CI/CD

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Build the app
      run: npm run build

    - name: Run tests
      run: npm test

    - name: Deploy to Firebase Hosting
      env:
        FIREBASE_TOKEN: ${{ secrets.FIREBASE_TOKEN }}
      run: |
        npm install -g firebase-tools
        firebase deploy --only hosting

Explicació dels Passos de Desplegament

  • Deploy to Firebase Hosting: Aquest pas instal·la Firebase CLI i desplega l'aplicació a Firebase Hosting.
    • env: Configura la variable d'entorn FIREBASE_TOKEN, que és necessària per a l'autenticació. Necessites crear un token de Firebase i afegir-lo als secrets del teu repositori de GitHub.

Conclusió

En aquest tema, hem cobert els conceptes bàsics de la Integració Contínua i el Lliurament Continu per a una aplicació Ionic. Hem configurat una pipeline de CI utilitzant GitHub Actions i l'hem configurat per construir, provar i desplegar la nostra aplicació a Firebase Hosting. Implementant CI/CD, pots assegurar-te que el teu codi estigui sempre en un estat desplegable i lliurar noves característiques i correccions de manera més ràpida i fiable.
