Utilitzant Firebase amb Ionic ►

En aquesta secció, explorarem com emmagatzemar dades localment en una aplicació Ionic. Això és crucial per crear aplicacions que puguin funcionar fora de línia o que necessitin guardar preferències d'usuari, configuracions o altres dades.

Conceptes Clau

  • Emmagatzematge Local: Un mecanisme simple d'emmagatzematge de claus i valors.
  • SQLite: Un motor de base de dades SQL complet.
  • Ionic Storage: Una API unificada per a diferents motors d'emmagatzematge.

Emmagatzematge Local

L'Emmagatzematge Local és un mecanisme d'emmagatzematge web que et permet emmagatzemar dades al navegador. És simple d'utilitzar però té limitacions en termes de mida d'emmagatzematge i complexitat de dades.

Utilitzant l'Emmagatzematge Local

Per utilitzar l'Emmagatzematge Local en una aplicació Ionic, pots interactuar directament amb l'objecte window.localStorage.

Exemple

// Desa dades a l'Emmagatzematge Local
window.localStorage.setItem('username', 'john_doe');

// Recupera dades de l'Emmagatzematge Local
const username = window.localStorage.getItem('username');
console.log(username); // Sortida: john_doe

// Elimina dades de l'Emmagatzematge Local
window.localStorage.removeItem('username');

Limitacions

  • Capacitat d'emmagatzematge limitada (normalment al voltant de 5MB).
  • Només suporta tipus de dades string.
  • API síncrona, que podria bloquejar el fil principal.

SQLite

SQLite és un motor de base de dades SQL potent i complet que es pot incrustar a la teva aplicació Ionic. És adequat per a necessitats d'emmagatzematge de dades més complexes.

Configurant SQLite

Per utilitzar SQLite en una aplicació Ionic, necessites instal·lar el plugin Cordova SQLite.

ionic cordova plugin add cordova-sqlite-storage
npm install @ionic-native/sqlite

Utilitzant SQLite

Exemple

import { SQLite, SQLiteObject } from '@ionic-native/sqlite/ngx';

constructor(private sqlite: SQLite) {}

this.sqlite.create({
  name: 'data.db',
  location: 'default'
})
.then((db: SQLiteObject) => {

  db.executeSql('create table danceMoves(name VARCHAR(32))', [])
    .then(() => console.log('Executed SQL'))
    .catch(e => console.log(e));

})
.catch(e => console.log(e));

Avantatges

  • Suporta consultes i transaccions complexes.
  • Pot gestionar grans quantitats de dades.
  • API asíncrona, que és millor per al rendiment.

Ionic Storage

Ionic Storage proporciona una API simple i unificada per a diferents motors d'emmagatzematge, incloent l'Emmagatzematge Local, IndexedDB i SQLite. És altament recomanable per a la majoria dels casos d'ús a causa de la seva flexibilitat i facilitat d'ús.

Configurant Ionic Storage

Per utilitzar Ionic Storage, necessites instal·lar els paquets necessaris.

npm install @ionic/storage-angular
npm install @ionic/storage

Configurant Ionic Storage

Necessites configurar Ionic Storage en el teu mòdul d'aplicació.

import { IonicStorageModule } from '@ionic/storage-angular';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,
    IonicModule.forRoot(),
    AppRoutingModule,
    IonicStorageModule.forRoot()
  ],
  bootstrap: [AppComponent]
})
export class AppModule {}

Utilitzant Ionic Storage

Exemple

import { Storage } from '@ionic/storage-angular';

constructor(private storage: Storage) {
  this.init();
}

async init() {
  await this.storage.create();
}

// Desa dades
this.storage.set('username', 'john_doe');

// Recupera dades
this.storage.get('username').then((val) => {
  console.log(val); // Sortida: john_doe
});

// Elimina dades
this.storage.remove('username');

Avantatges

  • API unificada per a diferents motors d'emmagatzematge.
  • API asíncrona.
  • Fàcil de configurar i utilitzar.

Resum

En aquesta secció, hem explorat diferents mètodes per emmagatzemar dades localment en una aplicació Ionic:

  • Emmagatzematge Local: Simple i fàcil d'utilitzar però limitat en capacitat i funcionalitat.
  • SQLite: Potent i adequat per a necessitats d'emmagatzematge de dades complexes.
  • Ionic Storage: Proporciona una API unificada per a diferents motors d'emmagatzematge, fent-lo flexible i fàcil d'utilitzar.

Escollir el mecanisme d'emmagatzematge adequat depèn dels requisits de la teva aplicació, com la quantitat de dades, la complexitat de les consultes i la necessitat de funcionalitat fora de línia.
