Nivell Principiant

Comprensió dels Conceptes Bàsics de Jenkins

  • Pregunta 1: Què és Jenkins i per a què s'utilitza principalment?

    • Resposta: Jenkins és un servidor d'automatització de codi obert utilitzat per automatitzar parts del procés de desenvolupament de programari, com ara la construcció, les proves i el desplegament de codi.

  • Pregunta 2: Com s'instal·la Jenkins en una màquina local?

    • Resposta: Jenkins es pot instal·lar en una màquina local descarregant l'arxiu WAR de Jenkins des del lloc web oficial i executant-lo amb una comanda Java: 
      java -jar jenkins.war
      Alternativament, Jenkins es pot instal·lar utilitzant gestors de paquets com apt per a sistemes basats en Debian o brew per a macOS.

Configuració Bàsica

  • Pregunta 3: Descriu els passos per crear una nova feina a Jenkins.

    • Resposta:
      1. Obre el tauler de control de Jenkins.
      2. Fes clic a "New Item".
      3. Introdueix un nom per a la feina.
      4. Selecciona el tipus de feina (per exemple, projecte Freestyle).
      5. Fes clic a "OK".
      6. Configura els paràmetres de la feina i desa.

  • Pregunta 4: Com configures una feina de Jenkins per obtenir codi d'un repositori Git?

    • Resposta:
      1. A la pàgina de configuració de la feina, ves a la secció "Source Code Management".
      2. Selecciona "Git".
      3. Introdueix la URL del repositori.
      4. Proporciona les credencials si és necessari.
      5. Especifica la branca a construir (opcional).

Nivell Intermedi

Pipeline com a Codi

  • Pregunta 5: Què és un Pipeline de Jenkins i per què és útil?

    • Resposta: Un Pipeline de Jenkins és un conjunt de plugins que suporta la implementació i integració de pipelines de lliurament continu a Jenkins. Permet definir tot el procés de construcció, incloent les etapes de construcció, prova i desplegament, com a codi, cosa que facilita la gestió i el control de versions.

  • Pregunta 6: Escriu un script senzill de Pipeline de Jenkins per construir un projecte Java utilitzant Maven.

    • Resposta: 
      pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }
    }
}

Gestió de Nodes de Jenkins

  • Pregunta 7: Explica la diferència entre un master de Jenkins i un agent de Jenkins.

    • Resposta: El master de Jenkins és la unitat de control central que gestiona l'entorn de Jenkins, maneja la programació de feines i proporciona la interfície d'usuari. Els agents de Jenkins (o esclaus) són màquines que realitzen les tasques de construcció real segons les instruccions del master.

  • Pregunta 8: Com afegeixes un nou node agent a Jenkins?

    • Resposta:
      1. Ves a "Manage Jenkins" > "Manage Nodes and Clouds".
      2. Fes clic a "New Node".
      3. Introdueix un nom per al node i selecciona "Permanent Agent".
      4. Configura els paràmetres del node, com ara el directori arrel remot i el mètode de llançament.
      5. Desa i llança l'agent.

Nivell Avançat

Tècniques Avançades de Pipeline

  • Pregunta 9: Com pots utilitzar Jenkins per desplegar una aplicació en un clúster de Kubernetes?
    • Resposta:
      1. Instal·la el plugin de Kubernetes a Jenkins.
      2. Configura les credencials de Kubernetes a Jenkins.
      3. Crea un script de Pipeline de Jenkins que utilitzi el plugin de Kubernetes per desplegar l'aplicació:
      pipeline {
    agent any
    environment {
        KUBECONFIG = credentials('kubeconfig')
    }
    stages {
        stage('Deploy') {
            steps {
                sh 'kubectl apply -f deployment.yaml'
            }
        }
    }
}

Seguretat i Millors Pràctiques

  • Pregunta 10: Quines són algunes de les millors pràctiques per assegurar una instància de Jenkins?
    • Resposta:
      • Utilitza HTTPS per assegurar la comunicació.
      • Activa i configura l'autenticació i l'autorització.
      • Actualitza regularment Jenkins i els seus plugins.
      • Utilitza el control d'accés basat en rols (RBAC) per limitar els permisos dels usuaris.
      • Fes còpies de seguretat regulars de les configuracions i dades de Jenkins.

Conclusió

Aquest conjunt de preguntes d'examen de mostra cobreix una gamma de temes des dels conceptes bàsics de Jenkins fins a tècniques avançades de pipeline i millors pràctiques de seguretat. Practicant aquestes preguntes, els estudiants poden reforçar la seva comprensió de Jenkins i preparar-se per a escenaris del món real on necessitin implementar i gestionar Jenkins en un entorn professional.
