Introducció a Pipeline com a Codi

Pipeline com a Codi és una pràctica on tot el pipeline de construcció, prova i desplegament es defineix i es gestiona a través de codi. Aquest enfocament aporta els beneficis del control de versions, la col·laboració i l'automatització al pipeline de CI/CD.

  • Definició: Pipeline com a Codi implica escriure la configuració del pipeline en un llenguatge de programació o script.
  • Beneficis:
    • Control de versions de les configuracions del pipeline.
    • Col·laboració i revisió més fàcil.
    • Reutilització i modularitat.
    • Automatització i consistència.

Conceptes Bàsics de Jenkins Pipeline

Jenkins Pipeline és un conjunt de plugins que suporta la implementació i integració de pipelines de lliurament continu a Jenkins.

  • Tipus de Pipelines de Jenkins:
    • Pipeline Declaratiu: Una sintaxi més estructurada i senzilla.
    • Pipeline Escrit: Una sintaxi més flexible i potent.

Pipeline Declaratiu

Pipeline Declaratiu proporciona una sintaxi més senzilla i es recomana per a la majoria d'usuaris.

Exemple:

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}
  • Explicació:
    • pipeline: L'element arrel d'un Pipeline Declaratiu.
    • agent: Defineix on s'executarà el pipeline o una etapa específica.
    • stages: Conté una seqüència d'una o més directives d'etapa.
    • stage: Representa una fase en el pipeline (per exemple, Build, Test, Deploy).
    • steps: Conté una sèrie de passos a executar en una etapa.

Pipeline Escrit

Pipeline Escrit utilitza una sintaxi basada en Groovy més flexible.

Exemple:

node {
    stage('Build') {
        echo 'Building...'
    }
    stage('Test') {
        echo 'Testing...'
    }
    stage('Deploy') {
        echo 'Deploying...'
    }
}
  • Explicació:
    • node: Assigna un espai de treball i executa el pipeline en un agent disponible.
    • stage: Similar al Pipeline Declaratiu, representa una fase en el pipeline.
    • echo: Imprimeix un missatge a la sortida de la consola.

Conceptes Avançats de Pipeline

A mesura que et familiaritzis amb els Pipelines de Jenkins, pots explorar conceptes avançats per millorar els teus pipelines.

  • Execució Paral·lela: Executar múltiples etapes o passos simultàniament.
  • Biblioteques de Pipeline: Reutilitzar codi comú de pipeline en múltiples pipelines.
  • Variables d'Entorn: Utilitzar i gestionar variables d'entorn dins dels pipelines.
  • Gestió de Credencials: Gestionar de manera segura informació sensible com contrasenyes i claus API.

Execució Paral·lela

L'execució paral·lela et permet executar múltiples etapes o passos simultàniament.

Exemple:

pipeline {
    agent any
    stages {
        stage('Parallel Stage') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        echo 'Running unit tests...'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        echo 'Running integration tests...'
                    }
                }
            }
        }
    }
}
  • Explicació:
    • parallel: Defineix múltiples etapes o passos a executar simultàniament.

Biblioteques de Pipeline

Les biblioteques de pipeline et permeten compartir i reutilitzar codi en múltiples pipelines.

Exemple:

@Library('my-shared-library') _
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                mySharedLibrary.build()
            }
        }
    }
}
  • Explicació:
    • @Library('my-shared-library'): Importa una biblioteca compartida.
    • mySharedLibrary.build(): Crida una funció de la biblioteca compartida.

Variables d'Entorn

Les variables d'entorn es poden utilitzar per passar configuracions i informació sensible.

Exemple:

pipeline {
    agent any
    environment {
        MY_VAR = 'some_value'
    }
    stages {
        stage('Print Env Var') {
            steps {
                echo "MY_VAR is ${env.MY_VAR}"
            }
        }
    }
}
  • Explicació:
    • environment: Defineix variables d'entorn per al pipeline.
    • ${env.MY_VAR}: Accedeix al valor de la variable d'entorn.

Gestió de Credencials

Jenkins proporciona una manera segura de gestionar credencials.

Exemple:

pipeline {
    agent any
    environment {
        MY_SECRET = credentials('my-credentials-id')
    }
    stages {
        stage('Use Credentials') {
            steps {
                echo "Using secret: ${env.MY_SECRET}"
            }
        }
    }
}
  • Explicació:
    • credentials('my-credentials-id'): Recupera les credencials emmagatzemades a Jenkins.

Conclusió

Pipeline com a Codi a Jenkins et permet definir els teus pipelines de CI/CD d'una manera estructurada i controlada per versions. Començant amb Pipelines Declaratius i Escrits bàsics, pots explorar gradualment conceptes avançats com l'execució paral·lela, biblioteques de pipeline, variables d'entorn i gestió de credencials. En dominar aquests conceptes, pots crear pipelines robustos, mantenibles i escalables que optimitzin el teu procés de lliurament de programari.
