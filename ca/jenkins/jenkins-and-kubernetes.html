Introducció a Jenkins i Kubernetes

Jenkins i Kubernetes són eines poderoses que, quan es combinen, poden millorar significativament la teva pipeline de CI/CD. Jenkins és un servidor d'automatització de codi obert que ajuda a automatitzar les parts del desenvolupament de programari relacionades amb la construcció, prova i desplegament, facilitant la integració contínua i el lliurament continu. Kubernetes, d'altra banda, és una plataforma d'orquestració de contenidors de codi obert que automatitza el desplegament, l'escalat i la gestió d'aplicacions contenidoritzades.

Per què utilitzar Jenkins amb Kubernetes?

  • Escalabilitat: Kubernetes pot escalar els agents de Jenkins amunt i avall segons la càrrega de treball.
  • Aïllament: Cada treball de Jenkins pot executar-se en el seu propi entorn aïllat.
  • Gestió de recursos: Kubernetes gestiona els recursos de manera eficient, assegurant un ús òptim de la CPU i la memòria.
  • Alta disponibilitat: Kubernetes assegura que Jenkins estigui altament disponible i resilient a fallades.

Configuració de Jenkins a Kubernetes

Prerequisits

  • Coneixement bàsic de Jenkins i Kubernetes.
  • Un clúster de Kubernetes en funcionament.
  • Eina de línia de comandes kubectl instal·lada i configurada.
  • Gestor de paquets Helm instal·lat.

Passos per desplegar Jenkins a Kubernetes

  1. Afegeix el repositori Helm de Jenkins:

    helm repo add jenkins https://charts.jenkins.io
helm repo update

  2. Instal·la Jenkins utilitzant Helm:

    helm install jenkins jenkins/jenkins --namespace jenkins --create-namespace

  3. Accedeix a Jenkins:

    • Obteniu la contrasenya d'administrador de Jenkins: 
      printf $(kubectl get secret --namespace jenkins jenkins -o jsonpath="{.data.jenkins-admin-password}" | base64 --decode); echo
    • Accedeix a Jenkins utilitzant el servei de Kubernetes: 
      kubectl --namespace jenkins port-forward svc/jenkins 8080:8080
    • Obre el teu navegador i ves a http://localhost:8080.

Configuració de Jenkins per utilitzar agents de Kubernetes

Instal·lació del plugin de Kubernetes

  1. Ves a Manage Jenkins > Manage Plugins.
  2. Cerca Kubernetes i instal·la el plugin.

Configuració del núvol de Kubernetes a Jenkins

  1. Ves a Manage Jenkins > Configure System.
  2. Desplaça't cap avall fins a Cloud i fes clic a Add a new cloud > Kubernetes.
  3. Omple els detalls del clúster de Kubernetes:
    • Kubernetes URL: La URL del teu servidor API de Kubernetes.
    • Jenkins URL: La URL de la teva instància de Jenkins.
    • Credencials: Afegeix credencials de Kubernetes si és necessari.
  4. Configura les plantilles de Pod:
    • Defineix les plantilles de pod que Jenkins utilitzarà per crear agents.
    • Especifica les imatges de contenidors i els límits de recursos.

Exemple de configuració de plantilla de Pod

apiVersion: v1
kind: Pod
metadata:
  name: jenkins-agent
spec:
  containers:
  - name: jnlp
    image: jenkins/inbound-agent:latest
    args: ['$(JENKINS_SECRET)', '$(JENKINS_NAME)']
  - name: maven
    image: maven:3.6.3-jdk-8
    command: ['cat']
    tty: true
  - name: docker
    image: docker:19.03.12
    command: ['cat']
    tty: true

Executar treballs de Jenkins a Kubernetes

Creació d'un treball de pipeline simple

  1. Ves a New Item > Pipeline.
  2. En la definició del pipeline, utilitza el següent script: 
    pipeline {
    agent {
        kubernetes {
            yaml """
            apiVersion: v1
            kind: Pod
            spec:
              containers:
              - name: maven
                image: maven:3.6.3-jdk-8
                command:
                - cat
                tty: true
            """
        }
    }
    stages {
        stage('Build') {
            steps {
                container('maven') {
                    sh 'mvn --version'
                }
            }
        }
    }
}

Exemple avançat de pipeline

pipeline {
    agent {
        kubernetes {
            yaml """
            apiVersion: v1
            kind: Pod
            spec:
              containers:
              - name: maven
                image: maven:3.6.3-jdk-8
                command:
                - cat
                tty: true
              - name: docker
                image: docker:19.03.12
                command:
                - cat
                tty: true
            """
        }
    }
    stages {
        stage('Build') {
            steps {
                container('maven') {
                    sh 'mvn clean install'
                }
            }
        }
        stage('Docker Build') {
            steps {
                container('docker') {
                    sh 'docker build -t myapp:latest .'
                }
            }
        }
    }
}

Conclusió

Integrar Jenkins amb Kubernetes proporciona una pipeline de CI/CD robusta, escalable i eficient. Aprofitant les capacitats d'orquestració de Kubernetes, Jenkins pot escalar dinàmicament els seus agents, assegurant una utilització òptima dels recursos i l'aïllament. Aquesta configuració no només millora el rendiment i la fiabilitat dels teus processos de CI/CD, sinó que també simplifica la gestió dels entorns de construcció.
