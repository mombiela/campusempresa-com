Introducció a Jenkins i Maven

Jenkins i Maven són eines potents utilitzades en el cicle de vida del desenvolupament de programari per automatitzar i gestionar el procés de construcció. Jenkins és un servidor d'automatització de codi obert, mentre que Maven és una eina d'automatització de construcció utilitzada principalment per a projectes Java.

Conceptes Clau

  • Jenkins: Un servidor d'automatització de codi obert que ajuda a automatitzar parts del procés de desenvolupament de programari, com ara la construcció, les proves i el desplegament de codi.
  • Maven: Una eina d'automatització de construcció utilitzada per gestionar les dependències del projecte, compilar codi, executar proves i empaquetar el codi en un format desplegable.

Configuració de Jenkins i Maven

Instal·lant Jenkins

  1. Descarregar Jenkins: Visita la pàgina de descàrrega de Jenkins i tria l'instal·lador adequat per al teu sistema operatiu.
  2. Instal·lar Jenkins: Segueix les instruccions d'instal·lació per al teu sistema operatiu.
  3. Iniciar Jenkins: Un cop instal·lat, inicia Jenkins executant la comanda o servei adequat.

Instal·lant Maven

  1. Descarregar Maven: Visita la pàgina de descàrrega de Maven i descarrega l'arxiu zip binari.
  2. Instal·lar Maven: Extreu l'arxiu a un directori de la teva elecció.
  3. Configurar Variables d'Entorn:
    • Afegeix la variable d'entorn MAVEN_HOME apuntant al directori de Maven.
    • Afegeix el directori bin de Maven a la variable d'entorn PATH.

Verificant les Instal·lacions

  • Jenkins: Obre un navegador web i navega a http://localhost:8080. Hauries de veure el tauler de Jenkins.
  • Maven: Obre un terminal i executa mvn -v. Hauries de veure la informació de la versió de Maven.

Integrant Jenkins amb Maven

Configurant Jenkins per Utilitzar Maven

  1. Instal·lar el Plugin de Maven:
    • Ves a Manage Jenkins > Manage Plugins.
    • Cerca el plugin "Maven Integration" i instal·la'l.
  2. Configurar Maven a Jenkins:
    • Ves a Manage Jenkins > Global Tool Configuration.
    • A la secció "Maven", afegeix una nova instal·lació de Maven especificant el nom i el camí al directori home de Maven.

Creant un Projecte Maven a Jenkins

  1. Crear una Nova Tasca:
    • Ves al tauler de Jenkins i fes clic a New Item.
    • Introdueix un nom per a la tasca i selecciona Maven project.
  2. Configurar la Tasca:
    • A la pàgina de configuració de la tasca, especifica els detalls de SCM (Gestió de Codi Font), com ara l'URL del repositori Git.
    • A la secció "Build", especifica els objectius i opcions per a Maven, com ara clean install.

Exemple: Configuració Simple de Projecte Maven

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0-SNAPSHOT</version>
    <dependencies>
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>
  • Explicació:
    • <groupId>: El grup o organització a la qual pertany el projecte.
    • <artifactId>: L'identificador únic per al projecte.
    • <version>: La versió del projecte.
    • <dependencies>: La llista de dependències requerides pel projecte.

Integració Avançada de Jenkins i Maven

Pipelines de Jenkins amb Maven

  • Pipeline Declaratiu: 
    pipeline {
    agent any
    tools {
        maven 'Maven 3.6.3'
    }
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/example/my-app.git'
            }
        }
        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
    }
}
    • Explicació:
      • agent any: Executa el pipeline en qualsevol agent disponible.
      • tools: Especifica la versió de Maven a utilitzar.
      • stages: Defineix les etapes del pipeline, com ara Checkout, Build i Test.

Gestió de Dependències i Plugins

  • Gestió de Dependències: Utilitza la secció <dependencyManagement> al pom.xml per gestionar les versions de les dependències.
  • Plugins: Utilitza la secció <plugins> per configurar plugins de Maven, com ara el plugin Surefire per executar proves.

Exemple: Configuració Avançada de `pom.xml`

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0-SNAPSHOT</version>
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework</groupId>
                <artifactId>spring-core</artifactId>
                <version>5.2.9.RELEASE</version>
            </dependency>
        </dependencies>
    </dependencyManagement>
    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-surefire-plugin</artifactId>
                <version>2.22.2</version>
            </plugin>
        </plugins>
    </build>
</project>
  • Explicació:
    • <dependencyManagement>: Gestiona les versions de les dependències.
    • <build>: Configura plugins de construcció, com ara el plugin Surefire per executar proves.

Conclusió

Integrar Jenkins i Maven proporciona una solució robusta per automatitzar els processos de construcció, prova i desplegament en el desenvolupament de programari. Configurant Jenkins per utilitzar Maven, pots agilitzar el teu flux de treball de desenvolupament, assegurar la consistència i millorar la productivitat. Tant si estàs treballant en un projecte simple com si estàs gestionant dependències i plugins complexos, Jenkins i Maven junts ofereixen una combinació potent per a la integració contínua i el lliurament continu (CI/CD).
