Les Llibreries de Pipeline a Jenkins són una característica poderosa que permet crear codi reutilitzable que es pot compartir entre múltiples pipelines de Jenkins. Això ajuda a mantenir la consistència, reduir la redundància i millorar la gestió dels teus treballs de Jenkins.

Introducció a les Llibreries de Pipeline

  • Definició: Les Llibreries de Pipeline són llibreries compartides que contenen codi reutilitzable per a pipelines de Jenkins.
  • Propòsit: Ajuden a modularitzar i reutilitzar codi a través de diferents pipelines, fent més fàcil gestionar i mantenir els treballs de Jenkins.
  • Tipus: Hi ha dos tipus principals de llibreries:
    • Llibreries Globals: Disponibles per a tots els pipelines en una instància de Jenkins.
    • Llibreries Locals: Específiques per a un pipeline o treball particular.

Configuració de les Llibreries de Pipeline

Llibreries Globals

  1. Configuració:

    • Navega a Manage Jenkins > Configure System.
    • Desplaça't cap avall fins a la secció Global Pipeline Libraries.
    • Afegeix una nova llibreria amb els següents detalls:
      • Nom: Un nom únic per a la llibreria.
      • Versió per Defecte: La versió per defecte a utilitzar si no se'n especifica cap al pipeline.
      • Mètode de Recuperació: Com ha de recuperar Jenkins la llibreria (per exemple, des d'un repositori Git).

  2. Exemple de Configuració:

    library identifier: 'my-shared-library@master'

Llibreries Locals

  1. Configuració:

    • Les llibreries locals es guarden típicament dins del mateix repositori que el Jenkinsfile.
    • Es carreguen utilitzant l'anotació @Library.

  2. Exemple de Configuració:

    @Library('my-local-library') _

Ús de les Llibreries de Pipeline

Ús Bàsic

  • Carregar Llibreries: Utilitza l'anotació @Library per carregar una llibreria al teu Jenkinsfile.
  • Cridar Funcions: Un cop carregada, pots cridar funcions definides a la llibreria.

Exemple

  1. Codi de la Llibreria (guardat al directori vars):

    // vars/helloWorld.groovy
def call() {
    echo 'Hello, World!'
}

  2. Jenkinsfile:

    @Library('my-shared-library') _

pipeline {
    agent any
    stages {
        stage('Example') {
            steps {
                helloWorld()
            }
        }
    }
}

Ús Avançat

Passos i Classes Personalitzades

  • Passos Personalitzats: Defineix passos reutilitzables al directori vars.
  • Classes: Defineix lògica més complexa utilitzant classes Groovy al directori src.

Exemple

  1. Pas Personalitzat:

    // vars/customStep.groovy
def call(String name) {
    echo "Hello, ${name}!"
}

  2. Classe:

    // src/com/example/Utility.groovy
package com.example

class Utility {
    static void printMessage(String message) {
        echo message
    }
}

  3. Jenkinsfile:

    @Library('my-shared-library') _
import com.example.Utility

pipeline {
    agent any
    stages {
        stage('Example') {
            steps {
                customStep('Jenkins')
                script {
                    Utility.printMessage('This is a custom message.')
                }
            }
        }
    }
}

Millors Pràctiques

  • Control de Versions: Controla sempre les versions de les teves llibreries per assegurar la consistència.
  • Documentació: Documenta bé les teves llibreries per fer-les més fàcils d'utilitzar i entendre.
  • Proves: Prova les teves llibreries a fons abans d'utilitzar-les en pipelines de producció.

Conclusió

Les Llibreries de Pipeline a Jenkins proporcionen una manera robusta de gestionar i reutilitzar codi a través de múltiples pipelines. Configurant llibreries globals o locals, pots modularitzar els teus Jenkinsfiles, fent-los més fàcils de mantenir i estendre. Ja sigui que estiguis definint passos reutilitzables simples o classes complexes, les Llibreries de Pipeline poden millorar significativament els teus fluxos de treball de CI/CD. Recorda seguir les millors pràctiques com el control de versions, la documentació i les proves per assegurar que les teves llibreries siguin fiables i fàcils d'utilitzar.
