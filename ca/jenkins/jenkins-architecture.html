Introducció

Jenkins és un servidor d'automatització de codi obert que permet als desenvolupadors construir, provar i desplegar el seu programari. Entendre la seva arquitectura és crucial per utilitzar efectivament Jenkins en la teva pipeline de CI/CD. Aquesta secció cobrirà els components clau i les seves interaccions dins de Jenkins.

Components Clau de l'Arquitectura de Jenkins

1. Jenkins Master

El Jenkins Master és la unitat de control central de Jenkins. Orquestra tot el procés de CI/CD.

  • Responsabilitats:
    • Programar les tasques de construcció.
    • Enviar les construccions als nodes (esclaus) per a la seva execució.
    • Monitoritzar els nodes (esclaus).
    • Registrar i presentar els resultats de les construccions.
    • Gestionar les sol·licituds i configuracions dels usuaris.

2. Nodes de Jenkins (Esclaus)

Els nodes, també coneguts com a esclaus, són màquines que realitzen les tasques de construcció reals.

  • Responsabilitats:
    • Executar les tasques de construcció enviades pel master.
    • Informar dels resultats de la construcció al master.
    • Es poden configurar per executar tipus específics de tasques.

3. Executors

Els executors són les unitats computacionals en un node que executen les tasques de construcció.

  • Punts Clau:
    • Cada node pot tenir un o més executors.
    • El nombre d'executors determina quantes tasques pot executar un node simultàniament.

4. Tasques de Construcció

Les tasques de construcció són les unitats de treball bàsiques en Jenkins.

  • Tipus de Tasques:
    • Projecte Freestyle: Tipus de projecte bàsic per a construccions senzilles.
    • Pipeline: Defineix tot el procés de construcció com a codi.
    • Pipeline Multibranch: Crea automàticament pipelines per a cada branca en un repositori.
    • Carpeta: Organitza les tasques en una estructura jeràrquica.

5. Plugins

Els plugins amplien la funcionalitat de Jenkins.

  • Exemples:
    • Plugin de Git: Integra el control de versions de Git.
    • Plugin de Pipeline: Permet pipeline com a codi.
    • Plugin de Docker: Permet a Jenkins utilitzar contenidors Docker com a entorns de construcció.

Exemple de Configuració

Configuració d'un Jenkins Master i Esclau

Configuració del Jenkins Master

  1. Instal·lar Jenkins:

    • Descarrega i instal·la Jenkins des del lloc web oficial.
    • Inicia Jenkins i completa l'assistent de configuració inicial.

  2. Configurar Nodes:

    • Navega a Manage Jenkins > Manage Nodes and Clouds.
    • Fes clic a New Node i configura els detalls del node.

Configuració de l'Esclau de Jenkins

  1. Llançar l'Agent de l'Esclau:

    • A la màquina esclau, descarrega l'agent.jar des del master de Jenkins.
    • Executa l'agent utilitzant la comanda: 
      java -jar agent.jar -jnlpUrl <JENKINS_MASTER_URL>/computer/<NODE_NAME>/slave-agent.jnlp -secret <SECRET_KEY>

  2. Verificar la Connexió:

    • Assegura't que el node esclau està connectat i en línia al tauler de control de Jenkins.

Resum

Entendre l'arquitectura de Jenkins és fonamental per configurar i gestionar una pipeline de CI/CD robusta. El Jenkins Master orquestra el procés, mentre que els nodes (esclaus) executen les tasques de construcció. Els executors en els nodes executen les tasques, i els plugins amplien les capacitats de Jenkins. Dominant aquests components, pots gestionar i escalar eficientment el teu entorn de Jenkins.

En les properes seccions, aprofundirem en la configuració de Jenkins, la creació de pipelines i la integració de diverses eines per millorar els teus fluxos de treball de CI/CD.
