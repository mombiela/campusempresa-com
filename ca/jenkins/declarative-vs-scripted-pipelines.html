Introducció

Les pipelines de Jenkins són un conjunt de plugins que suporten la implementació i integració de pipelines de lliurament continu a Jenkins. Hi ha dos tipus de pipelines a Jenkins: Declaratives i Escrites. Entendre les diferències entre aquests dos tipus és crucial per crear pipelines CI/CD eficients i mantenibles.

Conceptes Clau

Pipelines Declaratives

  • Estructura: Les pipelines declaratives proporcionen una sintaxi més estructurada i opinada.
  • Facilitat d'ús: Més fàcil per a principiants a causa de la seva sintaxi més simple i llegible.
  • Gestió d'Errors: Gestió d'errors integrada i accions posteriors a la construcció.
  • Validació: Jenkins pot validar la sintaxi de la pipeline abans de l'execució.
  • Etapes i Passos: Etapes i passos clarament definits.

Pipelines Escrites

  • Flexibilitat: Més flexible i potent, permetent scripts complexos.
  • Sintaxi Groovy: Utilitza el llenguatge de script Groovy, que pot ser més desafiant per a principiants.
  • Personalització: Major control sobre l'execució de la pipeline.
  • Llegat: Forma més antiga i tradicional de definir pipelines de Jenkins.

Exemple de Pipeline Declarativa

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Afegiu els passos de construcció aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Afegiu els passos de prova aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Afegiu els passos de desplegament aquí
            }
        }
    }

    post {
        always {
            echo 'Això sempre s'executarà'
        }
        success {
            echo 'Això s'executarà només si la construcció té èxit'
        }
        failure {
            echo 'Això s'executarà només si la construcció falla'
        }
    }
}

Explicació

  • pipeline: L'element arrel que defineix la pipeline.
  • agent: Especifica on s'executa la pipeline o una etapa específica.
  • stages: Conté una seqüència d'etapes a executar.
  • steps: Defineix les tasques reals a realitzar en cada etapa.
  • post: Defineix les accions a prendre després de l'execució de la pipeline.

Exemple de Pipeline Escrita

node {
    try {
        stage('Build') {
            echo 'Building...'
            // Afegiu els passos de construcció aquí
        }
        stage('Test') {
            echo 'Testing...'
            // Afegiu els passos de prova aquí
        }
        stage('Deploy') {
            echo 'Deploying...'
            // Afegiu els passos de desplegament aquí
        }
    } catch (Exception e) {
        echo 'S'ha produït un error: ' + e.toString()
    } finally {
        echo 'Això sempre s'executarà'
    }
}

Explicació

  • node: L'element arrel que defineix la pipeline escrita.
  • try-catch-finally: Utilitzat per a la gestió d'errors.
  • stage: Defineix una etapa en la pipeline.
  • echo: Imprimeix un missatge a la consola.

Taula Comparativa

| Característica | Pipeline Declarativa | Pipeline Escrita | |-------------------------|-------------------------------------------|------------------------------------------| | Sintaxi | Més estructurada i opinada | Flexible i potent | | Facilitat d'ús | Més fàcil per a principiants | Requereix coneixements de Groovy | | Gestió d'Errors | Gestió d'errors integrada | Gestió d'errors personalitzada amb try-catch | | Validació | Jenkins pot validar abans de l'execució | No hi ha validació prèvia a l'execució | | Personalització | Personalització limitada | Altament personalitzable | | Corba d'aprenentatge | Més baixa | Més alta |

Conclusió

Tant les pipelines Declaratives com les Escrites tenen els seus propis avantatges i casos d'ús. Les pipelines declaratives són ideals per a principiants i per a pipelines més simples i directes a causa de la seva llegibilitat i característiques integrades. Les pipelines escrites, en canvi, ofereixen una major flexibilitat i control, fent-les adequades per a fluxos de treball CI/CD més complexos i personalitzats. Entendre els punts forts i les limitacions de cada tipus t'ajudarà a triar l'enfocament adequat per a les teves pipelines de Jenkins.
