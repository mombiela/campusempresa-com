Introducció

Els pipelines de Jenkins són una manera poderosa de definir els teus processos de construcció, prova i desplegament. Aquesta secció cobrirà les millors pràctiques per assegurar que els teus pipelines siguin eficients, mantenibles i escalables.

Conceptes Clau

  • Pipelines Declaratius vs. Scriptats: Entén les diferències i quan utilitzar cadascun.
  • Pipeline com a Codi: Emmagatzema les teves definicions de pipeline en el control de versions.
  • Pipelines Modulars: Divideix pipelines complexos en components més petits i reutilitzables.
  • Gestió d'Errors: Implementa una gestió d'errors robusta per gestionar fallades de manera elegant.
  • Seguretat: Assegura els teus pipelines per protegir informació sensible.

Pipelines Declaratius vs. Scriptats

Pipelines Declaratius

  • Sintaxi més Simple: Més fàcil d'entendre per a principiants.
  • Estructurat: Enfoca un enfocament més estructurat.
  • Exemple: 
    pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}

Pipelines Scriptats

  • Flexibilitat: Més control sobre el pipeline.
  • Complexitat: Pot ser més difícil de llegir i mantenir.
  • Exemple: 
    node {
    stage('Build') {
        echo 'Building...'
    }
    stage('Test') {
        echo 'Testing...'
    }
    stage('Deploy') {
        echo 'Deploying...'
    }
}

Pipeline com a Codi

  • Control de Versions: Emmagatzema el teu Jenkinsfile en el teu repositori.
  • Col·laboració: Permet la col·laboració en equip i revisions de codi.
  • Exemple: 
    pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/your-repo.git'
            }
        }
        // Altres etapes...
    }
}

Pipelines Modulars

  • Biblioteques Compartides: Utilitza biblioteques compartides per reutilitzar codi comú de pipeline.
  • Exemple: 
    @Library('my-shared-library') _
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                buildStep()
            }
        }
        // Altres etapes...
    }
}

Gestió d'Errors

  • Blocs Try-Catch: Utilitza blocs try-catch per gestionar errors.
  • Accions Posteriors: Utilitza seccions post per definir accions en cas d'èxit, fallada o sempre.
  • Exemple: 
    pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                script {
                    try {
                        // Passos de construcció
                    } catch (Exception e) {
                        echo "La construcció ha fallat: ${e.message}"
                    }
                }
            }
        }
    }
    post {
        always {
            echo 'Netejant...'
        }
        success {
            echo 'Construcció exitosa!'
        }
        failure {
            echo 'La construcció ha fallat!'
        }
    }
}

Seguretat

  • Gestió de Credencials: Utilitza les credencials de Jenkins per gestionar informació sensible.
  • Exemple: 
    pipeline {
    agent any
    environment {
        SECRET = credentials('my-secret')
    }
    stages {
        stage('Build') {
            steps {
                echo "Utilitzant secret: ${SECRET}"
            }
        }
    }
}

Conclusió

Seguint aquestes millors pràctiques, pots crear pipelines de Jenkins que siguin eficients, mantenibles i segurs. Recorda triar el tipus de pipeline adequat per a les teves necessitats, modularitzar el teu codi, gestionar errors de manera elegant i sempre prioritzar la seguretat.
