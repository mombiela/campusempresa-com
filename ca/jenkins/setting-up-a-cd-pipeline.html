Introducció

El lliurament continu (CD) és una pràctica de desenvolupament de programari on els canvis de codi es preparen automàticament per a una versió a producció. Jenkins és una eina popular per implementar pipelines de CD. Aquesta secció et guiarà a través de la configuració d'un pipeline de CD utilitzant Jenkins, des de nivells bàsics fins a avançats.

Prerequisits

Abans de configurar un pipeline de CD, assegura't de tenir el següent:

  • Jenkins instal·lat i en funcionament.
  • Comprensió bàsica dels treballs i pipelines de Jenkins.
  • Repositori de codi font (per exemple, GitHub, GitLab).
  • Aplicació per desplegar (per exemple, una aplicació web senzilla).

Conceptes Bàsics

  • Lliurament Continu (CD): La pràctica de desplegar automàticament els canvis de codi a un entorn de staging o producció.
  • Pipeline: Una sèrie de passos automatitzats per construir, provar i desplegar codi.
  • Jenkinsfile: Un fitxer de text que conté la definició d'un Pipeline de Jenkins i es registra en el control de versions.

Configuració d'un Pipeline de CD Bàsic

Pas 1: Crear un Treball de Pipeline a Jenkins

  1. Obre Jenkins i navega a Nou Element.
  2. Introdueix un nom per al teu pipeline i selecciona Pipeline, després fes clic a D'acord.
  3. En la configuració del pipeline, desplaça't cap avall fins a la secció Pipeline.
  4. Selecciona Pipeline script from SCM.
  5. Tria el teu SCM (per exemple, Git) i proporciona l'URL del repositori.
  6. Especifica la ruta del script (per exemple, Jenkinsfile).

Pas 2: Escriure un Jenkinsfile Bàsic

Crea un Jenkinsfile en el teu repositori amb el següent contingut:

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Afegeix passos de construcció aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Afegeix passos de prova aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Afegeix passos de desplegament aquí
            }
        }
    }
}

Pas 3: Desencadenar el Pipeline

  1. Desa el Jenkinsfile en el teu repositori.
  2. Torna a Jenkins i desencadena el pipeline fent clic a Build Now.

Conceptes Intermedis

Afegir Notificacions

Per rebre notificacions sobre l'estat del pipeline, pots utilitzar plugins com Email Extension o Slack Notification.

Exemple: Notificació per Correu Electrònic

Afegeix els següents passos al Jenkinsfile:

post {
    success {
        mail to: 'team@example.com',
             subject: "Pipeline Success: ${currentBuild.fullDisplayName}",
             body: "Good news! The pipeline has succeeded."
    }
    failure {
        mail to: 'team@example.com',
             subject: "Pipeline Failure: ${currentBuild.fullDisplayName}",
             body: "Unfortunately, the pipeline has failed. Please check the logs."
    }
}

Etapes Paral·leles

Per accelerar el pipeline, pots executar etapes en paral·lel.

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        echo 'Running unit tests...'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        echo 'Running integration tests...'
                    }
                }
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}

Conceptes Avançats

Utilitzar Docker en Pipelines

Pots utilitzar contenidors Docker per proporcionar un entorn consistent per a les teves construccions.

pipeline {
    agent {
        docker {
            image 'maven:3.6.3-jdk-8'
            args '-v /root/.m2:/root/.m2'
        }
    }

    stages {
        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}

Desplegar a Kubernetes

Per a desplegaments avançats, pots desplegar la teva aplicació a un clúster de Kubernetes.

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            steps {
                withKubeConfig([credentialsId: 'kubeconfig-credentials']) {
                    sh 'kubectl apply -f deployment.yaml'
                }
            }
        }
    }
}

Conclusió

Configurar un pipeline de CD a Jenkins implica crear un treball de pipeline, escriure un Jenkinsfile, i configurar etapes per construir, provar i desplegar. A mesura que avancis, pots afegir notificacions, executar etapes en paral·lel, utilitzar Docker i desplegar a Kubernetes. El lliurament continu amb Jenkins ajuda a automatitzar el procés de desplegament, assegurant versions més ràpides i fiables.
