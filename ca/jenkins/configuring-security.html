Introducció

Configurar la seguretat a Jenkins és crucial per protegir les teves pipelines CI/CD, dades sensibles i infraestructura. Aquesta secció et guiarà a través dels conceptes bàsics de la seguretat de Jenkins, des de la configuració de l'autenticació d'usuaris fins a configuracions de seguretat avançades.

Autenticació d'Usuaris

L'autenticació d'usuaris és la primera línia de defensa per assegurar la teva instància de Jenkins. Garanteix que només els usuaris autoritzats puguin accedir i realitzar accions a Jenkins.

Configurant la Seguretat de Jenkins

  1. Habilitar la Seguretat:

    • Navega a Manage Jenkins > Configure Global Security.
    • Marca l'opció Enable security.

  2. Triar el Regne de Seguretat:

    • Base de dades d'usuaris pròpia de Jenkins: Simple i adequat per a equips petits.
    • LDAP: Integra amb el servidor LDAP de la teva organització.
    • Active Directory: Utilitza Active Directory per a l'autenticació.
    • SAML: Utilitza SAML per a Single Sign-On (SSO).

  3. Configurar l'Autorització:

    • Seguretat basada en matriu: Control granular sobre els permisos.
    • Estratègia d'Autorització basada en Matriu de Projectes: Controla els permisos a nivell de projecte.
    • Estratègia basada en rols: Defineix rols i assigna permisos als rols.

Exemple: Configurant la Base de Dades d'Usuaris Pròpia de Jenkins

# Navega a Manage Jenkins > Configure Global Security
security:
  enable: true
  realm: jenkins
  authorization: matrix
  users:
    - username: admin
      password: admin_password
      roles: [admin]
    - username: developer
      password: dev_password
      roles: [developer]

Assegurant Jenkins amb SSL

Assegurar Jenkins amb SSL garanteix que les dades transmeses entre el servidor Jenkins i els usuaris estiguin xifrades.

Passos per Habilitar SSL

  1. Generar un Keystore:

    keytool -genkey -alias jenkins -keyalg RSA -keystore jenkins.keystore

  2. Configurar Jenkins per Utilitzar SSL:

    • Edita el fitxer de configuració de Jenkins (per exemple, jenkins.xml o jenkins.service).
    • Afegeix els següents arguments: 
      <arguments>--httpPort=-1 --httpsPort=8443 --httpsKeyStore=/path/to/jenkins.keystore --httpsKeyStorePassword=your_password</arguments>

  3. Reiniciar Jenkins:

    sudo systemctl restart jenkins

Control d'Accés Basat en Rols (RBAC)

RBAC et permet definir rols amb permisos específics i assignar aquests rols a usuaris o grups.

Configurant RBAC

  1. Instal·lar el Plugin de Estratègia de Rols:

    • Navega a Manage Jenkins > Manage Plugins.
    • Instal·la el Role Strategy Plugin.

  2. Configurar Rols:

    • Navega a Manage Jenkins > Manage and Assign Roles > Manage Roles.
    • Defineix rols i assigna permisos.

  3. Assignar Rols a Usuaris:

    • Navega a Manage Jenkins > Manage and Assign Roles > Assign Roles.
    • Assigna rols a usuaris o grups.

Exemple: Definint i Assignant Rols

roles:
  - name: admin
    permissions: [Overall/Administer, Job/Configure, Job/Delete]
  - name: developer
    permissions: [Job/Read, Job/Build, Job/Cancel]

assignments:
  - user: admin
    roles: [admin]
  - user: developer
    roles: [developer]

Millors Pràctiques de Seguretat

  • Actualitza Regularment Jenkins: Mantingues Jenkins i els plugins actualitzats per mitigar vulnerabilitats.
  • Utilitza Contrasenyes Fortes: Enforça polítiques de contrasenyes fortes.
  • Limita l'Ús de Plugins: Instal·la només els plugins necessaris per reduir la superfície d'atac.
  • Fes Còpies de Seguretat Regularment: Fes còpies de seguretat regulars de les configuracions i dades de Jenkins.
  • Audita els Registres: Revisa regularment els registres d'auditoria per detectar activitats sospitoses.

Conclusió

Configurar la seguretat a Jenkins és essencial per protegir les teves pipelines CI/CD i dades sensibles. Configurant l'autenticació d'usuaris, habilitant SSL, implementant RBAC i seguint les millors pràctiques de seguretat, pots millorar significativament la seguretat de la teva instància de Jenkins. Revisa i actualitza regularment les teves configuracions de seguretat per adaptar-te a noves amenaces i vulnerabilitats.
