Visió general

Les Pipelines de Jenkins són una manera poderosa de definir i automatitzar els teus processos de construcció, prova i desplegament. Aquesta secció t'introduirà als conceptes bàsics de les Pipelines de Jenkins, incloent la seva estructura, components i com crear-les i gestionar-les.

Què és una Pipeline de Jenkins?

Una Pipeline de Jenkins és un conjunt de plugins que suporta la implementació i integració de pipelines de lliurament continu a Jenkins. Et permet definir tot el teu procés de construcció, que típicament inclou etapes per construir, provar i desplegar la teva aplicació.

Conceptes clau

  • Pipeline: Un model definit per l'usuari d'una pipeline de lliurament continu.
  • Node: Una màquina on s'executa Jenkins.
  • Stage: Un bloc que conté una sèrie de passos.
  • Step: Una tasca única que forma part del procés, com executar un script o fer el checkout del codi d'un repositori.

Tipus de Pipelines de Jenkins

Hi ha dos tipus principals de Pipelines de Jenkins:

  1. Pipeline Declarativa: Una sintaxi més estructurada i senzilla.
  2. Pipeline Escrita: Una sintaxi més flexible i complexa.

Pipeline Declarativa

La sintaxi de la Pipeline Declarativa proporciona una manera més simplificada i opinada de definir la teva pipeline. Utilitza una estructura predefinida que fa que sigui més fàcil de llegir i escriure.

Exemple

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}
  • pipeline: L'element arrel que defineix la pipeline.
  • agent: Especifica on s'executarà la pipeline o una etapa específica.
  • stages: Conté una seqüència d'una o més directives d'etapa.
  • stage: Defineix una etapa de la pipeline.
  • steps: Conté una seqüència d'una o més directives de pas.

Pipeline Escrita

La sintaxi de la Pipeline Escrita es basa en Groovy i proporciona més flexibilitat i control sobre la pipeline.

Exemple

node {
    stage('Build') {
        echo 'Building...'
    }
    stage('Test') {
        echo 'Testing...'
    }
    stage('Deploy') {
        echo 'Deploying...'
    }
}
  • node: Defineix un bloc de node, que és una part fonamental de les Pipelines Escrites.
  • stage: Defineix una etapa de la pipeline.
  • echo: Un pas senzill que imprimeix un missatge a la consola.

Creant la teva primera Pipeline

Per crear la teva primera Pipeline de Jenkins, segueix aquests passos:

  1. Obre Jenkins: Navega a la teva instància de Jenkins.
  2. Crea una nova feina: Fes clic a "New Item" i selecciona "Pipeline".
  3. Configura la Pipeline:
    • Introdueix un nom per a la teva pipeline.
    • A la secció "Pipeline", tria entre "Pipeline script" o "Pipeline script from SCM".
    • Si tries "Pipeline script", pots introduir directament el codi de la teva pipeline.
    • Si tries "Pipeline script from SCM", has d'especificar el repositori de control de versions on es troba el script de la teva pipeline.

Exemple de configuració

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}

Millors pràctiques

  • Mantén-ho senzill: Comença amb una pipeline senzilla i afegeix complexitat gradualment.
  • Utilitza sintaxi declarativa: Prefereix les Pipelines Declaratives per a la majoria dels casos d'ús degut a la seva simplicitat i llegibilitat.
  • Control de versions: Emmagatzema els teus scripts de pipeline en control de versions per fer un seguiment dels canvis i col·laborar amb el teu equip.
  • Modularitza: Divideix la teva pipeline en etapes i passos reutilitzables.

Conclusió

Les Pipelines de Jenkins proporcionen un marc robust per automatitzar els teus processos de construcció, prova i desplegament. Comprenent els conceptes bàsics de les Pipelines Declaratives i Escrites, pots començar a crear pipelines eficients i fàcils de mantenir. Recorda seguir les millors pràctiques per assegurar que les teves pipelines siguin senzilles, llegibles i controlades per versions.
