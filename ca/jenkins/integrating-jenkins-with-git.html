Introducció

Integrar Jenkins amb Git és un pas crucial en la configuració d'una pipeline d'integració contínua/desplegament continu (CI/CD). Aquesta secció et guiarà a través del procés de connectar Jenkins amb un repositori Git, permetent construccions i desplegaments automatitzats.

Prerequisits

Abans de començar, assegura't de tenir el següent:

  • Jenkins instal·lat i en funcionament.
  • Un repositori Git (GitHub, GitLab, Bitbucket, etc.).
  • Coneixements bàsics de Jenkins i Git.

Configurar Jenkins amb Git

Instal·lar el Plugin de Git a Jenkins

Per integrar Jenkins amb Git, necessites instal·lar el plugin de Git.

  1. Obre Jenkins i navega a Manage Jenkins.
  2. Fes clic a Manage Plugins.
  3. Ves a la pestanya Available i cerca Git Plugin.
  4. Selecciona el plugin i fes clic a Install without restart.

Configurar Git a Jenkins

Després d'instal·lar el plugin de Git, has de configurar Jenkins per utilitzar Git.

  1. Ves a Manage Jenkins > Global Tool Configuration.
  2. Desplaça't cap avall fins a Git i fes clic a Add Git.
  3. Proporciona un nom (per exemple, Default Git) i especifica el camí a l'executable de Git si és necessari.
  4. Fes clic a Save.

Crear una Feina de Jenkins per a un Projecte Git

Ara, creem una feina de Jenkins que extregui codi d'un repositori Git i el construeixi.

  1. Des del tauler de control de Jenkins, fes clic a New Item.
  2. Introdueix un nom per a la teva feina i selecciona Freestyle project.
  3. Fes clic a OK.

Configurar la Gestió del Codi Font

  1. A la pàgina de configuració de la feina, desplaça't cap avall fins a Source Code Management.
  2. Selecciona Git.
  3. Introdueix l'URL del repositori (per exemple, https://github.com/username/repository.git).
  4. Si el repositori és privat, proporciona les credencials necessàries fent clic a Add i introduint les teves credencials de Git.
  5. Especifica la branca a construir (per exemple, main).

Afegir Triggers de Construcció

Per automatitzar el procés de construcció, pots configurar triggers de construcció.

  1. Desplaça't cap avall fins a Build Triggers.
  2. Selecciona Poll SCM i introdueix un horari (per exemple, H/5 * * * * per fer polling cada 5 minuts).
  3. Alternativament, pots utilitzar GitHub hook trigger for GITScm polling si has configurat webhooks al teu repositori de GitHub.

Definir Passos de Construcció

  1. Desplaça't cap avall fins a Build.
  2. Fes clic a Add build step i selecciona Execute shell.
  3. Introdueix les comandes de construcció (per exemple, ./build.sh).

Accions Post-construcció

  1. Desplaça't cap avall fins a Post-build Actions.
  2. Afegeix qualsevol pas post-construcció necessari, com ara arxivar artefactes o enviar notificacions.

Configuració Exemple

Aquí tens un exemple d'un script shell senzill que es podria utilitzar en el pas de construcció:

#!/bin/bash
echo "Construint el projecte..."
# Comanda de construcció exemple
make
echo "Construcció completada amb èxit."

Conclusió

Integrar Jenkins amb Git és un pas fonamental per automatitzar el teu flux de treball de desenvolupament. Seguint els passos descrits anteriorment, pots configurar Jenkins per extreure codi d'un repositori Git, construir-lo i activar construccions automàticament basades en canvis al repositori. Aquesta integració no només estalvia temps, sinó que també assegura que el teu codi estigui sempre en un estat desplegable.

A les següents seccions, explorarem temes més avançats com configurar pipelines, gestionar processos de construcció complexos i integrar amb altres eines i serveis.
