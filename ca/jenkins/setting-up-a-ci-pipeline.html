Introducció a la Integració Contínua (CI)

  • Definició: La Integració Contínua (CI) és una pràctica de desenvolupament on els desenvolupadors integren codi en un repositori compartit freqüentment, idealment diverses vegades al dia.
  • Beneficis:
    • Detecció primerenca d'errors.
    • Millora de la qualitat del programari.
    • Lliurament més ràpid d'actualitzacions.

Introducció a Jenkins

  • Què és Jenkins?: Jenkins és un servidor d'automatització de codi obert que ajuda a automatitzar parts del procés de desenvolupament de programari, incloent la construcció, prova i desplegament de codi.
  • Característiques Clau:
    • Extensible amb plugins.
    • Suporta construccions distribuïdes.
    • Fàcil d'instal·lar i configurar.

Configuració de Jenkins

  • Instal·lació:
    • Windows: Descarrega l'instal·lador de Jenkins des del lloc web oficial i segueix l'assistent de configuració.
    • Linux: Utilitza gestors de paquets com apt o yum per instal·lar Jenkins.
    • Docker: Executa Jenkins en un contenidor Docker utilitzant la comanda: 
      docker run -p 8080:8080 -p 50000:50000 jenkins/jenkins:lts
  • Configuració Inicial:
    • Accedeix a Jenkins via http://localhost:8080.
    • Desbloqueja Jenkins utilitzant la contrasenya inicial d'administrador.
    • Instal·la els plugins suggerits.
    • Crea el primer usuari administrador.

Creació del teu Primer Job a Jenkins

  • Projecte Freestyle:
    • Navega a New Item > Freestyle project.
    • Introdueix un nom per al job i fes clic a OK.
    • Configura el job:
      • Gestió del Codi Font: Connecta't a un repositori Git.
      • Triggers de Construcció: Configura triggers com Poll SCM o Build periodically.
      • Passos de Construcció: Afegeix passos de construcció com Execute shell o Invoke Ant.
      • Accions Post-construcció: Configura accions com Archive the artifacts o Email notifications.

Exemple: Construcció amb Script de Shell Simple

echo "Building the project..."
# Simulate build process
sleep 5
echo "Build completed!"

Configuració d'una Pipeline de CI amb Jenkins

  • Pipeline com a Codi: Utilitza Jenkins Pipeline (Jenkinsfile) per definir la teva pipeline de CI com a codi.
  • Creació d'un Job de Pipeline:
    • Navega a New Item > Pipeline.
    • Introdueix un nom per al job i fes clic a OK.
    • Configura la pipeline:
      • Definició de la Pipeline: Tria Pipeline script from SCM si el teu Jenkinsfile està en un repositori, o Pipeline script per escriure'l directament.

Exemple de Jenkinsfile

pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/your-repo/your-project.git'
            }
        }
        stage('Build') {
            steps {
                sh './build.sh'
            }
        }
        stage('Test') {
            steps {
                sh './test.sh'
            }
        }
        stage('Deploy') {
            steps {
                sh './deploy.sh'
            }
        }
    }
    post {
        success {
            echo 'Pipeline completed successfully!'
        }
        failure {
            echo 'Pipeline failed!'
        }
    }
}

Conceptes Avançats de Pipeline de CI

  • Etapes Paral·leles: Executa múltiples etapes en paral·lel per accelerar la pipeline.
  • Execució Condicional: Utilitza condicions per executar etapes basades en certs criteris.
  • Biblioteques Compartides: Reutilitza codi comú de pipeline a través de múltiples Jenkinsfiles.

Exemple: Etapes Paral·leles

pipeline {
    agent any
    stages {
        stage('Parallel Stage') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        sh './run-unit-tests.sh'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        sh './run-integration-tests.sh'
                    }
                }
            }
        }
    }
}

Conclusió

  • Resum:
    • Jenkins és una eina poderosa per configurar pipelines de CI.
    • Comença amb configuracions bàsiques de job i gradualment passa a pipelines més complexes.
    • Utilitza Jenkinsfile per definir la teva pipeline de CI com a codi per a un millor control de versions i reutilització.
  • Pròxims Passos:
    • Explora els plugins de Jenkins per ampliar la funcionalitat.
    • Integra Jenkins amb altres eines com Docker, Kubernetes i serveis al núvol.
    • Millora contínuament la teva pipeline de CI per adaptar-se a les necessitats del teu projecte.
