Introducció a Jenkins i Serveis al Núvol

En aquesta secció, explorarem com Jenkins s'integra amb diversos serveis al núvol per millorar els fluxos de treball de la Integració Contínua i el Desplegament Continu (CI/CD). Els serveis al núvol proporcionen recursos escalables i flexibles que es poden aprofitar per millorar l'eficiència i la fiabilitat dels teus fluxos de treball de Jenkins.

Conceptes Clau

  • Serveis al Núvol: Plataformes que proporcionen diversos serveis de computació a través d'internet, com AWS, Azure i Google Cloud.
  • Fluxos de Treball CI/CD: Fluxos de treball automatitzats que integren canvis de codi, executen proves i despleguen aplicacions.
  • Plugins de Jenkins: Extensions que afegeixen funcionalitat a Jenkins, incloent integracions amb serveis al núvol.

Configuració de Jenkins amb Serveis al Núvol

Prerequisits

  • Coneixement bàsic de Jenkins i fluxos de treball CI/CD.
  • Un compte amb un proveïdor de serveis al núvol (per exemple, AWS, Azure, Google Cloud).

Instal·lació de Jenkins en un Servei al Núvol

  1. AWS EC2:

    • Llança una instància EC2.
    • SSH a la instància i instal·la Jenkins.
    • Configura grups de seguretat per permetre l'accés a Jenkins.
    # Actualitza l'índex de paquets
sudo apt-get update

# Instal·la Java
sudo apt-get install openjdk-11-jdk -y

# Afegeix el repositori i la clau de Jenkins
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'

# Instal·la Jenkins
sudo apt-get update
sudo apt-get install jenkins -y

# Inicia Jenkins
sudo systemctl start jenkins
sudo systemctl enable jenkins

  2. Azure VM:

    • Crea una Màquina Virtual a Azure.
    • SSH a la VM i segueix passos similars per instal·lar Jenkins com a AWS.

  3. Google Cloud Compute Engine:

    • Crea una instància de Compute Engine.
    • SSH a la instància i instal·la Jenkins.

Configuració de Jenkins amb Plugins de Núvol

  • Plugin AWS: Permet que Jenkins interactuï amb serveis d'AWS.
  • Plugin Azure: Integra Jenkins amb serveis d'Azure.
  • Plugin Google Cloud: Connecta Jenkins amb serveis de Google Cloud.

Exemple: Configuració del Plugin AWS

  1. Instal·la el plugin AWS des del Gestor de Plugins de Jenkins.

  2. Configura les credencials d'AWS a Jenkins:

    • Ves a Manage Jenkins > Manage Credentials.
    • Afegeix credencials d'AWS (Access Key ID i Secret Access Key).

  3. Utilitza el plugin AWS en un pipeline de Jenkins:

    pipeline {
    agent any
    environment {
        AWS_ACCESS_KEY_ID = credentials('aws-access-key-id')
        AWS_SECRET_ACCESS_KEY = credentials('aws-secret-access-key')
    }
    stages {
        stage('Deploy to S3') {
            steps {
                script {
                    sh '''
                    aws s3 cp my-app.zip s3://my-bucket/my-app.zip
                    '''
                }
            }
        }
    }
}

Integracions Avançades de Jenkins i el Núvol

Ús de Jenkins amb Kubernetes

  • Kubernetes: Una plataforma de codi obert per automatitzar el desplegament, l'escalat i les operacions de contenidors d'aplicacions.
  • Plugin Kubernetes de Jenkins: Permet que Jenkins provisioni dinàmicament pods de Kubernetes per executar treballs.

Exemple: Pipeline de Jenkins amb Kubernetes

  1. Instal·la el plugin Kubernetes a Jenkins.

  2. Configura els detalls del clúster de Kubernetes a Jenkins.

  3. Crea un pipeline de Jenkins que utilitzi pods de Kubernetes:

    pipeline {
    agent {
        kubernetes {
            yaml """
            apiVersion: v1
            kind: Pod
            spec:
              containers:
              - name: jnlp
                image: jenkins/inbound-agent
                args: ['\$(JENKINS_SECRET)', '\$(JENKINS_NAME)']
              - name: maven
                image: maven:3.6.3-jdk-11
                command:
                - cat
                tty: true
            """
        }
    }
    stages {
        stage('Build') {
            steps {
                container('maven') {
                    sh 'mvn clean install'
                }
            }
        }
    }
}

Jenkins i Arquitectures Serverless

  • Serverless: Un model d'execució de computació en el núvol on el proveïdor del núvol gestiona dinàmicament l'assignació de recursos de màquina.
  • AWS Lambda: Un servei de computació serverless que executa codi en resposta a esdeveniments.

Exemple: Activació d'AWS Lambda des de Jenkins

  1. Instal·la el plugin AWS Lambda a Jenkins.

  2. Configura les credencials d'AWS i els detalls de la funció Lambda.

  3. Crea un pipeline de Jenkins per invocar una funció Lambda:

    pipeline {
    agent any
    environment {
        AWS_ACCESS_KEY_ID = credentials('aws-access-key-id')
        AWS_SECRET_ACCESS_KEY = credentials('aws-secret-access-key')
    }
    stages {
        stage('Invoke Lambda') {
            steps {
                script {
                    sh '''
                    aws lambda invoke --function-name myLambdaFunction --payload '{"key": "value"}' response.json
                    '''
                }
            }
        }
    }
}

Conclusió

Integrar Jenkins amb serveis al núvol pot millorar significativament els teus fluxos de treball CI/CD aprofitant l'escalabilitat, la flexibilitat i les diverses eines proporcionades per les plataformes de núvol. Tant si estàs desplegant aplicacions a AWS, gestionant recursos a Azure, o executant contenidors a Kubernetes, Jenkins ofereix una àmplia gamma de plugins i integracions per optimitzar els teus fluxos de treball. Comprenent i utilitzant aquestes integracions, pots construir fluxos de treball CI/CD més eficients, fiables i escalables.
