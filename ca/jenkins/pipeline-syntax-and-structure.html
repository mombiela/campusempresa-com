Introducció a les Pipelines de Jenkins

Les Pipelines de Jenkins són un conjunt de plugins que suporten la implementació i integració de pipelines de lliurament continu a Jenkins. Proporcionen una manera de definir tot el procés de construcció, que inclou construir, provar i desplegar aplicacions.

Conceptes Clau de les Pipelines de Jenkins

  • Pipeline: Un model definit per l'usuari d'una pipeline de lliurament continu.
  • Node: Una màquina on Jenkins executa les tasques.
  • Stage: Un bloc que conté una sèrie de passos.
  • Step: Una tasca única que indica a Jenkins què fer en un moment determinat.

Tipus de Pipelines de Jenkins

  • Pipeline Declarativa: Una sintaxi més estructurada i senzilla per definir pipelines.
  • Pipeline Escrita: Una sintaxi més flexible i potent basada en Groovy.

Sintaxi de la Pipeline Declarativa

Les Pipelines Declaratives proporcionen una sintaxi més directa i opinada per definir la teva pipeline.

Estructura Bàsica

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
            }
        }
    }
}
  • pipeline: L'element arrel que defineix la pipeline.
  • agent: Defineix on s'executarà la pipeline o una etapa específica.
  • stages: Conté una seqüència d'una o més directives d'etapa.
  • stage: Defineix un subconjunt conceptualment distint de tasques realitzades a través de la pipeline.
  • steps: Conté una sèrie d'un o més passos a executar.

Exemple: Pipeline Declarativa amb Condicions

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
            }
        }
        stage('Test') {
            when {
                branch 'master'
            }
            steps {
                echo 'Testing...'
            }
        }
        stage('Deploy') {
            when {
                branch 'master'
            }
            steps {
                echo 'Deploying...'
            }
        }
    }
}
  • when: Permet l'execució d'etapes basades en condicions.

Sintaxi de la Pipeline Escrita

Les Pipelines Escrites proporcionen més flexibilitat i es basen en Groovy.

Estructura Bàsica

node {
    stage('Build') {
        echo 'Building...'
    }
    stage('Test') {
        echo 'Testing...'
    }
    stage('Deploy') {
        echo 'Deploying...'
    }
}
  • node: Una màquina on Jenkins executa la pipeline.
  • stage: Defineix un subconjunt conceptualment distint de tasques realitzades a través de la pipeline.
  • echo: Imprimeix un missatge a la sortida de la consola.

Exemple: Pipeline Escrita amb Etapes Paral·leles

node {
    stage('Build') {
        echo 'Building...'
    }
    stage('Test') {
        parallel {
            stage('Unit Tests') {
                steps {
                    echo 'Running unit tests...'
                }
            }
            stage('Integration Tests') {
                steps {
                    echo 'Running integration tests...'
                }
            }
        }
    }
    stage('Deploy') {
        echo 'Deploying...'
    }
}
  • parallel: Permet l'execució d'etapes en paral·lel.

Comparació de Pipelines Declaratives i Escrites

| Característica | Pipeline Declarativa | Pipeline Escrita | |------------------------|-------------------------------------------|------------------------------------------| | Sintaxi | Més estructurada i senzilla | Més flexible i potent | | Corba d'Aprenentatge | Més fàcil per a principiants | Corba d'aprenentatge més pronunciada | | Gestió d'Errors | Gestió d'errors integrada | Requereix gestió d'errors manual | | Flexibilitat | Limitada a l'estructura predefinida | Altament flexible amb scripting Groovy |

Conclusió

Entendre la sintaxi i l'estructura de les Pipelines de Jenkins és crucial per automatitzar els teus processos de CI/CD. Les Pipelines Declaratives ofereixen una manera més senzilla i estructurada de definir les teves pipelines, fent-les ideals per a principiants. Les Pipelines Escrites, d'altra banda, proporcionen una major flexibilitat i potència, adequades per a escenaris més complexos. En dominar ambdós tipus, pots aprofitar tot el potencial de Jenkins per optimitzar el teu flux de treball de desenvolupament.
