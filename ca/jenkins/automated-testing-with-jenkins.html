Introducció a les proves automatitzades

Les proves automatitzades són una part crucial del desenvolupament de programari modern. Ajuda a assegurar que els canvis en el codi no trenquin la funcionalitat existent i que les noves característiques funcionin com s'espera. Jenkins, un servidor d'automatització de codi obert popular, s'utilitza àmpliament per automatitzar diverses etapes del cicle de vida del desenvolupament de programari, incloent-hi les proves.

Configuració de Jenkins per a proves automatitzades

Abans d'entrar en les proves automatitzades, cal configurar Jenkins. Això implica:

  • Instal·lar Jenkins: Descarregar i instal·lar Jenkins des del lloc web oficial.
  • Configurar Jenkins: Configurar la configuració inicial, incloent-hi els paràmetres de seguretat i els connectors.
  • Crear una feina de Jenkins: Definir una feina que executarà les teves proves.

Instal·lar Jenkins

Per instal·lar Jenkins, segueix aquests passos:

  1. Descarregar Jenkins: Visita la pàgina de descàrrega de Jenkins i tria la versió adequada per al teu sistema operatiu.
  2. Instal·lar Jenkins: Segueix les instruccions d'instal·lació per al teu sistema operatiu.
  3. Iniciar Jenkins: Un cop instal·lat, inicia Jenkins i accedeix a la interfície web a http://localhost:8080.

Configurar Jenkins

Després de la instal·lació, configura Jenkins:

  1. Configuració inicial: Completa l'assistent de configuració inicial.
  2. Instal·lar connectors: Instal·la els connectors necessaris com Git, Maven, i qualsevol connector específic per a proves com JUnit o Selenium.
  3. Crear usuaris: Configura comptes d'usuari i permisos.

Crear una feina de Jenkins

Per crear una feina de Jenkins per a proves automatitzades:

  1. Nou element: Fes clic a "Nou element" al tauler de control de Jenkins.
  2. Tipus de feina: Tria "Projecte freestyle" i dóna-li un nom.
  3. Gestió del codi font: Configura el repositori de codi font (per exemple, Git).
  4. Desencadenants de construcció: Configura desencadenants com "Construir periòdicament" o "Construir quan es fa un canvi a GitHub".
  5. Passos de construcció: Afegeix passos de construcció per compilar el codi i executar les proves.

Escriure i executar proves a Jenkins

Jenkins admet diversos marcs de proves i eines. Aquí, cobrirem:

  • Proves unitàries amb JUnit
  • Proves d'integració amb Maven
  • Proves d'interfície d'usuari amb Selenium

Proves unitàries amb JUnit

JUnit és un marc popular per a proves unitàries en Java. Per executar proves JUnit a Jenkins:

  1. Afegeix el connector JUnit: Assegura't que el connector JUnit està instal·lat.
  2. Escriu proves: Escriu proves JUnit al teu projecte.
  3. Configura el pas de construcció: Afegeix un pas de construcció per executar les proves.

Exemple de prova JUnit:

import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {
    @Test
    public void testAdd() {
        Calculator calc = new Calculator();
        assertEquals(5, calc.add(2, 3));
    }
}

Proves d'integració amb Maven

Maven és una eina d'automatització de construcció que també pot executar proves d'integració. Per configurar proves d'integració Maven a Jenkins:

  1. Afegeix el connector Maven: Assegura't que el connector d'integració Maven està instal·lat.
  2. Escriu proves: Escriu proves d'integració utilitzant un marc com TestNG o JUnit.
  3. Configura el pas de construcció: Afegeix un pas de construcció per executar mvn test.

Exemple de configuració Maven:

<project>
    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-surefire-plugin</artifactId>
                <version>2.22.2</version>
            </plugin>
        </plugins>
    </build>
</project>

Proves d'interfície d'usuari amb Selenium

Selenium és un marc per automatitzar navegadors web. Per executar proves Selenium a Jenkins:

  1. Afegeix el connector Selenium: Assegura't que el connector Selenium està instal·lat.
  2. Escriu proves: Escriu proves Selenium al teu projecte.
  3. Configura el pas de construcció: Afegeix un pas de construcció per executar les proves.

Exemple de prova Selenium:

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class GoogleSearchTest {
    @Test
    public void testSearch() {
        WebDriver driver = new ChromeDriver();
        driver.get("http://www.google.com");
        assertEquals("Google", driver.getTitle());
        driver.quit();
    }
}

Tècniques avançades de proves

A mesura que et familiaritzis més amb Jenkins, pots explorar tècniques avançades de proves:

  • Proves en paral·lel: Executa proves en paral·lel per reduir el temps de construcció.
  • Pipeline com a codi: Utilitza Jenkins Pipeline per definir el teu procés de construcció com a codi.
  • Informes de proves: Genera i publica informes de proves.

Proves en paral·lel

Per executar proves en paral·lel:

  1. Configura l'execució en paral·lel: Utilitza la sintaxi de Jenkins Pipeline per definir etapes paral·leles.
  2. Distribueix les proves: Divideix les proves entre diversos nodes o contenidors.

Exemple de pipeline de proves en paral·lel:

pipeline {
    agent any
    stages {
        stage('Parallel Tests') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        sh 'mvn test -Dtest=UnitTestSuite'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        sh 'mvn test -Dtest=IntegrationTestSuite'
                    }
                }
            }
        }
    }
}

Pipeline com a codi

Jenkins Pipeline et permet definir el teu procés de construcció com a codi. Això es pot fer utilitzant un Jenkinsfile.

Exemple de Jenkinsfile:

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Deploy') {
            steps {
                sh 'mvn deploy'
            }
        }
    }
}

Informes de proves

Per generar i publicar informes de proves:

  1. Configura el connector d'informes de proves: Utilitza connectors com JUnit o Allure Report.
  2. Publica informes: Afegeix accions post-construcció per publicar els resultats de les proves.

Exemple de configuració d'informes de proves:

pipeline {
    agent any
    stages {
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}

Conclusió

Les proves automatitzades amb Jenkins són una manera poderosa d'assegurar la qualitat i la fiabilitat del teu programari. Configurant Jenkins, escrivint i executant proves, i explorant tècniques avançades, pots crear una pipeline CI/CD robusta que detecti problemes aviat i millori el teu flux de treball de desenvolupament.
