Introducció a Jenkins

Jenkins és un servidor d'automatització de codi obert que permet als desenvolupadors construir, provar i desplegar les seves aplicacions de manera fiable i eficient. Aquesta secció cobrirà els conceptes bàsics de Jenkins, la seva arquitectura i com encaixa en el pipeline de CI/CD.

Configuració de Jenkins

Abans de desplegar aplicacions, cal configurar Jenkins. Aquesta secció et guiarà a través de la instal·lació i la configuració inicial de Jenkins.

  • Instal·lació: Jenkins es pot instal·lar en diversos sistemes operatius, incloent Windows, macOS i Linux.
  • Configuració: La configuració inicial inclou configurar la seguretat, crear comptes d'usuari i instal·lar els plugins necessaris.

Exemple: Instal·lant Jenkins a Ubuntu

# Actualitza l'índex de paquets
sudo apt update

# Instal·la Java (Jenkins requereix Java per funcionar)
sudo apt install openjdk-11-jre

# Afegeix el repositori Debian de Jenkins
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'

# Actualitza l'índex de paquets de nou
sudo apt update

# Instal·la Jenkins
sudo apt install jenkins

# Inicia Jenkins
sudo systemctl start jenkins

# Activa Jenkins perquè s'iniciï en arrencar
sudo systemctl enable jenkins

Creant el teu Primer Job a Jenkins

Un job a Jenkins és una tasca o un conjunt de tasques que Jenkins executa. Aquesta secció et guiarà a través de la creació del teu primer job a Jenkins.

  • Projecte Freestyle: Un tipus de projecte bàsic que et permet configurar i executar tasques simples.
  • Projecte Pipeline: Un tipus de projecte més avançat que utilitza un Jenkinsfile per definir els passos del teu pipeline de CI/CD.

Exemple: Creant un Projecte Freestyle

  1. Obre Jenkins al teu navegador web.
  2. Fes clic a "New Item" al menú de l'esquerra.
  3. Introdueix un nom per al teu projecte i selecciona "Freestyle project".
  4. Fes clic a "OK" i configura els paràmetres del teu projecte.
  5. Afegeix passos de construcció, com executar comandes shell o scripts.
  6. Desa el teu projecte i fes clic a "Build Now" per executar-lo.

Pipelines de Jenkins

Els Pipelines de Jenkins són una manera poderosa de definir els teus processos de construcció, prova i desplegament com a codi. Aquesta secció t'introduirà als Pipelines de Jenkins i com crear-los.

  • Pipeline Declaratiu: Una sintaxi més simple i llegible per definir pipelines.
  • Pipeline Escripturat: Una sintaxi més flexible i poderosa per a casos d'ús avançats.

Exemple: Pipeline Declaratiu

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Afegeix passos de construcció aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Afegeix passos de prova aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Afegeix passos de desplegament aquí
            }
        }
    }
}

Integrant Jenkins amb Sistemes de Control de Versions

Jenkins es pot integrar amb diversos sistemes de control de versions (VCS) com Git, SVN i Mercurial. Aquesta secció es centrarà en integrar Jenkins amb Git.

  • Plugin de Git: El plugin de Git permet a Jenkins obtenir codi d'un repositori Git.
  • Webhooks: Els webhooks poden desencadenar jobs de Jenkins automàticament quan es fan canvis al repositori.

Exemple: Configurant Jenkins amb Git

  1. Instal·la el plugin de Git des del gestor de plugins de Jenkins.
  2. Crea un nou job a Jenkins o configura'n un d'existent.
  3. A la secció "Source Code Management", selecciona "Git".
  4. Introdueix l'URL del repositori i les credencials si és necessari.
  5. Configura els desencadenadors de construcció, com el polling del repositori o l'ús de webhooks.

Desplegant Aplicacions

Desplegar aplicacions és una part crítica del pipeline de CI/CD. Aquesta secció cobrirà diferents estratègies de desplegament i com implementar-les a Jenkins.

  • Desplegament Continu: Desplegar automàticament cada canvi que passi les proves.
  • Lliurament Continu: Preparar automàticament cada canvi per al desplegament però requerir aprovació manual per desplegar.
  • Desplegament Blue-Green: Desplegar a un nou entorn (blue) mentre es manté l'entorn antic (green) en funcionament, després canviar el trànsit al nou entorn.

Exemple: Desplegant una Aplicació Web

pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Afegeix passos de construcció aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Afegeix passos de prova aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                sh 'scp -r ./build user@server:/path/to/deploy'
                sh 'ssh user@server "systemctl restart webapp"'
            }
        }
    }
}

Monitorització i Notificacions

Monitoritzar els teus jobs de Jenkins i rebre notificacions sobre el seu estat és crucial per mantenir un pipeline de CI/CD saludable.

  • Estat de la Construcció: Jenkins proporciona diverses maneres de monitoritzar l'estat de les teves construccions, incloent la interfície web, l'historial de construccions i la sortida de la consola.
  • Notificacions: Jenkins pot enviar notificacions via correu electrònic, Slack o altres eines de comunicació.

Exemple: Configurant Notificacions per Correu Electrònic

  1. Instal·la el Plugin d'Extensió de Correu Electrònic des del gestor de plugins de Jenkins.
  2. Configura el servidor SMTP a la configuració global de Jenkins.
  3. Al teu job de Jenkins, afegeix una "Post-build Action" per enviar una notificació per correu electrònic.
  4. Configura els destinataris i el contingut del correu electrònic.

Conclusió

Desplegar aplicacions amb Jenkins implica configurar Jenkins, crear jobs, definir pipelines, integrar-se amb sistemes de control de versions i implementar estratègies de desplegament. Seguint aquests passos, pots automatitzar el teu pipeline de CI/CD i assegurar desplegaments d'aplicacions fiables i eficients. Recorda monitoritzar les teves construccions i configurar notificacions per estar informat sobre l'estat dels teus desplegaments.
