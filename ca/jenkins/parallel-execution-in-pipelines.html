Introducció a l'Execució Paral·lela

L'execució paral·lela en pipelines de Jenkins permet que múltiples tasques s'executin simultàniament, cosa que pot reduir significativament el temps total de construcció. Això és particularment útil per a projectes grans amb múltiples etapes independents.

Conceptes Clau

  • Etapes: Agrupació lògica de passos en un pipeline.
  • Etapes Paral·leles: Etapes que s'executen simultàniament.
  • Sintaxi Declarativa de Pipeline: Una sintaxi més estructurada i fàcil de llegir per definir pipelines de Jenkins.

Configuració de l'Execució Paral·lela

Execució Paral·lela Bàsica

Per configurar l'execució paral·lela en un pipeline de Jenkins, utilitzeu la directiva parallel dins d'una stage. Aquí teniu un exemple senzill:

pipeline {
    agent any
    stages {
        stage('Parallel Stage') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        echo 'Running Unit Tests'
                        // Afegiu les vostres comandes de proves unitàries aquí
                    }
                }
                stage('Integration Tests') {
                    steps {
                        echo 'Running Integration Tests'
                        // Afegiu les vostres comandes de proves d'integració aquí
                    }
                }
                stage('UI Tests') {
                    steps {
                        echo 'Running UI Tests'
                        // Afegiu les vostres comandes de proves d'UI aquí
                    }
                }
            }
        }
    }
}

Explicació

  • pipeline: Defineix el pipeline de Jenkins.
  • agent any: Executa el pipeline en qualsevol agent disponible.
  • stages: Conté totes les etapes del pipeline.
  • stage('Parallel Stage'): Una etapa que conté tasques paral·leles.
  • parallel: Directiva per executar etapes niades simultàniament.
  • stage('Unit Tests'), stage('Integration Tests'), stage('UI Tests'): Etapes individuals que s'executaran en paral·lel.

Execució Paral·lela Avançada

Etapes Paral·leles Niades

Podeu niar etapes paral·leles per crear fluxos de treball més complexos. Aquí teniu un exemple:

pipeline {
    agent any
    stages {
        stage('Build and Test') {
            parallel {
                stage('Build') {
                    steps {
                        echo 'Building the project'
                        // Afegiu les comandes de construcció aquí
                    }
                }
                stage('Test') {
                    parallel {
                        stage('Unit Tests') {
                            steps {
                                echo 'Running Unit Tests'
                                // Afegiu les vostres comandes de proves unitàries aquí
                            }
                        }
                        stage('Integration Tests') {
                            steps {
                                echo 'Running Integration Tests'
                                // Afegiu les vostres comandes de proves d'integració aquí
                            }
                        }
                    }
                }
            }
        }
    }
}

Explicació

  • Etapes paral·leles niades: L'etapa Test conté les seves pròpies etapes paral·leles (Unit Tests i Integration Tests), demostrant com podeu crear fluxos d'execució paral·lela més complexos.

Gestió de Fallades en Etapes Paral·leles

Estratègia de Fallada Ràpida

Per defecte, si una de les etapes paral·leles falla, les altres continuaran executant-se. No obstant això, podeu configurar el pipeline perquè falli ràpidament utilitzant l'opció failFast.

pipeline {
    agent any
    stages {
        stage('Parallel Stage') {
            parallel failFast: true, stages: {
                stage('Unit Tests') {
                    steps {
                        echo 'Running Unit Tests'
                        // Simular una fallada
                        error 'Unit Tests failed'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        echo 'Running Integration Tests'
                        // Afegiu les vostres comandes de proves d'integració aquí
                    }
                }
                stage('UI Tests') {
                    steps {
                        echo 'Running UI Tests'
                        // Afegiu les vostres comandes de proves d'UI aquí
                    }
                }
            }
        }
    }
}

Explicació

  • failFast: true: Si alguna de les etapes paral·leles falla, les etapes restants seran avortades immediatament.

Conclusió

L'execució paral·lela en pipelines de Jenkins és una característica potent que pot accelerar significativament els vostres processos de construcció i proves. Comprenent com configurar i gestionar etapes paral·leles, podeu optimitzar els vostres fluxos de treball de CI/CD per obtenir un millor rendiment i eficiència. Tant si executeu tasques paral·leles senzilles com etapes niades complexes, Jenkins proporciona la flexibilitat per gestionar diversos escenaris de manera efectiva.
