Introducció als Plugins de Jenkins

Jenkins és un servidor d'automatització potent que admet la construcció, el desplegament i l'automatització de qualsevol projecte. Una de les seves característiques més poderoses és la seva extensibilitat a través de plugins. Aquesta secció et guiarà a través dels conceptes bàsics d'escriure plugins per a Jenkins, des de la configuració del teu entorn de desenvolupament fins a la creació i el desplegament del teu primer plugin.

Conceptes Clau

  • Plugin de Jenkins: Un component de programari que afegeix capacitats específiques a Jenkins.
  • Arquitectura de Plugins de Jenkins: Jenkins utilitza una arquitectura modular on els plugins poden estendre la seva funcionalitat bàsica.
  • Maven: Una eina d'automatització de construcció utilitzada principalment per a projectes Java, essencial per al desenvolupament de plugins de Jenkins.

Configuració del Teu Entorn de Desenvolupament

Abans de començar a escriure plugins per a Jenkins, necessites configurar el teu entorn de desenvolupament.

Prerequisits

  • Java Development Kit (JDK): Assegura't de tenir instal·lat el JDK 8 o posterior.
  • Apache Maven: Maven és necessari per construir plugins de Jenkins.
  • Entorn de Desenvolupament Integrat (IDE): IntelliJ IDEA, Eclipse, o qualsevol altre IDE per a Java.

Passos per Configurar

  1. Instal·la el JDK: Descarrega i instal·la el JDK des del lloc web d'Oracle.
  2. Instal·la Maven: Descarrega i instal·la Maven des del lloc web d'Apache Maven.
  3. Configura el Teu IDE: Configura el teu IDE per utilitzar el JDK i Maven instal·lats.

Creació del Teu Primer Plugin per a Jenkins

En aquesta secció, crearem un plugin senzill per a Jenkins que afegeix un nou pas de construcció.

Guia Pas a Pas

  1. Genera l'Esquelet del Plugin: Utilitza l'arquetip de Maven per generar una estructura bàsica de plugin.

    mvn archetype:generate -Dfilter=io.jenkins.archetypes:

    Segueix les indicacions per seleccionar l'arquetip io.jenkins.archetypes:empty-plugin.

  2. Entén l'Estructura Generada:

    • pom.xml: El fitxer descriptor del projecte Maven.
    • src/main/java: Conté el codi font Java.
    • src/main/resources: Conté els recursos del plugin, com ara fitxers de configuració.

  3. Implementa la Lògica del Plugin:

    • Crea una nova classe Java a src/main/java que estengui Builder.
    • Anota la classe amb @Extension per registrar-la com una extensió de Jenkins.
    package com.example.jenkins;

import hudson.Extension;
import hudson.model.AbstractBuild;
import hudson.model.BuildListener;
import hudson.tasks.Builder;
import hudson.tasks.BuildStepDescriptor;
import hudson.tasks.BuildStepMonitor;
import hudson.tasks.Publisher;
import org.kohsuke.stapler.DataBoundConstructor;

public class HelloWorldBuilder extends Builder {

    private final String name;

    @DataBoundConstructor
    public HelloWorldBuilder(String name) {
        this.name = name;
    }

    @Override
    public boolean perform(AbstractBuild build, BuildListener listener) {
        listener.getLogger().println("Hello, " + name + "!");
        return true;
    }

    @Extension
    public static final class DescriptorImpl extends BuildStepDescriptor<Builder> {
        @Override
        public boolean isApplicable(Class<? extends AbstractBuild> aClass) {
            return true;
        }

        @Override
        public String getDisplayName() {
            return "Say Hello World";
        }
    }
}

  4. Construeix el Plugin:

    mvn clean install

  5. Desplega el Plugin:

    • Copia el fitxer .hpi generat des del directori target al directori de plugins de Jenkins.
    • Reinicia Jenkins per carregar el nou plugin.

Desenvolupament Avançat de Plugins

Un cop et sentis còmode amb la creació de plugins bàsics, pots explorar temes més avançats.

Conceptes Clau Avançats

  • Punts d'Extensió de Jenkins: Interfícies o classes abstractes que els plugins poden implementar per estendre la funcionalitat de Jenkins.
  • API de Jenkins: El conjunt de classes i interfícies Java proporcionades per Jenkins per al desenvolupament de plugins.
  • Proves Unitàries: Escriure proves per al teu plugin utilitzant JenkinsRule i altres utilitats de prova.

Exemple: Afegir una Pàgina de Configuració

Per afegir una pàgina de configuració al teu plugin, necessites crear un fitxer config.jelly al directori src/main/resources.

<j:jelly xmlns:j="jelly:core" xmlns:st="jelly:stapler" xmlns:f="/lib/form">
    <f:entry title="Name" field="name">
        <f:textbox />
    </f:entry>
</j:jelly>

Conclusió

Escriure plugins per a Jenkins et permet estendre les capacitats de Jenkins per satisfer les teves necessitats específiques. Entenent els conceptes bàsics del desenvolupament de plugins, configurant el teu entorn de desenvolupament i seguint un enfocament estructurat per crear i desplegar plugins, pots aprofitar tot el potencial de Jenkins. A mesura que avancis, explorar els punts d'extensió i les API de Jenkins et permetrà crear plugins més sofisticats i potents. Feliç programació!
