Què és CI/CD?

La Integració Contínua (CI) i el Desplegament Continu (CD) són pràctiques en el desenvolupament de programari destinades a millorar la qualitat i la velocitat de lliurament del programari.

  • Integració Contínua (CI): La pràctica de fusionar totes les còpies de treball dels desenvolupadors en una línia principal compartida diverses vegades al dia.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa totes les etapes del pipeline de producció a l'entorn de producció.

Beneficis de CI/CD

  • Temps de Mercat més Ràpid: Automatitza el procés d'integració i desplegament de codi, reduint el temps necessari per llançar noves funcionalitats.
  • Millora de la Qualitat del Codi: Les proves automatitzades s'executen amb cada canvi, detectant errors a l'inici del cicle de desenvolupament.
  • Reducció de l'Esforç Manual: L'automatització redueix la necessitat d'intervenció manual, alliberant els desenvolupadors per centrar-se en escriure codi.

Conceptes Clau en CI/CD

  • Automatització de la Construcció: Compilar i construir el codi automàticament.
  • Proves Automatitzades: Executar proves automàticament per garantir la qualitat del codi.
  • Integració Contínua: Integrar freqüentment els canvis de codi en un repositori compartit.
  • Desplegament Continu: Desplegar automàticament els canvis de codi a producció.
  • Sistemes de Control de Versions (VCS): Eines com Git que gestionen els canvis en el codi font.

Visió General de Jenkins

Jenkins és un servidor d'automatització de codi obert que permet als desenvolupadors construir, provar i desplegar el seu programari.

  • Plugins: Jenkins té un ric ecosistema de plugins que amplien la seva funcionalitat.
  • Pipeline: Jenkins Pipeline és un conjunt de plugins que suporten la implementació i integració de pipelines de lliurament continu.
  • Pipelines Declaratius i Scriptats: Jenkins suporta dos tipus de pipelines, Declaratius i Scriptats.

Configuració de Jenkins

Prerequisits

  • Java Development Kit (JDK)
  • Navegador Web
  • Connexió a Internet

Passos d'Instal·lació

  1. Descarregar Jenkins: Visita el lloc web de Jenkins i descarrega l'instal·lador adequat per al teu sistema operatiu.
  2. Instal·lar Jenkins: Segueix les instruccions d'instal·lació per al teu sistema operatiu.
  3. Iniciar Jenkins: Executa Jenkins i obre'l en el teu navegador web a http://localhost:8080.

Configuració Inicial

  • Desbloquejar Jenkins: Utilitza la contrasenya d'administrador inicial proporcionada durant la instal·lació.
  • Instal·lar Plugins Suggerits: Jenkins recomanarà un conjunt de plugins per instal·lar.
  • Crear Usuari Administrador: Configura el compte d'usuari administrador inicial.

Creació del teu Primer Job a Jenkins

Guia Pas a Pas

  1. Crear un Nou Job: Fes clic a "New Item" al tauler de Jenkins.
  2. Introduir Nom del Job: Proporciona un nom per al teu job.
  3. Seleccionar Tipus de Job: Tria "Freestyle project" i fes clic a "OK".
  4. Configurar la Gestió del Codi Font: Connecta't al teu sistema de control de versions (per exemple, Git). 
    git 'https://github.com/your-repo.git'
  5. Afegir Passos de Construcció: Afegeix passos per compilar i construir el teu codi. 
    mvn clean install
  6. Afegir Accions Post-Construcció: Afegeix accions com enviar notificacions o arxivar artefactes.
  7. Guardar i Construir: Guarda la configuració del job i fes clic a "Build Now".

Pipeline de Jenkins

Exemple de Pipeline Declaratiu

Un exemple senzill de Pipeline Declaratiu de Jenkins:

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                sh 'mvn clean install'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                sh 'mvn test'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                sh 'scp target/my-app.jar user@server:/path/to/deploy'
            }
        }
    }
}

Explicació

  • agent any: Executa el pipeline en qualsevol agent disponible.
  • stages: Defineix les etapes del pipeline.
  • steps: Els passos individuals a executar en cada etapa.
