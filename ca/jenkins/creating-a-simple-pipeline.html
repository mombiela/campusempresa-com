Creació d'un Pipeline Simple
Sintaxi i estructura de la pipeline ►

Introducció

En aquesta secció, cobrirem els conceptes bàsics de la creació d'un pipeline simple a Jenkins. Un pipeline automatitza el procés de construir, provar i desplegar codi, fent-lo una eina essencial per a la integració contínua i el lliurament continu (CI/CD).

Conceptes Clau

  • Pipeline: Una sèrie de passos automatitzats per construir, provar i desplegar codi.
  • Jenkinsfile: Un fitxer de text que conté la definició d'un Pipeline de Jenkins i es guarda en el control de versions.
  • Etapes: Diferents fases del pipeline, com ara construir, provar i desplegar.
  • Passos: Tasques individuals dins d'una etapa.

Configuració de Jenkins

Abans de crear un pipeline, assegura't de tenir Jenkins instal·lat i en funcionament. Pots descarregar Jenkins des del lloc web oficial.

Creació del Teu Primer Pipeline

Pas 1: Crear una Nova Feina de Pipeline

  1. Obre Jenkins al teu navegador web.
  2. Fes clic a New Item al menú de l'esquerra.
  3. Introdueix un nom per al teu pipeline i selecciona Pipeline com a tipus de projecte.
  4. Fes clic a OK.

Pas 2: Definir el Pipeline en un Jenkinsfile

  1. A la pàgina de configuració del pipeline, desplaça't cap avall fins a la secció Pipeline.
  2. Selecciona Pipeline script del desplegable Definition.
  3. Introdueix el següent script a l'àrea de script:
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Afegeix passos de construcció aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Afegeix passos de prova aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Afegeix passos de desplegament aquí
            }
        }
    }
}

Explicació de l'Script

  • pipeline { ... }: Defineix l'inici i el final del pipeline.
  • agent any: Especifica que el pipeline pot executar-se en qualsevol agent disponible.
  • stages { ... }: Conté totes les etapes del pipeline.
  • stage('Nom de l'Etapa') { ... }: Defineix una etapa amb un nom específic.
  • steps { ... }: Conté els passos a executar dins d'una etapa.
  • echo 'Missatge': Imprimeix un missatge a la sortida de la consola.

Pas 3: Desar i Executar el Pipeline

  1. Fes clic a Save per desar la configuració del teu pipeline.
  2. Fes clic a Build Now al menú de l'esquerra per executar el pipeline.
  3. Monitoritza el progrés a Build History i Console Output.

Afegint Més Funcionalitat

Utilitzant el Control de Versions

Per fer el teu pipeline més robust, guarda el teu Jenkinsfile en el control de versions (per exemple, GitHub). Modifica la configuració del pipeline per utilitzar l'opció Pipeline script from SCM i especifica l'URL del repositori i la branca.

Exemple de Jenkinsfile des de GitHub

pipeline {
    agent any

    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/your-repo/your-project.git'
            }
        }
        stage('Build') {
            steps {
                echo 'Building...'
                // Afegeix passos de construcció aquí
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Afegeix passos de prova aquí
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Afegeix passos de desplegament aquí
            }
        }
    }
}

Conclusió

Crear un pipeline simple a Jenkins implica definir etapes i passos en un Jenkinsfile. Aquest fitxer es pot guardar en el control de versions per a una millor gestió i versionat. Seguint els passos descrits en aquesta secció, pots automatitzar els teus processos de construcció, prova i desplegament, fent el teu flux de treball de desenvolupament més eficient i fiable.
