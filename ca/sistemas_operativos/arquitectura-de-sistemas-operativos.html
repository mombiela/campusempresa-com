Introducció

L'arquitectura d'un sistema operatiu (SO) defineix la seva estructura i la manera en què els seus components interactuen entre si. Comprendre aquesta arquitectura és crucial per a qualsevol professional de TI, ja que proporciona una base sòlida per entendre com els sistemes operatius gestionen els recursos i executen aplicacions.

Components Principals d'un Sistema Operatiu

Un sistema operatiu es compon de diversos components clau, cadascun amb funcions específiques:

  • Nucli (Kernel): És el cor del sistema operatiu. Gestiona els recursos del sistema i permet la comunicació entre el maquinari i el programari.
  • Gestor de processos: Maneja la creació, planificació i terminació de processos.
  • Gestor de memòria: Administra la memòria principal, assignant i alliberant espai segons sigui necessari.
  • Sistema de fitxers: Organitza i emmagatzema les dades en dispositius d'emmagatzematge.
  • Controladors de dispositius: Faciliten la comunicació entre el sistema operatiu i el maquinari.
  • Interfície d'usuari: Proporciona una forma perquè els usuaris interactuïn amb el sistema operatiu, ja sigui a través d'una interfície gràfica (GUI) o una línia de comandes (CLI).

Tipus d'Arquitectura de Sistemes Operatius

Hi ha diversos tipus d'arquitectura de sistemes operatius, cadascun amb els seus avantatges i desavantatges:

  • Monolítica: Tots els components del sistema operatiu estan integrats en un únic bloc de codi. Exemples: Unix, Linux.
  • Microkernel: Només les funcions essencials s'executen en el nucli, mentre que altres serveis s'executen en espai d'usuari. Exemples: Minix, QNX.
  • Exokernel: Permet a les aplicacions gestionar els recursos del maquinari directament, proporcionant un mínim d'abstracció. Exemple: Exokernel.
  • Híbrida: Combina elements de les arquitectures monolítica i microkernel. Exemples: Windows NT, MacOS.

Exemple de Codi: Gestió de Processos en un Sistema Operatiu

A continuació, es presenta un exemple de com un sistema operatiu podria gestionar processos utilitzant una cua de planificació simple en C:

#include <stdio.h>
#include <stdlib.h>

typedef struct Procés {
    int id;
    struct Procés* següent;
} Procés;

Procés* crear_procés(int id) {
    Procés* nou_procés = (Procés*)malloc(sizeof(Procés));
    nou_procés->id = id;
    nou_procés->següent = NULL;
    return nou_procés;
}

void afegir_procés(Procés** cap, int id) {
    Procés* nou_procés = crear_procés(id);
    if (*cap == NULL) {
        *cap = nou_procés;
    } else {
        Procés* temp = *cap;
        while (temp->següent != NULL) {
            temp = temp->següent;
        }
        temp->següent = nou_procés;
    }
}

void mostrar_processos(Procés* cap) {
    Procés* temp = cap;
    while (temp != NULL) {
        printf("Procés ID: %d\n", temp->id);
        temp = temp->següent;
    }
}

int main() {
    Procés* cap = NULL;
    afegir_procés(&cap, 1);
    afegir_procés(&cap, 2);
    afegir_procés(&cap, 3);
    mostrar_processos(cap);
    return 0;
}

Explicació del Codi

  • Estructura Procés: Defineix un procés amb un identificador i un punter al següent procés.
  • crear_procés: Funció per crear un nou procés.
  • afegir_procés: Funció per afegir un procés a la cua de planificació.
  • mostrar_processos: Funció per mostrar tots els processos en la cua.
  • main: Funció principal que crea i mostra una llista de processos.

Conclusió

L'arquitectura d'un sistema operatiu és fonamental per al seu funcionament i eficiència. Comprendre els diferents tipus d'arquitectures i els seus components permet als professionals de TI prendre decisions informades sobre el disseny i la implementació de sistemes operatius. A més, el coneixement pràctic, com la gestió de processos, és essencial per aplicar aquests conceptes en situacions del món real.
