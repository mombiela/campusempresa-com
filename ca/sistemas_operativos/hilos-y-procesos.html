Introducció

En els sistemes operatius, els conceptes de fils i processos són fonamentals per entendre com s'executen les tasques i com es gestionen els recursos del sistema. Aquest tema aborda les diferències entre fils i processos, les seves característiques, avantatges i desavantatges, així com exemples pràctics per il·lustrar el seu ús.

Processos

Un procés és una instància d'un programa en execució. Cada procés té el seu propi espai d'adreces en memòria, la qual cosa significa que un procés no pot accedir directament a la memòria d'un altre procés.

Característiques dels Processos

  • Aïllament: Cada procés opera en el seu propi espai de memòria.
  • Recursos: Cada procés té els seus propis recursos, com ara fitxers oberts i variables d'entorn.
  • Context d'Execució: Cada procés té el seu propi context d'execució, que inclou el comptador de programa, registres i pila.

Exemple de Codi en C

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main() {
    pid_t pid = fork();

    if (pid < 0) {
        fprintf(stderr, "Fork failed");
        return 1;
    } else if (pid == 0) {
        printf("This is the child process\n");
    } else {
        printf("This is the parent process\n");
    }

    return 0;
}

En aquest exemple, la funció fork() crea un nou procés. El procés fill i el procés pare executen el mateix codi, però poden diferenciar-se pel valor de retorn de fork().

Fils

Un fil és la unitat més petita de processament que pot ser programada per un sistema operatiu. Els fils comparteixen el mateix espai d'adreces dins d'un procés, la qual cosa permet una comunicació més eficient però també introdueix riscos de sincronització.

Característiques dels Fils

  • Compartició de Recursos: Els fils dins del mateix procés comparteixen el mateix espai de memòria i recursos.
  • Menor Sobrecàrrega: Crear i destruir fils és menys costós en termes de recursos que crear i destruir processos.
  • Comunicació Ràpida: La comunicació entre fils és més ràpida a causa de la compartició de memòria.

Exemple de Codi en C amb Pthreads

#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

void* print_message(void* ptr) {
    char* message = (char*) ptr;
    printf("%s\n", message);
    return NULL;
}

int main() {
    pthread_t thread1, thread2;
    char* message1 = "Thread 1";
    char* message2 = "Thread 2";

    pthread_create(&thread1, NULL, print_message, (void*) message1);
    pthread_create(&thread2, NULL, print_message, (void*) message2);

    pthread_join(thread1, NULL);
    pthread_join(thread2, NULL);

    return 0;
}

En aquest exemple, es creen dos fils que executen la funció print_message. Els fils s'executen concurrentment i comparteixen el mateix espai de memòria.

Comparació entre Fils i Processos

| Característica | Processos | Fils | |----------------------|-------------------------------------|-----------------------------------| | Espai de Memòria | Aïllat | Compartit | | Creació | Costosa | Menys costosa | | Comunicació | A través d'IPC (Inter-Process Communication) | Directa a través de memòria compartida | | Seguretat | Alta (degut a l'aïllament) | Baixa (degut a la compartició de memòria) | | Context d'Execució | Independent | Compartit |

Conclusió

Entendre la diferència entre fils i processos és crucial per dissenyar aplicacions eficients i segures. Els processos ofereixen aïllament i seguretat, però a un cost més gran en termes de recursos. Els fils, d'altra banda, permeten una comunicació més ràpida i eficient, però requereixen una gestió acurada per evitar problemes de sincronització. Conèixer quan i com utilitzar cadascun és una habilitat essencial per a qualsevol professional de TI.
