Introducció

Un sistema operatiu (SO) és el programari fonamental que permet la gestió dels recursos de maquinari i programari d'un ordinador. Actua com a intermediari entre els usuaris i el maquinari del sistema, facilitant l'execució d'aplicacions i la gestió de recursos. En aquesta secció, explorarem els components principals d'un sistema operatiu i la seva funció en el sistema.

Components Principals

1. Nucli (Kernel)

El nucli és el cor del sistema operatiu. És responsable de gestionar les operacions més bàsiques i essencials del sistema. Les seves funcions inclouen:

  • Gestió de processos: Controla la creació, execució i terminació de processos.
  • Gestió de memòria: Administra la memòria principal (RAM), assignant i alliberant espai segons sigui necessari.
  • Gestió de dispositius: Coordina la comunicació entre el maquinari i el programari.
  • Gestió d'arxius: Maneja la creació, lectura, escriptura i eliminació d'arxius en el sistema d'arxius.

Exemple de codi: Creació d'un procés en C

#include <stdio.h>
#include <unistd.h>

int main() {
    pid_t pid = fork();

    if (pid == 0) {
        // Procés fill
        printf("Sóc el procés fill\n");
    } else if (pid > 0) {
        // Procés pare
        printf("Sóc el procés pare\n");
    } else {
        // Error en crear el procés
        printf("Error en crear el procés\n");
    }

    return 0;
}

2. Gestor de Memòria

El gestor de memòria és responsable de l'assignació i alliberament de memòria als processos en execució. Les seves funcions inclouen:

  • Assignació de memòria: Distribueix la memòria disponible entre els processos.
  • Paginació i segmentació: Tècniques per gestionar la memòria de manera eficient.
  • Memòria virtual: Permet que els processos utilitzin més memòria de la que físicament està disponible mitjançant l'ús d'espai en disc.

3. Sistema d'Arxius

El sistema d'arxius organitza i emmagatzema les dades en el disc dur. Les seves funcions inclouen:

  • Estructura de directoris: Organitza els arxius en una jerarquia de directoris.
  • Permisos d'arxius: Controla l'accés als arxius mitjançant permisos de lectura, escriptura i execució.
  • Operacions d'arxius: Facilita la creació, lectura, escriptura i eliminació d'arxius.

Taula: Comparació de sistemes d'arxius

4. Gestor de Dispositius

El gestor de dispositius controla i coordina l'ús dels dispositius de maquinari. Les seves funcions inclouen:

  • Controladors de dispositius: Programari que permet la comunicació entre el sistema operatiu i el maquinari.
  • Interrupcions: Mecanisme per gestionar esdeveniments asíncrons dels dispositius.
  • Buffering i Spooling: Tècniques per gestionar l'entrada i sortida de dades dels dispositius.

5. Interfície d'Usuari

La interfície d'usuari permet la interacció entre l'usuari i el sistema operatiu. Pot ser:

  • Interfície de línia de comandes (CLI): Permet als usuaris interactuar amb el sistema mitjançant comandes de text.
  • Interfície gràfica d'usuari (GUI): Proporciona una interacció més intuïtiva mitjançant finestres, icones i menús.

Exemple de comandes en CLI

# Llistar arxius en el directori actual
ls

# Canviar de directori
cd /ruta/al/directori

# Crear un nou arxiu
touch nou_arxiu.txt

Conclusió

Els components principals d'un sistema operatiu són essencials per al seu funcionament i eficiència. El nucli, el gestor de memòria, el sistema d'arxius, el gestor de dispositius i la interfície d'usuari treballen junts per proporcionar un entorn estable i eficient per a l'execució d'aplicacions. Comprendre aquests components és fonamental per a qualsevol professional de TI que desitgi aprofundir en el disseny i l'administració de sistemes operatius.
