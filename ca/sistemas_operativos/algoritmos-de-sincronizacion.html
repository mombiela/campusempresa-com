Introducció

En els sistemes operatius, la sincronització és crucial per gestionar l'execució concurrent de processos i threads. Els algoritmes de sincronització asseguren que els recursos compartits s'utilitzin de manera segura i eficient, evitant condicions de carrera i altres problemes de concurrència.

Conceptes Clau

  • Condició de carrera: Passa quan dos o més processos accedeixen i manipulen dades compartides de manera concurrent, i el resultat depèn de l'ordre d'execució.
  • Secció crítica: Part del codi on s'accedeix a recursos compartits i que ha d'executar-se de manera atòmica per evitar inconsistències.
  • Mutex (Mutual Exclusion): Mecanisme que assegura que només un procés o thread pugui accedir a una secció crítica a la vegada.
  • Semàfor: Variable o abstracte de programació que s'utilitza per controlar l'accés a un recurs comú per múltiples processos en un entorn concurrent.
  • Monitors: Estructura de sincronització que permet que només un thread accedeixi a una secció crítica a la vegada, encapsulant tant la mutual exclusion com la condició de sincronització.

Algoritmes de Sincronització

Mutex

Un mutex és una eina bàsica per a la sincronització que assegura l'exclusió mútua. Només un thread pot posseir el mutex a la vegada, i altres threads han d'esperar fins que el mutex sigui alliberat.

#include <pthread.h>
#include <stdio.h>

pthread_mutex_t lock;

void *thread_function(void *arg) {
    pthread_mutex_lock(&lock);
    // Secció crítica
    printf("Thread %d està en la secció crítica\n", *(int *)arg);
    pthread_mutex_unlock(&lock);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    pthread_mutex_init(&lock, NULL);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    pthread_mutex_destroy(&lock);
    return 0;
}

Semàfors

Els semàfors són més flexibles que els mutexes i poden ser utilitzats per sincronitzar múltiples threads i processos. Hi ha dos tipus de semàfors: binaris i comptadors.

#include <pthread.h>
#include <semaphore.h>
#include <stdio.h>

sem_t semaphore;

void *thread_function(void *arg) {
    sem_wait(&semaphore);
    // Secció crítica
    printf("Thread %d està en la secció crítica\n", *(int *)arg);
    sem_post(&semaphore);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    sem_init(&semaphore, 0, 1);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    sem_destroy(&semaphore);
    return 0;
}

Monitors

Els monitors encapsulen tant l'exclusió mútua com la condició de sincronització, proporcionant una forma més estructurada de manejar la concurrència.

class Monitor {
    private int count = 0;

    public synchronized void increment() {
        count++;
        System.out.println("Count incrementat: " + count);
    }

    public synchronized void decrement() {
        count--;
        System.out.println("Count decrementat: " + count);
    }
}

public class MonitorExample {
    public static void main(String[] args) {
        Monitor monitor = new Monitor();

        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                monitor.increment();
            }
        });

        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                monitor.decrement();
            }
        });

        t1.start();
        t2.start();
    }
}

Conclusió

Els algoritmes de sincronització són fonamentals per al correcte funcionament de sistemes operatius que gestionen múltiples processos i threads. Entendre com i quan utilitzar mutexes, semàfors i monitors és essencial per evitar problemes de concurrència i assegurar la integritat de les dades compartides. A través d'exemples pràctics, hem vist com implementar i utilitzar aquests mecanismes en diferents llenguatges de programació.
