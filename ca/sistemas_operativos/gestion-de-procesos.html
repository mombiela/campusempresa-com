Introducció

La gestió de processos és una de les funcions fonamentals d'un sistema operatiu. Un procés és un programa en execució, i el sistema operatiu ha de gestionar aquests processos per assegurar que els recursos del sistema s'utilitzin de manera eficient i que els programes s'executin correctament.

Conceptes Clau

  • Procés: Un programa en execució, que inclou el codi del programa, les seves dades, i l'estat actual de l'execució.
  • Estat del procés: Els processos poden estar en diferents estats, com "nou", "en execució", "esperant", "llest" i "acabat".
  • PCB (Process Control Block): Estructura de dades que conté tota la informació sobre un procés, com el seu estat, comptador de programa, registres de CPU, informació de memòria, etc.
  • Context Switch: El procés de guardar l'estat d'un procés i carregar l'estat d'un altre. És essencial per a la multitasca.
  • Scheduler: Component del sistema operatiu que decideix quin procés s'executarà a continuació.
  • Cues de processos: Estructures de dades que mantenen els processos en diferents estats (llest, esperant, etc.).

Estats del Procés

Els processos poden estar en un dels següents estats:

| Estat | Descripció | |-----------|-----------------------------------------------------------------------------| | Nou | El procés està sent creat. | | Llest | El procés està preparat per executar-se i esperant assignació de CPU. | | En execució | El procés està sent executat per la CPU. | | Esperant | El procés està esperant algun esdeveniment (com I/O) per continuar. | | Acabat | El procés ha acabat la seva execució. |

Exemple de Codi: Creació i Gestió de Processos en C

A continuació, es mostra un exemple bàsic de com crear i gestionar processos en C utilitzant les crides al sistema fork() i wait().

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();

    if (pid < 0) {
        // Error al crear el procés
        fprintf(stderr, "Fork failed\n");
        return 1;
    } else if (pid == 0) {
        // Codi del procés fill
        printf("Procés fill: PID = %d\n", getpid());
        execlp("/bin/ls", "ls", NULL);
    } else {
        // Codi del procés pare
        printf("Procés pare: PID = %d\n", getpid());
        wait(NULL); // Espera que el procés fill acabi
        printf("Procés fill acabat\n");
    }

    return 0;
}

Explicació del Codi

  • fork(): Crea un nou procés. El procés fill és una còpia del procés pare.
  • pid < 0: Indica que hi ha hagut un error al crear el procés.
  • pid == 0: Codi que s'executa en el procés fill.
  • execlp(): Reemplaça el programa en el procés fill amb el programa especificat (en aquest cas, ls).
  • wait(): El procés pare espera que el procés fill acabi abans de continuar.

Scheduler i Algoritmes de Planificació

El scheduler és responsable de decidir quin procés s'executarà a continuació. Hi ha diversos algoritmes de planificació, cadascun amb els seus avantatges i desavantatges:

  • First-Come, First-Served (FCFS): Els processos s'executen en l'ordre en què arriben.
  • Shortest Job Next (SJN): El procés amb el menor temps d'execució s'executa primer.
  • Round Robin (RR): Cada procés rep una quantitat fixa de temps de CPU en un cicle.
  • Priority Scheduling: Els processos s'executen segons la seva prioritat.

Comparació d'Algoritmes de Planificació

| Algoritme | Avantatges | Desavantatges | |-----------|-------------------------------------------------|--------------------------------------------------| | FCFS | Simple d'implementar | Pot causar temps d'espera llargs | | SJN | Minimitza el temps d'espera mitjà | Difícil de predir el temps d'execució | | RR | Just, cada procés rep temps de CPU | Pot tenir un alt overhead de context switch | | Priority | Processos importants s'executen primer | Pot causar inanició de processos de baixa prioritat |

Conclusió

La gestió de processos és crucial per al funcionament eficient d'un sistema operatiu. Comprendre com es creen, gestionen i planifiquen els processos és fonamental per a qualsevol professional de TI. Els conceptes d'estats de processos, PCB, context switch i els diferents algoritmes de planificació són essencials per entendre com els sistemes operatius gestionen la multitasca i optimitzen l'ús de recursos.

Aquest coneixement no només és teòric, sinó que també té aplicacions pràctiques en la programació i administració de sistemes, el que permet als professionals dissenyar i mantenir sistemes més eficients i robustos.
