Introducció

La gestió de l'emmagatzematge és una de les funcions crítiques d'un sistema operatiu. S'encarrega d'administrar l'espai d'emmagatzematge en dispositius com discos durs, SSDs i altres mitjans d'emmagatzematge. Aquest tema abasta com els sistemes operatius organitzen, gestionen i accedeixen a les dades emmagatzemades.

Conceptes Clau

  • Sistema de Fitxers: Estructura que un sistema operatiu utilitza per organitzar i gestionar fitxers en un dispositiu d'emmagatzematge.
  • Particions: Divisions lògiques d'un disc físic que permeten gestionar diferents sistemes de fitxers en un mateix dispositiu.
  • Blocs: Unitats mínimes d'emmagatzematge que el sistema operatiu pot gestionar.
  • Fragmentació: Problema que ocorre quan els fitxers es divideixen en parts no contigües, la qual cosa pot afectar el rendiment.
  • Journaling: Tècnica utilitzada per millorar la fiabilitat del sistema de fitxers mitjançant el registre de canvis pendents.

Sistemes de Fitxers

Els sistemes de fitxers són fonamentals per a la gestió de l'emmagatzematge. Alguns dels sistemes de fitxers més comuns inclouen:

| Sistema de Fitxers | Característiques Principals | Ús Comú | |--------------------|-----------------------------|---------| | FAT32 | Simplicitat, compatibilitat | Dispositius USB, targetes SD | | NTFS | Seguretat, suport per a fitxers grans | Windows | | ext4 | Rendiment, suport per a grans volums | Linux | | APFS | Optimització per a SSD, xifrat | macOS |

Exemple de Creació de Partició i Format en Linux

Per crear una partició i formatar-la amb el sistema de fitxers ext4 en Linux, es poden seguir els següents passos:

  1. Crear una nova partició:

    sudo fdisk /dev/sda
    • Dins de fdisk, usar els comandos n per crear una nova partició i w per escriure els canvis.

  2. Formatar la partició:

    sudo mkfs.ext4 /dev/sda1

  3. Muntar la partició:

    sudo mount /dev/sda1 /mnt

Gestió d'Espai

La gestió de l'espai en un sistema operatiu inclou diverses tècniques i estratègies per optimitzar l'ús de l'emmagatzematge.

  • Assignació Contigua: Assigna blocs contigus per a un fitxer, la qual cosa facilita l'accés seqüencial però pot portar a la fragmentació externa.
  • Assignació Enllaçada: Cada fitxer és una llista de blocs, la qual cosa elimina la fragmentació externa però pot ser més lent per a l'accés aleatori.
  • Assignació Indexada: Utilitza una estructura d'índex per mantenir una llista de blocs, combinant els avantatges de les dues tècniques anteriors.

Exemple de Fragmentació

Suposem que tenim un disc amb blocs numerats de l'1 al 10. Si un fitxer ocupa els blocs 1, 2, 3 i després un altre fitxer ocupa els blocs 4, 5, 6, i el primer fitxer s'expandeix i necessita més blocs, podria acabar ocupant els blocs 7, 8, 9, 10, resultant en fragmentació.

Tècniques d'Optimització

  • Desfragmentació: Procés de reorganitzar els blocs de fitxers perquè siguin contigus.
  • Emmagatzematge en Memòria Cau: Ús de memòria ràpida per emmagatzemar dades freqüentment accedides.
  • RAID (Redundant Array of Independent Disks): Combina múltiples discos per millorar la redundància i el rendiment.

Conclusió

La gestió de l'emmagatzematge és essencial per al rendiment i la fiabilitat d'un sistema operatiu. Comprendre com funcionen els sistemes de fitxers, les tècniques d'assignació i les estratègies d'optimització pot ajudar els professionals de TI a prendre decisions informades sobre la configuració i el manteniment de sistemes d'emmagatzematge.

Aquest tema proporciona una base sòlida per entendre com els sistemes operatius gestionen l'emmagatzematge, preparant els estudiants per enfrontar desafiaments més avançats en l'administració de sistemes.
