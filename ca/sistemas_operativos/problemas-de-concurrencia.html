Introducció

En els sistemes operatius moderns, la concurrència és un aspecte crucial que permet l'execució simultània de múltiples processos o fils. No obstant això, la concurrència també introdueix problemes complexos com les condicions de carrera i els deadlocks. Aquest tema abordarà aquests problemes, les seves causes, i com poden ser mitigats.

Condicions de carrera

Definició

Una condició de carrera ocorre quan el comportament d'un programari depèn de l'ordre en què s'executen les operacions dels fils o processos concurrents. Això pot portar a resultats inesperats i errors difícils de reproduir.

Exemple de condició de carrera

Considerem un comptador compartit entre dos fils:

#include <stdio.h>
#include <pthread.h>

int counter = 0;
pthread_mutex_t lock;

void* increment(void* arg) {
    for (int i = 0; i < 1000000; i++) {
        pthread_mutex_lock(&lock);
        counter++;
        pthread_mutex_unlock(&lock);
    }
    return NULL;
}

int main() {
    pthread_t t1, t2;
    pthread_mutex_init(&lock, NULL);

    pthread_create(&t1, NULL, increment, NULL);
    pthread_create(&t2, NULL, increment, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    printf("Final counter value: %d\n", counter);
    pthread_mutex_destroy(&lock);
    return 0;
}

Mitigació

  • Bloquejos (Locks): Utilitzar mecanismes de sincronització com mutexos per assegurar que només un fil pugui accedir a una secció crítica a la vegada.
  • Variables atòmiques: Utilitzar variables atòmiques que garanteixen operacions indivisibles.

Deadlocks

Definició

Un deadlock és una situació en la qual dos o més processos o fils estan bloquejats indefinidament, esperant recursos que estan sent retinguts per altres processos o fils en el mateix conjunt.

Exemple de deadlock

Considerem dos fils que intenten adquirir dos bloquejos en ordre invers:

#include <stdio.h>
#include <pthread.h>

pthread_mutex_t lock1;
pthread_mutex_t lock2;

void* thread1(void* arg) {
    pthread_mutex_lock(&lock1);
    sleep(1); // Simula alguna operació
    pthread_mutex_lock(&lock2);

    printf("Thread 1 acquired both locks\n");

    pthread_mutex_unlock(&lock2);
    pthread_mutex_unlock(&lock1);
    return NULL;
}

void* thread2(void* arg) {
    pthread_mutex_lock(&lock2);
    sleep(1); // Simula alguna operació
    pthread_mutex_lock(&lock1);

    printf("Thread 2 acquired both locks\n");

    pthread_mutex_unlock(&lock1);
    pthread_mutex_unlock(&lock2);
    return NULL;
}

int main() {
    pthread_t t1, t2;
    pthread_mutex_init(&lock1, NULL);
    pthread_mutex_init(&lock2, NULL);

    pthread_create(&t1, NULL, thread1, NULL);
    pthread_create(&t2, NULL, thread2, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    pthread_mutex_destroy(&lock1);
    pthread_mutex_destroy(&lock2);
    return 0;
}

Mitigació

  • Evitar l'espera circular: Assegurar-se que tots els processos adquireixin els recursos en el mateix ordre.
  • Detecció i recuperació: Implementar algorismes que detectin deadlocks i prenguin accions correctives.
  • Assignació de recursos: Utilitzar tècniques com l'algorisme del banquer per evitar l'assignació de recursos que pugui portar a un deadlock.

Comparació de Condicions de Carrera i Deadlocks

| Aspecte | Condició de Carrera | Deadlock | |------------------------|---------------------------------------------|-------------------------------------------| | Definició | Comportament depèn de l'ordre d'execució | Bloqueig indefinit esperant recursos | | Causa | Accés concurrent a recursos compartits | Espera circular de recursos | | Mitigació | Bloquejos, variables atòmiques | Evitar espera circular, detecció i recuperació | | Detecció | Difícil de reproduir i detectar | Algorismes específics poden detectar-lo |

Conclusió

Els problemes de concurrència com les condicions de carrera i els deadlocks són desafiaments crítics en el disseny de sistemes operatius i aplicacions concurrents. Comprendre aquests problemes i aplicar tècniques adequades per mitigar-los és essencial per desenvolupar programari robust i eficient.
