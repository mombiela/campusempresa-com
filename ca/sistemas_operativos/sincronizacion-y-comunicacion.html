Introducció

La sincronització i comunicació entre processos són aspectes fonamentals en els sistemes operatius. Aquests conceptes permeten que múltiples processos treballin de manera coordinada, evitant conflictes i garantint la integritat de les dades. En aquesta secció, explorarem els mecanismes i tècniques utilitzats per aconseguir una sincronització i comunicació efectiva entre processos.

Sincronització de Processos

La sincronització de processos és crucial per evitar condicions de carrera i assegurar que els processos comparteixin recursos de manera segura.

Conceptes Clau

  • Condició de carrera: Ocorre quan dos o més processos accedeixen i manipulen dades compartides simultàniament, la qual cosa pot portar a resultats inconsistents.
  • Secció crítica: Part del codi on el procés accedeix a recursos compartits.
  • Mutex (Mutual Exclusion): Mecanisme que assegura que només un procés pugui accedir a la secció crítica a la vegada.
  • Semàfors: Variables utilitzades per controlar l'accés a recursos compartits.

Exemple de Mutex en C

#include <pthread.h>
#include <stdio.h>

pthread_mutex_t lock;

void *thread_function(void *arg) {
    pthread_mutex_lock(&lock);
    // Secció crítica
    printf("Thread %d està en la secció crítica\n", *(int *)arg);
    pthread_mutex_unlock(&lock);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    pthread_mutex_init(&lock, NULL);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    pthread_mutex_destroy(&lock);
    return 0;
}

Exemple de Semàfor en C

#include <pthread.h>
#include <semaphore.h>
#include <stdio.h>

sem_t sem;

void *thread_function(void *arg) {
    sem_wait(&sem);
    // Secció crítica
    printf("Thread %d està en la secció crítica\n", *(int *)arg);
    sem_post(&sem);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    sem_init(&sem, 0, 1);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    sem_destroy(&sem);
    return 0;
}

Comunicació entre Processos (IPC)

La comunicació entre processos (IPC) permet que els processos intercanviïn dades i senyals. Existeixen diversos mecanismes d'IPC, cadascun amb els seus propis avantatges i desavantatges.

Mecanismes d'IPC

  • Pipes: Permeten la comunicació unidireccional entre processos relacionats.
  • FIFOs (Named Pipes): Similar als pipes, però permeten comunicació bidireccional i entre processos no relacionats.
  • Sockets: Permeten la comunicació entre processos en diferents màquines a través d'una xarxa.
  • Memòria compartida: Permet que múltiples processos accedeixin a una regió de memòria comuna.
  • Missatges: Permeten l'intercanvi de missatges entre processos.

Exemple de Pipe en C

#include <stdio.h>
#include <unistd.h>

int main() {
    int fd[2];
    pipe(fd);
    pid_t pid = fork();

    if (pid == 0) {
        // Procés fill
        close(fd[0]);
        char message[] = "Hola des del procés fill";
        write(fd[1], message, sizeof(message));
        close(fd[1]);
    } else {
        // Procés pare
        close(fd[1]);
        char buffer[100];
        read(fd[0], buffer, sizeof(buffer));
        printf("Missatge rebut: %s\n", buffer);
        close(fd[0]);
    }

    return 0;
}

Exemple de Memòria Compartida en C

#include <stdio.h>
#include <stdlib.h>
#include <sys/ipc.h>
#include <sys/shm.h>
#include <sys/types.h>

int main() {
    key_t key = ftok("shmfile", 65);
    int shmid = shmget(key, 1024, 0666 | IPC_CREAT);
    char *str = (char *)shmat(shmid, (void *)0, 0);

    if (fork() == 0) {
        // Procés fill
        printf("Escrivint en memòria compartida\n");
        sprintf(str, "Hola des del procés fill");
        shmdt(str);
    } else {
        // Procés pare
        wait(NULL);
        printf("Llegint de memòria compartida: %s\n", str);
        shmdt(str);
        shmctl(shmid, IPC_RMID, NULL);
    }

    return 0;
}

Conclusió

La sincronització i comunicació entre processos són components essencials en els sistemes operatius moderns. La sincronització assegura que els processos no interfereixin entre si en accedir a recursos compartits, mentre que la comunicació entre processos permet l'intercanvi de dades i senyals. Comprendre aquests conceptes i les seves implementacions pràctiques és crucial per al desenvolupament d'aplicacions robustes i eficients en entorns multitasca.
