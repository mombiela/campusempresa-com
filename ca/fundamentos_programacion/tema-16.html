Introducció

L'anàlisi d'algoritmes és una part fonamental de la programació que se centra a determinar l'eficiència i la viabilitat dels algoritmes. Aquesta anàlisi ens permet comparar diferents algoritmes i triar el més adequat per resoldre un problema específic.

Conceptes Clau

  • Algoritme: Un conjunt d'instruccions ben definides per resoldre un problema o realitzar una tasca.
  • Complexitat Temporal: Mesura el temps que triga un algoritme a executar-se en funció de la mida de l'entrada.
  • Complexitat Espacial: Mesura la quantitat de memòria que un algoritme necessita en funció de la mida de l'entrada.
  • Notació Big O: Una notació matemàtica que descriu el comportament asimptòtic d'un algoritme, és a dir, com es comporta quan la mida de l'entrada tendeix a infinit.

Complexitat Temporal

La complexitat temporal es classifica generalment en diverses categories, que es descriuen a continuació:

| Notació Big O | Nom | Exemple d'Algoritme | |---------------|------------------|-------------------------------| | O(1) | Constant | Accés a un element d'un array | | O(log n) | Logarítmica | Cerca binària | | O(n) | Lineal | Cerca lineal | | O(n log n) | Linealítmica | Ordenament ràpid (Quicksort) | | O(n^2) | Quadràtica | Ordenament per bombolla | | O(2^n) | Exponencial | Algoritmes de força bruta |

Exemple de Complexitat Temporal

Considerem dos algoritmes per trobar el número màxim en una llista de números.

Algoritme 1: Cerca Lineal

def trobar_maxim(llista):
    maxim = llista[0]
    for numero in llista:
        if numero > maxim:
            maxim = numero
    return maxim
  • Complexitat Temporal: O(n), on n és el nombre d'elements en la llista.

Algoritme 2: Cerca Binària (requereix llista ordenada)

def trobar_maxim_binari(llista):
    return llista[-1]  # Assumint que la llista està ordenada
  • Complexitat Temporal: O(1), ja que només accedeix a l'últim element de la llista.

Complexitat Espacial

La complexitat espacial es refereix a la quantitat de memòria addicional que un algoritme necessita per executar-se. Es classifica de manera similar a la complexitat temporal.

Exemple de Complexitat Espacial

Considerem dos algoritmes per calcular la sèrie de Fibonacci.

Algoritme 1: Recursiu

def fibonacci_recursiu(n):
    if n <= 1:
        return n
    else:
        return fibonacci_recursiu(n-1) + fibonacci_recursiu(n-2)
  • Complexitat Espacial: O(n), a causa de la pila de trucades recursives.

Algoritme 2: Iteratiu

def fibonacci_iteratiu(n):
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a
  • Complexitat Espacial: O(1), ja que només utilitza un espai constant addicional.

Notació Big O

La notació Big O s'utilitza per descriure l'eficiència d'un algoritme en termes de temps i espai. Aquí hi ha alguns exemples comuns:

  • O(1): Temps constant, no depèn de la mida de l'entrada.
  • O(n): Temps lineal, creix proporcionalment a la mida de l'entrada.
  • O(n^2): Temps quadràtic, creix proporcionalment al quadrat de la mida de l'entrada.

Conclusió

L'anàlisi d'algoritmes és crucial per desenvolupar programes eficients i escalables. Comprendre la complexitat temporal i espacial, així com la notació Big O, permet als programadors triar i dissenyar algoritmes que optimitzin el rendiment i l'ús de recursos. A mesura que avancis en la teva carrera de programació, aquestes habilitats t'ajudaran a abordar problemes més complexos de manera efectiva.
