Introducció

En aquest tema, explorarem dues estructures de dades fonamentals en la programació: les piles i les cues. Aquestes estructures són essencials per a la gestió de dades en molts algorismes i aplicacions. Aprendrem què són, com funcionen, i com implementar-les en codi.

Piles

Definició

Una pila és una estructura de dades que segueix el principi LIFO (Last In, First Out), és a dir, l'últim element en entrar és el primer en sortir.

Operacions Bàsiques

  • Push: Afegir un element al capdamunt de la pila.
  • Pop: Eliminar l'element del capdamunt de la pila.
  • Peek: Obtenir el valor de l'element al capdamunt sense eliminar-lo.
  • isEmpty: Verificar si la pila està buida.

Exemple de Codi en Python

class Pila:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        if not self.is_empty():
            return self.items.pop()
        else:
            return None

    def peek(self):
        if not self.is_empty():
            return self.items[-1]
        else:
            return None

    def is_empty(self):
        return len(self.items) == 0

# Exemple d'ús
pila = Pila()
pila.push(1)
pila.push(2)
print(pila.peek())  # Sortida: 2
print(pila.pop())   # Sortida: 2
print(pila.pop())   # Sortida: 1
print(pila.is_empty())  # Sortida: True

Aplicacions Comunes

  • Avaluació d'expressions: Utilitzades en l'avaluació d'expressions aritmètiques i lògiques.
  • Recursió: Les piles són fonamentals per a la implementació de crides recursives.
  • Desfer/refer: Funcionalitats en editors de text i altres programes.

Cues

Definició

Una cua és una estructura de dades que segueix el principi FIFO (First In, First Out), és a dir, el primer element en entrar és el primer en sortir.

Operacions Bàsiques

  • Enqueue: Afegir un element al final de la cua.
  • Dequeue: Eliminar el primer element de la cua.
  • Front: Obtenir el valor del primer element sense eliminar-lo.
  • isEmpty: Verificar si la cua està buida.

Exemple de Codi en Python

class Cua:
    def __init__(self):
        self.items = []

    def enqueue(self, item):
        self.items.append(item)

    def dequeue(self):
        if not self.is_empty():
            return self.items.pop(0)
        else:
            return None

    def front(self):
        if not self.is_empty():
            return self.items[0]
        else:
            return None

    def is_empty(self):
        return len(self.items) == 0

# Exemple d'ús
cua = Cua()
cua.enqueue(1)
cua.enqueue(2)
print(cua.front())  # Sortida: 1
print(cua.dequeue())  # Sortida: 1
print(cua.dequeue())  # Sortida: 2
print(cua.is_empty())  # Sortida: True

Aplicacions Comunes

  • Gestió de tasques: Utilitzades en sistemes operatius per gestionar processos.
  • Sistemes d'impressió: Maneig de treballs d'impressió en una impressora.
  • Xarxes: Gestió de paquets de dades en xarxes de comunicació.

Comparació entre Piles i Cues

| Característica | Pila (LIFO) | Cua (FIFO) | |----------------|-------------|-------------| | Ordre d'accés | Últim en entrar, primer en sortir | Primer en entrar, primer en sortir | | Operacions principals | Push, Pop, Peek, isEmpty | Enqueue, Dequeue, Front, isEmpty | | Exemples d'ús | Avaluació d'expressions, Recursió, Desfer/refer | Gestió de tasques, Sistemes d'impressió, Xarxes |

Conclusió

Les piles i les cues són estructures de dades fonamentals que tot programador ha de conèixer. Entendre com funcionen i com implementar-les és crucial per resoldre una varietat de problemes en la programació. Practicar amb exemples i exercicis ajudarà a consolidar aquests conceptes i a aplicar-los en situacions reals.
