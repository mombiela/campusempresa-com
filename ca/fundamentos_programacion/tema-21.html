Introducció

La depuració i el maneig d'errors són habilitats essencials per a qualsevol programador. La capacitat d'identificar, diagnosticar i corregir errors en el codi no només millora la qualitat del programari, sinó que també augmenta l'eficiència del desenvolupament.

Conceptes Clau

  • Depuració: Procés d'identificar i corregir errors o bugs en el codi.
  • Errors de Sintaxi: Errors que ocorren quan el codi no segueix les regles del llenguatge de programació.
  • Errors d'Execució: Errors que ocorren mentre el programa està en execució.
  • Errors Lògics: Errors que ocorren quan el programa no produeix el resultat esperat, encara que no hi hagi errors de sintaxi o execució.
  • Maneig d'Excepcions: Tècnica per manejar errors d'execució de manera controlada.

Eines de Depuració

  • Depuradors Integrats (IDEs): Eines com Visual Studio Code, PyCharm, Eclipse, etc., que inclouen funcionalitats de depuració.
  • Impressions de Consola: Ús de print en Python o console.log en JavaScript per rastrejar el flux del programa.
  • Breakpoints: Punts en el codi on l'execució es deté per inspeccionar l'estat del programa.

Tècniques de Depuració

Ús d'Impressions de Consola

Una tècnica bàsica però efectiva és inserir impressions en el codi per verificar l'estat de les variables i el flux del programa.

def suma(a, b):
    print(f"Valors rebuts: a={a}, b={b}")
    resultat = a + b
    print(f"Resultat de la suma: {resultat}")
    return resultat

suma(3, 4)

Ús de Breakpoints

Els breakpoints permeten detenir l'execució del programa en un punt específic per inspeccionar l'estat del programa.

  1. Establir un Breakpoint: Fer clic en el marge esquerre de l'editor de codi per establir un breakpoint.
  2. Executar en Mode Depuració: Iniciar el programa en mode depuració perquè es detingui en els breakpoints.
  3. Inspeccionar Variables: Revisar el valor de les variables en el punt d'interrupció.

Depuració Pas a Pas

Permet executar el codi línia per línia per observar com canvien les variables i el flux del programa.

  • Step Over: Executa la següent línia de codi, però no entra en funcions cridades.
  • Step Into: Entra en la següent funció cridada.
  • Step Out: Surt de la funció actual i torna al punt de crida.

Maneig d'Errors

Excepcions

Les excepcions són esdeveniments que ocorren durant l'execució del programa i que interrompen el seu flux normal.

Exemple en Python

try:
    resultat = 10 / 0
except ZeroDivisionError as e:
    print(f"Error: {e}")
finally:
    print("Aquesta línia s'executa sempre.")

Tipus Comuns d'Excepcions

| Llenguatge | Excepció Comuna | Descripció | |------------|-----------------|------------| | Python | ZeroDivisionError | Divisió per zero | | Java | NullPointerException | Accés a un objecte nul | | JavaScript | TypeError | Operació amb un tipus de dada incorrecte |

Creació d'Excepcions Personalitzades

En alguns casos, pot ser útil definir excepcions personalitzades per manejar errors específics de l'aplicació.

Exemple en Python

class CustomError(Exception):
    pass

try:
    raise CustomError("Aquest és un error personalitzat")
except CustomError as e:
    print(f"Error personalitzat: {e}")

Conclusió

La depuració i el maneig d'errors són components crítics en el desenvolupament de programari. Utilitzar eines de depuració i tècniques adequades pot estalviar temps i esforç, i manejar les excepcions de manera efectiva pot fer que el codi sigui més robust i fàcil de mantenir. Practicar aquestes habilitats és essencial per convertir-se en un programador competent i eficient.
