Introducció

En aquesta secció, explorarem els conceptes fonamentals de les bases de dades, essencials per a qualsevol professional de TI. Comprendre aquests conceptes és crucial per dissenyar, implementar i gestionar bases de dades de manera efectiva.

Què és una Base de Dades?

Una base de dades és una col·lecció organitzada de dades que s'emmagatzemen i s'accedeixen electrònicament. Les bases de dades permeten emmagatzemar grans quantitats d'informació de manera estructurada i eficient, facilitant la recuperació i manipulació de dades.

Tipus de Bases de Dades

Hi ha diversos tipus de bases de dades, cadascuna amb les seves característiques i usos específics:

  • Bases de Dades Relacionals (RDBMS): Utilitzen taules per emmagatzemar dades i relacions entre ells. Exemples: MySQL, PostgreSQL, Oracle.
  • Bases de Dades NoSQL: Dissenyades per manejar grans volums de dades no estructurades. Exemples: MongoDB, Cassandra, Redis.
  • Bases de Dades Orientades a Objectes: Emmagatzemen dades en forma d'objectes, com en la programació orientada a objectes. Exemples: db4o, ObjectDB.
  • Bases de Dades Distribuïdes: Les dades es distribueixen en diferents ubicacions físiques. Exemples: Google Spanner, Amazon DynamoDB.

Components d'una Base de Dades

Una base de dades típica consta de diversos components clau:

  • Taules: Estructures que emmagatzemen dades en files i columnes.
  • Registres: Files individuals en una taula que representen una entrada única.
  • Camps: Columnes en una taula que representen un atribut específic de les dades.
  • Índexs: Estructures que milloren la velocitat de recuperació de dades.
  • Consultes: Instruccions per recuperar dades específiques de la base de dades.

Model Relacional

El model relacional és el més comunament utilitzat en bases de dades. Es basa en la teoria de conjunts i relacions matemàtiques. Aquí hi ha alguns conceptes clau:

  • Entitat: Un objecte del món real que es pot identificar de manera única.
  • Atribut: Una propietat o característica d'una entitat.
  • Relació: Una associació entre dues o més entitats.

Exemple de Taula Relacional

| ID | Nom | Edat | Ciutat | |-----|----------|------|------------| | 1 | Joan | 28 | Madrid | | 2 | Anna | 34 | Barcelona | | 3 | Pere | 45 | València |

En aquesta taula, cada fila representa un registre únic, i cada columna representa un atribut de les dades.

Llenguatge de Consulta Estructurat (SQL)

SQL és el llenguatge estàndard per interactuar amb bases de dades relacionals. Permet realitzar diverses operacions com inserir, actualitzar, eliminar i consultar dades.

Exemples de Consultes SQL

  • Seleccionar Dades:

    SELECT Nom, Ciutat FROM Usuaris WHERE Edat > 30;

    Aquesta consulta selecciona els noms i ciutats dels usuaris la edat dels quals és superior a 30.

  • Inserir Dades:

    INSERT INTO Usuaris (Nom, Edat, Ciutat) VALUES ('Carles', 29, 'Sevilla');

    Aquesta consulta insereix un nou registre a la taula Usuaris.

  • Actualitzar Dades:

    UPDATE Usuaris SET Ciutat = 'Bilbao' WHERE Nom = 'Anna';

    Aquesta consulta actualitza la ciutat d'Anna a Bilbao.

  • Eliminar Dades:

    DELETE FROM Usuaris WHERE Nom = 'Pere';

    Aquesta consulta elimina el registre de Pere de la taula Usuaris.

Integritat de Dades

La integritat de dades assegura que les dades a la base de dades siguin precises i consistents. Hi ha diversos tipus d'integritat:

  • Integritat d'Entitat: Cada taula ha de tenir una clau primària única.
  • Integritat Referencial: Les claus externes han de coincidir amb les claus primàries corresponents.
  • Integritat de Domini: Els valors dels atributs han d'estar dins d'un rang permès.

Conclusió

Comprendre els conceptes bàsics de bases de dades és fonamental per a qualsevol professional de TI. Des dels tipus de bases de dades fins a l'ús de SQL i la integritat de dades, aquests conceptes formen la base per treballar eficaçment amb bases de dades. En les properes seccions, aprofundirem en cadascun d'aquests temes per proporcionar una comprensió més detallada i pràctica.
