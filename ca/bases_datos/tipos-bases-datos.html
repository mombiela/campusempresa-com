Tipus de Bases de Dades

Introducció

En el món de la tecnologia de la informació, les bases de dades són fonamentals per emmagatzemar, gestionar i recuperar dades de manera eficient. Hi ha diversos tipus de bases de dades, cadascuna dissenyada per satisfer diferents necessitats i casos d'ús. En aquesta secció, explorarem els tipus més comuns de bases de dades, les seves característiques i exemples d'ús.

Bases de Dades Relacionals (RDBMS)

Les bases de dades relacionals són les més comunes i es basen en el model relacional, que organitza les dades en taules amb files i columnes.

  • Característiques:

    • Utilitzen SQL (Structured Query Language) per a la gestió i consulta de dades.
    • Garanteixen la integritat referencial mitjançant claus primàries i foranes.
    • Suporten transaccions ACID (Atomicitat, Consistència, Aïllament, Durabilitat).

  • Exemples de RDBMS:

    • MySQL
    • PostgreSQL
    • Oracle Database
    • Microsoft SQL Server

  • Exemple de codi SQL:

CREATE TABLE empleats (
    id INT PRIMARY KEY,
    nom VARCHAR(100),
    lloc VARCHAR(50),
    salari DECIMAL(10, 2)
);

INSERT INTO empleats (id, nom, lloc, salari) VALUES (1, 'Joan Pérez', 'Desenvolupador', 50000.00);

SELECT * FROM empleats;

Bases de Dades NoSQL

Les bases de dades NoSQL estan dissenyades per manejar grans volums de dades no estructurades o semi-estructurades i ofereixen flexibilitat en el model de dades.

  • Tipus de Bases de Dades NoSQL:

    • Documentals: Emmagatzemen dades en documents JSON o BSON.
      • Exemple: MongoDB
    • Clau-Valor: Emmagatzemen dades com a parells clau-valor.
      • Exemple: Redis
    • Columnar: Emmagatzemen dades en columnes en lloc de files.
      • Exemple: Apache Cassandra
    • Grafs: Emmagatzemen dades en nodes i relacions.
      • Exemple: Neo4j

  • Exemple de codi MongoDB:

db.empleats.insertOne({
    id: 1,
    nom: "Joan Pérez",
    lloc: "Desenvolupador",
    salari: 50000.00
});

db.empleats.find();

Bases de Dades Orientades a Objectes

Aquestes bases de dades emmagatzemen dades en forma d'objectes, com en la programació orientada a objectes.

  • Característiques:

    • Integren conceptes de bases de dades i programació orientada a objectes.
    • Suporten herència, encapsulament i polimorfisme.

  • Exemples:

    • db4o
    • ObjectDB

Bases de Dades en Memòria

Aquestes bases de dades emmagatzemen dades en la memòria principal (RAM) per proporcionar accés ràpid i de baixa latència.

  • Característiques:

    • Alta velocitat de lectura i escriptura.
    • Adequades per a aplicacions que requereixen respostes en temps real.

  • Exemples:

    • Redis
    • Memcached

Bases de Dades Distribuïdes

Les bases de dades distribuïdes emmagatzemen dades en múltiples nodes o servidors, proporcionant alta disponibilitat i escalabilitat.

  • Característiques:

    • Dades replicades en diversos nodes.
    • Tolerància a fallades i alta disponibilitat.

  • Exemples:

    • Apache Cassandra
    • Google Spanner

Comparació de Tipus de Bases de Dades

| Tipus de Base de Dades | Exemples | Característiques Clau | |-----------------------------|------------------|------------------------------------------------------------| | Relacionals (RDBMS) | MySQL, PostgreSQL| Ús de SQL, integritat referencial, transaccions ACID | | NoSQL Documentals | MongoDB | Emmagatzematge en documents JSON/BSON | | NoSQL Clau-Valor | Redis | Parells clau-valor, alta velocitat | | NoSQL Columnar | Apache Cassandra | Emmagatzematge en columnes, escalabilitat | | NoSQL Grafs | Neo4j | Emmagatzematge en nodes i relacions | | Orientades a Objectes | db4o, ObjectDB | Integració amb programació orientada a objectes | | En Memòria | Redis, Memcached | Alta velocitat d'accés, emmagatzematge en RAM | | Distribuïdes | Apache Cassandra | Dades replicades, alta disponibilitat, tolerància a fallades|

Conclusió

Comprendre els diferents tipus de bases de dades i les seves característiques és essencial per seleccionar la solució adequada per a cada cas d'ús. Les bases de dades relacionals són ideals per a aplicacions amb dades estructurades i transaccions complexes, mentre que les bases de dades NoSQL ofereixen flexibilitat i escalabilitat per a dades no estructurades. Les bases de dades en memòria i distribuïdes proporcionen solucions per a aplicacions que requereixen alta velocitat i disponibilitat. En triar una base de dades, és important considerar els requisits específics de l'aplicació i les característiques de cada tipus de base de dades.
