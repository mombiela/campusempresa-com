Introducció als Diagrames ER

Els Diagrames Entitat-Relació (ER) són una eina fonamental en el disseny de bases de dades. Permeten visualitzar l'estructura lògica d'una base de dades de manera gràfica, facilitant la comprensió i el disseny de la mateixa.

  • Entitat: Representa un objecte o concepte del món real que té existència independent. Exemples: Client, Producte, Comanda.
  • Atribut: Propietat o característica d'una entitat. Exemples: nom, preu, data.
  • Relació: Associació entre dues o més entitats. Exemples: Un client realitza una comanda, un producte pertany a una categoria.

Components d'un Diagrama ER

Entitats

Les entitats són representades per rectangles i poden ser de dos tipus:

  • Entitats fortes: Tenen una existència independent. Exemple: Client.
  • Entitats febles: Depenen d'una altra entitat per a la seva existència. Exemple: Detall de Comanda.

Atributs

Els atributs són representats per òvals i es connecten a la seva entitat corresponent mitjançant una línia.

  • Atributs simples: No es poden dividir en subparts. Exemple: Número de telèfon.
  • Atributs compostos: Poden dividir-se en subparts. Exemple: Adreça (Carrer, Ciutat, Codi Postal).
  • Atributs multivaluats: Poden tenir múltiples valors. Exemple: Telèfons de contacte.
  • Atributs derivats: Es poden calcular a partir d'altres atributs. Exemple: Edat (a partir de la data de naixement).

Relacions

Les relacions són representades per rombes i connecten dues o més entitats.

  • Relacions binàries: Involucren dues entitats. Exemple: Un client realitza una comanda.
  • Relacions ternàries: Involucren tres entitats. Exemple: Un proveïdor subministra productes a una botiga.

Cardinalitat

La cardinalitat defineix el nombre d'instàncies d'una entitat que poden estar associades amb una instància d'una altra entitat.

  • Un a un (1:1): Una instància d'una entitat s'associa amb una instància d'una altra entitat.
  • Un a molts (1:N): Una instància d'una entitat s'associa amb múltiples instàncies d'una altra entitat.
  • Molts a molts (M:N): Múltiples instàncies d'una entitat s'associen amb múltiples instàncies d'una altra entitat.

Exemple de Diagrama ER

A continuació, es presenta un exemple d'un diagrama ER per a una base de dades d'una botiga en línia:

[Client]---(Realitza)---[Comanda]---(Conté)---[Producte]
   |                        |                        |
[ID_Client]              [ID_Comanda]             [ID_Producte]
[Nom]                    [Data]                   [Nom]
[Correu]                 [Total]                  [Preu]

En aquest exemple:

  • Entitats: Client, Comanda, Producte.
  • Relacions: Realitza, Conté.
  • Atributs: ID_Client, Nom, Correu, ID_Comanda, Data, Total, ID_Producte, Preu.

Exercici Pràctic

Dissenya un diagrama ER per a una base de dades d'una biblioteca. Considera les següents entitats i atributs:

  • Llibre: ISBN, Títol, Autor, Any de Publicació.
  • Usuari: ID_Usuari, Nom, Adreça, Telèfon.
  • Préstec: ID_Préstec, Data de Préstec, Data de Devolució.

Defineix les relacions entre aquestes entitats i la seva cardinalitat.

Conclusió

Els Diagrames Entitat-Relació són una eina essencial per al disseny de bases de dades. Permeten representar de manera clara i concisa les entitats, atributs i relacions, facilitant la comprensió i el desenvolupament de l'estructura lògica de la base de dades. Practicar el disseny de diagrames ER és fonamental per adquirir habilitats sòlides en la modelació de bases de dades.
