Introducció

El model relacional és un dels pilars fonamentals en el disseny i gestió de bases de dades. Proposat per Edgar F. Codd el 1970, aquest model organitza les dades en taules (o relacions) que permeten una manipulació eficient i una estructura clara.

Conceptes Clau

  • Relació: Una taula en una base de dades relacional.
  • Tupla: Una fila en una taula, que representa un registre únic.
  • Atribut: Una columna en una taula, que representa una característica del registre.
  • Clau Primària: Un atribut o conjunt d'atributs que identifiquen de manera única una tupla en una taula.
  • Clau Forana: Un atribut en una taula que és una clau primària en una altra taula, utilitzat per establir relacions entre taules.
  • Esquema de Relació: L'estructura d'una taula, definida pels seus atributs i els seus tipus de dades.

Estructura d'una Taula

Una taula en el model relacional es pot visualitzar com una matriu de files i columnes. Cada fila representa un registre únic i cada columna representa un atribut del registre.

Exemple d'una Taula

Suposem que tenim una taula anomenada Clients:

| ClientID | Nom | Cognom | Email | |----------|----------|----------|----------------------| | 1 | Joan | Pérez | joan.perez@mail.com | | 2 | Maria | Garcia | maria.garcia@mail.com| | 3 | Carles | López | carles.lopez@mail.com|

  • ClientID: Clau primària.
  • Nom: Atribut que emmagatzema el nom del client.
  • Cognom: Atribut que emmagatzema el cognom del client.
  • Email: Atribut que emmagatzema el correu electrònic del client.

Relacions entre Taules

Les relacions entre taules s'estableixen mitjançant claus foranes. Això permet la normalització de dades i evita la redundància.

Exemple de Relacions

Suposem que tenim una altra taula anomenada Comandes:

| ComandaID | ClientID | Data | Total | |-----------|----------|------------|--------| | 101 | 1 | 2023-01-15 | 150.00 | | 102 | 2 | 2023-01-16 | 200.00 | | 103 | 1 | 2023-01-17 | 100.00 |

  • ComandaID: Clau primària.
  • ClientID: Clau forana que referència a ClientID en la taula Clients.
  • Data: Atribut que emmagatzema la data de la comanda.
  • Total: Atribut que emmagatzema el total de la comanda.

Exemples de Codi SQL

Creació de Taules

CREATE TABLE Clients (
    ClientID INT PRIMARY KEY,
    Nom VARCHAR(50),
    Cognom VARCHAR(50),
    Email VARCHAR(100)
);

CREATE TABLE Comandes (
    ComandaID INT PRIMARY KEY,
    ClientID INT,
    Data DATE,
    Total DECIMAL(10, 2),
    FOREIGN KEY (ClientID) REFERENCES Clients(ClientID)
);

Inserció de Dades

INSERT INTO Clients (ClientID, Nom, Cognom, Email) VALUES
(1, 'Joan', 'Pérez', 'joan.perez@mail.com'),
(2, 'Maria', 'Garcia', 'maria.garcia@mail.com'),
(3, 'Carles', 'López', 'carles.lopez@mail.com');

INSERT INTO Comandes (ComandaID, ClientID, Data, Total) VALUES
(101, 1, '2023-01-15', 150.00),
(102, 2, '2023-01-16', 200.00),
(103, 1, '2023-01-17', 100.00);

Consultes Bàsiques

  • Seleccionar tots els clients:

    SELECT * FROM Clients;

  • Seleccionar totes les comandes d'un client específic:

    SELECT * FROM Comandes WHERE ClientID = 1;

Conclusió

El model relacional és una eina poderosa per a l'organització i gestió de dades en bases de dades. Comprendre els seus conceptes bàsics, com relacions, claus primàries i foranes, i com estructurar i manipular dades mitjançant SQL, és essencial per a qualsevol professional de TI. Aquest coneixement permet dissenyar sistemes de bases de dades eficients i escalables, facilitant la integritat i consistència de les dades.
