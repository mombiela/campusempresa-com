Introducció

En aquest cas d'estudi, explorarem com dissenyar i gestionar una base de dades relacional per a una petita empresa de comerç electrònic. Aquest exercici t'ajudarà a comprendre els conceptes fonamentals de les bases de dades relacionals i com aplicar-los en un escenari del món real.

Requisits del Sistema

Abans de començar amb el disseny de la base de dades, és crucial entendre els requisits del sistema. L'empresa de comerç electrònic necessita:

  • Gestió de Productes: Informació sobre els productes que venen, incloent nom, descripció, preu i estoc.
  • Gestió de Clients: Informació sobre els clients, incloent nom, adreça, correu electrònic i número de telèfon.
  • Gestió de Comandes: Informació sobre les comandes realitzades pels clients, incloent data de la comanda, productes comprats i quantitat.
  • Gestió de Pagaments: Informació sobre els pagaments realitzats pels clients, incloent mètode de pagament i estat del pagament.

Disseny de la Base de Dades

Entitats i Relacions

Per satisfer els requisits del sistema, identifiquem les següents entitats i les seves relacions:

  • Producte: Representa els articles que es venen.
  • Client: Representa els compradors.
  • Comanda: Representa les ordres de compra realitzades pels clients.
  • Pagament: Representa els pagaments realitzats pels clients.

Les relacions entre aquestes entitats són les següents:

  • Un client pot realitzar múltiples comandes.
  • Una comanda pot incloure múltiples productes.
  • Una comanda pot tenir un pagament associat.

Esquema de la Base de Dades

A continuació, es presenta l'esquema de la base de dades amb les taules i les seves relacions:

| Taula | Columnes | |----------|--------------------------------------------------------------------------| | Producte | id_producte (PK), nom, descripció, preu, estoc | | Client | id_client (PK), nom, adreça, correu_electronic, telèfon | | Comanda | id_comanda (PK), id_client (FK), data_comanda | | Comanda_Producte | id_comanda (FK), id_producte (FK), quantitat | | Pagament | id_pagament (PK), id_comanda (FK), metode_pagament, estat_pagament |

Diagrama Entitat-Relació (ERD)

Un diagrama ERD visualitza les entitats i les seves relacions:

Client (1) ---- (N) Comanda
Comanda (1) ---- (N) Comanda_Producte
Producte (1) ---- (N) Comanda_Producte
Comanda (1) ---- (1) Pagament

Implementació en SQL

A continuació, es mostra com crear les taules en SQL:

CREATE TABLE Producte (
    id_producte INT PRIMARY KEY,
    nom VARCHAR(100),
    descripció TEXT,
    preu DECIMAL(10, 2),
    estoc INT
);

CREATE TABLE Client (
    id_client INT PRIMARY KEY,
    nom VARCHAR(100),
    adreça TEXT,
    correu_electronic VARCHAR(100),
    telèfon VARCHAR(15)
);

CREATE TABLE Comanda (
    id_comanda INT PRIMARY KEY,
    id_client INT,
    data_comanda DATE,
    FOREIGN KEY (id_client) REFERENCES Client(id_client)
);

CREATE TABLE Comanda_Producte (
    id_comanda INT,
    id_producte INT,
    quantitat INT,
    PRIMARY KEY (id_comanda, id_producte),
    FOREIGN KEY (id_comanda) REFERENCES Comanda(id_comanda),
    FOREIGN KEY (id_producte) REFERENCES Producte(id_producte)
);

CREATE TABLE Pagament (
    id_pagament INT PRIMARY KEY,
    id_comanda INT,
    metode_pagament VARCHAR(50),
    estat_pagament VARCHAR(50),
    FOREIGN KEY (id_comanda) REFERENCES Comanda(id_comanda)
);

Exemples de Consultes

Inserir Dades

INSERT INTO Producte (id_producte, nom, descripció, preu, estoc) VALUES
(1, 'Laptop', 'Laptop de 15 polzades', 1200.00, 10),
(2, 'Mouse', 'Mouse sense fil', 25.00, 50);

INSERT INTO Client (id_client, nom, adreça, correu_electronic, telèfon) VALUES
(1, 'Joan Perez', 'Carrer Fals 123', 'joan.perez@example.com', '555-1234');

INSERT INTO Comanda (id_comanda, id_client, data_comanda) VALUES
(1, 1, '2023-10-01');

INSERT INTO Comanda_Producte (id_comanda, id_producte, quantitat) VALUES
(1, 1, 1),
(1, 2, 2);

INSERT INTO Pagament (id_pagament, id_comanda, metode_pagament, estat_pagament) VALUES
(1, 1, 'Targeta de Crèdit', 'Completat');

Consultar Dades

-- Obtenir tots els productes
SELECT * FROM Producte;

-- Obtenir totes les comandes d'un client específic
SELECT * FROM Comanda WHERE id_client = 1;

-- Obtenir detalls d'una comanda específica
SELECT p.nom, pp.quantitat
FROM Comanda_Producte pp
JOIN Producte p ON pp.id_producte = p.id_producte
WHERE pp.id_comanda = 1;

Conclusió

Aquest cas d'estudi ha demostrat com dissenyar i gestionar una base de dades relacional per a una empresa de comerç electrònic. Hem cobert des de la identificació de requisits fins a la implementació en SQL, proporcionant una visió completa del procés. Practicar amb aquests conceptes t'ajudarà a desenvolupar una comprensió sòlida de les bases de dades relacionals i la seva aplicació en escenaris del món real.
