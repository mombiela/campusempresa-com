Introducció

En aquest cas d'estudi, explorarem les bases de dades no relacionals, també conegudes com NoSQL. Aquestes bases de dades són essencials per manejar grans volums de dades no estructurades i semiestructurades, i són àmpliament utilitzades en aplicacions modernes com xarxes socials, anàlisi de big data i sistemes de recomanació.

Tipus de Bases de Dades No Relacionals

Hi ha diversos tipus de bases de dades NoSQL, cadascuna dissenyada per a casos d'ús específics:

  • Documentals: Emmagatzemen dades en documents similars a JSON.
  • Clau-Valor: Emmagatzemen dades com a parells clau-valor.
  • Columnar: Emmagatzemen dades en columnes en lloc de files.
  • Grafos: Emmagatzemen dades en nodes i arestes, ideals per representar relacions complexes.

Comparació de Tipus de Bases de Dades No Relacionals

| Tipus | Exemple de Base de Dades | Cas d'Ús Principal | |----------------|--------------------------|----------------------------------------| | Documentals | MongoDB | Aplicacions web, catàlegs de productes | | Clau-Valor | Redis | Caché, sessions d'usuari | | Columnar | Cassandra | Anàlisi de big data, registres d'esdeveniments | | Grafos | Neo4j | Xarxes socials, sistemes de recomanació |

Cas d'Estudi: Implementació de MongoDB

Descripció del Projecte

Suposem que estem desenvolupant una aplicació de comerç electrònic que necessita emmagatzemar informació sobre productes, usuaris i comandes. MongoDB és una excel·lent opció a causa de la seva flexibilitat i capacitat per manejar dades semiestructurades.

Estructura de Dades

En MongoDB, les dades s'emmagatzemen en documents dins de col·leccions. A continuació, es mostra un exemple de com podríem estructurar els documents per a la nostra aplicació:

Col·lecció de Productes

{
  "_id": "12345",
  "nom": "Samarreta",
  "preu": 19.99,
  "categoria": "Roba",
  "stock": 100,
  "descripció": "Samarreta de cotó 100%."
}

Col·lecció d'Usuaris

{
  "_id": "67890",
  "nom": "Joan Pérez",
  "email": "joan.perez@example.com",
  "adreça": {
    "carrer": "Carrer Falsa 123",
    "ciutat": "Ciutat Fictícia",
    "codi_postal": "12345"
  }
}

Col·lecció de Comandes

{
  "_id": "54321",
  "usuari_id": "67890",
  "productes": [
    {
      "producte_id": "12345",
      "quantitat": 2
    }
  ],
  "data": "2023-10-01",
  "total": 39.98
}

Operacions Bàsiques

Inserir Documents

db.productes.insertOne({
  "_id": "12345",
  "nom": "Samarreta",
  "preu": 19.99,
  "categoria": "Roba",
  "stock": 100,
  "descripció": "Samarreta de cotó 100%."
});

Consultar Documents

db.productes.find({ "categoria": "Roba" });

Actualitzar Documents

db.productes.updateOne(
  { "_id": "12345" },
  { $set: { "preu": 17.99 } }
);

Eliminar Documents

db.productes.deleteOne({ "_id": "12345" });

Avantatges i Desavantatges de MongoDB

Avantatges

  • Flexibilitat: Permet emmagatzemar dades semiestructurades.
  • Escalabilitat: Fàcil d'escalar horitzontalment.
  • Rendiment: Alta velocitat de lectura i escriptura.

Desavantatges

  • Consistència: Pot no ser tan consistent com les bases de dades relacionals.
  • Consultes Complexes: Menys eficient per a consultes molt complexes.
  • Transaccions: Suport limitat per a transaccions ACID.

Conclusió

Les bases de dades no relacionals, com MongoDB, ofereixen una gran flexibilitat i escalabilitat per manejar dades semiestructurades i no estructurades. Són ideals per a aplicacions modernes que requereixen un rendiment ràpid i la capacitat d'escalar horitzontalment. No obstant això, és important considerar les seves limitacions en termes de consistència i suport per a transaccions complexes.

Aquest cas d'estudi proporciona una visió general de com implementar i utilitzar MongoDB en una aplicació de comerç electrònic, destacant els seus avantatges i desavantatges.
