Introducció a la Desnormalització

La desnormalització és el procés d'optimització d'una base de dades relacional en el qual s'afegeixen dades redundants per millorar el rendiment de les consultes. Encara que la normalització és crucial per eliminar redundàncies i assegurar la integritat de les dades, en alguns casos, la desnormalització pot ser necessària per millorar l'eficiència de les operacions de lectura.

Conceptes Clau

  • Normalització: Procés d'estructuració d'una base de dades per reduir la redundància i millorar la integritat de les dades.
  • Desnormalització: Procés d'afegir redundància a una base de dades per millorar el rendiment de les consultes.
  • Redundància: Duplicació de dades en una base de dades.
  • Integritat de les Dades: Garantia que les dades són precises i consistents.

Avantatges de la Desnormalització

  • Millora del Rendiment de Consultes: En reduir la quantitat d'unions (joins) necessàries en les consultes, es pot millorar significativament el temps de resposta.
  • Simplificació de Consultes: Les consultes poden ser més simples i fàcils d'escriure i mantenir.
  • Reducció de Càrregues de Treball: Menys operacions de lectura i escriptura en taules relacionades poden reduir la càrrega en el sistema.

Desavantatges de la Desnormalització

  • Augment de la Redundància: Pot portar a la duplicació de dades, la qual cosa augmenta l'espai d'emmagatzematge necessari.
  • Complexitat en el Manteniment: L'actualització de dades redundants pot ser més complexa i propensa a errors.
  • Risc d'Inconsistències: La duplicació de dades pot portar a inconsistències si no es gestionen adequadament les actualitzacions.

Exemples de Desnormalització

Exemple 1: Desnormalització d'una Taula de Comandes

Suposem que tenim una base de dades d'una botiga en línia amb les següents taules normalitzades:

  • Clients: Informació sobre els clients.
  • Comandes: Informació sobre les comandes realitzades.
  • Productes: Informació sobre els productes disponibles.

En una base de dades normalitzada, podríem tenir les següents taules:

CREATE TABLE Clients (
    ClientID INT PRIMARY KEY,
    Nom VARCHAR(100),
    Email VARCHAR(100)
);

CREATE TABLE Productes (
    ProducteID INT PRIMARY KEY,
    Nom VARCHAR(100),
    Preu DECIMAL(10, 2)
);

CREATE TABLE Comandes (
    ComandaID INT PRIMARY KEY,
    ClientID INT,
    Data DATE,
    FOREIGN KEY (ClientID) REFERENCES Clients(ClientID)
);

CREATE TABLE DetallsComanda (
    DetallID INT PRIMARY KEY,
    ComandaID INT,
    ProducteID INT,
    Quantitat INT,
    FOREIGN KEY (ComandaID) REFERENCES Comandes(ComandaID),
    FOREIGN KEY (ProducteID) REFERENCES Productes(ProducteID)
);

Per millorar el rendiment de les consultes, podríem desnormalitzar la taula de comandes per incloure informació del client i del producte directament en la taula de detalls de la comanda:

CREATE TABLE DetallsComandaDesnormalitzat (
    DetallID INT PRIMARY KEY,
    ComandaID INT,
    ClientID INT,
    NomClient VARCHAR(100),
    EmailClient VARCHAR(100),
    ProducteID INT,
    NomProducte VARCHAR(100),
    PreuProducte DECIMAL(10, 2),
    Quantitat INT
);

Exemple 2: Desnormalització en un Sistema de Reports

En un sistema de reports, és comú desnormalitzar dades per millorar el rendiment de les consultes agregades. Per exemple, en lloc de calcular el total de vendes en temps real, podríem emmagatzemar el total de vendes pre-calculat en una taula desnormalitzada.

CREATE TABLE VendesMensuals (
    Mes INT,
    Any INT,
    TotalVendes DECIMAL(15, 2)
);

Comparació: Normalització vs Desnormalització

| Aspecte | Normalització | Desnormalització | |------------------------|----------------------------------------|----------------------------------------| | Redundància | Baixa | Alta | | Integritat de Dades | Alta | Baixa | | Complexitat de Consultes | Alta | Baixa | | Rendiment de Consultes | Baix (en consultes complexes) | Alt (en consultes simples) | | Manteniment | Senzill | Complex |

Conclusió

La desnormalització és una tècnica útil per millorar el rendiment de les consultes en bases de dades relacionals, especialment en sistemes on les operacions de lectura són més freqüents que les d'escriptura. No obstant això, és important balancejar els avantatges i desavantatges, ja que la desnormalització pot portar a redundàncies i complicacions en el manteniment de les dades. La decisió de desnormalitzar ha de basar-se en una anàlisi acurada dels requisits de rendiment i l'estructura de la base de dades.
