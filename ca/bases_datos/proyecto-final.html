Descripció del Projecte

El projecte final del curs de Fonaments de Bases de Dades té com a objectiu aplicar els coneixements adquirits al llarg del curs en un projecte pràctic. Els estudiants hauran de dissenyar, implementar i consultar una base de dades relacional que resolgui un problema específic. Aquest projecte permetrà als estudiants demostrar la seva comprensió dels conceptes clau de bases de dades, així com les seves habilitats en disseny i consulta de bases de dades.

Requisits del Projecte

  • Tema del Projecte: Els estudiants han de triar un tema per a la seva base de dades. Pot ser qualsevol tema que els interessi, com una base de dades per a una biblioteca, un sistema de gestió d'inventaris, una xarxa social, etc.
  • Disseny de la Base de Dades:
    • Crear un diagrama entitat-relació (ER) que representi les entitats, relacions i atributs de la base de dades.
    • Normalitzar les taules fins a la tercera forma normal (3NF).
  • Implementació:
    • Crear les taules en un sistema de gestió de bases de dades (DBMS) com MySQL, PostgreSQL o SQLite.
    • Inserir dades de prova a les taules.
  • Consultes:
    • Escriure i executar consultes SQL per recuperar informació de la base de dades.
    • Incloure consultes que utilitzin SELECT, JOIN, GROUP BY, HAVING i ORDER BY.

Lliurables

  1. Document de Disseny:
    • Diagrama ER.
    • Descripció de les entitats i relacions.
    • Justificació de la normalització.
  2. Script SQL:
    • Codi per crear les taules.
    • Codi per inserir dades de prova.
    • Consultes SQL.
  3. Informe Final:
    • Descripció del tema triat.
    • Explicació del disseny de la base de dades.
    • Resultats de les consultes SQL.
    • Conclusions i aprenentatges.

Exemple de Projecte

Tema: Sistema de Gestió de Biblioteca

Diagrama ER

Descripció d'Entitats i Relacions

  • Entitat Llibre:
    • Atributs: ISBN, Títol, Autor, Any de Publicació, Gènere
  • Entitat Usuari:
    • Atributs: ID Usuari, Nom, Correu Electrònic, Data de Registre
  • Entitat Préstec:
    • Atributs: ID Préstec, ID Usuari, ISBN, Data de Préstec, Data de Devolució
  • Relacions:
    • Un Usuari pot prendre en préstec múltiples Llibres.
    • Un Llibre pot ser prestat a múltiples Usuaris en diferents moments.

Script SQL

-- Creació de taules
CREATE TABLE Llibre (
    ISBN VARCHAR(13) PRIMARY KEY,
    Titol VARCHAR(255) NOT NULL,
    Autor VARCHAR(255) NOT NULL,
    Any_Publicacio INT,
    Genere VARCHAR(100)
);

CREATE TABLE Usuari (
    ID_Usuari INT PRIMARY KEY AUTO_INCREMENT,
    Nom VARCHAR(255) NOT NULL,
    Correu_Electronic VARCHAR(255) UNIQUE NOT NULL,
    Data_Registro DATE NOT NULL
);

CREATE TABLE Prestec (
    ID_Prestec INT PRIMARY KEY AUTO_INCREMENT,
    ID_Usuari INT,
    ISBN VARCHAR(13),
    Data_Prestec DATE NOT NULL,
    Data_Devolucio DATE,
    FOREIGN KEY (ID_Usuari) REFERENCES Usuari(ID_Usuari),
    FOREIGN KEY (ISBN) REFERENCES Llibre(ISBN)
);

-- Inserció de dades de prova
INSERT INTO Llibre (ISBN, Titol, Autor, Any_Publicacio, Genere) VALUES

INSERT INTO Prestec (ID_Usuari, ISBN, Data_Prestec, Data_Devolucio) VALUES
(1, '978-3-16-148410-0', '2023-03-01', '2023-03-15'),
(2, '978-0-14-044913-6', '2023-03-05', NULL);

-- Consultes SQL
-- 1. Obtenir tots els llibres prestats actualment
SELECT L.Titol, U.Nom, P.Data_Prestec
FROM Prestec P
JOIN Llibre L ON P.ISBN = L.ISBN
JOIN Usuari U ON P.ID_Usuari = U.ID_Usuari
WHERE P.Data_Devolucio IS NULL;

-- 2. Comptar el nombre de préstecs per usuari
SELECT U.Nom, COUNT(P.ID_Prestec) AS Numero_Prestecs
FROM Usuari U
JOIN Prestec P ON U.ID_Usuari = P.ID_Usuari
GROUP BY U.Nom
ORDER BY Numero_Prestecs DESC;
