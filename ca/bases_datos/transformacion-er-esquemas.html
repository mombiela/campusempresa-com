Introducció

En aquesta secció, aprendrem com transformar un Diagrama d'Entitat-Relació (ER) en un esquema relacional. Aquest procés és fonamental per dissenyar bases de dades relacionals a partir de models conceptuals, assegurant que l'estructura de la base de dades sigui eficient i coherent.

Conceptes Clau

  • Diagrama ER: Representació gràfica d'entitats i les seves relacions.
  • Esquema Relacional: Conjunt de taules que representen les entitats i relacions d'un diagrama ER en una base de dades relacional.
  • Entitat: Objecte del món real que es distingeix d'altres objectes.
  • Atribut: Propietat o característica d'una entitat.
  • Clau Primària: Atribut o conjunt d'atributs que identifiquen de manera única una entitat.
  • Clau Forana: Atribut en una taula que es refereix a la clau primària d'una altra taula.

Passos per a la Transformació

1. Transformació d'Entitats Simples

Cada entitat simple en el diagrama ER es converteix en una taula en l'esquema relacional.

  • Exemple:

    • Entitat Estudiant amb atributs ID, Nom, Edat.

    • Taula resultant:

      CREATE TABLE Estudiant (
    ID INT PRIMARY KEY,
    Nom VARCHAR(100),
    Edat INT
);

2. Transformació de Relacions

Les relacions entre entitats es transformen en taules addicionals o s'incorporen en les taules existents mitjançant claus foranes.

Relacions Un a Un (1:1)

  • Exemple:

    • Entitats Persona i Passaport amb una relació 1:1.

    • Taula resultant:

      CREATE TABLE Persona (
    PersonaID INT PRIMARY KEY,
    Nom VARCHAR(100)
);

CREATE TABLE Passaport (
    PassaportID INT PRIMARY KEY,
    PersonaID INT,
    DataExpiracio DATE,
    FOREIGN KEY (PersonaID) REFERENCES Persona(PersonaID)
);

Relacions Un a Molts (1:N)

  • Exemple:

    • Entitats Professor i Curs amb una relació 1:N.

    • Taula resultant:

      CREATE TABLE Professor (
    ProfessorID INT PRIMARY KEY,
    Nom VARCHAR(100)
);

CREATE TABLE Curs (
    CursID INT PRIMARY KEY,
    Nom VARCHAR(100),
    ProfessorID INT,
    FOREIGN KEY (ProfessorID) REFERENCES Professor(ProfessorID)
);

Relacions Molts a Molts (M:N)

  • Exemple:

    • Entitats Estudiant i Curs amb una relació M:N.

    • Taula resultant:

      CREATE TABLE Estudiant (
    EstudiantID INT PRIMARY KEY,
    Nom VARCHAR(100)
);

CREATE TABLE Curs (
    CursID INT PRIMARY KEY,
    Nom VARCHAR(100)
);

CREATE TABLE EstudiantCurs (
    EstudiantID INT,
    CursID INT,
    PRIMARY KEY (EstudiantID, CursID),
    FOREIGN KEY (EstudiantID) REFERENCES Estudiant(EstudiantID),
    FOREIGN KEY (CursID) REFERENCES Curs(CursID)
);

3. Transformació d'Atributs Compostos

Els atributs compostos es descomponen en atributs simples.

  • Exemple:

    • Atribut compost Adreça amb sub-atributs Carrer, Ciutat, Codi Postal.

    • Taula resultant:

      CREATE TABLE Client (
    ClientID INT PRIMARY KEY,
    Nom VARCHAR(100),
    Carrer VARCHAR(100),
    Ciutat VARCHAR(100),
    CodiPostal VARCHAR(10)
);

4. Transformació d'Atributs Multivaluats

Els atributs multivaluats es transformen en taules separades.

  • Exemple:

    • Entitat Empleat amb atribut multivaluat Telèfons.

    • Taula resultant:

      CREATE TABLE Empleat (
    EmpleatID INT PRIMARY KEY,
    Nom VARCHAR(100)
);

CREATE TABLE Telefon (
    EmpleatID INT,
    NumeroTelefon VARCHAR(15),
    PRIMARY KEY (EmpleatID, NumeroTelefon),
    FOREIGN KEY (EmpleatID) REFERENCES Empleat(EmpleatID)
);

Exercici Pràctic

Transforma el següent diagrama ER en un esquema relacional:

  • Entitats: Autor (ID, Nom), Llibre (ISBN, Títol, Any), Editorial (ID, Nom).
  • Relacions:
    • Autor escriu Llibre (M:N).
    • Llibre és publicat per Editorial (N:1).

Conclusió

La transformació de diagrames ER a esquemes relacionals és un pas crucial en el disseny de bases de dades. Aquest procés assegura que l'estructura de la base de dades sigui lògica i eficient, facilitant la implementació i el manteniment de la base de dades. Practicar aquests passos amb diferents diagrames ER ajudarà a consolidar aquests conceptes i habilitats.
