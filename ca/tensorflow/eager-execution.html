L'Execució Eager és un entorn de programació imperativa que avalua les operacions immediatament, sense construir gràfics. Això fa que sigui més fàcil començar amb TensorFlow i depurar models, ja que es comporta més com el codi estàndard de Python.

Introducció a l'Execució Eager

  • Programació Imperativa: L'Execució Eager permet que les operacions de TensorFlow s'executin immediatament quan es criden des de Python.
  • Facilitat de Depuració: Com que les operacions s'executen immediatament, pots utilitzar eines de depuració estàndard de Python com pdb.
  • Desenvolupament Interactiu: L'Execució Eager és particularment útil en entorns interactius com els notebooks de Jupyter.

Activació de l'Execució Eager

Per defecte, TensorFlow 2.x té l'Execució Eager activada. No obstant això, si estàs utilitzant TensorFlow 1.x, cal activar-la explícitament.

TensorFlow 2.x

import tensorflow as tf

# L'Execució Eager està activada per defecte en TensorFlow 2.x
print(tf.executing_eagerly())  # Hauria d'imprimir: True

TensorFlow 1.x

import tensorflow as tf

# Activar l'Execució Eager
tf.enable_eager_execution()

print(tf.executing_eagerly())  # Hauria d'imprimir: True

Operacions Bàsiques amb l'Execució Eager

Amb l'Execució Eager, les operacions de TensorFlow retornen valors concrets en lloc de construir un gràfic computacional.

Exemple: Operacions Aritmètiques Bàsiques

import tensorflow as tf

# Definir constants
a = tf.constant(2)
b = tf.constant(3)

# Realitzar operacions
add = tf.add(a, b)
multiply = tf.multiply(a, b)

print(f"Addició: {add.numpy()}")  # Sortida: 5
print(f"Multiplicació: {multiply.numpy()}")  # Sortida: 6

Exemple: Operacions amb Matrius

import tensorflow as tf

# Definir matrius
matrix1 = tf.constant([[1, 2], [3, 4]])
matrix2 = tf.constant([[5, 6], [7, 8]])

# Realitzar multiplicació de matrius
product = tf.matmul(matrix1, matrix2)

print(f"Producte de Matrius:\n{product.numpy()}")
# Sortida:
# [[19 22]
#  [43 50]]

Beneficis de l'Execució Eager

  • Sintaxi Intuïtiva: El codi sembla i es comporta com el codi estàndard de Python.
  • Execució Immediata: No cal compilar un gràfic abans d'executar operacions.
  • Facilitat de Depuració: Utilitza directament les eines de depuració de Python.
  • Desenvolupament Interactiu: Ideal per a entorns interactius com els notebooks de Jupyter.

Limitacions de l'Execució Eager

  • Rendiment: L'Execució Eager pot ser més lenta que l'execució de gràfics per a models a gran escala.
  • Desplegament: Algunes eines i entorns de desplegament poden preferir encara l'execució de gràfics.

Transició de l'Execució Eager a l'Execució de Gràfics

Per a aplicacions crítiques en rendiment, potser voldràs convertir el teu codi d'Execució Eager a execució de gràfics utilitzant tf.function.

Exemple: Utilitzant `tf.function`

import tensorflow as tf

@tf.function
def add(a, b):
    return a + b

a = tf.constant(2)
b = tf.constant(3)

result = add(a, b)
print(f"Resultat: {result.numpy()}")  # Sortida: 5

Resum

L'Execució Eager en TensorFlow proporciona una manera intuïtiva i interactiva de desenvolupar i depurar models. Està activada per defecte en TensorFlow 2.x i es pot activar explícitament en TensorFlow 1.x. Tot i que ofereix molts beneficis, incloent facilitat d'ús i execució immediata, pot no ser adequada per a totes les aplicacions crítiques en rendiment. Per a aquests casos, la transició a l'execució de gràfics utilitzant tf.function pot proporcionar les millores de rendiment necessàries.
