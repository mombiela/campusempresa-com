Introducció

La gestió d'identitats és un component crucial en les arquitectures distribuïdes. Es refereix a l'administració de la informació dels usuaris i els seus permisos dins d'un sistema. Aquest tema abasta des de la creació i manteniment d'identitats fins a l'autenticació i autorització d'usuaris.

Conceptes Clau

  • Identitat Digital: Representació electrònica de la informació d'un usuari.
  • Autenticació: Procés de verificar la identitat d'un usuari.
  • Autorització: Procés de determinar els permisos d'un usuari autenticat.
  • Single Sign-On (SSO): Mecanisme que permet als usuaris autenticar-se una sola vegada i accedir a múltiples sistemes.
  • Federació d'Identitats: Permet als usuaris utilitzar les mateixes credencials en diferents dominis de seguretat.
  • Gestió d'Accés: Control de qui té accés a quins recursos i en quines condicions.

Autenticació

L'autenticació és el primer pas en la gestió d'identitats. A continuació, es presenten alguns mètodes comuns d'autenticació:

  • Contrasenyes: Mètode més comú, encara que menys segur.
  • Autenticació de Dos Factors (2FA): Combina alguna cosa que l'usuari sap (contrasenya) amb alguna cosa que l'usuari té (token).
  • Autenticació Biomètrica: Utilitza característiques físiques de l'usuari, com empremtes dactilars o reconeixement facial.

Exemple de Codi: Autenticació Bàsica amb Contrasenya

def authenticate(username, password):
    stored_password = get_stored_password(username)
    if stored_password == password:
        return True
    else:
        return False

def get_stored_password(username):
    # Simulació d'una base de dades d'usuaris
    user_db = {
        'user1': 'password123',
        'user2': 'securepassword'
    }
    return user_db.get(username, None)

# Exemple d'ús
username = 'user1'
password = 'password123'
if authenticate(username, password):
    print("Autenticació exitosa")
else:
    print("Autenticació fallida")

Autorització

Un cop autenticat l'usuari, és necessari determinar quines accions pot realitzar. Els mètodes comuns d'autorització inclouen:

  • Llistes de Control d'Accés (ACL): Especifiquen quins usuaris o grups poden accedir a certs recursos.
  • Rols Basats en Accés (RBAC): Assigna permisos a rols en lloc d'usuaris individuals.
  • Polítiques Basades en Atributs (ABAC): Utilitza atributs de l'usuari i de l'entorn per prendre decisions d'accés.

Exemple de Codi: Autorització Basada en Rols

def authorize(user_role, resource):
    role_permissions = {
        'admin': ['read', 'write', 'delete'],
        'user': ['read', 'write'],
        'guest': ['read']
    }
    return resource in role_permissions.get(user_role, [])

# Exemple d'ús
user_role = 'user'
resource = 'write'
if authorize(user_role, resource):
    print("Accés concedit")
else:
    print("Accés denegat")

Single Sign-On (SSO)

El SSO permet als usuaris autenticar-se una vegada i accedir a múltiples aplicacions sense necessitat de tornar a autenticar-se. Això s'aconsegueix mitjançant protocols com OAuth, SAML i OpenID Connect.

Comparació de Protocols SSO

| Protocol | Descripció | Ús Comú | |----------------|--------------------------------------------------|---------------------------------| | OAuth | Protocol d'autorització | API i aplicacions mòbils | | SAML | Protocol d'autenticació i autorització | Aplicacions empresarials | | OpenID Connect | Extensió d'OAuth per a autenticació | Aplicacions web i mòbils |

Federació d'Identitats

La federació d'identitats permet als usuaris utilitzar les mateixes credencials en diferents dominis de seguretat. Això és comú en entorns empresarials on els usuaris necessiten accedir a recursos en múltiples organitzacions.

Gestió d'Accés

La gestió d'accés implica la implementació de polítiques i mecanismes per controlar qui té accés a quins recursos i en quines condicions. Això pot incloure:

  • Polítiques de Contrasenyes: Regles sobre la complexitat i expiració de contrasenyes.
  • Auditories d'Accés: Registre i revisió d'accessos per detectar i prevenir activitats no autoritzades.
  • Gestió de Sessions: Control de la durada i validesa de les sessions d'usuari.

Conclusió

La gestió d'identitats és fonamental per a la seguretat i eficiència en les arquitectures distribuïdes. Comprendre els conceptes d'autenticació, autorització, SSO, federació d'identitats i gestió d'accés és essencial per dissenyar sistemes segurs i escalables. A través d'exemples pràctics i comparacions de protocols, hem explorat com implementar i gestionar identitats de manera efectiva.
