Introducció

El monitoratge de sistemes distribuïts és una pràctica essencial per garantir la disponibilitat, rendiment i seguretat de les aplicacions que operen en un entorn distribuït. Aquest tema cobreix els conceptes fonamentals, eines i millors pràctiques per monitorar sistemes distribuïts de manera efectiva.

Conceptes Clau

  • Sistemes Distribuïts: Conjunt d'ordinadors independents que semblen un únic sistema coherent per als usuaris.
  • Monitoratge: Procés de recopilar, analitzar i visualitzar dades sobre el rendiment i estat d'un sistema.
  • Mètriques: Dades quantitatives que reflecteixen el rendiment del sistema, com latència, rendiment, ús de CPU, etc.
  • Alertes: Notificacions automàtiques que es generen quan una mètrica supera un llindar predefinit.
  • Logs: Registres detallats d'esdeveniments que ocorren en el sistema, útils per al diagnòstic i resolució de problemes.

Eines de Monitoratge

Existeixen diverses eines per al monitoratge de sistemes distribuïts, cadascuna amb les seves pròpies característiques i avantatges.

Prometheus

  • Descripció: Sistema de monitoratge i alerta de codi obert dissenyat per a la fiabilitat i escalabilitat.
  • Característiques:
    • Recol·lecció de mètriques mitjançant "scraping".
    • Llenguatge de consulta PromQL.
    • Integració amb Grafana per a visualització.
  • Exemple de Configuració: 
    global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'example'
    static_configs:
      - targets: ['localhost:9090']

Grafana

  • Descripció: Plataforma de codi obert per a la visualització i anàlisi de mètriques.
  • Característiques:
    • Suport per a múltiples fonts de dades.
    • Dashboards personalitzables.
    • Alertes basades en regles.
  • Exemple de Dashboard: 
    {
  "dashboard": {
    "id": null,
    "title": "Example Dashboard",
    "panels": [
      {
        "type": "graph",
        "title": "CPU Usage",
        "targets": [
          {
            "expr": "rate(node_cpu_seconds_total[5m])",
            "legendFormat": "{{cpu}}"
          }
        ]
      }
    ]
  }
}

ELK Stack (Elasticsearch, Logstash, Kibana)

  • Descripció: Conjunt d'eines per a la cerca, anàlisi i visualització de logs.
  • Característiques:
    • Elasticsearch per a emmagatzematge i cerca.
    • Logstash per a processament de logs.
    • Kibana per a visualització.
  • Exemple de Configuració de Logstash: 
    input {
  file {
    path => "/var/log/syslog"
    start_position => "beginning"
  }
}

filter {
  grok {
    match => { "message" => "%{SYSLOGTIMESTAMP:timestamp} %{SYSLOGHOST:host} %{DATA:program}: %{GREEDYDATA:message}" }
  }
}

output {
  elasticsearch {
    hosts => ["localhost:9200"]
  }
}

Millors Pràctiques

  • Definir Mètriques Clau: Identificar i monitorar les mètriques més rellevants per al rendiment i la salut del sistema.
  • Automatitzar Alertes: Configurar alertes per detectar problemes abans que afectin els usuaris.
  • Centralitzar Logs: Utilitzar una solució centralitzada per a la recopilació i anàlisi de logs.
  • Visualització Efectiva: Crear dashboards clars i concisos que permetin una ràpida interpretació de les dades.
  • Revisar i Ajustar: Revisar periòdicament les mètriques i alertes per ajustar-les segons sigui necessari.

Exercicis Pràctics

  1. Configurar Prometheus i Grafana:

    • Instal·lar Prometheus i Grafana en un entorn local.
    • Configurar Prometheus per recopilar mètriques d'un servei simple.
    • Crear un dashboard en Grafana per visualitzar les mètriques recopilades.

  2. Implementar ELK Stack:

    • Instal·lar Elasticsearch, Logstash i Kibana.
    • Configurar Logstash per processar i enviar logs a Elasticsearch.
    • Crear visualitzacions en Kibana per analitzar els logs.

Conclusió

El monitoratge de sistemes distribuïts és crucial per mantenir l'operativitat i rendiment de les aplicacions en entorns complexos. Utilitzant eines com Prometheus, Grafana i ELK Stack, juntament amb les millors pràctiques, els professionals poden assegurar-se que els seus sistemes funcionin de manera eficient i puguin respondre ràpidament a qualsevol problema que sorgeixi.
