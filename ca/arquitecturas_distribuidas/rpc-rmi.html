Introducció

En el context de les arquitectures distribuïdes, la comunicació entre processos que s'executen en diferents màquines és fonamental. Dues de les tecnologies més utilitzades per facilitar aquesta comunicació són RPC (Remote Procedure Call) i RMI (Remote Method Invocation). Aquest tema abordarà els conceptes, diferències, i exemples pràctics de totes dues tecnologies.

Conceptes Clau

RPC (Remote Procedure Call)

  • Definició: RPC és un protocol que permet a un programa executar procediments en una altra adreça d'espai de memòria (generalment en una altra màquina) com si fossin crides locals.
  • Funcionament:
    • Client: Realitza la crida al procediment remot.
    • Servidor: Conté la implementació del procediment.
    • Stub: Actua com un intermediari en el costat del client.
    • Skeleton: Actua com un intermediari en el costat del servidor.
  • Avantatges:
    • Transparència en la crida de procediments.
    • Simplificació de la comunicació entre processos.
  • Desavantatges:
    • Pot ser complex d'implementar i depurar.
    • Dependència de la xarxa i latència.

RMI (Remote Method Invocation)

  • Definició: RMI és una tecnologia específica de Java que permet la invocació de mètodes en objectes que resideixen en diferents màquines virtuals Java (JVM).
  • Funcionament:
    • Client: Invoca mètodes en l'objecte remot.
    • Servidor: Proporciona la implementació de l'objecte remot.
    • Stub: Proxy en el costat del client que representa l'objecte remot.
    • Skeleton: Proxy en el costat del servidor que maneja la invocació del mètode (en versions anteriors de Java).
  • Avantatges:
    • Integració nativa amb Java.
    • Suport per a la transmissió d'objectes complexos.
  • Desavantatges:
    • Limitat a aplicacions Java.
    • Requereix configuració i maneig d'excepcions específiques de RMI.

Comparació entre RPC i RMI

| Característica | RPC | RMI | |-------------------------|------------------------------------------|------------------------------------------| | Llenguatge | Multillenguatge | Exclusiu de Java | | Transparència | Procediments | Mètodes | | Intermediaris | Stub i Skeleton | Stub i Skeleton (en versions anteriors) | | Complexitat | Moderada | Alta (degut a la configuració de RMI) | | Suport d'Objectes | Limitat | Complet |

Exemples de Codi

Exemple de RPC en Python

# rpc_server.py
import xmlrpc.server

def add(x, y):
    return x + y

server = xmlrpc.server.SimpleXMLRPCServer(("localhost", 8000))
server.register_function(add, "add")
print("Server is running...")
server.serve_forever()
# rpc_client.py
import xmlrpc.client

proxy = xmlrpc.client.ServerProxy("http://localhost:8000/")
result = proxy.add(5, 3)
print(f"Result of add(5, 3): {result}")

Exemple de RMI en Java

// RemoteInterface.java
import java.rmi.Remote;
import java.rmi.RemoteException;

public interface RemoteInterface extends Remote {
    int add(int x, int y) throws RemoteException;
}
// RemoteImplementation.java
import java.rmi.server.UnicastRemoteObject;
import java.rmi.RemoteException;

public class RemoteImplementation extends UnicastRemoteObject implements RemoteInterface {
    protected RemoteImplementation() throws RemoteException {
        super();
    }

    public int add(int x, int y) throws RemoteException {
        return x + y;
    }
}
// Server.java
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;

public class Server {
    public static void main(String[] args) {
        try {
            RemoteImplementation obj = new RemoteImplementation();
            Registry registry = LocateRegistry.createRegistry(1099);
            registry.bind("RemoteInterface", obj);
            System.out.println("Server is running...");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
// Client.java
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;

public class Client {
    public static void main(String[] args) {
        try {
            Registry registry = LocateRegistry.getRegistry("localhost", 1099);
            RemoteInterface stub = (RemoteInterface) registry.lookup("RemoteInterface");
            int result = stub.add(5, 3);
            System.out.println("Result of add(5, 3): " + result);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Conclusió

RPC i RMI són tecnologies essencials per a la comunicació en arquitectures distribuïdes. Mentre que RPC ofereix una solució més general i multillenguatge, RMI proporciona una integració més profunda i robusta dins de l'ecosistema Java. Comprendre les seves diferències, avantatges i desavantatges és crucial per triar la tecnologia adequada segons les necessitats específiques de cada projecte.
