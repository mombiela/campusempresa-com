El Projecte Sobre nosaltres Contribuir Donacions Llicència

La gestió de concurrència és un aspecte crucial en les arquitectures distribuïdes, ja que permet que múltiples processos o fils d'execució operin simultàniament sense conflictes. Aquest tema aborda els conceptes fonamentals, tècniques i eines utilitzades per manejar la concurrència en sistemes distribuïts.

Conceptes Clau

  • Concurrència: La capacitat d'un sistema per executar múltiples tasques al mateix temps.
  • Paral·lelisme: L'execució simultània de múltiples tasques en diferents processadors o nuclis.
  • Condicions de cursa: Situacions on el resultat d'un procés depèn de l'ordre d'execució d'altres processos.
  • Secció crítica: Part del codi que ha de ser executada de manera exclusiva per un sol fil o procés a la vegada.
  • Bloqueig (Lock): Mecanisme per assegurar que només un fil o procés pugui accedir a una secció crítica a la vegada.
  • Deadlock (Interbloqueig): Situació on dos o més processos es bloquegen mútuament esperant recursos que mai seran alliberats.
  • Starvation (Fam): Situació on un procés mai obté els recursos necessaris per continuar la seva execució.

Tècniques de Gestió de Concurrència

Bloquejos i Semàfors

  • Mutex (Mutual Exclusion): Un tipus de bloqueig que permet que només un fil accedeixi a una secció crítica a la vegada.
  • Semàfors: Variables que s'utilitzen per controlar l'accés a recursos compartits. Poden ser binaris (0 o 1) o comptar (permeten un nombre específic d'accessos simultanis).

Exemple de Mutex en Python

import threading

# Recurs compartit
counter = 0
counter_lock = threading.Lock()

def increment_counter():
    global counter
    with counter_lock:
        counter += 1

threads = []
for i in range(10):
    thread = threading.Thread(target=increment_counter)
    threads.append(thread)
    thread.start()

for thread in threads:
    thread.join()

print(f"Final counter value: {counter}")

Monitors

  • Monitors: Estructures que combinen la mútua exclusió i la sincronització de fils. Proporcionen mètodes que són executats en exclusió mútua.

Algoritmes de Consens

  • Paxos: Un algoritme de consens que assegura que un grup de nodes en un sistema distribuït acordin un valor proposat.
  • Raft: Un algoritme de consens més senzill i comprensible que Paxos, utilitzat per a la replicació de logs.

Transaccions Distribuïdes

  • Two-Phase Commit (2PC): Un protocol que assegura que totes les parts d'una transacció distribuïda acordin en realitzar la transacció o revertir-la.
  • Three-Phase Commit (3PC): Una millora del 2PC que afegeix una fase addicional per reduir la possibilitat d'interbloquejos.

Eines i Biblioteques

  • Zookeeper: Un servei centralitzat per mantenir la configuració i proporcionar serveis de sincronització.
  • Redis: Una base de dades en memòria que suporta operacions atòmiques i estructures de dades avançades.
  • Akka: Un toolkit per construir aplicacions concurrents i distribuïdes en la JVM.

Exemple de Concurrència amb Akka en Scala

import akka.actor.{Actor, ActorSystem, Props}

case object Increment
case object GetCount

class Counter extends Actor {
  var count = 0

  def receive = {
    case Increment => count += 1
    case GetCount => sender() ! count
  }
}

object Main extends App {
  val system = ActorSystem("CounterSystem")
  val counter = system.actorOf(Props[Counter], "counter")

  counter ! Increment
  counter ! Increment
  counter ! GetCount

  system.terminate()
}

Conclusió

La gestió de concurrència és essencial per al correcte funcionament de sistemes distribuïts. Comprendre i aplicar tècniques com bloquejos, semàfors, monitors i algoritmes de consens és crucial per evitar problemes com condicions de cursa, interbloquejos i fam. Eines i biblioteques com Zookeeper, Redis i Akka faciliten la implementació de solucions concurrents i distribuïdes, permetent als desenvolupadors construir sistemes robustos i eficients.
