En aquesta secció, explorarem exemples d'aplicacions que utilitzen arquitectures distribuïdes. Aquestes aplicacions abasten una varietat d'indústries i casos d'ús, la qual cosa demostra la versatilitat i el poder de les arquitectures distribuïdes.

Aplicacions Web Escalables

Les aplicacions web escalables són un exemple clàssic d'arquitectures distribuïdes. Aquestes aplicacions han de manejar grans volums de trànsit i dades, la qual cosa requereix una distribució eficient de recursos.

Exemple: Xarxes Socials

Les xarxes socials com Facebook, Twitter i LinkedIn utilitzen arquitectures distribuïdes per manejar milions d'usuaris simultàniament. Aquestes arquitectures permeten:

  • Balanceig de càrrega: Distribuir el trànsit entre múltiples servidors per evitar sobrecàrregues.
  • Replicació de dades: Mantenir còpies de dades en diferents ubicacions per millorar la disponibilitat i la velocitat d'accés.
  • Microserveis: Dividir l'aplicació en serveis més petits i manejables que poden desenvolupar-se, desplegar-se i escalar-se de forma independent.

Codi d'Exemple: Balanceig de Càrrega amb Nginx

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

En aquest exemple, Nginx es configura per distribuir el trànsit entre tres servidors backend.

Sistemes d'Emmagatzematge Distribuït

Els sistemes d'emmagatzematge distribuït són essencials per manejar grans volums de dades i garantir la disponibilitat i durabilitat dels mateixos.

Exemple: Hadoop

Hadoop és un marc de treball que permet el processament distribuït de grans conjunts de dades a través de clústers d'ordinadors. Utilitza el sistema de fitxers distribuït HDFS (Hadoop Distributed File System) per emmagatzemar dades de manera redundant.

  • MapReduce: Un model de programació per al processament de grans conjunts de dades.
  • HDFS: Sistema de fitxers distribuït que emmagatzema dades en blocs distribuïts en múltiples nodes.

Codi d'Exemple: Job de MapReduce en Hadoop

public class WordCount {
    public static class TokenizerMapper extends Mapper<Object, Text, Text, IntWritable> {
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();

        public void map(Object key, Text value, Context context) throws IOException, InterruptedException {
            StringTokenizer itr = new StringTokenizer(value.toString());
            while (itr.hasMoreTokens()) {
                word.set(itr.nextToken());
                context.write(word, one);
            }
        }
    }

    public static class IntSumReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
        public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            context.write(key, new IntWritable(sum));
        }
    }

    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "word count");
        job.setJarByClass(WordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}

Aquest exemple mostra una feina de MapReduce per comptar paraules en Hadoop.

Serveis de Missatgeria i Cues

Els serveis de missatgeria i cues són fonamentals en les arquitectures distribuïdes per a la comunicació asíncrona entre diferents components del sistema.

Exemple: RabbitMQ

RabbitMQ és un sistema de missatgeria que implementa el protocol AMQP (Advanced Message Queuing Protocol). S'utilitza per enviar i rebre missatges entre diferents parts d'una aplicació distribuïda.

  • Productors: Envien missatges a la cua.
  • Consumidors: Reben missatges de la cua.
  • Intercanvis: Dirigeixen els missatges a les cues adequades.

Codi d'Exemple: Enviament i Recepció de Missatges amb RabbitMQ

# Enviament de missatges (Productor)
import pika

connection = pika.BlockingConnection(pika.ConnectionParameters('localhost'))
channel = connection.channel()

channel.queue_declare(queue='hello')

channel.basic_publish(exchange='',
                      routing_key='hello',
                      body='Hello World!')
print(" [x] Sent 'Hello World!'")
connection.close()

# Recepció de missatges (Consumidor)
import pika

connection = pika.BlockingConnection(pika.ConnectionParameters('localhost'))
channel = connection.channel()

channel.queue_declare(queue='hello')

def callback(ch, method, properties, body):
    print(" [x] Received %r" % body)

channel.basic_consume(queue='hello',
                      on_message_callback=callback,
                      auto_ack=True)

print(' [*] Waiting for messages. To exit press CTRL+C')
channel.start_consuming()

En aquest exemple, un productor envia un missatge "Hello World!" a una cua, i un consumidor rep el missatge de la cua.

Conclusió

Les arquitectures distribuïdes són essencials per construir aplicacions escalables, robustes i eficients. A través d'exemples com aplicacions web escalables, sistemes d'emmagatzematge distribuït i serveis de missatgeria, hem vist com aquestes arquitectures s'apliquen en el món real. La comprensió i l'ús adequat d'aquestes arquitectures permeten als desenvolupadors crear sistemes que poden manejar grans volums de dades i trànsit, garantint al mateix temps la disponibilitat i la resiliència.
