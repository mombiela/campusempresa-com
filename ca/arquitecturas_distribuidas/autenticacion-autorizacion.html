Introducció

En les arquitectures distribuïdes, l'autenticació i l'autorització són components crítics per garantir la seguretat i l'accés adequat als recursos. Aquest tema cobreix els conceptes fonamentals, les diferències entre autenticació i autorització, i les millors pràctiques per implementar-les en sistemes distribuïts.

Conceptes Clau

  • Autenticació: Procés de verificar la identitat d'un usuari o sistema.
  • Autorització: Procés de determinar si un usuari o sistema autenticat té permís per accedir a un recurs específic.
  • Token d'Autenticació: Un objecte que representa la identitat de l'usuari i s'utilitza per accedir als recursos.
  • OAuth: Un protocol d'autorització que permet a les aplicacions obtenir accés limitat als recursos d'un usuari.
  • JWT (JSON Web Token): Un estàndard obert per crear tokens d'accés que permeten la transmissió segura d'informació entre parts com un objecte JSON.

Autenticació

L'autenticació és el primer pas en el procés de seguretat. S'assegura que l'usuari o sistema és qui diu ser. Hi ha diversos mètodes d'autenticació:

  • Contrasenyes: El mètode més comú, on l'usuari proporciona un nom d'usuari i una contrasenya.
  • Autenticació de Dos Factors (2FA): Afegeix una capa addicional de seguretat en requerir un segon factor, com un codi enviat al telèfon de l'usuari.
  • Certificats Digitals: Utilitzen criptografia per verificar la identitat de l'usuari.
  • Biometria: Utilitza característiques físiques de l'usuari, com empremtes dactilars o reconeixement facial.

Exemple d'Autenticació amb JWT

import jwt
import datetime

# Clau secreta per signar el token
SECRET_KEY = 'la_meva_clau_secreta'

# Dades de l'usuari
user_data = {
    'username': 'usuari123',
    'exp': datetime.datetime.utcnow() + datetime.timedelta(hours=1)  # Expiració del token
}

# Crear el token
token = jwt.encode(user_data, SECRET_KEY, algorithm='HS256')
print(f'Token d'autenticació: {token}')

Autorització

Un cop un usuari ha estat autenticat, el següent pas és l'autorització. Aquest procés determina quins recursos i accions estan permesos per a l'usuari autenticat.

  • Llistes de Control d'Accés (ACL): Defineixen quins usuaris o sistemes tenen accés a quins recursos.
  • Rols i Permisos: Assignen permisos a rols específics, i després els usuaris s'assignen a aquests rols.
  • OAuth: Permet a les aplicacions obtenir accés limitat als recursos d'un usuari sense compartir les credencials de l'usuari.

Exemple d'Autorització amb Rols

# Definició de rols i permisos
roles = {
    'admin': ['llegir', 'escriure', 'esborrar'],
    'usuari': ['llegir']
}

# Funció per verificar permisos
def verificar_permís(rol, acció):
    if acció in roles.get(rol, []):
        return True
    return False

# Verificar si un usuari amb rol 'admin' pot 'esborrar'
print(verificar permís('admin', 'esborrar'))  # Sortida: True

Comparació d'Autenticació i Autorització

| Característica | Autenticació | Autorització | |--------------------|---------------------------------------|---------------------------------------| | Propòsit | Verificar la identitat de l'usuari | Determinar l'accés a recursos | | Quan ocorre | Abans de l'autorització | Després de l'autenticació | | Exemple | Usuari ingressa nom d'usuari i contrasenya | Usuari pot accedir a un fitxer específic |

Conclusió

L'autenticació i l'autorització són components essencials en qualsevol arquitectura distribuïda. L'autenticació assegura que els usuaris siguin qui diuen ser, mentre que l'autorització determina quins recursos poden accedir aquests usuaris. Implementar aquests processos de manera efectiva és crucial per mantenir la seguretat i la integritat dels sistemes distribuïts.
