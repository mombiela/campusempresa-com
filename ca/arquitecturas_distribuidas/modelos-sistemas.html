Models de Sistemes Distribuïts

Introducció

Els sistemes distribuïts són una col·lecció d'ordinadors independents que semblen un únic sistema coherent per als usuaris. En aquest tema, explorarem els diferents models de sistemes distribuïts, les seves característiques, avantatges i desavantatges, i com s'apliquen a la pràctica.

Models de Sistemes Distribuïts

1. Model Client-Servidor

  • Descripció: En aquest model, els clients sol·liciten serveis i els servidors proporcionen aquests serveis.
  • Característiques:
    • Centralització de recursos en el servidor.
    • Clients lleugers que sol·liciten serveis.
    • Comunicació a través de protocols de xarxa.
  • Avantatges:
    • Fàcil de gestionar i mantenir.
    • Escalabilitat vertical (millora del servidor).
  • Desavantatges:
    • Punt únic de fallada.
    • Escalabilitat limitada pel servidor.

Exemple de Codi

# Exemple d'un servidor simple en Python usant sockets
import socket

def start_server():
    server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    server_socket.bind(('localhost', 8080))
    server_socket.listen(5)
    print("Servidor iniciat i escoltant en el port 8080")

    while True:
        client_socket, address = server_socket.accept()
        print(f"Connexió establerta amb {address}")
        client_socket.send(b'Hola des del servidor!')
        client_socket.close()

start_server()

2. Model Peer-to-Peer (P2P)

  • Descripció: En aquest model, cada node actua tant com a client com a servidor, compartint recursos directament entre ells.
  • Característiques:
    • Descentralització.
    • Cada node té capacitats equivalents.
    • Comunicació directa entre nodes.
  • Avantatges:
    • Alta disponibilitat i redundància.
    • Escalabilitat horitzontal (afegir més nodes).
  • Desavantatges:
    • Gestió i coordinació complexes.
    • Seguretat i fiabilitat poden ser un problema.

Exemple de Codi

# Exemple d'un node P2P simple en Python usant sockets
import socket
import threading

def handle_client(client_socket):
    request = client_socket.recv(1024)
    print(f"Rebut: {request.decode('utf-8')}")
    client_socket.send(b'Hola des del node P2P!')
    client_socket.close()

def start_node():
    node_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    node_socket.bind(('localhost', 8081))
    node_socket.listen(5)
    print("Node P2P iniciat i escoltant en el port 8081")

    while True:
        client_socket, address = node_socket.accept()
        print(f"Connexió establerta amb {address}")
        client_handler = threading.Thread(target=handle_client, args=(client_socket,))
        client_handler.start()

start_node()

3. Model de Computació en el Núvol

  • Descripció: Aquest model proporciona recursos de computació i emmagatzematge a través d'internet.
  • Característiques:
    • Serveis sota demanda.
    • Escalabilitat elàstica.
    • Pagament per ús.
  • Avantatges:
    • Reducció de costos d'infraestructura.
    • Flexibilitat i escalabilitat.
  • Desavantatges:
    • Dependència del proveïdor de serveis.
    • Preocupacions de seguretat i privacitat.

Exemple d'Ús

# Exemple d'ús d'AWS S3 per a emmagatzematge en el núvol
import boto3

def upload_to_s3(file_name, bucket, object_name=None):
    s3_client = boto3.client('s3')
    try:
        s3_client.upload_file(file_name, bucket, object_name or file_name)
        print(f"Arxiu {file_name} pujat a {bucket}")
    except Exception as e:
        print(f"Error pujant arxiu: {e}")

upload_to_s3('el_meu_arxiu.txt', 'el_meu-bucket')

Conclusió

Els models de sistemes distribuïts ofereixen diferents enfocaments per a la gestió i distribució de recursos. L'elecció del model adequat depèn de les necessitats específiques del sistema, incloent l'escalabilitat, la gestió, la seguretat i els costos. Comprendre aquests models és crucial per dissenyar arquitectures distribuïdes eficients i efectives.
