Introducció

En aquesta secció, explorarem casos reals d'arquitectures distribuïdes implementades en diverses indústries. Analitzarem com aquestes arquitectures han resolt problemes específics i els beneficis que han aportat. Aquest estudi de casos et permetrà comprendre millor com aplicar els conceptes teòrics en situacions pràctiques.

Cas 1: Netflix

Descripció del Problema

Netflix necessitava una infraestructura que pogués manejar milions d'usuaris simultàniament, oferint una experiència de transmissió de vídeo sense interrupcions.

Solució Implementada

  • Microserveis: Netflix va adoptar una arquitectura de microserveis per dividir la seva aplicació monolítica en serveis més petits i manejables.
  • AWS (Amazon Web Services): Van utilitzar el núvol d'AWS per escalar els seus serveis de manera flexible i eficient.
  • Eureka: Van implementar un servei de descobriment perquè els microserveis poguessin trobar-se entre si.
  • Hystrix: Van utilitzar aquest patró de circuit per manejar fallades i latències en els serveis.

Exemple de Codi

// Exemple d'un microservei en Spring Boot
@SpringBootApplication
public class NetflixServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(NetflixServiceApplication.class, args);
    }
}

@RestController
public class MovieController {
    @GetMapping("/movies")
    public List<String> getMovies() {
        return Arrays.asList("Movie 1", "Movie 2", "Movie 3");
    }
}

Beneficis

  • Escalabilitat: L'arquitectura de microserveis va permetre a Netflix escalar individualment cada servei segons la demanda.
  • Resiliència: Amb Hystrix, Netflix va poder manejar fallades de manera eficient, millorant la disponibilitat del servei.
  • Flexibilitat: L'adopció d'AWS va permetre a Netflix ajustar la seva infraestructura ràpidament segons les necessitats.

Cas 2: Uber

Descripció del Problema

Uber necessitava una arquitectura que pogués manejar milions de sol·licituds de viatge en temps real, garantint la disponibilitat i la baixa latència.

Solució Implementada

  • Microserveis: Uber també va adoptar una arquitectura de microserveis per manejar diferents aspectes de la seva plataforma, com la gestió d'usuaris, l'assignació de viatges i el processament de pagaments.
  • Apache Kafka: Van utilitzar Kafka per a la transmissió de dades en temps real entre els microserveis.
  • Cassandra: Van implementar Cassandra com a base de dades distribuïda per manejar grans volums de dades amb alta disponibilitat.

Exemple de Codi

# Exemple de productor Kafka en Python
from kafka import KafkaProducer
import json

producer = KafkaProducer(bootstrap_servers='localhost:9092',
                         value_serializer=lambda v: json.dumps(v).encode('utf-8'))

data = {'user_id': '123', 'location': 'San Francisco'}
producer.send('ride_requests', value=data)
producer.flush()

Beneficis

  • Baixa Latència: Kafka va permetre a Uber processar dades en temps real, reduint la latència en l'assignació de viatges.
  • Alta Disponibilitat: Cassandra va proporcionar una base de dades distribuïda que garantia la disponibilitat de les dades fins i tot en cas de fallades.
  • Escalabilitat: L'arquitectura de microserveis va permetre a Uber escalar diferents parts del seu sistema de manera independent.

Cas 3: Airbnb

Descripció del Problema

Airbnb necessitava una infraestructura que pogués manejar milions de llistats i reserves, oferint una experiència d'usuari fluida i fiable.

Solució Implementada

  • Microserveis: Airbnb va adoptar una arquitectura de microserveis per manejar diferents funcionalitats com la cerca de llistats, la gestió de reserves i la comunicació entre usuaris.
  • Apache Kafka: Van utilitzar Kafka per a la transmissió d'esdeveniments entre microserveis.
  • MySQL i Redis: Van implementar MySQL per a l'emmagatzematge de dades relacionals i Redis per a l'emmagatzematge en memòria cau.

Exemple de Codi

# Exemple de consumidor Kafka en Ruby
require 'kafka'

kafka = Kafka.new(seed_brokers: ['localhost:9092'])

consumer = kafka.consumer(group_id: 'booking_group')
consumer.subscribe('booking_events')

consumer.each_message do |message|
  puts message.value
end

Beneficis

  • Escalabilitat: L'arquitectura de microserveis va permetre a Airbnb escalar diferents parts del seu sistema segons la demanda.
  • Rendiment: L'ús de Redis com a memòria cau va millorar significativament el rendiment de les consultes freqüents.
  • Flexibilitat: Kafka va permetre una comunicació eficient entre microserveis, facilitant la transmissió d'esdeveniments en temps real.

Conclusió

L'estudi d'aquests casos reals demostra com les arquitectures distribuïdes poden resoldre problemes complexos en diferents indústries. L'adopció de microserveis, juntament amb tecnologies com AWS, Kafka, Cassandra i Redis, ha permès a empreses com Netflix, Uber i Airbnb escalar els seus sistemes, millorar la resiliència i oferir una millor experiència d'usuari. Aquests exemples proporcionen una guia pràctica sobre com aplicar conceptes d'arquitectures distribuïdes en situacions del món real.
