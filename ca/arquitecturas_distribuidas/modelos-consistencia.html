Introducció

En les arquitectures distribuïdes, la consistència de les dades és un aspecte crucial que determina com les dades es sincronitzen i es mantenen coherents a través de múltiples nodes. Els models de consistència defineixen les garanties que un sistema distribuït ofereix en termes de visibilitat i ordre de les actualitzacions de dades.

Tipus de Models de Consistència

Consistència Forta

  • Definició: Garanteix que qualsevol lectura de dades sempre torna el valor més recent escrit.
  • Avantatges:
    • Proporciona una visió coherent i actualitzada de les dades.
    • Simplifica el desenvolupament d'aplicacions.
  • Desavantatges:
    • Pot implicar latències més altes a causa de la necessitat de sincronització entre nodes.
    • Menor tolerància a fallades.

Consistència Eventual

  • Definició: Garanteix que, en absència de noves actualitzacions, tots els nodes eventualment convergiran al mateix valor.
  • Avantatges:
    • Alta disponibilitat i baixa latència.
    • Millor tolerància a fallades.
  • Desavantatges:
    • Les dades poden estar temporalment inconsistents.
    • Requereix lògica addicional en l'aplicació per manejar la inconsistència temporal.

Consistència Causal

  • Definició: Assegura que les operacions que són causalment relacionades es veuen en el mateix ordre en tots els nodes.
  • Avantatges:
    • Proporciona una consistència més forta que l'eventual sense el cost complet de la consistència forta.
    • Adequada per a aplicacions col·laboratives.
  • Desavantatges:
    • Més complexa d'implementar que la consistència eventual.
    • Pot requerir més metadades per rastrejar dependències causals.

Consistència de Sessió

  • Definició: Garanteix que totes les operacions dins d'una sessió d'usuari vegin un ordre consistent d'operacions.
  • Avantatges:
    • Proporciona una experiència d'usuari coherent.
    • Menys costosa que la consistència forta.
  • Desavantatges:
    • No garanteix consistència entre diferents sessions.
    • Pot ser insuficient per a aplicacions que requereixen una consistència global.

Comparació de Models de Consistència

| Model de Consistència | Garanties de Consistència | Latència | Disponibilitat | Complexitat d'Implementació | |-----------------------|---------------------------|----------|----------------|-----------------------------| | Forta | Alta | Alta | Baixa | Alta | | Eventual | Baixa | Baixa | Alta | Baixa | | Causal | Mitjana | Mitjana | Mitjana | Mitjana | | Sessió | Mitjana (dins de la sessió) | Mitjana | Mitjana | Mitjana |

Exemples de Codi

Consistència Eventual amb Redis

import redis
import time

# Connexió a Redis
r = redis.Redis(host='localhost', port=6379, db=0)

# Escriptura de dades
r.set('key', 'value1')
print("Escrit 'value1' en 'key'")

# Simulació de retard
time.sleep(2)

# Lectura de dades
value = r.get('key')
print(f"Llegit '{value.decode('utf-8')}' de 'key'")

# Escriptura d'un nou valor
r.set('key', 'value2')
print("Escrit 'value2' en 'key'")

# Lectura immediata del nou valor
value = r.get('key')
print(f"Llegit '{value.decode('utf-8')}' de 'key'")

Consistència Forta amb Zookeeper

from kazoo.client import KazooClient

# Connexió a Zookeeper
zk = KazooClient(hosts='127.0.0.1:2181')
zk.start()

# Escriptura de dades
zk.create("/mykey", b"value1")
print("Escrit 'value1' en '/mykey'")

# Lectura de dades
data, stat = zk.get("/mykey")
print(f"Llegit '{data.decode('utf-8')}' de '/mykey'")

# Escriptura d'un nou valor
zk.set("/mykey", b"value2")
print("Escrit 'value2' en '/mykey'")

# Lectura immediata del nou valor
data, stat = zk.get("/mykey")
print(f"Llegit '{data.decode('utf-8')}' de '/mykey'")

# Tancament de la connexió
zk.stop()

Conclusió

Els models de consistència són fonamentals per dissenyar sistemes distribuïts eficients i fiables. L'elecció del model adequat depèn de les necessitats específiques de l'aplicació, incloent-hi requisits de latència, disponibilitat i consistència. Comprendre els avantatges i desavantatges de cada model permet als arquitectes de sistemes prendre decisions informades per equilibrar aquests factors i construir sistemes robustos.
