Introducció

Apache Spark és un motor de processament de dades en clúster de codi obert que s'utilitza per al processament en temps real i en lots. Aquest tema se centra en com Spark pot ser utilitzat per al processament en temps real, una capacitat crucial en arquitectures distribuïdes modernes.

Conceptes Clau

  • Processament en Temps Real: La capacitat de processar dades a mesura que es generen o reben, amb una latència mínima.
  • Apache Spark: Un motor d'anàlisi unificat que suporta processament en temps real i en lots.
  • Spark Streaming: Una extensió de Spark que permet el processament de fluxos de dades en temps real.
  • DStreams (Discretized Streams): L'abstracció principal en Spark Streaming, que representa un flux continu de dades.
  • Structured Streaming: Una API més recent i d'alt nivell per al processament de fluxos de dades en temps real.

Spark Streaming

Spark Streaming permet el processament en temps real de fluxos de dades. Les dades poden provenir de diverses fonts com Kafka, Flume, Kinesis, o sockets TCP.

Exemple de Codi: Spark Streaming amb Socket

from pyspark import SparkContext
from pyspark.streaming import StreamingContext

# Crear un context de Spark
sc = SparkContext("local[2]", "NetworkWordCount")
ssc = StreamingContext(sc, 1)

# Crear un DStream que connecti a un host i port
lines = ssc.socketTextStream("localhost", 9999)

# Realitzar operacions de transformació
words = lines.flatMap(lambda line: line.split(" "))
pairs = words.map(lambda word: (word, 1))
wordCounts = pairs.reduceByKey(lambda x, y: x + y)

# Imprimir el resultat
wordCounts.pprint()

# Iniciar el processament
ssc.start()
ssc.awaitTermination()

Explicació del Codi

  • SparkContext: Inicialitza el context de Spark.
  • StreamingContext: Inicialitza el context de streaming amb un interval de lot d'1 segon.
  • socketTextStream: Connecta a un socket TCP en l'host i port especificats.
  • Transformacions: flatMap, map, i reduceByKey s'utilitzen per transformar i agregar les dades.
  • pprint: Imprimeix els resultats a la consola.

Structured Streaming

Structured Streaming és una API més recent que proporciona una forma més senzilla i robusta de processar fluxos de dades en temps real.

Exemple de Codi: Structured Streaming amb Kafka

from pyspark.sql import SparkSession

# Crear una sessió de Spark
spark = SparkSession.builder.appName("StructuredKafkaWordCount").getOrCreate()

# Llegir dades de Kafka
df = spark.readStream.format("kafka") \
    .option("kafka.bootstrap.servers", "localhost:9092") \
    .option("subscribe", "test_topic") \
    .load()

# Seleccionar i transformar les dades
words = df.selectExpr("CAST(value AS STRING)").alias("words")
wordCounts = words.groupBy("value").count()

# Escriure els resultats a la consola
query = wordCounts.writeStream.outputMode("complete").format("console").start()

# Iniciar el processament
query.awaitTermination()

Explicació del Codi

  • SparkSession: Inicialitza la sessió de Spark.
  • readStream: Llegeix dades en temps real des de Kafka.
  • selectExpr: Selecciona i transforma les dades.
  • groupBy: Agrupa les dades per valor i compta les ocurrències.
  • writeStream: Escriu els resultats a la consola.

Comparació entre Spark Streaming i Structured Streaming

| Característica | Spark Streaming | Structured Streaming | |-------------------------|--------------------------------------|-------------------------------------| | API | DStreams | DataFrames/Datasets | | Latència | Baixa | Baixa | | Complexitat | Mitjana | Baixa | | Tolerància a fallades | Alta | Alta | | Integració amb Kafka | Bona | Excel·lent |

Conclusió

Apache Spark ofereix potents capacitats per al processament en temps real a través de Spark Streaming i Structured Streaming. Mentre que Spark Streaming utilitza DStreams, Structured Streaming proporciona una API més moderna i senzilla basada en DataFrames i Datasets. Ambdues opcions són robustes i adequades per a diferents casos d'ús en arquitectures distribuïdes.

El coneixement d'aquestes eines i tècniques és essencial per a qualsevol professional que treballi amb grans volums de dades i necessiti processar-les en temps real.
