Introducció

En aquesta secció del curs sobre Arquitectures Distribuïdes, explorarem el concepte d'APIs i Microserveis. Entendrem com aquestes tecnologies permeten la construcció d'aplicacions escalables, mantenibles i eficients.

APIs (Application Programming Interfaces)

Conceptes Clau

  • Definició: Una API és un conjunt de regles que permet a diferents programaris comunicar-se entre si.
  • Tipus d'APIs:
    • REST (Representational State Transfer): Utilitza HTTP requests per accedir i manipular dades.
    • SOAP (Simple Object Access Protocol): Protocol basat en XML per intercanviar informació estructurada.
    • GraphQL: Llenguatge de consulta per a APIs que permet sol·licitar exactament les dades necessàries.
  • Components d'una API:
    • Endpoint: URL on es pot accedir a un recurs.
    • Mètodes HTTP: GET, POST, PUT, DELETE, etc.
    • Headers: Informació addicional enviada amb la sol·licitud o resposta.
    • Body: Contingut de la sol·licitud o resposta, generalment en format JSON o XML.

Exemple de Codi

Exemple d'una API REST en Node.js amb Express

const express = require('express');
const app = express();
const port = 3000;

app.use(express.json());

let items = [
  { id: 1, name: 'Item 1' },
  { id: 2, name: 'Item 2' }
];

// GET all items
app.get('/api/items', (req, res) => {
  res.json(items);
});

// GET item by ID
app.get('/api/items/:id', (req, res) => {
  const item = items.find(i => i.id === parseInt(req.params.id));
  if (!item) return res.status(404).send('Item not found');
  res.json(item);
});

// POST create new item
app.post('/api/items', (req, res) => {
  const newItem = {
    id: items.length + 1,
    name: req.body.name
  };
  items.push(newItem);
  res.status(201).json(newItem);
});

// PUT update item
app.put('/api/items/:id', (req, res) => {
  const item = items.find(i => i.id === parseInt(req.params.id));
  if (!item) return res.status(404).send('Item not found');
  item.name = req.body.name;
  res.json(item);
});

// DELETE item
app.delete('/api/items/:id', (req, res) => {
  const itemIndex = items.findIndex(i => i.id === parseInt(req.params.id));
  if (itemIndex === -1) return res.status(404).send('Item not found');
  items.splice(itemIndex, 1);
  res.status(204).send();
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});

Exercici

  1. Crear una API REST: Utilitza Node.js i Express per crear una API que gestioni una llista de tasques (CRUD: Create, Read, Update, Delete).
  2. Documentar l'API: Usa Swagger per documentar la teva API.

Microserveis

Conceptes Clau

  • Definició: Arquitectura que estructura una aplicació com un conjunt de serveis petits i autònoms.
  • Característiques:
    • Desacoblament: Cada microservei és independent.
    • Escalabilitat: Els microserveis poden escalar-se individualment.
    • Desplegament Independent: Cada microservei pot desplegar-se sense afectar els altres.
    • Tecnologia Heterogènia: Diferents microserveis poden usar diferents tecnologies i llenguatges de programació.
  • Comunicació entre Microserveis:
    • Síncrona: Usant HTTP/REST o gRPC.
    • Asíncrona: Usant cues de missatges com RabbitMQ o Kafka.

Exemple de Codi

Exemple de Microservei en Spring Boot

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.web.bind.annotation.*;

import java.util.ArrayList;
import java.util.List;

@SpringBootApplication
public class ItemServiceApplication {

    public static void main(String[] args) {
        SpringApplication.run(ItemServiceApplication.class, args);
    }
}

@RestController
@RequestMapping("/api/items")
class ItemController {

    private List<Item> items = new ArrayList<>();

    @GetMapping
    public List<Item> getAllItems() {
        return items;
    }

    @GetMapping("/{id}")
    public Item getItemById(@PathVariable int id) {
        return items.stream().filter(item -> item.getId() == id).findFirst().orElse(null);
    }

    @PostMapping
    public Item createItem(@RequestBody Item item) {
        items.add(item);
        return item;
    }

    @PutMapping("/{id}")
    public Item updateItem(@PathVariable int id, @RequestBody Item newItem) {
        Item item = items.stream().filter(i -> i.getId() == id).findFirst().orElse(null);
        if (item != null) {
            item.setName(newItem.getName());
        }
        return item;
    }

    @DeleteMapping("/{id}")
    public void deleteItem(@PathVariable int id) {
        items.removeIf(item -> item.getId() == id);
    }
}

class Item {
    private int id;
    private String name;

    // Getters and Setters
}

Exercici

  1. Crear un Microservei: Utilitza Spring Boot per crear un microservei que gestioni una llista de productes.
  2. Desplegar el Microservei: Usa Docker per conteniritzar i desplegar el teu microservei.
