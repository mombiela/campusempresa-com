Introducció

Els sistemes de fitxers distribuïts (DFS, per les seves sigles en anglès) són una peça fonamental en les arquitectures distribuïdes. Permeten que múltiples usuaris i aplicacions accedeixin i comparteixin fitxers a través d'una xarxa, com si estiguessin en un sol sistema de fitxers local. Aquest tema cobreix els conceptes clau, arquitectures, exemples i exercicis pràctics per entendre i treballar amb sistemes de fitxers distribuïts.

Conceptes Clau

  • Transparència: Els usuaris han de percebre el sistema de fitxers distribuït com un únic sistema de fitxers unificat.
  • Escalabilitat: Capacitat del sistema per manejar un augment en la càrrega de treball sense degradar el rendiment.
  • Fiabilitat i Disponibilitat: Garantir que les dades estiguin disponibles i siguin accessibles fins i tot en cas de fallades.
  • Consistència: Assegurar que totes les còpies d'un fitxer siguin iguals en tots els nodes del sistema.
  • Seguretat: Protecció de les dades contra accessos no autoritzats i corrupció.

Arquitectures de Sistemes de Fitxers Distribuïts

Arquitectura Client-Servidor

  • Client: Sol·licita serveis d'emmagatzematge i recuperació de fitxers.
  • Servidor: Proporciona serveis d'emmagatzematge i recuperació de fitxers.

Arquitectura Peer-to-Peer (P2P)

  • Nodes iguals: Tots els nodes actuen tant com a clients i servidors.
  • Distribució de càrrega: La càrrega de treball es distribueix equitativament entre tots els nodes.

Exemples de Sistemes de Fitxers Distribuïts

NFS (Network File System)

  • Protocol: Utilitza el protocol RPC (Remote Procedure Call) per a la comunicació.
  • Transparència: Permet als usuaris accedir a fitxers remots com si estiguessin en el seu propi sistema.
# Exemple de muntatge d'un sistema de fitxers NFS
sudo mount -t nfs 192.168.1.100:/shared_folder /mnt/nfs

HDFS (Hadoop Distributed File System)

  • Disseny: Optimitzat per a l'emmagatzematge de grans volums de dades.
  • Components: NameNode (gestiona la metadata) i DataNodes (emmagatzemen les dades reals).
// Exemple de codi Java per llegir un fitxer en HDFS
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import java.io.BufferedReader;
import java.io.InputStreamReader;

public class HDFSExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path filePath = new Path("/user/hadoop/input.txt");
        BufferedReader br = new BufferedReader(new InputStreamReader(fs.open(filePath)));
        String line;
        while ((line = br.readLine()) != null) {
            System.out.println(line);
        }
        br.close();
    }
}

Comparació de Sistemes de Fitxers Distribuïts

| Característica | NFS | HDFS | |----------------------|---------------------------------|---------------------------------| | Transparència | Alta | Mitjana | | Escalabilitat | Limitada | Alta | | Fiabilitat | Mitjana | Alta | | Consistència | Alta | Eventual | | Seguretat | Mitjana | Alta |

Exercicis Pràctics

  1. Muntar un sistema de fitxers NFS:

    • Configura un servidor NFS en una màquina virtual.
    • Munta el sistema de fitxers NFS en una altra màquina virtual i accedeix als fitxers compartits.

  2. Treballar amb HDFS:

    • Instal·la Hadoop en un clúster de màquines virtuals.
    • Carrega un fitxer gran en HDFS i escriu un programa en Java per llegir i processar el fitxer.

Conclusió

Els sistemes de fitxers distribuïts són essencials per a les arquitectures distribuïdes modernes, proporcionant una forma eficient i fiable de gestionar i accedir a grans volums de dades. Comprendre els conceptes clau, les arquitectures i els exemples pràctics com NFS i HDFS és crucial per a qualsevol professional que treballi en aquest camp.
