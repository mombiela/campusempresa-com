Algoritmes de Consens

Introducció

En sistemes distribuïts, el consens és un problema fonamental que implica assolir un acord entre múltiples nodes o processos. Aquest acord és crucial per mantenir la coherència i la disponibilitat del sistema, especialment en presència de fallades. Els algorismes de consens són essencials per coordinar accions i prendre decisions de manera fiable en un entorn distribuït.

Conceptes Clau

  • Consens: Procés mitjançant el qual un grup de nodes en un sistema distribuït acorda sobre un valor únic.
  • Node: Cadascun dels participants en el sistema distribuït.
  • Fallades Bizantines: Tipus de fallada en què els nodes poden fallar i enviar informació incorrecta o maliciosa.
  • Tolerància a Fallades: Capacitat del sistema per continuar funcionant correctament fins i tot si alguns dels seus components fallen.

Algorismes de Consens Populars

Paxos

Paxos és un dels algorismes de consens més coneguts i utilitzats en sistemes distribuïts. Va ser proposat per Leslie Lamport i és conegut per la seva robustesa i tolerància a fallades.

Característiques de Paxos

  • Tolerància a fallades: Pot tolerar fallades de fins a menys de la meitat dels nodes.
  • Rendiment: Pot ser menys eficient en termes de latència a causa de múltiples rondes de comunicació.

Exemple de Codi

class PaxosNode:
    def __init__(self, node_id):
        self.node_id = node_id
        self.promised_id = None
        self.accepted_id = None
        self.accepted_value = None

    def prepare(self, proposal_id):
        if self.promised_id is None or proposal_id > self.promised_id:
            self.promised_id = proposal_id
            return True
        return False

    def accept(self, proposal_id, value):
        if self.promised_id is None or proposal_id >= self.promised_id:
            self.promised_id = proposal_id
            self.accepted_id = proposal_id
            self.accepted_value = value
            return True
        return False

Raft

Raft és un altre algorisme de consens que va ser dissenyat per ser més comprensible que Paxos. Va ser introduït per Diego Ongaro i John Ousterhout.

Característiques de Raft

  • Simplicitat: Més fàcil d'entendre i implementar que Paxos.
  • Rols definits: Líder, seguidor i candidat.

Exemple de Codi

class RaftNode:
    def __init__(self, node_id):
        self.node_id = node_id
        self.term = 0
        self.voted_for = None
        self.log = []

    def request_vote(self, candidate_id, candidate_term):
        if candidate_term > self.term:
            self.term = candidate_term
            self.voted_for = candidate_id
            return True
        return False

    def append_entries(self, leader_term, entries):
        if leader_term >= self.term:
            self.term = leader_term
            self.log.extend(entries)
            return True
        return False

Comparació de Paxos i Raft

| Característica | Paxos | Raft | |---------------------|------------------------------|------------------------------| | Simplicitat | Complex | Més simple | | Rendiment | Latència alta | Latència baixa | | Rols | No definits clarament | Líder, seguidor, candidat | | Tolerància a fallades | Alta | Alta |

Conclusió

Els algorismes de consens són fonamentals per a l'operació de sistemes distribuïts, permetent que múltiples nodes arribin a un acord malgrat fallades i condicions adverses. Paxos i Raft són dos dels algorismes més utilitzats, cadascun amb els seus propis avantatges i desavantatges. Comprendre aquests algorismes és crucial per dissenyar i mantenir sistemes distribuïts robustos i eficients.
