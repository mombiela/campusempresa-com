Introducció als Principis de Disseny en Microserveis

Els microserveis són una arquitectura de programari que permet desenvolupar aplicacions com una col·lecció de serveis petits i autònoms, cadascun executant-se en el seu propi procés i comunicant-se mitjançant protocols lleugers. Aquest enfocament ofereix diverses avantatges, com escalabilitat, flexibilitat i facilitat de manteniment. No obstant això, per aprofitar al màxim aquests beneficis, és crucial seguir certs principis de disseny.

Principis Clau de Disseny

1. Descomposició per Domini

  • Definició: Dividir l'aplicació en serveis basats en dominis de negoci.
  • Exemple: En una aplicació de comerç electrònic, podries tenir microserveis separats per a Usuaris, Comandes, Inventari i Pagaments.
+------------------+   +------------------+   +------------------+
|   Servei de      |   |   Servei de      |   |   Servei de      |
|     Usuaris      |   |     Comandes     |   |    Inventari     |
+------------------+   +------------------+   +------------------+

2. Independència d'Implementació

  • Definició: Cada microservei ha de ser independent en termes d'implementació i desplegament.
  • Exemple: Un servei pot estar escrit en Python mentre que un altre pot estar en Java, i tots dos poden ser desplegats i actualitzats de manera independent.
+------------------+   +------------------+
|   Servei en      |   |   Servei en      |
|     Python       |   |      Java        |
+------------------+   +------------------+

3. Comunicació Lleugera

  • Definició: Els microserveis han de comunicar-se entre si utilitzant protocols lleugers com HTTP/REST o missatges a través de cues.
  • Exemple: Un servei de Comandes pot enviar una sol·licitud HTTP al servei d'Inventari per verificar la disponibilitat d'un producte.
# Exemple de sol·licitud HTTP en Python
import requests

response = requests.get('http://inventari/comprovar_disponibilitat?producte_id=123')
if response.status_code == 200:
    disponibilitat = response.json()['disponibilitat']

4. Gestió de Dades Descentralitzada

  • Definició: Cada microservei ha de gestionar la seva pròpia base de dades.
  • Exemple: El servei d'Usuaris pot tenir una base de dades PostgreSQL, mentre que el servei de Comandes pot usar MongoDB.

| Servei | Base de Dades | |-----------|----------------| | Usuaris | PostgreSQL | | Comandes | MongoDB | | Inventari | MySQL |

5. Tolerància a Fallades

  • Definició: Els microserveis han de dissenyar-se per ser resilients i manejar fallades de manera graciosa.
  • Exemple: Implementar circuit breakers i retries per manejar fallades temporals en la comunicació entre serveis.
# Exemple d'implementació d'un retry en Python
import requests
from requests.adapters import HTTPAdapter
from requests.packages.urllib3.util.retry import Retry

session = requests.Session()
retry = Retry(total=3, backoff_factor=1, status_forcelist=[500, 502, 503, 504])
adapter = HTTPAdapter(max_retries=retry)
session.mount('http://', adapter)

response = session.get('http://inventari/comprovar_disponibilitat?producte_id=123')

6. Monitoratge i Registre

  • Definició: Implementar monitoratge i registre centralitzat per a tots els microserveis.
  • Exemple: Utilitzar eines com ELK Stack (Elasticsearch, Logstash, Kibana) per centralitzar i analitzar logs.

|   Servei de      |   |   Servei de      |   |   Servei de      |
|     Usuaris      |   |     Comandes     |   |    Inventari     |
+------------------+   +------------------+   +------------------+
        |                    |                    |
        +--------------------+--------------------+
                             |
                      +-------------+
                      |   ELK Stack |
                      +-------------+

Conclusió

El disseny de microserveis requereix una acurada consideració de diversos principis clau per garantir que els serveis siguin escalables, flexibles i fàcils de mantenir. En seguir aquests principis, pots construir sistemes robustos i eficients que aprofitin al màxim els avantatges de l'arquitectura de microserveis.
