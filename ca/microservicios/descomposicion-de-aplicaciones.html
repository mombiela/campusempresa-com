Introducció

La descomposició d'aplicacions és un pas crucial en la transició d'una arquitectura monolítica a una basada en microserveis. Aquest procés implica dividir una aplicació gran i complexa en components més petits i manejables, cadascun dels quals pot ser desenvolupat, desplegat i escalat de manera independent.

Conceptes Clau

  • Monòlit: Una aplicació monolítica és una aplicació única i unificada on tots els components estan interconnectats i es despleguen junts.
  • Microservei: Un microservei és una petita aplicació independent que realitza una funció específica dins d'un sistema més gran.
  • Descomposició: El procés de dividir una aplicació monolítica en microserveis.

Estratègies de Descomposició

Descomposició per Funcionalitat

Dividir l'aplicació en microserveis basats en les funcionalitats o mòduls principals.

  • Exemple: En una aplicació de comerç electrònic, podríem tenir microserveis per al catàleg de productes, el carret de la compra, el processament de pagaments, etc.
+----------------+     +----------------+     +----------------+
|  Product       |     |  Cart          |     |  Payment       |
|  Microservice  |     |  Microservice  |     |  Microservice  |
+----------------+     +----------------+     +----------------+

Descomposició per Subdominis

Utilitzar els principis de disseny de domini (DDD) per identificar subdominis dins de l'aplicació i crear microserveis per a cadascun.

  • Exemple: En una aplicació de gestió de projectes, podríem tenir subdominis com gestió d'usuaris, gestió de projectes i gestió de tasques.
+----------------+     +----------------+     +----------------+
|  User          |     |  Project       |     |  Task          |
|  Microservice  |     |  Microservice  |     |  Microservice  |
+----------------+     +----------------+     +----------------+

Descomposició per Capes

Dividir l'aplicació en microserveis basats en les capes d'arquitectura, com la capa de presentació, la capa de negoci i la capa de dades.

  • Exemple: En una aplicació de reserves de vols, podríem tenir microserveis per a la interfície d'usuari, la lògica de negoci i l'accés a dades.
+----------------+     +----------------+     +----------------+
|  UI            |     |  Business      |     |  Data          |
|  Microservice  |     |  Logic         |     |  Access        |
+----------------+     +----------------+     +----------------+

Exemple de Descomposició

Considerem una aplicació monolítica de blog que inclou funcionalitats com gestió d'usuaris, creació de posts, comentaris i notificacions. A continuació, es mostra com podríem descompondre aquesta aplicació en microserveis.

Monòlit Original

+------------------------------------------------+
|                    Blog App                    |
|------------------------------------------------|
|  Users  |  Posts  |  Comments  |  Notifications|
+------------------------------------------------+

Descomposició en Microserveis

+----------------+     +----------------+     +----------------+     +----------------+
|  User          |     |  Post          |     |  Comment       |     |  Notification  |
|  Microservice  |     |  Microservice  |     |  Microservice  |     |  Microservice  |
+----------------+     +----------------+     +----------------+     +----------------+

Exemple de Codi

Monòlit

# Monolithic Blog Application
class BlogApp:
    def __init__(self):
        self.users = []
        self.posts = []
        self.comments = []
        self.notifications = []

    def add_user(self, user):
        self.users.append(user)
        self.notify("New user added")

    def add_post(self, post):
        self.posts.append(post)
        self.notify("New post added")

    def add_comment(self, comment):
        self.comments.append(comment)
        self.notify("New comment added")

    def notify(self, message):
        self.notifications.append(message)

Microserveis

User Microservice
# User Microservice
class UserMicroservice:
    def __init__(self):
        self.users = []

    def add_user(self, user):
        self.users.append(user)
        # Notify other services
        NotificationMicroservice().notify("New user added")

# Post Microservice
class PostMicroservice:

class NotificationMicroservice:
    def __init__(self):

    def notify(self, message):
        self.notifications.append(message)

Conclusió

La descomposició d'aplicacions és un procés essencial per adoptar una arquitectura de microserveis. En dividir una aplicació monolítica en microserveis més petits i manejables, es poden obtenir beneficis significatius en termes d'escalabilitat, mantenibilitat i desplegament independent. Les estratègies de descomposició, com la descomposició per funcionalitat, subdominis i capes, proporcionen diferents enfocaments per aconseguir aquesta transició de manera efectiva.
