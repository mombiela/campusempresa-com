Introducció

En el context de microserveis, la seguretat en la comunicació és crucial per garantir que les dades intercanviades entre serveis siguin confidencials, íntegres i autèntiques. Aquest tema aborda els principis i pràctiques essencials per assegurar la comunicació entre microserveis.

Conceptes Clau

  • Confidencialitat: Assegurar que les dades transmeses no siguin accessibles per entitats no autoritzades.
  • Integritat: Garantir que les dades no siguin alterades durant la transmissió.
  • Autenticació: Verificar la identitat de les entitats que es comuniquen.
  • Autorització: Controlar l'accés a recursos i operacions basant-se en la identitat autenticada.
  • No repudi: Assegurar que una entitat no pugui negar haver realitzat una acció.

Mètodes de Seguretat

Xifrat de Dades en Trànsit

El xifrat de dades en trànsit és fonamental per protegir la informació mentre es mou entre microserveis.

  • TLS (Transport Layer Security): Protocol estàndard per xifrar la comunicació a la xarxa.

    - Ús de certificats SSL/TLS per establir connexions segures.
- Configuració de TLS en servidors i clients per assegurar la comunicació.

Exemple de Configuració de TLS en un Servidor Spring Boot

server:
  ssl:
    key-store: classpath:keystore.p12
    key-store-password: password
    key-store-type: PKCS12
    key-alias: tomcat

Autenticació i Autorització

  • OAuth 2.0: Protocol d'autorització que permet a les aplicacions obtenir accés limitat als recursos d'usuari.
  • JWT (JSON Web Tokens): Mètode per representar claims de forma segura entre dues parts.

Exemple d'Ús de JWT en Node.js

const jwt = require('jsonwebtoken');

const token = jwt.sign({ userId: 123 }, 'secretKey', { expiresIn: '1h' });

jwt.verify(token, 'secretKey', (err, decoded) => {
  if (err) {
    console.log('Token no vàlid');
  } else {
    console.log('Token vàlid', decoded);
  }
});

Seguretat en l'API

  • API Gateway: Actua com un punt d'entrada per a totes les sol·licituds de microserveis, proporcionant autenticació, autorització i altres polítiques de seguretat.
  • Rate Limiting: Limitar el nombre de sol·licituds a l'API per prevenir abusos.

Exemple de Configuració de Rate Limiting en NGINX

http {
  limit_req_zone $binary_remote_addr zone=mylimit:10m rate=1r/s;

  server {
    location /api/ {
      limit_req zone=mylimit burst=5;
      proxy_pass http://backend;
    }
  }
}

Comparació de Mètodes de Seguretat

| Mètode | Propòsit | Exemple d'Ús | |----------------------|-----------------------------------|----------------------------------| | TLS | Xifrat de dades en trànsit | Configuració de SSL/TLS | | OAuth 2.0 | Autorització | Accés a recursos d'usuari | | JWT | Autenticació i Autorització | Tokens per sessions d'usuari | | API Gateway | Punt d'entrada segur | Autenticació i polítiques | | Rate Limiting | Prevenció d'abusos | Configuració en NGINX |

Conclusió

La seguretat en la comunicació entre microserveis és essencial per protegir les dades i garantir la integritat i autenticitat de les interaccions. Implementar pràctiques com el xifrat de dades en trànsit, autenticació i autorització robustes, i l'ús d'API Gateways i Rate Limiting, són passos fonamentals per assegurar un entorn de microserveis segur i fiable.
