Introducció

La gestió d'errors i la recuperació són aspectes crítics en l'arquitectura de microserveis. Atès que els microserveis són sistemes distribuïts, els errors són inevitables i han de ser gestionats de manera eficient per assegurar la resiliència i la disponibilitat del sistema.

Conceptes Clau

  • Resiliència: Capacitat del sistema per recuperar-se de fallades i continuar funcionant.
  • Tolerància a fallades: Habilitat del sistema per seguir operant correctament en cas de fallades d'alguns dels seus components.
  • Degradació graciosa: Estratègia per reduir la funcionalitat d'un sistema en lloc de fallar completament.
  • Circuit Breaker: Patró que evita que una aplicació intenti realitzar una operació que probablement fallarà.
  • Retry: Estratègia per reintentar una operació fallida després d'un interval de temps.
  • Fallback: Provisió d'una resposta alternativa quan una operació falla.

Estratègies de Gestió d'Errors

Circuit Breaker

El patró Circuit Breaker és fonamental per prevenir que un sistema distribuït es vegi aclaparat per fallades repetides. Funciona com un interruptor que talla el flux de sol·licituds a un servei que està fallant.

Exemple de Circuit Breaker en Spring Boot

import org.springframework.cloud.client.circuitbreaker.CircuitBreakerFactory;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class MyController {

    private final CircuitBreakerFactory circuitBreakerFactory;

    public MyController(CircuitBreakerFactory circuitBreakerFactory) {
        this.circuitBreakerFactory = circuitBreakerFactory;
    }

    @GetMapping("/example")
    public String example() {
        return circuitBreakerFactory.create("circuitbreaker").run(
            () -> remoteServiceCall(),
            throwable -> fallbackMethod()
        );
    }

    private String remoteServiceCall() {
        // Lògica per trucar a un servei remot
        return "Success";
    }

    private String fallbackMethod() {
        // Lògica de fallback
        return "Fallback response";
    }
}

Retry

El patró Retry intenta reintentar una operació fallida després d'un interval de temps. És útil per gestionar fallades temporals.

Exemple de Retry en Spring Boot

import org.springframework.retry.annotation.Backoff;
import org.springframework.retry.annotation.Retryable;
import org.springframework.stereotype.Service;

@Service
public class MyService {

    @Retryable(
        value = {RemoteServiceException.class},
        maxAttempts = 4,
        backoff = @Backoff(delay = 2000)
    )
    public String callRemoteService() throws RemoteServiceException {
        // Lògica per trucar a un servei remot
        if (someConditionFails()) {
            throw new RemoteServiceException("Service failed");
        }
        return "Success";
    }
}

Fallback

El patró Fallback proporciona una resposta alternativa quan una operació falla. És una estratègia per assegurar que el sistema degradi la seva funcionalitat de manera controlada.

Exemple de Fallback en Hystrix

import com.netflix.hystrix.contrib.javanica.annotation.HystrixCommand;
import org.springframework.stereotype.Service;

@Service
public class MyService {

    @HystrixCommand(fallbackMethod = "fallbackMethod")
    public String callRemoteService() {
        // Lògica per trucar a un servei remot
        if (someConditionFails()) {
            throw new RuntimeException("Service failed");
        }
        return "Success";
    }

    public String fallbackMethod() {
        // Lògica de fallback
        return "Fallback response";
    }
}

Comparació d'Estratègies

| Estratègia | Avantatges | Desavantatges | |------------------|--------------------------------------------------------------------------|------------------------------------------------------------------| | Circuit Breaker | Prevé sobrecàrrega, millora la resiliència | Pot ser complex d'implementar i ajustar | | Retry | Gestiona fallades temporals, millora la disponibilitat | Pot causar sobrecàrrega si no es configura adequadament | | Fallback | Assegura una resposta controlada, millora l'experiència de l'usuari | Pot ocultar problemes subjacents si s'utilitza excessivament |

Conclusió

La gestió d'errors i la recuperació són essencials per mantenir la resiliència i disponibilitat en sistemes de microserveis. Implementar estratègies com Circuit Breaker, Retry i Fallback pot ajudar a gestionar fallades de manera eficient i assegurar que el sistema degradi la seva funcionalitat de manera controlada en lloc de fallar completament. L'elecció de l'estratègia adequada depèn del context i els requisits específics del sistema.
