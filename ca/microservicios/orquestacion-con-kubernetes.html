Introducció a Kubernetes

Kubernetes és una plataforma d'orquestració de contenidors que automatitza el desplegament, la gestió i l'escalabilitat d'aplicacions en contenidors. És una eina essencial en el desenvolupament i operació de microserveis a causa de la seva capacitat per manejar aplicacions complexes i distribuïdes.

Conceptes Clau

  • Node: Una màquina física o virtual en el clúster de Kubernetes.
  • Pod: La unitat més petita i bàsica de desplegament en Kubernetes, que pot contenir un o més contenidors.
  • Cluster: Un conjunt de nodes que executen aplicacions en contenidors gestionats per Kubernetes.
  • Namespace: Un mecanisme per dividir recursos de clúster entre múltiples usuaris.
  • Deployment: Un recurs que defineix com desplegar i gestionar un conjunt de Pods.
  • Service: Un recurs que defineix una política per accedir a un conjunt de Pods.

Arquitectura de Kubernetes

Kubernetes segueix una arquitectura de clúster mestre-esclau. El clúster mestre gestiona l'estat del clúster i els nodes esclaus executen les aplicacions.

Components del Mestre

  • API Server: Punt d'entrada per a totes les operacions del clúster.
  • etcd: Magatzem de dades distribuït que guarda la configuració del clúster.
  • Controller Manager: Gestiona els controladors que regulen l'estat del clúster.
  • Scheduler: Assigna Pods als nodes disponibles.

Components del Node

  • Kubelet: Agent que assegura que els contenidors s'executin en un Pod.
  • Kube-proxy: Gestiona la xarxa de serveis en cada node.
  • Container Runtime: Programari que executa els contenidors (per exemple, Docker).

Desplegament d'Aplicacions amb Kubernetes

Creació d'un Pod

Un Pod és la unitat bàsica de desplegament en Kubernetes. Aquí hi ha un exemple d'un fitxer YAML per crear un Pod:

apiVersion: v1
kind: Pod
metadata:
  name: my-pod
spec:
  containers:
  - name: my-container
    image: nginx
    ports:
    - containerPort: 80

Per desplegar aquest Pod, s'utilitza la comanda:

kubectl apply -f my-pod.yaml

Creació d'un Deployment

Un Deployment gestiona la creació i actualització de Pods. Aquí hi ha un exemple d'un fitxer YAML per a un Deployment:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-container
        image: nginx
        ports:
        - containerPort: 80

Per desplegar aquest Deployment, s'utilitza la comanda:

kubectl apply -f my-deployment.yaml

Creació d'un Service

Un Service exposa un conjunt de Pods com un servei de xarxa. Aquí hi ha un exemple d'un fitxer YAML per a un Service:

apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
  type: LoadBalancer

Per desplegar aquest Service, s'utilitza la comanda:

kubectl apply -f my-service.yaml

Escalabilitat i Gestió d'Estats

Escalat Horitzontal

Kubernetes permet escalar aplicacions horitzontalment (afegint més rèpliques de Pods) de manera senzilla:

kubectl scale deployment my-deployment --replicas=5

Actualitzacions i Rollbacks

Kubernetes facilita les actualitzacions i rollbacks d'aplicacions:

  • Actualització:
kubectl set image deployment/my-deployment my-container=nginx:1.19.0
  • Rollback:
kubectl rollout undo deployment/my-deployment

Monitoratge i Logging

Monitoratge

Kubernetes proporciona mètriques a través d'eines com Prometheus i Grafana per monitorar el rendiment de les aplicacions.

Logging

Els logs dels contenidors poden ser accedits usant:

kubectl logs my-pod

Conclusió

Kubernetes és una eina poderosa per a l'orquestració de contenidors, essencial per al desplegament i gestió de microserveis. Amb la seva capacitat per automatitzar l'escalat, les actualitzacions i la recuperació davant fallades, Kubernetes facilita la gestió d'aplicacions complexes i distribuïdes. A través d'aquest mòdul, hem cobert els conceptes bàsics, l'arquitectura, i les operacions essencials per treballar amb Kubernetes.
