Introducció

Els microserveis han revolucionat la forma en què es desenvolupen i despleguen les aplicacions modernes. Aquest enfocament arquitectònic permet als equips de desenvolupament crear aplicacions més escalables, flexibles i fàcils de mantenir. En aquesta secció, explorarem la història i evolució dels microserveis, des dels seus orígens fins a la seva adopció generalitzada en la indústria.

Orígens dels Microserveis

  • Monòlits: Inicialment, les aplicacions es desenvolupaven com a monòlits, on tots els components estaven integrats en una única unitat de desplegament.
  • Desafiaments dels Monòlits:
    • Dificultat per escalar.
    • Problemes de manteniment i actualització.
    • Dependències complexes entre components.

Evolució cap als Microserveis

Primers Intentos de Descomposició

  • Arquitectura en Capes: Separació de la lògica de presentació, lògica de negoci i accés a dades.
  • Serveis SOA (Arquitectura Orientada a Serveis):
    • Introducció de serveis independents.
    • Ús de protocols com SOAP per a la comunicació.
    • Problemes de complexitat i sobrecàrrega.

Naixement dels Microserveis

  • Concepte de Microserveis:
    • Introduït per Martin Fowler i James Lewis el 2014.
    • Enfocament en la creació de serveis petits, autònoms i desplegables independentment.
  • Característiques Clau:
    • Desplegament independent.
    • Escalabilitat horitzontal.
    • Comunicació a través d'APIs lleugeres (generalment HTTP/REST).

Adopció i Maduresa

  • Adopció Primerenca:
    • Empreses com Netflix, Amazon i Spotify van adoptar microserveis per millorar l'escalabilitat i agilitat.
  • Eines i Tecnologies:
    • Contenidors (Docker) i orquestradors (Kubernetes) van facilitar el desplegament i gestió de microserveis.
    • Frameworks i llibreries específiques per a microserveis (Spring Boot, Micronaut).

Comparació: Monòlits vs Microserveis

| Aspecte | Monòlits | Microserveis | |------------------------|----------------------------------------|-----------------------------------------| | Desplegament | Un únic desplegament | Desplegament independent de serveis | | Escalabilitat | Escalabilitat vertical | Escalabilitat horitzontal | | Manteniment | Complex, afecta tota l'aplicació | Més senzill, afecta només el servei | | Comunicació | Interna, dins del mateix procés | Externa, a través d'APIs | | Dependències | Altament acoblades | Feblement acoblades |

Exemple de Codi

Monòlit

public class MonolithicApplication {
    public static void main(String[] args) {
        UserService userService = new UserService();
        OrderService orderService = new OrderService();
        
        // Lògica de negoci combinada
        userService.createUser("John Doe");
        orderService.createOrder("John Doe", "Product 1");
    }
}

Microservei

UserService

@RestController
@RequestMapping("/users")
public class UserService {
    
    @PostMapping
    public ResponseEntity<String> createUser(@RequestBody String name) {
        // Lògica per crear usuari
        return ResponseEntity.ok("User created: " + name);
    }
}

OrderService

@RestController
@RequestMapping("/orders")
public class OrderService {
    
    @PostMapping
    public ResponseEntity<String> createOrder(@RequestBody OrderRequest orderRequest) {
        // Lògica per crear ordre
        return ResponseEntity.ok("Order created for user: " + orderRequest.getUserName());
    }
}

Conclusió

La transició d'arquitectures monolítiques a microserveis ha permès a les organitzacions millorar l'escalabilitat, flexibilitat i velocitat de desenvolupament de les seves aplicacions. Tot i que els microserveis presenten desafiaments propis, com la gestió de la comunicació entre serveis i l'orquestració, els avantatges superen amb escreix aquests inconvenients. Amb el suport d'eines modernes i pràctiques recomanades, els microserveis s'han convertit en una elecció popular per al desenvolupament d'aplicacions complexes i de gran escala.
