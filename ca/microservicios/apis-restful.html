Introducció a les APIs RESTful

Les APIs RESTful són un component essencial en l'arquitectura de microserveis. REST (Representational State Transfer) és un estil d'arquitectura que utilitza HTTP per a la comunicació entre sistemes. En aquest tema, explorarem els principis fonamentals de les APIs RESTful, la seva estructura i com s'integren en un entorn de microserveis.

Principis de les APIs RESTful

Perquè una API sigui considerada RESTful, ha d'adherir-se a certs principis:

  • Client-Servidor: L'arquitectura ha de separar les responsabilitats del client i del servidor.
  • Sense Estat: Cada sol·licitud del client al servidor ha de contenir tota la informació necessària per entendre i processar la sol·licitud.
  • Cacheable: Les respostes han de ser explícitament marcades com a cacheables o no cacheables.
  • Interfície Uniforme: Ha d'haver-hi una interfície uniforme que permeti la interacció entre el client i el servidor de manera estàndard.
  • Sistema en Capes: L'arquitectura pot estar composta per múltiples capes, cadascuna amb la seva pròpia funcionalitat.
  • Codi a Demanda (opcional): Els servidors poden proporcionar codi executable al client quan sigui necessari.

Estructura d'una API RESTful

Recursos

En REST, els recursos són les peces fonamentals que s'exposen a través de l'API. Cada recurs és identificat per una URL única.

  • Exemple de URL de recurs: https://api.exemple.com/usuaris/123

Mètodes HTTP

Els mètodes HTTP són utilitzats per realitzar operacions sobre els recursos. Els mètodes més comuns són:

  • GET: Recuperar informació d'un recurs.
  • POST: Crear un nou recurs.
  • PUT: Actualitzar un recurs existent.
  • DELETE: Eliminar un recurs.

Exemples de Codi

Exemple d'una sol·licitud GET

GET /usuaris/123 HTTP/1.1
Host: api.exemple.com

Exemple d'una sol·licitud POST

POST /usuaris HTTP/1.1
Host: api.exemple.com
Content-Type: application/json

{
  "nom": "Joan Perez",
  "email": "joan.perez@exemple.com"
}

Exemple d'una sol·licitud PUT

PUT /usuaris/123 HTTP/1.1
Host: api.exemple.com
Content-Type: application/json

{
  "nom": "Joan Perez",
  "email": "joan.perez@exemple.com"
}

Exemple d'una sol·licitud DELETE

DELETE /usuaris/123 HTTP/1.1
Host: api.exemple.com

Bones Pràctiques

  • Versionat de l'API: Utilitzar versionat en les URLs per manejar canvis en l'API.
    • Exemple: https://api.exemple.com/v1/usuaris
  • Gestió d'Errors: Proveir missatges d'error clars i codis d'estat HTTP adequats.
    • Exemple: 404 Not Found per a recursos no trobats.
  • Documentació: Mantenir una documentació clara i actualitzada de l'API.
  • Seguretat: Implementar autenticació i autorització adequades, com OAuth.

Comparació de Mètodes HTTP

| Mètode | Descripció | Idempotent | Segur | |---------|-------------------------------------|-------------|--------| | GET | Recuperar un recurs | Sí | Sí | | POST | Crear un nou recurs | No | No | | PUT | Actualitzar un recurs existent | Sí | No | | DELETE | Eliminar un recurs | Sí | No |

Conclusió

Les APIs RESTful són una eina poderosa per a la comunicació entre microserveis. En seguir els principis i bones pràctiques de REST, es pot construir una API robusta i escalable. En el següent tema, explorarem com integrar aquestes APIs RESTful en una arquitectura de microserveis i com gestionar la comunicació entre diferents serveis.
