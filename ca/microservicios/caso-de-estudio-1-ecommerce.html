Introducció

En aquest cas d'estudi, explorarem com una aplicació de comerç electrònic pot beneficiar-se d'una arquitectura de microserveis. Analitzarem els components clau, les interaccions entre ells i com es poden implementar utilitzant microserveis per millorar l'escalabilitat, la flexibilitat i la mantenibilitat del sistema.

Components del Sistema E-commerce

Un sistema de comerç electrònic típic pot dividir-se en diversos components clau, cadascun dels quals pot ser implementat com un microservei independent:

  • Servei de Catàleg de Productes: Gestiona la informació sobre els productes disponibles.
  • Servei de Carret de Compres: Maneja les operacions relacionades amb el carret de compres de l'usuari.
  • Servei de Comanda: Processa i gestiona les comandes realitzades pels usuaris.
  • Servei de Pagament: Gestiona les transaccions de pagament.
  • Servei d'Usuari: Maneja la informació i autenticació dels usuaris.
  • Servei de Notificacions: Envia notificacions als usuaris sobre l'estat de les seves comandes.

Interaccions entre Microserveis

Els microserveis han de comunicar-se entre si per proporcionar una funcionalitat completa. A continuació, es mostra un diagrama simplificat de les interaccions entre els microserveis:

| Microservei | Interacció | |-------------|-------------| | Catàleg de Productes | Proporciona informació de productes al Carret de Compres i al Servei de Comanda. | | Carret de Compres | Sol·licita informació de productes i envia dades al Servei de Comanda. | | Comanda | Rep dades del Carret de Compres i sol·licita transaccions al Servei de Pagament. | | Pagament | Processa pagaments i notifica al Servei de Comanda sobre l'estat de la transacció. | | Usuari | Autentica usuaris i proporciona informació al Carret de Compres i al Servei de Comanda. | | Notificacions | Envia actualitzacions als usuaris sobre les seves comandes i transaccions. |

Exemple d'Implementació

A continuació, es presenta un exemple de com es podria implementar el Servei de Catàleg de Productes utilitzant Node.js i Express.

const express = require('express');
const app = express();
const port = 3000;

let products = [
  { id: 1, name: 'Laptop', price: 1000 },
  { id: 2, name: 'Smartphone', price: 500 },
  { id: 3, name: 'Tablet', price: 300 }
];

app.get('/products', (req, res) => {
  res.json(products);
});

app.get('/products/:id', (req, res) => {
  const product = products.find(p => p.id === parseInt(req.params.id));
  if (product) {
    res.json(product);
  } else {
    res.status(404).send('Product not found');
  }
});

app.listen(port, () => {
  console.log(`Product service listening at http://localhost:${port}`);
});

Explicació del Codi

  • Express: Utilitzem el framework Express per crear un servidor HTTP.
  • Rutes: Definim dues rutes, una per obtenir tots els productes (/products) i una altra per obtenir un producte específic pel seu ID (/products/:id).
  • Dades de Productes: Utilitzem un array simple per emmagatzemar les dades dels productes. En una aplicació real, aquestes dades probablement vindrien d'una base de dades.

Conclusió

Implementar una arquitectura de microserveis per a un sistema de comerç electrònic pot oferir nombrosos avantatges, incloent-hi una millor escalabilitat, flexibilitat i mantenibilitat. Cada component del sistema pot desenvolupar-se, desplegar-se i escalar-se de manera independent, cosa que facilita la gestió i l'evolució del sistema al llarg del temps.

Aquest cas d'estudi proporciona una visió general de com es poden estructurar i comunicar els microserveis en una aplicació de comerç electrònic. En les pròximes seccions del curs, aprofundirem en tècniques avançades i millors pràctiques per implementar i gestionar microserveis en un entorn de producció.
