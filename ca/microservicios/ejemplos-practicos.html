En aquesta secció, explorarem exemples pràctics que t'ajudaran a entendre com implementar i gestionar microserveis en un entorn real. Aquests exemples estan dissenyats per ser clars i comprensibles, fins i tot si ets un principiant en el tema.

Exemple 1: Creació d'un Microservei Bàsic

Descripció de l'Exemple

En aquest primer exemple, crearem un microservei bàsic utilitzant Spring Boot, un framework popular en l'ecosistema de Java. Aquest microservei serà responsable de gestionar informació d'usuaris.

Passos per Crear el Microservei

  1. Configuració del Projecte:

    • Crear un nou projecte Spring Boot utilitzant Spring Initializr.
    • Seleccionar les dependències necessàries: Spring Web, Spring Data JPA, i H2 Database.

  2. Definició del Model:

    • Crear una classe User que representi l'entitat d'usuari.
@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private String email;

    // Getters and Setters
}
  1. Creació del Repositori:
    • Crear una interfície UserRepository que estengui JpaRepository.
public interface UserRepository extends JpaRepository<User, Long> {
}
  1. Implementació del Controlador:
    • Crear un controlador UserController per manejar les sol·licituds HTTP.
@RestController
@RequestMapping("/users")
public class UserController {
    @Autowired
    private UserRepository userRepository;

    @GetMapping
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    @PostMapping
    public User createUser(@RequestBody User user) {
        return userRepository.save(user);
    }
}

Execució i Proves

  • Executar l'aplicació i utilitzar eines com Postman per provar els endpoints /users (GET i POST).

Exemple 2: Comunicació entre Microserveis

Descripció de l'Exemple

En aquest exemple, veurem com dos microserveis poden comunicar-se entre si utilitzant REST API. Tindrem un microservei de Order que es comunicarà amb el microservei de User per obtenir informació de l'usuari.

Passos per Configurar la Comunicació

  1. Creació del Microservei d'Ordre:

    • Configurar un nou projecte Spring Boot per al microservei de Order.

  2. Definició del Model d'Ordre:

    • Crear una classe Order que representi l'entitat d'ordre.
@Entity
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private Long userId;
    private String product;
    private int quantity;

    // Getters and Setters
}
  1. Implementació del Client REST:
    • Crear una classe UserClient per comunicar-se amb el microservei de User.
@Service
public class UserClient {
    @Autowired
    private RestTemplate restTemplate;

    public User getUserById(Long userId) {
        return restTemplate.getForObject("http://localhost:8080/users/" + userId, User.class);
    }
}
  1. Configuració del Controlador d'Ordre:
    • Crear un controlador OrderController que utilitzi UserClient.
@RestController
@RequestMapping("/orders")
public class OrderController {
    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private UserClient userClient;

    @PostMapping
    public Order createOrder(@RequestBody Order order) {
        User user = userClient.getUserById(order.getUserId());
        if (user != null) {
            return orderRepository.save(order);
        } else {
            throw new RuntimeException("User not found");
        }
    }
}

Execució i Proves

  • Executar ambdós microserveis (User i Order).
  • Utilitzar Postman per crear ordres i verificar la comunicació entre els microserveis.

Conclusió

En aquesta secció, hem vist dos exemples pràctics que il·lustren com crear i gestionar microserveis. El primer exemple es va centrar en la creació d'un microservei bàsic, mentre que el segon va mostrar com els microserveis poden comunicar-se entre si. Aquests exemples són fonamentals per entendre els conceptes clau dels microserveis i et serviran com a base per a projectes més complexos.
