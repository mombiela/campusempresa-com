Introducció

En aquesta secció del curs, explorarem dues arquitectures modernes que estan transformant la manera en què desenvolupem i despleguem aplicacions: Microserveis i Serverless. Ambdues arquitectures ofereixen avantatges significatius en termes d'escalabilitat, manteniment i eficiència, però també presenten desafiaments únics.

Microserveis

Què són els Microserveis?

Els microserveis són una arquitectura de programari que estructura una aplicació com un conjunt de serveis petits i autònoms, cadascun executant un procés únic i comunicant-se a través d'interfícies ben definides.

  • Descomposició: Cada servei és una unitat funcional independent.
  • Desplegament Independent: Els serveis poden ser desplegats i escalats de manera independent.
  • Comunicació: Generalment es comuniquen a través d'APIs RESTful o missatgeria.

Avantatges dels Microserveis

  • Escalabilitat: Permet escalar només els serveis que ho necessiten.
  • Flexibilitat Tecnològica: Cada servei pot ser desenvolupat amb diferents tecnologies.
  • Manteniment: Facilita la identificació i resolució de problemes.

Desafiaments dels Microserveis

  • Complexitat: Augmenta la complexitat de la gestió i el monitoratge.
  • Comunicació: La comunicació entre serveis pot ser un coll d'ampolla.
  • Consistència de Dades: Mantenir la consistència de dades pot ser complicat.

Exemple de Microserveis

# Servei d'Usuari
from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/user/<int:user_id>', methods=['GET'])
def get_user(user_id):
    # Simulació d'obtenció de dades d'usuari
    user = {"id": user_id, "name": "John Doe"}
    return jsonify(user)

if __name__ == '__main__':
    app.run(port=5000)

# Servei de Comanda
from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/order/<int:order_id>', methods=['GET'])
def get_order(order_id):
    # Simulació d'obtenció de dades de comanda
    order = {"id": order_id, "item": "Laptop", "quantity": 1}
    return jsonify(order)

if __name__ == '__main__':
    app.run(port=5001)

Serverless

Què és Serverless?

Serverless és un model d'execució al núvol on el proveïdor de serveis al núvol gestiona la infraestructura i l'assignació de recursos automàticament. Els desenvolupadors només es preocupen pel codi.

  • Funcionalitat com a Servei (FaaS): Executa funcions en resposta a esdeveniments.
  • Backend com a Servei (BaaS): Serveis gestionats com bases de dades, autenticació, etc.

Avantatges de Serverless

  • Cost-Eficiència: Pagament per ús, sense necessitat de mantenir servidors.
  • Escalabilitat Automàtica: Escala automàticament segons la demanda.
  • Reducció de Tasques Operatives: Elimina la necessitat de gestionar la infraestructura.

Desafiaments de Serverless

  • Latència: Pot haver-hi latència en l'arrencada de funcions (cold start).
  • Limitacions d'Execució: Restriccions en el temps d'execució i recursos.
  • Debugging i Monitoratge: Pot ser més complicat que en arquitectures tradicionals.

Exemple de Serverless amb AWS Lambda

import json

def lambda_handler(event, context):
    # Simulació de processament d'esdeveniment
    user_id = event.get('user_id', 1)
    user = {"id": user_id, "name": "John Doe"}
    
    return {
        'statusCode': 200,
        'body': json.dumps(user)
    }

Comparació entre Microserveis i Serverless

| Característica | Microserveis | Serverless | |---------------------------|---------------------------------------|------------------------------------------| | Gestió d'Infraestructura | Requereix gestió de contenidors i VMs | Gestionat pel proveïdor del núvol | | Escalabilitat | Escalabilitat manual o automatitzada | Escalabilitat automàtica | | Cost | Costos fixos i variables | Pagament per ús | | Desplegament | Desplegament de serveis independents | Desplegament de funcions individuals | | Latència | Baixa latència | Pot haver-hi latència en l'arrencada |

Conclusió

Tant els microserveis com el serverless són arquitectures poderoses que poden millorar significativament l'eficiència i escalabilitat de les aplicacions. L'elecció entre una i altra dependrà de les necessitats específiques del projecte, l'equip i la infraestructura existent. Comprendre els avantatges i desafiaments de cadascuna és crucial per prendre decisions informades i dissenyar sistemes robustos i eficients.
