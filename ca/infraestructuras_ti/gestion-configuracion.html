Introducció a la Gestió de Configuració

La gestió de configuració és un procés essencial en l'administració d'infraestructures TI, que se centra a mantenir la consistència del rendiment, la funcionalitat i els atributs físics d'un sistema al llarg del temps. Aquest procés és crucial per garantir que tots els components d'un sistema estiguin correctament configurats i funcionin de manera òptima.

Objectius de la Gestió de Configuració

  • Control de Canvis: Assegurar que tots els canvis en la infraestructura es realitzin de manera controlada i documentada.
  • Integritat del Sistema: Mantenir la integritat i la coherència del sistema al llarg del temps.
  • Auditoria i Compliment: Facilitar l'auditoria i el compliment de normatives i estàndards.
  • Optimització de Recursos: Millorar la utilització dels recursos i reduir el temps d'inactivitat.

Components de la Gestió de Configuració

Elements de Configuració (CI)

  • Maquinari: Servidors, routers, switches, etc.
  • Programari: Sistemes operatius, aplicacions, bases de dades.
  • Documentació: Manuals, polítiques, procediments.

Base de Dades de Gestió de Configuració (CMDB)

Una CMDB és una base de dades que conté tots els elements de configuració i les seves relacions. És fonamental per a la gestió de configuració ja que proporciona una visió centralitzada i actualitzada de l'estat del sistema.

| Element de Configuració | Descripció | Estat | |-------------------------|------------|-------| | Servidor Web | Apache HTTP Server | Actiu | | Base de Dades | MySQL 5.7 | Actiu | | Router | Cisco 2901 | En manteniment |

Eines de Gestió de Configuració

Ansible

Ansible és una eina d'automatització que permet la gestió de configuració mitjançant scripts YAML. És coneguda per la seva simplicitat i facilitat d'ús.

Exemple de Playbook d'Ansible

---
- name: Configurar servidor web
  hosts: webservers
  become: yes
  tasks:
    - name: Instal·lar Apache
      apt:
        name: apache2
        state: present

    - name: Iniciar servei Apache
      service:
        name: apache2
        state: started

Puppet

Puppet és una altra eina popular que utilitza un llenguatge declaratiu per definir la configuració del sistema.

Exemple de Manifest de Puppet

node 'webserver' {
  package { 'apache2':
    ensure => installed,
  }

  service { 'apache2':
    ensure => running,
    enable => true,
  }
}

Bones Pràctiques en la Gestió de Configuració

  • Documentació Completa: Mantenir una documentació detallada i actualitzada de tots els elements de configuració.
  • Automatització: Utilitzar eines d'automatització per reduir errors humans i millorar l'eficiència.
  • Revisió i Auditoria: Realitzar revisions periòdiques i auditories per assegurar la conformitat amb les polítiques i procediments establerts.
  • Gestió de Canvis: Implementar un procés formal de gestió de canvis per avaluar i aprovar modificacions en la infraestructura.

Conclusió

La gestió de configuració és un component crític en l'administració d'infraestructures TI. A través de la identificació, control i documentació dels elements de configuració, es pot assegurar l'estabilitat, seguretat i eficiència del sistema. Eines com Ansible i Puppet faciliten l'automatització d'aquests processos, permetent als professionals de TI mantenir un entorn controlat i ben gestionat.
