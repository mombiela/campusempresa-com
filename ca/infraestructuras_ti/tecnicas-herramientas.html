Introducció

L'alta disponibilitat (HA) és un aspecte crític en la infraestructura de TI, especialment en entorns on la continuïtat del servei és essencial. Aquest tema cobreix les tècniques i eines que s'utilitzen per garantir que els sistemes i aplicacions estiguin disponibles amb el mínim temps d'inactivitat.

Conceptes Clau d'Alta Disponibilitat

  • Redundància: Implementació de components duplicats per evitar punts únics de fallada.
  • Failover: Procés de canviar automàticament a un sistema de suport quan el sistema principal falla.
  • Load Balancing: Distribució del trànsit de xarxa o aplicació entre diversos servidors per assegurar que cap no es sobrecarregui.
  • Clustering: Agrupació de múltiples servidors perquè treballin junts com un sol sistema.
  • Replicació de Dades: Còpia de dades en múltiples ubicacions per assegurar la seva disponibilitat en cas de fallada.

Tècniques per a Alta Disponibilitat

Redundància

La redundància implica tenir múltiples components que puguin assumir la càrrega si un falla. Això pot aplicar-se a maquinari, programari, xarxes i emmagatzematge.

  • Maquinari Redundant: Ús de servidors, discos durs i fonts d'alimentació duplicades.
  • Programari Redundant: Implementació d'aplicacions en múltiples servidors.

Failover

El failover és crucial per a l'alta disponibilitat. Aquest procés pot ser manual o automàtic.

  • Failover Manual: Requereix intervenció humana per canviar a un sistema de suport.
  • Failover Automàtic: Utilitza programari per detectar fallades i canviar automàticament al sistema de suport.

Load Balancing

El balanceig de càrrega distribueix el trànsit entre diversos servidors per evitar sobrecàrregues i millorar la disponibilitat.

  • Tipus de Load Balancers:
    • DNS Load Balancing: Distribueix el trànsit a través de diferents adreces IP.
    • Hardware Load Balancers: Dispositius físics que distribueixen el trànsit.
    • Software Load Balancers: Aplicacions que gestionen la distribució del trànsit.

Clustering

El clustering agrupa múltiples servidors perquè treballin junts com un sol sistema, millorant la disponibilitat i escalabilitat.

  • Tipus de Clústers:
    • Active-Active Clusters: Tots els nodes estan actius i comparteixen la càrrega.
    • Active-Passive Clusters: Un node està actiu mentre que l'altre està en espera i s'activa només en cas de fallada.

Replicació de Dades

La replicació de dades assegura que les dades estiguin disponibles en múltiples ubicacions, cosa que és crucial per a la recuperació davant desastres.

  • Tipus de Replicació:
    • Síncrona: Les dades es replicen en temps real.
    • Asíncrona: Les dades es replicen amb un retard.

Eines per a Alta Disponibilitat

Eines de Failover

  • Keepalived: Utilitzat per a failover i balanceig de càrrega en entorns Linux.
  • Heartbeat: Proporciona failover i monitoratge d'alta disponibilitat.

Eines de Load Balancing

  • HAProxy: Un balancejador de càrrega d'alt rendiment i proxy per a TCP i HTTP.
  • Nginx: A més de ser un servidor web, també pot actuar com un balancejador de càrrega.

Eines de Clustering

  • Pacemaker: Un gestor de clústers d'alta disponibilitat.
  • Corosync: Proporciona serveis de missatgeria i quorum per a clústers.

Eines de Replicació de Dades

  • DRBD (Distributed Replicated Block Device): Permet la replicació de dades en temps real entre servidors.
  • GlusterFS: Un sistema de fitxers distribuït que permet la replicació de dades.

Exemple de Configuració de HAProxy

A continuació, es mostra un exemple bàsic de configuració de HAProxy per a balanceig de càrrega:

global
    log /dev/log    local0
    log /dev/log    local1 notice
    chroot /var/lib/haproxy
    stats socket /run/haproxy/admin.sock mode 660 level admin
    stats timeout 30s
    user haproxy
    group haproxy
    daemon

defaults
    log     global
    mode    http
    option  httplog
    option  dontlognull
    timeout connect 5000
    timeout client  50000
    timeout server  50000

frontend http_front
    bind *:80
    default_backend http_back

backend http_back
    balance roundrobin
    server server1 192.168.1.1:80 check
    server server2 192.168.1.2:80 check

En aquest exemple:

  • La secció global defineix configuracions globals per a HAProxy.
  • La secció defaults defineix configuracions per defecte per a totes les seccions.
  • La secció frontend defineix l'entrada de trànsit.
  • La secció backend defineix els servidors que rebran el trànsit balancejat.

Conclusió

L'alta disponibilitat és essencial per garantir la continuïtat del servei en infraestructures de TI. Implementar tècniques com redundància, failover, balanceig de càrrega, clustering i replicació de dades, juntament amb eines adequades, pot minimitzar el temps d'inactivitat i assegurar que els sistemes estiguin sempre disponibles.
