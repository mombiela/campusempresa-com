Infraestructura com a Codi (IaC) ►

Introducció

La virtualització i els contenidors són tecnologies clau en la infraestructura de TI moderna. Permeten una millor utilització dels recursos, major flexibilitat i escalabilitat, i una gestió més eficient de les aplicacions i serveis.

Virtualització

La virtualització és una tecnologia que permet crear múltiples entorns simulats o virtuals a partir d'un únic sistema físic.

Conceptes Clau

  • Hipervisor: Programari que permet crear i gestionar màquines virtuals (VMs). Hi ha dos tipus:
    • Tipus 1: Corre directament sobre el maquinari (e.g., VMware ESXi, Microsoft Hyper-V).
    • Tipus 2: Corre sobre un sistema operatiu amfitrió (e.g., VMware Workstation, Oracle VirtualBox).
  • Màquina Virtual (VM): Un entorn virtual que emula un sistema físic complet, incloent-hi el sistema operatiu i aplicacions.
  • Snapshots: Captures de l'estat d'una VM en un moment específic, permetent restaurar aquest estat posteriorment.

Avantatges

  • Aïllament: Cada VM està aïllada de la resta, cosa que millora la seguretat i estabilitat.
  • Consolidació: Permet executar múltiples VMs en un sol servidor físic, optimitzant l'ús de recursos.
  • Flexibilitat: Facilita la creació i eliminació d'entorns segons sigui necessari.

Exemple de Configuració d'una VM amb VirtualBox

# Instal·lar VirtualBox a Ubuntu
sudo apt update
sudo apt install virtualbox

# Crear una nova VM
VBoxManage createvm --name "MiVM" --register

# Configurar la VM
VBoxManage modifyvm "MiVM" --memory 2048 --cpus 2 --nic1 nat

# Afegir un disc dur virtual
VBoxManage createhd --filename "MiVM.vdi" --size 20000

# Adjuntar el disc dur a la VM
VBoxManage storagectl "MiVM" --name "SATA Controller" --add sata --controller IntelAhci
VBoxManage storageattach "MiVM" --storagectl "SATA Controller" --port 0 --device 0 --type hdd --medium "MiVM.vdi"

# Iniciar la VM
VBoxManage startvm "MiVM"

Contenidors

Els contenidors són una forma lleugera de virtualització que permet empaquetar una aplicació i les seves dependències en un sol paquet executable.

Conceptes Clau

  • Docker: La plataforma de contenidors més popular que permet crear, desplegar i gestionar contenidors.
  • Imatge: Un paquet immutable que conté tot el necessari per executar una aplicació (codi, runtime, biblioteques, etc.).
  • Contenidor: Una instància en execució d'una imatge.
  • Orquestració: Eines com Kubernetes que gestionen la implementació, escalat i operació de contenidors.

Avantatges

  • Lleugeresa: Els contenidors comparteixen el nucli del sistema operatiu, cosa que els fa més lleugers que les VMs.
  • Portabilitat: Les imatges de contenidors poden executar-se en qualsevol entorn que suporti Docker.
  • Escalabilitat: Faciliten l'escalat horitzontal d'aplicacions.

Exemple de Configuració d'un Contenidor amb Docker

# Instal·lar Docker a Ubuntu
sudo apt update
sudo apt install docker.io

# Iniciar el servei Docker
sudo systemctl start docker
sudo systemctl enable docker

# Descarregar una imatge de Docker
sudo docker pull nginx

# Executar un contenidor basat en la imatge descarregada
sudo docker run -d -p 80:80 --name mi_nginx nginx

# Llistar els contenidors en execució
sudo docker ps

# Aturar i eliminar el contenidor
sudo docker stop mi_nginx
sudo docker rm mi_nginx

Comparació entre Virtualització i Contenidors

| Característica | Virtualització (VMs) | Contenidors | |-----------------------|-------------------------------------------|--------------------------------------| | Aïllament | Alt (cada VM té el seu propi SO) | Mig (comparteixen el nucli del SO) | | Ús de Recursos | Major (cada VM necessita el seu propi SO) | Menor (comparteixen el nucli del SO) | | Portabilitat | Menor (depenen de l'hipervisor) | Major (poden executar-se en qualsevol entorn Docker) | | Temps d'Arrencada | Lent (arrencada completa del SO) | Ràpid (arrencada del contenidor) | | Escalabilitat | Menor (més recursos necessaris per VM) | Major (més lleuger i fàcil d'escalar) |

Conclusió

La virtualització i els contenidors són tecnologies complementàries que juguen un paper crucial en la infraestructura de TI moderna. La virtualització ofereix un alt grau d'aïllament i flexibilitat, mentre que els contenidors proporcionen una solució lleugera i altament portable per a la implementació d'aplicacions. L'elecció entre una o altra dependrà de les necessitats específiques de cada projecte i entorn.
