Introducció

El disseny i l'arquitectura de xarxes són components essencials en la infraestructura de TI. Aquest tema abasta els principis fonamentals, les millors pràctiques i les tecnologies utilitzades per construir xarxes eficients, segures i escalables.

Conceptes Clau

  • Topologia de Xarxa: La disposició física o lògica dels dispositius en una xarxa.

    • Topologia en Estrella
    • Topologia en Bus
    • Topologia en Anell
    • Topologia en Malla

  • Models de Referència:

    • Model OSI (Open Systems Interconnection): Divideix les funcions de xarxa en 7 capes.
    • Model TCP/IP: Més pràctic i utilitzat, amb 4 capes principals.

  • Dispositius de Xarxa:

    • Routers: Encaminen paquets de dades entre xarxes.
    • Switches: Connecten dispositius dins de la mateixa xarxa.
    • Firewalls: Protegeixen la xarxa d'accessos no autoritzats.
    • Access Points: Permeten la connexió sense fils de dispositius.

Disseny de Xarxa

Topologies de Xarxa

  • Topologia en Estrella: Tots els dispositius estan connectats a un node central.

    • Avantatges: Fàcil d'instal·lar i gestionar, fallades en un cable no afecten altres dispositius.
    • Desavantatges: Si el node central falla, tota la xarxa cau.

  • Topologia en Bus: Tots els dispositius comparteixen un únic cable de comunicació.

    • Avantatges: Econòmica i fàcil d'instal·lar.
    • Desavantatges: Difícil de solucionar problemes, si el cable principal falla, tota la xarxa cau.

  • Topologia en Anell: Cada dispositiu té dues connexions, una a cada veí, formant un anell.

    • Avantatges: Les dades flueixen en una direcció, reduint col·lisions.
    • Desavantatges: Si un dispositiu falla, pot afectar tota la xarxa.

  • Topologia en Malla: Cada dispositiu està connectat a diversos altres dispositius.

    • Avantatges: Alta redundància i fiabilitat.
    • Desavantatges: Costosa i complexa d'instal·lar.

Models de Referència

Model OSI

| Capa | Funció | |------|---------| | 1. Física | Transmissió de bits a través de mitjans físics | | 2. Enllaç de Dades | Establiment d'enllaços de dades fiables | | 3. Xarxa | Determinació de rutes i adreçament | | 4. Transport | Transferència de dades fiable i control de flux | | 5. Sessió | Gestió de sessions de comunicació | | 6. Presentació | Traducció, xifrat i compressió de dades | | 7. Aplicació | Serveis de xarxa per a aplicacions |

Model TCP/IP

| Capa | Funció | |------|---------| | 1. Accés a la Xarxa | Comunicació amb el maquinari de xarxa | | 2. Internet | Enrutament de paquets a través de xarxes | | 3. Transport | Comunicació de punta a punta | | 4. Aplicació | Serveis de xarxa per a aplicacions |

Dispositius de Xarxa

  • Routers: Utilitzen protocols d'enrutament com OSPF i BGP per determinar la millor ruta per a les dades.
  • Switches: Operen a la capa 2 del model OSI i utilitzen adreces MAC per enviar dades al dispositiu correcte.
  • Firewalls: Poden ser basats en maquinari o programari i utilitzen regles per permetre o bloquejar trànsit.
  • Access Points: Faciliten la connexió de dispositius sense fils a la xarxa cablejada.

Exemple de Configuració de Xarxa

# Configuració bàsica d'un router Cisco
enable
configure terminal
hostname Router1
interface GigabitEthernet0/0
 ip address 192.168.1.1 255.255.255.0
 no shutdown
exit
interface GigabitEthernet0/1
 ip address 10.0.0.1 255.255.255.0
 no shutdown
exit
router ospf 1
 network 192.168.1.0 0.0.0.255 area 0
 network 10.0.0.0 0.0.0.255 area 0
end
write memory

Conclusió

El disseny i l'arquitectura de xarxes són fonamentals per a la creació d'infraestructures TI robustes i eficients. Comprendre les diferents topologies, models de referència i dispositius de xarxa permet als professionals de TI construir i mantenir xarxes que suportin les necessitats de les organitzacions modernes.
