La Programació Orientada a Objectes (OOP) és un paradigma de programació que utilitza "objectes" per dissenyar aplicacions i programes informàtics. En R, la OOP s'implementa a través de diferents sistemes, principalment S3, S4 i Classes de Referència (R5). Aquesta secció et guiarà a través d'aquests sistemes, proporcionant explicacions clares, exemples i exercicis.

Introducció a la Programació Orientada a Objectes

  • Definició: La OOP és un paradigma de programació basat en el concepte d'"objectes", que poden contenir dades i codi per manipular aquestes dades.
  • Beneficis:
    • Encapsulació: Agrupació de dades amb mètodes que operen sobre aquestes dades.
    • Herència: Mecanisme per crear noves classes a partir de les existents.
    • Polimorfisme: Capacitat de processar objectes de manera diferent segons la seva classe.

Sistema S3

El sistema S3 és el sistema OOP més senzill i més utilitzat en R. És informal i es basa en convencions de noms.

Creació d'Objectes S3

  • Creació d'un objecte S3: Un objecte S3 és normalment una llista amb un atribut de classe.
# Crear un objecte S3 senzill
person <- list(name = "John Doe", age = 30)
class(person) <- "Person"
  • Definició de mètodes S3: Els mètodes són funcions que operen sobre objectes d'una classe específica.
# Definir un mètode print per a la classe Person
print.Person <- function(x) {
  cat("Nom:", x$name, "\nEdat:", x$age, "\n")
}

# Provar el mètode print
print(person)

Exercicis

  1. Crear un objecte S3 per a un "Cotxe" amb atributs com make, model i year.
  2. Definir un mètode print per a la classe "Cotxe".

Sistema S4

El sistema S4 és més formal i rigorós que el S3, proporcionant millor validació de dades i encapsulació.

Creació de Classes i Objectes S4

  • Definició d'una classe S4: Utilitza la funció setClass.
# Definir una classe S4
setClass("Person",
         slots = list(name = "character", age = "numeric"))

# Crear un objecte S4
john <- new("Person", name = "John Doe", age = 30)
  • Definició de mètodes S4: Utilitza la funció setMethod.
# Definir un mètode show per a la classe Person
setMethod("show", "Person", function(object) {
  cat("Nom:", object@name, "\nEdat:", object@age, "\n")
})

# Provar el mètode show
john

Exercicis

  1. Definir una classe S4 per a un "Cotxe" amb slots per a make, model i year.
  2. Crear un objecte S4 per a un cotxe i definir un mètode show per a aquest.

Classes de Referència (R5)

Les Classes de Referència (R5) proporcionen un enfocament OOP més tradicional, similar a altres llenguatges de programació com Java o C++.

Creació de Classes de Referència

  • Definició d'una Classe de Referència: Utilitza la funció setRefClass.
# Definir una Classe de Referència
Person <- setRefClass("Person",
                      fields = list(name = "character", age = "numeric"),
                      methods = list(
                        show = function() {
                          cat("Nom:", name, "\nEdat:", age, "\n")
                        }
                      ))

# Crear una instància de la Classe de Referència
john <- Person$new(name = "John Doe", age = 30)
john$show()

Exercicis

  1. Definir una Classe de Referència per a un "Cotxe" amb camps per a make, model i year.
  2. Crear una instància de la classe Cotxe i definir un mètode per mostrar els seus detalls.

Comparació dels Sistemes OOP en R

| Característica | S3 | S4 | Classes de Referència (R5) | |----------------|---------------|---------------|----------------------------| | Formalitat | Informal | Formal | Formal | | Validació de Dades | No | Sí | Sí | | Encapsulació | Parcial | Completa | Completa | | Herència | Sí | Sí | Sí | | Polimorfisme | Sí | Sí | Sí |

Conclusió

La Programació Orientada a Objectes en R ofereix múltiples sistemes per adaptar-se a diferents necessitats i nivells de formalitat. El sistema S3 és senzill i flexible, adequat per a tasques ràpides i informals. El sistema S4 proporciona una millor validació de dades i encapsulació, ideal per a aplicacions complexes. Les Classes de Referència (R5) ofereixen un enfocament OOP més tradicional, similar a altres llenguatges de programació.

Comprenent i practicant aquests sistemes, pots aprofitar el poder de la OOP per escriure codi més organitzat, reutilitzable i mantenible en R.
