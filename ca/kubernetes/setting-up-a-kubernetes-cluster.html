Configurar un clúster de Kubernetes és una habilitat fonamental per a qualsevol persona que vulgui gestionar aplicacions contenidoritzades a escala. Aquesta secció et guiarà a través del procés de configurar un clúster de Kubernetes des de zero, cobrint tant conceptes teòrics com passos pràctics.

Prerequisits

Abans de configurar un clúster de Kubernetes, assegura't de tenir el següent:

  • Comprensió bàsica de la contenidorització (Docker).
  • Una màquina amb almenys 2 CPU i 2GB de RAM.
  • Un sistema operatiu basat en Linux (es recomana Ubuntu 20.04).
  • Connexió a Internet per descarregar els paquets necessaris.

Conceptes Clau

  • Clúster de Kubernetes: Un conjunt de nodes (màquines) que executen aplicacions contenidoritzades gestionades per Kubernetes.
  • Node Mestre: El node que controla i gestiona el clúster.
  • Nodes Treballadors: Nodes que executen les aplicacions contenidoritzades.
  • kubectl: L'eina de línia de comandes per interactuar amb el clúster de Kubernetes.

Guia Pas a Pas

1. Instal·lar Docker

Kubernetes utilitza Docker per gestionar contenidors. Instal·la Docker en tots els nodes (tant mestres com treballadors).

sudo apt-get update
sudo apt-get install -y docker.io
sudo systemctl start docker
sudo systemctl enable docker

2. Instal·lar kubeadm, kubelet, i kubectl

Aquests són els components essencials per configurar i gestionar un clúster de Kubernetes.

sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl
sudo curl -fsSL https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
sudo bash -c 'cat <<EOF >/etc/apt/sources.list.d/kubernetes.list
deb https://apt.kubernetes.io/ kubernetes-xenial main
EOF'
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

3. Inicialitzar el Node Mestre

Inicialitza el node mestre utilitzant kubeadm.

sudo kubeadm init --pod-network-cidr=192.168.0.0/16

Després de la inicialització, veuràs una comanda per unir nodes treballadors al clúster. Guarda aquesta comanda per a ús posterior.

4. Configurar kubectl per al Node Mestre

Configura kubectl per al node mestre.

mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

5. Instal·lar un Complement de Xarxa de Pods

Una xarxa de pods permet la comunicació entre nodes. Instal·la un complement de xarxa com Calico.

kubectl apply -f https://docs.projectcalico.org/manifests/calico.yaml

6. Unir Nodes Treballadors al Clúster

Executa la comanda d'unió en cada node treballador. Aquesta comanda es va proporcionar al final del procés kubeadm init.

sudo kubeadm join <master-node-ip>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>

7. Verificar el Clúster

Comprova l'estat dels nodes per assegurar-te que tots formen part del clúster.

kubectl get nodes

Resolució de Problemes

  • Problema: Els nodes treballadors no s'uneixen al clúster.

    • Solució: Assegura't que el token i el hash són correctes i que els nodes treballadors poden arribar al node mestre.

  • Problema: Els pods no s'inicien.

    • Solució: Comprova la instal·lació del complement de xarxa de pods i assegura't que tots els nodes estan en estat Ready.

Conclusió

Configurar un clúster de Kubernetes implica diversos passos, des d'instal·lar els components necessaris fins a configurar la xarxa i unir nodes treballadors. Seguint aquesta guia, hauràs de tenir un clúster de Kubernetes funcional preparat per desplegar i gestionar aplicacions contenidoritzades. Recorda comprovar regularment l'estat dels teus nodes i resoldre qualsevol problema que sorgeixi per mantenir un clúster saludable.
