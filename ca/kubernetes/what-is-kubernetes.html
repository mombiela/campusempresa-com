Introducció

Kubernetes, sovint abreujat com K8s, és una plataforma de codi obert dissenyada per automatitzar el desplegament, l'escalat i l'operació de contenidors d'aplicacions. Originalment desenvolupat per Google, Kubernetes ara és mantingut per la Cloud Native Computing Foundation (CNCF).

Conceptes Clau

  • Orquestració de Contenidors: Kubernetes gestiona el cicle de vida de les aplicacions contenidoritzades a través d'un clúster de nodes.
  • Clúster: Un conjunt de nodes (màquines) que executen aplicacions contenidoritzades gestionades per Kubernetes.
  • Node: Una sola màquina en un clúster de Kubernetes, que pot ser una màquina física o virtual.
  • Pod: La unitat de desplegament més petita en Kubernetes, que pot contenir un o més contenidors.
  • Servei: Una abstracció que defineix un conjunt lògic de Pods i una política per accedir-hi.
  • Namespace: Una manera de dividir els recursos del clúster entre múltiples usuaris.

Per què Kubernetes?

  • Escalabilitat: Escalar automàticament les aplicacions amunt i avall segons la demanda.
  • Alta Disponibilitat: Assegurar que les aplicacions estiguin sempre en funcionament i accessibles.
  • Eficiència de Recursos: Optimitzar l'ús dels recursos de maquinari subjacents.
  • Portabilitat: Executar aplicacions de manera consistent en diferents entorns (on-premises, núvol, híbrid).

Arquitectura Bàsica

L'arquitectura de Kubernetes consta dels següents components:

| Component | Descripció | |------------------|-----------------------------------------------------------------------------| | Node Mestre | Gestiona el clúster de Kubernetes. | | Nodes Treballadors| Executen les aplicacions contenidoritzades. | | etcd | Un magatzem de claus-valor consistent i altament disponible utilitzat per a dades de configuració.| | kube-apiserver | Exposa l'API de Kubernetes. | | kube-scheduler | Assigna càrregues de treball als nodes segons la disponibilitat de recursos.| | kube-controller-manager | Gestiona diversos controladors que regulen l'estat del clúster. | | kubelet | Un agent que s'executa en cada node treballador i assegura que els contenidors estiguin en funcionament.| | kube-proxy | Manté les regles de xarxa als nodes. |

Exemple: Desplegant una Aplicació Simple

Pas 1: Crear un Deployment

Un Deployment assegura que un nombre especificat de rèpliques de pod estiguin en funcionament en tot moment.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

Pas 2: Aplicar el Deployment

Utilitza l'eina de línia de comandes kubectl per aplicar la configuració del deployment.

kubectl apply -f nginx-deployment.yaml

Pas 3: Verificar el Deployment

Comprova l'estat del deployment per assegurar-te que està funcionant correctament.

kubectl get deployments
kubectl get pods

Pas 4: Exposar el Deployment

Crea un Servei per exposar el deployment al trànsit extern.

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer

Aplica la configuració del servei:

kubectl apply -f nginx-service.yaml

Conclusió

Kubernetes és una eina poderosa per gestionar aplicacions contenidoritzades a escala. Proporciona un marc robust per desplegar, escalar i mantenir aplicacions, assegurant alta disponibilitat i una utilització eficient dels recursos. Entenent els conceptes bàsics i l'arquitectura de Kubernetes, pots començar a aprofitar les seves capacitats per agilitzar els teus processos de desplegament d'aplicacions.
