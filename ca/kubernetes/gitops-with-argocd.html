GitOps amb ArgoCD

Introducció a GitOps

  • Definició: GitOps és un conjunt de pràctiques per gestionar la infraestructura i les configuracions d'aplicacions utilitzant Git.
  • Principis Bàsics:
    • Descripcions Declaratives: Totes les configuracions es descriuen de manera declarativa.
    • Control de Versions: Les configuracions es guarden en un sistema de control de versions (per exemple, Git).
    • Aplicació Automàtica: Els canvis en les configuracions s'apliquen automàticament al sistema.
    • Reconciliació Contínua: El sistema assegura contínuament que l'estat actual coincideixi amb l'estat desitjat.

Introducció a ArgoCD

  • Definició: ArgoCD és una eina de lliurament continu declarativa, GitOps, per a Kubernetes.
  • Característiques Clau:
    • GitOps Declaratiu: Utilitza repositoris Git com a font de veritat per a l'estat desitjat de les aplicacions.
    • Sincronització Automàtica: Sincronitza automàticament l'estat del clúster amb l'estat definit a Git.
    • Autocuració: Corregeix automàticament la desviació entre l'estat desitjat i l'estat actual.
    • Suport Multi-Clúster: Gestiona aplicacions a través de múltiples clústers de Kubernetes.

Configuració d'ArgoCD

  • Prerequisits:

    • Un clúster de Kubernetes en funcionament.
    • Eina de línia de comandes kubectl configurada per interactuar amb el teu clúster.
    • Un repositori Git amb manifests de Kubernetes.

  • Passos d'Instal·lació:

    1. Instal·lar ArgoCD a Kubernetes: 
      kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
    2. Accedir al Servidor API d'ArgoCD: 
      kubectl port-forward svc/argocd-server -n argocd 8080:443
    3. Iniciar Sessió a ArgoCD: 
      # Obtenir la contrasenya inicial d'administrador
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
# Obrir la interfície d'usuari d'ArgoCD al teu navegador a https://localhost:8080

Configuració d'ArgoCD

  • Connectar un Repositori Git:

    • Crear una Aplicació: 
      argocd app create my-app 
            --repo https://github.com/my-org/my-repo.git 
            --path path/to/manifests 
            --dest-server https://kubernetes.default.svc 
            --dest-namespace default
    • Sincronitzar l'Aplicació: 
      argocd app sync my-app

  • Configuració de l'Aplicació:

    • Exemple de Manifest d'Aplicació: 
      apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: guestbook
  namespace: argocd
spec:
  project: default
  source:
    repoURL: 'https://github.com/argoproj/argocd-example-apps'
    targetRevision: HEAD
    path: guestbook
  destination:
    server: 'https://kubernetes.default.svc'
    namespace: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true

Funcions Avançades d'ArgoCD

  • Polítiques de Sincronització Automàtica:

    • Autocuració: Corregeix automàticament qualsevol desviació entre l'estat desitjat i l'estat actual.
    • Poda: Elimina automàticament els recursos que ja no estan definits al repositori Git.

  • Gestió Multi-Clúster:

    • Afegir un Clúster: 
      argocd cluster add <context-name>

  • Comprovacions de Salut Personalitzades:

    • Definir Comprovacions de Salut Personalitzades: 
      apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: guestbook
  namespace: argocd
spec:
  project: default
  source:
    repoURL: 'https://github.com/argoproj/argocd-example-apps'
    targetRevision: HEAD
    path: guestbook
  destination:
    server: 'https://kubernetes.default.svc'
    namespace: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
  healthChecks:
    - apiVersion: apps/v1
      kind: Deployment
      name: guestbook-ui
      health:
        status: Healthy
        message: 'Guestbook UI is healthy'

Conclusió

  • Resum:
    • GitOps amb ArgoCD proporciona una manera poderosa de gestionar aplicacions de Kubernetes de manera declarativa utilitzant Git com a font de veritat.
    • ArgoCD automatitza el procés de desplegament, assegurant que l'estat actual del clúster coincideixi amb l'estat desitjat definit a Git.
    • Funcions avançades com les polítiques de sincronització automàtica, la gestió multi-clúster i les comprovacions de salut personalitzades fan d'ArgoCD una eina versàtil per al lliurament continu de Kubernetes.

Dominant GitOps amb ArgoCD, pots aconseguir desplegaments més fiables i consistents, reduir la intervenció manual i millorar l'estabilitat general dels teus entorns de Kubernetes.
