Escalar aplicacions és un aspecte fonamental de la gestió de càrregues de treball a Kubernetes. Aquest tema et guiarà a través dels conceptes, eines i millors pràctiques per escalar aplicacions des d'un nivell principiant fins a un nivell avançat.

Introducció a l'Escalat

L'escalat a Kubernetes implica ajustar el nombre de rèpliques de la teva aplicació per gestionar càrregues variables. Això es pot fer manualment o automàticament.

Conceptes Clau

  • ReplicaSet: Assegura que un nombre especificat de rèpliques de pod estiguin en execució en qualsevol moment.
  • Horizontal Pod Autoscaler (HPA): Escala automàticament el nombre de pods basant-se en la utilització de CPU observada o altres mètriques seleccionades.
  • Vertical Pod Autoscaler (VPA): Ajusta els límits de recursos i les sol·licituds per als contenidors en els pods.
  • Cluster Autoscaler: Ajusta la mida del clúster de Kubernetes afegint o eliminant nodes.

Escalat Manual

L'escalat manual implica establir explícitament el nombre de rèpliques per a la teva aplicació.

Exemple

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app-container
        image: my-app-image
        ports:
        - containerPort: 80

En aquest exemple, el camp replicas està establert a 3, el que significa que Kubernetes mantindrà 3 rèpliques del pod my-app.

Escalar amb kubectl

També pots escalar la teva aplicació utilitzant la comanda kubectl:

kubectl scale deployment my-app --replicas=5

Aquesta comanda escala el desplegament my-app a 5 rèpliques.

Horizontal Pod Autoscaler (HPA)

HPA escala automàticament el nombre de pods en un desplegament o conjunt de rèpliques basant-se en les mètriques observades.

Exemple

Primer, assegura't que el Metrics Server està instal·lat al teu clúster:

kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

Després, crea un recurs HPA:

apiVersion: autoscaling/v1
kind: HorizontalPodAutoscaler
metadata:
  name: my-app-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  minReplicas: 1
  maxReplicas: 10
  targetCPUUtilizationPercentage: 50

Aquest HPA escalarà el desplegament my-app entre 1 i 10 rèpliques per mantenir una utilització mitjana de CPU del 50%.

Aplicar l'HPA

kubectl apply -f my-app-hpa.yaml

Vertical Pod Autoscaler (VPA)

VPA ajusta les sol·licituds i límits de recursos per als contenidors en els teus pods.

Exemple

Primer, instal·la els components VPA:

kubectl apply -f https://github.com/kubernetes/autoscaler/releases/latest/download/vertical-pod-autoscaler.yaml

Després, crea un recurs VPA:

apiVersion: autoscaling.k8s.io/v1
kind: VerticalPodAutoscaler
metadata:
  name: my-app-vpa
spec:
  targetRef:
    apiVersion: "apps/v1"
    kind:       Deployment
    name:       my-app
  updatePolicy:
    updateMode: "Auto"

Aquest VPA ajustarà automàticament les sol·licituds i límits de recursos per al desplegament my-app.

Aplicar el VPA

kubectl apply -f my-app-vpa.yaml

Cluster Autoscaler

Cluster Autoscaler ajusta el nombre de nodes en el teu clúster basant-se en els requisits de recursos dels teus pods.

Exemple

Cluster Autoscaler es configura típicament a través del teu proveïdor de núvol. Per exemple, a Google Kubernetes Engine (GKE), pots habilitar-lo a través de la consola de GKE o utilitzant comandes gcloud.

Habilitar Cluster Autoscaler a GKE

gcloud container clusters update my-cluster --enable-autoscaling --min-nodes=1 --max-nodes=10 --zone=us-central1-a

Resum

Escalar aplicacions a Kubernetes és essencial per mantenir el rendiment i l'eficiència. Pots escalar manualment o utilitzar funcions d'autoscalat com HPA, VPA i Cluster Autoscaler per automatitzar el procés. Entendre aquestes eines i com aplicar-les t'ajudarà a gestionar les teves càrregues de treball de manera efectiva.

En dominar aquestes tècniques d'escalat, pots assegurar-te que les teves aplicacions siguin resilients, rendibles i capaces de gestionar càrregues variables.
