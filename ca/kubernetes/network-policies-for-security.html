Introducció

Les Polítiques de Xarxa de Kubernetes són crucials per assegurar el teu clúster controlant la comunicació entre pods i serveis. Aquest tema et guiarà des dels conceptes bàsics fins als avançats, assegurant que entenguis com implementar i gestionar les polítiques de xarxa de manera efectiva.

Què són les Polítiques de Xarxa?

Les Polítiques de Xarxa en Kubernetes són regles que defineixen com els pods es comuniquen entre ells i amb altres punts finals de la xarxa. S'implementen utilitzant el recurs NetworkPolicy de Kubernetes.

Conceptes Clau

  • Selector de Pod: Especifica el grup de pods als quals s'aplica la política.
  • Regles d'Ingress: Defineixen el trànsit entrant permès als pods seleccionats.
  • Regles d'Egress: Defineixen el trànsit sortint permès dels pods seleccionats.
  • Namespaces: Les polítiques de xarxa es poden aplicar dins d'un namespace específic o a través de múltiples namespaces.

Exemple Bàsic de Política de Xarxa

Comencem amb un exemple senzill per permetre el trànsit només des de pods específics.

Exemple: Permetre el Trànsit des de Pods Específics

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-specific-pods
  namespace: default
spec:
  podSelector:
    matchLabels:
      role: db
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: frontend

Explicació

  • apiVersion: Especifica la versió de l'API.
  • kind: Defineix el tipus de recurs, que és NetworkPolicy.
  • metadata: Conté el nom i el namespace de la política.
  • spec: La especificació de la política.
    • podSelector: Selecciona pods amb l'etiqueta role: db.
    • ingress: Permet el trànsit entrant des de pods amb l'etiqueta role: frontend.

Conceptes Avançats de Polítiques de Xarxa

A mesura que et familiaritzis amb les polítiques bàsiques, pots explorar funcions més avançades.

Combinació de Regles d'Ingress i Egress

Pots definir tant regles d'ingress com d'egress en una sola política.

Exemple: Regles Combinades d'Ingress i Egress

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: combined-policy
  namespace: default
spec:
  podSelector:
    matchLabels:
      role: backend
  ingress:
  - from:
    - podSelector:
        matchLabels:
          role: frontend
  egress:
  - to:
    - podSelector:
        matchLabels:
          role: db

Explicació

  • podSelector: Selecciona pods amb l'etiqueta role: backend.
  • ingress: Permet el trànsit entrant des de pods amb l'etiqueta role: frontend.
  • egress: Permet el trànsit sortint cap a pods amb l'etiqueta role: db.

Ús de Selectors de Namespace

Les polítiques de xarxa també poden utilitzar selectors de namespace per controlar el trànsit a través de diferents namespaces.

Exemple: Selector de Namespace

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: namespace-selector
  namespace: default
spec:
  podSelector:
    matchLabels:
      role: api
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          environment: production

Explicació

  • namespaceSelector: Selecciona namespaces amb l'etiqueta environment: production.
  • ingress: Permet el trànsit entrant des de qualsevol pod en namespaces etiquetats com environment: production.

Millors Pràctiques per a les Polítiques de Xarxa

  • Menor Privilegi: Comença amb una política de denegació per defecte i després afegeix regles específiques de permís.
  • Consistència d'Etiquetes: Utilitza etiquetatge consistent a través dels teus pods i namespaces per simplificar la gestió de polítiques.
  • Proves: Prova les teves polítiques de xarxa en un entorn de staging abans d'aplicar-les a producció.

Conclusió

Les Polítiques de Xarxa són una eina poderosa per assegurar el teu clúster de Kubernetes controlant la comunicació entre pods. Entenent i implementant aquestes polítiques, pots assegurar que les teves aplicacions siguin segures i compleixin amb les polítiques organitzacionals. Comença amb polítiques bàsiques i gradualment passa a configuracions més complexes a mesura que les teves necessitats evolucionin.
