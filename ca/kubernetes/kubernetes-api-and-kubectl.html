Introducció

En aquesta secció, explorarem l'API de Kubernetes i kubectl, l'eina de línia de comandes per interactuar amb clústers de Kubernetes. Entendre aquestes eines és essencial per gestionar i automatitzar tasques dins d'un entorn Kubernetes.

API de Kubernetes

L'API de Kubernetes és la interfície a través de la qual els usuaris, components externs i components interns es comuniquen entre si. És una API RESTful que et permet realitzar operacions CRUD (Crear, Llegir, Actualitzar, Esborrar) sobre recursos de Kubernetes.

Conceptes Clau

  • Servidor API: El component central del pla de control de Kubernetes que exposa l'API de Kubernetes.
  • Recursos: Objectes en l'API de Kubernetes, com ara Pods, Serveis, Desplegaments, etc.
  • Endpoints: URLs específiques que representen diferents operacions sobre recursos.

Exemple: Accedir a l'API de Kubernetes

Pots interactuar amb l'API de Kubernetes directament utilitzant sol·licituds HTTP. Aquí tens un exemple de com llistar tots els Pods en un espai de noms utilitzant curl:

curl -X GET https://<kubernetes-api-server>/api/v1/namespaces/default/pods \
  -H "Authorization: Bearer <your-access-token>" \
  -H "Accept: application/json"

kubectl

kubectl és l'eina de línia de comandes per interactuar amb l'API de Kubernetes. Simplifica el procés de gestionar recursos de Kubernetes proporcionant un conjunt de comandes que abstrauen les crides API subjacents.

Conceptes Clau

  • Comandes: Les comandes de kubectl segueixen el patró kubectl <comanda> <recurs> <nom>.
  • Configuració: kubectl utilitza un fitxer de configuració (normalment situat a ~/.kube/config) per accedir al clúster de Kubernetes.
  • Contextos: Els contextos en el fitxer de configuració et permeten canviar entre diferents clústers i espais de noms.

Comandes Bàsiques

Aquí tens algunes comandes bàsiques de kubectl per començar:

  • Obtenir Pods: Llistar tots els Pods en l'espai de noms per defecte.

    kubectl get pods

  • Descriure Pod: Obtenir informació detallada sobre un Pod específic.

    kubectl describe pod <pod-name>

  • Crear Recurs: Crear un recurs a partir d'un fitxer YAML.

    kubectl apply -f <file.yaml>

  • Esborrar Recurs: Esborrar un recurs específic.

    kubectl delete <resource-type> <resource-name>

Comandes Avançades

Per a operacions més avançades, kubectl proporciona comandes addicionals:

  • Escalar Desplegament: Escalar un desplegament a un nombre especificat de rèpliques.

    kubectl scale deployment <deployment-name> --replicas=<number>

  • Actualització Progressiva: Realitzar una actualització progressiva en un desplegament.

    kubectl rollout restart deployment <deployment-name>

  • Redirecció de Ports: Redirigir un port local a un port en un Pod.

    kubectl port-forward <pod-name> <local-port>:<pod-port>

Gestió de Configuració

Gestionar múltiples clústers i espais de noms es pot fer utilitzant contextos en el fitxer de configuració de kubectl.

  • Veure Context Actual:

    kubectl config current-context

  • Establir Context:

    kubectl config use-context <context-name>

  • Llistar Contextos:

    kubectl config get-contexts

Conclusió

Entendre l'API de Kubernetes i kubectl és fonamental per gestionar efectivament clústers de Kubernetes. L'API proporciona una interfície potent per a l'automatització i la integració, mentre que kubectl ofereix una manera fàcil d'usar per interactuar amb el clúster. Dominar aquestes eines millorarà significativament la teva capacitat per desplegar, gestionar i resoldre problemes d'aplicacions en Kubernetes.
