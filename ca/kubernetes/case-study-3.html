Introducció

En aquest estudi de cas, explorarem com implementar la Integració Contínua i el Desplegament Continu (CI/CD) utilitzant Kubernetes. Això inclourà configurar una pipeline de CI/CD, integrar-se amb sistemes de control de versions i desplegar aplicacions a un clúster de Kubernetes.

Conceptes Clau

  • Integració Contínua (CI): La pràctica de construir i provar automàticament els canvis de codi, assegurant que la base de codi es mantingui en un estat desplegable.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament els canvis de codi a un entorn de producció després de passar la pipeline de CI.
  • Kubernetes: Una plataforma de codi obert dissenyada per automatitzar el desplegament, l'escalat i l'operació de contenidors d'aplicacions.

Configuració de l'Entorn

Per implementar CI/CD amb Kubernetes, necessitaràs les següents eines:

  • Git: Sistema de control de versions.
  • Jenkins: Servidor d'automatització de codi obert per construir pipelines de CI/CD.
  • Docker: Plataforma de contenidorització.
  • Clúster de Kubernetes: L'entorn on es desplegaran les teves aplicacions.

Implementació Pas a Pas

1. Configuració de Jenkins

Primer, instal·la Jenkins i configura'l per treballar amb el teu clúster de Kubernetes.

# Instal·la Jenkins utilitzant Docker
docker run -d -p 8080:8080 -p 50000:50000 jenkins/jenkins:lts

2. Configuració de Jenkins per Kubernetes

Instal·la el plugin de Kubernetes a Jenkins i configura'l per connectar-se al teu clúster de Kubernetes.

  • Ves al Tauler de Jenkins > Gestiona Jenkins > Gestiona Plugins.
  • Instal·la el plugin "Kubernetes".
  • Configura el plugin de Kubernetes amb els detalls del teu clúster.

3. Creació d'una Pipeline de Jenkins

Crea un script de pipeline de Jenkins per automatitzar el procés de construcció, prova i desplegament.

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                script {
                    docker.build('my-app')
                }
            }
        }
        stage('Test') {
            steps {
                script {
                    docker.image('my-app').inside {
                        sh 'make test'
                    }
                }
            }
        }
        stage('Deploy') {
            steps {
                script {
                    kubernetesDeploy(
                        configs: 'k8s/deployment.yaml',
                        kubeconfigId: 'kube-config'
                    )
                }
            }
        }
    }
}

4. Escritura de la Configuració de Desplegament de Kubernetes

Crea un fitxer de configuració de desplegament de Kubernetes (k8s/deployment.yaml).

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:latest
        ports:
        - containerPort: 80

5. Integració amb Git

Configura Jenkins per desencadenar construccions en canvis de codi.

  • Ves al Tauler de Jenkins > Nou Element > Pipeline.
  • Configura la pipeline per utilitzar el teu repositori de Git.
  • Configura un webhook al teu repositori de Git per desencadenar construccions de Jenkins en esdeveniments de push.

Conclusió

En aquest estudi de cas, hem recorregut el procés de configurar una pipeline de CI/CD utilitzant Jenkins i Kubernetes. Seguint aquests passos, pots automatitzar la construcció, prova i desplegament de les teves aplicacions, assegurant un flux de treball de desenvolupament suau i eficient.

Implementant CI/CD, pots:

  • Reduir errors manuals.
  • Augmentar la velocitat de desplegament.
  • Millorar la qualitat del codi mitjançant proves automatitzades.

Aquest estudi de cas proporciona una comprensió fonamental de CI/CD amb Kubernetes, que pots ampliar per adaptar-lo a les necessitats específiques del teu projecte.
