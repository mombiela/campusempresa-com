Introducció a les Classes d'emmagatzematge

Les Classes d'emmagatzematge a Kubernetes proporcionen una manera de descriure les "classes" d'emmagatzematge disponibles en un clúster. Permeten als administradors definir diferents tipus d'emmagatzematge (per exemple, SSDs, HDDs) i les seves propietats (per exemple, rendiment, disponibilitat).

Conceptes Clau

  • StorageClass: Un recurs de Kubernetes que defineix una classe d'emmagatzematge.
  • PersistentVolume (PV): Una peça d'emmagatzematge en el clúster que ha estat provisionada per un administrador o provisionada dinàmicament utilitzant StorageClasses.
  • PersistentVolumeClaim (PVC): Una sol·licitud d'emmagatzematge per part d'un usuari. Pot sol·licitar una mida específica i modes d'accés.
  • Dynamic Provisioning: El procés de crear automàticament un PersistentVolume quan es crea un PersistentVolumeClaim.

Creació d'una StorageClass

Per crear una StorageClass, cal definir-la en un fitxer YAML. Aquí teniu un exemple:

apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: fast
provisioner: kubernetes.io/aws-ebs
parameters:
  type: gp2
  fsType: ext4

Explicació

  • apiVersion: La versió de l'API de Kubernetes.
  • kind: El tipus de recurs, en aquest cas, StorageClass.
  • metadata: Metadades sobre el recurs, incloent-hi el seu nom.
  • provisioner: El tipus de provisioner a utilitzar (per exemple, AWS EBS, GCE PD).
  • parameters: Paràmetres específics per al provisioner, com ara el tipus de disc i el sistema de fitxers.

Ús d'una StorageClass

Per utilitzar una StorageClass, creeu un PersistentVolumeClaim que la referenciï. Aquí teniu un exemple:

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: mypvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
  storageClassName: fast

Explicació

  • apiVersion: La versió de l'API de Kubernetes.
  • kind: El tipus de recurs, en aquest cas, PersistentVolumeClaim.
  • metadata: Metadades sobre el recurs, incloent-hi el seu nom.
  • spec: L'especificació de la sol·licitud.
    • accessModes: Els modes d'accés per al volum (per exemple, ReadWriteOnce, ReadOnlyMany, ReadWriteMany).
    • resources: Els recursos sol·licitats, incloent-hi la quantitat d'emmagatzematge.
    • storageClassName: El nom de la StorageClass a utilitzar.

Provisionament Dinàmic

Quan es crea un PersistentVolumeClaim i es referència una StorageClass, Kubernetes crearà automàticament un PersistentVolume que coincideixi amb els requisits de la sol·licitud.

Exemple

Aquí teniu un exemple d'un PersistentVolume provisionat dinàmicament:

apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv0001
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  storageClassName: fast
  awsElasticBlockStore:
    volumeID: vol-0a89b7e7c6e5f8b9a
    fsType: ext4

Explicació

  • apiVersion: La versió de l'API de Kubernetes.
  • kind: El tipus de recurs, en aquest cas, PersistentVolume.
  • metadata: Metadades sobre el recurs, incloent-hi el seu nom.
  • spec: L'especificació del volum.
    • capacity: La capacitat del volum.
    • accessModes: Els modes d'accés per al volum.
    • persistentVolumeReclaimPolicy: La política de recuperació del volum (per exemple, Retain, Recycle, Delete).
    • storageClassName: El nom de la StorageClass.
    • awsElasticBlockStore: Configuració específica per als volums AWS EBS.

Temes Avançats

Paràmetres Personalitzats

Podeu definir paràmetres personalitzats a la vostra StorageClass per ajustar la provisió d'emmagatzematge. Per exemple, podeu especificar IOPS per AWS EBS:

apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: high-iops
provisioner: kubernetes.io/aws-ebs
parameters:
  type: io1
  iopsPerGB: "10"
  fsType: ext4

Polítiques de Recuperació

Les polítiques de recuperació determinen què passa amb un PersistentVolume quan es allibera. Els valors possibles són:

  • Retain: El volum es reté després que el PVC es suprimeixi.
  • Recycle: El volum es neteja i es torna a posar a disposició.
  • Delete: El volum es suprimeix.

Modes de Vinculació

Els modes de vinculació controlen quan el PersistentVolume es vincula al PersistentVolumeClaim. Els valors possibles són:

  • Immediate: El volum es vincula immediatament.
  • WaitForFirstConsumer: El volum es vincula només quan un pod utilitza el PVC.

Conclusió

Les Classes d'emmagatzematge a Kubernetes proporcionen una manera flexible i potent de gestionar l'emmagatzematge en el vostre clúster. Definint diferents classes d'emmagatzematge, podeu assegurar-vos que les vostres aplicacions tinguin el tipus d'emmagatzematge adequat per a les seves necessitats. El provisionament dinàmic simplifica el procés de gestió de l'emmagatzematge, permetent-vos centrar-vos en les vostres aplicacions en lloc de la infraestructura subjacent. Entendre i utilitzar les Classes d'emmagatzematge de manera efectiva pot millorar molt el rendiment i la fiabilitat de les vostres implementacions de Kubernetes.
