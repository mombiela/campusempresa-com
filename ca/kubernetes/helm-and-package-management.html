Introducció a Helm

Helm és un gestor de paquets per a Kubernetes que t'ajuda a gestionar aplicacions de Kubernetes. Helm utilitza un format de paquet anomenat charts, que són col·leccions d'arxius que descriuen un conjunt relacionat de recursos de Kubernetes.

Conceptes Clau

  • Charts: Paquets de Helm que contenen totes les definicions de recursos necessàries per executar una aplicació, eina o servei dins d'un clúster de Kubernetes.
  • Repositoris: Ubicacions on es guarden i es comparteixen els charts.
  • Releases: Instàncies de charts executant-se en un clúster de Kubernetes.

Instal·lant Helm

Per començar amb Helm, necessites instal·lar-lo a la teva màquina local. Segueix aquests passos:

  1. Descarrega Helm:

    curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

  2. Verifica la Instal·lació:

    helm version

Helm Charts

Els charts de Helm són la pedra angular de Helm. Defineixen l'estructura i la configuració de les teves aplicacions de Kubernetes.

Creant un Helm Chart

Per crear un nou Helm chart, utilitza la següent comanda:

helm create mychart

Aquesta comanda genera una estructura de directoris com aquesta:

mychart/
  Chart.yaml          # Un arxiu YAML que conté informació sobre el chart
  values.yaml         # Els valors de configuració per defecte per a aquest chart
  charts/             # Un directori que conté qualsevol chart del qual depengui aquest chart
  templates/          # Un directori de plantilles que, quan es combinen amb valors, generen arxius manifest vàlids de Kubernetes

Entenent Chart.yaml

L'arxiu Chart.yaml conté metadades sobre el chart:

apiVersion: v2
name: mychart
description: Un chart de Helm per a Kubernetes
version: 0.1.0
appVersion: "1.0"

Utilitzant values.yaml

L'arxiu values.yaml et permet especificar valors per defecte per a les teves plantilles:

replicaCount: 1
image:
  repository: nginx
  tag: stable
  pullPolicy: IfNotPresent
service:
  type: ClusterIP
  port: 80

Desplegant Aplicacions amb Helm

Desplegar una aplicació utilitzant Helm és senzill. Pots utilitzar la comanda helm install per desplegar un chart.

Instal·lant un Chart

helm install myrelease mychart
  • myrelease: El nom que vols donar a aquesta release.
  • mychart: El camí al chart que vols instal·lar.

Actualitzant una Release

Per actualitzar una release existent, utilitza la comanda helm upgrade:

helm upgrade myrelease mychart

Desinstal·lant una Release

Per desinstal·lar una release, utilitza la comanda helm uninstall:

helm uninstall myrelease

Repositoris de Helm

Els repositoris de Helm són ubicacions on es guarden i es comparteixen els charts. El repositori per defecte de Helm és Helm Hub, però també pots afegir altres repositoris.

Afegint un Repositori

helm repo add stable https://charts.helm.sh/stable

Llistant Repositoris

helm repo list

Cercant Charts

helm search repo nginx

Funcions Avançades de Helm

Helm ofereix diverses funcions avançades que poden ajudar-te a gestionar aplicacions complexes.

Helm Hooks

Els hooks de Helm et permeten intervenir en certs punts del cicle de vida d'una release, com abans o després d'una actualització.

Plugins de Helm

Els plugins de Helm amplien la funcionalitat de Helm. Pots instal·lar plugins utilitzant la comanda helm plugin.

Plantilles de Helm

Helm utilitza plantilles de Go per generar manifests de Kubernetes. Pots utilitzar funcions de plantilla per manipular dades i crear configuracions dinàmiques.

Exemple d'una Plantilla

apiVersion: v1
kind: Pod
metadata:
  name: {{ .Release.Name }}-pod
spec:
  containers:
    - name: {{ .Chart.Name }}
      image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"

Conclusió

Helm és una eina poderosa per gestionar aplicacions de Kubernetes. Simplifica el procés de desplegament i proporciona una manera consistent de empaquetar i compartir aplicacions. Entenent els charts de Helm, els repositoris i les funcions avançades, pots gestionar efectivament les teves aplicacions de Kubernetes des del desenvolupament fins a la producció.
