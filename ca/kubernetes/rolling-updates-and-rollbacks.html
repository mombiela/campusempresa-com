Introducció

Les actualitzacions progressives i les reversions són conceptes essencials en Kubernetes per gestionar els desplegaments d'aplicacions. Permeten actualitzar les teves aplicacions sense temps d'inactivitat i revertir a versions anteriors si alguna cosa surt malament.

Conceptes Clau

  • Actualització Progressiva: Una estratègia de desplegament que actualitza els pods de manera incremental per assegurar un mínim temps d'inactivitat.
  • Reversió: El procés de tornar a una versió de desplegament anterior si l'actual falla o té problemes.
  • Desplegament: Un recurs de Kubernetes que gestiona un conjunt de pods idèntics, assegurant que un nombre especificat d'ells estiguin en execució.

Actualitzacions Progressives

Com Funcionen les Actualitzacions Progressives

Durant una actualització progressiva, Kubernetes reemplaça incrementalment els pods antics per nous. Això assegura que la teva aplicació romangui disponible durant tot el procés d'actualització.

Passos per Realitzar una Actualització Progressiva

  1. Actualitza el Desplegament: Modifica la configuració del desplegament per especificar la nova versió de l'aplicació.
  2. Aplica l'Actualització: Utilitza kubectl apply per aplicar els canvis.
  3. Monitora l'Actualització: Comprova l'estat del desplegament per assegurar-te que l'actualització està procedint com s'espera.

Exemple: Realitzar una Actualització Progressiva

Suposem que tens un desplegament anomenat my-app i vols actualitzar la seva imatge a una nova versió.

  1. Visualitza la configuració actual del desplegament:

    kubectl get deployment my-app -o yaml

  2. Actualitza el desplegament per utilitzar la nova imatge:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app-container
        image: my-app:2.0  # Actualitza la versió de la imatge aquí

  3. Aplica la configuració actualitzada:

    kubectl apply -f my-app-deployment.yaml

  4. Monitora l'estat del desplegament:

    kubectl rollout status deployment/my-app

Estratègies d'Actualització Progressiva

  • Max Surge: El nombre màxim de pods que es poden crear per sobre del nombre desitjat de pods durant l'actualització.
  • Max Unavailable: El nombre màxim de pods que poden estar no disponibles durant l'actualització.

Exemple de configuració:

spec:
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 1

Reversions

Com Funcionen les Reversions

Una reversió torna el desplegament a una versió anterior si la versió actual té problemes. Kubernetes manté un historial de desplegaments anteriors, facilitant la tornada a un estat estable.

Passos per Realitzar una Reversió

  1. Comprova l'Historial de Desplegaments: Visualitza l'historial de desplegaments anteriors.
  2. Reverteix a una Revisió Anterior: Utilitza kubectl rollout undo per tornar a una revisió específica.
  3. Monitora la Reversió: Assegura't que la reversió és exitosa i que l'aplicació és estable.

Exemple: Realitzar una Reversió

  1. Visualitza l'historial del desplegament:

    kubectl rollout history deployment/my-app

  2. Reverteix a una revisió anterior:

    kubectl rollout undo deployment/my-app --to-revision=2

  3. Monitora l'estat de la reversió:

    kubectl rollout status deployment/my-app

Conclusió

Les actualitzacions progressives i les reversions són característiques poderoses en Kubernetes que ajuden a assegurar que les teves aplicacions s'actualitzin de manera suau i es puguin revertir ràpidament si sorgeixen problemes. Entenent i utilitzant aquestes característiques, pots mantenir una alta disponibilitat i fiabilitat per a les teves aplicacions.
