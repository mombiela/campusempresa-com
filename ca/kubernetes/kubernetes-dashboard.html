Introducció al Kubernetes Dashboard

El Kubernetes Dashboard és una interfície d'usuari basada en web que et permet gestionar i monitoritzar els teus clústers de Kubernetes. Proporciona una representació visual dels recursos i càrregues de treball del clúster, facilitant la comprensió i gestió de les teves aplicacions.

Conceptes Clau

  • Gestió del Clúster: Visualitza i gestiona recursos del clúster com nodes, namespaces i càrregues de treball.
  • Gestió de Càrregues de Treball: Desplega, escala i resol problemes d'aplicacions.
  • Monitorització de Recursos: Monitoritza l'ús de recursos i la salut del clúster.
  • Gestió d'Usuaris: Gestiona el control d'accés i permisos.

Instal·lació del Kubernetes Dashboard

Per utilitzar el Kubernetes Dashboard, cal instal·lar-lo al teu clúster. Segueix aquests passos per instal·lar el Dashboard.

Instal·lació Pas a Pas

  1. Desplega el Dashboard:

    kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.3.1/aio/deploy/recommended.yaml

  2. Crea un Compte de Servei:

    kubectl create serviceaccount dashboard-admin-sa
kubectl create clusterrolebinding dashboard-admin-sa --clusterrole=cluster-admin --serviceaccount=default:dashboard-admin-sa

  3. Obtén el Bearer Token:

    kubectl get secret $(kubectl get serviceaccount dashboard-admin-sa -o jsonpath="{.secrets[0].name}") -o jsonpath="{.data.token}" | base64 --decode

  4. Accedeix al Dashboard:

    kubectl proxy

    Obre la següent URL al teu navegador web:

    http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy/

Utilitzant el Kubernetes Dashboard

Un cop el Dashboard està instal·lat i funcionant, pots començar a utilitzar-lo per gestionar el teu clúster.

Navegant pel Dashboard

  • Visió General: Proporciona un resum de l'estat del clúster, incloent nodes, namespaces i càrregues de treball.
  • Càrregues de Treball: Gestiona desplegaments, pods, replica sets i treballs.
  • Serveis i Ingress: Gestiona serveis, recursos ingress i polítiques de xarxa.
  • Configuració i Emmagatzematge: Gestiona ConfigMaps, secrets, reclamacions de volums persistents i classes d'emmagatzematge.
  • Clúster: Visualitza i gestiona nodes, namespaces i rols.
  • Configuració: Configura els ajustaments del Dashboard i gestiona l'accés d'usuaris.

Exemple: Desplegant una Aplicació

  1. Crea un Desplegament:

    • Navega a la secció "Càrregues de Treball".
    • Fes clic a "Desplegaments" i després a "Crear".
    • Omple els detalls necessaris com el nom de l'aplicació, la imatge del contenidor i el nombre de rèpliques.
    • Fes clic a "Desplegar".

  2. Exposa el Desplegament:

    • Navega a la secció "Serveis i Ingress".
    • Fes clic a "Serveis" i després a "Crear".
    • Selecciona el desplegament que has creat i configura el tipus de servei (per exemple, ClusterIP, NodePort, LoadBalancer).
    • Fes clic a "Crear".

Monitorització de Recursos

  • Ús de Recursos: Visualitza l'ús de CPU i memòria per nodes i pods.
  • Registres: Accedeix als registres dels pods individuals per resoldre problemes.
  • Esdeveniments: Visualitza esdeveniments relacionats amb els recursos del clúster per entendre quines accions s'han pres.

Conclusió

El Kubernetes Dashboard és una eina poderosa per gestionar i monitoritzar els teus clústers de Kubernetes. Proporciona una interfície fàcil d'usar per desplegar aplicacions, monitoritzar recursos i resoldre problemes. Seguint els passos d'instal·lació i explorant les característiques del Dashboard, pots gestionar efectivament el teu entorn Kubernetes.

Resum

  • El Kubernetes Dashboard és una interfície d'usuari basada en web per gestionar clústers de Kubernetes.
  • Et permet desplegar aplicacions, monitoritzar recursos i gestionar l'accés d'usuaris.
  • La instal·lació implica desplegar el Dashboard, crear un compte de servei i accedir al Dashboard mitjançant un proxy.
  • El Dashboard proporciona diverses seccions per gestionar càrregues de treball, serveis, configuracions i el clúster en si.

Dominant el Kubernetes Dashboard, pots millorar la teva capacitat per gestionar i optimitzar els teus clústers de Kubernetes de manera eficient.
