Introducció

Kubernetes és una eina poderosa per gestionar aplicacions contenidoritzades en producció. No obstant això, per garantir la fiabilitat, escalabilitat i seguretat, és essencial seguir les millors pràctiques. Aquesta secció cobrirà conceptes clau i estratègies per desplegar Kubernetes en un entorn de producció.

Gestió de Recursos

Una gestió eficient dels recursos és crucial per mantenir el rendiment de l'aplicació i la rendibilitat.

  • Sol·licituds i Límits: Defineix sol·licituds i límits de recursos per a CPU i memòria per garantir una assignació justa de recursos i prevenir la manca de recursos.

    apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    resources:
      requests:
        memory: "64Mi"
        cpu: "250m"
      limits:
        memory: "128Mi"
        cpu: "500m"

  • Espais de Noms: Utilitza espais de noms per organitzar i gestionar els recursos de manera eficient.

    kubectl create namespace production

Seguretat

Assegurar el teu clúster de Kubernetes és fonamental per protegir dades sensibles i garantir el compliment.

  • Control d'Accés Basat en Rols (RBAC): Implementa RBAC per controlar qui pot accedir i realitzar accions al clúster.

    apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "watch", "list"]

  • Polítiques de Xarxa: Defineix polítiques de xarxa per controlar el trànsit entre pods.

    apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-same-namespace
  namespace: default
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector: {}

Monitorització i Registre

La monitorització i el registre són essencials per mantenir la salut i el rendiment de les teves aplicacions.

  • Prometheus i Grafana: Utilitza Prometheus per a la monitorització i Grafana per a la visualització.

    kubectl apply -f https://raw.githubusercontent.com/prometheus-operator/prometheus-operator/master/bundle.yaml

  • ELK Stack: Implementa l'ELK (Elasticsearch, Logstash, Kibana) stack per al registre centralitzat.

    kubectl apply -f https://download.elastic.co/downloads/eck/1.7.1/all-in-one.yaml

Escalabilitat

Assegura't que les teves aplicacions poden gestionar una càrrega augmentada escalant eficientment.

  • Autoscalador Horitzontal de Pods (HPA): Utilitza HPA per escalar automàticament el nombre de rèpliques de pods basant-se en la utilització de CPU o altres mètriques.

    apiVersion: autoscaling/v1
kind: HorizontalPodAutoscaler
metadata:
  name: example-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: example-deployment
  minReplicas: 1
  maxReplicas: 10
  targetCPUUtilizationPercentage: 50

  • Autoscalador de Clúster: Ajusta automàticament la mida del clúster de Kubernetes basant-se en l'ús de recursos.

    kubectl apply -f https://github.com/kubernetes/autoscaler/releases/download/cluster-autoscaler-1.20.0/cluster-autoscaler-autodiscover.yaml

Còpia de Seguretat i Recuperació de Desastres

Prepara't per a fallades inesperades implementant estratègies robustes de còpia de seguretat i recuperació de desastres.

  • Còpia de Seguretat d'Etcd: Fes còpies de seguretat regulars de la base de dades etcd, que emmagatzema l'estat del clúster.

    ETCDCTL_API=3 etcdctl snapshot save snapshot.db

  • Còpies de Seguretat d'Aplicacions: Utilitza eines com Velero per fer còpies de seguretat i restaurar recursos de Kubernetes i volums persistents.

    velero install --provider aws --bucket <YOUR_BUCKET> --secret-file ./credentials-velero

Conclusió

Seguir aquestes millors pràctiques t'ajudarà a gestionar els teus clústers de Kubernetes de manera més efectiva en un entorn de producció. Enfocant-te en la gestió de recursos, seguretat, monitorització, escalabilitat i recuperació de desastres, pots assegurar-te que les teves aplicacions siguin fiables, segures i amb bon rendiment.
