Introducció

En Kubernetes, assegurar que les teves aplicacions funcionen correctament és crucial. Les comprovacions de salut i les sondes són mecanismes que Kubernetes utilitza per monitoritzar la salut de les teves aplicacions. Aquesta secció cobrirà els conceptes bàsics de les comprovacions de salut i les sondes, els seus tipus, i com implementar-les en els teus clústers de Kubernetes.

Conceptes Clau

  • Comprovacions de Salut: Mecanismes per determinar si una aplicació està funcionant com s'espera.
  • Sondes: Tipus específics de comprovacions de salut que Kubernetes utilitza per monitoritzar l'estat dels contenidors.

Tipus de Sondes

Kubernetes suporta tres tipus de sondes:

  1. Sondes de Vida: Determinen si un contenidor està funcionant. Si la sonda de vida falla, Kubernetes matarà el contenidor i el reiniciarà.
  2. Sondes de Preparació: Determinen si un contenidor està llest per començar a acceptar tràfic. Si la sonda de preparació falla, Kubernetes eliminarà el contenidor dels punts finals del servei.
  3. Sondes d'Arrencada: S'utilitzen per comprovar si un contenidor ha començat amb èxit. Si la sonda d'arrencada falla, Kubernetes matarà el contenidor i el reiniciarà.

Implementació de Sondes

Sondes de Vida

Les sondes de vida s'utilitzen per comprovar si l'aplicació dins d'un contenidor està viva. Si la sonda de vida falla, Kubernetes reiniciarà el contenidor.

Exemple:

apiVersion: v1
kind: Pod
metadata:
  name: liveness-example
spec:
  containers:
  - name: myapp
    image: myapp:latest
    livenessProbe:
      httpGet:
        path: /healthz
        port: 8080
      initialDelaySeconds: 3
      periodSeconds: 3
  • httpGet: Especifica el punt final HTTP per comprovar.
  • initialDelaySeconds: Temps d'espera abans de realitzar la primera sonda.
  • periodSeconds: Temps entre sondes successives.

Sondes de Preparació

Les sondes de preparació s'utilitzen per comprovar si l'aplicació està llesta per servir tràfic. Si la sonda de preparació falla, el contenidor serà eliminat dels punts finals del servei.

Exemple:

apiVersion: v1
kind: Pod
metadata:
  name: readiness-example
spec:
  containers:
  - name: myapp
    image: myapp:latest
    readinessProbe:
      httpGet:
        path: /ready
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 5
  • httpGet: Especifica el punt final HTTP per comprovar.
  • initialDelaySeconds: Temps d'espera abans de realitzar la primera sonda.
  • periodSeconds: Temps entre sondes successives.

Sondes d'Arrencada

Les sondes d'arrencada s'utilitzen per comprovar si l'aplicació ha començat amb èxit. Són útils per a aplicacions que tenen temps d'arrencada llargs.

Exemple:

apiVersion: v1
kind: Pod
metadata:
  name: startup-example
spec:
  containers:
  - name: myapp
    image: myapp:latest
    startupProbe:
      httpGet:
        path: /startup
        port: 8080
      initialDelaySeconds: 10
      periodSeconds: 10
      failureThreshold: 30
  • httpGet: Especifica el punt final HTTP per comprovar.
  • initialDelaySeconds: Temps d'espera abans de realitzar la primera sonda.
  • periodSeconds: Temps entre sondes successives.
  • failureThreshold: Nombre de fallades consecutives abans que la sonda es consideri fallida.

Comparació de Sondes

| Tipus de Sonda | Propòsit | Acció en cas de Fallada | Cas d'Ús Típic | |-----------------|-------------------------------------------|------------------------------------|--------------------------------------| | Sonda de Vida | Comprovar si el contenidor està viu | Reiniciar el contenidor | Detectar i recuperar-se de bloquejos | | Sonda de Preparació | Comprovar si el contenidor està llest per servir | Eliminar dels punts finals del servei | Assegurar que només els contenidors saludables rebin tràfic | | Sonda d'Arrencada | Comprovar si el contenidor ha començat | Reiniciar el contenidor | Gestionar aplicacions amb temps d'arrencada llargs |

Conclusió

Les comprovacions de salut i les sondes són essencials per mantenir la fiabilitat i disponibilitat de les teves aplicacions en Kubernetes. Implementant correctament les sondes de vida, preparació i arrencada, pots assegurar que les teves aplicacions funcionen correctament i poden recuperar-se automàticament de fallades. Entendre i utilitzar aquestes sondes de manera efectiva t'ajudarà a construir desplegaments de Kubernetes robustos i resilients.
