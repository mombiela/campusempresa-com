En aquesta secció, explorarem el concepte de Serveis en Kubernetes, que són essencials per permetre la comunicació entre diferents components d'una aplicació Kubernetes. Cobrirem els conceptes bàsics, diferents tipus de serveis i configuracions avançades.

Què és un Servei?

Un Servei en Kubernetes és una abstracció que defineix un conjunt lògic de Pods i una política per accedir-hi. Els Serveis permeten la comunicació entre diferents parts de la teva aplicació i poden exposar la teva aplicació al món exterior.

Conceptes Clau

  • Pod: La unitat desplegable més petita en Kubernetes, que pot contenir un o més contenidors.
  • Servei: Una abstracció que defineix un conjunt lògic de Pods i una política per accedir-hi.
  • Selector d'Etiquetes: Un mecanisme per seleccionar un grup de Pods basat en les seves etiquetes.
  • ClusterIP: El tipus de Servei per defecte, que exposa el Servei en una IP interna del clúster.
  • NodePort: Exposa el Servei en la IP de cada Node en un port estàtic.
  • LoadBalancer: Exposa el Servei externament utilitzant un equilibrador de càrrega del proveïdor de núvol.
  • ExternalName: Mapa un Servei a un nom DNS.

Tipus de Serveis

Kubernetes suporta diversos tipus de Serveis, cadascun dissenyat per a diferents casos d'ús.

ClusterIP

El tipus de Servei per defecte, que exposa el Servei en una adreça IP interna dins del clúster. Aquest tipus s'utilitza per a la comunicació interna entre Pods.

apiVersion: v1
kind: Service
metadata:
  name: my-clusterip-service
spec:
  selector:
    app: MyApp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376

NodePort

Exposa el Servei en la IP de cada Node en un port estàtic. Aquest tipus permet que el trànsit extern accedeixi al Servei.

apiVersion: v1
kind: Service
metadata:
  name: my-nodeport-service
spec:
  type: NodePort
  selector:
    app: MyApp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376
      nodePort: 30007

LoadBalancer

Exposa el Servei externament utilitzant un equilibrador de càrrega del proveïdor de núvol. Aquest tipus és útil per a serveis que necessiten ser accessibles des de fora del clúster.

apiVersion: v1
kind: Service
metadata:
  name: my-loadbalancer-service
spec:
  type: LoadBalancer
  selector:
    app: MyApp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376

ExternalName

Mapa un Servei a un nom DNS. Aquest tipus és útil per integrar-se amb serveis externs.

apiVersion: v1
kind: Service
metadata:
  name: my-externalname-service
spec:
  type: ExternalName
  externalName: my.database.example.com

Configuracions Avançades

Serveis Sense Capçalera

Un Servei sense capçalera és un Servei sense una ClusterIP. Aquest tipus de Servei és útil per a StatefulSets i altres escenaris on necessites accés directe als Pods.

apiVersion: v1
kind: Service
metadata:
  name: my-headless-service
spec:
  clusterIP: None
  selector:
    app: MyApp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376

Descobriment de Serveis

Kubernetes proporciona mecanismes de descobriment de serveis integrats. Els Pods poden descobrir serveis mitjançant variables d'entorn o DNS.

  • Variables d'Entorn: Kubernetes crea automàticament variables d'entorn per a cada Servei.
  • DNS: Els clústers de Kubernetes inclouen un servidor DNS que assigna automàticament noms DNS als Serveis.

Mesh de Serveis

Una mesh de serveis és una capa d'infraestructura dedicada a gestionar la comunicació servei a servei. Implementacions populars de mesh de serveis inclouen Istio, Linkerd i Consul.

Conclusió

Els Serveis en Kubernetes són un concepte fonamental que permet la comunicació entre diferents parts de la teva aplicació. Entendre els diferents tipus de Serveis i els seus casos d'ús és crucial per dissenyar i desplegar aplicacions escalables i fiables en Kubernetes. Des de la comunicació interna bàsica amb ClusterIP fins a l'exposició de serveis externament amb LoadBalancer, Kubernetes proporciona una varietat d'opcions per satisfer les necessitats de la teva aplicació. Les configuracions avançades com els serveis sense capçalera i les mesh de serveis milloren encara més la flexibilitat i les capacitats dels teus desplegaments en Kubernetes.
