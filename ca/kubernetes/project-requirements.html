Requisits del Projecte
Implementació del Projecte ►

En aquesta secció, cobrirem els requisits essencials per configurar i gestionar un projecte de Kubernetes. Entendre aquests requisits és crucial tant per a principiants com per a usuaris avançats per assegurar una implementació de Kubernetes suau i eficient.

Requisits de Maquinari

Abans de configurar Kubernetes, és important assegurar-se que el teu maquinari compleix les especificacions necessàries.

  • CPU: Almenys 2 nuclis per màquina.
  • Memòria: Mínim 2GB de RAM per màquina.
  • Emmagatzematge: Almenys 10GB d'espai en disc per màquina.
  • Xarxa: Connexió de xarxa fiable entre totes les màquines del clúster.

Requisits de Programari

Kubernetes depèn de diversos components de programari. Assegura't que el següent programari estigui instal·lat i configurat correctament:

  • Sistema Operatiu: Linux (Ubuntu, CentOS, etc.)
  • Runtime de Contenidors: Docker, containerd, o CRI-O.
  • Kubeadm, Kubelet, i Kubectl: Aquests són eines essencials de Kubernetes.
  • Plugin de Xarxa: Flannel, Calico, o un altre plugin compatible amb CNI.

Exemple: Instal·lant Docker a Ubuntu

sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl software-properties-common
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
sudo apt-get update
sudo apt-get install -y docker-ce

Requisits del Clúster

Quan configuris un clúster de Kubernetes, considera el següent:

  • Nodes Master: Almenys un node master per gestionar el clúster.
  • Nodes Worker: Un o més nodes worker per executar les teves aplicacions.
  • Alta Disponibilitat: Per a entorns de producció, considera múltiples nodes master per a alta disponibilitat.

Exemple: Inicialitzant el Node Master

sudo kubeadm init --pod-network-cidr=10.244.0.0/16

Després d'executar la comanda anterior, segueix les instruccions proporcionades per configurar kubectl per a l'usuari root o un usuari regular.

Requisits de Xarxa

La xarxa és un aspecte crític de Kubernetes. Assegura el següent:

  • Xarxa de Pods: Un bloc CIDR per a les adreces IP dels pods.
  • Xarxa de Serveis: Un bloc CIDR per a les adreces IP dels serveis.
  • DNS: Kubernetes requereix un servidor DNS per al descobriment de serveis.

Exemple: Aplicant un Plugin de Xarxa

kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml

Requisits de Seguretat

La seguretat és primordial en qualsevol implementació de Kubernetes. Considera el següent:

  • RBAC (Control d'Accés Basat en Rols): Implementa RBAC per controlar l'accés al clúster.
  • Certificats TLS: Utilitza certificats TLS per assegurar la comunicació entre components.
  • Polítiques de Xarxa: Defineix polítiques de xarxa per controlar el trànsit entre pods.

Exemple: Habilitant RBAC

RBAC està habilitat per defecte a Kubernetes 1.6 i versions posteriors. Per crear un rol i vincular-lo a un usuari:

apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "watch", "list"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: read-pods
  namespace: default
subjects:
- kind: User
  name: jane
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io

Conclusió

Entendre i complir els requisits del projecte és el primer pas cap a una implementació exitosa de Kubernetes. Assegura't que els teus requisits de maquinari, programari, clúster, xarxa i seguretat estiguin planificats i implementats a fons. Aquest coneixement fonamental obrirà el camí per a temes i pràctiques més avançades de Kubernetes.
