Alta Disponibilitat a Kubernetes
Recuperació de Desastres ►

L'Alta Disponibilitat (HA) és un aspecte crucial de qualsevol clúster de Kubernetes de grau de producció. Assegura que les teves aplicacions i serveis romanguin disponibles fins i tot en cas de fallades. Aquest tema cobrirà els fonaments de l'HA a Kubernetes, incloent conceptes clau, arquitectura i exemples pràctics.

Conceptes Clau d'Alta Disponibilitat

  • Redundància: Tenir múltiples instàncies de components crítics per evitar punts únics de fallada.
  • Failover: El procés de canviar a un sistema redundant o de reserva en cas de fallada del sistema principal.
  • Balanceig de Càrrega: Distribuir el tràfic de xarxa o aplicació entre múltiples servidors per assegurar que cap servidor es converteixi en un coll d'ampolla.
  • Replicació: Mantenir múltiples còpies de dades per assegurar disponibilitat i fiabilitat.

Arquitectura d'Alta Disponibilitat a Kubernetes

Alta Disponibilitat del Node Mestre

El node mestre de Kubernetes és responsable de gestionar el clúster. Assegurar la seva alta disponibilitat implica:

  • Múltiples Nodes Mestres: Desplegar múltiples nodes mestres per evitar un punt únic de fallada.
  • Clúster Etcd: Utilitzar un clúster etcd per emmagatzemar l'estat del clúster de manera fiable.
  • Balancejador de Càrrega: Col·locar un balancejador de càrrega davant dels nodes mestres per distribuir les sol·licituds.

Exemple: Configuració de Múltiples Nodes Mestres

apiVersion: v1
kind: Pod
metadata:
  name: etcd
  namespace: kube-system
spec:
  containers:
  - name: etcd
    image: quay.io/coreos/etcd:v3.4.3
    command:
    - etcd
    - --advertise-client-urls=https://<MASTER_IP>:2379
    - --initial-advertise-peer-urls=https://<MASTER_IP>:2380
    - --initial-cluster=<MASTER_NAME>=https://<MASTER_IP>:2380
    - --initial-cluster-state=new
    - --initial-cluster-token=etcd-cluster-1
    - --name=<MASTER_NAME>
    - --peer-client-cert-auth=true
    - --peer-key-file=/etc/kubernetes/pki/etcd/peer.key
    - --peer-cert-file=/etc/kubernetes/pki/etcd/peer.crt
    - --peer-trusted-ca-file=/etc/kubernetes/pki/etcd/ca.crt
    - --client-cert-auth=true
    - --key-file=/etc/kubernetes/pki/etcd/server.key
    - --cert-file=/etc/kubernetes/pki/etcd/server.crt
    - --trusted-ca-file=/etc/kubernetes/pki/etcd/ca.crt
    - --data-dir=/var/lib/etcd

Alta Disponibilitat del Node de Treball

Els nodes de treball executen les càrregues de treball reals de l'aplicació. Assegurar la seva alta disponibilitat implica:

  • Pools de Nodes: Agrupar nodes en pools per gestionar-los i escalar-los eficientment.
  • Replicació de Pods: Utilitzar ReplicaSets o Deployments per assegurar que múltiples instàncies d'un pod estiguin en execució.
  • Anti-Afinitat de Pods: Assegurar que els pods estiguin distribuïts entre diferents nodes per evitar un punt únic de fallada.

Exemple: Utilització de ReplicaSets per a la Replicació de Pods

apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: nginx-replicaset
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

Balanceig de Càrrega

El balanceig de càrrega és essencial per distribuir el tràfic entre múltiples nodes i pods. Kubernetes proporciona diverses maneres d'aconseguir el balanceig de càrrega:

  • Servei: Un Servei de Kubernetes pot exposar una aplicació que s'executa en un conjunt de Pods com un servei de xarxa.
  • Ingress: Un objecte API que gestiona l'accés extern als serveis, típicament HTTP.

Exemple: Creació d'un Servei amb Balanceig de Càrrega

apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: MyApp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376
  type: LoadBalancer

Conclusió

L'Alta Disponibilitat a Kubernetes s'aconsegueix mitjançant la redundància, mecanismes de failover, balanceig de càrrega i replicació. Configurant múltiples nodes mestres, utilitzant ReplicaSets i aprofitant els Serveis i Ingress de Kubernetes, pots assegurar que les teves aplicacions romanguin disponibles fins i tot en cas de fallades. Entendre i implementar aquests conceptes és crucial per mantenir un entorn de Kubernetes robust i fiable.
