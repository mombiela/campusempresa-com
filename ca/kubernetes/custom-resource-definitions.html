Introducció a les Definicions de Recursos Personalitzats (CRDs)

Les Definicions de Recursos Personalitzats (CRDs) són una característica poderosa a Kubernetes que et permeten estendre l'API de Kubernetes amb els teus propis recursos personalitzats. Això et permet crear i gestionar recursos que no formen part de l'API predeterminada de Kubernetes.

Conceptes Clau

  • Recursos Personalitzats (CRs): Són extensions de l'API de Kubernetes que et permeten emmagatzemar i recuperar dades estructurades.
  • Definicions de Recursos Personalitzats (CRDs): Defineixen l'esquema per als teus recursos personalitzats, permetent a Kubernetes entendre'ls i gestionar-los.
  • Controlador: Un bucle de control que observa l'estat dels teus recursos personalitzats i fa canvis per moure l'estat actual cap a l'estat desitjat.

Creació d'una Definició de Recurs Personalitzat

Per crear una CRD, has de definir-la en un fitxer YAML i aplicar-la al teu clúster de Kubernetes.

Exemple de YAML de CRD

apiVersion: apiextensions.k8s.io/v1
kind: CustomResourceDefinition
metadata:
  name: crontabs.example.com
spec:
  group: example.com
  versions:
    - name: v1
      served: true
      storage: true
      schema:
        openAPIV3Schema:
          type: object
          properties:
            spec:
              type: object
              properties:
                cronSpec:
                  type: string
                image:
                  type: string
                replicas:
                  type: integer
  scope: Namespaced
  names:
    plural: crontabs
    singular: crontab
    kind: CronTab
    shortNames:
    - ct

Explicació

  • apiVersion: Especifica la versió de l'API.
  • kind: Especifica el tipus de recurs, que és CustomResourceDefinition.
  • metadata: Conté el nom de la CRD.
  • spec: Defineix el grup, versions, abast i noms per al recurs personalitzat.
    • group: Un agrupament lògic per als teus recursos personalitzats.
    • versions: Llista les versions del recurs personalitzat.
    • scope: Pot ser Namespaced o Cluster.
    • names: Defineix els noms per al recurs personalitzat.

Creació de Recursos Personalitzats

Un cop aplicada la CRD, pots crear instàncies del teu recurs personalitzat.

Exemple de YAML de Recurs Personalitzat

apiVersion: example.com/v1
kind: CronTab
metadata:
  name: my-crontab
spec:
  cronSpec: "* * * * */5"
  image: my-cron-image
  replicas: 3

Explicació

  • apiVersion: Especifica la versió de l'API i el grup.
  • kind: Especifica el tipus de recurs personalitzat.
  • metadata: Conté el nom del recurs personalitzat.
  • spec: Defineix l'estat desitjat del recurs personalitzat.

Gestió de Recursos Personalitzats

Pots gestionar els teus recursos personalitzats utilitzant les comandes estàndard de kubectl.

Comandes d'Exemple

  • Crear un Recurs Personalitzat: 
    kubectl apply -f my-crontab.yaml
  • Obtenir Recursos Personalitzats: 
    kubectl get crontabs
  • Descriure un Recurs Personalitzat: 
    kubectl describe crontab my-crontab
  • Eliminar un Recurs Personalitzat: 
    kubectl delete crontab my-crontab

Temes Avançats

Validació i Assignació de Valors Predeterminats

Pots afegir validació i assignació de valors predeterminats a les teves CRDs per assegurar la integritat de les dades i proporcionar valors predeterminats.

Exemple d'Esquema de Validació

schema:
  openAPIV3Schema:
    type: object
    properties:
      spec:
        type: object
        properties:
          cronSpec:
            type: string
            pattern: '^[0-9*/,-]+ [0-9*/,-]+ [0-9*/,-]+ [0-9*/,-]+ [0-9*/,-]+$'
          image:
            type: string
          replicas:
            type: integer
            minimum: 1
            maximum: 10

Explicació

  • pattern: Assegura que el cronSpec segueixi un format cron vàlid.
  • minimum i maximum: Asseguren que el valor de replicas estigui entre 1 i 10.

Controladors Personalitzats

Per gestionar el cicle de vida dels teus recursos personalitzats, pots escriure controladors personalitzats utilitzant marcs com Kubebuilder o Operator SDK.

Conclusió

Les Definicions de Recursos Personalitzats (CRDs) proporcionen una manera flexible d'estendre l'API de Kubernetes amb els teus propis recursos personalitzats. Definint CRDs, pots crear, gestionar i validar recursos personalitzats, permetent-te adaptar Kubernetes a les teves necessitats específiques. Característiques avançades com la validació, l'assignació de valors predeterminats i els controladors personalitzats milloren encara més les capacitats de les CRDs, convertint-les en una eina poderosa per als desenvolupadors de Kubernetes.
