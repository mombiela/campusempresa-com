Introducció a les Xarxes de Clúster

Les xarxes de clúster a Kubernetes són un concepte fonamental que garanteix la comunicació entre diversos components dins d'un clúster de Kubernetes. Aquesta secció cobrirà els conceptes bàsics de les xarxes de clúster, incloent com els pods es comuniquen entre ells, serveis i xarxes externes.

Conceptes Clau

  • Xarxa de Pod: Cada pod en un clúster de Kubernetes obté la seva pròpia adreça IP, la qual cosa permet la comunicació directa entre pods.
  • Xarxa de Servei: Els serveis proporcionen una adreça IP estable i un nom DNS a un conjunt de pods, permetent una comunicació fiable.
  • ClusterIP: El tipus de servei per defecte que exposa el servei en una IP interna del clúster.
  • NodePort: Exposa el servei en la IP de cada Node en un port estàtic.
  • LoadBalancer: Exposa el servei externament utilitzant un equilibrador de càrrega del proveïdor de núvol.
  • Polítiques de Xarxa: Defineixen com es permet que els pods es comuniquin entre ells i altres punts finals de la xarxa.

Comunicació Pod-a-Pod

A Kubernetes, cada pod obté la seva pròpia adreça IP, i els pods poden comunicar-se entre ells utilitzant aquestes adreces IP.

Exemple

apiVersion: v1
kind: Pod
metadata:
  name: pod-a
spec:
  containers:
  - name: container-a
    image: nginx
---
apiVersion: v1
kind: Pod
metadata:
  name: pod-b
spec:
  containers:
  - name: container-b
    image: busybox
    command: ["sh", "-c", "sleep 3600"]
  • Explicació: En aquest exemple, pod-a i pod-b són dos pods que s'executen en el mateix clúster. pod-b pot comunicar-se amb pod-a utilitzant l'adreça IP de pod-a.

Serveis

Els serveis a Kubernetes proporcionen un punt final estable per accedir a un conjunt de pods.

Tipus de Serveis

  • ClusterIP: Exposa el servei en una IP interna del clúster.
  • NodePort: Exposa el servei en la IP de cada Node en un port estàtic.
  • LoadBalancer: Exposa el servei externament utilitzant un equilibrador de càrrega del proveïdor de núvol.

Exemple

apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: MyApp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376
  type: ClusterIP
  • Explicació: Aquest servei selecciona pods amb l'etiqueta app: MyApp i redirigeix el trànsit del port 80 al port 9376 en els pods seleccionats.

Polítiques de Xarxa

Les polítiques de xarxa et permeten controlar la comunicació entre pods.

Exemple

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-nginx
spec:
  podSelector:
    matchLabels:
      app: nginx
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: frontend
  • Explicació: Aquesta política de xarxa permet el trànsit entrant als pods amb l'etiqueta app: nginx només des dels pods amb l'etiqueta app: frontend.

Comparació de Tipus de Serveis

| Tipus de Servei | Accés Intern | Accés Extern | Cas d'Ús | |-----------------|--------------|--------------|----------| | ClusterIP | Sí | No | Comunicació interna entre pods | | NodePort | Sí | Sí | Exposar serveis en un port estàtic en cada node | | LoadBalancer | Sí | Sí | Exposar serveis externament utilitzant un equilibrador de càrrega del proveïdor de núvol |

Conclusió

Les xarxes de clúster a Kubernetes són essencials per garantir una comunicació fluida entre diversos components dins d'un clúster. Entendre la comunicació pod-a-pod, serveis i polítiques de xarxa és crucial per gestionar i assegurar les teves aplicacions a Kubernetes. En dominar aquests conceptes, pots dissenyar i desplegar aplicacions escalables i segures en un entorn Kubernetes.
