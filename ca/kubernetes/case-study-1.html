Introducció

Migrar una aplicació a Kubernetes pot ser un procés complex però gratificant. Aquest estudi de cas et guiarà pels passos de migrar una aplicació web senzilla a Kubernetes, destacant conceptes clau, millors pràctiques i errors comuns.

Conceptes clau

  • Containers: Paquets lleugers, autònoms i executables que inclouen tot el necessari per executar un programari.
  • Kubernetes: Una plataforma de codi obert dissenyada per automatitzar el desplegament, l'escalat i l'operació de contenidors d'aplicacions.
  • Pods: Les unitats desplegables més petites a Kubernetes, que poden contenir un o més contenidors.
  • Services: Abstraccions que defineixen un conjunt lògic de Pods i una política per accedir-hi.
  • Deployments: Proporcionen actualitzacions declaratives a les aplicacions, assegurant que l'estat desitjat es mantingui.

Prerequisits

Abans de començar la migració, assegura't de tenir:

  • Un coneixement bàsic de Docker i la containerització.
  • Un clúster de Kubernetes en funcionament (per exemple, Minikube per al desenvolupament local o un proveïdor de núvol com GKE, EKS o AKS).
  • L'eina de línia de comandes kubectl instal·lada i configurada per interactuar amb el teu clúster.

Migració pas a pas

Pas 1: Containeritzar l'aplicació

Primer, necessitem containeritzar la nostra aplicació web utilitzant Docker.

Exemple de Dockerfile

# Utilitza una imatge oficial de Node.js com a imatge base
FROM node:14

# Estableix el directori de treball
WORKDIR /usr/src/app

# Copia package.json i instal·la les dependències
COPY package*.json ./
RUN npm install

# Copia la resta del codi de l'aplicació
COPY . .

# Exposa el port en què s'executa l'aplicació
EXPOSE 8080

# Defineix la comanda per executar l'aplicació
CMD ["node", "app.js"]
  • FROM: Especifica la imatge base.
  • WORKDIR: Estableix el directori de treball dins del contenidor.
  • COPY: Copia fitxers des de l'amfitrió al contenidor.
  • RUN: Executa comandes dins del contenidor.
  • EXPOSE: Informa Docker que el contenidor escolta en els ports de xarxa especificats.
  • CMD: Proporciona la comanda per defecte per executar l'aplicació.

Pas 2: Construir i pujar la imatge Docker

Construeix la imatge Docker i puja-la a un registre de contenidors (per exemple, Docker Hub).

# Construeix la imatge Docker
docker build -t yourusername/yourapp:latest .

# Puja la imatge a Docker Hub
docker push yourusername/yourapp:latest

Pas 3: Crear manifests de Kubernetes

Crea manifests de Kubernetes per al Deployment i el Service.

Manifest de Deployment (deployment.yaml)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: webapp-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: webapp
  template:
    metadata:
      labels:
        app: webapp
    spec:
      containers:
      - name: webapp
        image: yourusername/yourapp:latest
        ports:
        - containerPort: 8080
  • replicas: Nombre de rèpliques de pod.
  • selector: Identifica els pods gestionats pel deployment.
  • template: Defineix la plantilla de pod.

Manifest de Service (service.yaml)

apiVersion: v1
kind: Service
metadata:
  name: webapp-service
spec:
  selector:
    app: webapp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 8080
  type: LoadBalancer
  • selector: Coincideix amb els pods a exposar.
  • ports: Defineix els mapatges de ports.
  • type: Especifica el tipus de servei (per exemple, LoadBalancer per a accés extern).

Pas 4: Desplegar a Kubernetes

Aplica els manifests per desplegar l'aplicació al clúster de Kubernetes.

# Aplica el deployment
kubectl apply -f deployment.yaml

# Aplica el servei
kubectl apply -f service.yaml

Pas 5: Verificar el desplegament

Comprova l'estat del deployment i el servei.

# Obten l'estat del deployment
kubectl get deployments

# Obten l'estat dels pods
kubectl get pods

# Obten l'estat del servei
kubectl get services

Conclusió

Migrar a Kubernetes implica diversos passos, incloent containeritzar l'aplicació, crear manifests de Kubernetes i desplegar al clúster. Seguint aquest enfocament estructurat, pots assegurar un procés de migració suau. Kubernetes proporciona eines poderoses per gestionar aplicacions containeritzades, facilitant l'escalat i el manteniment dels teus serveis.

En aquest estudi de cas, hem cobert:

  • Containeritzar una aplicació web amb Docker.
  • Crear manifests de Kubernetes per al deployment i el servei.
  • Desplegar i verificar l'aplicació en un clúster de Kubernetes.

Aquest coneixement fonamental t'ajudarà a abordar escenaris més complexos de Kubernetes en el futur.
