Els espais de noms són un concepte fonamental a Kubernetes, proporcionant un mecanisme per aïllar i gestionar recursos dins d'un clúster. Aquest tema cobrirà els espais de noms des d'un nivell principiant fins a un nivell avançat, assegurant que entenguis el seu propòsit, com crear-los i gestionar-los, i casos d'ús avançats.

Introducció als Espais de Noms

Els espais de noms a Kubernetes s'utilitzen per dividir els recursos del clúster entre múltiples usuaris. Proporcionen un abast per als noms, permetent-te crear recursos amb el mateix nom en diferents espais de noms.

Conceptes Clau

  • Aïllament: Els espais de noms permeten l'aïllament de recursos dins d'un clúster.
  • Gestió de Recursos: Ajuden a gestionar recursos com les quotes de CPU i memòria.
  • Control d'Accés: Els espais de noms es poden utilitzar per aplicar diferents controls d'accés a diferents parts del clúster.

Escenari Exemple

Imagina que tens un clúster de Kubernetes utilitzat per múltiples equips: Desenvolupament, Proves i Producció. Utilitzant espais de noms, pots crear entorns separats per a cada equip, assegurant que els seus recursos i configuracions no interfereixin entre ells.

Creació i Gestió d'Espais de Noms

Crear i gestionar espais de noms a Kubernetes és senzill. Pots utilitzar l'eina de línia de comandes kubectl per realitzar aquestes accions.

Creació d'un Espai de Noms

Per crear un espai de noms, pots utilitzar la següent comanda:

kubectl create namespace <namespace-name>

Per exemple, per crear un espai de noms per a l'equip de desenvolupament:

kubectl create namespace development

Llistar Espais de Noms

Per llistar tots els espais de noms en el teu clúster, utilitza:

kubectl get namespaces

Eliminar un Espai de Noms

Per eliminar un espai de noms, utilitza la següent comanda:

kubectl delete namespace <namespace-name>

Per exemple, per eliminar l'espai de noms de desenvolupament:

kubectl delete namespace development

Treballant amb Espais de Noms

Un cop creats els espais de noms, pots desplegar recursos en ells i gestionar-los en conseqüència.

Desplegar Recursos en un Espai de Noms

Per desplegar un recurs en un espai de noms específic, pots especificar l'espai de noms en el fitxer de definició del recurs o utilitzar el flag --namespace amb kubectl.

Exemple de Definició de Recurs

apiVersion: v1
kind: Pod
metadata:
  name: my-pod
  namespace: development
spec:
  containers:
  - name: my-container
    image: nginx

Utilitzant kubectl amb Espai de Noms

kubectl apply -f my-pod.yaml --namespace=development

Canviar Entre Espais de Noms

Per canviar entre espais de noms en el teu context de kubectl, pots utilitzar:

kubectl config set-context --current --namespace=<namespace-name>

Per exemple, per canviar a l'espai de noms de desenvolupament:

kubectl config set-context --current --namespace=development

Gestió Avançada d'Espais de Noms

La gestió avançada d'espais de noms implica establir quotes de recursos, límits i aplicar polítiques de xarxa.

Quotes de Recursos

Les quotes de recursos asseguren que un espai de noms no utilitzi més recursos dels que té assignats.

Exemple de Quota de Recursos

apiVersion: v1
kind: ResourceQuota
metadata:
  name: cpu-mem-quota
  namespace: development
spec:
  hard:
    requests.cpu: "1"
    requests.memory: 1Gi
    limits.cpu: "2"
    limits.memory: 2Gi

Aplica la quota de recursos amb:

kubectl apply -f resource-quota.yaml

Polítiques de Xarxa

Les polítiques de xarxa controlen la comunicació entre pods dins d'un espai de noms.

Exemple de Política de Xarxa

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-same-namespace
  namespace: development
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector: {}
  egress:
  - to:
    - podSelector: {}

Aplica la política de xarxa amb:

kubectl apply -f network-policy.yaml

Conclusió

Els espais de noms són una característica poderosa a Kubernetes que ajuden a organitzar i gestionar recursos dins d'un clúster. Proporcionen aïllament, gestió de recursos i control d'accés, facilitant la gestió d'entorns complexos. Entenent com crear, gestionar i utilitzar espais de noms de manera efectiva, pots assegurar-te que el teu clúster de Kubernetes estigui ben organitzat i segur.

En resum:

  • Els espais de noms proporcionen aïllament i gestió de recursos.
  • Pots crear, llistar i eliminar espais de noms utilitzant kubectl.
  • Desplega recursos en espais de noms específics i canvia de context segons sigui necessari.
  • Utilitza funcions avançades com quotes de recursos i polítiques de xarxa per gestionar espais de noms de manera efectiva.
