Kubeadm és una eina que t'ajuda a arrencar un clúster mínim viable de Kubernetes que compleix amb les millors pràctiques. Realitza les accions necessàries per posar en marxa un clúster mínim viable i segur d'una manera fàcil d'usar.

Introducció a Kubeadm

  • Què és Kubeadm?

    • Kubeadm és una eina proporcionada per Kubernetes per simplificar el procés de configuració d'un clúster de Kubernetes.
    • Gestiona la instal·lació i configuració dels components del pla de control i dels nodes de treball.

  • Per què utilitzar Kubeadm?

    • Simplifica el procés de configuració del clúster.
    • Assegura les millors pràctiques i configuracions de seguretat.
    • Proporciona una manera consistent i repetible de crear clústers.

Instal·lació de Kubeadm

Prerequisits

  • Sistema Operatiu: Linux (Ubuntu, CentOS, etc.)
  • Requisits de Maquinari:
    • 2 GB o més de RAM per màquina (menys deixarà poc espai per a les teves aplicacions).
    • 2 CPUs o més.
    • Connectivitat de xarxa completa entre totes les màquines del clúster (xarxa pública o privada està bé).
    • Nom d'amfitrió únic, adreça MAC i product_uuid per a cada node.
  • Requisits de Programari:
    • Docker (o un altre entorn de contenidors).
    • Kubeadm, kubelet i kubectl instal·lats en tots els nodes.

Passos d'Instal·lació

  1. Instal·lar Docker:

    sudo apt-get update
sudo apt-get install -y docker.io
sudo systemctl enable docker
sudo systemctl start docker

  2. Instal·lar Kubeadm, Kubelet i Kubectl:

    sudo apt-get update && sudo apt-get install -y apt-transport-https curl
curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
cat <<EOF | sudo tee /etc/apt/sources.list.d/kubernetes.list
deb https://apt.kubernetes.io/ kubernetes-xenial main
EOF
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

Arrencar un Clúster amb Kubeadm

Inicialitzar el Pla de Control

  • Comanda:

    sudo kubeadm init --pod-network-cidr=10.244.0.0/16

  • Explicació:

    • --pod-network-cidr=10.244.0.0/16: Aquesta bandera especifica el CIDR per a la xarxa de pods. Aquest exemple utilitza el CIDR per defecte de Flannel.

  • Passos Post-Inicialització:

    mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

Configurar una Xarxa de Pods

  • Instal·lar Flannel: 
    kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml

Unir Nodes de Treball

  • Comanda:

    • Després d'inicialitzar el pla de control, kubeadm mostrarà una comanda kubeadm join. Executa aquesta comanda en cada node de treball per unir-los al clúster.

  • Exemple:

    sudo kubeadm join <control-plane-host>:<control-plane-port> --token <token> --discovery-token-ca-cert-hash sha256:<hash>

Gestionar el Clúster

Comandes Comunes

  • Comprovar l'Estat dels Nodes:

    kubectl get nodes

  • Comprovar l'Estat dels Pods:

    kubectl get pods --all-namespaces

  • Desplegar una Aplicació:

    kubectl create deployment nginx --image=nginx

  • Exposar el Desplegament:

    kubectl expose deployment nginx --port=80 --type=NodePort

Resolució de Problemes

  • Problemes Comuns:

    • Pods no arrencant: Comprova els registres del pod utilitzant kubectl logs <pod-name>.
    • Node no preparat: Assegura't que el servei kubelet està en funcionament i comprova l'estat del node utilitzant kubectl describe node <node-name>.

  • Comandes Útils:

    • kubectl describe pod <pod-name>: Proporciona informació detallada sobre un pod.
    • kubectl logs <pod-name>: Recupera els registres d'un pod.

Conclusió

Kubeadm és una eina poderosa que simplifica el procés de configuració d'un clúster de Kubernetes. Seguint les millors pràctiques i proporcionant un procés de configuració consistent, assegura que el teu clúster sigui segur i fiable. Tant si estàs configurant un petit clúster de desenvolupament com un gran entorn de producció, Kubeadm proporciona les eines necessàries per començar ràpidament i eficientment.
