Introducció a l'Arquitectura de Kubernetes

Kubernetes és una plataforma de codi obert dissenyada per automatitzar el desplegament, l'escalat i l'operació de contenidors d'aplicacions. Entendre la seva arquitectura és crucial per gestionar i desplegar aplicacions de manera efectiva.

Components Clau de l'Arquitectura de Kubernetes

Node Mestre

El node mestre és responsable de gestionar el clúster de Kubernetes. Coordina totes les activitats dins del clúster, com ara la programació, l'escalat i l'actualització d'aplicacions.

  • API Server: Actua com el front-end per al pla de control de Kubernetes. Exposa l'API de Kubernetes.
  • etcd: Un magatzem de claus-valor consistent i altament disponible utilitzat com a magatzem de suport de Kubernetes per a totes les dades del clúster.
  • Controller Manager: Executa processos de control que gestionen tasques rutinàries al clúster.
  • Scheduler: Assigna càrregues de treball a nodes específics basant-se en la disponibilitat de recursos i altres restriccions.

Nodes Treballadors

Els nodes treballadors són les màquines on s'executen realment les càrregues de treball de les aplicacions. Cada node treballador conté els serveis necessaris per executar pods i és gestionat pel node mestre.

  • Kubelet: Un agent que s'executa en cada node treballador i assegura que els contenidors s'estan executant en un pod.
  • Kube-proxy: Manté les regles de xarxa als nodes, permetent la comunicació de xarxa amb els teus pods.
  • Container Runtime: Programari responsable d'executar contenidors (per exemple, Docker, containerd).

Pods

Els pods són les unitats desplegables més petites a Kubernetes. Un pod pot contenir un o més contenidors que comparteixen emmagatzematge, xarxa i una especificació de com executar els contenidors.

Serveis

Els serveis a Kubernetes defineixen un conjunt lògic de pods i una política per accedir-hi. Permeten la comunicació entre diferents parts d'una aplicació.

Volums Persistents

Els Volums Persistents (PVs) proporcionen recursos d'emmagatzematge en un clúster de Kubernetes. Són independents del cicle de vida dels pods.

Pla de Control de Kubernetes

El pla de control gestiona el clúster de Kubernetes i consisteix en els components del node mestre.

API Server

L'API Server és el punt d'entrada per a totes les comandes REST utilitzades per controlar el clúster. Processa les sol·licituds REST, les valida i executa la lògica de negoci associada.

# Exemple d'una sol·licitud d'API de Kubernetes per crear un pod
apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx

etcd

etcd és un magatzem de claus-valor distribuït utilitzat per emmagatzemar totes les dades utilitzades per gestionar el clúster. És la font única de veritat per a l'estat del clúster.

Controller Manager

El Controller Manager executa diversos controladors que regulen l'estat del clúster. Per exemple, el Node Controller assegura que els nodes estiguin en funcionament.

Scheduler

El Scheduler assigna pods a nodes basant-se en els requisits de recursos i altres restriccions. Assegura que les càrregues de treball estiguin equilibrades a través del clúster.

Components del Node

Els components del node s'executen en cada node, mantenint els pods en execució i proporcionant l'entorn d'execució de Kubernetes.

Kubelet

El Kubelet assegura que els contenidors s'estiguin executant en un pod. Comunica amb l'API Server per obtenir l'estat desitjat dels pods i assegura que els contenidors s'executin com s'espera.

Kube-proxy

Kube-proxy manté les regles de xarxa als nodes. Permet la comunicació de xarxa amb els teus pods des de sessions de xarxa dins o fora del teu clúster.

Container Runtime

El runtime de contenidors és responsable d'executar contenidors. Kubernetes suporta diversos runtimes de contenidors, incloent Docker i containerd.

Xarxes a Kubernetes

Les xarxes de Kubernetes permeten la comunicació entre contenidors, pods i serveis. Abstrauen els detalls de la xarxa subjacent i proporcionen una interfície consistent per a la comunicació.

  • Xarxa del Clúster: Proporciona comunicació entre tots els pods a través dels nodes.
  • Xarxa de Serveis: Proporciona adreces IP estables i noms DNS per als serveis.
  • Ingress: Gestiona l'accés extern als serveis, típicament HTTP.

Emmagatzematge a Kubernetes

Kubernetes proporciona diverses opcions d'emmagatzematge per persistir dades més enllà del cicle de vida dels pods.

  • Volums: Adjuntats als pods i proporcionen emmagatzematge per als contenidors.
  • Volums Persistents (PVs): Recursos d'emmagatzematge a nivell de clúster que són independents dels pods.
  • Reclamacions de Volums Persistents (PVCs): Sol·licituds d'emmagatzematge per part d'un usuari.

Conclusió

Entendre l'arquitectura de Kubernetes és fonamental per gestionar i desplegar aplicacions de manera efectiva. L'arquitectura consisteix en nodes mestres i treballadors, amb components clau com l'API Server, etcd i Kubelet assegurant que el clúster funcioni sense problemes. Les xarxes i l'emmagatzematge també són aspectes crucials, proporcionant comunicació i persistència de dades per a les aplicacions.

En dominar aquests conceptes, estaràs ben equipat per aprofitar Kubernetes per a un desplegament d'aplicacions escalable i resilient.
