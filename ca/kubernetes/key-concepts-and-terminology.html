Introducció

Kubernetes, sovint abreujat com a K8s, és una plataforma de codi obert dissenyada per automatitzar el desplegament, l'escalat i l'operació de contenidors d'aplicacions. Entendre els conceptes i la terminologia clau és crucial per dominar Kubernetes des de nivells principiants fins a avançats.

Conceptes Bàsics

Clúster

  • Definició: Un conjunt de nodes (màquines) que executen aplicacions contenidoritzades gestionades per Kubernetes.
  • Components:
    • Node Mestre: Gestiona el clúster i coordina totes les activitats.
    • Nodes Treballadors: Executen les aplicacions contenidoritzades.

Node

  • Definició: Una màquina individual en el clúster de Kubernetes, que pot ser una màquina física o virtual.
  • Tipus:
    • Node Mestre: Gestiona el pla de control.
    • Node Treballador: Executa les càrregues de treball de l'aplicació.

Pod

  • Definició: L'objecte més petit i senzill de Kubernetes. Un pod representa una única instància d'un procés en execució en el teu clúster.
  • Característiques:
    • Pot contenir un o més contenidors.
    • Comparteix recursos d'emmagatzematge i xarxa.
apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx

Servei

  • Definició: Una abstracció que defineix un conjunt lògic de Pods i una política per accedir-hi.
  • Tipus:
    • ClusterIP: Exposa el servei en una IP interna del clúster.
    • NodePort: Exposa el servei en la IP de cada Node en un port estàtic.
    • LoadBalancer: Exposa el servei externament utilitzant un equilibrador de càrrega del proveïdor de núvol.
apiVersion: v1
kind: Service
metadata:
  name: example-service
spec:
  selector:
    app: example
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376
  type: ClusterIP

Desplegament

  • Definició: Proporciona actualitzacions declaratives a les aplicacions, assegurant que el nombre desitjat de Pods estiguin en execució.
  • Característiques:
    • Actualitzacions contínues.
    • Reversions.
apiVersion: apps/v1
kind: Deployment
metadata:
  name: example-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

Espai de Noms

  • Definició: Una manera de dividir els recursos del clúster entre múltiples usuaris.
  • Usos:
    • Separació d'entorns (per exemple, desenvolupament, preproducció, producció).
    • Gestió de quotes de recursos.
apiVersion: v1
kind: Namespace
metadata:
  name: example-namespace

Conceptes Avançats

StatefulSet

  • Definició: Gestiona el desplegament i l'escalat d'un conjunt de Pods, i proporciona garanties sobre l'ordre i la unicitat d'aquests Pods.
  • Usos:
    • Aplicacions amb estat com bases de dades.
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: example-statefulset
spec:
  serviceName: "example"
  replicas: 3
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

DaemonSet

  • Definició: Assegura que tots (o alguns) Nodes executin una còpia d'un Pod.
  • Usos:
    • Monitorització de Nodes.
    • Recollida de registres.
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: example-daemonset
spec:
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

ConfigMap i Secret

  • ConfigMap:
    • Definició: Utilitzat per emmagatzemar dades no confidencials en parelles clau-valor.
    • Usos: Configuració de paràmetres.
apiVersion: v1
kind: ConfigMap
metadata:
  name: example-configmap
data:
  example.property: "exampleValue"
  • Secret:
    • Definició: Utilitzat per emmagatzemar informació sensible, com contrasenyes, tokens OAuth i claus ssh.
    • Usos: Emmagatzematge segur de dades sensibles.
apiVersion: v1
kind: Secret
metadata:
  name: example-secret
type: Opaque
data:
  password: cGFzc3dvcmQ=

Conclusió

Entendre aquests conceptes i terminologia clau és essencial per utilitzar Kubernetes de manera efectiva. Des dels blocs de construcció bàsics com Pods i Serveis fins a estructures més avançades com StatefulSets i DaemonSets, cada component juga un paper crític en l'ecosistema de Kubernetes. La mestria d'aquests conceptes proporcionarà una base sòlida per a una exploració més profunda i un ús avançat de Kubernetes.
