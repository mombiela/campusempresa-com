Les Polítiques de Seguretat de Pods (PSPs) són un aspecte crític de la seguretat de Kubernetes, proporcionant un mecanisme per controlar els aspectes relacionats amb la seguretat de la creació i actualització de pods. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de les PSPs, assegurant una comprensió completa.

Introducció a les Polítiques de Seguretat de Pods

Les Polítiques de Seguretat de Pods són recursos a nivell de clúster que controlen els aspectes sensibles a la seguretat de l'especificació dels pods. Permeten als administradors definir un conjunt de condicions que un pod ha de complir per ser acceptat al sistema.

Conceptes Clau

  • Política de Seguretat de Pod (PSP): Un recurs a nivell de clúster que defineix un conjunt de condicions que un pod ha de complir per ser acceptat al sistema.
  • Controlador d'Admissió: Un fragment de codi que intercepta les sol·licituds al servidor API de Kubernetes abans de la persistència de l'objecte, però després que la sol·licitud hagi estat autenticada i autoritzada.
  • Context de Seguretat: Defineix els paràmetres de privilegi i control d'accés per a un Pod o Contenidor.

Creació d'una Política de Seguretat de Pod

Per crear una PSP, necessites definir un fitxer de configuració YAML que especifiqui els requisits de seguretat per als teus pods.

Exemple de YAML de PSP

apiVersion: policy/v1beta1
kind: PodSecurityPolicy
metadata:
  name: example-psp
spec:
  privileged: false
  allowPrivilegeEscalation: false
  requiredDropCapabilities:
    - ALL
  volumes:
    - 'configMap'
    - 'emptyDir'
    - 'projected'
    - 'secret'
  hostNetwork: false
  hostIPC: false
  hostPID: false
  runAsUser:
    rule: 'MustRunAsNonRoot'
  seLinux:
    rule: 'RunAsAny'
  supplementalGroups:
    rule: 'MustRunAs'
    ranges:
      - min: 1
        max: 65535
  fsGroup:
    rule: 'MustRunAs'
    ranges:
      - min: 1
        max: 65535

Explicació

  • privileged: Assegura que no es permetin contenidors privilegiats.
  • allowPrivilegeEscalation: Evita l'escalada de privilegis.
  • requiredDropCapabilities: Elimina totes les capacitats per minimitzar la superfície d'atac potencial.
  • volumes: Restringeix els tipus de volums que es poden utilitzar.
  • hostNetwork, hostIPC, hostPID: Evita l'ús de la xarxa, IPC i espais de noms PID de l'amfitrió.
  • runAsUser: Assegura que els contenidors s'executin com a usuari no root.
  • seLinux: Permet qualsevol context SELinux.
  • supplementalGroups: Requereix que els contenidors s'executin amb grups suplementaris específics.
  • fsGroup: Requereix que els contenidors s'executin amb grups de sistema de fitxers específics.

Aplicació d'una Política de Seguretat de Pod

Per aplicar una PSP, necessites habilitar el controlador d'admissió PodSecurityPolicy i crear els RoleBindings o ClusterRoleBindings apropiats.

Habilitant el Controlador d'Admissió

Edita la configuració del servidor API de Kubernetes per incloure el controlador d'admissió PodSecurityPolicy.

--enable-admission-plugins=PodSecurityPolicy

Creació de RoleBindings

Necessites crear RoleBindings o ClusterRoleBindings per vincular la PSP a usuaris o comptes de servei específics.

apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: use-example-psp
  namespace: default
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: Role
  name: use-example-psp
subjects:
- kind: User
  name: example-user
  apiGroup: rbac.authorization.k8s.io

Explicació

  • roleRef: Referencia el rol que concedeix permisos per utilitzar la PSP.
  • subjects: Especifica els usuaris o comptes de servei als quals s'aplica el RoleBinding.

Ús Avançat de les Polítiques de Seguretat de Pods

Combinació de Múltiples PSPs

Pots crear múltiples PSPs per atendre diferents requisits de seguretat per a diferents espais de noms o aplicacions.

Exemple

apiVersion: policy/v1beta1
kind: PodSecurityPolicy
metadata:
  name: restricted-psp
spec:
  privileged: false
  allowPrivilegeEscalation: false
  requiredDropCapabilities:
    - ALL
  runAsUser:
    rule: 'MustRunAsNonRoot'
  seLinux:
    rule: 'RunAsAny'
  supplementalGroups:
    rule: 'MustRunAs'
    ranges:
      - min: 1
        max: 65535
  fsGroup:
    rule: 'MustRunAs'
    ranges:
      - min: 1
        max: 65535
apiVersion: policy/v1beta1
kind: PodSecurityPolicy
metadata:
  name: permissive-psp
spec:
  privileged: true
  allowPrivilegeEscalation: true
  runAsUser:
    rule: 'RunAsAny'
  seLinux:
    rule: 'RunAsAny'
  supplementalGroups:
    rule: 'RunAsAny'
  fsGroup:
    rule: 'RunAsAny'

Explicació

  • restricted-psp: Enforça polítiques de seguretat estrictes.
  • permissive-psp: Permet configuracions més permissives per a casos d'ús específics.

Conclusió

Les Polítiques de Seguretat de Pods són una eina poderosa per fer complir els estàndards de seguretat en un clúster de Kubernetes. Comprenent i implementant PSPs, pots millorar significativament la postura de seguretat del teu entorn de Kubernetes. Recorda:

  • Defineix PSPs clares i concises adaptades als teus requisits de seguretat.
  • Aplica les PSPs utilitzant RoleBindings o ClusterRoleBindings.
  • Revisa i actualitza regularment les PSPs per adaptar-te a les necessitats de seguretat en evolució.

Dominant les PSPs, assegures que els teus clústers de Kubernetes siguin segurs i compleixin amb les millors pràctiques.
