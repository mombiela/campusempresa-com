Introducció a l'EFK Stack

L'EFK stack és un conjunt d'eines potent utilitzat per al registre i monitoratge en entorns Kubernetes. Consisteix en:

  • Elasticsearch: Un motor de cerca i analítica.
  • Fluentd: Un col·lector de dades per a un registre unificat.
  • Kibana: Una eina de visualització per a dades d'Elasticsearch.

Per què utilitzar EFK a Kubernetes?

  • Registre Centralitzat: Recollir registres de tots els pods i nodes en un sol lloc.
  • Escalabilitat: Gestionar grans volums de registres de manera eficient.
  • Cerca i Anàlisi: Utilitzar Elasticsearch per cercar i analitzar registres.
  • Visualització: Utilitzar Kibana per crear panells i visualitzar dades de registres.

Configuració de l'EFK Stack a Kubernetes

Prerequisits

  • Un clúster Kubernetes en funcionament.
  • Eina de línia de comandes kubectl configurada per interactuar amb el teu clúster.

Pas 1: Desplegar Elasticsearch

Elasticsearch emmagatzemarà i indexarà els registres recollits del clúster Kubernetes.

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: elasticsearch
spec:
  serviceName: "elasticsearch"
  replicas: 1
  selector:
    matchLabels:
      app: elasticsearch
  template:
    metadata:
      labels:
        app: elasticsearch
    spec:
      containers:
      - name: elasticsearch
        image: docker.elastic.co/elasticsearch/elasticsearch:7.10.1
        ports:
        - containerPort: 9200
          name: http
        - containerPort: 9300
          name: transport
        env:
        - name: discovery.type
          value: single-node

Pas 2: Desplegar Fluentd

Fluentd recollirà registres dels nodes Kubernetes i els reenviarà a Elasticsearch.

apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: fluentd
  labels:
    k8s-app: fluentd-logging
spec:
  selector:
    matchLabels:
      name: fluentd
  template:
    metadata:
      labels:
        name: fluentd
    spec:
      containers:
      - name: fluentd
        image: fluent/fluentd:v1.11.5-debian-1.0
        env:
        - name: FLUENT_ELASTICSEARCH_HOST
          value: "elasticsearch"
        - name: FLUENT_ELASTICSEARCH_PORT
          value: "9200"
        volumeMounts:
        - name: varlog
          mountPath: /var/log
        - name: varlibdockercontainers
          mountPath: /var/lib/docker/containers
          readOnly: true
      volumes:
      - name: varlog
        hostPath:
          path: /var/log
      - name: varlibdockercontainers
        hostPath:
          path: /var/lib/docker/containers

Pas 3: Desplegar Kibana

Kibana proporcionarà una interfície web per visualitzar els registres emmagatzemats a Elasticsearch.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: kibana
spec:
  replicas: 1
  selector:
    matchLabels:
      app: kibana
  template:
    metadata:
      labels:
        app: kibana
    spec:
      containers:
      - name: kibana
        image: docker.elastic.co/kibana/kibana:7.10.1
        ports:
        - containerPort: 5601
        env:
        - name: ELASTICSEARCH_HOSTS
          value: "http://elasticsearch:9200"

Pas 4: Exposar Kibana

Crear un servei per exposar Kibana de manera que puguis accedir-hi des del teu navegador.

apiVersion: v1
kind: Service
metadata:
  name: kibana
spec:
  selector:
    app: kibana
  ports:
  - protocol: TCP
    port: 5601
    targetPort: 5601
  type: NodePort

Accedir a Kibana

  • Obtenir el NodePort assignat al servei Kibana: 
    kubectl get svc kibana
  • Accedir a Kibana utilitzant la IP del Node i el NodePort al teu navegador: 
    http://<NodeIP>:<NodePort>

Conclusió

L'EFK stack proporciona una solució robusta per al registre i monitoratge en entorns Kubernetes. Desplegant Elasticsearch, Fluentd i Kibana, pots centralitzar, cercar i visualitzar registres de tot el teu clúster Kubernetes, facilitant la monitorització i resolució de problemes de les teves aplicacions.
