Introducció

A Kubernetes, gestionar dades de configuració i informació sensible és crucial per al desplegament i operació d'aplicacions. ConfigMaps i Secrets són dos recursos fonamentals que ajuden en aquest sentit. Aquesta secció cobrirà els conceptes bàsics i l'ús avançat de ConfigMaps i Secrets, proporcionant explicacions clares, exemples i exercicis.

ConfigMaps

Què és un ConfigMap?

  • Un ConfigMap és un objecte API utilitzat per emmagatzemar dades no confidencials en parells clau-valor.
  • Els ConfigMaps permeten deslligar els artefactes de configuració del contingut de la imatge per mantenir les aplicacions contenidoritzades portables.

Crear un ConfigMap

Pots crear un ConfigMap utilitzant un fitxer YAML o directament des de la línia de comandes.

Utilitzant un fitxer YAML

apiVersion: v1
kind: ConfigMap
metadata:
  name: example-config
data:
  database_url: "mongodb://localhost:27017"
  log_level: "DEBUG"

Per crear el ConfigMap:

kubectl apply -f configmap.yaml

Utilitzant la línia de comandes

kubectl create configmap example-config --from-literal=database_url="mongodb://localhost:27017" --from-literal=log_level="DEBUG"

Utilitzar ConfigMaps en Pods

Pots utilitzar ConfigMaps en Pods a través de variables d'entorn o muntatges de volums.

Variables d'entorn

apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    env:
    - name: DATABASE_URL
      valueFrom:
        configMapKeyRef:
          name: example-config
          key: database_url

Muntatges de volums

apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    volumeMounts:
    - name: config-volume
      mountPath: /etc/config
  volumes:
  - name: config-volume
    configMap:
      name: example-config

Millors pràctiques per a ConfigMaps

  • Utilitza ConfigMaps per a dades de configuració no sensibles.
  • Mantingues els ConfigMaps petits i manejables.
  • Versiona els teus ConfigMaps per fer un seguiment dels canvis.

Secrets

Què és un Secret?

  • Un Secret és un objecte API que emmagatzema dades sensibles com contrasenyes, tokens OAuth i claus SSH.
  • Els Secrets estan codificats en base64 per evitar l'exposició accidental, però no estan xifrats per defecte.

Crear un Secret

Pots crear un Secret utilitzant un fitxer YAML o directament des de la línia de comandes.

Utilitzant un fitxer YAML

apiVersion: v1
kind: Secret
metadata:
  name: example-secret
type: Opaque
data:
  username: YWRtaW4=  # codificat en base64 'admin'
  password: MWYyZDFlMmU2N2Rm  # codificat en base64 '1f2d1e2e67df'

Per crear el Secret:

kubectl apply -f secret.yaml

Utilitzant la línia de comandes

kubectl create secret generic example-secret --from-literal=username=admin --from-literal=password=1f2d1e2e67df

Utilitzar Secrets en Pods

Pots utilitzar Secrets en Pods a través de variables d'entorn o muntatges de volums.

Variables d'entorn

apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    env:
    - name: USERNAME
      valueFrom:
        secretKeyRef:
          name: example-secret
          key: username

Muntatges de volums

apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    volumeMounts:
    - name: secret-volume
      mountPath: /etc/secret
  volumes:
  - name: secret-volume
    secret:
      secretName: example-secret

Millors pràctiques per a Secrets

  • Utilitza Secrets per a dades sensibles.
  • Activa el xifrat en repòs per als Secrets.
  • Limita l'accés als Secrets utilitzant RBAC (Control d'Accés Basat en Rols).

Conclusió

Els ConfigMaps i Secrets són eines essencials a Kubernetes per gestionar dades de configuració i informació sensible. Els ConfigMaps són adequats per a dades no confidencials, mentre que els Secrets s'han d'utilitzar per a informació sensible. L'ús adequat i les millors pràctiques asseguren que les teves aplicacions siguin tant flexibles com segures. Entenent i implementant aquests recursos, pots millorar la portabilitat i la seguretat dels teus desplegaments a Kubernetes.
