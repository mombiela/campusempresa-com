Introducció

L'automatització dels desplegaments en Kubernetes és crucial per mantenir la consistència, reduir errors i millorar la velocitat de llançament de noves funcionalitats. Aquesta secció et guiarà des dels conceptes bàsics fins a tècniques avançades d'automatització de desplegaments utilitzant Kubernetes.

Conceptes Clau

  • Deployment: Un recurs de Kubernetes que proporciona actualitzacions declaratives a les aplicacions.
  • ReplicaSet: Assegura que un nombre especificat de rèpliques de pod estiguin en execució en qualsevol moment.
  • Rolling Updates: Substitueix gradualment les versions antigues dels pods per noves.
  • Blue-Green Deployment: Manté dos entorns idèntics, un en viu i un inactiu, per canviar entre ells.
  • Canary Deployment: Desvia gradualment el trànsit de la versió antiga a la nova versió.
  • CI/CD Pipelines: Les pipelines d'Integració Contínua i Desplegament Continu automatitzen el procés de construir, provar i desplegar codi.

Automatització Bàsica del Desplegament

Creació d'un Deployment

Es pot crear un desplegament senzill utilitzant un fitxer YAML. A continuació es mostra un exemple de configuració bàsica de desplegament:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app-container
        image: my-app-image:1.0
        ports:
        - containerPort: 80
  • apiVersion: Especifica la versió de l'API.
  • kind: El tipus de recurs de Kubernetes.
  • metadata: Metadades sobre el desplegament, incloent el nom.
  • spec: L'especificació del desplegament, incloent el nombre de rèpliques i la plantilla del pod.

Aplicació del Desplegament

Per aplicar el desplegament, utilitza la següent comanda:

kubectl apply -f my-app-deployment.yaml

Aquesta comanda crearà el desplegament i iniciarà el nombre especificat de rèpliques.

Estratègies Intermèdies de Desplegament

Rolling Updates

Les actualitzacions progressives permeten actualitzar l'aplicació sense temps d'inactivitat. Kubernetes gestiona el procés d'actualització substituint gradualment els pods antics per nous.

Per actualitzar el desplegament, modifica la versió de la imatge al fitxer YAML:

spec:
  template:
    spec:
      containers:
      - name: my-app-container
        image: my-app-image:2.0

Aplica els canvis:

kubectl apply -f my-app-deployment.yaml

Blue-Green Deployment

El desplegament Blue-Green implica mantenir dos entorns. Aquí tens un enfocament simplificat:

  1. Desplega la nova versió (green) al costat de la versió antiga (blue).
  2. Canvia el servei per apuntar a la nova versió.

Exemple:

apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app-green
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

Canviar el selector del servei a app: my-app-green redirigirà el trànsit a la nova versió.

Tècniques Avançades de Desplegament

Canary Deployment

Els desplegaments canary desvien gradualment el trànsit a la nova versió. Això es pot aconseguir utilitzant múltiples desplegaments i serveis.

  1. Crea un nou desplegament per a la versió canary.
  2. Utilitza un mesh de serveis o un controlador d'ingress per dirigir un petit percentatge de trànsit al desplegament canary.

Exemple utilitzant Istio:

apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: my-app
spec:
  hosts:
  - my-app.example.com
  http:
  - route:
    - destination:
        host: my-app
        subset: v1
      weight: 90
    - destination:
        host: my-app
        subset: v2
      weight: 10

CI/CD Pipelines

Integra els desplegaments de Kubernetes a la teva pipeline de CI/CD utilitzant eines com Jenkins, GitLab CI o GitHub Actions.

Exemple de workflow de GitHub Actions:

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout code
      uses: actions/checkout@v2
    - name: Build Docker image
      run: docker build -t my-app-image:${{ github.sha }} .
    - name: Push Docker image
      run: docker push my-app-image:${{ github.sha }}
    - name: Deploy to Kubernetes
      run: |
        kubectl set image deployment/my-app my-app-container=my-app-image:${{ github.sha }}

Conclusió

L'automatització dels desplegaments en Kubernetes millora la fiabilitat i l'eficiència del procés de lliurament de la teva aplicació. Comprenent i implementant diverses estratègies de desplegament com les actualitzacions progressives, els desplegaments blue-green i els desplegaments canary, pots assegurar llançaments suaus i controlats. Integrar aquestes pràctiques a les pipelines de CI/CD agilitza encara més el procés, permetent el desplegament continu i automatitzat de noves funcionalitats i actualitzacions.
