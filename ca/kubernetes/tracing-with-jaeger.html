Introducció a la Traçabilitat Distribuïda

La traçabilitat distribuïda és un mètode utilitzat per rastrejar i monitoritzar les sol·licituds a mesura que flueixen a través de diversos serveis en una arquitectura de microserveis. Ajuda a identificar colls d'ampolla de rendiment, entendre les dependències dels serveis i depurar problemes complexos.

Què és Jaeger?

Jaeger és una eina de traçabilitat distribuïda de codi obert, de punta a punta, que ajuda a monitoritzar i solucionar problemes en sistemes distribuïts basats en microserveis. S'utilitza per a:

  • Optimització del rendiment
  • Anàlisi de la causa arrel
  • Anàlisi de dependències de serveis

Configuració de Jaeger a Kubernetes

Per començar a utilitzar Jaeger al teu clúster de Kubernetes, necessites desplegar els components de Jaeger. Això es pot fer utilitzant Helm, un gestor de paquets per a Kubernetes.

Prerequisits

  • Un clúster de Kubernetes en funcionament
  • Helm instal·lat a la teva màquina local

Instal·lació de Jaeger amb Helm

  1. Afegeix el repositori Helm de Jaeger:

    helm repo add jaegertracing https://jaegertracing.github.io/helm-charts
helm repo update

  2. Instal·la Jaeger:

    helm install jaeger jaegertracing/jaeger

  3. Verifica la instal·lació:

    kubectl get all -l app.kubernetes.io/instance=jaeger

Instrumentació de la teva Aplicació

Per rastrejar sol·licituds, necessites instrumentar el codi de la teva aplicació per enviar dades de traçabilitat a Jaeger.

Exemple: Instrumentació d'una Aplicació Python

  1. Instal·la la biblioteca client de Jaeger:

    pip install jaeger-client

  2. Afegeix traçabilitat a la teva aplicació:

    from jaeger_client import Config
from flask import Flask

app = Flask(__name__)

def init_tracer(service_name='my-service'):
    config = Config(
        config={
            'sampler': {'type': 'const', 'param': 1},
            'logging': True,
        },
        service_name=service_name,
    )
    return config.initialize_tracer()

tracer = init_tracer()

@app.route('/')
def hello():
    with tracer.start_span('hello-world') as span:
        span.set_tag('example-tag', 'hello-world')
        return 'Hello, World!'

if __name__ == '__main__':
    app.run()

Visualització de Traces a la Interfície de Jaeger

Un cop la teva aplicació està instrumentada i en funcionament, pots veure les traces a la interfície de Jaeger.

  1. Accedeix a la interfície de Jaeger:

    kubectl port-forward svc/jaeger-query 16686:16686

    Obre el teu navegador i navega a http://localhost:16686.

  2. Cerca traces:

    • Selecciona el teu servei des del menú desplegable.
    • Fes clic a "Find Traces" per veure les dades de traçabilitat.

Configuració Avançada

Jaeger proporciona diverses opcions de configuració per personalitzar el seu comportament.

Estratègies de Mostreig

El mostreig determina quines traces es recullen i s'envien a Jaeger. Les estratègies comunes inclouen:

  • Mostreig Constant: Sempre mostreja un percentatge fix de traces.
  • Mostreig Probabilístic: Mostreja traces basades en una probabilitat.

Exemple: Configuració del Mostreig a Kubernetes

  1. Crea un ConfigMap per al mostreig:

    apiVersion: v1
kind: ConfigMap
metadata:
  name: jaeger-sampling
data:
  sampling.json: |
    {
      "service_strategies": [
        {
          "service": "my-service",
          "type": "probabilistic",
          "param": 0.1
        }
      ]
    }

  2. Aplica el ConfigMap:

    kubectl apply -f jaeger-sampling-configmap.yaml

  3. Actualitza el desplegament de Jaeger per utilitzar el ConfigMap:

    spec:
  containers:
  - name: jaeger-agent
    volumeMounts:
    - name: sampling-config
      mountPath: /etc/jaeger/sampling
      subPath: sampling.json
  volumes:
  - name: sampling-config
    configMap:
      name: jaeger-sampling

Conclusió

La traçabilitat amb Jaeger en un entorn Kubernetes proporciona informació poderosa sobre la teva arquitectura de microserveis. Seguint els passos descrits, pots configurar Jaeger, instrumentar les teves aplicacions i començar a analitzar traces per optimitzar el rendiment i solucionar problemes. Les configuracions avançades com les estratègies de mostreig et permeten ajustar la traçabilitat segons les teves necessitats.
