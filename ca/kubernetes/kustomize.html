Kustomize és una eina per personalitzar configuracions de Kubernetes. Et permet gestionar i personalitzar manifestos de Kubernetes sense necessitat d'utilitzar plantilles. Aquest tema et guiarà des dels conceptes bàsics de Kustomize fins a l'ús avançat, proporcionant explicacions clares, exemples i exercicis.

Introducció a Kustomize

Kustomize està integrat en kubectl i proporciona una manera de gestionar recursos de Kubernetes de manera declarativa. Et permet definir un conjunt base de recursos i després crear superposicions per personalitzar aquests recursos per a diferents entorns.

Conceptes Clau

  • Base: Un conjunt de manifestos de Kubernetes que defineixen els recursos comuns.
  • Superposició: Personalitzacions aplicades sobre els manifestos base per a diferents entorns (per exemple, desenvolupament, preproducció, producció).
  • kustomization.yaml: El fitxer de configuració utilitzat per Kustomize per definir com s'han de personalitzar els recursos.

Començant amb Kustomize

Instal·lant Kustomize

Kustomize està integrat en kubectl, però també el pots instal·lar com una eina independent.

Utilitzant kubectl

Kustomize està inclòs en kubectl a partir de la versió 1.14. El pots utilitzar directament amb la comanda kubectl kustomize.

kubectl kustomize <path-to-kustomization-directory>

Instal·lació Independent

Per instal·lar Kustomize com una eina independent, segueix les instruccions a la pàgina de GitHub de Kustomize.

Creant una Base

Una base és un conjunt de manifestos de Kubernetes que defineixen els recursos comuns. Creem una base simple per a un Deployment i un Service.

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:1.0.0
        ports:
        - containerPort: 80
# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

Creant un kustomization.yaml

Crea un fitxer kustomization.yaml per definir la base.

# kustomization.yaml
resources:
- deployment.yaml
- service.yaml

Aplicant la Base

Per aplicar la base, utilitza la següent comanda:

kubectl apply -k <path-to-base-directory>

Creant Superposicions

Les superposicions et permeten personalitzar els recursos base per a diferents entorns.

Exemple: Superposició de Desenvolupament

Crea un directori per a la superposició de desenvolupament i un fitxer kustomization.yaml dins d'aquest.

# overlays/development/kustomization.yaml
resources:
- ../../base

patchesStrategicMerge:
- deployment-patch.yaml

Crea un fitxer de pedaç per personalitzar el Deployment per a l'entorn de desenvolupament.

# overlays/development/deployment-patch.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 1
  template:
    spec:
      containers:
      - name: my-app
        image: my-app:1.0.1

Aplicant la Superposició

Per aplicar la superposició de desenvolupament, utilitza la següent comanda:

kubectl apply -k overlays/development

Funcions Avançades de Kustomize

Utilitzant ConfigMaps i Secrets

Kustomize et permet generar ConfigMaps i Secrets a partir de fitxers o literals.

# kustomization.yaml
configMapGenerator:
- name: my-config
  files:
  - config.properties

secretGenerator:
- name: my-secret
  literals:
  - username=admin
  - password=secret

Etiquetes i Anotacions Comunes

Pots afegir etiquetes i anotacions comunes a tots els recursos.

# kustomization.yaml
commonLabels:
  app: my-app
commonAnnotations:
  environment: development

Transformacions

Kustomize suporta diverses transformacions, com namePrefix, nameSuffix i transformacions d'imatge.

# kustomization.yaml
namePrefix: dev-
nameSuffix: -v1

images:
- name: my-app
  newName: my-app
  newTag: 1.0.1

Conclusió

Kustomize és una eina poderosa per gestionar i personalitzar configuracions de Kubernetes. Et permet definir un conjunt base de recursos i crear superposicions per a diferents entorns, facilitant la gestió de desplegaments complexos. Entenent i utilitzant Kustomize, pots optimitzar els teus fluxos de treball de Kubernetes i millorar la mantenibilitat de les teves configuracions.

Resum

  • Base: Conjunt comú de manifestos de Kubernetes.
  • Superposició: Personalitzacions per a diferents entorns.
  • kustomization.yaml: Fitxer de configuració per a Kustomize.
  • Funcions Clau: Generació de ConfigMap i Secret, etiquetes i anotacions comunes, transformacions.

Practica utilitzant Kustomize creant les teves pròpies bases i superposicions, i explora les seves funcions avançades per aprofitar al màxim les seves capacitats en els teus desplegaments de Kubernetes.
