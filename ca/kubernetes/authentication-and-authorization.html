Introducció

L'autenticació i l'autorització són components crítics per assegurar un clúster de Kubernetes. L'autenticació verifica la identitat d'un usuari o servei, mentre que l'autorització determina quines accions poden realitzar. Aquesta secció et guiarà des dels conceptes bàsics fins als avançats d'autenticació i autorització en Kubernetes.

Autenticació

Visió General

L'autenticació en Kubernetes és el procés de verificar la identitat d'un usuari o servei. Kubernetes admet múltiples mètodes d'autenticació, incloent:

  • Certificats de Client
  • Tokens Bearer
  • Tokens OpenID Connect
  • Autenticació de Token Webhook

Certificats de Client

Els certificats de client s'utilitzen per autenticar usuaris i serveis. Kubernetes utilitza certificats TLS per assegurar la comunicació entre el client i el servidor API.

Exemple

apiVersion: v1
kind: Config
clusters:
- cluster:
    certificate-authority: /path/to/ca.crt
    server: https://kubernetes.example.com
  name: example-cluster
contexts:
- context:
    cluster: example-cluster
    user: example-user
  name: example-context
current-context: example-context
users:
- name: example-user
  user:
    client-certificate: /path/to/client.crt
    client-key: /path/to/client.key

Tokens Bearer

Els tokens bearer són un tipus de token que es poden utilitzar per autenticar sol·licituds al servidor API de Kubernetes. Aquests tokens s'utilitzen típicament per comptes de servei.

Exemple

apiVersion: v1
kind: ServiceAccount
metadata:
  name: example-service-account

Tokens OpenID Connect

Els tokens OpenID Connect (OIDC) s'utilitzen per autenticar usuaris a través d'un proveïdor d'identitat extern.

Exemple

apiVersion: v1
kind: Config
clusters:
- cluster:
    certificate-authority: /path/to/ca.crt
    server: https://kubernetes.example.com
  name: example-cluster
contexts:
- context:
    cluster: example-cluster
    user: oidc-user
  name: example-context
current-context: example-context
users:
- name: oidc-user
  user:
    auth-provider:
      name: oidc
      config:
        client-id: your-client-id
        client-secret: your-client-secret
        id-token: your-id-token
        idp-issuer-url: https://accounts.google.com
        refresh-token: your-refresh-token

Autenticació de Token Webhook

L'autenticació de token webhook et permet integrar-te amb un servei d'autenticació extern.

Exemple

apiVersion: v1
kind: Config
clusters:
- cluster:
    certificate-authority: /path/to/ca.crt
    server: https://kubernetes.example.com
  name: example-cluster
contexts:
- context:
    cluster: example-cluster
    user: webhook-user
  name: example-context
current-context: example-context
users:
- name: webhook-user
  user:
    token: your-webhook-token

Autorització

Visió General

L'autorització en Kubernetes determina quines accions un usuari o servei està autoritzat a realitzar. Kubernetes admet diversos mètodes d'autorització, incloent:

  • Control d'Accés Basat en Rols (RBAC)
  • Control d'Accés Basat en Atributs (ABAC)
  • Autorització Webhook

Control d'Accés Basat en Rols (RBAC)

RBAC és el mètode d'autorització més utilitzat en Kubernetes. Utilitza rols i vinculacions de rols per controlar l'accés als recursos.

Exemple

apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: example-role
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "list", "watch"]
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: example-rolebinding
  namespace: default
subjects:
- kind: User
  name: example-user
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: example-role
  apiGroup: rbac.authorization.k8s.io

Control d'Accés Basat en Atributs (ABAC)

ABAC utilitza polítiques definides en JSON per controlar l'accés als recursos.

Exemple

{
  "apiVersion": "abac.authorization.kubernetes.io/v1beta1",
  "kind": "Policy",
  "spec": {
    "user": "example-user",
    "namespace": "default",
    "resource": "pods",
    "readonly": true
  }
}

Autorització Webhook

L'autorització webhook et permet integrar-te amb un servei d'autorització extern.

Exemple

apiVersion: v1
kind: Config
clusters:
- cluster:
    certificate-authority: /path/to/ca.crt
    server: https://kubernetes.example.com
  name: example-cluster
contexts:
- context:
    cluster: example-cluster
    user: webhook-user
  name: example-context
current-context: example-context
users:
- name: webhook-user
  user:
    token: your-webhook-token

Conclusió

L'autenticació i l'autorització són essencials per assegurar un clúster de Kubernetes. Comprenent i implementant els diversos mètodes disponibles, pots assegurar-te que només els usuaris i serveis autoritzats tinguin accés als teus recursos. Aquesta secció ha cobert els conceptes bàsics fins als avançats, incloent certificats de client, tokens bearer, tokens OpenID Connect, autenticació de token webhook, RBAC, ABAC i autorització webhook.
