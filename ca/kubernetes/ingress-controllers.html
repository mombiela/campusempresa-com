Introducció als Controladors d'Ingress

Els Controladors d'Ingress són un component crític a Kubernetes per gestionar l'accés extern als serveis dins d'un clúster. Proporcionen una manera d'exposar rutes HTTP i HTTPS des de fora del clúster als serveis dins del clúster.

Conceptes Clau

  • Recurs d'Ingress: Una col·lecció de regles que permeten que les connexions entrants arribin als serveis del clúster.
  • Controlador d'Ingress: Un dimoni que vigila el servidor API de Kubernetes per a actualitzacions dels recursos d'Ingress i configura el balancejador de càrrega en conseqüència.

Configuració d'un Controlador d'Ingress

Per utilitzar un recurs d'Ingress, necessites tenir un Controlador d'Ingress executant-se en el teu clúster. Hi ha diferents Controladors d'Ingress disponibles, com NGINX, Traefik i HAProxy.

Instal·lació del Controlador d'Ingress NGINX

NGINX és un dels Controladors d'Ingress més populars. Aquí tens com instal·lar-lo:

kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/cloud/deploy.yaml

Aquesta comanda desplegarà el Controlador d'Ingress NGINX en el teu clúster.

Creació d'un Recurs d'Ingress

Un cop el Controlador d'Ingress està instal·lat, pots crear un recurs d'Ingress per definir com el trànsit extern ha de ser dirigit als teus serveis.

Exemple de Recurs d'Ingress

Aquí tens un exemple d'un recurs d'Ingress:

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: example-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  rules:
  - host: example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: example-service
            port:
              number: 80

Explicació

  • apiVersion: Especifica la versió de l'API.
  • kind: Defineix el tipus de recurs, que és Ingress.
  • metadata: Conté metadades sobre el recurs d'Ingress, com el seu nom.
  • annotations: Proporcionen opcions de configuració addicionals per al Controlador d'Ingress.
  • spec: Defineix l'estat desitjat de l'Ingress.
    • rules: Una llista de regles per dirigir el trànsit.
      • host: El nom de domini per a la regla.
      • http: Especifica que la regla s'aplica al trànsit HTTP.
        • paths: Una llista de camins a coincidir.
          • path: El camí URL a coincidir.
          • pathType: El tipus de coincidència de camí (per exemple, Prefix).
          • backend: Especifica el servei backend al qual dirigir el trànsit.
            • service: El nom i el port del servei.

Funcions Avançades d'Ingress

Els Controladors d'Ingress ofereixen funcions avançades com la terminació SSL, la reescriptura d'URL i el balanceig de càrrega.

Terminació SSL

Per habilitar la terminació SSL, necessites crear un secret TLS i referenciar-lo en el teu recurs d'Ingress.

Creació d'un Secret TLS

kubectl create secret tls example-tls --cert=path/to/tls.crt --key=path/to/tls.key

Actualització del Recurs d'Ingress

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: example-ingress
spec:
  tls:
  - hosts:
    - example.com
    secretName: example-tls
  rules:
  - host: example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: example-service
            port:
              number: 80

Reescriptura d'URL

La reescriptura d'URL es pot configurar utilitzant anotacions. Per exemple, per reescriure tots els camins a l'arrel:

metadata:
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /

Resolució de Problemes amb els Controladors d'Ingress

Els problemes comuns amb els Controladors d'Ingress inclouen configuracions incorrectes i problemes de xarxa. Aquí tens alguns consells per resoldre problemes:

  • Comprova els Registres del Controlador d'Ingress: Utilitza kubectl logs per veure els registres del Controlador d'Ingress.
  • Valida el Recurs d'Ingress: Assegura't que el recurs d'Ingress està correctament definit.
  • Polítiques de Xarxa: Verifica que les polítiques de xarxa no estiguin bloquejant el trànsit.

Conclusió

Els Controladors d'Ingress són essencials per gestionar l'accés extern als serveis en un clúster de Kubernetes. Entenent com configurar i configurar els Controladors d'Ingress, pots dirigir efectivament el trànsit cap a les teves aplicacions i aprofitar funcions avançades com la terminació SSL i la reescriptura d'URL.
