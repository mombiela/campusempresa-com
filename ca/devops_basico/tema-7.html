Introducció

En aquesta secció del curs sobre DevOps Bàsic, aprendrem a configurar un pipeline d'Integració Contínua (CI). La Integració Contínua és una pràctica essencial en DevOps que permet als equips de desenvolupament integrar el seu treball freqüentment, cosa que facilita la detecció primerenca d'errors i millora la qualitat del programari.

Conceptes Clau

  • Pipeline de CI: Un conjunt de processos automatitzats que permeten la integració contínua del codi.
  • Repositori de Codi: Lloc on s'emmagatzema el codi font, comunament en plataformes com GitHub, GitLab o Bitbucket.
  • Build: Procés de compilació del codi font.
  • Test: Execució de proves automatitzades per assegurar la qualitat del codi.
  • Deploy: Desplegament del codi en un entorn de prova o producció.

Eines Comunes per a CI

| Eina | Descripció | |--------------|-----------------------------------------------------------------------------| | Jenkins | Una de les eines de CI més populars i àmpliament utilitzada. | | Travis CI | Servei de CI basat en el núvol que s'integra bé amb GitHub. | | CircleCI | Plataforma de CI/CD que permet l'automatització de tot el cicle de vida. | | GitLab CI/CD | Integrat en GitLab, permet configurar pipelines directament en el repositori. |

Configuració Bàsica d'un Pipeline de CI amb Jenkins

Pas 1: Instal·lació de Jenkins

  1. Descarregar Jenkins: Visita la pàgina oficial de Jenkins i descarrega la versió adequada per al teu sistema operatiu.
  2. Instal·lar Jenkins: Segueix les instruccions d'instal·lació específiques per al teu sistema operatiu.
  3. Iniciar Jenkins: Un cop instal·lat, inicia Jenkins i accedeix a la interfície web a través de http://localhost:8080.

Pas 2: Configuració Inicial

  1. Desbloquejar Jenkins: Utilitza la contrasenya inicial proporcionada durant la instal·lació.
  2. Instal·lar Plugins Suggerits: Jenkins et suggerirà una llista de plugins essencials. Instal·la'ls per tenir una configuració bàsica funcional.
  3. Crear un Usuari Administrador: Configura un usuari administrador per gestionar Jenkins.

Pas 3: Crear un Nou Job

  1. Nou Ítem: A la interfície de Jenkins, selecciona "Nou Ítem".
  2. Nom del Job: Assigna un nom al teu job i selecciona "Freestyle project".
  3. Configuració del Repositori: A la secció "Source Code Management", selecciona "Git" i proporciona la URL del teu repositori.

Pas 4: Configuració del Pipeline

  1. Build Triggers: Configura els triggers perquè el pipeline s'executi automàticament. Per exemple, pots utilitzar "GitHub hook trigger for GITScm polling".
  2. Build Steps: Afegeix passos de construcció. Per exemple, si estàs utilitzant Maven, selecciona "Invoke top-level Maven targets" i proporciona els objectius de Maven com clean install.
  3. Post-build Actions: Afegeix accions post-construcció com "Publish JUnit test result report" per publicar els resultats de les proves.

Exemple de Configuració d'un Job en Jenkins

pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/el-teu-repositori.git'
            }
        }
        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Deploy') {
            steps {
                sh 'mvn deploy'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}

Conclusió

La configuració d'un pipeline de CI és un pas crucial en la implementació de pràctiques DevOps. Utilitzant eines com Jenkins, pots automatitzar la integració i desplegament del teu codi, millorant l'eficiència i qualitat del teu desenvolupament. En aquesta secció, hem cobert els conceptes bàsics i proporcionat un exemple pràctic perquè puguis començar a configurar el teu propi pipeline de CI.
