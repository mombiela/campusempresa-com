Introducció a la Integració Contínua (CI)

La Integració Contínua (CI) és una pràctica de desenvolupament de programari on els desenvolupadors integren el seu codi en un repositori compartit diverses vegades al dia. Cada integració és verificada per una compilació automatitzada i proves per detectar errors el més aviat possible.

Jenkins

Jenkins és una de les eines de CI més populars i àmpliament utilitzades. És un servidor d'automatització de codi obert que admet la construcció, implementació i automatització de qualsevol projecte.

Característiques Clau de Jenkins

  • Open Source: Gratuït i de codi obert.
  • Extensible: Més de 1000 plugins per integrar amb diverses eines de desenvolupament.
  • Configuració: Configuració basada en GUI i scripts.
  • Suport Multiplataforma: Funciona en Windows, macOS i Linux.

Exemple de Configuració de Jenkins

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                sh 'make' // Comando per construir el projecte
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                sh 'make test' // Comando per executar proves
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                sh 'make deploy' // Comando per desplegar el projecte
            }
        }
    }
}

Travis CI

Travis CI és una eina de CI basada en el núvol que s'integra perfectament amb GitHub. És coneguda per la seva facilitat d'ús i configuració ràpida.

Característiques Clau de Travis CI

  • Integració amb GitHub: Fàcil integració amb repositoris de GitHub.
  • Configuració Senzilla: Configuració mitjançant un fitxer .travis.yml.
  • Suport Multillenguatge: Admet múltiples llenguatges de programació.
  • Entorn d'Execució en el Núvol: No requereix instal·lació local.

Exemple de Configuració de Travis CI

language: python
python:
  - "3.8"
# Instal·lació de dependències
install:
  - pip install -r requirements.txt
# Script de proves
script:
  - pytest

CircleCI

CircleCI és una altra eina de CI basada en el núvol que es destaca per la seva velocitat i flexibilitat. Ofereix una integració profunda amb GitHub i Bitbucket.

Característiques Clau de CircleCI

  • Configuració en YAML: Configuració mitjançant un fitxer config.yml.
  • Paral·lelització: Execució de tasques en paral·lel per accelerar el procés.
  • Integració amb VCS: Suport per a GitHub i Bitbucket.
  • Entorn d'Execució en el Núvol i Local: Opcions per executar en el núvol o en servidors locals.

Exemple de Configuració de CircleCI

version: 2.1
jobs:
  build:
    docker:
      - image: circleci/python:3.8
    steps:
      - checkout
      - run:
          name: Install dependencies
          command: |
            python -m venv venv
            . venv/bin/activate
            pip install -r requirements.txt
      - run:
          name: Run tests
          command: |
            . venv/bin/activate
            pytest

Comparació d'Eines de CI

Conclusió

Les eines de CI com Jenkins, Travis CI i CircleCI són fonamentals per implementar la pràctica d'Integració Contínua en projectes de programari. Cada eina té els seus propis avantatges i característiques úniques que s'adapten a diferents necessitats i entorns de desenvolupament. Jenkins és altament configurable i extensible, Travis CI és fàcil d'usar i s'integra bé amb GitHub, mentre que CircleCI ofereix flexibilitat i velocitat amb suport per a múltiples VCS. L'elecció de l'eina adequada dependrà dels requisits específics del teu projecte i equip.
