El Projecte Sobre nosaltres Contribuir Donacions Llicència

En aquest tema, explorarem les diferències entre Integració Contínua (CI) i Lliurament Continu (CD), dues pràctiques fonamentals en l'àmbit de DevOps. Aquestes pràctiques ajuden a millorar l'eficiència i la qualitat del desenvolupament de programari.

Integració Contínua (CI)

La Integració Contínua és una pràctica de desenvolupament de programari on els desenvolupadors integren el seu codi de manera freqüent en un repositori compartit. Cada integració és verificada per una build automatitzada i proves per detectar errors el més aviat possible.

Conceptes Clau de CI

  • Freqüència d'Integració: Els desenvolupadors integren el seu codi diverses vegades al dia.
  • Build Automatitzada: Cada integració dispara una build automatitzada que compila el codi.
  • Proves Automatitzades: S'executen proves automatitzades per assegurar que la nova integració no trenca el sistema.
  • Detecció Primerenca d'Errors: En integrar freqüentment, els errors es detecten i corregeixen ràpidament.

Exemple de Flux de CI

  1. Un desenvolupador realitza canvis en el codi i els puja al repositori.
  2. Un servidor de CI detecta el canvi i dispara una build.
  3. La build inclou la compilació del codi i l'execució de proves automatitzades.
  4. Si la build i les proves són exitoses, el codi s'integra en la branca principal.
# Exemple de configuració de CI usant GitHub Actions
name: CI Pipeline

on: [push]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up JDK 11
      uses: actions/setup-java@v1
      with:
        java-version: 11
    - name: Build with Gradle
      run: ./gradlew build
    - name: Run tests
      run: ./gradlew test

Lliurament Continu (CD)

El Lliurament Continu és una pràctica que extén la Integració Contínua assegurant que el codi que passa les proves automatitzades pot ser desplegat automàticament en un entorn de producció.

Conceptes Clau de CD

  • Desplegament Automatitzat: El codi que passa les proves és desplegat automàticament.
  • Entorns de Prova i Producció: S'utilitzen múltiples entorns (desenvolupament, prova, producció) per assegurar la qualitat.
  • Feedback Ràpid: Els equips reben feedback ràpid sobre l'estat del codi en producció.
  • Menor Risc de Desplegament: Els desplegaments freqüents i automatitzats redueixen el risc d'errors en producció.

Exemple de Flux de CD

  1. El codi passa les proves automatitzades en l'entorn de CI.
  2. El codi és desplegat automàticament en un entorn de prova.
  3. Si les proves en l'entorn de prova són exitoses, el codi es desplega en producció.
# Exemple de configuració de CD usant GitHub Actions
name: CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Deploy to Production
      run: ./deploy.sh

Comparació entre CI i CD

| Característica | Integració Contínua (CI) | Lliurament Continu (CD) | |----------------------------|--------------------------|--------------------------------| | Freqüència d'Integració | Diverses vegades al dia | Basat en la integració exitosa | | Build Automatitzada | Sí | Sí | | Proves Automatitzades | Sí | Sí | | Desplegament Automatitzat | No | Sí | | Entorns | Desenvolupament | Desenvolupament, Prova, Producció | | Feedback | Ràpid | Molt ràpid | | Risc de Desplegament | Mitjà | Baix |

Conclusió

La Integració Contínua (CI) i el Lliurament Continu (CD) són pràctiques essencials en DevOps que milloren la qualitat i l'eficiència del desenvolupament de programari. CI se centra en integrar i verificar el codi freqüentment, mentre que CD assegura que el codi que passa les proves pot ser desplegat automàticament en producció. Juntes, aquestes pràctiques permeten als equips de desenvolupament lliurar programari d'alta qualitat de manera ràpida i fiable.
