Introducció

El projecte final del curs de DevOps Bàsic té com a objectiu integrar i aplicar els coneixements adquirits al llarg del curs. Els estudiants treballaran en un projecte pràctic que simula un entorn real de DevOps, permetent-los experimentar amb les eines i pràctiques essencials de DevOps.

Objectius del Projecte

  • Implementar un pipeline de CI/CD: Configurar un pipeline d'Integració Contínua i Desplegament Continu utilitzant eines com Jenkins, GitLab CI, o GitHub Actions.
  • Automatització d'Infraestructura: Utilitzar eines d'infraestructura com a codi (IaC) com Terraform o Ansible per automatitzar la provisió de recursos.
  • Monitoratge i Logging: Implementar solucions de monitoratge i logging per assegurar l'observabilitat del sistema.
  • Gestió de Contenidors: Utilitzar Docker per a la creació i gestió de contenidors, i Kubernetes per a l'orquestració de contenidors.

Requisits del Projecte

Pipeline de CI/CD

  • Repositori de Codi: Configurar un repositori a GitHub o GitLab.
  • Integració Contínua: Configurar un pipeline que executi proves automàtiques cada vegada que es realitzi un commit.
  • Desplegament Continu: Configurar el pipeline per desplegar automàticament l'aplicació en un entorn de staging i, posteriorment, en producció.
# Exemple de pipeline a GitLab CI
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - echo "Building the application..."
    - ./build.sh

test:
  stage: test
  script:
    - echo "Running tests..."
    - ./run_tests.sh

deploy:
  stage: deploy
  script:
    - echo "Deploying to production..."
    - ./deploy.sh
  only:
    - master

Automatització d'Infraestructura

  • Provisió de Recursos: Utilitzar Terraform per crear i gestionar recursos al núvol (AWS, Azure, GCP).
  • Configuració de Servidors: Utilitzar Ansible per configurar servidors i desplegar aplicacions.
# Exemple de configuració de Terraform
provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "WebServer"
  }
}

Monitoratge i Logging

  • Monitoratge: Implementar Prometheus per a la recol·lecció de mètriques i Grafana per a la visualització.
  • Logging: Configurar ELK Stack (Elasticsearch, Logstash, Kibana) per a la gestió de logs.
# Exemple de configuració de Prometheus
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

Gestió de Contenidors

  • Docker: Crear Dockerfiles per a l'aplicació i les seves dependències.
  • Kubernetes: Desplegar l'aplicació en un clúster de Kubernetes.
# Exemple de Dockerfile
FROM node:14
WORKDIR /app
COPY package.json .
RUN npm install
COPY . .
CMD ["npm", "start"]
# Exemple de configuració de Kubernetes
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: web
        image: myapp:latest
        ports:
        - containerPort: 80

Conclusió

El projecte final és una oportunitat per aplicar de manera pràctica els conceptes i eines apresos durant el curs de DevOps Bàsic. En completar aquest projecte, els estudiants hauran adquirit experiència en la configuració de pipelines de CI/CD, l'automatització d'infraestructura, el monitoratge i logging, i la gestió de contenidors. Aquest projecte no només consolidarà els seus coneixements, sinó que també els proporcionarà una base sòlida per afrontar desafiaments reals en l'àmbit de DevOps.
