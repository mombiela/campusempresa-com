Introducció a les proves automatitzades en CI

Les proves automatitzades són una part crucial del procés d'Integració Contínua (CI). Permeten als equips de desenvolupament detectar errors ràpidament, assegurant que el codi nou no trenqui funcionalitats existents. En aquesta secció, explorarem els conceptes bàsics de les proves automatitzades i com s'integren en un pipeline de CI.

Conceptes clau

  • Integració Contínua (CI): Pràctica d'integrar el treball de tots els desenvolupadors en un repositori compartit diverses vegades al dia.
  • Proves automatitzades: Scripts que verifiquen automàticament que el codi funciona com s'espera.
  • Pipeline de CI: Seqüència de passos automatitzats que s'executen cada vegada que es realitza un canvi en el codi.

Tipus de proves automatitzades

  • Proves unitàries: Verifiquen el funcionament de components individuals del codi.
  • Proves d'integració: Asseguren que diferents mòduls o serveis funcionin bé junts.
  • Proves funcionals: Avaluen el sistema complet per assegurar-se que compleix amb els requisits especificats.
  • Proves de regressió: Garanteixen que els canvis recents no hagin introduït nous errors en el codi existent.

Eines comunes per a proves automatitzades

| Eina | Tipus de prova | Descripció | |---------------------|----------------------|-----------------------------------------------------------------------------| | JUnit | Proves unitàries | Framework de proves unitàries per a Java. | | Selenium | Proves funcionals | Eina per a proves d'aplicacions web. | | Postman | Proves d'integració | Plataforma per a proves d'API. | | Jenkins | CI/CD | Eina d'automatització per construir, provar i desplegar aplicacions. |

Integració de proves automatitzades en un pipeline de CI

Pas 1: Configuració de l'entorn de proves

Abans d'executar les proves, és crucial configurar l'entorn de proves. Això inclou la instal·lació de dependències i la configuració de bases de dades o serveis necessaris.

# Exemple de configuració en un fitxer Jenkinsfile
pipeline {
    agent any
    stages {
        stage('Setup') {
            steps {
                sh 'npm install'  # Instal·lació de dependències
            }
        }
    }
}

Pas 2: Execució de proves

Un cop l'entorn està configurat, es poden executar les proves. Això es fa generalment en una etapa separada del pipeline.

pipeline {
    agent any
    stages {
        stage('Setup') {
            steps {
                sh 'npm install'
            }
        }
        stage('Test') {
            steps {
                sh 'npm test'  # Execució de proves
            }
        }
    }
}

Pas 3: Informe de resultats

Després d'executar les proves, és important generar i emmagatzemar els informes de resultats per a la seva revisió.

pipeline {
    agent any
    stages {
        stage('Setup') {
            steps {
                sh 'npm install'
            }
        }
        stage('Test') {
            steps {
                sh 'npm test'
                junit 'reports/**/*.xml'  # Publicació de resultats de proves
            }
        }
    }
}

Exemple pràctic

Suposem que estem treballant en una aplicació web en Node.js i volem integrar proves automatitzades en el nostre pipeline de CI utilitzant Jenkins.

  1. Configurar Jenkinsfile: Creem un fitxer Jenkinsfile en el directori arrel del nostre projecte.
pipeline {
    agent any
    stages {
        stage('Setup') {
            steps {
                sh 'npm install'
            }
        }
        stage('Test') {
            steps {
                sh 'npm test'
            }
        }
        stage('Report') {
            steps {
                junit 'reports/**/*.xml'
            }
        }
    }
}
  1. Configurar proves en package.json: Assegurem que el nostre fitxer package.json tingui un script per executar les proves.
{
  "scripts": {
    "test": "mocha tests/**/*.js --reporter junit --reporter-options mochaFile=reports/test-results.xml"
  }
}

Conclusió

Les proves automatitzades són essencials per mantenir la qualitat del codi en un entorn d'Integració Contínua. En integrar aquestes proves en el pipeline de CI, els equips poden detectar i corregir errors ràpidament, millorant l'eficiència i la fiabilitat del desenvolupament de programari. Amb eines adequades i una configuració correcta, les proves automatitzades poden ser una poderosa addició a qualsevol procés de CI.
