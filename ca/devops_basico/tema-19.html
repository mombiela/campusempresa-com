Introducció a Kubernetes

Kubernetes és una plataforma d'orquestració de contenidors de codi obert que automatitza la implementació, l'escalat i la gestió d'aplicacions en contenidors. Va ser desenvolupat originalment per Google i ara és mantingut per la Cloud Native Computing Foundation (CNCF).

Conceptes Clau

  • Node: Una màquina (física o virtual) en el clúster de Kubernetes.
  • Clúster: Conjunt de nodes gestionats per Kubernetes.
  • Pod: Unitat mínima de desplegament en Kubernetes, que pot contenir un o més contenidors.
  • Servei: Abstracció que defineix un conjunt lògic de Pods i una política per accedir a ells.
  • Namespace: Mecanisme per aïllar recursos dins d'un clúster.
  • ConfigMap i Secret: Mecanismes per injectar configuracions i dades sensibles en els Pods.

Arquitectura de Kubernetes

Kubernetes segueix una arquitectura mestre-esclau:

  • Master Node: Gestiona el clúster i coordina les activitats.

    • API Server: Punt d'entrada per a totes les operacions d'administració del clúster.
    • etcd: Magatzem de dades distribuït que guarda la configuració del clúster.
    • Controller Manager: Executa els controladors que regulen l'estat del clúster.
    • Scheduler: Assigna Pods als nodes disponibles.

  • Worker Nodes: Executen les aplicacions en contenidors.

    • Kubelet: Agent que assegura que els contenidors estan corrent en un Pod.
    • Kube-proxy: Gestiona les regles de xarxa en els nodes.
    • Container Runtime: Programari que executa els contenidors (per exemple, Docker).

Desplegament d'Aplicacions en Kubernetes

Creació d'un Pod

Un Pod és la unitat bàsica d'execució en Kubernetes. A continuació es mostra un exemple d'un fitxer YAML per crear un Pod:

apiVersion: v1
kind: Pod
metadata:
  name: my-pod
spec:
  containers:
  - name: my-container
    image: nginx:latest
    ports:
    - containerPort: 80

Per desplegar aquest Pod, s'utilitza el comandament kubectl:

kubectl apply -f my-pod.yaml

Creació d'un Deployment

Un Deployment proporciona actualitzacions declaratives per a Pods i ReplicaSets. Aquí hi ha un exemple d'un fitxer YAML per a un Deployment:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-container
        image: nginx:latest
        ports:
        - containerPort: 80

Per desplegar aquest Deployment, s'utilitza el comandament:

kubectl apply -f my-deployment.yaml

Serveis en Kubernetes

Un Servei en Kubernetes exposa un o més Pods a través d'una adreça IP única. Aquí hi ha un exemple d'un fitxer YAML per a un Servei:

apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
  type: ClusterIP

Per crear aquest Servei, s'utilitza el comandament:

kubectl apply -f my-service.yaml

Exercicis Pràctics

  1. Crear un Pod: Escriu un fitxer YAML per crear un Pod que executi una aplicació simple (per exemple, un servidor web Nginx) i desplega el Pod en el teu clúster.
  2. Crear un Deployment: Escriu un fitxer YAML per a un Deployment que desplegui tres rèpliques d'una aplicació i desplega el Deployment en el teu clúster.
  3. Exposar un Servei: Escriu un fitxer YAML per a un Servei que exposi el Deployment creat anteriorment i desplega el Servei en el teu clúster.

Conclusió

Kubernetes és una eina poderosa per a l'orquestració de contenidors, que facilita la gestió d'aplicacions en contenidors a gran escala. Comprendre els conceptes bàsics i l'arquitectura de Kubernetes és essencial per a qualsevol professional de DevOps. Amb la pràctica, podràs desplegar, escalar i gestionar aplicacions de manera eficient en un entorn de producció.
