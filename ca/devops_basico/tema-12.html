Introducció

En aquesta secció del curs sobre DevOps Bàsic, aprendrem a implementar un pipeline de Continuous Deployment (CD). Un pipeline de CD és una sèrie de passos automatitzats que permeten el lliurament continu de programari des del desenvolupament fins a la producció. Aquest procés assegura que cada canvi en el codi font passi per una sèrie de proves i validacions abans de ser desplegat automàticament en l'entorn de producció.

Conceptes Clau

  • Continuous Deployment (CD): Procés automatitzat que permet el lliurament continu de programari a producció.
  • Pipeline: Sèrie de passos o etapes que el codi ha de passar abans de ser desplegat.
  • Automatització: Ús d'eines i scripts per realitzar tasques repetitives sense intervenció humana.
  • Integració Contínua (CI): Procés d'integrar canvis de codi freqüentment i verificar la seva validesa mitjançant proves automatitzades.

Components d'un Pipeline de CD

  1. Repositori de Codi Font:

    • Emmagatzema el codi del projecte.
    • Exemples: GitHub, GitLab, Bitbucket.

  2. Servidor d'Integració Contínua (CI):

    • Executa proves automatitzades i construeix el codi.
    • Exemples: Jenkins, Travis CI, CircleCI.

  3. Eina de Gestió de Configuració:

    • Gestiona la configuració i l'estat de l'entorn.
    • Exemples: Ansible, Chef, Puppet.

  4. Entorn de Proves:

    • Entorn on s'executen proves automatitzades.
    • Pot ser un entorn de staging o pre-producció.

  5. Entorn de Producció:

    • Entorn final on es desplega el programari per als usuaris finals.

Passos per Implementar un Pipeline de CD

1. Configuració del Repositori de Codi Font

  • Crear un repositori en una plataforma com GitHub.
  • Configurar branques per a desenvolupament i producció.
  • Establir polítiques de revisió de codi.
# Crear un nou repositori
git init my-project
cd my-project
# Afegir un fitxer README
echo "# My Project" > README.md
# Afegir fitxers al repositori
git add .
# Fer commit inicial
git commit -m "Initial commit"
# Connectar amb el repositori remot
git remote add origin https://github.com/usuari/my-project.git
# Empènyer canvis a la branca principal
git push -u origin main

2. Configuració del Servidor d'Integració Contínua (CI)

  • Configurar un servidor CI com Jenkins.
  • Crear un fitxer de configuració per al pipeline (per exemple, Jenkinsfile).
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Comandes per construir el projecte
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Comandes per executar proves
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Comandes per desplegar el projecte
            }
        }
    }
}

3. Automatització de Proves

  • Escriure proves unitàries i d'integració.
  • Configurar el servidor CI per executar aquestes proves automàticament.
# Exemple de prova unitaria en Python
import unittest

class TestMyFunction(unittest.TestCase):
    def test_addition(self):
        self.assertEqual(1 + 1, 2)

if __name__ == '__main__':
    unittest.main()

4. Desplegament Automàtic

  • Configurar scripts de desplegament.
  • Utilitzar eines de gestió de configuració per automatitzar el desplegament.
# Exemple de playbook d'Ansible per desplegar una aplicació
- hosts: webservers
  tasks:
    - name: Ensure the latest version of the application is installed
      git:
        repo: 'https://github.com/usuari/my-project.git'
        dest: /var/www/my-project
        version: main
    - name: Restart web server
      service:
        name: apache2
        state: restarted

Conclusió

La implementació d'un pipeline de CD és essencial per assegurar el lliurament continu i eficient de programari. En automatitzar els passos des de la integració fins al desplegament, es minimitzen els errors humans i s'accelera el cicle de desenvolupament. En aquesta secció, hem cobert els components clau i els passos necessaris per configurar un pipeline de CD bàsic. Amb aquesta base, els professionals de TI poden construir pipelines més complexos i adaptats a les seves necessitats específiques.
