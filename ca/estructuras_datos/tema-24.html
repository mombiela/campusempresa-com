Introducció als Arbres B

Els Arbres B són una estructura de dades d'arbre autoequilibrat que manté dades ordenades i permet cerques, insercions, eliminacions i accessos seqüencials en temps logarítmic. Són especialment útils en sistemes de bases de dades i sistemes de fitxers.

Característiques Clau dels Arbres B

  • Autoequilibrat: Els Arbres B mantenen el seu equilibri automàticament durant les operacions d'inserció i eliminació.
  • Nodes amb múltiples fills: A diferència dels arbres binaris, els nodes en un Arbre B poden tenir més de dos fills.
  • Alçada baixa: L'alçada d'un Arbre B és baixa en comparació amb altres arbres, cosa que permet un accés més ràpid a les dades.
  • Ús eficient de l'emmagatzematge: Els Arbres B estan dissenyats per minimitzar el nombre d'accessos a disc, cosa que els fa ideals per a sistemes d'emmagatzematge en disc.

Estructura d'un Arbre B

Un Arbre B d'ordre m té les següents propietats:

  • Cada node pot tenir com a màxim m fills.
  • Cada node intern (excepte l'arrel) té almenys ⌈m/2⌉ fills.
  • L'arrel té almenys dos fills si no és una fulla.
  • Tots els nodes fulla estan al mateix nivell.
  • Un node amb k fills conté k-1 claus.

Exemple d'un Arbre B d'Ordre 3

Considerem un Arbre B d'ordre 3 (també conegut com un 2-3 arbre):

        [10 | 20]
       /    |    \
  [5 | 7]  [15]  [25 | 30]

En aquest exemple:

  • L'arrel té 2 claus i 3 fills.
  • Cada node fulla té entre 1 i 2 claus.

Operacions en Arbres B

Inserció

La inserció en un Arbre B implica trobar la posició correcta per a la nova clau i, si és necessari, dividir els nodes que excedeixen el nombre màxim de claus permeses.

Exemple d'Inserció

Suposem que volem inserir la clau 12 en el següent Arbre B d'ordre 3:

        [10 | 20]
       /    |    \
  [5 | 7]  [15]  [25 | 30]
  1. Troba la fulla adequada per a la clau 12, que és el node [15].
  2. Insereix 12 en el node [15], resultant en [12 | 15].
  3. Si el node excedeix el nombre màxim de claus, divideix-lo i ajusta l'arbre.

Eliminació

L'eliminació en un Arbre B pot ser més complexa que la inserció, ja que pot requerir la redistribució de claus o la fusió de nodes per mantenir les propietats de l'arbre.

Exemple d'Eliminació

Suposem que volem eliminar la clau 20 del següent Arbre B d'ordre 3:

        [10 | 20]
       /    |    \
  [5 | 7]  [15]  [25 | 30]
  1. Troba la clau 20 i elimina-la.
  2. Si l'eliminació deixa un node amb menys de ⌈m/2⌉ claus, redistribueix o fusiona nodes per mantenir l'equilibri.

Exemple de Codi en Python

A continuació, es presenta un exemple bàsic d'implementació d'un Arbre B en Python:

class BTreeNode:
    def __init__(self, t, leaf=False):
        self.t = t  # Grau mínim (ordre)
        self.leaf = leaf  # True si el node és fulla
        self.keys = []  # Llista de claus
        self.children = []  # Llista de fills

class BTree:
    def __init__(self, t):
        self.root = BTreeNode(t, True)
        self.t = t

    def traverse(self, node):
        for i in range(len(node.keys)):
            if not node.leaf:
                self.traverse(node.children[i])
            print(node.keys[i], end=" ")
        if not node.leaf:
            self.traverse(node.children[len(node.keys)])

    def insert(self, k):
        root = self.root
        if len(root.keys) == 2 * self.t - 1:
            temp = BTreeNode(self.t)
            self.root = temp
            temp.children.insert(0, root)
            self.split_child(temp, 0)
            self.insert_non_full(temp, k)
        else:
            self.insert_non_full(root, k)

    def split_child(self, parent, i):
        t = self.t
        y = parent.children[i]
        z = BTreeNode(t, y.leaf)
        parent.children.insert(i + 1, z)
        parent.keys.insert(i, y.keys[t - 1])
        z.keys = y.keys[t:(2 * t - 1)]
        y.keys = y.keys[0:(t - 1)]
        if not y.leaf:
            z.children = y.children[t:(2 * t)]
            y.children = y.children[0:t]

    def insert_non_full(self, node, k):
        i = len(node.keys) - 1
        if node.leaf:
            node.keys.append(0)
            while i >= 0 and k < node.keys[i]:
                node.keys[i + 1] = node.keys[i]
                i -= 1
            node.keys[i + 1] = k
        else:
            while i >= 0 and k < node.keys[i]:
                i -= 1
            i += 1
            if len(node.children[i].keys) == 2 * self.t - 1:
                self.split_child(node, i)
                if k > node.keys[i]:
                    i += 1
            self.insert_non_full(node.children[i], k)

# Exemple d'ús
b_tree = BTree(3)
keys = [10, 20, 5, 6, 12, 30, 7, 17]
for key in keys:
    b_tree.insert(key)

b_tree.traverse(b_tree.root)

Conclusió

Els Arbres B són una estructura de dades fonamental en la informàtica, especialment en l'àmbit de les bases de dades i sistemes de fitxers. La seva capacitat per mantenir l'equilibri i minimitzar els accessos a disc els fa extremadament eficients per a operacions de cerca, inserció i eliminació. Comprendre com funcionen i com implementar-los és essencial per a qualsevol professional de TI que treballi amb grans volums de dades.
