Introducció

En l'estudi de les estructures de dades, els grafs són una eina fonamental per modelar relacions complexes entre diferents entitats. Els grafs poden classificar-se en dos tipus principals: dirigits i no dirigits. En aquesta secció, explorarem les característiques, diferències i aplicacions d'aquests dos tipus de grafs.

Grafs No Dirigits

Un graf no dirigit és una estructura composta per un conjunt de nodes (o vèrtexs) i un conjunt d'arestes (o enllaços) que connecten parells de nodes. En un graf no dirigit, les arestes no tenen una direcció específica, la qual cosa significa que la relació entre els nodes és bidireccional.

Característiques dels Grafs No Dirigits

  • Simetria: Si existeix una aresta entre el node A i el node B, llavors també existeix una aresta entre el node B i el node A.
  • Representació: Es pot representar mitjançant una matriu d'adjacència simètrica o una llista d'adjacència.

Exemple de Graf No Dirigit

Considerem un graf no dirigit amb els nodes A, B, C i D, i les següents arestes: (A-B), (A-C), (B-D), (C-D).

Representació en Llista d'Adjacència

A: B, C
B: A, D
C: A, D
D: B, C

Representació en Matriu d'Adjacència

| | A | B | C | D | |---|---|---|---|---| | A | 0 | 1 | 1 | 0 | | B | 1 | 0 | 0 | 1 | | C | 1 | 0 | 0 | 1 | | D | 0 | 1 | 1 | 0 |

Exemple de Codi en Python

# Representació d'un graf no dirigit usant un diccionari (llista d'adjacència)
graf_no_dirigit = {
    'A': ['B', 'C'],
    'B': ['A', 'D'],
    'C': ['A', 'D'],
    'D': ['B', 'C']
}

# Imprimir el graf
for node, adjacents in graf_no_dirigit.items():
    print(f"{node}: {', '.join(adjacents)}")

Grafs Dirigits

Un graf dirigit, també conegut com a dígraf, és una estructura en la qual les arestes tenen una direcció específica, la qual cosa significa que la relació entre els nodes és unidireccional.

Característiques dels Grafs Dirigits

  • Direcció: Cada aresta té una direcció que va des d'un node origen fins a un node destí.
  • Representació: Es pot representar mitjançant una matriu d'adjacència no simètrica o una llista d'adjacència.

Exemple de Graf Dirigit

Considerem un graf dirigit amb els nodes A, B, C i D, i les següents arestes: (A->B), (A->C), (B->D), (C->D).

Representació en Llista d'Adjacència

A: B, C
B: D
C: D
D:

Representació en Matriu d'Adjacència

| | A | B | C | D | |---|---|---|---|---| | A | 0 | 1 | 1 | 0 | | B | 0 | 0 | 0 | 1 | | C | 0 | 0 | 0 | 1 | | D | 0 | 0 | 0 | 0 |

Exemple de Codi en Python

# Representació d'un graf dirigit usant un diccionari (llista d'adjacència)
graf_dirigit = {
    'A': ['B', 'C'],
    'B': ['D'],
    'C': ['D'],
    'D': []
}

# Imprimir el graf
for node, adjacents in graf_dirigit.items():
    print(f"{node}: {', '.join(adjacents)}")

Comparació entre Grafs Dirigits i No Dirigits

| Característica | Graf No Dirigit | Graf Dirigit | |----------------------|-----------------|---------------| | Simetria | Sí | No | | Direcció d'arestes | No | Sí | | Representació | Matriu/Ll. Adj. Simètrica | Matriu/Ll. Adj. No Simètrica | | Aplicacions | Xarxes socials, Mapes de carreteres | Fluxos de treball, Xarxes de comunicació |

Conclusió

Els grafs dirigits i no dirigits són eines poderoses per modelar diferents tipus de relacions i estructures en l'àmbit de les ciències de la computació i altres disciplines. Comprendre les diferències i característiques de cada tipus de graf és essencial per seleccionar l'estructura adequada segons el problema que es desitja resoldre. A través d'exemples i representacions, hem explorat com es poden implementar i utilitzar aquests grafs en la pràctica.
