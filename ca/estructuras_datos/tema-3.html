Introducció

Les estructures de dades són una part fonamental de la informàtica i la programació. Permeten organitzar i emmagatzemar dades de manera eficient, cosa que és crucial per al rendiment dels algorismes i aplicacions. En aquesta secció, explorarem els tipus més comuns d'estructures de dades, les seves característiques, avantatges i desavantatges, així com exemples pràctics del seu ús.

Estructures de Dades Lineals

Arrays (Matrius)

  • Descripció: Una col·lecció d'elements, tots del mateix tipus, emmagatzemats en ubicacions de memòria contigües.
  • Avantatges:
    • Accés ràpid als elements mitjançant índexs.
    • Fàcil d'implementar i usar.
  • Desavantatges:
    • Mida fixa.
    • Costós en termes de temps per a operacions d'inserció i eliminació.
# Exemple d'una matriu en Python
array = [1, 2, 3, 4, 5]
print(array[2])  # Sortida: 3

Linked Lists (Llistes Enllaçades)

  • Descripció: Una col·lecció de nodes on cada node conté una dada i una referència (enllaç) al següent node en la seqüència.
  • Avantatges:
    • Mida dinàmica.
    • Insercions i eliminacions eficients.
  • Desavantatges:
    • Accés lent als elements (cal recórrer la llista).
    • Major ús de memòria a causa dels punters.
# Exemple d'una llista enllaçada en Python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        last = self.head
        while last.next:
            last = last.next
        last.next = new_node

# Ús de la llista enllaçada
ll = LinkedList()
ll.append(1)
ll.append(2)
ll.append(3)

Stacks (Piles)

  • Descripció: Una col·lecció d'elements amb accés restringit, seguint el principi LIFO (Last In, First Out).
  • Operacions principals:
    • push: Afegir un element al cim.
    • pop: Eliminar l'element del cim.
  • Avantatges:
    • Simplicitat en la implementació.
    • Útil per a problemes de recursió i reversió.
  • Desavantatges:
    • Accés limitat als elements.
# Exemple d'una pila en Python
stack = []
stack.append(1)
stack.append(2)
stack.append(3)
print(stack.pop())  # Sortida: 3

Queues (Cues)

  • Descripció: Una col·lecció d'elements que segueix el principi FIFO (First In, First Out).
  • Operacions principals:
    • enqueue: Afegir un element al final.
    • dequeue: Eliminar l'element del davant.
  • Avantatges:
    • Útil per a la gestió de tasques i processos.
  • Desavantatges:
    • Accés limitat als elements.
# Exemple d'una cua en Python
from collections import deque

queue = deque()
queue.append(1)
queue.append(2)
queue.append(3)
print(queue.popleft())  # Sortida: 1

Estructures de Dades No Lineals

Trees (Arbres)

  • Descripció: Una estructura jeràrquica que consisteix en nodes, amb un node arrel i subnodes fills.
  • Tipus comuns:
    • Arbre Binari
    • Arbre Binari de Cerca (BST)
    • Arbre AVL
  • Avantatges:
    • Eficient per a operacions de cerca, inserció i eliminació.
  • Desavantatges:
    • Complexitat en la implementació i manteniment.
# Exemple d'un arbre binari en Python
class TreeNode:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

# Creació d'un arbre binari
root = TreeNode(1)
root.left = TreeNode(2)
root.right = TreeNode(3)

Graphs (Grafs)

  • Descripció: Una col·lecció de nodes (vèrtexs) i arestes (connexions) entre ells.
  • Tipus comuns:
    • Graf dirigit
    • Graf no dirigit
    • Graf ponderat
  • Avantatges:
    • Modela relacions complexes entre dades.
  • Desavantatges:
    • Complexitat en la representació i algorismes.
# Exemple d'un graf en Python usant un diccionari
graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'],
    'E': ['B', 'F'],
    'F': ['C', 'E']
}

Conclusió

Les estructures de dades són essencials per a l'eficiència i eficàcia dels algorismes i aplicacions. Cada tipus d'estructura de dades té els seus propis avantatges i desavantatges, i l'elecció de l'estructura adequada depèn del problema específic que s'estigui resolent. Comprendre aquestes estructures i les seves característiques és crucial per a qualsevol professional de TI que desitgi optimitzar el rendiment dels seus programes i sistemes.
