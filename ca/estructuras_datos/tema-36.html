Introducció

La selecció adequada d'estructures de dades és crucial per al rendiment i l'eficiència de les aplicacions. Aquest tema abordarà com triar l'estructura de dades correcta segons el problema que es desitja resoldre. Explorarem diferents escenaris i discutirem les estructures de dades més adequades per a cadascun.

Conceptes Clau

  • Eficiència de Temps i Espai: L'elecció d'una estructura de dades pot afectar significativament el temps d'execució i l'ús de memòria.
  • Operacions Comunes: Inserció, eliminació, cerca i accés són operacions que s'han de considerar en seleccionar una estructura de dades.
  • Escenaris d'Ús: Diferents problemes requereixen diferents estructures de dades per ser resolts de manera òptima.

Estructures de Dades Comunes i els seus Usos

Arrays

  • Avantatges:
    • Accés ràpid a elements mitjançant índexs.
    • Eficient en termes d'ús de memòria.
  • Desavantatges:
    • Mida fixa (en la majoria dels llenguatges).
    • Costós en termes de temps per a insercions i eliminacions.
  • Exemple d'Ús:
    • Problema: Necessites emmagatzemar una llista d'elements que no canviarà de mida.
    • Solució: Usar un array.
# Exemple d'ús d'un array en Python
array = [1, 2, 3, 4, 5]
print(array[2])  # Accés ràpid al tercer element

Llistes Enllaçades

  • Avantatges:
    • Mida dinàmica.
    • Insercions i eliminacions eficients.
  • Desavantatges:
    • Accés lent a elements (ha de recórrer la llista).
    • Major ús de memòria a causa dels punters.
  • Exemple d'Ús:
    • Problema: Necessites una col·lecció d'elements on les insercions i eliminacions són freqüents.
    • Solució: Usar una llista enllaçada.
# Exemple d'ús d'una llista enllaçada en Python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        last = self.head
        while last.next:
            last = last.next
        last.next = new_node

# Crear una llista enllaçada i afegir elements
ll = LinkedList()
ll.append(1)
ll.append(2)
ll.append(3)

Piles (Stacks)

  • Avantatges:
    • Operacions LIFO (Last In, First Out).
    • Insercions i eliminacions eficients.
  • Desavantatges:
    • Accés limitat (només a l'element superior).
  • Exemple d'Ús:
    • Problema: Necessites una estructura per manejar operacions de desfer/refer.
    • Solució: Usar una pila.
# Exemple d'ús d'una pila en Python
stack = []
stack.append(1)
stack.append(2)
stack.append(3)
print(stack.pop())  # Elimina i retorna l'últim element afegit (3)

Cues (Queues)

  • Avantatges:
    • Operacions FIFO (First In, First Out).
    • Insercions i eliminacions eficients.
  • Desavantatges:
    • Accés limitat (només al primer i últim element).
  • Exemple d'Ús:
    • Problema: Necessites una estructura per manejar tasques en ordre d'arribada.
    • Solució: Usar una cua.
# Exemple d'ús d'una cua en Python
from collections import deque

queue = deque()
queue.append(1)
queue.append(2)
queue.append(3)
print(queue.popleft())  # Elimina i retorna el primer element afegit (1)

Taules Hash (Hash Tables)

  • Avantatges:
    • Accés ràpid a elements mitjançant claus.
    • Eficient per a cerques, insercions i eliminacions.
  • Desavantatges:
    • Col·lisions de hash poden afectar el rendiment.
    • Ús de memòria pot ser alt.
  • Exemple d'Ús:
    • Problema: Necessites una estructura per emmagatzemar parells clau-valor amb accés ràpid.
    • Solució: Usar una taula hash.
# Exemple d'ús d'una taula hash en Python
hash_table = {}
hash_table['key1'] = 'value1'
hash_table['key2'] = 'value2'
print(hash_table['key1'])  # Accés ràpid al valor associat amb 'key1'

Comparació d'Estructures de Dades

| Estructura de Dades | Accés | Inserció | Eliminació | Ús de Memòria | Escenaris Comuns | |---------------------|-------|----------|------------|---------------|------------------| | Array | O(1) | O(n) | O(n) | Baix | Llistes estàtiques| | Llista Enllaçada | O(n) | O(1) | O(1) | Alt | Llistes dinàmiques| | Pila | O(n) | O(1) | O(1) | Baix | Desfer/refer | | Cua | O(n) | O(1) | O(1) | Baix | Tasques en ordre | | Taula Hash | O(1) | O(1) | O(1) | Mitjà | Parells clau-valor|

Conclusió

La selecció de l'estructura de dades adequada depèn del problema específic que es desitja resoldre i de les operacions que es realitzaran amb major freqüència. Comprendre els avantatges i desavantatges de cada estructura de dades permet prendre decisions informades que optimitzen el rendiment i l'eficiència de les aplicacions.
