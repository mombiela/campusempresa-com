Optimització del Rendiment en TypeScript
Manteniment de la seguretat de tipus ►

Introducció

L'optimització del rendiment és un aspecte crucial del desenvolupament de programari, assegurant que les teves aplicacions TypeScript funcionin de manera eficient i efectiva. Aquesta secció cobrirà diverses tècniques i millors pràctiques per optimitzar el codi TypeScript, des de nivells bàsics fins a avançats.

Conceptes Clau

  • Eficiència del Codi: Escriure codi que funcioni bé i utilitzi els recursos de manera sàvia.
  • Gestió de Memòria: Ús eficient de la memòria per evitar fuites i consum excessiu.
  • Programació Asíncrona: Aprofitar les operacions asíncrones per millorar el rendiment.
  • Perfilat i Benchmarking: Eines i tècniques per mesurar i millorar el rendiment.

Eficiència del Codi

Evitar Càlculs Innecessaris

Minimitza els càlculs redundants emmagatzemant els resultats en variables.

// Ineficient
for (let i = 0; i < array.length; i++) {
  console.log(array.length); // array.length es calcula en cada iteració
}

// Eficient
const arrayLength = array.length;
for (let i = 0; i < arrayLength; i++) {
  console.log(arrayLength); // array.length es calcula una vegada
}

Utilitzar Estructures de Dades Apropiades

Escull l'estructura de dades adequada per a les teves necessitats per millorar el rendiment.

// Ineficient: Utilitzar una matriu per a parells clau-valor
const data = [['key1', 'value1'], ['key2', 'value2']];
const value = data.find(item => item[0] === 'key1')[1];

// Eficient: Utilitzar un Map per a parells clau-valor
const dataMap = new Map<string, string>([['key1', 'value1'], ['key2', 'value2']]);
const value = dataMap.get('key1');

Gestió de Memòria

Evitar Fuites de Memòria

Assegura't que els objectes es desreferencien correctament quan ja no es necessiten.

// Potencial fuita de memòria
let cache: { [key: string]: any } = {};
function addToCache(key: string, value: any) {
  cache[key] = value;
}

// Neteja correcta de la memòria cau
function clearCache() {
  cache = {};
}

Utilitzar Referències Febles

Utilitza WeakMap o WeakSet per a objectes que no haurien d'evitar la recollida d'escombraries.

const weakMap = new WeakMap<object, string>();
let obj = {};
weakMap.set(obj, 'some value');

// obj pot ser recollit com a escombraries quan ja no s'utilitza
obj = null;

Programació Asíncrona

Utilitzar Promeses i Async/Await

Aprofita les operacions asíncrones per evitar bloquejar el fil principal.

// Ineficient: Bloquejant el fil principal
function fetchData() {
  const response = fetch('https://api.example.com/data');
  const data = response.json();
  console.log(data);
}

// Eficient: Utilitzant async/await
async function fetchData() {
  const response = await fetch('https://api.example.com/data');
  const data = await response.json();
  console.log(data);
}

Debouncing i Throttling

Controla la velocitat d'execució de la funció per millorar el rendiment.

// Debouncing: Executar la funció després d'un retard
function debounce(func: Function, delay: number) {
  let timeoutId: number;
  return function(...args: any[]) {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => func(...args), delay);
  };
}

// Throttling: Executar la funció com a màxim una vegada en un període especificat
function throttle(func: Function, limit: number) {
  let lastFunc: number;
  let lastRan: number;
  return function(...args: any[]) {
    if (!lastRan) {
      func(...args);
      lastRan = Date.now();
    } else {
      clearTimeout(lastFunc);
      lastFunc = setTimeout(() => {
        if (Date.now() - lastRan >= limit) {
          func(...args);
          lastRan = Date.now();
        }
      }, limit - (Date.now() - lastRan));
    }
  };
}

Perfilat i Benchmarking

Utilitzar Eines de Rendiment

Utilitza les eines de desenvolupador del navegador i els hooks de rendiment de Node.js per perfilar i fer benchmarking del teu codi.

// Navegador: Utilitzant console.time i console.timeEnd
console.time('fetchData');
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.timeEnd('fetchData');
    console.log(data);
  });

// Node.js: Utilitzant hooks de rendiment
const { performance } = require('perf_hooks');
performance.mark('A');
setTimeout(() => {
  performance.mark('B');
  performance.measure('A to B', 'A', 'B');
  const measure = performance.getEntriesByName('A to B')[0];
  console.log(measure.duration);
}, 1000);

Conclusió

L'optimització del rendiment en TypeScript implica escriure codi eficient, gestionar la memòria de manera efectiva, aprofitar la programació asíncrona i utilitzar eines de perfilat per mesurar i millorar el rendiment. Seguint aquestes millors pràctiques, pots assegurar-te que les teves aplicacions TypeScript funcionin de manera suau i eficient.
