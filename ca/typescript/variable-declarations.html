Declaracions de Variables
Tipus primitius ►

Introducció

Les declaracions de variables són un concepte fonamental en TypeScript, com ho són en qualsevol llenguatge de programació. Entendre com declarar i utilitzar variables de manera efectiva és crucial per escriure codi net, eficient i lliure d'errors.

Declaracions Bàsiques de Variables

En TypeScript, pots declarar variables utilitzant tres paraules clau: var, let i const.

`var`

  • La paraula clau var s'utilitza per declarar una variable que es pot reassignar.
  • Les variables declarades amb var tenen àmbit de funció o àmbit global, no àmbit de bloc.
var message = "Hola, Món!";
console.log(message); // Sortida: Hola, Món!

if (true) {
    var count = 10;
}
console.log(count); // Sortida: 10 (accessible fora del bloc)

`let`

  • La paraula clau let s'utilitza per declarar una variable amb àmbit de bloc que es pot reassignar.
  • Es prefereix sobre var a causa del seu comportament d'àmbit de bloc.
let message = "Hola, TypeScript!";
console.log(message); // Sortida: Hola, TypeScript!

if (true) {
    let count = 10;
    console.log(count); // Sortida: 10 (accessible dins del bloc)
}
console.log(count); // Error: count no està definit (no accessible fora del bloc)

`const`

  • La paraula clau const s'utilitza per declarar una variable amb àmbit de bloc que no es pot reassignar.
  • S'utilitza per a variables que han de romandre constants durant tot el programa.
const message = "Hola, TypeScript!";
console.log(message); // Sortida: Hola, TypeScript!

message = "Nou Missatge"; // Error: Assignació a una variable constant.

Anotacions de Tipus

TypeScript et permet especificar el tipus d'una variable utilitzant anotacions de tipus. Això ajuda a detectar errors en temps de compilació i fa que el codi sigui més llegible.

let message: string = "Hola, TypeScript!";
let count: number = 10;
let isActive: boolean = true;

Inferència de Tipus

TypeScript pot inferir automàticament el tipus d'una variable basant-se en el valor assignat. Això significa que no sempre cal especificar explícitament el tipus.

let message = "Hola, TypeScript!"; // TypeScript infereix que message és de tipus string
let count = 10; // TypeScript infereix que count és de tipus number
let isActive = true; // TypeScript infereix que isActive és de tipus boolean

Àmbit de Variables

Entendre l'àmbit de les variables és crucial per escriure codi mantenible. L'àmbit determina la visibilitat i la vida útil d'una variable.

Àmbit Global

  • Les variables declarades fora de qualsevol funció o bloc estan en l'àmbit global.
  • Es poden accedir des de qualsevol lloc del codi.
var globalMessage = "Sóc global";

function showMessage() {
    console.log(globalMessage); // Sortida: Sóc global
}

Àmbit de Funció

  • Les variables declarades dins d'una funció estan en l'àmbit de funció.
  • Només es poden accedir dins d'aquesta funció.
function showMessage() {
    var localMessage = "Sóc local";
    console.log(localMessage); // Sortida: Sóc local
}
console.log(localMessage); // Error: localMessage no està definit

Àmbit de Bloc

  • Les variables declarades amb let o const dins d'un bloc (per exemple, dins d'una instrucció if o un bucle) estan en l'àmbit de bloc.
  • Només es poden accedir dins d'aquest bloc.
if (true) {
    let blockMessage = "Sóc d'àmbit de bloc";
    console.log(blockMessage); // Sortida: Sóc d'àmbit de bloc
}
console.log(blockMessage); // Error: blockMessage no està definit

Resum

  • Utilitza var per a variables amb àmbit de funció o àmbit global, però prefereix let i const per a variables amb àmbit de bloc.
  • Utilitza let per a variables que es poden reassignar i const per a variables que han de romandre constants.
  • Les anotacions de tipus ajuden a detectar errors i fan que el codi sigui més llegible, però TypeScript també pot inferir tipus.
  • Entendre l'àmbit de les variables (global, funció i bloc) és essencial per escriure codi mantenible.

Dominant les declaracions de variables en TypeScript, estableixes una base sòlida per escriure aplicacions TypeScript robustes i eficients.
