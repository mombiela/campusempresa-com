Introducció

L'estil i el format del codi són crucials per mantenir la llegibilitat, la consistència i la col·laboració en projectes de TypeScript. Aquesta secció cobrirà les millors pràctiques, eines i directrius per ajudar-te a escriure codi de TypeScript net i mantenible.

Conceptes Clau

  • Consistència: Assegurar que el teu codi segueixi un estil consistent a tot el projecte.
  • Llegibilitat: Escriure codi que sigui fàcil de llegir i entendre.
  • Eines: Utilitzar eines per automatitzar el format del codi i fer complir les directrius d'estil.

Millors Pràctiques

Convencions de Nomenclatura

  • Variables i Funcions: Utilitza camelCase. 
    let userName: string;
function getUserInfo() {}
  • Classes i Interfícies: Utilitza PascalCase. 
    class UserProfile {}
interface User {
  name: string;
  age: number;
}
  • Constants: Utilitza UPPER_CASE amb guions baixos. 
    const MAX_USERS = 100;

Indentació i Espaiat

  • Utilitza 2 o 4 espais per a la indentació (tria un i mantingues-lo).
  • Afegeix espais al voltant dels operadors i després de les comes. 
    let sum = a + b;
let numbers = [1, 2, 3];

Punt i Coma

  • Utilitza sempre punt i coma per acabar les instruccions. 
    let user = 'John';
console.log(user);

Longitud de la Línia

  • Mantingues les línies per sota de 80 o 100 caràcters per millorar la llegibilitat.

Claus i Blocs

  • Utilitza claus per a totes les estructures de control, fins i tot blocs d'una sola línia. 
    if (isValid) {
  console.log('Valid');
} else {
  console.log('Invalid');
}

Eines

Prettier

Prettier és un formatejador de codi opinat que suporta TypeScript. Fa complir un estil consistent analitzant el teu codi i reimprimint-lo amb les seves pròpies regles.

  • Instal·lació: 
    npm install --save-dev prettier
  • Configuració: Crea un fitxer .prettierrc a l'arrel del teu projecte. 
    {
  "semi": true,
  "singleQuote": true,
  "trailingComma": "all",
  "tabWidth": 2
}
  • Ús: 
    npx prettier --write src/

ESLint

ESLint és una eina d'anàlisi de codi estàtic per identificar patrons problemàtics en codi JavaScript i TypeScript.

  • Instal·lació: 
    npm install --save-dev eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
  • Configuració: Crea un fitxer .eslintrc.json. 
    {
  "parser": "@typescript-eslint/parser",
  "plugins": ["@typescript-eslint"],
  "extends": ["eslint:recommended", "plugin:@typescript-eslint/recommended"],
  "rules": {
    "semi": ["error", "always"],
    "quotes": ["error", "single"]
  }
}
  • Ús: 
    npx eslint src/**/*.ts

Exemple

Aquí tens un exemple d'un fitxer TypeScript ben formatat:

// Importació de mòduls necessaris
import { User } from './models/User';

// Constants
const MAX_USERS = 100;

// Definició de la classe
class UserProfile {
  private user: User;

  constructor(user: User) {
    this.user = user;
  }

  public getUserInfo(): string {
    return `Name: ${this.user.name}, Age: ${this.user.age}`;
  }
}

// Funció per crear un nou perfil d'usuari
function createUserProfile(name: string, age: number): UserProfile {
  const user: User = { name, age };
  return new UserProfile(user);
}

// Exemple d'ús
const userProfile = createUserProfile('John Doe', 30);
console.log(userProfile.getUserInfo());

Conclusió

Adherir-se a un estil de codi consistent i a les directrius de format és essencial per escriure codi de TypeScript net i mantenible. Seguint les millors pràctiques i aprofitant eines com Prettier i ESLint, pots assegurar-te que la teva base de codi es mantingui llegible i fàcil de treballar, tant per a tu com per als altres.
