Desplegar Aplicacions TypeScript

Desplegar aplicacions TypeScript implica compilar el codi TypeScript a JavaScript i assegurar-se que l'aplicació funcioni sense problemes en l'entorn de destinació. Aquesta secció et guiarà a través del procés de preparar, construir i desplegar aplicacions TypeScript.

Conceptes Clau

  • Compilació: Transformar el codi TypeScript en JavaScript.
  • Agrupació: Combinar múltiples fitxers JavaScript en un sol fitxer o uns pocs fitxers.
  • Transpilació: Convertir TypeScript a JavaScript que sigui compatible amb navegadors més antics.
  • Desplegament: Moure l'aplicació a un entorn de producció.

Configuració del Projecte

Abans de desplegar, assegura't que el teu projecte TypeScript estigui correctament configurat.

  1. Inicialitzar un Projecte TypeScript:

    npm init -y
npm install typescript --save-dev
npx tsc --init

  2. Configurar tsconfig.json:

    {
  "compilerOptions": {
    "target": "es5",
    "module": "commonjs",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "**/*.spec.ts"]
}

Compilar TypeScript a JavaScript

  1. Compilar el Codi:

    npx tsc

    Aquesta comanda compilarà tots els fitxers TypeScript al directori src i generarà fitxers JavaScript al directori dist.

  2. Verificar la Sortida: Assegura't que el directori dist contingui els fitxers JavaScript compilats.

Agrupar l'Aplicació

Agrupar ajuda a reduir el nombre de sol·licituds HTTP i optimitzar el temps de càrrega de la teva aplicació.

  1. Instal·lar Webpack:

    npm install webpack webpack-cli ts-loader --save-dev

  2. Configurar Webpack: Crear un fitxer webpack.config.js:

    const path = require('path');

module.exports = {
  entry: './src/index.ts',
  module: {
    rules: [
      {
        test: /\.ts$/,
        use: 'ts-loader',
        exclude: /node_modules/
      }
    ]
  },
  resolve: {
    extensions: ['.ts', '.js']
  },
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  }
};

  3. Construir l'Aplicació:

    npx webpack --mode production

Transpilar per a Navegadors Més Antics

Per assegurar la compatibilitat amb navegadors més antics, potser necessitis transpilar el teu codi JavaScript.

  1. Instal·lar Babel:

    npm install @babel/core @babel/preset-env babel-loader --save-dev

  2. Configurar Babel: Actualitza webpack.config.js:

    const path = require('path');

module.exports = {
  entry: './src/index.ts',
  module: {
    rules: [
      {
        test: /\.ts$/,
        use: 'ts-loader',
        exclude: /node_modules/
      },
      {
        test: /\.js$/,
        use: 'babel-loader',
        exclude: /node_modules/
      }
    ]
  },
  resolve: {
    extensions: ['.ts', '.js']
  },
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  }
};

  3. Crear .babelrc:

    {
  "presets": ["@babel/preset-env"]
}

  4. Construir l'Aplicació:

    npx webpack --mode production

Desplegar a un Servidor

  1. Preparar el Paquet de Desplegament: Assegura't que el directori dist contingui tots els fitxers necessaris.

  2. Triar un Servei d'Allotjament:

    • Allotjament Estàtic: Serveis com GitHub Pages, Netlify o Vercel.
    • Allotjament de Servidor: Serveis com Heroku, AWS o DigitalOcean.

  3. Desplegar l'Aplicació:

    • Per a Allotjament Estàtic:
      • Segueix les instruccions específiques del servei d'allotjament per desplegar el directori dist.
    • Per a Allotjament de Servidor:
      • Puja el directori dist al teu servidor.
      • Configura el servidor per servir el fitxer index.html des del directori dist.

Conclusió

Desplegar aplicacions TypeScript implica diversos passos, incloent compilar TypeScript a JavaScript, agrupar l'aplicació, assegurar la compatibilitat amb navegadors més antics i finalment desplegar a un servei d'allotjament. Seguint aquests passos, pots assegurar-te que la teva aplicació TypeScript estigui llesta per a producció i pugui ser accedida pels usuaris sense problemes.
