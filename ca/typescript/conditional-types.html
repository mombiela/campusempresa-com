Tipus condicionals a TypeScript
Tipus mapats ►

Introducció

Els tipus condicionals a TypeScript et permeten crear tipus que depenen d'una condició. Són una característica poderosa que pot fer que les teves definicions de tipus siguin més flexibles i expressives. Aquest tema et guiarà des dels conceptes bàsics fins a l'ús avançat dels tipus condicionals.

Sintaxi bàsica

Els tipus condicionals segueixen una sintaxi simple:

T extends U ? X : Y
  • T extends U és la condició.
  • Si la condició és certa, el tipus es resol a X.
  • Si la condició és falsa, el tipus es resol a Y.

Exemple

Comencem amb un exemple bàsic:

type IsString<T> = T extends string ? "Yes" : "No";

type A = IsString<string>; // "Yes"
type B = IsString<number>; // "No"

En aquest exemple:

  • IsString<string> s'avalua com a "Yes" perquè string s'estén de string.
  • IsString<number> s'avalua com a "No" perquè number no s'estén de string.

Tipus condicionals amb genèrics

Els tipus condicionals es tornen més poderosos quan s'utilitzen amb genèrics.

Exemple

Considera una funció que retorna diferents tipus basats en una condició:

type ReturnType<T> = T extends () => infer R ? R : never;

type Func = () => string;
type Result = ReturnType<Func>; // string

Aquí:

  • ReturnType<T> comprova si T és un tipus de funció.
  • Si és cert, extreu el tipus de retorn R utilitzant infer.
  • Si és fals, es resol a never.

Tipus condicionals distributius

Els tipus condicionals es distribueixen sobre unions. Això significa que si apliques un tipus condicional a un tipus d'unió, TypeScript aplicarà el tipus condicional a cada membre de la unió individualment.

Exemple

type ToArray<T> = T extends any ? T[] : never;

type StrArray = ToArray<string>; // string[]
type NumArray = ToArray<number>; // number[]
type UnionArray = ToArray<string | number>; // string[] | number[]

En aquest exemple:

  • ToArray<string | number> es distribueix sobre la unió, resultant en string[] | number[].

Ús avançat

Els tipus condicionals es poden combinar amb altres característiques de TypeScript per crear definicions de tipus complexes i poderoses.

Exemple: Filtrant tipus

Pots utilitzar tipus condicionals per filtrar certs tipus d'una unió.

type Exclude<T, U> = T extends U ? never : T;

type T1 = Exclude<"a" | "b" | "c", "a">; // "b" | "c"
type T2 = Exclude<string | number | (() => void), Function>; // string | number

Aquí:

  • Exclude<T, U> elimina tipus de T que són assignables a U.

Aplicació pràctica

Els tipus condicionals s'utilitzen sovint en projectes reals de TypeScript per crear tipus més precisos i flexibles.

Exemple: Deep Partial

Un tipus d'utilitat comú és DeepPartial, que fa que totes les propietats d'un tipus siguin opcionals, de manera recursiva.

type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface User {
  name: string;
  address: {
    street: string;
    city: string;
  };
}

type PartialUser = DeepPartial<User>;
/*
PartialUser és:
{
  name?: string;
  address?: {
    street?: string;
    city?: string;
  };
}
*/

En aquest exemple:

  • DeepPartial<T> utilitza un tipus condicional per comprovar si una propietat és un objecte.
  • Si ho és, aplica DeepPartial de manera recursiva.

Conclusió

Els tipus condicionals a TypeScript proporcionen una manera poderosa de crear definicions de tipus flexibles i expressives. Et permeten definir tipus que depenen de condicions, treballar amb genèrics i fins i tot distribuir-se sobre unions. En dominar els tipus condicionals, podràs escriure codi TypeScript més robust i mantenible.
