Sobrecàrregues de Funcions en TypeScript
Funcions fletxa ►

La sobrecàrrega de funcions és una característica de TypeScript que et permet definir múltiples signatures per a una sola funció. Això pot ser particularment útil quan vols que una funció gestioni diferents tipus de paràmetres o retorni diferents tipus de resultats segons l'entrada.

Conceptes Clau

  • Signatura de Funció: La combinació del nom d'una funció i els seus tipus de paràmetres.
  • Signatura de Sobrecàrrega: Diferents versions d'una signatura de funció que una funció pot acceptar.
  • Signatura d'Implementació: La implementació real de la funció que gestiona totes les sobrecàrregues.

Per què Utilitzar Sobrecàrregues de Funcions?

  • Flexibilitat: Permet que una sola funció gestioni diferents tipus d'entrades.
  • Llegibilitat: Fa que el codi sigui més fàcil d'entendre definint clarament quins tipus d'entrades pot acceptar una funció.
  • Seguretat de Tipus: Assegura que la funció s'utilitzi correctament segons les seves signatures definides.

Sintaxi Bàsica

En TypeScript, les sobrecàrregues de funcions es defineixen escrivint múltiples signatures de funció seguides d'una sola implementació.

// Signatures de sobrecàrrega
function add(a: number, b: number): number;
function add(a: string, b: string): string;

// Signatura d'implementació
function add(a: any, b: any): any {
    return a + b;
}

// Ús
console.log(add(1, 2)); // Sortida: 3
console.log(add("Hello, ", "world!")); // Sortida: Hello, world!

Exemple Detallat

Creem un exemple més detallat per entendre millor les sobrecàrregues de funcions. Suposem que volem crear una funció que pugui concatenar dues cadenes o sumar dos números.

// Signatures de sobrecàrrega
function combine(a: number, b: number): number;
function combine(a: string, b: string): string;

// Signatura d'implementació
function combine(a: any, b: any): any {
    if (typeof a === "number" && typeof b === "number") {
        return a + b;
    } else if (typeof a === "string" && typeof b === "string") {
        return a + b;
    }
    throw new Error("Arguments invàlids. Ambdós arguments han de ser del mateix tipus.");
}

// Ús
console.log(combine(10, 20)); // Sortida: 30
console.log(combine("foo", "bar")); // Sortida: foobar

Exemple Avançat amb Paràmetres Opcional

Les sobrecàrregues de funcions també poden gestionar paràmetres opcionals. Creem una funció que pugui retornar el nom complet d'una persona o només el primer nom si no es proporciona el cognom.

// Signatures de sobrecàrrega
function getName(firstName: string, lastName: string): string;
function getName(firstName: string): string;

// Signatura d'implementació
function getName(firstName: string, lastName?: string): string {
    if (lastName) {
        return `${firstName} ${lastName}`;
    } else {
        return firstName;
    }
}

// Ús
console.log(getName("John", "Doe")); // Sortida: John Doe
console.log(getName("John")); // Sortida: John

Taula Comparativa

| Característica | Descripció | |---------------------|-----------------------------------------------------------------------------| | Signatures de Sobrecàrrega | Múltiples signatures de funció que defineixen diferents maneres de cridar la funció.| | Implementació | La funció real que gestiona totes les sobrecàrregues. | | Seguretat de Tipus | Assegura que la funció es cridi amb els tipus correctes d'arguments. | | Flexibilitat | Permet que una sola funció gestioni diferents tipus d'entrades. |

Conclusió

Les sobrecàrregues de funcions en TypeScript proporcionen una manera poderosa de crear funcions flexibles i segures en termes de tipus. Definint múltiples signatures per a una sola funció, pots gestionar diferents tipus d'entrades i assegurar-te que la teva funció s'utilitzi correctament. Aquesta característica millora la llegibilitat i mantenibilitat del codi, fent més fàcil treballar amb funcions complexes.

Entenent i utilitzant les sobrecàrregues de funcions, pots escriure codi TypeScript més robust i versàtil, atenent a una varietat de casos d'ús amb una sola funció ben definida.
