Proves de Cap a Cap en TypeScript
Opcions del compilador de TypeScript ►

Les proves de cap a cap (E2E) són una part crucial del cicle de vida del desenvolupament de programari, assegurant que tot el flux de l'aplicació funcioni com s'espera. Aquest tema et guiarà a través dels conceptes bàsics de les proves E2E en TypeScript, des de la configuració del teu entorn fins a escriure i executar les teves primeres proves.

Què són les Proves de Cap a Cap?

Les proves de cap a cap impliquen provar la funcionalitat completa d'una aplicació, des de la interfície d'usuari fins al backend i la base de dades. Simulen escenaris reals d'usuari per validar el comportament del sistema.

  • Propòsit: Assegurar que l'aplicació funcioni com un tot.
  • Abast: Cobreix tota l'aplicació, incloent-hi la UI, APIs, bases de dades i altres serveis integrats.
  • Eines: Eines comunes inclouen Cypress, Selenium i Playwright.

Configurant el Teu Entorn

Abans d'escriure proves E2E, necessites configurar el teu entorn de proves. Per a aquesta guia, utilitzarem Cypress, un marc de proves E2E popular.

Instal·lant Cypress

Primer, instal·la Cypress via npm:

npm install cypress --save-dev

Configurant Cypress

Després de la instal·lació, necessites configurar Cypress. Crea un fitxer cypress.json a l'arrel del teu projecte:

{
  "baseUrl": "http://localhost:3000",
  "integrationFolder": "cypress/integration"
}
  • baseUrl: La URL on la teva aplicació està executant-se.
  • integrationFolder: La carpeta on es trobaran els teus fitxers de prova.

Escriure la Teva Primera Prova E2E

Escrivim una prova E2E simple per comprovar si la pàgina d'inici es carrega correctament.

Creant un Fitxer de Prova

Crea un nou fitxer anomenat home.spec.ts a la carpeta cypress/integration:

describe('Homepage', () => {
  it('should load the homepage', () => {
    cy.visit('/');
    cy.contains('Welcome to My App');
  });
});

Explicació

  • describe: Agrupa proves relacionades.
  • it: Defineix un cas de prova únic.
  • cy.visit('/'): Navega a la pàgina d'inici.
  • cy.contains('Welcome to My App'): Comprova si el text "Welcome to My App" és present a la pàgina.

Executant les Teves Proves

Per executar les teves proves, utilitza la següent comanda:

npx cypress open

Això obrirà el Cypress Test Runner, on podràs seleccionar i executar les teves proves.

Conceptes Avançats de Proves E2E

A mesura que et sentis més còmode amb les proves E2E, pots explorar conceptes més avançats.

Gestionant l'Autenticació

Moltes aplicacions requereixen autenticació. Pots gestionar això en Cypress utilitzant comandes personalitzades.

Exemple: Comanda Personalitzada per a l'Inici de Sessió

Crea un nou fitxer anomenat commands.ts a la carpeta cypress/support:

Cypress.Commands.add('login', (username, password) => {
  cy.request('POST', '/api/login', {
    username,
    password
  }).then((response) => {
    window.localStorage.setItem('authToken', response.body.token);
  });
});

Utilitza aquesta comanda personalitzada en les teves proves:

describe('Dashboard', () => {
  beforeEach(() => {
    cy.login('testuser', 'password123');
  });

  it('should load the dashboard', () => {
    cy.visit('/dashboard');
    cy.contains('Dashboard');
  });
});

Simulant Peticions API

Per aïllar les proves del frontend de les dependències del backend, pots simular peticions API.

Exemple: Simulant una Petició API

describe('User Profile', () => {
  beforeEach(() => {
    cy.intercept('GET', '/api/user', {
      statusCode: 200,
      body: { name: 'John Doe', email: 'john.doe@example.com' }
    }).as('getUser');
  });

  it('should display user profile', () => {
    cy.visit('/profile');
    cy.wait('@getUser');
    cy.contains('John Doe');
    cy.contains('john.doe@example.com');
  });
});

Conclusió

Les proves de cap a cap són essencials per assegurar la fiabilitat i funcionalitat de la teva aplicació. Utilitzant eines com Cypress i escrivint proves completes, pots detectar errors aviat i millorar la qualitat general del teu programari. A mesura que avancis, explora escenaris més complexos i integra les proves E2E en la teva pipeline d'integració contínua per a proves automatitzades.
