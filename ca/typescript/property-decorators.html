Decoradors de Propietats en TypeScript
Decoradors de paràmetre ►

Els decoradors de propietats són una característica poderosa en TypeScript que et permet modificar o anotar propietats dins d'una classe. Proporcionen una manera d'afegir metadades o canviar el comportament de les propietats d'una manera declarativa. Aquest tema et guiarà des dels conceptes bàsics fins a l'ús avançat dels decoradors de propietats.

Introducció als Decoradors de Propietats

  • Definició: Els decoradors de propietats són funcions que s'apliquen a les propietats d'una classe.
  • Propòsit: Es poden utilitzar per afegir metadades, realitzar validacions o modificar el comportament de les propietats.

Sintaxi Bàsica

Un decorador de propietat és una funció que pren dos arguments:

  1. L'objectiu (el prototip de la classe per a propietats d'instància o la funció constructora per a propietats estàtiques).
  2. El nom de la propietat.
function MyPropertyDecorator(target: any, propertyKey: string) {
    console.log(`PropertyDecorator called on: ${propertyKey}`);
}

class ExampleClass {
    @MyPropertyDecorator
    myProperty: string;
}

En aquest exemple:

  • MyPropertyDecorator és un decorador senzill que registra el nom de la propietat a la qual s'aplica.
  • @MyPropertyDecorator s'utilitza per anotar myProperty en ExampleClass.

Casos Pràctics

Afegint Metadades

Els decoradors de propietats es poden utilitzar per afegir metadades a les propietats, cosa que pot ser útil per a diversos propòsits com la validació, la serialització o l'ORM (Mapatge Objecte-Relacional).

function Required(target: any, propertyKey: string) {
    if (!target.constructor.requiredProperties) {
        target.constructor.requiredProperties = [];
    }
    target.constructor.requiredProperties.push(propertyKey);
}

class User {
    @Required
    name: string;

    @Required
    email: string;
}

console.log(User.requiredProperties); // Sortida: ['name', 'email']

Validació

Pots utilitzar decoradors de propietats per fer complir regles de validació a les propietats.

function MinLength(length: number) {
    return function (target: any, propertyKey: string) {
        let value: string;

        const getter = () => value;
        const setter = (newVal: string) => {
            if (newVal.length < length) {
                throw new Error(`The length of ${propertyKey} should be at least ${length}`);
            }
            value = newVal;
        };

        Object.defineProperty(target, propertyKey, {
            get: getter,
            set: setter,
            enumerable: true,
            configurable: true
        });
    };
}

class Person {
    @MinLength(3)
    name: string;
}

const person = new Person();
person.name = "Jo"; // Llença un error: The length of name should be at least 3

Ús Avançat

Combinant Diversos Decoradors

Pots aplicar diversos decoradors a una sola propietat. S'apliquen en l'ordre en què es declaren.

function FirstDecorator(target: any, propertyKey: string) {
    console.log(`FirstDecorator called on: ${propertyKey}`);
}

function SecondDecorator(target: any, propertyKey: string) {
    console.log(`SecondDecorator called on: ${propertyKey}`);
}

class Demo {
    @FirstDecorator
    @SecondDecorator
    demoProperty: string;
}

// Sortida:
// SecondDecorator called on: demoProperty
// FirstDecorator called on: demoProperty

Utilitzant Decoradors amb Reflexió de Metadades

TypeScript admet la reflexió de metadades a través de la biblioteca reflect-metadata, que es pot utilitzar per emmagatzemar i recuperar metadades sobre propietats.

import 'reflect-metadata';

function ReflectMetadataExample(target: any, propertyKey: string) {
    Reflect.defineMetadata('exampleMetadata', 'someValue', target, propertyKey);
}

class Sample {
    @ReflectMetadataExample
    sampleProperty: string;
}

const metadataValue = Reflect.getMetadata('exampleMetadata', Sample.prototype, 'sampleProperty');
console.log(metadataValue); // Sortida: 'someValue'

Conclusió

Els decoradors de propietats en TypeScript proporcionen una manera poderosa d'afegir metadades, fer complir la validació i modificar el comportament de les propietats de la classe. Es poden utilitzar en una varietat d'escenaris, des de la simple registració fins a la validació complexa i la gestió de metadades. Comprenent i utilitzant els decoradors de propietats, pots escriure codi més declaratiu i mantenible.

En aquest tema, hem cobert:

  • La sintaxi bàsica i el propòsit dels decoradors de propietats.
  • Casos pràctics incloent l'afegit de metadades i la validació.
  • Ús avançat com la combinació de diversos decoradors i l'ús de la reflexió de metadades.

Experimenta amb els decoradors de propietats en els teus projectes per veure com poden simplificar i millorar el teu codi!
