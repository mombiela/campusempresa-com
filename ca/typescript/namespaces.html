Espais de noms a TypeScript
Resolució de mòduls ►

Els espais de noms a TypeScript proporcionen una manera d'organitzar i agrupar codi relacionat, fent-lo més fàcil de gestionar i evitant conflictes de noms. Són particularment útils en grans bases de codi on múltiples desenvolupadors poden estar treballant en diferents parts de l'aplicació.

Introducció als Espais de Noms

Els espais de noms són una característica específica de TypeScript que et permet agrupar codi relacionat. Són similars als mòduls però s'utilitzen per organitzar codi dins d'un sol fitxer o a través de múltiples fitxers.

Conceptes Clau

  • Declaració d'Espai de Noms: Utilitza la paraula clau namespace per declarar un espai de noms.
  • Exportació de Membres: Utilitza la paraula clau export per fer variables, funcions, classes o interfícies disponibles fora de l'espai de noms.
  • Espais de Noms Imbricats: Els espais de noms poden estar imbricats dins d'altres espais de noms.
  • Referència d'Espais de Noms: Utilitza la directiva /// <reference path="..."/> per referenciar espais de noms a través de diferents fitxers.

Creació i Ús d'Espais de Noms

Declaració Bàsica d'Espai de Noms

Per declarar un espai de noms, utilitza la paraula clau namespace seguida del nom de l'espai de noms i un bloc de codi.

namespace Utility {
    export function log(message: string): void {
        console.log(message);
    }

    export function error(message: string): void {
        console.error(message);
    }
}

En aquest exemple, hem creat un espai de noms anomenat Utility amb dues funcions exportades: log i error.

Accés als Membres de l'Espai de Noms

Per accedir als membres d'un espai de noms, utilitza la notació de punt.

Utility.log("Aquest és un missatge de registre.");
Utility.error("Aquest és un missatge d'error.");

Espais de Noms Imbricats

Els espais de noms poden estar imbricats per organitzar encara més el codi.

namespace App {
    export namespace Models {
        export class User {
            constructor(public name: string, public age: number) {}
        }
    }

    export namespace Services {
        export function getUserInfo(user: Models.User): string {
            return `${user.name} té ${user.age} anys.`;
        }
    }
}

Accés als membres d'espais de noms imbricats:

const user = new App.Models.User("Alice", 30);
console.log(App.Services.getUserInfo(user));

Espais de Noms a Través de Múltiples Fitxers

Per utilitzar espais de noms a través de múltiples fitxers, necessites referenciar-los utilitzant la directiva /// <reference path="..."/>.

Fitxer: `models.ts`

namespace App.Models {
    export class User {
        constructor(public name: string, public age: number) {}
    }
}

Fitxer: `services.ts`

/// <reference path="models.ts" />

namespace App.Services {
    export function getUserInfo(user: App.Models.User): string {
        return `${user.name} té ${user.age} anys.`;
    }
}

Fitxer: `main.ts`

/// <reference path="models.ts" />
/// <reference path="services.ts" />

const user = new App.Models.User("Alice", 30);
console.log(App.Services.getUserInfo(user));

Conclusió

Els espais de noms a TypeScript són una eina poderosa per organitzar i gestionar codi, especialment en projectes grans. Ajuden a evitar conflictes de noms i fan que la base de codi sigui més mantenible. Entenent com declarar, accedir i referenciar espais de noms, pots estructurar efectivament les teves aplicacions TypeScript.

Resum

  • Espais de Noms: Agrupa codi relacionat utilitzant la paraula clau namespace.
  • Exportació de Membres: Utilitza la paraula clau export per fer els membres accessibles fora de l'espai de noms.
  • Espais de Noms Imbricats: Organitza el codi encara més imbricant espais de noms.
  • Referència a Través de Fitxers: Utilitza la directiva /// <reference path="..."/> per referenciar espais de noms en diferents fitxers.

Dominant aquests conceptes, estaràs ben preparat per gestionar projectes complexos de TypeScript amb facilitat.
