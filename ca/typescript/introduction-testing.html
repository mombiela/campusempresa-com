Introducció a les Proves
Proves unitàries amb Jest ►

Les proves són una part crucial del desenvolupament de programari que assegura que el teu codi es comporti com s'espera. En aquesta secció, explorarem els conceptes bàsics de les proves en TypeScript, cobrint conceptes clau, eines i pràctiques.

Per què són importants les proves

  • Assegurament de la Qualitat: Assegura que el programari compleixi els estàndards requerits.
  • Detecció d'Errors: Identifica problemes a l'inici del procés de desenvolupament.
  • Documentació: Les proves poden servir com a documentació de com ha de funcionar el codi.
  • Seguretat en la Refactorització: Et permet fer canvis amb confiança que no has trencat la funcionalitat existent.

Tipus de Proves

  • Proves Unitàries: Prova unitats o components individuals del codi.
  • Proves d'Integració: Prova la interacció entre diferents unitats o components.
  • Proves de Cap a Cap (E2E): Prova tot el flux de l'aplicació de principi a fi.

Eines de Prova per a TypeScript

  • Jest: Un marc de proves popular amb suport integrat per a TypeScript.
  • Mocha: Un marc de proves flexible que es pot utilitzar amb diverses biblioteques d'assertions.
  • Chai: Una biblioteca d'assertions sovint utilitzada amb Mocha.
  • Sinon: Una biblioteca per crear espies, mocks i stubs.

Configuració d'un Entorn de Proves

Per començar amb les proves en TypeScript, necessites configurar el teu entorn. Aquí, utilitzarem Jest com a exemple.

Instal·lació de Jest

Primer, instal·la Jest i el seu suport per a TypeScript:

npm install --save-dev jest ts-jest @types/jest

Configuració de Jest

Crea un fitxer jest.config.js per configurar Jest per a TypeScript:

module.exports = {
  preset: 'ts-jest',
  testEnvironment: 'node',
  testMatch: ['**/?(*.)+(spec|test).[jt]s?(x)'],
};

Escriure la teva Primera Prova

Escrivim una funció simple i una prova per a ella.

Exemple de Funció

Crea un fitxer sum.ts:

export function sum(a: number, b: number): number {
  return a + b;
}

Exemple de Prova

Crea un fitxer sum.test.ts:

import { sum } from './sum';

test('suma 1 + 2 per igualar 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Executar la Prova

Executa la prova utilitzant la següent comanda:

npx jest

Hauries de veure una sortida indicant que la prova ha passat.

Millors Pràctiques per a les Proves

  • Escriu Proves Aviat: Comença a escriure proves tan aviat com escriguis el teu codi.
  • Mantingues les Proves Petites i Focalitzades: Cada prova hauria de centrar-se en una sola funcionalitat.
  • Utilitza Noms Descriptius: Nomena les teves proves clarament per descriure què estan provant.
  • Mock de Dependències Externes: Utilitza mocks per aïllar la unitat de codi que s'està provant.

Conclusió

Les proves són una habilitat essencial per a qualsevol desenvolupador. Escriure proves assegura que el teu codi TypeScript sigui fiable, mantenible i lliure d'errors. Comença amb proves unitàries simples i gradualment passa a proves d'integració i de cap a cap més complexes. Utilitza eines com Jest per agilitzar el procés de proves i segueix les millors pràctiques per fer que les teves proves siguin efectives i eficients.
