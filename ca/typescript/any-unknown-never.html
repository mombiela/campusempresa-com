Tipus Any, Unknown i Never
Tipus de funcions ►

Introducció

En TypeScript, entendre els tipus any, unknown i never és crucial per escriure codi robust i segur en termes de tipus. Aquests tipus ajuden a gestionar diferents escenaris on la informació de tipus és flexible, desconeguda o impossible.

El tipus `any`

El tipus any és un tipus poderós però potencialment perillós en TypeScript. Et permet evitar la comprovació de tipus, fent-lo útil per a escenaris on no coneixes el tipus per avançat o quan migres codi JavaScript a TypeScript.

Conceptes Clau

  • Flexibilitat: any pot contenir qualsevol valor, i pots realitzar qualsevol operació sobre ell sense que TypeScript es queixi.
  • Seguretat de Tipus: Utilitzar any elimina els beneficis de la comprovació de tipus de TypeScript, cosa que pot conduir a errors en temps d'execució.

Exemple

let value: any;

value = 5; // OK
value = "Hola"; // OK
value = true; // OK

// Sense comprovació de tipus
value.toUpperCase(); // No hi ha error, però podria causar un error en temps d'execució si value no és una cadena

El tipus `unknown`

El tipus unknown és més segur que any perquè t'obliga a realitzar comprovacions de tipus abans de realitzar operacions sobre el valor. És útil quan vols acceptar qualsevol valor però encara assegurar la seguretat de tipus.

Conceptes Clau

  • Seguretat de Tipus: A diferència de any, no pots realitzar operacions sobre un tipus unknown sense primer restringir-lo a un tipus específic.
  • Guardes de Tipus: Has d'utilitzar guardes de tipus o afirmacions de tipus per treballar amb valors unknown.

Exemple

let value: unknown;

value = 5; // OK
value = "Hola"; // OK
value = true; // OK

// Comprovació de tipus requerida
if (typeof value === "string") {
    value.toUpperCase(); // OK
} else {
    // value.toUpperCase(); // Error: L'objecte és de tipus 'unknown'
}

El tipus `never`

El tipus never representa valors que mai ocorren. Sovint s'utilitza per a funcions que mai retornen o per a comprovacions exhaustives de tipus.

Conceptes Clau

  • Codi Inassolible: Funcions que llancen excepcions o tenen bucles infinits poden retornar never.
  • Comprovació d'Exhaustivitat: Útil en instruccions switch per assegurar que es gestionen tots els casos possibles.

Exemple

function error(message: string): never {
    throw new Error(message);
}

function infiniteLoop(): never {
    while (true) {}
}

// Comprovació d'exhaustivitat
type Shape = "circle" | "square";

function getArea(shape: Shape): number {
    switch (shape) {
        case "circle":
            return Math.PI * 1 * 1; // Càlcul d'àrea d'exemple
        case "square":
            return 1 * 1; // Càlcul d'àrea d'exemple
        default:
            const _exhaustiveCheck: never = shape;
            return _exhaustiveCheck; // Error si s'afegeix una nova forma però no es gestiona
    }
}

Conclusió

Entendre els tipus any, unknown i never en TypeScript és essencial per escriure codi flexible però segur en termes de tipus. Mentre que any proporciona flexibilitat, sacrifica la seguretat de tipus. El tipus unknown ofereix una alternativa més segura en fer complir les comprovacions de tipus. El tipus never és útil per a funcions que mai retornen i per assegurar comprovacions exhaustives de tipus. En dominar aquests tipus, pots escriure codi TypeScript més robust i mantenible.
