Introducció a les classes abstractes

Les classes abstractes són un concepte fonamental en la programació orientada a objectes (OOP) que permeten definir una classe base amb funcionalitat comuna, mentre es difereixen alguns detalls d'implementació a les classes derivades. En TypeScript, les classes abstractes proporcionen una manera de crear plànols per a altres classes.

Conceptes clau

  • Classe abstracta: Una classe que no es pot instanciar directament. Està destinada a ser subclassificada.
  • Mètode abstracte: Un mètode que es declara en una classe abstracta però que no té una implementació. Les subclasses estan obligades a proporcionar una implementació per a aquests mètodes.
  • Herència: Les classes abstractes s'utilitzen com a classes base de les quals altres classes poden heretar.

Definició de classes abstractes

Per definir una classe abstracta en TypeScript, utilitza la paraula clau abstract. Les classes abstractes poden contenir tant mètodes abstractes (sense implementació) com mètodes concrets (amb implementació).

Exemple

abstract class Animal {
    // Mètode abstracte (ha de ser implementat en les classes derivades)
    abstract makeSound(): void;

    // Mètode concret
    move(): void {
        console.log("Moving...");
    }
}

En aquest exemple, Animal és una classe abstracta amb un mètode abstracte makeSound i un mètode concret move.

Implementació de classes abstractes

Quan una classe hereta d'una classe abstracta, ha d'implementar tots els mètodes abstractes de la classe pare.

Exemple

class Dog extends Animal {
    makeSound(): void {
        console.log("Bark!");
    }
}

const myDog = new Dog();
myDog.makeSound(); // Output: Bark!
myDog.move();      // Output: Moving...

En aquest exemple, Dog és una subclasse d'Animal i proporciona una implementació per al mètode makeSound.

Usos de les classes abstractes

Les classes abstractes són útils quan vols:

  • Definir una interfície comuna per a un grup de classes relacionades.
  • Proporcionar alguna funcionalitat compartida mentre forces les subclasses a implementar mètodes específics.
  • Assegurar-te que certs mètodes són implementats en les classes derivades.

Comparació: Classes abstractes vs Interfícies

Tant les classes abstractes com les interfícies s'utilitzen per definir l'estructura d'una classe, però tenen algunes diferències.

| Característica | Classe abstracta | Interfície | |---------------------|---------------------------------------|------------------------------------| | Herència | Pot heretar d'una classe abstracta | Pot implementar múltiples interfícies | | Implementació de mètodes | Pot tenir mètodes concrets | No pot tenir mètodes concrets | | Propietats | Pot tenir propietats amb implementacions | Només declaracions, sense implementacions | | Instanciació | No es pot instanciar directament | No es pot instanciar directament |

Exemple d'interfície

interface AnimalInterface {
    makeSound(): void;
    move(): void;
}

class Cat implements AnimalInterface {
    makeSound(): void {
        console.log("Meow!");
    }

    move(): void {
        console.log("Moving silently...");
    }
}

En aquest exemple, Cat implementa la AnimalInterface i proporciona implementacions per als mètodes makeSound i move.

Conclusió

Les classes abstractes en TypeScript són una eina poderosa per crear un plànol comú per a classes relacionades. Permeten definir mètodes que han de ser implementats per les subclasses, assegurant una interfície consistent mentre es proporciona funcionalitat compartida. Entendre quan i com utilitzar les classes abstractes, així com les diferències entre classes abstractes i interfícies, és crucial per a una programació efectiva en TypeScript.

Dominant les classes abstractes, pots crear bases de codi més organitzades, mantenibles i escalables.
