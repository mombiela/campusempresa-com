Introducció als Decoradors de Classes

Els decoradors de classes són una característica poderosa en TypeScript que et permet modificar o anotar classes. Són funcions que prenen un constructor de classe com a argument i retornen un nou constructor o modifiquen l'existent.

Conceptes Clau

  • Funció Decoradora: Una funció que pren un constructor de classe com a argument.
  • Anotació: Afegir metadades a una classe.
  • Modificació: Canviar el comportament o les propietats d'una classe.

Sintaxi Bàsica dels Decoradors de Classes

Un decorador de classe es defineix amb el símbol @ seguit del nom de la funció decoradora.

function simpleDecorator(constructor: Function) {
    console.log("SimpleDecorator called.");
}

@simpleDecorator
class MyClass {
    constructor() {
        console.log("MyClass instance created.");
    }
}

const myInstance = new MyClass();
// Sortida:
// SimpleDecorator called.
// MyClass instance created.

Explicació

  • La funció simpleDecorator registra un missatge quan és cridada.
  • La sintaxi @simpleDecorator aplica el decorador a MyClass.
  • Quan MyClass és instanciat, la funció decoradora s'executa primer.

Crear Decoradors de Classes Útils

Els decoradors de classes es poden utilitzar per a diversos propòsits, com ara registre, validació o modificació de propietats de la classe.

Exemple: Decorador de Registre

function logClass(constructor: Function) {
    console.log(`Class ${constructor.name} is created.`);
}

@logClass
class User {
    constructor(public name: string) {}
}

const user = new User("Alice");
// Sortida:
// Class User is created.

Explicació

  • El decorador logClass registra el nom de la classe quan és creada.
  • La classe User està anotada amb @logClass, així que quan es crea una instància de User, el decorador registra el nom de la classe.

Ús Avançat: Modificar el Comportament de la Classe

Els decoradors de classes també poden modificar el comportament de la classe retornant un nou constructor.

Exemple: Afegir una Marca de Temps

function addTimestamp<T extends { new(...args: any[]): {} }>(constructor: T) {
    return class extends constructor {
        timestamp = new Date();
    }
}

@addTimestamp
class Document {
    constructor(public title: string) {}
}

const doc = new Document("My Document");
console.log(doc.timestamp); // Mostra la data i hora actuals

Explicació

  • El decorador addTimestamp afegeix una propietat timestamp a la classe.
  • Retorna una nova classe que extén la classe original i afegeix la propietat timestamp.
  • Quan es crea una instància de Document, inclou la propietat timestamp amb la data i hora actuals.

Combinació de Diversos Decoradors

Pots aplicar diversos decoradors a una sola classe. S'executen en l'ordre en què es declaren, de dalt a baix.

Exemple: Combinació de Decoradors

function firstDecorator(constructor: Function) {
    console.log("FirstDecorator called.");
}

function secondDecorator(constructor: Function) {
    console.log("SecondDecorator called.");
}

@firstDecorator
@secondDecorator
class Product {
    constructor(public name: string) {}
}

const product = new Product("Laptop");
// Sortida:
// FirstDecorator called.
// SecondDecorator called.

Explicació

  • firstDecorator i secondDecorator s'apliquen a la classe Product.
  • Ambdós decoradors registren un missatge quan la classe és creada.
  • Els decoradors s'executen en l'ordre en què es declaren.

Conclusió

Els decoradors de classes en TypeScript proporcionen una manera flexible de modificar i anotar classes. Es poden utilitzar per a registre, validació, afegir propietats, o fins i tot alterar el comportament de la classe. Entendre com crear i aplicar decoradors de classes pot millorar enormement les teves habilitats de programació en TypeScript.

Resum

  • Els decoradors de classes són funcions que prenen un constructor de classe com a argument.
  • Es poden utilitzar per anotar o modificar classes.
  • Els decoradors s'apliquen utilitzant el símbol @.
  • Es poden combinar diversos decoradors i s'executen en l'ordre en què es declaren.

En dominar els decoradors de classes, pots escriure codi més modular, reutilitzable i mantenible en TypeScript.
