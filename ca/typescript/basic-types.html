Tipus Bàsics
Anotacions de tipus ►

En aquesta secció, cobrirem els tipus bàsics en TypeScript. Entendre aquests tipus és fonamental per escriure codi robust i segur en termes de tipus. Explorarem els següents tipus:

  • boolean
  • number
  • string
  • array
  • tuple
  • enum
  • any
  • void
  • null i undefined
  • never

Boolean

El tipus boolean representa un valor lògic que pot ser true o false.

let isDone: boolean = false;

Number

El tipus number s'utilitza per representar tant nombres enters com nombres de punt flotant.

let decimal: number = 6;
let hex: number = 0xf00d;
let binary: number = 0b1010;
let octal: number = 0o744;

String

El tipus string s'utilitza per representar dades textuals. Pots utilitzar cometes simples ('), cometes dobles (") o literals de plantilla (`).

let color: string = "blue";
color = 'red';

let fullName: string = `John Doe`;
let age: number = 30;
let sentence: string = `Hello, my name is ${fullName}. I'll be ${age + 1} years old next month.`;

Array

El tipus array es pot escriure de dues maneres. La primera manera utilitza el tipus seguit de [], i la segona manera utilitza un tipus genèric d'array, Array<type>.

let list: number[] = [1, 2, 3];
let list: Array<number> = [1, 2, 3];

Tuple

El tipus tuple et permet expressar un array amb un nombre fix d'elements els tipus dels quals són coneguts però poden ser diferents.

let x: [string, number];
x = ["hello", 10]; // OK
// x = [10, "hello"]; // Error

Enum

El tipus enum és una manera de donar noms més amigables a conjunts de valors numèrics.

enum Color {Red, Green, Blue}
let c: Color = Color.Green;

També pots personalitzar els valors numèrics:

enum Color {Red = 1, Green, Blue}
let c: Color = Color.Green;

O assignar valors arbitraris:

enum Color {Red = 1, Green = 2, Blue = 4}
let c: Color = Color.Blue;

Any

El tipus any és una manera poderosa de treballar amb JavaScript existent, permetent-te no fer comprovacions de tipus.

let notSure: any = 4;
notSure = "maybe a string instead";
notSure = false; // okay, definitely a boolean

Void

El tipus void s'utilitza per indicar l'absència de qualsevol tipus, comunament utilitzat com el tipus de retorn de funcions que no retornen cap valor.

function warnUser(): void {
    console.log("This is my warning message");
}

Null i Undefined

TypeScript té dos tipus especials, null i undefined, que tenen els seus propis tipus. Per defecte, null i undefined són subtipus de tots els altres tipus.

let u: undefined = undefined;
let n: null = null;

Never

El tipus never representa el tipus de valors que mai ocorren. Per exemple, never és el tipus de retorn per a una funció que sempre llença una excepció o una que mai retorna.

function error(message: string): never {
    throw new Error(message);
}

function fail() {
    return error("Something failed");
}

function infiniteLoop(): never {
    while (true) {}
}

Conclusió

Entendre i utilitzar els tipus bàsics en TypeScript és essencial per escriure codi segur en termes de tipus. Aquests tipus t'ajuden a detectar errors a l'inici del procés de desenvolupament i fan que el teu codi sigui més llegible i mantenible. A mesura que avancis en aquest curs, veuràs com aquests tipus bàsics formen la base per a construccions de tipus més complexes i com es poden utilitzar per crear aplicacions robustes.
