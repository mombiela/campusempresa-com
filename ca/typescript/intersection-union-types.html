Tipus d'Intersecció i Unió en TypeScript
Guàrdies de tipus ►

Introducció

En TypeScript, entendre els tipus d'intersecció i unió és crucial per crear sistemes de tipus flexibles i robustos. Aquests tipus permeten combinar múltiples tipus en un, permetent definicions de tipus més precises i codi més segur.

Tipus d'Unió

Els tipus d'unió permeten que una variable sigui d'un de diversos tipus. Això és útil quan un valor pot ser de més d'un tipus.

Conceptes Clau

  • Definició: Un tipus d'unió es defineix utilitzant el símbol | (barra vertical).
  • Ús: Els tipus d'unió s'utilitzen quan un valor pot ser d'un de diversos tipus.

Exemples

let value: string | number;

value = "Hola"; // vàlid
value = 42;     // vàlid
value = true;   // invàlid, TypeScript llançarà un error

Explicació

En l'exemple anterior, value pot ser o bé un string o un number. Assignar un valor booleà a value resultarà en un error de TypeScript perquè boolean no forma part del tipus d'unió.

Usos Pràctics

  • Paràmetres de Funció: Quan una funció pot acceptar múltiples tipus.
  • Respostes d'API: Quan una API pot retornar diferents tipus de dades.
function printId(id: string | number) {
    console.log(`ID: ${id}`);
}

printId("abc123"); // vàlid
printId(123);      // vàlid

Tipus d'Intersecció

Els tipus d'intersecció combinen múltiples tipus en un. Això significa que un valor ha de satisfer tots els tipus combinats.

Conceptes Clau

  • Definició: Un tipus d'intersecció es defineix utilitzant el símbol & (i comercial).
  • Ús: Els tipus d'intersecció s'utilitzen quan un valor ha de tenir totes les propietats dels tipus combinats.

Exemples

interface Person {
    name: string;
}

interface Employee {
    employeeId: number;
}

type EmployeePerson = Person & Employee;

const employee: EmployeePerson = {
    name: "John Doe",
    employeeId: 1234
};

Explicació

En l'exemple anterior, EmployeePerson és un tipus d'intersecció que combina Person i Employee. L'objecte employee ha de tenir tant les propietats name com employeeId.

Usos Pràctics

  • Combinació d'Interfícies: Quan un objecte necessita satisfer múltiples interfícies.
  • Millora de Tipus: Quan s'estenen tipus amb propietats addicionals.
interface Address {
    street: string;
    city: string;
}

type EmployeeWithAddress = Employee & Address;

const employeeWithAddress: EmployeeWithAddress = {
    employeeId: 1234,
    street: "123 Main St",
    city: "Anytown"
};

Comparació de Tipus d'Unió i Intersecció

| Característica | Tipus d'Unió (|) | Tipus d'Intersecció (&) | |-------------------------|--------------------------------|----------------------------------| | Definició | type AorB = A \| B | type AandB = A & B | | Ús | El valor pot ser un dels tipus | El valor ha de satisfer tots els tipus | | Exemple | string \| number | Person & Employee | | Usos Pràctics | Paràmetres de funció, respostes d'API | Combinació d'interfícies, millora de tipus |

Conclusió

Entendre i utilitzar els tipus d'intersecció i unió en TypeScript pot millorar molt la flexibilitat i seguretat del teu codi. Els tipus d'unió permeten definicions de variables més versàtils, mentre que els tipus d'intersecció asseguren que els objectes compleixin múltiples requisits de tipus. Dominar aquests conceptes t'ajudarà a escriure codi TypeScript més robust i mantenible.
