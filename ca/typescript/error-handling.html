Gestió d'Errors en TypeScript
Refactorització de codi TypeScript ►

La gestió d'errors és un aspecte crucial de la programació que assegura que la teva aplicació pugui gestionar situacions inesperades de manera elegant. En TypeScript, la gestió d'errors és similar a JavaScript però amb funcions de seguretat de tipus afegides. Aquesta secció et guiarà des dels conceptes bàsics fins als avançats de la gestió d'errors en TypeScript.

Conceptes Bàsics de la Gestió d'Errors

La Sentència `try...catch`

La sentència try...catch et permet capturar errors que es produeixen durant l'execució i gestionar-los adequadament.

try {
    // Codi que pot llançar un error
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    // Gestiona l'error
    console.error("S'ha produït un error:", error);
}
  • Bloc try: Conté el codi que podria llançar un error.
  • Bloc catch: Conté el codi per gestionar l'error.

Llançament d'Errors

Pots llançar errors utilitzant la sentència throw. Això és útil per a la gestió d'errors personalitzada.

function riskyOperation() {
    if (Math.random() > 0.5) {
        throw new Error("Alguna cosa ha anat malament!");
    }
    return "Èxit!";
}

try {
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    console.error("S'ha capturat un error:", error.message);
}

Gestió d'Errors Avançada

Classes d'Errors Personalitzades

Crear classes d'errors personalitzades pot ajudar-te a gestionar diferents tipus d'errors de manera més efectiva.

class CustomError extends Error {
    constructor(message: string) {
        super(message);
        this.name = "CustomError";
    }
}

function riskyOperation() {
    throw new CustomError("Aquest és un error personalitzat!");
}

try {
    riskyOperation();
} catch (error) {
    if (error instanceof CustomError) {
        console.error("S'ha capturat un error personalitzat:", error.message);
    } else {
        console.error("S'ha capturat un error desconegut:", error);
    }
}

Ús del Bloc `finally`

El bloc finally conté codi que s'executarà independentment de si s'ha llançat un error o no.

try {
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    console.error("S'ha produït un error:", error);
} finally {
    console.log("Això sempre s'executarà.");
}

Gestió d'Errors amb Promeses

Gestionar errors en codi asíncron és crucial. Les promeses proporcionen un mètode catch per gestionar errors.

function asyncOperation(): Promise<string> {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            if (Math.random() > 0.5) {
                resolve("Èxit!");
            } else {
                reject(new Error("L'operació asíncrona ha fallat!"));
            }
        }, 1000);
    });
}

asyncOperation()
    .then(result => {
        console.log(result);
    })
    .catch(error => {
        console.error("S'ha capturat un error:", error.message);
    });

Gestió d'Errors amb `async/await`

La sintaxi async/await facilita el treball amb codi asíncron i la gestió d'errors.

async function performAsyncOperation() {
    try {
        let result = await asyncOperation();
        console.log(result);
    } catch (error) {
        console.error("S'ha capturat un error:", error.message);
    }
}

performAsyncOperation();

Conclusió

La gestió d'errors en TypeScript implica l'ús de sentències try...catch, la creació de classes d'errors personalitzades i la gestió d'errors en codi asíncron utilitzant Promeses i async/await. Comprenent i implementant aquestes tècniques, pots crear aplicacions robustes que puguin gestionar situacions inesperades de manera elegant.
