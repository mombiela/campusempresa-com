Mixins en TypeScript
Tipus condicionals ►

Els mixins són una característica poderosa en TypeScript que et permeten compondre classes a partir de components reutilitzables. Aquest tema et guiarà des dels conceptes bàsics fins a l'ús avançat dels mixins en TypeScript.

Què són els Mixins?

  • Definició: Els mixins són una manera de compondre classes a partir de peces de funcionalitat reutilitzables.
  • Propòsit: Ajuden a evitar les limitacions de la herència simple permetent un comportament semblant a la herència múltiple.

Conceptes Bàsics

Crear un Mixin

Per crear un mixin, defineixes una funció que pren una classe i retorna una nova classe que extén la classe original amb propietats o mètodes addicionals.

function Timestamped<TBase extends new (...args: any[]) => {}>(Base: TBase) {
    return class extends Base {
        timestamp = new Date();
    };
}

Aplicar un Mixin

Pots aplicar un mixin a una classe passant la classe a la funció del mixin.

class BasicClass {
    name: string;
    constructor(name: string) {
        this.name = name;
    }
}

const TimestampedClass = Timestamped(BasicClass);

const instance = new TimestampedClass("Example");
console.log(instance.name); // Output: Example
console.log(instance.timestamp); // Output: Current date and time

Conceptes Avançats

Combinació de Múltiples Mixins

Pots combinar múltiples mixins per crear classes més complexes.

function Activatable<TBase extends new (...args: any[]) => {}>(Base: TBase) {
    return class extends Base {
        isActive = false;
        activate() {
            this.isActive = true;
        }
        deactivate() {
            this.isActive = false;
        }
    };
}

const TimestampedActivatableClass = Activatable(Timestamped(BasicClass));

const instance = new TimestampedActivatableClass("Example");
instance.activate();
console.log(instance.isActive); // Output: true
console.log(instance.timestamp); // Output: Current date and time

Seguretat de Tipus amb Mixins

TypeScript proporciona una tipificació forta per als mixins, assegurant que les classes resultants tinguin els tipus correctes.

type Constructor<T = {}> = new (...args: any[]) => T;

function Loggable<TBase extends Constructor>(Base: TBase) {
    return class extends Base {
        log() {
            console.log(this);
        }
    };
}

const LoggableClass = Loggable(TimestampedActivatableClass);

const loggableInstance = new LoggableClass("Example");
loggableInstance.log(); // Output: instance details

Usos Pràctics

Millorar Classes Existents

Els mixins es poden utilitzar per afegir funcionalitat a classes existents sense modificar el seu codi font.

class User {
    constructor(public name: string) {}
}

const TimestampedUser = Timestamped(User);

const user = new TimestampedUser("Alice");
console.log(user.name); // Output: Alice
console.log(user.timestamp); // Output: Current date and time

Crear Components Reutilitzables

Els mixins et permeten crear components reutilitzables que es poden aplicar a múltiples classes.

class Product {
    constructor(public title: string) {}
}

const TimestampedProduct = Timestamped(Product);

const product = new TimestampedProduct("Laptop");
console.log(product.title); // Output: Laptop
console.log(product.timestamp); // Output: Current date and time

Conclusió

Els mixins en TypeScript proporcionen una manera flexible de compondre classes a partir de peces de funcionalitat reutilitzables. Et permeten millorar classes existents i crear components reutilitzables, fent que el teu codi sigui més modular i mantenible. Entenent i aplicant mixins, pots aprofitar el poder d'un comportament semblant a la herència múltiple en TypeScript mentre mantens una forta seguretat de tipus.
