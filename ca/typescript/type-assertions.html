Les assercions de tipus a TypeScript
Tipus literals ►

TypeScript és un llenguatge potent que es basa en JavaScript afegint definicions de tipus estàtics. Una de les característiques clau que proporciona TypeScript són les assercions de tipus. Aquest tema et guiarà per entendre i utilitzar les assercions de tipus de manera efectiva, des de nivells principiants fins a avançats.

Què són les assercions de tipus?

Les assercions de tipus són una manera de dir al compilador de TypeScript que tracti un valor com un tipus específic. Això pot ser útil quan saps més sobre el tipus d'un valor del que sap TypeScript.

  • Les assercions de tipus no realitzen comprovacions de tipus ni reestructuració.
  • Són purament una manera d'informar el compilador sobre el tipus d'una variable.

Sintaxi bàsica

Hi ha dues maneres de realitzar assercions de tipus a TypeScript:

  1. Sintaxi de claudàtors angulats:

    let someValue: any = "this is a string";
let strLength: number = (<string>someValue).length;

  2. Sintaxi as:

    let someValue: any = "this is a string";
let strLength: number = (someValue as string).length;

Tots dos exemples diuen al compilador de TypeScript que tracti someValue com una cadena, permetent-nos accedir a la propietat length.

Quan utilitzar les assercions de tipus

Les assercions de tipus són útils en diversos escenaris:

  • Quan es treballa amb elements del DOM:

    let inputElement = document.getElementById("myInput") as HTMLInputElement;
inputElement.value = "Hello, World!";

  • Quan es treballa amb biblioteques de tercers:

    declare const someLibrary: any;
let result = (someLibrary as SomeLibraryType).doSomething();

  • Quan es redueixen tipus en declaracions condicionals:

    function getLength(value: string | number): number {
  if ((value as string).length !== undefined) {
    return (value as string).length;
  } else {
    return value.toString().length;
  }
}

Ús avançat

Assercions de tipus vs Conversió de tipus

Les assercions de tipus són diferents de la conversió de tipus en altres llenguatges. Les assercions de tipus no canvien el tipus en temps d'execució d'una variable; només afecten la comprovació de tipus en temps de compilació.

Combinació amb tipus d'unió

Les assercions de tipus poden ser particularment útils quan es treballa amb tipus d'unió:

function processValue(value: string | number) {
  if (typeof value === "string") {
    console.log((value as string).toUpperCase());
  } else {
    console.log((value as number).toFixed(2));
  }
}

Assercions de no nul·litat

TypeScript també proporciona un operador d'asseració de no nul·litat (!) que es pot utilitzar per afirmar que un valor no és null o undefined:

let someValue: string | null = "Hello, World!";
console.log(someValue!.toUpperCase());

Trampes comunes

  • Ús excessiu de les assercions de tipus: Confiar massa en les assercions de tipus pot conduir a un codi menys segur en termes de tipus. Sempre intenta utilitzar la inferència de tipus i els guardians de tipus quan sigui possible.
  • Assercions incorrectes: Afirmar un valor a un tipus incorrecte pot conduir a errors en temps d'execució.

Conclusió

Les assercions de tipus són una característica poderosa a TypeScript que et permeten anul·lar la inferència de tipus del compilador quan tens més informació sobre els tipus en el teu codi. S'han d'utilitzar amb prudència per assegurar que el teu codi es mantingui segur en termes de tipus i mantenible.

Entenent i utilitzant les assercions de tipus de manera efectiva, pots aprofitar al màxim el sistema de tipus de TypeScript i escriure codi més robust i lliure d'errors.
