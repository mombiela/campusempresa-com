Refactorització de Codi TypeScript
Optimització del rendiment ►

La refactorització és el procés de reestructurar el codi existent sense canviar el seu comportament extern. En TypeScript, la refactorització pot ajudar a millorar la llegibilitat, la mantenibilitat i el rendiment del codi. Aquesta secció et guiarà a través dels principis i tècniques de refactorització de codi TypeScript, des de nivells bàsics fins a avançats.

Per què Refactoritzar?

  • Millorar la Llegibilitat: Fa que el codi sigui més fàcil d'entendre.
  • Millorar la Mantenibilitat: Simplifica les modificacions futures.
  • Optimitzar el Rendiment: Millora l'eficiència del codi.
  • Reduir la Complexitat: Descompon el codi complex en parts més senzilles.

Tècniques Bàsiques de Refactorització

Canviar el Nom de Variables i Funcions

Canviar el nom de variables i funcions a noms més descriptius pot millorar significativament la llegibilitat del codi.

Abans:

function calc(a: number, b: number): number {
    return a + b;
}

const x = 5;
const y = 10;
console.log(calc(x, y));

Després:

function calculateSum(firstNumber: number, secondNumber: number): number {
    return firstNumber + secondNumber;
}

const firstValue = 5;
const secondValue = 10;
console.log(calculateSum(firstValue, secondValue));

Extreure Funcions

Extreure parts d'una funció en funcions més petites i reutilitzables pot fer que el codi sigui més modular i fàcil de provar.

Abans:

function processNumbers(numbers: number[]): number {
    let total = 0;
    for (let i = 0; i < numbers.length; i++) {
        total += numbers[i];
    }
    return total / numbers.length;
}

Després:

function sum(numbers: number[]): number {
    return numbers.reduce((acc, curr) => acc + curr, 0);
}

function average(numbers: number[]): number {
    return sum(numbers) / numbers.length;
}

function processNumbers(numbers: number[]): number {
    return average(numbers);
}

Simplificar Expressions Condicionals

Simplificar expressions condicionals complexes pot fer que el codi sigui més fàcil de seguir.

Abans:

function getDiscount(price: number): number {
    if (price > 100) {
        return price * 0.1;
    } else {
        return 0;
    }
}

Després:

function getDiscount(price: number): number {
    return price > 100 ? price * 0.1 : 0;
}

Tècniques Intermèdies de Refactorització

Utilitzar Aliases de Tipus i Interfícies

Els aliases de tipus i les interfícies poden fer que els tipus complexos siguin més llegibles i reutilitzables.

Abans:

function printUser(user: { name: string; age: number; email: string }): void {
    console.log(`Name: ${user.name}, Age: ${user.age}, Email: ${user.email}`);
}

Després:

interface User {
    name: string;
    age: number;
    email: string;
}

function printUser(user: User): void {
    console.log(`Name: ${user.name}, Age: ${user.age}, Email: ${user.email}`);
}

Eliminar Nombres Màgics

Substituir nombres màgics per constants amb nom pot fer que el codi sigui més comprensible.

Abans:

function calculateArea(radius: number): number {
    return 3.14 * radius * radius;
}

Després:

const PI = 3.14;

function calculateArea(radius: number): number {
    return PI * radius * radius;
}

Tècniques Avançades de Refactorització

Aplicar Patrons de Disseny

Utilitzar patrons de disseny pot ajudar a resoldre problemes comuns d'una manera més estructurada.

Abans:

class UserService {
    private users: string[] = [];

    addUser(user: string): void {
        this.users.push(user);
    }

    getUsers(): string[] {
        return this.users;
    }
}

Després (Patró Singleton):

class UserService {
    private static instance: UserService;
    private users: string[] = [];

    private constructor() {}

    static getInstance(): UserService {
        if (!UserService.instance) {
            UserService.instance = new UserService();
        }
        return UserService.instance;
    }

    addUser(user: string): void {
        this.users.push(user);
    }

    getUsers(): string[] {
        return this.users;
    }
}

const userService = UserService.getInstance();
userService.addUser('John Doe');
console.log(userService.getUsers());

Refactoritzar per al Rendiment

Optimitzar el codi per al rendiment pot implicar diverses tècniques, com ara la memoització o la càrrega mandrosa.

Abans:

function fibonacci(n: number): number {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

Després (Memoització):

const memo: { [key: number]: number } = {};

function fibonacci(n: number): number {
    if (n <= 1) return n;
    if (memo[n]) return memo[n];
    memo[n] = fibonacci(n - 1) + fibonacci(n - 2);
    return memo[n];
}

Conclusió

La refactorització és una habilitat crucial per a qualsevol desenvolupador. Ajuda a millorar la qualitat del teu codi, fent-lo més llegible, mantenible i eficient. Aplicant les tècniques discutides en aquesta secció, pots assegurar-te que el teu codi TypeScript es mantingui net i robust a mesura que els teus projectes creixin en complexitat. Recorda, l'objectiu de la refactorització no és només fer que el codi funcioni, sinó fer que funcioni bé.
