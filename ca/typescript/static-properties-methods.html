Propietats i Mètodes Estàtics
Tipus d'intersecció i unió ►

Les propietats i mètodes estàtics són un concepte essencial en TypeScript i en la programació orientada a objectes. Pertanyen a la classe en si mateixa en lloc de a les instàncies de la classe. Aquest tema cobrirà els conceptes bàsics de les propietats i mètodes estàtics, el seu ús, i proporcionarà exemples pràctics per consolidar la teva comprensió.

Què són les Propietats i Mètodes Estàtics?

  • Propietats Estàtiques: Aquestes són propietats que pertanyen a la classe en si mateixa, no a cap instància específica de la classe.
  • Mètodes Estàtics: Aquests són mètodes que es poden cridar a la classe en si mateixa, no a les instàncies de la classe.

Per què Utilitzar Propietats i Mètodes Estàtics?

  • Dades Compartides: Les propietats estàtiques es poden utilitzar per emmagatzemar dades que es comparteixen entre totes les instàncies d'una classe.
  • Funcions d'Utilitat: Els mètodes estàtics sovint s'utilitzen per a funcions d'utilitat que no requereixen cap dada de les instàncies de la classe.

Definició de Propietats i Mètodes Estàtics

En TypeScript, les propietats i mètodes estàtics es defineixen utilitzant la paraula clau static.

Exemple: Propietats Estàtiques

class MyClass {
    static staticProperty: string = "Sóc una propietat estàtica";

    static showStaticProperty() {
        console.log(MyClass.staticProperty);
    }
}

// Accés a la propietat estàtica sense crear una instància
console.log(MyClass.staticProperty); // Sortida: Sóc una propietat estàtica

Exemple: Mètodes Estàtics

class MathUtils {
    static add(a: number, b: number): number {
        return a + b;
    }
}

// Crida al mètode estàtic sense crear una instància
console.log(MathUtils.add(5, 3)); // Sortida: 8

Usos Pràctics

Patró Singleton

El patró Singleton assegura que una classe només tingui una instància i proporciona un punt d'accés global a aquesta. Les propietats i mètodes estàtics sovint s'utilitzen per implementar aquest patró.

class Singleton {
    private static instance: Singleton;

    private constructor() {
        // Constructor privat per evitar la instanciació
    }

    static getInstance(): Singleton {
        if (!Singleton.instance) {
            Singleton.instance = new Singleton();
        }
        return Singleton.instance;
    }

    showMessage(): void {
        console.log("Hola, sóc una instància singleton!");
    }
}

const singleton1 = Singleton.getInstance();
const singleton2 = Singleton.getInstance();

singleton1.showMessage(); // Sortida: Hola, sóc una instància singleton!
console.log(singleton1 === singleton2); // Sortida: true

Classes d'Utilitat

Les classes d'utilitat sovint contenen mètodes estàtics que realitzen tasques comunes.

class StringUtils {
    static toUpperCase(str: string): string {
        return str.toUpperCase();
    }

    static toLowerCase(str: string): string {
        return str.toLowerCase();
    }
}

console.log(StringUtils.toUpperCase("hola")); // Sortida: HOLA
console.log(StringUtils.toLowerCase("MÓN")); // Sortida: món

Punts Clau a Recordar

  • Les propietats i mètodes estàtics pertanyen a la classe en si mateixa, no a les instàncies de la classe.
  • Utilitza la paraula clau static per definir propietats i mètodes estàtics.
  • Els mètodes estàtics sovint s'utilitzen per a funcions d'utilitat i dades compartides.
  • Les propietats i mètodes estàtics es poden accedir directament utilitzant el nom de la classe.

Conclusió

Les propietats i mètodes estàtics són eines poderoses en TypeScript que et permeten definir propietats i mètodes que pertanyen a la classe en si mateixa en lloc de a les instàncies de la classe. Són particularment útils per a dades compartides, funcions d'utilitat i patrons de disseny com el patró Singleton. Entendre com utilitzar les propietats i mètodes estàtics de manera efectiva et pot ajudar a escriure codi més organitzat i eficient.
