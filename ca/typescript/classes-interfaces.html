Classes i interfícies en TypeScript
Herència ►

Introducció

En TypeScript, les classes i les interfícies són blocs de construcció fonamentals per crear codi robust i mantenible. Aquesta secció et guiarà des dels conceptes bàsics fins als avançats, assegurant que entenguis com definir, implementar i utilitzar classes i interfícies de manera efectiva.

Classes en TypeScript

Estructura bàsica d'una classe

Una classe en TypeScript és un plànol per crear objectes amb propietats i mètodes predefinits.

class Person {
    name: string;
    age: number;

    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }

    greet() {
        console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    }
}

const person1 = new Person('Alice', 30);
person1.greet(); // Output: Hello, my name is Alice and I am 30 years old.
  • Propietats: Variables que contenen dades específiques per a cada instància de la classe.
  • Constructor: Un mètode especial per inicialitzar nous objectes.
  • Mètodes: Funcions definides dins d'una classe que operen sobre les propietats de la classe.

Herència

L'herència permet que una classe estengui una altra classe, heretant les seves propietats i mètodes.

class Employee extends Person {
    employeeId: number;

    constructor(name: string, age: number, employeeId: number) {
        super(name, age);
        this.employeeId = employeeId;
    }

    displayEmployeeId() {
        console.log(`Employee ID: ${this.employeeId}`);
    }
}

const employee1 = new Employee('Bob', 25, 12345);
employee1.greet(); // Output: Hello, my name is Bob and I am 25 years old.
employee1.displayEmployeeId(); // Output: Employee ID: 12345
  • Paraula clau extends: Utilitzada per crear una subclasse.
  • Paraula clau super: Crida el constructor de la classe base.

Modificadors d'accés

TypeScript proporciona modificadors d'accés per controlar la visibilitat dels membres de la classe.

  • public: Accessible des de qualsevol lloc.
  • private: Accessible només dins de la classe.
  • protected: Accessible dins de la classe i les seves subclasses.
class Animal {
    public name: string;
    private age: number;
    protected species: string;

    constructor(name: string, age: number, species: string) {
        this.name = name;
        this.age = age;
        this.species = species;
    }

    public getAge() {
        return this.age;
    }
}

class Dog extends Animal {
    constructor(name: string, age: number, species: string) {
        super(name, age, species);
    }

    public getSpecies() {
        return this.species;
    }
}

const dog1 = new Dog('Buddy', 5, 'Canine');
console.log(dog1.name); // Output: Buddy
console.log(dog1.getAge()); // Output: 5
console.log(dog1.getSpecies()); // Output: Canine

Interfícies en TypeScript

Definició d'interfícies

Les interfícies defineixen l'estructura a la qual un objecte ha d'adherir-se.

interface Person {
    name: string;
    age: number;
    greet(): void;
}

const person: Person = {
    name: 'Charlie',
    age: 40,
    greet() {
        console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    }
};

person.greet(); // Output: Hello, my name is Charlie and I am 40 years old.
  • Propietats: Defineixen les propietats requerides i els seus tipus.
  • Mètodes: Defineixen els mètodes requerits i les seves signatures.

Implementació d'interfícies en classes

Les classes poden implementar interfícies per assegurar-se que s'adhereixen a una estructura específica.

interface Drivable {
    startEngine(): void;
    stopEngine(): void;
}

class Car implements Drivable {
    startEngine() {
        console.log('Engine started.');
    }

    stopEngine() {
        console.log('Engine stopped.');
    }
}

const myCar = new Car();
myCar.startEngine(); // Output: Engine started.
myCar.stopEngine(); // Output: Engine stopped.
  • Paraula clau implements: Assegura que la classe s'adhereix a l'estructura de la interfície.

Extensió d'interfícies

Les interfícies poden estendre altres interfícies, permetent estructures més complexes.

interface Animal {
    name: string;
    age: number;
}

interface Mammal extends Animal {
    hasFur: boolean;
}

const mammal: Mammal = {
    name: 'Elephant',
    age: 10,
    hasFur: false
};

console.log(mammal); // Output: { name: 'Elephant', age: 10, hasFur: false }

Conceptes avançats

Classes abstractes

Les classes abstractes no es poden instanciar directament i estan destinades a ser subclassificades.

abstract class Shape {
    abstract area(): number;

    displayArea() {
        console.log(`Area: ${this.area()}`);
    }
}

class Circle extends Shape {
    radius: number;

    constructor(radius: number) {
        super();
        this.radius = radius;
    }

    area(): number {
        return Math.PI * this.radius * this.radius;
    }
}

const circle = new Circle(5);
circle.displayArea(); // Output: Area: 78.53981633974483
  • Paraula clau abstract: Utilitzada per definir classes i mètodes abstractes.

Tipus d'intersecció

Combinació de múltiples tipus en un utilitzant tipus d'intersecció.

interface CanFly {
    fly(): void;
}

interface CanSwim {
    swim(): void;
}

type AmphibiousVehicle = CanFly & CanSwim;

const vehicle: AmphibiousVehicle = {
    fly() {
        console.log('Flying.');
    },
    swim() {
        console.log('Swimming.');
    }
};

vehicle.fly(); // Output: Flying.
vehicle.swim(); // Output: Swimming.

Conclusió

Entendre les classes i les interfícies en TypeScript és crucial per construir aplicacions escalables i mantenibles. Les classes proporcionen un plànol per crear objectes, mentre que les interfícies defineixen l'estructura a la qual els objectes han d'adherir-se. En dominar aquests conceptes, podràs escriure codi més robust i segur en termes de tipus.
