Resolució de Mòduls
Importacions dinàmiques ►

La resolució de mòduls és un concepte crític en TypeScript que determina com el compilador troba i connecta els mòduls en el teu codi. Entendre la resolució de mòduls ajuda a organitzar eficientment la teva base de codi i a evitar errors comuns relacionats amb les importacions de mòduls.

Conceptes Clau

  • Mòdul: Un fitxer que conté codi TypeScript. Els mòduls poden exportar i importar funcionalitats.
  • Resolució de Mòduls: El procés que utilitza el compilador de TypeScript per localitzar i carregar mòduls.
  • Especificador de Mòdul: La cadena en la declaració import que indica al compilador quin mòdul carregar.
  • Base URL: Una opció del compilador que estableix el directori base per a noms de mòduls no relatius.
  • Paths: Una opció del compilador que permet definir alias de mòduls.

Estratègies de Resolució de Mòduls

TypeScript utilitza dues estratègies principals per a la resolució de mòduls:

  1. Clàssica: L'estratègia per defecte utilitzada en TypeScript 1.6 i anteriors.
  2. Node: L'estratègia que imita la resolució de mòduls de Node.js, adequada per a aplicacions Node.js.

Resolució Clàssica de Mòduls

En l'estratègia clàssica, TypeScript intenta localitzar el mòdul seguint l'ordre següent:

  1. Ruta Relativa: Si l'especificador del mòdul comença amb ./ o ../, TypeScript el tracta com una ruta relativa.
  2. Ruta No Relativa: Si l'especificador del mòdul no comença amb ./ o ../, TypeScript el tracta com una ruta no relativa i cerca en el baseUrl i paths.

Resolució de Mòduls de Node

L'estratègia de Node és més complexa i segueix aquests passos:

  1. Ruta Relativa: Similar a l'estratègia clàssica, si l'especificador del mòdul comença amb ./ o ../, TypeScript el tracta com una ruta relativa.
  2. Ruta No Relativa: Per a rutes no relatives, TypeScript cerca en els directoris node_modules, començant pel directori actual i pujant per l'arbre de directoris.
  3. Extensions de Fitxers: TypeScript intenta resoldre el mòdul afegint extensions .ts, .tsx i .d.ts.
  4. Fitxers Index: Si l'especificador del mòdul apunta a un directori, TypeScript cerca un fitxer index.ts, index.tsx o index.d.ts en aquest directori.

Configurant la Resolució de Mòduls

Pots configurar la resolució de mòduls en el fitxer tsconfig.json utilitzant l'opció moduleResolution.

{
  "compilerOptions": {
    "moduleResolution": "node", // o "classic"
    "baseUrl": "./src",
    "paths": {
      "@app/*": ["app/*"]
    }
  }
}
  • moduleResolution: Especifica l'estratègia de resolució de mòduls (node o classic).
  • baseUrl: Estableix el directori base per resoldre noms de mòduls no relatius.
  • paths: Defineix alias de mòduls, permetent mapar especificadors de mòduls a rutes específiques.

Exemples

Importació de Mòduls Relatius

// Fitxer: src/utils/math.ts
export function add(a: number, b: number): number {
  return a + b;
}

// Fitxer: src/index.ts
import { add } from './utils/math';

console.log(add(2, 3)); // Sortida: 5

Importació de Mòduls No Relatius amb Base URL i Paths

// Fitxer: src/app/services/logger.ts
export function log(message: string): void {
  console.log(message);
}

// Fitxer: src/index.ts
import { log } from '@app/services/logger';

log('Hello, World!'); // Sortida: Hello, World!

Resolució de Mòduls de Node

// Fitxer: src/index.ts
import * as fs from 'fs';

fs.writeFileSync('test.txt', 'Hello, World!');

Conclusió

Entendre la resolució de mòduls en TypeScript és essencial per gestionar dependències i organitzar la teva base de codi de manera efectiva. Configurant adequadament el fitxer tsconfig.json i coneixent les diferències entre les estratègies de resolució de mòduls clàssica i de Node, pots evitar errors comuns i assegurar-te que els teus mòduls es localitzen i carreguen correctament pel compilador de TypeScript.
