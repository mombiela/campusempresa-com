Introducció a TypeScript amb Express

TypeScript és un llenguatge potent que es basa en JavaScript afegint definicions de tipus estàtics. Express és un framework minimalista i flexible per aplicacions web de Node.js que proporciona un conjunt robust de característiques per a aplicacions web i mòbils. Combinar TypeScript amb Express permet als desenvolupadors escriure codi més robust i mantenible.

Configuració de TypeScript amb Express

Abans de començar a codificar, necessitem configurar el nostre entorn de desenvolupament.

Prerequisits

  • Node.js instal·lat
  • npm (Node Package Manager) instal·lat

Passos per Configurar

  1. Inicialitzar un nou projecte de Node.js:

    mkdir typescript-express-app
cd typescript-express-app
npm init -y

  2. Instal·lar TypeScript i Express:

    npm install typescript ts-node @types/node @types/express express

  3. Crear un fitxer tsconfig.json:

    {
  "compilerOptions": {
    "target": "ES6",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "outDir": "./dist"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}

  4. Crear l'estructura del projecte:

    mkdir src
touch src/index.ts

Construir un Servidor Bàsic d'Express amb TypeScript

Creem un servidor simple d'Express utilitzant TypeScript.

Escriure el Codi del Servidor

  1. Importar els mòduls necessaris:

    import express, { Request, Response } from 'express';

  2. Inicialitzar l'aplicació d'Express:

    const app = express();
const port = 3000;

  3. Definir una ruta bàsica:

    app.get('/', (req: Request, res: Response) => {
  res.send('Hello, TypeScript with Express!');
});

  4. Iniciar el servidor:

    app.listen(port, () => {
  console.log(`Server is running at http://localhost:${port}`);
});

  5. Fitxer complet src/index.ts:

    import express, { Request, Response } from 'express';

const app = express();
const port = 3000;

app.get('/', (req: Request, res: Response) => {
  res.send('Hello, TypeScript with Express!');
});

app.listen(port, () => {
  console.log(`Server is running at http://localhost:${port}`);
});

  6. Executar el servidor:

    npx ts-node src/index.ts

Afegir Middleware

Les funcions middleware són funcions que tenen accés a l'objecte de sol·licitud (req), l'objecte de resposta (res), i la següent funció middleware en el cicle de sol·licitud-resposta de l'aplicació.

Exemple de Middleware

  1. Crear un middleware de registre:

    const loggerMiddleware = (req: Request, res: Response, next: Function) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

  2. Utilitzar el middleware en l'aplicació:

    app.use(loggerMiddleware);

  3. Fitxer complet src/index.ts amb middleware:

    import express, { Request, Response } from 'express';

const app = express();
const port = 3000;

const loggerMiddleware = (req: Request, res: Response, next: Function) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

app.use(loggerMiddleware);

app.get('/', (req: Request, res: Response) => {
  res.send('Hello, TypeScript with Express!');
});

app.listen(port, () => {
  console.log(`Server is running at http://localhost:${port}`);
});

Temes Avançats

Definicions de Tipus per a Request i Response

TypeScript ens permet definir tipus personalitzats per als objectes de sol·licitud i resposta per garantir la seguretat de tipus.

  1. Definir un tipus personalitzat per al cos de la sol·licitud:

    interface UserRequest extends Request {
  body: {
    name: string;
    age: number;
  };
}

  2. Utilitzar el tipus personalitzat en una ruta:

    app.post('/user', (req: UserRequest, res: Response) => {
  const { name, age } = req.body;
  res.send(`User ${name} is ${age} years old`);
});

Middleware de Gestió d'Errors

El middleware de gestió d'errors es defineix de la mateixa manera que altres middleware, excepte que té quatre arguments en lloc de tres: (err, req, res, next).

  1. Crear un middleware de gestió d'errors:

    const errorHandler = (err: Error, req: Request, res: Response, next: Function) => {
  console.error(err.message);
  res.status(500).send('Internal Server Error');
};

  2. Utilitzar el middleware de gestió d'errors:

    app.use(errorHandler);

Utilitzar Async/Await amb Express

Gestionar operacions asíncrones és crucial en aplicacions web modernes.

  1. Crear un controlador de ruta asíncron:

    app.get('/data', async (req: Request, res: Response) => {
  try {
    const data = await fetchDataFromDatabase();
    res.json(data);
  } catch (error) {
    res.status(500).send('Error fetching data');
  }
});

  2. Funció simulada fetchDataFromDatabase:

    const fetchDataFromDatabase = async () => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve({ message: 'Data fetched successfully' });
    }, 1000);
  });
};

Conclusió

Combinar TypeScript amb Express millora l'experiència de desenvolupament proporcionant seguretat de tipus i millors eines. Aquest curs ha cobert la configuració d'un projecte TypeScript amb Express, la creació d'un servidor bàsic, l'afegiment de middleware, la definició de tipus personalitzats, la gestió d'errors i l'ús d'async/await. Amb aquestes habilitats, pots construir aplicacions web robustes i mantenibles.
