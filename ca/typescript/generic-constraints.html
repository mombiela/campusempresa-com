Restriccions Genèriques
Ús de genèrics amb interfícies ►

En TypeScript, els genèrics proporcionen una manera de crear components reutilitzables que funcionen amb una varietat de tipus de dades. No obstant això, hi ha moments en què necessites restringir els tipus que es poden utilitzar amb un genèric. Aquí és on entren en joc les restriccions genèriques.

Què són les Restriccions Genèriques?

Les restriccions genèriques et permeten especificar que un tipus genèric ha de complir una certa estructura o heretar d'un tipus específic. Això assegura que el tipus genèric tingui les propietats o mètodes que necessites utilitzar dins de la teva funció o classe.

Conceptes Clau

  • Genèrics: Una manera de crear components que poden treballar amb qualsevol tipus de dades.
  • Restriccions: Restriccions que limiten els tipus que es poden utilitzar amb un genèric.

Per què Utilitzar Restriccions Genèriques?

Utilitzar restriccions genèriques pot ajudar-te a:

  • Assegurar la seguretat de tipus restringint els tipus que es poden utilitzar.
  • Accedir a propietats o mètodes específics del tipus genèric.
  • Millorar la llegibilitat i mantenibilitat del codi definint clarament l'estructura esperada dels tipus.

Com Aplicar Restriccions Genèriques

Pots aplicar restriccions genèriques utilitzant la paraula clau extends. Aquesta paraula clau et permet especificar que un tipus genèric ha d'extendre un tipus o interfície particular.

Exemple: Restricció Genèrica Bàsica

Comencem amb un exemple senzill on volem assegurar-nos que un tipus genèric tingui una propietat length.

interface HasLength {
  length: number;
}

function logLength<T extends HasLength>(item: T): void {
  console.log(item.length);
}

logLength("Hello, World!"); // Funciona perquè la cadena té una propietat length
logLength([1, 2, 3, 4, 5]); // Funciona perquè l'array té una propietat length
// logLength(123); // Error: Argument of type 'number' is not assignable to parameter of type 'HasLength'

En aquest exemple:

  • Definim una interfície HasLength amb una propietat length.
  • La funció genèrica logLength utilitza T extends HasLength per assegurar-se que el tipus T tingui una propietat length.
  • La funció funciona amb cadenes i arrays perquè ambdós tenen una propietat length, però llançarà un error si intentes passar un número.

Utilitzant Múltiples Restriccions

També pots aplicar múltiples restriccions a un tipus genèric utilitzant tipus d'intersecció.

Exemple: Múltiples Restriccions

interface HasName {
  name: string;
}

interface HasAge {
  age: number;
}

function logPerson<T extends HasName & HasAge>(person: T): void {
  console.log(`Name: ${person.name}, Age: ${person.age}`);
}

const person = { name: "Alice", age: 30, occupation: "Engineer" };
logPerson(person); // Funciona perquè la persona té tant propietats name com age
// logPerson({ name: "Bob" }); // Error: Property 'age' is missing

En aquest exemple:

  • Definim dues interfícies, HasName i HasAge.
  • La funció genèrica logPerson utilitza T extends HasName & HasAge per assegurar-se que el tipus T tingui tant propietats name com age.
  • La funció funciona amb un objecte que té ambdues propietats però llançarà un error si falta alguna propietat.

Casos Pràctics

Exemple: Classes Genèriques amb Restriccions

Les restriccions genèriques també són útils en classes. Considerem una classe que gestiona una col·lecció d'ítems, on cada ítem ha de tenir una propietat id.

interface Identifiable {
  id: number;
}

class ItemCollection<T extends Identifiable> {
  private items: T[] = [];

  addItem(item: T): void {
    this.items.push(item);
  }

  getItemById(id: number): T | undefined {
    return this.items.find(item => item.id === id);
  }
}

const collection = new ItemCollection<{ id: number; name: string }>();
collection.addItem({ id: 1, name: "Item 1" });
collection.addItem({ id: 2, name: "Item 2" });

console.log(collection.getItemById(1)); // { id: 1, name: "Item 1" }
console.log(collection.getItemById(3)); // undefined

En aquest exemple:

  • Definim una interfície Identifiable amb una propietat id.
  • La classe ItemCollection utilitza T extends Identifiable per assegurar-se que els ítems tinguin una propietat id.
  • La classe proporciona mètodes per afegir ítems i recuperar ítems pel seu id.

Conclusió

Les restriccions genèriques en TypeScript són una característica poderosa que et permet crear codi més robust i segur. Utilitzant restriccions, pots assegurar-te que els teus genèrics tinguin les propietats o mètodes necessaris, fent el teu codi més previsible i fàcil de mantenir.

  • Genèrics: Permeten components reutilitzables per a diversos tipus de dades.
  • Restriccions: Restringeixen els tipus que es poden utilitzar amb genèrics.
  • Ús Pràctic: Asseguren la seguretat de tipus i accedeixen a propietats o mètodes necessaris.

Entendre i utilitzar les restriccions genèriques de manera efectiva t'ajudarà a escriure codi TypeScript més flexible i fiable.
