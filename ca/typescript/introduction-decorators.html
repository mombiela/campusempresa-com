Què són els Decoradors?

Els decoradors són un tipus especial de declaració que es pot adjuntar a una classe, mètode, accessor, propietat o paràmetre. Els decoradors són una proposta de fase 2 per a JavaScript i estan disponibles com una característica experimental en TypeScript. Proporcionen una manera d'afegir tant anotacions com una sintaxi de meta-programació per a les declaracions de classes i membres.

Per què Utilitzar Decoradors?

  • Reutilització de Codi: Els decoradors permeten reutilitzar la lògica comuna en diverses classes o mètodes.
  • Separació de Responsabilitats: Ajuden a separar preocupacions transversals com el registre, la validació i la gestió d'errors de la lògica principal del negoci.
  • Millora de la Llegibilitat: Utilitzant decoradors, pots fer que el teu codi sigui més llegible i mantenible.

Tipus de Decoradors

  • Decoradors de Classe: Utilitzats per modificar o anotar una classe.
  • Decoradors de Mètode: Utilitzats per modificar o anotar un mètode.
  • Decoradors d'Accessor: Utilitzats per modificar o anotar un getter o setter.
  • Decoradors de Propietat: Utilitzats per modificar o anotar una propietat.
  • Decoradors de Paràmetre: Utilitzats per modificar o anotar un paràmetre.

Sintaxi Bàsica

La sintaxi bàsica per a un decorador en TypeScript és la següent:

function decoratorName(target: any, propertyKey?: string, descriptor?: PropertyDescriptor) {
    // Lògica del decorador aquí
}

Exemples

Decorador de Classe

Un decorador de classe s'aplica al constructor de la classe. Es pot utilitzar per modificar o reemplaçar la definició de la classe.

function sealed(constructor: Function) {
    Object.seal(constructor);
    Object.seal(constructor.prototype);
}

@sealed
class Greeter {
    greeting: string;
    constructor(message: string) {
        this.greeting = message;
    }
    greet() {
        return "Hola, " + this.greeting;
    }
}

En aquest exemple, el decorador sealed segella la classe Greeter, impedint que s'hi afegeixin noves propietats.

Decorador de Mètode

Un decorador de mètode s'aplica a un mètode d'una classe. Es pot utilitzar per modificar el comportament del mètode.

function log(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    const originalMethod = descriptor.value;

    descriptor.value = function (...args: any[]) {
        console.log(`Cridant ${propertyKey} amb arguments: ${JSON.stringify(args)}`);
        const result = originalMethod.apply(this, args);
        console.log(`Resultat: ${result}`);
        return result;
    };

    return descriptor;
}

class Calculator {
    @log
    add(a: number, b: number): number {
        return a + b;
    }
}

const calculator = new Calculator();
calculator.add(2, 3);

En aquest exemple, el decorador log registra el nom del mètode i els arguments abans i després de l'execució del mètode.

Decorador de Propietat

Un decorador de propietat s'aplica a una propietat d'una classe. Es pot utilitzar per modificar la propietat.

function readonly(target: any, propertyKey: string) {
    Object.defineProperty(target, propertyKey, {
        writable: false
    });
}

class Person {
    @readonly
    name: string;

    constructor(name: string) {
        this.name = name;
    }
}

const person = new Person("John");
person.name = "Doe"; // Error: No es pot assignar a 'name' perquè és una propietat de només lectura.

En aquest exemple, el decorador readonly fa que la propietat name sigui de només lectura.

Conclusió

Els decoradors en TypeScript proporcionen una manera poderosa d'afegir anotacions i capacitats de meta-programació a les teves classes i els seus membres. Ajuden a millorar la reutilització del codi, la separació de responsabilitats i la llegibilitat. Entenent i utilitzant decoradors, pots escriure aplicacions TypeScript més mantenibles i escalables.
