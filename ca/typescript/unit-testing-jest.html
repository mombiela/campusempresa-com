Introducció a les Proves Unitàries

Les proves unitàries són un mètode de prova de programari on es proven unitats o components individuals d'un programari. L'objectiu és validar que cada unitat del programari funcioni com s'espera.

  • Unitat: La part més petita testejable de qualsevol programari.
  • Prova Unitària: Un fragment de codi escrit per un desenvolupador que executa una petita part del codi que s'està provant i comprova si els resultats són els esperats.

Per què Utilitzar Jest per a les Proves Unitàries?

Jest és un marc de proves de JavaScript encantador amb un enfocament en la simplicitat. Funciona amb projectes que utilitzen Babel, TypeScript, Node.js, React, Angular, Vue.js, i més.

  • Zero configuració: Jest pretén funcionar fora de la caixa.
  • Snapshots: Captura l'estat de la teva interfície d'usuari per assegurar-te que no canviï inesperadament.
  • Aïllat: Les proves s'executen en paral·lel en els seus propis processos per maximitzar el rendiment.

Configurar Jest amb TypeScript

Per començar amb Jest en un projecte TypeScript, segueix aquests passos:

  1. Instal·la Jest i les seves dependències de TypeScript:

    npm install --save-dev jest ts-jest @types/jest

  2. Configura Jest: Crea un fitxer jest.config.js a l'arrel del teu projecte:

    module.exports = {
  preset: 'ts-jest',
  testEnvironment: 'node',
};

  3. Afegeix un script de prova a package.json:

    "scripts": {
  "test": "jest"
}

Escriure la Teva Primera Prova

Escrivim una funció simple i una prova corresponent.

Exemple de Funció

Crea un fitxer anomenat sum.ts:

export function sum(a: number, b: number): number {
  return a + b;
}

Exemple de Prova

Crea un fitxer anomenat sum.test.ts:

import { sum } from './sum';

test('suma 1 + 2 per igualar 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Executar la Prova

Executa la següent comanda per executar les teves proves:

npm test

Tècniques Avançades de Prova

Mocking

El mocking és una tècnica per substituir objectes complexos amb versions simplificades que simulen el comportament dels objectes reals.

Exemple amb Mocking

Suposa que tens una funció que obté dades d'una API:

import axios from 'axios';

export async function fetchData(url: string): Promise<any> {
  const response = await axios.get(url);
  return response.data;
}

Pots fer un mock d'axios en la teva prova:

import axios from 'axios';
import { fetchData } from './fetchData';

jest.mock('axios');
const mockedAxios = axios as jest.Mocked<typeof axios>;

test('obté dades correctament d'una API', async () => {
  const data = { name: 'John Doe' };
  mockedAxios.get.mockResolvedValue({ data });

  const result = await fetchData('https://api.example.com/user');
  expect(result).toEqual(data);
});

Proves de Snapshot

Les proves de snapshot són una manera de provar la interfície d'usuari comparant l'estat actual de la interfície amb un estat guardat prèviament.

Exemple amb Proves de Snapshot

import renderer from 'react-test-renderer';
import MyComponent from './MyComponent';

test('es renderitza correctament', () => {
  const tree = renderer.create(<MyComponent />).toJSON();
  expect(tree).toMatchSnapshot();
});

Conclusió

Les proves unitàries són una part essencial del procés de desenvolupament de programari, assegurant que els components individuals funcionin com s'espera. Jest proporciona un marc robust per escriure i executar proves en projectes TypeScript, oferint característiques com zero configuració, mocking, i proves de snapshot. Integrant Jest en els teus projectes TypeScript, pots millorar la qualitat del codi i la mantenibilitat, fent el teu procés de desenvolupament més eficient i fiable.
