Introducció

En TypeScript, les funcions es poden definir amb paràmetres opcionals i per defecte. Aquestes característiques milloren la flexibilitat i la robustesa del teu codi permetent que les funcions gestionin casos on alguns arguments poden faltar o tenir valors per defecte.

Paràmetres opcionals

Els paràmetres opcionals són paràmetres que poden o no ser proporcionats quan es crida una funció. Es defineixen utilitzant un signe d'interrogació (?) després del nom del paràmetre.

Conceptes clau

  • Els paràmetres opcionals han de venir després dels paràmetres requerits en la signatura de la funció.
  • Si no es proporciona un paràmetre opcional, el seu valor és undefined.

Exemple

function greet(name: string, greeting?: string): string {
    if (greeting) {
        return `${greeting}, ${name}!`;
    } else {
        return `Hello, ${name}!`;
    }
}

console.log(greet("Alice")); // Output: Hello, Alice!
console.log(greet("Bob", "Good morning")); // Output: Good morning, Bob!

En aquest exemple:

  • El paràmetre greeting és opcional.
  • Si no es proporciona greeting, la funció per defecte diu "Hello".

Paràmetres per defecte

Els paràmetres per defecte et permeten especificar un valor per defecte per a un paràmetre. Si no es proporciona l'argument, s'utilitza el valor per defecte.

Conceptes clau

  • Els paràmetres per defecte es poden utilitzar en qualsevol posició de la llista de paràmetres.
  • Si es proporciona un valor per a un paràmetre per defecte, aquest valor sobreescriu el valor per defecte.

Exemple

function greet(name: string, greeting: string = "Hello"): string {
    return `${greeting}, ${name}!`;
}

console.log(greet("Alice")); // Output: Hello, Alice!
console.log(greet("Bob", "Good morning")); // Output: Good morning, Bob!

En aquest exemple:

  • El paràmetre greeting té un valor per defecte de "Hello".
  • Si no es proporciona greeting, s'utilitza "Hello".

Combinació de paràmetres opcionals i per defecte

Pots combinar paràmetres opcionals i per defecte en una funció. No obstant això, tingues en compte que els paràmetres opcionals han de venir després dels paràmetres requerits.

Exemple

function greet(name: string, greeting?: string, punctuation: string = "!"): string {
    if (greeting) {
        return `${greeting}, ${name}${punctuation}`;
    } else {
        return `Hello, ${name}${punctuation}`;
    }
}

console.log(greet("Alice")); // Output: Hello, Alice!
console.log(greet("Bob", "Good morning")); // Output: Good morning, Bob!
console.log(greet("Charlie", undefined, ".")); // Output: Hello, Charlie.

En aquest exemple:

  • El paràmetre greeting és opcional.
  • El paràmetre punctuation té un valor per defecte de "!".
  • Si no es proporciona greeting, la funció per defecte diu "Hello".

Conclusió

Entendre els paràmetres opcionals i per defecte en TypeScript et permet escriure funcions més flexibles i mantenibles. Els paràmetres opcionals proporcionen una manera de gestionar casos on alguns arguments poden faltar, mentre que els paràmetres per defecte asseguren que les teves funcions tinguin valors per defecte raonables quan no es proporcionen arguments. Combinant aquestes característiques, pots crear funcions robustes que gestionin una varietat d'escenaris d'entrada de manera elegant.
