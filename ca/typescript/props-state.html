Introducció

En aquesta secció, explorarem els conceptes de Props i Estat en TypeScript, particularment en el context de React. Entendre aquests conceptes és crucial per construir aplicacions web dinàmiques i interactives.

Props

Props (abreviatura de propietats) són atributs de només lectura passats d'un component pare a un component fill. S'utilitzen per passar dades i controladors d'esdeveniments cap avall en l'arbre de components.

Conceptes Clau

  • Immutabilitat: Les props són immutables, és a dir, no poden ser canviades pel component fill.
  • Seguretat de Tipus: TypeScript et permet definir els tipus de les props, proporcionant millor seguretat de tipus i autocompleció.

Exemple

Creem un component React senzill que utilitza props en TypeScript.

import React from 'react';

// Definir el tipus per a les props
interface GreetingProps {
  name: string;
  age: number;
}

// Component funcional que utilitza props
const Greeting: React.FC<GreetingProps> = ({ name, age }) => {
  return (
    <div>
      <h1>Hola, {name}!</h1>
      <p>Tens {age} anys.</p>
    </div>
  );
};

// Ús
const App: React.FC = () => {
  return (
    <div>
      <Greeting name="Alice" age={30} />
    </div>
  );
};

export default App;

En aquest exemple:

  • Definim una interfície GreetingProps per especificar els tipus de name i age.
  • El component Greeting utilitza aquestes props per renderitzar un missatge.
  • El component App passa les props a Greeting.

Estat

L'estat és una manera de gestionar dades que poden canviar amb el temps dins d'un component. A diferència de les props, l'estat és mutable i pot ser actualitzat pel mateix component.

Conceptes Clau

  • Mutabilitat: L'estat pot ser actualitzat utilitzant la funció setState.
  • Local al Component: L'estat és local al component i no pot ser accedit per altres components a menys que es passi com a props.

Exemple

Creem un component React senzill que utilitza estat en TypeScript.

import React, { useState } from 'react';

// Component funcional que utilitza estat
const Counter: React.FC = () => {
  // Definir estat amb valor inicial
  const [count, setCount] = useState<number>(0);

  return (
    <div>
      <h1>Comptador: {count}</h1>
      <button onClick={() => setCount(count + 1)}>Incrementar</button>
      <button onClick={() => setCount(count - 1)}>Decrementar</button>
    </div>
  );
};

// Ús
const App: React.FC = () => {
  return (
    <div>
      <Counter />
    </div>
  );
};

export default App;

En aquest exemple:

  • Utilitzem el hook useState per crear una variable d'estat count amb un valor inicial de 0.
  • La funció setCount s'utilitza per actualitzar l'estat.
  • El component renderitza botons per incrementar i decrementar el comptador.

Comparació entre Props i Estat

| Característica | Props | Estat | |----------------|--------------------------------|--------------------------------| | Mutabilitat | Immutable | Mutable | | Àmbit | Passat de pare a fill | Local al component | | Mètode d'Actualització | No pot ser actualitzat pel fill | Actualitzat utilitzant setState | | Ús | Passar dades i controladors d'esdeveniments | Gestionar dades dinàmiques dins del component |

Conclusió

Entendre les props i l'estat és fonamental per construir aplicacions React amb TypeScript. Les props et permeten passar dades i funcions cap avall en l'arbre de components, mentre que l'estat et permet gestionar dades dinàmiques dins d'un component. Aprofitant la seguretat de tipus de TypeScript, pots crear codi més robust i mantenible. Practica utilitzant props i estat en els teus projectes per consolidar aquests conceptes.
