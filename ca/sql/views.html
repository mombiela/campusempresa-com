Introducció a les Vistes

Les vistes en SQL són taules virtuals que proporcionen una manera de simplificar consultes complexes, millorar la seguretat i presentar dades en un format específic. No emmagatzemen dades per si mateixes, sinó que obtenen dades de les taules subjacents sempre que s'accedeix a elles.

Conceptes Clau

  • Taula Virtual: Una vista es comporta com una taula però no emmagatzema dades físicament.
  • Simplificació de Consultes: Les vistes poden encapsular consultes complexes, fent-les més fàcils de reutilitzar.
  • Seguretat: Les vistes poden restringir l'accés a columnes o files específiques de dades.
  • Abstracció de Dades: Les vistes poden presentar dades en un format o estructura diferent de les taules subjacents.

Creació de Vistes

Per crear una vista, utilitzeu la instrucció CREATE VIEW seguida del nom de la vista i la consulta SELECT que defineix la vista.

Sintaxi

CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

Exemple

Suposem que tenim una taula employees amb la següent estructura:

CREATE TABLE employees (
    employee_id INT,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    department VARCHAR(50),
    salary DECIMAL(10, 2)
);

Volem crear una vista que mostri només els empleats del departament de 'Sales'.

CREATE VIEW sales_employees AS
SELECT employee_id, first_name, last_name, salary
FROM employees
WHERE department = 'Sales';

Ús de les Vistes

Un cop creada una vista, podeu consultar-la igual que una taula regular.

Exemple

SELECT * FROM sales_employees;

Aquesta consulta retornarà totes les columnes i files de la vista sales_employees.

Actualització de Vistes

Les vistes es poden actualitzar en alguns casos, però hi ha restriccions. La capacitat d'actualitzar una vista depèn de la complexitat de la instrucció SELECT utilitzada per crear-la.

Exemple

UPDATE sales_employees
SET salary = salary * 1.1
WHERE employee_id = 101;

Aquesta consulta actualitzarà el salari de l'empleat amb employee_id 101 a la vista sales_employees, i conseqüentment, a la taula subjacents employees.

Eliminació de Vistes

Si ja no necessiteu una vista, podeu eliminar-la utilitzant la instrucció DROP VIEW.

Sintaxi

DROP VIEW view_name;

Exemple

DROP VIEW sales_employees;

Conceptes Avançats de Vistes

Vistes Indexades

En algunes bases de dades SQL, podeu crear vistes indexades per millorar el rendiment. Aquestes també es coneixen com a vistes materialitzades.

Sintaxi

CREATE VIEW view_name WITH SCHEMABINDING AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

Exemple

CREATE VIEW sales_summary WITH SCHEMABINDING AS
SELECT department, COUNT(*) AS employee_count, AVG(salary) AS avg_salary
FROM employees
GROUP BY department;

Actualització de Vistes Materialitzades

Les vistes materialitzades necessiten ser actualitzades per reflectir els canvis a les taules subjacents.

Sintaxi

REFRESH MATERIALIZED VIEW view_name;

Exemple

REFRESH MATERIALIZED VIEW sales_summary;

Conclusió

Les vistes són una característica poderosa en SQL que permet simplificar consultes complexes, millorar la seguretat i presentar dades en un format específic. Es poden crear, consultar, actualitzar i eliminar igual que les taules regulars, però no emmagatzemen dades per si mateixes. Conceptes avançats com les vistes indexades poden optimitzar encara més el rendiment per a casos d'ús específics. Entendre i utilitzar les vistes de manera efectiva pot millorar enormement les vostres habilitats de programació SQL.
