Introducció als Triggers

Els triggers són tipus especials de procediments emmagatzemats que s'executen automàticament o es disparen quan es produeixen certs esdeveniments a la base de dades. S'utilitzen per fer complir les regles de negoci, validar dades i mantenir registres d'auditoria.

Conceptes Clau

  • Esdeveniment: L'acció que fa que el trigger es dispari (per exemple, INSERT, UPDATE, DELETE).
  • Moment: Quan s'executa el trigger en relació amb l'esdeveniment (per exemple, BEFORE, AFTER).
  • Àmbit: La taula o vista a la qual està adjunt el trigger.

Creació de Triggers

Per crear un trigger, s'utilitza la instrucció CREATE TRIGGER. La sintaxi bàsica és:

CREATE TRIGGER nom_del_trigger
{BEFORE | AFTER} {INSERT | UPDATE | DELETE}
ON nom_de_la_taula
FOR EACH ROW
BEGIN
    -- Lògica del trigger aquí
END;

Exemple: Trigger Bàsic d'Inserció

Aquest exemple demostra un trigger que registra operacions d'inserció en una taula users.

CREATE TABLE users (
    id INT PRIMARY KEY,
    username VARCHAR(50),
    created_at TIMESTAMP
);

CREATE TABLE user_log (
    log_id INT PRIMARY KEY,
    user_id INT,
    action VARCHAR(50),
    log_time TIMESTAMP
);

CREATE TRIGGER log_user_insert
AFTER INSERT ON users
FOR EACH ROW
BEGIN
    INSERT INTO user_log (log_id, user_id, action, log_time)
    VALUES (NULL, NEW.id, 'INSERT', NOW());
END;

Explicació

  • Nom del Trigger: log_user_insert
  • Moment: AFTER
  • Esdeveniment: INSERT
  • Taula: users
  • Lògica: Insereix una entrada de registre a la taula user_log cada vegada que s'afegeix un nou usuari a la taula users.

Conceptes Avançats de Triggers

Lògica Condicional en Triggers

Es pot incloure lògica condicional dins dels triggers per realitzar diferents accions segons les dades que s'insereixen, actualitzen o eliminen.

CREATE TRIGGER update_user_log
AFTER UPDATE ON users
FOR EACH ROW
BEGIN
    IF OLD.username != NEW.username THEN
        INSERT INTO user_log (log_id, user_id, action, log_time)
        VALUES (NULL, NEW.id, 'USERNAME_CHANGE', NOW());
    END IF;
END;

Explicació

  • Condició: Comprova si el username ha canviat.
  • Acció: Registra el canvi si es compleix la condició.

Ús de Triggers per a la Validació de Dades

Els triggers també es poden utilitzar per fer complir les regles d'integritat de les dades.

CREATE TRIGGER validate_user_age
BEFORE INSERT ON users
FOR EACH ROW
BEGIN
    IF NEW.age < 18 THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'User must be at least 18 years old';
    END IF;
END;

Explicació

  • Moment: BEFORE
  • Esdeveniment: INSERT
  • Validació: Assegura que l'edat de l'usuari sigui almenys de 18 anys.

Gestió de Triggers

Visualització de Triggers

Es poden veure els triggers existents utilitzant la següent consulta:

SHOW TRIGGERS;

Eliminació de Triggers

Per eliminar un trigger, s'utilitza la instrucció DROP TRIGGER:

DROP TRIGGER nom_del_trigger;

Millors Pràctiques

  • Mantenir els Triggers Simples: La lògica complexa pot fer que els triggers siguin difícils de mantenir i depurar.
  • Minimitzar els Efectes Secundaris: Evitar fer canvis en altres taules a menys que sigui necessari.
  • Documentar els Triggers: Documentar clarament el propòsit i la lògica de cada trigger.

Conclusió

Els triggers són eines poderoses en SQL que permeten automatitzar tasques i fer complir regles a nivell de base de dades. Entenent com crear i gestionar triggers, es pot assegurar la integritat de les dades, mantenir registres d'auditoria i implementar lògica de negoci complexa directament dins de la base de dades. Recorda utilitzar els triggers amb prudència i mantenir-los tan simples i clars com sigui possible per mantenir el rendiment i la llegibilitat de la teva base de dades.
