Procediments Emmagatzemats
Desencadenants ►

Els procediments emmagatzemats són una característica poderosa en SQL que permet encapsular i reutilitzar codi SQL. Poden ajudar a millorar el rendiment, la mantenibilitat i la seguretat de les operacions de la teva base de dades. Aquesta secció et guiarà des dels conceptes bàsics fins als avançats dels procediments emmagatzemats.

Introducció als Procediments Emmagatzemats

  • Definició: Un procediment emmagatzemat és una col·lecció precompilada d'una o més instruccions SQL emmagatzemades al servidor de bases de dades.
  • Beneficis:
    • Rendiment: Reducció del trànsit de xarxa i millora de la velocitat d'execució.
    • Mantenibilitat: Lògica centralitzada que es pot reutilitzar en diverses aplicacions.
    • Seguretat: Accés controlat a les dades a través de la gestió de permisos.

Creació de Procediments Emmagatzemats

Sintaxi Bàsica

Per crear un procediment emmagatzemat, utilitzes la instrucció CREATE PROCEDURE seguida del nom del procediment i les instruccions SQL que conté.

CREATE PROCEDURE procedure_name
AS
BEGIN
    -- Les instruccions SQL van aquí
END;

Exemple: Procediment Emmagatzemat Simple

Creem un procediment emmagatzemat simple que recupera tots els registres d'una taula Customers.

CREATE PROCEDURE GetAllCustomers
AS
BEGIN
    SELECT * FROM Customers;
END;
  • Explicació:
    • CREATE PROCEDURE GetAllCustomers: Defineix un nou procediment emmagatzemat anomenat GetAllCustomers.
    • AS BEGIN ... END;: Encapsula les instruccions SQL a executar.

Execució de Procediments Emmagatzemats

Per executar un procediment emmagatzemat, utilitzes la instrucció EXEC o EXECUTE.

EXEC GetAllCustomers;

Exemple: Execució d'un Procediment Emmagatzemat

EXEC GetAllCustomers;
  • Explicació:
    • EXEC GetAllCustomers;: Executa el procediment emmagatzemat GetAllCustomers, que recupera tots els registres de la taula Customers.

Paràmetres en Procediments Emmagatzemats

Els procediments emmagatzemats poden acceptar paràmetres, permetent-te passar dades a ells.

Sintaxi amb Paràmetres

CREATE PROCEDURE procedure_name
    @parameter_name datatype,
    @another_parameter datatype
AS
BEGIN
    -- Instruccions SQL utilitzant paràmetres
END;

Exemple: Procediment Emmagatzemat amb Paràmetres

Creem un procediment emmagatzemat que recupera registres de clients basats en el seu país.

CREATE PROCEDURE GetCustomersByCountry
    @Country NVARCHAR(50)
AS
BEGIN
    SELECT * FROM Customers WHERE Country = @Country;
END;
  • Explicació:
    • @Country NVARCHAR(50): Defineix un paràmetre anomenat @Country de tipus NVARCHAR(50).
    • WHERE Country = @Country;: Filtra la taula Customers basant-se en el paràmetre Country.

Execució d'un Procediment Emmagatzemat amb Paràmetres

EXEC GetCustomersByCountry @Country = 'USA';
  • Explicació:
    • @Country = 'USA': Passa el valor 'USA' al paràmetre @Country.

Conceptes Avançats

Paràmetres de Sortida

Els procediments emmagatzemats també poden retornar valors a través de paràmetres de sortida.

Sintaxi amb Paràmetres de Sortida

CREATE PROCEDURE procedure_name
    @input_parameter datatype,
    @output_parameter datatype OUTPUT
AS
BEGIN
    -- Instruccions SQL
    SET @output_parameter = some_value;
END;

Exemple: Procediment Emmagatzemat amb Paràmetres de Sortida

Creem un procediment emmagatzemat que retorna el nombre total de clients en un país específic.

CREATE PROCEDURE GetCustomerCountByCountry
    @Country NVARCHAR(50),
    @CustomerCount INT OUTPUT
AS
BEGIN
    SELECT @CustomerCount = COUNT(*) FROM Customers WHERE Country = @Country;
END;
  • Explicació:
    • @CustomerCount INT OUTPUT: Defineix un paràmetre de sortida anomenat @CustomerCount de tipus INT.
    • SELECT @CustomerCount = COUNT(*) ...: Assigna el recompte de clients al paràmetre @CustomerCount.

Execució d'un Procediment Emmagatzemat amb Paràmetres de Sortida

DECLARE @Count INT;
EXEC GetCustomerCountByCountry @Country = 'USA', @CustomerCount = @Count OUTPUT;
SELECT @Count AS CustomerCount;
  • Explicació:
    • DECLARE @Count INT;: Declara una variable per contenir el valor de sortida.
    • EXEC GetCustomerCountByCountry ...: Executa el procediment emmagatzemat i assigna la sortida a @Count.
    • SELECT @Count AS CustomerCount;: Mostra el valor de sortida.

Gestió d'Errors en Procediments Emmagatzemats

Bloc TRY...CATCH

Pots utilitzar el bloc TRY...CATCH per gestionar errors dins dels procediments emmagatzemats.

Exemple: Gestió d'Errors

CREATE PROCEDURE SafeInsertCustomer
    @CustomerName NVARCHAR(50),
    @Country NVARCHAR(50)
AS
BEGIN
    BEGIN TRY
        INSERT INTO Customers (CustomerName, Country) VALUES (@CustomerName, @Country);
    END TRY
    BEGIN CATCH
        SELECT ERROR_MESSAGE() AS ErrorMessage;
    END CATCH
END;
  • Explicació:
    • BEGIN TRY ... END TRY: Encapsula les instruccions SQL que podrien causar un error.
    • BEGIN CATCH ... END CATCH: Gestiona qualsevol error que es produeixi en el bloc TRY.

Conclusió

Els procediments emmagatzemats són una característica fonamental en SQL que poden millorar enormement l'eficiència, la mantenibilitat i la seguretat de les teves operacions de bases de dades. Entenent com crear, executar i gestionar procediments emmagatzemats, pots aprofitar tot el seu potencial per construir aplicacions de bases de dades robustes i escalables.
