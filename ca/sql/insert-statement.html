El Projecte Sobre nosaltres Contribuir Donacions Llicència

La declaració INSERT en SQL s'utilitza per afegir noves files de dades a una taula. Aquesta és una operació fonamental en la gestió de bases de dades, permetent-te omplir les taules amb les dades necessàries per a les teves aplicacions.

Conceptes Clau

  • Sintaxi: L'estructura bàsica de la declaració INSERT.
  • Inserció d'una Sola Fila: Com inserir una sola fila en una taula.
  • Inserció de Múltiples Files: Com inserir múltiples files en una sola declaració.
  • Inserció de Dades d'una Altra Taula: Utilitzant INSERT INTO ... SELECT per copiar dades d'una taula a una altra.
  • Gestió de Valors NULL: Com tractar les columnes que poden tenir valors NULL.
  • Gestió d'Errors: Errors comuns i com gestionar-los.

Sintaxi

La sintaxi bàsica per a la declaració INSERT és la següent:

INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);
  • table_name: El nom de la taula on vols inserir les dades.
  • column1, column2, column3, ...: Les columnes de la taula on s'inseriran les dades.
  • value1, value2, value3, ...: Els valors que s'inseriran a les columnes especificades.

Inserció d'una Sola Fila

Per inserir una sola fila en una taula, pots utilitzar el següent exemple:

INSERT INTO employees (first_name, last_name, email, hire_date)
VALUES ('John', 'Doe', 'john.doe@example.com', '2023-10-01');

En aquest exemple:

  • La taula employees té columnes first_name, last_name, email i hire_date.
  • La clàusula VALUES especifica les dades que s'inseriran en aquestes columnes.

Inserció de Múltiples Files

Pots inserir múltiples files en una taula en una sola declaració INSERT:

INSERT INTO employees (first_name, last_name, email, hire_date)
VALUES 
    ('Jane', 'Smith', 'jane.smith@example.com', '2023-10-02'),
    ('Alice', 'Johnson', 'alice.johnson@example.com', '2023-10-03'),
    ('Bob', 'Brown', 'bob.brown@example.com', '2023-10-04');

Aquest exemple insereix tres files a la taula employees d'una sola vegada.

Inserció de Dades d'una Altra Taula

També pots inserir dades en una taula des d'una altra taula utilitzant la declaració INSERT INTO ... SELECT:

INSERT INTO employees_archive (first_name, last_name, email, hire_date)
SELECT first_name, last_name, email, hire_date
FROM employees
WHERE hire_date < '2023-01-01';

En aquest exemple:

  • Les dades es copien de la taula employees a la taula employees_archive.
  • Només s'insereixen les files on hire_date és anterior a l'1 de gener de 2023.

Gestió de Valors NULL

Si una columna pot acceptar valors NULL, pots inserir NULL explícitament o ometre la columna de la declaració INSERT:

INSERT INTO employees (first_name, last_name, email)
VALUES ('Charlie', 'Davis', NULL);

O:

INSERT INTO employees (first_name, last_name)
VALUES ('Charlie', 'Davis');

En ambdós casos, la columna email tindrà un valor NULL per a la nova fila.

Gestió d'Errors

Els errors comuns que pots trobar quan utilitzes la declaració INSERT inclouen:

  • Error de Clau Duplicada: Es produeix quan intentes inserir una fila amb una clau primària que ja existeix.
  • Desajust de Tipus de Dades: Passa quan el valor que s'està inserint no coincideix amb el tipus de dades de la columna.
  • Violació de Restriccions: Es produeix quan les dades que s'estan inserint violen una restricció de la taula, com una restricció de clau forana.

Exemple de gestió d'un error de clau duplicada:

INSERT INTO employees (id, first_name, last_name, email)
VALUES (1, 'David', 'Wilson', 'david.wilson@example.com')
ON CONFLICT (id) DO NOTHING;

En aquest exemple, si ja existeix una fila amb id 1, la clàusula ON CONFLICT evita la inserció i no fa res.

Conclusió

La declaració INSERT és una eina poderosa en SQL per afegir noves dades a les teves taules. Entenent la seva sintaxi i diversos casos d'ús, pots gestionar eficientment les dades de la teva base de dades. Ja sigui que estiguis inserint una sola fila, múltiples files o copiant dades d'una altra taula, dominar la declaració INSERT és essencial per a qualsevol programador SQL.
