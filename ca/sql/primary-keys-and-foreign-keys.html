Introducció

En SQL, les claus primàries i les claus foranes són conceptes fonamentals utilitzats per establir i fer complir les relacions entre taules. Entendre aquestes claus és crucial per dissenyar bases de dades robustes i eficients.

Claus Primàries

Una clau primària és un identificador únic per a un registre en una taula. Assegura que cada registre pugui ser identificat i recuperat de manera única.

Conceptes Clau

  • Unicitat: Cada valor a la columna (o columnes) de la clau primària ha de ser únic.
  • No nul·litat: Les columnes de la clau primària no poden contenir valors NULL.
  • Simple o Compost: Una clau primària pot consistir en una sola columna o múltiples columnes (clau composta).

Exemple

Considera una taula Students on cada estudiant té un ID únic:

CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    BirthDate DATE
);

En aquest exemple:

  • StudentID és la clau primària.
  • Identifica de manera única cada estudiant a la taula Students.

Claus Foranes

Una clau forana és una columna o un conjunt de columnes en una taula que identifica de manera única una fila d'una altra taula. Estableix un enllaç entre les dades de les dues taules.

Conceptes Clau

  • Integritat referencial: Assegura que el valor a la columna (o columnes) de la clau forana coincideixi amb un valor a la columna (o columnes) de la clau primària de la taula referenciada.
  • Relació Pare-Fill: La taula que conté la clau forana és la taula fill, i la taula que conté la clau primària és la taula pare.

Exemple

Considera una taula Enrollments que registra quins estudiants estan matriculats en quins cursos:

CREATE TABLE Enrollments (
    EnrollmentID INT PRIMARY KEY,
    StudentID INT,
    CourseID INT,
    EnrollmentDate DATE,
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID)
);

En aquest exemple:

  • StudentID a la taula Enrollments és una clau forana.
  • Referencia StudentID a la taula Students, establint una relació entre les dues taules.

Combinació de Claus Primàries i Claus Foranes

Per il·lustrar com funcionen conjuntament les claus primàries i les claus foranes, afegim una taula Courses i modifiquem la taula Enrollments per incloure una referència de clau forana a Courses.

Exemple

CREATE TABLE Courses (
    CourseID INT PRIMARY KEY,
    CourseName VARCHAR(100),
    Credits INT
);

CREATE TABLE Enrollments (
    EnrollmentID INT PRIMARY KEY,
    StudentID INT,
    CourseID INT,
    EnrollmentDate DATE,
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
    FOREIGN KEY (CourseID) REFERENCES Courses(CourseID)
);

En aquest exemple:

  • CourseID a la taula Courses és la clau primària.
  • CourseID a la taula Enrollments és una clau forana que referencia CourseID a la taula Courses.
  • Aquesta configuració assegura que cada registre de matrícula estigui enllaçat a un estudiant vàlid i a un curs vàlid.

Resum

  • Claus Primàries: Asseguren que cada registre en una taula sigui únic i identificable.
  • Claus Foranes: Estableixen relacions entre taules, assegurant la integritat referencial.
  • Ús: Les claus primàries s'utilitzen per identificar de manera única els registres, mentre que les claus foranes enllacen registres entre taules.

Entendre i implementar correctament les claus primàries i les claus foranes és essencial per mantenir la integritat de les dades i establir relacions clares dins de la teva base de dades.
