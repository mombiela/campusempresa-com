Introducció

En aquesta secció, explorarem com es pot utilitzar SQL conjuntament amb bases de dades NoSQL. Tot i que SQL està tradicionalment associat amb bases de dades relacionals, moltes bases de dades NoSQL ara ofereixen capacitats de consulta similars a SQL. Això permet als desenvolupadors aprofitar les seves habilitats en SQL mentre treballen amb bases de dades NoSQL.

Conceptes Clau

  • Bases de Dades NoSQL: Aquestes bases de dades estan dissenyades per manejar grans volums de dades no estructurades o semi-estructurades. Es classifiquen en quatre tipus principals: Document, Clau-Valor, Família de Columnes i Gràfics.
  • Llenguatges de Consulta Similars a SQL: Moltes bases de dades NoSQL proporcionen els seus propis llenguatges de consulta que són similars a SQL, facilitant que els desenvolupadors familiaritzats amb SQL treballin amb bases de dades NoSQL.
  • Integració: Comprendre com integrar consultes SQL amb bases de dades NoSQL pot millorar les capacitats de recuperació i manipulació de dades.

Llenguatges de Consulta Similars a SQL en Bases de Dades NoSQL

MongoDB

MongoDB és una base de dades NoSQL popular basada en documents. Utilitza un llenguatge de consulta anomenat MongoDB Query Language (MQL), que té algunes similituds amb SQL.

Exemple

// SQL: SELECT * FROM users WHERE age > 25;
db.users.find({ age: { $gt: 25 } });
  • db.users.find és l'equivalent de MongoDB a la instrucció SELECT de SQL.
  • { age: { $gt: 25 } } és la sintaxi de MongoDB per filtrar documents on el camp edat és superior a 25.

Cassandra

Cassandra és una base de dades NoSQL de família de columnes que utilitza un llenguatge de consulta anomenat Cassandra Query Language (CQL), que és molt similar a SQL.

Exemple

-- SQL: SELECT * FROM users WHERE age > 25;
SELECT * FROM users WHERE age > 25;
  • La consulta CQL és gairebé idèntica a la consulta SQL, facilitant l'adaptació dels usuaris de SQL.

Couchbase

Couchbase és una base de dades NoSQL basada en documents que ofereix un llenguatge de consulta anomenat N1QL, que és molt similar a SQL.

Exemple

-- SQL: SELECT * FROM users WHERE age > 25;
SELECT * FROM users WHERE age > 25;
  • Les consultes N1QL estan dissenyades per ser tan properes a SQL com sigui possible, proporcionant una transició suau per als usuaris de SQL.

Exercicis Pràctics

Exercici 1: Consulta MongoDB

Converteix la següent consulta SQL en una consulta MongoDB:

SELECT name, age FROM users WHERE age < 30;

Solució

db.users.find({ age: { $lt: 30 } }, { name: 1, age: 1, _id: 0 });

Exercici 2: Consulta Cassandra

Escriu una consulta CQL per recuperar tots els registres de la taula employees on el department sigui 'HR'.

Solució

SELECT * FROM employees WHERE department = 'HR';

Exercici 3: Consulta Couchbase

Converteix la següent consulta SQL en una consulta N1QL:

SELECT title, author FROM books WHERE year_published > 2000;

Solució

SELECT title, author FROM books WHERE year_published > 2000;

Conclusió

Utilitzar SQL amb bases de dades NoSQL permet als desenvolupadors aprofitar les seves habilitats existents en SQL mentre aprofiten l'escalabilitat i flexibilitat de les bases de dades NoSQL. Comprenent els llenguatges de consulta similars a SQL proporcionats per les bases de dades NoSQL, els desenvolupadors poden consultar i manipular dades de manera eficient a través de diferents tipus de bases de dades. Aquesta integració amplia l'abast de la gestió de dades i obre noves possibilitats per manejar conjunts de dades diversos.
