Introducció a UPSERT (MERGE)

La instrucció UPSERT (MERGE) és una potent comanda SQL que et permet inserir nous registres o actualitzar registres existents dins d'una taula. Això és particularment útil quan necessites sincronitzar dades entre taules o quan vols assegurar-te que les dades estan actualitzades sense haver de realitzar operacions INSERT i UPDATE per separat.

Conceptes Clau

  • UPSERT: Una combinació de "UPDATE" i "INSERT". Assegura que si un registre existeix, s'actualitza; si no existeix, s'insereix.
  • MERGE: La comanda estàndard SQL que realitza l'operació UPSERT. Combina la funcionalitat d'INSERT i UPDATE en una sola instrucció.

Sintaxi Bàsica

La sintaxi bàsica de la instrucció MERGE és la següent:

MERGE INTO target_table AS target
USING source_table AS source
ON target.matching_column = source.matching_column
WHEN MATCHED THEN
    UPDATE SET target.column1 = source.column1, target.column2 = source.column2
WHEN NOT MATCHED THEN
    INSERT (column1, column2) VALUES (source.column1, source.column2);

Explicació

  • MERGE INTO target_table: Especifica la taula objectiu on les dades seran inserides o actualitzades.
  • USING source_table: Especifica la taula font que conté les noves dades.
  • ON target.matching_column = source.matching_column: Defineix la condició per a coincidir registres entre les taules objectiu i font.
  • WHEN MATCHED THEN UPDATE: Especifica l'acció a prendre quan es troba una coincidència.
  • WHEN NOT MATCHED THEN INSERT: Especifica l'acció a prendre quan no es troba una coincidència.

Exemple

Considera dues taules: employees (objectiu) i new_employees (font).

Estructures de les Taules

employees

| employee_id | nom | departament | |-------------|---------|-------------| | 1 | Alice | HR | | 2 | Bob | IT | | 3 | Charlie | Finance |

new_employees

| employee_id | nom | departament | |-------------|---------|-------------| | 2 | Bob | Marketing | | 4 | David | IT |

Instrucció MERGE

MERGE INTO employees AS e
USING new_employees AS ne
ON e.employee_id = ne.employee_id
WHEN MATCHED THEN
    UPDATE SET e.name = ne.name, e.department = ne.department
WHEN NOT MATCHED THEN
    INSERT (employee_id, name, department) VALUES (ne.employee_id, ne.name, ne.department);

Resultat

Després d'executar la instrucció MERGE anterior, la taula employees s'actualitzarà de la següent manera:

| employee_id | nom | departament | |-------------|---------|-------------| | 1 | Alice | HR | | 2 | Bob | Marketing | | 3 | Charlie | Finance | | 4 | David | IT |

Ús Avançat

Maneig de Múltiples Condicions

Pots manejar múltiples condicions dins de la instrucció MERGE per realitzar diferents accions basades en diversos criteris.

MERGE INTO employees AS e
USING new_employees AS ne
ON e.employee_id = ne.employee_id
WHEN MATCHED AND e.department != ne.department THEN
    UPDATE SET e.department = ne.department
WHEN MATCHED AND e.department = ne.department THEN
    UPDATE SET e.name = ne.name
WHEN NOT MATCHED THEN
    INSERT (employee_id, name, department) VALUES (ne.employee_id, ne.name, ne.department);

Ús de Subconsultes

També pots utilitzar subconsultes en la clàusula USING per generar dinàmicament les dades font.

MERGE INTO employees AS e
USING (SELECT employee_id, name, department FROM new_employees WHERE department = 'IT') AS ne
ON e.employee_id = ne.employee_id
WHEN MATCHED THEN
    UPDATE SET e.name = ne.name, e.department = ne.department
WHEN NOT MATCHED THEN
    INSERT (employee_id, name, department) VALUES (ne.employee_id, ne.name, ne.department);

Conclusió

La instrucció UPSERT (MERGE) és una manera versàtil i eficient de gestionar la sincronització de dades i assegurar la integritat de les dades en bases de dades SQL. Combinant la funcionalitat de les operacions INSERT i UPDATE, simplifica el procés de mantenir registres actualitzats. Entendre i utilitzar la instrucció MERGE pot millorar significativament les teves habilitats de programació SQL i agilitzar les teves tasques de gestió de bases de dades.
