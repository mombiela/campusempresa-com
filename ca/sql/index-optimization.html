Introducció als Índexs

Els índexs són essencials per optimitzar el rendiment de les consultes SQL. Permeten que la base de dades trobi i recuperi files específiques molt més ràpidament del que podria sense un índex.

Conceptes Clau

  • Índex: Un objecte de base de dades que millora la velocitat de les operacions de recuperació de dades en una taula.
  • Clau Primària: Crea automàticament un índex únic a la columna de la clau primària.
  • Índex Únic: Assegura que els valors a la(s) columna(es) indexada(es) siguin únics.
  • Índex No Únic: Permet valors duplicats a la(s) columna(es) indexada(es).
  • Índex Agrupat: Ordena i emmagatzema les files de dades a la taula basant-se en els seus valors de clau.
  • Índex No Agrupat: Conté una còpia de part de les dades de la taula i un punter a les dades reals.

Creació d'Índexs

Crear índexs és senzill però requereix entendre les dades i les consultes per optimitzar el rendiment de manera efectiva.

Sintaxi

CREATE INDEX nom_index
ON nom_taula (columna1, columna2, ...);

Exemple

CREATE INDEX idx_client_cognom
ON clients (cognom);

Això crea un índex a la columna cognom de la taula clients, accelerant les consultes que filtren o ordenen per cognom.

Tipus d'Índexs

Diferents tipus d'índexs serveixen per a diferents propòsits i tenen característiques úniques.

Índexs Agrupats vs. No Agrupats

| Característica | Índex Agrupat | Índex No Agrupat | |--------------------|------------------------------------------|------------------------------------------| | Emmagatzematge de Dades | Les files de dades s'emmagatzemen en l'ordre de l'índex | Separat de les files de dades | | Nombre per Taula | Un per taula | Múltiples per taula | | Cas d'Ús | Clau primària, columnes consultades freqüentment | Columnes utilitzades freqüentment en claus WHERE |

Índexs Únics vs. No Únics

| Característica | Índex Únic | Índex No Únic | |--------------------|------------------------------------------|------------------------------------------| | Unicitat | Assegura que tots els valors siguin únics | Permet valors duplicats | | Cas d'Ús | Columnes que han de tenir valors únics | Columnes utilitzades per a la recuperació ràpida de dades |

Estratègies d'Indexació

Estratègies d'indexació efectives poden millorar significativament el rendiment de les consultes.

Conceptes Clau

  • Indexació Selectiva: Indexar columnes que es fan servir freqüentment en claus WHERE i que tenen alta selectivitat.
  • Índexs Compostos: Indexar múltiples columnes juntes per optimitzar consultes que filtren en múltiples columnes.
  • Índexs de Cobertura: Incloure totes les columnes necessàries per a una consulta per evitar accedir a les dades de la taula.

Exemple

CREATE INDEX idx_comanda_client_data
ON comandes (id_client, data_comanda);

Aquest índex compost optimitza les consultes que filtren tant per id_client com per data_comanda.

Manteniment d'Índexs

Els índexs requereixen manteniment per assegurar que continuïn proporcionant beneficis de rendiment.

Conceptes Clau

  • Reconstrucció d'Índexs: Reconstruir periòdicament els índexs per desfragmentar-los i optimitzar-los.
  • Eliminació d'Índexs No Utilitzats: Eliminar índexs que no són utilitzats per les consultes per estalviar espai d'emmagatzematge i millorar el rendiment.

Exemple

-- Reconstruir un índex
ALTER INDEX idx_client_cognom
ON clients REBUILD;

-- Eliminar un índex no utilitzat
DROP INDEX idx_antic_index
ON comandes;

Monitorització i Anàlisi d'Índexs

Monitoritzar i analitzar regularment els índexs per assegurar que s'utilitzen de manera efectiva.

Conceptes Clau

  • Estadístiques d'Ús d'Índexs: Fer un seguiment de la freqüència amb què els índexs són utilitzats per les consultes.
  • Plans d'Execució: Analitzar els plans d'execució de les consultes per entendre com s'utilitzen els índexs.

Exemple

-- Veure estadístiques d'ús d'índexs
SELECT * FROM sys.dm_db_index_usage_stats
WHERE object_id = OBJECT_ID('clients');

-- Analitzar un pla d'execució de consulta
EXPLAIN ANALYZE
SELECT * FROM clients
WHERE cognom = 'Smith';

Conclusió

L'optimització d'índexs és un aspecte crític de l'ajust del rendiment SQL. Entenent els diferents tipus d'índexs, creant estratègies d'indexació efectives, mantenint els índexs i monitoritzant-ne l'ús, es pot millorar significativament el rendiment de les consultes SQL. Revisa i ajusta regularment les teves estratègies d'indexació basant-te en les necessitats canviants de la teva base de dades i consultes.
