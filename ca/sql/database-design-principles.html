Introducció al Disseny de Bases de Dades

El disseny de bases de dades és un aspecte crucial per crear bases de dades eficients i escalables. Implica estructurar una base de dades de manera que es redueixi la redundància i es garanteixi la integritat de les dades. Aquesta secció cobrirà els principis fonamentals del disseny de bases de dades.

Conceptes Clau

  • Normalització: El procés d'organitzar les dades per minimitzar la redundància.
  • Desnormalització: El procés de combinar taules per millorar el rendiment de lectura.
  • Claus Primàries: Identificadors únics per als registres en una taula.
  • Claus Foranes: Camps que creen una relació entre dues taules.
  • Índexs: Estructures que milloren la velocitat de recuperació de dades.

Normalització

La normalització implica dividir una base de dades en dues o més taules i definir relacions entre les taules. L'objectiu és reduir la redundància i la dependència.

Formes Normals

  • Primera Forma Normal (1NF): Garanteix que cada columna contingui valors atòmics (indivisibles) i que cada registre sigui únic.
  • Segona Forma Normal (2NF): S'aconsegueix quan la taula està en 1NF i tots els atributs no clau són completament dependents funcionalment de la clau primària.
  • Tercera Forma Normal (3NF): S'aconsegueix quan la taula està en 2NF i tots els atributs són funcionalment dependents només de la clau primària.

Exemple

Considerem una taula que emmagatzema informació sobre estudiants i els seus cursos:

CREATE TABLE StudentCourses (
    StudentID INT,
    StudentName VARCHAR(100),
    CourseID INT,
    CourseName VARCHAR(100),
    InstructorName VARCHAR(100)
);

Per normalitzar aquesta taula a 1NF, assegurem que cada columna contingui valors atòmics. Per passar a 2NF i 3NF, dividim la taula en múltiples taules:

CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    StudentName VARCHAR(100)
);

CREATE TABLE Courses (
    CourseID INT PRIMARY KEY,
    CourseName VARCHAR(100),
    InstructorName VARCHAR(100)
);

CREATE TABLE StudentCourses (
    StudentID INT,
    CourseID INT,
    PRIMARY KEY (StudentID, CourseID),
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
    FOREIGN KEY (CourseID) REFERENCES Courses(CourseID)
);

Desnormalització

La desnormalització és el procés de combinar taules per reduir la complexitat de les consultes i millorar el rendiment de lectura. Sovint s'utilitza en sistemes de magatzem de dades i OLAP.

Exemple

Si necessitem consultar freqüentment els noms dels estudiants juntament amb els noms dels seus cursos i els noms dels instructors, podríem desnormalitzar les taules:

CREATE TABLE StudentCourseDetails (
    StudentID INT,
    StudentName VARCHAR(100),
    CourseID INT,
    CourseName VARCHAR(100),
    InstructorName VARCHAR(100),
    PRIMARY KEY (StudentID, CourseID)
);

Claus Primàries i Foranes

Les claus primàries identifiquen de manera única els registres en una taula, mentre que les claus foranes estableixen relacions entre taules.

Exemple

En les taules normalitzades anteriors, StudentID és la clau primària en la taula Students, i CourseID és la clau primària en la taula Courses. La taula StudentCourses utilitza tant StudentID com CourseID com a claus foranes per crear relacions.

CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    StudentName VARCHAR(100)
);

CREATE TABLE Courses (
    CourseID INT PRIMARY KEY,
    CourseName VARCHAR(100),
    InstructorName VARCHAR(100)
);

CREATE TABLE StudentCourses (
    StudentID INT,
    CourseID INT,
    PRIMARY KEY (StudentID, CourseID),
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
    FOREIGN KEY (CourseID) REFERENCES Courses(CourseID)
);

Índexs

Els índexs s'utilitzen per accelerar la recuperació de dades. Es poden crear en una o més columnes d'una taula.

Exemple

Crear un índex en la columna StudentName en la taula Students:

CREATE INDEX idx_student_name ON Students(StudentName);

Aquest índex farà que les consultes que filtren per StudentName siguin més ràpides.

Conclusió

Entendre i aplicar els principis de disseny de bases de dades és essencial per crear bases de dades eficients i escalables. La normalització ajuda a reduir la redundància i mantenir la integritat de les dades, mentre que la desnormalització pot millorar el rendiment de lectura en certs escenaris. L'ús adequat de claus primàries, claus foranes i índexs garanteix que la base de dades sigui tant relacional com eficient. En dominar aquests principis, pots dissenyar bases de dades que siguin robustes, eficients i fàcils de mantenir.
