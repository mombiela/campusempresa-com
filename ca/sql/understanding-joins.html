Introducció

En SQL, les unions s'utilitzen per combinar files de dues o més taules basades en una columna relacionada entre elles. Comprendre les unions és crucial per consultar bases de dades relacionals de manera efectiva.

Tipus d'unions

Unió interna

Una unió interna retorna només les files que tenen valors coincidents en ambdues taules.

Sintaxi:

SELECT columns
FROM table1
INNER JOIN table2
ON table1.common_column = table2.common_column;

Exemple:

Considerem dues taules, employees i departments:

employees | employee_id | name | department_id | |-------------|--------|---------------| | 1 | Alice | 1 | | 2 | Bob | 2 | | 3 | Charlie| 3 |

departments | department_id | department_name | |---------------|-----------------| | 1 | HR | | 2 | IT | | 4 | Marketing |

SELECT employees.name, departments.department_name
FROM employees
INNER JOIN departments
ON employees.department_id = departments.department_id;

Resultat: | name | department_name | |--------|-----------------| | Alice | HR | | Bob | IT |

Unió esquerra (externa)

Una unió esquerra retorna totes les files de la taula esquerra, i les files coincidents de la taula dreta. Si no es troba cap coincidència, es retornen valors NULL per a les columnes de la taula dreta.

Sintaxi:

SELECT columns
FROM table1
LEFT JOIN table2
ON table1.common_column = table2.common_column;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
LEFT JOIN departments
ON employees.department_id = departments.department_id;

Resultat: | name | department_name | |---------|-----------------| | Alice | HR | | Bob | IT | | Charlie | NULL |

Unió dreta (externa)

Una unió dreta retorna totes les files de la taula dreta, i les files coincidents de la taula esquerra. Si no es troba cap coincidència, es retornen valors NULL per a les columnes de la taula esquerra.

Sintaxi:

SELECT columns
FROM table1
RIGHT JOIN table2
ON table1.common_column = table2.common_column;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
RIGHT JOIN departments
ON employees.department_id = departments.department_id;

Resultat: | name | department_name | |--------|-----------------| | Alice | HR | | Bob | IT | | NULL | Marketing |

Unió completa (externa)

Una unió completa retorna totes les files quan hi ha una coincidència en qualsevol de les taules esquerra o dreta. Si no hi ha coincidència, el resultat és NULL en el costat que no té coincidència.

Sintaxi:

SELECT columns
FROM table1
FULL OUTER JOIN table2
ON table1.common_column = table2.common_column;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
FULL OUTER JOIN departments
ON employees.department_id = departments.department_id;

Resultat: | name | department_name | |---------|-----------------| | Alice | HR | | Bob | IT | | Charlie | NULL | | NULL | Marketing |

Unió creuada

Una unió creuada retorna el producte cartesià de les dues taules, és a dir, retorna totes les combinacions possibles de files de les taules.

Sintaxi:

SELECT columns
FROM table1
CROSS JOIN table2;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
CROSS JOIN departments;

Resultat: | name | department_name | |---------|-----------------| | Alice | HR | | Alice | IT | | Alice | Marketing | | Bob | HR | | Bob | IT | | Bob | Marketing | | Charlie | HR | | Charlie | IT | | Charlie | Marketing |

Conclusió

Les unions són un aspecte fonamental de SQL que et permeten combinar dades de múltiples taules basades en columnes relacionades. Comprendre els diferents tipus d'unions—interna, esquerra, dreta, completa i creuada—et permetrà escriure consultes més complexes i eficients. Practica utilitzant aquestes unions amb diversos conjunts de dades per convertir-te en un expert en consultes SQL.
