Introducció a les Subqueries

Les subqueries, també conegudes com a consultes internes o consultes niades, són consultes dins d'una altra consulta SQL. S'utilitzen per realitzar operacions que requereixen múltiples passos, com ara filtrar, agregar o unir dades de manera més complexa.

Conceptes Clau

  • Subquery: Una consulta niada dins d'una altra consulta.
  • Consulta Externa: La consulta principal que conté la subquery.
  • Subquery d'una sola fila: Retorna una sola fila de resultats.
  • Subquery de múltiples files: Retorna múltiples files de resultats.
  • Subquery Correlacionada: Una subquery que fa referència a columnes de la consulta externa.

Tipus de Subqueries

Subqueries d'una sola fila

Les subqueries d'una sola fila retornen una fila de resultats i sovint s'utilitzen amb operadors de comparació com =, >, <, >=, <=, i <>.

Exemple

SELECT employee_id, first_name, last_name
FROM employees
WHERE department_id = (
    SELECT department_id
    FROM departments
    WHERE department_name = 'Sales'
);
  • Explicació: Aquesta consulta recupera l'employee_id, first_name, i last_name dels empleats que treballen al departament de 'Sales'. La subquery troba l'department_id per a 'Sales', i la consulta externa utilitza aquest department_id per filtrar els empleats.

Subqueries de múltiples files

Les subqueries de múltiples files retornen més d'una fila i sovint s'utilitzen amb operadors com IN, ANY, o ALL.

Exemple

SELECT employee_id, first_name, last_name
FROM employees
WHERE department_id IN (
    SELECT department_id
    FROM departments
    WHERE location_id = 1700
);
  • Explicació: Aquesta consulta recupera l'employee_id, first_name, i last_name dels empleats que treballen en departaments situats a location_id 1700. La subquery retorna tots els department_ids a la ubicació 1700, i la consulta externa filtra els empleats basant-se en aquests department_ids.

Subqueries Correlacionades

Les subqueries correlacionades fan referència a columnes de la consulta externa i s'avaluen una vegada per cada fila processada per la consulta externa.

Exemple

SELECT employee_id, first_name, last_name
FROM employees e
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department_id = e.department_id
);
  • Explicació: Aquesta consulta recupera l'employee_id, first_name, i last_name dels empleats el salari dels quals és superior a la mitjana salarial del seu departament. La subquery calcula la mitjana salarial per a cada departament, i la consulta externa compara el salari de cada empleat amb aquesta mitjana.

Exercicis Pràctics

Exercici 1: Subquery d'una sola fila

Escriu una consulta per trobar el nom de l'empleat que té el salari més alt.

SELECT first_name, last_name
FROM employees
WHERE salary = (
    SELECT MAX(salary)
    FROM employees
);

Exercici 2: Subquery de múltiples files

Escriu una consulta per trobar els noms dels empleats que treballen al mateix departament que 'John Doe'.

SELECT first_name, last_name
FROM employees
WHERE department_id = (
    SELECT department_id
    FROM employees
    WHERE first_name = 'John' AND last_name = 'Doe'
);

Exercici 3: Subquery Correlacionada

Escriu una consulta per trobar els noms dels empleats que guanyen més que la mitjana salarial del seu departament.

SELECT first_name, last_name
FROM employees e
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department_id = e.department_id
);

Conclusió

Les subqueries són eines poderoses en SQL que permeten una recuperació de dades més complexa i flexible. Es poden utilitzar per realitzar operacions que serien difícils o impossibles amb una sola consulta. Entendre com utilitzar subqueries d'una sola fila, de múltiples files i correlacionades millorarà la teva capacitat per escriure consultes SQL eficients i efectives. Practica utilitzant subqueries en diferents escenaris per convertir-te en un expert en la seva aplicació.
