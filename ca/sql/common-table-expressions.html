Introducció a les CTEs

Les Expressions de Taula Comunes (CTEs) són una característica poderosa en SQL que permeten crear conjunts de resultats temporals que es poden referenciar dins d'una instrucció SELECT, INSERT, UPDATE o DELETE. Poden fer que les consultes complexes siguin més fàcils de llegir i mantenir.

  • Definició: Una CTE és un conjunt de resultats temporal que es defineix dins de l'àmbit d'execució d'una sola instrucció SELECT, INSERT, UPDATE o DELETE.
  • Sintaxi: La sintaxi bàsica per a una CTE és la següent: 
    WITH cte_name AS (
    -- Consulta CTE
    SELECT column1, column2
    FROM table_name
    WHERE condition
)
-- Consulta principal
SELECT column1, column2
FROM cte_name;

Beneficis d'Utilitzar CTEs

  • Millora de la Llegibilitat: Les CTEs poden descompondre consultes complexes en parts més senzilles.
  • Reutilització: Pots referenciar la CTE múltiples vegades dins de la consulta principal.
  • Recursió: Les CTEs suporten consultes recursives, que són útils per a dades jeràrquiques.

Exemple Bàsic d'una CTE

Comencem amb un exemple senzill per entendre com funcionen les CTEs.

Exemple: Seleccionar Empleats d'un Departament

Suposem que tenim una taula employees i volem seleccionar empleats del departament de 'Sales'.

WITH SalesEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Sales'
)
SELECT employee_id, first_name, last_name
FROM SalesEmployees;
  • Explicació:
    • La CTE SalesEmployees es defineix per seleccionar empleats del departament de 'Sales'.
    • La consulta principal després selecciona de la CTE SalesEmployees.

Ús Avançat de les CTEs

Múltiples CTEs

Pots definir múltiples CTEs en una sola consulta separant-les amb comes.

WITH SalesEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Sales'
),
MarketingEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Marketing'
)
SELECT * FROM SalesEmployees
UNION ALL
SELECT * FROM MarketingEmployees;

CTEs Recursives

Les CTEs recursives són útils per a dades jeràrquiques, com ara organigrames o estructures d'arbre.

Exemple: Jerarquia Organitzativa

Suposem que tenim una taula employees amb una columna manager_id que fa referència al gestor de l'empleat.

WITH RECURSIVE OrgChart AS (
    SELECT employee_id, first_name, last_name, manager_id
    FROM employees
    WHERE manager_id IS NULL
    UNION ALL
    SELECT e.employee_id, e.first_name, e.last_name, e.manager_id
    FROM employees e
    INNER JOIN OrgChart o ON e.manager_id = o.employee_id
)
SELECT * FROM OrgChart;
  • Explicació:
    • El cas base selecciona empleats sense gestor (manager_id IS NULL).
    • La part recursiva uneix la taula employees amb la CTE OrgChart per trobar empleats gestionats per aquells a OrgChart.

Exercicis Pràctics

  1. CTE Bàsica: Crea una CTE per seleccionar productes amb un preu superior a 100 $ d'una taula products.
  2. Múltiples CTEs: Crea dues CTEs per seleccionar empleats dels departaments de 'HR' i 'IT', després combina els resultats.
  3. CTE Recursiva: Crea una CTE recursiva per mostrar una jerarquia de categories d'una taula categories amb category_id i parent_category_id.

Conclusió

Les CTEs són una eina versàtil en SQL que poden simplificar consultes complexes, millorar la llegibilitat i gestionar dades jeràrquiques de manera eficient. En dominar les CTEs, pots escriure consultes SQL més mantenibles i eficients.

  • Punts Clau:
    • Les CTEs proporcionen una manera de crear conjunts de resultats temporals.
    • Milloren la llegibilitat i mantenibilitat de les consultes.
    • Les CTEs recursives són poderoses per treballar amb dades jeràrquiques.

Practica utilitzant CTEs en les teves consultes SQL per esdevenir més proficient i aprofitar tot el seu potencial.
