Introducció als Índexs

Els índexs són taules de cerca especials que el motor de cerca de la base de dades pot utilitzar per accelerar la recuperació de dades. S'utilitzen per localitzar ràpidament dades sense haver de cercar cada fila en una taula de base de dades cada vegada que s'accedeix a una taula de base de dades.

Conceptes Clau

  • Índex: Un objecte de base de dades que millora la velocitat de les operacions de recuperació de dades en una taula.
  • Índex de Clau Primària: Es crea automàticament quan es defineix una clau primària.
  • Índex Únic: Assegura que tots els valors en l'índex siguin únics.
  • Índex No Únic: Permet valors duplicats i s'utilitza per millorar el rendiment de les consultes.
  • Índex Agrupat: Ordena i emmagatzema les files de dades en la taula basant-se en la clau de l'índex.
  • Índex No Agrupat: Conté una llista ordenada de referències a les dades de la taula.

Creació d'Índexs

Crear un índex implica especificar la taula i la(s) columna(es) a indexar.

Sintaxi

CREATE INDEX nom_índex
ON nom_taula (columna1, columna2, ...);

Exemple

CREATE INDEX idx_cognom
ON empleats (cognom);

Això crea un índex anomenat idx_cognom a la columna cognom de la taula empleats.

Tipus d'Índexs

Índex de Clau Primària

Es crea automàticament quan es defineix una clau primària.

CREATE TABLE empleats (
    id INT PRIMARY KEY,
    nom VARCHAR(50),
    cognom VARCHAR(50)
);

En aquest exemple, es crea automàticament un índex a la columna id.

Índex Únic

Assegura que tots els valors en l'índex siguin únics.

CREATE UNIQUE INDEX idx_unique_email
ON empleats (email);

Això crea un índex únic a la columna email, assegurant que no hi hagi correus electrònics duplicats.

Índex No Únic

Permet valors duplicats i s'utilitza per millorar el rendiment de les consultes.

CREATE INDEX idx_departament
ON empleats (departament);

Això crea un índex no únic a la columna departament.

Índex Agrupat

Ordena i emmagatzema les files de dades en la taula basant-se en la clau de l'índex. Una taula només pot tenir un índex agrupat.

CREATE CLUSTERED INDEX idx_empleat_id
ON empleats (id);

Això crea un índex agrupat a la columna id.

Índex No Agrupat

Conté una llista ordenada de referències a les dades de la taula. Una taula pot tenir múltiples índexs no agrupats.

CREATE NONCLUSTERED INDEX idx_cognom_nom
ON empleats (cognom, nom);

Això crea un índex no agrupat a les columnes cognom i nom.

Gestió d'Índexs

Eliminar un Índex

Per eliminar un índex, utilitza l'instrucció DROP INDEX.

DROP INDEX idx_cognom;

Això elimina l'índex idx_cognom de la taula.

Reconstruir un Índex

Per reconstruir un índex, cosa que pot ser útil per desfragmentar, utilitza l'instrucció ALTER INDEX.

ALTER INDEX idx_cognom
ON empleats
REBUILD;

Això reconstrueix l'índex idx_cognom a la taula empleats.

Consideracions de Rendiment

  • Rendiment de Lectura: Els índexs milloren significativament el rendiment de lectura permetent que la base de dades trobi dades ràpidament.
  • Rendiment d'Escriptura: Els índexs poden alentir les operacions d'escriptura (INSERT, UPDATE, DELETE) perquè l'índex s'ha d'actualitzar cada vegada que es modifiquen les dades.
  • Emmagatzematge: Els índexs consumeixen espai addicional en disc.

Exemple: Rendiment de la Consulta

Sense Índex:

SELECT * FROM empleats WHERE cognom = 'Smith';

Amb Índex:

CREATE INDEX idx_cognom ON empleats (cognom);

SELECT * FROM empleats WHERE cognom = 'Smith';

La segona consulta s'executarà més ràpidament gràcies a l'índex a la columna cognom.

Conclusió

Els índexs són eines poderoses per optimitzar el rendiment de la base de dades, especialment per a operacions amb moltes lectures. Entendre els diferents tipus d'índexs i els seus casos d'ús adequats és crucial per a una gestió eficient de la base de dades. Tot i que els índexs poden accelerar significativament la recuperació de dades, tenen desavantatges en termes de rendiment d'escriptura i requisits d'emmagatzematge. Gestionar i mantenir adequadament els índexs és essencial per assegurar un rendiment òptim de la base de dades.
