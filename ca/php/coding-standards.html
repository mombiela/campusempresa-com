Introducció als Estàndards de Codificació

Els estàndards de codificació són un conjunt de directrius i millors pràctiques per escriure codi d'una manera consistent i llegible. Ajuden a assegurar que el codi sigui mantenible, escalable i comprensible per altres desenvolupadors. En PHP, adherir-se als estàndards de codificació és crucial per a projectes col·laboratius i manteniment a llarg termini.

Per què Importen els Estàndards de Codificació

  • Llegibilitat: Fa que el codi sigui més fàcil de llegir i entendre.
  • Mantenibilitat: Simplifica el procés d'actualització i correcció del codi.
  • Col·laboració: Assegura consistència entre el codi de diferents desenvolupadors.
  • Reducció d'Errors: Ajuda a identificar i evitar errors comuns.

Estàndards de Codificació en PHP

Estàndards PSR

El PHP-FIG (Framework Interoperability Group) ha definit diverses Recomanacions d'Estàndards PHP (PSRs) que són àmpliament adoptades en la comunitat PHP.

PSR-1: Estàndard Bàsic de Codificació

  • Fitxers: El codi PHP HA D'UTILITZAR l'etiqueta d'obertura <?php i NO HA D'UTILITZAR l'etiqueta de tancament ?>.
  • Codificació de Caràcters: El codi PHP HA D'UTILITZAR només UTF-8 sense BOM.
  • Espais de Noms i Classes: Els noms de les classes HAN DE ser declarats en StudlyCaps. Les constants de les classes HAN DE ser declarades en majúscules amb separadors de guió baix.
  • Mètodes: Els noms dels mètodes HAN DE ser declarats en camelCase.
<?php

namespace MyProject;

class UserProfile
{
    const MAX_LOGIN_ATTEMPTS = 5;

    public function getUserName()
    {
        // Implementació del mètode
    }
}

PSR-2: Guia d'Estil de Codificació

  • Indentació: El codi HA D'UTILITZAR 4 espais per a la indentació, no tabuladors.
  • Longitud de Línia: Les línies NO HAURIEN de ser més llargues de 80 caràcters; 120 caràcters és acceptable.
  • Estructures de Control: Les paraules clau de les estructures de control HAN DE tenir un espai després d'elles; les crides a mètodes i funcions NO HAN DE.
if ($condition) {
    // Bloc de codi
} elseif ($anotherCondition) {
    // Un altre bloc de codi
} else {
    // Bloc else
}

PSR-4: Estàndard d'Autocarregament

  • Espai de Noms: Els espais de noms i els noms de les classes HAN DE seguir l'estructura de directoris.
  • Autocarregament: Les classes HAN DE ser autocarregades segons l'estàndard PSR-4.
// Estructura de directoris
// src/
// └── MyProject/
//     └── UserProfile.php

// Exemple d'autocarregament
spl_autoload_register(function ($class) {
    $prefix = 'MyProject\\';
    $base_dir = __DIR__ . '/src/';
    $len = strlen($prefix);
    if (strncmp($prefix, $class, $len) !== 0) {
        return;
    }
    $relative_class = substr($class, $len);
    $file = $base_dir . str_replace('\\', '/', $relative_class) . '.php';
    if (file_exists($file)) {
        require $file;
    }
});

Millors Pràctiques

Convencions de Nomenclatura

  • Variables: Utilitza noms descriptius en camelCase.
  • Funcions: Utilitza noms descriptius en camelCase.
  • Classes: Utilitza StudlyCaps per als noms de les classes.
  • Constants: Utilitza UPPER_CASE amb guions baixos.
<?php

class UserProfile
{
    private $userName;
    private $userEmail;

    public function setUserName($name)
    {
        $this->userName = $name;
    }

    public function getUserName()
    {
        return $this->userName;
    }
}

Comentaris i Documentació

  • Comentaris en Línia: Utilitza // per a comentaris d'una sola línia i /* ... */ per a comentaris de múltiples línies.
  • DocBlocks: Utilitza PHPDoc per documentar funcions, classes i mètodes.
<?php

/**
 * Classe UserProfile
 * 
 * Aquesta classe gestiona les operacions del perfil d'usuari.
 */
class UserProfile
{
    /**
     * @var string $userName El nom de l'usuari.
     */
    private $userName;

    /**
     * Estableix el nom de l'usuari.
     *
     * @param string $name El nom de l'usuari.
     */
    public function setUserName($name)
    {
        $this->userName = $name;
    }

    /**
     * Obté el nom de l'usuari.
     *
     * @return string El nom de l'usuari.
     */
    public function getUserName()
    {
        return $this->userName;
    }
}

Eines per Enforçar els Estàndards de Codificació

  • PHP_CodeSniffer: Una eina que detecta violacions d'un estàndard de codificació definit.
  • PHP-CS-Fixer: Una eina que corregeix automàticament problemes d'estàndards de codificació.
  • PHPStan: Una eina d'anàlisi estàtica que ajuda a trobar errors en el teu codi.

Conclusió

Adherir-se als estàndards de codificació en PHP és essencial per escriure codi net, mantenible i escalable. Seguint els estàndards PSR i les millors pràctiques, pots assegurar que el teu codi sigui consistent i fàcil d'entendre, cosa especialment important en entorns col·laboratius. Utilitza eines com PHP_CodeSniffer i PHP-CS-Fixer per enforçar aquests estàndards i mantenir un codi d'alta qualitat.
