Gestió de Dades de Formularis

La gestió de dades de formularis és un aspecte crucial del desenvolupament web en PHP. Aquest tema et guiarà a través dels conceptes bàsics de captura i processament de dades de formularis, així com tècniques més avançades per a la validació i seguretat.

Introducció a la Gestió de Formularis

Els formularis són una part fonamental de les aplicacions web, permetent als usuaris enviar dades que poden ser processades pel servidor. PHP proporciona diverses maneres de gestionar les dades de formularis de manera eficient.

Conceptes Clau

  • Elements de Formulari: Elements HTML com <input>, <textarea> i <select> que recullen l'entrada de l'usuari.
  • Mètodes de Formulari: Mètodes GET i POST utilitzats per enviar dades de formularis al servidor.
  • Superglobals: Arrays de PHP com $_GET i $_POST que emmagatzemen dades de formularis.

Creació d'un Formulari Simple

Comencem amb un formulari HTML bàsic que utilitza el mètode POST per enviar dades a un script PHP.

<!DOCTYPE html>
<html>
<head>
    <title>Formulari Simple</title>
</head>
<body>
    <form action="process_form.php" method="post">
        <label for="name">Nom:</label>
        <input type="text" id="name" name="name"><br><br>
        <label for="email">Correu electrònic:</label>
        <input type="email" id="email" name="email"><br><br>
        <input type="submit" value="Enviar">
    </form>
</body>
</html>

Explicació

  • action: Especifica el script PHP que processarà les dades del formulari.
  • method: Defineix com s'enviaran les dades del formulari (POST en aquest cas).
  • atribut name: Cada element d'entrada ha de tenir un atribut name per ser accessible en el script PHP.

Processament de Dades de Formularis en PHP

En el fitxer process_form.php, pots accedir a les dades del formulari utilitzant la superglobal $_POST.

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $name = $_POST['name'];
    $email = $_POST['email'];

    echo "Nom: " . htmlspecialchars($name) . "<br>";
    echo "Correu electrònic: " . htmlspecialchars($email) . "<br>";
}
?>

Explicació

  • $_SERVER["REQUEST_METHOD"]: Comprova si el formulari s'ha enviat utilitzant el mètode POST.
  • $_POST['name'] i $_POST['email']: Accedeix a les dades del formulari.
  • htmlspecialchars(): Converteix caràcters especials en entitats HTML per prevenir atacs XSS.

Validació de Dades de Formularis

La validació assegura que les dades enviades per l'usuari compleixin certs criteris abans de processar-les.

Exemple: Validació de Nom i Correu Electrònic

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $name = trim($_POST['name']);
    $email = trim($_POST['email']);
    $errors = [];

    if (empty($name)) {
        $errors[] = "El nom és obligatori.";
    }

    if (empty($email)) {
        $errors[] = "El correu electrònic és obligatori.";
    } elseif (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $errors[] = "Format de correu electrònic no vàlid.";
    }

    if (empty($errors)) {
        echo "Nom: " . htmlspecialchars($name) . "<br>";
        echo "Correu electrònic: " . htmlspecialchars($email) . "<br>";
    } else {
        foreach ($errors as $error) {
            echo $error . "<br>";
        }
    }
}
?>

Explicació

  • trim(): Elimina espais en blanc del començament i final d'una cadena.
  • filter_var(): Valida el format del correu electrònic.
  • Gestió d'Errors: Recull i mostra errors si la validació falla.

Tècniques Avançades de Gestió de Formularis

Gestió de Pujades d'Arxius

PHP et permet gestionar pujades d'arxius a través de formularis. Aquí tens un exemple bàsic:

<!DOCTYPE html>
<html>
<head>
    <title>Formulari de Pujada d'Arxius</title>
</head>
<body>
    <form action="upload.php" method="post" enctype="multipart/form-data">
        <label for="file">Tria un arxiu:</label>
        <input type="file" id="file" name="file"><br><br>
        <input type="submit" value="Pujar">
    </form>
</body>
</html>

Processament de Pujades d'Arxius

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $target_dir = "uploads/";
    $target_file = $target_dir . basename($_FILES["file"]["name"]);
    $uploadOk = 1;
    $fileType = strtolower(pathinfo($target_file, PATHINFO_EXTENSION));

    // Comprova si l'arxiu és una imatge real
    $check = getimagesize($_FILES["file"]["tmp_name"]);
    if ($check !== false) {
        echo "L'arxiu és una imatge - " . $check["mime"] . ".<br>";
        $uploadOk = 1;
    } else {
        echo "L'arxiu no és una imatge.<br>";
        $uploadOk = 0;
    }

    // Comprova si l'arxiu ja existeix
    if (file_exists($target_file)) {
        echo "Ho sentim, l'arxiu ja existeix.<br>";
        $uploadOk = 0;
    }

    // Comprova la mida de l'arxiu
    if ($_FILES["file"]["size"] > 500000) {
        echo "Ho sentim, el teu arxiu és massa gran.<br>";
        $uploadOk = 0;
    }

    // Permet només certs formats d'arxiu
    if ($fileType != "jpg" && $fileType != "png" && $fileType != "jpeg" && $fileType != "gif") {
        echo "Ho sentim, només es permeten arxius JPG, JPEG, PNG i GIF.<br>";
        $uploadOk = 0;
    }

    // Comprova si $uploadOk està establert a 0 per un error
    if ($uploadOk == 0) {
        echo "Ho sentim, el teu arxiu no s'ha pujat.<br>";
    // Si tot està bé, intenta pujar l'arxiu
    } else {
        if (move_uploaded_file($_FILES["file"]["tmp_name"], $target_file)) {
            echo "L'arxiu ". htmlspecialchars(basename($_FILES["file"]["name"])). " s'ha pujat.<br>";
        } else {
            echo "Ho sentim, hi ha hagut un error en pujar el teu arxiu.<br>";
        }
    }
}
?>

Explicació

  • $_FILES: Array superglobal utilitzat per gestionar pujades d'arxius.
  • getimagesize(): Comprova si l'arxiu és una imatge real.
  • move_uploaded_file(): Mou l'arxiu pujat al directori especificat.

Conclusió

La gestió de dades de formularis en PHP implica crear formularis HTML, capturar i processar les dades, validar les entrades i gestionar pujades d'arxius. En dominar aquestes tècniques, pots construir aplicacions web robustes i segures que interactuen efectivament amb els usuaris. Recorda sempre validar i sanejar les entrades dels usuaris per mantenir la seguretat i integritat de la teva aplicació.
