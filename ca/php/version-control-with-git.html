Introducció al Control de Versions

El control de versions és un sistema que registra els canvis en un fitxer o conjunt de fitxers al llarg del temps perquè puguis recuperar versions específiques més tard. Git és un dels sistemes de control de versions més populars utilitzats avui en dia.

  • Per què utilitzar el control de versions?
    • Fer un seguiment dels canvis al llarg del temps.
    • Col·laborar amb altres persones.
    • Revertir a versions anteriors.
    • Fer una còpia de seguretat del teu treball.

Començant amb Git

Abans de submergir-te en Git, assegura't de tenir-lo instal·lat al teu sistema. Pots descarregar-lo des de git-scm.com.

Comandes bàsiques de Git

Aquí tens algunes comandes fonamentals de Git per començar:

  • Inicialitzar un repositori:

    git init

    Aquesta comanda inicialitza un nou repositori Git al teu directori actual.

  • Clonar un repositori:

    git clone <repository_url>

    Aquesta comanda crea una còpia d'un repositori existent.

  • Comprovar l'estat del teu repositori:

    git status

    Aquesta comanda mostra l'estat dels canvis com a no seguits, modificats o preparats.

  • Afegir fitxers a l'àrea de preparació:

    git add <file_name>

    Aquesta comanda prepara fitxers específics per al següent commit. Utilitza git add . per preparar tots els canvis.

  • Fer commit dels canvis:

    git commit -m "El teu missatge de commit"

    Aquesta comanda fa commit dels canvis preparats al repositori amb un missatge.

  • Empènyer els canvis a un repositori remot:

    git push origin <branch_name>

    Aquesta comanda envia els teus commits locals al repositori remot.

  • Extreure canvis d'un repositori remot:

    git pull origin <branch_name>

    Aquesta comanda obté i fusiona els canvis del repositori remot al teu repositori local.

Creació de branques i fusió

La creació de branques és una característica poderosa de Git que et permet crear línies de desenvolupament separades.

Creació i gestió de branques

  • Crear una nova branca:

    git branch <branch_name>

    Aquesta comanda crea una nova branca.

  • Canviar a una branca:

    git checkout <branch_name>

    Aquesta comanda canvia a la branca especificada.

  • Crear i canviar a una nova branca:

    git checkout -b <branch_name>

    Aquesta comanda crea una nova branca i hi canvia immediatament.

Fusió de branques

  • Fusionar una branca a la branca actual: 
    git merge <branch_name>
    Aquesta comanda fusiona la branca especificada a la branca actual.

Gestió de conflictes de fusió

De vegades, pots trobar-te amb conflictes durant una fusió. Git marcarà els conflictes als fitxers afectats, i hauràs de resoldre'ls manualment.

  • Veure conflictes:

    git status

    Aquesta comanda et mostrarà quins fitxers tenen conflictes.

  • Després de resoldre els conflictes, afegeix els fitxers resolts:

    git add <file_name>

  • Fer commit de la fusió:

    git commit

Treballant amb repositoris remots

Els repositoris remots són versions del teu projecte que estan allotjades a internet o a la xarxa.

Afegir un repositori remot

  • Afegir un repositori remot: 
    git remote add origin <repository_url>
    Aquesta comanda afegeix un repositori remot.

Obtenir i extreure de repositoris remots

  • Obtenir canvis d'un repositori remot:

    git fetch origin

    Aquesta comanda obté canvis del repositori remot però no els fusiona.

  • Extreure canvis d'un repositori remot:

    git pull origin <branch_name>

    Aquesta comanda obté i fusiona canvis del repositori remot.

Empènyer a repositoris remots

  • Empènyer canvis a un repositori remot: 
    git push origin <branch_name>
    Aquesta comanda envia els teus commits locals al repositori remot.

Funcions avançades de Git

Emmagatzematge de canvis

L'emmagatzematge et permet guardar els teus canvis temporalment sense fer commit.

  • Emmagatzemar canvis:

    git stash

    Aquesta comanda emmagatzema els teus canvis.

  • Aplicar canvis emmagatzemats:

    git stash apply

    Aquesta comanda aplica els canvis emmagatzemats.

Rebase

El rebase és una manera d'integrar canvis d'una branca a una altra.

  • Fer rebase d'una branca: 
    git rebase <branch_name>
    Aquesta comanda fa rebase de la branca actual a la branca especificada.

Etiquetatge

Les etiquetes s'utilitzen per marcar punts específics en la història com a importants.

  • Crear una etiqueta:

    git tag <tag_name>

    Aquesta comanda crea una etiqueta.

  • Empènyer etiquetes a un repositori remot:

    git push origin --tags

    Aquesta comanda envia totes les etiquetes al repositori remot.

Conclusió

Git és una eina essencial per a qualsevol desenvolupador, proporcionant funcions poderoses per al control de versions i la col·laboració. En dominar Git, pots gestionar els teus projectes PHP de manera més efectiva, col·laborar amb altres persones i mantenir un historial del teu treball. Practica les comandes i conceptes discutits en aquest tema per convertir-te en un expert en l'ús de Git per al teu desenvolupament PHP.
