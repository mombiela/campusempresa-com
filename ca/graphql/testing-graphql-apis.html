Proves d'APIs GraphQL

Les proves són una part crucial del desenvolupament d'APIs GraphQL robustes i fiables. Aquesta secció et guiarà a través del procés de provar APIs GraphQL, des de conceptes bàsics fins a tècniques avançades.

Introducció a les Proves d'APIs GraphQL

Les proves asseguren que la teva API GraphQL es comporti com s'espera i ajuden a detectar errors en les primeres fases del procés de desenvolupament. Hi ha diversos tipus de proves que pots realitzar en una API GraphQL:

  • Proves Unitàries: Prova resolutors individuals i altres petites unitats de codi.
  • Proves d'Integració: Prova com diferents parts de la teva aplicació treballen juntes.
  • Proves de Cap a Cap (E2E): Prova tot el flux de l'aplicació des del client fins al servidor i de tornada.

Eines per a Provar APIs GraphQL

Diverses eines poden ajudar-te a provar APIs GraphQL de manera efectiva:

  • Jest: Un marc de proves de JavaScript que funciona bé amb GraphQL.
  • Utilitats de Prova d'Apollo Server: Eines proporcionades per Apollo Server per facilitar les proves.
  • GraphQL Playground: Un IDE interactiu en el navegador per a GraphQL que es pot utilitzar per a proves manuals.

Escriure Proves Unitàries per a Resolutors GraphQL

Les proves unitàries es concentren en resolutors individuals per assegurar-se que retornin els resultats esperats. Aquí tens com pots escriure proves unitàries utilitzant Jest.

Exemple: Provar un Resolutor amb Jest

Suposa que tens el següent resolutor:

const resolvers = {
  Query: {
    user: (parent, args, context, info) => {
      return context.dataSources.userAPI.getUserById(args.id);
    },
  },
};

Pots escriure una prova unitària per a aquest resolutor de la següent manera:

const { ApolloServer } = require('apollo-server');
const { createTestClient } = require('apollo-server-testing');
const gql = require('graphql-tag');

// Font de dades simulada
const mockUserAPI = {
  getUserById: jest.fn(),
};

// Crear un servidor de prova
const server = new ApolloServer({
  typeDefs,
  resolvers,
  dataSources: () => ({
    userAPI: mockUserAPI,
  }),
});

const { query } = createTestClient(server);

describe('Query.user', () => {
  it('recupera un usuari per ID', async () => {
    // Preparació
    const user = { id: '1', name: 'John Doe' };
    mockUserAPI.getUserById.mockResolvedValue(user);

    // Acció
    const res = await query({
      query: gql`
        query getUser($id: ID!) {
          user(id: $id) {
            id
            name
          }
        }
      `,
      variables: { id: '1' },
    });

    // Afirmació
    expect(res.data.user).toEqual(user);
  });
});

Escriure Proves d'Integració

Les proves d'integració asseguren que diferents parts de la teva aplicació treballin juntes correctament. Aquestes proves típicament impliquen configurar un servidor de prova i fer sol·licituds a aquest.

Exemple: Prova d'Integració amb Apollo Server

const { ApolloServer } = require('apollo-server');
const { createTestClient } = require('apollo-server-testing');
const gql = require('graphql-tag');

// Defineix el teu esquema i resolutors
const typeDefs = gql`
  type User {
    id: ID!
    name: String!
  }

  type Query {
    user(id: ID!): User
  }
`;

const resolvers = {
  Query: {
    user: (parent, args, context, info) => {
      return { id: args.id, name: 'John Doe' };
    },
  },
};

// Crear un servidor de prova
const server = new ApolloServer({
  typeDefs,
  resolvers,
});

const { query } = createTestClient(server);

describe('Prova d\'Integració', () => {
  it('recupera un usuari per ID', async () => {
    const res = await query({
      query: gql`
        query getUser($id: ID!) {
          user(id: $id) {
            id
            name
          }
        }
      `,
      variables: { id: '1' },
    });

    expect(res.data.user).toEqual({ id: '1', name: 'John Doe' });
  });
});

Escriure Proves de Cap a Cap (E2E)

Les proves de cap a cap simulen interaccions reals dels usuaris amb la teva aplicació. Aquestes proves són típicament més complexes i impliquen configurar tota l'estaca de l'aplicació.

Exemple: Prova E2E amb Cypress

Cypress és una eina popular per a proves E2E. Aquí tens un exemple bàsic de com podries utilitzar Cypress per provar una API GraphQL.

  1. Instal·la Cypress:

    npm install cypress --save-dev

  2. Escriu una Prova de Cypress:

    // cypress/integration/graphql_spec.js
describe('API GraphQL', () => {
  it('recupera un usuari per ID', () => {
    cy.request({
      method: 'POST',
      url: 'http://localhost:4000/graphql',
      body: {
        query: `
          query getUser($id: ID!) {
            user(id: $id) {
              id
              name
            }
          }
        `,
        variables: { id: '1' },
      },
    }).then((response) => {
      expect(response.body.data.user).to.have.property('id', '1');
      expect(response.body.data.user).to.have.property('name', 'John Doe');
    });
  });
});

Conclusió

Provar APIs GraphQL és essencial per assegurar la fiabilitat i correcció de la teva aplicació. Escriure proves unitàries, proves d'integració i proves de cap a cap et permet detectar errors d'hora i assegurar que la teva API es comporti com s'espera. Utilitzar eines com Jest, les Utilitats de Prova d'Apollo Server i Cypress pot fer que el procés de proves sigui més eficient i efectiu.
