Introducció a les Subscripcions GraphQL

Les Subscripcions GraphQL són una característica poderosa que permet als clients rebre actualitzacions en temps real des del servidor. A diferència de les consultes i mutacions, que són operacions d'una sola vegada, les subscripcions mantenen una connexió activa i envien actualitzacions al client sempre que es produeixen esdeveniments específics al servidor.

Conceptes Clau

  • Comunicació en temps real: Les subscripcions permeten la comunicació en temps real entre el client i el servidor.
  • WebSockets: Les subscripcions solen utilitzar WebSockets per mantenir una connexió persistent.
  • Mecanisme Pub/Sub: Les subscripcions sovint es basen en un mecanisme de publicació/subscripció per difondre actualitzacions als clients subscrits.

Configuració de les Subscripcions GraphQL

Per utilitzar les Subscripcions GraphQL, cal configurar un servidor WebSocket i configurar el servidor GraphQL per gestionar operacions de subscripció.

Configuració del Servidor

  1. Instal·lar Dependències:

    • graphql-subscriptions: Proporciona una implementació simple de pub/sub.
    • subscriptions-transport-ws: Transport WebSocket per a subscripcions GraphQL.
    npm install graphql-subscriptions subscriptions-transport-ws

  2. Configurar el Servidor WebSocket:

    • Configurar un servidor WebSocket per gestionar les sol·licituds de subscripció.
    const { createServer } = require('http');
const { execute, subscribe } = require('graphql');
const { SubscriptionServer } = require('subscriptions-transport-ws');
const { makeExecutableSchema } = require('@graphql-tools/schema');

const typeDefs = `
  type Query {
    _: Boolean
  }
  type Subscription {
    messageSent: String
  }
`;

const resolvers = {
  Subscription: {
    messageSent: {
      subscribe: (_, __, { pubsub }) => pubsub.asyncIterator('MESSAGE_SENT')
    }
  }
};

const schema = makeExecutableSchema({ typeDefs, resolvers });

const server = createServer((req, res) => {
  res.writeHead(404);
  res.end();
});

server.listen(4000, () => {
  new SubscriptionServer({
    execute,
    subscribe,
    schema
  }, {
    server,
    path: '/subscriptions',
  });
  console.log('El servidor WebSocket està funcionant a ws://localhost:4000/subscriptions');
});

  3. Publicar Esdeveniments:

    • Utilitzar un mecanisme pub/sub per publicar esdeveniments als quals els clients es poden subscriure.
    const { PubSub } = require('graphql-subscriptions');
const pubsub = new PubSub();

// Exemple de publicació d'un esdeveniment
setInterval(() => {
  pubsub.publish('MESSAGE_SENT', { messageSent: 'Hola Món!' });
}, 1000);

Configuració del Client

  1. Instal·lar Dependències:

    • subscriptions-transport-ws: Transport WebSocket per a subscripcions GraphQL.
    • apollo-client i apollo-link-ws: Apollo Client i enllaç WebSocket per gestionar subscripcions.
    npm install subscriptions-transport-ws @apollo/client

  2. Configurar Apollo Client:

    • Configurar Apollo Client per utilitzar l'enllaç WebSocket per a subscripcions.
    import { ApolloClient, InMemoryCache, split } from '@apollo/client';
import { WebSocketLink } from '@apollo/client/link/ws';
import { getMainDefinition } from '@apollo/client/utilities';

const wsLink = new WebSocketLink({
  uri: 'ws://localhost:4000/subscriptions',
  options: {
    reconnect: true
  }
});

const client = new ApolloClient({
  link: wsLink,
  cache: new InMemoryCache()
});

  3. Subscriure's a Esdeveniments:

    • Utilitzar Apollo Client per subscriure's a esdeveniments.
    import { gql, useSubscription } from '@apollo/client';

const MESSAGE_SENT_SUBSCRIPTION = gql`
  subscription OnMessageSent {
    messageSent
  }
`;

function Messages() {
  const { data, loading } = useSubscription(MESSAGE_SENT_SUBSCRIPTION);

  if (loading) return <p>Carregant...</p>;
  return <p>Nou missatge: {data.messageSent}</p>;
}

Conclusió

Les Subscripcions GraphQL proporcionen una manera robusta d'implementar funcions en temps real a les teves aplicacions. Aprofitant els WebSockets i un mecanisme pub/sub, pots enviar actualitzacions als clients tan aviat com es produeixin. Configurar subscripcions implica configurar tant el servidor com el client per gestionar connexions WebSocket i gestionar esdeveniments de subscripció. Amb aquest coneixement, ara pots implementar funcionalitats en temps real a les teves aplicacions GraphQL, millorant l'experiència de l'usuari amb actualitzacions en directe.
