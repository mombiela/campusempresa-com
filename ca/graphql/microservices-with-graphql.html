Introducció als Microserveis

  • Definició: Els microserveis són un estil arquitectònic que estructura una aplicació com una col·lecció de serveis poc acoblats.
  • Beneficis:
    • Escalabilitat
    • Flexibilitat
    • Desplegament independent
  • Reptes:
    • Complexitat en la gestió
    • Consistència de dades
    • Comunicació entre serveis

Introducció a GraphQL

  • Definició: GraphQL és un llenguatge de consultes per a APIs i un temps d'execució per executar aquestes consultes utilitzant un sistema de tipus que defineixes per a les teves dades.
  • Beneficis:
    • Obtenció eficient de dades
    • Esquema fortament tipat
    • Flexibilitat en les consultes

Per què utilitzar GraphQL amb microserveis?

  • Porta d'entrada API unificada: GraphQL pot actuar com un únic punt d'entrada per a múltiples microserveis.
  • Obtenció eficient de dades: Redueix la sobre-obtenció i la sota-obtenció de dades.
  • Unió d'esquemes: Combina múltiples esquemes GraphQL en un únic esquema.

Configuració d'un servidor bàsic de GraphQL

  • Dependències:
    • express
    • express-graphql
    • graphql
  • Codi d'exemple: 
    const express = require('express');
const { graphqlHTTP } = require('express-graphql');
const { buildSchema } = require('graphql');

// Construeix un esquema, utilitzant el llenguatge d'esquemes de GraphQL
const schema = buildSchema(`
  type Query {
    hello: String
  }
`);

// El root proporciona una funció resolutiva per a cada punt final de l'API
const root = {
  hello: () => {
    return 'Hello world!';
  },
};

const app = express();
app.use('/graphql', graphqlHTTP({
  schema: schema,
  rootValue: root,
  graphiql: true,
}));

app.listen(4000, () => console.log('Now browse to localhost:4000/graphql'));

Integració de microserveis amb GraphQL

  • Comunicació de serveis:
    • API REST
    • gRPC
    • Accés directe a la base de dades
  • Exemple: 
    const fetch = require('node-fetch');

const schema = buildSchema(`
  type Query {
    user(id: Int!): User
  }

  type User {
    id: Int
    name: String
    email: String
  }
`);

const root = {
  user: async ({ id }) => {
    const response = await fetch(`http://user-service/users/${id}`);
    return response.json();
  },
};

const app = express();
app.use('/graphql', graphqlHTTP({
  schema: schema,
  rootValue: root,
  graphiql: true,
}));

app.listen(4000, () => console.log('Now browse to localhost:4000/graphql'));

Unió d'esquemes i Federació

  • Unió d'esquemes: Combina múltiples esquemes GraphQL en un únic esquema.
  • Federació d'Apollo: Un enfocament més recent per a la unió d'esquemes que permet una arquitectura més modular i escalable.

Exemple d'unió d'esquemes

const { mergeSchemas } = require('graphql-tools');
const { makeExecutableSchema } = require('graphql-tools');

// Defineix esquemes per a diferents serveis
const userSchema = makeExecutableSchema({
  typeDefs: `
    type User {
      id: ID!
      name: String
    }
    type Query {
      user(id: ID!): User
    }
  `,
  resolvers: {
    Query: {
      user: () => ({ id: 1, name: 'John Doe' }),
    },
  },
});

const productSchema = makeExecutableSchema({
  typeDefs: `
    type Product {
      id: ID!
      name: String
    }
    type Query {
      product(id: ID!): Product
    }
  `,
  resolvers: {
    Query: {
      product: () => ({ id: 1, name: 'Laptop' }),
    },
  },
});

// Uneix esquemes
const schema = mergeSchemas({
  schemas: [userSchema, productSchema],
});

const app = express();
app.use('/graphql', graphqlHTTP({
  schema: schema,
  graphiql: true,
}));

app.listen(4000, () => console.log('Now browse to localhost:4000/graphql'));

Millors pràctiques

  • Modularització: Mantingues els teus serveis i esquemes modulars.
  • Gestió d'errors: Implementa mecanismes robustos de gestió d'errors.
  • Seguretat: Assegura la comunicació segura entre serveis i protegeix el teu punt final de GraphQL.
  • Rendiment: Optimitza les teves consultes i utilitza la memòria cau quan sigui apropiat.

Conclusió

Integrar GraphQL amb microserveis pot millorar significativament la flexibilitat i l'eficiència de la teva arquitectura API. Utilitzant GraphQL com una porta d'entrada API unificada, pots simplificar l'obtenció de dades i millorar l'experiència del desenvolupador. No obstant això, és essencial seguir les millors pràctiques per gestionar la complexitat, garantir la seguretat i mantenir el rendiment.
