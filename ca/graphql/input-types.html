Introducció

En GraphQL, els tipus d'entrada s'utilitzen per definir l'estructura de les dades que es poden enviar al servidor. Són crucials per crear mutacions i consultes que requereixen entrades complexes. Entendre els tipus d'entrada és essencial per construir APIs de GraphQL robustes i flexibles.

Conceptes clau

  • Tipus d'entrada: Tipus especials utilitzats per definir l'estructura de les dades d'entrada.
  • Escalars: Tipus de dades bàsics com Int, Float, String, Boolean i ID.
  • Tipus d'entrada personalitzats: Tipus definits per l'usuari que es poden utilitzar per encapsular estructures d'entrada complexes.
  • Modificadors de no nul i llista: Millores als tipus d'entrada per especificar camps requerits i arrays.

Escalars

Els escalars són la forma més simple de tipus d'entrada. Representen tipus de dades primitives.

Exemple

type Query {
  getUser(id: ID!): User
}

type User {
  id: ID!
  name: String!
  age: Int
}

En aquest exemple, ID, String i Int són tipus escalars.

Tipus d'entrada personalitzats

Els tipus d'entrada personalitzats permeten definir estructures més complexes per a les teves entrades. Es defineixen utilitzant la paraula clau input.

Exemple

input UserInput {
  name: String!
  age: Int
}

type Mutation {
  createUser(input: UserInput!): User
}

Aquí, UserInput és un tipus d'entrada personalitzat que encapsula els camps name i age.

Modificadors de no nul i llista

Els modificadors de no nul (!) i llista ([]) es poden utilitzar amb tipus d'entrada per fer complir camps requerits i arrays d'entrades.

Exemple

input UserInput {
  name: String!
  age: Int
  hobbies: [String!]
}

type Mutation {
  createUser(input: UserInput!): User
}

En aquest exemple:

  • name és un camp requerit.
  • age és opcional.
  • hobbies és una llista opcional de cadenes requerides.

Ús dels tipus d'entrada en mutacions

Els tipus d'entrada s'utilitzen més comunament en mutacions per passar dades estructurades al servidor.

Exemple

input PostInput {
  title: String!
  content: String!
  tags: [String!]
}

type Mutation {
  createPost(input: PostInput!): Post
}

type Post {
  id: ID!
  title: String!
  content: String!
  tags: [String!]
}

Aquí, PostInput s'utilitza per encapsular les dades necessàries per crear una nova publicació.

Comparació d'escalars i tipus d'entrada personalitzats

| Característica | Escalars | Tipus d'entrada personalitzats | |------------------|------------------------|-------------------------------| | Definició | Tipus primitius integrats | Estructures definides per l'usuari | | Complexitat | Simple | Pot ser complex | | Usos | Camps bàsics | Entrades complexes per a mutacions i consultes |

Conclusió

Els tipus d'entrada en GraphQL són essencials per definir l'estructura de les dades que es poden enviar al servidor. Els escalars proporcionen tipus de dades bàsics, mentre que els tipus d'entrada personalitzats permeten estructures més complexes. Entendre com utilitzar els modificadors de no nul i llista pot ajudar a fer complir la integritat de les dades i la flexibilitat en la teva API de GraphQL. Dominant els tipus d'entrada, pots crear esquemes de GraphQL més robustos i flexibles.
