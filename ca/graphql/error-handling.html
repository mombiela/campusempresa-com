La gestió d'errors és un aspecte crucial de qualsevol aplicació, i GraphQL no és una excepció. Aquest tema et guiarà a través dels conceptes bàsics de la gestió d'errors en GraphQL, des de la comprensió de les respostes d'error per defecte fins a la implementació de mecanismes de gestió d'errors personalitzats.

Introducció a la Gestió d'Errors en GraphQL

GraphQL té un mecanisme integrat per gestionar errors. Quan un servidor GraphQL troba un error mentre resol una consulta, retorna una matriu errors en la resposta.

Resposta d'Error per Defecte

Quan es produeix un error, la resposta GraphQL inclou una matriu errors al costat del camp data. Aquí tens un exemple:

query {
  user(id: "1") {
    name
    email
  }
}

Si l'usuari amb id: "1" no existeix, la resposta podria semblar així:

{
  "data": {
    "user": null
  },
  "errors": [
    {
      "message": "User not found",
      "locations": [
        {
          "line": 2,
          "column": 3
        }
      ],
      "path": ["user"]
    }
  ]
}
  • data: Conté les dades parcials que es van poder resoldre.
  • errors: Una matriu d'objectes d'error amb detalls sobre cada error.

Tipus d'Errors en GraphQL

Errors de Validació

Aquests errors es produeixen quan la consulta no es conforma amb l'esquema GraphQL.

Exemple:

query {
  user(id: "1") {
    nonExistentField
  }
}

Resposta:

{
  "errors": [
    {
      "message": "Cannot query field 'nonExistentField' on type 'User'.",
      "locations": [
        {
          "line": 3,
          "column": 5
        }
      ]
    }
  ]
}

Errors d'Execució

Aquests errors es produeixen durant l'execució de la consulta, com quan un resolutor llança una excepció.

Exemple:

query {
  user(id: "invalid-id") {
    name
  }
}

Resposta:

{
  "data": {
    "user": null
  },
  "errors": [
    {
      "message": "Invalid user ID",
      "locations": [
        {
          "line": 2,
          "column": 3
        }
      ],
      "path": ["user"]
    }
  ]
}

Gestió d'Errors Personalitzada

Per proporcionar missatges d'error més significatius i gestionar errors de manera elegant, pots implementar una gestió d'errors personalitzada en el teu servidor GraphQL.

Utilitzant Apollo Server

Apollo Server et permet personalitzar la gestió d'errors utilitzant la funció formatError.

Exemple:

const { ApolloServer, gql } = require('apollo-server');

const typeDefs = gql`
  type Query {
    user(id: ID!): User
  }

  type User {
    id: ID!
    name: String!
  }
`;

const resolvers = {
  Query: {
    user: (parent, args, context, info) => {
      if (args.id !== "1") {
        throw new Error("User not found");
      }
      return { id: "1", name: "John Doe" };
    }
  }
};

const server = new ApolloServer({
  typeDefs,
  resolvers,
  formatError: (err) => {
    return {
      message: err.message,
      code: err.extensions.code || 'INTERNAL_SERVER_ERROR',
      locations: err.locations,
      path: err.path
    };
  }
});

server.listen().then(({ url }) => {
  console.log(`🚀 Server ready at ${url}`);
});

En aquest exemple, la funció formatError personalitza la resposta d'error per incloure un camp code.

Codis d'Error

Pots definir codis d'error personalitzats per categoritzar errors.

Exemple:

const { ApolloServer, gql, ApolloError } = require('apollo-server');

const typeDefs = gql`
  type Query {
    user(id: ID!): User
  }

  type User {
    id: ID!
    name: String!
  }
`;

const resolvers = {
  Query: {
    user: (parent, args, context, info) => {
      if (args.id !== "1") {
        throw new ApolloError("User not found", "USER_NOT_FOUND");
      }
      return { id: "1", name: "John Doe" };
    }
  }
};

const server = new ApolloServer({
  typeDefs,
  resolvers,
  formatError: (err) => {
    return {
      message: err.message,
      code: err.extensions.code,
      locations: err.locations,
      path: err.path
    };
  }
});

server.listen().then(({ url }) => {
  console.log(`🚀 Server ready at ${url}`);
});

En aquest exemple, la classe ApolloError s'utilitza per llançar errors amb codis personalitzats.

Conclusió

La gestió d'errors en GraphQL és essencial per construir aplicacions robustes. Comprenent les respostes d'error per defecte i implementant una gestió d'errors personalitzada, pots proporcionar missatges d'error més significatius i millorar l'experiència general de l'usuari. Recorda:

  • Gestionar adequadament els errors de validació i execució.
  • Personalitzar les respostes d'error utilitzant eines com Apollo Server.
  • Definir i utilitzar codis d'error personalitzats per categoritzar errors.

Amb aquestes pràctiques, estaràs ben equipat per gestionar errors de manera efectiva en les teves aplicacions GraphQL.
