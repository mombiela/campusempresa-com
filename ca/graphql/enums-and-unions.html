Introducció

En aquesta secció, explorarem dues característiques importants de GraphQL: Enums i Unions. Aquestes característiques ajuden a definir esquemes més precisos i flexibles, fent que la teva API GraphQL sigui més robusta i fàcil d'utilitzar.

Enums en GraphQL

Què són els Enums?

  • Els Enums (abreviatura d'Enumeracions) són un tipus especial d'escalar que restringeix el valor a un conjunt predefinit de constants.
  • Són útils per representar un conjunt fix de valors possibles, com ara dies de la setmana, rols d'usuari o codis d'estat.

Definint Enums

Per definir un Enum en GraphQL, utilitzes la paraula clau enum seguida del nom de l'Enum i el conjunt de valors possibles.

enum Role {
  ADMIN
  USER
  GUEST
}

Utilitzant Enums en l'Esquema

Els Enums es poden utilitzar com a tipus de camp en el teu esquema GraphQL. Aquí tens un exemple de com utilitzar l'enum Role en un tipus User:

type User {
  id: ID!
  name: String!
  role: Role!
}

Exemple de Consulta amb Enums

Aquí tens com podries consultar un usuari i obtenir el seu rol:

query {
  user(id: "1") {
    name
    role
  }
}

Exemple de Mutació amb Enums

També pots utilitzar Enums en mutacions. Per exemple, establint el rol d'un usuari:

mutation {
  updateUserRole(id: "1", role: ADMIN) {
    id
    name
    role
  }
}

Unions en GraphQL

Què són les Unions?

  • Les Unions permeten que un camp retorni un dels diversos tipus diferents.
  • A diferència de les interfícies, les unions no comparteixen cap camp en comú.

Definint Unions

Per definir una Union, utilitzes la paraula clau union seguida del nom de la Union i els possibles tipus que pot retornar.

union SearchResult = User | Post

Utilitzant Unions en l'Esquema

Les Unions es poden utilitzar com a tipus de camp en el teu esquema GraphQL. Aquí tens un exemple de com utilitzar la union SearchResult en una consulta:

type Query {
  search(text: String!): [SearchResult!]!
}

Exemple de Consulta amb Unions

Aquí tens com podries consultar una cerca i obtenir diferents tipus de resultats:

query {
  search(text: "GraphQL") {
    ... on User {
      id
      name
    }
    ... on Post {
      id
      title
      content
    }
  }
}

Resolent Unions

Quan utilitzes unions, necessites definir com resoldre el tipus de la union en el teu codi del servidor. Aquí tens un exemple en JavaScript utilitzant Apollo Server:

const resolvers = {
  SearchResult: {
    __resolveType(obj, context, info) {
      if (obj.name) {
        return 'User';
      }
      if (obj.title) {
        return 'Post';
      }
      return null;
    },
  },
  Query: {
    search: (parent, args, context, info) => {
      // Implementa la lògica de cerca aquí
      return [
        { id: '1', name: 'Alice' },
        { id: '2', title: 'GraphQL Introduction', content: '...' },
      ];
    },
  },
};

Conclusió

Els Enums i les Unions són característiques poderoses en GraphQL que et permeten crear esquemes més precisos i flexibles. Els Enums restringeixen un camp a un conjunt de valors predefinits, fent que la teva API sigui més predictible i fàcil d'utilitzar. Les Unions permeten que un camp retorni diferents tipus, proporcionant flexibilitat en com estructures les teves dades.

Entenent i utilitzant els Enums i les Unions, pots millorar la robustesa i usabilitat de les teves APIs GraphQL, fent-les més intuïtives per als clients que hi interactuen.
