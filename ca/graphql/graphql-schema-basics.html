Introducció a l'esquema GraphQL

Un esquema GraphQL és una peça central de qualsevol implementació de servidor GraphQL. Defineix els tipus de dades que es poden consultar i les relacions entre aquests tipus. Entendre l'esquema és crucial tant per dissenyar com per consultar una API GraphQL.

Conceptes clau

  • Tipus: Els blocs de construcció d'un esquema GraphQL. Els tipus defineixen la forma de les dades.
  • Consultes: Les operacions de lectura que els clients poden realitzar per obtenir dades.
  • Mutacions: Les operacions d'escriptura que els clients poden realitzar per modificar dades.
  • Resolvers: Funcions que resolen els valors per als camps de l'esquema.

Definició de tipus

A GraphQL, defineixes tipus utilitzant la paraula clau type. Cada tipus té camps, i cada camp té un tipus.

type Book {
  id: ID!
  title: String!
  author: Author!
}

type Author {
  id: ID!
  name: String!
  books: [Book!]!
}

Explicació:

  • Book i Author són tipus personalitzats.
  • ID! és un identificador únic no nul.
  • String! és una cadena no nul·la.
  • Author! indica que el camp author en Book no pot ser nul.
  • [Book!]! indica una llista no nul·la d'objectes Book no nuls.

Consultes

Les consultes es defineixen utilitzant la paraula clau type Query. Especificen quines dades es poden obtenir i com.

type Query {
  books: [Book]
  book(id: ID!): Book
  authors: [Author]
  author(id: ID!): Author
}

Explicació:

  • books retorna una llista d'objectes Book.
  • book(id: ID!) retorna un únic objecte Book basat en l'id proporcionat.
  • authors retorna una llista d'objectes Author.
  • author(id: ID!) retorna un únic objecte Author basat en l'id proporcionat.

Mutacions

Les mutacions es defineixen utilitzant la paraula clau type Mutation. Especificen quines dades es poden modificar i com.

type Mutation {
  addBook(title: String!, authorId: ID!): Book
  updateBook(id: ID!, title: String, authorId: ID): Book
  deleteBook(id: ID!): Book
}

Explicació:

  • addBook(title: String!, authorId: ID!) afegeix un nou Book i retorna el Book afegit.
  • updateBook(id: ID!, title: String, authorId: ID) actualitza un Book existent i retorna el Book actualitzat.
  • deleteBook(id: ID!) elimina un Book existent i retorna el Book eliminat.

Resolvers

Els resolvers són funcions que proporcionen la lògica per obtenir o modificar les dades per a cada camp de l'esquema.

const resolvers = {
  Query: {
    books: () => getAllBooks(),
    book: (parent, args) => getBookById(args.id),
    authors: () => getAllAuthors(),
    author: (parent, args) => getAuthorById(args.id),
  },
  Mutation: {
    addBook: (parent, args) => addNewBook(args.title, args.authorId),
    updateBook: (parent, args) => updateExistingBook(args.id, args.title, args.authorId),
    deleteBook: (parent, args) => deleteExistingBook(args.id),
  },
  Book: {
    author: (book) => getAuthorById(book.authorId),
  },
  Author: {
    books: (author) => getBooksByAuthorId(author.id),
  },
};

Explicació:

  • Els resolvers de Query s'encarreguen d'obtenir dades.
  • Els resolvers de Mutation s'encarreguen de modificar dades.
  • Els resolvers específics de camp com Book.author i Author.books s'encarreguen d'obtenir dades niades.

Resum

En aquesta secció, hem cobert els conceptes bàsics dels esquemes GraphQL, incloent:

  • Definir tipus per donar forma a les dades.
  • Crear consultes per obtenir dades.
  • Crear mutacions per modificar dades.
  • Implementar resolvers per proporcionar la lògica per obtenir i modificar dades.

Entendre aquests fonaments és crucial per construir i consultar una API GraphQL de manera efectiva. A mesura que progressis, aprendràs conceptes més avançats com interfícies, unions i escalars personalitzats per millorar encara més els teus esquemes GraphQL.
