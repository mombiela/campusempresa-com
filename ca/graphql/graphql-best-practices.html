Introducció

GraphQL és un llenguatge de consulta potent per a la teva API, que proporciona una alternativa més eficient, potent i flexible al REST. No obstant això, per aprofitar al màxim les seves capacitats, és essencial seguir les millors pràctiques. Aquesta secció cobrirà les millors pràctiques per dissenyar, implementar i mantenir APIs de GraphQL.

Disseny de l'Esquema

L'esquema és el nucli de qualsevol API de GraphQL. Un esquema ben dissenyat és crucial per a la usabilitat i mantenibilitat de l'API.

  • Utilitza Noms Descriptius: Assegura't que els noms de camps i tipus siguin descriptius i intuïtius.
  • Organitza els Tipus Lògicament: Agrupa camps i tipus relacionats junts.
  • Evita la Niuada Profunda: Les consultes profundament niuades poden provocar problemes de rendiment.
  • Utilitza Enums per a Conjunts Fixos: Quan un camp pot tenir un conjunt limitat de valors, utilitza enums en lloc de cadenes de text.

Exemple

type User {
  id: ID!
  name: String!
  email: String!
  role: UserRole!
}

enum UserRole {
  ADMIN
  USER
  GUEST
}

Disseny de Consultes

Dissenyar consultes eficients i efectives és essencial per al rendiment i la usabilitat.

  • Limita la Profunditat de la Consulta: Prevé consultes excessivament complexes que poden degradar el rendiment.
  • Utilitza Àlies i Fragments: Els àlies i fragments poden simplificar les consultes i millorar la llegibilitat.
  • Paginar Llistes Grans: Utilitza la paginació per gestionar grans conjunts de dades de manera eficient.

Exemple

query GetUsers {
  users(first: 10, after: "cursor") {
    edges {
      node {
        id
        name
      }
    }
    pageInfo {
      endCursor
      hasNextPage
    }
  }
}

Gestió d'Errors

Una gestió adequada dels errors assegura que els clients puguin gestionar els problemes de manera elegant.

  • Utilitza Codis d'Error Personalitzats: Proporciona codis d'error i missatges significatius.
  • Valida les Entrades: Valida les entrades per evitar que dades invàlides causin errors.
  • Utilitza Blocs Try-Catch: Gestiona les excepcions de manera elegant en els teus resolvers.

Exemple

const resolvers = {
  Query: {
    user: async (_, { id }) => {
      try {
        const user = await getUserById(id);
        if (!user) {
          throw new Error('User not found');
        }
        return user;
      } catch (error) {
        return new Error('Failed to fetch user');
      }
    },
  },
};

Seguretat

La seguretat és primordial en qualsevol disseny d'API.

  • Limitació de Ràtio: Implementa la limitació de ràtio per prevenir abusos.
  • Autenticació i Autorització: Assegura't que només els usuaris autenticats i autoritzats puguin accedir a certes dades.
  • Sanitització d'Entrades: Sanitiza les entrades per prevenir atacs d'injecció.

Exemple

const resolvers = {
  Query: {
    user: async (_, { id }, { user }) => {
      if (!user) {
        throw new Error('Not authenticated');
      }
      const userData = await getUserById(id);
      if (userData.id !== user.id) {
        throw new Error('Not authorized');
      }
      return userData;
    },
  },
};

Optimització del Rendiment

Optimitzar el rendiment és crucial per a una API responsiva.

  • Agrupació de Peticions: Utilitza eines com DataLoader per agrupar i emmagatzemar en memòria cau les peticions.
  • Optimitza els Resolvers: Assegura't que els resolvers siguin eficients i evitin càlculs innecessaris.
  • Utilitza Consultes Persistents: Predefineix consultes per reduir la càrrega de parsing i validació.

Exemple

const DataLoader = require('dataloader');

const userLoader = new DataLoader(keys => batchGetUsers(keys));

const resolvers = {
  Query: {
    user: (_, { id }) => userLoader.load(id),
  },
};

Eines i Monitorització

Utilitzar les eines adequades pot ajudar a mantenir i monitoritzar la teva API de GraphQL de manera efectiva.

  • Documentació de l'Esquema: Utilitza eines com GraphiQL o Apollo Studio per documentar el teu esquema.
  • Registre i Monitorització: Implementa registre i monitorització per fer un seguiment de l'ús i el rendiment de l'API.
  • Proves: Escriu proves completes per al teu esquema i resolvers.

Exemple

const { ApolloServer } = require('apollo-server');
const { typeDefs, resolvers } = require('./schema');
const { createLogger } = require('winston');

const logger = createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.Console(),
  ],
});

const server = new ApolloServer({
  typeDefs,
  resolvers,
  context: ({ req }) => {
    const user = getUserFromToken(req.headers.authorization);
    return { user, logger };
  },
});

server.listen().then(({ url }) => {
  console.log(`🚀 Server ready at ${url}`);
});

Conclusió

Seguir les millors pràctiques en el disseny de l'esquema de GraphQL, el disseny de consultes, la gestió d'errors, la seguretat, l'optimització del rendiment i les eines pot millorar significativament l'eficiència, la seguretat i la mantenibilitat de la teva API de GraphQL. Seguint aquestes directrius, pots assegurar-te que la teva API sigui robusta, escalable i fàcil d'utilitzar.
