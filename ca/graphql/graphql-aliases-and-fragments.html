Introducció

En aquesta secció, explorarem dues característiques poderoses de GraphQL: Àlies i Fragments. Aquestes característiques ajuden a optimitzar i organitzar les teves consultes, fent-les més llegibles i eficients.

Àlies

Els àlies en GraphQL et permeten canviar el nom del resultat d'un camp per evitar conflictes o per fer la resposta més llegible.

Conceptes Clau

  • Evitar Conflictes de Noms: Quan es consulta el mateix camp diverses vegades amb diferents arguments.
  • Millorar la Llegibilitat: Fer la resposta més comprensible utilitzant noms significatius.

Exemple

Considera un escenari on vols obtenir el nom de dos usuaris diferents en una sola consulta. Sense àlies, això seria problemàtic perquè els noms dels camps entrarien en conflicte.

query {
  user1: user(id: "1") {
    name
  }
  user2: user(id: "2") {
    name
  }
}

Explicació

  • user1 i user2 són àlies per al camp user.
  • Això et permet obtenir dades per a dos usuaris diferents sense cap conflicte de noms.

Exercici

Intenta escriure una consulta utilitzant àlies per obtenir els noms i les adreces de correu electrònic de dos usuaris diferents.

Fragments

Els fragments en GraphQL et permeten reutilitzar parts de les teves consultes. Això és particularment útil quan tens múltiples consultes que comparteixen camps comuns.

Conceptes Clau

  • Reutilització: Escriu una vegada, utilitza múltiples vegades.
  • Mantenibilitat: Més fàcil d'actualitzar camps comuns en un sol lloc.

Exemple

Suposa que tens una consulta que obté detalls d'usuari i una altra consulta que obté detalls de publicació, ambdues incloent alguns camps comuns com id i createdAt.

fragment userDetails on User {
  id
  name
  email
}

fragment postDetails on Post {
  id
  title
  createdAt
}

query {
  user(id: "1") {
    ...userDetails
  }
  post(id: "1") {
    ...postDetails
  }
}

Explicació

  • fragment userDetails on User defineix un fragment reutilitzable per als camps d'usuari.
  • fragment postDetails on Post defineix un fragment reutilitzable per als camps de publicació.
  • La sintaxi ...userDetails i ...postDetails s'utilitza per incloure aquests fragments en la consulta.

Exercici

Crea un fragment per a un tipus Comment que inclogui id, content i author. Utilitza aquest fragment en una consulta per obtenir comentaris per a una publicació específica.

Conclusió

Els àlies i els fragments són eines essencials en GraphQL que ajuden a escriure consultes més eficients, llegibles i mantenibles. Els àlies et permeten evitar conflictes de noms i millorar la llegibilitat de les teves respostes, mentre que els fragments et permeten reutilitzar parts comunes de les teves consultes, fent el teu codi més mantenible. En dominar aquestes característiques, pots millorar significativament les teves capacitats de consulta en GraphQL.
