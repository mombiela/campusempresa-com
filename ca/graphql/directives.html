Les directives són una característica poderosa en GraphQL que et permeten alterar dinàmicament l'execució de les teves consultes i esquemes. Proporcionen una manera de personalitzar el comportament del teu servidor GraphQL i les interaccions del client. En aquesta secció, explorarem què són les directives, com utilitzar-les i alguns casos d'ús comuns.

Què són les Directives?

Les directives són anotacions especials en GraphQL que es poden adjuntar a camps, fragments o operacions per modificar el seu comportament. Estan prefixades amb un símbol @ i es poden utilitzar per incloure o excloure condicionalment parts d'una consulta, entre altres coses.

Conceptes Clau

  • Anotació: Les directives són anotacions que es poden afegir a consultes i esquemes GraphQL.
  • Execució Condicional: Permeten la inclusió o exclusió condicional de camps i fragments.
  • Directives Personalitzades: Pots definir les teves pròpies directives per ampliar la funcionalitat del teu servidor GraphQL.

Directives Integrades

GraphQL ve amb dues directives integrades: @include i @skip.

Directiva @include

La directiva @include inclou condicionalment un camp o fragment només si l'argument especificat és cert.

Sintaxi

directive @include(if: Boolean!) on FIELD | FRAGMENT_SPREAD | INLINE_FRAGMENT

Exemple

query GetUser($includeAddress: Boolean!) {
  user(id: "1") {
    name
    email
    address @include(if: $includeAddress) {
      street
      city
      country
    }
  }
}

En aquest exemple, el camp address només s'inclourà en la resposta si la variable $includeAddress és certa.

Directiva @skip

La directiva @skip exclou condicionalment un camp o fragment si l'argument especificat és cert.

Sintaxi

directive @skip(if: Boolean!) on FIELD | FRAGMENT_SPREAD | INLINE_FRAGMENT

Exemple

query GetUser($skipAddress: Boolean!) {
  user(id: "1") {
    name
    email
    address @skip(if: $skipAddress) {
      street
      city
      country
    }
  }
}

En aquest exemple, el camp address s'exclourà de la resposta si la variable $skipAddress és certa.

Directives Personalitzades

Pots definir les teves pròpies directives per afegir comportament personalitzat al teu servidor GraphQL. Les directives personalitzades es defineixen a l'esquema i es poden utilitzar per implementar funcions com l'autenticació, el registre o la transformació de dades.

Definint una Directiva Personalitzada

Per definir una directiva personalitzada, cal afegir-la al teu esquema utilitzant la paraula clau directive.

Exemple

directive @auth(requires: Role = ADMIN) on FIELD_DEFINITION

enum Role {
  ADMIN
  USER
}

En aquest exemple, definim una directiva @auth que pren un argument requires de tipus Role.

Utilitzant una Directiva Personalitzada

Un cop definida, pots utilitzar la directiva personalitzada en el teu esquema.

Exemple

type Query {
  secretData: String @auth(requires: ADMIN)
}

En aquest exemple, el camp secretData està protegit per la directiva @auth, que requereix que l'usuari tingui un rol ADMIN.

Implementant la Lògica d'una Directiva Personalitzada

Per implementar la lògica d'una directiva personalitzada, cal escriure un resolutor de directiva en el codi del teu servidor.

Exemple (JavaScript amb Apollo Server)

const { SchemaDirectiveVisitor } = require('apollo-server');
const { defaultFieldResolver } = require('graphql');

class AuthDirective extends SchemaDirectiveVisitor {
  visitFieldDefinition(field) {
    const { resolve = defaultFieldResolver } = field;
    const { requires } = this.args;

    field.resolve = async function (...args) {
      const context = args[2];
      if (!context.user || !context.user.roles.includes(requires)) {
        throw new Error("Not authorized");
      }
      return resolve.apply(this, args);
    };
  }
}

const typeDefs = `
  directive @auth(requires: Role = ADMIN) on FIELD_DEFINITION

  enum Role {
    ADMIN
    USER
  }

  type Query {
    secretData: String @auth(requires: ADMIN)
  }
`;

const resolvers = {
  Query: {
    secretData: () => "This is secret data",
  },
};

const server = new ApolloServer({
  typeDefs,
  resolvers,
  schemaDirectives: {
    auth: AuthDirective,
  },
  context: ({ req }) => {
    // Mock user context
    return {
      user: {
        roles: ['USER'],
      },
    };
  },
});

server.listen().then(({ url }) => {
  console.log(`🚀 Server ready at ${url}`);
});

En aquest exemple, la classe AuthDirective estèn SchemaDirectiveVisitor i sobreescriu el mètode visitFieldDefinition per implementar la lògica d'autorització.

Conclusió

Les directives en GraphQL són una eina versàtil que et pot ajudar a controlar l'execució de les teves consultes i ampliar la funcionalitat del teu servidor GraphQL. Comprenent i utilitzant tant les directives integrades com les personalitzades, pots crear aplicacions GraphQL més dinàmiques i poderoses. Tant si necessites incloure camps condicionalment, fer complir l'autorització o implementar comportaments personalitzats, les directives proporcionen una solució flexible per satisfer les teves necessitats.
