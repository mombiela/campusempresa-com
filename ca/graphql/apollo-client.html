Apollo Client és una biblioteca integral de gestió d'estat per a JavaScript que et permet gestionar tant dades locals com remotes amb GraphQL. Està dissenyada per treballar perfectament amb qualsevol servidor GraphQL i es pot integrar amb qualsevol framework de JavaScript.

Introducció a Apollo Client

  • Què és Apollo Client?

    • Una biblioteca de gestió d'estat per a aplicacions JavaScript.
    • Treballa amb GraphQL per gestionar tant dades locals com remotes.
    • Es pot integrar amb qualsevol framework de JavaScript (React, Vue, Angular, etc.).

  • Per què utilitzar Apollo Client?

    • Simplifica la recuperació de dades i la gestió d'estat.
    • Proporciona eines potents per a la memòria cau, la gestió d'errors i més.
    • Millora la productivitat del desenvolupador amb funcions com l'agrupació de consultes i les actualitzacions d'UI optimistes.

Configuració d'Apollo Client

Instal·lació

Per començar amb Apollo Client, necessites instal·lar els paquets necessaris. Per a una aplicació React, normalment instal·laries @apollo/client i graphql.

npm install @apollo/client graphql

Configuració bàsica

Després d'instal·lar els paquets, necessites configurar Apollo Client a la teva aplicació. Aquí tens un exemple bàsic de com configurar Apollo Client en una aplicació React:

import React from 'react';
import { ApolloClient, InMemoryCache, ApolloProvider } from '@apollo/client';

const client = new ApolloClient({
  uri: 'https://your-graphql-endpoint.com/graphql',
  cache: new InMemoryCache()
});

const App = () => (
  <ApolloProvider client={client}>
    <YourComponent />
  </ApolloProvider>
);

export default App;
  • Explicació:
    • ApolloClient: Crea una nova instància d'Apollo Client.
    • InMemoryCache: Un mecanisme de memòria cau proporcionat per Apollo Client.
    • ApolloProvider: Un component de React que proporciona la instància d'Apollo Client al teu arbre de components React.

Recuperació de dades amb consultes

Consulta bàsica

Per recuperar dades, utilitzes el hook useQuery proporcionat per Apollo Client. Aquí tens un exemple de com recuperar dades d'un endpoint GraphQL:

import React from 'react';
import { useQuery, gql } from '@apollo/client';

const GET_DATA = gql`
  query GetData {
    data {
      id
      name
    }
  }
`;

const DataComponent = () => {
  const { loading, error, data } = useQuery(GET_DATA);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error.message}</p>;

  return (
    <ul>
      {data.data.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
};

export default DataComponent;
  • Explicació:
    • gql: Una funció per analitzar consultes GraphQL.
    • useQuery: Un hook per executar la consulta i gestionar el seu estat (loading, error, data).

Modificació de dades amb mutacions

Mutació bàsica

Per modificar dades, utilitzes el hook useMutation. Aquí tens un exemple de com realitzar una mutació:

import React, { useState } from 'react';
import { useMutation, gql } from '@apollo/client';

const ADD_DATA = gql`
  mutation AddData($name: String!) {
    addData(name: $name) {
      id
      name
    }
  }
`;

const AddDataComponent = () => {
  const [name, setName] = useState('');
  const [addData, { data, loading, error }] = useMutation(ADD_DATA);

  const handleSubmit = (e) => {
    e.preventDefault();
    addData({ variables: { name } });
  };

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error.message}</p>;

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
      />
      <button type="submit">Add Data</button>
      {data && <p>Added: {data.addData.name}</p>}
    </form>
  );
};

export default AddDataComponent;
  • Explicació:
    • useMutation: Un hook per executar la mutació i gestionar el seu estat (loading, error, data).
    • variables: Un objecte que conté les variables requerides per la mutació.

Funcions avançades

Memòria cau

Apollo Client proporciona un mecanisme de memòria cau potent per millorar el rendiment i reduir les sol·licituds de xarxa. InMemoryCache és la implementació de memòria cau per defecte.

  • Configuració de la memòria cau:
    • Pots configurar la memòria cau per gestionar com es normalitzen i s'emmagatzemen les dades.
    • Exemple: Personalització de les claus de la memòria cau.
const cache = new InMemoryCache({
  typePolicies: {
    Query: {
      fields: {
        data: {
          keyArgs: false,
          merge(existing = [], incoming) {
            return [...existing, ...incoming];
          }
        }
      }
    }
  }
});

Gestió d'errors

Apollo Client proporciona mecanismes de gestió d'errors integrats per gestionar els errors de manera elegant.

  • Exemple de gestió d'errors:
const client = new ApolloClient({
  uri: 'https://your-graphql-endpoint.com/graphql',
  cache: new InMemoryCache(),
  onError: ({ graphQLErrors, networkError }) => {
    if (graphQLErrors) {
      graphQLErrors.forEach(({ message, locations, path }) =>
        console.log(`[GraphQL error]: Message: ${message}, Location: ${locations}, Path: ${path}`)
      );
    }
    if (networkError) {
      console.log(`[Network error]: ${networkError}`);
    }
  }
});

Conclusió

Apollo Client és una eina poderosa per gestionar dades GraphQL en aplicacions JavaScript. Simplifica la recuperació de dades, la memòria cau i la gestió d'estat, facilitant la construcció d'aplicacions robustes i eficients. Entenent els conceptes bàsics de la configuració d'Apollo Client, la recuperació i modificació de dades, i aprofitant funcions avançades com la memòria cau i la gestió d'errors, pots integrar efectivament GraphQL als teus projectes i millorar el teu flux de treball de desenvolupament.
