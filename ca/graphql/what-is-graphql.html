GraphQL vs REST ►

Introducció

GraphQL és un llenguatge de consulta per a APIs i un entorn d'execució per executar aquestes consultes utilitzant un sistema de tipus que defineixes per a les teves dades. Va ser desenvolupat per Facebook el 2012 i alliberat com a projecte de codi obert el 2015. GraphQL proporciona una alternativa més eficient, potent i flexible a l'API REST tradicional.

Conceptes Clau

  • Llenguatge de Consulta: GraphQL permet als clients sol·licitar exactament les dades que necessiten, fent possible obtenir totes les dades necessàries en una sola sol·licitud.
  • Entorn d'Execució: GraphQL executa consultes contra un esquema predefinit, assegurant que les dades retornades coincideixin amb la sol·licitud del client.
  • Esquema: Un sistema de tipus fort que defineix les capacitats de l'API i la forma de les dades.
  • Resolvers: Funcions que gestionen la recuperació de les dades per a cada camp en una consulta.

Beneficis de GraphQL

  • Recuperació Eficient de Dades: Els clients poden sol·licitar només les dades que necessiten, reduint problemes de sobrecàrrega i subcàrrega de dades.
  • Un Sol Punt d'Accés: A diferència de REST, que sovint requereix múltiples punts d'accés, GraphQL utilitza un sol punt d'accés per accedir a totes les dades.
  • Esquema Fortament Tipat: L'esquema defineix l'estructura de l'API, fent-la més fàcil d'entendre i utilitzar.
  • Introspecció: Els clients poden consultar l'esquema mateix per entendre les dades i operacions disponibles.

Estructura Bàsica d'una Consulta GraphQL

Una consulta GraphQL està estructurada per reflectir la forma de les dades JSON que esperes rebre. Aquí tens un exemple senzill:

{
  user(id: "1") {
    id
    name
    email
  }
}

Explicació:

  • user(id: "1"): Aquest és un camp amb un argument. Sol·licita un usuari amb l'ID 1.
  • id, name, email: Aquests són els camps sol·licitats per a l'objecte usuari.

Estructura Bàsica d'un Esquema GraphQL

Un esquema GraphQL es defineix utilitzant un sistema de tipus. Aquí tens un exemple de definició d'esquema:

type User {
  id: ID!
  name: String!
  email: String!
}

type Query {
  user(id: ID!): User
}

Explicació:

  • type User: Defineix un tipus Usuari amb els camps id, name i email.
  • type Query: Defineix el punt d'entrada per a l'API, especificant que una consulta user retornarà un objecte Usuari.

Exemple d'un Resolver

Els resolvers són funcions que resolen un valor per a un tipus o camp en un esquema. Aquí tens un exemple de resolver per a la consulta user:

const resolvers = {
  Query: {
    user: (parent, args, context, info) => {
      // Recupera les dades de l'usuari d'una font de dades, per exemple, una base de dades
      return {
        id: args.id,
        name: "John Doe",
        email: "john.doe@example.com"
      };
    }
  }
};

Explicació:

  • user Resolver: Aquesta funció pren arguments i retorna un objecte usuari. En un escenari real, recuperaria dades d'una base de dades o una altra font de dades.

Comparació: GraphQL vs REST

| Característica | GraphQL | REST | |-----------------------|----------------------------------------------|-----------------------------------------------| | Recuperació de Dades | Els clients sol·liciten exactament el que necessiten | Els punts d'accés fixos retornen dades predefinides | | Punts d'Accés | Un sol punt d'accés per a totes les consultes | Múltiples punts d'accés per a diferents recursos | | Esquema | Esquema fortament tipat | Sense esquema integrat | | Sobrecarrega/Subcarrega de Dades | Evita ambdues amb consultes precises | Problema comú degut als punts d'accés fixos | | Versionat | No cal versionat | Sovint requereix versionat |

Conclusió

GraphQL és una eina potent per construir APIs que permeten als clients sol·licitar exactament les dades que necessiten. El seu esquema fortament tipat, un sol punt d'accés i capacitats eficients de recuperació de dades el fan una alternativa atractiva a les APIs REST tradicionals. Entendre l'estructura bàsica de les consultes, esquemes i resolvers és essencial per començar amb GraphQL. A mesura que progressis, descobriràs característiques més avançades i millors pràctiques que poden millorar encara més la teva experiència de desenvolupament d'APIs.
