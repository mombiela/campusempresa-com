Introducció a les Interfícies

Les interfícies en GraphQL són una manera poderosa de definir un contracte que múltiples tipus poden implementar. Permeten crear un conjunt de camps que diferents tipus han d'incloure, assegurant la consistència en el teu esquema.

  • Definició: Una interfície és un tipus abstracte que inclou un cert conjunt de camps que un tipus ha d'implementar.
  • Propòsit: Les interfícies s'utilitzen per definir una estructura comuna per a diferents tipus, permetent el polimorfisme en el teu esquema GraphQL.

Definint Interfícies

Per definir una interfície en GraphQL, utilitzes la paraula clau interface seguida del nom de la interfície i els camps que inclou.

Exemple

interface Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
}

En aquest exemple:

  • Character és la interfície.
  • Inclou tres camps: id, name, i appearsIn.

Implementant Interfícies

Els tipus que implementen una interfície han d'incloure tots els camps definits en la interfície.

Exemple

type Human implements Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
  totalCredits: Int
}

type Droid implements Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
  primaryFunction: String
}

En aquest exemple:

  • Human i Droid implementen la interfície Character.
  • Ambdós tipus inclouen els camps id, name, i appearsIn com requerit per la interfície Character.
  • També inclouen camps addicionals específics per a cada tipus (totalCredits per a Human i primaryFunction per a Droid).

Fent Consultes a Interfícies

Quan fas una consulta a una interfície, pots demanar camps que formen part de la interfície, i també pots utilitzar fragments en línia per demanar camps específics dels tipus que implementen la interfície.

Exemple

{
  characters {
    id
    name
    appearsIn
    ... on Human {
      totalCredits
    }
    ... on Droid {
      primaryFunction
    }
  }
}

En aquesta consulta:

  • characters és un camp que retorna una llista de tipus de la interfície Character.
  • La consulta demana els camps id, name, i appearsIn, que formen part de la interfície Character.
  • Els fragments en línia (... on Human i ... on Droid) s'utilitzen per demanar camps específics dels tipus Human i Droid.

Interfícies vs. Unions

Tot i que tant les interfícies com les unions permeten el polimorfisme, serveixen per a propòsits diferents.

| Característica | Interfície | Unió | |------------------|------------------------------------------------|------------------------------------------------| | Camps Comuns | Sí | No | | Comprovació de Tipus | Pot comprovar camps específics | Només pot comprovar el tipus | | Cas d'Ús | Quan els tipus comparteixen camps comuns | Quan els tipus no comparteixen camps comuns | | Sintaxi | Paraula clau interface | Paraula clau union |

Exemple d'Unió

union SearchResult = Human | Droid | Starship

{
  search(text: "R2") {
    ... on Human {
      name
    }
    ... on Droid {
      primaryFunction
    }
    ... on Starship {
      model
    }
  }
}

En aquest exemple:

  • SearchResult és una unió de Human, Droid, i Starship.
  • La consulta utilitza fragments en línia per demanar camps específics de cada tipus en la unió.

Conclusió

Les interfícies en GraphQL proporcionen una manera robusta d'imposar una estructura consistent en diferents tipus. Definint camps comuns en una interfície i implementant-los en diversos tipus, pots assegurar que el teu esquema es mantingui organitzat i previsible. Entendre com definir, implementar i fer consultes a interfícies és essencial per crear APIs GraphQL flexibles i mantenibles.
