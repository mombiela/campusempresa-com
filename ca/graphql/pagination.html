La paginació és un concepte crucial en GraphQL per gestionar grans conjunts de dades. Permet als clients sol·licitar dades en fragments, fent que la recuperació de dades sigui més eficient i manejable. Aquest tema cobrirà les bases de la paginació, diferents estratègies de paginació i tècniques avançades.

Introducció a la Paginació

La paginació és el procés de dividir un gran conjunt de dades en fragments més petits i manejables o pàgines. Això és essencial per al rendiment i la usabilitat, especialment quan es tracta de grans conjunts de dades.

Conceptes Clau

  • Pàgina: Un subconjunt del conjunt de dades total.
  • Cursor: Un punter a un element específic del conjunt de dades, utilitzat per mantenir el seguiment de la posició actual.
  • Desplaçament: El nombre d'elements a saltar abans de començar a recollir el conjunt de resultats.
  • Límit: El nombre màxim d'elements a retornar en una sola pàgina.

Estratègies de Paginació

Hi ha dues estratègies principals per implementar la paginació en GraphQL: Paginació basada en desplaçament i Paginació basada en cursor.

Paginació basada en desplaçament

La paginació basada en desplaçament utilitza un desplaçament i un límit per obtenir un subconjunt específic de dades.

Exemple

query {
  posts(offset: 0, limit: 10) {
    id
    title
    content
  }
}

Explicació

  • offset: El nombre d'elements a saltar.
  • limit: El nombre d'elements a retornar.

Aquesta consulta obté els primers 10 posts. Per obtenir els següents 10 posts, caldria establir offset a 10.

Paginació basada en cursor

La paginació basada en cursor utilitza un cursor per mantenir el seguiment de la posició actual en el conjunt de dades. Aquest mètode és més eficient per a grans conjunts de dades i proporciona un millor rendiment.

Exemple

query {
  posts(first: 10, after: "cursor123") {
    edges {
      node {
        id
        title
        content
      }
      cursor
    }
    pageInfo {
      endCursor
      hasNextPage
    }
  }
}

Explicació

  • first: El nombre d'elements a retornar.
  • after: El cursor que apunta a l'últim element de la pàgina anterior.
  • edges: Conté les dades reals i el cursor per a cada element.
  • pageInfo: Proporciona informació sobre l'estat de la paginació, com endCursor i hasNextPage.

Implementació de la Paginació en GraphQL

Definició de l'Esquema

Defineix l'esquema per donar suport a la paginació. Aquí tens un exemple d'esquema per a la paginació basada en cursor:

type Post {
  id: ID!
  title: String!
  content: String!
}

type PostEdge {
  node: Post
  cursor: String!
}

type PageInfo {
  endCursor: String
  hasNextPage: Boolean!
}

type PostConnection {
  edges: [PostEdge]
  pageInfo: PageInfo!
}

type Query {
  posts(first: Int, after: String): PostConnection
}

Implementació del Resolver

Implementa el resolver per gestionar la lògica de la paginació. Aquí tens un exemple de resolver per a la paginació basada en cursor:

const posts = [
  // Array d'objectes post
];

const resolvers = {
  Query: {
    posts: (parent, { first, after }) => {
      const startIndex = after ? posts.findIndex(post => post.id === after) + 1 : 0;
      const paginatedPosts = posts.slice(startIndex, startIndex + first);

      const edges = paginatedPosts.map(post => ({
        node: post,
        cursor: post.id,
      }));

      const endCursor = edges.length > 0 ? edges[edges.length - 1].cursor : null;
      const hasNextPage = startIndex + first < posts.length;

      return {
        edges,
        pageInfo: {
          endCursor,
          hasNextPage,
        },
      };
    },
  },
};

Tècniques Avançades de Paginació

Paginació Bidireccional

La paginació bidireccional permet als clients navegar tant cap endavant com cap enrere pel conjunt de dades. Això es pot aconseguir afegint arguments before i last a la consulta.

Exemple

query {
  posts(last: 10, before: "cursor123") {
    edges {
      node {
        id
        title
        content
      }
      cursor
    }
    pageInfo {
      startCursor
      hasPreviousPage
    }
  }
}

Combinació de Filtres amb Paginació

La combinació de filtres amb paginació permet als clients refinar les seves sol·licituds de dades mentre es beneficien de la paginació.

Exemple

query {
  posts(first: 10, after: "cursor123", filter: { author: "John Doe" }) {
    edges {
      node {
        id
        title
        content
      }
      cursor
    }
    pageInfo {
      endCursor
      hasNextPage
    }
  }
}

Conclusió

La paginació és un concepte fonamental en GraphQL que ajuda a gestionar grans conjunts de dades de manera eficient. Comprenent i implementant diferents estratègies de paginació, com la paginació basada en desplaçament i la paginació basada en cursor, pots millorar el rendiment i la usabilitat de les teves APIs de GraphQL. Tècniques avançades com la paginació bidireccional i la combinació de filtres amb paginació proporcionen encara més flexibilitat i control sobre la recuperació de dades.
