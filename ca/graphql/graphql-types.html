Introducció als tipus de GraphQL

GraphQL és un llenguatge de consulta per a APIs que permet als clients sol·licitar exactament les dades que necessiten. Un dels conceptes clau en GraphQL és el seu sistema de tipus, que defineix l'estructura de les dades que es poden consultar. Entendre els tipus de GraphQL és essencial tant per definir el teu esquema com per escriure consultes efectives.

Tipus bàsics

GraphQL ve amb un conjunt de tipus escalars bàsics que representen els tipus de dades més comuns.

  • Int: Un enter de 32 bits amb signe.
  • Float: Un valor de punt flotant de doble precisió amb signe.
  • String: Una seqüència de caràcters UTF-8.
  • Boolean: Un valor veritable o fals.
  • ID: Un identificador únic, sovint utilitzat per tornar a obtenir un objecte o com a clau per a una memòria cau.

Exemple

type Query {
  id: ID
  name: String
  age: Int
  isActive: Boolean
  score: Float
}

Tipus d'objecte

Els tipus d'objecte són el tipus més comú en un esquema de GraphQL. Representen una col·lecció de camps, cadascun dels quals pot ser un altre tipus.

Exemple

type User {
  id: ID!
  name: String!
  age: Int
  email: String
}

En aquest exemple:

  • id i name són camps no nul·lables (denotats per !).
  • age i email són camps nul·lables.

Tipus de consulta i mutació

En GraphQL, l'esquema defineix dos tipus especials: Query i Mutation.

  • Query: Defineix els punts d'entrada per a operacions de lectura.
  • Mutation: Defineix els punts d'entrada per a operacions d'escriptura.

Exemple

type Query {
  user(id: ID!): User
  users: [User]
}

type Mutation {
  createUser(name: String!, email: String!): User
}

En aquest exemple:

  • El tipus Query té dos camps: user i users.
  • El tipus Mutation té un camp: createUser.

Tipus d'enumeració

Els tipus d'enumeració són un tipus especial d'escalar que està restringit a un conjunt particular de valors permesos.

Exemple

enum Role {
  ADMIN
  USER
  GUEST
}

type User {
  id: ID!
  name: String!
  role: Role!
}

En aquest exemple:

  • L'enumeració Role pot tenir un dels tres valors: ADMIN, USER o GUEST.

Tipus de llista i no nul·lables

GraphQL també admet tipus de llista i no nul·lables per definir estructures més complexes.

  • Llista: Representa una matriu d'un cert tipus.
  • No nul·lable: Assegura que un camp no pot ser nul.

Exemple

type Query {
  users: [User!]!
}

En aquest exemple:

  • users és una llista no nul·lable d'objectes User no nul·lables.

Tipus d'entrada

Els tipus d'entrada s'utilitzen com a arguments en consultes i mutacions. Permeten passar objectes complexos com a arguments.

Exemple

input UserInput {
  name: String!
  email: String!
  age: Int
}

type Mutation {
  createUser(input: UserInput!): User
}

En aquest exemple:

  • UserInput és un tipus d'entrada utilitzat en la mutació createUser.

Tipus d'interfície

Els tipus d'interfície són tipus abstractes que defineixen un conjunt de camps que múltiples tipus han d'incloure.

Exemple

interface Character {
  id: ID!
  name: String!
}

type Human implements Character {
  id: ID!
  name: String!
  homePlanet: String
}

type Droid implements Character {
  id: ID!
  name: String!
  primaryFunction: String
}

type Query {
  characters: [Character]
}

En aquest exemple:

  • Character és una interfície implementada per Human i Droid.

Tipus d'unió

Els tipus d'unió són similars a les interfícies però no requereixen que els tipus comparteixin cap camp.

Exemple

union SearchResult = Human | Droid

type Query {
  search(text: String!): [SearchResult]
}

En aquest exemple:

  • SearchResult és un tipus d'unió que pot ser Human o Droid.

Conclusió

Entendre els tipus de GraphQL és fonamental per dissenyar un esquema de GraphQL robust i eficient. Aprofitant els diversos tipus disponibles, pots crear un esquema que representi amb precisió les teves dades i admeti consultes i mutacions complexes. A mesura que progressis, descobriràs que dominar aquests tipus et permetrà construir APIs més potents i flexibles.
