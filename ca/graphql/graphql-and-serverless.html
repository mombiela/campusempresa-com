Introducció a l'Arquitectura Serverless

  • Definició: L'arquitectura serverless permet als desenvolupadors construir i executar aplicacions sense gestionar la infraestructura del servidor. El proveïdor de núvol provisiona, escala i gestiona automàticament la infraestructura necessària per executar el codi.
  • Beneficis:
    • Eficiència de Costos: Paga només pel temps de computació que consumeixes.
    • Escalabilitat: Escala automàticament amb el nombre de sol·licituds.
    • Complexitat Operativa Reduïda: Centra't en escriure codi sense preocupar-te per la gestió del servidor.

Integració de GraphQL amb Serverless

  • Per què Utilitzar GraphQL amb Serverless?:
    • Obtenció Eficient de Dades: GraphQL permet als clients sol·licitar exactament les dades que necessiten, reduint el sobreaprofitament i el subaprofitament.
    • Flexibilitat: S'adapta fàcilment als requisits canviants sense modificar el backend.
    • Rendiment: Les funcions serverless es poden optimitzar per a consultes i mutacions específiques de GraphQL.

Configuració d'una API GraphQL Serverless

  • Triar un Proveïdor Serverless:

    • AWS Lambda
    • Azure Functions
    • Google Cloud Functions
    • Netlify Functions

  • Exemple de Configuració amb AWS Lambda:

    1. Crear una nova funció AWS Lambda:
      • Ves a la Consola de Gestió d'AWS.
      • Navega a Lambda i crea una nova funció.
      • Tria un runtime (per exemple, Node.js).
    2. Instal·lar Dependències: 
      npm init -y
npm install graphql express express-graphql
    3. Escriure la Funció Lambda: 
      const { ApolloServer, gql } = require('apollo-server-lambda');

// Defineix el teu esquema
const typeDefs = gql`
  type Query {
    hello: String
  }
`;

// Defineix els teus resolvers
const resolvers = {
  Query: {
    hello: () => 'Hello, world!',
  },
};

// Crea el servidor Apollo
const server = new ApolloServer({ typeDefs, resolvers });

exports.handler = server.createHandler();
    4. Desplegar la Funció:
      • Empaqueta el teu codi i dependències.
      • Carrega el paquet a AWS Lambda.
      • Configura un API Gateway per activar la funció Lambda.

Exemple: Construir una API GraphQL Serverless amb Netlify Functions

  1. Configurar un Nou Projecte a Netlify:
    • Crea un nou repositori a GitHub.
    • Enllaça el repositori a Netlify.
  2. Instal·lar Dependències: 
    npm init -y
npm install graphql apollo-server-lambda
  3. Crear el Gestor de GraphQL: 
    // netlify/functions/graphql.js
const { ApolloServer, gql } = require('apollo-server-lambda');

const typeDefs = gql`
  type Query {
    hello: String
  }
`;

const resolvers = {
  Query: {
    hello: () => 'Hello, Netlify!',
  },
};

const server = new ApolloServer({ typeDefs, resolvers });

exports.handler = server.createHandler();
  4. Desplegar a Netlify:
    • Puja el teu codi a GitHub.
    • Netlify construirà i desplegarà automàticament el teu projecte.

Millors Pràctiques per a APIs GraphQL Serverless

  • Optimitzar els Temps de Cold Start:
    • Minimitza la mida del teu paquet de desplegament.
    • Utilitza biblioteques lleugeres.
  • Gestió d'Errors:
    • Implementa una gestió d'errors robusta en els teus resolvers.
    • Utilitza eines com Sentry per al monitoratge.
  • Seguretat:
    • Utilitza mecanismes d'autenticació i autorització.
    • Valida les dades d'entrada per prevenir atacs d'injecció.

Conclusió

Integrar GraphQL amb l'arquitectura serverless proporciona una combinació poderosa per construir APIs escalables, eficients i rendibles. Aprofitant els beneficis de totes dues tecnologies, els desenvolupadors poden crear aplicacions flexibles i de gran rendiment sense la càrrega de gestionar la infraestructura del servidor. Tant si utilitzes AWS Lambda, Netlify Functions o un altre proveïdor serverless, els principis i pràctiques es mantenen consistents, assegurant una experiència de desenvolupament fluida.
