Introducció a GraphiQL i Playground

GraphiQL i Playground són dues eines populars utilitzades per interactuar amb APIs GraphQL. Proporcionen una interfície fàcil d'usar per escriure, provar i depurar consultes i mutacions GraphQL. Aquesta secció cobrirà els conceptes bàsics d'aquestes eines, les seves característiques i com utilitzar-les de manera efectiva.

Què és GraphiQL?

GraphiQL és un IDE al navegador per explorar GraphQL. Permet als desenvolupadors escriure consultes, veure resultats en temps real i explorar l'esquema.

Característiques Clau de GraphiQL

  • Consulta Interactiva: Escriu i executa consultes i mutacions.
  • Exploració de l'Esquema: Navega per l'esquema per entendre els tipus i camps disponibles.
  • Resultats en Temps Real: Veu els resultats de les teves consultes instantàniament.
  • Documentació: Documentació en línia per a tipus i camps.

Utilitzant GraphiQL

Per utilitzar GraphiQL, normalment has de navegar a l'endpoint /graphql del teu servidor GraphQL. Aquí tens un exemple bàsic de com utilitzar GraphiQL:

  1. Obre GraphiQL: Navega a http://localhost:4000/graphql (assumint que el teu servidor està funcionant a localhost i port 4000).
  2. Escriu una Consulta: Introdueix una consulta al panell esquerre. 
    {
  user(id: "1") {
    id
    name
    email
  }
}
  3. Executa la Consulta: Prem el botó "Executar" (normalment un botó de reproducció) per veure els resultats al panell dret.

Exemple de Consulta en GraphiQL

{
  user(id: "1") {
    id
    name
    email
  }
}

Aquesta consulta obté l'id, nom i correu electrònic d'un usuari amb l'ID "1".

Què és Playground?

Playground és una altra eina poderosa per interactuar amb APIs GraphQL. Ofereix característiques més avançades en comparació amb GraphiQL i sovint s'utilitza en entorns de producció.

Característiques Clau de Playground

  • Consulta Interactiva: Similar a GraphiQL, permet escriure i executar consultes.
  • Exploració de l'Esquema: Navega i entén l'esquema.
  • Resultats en Temps Real: Retroalimentació immediata sobre les consultes.
  • Característiques Avançades: Inclou característiques com historial de consultes, pestanyes i personalització de configuracions.

Utilitzant Playground

Playground es pot accedir de manera similar a GraphiQL, sovint a l'endpoint /playground. Aquí tens com utilitzar Playground:

  1. Obre Playground: Navega a http://localhost:4000/playground.
  2. Escriu una Consulta: Introdueix una consulta a l'editor de consultes. 
    {
  user(id: "1") {
    id
    name
    email
  }
}
  3. Executa la Consulta: Prem el botó "Executar" per veure els resultats.

Exemple de Consulta en Playground

{
  user(id: "1") {
    id
    name
    email
  }
}

Aquesta consulta és idèntica a la utilitzada en GraphiQL i demostra la consistència entre les eines.

Comparació de GraphiQL i Playground

| Característica | GraphiQL | Playground | |------------------------|---------------------------|---------------------------| | Consulta Interactiva | Sí | Sí | | Exploració de l'Esquema| Sí | Sí | | Resultats en Temps Real| Sí | Sí | | Historial de Consultes | No | Sí | | Pestanyes | No | Sí | | Personalització de Configuracions | Limitada | Extensa |

Conclusió

GraphiQL i Playground són eines essencials per a qualsevol desenvolupador de GraphQL. Proporcionen interfícies poderoses per escriure, provar i depurar consultes i mutacions. Mentre que GraphiQL és més senzill i sovint suficient per a necessitats bàsiques, Playground ofereix característiques avançades que poden ser molt útils en escenaris de desenvolupament més complexos. Entendre com utilitzar ambdues eines de manera efectiva augmentarà enormement la teva productivitat i eficiència quan treballis amb APIs GraphQL.
