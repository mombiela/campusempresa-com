Introducció a GraphQL i WebSockets

GraphQL és un llenguatge de consulta per a APIs i un temps d'execució per executar aquestes consultes utilitzant un sistema de tipus que defineixes per a les teves dades. Els WebSockets proporcionen un canal de comunicació de doble sentit sobre una única connexió TCP, permetent la transferència de dades en temps real entre client i servidor.

Combinar GraphQL amb WebSockets permet actualitzacions i subscripcions en temps real, fent possible construir aplicacions altament interactives i dinàmiques.

Conceptes Clau

  • GraphQL: Un llenguatge de consulta per a la teva API, i un temps d'execució del costat del servidor per executar consultes utilitzant un sistema de tipus que defineixes per a les teves dades.
  • WebSockets: Un protocol que proporciona canals de comunicació de doble sentit sobre una única connexió TCP.
  • Subscripcions: Una característica de GraphQL que permet als clients subscriure's a esdeveniments específics i rebre actualitzacions en temps real.

Configurant GraphQL amb WebSockets

Per utilitzar GraphQL amb WebSockets, necessites configurar un servidor GraphQL que suporti subscripcions. Una opció popular és Apollo Server.

Instal·lant Dependències

Primer, instal·la els paquets necessaris:

npm install apollo-server graphql subscriptions-transport-ws

Configurant Apollo Server amb Suport per a WebSocket

Aquí tens una configuració bàsica per a un Apollo Server amb suport per a WebSocket:

const { ApolloServer, gql } = require('apollo-server');
const { createServer } = require('http');
const { execute, subscribe } = require('graphql');
const { SubscriptionServer } = require('subscriptions-transport-ws');

// Defineix el teu esquema
const typeDefs = gql`
  type Query {
    hello: String
  }

  type Subscription {
    time: String
  }
`;

// Defineix els teus resolvers
const resolvers = {
  Query: {
    hello: () => 'Hello world!',
  },
  Subscription: {
    time: {
      subscribe: (_, __, { pubsub }) => pubsub.asyncIterator('TIME'),
    },
  },
};

// Crea una instància d'ApolloServer
const server = new ApolloServer({
  typeDefs,
  resolvers,
  context: ({ req }) => ({
    pubsub,
  }),
});

// Crea un servidor HTTP
const httpServer = createServer(server);

// Configura WebSocket per a subscripcions
server.installSubscriptionHandlers(httpServer);

// Inicia el servidor
httpServer.listen(4000, () => {
  console.log(`El servidor està funcionant a http://localhost:4000${server.graphqlPath}`);
  console.log(`Subscripcions llestes a ws://localhost:4000${server.subscriptionsPath}`);
});

Publicant Esdeveniments

Per publicar esdeveniments als subscriptors, pots utilitzar una instància de PubSub del paquet graphql-subscriptions:

const { PubSub } = require('graphql-subscriptions');
const pubsub = new PubSub();

// Publica un esdeveniment cada segon
setInterval(() => {
  const time = new Date().toISOString();
  pubsub.publish('TIME', { time });
}, 1000);

Exemple: Aplicació de Xat en Temps Real

Construïm una simple aplicació de xat en temps real utilitzant GraphQL i WebSockets.

Definició de l'Esquema

Defineix l'esquema per a missatges i subscripcions:

const typeDefs = gql`
  type Message {
    id: ID!
    content: String!
    user: String!
  }

  type Query {
    messages: [Message!]
  }

  type Mutation {
    postMessage(user: String!, content: String!): ID!
  }

  type Subscription {
    messages: [Message!]
  }
`;

Resolvers

Implementa els resolvers per a consultes, mutacions i subscripcions:

const messages = [];
const resolvers = {
  Query: {
    messages: () => messages,
  },
  Mutation: {
    postMessage: (parent, { user, content }) => {
      const id = messages.length;
      messages.push({ id, user, content });
      pubsub.publish('MESSAGES', { messages });
      return id;
    },
  },
  Subscription: {
    messages: {
      subscribe: () => pubsub.asyncIterator('MESSAGES'),
    },
  },
};

Implementació del Client

Al costat del client, pots utilitzar Apollo Client per connectar-te al servidor i gestionar subscripcions:

import { ApolloClient, InMemoryCache, gql } from '@apollo/client';
import { WebSocketLink } from '@apollo/client/link/ws';
import { split } from '@apollo/client';
import { getMainDefinition } from '@apollo/client/utilities';
import { HttpLink } from '@apollo/client';

// Crea un enllaç HTTP
const httpLink = new HttpLink({
  uri: 'http://localhost:4000/graphql',
});

// Crea un enllaç WebSocket
const wsLink = new WebSocketLink({
  uri: 'ws://localhost:4000/graphql',
  options: {
    reconnect: true,
  },
});

// Divideix els enllaços basant-se en el tipus d'operació
const link = split(
  ({ query }) => {
    const definition = getMainDefinition(query);
    return (
      definition.kind === 'OperationDefinition' &&
      definition.operation === 'subscription'
    );
  },
  wsLink,
  httpLink,
);

// Crea una instància d'Apollo Client
const client = new ApolloClient({
  link,
  cache: new InMemoryCache(),
});

// Subscriu-te als missatges
client
  .subscribe({
    query: gql`
      subscription {
        messages {
          id
          content
          user
        }
      }
    `,
  })
  .subscribe({
    next({ data }) {
      console.log(data);
    },
  });

Conclusió

GraphQL i WebSockets junts proporcionen un conjunt d'eines poderoses per construir aplicacions en temps real. Mitjançant l'ús de subscripcions de GraphQL, pots crear experiències d'usuari dinàmiques i interactives que s'actualitzen en temps real. Aquesta combinació és especialment útil per a aplicacions com aplicacions de xat, actualitzacions en viu d'esports i eines de col·laboració.

En aquesta secció, has après a configurar un servidor GraphQL amb suport per a WebSocket, implementar subscripcions i construir una aplicació de xat en temps real. Amb aquestes habilitats, ara pots explorar casos d'ús més avançats i optimitzar la gestió de dades en temps real.
