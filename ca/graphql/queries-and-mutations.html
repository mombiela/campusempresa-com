Introducció a GraphQL

GraphQL és un llenguatge de consulta per a APIs i un entorn d'execució per executar aquestes consultes utilitzant un sistema de tipus que defineixes per a les teves dades. Permet als clients sol·licitar exactament les dades que necessiten, fent-lo més eficient que les APIs REST tradicionals.

Consultes

Les consultes en GraphQL s'utilitzen per obtenir dades del servidor. Són anàlogues a les sol·licituds GET en APIs REST.

Estructura Bàsica d'una Consulta

Una consulta bàsica de GraphQL es veu així:

{
  user(id: "1") {
    id
    name
    email
  }
}
  • Explicació:
    • La consulta sol·licita un user amb un id de "1".
    • Els camps id, name i email estan especificats, el que significa que només aquests camps seran retornats.

Consultes Imbricades

GraphQL permet consultes imbricades, permetent obtenir dades relacionades en una sola sol·licitud.

{
  user(id: "1") {
    id
    name
    email
    posts {
      title
      content
    }
  }
}
  • Explicació:
    • A més de la informació bàsica de l'usuari, aquesta consulta obté els posts associats amb l'usuari.
    • Per a cada post, només es sol·liciten els camps title i content.

Arguments

Es poden passar arguments als camps per filtrar o personalitzar les dades retornades.

{
  posts(limit: 10, sort: "DESC") {
    title
    content
  }
}
  • Explicació:
    • El camp posts accepta arguments limit i sort per controlar el nombre de posts retornats i el seu ordre.

Àlies

Els àlies permeten canviar el nom del resultat d'un camp per evitar conflictes o per fer la resposta més llegible.

{
  firstUser: user(id: "1") {
    name
  }
  secondUser: user(id: "2") {
    name
  }
}
  • Explicació:
    • El camp user es consulta dues vegades amb diferents valors id, però els resultats es canvien de nom com firstUser i secondUser.

Mutacions

Les mutacions en GraphQL s'utilitzen per modificar dades al servidor. Són anàlogues a les sol·licituds POST, PUT, DELETE en APIs REST.

Estructura Bàsica d'una Mutació

Una mutació bàsica es veu així:

mutation {
  createUser(name: "John Doe", email: "john@example.com") {
    id
    name
    email
  }
}
  • Explicació:
    • La mutació createUser crea un nou usuari amb el name i email especificats.
    • La resposta inclou el id, name i email del nou usuari creat.

Actualització de Dades

Les mutacions també es poden utilitzar per actualitzar dades existents.

mutation {
  updateUser(id: "1", email: "newemail@example.com") {
    id
    name
    email
  }
}
  • Explicació:
    • La mutació updateUser actualitza el correu electrònic de l'usuari amb id "1".
    • La resposta inclou la informació actualitzada de l'usuari.

Eliminació de Dades

Les mutacions també poden eliminar dades.

mutation {
  deleteUser(id: "1") {
    id
    name
  }
}
  • Explicació:
    • La mutació deleteUser elimina l'usuari amb id "1".
    • La resposta inclou el id i name de l'usuari eliminat.

Combinació de Consultes i Mutacions

Pots combinar múltiples consultes i mutacions en una sola sol·licitud.

mutation {
  createUser(name: "Jane Doe", email: "jane@example.com") {
    id
    name
  }
}

query {
  users {
    id
    name
    email
  }
}
  • Explicació:
    • La mutació createUser crea un nou usuari.
    • La consulta users obté la llista de tots els usuaris.

Conclusió

  • Consultes: S'utilitzen per obtenir dades. Poden ser imbricades, acceptar arguments i utilitzar àlies.
  • Mutacions: S'utilitzen per modificar dades. Poden crear, actualitzar o eliminar dades.
  • Combinar consultes i mutacions en una sola sol·licitud pot optimitzar l'ús de la xarxa.

Entendre com utilitzar efectivament les consultes i mutacions en GraphQL et permetrà construir APIs eficients i flexibles. Practica escrivint les teves pròpies consultes i mutacions per familiaritzar-te amb la sintaxi i les capacitats de GraphQL.
