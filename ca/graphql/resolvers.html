Els resolvers són un concepte fonamental en GraphQL. Són responsables d'obtenir les dades per als camps del teu esquema. Aquest tema et guiarà des dels conceptes bàsics dels resolvers fins a l'ús avançat, assegurant que entenguis com utilitzar-los efectivament en les teves aplicacions GraphQL.

Què són els Resolvers?

  • Definició: Els resolvers són funcions que gestionen l'obtenció de dades per a un camp específic en un esquema GraphQL.
  • Propòsit: Connecten l'esquema amb les fonts de dades subjacents, com bases de dades o APIs.

Estructura Bàsica dels Resolvers

Una funció resolver típicament segueix aquesta estructura:

const resolvers = {
  Query: {
    fieldName: (parent, args, context, info) => {
      // Obtenir i retornar dades per al camp
    },
  },
};
  • parent: El resultat del resolver anterior en la cadena de resolvers.
  • args: Els arguments proporcionats al camp en la consulta.
  • context: Un objecte compartit passat a cada resolver, útil per a l'autenticació, connexions a bases de dades, etc.
  • info: Informació sobre l'estat d'execució de la consulta.

Exemple: Resolver Bàsic

Creem un resolver senzill per a un camp hello que retorna un missatge de salutació.

const resolvers = {
  Query: {
    hello: () => {
      return "Hola, món!";
    },
  },
};

Gestió d'Arguments

Els resolvers poden acceptar arguments per filtrar o modificar les dades que retornen.

Exemple: Resolver amb Arguments

Considera un camp user que obté un usuari per ID:

const resolvers = {
  Query: {
    user: (parent, args, context, info) => {
      const { id } = args;
      return context.db.getUserById(id);
    },
  },
};

En aquest exemple:

  • L'objecte args conté els arguments passats al camp user.
  • context.db.getUserById(id) obté l'usuari d'una base de dades.

Context en els Resolvers

El paràmetre context és útil per compartir dades entre resolvers, com informació d'autenticació o connexions a bases de dades.

Exemple: Utilitzant Context

const resolvers = {
  Query: {
    currentUser: (parent, args, context, info) => {
      return context.user;
    },
  },
};

En aquest exemple:

  • context.user conté la informació de l'usuari autenticat.

Resolvers Niuats

Els resolvers poden estar niuats per gestionar estructures de dades complexes.

Exemple: Resolvers Niuats

Considera un tipus Post amb un camp author:

const resolvers = {
  Query: {
    post: (parent, args, context, info) => {
      return context.db.getPostById(args.id);
    },
  },
  Post: {
    author: (parent, args, context, info) => {
      return context.db.getUserById(parent.authorId);
    },
  },
};

En aquest exemple:

  • El resolver post obté una publicació per ID.
  • El resolver author obté l'autor de la publicació utilitzant l'authorId de l'objecte de la publicació pare.

Gestió d'Errors en els Resolvers

La gestió adequada d'errors és crucial per construir APIs GraphQL robustes.

Exemple: Gestió d'Errors

const resolvers = {
  Query: {
    user: async (parent, args, context, info) => {
      try {
        const user = await context.db.getUserById(args.id);
        if (!user) {
          throw new Error("Usuari no trobat");
        }
        return user;
      } catch (error) {
        throw new Error(`No s'ha pogut obtenir l'usuari: ${error.message}`);
      }
    },
  },
};

En aquest exemple:

  • Els errors són capturats i es llança un missatge d'error significatiu.

Tècniques Avançades de Resolvers

Càrrega per Lots amb DataLoader

DataLoader és una utilitat per carregar i emmagatzemar en memòria cau sol·licituds de bases de dades, reduint el nombre de trucades a la base de dades.

Exemple: Utilitzant DataLoader

const DataLoader = require('dataloader');

const userLoader = new DataLoader(keys => context.db.batchGetUsers(keys));

const resolvers = {
  Query: {
    users: (parent, args, context, info) => {
      return userLoader.loadMany(args.ids);
    },
  },
};

En aquest exemple:

  • DataLoader carrega i emmagatzema en memòria cau sol·licituds per obtenir usuaris per IDs.

Conclusió

Els resolvers són l'espina dorsal de qualsevol servidor GraphQL, connectant el teu esquema amb les teves fonts de dades. Entenent els conceptes bàsics i les tècniques avançades dels resolvers, pots construir APIs GraphQL eficients i escalables. Recorda gestionar els arguments, utilitzar el context amb saviesa, gestionar resolvers niuats i implementar una gestió adequada d'errors per assegurar una aplicació robusta. Les tècniques avançades com DataLoader poden optimitzar encara més les teves estratègies d'obtenció de dades.
