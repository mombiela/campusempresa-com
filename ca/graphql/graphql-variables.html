Introducció a les Variables de GraphQL

Les variables de GraphQL et permeten passar arguments dinàmicament a les teves consultes i mutacions, fent que les teves operacions de GraphQL siguin més flexibles i reutilitzables. En lloc d'incloure valors directament a les teves consultes, pots utilitzar variables per passar diferents valors en temps d'execució.

Conceptes Clau

  • Variables: Marcadors de posició a les teves consultes i mutacions de GraphQL que poden ser reemplaçats amb valors reals en temps d'execució.
  • Definicions de Variables: La part de la consulta de GraphQL on declares les variables que tens la intenció d'utilitzar.
  • Ús de Variables: La part de la consulta de GraphQL on utilitzes les variables declarades.

Definició i Ús de Variables

Per utilitzar variables en una consulta o mutació de GraphQL, segueix aquests passos:

  1. Declara les variables a la consulta o mutació.
  2. Passa les variables quan executis la consulta o mutació.

Exemple: Consulta amb Variables

Pas 1: Declara les Variables

A la consulta, declara les variables amb un prefix $ i especifica els seus tipus.

query GetUser($id: ID!) {
  user(id: $id) {
    id
    name
    email
  }
}
  • $id: ID! declara una variable anomenada id de tipus ID que és no nul·la (!).

Pas 2: Passa les Variables

Quan executis la consulta, passa les variables en un objecte variables separat.

{
  "query": "query GetUser($id: ID!) { user(id: $id) { id name email } }",
  "variables": {
    "id": "123"
  }
}

Exemple: Mutació amb Variables

Pas 1: Declara les Variables

A la mutació, declara les variables amb un prefix $ i especifica els seus tipus.

mutation CreateUser($name: String!, $email: String!) {
  createUser(name: $name, email: $email) {
    id
    name
    email
  }
}
  • $name: String! i $email: String! declaren variables anomenades name i email de tipus String que són no nul·les.

Pas 2: Passa les Variables

Quan executis la mutació, passa les variables en un objecte variables separat.

{
  "query": "mutation CreateUser($name: String!, $email: String!) { createUser(name: $name, email: $email) { id name email } }",
  "variables": {
    "name": "John Doe",
    "email": "john.doe@example.com"
  }
}

Beneficis d'Utilitzar Variables

  • Reutilització: Pots reutilitzar la mateixa consulta o mutació amb diferents valors de variables.
  • Seguretat: Ajuda a prevenir atacs d'injecció separant l'estructura de la consulta de les dades.
  • Llegibilitat: Fa que les consultes i mutacions siguin més netes i fàcils de llegir.

Tipus Comuns de Variables

  • Escalars: Tipus de dades bàsics com Int, Float, String, Boolean i ID.
  • Enums: Tipus escalars personalitzats que representen un conjunt de valors anomenats.
  • Objectes d'Entrada: Objectes complexos que poden ser passats com una sola variable.

Exemple: Objecte d'Entrada

Pas 1: Defineix el Tipus d'Objecte d'Entrada

Al teu esquema de GraphQL, defineix un tipus d'objecte d'entrada.

input UserInput {
  name: String!
  email: String!
}

Pas 2: Utilitza l'Objecte d'Entrada en una Mutació

Declara la variable de l'objecte d'entrada i utilitza-la a la mutació.

mutation CreateUser($input: UserInput!) {
  createUser(input: $input) {
    id
    name
    email
  }
}

Pas 3: Passa l'Objecte d'Entrada

Quan executis la mutació, passa l'objecte d'entrada en l'objecte variables.

{
  "query": "mutation CreateUser($input: UserInput!) { createUser(input: $input) { id name email } }",
  "variables": {
    "input": {
      "name": "Jane Doe",
      "email": "jane.doe@example.com"
    }
  }
}

Conclusió

Les variables de GraphQL són una característica poderosa que millora la flexibilitat, seguretat i llegibilitat de les teves operacions de GraphQL. Entenent com declarar, utilitzar i passar variables, pots crear consultes i mutacions més dinàmiques i reutilitzables. Practica l'ús de variables en diferents escenaris per convertir-te en un expert en aprofitar aquesta característica a les teves aplicacions de GraphQL.
