En aquesta secció, ens endinsarem en el procés d'implementació del backend per a una API GraphQL. Això inclou configurar el servidor, definir l'esquema i crear resolvers. Al final d'aquesta secció, tindràs una comprensió sòlida de com construir un backend funcional de GraphQL.

Configuració del Servidor

Per començar, necessitem configurar un servidor que pugui gestionar sol·licituds GraphQL. Utilitzarem Node.js amb el framework Express i la biblioteca Apollo Server.

Instal·lació de Dependències

Primer, instal·lem els paquets necessaris:

npm init -y
npm install express apollo-server-express graphql

Creació del Servidor

A continuació, crea un fitxer anomenat server.js i afegeix el següent codi per configurar el servidor:

const express = require('express');
const { ApolloServer, gql } = require('apollo-server-express');

const app = express();

// Defineix un esquema simple
const typeDefs = gql`
  type Query {
    hello: String
  }
`;

// Defineix resolvers
const resolvers = {
  Query: {
    hello: () => 'Hola, món!',
  },
};

// Crea una instància d'Apollo Server
const server = new ApolloServer({ typeDefs, resolvers });

// Aplica el middleware d'Apollo GraphQL i estableix el camí a /graphql
server.applyMiddleware({ app, path: '/graphql' });

// Inicia el servidor
app.listen({ port: 4000 }, () =>
  console.log(`Servidor llest a http://localhost:4000${server.graphqlPath}`)
);

Explicació

  • Dependències: Importem express, ApolloServer i gql dels seus respectius paquets.
  • Definició de l'Esquema: Definim un esquema simple amb un únic tipus Query que retorna una cadena.
  • Resolvers: Creem un resolver per a la consulta hello que retorna una cadena estàtica.
  • Apollo Server: Creem una instància d'ApolloServer amb l'esquema i els resolvers, i després l'apliquem com a middleware a l'aplicació Express.
  • Inici del Servidor: Finalment, iniciem el servidor al port 4000.

Definició de l'Esquema

L'esquema és una part crucial de qualsevol API GraphQL. Defineix els tipus, consultes i mutacions que els clients poden executar.

Esquema Bàsic

Ampliem el nostre esquema per incloure tipus i relacions més complexes. Actualitza els typeDefs a server.js:

const typeDefs = gql`
  type Book {
    title: String
    author: Author
  }

  type Author {
    name: String
    books: [Book]
  }

  type Query {
    books: [Book]
    authors: [Author]
  }
`;

Explicació

  • Tipus: Definim els tipus Book i Author. Cada Book té un title i un author, i cada Author té un name i una llista de books.
  • Consultes: Afegim consultes per obtenir llistes de llibres i autors.

Creació de Resolvers

Els resolvers proporcionen la lògica per obtenir les dades definides a l'esquema.

Resolvers Bàsics

Actualitza els resolvers a server.js:

const books = [
  { title: 'The Awakening', author: 'Kate Chopin' },
  { title: 'City of Glass', author: 'Paul Auster' },
];

const authors = [
  { name: 'Kate Chopin', books: ['The Awakening'] },
  { name: 'Paul Auster', books: ['City of Glass'] },
];

const resolvers = {
  Query: {
    books: () => books,
    authors: () => authors,
  },
  Book: {
    author: (book) => authors.find((author) => author.name === book.author),
  },
  Author: {
    books: (author) => books.filter((book) => book.author === author.name),
  },
};

Explicació

  • Dades Estàtiques: Utilitzem arrays estàtics per a llibres i autors per simplicitat.
  • Resolvers de Consultes: Proporcionem resolvers per a les consultes books i authors que retornen les dades estàtiques.
  • Resolvers de Camps: Afegim resolvers de camps per a Book.author i Author.books per resoldre les relacions entre llibres i autors.

Conclusió

En aquesta secció, hem cobert els conceptes bàsics de la configuració d'un backend GraphQL utilitzant Node.js, Express i Apollo Server. Hem definit un esquema simple i hem creat resolvers per obtenir dades. Aquest coneixement fonamental et permetrà construir APIs GraphQL més complexes a mesura que avancis en el curs.

En entendre com configurar el servidor, definir l'esquema i crear resolvers, ara estàs equipat per gestionar els components principals d'un backend GraphQL. En les properes seccions, explorarem temes més avançats com l'autenticació, la gestió d'errors i l'optimització del rendiment.
