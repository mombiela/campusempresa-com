Construir un Servidor Web Simple en Haskell
Accés a Bases de Dades ►

Introducció

En aquest tema, explorarem com construir un servidor web simple utilitzant Haskell. Això cobrirà els conceptes bàsics de la configuració d'un entorn Haskell, la comprensió de HTTP i l'ús de les biblioteques wai i warp per crear un servidor web funcional.

Prerequisits

Abans de començar, assegura't de tenir:

  • Comprensió bàsica de la sintaxi i les funcions de Haskell.
  • Haskell instal·lat a la teva màquina (GHC i Cabal o Stack).

Configurant l'Entorn

Per començar, necessitem configurar el nostre entorn Haskell i instal·lar les biblioteques necessàries.

Instal·lant Haskell

  1. GHC (Glasgow Haskell Compiler): El compilador principal per a Haskell.
  2. Cabal o Stack: Eines de construcció per gestionar projectes Haskell.

Instal·lant Biblioteques

Utilitzarem les biblioteques wai (Web Application Interface) i warp (una biblioteca de servidor HTTP ràpida).

cabal update
cabal install wai warp

O, si utilitzes Stack:

stack update
stack install wai warp

Comprenent els Conceptes Bàsics de HTTP

Abans de submergir-nos en el codi, entenem breument HTTP, el protocol utilitzat pels servidors web per comunicar-se amb els clients (navegadors).

  • Request: Enviat pel client al servidor, inclou mètodes com GET, POST, etc.
  • Response: Enviat pel servidor al client, inclou codis d'estat (200, 404, etc.) i dades.

Construint el Servidor Web

Creant un Servidor Bàsic

Comencem creant un servidor web simple que respongui amb "Hello, World!" a qualsevol sol·licitud.

  1. Crea un nou fitxer Haskell:

    touch Main.hs

  2. Escriu el següent codi a Main.hs:

    {-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app _ respond = respond $ responseLBS status200 [("Content-Type", "text/plain")] "Hello, World!"

Explicació

  • Imports: Importem els mòduls necessaris de wai, warp i http-types.
  • Funció Principal: Inicia el servidor al port 8080 i utilitza la funció app per gestionar les sol·licituds.
  • Funció App: Defineix la lògica de l'aplicació, que respon amb un estat 200 i el missatge "Hello, World!".

Executant el Servidor

Per executar el servidor, utilitza la següent comanda:

runhaskell Main.hs

Obre el teu navegador i navega a http://localhost:8080 per veure "Hello, World!".

Afegint Rutes

A continuació, afegim algunes rutes per gestionar diferents camins.

Exemple de Codi

Actualitza Main.hs per incloure el routing:

{-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application, pathInfo)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200, status404)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app req respond = case pathInfo req of
    ["hello"] -> respond $ responseLBS status200 [("Content-Type", "text/plain")] "Hello, World!"
    ["goodbye"] -> respond $ responseLBS status200 [("Content-Type", "text/plain")] "Goodbye, World!"
    _ -> respond $ responseLBS status404 [("Content-Type", "text/plain")] "404 - Not Found"

Explicació

  • pathInfo: Extreu el camí de la sol·licitud.
  • Coincidència de Casos: Coincideix el camí amb rutes predefinides i respon en conseqüència.

Executant el Servidor

Executa el servidor de nou i navega a:

  • http://localhost:8080/hello per veure "Hello, World!"
  • http://localhost:8080/goodbye per veure "Goodbye, World!"
  • Qualsevol altre camí retornarà "404 - Not Found".

Gestionant Paràmetres de Consulta

Gestionem els paràmetres de consulta per fer el nostre servidor més dinàmic.

Exemple de Codi

Actualitza Main.hs per gestionar els paràmetres de consulta:

{-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application, queryString)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200, status400)
import Data.Maybe (fromMaybe)
import Data.ByteString.Lazy.Char8 (pack)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app req respond = case pathInfo req of
    ["greet"] -> do
        let name = fromMaybe "World" $ lookup "name" $ queryString req
        respond $ responseLBS status200 [("Content-Type", "text/plain")] (pack $ "Hello, " ++ show name ++ "!")
    _ -> respond $ responseLBS status400 [("Content-Type", "text/plain")] "400 - Bad Request"

Explicació

  • queryString: Extreu els paràmetres de consulta de la sol·licitud.
  • fromMaybe: Proporciona un valor per defecte si no es troba el paràmetre.
  • pack: Converteix una cadena a un ByteString.

Executant el Servidor

Executa el servidor i navega a:

  • http://localhost:8080/greet?name=Haskell per veure "Hello, Haskell!"

Conclusió

En aquest tema, hem cobert els conceptes bàsics de la construcció d'un servidor web simple en Haskell utilitzant les biblioteques wai i warp. Hem començat amb un servidor bàsic, hem afegit routing i hem gestionat paràmetres de consulta. Aquesta base et permetrà construir aplicacions web més complexes en Haskell.

Resum

  • Hem configurat l'entorn Haskell i hem instal·lat les biblioteques necessàries.
  • Hem construït un servidor web bàsic que respon amb "Hello, World!".
  • Hem afegit routing per gestionar diferents camins.
  • Hem gestionat paràmetres de consulta per fer el servidor dinàmic.

Amb aquestes habilitats, ara pots explorar temes més avançats com middleware, integració de bases de dades i routing més complex.
