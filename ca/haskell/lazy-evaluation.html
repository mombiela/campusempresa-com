Avaluació mandrosa a Haskell
Estructures de Dades Infinites ►

L'avaluació mandrosa és una de les característiques definidores de Haskell. Permet al llenguatge ajornar els càlculs fins que els seus resultats siguin realment necessaris, cosa que pot conduir a millores en el rendiment i la capacitat de treballar amb estructures de dades infinites.

Què és l'avaluació mandrosa?

L'avaluació mandrosa, també coneguda com a call-by-need, és una estratègia on les expressions no s'avaluen quan s'assignen a variables, sinó quan els seus resultats són requerits per altres càlculs.

Conceptes clau

  • Càlcul ajornat: Les expressions només s'avaluen quan els seus valors són necessaris.
  • Thunks: Un thunk és un càlcul ajornat. Quan una expressió s'ajorna, es guarda com un thunk.
  • Memoització: Un cop un thunk s'avalua, el seu resultat es guarda per evitar una reavaluació.

Beneficis de l'avaluació mandrosa

  • Eficiència: Evita càlculs innecessaris.
  • Modularitat: Permet la definició d'estructures de control com a funcions regulars.
  • Estructures de dades infinites: Permet l'ús de llistes infinites i altres estructures de dades.

Exemples d'avaluació mandrosa

Exemple bàsic

Considera el següent codi Haskell:

square :: Int -> Int
square x = x * x

main :: IO ()
main = do
    let a = square 3
    let b = square 4
    print (a + b)

En aquest exemple, square 3 i square 4 no s'avaluen immediatament quan s'assignen a a i b. Només s'avaluen quan a + b és necessari per a la funció print.

Llistes infinites

Un dels usos més poderosos de l'avaluació mandrosa és amb llistes infinites:

naturals :: [Int]
naturals = [0..]

main :: IO ()
main = print (take 10 naturals)

Aquí, naturals és una llista infinita de nombres naturals. Gràcies a l'avaluació mandrosa, Haskell pot manejar aquesta llista infinita i només calcular els primers 10 elements quan take 10 naturals s'avalua.

Avaluació mandrosa a la pràctica

Evitant càlculs innecessaris

L'avaluació mandrosa pot ajudar a evitar càlculs innecessaris. Considera el següent exemple:

expensiveComputation :: Int -> Int
expensiveComputation x = x * x * x * x

main :: IO ()
main = do
    let result = if False then expensiveComputation 100000 else 0
    print result

En aquest cas, expensiveComputation 100000 mai s'avalua perquè la condició és False. Amb l'avaluació impacient, aquest càlcul es realitzaria independentment de la condició.

Treballant amb dades grans

L'avaluació mandrosa et permet treballar amb estructures de dades grans de manera eficient:

largeList :: [Int]
largeList = [1..1000000]

main :: IO ()
main = print (sum (take 100 largeList))

Aquí, largeList és una llista amb un milió d'elements, però només es sumen els primers 100 elements. Gràcies a l'avaluació mandrosa, la llista completa no es genera de cop.

Conclusió

L'avaluació mandrosa és una característica poderosa de Haskell que permet càlculs eficients, codi modular i l'ús d'estructures de dades infinites. Entenent i aprofitant l'avaluació mandrosa, pots escriure programes Haskell més eficients i expressius. Recorda:

  • Els càlculs s'ajornen fins que els seus resultats són necessaris.
  • Els thunks representen càlculs ajornats.
  • La memoització evita la reavaluació de les expressions.

Amb aquests conceptes en ment, pots aprofitar al màxim l'avaluació mandrosa de Haskell per escriure codi més eficient i elegant.
