Introducció a l'Accés a Bases de Dades

L'accés a bases de dades és un aspecte crucial de moltes aplicacions, permetent l'emmagatzematge, recuperació i manipulació de dades. En Haskell, l'accés a bases de dades es pot aconseguir utilitzant diverses biblioteques i frameworks. Aquesta secció et guiarà a través dels conceptes bàsics de l'accés a bases de dades en Haskell, des de la configuració d'una connexió fins a la realització d'operacions CRUD (Crear, Llegir, Actualitzar, Esborrar).

Configuració de l'Entorn

Abans de submergir-nos en el codi, configurem l'entorn necessari.

  1. Instal·la GHC i Cabal: Assegura't de tenir el Glasgow Haskell Compiler (GHC) i Cabal instal·lats.
  2. Crea un Nou Projecte: Utilitza Cabal per crear un nou projecte Haskell. 
    cabal init
  3. Afegeix Dependències: Afegeix les biblioteques necessàries al teu fitxer cabal. Per a l'accés a bases de dades, utilitzarem persistent i persistent-sqlite per a SQLite com a exemple. 
    build-depends: base >=4.7 && <5,
               persistent,
               persistent-sqlite,
               persistent-template,
               text

Connexió a una Base de Dades

Per connectar-te a una base de dades, necessites definir el teu esquema de base de dades i configurar la connexió.

Definició de l'Esquema

Defineix el teu esquema de base de dades utilitzant la sintaxi Template Haskell de Persistent.

{-# LANGUAGE GADTs #-}
{-# LANGUAGE OverloadedStrings #-}
{-# LANGUAGE QuasiQuotes #-}
{-# LANGUAGE TemplateHaskell #-}
{-# LANGUAGE TypeFamilies #-}

import Database.Persist
import Database.Persist.Sqlite
import Database.Persist.TH
import Control.Monad.IO.Class (liftIO)

share [mkPersist sqlSettings, mkMigrate "migrateAll"] [persistLowerCase|
Person
    name String
    age Int
    deriving Show
|]

Configuració de la Connexió

Configura la connexió a la base de dades SQLite i executa la migració per crear les taules.

main :: IO ()
main = runSqlite "test.db" $ do
    runMigration migrateAll
    liftIO $ putStrLn "Base de dades connectada i esquema migrat."

Realització d'Operacions CRUD

Un cop la connexió està configurada, pots realitzar operacions CRUD.

Crear

Insereix un nou registre a la base de dades.

addPerson :: String -> Int -> IO ()
addPerson name age = runSqlite "test.db" $ do
    personId <- insert $ Person name age
    liftIO $ putStrLn $ "Persona inserida amb ID: " ++ show personId

Llegir

Recupera registres de la base de dades.

getPeople :: IO ()
getPeople = runSqlite "test.db" $ do
    people <- selectList [] []
    liftIO $ mapM_ (print . entityVal) people

Actualitzar

Actualitza un registre existent.

updatePersonAge :: PersonId -> Int -> IO ()
updatePersonAge personId newAge = runSqlite "test.db" $ do
    update personId [PersonAge =. newAge]
    liftIO $ putStrLn $ "Persona actualitzada amb ID: " ++ show personId

Esborrar

Esborra un registre de la base de dades.

deletePerson :: PersonId -> IO ()
deletePerson personId = runSqlite "test.db" $ do
    delete personId
    liftIO $ putStrLn $ "Persona esborrada amb ID: " ++ show personId

Temes Avançats

Per a usuaris avançats, Haskell ofereix operacions de bases de dades més sofisticades i optimitzacions.

Transaccions

Realitza múltiples operacions dins d'una transacció per assegurar l'atomicitat.

transactionExample :: IO ()
transactionExample = runSqlite "test.db" $ do
    personId <- insert $ Person "Alice" 30
    update personId [PersonAge =. 31]
    delete personId
    liftIO $ putStrLn "Transacció completada."

Optimització de Consultes

Utilitza índexs i consultes optimitzades per millorar el rendiment.

-- Exemple de creació d'un índex
share [mkPersist sqlSettings, mkMigrate "migrateAll"] [persistLowerCase|
Person
    name String
    age Int
    deriving Show
    UniqueName name
|]

Conclusió

En aquesta secció, hem cobert els conceptes bàsics de l'accés a bases de dades en Haskell utilitzant la biblioteca Persistent. Has après com configurar una connexió, definir un esquema i realitzar operacions CRUD. Per a usuaris avançats, hem tocat temes com les transaccions i l'optimització de consultes. Amb aquestes habilitats, pots gestionar eficientment operacions de bases de dades en les teves aplicacions Haskell.
