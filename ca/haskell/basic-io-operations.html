Introducció

A Haskell, les operacions d'entrada/sortida (I/O) es gestionen de manera diferent en comparació amb els llenguatges imperatius. Haskell és un llenguatge purament funcional, el que significa que les funcions no tenen efectes secundaris. No obstant això, les operacions d'I/O inherentment tenen efectes secundaris, així que Haskell utilitza un tipus especial anomenat IO per gestionar aquestes operacions. Aquesta secció cobrirà els conceptes bàsics de l'I/O a Haskell, incloent la lectura i escriptura a la consola, treballar amb fitxers i entendre el tipus IO.

Conceptes clau

  • Funcions pures vs. Accions d'I/O: Entendre la diferència entre funcions pures i accions d'I/O.
  • El tipus IO: Introducció al tipus IO i com encapsula els efectes secundaris.
  • Funcions bàsiques d'I/O: Utilitzar putStrLn, getLine, readFile i writeFile.
  • Combinació d'accions d'I/O: Utilitzar la notació do per seqüenciar accions d'I/O.

Funcions pures vs. Accions d'I/O

A Haskell, les funcions pures sempre produeixen la mateixa sortida per la mateixa entrada i no tenen efectes secundaris. Les accions d'I/O, en canvi, interactuen amb el món exterior i poden produir resultats diferents cada vegada que s'executen.

-- Exemple de funció pura
add :: Int -> Int -> Int
add x y = x + y

-- Exemple d'acció d'I/O
main :: IO ()
main = do
    putStrLn "Enter your name:"
    name <- getLine
    putStrLn ("Hello, " ++ name ++ "!")

El tipus `IO`

El tipus IO s'utilitza per representar accions que realitzen I/O. Una acció IO és un valor que, quan s'executa, interactua amb el món exterior i produeix un resultat.

-- Exemple d'una acció IO
main :: IO ()
main = putStrLn "Hello, World!"

En l'exemple anterior, putStrLn "Hello, World!" és una acció IO que imprimeix una cadena a la consola.

Funcions bàsiques d'I/O

`putStrLn`

putStrLn s'utilitza per imprimir una cadena seguida d'un salt de línia a la consola.

main :: IO ()
main = putStrLn "Hello, World!"

`getLine`

getLine llegeix una línia d'entrada de la consola.

main :: IO ()
main = do
    putStrLn "Enter your name:"
    name <- getLine
    putStrLn ("Hello, " ++ name ++ "!")

`readFile` i `writeFile`

readFile llegeix el contingut d'un fitxer com una cadena, i writeFile escriu una cadena a un fitxer.

main :: IO ()
main = do
    contents <- readFile "example.txt"
    putStrLn contents

writeExample :: IO ()
writeExample = do
    writeFile "example.txt" "This is an example file."

Combinació d'accions d'I/O

Per realitzar múltiples accions d'I/O en seqüència, utilitzem la notació do. Això ens permet escriure una sèrie d'accions que s'executen en ordre.

main :: IO ()
main = do
    putStrLn "Enter the first number:"
    num1 <- getLine
    putStrLn "Enter the second number:"
    num2 <- getLine
    let sum = (read num1 :: Int) + (read num2 :: Int)
    putStrLn ("The sum is: " ++ show sum)

En aquest exemple, llegim dos números de l'usuari, calculem la seva suma i imprimim el resultat.

Conclusió

Entendre les operacions bàsiques d'I/O a Haskell és crucial per interactuar amb el món exterior. El tipus IO encapsula els efectes secundaris, permetent que Haskell mantingui la seva puresa funcional. Utilitzant funcions com putStrLn, getLine, readFile i writeFile, i combinant-les amb la notació do, pots realitzar operacions d'I/O complexes d'una manera estructurada i llegible.
