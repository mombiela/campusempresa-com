Proves i Depuració en Haskell
Projecte: Construcció d'una Aplicació Haskell ►

Introducció

Les proves i la depuració són aspectes crucials del desenvolupament de programari. En Haskell, aquestes pràctiques asseguren que el teu codi sigui correcte, eficient i mantenible. Aquesta secció cobrirà els conceptes bàsics de les proves i la depuració en Haskell, des de nivells principiants fins a avançats.

Proves en Haskell

Per què són importants les proves

  • Asseguren la correcció del codi.
  • Faciliten el manteniment del codi.
  • Ajuden a identificar errors aviat.
  • Proporcionen documentació sobre el comportament del codi.

Tipus de proves

  • Proves Unitàries: Proves d'unitats individuals de codi.
  • Proves Basades en Propietats: Proves de propietats que haurien de mantenir-se per a un rang d'entrades.
  • Proves d'Integració: Proves de parts combinades d'una aplicació per assegurar-se que funcionen juntes.

Proves Unitàries amb HUnit

HUnit és un marc de proves unitàries per a Haskell, similar a JUnit per a Java.

Instal·lació

Per instal·lar HUnit, afegeix-lo al fitxer .cabal del teu projecte:

build-depends: base >= 4.7 && < 5, HUnit >= 1.6 && < 2

Exemple

import Test.HUnit

-- Funció a provar
add :: Int -> Int -> Int
add x y = x + y

-- Casos de prova
test1 = TestCase (assertEqual "per (add 1 2)," 3 (add 1 2))
test2 = TestCase (assertEqual "per (add (-1) 1)," 0 (add (-1) 1))

-- Agrupació de proves
tests = TestList [TestLabel "test1" test1, TestLabel "test2" test2]

-- Execució de proves
main :: IO Counts
main = runTestTT tests

Proves Basades en Propietats amb QuickCheck

QuickCheck és una biblioteca per a proves aleatòries de propietats de programes.

Instal·lació

Per instal·lar QuickCheck, afegeix-lo al fitxer .cabal del teu projecte:

build-depends: base >= 4.7 && < 5, QuickCheck >= 2.14 && < 3

Exemple

import Test.QuickCheck

-- Propietat a provar
prop_reverse :: [Int] -> Bool
prop_reverse xs = reverse (reverse xs) == xs

-- Execució de la prova de propietat
main :: IO ()
main = quickCheck prop_reverse

Proves d'Integració

Les proves d'integració en Haskell sovint impliquen combinar proves unitàries i proves basades en propietats per assegurar-se que diferents parts de l'aplicació funcionen juntes com s'espera.

Depuració en Haskell

Tècniques Comunes de Depuració

  • Depuració amb Print: Utilitzar instruccions print per traçar l'execució del codi.
  • Depurador de GHCi: Depuració interactiva utilitzant GHCi.
  • Registre: Utilitzar biblioteques de registre per enregistrar informació d'execució.

Depuració amb Print

Utilitzar instruccions print per entendre el flux del programa.

factorial :: Int -> Int
factorial 0 = 1
factorial n = n * factorial (n - 1)

main :: IO ()
main = do
    let result = factorial 5
    print result

Depurador de GHCi

GHCi proporciona un entorn interactiu per depurar codi Haskell.

Exemple

-- Carregar el fitxer en GHCi
:load MyModule.hs

-- Establir un punt de ruptura
:break 10

-- Executar la funció
main

-- Passar pel codi
:step

Registre

Utilitzar la biblioteca hslogger per al registre.

Instal·lació

Per instal·lar hslogger, afegeix-lo al fitxer .cabal del teu projecte:

build-depends: base >= 4.7 && < 5, hslogger >= 1.3 && < 2

Exemple

import System.Log.Logger

main :: IO ()
main = do
    updateGlobalLogger rootLoggerName (setLevel DEBUG)
    debugM "Main" "Aquest és un missatge de depuració"
    infoM "Main" "Aquest és un missatge d'informació"
    warningM "Main" "Aquest és un missatge d'advertència"
    errorM "Main" "Aquest és un missatge d'error"

Conclusió

Les proves i la depuració són pràctiques essencials en la programació en Haskell. Les proves unitàries amb HUnit i les proves basades en propietats amb QuickCheck ajuden a assegurar la correcció del codi. Les tècniques de depuració com la depuració amb print, l'ús del depurador de GHCi i el registre són eines invaluables per identificar i solucionar problemes en el teu codi. Dominant aquestes tècniques, podràs escriure programes en Haskell més fiables i mantenibles.
