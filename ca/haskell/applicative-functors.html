Introducció als Funtors Aplicatius

Els Funtors Aplicatius són una característica poderosa en Haskell que permet l'aplicació de funcions elevades sobre un context computacional. Són més poderosos que els Funtors però menys poderosos que els Monads. Entendre els Funtors Aplicatius és crucial per escriure codi Haskell concís i expressiu.

Conceptes Clau

  • Functor: Una classe de tipus que permet aplicar una funció a valors embolcallats en un context.
  • Funtor Aplicatiu: Estén Functor permetent que funcions que estan embolcallades en un context siguin aplicades a valors embolcallats en un context.
  • Pure: Una funció que pren un valor i el retorna embolcallat en un context.
  • <*> (Apply): Un operador que aplica una funció embolcallada en un context a un valor embolcallat en un context.

Resum de Functor

Abans d'endinsar-nos en els Funtors Aplicatius, repassem ràpidament els Funtors.

class Functor f where
    fmap :: (a -> b) -> f a -> f b

Exemple amb Maybe:

fmap (+1) (Just 2) -- Resultat: Just 3
fmap (+1) Nothing  -- Resultat: Nothing

Conceptes Bàsics de Funtor Aplicatiu

La classe de tipus Applicative es defineix de la següent manera:

class Functor f => Applicative f where
    pure :: a -> f a
    (<*>) :: f (a -> b) -> f a -> f b

Pure

La funció pure pren un valor i l'embolcalla en un context.

pure 3 :: Maybe Int -- Resultat: Just 3
pure 3 :: [Int]     -- Resultat: [3]

Apply (<*>)

L'operador <*> aplica una funció embolcallada en un context a un valor embolcallat en un context.

(Just (+1)) <*> (Just 2) -- Resultat: Just 3
Nothing <*> (Just 2)     -- Resultat: Nothing
(Just (+1)) <*> Nothing  -- Resultat: Nothing

Exemple amb Llistes

Les llistes també són Funtors Aplicatius.

pure (+1) <*> [1, 2, 3] -- Resultat: [2, 3, 4]
[(+1), (*2)] <*> [1, 2, 3] -- Resultat: [2, 3, 4, 2, 4, 6]

Combinació de Funtors Aplicatius

Els Funtors Aplicatius es poden combinar per aplicar funcions a múltiples valors embolcallats.

pure (+) <*> Just 3 <*> Just 5 -- Resultat: Just 8
pure (+) <*> Just 3 <*> Nothing -- Resultat: Nothing

Exemple amb Funcions

Les funcions en si mateixes poden ser Funtors Aplicatius.

pure (+) <*> (+3) <*> (*2) $ 5 -- Resultat: 16
-- Explicació: ((+3) 5) + ((*2) 5) = 8 + 10 = 18

Exemple Pràctic: Validació

Els Funtors Aplicatius són útils en escenaris com la validació on múltiples càlculs necessiten ser combinats.

import Control.Applicative

data Validation e a = Failure e | Success a deriving (Show)

instance Functor (Validation e) where
    fmap _ (Failure e) = Failure e
    fmap f (Success a) = Success (f a)

instance Monoid e => Applicative (Validation e) where
    pure = Success
    (Failure e1) <*> (Failure e2) = Failure (e1 <> e2)
    (Failure e) <*> _ = Failure e
    _ <*> (Failure e) = Failure e
    (Success f) <*> (Success a) = Success (f a)

validateName :: String -> Validation [String] String
validateName name
    | null name = Failure ["Name cannot be empty"]
    | otherwise = Success name

validateAge :: Int -> Validation [String] Int
validateAge age
    | age <= 0 = Failure ["Age must be positive"]
    | otherwise = Success age

validatePerson :: String -> Int -> Validation [String] (String, Int)
validatePerson name age = (,) <$> validateName name <*> validateAge age

-- Exemple d'ús:
-- validatePerson "" 0
-- Resultat: Failure ["Name cannot be empty","Age must be positive"]
-- validatePerson "Alice" 30
-- Resultat: Success ("Alice", 30)

Conclusió

Els Funtors Aplicatius proporcionen un punt intermedi entre els Funtors i els Monads, permetent un codi més flexible i expressiu. Són particularment útils en escenaris on múltiples càlculs independents necessiten ser combinats, com la validació. Entendre els Funtors Aplicatius millorarà la teva capacitat per escriure codi Haskell concís i poderós.
