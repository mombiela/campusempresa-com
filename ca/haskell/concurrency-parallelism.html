Introducció

La concurrència i el paral·lelisme són conceptes essencials en la programació moderna, especialment per millorar el rendiment i la capacitat de resposta en aplicacions. Haskell, amb el seu fort èmfasi en la programació funcional, proporciona eines i biblioteques robustes per manejar aquests conceptes de manera eficient.

Conceptes clau

  • Concurrència: La capacitat de manejar múltiples tasques al mateix temps, avançant en cadascuna sense necessàriament executar-les simultàniament.
  • Paral·lelisme: L'execució simultània de múltiples tasques, típicament per accelerar els càlculs.

Concurrència en Haskell

El monad `IO` i la concurrència

Haskell utilitza el monad IO per manejar efectes secundaris, incloent-hi la concurrència. El mòdul Control.Concurrent proporciona les eines necessàries.

Creació de fils

import Control.Concurrent

main :: IO ()
main = do
    forkIO $ putStrLn "Hola des del nou fil!"
    putStrLn "Hola des del fil principal!"
  • forkIO: Crea un nou fil lleuger per executar l'acció IO donada.
  • El fil principal i el nou fil s'executen concurrentment.

MVars per a la comunicació

MVar és una ubicació mutable que està buida o conté un valor, utilitzada per a la comunicació entre fils.

import Control.Concurrent
import Control.Concurrent.MVar

main :: IO ()
main = do
    mvar <- newEmptyMVar
    forkIO $ do
        putStrLn "Fil fill: Posant valor a MVar"
        putMVar mvar "Hola des de MVar"
    value <- takeMVar mvar
    putStrLn $ "Fil principal: Valor rebut: " ++ value
  • newEmptyMVar: Crea un MVar buit.
  • putMVar: Posa un valor al MVar.
  • takeMVar: Pren un valor del MVar, bloquejant-se si està buit.

Paral·lelisme en Haskell

Les funcions `par` i `pseq`

El mòdul Control.Parallel de Haskell proporciona construccions bàsiques de paral·lelisme.

import Control.Parallel

main :: IO ()
main = do
    let x = expensiveComputation 1
        y = expensiveComputation 2
    x `par` y `pseq` print (x + y)

expensiveComputation :: Int -> Int
expensiveComputation n = sum [1..1000000 * n]
  • par: Suggerix que el seu primer argument es pot avaluar en paral·lel amb el segon.
  • pseq: Assegura que el seu primer argument s'avalua abans que el segon.

Estratègies per al paral·lelisme

El mòdul Control.Parallel.Strategies proporciona abstraccions de més alt nivell per al paral·lelisme.

import Control.Parallel.Strategies

main :: IO ()
main = do
    let results = parMap rpar expensiveComputation [1..4]
    print results

expensiveComputation :: Int -> Int
expensiveComputation n = sum [1..1000000 * n]
  • parMap: Una versió paral·lela de map.
  • rpar: Una estratègia que avalua cada element en paral·lel.

Comparació entre concurrència i paral·lelisme

| Característica | Concurrència | Paral·lelisme | |-------------------|--------------------------------------------------|--------------------------------------------------| | Propòsit | Manejar múltiples tasques concurrentment | Accelerar càlculs executant tasques en paral·lel | | Mòdul principal | Control.Concurrent | Control.Parallel, Control.Parallel.Strategies| | Funcions clau | forkIO, MVar | par, pseq, parMap, rpar | | Usos típics | Operacions d'E/S, aplicacions responsives | Càlculs intensius de CPU, processament de dades |

Conclusió

La concurrència i el paral·lelisme són eines poderoses en Haskell que poden millorar significativament el rendiment i la capacitat de resposta de les teves aplicacions. Entenent i utilitzant els mòduls Control.Concurrent i Control.Parallel, pots gestionar efectivament múltiples tasques i aprofitar al màxim els processadors multicore. Practica amb els exemples proporcionats i explora més per dominar aquests conceptes en Haskell.
