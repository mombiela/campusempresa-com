Introducció

L'estil i el format del codi són crucials en la programació en Haskell per assegurar la llegibilitat, mantenibilitat i col·laboració. Aquesta secció cobrirà les millors pràctiques per escriure codi Haskell net i consistent, des de regles bàsiques de format fins a guies estilístiques avançades.

Regles Bàsiques de Format

Indentació

  • Consistència: Utilitza una indentació consistent al llarg del teu codi. L'estàndard és de 2 o 4 espais per nivell d'indentació.
  • Sense Tabuladors: Evita utilitzar tabuladors; utilitza espais per a la indentació.
-- Bona indentació
main :: IO ()
main = do
  putStrLn "Hello, World!"
  putStrLn "Welcome to Haskell!"

-- Mala indentació
main :: IO ()
main = do
 putStrLn "Hello, World!"
  putStrLn "Welcome to Haskell!"

Longitud de la Línia

  • Limitar la Longitud de la Línia: Mantingues les línies amb un màxim de 80 caràcters per millorar la llegibilitat.
  • Trencar Línies: Si una línia supera els 80 caràcters, trenca-la en múltiples línies.
-- Bona longitud de línia
longFunctionName :: Int -> Int -> Int -> Int
longFunctionName x y z = x + y + z

-- Mala longitud de línia
longFunctionName :: Int -> Int -> Int -> Int = x + y + z

Espais en Blanc

  • Al Voltant dels Operadors: Utilitza espais al voltant dels operadors per a una millor llegibilitat.
  • Dins dels Parèntesis: Evita espais dins dels parèntesis.
-- Bon ús dels espais en blanc
sum = a + b * c

-- Mal ús dels espais en blanc
sum=a+b*c

Convencions de Nomenclatura

Variables i Funcions

  • Camel Case: Utilitza camelCase per als noms de variables i funcions.
  • Noms Descriptius: Tria noms descriptius que transmetin la finalitat de la variable o funció.
-- Bona nomenclatura
calculateSum :: Int -> Int -> Int
calculateSum firstNumber secondNumber = firstNumber + secondNumber

-- Mala nomenclatura
cs :: Int -> Int -> Int
cs x y = x + y

Tipus i Constructors

  • Pascal Case: Utilitza PascalCase per als noms de tipus i constructors.
-- Bona nomenclatura
data Person = Person { firstName :: String, lastName :: String }

-- Mala nomenclatura
data person = person { firstname :: String, lastname :: String }

Comentaris

Comentaris en Línia

  • Finalitat: Utilitza comentaris en línia per explicar lògica complexa o detalls importants.
  • Col·locació: Col·loca els comentaris a la mateixa línia o a sobre del codi que descriuen.
-- Bon comentari en línia
factorial :: Int -> Int
factorial n
  | n == 0    = 1  -- Cas base: 0! és 1
  | otherwise = n * factorial (n - 1)

-- Mal comentari en línia
factorial :: Int -> Int
factorial n
  | n == 0 = 1 -- Cas base: 0! és 1
  | otherwise = n * factorial (n - 1)

Comentaris en Bloc

  • Ús: Utilitza comentaris en bloc per a grans seccions de codi o documentació a nivell de mòdul.
  • Format: Comença i acaba els comentaris en bloc amb {- i -}.
{- 
  Aquesta funció calcula el factorial d'un nombre.
  Utilitza la recursió per calcular el resultat.
-}
factorial :: Int -> Int
factorial n
  | n == 0    = 1
  | otherwise = n * factorial (n - 1)

Guies Estilístiques Avançades

Composició de Funcions

  • Utilitza Composició: Prefereix la composició de funcions sobre les crides de funcions niades per a més claredat.
-- Bona composició de funcions
processData = map toUpper . filter isLetter

-- Males crides de funcions niades
processData xs = map toUpper (filter isLetter xs)

Estil Point-Free

  • Point-Free: Utilitza l'estil point-free quan sigui apropiat per fer el codi més concís.
-- Bon estil point-free
sumSquares = sum . map (^2)

-- Mal estil point-free
sumSquares xs = sum (map (^2) xs)

Conclusió

Adherir-se a les guies d'estil i format del codi en Haskell és essencial per escriure codi net, llegible i mantenible. Seguint una indentació consistent, convencions de nomenclatura i pràctiques de comentaris, pots millorar la qualitat dels teus programes Haskell i facilitar la col·laboració amb altres desenvolupadors. Recorda utilitzar tècniques avançades com la composició de funcions i l'estil point-free amb prudència per millorar la claredat i concisió del codi.
