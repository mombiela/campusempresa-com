Introducció al Maneig d'Excepcions

El maneig d'excepcions és un aspecte crucial de la programació que permet als desenvolupadors gestionar errors i condicions excepcionals de manera elegant. En Haskell, el maneig d'excepcions s'aborda de manera diferent en comparació amb els llenguatges imperatius a causa de la seva naturalesa funcional.

Conceptes Bàsics

  • Excepció: Un esdeveniment inesperat que interromp el flux normal d'un programa.
  • Maneig d'Excepcions: Mecanismes per capturar i gestionar excepcions per evitar que el programa es bloquegi.

Tipus d'Excepcions

Haskell categoritza les excepcions en:

  • Excepcions Sincròniques: Errors que ocorren durant l'execució d'un programa, com ara divisió per zero o fitxer no trobat.
  • Excepcions Asincròniques: Esdeveniments externs que interrompen el programa, com ara interrupcions d'usuari o temps d'espera esgotats.

Maneig d'Excepcions amb `Maybe` i `Either`

Haskell proporciona dos tipus principals per gestionar errors de manera funcional:

  • Maybe: Representa un valor opcional que pot ser Just un valor o Nothing.
  • Either: Representa un valor que pot ser d'un de dos tipus, típicament utilitzat per al maneig d'errors on Left representa un error i Right representa un èxit.

Exemple: Usant `Maybe`

safeDivide :: Double -> Double -> Maybe Double
safeDivide _ 0 = Nothing
safeDivide x y = Just (x / y)

main :: IO ()
main = do
    let result = safeDivide 10 2
    case result of
        Nothing -> putStrLn "Divisió per zero!"
        Just value -> putStrLn $ "Resultat: " ++ show value

Exemple: Usant `Either`

safeDivide :: Double -> Double -> Either String Double
safeDivide _ 0 = Left "Divisió per zero!"
safeDivide x y = Right (x / y)

main :: IO ()
main = do
    let result = safeDivide 10 0
    case result of
        Left err -> putStrLn err
        Right value -> putStrLn $ "Resultat: " ++ show value

Maneig d'Excepcions Avançat amb `Control.Exception`

Per a escenaris més complexos, Haskell proporciona el mòdul Control.Exception que permet un control més detallat sobre les excepcions.

Funcions Clau

  • throw: Llença una excepció.
  • catch: Captura una excepció.
  • try: Intenta una acció i retorna una excepció o un resultat.

Exemple: Usant `Control.Exception`

import Control.Exception

main :: IO ()
main = do
    result <- try (evaluate (5 `div` 0)) :: IO (Either SomeException Int)
    case result of
        Left ex -> putStrLn $ "Excepció capturada: " ++ show ex
        Right value -> putStrLn $ "Resultat: " ++ show value

Excepcions Personalitzades

Pots definir les teves pròpies excepcions creant un nou tipus de dades i fent-lo una instància de la classe Exception.

Exemple: Definint i Usant Excepcions Personalitzades

import Control.Exception

data MyException = MyException String deriving (Show)

instance Exception MyException

main :: IO ()
main = do
    result <- try (throwIO (MyException "Alguna cosa ha anat malament")) :: IO (Either MyException ())
    case result of
        Left ex -> putStrLn $ "Excepció personalitzada capturada: " ++ show ex
        Right _ -> putStrLn "Cap excepció"

Resum

El maneig d'excepcions en Haskell es pot abordar utilitzant tipus simples com Maybe i Either per a la gestió bàsica d'errors, o tècniques més avançades utilitzant el mòdul Control.Exception per a escenaris complexos. Entendre aquests mecanismes et permetrà escriure programes en Haskell robustos i tolerants a fallades.
