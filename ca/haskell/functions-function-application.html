Introducció a les Funcions en Haskell

Les funcions són una part fonamental de Haskell, un llenguatge de programació purament funcional. Entendre com definir i aplicar funcions és crucial per escriure codi Haskell efectiu.

Conceptes Clau

  • Funcions Pures: Les funcions en Haskell són pures, és a dir, sempre produeixen la mateixa sortida per la mateixa entrada i no tenen efectes secundaris.
  • Funcions de Primera Classe: Les funcions en Haskell poden ser passades com a arguments a altres funcions, retornades com a valors d'altres funcions i assignades a variables.
  • Aplicació de Funcions: Aplicar una funció als seus arguments és senzill i no requereix parèntesis.

Definició de Funcions

En Haskell, les funcions es defineixen utilitzant el símbol =. El nom de la funció és seguit pels seus paràmetres, i el cos de la funció segueix el =.

Exemple

add :: Int -> Int -> Int
add x y = x + y
  • add és el nom de la funció.
  • x i y són paràmetres.
  • x + y és el cos de la funció.

Explicació

  • La signatura de tipus Int -> Int -> Int indica que add pren dos arguments Int i retorna un Int.
  • La funció add pren dos enters i retorna la seva suma.

Aplicació de Funcions

Aplicar una funció als seus arguments en Haskell és simple i no requereix parèntesis.

Exemple

result = add 3 5
  • add 3 5 aplica la funció add als arguments 3 i 5.
  • result serà 8.

Explicació

  • L'aplicació de funcions es fa escrivint el nom de la funció seguit dels seus arguments separats per espais.
  • No es necessiten parèntesis tret que vulguis canviar l'ordre d'avaluació.

Funcions d'Ordre Superior

Les funcions d'ordre superior són funcions que prenen altres funcions com a arguments o retornen funcions com a resultats.

Exemple

applyTwice :: (a -> a) -> a -> a
applyTwice f x = f (f x)
  • applyTwice és una funció d'ordre superior.
  • Pren una funció f i un valor x, i aplica f a x dues vegades.

Explicació

  • La signatura de tipus (a -> a) -> a -> a indica que applyTwice pren una funció de a a a i un valor de tipus a, i retorna un valor de tipus a.
  • f (f x) aplica la funció f a x dues vegades.

Funcions Anònimes (Lambdas)

Les funcions anònimes, o lambdas, són funcions sense noms. Són útils per a funcions curtes i d'un sol ús.

Exemple

increment = \x -> x + 1
  • \x -> x + 1 és una funció anònima que incrementa el seu argument en 1.

Explicació

  • La barra invertida \ s'utilitza per introduir una lambda.
  • x és el paràmetre, i x + 1 és el cos de la funció.

Currificació

En Haskell, totes les funcions són currificades, és a dir, prenen un argument a la vegada i retornen una nova funció que pren el següent argument.

Exemple

multiply :: Int -> Int -> Int
multiply x y = x * y

double = multiply 2
  • multiply és una funció currificada.
  • double és una nova funció creada aplicant multiply a 2.

Explicació

  • multiply pren un Int i retorna una funció que pren un altre Int i retorna el seu producte.
  • double és una funció que multiplica el seu argument per 2.

Composició de Funcions

La composició de funcions és combinar dues o més funcions per formar una nova funció.

Exemple

compose :: (b -> c) -> (a -> b) -> a -> c
compose f g x = f (g x)

incrementAndDouble = compose (*2) (+1)
  • compose és una funció que pren dues funcions f i g i retorna la seva composició.
  • incrementAndDouble és una nova funció que incrementa el seu argument i després el duplica.

Explicació

  • La signatura de tipus (b -> c) -> (a -> b) -> a -> c indica que compose pren dues funcions i retorna una nova funció.
  • f (g x) aplica g a x i després aplica f al resultat.

Resum

En aquesta secció, hem cobert els conceptes bàsics de les funcions i l'aplicació de funcions en Haskell:

  • Com definir funcions.
  • Com aplicar funcions als arguments.
  • Funcions d'ordre superior i els seus usos.
  • Funcions anònimes (lambdas) per a funcions curtes i d'un sol ús.
  • Currificació i les seves implicacions per a l'aplicació de funcions.
  • Composició de funcions per combinar funcions.

Entendre aquests conceptes és essencial per escriure codi Haskell efectiu i servirà com a base per a temes més avançats en programació funcional.
