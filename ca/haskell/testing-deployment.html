Introducció

En aquesta secció, cobrirem els aspectes essencials de les proves i el desplegament d'aplicacions Haskell. Les proves asseguren que el teu codi es comporti com s'espera, mentre que el desplegament implica fer que la teva aplicació estigui disponible per a l'ús. Començarem amb conceptes bàsics de proves i gradualment passarem a temes més avançats, incloent la integració contínua i les estratègies de desplegament.

Proves en Haskell

Per què les proves són importants

  • Assegura la qualitat del codi: Ajuda a detectar errors i fallades aviat.
  • Facilita la refactorització: Fa que sigui més fàcil canviar el codi amb confiança.
  • Documentació: Les proves poden servir com a documentació addicional de com ha de funcionar el codi.

Tipus de proves

  • Proves unitàries: Prova components o funcions individuals.
  • Proves d'integració: Prova com diferents parts de l'aplicació treballen juntes.
  • Proves basades en propietats: Utilitza propietats per generar casos de prova automàticament.

Configuració d'un entorn de proves

Per començar amb les proves en Haskell, necessitaràs configurar un entorn de proves. Les biblioteques més utilitzades són Hspec per a proves unitàries i QuickCheck per a proves basades en propietats.

Instal·lació de Hspec i QuickCheck

Pots afegir aquestes biblioteques al teu projecte modificant el teu fitxer package.yaml o *.cabal:

dependencies:
  - hspec
  - QuickCheck

Escriure proves unitàries amb Hspec

Hspec és un marc de proves inspirat en RSpec de Ruby. Et permet escriure proves llegibles i mantenibles.

Exemple: Provar una funció simple

Escrivim una prova per a una funció simple que suma dos números:

-- src/Lib.hs
module Lib (add) where

add :: Int -> Int -> Int
add x y = x + y
-- test/Spec.hs
import Test.Hspec
import Lib (add)

main :: IO ()
main = hspec $ do
  describe "add" $ do
    it "adds two numbers" $ do
      add 1 2 `shouldBe` 3

Per executar les proves, utilitza la següent comanda:

stack test

Proves basades en propietats amb QuickCheck

QuickCheck et permet definir propietats que les teves funcions haurien de satisfer i després genera casos de prova aleatoris per comprovar aquestes propietats.

Exemple: Provar la funció add

-- test/Spec.hs
import Test.Hspec
import Test.QuickCheck
import Lib (add)

main :: IO ()
main = hspec $ do
  describe "add" $ do
    it "is commutative" $ property $
      \x y -> add x y == add y x

Desplegament

Preparació per al desplegament

Abans de desplegar la teva aplicació Haskell, assegura't que:

  • Totes les proves passin.
  • L'aplicació es construeixi amb èxit.
  • Les dependències estiguin correctament gestionades.

Construir l'aplicació

Pots construir la teva aplicació Haskell utilitzant Stack o Cabal. Per exemple, utilitzant Stack:

stack build

Estratègies de desplegament

  • Desplegament manual: Copiar manualment els artefactes de construcció al servidor.
  • Desplegament automatitzat: Utilitzar pipelines de CI/CD per automatitzar el procés de desplegament.

Integració contínua i desplegament continu (CI/CD)

CI/CD automatitza el procés de provar i desplegar la teva aplicació. Eines populars de CI/CD inclouen GitHub Actions, Travis CI i CircleCI.

Exemple: Workflow de GitHub Actions

Crea un fitxer .github/workflows/ci.yml:

name: CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Set up Haskell
      uses: actions/setup-haskell@v1
      with:
        ghc-version: '8.10.4'
    - name: Install dependencies
      run: stack setup
    - name: Build
      run: stack build
    - name: Run tests
      run: stack test

Conclusió

Les proves i el desplegament són aspectes crucials del desenvolupament de programari en Haskell. Escriure proves completes utilitzant Hspec i QuickCheck pot assegurar que el teu codi sigui fiable i mantenible. Automatitzar el procés de desplegament amb eines de CI/CD com GitHub Actions pot estalviar temps i reduir errors, fent que el teu flux de treball de desenvolupament sigui més eficient.
