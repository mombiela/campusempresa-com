Introducció a les eines de construcció en Haskell

Haskell, com molts altres llenguatges de programació, té el seu propi conjunt d'eines per gestionar dependències, construir projectes i manejar la distribució de paquets. Les dues eines principals de construcció en l'ecosistema Haskell són Cabal i Stack. Entendre aquestes eines és crucial per a un desenvolupament eficient en Haskell.

Cabal

Cabal és un sistema per construir i empaquetar biblioteques i programes Haskell. És part de la Plataforma Haskell i ha existit durant molt de temps.

Conceptes clau de Cabal

  • Fitxer de descripció del paquet: Cada paquet Haskell té un fitxer .cabal que descriu el paquet, les seves dependències i altres metadades.
  • Hackage: El repositori central de paquets per a biblioteques Haskell.
  • Sandboxing: Una manera d'aïllar les dependències d'un projecte per evitar conflictes.

Comandes bàsiques de Cabal

  • Inicialitzar un nou projecte:

    cabal init

    Aquesta comanda crea un nou fitxer .cabal i configura l'estructura del projecte.

  • Construir un projecte:

    cabal build

    Aquesta comanda compila el projecte i les seves dependències.

  • Executar un projecte:

    cabal run

    Aquesta comanda construeix i executa el projecte.

  • Instal·lar dependències:

    cabal install --only-dependencies

    Aquesta comanda instal·la totes les dependències llistades en el fitxer .cabal.

Exemple: Creació d'un projecte Cabal simple

  1. Inicialitzar el projecte:

    cabal init

    Segueix les indicacions per configurar el projecte.

  2. Editar el fitxer .cabal:

    name:                my-project
version:             0.1.0.0
build-type:          Simple
cabal-version:       >=1.10

library
  exposed-modules:     MyLib
  build-depends:       base >=4.7 && <5
  hs-source-dirs:      src
  default-language:    Haskell2010

  3. Crear el directori de fonts i un mòdul simple:

    mkdir src
echo "module MyLib where\n\nhello :: String\nhello = \"Hello, World!\"" > src/MyLib.hs

  4. Construir i executar el projecte:

    cabal build
cabal repl

Stack

Stack és una eina de construcció més nova que té com a objectiu proporcionar una experiència més amigable per a l'usuari i una millor gestió de dependències.

Conceptes clau de Stack

  • Stackage: Un conjunt de paquets seleccionats que se sap que funcionen bé junts.
  • Snapshots: Conjunts específics de versions de paquets que es proven junts.
  • Configuració del projecte: Stack utilitza un fitxer stack.yaml per configurar el projecte.

Comandes bàsiques de Stack

  • Inicialitzar un nou projecte:

    stack new my-project

    Aquesta comanda crea un nou projecte amb una plantilla per defecte.

  • Construir un projecte:

    stack build

    Aquesta comanda compila el projecte i les seves dependències.

  • Executar un projecte:

    stack run

    Aquesta comanda construeix i executa el projecte.

  • Instal·lar dependències:

    stack setup

    Aquesta comanda configura el compilador i instal·la les dependències.

Exemple: Creació d'un projecte Stack simple

  1. Inicialitzar el projecte:

    stack new my-project
cd my-project

  2. Editar el fitxer stack.yaml:

    resolver: lts-18.18

packages:
- .
extra-deps: []

  3. Editar el fitxer package.yaml:

    name: my-project
version: 0.1.0.0
dependencies:
- base >= 4.7 && < 5

library:
  source-dirs: src
  exposed-modules: MyLib

  4. Crear el directori de fonts i un mòdul simple:

    mkdir src
echo "module MyLib where\n\nhello :: String\nhello = \"Hello, World!\"" > src/MyLib.hs

  5. Construir i executar el projecte:

    stack build
stack ghci

Comparació de Cabal i Stack

| Característica | Cabal | Stack | |------------------------|--------------------------------------------|--------------------------------------------| | Repositori de paquets | Hackage | Stackage | | Gestió de dependències | Manual, amb sandboxing | Automàtica, amb snapshots seleccionats | | Fitxer de configuració | Fitxer .cabal | stack.yaml i package.yaml | | Facilitat d'ús | Requereix més configuració manual | Més amigable per a l'usuari, especialment per a principiants | | Aïllament | Sandboxing (opcional) | Sempre aïllat |

Conclusió

Tant Cabal com Stack són eines poderoses per gestionar projectes Haskell, cadascuna amb els seus punts forts. Cabal ofereix flexibilitat i ha estat un element bàsic en la comunitat Haskell durant anys, mentre que Stack proporciona una experiència més simplificada i amigable per a l'usuari, especialment per a principiants. Entendre ambdues eines et donarà una base sòlida per gestionar projectes Haskell de manera efectiva.
