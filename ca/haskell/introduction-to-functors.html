Què és un Functor?

A Haskell, un Functor és una classe de tipus que representa un tipus que es pot mapar. És un concepte fonamental en la programació funcional i s'utilitza per aplicar una funció a valors embolcallats en un context, com ara una llista, Maybe, o altres estructures de dades.

Conceptes Clau

  • Classe de Tipus: Una classe de tipus defineix un conjunt de funcions que poden ser implementades per diferents tipus.
  • Mapatge: Aplicar una funció a cada element en una estructura de dades.
  • Context: L'estructura que conté els valors, com ara una llista o Maybe.

Classe de Tipus Functor

La classe de tipus Functor es defineix al Prelude de la següent manera:

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • fmap és la funció principal de la classe de tipus Functor.
  • Pren una funció (a -> b) i un functor f a, i retorna un nou functor f b.

Exemple: Instància de Functor per a Llista

El tipus de llista [] és un functor. Així és com funciona:

instance Functor [] where
    fmap = map
  • La funció map aplica una funció donada a cada element d'una llista.

Exemple de Codi

-- Utilitzant fmap amb una llista
fmap (+1) [1, 2, 3]  -- Resultat: [2, 3, 4]

En aquest exemple, fmap aplica la funció (+1) a cada element de la llista [1, 2, 3], resultant en [2, 3, 4].

Lleis dels Functors

Els functors han d'obeir dues lleis per assegurar un comportament consistent:

  1. Llei de la Identitat: fmap id == id
  2. Llei de la Composició: fmap (f . g) == fmap f . fmap g

Llei de la Identitat

Aplicar la funció d'identitat id utilitzant fmap hauria de retornar el functor original.

Exemple de Codi

fmap id [1, 2, 3]  -- Resultat: [1, 2, 3]

Llei de la Composició

Mapar la composició de dues funcions hauria de ser el mateix que mapar una funció i després l'altra.

Exemple de Codi

fmap ((+1) . (*2)) [1, 2, 3]  -- Resultat: [3, 5, 7]
-- Equivalent a
fmap (+1) (fmap (*2) [1, 2, 3])  -- Resultat: [3, 5, 7]

Instàncies de Functor per a Altres Tipus

Functor Maybe

El tipus Maybe també és un functor. Representa càlculs que poden fallar.

instance Functor Maybe where
    fmap _ Nothing  = Nothing
    fmap f (Just x) = Just (f x)

Exemple de Codi

fmap (+1) (Just 5)  -- Resultat: Just 6
fmap (+1) Nothing   -- Resultat: Nothing

Functor Either

El tipus Either representa càlculs que poden resultar en un error.

instance Functor (Either a) where
    fmap _ (Left x)  = Left x
    fmap f (Right y) = Right (f y)

Exemple de Codi

fmap (+1) (Right 5)  -- Resultat: Right 6
fmap (+1) (Left "Error")  -- Resultat: Left "Error"

Usos Pràctics

Transformant Dades en una Llista

-- Duplicant cada element en una llista
fmap (*2) [1, 2, 3, 4]  -- Resultat: [2, 4, 6, 8]

Gestionant Valors Opcional

-- Incrementant un valor dins d'un Maybe
fmap (+1) (Just 10)  -- Resultat: Just 11
fmap (+1) Nothing    -- Resultat: Nothing

Conclusió

Els functors són una abstracció poderosa a Haskell que et permeten aplicar funcions a valors embolcallats en un context. Entenent i utilitzant la classe de tipus Functor, pots escriure codi més genèric i reutilitzable. Recorda les lleis dels Functors per assegurar que les teves instàncies es comporten de manera consistent. Els functors són només el començament; obren el camí a conceptes més avançats com els Applicatives i els Monads.
