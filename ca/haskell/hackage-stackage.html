Introducció

Hackage i Stackage són dos components essencials de l'ecosistema Haskell que ajuden a gestionar i distribuir biblioteques i paquets Haskell. Entendre aquestes eines és crucial per als desenvolupadors de Haskell, des de principiants fins a usuaris avançats.

Hackage

Hackage és l'arxiu central de paquets per a biblioteques Haskell. Allotja una vasta col·lecció de paquets Haskell contribuïts per la comunitat.

Conceptes Clau

  • Repositori de Paquets: Hackage serveix com a repositori on els desenvolupadors poden pujar i compartir els seus paquets Haskell.
  • Cabal: El sistema de construcció i gestor de paquets per a Haskell, que interactua amb Hackage per descarregar i gestionar dependències.
  • Metadades: Cada paquet a Hackage inclou metadades com ara versió, dependències i documentació.

Utilitzant Hackage

Per utilitzar Hackage, normalment interactues amb ell a través de l'eina de línia de comandes cabal.

Exemple: Instal·lant un Paquet des de Hackage

cabal update
cabal install <package-name>

Explicació:

  1. cabal update actualitza la llista local de paquets disponibles des de Hackage.
  2. cabal install <package-name> instal·la el paquet especificat.

Publicant un Paquet a Hackage

Per publicar el teu propi paquet a Hackage, has de seguir aquests passos:

  1. Prepara el teu paquet: Assegura't que el teu paquet té un fitxer .cabal amb totes les metadades necessàries.
  2. Construeix i prova: Utilitza cabal build i cabal test per assegurar-te que el teu paquet es construeix i funciona correctament.
  3. Puja: Utilitza cabal upload per pujar el teu paquet a Hackage.

Exemple: Pujant un Paquet

cabal sdist
cabal upload dist/<package-name-version>.tar.gz

Explicació:

  1. cabal sdist crea una distribució de codi font del teu paquet.
  2. cabal upload puja el tarball generat a Hackage.

Stackage

Stackage és una font estable de paquets Haskell que proporciona conjunts curats de paquets coneguts per funcionar bé junts.

Conceptes Clau

  • LTS (Suport a Llarg Termini): Stackage proporciona versions LTS, que són instantànies estables de paquets Haskell.
  • Estabilitat: Stackage assegura que els paquets en una instantània són compatibles entre ells, reduint problemes de dependències.
  • Stack: Una eina de construcció que treballa amb Stackage per gestionar les dependències del projecte.

Utilitzant Stackage

Per utilitzar Stackage, normalment interactues amb ell a través de l'eina de línia de comandes stack.

Exemple: Creant un Nou Projecte amb Stack

stack new my-project
cd my-project
stack setup
stack build

Explicació:

  1. stack new my-project crea un nou projecte Haskell utilitzant una plantilla.
  2. cd my-project canvia el directori al nou projecte.
  3. stack setup instal·la la versió adequada de GHC (Glasgow Haskell Compiler).
  4. stack build construeix el projecte utilitzant la instantània de Stackage.

Configurant Instantànies de Stackage

Pots especificar quina instantània de Stackage utilitzar al teu fitxer stack.yaml.

Exemple: Especificant una Instantània

resolver: lts-18.18

Explicació:

  • El camp resolver especifica la versió de la instantània de Stackage a utilitzar per al teu projecte.

Conclusió

Hackage i Stackage són eines fonamentals en l'ecosistema Haskell que faciliten la gestió de paquets i asseguren l'estabilitat en els projectes Haskell. Hackage serveix com a repositori de paquets complet, mentre que Stackage proporciona conjunts curats i estables de paquets. En dominar aquestes eines, els desenvolupadors de Haskell poden gestionar eficientment les dependències i mantenir projectes robustos.
