Polimorfisme en Haskell
Estructures de Dades Recursives ►

El polimorfisme és un concepte central en Haskell que permet que les funcions operin sobre diferents tipus. Millora la reutilització i la flexibilitat del codi. Aquest tema et guiarà per entendre el polimorfisme en Haskell, des de nivells bàsics fins a avançats.

Introducció al Polimorfisme

El polimorfisme en Haskell es pot categoritzar àmpliament en dos tipus:

  • Polimorfisme Paramètric: Funcions o tipus de dades que poden operar sobre qualsevol tipus.
  • Polimorfisme Ad-hoc: Funcions que poden operar sobre diferents tipus però es comporten de manera diferent segons el tipus.

Polimorfisme Paramètric

El polimorfisme paramètric et permet escriure funcions genèriques que poden manejar valors de manera uniforme sense dependre del seu tipus.

Exemple: Una Funció d'Identitat Genèrica

identity :: a -> a
identity x = x
  • identity és una funció que pren un valor de qualsevol tipus a i retorna un valor del mateix tipus.
  • La variable de tipus a pot representar qualsevol tipus, fent que identity sigui una funció polimòrfica.

Exemple: Una Funció de Parell Genèrica

makePair :: a -> b -> (a, b)
makePair x y = (x, y)
  • makePair pren dos valors de tipus potencialment diferents a i b i retorna una tupla que conté ambdós valors.
  • Aquesta funció és polimòrfica en ambdós arguments.

Polimorfisme Ad-hoc

El polimorfisme ad-hoc s'aconsegueix a través de classes de tipus, permetent que les funcions operin de manera diferent segons el tipus dels seus arguments.

Exemple: La Classe de Tipus `Eq`

La classe de tipus Eq s'utilitza per a tipus que suporten la prova d'igualtat.

class Eq a where
    (==) :: a -> a -> Bool
    (/=) :: a -> a -> Bool
  • Els tipus que són instàncies de Eq han d'implementar les funcions (==) i (/=).

Exemple: Fer un Tipus una Instància de `Eq`

data Color = Red | Green | Blue

instance Eq Color where
    Red == Red = True
    Green == Green = True
    Blue == Blue = True
    _ == _ = False
  • Aquí, Color es fa una instància de Eq, definint com es determina la igualtat per als valors de Color.

Exemple: La Classe de Tipus `Show`

La classe de tipus Show s'utilitza per a tipus els valors dels quals es poden convertir en cadenes de text.

class Show a where
    show :: a -> String
  • Els tipus que són instàncies de Show han d'implementar la funció show.

Exemple: Fer un Tipus una Instància de `Show`

data Color = Red | Green | Blue

instance Show Color where
    show Red = "Red"
    show Green = "Green"
    show Blue = "Blue"
  • Aquí, Color es fa una instància de Show, definint com es converteixen els valors de Color en cadenes de text.

Polimorfisme Avançat

Tipus de Kinds Superiors

Els tipus de kinds superiors et permeten abstraure sobre constructors de tipus. Aquest és una forma més avançada de polimorfisme.

Exemple: Classe de Tipus Functor

La classe de tipus Functor és per a tipus que es poden mapar.

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • f és un constructor de tipus que pren un argument de tipus.
  • fmap aplica una funció a cada element dins d'un functor.

Exemple: Fer un Tipus una Instància de `Functor`

instance Functor Maybe where
    fmap _ Nothing = Nothing
    fmap f (Just x) = Just (f x)
  • Aquí, Maybe es fa una instància de Functor, definint com funciona fmap per a Maybe.

Conclusió

El polimorfisme en Haskell et permet escriure codi flexible i reutilitzable. Entenent el polimorfisme paramètric i el polimorfisme ad-hoc, pots crear funcions i tipus que funcionen en una àmplia gamma d'escenaris. A mesura que avances, els tipus de kinds superiors i les classes de tipus com Functor et permetran abstraure patrons encara més complexos, fent que el teu codi Haskell sigui tant poderós com elegant.
