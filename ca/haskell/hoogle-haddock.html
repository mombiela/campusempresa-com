Introducció

En aquesta secció, explorarem dues eines essencials per als desenvolupadors de Haskell: Hoogle i Haddock. Aquestes eines són invaluables per cercar i documentar codi Haskell, respectivament. Entendre com utilitzar-les de manera efectiva pot millorar significativament la teva productivitat i la qualitat del teu codi.

Hoogle

Hoogle és un motor de cerca d'API de Haskell que et permet cercar biblioteques, funcions i tipus de Haskell. És particularment útil quan saps quin tipus de funció necessites però no en coneixes el nom o quina biblioteca la proporciona.

Característiques Clau de Hoogle

  • Cerca Basada en Tipus: Pots cercar per signatures de tipus de funcions.
  • Cerca de Biblioteques: Troba funcions i tipus dins de biblioteques específiques.
  • Enllaços de Documentació: Enllaços directes a la documentació dels resultats de la cerca.

Utilitzant Hoogle

Per utilitzar Hoogle, pots visitar el lloc web de Hoogle o utilitzar-lo des de la línia de comandes.

Exemple: Cercant per Signatura de Tipus

Suposem que necessites una funció que prengui una llista d'enters i retorni un enter. Pots cercar-ho utilitzant la signatura de tipus [Int] -> Int.

-- Consulta de cerca de Hoogle
[Int] -> Int

Aquesta cerca podria retornar funcions com sum, length o product.

Exemple: Cercant per Nom de Funció

Si coneixes el nom de la funció, pots cercar directament:

-- Consulta de cerca de Hoogle
map

Això retornarà la funció map juntament amb la seva documentació i exemples d'ús.

Instal·lant Hoogle Localment

Per instal·lar Hoogle localment, pots utilitzar les següents comandes:

cabal update
cabal install hoogle
hoogle generate

Després de la instal·lació, pots utilitzar Hoogle des de la línia de comandes:

hoogle "[Int] -> Int"

Haddock

Haddock és l'eina estàndard per generar documentació a partir de codi font de Haskell anotat. Genera documentació HTML que és fàcil de navegar i entendre.

Característiques Clau de Haddock

  • Generació Automàtica de Documentació: Genera documentació a partir de comentaris en el codi font.
  • Codi Font amb Hiperenllaços: Enllaços entre diferents parts del codi i la documentació.
  • Sortida Personalitzable: Permet la personalització de la documentació generada.

Escriure Comentaris Haddock

Els comentaris Haddock s'escriuen utilitzant una sintaxi especial dins dels teus fitxers de codi Haskell.

Exemple: Comentari Haddock Bàsic

Aquí tens un exemple d'un comentari Haddock senzill:

-- | Aquesta funció suma dos enters.
-- Pren dos arguments i retorna la seva suma.
add :: Int -> Int -> Int
add x y = x + y

Exemple: Documentant un Mòdul

També pots documentar mòduls sencers:

{-|
Mòdul      : MyModule
Descripció : Aquest mòdul proporciona funcions aritmètiques bàsiques.
-}
module MyModule (add, subtract) where

-- | Suma dos enters.
add :: Int -> Int -> Int
add x y = x + y

-- | Resta el segon enter del primer.
subtract :: Int -> Int -> Int
subtract x y = x - y

Generant Documentació amb Haddock

Per generar documentació, utilitza la següent comanda:

cabal haddock

Això crearà documentació HTML al directori dist/doc/html del teu projecte.

Conclusió

Hoogle i Haddock són eines poderoses que poden millorar enormement la teva experiència de desenvolupament en Haskell. Hoogle t'ajuda a trobar les funcions i tipus que necessites, mentre que Haddock et permet crear documentació completa i navegable per al teu codi. Dominant aquestes eines, pots millorar tant la teva productivitat com la qualitat dels teus projectes en Haskell.
