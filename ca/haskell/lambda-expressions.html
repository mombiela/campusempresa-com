Expressions Lambda en Haskell
Currificació i Aplicació Parcial ►

Les expressions lambda, també conegudes com a funcions anònimes, són una característica poderosa en Haskell que et permeten crear funcions sense donar-los un nom. Són particularment útils per a funcions curtes i d'un sol ús que s'utilitzen només una vegada o unes poques vegades.

Introducció a les Expressions Lambda

  • Definició: Una expressió lambda és una manera de definir una funció anònima utilitzant el símbol \ (barra invertida).
  • Sintaxi: La sintaxi general per a una expressió lambda és \arguments -> expressió.

Sintaxi Bàsica i Ús

Les expressions lambda es defineixen utilitzant la barra invertida (\) seguida dels arguments, una fletxa (->), i el cos de la funció.

-- Una simple expressió lambda que suma dos números
add = \x y -> x + y

-- Utilitzant l'expressió lambda
main = print (add 3 5)  -- Sortida: 8

Exemples Pràctics

Les expressions lambda sovint s'utilitzen en funcions d'ordre superior com map, filter, i foldr.

-- Utilitzant una expressió lambda amb map per incrementar cada element d'una llista
incremented = map (\x -> x + 1) [1, 2, 3, 4]
-- Sortida: [2, 3, 4, 5]

-- Utilitzant una expressió lambda amb filter per seleccionar números parells
evens = filter (\x -> x `mod` 2 == 0) [1, 2, 3, 4, 5, 6]
-- Sortida: [2, 4, 6]

-- Utilitzant una expressió lambda amb foldr per sumar una llista
sumList = foldr (\x acc -> x + acc) 0 [1, 2, 3, 4]
-- Sortida: 10

Ús Avançat de les Expressions Lambda

Expressions Lambda Imbricades

Les expressions lambda poden ser imbricades per crear funcions més complexes.

-- Una expressió lambda imbricada per sumar tres números
addThree = \x -> \y -> \z -> x + y + z

-- Utilitzant l'expressió lambda imbricada
main = print (addThree 1 2 3)  -- Sortida: 6

Currificació amb Expressions Lambda

Les funcions en Haskell són currificades per defecte, el que significa que poden ser aplicades parcialment. Les expressions lambda també poden ser currificades.

-- Una expressió lambda currificada
curriedAdd = \x -> (\y -> x + y)

-- Aplicant parcialment l'expressió lambda currificada
addFive = curriedAdd 5

-- Utilitzant la funció parcialment aplicada
main = print (addFive 10)  -- Sortida: 15

Expressions Lambda en Comprensions de Llistes

Les expressions lambda poden ser utilitzades dins de comprensions de llistes per a filtratges i mapatges més complexos.

-- Utilitzant una expressió lambda en una comprensió de llista
squares = [ (\x -> x * x) x | x <- [1..5] ]
-- Sortida: [1, 4, 9, 16, 25]

Errors Comuns i Millors Pràctiques

  • Llegibilitat: Tot i que les expressions lambda són concises, un ús excessiu pot fer que el codi sigui més difícil de llegir. Utilitza-les amb prudència.
  • Rendiment: Les expressions lambda poden de vegades causar problemes de rendiment si no s'utilitzen amb cura, especialment en bucles ajustats o funcions recursives.
  • Depuració: Com que les expressions lambda són anònimes, la depuració pot ser més difícil. Donar noms a les funcions pot de vegades facilitar la depuració.

Conclusió

Les expressions lambda són una característica versàtil i poderosa en Haskell que et permeten crear funcions anònimes sobre la marxa. Són particularment útils en funcions d'ordre superior i comprensions de llistes. No obstant això, és important utilitzar-les amb prudència per mantenir la llegibilitat i el rendiment del codi. Entenent i practicant amb expressions lambda, pots escriure codi Haskell més concís i expressiu.
