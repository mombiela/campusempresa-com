Introducció a les Monads

Les monads són una manera potent i flexible de gestionar càlculs en Haskell. Proporcionen una manera d'estructurar programes de manera genèrica. Aquesta secció cobrirà tres monads comuns: Maybe, Either i List.

Maybe Monad

La monad Maybe s'utilitza per representar càlculs que podrien fallar. Encapsula un valor opcional.

Definició

data Maybe a = Nothing | Just a

Conceptes Clau

  • Nothing: Representa un fracàs o absència de valor.
  • Just a: Representa un càlcul reeixit amb un valor a.

Exemple d'Ús

safeDivide :: Double -> Double -> Maybe Double
safeDivide _ 0 = Nothing
safeDivide x y = Just (x / y)

Instància de Monad

instance Monad Maybe where
    return = Just
    Nothing >>= _ = Nothing
    Just x >>= f = f x

Exemple amb Notació `do`

maybeExample :: Maybe Double
maybeExample = do
    x <- Just 10
    y <- Just 2
    safeDivide x y

Either Monad

La monad Either s'utilitza per a càlculs que poden fallar amb un missatge d'error. Encapsula un valor que pot ser d'un dels dos tipus.

Definició

data Either a b = Left a | Right b

Conceptes Clau

  • Left a: Representa un error amb un valor a.
  • Right b: Representa un càlcul reeixit amb un valor b.

Exemple d'Ús

safeDivide :: Double -> Double -> Either String Double
safeDivide _ 0 = Left "Division by zero"
safeDivide x y = Right (x / y)

Instància de Monad

instance Monad (Either e) where
    return = Right
    Left l >>= _ = Left l
    Right r >>= f = f r

Exemple amb Notació `do`

eitherExample :: Either String Double
eitherExample = do
    x <- Right 10
    y <- Right 2
    safeDivide x y

List Monad

La monad List s'utilitza per a càlculs que poden retornar múltiples resultats. Encapsula una llista de valors.

Definició

data [] a = [] | a : [a]

Conceptes Clau

  • []: Representa una llista buida.
  • a : [a]: Representa una llista amb cap a i cua [a].

Exemple d'Ús

pairs :: [a] -> [b] -> [(a, b)]
pairs xs ys = [(x, y) | x <- xs, y <- ys]

Instància de Monad

instance Monad [] where
    return x = [x]
    xs >>= f = concat (map f xs)

Exemple amb Notació `do`

listExample :: [(Int, Char)]
listExample = do
    x <- [1, 2, 3]
    y <- ['a', 'b']
    return (x, y)

Comparació de Monads

| Monad | Propòsit | Tipus d'Èxit | Tipus de Fracàs | |--------|---------------------------------|--------------|-----------------------| | Maybe | Valor opcional | Just a | Nothing | | Either | Càlcul amb missatge d'error | Right b | Left a | | List | Múltiples resultats | [a] | [] (llista buida) |

Conclusió

Les monads com Maybe, Either i List proporcionen abstraccions potents per gestionar diferents tipus de càlculs en Haskell. Entendre aquestes monads i el seu ús pot millorar enormement la teva capacitat per escriure programes robustos i flexibles en Haskell. Practica utilitzant aquestes monads en diversos escenaris per familiaritzar-te amb els seus patrons i idiomes.
