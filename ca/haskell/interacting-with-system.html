Interactuar amb el Sistema
Gestió d'Excepcions ►

En aquesta secció, explorarem com els programes Haskell poden interactuar amb el sistema. Això inclou llegir i escriure fitxers, executar comandes del sistema i gestionar arguments de la línia de comandes. Aquestes capacitats són essencials per crear aplicacions del món real que necessiten interactuar amb el seu entorn.

Entrada/Sortida de Fitxers

L'entrada/sortida de fitxers (Input/Output) és un aspecte fonamental de la interacció amb el sistema. Haskell proporciona diverses funcions per llegir i escriure fitxers.

Llegir d'un Fitxer

Per llegir d'un fitxer, pots utilitzar la funció readFile. Aquesta funció pren una ruta de fitxer com a argument i retorna el contingut del fitxer com una cadena.

import System.IO

main :: IO ()
main = do
    contents <- readFile "example.txt"
    putStrLn contents

Explicació:

  • readFile "example.txt" llegeix el contingut de example.txt.
  • contents <- readFile "example.txt" assigna el contingut del fitxer a la variable contents.
  • putStrLn contents imprimeix el contingut a la consola.

Escriure a un Fitxer

Per escriure a un fitxer, pots utilitzar la funció writeFile. Aquesta funció pren una ruta de fitxer i una cadena per escriure al fitxer.

import System.IO

main :: IO ()
main = do
    let content = "Hello, Haskell!"
    writeFile "output.txt" content

Explicació:

  • let content = "Hello, Haskell!" defineix la cadena que s'ha d'escriure.
  • writeFile "output.txt" content escriu la cadena a output.txt.

Afegir a un Fitxer

Per afegir a un fitxer, pots utilitzar la funció appendFile. Aquesta funció pren una ruta de fitxer i una cadena per afegir al fitxer.

import System.IO

main :: IO ()
main = do
    let additionalContent = "\nAppended text."
    appendFile "output.txt" additionalContent

Explicació:

  • let additionalContent = "\nAppended text." defineix la cadena que s'ha d'afegir.
  • appendFile "output.txt" additionalContent afegeix la cadena a output.txt.

Executar Comandes del Sistema

Haskell pot executar comandes del sistema utilitzant el mòdul System.Process. Aquest mòdul proporciona funcions per executar comandes externes i capturar-ne la sortida.

Executar una Comanda Simple

Per executar una comanda simple, pots utilitzar la funció system.

import System.Process

main :: IO ()
main = do
    exitCode <- system "ls"
    print exitCode

Explicació:

  • system "ls" executa la comanda ls (llistar el contingut del directori).
  • exitCode <- system "ls" captura el codi de sortida de la comanda.
  • print exitCode imprimeix el codi de sortida a la consola.

Capturar la Sortida d'una Comanda

Per capturar la sortida d'una comanda, pots utilitzar la funció readProcess.

import System.Process

main :: IO ()
main = do
    output <- readProcess "echo" ["Hello, Haskell!"] ""
    putStrLn output

Explicació:

  • readProcess "echo" ["Hello, Haskell!"] "" executa la comanda echo amb l'argument "Hello, Haskell!".
  • output <- readProcess "echo" ["Hello, Haskell!"] "" captura la sortida de la comanda.
  • putStrLn output imprimeix la sortida a la consola.

Gestionar Arguments de la Línia de Comandes

Haskell proporciona el mòdul System.Environment per gestionar arguments de la línia de comandes.

Accedir als Arguments de la Línia de Comandes

Per accedir als arguments de la línia de comandes, pots utilitzar la funció getArgs.

import System.Environment

main :: IO ()
main = do
    args <- getArgs
    print args

Explicació:

  • getArgs recupera la llista d'arguments de la línia de comandes.
  • args <- getArgs assigna la llista d'arguments a la variable args.
  • print args imprimeix la llista d'arguments a la consola.

Utilitzar Arguments de la Línia de Comandes

Pots utilitzar arguments de la línia de comandes per personalitzar el comportament del teu programa.

import System.Environment

main :: IO ()
main = do
    args <- getArgs
    case args of
        [fileName] -> do
            contents <- readFile fileName
            putStrLn contents
        _ -> putStrLn "Usage: runhaskell program.hs <filename>"

Explicació:

  • case args of fa una coincidència de patrons amb la llista d'arguments.
  • [fileName] -> do ... gestiona el cas en què es proporciona exactament un argument (el nom del fitxer).
  • _ -> putStrLn "Usage: runhaskell program.hs <filename>" gestiona qualsevol altre cas, imprimint un missatge d'ús.

Resum

En aquesta secció, hem cobert com interactuar amb el sistema en Haskell. Hem après com:

  • Llegir i escriure fitxers utilitzant readFile, writeFile i appendFile.
  • Executar comandes del sistema utilitzant system i readProcess.
  • Gestionar arguments de la línia de comandes utilitzant getArgs.

Aquestes capacitats permeten que els programes Haskell interactuïn amb el seu entorn, fent-los més versàtils i útils en aplicacions del món real.
