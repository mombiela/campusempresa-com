Introducció a les Classes de Tipus

Les classes de tipus en Haskell són una característica poderosa que et permet definir funcions que poden operar sobre una varietat de tipus diferents. Són similars a les interfícies en altres llenguatges però més flexibles i poderoses.

  • Classe de Tipus: Una col·lecció de tipus que suporten certes operacions.
  • Instància: Una implementació específica d'una classe de tipus per a un tipus particular.

Exemple Bàsic: La Classe de Tipus `Eq`

La classe de tipus Eq s'utilitza per a tipus que suporten la prova d'igualtat.

class Eq a where
    (==) :: a -> a -> Bool
    (/=) :: a -> a -> Bool
  • class Eq a where: Defineix una classe de tipus Eq amb una variable de tipus a.
  • (==) i (/=): Funcions que han de ser implementades per qualsevol tipus que sigui una instància de Eq.

Creació d'Instàncies

Per fer que un tipus sigui una instància d'una classe de tipus, has de proporcionar implementacions per a les funcions requerides.

data Color = Red | Green | Blue

instance Eq Color where
    Red == Red = True
    Green == Green = True
    Blue == Blue = True
    _ == _ = False
  • data Color = Red | Green | Blue: Defineix un nou tipus de dades Color.
  • instance Eq Color where: Fa que Color sigui una instància de Eq definint com funciona == per a Color.

Classes de Tipus Comunes

Aquí tens algunes de les classes de tipus més utilitzades en Haskell:

| Classe de Tipus | Descripció | |-----------------|-------------| | Eq | Prova d'igualtat | | Ord | Operacions de comparació | | Show | Conversió a una cadena | | Read | Anàlisi des d'una cadena | | Num | Operacions numèriques |

La Classe de Tipus `Ord`

La classe de tipus Ord és per a tipus que tenen un ordre.

class Eq a => Ord a where
    compare :: a -> a -> Ordering
    (<) :: a -> a -> Bool
    (<=) :: a -> a -> Bool
    (>) :: a -> a -> Bool
    (>=) :: a -> a -> Bool
  • class Eq a => Ord a where: Ord és una subclasse de Eq, el que significa que qualsevol tipus que sigui una instància de Ord també ha de ser una instància de Eq.

Exemple: Instància `Ord` per a `Color`

instance Ord Color where
    compare Red Red = EQ
    compare Red _ = LT
    compare _ Red = GT
    compare Green Green = EQ
    compare Green _ = LT
    compare _ Green = GT
    compare Blue Blue = EQ
  • compare: Defineix com comparar dos valors de Color.

Classes de Tipus Avançades

Functor

La classe de tipus Functor s'utilitza per a tipus que es poden mapar.

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • class Functor f where: Defineix una classe de tipus Functor amb una variable de tipus f.
  • fmap: Una funció que aplica una funció a cada element d'una estructura.

Exemple: Instància `Functor` per a `Maybe`

instance Functor Maybe where
    fmap _ Nothing = Nothing
    fmap f (Just x) = Just (f x)
  • instance Functor Maybe where: Fa que Maybe sigui una instància de Functor.
  • fmap _ Nothing = Nothing: Si el valor és Nothing, fmap retorna Nothing.
  • fmap f (Just x) = Just (f x): Si el valor és Just x, fmap aplica f a x.

Exercicis

  1. Crea una instància de Eq per a un tipus de dades personalitzat Shape amb constructors Circle, Square i Triangle.
  2. Implementa la classe de tipus Ord per al tipus Shape.
  3. Defineix una instància de Functor per a un tipus de dades personalitzat Box que embolcalla un sol valor.

Conclusió

Les classes de tipus en Haskell proporcionen una manera de definir interfícies genèriques que poden ser implementades per diferents tipus. Són una pedra angular del sistema de tipus de Haskell, permetent el polimorfisme i la reutilització de codi. Entenent i utilitzant les classes de tipus, pots escriure codi més flexible i reutilitzable.
