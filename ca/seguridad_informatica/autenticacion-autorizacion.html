Introducció

En l'àmbit de la seguretat informàtica, l'autenticació i l'autorització són dos conceptes fonamentals que garanteixen que els usuaris i sistemes accedeixin als recursos de manera segura i controlada. Encara que sovint s'utilitzen de manera intercanviable, aquests termes tenen significats i funcions diferents.

Autenticació

L'autenticació és el procés de verificar la identitat d'un usuari o sistema. Aquest procés assegura que qui intenta accedir a un recurs és qui diu ser.

Mètodes d'Autenticació

  • Contrasenyes: El mètode més comú, on l'usuari proporciona una combinació de nom d'usuari i contrasenya.
  • Autenticació de Dos Factors (2FA): Combina alguna cosa que l'usuari sap (contrasenya) amb alguna cosa que l'usuari té (un dispositiu mòbil).
  • Biometria: Utilitza característiques físiques de l'usuari, com empremtes dactilars o reconeixement facial.
  • Certificats Digitals: Utilitzen criptografia per verificar la identitat de l'usuari o dispositiu.

Exemple de Codi: Autenticació Bàsica amb Contrasenya

# Exemple en Python per a autenticació bàsica
usuaris = {
    "usuari1": "contrasenya123",
    "usuari2": "contrasenya456"
}

def autenticar(usuari, contrasenya):
    if usuari in usuaris and usuaris[usuari] == contrasenya:
        return "Autenticació exitosa"
    else:
        return "Autenticació fallida"

# Prova d'autenticació
print(autenticar("usuari1", "contrasenya123"))  # Sortida: Autenticació exitosa
print(autenticar("usuari1", "contrasenyaIncorrecta"))  # Sortida: Autenticació fallida

Autorització

L'autorització és el procés de determinar si un usuari autenticat té permís per accedir a un recurs específic. Aquest procés assegura que els usuaris només puguin realitzar accions per a les quals tenen permisos explícits.

Mètodes d'Autorització

  • Llistes de Control d'Accés (ACL): Especifiquen quins usuaris o sistemes poden accedir a certs recursos i quines operacions poden realitzar.
  • Rols i Permisos: Els usuaris es agrupen en rols, i els permisos s'assignen a aquests rols en lloc d'usuaris individuals.
  • Polítiques Basades en Atributs (ABAC): Utilitzen atributs de l'usuari, del recurs i de l'entorn per prendre decisions d'autorització.

Exemple de Codi: Autorització amb Rols

# Exemple en Python per a autorització basada en rols
rols_permisos = {
    "admin": ["llegir", "escriure", "esborrar"],
    "usuari": ["llegir"]
}

usuaris_rols = {
    "usuari1": "admin",
    "usuari2": "usuari"
}

def autoritzar(usuari, accio):
    rol = usuaris_rols.get(usuari)
    if rol and accio in rols_permisos.get(rol, []):
        return "Autorització concedida"
    else:
        return "Autorització denegada"

# Prova d'autorització
print(autoritzar("usuari1", "esborrar"))  # Sortida: Autorització concedida
print(autoritzar("usuari2", "esborrar"))  # Sortida: Autorització denegada

Comparació entre Autenticació i Autorització

| Característica | Autenticació | Autorització | |--------------------|---------------------------------------|---------------------------------------| | Propòsit | Verificar la identitat de l'usuari | Determinar els permisos de l'usuari | | Quan ocorre | Abans de l'autorització | Després de l'autenticació | | Exemples comuns | Contrasenyes, 2FA, biometria | ACL, rols i permisos, ABAC |

Conclusió

L'autenticació i l'autorització són components essencials de qualsevol sistema de seguretat informàtica. Mentre que l'autenticació se centra en verificar la identitat de l'usuari, l'autorització s'encarrega de gestionar els permisos i accessos als recursos. Comprendre i aplicar correctament aquests conceptes és crucial per protegir la informació i els sistemes d'accessos no autoritzats.
