L'autenticació i l'autorització són components crítics de la seguretat de la base de dades. Aquesta secció et guiarà a través dels conceptes i la implementació de l'autenticació i l'autorització a MongoDB, des de nivells bàsics fins a avançats.

Introducció a l'Autenticació i l'Autorització

  • Autenticació: El procés de verificar la identitat d'un usuari o procés.
  • Autorització: El procés de determinar què pot fer un usuari o procés autenticat.

Autenticació a MongoDB

Conceptes Bàsics

  • Usuari: Una entitat que pot ser autenticada.
  • Rol: Un conjunt de privilegis que es poden assignar a un usuari.
  • Privilegi: Una acció específica que un usuari pot realitzar sobre un recurs.

Activació de l'Autenticació

Per defecte, MongoDB no requereix autenticació. Per activar l'autenticació, cal iniciar el servidor MongoDB amb l'opció --auth.

mongod --auth --dbpath /data/db

Creació d'Usuaris

Per crear un usuari, cal connectar-se a la instància de MongoDB sense autenticació i utilitzar el mètode db.createUser.

use admin
db.createUser({
  user: "admin",
  pwd: "password123",
  roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
})

Autenticació d'Usuaris

Un cop creats els usuaris, poden autenticar-se utilitzant el mètode db.auth.

use admin
db.auth("admin", "password123")

Autorització a MongoDB

Control d'Accés Basat en Rols (RBAC)

MongoDB utilitza el Control d'Accés Basat en Rols (RBAC) per gestionar l'autorització. Els rols atorguen als usuaris privilegis específics sobre recursos.

Rols Integrats

MongoDB proporciona diversos rols integrats, com ara:

  • read: Atorga accés de només lectura a una base de dades.
  • readWrite: Atorga accés de lectura i escriptura a una base de dades.
  • dbAdmin: Atorga drets administratius a una base de dades.
  • userAdmin: Atorga la capacitat de crear i modificar rols i usuaris en una base de dades.

Rols Personalitzats

També es poden crear rols personalitzats per satisfer necessitats específiques.

db.createRole({
  role: "customRole",
  privileges: [
    { resource: { db: "test", collection: "" }, actions: [ "find", "insert" ] }
  ],
  roles: []
})

Assignació de Rols a Usuaris

Els rols es poden assignar als usuaris quan es creen o es modifiquen.

db.createUser({
  user: "appUser",
  pwd: "appPassword",
  roles: [ { role: "readWrite", db: "test" } ]
})

Exemples Pràctics

Exemple 1: Creació d'un Usuari Administrador

  1. Inicia MongoDB sense autenticació.
  2. Crea un usuari administrador.
use admin
db.createUser({
  user: "admin",
  pwd: "adminPass",
  roles: [ { role: "root", db: "admin" } ]
})
  1. Reinicia MongoDB amb l'autenticació activada.
mongod --auth --dbpath /data/db
  1. Autentica't com a usuari administrador.
use admin
db.auth("admin", "adminPass")

Exemple 2: Creació i Autenticació d'un Usuari Regular

  1. Autentica't com a usuari administrador.
use admin
db.auth("admin", "adminPass")
  1. Crea un usuari regular amb accés de lectura i escriptura a la base de dades test.
use test
db.createUser({
  user: "testUser",
  pwd: "testPass",
  roles: [ { role: "readWrite", db: "test" } ]
})
  1. Autentica't com a usuari regular.
use test
db.auth("testUser", "testPass")

Resum

En aquesta secció, hem cobert els conceptes bàsics de l'autenticació i l'autorització a MongoDB. Hem après a:

  • Activar l'autenticació.
  • Crear i autenticar usuaris.
  • Utilitzar rols integrats i crear rols personalitzats.
  • Assignar rols als usuaris.

Entendre aquests conceptes és crucial per assegurar les teves instàncies de MongoDB i garantir que només els usuaris autoritzats puguin accedir i manipular les teves dades.
