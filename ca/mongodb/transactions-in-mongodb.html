Introducció a les Transaccions

Les transaccions a MongoDB permeten executar múltiples operacions en aïllament i assegurar que totes tinguin èxit o totes fallin. Això és crucial per mantenir la integritat de les dades, especialment en aplicacions que requereixen propietats d'atomicitat, consistència, aïllament i durabilitat (ACID).

Conceptes Clau

  • Atomicitat: Assegura que una sèrie d'operacions es completin totalment o no es completin en absolut.
  • Consistència: Garanteix que la base de dades es mantingui en un estat vàlid abans i després de la transacció.
  • Aïllament: Assegura que les transaccions es processin de manera segura i independent al mateix temps sense interferències.
  • Durabilitat: Assegura que una vegada una transacció s'ha compromès, es mantindrà així, fins i tot en cas de fallada del sistema.

Transaccions de Document Únic

MongoDB proporciona automàticament atomicitat per a operacions de document únic. Això significa que qualsevol operació d'actualització, inserció o eliminació en un sol document és atòmica per defecte.

Exemple

db.collection.updateOne(
  { _id: 1 },
  { $set: { status: "active" } }
);

En aquest exemple, l'operació d'actualització en el document amb _id: 1 és atòmica.

Transaccions de Múltiples Documents

A partir de MongoDB 4.0, es suporten transaccions de múltiples documents, permetent agrupar múltiples operacions en una sola transacció.

Exemple

const session = client.startSession();

session.startTransaction();
try {
  db.collection1.insertOne({ _id: 1, name: "Alice" }, { session });
  db.collection2.insertOne({ _id: 2, name: "Bob" }, { session });
  
  session.commitTransaction();
} catch (error) {
  session.abortTransaction();
  throw error;
} finally {
  session.endSession();
}

En aquest exemple:

  1. Es comença una sessió.
  2. S'inicia una transacció.
  3. Es realitzen dues operacions d'inserció dins de la transacció.
  4. La transacció es compromet si ambdues operacions tenen èxit.
  5. Si es produeix un error, la transacció s'aborta.

Opcions de Transacció

Les transaccions es poden configurar amb diverses opcions per controlar el seu comportament.

Exemple

const session = client.startSession();
const transactionOptions = {
  readConcern: { level: "snapshot" },
  writeConcern: { w: "majority" },
  readPreference: "primary"
};

session.startTransaction(transactionOptions);
try {
  db.collection1.insertOne({ _id: 1, name: "Alice" }, { session });
  db.collection2.insertOne({ _id: 2, name: "Bob" }, { session });
  
  session.commitTransaction();
} catch (error) {
  session.abortTransaction();
  throw error;
} finally {
  session.endSession();
}

En aquest exemple, la transacció es configura amb opcions específiques de readConcern, writeConcern i readPreference.

Gestió d'Errors en Transaccions

Una correcta gestió d'errors és crucial per assegurar que les transaccions es comprometin o s'abortin correctament.

Exemple

const session = client.startSession();

session.startTransaction();
try {
  db.collection1.insertOne({ _id: 1, name: "Alice" }, { session });
  db.collection2.insertOne({ _id: 2, name: "Bob" }, { session });
  
  session.commitTransaction();
} catch (error) {
  session.abortTransaction();
  console.error("Transacció abortada a causa d'un error: ", error);
} finally {
  session.endSession();
}

En aquest exemple, qualsevol error durant la transacció farà que s'aborti, i l'error es registrarà.

Conclusió

Les transaccions a MongoDB proporcionen una manera poderosa d'assegurar la integritat i consistència de les dades a través de múltiples operacions. Entenent i utilitzant operacions atòmiques de document únic i transaccions de múltiples documents, pots construir aplicacions robustes que gestionin manipulacions complexes de dades de manera fiable. Recorda gestionar sempre els errors de manera adequada per mantenir l'estabilitat i la fiabilitat de les teves operacions de base de dades.
