Introducció

A MongoDB, una relació un a un és un tipus de relació de dades on un únic document en una col·lecció està associat amb un únic document en una altra col·lecció. Això és útil per a escenaris on vols separar dades en diferents col·leccions però mantenir un enllaç directe entre documents relacionats.

Conceptes Clau

  • Relació Un a Un: Una relació on cada document en una col·lecció està enllaçat exactament amb un document en una altra col·lecció.
  • Emmagatzematge: Emmagatzemar dades relacionades dins del mateix document.
  • Referència: Emmagatzemar dades relacionades en documents separats i enllaçar-los utilitzant referències.

Emmagatzematge vs. Referència

Emmagatzematge

  • Pros:
    • Simplicitat: Totes les dades relacionades s'emmagatzemen en un únic document.
    • Rendiment: Operacions de lectura més ràpides ja que totes les dades es recuperen en una sola consulta.
  • Contres:
    • Mida del Document: MongoDB té un límit de mida de document de 16MB, que pot ser una restricció.
    • Redundància: Duplicació de dades si les dades emmagatzemades s'utilitzen en múltiples llocs.

Referència

  • Pros:
    • Flexibilitat: Les dades es poden normalitzar i reutilitzar en diferents col·leccions.
    • Escalabilitat: Millor per a grans conjunts de dades ja que evita el límit de mida de document de 16MB.
  • Contres:
    • Complexitat: Requereix múltiples consultes per recuperar dades relacionades.
    • Rendiment: Operacions de lectura més lentes degut a la necessitat de consultes addicionals.

Implementació de Relacions Un a Un

Utilitzant Emmagatzematge

Quan utilitzes emmagatzematge, emmagatzemes el document relacionat directament dins del document pare.

Exemple

Suposem que tenim una col·lecció users i cada usuari té un perfil únic.

{
  "_id": ObjectId("60c72b2f9b1d8b1f8c8b4567"),
  "username": "johndoe",
  "email": "johndoe@example.com",
  "profile": {
    "age": 30,
    "gender": "male",
    "address": "123 Main St"
  }
}

En aquest exemple, el document profile està emmagatzemat dins del document users.

Utilitzant Referència

Quan utilitzes referència, emmagatzemes el document relacionat en una col·lecció separada i els enllaces utilitzant una referència.

Exemple

Separem els users i profiles en dues col·leccions.

Col·lecció d'Usuaris:

{
  "_id": ObjectId("60c72b2f9b1d8b1f8c8b4567"),
  "username": "johndoe",
  "email": "johndoe@example.com",
  "profile_id": ObjectId("60c72b3f9b1d8b1f8c8b4568")
}

Col·lecció de Perfils:

{
  "_id": ObjectId("60c72b3f9b1d8b1f8c8b4568"),
  "age": 30,
  "gender": "male",
  "address": "123 Main St"
}

En aquest exemple, el document users conté una referència al document profiles utilitzant el camp profile_id.

Consultes de Relacions Un a Un

Consultar Documents Emmagatzemats

Per recuperar un usuari juntament amb el seu perfil emmagatzemat, simplement consultes la col·lecció users.

Exemple

db.users.findOne({ username: "johndoe" })

Consultar Documents Referenciats

Per recuperar un usuari i el seu perfil, necessites realitzar dues consultes: una per obtenir l'usuari i una altra per obtenir el perfil.

Exemple

const user = db.users.findOne({ username: "johndoe" });
const profile = db.profiles.findOne({ _id: user.profile_id });

Conclusió

Les relacions un a un a MongoDB es poden implementar utilitzant emmagatzematge o referència, cadascuna amb els seus avantatges i desavantatges. L'emmagatzematge és més senzill i ràpid per a operacions de lectura però pot conduir a mides de document més grans. La referència és més flexible i escalable però requereix consultes més complexes. Entendre aquests compromisos t'ajudarà a dissenyar esquemes de MongoDB eficients i mantenibles.
