Introducció

La monitorització i el perfilatge són pràctiques essencials per mantenir la salut i el rendiment de les teves bases de dades MongoDB. Aquesta secció cobrirà les eines i tècniques disponibles per monitoritzar i perfilar MongoDB, des de nivells bàsics fins a avançats.

Monitorització de MongoDB

Conceptes Clau

  • Monitorització: El procés de recollir, analitzar i utilitzar informació per seguir el rendiment i la salut d'un desplegament de MongoDB.
  • Mètriques: Mesures quantitatives utilitzades per avaluar el rendiment de diversos aspectes de MongoDB.
  • Alertes: Notificacions activades quan certes mètriques superen llindars predefinits.

Eines per a la Monitorització

  • MongoDB Atlas: Un servei de base de dades al núvol totalment gestionat que inclou monitorització i alertes integrades.
  • MongoDB Cloud Manager: Un servei per gestionar, monitoritzar i fer còpies de seguretat dels desplegaments de MongoDB.
  • mongostat: Una eina de línia de comandes que proporciona una visió ràpida de l'estat d'una instància de MongoDB.
  • mongotop: Una eina de línia de comandes que segueix i informa de l'activitat de lectura i escriptura en una instància de MongoDB.

Exemple: Ús de `mongostat`

mongostat proporciona una visió ràpida de l'estat d'una instància de MongoDB, mostrant mètriques com insercions, consultes, actualitzacions, eliminacions i més.

mongostat --host your_mongodb_host --port your_mongodb_port

Sortida:

insert query update delete getmore command flushes mapped  vsize   res faults  locked db idx miss %     qr|qw   ar|aw  netIn netOut  conn       time
    0     0      0      0       0     2|0       0       0  1.12G  0.00k       0       0.0          0|0       0|0    1k     1k     1   12:00:00

Exemple: Ús de `mongotop`

mongotop proporciona una descripció del temps dedicat a llegir i escriure dades en cada col·lecció.

mongotop --host your_mongodb_host --port your_mongodb_port

Sortida:

                    ns    total    read    write    2019-01-01T00:00:00Z
       test.collection    0ms      0ms      0ms

Perfilatge de MongoDB

Conceptes Clau

  • Perfilatge: El procés de registrar i analitzar el rendiment de les operacions de la base de dades per identificar consultes lentes i altres colls d'ampolla de rendiment.
  • Nivells de Perfilatge: MongoDB proporciona diferents nivells de perfilatge per controlar la quantitat de dades recollides.

Nivells de Perfilatge

  • Nivell 0: Sense perfilatge.
  • Nivell 1: Perfilatge d'operacions lentes.
  • Nivell 2: Perfilatge de totes les operacions.

Activació del Perfilatge

Per activar el perfilatge, pots utilitzar el mètode db.setProfilingLevel().

// Activar el perfilatge per a operacions lentes (el llindar per defecte és 100ms)
db.setProfilingLevel(1);

// Activar el perfilatge per a totes les operacions
db.setProfilingLevel(2);

// Desactivar el perfilatge
db.setProfilingLevel(0);

Visualització de Dades de Perfilatge

Les dades de perfilatge es guarden a la col·lecció system.profile. Pots consultar aquesta col·lecció per analitzar el rendiment de les teves operacions de base de dades.

// Trobar totes les dades de perfilatge
db.system.profile.find().pretty();

// Trobar operacions lentes (per exemple, operacions que duren més de 200ms)
db.system.profile.find({ millis: { $gt: 200 } }).pretty();

Monitorització i Perfilatge Avançats

Alertes i Panells Personalitzats

  • Alertes Personalitzades: Configura alertes personalitzades a MongoDB Atlas o Cloud Manager per notificar-te quan certes mètriques superin llindars.
  • Panells: Crea panells personalitzats per visualitzar mètriques clau i tendències al llarg del temps.

Integració amb Eines de Tercers

  • Prometheus: Utilitza el MongoDB Exporter per integrar les mètriques de MongoDB amb Prometheus per a una monitorització i alertes avançades.
  • Grafana: Visualitza les mètriques de MongoDB a Grafana integrant-lo amb Prometheus.

Exemple: Configuració d'una Alerta Personalitzada a MongoDB Atlas

  1. Navega a la secció d'Alertes a MongoDB Atlas.
  2. Fes clic a "Crear Alerta".
  3. Tria la mètrica que vols monitoritzar (per exemple, ús de CPU, ús de memòria).
  4. Configura el llindar i els paràmetres de notificació.
  5. Desa l'alerta.

Conclusió

La monitorització i el perfilatge són crítics per mantenir el rendiment i la fiabilitat dels teus desplegaments de MongoDB. Utilitzant les eines i tècniques cobertes en aquesta secció, pots obtenir informació valuosa sobre la salut de la teva base de dades, identificar colls d'ampolla de rendiment i prendre mesures proactives per assegurar un rendiment òptim. Ja sigui utilitzant eines integrades com mongostat i mongotop, o integrant amb solucions de tercers com Prometheus i Grafana, una monitorització i perfilatge efectius t'ajudaran a gestionar les teves bases de dades MongoDB de manera més eficient.
