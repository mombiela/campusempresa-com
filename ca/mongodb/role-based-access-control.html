Introducció a RBAC

El Control d'Accés Basat en Rols (RBAC) és un mètode per regular l'accés a recursos informàtics o de xarxa basat en els rols dels usuaris individuals dins d'una empresa. A MongoDB, RBAC s'utilitza per gestionar permisos i assegurar-se que els usuaris tenen els nivells d'accés adequats per realitzar les seves tasques.

Conceptes Clau

  • Rol: Una col·lecció de privilegis que es poden assignar a un usuari.
  • Privilegi: Una acció específica que un usuari pot realitzar en un recurs.
  • Usuari: Un individu a qui se li assignen un o més rols.

Configuració de RBAC a MongoDB

Creació d'Usuaris

Per crear un usuari a MongoDB, s'utilitza el mètode db.createUser(). Cada usuari ha de tenir assignat almenys un rol.

db.createUser({
  user: "username",
  pwd: "password",
  roles: [
    { role: "readWrite", db: "exampleDB" }
  ]
});
  • user: El nom d'usuari.
  • pwd: La contrasenya per a l'usuari.
  • roles: Una matriu de rols assignats a l'usuari.

Rols Integrats

MongoDB proporciona diversos rols integrats que cobreixen casos d'ús comuns. Alguns dels rols integrats clau inclouen:

| Rol | Descripció | |---------------|-------------------------------------------------| | read | Concedeix accés de només lectura a la base de dades. | | readWrite | Concedeix accés de lectura i escriptura a la base de dades. | | dbAdmin | Proporciona privilegis administratius a la base de dades. | | userAdmin | Permet la gestió d'usuaris dins de la base de dades. | | clusterAdmin| Concedeix privilegis administratius al clúster. |

Rols Personalitzats

També podeu crear rols personalitzats per satisfer requisits específics.

db.createRole({
  role: "customRole",
  privileges: [
    { resource: { db: "exampleDB", collection: "" }, actions: ["find", "insert"] }
  ],
  roles: []
});
  • role: El nom del rol personalitzat.
  • privileges: Una matriu de privilegis, on cada privilegi especifica el recurs i les accions permeses.
  • roles: Una matriu de rols que aquest rol hereta.

Gestió de Rols d'Usuari

Assignació de Rols a Usuaris

Podeu assignar rols addicionals a un usuari existent utilitzant el mètode db.grantRolesToUser().

db.grantRolesToUser("username", [{ role: "dbAdmin", db: "exampleDB" }]);

Revocació de Rols d'Usuaris

Per revocar rols d'un usuari, utilitzeu el mètode db.revokeRolesFromUser().

db.revokeRolesFromUser("username", [{ role: "readWrite", db: "exampleDB" }]);

Exemples Pràctics

Exemple 1: Creació d'un Usuari de Només Lectura

db.createUser({
  user: "readonlyUser",
  pwd: "readonlyPass",
  roles: [
    { role: "read", db: "exampleDB" }
  ]
});

Això crea un usuari readonlyUser amb accés de només lectura a exampleDB.

Exemple 2: Creació d'un Usuari amb Rol Personalitzat

db.createRole({
  role: "dataAnalyst",
  privileges: [
    { resource: { db: "exampleDB", collection: "" }, actions: ["find"] }
  ],
  roles: []
});

db.createUser({
  user: "analystUser",
  pwd: "analystPass",
  roles: [
    { role: "dataAnalyst", db: "exampleDB" }
  ]
});

Això crea un rol personalitzat dataAnalyst amb accés de només lectura i l'assigna a analystUser.

Conclusió

RBAC a MongoDB és una característica potent que permet gestionar els permisos dels usuaris de manera efectiva. Entenent i utilitzant rols, privilegis i usuaris, podeu assegurar-vos que la vostra implementació de MongoDB sigui segura i que els usuaris tinguin els nivells d'accés adequats per realitzar les seves tasques. Tant si utilitzeu rols integrats com si creeu rols personalitzats, RBAC proporciona la flexibilitat necessària per gestionar diverses situacions de control d'accés.
