Les operacions d'agrupació processen registres de dades i retornen resultats calculats. MongoDB proporciona un potent marc d'agrupació que et permet realitzar processament i anàlisi de dades complexos directament dins de la base de dades.

Conceptes Clau de l'Agrupació

  • Agrupació per Canonada: Un marc per a l'agrupació de dades modelat sobre el concepte de canonades de processament de dades. Els documents entren en una canonada de múltiples etapes que transforma els documents en resultats agregats.
  • Etapes: Cada etapa de la canonada realitza una operació sobre els documents d'entrada. Les etapes poden filtrar, ordenar, agrupar, remodelar i modificar els documents que passen per la canonada.
  • Expressions: Utilitzades dins de les etapes per calcular valors, realitzar operacions i transformar dades.

Agrupació per Canonada

L'agrupació per canonada és una sèrie d'etapes que processen documents:

  1. $match: Filtra documents per passar només aquells que compleixen la(s) condició(ns) especificada(es).
  2. $group: Agrupa els documents d'entrada per una expressió d'identificador especificada i aplica les expressions acumuladores a cada grup.
  3. $project: Remodela cada document en el flux, com ara afegint, eliminant o canviant el nom de camps.
  4. $sort: Reordena els documents en el flux per una clau d'ordenació especificada.
  5. $limit: Restringeix el nombre de documents en la sortida.
  6. $skip: Salta un nombre especificat de documents de la canonada i passa la resta.

Exemple d'Agrupació per Canonada

Considera una col·lecció orders amb documents com:

{
  "_id": 1,
  "product": "Laptop",
  "quantity": 2,
  "price": 1000,
  "customer": "John Doe"
}

Per calcular les vendes totals per a cada producte, pots utilitzar la següent agrupació per canonada:

db.orders.aggregate([
  {
    $group: {
      _id: "$product",
      totalSales: { $sum: { $multiply: ["$quantity", "$price"] } }
    }
  }
])

Explicació:

  • $group: Agrupa els documents pel camp product.
  • _id: El camp pel qual agrupar els documents.
  • totalSales: Calcula les vendes totals multiplicant quantity i price per a cada document i sumant els resultats.

Etapes Comunes d'Agrupació

$match

Filtra documents per passar només aquells que compleixen la(s) condició(ns) especificada(es).

db.orders.aggregate([
  { $match: { customer: "John Doe" } }
])

$group

Agrupa els documents d'entrada per una expressió d'identificador especificada i aplica les expressions acumuladores a cada grup.

db.orders.aggregate([
  {
    $group: {
      _id: "$customer",
      totalQuantity: { $sum: "$quantity" }
    }
  }
])

$project

Remodela cada document en el flux, com ara afegint, eliminant o canviant el nom de camps.

db.orders.aggregate([
  {
    $project: {
      product: 1,
      totalCost: { $multiply: ["$quantity", "$price"] }
    }
  }
])

$sort

Reordena els documents en el flux per una clau d'ordenació especificada.

db.orders.aggregate([
  { $sort: { totalSales: -1 } }
])

$limit

Restringeix el nombre de documents en la sortida.

db.orders.aggregate([
  { $limit: 5 }
])

$skip

Salta un nombre especificat de documents de la canonada i passa la resta.

db.orders.aggregate([
  { $skip: 10 }
])

Conclusió

L'agrupació en MongoDB és una eina poderosa per a l'anàlisi i transformació de dades. Entenent i utilitzant la canonada d'agrupació i les seves etapes, pots realitzar consultes complexes i obtenir informació significativa de les teves dades. Practica utilitzant diferents etapes i expressions per convertir-te en un expert en la construcció de canonades d'agrupació eficients.
