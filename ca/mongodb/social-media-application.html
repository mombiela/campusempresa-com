Introducció

En aquest curs, explorarem com utilitzar MongoDB per construir una aplicació de xarxes socials. Començarem amb els conceptes bàsics de MongoDB i progressivament avançarem cap a temes més avançats. Al final d'aquest curs, tindràs una comprensió sòlida de MongoDB i de com utilitzar-lo per gestionar dades per a una aplicació de xarxes socials.

Començant amb MongoDB

Què és MongoDB?

  • MongoDB és una base de dades NoSQL que emmagatzema dades en documents similars a JSON.
  • Està dissenyat per a l'escalabilitat i la flexibilitat, fent-lo ideal per a aplicacions web modernes.

Instal·lant MongoDB

  • Segueix la guia oficial d'instal·lació de MongoDB per al teu sistema operatiu.
  • Verifica la instal·lació executant mongo --version al teu terminal.

Operacions bàsiques de MongoDB

  • Connectant-se a MongoDB: Utilitza la shell de MongoDB o un client com MongoDB Compass.
  • Operacions CRUD:
    • Crear: Inserir documents en una col·lecció.
    • Llegir: Consultar documents d'una col·lecció.
    • Actualitzar: Modificar documents existents.
    • Eliminar: Eliminar documents d'una col·lecció.
// Connectant-se a MongoDB
const { MongoClient } = require('mongodb');
const uri = "your_mongodb_connection_string";
const client = new MongoClient(uri, { useNewUrlParser: true, useUnifiedTopology: true });

async function run() {
    try {
        await client.connect();
        console.log("Connected to MongoDB");
    } finally {
        await client.close();
    }
}
run().catch(console.dir);

Dissenyant l'Esquema de l'Aplicació de Xarxes Socials

Col·lecció d'Usuaris

  • Camps:
    • username: String, identificador únic per a l'usuari.
    • email: String, adreça de correu electrònic única.
    • password: String, contrasenya encriptada.
    • profile: Objecte que conté informació del perfil de l'usuari.
const userSchema = {
    username: "john_doe",
    email: "john@example.com",
    password: "hashed_password",
    profile: {
        fullName: "John Doe",
        bio: "Software Developer",
        avatar: "url_to_avatar"
    }
};

Col·lecció de Publicacions

  • Camps:
    • userId: ObjectId, referència a l'usuari que ha creat la publicació.
    • content: String, el contingut de la publicació.
    • createdAt: Date, marca de temps de quan es va crear la publicació.
    • likes: Array d'ObjectIds, referències als usuaris que han agradat la publicació.
    • comments: Array d'objectes de comentaris.
const postSchema = {
    userId: ObjectId("user_id"),
    content: "This is a sample post",
    createdAt: new Date(),
    likes: [ObjectId("user_id_1"), ObjectId("user_id_2")],
    comments: [
        {
            userId: ObjectId("commenter_id"),
            comment: "Nice post!",
            createdAt: new Date()
        }
    ]
};

Col·lecció de Comentaris

  • Camps:
    • postId: ObjectId, referència a la publicació.
    • userId: ObjectId, referència a l'usuari que ha comentat.
    • comment: String, el text del comentari.
    • createdAt: Date, marca de temps de quan es va crear el comentari.
const commentSchema = {
    postId: ObjectId("post_id"),
    userId: ObjectId("user_id"),
    comment: "This is a comment",
    createdAt: new Date()
};

Característiques Avançades de MongoDB

Indexació

  • Propòsit: Millorar el rendiment de les consultes.
  • Tipus:
    • Índex de camp únic
    • Índex compost
    • Índex de text
// Creant un índex en el camp username
db.users.createIndex({ username: 1 });

// Creant un índex compost en email i username
db.users.createIndex({ email: 1, username: 1 });

Marc d'Agrupació

  • Propòsit: Realitzar processament i anàlisi de dades complexos.
  • Etapes:
    • $match: Filtrar documents.
    • $group: Agrupar documents per un camp especificat.
    • $sort: Ordenar documents.
// Canalització d'agrupació per obtenir el nombre de publicacions per usuari
db.posts.aggregate([
    { $group: { _id: "$userId", postCount: { $sum: 1 } } },
    { $sort: { postCount: -1 } }
]);

Transaccions

  • Propòsit: Assegurar la consistència de les dades a través de múltiples operacions.
  • Ús: Utilitza transaccions per a operacions que necessiten ser atòmiques.
async function createPostWithTransaction(client, userId, content) {
    const session = client.startSession();
    session.startTransaction();
    try {
        const postsCollection = client.db("social_media").collection("posts");
        const result = await postsCollection.insertOne({ userId, content, createdAt: new Date() }, { session });
        await session.commitTransaction();
        console.log("Post created successfully");
    } catch (error) {
        await session.abortTransaction();
        console.error("Transaction aborted due to an error: ", error);
    } finally {
        session.endSession();
    }
}
