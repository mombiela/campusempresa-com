Utilitzant `$lookup` per a Joins

Introducció a `$lookup`

MongoDB és una base de dades NoSQL que no suporta joins tradicionals de SQL. No obstant això, MongoDB proporciona l'etapa d'agregació $lookup per realitzar operacions de join entre col·leccions.

  • Propòsit: $lookup s'utilitza per unir documents de diferents col·leccions.
  • Sintaxi: 
    {
  $lookup: {
    from: <col·lecció a unir>,
    localField: <camp dels documents d'entrada>,
    foreignField: <camp dels documents de la col·lecció "from">,
    as: <camp de sortida de l'array>
  }
}

Ús bàsic de `$lookup`

Comencem amb un exemple senzill per entendre com funciona $lookup.

Escenari d'Exemple

Tenim dues col·leccions: orders i customers.

Col·lecció orders

[
  { "_id": 1, "product": "Laptop", "customerId": 101 },
  { "_id": 2, "product": "Phone", "customerId": 102 },
  { "_id": 3, "product": "Tablet", "customerId": 101 }
]

Col·lecció customers

[
  { "_id": 101, "name": "John Doe", "email": "john@example.com" },
  { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
]

Realitzant un Join

Volem unir la col·lecció orders amb la col·lecció customers per incloure els detalls del client en cada comanda.

Pipeline d'Agregació

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  }
])

Resultat

[
  {
    "_id": 1,
    "product": "Laptop",
    "customerId": 101,
    "customerDetails": [
      { "_id": 101, "name": "John Doe", "email": "john@example.com" }
    ]
  },
  {
    "_id": 2,
    "product": "Phone",
    "customerId": 102,
    "customerDetails": [
      { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
    ]
  },
  {
    "_id": 3,
    "product": "Tablet",
    "customerId": 101,
    "customerDetails": [
      { "_id": 101, "name": "John Doe", "email": "john@example.com" }
    ]
  }
]

Ús avançat de `$lookup`

Desenrotllant l'Array Unit

Per defecte, $lookup crea un array de documents coincidents. Per aplanar aquest array, utilitzem l'etapa $unwind.

Exemple

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  },
  { $unwind: "$customerDetails" }
])

Resultat

[
  {
    "_id": 1,
    "product": "Laptop",
    "customerId": 101,
    "customerDetails": { "_id": 101, "name": "John Doe", "email": "john@example.com" }
  },
  {
    "_id": 2,
    "product": "Phone",
    "customerId": 102,
    "customerDetails": { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
  },
  {
    "_id": 3,
    "product": "Tablet",
    "customerId": 101,
    "customerDetails": { "_id": 101, "name": "John Doe", "email": "john@example.com" }
  }
]

Gestionant Documents No Coincidents

Per incloure documents que no tenen una coincidència en la col·lecció unida, utilitzeu l'opció preserveNullAndEmptyArrays amb $unwind.

Exemple

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  },
  { $unwind: { path: "$customerDetails", preserveNullAndEmptyArrays: true } }
])

Resultat

Si hi hagués comandes amb customerId no present en la col·lecció customers, aquestes comandes encara apareixerien en el resultat amb customerDetails establert a null.

Resum

  • Join Bàsic: Utilitzeu $lookup per unir documents de diferents col·leccions.
  • Aplanant Resultats: Utilitzeu $unwind per aplanar l'array creat per $lookup.
  • Gestionant Documents No Coincidents: Utilitzeu preserveNullAndEmptyArrays amb $unwind per incloure documents sense coincidències.

Dominant $lookup, podeu realitzar efectivament operacions de join a MongoDB, permetent-vos combinar dades de múltiples col·leccions d'una manera flexible i potent.
