Introducció

A MongoDB, entendre els tipus de dades és crucial per a un disseny i consulta de bases de dades efectius. MongoDB admet un conjunt ric de tipus de dades que et permeten emmagatzemar i manipular dades en diverses formes. Aquesta secció cobrirà els principals tipus de dades disponibles a MongoDB, proporcionant explicacions, exemples i exercicis per ajudar-te a comprendre el seu ús.

Tipus de dades clau

String

  • Descripció: Utilitzat per emmagatzemar dades de text.
  • Exemple: 
    { "name": "Alice" }
  • Ús: Comunament utilitzat per emmagatzemar noms, adreces i altra informació textual.

Integer

  • Descripció: Utilitzat per emmagatzemar dades numèriques sense component fraccionari.
  • Exemple: 
    { "age": 30 }
  • Ús: Ideal per comptar, indexar i emmagatzemar nombres enters.

Double

  • Descripció: Utilitzat per emmagatzemar nombres de punt flotant.
  • Exemple: 
    { "price": 19.99 }
  • Ús: Adequat per emmagatzemar preus, mesures i altres valors decimals.

Boolean

  • Descripció: Representa un valor binari, ja sigui true o false.
  • Exemple: 
    { "isActive": true }
  • Ús: Útil per a banderes, indicadors d'estat i condicions binàries.

Array

  • Descripció: Utilitzat per emmagatzemar llistes de valors.
  • Exemple: 
    { "tags": ["mongodb", "database", "nosql"] }
  • Ús: Ideal per emmagatzemar múltiples valors en un sol camp, com ara etiquetes, categories o llistes d'articles.

Object

  • Descripció: Utilitzat per emmagatzemar documents incrustats.
  • Exemple: 
    { 
  "address": {
    "street": "123 Main St",
    "city": "Anytown",
    "zip": "12345"
  }
}
  • Ús: Útil per a la nidificació de dades relacionades dins d'un sol document.

Date

  • Descripció: Utilitzat per emmagatzemar dates i hores.
  • Exemple: 
    { "createdAt": ISODate("2023-10-01T00:00:00Z") }
  • Ús: Essencial per a la marca de temps d'esdeveniments, programació i seguiment de dades basades en el temps.

ObjectId

  • Descripció: Un identificador únic per a documents.
  • Exemple: 
    { "_id": ObjectId("507f1f77bcf86cd799439011") }
  • Ús: Generat automàticament per a cada document en una col·lecció, assegurant la unicitat.

Null

  • Descripció: Representa l'absència d'un valor.
  • Exemple: 
    { "middleName": null }
  • Ús: Útil per a camps que són opcionals o desconeguts.

Binary Data

  • Descripció: Utilitzat per emmagatzemar dades binàries.
  • Exemple: 
    { "file": BinData(0, "base64encodedstring") }
  • Ús: Adequat per emmagatzemar imatges, arxius i altres continguts binaris.

Regular Expression

  • Descripció: Utilitzat per emmagatzemar expressions regulars.
  • Exemple: 
    { "pattern": /abc/i }
  • Ús: Útil per a la coincidència de patrons dins de cadenes.

Comparació de tipus de dades

| Tipus de dades | Descripció | Exemple | |---------------------|--------------------------------------|----------------------------------------------| | String | Dades de text | { "name": "Alice" } | | Integer | Nombres enters | { "age": 30 } | | Double | Nombres de punt flotant | { "price": 19.99 } | | Boolean | Valors binaris | { "isActive": true } | | Array | Llistes de valors | { "tags": ["mongodb", "database", "nosql"] } | | Object | Documents incrustats | { "address": { "street": "123 Main St" } } | | Date | Dates i hores | { "createdAt": ISODate("2023-10-01T00:00:00Z") } | | ObjectId | Identificador únic | { "_id": ObjectId("507f1f77bcf86cd799439011") } | | Null | Absència de valor | { "middleName": null } | | Binary Data | Contingut binari | { "file": BinData(0, "base64encodedstring") } | | Regular Expression | Coincidència de patrons | { "pattern": /abc/i } |

Exercicis

  1. Crea un document amb diversos tipus de dades:

    • Crea un document de MongoDB que inclogui almenys cinc tipus de dades diferents.
    • Exemple: 
      {
  "name": "John Doe",
  "age": 25,
  "isActive": true,
  "tags": ["developer", "blogger"],
  "address": {
    "street": "456 Elm St",
    "city": "Somewhere",
    "zip": "67890"
  },
  "createdAt": ISODate("2023-10-01T00:00:00Z")
}

  2. Consulta documents per tipus de dades:

    • Escriu una consulta per trobar tots els documents on el camp isActive sigui true.
    • Exemple: 
      db.collection.find({ "isActive": true })

  3. Actualitza un document amb un nou tipus de dades:

    • Afegeix un nou camp lastLogin amb la data actual a un document existent.
    • Exemple: 
      db.collection.updateOne(
  { "name": "John Doe" },
  { $set: { "lastLogin": new Date() } }
)

Conclusió

Entendre i utilitzar efectivament els tipus de dades de MongoDB és fonamental per dissenyar bases de dades robustes i eficients. Cada tipus de dada serveix a un propòsit específic i pot influir molt en com emmagatzemes, consultes i manipules les dades. En dominar aquests tipus de dades, pots assegurar-te que les teves aplicacions de MongoDB siguin tant rendibles com escalables.
