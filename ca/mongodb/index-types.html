Els índexs són un aspecte crucial de MongoDB que milloren significativament el rendiment de les consultes. Aquesta secció cobrirà els diferents tipus d'índexs disponibles a MongoDB, els seus casos d'ús i com implementar-los.

Introducció als Índexs

  • Definició: Un índex és una estructura de dades que millora la velocitat de les operacions de recuperació de dades en una taula de base de dades a costa d'escriptures addicionals i espai d'emmagatzematge.
  • Propòsit: Els índexs s'utilitzen per localitzar ràpidament dades sense haver de cercar cada document en una col·lecció.

Tipus d'Índexs

Índex de Camp Únic

  • Descripció: Índexs en un sol camp.
  • Cas d'Ús: Quan les consultes filtren o ordenen freqüentment per un sol camp.
  • Exemple: 
    db.collection.createIndex({ fieldName: 1 });
    Això crea un índex ascendent en fieldName.

Índex Compost

  • Descripció: Índexs en múltiples camps.
  • Cas d'Ús: Quan les consultes filtren o ordenen per múltiples camps.
  • Exemple: 
    db.collection.createIndex({ field1: 1, field2: -1 });
    Això crea un índex ascendent en field1 i un índex descendent en field2.

Índex Multiclau

  • Descripció: Índexs en camps d'array.
  • Cas d'Ús: Quan els camps contenen arrays i les consultes necessiten cercar dins d'aquests arrays.
  • Exemple: 
    db.collection.createIndex({ arrayField: 1 });
    Això crea un índex en els elements de arrayField.

Índex de Text

  • Descripció: Índexs per a la cerca de text.
  • Cas d'Ús: Quan es realitzen cerques de text dins del contingut de cadenes.
  • Exemple: 
    db.collection.createIndex({ fieldName: "text" });
    Això crea un índex de text en fieldName.

Índex Geoespacial

  • Descripció: Índexs per a consultes geoespacials.
  • Cas d'Ús: Quan es consulten dades geogràfiques.
  • Exemple: 
    db.collection.createIndex({ location: "2dsphere" });
    Això crea un índex 2dsphere en location.

Índex Hashed

  • Descripció: Índexs que emmagatzemen valors hashed del camp indexat.
  • Cas d'Ús: Quan es distribueixen dades a través d'un clúster fragmentat.
  • Exemple: 
    db.collection.createIndex({ fieldName: "hashed" });
    Això crea un índex hashed en fieldName.

Índex de Comodí

  • Descripció: Índexs en tots els camps dins dels documents.
  • Cas d'Ús: Quan s'indexen camps dinàmics o imprevisibles.
  • Exemple: 
    db.collection.createIndex({ "$**": 1 });
    Això crea un índex de comodí en tots els camps.

Comparació dels Tipus d'Índexs

| Tipus d'Índex | Cas d'Ús | Codi d'Exemple | |------------------|-----------------------------------------------|---------------------------------------------------| | Camp Únic | Consultes de camp únic | db.collection.createIndex({ fieldName: 1 }); | | Compost | Consultes de múltiples camps | db.collection.createIndex({ field1: 1, field2: -1 }); | | Multiclau | Consultes de camps d'array | db.collection.createIndex({ arrayField: 1 }); | | Text | Cerca de text | db.collection.createIndex({ fieldName: "text" }); | | Geoespacial | Consultes de dades geogràfiques | db.collection.createIndex({ location: "2dsphere" }); | | Hashed | Distribució de clúster fragmentat | db.collection.createIndex({ fieldName: "hashed" }); | | Comodí | Camps dinàmics o imprevisibles | db.collection.createIndex({ "$**": 1 }); |

Conclusió

Els índexs són una eina poderosa a MongoDB que poden millorar enormement el rendiment de les consultes. Entendre els diferents tipus d'índexs i els seus casos d'ús et permet optimitzar la teva base de dades de manera efectiva. Considera sempre les necessitats específiques de les teves consultes i l'estructura de dades a l'hora de triar el tipus d'índex adequat. Recorda que, mentre els índexs acceleren les operacions de lectura, poden afegir càrrega a les operacions d'escriptura i consumir espai d'emmagatzematge addicional.
