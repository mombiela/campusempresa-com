Introducció

A MongoDB, una relació un a molts és un patró comú de disseny de bases de dades on un únic document (el "un") està relacionat amb múltiples documents (els "molts"). Aquest patró és essencial per modelar escenaris del món real com ara una entrada de blog amb múltiples comentaris, un usuari amb múltiples comandes, o una categoria amb múltiples productes.

Conceptes Clau

  • Relació Un a Molts: Un únic document en una col·lecció està relacionat amb múltiples documents en una altra col·lecció.
  • Embutxacament: Emmagatzemar documents relacionats dins d'un únic document.
  • Referenciació: Emmagatzemar referències a documents relacionats en col·leccions separades.

Embutxacament de Documents

L'embutxacament és un mètode on les dades relacionades s'emmagatzemen dins del mateix document. Això és útil quan les dades relacionades es consulten freqüentment juntes.

Exemple: Entrada de Blog amb Comentaris

{
  "_id": 1,
  "title": "Introducció a MongoDB",
  "content": "MongoDB és una base de dades NoSQL...",
  "comments": [
    {
      "user": "Alice",
      "comment": "Gran entrada!",
      "date": "2023-10-01"
    },
    {
      "user": "Bob",
      "comment": "Molt informatiu.",
      "date": "2023-10-02"
    }
  ]
}

Avantatges de l'Embutxacament

  • Rendiment: Operacions de lectura més ràpides ja que les dades relacionades estan en un únic document.
  • Atomicitat: Les actualitzacions del document són atòmiques.

Desavantatges de l'Embutxacament

  • Mida del Document: Els documents de MongoDB tenen un límit de mida de 16MB. Embutxacant massa dades es pot superar aquest límit.
  • Duplicació de Dades: Si les dades embutxacades s'utilitzen en múltiples llocs, pot conduir a la duplicació.

Referenciació de Documents

La referenciació és un mètode on les dades relacionades s'emmagatzemen en documents separats, i es fan servir referències (normalment ObjectIDs) per enllaçar-los. Això és útil quan les dades relacionades són grans o es consulten freqüentment de manera independent.

Exemple: Usuaris i Comandes

Col·lecció d'Usuaris

{
  "_id": 1,
  "name": "Alice",
  "email": "alice@example.com",
  "orders": [101, 102]
}

Col·lecció de Comandes

{
  "_id": 101,
  "product": "Portàtil",
  "quantity": 1,
  "price": 1200
}
{
  "_id": 102,
  "product": "Ratolí",
  "quantity": 2,
  "price": 40
}

Avantatges de la Referenciació

  • Flexibilitat: Més fàcil de gestionar conjunts de dades grans i relacions.
  • Normalització: Redueix la duplicació de dades.

Desavantatges de la Referenciació

  • Rendiment: Requereix consultes addicionals per obtenir les dades relacionades.
  • Complexitat: Consultes i gestió de dades més complexes.

Triar Entre Embutxacament i Referenciació

  • Utilitza Embutxacament quan:
    • Les dades relacionades es consulten freqüentment juntes.
    • Les dades relacionades no són grans i no creixen indefinidament.
  • Utilitza Referenciació quan:
    • Les dades relacionades són grans o creixen indefinidament.
    • Les dades relacionades es consulten freqüentment de manera independent.

Exercicis Pràctics

  1. Exercici d'Embutxacament:

    • Crea una col·lecció anomenada posts i insereix un document amb comentaris embutxacats.
    • Consulta la col·lecció posts per obtenir l'entrada juntament amb els seus comentaris.

  2. Exercici de Referenciació:

    • Crea dues col·leccions anomenades users i orders.
    • Insereix documents a ambdues col·leccions i estableix una referència entre un usuari i les seves comandes.
    • Consulta la col·lecció users per obtenir un usuari i les seves comandes utilitzant les referències.

Conclusió

Entendre les relacions un a molts a MongoDB és crucial per dissenyar bases de dades eficients i escalables. Triant el mètode adequat (embutxacament o referenciació) segons les necessitats de la teva aplicació, pots optimitzar el rendiment i la mantenibilitat. Practica amb exemples del món real per consolidar la teva comprensió i convertir-te en un expert en modelar relacions un a molts a MongoDB.
