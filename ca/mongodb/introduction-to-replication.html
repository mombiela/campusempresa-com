La replicació en MongoDB és una característica clau que assegura alta disponibilitat i redundància de dades. Aquest tema cobrirà els conceptes bàsics de la replicació, els seus beneficis i com configurar la replicació en MongoDB.

Què és la Replicació?

La replicació és el procés de sincronitzar dades entre múltiples servidors. En MongoDB, la replicació s'aconsegueix a través d'un conjunt de rèpliques, que és un grup d'instàncies mongod que mantenen el mateix conjunt de dades.

Conceptes Clau

  • Conjunt de Rèpliques: Un grup d'instàncies mongod que repliquen dades entre elles.
  • Node Primari: El node que rep totes les operacions d'escriptura.
  • Nodes Secundaris: Nodes que repliquen les dades del node primari i poden servir operacions de lectura.
  • Àrbitre: Un node que participa en eleccions però no conté dades.

Beneficis de la Replicació

La replicació proporciona diversos beneficis:

  • Alta Disponibilitat: Assegura que les dades estiguin disponibles fins i tot si alguns nodes fallen.
  • Redundància de Dades: Es mantenen múltiples còpies de les dades per prevenir la pèrdua de dades.
  • Escalabilitat de Lectura: Els nodes secundaris poden servir operacions de lectura, distribuint la càrrega.

Configuració d'un Conjunt de Rèpliques

Per configurar un conjunt de rèpliques en MongoDB, segueix aquests passos:

Pas 1: Iniciar Instàncies de MongoDB

Inicia múltiples instàncies mongod en diferents ports o màquines. Per simplicitat, utilitzarem tres instàncies en la mateixa màquina amb diferents ports.

# Iniciar la primera instància
mongod --port 27017 --dbpath /data/db1 --replSet rs0

# Iniciar la segona instància
mongod --port 27018 --dbpath /data/db2 --replSet rs0

# Iniciar la tercera instància
mongod --port 27019 --dbpath /data/db3 --replSet rs0

Pas 2: Inicialitzar el Conjunt de Rèpliques

Connecta't a una de les instàncies utilitzant la shell de mongo i inicialitza el conjunt de rèpliques.

// Connectar-se a la primera instància
mongo --port 27017

// Inicialitzar el conjunt de rèpliques
rs.initiate({
  _id: "rs0",
  members: [
    { _id: 0, host: "localhost:27017" },
    { _id: 1, host: "localhost:27018" },
    { _id: 2, host: "localhost:27019" }
  ]
})

Pas 3: Verificar el Conjunt de Rèpliques

Comprova l'estat del conjunt de rèpliques per assegurar-te que està configurat correctament.

rs.status()

Exemple: Afegir un Àrbitre

Es pot afegir un àrbitre al conjunt de rèpliques per participar en eleccions sense contenir dades.

// Afegir un àrbitre al conjunt de rèpliques
rs.addArb("localhost:27020")

Explicació del Codi

  • rs.initiate(): Inicialitza el conjunt de rèpliques amb la configuració especificada.
  • rs.status(): Proporciona l'estat actual del conjunt de rèpliques.
  • rs.addArb(): Afegeix un àrbitre al conjunt de rèpliques.

Conclusió

La replicació és una característica fonamental en MongoDB que assegura alta disponibilitat, redundància de dades i escalabilitat de lectura. Configurant un conjunt de rèpliques, pots distribuir dades entre múltiples nodes, proporcionant un sistema de bases de dades robust i resilient. Entendre i implementar la replicació és crucial per mantenir un desplegament fiable de MongoDB.
