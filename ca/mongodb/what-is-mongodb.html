Introducció a MongoDB

MongoDB és una base de dades NoSQL popular coneguda per la seva flexibilitat, escalabilitat i facilitat d'ús. A diferència de les bases de dades relacionals tradicionals, MongoDB emmagatzema dades en un format similar a JSON, cosa que permet un emmagatzematge de dades més dinàmic i no estructurat.

Conceptes Clau

  • Base de Dades NoSQL: MongoDB és una base de dades NoSQL, el que significa que no utilitza l'estructura tradicional de bases de dades relacionals basades en taules.
  • Orientada a Documents: Les dades s'emmagatzemen en documents, que són similars als objectes JSON.
  • Sense Esquema: MongoDB no requereix un esquema predefinit, permetent models de dades més flexibles.
  • Escalabilitat: MongoDB està dissenyat per escalar distribuint dades a través de múltiples servidors.

Emmagatzematge Orientat a Documents

A MongoDB, les dades s'emmagatzemen en documents dins de col·leccions. Un document és un conjunt de parells clau-valor, i una col·lecció és un grup de documents.

Exemple de Document

{
  "name": "John Doe",
  "age": 29,
  "email": "john.doe@example.com",
  "address": {
    "street": "123 Main St",
    "city": "Anytown",
    "state": "CA",
    "zip": "12345"
  }
}
  • Parells Clau-Valor: Cada peça de dades s'emmagatzema com un parell clau-valor.
  • Documents Imbricats: Els documents poden contenir documents imbricats, permetent estructures de dades complexes.

Col·leccions

Una col·lecció a MongoDB és anàloga a una taula en una base de dades relacional però sense un esquema fix.

Exemple de Col·lecció

[
  {
    "name": "John Doe",
    "age": 29,
    "email": "john.doe@example.com"
  },
  {
    "name": "Jane Smith",
    "age": 34,
    "email": "jane.smith@example.com"
  }
]
  • Esquema Flexible: Cada document en una col·lecció pot tenir una estructura diferent.
  • Dades Dinàmiques: Les col·leccions poden créixer i canviar amb el temps sense requerir modificacions d'esquema.

Operacions Bàsiques

Inserir Documents

Per inserir un document en una col·lecció, utilitzes els mètodes insertOne o insertMany.

db.users.insertOne({
  "name": "John Doe",
  "age": 29,
  "email": "john.doe@example.com"
});

Consultar Documents

Per recuperar documents d'una col·lecció, utilitzes el mètode find.

db.users.find({ "age": { "$gt": 25 } });
  • Operadors de Consulta: MongoDB proporciona diversos operadors com $gt (més gran que) per filtrar documents.

Actualitzar Documents

Per actualitzar documents, utilitzes els mètodes updateOne o updateMany.

db.users.updateOne(
  { "name": "John Doe" },
  { "$set": { "age": 30 } }
);
  • Operadors d'Actualització: Operadors com $set s'utilitzen per especificar els camps a actualitzar.

Eliminar Documents

Per eliminar documents, utilitzes els mètodes deleteOne o deleteMany.

db.users.deleteOne({ "name": "John Doe" });

Avantatges de MongoDB

  • Flexibilitat: La naturalesa sense esquema de MongoDB permet un desenvolupament i iteració ràpids.
  • Escalabilitat: El sharding i la replicació integrats suporten l'escalabilitat horitzontal.
  • Rendiment: Optimitzat per a operacions de lectura i escriptura, fent-lo adequat per a aplicacions amb molt de trànsit.

Conclusió

MongoDB és una potent base de dades NoSQL que ofereix flexibilitat, escalabilitat i facilitat d'ús. El seu enfocament orientat a documents permet estructures de dades dinàmiques i complexes, fent-lo una excel·lent opció per a aplicacions modernes. Entenent les operacions bàsiques i els avantatges de MongoDB, pots aprofitar les seves característiques per construir aplicacions robustes i escalables.
