Introducció a la Canalització d'Agrupació

La Canalització d'Agrupació és un marc potent en MongoDB per a l'agrupació de dades, que et permet processar registres de dades i retornar resultats calculats. És similar al concepte de canalitzacions de processament de dades en sistemes Unix-like, on les dades passen a través d'una sèrie d'etapes, cadascuna realitzant una operació específica.

Conceptes Clau

  • Etapes: Cada etapa en la canalització realitza una operació sobre els documents d'entrada i passa el resultat a la següent etapa.
  • Operadors: Els operadors defineixen les operacions que s'han de realitzar sobre les dades dins de cada etapa.
  • Canalització: Una seqüència d'etapes a través de les quals flueixen les dades.

Etapes Bàsiques en la Canalització d'Agrupació

$match

Filtra els documents per passar només aquells que compleixen la condició especificada a la següent etapa.

db.orders.aggregate([
  { $match: { status: "A" } }
])

Exemple: Aquesta etapa filtra documents on el camp status és "A".

$group

Agrupa els documents d'entrada per una expressió d'identificador especificada i aplica les expressions d'acumulador a cada grup.

db.orders.aggregate([
  { $group: { _id: "$cust_id", total: { $sum: "$amount" } } }
])

Exemple: Aquesta etapa agrupa documents per cust_id i calcula el total de l'import per a cada client.

$project

Reestructura cada document en el flux, com ara afegint nous camps o eliminant camps existents.

db.orders.aggregate([
  { $project: { item: 1, total: { $multiply: ["$price", "$quantity"] } } }
])

Exemple: Aquesta etapa projecta el camp item i calcula un nou camp total multiplicant price i quantity.

$sort

Ordena tots els documents d'entrada i els retorna a la canalització en ordre ordenat.

db.orders.aggregate([
  { $sort: { total: -1 } }
])

Exemple: Aquesta etapa ordena els documents pel camp total en ordre descendent.

$limit

Limita el nombre de documents passats a la següent etapa.

db.orders.aggregate([
  { $limit: 5 }
])

Exemple: Aquesta etapa limita la sortida als primers 5 documents.

$skip

Salta els primers N documents i passa els documents restants a la següent etapa.

db.orders.aggregate([
  { $skip: 10 }
])

Exemple: Aquesta etapa salta els primers 10 documents.

Etapes i Operadors Avançats

$unwind

Desconstrueix un camp de matriu dels documents d'entrada per produir un document per a cada element.

db.orders.aggregate([
  { $unwind: "$items" }
])

Exemple: Aquesta etapa desconstrueix el camp de matriu items, produint un document per a cada element.

$lookup

Realitza una unió externa esquerra a una altra col·lecció en la mateixa base de dades per filtrar documents de la col·lecció "unida" per al processament.

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "cust_id",
      foreignField: "cust_id",
      as: "customer_info"
    }
  }
])

Exemple: Aquesta etapa uneix la col·lecció orders amb la col·lecció customers basant-se en el camp cust_id.

$facet

Processa múltiples canalitzacions d'agrupació dins d'una sola etapa en el mateix conjunt de documents d'entrada.

db.orders.aggregate([
  {
    $facet: {
      "categorizedByPrice": [
        { $match: { price: { $exists: 1 } } },
        { $bucket: { groupBy: "$price", boundaries: [0, 50, 100, 200], default: "Other" } }
      ],
      "categorizedByQuantity": [
        { $match: { quantity: { $exists: 1 } } },
        { $bucket: { groupBy: "$quantity", boundaries: [0, 20, 50, 100], default: "Other" } }
      ]
    }
  }
])

Exemple: Aquesta etapa categoritza documents per preu i quantitat utilitzant dues canalitzacions separades.

Exemples Pràctics

Exemple 1: Informe de Vendes

Genera un informe de vendes mostrant el total de l'import de les vendes per a cada categoria de producte.

db.sales.aggregate([
  { $match: { date: { $gte: ISODate("2023-01-01"), $lt: ISODate("2024-01-01") } } },
  { $group: { _id: "$category", totalSales: { $sum: "$amount" } } },
  { $sort: { totalSales: -1 } }
])

Explicació: Aquesta canalització filtra vendes dins de l'any 2023, les agrupa per categoria, calcula el total de l'import de les vendes per a cada categoria, i ordena els resultats en ordre descendent.

Exemple 2: Freqüència de Compres dels Clients

Troba el nombre de compres realitzades per cada client.

db.orders.aggregate([
  { $group: { _id: "$cust_id", purchaseCount: { $sum: 1 } } },
  { $sort: { purchaseCount: -1 } }
])

Explicació: Aquesta canalització agrupa comandes per ID de client, compta el nombre de compres per a cada client, i ordena els resultats en ordre descendent.

Conclusió

La Canalització d'Agrupació en MongoDB és una eina versàtil i potent per al processament i l'anàlisi de dades. Enllaçant múltiples etapes, pots realitzar transformacions i càlculs complexos de dades de manera eficient. Comprendre les etapes bàsiques i avançades, juntament amb exemples pràctics, t'ajudarà a aprofitar tot el potencial del marc d'agrupació de MongoDB en les teves aplicacions.
