Configuració de Clústers Fragmentats
Gestionant Clústers Fragmentats ►

El fragmentat és un mètode per distribuir dades a través de múltiples màquines. MongoDB utilitza el fragmentat per donar suport a desplegaments amb conjunts de dades molt grans i operacions d'alt rendiment. Aquesta secció et guiarà a través del procés de configuració de clústers fragmentats en MongoDB, des dels conceptes bàsics fins a configuracions avançades.

Introducció al Fragmentat

  • Definició: El fragmentat és el procés d'emmagatzemar registres de dades a través de múltiples màquines i és l'enfocament de MongoDB per satisfer les demandes de creixement de dades.
  • Propòsit: Ajuda en l'escalat horitzontal distribuint dades a través de múltiples servidors.
  • Components:
    • Fragments: Aquests són les unitats d'emmagatzematge de dades. Cada fragment conté un subconjunt de les dades fragmentades.
    • Servidors de Configuració: Aquests emmagatzemen metadades i configuracions per al clúster.
    • Enrutadors de Consultes (mongos): Aquests són la interfície entre les aplicacions clients i el clúster fragmentat.

Configuració d'un Clúster Fragmentat

Prerequisits

  • Instal·lació de MongoDB: Assegura't que MongoDB està instal·lat en tots els servidors que formaran part del clúster fragmentat.
  • Configuració de Xarxa: Tots els servidors han de poder comunicar-se entre ells.

Configuració Pas a Pas

  1. Inicia els Servidors de Configuració:

    • Els servidors de configuració emmagatzemen metadades sobre el clúster fragmentat.
    • Inicia tres instàncies de servidor de configuració per redundància i fiabilitat.
    mongod --configsvr --replSet configReplSet --dbpath /data/configdb --port 27019

  2. Inicia el Conjunt de Rèpliques del Servidor de Configuració:

    • Connecta't a un dels servidors de configuració i inicia el conjunt de rèpliques.
    mongo --port 27019
rs.initiate({
  _id: "configReplSet",
  configsvr: true,
  members: [
    { _id: 0, host: "configServer1:27019" },
    { _id: 1, host: "configServer2:27019" },
    { _id: 2, host: "configServer3:27019" }
  ]
})

  3. Inicia els Servidors de Fragment:

    • Cada fragment és un conjunt de rèpliques. Inicia les instàncies mongod per a cada fragment.
    mongod --shardsvr --replSet shardReplSet1 --dbpath /data/shard1 --port 27018

  4. Inicia el Conjunt de Rèpliques del Fragment:

    • Connecta't a un dels servidors de fragment i inicia el conjunt de rèpliques.
    mongo --port 27018
rs.initiate({
  _id: "shardReplSet1",
  members: [
    { _id: 0, host: "shardServer1:27018" },
    { _id: 1, host: "shardServer2:27018" },
    { _id: 2, host: "shardServer3:27018" }
  ]
})

  5. Inicia les Instàncies mongos:

    • Les instàncies mongos enrutaran les consultes al fragment adequat.
    mongos --configdb configReplSet/configServer1:27019,configServer2:27019,configServer3:27019 --port 27017

  6. Afegeix Fragments al Clúster:

    • Connecta't a la instància mongos i afegeix els fragments.
    mongo --port 27017
sh.addShard("shardReplSet1/shardServer1:27018,shardServer2:27018,shardServer3:27018")

Fragmentació d'una Col·lecció

  1. Habilita el Fragmentat per a una Base de Dades:

    sh.enableSharding("myDatabase")

  2. Fragmenta una Col·lecció:

    • Tria una clau de fragment que determini com es distribueixen les dades.
    sh.shardCollection("myDatabase.myCollection", { shardKey: 1 })

Millors Pràctiques

  • Selecció de la Clau de Fragment: Tria una clau de fragment que asseguri una distribució uniforme de les dades.
  • Monitorització: Monitoritza regularment el rendiment i la salut del clúster.
  • Còpia de Seguretat: Implementa una estratègia de còpia de seguretat robusta.

Conclusió

Configurar un clúster fragmentat en MongoDB implica configurar servidors de configuració, servidors de fragment i instàncies mongos. Una configuració i gestió adequades d'aquests components asseguren una distribució eficient de les dades i una alta disponibilitat. Seguint els passos descrits en aquesta guia, pots configurar un clúster fragmentat que escala horitzontalment i gestiona grans volums de dades de manera efectiva.
