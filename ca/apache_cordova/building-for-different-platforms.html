En aquesta secció, explorarem com construir i desplegar aplicacions Apache Cordova per a diferents plataformes. Apache Cordova et permet crear aplicacions mòbils utilitzant HTML, CSS i JavaScript, i després desplegar-les en múltiples plataformes com Android, iOS i Windows. Aquesta flexibilitat és un dels avantatges clau d'utilitzar Cordova.

Conceptes Clau

  • Desenvolupament Multiplataforma: Escriu el teu codi una vegada i desplega'l en múltiples plataformes.
  • Codi Específic de Plataforma: Gestiona els requisits i funcionalitats específiques de cada plataforma.
  • Eines de Construcció: Utilitza Cordova CLI i SDKs específics de plataforma per construir i desplegar aplicacions.

Configuració del Teu Entorn

Abans de començar a construir per a diferents plataformes, assegura't de tenir les eines necessàries instal·lades:

  • Node.js i npm: Necessaris per a Cordova CLI.
  • Cordova CLI: Instal·la'l utilitzant npm. 
    npm install -g cordova
  • SDKs de Plataforma: Instal·la els SDKs per a les plataformes que vols objectiu (per exemple, Android Studio per a Android, Xcode per a iOS).

Afegir Plataformes

Per construir la teva aplicació Cordova per a una plataforma específica, necessites afegir aquesta plataforma al teu projecte.

Android

  1. Afegir Plataforma Android:

    cordova platform add android

  2. Construir per a Android:

    cordova build android

  3. Executar en un Dispositiu o Emulador Android:

    cordova run android

iOS

  1. Afegir Plataforma iOS:

    cordova platform add ios

  2. Construir per a iOS:

    cordova build ios

  3. Executar en un Dispositiu o Simulador iOS:

    cordova run ios

Windows

  1. Afegir Plataforma Windows:

    cordova platform add windows

  2. Construir per a Windows:

    cordova build windows

  3. Executar en un Dispositiu o Emulador Windows:

    cordova run windows

Gestionar Codi Específic de Plataforma

De vegades, pot ser necessari escriure codi que sigui específic per a una plataforma particular. Cordova proporciona mecanismes per gestionar això.

Utilitzant `cordova.platformId`

Pots utilitzar la propietat cordova.platformId per determinar la plataforma actual i executar codi específic de plataforma.

if (cordova.platformId === 'android') {
    // Codi específic per a Android
} else if (cordova.platformId === 'ios') {
    // Codi específic per a iOS
} else if (cordova.platformId === 'windows') {
    // Codi específic per a Windows
}

Utilitzant Fitxers Específics de Plataforma

Cordova et permet crear versions específiques de fitxers per a cada plataforma. Per exemple, pots tenir index.android.html i index.ios.html per a Android i iOS, respectivament. Cordova utilitzarà automàticament el fitxer apropiat per a la plataforma.

Construir i Desplegar

Construir per a Publicació

Quan estiguis llest per publicar la teva aplicació, necessites construir-la en mode de publicació.

  • Android:

    cordova build android --release

  • iOS:

    cordova build ios --release

  • Windows:

    cordova build windows --release

Signar i Publicar

Cada plataforma té els seus propis requisits per signar i publicar aplicacions.

  • Android: Signa l'APK utilitzant jarsigner i alinea'l utilitzant zipalign.
  • iOS: Utilitza Xcode per arxivar i enviar la teva aplicació a l'App Store.
  • Windows: Utilitza Visual Studio per empaquetar i publicar la teva aplicació a la Microsoft Store.

Conclusió

Construir per a diferents plataformes amb Apache Cordova implica configurar el teu entorn, afegir les plataformes necessàries al teu projecte, gestionar codi específic de plataforma, i finalment construir i desplegar la teva aplicació. Seguint aquests passos, pots crear eficientment aplicacions mòbils multiplataforma utilitzant una única base de codi. Aquest enfocament no només estalvia temps de desenvolupament sinó que també assegura una experiència d'usuari consistent a través de diferents dispositius.
