El Projecte Sobre nosaltres Contribuir Donacions Llicència

La Integració Contínua (CI) és una pràctica de desenvolupament on els desenvolupadors integren codi en un repositori compartit freqüentment, preferiblement diverses vegades al dia. Cada integració es pot verificar amb una compilació automatitzada i proves automatitzades. Aquesta pràctica ajuda a detectar errors ràpidament i a millorar la qualitat del programari. En aquesta secció, explorarem com configurar la CI per a un projecte d'Apache Cordova.

Introducció a la Integració Contínua

  • Definició: La Integració Contínua és la pràctica de fusionar totes les còpies de treball dels desenvolupadors en una línia principal compartida diverses vegades al dia.
  • Beneficis:
    • Detecció precoç d'errors d'integració.
    • Reducció de problemes d'integració.
    • Retroalimentació contínua sobre la salut del codi.

Triar una Eina de CI

Hi ha diverses eines de CI disponibles, cadascuna amb els seus punts forts i febles. Algunes eines de CI populars inclouen:

  • Jenkins: Un servidor d'automatització de codi obert.
  • Travis CI: Un servei d'integració contínua allotjat utilitzat per construir i provar projectes de programari allotjats a GitHub.
  • CircleCI: Una eina de CI/CD que automatitza el procés de desenvolupament de programari utilitzant integració contínua i lliurament continu.
  • GitHub Actions: Una eina de CI/CD integrada directament a GitHub.

Per a aquest curs, utilitzarem GitHub Actions per la seva integració perfecta amb els repositoris de GitHub i la seva facilitat d'ús.

Configuració de GitHub Actions per a Apache Cordova

Pas 1: Crear un Repositori a GitHub

  1. Crea un nou repositori a GitHub per al teu projecte d'Apache Cordova.
  2. Clona el repositori a la teva màquina local i configura el teu projecte Cordova si encara no ho has fet.

Pas 2: Afegir un Workflow de GitHub Actions

  1. Crea un directori .github/workflows a l'arrel del teu repositori.
  2. Crea un nou fitxer en aquest directori anomenat ci.yml.

Pas 3: Definir el Workflow

Aquí tens un exemple de workflow simple de GitHub Actions per a un projecte d'Apache Cordova:

name: Cordova CI

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install Cordova
      run: npm install -g cordova

    - name: Install project dependencies
      run: npm install

    - name: Build Cordova project
      run: cordova build

Explicació del Workflow

  • name: El nom del workflow.
  • on: Especifica els esdeveniments que desencadenen el workflow. En aquest cas, el workflow s'executa en empentes i sol·licituds d'extracció a la branca main.
  • jobs: Defineix les tasques que s'executaran com a part del workflow.
    • build: El nom de la tasca.
    • runs-on: Especifica el tipus de màquina per executar la tasca. ubuntu-latest és una opció comunament utilitzada.
    • steps: La seqüència de passos per executar en la tasca.
      • Checkout code: Utilitza l'acció actions/checkout per comprovar el codi del repositori.
      • Set up Node.js: Utilitza l'acció actions/setup-node per configurar Node.js.
      • Install Cordova: Instal·la Cordova globalment utilitzant npm.
      • Install project dependencies: Instal·la les dependències npm del projecte.
      • Build Cordova project: Construeix el projecte Cordova.

Pas 4: Fer Commit i Empènyer

  1. Fes commit del fitxer ci.yml al teu repositori.
  2. Empeny les canvis a GitHub.

GitHub Actions detectarà automàticament el nou fitxer de workflow i començarà a executar el procés de CI basat en els desencadenants definits.

Conclusió

Configurar la Integració Contínua per a un projecte d'Apache Cordova utilitzant GitHub Actions és un procés senzill que pot millorar significativament el teu flux de treball de desenvolupament. Automatitzant el procés de construcció i prova, pots assegurar-te que el teu codi es manté saludable i que els problemes d'integració es detecten aviat. Aquesta configuració es pot ampliar amb passos addicionals, com ara executar proves o desplegar l'aplicació, per millorar encara més la teva pipeline de CI.
