El Projecte Sobre nosaltres Contribuir Donacions Llicència

Les notificacions push són una manera poderosa de mantenir els usuaris compromesos amb la teva aplicació mitjançant la lliurament d'informació oportuna i rellevant directament als seus dispositius. Aquest tema et guiarà a través del procés d'implementació de notificacions push en una aplicació Apache Cordova, des de nivells principiants fins a avançats.

Introducció a les Notificacions Push

Les notificacions push permeten que les aplicacions notifiquin els usuaris de nous missatges o esdeveniments fins i tot quan l'usuari no està utilitzant activament l'aplicació. Són una característica essencial per a molts tipus d'aplicacions, incloent-hi xarxes socials, notícies i aplicacions de comerç electrònic.

Conceptes Clau

  • Servei de Notificacions Push: Un servei que envia notificacions als dispositius. Exemples inclouen Firebase Cloud Messaging (FCM) per a Android i Apple Push Notification Service (APNs) per a iOS.
  • Token de Dispositiu: Un identificador únic assignat a cada dispositiu pel servei de notificacions push, utilitzat per enviar notificacions a aquest dispositiu específic.
  • Payload: Les dades enviades en la notificació, que poden incloure un missatge, títol i dades addicionals.

Configuració del Teu Entorn

Abans de poder implementar notificacions push, necessites configurar el teu entorn de desenvolupament.

Prerequisits

  • Node.js i npm: Assegura't de tenir instal·lats Node.js i npm.
  • Apache Cordova: Instal·la Cordova globalment utilitzant npm. 
    npm install -g cordova
  • Projecte Cordova: Crea un nou projecte Cordova o navega al teu projecte existent. 
    cordova create MyApp
cd MyApp

Afegint el Plugin Push

Per gestionar les notificacions push en Cordova, necessites afegir un plugin que proporcioni la funcionalitat necessària.

Instal·lant el Plugin

  • Utilitza el Cordova Push Plugin, que suporta tant FCM com APNs. 
    cordova plugin add phonegap-plugin-push

Configurant les Notificacions Push

Configuració d'Android

  1. Configuració de Firebase: Crea un projecte Firebase i afegeix la teva aplicació Android a aquest.
  2. Descarrega google-services.json: Col·loca aquest fitxer al directori platforms/android/app.
  3. Modifica config.xml: Afegeix les següents línies al teu fitxer config.xml. 
    <platform name="android">
    <resource-file src="google-services.json" target="app/google-services.json" />
</platform>

Configuració d'iOS

  1. Compte de Desenvolupador d'Apple: Assegura't de tenir un compte de desenvolupador d'Apple.
  2. Configuració d'APNs: Crea una clau APNs i descarrega el fitxer .p8.
  3. Modifica config.xml: Afegeix les següents línies al teu fitxer config.xml. 
    <platform name="ios">
    <config-file platform="ios" target="*-Info.plist" parent="UIApplicationExitsOnSuspend">
        <false/>
    </config-file>
</platform>

Implementant Notificacions Push

Inicialitzant el Plugin

  • Afegeix el següent codi al teu index.js o fitxer principal de JavaScript per inicialitzar el plugin push. 
    document.addEventListener('deviceready', function () {
    var push = PushNotification.init({
        android: {
            senderID: "YOUR_SENDER_ID"
        },
        ios: {
            alert: "true",
            badge: "true",
            sound: "true"
        }
    });

    push.on('registration', function (data) {
        console.log("Dispositiu registrat amb token:", data.registrationId);
        // Envia el ID de registre al teu servidor
    });

    push.on('notification', function (data) {
        console.log("Notificació rebuda:", data.message);
        // Gestiona la notificació
    });

    push.on('error', function (e) {
        console.error("Error de notificació push:", e.message);
    });
}, false);

Enviant Notificacions

  • Utilitza el teu servidor backend per enviar notificacions. Aquí tens un exemple utilitzant Node.js i FCM. 
    const admin = require('firebase-admin');
const serviceAccount = require('./path/to/serviceAccountKey.json');

admin.initializeApp({
    credential: admin.credential.cert(serviceAccount)
});

const registrationToken = 'DEVICE_REGISTRATION_TOKEN';
const message = {
    notification: {
        title: 'Hola',
        body: 'Món'
    },
    token: registrationToken
};

admin.messaging().send(message)
    .then((response) => {
        console.log('Missatge enviat amb èxit:', response);
    })
    .catch((error) => {
        console.error('Error enviant el missatge:', error);
    });

Temes Avançats

Gestionant Notificacions en Segon Pla

  • Modifica la teva aplicació Cordova per gestionar notificacions quan l'aplicació està en segon pla. 
    push.on('notification', function (data) {
    if (data.additionalData.foreground) {
        // L'aplicació està en primer pla
        console.log("Notificació en primer pla:", data.message);
    } else {
        // L'aplicació està en segon pla
        console.log("Notificació en segon pla:", data.message);
    }
});

Accions Personalitzades de Notificació

  • Afegeix accions personalitzades a les teves notificacions, com botons per a respostes ràpides. 
    const message = {
    notification: {
        title: 'Nou Missatge',
        body: 'Tens un nou missatge'
    },
    android: {
        notification: {
            clickAction: 'OPEN_ACTIVITY_1'
        }
    },
    token: registrationToken
};

Conclusió

Implementar notificacions push en una aplicació Apache Cordova implica configurar el teu entorn, configurar els serveis necessaris i utilitzar el Cordova Push Plugin per gestionar les notificacions. Seguint aquesta guia, pots assegurar-te que la teva aplicació pugui enviar i rebre notificacions push de manera efectiva, mantenint els teus usuaris compromesos i informats.
