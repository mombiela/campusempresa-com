Introducció

Apache Cordova permet als desenvolupadors crear aplicacions mòbils utilitzant HTML, CSS i JavaScript. No obstant això, hi ha moments en què necessites accedir a funcions específiques de la plataforma o optimitzar el rendiment integrant codi natiu. Aquesta secció et guiarà a través del procés d'integració de codi natiu en el teu projecte Cordova, des de nivells bàsics fins a avançats.

Per què integrar codi natiu?

  • Accés a funcions específiques de la plataforma: Algunes funcions no estan disponibles a través de les API de JavaScript de Cordova.
  • Optimització del rendiment: El codi natiu pot ser més eficient per a certes tasques.
  • Funcionalitat personalitzada: Implementa funcions úniques que no estan cobertes pels plugins existents.

Prerequisits

  • Comprensió bàsica de l'estructura del projecte Cordova.
  • Familiaritat amb JavaScript.
  • Coneixements bàsics de llenguatges de programació natius (Java per a Android, Swift/Objective-C per a iOS).

Configuració del teu entorn

Abans d'integrar codi natiu, assegura't que el teu entorn de desenvolupament està configurat correctament.

Android

  • Instal·la Android Studio.
  • Assegura't de tenir l'SDK d'Android i les eines de compilació necessàries.

iOS

  • Instal·la Xcode.
  • Assegura't de tenir l'últim SDK d'iOS.

Creació d'un Plugin Cordova

Els plugins de Cordova són el pont entre JavaScript i el codi natiu. Segueix aquests passos per crear un plugin senzill.

Pas 1: Crear l'estructura del Plugin

Utilitza el CLI de Cordova per crear un nou plugin.

cordova plugin create my-plugin com.example.myplugin MyPlugin

Aquesta comanda crea una estructura bàsica de plugin amb els següents fitxers:

  • plugin.xml: Fitxer de configuració.
  • src/ios/MyPlugin.m: Codi natiu per a iOS.
  • src/android/MyPlugin.java: Codi natiu per a Android.
  • www/MyPlugin.js: Interfície de JavaScript.

Pas 2: Implementar la interfície de JavaScript

Edita www/MyPlugin.js per definir la interfície de JavaScript.

var exec = require('cordova/exec');

exports.coolMethod = function(arg0, success, error) {
    exec(success, error, 'MyPlugin', 'coolMethod', [arg0]);
};

Pas 3: Implementar el codi natiu

Android

Edita src/android/MyPlugin.java.

package com.example.myplugin;

import org.apache.cordova.CordovaPlugin;
import org.apache.cordova.CallbackContext;
import org.json.JSONArray;
import org.json.JSONException;
import org.json.JSONObject;

public class MyPlugin extends CordovaPlugin {
    @Override
    public boolean execute(String action, JSONArray args, CallbackContext callbackContext) throws JSONException {
        if (action.equals("coolMethod")) {
            String message = args.getString(0);
            this.coolMethod(message, callbackContext);
            return true;
        }
        return false;
    }

    private void coolMethod(String message, CallbackContext callbackContext) {
        if (message != null && message.length() > 0) {
            callbackContext.success(message);
        } else {
            callbackContext.error("Expected one non-empty string argument.");
        }
    }
}

iOS

Edita src/ios/MyPlugin.m.

#import <Cordova/CDV.h>

@interface MyPlugin : CDVPlugin
- (void)coolMethod:(CDVInvokedUrlCommand*)command;
@end

@implementation MyPlugin

- (void)coolMethod:(CDVInvokedUrlCommand*)command {
    CDVPluginResult* pluginResult = nil;
    NSString* echo = [command.arguments objectAtIndex:0];

    if (echo != nil && [echo length] > 0) {
        pluginResult = [CDVPluginResult resultWithStatus:CDVCommandStatus_OK messageAsString:echo];
    } else {
        pluginResult = [CDVPluginResult resultWithStatus:CDVCommandStatus_ERROR messageAsString:@"Expected one non-empty string argument."];
    }

    [self.commandDelegate sendPluginResult:pluginResult callbackId:command.callbackId];
}

@end

Pas 4: Definir el Plugin en `plugin.xml`

Edita plugin.xml per definir les metadades del plugin i les configuracions específiques de la plataforma.

<plugin id="com.example.myplugin" version="1.0.0" xmlns="http://apache.org/cordova/ns/plugins/1.0" xmlns:android="http://schemas.android.com/apk/res/android">
    <name>MyPlugin</name>
    <js-module src="www/MyPlugin.js" name="MyPlugin">
        <clobbers target="cordova.plugins.MyPlugin" />
    </js-module>

    <platform name="android">
        <config-file target="res/xml/config.xml" parent="/*">
            <feature name="MyPlugin">
                <param name="android-package" value="com.example.myplugin.MyPlugin" />
            </feature>
        </config-file>
        <source-file src="src/android/MyPlugin.java" target-dir="src/com/example/myplugin" />
    </platform>

    <platform name="ios">
        <config-file target="config.xml" parent="/*">
            <feature name="MyPlugin">
                <param name="ios-package" value="MyPlugin" />
            </feature>
        </config-file>
        <source-file src="src/ios/MyPlugin.m" />
    </platform>
</plugin>

Ús del Plugin en la teva App Cordova

Després de crear el plugin, pots utilitzar-lo en la teva aplicació Cordova.

Pas 1: Instal·lar el Plugin

cordova plugin add path/to/my-plugin

Pas 2: Cridar el Plugin des de JavaScript

document.addEventListener('deviceready', function() {
    cordova.plugins.MyPlugin.coolMethod('Hello, World!', function(success) {
        console.log('Success: ' + success);
    }, function(error) {
        console.error('Error: ' + error);
    });
}, false);

Temes Avançats

Gestió d'Operacions Asíncrones

Per a operacions més complexes, com sol·licituds de xarxa o I/O de fitxers, potser necessitis gestionar tasques asíncrones en el teu codi natiu.

Exemple d'Android

private void coolMethod(String message, final CallbackContext callbackContext) {
    cordova.getThreadPool().execute(new Runnable() {
        public void run() {
            // Realitza la tasca de llarga durada aquí
            callbackContext.success("Task completed");
        }
    });
}

Exemple d'iOS

- (void)coolMethod:(CDVInvokedUrlCommand*)command {
    [self.commandDelegate runInBackground:^{
        // Realitza la tasca de llarga durada aquí
        CDVPluginResult* pluginResult = [CDVPluginResult resultWithStatus:CDVCommandStatus_OK messageAsString:@"Task completed"];
        [self.commandDelegate sendPluginResult:pluginResult callbackId:command.callbackId];
    }];
}

Gestió d'Errors

La gestió adequada d'errors és crucial per a un plugin robust.

Exemple d'Android

private void coolMethod(String message, CallbackContext callbackContext) {
    try {
        // El teu codi aquí
        callbackContext.success("Success message");
    } catch (Exception e) {
        callbackContext.error("Error message: " + e.getMessage());
    }
}

Exemple d'iOS

- (void)coolMethod:(CDVInvokedUrlCommand*)command {
    @try {
        // El teu codi aquí
        CDVPluginResult* pluginResult = [CDVPluginResult resultWithStatus:CDVCommandStatus_OK messageAsString:@"Success message"];
        [self.commandDelegate sendPluginResult:pluginResult callbackId:command.callbackId];
    } @catch (NSException *exception) {
        CDVPluginResult* pluginResult = [CDVPluginResult resultWithStatus:CDVCommandStatus_ERROR messageAsString:exception.reason];
        [self.commandDelegate sendPluginResult:pluginResult callbackId:command.callbackId];
    }
}

Conclusió

Integrar codi natiu en el teu projecte Cordova pot millorar significativament les capacitats i el rendiment de les teves aplicacions mòbils. Creant plugins personalitzats, pots accedir a funcions específiques de la plataforma, optimitzar el rendiment i implementar funcionalitats úniques. Aquesta guia ha proporcionat un enfocament pas a pas per crear i utilitzar plugins de Cordova, des de la configuració bàsica fins a la gestió avançada d'errors i operacions asíncrones. Amb aquestes habilitats, pots ampliar el poder de les teves aplicacions Cordova i oferir una experiència d'usuari més rica.
