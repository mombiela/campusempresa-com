La depuració de codi natiu en aplicacions Apache Cordova pot ser un repte, però és crucial per assegurar que la teva aplicació funcioni correctament en diferents plataformes. Aquest tema et guiarà a través del procés de depuració de codi natiu, des de nivells bàsics fins a avançats.

Introducció al Codi Natiu en Cordova

  • Codi Natiu: Codi escrit en el llenguatge natiu de la plataforma (Java per Android, Swift/Objective-C per iOS).
  • Plugins de Cordova: Et permeten escriure codi natiu que pot ser invocat des de JavaScript.

Configuració del Teu Entorn

Abans de poder depurar codi natiu, assegura't que el teu entorn de desenvolupament estigui correctament configurat.

  • Android:
    • Instal·la Android Studio.
    • Assegura't de tenir l'SDK d'Android i les eines de plataforma necessàries.
  • iOS:
    • Instal·la Xcode.
    • Assegura't de tenir els SDKs d'iOS necessaris.

Depuració de Codi Natiu en Android

Utilitzant Android Studio

  1. Obre el Teu Projecte:

    • Obre el teu projecte Cordova a Android Studio.
    • Navega al directori platforms/android.

  2. Configura Punts de Trencament:

    • Obre el fitxer Java que vols depurar.
    • Fes clic al marge esquerre per configurar punts de trencament.

  3. Executa en Mode Depuració:

    • Fes clic a la icona de 'Debug' o utilitza Shift + F9.
    • Selecciona el teu dispositiu connectat o emulador.

  4. Inspecciona Variables i Pila de Trucades:

    • Utilitza les finestres de 'Variables' i 'Call Stack' per inspeccionar l'estat de la teva aplicació.
// Exemple de codi Java amb un punt de trencament
public class ExamplePlugin extends CordovaPlugin {
    @Override
    public boolean execute(String action, JSONArray args, CallbackContext callbackContext) throws JSONException {
        if (action.equals("coolMethod")) {
            String message = args.getString(0);
            this.coolMethod(message, callbackContext);
            return true;
        }
        return false;
    }

    private void coolMethod(String message, CallbackContext callbackContext) {
        // Configura un punt de trencament aquí
        if (message != null && message.length() > 0) {
            callbackContext.success(message);
        } else {
            callbackContext.error("Expected one non-empty string argument.");
        }
    }
}

Utilitzant Logcat

  • Logcat: Una eina per veure missatges de registre des del teu dispositiu Android.
  • Afegint Declaracions de Registre:
    • Utilitza Log.d, Log.i, Log.w, i Log.e per imprimir missatges de depuració, informació, advertència i error.
// Exemple d'ús de Logcat
import android.util.Log;

public class ExamplePlugin extends CordovaPlugin {
    private static final String TAG = "ExamplePlugin";

    private void coolMethod(String message, CallbackContext callbackContext) {
        Log.d(TAG, "coolMethod called with message: " + message);
        if (message != null && message.length() > 0) {
            callbackContext.success(message);
        } else {
            callbackContext.error("Expected one non-empty string argument.");
        }
    }
}

Depuració de Codi Natiu en iOS

Utilitzant Xcode

  1. Obre el Teu Projecte:

    • Obre el teu projecte Cordova a Xcode.
    • Navega al directori platforms/ios i obre el fitxer .xcworkspace.

  2. Configura Punts de Trencament:

    • Obre el fitxer Swift/Objective-C que vols depurar.
    • Fes clic al marge esquerre per configurar punts de trencament.

  3. Executa en Mode Depuració:

    • Fes clic al botó 'Run' o utilitza Cmd + R.
    • Selecciona el teu dispositiu connectat o simulador.

  4. Inspecciona Variables i Pila de Trucades:

    • Utilitza les finestres de 'Variables' i 'Call Stack' per inspeccionar l'estat de la teva aplicació.
// Exemple de codi Swift amb un punt de trencament
@objc(ExamplePlugin) class ExamplePlugin: CDVPlugin {
    @objc(coolMethod:)
    func coolMethod(command: CDVInvokedUrlCommand) {
        let message = command.arguments[0] as? String ?? ""
        // Configura un punt de trencament aquí
        if message.count > 0 {
            let pluginResult = CDVPluginResult(status: CDVCommandStatus_OK, messageAs: message)
            self.commandDelegate.send(pluginResult, callbackId: command.callbackId)
        } else {
            let pluginResult = CDVPluginResult(status: CDVCommandStatus_ERROR, messageAs: "Expected one non-empty string argument.")
            self.commandDelegate.send(pluginResult, callbackId: command.callbackId)
        }
    }
}

Utilitzant NSLog

  • NSLog: Una funció per imprimir missatges de registre a la consola.
  • Afegint Declaracions de Registre:
    • Utilitza NSLog per imprimir missatges de depuració.
// Exemple d'ús de NSLog
@objc(ExamplePlugin) class ExamplePlugin: CDVPlugin {
    @objc(coolMethod:)
    func coolMethod(command: CDVInvokedUrlCommand) {
        let message = command.arguments[0] as? String ?? ""
        NSLog("coolMethod called with message: %@", message)
        if message.count > 0 {
            let pluginResult = CDVPluginResult(status: CDVCommandStatus_OK, messageAs: message)
            self.commandDelegate.send(pluginResult, callbackId: command.callbackId)
        } else {
            let pluginResult = CDVPluginResult(status: CDVCommandStatus_ERROR, messageAs: "Expected one non-empty string argument.")
            self.commandDelegate.send(pluginResult, callbackId: command.callbackId)
        }
    }
}

Tècniques Avançades de Depuració

  • Depuració Remota: Utilitza eines com Chrome DevTools per Android o Safari Web Inspector per iOS per depurar JavaScript i codi natiu.
  • Perfilat: Utilitza Android Studio Profiler o Xcode Instruments per analitzar problemes de rendiment.
  • Registres de Fallades: Analitza els registres de fallades per identificar i solucionar problemes que causen que la teva aplicació falli.

Conclusió

La depuració de codi natiu en Apache Cordova requereix una bona comprensió de l'entorn de desenvolupament i les eines disponibles per a cada plataforma. Configurant punts de trencament, utilitzant eines de registre i aprofitant tècniques avançades de depuració, pots identificar i resoldre efectivament problemes en el teu codi natiu. Això assegura una experiència més fluida i fiable per als usuaris de la teva aplicació.
