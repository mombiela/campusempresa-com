Introducció a la Depuració de Scripts DCL

La depuració és una habilitat essencial per a qualsevol programador, i és particularment important quan es treballa amb scripts DCL (Digital Command Language) a OpenVMS. Aquesta secció et guiarà a través del procés d'identificació i correcció d'errors en els teus scripts DCL, des de nivells principiants fins a avançats.

Tipus Comuns d'Errors en Scripts DCL

Entendre els tipus d'errors que podries trobar és el primer pas en la depuració. Aquí tens alguns errors comuns:

  • Errors de Sintaxi: Errors en la sintaxi de l'script.
  • Errors Lògics: Errors en la lògica que produeixen resultats incorrectes.
  • Errors d'Execució: Errors que es produeixen quan s'executa l'script.

Tècniques Bàsiques de Depuració

Ús de `SET VERIFY`

La comanda SET VERIFY és una eina senzilla però poderosa per a la depuració. Mostra cada línia de comanda abans que s'executi, ajudant-te a traçar el flux del teu script.

$ SET VERIFY
$! Les teves comandes de script aquí
$ SET NOVERIFY

Exemple

$ SET VERIFY
$ WRITE SYS$OUTPUT "Iniciant l'script"
$! Error de sintaxi intencionat
$ WRITE SYS$OUTPUT "Aquesta línia té un error
$ SET NOVERIFY

En aquest exemple, SET VERIFY t'ajudarà a identificar la cometa de tancament que falta en la segona comanda WRITE.

Ús de `SHOW SYMBOL`

La comanda SHOW SYMBOL mostra el valor actual d'un símbol (variable). Això és útil per comprovar els valors de les variables en diferents punts del teu script.

$ MYVAR = "Hola, Món!"
$ SHOW SYMBOL MYVAR

Exemple

$ MYVAR = "Hola, Món!"
$ SHOW SYMBOL MYVAR
$ MYVAR = "Adéu, Món!"
$ SHOW SYMBOL MYVAR

Això mostrarà el valor de MYVAR en dos punts diferents de l'script.

Tècniques Intermèdies de Depuració

Ús de `ON` Statements

La instrucció ON et permet especificar accions a prendre quan es produeixen certes condicions, com errors.

$ ON WARNING THEN GOTO ErrorHandler
$! Les teves comandes de script aquí
$ EXIT
$ErrorHandler:
$ WRITE SYS$OUTPUT "S'ha produït un avís."
$ EXIT

Exemple

$ ON WARNING THEN GOTO ErrorHandler
$ WRITE SYS$OUTPUT "Iniciant l'script"
$! Error d'execució intencionat
$ OPEN/READ FILE MYFILE "inexistent.txt"
$ EXIT
$ErrorHandler:
$ WRITE SYS$OUTPUT "S'ha produït un avís."
$ EXIT

En aquest exemple, si el fitxer inexistent.txt no existeix, l'script saltarà a l'etiqueta ErrorHandler.

Ús de `SET BREAK`

La comanda SET BREAK et permet establir punts de ruptura en el teu script, pausant l'execució en punts especificats.

$ SET BREAK
$! Les teves comandes de script aquí
$ SET NOBREAK

Exemple

$ SET BREAK
$ WRITE SYS$OUTPUT "Iniciant l'script"
$! Error lògic intencionat
$ MYVAR = 10
$ IF MYVAR .EQ. 20 THEN WRITE SYS$OUTPUT "MYVAR és 20"
$ SET NOBREAK

Això pausarà l'execució de l'script, permetent-te inspeccionar l'estat de les teves variables i la lògica.

Tècniques Avançades de Depuració

Ús de `LIB$SIGNAL` i `LIB$STOP`

Les rutines LIB$SIGNAL i LIB$STOP es poden utilitzar per generar i gestionar condicions i errors específics.

$ CALL LIB$SIGNAL(%X'00010001')
$ CALL LIB$STOP(%X'00010001')

Exemple

$ ON SEVERE_ERROR THEN GOTO SevereErrorHandler
$ CALL LIB$SIGNAL(%X'00010001')
$ EXIT
$SevereErrorHandler:
$ WRITE SYS$OUTPUT "S'ha produït un error greu."
$ EXIT

En aquest exemple, LIB$SIGNAL genera una condició que desencadena el SevereErrorHandler.

Ús d'Eines de Depuració

OpenVMS proporciona diverses eines de depuració que es poden utilitzar per a una depuració més avançada, com el Depurador d'OpenVMS.

  • Establiment de Punts de Ruptura: Et permet pausar l'execució en línies específiques.
  • Execució Pas a Pas: Executa el teu script línia per línia.
  • Inspecció de Variables: Inspecciona els valors de les variables en diferents punts.

Conclusió

La depuració de scripts DCL a OpenVMS implica una varietat de tècniques, des de comandes bàsiques com SET VERIFY i SHOW SYMBOL fins a mètodes més avançats com les instruccions ON i LIB$SIGNAL. En dominar aquestes tècniques, podràs identificar i corregir errors de manera eficient en els teus scripts, assegurant que funcionin de manera suau i correcta. Practica aquestes tècniques regularment per convertir-te en un expert en la depuració de scripts DCL.
