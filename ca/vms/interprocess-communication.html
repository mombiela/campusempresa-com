La Comunicació Interprocessos (IPC) és un concepte fonamental en els sistemes operatius, permetent que els processos intercanviïn dades i sincronitzin les seves accions. En OpenVMS, els mecanismes d'IPC són robustos i versàtils, proporcionant diversos mètodes perquè els processos es comuniquin de manera efectiva. Aquest tema cobrirà l'IPC des de nivells bàsics fins a avançats, assegurant una comprensió completa dels conceptes i les implementacions pràctiques.

Introducció a la Comunicació Interprocessos

  • Definició: L'IPC és un mecanisme que permet als processos comunicar-se entre ells i sincronitzar les seves accions.
  • Importància: L'IPC és crucial per a entorns de multitarea on els processos necessiten compartir dades i recursos de manera eficient.

Mecanismes d'IPC en OpenVMS

OpenVMS proporciona diversos mecanismes d'IPC, incloent:

  1. Busties
  2. Seccions Globals
  3. Flags d'Esdeveniments
  4. Noms Lògics
  5. Cues

Busties

Les busties són una manera simple i efectiva perquè els processos enviïn i rebin missatges.

  • Concepte: Una bustia és un buffer al qual els processos poden escriure i llegir.
  • Ús: Adequat per a missatges petits i notificacions.

Exemple: Creació i Ús d'una Bustia

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    unsigned short channel;
    char msg[] = "Hello, OpenVMS!";
    char buffer[20];
    unsigned short iosb[4];

    // Crear una bustia
    int status = sys$crembx(0, &channel, 0, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error creant bustia: %d\n", status);
        return 1;
    }

    // Escriure a la bustia
    status = sys$qiow(0, channel, IO$_WRITEVBLK, iosb, 0, 0, msg, sizeof(msg) - 1, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error escrivint a la bustia: %d\n", status);
        return 1;
    }

    // Llegir de la bustia
    status = sys$qiow(0, channel, IO$_READVBLK, iosb, 0, 0, buffer, sizeof(buffer) - 1, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error llegint de la bustia: %d\n", status);
        return 1;
    }

    buffer[iosb[1]] = '\0'; // Terminar la cadena amb nul
    printf("Missatge rebut: %s\n", buffer);

    // Desassignar el canal
    sys$dassgn(channel);

    return 0;
}

Seccions Globals

Les seccions globals permeten als processos compartir memòria.

  • Concepte: Una secció global és un segment de memòria compartida que múltiples processos poden mapar en el seu espai d'adreces.
  • Ús: Adequat per a compartir dades grans i estructures de dades complexes.

Exemple: Creació i Ús d'una Secció Global

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    struct dsc$descriptor_s name = { 0, DSC$K_DTYPE_T, DSC$K_CLASS_S, 0 };
    unsigned long gsd[2];
    unsigned long *shared_memory;
    unsigned long status;

    // Definir el nom de la secció global
    name.dsc$w_length = strlen("MY_GLOBAL_SECTION");
    name.dsc$a_pointer = "MY_GLOBAL_SECTION";

    // Crear la secció global
    status = sys$crmpsc(gsd, 0, 0, SEC$M_GBL, 0, 0, &name, 0, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error creant secció global: %d\n", status);
        return 1;
    }

    // Mapar la secció global en l'espai d'adreces del procés
    status = sys$mgblsc(gsd, &shared_memory, 0, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error mapant secció global: %d\n", status);
        return 1;
    }

    // Utilitzar la memòria compartida
    shared_memory[0] = 42;
    printf("Valor de la memòria compartida: %lu\n", shared_memory[0]);

    // Desmapar la secció global
    sys$deltva(&shared_memory, 0, 0);

    return 0;
}

Flags d'Esdeveniments

Els flags d'esdeveniments s'utilitzen per a la sincronització de processos.

  • Concepte: Un flag d'esdeveniment és un flag binari que els processos poden establir, esborrar i esperar.
  • Ús: Adequat per a senyalitzar entre processos.

Exemple: Ús de Flags d'Esdeveniments

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    unsigned long efn = 0; // Número del flag d'esdeveniment
    unsigned long status;

    // Establir el flag d'esdeveniment
    status = sys$setef(efn);
    if (status != SS$_NORMAL) {
        printf("Error establint flag d'esdeveniment: %d\n", status);
        return 1;
    }

    // Esperar el flag d'esdeveniment
    status = sys$waitfr(efn);
    if (status != SS$_NORMAL) {
        printf("Error esperant flag d'esdeveniment: %d\n", status);
        return 1;
    }

    printf("Flag d'esdeveniment senyalitzat.\n");

    return 0;
}

Noms Lògics

Els noms lògics s'utilitzen per definir noms simbòlics per a recursos del sistema.

  • Concepte: Un nom lògic és una cadena que es mapar a un recurs del sistema, com ara un fitxer o dispositiu.
  • Ús: Adequat per a l'abstracció de noms de recursos.

Exemple: Creació i Ús de Noms Lògics

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    struct dsc$descriptor_s logname = { 0, DSC$K_DTYPE_T, DSC$K_CLASS_S, 0 };
    struct dsc$descriptor_s logvalue = { 0, DSC$K_DTYPE_T, DSC$K_CLASS_S, 0 };
    unsigned long status;

    // Definir el nom lògic i el valor
    logname.dsc$w_length = strlen("MY_LOGICAL_NAME");
    logname.dsc$a_pointer = "MY_LOGICAL_NAME";
    logvalue.dsc$w_length = strlen("MY_LOGICAL_VALUE");
    logvalue.dsc$a_pointer = "MY_LOGICAL_VALUE";

    // Crear el nom lògic
    status = sys$crelnm(0, &logname, &logvalue, 0);
    if (status != SS$_NORMAL) {
        printf("Error creant nom lògic: %d\n", status);
        return 1;
    }

    // Traduir el nom lògic
    char buffer[256];
    struct dsc$descriptor_s result = { sizeof(buffer) - 1, DSC$K_DTYPE_T, DSC$K_CLASS_S, buffer };
    status = sys$trnlnm(0, &logname, &result, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error traduint nom lògic: %d\n", status);
        return 1;
    }

    buffer[result.dsc$w_length] = '\0'; // Terminar la cadena amb nul
    printf("Valor del nom lògic: %s\n", buffer);

    return 0;
}

Cues

Les cues s'utilitzen per a la transmissió de missatges entre processos.

  • Concepte: Una cua és una estructura de dades que conté missatges perquè els processos els llegeixin i escriguin.
  • Ús: Adequat per a la transmissió de missatges complexos i el control de treballs.

Exemple: Ús de Cues

#include <stdio.h>
#include <descrip.h>
#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    struct dsc$descriptor_s queue_name = { 0, DSC$K_DTYPE_T, DSC$K_CLASS_S, 0 };
    unsigned short iosb[4];
    char msg[] = "Hello, Queue!";
    char buffer[20];
    unsigned long status;

    // Definir el nom de la cua
    queue_name.dsc$w_length = strlen("MY_QUEUE");
    queue_name.dsc$a_pointer = "MY_QUEUE";

    // Crear la cua
    status = sys$crembx(0, &queue_name, 0, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error creant cua: %d\n", status);
        return 1;
    }

    // Escriure a la cua
    status = sys$qiow(0, queue_name, IO$_WRITEVBLK, iosb, 0, 0, msg, sizeof(msg) - 1, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error escrivint a la cua: %d\n", status);
        return 1;
    }

    // Llegir de la cua
    status = sys$qiow(0, queue_name, IO$_READVBLK, iosb, 0, 0, buffer, sizeof(buffer) - 1, 0, 0, 0, 0);
    if (status != SS$_NORMAL) {
        printf("Error llegint de la cua: %d\n", status);
        return 1;
    }

    buffer[iosb[1]] = '\0'; // Terminar la cadena amb nul
    printf("Missatge rebut: %s\n", buffer);

    return 0;
}

Conclusió

La Comunicació Interprocessos en OpenVMS és una característica poderosa que permet als processos comunicar-se i sincronitzar-se de manera efectiva. Comprenent i utilitzant els diversos mecanismes d'IPC com busties, seccions globals, flags d'esdeveniments, noms lògics i cues, els programadors poden desenvolupar aplicacions robustes i eficients. Cada mètode té els seus casos d'ús específics i avantatges, fent d'OpenVMS una plataforma versàtil per a la comunicació de processos.
