Visió General

OpenVMS (Open Virtual Memory System) és un sistema operatiu de memòria virtual multiusuari i multiprocés dissenyat per a l'ús en temps compartit, processament per lots i processament de transaccions. Aquest tema us introduirà als serveis del sistema proporcionats per OpenVMS, que són essencials per interactuar amb el sistema operatiu a un nivell baix.

Conceptes Clau

  • Serveis del Sistema: Són rutines proporcionades pel sistema operatiu OpenVMS que les aplicacions poden cridar per realitzar diverses tasques com el control de processos, operacions de fitxers i comunicació entre processos.
  • Cridar Serveis del Sistema: Els serveis del sistema es criden típicament utilitzant una sintaxi específica en llenguatges de programació com C o assemblador.

Serveis del Sistema Comuns

Aquí teniu alguns dels serveis del sistema més utilitzats a OpenVMS:

  • $CREPRC: Crear un procés.
  • $DELPRC: Eliminar un procés.
  • $GETJPI: Obtenir informació del treball/procés.
  • $QIO: Enviar una sol·licitud d'E/S a la cua.
  • $SNDJBC: Enviar un treball a una cua de lots.

Cridar Serveis del Sistema en C

Per cridar un servei del sistema en C, típicament s'inclouen els fitxers de capçalera apropiats i s'utilitza el nom de la funció del servei del sistema. Aquí teniu un exemple de com cridar el servei $GETJPI per obtenir informació sobre un procés.

Exemple: Obtenir Informació del Procés

#include <stdio.h>
#include <descrip.h>
#include <ssdef.h>
#include <jpidef.h>
#include <lib$routines.h>
#include <starlet.h>

int main() {
    unsigned long status;
    unsigned long pid = 0; // 0 significa procés actual
    unsigned long item_code = JPI$_PID; // Obtenir l'ID del procés
    unsigned long result;

    status = sys$getjpiw(0, &pid, 0, &item_code, &result, 0, 0, 0);

    if (status == SS$_NORMAL) {
        printf("ID del procés: %lu\n", result);
    } else {
        printf("Error en obtenir informació del procés\n");
    }

    return 0;
}
  • Fitxers de Capçalera: Incloure els fitxers de capçalera necessaris per als serveis del sistema.
  • Variables: Definir variables per a l'estat, l'ID del procés, el codi de l'element i el resultat.
  • sys$getjpiw: Cridar el servei del sistema $GETJPI.
  • Comprovació de l'Estat: Comprovar si la crida ha estat exitosa i imprimir l'ID del procés.

Utilitzar Serveis del Sistema en Assemblador

Per aquells que prefereixen o necessiten utilitzar el llenguatge d'assemblador, aquí teniu un exemple de com cridar el servei $CREPRC per crear un nou procés.

Exemple: Crear un Procés en Assemblador

        .TITLE  CREATE_PROCESS
        .IDENT  /V1.0/

        .MACRO  $CREPRC, PID, NAME, STATUS
        MOV     R0, PID
        MOV     R1, NAME
        MOV     R2, STATUS
        CALLS   #3, SYS$CREPRC
        .ENDM

        .ENTRY  START
        .PSECT  DATA, RW, NOEXE

PID:    .LONG   0
NAME:   .ASCID  /MYPROCESS/
STATUS: .LONG   0

        .PSECT  CODE, RO, EXEC

START:  $CREPRC PID, NAME, STATUS
        BR      EXIT

EXIT:   HALT

        .END    START
  • .MACRO: Definir una macro per al servei del sistema $CREPRC.
  • Instruccions MOV: Moure els paràmetres als registres.
  • CALLS: Cridar el servei del sistema.
  • .PSECT: Definir seccions per a dades i codi.
  • HALT: Finalitzar el programa.

Conclusió

Entendre i utilitzar els serveis del sistema OpenVMS és crucial per a una programació efectiva a nivell de sistema a la plataforma OpenVMS. Aquesta introducció ha cobert els conceptes bàsics sobre què són els serveis del sistema, com cridar-los en C i assemblador, i ha proporcionat exemples per il·lustrar el seu ús. A mesura que avanceu, trobareu serveis i escenaris més complexos, però el coneixement fonamental proporcionat aquí us servirà com a punt de partida sòlid.
