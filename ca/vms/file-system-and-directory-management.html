Introducció

OpenVMS és un sistema operatiu de memòria virtual multiusuari i multiprocés dissenyat per a l'ús en temps compartit, processament per lots i processament de transaccions. Entendre com gestionar fitxers i directoris és crucial per a una administració del sistema i operacions d'usuari efectives.

Conceptes clau

  • Estructura del sistema de fitxers: OpenVMS utilitza una estructura de sistema de fitxers jeràrquica.
  • Directoris: Els directoris s'utilitzen per organitzar fitxers.
  • Convencions de nomenclatura de fitxers: OpenVMS té regles específiques per nomenar fitxers.
  • Operacions de fitxers: Les operacions bàsiques inclouen crear, llegir, escriure i eliminar fitxers.
  • Operacions de directoris: Les operacions bàsiques inclouen crear, llistar i eliminar directoris.

Estructura del sistema de fitxers

OpenVMS utilitza un sistema de fitxers jeràrquic on els fitxers s'organitzen en directoris. El directori arrel és el directori de nivell superior, i tots els altres directoris es ramifiquen des d'ell.

Directoris

Els directoris a OpenVMS s'utilitzen per organitzar fitxers de manera lògica. Cada directori pot contenir fitxers i subdirectoris.

Crear un directori

Per crear un directori, utilitza la comanda CREATE/DIRECTORY:

$ CREATE/DIRECTORY [MYDIR]

Aquesta comanda crea un directori anomenat MYDIR.

Llistar el contingut del directori

Per llistar el contingut d'un directori, utilitza la comanda DIRECTORY:

$ DIRECTORY [MYDIR]

Aquesta comanda llista tots els fitxers i subdirectoris dins de MYDIR.

Eliminar un directori

Per eliminar un directori, utilitza la comanda DELETE:

$ DELETE [MYDIR]

Nota: El directori ha d'estar buit abans de poder ser eliminat.

Convencions de nomenclatura de fitxers

Els noms de fitxers a OpenVMS tenen el següent format:

filename.extension;version
  • Nom del fitxer: Fins a 39 caràcters.
  • Extensió: Fins a 39 caràcters.
  • Versió: Un número que permet múltiples versions del mateix fitxer.

Exemple

$ CREATE MYFILE.TXT;1

Aquesta comanda crea un fitxer anomenat MYFILE.TXT amb la versió 1.

Operacions de fitxers

Crear un fitxer

Per crear un fitxer, utilitza la comanda CREATE:

$ CREATE MYFILE.TXT

Aquesta comanda obre un editor de text per crear MYFILE.TXT.

Llegir un fitxer

Per llegir un fitxer, utilitza la comanda TYPE:

$ TYPE MYFILE.TXT

Aquesta comanda mostra el contingut de MYFILE.TXT.

Escriure a un fitxer

Per escriure a un fitxer, utilitza la comanda EDIT:

$ EDIT MYFILE.TXT

Aquesta comanda obre un editor de text per modificar MYFILE.TXT.

Eliminar un fitxer

Per eliminar un fitxer, utilitza la comanda DELETE:

$ DELETE MYFILE.TXT;1

Aquesta comanda elimina la versió 1 de MYFILE.TXT.

Operacions de directoris

Crear un subdirectori

Per crear un subdirectori dins d'un directori existent:

$ CREATE/DIRECTORY [MYDIR.SUBDIR]

Aquesta comanda crea un subdirectori anomenat SUBDIR dins de MYDIR.

Navegar pels directoris

Per canviar el directori actual, utilitza la comanda SET DEFAULT:

$ SET DEFAULT [MYDIR]

Aquesta comanda estableix MYDIR com el directori de treball actual.

Llistar tots els directoris

Per llistar tots els directoris des de l'arrel:

$ DIRECTORY [000000...]

Aquesta comanda llista tots els directoris començant des de l'arrel.

Conclusió

Entendre la gestió del sistema de fitxers i directoris a OpenVMS és essencial per a un ús eficient del sistema. En dominar aquestes comandes i conceptes bàsics, els usuaris poden organitzar i gestionar efectivament els seus fitxers i directoris. Practica aquestes comandes per convertir-te en un expert en la navegació i manipulació del sistema de fitxers d'OpenVMS.
