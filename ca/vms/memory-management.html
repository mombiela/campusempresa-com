Conceptes de Xarxes en OpenVMS ►

La gestió de memòria és un aspecte crític de qualsevol sistema operatiu, i OpenVMS no és una excepció. Aquest tema et guiarà a través dels fonaments de la gestió de memòria en OpenVMS, progressant des de conceptes bàsics fins a avançats.

Introducció a la Gestió de Memòria

La gestió de memòria en OpenVMS implica l'assignació, desassignació i organització dels recursos de memòria. Entendre aquests conceptes és crucial per al desenvolupament eficient d'aplicacions i el rendiment del sistema.

Conceptes Clau

  • Memòria Virtual: OpenVMS utilitza memòria virtual per proporcionar a cada procés el seu propi espai d'adreces.
  • Memòria Física: La RAM real instal·lada en el sistema.
  • Paginació: El procés de moure dades entre la memòria física i l'emmagatzematge en disc.
  • Intercanvi: Mou tot el procés entre la memòria física i l'emmagatzematge en disc.

Memòria Virtual

La memòria virtual permet a OpenVMS utilitzar espai en disc com una extensió de la RAM, permetent al sistema executar aplicacions més grans del que permetria la memòria física.

Espai d'Adreces

  • Espai d'Adreces del Procés: Cada procés en OpenVMS té el seu propi espai d'adreces virtual.
  • Espai d'Adreces del Sistema: Compartit per tots els processos, utilitzat per recursos de tot el sistema.

Exemple: Assignació de Memòria Virtual

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    ptr = (int *)malloc(100 * sizeof(int)); // Assignar memòria per a 100 enters
    if (ptr == NULL) {
        printf("L'assignació de memòria ha fallat\n");
        return 1;
    }
    // Utilitzar la memòria assignada
    for (int i = 0; i < 100; i++) {
        ptr[i] = i;
    }
    // Alliberar la memòria assignada
    free(ptr);
    return 0;
}
  • Explicació: La funció malloc assigna memòria en l'espai d'adreces virtual del procés. La funció free la desassigna.

Memòria Física

La memòria física es refereix a la RAM real instal·lada en el sistema. OpenVMS gestiona la memòria física per optimitzar el rendiment i assegurar l'estabilitat.

Pàgines de Memòria

  • Mida de la Pàgina: Normalment 512 bytes o 8192 bytes, depenent de l'arquitectura del sistema.
  • Marcs de Pàgina: Blocs de mida fixa de memòria física.

Exemple: Veure l'Ús de la Memòria Física

$ SHOW MEMORY /PHYSICAL
  • Explicació: Aquesta comanda mostra l'ús actual de la memòria física en el sistema.

Paginació i Intercanvi

La paginació i l'intercanvi són tècniques utilitzades per gestionar la memòria de manera eficient movent dades entre la memòria física i l'emmagatzematge en disc.

Paginació

  • Error de Pàgina: Es produeix quan un procés intenta accedir a una pàgina que no està actualment en la memòria física.
  • Reemplaçament de Pàgina: El procés de moure una pàgina fora de la memòria física al disc per fer espai per a una nova pàgina.

Intercanvi

  • Intercanvi de Procés: Mou tot el procés entre la memòria física i l'emmagatzematge en disc per alliberar memòria.

Exemple: Configuració de Fitxers de Paginació

$ CREATE /PAGEFILE SYS$SYSTEM:PAGEFILE.SYS /SIZE=10000
$ INSTALL /PAGEFILE SYS$SYSTEM:PAGEFILE.SYS
  • Explicació: Aquestes comandes creen i instal·len un fitxer de paginació, que s'utilitza per emmagatzemar pàgines intercanviades fora de la memòria física.

Gestió Avançada de Memòria

Les tècniques avançades de gestió de memòria en OpenVMS inclouen l'ajust dels paràmetres del sistema i l'ús de serveis específics del sistema per a un rendiment òptim.

Paràmetres del Sistema

  • VIRTUALPAGECNT: Nombre màxim de pàgines virtuals que un procés pot utilitzar.
  • WSDEFAULT: Mida predeterminada del conjunt de treball per als processos.

Exemple: Ajust dels Paràmetres del Sistema

$ SET PARAMETER VIRTUALPAGECNT 50000
$ SET PARAMETER WSDEFAULT 1024
  • Explicació: Aquestes comandes estableixen el nombre màxim de pàgines virtuals i la mida predeterminada del conjunt de treball per als processos.

Serveis del Sistema

  • SYS$CRMPSC: Crear i mapar una secció de memòria.
  • SYS$DELTVA: Eliminar un rang d'adreces virtuals.

Exemple: Ús de SYS$CRMPSC

#include <lib$routines.h>
#include <ssdef.h>
#include <starlet.h>

int main() {
    unsigned long status;
    unsigned long start_va = 0;
    unsigned long end_va = 0;
    unsigned long length = 8192; // 8KB

    status = sys$crmpsc(&start_va, &end_va, 0, 0, 0, 0, 0, 0, 0, &length, 0, 0);
    if (status != SS$_NORMAL) {
        printf("El mapeig de memòria ha fallat\n");
        return 1;
    }
    // Utilitzar la memòria mapejada
    // ...

    // Desmapar la memòria
    sys$deltva(&start_va, &end_va, 0);
    return 0;
}
  • Explicació: El servei sys$crmpsc mapar una secció de memòria, i sys$deltva la desmapa.

Conclusió

La gestió de memòria en OpenVMS és un tema complex però essencial. Entenent la memòria virtual i física, la paginació, l'intercanvi i les tècniques avançades, pots optimitzar les teves aplicacions i assegurar un ús eficient dels recursos del sistema. Practica aquests conceptes amb els exemples i comandes proporcionats per obtenir una comprensió més profunda i habilitat en la gestió de memòria en OpenVMS.
