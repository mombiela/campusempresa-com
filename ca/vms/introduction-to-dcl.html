Visió General

Digital Command Language (DCL) és el llenguatge de comandes utilitzat en el sistema operatiu OpenVMS. És un llenguatge de script potent que permet als usuaris interactuar amb el sistema, automatitzar tasques i gestionar els recursos del sistema de manera eficient.

Conceptes Clau

  • Estructura de Comandes: Les comandes DCL normalment segueixen una estructura verb-nom, fent-les intuïtives i fàcils de recordar.
  • Símbols i Variables: DCL utilitza símbols (variables) per emmagatzemar dades que es poden reutilitzar al llarg dels scripts.
  • Noms Lògics: Els noms lògics són àlies per a fitxers, dispositius o altres recursos del sistema.
  • Procediments de Comandes: Aquests són scripts escrits en DCL per automatitzar tasques.

Comandes Bàsiques

Aquí tens algunes comandes bàsiques de DCL per començar:

  • SHOW: Mostra informació sobre el sistema o l'entorn de l'usuari.

    $ SHOW TIME

    Aquesta comanda mostra l'hora actual del sistema.

  • DIR: Llista els fitxers en un directori.

    $ DIR

    Aquesta comanda llista tots els fitxers en el directori actual.

  • TYPE: Mostra el contingut d'un fitxer.

    $ TYPE filename.txt

    Aquesta comanda mostra el contingut de filename.txt.

Símbols i Variables

Els símbols en DCL s'utilitzen per emmagatzemar valors que es poden reutilitzar. Són similars a les variables en altres llenguatges de programació.

  • Definint un Símbol:

    $ MY_SYMBOL = "Hello, OpenVMS!"

    Aquesta comanda defineix un símbol MY_SYMBOL amb el valor "Hello, OpenVMS!".

  • Utilitzant un Símbol:

    $ SHOW SYMBOL MY_SYMBOL

    Aquesta comanda mostra el valor de MY_SYMBOL.

Noms Lògics

Els noms lògics s'utilitzen per crear àlies per als recursos del sistema.

  • Definint un Nom Lògic:

    $ DEFINE MY_DISK DKA0:

    Aquesta comanda defineix un nom lògic MY_DISK per al disc DKA0:.

  • Utilitzant un Nom Lògic:

    $ DIR MY_DISK:[000000]

    Aquesta comanda llista els fitxers en el directori arrel de DKA0: utilitzant el nom lògic MY_DISK.

Procediments de Comandes

Els procediments de comandes són scripts escrits en DCL per automatitzar tasques. Es guarden amb una extensió .COM.

  • Creant un Procediment de Comandes:

    $! This is a comment
$ WRITE SYS$OUTPUT "Hello, OpenVMS!"

    Guarda l'script anterior com hello.com.

  • Executant un Procediment de Comandes:

    $ @hello.com

    Aquesta comanda executa l'script hello.com.

Exemple: Automatitzant una Còpia de Seguretat

Aquí tens un exemple d'un procediment de comandes senzill per automatitzar un procés de còpia de seguretat:

$! backup.com - A simple backup script
$ SET NOON
$ BACKUP DKA0:[USER]*.* DKA1:[BACKUP]/SAVE
$ IF $STATUS .EQS. %X00000001 THEN -
  WRITE SYS$OUTPUT "Backup completed successfully."
$ ELSE -
  WRITE SYS$OUTPUT "Backup failed."
  • Explicació:
    • $ SET NOON: Desactiva la gestió d'errors per evitar que l'script s'aturi en cas d'errors.
    • $ BACKUP DKA0:[USER]*.* DKA1:[BACKUP]/SAVE: Fa una còpia de seguretat de tots els fitxers de DKA0:[USER] a DKA1:[BACKUP].
    • $ IF $STATUS .EQS. %X00000001 THEN -: Comprova si la còpia de seguretat ha estat exitosa.
    • WRITE SYS$OUTPUT "Backup completed successfully.": Imprimeix un missatge d'èxit.
    • WRITE SYS$OUTPUT "Backup failed.": Imprimeix un missatge de fallada si la còpia de seguretat no ha estat exitosa.

Conclusió

DCL és un llenguatge de comandes versàtil i potent que és essencial per gestionar i automatitzar tasques en l'entorn OpenVMS. Comprenent les comandes bàsiques, els símbols, els noms lògics i els procediments de comandes, pots començar a aprofitar DCL per millorar la teva productivitat i capacitats de gestió del sistema. A mesura que progressis, descobriràs funcions i tècniques més avançades que ampliaran encara més la teva capacitat per automatitzar i controlar el teu sistema OpenVMS.
