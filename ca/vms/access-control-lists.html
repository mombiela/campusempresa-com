Les Llistes de Control d'Accés (ACLs) són un aspecte fonamental de la seguretat a OpenVMS, proporcionant un control detallat sobre l'accés als recursos del sistema. Aquest tema et guiarà des dels conceptes bàsics fins a l'ús avançat de les ACLs a OpenVMS.

Introducció a les ACLs

  • Definició: Les ACLs són llistes de permisos adjuntes a un objecte (com un fitxer o directori) que especifiquen quins usuaris o processos del sistema poden accedir a l'objecte i quines operacions poden realitzar.
  • Propòsit: Millorar la seguretat permetent un control d'accés detallat més enllà dels permisos estàndard dels fitxers.

Conceptes Bàsics

Comprensió dels Components de les ACLs

  • Entrades: Cada ACL consta de múltiples entrades, on cada entrada especifica un usuari o grup i els permisos concedits o denegats.
  • Permisos: Els permisos comuns inclouen llegir, escriure, executar, eliminar i controlar l'accés.

ACLs Per Defecte vs. ACLs Explícites

  • ACLs Per Defecte: Aplicades automàticament a nous objectes creats dins d'un directori.
  • ACLs Explícites: Específicament establertes en objectes individuals.

Configuració de les ACLs

Visualització de les ACLs

Per veure les ACLs d'un fitxer o directori, utilitza la comanda SHOW SECURITY:

$ SHOW SECURITY file.txt

Afegir Entrades d'ACL

Per afegir una entrada d'ACL, utilitza la comanda SET SECURITY:

$ SET SECURITY/ACL=(IDENTIFIER=USERNAME,ACCESS=READ+WRITE) file.txt
  • IDENTIFIER: Especifica l'usuari o grup.
  • ACCESS: Especifica els permisos.

Eliminar Entrades d'ACL

Per eliminar una entrada d'ACL, utilitza la comanda SET SECURITY/ACL=REMOVE:

$ SET SECURITY/ACL=(IDENTIFIER=USERNAME,ACCESS=NONE) file.txt

Gestió Avançada d'ACLs

Combinació d'ACLs amb Permisos Estàndard

  • Permisos Estàndard: Permisos bàsics com propietari, grup i món.
  • Precedència de les ACLs: Les ACLs tenen precedència sobre els permisos estàndard, proporcionant un control més específic.

Herència de les ACLs

  • Herència: Les ACLs poden ser heretades dels directoris pares, assegurant polítiques de seguretat consistents.
  • Establiment de l'Herència: Utilitza l'atribut DEFAULT_PROTECTION per establir ACLs heretades.

Exemple: ACLs Heretades

$ SET SECURITY/ACL=(IDENTIFIER=GROUP1,ACCESS=READ+WRITE,DEFAULT_PROTECTION) directory

Ús de Comodins amb les ACLs

  • Comodins: Aplica ACLs a múltiples fitxers o directoris utilitzant comodins.
  • Exemple:
$ SET SECURITY/ACL=(IDENTIFIER=USERNAME,ACCESS=READ+WRITE) *.txt

Resolució de Problemes amb les ACLs

Problemes Comuns

  • Permís Denegat: Assegura't que l'usuari o grup correcte està especificat i que els permisos necessaris estan concedits.
  • ACLs Conflictius: Revisa les entrades d'ACL superposades o conflictuoses que podrien causar un comportament inesperat.

Consells de Depuració

  • Sortida Verbosa: Utilitza opcions verboses amb les comandes per obtenir informació detallada.
  • Registres d'Auditoria: Revisa els registres d'auditoria del sistema per a intents d'accés i fallades.

Conclusió

Les Llistes de Control d'Accés (ACLs) a OpenVMS proporcionen un mecanisme potent per gestionar l'accés als recursos del sistema. Comprenent i utilitzant les ACLs, pots millorar la seguretat del teu entorn OpenVMS, assegurant que només els usuaris i processos autoritzats poden accedir a dades sensibles i realitzar operacions crítiques. Practica establint, modificant i resolent problemes amb les ACLs per convertir-te en un expert en la gestió del control d'accés a OpenVMS.
