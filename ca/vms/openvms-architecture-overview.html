Introducció

OpenVMS (Open Virtual Memory System) és un sistema operatiu multiusuari, multiprocés basat en memòria virtual dissenyat per a l'ús en temps compartit, processament per lots i processament de transaccions. Aquesta secció proporciona una visió general de l'arquitectura d'OpenVMS, cobrint els seus components clau i com interactuen.

Components clau de l'arquitectura d'OpenVMS

1. Nucli

El nucli és el component central d'OpenVMS, responsable de gestionar els recursos del sistema i el maquinari. Inclou:

  • Gestió de processos: Gestiona la creació, planificació i terminació de processos.
  • Gestió de memòria: Gestiona la memòria física i virtual, incloent paginació i intercanvi.
  • Gestió d'E/S: Gestiona les operacions d'entrada/sortida i els controladors de dispositius.
  • Comunicació entre processos (IPC): Facilita la comunicació entre processos.

2. Executiu

La capa executiva proporciona serveis de nivell superior construïts sobre el nucli. Inclou:

  • Sistema de fitxers: Gestiona fitxers i directoris, proporcionant control d'accés i operacions de fitxers.
  • Xarxes: Suporta protocols de xarxa i comunicació.
  • Seguretat: Implementa polítiques de seguretat, autenticació d'usuaris i control d'accés.

3. Serveis del sistema

Els serveis del sistema són API proporcionades per OpenVMS per al desenvolupament d'aplicacions. Inclouen:

  • Rutines del sistema: Funcions per al control de processos, operacions de fitxers i gestió del sistema.
  • Biblioteques d'execució (RTLs): Biblioteques que proporcionen funcionalitats addicionals per a aplicacions.

4. Mode d'usuari

El mode d'usuari és on s'executen les aplicacions i utilitats d'usuari. Inclou:

  • Interfície de línia de comandes (CLI): El llenguatge de comandes digital (DCL) per interactuar amb el sistema.
  • Interfície gràfica d'usuari (GUI): DECwindows per a la interacció gràfica.
  • Aplicacions d'usuari: Aplicacions personalitzades i de tercers que s'executen a OpenVMS.

Gestió de processos

Estats del procés

Els processos d'OpenVMS poden estar en diversos estats:

  • Executant: El procés s'està executant actualment.
  • Esperant: El procés està esperant un esdeveniment o recurs.
  • Suspès: El procés està en pausa i es pot reprendre més tard.
  • Terminat: El procés ha completat l'execució.

Bloc de control de processos (PCB)

El PCB és una estructura de dades que emmagatzema informació sobre un procés, incloent:

  • Identificador de procés (PID)
  • Estat del procés
  • Registres de la CPU
  • Informació de gestió de memòria

Exemple: Creació d'un procés

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int main() {
    pid_t pid = fork();
    if (pid == 0) {
        // Procés fill
        printf("Procés fill\n");
        execlp("/bin/ls", "ls", NULL);
    } else if (pid > 0) {
        // Procés pare
        printf("Procés pare\n");
        wait(NULL);
    } else {
        // Error en el fork
        printf("Error en el fork\n");
        return 1;
    }
    return 0;
}

En aquest exemple, la crida al sistema fork() crea un nou procés. El procés fill executa la comanda ls, mentre que el procés pare espera que el fill acabi.

Gestió de memòria

Memòria virtual

OpenVMS utilitza memòria virtual per proporcionar a cada procés el seu propi espai d'adreces. Els conceptes clau inclouen:

  • Paginació: Dividir la memòria en pàgines de mida fixa.
  • Intercanvi: Moure processos complets entre la memòria física i el disc.

Exemple: Assignació de memòria

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr = (int *)malloc(sizeof(int) * 10);
    if (ptr == NULL) {
        printf("Assignació de memòria fallida\n");
        return 1;
    }
    for (int i = 0; i < 10; i++) {
        ptr[i] = i;
    }
    for (int i = 0; i < 10; i++) {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}

Aquest exemple demostra l'assignació dinàmica de memòria utilitzant malloc i free.

Gestió d'E/S

Controladors de dispositius

Els controladors de dispositius a OpenVMS gestionen la comunicació entre el sistema operatiu i els dispositius de maquinari. Gestionen:

  • Inicialització: Configuració del dispositiu.
  • Transferència de dades: Lectura i escriptura al dispositiu.
  • Gestió d'interrupcions: Resposta a interrupcions de maquinari.

Exemple: Lectura d'un fitxer

#include <stdio.h>

int main() {
    FILE *file = fopen("example.txt", "r");
    if (file == NULL) {
        printf("Error en obrir el fitxer\n");
        return 1;
    }
    char buffer[100];
    while (fgets(buffer, sizeof(buffer), file) != NULL) {
        printf("%s", buffer);
    }
    fclose(file);
    return 0;
}

Aquest exemple mostra com llegir dades d'un fitxer utilitzant funcions d'E/S estàndard.

Conclusió

L'arquitectura d'OpenVMS està dissenyada per proporcionar entorns de computació robustos, escalables i segurs. Comprendre els seus components clau—nucli, executiu, serveis del sistema i mode d'usuari—ajuda a utilitzar i desenvolupar aplicacions per a OpenVMS de manera efectiva. Dominant la gestió de processos, la gestió de memòria i la gestió d'E/S, els programadors poden aprofitar tot el potencial d'OpenVMS en els seus projectes.
