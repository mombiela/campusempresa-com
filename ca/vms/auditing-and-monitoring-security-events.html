Introducció

L'auditoria i el monitoratge d'esdeveniments de seguretat són components crítics per mantenir un entorn OpenVMS segur. Aquest tema et guiarà a través dels conceptes bàsics per configurar i gestionar auditories de seguretat, entendre els registres d'auditoria i utilitzar eines per monitoritzar esdeveniments de seguretat des de nivells bàsics fins a avançats.

Conceptes Clau

  • Auditoria: El procés de registrar i examinar activitats per assegurar el compliment de les polítiques de seguretat.
  • Monitoratge: L'observació contínua de les activitats del sistema per detectar i respondre a incidents de seguretat.
  • Esdeveniments de Seguretat: Accions o esdeveniments que poden afectar la seguretat del sistema, com intents d'inici de sessió, accessos a fitxers i canvis al sistema.

Configuració de l'Auditoria

Activació de l'Auditoria

Per activar l'auditoria a OpenVMS, cal configurar el Servidor d'Auditoria de Seguretat. Això implica configurar el servidor d'auditoria i definir quins esdeveniments auditar.

$ SET AUDIT /ENABLE
$ SET AUDIT /SERVER=START

Definició d'Esdeveniments d'Auditoria

Especifica quins esdeveniments vols auditar. Els esdeveniments comuns inclouen intents d'inici de sessió, accessos a fitxers i canvis al sistema.

$ SET AUDIT /ALARM=(LOGIN=ALL)
$ SET AUDIT /AUDIT=(LOGIN=ALL)
$ SET AUDIT /ALARM=(ACCESS=ALL)
$ SET AUDIT /AUDIT=(ACCESS=ALL)

Configuració de Destinacions d'Auditoria

Els registres d'auditoria es poden enviar a diverses destinacions, com un fitxer o la consola de l'operador.

$ SET AUDIT /DESTINATION=FILE=SYS$MANAGER:SECURITY_AUDIT.DAT
$ SET AUDIT /DESTINATION=OPERATOR

Entenent els Registres d'Auditoria

Visualització dels Registres d'Auditoria

Els registres d'auditoria contenen registres de tots els esdeveniments auditats. Utilitza el comandament ANALYZE/AUDIT per veure i analitzar aquests registres.

$ ANALYZE/AUDIT /FULL SYS$MANAGER:SECURITY_AUDIT.DAT

Interpretació dels Registres d'Auditoria

Els registres d'auditoria proporcionen informació detallada sobre cada esdeveniment, incloent-hi l'hora, l'usuari i l'acció realitzada.

Exemple d'un registre d'auditoria:

Security Audit Record
----------------------
Event: LOGIN
Time: 10-OCT-2023 10:00:00.00
User: SYSTEM
Node: NODE1
Status: SUCCESSFUL

Monitoratge d'Esdeveniments de Seguretat

Monitoratge en Temps Real

Per al monitoratge en temps real, pots utilitzar la utilitat MONITOR per observar les activitats del sistema a mesura que succeeixen.

$ MONITOR SYSTEM

Ús d'Alarmes de Seguretat

Les alarmes de seguretat notifiquen als administradors sobre esdeveniments de seguretat crítics en temps real.

$ SET AUDIT /ALARM=(LOGIN=FAILURE)

Automatització de Respostes

Automatitza les respostes a certs esdeveniments de seguretat utilitzant procediments de comandament o scripts.

Exemple d'un procediment de comandament per bloquejar un compte després de múltiples intents fallits d'inici de sessió:

$! Lock account after 3 failed login attempts
$ SET AUDIT /ALARM=(LOGIN=FAILURE)
$ MONITOR LOGIN /FAILURE /THRESHOLD=3 /ACTION="SET ACCOUNT /FLAGS=DISUSER"

Tècniques Avançades d'Auditoria

Polítiques d'Auditoria Personalitzades

Crea polítiques d'auditoria personalitzades adaptades als requisits específics de seguretat de la teva organització.

$ SET AUDIT /POLICY=(LOGIN=ALL, ACCESS=(READ,WRITE,DELETE))

Integració amb Eines Externes

Integra l'auditoria d'OpenVMS amb eines externes de gestió d'informació i esdeveniments de seguretat (SIEM) per a una anàlisi i informes millorats.

$ SET AUDIT /DESTINATION=SYS$MANAGER:SECURITY_AUDIT.DAT
$ COPY SYS$MANAGER:SECURITY_AUDIT.DAT SIEM_SERVER::DISK$:[AUDIT_LOGS]

Conclusió

L'auditoria i el monitoratge d'esdeveniments de seguretat a OpenVMS són pràctiques essencials per mantenir un sistema segur. Activant l'auditoria, definint esdeveniments rellevants i configurant destinacions d'auditoria, pots fer un seguiment i analitzar efectivament les activitats relacionades amb la seguretat. El monitoratge en temps real i les respostes automatitzades milloren encara més la teva capacitat per gestionar incidents de seguretat. A mesura que avancis, considera personalitzar les polítiques d'auditoria i integrar-les amb eines externes per satisfer les necessitats específiques de la teva organització.
