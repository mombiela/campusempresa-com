Introducció

Els treballs batch són essencials per automatitzar tasques repetitives a OpenVMS. Aquest tema et guiarà a través dels conceptes bàsics de la creació i gestió de treballs batch, progressant cap a tècniques més avançades.

Què és un Treball Batch?

  • Definició: Un treball batch és un procés no interactiu que s'executa en segon pla.
  • Propòsit: Automatitzar tasques repetitives, programar tasques per executar-se en moments específics i gestionar els recursos del sistema de manera eficient.

Creació d'un Treball Batch Simple

Escriure un Fitxer de Comandes Batch

Un fitxer de comandes batch conté una sèrie de comandes DCL (Digital Command Language) per ser executades.

Exemple:

$! Exemple de fitxer de comandes batch
$ WRITE SYS$OUTPUT "Iniciant treball batch..."
$ DIRECTORY SYS$LOGIN
$ WRITE SYS$OUTPUT "Treball batch completat."
  • Explicació:
    • $! és un comentari.
    • WRITE SYS$OUTPUT mostra text al terminal.
    • DIRECTORY SYS$LOGIN llista els fitxers al directori d'inici de sessió de l'usuari.

Enviar un Treball Batch

Utilitza la comanda SUBMIT per enviar un treball batch.

Exemple:

$ SUBMIT my_batch_job.com
  • Explicació:
    • SUBMIT és la comanda per enviar un treball batch.
    • my_batch_job.com és el nom del fitxer de comandes batch.

Gestió de Treballs Batch

Monitoritzar Treballs Batch

Utilitza la comanda SHOW QUEUE per monitoritzar els treballs batch.

Exemple:

$ SHOW QUEUE/BATCH
  • Explicació:
    • SHOW QUEUE mostra l'estat de les cues.
    • /BATCH filtra la visualització per mostrar només les cues batch.

Eliminar un Treball Batch

Utilitza la comanda DELETE/ENTRY per eliminar un treball batch.

Exemple:

$ DELETE/ENTRY=12345
  • Explicació:
    • DELETE/ENTRY elimina un treball específic.
    • 12345 és el número d'entrada del treball.

Tècniques Avançades de Treballs Batch

Programar Treballs Batch

Utilitza el qualificador /AFTER amb la comanda SUBMIT per programar un treball batch.

Exemple:

$ SUBMIT/AFTER="TOMORROW+08:00" my_batch_job.com
  • Explicació:
    • /AFTER="TOMORROW+08:00" programa el treball per executar-se a les 8 del matí del dia següent.

Utilitzar Noms Lògics i Símbols

Els noms lògics i els símbols poden fer els treballs batch més flexibles.

Exemple:

$ DEFINE MYDIR SYS$LOGIN
$ DIRECTORY MYDIR
  • Explicació:
    • DEFINE MYDIR SYS$LOGIN crea un nom lògic MYDIR per al directori d'inici de sessió.
    • DIRECTORY MYDIR utilitza el nom lògic per llistar el contingut del directori.

Gestió d'Errors en Treballs Batch

Utilitza comandes condicionals per gestionar errors.

Exemple:

$ SET NOON
$ COPY file1.txt file2.txt
$ IF $STATUS .NE. 1 THEN WRITE SYS$OUTPUT "Error copiant el fitxer."
  • Explicació:
    • SET NOON evita que el treball s'aturi en cas d'errors.
    • IF $STATUS .NE. 1 comprova si la darrera comanda ha fallat.

Resum

  • Els treballs batch automatitzen tasques i gestionen els recursos del sistema.
  • Crear treballs batch implica escriure fitxers de comandes DCL i utilitzar la comanda SUBMIT.
  • Gestionar treballs batch inclou monitoritzar amb SHOW QUEUE i eliminar amb DELETE/ENTRY.
  • Tècniques avançades inclouen programació, ús de noms lògics i gestió d'errors.

En dominar aquests conceptes, podràs crear i gestionar treballs batch a OpenVMS de manera eficient, millorant la teva productivitat i capacitats de gestió del sistema.
