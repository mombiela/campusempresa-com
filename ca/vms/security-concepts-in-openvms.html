Introducció a la Seguretat d'OpenVMS

OpenVMS és conegut per les seves robustes característiques de seguretat, que són essencials per mantenir la integritat, la confidencialitat i la disponibilitat de les dades. Aquesta secció proporciona una visió general dels conceptes fonamentals de seguretat a OpenVMS.

  • Autenticació d'Usuaris: Assegura que només els usuaris autoritzats puguin accedir al sistema.
  • Control d'Accés: Gestiona els permisos per a fitxers, directoris i altres recursos.
  • Auditoria: Fa un seguiment i registra esdeveniments relacionats amb la seguretat.
  • Xifrat: Protegeix les dades en trànsit i en repòs.

Autenticació d'Usuaris

L'autenticació d'usuaris a OpenVMS implica verificar la identitat dels usuaris que intenten accedir al sistema.

  • Nom d'Usuari i Contrasenya: El mètode principal per a l'autenticació d'usuaris.
  • Autenticació de Doble Factor (2FA): Afegeix una capa extra de seguretat requerint una segona forma de verificació.

Exemple: Crear un Compte d'Usuari

$ SET DEFAULT SYS$SYSTEM
$ RUN AUTHORIZE
UAF> ADD username /PASSWORD=password /OWNER="User Name" /DEVICE=SYS$SYSDEVICE /DIRECTORY=[username]
UAF> EXIT

Explicació:

  • SET DEFAULT SYS$SYSTEM: Estableix el directori per defecte al directori del sistema.
  • RUN AUTHORIZE: Executa l'utilitat AUTHORIZE.
  • ADD username /PASSWORD=password /OWNER="User Name" /DEVICE=SYS$SYSDEVICE /DIRECTORY=[username]: Afegeix un nou usuari amb els atributs especificats.

Control d'Accés

El control d'accés a OpenVMS es gestiona a través de Llistes de Control d'Accés (ACLs) i codis de protecció de fitxers.

  • Codis de Protecció de Fitxers: Defineixen els permisos d'accés per al propietari, grup, món i sistema.
  • Llistes de Control d'Accés (ACLs): Proporcionen un control més granular sobre l'accés als recursos.

Exemple: Establir Codis de Protecció de Fitxers

$ SET PROTECTION=(S:RWED,O:RWED,G:RE,W:RE) filename.txt

Explicació:

  • SET PROTECTION=(S:RWED,O:RWED,G:RE,W:RE) filename.txt: Estableix el codi de protecció per a filename.txt on:
    • S (Sistema): Llegir, Escriure, Executar, Esborrar
    • O (Propietari): Llegir, Escriure, Executar, Esborrar
    • G (Grup): Llegir, Executar
    • W (Món): Llegir, Executar

Auditoria

L'auditoria a OpenVMS implica monitoritzar i registrar esdeveniments relacionats amb la seguretat per detectar i respondre a possibles violacions de seguretat.

  • Servidor d'Auditoria: Recull i emmagatzema registres d'auditoria.
  • Polítiques d'Auditoria: Defineixen quins esdeveniments són auditats.

Exemple: Activar l'Auditoria per a Fallades d'Inici de Sessió

$ SET AUDIT /ENABLE=LOGIN_FAILURE

Explicació:

  • SET AUDIT /ENABLE=LOGIN_FAILURE: Activa l'auditoria per a esdeveniments de fallada d'inici de sessió.

Xifrat

El xifrat a OpenVMS assegura que les dades estiguin protegides tant en trànsit com en repòs.

  • Xifrat de Dades: Protegeix les dades emmagatzemades en disc.
  • Xifrat de Xarxa: Assegura les dades transmeses per la xarxa.

Exemple: Xifrar un Fitxer

$ ENCRYPT/KEY=mykey filename.txt

Explicació:

  • ENCRYPT/KEY=mykey filename.txt: Xifra filename.txt utilitzant la clau mykey.

Resum

Comprendre i implementar conceptes de seguretat a OpenVMS és crucial per protegir la integritat del sistema i la confidencialitat de les dades. Les àrees clau inclouen l'autenticació d'usuaris, el control d'accés, l'auditoria i el xifrat. Dominant aquests conceptes, pots assegurar un entorn OpenVMS segur.

  • Autenticació d'Usuaris: Verificar les identitats dels usuaris.
  • Control d'Accés: Gestionar permisos i accessos.
  • Auditoria: Monitoritzar i registrar esdeveniments de seguretat.
  • Xifrat: Protegir les dades en trànsit i en repòs.

Seguint aquestes pràctiques, pots mantenir un sistema OpenVMS segur i fiable.
