Introducció

Desplegar i mantenir aplicacions a OpenVMS requereix una comprensió sòlida de les característiques úniques i les eines del sistema operatiu. Aquesta secció et guiarà a través del procés des d'un nivell principiant fins a un nivell avançat, cobrint conceptes clau, exemples pràctics i millors pràctiques.

Comprendre l'Entorn OpenVMS

Abans d'endinsar-nos en el desplegament, és essencial comprendre l'entorn OpenVMS:

  • DCL (Digital Command Language): El llenguatge de comandes utilitzat a OpenVMS.
  • Sistema de Fitxers: OpenVMS utilitza un sistema de fitxers jeràrquic.
  • Comptes d'Usuari i Privilegis: Gestionar els permisos d'usuari és crucial per a un desplegament segur.

Preparar l'Aplicació per al Desplegament

Compilar i Enllaçar

  • Compilar: Convertir el codi font en codi objecte.
  • Enllaçar: Combinar el codi objecte amb biblioteques per crear un executable.

Exemple:

$ FORTRAN myprogram.for
$ LINK myprogram

Explicació:

  • FORTRAN myprogram.for compila el fitxer font Fortran.
  • LINK myprogram enllaça el fitxer objecte compilat per crear un executable.

Crear Procediments de Comanda

  • Procediments de Comanda: Scripts escrits en DCL per automatitzar tasques.
  • Exemple de Procediment de Comanda:
$! Aquest és un procediment de comanda d'exemple
$ SET DEFAULT SYS$USER:[MYDIR]
$ RUN MYPROGRAM.EXE

Explicació:

  • SET DEFAULT SYS$USER:[MYDIR] canvia el directori de treball.
  • RUN MYPROGRAM.EXE executa el programa.

Desplegar l'Aplicació

Copiar Fitxers

  • Utilitzar la Comanda COPY: Transferir fitxers al directori de destinació.
$ COPY MYPROGRAM.EXE SYS$USER:[TARGETDIR]

Explicació:

  • COPY MYPROGRAM.EXE SYS$USER:[TARGETDIR] copia l'executable al directori de destinació.

Configurar Noms Lògics

  • Noms Lògics: Àlies per a especificacions de fitxers o dispositius.
$ DEFINE MYAPP SYS$USER:[TARGETDIR]MYPROGRAM.EXE

Explicació:

  • DEFINE MYAPP SYS$USER:[TARGETDIR]MYPROGRAM.EXE crea un nom lògic MYAPP per a l'executable.

Gestionar l'Accés d'Usuaris

  • Concedir Accés: Utilitzar la utilitat AUTHORIZE per gestionar els permisos d'usuari.
$ RUN SYS$SYSTEM:AUTHORIZE
UAF> ADD USERNAME /PASSWORD=secret /DEVICE=SYS$USER /DIRECTORY=[USERNAME]
UAF> EXIT

Explicació:

  • ADD USERNAME /PASSWORD=secret /DEVICE=SYS$USER /DIRECTORY=[USERNAME] crea un nou usuari amb els atributs especificats.

Mantenir l'Aplicació

Monitoritzar el Rendiment

  • Utilitzar la Utilitat MONITOR: Fer un seguiment del rendiment del sistema.
$ MONITOR SYSTEM

Explicació:

  • MONITOR SYSTEM mostra les mètriques de rendiment del sistema.

Actualitzar l'Aplicació

  • Substituir Executables: Utilitzar RENAME i COPY per actualitzar fitxers.
$ RENAME SYS$USER:[TARGETDIR]MYPROGRAM.EXE MYPROGRAM_OLD.EXE
$ COPY NEW_MYPROGRAM.EXE SYS$USER:[TARGETDIR]MYPROGRAM.EXE

Explicació:

  • RENAME canvia el nom de l'antic executable.
  • COPY col·loca el nou executable al directori de destinació.

Còpia de Seguretat i Recuperació

  • Utilitzar la Utilitat BACKUP: Crear còpies de seguretat dels fitxers importants.
$ BACKUP SYS$USER:[TARGETDIR]*.*;* SYS$USER:[BACKUPDIR]

Explicació:

  • BACKUP SYS$USER:[TARGETDIR]*.*;* SYS$USER:[BACKUPDIR] fa una còpia de seguretat de tots els fitxers del directori de destinació al directori de còpia de seguretat.

Conclusió

Desplegar i mantenir aplicacions a OpenVMS implica comprendre l'entorn, preparar l'aplicació, desplegar-la correctament i mantenir-la a través del monitoratge, actualitzacions i còpies de seguretat. Seguint aquests passos i utilitzant els exemples proporcionats, pots assegurar un procés de desplegament i manteniment fluid per a les teves aplicacions a OpenVMS.
