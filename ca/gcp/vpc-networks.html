Introducció a les Xarxes VPC

Les Xarxes Virtual Private Cloud (VPC) són components fonamentals a Google Cloud Platform (GCP) que et permeten crear xarxes virtuals aïllades per allotjar els teus recursos de GCP. Entendre les Xarxes VPC és crucial per dissenyar arquitectures de núvol segures i eficients.

Conceptes Clau

  • Xarxa VPC: Una versió virtual d'una xarxa física, implementada a GCP.
  • Subxarxes: Segments d'una xarxa VPC que poden abastar diverses regions.
  • Rangs d'Adreces IP: Defineixen el rang d'adreces IP disponibles per als recursos dins d'una subxarxa.
  • Regles de Tallafocs: Controlen el trànsit cap a i des de les instàncies dins d'una xarxa VPC.
  • Rutes: Determinen els camins que el trànsit de xarxa pren d'una instància a una altra.

Creació i Configuració de Xarxes VPC

Creació d'una Xarxa VPC

Per crear una xarxa VPC, pots utilitzar la Consola de GCP, l'eina de línia de comandes gcloud, o Terraform per a infraestructura com a codi.

Utilitzant la Consola de GCP

  1. Navega a la secció de xarxes VPC a la Consola de GCP.
  2. Fes clic a "Crear xarxa VPC".
  3. Proporciona un nom per a la teva xarxa.
  4. Tria el mode de creació de subxarxes (Automàtic o Personalitzat).
  5. Defineix el rang d'adreces IP per a cada subxarxa.
  6. Fes clic a "Crear".

Utilitzant l'Eina de Línia de Comandes gcloud

gcloud compute networks create my-vpc-network --subnet-mode=custom
gcloud compute networks subnets create my-subnet \
    --network=my-vpc-network \
    --range=10.0.0.0/24 \
    --region=us-central1

Configuració de Regles de Tallafocs

Les regles de tallafocs són essencials per controlar el trànsit cap a i des de la teva xarxa VPC.

Exemple: Permetre Accés SSH

gcloud compute firewall-rules create allow-ssh \
    --network my-vpc-network \
    --allow tcp:22 \
    --source-ranges 0.0.0.0/0 \
    --target-tags ssh-access

Configuració de Rutes

Les rutes determinen com es dirigeixen els paquets dins de la teva xarxa VPC.

Exemple: Creació d'una Ruta Estàtica

gcloud compute routes create my-route \
    --network my-vpc-network \
    --destination-range 10.1.0.0/16 \
    --next-hop-instance my-instance \
    --next-hop-instance-zone us-central1-a

Funcions Avançades de Xarxes VPC

VPC Compartida

La VPC Compartida permet que múltiples projectes comparteixin una xarxa VPC comuna, permetent un control centralitzat sobre els recursos de xarxa.

Configuració de VPC Compartida

  1. Designa un projecte amfitrió.
  2. Adjunta projectes de servei al projecte amfitrió.
  3. Concedeix els rols IAM necessaris per gestionar la VPC compartida.

Interconnexió de Xarxes VPC

L'Interconnexió de Xarxes VPC permet connectar dues xarxes VPC de manera privada utilitzant adreces IP internes.

Creació d'una Connexió d'Interconnexió de Xarxes VPC

gcloud compute networks peerings create my-peering \
    --network my-vpc-network \
    --peer-network peer-vpc-network \
    --auto-create-routes

Accés Privat a Google

L'Accés Privat a Google permet que les instàncies en una xarxa VPC arribin a les API i serveis de Google utilitzant adreces IP internes.

Activació de l'Accés Privat a Google

  1. Navega als detalls de la subxarxa a la Consola de GCP.
  2. Activa "Accés Privat a Google" per a la subxarxa.

Millors Pràctiques

  • Dissenya per a l'Escalabilitat: Planifica els teus rangs d'adreces IP i subxarxes per acomodar el creixement futur.
  • Implementa el Principi de Menor Privilegi: Utilitza regles de tallafocs per restringir l'accés només al que és necessari.
  • Monitora i Audita: Revisa regularment les configuracions de xarxa i els registres per assegurar el compliment i la seguretat.

Conclusió

Les Xarxes VPC a GCP proporcionen una manera flexible i segura de gestionar els teus recursos al núvol. Entenent els conceptes bàsics de la creació, configuració i funcions avançades com la VPC Compartida i l'Interconnexió de Xarxes VPC, pots dissenyar arquitectures de xarxa robustes que compleixin amb les necessitats de la teva organització. Segueix sempre les millors pràctiques per assegurar que la teva xarxa es mantingui segura i escalable.
