Introducció

Google Cloud Platform (GCP) ofereix eines potents per monitoritzar i depurar aplicacions. Cloud Trace i Cloud Debugger són dues d'aquestes eines que ajuden els desenvolupadors a obtenir informació sobre el rendiment de les seves aplicacions i diagnosticar problemes en temps real.

Cloud Trace

Cloud Trace és un sistema de traçat distribuït que recull dades de latència de les teves aplicacions i les mostra a la Google Cloud Console. T'ajuda a entendre com es propaguen les sol·licituds a través de la teva aplicació i a identificar colls d'ampolla.

Conceptes Clau

  • Traça: Una traça registra el camí d'una sol·licitud a mesura que es mou a través de diversos serveis.
  • Span: Un span representa una única operació dins d'una traça. Inclou metadades com ara l'hora d'inici i final, i qualsevol etiqueta associada.
  • Anàlisi de Latència: Cloud Trace t'ajuda a analitzar la latència de la teva aplicació mostrant quant temps triga cada sol·licitud i on es produeixen els retards.

Configuració de Cloud Trace

  1. Habilitar l'API de Cloud Trace:

    • Ves a la Consola de GCP.
    • Navega a APIs & Services > Library.
    • Cerca "Cloud Trace API" i habilita-la.

  2. Instal·lar el SDK de Trace:

    • Per Python: 
      pip install google-cloud-trace
    • Per Node.js: 
      npm install @google-cloud/trace-agent

  3. Inicialitzar Trace a la Teva Aplicació:

    • Exemple en Python: 
      from google.cloud import trace

trace_client = trace.Client()
trace_client.setup_logging()
    • Exemple en Node.js: 
      require('@google-cloud/trace-agent').start();

Visualització de Traces

  • Navega a la secció de Cloud Trace a la Consola de GCP.
  • Utilitza la Llista de Traces per veure traces individuals i els seus detalls.
  • Analitza les dades de latència per identificar colls d'ampolla de rendiment.

Cloud Debugger

Cloud Debugger et permet inspeccionar l'estat d'una aplicació en qualsevol ubicació de codi en temps real sense aturar o alentir l'aplicació.

Conceptes Clau

  • Snapshot: Un snapshot captura les variables locals i la pila de trucades en una línia de codi específica.
  • Logpoint: Un logpoint et permet injectar declaracions de registre a la teva aplicació en execució sense redeployar-la.

Configuració de Cloud Debugger

  1. Habilitar l'API de Cloud Debugger:

    • Ves a la Consola de GCP.
    • Navega a APIs & Services > Library.
    • Cerca "Cloud Debugger API" i habilita-la.

  2. Instal·lar el SDK de Debugger:

    • Per Python: 
      pip install google-cloud-debugger
    • Per Node.js: 
      npm install @google-cloud/debug-agent

  3. Inicialitzar Debugger a la Teva Aplicació:

    • Exemple en Python: 
      import googleclouddebugger

googleclouddebugger.enable()
    • Exemple en Node.js: 
      require('@google-cloud/debug-agent').start();

Ús de Cloud Debugger

  • Navega a la secció de Cloud Debugger a la Consola de GCP.
  • Configura breakpoints al teu codi per capturar snapshots.
  • Utilitza logpoints per afegir registres dinàmicament.

Exemple: Configuració d'un Snapshot

  • Python: 
    def example_function():
    x = 10
    y = 20
    z = x + y  # Configura un snapshot aquí per inspeccionar les variables x, y, i z
    return z
  • Node.js: 
    function exampleFunction() {
    let x = 10;
    let y = 20;
    let z = x + y; // Configura un snapshot aquí per inspeccionar les variables x, y, i z
    return z;
}

Conclusió

Cloud Trace i Cloud Debugger són eines essencials per monitoritzar i depurar aplicacions a GCP. Cloud Trace t'ajuda a entendre el rendiment i la latència de la teva aplicació, mentre que Cloud Debugger et permet inspeccionar i depurar el teu codi en temps real sense interrompre el flux de l'aplicació. Integrant aquestes eines al teu flux de treball de desenvolupament, pots millorar significativament la fiabilitat i el rendiment de les teves aplicacions.
