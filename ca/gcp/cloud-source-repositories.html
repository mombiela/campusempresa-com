Introducció als Repositoris de codi font al núvol

Els Repositoris de codi font al núvol (CSR) són un repositori Git complet, escalable i segur allotjat a la Plataforma de Google Cloud (GCP). Permet als desenvolupadors col·laborar en el codi, gestionar versions i integrar-se amb altres serveis de GCP sense problemes.

Conceptes clau

  • Git: Un sistema de control de versions distribuït utilitzat per fer un seguiment dels canvis en el codi font durant el desenvolupament de programari.
  • Repositori: Una ubicació d'emmagatzematge per a paquets de programari, sovint utilitzada per emmagatzemar múltiples versions de codi.
  • Branques: Crear una línia de desenvolupament separada en un repositori.
  • Fusionar: Combinar canvis de diferents branques en una sola branca.

Configuració dels Repositoris de codi font al núvol

Crear un repositori

  1. Habilitar l'API dels Repositoris de codi font al núvol:

    • Aneu a la Consola de GCP.
    • Navegueu al tauler de control d'API i Serveis.
    • Habiliteu l'API dels Repositoris de codi font al núvol.

  2. Crear un repositori:

    • Aneu a la pàgina dels Repositoris de codi font al núvol a la Consola de GCP.
    • Feu clic a "Afegeix repositori".
    • Trieu "Crea un nou repositori".
    • Proporcioneu un nom per al vostre repositori i feu clic a "Crea".

Clonar un repositori

Per clonar un repositori, utilitzeu la següent comanda al vostre terminal:

git clone https://source.developers.google.com/p/PROJECT_ID/r/REPOSITORY_NAME

Substituïu PROJECT_ID i REPOSITORY_NAME pel vostre ID de projecte i nom del repositori real.

Operacions bàsiques de Git

Cometre canvis

  1. Estagejar canvis:

    git add .

    Aquesta comanda estageja tots els canvis al directori actual.

  2. Cometre canvis:

    git commit -m "El teu missatge de commit"

    Aquesta comanda commet els canvis estagejats amb un missatge descriptiu.

Empènyer canvis

Per empènyer els vostres canvis al repositori remot:

git push origin main

Substituïu main pel nom de la vostra branca si és diferent.

Funcions avançades

Branques i fusions

  • Crear una branca:

    git checkout -b new-feature

    Aquesta comanda crea una nova branca anomenada new-feature i hi canvia.

  • Fusionar una branca:

    git checkout main
git merge new-feature

    Aquesta comanda fusiona la branca new-feature a la branca main.

Integració amb altres serveis de GCP

  • Cloud Build: Automatitzar compilacions i desplegaments.
  • Cloud Functions: Activar funcions basades en esdeveniments del repositori.
  • Cloud Monitoring: Supervisar l'activitat i el rendiment del repositori.

Control d'accés

  • Rols IAM: Assignar rols com Visualitzador, Editor i Propietari per controlar l'accés als repositoris.
  • Protecció de branques: Aplicar regles a les branques per evitar canvis no autoritzats.

Exemple: Configuració de la integració contínua amb Cloud Build

  1. Crear un fitxer cloudbuild.yaml:

    steps:
- name: 'gcr.io/cloud-builders/git'
  args: ['clone', 'https://source.developers.google.com/p/PROJECT_ID/r/REPOSITORY_NAME']
- name: 'gcr.io/cloud-builders/docker'
  args: ['build', '-t', 'gcr.io/PROJECT_ID/IMAGE_NAME', '.']
images:
- 'gcr.io/PROJECT_ID/IMAGE_NAME'

    Substituïu PROJECT_ID, REPOSITORY_NAME i IMAGE_NAME pel vostre ID de projecte, nom del repositori i nom de la imatge desitjada.

  2. Activar una compilació:

    • Aneu a la pàgina de Cloud Build a la Consola de GCP.
    • Feu clic a "Triggers" i configureu un nou trigger per executar-se en commits al vostre repositori.

Conclusió

Els Repositoris de codi font al núvol proporcionen una solució robusta i integrada per gestionar el vostre codi font a GCP. Des d'operacions bàsiques de Git fins a integracions avançades amb altres serveis de GCP, CSR ofereix un conjunt complet d'eines per als desenvolupadors. Comprenent i utilitzant aquestes funcions, podeu millorar el vostre flux de treball de desenvolupament i assegurar-vos que el vostre codi sigui segur, escalable i gestionat de manera eficient.
