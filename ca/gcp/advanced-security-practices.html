Introducció

En aquesta secció, aprofundirem en les pràctiques de seguretat avançades a Google Cloud Platform (GCP). Això inclou entendre i implementar mesures de seguretat robustes per protegir les teves aplicacions i dades. Cobrirem temes com la Gestió d'Identitat i Accés (IAM), seguretat de xarxa, xifrat de dades, i monitoratge i registre.

Gestió d'Identitat i Accés (IAM)

IAM és un component crític de la seguretat de GCP. Et permet gestionar l'accés als recursos definint qui (identitat) té quin accés (rol) a quin recurs.

  • Principis de IAM:

    • Privilegi Mínim: Concedeix només els permisos necessaris per realitzar una tasca.
    • Separació de Funcions: Divideix les responsabilitats per reduir el risc de frau o error.
    • Control d'Accés Basat en Rols (RBAC): Assigna rols als usuaris basant-se en les seves funcions laborals.

  • Components de IAM:

    • Identitats: Usuaris, grups, comptes de servei.
    • Rols: Rols predefinits, rols personalitzats.
    • Polítiques: Vinculacions de rols a identitats.

Exemple: Crear un Rol Personalitzat

gcloud iam roles create customRoleName \
  --project=my-project \
  --title="Custom Role" \
  --description="A custom role with specific permissions" \
  --permissions=compute.instances.start,compute.instances.stop

Aquesta comanda crea un rol personalitzat amb permisos per iniciar i aturar instàncies de Compute Engine.

Seguretat de Xarxa

Assegurar la teva xarxa és crucial per protegir els teus recursos d'accés no autoritzat i atacs.

  • VPC (Virtual Private Cloud):

    • Subxarxes: Divideix la teva VPC en subxarxes per a una millor organització i seguretat.
    • Regles de Tallafocs: Controla el trànsit cap a i des de les teves instàncies.
    • Accés Privat a Google: Permet que les instàncies sense adreces IP externes arribin als serveis de Google.

  • Millors Pràctiques:

    • Utilitza VPC Service Controls: Defineix perímetres de seguretat al voltant dels teus recursos de GCP.
    • Implementa Regles de Tallafocs: Restringeix el trànsit basant-se en rangs d'IP, protocols i ports.
    • Utilitza IPs Privades: Limita l'exposició utilitzant adreces IP privades per a la comunicació interna.

Exemple: Crear una Regla de Tallafocs

gcloud compute firewall-rules create allow-ssh \
  --network default \
  --allow tcp:22 \
  --source-ranges 0.0.0.0/0 \
  --target-tags ssh

Aquesta comanda crea una regla de tallafocs per permetre el trànsit SSH des de qualsevol adreça IP.

Xifrat de Dades

El xifrat de dades és essencial per protegir informació sensible tant en repòs com en trànsit.

  • Xifrat en Repòs:

    • Claus de Xifrat Gestionades pel Client (CMEK): Utilitza les teves pròpies claus de xifrat.
    • Claus de Xifrat Subministrades pel Client (CSEK): Subministra les teves pròpies claus per al xifrat.

  • Xifrat en Trànsit:

    • TLS/SSL: Utilitza Transport Layer Security (TLS) o Secure Sockets Layer (SSL) per xifrar dades en trànsit.

Exemple: Activar CMEK per a un Bucket d'Emmagatzematge

gsutil kms authorize -p my-project -k projects/my-project/locations/global/keyRings/my-key-ring/cryptoKeys/my-key
gsutil mb -p my-project -l us-central1 -b on gs://my-secure-bucket
gsutil bucketpolicyonly set on gs://my-secure-bucket
gsutil kms encryption set projects/my-project/locations/global/keyRings/my-key-ring/cryptoKeys/my-key gs://my-secure-bucket

Aquesta seqüència de comandes autoritza, crea i configura un bucket d'emmagatzematge per utilitzar una clau de xifrat gestionada pel client.

Monitoratge i Registre

El monitoratge i registre són vitals per detectar i respondre a incidents de seguretat.

  • Cloud Logging: Recull i emmagatzema registres dels serveis de GCP.

  • Cloud Monitoring: Monitoritza el rendiment i la salut de les teves aplicacions.

  • Security Command Center: Obté visibilitat sobre la teva postura de seguretat.

  • Millors Pràctiques:

    • Activa Registres d'Auditoria: Fes un seguiment de l'accés i els canvis als teus recursos.
    • Configura Alertes: Configura alertes per a activitats sospitoses.
    • Revisa Regularment els Registres: Analitza els registres per identificar possibles problemes de seguretat.

Exemple: Configurar una Alerta Basada en Registres

gcloud logging sinks create my-sink \
  storage.googleapis.com/my-bucket \
  --log-filter='resource.type="gce_instance" AND severity>=ERROR'

Aquesta comanda crea un dipòsit de registres que exporta registres d'instàncies GCE amb severitat ERROR o superior a un bucket de Cloud Storage.

Conclusió

En aquesta secció, hem cobert pràctiques de seguretat avançades a GCP, incloent IAM, seguretat de xarxa, xifrat de dades, i monitoratge i registre. Implementant aquestes pràctiques, pots millorar significativament la seguretat del teu entorn GCP. Mantén-te sempre actualitzat amb les últimes característiques de seguretat i millors pràctiques per protegir els teus recursos de manera efectiva.
