Introducció a IAM

La Gestió d'Identitats i Accessos (IAM) és un component crític de Google Cloud Platform (GCP) que et permet gestionar l'accés als recursos definint qui (identitat) té quin accés (rol) a quins recursos.

Conceptes Clau

  • Identitat: Representa usuaris, grups o serveis que necessiten accés als recursos de GCP.
  • Rol: Una col·lecció de permisos que es poden assignar a identitats.
  • Permís: Defineix quines accions es poden realitzar en un recurs.
  • Política: Una col·lecció de vinculacions que associen rols amb identitats.

Conceptes Bàsics de IAM

Comprensió de les Identitats

  • Comptes de Google: Usuaris individuals amb un compte de Google.
  • Comptes de Servei: Comptes especials utilitzats per aplicacions o màquines virtuals per fer trucades API autoritzades.
  • Grups de Google: Col·leccions de comptes de Google.
  • Dominis de Cloud Identity: Dominis gestionats per organitzacions.

Comprensió dels Rols

  • Rols Primitivos: Rols bàsics que inclouen Propietari, Editor i Visualitzador.
  • Rols Predefinits: Rols granulars creats per Google que proporcionen permisos específics.
  • Rols Personalitzats: Rols definits per l'usuari que proporcionen un control detallat sobre els permisos.

Assignació de Rols a Identitats

Per assignar rols a identitats, crees una vinculació de política IAM. Aquí tens un exemple de com assignar el rol de Visualitzador a un usuari:

bindings:
- members:
  - user:example-user@gmail.com
  role: roles/viewer

Conceptes Intermedis de IAM

Gestió de Comptes de Servei

Els comptes de servei són crucials per a aplicacions que s'executen a GCP. Aquí tens com crear i gestionar-los:

Creació d'un Compte de Servei

gcloud iam service-accounts create my-service-account \
    --display-name "My Service Account"

Assignació de Rols a un Compte de Servei

gcloud projects add-iam-policy-binding my-project \
    --member "serviceAccount:my-service-account@my-project.iam.gserviceaccount.com" \
    --role "roles/editor"

Ús de Polítiques IAM

Les polítiques IAM defineixen qui té quin tipus d'accés. Aquí tens un exemple d'una política IAM:

{
  "bindings": [
    {
      "role": "roles/viewer",
      "members": [
        "user:example-user@gmail.com"
      ]
    },
    {
      "role": "roles/editor",
      "members": [
        "serviceAccount:my-service-account@my-project.iam.gserviceaccount.com"
      ]
    }
  ]
}

Conceptes Avançats de IAM

Rols Personalitzats

Els rols personalitzats et permeten definir permisos específics adaptats a les teves necessitats.

Creació d'un Rol Personalitzat

gcloud iam roles create myCustomRole \
    --project my-project \
    --title "My Custom Role" \
    --permissions compute.instances.start,compute.instances.stop \
    --stage GA

Auditoria i Monitorització de IAM

Monitoritzar les activitats de IAM és essencial per a la seguretat i el compliment.

Activació dels Registres d'Auditoria

gcloud logging sinks create my-audit-logs \
    storage.googleapis.com/my-audit-logs-bucket \
    --log-filter='resource.type="gce_instance"'

Millors Pràctiques

  • Principi del Menor Privilegi: Concedeix només els permisos necessaris perquè un usuari pugui realitzar la seva feina.
  • Auditories Regulars: Revisa regularment les polítiques i rols de IAM.
  • Ús de Grups: Gestiona l'accés assignant rols a grups en lloc d'usuaris individuals.

Conclusió

IAM a GCP és una eina poderosa per gestionar l'accés als teus recursos al núvol. Comprenent i utilitzant identitats, rols i polítiques, pots assegurar-te que els teus recursos estiguin segurs i que els usuaris tinguin el nivell d'accés adequat. Recorda seguir les millors pràctiques com el principi del menor privilegi i les auditories regulars per mantenir un entorn segur.
