Introducció a les Arquitectures sense servidor

La computació sense servidor et permet construir i executar aplicacions i serveis sense pensar en els servidors. Amb el model sense servidor, el proveïdor de núvol provisiona, escala i gestiona automàticament la infraestructura necessària per executar el teu codi.

Conceptes Clau

  • Computació sense servidor: Un model d'execució de computació en núvol on el proveïdor de núvol gestiona dinàmicament l'assignació de recursos de màquina.
  • Funció com a Servei (FaaS): Una categoria de serveis de computació en núvol que proporciona una plataforma que permet als clients desenvolupar, executar i gestionar funcionalitats d'aplicacions sense la complexitat de construir i mantenir la infraestructura.
  • Arquitectura impulsada per esdeveniments: Un patró d'arquitectura de programari que promou la producció, detecció, consum i reacció als esdeveniments.

Començant amb Google Cloud Functions

Google Cloud Functions és una solució de computació lleugera, basada en esdeveniments i asíncrona que et permet crear funcions petites i de propòsit únic que responen a esdeveniments del núvol sense la necessitat de gestionar un servidor o un entorn d'execució.

Creant la teva primera Cloud Function

  1. Configura el teu projecte GCP:

    • Assegura't de tenir un compte GCP i crea un nou projecte.
    • Activa l'API de Cloud Functions.

  2. Escriu la teva funció:

    • Crea una funció simple que respongui a sol·licituds HTTP.
// index.js
exports.helloWorld = (req, res) => {
  res.send('Hola, Món!');
};
  1. Desplega la teva funció:
    • Utilitza l'eina de línia de comandes gcloud per desplegar la teva funció.
gcloud functions deploy helloWorld --runtime nodejs14 --trigger-http --allow-unauthenticated
  1. Prova la teva funció:
    • Després del desplegament, rebràs una URL. Obre aquesta URL al teu navegador per veure el resultat.

Funcions avançades de Cloud Functions

Variables d'entorn

  • Configuració de variables d'entorn: Utilitza variables d'entorn per configurar les teves funcions sense canviar el codi.
gcloud functions deploy helloWorld --set-env-vars GREETING="Hola, GCP!"
  • Accés a les variables d'entorn:
exports.helloWorld = (req, res) => {
  const greeting = process.env.GREETING || 'Hola, Món!';
  res.send(greeting);
};

Connexió a altres serveis de GCP

  • Cloud Storage: Activa funcions en canvis de fitxers a Cloud Storage.
exports.processFile = (event, context) => {
  const file = event.name;
  console.log(`Processant fitxer: ${file}`);
};
  • Cloud Pub/Sub: Activa funcions en missatges de Pub/Sub.
exports.subscribe = (message, context) => {
  const data = Buffer.from(message.data, 'base64').toString();
  console.log(`Missatge rebut: ${data}`);
};

Google Cloud Run

Google Cloud Run et permet executar contenidors sense estat que poden ser invocats mitjançant sol·licituds HTTP. Abstracte la gestió de la infraestructura, permetent-te centrar en escriure codi.

Desplegant un contenidor a Cloud Run

  1. Crea un Dockerfile:
# Utilitza la imatge oficial de Node.js.
FROM node:14

# Crea i canvia al directori de l'aplicació.
WORKDIR /usr/src/app

# Copia els manifests de dependències de l'aplicació a la imatge del contenidor.
COPY package*.json ./

# Instal·la les dependències.
RUN npm install

# Copia el codi local a la imatge del contenidor.
COPY . .

# Executa el servei web en iniciar el contenidor.
CMD [ "node", "index.js" ]

# Documenta que el servei escolta al port 8080.
EXPOSE 8080
  1. Construeix i puja la imatge del contenidor:
gcloud builds submit --tag gcr.io/PROJECT-ID/helloworld
  1. Desplega a Cloud Run:
gcloud run deploy --image gcr.io/PROJECT-ID/helloworld --platform managed
  1. Prova el teu servei:
    • Després del desplegament, rebràs una URL. Obre aquesta URL al teu navegador per veure el resultat.

Millors pràctiques per a arquitectures sense servidor

  • Funcions sense estat: Assegura't que les teves funcions siguin sense estat per permetre una escalabilitat fàcil.
  • Gestió eficient de recursos: Utilitza variables d'entorn i gestió de secrets per gestionar la configuració.
  • Monitorització i registre: Utilitza les eines de monitorització i registre de GCP per fer un seguiment del rendiment i errors de les teves funcions.

Conclusió

Les arquitectures sense servidor a GCP proporcionen una manera poderosa de construir aplicacions escalables i eficients sense la càrrega de gestionar la infraestructura. Aprofitant serveis com Google Cloud Functions i Google Cloud Run, pots centrar-te en escriure codi i oferir valor als teus usuaris. Recorda seguir les millors pràctiques per assegurar que les teves aplicacions siguin robustes i mantenibles.
