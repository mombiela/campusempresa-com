Introducció

En aquesta secció, aprofundirem en els conceptes avançats de xarxes a Google Cloud Platform (GCP). Això inclou entendre les configuracions de Virtual Private Cloud (VPC), la connectivitat híbrida, el balanç de càrrega i les mesures de seguretat. Al final d'aquest tema, hauríeu de ser capaços de dissenyar i implementar solucions de xarxes complexes a GCP.

Virtual Private Cloud (VPC)

Una Virtual Private Cloud (VPC) és una xarxa virtual dedicada al vostre projecte de Google Cloud. És un recurs global que consisteix en una llista de subxarxes regionals, una taula de rutes i regles de tallafocs.

Conceptes Clau

  • Subxarxes: Recursos regionals que divideixen la vostra xarxa VPC en segments més petits.
  • Rutes: Defineixen camins perquè el trànsit viatgi dins de la vostra xarxa VPC.
  • Regles de Tallafocs: Controlen el trànsit cap a i des de les instàncies basant-se en regles especificades.

Exemple: Crear una VPC amb Subxarxes

gcloud compute networks create my-vpc --subnet-mode=custom
gcloud compute networks subnets create subnet-us-central1 \
    --network=my-vpc \
    --region=us-central1 \
    --range=10.0.0.0/24

En aquest exemple, creem una VPC en mode personalitzat anomenada my-vpc i una subxarxa anomenada subnet-us-central1 a la regió us-central1 amb un rang CIDR de 10.0.0.0/24.

Connectivitat Híbrida

La connectivitat híbrida permet connectar la vostra xarxa local a la vostra xarxa VPC de GCP. Això es pot aconseguir mitjançant diversos mètodes com VPN, Interconnect i Peering.

Conceptes Clau

  • Cloud VPN: Estableix una connexió segura a través d'internet.
  • Cloud Interconnect: Proporciona una connexió física directa entre la vostra xarxa local i GCP.
  • VPC Peering: Connecta dues xarxes VPC per permetre la comunicació utilitzant adreces IP internes.

Exemple: Configurar una Cloud VPN

gcloud compute vpn gateways create my-vpn-gateway \
    --network=my-vpc \
    --region=us-central1

gcloud compute vpn tunnels create my-vpn-tunnel \
    --peer-address=203.0.113.1 \
    --region=us-central1 \
    --ike-version=2 \
    --shared-secret=my-secret \
    --target-vpn-gateway=my-vpn-gateway

En aquest exemple, creem una passarel·la VPN i un túnel VPN per establir una connexió segura amb una xarxa local amb una adreça de parella de 203.0.113.1.

Balanç de Càrrega

El balanç de càrrega distribueix el trànsit entrant entre diverses instàncies per garantir una alta disponibilitat i fiabilitat.

Conceptes Clau

  • Balanç de Càrrega HTTP(S): Distribueix el trànsit HTTP i HTTPS.
  • Balanç de Càrrega TCP/UDP: Distribueix el trànsit TCP i UDP.
  • Balanç de Càrrega Intern: Distribueix el trànsit dins d'una VPC.

Exemple: Configurar un Balançador de Càrrega HTTP

gcloud compute instance-groups unmanaged create my-instance-group --zone=us-central1-a

gcloud compute instance-groups unmanaged add-instances my-instance-group \
    --instances=instance-1,instance-2 --zone=us-central1-a

gcloud compute backend-services create my-backend-service \
    --protocol=HTTP \
    --port-name=http \
    --health-checks=my-health-check \
    --global

gcloud compute url-maps create my-url-map \
    --default-service=my-backend-service

gcloud compute target-http-proxies create my-http-proxy \
    --url-map=my-url-map

gcloud compute forwarding-rules create my-http-rule \
    --global \
    --target-http-proxy=my-http-proxy \
    --ports=80

En aquest exemple, creem un grup d'instàncies no gestionat, afegim instàncies a aquest grup i configurem un balançador de càrrega HTTP per distribuir el trànsit a aquestes instàncies.

Mesures de Seguretat

La seguretat és un aspecte crític de les xarxes. GCP proporciona diverses eines i característiques per assegurar la vostra xarxa.

Conceptes Clau

  • Regles de Tallafocs: Controlen el trànsit cap a i des de les instàncies.
  • Cloud Armor: Proporciona protecció contra DDoS i capacitats de WAF.
  • Accés Privat a Google: Permet que les instàncies arribin a les API i serveis de Google utilitzant adreces IP internes.

Exemple: Crear una Regla de Tallafocs

gcloud compute firewall-rules create allow-http \
    --network=my-vpc \
    --allow=tcp:80 \
    --source-ranges=0.0.0.0/0 \
    --target-tags=http-server

En aquest exemple, creem una regla de tallafocs per permetre el trànsit HTTP (port 80) des de qualsevol font a les instàncies etiquetades amb http-server.

Conclusió

En aquest tema, hem cobert conceptes avançats de xarxes a GCP, incloent configuracions de VPC, connectivitat híbrida, balanç de càrrega i mesures de seguretat. En dominar aquests conceptes, podeu dissenyar i implementar solucions de xarxes robustes i segures a Google Cloud Platform. Continueu practicant amb escenaris del món real per consolidar la vostra comprensió i habilitats.
