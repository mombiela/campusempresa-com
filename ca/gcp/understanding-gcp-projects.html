Introducció als Projectes de GCP

Google Cloud Platform (GCP) organitza recursos en projectes, que proporcionen una manera de grup i gestionar recursos relacionats. Comprendre els projectes de GCP és fonamental per utilitzar efectivament els serveis de GCP.

Conceptes Clau

  • Projecte: Un contenidor per a recursos com màquines virtuals, bases de dades i emmagatzematge.
  • ID del Projecte: Un identificador únic per al teu projecte.
  • Nom del Projecte: Un nom llegible per a humans per al teu projecte.
  • Número del Projecte: Un número únic assignat per GCP.
  • Compte de Facturació: Enllaçat a un projecte per gestionar els costos.
  • IAM (Gestió d'Identitat i Accés): Controla l'accés als recursos dins d'un projecte.

Crear un Projecte de GCP

Per començar a utilitzar GCP, necessites crear un projecte. Això es pot fer a través del Consola de GCP, gcloud CLI, o API.

Utilitzant la Consola de GCP

  1. Navega a la Consola de GCP.
  2. Fes clic al desplegable del projecte a la part superior de la pàgina.
  3. Selecciona "Nou Projecte."
  4. Introdueix el nom del projecte, ID, i selecciona un compte de facturació.
  5. Fes clic a "Crear."

Utilitzant gcloud CLI

# Establir un ID de projecte per defecte
PROJECT_ID="my-gcp-project"

# Crear un nou projecte
gcloud projects create $PROJECT_ID

# Enllaçar el projecte a un compte de facturació
BILLING_ACCOUNT=$(gcloud beta billing accounts list --format="value(name)" --limit=1)
gcloud beta billing projects link $PROJECT_ID --billing-account=$BILLING_ACCOUNT

Utilitzant API

from googleapiclient import discovery
from oauth2client.client import GoogleCredentials

credentials = GoogleCredentials.get_application_default()
service = discovery.build('cloudresourcemanager', 'v1', credentials=credentials)

project_body = {
    'name': 'My GCP Project',
    'projectId': 'my-gcp-project'
}

request = service.projects().create(body=project_body)
response = request.execute()

print(response)

Gestionar Projectes de GCP

Un cop creat un projecte, pots gestionar-lo afegint recursos, establint permisos i monitoritzant l'ús.

Afegir Recursos

Es poden afegir recursos com instàncies de Compute Engine, cubs de Cloud Storage i conjunts de dades de BigQuery al teu projecte.

Establir Permisos

Utilitza IAM per controlar qui té accés al teu projecte i què poden fer.

  • Rols: Defineixen quines accions estan permeses.
  • Membres: Usuaris, grups o comptes de servei.
  • Polítiques: Enllacen rols a membres.

Exemple d'afegir un rol a un membre:

gcloud projects add-iam-policy-binding $PROJECT_ID \
    --member="user:example@gmail.com" \
    --role="roles/viewer"

Monitorització i Facturació

Utilitza la Consola de GCP per monitoritzar l'ús de recursos i gestionar la facturació.

  • Stackdriver Monitoring: Monitoritza el rendiment i el temps d'activitat.
  • Informes de Facturació: Fes un seguiment dels costos i estableix pressupostos.

Millors Pràctiques

  • Organitzar Projectes: Utilitza projectes per separar entorns (per exemple, desenvolupament, estadi, producció).
  • Utilitzar Etiquetes: Etiqueta recursos amb etiquetes per a una millor organització i seguiment de costos.
  • Establir Pressupostos i Alertes: Evita costos inesperats establint pressupostos i alertes.

Conclusió

Comprendre i gestionar els projectes de GCP és crucial per utilitzar efectivament Google Cloud Platform. Organitzant recursos en projectes, establint permisos adequats i monitoritzant l'ús, pots assegurar operacions al núvol eficients i segures.
