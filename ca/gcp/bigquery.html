Introducció a BigQuery

BigQuery és un magatzem de dades totalment gestionat i sense servidor que permet una anàlisi escalable de petabytes de dades. Forma part de Google Cloud Platform (GCP) i està dissenyat per fer que l'anàlisi de dades sigui ràpida i fàcil.

Conceptes Clau

  • Arquitectura sense servidor: No cal gestionar la infraestructura.
  • Escalabilitat: Gestiona grans conjunts de dades de manera eficient.
  • Consultes SQL: Utilitza SQL estàndard per consultar dades.
  • Integració: Funciona perfectament amb altres serveis de GCP.

Configuració de BigQuery

Creació d'un Projecte BigQuery

  1. Inicia sessió a Google Cloud Console.

  2. Crea un nou projecte:

    • Navega a la pàgina de selecció de projectes.
    • Fes clic a "Nou Projecte".
    • Introdueix el nom del projecte i el compte de facturació.
    • Fes clic a "Crear".

  3. Habilita l'API de BigQuery:

    • Ves a la biblioteca d'API.
    • Cerca "BigQuery API".
    • Fes clic a "Habilitar".

Creació de Conjunts de Dades i Taules

  • Conjunt de dades: Un contenidor per a taules.
  • Taula: Emmagatzema les dades reals.

Exemple: Creació d'un Conjunt de Dades i una Taula

-- Crear un conjunt de dades
CREATE SCHEMA my_dataset;

-- Crear una taula
CREATE TABLE my_dataset.my_table (
  id INT64,
  name STRING,
  age INT64
);

Consultar Dades a BigQuery

Escriure Consultes SQL

BigQuery utilitza la sintaxi SQL estàndard. Aquí hi ha alguns exemples de consultes bàsiques:

Seleccionar Dades

SELECT name, age
FROM my_dataset.my_table
WHERE age > 21;

Agregar Dades

SELECT COUNT(*) AS total, AVG(age) AS average_age
FROM my_dataset.my_table;

Consultes SQL Avançades

  • JOINs: Combina files de dues o més taules.
  • Subconsultes: Consultes niades dins d'una consulta principal.
  • Funcions de Finestra: Realitzen càlculs a través d'un conjunt de files de la taula.

Exemple: Utilitzar JOINs

SELECT a.name, b.department
FROM my_dataset.employees a
JOIN my_dataset.departments b
ON a.department_id = b.id;

Optimització del Rendiment

Particionament i Agrupament

  • Particionament: Divideix una taula en segments.
  • Agrupament: Organitza les dades basant-se en una o més columnes.

Exemple: Creació d'una Taula Particionada

CREATE TABLE my_dataset.sales (
  sale_id INT64,
  sale_date DATE,
  amount FLOAT64
)
PARTITION BY sale_date;

Millors Pràctiques

  • Utilitza SELECT * amb moderació: Selecciona només les columnes necessàries.
  • Filtra aviat: Aplica filtres tan aviat com sigui possible a la teva consulta.
  • Evita consultes N+1: Utilitza JOINs en lloc de múltiples consultes.

Integració de BigQuery amb Altres Serveis de GCP

Dataflow

  • Pipelines ETL: Utilitza Dataflow per transformar i carregar dades a BigQuery.

Data Studio

  • Visualització: Crea taulers interactius utilitzant Data Studio.

Cloud Functions

  • Automatització: Desencadena operacions de BigQuery utilitzant Cloud Functions.

Seguretat i Control d'Accés

Gestió de Permisos

  • Rols IAM: Assigna rols a usuaris i grups.
  • Permisos de Conjunt de Dades i Taula: Controla l'accés a nivell de conjunt de dades i taula.

Exemple: Concedir Accés

GRANT roles/bigquery.dataViewer
ON my_dataset.my_table
TO 'user:example@example.com';

Conclusió

BigQuery és una eina poderosa per a l'anàlisi de dades a GCP, oferint escalabilitat, facilitat d'ús i integració amb altres serveis de GCP. Entenent els seus conceptes clau, configurant projectes, escrivint consultes eficients i integrant amb altres eines, pots aprofitar BigQuery per obtenir informació valuosa de les teves dades.
