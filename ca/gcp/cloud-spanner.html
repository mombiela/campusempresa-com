Introducció a Cloud Spanner

Cloud Spanner és un servei de base de dades completament gestionat, escalable, distribuït globalment i fortament consistent proporcionat per Google Cloud Platform (GCP). Combina els beneficis de l'estructura de bases de dades relacionals amb l'escala horitzontal no relacional.

Conceptes Clau

  • Completament Gestionat: Google s'encarrega de tot el manteniment, actualitzacions i escalabilitat.
  • Escalabilitat: Escala horitzontalment de manera fluida per manejar càrregues augmentades.
  • Distribució Global: Les dades es poden distribuir a través de múltiples regions.
  • Consistència Forta: Assegura que totes les lectures i escriptures siguin consistents globalment.
  • Suport SQL: Suporta ANSI SQL per a la consulta de dades.

Configuració de Cloud Spanner

Per començar a utilitzar Cloud Spanner, necessites configurar una instància i crear una base de dades.

Passos per Configurar

  1. Crear una Instància de Cloud Spanner:

    • Ves a la Consola de GCP.
    • Navega a Spanner.
    • Fes clic a "Crear Instància".
    • Omple els detalls de la instància (nom, ID de la instància, configuració, nodes).
    • Fes clic a "Crear".

  2. Crear una Base de Dades:

    • Després de crear una instància, fes clic al nom de la instància.
    • Fes clic a "Crear Base de Dades".
    • Proporciona un nom per a la base de dades.
    • Opcionalment, pots definir l'esquema durant la creació.
    • Fes clic a "Crear".

Exemple: Creació d'una Instància i Base de Dades

gcloud spanner instances create my-instance \
    --config=regional-us-central1 \
    --description="My Spanner Instance" \
    --nodes=1

gcloud spanner databases create my-database \
    --instance=my-instance

Disseny de l'Esquema

Dissenyar l'esquema en Cloud Spanner implica definir taules, columnes i relacions.

Exemple d'Esquema

CREATE TABLE Users (
    UserId STRING(36) NOT NULL,
    UserName STRING(128),
    UserEmail STRING(128),
) PRIMARY KEY (UserId);

CREATE TABLE Orders (
    OrderId STRING(36) NOT NULL,
    UserId STRING(36),
    OrderDate TIMESTAMP,
    OrderAmount FLOAT64,
) PRIMARY KEY (OrderId),
  INTERLEAVE IN PARENT Users ON DELETE CASCADE;

Punts Clau

  • Claus Primàries: Essencials per identificar únicament els registres.
  • Intercalat: Ajuda a optimitzar les consultes agrupant físicament les files relacionades.

Consulta de Dades

Cloud Spanner suporta ANSI SQL, facilitant la consulta de dades.

Consultes Bàsiques

-- Seleccionar tots els usuaris
SELECT * FROM Users;

-- Seleccionar columnes específiques
SELECT UserName, UserEmail FROM Users;

-- Unir taules
SELECT Users.UserName, Orders.OrderAmount
FROM Users
JOIN Orders ON Users.UserId = Orders.UserId;

Consultes Avançades

-- Agregació
SELECT UserId, COUNT(*) as OrderCount
FROM Orders
GROUP BY UserId;

-- Filtratge
SELECT * FROM Orders
WHERE OrderAmount > 100.00;

-- Subconsultes
SELECT UserName
FROM Users
WHERE UserId IN (SELECT UserId FROM Orders WHERE OrderAmount > 100.00);

Optimització del Rendiment

Optimitzar el rendiment en Cloud Spanner implica entendre i utilitzar les seves característiques úniques.

Consells per a l'Optimització

  • Disseny de l'Esquema: Dissenya correctament l'esquema amb intercalat i claus primàries.
  • Optimització de Consultes: Utilitza índexs i evita escanejos complets de taules.
  • Equilibri de Càrrega: Distribueix les dades uniformement entre els nodes.

Exemple: Creació d'Índexs

CREATE INDEX UserEmailIndex ON Users(UserEmail);

Còpia de Seguretat i Restauració

Cloud Spanner proporciona mecanismes per fer còpies de seguretat i restaurar dades.

Còpia de Seguretat

gcloud spanner backups create my-backup \
    --instance=my-instance \
    --database=my-database \
    --expire-time="2023-12-31T23:59:59Z"

Restauració

gcloud spanner databases restore my-database-restored \
    --instance=my-instance \
    --backup=my-backup

Conclusió

Cloud Spanner és un servei de base de dades potent i completament gestionat que ofereix l'escalabilitat i la consistència necessàries per a aplicacions modernes. Entenent la seva configuració, disseny d'esquema, capacitats de consulta, optimització del rendiment i funcions de còpia de seguretat/restauració, pots aprofitar efectivament Cloud Spanner per a les teves necessitats de base de dades.
