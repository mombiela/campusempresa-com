Google Cloud Platform's Compute Engine és una potent Infraestructura-com-a-Servei (IaaS) que et permet executar màquines virtuals a la infraestructura de Google. Aquest tema et guiarà des dels conceptes bàsics fins als avançats de Compute Engine, ajudant-te a entendre com aprofitar-lo per a les teves aplicacions.

Introducció a Compute Engine

  • Què és Compute Engine?

    • Compute Engine és un servei que proporciona màquines virtuals (VMs) escalables i d'alt rendiment a Google Cloud Platform.
    • Et permet executar càrregues de treball a gran escala en màquines virtuals allotjades a la infraestructura de Google.

  • Funcionalitats Clau

    • Màquines virtuals personalitzables
    • Tipus de màquines predefinides
    • Discos persistents
    • Equilibri de càrrega global
    • Escalabilitat automàtica
    • Integració amb altres serveis de GCP

Començant amb Compute Engine

  • Configuració de la teva primera instància de VM
    • Pas 1: Crear un Projecte
      • Ves al Consola de GCP.
      • Crea un nou projecte o selecciona'n un d'existent.
    • Pas 2: Habilitar l'API de Compute Engine
      • Navega al tauler de control d'API i Serveis.
      • Habilita l'API de Compute Engine.
    • Pas 3: Crear una Instància de VM
      • Ves a la secció de Compute Engine.
      • Fes clic a "Crear Instància".
      • Configura la instància (nom, regió, zona, tipus de màquina, disc d'arrencada, etc.).
      • Fes clic a "Crear".
# Exemple: Creant una instància de VM utilitzant l'eina de línia de comandes gcloud
gcloud compute instances create my-instance \
    --zone=us-central1-a \
    --machine-type=e2-medium \
    --image-family=debian-10 \
    --image-project=debian-cloud
  • Connectant-te a la teva Instància de VM
    • Utilitza SSH per connectar-te a la teva instància de VM des de la Consola de GCP.
    • Alternativament, utilitza l'eina de línia de comandes gcloud:
# Exemple: Connectant-se a una instància de VM utilitzant gcloud
gcloud compute ssh my-instance --zone=us-central1-a

Gestionant Instàncies de VM

  • Cicle de Vida de la Instància
    • Iniciant i Aturant Instàncies
      • Inicia, atura i reinicia instàncies des de la Consola de GCP o utilitzant comandes gcloud.
# Exemple: Iniciant una instància de VM
gcloud compute instances start my-instance --zone=us-central1-a

# Exemple: Aturant una instància de VM
gcloud compute instances stop my-instance --zone=us-central1-a
  • Eliminant Instàncies
    • Elimina instàncies quan ja no siguin necessàries per evitar costos innecessaris.
# Exemple: Eliminant una instància de VM
gcloud compute instances delete my-instance --zone=us-central1-a
  • Plantilles d'Instància i Grups
    • Plantilles d'Instància
      • Defineix una plantilla per a les instàncies de VM per assegurar la consistència.
    • Grups d'Instàncies Gestionades
      • Utilitza grups d'instàncies gestionades per a l'autoescalabilitat i l'equilibri de càrrega.
# Exemple: Creant una plantilla d'instància
gcloud compute instance-templates create my-template \
    --machine-type=e2-medium \
    --image-family=debian-10 \
    --image-project=debian-cloud

# Exemple: Creant un grup d'instàncies gestionades
gcloud compute instance-groups managed create my-group \
    --base-instance-name=my-instance \
    --template=my-template \
    --size=3 \
    --zone=us-central1-a

Funcionalitats Avançades de Compute Engine

  • Tipus de Màquines Personalitzades
    • Crea tipus de màquines personalitzades per optimitzar el cost i el rendiment per a càrregues de treball específiques.
# Exemple: Creant una instància de VM amb un tipus de màquina personalitzada
gcloud compute instances create custom-instance \
    --custom-cpu=4 \
    --custom-memory=10GB \
    --zone=us-central1-a \
    --image-family=debian-10 \
    --image-project=debian-cloud
  • Discos Persistents
    • Adjunta i gestiona discos persistents per emmagatzemar dades més enllà del cicle de vida d'una instància de VM.
# Exemple: Creant i adjuntant un disc persistent
gcloud compute disks create my-disk --size=200GB --zone=us-central1-a
gcloud compute instances attach-disk my-instance --disk=my-disk --zone=us-central1-a
  • Xarxes
    • Configura VPCs, subxarxes i regles de tallafocs per controlar el trànsit de xarxa cap a i des de les teves instàncies de VM.
# Exemple: Creant una regla de tallafocs per permetre el trànsit SSH
gcloud compute firewall-rules create allow-ssh \
    --allow=tcp:22 \
    --network=default
  • Equilibri de Càrrega
    • Distribueix el trànsit entre múltiples instàncies de VM utilitzant l'equilibri de càrrega global.
# Exemple: Creant un equilibrador de càrrega HTTP global
gcloud compute backend-services create my-backend-service \
    --protocol=HTTP \
    --global

gcloud compute url-maps create my-url-map \
    --default-service=my-backend-service

gcloud compute target-http-proxies create my-http-proxy \
    --url-map=my-url-map

gcloud compute forwarding-rules create my-http-rule \
    --global \
    --target-http-proxy=my-http-proxy \
    --ports=80

Millors Pràctiques

  • Seguretat

    • Utilitza rols IAM i comptes de servei per controlar l'accés.
    • Habilita OS Login per a la gestió d'accés SSH.
    • Actualitza i aplica regularment pegats a les teves instàncies de VM.

  • Gestió de Costos

    • Utilitza descomptes per ús sostingut i contractes d'ús compromès.
    • Monitoritza i optimitza l'ús de recursos.

  • Optimització del Rendiment

    • Utilitza tipus de màquines personalitzades per a càrregues de treball específiques.
    • Aprofita l'autoescalabilitat per a càrregues de treball dinàmiques.

Conclusió

Compute Engine de Google Cloud Platform ofereix un entorn robust i flexible per executar màquines virtuals. Des de la creació de la teva primera instància de VM fins a la gestió de càrregues de treball complexes amb tipus de màquines personalitzades, discos persistents i equilibri de càrrega, Compute Engine proporciona les eines necessàries per construir aplicacions escalables i eficients. Seguint les millors pràctiques de seguretat, gestió de costos i optimització del rendiment, pots maximitzar els beneficis d'utilitzar Compute Engine en els teus projectes.
