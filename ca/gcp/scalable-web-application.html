app = Flask(__name__)

5. Gestió de Bases de Dades

  • Triar el servei de base de dades adequat:
    • Cloud SQL vs. Firestore vs. BigQuery
  • Configuració de Cloud SQL
  • Connexió de la vostra aplicació a Cloud SQL

Exemple: Connexió d'una Aplicació Flask a Cloud SQL

# app.py
import os
import sqlalchemy

db_user = os.environ.get('DB_USER')
db_pass = os.environ.get('DB_PASS')
db_name = os.environ.get('DB_NAME')
db_host = os.environ.get('DB_HOST')

db = sqlalchemy.create_engine(
    sqlalchemy.engine.url.URL(
        drivername='mysql+pymysql',
        username=db_user,
        password=db_pass,
        database=db_name,
        host=db_host
    )
)

@app.route('/users')
def users():
    with db.connect() as conn:
        result = conn.execute('SELECT * FROM users')
        users = [row['name'] for row in result]
    return ', '.join(users)

8. Temes Avançats

  • Utilització de Kubernetes amb GKE
  • Implementació de CI/CD amb Cloud Build
  • Arquitectura sense servidor amb Cloud Functions
  • Xarxes avançades amb VPC

Exemple: Desplegament d'una Aplicació Contenidoritzada amb GKE

# Construir la imatge Docker
docker build -t gcr.io/[PROJECT_ID]/my-app:v1 .

# Pujar la imatge a Container Registry
docker push gcr.io/[PROJECT_ID]/my-app:v1

# Crear un clúster GKE
gcloud container clusters create my-cluster

# Desplegar l'aplicació a GKE
kubectl run my-app --image=gcr.io/[PROJECT_ID]/my-app:v1 --port=8080
kubectl expose deployment my-app --type=LoadBalancer --port 80 --target-port 8080

Conclusió

Construir una aplicació web escalable a GCP implica comprendre i utilitzar diversos serveis i eines proporcionats per la plataforma. Des de la configuració del vostre entorn fins al desplegament i l'escalabilitat de la vostra aplicació, aquest curs ha cobert els passos essencials i les millors pràctiques. Seguint aquestes directrius i exemples, estareu ben equipats per crear aplicacions web robustes i escalables a GCP.
