Introducció a Cloud Build

Google Cloud Build és una plataforma d'integració contínua i lliurament continu (CI/CD) que automatitza el procés de construcció, prova i desplegament d'aplicacions. Admet una àmplia gamma de llenguatges i entorns, cosa que el converteix en una eina versàtil per als desenvolupadors.

Conceptes Clau

  • Triggers de Construcció: Inicien automàticament les construccions basades en esdeveniments com commits de codi.
  • Passos de Construcció: Defineixen la seqüència d'operacions en un procés de construcció.
  • Artifacts de Construcció: Sortides generades pel procés de construcció, com binaris o imatges Docker.
  • Configuració de Construcció: Fitxers YAML o JSON que especifiquen els passos de construcció i l'entorn.

Configuració de Cloud Build

Per començar amb Cloud Build, cal configurar el teu projecte de Google Cloud i habilitar l'API de Cloud Build.

Passos per Configurar

  1. Crear un Projecte de Google Cloud:

    • Ves al Google Cloud Console.
    • Fes clic al desplegable de projectes i selecciona "Nou Projecte".
    • Introdueix un nom de projecte i fes clic a "Crear".

  2. Habilitar l'API de Cloud Build:

    • Navega a la Biblioteca d'APIs.
    • Cerca "Cloud Build API" i fes clic sobre ella.
    • Fes clic a "Habilitar".

  3. Instal·lar el Cloud SDK:

    • Segueix les instruccions a la pàgina d'instal·lació del Google Cloud SDK.

  4. Autenticar-se amb Google Cloud:

    gcloud auth login
gcloud config set project [PROJECT_ID]

Creació d'una Construcció Simple

Creem una configuració de construcció simple per entendre els conceptes bàsics.

Exemple: Construcció d'una Imatge Docker

  1. Crear un fitxer cloudbuild.yaml:

    steps:
- name: 'gcr.io/cloud-builders/docker'
  args: ['build', '-t', 'gcr.io/[PROJECT_ID]/my-app', '.']
images:
- 'gcr.io/[PROJECT_ID]/my-app'

  2. Executar la Construcció:

    gcloud builds submit --config cloudbuild.yaml .

Explicació

  • steps: Defineix les accions a realitzar. Aquí, utilitzem el constructor Docker per construir una imatge.
  • name: Especifica la imatge del constructor.
  • args: Arguments passats al constructor. En aquest cas, estem construint una imatge Docker.
  • images: Especifica les imatges que es pujaran al Container Registry.

Configuracions de Construcció Avançades

A mesura que et familiaritzis amb Cloud Build, podràs crear configuracions de construcció més complexes.

Exemple: Construcció Multi-Passos

steps:
- name: 'gcr.io/cloud-builders/npm'
  args: ['install']
- name: 'gcr.io/cloud-builders/npm'
  args: ['test']
- name: 'gcr.io/cloud-builders/docker'
  args: ['build', '-t', 'gcr.io/[PROJECT_ID]/my-app', '.']
images:
- 'gcr.io/[PROJECT_ID]/my-app'

Explicació

  • npm install: Instal·la les dependències.
  • npm test: Executa les proves.
  • docker build: Construeix la imatge Docker.

Triggers de Construcció

Els triggers de construcció permeten automatitzar les construccions basades en esdeveniments com commits de codi.

Creació d'un Trigger de Construcció

  1. Navegar a Triggers de Cloud Build:

    • Ves a la pàgina de Triggers de Cloud Build.

  2. Crear un Trigger:

    • Fes clic a "Crear Trigger".
    • Selecciona el teu repositori i branca.
    • Especifica el fitxer de configuració de construcció (cloudbuild.yaml).

Exemple: Configuració del Trigger

triggerTemplate:
  projectId: [PROJECT_ID]
  repoName: [REPO_NAME]
  branchName: 'main'

Conclusió

Cloud Build és una eina poderosa per automatitzar els processos de construcció, prova i desplegament. Entenent els conceptes bàsics i explorant gradualment configuracions avançades, pots optimitzar el teu flux de treball de desenvolupament i assegurar construccions consistents i fiables. Comença amb configuracions de construcció simples i incorpora progressivament passos i triggers més complexos per aprofitar al màxim les capacitats de Cloud Build.
