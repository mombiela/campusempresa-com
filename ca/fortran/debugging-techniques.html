La depuració és una habilitat essencial per a qualsevol programador. Aquesta secció cobrirà diverses tècniques de depuració en Fortran, des de nivells bàsics fins a avançats. Al final d'aquest tema, hauríeu de ser capaços d'identificar i corregir errors en el vostre codi Fortran de manera eficient.

Introducció a la Depuració

La depuració és el procés d'identificar, aïllar i corregir errors o bugs en el vostre codi. Una depuració efectiva pot estalviar-vos temps i millorar la qualitat del vostre programari.

Conceptes Clau

  • Errors de Sintaxi: Errors en el codi que violen les regles del llenguatge de programació.
  • Errors d'Execució: Errors que es produeixen mentre el programa s'està executant.
  • Errors Lògics: Errors que fan que el programa funcioni incorrectament però no es bloquegi.

Tècniques Bàsiques de Depuració

Sentències Print

Una de les maneres més senzilles de depurar és utilitzar sentències print per mostrar els valors de les variables en diferents punts del vostre programa.

program debug_example
    implicit none
    integer :: i, sum

    sum = 0
    do i = 1, 10
        sum = sum + i
        print *, 'i = ', i, ' sum = ', sum
    end do
end program debug_example

Comprovació de Valors de Variables

Assegureu-vos que les variables estan inicialitzades i tenen els valors esperats.

program check_variables
    implicit none
    integer :: a, b, c

    a = 5
    b = 10
    c = a + b
    print *, 'a = ', a, ' b = ', b, ' c = ', c
end program check_variables

Tècniques Intermèdies de Depuració

Ús d'Eines de Depuració

La majoria de compiladors Fortran venen amb eines de depuració. Per exemple, gdb (GNU Debugger) es pot utilitzar amb programes Fortran compilats amb gfortran.

Exemple: Ús de gdb

  1. Compileu el programa amb informació de depuració: 
    gfortran -g -o myprogram myprogram.f90
  2. Executeu gdb: 
    gdb ./myprogram
  3. Establiu punts de ruptura i executeu el programa: 
    (gdb) break main
(gdb) run

Anàlisi de Core Dumps

Un core dump és un fitxer que captura la memòria d'un programa en un moment específic, generalment quan el programa es bloqueja. Això es pot analitzar per entendre què va fallar.

ulimit -c unlimited  # Habilitar core dumps
./myprogram          # Executar el programa per generar un core dump
gdb ./myprogram core # Analitzar el core dump amb gdb

Tècniques Avançades de Depuració

Perfilat i Anàlisi de Rendiment

El perfilat ajuda a identificar colls d'ampolla de rendiment en el vostre codi. Eines com gprof es poden utilitzar per a aquest propòsit.

Exemple: Ús de gprof

  1. Compileu el programa amb el perfilat habilitat: 
    gfortran -pg -o myprogram myprogram.f90
  2. Executeu el programa per generar dades de perfilat: 
    ./myprogram
  3. Analitzeu les dades de perfilat: 
    gprof ./myprogram gmon.out > analysis.txt

Comprovació de Memòria

Eines com valgrind poden ajudar a detectar fuites de memòria i altres problemes relacionats amb la memòria.

Exemple: Ús de valgrind

  1. Executeu el programa amb valgrind: 
    valgrind ./myprogram

Depuració de Programes Paral·lels

Els programes Fortran sovint utilitzen biblioteques de computació paral·lela com MPI o OpenMP. La depuració de programes paral·lels pot ser més complexa i pot requerir eines especialitzades com TotalView o DDT.

Conclusió

La depuració és una habilitat crítica que millora amb la pràctica i l'experiència. Utilitzant les tècniques discutides en aquesta secció, podeu identificar i corregir errors de manera efectiva en els vostres programes Fortran. Recordeu començar amb mètodes senzills com les sentències print i avançar gradualment cap a eines més avançades segons sigui necessari. Bona depuració!
