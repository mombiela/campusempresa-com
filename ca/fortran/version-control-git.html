Introducció al Control de Versions

  • Definició: El control de versions és un sistema que registra els canvis en un fitxer o conjunt de fitxers al llarg del temps perquè puguis recuperar versions específiques més tard.
  • Importància: Permet que diverses persones treballin en un projecte simultàniament, fa un seguiment dels canvis i ajuda a gestionar la història del projecte.

Començant amb Git

  • Què és Git?: Git és un sistema de control de versions distribuït dissenyat per gestionar tot, des de projectes petits fins a molt grans, amb velocitat i eficiència.
  • Instal·lació:
    • Windows: Descarrega des de git-scm.com i segueix les instruccions d'instal·lació.
    • macOS: Instal·la via Homebrew amb brew install git.
    • Linux: Utilitza el gestor de paquets, per exemple, sudo apt-get install git per a sistemes basats en Debian.

Comandes bàsiques de Git

  • Configuració de Git: 
    git config --global user.name "El teu Nom"
git config --global user.email "el.teu.email@example.com"
  • Inicialitzar un Repositori: 
    git init
  • Clonar un Repositori: 
    git clone https://github.com/username/repository.git
  • Comprovar l'Estat: 
    git status
  • Afegir Fitxers: 
    git add filename
  • Cometre Canvis: 
    git commit -m "Missatge del commit"
  • Empènyer Canvis: 
    git push origin main

Treballant amb Branques

  • Crear una Branca: 
    git branch new-branch
  • Canviar de Branca: 
    git checkout new-branch
  • Fusionar Branques: 
    git checkout main
git merge new-branch

Col·laborant amb Git

  • Fork d'un Repositori: Fes un fork d'un repositori a GitHub per crear la teva pròpia còpia.
  • Pull Requests: Envia canvis al repositori original mitjançant pull requests.
  • Gestionar Conflictes de Fusió:
    • Identificar el Conflicte: Git marcarà els conflictes en els fitxers.
    • Resoldre el Conflicte: Edita els fitxers per resoldre els conflictes.
    • Cometre la Resolució: 
      git add resolved-file
git commit -m "Resolved merge conflict"

Funcions Avançades de Git

  • Emmagatzemar Canvis: 
    git stash
git stash pop
  • Rebase: 
    git rebase main
  • Etiquetar: 
    git tag -a v1.0 -m "Versió 1.0"
git push origin v1.0

Exemple: Gestionar un Projecte Fortran amb Git

Exemple Pas a Pas

  1. Inicialitzar el Repositori: 
    git init fortran-project
cd fortran-project
  2. Crear un Fitxer Fortran: 
    ! hello.f90
PROGRAM HelloWorld
  PRINT *, "Hello, World!"
END PROGRAM HelloWorld
  3. Afegir i Cometre: 
    git add hello.f90
git commit -m "Add HelloWorld program"
  4. Crear una Nova Branca per a la Funcionalitat: 
    git checkout -b feature-addition
  5. Modificar el Fitxer Fortran: 
    ! hello.f90
PROGRAM HelloWorld
  PRINT *, "Hello, World!"
  PRINT *, "This is a new feature."
END PROGRAM HelloWorld
  6. Afegir i Cometre Canvis: 
    git add hello.f90
git commit -m "Add new feature to HelloWorld program"
  7. Fusionar la Branca de Funcionalitat: 
    git checkout main
git merge feature-addition

Resum

En aquest tema, hem cobert els conceptes bàsics del control de versions amb Git, incloent la instal·lació, comandes bàsiques, branques, col·laboració i funcions avançades. També hem proporcionat un exemple pràctic de com gestionar un projecte Fortran utilitzant Git. En dominar aquestes habilitats, podràs gestionar eficientment el teu codi, col·laborar amb altres i mantenir una història robusta del projecte.
