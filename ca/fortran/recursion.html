Introducció a la Recursió

La recursió és una tècnica de programació on una funció es crida a si mateixa per resoldre un problema. És particularment útil per a problemes que es poden descompondre en subproblemes més petits i similars. En Fortran, la recursió es pot implementar utilitzant funcions recursives.

Conceptes Clau

  • Cas Base: La condició sota la qual la funció recursiva deixa de cridar-se a si mateixa.
  • Cas Recursiu: La part de la funció on la funció es crida a si mateixa amb un argument modificat.
  • Desbordament de Pila: Un risc potencial quan la profunditat de la recursió és massa alta, provocant un ús excessiu de memòria.

Exemple Bàsic de Recursió

Comencem amb un exemple senzill: calcular el factorial d'un número.

Funció Factorial

El factorial d'un número \( n \) (denotat com \( n! \)) és el producte de tots els enters positius menors o iguals a \( n \). El factorial de 0 es defineix com 1.

Exemple de Codi

program factorial_example
    implicit none
    integer :: n, result

    print *, "Enter a number:"
    read *, n

    result = factorial(n)
    print *, "Factorial of ", n, " is ", result
end program factorial_example

recursive function factorial(n) result(fact)
    integer, intent(in) :: n
    integer :: fact

    if (n <= 1) then
        fact = 1
    else
        fact = n * factorial(n - 1)
    end if
end function factorial

Explicació

  • Cas Base: Quan \( n \leq 1 \), la funció retorna 1.
  • Cas Recursiu: La funció es crida a si mateixa amb \( n - 1 \) i multiplica el resultat per \( n \).

Exemple Avançat de Recursió

Exploreu un exemple més complex: la seqüència de Fibonacci.

Seqüència de Fibonacci

La seqüència de Fibonacci és una sèrie de números on cada número és la suma dels dos precedents, normalment començant amb 0 i 1.

Exemple de Codi

program fibonacci_example
    implicit none
    integer :: n, i

    print *, "Enter the number of terms:"
    read *, n

    do i = 0, n-1
        print *, "Fibonacci(", i, ") = ", fibonacci(i)
    end do
end program fibonacci_example

recursive function fibonacci(n) result(fib)
    integer, intent(in) :: n
    integer :: fib

    if (n == 0) then
        fib = 0
    elseif (n == 1) then
        fib = 1
    else
        fib = fibonacci(n - 1) + fibonacci(n - 2)
    end if
end function fibonacci

Explicació

  • Casos Base: Quan \( n = 0 \), la funció retorna 0. Quan \( n = 1 \), la funció retorna 1.
  • Cas Recursiu: La funció es crida a si mateixa amb \( n - 1 \) i \( n - 2 \) i suma els resultats.

Optimització de la Recursió

La recursió pot ser ineficient per a problemes com la seqüència de Fibonacci a causa de càlculs repetits. Una manera d'optimitzar és utilitzant memoització o enfocaments iteratius.

Exemple de Memoització

La memoització implica emmagatzemar els resultats de crides de funcions costoses i reutilitzar-los quan els mateixos inputs es repeteixen.

Exemple de Codi

program fibonacci_memoization_example
    implicit none
    integer, parameter :: max_n = 100
    integer :: n, i
    integer, dimension(0:max_n) :: memo

    memo = -1

    print *, "Enter the number of terms:"
    read *, n

    do i = 0, n-1
        print *, "Fibonacci(", i, ") = ", fibonacci_memo(i, memo)
    end do
end program fibonacci_memoization_example

recursive function fibonacci_memo(n, memo) result(fib)
    integer, intent(in) :: n
    integer, dimension(:), intent(inout) :: memo
    integer :: fib

    if (memo(n) /= -1) then
        fib = memo(n)
    else
        if (n == 0) then
            fib = 0
        elseif (n == 1) then
            fib = 1
        else
            fib = fibonacci_memo(n - 1, memo) + fibonacci_memo(n - 2, memo)
        end if
        memo(n) = fib
    end if
end function fibonacci_memo

Explicació

  • Array Memo: Un array memo s'utilitza per emmagatzemar els resultats dels càlculs de Fibonacci.
  • Comprovar Memo: Abans de realitzar una crida recursiva, la funció comprova si el resultat ja és a l'array memo.

Conclusió

La recursió és una eina poderosa en Fortran per resoldre problemes que es poden descompondre en subproblemes més petits. No obstant això, és important gestionar correctament els casos base per evitar recursions infinites i possibles desbordaments de pila. Per a problemes complexos, optimitzar la recursió amb tècniques com la memoització pot millorar significativament el rendiment. Comprenent i aplicant aquests conceptes, podeu utilitzar efectivament la recursió en els vostres programes Fortran.
