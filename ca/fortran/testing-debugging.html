Proves i Depuració en Fortran

Introducció

Les proves i la depuració són aspectes crucials del desenvolupament de programari, assegurant que els teus programes Fortran s'executin correctament i eficientment. Aquesta secció cobrirà els conceptes bàsics de les proves i la depuració, progressant des de nivells principiants fins a avançats.

Conceptes Bàsics de les Proves

Què són les Proves?

  • Definició: Les proves són el procés d'executar un programa amb la intenció de trobar errors.
  • Propòsit: Assegurar que el programa es comporti com s'espera sota diverses condicions.

Tipus de Proves

  • Proves Unitàries: Provar components o funcions individuals.
  • Proves d'Integració: Provar parts combinades d'una aplicació per assegurar que funcionin juntes.
  • Proves de Sistema: Provar el sistema complet com un tot.
  • Proves d'Acceptació: Provar per assegurar que el sistema compleixi els requisits.

Escriure Casos de Prova

  • Identificar Escenaris de Prova: Determinar què necessita ser provat.
  • Crear Dades de Prova: Preparar dades d'entrada per a les proves.
  • Definir Resultats Esperats: Especificar quin hauria de ser el resultat correcte.

Exemple d'un Cas de Prova Simple

program test_addition
  implicit none
  integer :: result

  result = add(2, 3)
  if (result /= 5) then
    print *, "Test Failed: Expected 5, got ", result
  else
    print *, "Test Passed"
  end if
end program test_addition

function add(a, b)
  integer :: add
  integer, intent(in) :: a, b
  add = a + b
end function add
  • Explicació: Aquest programa prova la funció add. Comprova si sumar 2 i 3 resulta en 5.

Conceptes Bàsics de la Depuració

Què és la Depuració?

  • Definició: La depuració és el procés d'identificar, analitzar i eliminar errors d'un programa.
  • Propòsit: Corregir errors i assegurar que el programa s'executi correctament.

Tècniques Comunes de Depuració

  • Sentències de Print: Inserir sentències de print per mostrar valors de variables i el flux del programa.
  • Ús de Depuradors: Utilitzar eines com GDB (GNU Debugger) per a una depuració més avançada.
  • Revisió de Codi: Revisar el codi amb companys per trobar possibles problemes.

Exemple de Depuració amb Sentències de Print

program debug_example
  implicit none
  integer :: a, b, result

  a = 5
  b = 0
  print *, "Before division: a =", a, ", b =", b
  result = a / b
  print *, "After division: result =", result
end program debug_example
  • Explicació: Aquest programa fallarà a causa de la divisió per zero. Les sentències de print ajuden a identificar els valors de a i b abans que ocorri l'error.

Tècniques Avançades de Proves

Proves Automatitzades

  • Definició: Ús de scripts i eines per executar proves automàticament.
  • Eines: Frameworks com pFUnit per a Fortran.

Exemple de Proves Automatitzades amb pFUnit

module test_addition
  use pfunit
  implicit none

  @test
  subroutine test_add
    integer :: result
    result = add(2, 3)
    @assertEqual(result, 5)
  end subroutine test_add

  function add(a, b)
    integer :: add
    integer, intent(in) :: a, b
    add = a + b
  end function add
end module test_addition
  • Explicació: Aquest mòdul utilitza pFUnit per automatitzar la prova de la funció add.

Tècniques Avançades de Depuració

Ús de GDB amb Fortran

  • Establir Punts de Trencament: Pausar l'execució en línies específiques.
  • Inspeccionar Variables: Comprovar els valors de les variables en diferents punts.
  • Passar per Codi: Executar el codi línia per línia per trobar errors.

Exemple d'Ús de GDB

  1. Compilar amb Informació de Depuració: 
    gfortran -g -o debug_example debug_example.f90
  2. Executar GDB: 
    gdb ./debug_example
  3. Establir Punt de Trencament i Executar: 
    (gdb) break debug_example.f90:6
(gdb) run

Conclusió

Les proves i la depuració són habilitats essencials per a qualsevol programador de Fortran. Començant amb tècniques bàsiques com escriure casos de prova i utilitzar sentències de print, pots progressar a mètodes més avançats com les proves automatitzades amb pFUnit i l'ús de GDB per a la depuració. En dominar aquestes tècniques, pots assegurar que els teus programes Fortran siguin robustos, fiables i eficients.
