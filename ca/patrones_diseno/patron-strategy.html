Introducció al Patró Strategy

El Patró Strategy és un patró de disseny de comportament que permet definir una família d'algoritmes, encapsular cadascun d'ells i fer-los intercanviables. Aquest patró permet que l'algoritme variï independentment dels clients que l'utilitzen.

Conceptes Clau

  • Encapsulació d'Algoritmes: Cada algoritme s'encapsula en la seva pròpia classe.
  • Intercanviabilitat: Els algoritmes poden ser intercanviats sense modificar el codi del client.
  • Context: La classe que utilitza l'algoritme i que pot canviar d'estratègia en temps d'execució.

Estructura del Patró Strategy

El Patró Strategy es compon dels següents components:

  1. Estratègia (Strategy): Interfície comuna per a tots els algoritmes.
  2. Estratègies Concretes (ConcreteStrategy): Implementacions específiques de la interfície d'estratègia.
  3. Context (Context): Classe que utilitza una estratègia concreta.

Diagrama UML

| Component | Descripció | |------------------|-----------------------------------------------------------------------------| | Strategy | Interfície comuna per a tots els algoritmes. | | ConcreteStrategyA | Implementació concreta de l'estratègia A. | | ConcreteStrategyB | Implementació concreta de l'estratègia B. | | Context | Classe que utilitza una estratègia concreta i pot canviar-la en temps d'execució. |

Exemple de Codi en Python

A continuació, es presenta un exemple de com implementar el Patró Strategy en Python.

Definició de la Interfície d'Estratègia

from abc import ABC, abstractmethod

class Strategy(ABC):
    @abstractmethod
    def execute(self, data):
        pass

Implementacions Concretes d'Estratègia

class ConcreteStrategyA(Strategy):
    def execute(self, data):
        return sorted(data)

class ConcreteStrategyB(Strategy):
    def execute(self, data):
        return sorted(data, reverse=True)

Classe Context

class Context:
    def __init__(self, strategy: Strategy):
        self._strategy = strategy

    def set_strategy(self, strategy: Strategy):
        self._strategy = strategy

    def execute_strategy(self, data):
        return self._strategy.execute(data)

Ús del Patró Strategy

if __name__ == "__main__":
    data = [5, 2, 9, 1]

    context = Context(ConcreteStrategyA())
    print("Ordre Ascendent:", context.execute_strategy(data))

    context.set_strategy(ConcreteStrategyB())
    print("Ordre Descendent:", context.execute_strategy(data))

Avantatges i Desavantatges

Avantatges

  • Flexibilitat: Permet canviar d'algoritme en temps d'execució.
  • Manteniment: Facilita l'addició de noves estratègies sense modificar el codi existent.
  • Reutilització: Les estratègies poden ser reutilitzades en diferents contextos.

Desavantatges

  • Complexitat: Augmenta la complexitat del codi a causa de la introducció de múltiples classes.
  • Sobrecàrrega: Pot introduir una lleugera sobrecàrrega a causa de la delegació de tasques.

Conclusió

El Patró Strategy és una poderosa eina per manejar algoritmes intercanviables i encapsular comportaments específics. En utilitzar aquest patró, els desenvolupadors poden crear sistemes més flexibles i mantenibles, permetent que els algoritmes variïn independentment dels clients que els utilitzen. No obstant això, és important considerar la complexitat addicional que pot introduir en el disseny del sistema.
