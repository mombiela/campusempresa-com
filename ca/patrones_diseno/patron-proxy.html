Introducció al Patró Proxy

El patró Proxy és un patró de disseny estructural que proporciona un substitut o marcador de posició per a un altre objecte. Un proxy controla l'accés a un objecte, permetent realitzar certes accions abans o després de la sol·licitud a l'objecte real.

Objectius del Patró Proxy

  • Control d'accés: Restringir l'accés a certs mètodes o dades.
  • Optimització: Carregar objectes pesats només quan es necessiten.
  • Seguretat: Protegir l'objecte real d'accés no autoritzat.
  • Registre i monitoratge: Registrar les sol·licituds i respostes per a auditoria o monitoratge.

Tipus de Proxy

  • Proxy Virtual: Gestiona la creació i l'accés a objectes costosos en termes de recursos.
  • Proxy de Protecció: Controla l'accés a un objecte, permetent diferents permisos per a diferents usuaris.
  • Proxy Remot: Representa un objecte que resideix en un espai d'adreces diferent (per exemple, en una màquina remota).
  • Proxy de Caché: Emmagatzema els resultats d'operacions costoses per evitar repetir-les.

Estructura del Patró Proxy

El patró Proxy generalment involucra els següents components:

  • Subject (Subjecte): Defineix la interfície comuna per al RealSubject i el Proxy.
  • RealSubject (Subjecte Real): L'objecte real que el proxy representa.
  • Proxy: Manté una referència al RealSubject i controla l'accés a ell.

Diagrama UML

| Component | Descripció | |--------------|-----------------------------------------------------------------------------| | Subject | Interfície comuna per a RealSubject i Proxy. | | RealSubject | L'objecte real que s'està representant. | | Proxy | Controla l'accés al RealSubject i pot afegir funcionalitat addicional. |

Exemple de Codi

A continuació, es presenta un exemple en Java per il·lustrar el patró Proxy:

// Interfície comuna
public interface Image {
    void display();
}

// Subjecte Real
public class RealImage implements Image {
    private String fileName;

    public RealImage(String fileName) {
        this.fileName = fileName;
        loadFromDisk();
    }

    private void loadFromDisk() {
        System.out.println("Loading " + fileName);
    }

    @Override
    public void display() {
        System.out.println("Displaying " + fileName);
    }
}

// Proxy
public class ProxyImage implements Image {
    private RealImage realImage;
    private String fileName;

    public ProxyImage(String fileName) {
        this.fileName = fileName;
    }

    @Override
    public void display() {
        if (realImage == null) {
            realImage = new RealImage(fileName);
        }
        realImage.display();
    }
}

// Client
public class ProxyPatternDemo {
    public static void main(String[] args) {
        Image image = new ProxyImage("test_image.jpg");

        // La imatge es carregarà des del disc
        image.display();
        System.out.println("");

        // La imatge no es carregarà des del disc
        image.display();
    }
}

Explicació del Codi

  • Interfície Image: Defineix el mètode display() que serà implementat per RealImage i ProxyImage.
  • RealImage: Implementa la interfície Image i representa l'objecte real que es carrega des del disc.
  • ProxyImage: Implementa la interfície Image i controla l'accés a RealImage. Només carrega RealImage quan és necessari.
  • ProxyPatternDemo: Classe client que utilitza ProxyImage per accedir a RealImage.

Avantatges i Desavantatges

Avantatges

  • Control d'accés: Permet controlar l'accés a l'objecte real.
  • Optimització de recursos: Carrega objectes pesats només quan són necessaris.
  • Seguretat: Pot afegir capes de seguretat addicionals.
  • Monitoratge i registre: Facilita el registre de les sol·licituds i respostes.

Desavantatges

  • Complexitat addicional: Afegeix complexitat al sistema a causa de la introducció de noves classes.
  • Rendiment: Pot introduir una lleugera sobrecàrrega a causa de la delegació de sol·licituds.

Conclusió

El patró Proxy és una eina poderosa per controlar l'accés a objectes, optimitzar l'ús de recursos i afegir capes de seguretat i monitoratge. No obstant això, ha de ser utilitzat amb cura per evitar afegir complexitat innecessària al sistema. En comprendre i aplicar correctament aquest patró, els desenvolupadors poden crear sistemes més eficients i segurs.
