Introducció

El Patró Observer és un dels patrons de disseny de comportament més utilitzats en el desenvolupament de programari. Aquest patró defineix una relació de dependència un-a-molts entre objectes, de manera que quan un objecte canvia el seu estat, tots els seus dependents són notificats i actualitzats automàticament.

Conceptes Clau

  • Subjecte (Subject): L'objecte que manté una llista de dependents (observadors) i notifica a aquests quan el seu estat canvia.
  • Observador (Observer): L'objecte que desitja ser notificat quan l'estat del subjecte canvia.
  • Notificació: El mecanisme mitjançant el qual el subjecte informa els observadors sobre els canvis en el seu estat.

Estructura

El patró Observer generalment es compon de les següents classes:

| Component | Descripció | |-----------|-------------| | Subject | Manté una llista d'observadors i proporciona mètodes per afegir, eliminar i notificar observadors. | | Observer | Defineix una interfície d'actualització que és implementada per tots els observadors concrets. | | ConcreteSubject | Implementa el Subject i emmagatzema l'estat d'interès per als observadors. | | ConcreteObserver | Implementa la interfície Observer i manté una referència al ConcreteSubject per sincronitzar el seu estat. |

Exemple de Codi

A continuació, es presenta un exemple en Python per il·lustrar el Patró Observer:

class Subject:
    def __init__(self):
        self._observers = []

    def attach(self, observer):
        self._observers.append(observer)

    def detach(self, observer):
        self._observers.remove(observer)

    def notify(self):
        for observer in self._observers:
            observer.update(self)

class Observer:
    def update(self, subject):
        pass

class ConcreteSubject(Subject):
    def __init__(self):
        super().__init__()
        self._state = None

    @property
    def state(self):
        return self._state

    @state.setter
    def state(self, value):
        self._state = value
        self.notify()

class ConcreteObserver(Observer):
    def update(self, subject):
        print(f"Observer: Reacted to the state change to {subject.state}")

# Ús del patró Observer
subject = ConcreteSubject()
observer1 = ConcreteObserver()
observer2 = ConcreteObserver()

subject.attach(observer1)
subject.attach(observer2)

subject.state = 'State 1'
subject.state = 'State 2'

Explicació del Codi

  1. Subject: Classe base que manté una llista d'observadors i proporciona mètodes per afegir (attach), eliminar (detach) i notificar (notify) observadors.
  2. Observer: Classe base que defineix la interfície update que ha de ser implementada pels observadors concrets.
  3. ConcreteSubject: Estén Subject i afegeix un estat que, quan canvia, notifica a tots els observadors.
  4. ConcreteObserver: Implementa la interfície Observer i reacciona als canvis en l'estat del ConcreteSubject.

Avantatges i Desavantatges

Avantatges

  • Desacoblament: Els subjectes i observadors estan desacoblats, cosa que facilita la reutilització i el manteniment del codi.
  • Flexibilitat: Els observadors poden ser afegits o eliminats en temps d'execució.

Desavantatges

  • Complexitat: Pot afegir complexitat al sistema a causa de la gestió de múltiples dependències.
  • Rendiment: Si hi ha molts observadors, la notificació pot tornar-se costosa en termes de rendiment.

Conclusió

El Patró Observer és una eina poderosa per gestionar la comunicació entre objectes en un sistema, especialment quan es requereix que múltiples components reaccionin a canvis en l'estat d'un altre component. En entendre i aplicar aquest patró, els desenvolupadors poden crear sistemes més flexibles i mantenibles.
