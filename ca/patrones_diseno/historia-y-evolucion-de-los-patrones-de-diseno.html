Introducció

Els patrons de disseny són solucions reutilitzables a problemes comuns en el disseny de programari. Aquest tema explora la història i evolució dels patrons de disseny, proporcionant una base sòlida per entendre la seva importància i aplicació en el desenvolupament de programari modern.

Orígens dels Patrons de Disseny

  • Christopher Alexander (1977): El concepte de patrons de disseny es va originar en l'arquitectura amb Christopher Alexander, qui va introduir la idea en el seu llibre "A Pattern Language". Alexander va descriure patrons com solucions a problemes recurrents en el disseny arquitectònic.
  • Adopció en Programari (1980s-1990s): La idea va ser adoptada per la comunitat de programari a la dècada de 1980 i 1990. Els patrons de disseny en programari es van popularitzar amb la publicació del llibre "Design Patterns: Elements of Reusable Object-Oriented Software" per Erich Gamma, Richard Helm, Ralph Johnson i John Vlissides, coneguts com la "Banda dels Quatre" (Gang of Four, GoF).

La Banda dels Quatre (Gang of Four, GoF)

  • Publicació del Llibre (1994): El llibre de la GoF és considerat la bíblia dels patrons de disseny en programari. Va introduir 23 patrons de disseny que es classifiquen en tres categories principals:
    • Patrons Creacionals: Tracten amb la creació d'objectes.
    • Patrons Estructurals: Tracten amb la composició de classes o objectes.
    • Patrons de Comportament: Tracten amb la interacció i responsabilitat entre objectes.

Exemple d'un Patró de Disseny: Singleton

El patró Singleton assegura que una classe tingui només una instància i proporciona un punt d'accés global a ella.

public class Singleton {
    private static Singleton instance;

    private Singleton() {
        // Constructor privat per evitar instanciació
    }

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
  • Explicació:
    • Constructor Privat: Evita que altres classes creïn una nova instància.
    • Mètode estàtic getInstance: Proporciona un punt d'accés global a la instància única.

Evolució i Noves Tendències

  • Patrons de Disseny en la Programació Funcional: Amb l'auge de la programació funcional, han sorgit nous patrons que s'adapten a aquest paradigma, com el patró de funció pura i el patró de composició de funcions.
  • Patrons de Disseny en Arquitectures Modernes: En arquitectures modernes com Microserveis i Serverless, els patrons de disseny han evolucionat per abordar problemes específics d'aquestes arquitectures, com el patró Circuit Breaker i el patró Event Sourcing.

Comparació de Patrons Clàssics i Moderns

| Característica | Patrons Clàssics (GoF) | Patrons Moderns (Microserveis/Serverless) | |-------------------------------|-------------------------|-------------------------------------------| | Enfocament | Orientat a objectes | Orientat a serveis | | Exemples Comuns | Singleton, Factory | Circuit Breaker, Event Sourcing | | Escalabilitat | Limitada | Alta | | Desplegament | Monolític | Distribuït |

Conclusió

La història i evolució dels patrons de disseny reflecteixen l'adaptació contínua del desenvolupament de programari a nous desafiaments i paradigmes. Des dels seus orígens en l'arquitectura fins a la seva adopció i expansió en el programari, els patrons de disseny han demostrat ser eines valuoses per crear solucions robustes i reutilitzables. Entendre aquesta evolució ajuda els professionals de TI a aplicar aquests patrons de manera efectiva en els seus projectes actuals i futurs.
