Introducció

El Patró Decorator és un patró de disseny estructural que permet afegir funcionalitats a un objecte de manera dinàmica. Aquest patró és útil quan volem estendre les capacitats d'una classe sense modificar el seu codi original, promovent així el principi d'obert/tancat (Open/Closed Principle) de SOLID.

Conceptes Clau

  • Composició sobre herència: El Patró Decorator afavoreix la composició en lloc de la herència per estendre les funcionalitats dels objectes.
  • Interfície comuna: Tant l'objecte original com els decoradors implementen una interfície comuna, la qual cosa permet que els decoradors puguin ser intercanviables.
  • Encapsulament: Els decoradors encapsulen l'objecte original i poden afegir o modificar el seu comportament.

Estructura

El Patró Decorator es compon dels següents elements:

  • Component (Component): Defineix la interfície comuna per als objectes que poden tenir responsabilitats addicionals.
  • Component Concret (ConcreteComponent): Implementa la interfície del Component. És l'objecte al qual se li poden afegir responsabilitats addicionals.
  • Decorador (Decorator): Manté una referència al Component i defineix una interfície que compleix amb la del Component.
  • Decorador Concret (ConcreteDecorator): Estén la funcionalitat del Component, afegint responsabilitats addicionals.

Exemple de Codi

A continuació, es mostra un exemple en Python per il·lustrar com funciona el Patró Decorator.

from abc import ABC, abstractmethod

# Component
class Beverage(ABC):
    @abstractmethod
    def cost(self) -> float:
        pass

# Component Concret
class Espresso(Beverage):
    def cost(self) -> float:
        return 1.99

# Decorador
class AddonDecorator(Beverage, ABC):
    def __init__(self, beverage: Beverage):
        self._beverage = beverage

# Decorador Concret
class Milk(AddonDecorator):
    def cost(self) -> float:
        return self._beverage.cost() + 0.50

class Mocha(AddonDecorator):
    def cost(self) -> float:
        return self._beverage.cost() + 0.75

# Ús del Patró Decorator
beverage = Espresso()
print(f"Cost of Espresso: ${beverage.cost()}")

beverage_with_milk = Milk(beverage)
print(f"Cost of Espresso with Milk: ${beverage_with_milk.cost()}")

beverage_with_milk_and_mocha = Mocha(beverage_with_milk)
print(f"Cost of Espresso with Milk and Mocha: ${beverage_with_milk_and_mocha.cost()}")

Explicació del Codi

  1. Component: Beverage és una classe abstracta que defineix el mètode cost.
  2. Component Concret: Espresso és una implementació concreta de Beverage.
  3. Decorador: AddonDecorator és una classe abstracta que implementa Beverage i conté una referència a un objecte Beverage.
  4. Decoradors Concrets: Milk i Mocha són decoradors concrets que afegeixen funcionalitats addicionals a l'objecte Beverage.

Comparació: Herència vs Decorator

| Característica | Herència | Decorator | |----------------------|-----------------------------------|------------------------------------| | Extensibilitat | Limitada a la jerarquia de classes | Molt flexible, es poden combinar múltiples decoradors | | Complexitat | Pot tornar-se complexa amb moltes subclasses | Més senzilla, ja que s'afegeixen funcionalitats de manera modular | | Reutilització | Menys reutilitzable | Alta reutilització de decoradors |

Conclusió

El Patró Decorator és una poderosa eina per afegir funcionalitats als objectes de manera dinàmica i flexible. En afavorir la composició sobre la herència, aquest patró permet crear sistemes més modulars i fàcils de mantenir. És especialment útil en situacions on les funcionalitats addicionals poden ser combinades de diferents maneres, proporcionant una gran flexibilitat en el disseny del programari.
