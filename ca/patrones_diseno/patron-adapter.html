Introducció

El Patró Adapter és un dels patrons de disseny estructurals que permet que dues interfícies incompatibles treballin juntes. Aquest patró actua com un pont entre dues classes, permetent que una classe existent funcioni amb altres sense modificar el seu codi font.

Objectius

  • Comprendre el propòsit i la utilitat del Patró Adapter.
  • Aprendre a implementar el Patró Adapter en un projecte.
  • Identificar situacions en les quals el Patró Adapter és aplicable.

Conceptes Clau

  • Adapter: Classe que converteix la interfície d'una classe en una altra interfície que el client espera.
  • Client: Classe que interactua amb l'Adapter.
  • Adaptee: Classe existent que necessita ser adaptada.
  • Target: Interfície esperada pel client.

Exemple de Codi

Exemple en Java

Suposem que tenim una classe Adaptee que té un mètode specificRequest(), i volem que sigui compatible amb una interfície Target que té un mètode request().

Adaptee.java

public class Adaptee {
    public void specificRequest() {
        System.out.println("Called specificRequest()");
    }
}

Target.java

public interface Target {
    void request();
}

Adapter.java

public class Adapter implements Target {
    private Adaptee adaptee;

    public Adapter(Adaptee adaptee) {
        this.adaptee = adaptee;
    }

    @Override
    public void request() {
        adaptee.specificRequest();
    }
}

Client.java

public class Client {
    public static void main(String[] args) {
        Adaptee adaptee = new Adaptee();
        Target target = new Adapter(adaptee);
        target.request();
    }
}

Explicació del Codi

  • Adaptee: Classe existent amb el mètode specificRequest().
  • Target: Interfície que defineix el mètode request().
  • Adapter: Classe que implementa la interfície Target i tradueix la crida al mètode specificRequest() de Adaptee.
  • Client: Classe que utilitza la interfície Target per cridar el mètode request().

Avantatges i Desavantatges

Avantatges

  • Reutilització de Codi: Permet reutilitzar classes existents sense modificar-les.
  • Flexibilitat: Facilita la integració de classes amb interfícies incompatibles.

Desavantatges

  • Complexitat: Pot afegir complexitat addicional al sistema.
  • Rendiment: Pot introduir una lleugera sobrecàrrega a causa de la traducció de crides.

Casos d'Ús

  • Integració de Llibreries: Quan es necessita integrar una llibreria de tercers amb una interfície específica.
  • Sistemes Llegats: Quan es necessita que un sistema llegat funcioni amb noves interfícies.

Conclusió

El Patró Adapter és una eina poderosa per resoldre problemes d'incompatibilitat d'interfícies en el disseny de programari. En entendre i aplicar aquest patró, els desenvolupadors poden crear sistemes més flexibles i reutilitzables, facilitant la integració de components i l'evolució del programari.

Aquest tema proporciona una base sòlida per comprendre i aplicar el Patró Adapter en projectes de programari. Assegura't de practicar amb diferents exemples i casos d'ús per dominar aquest patró de disseny.
