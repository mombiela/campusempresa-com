Els patrons de disseny són solucions reutilitzables a problemes comuns en el disseny de programari. Es classifiquen en tres categories principals: Creacionals, Estructurals i de Comportament. Cada categoria aborda un aspecte diferent del disseny de programari i proporciona un conjunt de patrons que poden ser aplicats en diverses situacions.

Patrons Creacionals

Els patrons creacionals se centren en la forma en què es creen els objectes. Aquests patrons ajuden a fer el sistema independent de com es creen, componen i representen els objectes.

  • Singleton: Garanteix que una classe tingui una única instància i proporciona un punt d'accés global a ella.
  • Factory Method: Defineix una interfície per crear un objecte, però permet a les subclasses alterar el tipus d'objectes que es crearan.
  • Abstract Factory: Proporciona una interfície per crear famílies d'objectes relacionats o dependents sense especificar les seves classes concretes.
  • Builder: Separa la construcció d'un objecte complex de la seva representació, permetent que el mateix procés de construcció creï diferents representacions.
  • Prototype: Permet crear nous objectes copiant un objecte existent, conegut com a prototip.

Exemple de Singleton en Python

class Singleton:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super(Singleton, cls).__new__(cls)
        return cls._instance

# Ús del patró Singleton
singleton1 = Singleton()
singleton2 = Singleton()

print(singleton1 is singleton2)  # True

Patrons Estructurals

Els patrons estructurals es preocupen de la composició de classes i objectes. Aquests patrons ajuden a garantir que si una part d'un sistema canvia, l'estructura general del sistema no es vegi afectada.

  • Adapter: Permet que les interfícies incompatibles treballin juntes.
  • Bridge: Desacobla una abstracció de la seva implementació, permetent que ambdues variïn independentment.
  • Composite: Compon objectes en estructures d'arbre per representar jerarquies part-tot.
  • Decorator: Afegeix responsabilitats addicionals a un objecte de manera dinàmica.
  • Facade: Proporciona una interfície simplificada a un conjunt d'interfícies en un subsistema.
  • Flyweight: Utilitza l'ús compartit per suportar eficientment grans quantitats d'objectes de gra fi.
  • Proxy: Proporciona un substitut o marcador de posició per a un altre objecte per controlar l'accés a ell.

Exemple d'Adapter en Python

class EuropeanSocket:
    def voltage(self):
        return 230

class USASocket:
    def voltage(self):
        return 120

class Adapter:
    def __init__(self, socket):
        self.socket = socket

    def voltage(self):
        return self.socket.voltage()

# Ús del patró Adapter
european_socket = EuropeanSocket()
adapter = Adapter(european_socket)
print(adapter.voltage())  # 230

Patrons de Comportament

Els patrons de comportament se centren en la comunicació entre objectes. Aquests patrons ajuden a definir com els objectes interactuen i es comuniquen entre si.

  • Chain of Responsibility: Permet que més d'un objecte gestioni una sol·licitud passant la sol·licitud al llarg d'una cadena de manejadors.
  • Command: Encapsula una sol·licitud com un objecte, permetent parametritzar els clients amb diferents sol·licituds, cues o registres de sol·licituds.
  • Interpreter: Donat un llenguatge, defineix una representació per a la seva gramàtica juntament amb un intèrpret que utilitza la representació per interpretar oracions en el llenguatge.
  • Iterator: Proporciona una forma d'accedir seqüencialment als elements d'un objecte agregat sense exposar la seva representació subjacent.
  • Mediator: Defineix un objecte que encapsula com un conjunt d'objectes interactuen.
  • Memento: Sense violar la encapsulació, captura i externalitza l'estat intern d'un objecte perquè l'objecte pugui ser restaurat a aquest estat més tard.
  • Observer: Defineix una dependència d'un a molts entre objectes perquè quan un objecte canviï d'estat, tots els seus dependents siguin notificats i actualitzats automàticament.
  • State: Permet a un objecte alterar el seu comportament quan el seu estat intern canvia.
  • Strategy: Defineix una família d'algoritmes, encapsula cadascun i els fa intercanviables.
  • Template Method: Defineix l'esquelet d'un algoritme en una operació, diferint alguns passos a les subclasses.
  • Visitor: Representa una operació que es farà en els elements d'una estructura d'objectes.

Exemple d'Observer en Python

class Subject:
    def __init__(self):
        self._observers = []

    def attach(self, observer):
        self._observers.append(observer)

    def detach(self, observer):
        self._observers.remove(observer)

    def notify(self):
        for observer in self._observers:
            observer.update(self)

class Observer:
    def update(self, subject):
        pass

class ConcreteObserver(Observer):
    def update(self, subject):
        print("Observer notified")

# Ús del patró Observer
subject = Subject()
observer = ConcreteObserver()
subject.attach(observer)
subject.notify()  # Observer notified

Conclusió

La classificació dels patrons de disseny en Creacionals, Estructurals i de Comportament proporciona una forma organitzada d'abordar problemes comuns en el disseny de programari. Cada categoria aborda diferents aspectes del disseny, des de la creació d'objectes fins a l'estructura i la comunicació entre ells. Comprendre i aplicar aquests patrons pot millorar significativament la qualitat i la mantenibilitat del programari.
