Introducció

El Patró Iterator és un dels patrons de disseny de comportament més utilitzats en el desenvolupament de programari. El seu propòsit principal és proporcionar una forma d'accedir seqüencialment als elements d'una col·lecció sense exposar la seva representació subjacent.

Objectius del Patró Iterator

  • Separació de responsabilitats: Permet que la col·lecció i l'iterador tinguin responsabilitats separades.
  • Accés uniforme: Proporciona una interfície uniforme per recórrer diferents tipus de col·leccions.
  • Encapsulament: Oculta la complexitat de l'estructura interna de la col·lecció.

Components del Patró Iterator

El Patró Iterator generalment es compon dels següents elements:

  • Iterador (Iterator): Defineix la interfície per accedir i recórrer els elements.
  • Iterador Concret (Concrete Iterator): Implementa la interfície de l'iterador i manté l'estat de la posició actual en la col·lecció.
  • Col·lecció (Aggregate): Defineix una interfície per crear un iterador.
  • Col·lecció Concreta (Concrete Aggregate): Implementa la interfície de la col·lecció i retorna una instància de l'iterador concret.

Exemple d'Implementació en Java

A continuació, es presenta un exemple de com implementar el Patró Iterator en Java.

Interfície de l'Iterador

public interface Iterator {
    boolean hasNext();
    Object next();
}

Iterador Concret

public class ConcreteIterator implements Iterator {
    private List<Object> collection;
    private int position = 0;

    public ConcreteIterator(List<Object> collection) {
        this.collection = collection;
    }

    @Override
    public boolean hasNext() {
        return position < collection.size();
    }

    @Override
    public Object next() {
        if (this.hasNext()) {
            return collection.get(position++);
        }
        return null;
    }
}

Interfície de la Col·lecció

public interface Aggregate {
    Iterator createIterator();
}

Col·lecció Concreta

import java.util.List;
import java.util.ArrayList;

public class ConcreteAggregate implements Aggregate {
    private List<Object> items = new ArrayList<>();

    public void addItem(Object item) {
        items.add(item);
    }

    @Override
    public Iterator createIterator() {
        return new ConcreteIterator(items);
    }
}

Ús del Patró Iterator

public class Main {
    public static void main(String[] args) {
        ConcreteAggregate collection = new ConcreteAggregate();
        collection.addItem("Item 1");
        collection.addItem("Item 2");
        collection.addItem("Item 3");

        Iterator iterator = collection.createIterator();
        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }
    }
}

Avantatges i Desavantatges

Avantatges

  • Simplicitat: Facilita el recorregut de col·leccions sense necessitat de conèixer la seva estructura interna.
  • Flexibilitat: Permet canviar la col·lecció subjacent sense modificar el codi que la recorre.
  • Reusabilitat: Els iteradors poden ser reutilitzats per a diferents col·leccions.

Desavantatges

  • Sobrecàrrega: Pot afegir complexitat addicional en casos on no és necessari.
  • Rendiment: En alguns casos, pot afectar el rendiment a causa de la creació de múltiples objectes iteradors.

Conclusió

El Patró Iterator és una eina poderosa per manejar col·leccions de manera eficient i flexible. En separar la lògica de recorregut de l'estructura de la col·lecció, s'aconsegueix un codi més net i mantenible. Aquest patró és especialment útil en situacions on es requereix un accés seqüencial als elements d'una col·lecció sense exposar la seva estructura interna.
