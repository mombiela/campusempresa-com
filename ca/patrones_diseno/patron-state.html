Introducció al Patró State

El Patró State és un patró de disseny de comportament que permet a un objecte canviar el seu comportament quan el seu estat intern canvia. Aquest patró és especialment útil quan un objecte ha de canviar el seu comportament en temps d'execució depenent del seu estat.

Conceptes Clau

  • Estat: Representa una condició particular en la qual es troba un objecte.
  • Context: L'objecte que canvia el seu comportament en funció del seu estat.
  • Transició d'Estat: El canvi d'un estat a un altre dins del context.

Estructura del Patró State

El Patró State generalment es compon dels següents components:

  1. Context: Manté una instància d'una subclasse concreta de State que defineix l'estat actual.
  2. Estat Abstracte (State): Defineix una interfície per encapsular el comportament associat amb un estat particular del context.
  3. Estats Concrets: Implementen el comportament específic de cada estat.

Diagrama de Classes

| Component | Descripció | |-----------|------------| | Context | Manté una referència a una instància d'un objecte State que defineix l'estat actual. | | State | Defineix una interfície per encapsular el comportament associat amb un estat particular del Context. | | ConcreteStateA | Implementa el comportament associat amb un estat del Context. | | ConcreteStateB | Implementa el comportament associat amb un altre estat del Context. |

Exemple de Codi

A continuació, es presenta un exemple en Python per il·lustrar el Patró State:

class State:
    def handle(self, context):
        raise NotImplementedError("Ha d'implementar el mètode handle")

class ConcreteStateA(State):
    def handle(self, context):
        print("Estat A: Gestiono la sol·licitud i canvio a l'Estat B")
        context.state = ConcreteStateB()

class ConcreteStateB(State):
    def handle(self, context):
        print("Estat B: Gestiono la sol·licitud i canvio a l'Estat A")
        context.state = ConcreteStateA()

class Context:
    def __init__(self, state):
        self.state = state

    def request(self):
        self.state.handle(self)

# Ús del Patró State
context = Context(ConcreteStateA())
context.request()  # Estat A gestiona la sol·licitud i canvia a l'Estat B
context.request()  # Estat B gestiona la sol·licitud i canvia a l'Estat A

Explicació del Codi

  • State: És una classe abstracta que defineix el mètode handle que ha de ser implementat pels estats concrets.
  • ConcreteStateA i ConcreteStateB: Implementen el mètode handle i defineixen el comportament específic per a cada estat.
  • Context: Manté una referència a l'estat actual i delega la sol·licitud a l'estat actual mitjançant el mètode request.

Avantatges i Desavantatges del Patró State

Avantatges

  • Flexibilitat: Permet canviar el comportament de l'objecte en temps d'execució.
  • Claredat: Millora la llegibilitat del codi en separar el comportament en diferents classes d'estat.
  • Manteniment: Facilita el manteniment i l'extensió del codi en afegir nous estats sense modificar el context.

Desavantatges

  • Complexitat: Pot augmentar la complexitat del codi a causa de la creació de múltiples classes d'estat.
  • Sobrecàrrega: Pot introduir una sobrecàrrega addicional si els estats són molt simples i no justifiquen l'ús del patró.

Conclusió

El Patró State és una poderosa eina per gestionar el comportament d'un objecte basat en el seu estat intern. En separar els estats en classes distintes, es millora la claredat i la mantenibilitat del codi. No obstant això, és important avaluar si la complexitat addicional és justificada per la naturalesa del problema que s'està resolent.
