Introducció

El Patró Bridge és un patró de disseny estructural que desacobla una abstracció de la seva implementació, permetent que ambdues variïn de manera independent. Aquest patró és útil quan es desitja evitar un creixement exponencial de classes a causa de la combinació de múltiples variants d'abstracció i implementació.

Conceptes Clau

  • Abstracció: La interfície d'alt nivell que defineix les operacions.
  • Implementació: La interfície que defineix les operacions primitives.
  • Refinament d'Abstracció: Una extensió de l'abstracció que afegeix més operacions.
  • Implementació Concreta: Una implementació específica de la interfície d'implementació.

Estructura del Patró Bridge

El Patró Bridge es compon dels següents components:

  • Abstracció: Defineix la interfície i manté una referència a un objecte de tipus Implementació.
  • Implementació: Defineix la interfície per a totes les implementacions concretes.
  • Refinament d'Abstracció: Estén la interfície d'Abstracció.
  • Implementació Concreta: Implementa la interfície d'Implementació.

Diagrama UML

| Component | Descripció | |--------------------------|----------------------------------------------------------------------------| | Abstraction | Defineix la interfície i manté una referència a un objecte de tipus Implementor. | | Implementor | Defineix la interfície per a totes les implementacions concretes. | | RefinedAbstraction | Estén la interfície de Abstraction. | | ConcreteImplementor | Implementa la interfície de Implementor. |

Exemple de Codi

A continuació, es presenta un exemple en Python per il·lustrar el Patró Bridge:

# Implementor
class DrawingAPI:
    def draw_circle(self, x, y, radius):
        pass

# ConcreteImplementor 1
class DrawingAPI1(DrawingAPI):
    def draw_circle(self, x, y, radius):
        print(f"API1.circle at {x}:{y} radius {radius}")

# ConcreteImplementor 2
class DrawingAPI2(DrawingAPI):
    def draw_circle(self, x, y, radius):
        print(f"API2.circle at {x}:{y} radius {radius}")

# Abstraction
class Shape:
    def __init__(self, drawing_api):
        self.drawing_api = drawing_api

    def draw(self):
        pass

    def resize_by_percentage(self, pct):
        pass

# RefinedAbstraction
class CircleShape(Shape):
    def __init__(self, x, y, radius, drawing_api):
        super().__init__(drawing_api)
        self.x = x
        self.y = y
        self.radius = radius

    def draw(self):
        self.drawing_api.draw_circle(self.x, self.y, self.radius)

    def resize_by_percentage(self, pct):
        self.radius *= (1 + pct / 100)

# Ús del patró Bridge
circle1 = CircleShape(1, 2, 3, DrawingAPI1())
circle2 = CircleShape(5, 7, 11, DrawingAPI2())

circle1.draw()
circle2.draw()

circle1.resize_by_percentage(50)
circle1.draw()

Explicació del Codi

  • DrawingAPI: És la interfície d'implementació que defineix el mètode draw_circle.
  • DrawingAPI1 i DrawingAPI2: Són implementacions concretes de DrawingAPI que proporcionen diferents maneres de dibuixar un cercle.
  • Shape: És l'abstracció que manté una referència a un objecte de tipus DrawingAPI.
  • CircleShape: És una extensió de Shape que afegeix operacions específiques per a un cercle, com draw i resize_by_percentage.

Conclusió

El Patró Bridge és una solució efectiva per desacoblar una abstracció de la seva implementació, permetent que ambdues evolucionin de manera independent. Això és especialment útil en sistemes complexos on les combinacions d'abstracció i implementació poden créixer exponencialment. En aplicar aquest patró, es millora la flexibilitat i la mantenibilitat del codi.
