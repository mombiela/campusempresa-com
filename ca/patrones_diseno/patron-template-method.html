Introducció

El patró Template Method és un patró de disseny de comportament que defineix l'esquelet d'un algorisme en una operació, diferint alguns passos a les subclasses. Permet que les subclasses redefineixin certs passos d'un algorisme sense canviar l'estructura del mateix.

Conceptes Clau

  • Classe Abstracta: Defineix l'esquelet de l'algorisme i conté el mètode template.
  • Mètode Template: Un mètode en la classe abstracta que defineix l'algorisme i crida als mètodes abstractes.
  • Mètodes Abstractes: Mètodes que han de ser implementats per les subclasses.
  • Guany: Promou la reutilització de codi i l'extensió del comportament sense modificar el codi existent.

Estructura del Patró

  1. Classe Abstracta: Conté el mètode template i els mètodes abstractes.
  2. Subclasses Concretes: Implementen els mètodes abstractes definits en la classe abstracta.

Diagrama de Classes

| Classe Abstracta | Subclasse Concreta | |------------------|--------------------| | + templateMethod() | + metodoConcreto1() | | + metodoAbstracto1() | + metodoConcreto2() | | + metodoAbstracto2() | |

Exemple de Codi

A continuació, es presenta un exemple en Python per il·lustrar el patró Template Method:

from abc import ABC, abstractmethod

class AbstractClass(ABC):
    def template_method(self):
        self.base_operation1()
        self.required_operations1()
        self.base_operation2()
        self.hook1()
        self.required_operations2()
        self.base_operation3()
        self.hook2()

    def base_operation1(self):
        print("AbstractClass diu: Estic fent la major part de la feina")

    def base_operation2(self):
        print("AbstractClass diu: Però permeto que les subclasses sobrescriguin algunes operacions")

    def base_operation3(self):
        print("AbstractClass diu: Però jo sóc qui crida a les operacions")

    @abstractmethod
    def required_operations1(self):
        pass

    @abstractmethod
    def required_operations2(self):
        pass

    def hook1(self):
        pass

    def hook2(self):
        pass

class ConcreteClass1(AbstractClass):
    def required_operations1(self):
        print("ConcreteClass1 diu: He implementat Operation1")

    def required_operations2(self):
        print("ConcreteClass1 diu: He implementat Operation2")

class ConcreteClass2(AbstractClass):
    def required_operations1(self):
        print("ConcreteClass2 diu: He implementat Operation1")

    def required_operations2(self):
        print("ConcreteClass2 diu: He implementat Operation2")

    def hook1(self):
        print("ConcreteClass2 diu: He sobrescrit Hook1")

def client_code(abstract_class: AbstractClass):
    abstract_class.template_method()

if __name__ == "__main__":
    print("El mateix codi de client pot funcionar amb diferents subclasses:")
    client_code(ConcreteClass1())
    print("")
    print("El mateix codi de client pot funcionar amb diferents subclasses:")
    client_code(ConcreteClass2())

Explicació del Codi

  • AbstractClass: Defineix el mètode template_method que estableix l'esquelet de l'algorisme. Inclou mètodes abstractes (required_operations1 i required_operations2) que han de ser implementats per les subclasses.
  • ConcreteClass1 i ConcreteClass2: Implementen els mètodes abstractes definits en AbstractClass. ConcreteClass2 també sobrescriu un mètode hook1 opcional.
  • client_code: Funció que accepta una instància de AbstractClass i crida al mètode template_method.

Conclusió

El patró Template Method és útil quan es desitja definir l'esquelet d'un algorisme en una classe base i permetre que les subclasses implementin detalls específics. Això promou la reutilització de codi i facilita l'extensió del comportament sense modificar el codi existent. És especialment útil en situacions on es tenen algorismes similars amb passos específics que varien entre implementacions.
