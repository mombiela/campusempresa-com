Introducció

En aquesta secció, explorarem com els patrons de disseny s'apliquen en situacions del món real. Analitzarem casos específics i veurem com els patrons de disseny poden resoldre problemes comuns en el desenvolupament de programari. Aquest enfocament pràctic t'ajudarà a entendre millor la utilitat i la implementació d'aquests patrons.

Cas 1: Sistema de Gestió de Comandes

Context

Una empresa de comerç electrònic necessita un sistema per gestionar les comandes dels seus clients. El sistema ha de ser flexible i escalable per manejar diferents tipus de comandes i mètodes de pagament.

Problema

El sistema actual és monolític i difícil de mantenir. Cada vegada que s'afegeix un nou tipus de comanda o mètode de pagament, es requereix una reescriptura significativa del codi.

Solució: Patró de Disseny Estratègia

El patró de disseny Estratègia permet definir una família d'algoritmes, encapsular cadascun d'ells i fer-los intercanviables. Aquest patró permet que l'algoritme variï independentment dels clients que l'utilitzen.

Implementació

  1. Definir la interfície de l'estratègia:

    public interface PaymentStrategy {
    void pay(int amount);
}

  2. Implementar diferents estratègies de pagament:

    public class CreditCardPayment implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " using Credit Card.");
    }
}

public class PayPalPayment implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " using PayPal.");
    }
}

  3. Context que utilitza l'estratègia:

    public class Order {
    private PaymentStrategy paymentStrategy;

    public void setPaymentStrategy(PaymentStrategy paymentStrategy) {
        this.paymentStrategy = paymentStrategy;
    }

    public void processOrder(int amount) {
        paymentStrategy.pay(amount);
    }
}

  4. Ús del patró Estratègia:

    public class Main {
    public static void main(String[] args) {
        Order order = new Order();

        // Pagar amb targeta de crèdit
        order.setPaymentStrategy(new CreditCardPayment());
        order.processOrder(100);

        // Pagar amb PayPal
        order.setPaymentStrategy(new PayPalPayment());
        order.processOrder(200);
    }
}

Beneficis

  • Flexibilitat: Fàcil d'afegir nous mètodes de pagament sense modificar el codi existent.
  • Mantenibilitat: Separació clara de les responsabilitats.

Cas 2: Sistema de Notificacions

Context

Una aplicació de missatgeria necessita enviar notificacions als usuaris a través de diferents canals com email, SMS i notificacions push.

Problema

El codi per enviar notificacions està dispers i duplicat en diverses parts de l'aplicació, cosa que en dificulta el manteniment i l'escalabilitat.

Solució: Patró de Disseny Observador

El patró Observador defineix una dependència un-a-molts entre objectes, de manera que quan un objecte canvia d'estat, tots els seus dependents són notificats i actualitzats automàticament.

Implementació

  1. Definir la interfície de l'observador:

    public interface Observer {
    void update(String message);
}

  2. Implementar diferents observadors:

    public class EmailNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("Email notification: " + message);
    }
}

public class SMSNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("SMS notification: " + message);
    }
}

public class PushNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("Push notification: " + message);
    }
}

  3. Definir el subjecte que notifica els observadors:

    import java.util.ArrayList;
import java.util.List;

public class NotificationService {
    private List<Observer> observers = new ArrayList<>();

    public void addObserver(Observer observer) {
        observers.add(observer);
    }

    public void removeObserver(Observer observer) {
        observers.remove(observer);
    }

    public void notifyObservers(String message) {
        for (Observer observer : observers) {
            observer.update(message);
        }
    }
}

  4. Ús del patró Observador:

    public class Main {
    public static void main(String[] args) {
        NotificationService notificationService = new NotificationService();

        Observer emailNotifier = new EmailNotifier();
        Observer smsNotifier = new SMSNotifier();
        Observer pushNotifier = new PushNotifier();

        notificationService.addObserver(emailNotifier);
        notificationService.addObserver(smsNotifier);
        notificationService.addObserver(pushNotifier);

        notificationService.notifyObservers("New message received!");
    }
}

Beneficis

  • Desacoblament: Els observadors estan desacoblats del subjecte.
  • Escalabilitat: Fàcil d'afegir nous tipus de notificacions.

Conclusió

L'estudi de casos reals ens mostra com els patrons de disseny poden resoldre problemes comuns en el desenvolupament de programari. En aplicar patrons com Estratègia i Observador, podem crear sistemes més flexibles, mantenibles i escalables. Aquests exemples pràctics il·lustren la importància dels patrons de disseny i com poden ser implementats en situacions del món real.
