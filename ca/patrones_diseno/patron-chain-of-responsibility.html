Introducció

El patró Chain of Responsibility és un patró de disseny de comportament que permet a múltiples objectes manejar una sol·licitud sense acoblar explícitament el remitent de la sol·licitud al seu receptor. Aquest patró crea una cadena d'objectes receptors i passa la sol·licitud al llarg de la cadena fins que un objecte la maneja.

Objectius d'Aprenentatge

  • Comprendre el propòsit i l'estructura del patró Chain of Responsibility.
  • Identificar quan i com aplicar aquest patró en el desenvolupament de programari.
  • Implementar el patró Chain of Responsibility en un exemple pràctic.

Conceptes Clau

  • Responsabilitat: Cada objecte en la cadena té l'oportunitat de manejar la sol·licitud.
  • Desacoblament: El remitent de la sol·licitud no necessita conèixer quin objecte en la cadena manejarà la sol·licitud.
  • Flexibilitat: És fàcil afegir o canviar els manejadors en la cadena sense modificar el codi del remitent.

Estructura del Patró

El patró Chain of Responsibility es compon dels següents components:

  • Handler (Manejador): Defineix una interfície per manejar les sol·licituds.
  • ConcreteHandler (Manejador Concret): Maneja les sol·licituds que li corresponen i passa la sol·licitud al següent manejador en la cadena si no pot manejar-la.
  • Client (Client): Inicia la sol·licitud a un dels manejadors en la cadena.

Diagrama de Classes

| Component | Descripció | |------------------|-----------------------------------------------------------------------------| | Handler | Interfície que declara el mètode per manejar les sol·licituds. | | ConcreteHandler1 | Implementa el mètode de maneig de sol·licituds i passa la sol·licitud si no pot manejar-la. | | ConcreteHandler2 | Similar a ConcreteHandler1, maneja sol·licituds específiques. | | Client | Inicia la sol·licitud a un dels manejadors en la cadena. |

Exemple Pràctic

Implementarem un exemple simple en Python on diferents manejadors processen una sol·licitud de suport tècnic.

Codi d'Exemple

class Handler:
    def __init__(self, successor=None):
        self._successor = successor

    def handle(self, request):
        raise NotImplementedError("Ha d'implementar el mètode handle")

class LevelOneSupport(Handler):
    def handle(self, request):
        if request == "simple issue":
            return "LevelOneSupport: Resolved simple issue"
        elif self._successor:
            return self._successor.handle(request)

class LevelTwoSupport(Handler):
    def handle(self, request):
        if request == "complex issue":
            return "LevelTwoSupport: Resolved complex issue"
        elif self._successor:
            return self._successor.handle(request)

class LevelThreeSupport(Handler):
    def handle(self, request):
        if request == "critical issue":
            return "LevelThreeSupport: Resolved critical issue"
        elif self._successor:
            return self._successor.handle(request)

# Configuració de la cadena de responsabilitat
level_three = LevelThreeSupport()
level_two = LevelTwoSupport(level_three)
level_one = LevelOneSupport(level_two)

# Client fa una sol·licitud
requests = ["simple issue", "complex issue", "critical issue", "unknown issue"]

for req in requests:
    result = level_one.handle(req)
    print(f"Request: {req} -> {result}")

Explicació del Codi

  • Handler: Classe base que defineix la interfície handle.
  • LevelOneSupport, LevelTwoSupport, LevelThreeSupport: Manejadors concrets que implementen la lògica per manejar diferents tipus de sol·licituds.
  • Configuració de la cadena: Es crea una cadena de manejadors on LevelOneSupport passa la sol·licitud a LevelTwoSupport si no pot manejar-la, i així successivament.
  • Client: Realitza diverses sol·licituds i la cadena de manejadors processa cadascuna.

Conclusió

El patró Chain of Responsibility és útil per a situacions on múltiples objectes poden manejar una sol·licitud i el remitent no necessita saber quin objecte la manejarà. Aquest patró promou el desacoblament i la flexibilitat en el maneig de sol·licituds. En implementar aquest patró, és important dissenyar acuradament la cadena de manejadors per assegurar que totes les sol·licituds siguin processades adequadament.
