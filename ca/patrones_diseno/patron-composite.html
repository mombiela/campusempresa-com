Introducció

El patró Composite és un patró de disseny estructural que permet tractar objectes individuals i compostos de manera uniforme. Aquest patró és útil quan necessites representar jerarquies d'objectes i desitges que els clients interactuïn amb aquests objectes de manera consistent, sense preocupar-se si estan tractant amb un objecte simple o compost.

Conceptes Clau

  • Component: Defineix la interfície per a tots els objectes en la composició, tant per als objectes simples com per als compostos.
  • Fulla: Representa els objectes individuals en la composició. Una fulla no té fills.
  • Compost: Defineix el comportament dels components que tenen fills. Implementa operacions relacionades amb els fills i pot afegir, eliminar o accedir a ells.

Estructura del Patró Composite

El patró Composite es pot visualitzar amb la següent estructura:

| Component | Fulla | Compost | |-----------|-------|---------| | + operació() | + operació() | + operació() | | | | + afegir(Component) | | | | + eliminar(Component) | | | | + obtenirFill(int) |

Exemple de Codi

A continuació, es presenta un exemple en Java per il·lustrar com implementar el patró Composite.

// Interfície Component
interface Component {
    void operacio();
}

// Classe Fulla
class Fulla implements Component {
    private String nom;

    public Fulla(String nom) {
        this.nom = nom;
    }

    @Override
    public void operacio() {
        System.out.println("Fulla " + nom + " operació realitzada.");
    }
}

// Classe Compost
class Compost implements Component {
    private List<Component> fills = new ArrayList<>();

    @Override
    public void operacio() {
        for (Component fill : fills) {
            fill.operacio();
        }
    }

    public void afegir(Component component) {
        fills.add(component);
    }

    public void eliminar(Component component) {
        fills.remove(component);
    }

    public Component obtenirFill(int index) {
        return fills.get(index);
    }
}

// Ús del patró Composite
public class Client {
    public static void main(String[] args) {
        Component fulla1 = new Fulla("1");
        Component fulla2 = new Fulla("2");

        Compost compost = new Compost();
        compost.afegir(fulla1);
        compost.afegir(fulla2);

        Compost compostMajor = new Compost();
        compostMajor.afegir(compost);

        compostMajor.operacio();
    }
}

Explicació del Codi

  1. Interfície Component: Defineix l'operació operacio() que serà implementada tant per les fulles com pels compostos.
  2. Classe Fulla: Implementa la interfície Component i defineix l'operació específica per a les fulles.
  3. Classe Compost: Implementa la interfície Component i maneja una llista de fills que també són components. Implementa mètodes per afegir, eliminar i obtenir fills.
  4. Client: Crea instàncies de fulles i compostos, i demostra com es poden combinar i operar de manera uniforme.

Conclusió

El patró Composite és una poderosa eina per manejar jerarquies d'objectes de manera uniforme. Permet als clients tractar objectes individuals i compostos de la mateixa manera, simplificant el codi i millorant la flexibilitat. Aquest patró és especialment útil en aplicacions que manegen estructures d'arbre, com sistemes de fitxers, interfícies gràfiques i representacions de documents.

Recorda que la clau del patró Composite és la recursivitat i la capacitat de tractar objectes individuals i compostos de manera uniforme. Practica implementant aquest patró en diferents escenaris per comprendre millor la seva utilitat i aplicació.
