Introducció

Els patrons de disseny són solucions reutilitzables a problemes comuns que es presenten en el disseny de programari. Aquests patrons no són peces de codi específiques, sinó més aviat descripcions o plantilles que poden ser aplicades a diferents situacions per resoldre problemes de disseny recurrents.

Importància dels Patrons de Disseny

  • Reutilització de Solucions Provades: Els patrons de disseny proporcionen solucions que han estat provades i refinades al llarg del temps.
  • Comunicació Eficaç: Faciliten la comunicació entre desenvolupadors en proporcionar un llenguatge comú.
  • Millora de la Mantenibilitat: Ajuden a crear sistemes més fàcils de mantenir i estendre.
  • Reducció d'Errors: En utilitzar solucions provades, es redueixen les possibilitats de cometre errors comuns.

Classificació dels Patrons de Disseny

Els patrons de disseny es divideixen en tres categories principals:

  1. Patrons Creacionals: Es concentren en la creació d'objectes, ajudant a fer el sistema independent de com es creen, componen i representen els objectes.

    • Exemples: Singleton, Factory Method, Abstract Factory, Builder, Prototype.

  2. Patrons Estructurals: Es preocupen de la composició de classes o objectes per formar estructures més grans.

    • Exemples: Adapter, Composite, Proxy, Flyweight, Facade, Bridge, Decorator.

  3. Patrons de Comportament: Es concentren en la interacció i responsabilitat entre els objectes.

    • Exemples: Observer, Strategy, Command, Chain of Responsibility, Mediator, Memento, State, Visitor, Template Method, Iterator.

Exemples de Patrons de Disseny

Singleton

El patró Singleton assegura que una classe tingui només una instància i proporciona un punt d'accés global a ella.

public class Singleton {
    private static Singleton instance;

    private Singleton() {
        // Constructor privat per evitar instanciació
    }

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

Factory Method

El patró Factory Method defineix una interfície per crear un objecte, però permet a les subclasses alterar el tipus d'objectes que es crearan.

abstract class Product {
    public abstract void use();
}

class ConcreteProductA extends Product {
    public void use() {
        System.out.println("Using Product A");
    }
}

class ConcreteProductB extends Product {
    public void use() {
        System.out.println("Using Product B");
    }
}

abstract class Creator {
    public abstract Product factoryMethod();

    public void someOperation() {
        Product product = factoryMethod();
        product.use();
    }
}

class ConcreteCreatorA extends Creator {
    public Product factoryMethod() {
        return new ConcreteProductA();
    }
}

class ConcreteCreatorB extends Creator {
    public Product factoryMethod() {
        return new ConcreteProductB();
    }
}

Comparació de Patrons

| Categoria | Propòsit | Exemple | |-------------------|--------------------------------------------------------------------------|------------------------------| | Creacionals | Gestionen la creació d'objectes | Singleton, Factory Method | | Estructurals | Composen classes o objectes en estructures més grans | Adapter, Composite | | Comportament | Gestionen la interacció i responsabilitat entre objectes | Observer, Strategy |

Conclusió

Els patrons de disseny són eines essencials per a qualsevol desenvolupador de programari. No només proporcionen solucions provades i refinades per a problemes comuns, sinó que també milloren la comunicació entre els desenvolupadors i la mantenibilitat del codi. En entendre i aplicar aquests patrons, els desenvolupadors poden crear sistemes més robustos i flexibles.
