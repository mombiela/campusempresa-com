Introducció

En el desenvolupament de programari, els patrons de disseny són solucions provades i reutilitzables per a problemes comuns que sorgeixen durant el disseny de sistemes. La identificació correcta de problemes i la selecció adequada de patrons de disseny són habilitats crucials per a qualsevol professional de TI. Aquest tema et guiarà a través del procés d'identificar problemes en el disseny de programari i seleccionar els patrons de disseny més adequats per resoldre'ls.

Identificació de Problemes

Per identificar problemes en el disseny de programari, és essencial comprendre el context i els requisits del sistema. Aquí hi ha alguns passos clau:

  1. Anàlisi de Requisits: Revisa els requisits del sistema per entendre les funcionalitats i restriccions.
  2. Identificació de Problemes Comuns: Busca problemes comuns com la duplicació de codi, la rigidesa del sistema, la falta de reutilització, etc.
  3. Entrevistes i Reunions: Parla amb els stakeholders i l'equip de desenvolupament per identificar problemes específics.
  4. Revisió de Codi: Analitza el codi existent per detectar problemes de disseny.

Selecció de Patrons de Disseny

Un cop s'han identificat els problemes, el següent pas és seleccionar els patrons de disseny adequats. Aquí hi ha alguns patrons de disseny comuns i els problemes que resolen:

  • Singleton: Garanteix que una classe tingui només una instància i proporciona un punt d'accés global a ella.

    • Problema: Necessitat d'una única instància d'una classe.
    • Exemple: 
      public class Singleton {
    private static Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

  • Factory Method: Defineix una interfície per crear un objecte, però permet a les subclasses alterar el tipus d'objectes que es creen.

    • Problema: Necessitat de crear objectes sense especificar la classe exacta de l'objecte que es crearà.
    • Exemple: 
      public abstract class Product {
    public abstract void use();
}

public class ConcreteProductA extends Product {
    public void use() {
        System.out.println("Using ConcreteProductA");
    }
}

public class ConcreteProductB extends Product {
    public void use() {
        System.out.println("Using ConcreteProductB");
    }
}

public abstract class Creator {
    public abstract Product factoryMethod();
}

public class ConcreteCreatorA extends Creator {
    public Product factoryMethod() {
        return new ConcreteProductA();
    }
}

public class ConcreteCreatorB extends Creator {
    public Product factoryMethod() {
        return new ConcreteProductB();
    }
}

  • Observer: Defineix una dependència d'un a molts entre objectes, de manera que quan un dels objectes canvia el seu estat, tots els seus dependents són notificats i actualitzats automàticament.

    • Problema: Necessitat de notificar a múltiples objectes sobre canvis en l'estat d'un altre objecte.
    • Exemple: 
      import java.util.ArrayList;
import java.util.List;

public class Subject {
    private List<Observer> observers = new ArrayList<>();
    private int state;

    public int getState() {
        return state;
    }

    public void setState(int state) {
        this.state = state;
        notifyAllObservers();
    }

    public void attach(Observer observer) {
        observers.add(observer);
    }

    public void notifyAllObservers() {
        for (Observer observer : observers) {
            observer.update();
        }
    }
}

public abstract class Observer {
    protected Subject subject;
    public abstract void update();
}

public class ConcreteObserver extends Observer {
    public ConcreteObserver(Subject subject) {
        this.subject = subject;
        this.subject.attach(this);
    }

    public void update() {
        System.out.println("State changed to: " + subject.getState());
    }
}

Comparació de Patrons

| Patró de Disseny | Problema que Resol | Exemple d'Ús | |------------------|---------------------|---------------| | Singleton | Necessitat d'una única instància d'una classe | Gestió de configuració global | | Factory Method | Creació d'objectes sense especificar la classe exacta | Creació de diferents tipus de productes | | Observer | Notificació de canvis d'estat a múltiples objectes | Implementació d'esdeveniments i subscriptors |

Conclusió

La identificació de problemes i la selecció de patrons de disseny són passos fonamentals en el desenvolupament de programari. En comprendre els problemes comuns i els patrons de disseny disponibles, pots crear solucions més eficients i mantenibles. Practicar aquests conceptes amb exemples i exercicis t'ajudarà a dominar aquestes habilitats essencials.
