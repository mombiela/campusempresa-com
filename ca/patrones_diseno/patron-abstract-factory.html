Introducció

El patró Abstract Factory és un dels patrons de disseny creacionals més importants en el desenvolupament de programari. Aquest patró proporciona una interfície per crear famílies d'objectes relacionats o dependents sense especificar les seves classes concretes. És especialment útil quan el sistema ha de ser independent de com es creen, componen i representen els seus productes.

Objectiu del Patró Abstract Factory

  • Desacoblar la creació d'objectes: Permet que el codi client no depengui de les classes concretes dels objectes que necessita.
  • Facilitar la intercanviabilitat de famílies de productes: Permet canviar fàcilment la família de productes que s'utilitza en el sistema.
  • Promoure la coherència entre objectes: Assegura que els objectes creats siguin compatibles entre si.

Estructura del Patró Abstract Factory

El patró Abstract Factory es compon dels següents elements:

  • AbstractFactory: Declara una interfície per a operacions que creen productes abstractes.
  • ConcreteFactory: Implementa les operacions per crear objectes concrets.
  • AbstractProduct: Declara una interfície per a un tipus de producte.
  • ConcreteProduct: Defineix un objecte de producte que serà creat per la fàbrica concreta corresponent.
  • Client: Utilitza únicament les interfícies declarades per AbstractFactory i AbstractProduct.

Diagrama de Classes

| Element | Descripció | |----------------|-----------------------------------------------------------------------------| | AbstractFactory | Interfície que declara mètodes per crear productes abstractes. | | ConcreteFactory | Implementa els mètodes de AbstractFactory per crear productes concrets. | | AbstractProduct | Interfície per a un tipus de producte. | | ConcreteProduct | Implementa la interfície AbstractProduct per a un producte específic. | | Client | Utilitza les interfícies de AbstractFactory i AbstractProduct per a la seva lògica. |

Exemple de Codi

A continuació, es presenta un exemple en Python per il·lustrar el patró Abstract Factory.

from abc import ABC, abstractmethod

# Abstract Products
class AbstractProductA(ABC):
    @abstractmethod
    def operation_a(self) -> str:
        pass

class AbstractProductB(ABC):
    @abstractmethod
    def operation_b(self) -> str:
        pass

# Concrete Products
class ConcreteProductA1(AbstractProductA):
    def operation_a(self) -> str:
        return "Result of ConcreteProductA1"

class ConcreteProductA2(AbstractProductA):
    def operation_a(self) -> str:
        return "Result of ConcreteProductA2"

class ConcreteProductB1(AbstractProductB):
    def operation_b(self) -> str:
        return "Result of ConcreteProductB1"

class ConcreteProductB2(AbstractProductB):
    def operation_b(self) -> str:
        return "Result of ConcreteProductB2"

# Abstract Factory
class AbstractFactory(ABC):
    @abstractmethod
    def create_product_a(self) -> AbstractProductA:
        pass

    @abstractmethod
    def create_product_b(self) -> AbstractProductB:
        pass

# Concrete Factories
class ConcreteFactory1(AbstractFactory):
    def create_product_a(self) -> AbstractProductA:
        return ConcreteProductA1()

    def create_product_b(self) -> AbstractProductB:
        return ConcreteProductB1()

class ConcreteFactory2(AbstractFactory):
    def create_product_a(self) -> AbstractProductA:
        return ConcreteProductA2()

    def create_product_b(self) -> AbstractProductB:
        return ConcreteProductB2()

# Client Code
def client_code(factory: AbstractFactory) -> None:
    product_a = factory.create_product_a()
    product_b = factory.create_product_b()

    print(product_a.operation_a())
    print(product_b.operation_b())

if __name__ == "__main__":
    print("Client: Testing client code with the first factory type:")
    client_code(ConcreteFactory1())

    print("\nClient: Testing the same client code with the second factory type:")
    client_code(ConcreteFactory2())

Explicació del Codi

  • AbstractProductA i AbstractProductB: Defineixen interfícies per als productes A i B.
  • ConcreteProductA1, ConcreteProductA2, ConcreteProductB1, ConcreteProductB2: Implementen les interfícies de productes concrets.
  • AbstractFactory: Defineix mètodes abstractes per crear productes A i B.
  • ConcreteFactory1 i ConcreteFactory2: Implementen els mètodes de AbstractFactory per crear productes concrets.
  • client_code: Funció que utilitza la fàbrica abstracta per crear productes sense conèixer les seves classes concretes.

Conclusió

El patró Abstract Factory és una eina poderosa per a la creació de famílies d'objectes relacionats sense acoblar el codi client a les classes concretes. Aquest patró facilita la intercanviabilitat i coherència dels productes, el que resulta en un codi més flexible i mantenible. En entendre i aplicar aquest patró, els desenvolupadors poden dissenyar sistemes més robustos i escalables.
