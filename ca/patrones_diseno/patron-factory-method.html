Introducció

El patró Factory Method és un dels patrons de disseny creacionals més importants i utilitzats en el desenvolupament de programari. El seu principal objectiu és definir una interfície per crear un objecte, però permet a les subclasses alterar el tipus d'objectes que es crearan.

Conceptes Clau

  • Propòsit: Proveir una interfície per a la creació d'objectes en una superclasse, mentre permet a les subclasses modificar el tipus d'objectes que es crearan.
  • Aplicabilitat: S'utilitza quan una classe no pot anticipar el tipus d'objectes que ha de crear o quan una classe vol que les seves subclasses especifiquin els objectes que crea.
  • Participants:
    • Product: Defineix la interfície dels objectes que el mètode de fàbrica crea.
    • ConcreteProduct: Implementa la interfície Product.
    • Creator: Declara el mètode de fàbrica, que retorna un objecte de tipus Product.
    • ConcreteCreator: Sobrescriu el mètode de fàbrica per retornar una instància de ConcreteProduct.

Exemple de Codi

A continuació, es presenta un exemple en Python per il·lustrar el patró Factory Method:

from abc import ABC, abstractmethod

# Product
class Button(ABC):
    @abstractmethod
    def render(self):
        pass

# ConcreteProduct
class WindowsButton(Button):
    def render(self):
        return "Render a button in Windows style"

class HTMLButton(Button):
    def render(self):
        return "Render a button in HTML style"

# Creator
class Dialog(ABC):
    @abstractmethod
    def create_button(self) -> Button:
        pass

    def render(self):
        button = self.create_button()
        return button.render()

# ConcreteCreator
class WindowsDialog(Dialog):
    def create_button(self) -> Button:
        return WindowsButton()

class WebDialog(Dialog):
    def create_button(self) -> Button:
        return HTMLButton()

# Client
def client_code(dialog: Dialog):
    print(dialog.render())

# Ús
if __name__ == "__main__":
    dialog = WindowsDialog()
    client_code(dialog)

    dialog = WebDialog()
    client_code(dialog)

Explicació del Codi

  • Product: Button és una interfície abstracta que defineix el mètode render.
  • ConcreteProduct: WindowsButton i HTMLButton són implementacions concretes de Button.
  • Creator: Dialog és una classe abstracta que declara el mètode de fàbrica create_button.
  • ConcreteCreator: WindowsDialog i WebDialog són subclasses de Dialog que implementen el mètode de fàbrica per crear instàncies de WindowsButton i HTMLButton, respectivament.
  • Client: La funció client_code utilitza el mètode render de Dialog per mostrar el botó creat.

Comparació amb Altres Patrons Creacionals

| Patró | Propòsit | Exemple d'Ús | |----------------------|--------------------------------------------------------------------------|------------------------------------------------------------------------------| | Factory Method | Defineix una interfície per crear un objecte, però deixa que les subclasses decideixin quina classe instanciar. | Crear diferents tipus de botons (Windows, HTML) sense canviar el codi del client. | | Abstract Factory | Proveir una interfície per crear famílies d'objectes relacionats o dependents sense especificar les seves classes concretes. | Crear famílies de productes (botons, checkboxes) per a diferents plataformes (Windows, Mac). | | Singleton | Assegurar que una classe tingui només una instància i proporcionar un punt d'accés global a ella. | Administrar una única instància d'una classe de configuració en tota l'aplicació. |

Conclusió

El patró Factory Method és una eina poderosa per a la creació d'objectes en un sistema de programari. Permet a les subclasses decidir quins objectes crear, promovent la flexibilitat i l'extensibilitat del codi. En comprendre i aplicar aquest patró, els desenvolupadors poden dissenyar sistemes més robustos i mantenibles.
