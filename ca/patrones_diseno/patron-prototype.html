Introducció

El patró Prototype és un dels patrons de disseny creacionals que permet la creació de nous objectes a partir d'una instància prototípica, clonant-la. Aquest patró és útil quan la creació d'un objecte és costosa en termes de temps o recursos, i es desitja evitar la repetició de la configuració inicial.

Conceptes Clau

  • Prototip: Un objecte existent que s'utilitza com a base per crear nous objectes.
  • Clonació: El procés de crear una còpia exacta del prototip.
  • Deep Copy vs Shallow Copy: Diferència entre copiar només les referències dels objectes (shallow) i copiar els objectes complets (deep).

Estructura del Patró Prototype

El patró Prototype generalment inclou els següents components:

  1. Prototype: Una interfície que declara el mètode de clonació.
  2. ConcretePrototype: Una classe que implementa la interfície Prototype i defineix com s'ha de clonar.
  3. Client: La classe que utilitza el prototip per crear nous objectes.

Exemple de Codi

A continuació, es presenta un exemple en Python per il·lustrar el patró Prototype:

import copy

class Prototype:
    def clone(self):
        pass

class ConcretePrototype1(Prototype):
    def __init__(self, value):
        self.value = value

    def clone(self):
        return copy.deepcopy(self)

class ConcretePrototype2(Prototype):
    def __init__(self, value):
        self.value = value

    def clone(self):
        return copy.deepcopy(self)

# Client que utilitza el prototip
if __name__ == "__main__":
    prototype1 = ConcretePrototype1("Prototype 1")
    prototype2 = ConcretePrototype2("Prototype 2")

    clone1 = prototype1.clone()
    clone2 = prototype2.clone()

    print(f"Original: {prototype1.value}, Clon: {clone1.value}")
    print(f"Original: {prototype2.value}, Clon: {clone2.value}")

Explicació del Codi

  • Prototype: Defineix la interfície clone que serà implementada per les classes concretes.
  • ConcretePrototype1 i ConcretePrototype2: Implementen el mètode clone utilitzant copy.deepcopy per realitzar una còpia profunda de l'objecte.
  • Client: Crea instàncies dels prototips i després les clona, demostrant que els objectes clonats tenen els mateixos valors que els originals.

Avantatges i Desavantatges

Avantatges

  • Reducció de Costos: Evita la costosa creació d'objectes des de zero.
  • Flexibilitat: Permet afegir o eliminar prototips en temps d'execució.
  • Simplicitat: Facilita la creació d'objectes complexos.

Desavantatges

  • Complexitat en la Clonació: La clonació profunda pot ser complicada si els objectes tenen referències circulars.
  • Manteniment: Pot ser difícil mantenir el codi si hi ha molts prototips diferents.

Comparació amb Altres Patrons Creacionals

| Patró | Propòsit | Exemple d'Ús | |-----------------|--------------------------------------------------------------------------|-------------------------------------------------------------------------------| | Singleton | Garanteix que una classe tingui només una instància i proporciona un punt global d'accés a ella. | Logger, Configuració d'Aplicació | | Factory Method | Defineix una interfície per crear un objecte, però deixa que les subclasses decideixin quina classe instanciar. | Creació d'objectes en biblioteques de tercers | | Abstract Factory| Proporciona una interfície per crear famílies d'objectes relacionats o dependents sense especificar les seves classes concretes. | Sistemes que han de ser independents de com es creen i munten els seus productes| | Builder | Separa la construcció d'un objecte complex de la seva representació, permetent que el mateix procés de construcció creï diferents representacions. | Creació d'objectes complexos com configuracions de UI | | Prototype | Permet la creació de nous objectes mitjançant la clonació d'una instància prototípica. | Clonació d'objectes configurats prèviament |

Conclusió

El patró Prototype és una eina poderosa per a la creació d'objectes quan es desitja evitar el cost d'inicialització repetitiva. Permet clonar objectes existents, la qual cosa pot ser especialment útil en aplicacions on els objectes són costosos de crear o configurar. No obstant això, és important tenir en compte les possibles complicacions en la clonació profunda i el manteniment del codi.

Aquest patró és una addició valuosa al conjunt d'eines de qualsevol desenvolupador de programari, proporcionant flexibilitat i eficiència en la creació d'objectes.
