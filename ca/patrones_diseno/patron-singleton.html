Introducció

El Patró Singleton és un dels patrons de disseny creacionals més coneguts i utilitzats en el desenvolupament de programari. El seu principal objectiu és garantir que una classe tingui una única instància i proporcionar un punt d'accés global a aquesta instància.

Conceptes Clau

  • Instància Única: El patró assegura que només existeixi una instància d'una classe en tot el sistema.
  • Accés Global: Proporciona un mètode global per accedir a aquesta única instància.
  • Control de Concurrència: En entorns multihil, és crucial assegurar que la instància única no es creï més d'una vegada.

Implementació del Patró Singleton

Exemple en Java

A continuació, es mostra un exemple bàsic de com implementar el patró Singleton en Java:

public class Singleton {
    // Instància única de la classe
    private static Singleton uniqueInstance;

    // Constructor privat per evitar la instanciació directa
    private Singleton() {}

    // Mètode públic per obtenir la instància única
    public static Singleton getInstance() {
        if (uniqueInstance == null) {
            uniqueInstance = new Singleton();
        }
        return uniqueInstance;
    }

    // Altres mètodes de la classe
    public void showMessage() {
        System.out.println("Hola des de Singleton!");
    }
}

Exemple en Python

A continuació, es mostra un exemple bàsic de com implementar el patró Singleton en Python:

class Singleton:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super(Singleton, cls).__new__(cls)
        return cls._instance

    def show_message(self):
        print("Hola des de Singleton!")

# Ús del Singleton
singleton = Singleton()
singleton.show_message()

Comparació d'Implementacions

| Característica | Java | Python | |-----------------------|------------------------------------------|------------------------------------------| | Declaració d'Instància Única | private static Singleton uniqueInstance; | _instance = None | | Constructor Privat | private Singleton() {} | def __new__(cls): | | Mètode d'Accés | public static Singleton getInstance() | def __new__(cls): | | Control de Concurrència | Necessari en entorns multihil | No és necessari en la majoria dels casos |

Avantatges i Desavantatges

Avantatges

  • Control d'Accés: Facilita el control d'accés a la instància única.
  • Reducció de Recursos: Evita la creació de múltiples instàncies, estalviant recursos.
  • Consistència: Garanteix que tots els usuaris del Singleton treballin amb la mateixa instància.

Desavantatges

  • Dificultat en Proves Unitàries: Pot complicar les proves unitàries a causa de la dependència global.
  • Problemes de Concurrència: En entorns multihil, s'ha de tenir cura per evitar problemes de concurrència.
  • Acoblament: Pot augmentar l'acoblament entre classes, cosa que dificulta el manteniment.

Exercicis

  1. Implementació Bàsica: Implementa el patró Singleton en un llenguatge de la teva elecció i crea una classe que imprimeixi un missatge.
  2. Control de Concurrència: Modifica la implementació del Singleton en Java perquè sigui segura en un entorn multihil.
  3. Proves Unitàries: Escriu proves unitàries per verificar que la classe Singleton sempre retorna la mateixa instància.

Conclusió

El Patró Singleton és una eina poderosa per garantir que una classe tingui una única instància i proporcionar un punt d'accés global a aquesta instància. Tot i que té els seus avantatges, també presenta desafiaments, especialment en entorns multihil i en proves unitàries. Comprendre quan i com utilitzar aquest patró és crucial per a la seva correcta aplicació en el desenvolupament de programari.
