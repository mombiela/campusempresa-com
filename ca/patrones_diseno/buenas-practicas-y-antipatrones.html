Introducció

En el desenvolupament de programari, els patrons de disseny són solucions provades i optimitzades per a problemes comuns. No obstant això, és igualment important conèixer les bones pràctiques i els antipatrons per evitar errors comuns i millorar la qualitat del codi.

Bones Pràctiques

Les bones pràctiques són mètodes i tècniques recomanades que ajuden a millorar la qualitat, mantenibilitat i eficiència del codi. A continuació, es presenten algunes de les més importants:

  • Principi de Responsabilitat Única (SRP): Cada classe o mòdul ha de tenir una única responsabilitat o raó per canviar.
  • Principi d'Obert/Tancat (OCP): Les entitats de programari han d'estar obertes per a l'extensió, però tancades per a la modificació.
  • Principi de Substitució de Liskov (LSP): Els objectes d'una classe derivada han de poder reemplaçar objectes de la classe base sense alterar el comportament del programa.
  • Principi de Segregació d'Interfícies (ISP): Els clients no han de veure's obligats a dependre d'interfícies que no utilitzen.
  • Principi d'Inversió de Dependència (DIP): Els mòduls d'alt nivell no han de dependre de mòduls de baix nivell. Ambdós han de dependre d'abstraccions.

Exemple de Codi: Principi de Responsabilitat Única

// Violació del SRP: La classe té múltiples responsabilitats
public class User {
    public void login(String username, String password) {
        // lògica d'autenticació
    }

    public void register(String username, String password) {
        // lògica de registre
    }

    public void sendEmail(String message) {
        // lògica d'enviament de correu
    }
}

// Aplicació del SRP: Separació de responsabilitats
public class User {
    public void login(String username, String password) {
        // lògica d'autenticació
    }

    public void register(String username, String password) {
        // lògica de registre
    }
}

public class EmailService {
    public void sendEmail(String message) {
        // lògica d'enviament de correu
    }
}

Antipatrons

Els antipatrons són solucions que poden semblar efectives a primera vista però que resulten en problemes a llarg termini. Conèixer-los és crucial per evitar-los i millorar la qualitat del programari.

  • God Object: Un objecte que sap massa o fa massa, violant el principi de responsabilitat única.
  • Spaghetti Code: Codi desorganitzat i difícil de seguir a causa de la manca d'estructura.
  • Golden Hammer: Ús excessiu d'una solució coneguda per a tots els problemes, sense considerar si és la més adequada.
  • Copy-Paste Programming: Reutilització de codi mitjançant còpia i pega, cosa que pot portar a inconsistències i errors difícils de mantenir.
  • Premature Optimization: Optimització del codi abans que sigui necessari, cosa que pot complicar el disseny i la implementació.

Exemple de Codi: God Object

// Exemple de God Object
public class GodObject {
    public void manageUser() {
        // lògica de gestió d'usuaris
    }

    public void manageOrders() {
        // lògica de gestió de comandes
    }

    public void manageInventory() {
        // lògica de gestió d'inventari
    }
}

// Refactorització per evitar God Object
public class UserManager {
    public void manageUser() {
        // lògica de gestió d'usuaris
    }
}

public class OrderManager {
    public void manageOrders() {
        // lògica de gestió de comandes
    }
}

public class InventoryManager {
    public void manageInventory() {
        // lògica de gestió d'inventari
    }
}

Comparació de Bones Pràctiques i Antipatrons

| Bones Pràctiques | Antipatrons | |---------------------------------|---------------------------------| | Principi de Responsabilitat Única | God Object | | Principi d'Obert/Tancat | Spaghetti Code | | Principi de Substitució de Liskov | Golden Hammer | | Principi de Segregació d'Interfícies | Copy-Paste Programming | | Principi d'Inversió de Dependència | Premature Optimization |

Conclusió

Conèixer i aplicar bones pràctiques en el desenvolupament de programari és essencial per crear codi d'alta qualitat i fàcil de mantenir. Al mateix temps, és crucial identificar i evitar els antipatrons que poden portar a problemes a llarg termini. En seguir aquests principis i evitar aquests errors comuns, els desenvolupadors poden millorar significativament l'eficiència i l'efectivitat dels seus projectes de programari.
