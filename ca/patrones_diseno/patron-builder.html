Introducció

El Patró Builder és un dels patrons de disseny creacionals que s'utilitza per construir objectes complexos pas a pas. Aquest patró és especialment útil quan un objecte pot ser construït de diferents maneres o quan el procés de construcció implica diversos passos.

Conceptes Clau

  • Separació de construcció i representació: El Patró Builder separa la construcció d'un objecte de la seva representació, permetent que el mateix procés de construcció pugui crear diferents representacions.
  • Director: És l'encarregat de controlar el procés de construcció. Utilitza un objecte Builder per construir el producte.
  • Builder: Defineix una interfície per crear les parts de l'objecte Producte.
  • ConcreteBuilder: Implementa la interfície Builder i construeix les parts del producte.
  • Producte: L'objecte complex que es construeix.

Estructura del Patró Builder

| Component | Descripció | |----------------|-----------------------------------------------------------------------------| | Director | Controla el procés de construcció utilitzant un objecte Builder. | | Builder | Defineix una interfície per crear les parts de l'objecte Producte. | | ConcreteBuilder| Implementa la interfície Builder i construeix les parts del producte. | | Producte | L'objecte complex que es construeix. |

Exemple de Codi

A continuació, es presenta un exemple en Python per il·lustrar el Patró Builder.

# Producte
class Pizza:
    def __init__(self):
        self.dough = None
        self.sauce = None
        self.topping = None

    def __str__(self):
        return f'Dough: {self.dough}, Sauce: {self.sauce}, Topping: {self.topping}'

# Builder
class PizzaBuilder:
    def __init__(self):
        self.pizza = Pizza()

    def set_dough(self, dough):
        self.pizza.dough = dough
        return self

    def set_sauce(self, sauce):
        self.pizza.sauce = sauce
        return self

    def set_topping(self, topping):
        self.pizza.topping = topping
        return self

    def build(self):
        return self.pizza

# Director
class Director:
    def __init__(self, builder):
        self._builder = builder

    def construct_pizza(self):
        self._builder.set_dough("cross").set_sauce("tomato").set_topping("mozzarella")
        return self._builder.build()

# Ús del Patró Builder
builder = PizzaBuilder()
director = Director(builder)
pizza = director.construct_pizza()
print(pizza)

Explicació del Codi

  • Producte (Pizza): La classe Pizza representa l'objecte complex que es construirà.
  • Builder (PizzaBuilder): La classe PizzaBuilder proporciona mètodes per configurar les diferents parts de la pizza.
  • Director: La classe Director controla el procés de construcció i utilitza el PizzaBuilder per construir una pizza.
  • Ús del Patró Builder: Es crea una instància de PizzaBuilder i Director, i s'utilitza el director per construir la pizza.

Avantatges del Patró Builder

  • Millora la llegibilitat del codi: En separar la construcció de l'objecte de la seva representació, el codi és més fàcil d'entendre i mantenir.
  • Facilita la creació d'objectes complexos: Permet construir objectes pas a pas, cosa que és útil per a objectes que requereixen múltiples configuracions.
  • Flexibilitat: El mateix procés de construcció pot crear diferents representacions de l'objecte.

Desavantatges del Patró Builder

  • Complexitat addicional: Introdueix més classes i pot fer que el disseny sigui més complex.
  • No sempre necessari: Per a objectes simples, l'ús del Patró Builder pot ser excessiu.

Conclusió

El Patró Builder és una eina poderosa per a la construcció d'objectes complexos d'una manera controlada i flexible. En separar la construcció de l'objecte de la seva representació, aquest patró facilita la creació de diferents versions d'un objecte sense duplicar codi. Encara que introdueix certa complexitat addicional, els seus beneficis en termes de llegibilitat i flexibilitat sovint superen aquests inconvenients, especialment en projectes grans i complexos.
