Introducció

En aquesta secció, explorarem els conceptes de Joins i Unions a BigQuery. Aquestes són operacions fonamentals per combinar dades de diverses taules. Entendre com utilitzar-les de manera efectiva és crucial per a l'anàlisi i la manipulació de dades.

Joins

Els Joins s'utilitzen per combinar files de dues o més taules basant-se en una columna relacionada entre elles. BigQuery admet diversos tipus de joins, cadascun amb diferents propòsits.

Tipus de Joins

  • INNER JOIN: Retorna només les files que tenen valors coincidents en ambdues taules.
  • LEFT JOIN (o LEFT OUTER JOIN): Retorna totes les files de la taula esquerra i les files coincidents de la taula dreta. Si no es troba cap coincidència, es retornen valors NULL per a les columnes de la taula dreta.
  • RIGHT JOIN (o RIGHT OUTER JOIN): Retorna totes les files de la taula dreta i les files coincidents de la taula esquerra. Si no es troba cap coincidència, es retornen valors NULL per a les columnes de la taula esquerra.
  • FULL JOIN (o FULL OUTER JOIN): Retorna totes les files quan hi ha una coincidència en una de les taules. Si no hi ha coincidència, el resultat és NULL en el costat que no té coincidència.
  • CROSS JOIN: Retorna el producte cartesià de les dues taules, és a dir, totes les combinacions possibles de files.

Sintaxi i exemples

INNER JOIN

SELECT a.column1, b.column2
FROM table1 a
INNER JOIN table2 b
ON a.common_column = b.common_column;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
INNER JOIN departments
ON employees.department_id = departments.department_id;

Explicació: Aquesta consulta recupera els noms dels empleats i els noms dels departaments corresponents on hi ha una coincidència en department_id.

LEFT JOIN

SELECT a.column1, b.column2
FROM table1 a
LEFT JOIN table2 b
ON a.common_column = b.common_column;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
LEFT JOIN departments
ON employees.department_id = departments.department_id;

Explicació: Aquesta consulta recupera tots els noms dels empleats i els noms dels seus departaments. Si un empleat no pertany a cap departament, el nom del departament serà NULL.

RIGHT JOIN

SELECT a.column1, b.column2
FROM table1 a
RIGHT JOIN table2 b
ON a.common_column = b.common_column;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
RIGHT JOIN departments
ON employees.department_id = departments.department_id;

Explicació: Aquesta consulta recupera tots els noms dels departaments i els noms dels empleats en aquests departaments. Si un departament no té empleats, el nom de l'empleat serà NULL.

FULL JOIN

SELECT a.column1, b.column2
FROM table1 a
FULL JOIN table2 b
ON a.common_column = b.common_column;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
FULL JOIN departments
ON employees.department_id = departments.department_id;

Explicació: Aquesta consulta recupera tots els noms dels empleats i els noms dels departaments, incloent aquells empleats que no pertanyen a cap departament i departaments que no tenen empleats.

CROSS JOIN

SELECT a.column1, b.column2
FROM table1 a
CROSS JOIN table2 b;

Exemple:

SELECT employees.name, departments.department_name
FROM employees
CROSS JOIN departments;

Explicació: Aquesta consulta recupera el producte cartesià dels empleats i els departaments, és a dir, totes les combinacions possibles d'empleat i departament.

Unions

Les unions s'utilitzen per combinar els resultats de dues o més consultes SELECT en un únic conjunt de resultats. Les consultes han de tenir el mateix nombre de columnes i tipus de dades compatibles en les columnes corresponents.

Tipus d'Unions

  • UNION: Combina els resultats de dues consultes i elimina les files duplicades.
  • UNION ALL: Combina els resultats de dues consultes i inclou totes les files, incloent-hi els duplicats.

Sintaxi i exemples

UNION

SELECT column1, column2 FROM table1
UNION
SELECT column1, column2 FROM table2;

Exemple:

SELECT name FROM employees
UNION
SELECT name FROM contractors;

Explicació: Aquesta consulta recupera una llista de noms únics tant d'empleats com de contractistes.

UNION ALL

SELECT column1, column2 FROM table1
UNION ALL
SELECT column1, column2 FROM table2;

Exemple:

SELECT name FROM employees
UNION ALL
SELECT name FROM contractors;

Explicació: Aquesta consulta recupera una llista de noms tant d'empleats com de contractistes, incloent-hi els duplicats.

Conclusió

Entendre i utilitzar Joins i Unions a BigQuery et permet combinar i analitzar dades de diverses taules de manera efectiva. Els Joins ajuden a fusionar files basant-se en columnes relacionades, mentre que les Unions ajuden a apilar resultats de múltiples consultes. Dominar aquestes operacions millorarà significativament les teves habilitats de manipulació de dades a BigQuery.
