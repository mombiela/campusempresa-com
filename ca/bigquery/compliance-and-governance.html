Introducció

BigQuery, com a magatzem de dades totalment gestionat, proporciona funcions robustes per garantir el compliment i la governança. Aquesta secció cobrirà els aspectes essencials del compliment i la governança a BigQuery, des de conceptes bàsics fins a pràctiques avançades.

Conceptes Clau

  • Compliment: Assegurar que la gestió i el processament de dades compleixin els requisits normatius.
  • Governança: Establir polítiques i procediments per gestionar la qualitat, la seguretat i l'ús de les dades.

Seguretat i Privacitat de les Dades

Control d'Accés

  • Gestió d'Identitat i Accés (IAM): BigQuery utilitza IAM per gestionar l'accés als recursos.
    • Rols: Rols predefinits (per exemple, bigquery.user, bigquery.admin) i rols personalitzats.
    • Permisos: Accions específiques que es poden realitzar (per exemple, bigquery.tables.getData).
-- Exemple: Concedir un rol a un usuari
GRANT `roles/bigquery.dataViewer` ON PROJECT `my-project` TO `user:example@example.com`;

Xifratge de Dades

  • Xifratge en Repòs: Les dades es xifren utilitzant claus gestionades per Google per defecte.
  • Claus de Xifratge Gestionades pel Client (CMEK): Opció per utilitzar les teves pròpies claus de xifratge.
-- Exemple: Crear un conjunt de dades amb CMEK
CREATE SCHEMA my_dataset
OPTIONS (
  location = 'US',
  default_kms_key_name = 'projects/my-project/locations/us/keyRings/my-keyring/cryptoKeys/my-key'
);

Governança de Dades

Llinatge de Dades

  • Seguiment del Llinatge de Dades: Entendre el flux de dades des de la font fins a la destinació.
    • Registres d'Auditoria: Utilitzar Cloud Audit Logs per seguir l'accés i els canvis.
    • Catàleg de Dades: Utilitzar Data Catalog per a la gestió de metadades i el seguiment del llinatge.

Qualitat de les Dades

  • Validació de Dades: Implementar comprovacions per assegurar l'exactitud i la consistència de les dades.
    • Restriccions: Utilitzar restriccions SQL per fer complir la integritat de les dades.
    • Perfilat de Dades: Analitzar les dades per entendre la seva estructura i qualitat.
-- Exemple: Afegir una restricció NOT NULL
CREATE TABLE my_table (
  id INT64 NOT NULL,
  name STRING NOT NULL
);

Retenció i Eliminació de Dades

  • Polítiques de Retenció: Establir polítiques per retenir dades durant un període específic.
  • Polítiques d'Eliminació: Assegurar que les dades es suprimeixin de manera segura després del seu període de retenció.
-- Exemple: Establir un temps d'expiració per a una taula
CREATE TABLE my_table (
  id INT64,
  name STRING
)
OPTIONS (
  expiration_timestamp = TIMESTAMP '2023-12-31 23:59:59 UTC'
);

Auditoria i Monitoratge

Registres d'Auditoria

  • Tipus de Registres: Activitat d'Administrador, Accés a Dades, Esdeveniments del Sistema i Registres de Polítiques Denegades.
  • Anàlisi de Registres: Utilitzar BigQuery per analitzar els registres d'auditoria per al compliment i el monitoratge de la seguretat.
-- Exemple: Consultar registres d'auditoria
SELECT
  protopayload_auditlog.methodName,
  protopayload_auditlog.resourceName,
  timestamp
FROM
  `my-project.audit_logs.cloudaudit_googleapis_com_activity_*`
WHERE
  protopayload_auditlog.methodName = 'google.bigquery.v2.JobService.InsertJob';

Monitoratge

  • Eines de Monitoratge: Utilitzar Cloud Monitoring i Cloud Logging per monitorar l'ús i el rendiment de BigQuery.
  • Alertes: Configurar alertes per a esdeveniments o llindars específics.
-- Exemple: Crear una política d'alerta
gcloud alpha monitoring policies create \
  --display-name="BigQuery Job Failure Alert" \
  --combiner="OR" \
  --conditions="condition_threshold.json"

Conclusió

En aquest tema, hem explorat els aspectes crítics del compliment i la governança a BigQuery. Hem cobert el control d'accés, el xifratge de dades, el llinatge de dades, la qualitat de les dades, les polítiques de retenció i l'auditoria i el monitoratge. Implementant aquestes pràctiques, pots assegurar que el teu entorn BigQuery sigui segur, complidor i ben governat.
