Introducció

L'auditoria i el monitoratge són aspectes crucials per gestionar entorns de BigQuery, assegurant la seguretat de les dades, el compliment normatiu i l'optimització del rendiment. Aquesta secció et guiarà des de les tècniques bàsiques fins a les avançades per a l'auditoria i el monitoratge a BigQuery.

Conceptes clau

  • Auditoria: Seguiment i registre de les activitats dels usuaris i l'accés a les dades.
  • Monitoratge: Observació del rendiment del sistema i l'ús de recursos per mantenir una operació òptima.

Auditoria a BigQuery

Comprensió dels registres d'auditoria

BigQuery s'integra amb el registre d'auditoria de Google Cloud per proporcionar registres detallats de les operacions realitzades en els teus conjunts de dades, taules i treballs.

  • Registres d'activitat administrativa: Registren accions administratives com la creació o eliminació de conjunts de dades.
  • Registres d'accés a les dades: Fan el seguiment de les operacions de lectura i escriptura sobre les dades.
  • Registres d'esdeveniments del sistema: Capturen esdeveniments del sistema com fallades de màquines virtuals o problemes de xarxa.

Activació dels registres d'auditoria

Els registres d'auditoria estan activats per defecte per a la majoria dels serveis de Google Cloud, incloent BigQuery. No obstant això, pots configurar-los i gestionar-los a través de la Consola de Google Cloud.

Visualització dels registres d'auditoria

Pots veure els registres d'auditoria utilitzant la Consola de Google Cloud, l'eina de línia de comandes gcloud o l'API de Cloud Logging.

Exemple: Visualització dels registres d'auditoria a la Consola de Google Cloud

  1. Ves a la Consola de Google Cloud.
  2. Navega a Logging > Logs Explorer.
  3. Utilitza el creador de consultes per filtrar els registres per a BigQuery.

Exemple: Utilització de gcloud per veure els registres

gcloud logging read "resource.type=bigquery_resource"

Anàlisi dels registres d'auditoria

Els registres d'auditoria es poden exportar a BigQuery per a una anàlisi detallada.

Exemple: Exportació de registres a BigQuery

  1. Ves a la Consola de Google Cloud.
  2. Navega a Logging > Logs Router.
  3. Crea un sink per exportar els registres a un conjunt de dades de BigQuery.
gcloud logging sinks create my-sink bigquery.googleapis.com/projects/my-project/datasets/my_dataset

Monitoratge a BigQuery

Monitoratge de mètriques

BigQuery proporciona diverses mètriques que es poden monitorar per assegurar un rendiment òptim i un ús adequat dels recursos.

  • Rendiment de consultes: Mètriques com el temps d'execució de consultes, bytes processats i slots utilitzats.
  • Ús de recursos: Mètriques per a la utilització de CPU, memòria i emmagatzematge.
  • Taxes d'error: Monitoratge de les taxes d'error per identificar i solucionar problemes.

Utilització de Cloud Monitoring

Google Cloud Monitoring et permet crear panells, configurar alertes i visualitzar mètriques.

Exemple: Creació d'un panell

  1. Ves a la Consola de Google Cloud.
  2. Navega a Monitoring > Dashboards.
  3. Crea un nou panell i afegeix widgets per visualitzar les mètriques de BigQuery.

Configuració d'alertes

Les alertes es poden configurar per notificar-te quan es compleixin condicions específiques, com ara alta latència de consultes o ús excessiu de recursos.

Exemple: Configuració d'una alerta

  1. Ves a la Consola de Google Cloud.
  2. Navega a Monitoring > Alerting.
  3. Crea una nova política d'alerta amb condicions basades en les mètriques de BigQuery.
gcloud alpha monitoring policies create \
  --display-name="High Query Latency" \
  --conditions="metric.type=\"bigquery.googleapis.com/query/latency\" AND metric.value > 1000"

Conclusió

L'auditoria i el monitoratge són pràctiques essencials per mantenir la seguretat, el compliment normatiu i el rendiment del teu entorn de BigQuery. Mitjançant l'ús de les eines de registre d'auditoria i monitoratge de Google Cloud, pots obtenir informació valuosa sobre les activitats dels usuaris, el rendiment del sistema i l'ús de recursos, permetent-te prendre decisions informades i mantenir una infraestructura de dades robusta.
