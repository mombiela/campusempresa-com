Introducció a les CTEs

Les Expressions de Taula Comunes (CTEs) són una característica poderosa en SQL que et permeten crear conjunts de resultats temporals que poden ser referenciats dins d'una instrucció SELECT, INSERT, UPDATE o DELETE. A BigQuery, les CTEs poden ajudar a simplificar consultes complexes, millorar la llegibilitat i fer que el teu codi SQL sigui més mantenible.

Conceptes Clau

  • Definició de CTE: Una CTE es defineix utilitzant la paraula clau WITH seguida d'una consulta que defineix el conjunt de resultats temporals.
  • Àmbit: La CTE només està disponible dins de l'àmbit d'execució de la instrucció en què es defineix.
  • Múltiples CTEs: Pots definir múltiples CTEs en una sola consulta, i poden referenciar-se entre elles.

Sintaxi Bàsica de les CTEs

La sintaxi bàsica per a una CTE a BigQuery és la següent:

WITH cte_name AS (
    SELECT column1, column2, ...
    FROM table_name
    WHERE condition
)
SELECT column1, column2, ...
FROM cte_name;

Exemple

Comencem amb un exemple senzill. Suposem que tenim una taula anomenada sales amb les següents columnes: sale_id, product_id, quantity i sale_date.

WITH recent_sales AS (
    SELECT sale_id, product_id, quantity, sale_date
    FROM sales
    WHERE sale_date > '2023-01-01'
)
SELECT *
FROM recent_sales;

En aquest exemple:

  • Definim una CTE anomenada recent_sales que selecciona totes les vendes realitzades després de l'1 de gener de 2023.
  • Després seleccionem totes les columnes de la CTE recent_sales.

Ús Avançat de les CTEs

CTEs Recursives

Les CTEs recursives s'utilitzen per realitzar operacions que requereixen processament iteratiu, com ara travessar estructures de dades jeràrquiques.

Sintaxi

WITH RECURSIVE cte_name AS (
    initial_query
    UNION ALL
    recursive_query
)
SELECT *
FROM cte_name;

Exemple

Considera una taula employees amb columnes employee_id, manager_id i employee_name. Volem llistar tots els empleats i els seus gerents en ordre jeràrquic.

WITH RECURSIVE employee_hierarchy AS (
    SELECT employee_id, manager_id, employee_name, 1 AS level
    FROM employees
    WHERE manager_id IS NULL
    UNION ALL
    SELECT e.employee_id, e.manager_id, e.employee_name, eh.level + 1
    FROM employees e
    JOIN employee_hierarchy eh ON e.manager_id = eh.employee_id
)
SELECT *
FROM employee_hierarchy;

En aquest exemple:

  • La consulta inicial selecciona tots els gerents de nivell superior (on manager_id és NULL).
  • La consulta recursiva uneix la taula employees amb la CTE employee_hierarchy per trobar empleats que reporten a cada gerent.

Aplicacions Pràctiques de les CTEs

Simplificant Consultes Complexes

Les CTEs poden descompondre consultes complexes en parts més simples i manejables.

Exemple

Suposem que volem trobar les vendes totals per a cada producte a la taula sales i després filtrar productes amb vendes totals superiors a 1000.

WITH product_sales AS (
    SELECT product_id, SUM(quantity) AS total_sales
    FROM sales
    GROUP BY product_id
)
SELECT product_id, total_sales
FROM product_sales
WHERE total_sales > 1000;

Millorant la Llegibilitat

Les CTEs poden fer que el teu codi SQL sigui més llegible donant noms significatius als conjunts de resultats intermedis.

Exemple

Considera una consulta que calcula les vendes mitjanes per producte i després troba productes amb vendes superiors a la mitjana.

WITH avg_sales AS (
    SELECT AVG(total_sales) AS avg_total_sales
    FROM (
        SELECT product_id, SUM(quantity) AS total_sales
        FROM sales
        GROUP BY product_id
    )
),
above_avg_sales AS (
    SELECT product_id, SUM(quantity) AS total_sales
    FROM sales
    GROUP BY product_id
    HAVING SUM(quantity) > (SELECT avg_total_sales FROM avg_sales)
)
SELECT *
FROM above_avg_sales;

Conclusió

Les Expressions de Taula Comunes (CTEs) són una eina versàtil a BigQuery que poden ajudar a simplificar consultes complexes, millorar la llegibilitat i gestionar dades jeràrquiques. Utilitzant CTEs, pots descompondre les teves consultes SQL en parts més manejables, fent que el teu codi sigui més fàcil d'entendre i mantenir. Tant si treballes amb CTEs simples com recursives, dominar aquesta característica millorarà significativament les teves capacitats de consulta SQL.
