Introducció

En aquesta secció, explorarem els conceptes fonamentals dels tipus de dades i esquemes a BigQuery. Entendre aquests conceptes és crucial per emmagatzemar, consultar i analitzar dades de manera eficient. Cobriren els tipus de dades bàsics, com definir esquemes i temes avançats com camps niats i repetits.

Tipus de Dades Bàsics

BigQuery admet una varietat de tipus de dades per acomodar diferents tipus de dades. Aquests són els principals tipus de dades que trobaràs:

  • STRING: Utilitzat per a dades de text.
  • INT64: Utilitzat per a nombres enters.
  • FLOAT64: Utilitzat per a nombres de punt flotant.
  • BOOL: Utilitzat per a valors booleans (TRUE o FALSE).
  • BYTES: Utilitzat per a dades binàries.
  • DATE: Utilitzat per a dates del calendari (sense hora).
  • DATETIME: Utilitzat per a data i hora (sense fus horari).
  • TIMESTAMP: Utilitzat per a data i hora amb fus horari.
  • TIME: Utilitzat per a l'hora del dia (sense data).
  • GEOGRAPHY: Utilitzat per a dades geogràfiques.

Exemple

CREATE TABLE my_dataset.my_table (
  name STRING,
  age INT64,
  height FLOAT64,
  is_student BOOL,
  profile_picture BYTES,
  birth_date DATE,
  last_login TIMESTAMP,
  daily_schedule TIME
);

En aquest exemple, definim una taula amb diverses columnes, cadascuna utilitzant un tipus de dada diferent.

Definició d'Esquemes

Un esquema a BigQuery és un pla que defineix l'estructura d'una taula, incloent els noms i tipus de dades de les columnes. Els esquemes es poden definir manualment o inferir automàticament quan es carrega la dada.

Definició Manual d'Esquema

Quan es crea una taula, es pot especificar l'esquema explícitament:

CREATE TABLE my_dataset.user_info (
  user_id INT64,
  user_name STRING,
  email STRING,
  signup_date DATE
);

Inferència Automàtica d'Esquema

BigQuery també pot inferir l'esquema de les dades quan es carrega des d'un fitxer:

bq load --autodetect --source_format=CSV my_dataset.user_info gs://my_bucket/user_info.csv

Camps Niats i Repetits

BigQuery admet camps niats i repetits, que permeten emmagatzemar estructures de dades complexes dins d'una taula. Això és particularment útil per manejar dades semblants a JSON.

Camps Niats

Els camps niats són camps que contenen altres camps. Es defineixen utilitzant el tipus de dada STRUCT.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  customer STRUCT<
    customer_id INT64,
    customer_name STRING,
    contact_info STRUCT<
      email STRING,
      phone STRING
    >
  >,
  order_date DATE
);

Camps Repetits

Els camps repetits són arrays de valors. Es defineixen utilitzant el tipus de dada ARRAY.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  items ARRAY<STRUCT<
    item_id INT64,
    item_name STRING,
    quantity INT64
  >>,
  order_date DATE
);

Combinació de Camps Niats i Repetits

Es poden combinar camps niats i repetits per crear estructures de dades encara més complexes.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  customer STRUCT<
    customer_id INT64,
    customer_name STRING
  >,
  items ARRAY<STRUCT<
    item_id INT64,
    item_name STRING,
    quantity INT64
  >>,
  order_date DATE
);

Conclusió

Entendre els tipus de dades i esquemes és fonamental per treballar de manera efectiva amb BigQuery. En dominar aquests conceptes, pots dissenyar estructures de dades eficients i escalables que satisfacin les teves necessitats analítiques. Tant si tractes amb taules simples com amb camps niats i repetits complexos, BigQuery proporciona la flexibilitat per manejar una àmplia gamma d'escenaris de dades.
