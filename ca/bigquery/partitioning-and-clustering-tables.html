Introducció

El particionament i el clustering són tècniques essencials a BigQuery que ajuden a optimitzar el rendiment de les consultes i gestionar grans conjunts de dades de manera eficient. Aquesta secció cobrirà els conceptes bàsics del particionament i el clustering, els seus beneficis i com implementar-los a BigQuery.

Particionament de Taules

Què és el Particionament?

El particionament és el procés de dividir una taula gran en peces més petites i manejables anomenades particions. Cada partició es guarda per separat, cosa que pot millorar significativament el rendiment de les consultes i reduir els costos.

Tipus de Particionament

  • Particionament basat en el temps: Divideix la taula en funció d'una columna de marca de temps o data.
  • Particionament per rang d'enters: Divideix la taula en funció d'una columna d'enters.

Beneficis del Particionament

  • Millora del Rendiment de les Consultes: Les consultes poden escanejar només les particions rellevants en lloc de tota la taula.
  • Eficiència de Costos: Redueix la quantitat de dades escanejades, disminuint així els costos de les consultes.
  • Gestió de Dades: Facilita la gestió i manteniment de grans conjunts de dades.

Implementació del Particionament basat en el temps

Per crear una taula particionada basada en una columna de marca de temps:

CREATE TABLE my_dataset.my_partitioned_table
PARTITION BY DATE(timestamp_column)
AS
SELECT * FROM my_dataset.my_source_table;

Implementació del Particionament per rang d'enters

Per crear una taula particionada basada en una columna d'enters:

CREATE TABLE my_dataset.my_partitioned_table
PARTITION BY RANGE_BUCKET(integer_column, GENERATE_ARRAY(1, 100, 10))
AS
SELECT * FROM my_dataset.my_source_table;

Consultar Taules Particionades

Quan es consulten taules particionades, utilitzeu la columna de partició a la clàusula WHERE per limitar les dades escanejades:

SELECT *
FROM my_dataset.my_partitioned_table
WHERE DATE(timestamp_column) = '2023-01-01';

Clustering de Taules

Què és el Clustering?

El clustering organitza les dades dins d'una taula particionada en funció dels valors d'una o més columnes. Això pot optimitzar encara més el rendiment de les consultes reduint la quantitat de dades escanejades dins de cada partició.

Beneficis del Clustering

  • Millora del Rendiment de les Consultes: Millora l'eficiència de les consultes que filtren o agreguen sobre columnes agrupades.
  • Reducció de l'Escaneig de Dades: Limita les dades escanejades dins de cada partició, cosa que condueix a una execució més ràpida de les consultes.

Implementació del Clustering

Per crear una taula agrupada:

CREATE TABLE my_dataset.my_clustered_table
PARTITION BY DATE(timestamp_column)
CLUSTER BY column1, column2
AS
SELECT * FROM my_dataset.my_source_table;

Consultar Taules Agrupades

Quan es consulten taules agrupades, utilitzeu les columnes agrupades a la clàusula WHERE per aprofitar el clustering:

SELECT *
FROM my_dataset.my_clustered_table
WHERE column1 = 'value1' AND column2 = 'value2';

Millors Pràctiques

Elecció de Columnes de Partició i Agrupació

  • Columnes de Partició: Trieu columnes que s'utilitzin freqüentment en consultes basades en dates o rangs.
  • Columnes d'Agrupació: Trieu columnes que s'utilitzin sovint en operacions de filtratge o agregació.

Combinació de Particionament i Clustering

La combinació d'ambdues tècniques pot proporcionar el millor rendiment per a grans conjunts de dades. Per exemple, particioneu per data i agrupeu per columnes consultades freqüentment:

CREATE TABLE my_dataset.my_optimized_table
PARTITION BY DATE(timestamp_column)
CLUSTER BY column1, column2
AS
SELECT * FROM my_dataset.my_source_table;

Conclusió

El particionament i el clustering són tècniques poderoses a BigQuery que poden millorar significativament el rendiment de les consultes i gestionar grans conjunts de dades de manera eficient. Comprenent i implementant aquestes tècniques, podeu optimitzar l'ús de BigQuery, reduir costos i millorar la gestió de dades. Trieu sempre les columnes de partició i agrupació en funció dels vostres patrons de consulta per maximitzar els beneficis.
