Introducció al Xifrat de Dades

El xifrat de dades és un aspecte fonamental de la seguretat de dades, assegurant que la informació sensible estigui protegida contra l'accés no autoritzat. A BigQuery, el xifrat s'utilitza per protegir les dades en repòs i en trànsit.

Conceptes Clau

  • Xifrat: El procés de convertir dades en un format codificat que només pot ser llegit per algú que tingui la clau de desxifrat.
  • Desxifrat: El procés de convertir dades xifrades de nou al seu format original.
  • Xifrat en Repòs: Protegir les dades emmagatzemades en disc.
  • Xifrat en Trànsit: Protegir les dades mentre es mouen a través de xarxes.
  • Claus de Xifrat Gestionades pel Client (CMEK): Claus gestionades pel client per xifrar dades.
  • Claus de Xifrat Gestionades per Google (GMEK): Claus gestionades per Google per xifrar dades.

Xifrat en Repòs

BigQuery xifra automàticament totes les dades en repòs utilitzant claus de xifrat gestionades per Google. No obstant això, per a un control addicional, podeu utilitzar Claus de Xifrat Gestionades pel Client (CMEK).

Utilitzant Claus de Xifrat Gestionades per Google (GMEK)

Per defecte, BigQuery utilitza GMEK per xifrar dades en repòs. Aquest procés és transparent per a l'usuari i no requereix configuració addicional.

Utilitzant Claus de Xifrat Gestionades pel Client (CMEK)

CMEK us permet utilitzar les vostres pròpies claus de xifrat emmagatzemades a Google Cloud Key Management Service (KMS).

Passos per Utilitzar CMEK a BigQuery

  1. Creeu un Anell de Claus i una Clau a Google Cloud KMS:

    gcloud kms keyrings create my-key-ring --location=global
gcloud kms keys create my-key --location=global --keyring=my-key-ring --purpose=encryption

  2. Concediu Accés al Compte de Servei de BigQuery a la Clau:

    gcloud kms keys add-iam-policy-binding my-key 
            --location=global 
            --keyring=my-key-ring 
            --member=serviceAccount:your-bigquery-service-account 
            --role=roles/cloudkms.cryptoKeyEncrypterDecrypter

  3. Creeu un Dataset de BigQuery amb CMEK:

    CREATE SCHEMA my_dataset
OPTIONS(
    kms_key_name="projects/my-project/locations/global/keyRings/my-key-ring/cryptoKeys/my-key"
);

Xifrat en Trànsit

BigQuery xifra les dades en trànsit utilitzant TLS (Transport Layer Security) per assegurar la integritat i privacitat de les dades mentre es mouen entre els vostres dispositius i els servidors de Google.

Assegurant Connexions Segures

  • Utilitzeu sempre HTTPS per interactuar amb BigQuery.
  • Utilitzeu API segures i biblioteques de clients proporcionades per Google Cloud.

Tècniques Avançades de Xifrat

Per a usuaris avançats, BigQuery admet funcions de xifrat addicionals com:

  • Xifrat a Nivell de Columna: Xifrant columnes específiques dins d'un dataset.
  • Xifrat Híbrid: Combinant xifrat simètric i asimètric per a una seguretat millorada.

Exemple: Xifrat a Nivell de Columna

Per xifrar columnes específiques, podeu utilitzar funcions com AEAD.ENCRYPT i AEAD.DECRYPT.

Xifrant una Columna

DECLARE key BYTES = FROM_HEX('your-hex-encoded-key');
DECLARE plaintext STRING = 'Dades Sensibles';

SET encrypted_data = AEAD.ENCRYPT(key, plaintext, '');

Desxifrant una Columna

DECLARE key BYTES = FROM_HEX('your-hex-encoded-key');
DECLARE ciphertext BYTES = 'your-encrypted-data';

SET decrypted_data = AEAD.DECRYPT(key, ciphertext, '');

Resum

El xifrat de dades a BigQuery és un component crític de la seguretat de dades, assegurant que les vostres dades estiguin protegides tant en repòs com en trànsit. Comprenent i utilitzant tant claus de xifrat gestionades per Google com gestionades pel client, podeu millorar la seguretat dels vostres datasets. A més, tècniques avançades de xifrat com el xifrat a nivell de columna proporcionen una major granularitat i control sobre la protecció de dades.

Seguint les millors pràctiques i aprofitant les funcions de xifrat proporcionades per BigQuery, podeu assegurar que les vostres dades es mantinguin segures i compleixin amb els estàndards de la indústria.
