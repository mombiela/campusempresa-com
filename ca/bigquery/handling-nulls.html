Gestionar els NULLs a BigQuery

Introducció als NULLs

  • Definició: En SQL, NULL representa un valor absent o indefinit.
  • Importància: Entendre com gestionar els valors NULL és crucial per a una anàlisi i manipulació de dades precisa.

Identificar els NULLs

  • IS NULL: S'utilitza per comprovar si un valor és NULL.
  • IS NOT NULL: S'utilitza per comprovar si un valor no és NULL.
SELECT *
FROM my_table
WHERE column_name IS NULL;
SELECT *
FROM my_table
WHERE column_name IS NOT NULL;

Gestionar els NULLs en Consultes

Utilitzant COALESCE

  • Propòsit: Substitueix NULL per un valor especificat.
  • Sintaxi: COALESCE(expression, replacement_value)
SELECT COALESCE(column_name, 'default_value') AS column_name
FROM my_table;

Utilitzant IFNULL

  • Propòsit: Similar a COALESCE, però només pren dos arguments.
  • Sintaxi: IFNULL(expression, replacement_value)
SELECT IFNULL(column_name, 'default_value') AS column_name
FROM my_table;

Utilitzant NULLIF

  • Propòsit: Retorna NULL si dues expressions són iguals.
  • Sintaxi: NULLIF(expression1, expression2)
SELECT NULLIF(column_name, 'value_to_compare') AS column_name
FROM my_table;

Agregar amb NULLs

  • COUNT: COUNT(column_name) ignora els valors NULL, mentre que COUNT(*) els inclou.
  • SUM, AVG, MIN, MAX: Aquestes funcions ignoren els valors NULL.
SELECT COUNT(column_name) AS non_null_count,
       COUNT(*) AS total_count
FROM my_table;

Gestió Avançada dels NULLs

Utilitzant Sentències CASE

  • Propòsit: Proporciona una gestió condicional més complexa dels valors NULL.
  • Sintaxi: CASE WHEN condition THEN result ELSE result END
SELECT CASE 
           WHEN column_name IS NULL THEN 'No Value'
           ELSE column_name
       END AS column_name
FROM my_table;

NULLs en Unions

  • LEFT JOIN: Inclou tots els registres de la taula esquerra i els registres coincidents de la taula dreta. Si no hi ha coincidència, es retornen valors NULL per a les columnes de la taula dreta.
  • INNER JOIN: Exclou els registres amb valors NULL en la condició de la unió.
SELECT a.*, b.*
FROM table_a a
LEFT JOIN table_b b
ON a.id = b.id;

NULLs en Tipus de Dades

  • String: NULL és diferent d'una cadena buida ''.
  • Numeric: NULL és diferent de zero 0.

| Tipus de Dada | Exemple de NULL | Exemple de No-NULL | |---------------|-----------------|--------------------| | String | NULL | '' | | Numeric | NULL | 0 |

Conclusió

Gestionar els valors NULL de manera efectiva és essencial per a una anàlisi de dades precisa a BigQuery. Utilitzant funcions com COALESCE, IFNULL i NULLIF, així com entenent com els valors NULL interactuen amb unions i agregacions, pots assegurar-te que les teves consultes retornin els resultats correctes. Sempre tingues en compte com es tracten els valors NULL en diferents contextos per evitar resultats inesperats en la teva anàlisi de dades.
