En aquesta secció, explorarem com gestionar el control d'accés a BigQuery. El control d'accés és crucial per assegurar que només els usuaris autoritzats puguin accedir o modificar les teves dades. Cobrirem els conceptes bàsics del control d'accés, com establir permisos i tècniques avançades per gestionar l'accés.

Introducció al Control d'Accés

El control d'accés a BigQuery es gestiona a través de la Gestió d'Identitat i Accés (IAM). IAM et permet controlar qui té accés als teus recursos de BigQuery i què poden fer amb aquests recursos.

  • Rols IAM: Els rols són una col·lecció de permisos. BigQuery proporciona rols predefinits que pots assignar als usuaris.
  • Polítiques IAM: Les polítiques s'utilitzen per atorgar rols a usuaris, grups o comptes de servei.

Configuració dels Rols IAM

BigQuery proporciona diversos rols predefinits que pots utilitzar per gestionar l'accés. Aquests rols van des de permisos amplis fins a permisos molt específics.

Rols Predefinits

  • Visualitzador: Accés de només lectura a conjunts de dades i taules.
  • Editor: Accés de lectura i escriptura a conjunts de dades i taules.
  • Propietari: Control total sobre conjunts de dades i taules, inclosa la capacitat d'atorgar accés a altres.
  • Administrador de BigQuery: Control total sobre tots els recursos de BigQuery en un projecte.

Rols Personalitzats

Si els rols predefinits no satisfan les teves necessitats, pots crear rols personalitzats amb permisos específics.

Assignació de Rols

Per assignar rols, has d'actualitzar la política IAM del teu projecte, conjunt de dades o taula.

Exemple: Assignació d'un Rol a un Usuari

# Assigna el rol de Visualitzador de Dades de BigQuery a un usuari per a un conjunt de dades específic
gcloud projects add-iam-policy-binding my-project \
    --member=user:example-user@gmail.com \
    --role=roles/bigquery.dataViewer

Exemple: Assignació d'un Rol a un Compte de Servei

# Assigna el rol d'Editor de Dades de BigQuery a un compte de servei per a un conjunt de dades específic
gcloud projects add-iam-policy-binding my-project \
    --member=serviceAccount:my-service-account@my-project.iam.gserviceaccount.com \
    --role=roles/bigquery.dataEditor

Gestió de Permisos de Conjunt de Dades i Taula

També pots establir permisos a nivell de conjunt de dades i taula per a un control més granular.

Exemple: Atorgar Accés a un Conjunt de Dades

# Atorga el rol de Visualitzador de Dades de BigQuery a un usuari per a un conjunt de dades específic
bq update --dataset --add_iam_policy_binding \
    my-project:my_dataset \
    user:example-user@gmail.com:roles/bigquery.dataViewer

Exemple: Atorgar Accés a una Taula

# Atorga el rol d'Editor de Dades de BigQuery a un usuari per a una taula específica
bq update --table --add_iam_policy_binding \
    my-project:my_dataset.my_table \
    user:example-user@gmail.com:roles/bigquery.dataEditor

Tècniques Avançades de Control d'Accés

Ús de Llistes de Control d'Accés (ACLs)

Les ACLs proporcionen una manera de gestionar l'accés a un nivell més granular que els rols IAM. Pots utilitzar ACLs per atorgar accés a columnes específiques dins d'una taula.

Exemple: Establir Permisos a Nivell de Columna

GRANT SELECT(column_name) ON TABLE my_dataset.my_table TO 'user:example-user@gmail.com';

Ús de Vistes Autoritzades

Les vistes autoritzades et permeten compartir resultats de consultes amb usuaris específics sense donar-los accés a les taules subjacents.

Exemple: Crear una Vista Autoritzada

  1. Crear una vista:

    CREATE VIEW my_dataset.my_view AS
SELECT column1, column2
FROM my_dataset.my_table;

  2. Autoritzar la vista:

    bq update --view --add_iam_policy_binding 
            my-project:my_dataset.my_view 
            user:example-user@gmail.com:roles/bigquery.dataViewer

Conclusió

Gestionar el control d'accés a BigQuery és essencial per mantenir la seguretat i la integritat de les teves dades. Entenent i utilitzant els rols IAM, polítiques, ACLs i vistes autoritzades, pots assegurar-te que només els usuaris autoritzats tinguin accés als teus recursos de BigQuery. Practica assignant rols i establint permisos per convertir-te en un expert en la gestió del control d'accés a BigQuery.
