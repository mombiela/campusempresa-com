La indexació de dades a Elasticsearch és un concepte fonamental que et permet emmagatzemar, cercar i analitzar grans volums de dades de manera ràpida i gairebé en temps real. Aquest tema cobrirà els conceptes bàsics de la indexació, com crear i gestionar índexs, i tècniques avançades per optimitzar el teu procés de indexació de dades.

Introducció a la Indexació

La indexació a Elasticsearch és el procés d'afegir dades a un índex. Un índex és una col·lecció de documents que comparteixen característiques similars. Cada document és un objecte JSON que conté camps, que són les dades que vols cercar i analitzar.

Conceptes Clau

  • Índex: Una col·lecció de documents.
  • Document: Un objecte JSON que conté dades.
  • Camp: Una parella clau-valor dins d'un document.
  • Mapeig: Defineix com es guarden i indexen els documents i els seus camps.

Creació d'un Índex

Per crear un índex a Elasticsearch, pots utilitzar la sol·licitud PUT. Aquí tens un exemple senzill:

PUT /my_first_index
{
  "settings": {
    "number_of_shards": 1,
    "number_of_replicas": 1
  }
}
  • number_of_shards: Defineix el nombre de fragments primaris.
  • number_of_replicas: Defineix el nombre de fragments de rèplica.

Exemple Explicat

En l'exemple anterior, creem un índex anomenat my_first_index amb 1 fragment primari i 1 fragment de rèplica. Els fragments són les unitats bàsiques d'emmagatzematge a Elasticsearch, i les rèpliques són còpies dels fragments per a la tolerància a fallades.

Afegir Documents a un Índex

Per afegir un document a un índex, utilitzes la sol·licitud POST o PUT. Aquí tens un exemple:

POST /my_first_index/_doc/1
{
  "name": "John Doe",
  "age": 30,
  "occupation": "Software Engineer"
}
  • /my_first_index/_doc/1: Especifica l'índex i l'ID del document.
  • name, age, occupation: Camps dins del document.

Exemple Explicat

En aquest exemple, afegim un document amb ID 1 a l'índex my_first_index. El document conté tres camps: name, age i occupation.

Actualització de Documents

Per actualitzar un document existent, pots utilitzar la sol·licitud POST amb l'endpoint _update. Aquí tens un exemple:

POST /my_first_index/_update/1
{
  "doc": {
    "age": 31
  }
}

Exemple Explicat

En aquest exemple, actualitzem el document amb ID 1 a l'índex my_first_index, canviant el camp age a 31.

Eliminació de Documents

Per eliminar un document, utilitzes la sol·licitud DELETE. Aquí tens un exemple:

DELETE /my_first_index/_doc/1

Exemple Explicat

En aquest exemple, eliminem el document amb ID 1 de l'índex my_first_index.

Tècniques Avançades de Indexació

Indexació Massiva

Per obtenir un millor rendiment, especialment quan es treballa amb grans conjunts de dades, pots utilitzar l'API _bulk per indexar múltiples documents en una sola sol·licitud. Aquí tens un exemple:

POST /_bulk
{ "index": { "_index": "my_first_index", "_id": "2" } }
{ "name": "Jane Doe", "age": 25, "occupation": "Data Scientist" }
{ "index": { "_index": "my_first_index", "_id": "3" } }
{ "name": "Alice Smith", "age": 28, "occupation": "Product Manager" }

Exemple Explicat

En aquest exemple, utilitzem l'API _bulk per afegir dos documents a l'índex my_first_index en una sola sol·licitud.

Aliases d'Índex

Els aliases d'índex et permeten utilitzar un sol nom per referir-te a un o més índexs. Això pot ser útil per gestionar versions d'índexs i realitzar reindexacions sense temps d'inactivitat. Aquí tens un exemple:

POST /_aliases
{
  "actions": [
    { "add": { "index": "my_first_index", "alias": "current_index" } }
  ]
}

Exemple Explicat

En aquest exemple, creem un alias anomenat current_index que apunta a l'índex my_first_index.

Conclusió

La indexació de dades a Elasticsearch és un pas crucial per emmagatzemar i recuperar dades de manera eficient. Entenent els conceptes bàsics de la creació d'índexs, l'afegiment, l'actualització i l'eliminació de documents, així com tècniques avançades com la indexació massiva i els aliases d'índex, pots optimitzar l'ús d'Elasticsearch per obtenir un millor rendiment i escalabilitat.
