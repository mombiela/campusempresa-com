Introducció al Mapeig en Elasticsearch

El mapeig en Elasticsearch és el procés de definir com es guarden i s'indexen un document i els camps que conté. És similar a definir un esquema en una base de dades relacional. Entendre el mapeig és crucial per a un indexat i consulta de dades eficients.

Conceptes Clau

  • Document: Un objecte JSON que es guarda a Elasticsearch.
  • Índex: Una col·lecció de documents.
  • Camp: Una parella clau-valor en un document.
  • Mapeig: El procés de definir com es guarden i s'indexen els documents i els seus camps.

Mapeig Bàsic

Quan indexeu un document a Elasticsearch sense definir explícitament un mapeig, Elasticsearch crearà automàticament un mapeig dinàmic per a vosaltres. No obstant això, per a un millor control i rendiment, sovint és beneficiós definir els vostres propis mapeigs.

Exemple de Mapeig Dinàmic

PUT /my_index/_doc/1
{
  "name": "John Doe",
  "age": 30,
  "email": "john.doe@example.com"
}

En aquest exemple, Elasticsearch inferirà automàticament els tipus de dades dels camps.

Definició de Mapeigs Explícits

Podeu definir mapeigs explícits quan creeu un índex per assegurar-vos que els camps s'indexen correctament.

PUT /my_index
{
  "mappings": {
    "properties": {
      "name": {
        "type": "text"
      },
      "age": {
        "type": "integer"
      },
      "email": {
        "type": "keyword"
      }
    }
  }
}

En aquest exemple:

  • name es defineix com text, adequat per a la cerca de text complet.
  • age es defineix com integer, adequat per a operacions numèriques.
  • email es defineix com keyword, adequat per a coincidències exactes.

Tipus de Dades a Elasticsearch

Elasticsearch admet una varietat de tipus de dades, cadascun optimitzat per a diferents tipus de dades i consultes.

Tipus de Dades Bàsics

  • Tipus de Cadena:
    • text: Utilitzat per a la cerca de text complet.
    • keyword: Utilitzat per a coincidències exactes.
  • Tipus Numèrics:
    • integer: Enter de 32 bits.
    • long: Enter de 64 bits.
    • float: Punt flotant de precisió simple.
    • double: Punt flotant de precisió doble.
  • Data: Emmagatzema dates en diversos formats.
  • Boolean: Emmagatzema true o false.

Exemple de Diferents Tipus de Dades

PUT /my_index
{
  "mappings": {
    "properties": {
      "title": {
        "type": "text"
      },
      "price": {
        "type": "float"
      },
      "release_date": {
        "type": "date"
      },
      "in_stock": {
        "type": "boolean"
      }
    }
  }
}

En aquest exemple:

  • title és un camp text per a la cerca de text complet.
  • price és un camp float per a operacions numèriques.
  • release_date és un camp date per a operacions de data.
  • in_stock és un camp boolean per a valors vertader/fals.

Tècniques Avançades de Mapeig

Objectes Imbricats

Els objectes imbricats us permeten indexar arrays d'objectes i consultar-los com si fossin documents separats.

PUT /my_index
{
  "mappings": {
    "properties": {
      "user": {
        "type": "nested",
        "properties": {
          "first_name": {
            "type": "text"
          },
          "last_name": {
            "type": "text"
          }
        }
      }
    }
  }
}

Camps Múltiples

Els camps múltiples permeten que un sol camp s'indexi de diverses maneres per a diferents tipus de consultes.

PUT /my_index
{
  "mappings": {
    "properties": {
      "name": {
        "type": "text",
        "fields": {
          "keyword": {
            "type": "keyword"
          }
        }
      }
    }
  }
}

En aquest exemple, el camp name s'indexa tant com text per a la cerca de text complet com keyword per a coincidències exactes.

Conclusió

Entendre i definir mapeigs a Elasticsearch és essencial per a un indexat i consulta de dades eficients. Definint mapeigs explícits, podeu optimitzar els vostres índexs d'Elasticsearch per a un millor rendiment i resultats de cerca més precisos. Recordeu triar els tipus de dades adequats per als vostres camps i considereu tècniques avançades com els objectes imbricats i els camps múltiples per gestionar estructures de dades complexes.
