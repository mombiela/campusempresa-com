Introducció

El monitoratge i el manteniment són aspectes crítics de la gestió d'un clúster Elasticsearch. Un monitoratge adequat assegura que puguis detectar i abordar problemes abans que afectin el rendiment del teu sistema. Les tasques de manteniment ajuden a mantenir el teu clúster funcionant de manera suau i eficient.

Monitoratge d'Elasticsearch

Conceptes Clau

  • Salut del Clúster: Indica l'estat general del clúster.
  • Estadístiques del Node: Proporciona informació detallada sobre cada node del clúster.
  • Estadístiques de l'Índex: Ofereix informació sobre el rendiment i l'estat dels índexs.
  • Registre: Captura esdeveniments i errors per a la resolució de problemes.

Salut del Clúster

La salut del clúster es pot comprovar utilitzant l'API _cluster/health. L'estat de salut pot ser verd, groc o vermell.

GET /_cluster/health
  • Verd: Tots els fragments primaris i rèpliques estan actius.
  • Groc: Tots els fragments primaris estan actius, però algunes rèpliques no ho estan.
  • Vermell: Alguns fragments primaris no estan actius.

Estadístiques del Node

Les estadístiques del node proporcionen informació detallada sobre el rendiment i l'ús de recursos de cada node.

GET /_nodes/stats
  • Índexs: Informació sobre operacions d'índex.
  • OS: Mètriques del sistema operatiu.
  • Procés: Mètriques del procés JVM.
  • JVM: Mètriques de la Màquina Virtual Java.
  • Pool de Fils: Estadístiques del pool de fils.

Estadístiques de l'Índex

Les estadístiques de l'índex donen informació sobre el rendiment i l'estat dels índexs.

GET /_stats
  • Docs: Nombre de documents.
  • Emmagatzematge: Mida de l'emmagatzematge.
  • Indexació: Operacions d'indexació.
  • Cerca: Operacions de cerca.

Registre

Els registres d'Elasticsearch són essencials per a la resolució de problemes i per entendre el comportament del teu clúster. Els registres es poden trobar al directori logs de la teva instal·lació d'Elasticsearch.

Tasques de Manteniment

Conceptes Clau

  • Instantània i Restauració: Còpia de seguretat i restauració de les teves dades.
  • Gestió d'Índexs: Optimitza i gestiona els índexs.
  • Actualitzacions del Clúster: Actualitza les versions d'Elasticsearch.
  • Gestió de Nodes: Afegeix o elimina nodes del clúster.

Instantània i Restauració

Les instantànies són còpies de seguretat dels teus índexs que es poden emmagatzemar en un repositori. Pots restaurar aquestes instantànies per recuperar dades.

Crear una Instantània

PUT /_snapshot/my_backup/snapshot_1

Restaurar una Instantània

POST /_snapshot/my_backup/snapshot_1/_restore

Gestió d'Índexs

Gestionar regularment els índexs ajuda a mantenir el rendiment i l'eficiència.

Fusió Forçada

La fusió forçada redueix el nombre de segments en cada fragment.

POST /my_index/_forcemerge

Eliminar Índexs Antics

Automatitza l'eliminació d'índexs antics per estalviar espai.

DELETE /my_old_index

Actualitzacions del Clúster

Actualitzar Elasticsearch implica diversos passos per assegurar una transició suau.

Comprovar Compatibilitat

Assegura't que els teus plugins i configuracions siguin compatibles amb la nova versió.

GET /_nodes/plugins

Realitzar una Actualització Progressiva

Actualitza els nodes un a un per minimitzar el temps d'inactivitat.

Gestió de Nodes

Afegir o eliminar nodes pot ajudar a escalar el teu clúster o desmantellar maquinari antic.

Afegir un Node

Actualitza el fitxer de configuració elasticsearch.yml i inicia el nou node.

Eliminar un Node

Utilitza l'API _cluster/settings per excloure el node i després apaga'l.

PUT /_cluster/settings
{
  "transient": {
    "cluster.routing.allocation.exclude._name": "node_name"
  }
}

Conclusió

El monitoratge i el manteniment són essencials per a la salut i el rendiment del teu clúster Elasticsearch. Comprovant regularment la salut del clúster, monitoritzant les estadístiques de nodes i índexs, i realitzant tasques de manteniment rutinàries com instantànies i restauracions, gestió d'índexs i gestió de nodes, pots assegurar-te que el teu clúster funcioni de manera suau i eficient. Un registre adequat i actualitzacions oportunes també juguen un paper crucial en el manteniment d'un entorn Elasticsearch robust.
