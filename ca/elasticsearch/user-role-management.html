La gestió d'usuaris i rols és un aspecte crucial per assegurar el teu clúster d'Elasticsearch. Aquest tema et guiarà des dels conceptes bàsics fins als avançats, assegurant que entenguis com gestionar usuaris i rols de manera efectiva.

Introducció a la Gestió d'Usuaris i Rols

Elasticsearch proporciona un model de seguretat robust que inclou l'autenticació d'usuaris i el control d'accés basat en rols (RBAC). Això assegura que només els usuaris autoritzats puguin accedir i realitzar accions sobre les teves dades.

Conceptes Clau

  • Usuari: Un compte individual que pot autenticar-se i realitzar accions a Elasticsearch.
  • Rol: Un conjunt de permisos que es poden assignar als usuaris per controlar el seu accés als recursos.
  • RBAC (Control d'Accés Basat en Rols): Un mètode de regulació de l'accés als recursos basat en els rols assignats als usuaris individuals.

Creació i Gestió d'Usuaris

Els usuaris a Elasticsearch són entitats que poden autenticar-se i realitzar accions basades en els rols assignats.

Creació d'un Usuari

Per crear un usuari, pots utilitzar la interfície de Kibana o l'API d'Elasticsearch. Aquí, ens centrarem en l'ús de l'API.

PUT /_security/user/john_doe
{
  "password" : "password123",
  "roles" : [ "admin", "read_only" ],
  "full_name" : "John Doe",
  "email" : "john.doe@example.com"
}
  • Endpoint: /security/user/{username}
  • Camps:
    • password: La contrasenya de l'usuari.
    • roles: Una matriu de rols assignats a l'usuari.
    • full_name: El nom complet de l'usuari.
    • email: L'adreça de correu electrònic de l'usuari.

Gestió d'Usuaris

Pots actualitzar, eliminar o recuperar informació d'usuaris utilitzant l'API d'Elasticsearch.

  • Actualitzar Usuari:

    POST /_security/user/john_doe/_password
{
  "password" : "new_password123"
}

  • Eliminar Usuari:

    DELETE /_security/user/john_doe

  • Recuperar Informació d'Usuari:

    GET /_security/user/john_doe

Creació i Gestió de Rols

Els rols defineixen els permisos que tenen els usuaris dins del clúster d'Elasticsearch.

Creació d'un Rol

Els rols es poden crear utilitzant l'API d'Elasticsearch.

PUT /_security/role/read_only
{
  "cluster": ["monitor"],
  "indices": [
    {
      "names": [ "index1", "index2" ],
      "privileges": [ "read" ]
    }
  ]
}
  • Endpoint: /security/role/{role_name}
  • Camps:
    • cluster: Permisos a nivell de clúster.
    • indices: Permisos específics d'índex.

Gestió de Rols

Pots actualitzar, eliminar o recuperar informació de rols utilitzant l'API d'Elasticsearch.

  • Actualitzar Rol:

    PUT /_security/role/read_only
{
  "cluster": ["monitor"],
  "indices": [
    {
      "names": [ "index1", "index2" ],
      "privileges": [ "read", "write" ]
    }
  ]
}

  • Eliminar Rol:

    DELETE /_security/role/read_only

  • Recuperar Informació de Rol:

    GET /_security/role/read_only

Assignació de Rols als Usuaris

Els rols es poden assignar als usuaris durant la creació de l'usuari o actualitzar-se més tard.

Assignació de Rols Durant la Creació de l'Usuari

Com es mostra en l'exemple de creació d'usuari:

PUT /_security/user/john_doe
{
  "password" : "password123",
  "roles" : [ "admin", "read_only" ],
  "full_name" : "John Doe",
  "email" : "john.doe@example.com"
}

Actualització de Rols d'Usuari

Pots actualitzar els rols assignats a un usuari utilitzant la següent crida d'API:

POST /_security/user/john_doe/_roles
{
  "roles" : [ "admin", "read_only", "new_role" ]
}

Millors Pràctiques

  • Utilitza Contrasenyes Fortes: Assegura't que les contrasenyes dels usuaris siguin fortes i es canviïn regularment.
  • Principi del Menor Privilegi: Assigna els permisos mínims necessaris als usuaris.
  • Auditories Regulars: Revisa regularment els rols i permisos dels usuaris per assegurar-te que estiguin actualitzats i siguin adequats.

Conclusió

La gestió d'usuaris i rols a Elasticsearch és essencial per assegurar el teu clúster i assegurar que només els usuaris autoritzats puguin accedir i realitzar accions sobre les teves dades. Entenent com crear, gestionar i assignar rols als usuaris, pots implementar un model de seguretat robust que protegeixi el teu entorn d'Elasticsearch.
