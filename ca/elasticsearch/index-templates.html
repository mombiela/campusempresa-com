Les plantilles d'índex a Elasticsearch et permeten definir configuracions, mapatges i àlies que s'apliquen automàticament als nous índexs. Són una característica potent per gestionar i estandarditzar la configuració dels índexs en el teu clúster d'Elasticsearch.

Conceptes Clau

  • Plantilla d'Índex: Un conjunt de configuracions que es poden aplicar als nous índexs.
  • Configuracions: Opcions de configuració per a un índex, com ara el nombre de fragments i rèpliques.
  • Mapatges: Defineixen com es guarden i s'indexen els documents i els seus camps.
  • Àlies: Noms alternatius per als índexs, que poden simplificar les consultes i la gestió d'índexs.

Creació d'una Plantilla d'Índex

Per crear una plantilla d'índex, cal definir-la utilitzant un objecte JSON i enviar-la al punt final _template. Aquí tens un exemple bàsic:

PUT _template/my_template
{
  "index_patterns": ["log-*"],
  "settings": {
    "number_of_shards": 1
  },
  "mappings": {
    "properties": {
      "timestamp": {
        "type": "date"
      },
      "message": {
        "type": "text"
      }
    }
  },
  "aliases": {
    "logs": {}
  }
}

Explicació

  • index_patterns: Especifica els patrons de nom d'índex als quals s'ha d'aplicar aquesta plantilla. En aquest cas, s'aplica a qualsevol índex que comenci amb "log-".
  • settings: Defineix les configuracions de l'índex. Aquí, establim el nombre de fragments a 1.
  • mappings: Defineix els mapatges de camps. Tenim dos camps: timestamp de tipus date i message de tipus text.
  • aliases: Defineix un àlies anomenat logs per als índexs que coincideixen amb el patró.

Aplicació de Plantilles d'Índex

Quan crees un nou índex que coincideix amb el patró especificat a la plantilla, Elasticsearch aplica automàticament les configuracions, mapatges i àlies de la plantilla al nou índex.

Per exemple, creant un índex anomenat log-2023-10-01:

PUT log-2023-10-01

Aquest índex heretarà les configuracions, mapatges i àlies definits a my_template.

Gestió de Plantilles d'Índex

Llistar Plantilles d'Índex

Per llistar totes les plantilles d'índex, utilitza la següent comanda:

GET _template

Recuperar una Plantilla Específica

Per recuperar una plantilla específica, utilitza:

GET _template/my_template

Actualització d'una Plantilla d'Índex

Per actualitzar una plantilla existent, utilitza la mateixa petició PUT amb l'objecte JSON actualitzat. Elasticsearch sobreescriurà la plantilla existent amb la nova definició.

Eliminació d'una Plantilla d'Índex

Per eliminar una plantilla d'índex, utilitza la següent comanda:

DELETE _template/my_template

Exemple: Plantilla d'Índex Avançada

Aquí tens un exemple més avançat que inclou configuracions i mapatges addicionals:

PUT _template/advanced_template
{
  "index_patterns": ["data-*"],
  "order": 1,
  "settings": {
    "number_of_shards": 3,
    "number_of_replicas": 2,
    "refresh_interval": "30s"
  },
  "mappings": {
    "properties": {
      "user_id": {
        "type": "keyword"
      },
      "created_at": {
        "type": "date",
        "format": "yyyy-MM-dd HH:mm:ss||epoch_millis"
      },
      "content": {
        "type": "text",
        "analyzer": "standard"
      }
    }
  },
  "aliases": {
    "data_alias": {}
  }
}

Explicació

  • order: Determina l'ordre d'aplicació si coincideixen múltiples plantilles. Els valors més alts tenen més precedència.
  • refresh_interval: Estableix l'interval en què s'actualitza l'índex.
  • mappings: Inclou un camp user_id de tipus keyword, un camp created_at amb un format de data personalitzat, i un camp content amb un analitzador estàndard.

Conclusió

Les plantilles d'índex a Elasticsearch són essencials per gestionar i estandarditzar la configuració dels índexs. Ajuden a garantir la consistència entre els índexs i simplifiquen el procés de creació d'índexs. Entenent i utilitzant les plantilles d'índex, pots optimitzar el rendiment i la mantenibilitat del teu clúster d'Elasticsearch.
