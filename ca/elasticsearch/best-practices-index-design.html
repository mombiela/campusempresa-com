Introducció

Elasticsearch és un potent motor de cerca i analítica que depèn molt d'índexs ben dissenyats per funcionar de manera eficient. Aquesta secció cobrirà les millors pràctiques per dissenyar índexs a Elasticsearch, des de conceptes bàsics fins a tècniques avançades.

Comprendre els Índexs

  • Definició: Un índex a Elasticsearch és una col·lecció de documents que comparteixen característiques similars.
  • Propòsit: Els índexs s'utilitzen per organitzar dades i fer-les cercables.

Conceptes Clau en el Disseny d'Índexs

  • Shards: Elasticsearch divideix els índexs en peces més petites anomenades shards, que es poden distribuir a través de múltiples nodes.
  • Rèpliques: Còpies de shards que proporcionen redundància i milloren el rendiment de cerca.
  • Mappings: Defineixen l'estructura dels documents en un índex, incloent-hi tipus de camps i analitzadors.

Millors Pràctiques

1. Planifica l'Estructura del Teu Índex

  • Tipus de Documents: Evita utilitzar múltiples tipus en un sol índex. En lloc d'això, utilitza un sol tipus per índex.
  • Nomenclatura de Camps: Utilitza noms de camps clars, consistents i descriptius.
  • Evita la Nesting Profunda: Limita l'ús d'objectes nested per evitar problemes de rendiment.

2. Optimitza l'Assignació de Shards

  • Mida de Shard: Apunta a mides de shard entre 10GB i 50GB.
  • Nombre de Shards: Equilibra el nombre de shards per evitar tenir massa shards petits o massa pocs shards grans.
  • Rèpliques: Estableix un nombre adequat de rèpliques segons la teva necessitat de redundància i rendiment de lectura.

3. Utilitza Mappings Adequats

  • Tipus de Camps: Tria els tipus de camps correctes (per exemple, text, keyword, date, integer).
  • Analitzadors: Utilitza analitzadors personalitzats si els predeterminats no compleixen els teus requisits.
  • Mapping Dinàmic: Desactiva el mapping dinàmic en entorns de producció per evitar tipus de camps inesperats.

4. Estratègies d'Indexació

  • Indexació en Bloc: Utilitza la indexació en bloc per millorar el rendiment d'indexació.
  • Interval de Refresc: Ajusta l'interval de refresc per equilibrar entre la velocitat d'indexació i la latència de cerca.
  • Plantilles d'Índex: Utilitza plantilles d'índex per aplicar configuracions i mappings consistents a nous índexs.

5. Monitoritza i Mantén els Índexs

  • Gestió del Cicle de Vida de l'Índex (ILM): Utilitza polítiques ILM per automatitzar tasques de gestió d'índexs com rollover, shrink i delete.
  • Eines de Monitorització: Utilitza eines de monitorització d'Elasticsearch per fer un seguiment del rendiment i la salut dels índexs.
  • Reindexació: Planifica la reindexació quan facis canvis significatius en els mappings o configuracions.

Exemples

Exemple 1: Creació d'un Índex amb Mappings Adequats

PUT /my_index
{
  "settings": {
    "number_of_shards": 3,
    "number_of_replicas": 1
  },
  "mappings": {
    "properties": {
      "title": {
        "type": "text",
        "analyzer": "standard"
      },
      "author": {
        "type": "keyword"
      },
      "publish_date": {
        "type": "date"
      },
      "content": {
        "type": "text",
        "analyzer": "english"
      }
    }
  }
}
  • Explicació: Aquest exemple crea un índex amb 3 shards primaris i 1 rèplica. Els mappings defineixen l'estructura dels documents, especificant tipus de camps i analitzadors.

Exemple 2: Utilització de la Indexació en Bloc

POST /_bulk
{ "index" : { "_index" : "my_index", "_id" : "1" } }
{ "title" : "Elasticsearch Basics", "author" : "John Doe", "publish_date" : "2023-01-01", "content" : "Introduction to Elasticsearch." }
{ "index" : { "_index" : "my_index", "_id" : "2" } }
{ "title" : "Advanced Elasticsearch", "author" : "Jane Smith", "publish_date" : "2023-02-01", "content" : "Deep dive into Elasticsearch." }
  • Explicació: Aquest exemple demostra com utilitzar l'API bulk per indexar múltiples documents en una sola sol·licitud, que és més eficient que indexar documents un per un.

Conclusió

Dissenyar índexs a Elasticsearch requereix una planificació acurada i consideració de diversos factors com l'assignació de shards, mappings i estratègies d'indexació. Seguint les millors pràctiques, pots assegurar-te que els teus índexs estan optimitzats per al rendiment, l'escalabilitat i el manteniment. La monitorització i el manteniment regulars també són crucials per mantenir els teus índexs saludables i amb bon rendiment.
