Introducció

La resolució de problemes d'Elasticsearch implica identificar, diagnosticar i resoldre problemes que poden sorgir durant la seva operació. Aquesta secció cobrirà problemes comuns, les seves causes i solucions, des de nivells bàsics fins a avançats.

Problemes comuns i solucions

Salut del clúster

  • Estat groc: Indica que algunes rèpliques no estan assignades.
    • Solució: Comproveu si hi ha prou nodes per assignar rèpliques. Utilitzeu l'API _cluster/health per obtenir més detalls. 
      GET /_cluster/health
  • Estat vermell: Indica que algunes particions primàries no estan assignades.
    • Solució: Investigueu els registres per trobar errors, comproveu l'espai en disc i assegureu-vos que tots els nodes estiguin en funcionament.

Problemes de nodes

  • El node no s'uneix al clúster: Un node pot no unir-se al clúster a causa de problemes de xarxa o errors de configuració.
    • Solució: Verifiqueu la connectivitat de la xarxa i assegureu-vos que la configuració de elasticsearch.yml sigui correcta, especialment els paràmetres cluster.name i discovery.seed_hosts. 
      cluster.name: my-cluster
discovery.seed_hosts: ["host1", "host2"]

Problemes d'indexació

  • Indexació lenta: El rendiment de la indexació pot degradar-se per diversos factors.
    • Solució: Comproveu els colls d'ampolla de recursos (CPU, memòria, I/O de disc), optimitzeu els mapatges i considereu utilitzar la indexació massiva. 
      POST /_bulk
{ "index" : { "_index" : "test", "_id" : "1" } }
{ "field1" : "value1" }

Problemes de cerca

  • Cerques lentes: El rendiment lent de les cerques pot ser causat per consultes ineficients o manca de recursos.
    • Solució: Utilitzeu l'API _search amb perfilat per identificar les parts lentes de la consulta. 
      GET /_search
{
  "profile": true,
  "query": {
    "match": {
      "field1": "value1"
    }
  }
}

Problemes de particions

  • Particions no assignades: Les particions poden romandre no assignades per diverses raons.
    • Solució: Utilitzeu l'API _cluster/allocation/explain per obtenir informació detallada i prendre mesures correctives. 
      GET /_cluster/allocation/explain

Resolució de problemes avançada

Monitoratge i registre

  • Monitoratge: Utilitzeu les funcions de monitoratge integrades d'Elasticsearch o eines externes com Kibana, Prometheus o Grafana.
    • Exemple: Activeu el monitoratge a elasticsearch.yml. 
      xpack.monitoring.enabled: true
  • Registre: Configureu els nivells de registre i reviseu els registres per trobar errors i advertències.
    • Exemple: Establiu el nivell de registre a log4j2.properties. 
      logger.action.level = debug

Problemes de memòria heap

  • OutOfMemoryError: Elasticsearch pot quedar-se sense memòria heap.
    • Solució: Augmenteu la mida de la heap a jvm.options i assegureu-vos que estigui establerta al 50% de la RAM disponible. 
      -Xms4g
-Xmx4g

Problemes de xarxa

  • Particions de xarxa: Els problemes de xarxa poden causar que els nodes perdin connectivitat.
    • Solució: Assegureu connexions de xarxa estables i configureu els paràmetres discovery.zen.fd per gestionar les particions de xarxa. 
      discovery.zen.fd.ping_timeout: 30s
discovery.zen.fd.ping_retries: 3

Conclusió

La resolució de problemes d'Elasticsearch requereix un enfocament sistemàtic per identificar i resoldre problemes. Comprenent els problemes comuns i les seves solucions, podeu mantenir un clúster d'Elasticsearch saludable i amb bon rendiment. El monitoratge regular, una configuració adequada i una intervenció oportuna són claus per prevenir i resoldre problemes de manera efectiva.
