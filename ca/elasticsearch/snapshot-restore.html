Introducció

Instantània i Restauració és una característica crucial a Elasticsearch que et permet fer còpies de seguretat de les dades del teu clúster i restaurar-les quan sigui necessari. Això és essencial per a la recuperació de dades, la migració i per assegurar la integritat de les dades al llarg del temps.

Conceptes Clau

  • Instantània: Una còpia de seguretat de les dades emmagatzemades en un clúster d'Elasticsearch en un moment específic.
  • Repositori: Una ubicació d'emmagatzematge on es guarden les instantànies. Això pot ser un sistema de fitxers compartit, Amazon S3, HDFS, etc.
  • Restauració: El procés de recuperar dades d'una instantània.

Configuració d'un Repositori

Abans de fer instantànies, cal configurar un repositori on es guardaran les instantànies.

Exemple: Creació d'un Repositori de Sistema de Fitxers

PUT /_snapshot/my_backup
{
  "type": "fs",
  "settings": {
    "location": "/mount/backups/my_backup"
  }
}
  • type: Especifica el tipus de repositori. Aquí, fs significa sistema de fitxers.
  • location: El camí on es guardaran les instantànies.

Fer una Instantània

Un cop el repositori està configurat, pots fer una instantània del teu clúster.

Exemple: Fer una Instantània

PUT /_snapshot/my_backup/snapshot_1
{
  "indices": "index_1,index_2",
  "ignore_unavailable": true,
  "include_global_state": false
}
  • indices: Especifica quins índexs incloure a la instantània.
  • ignore_unavailable: Si és cert, el procés d'instantània ignorarà els índexs que falten o estan tancats.
  • include_global_state: Si és fals, l'estat global del clúster (per exemple, plantilles, configuracions persistents) no s'inclou a la instantània.

Restaurar des d'una Instantània

Per restaurar dades d'una instantània, cal especificar la instantània i els índexs que vols restaurar.

Exemple: Restaurar una Instantània

POST /_snapshot/my_backup/snapshot_1/_restore
{
  "indices": "index_1",
  "ignore_unavailable": true,
  "include_global_state": false,
  "rename_pattern": "index_(.+)",
  "rename_replacement": "restored_index_$1"
}
  • indices: Especifica quins índexs restaurar.
  • ignore_unavailable: Si és cert, el procés de restauració ignorarà els índexs que falten a la instantània.
  • include_global_state: Si és fals, l'estat global del clúster no es restaura.
  • rename_pattern: Un patró regex per coincidir amb els noms dels índexs.
  • rename_replacement: Una cadena de reemplaçament per canviar el nom dels índexs durant la restauració.

Monitorització d'Instantànies i Restauracions

Pots monitoritzar l'estat de les teves instantànies i restauracions per assegurar-te que es completen amb èxit.

Exemple: Comprovar l'Estat de la Instantània

GET /_snapshot/my_backup/snapshot_1

Exemple: Comprovar l'Estat de la Restauració

GET /_cat/recovery?v

Millors Pràctiques

  • Còpies de Seguretat Regulars: Programa instantànies regulars per assegurar-te que sempre tens una còpia de seguretat recent.
  • Prova de Restauracions: Prova periòdicament la restauració des de les instantànies per assegurar-te que el teu procés de còpia de seguretat funciona correctament.
  • Repositoris Segurs: Assegura't que els teus repositoris d'instantànies són segurs i tenen els controls d'accés adequats.

Conclusió

Instantània i Restauració a Elasticsearch és una característica potent que t'ajuda a gestionar la seguretat i la integritat de les teves dades. Configurant repositoris, fent instantànies regulars i provant restauracions, pots assegurar-te que les teves dades estan sempre protegides i recuperables.
