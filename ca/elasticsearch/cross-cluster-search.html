La Cerca entre Clústers (CCS) és una característica potent d'Elasticsearch que et permet cercar a través de múltiples clústers com si fossin un sol clúster. Això és particularment útil per a organitzacions que tenen dades distribuïdes a diferents ubicacions geogràfiques o necessiten gestionar dades a través de múltiples clústers per a escalabilitat i redundància.

Conceptes Clau

  • Clúster: Una col·lecció d'un o més nodes (servidors) que junts contenen totes les teves dades i proporcionen capacitats d'indexació i cerca federades.
  • Cerca entre Clústers: La capacitat de realitzar consultes de cerca a través de múltiples clústers d'Elasticsearch.
  • Clúster Remot: Un clúster que està configurat per ser cercable des d'un altre clúster.

Configuració de la Cerca entre Clústers

Configuració de Clústers Remots

Per habilitar la Cerca entre Clústers, primer has de configurar els clústers remots. Això implica configurar els clústers per reconèixer-se entre ells.

  1. Afegir Configuracions de Clúster Remot: Pots afegir configuracions de clúster remot al fitxer de configuració elasticsearch.yml o mitjançant l'API de configuració del clúster.

    cluster:
  remote:
    cluster_one:
      seeds: ["host1:9300", "host2:9300"]
    cluster_two:
      seeds: ["host3:9300", "host4:9300"]

  2. Utilitzant l'API de Configuració del Clúster: Pots afegir dinàmicament clústers remots utilitzant la següent crida API:

    PUT /_cluster/settings
{
  "persistent": {
    "cluster.remote.cluster_one.seeds": ["host1:9300", "host2:9300"],
    "cluster.remote.cluster_two.seeds": ["host3:9300", "host4:9300"]
  }
}

Realitzant Consultes a través de Clústers

Un cop els clústers remots estan configurats, pots realitzar cerques a través d'ells utilitzant la següent sintaxi:

GET /cluster_one:index_name,cluster_two:index_name/_search
{
  "query": {
    "match_all": {}
  }
}

En aquest exemple, cluster_one:index_name i cluster_two:index_name especifiquen els índexs en els respectius clústers.

Exemple: Consulta de Cerca entre Clústers

Vegem un exemple pràctic on cerquem documents que continguin el terme "Elasticsearch" a través de dos clústers.

GET /cluster_one:logs-2023.01,cluster_two:logs-2023.01/_search
{
  "query": {
    "match": {
      "message": "Elasticsearch"
    }
  }
}

Explicació

  • GET /cluster_one:logs-2023.01,cluster_two:logs-2023.01/_search: Aquesta línia especifica que la cerca s'ha de realitzar a l'índex logs-2023.01 tant en cluster_one com en cluster_two.
  • "query": { "match": { "message": "Elasticsearch" } }: Aquesta part de la consulta especifica que estem buscant documents on el camp message contingui el terme "Elasticsearch".

Consideracions de Rendiment

Quan realitzis Cerques entre Clústers, és important considerar el següent:

  • Latència de Xarxa: Cercar a través de clústers situats en diferents regions geogràfiques pot introduir latència de xarxa.
  • Utilització de Recursos: Les Cerques entre Clústers poden ser intensives en recursos. Assegura't que els teus clústers estan adequadament provisionats per manejar la càrrega.
  • Seguretat: Assegura't que la comunicació segura (per exemple, TLS) està configurada entre clústers per protegir les dades en trànsit.

Conclusió

La Cerca entre Clústers en Elasticsearch és una característica potent que et permet cercar a través de múltiples clústers sense problemes. Configurant clústers remots i utilitzant la sintaxi de consulta adequada, pots aprofitar aquesta capacitat per gestionar i cercar dades distribuïdes de manera efectiva. Considera sempre les implicacions de rendiment i seguretat quan configuris la Cerca entre Clústers per assegurar operacions òptimes i segures.
