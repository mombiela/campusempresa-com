Introducció a la Cerca en Elasticsearch

Elasticsearch és un motor de cerca potent que et permet emmagatzemar, cercar i analitzar grans volums de dades ràpidament i en temps gairebé real. Aquesta secció t'introduirà als conceptes bàsics de la cerca de dades en Elasticsearch.

  • Cerca de text complet: Elasticsearch destaca en la cerca de text complet, que significa cercar text dins dels documents.
  • Cerca estructurada: Elasticsearch també suporta la cerca estructurada, que implica cercar camps específics dins dels documents.
  • Filtratge: El filtratge et permet restringir els resultats de la cerca basant-te en criteris específics.

Consultes de Cerca Bàsiques

Elasticsearch utilitza un llenguatge de consulta anomenat Query DSL (Domain Specific Language) per cercar dades. Comencem amb algunes consultes de cerca bàsiques.

Consulta Match

La consulta match s'utilitza per cercar un terme específic en un camp.

GET /my_index/_search
{
  "query": {
    "match": {
      "message": "search term"
    }
  }
}
  • Explicació: Aquesta consulta cerca documents en l'índex my_index on el camp message conté el terme "search term".

Consulta Term

La consulta term s'utilitza per a coincidències exactes.

GET /my_index/_search
{
  "query": {
    "term": {
      "status": "active"
    }
  }
}
  • Explicació: Aquesta consulta cerca documents on el camp status coincideix exactament amb "active".

Consultes de Cerca Avançades

A mesura que et familiaritzis amb les consultes bàsiques, pots començar a explorar capacitats de cerca més avançades.

Consulta Bool

La consulta bool et permet combinar múltiples consultes utilitzant lògica booleana.

GET /my_index/_search
{
  "query": {
    "bool": {
      "must": [
        { "match": { "message": "search term" } },
        { "term": { "status": "active" } }
      ],
      "filter": [
        { "range": { "date": { "gte": "2023-01-01" } } }
      ]
    }
  }
}
  • Explicació: Aquesta consulta cerca documents que han de coincidir tant amb el camp message que conté "search term" com amb el camp status que coincideix exactament amb "active". A més, filtra els resultats per incloure només documents on el camp date és més gran o igual a "2023-01-01".

Agregacions

Les agregacions et permeten realitzar càlculs complexos i anàlisis de dades.

GET /my_index/_search
{
  "aggs": {
    "status_count": {
      "terms": {
        "field": "status"
      }
    }
  }
}
  • Explicació: Aquesta consulta realitza una agregació per comptar el nombre de documents per a cada valor únic en el camp status.

Filtratge vs. Consulta

Entendre la diferència entre filtratge i consulta és crucial per optimitzar les teves cerques.

| Característica | Filtratge | Consulta | |----------------|-----------|----------| | Ús | Restringir resultats basant-se en criteris | Cercar documents rellevants basant-se en puntuació de rellevància | | Rendiment | Més ràpid, ja que no implica puntuació | Més lent, ja que implica puntuació | | Exemple | filter en consulta bool | must en consulta bool |

Paginació

Elasticsearch suporta la paginació per gestionar grans conjunts de resultats de cerca.

From i Size

Els paràmetres from i size controlen la paginació.

GET /my_index/_search
{
  "from": 10,
  "size": 10,
  "query": {
    "match_all": {}
  }
}
  • Explicació: Aquesta consulta recupera la segona pàgina de resultats, assumint que cada pàgina conté 10 resultats.

Ordenació

L'ordenació et permet ordenar els resultats de la cerca basant-te en camps específics.

GET /my_index/_search
{
  "sort": [
    { "date": { "order": "desc" } }
  ],
  "query": {
    "match_all": {}
  }
}
  • Explicació: Aquesta consulta ordena els resultats de la cerca pel camp date en ordre descendent.

Conclusió

En aquest tema, hem cobert els conceptes bàsics de la cerca de dades en Elasticsearch, incloent consultes de cerca bàsiques i avançades, filtratge, paginació i ordenació. Entenent aquests conceptes, pots cercar i analitzar dades emmagatzemades en Elasticsearch de manera efectiva. A mesura que progressis, podràs explorar consultes i agregacions més complexes per aprofitar tot el potencial d'Elasticsearch.
