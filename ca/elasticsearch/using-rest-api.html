Introducció

Elasticsearch és un potent motor de cerca i analítica que proporciona una API RESTful per interactuar amb les teves dades. Aquesta secció cobrirà els conceptes bàsics d'ús de l'API REST d'Elasticsearch, incloent com realitzar operacions CRUD, consultes de cerca i funcionalitats més avançades.

Conceptes Clau

  • API REST: Un conjunt de serveis web que et permeten interactuar amb Elasticsearch utilitzant mètodes HTTP.
  • Operacions CRUD: Operacions de Crear, Llegir, Actualitzar i Eliminar documents.
  • Índex: Una col·lecció de documents a Elasticsearch.
  • Document: Un registre únic en un índex d'Elasticsearch.
  • Endpoint: Un camí URL específic que correspon a una operació a l'API d'Elasticsearch.

Configuració

Abans de poder utilitzar l'API REST d'Elasticsearch, necessites tenir Elasticsearch instal·lat i en funcionament. Pots interactuar amb l'API utilitzant eines com curl, Postman, o qualsevol client HTTP en el teu llenguatge de programació preferit.

Operacions CRUD Bàsiques

Crear un Índex

Per crear un índex, utilitzes el mètode PUT. Aquí tens un exemple de creació d'un índex anomenat my_index:

PUT /my_index

Afegir un Document

Per afegir un document a un índex, utilitzes el mètode POST. Aquí tens un exemple d'afegir un document a my_index:

POST /my_index/_doc/1
{
  "name": "John Doe",
  "age": 30,
  "occupation": "Software Engineer"
}

Recuperar un Document

Per recuperar un document, utilitzes el mètode GET. Aquí tens un exemple de recuperació del document amb ID 1 de my_index:

GET /my_index/_doc/1

Actualitzar un Document

Per actualitzar un document, utilitzes el mètode POST amb l'endpoint _update. Aquí tens un exemple d'actualització del document amb ID 1:

POST /my_index/_doc/1/_update
{
  "doc": {
    "age": 31
  }
}

Eliminar un Document

Per eliminar un document, utilitzes el mètode DELETE. Aquí tens un exemple d'eliminació del document amb ID 1:

DELETE /my_index/_doc/1

Cercar Documents

Cerca Bàsica

Per realitzar una cerca bàsica, utilitzes l'endpoint _search amb el mètode GET. Aquí tens un exemple de cerca de documents a my_index on el camp name és John Doe:

GET /my_index/_search
{
  "query": {
    "match": {
      "name": "John Doe"
    }
  }
}

Cerca Avançada

Elasticsearch suporta consultes de cerca més avançades, com ara consultes booleanes, consultes de rang i agregacions. Aquí tens un exemple d'una consulta booleana:

GET /my_index/_search
{
  "query": {
    "bool": {
      "must": [
        { "match": { "occupation": "Software Engineer" } }
      ],
      "filter": [
        { "range": { "age": { "gte": 30 } } }
      ]
    }
  }
}

Gestió d'Índexs

Comprovar l'Estat de l'Índex

Per comprovar l'estat d'un índex, utilitzes l'endpoint _stats. Aquí tens un exemple:

GET /my_index/_stats

Eliminar un Índex

Per eliminar un índex, utilitzes el mètode DELETE. Aquí tens un exemple:

DELETE /my_index

Conclusió

L'API REST d'Elasticsearch proporciona una manera potent i flexible d'interactuar amb les teves dades. Entenent les operacions CRUD bàsiques, les funcionalitats de cerca i la gestió d'índexs, pots utilitzar Elasticsearch de manera efectiva per emmagatzemar, recuperar i analitzar les teves dades. A mesura que et familiaritzis més amb l'API, podràs explorar funcions més avançades i optimitzacions per millorar encara més la teva experiència amb Elasticsearch.
