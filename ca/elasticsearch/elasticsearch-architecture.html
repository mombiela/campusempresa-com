Introducció

Elasticsearch és un motor de cerca i analítica distribuït i RESTful capaç de resoldre un nombre creixent de casos d'ús. Entendre la seva arquitectura és crucial per desplegar i gestionar eficaçment els clústers d'Elasticsearch. Aquesta secció cobrirà els components i conceptes fonamentals de l'arquitectura d'Elasticsearch.

Components Clau

Clúster

  • Definició: Un clúster és una col·lecció d'un o més nodes (servidors) que junts contenen totes les teves dades i proporcionen capacitats d'indexació i cerca federades.
  • Nom del Clúster: Cada clúster s'identifica per un nom únic, que per defecte és "elasticsearch". Els nodes només poden unir-se a clústers amb el mateix nom.

Node

  • Definició: Un node és un servidor únic que forma part del clúster, emmagatzema dades i participa en les capacitats d'indexació i cerca del clúster.
  • Rols del Node:
    • Node Mestre: Responsable de la configuració i operacions a nivell de clúster.
    • Node de Dades: Emmagatzema dades i executa operacions relacionades amb les dades com CRUD, cerca i agregacions.
    • Node d'Ingestió: Preprocessa documents abans de la indexació.
    • Node Coordinador: Enruta sol·licituds, gestiona la fase de reducció de cerca i distribueix la indexació massiva.

Índex

  • Definició: Un índex és una col·lecció de documents que tenen característiques similars.
  • Estructura: Un índex s'identifica per un nom i conté múltiples tipus (desaconsellat en versions més noves), que al seu torn contenen múltiples documents.

Document

  • Definició: Un document és una unitat bàsica d'informació que es pot indexar. S'expressa en format JSON.
  • Exemple: 
    {
  "user": "john_doe",
  "post_date": "2023-10-01",
  "message": "Elasticsearch is awesome!"
}

Fragment

  • Definició: Un índex es pot dividir en múltiples peces anomenades fragments. Cada fragment és un "índex" completament funcional i independent que es pot allotjar en qualsevol node del clúster.
  • Tipus:
    • Fragment Primari: Fragment original on es indexen les dades.
    • Fragment de Rèplica: Còpia del fragment primari, utilitzat per a la recuperació d'errors i per augmentar el rendiment de cerca.

Rèplica

  • Definició: Una rèplica és una còpia d'un fragment, proporcionant redundància i augmentant el rendiment de cerca.
  • Configuració de Rèpliques: Configurable a nivell d'índex, permetent especificar el nombre de rèpliques per a cada fragment primari.

Flux de Dades a Elasticsearch

Procés d'Indexació

  1. Enviament de Document: Es presenta un document a un índex.
  2. Enrutament: El document es dirigeix a un fragment primari.
  3. Ingestió: Si s'utilitza un node d'ingestió, el document es preprocessa.
  4. Emmagatzematge: El document s'emmagatzema en el fragment primari i es replica als fragments de rèplica.

Procés de Cerca

  1. Enrutament de Sol·licitud: S'envia una sol·licitud de cerca a un node coordinador.
  2. Distribució de Consulta: El node coordinador envia la consulta als fragments rellevants.
  3. Execució: Cada fragment executa la consulta i retorna els resultats al node coordinador.
  4. Agregació: El node coordinador agrega els resultats i retorna la resposta final.

Exemple: Crear un Índex i Afegir un Document

Crear un Índex

PUT /my_index
{
  "settings": {
    "number_of_shards": 3,
    "number_of_replicas": 2
  }
}
  • Explicació: Aquesta comanda crea un índex anomenat my_index amb 3 fragments primaris i 2 rèpliques per a cada fragment.

Afegir un Document

POST /my_index/_doc/1
{
  "user": "john_doe",
  "post_date": "2023-10-01",
  "message": "Elasticsearch is awesome!"
}
  • Explicació: Aquesta comanda afegeix un document amb ID 1 a l'índex my_index.

Conclusió

Entendre l'arquitectura d'Elasticsearch és fonamental per gestionar i escalar eficientment les teves solucions de cerca i analítica. Components clau com clústers, nodes, índexs, documents, fragments i rèpliques formen l'esquelet d'Elasticsearch. Comprenent aquests conceptes, pots dissenyar, desplegar i solucionar millor els problemes dels clústers d'Elasticsearch per satisfer les teves necessitats específiques.
