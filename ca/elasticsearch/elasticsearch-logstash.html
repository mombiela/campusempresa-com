Introducció a Logstash

Logstash és una eina potent de canalització de processament de dades que pot ingerir dades de diverses fonts, transformar-les i enviar-les a la destinació desitjada, com Elasticsearch. Aquesta secció us introduirà als conceptes bàsics de Logstash i com s'integra amb Elasticsearch.

  • Què és Logstash?

    • Una canalització de processament de dades del costat del servidor.
    • Ingereix dades de múltiples fonts simultàniament.
    • Transforma dades utilitzant connectors de filtre.
    • Envia dades a diverses destinacions, incloent Elasticsearch.

  • Per què utilitzar Logstash amb Elasticsearch?

    • Integració perfecta per a la ingesta de dades.
    • Processament i transformació de dades en temps real.
    • Capacitats millorades d'enriquiment de dades.

Configuració de Logstash

Abans d'entrar en l'ús de Logstash amb Elasticsearch, cal configurar Logstash al vostre sistema.

  • Passos d'instal·lació:

    1. Descarregueu Logstash des del lloc web oficial d'Elastic.
    2. Extraieu el fitxer descarregat.
    3. Navegueu al directori de Logstash.
    4. Executeu Logstash utilitzant la comanda:
    bin/logstash -e 'input { stdin { } } output { stdout {} }'

  • Configuració bàsica:

    • Els fitxers de configuració de Logstash estan escrits en un format similar a JSON.
    • Un fitxer de configuració bàsic consta de tres seccions: input, filter i output.

Configuració de Logstash per a Elasticsearch

Per enviar dades de Logstash a Elasticsearch, cal configurar adequadament les seccions input i output.

  • Secció d'entrada:

    • Defineix la font de les dades.
    • Exemple: 
      input {
  file {
    path => "/path/to/your/logfile.log"
    start_position => "beginning"
  }
}

  • Secció de filtre:

    • S'utilitza per transformar i enriquir les dades.
    • Exemple: 
      filter {
  grok {
    match => { "message" => "%{COMBINEDAPACHELOG}" }
  }
  date {
    match => [ "timestamp" , "dd/MMM/yyyy:HH:mm:ss Z" ]
  }
}

  • Secció de sortida:

    • Defineix la destinació per a les dades processades.
    • Exemple: 
      output {
  elasticsearch {
    hosts => ["localhost:9200"]
    index => "logstash-%{+YYYY.MM.dd}"
  }
}

Exemple pràctic: Ingesta de registres d'Apache

Vegem un exemple pràctic d'ingesta de registres d'Apache a Elasticsearch utilitzant Logstash.

  1. Creeu un fitxer de configuració de Logstash:

    input {
  file {
    path => "/var/log/apache2/access.log"
    start_position => "beginning"
  }
}

filter {
  grok {
    match => { "message" => "%{COMBINEDAPACHELOG}" }
  }
  date {
    match => [ "timestamp" , "dd/MMM/yyyy:HH:mm:ss Z" ]
  }
}

output {
  elasticsearch {
    hosts => ["localhost:9200"]
    index => "apache-logs-%{+YYYY.MM.dd}"
  }
}

  2. Executeu Logstash amb el fitxer de configuració:

    bin/logstash -f /path/to/logstash.conf

  3. Verifiqueu les dades a Elasticsearch:

    • Utilitzeu Kibana o l'API REST d'Elasticsearch per verificar que les dades s'han ingerit correctament.
    • Exemple de trucada a l'API REST: 
      curl -X GET "localhost:9200/apache-logs-*/_search?pretty"

Funcions avançades de Logstash

A mesura que us familiaritzeu més amb Logstash, podeu explorar les seves funcions avançades per millorar la vostra canalització de processament de dades.

  • Processament condicional:

    • Apliqueu filtres i sortides de manera condicional.
    • Exemple: 
      filter {
  if [status] == 404 {
    mutate { add_tag => "not_found" }
  }
}

  • Múltiples canalitzacions:

    • Executeu múltiples canalitzacions en una sola instància de Logstash.
    • Exemple de configuració: 
      - pipeline.id: apache_logs
  path.config: "/etc/logstash/conf.d/apache.conf"

- pipeline.id: syslogs
  path.config: "/etc/logstash/conf.d/syslog.conf"

Conclusió

Utilitzar Logstash amb Elasticsearch proporciona una solució robusta per ingerir, transformar i emmagatzemar dades. Entenent els conceptes bàsics de la configuració de Logstash i explorant les seves funcions avançades, podeu construir canalitzacions de dades eficients que s'adaptin a les vostres necessitats específiques. Tant si tracteu amb fitxers de registre, mètriques o altres tipus de dades, Logstash i Elasticsearch junts ofereixen una combinació poderosa per al processament i anàlisi de dades en temps real.
