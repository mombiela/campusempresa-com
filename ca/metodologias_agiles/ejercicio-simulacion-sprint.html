Introducció

En aquesta secció del curs sobre Metodologies Àgils, realitzarem un exercici pràctic per simular un sprint. Aquest exercici està dissenyat per proporcionar una experiència pràctica en la planificació, execució i revisió d'un sprint, seguint les millors pràctiques de Scrum.

Objectius de l'Exercici

  • Comprendre el cicle complet d'un sprint.
  • Practicar la planificació i estimació de tasques.
  • Experimentar amb l'execució i seguiment del progrés.
  • Realitzar una revisió i retrospectiva de l'sprint.

Materials Necessaris

  • Eina de gestió de projectes (JIRA, Trello, etc.)
  • Plantilles de planificació d'sprint
  • Històries d'usuari predefinides
  • Equip de treball (pot ser simulat)

Planificació de l'Sprint

Pas 1: Definició de l'Sprint

  • Durada de l'Sprint: 2 setmanes
  • Objectiu de l'Sprint: Implementar i provar una nova funcionalitat en l'aplicació web.

Pas 2: Selecció d'Històries d'Usuari

  • Història d'Usuari 1: Com a usuari, vull poder registrar-me a l'aplicació per accedir a funcionalitats exclusives.
  • Història d'Usuari 2: Com a usuari registrat, vull poder iniciar sessió per accedir al meu compte.
  • Història d'Usuari 3: Com a usuari, vull poder restablir la meva contrasenya en cas d'oblidar-la.

Pas 3: Estimació de Tasques

Utilitzarem la tècnica de Planning Poker per estimar les tasques. Cada membre de l'equip assignarà punts d'història a cada tasca basada en la seva complexitat.

| Història d'Usuari | Tasca | Estimació (Punts d'Història) | |---------------------|-------|---------------------------------| | Registre d'Usuari | Crear formulari de registre | 3 | | | Validar dades d'entrada | 2 | | | Guardar dades a la base de dades | 3 | | Inici de Sessió | Crear formulari d'inici de sessió | 2 | | | Validar credencials | 3 | | | Gestionar sessions d'usuari | 3 | | Restabliment de Contrasenya | Crear formulari de restabliment | 2 | | | Enviar correu de restabliment | 3 | | | Actualitzar contrasenya a la base de dades | 2 |

Execució de l'Sprint

Pas 4: Desenvolupament

Cada membre de l'equip prendrà una tasca del backlog i començarà a treballar-hi. Utilitzarem una eina de gestió de projectes per seguir el progrés.

Exemple de Codi: Validació de Dades de Registre

def validate_registration_data(username, email, password):
    if not username or not email or not password:
        return False, "All fields are required."
    if len(password) < 8:
        return False, "Password must be at least 8 characters long."
    if "@" not in email:
        return False, "Invalid email address."
    return True, "Validation successful."

Pas 5: Daily Stand-up

Cada dia, l'equip es reunirà breument per discutir:

  • El que van fer ahir.
  • El que planegen fer avui.
  • Qualsevol impediment que estiguin enfrontant.

Revisió de l'Sprint

Pas 6: Sprint Review

Al final de l'sprint, l'equip presentarà la feina completada als stakeholders. Es demostrarà la nova funcionalitat implementada i es recollirà feedback.

Pas 7: Sprint Retrospective

L'equip es reunirà per discutir:

  • El que va sortir bé durant l'sprint.
  • El que no va sortir bé.
  • Què es pot millorar per al proper sprint.

Conclusió

Aquest exercici pràctic de simulació d'un sprint proporciona una experiència completa del cicle de vida d'un sprint en Scrum. En seguir aquests passos, els professionals de TI poden millorar la seva comprensió i habilitats en la gestió de projectes àgils, la qual cosa els permetrà aplicar aquests coneixements en els seus entorns de treball reals.
