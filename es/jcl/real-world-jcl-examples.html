Introducción

En esta sección, exploraremos ejemplos del mundo real de Job Control Language (JCL) para ayudarte a entender cómo se usa JCL en escenarios prácticos. Cubriremos una gama de ejemplos desde niveles básicos hasta avanzados, proporcionando explicaciones claras y fragmentos de código para ilustrar cada concepto.

Ejemplos Básicos

Ejemplo 1: Envío de Trabajo Simple

Este ejemplo demuestra cómo enviar un trabajo simple que ejecuta un programa.

//SIMPLEJOB JOB (ACCT),'SIMPLE JOB',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=IEFBR14
  • Explicación:
    • //SIMPLEJOB JOB (ACCT),'SIMPLE JOB',CLASS=A,MSGCLASS=A: Esta línea define la tarjeta de trabajo con el nombre del trabajo SIMPLEJOB, información contable (ACCT), una descripción 'SIMPLE JOB', y especifica la clase de trabajo y la clase de mensaje.
    • //STEP1 EXEC PGM=IEFBR14: Esta línea define un paso llamado STEP1 que ejecuta el programa IEFBR14, un programa utilitario que no hace nada pero que a menudo se usa para pruebas.

Ejemplo 2: Copiar un Conjunto de Datos

Este ejemplo muestra cómo copiar un conjunto de datos usando la utilidad IEBGENER.

//COPYJOB  JOB (ACCT),'COPY DATASET',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=IEBGENER
//SYSPRINT DD SYSOUT=A
//SYSUT1   DD DSN=SOURCE.DATASET,DISP=SHR
//SYSUT2   DD DSN=TARGET.DATASET,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(TRK,(1,1)),DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)
//SYSIN    DD DUMMY
  • Explicación:
    • //COPYJOB JOB (ACCT),'COPY DATASET',CLASS=A,MSGCLASS=A: Define la tarjeta de trabajo.
    • //STEP1 EXEC PGM=IEBGENER: Ejecuta la utilidad IEBGENER.
    • //SYSPRINT DD SYSOUT=A: Dirige la salida de la utilidad a la salida del sistema.
    • //SYSUT1 DD DSN=SOURCE.DATASET,DISP=SHR: Especifica el conjunto de datos fuente.
    • //SYSUT2 DD DSN=TARGET.DATASET,DISP=(NEW,CATLG,DELETE),...: Especifica el conjunto de datos de destino con sus atributos.
    • //SYSIN DD DUMMY: Indica que no se necesitan declaraciones de control para IEBGENER.

Ejemplos Intermedios

Ejemplo 3: Pasos Condicionales del Trabajo

Este ejemplo demuestra cómo ejecutar pasos condicionalmente basados en el código de retorno de un paso anterior.

//CONDJOB  JOB (ACCT),'CONDITIONAL STEPS',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=PROG1
//STEP2    EXEC PGM=PROG2,COND=(4,LT,STEP1)
//STEP3    EXEC PGM=PROG3,COND=(0,EQ,STEP2)
  • Explicación:
    • //CONDJOB JOB (ACCT),'CONDITIONAL STEPS',CLASS=A,MSGCLASS=A: Define la tarjeta de trabajo.
    • //STEP1 EXEC PGM=PROG1: Ejecuta PROG1.
    • //STEP2 EXEC PGM=PROG2,COND=(4,LT,STEP1): Ejecuta PROG2 solo si el código de retorno de STEP1 es menor que 4.
    • //STEP3 EXEC PGM=PROG3,COND=(0,EQ,STEP2): Ejecuta PROG3 solo si el código de retorno de STEP2 es 0.

Ejemplo 4: Uso de Parámetros Simbólicos

Este ejemplo muestra cómo usar parámetros simbólicos para hacer que JCL sea más flexible.

//SYMBOLIC JOB (ACCT),'SYMBOLIC PARAMS',CLASS=A,MSGCLASS=A
//SET VAR1='MYDATA'
//STEP1    EXEC PGM=IEFBR14
//DD1      DD DSN=&VAR1..DATASET,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(TRK,(1,1)),DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)
  • Explicación:
    • //SYMBOLIC JOB (ACCT),'SYMBOLIC PARAMS',CLASS=A,MSGCLASS=A: Define la tarjeta de trabajo.
    • //SET VAR1='MYDATA': Define un parámetro simbólico VAR1.
    • //STEP1 EXEC PGM=IEFBR14: Ejecuta el programa IEFBR14.
    • //DD1 DD DSN=&VAR1..DATASET,...: Usa el parámetro simbólico VAR1 para definir el nombre del conjunto de datos.

Ejemplos Avanzados

Ejemplo 5: Uso de Procedimientos

Este ejemplo demuestra cómo usar procedimientos para simplificar JCL.

//PROCJOB  JOB (ACCT),'USING PROCEDURES',CLASS=A,MSGCLASS=A
//         EXEC MYPROC
  • Explicación:
    • //PROCJOB JOB (ACCT),'USING PROCEDURES',CLASS=A,MSGCLASS=A: Define la tarjeta de trabajo.
    • // EXEC MYPROC: Ejecuta un procedimiento predefinido MYPROC.

Ejemplo 6: Reiniciar un Trabajo

Este ejemplo muestra cómo reiniciar un trabajo desde un paso específico.

//RESTART JOB (ACCT),'RESTART JOB',CLASS=A,MSGCLASS=A,RESTART=STEP2
//STEP1    EXEC PGM=PROG1
//STEP2    EXEC PGM=PROG2
//STEP3    EXEC PGM=PROG3
  • Explicación:
    • //RESTART JOB (ACCT),'RESTART JOB',CLASS=A,MSGCLASS=A,RESTART=STEP2: Define la tarjeta de trabajo con un parámetro de reinicio para comenzar desde STEP2.
    • //STEP1 EXEC PGM=PROG1: Ejecuta PROG1.
    • //STEP2 EXEC PGM=PROG2: Ejecuta PROG2.
    • //STEP3 EXEC PGM=PROG3: Ejecuta PROG3.

Conclusión

En esta sección, hemos cubierto una variedad de ejemplos de JCL del mundo real que van desde niveles básicos hasta avanzados. Estos ejemplos ilustran cómo se puede usar JCL para enviar trabajos, copiar conjuntos de datos, ejecutar pasos condicionales, usar parámetros simbólicos, utilizar procedimientos y reiniciar trabajos. Al entender estos ejemplos, estarás mejor preparado para escribir y gestionar JCL en escenarios prácticos.
