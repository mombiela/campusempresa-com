Introducción

La declaración JOB es la primera declaración en un script JCL (Job Control Language). Proporciona al sistema información esencial sobre el trabajo, como el nombre del trabajo, información contable y varios parámetros a nivel de trabajo. Entender la declaración JOB es crucial para cualquiera que trabaje con JCL, ya que establece el escenario para la ejecución del trabajo.

Conceptos Clave

  • Nombre del Trabajo: Un identificador único para el trabajo.
  • Información Contable: Utilizada para fines de facturación y seguimiento.
  • Parámetros del Trabajo: Controlan varios aspectos de la ejecución del trabajo, como prioridad, clase y notificación.

Estructura Básica de una Declaración JOB

Una declaración JOB básica tiene la siguiente sintaxis:

//JOBNAME JOB (accounting_info), 'programmer_name', CLASS=class, MSGCLASS=msgclass, MSGLEVEL=msglevel, NOTIFY=userid

Ejemplo

//MYJOB01 JOB (12345), 'John Doe', CLASS=A, MSGCLASS=X, MSGLEVEL=(1,1), NOTIFY=&SYSUID

Explicación

  • MYJOB01: El nombre del trabajo, que debe ser único dentro del sistema.
  • (12345): Información contable para fines de facturación.
  • 'John Doe': El nombre del programador o propietario del trabajo.
  • CLASS=A: Especifica la clase del trabajo, que determina la prioridad y asignación de recursos del trabajo.
  • MSGCLASS=X: Especifica la clase de salida para mensajes relacionados con el trabajo.
  • MSGLEVEL=(1,1): Controla el nivel de detalle en los mensajes de salida del trabajo.
  • NOTIFY=&SYSUID: Especifica el usuario a notificar al completar el trabajo.

Parámetros Detallados

CLASS

  • Propósito: Determina la prioridad y asignación de recursos para el trabajo.
  • Valores: Típicamente, letras individuales (A, B, C, etc.) o números (1, 2, 3, etc.).

MSGCLASS

  • Propósito: Especifica la clase de salida para mensajes relacionados con el trabajo.
  • Valores: Letras individuales (A, B, C, etc.) o números (1, 2, 3, etc.).

MSGLEVEL

  • Propósito: Controla el nivel de detalle en los mensajes de salida del trabajo.
  • Valores: Un par de números (x,y) donde:
    • x: Controla la salida de declaraciones JCL.
    • y: Controla la salida de mensajes del sistema.

NOTIFY

  • Propósito: Especifica el usuario a notificar al completar el trabajo.
  • Valores: ID de usuario o &SYSUID para el usuario que envía el trabajo.

Uso Avanzado

Ejecución Condicional

Puedes usar parámetros condicionales para controlar la ejecución de trabajos basados en condiciones específicas.

Ejemplo

//MYJOB02 JOB (12345), 'Jane Doe', CLASS=B, MSGCLASS=Y, MSGLEVEL=(2,0), NOTIFY=&SYSUID, COND=(4,LT)

Explicación

  • COND=(4,LT): El trabajo solo se ejecutará si el código de retorno del paso de trabajo anterior es menor que 4.

Uso de JCLLIB

Puedes incluir una declaración JCLLIB para especificar una biblioteca de procedimientos JCL.

Ejemplo

//MYJOB03 JOB (12345), 'Alice Smith', CLASS=C, MSGCLASS=Z, MSGLEVEL=(1,1), NOTIFY=&SYSUID
//JCLLIB ORDER=(MY.PROCLIB)

Explicación

  • JCLLIB ORDER=(MY.PROCLIB): Especifica la biblioteca MY.PROCLIB para buscar procedimientos.

Conclusión

La declaración JOB es un componente fundamental de JCL que proporciona información esencial sobre el trabajo al sistema. Al entender su estructura y parámetros, puedes controlar efectivamente la ejecución del trabajo, priorizar tareas y gestionar salidas de trabajo. Dominar la declaración JOB es un paso crucial para volverse competente en JCL.
