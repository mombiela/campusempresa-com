Introducción

En el Lenguaje de Control de Trabajos (JCL), gestionar y organizar tus flujos de trabajo de manera eficiente es crucial. Dos declaraciones importantes que ayudan en este sentido son JCLLIB e INCLUDE. Esta sección cubrirá sus propósitos, uso y proporcionará ejemplos para ayudarte a entender cómo implementarlas en tus scripts JCL.

Declaración JCLLIB

La declaración JCLLIB se utiliza para especificar una o más bibliotecas que contienen procedimientos JCL (PROCs) que deseas incluir en tu trabajo. Esto ayuda a modularizar tus scripts JCL y reutilizar procedimientos comunes en múltiples trabajos.

Conceptos Clave

  • Propósito: Especificar las bibliotecas que contienen procedimientos JCL.
  • Sintaxis: //JCLLIB ORDER=(library1,library2,...)
  • Ubicación: Normalmente se coloca al principio del script JCL, antes de cualquier paso del trabajo que use los procedimientos.

Ejemplo

//MYJOB    JOB  (ACCT),'Example Job'
//JCLLIB   ORDER=(MY.PROCLIB, SYS1.PROCLIB)
//STEP1    EXEC  PROC=MYPROC

En este ejemplo:

  • MY.PROCLIB y SYS1.PROCLIB son las bibliotecas especificadas en la declaración JCLLIB.
  • MYPROC es un procedimiento ubicado en una de estas bibliotecas.

Declaración INCLUDE

La declaración INCLUDE te permite incluir un conjunto de declaraciones JCL desde un miembro de un conjunto de datos particionado (PDS). Esto es útil para incluir segmentos de código JCL comunes sin duplicarlos en múltiples trabajos.

Conceptos Clave

  • Propósito: Incluir segmentos de código JCL predefinidos.
  • Sintaxis: //INCLUDE MEMBER=membername
  • Ubicación: Puede colocarse en cualquier lugar del script JCL donde desees incluir el segmento de código JCL.

Ejemplo

//MYJOB    JOB  (ACCT),'Example Job'
//INCLUDE  MEMBER=COMMONJCL
//STEP1    EXEC  PGM=MYPROG

En este ejemplo:

  • COMMONJCL es un miembro de un PDS que contiene declaraciones JCL comunes.
  • La declaración INCLUDE insertará el contenido de COMMONJCL en ese punto del trabajo.

Combinando JCLLIB e INCLUDE

Puedes usar tanto las declaraciones JCLLIB como INCLUDE en el mismo script JCL para gestionar procedimientos y segmentos de código JCL comunes de manera eficiente.

Ejemplo

//MYJOB    JOB  (ACCT),'Example Job'
//JCLLIB   ORDER=(MY.PROCLIB, SYS1.PROCLIB)
//INCLUDE  MEMBER=COMMONJCL
//STEP1    EXEC  PROC=MYPROC

En este ejemplo:

  • La declaración JCLLIB especifica las bibliotecas que contienen procedimientos.
  • La declaración INCLUDE incluye el código JCL común del miembro COMMONJCL.
  • STEP1 ejecuta el procedimiento MYPROC de las bibliotecas especificadas.

Conclusión

Entender y usar las declaraciones JCLLIB e INCLUDE puede mejorar enormemente la modularidad y mantenibilidad de tus scripts JCL. Al especificar bibliotecas para procedimientos e incluir segmentos de código JCL comunes, puedes evitar redundancias y hacer que tus flujos de trabajo sean más organizados y fáciles de gestionar. Practica usando estas declaraciones en tus scripts JCL para volverte competente en la gestión eficiente de flujos de trabajo complejos.
