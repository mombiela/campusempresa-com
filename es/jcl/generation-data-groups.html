Introducción a los GDGs

Los Grupos de Datos de Generación (GDGs) son un método en JCL (Lenguaje de Control de Trabajos) para gestionar conjuntos de datos que están lógicamente relacionados y necesitan ser procesados en una secuencia específica. Los GDGs son particularmente útiles para manejar conjuntos de datos que se crean regularmente, como registros diarios o copias de seguridad.

Conceptos Clave

  • Grupo de Datos de Generación (GDG): Una colección de conjuntos de datos que están relacionados y pueden ser referenciados por un nombre común.
  • Generación: Cada conjunto de datos dentro de un GDG se denomina generación y se identifica por un número de generación.
  • Base: El nombre común utilizado para referenciar todo el GDG.
  • Número de Generación Relativa: Una forma de referenciar conjuntos de datos dentro de un GDG en relación con la generación actual.

Creación y Gestión de GDGs

Definición de una Base GDG

Para crear un GDG, primero necesitas definir una base GDG. Esto se hace usando la utilidad IDCAMS.

//DEFINEGDG EXEC PGM=IDCAMS
//SYSPRINT DD SYSOUT=*
//SYSIN    DD *
  DEFINE GDG (NAME(MY.GDG.BASE) LIMIT(5) SCRATCH)
/*
  • NAME: Especifica el nombre de la base GDG.
  • LIMIT: Especifica el número máximo de generaciones que se pueden mantener.
  • SCRATCH: Indica que las generaciones antiguas deben ser eliminadas cuando se exceda el límite.

Creación de una Nueva Generación

Para crear una nueva generación dentro de un GDG, usas el nombre de la base GDG seguido de (0).

//NEWGEN   EXEC PGM=IEFBR14
//NEWFILE  DD DSN=MY.GDG.BASE(+1),
//            DISP=(NEW,CATLG,DELETE),
//            SPACE=(TRK,(1,1)),
//            DCB=(LRECL=80,BLKSIZE=800,RECFM=FB)
  • MY.GDG.BASE(+1): Crea una nueva generación.
  • DISP: Especifica la disposición del conjunto de datos.
  • SPACE: Asigna espacio para el conjunto de datos.
  • DCB: Define los parámetros del bloque de control del conjunto de datos.

Acceso a Generaciones GDG

Puedes acceder a generaciones específicas usando números de generación relativa.

//ACCESSGEN EXEC PGM=IEFBR14
//OLDGEN   DD DSN=MY.GDG.BASE(-1),DISP=SHR
//CURGEN   DD DSN=MY.GDG.BASE(0),DISP=SHR
  • MY.GDG.BASE(-1): Se refiere a la generación anterior.
  • MY.GDG.BASE(0): Se refiere a la generación actual.

Gestión Avanzada de GDGs

Eliminación de Generaciones

Cuando se alcanza el límite de generaciones, la generación más antigua se elimina automáticamente si se usa la opción SCRATCH. Si se especifica NOSCRATCH, la generación más antigua se descataloga pero no se elimina.

Uso de GDGs en Procedimientos

Los GDGs pueden ser usados en procedimientos JCL para automatizar la creación y procesamiento de conjuntos de datos.

//PROC1    PROC
//STEP1    EXEC PGM=MYPROG
//INPUT    DD DSN=MY.GDG.BASE(0),DISP=SHR
//OUTPUT   DD DSN=MY.GDG.BASE(+1),DISP=(NEW,CATLG,DELETE),
//            SPACE=(TRK,(1,1)),
//            DCB=(LRECL=80,BLKSIZE=800,RECFM=FB)
/*
//PROC1    PEND
  • PROC: Define el inicio de un procedimiento.
  • PEND: Marca el final del procedimiento.

Comparación de Estructuras GDG

| Característica | Descripción | |------------------|-----------------------------------------------------------------------------| | Nombre Base | Nombre común para todas las generaciones. | | Generación | Conjunto de datos individual dentro del GDG. | | Generación Relativa | Número usado para referenciar conjuntos de datos en relación con la generación actual. | | Límite | Número máximo de generaciones mantenidas. | | SCRATCH | Opción para eliminar generaciones antiguas cuando se excede el límite. | | NOSCRATCH | Opción para descatalogar pero no eliminar generaciones antiguas cuando se excede el límite. |

Conclusión

Los Grupos de Datos de Generación (GDGs) en JCL proporcionan un mecanismo poderoso para gestionar conjuntos de datos que se crean y procesan en secuencia. Al entender cómo definir, crear y acceder a los GDGs, puedes gestionar eficazmente conjuntos de datos que necesitan ser manejados en un orden lógico. Las características avanzadas como la eliminación de generaciones y el uso de GDGs en procedimientos mejoran aún más su utilidad en flujos de trabajo JCL complejos.
