Introducción

Definir conjuntos de datos es un aspecto fundamental al trabajar con el Lenguaje de Control de Trabajos (JCL). Los conjuntos de datos son colecciones de datos que pueden ser utilizados para varios propósitos, como entrada, salida o almacenamiento intermedio durante la ejecución del trabajo. Esta sección cubrirá los conceptos básicos de la definición de conjuntos de datos en JCL, incluyendo sintaxis, parámetros y ejemplos.

Conceptos Clave

  • Conjunto de Datos: Una colección de datos, a menudo almacenada en disco o cinta, que puede ser accedida y manipulada por trabajos JCL.
  • Instrucción DD: La instrucción de Definición de Datos (DD) se utiliza en JCL para definir conjuntos de datos.
  • DSN (Nombre del Conjunto de Datos): El nombre asignado a un conjunto de datos.
  • DISP (Disposición): Especifica el estado del conjunto de datos (por ejemplo, NEW, OLD, MOD) y qué hacer con él después de que el paso del trabajo se complete.

Sintaxis de la Instrucción DD

La instrucción DD se utiliza para definir conjuntos de datos en JCL. Aquí está la sintaxis básica:

//DDNAME DD DSN=dataset.name,DISP=(status,normal-termination,abnormal-termination)
  • DDNAME: El nombre utilizado para referenciar el conjunto de datos dentro del JCL.
  • DSN: El nombre del conjunto de datos.
  • DISP: El parámetro de disposición, que incluye el estado y las acciones a tomar en caso de terminación normal y anormal.

Parámetro DISP

El parámetro DISP es crucial para la gestión de conjuntos de datos. Tiene tres sub-parámetros:

  1. Estado: Indica si el conjunto de datos es nuevo, viejo o debe ser modificado.
    • NEW: Crear un nuevo conjunto de datos.
    • OLD: Usar un conjunto de datos existente.
    • MOD: Modificar un conjunto de datos existente o crearlo si no existe.
  2. Terminación Normal: Acción a tomar si el paso del trabajo se completa exitosamente.
    • CATLG: Catalogar el conjunto de datos.
    • KEEP: Mantener el conjunto de datos.
    • DELETE: Eliminar el conjunto de datos.
  3. Terminación Anormal: Acción a tomar si el paso del trabajo falla.
    • CATLG: Catalogar el conjunto de datos.
    • KEEP: Mantener el conjunto de datos.
    • DELETE: Eliminar el conjunto de datos.

Ejemplo de Uso del Parámetro DISP

//MYDATA DD DSN=MY.DATASET,DISP=(NEW,CATLG,DELETE)

En este ejemplo:

  • El conjunto de datos MY.DATASET se crea nuevo.
  • Si el paso del trabajo se completa exitosamente, el conjunto de datos se cataloga.
  • Si el paso del trabajo falla, el conjunto de datos se elimina.

Definiendo Conjuntos de Datos Temporales

Los conjuntos de datos temporales se utilizan para almacenamiento intermedio y se eliminan automáticamente después de que el trabajo se completa. No requieren un nombre de conjunto de datos.

Ejemplo de Conjunto de Datos Temporal

//TEMP DD DSN=&&TEMPDS,DISP=(NEW,PASS,DELETE)

En este ejemplo:

  • &&TEMPDS indica un conjunto de datos temporal.
  • El conjunto de datos se crea nuevo.
  • Si el paso del trabajo se completa exitosamente, el conjunto de datos se pasa al siguiente paso.
  • Si el paso del trabajo falla, el conjunto de datos se elimina.

Definiendo Conjuntos de Datos VSAM

Los conjuntos de datos VSAM (Método de Acceso de Almacenamiento Virtual) se utilizan para acceso de alto rendimiento a los datos. Definir un conjunto de datos VSAM requiere parámetros adicionales.

Ejemplo de Definición de Conjunto de Datos VSAM

//VSAMDD DD DSN=MY.VSAM.DATASET,DISP=(NEW,CATLG,DELETE),
//       SPACE=(CYL,(5,1)),UNIT=SYSDA,
//       DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)

En este ejemplo:

  • MY.VSAM.DATASET es el nombre del conjunto de datos VSAM.
  • El conjunto de datos se crea nuevo, se cataloga en caso de terminación exitosa y se elimina en caso de fallo.
  • SPACE especifica la asignación de espacio primaria y secundaria.
  • UNIT especifica el tipo de dispositivo.
  • DCB (Bloque de Control de Datos) especifica las características del conjunto de datos.

Conclusión

Definir conjuntos de datos en JCL es esencial para gestionar datos durante la ejecución del trabajo. Entender la instrucción DD, el parámetro DISP y cómo definir tanto conjuntos de datos temporales como VSAM te permitirá controlar efectivamente el flujo de datos en tus trabajos JCL. Practica con diferentes definiciones de conjuntos de datos para volverte competente en el manejo de varios escenarios de datos en JCL.
