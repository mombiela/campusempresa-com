Introducción

El Lenguaje de Control de Trabajos (JCL) es un lenguaje de scripting utilizado en los sistemas operativos mainframe de IBM para instruir al sistema sobre cómo ejecutar un trabajo por lotes o iniciar un subsistema. Comprender la estructura básica de una declaración JCL es crucial para cualquiera que desee trabajar con sistemas mainframe.

Conceptos Clave

  • Declaraciones JCL: Instrucciones escritas en JCL para realizar tareas específicas.
  • Trabajo: Una unidad de trabajo que el sistema ejecuta.
  • Pasos: Tareas individuales dentro de un trabajo.
  • Parámetros: Instrucciones que definen cómo debe ejecutarse el trabajo o el paso.

Estructura Básica de una Declaración JCL

Una declaración JCL típicamente consta de tres partes:

  1. Identificador: Especifica el tipo de declaración.
  2. Campo de Nombre: Un campo opcional que nombra la declaración.
  3. Campo de Operación: Especifica la operación a realizar.
  4. Campo de Parámetros: Contiene parámetros que controlan la operación.

Ejemplo de una Declaración JCL

//JOBNAME  JOB  (ACCT),'PROGRAMMER',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID

Desglose del Ejemplo

  • Identificador: // indica el inicio de una declaración JCL.
  • Campo de Nombre: JOBNAME es el nombre del trabajo.
  • Campo de Operación: JOB especifica que esta es una declaración de trabajo.
  • Campo de Parámetros: Contiene parámetros como (ACCT), 'PROGRAMMER', CLASS=A, MSGCLASS=X, y NOTIFY=&SYSUID.

Declaraciones JCL Comunes

Declaración JOB

Define el inicio de un trabajo y proporciona parámetros a nivel de trabajo.

//MYJOB  JOB  (ACCT),'MY JOB',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID

Declaración EXEC

Especifica la ejecución de un programa o procedimiento.

//STEP1  EXEC  PGM=MYPROG

Declaración DD

Define los conjuntos de datos a utilizar en el paso del trabajo.

//INPUT  DD  DSN=MY.DATA.SET,DISP=SHR

Parámetros en Declaraciones JCL

Parámetros de la Declaración JOB

  • ACCT: Información contable.
  • 'MY JOB': Descripción del trabajo.
  • CLASS: Clase de trabajo.
  • MSGCLASS: Clase de salida.
  • NOTIFY: Usuario a notificar al completar el trabajo.

Parámetros de la Declaración EXEC

  • PGM: Nombre del programa a ejecutar.
  • PROC: Nombre del procedimiento si se utiliza un procedimiento catalogado o en línea.

Parámetros de la Declaración DD

  • DSN: Nombre del conjunto de datos.
  • DISP: Disposición del conjunto de datos (por ejemplo, NEW, OLD, SHR).
  • UNIT: Tipo de dispositivo.
  • SPACE: Asignación de espacio.

Ejemplo de Trabajo con Múltiples Pasos

//MYJOB  JOB  (ACCT),'MY JOB',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1  EXEC PGM=MYPROG
//INPUT  DD  DSN=MY.DATA.SET,DISP=SHR
//STEP2  EXEC PGM=ANOTHERPGM
//OUTPUT DD  DSN=ANOTHER.DATA.SET,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(CYL,(5,5),RLSE)

Explicación

  • STEP1 ejecuta MYPROG utilizando MY.DATA.SET como entrada.
  • STEP2 ejecuta ANOTHERPGM y crea ANOTHER.DATA.SET con una asignación de espacio y parámetros de disposición específicos.

Conclusión

Comprender la estructura básica de las declaraciones JCL es esencial para escribir scripts de trabajo efectivos en sistemas mainframe de IBM. Cada declaración tiene un propósito específico, y saber cómo configurar los parámetros correctamente puede impactar significativamente en la ejecución y gestión de los trabajos. Practica escribiendo scripts JCL simples para familiarizarte más con estos conceptos.
