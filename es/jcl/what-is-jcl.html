Introducción a JCL

El Lenguaje de Control de Trabajos (JCL) es un lenguaje de scripting utilizado en los sistemas operativos mainframe de IBM para instruir al sistema sobre cómo ejecutar un trabajo por lotes o iniciar un subsistema. Es esencial para gestionar y ejecutar trabajos en un entorno mainframe.

Conceptos Clave

  • Trabajo: Una unidad de trabajo que el sistema ejecutará.
  • Paso de Trabajo: Una operación única dentro de un trabajo.
  • Procedimiento (PROC): Un conjunto de declaraciones JCL que pueden reutilizarse en múltiples trabajos.
  • Conjunto de Datos: Un archivo o una colección de datos que JCL procesará.
  • DD (Definición de Datos): Declaraciones que definen los conjuntos de datos utilizados en un trabajo.

Estructura Básica de JCL

Un script JCL se compone de varios componentes clave:

  1. Declaración de Trabajo: Define el trabajo y proporciona información a nivel de trabajo.
  2. Declaración Exec: Especifica el programa o procedimiento a ejecutar.
  3. Declaración DD: Describe los conjuntos de datos y recursos necesarios para el trabajo.

Ejemplo de un Script JCL Simple

//MYJOB    JOB  'ACCOUNTING',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1    EXEC PGM=MYPROGRAM
//INPUT    DD   DSN=MY.INPUT.DATASET,DISP=SHR
//OUTPUT   DD   DSN=MY.OUTPUT.DATASET,DISP=(NEW,CATLG,DELETE),
//              SPACE=(CYL,(10,5)),UNIT=SYSDA

Explicación:

  • Declaración de Trabajo (//MYJOB JOB): Define el nombre del trabajo (MYJOB), información contable, clase de trabajo, clase de mensaje y notificación.
  • Declaración Exec (//STEP1 EXEC PGM=MYPROGRAM): Especifica el programa (MYPROGRAM) a ejecutar.
  • Declaraciones DD:
    • //INPUT DD: Define el conjunto de datos de entrada (MY.INPUT.DATASET) con una disposición (DISP) de SHR (compartido).
    • //OUTPUT DD: Define el conjunto de datos de salida (MY.OUTPUT.DATASET) con una disposición de NEW (nuevo conjunto de datos), CATLG (catalogar el conjunto de datos) y DELETE (eliminar si el trabajo falla). También especifica la asignación de espacio y la unidad.

Conceptos Avanzados

A medida que avances en JCL, encontrarás constructos y características más complejas:

Procedimientos (PROCs)

  • Definición: Un PROC es un conjunto de declaraciones JCL que pueden incluirse en múltiples trabajos para evitar redundancias.
  • Ejemplo: 
    //MYPROC PROC
//STEP1  EXEC PGM=MYPROGRAM
//INPUT  DD   DSN=&INDSN,DISP=SHR
//OUTPUT DD   DSN=&OUTDSN,DISP=(NEW,CATLG,DELETE),
//              SPACE=(CYL,(10,5)),UNIT=SYSDA

Procesamiento Condicional

  • IF/THEN/ELSE: Permite la ejecución condicional de pasos de trabajo basándose en el código de retorno de pasos anteriores.
  • Ejemplo: 
    //STEP1  EXEC PGM=MYPROGRAM
//IF (STEP1.RC = 0) THEN
//STEP2  EXEC PGM=ANOTHERPGM
//ELSE
//STEP3  EXEC PGM=ERRORPGM
//ENDIF

Parámetros Simbólicos

  • Definición: Los parámetros simbólicos permiten la parametrización de scripts JCL, haciéndolos más flexibles y reutilizables.
  • Ejemplo: 
    //MYJOB  JOB  'ACCOUNTING',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1  EXEC PGM=MYPROGRAM,PARM='&MYVAR'

Conclusión

JCL es un lenguaje poderoso para gestionar y ejecutar trabajos en mainframes de IBM. Comprender su estructura básica, componentes clave y características avanzadas es crucial para trabajar eficientemente en un entorno mainframe. A medida que progreses de niveles principiantes a avanzados, podrás escribir scripts JCL más complejos y eficientes, aprovechando procedimientos, procesamiento condicional y parámetros simbólicos para optimizar tus flujos de trabajo.
