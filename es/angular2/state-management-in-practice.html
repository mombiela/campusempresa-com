Introducción a la Gestión del Estado

La gestión del estado es un aspecto crítico para construir aplicaciones de Angular escalables y mantenibles. Implica gestionar el estado de tu aplicación de una manera predecible y consistente.

  • Estado: Los datos que tu aplicación necesita para funcionar.
  • Gestión del Estado: Técnicas y bibliotecas utilizadas para gestionar el estado.

Por qué la Gestión del Estado es Importante

  • Predictibilidad: Asegura que el estado de la aplicación sea predecible.
  • Depuración: Facilita la depuración de aplicaciones.
  • Escalabilidad: Ayuda a escalar aplicaciones gestionando el estado de manera eficiente.
  • Consistencia: Asegura que el estado sea consistente en diferentes partes de la aplicación.

Técnicas Comunes de Gestión del Estado

  • Estado Local: Gestionado dentro de componentes individuales.
  • Gestión del Estado Basada en Servicios: Uso de servicios de Angular para gestionar el estado.
  • Bibliotecas de Gestión del Estado: Uso de bibliotecas como NgRx o Akita.

Gestión del Estado Local

El estado local se gestiona dentro de componentes individuales. Esto es adecuado para aplicaciones pequeñas o componentes aislados.

Ejemplo

import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  template: `
    <div>
      <p>Count: {{ count }}</p>
      <button (click)="increment()">Increment</button>
      <button (click)="decrement()">Decrement</button>
    </div>
  `
})
export class CounterComponent {
  count = 0;

  increment() {
    this.count++;
  }

  decrement() {
    this.count--;
  }
}
  • Explicación: El estado count se gestiona dentro del CounterComponent.

Gestión del Estado Basada en Servicios

Usar servicios de Angular para gestionar el estado permite compartir el estado entre múltiples componentes.

Ejemplo

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class CounterService {
  private count = 0;

  getCount() {
    return this.count;
  }

  increment() {
    this.count++;
  }

  decrement() {
    this.count--;
  }
}
import { Component } from '@angular/core';
import { CounterService } from './counter.service';

@Component({
  selector: 'app-counter',
  template: `
    <div>
      <p>Count: {{ counterService.getCount() }}</p>
      <button (click)="counterService.increment()">Increment</button>
      <button (click)="counterService.decrement()">Decrement</button>
    </div>
  `
})
export class CounterComponent {
  constructor(public counterService: CounterService) {}
}
  • Explicación: El CounterService gestiona el estado, y el CounterComponent interactúa con el servicio para actualizar y mostrar el estado.

Bibliotecas de Gestión del Estado

Para aplicaciones más grandes, usar bibliotecas de gestión del estado como NgRx o Akita puede ser beneficioso.

NgRx

NgRx es una popular biblioteca de gestión del estado para aplicaciones de Angular, inspirada en Redux.

Conceptos Clave

  • Store: Contiene el estado de la aplicación.
  • Actions: Describen cambios de estado.
  • Reducers: Manejan cambios de estado basados en acciones.
  • Selectors: Seleccionan partes del estado desde el store.

Ejemplo

Actions

import { createAction } from '@ngrx/store';

export const increment = createAction('[Counter] Increment');
export const decrement = createAction('[Counter] Decrement');

Reducer

import { createReducer, on } from '@ngrx/store';
import { increment, decrement } from './counter.actions';

export const initialState = 0;

const _counterReducer = createReducer(initialState,
  on(increment, state => state + 1),
  on(decrement, state => state - 1)
);

export function counterReducer(state, action) {
  return _counterReducer(state, action);
}

Store Module

import { NgModule } from '@angular/core';
import { StoreModule } from '@ngrx/store';
import { counterReducer } from './counter.reducer';

@NgModule({
  imports: [
    StoreModule.forRoot({ count: counterReducer })
  ]
})
export class AppModule {}

Component

import { Component } from '@angular/core';
import { Store } from '@ngrx/store';
import { increment, decrement } from './counter.actions';

@Component({
  selector: 'app-counter',
  template: `
    <div>
      <p>Count: {{ count$ | async }}</p>
      <button (click)="increment()">Increment</button>
      <button (click)="decrement()">Decrement</button>
    </div>
  `
})
export class CounterComponent {
  count$ = this.store.select('count');

  constructor(private store: Store<{ count: number }>) {}

  increment() {
    this.store.dispatch(increment());
  }

  decrement() {
    this.store.dispatch(decrement());
  }
}
  • Explicación: El estado se gestiona usando NgRx, con acciones, reducers y el módulo store.

Conclusión

La gestión del estado es esencial para construir aplicaciones robustas de Angular. Dependiendo del tamaño y la complejidad de tu aplicación, puedes elegir entre la gestión del estado local, la gestión del estado basada en servicios o el uso de bibliotecas de gestión del estado como NgRx. Cada técnica tiene sus propias ventajas y casos de uso, y entenderlas te ayudará a construir aplicaciones escalables y mantenibles.
