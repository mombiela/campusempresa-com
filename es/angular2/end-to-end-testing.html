Las pruebas de extremo a extremo (E2E) son una parte crucial del ciclo de vida del desarrollo de software, asegurando que todo el flujo de la aplicación funcione como se espera. En Angular 2+, las pruebas E2E se realizan típicamente usando Protractor, un marco de pruebas diseñado específicamente para aplicaciones Angular.

Introducción a las pruebas de extremo a extremo

  • Definición: Las pruebas E2E implican probar el flujo completo de una aplicación de principio a fin.
  • Propósito: Validar que los componentes integrados de una aplicación funcionen como se espera.
  • Herramientas: En Angular, Protractor es la herramienta principal utilizada para las pruebas E2E.

Configuración de Protractor

  • Instalación: Protractor se puede instalar a través de npm. 
    npm install -g protractor
  • Actualización de WebDriver: Asegúrate de que WebDriver esté actualizado. 
    webdriver-manager update
  • Configuración: Protractor requiere un archivo de configuración (protractor.conf.js).

Archivo de configuración de ejemplo

exports.config = {
  framework: 'jasmine',
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['spec.js'],
  capabilities: {
    'browserName': 'chrome'
  }
};
  • Explicación:
    • framework: Especifica el marco de pruebas (Jasmine en este caso).
    • seleniumAddress: La dirección del servidor Selenium en ejecución.
    • specs: Una matriz de archivos de prueba.
    • capabilities: Define el navegador que se utilizará para las pruebas.

Escribiendo pruebas E2E

  • Estructura básica: Las pruebas E2E en Protractor se escriben usando Jasmine.
  • Prueba de ejemplo: 
    describe('Angular App', function() {
  it('should have a title', function() {
    browser.get('http://localhost:4200');
    expect(browser.getTitle()).toEqual('My Angular App');
  });
});
  • Explicación:
    • describe: Define un conjunto de pruebas.
    • it: Define un caso de prueba.
    • browser.get: Navega a la URL especificada.
    • expect: Afirma el resultado esperado.

Escenarios comunes de pruebas E2E

  • Navegación de páginas: 
    it('should navigate to the about page', function() {
  element(by.linkText('About')).click();
  expect(browser.getCurrentUrl()).toContain('/about');
});
  • Interacciones con formularios: 
    it('should submit a form', function() {
  element(by.css('input[name="username"]')).sendKeys('testuser');
  element(by.css('input[name="password"]')).sendKeys('password');
  element(by.buttonText('Submit')).click();
  expect(element(by.css('.success-message')).isPresent()).toBe(true);
});

Técnicas avanzadas de pruebas E2E

  • Localizadores personalizados: Crear localizadores personalizados para elementos más complejos. 
    by.addLocator('buttonText', function(buttonText, opt_parentElement) {
  var using = opt_parentElement || document;
  return using.querySelectorAll('button:contains("' + buttonText + '")');
});
  • Manejo de operaciones asíncronas: Usar async y await para manejar operaciones asíncronas. 
    it('should wait for an element to be visible', async function() {
  await browser.get('http://localhost:4200');
  let myElement = element(by.css('.async-element'));
  await browser.wait(protractor.ExpectedConditions.visibilityOf(myElement), 5000);
  expect(myElement.isPresent()).toBe(true);
});

Mejores prácticas para pruebas E2E

  • Mantén las pruebas independientes: Cada prueba debe poder ejecutarse independientemente de las demás.
  • Usa objetos de página: Encapsula la lógica específica de la página en objetos de página para mejorar el mantenimiento.
  • Ejecuta pruebas con frecuencia: Integra las pruebas E2E en tu pipeline de CI/CD para detectar problemas temprano.

Conclusión

Las pruebas de extremo a extremo son una práctica esencial para asegurar la fiabilidad y corrección de tus aplicaciones Angular. Usando Protractor, puedes simular interacciones reales de usuarios y validar todo el flujo de la aplicación. Recuerda seguir las mejores prácticas para mantener tus pruebas mantenibles y eficientes.
