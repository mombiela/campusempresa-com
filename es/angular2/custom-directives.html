Introducción

Las directivas personalizadas en Angular 2+ te permiten extender la funcionalidad de los elementos HTML y crear componentes reutilizables. Las directivas pueden ser utilizadas para manipular el DOM, aplicar estilos o añadir comportamiento a los elementos.

Tipos de Directivas

Angular 2+ proporciona tres tipos de directivas:

  • Directivas de Componente: Estas son las directivas más comunes y se utilizan para crear componentes.
  • Directivas Estructurales: Estas directivas cambian la disposición del DOM añadiendo o eliminando elementos (por ejemplo, *ngIf, *ngFor).
  • Directivas de Atributo: Estas directivas cambian la apariencia o el comportamiento de un elemento, componente u otra directiva (por ejemplo, ngClass, ngStyle).

Creando una Directiva de Atributo Personalizada

Las directivas de atributo se utilizan para cambiar el comportamiento o la apariencia de un elemento. Vamos a crear una simple directiva de atributo personalizada que cambia el color de fondo de un elemento cuando se pasa el ratón sobre él.

Guía Paso a Paso

  1. Generar la Directiva: Usa Angular CLI para generar una nueva directiva.

    ng generate directive highlight

  2. Implementar la Directiva: Abre el archivo generado highlight.directive.ts e implementa la lógica de la directiva.

    import { Directive, ElementRef, HostListener, Input } from '@angular/core';

@Directive({
  selector: '[appHighlight]'
})
export class HighlightDirective {
  @Input() defaultColor: string = 'yellow';
  @Input('appHighlight') highlightColor: string;

  constructor(private el: ElementRef) {}

  @HostListener('mouseenter') onMouseEnter() {
    this.highlight(this.highlightColor || this.defaultColor);
  }

  @HostListener('mouseleave') onMouseLeave() {
    this.highlight(null);
  }

  private highlight(color: string) {
    this.el.nativeElement.style.backgroundColor = color;
  }
}

  3. Usar la Directiva en un Componente: Aplica la directiva a un elemento en tu plantilla de componente.

    <p appHighlight="lightblue" defaultColor="lightgreen">
  Pasa el ratón sobre este texto para ver el efecto de resaltado.
</p>

Explicación

  • @Directive: Decorador que marca una clase como una directiva de Angular.
  • selector: El selector CSS que identifica la directiva en una plantilla.
  • ElementRef: Servicio que otorga acceso directo al elemento del DOM.
  • @HostListener: Decorador que escucha eventos del DOM y dispara el método especificado.
  • @Input: Decorador que permite pasar datos a la directiva desde el componente padre.

Creando una Directiva Estructural Personalizada

Las directivas estructurales cambian la disposición del DOM añadiendo o eliminando elementos. Vamos a crear una directiva estructural personalizada que incluya condicionalmente una plantilla basada en una expresión booleana.

Guía Paso a Paso

  1. Generar la Directiva: Usa Angular CLI para generar una nueva directiva.

    ng generate directive unless

  2. Implementar la Directiva: Abre el archivo generado unless.directive.ts e implementa la lógica de la directiva.

    import { Directive, Input, TemplateRef, ViewContainerRef } from '@angular/core';

@Directive({
  selector: '[appUnless]'
})
export class UnlessDirective {
  private hasView = false;

  constructor(
    private templateRef: TemplateRef<any>,
    private viewContainer: ViewContainerRef
  ) {}

  @Input() set appUnless(condition: boolean) {
    if (!condition && !this.hasView) {
      this.viewContainer.createEmbeddedView(this.templateRef);
      this.hasView = true;
    } else if (condition && this.hasView) {
      this.viewContainer.clear();
      this.hasView = false;
    }
  }
}

  3. Usar la Directiva en un Componente: Aplica la directiva a un elemento en tu plantilla de componente.

    <div *appUnless="isVisible">
  Este texto se muestra solo cuando isVisible es falso.
</div>

Explicación

  • TemplateRef: Representa una plantilla incrustada que puede ser utilizada para instanciar vistas incrustadas.
  • ViewContainerRef: Representa un contenedor donde una o más vistas pueden ser adjuntadas.
  • @Input: Decorador que permite pasar datos a la directiva desde el componente padre.

Conclusión

Las directivas personalizadas en Angular 2+ son herramientas poderosas que te permiten extender la funcionalidad de los elementos HTML y crear componentes reutilizables. Al entender e implementar directivas de atributo y estructurales, puedes mejorar la interactividad y mantenibilidad de tu aplicación. Practica creando tus propias directivas personalizadas para volverte competente en el desarrollo con Angular.
