El manejo de errores es un aspecto crucial de cualquier aplicación, asegurando que tu aplicación pueda manejar situaciones inesperadas de manera elegante y proporcionar retroalimentación significativa a los usuarios. En esta sección, cubriremos el manejo de errores en Angular 2+ desde niveles principiantes hasta avanzados.

Introducción al Manejo de Errores

El manejo de errores en Angular implica capturar y gestionar errores que ocurren durante la ejecución de tu aplicación. Esto puede incluir errores en solicitudes HTTP, lógica de componentes y más.

Manejo Básico de Errores

Bloques Try-Catch

La forma más simple de manejo de errores en JavaScript es usando bloques try-catch. Esto se puede usar en componentes y servicios de Angular.

try {
  // Código que puede lanzar un error
} catch (error) {
  console.error('Ocurrió un error:', error);
}

Manejo de Errores HTTP

Al hacer solicitudes HTTP usando HttpClient de Angular, puedes manejar errores usando el operador catchError de RxJS.

import { HttpClient } from '@angular/common/http';
import { catchError } from 'rxjs/operators';
import { throwError } from 'rxjs';

constructor(private http: HttpClient) {}

getData() {
  this.http.get('https://api.example.com/data')
    .pipe(
      catchError(error => {
        console.error('Error HTTP:', error);
        return throwError(error);
      })
    )
    .subscribe(
      data => console.log('Datos:', data),
      error => console.error('Error de Suscripción:', error)
    );
}

Manejo Intermedio de Errores

Manejo Global de Errores

Angular proporciona una manera de manejar errores globalmente usando la clase ErrorHandler. Puedes crear un manejador de errores personalizado extendiendo esta clase.

import { ErrorHandler, Injectable } from '@angular/core';

@Injectable()
export class GlobalErrorHandler implements ErrorHandler {
  handleError(error: any): void {
    // Manejar el error aquí
    console.error('Error Global:', error);
  }
}

Para usar este manejador de errores global, proporciónalo en tu módulo de la aplicación.

import { NgModule, ErrorHandler } from '@angular/core';
import { GlobalErrorHandler } from './global-error-handler';

@NgModule({
  providers: [{ provide: ErrorHandler, useClass: GlobalErrorHandler }]
})
export class AppModule {}

Páginas de Error Personalizadas

Puedes crear páginas de error personalizadas para proporcionar una mejor experiencia de usuario cuando ocurra un error.

// error-page.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-error-page',
  template: `<h1>Ocurrió un error</h1><p>Por favor, inténtelo de nuevo más tarde.</p>`
})
export class ErrorPageComponent {}

Luego, configura tus rutas para mostrar la página de error.

import { RouterModule, Routes } from '@angular/router';
import { ErrorPageComponent } from './error-page/error-page.component';

const routes: Routes = [
  // Otras rutas
  { path: 'error', component: ErrorPageComponent },
  { path: '**', redirectTo: '/error' }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule {}

Manejo Avanzado de Errores

Interceptores de Errores

Los interceptores HTTP pueden ser usados para manejar errores globalmente para todas las solicitudes HTTP.

import { Injectable } from '@angular/core';
import { HttpInterceptor, HttpRequest, HttpHandler, HttpEvent, HttpErrorResponse } from '@angular/common/http';
import { Observable, throwError } from 'rxjs';
import { catchError } from 'rxjs/operators';

@Injectable()
export class ErrorInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    return next.handle(req).pipe(
      catchError((error: HttpErrorResponse) => {
        console.error('Error del Interceptor HTTP:', error);
        return throwError(error);
      })
    );
  }
}

Para usar el interceptor, proporciónalo en tu módulo de la aplicación.

import { HTTP_INTERCEPTORS } from '@angular/common/http';
import { ErrorInterceptor } from './error-interceptor';

@NgModule({
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: ErrorInterceptor, multi: true }
  ]
})
export class AppModule {}

Servicios de Registro de Errores

Para el manejo avanzado de errores, podrías querer registrar errores en un servicio externo.

import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';

@Injectable()
export class ErrorLoggingService {
  constructor(private http: HttpClient) {}

  logError(error: any): void {
    // Enviar el error a un servicio de registro externo
    this.http.post('https://logging-service.example.com/log', { error })
      .subscribe();
  }
}

Luego puedes usar este servicio en tu manejador de errores global.

import { ErrorHandler, Injectable } from '@angular/core';
import { ErrorLoggingService } from './error-logging.service';

@Injectable()
export class GlobalErrorHandler implements ErrorHandler {
  constructor(private errorLoggingService: ErrorLoggingService) {}

  handleError(error: any): void {
    console.error('Error Global:', error);
    this.errorLoggingService.logError(error);
  }
}

Conclusión

El manejo de errores es una parte esencial de la construcción de aplicaciones robustas en Angular. Al entender e implementar varias estrategias de manejo de errores, puedes asegurar que tu aplicación pueda manejar situaciones inesperadas de manera elegante y proporcionar una mejor experiencia de usuario. Desde bloques try-catch básicos hasta interceptores de errores avanzados y servicios de registro, Angular proporciona un conjunto completo de herramientas para gestionar errores de manera efectiva.
