Introducción a los Parámetros de Ruta

Los parámetros de ruta son una forma de pasar información a una ruta en Angular. Son esenciales para crear aplicaciones dinámicas y flexibles donde el mismo componente puede mostrar diferentes datos basados en los parámetros de la ruta.

Configuración de Parámetros de Ruta

Para usar parámetros de ruta, necesitas definirlos en tu configuración de rutas.

Definición de Parámetros de Ruta

Así es como defines parámetros de ruta en tu app-routing.module.ts:

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { UserComponent } from './user/user.component';

const routes: Routes = [
  { path: 'user/:id', component: UserComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

En este ejemplo:

  • path: 'user/:id' define una ruta con un parámetro id.

Acceso a los Parámetros de Ruta

Una vez que has definido los parámetros de ruta, puedes acceder a ellos en tu componente.

Usando ActivatedRoute

Para acceder a los parámetros de ruta, usas el servicio ActivatedRoute.

import { Component, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-user',
  template: `<h1>User ID: {{ userId }}</h1>`
})
export class UserComponent implements OnInit {
  userId: string;

  constructor(private route: ActivatedRoute) {}

  ngOnInit(): void {
    this.userId = this.route.snapshot.paramMap.get('id');
  }
}

En este ejemplo:

  • ActivatedRoute es inyectado en el componente.
  • this.route.snapshot.paramMap.get('id') recupera el parámetro id de la ruta.

Uso de Parámetros de Ruta en Plantillas

Puedes usar los parámetros de ruta recuperados directamente en tus plantillas.

<h1>User ID: {{ userId }}</h1>

Esto mostrará el ID del usuario en la página.

Reacción a Cambios en los Parámetros

A veces, los parámetros de ruta pueden cambiar mientras el componente aún está activo. Para manejar tales casos, necesitas suscribirte a los cambios de parámetros.

Suscripción a Cambios de Parámetros

Así es como puedes suscribirte a los cambios de parámetros de ruta:

import { Component, OnInit, OnDestroy } from '@angular/core';
import { ActivatedRoute } from '@angular/router';
import { Subscription } from 'rxjs';

@Component({
  selector: 'app-user',
  template: `<h1>User ID: {{ userId }}</h1>`
})
export class UserComponent implements OnInit, OnDestroy {
  userId: string;
  private routeSub: Subscription;

  constructor(private route: ActivatedRoute) {}

  ngOnInit(): void {
    this.routeSub = this.route.paramMap.subscribe(params => {
      this.userId = params.get('id');
    });
  }

  ngOnDestroy(): void {
    this.routeSub.unsubscribe();
  }
}

En este ejemplo:

  • this.route.paramMap.subscribe se usa para reaccionar a los cambios de parámetros.
  • ngOnDestroy asegura que la suscripción se limpie cuando el componente sea destruido.

Parámetros de Ruta Opcionales

Angular también soporta parámetros de ruta opcionales.

Definición de Parámetros de Ruta Opcionales

Los parámetros opcionales se definen usando la propiedad queryParams.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { UserComponent } from './user/user.component';

const routes: Routes = [
  { path: 'user', component: UserComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Acceso a Parámetros de Ruta Opcionales

Puedes acceder a los parámetros opcionales usando la propiedad queryParams de ActivatedRoute.

import { Component, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-user',
  template: `<h1>User ID: {{ userId }}</h1>`
})
export class UserComponent implements OnInit {
  userId: string;

  constructor(private route: ActivatedRoute) {}

  ngOnInit(): void {
    this.route.queryParams.subscribe(params => {
      this.userId = params['id'];
    });
  }
}

En este ejemplo:

  • this.route.queryParams.subscribe se usa para reaccionar a los cambios de parámetros opcionales.

Resumen

  • Parámetros de Ruta: Permiten pasar datos dinámicos a las rutas.
  • Definición de Parámetros: Usa :param en la ruta.
  • Acceso a Parámetros: Usa el servicio ActivatedRoute.
  • Reacción a Cambios: Suscríbete a paramMap o queryParams.
  • Parámetros Opcionales: Accedidos vía queryParams.

Entender y utilizar los parámetros de ruta de manera efectiva puede mejorar significativamente la flexibilidad y dinamismo de tus aplicaciones Angular.
