Introducción

TypeScript es un superconjunto de JavaScript con tipado estático opcional y otras características adicionales. Entender las variables y los tipos de datos en TypeScript es crucial para desarrollar aplicaciones robustas en Angular. Esta sección cubrirá los conceptos básicos de la declaración de variables y los diversos tipos de datos disponibles en TypeScript.

Declaración de Variables

En TypeScript, las variables pueden ser declaradas usando let, const y var. Cada una tiene su propio alcance y consideraciones de uso.

`let`

  • Alcance de bloque.
  • Puede ser reasignada.
let message: string = "¡Hola, TypeScript!";
message = "¡Hola, Angular!";
console.log(message); // Salida: ¡Hola, Angular!

`const`

  • Alcance de bloque.
  • No puede ser reasignada.
const pi: number = 3.14;
// pi = 3.14159; // Error: Asignación a una variable constante.
console.log(pi); // Salida: 3.14

`var`

  • Alcance de función.
  • Puede ser reasignada.
var greeting: string = "¡Hola!";
greeting = "¡Hola!";
console.log(greeting); // Salida: ¡Hola!

Tipos de Datos

TypeScript soporta varios tipos de datos, incluyendo tipos primitivos y tipos más complejos.

Tipos Primitivos

  • Number: Representa tanto números enteros como de punto flotante.
  • String: Representa datos de texto.
  • Boolean: Representa valores verdadero/falso.
  • Null: Representa una ausencia intencional de cualquier valor de objeto.
  • Undefined: Representa una variable no inicializada.
  • Symbol: Representa un valor primitivo único e inmutable.
let age: number = 25;
let name: string = "John Doe";
let isStudent: boolean = true;
let notAssigned: undefined = undefined;
let empty: null = null;

Array

Los arrays pueden ser definidos de dos maneras: usando el tipo seguido de [] o usando la sintaxis Array<type>.

let numbers: number[] = [1, 2, 3, 4, 5];
let fruits: Array<string> = ["Manzana", "Banana", "Cereza"];

Tuple

Las tuplas te permiten expresar un array con un número fijo de elementos cuyos tipos son conocidos.

let person: [string, number];
person = ["Alice", 30]; // Correcto
// person = [30, "Alice"]; // Error: El tipo 'number' no es asignable al tipo 'string'.

Enum

Los enums te permiten definir un conjunto de constantes con nombre.

enum Color {
    Rojo,
    Verde,
    Azul
}
let c: Color = Color.Verde;
console.log(c); // Salida: 1

Any

El tipo any te permite optar por no usar la verificación de tipos.

let randomValue: any = 10;
randomValue = "Hola";
randomValue = true;

Void

El tipo void se usa para indicar la ausencia de un valor, comúnmente usado como el tipo de retorno de funciones que no devuelven un valor.

function logMessage(message: string): void {
    console.log(message);
}

Object

El tipo object representa tipos no primitivos.

let user: { name: string; age: number } = {
    name: "John",
    age: 30
};

Inferencia de Tipos

TypeScript puede inferir el tipo de una variable basado en el valor asignado a ella.

let inferredString = "Esto es una cadena"; // inferredString es de tipo string
let inferredNumber = 42; // inferredNumber es de tipo number

Aserciones de Tipo

Las aserciones de tipo se usan para decirle al compilador que trate una variable como un tipo específico.

let someValue: any = "¡Hola, TypeScript!";
let strLength: number = (someValue as string).length;
console.log(strLength); // Salida: 17

Conclusión

Entender las variables y los tipos de datos en TypeScript es fundamental para construir aplicaciones en Angular. Al aprovechar el tipado estático de TypeScript, puedes escribir código más predecible y mantenible. Esta sección cubrió los conceptos básicos de la declaración de variables, los diversos tipos de datos disponibles y cómo usar la inferencia de tipos y las aserciones de tipo. Con este conocimiento, estás bien equipado para manejar datos de manera efectiva en tus proyectos de Angular.
