Configurar un proyecto es el primer paso en el desarrollo de una aplicación Angular. Esta sección te guiará a través del proceso de configuración de un proyecto Angular desde cero, asegurándote de tener todas las herramientas y configuraciones necesarias para comenzar a construir tu aplicación.

Requisitos Previos

Antes de configurar un proyecto Angular, asegúrate de tener lo siguiente instalado en tu máquina:

  • Node.js: Angular requiere Node.js para sus herramientas de construcción y desarrollo.
  • npm (Node Package Manager): Viene con Node.js y se usa para instalar Angular CLI y otras dependencias.
  • Angular CLI: Una herramienta de interfaz de línea de comandos que ayuda a crear, gestionar y construir aplicaciones Angular.

Instalación de Node.js y npm

  1. Descargar e Instalar Node.js:

    • Visita el sitio web oficial de Node.js.
    • Descarga la versión LTS (Long Term Support) para tu sistema operativo.
    • Sigue las instrucciones de instalación.

  2. Verificar la Instalación:

    • Abre tu terminal o símbolo del sistema.
    • Ejecuta los siguientes comandos para verificar las versiones instaladas: 
      node -v
npm -v

Instalación de Angular CLI

  1. Instalar Angular CLI Globalmente:

    • Abre tu terminal o símbolo del sistema.
    • Ejecuta el siguiente comando para instalar Angular CLI globalmente: 
      npm install -g @angular/cli

  2. Verificar la Instalación:

    • Ejecuta el siguiente comando para verificar la versión instalada de Angular CLI: 
      ng version

Creación de un Nuevo Proyecto Angular

Con Angular CLI instalado, ahora puedes crear un nuevo proyecto Angular.

  1. Crear un Nuevo Proyecto:

    • Navega al directorio donde deseas crear tu proyecto.
    • Ejecuta el siguiente comando, reemplazando my-angular-app con el nombre deseado para tu proyecto: 
      ng new my-angular-app

  2. Indicaciones de Configuración del Proyecto:

    • Se te pedirá que elijas algunas opciones de configuración:
      • Would you like to add Angular routing?: Elige Yes o No dependiendo de si necesitas enrutamiento.
      • Which stylesheet format would you like to use?: Elige tu formato de hoja de estilos preferido (CSS, SCSS, SASS, LESS, Stylus).

  3. Navegar al Directorio del Proyecto:

    • Después de que el proyecto se cree, navega al directorio del proyecto: 
      cd my-angular-app

  4. Servir la Aplicación:

    • Ejecuta el siguiente comando para iniciar el servidor de desarrollo: 
      ng serve
    • Abre tu navegador y navega a http://localhost:4200/ para ver tu nueva aplicación Angular en funcionamiento.

Estructura del Proyecto

Entender la estructura de un proyecto Angular es crucial para un desarrollo efectivo. Aquí tienes una visión general de los archivos y directorios clave:

  • e2e/: Pruebas de extremo a extremo para la aplicación.
  • node_modules/: Contiene todos los paquetes npm instalados para el proyecto.
  • src/: Carpeta principal que contiene el código de la aplicación.
    • app/: Contiene los módulos y componentes principales de la aplicación.
    • assets/: Contiene activos estáticos como imágenes y estilos.
    • environments/: Contiene archivos de configuración específicos del entorno.
    • index.html: El archivo HTML principal.
    • main.ts: El punto de entrada principal para la aplicación.
    • styles.css: Estilos globales para la aplicación.
  • angular.json: Archivo de configuración para Angular CLI.
  • package.json: Lista las dependencias y scripts del proyecto.
  • tsconfig.json: Archivo de configuración de TypeScript.

Conclusión

Configurar un proyecto Angular implica instalar las herramientas necesarias, crear un nuevo proyecto usando Angular CLI y entender la estructura del proyecto. Con estos pasos, ahora estás listo para comenzar a desarrollar tu aplicación Angular. En las siguientes secciones, profundizaremos en la construcción de componentes, servicios y otras partes esenciales de una aplicación Angular.
