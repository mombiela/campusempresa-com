Introducción a Angular Universal

Angular Universal es una tecnología que permite renderizar aplicaciones Angular en el lado del servidor. Esto puede mejorar el rendimiento, SEO y la experiencia del usuario al entregar páginas completamente renderizadas al cliente.

Conceptos Clave

  • Renderizado del Lado del Servidor (SSR): El proceso de renderizar páginas web en el servidor en lugar del cliente.
  • Angular Universal: Un conjunto de herramientas y bibliotecas que habilitan SSR para aplicaciones Angular.
  • Beneficios de SEO: Los motores de búsqueda pueden indexar páginas renderizadas en el servidor de manera más efectiva.
  • Mejoras en el Rendimiento: Tiempos de carga inicial más rápidos y mejor rendimiento percibido.

Configuración de Angular Universal

Para comenzar con Angular Universal, necesitas agregar los paquetes necesarios y configurar tu aplicación Angular.

Configuración Paso a Paso

  1. Instalar Paquetes de Angular Universal:

    ng add @nguniversal/express-engine

    Este comando instalará los paquetes requeridos y hará los cambios necesarios en tu proyecto Angular.

  2. Actualizar Configuración de Angular: El comando ng add actualizará automáticamente tu archivo angular.json para incluir configuraciones de renderizado del lado del servidor.

  3. Crear Módulo del Servidor: Angular Universal requiere un módulo del servidor para manejar el renderizado del lado del servidor.

    // src/app/app.server.module.ts
import { NgModule } from '@angular/core';
import { ServerModule } from '@angular/platform-server';
import { AppModule } from './app.module';
import { AppComponent } from './app.component';

@NgModule({
  imports: [
    AppModule,
    ServerModule,
  ],
  bootstrap: [AppComponent],
})
export class AppServerModule {}

  4. Crear Archivo del Servidor: Crea un archivo del servidor para inicializar la aplicación Angular en el servidor.

    // server.ts
import 'zone.js/dist/zone-node';
import { ngExpressEngine } from '@nguniversal/express-engine';
import * as express from 'express';
import { join } from 'path';
import { APP_BASE_HREF } from '@angular/common';
import { existsSync } from 'fs';

import { AppServerModule } from './src/main.server';
import { APP_BASE_HREF } from '@angular/common;

const app = express();
const PORT = process.env.PORT || 4000;
const DIST_FOLDER = join(process.cwd(), 'dist/browser');

app.engine('html', ngExpressEngine({
  bootstrap: AppServerModule,
}));

app.set('view engine', 'html');
app.set('views', DIST_FOLDER);

app.get('*.*', express.static(DIST_FOLDER, {
  maxAge: '1y'
}));

app.get('*', (req, res) => {
  res.render('index', { req, providers: [{ provide: APP_BASE_HREF, useValue: req.baseUrl }] });
});

app.listen(PORT, () => {
  console.log(`Servidor Node escuchando en http://localhost:${PORT}`);
});

  5. Construir y Servir: Construye la aplicación tanto para el cliente como para el servidor:

    npm run build:ssr
npm run serve:ssr

Configuración Avanzada

Una vez que tengas la configuración básica, puedes optimizar y configurar Angular Universal para tus necesidades específicas.

Carga Diferida de Módulos

La carga diferida puede configurarse para trabajar con Angular Universal para mejorar el rendimiento.

// app-routing.module.ts
const routes: Routes = [
  {
    path: '',
    loadChildren: () => import('./home/home.module').then(m => m.HomeModule)
  }
];

Manejo de Solicitudes API

El renderizado del lado del servidor puede manejar las solicitudes API de manera diferente. Asegúrate de que tu código del lado del servidor pueda manejar estas solicitudes adecuadamente.

// server.ts
app.get('/api/data', (req, res) => {
  res.json({ data: 'Datos del lado del servidor' });
});

Estrategias de Caché

Implementa estrategias de caché para mejorar el rendimiento y reducir la carga del servidor.

// server.ts
app.get('*', (req, res) => {
  const cacheKey = req.url;
  if (cache.has(cacheKey)) {
    res.send(cache.get(cacheKey));
  } else {
    res.render('index', { req }, (err, html) => {
      cache.set(cacheKey, html);
      res.send(html);
    });
  }
});

Conclusión

Angular Universal proporciona una forma poderosa de mejorar el rendimiento y SEO de tus aplicaciones Angular a través del renderizado del lado del servidor. Siguiendo los pasos de configuración, puedes aprovechar SSR para entregar aplicaciones web más rápidas y eficientes. Configuraciones avanzadas como la carga diferida, el manejo de solicitudes API y la implementación de estrategias de caché pueden optimizar aún más tu aplicación para un mejor rendimiento y escalabilidad.
