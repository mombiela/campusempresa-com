La interacción entre componentes es un concepto fundamental en Angular que permite a diferentes componentes comunicarse y compartir datos. Este tema cubrirá varios métodos de interacción entre componentes, desde técnicas básicas hasta avanzadas.

Comunicación de Padre a Hijo

La comunicación de padre a hijo es una de las formas más comunes de interacción entre componentes. Involucra pasar datos de un componente padre a un componente hijo usando la vinculación de propiedades.

Decorador Input

El decorador @Input se usa para definir propiedades de entrada en el componente hijo que pueden recibir datos del componente padre.

Ejemplo

Componente Padre (app.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<app-child [childProperty]="parentProperty"></app-child>`,
})
export class AppComponent {
  parentProperty = '¡Hola desde el Padre!';
}

Componente Hijo (child.component.ts):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `<p>{{ childProperty }}</p>`,
})
export class ChildComponent {
  @Input() childProperty: string;
}

En este ejemplo, la parentProperty del componente padre se pasa a la childProperty en el componente hijo usando el decorador @Input.

Comunicación de Hijo a Padre

La comunicación de hijo a padre implica enviar datos desde un componente hijo de vuelta a su componente padre. Esto se puede lograr usando el decorador @Output y EventEmitter.

Decorador Output y EventEmitter

El decorador @Output y EventEmitter se usan para crear eventos personalizados a los que el componente padre puede escuchar.

Ejemplo

Componente Hijo (child.component.ts):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `<button (click)="sendData()">Enviar Datos</button>`,
})
export class ChildComponent {
  @Output() dataEmitter = new EventEmitter<string>();

  sendData() {
    this.dataEmitter.emit('¡Hola desde el Hijo!');
  }
}

Componente Padre (app.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<app-child (dataEmitter)="receiveData($event)"></app-child>
             <p>{{ receivedData }}</p>`,
})
export class AppComponent {
  receivedData: string;

  receiveData(data: string) {
    this.receivedData = data;
  }
}

En este ejemplo, el componente hijo emite un evento con datos, que luego es recibido y manejado por el componente padre.

Comunicación entre Hermanos

Los componentes hermanos pueden comunicarse a través de un componente padre compartido o un servicio compartido.

Servicio Compartido

Un servicio compartido puede usarse para facilitar la comunicación entre componentes hermanos.

Ejemplo

Servicio Compartido (data.service.ts):

import { Injectable } from '@angular/core';
import { Subject } from 'rxjs';

@Injectable({
  providedIn: 'root',
})
export class DataService {
  private dataSubject = new Subject<string>();
  data$ = this.dataSubject.asObservable();

  sendData(data: string) {
    this.dataSubject.next(data);
  }
}

Componente Hermano 1 (sibling1.component.ts):

import { Component } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-sibling1',
  template: `<button (click)="sendData()">Enviar Datos al Hermano 2</button>`,
})
export class Sibling1Component {
  constructor(private dataService: DataService) {}

  sendData() {
    this.dataService.sendData('¡Hola desde el Hermano 1!');
  }
}

Componente Hermano 2 (sibling2.component.ts):

import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-sibling2',
  template: `<p>{{ receivedData }}</p>`,
})
export class Sibling2Component implements OnInit {
  receivedData: string;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.dataService.data$.subscribe((data) => {
      this.receivedData = data;
    });
  }
}

En este ejemplo, Sibling1Component envía datos a Sibling2Component a través del DataService.

Conclusión

La interacción entre componentes es un aspecto crucial para construir aplicaciones en Angular. Entender cómo comunicarse efectivamente entre componentes usando @Input, @Output y servicios compartidos te permitirá crear aplicaciones más dinámicas y receptivas. Al dominar estas técnicas, puedes asegurarte de que tus componentes trabajen juntos sin problemas, lo que lleva a una base de código más mantenible y escalable.
