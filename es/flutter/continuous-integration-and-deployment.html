La Integración Continua (CI) y el Despliegue Continuo (CD) son prácticas esenciales en el desarrollo de software moderno. Ayudan a garantizar que tu código esté siempre en un estado desplegable y que las nuevas características y correcciones de errores puedan ser entregadas de manera rápida y confiable. En esta sección, exploraremos cómo configurar CI/CD para un proyecto de Flutter, desde niveles principiantes hasta avanzados.

Introducción a CI/CD

  • Integración Continua (CI): La práctica de integrar automáticamente los cambios de código de múltiples contribuyentes en un repositorio compartido varias veces al día.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio que pase todas las etapas de tu pipeline de producción a tus usuarios.

Configuración de CI para Flutter

Elegir un Servicio de CI

Hay varios servicios de CI disponibles que soportan Flutter, incluyendo:

  • GitHub Actions
  • Travis CI
  • CircleCI
  • Bitrise

Para este curso, usaremos GitHub Actions ya que es gratuito para repositorios públicos y se integra perfectamente con GitHub.

Flujo de Trabajo Básico de CI con GitHub Actions

  1. Crea un directorio .github/workflows en tu proyecto de Flutter.
  2. Agrega un nuevo archivo de flujo de trabajo, por ejemplo, ci.yml.
name: Flutter CI

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Flutter
      uses: subosito/flutter-action@v1
      with:
        flutter-version: '2.5.0'
    - name: Install dependencies
      run: flutter pub get
    - name: Run tests
      run: flutter test

Explicación

  • name: El nombre del flujo de trabajo.
  • on: Especifica los eventos que desencadenan el flujo de trabajo. Aquí, se desencadena en pushes y pull requests a la rama main.
  • jobs: Define los trabajos que se ejecutarán como parte del flujo de trabajo.
  • steps: Los pasos individuales a ejecutar en el trabajo.
    • actions/checkout@v2: Revisa el repositorio.
    • subosito/flutter-action@v1: Configura Flutter.
    • flutter pub get: Instala dependencias.
    • flutter test: Ejecuta las pruebas.

Configuración de CD para Flutter

Desplegando a Firebase App Distribution

Firebase App Distribution es una gran herramienta para distribuir tus aplicaciones Flutter a testers.

  1. Agrega Firebase a tu proyecto de Flutter.
  2. Crea un nuevo archivo de flujo de trabajo, por ejemplo, cd.yml.
name: Flutter CD

on:
  push:
    tags:
      - 'v*.*.*'

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Flutter
      uses: subosito/flutter-action@v1
      with:
        flutter-version: '2.5.0'
    - name: Install dependencies
      run: flutter pub get
    - name: Build APK
      run: flutter build apk --release
    - name: Upload to Firebase App Distribution
      uses: wzieba/Firebase-Distribution-Github-Action@v1
      with:
        appId: ${{ secrets.FIREBASE_APP_ID }}
        token: ${{ secrets.FIREBASE_TOKEN }}
        groups: testers
        file: build/app/outputs/flutter-apk/app-release.apk

Explicación

  • on: El flujo de trabajo se desencadena en pushes a etiquetas que siguen el patrón v*.*.* (por ejemplo, v1.0.0).
  • steps:
    • flutter build apk --release: Construye el APK de lanzamiento.
    • wzieba/Firebase-Distribution-Github-Action@v1: Sube el APK a Firebase App Distribution.

Gestión de Secretos

  • FIREBASE_APP_ID: El ID de la aplicación para tu proyecto de Firebase.
  • FIREBASE_TOKEN: El token para la autenticación de Firebase CLI.

Puedes agregar estos secretos en la configuración de tu repositorio de GitHub bajo Settings > Secrets.

Prácticas Avanzadas de CI/CD

Ejecutando Pruebas de Integración

Las pruebas de integración pueden ejecutarse en dispositivos reales usando servicios como Firebase Test Lab.

- name: Run integration tests
  run: flutter drive --target=test_driver/app.dart

Cobertura de Código

Puedes generar informes de cobertura de código y subirlos a servicios como Codecov.

- name: Generate code coverage
  run: flutter test --coverage
- name: Upload coverage to Codecov
  uses: codecov/codecov-action@v1
  with:
    file: coverage/lcov.info

Linting y Formateo

Asegura la calidad del código agregando pasos de linting y formateo.

- name: Analyze code
  run: flutter analyze
- name: Format code
  run: flutter format --set-exit-if-changed .

Conclusión

Configurar CI/CD para tu proyecto de Flutter puede mejorar significativamente tu flujo de trabajo de desarrollo al automatizar tareas repetitivas y garantizar que tu código esté siempre en un estado desplegable. Siguiendo los pasos descritos en esta sección, puedes configurar un pipeline robusto de CI/CD que incluya la construcción, prueba y despliegue de tus aplicaciones Flutter.

Al dominar estas prácticas, podrás entregar aplicaciones de alta calidad más rápido y de manera más eficiente, convirtiéndote en un desarrollador más efectivo y productivo.
