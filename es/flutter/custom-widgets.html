En esta sección, exploraremos cómo crear widgets personalizados en Flutter. Los widgets personalizados son esenciales para construir código reutilizable y mantenible. Al final de esta sección, entenderás cómo crear y usar widgets personalizados, y cómo hacerlos flexibles y adaptables a diferentes casos de uso.

Introducción a los Widgets Personalizados

Los widgets personalizados son widgets definidos por el usuario que encapsulan una pieza de la interfaz de usuario y su comportamiento. Ayudan a descomponer interfaces de usuario complejas en piezas más pequeñas y manejables.

Conceptos Clave

  • StatelessWidget: Un widget que no mantiene ningún estado.
  • StatefulWidget: Un widget que mantiene estado y puede cambiar con el tiempo.
  • Composición: Construir widgets complejos combinando otros más simples.

Creando un Widget Personalizado Stateless

Un StatelessWidget es un widget que no cambia su estado una vez que se construye. Es ideal para contenido estático.

Ejemplo: Botón Personalizado Simple

import 'package:flutter/material.dart';

class CustomButton extends StatelessWidget {
  final String text;
  final VoidCallback onPressed;

  CustomButton({required this.text, required this.onPressed});

  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: onPressed,
      child: Text(text),
    );
  }
}

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Ejemplo de Botón Personalizado')),
        body: Center(
          child: CustomButton(
            text: 'Haz clic aquí',
            onPressed: () {
              print('Botón presionado');
            },
          ),
        ),
      ),
    );
  }
}

Explicación

  • Clase CustomButton: Esta clase extiende StatelessWidget y toma dos parámetros: text y onPressed.
  • Método build: Devuelve un ElevatedButton con el texto y la función onPressed proporcionados.

Creando un Widget Personalizado Stateful

Un StatefulWidget es un widget que puede cambiar su estado durante su vida útil. Es útil para contenido dinámico.

Ejemplo: Widget Contador Personalizado

import 'package:flutter/material.dart';

class CustomCounter extends StatefulWidget {
  @override
  _CustomCounterState createState() => _CustomCounterState();
}

class _CustomCounterState extends State<CustomCounter> {
  int _count = 0;

  void _increment() {
    setState(() {
      _count++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        Text('Cuenta: $_count', style: TextStyle(fontSize: 24)),
        SizedBox(height: 20),
        ElevatedButton(
          onPressed: _increment,
          child: Text('Incrementar'),
        ),
      ],
    );
  }
}

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Ejemplo de Contador Personalizado')),
        body: Center(
          child: CustomCounter(),
        ),
      ),
    );
  }
}

Explicación

  • Clase CustomCounter: Extiende StatefulWidget y crea su estado usando _CustomCounterState.
  • Clase _CustomCounterState: Gestiona el estado de CustomCounter. Incluye una variable _count y un método _increment que actualiza el estado.
  • Método build: Devuelve una Column con un widget Text que muestra la cuenta y un ElevatedButton para incrementar la cuenta.

Haciendo los Widgets Personalizados Flexibles

Los widgets personalizados deben ser flexibles y reutilizables. Esto se puede lograr utilizando parámetros y valores predeterminados.

Ejemplo: Botón Personalizado Flexible

import 'package:flutter/material.dart';

class FlexibleButton extends StatelessWidget {
  final String text;
  final VoidCallback onPressed;
  final Color color;
  final double fontSize;

  FlexibleButton({
    required this.text,
    required this.onPressed,
    this.color = Colors.blue,
    this.fontSize = 16.0,
  });

  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      style: ElevatedButton.styleFrom(primary: color),
      onPressed: onPressed,
      child: Text(
        text,
        style: TextStyle(fontSize: fontSize),
      ),
    );
  }
}

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Ejemplo de Botón Flexible')),
        body: Center(
          child: FlexibleButton(
            text: 'Haz clic aquí',
            onPressed: () {
              print('Botón presionado');
            },
            color: Colors.red,
            fontSize: 20.0,
          ),
        ),
      ),
    );
  }
}

Explicación

  • Clase FlexibleButton: Extiende StatelessWidget e incluye parámetros adicionales para color y fontSize con valores predeterminados.
  • Método build: Utiliza los parámetros proporcionados para estilizar el ElevatedButton.

Conclusión

Los widgets personalizados son una característica poderosa en Flutter que te permite crear componentes de interfaz de usuario reutilizables y mantenibles. Al entender la diferencia entre StatelessWidget y StatefulWidget, y aprender a hacer tus widgets flexibles, puedes construir interfaces de usuario complejas de manera más eficiente. Practica creando tus propios widgets personalizados para volverte competente en el desarrollo con Flutter.
