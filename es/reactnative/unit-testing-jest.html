Introducción a las Pruebas Unitarias

Las pruebas unitarias son un método de prueba de software donde unidades o componentes individuales de un software se prueban de forma aislada. En el contexto de React Native, las pruebas unitarias ayudan a asegurar que tus componentes y funciones funcionen como se espera.

¿Por Qué Usar Jest para Pruebas Unitarias?

Jest es un marco de pruebas de JavaScript mantenido por Facebook, diseñado para asegurar la corrección de cualquier base de código JavaScript. Es particularmente adecuado para probar aplicaciones de React y React Native debido a su simplicidad y potentes características.

  • Cero configuración: Jest funciona de inmediato para la mayoría de los proyectos de JavaScript.
  • Pruebas de instantáneas: Jest puede capturar instantáneas de árboles de React para asegurar la consistencia de la interfaz de usuario.
  • Simulación: Jest proporciona potentes capacidades de simulación para aislar componentes y funciones.

Configuración de Jest en un Proyecto de React Native

Instalación

Para comenzar con Jest en un proyecto de React Native, necesitas instalar Jest y sus dependencias.

npm install --save-dev jest @testing-library/react-native @testing-library/jest-native

Configuración

Crea un archivo jest.config.js en la raíz de tu proyecto para configurar Jest.

module.exports = {
  preset: 'react-native',
  setupFilesAfterEnv: ['@testing-library/jest-native/extend-expect'],
  transformIgnorePatterns: [
    'node_modules/(?!(jest-)?react-native|@react-native|@react-navigation)',
  ],
};

Escribiendo Tu Primera Prueba

Probando una Función Simple

Comencemos escribiendo una prueba para una función simple.

// sum.js
export const sum = (a, b) => a + b;
// sum.test.js
import { sum } from './sum';

test('suma 1 + 2 para igualar 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Ejecutando la Prueba

Puedes ejecutar tus pruebas usando el siguiente comando:

npm test

Probando Componentes de React Native

Componente de Ejemplo

Considera un componente simple de React Native:

// Greeting.js
import React from 'react';
import { Text } from 'react-native';

const Greeting = ({ name }) => {
  return <Text>Hello, {name}!</Text>;
};

export default Greeting;

Escribiendo una Prueba para el Componente

Para probar este componente, puedes usar @testing-library/react-native.

// Greeting.test.js
import React from 'react';
import { render } from '@testing-library/react-native';
import Greeting from './Greeting';

test('renderiza el mensaje de saludo correcto', () => {
  const { getByText } = render(<Greeting name="John" />);
  expect(getByText('Hello, John!')).toBeTruthy();
});

Pruebas de Instantáneas

Las pruebas de instantáneas son una característica de Jest que te permite tomar una "instantánea" de tu componente renderizado y compararlo con un archivo de instantánea de referencia almacenado junto a tus pruebas.

Creando una Prueba de Instantánea

// Greeting.test.js
import React from 'react';
import renderer from 'react-test-renderer';
import Greeting from './Greeting';

test('se renderiza correctamente', () => {
  const tree = renderer.create(<Greeting name="John" />).toJSON();
  expect(tree).toMatchSnapshot();
});

Cuando ejecutas esta prueba por primera vez, Jest creará un archivo de instantánea. En ejecuciones posteriores, Jest comparará la salida renderizada con la instantánea existente y te alertará si hay diferencias.

Simulación en Jest

La simulación es útil para aislar el componente o función que estás probando reemplazando dependencias con implementaciones simuladas.

Simulando un Módulo

Supongamos que tienes un módulo que obtiene datos de una API:

// api.js
export const fetchData = async () => {
  const response = await fetch('https://api.example.com/data');
  return response.json();
};

Puedes simular este módulo en tus pruebas:

// api.test.js
import { fetchData } from './api';

jest.mock('./api');

test('fetchData devuelve datos simulados', async () => {
  fetchData.mockResolvedValue({ data: 'mock data' });
  const data = await fetchData();
  expect(data).toEqual({ data: 'mock data' });
});

Conclusión

Las pruebas unitarias con Jest en un proyecto de React Native son una forma poderosa de asegurar que tus componentes y funciones funcionen como se espera. Al configurar Jest, escribir pruebas para funciones y componentes, usar pruebas de instantáneas y aprovechar las capacidades de simulación de Jest, puedes crear una suite de pruebas robusta que ayude a mantener la calidad y confiabilidad de tu base de código.

Recuerda, la clave para pruebas unitarias efectivas es escribir pruebas que sean aisladas, repetibles y fáciles de entender. ¡Felices pruebas!
