Introducción a CodePush

CodePush es un servicio en la nube que permite a los desarrolladores de React Native desplegar actualizaciones de aplicaciones móviles directamente en los dispositivos de sus usuarios. Esto permite correcciones rápidas de errores y mejoras de funciones sin necesidad de pasar por el proceso de revisión de la tienda de aplicaciones.

Conceptos Clave

  • Actualizaciones Over-the-Air (OTA): Actualizaciones que se entregan directamente al dispositivo del usuario sin requerir una nueva versión en la tienda de aplicaciones.
  • CodePush: Un servicio proporcionado por Microsoft que permite actualizaciones OTA para aplicaciones React Native.
  • App Center: Un conjunto de servicios proporcionados por Microsoft que incluye CodePush, entre otras herramientas para el desarrollo y despliegue de aplicaciones.

Configuración de CodePush

Prerrequisitos

  • Un proyecto de React Native.
  • Node.js y npm instalados.
  • Una cuenta en App Center.

Instalación

  1. Instalar la CLI de CodePush:

    npm install -g code-push-cli

  2. Vincular CodePush a tu proyecto de React Native:

    npm install --save react-native-code-push

  3. Vincular el plugin de CodePush:

    npx react-native link react-native-code-push

Configuración

  1. Crear una aplicación CodePush:

    appcenter apps create -d MyApp -o React-Native

  2. Obtener claves de despliegue:

    appcenter codepush deployment list -a <ownerName>/<appName> -k

  3. Configurar CodePush en tu aplicación:

    • Para iOS: Añade la clave de despliegue a tu archivo AppDelegate.m.
    • Para Android: Añade la clave de despliegue a tu archivo MainApplication.java.

Ejemplo de Configuración

iOS (AppDelegate.m):

#import <CodePush/CodePush.h>

- (NSURL *)sourceURLForBridge:(RCTBridge *)bridge
{
  #if DEBUG
    return [[RCTBundleURLProvider sharedSettings] jsBundleURLForBundleRoot:@"index" fallbackResource:nil];
  #else
    return [CodePush bundleURL];
  #endif
}

Android (MainApplication.java):

import com.microsoft.codepush.react.CodePush;

@Override
public String getJSBundleFile() {
  return CodePush.getJSBundleFile();
}

Desplegando Actualizaciones

Creando una Versión

  1. Empaqueta tu JavaScript:

    npx react-native bundle --platform ios --entry-file index.js --bundle-output ./ios/main.jsbundle --assets-dest ./ios

  2. Lanza la Actualización:

    appcenter codepush release-react -a <ownerName>/<appName> -d <deploymentName>

Verificando el Estado del Despliegue

  • Ver Historial de Despliegue: 
    appcenter codepush deployment history -a <ownerName>/<appName> -d <deploymentName>

Manejo de Actualizaciones en tu Aplicación

Ejemplo de Código

App.js:

import React from 'react';
import { View, Text } from 'react-native';
import CodePush from 'react-native-code-push';

class App extends React.Component {
  render() {
    return (
      <View>
        <Text>¡Bienvenido a la demostración de CodePush!</Text>
      </View>
    );
  }
}

let codePushOptions = { checkFrequency: CodePush.CheckFrequency.ON_APP_START };

App = CodePush(codePushOptions)(App);

export default App;

Explicación

  • Opciones de CodePush: La opción checkFrequency determina con qué frecuencia la aplicación busca actualizaciones. ON_APP_START significa que la aplicación buscará actualizaciones cada vez que se inicie.
  • Envolviendo el Componente de la Aplicación: La función CodePush envuelve el componente principal App para habilitar actualizaciones OTA.

Conclusión

Usar CodePush para actualizaciones over-the-air en React Native permite un despliegue rápido de correcciones de errores y nuevas funciones sin la necesidad de aprobaciones de la tienda de aplicaciones. Siguiendo los pasos de configuración y despliegue, puedes asegurarte de que tus usuarios siempre tengan la última versión de tu aplicación. Esto no solo mejora la experiencia del usuario, sino que también reduce el tiempo y esfuerzo necesarios para mantener y actualizar tu aplicación.
