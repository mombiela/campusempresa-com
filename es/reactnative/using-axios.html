Introducción a Axios

Axios es una popular biblioteca de JavaScript utilizada para hacer solicitudes HTTP desde el navegador y Node.js. Es ampliamente utilizada en aplicaciones React Native por su simplicidad y facilidad de uso.

Características Clave de Axios

  • Basado en Promesas: Axios utiliza Promesas, lo que facilita trabajar con solicitudes asíncronas.
  • Interceptores: Permite interceptar solicitudes o respuestas antes de que sean manejadas.
  • Transformación automática de datos JSON: Transforma automáticamente los datos JSON.
  • Manejo de errores: Proporciona una forma sencilla de manejar errores.
  • Cancelación de solicitudes: Soporta la cancelación de solicitudes.

Instalando Axios en un Proyecto React Native

Para usar Axios en un proyecto React Native, necesitas instalarlo a través de npm o yarn.

npm install axios
# o
yarn add axios

Haciendo una Solicitud GET Básica

Para hacer una solicitud GET usando Axios, puedes usar el método axios.get. A continuación se muestra un ejemplo simple de cómo obtener datos de una API.

import axios from 'axios';
import React, { useEffect, useState } from 'react';
import { View, Text } from 'react-native';

const App = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    axios.get('https://jsonplaceholder.typicode.com/posts/1')
      .then(response => {
        setData(response.data);
        setLoading(false);
      })
      .catch(error => {
        console.error('Error fetching data:', error);
        setLoading(false);
      });
  }, []);

  if (loading) {
    return <Text>Loading...</Text>;
  }

  return (
    <View>
      <Text>{data.title}</Text>
      <Text>{data.body}</Text>
    </View>
  );
};

export default App;

Explicación

  • Importando Axios: import axios from 'axios';
  • Gestión de Estado: Usando useState para gestionar los datos obtenidos y el estado de carga.
  • Hook de Efecto: Usando useEffect para obtener datos cuando el componente se monta.
  • Solicitud GET: axios.get para obtener datos de la API.
  • Manejo de Respuesta: Actualizando el estado con los datos obtenidos.
  • Manejo de Errores: Registrando errores en la consola.

Haciendo una Solicitud POST

Para enviar datos a una API, puedes usar el método axios.post. A continuación se muestra un ejemplo de cómo enviar datos a una API.

import axios from 'axios';
import React, { useState } from 'react';
import { View, Text, Button, TextInput } from 'react-native';

const App = () => {
  const [title, setTitle] = useState('');
  const [body, setBody] = useState('');
  const [response, setResponse] = useState(null);

  const handleSubmit = () => {
    axios.post('https://jsonplaceholder.typicode.com/posts', {
      title,
      body,
      userId: 1,
    })
    .then(response => {
      setResponse(response.data);
    })
    .catch(error => {
      console.error('Error posting data:', error);
    });
  };

  return (
    <View>
      <TextInput
        placeholder="Title"
        value={title}
        onChangeText={setTitle}
      />
      <TextInput
        placeholder="Body"
        value={body}
        onChangeText={setBody}
      />
      <Button title="Submit" onPress={handleSubmit} />
      {response && (
        <View>
          <Text>Response:</Text>
          <Text>{response.title}</Text>
          <Text>{response.body}</Text>
        </View>
      )}
    </View>
  );
};

export default App;

Explicación

  • Gestión de Estado: Usando useState para gestionar las entradas del formulario y los datos de respuesta.
  • Solicitud POST: axios.post para enviar datos a la API.
  • Manejo de Respuesta: Actualizando el estado con los datos de respuesta.
  • Manejo de Errores: Registrando errores en la consola.

Uso de Interceptores de Axios

Los interceptores te permiten ejecutar tu código o modificar la solicitud/respuesta antes de que sea manejada por then o catch.

Añadiendo un Interceptor de Solicitud

axios.interceptors.request.use(
  config => {
    // Hacer algo antes de que la solicitud sea enviada
    console.log('Request Interceptor:', config);
    return config;
  },
  error => {
    // Hacer algo con el error de la solicitud
    return Promise.reject(error);
  }
);

Añadiendo un Interceptor de Respuesta

axios.interceptors.response.use(
  response => {
    // Hacer algo con los datos de la respuesta
    console.log('Response Interceptor:', response);
    return response;
  },
  error => {
    // Hacer algo con el error de la respuesta
    return Promise.reject(error);
  }
);

Explicación

  • Interceptor de Solicitud: Te permite modificar la solicitud antes de que sea enviada.
  • Interceptor de Respuesta: Te permite modificar la respuesta antes de que sea manejada.

Manejo de Errores en Axios

El manejo de errores es crucial para una aplicación robusta. Axios proporciona una forma sencilla de manejar errores.

Ejemplo de Manejo de Errores

axios.get('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => {
    console.log('Data:', response.data);
  })
  .catch(error => {
    if (error.response) {
      // El servidor respondió con un estado diferente al rango 200
      console.error('Response Error:', error.response.data);
    } else if (error.request) {
      // La solicitud fue hecha pero no se recibió respuesta
      console.error('Request Error:', error.request);
    } else {
      // Algo más sucedió
      console.error('Error:', error.message);
    }
  });

Explicación

  • Tipos de Errores:
    • Error de Respuesta: Error de respuesta del servidor.
    • Error de Solicitud: No se recibió respuesta del servidor.
    • Otros Errores: Cualquier otro error.

Conclusión

Axios es una biblioteca poderosa y fácil de usar para hacer solicitudes HTTP en aplicaciones React Native. Simplifica el proceso de obtener y enviar datos, manejar errores y gestionar interceptores de solicitud/respuesta. Al entender y utilizar Axios, puedes mejorar las capacidades de red de tus aplicaciones React Native, haciéndolas más robustas y eficientes.
