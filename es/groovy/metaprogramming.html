Metaprogramación en Groovy
Constructores ►

La metaprogramación es una característica poderosa en Groovy que te permite escribir código que escribe o modifica otro código en tiempo de ejecución. Esto puede ser particularmente útil para crear aplicaciones más flexibles y dinámicas. En esta sección, exploraremos los conceptos básicos de la metaprogramación en Groovy, avanzando desde conceptos para principiantes hasta técnicas más avanzadas.

Introducción a la Metaprogramación

La metaprogramación en Groovy se puede dividir en dos categorías principales:

  • Metaprogramación en tiempo de compilación: Modificar el código durante el proceso de compilación.
  • Metaprogramación en tiempo de ejecución: Modificar el código mientras el programa se está ejecutando.

Conceptos Clave

  • Invocación Dinámica de Métodos: Agregar métodos a las clases en tiempo de ejecución.
  • ExpandoMetaClass: Una clase que te permite agregar métodos, propiedades y constructores a clases existentes.
  • Categorías: Agregar métodos temporalmente a clases dentro de un ámbito específico.
  • Transformaciones AST: Modificar el Árbol de Sintaxis Abstracta (AST) durante la compilación.

Invocación Dinámica de Métodos

La invocación dinámica de métodos te permite agregar métodos a las clases en tiempo de ejecución. Esto se puede hacer usando la propiedad metaClass.

Ejemplo

class Person {
    String name
}

Person.metaClass.sayHello = { ->
    println "Hello, my name is ${delegate.name}"
}

def john = new Person(name: 'John')
john.sayHello()  // Output: Hello, my name is John

En este ejemplo, agregamos dinámicamente un método sayHello a la clase Person.

ExpandoMetaClass

El ExpandoMetaClass te permite agregar métodos, propiedades y constructores a clases existentes. Está habilitado por defecto en Groovy.

Ejemplo

class Car {
    String model
}

Car.metaClass.startEngine = { ->
    println "Engine started for ${delegate.model}"
}

def tesla = new Car(model: 'Tesla Model S')
tesla.startEngine()  // Output: Engine started for Tesla Model S

Aquí, agregamos un método startEngine a la clase Car usando ExpandoMetaClass.

Categorías

Las categorías te permiten agregar métodos a clases dentro de un ámbito específico. Esto es útil cuando deseas extender clases temporalmente.

Ejemplo

class StringCategory {
    static String shout(String str) {
        return str.toUpperCase() + "!!!"
    }
}

use(StringCategory) {
    println "hello".shout()  // Output: HELLO!!!
}

En este ejemplo, definimos una clase StringCategory con un método shout y la usamos dentro de un ámbito específico.

Transformaciones AST

Las transformaciones AST (Árbol de Sintaxis Abstracta) te permiten modificar la estructura del código durante el proceso de compilación. Groovy proporciona varias transformaciones AST integradas, como @Delegate, @Immutable y @Singleton.

Ejemplo

import groovy.transform.Immutable

@Immutable
class Point {
    int x, y
}

def p = new Point(x: 10, y: 20)
println p  // Output: Point(10, 20)

En este ejemplo, la transformación @Immutable hace que la clase Point sea inmutable, lo que significa que sus propiedades no pueden ser cambiadas después de la inicialización.

Conclusión

La metaprogramación en Groovy proporciona herramientas poderosas para modificar y extender dinámicamente tu código. Al comprender y utilizar la invocación dinámica de métodos, ExpandoMetaClass, categorías y transformaciones AST, puedes crear aplicaciones más flexibles y dinámicas. A medida que te sientas más cómodo con estos conceptos, podrás aprovechar todo el poder de las capacidades de metaprogramación de Groovy.
