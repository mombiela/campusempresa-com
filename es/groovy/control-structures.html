Estructuras de Control en Groovy
Listas ►

Las estructuras de control son fundamentales en la programación ya que te permiten dictar el flujo de tu programa. En Groovy, las estructuras de control son similares a las de Java pero con algunas mejoras y azúcar sintáctica que las hacen más concisas y expresivas.

Declaraciones Condicionales

`if` y `else`

Las declaraciones if y else se utilizan para ejecutar bloques de código basados en una condición.

def age = 18

if (age >= 18) {
    println "Eres un adulto."
} else {
    println "Eres un menor."
}
  • Conceptos Clave:
    • La condición dentro de la declaración if debe evaluar a un booleano.
    • El bloque else es opcional y se ejecuta si la condición del if es falsa.

`else if`

La declaración else if te permite verificar múltiples condiciones.

def score = 85

if (score >= 90) {
    println "Calificación: A"
} else if (score >= 80) {
    println "Calificación: B"
} else if (score >= 70) {
    println "Calificación: C"
} else {
    println "Calificación: F"
}
  • Conceptos Clave:
    • Se pueden encadenar múltiples bloques else if.
    • La primera condición que evalúe a verdadero ejecutará su bloque.

Declaración Switch

La declaración switch en Groovy es más poderosa que en Java, permitiendo valores de caso más flexibles.

def fruit = "Apple"

switch (fruit) {
    case "Apple":
        println "Es una manzana."
        break
    case "Banana":
        println "Es un plátano."
        break
    default:
        println "Fruta desconocida."
}
  • Conceptos Clave:
    • La declaración switch evalúa la expresión y la compara con los valores de los casos.
    • La declaración break se usa para salir del bloque switch.
    • El caso default es opcional y se ejecuta si ningún otro caso coincide.

Estructuras de Bucle

Bucle `for`

El bucle for en Groovy puede iterar sobre un rango, lista o cualquier objeto iterable.

for (i in 1..5) {
    println "Iteración: $i"
}
  • Conceptos Clave:
    • La palabra clave in se usa para iterar sobre un rango o colección.
    • La variable del bucle (i en este caso) toma cada valor en el rango o colección.

Bucle `while`

El bucle while continúa ejecutándose mientras su condición sea verdadera.

def count = 1

while (count <= 5) {
    println "Cuenta: $count"
    count++
}
  • Conceptos Clave:
    • La condición se evalúa antes de cada iteración.
    • Si la condición es falsa inicialmente, el cuerpo del bucle no se ejecutará.

Bucle `do-while`

El bucle do-while es similar al bucle while pero garantiza que el cuerpo del bucle se ejecutará al menos una vez.

def count = 1

do {
    println "Cuenta: $count"
    count++
} while (count <= 5)
  • Conceptos Clave:
    • El cuerpo del bucle se ejecuta antes de que se verifique la condición.
    • Útil cuando el cuerpo del bucle necesita ejecutarse al menos una vez independientemente de la condición.

Bucle Mejorado con Groovy

Método `each`

Groovy proporciona el método each para iterar sobre colecciones de una manera más expresiva.

def list = [1, 2, 3, 4, 5]

list.each { item ->
    println "Elemento: $item"
}
  • Conceptos Clave:
    • El método each toma un closure que se ejecuta para cada elemento en la colección.
    • La variable del bucle (item en este caso) se define dentro del closure.

Método `collect`

El método collect transforma cada elemento en una colección y devuelve una nueva colección.

def list = [1, 2, 3, 4, 5]
def squaredList = list.collect { it * it }

println squaredList  // Salida: [1, 4, 9, 16, 25]
  • Conceptos Clave:
    • El método collect toma un closure que transforma cada elemento.
    • El resultado es una nueva colección con los elementos transformados.

Conclusión

Las estructuras de control en Groovy proporcionan formas poderosas y flexibles para controlar el flujo de tu programa. Desde declaraciones condicionales básicas y bucles hasta métodos mejorados como each y collect, Groovy ofrece un conjunto rico de herramientas para hacer tu código más expresivo y conciso. Entender estas estructuras de control es esencial para escribir programas efectivos en Groovy.
