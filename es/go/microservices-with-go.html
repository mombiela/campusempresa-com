Introducción a los Microservicios

  • Definición: Los microservicios son un estilo arquitectónico que estructura una aplicación como una colección de servicios acoplados de manera flexible.
  • Beneficios:
    • Escalabilidad
    • Flexibilidad
    • Despliegue independiente
    • Aislamiento de fallos
  • Desafíos:
    • Complejidad en la gestión
    • Latencia de red
    • Consistencia de datos

¿Por qué Go para Microservicios?

  • Rendimiento: Go es un lenguaje compilado, que ofrece alto rendimiento.
  • Concurrencia: Soporte incorporado para programación concurrente con goroutines.
  • Simplicidad: Fácil de aprender y escribir, con una sintaxis limpia.
  • Biblioteca Estándar: Rica biblioteca estándar que soporta la construcción de servidores web y el manejo de solicitudes HTTP.

Configurando tu Entorno Go

  • Instalar Go: Sigue la guía oficial de instalación de Go.
  • Configuración del Espacio de Trabajo: 
    mkdir -p ~/go/src/microservices
cd ~/go/src/microservices

Construyendo tu Primer Microservicio

  • Estructura del Proyecto: 
    microservices/
├── main.go
├── handlers/
│   └── hello.go
└── go.mod
  • main.go: 
    package main

import (
    "log"
    "net/http"
    "microservices/handlers"
)

func main() {
    http.HandleFunc("/hello", handlers.HelloHandler)
    log.Fatal(http.ListenAndServe(":8080", nil))
}
  • handlers/hello.go: 
    package handlers

import (
    "fmt"
    "net/http"
)

func HelloHandler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "¡Hola, Mundo!")
}
  • go.mod: 
    module microservices

go 1.17

Manejo de Solicitudes HTTP

  • Enrutamiento: Usa el paquete net/http para manejar rutas.
  • Ejemplo: 
    http.HandleFunc("/hello", handlers.HelloHandler)

Gestión de Datos

  • Usando JSON: 
    import (
    "encoding/json"
    "net/http"
)

type Response struct {
    Message string `json:"message"`
}

func HelloHandler(w http.ResponseWriter, r *http.Request) {
    response := Response{Message: "¡Hola, Mundo!"}
    json.NewEncoder(w).Encode(response)
}

Concurrencia con Goroutines

  • Goroutines: Hilos ligeros gestionados por el runtime de Go.
  • Ejemplo: 
    func main() {
    go func() {
        fmt.Println("Hola desde una goroutine")
    }()
    fmt.Println("Hola desde main")
}

Comunicación Entre Microservicios

  • APIs RESTful: Usa HTTP para la comunicación.
  • Ejemplo: 
    resp, err := http.Get("http://localhost:8080/hello")
if err != nil {
    log.Fatal(err)
}
defer resp.Body.Close()
body, err := ioutil.ReadAll(resp.Body)
if err != nil {
    log.Fatal(err)
}
fmt.Println(string(body))

Manejo de Errores

  • Mejores Prácticas:
    • Verifica errores después de cada operación.
    • Usa log.Fatal para errores críticos.
    • Devuelve códigos de estado HTTP significativos.

Pruebas de Microservicios

  • Pruebas Unitarias: 
    import (
    "net/http"
    "net/http/httptest"
    "testing"
)

func TestHelloHandler(t *testing.T) {
    req, err := http.NewRequest("GET", "/hello", nil)
    if err != nil {
        t.Fatal(err)
    }
    rr := httptest.NewRecorder()
    handler := http.HandlerFunc(HelloHandler)
    handler.ServeHTTP(rr, req)

    if status := rr.Code; status != http.StatusOK {
        t.Errorf("el manejador devolvió un código de estado incorrecto: obtuvo %v, esperaba %v", status, http.StatusOK)
    }

    expected := `{"message":"¡Hola, Mundo!"}`
    if rr.Body.String() != expected {
        t.Errorf("el manejador devolvió un cuerpo inesperado: obtuvo %v, esperaba %v", rr.Body.String(), expected)
    }
}

Estrategias de Despliegue

  • Docker: Conteneriza tus microservicios. 
    FROM golang:1.17-alpine
WORKDIR /app
COPY . .
RUN go build -o main .
CMD ["./main"]
  • Kubernetes: Orquesta tus contenedores.

Conclusión

Los microservicios con Go ofrecen una poderosa combinación de rendimiento, simplicidad y concurrencia. Siguiendo las mejores prácticas y aprovechando la robusta biblioteca estándar de Go, puedes construir microservicios escalables y mantenibles. Este curso ha cubierto los conceptos básicos de la configuración de un entorno Go, la construcción de microservicios, el manejo de solicitudes HTTP, la gestión de datos y el despliegue de tus servicios. Continúa explorando temas más avanzados como el descubrimiento de servicios, las puertas de enlace API y el rastreo distribuido para mejorar aún más tu arquitectura de microservicios.
