Introducción

En Go, el manejo de errores se realiza típicamente usando el tipo error. Sin embargo, hay situaciones en las que un error es tan grave que el programa no puede continuar ejecutándose normalmente. Para estos casos, Go proporciona los mecanismos panic y recover. Esta sección cubrirá cómo usar panic y recover, sus propósitos y mejores prácticas.

Pánico

Panic es una función incorporada que detiene el flujo ordinario de control y comienza a entrar en pánico. Cuando una función entra en pánico, deja de ejecutarse y cualquier función diferida se ejecuta en orden inverso. Después de que todas las funciones diferidas se han ejecutado, el programa se bloquea con un mensaje de registro.

Cuándo Usar Pánico

  • Cuando el programa no puede continuar ejecutándose, como cuando encuentra un error crítico.
  • Cuando se alcanza un estado inesperado que nunca debería ocurrir si el programa es correcto.

Ejemplo de Pánico

package main

import "fmt"

func main() {
    fmt.Println("Inicio del programa")
    panic("¡Algo salió mal!")
    fmt.Println("Esta línea no se ejecutará")
}

En este ejemplo, el programa imprimirá "Inicio del programa" y luego entrará en pánico con el mensaje "¡Algo salió mal!". La línea después del pánico no se ejecutará.

Recuperación

Recover es una función incorporada que recupera el control de una goroutine en pánico. Recover solo es útil dentro de funciones diferidas. Cuando se llama dentro de una función diferida, recover detiene el pánico y devuelve el valor que se pasó a la llamada de panic.

Cuándo Usar Recuperación

  • Para manejar un pánico y permitir que el programa continúe ejecutándose.
  • Para registrar el pánico y realizar la limpieza antes de que el programa se bloquee.

Ejemplo de Recuperación

package main

import "fmt"

func main() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recuperado del pánico:", r)
        }
    }()
    fmt.Println("Inicio del programa")
    panic("¡Algo salió mal!")
    fmt.Println("Esta línea no se ejecutará")
}

En este ejemplo, la función diferida usa recover para atrapar el pánico. El programa imprimirá "Inicio del programa" y luego "Recuperado del pánico: ¡Algo salió mal!". La línea después del pánico aún no se ejecutará, pero el programa no se bloqueará.

Mejores Prácticas

  • Usar pánico con moderación: Panic debe usarse para situaciones verdaderamente excepcionales. Para el manejo regular de errores, use el tipo error.
  • Recuperar en el nivel superior: Use recover en una función de nivel superior para asegurarse de que su programa pueda manejar los pánicos de manera elegante.
  • Registrar pánicos: Siempre registre el mensaje de pánico para ayudar con la depuración.
  • Evitar usar pánico y recuperación para el flujo de control: No deben usarse como sustituto del manejo regular de errores.

Conclusión

Panic y recover son herramientas poderosas en Go para manejar errores graves y estados inesperados. Aunque deben usarse con moderación, entender cómo usarlos efectivamente puede ayudarle a escribir programas más robustos y resilientes. Recuerde usar pánico solo para errores críticos y recuperar en un nivel alto para asegurarse de que su programa pueda manejar situaciones inesperadas de manera elegante.
