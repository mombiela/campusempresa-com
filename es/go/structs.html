Introducción a las Estructuras

Las estructuras en Go son tipos de datos compuestos que agrupan variables bajo un solo nombre. Estas variables, conocidas como campos, pueden ser de diferentes tipos. Las estructuras son útiles para crear estructuras de datos más complejas.

Conceptos Clave

  • Definición: Una estructura se define usando las palabras clave type y struct.
  • Campos: Variables dentro de una estructura, cada una con un nombre y tipo.
  • Instanciación: Creación de una instancia de una estructura.
  • Acceso a Campos: Uso de la notación de punto para acceder o modificar campos.
  • Campos Anónimos: Campos sin nombre, lo que permite definiciones más concisas.
  • Estructuras Embebidas: Estructuras dentro de estructuras, lo que permite la composición.

Definición y Uso de Estructuras

Definición Básica de una Estructura

Para definir una estructura, se usa la palabra clave type seguida del nombre de la estructura y la palabra clave struct.

type Person struct {
    Name string
    Age  int
}

Instanciación de una Estructura

Puedes crear una instancia de una estructura usando el nombre de la estructura seguido de llaves.

func main() {
    var p1 Person
    p1.Name = "Alice"
    p1.Age = 30

    p2 := Person{Name: "Bob", Age: 25}
    p3 := Person{"Charlie", 35} // Los nombres de los campos pueden omitirse si los valores están en orden

    fmt.Println(p1)
    fmt.Println(p2)
    fmt.Println(p3)
}

Acceso y Modificación de Campos

Los campos de una estructura pueden ser accedidos y modificados usando la notación de punto.

func main() {
    p := Person{Name: "Alice", Age: 30}
    fmt.Println(p.Name) // Salida: Alice
    p.Age = 31
    fmt.Println(p.Age)  // Salida: 31
}

Características Avanzadas de las Estructuras

Campos Anónimos

Los campos anónimos te permiten definir una estructura sin nombrar explícitamente los campos.

type Point struct {
    int
    int
}

func main() {
    p := Point{1, 2}
    fmt.Println(p.int) // Error: selector ambiguo p.int
    fmt.Println(p.int1) // Salida: 1
    fmt.Println(p.int2) // Salida: 2
}

Estructuras Embebidas

Las estructuras pueden ser embebidas dentro de otras estructuras, permitiendo la composición y reutilización de campos.

type Address struct {
    City, State string
}

type Employee struct {
    Name    string
    Age     int
    Address // Estructura embebida
}

func main() {
    e := Employee{
        Name: "John",
        Age:  28,
        Address: Address{
            City:  "New York",
            State: "NY",
        },
    }

    fmt.Println(e.Name)    // Salida: John
    fmt.Println(e.City)    // Salida: New York
    fmt.Println(e.Address) // Salida: {New York NY}
}

Métodos de Estructuras

Los métodos pueden ser asociados con estructuras, permitiéndote definir comportamientos para tus tipos de datos.

type Rectangle struct {
    Width, Height int
}

// Método con receptor de valor
func (r Rectangle) Area() int {
    return r.Width * r.Height
}

// Método con receptor de puntero
func (r *Rectangle) Scale(factor int) {
    r.Width *= factor
    r.Height *= factor
}

func main() {
    rect := Rectangle{Width: 10, Height: 5}
    fmt.Println(rect.Area()) // Salida: 50

    rect.Scale(2)
    fmt.Println(rect.Area()) // Salida: 200
}

Conclusión

Las estructuras en Go proporcionan una forma poderosa de agrupar datos relacionados y definir estructuras de datos complejas. Al entender cómo definir, instanciar y manipular estructuras, así como cómo usar características avanzadas como campos anónimos y estructuras embebidas, puedes escribir código Go más organizado y mantenible. Los métodos de estructuras mejoran aún más las capacidades de las estructuras al permitirte definir comportamientos asociados con tus tipos de datos.
