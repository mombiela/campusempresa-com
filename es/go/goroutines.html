Las goroutines son un concepto fundamental en Go que te permiten ejecutar funciones concurrentemente. Son ligeras y gestionadas por el runtime de Go, lo que las convierte en una forma eficiente de manejar múltiples tareas simultáneamente.

¿Qué es una Goroutine?

  • Una goroutine es una función que es capaz de ejecutarse concurrentemente con otras funciones.
  • Las goroutines son gestionadas por el runtime de Go, que maneja su planificación y ejecución.
  • Son mucho más ligeras que los hilos tradicionales, permitiéndote ejecutar miles de goroutines sin una sobrecarga significativa.

Creando una Goroutine

Para crear una goroutine, simplemente usas la palabra clave go seguida de la llamada a la función.

Ejemplo

package main

import (
    "fmt"
    "time"
)

func sayHello() {
    fmt.Println("¡Hola, Mundo!")
}

func main() {
    go sayHello() // Esto inicia una nueva goroutine
    time.Sleep(1 * time.Second) // Espera a que la goroutine termine
}

En este ejemplo:

  • go sayHello() inicia una nueva goroutine que ejecuta la función sayHello.
  • time.Sleep(1 * time.Second) se usa para darle tiempo a la goroutine para completar antes de que la función principal termine.

Sincronización

Las goroutines se ejecutan concurrentemente, lo que significa que pueden ejecutarse al mismo tiempo. Esto puede llevar a condiciones de carrera si acceden a recursos compartidos. Para manejar esto, Go proporciona varios mecanismos de sincronización.

WaitGroup

Un WaitGroup espera a que un conjunto de goroutines terminen.

Ejemplo

package main

import (
    "fmt"
    "sync"
)

func sayHello(wg *sync.WaitGroup) {
    defer wg.Done() // Notifica al WaitGroup que esta goroutine ha terminado
    fmt.Println("¡Hola, Mundo!")
}

func main() {
    var wg sync.WaitGroup
    wg.Add(1) // Añade una goroutine al WaitGroup
    go sayHello(&wg)
    wg.Wait() // Espera a que todas las goroutines en el WaitGroup terminen
}

En este ejemplo:

  • wg.Add(1) incrementa el contador del WaitGroup.
  • wg.Done() decrementa el contador cuando la goroutine termina.
  • wg.Wait() bloquea hasta que el contador sea cero.

Canales

Los canales proporcionan una forma para que las goroutines se comuniquen entre sí y sincronicen su ejecución.

Ejemplo

package main

import (
    "fmt"
)

func sayHello(ch chan string) {
    ch <- "¡Hola, Mundo!" // Envía un mensaje al canal
}

func main() {
    ch := make(chan string)
    go sayHello(ch)
    msg := <-ch // Recibe el mensaje del canal
    fmt.Println(msg)
}

En este ejemplo:

  • ch := make(chan string) crea un nuevo canal.
  • ch <- "¡Hola, Mundo!" envía un mensaje al canal.
  • msg := <-ch recibe el mensaje del canal.

Canales con Búfer

Los canales con búfer te permiten especificar la capacidad del canal, que puede almacenar múltiples valores antes de bloquearse.

Ejemplo

package main

import (
    "fmt"
)

func main() {
    ch := make(chan string, 2) // Crea un canal con búfer con capacidad 2
    ch <- "Hola"
    ch <- "Mundo"
    fmt.Println(<-ch) // Recibe el primer mensaje
    fmt.Println(<-ch) // Recibe el segundo mensaje
}

En este ejemplo:

  • ch := make(chan string, 2) crea un canal con búfer con una capacidad de 2.
  • El canal puede almacenar dos mensajes antes de bloquearse.

Declaración Select

La declaración select te permite esperar en múltiples operaciones de canal.

Ejemplo

package main

import (
    "fmt"
    "time"
)

func main() {
    ch1 := make(chan string)
    ch2 := make(chan string)

    go func() {
        time.Sleep(1 * time.Second)
        ch1 <- "uno"
    }()

    go func() {
        time.Sleep(2 * time.Second)
        ch2 <- "dos"
    }()

    for i := 0; i < 2; i++ {
        select {
        case msg1 := <-ch1:
            fmt.Println("Recibido", msg1)
        case msg2 := <-ch2:
            fmt.Println("Recibido", msg2)
        }
    }
}

En este ejemplo:

  • select espera a que ch1 o ch2 reciban un mensaje.
  • El primer mensaje recibido se imprime, y el bucle continúa hasta que ambos mensajes sean recibidos.

Conclusión

Las goroutines son una característica poderosa en Go que permiten la programación concurrente con facilidad. Al entender cómo crear y gestionar goroutines, usar mecanismos de sincronización como WaitGroup y canales, y manejar múltiples operaciones de canal con la declaración select, puedes escribir programas en Go eficientes y concurrentes.
