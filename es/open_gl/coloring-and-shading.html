Introducción al Coloreado en OpenGL

El coloreado es uno de los aspectos fundamentales del renderizado en OpenGL. Involucra asignar colores a los vértices, los cuales luego son interpolados a través de la superficie de las primitivas (triángulos, líneas, etc.).

Conceptos Clave

  • Colores de Vértice: Colores asignados a cada vértice.
  • Colores de Fragmento: Colores calculados para cada píxel (fragmento) durante la rasterización.
  • Interpolación: El proceso de mezclar colores de vértices a través de la superficie de una primitiva.

Ejemplo Básico de Coloreado

Aquí hay un ejemplo simple de cómo asignar colores a los vértices en OpenGL:

// Vertex Shader
#version 330 core
layout(location = 0) in vec3 aPos;
layout(location = 1) in vec3 aColor;

out vec3 ourColor;

void main()
{
    gl_Position = vec4(aPos, 1.0);
    ourColor = aColor;
}

// Fragment Shader
#version 330 core
in vec3 ourColor;
out vec4 FragColor;

void main()
{
    FragColor = vec4(ourColor, 1.0);
}

En este ejemplo:

  • Las posiciones y colores de los vértices se pasan al vertex shader.
  • El vertex shader produce el color para ser interpolado a través de la primitiva.
  • El fragment shader recibe el color interpolado y lo establece como el color de salida.

Introducción al Sombreado en OpenGL

El sombreado se refiere al proceso de calcular el color de cada píxel basado en la iluminación, las propiedades del material y otros factores.

Conceptos Clave

  • Sombreado Plano: Cada primitiva se sombrea con un solo color.
  • Sombreado Gouraud: Los colores se calculan en los vértices y se interpolan a través de la primitiva.
  • Sombreado Phong: Los colores se calculan por píxel, proporcionando efectos de iluminación más realistas.

Ejemplo Básico de Sombreado

Aquí hay un ejemplo simple de sombreado Phong en OpenGL:

// Vertex Shader
#version 330 core
layout(location = 0) in vec3 aPos;
layout(location = 1) in vec3 aNormal;

out vec3 FragPos;
out vec3 Normal;

uniform mat4 model;
uniform mat4 view;
uniform mat4 projection;

void main()
{
    FragPos = vec3(model * vec4(aPos, 1.0));
    Normal = mat3(transpose(inverse(model))) * aNormal;  
    gl_Position = projection * view * vec4(FragPos, 1.0);
}

// Fragment Shader
#version 330 core
in vec3 FragPos;
in vec3 Normal;

out vec4 FragColor;

uniform vec3 lightPos;
uniform vec3 viewPos;
uniform vec3 lightColor;
uniform vec3 objectColor;

void main()
{
    // Ambiente
    float ambientStrength = 0.1;
    vec3 ambient = ambientStrength * lightColor;
    
    // Difuso 
    vec3 norm = normalize(Normal);
    vec3 lightDir = normalize(lightPos - FragPos);
    float diff = max(dot(norm, lightDir), 0.0);
    vec3 diffuse = diff * lightColor;
    
    // Especular
    float specularStrength = 0.5;
    vec3 viewDir = normalize(viewPos - FragPos);
    vec3 reflectDir = reflect(-lightDir, norm);  
    float spec = pow(max(dot(viewDir, reflectDir), 0.0), 32);
    vec3 specular = specularStrength * spec * lightColor;  
    
    vec3 result = (ambient + diffuse + specular) * objectColor;
    FragColor = vec4(result, 1.0);
}

En este ejemplo:

  • El vertex shader calcula la posición y la normal de cada vértice.
  • El fragment shader calcula los componentes de iluminación ambiente, difusa y especular para determinar el color final de cada píxel.

Técnicas Avanzadas de Sombreado

A medida que progreses, puedes explorar técnicas de sombreado más avanzadas como:

  • Normal Mapping: Añade detalle a las superficies sin aumentar la complejidad de la geometría.
  • Parallax Mapping: Mejora la ilusión de profundidad en superficies texturizadas.
  • Renderizado Basado en Física (PBR): Simula las propiedades físicas de los materiales para un renderizado más realista.

Ejemplo de Normal Mapping

Aquí hay un breve ejemplo de normal mapping:

// Vertex Shader
#version 330 core
layout(location = 0) in vec3 aPos;
layout(location = 1) in vec3 aNormal;
layout(location = 2) in vec2 aTexCoords;
layout(location = 3) in vec3 aTangent;

out vec2 TexCoords;
out vec3 FragPos;
out mat3 TBN;

uniform mat4 model;
uniform mat4 view;
uniform mat4 projection;

void main()
{
    FragPos = vec3(model * vec4(aPos, 1.0));
    TexCoords = aTexCoords;
    
    vec3 T = normalize(mat3(model) * aTangent);
    vec3 N = normalize(mat3(model) * aNormal);
    vec3 B = cross(N, T);
    TBN = mat3(T, B, N);
    
    gl_Position = projection * view * vec4(FragPos, 1.0);
}

// Fragment Shader
#version 330 core
in vec2 TexCoords;
in vec3 FragPos;
in mat3 TBN;

out vec4 FragColor;

uniform sampler2D normalMap;
uniform vec3 lightPos;
uniform vec3 viewPos;
uniform vec3 lightColor;
uniform vec3 objectColor;

void main()
{
    vec3 normal = texture(normalMap, TexCoords).rgb;
    normal = normalize(normal * 2.0 - 1.0);
    normal = normalize(TBN * normal);
    
    // Cálculos de iluminación (similares al sombreado Phong)
    // ...
}

En este ejemplo:

  • El vertex shader calcula la matriz TBN para transformar normales del espacio tangente al espacio mundial.
  • El fragment shader usa el normal map para perturbar las normales de la superficie para efectos de iluminación más detallados.

Conclusión

El coloreado y el sombreado son técnicas esenciales en OpenGL para crear gráficos visualmente atractivos. Comenzando desde el coloreado básico de vértices, puedes progresar a técnicas de sombreado más avanzadas como el sombreado Phong y el normal mapping. Dominar estas técnicas mejorará significativamente el realismo y la calidad visual de tus aplicaciones OpenGL.
