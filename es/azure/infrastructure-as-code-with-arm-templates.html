Introducción a la Infraestructura como Código (IaC)

  • Definición: La Infraestructura como Código (IaC) es el proceso de gestionar y aprovisionar la infraestructura informática a través de archivos de definición legibles por máquina, en lugar de la configuración de hardware físico o herramientas de configuración interactivas.
  • Beneficios:
    • Consistencia y repetibilidad
    • Control de versiones
    • Automatización y eficiencia
    • Reducción del riesgo de error humano

Introducción a las Plantillas ARM

  • Definición: Las Plantillas de Azure Resource Manager (ARM) son archivos JSON que definen la infraestructura y configuración para tu solución en Azure.
  • Conceptos Clave:
    • Recursos: Los componentes individuales que deseas desplegar (por ejemplo, máquinas virtuales, cuentas de almacenamiento).
    • Parámetros: Valores que puedes pasar a la plantilla para personalizar el despliegue.
    • Variables: Valores que se utilizan dentro de la plantilla para simplificar expresiones complejas.
    • Salidas: Valores que se devuelven después del despliegue.
    • Funciones: Métodos predefinidos para simplificar las expresiones de la plantilla.

Estructura Básica de una Plantilla ARM

  • Estructura de la Plantilla: 
    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {},
  "variables": {},
  "resources": [],
  "outputs": {}
}

Creando Tu Primera Plantilla ARM

  • Ejemplo: Desplegando una Cuenta de Almacenamiento Simple 
    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Storage/storageAccounts",
      "apiVersion": "2021-04-01",
      "name": "mystorageaccount",
      "location": "[resourceGroup().location]",
      "sku": {
        "name": "Standard_LRS"
      },
      "kind": "StorageV2",
      "properties": {}
    }
  ]
}
  • Explicación:
    • type: El tipo de recurso.
    • apiVersion: La versión de la API a utilizar para este recurso.
    • name: El nombre de la cuenta de almacenamiento.
    • location: La ubicación donde se desplegará el recurso.
    • sku: El SKU de la cuenta de almacenamiento.
    • kind: El tipo de cuenta de almacenamiento.
    • properties: Propiedades adicionales para el recurso.

Parámetros y Variables

  • Uso de Parámetros: 
    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {
    "storageAccountType": {
      "type": "string",
      "defaultValue": "Standard_LRS",
      "allowedValues": [
        "Standard_LRS",
        "Standard_GRS",
        "Standard_ZRS",
        "Premium_LRS"
      ],
      "metadata": {
        "description": "El tipo de cuenta de almacenamiento a crear."
      }
    }
  },
  "resources": [
    {
      "type": "Microsoft.Storage/storageAccounts",
      "apiVersion": "2021-04-01",
      "name": "mystorageaccount",
      "location": "[resourceGroup().location]",
      "sku": {
        "name": "[parameters('storageAccountType')]"
      },
      "kind": "StorageV2",
      "properties": {}
    }
  ]
}
  • Explicación:
    • parameters: Define los parámetros que se pueden pasar a la plantilla.
    • defaultValue: El valor predeterminado si no se proporciona ninguno.
    • allowedValues: El conjunto de valores permitidos para el parámetro.
    • metadata: Información adicional sobre el parámetro.

Características Avanzadas de las Plantillas ARM

  • Plantillas Anidadas: Uso de plantillas vinculadas para modularizar tu despliegue.
  • Condiciones: Desplegar recursos condicionalmente basado en valores de parámetros.
  • Bucles: Desplegar múltiples instancias de un recurso utilizando bucles.

Ejemplo: Plantillas Anidadas

  • Plantilla Principal: 
    {
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Resources/deployments",
      "apiVersion": "2021-04-01",
      "name": "nestedTemplate",
      "properties": {
        "mode": "Incremental",
        "templateLink": {
          "uri": "https://example.com/nestedTemplate.json",
          "contentVersion": "1.0.0.0"
        }
      }
    }
  ]
}
  • Explicación:
    • templateLink: Especifica el URI de la plantilla anidada.

Mejores Prácticas para Plantillas ARM

  • Modularización: Divide plantillas grandes en plantillas más pequeñas y reutilizables.
  • Parametrización: Usa parámetros para hacer las plantillas flexibles y reutilizables.
  • Control de Versiones: Almacena las plantillas en un sistema de control de versiones como Git.
  • Documentación: Documenta tus plantillas para facilitar su mantenimiento y comprensión.

Conclusión

Las Plantillas ARM proporcionan una forma poderosa de gestionar y automatizar tu infraestructura en Azure. Al comprender la estructura básica, usar parámetros y variables, y aprovechar características avanzadas como plantillas anidadas, puedes crear despliegues de infraestructura robustos y mantenibles. Siempre sigue las mejores prácticas para asegurar que tus plantillas sean eficientes, reutilizables y fáciles de gestionar.
