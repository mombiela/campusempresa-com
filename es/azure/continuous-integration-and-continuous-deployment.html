Introducción a CI/CD

La Integración Continua (CI) y el Despliegue Continuo (CD) son prácticas esenciales en el desarrollo de software moderno. Las canalizaciones de CI/CD automatizan el proceso de integración de cambios de código, ejecución de pruebas y despliegue de aplicaciones, asegurando una entrega de software más rápida y confiable.

Conceptos Clave de CI/CD

  • Integración Continua (CI): La práctica de integrar automáticamente los cambios de código de múltiples contribuyentes en un repositorio compartido varias veces al día.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio de código que pasa las pruebas automatizadas a producción.
  • Canalización: Un conjunto de procesos automatizados que impulsan el desarrollo de software a través de varias etapas, desde la integración del código hasta el despliegue.
  • Compilación: El proceso de compilar el código fuente en código ejecutable.
  • Prueba: Pruebas automatizadas para verificar la corrección del código.
  • Despliegue: El proceso de entregar la aplicación a un entorno de producción.

Configuración de CI/CD en Azure

Azure proporciona varias herramientas y servicios para implementar canalizaciones de CI/CD, incluyendo Azure DevOps y GitHub Actions.

Azure DevOps

Azure DevOps es un conjunto de herramientas de desarrollo que soportan todo el ciclo de vida del desarrollo de software.

Creación de una Canalización de CI

  1. Crear un nuevo proyecto en Azure DevOps.
  2. Configurar un repositorio:
    • Navegar a Repos > Archivos.
    • Inicializar el repositorio con tu código.
  3. Definir una canalización de compilación:
    • Navegar a Pipelines > Builds.
    • Hacer clic en "New pipeline" y seleccionar tu repositorio.
    • Elegir una plantilla o comenzar con un trabajo vacío.
    • Definir los pasos de compilación en el archivo YAML.
# azure-pipelines.yml
trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- script: dotnet build --configuration Release
  displayName: 'Build project'

- script: dotnet test --no-build --verbosity normal
  displayName: 'Run tests'

Creación de una Canalización de CD

  1. Definir una canalización de lanzamiento:
    • Navegar a Pipelines > Releases.
    • Hacer clic en "New pipeline" y seleccionar una plantilla.
    • Definir las etapas (por ejemplo, Dev, QA, Producción).
  2. Agregar artefactos:
    • Vincular la canalización de compilación como fuente de artefactos.
  3. Definir tareas de despliegue:
    • Agregar tareas para desplegar la aplicación en el entorno deseado (por ejemplo, Azure App Service).
# Ejemplo de tarea de despliegue en YAML
- task: AzureRmWebAppDeployment@4
  inputs:
    azureSubscription: 'your-azure-subscription'
    appType: 'webApp'
    WebAppName: 'your-app-service-name'
    packageForLinux: '$(System.DefaultWorkingDirectory)/**/*.zip'

GitHub Actions

GitHub Actions es una herramienta de automatización integrada con GitHub que permite crear flujos de trabajo de CI/CD.

Creación de un Flujo de Trabajo de CI

  1. Crear un directorio .github/workflows en tu repositorio.
  2. Agregar un archivo de flujo de trabajo:
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up .NET
      uses: actions/setup-dotnet@v1
      with:
        dotnet-version: '5.x'
    - name: Build with dotnet
      run: dotnet build --configuration Release
    - name: Test with dotnet
      run: dotnet test --no-build --verbosity normal

Creación de un Flujo de Trabajo de CD

  1. Agregar un archivo de flujo de trabajo de despliegue:
# .github/workflows/cd.yml
name: CD

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up .NET
      uses: actions/setup-dotnet@v1
      with:
        dotnet-version: '5.x'
    - name: Deploy to Azure Web App
      uses: azure/webapps-deploy@v2
      with:
        app-name: 'your-app-service-name'
        publish-profile: ${{ secrets.AZURE_WEBAPP_PUBLISH_PROFILE }}
        package: .

Prácticas Avanzadas de CI/CD

  • Variables de Entorno y Secretos: Gestionar de manera segura la información sensible utilizando Azure Key Vault o GitHub Secrets.
  • Canalizaciones Multi-Etapa: Implementar canalizaciones con múltiples etapas para diferentes entornos (por ejemplo, Dev, QA, Producción).
  • Monitoreo y Alertas: Integrar herramientas de monitoreo como Azure Monitor para rastrear la salud de tus despliegues y configurar alertas para fallos.
  • Estrategias de Reversión: Implementar estrategias para revertir despliegues en caso de fallos.

Conclusión

Implementar canalizaciones de CI/CD en Azure utilizando Azure DevOps o GitHub Actions puede mejorar significativamente tu proceso de desarrollo de software al automatizar la integración de código, pruebas y despliegue. Siguiendo las mejores prácticas y aprovechando el robusto conjunto de herramientas de Azure, puedes lograr una entrega de software más rápida y confiable.
