Azure Policy es un servicio en Azure que se utiliza para crear, asignar y gestionar políticas. Estas políticas imponen diferentes reglas y efectos sobre tus recursos, para que esos recursos se mantengan conformes con tus estándares corporativos y acuerdos de nivel de servicio. Azure Policy hace esto ejecutando evaluaciones de tus recursos y escaneando aquellos que no cumplen con las políticas que has creado.

Introducción a Azure Policy

  • Definición: Azure Policy es una herramienta de gobernanza que te permite imponer estándares organizacionales y evaluar el cumplimiento a gran escala.
  • Propósito: Asegurar que los recursos cumplan con los estándares corporativos y los requisitos regulatorios.
  • Características Clave:
    • Definiciones de Políticas
    • Iniciativas
    • Asignaciones
    • Cumplimiento

Definiciones de Políticas

Una definición de política expresa qué evaluar y qué acción tomar. Estas definiciones pueden ser creadas desde cero o utilizando políticas integradas proporcionadas por Azure.

  • Estructura:
    • Modo: Especifica el alcance de la política (por ejemplo, Todos, Indexados).
    • Parámetros: Definen valores que pueden ser utilizados en la regla de la política.
    • Regla: La evaluación lógica que determina si un recurso es conforme.

Ejemplo de Definición de Política

{
  "properties": {
    "displayName": "Ubicaciones permitidas",
    "description": "Esta política te permite restringir las ubicaciones que tu organización puede especificar al desplegar recursos.",
    "parameters": {
      "allowedLocations": {
        "type": "Array",
        "metadata": {
          "description": "La lista de ubicaciones permitidas para los recursos.",
          "displayName": "Ubicaciones permitidas",
          "strongType": "location"
        }
      }
    },
    "policyRule": {
      "if": {
        "not": {
          "field": "location",
          "in": "[parameters('allowedLocations')]"
        }
      },
      "then": {
        "effect": "deny"
      }
    }
  }
}
  • Explicación:
    • displayName: Nombre de la política.
    • description: Descripción de lo que hace la política.
    • parameters: Define las ubicaciones permitidas.
    • policyRule: La regla verifica si la ubicación del recurso no está en las ubicaciones permitidas y niega el despliegue si no lo está.

Iniciativas

Una iniciativa es una colección de definiciones de políticas que se agrupan con un objetivo o propósito específico.

  • Propósito: Simplificar la gestión de múltiples políticas agrupándolas.
  • Estructura:
    • Definición de Iniciativa: Contiene múltiples definiciones de políticas.
    • Parámetros: Parámetros que pueden ser compartidos entre las políticas en la iniciativa.

Ejemplo de Definición de Iniciativa

{
  "properties": {
    "displayName": "Iniciativa de Seguridad",
    "description": "Asegura que los recursos cumplan con los estándares de seguridad.",
    "parameters": {},
    "policyDefinitions": [
      {
        "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/allowedLocations",
        "parameters": {
          "allowedLocations": {
            "value": ["eastus", "westus"]
          }
        }
      },
      {
        "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/requireTag",
        "parameters": {
          "tagName": {
            "value": "Environment"
          }
        }
      }
    ]
  }
}
  • Explicación:
    • displayName: Nombre de la iniciativa.
    • description: Descripción del propósito de la iniciativa.
    • policyDefinitions: Lista de definiciones de políticas incluidas en la iniciativa.

Asignaciones

Las asignaciones se utilizan para aplicar una política o iniciativa a un alcance específico, como una suscripción, grupo de recursos o recurso individual.

  • Alcance: Define dónde se aplica la política o iniciativa.
  • Parámetros: Valores proporcionados para los parámetros definidos en la política o iniciativa.

Ejemplo de Asignación

{
  "properties": {
    "displayName": "Imponer ubicaciones permitidas",
    "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/allowedLocations",
    "scope": "/subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}",
    "parameters": {
      "allowedLocations": {
        "value": ["eastus", "westus"]
      }
    }
  }
}
  • Explicación:
    • displayName: Nombre de la asignación.
    • policyDefinitionId: El ID de la definición de política que se está asignando.
    • scope: El alcance donde se aplica la política.
    • parameters: Valores para los parámetros definidos en la política.

Cumplimiento

El cumplimiento en Azure Policy se refiere al estado de los recursos en relación con las políticas asignadas a ellos.

  • Estado de Cumplimiento: Indica si los recursos son conformes o no conformes.
  • Informes de Cumplimiento: Proporcionan información detallada sobre el estado de cumplimiento de los recursos.

Visualización del Cumplimiento

Puedes ver los detalles de cumplimiento en el portal de Azure:

  1. Navega al servicio de Azure Policy.
  2. Selecciona la pestaña "Cumplimiento".
  3. Revisa el estado de cumplimiento de tus recursos.

Conclusión

Azure Policy es una herramienta poderosa para asegurar que tus recursos de Azure cumplan con los estándares organizacionales y los requisitos regulatorios. Al entender y utilizar definiciones de políticas, iniciativas, asignaciones e informes de cumplimiento, puedes gestionar y aplicar efectivamente la gobernanza en todo tu entorno de Azure.
