Visión General

La Red de Azure es un componente crítico de Microsoft Azure, proporcionando la infraestructura y los servicios necesarios para conectar y gestionar tus recursos en la nube. Esta sección te introducirá a los conceptos y servicios fundamentales dentro de la Red de Azure, ayudándote a entender cómo diseñar e implementar una arquitectura de red robusta en Azure.

Conceptos Clave

  • Redes Virtuales (VNets): La base de la red de Azure, permitiéndote crear redes aisladas dentro de la nube de Azure.
  • Subredes: Segmentos dentro de una VNet que ayudan a organizar y asegurar los recursos.
  • Grupos de Seguridad de Red (NSGs): Reglas que controlan el tráfico entrante y saliente hacia las interfaces de red (NICs), VMs y subredes.
  • Azure Load Balancer: Distribuye el tráfico de red entrante entre múltiples VMs para asegurar alta disponibilidad y fiabilidad.
  • Azure VPN Gateway: Establece conectividad segura entre tus VNets de Azure y redes locales.
  • Azure ExpressRoute: Proporciona una conexión dedicada y privada entre tu infraestructura local y los centros de datos de Azure.
  • Azure DNS: Un servicio de alojamiento para dominios DNS, proporcionando resolución de nombres utilizando la infraestructura de Microsoft Azure.
  • Azure Traffic Manager: Un balanceador de carga basado en DNS que te permite distribuir el tráfico de manera óptima a servicios en regiones globales de Azure.

Redes Virtuales (VNets)

Creación de una Red Virtual

Una Red Virtual (VNet) es una representación de tu propia red en la nube. Es un aislamiento lógico de la nube de Azure dedicado a tu suscripción.

# Crear un grupo de recursos
az group create --name MyResourceGroup --location eastus

# Crear una red virtual
az network vnet create \
  --resource-group MyResourceGroup \
  --name MyVNet \
  --address-prefix 10.0.0.0/16 \
  --subnet-name MySubnet \
  --subnet-prefix 10.0.1.0/24

Puntos Clave

  • Las VNets pueden ser segmentadas en subredes.
  • Las VNets pueden ser conectadas a otras VNets o redes locales.
  • Las VNets soportan varios servicios de Azure como VMs, Azure Kubernetes Service (AKS) y más.

Subredes

Las subredes te permiten dividir tu VNet en secciones más pequeñas y manejables. Cada subred puede alojar diferentes servicios y aplicar diferentes reglas de seguridad.

# Añadir una subred a una VNet existente
az network vnet subnet create \
  --resource-group MyResourceGroup \
  --vnet-name MyVNet \
  --name MyAdditionalSubnet \
  --address-prefix 10.0.2.0/24

Puntos Clave

  • Las subredes ayudan a organizar y asegurar los recursos.
  • Cada subred puede tener sus propios Grupos de Seguridad de Red (NSGs).

Grupos de Seguridad de Red (NSGs)

Los NSGs contienen reglas de seguridad que permiten o deniegan el tráfico entrante y saliente hacia y desde los recursos.

# Crear un Grupo de Seguridad de Red
az network nsg create \
  --resource-group MyResourceGroup \
  --name MyNSG

# Crear una regla de seguridad
az network nsg rule create \
  --resource-group MyResourceGroup \
  --nsg-name MyNSG \
  --name AllowSSH \
  --protocol Tcp \
  --direction Inbound \
  --source-address-prefix '*' \
  --source-port-range '*' \
  --destination-address-prefix '*' \
  --destination-port-range 22 \
  --access Allow \
  --priority 1000

Puntos Clave

  • Los NSGs pueden ser asociados con subredes o interfaces de red individuales.
  • Las reglas son evaluadas en orden de prioridad, con números más bajos teniendo mayor prioridad.

Azure Load Balancer

Azure Load Balancer distribuye el tráfico entrante entre instancias saludables de servicios en tu VNet.

# Crear una dirección IP pública
az network public-ip create \
  --resource-group MyResourceGroup \
  --name MyPublicIP

# Crear un balanceador de carga
az network lb create \
  --resource-group MyResourceGroup \
  --name MyLoadBalancer \
  --public-ip-address MyPublicIP \
  --frontend-ip-name MyFrontEndPool \
  --backend-pool-name MyBackEndPool

Puntos Clave

  • Soporta balanceo de carga tanto público como interno.
  • Asegura alta disponibilidad y fiabilidad distribuyendo el tráfico.

Azure VPN Gateway

Azure VPN Gateway proporciona conectividad segura entre sitios.

# Crear una puerta de enlace de red virtual
az network vnet-gateway create \
  --resource-group MyResourceGroup \
  --name MyVPNGateway \
  --public-ip-address MyPublicIP \
  --vnet MyVNet \
  --gateway-type Vpn \
  --vpn-type RouteBased \
  --sku VpnGw1 \
  --no-wait

Puntos Clave

  • Soporta conexiones sitio a sitio, punto a sitio y VNet a VNet.
  • Proporciona conexiones seguras y encriptadas.

Azure ExpressRoute

Azure ExpressRoute ofrece una conexión privada a Azure, evitando el internet público.

Puntos Clave

  • Proporciona mayor seguridad, fiabilidad y velocidad en comparación con VPN.
  • Ideal para escenarios que requieren conectividad consistente y de alto rendimiento.

Azure DNS

Azure DNS te permite alojar tus dominios DNS en Azure.

# Crear una zona DNS
az network dns zone create \
  --resource-group MyResourceGroup \
  --name mydomain.com

# Crear un registro DNS
az network dns record-set a add-record \
  --resource-group MyResourceGroup \
  --zone-name mydomain.com \
  --record-set-name www \
  --ipv4-address 10.0.0.4

Puntos Clave

  • Proporciona servicios DNS rápidos y fiables.
  • Integrado con otros servicios de Azure.

Azure Traffic Manager

Azure Traffic Manager es un balanceador de carga basado en DNS.

# Crear un perfil de Traffic Manager
az network traffic-manager profile create \
  --resource-group MyResourceGroup \
  --name MyTrafficManagerProfile \
  --routing-method Performance \
  --unique-dns-name myapptrafficmanager

# Añadir un punto final al perfil de Traffic Manager
az network traffic-manager endpoint create \
  --resource-group MyResourceGroup \
  --profile-name MyTrafficManagerProfile \
  --name MyEndpoint \
  --type azureEndpoints \
  --target-resource-id /subscriptions/{subscription-id}/resourceGroups/MyResourceGroup/providers/Microsoft.Network/publicIPAddresses/MyPublicIP

Puntos Clave

  • Distribuye el tráfico a través de múltiples regiones.
  • Mejora la disponibilidad y capacidad de respuesta de las aplicaciones.

Conclusión

La Red de Azure proporciona una suite completa de servicios para construir, gestionar y asegurar tu infraestructura de red en la nube. Al entender y utilizar estos servicios, puedes crear arquitecturas de red escalables, fiables y seguras que satisfagan las necesidades de tu negocio. Ya sea conectando VNets, asegurando recursos con NSGs, o optimizando el tráfico con Load Balancers y Traffic Manager, la Red de Azure tiene las herramientas que necesitas para tener éxito.
