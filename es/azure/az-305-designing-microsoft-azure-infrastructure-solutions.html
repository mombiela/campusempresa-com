Introducción

Bienvenido al curso AZ-305 sobre Diseño de Soluciones de Infraestructura de Microsoft Azure. Este curso está diseñado para llevarte de un nivel principiante a un nivel avanzado en la comprensión e implementación de soluciones de infraestructura en Azure. Ya sea que seas nuevo en Azure o busques profundizar tu conocimiento, este curso te proporcionará las habilidades necesarias para diseñar y gestionar la infraestructura de Azure de manera efectiva.

Esquema del Curso

  1. Introducción a Azure
  2. Diseño para Seguridad y Cumplimiento
  3. Diseño para Rendimiento y Escalabilidad
  4. Diseño para Optimización de Costos
  5. Diseño para Operaciones y Monitoreo
  6. Temas Avanzados y Mejores Prácticas

1. Introducción a Azure

Conceptos Clave

  • Visión General de Azure: Comprender los conceptos básicos de Microsoft Azure, incluida su infraestructura global, servicios y modelos de precios.
  • Portal de Azure: Navegar por el Portal de Azure y usarlo para gestionar recursos.
  • Grupos de Recursos: Organizar recursos utilizando grupos de recursos.
  • CLI de Azure y PowerShell: Gestionar recursos de Azure utilizando herramientas de línea de comandos.

Ejemplo: Crear una Máquina Virtual

# Usando Azure CLI para crear una máquina virtual
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Explicación:

  • az vm create: Comando para crear una máquina virtual.
  • --resource-group: Especifica el grupo de recursos.
  • --name: Nombre de la máquina virtual.
  • --image: Especifica la imagen del sistema operativo.
  • --admin-username: Nombre de usuario administrador para la VM.
  • --generate-ssh-keys: Genera claves SSH para acceso seguro.

2. Diseño para Seguridad y Cumplimiento

Conceptos Clave

  • Gestión de Identidad y Acceso (IAM): Usar Azure Active Directory (AAD) para la gestión de identidades.
  • Seguridad de Red: Implementar Grupos de Seguridad de Red (NSGs) y Firewall de Azure.
  • Protección de Datos: Encriptar datos en reposo y en tránsito.
  • Cumplimiento: Asegurar el cumplimiento con estándares y regulaciones de la industria.

Ejemplo: Configurar un Grupo de Seguridad de Red

# Crear un Grupo de Seguridad de Red
az network nsg create \
  --resource-group myResourceGroup \
  --name myNetworkSecurityGroup

# Crear una regla de seguridad
az network nsg rule create \
  --resource-group myResourceGroup \
  --nsg-name myNetworkSecurityGroup \
  --name AllowSSH \
  --protocol tcp \
  --priority 1000 \
  --destination-port-range 22 \
  --access allow

Explicación:

  • az network nsg create: Comando para crear un Grupo de Seguridad de Red.
  • az network nsg rule create: Comando para crear una regla de seguridad dentro del NSG.
  • --protocol: Especifica el protocolo (TCP).
  • --priority: Prioridad de la regla.
  • --destination-port-range: Rango de puertos (22 para SSH).
  • --access: Permitir o denegar tráfico.

3. Diseño para Rendimiento y Escalabilidad

Conceptos Clave

  • Escalabilidad: Comprender la escalabilidad vertical y horizontal.
  • Balanceo de Carga: Implementar Azure Load Balancer y Application Gateway.
  • Caché: Usar Azure Cache para Redis.
  • Red de Entrega de Contenido (CDN): Distribuir contenido globalmente con Azure CDN.

Ejemplo: Configurar un Balanceador de Carga

# Crear una dirección IP pública
az network public-ip create \
  --resource-group myResourceGroup \
  --name myPublicIP

# Crear un balanceador de carga
az network lb create \
  --resource-group myResourceGroup \
  --name myLoadBalancer \
  --public-ip-address myPublicIP \
  --frontend-ip-name myFrontEndPool \
  --backend-pool-name myBackEndPool

Explicación:

  • az network public-ip create: Comando para crear una dirección IP pública.
  • az network lb create: Comando para crear un balanceador de carga.
  • --frontend-ip-name: Nombre del grupo de IPs frontend.
  • --backend-pool-name: Nombre del grupo de IPs backend.

4. Diseño para Optimización de Costos

Conceptos Clave

  • Gestión de Costos: Usar Azure Cost Management y Billing.
  • Instancias Reservadas: Ahorrar costos con instancias reservadas.
  • Calculadora de Precios de Azure: Estimar costos con la Calculadora de Precios de Azure.
  • Monitoreo y Alertas: Configurar alertas de costos.

Ejemplo: Usar la Calculadora de Precios de Azure

  • Visita la Calculadora de Precios de Azure.
  • Selecciona los servicios que planeas usar.
  • Configura las opciones para cada servicio.
  • Revisa el costo estimado y ajusta según sea necesario.

5. Diseño para Operaciones y Monitoreo

Conceptos Clave

  • Azure Monitor: Monitorear recursos con Azure Monitor.
  • Log Analytics: Recopilar y analizar logs.
  • Automatización de Azure: Automatizar tareas con Azure Automation.
  • Respaldo y Recuperación ante Desastres: Implementar Azure Backup y Site Recovery.

Ejemplo: Configurar Azure Monitor

# Crear un espacio de trabajo de Log Analytics
az monitor log-analytics workspace create \
  --resource-group myResourceGroup \
  --workspace-name myWorkspace

# Habilitar monitoreo para una VM
az monitor diagnostic-settings create \
  --resource myVM \
  --resource-group myResourceGroup \
  --workspace myWorkspace \
  --name myDiagnosticSetting \
  --metrics '[{"category": "AllMetrics", "enabled": true}]' \
  --logs '[{"category": "Syslog", "enabled": true}]'

Explicación:

  • az monitor log-analytics workspace create: Comando para crear un espacio de trabajo de Log Analytics.
  • az monitor diagnostic-settings create: Comando para habilitar monitoreo para una VM.
  • --metrics: Especifica las métricas a recopilar.
  • --logs: Especifica los logs a recopilar.

6. Temas Avanzados y Mejores Prácticas

Conceptos Clave

  • Soluciones de Nube Híbrida: Integrar recursos locales y en la nube.
  • DevOps y CI/CD: Implementar prácticas de DevOps con Azure DevOps.
  • Redes Avanzadas: Usar Azure Virtual WAN y ExpressRoute.
  • Mejores Prácticas de Seguridad: Implementar medidas de seguridad avanzadas.

Ejemplo: Configurar una Canalización CI/CD con Azure DevOps

# azure-pipelines.yml
trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- script: dotnet build
  displayName: 'Build project'

- script: dotnet test
  displayName: 'Run tests'

Explicación:

  • trigger: Especifica la rama para activar la canalización.
  • pool: Especifica la imagen de VM a usar.
  • steps: Define los pasos en la canalización.
  • UseDotNet@2: Tarea para instalar el SDK de .NET.
  • dotnet build: Comando para construir el proyecto.
  • dotnet test: Comando para ejecutar pruebas.

Conclusión

En este curso, hemos cubierto los aspectos esenciales del diseño de soluciones de infraestructura de Microsoft Azure, desde conceptos básicos hasta prácticas avanzadas. Al comprender y aplicar estos principios, estarás bien equipado para diseñar, implementar y gestionar la infraestructura de Azure de manera efectiva. Recuerda explorar continuamente y mantenerte actualizado con los servicios en evolución de Azure y las mejores prácticas para asegurar un rendimiento óptimo, seguridad y eficiencia de costos en tus soluciones.
