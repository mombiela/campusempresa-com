Introducción

El monitoreo y los diagnósticos son aspectos cruciales de la gestión de aplicaciones y servicios en Azure. Este tema te guiará a través de los conceptos básicos de monitoreo y diagnósticos, progresando a técnicas avanzadas y herramientas disponibles en Azure.

Conceptos Clave

  • Monitoreo: El proceso de recopilar, analizar y usar información para rastrear el rendimiento y la salud de aplicaciones y servicios.
  • Diagnósticos: El proceso de identificar y resolver problemas dentro de aplicaciones y servicios.

Nivel Principiante

Azure Monitor

Azure Monitor es una solución integral para recopilar, analizar y actuar sobre la telemetría de tus entornos en la nube y locales.

  • Métricas: Datos numéricos que describen el rendimiento de tus recursos.
  • Registros: Registros detallados de eventos y actividades.

Configuración de Azure Monitor

  1. Crear un Espacio de Trabajo de Log Analytics:

    az monitor log-analytics workspace create --resource-group MyResourceGroup --workspace-name MyWorkspace

  2. Habilitar Configuraciones de Diagnóstico:

    az monitor diagnostic-settings create --resource MyResource --resource-group MyResourceGroup --workspace MyWorkspace --name MyDiagnosticSetting --logs '[{"category": "Administrative", "enabled": true}]'

Application Insights

Application Insights es un servicio extensible de Gestión del Rendimiento de Aplicaciones (APM) para desarrolladores web.

  • Datos de Telemetría: Incluye tasas de solicitudes, tiempos de respuesta y tasas de fallos.
  • Mapa de Aplicaciones: Representación visual de los componentes de tu aplicación.

Configuración de Application Insights

  1. Crear un Recurso de Application Insights:

    az monitor app-insights component create --app MyApp --location eastus --resource-group MyResourceGroup --application-type web

  2. Agregar la Clave de Instrumentación a Tu Aplicación:

    // En el código de tu aplicación
var telemetryClient = new TelemetryClient(new TelemetryConfiguration("your_instrumentation_key"));
telemetryClient.TrackEvent("Application Started");

Nivel Intermedio

Log Analytics

Log Analytics es una herramienta en el portal de Azure utilizada para editar y ejecutar consultas de registros a partir de los datos recopilados por Azure Monitor.

  • Lenguaje de Consulta Kusto (KQL): Utilizado para escribir consultas de registros.
  • Libros de Trabajo: Informes personalizables que visualizan datos de registros.

Escribir Consultas Básicas en KQL

// Obtener todos los registros de las últimas 24 horas
AzureActivity
| where TimeGenerated > ago(24h)

Alertas

Las alertas te notifican sobre condiciones críticas y potencialmente toman acciones correctivas.

  • Alertas de Métricas: Se activan cuando una métrica cruza un umbral.
  • Alertas de Registros: Se activan en base a los resultados de consultas de registros.

Crear una Alerta de Métrica

  1. Crear una Regla de Alerta: 
    az monitor metrics alert create --name MyAlertRule --resource-group MyResourceGroup --scopes /subscriptions/{subscription-id}/resourceGroups/{resource-group}/providers/Microsoft.Compute/virtualMachines/{vm-name} --condition "avg Percentage CPU > 80" --window-size 5m --evaluation-frequency 1m --action-group MyActionGroup

Nivel Avanzado

Diagnósticos Avanzados con Azure Monitor

  • Métricas Personalizadas: Definir y recopilar métricas personalizadas para tus aplicaciones.
  • Rastreo Distribuido: Rastrear el flujo de solicitudes a través de servicios.

Implementación de Métricas Personalizadas

  1. Definir Métricas Personalizadas: 
    var telemetryClient = new TelemetryClient();
telemetryClient.GetMetric("CustomMetric").TrackValue(42);

Monitoreo de Red

Azure Network Watcher proporciona herramientas para monitorear, diagnosticar y obtener información sobre el rendimiento y la salud de tu red.

  • Monitor de Conexión: Monitorea la conectividad entre recursos de Azure.
  • Monitor de Rendimiento de Red: Monitorea el rendimiento de la red y detecta problemas de red.

Configuración de Network Watcher

  1. Habilitar Network Watcher:

    az network watcher configure --locations eastus --resource-group NetworkWatcherRG --enabled

  2. Crear un Monitor de Conexión:

    az network watcher connection-monitor create --name MyConnectionMonitor --resource-group NetworkWatcherRG --location eastus --source-resource /subscriptions/{subscription-id}/resourceGroups/{resource-group}/providers/Microsoft.Compute/virtualMachines/{vm-name} --dest-address www.example.com --dest-port 80

Conclusión

El monitoreo y los diagnósticos en Azure son esenciales para mantener la salud y el rendimiento de tus aplicaciones y servicios. Comenzando con herramientas básicas como Azure Monitor y Application Insights, puedes progresar a técnicas más avanzadas como métricas personalizadas y monitoreo de red. Al dominar estas herramientas, puedes asegurar que tus aplicaciones funcionen sin problemas y de manera eficiente, y resolver rápidamente cualquier problema que surja.
