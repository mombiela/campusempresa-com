Introducción al Almacenamiento de Blobs

Azure Blob Storage es la solución de almacenamiento de objetos en la nube de Microsoft. El almacenamiento de blobs está optimizado para almacenar grandes cantidades de datos no estructurados, como texto o datos binarios.

Conceptos Clave

  • Blob: Un objeto binario grande que puede almacenar texto o datos binarios.
  • Contenedor: Un contenedor organiza un conjunto de blobs, similar a un directorio en un sistema de archivos.
  • Cuenta de Almacenamiento: Una cuenta de almacenamiento proporciona un espacio de nombres único en Azure para tus datos.

Tipos de Blobs

  • Blobs de Bloques: Utilizados para almacenar texto y datos binarios, ideales para archivos como documentos y archivos multimedia.
  • Blobs de Anexos: Optimizados para operaciones de anexado, comúnmente utilizados para registros.
  • Blobs de Páginas: Utilizados para operaciones de lectura/escritura aleatorias, adecuados para archivos de disco duro virtual (VHD).

Configuración del Almacenamiento de Blobs

Creación de una Cuenta de Almacenamiento

  1. Navegar al Portal de Azure: Ve al Portal de Azure.
  2. Crear un nuevo recurso: Haz clic en "Crear un recurso" y selecciona "Cuenta de almacenamiento".
  3. Configurar la cuenta de almacenamiento:
    • Suscripción: Elige tu suscripción.
    • Grupo de recursos: Crea un nuevo grupo de recursos o usa uno existente.
    • Nombre de la cuenta de almacenamiento: Proporciona un nombre único.
    • Región: Selecciona la región más cercana a tus usuarios.
    • Rendimiento: Elige entre Estándar y Premium.
    • Replicación: Selecciona la estrategia de replicación (LRS, GRS, RA-GRS, etc.).

Creación de un Contenedor

  1. Navegar a tu cuenta de almacenamiento: En el Portal de Azure, ve a tu cuenta de almacenamiento.
  2. Crear un contenedor: En la sección "Servicio de blobs", haz clic en "Contenedores" y luego en "+ Contenedor".
  3. Configurar el contenedor:
    • Nombre: Proporciona un nombre para tu contenedor.
    • Nivel de acceso público: Elige el nivel de acceso (Privado, Blob, Contenedor).

Trabajando con Blobs

Carga de Blobs

Puedes cargar blobs usando el Portal de Azure, Azure CLI o SDKs.

Usando el Portal de Azure

  1. Navegar a tu contenedor: En el Portal de Azure, ve a tu contenedor.
  2. Cargar un blob: Haz clic en "Cargar" y selecciona el archivo que deseas cargar.

Usando Azure CLI

# Cargar un archivo a un contenedor
az storage blob upload --account-name <nombre_cuenta_almacenamiento> --container-name <nombre_contenedor> --name <nombre_blob> --file <ruta_archivo>

Usando Azure SDK para Python

from azure.storage.blob import BlobServiceClient

# Inicializar una conexión al servicio de Blob
blob_service_client = BlobServiceClient.from_connection_string("<tu_cadena_conexion>")

# Obtener una referencia a un contenedor
container_client = blob_service_client.get_container_client("<nombre_contenedor>")

# Cargar un blob
with open("<ruta_archivo>", "rb") as data:
    container_client.upload_blob(name="<nombre_blob>", data=data)

Descarga de Blobs

Usando Azure CLI

# Descargar un blob de un contenedor
az storage blob download --account-name <nombre_cuenta_almacenamiento> --container-name <nombre_contenedor> --name <nombre_blob> --file <ruta_descarga>

Usando Azure SDK para Python

# Descargar un blob
blob_client = container_client.get_blob_client("<nombre_blob>")
with open("<ruta_descarga>", "wb") as download_file:
    download_file.write(blob_client.download_blob().readall())

Características Avanzadas del Almacenamiento de Blobs

Niveles de Almacenamiento de Blobs

Azure Blob Storage ofrece diferentes niveles de acceso para optimizar costos según los patrones de acceso a los datos:

  • Caliente: Optimizado para datos que se acceden con frecuencia.
  • Frío: Optimizado para datos que se acceden con poca frecuencia y se almacenan durante al menos 30 días.
  • Archivo: Optimizado para datos que rara vez se acceden y se almacenan durante al menos 180 días.

Gestión del Ciclo de Vida

Azure Blob Storage admite políticas de gestión del ciclo de vida para automatizar la transición de blobs entre niveles de acceso y la eliminación de blobs.

Ejemplo de Política

{
  "rules": [
    {
      "name": "move-to-cool",
      "enabled": true,
      "definition": {
        "actions": {
          "baseBlob": {
            "tierToCool": {
              "daysAfterModificationGreaterThan": 30
            }
          }
        },
        "filters": {
          "blobTypes": ["blockBlob"]
        }
      }
    }
  ]
}

Características de Seguridad

  • Firmas de Acceso Compartido (SAS): Proporciona acceso limitado a los recursos de tu cuenta de almacenamiento sin exponer tu clave de cuenta.
  • Integración con Azure Active Directory (AAD): Te permite usar AAD para autorizar solicitudes al almacenamiento de blobs.

Conclusión

Azure Blob Storage es una solución versátil y escalable para almacenar datos no estructurados en la nube. Al comprender sus conceptos clave, configurar cuentas de almacenamiento y contenedores, y aprovechar características avanzadas como los niveles de almacenamiento y la gestión del ciclo de vida, puedes gestionar eficientemente tus necesidades de almacenamiento de datos. Ya seas un principiante o un usuario avanzado, Azure Blob Storage ofrece las herramientas y la flexibilidad para satisfacer tus requisitos.
