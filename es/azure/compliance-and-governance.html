Introducción al Cumplimiento y Gobernanza

El cumplimiento y la gobernanza son aspectos críticos para gestionar los recursos en la nube de manera efectiva. Aseguran que tu organización cumpla con los requisitos legales, regulatorios y organizacionales mientras mantiene el control sobre tu entorno en la nube.

Conceptos Clave

  • Cumplimiento: Adherirse a leyes, regulaciones y políticas organizacionales.
  • Gobernanza: Establecer políticas, roles, responsabilidades y procesos para controlar y gestionar los recursos de TI.

Azure Policy

Azure Policy es un servicio en Azure que puedes usar para crear, asignar y gestionar políticas. Estas políticas imponen diferentes reglas y efectos sobre tus recursos, asegurando que esos recursos se mantengan conformes con tus estándares corporativos y acuerdos de nivel de servicio.

Creación y Asignación de Políticas

  1. Crear una Definición de Política: Define las reglas y efectos para tu política.
  2. Asignar la Política: Aplica la política a un ámbito específico, como una suscripción o grupo de recursos.
  3. Evaluar el Cumplimiento: Verifica el estado de cumplimiento de tus recursos en relación con la política.

Ejemplo: Creación de una Definición de Política

{
  "properties": {
    "displayName": "Ubicaciones permitidas",
    "policyType": "BuiltIn",
    "mode": "All",
    "parameters": {
      "allowedLocations": {
        "type": "Array",
        "metadata": {
          "description": "La lista de ubicaciones permitidas para los recursos.",
          "displayName": "Ubicaciones permitidas",
          "strongType": "location"
        }
      }
    },
    "policyRule": {
      "if": {
        "field": "location",
        "notIn": "[parameters('allowedLocations')]"
      },
      "then": {
        "effect": "deny"
      }
    }
  }
}

Esta política asegura que los recursos solo se puedan crear en ubicaciones especificadas.

Azure Blueprints

Azure Blueprints te permite definir un conjunto repetible de recursos de Azure que implementan y cumplen con estándares, patrones y requisitos. Los blueprints se utilizan para orquestar el despliegue de varios plantillas de recursos y otros artefactos, tales como:

  • Asignaciones de Roles
  • Asignaciones de Políticas
  • Plantillas de Azure Resource Manager (ARM)
  • Grupos de Recursos

Creación de un Blueprint

  1. Definir Artefactos: Especifica los recursos y configuraciones.
  2. Asignar el Blueprint: Aplica el blueprint a una suscripción o grupo de recursos.
  3. Rastrear y Gestionar: Monitorea el estado de despliegue y cumplimiento.

Ejemplo: Creación de un Blueprint

{
  "properties": {
    "displayName": "Blueprint de Ejemplo",
    "description": "Un blueprint de ejemplo para desplegar recursos.",
    "targetScope": "subscription",
    "parameters": {},
    "resourceGroups": {
      "myResourceGroup": {
        "properties": {
          "displayName": "Mi Grupo de Recursos",
          "description": "Grupo de recursos para mis recursos.",
          "location": "Oeste de EE.UU."
        }
      }
    },
    "artifacts": [
      {
        "type": "policyAssignment",
        "name": "allowedLocations",
        "properties": {
          "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/allowedLocations",
          "parameters": {
            "allowedLocations": {
              "value": ["Oeste de EE.UU.", "Este de EE.UU."]
            }
          }
        }
      }
    ]
  }
}

Este blueprint asegura que se cree un grupo de recursos y se asigne una política para restringir las ubicaciones de los recursos.

Plantillas de Azure Resource Manager (ARM)

Las plantillas ARM son archivos JSON que definen la infraestructura y configuración para tu solución en Azure. Te permiten desplegar, gestionar y configurar todos los recursos para tu solución de manera declarativa.

Características Clave

  • Sintaxis Declarativa: Define lo que deseas desplegar sin escribir la secuencia de comandos de programación.
  • Despliegues Repetibles: Asegura entornos consistentes en diferentes etapas.
  • Modular y Reutilizable: Divide las plantillas en componentes más pequeños y reutilizables.

Ejemplo: Plantilla ARM Básica

{
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentTemplate.json#",
  "contentVersion": "1.0.0.0",
  "resources": [
    {
      "type": "Microsoft.Storage/storageAccounts",
      "apiVersion": "2019-04-01",
      "name": "mystorageaccount",
      "location": "westus",
      "sku": {
        "name": "Standard_LRS"
      },
      "kind": "StorageV2",
      "properties": {}
    }
  ]
}

Esta plantilla despliega una cuenta de almacenamiento en la región "westus".

Conclusión

El cumplimiento y la gobernanza en Azure son esenciales para asegurar que tus recursos en la nube cumplan con los estándares legales, regulatorios y organizacionales. Al aprovechar herramientas como Azure Policy, Azure Blueprints y plantillas ARM, puedes automatizar y hacer cumplir estos estándares en todo tu entorno de Azure. Entender e implementar estas herramientas te ayudará a mantener el control y el cumplimiento, reduciendo riesgos y mejorando la gestión general de tus recursos en la nube.
