Introducción

En este módulo, exploraremos cómo configurar dominios personalizados y certificados SSL en Azure. Esto es esencial para asegurar tus aplicaciones web y proporcionar una apariencia profesional con nombres de dominio personalizados.

Dominios Personalizados

¿Qué es un Dominio Personalizado?

Un dominio personalizado es una dirección única y legible por humanos que puedes usar para acceder a tu aplicación web. En lugar de usar el dominio predeterminado proporcionado por Azure (por ejemplo, yourapp.azurewebsites.net), puedes usar un dominio como www.yourdomain.com.

¿Por qué Usar Dominios Personalizados?

  • Branding: Mejora la identidad de tu marca.
  • Confianza: Aumenta la confianza y credibilidad del usuario.
  • SEO: Mejora la optimización en motores de búsqueda.

Configuración de un Dominio Personalizado

  1. Compra un Dominio: Puedes comprar un dominio de registradores de dominios como GoDaddy, Namecheap, etc.
  2. Agregar Dominio Personalizado en Azure:
    • Navega a tu Aplicación Web en Azure.
    • Ve a la sección "Dominios personalizados".
    • Haz clic en "Agregar dominio personalizado".
    • Ingresa tu nombre de dominio personalizado.
  3. Verificar la Propiedad del Dominio:
    • Azure proporcionará un registro TXT que necesitas agregar a la configuración DNS de tu dominio.
    • Una vez que el registro TXT sea verificado, tu dominio estará vinculado a tu Aplicación Web en Azure.

Ejemplo: Agregar un Dominio Personalizado

# Navega a tu Aplicación Web en Azure
az webapp config hostname add --resource-group <ResourceGroupName> --webapp-name <AppName> --hostname www.yourdomain.com

Certificados SSL

¿Qué es un Certificado SSL?

Un certificado SSL (Secure Sockets Layer) encripta los datos transferidos entre el navegador del usuario y tu servidor web, asegurando la privacidad y seguridad de los datos.

¿Por qué Usar Certificados SSL?

  • Seguridad: Encripta datos sensibles.
  • Confianza: Los usuarios confían en sitios web con HTTPS.
  • SEO: Los motores de búsqueda prefieren sitios web habilitados con HTTPS.

Tipos de Certificados SSL

  • Certificados SSL Gratuitos: Proporcionados por servicios como Let's Encrypt.
  • Certificados SSL Pagos: Ofrecidos por varias Autoridades Certificadoras (CAs) con características adicionales y garantías.

Agregar un Certificado SSL a Azure

  1. Obtener un Certificado SSL:
    • Puedes obtener un certificado SSL gratuito de Let's Encrypt o comprar uno de una CA.
  2. Subir el Certificado SSL a Azure:
    • Navega a tu Aplicación Web en Azure.
    • Ve a la sección "Configuración TLS/SSL".
    • Haz clic en "Certificados de Clave Privada (.pfx)" y sube tu certificado.
  3. Vincular el Certificado SSL al Dominio Personalizado:
    • Después de subirlo, ve a la sección "Dominios personalizados".
    • Selecciona tu dominio personalizado y vincula el certificado SSL a él.

Ejemplo: Agregar un Certificado SSL

# Subir certificado SSL a la Aplicación Web en Azure
az webapp config ssl upload --resource-group <ResourceGroupName> --name <AppName> --certificate-file <PathToPfxFile> --certificate-password <Password>

# Vincular certificado SSL al dominio personalizado
az webapp config ssl bind --resource-group <ResourceGroupName> --name <AppName> --certificate-thumbprint <Thumbprint> --ssl-type SNI

Conclusión

Configurar dominios personalizados y certificados SSL en Azure es un paso crucial para asegurar y dar marca a tus aplicaciones web. Siguiendo los pasos descritos en este módulo, puedes mejorar la seguridad de tu aplicación y la confianza del usuario. Asegúrate siempre de que tus certificados SSL estén actualizados y renuévalos antes de que expiren para mantener una seguridad continua.
