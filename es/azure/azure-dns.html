Azure DNS es un servicio de alojamiento para dominios DNS, que proporciona resolución de nombres utilizando la infraestructura de Microsoft Azure. Al alojar tus dominios en Azure, puedes gestionar tus registros DNS utilizando las mismas credenciales, APIs, herramientas y facturación que tus otros servicios de Azure.

Introducción al DNS

  • DNS (Sistema de Nombres de Dominio): Un sistema jerárquico para traducir nombres de dominio amigables para los humanos (como www.example.com) en direcciones IP (como 192.0.2.1).
  • Registros DNS: Entradas en la base de datos DNS que asignan nombres de dominio a direcciones IP y otros recursos.

¿Por Qué Usar Azure DNS?

  • Alcance Global: Aprovecha la red global de servidores DNS de Microsoft para una resolución de nombres rápida y confiable.
  • Gestión Integrada: Gestiona los registros DNS utilizando el mismo portal de Azure, APIs y herramientas que otros servicios de Azure.
  • Seguridad: Benefíciate de las características de seguridad de Azure, incluyendo control de acceso basado en roles (RBAC) y registros de actividad.

Configuración de Azure DNS

Creación de una Zona DNS

Una zona DNS se utiliza para alojar los registros DNS de un dominio en particular.

  1. Navegar al Portal de Azure: Ve al portal de Azure en https://portal.azure.com.
  2. Crear un Recurso: Haz clic en "Crear un recurso" y busca "zona DNS".
  3. Configurar Zona DNS: Rellena los detalles necesarios como la suscripción, el grupo de recursos y el nombre de la zona DNS (por ejemplo, example.com).
  4. Revisar y Crear: Revisa tus configuraciones y haz clic en "Crear".

Añadir Registros DNS

Una vez creada la zona DNS, puedes añadir registros DNS.

  1. Navegar a la Zona DNS: Ve a la zona DNS que creaste.
  2. Añadir Conjunto de Registros: Haz clic en "+ Conjunto de registros" para añadir un nuevo registro DNS.
  3. Configurar Registro: Rellena los detalles como nombre, tipo (A, CNAME, MX, etc.), TTL (Tiempo de Vida) y valor.
  4. Guardar: Haz clic en "OK" para guardar el registro.

Ejemplo: Añadir un Registro A

Nombre: www
Tipo: A
TTL: 3600
Valor: 192.0.2.1

Este registro asigna www.example.com a la dirección IP 192.0.2.1.

Gestión de DNS con Azure CLI

Azure CLI proporciona una interfaz de línea de comandos para gestionar recursos de Azure, incluyendo DNS.

Instalación de Azure CLI

Sigue las instrucciones en Instalar Azure CLI para instalar Azure CLI en tu sistema.

Creación de una Zona DNS con Azure CLI

az network dns zone create --resource-group MyResourceGroup --name example.com

Añadir un Registro DNS con Azure CLI

az network dns record-set a add-record --resource-group MyResourceGroup --zone-name example.com --record-set-name www --ipv4-address 192.0.2.1

Características Avanzadas de DNS

Registros Alias

Los registros alias te permiten asignar un nombre DNS a un recurso de Azure, como una dirección IP pública o un perfil de Traffic Manager.

  • Caso de Uso: Simplifica la gestión de DNS cuando la dirección IP del recurso cambia.

Zonas DNS Privadas

Las zonas DNS privadas proporcionan resolución DNS para redes virtuales dentro de Azure.

  • Caso de Uso: Útil para escenarios donde necesitas resolución DNS para recursos internos que no están expuestos a internet.

Traffic Manager de DNS

Azure Traffic Manager es un balanceador de carga basado en DNS que te permite distribuir el tráfico de manera óptima a servicios en regiones globales de Azure.

  • Caso de Uso: Mejora la disponibilidad y capacidad de respuesta de tus aplicaciones al dirigir el tráfico de usuarios al punto final más cercano o más disponible.

Resumen

Azure DNS es un servicio poderoso y flexible para gestionar registros DNS en la nube. Se integra perfectamente con otros servicios de Azure, proporcionando una experiencia de gestión unificada. Al entender los conceptos básicos de DNS, configurar zonas DNS y aprovechar características avanzadas como registros alias y zonas DNS privadas, puedes gestionar eficazmente los registros DNS de tu dominio utilizando Azure DNS.

Puntos Clave

  • Azure DNS proporciona un servicio DNS globalmente distribuido y de alta disponibilidad.
  • Puedes gestionar zonas y registros DNS a través del portal de Azure, Azure CLI y APIs.
  • Características avanzadas como registros alias y zonas DNS privadas ofrecen flexibilidad y control adicionales.

Al dominar Azure DNS, puedes asegurar una resolución de nombres confiable y eficiente para tus aplicaciones y servicios.
