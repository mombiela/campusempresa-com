Introducción a la Computación Serverless

  • Definición: La computación serverless te permite construir y ejecutar aplicaciones y servicios sin gestionar la infraestructura. El proveedor de la nube provisiona, escala y gestiona automáticamente la infraestructura necesaria para ejecutar el código.
  • Beneficios:
    • Eficiencia de Costos: Paga solo por el tiempo de cómputo que consumes.
    • Escalabilidad: Escala automáticamente hacia arriba o hacia abajo según la demanda.
    • Reducción de Sobrecarga Operativa: No es necesario gestionar servidores o infraestructura.

¿Qué son las Azure Functions?

  • Definición: Azure Functions es un servicio de cómputo serverless que te permite ejecutar código impulsado por eventos sin tener que provisionar o gestionar explícitamente la infraestructura.
  • Características Clave:
    • Impulsado por Eventos: Se activa por eventos como solicitudes HTTP, temporizadores o mensajes de otros servicios de Azure.
    • Desarrollo Flexible: Soporta múltiples lenguajes de programación, incluyendo C#, JavaScript, Python y más.
    • Seguridad Integrada: Características de seguridad integradas como autenticación y autorización.

Empezando con Azure Functions

  • Creando tu Primera Función:
    1. Configura tu Entorno:
      • Instala las Azure Functions Core Tools.
      • Instala el Azure CLI.
    2. Crea una Function App: 
      az functionapp create --resource-group <ResourceGroupName> --consumption-plan-location <Region> --runtime <Runtime> --functions-version 3 --name <FunctionAppName> --storage-account <StorageAccountName>
    3. Crea una Nueva Función: 
      func init MyFunctionProj --dotnet
cd MyFunctionProj
func new --name MyHttpTrigger --template "HTTP trigger" --authlevel "anonymous"
    4. Ejecuta la Función Localmente: 
      func start
    5. Despliega la Función en Azure: 
      func azure functionapp publish <FunctionAppName>

Disparadores y Enlaces

  • Disparadores: Definen cómo se invoca una función.
    • Disparador HTTP: Invocado por solicitudes HTTP.
    • Disparador de Temporizador: Invocado en un horario.
    • Disparador de Cola: Invocado cuando se añade un mensaje a una Cola de Almacenamiento de Azure.
  • Enlaces: Conectan funciones a otros servicios.
    • Enlaces de Entrada: Recuperan datos de fuentes externas.
    • Enlaces de Salida: Envían datos a fuentes externas.

Ejemplo: Función de Disparador HTTP

using System.IO;
using Microsoft.AspNetCore.Mvc;
using Microsoft.Azure.WebJobs;
using Microsoft.Azure.WebJobs.Extensions.Http;
using Microsoft.AspNetCore.Http;
using Microsoft.Extensions.Logging;
using Newtonsoft.Json;

public static class MyHttpTrigger
{
    [FunctionName("MyHttpTrigger")]
    public static async Task<IActionResult> Run(
        [HttpTrigger(AuthorizationLevel.Function, "get", "post", Route = null)] HttpRequest req,
        ILogger log)
    {
        log.LogInformation("C# HTTP trigger function processed a request.");

        string name = req.Query["name"];

        string requestBody = await new StreamReader(req.Body).ReadToEndAsync();
        dynamic data = JsonConvert.DeserializeObject(requestBody);
        name = name ?? data?.name;

        return name != null
            ? (ActionResult)new OkObjectResult($"Hello, {name}")
            : new BadRequestObjectResult("Please pass a name on the query string or in the request body");
    }
}
  • Explicación:
    • La función se activa por una solicitud HTTP.
    • Lee el parámetro name de la cadena de consulta o del cuerpo de la solicitud.
    • Devuelve un mensaje de saludo si se proporciona name, de lo contrario, devuelve un mensaje de solicitud incorrecta.

Conceptos Avanzados

  • Funciones Durables: Extiende Azure Functions con flujos de trabajo con estado.
    • Funciones de Orquestador: Definen flujos de trabajo usando código.
    • Funciones de Actividad: Realizan tareas dentro del flujo de trabajo.
  • Escalabilidad y Rendimiento:
    • Plan de Consumo: Escala automáticamente según la demanda.
    • Plan Premium: Proporciona opciones mejoradas de rendimiento y escalabilidad.
  • Seguridad:
    • Autenticación y Autorización: Integración con Azure AD, proveedores OAuth, etc.
    • Identidades Administradas: Acceso seguro a recursos de Azure sin almacenar credenciales.

Monitoreo y Solución de Problemas

  • Application Insights: Monitorea y diagnostica problemas con tus funciones.
    • Registro: Captura registros para depuración.
    • Métricas: Rastrea métricas de rendimiento y uso.
    • Alertas: Configura alertas para condiciones específicas.

Conclusión

Azure Functions y la computación serverless ofrecen una forma poderosa de construir aplicaciones escalables e impulsadas por eventos sin la sobrecarga de gestionar infraestructura. Al entender los conceptos básicos de crear y desplegar funciones, trabajar con disparadores y enlaces, y explorar conceptos avanzados como las Funciones Durables y la seguridad, puedes aprovechar todo el potencial de Azure Functions para construir soluciones eficientes y rentables.
