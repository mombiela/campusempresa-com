Introducción al Azure Load Balancer

Azure Load Balancer es un balanceador de carga de Capa 4 (TCP, UDP) que distribuye el tráfico de red entrante entre múltiples máquinas virtuales (VM) o servicios. Garantiza alta disponibilidad y fiabilidad al distribuir el tráfico de manera eficiente.

Conceptos Clave

  • Balanceo de Carga: Distribuir el tráfico entrante entre múltiples servidores para asegurar que ningún servidor se sature.
  • Alta Disponibilidad: Garantizar que los servicios estén disponibles incluso si uno o más servidores fallan.
  • Escalabilidad: Capacidad de manejar una carga aumentada añadiendo más servidores.

Tipos de Azure Load Balancers

Azure ofrece dos tipos principales de balanceadores de carga:

Balanceador de Carga Básico

  • Caso de Uso: Adecuado para aplicaciones a pequeña escala o entornos de desarrollo/pruebas.
  • Características:
    • Limitado a un solo conjunto de disponibilidad o conjunto de escalado de máquinas virtuales.
    • No tiene SLA (Acuerdo de Nivel de Servicio) para el Balanceador de Carga Básico.
    • Limitado a 300 puntos finales.

Balanceador de Carga Estándar

  • Caso de Uso: Adecuado para entornos de producción que requieren alta disponibilidad y escalabilidad.
  • Características:
    • Soporta múltiples zonas de disponibilidad.
    • Proporciona un SLA más alto (99.99%).
    • Soporta hasta 1000 puntos finales.
    • Diagnósticos y métricas avanzadas.

| Característica | Balanceador de Carga Básico | Balanceador de Carga Estándar | |----------------------------|-----------------------------|-------------------------------| | SLA | Ninguno | 99.99% | | Zonas de Disponibilidad | No | Sí | | Puntos Finales Máximos | 300 | 1000 | | Diagnósticos y Métricas | Básico | Avanzado |

Configuración de un Azure Load Balancer

Requisitos Previos

  • Una cuenta de Azure.
  • Una red virtual con al menos dos máquinas virtuales.

Guía Paso a Paso

  1. Crear un Balanceador de Carga:

    • Navegar al portal de Azure.
    • Seleccionar "Crear un recurso" > "Redes" > "Balanceador de Carga".
    • Elegir la suscripción, grupo de recursos y región.
    • Seleccionar el tipo (Básico o Estándar) y SKU (Público o Interno).

  2. Configurar IP de Frontend:

    • En la configuración del Balanceador de Carga, seleccionar "Configuración de IP de Frontend".
    • Añadir una nueva configuración de IP de frontend y asignar una dirección IP pública o privada.

  3. Crear un Pool de Backend:

    • Seleccionar "Pools de Backend" y añadir un nuevo pool de backend.
    • Añadir las máquinas virtuales que recibirán el tráfico.

  4. Configurar Probes de Salud:

    • Seleccionar "Probes de Salud" y añadir un nuevo probe.
    • Configurar el probe con protocolo (TCP/HTTP), puerto y ajustes de intervalo.

  5. Crear Reglas de Balanceo de Carga:

    • Seleccionar "Reglas de Balanceo de Carga" y añadir una nueva regla.
    • Configurar la regla con IP de frontend, pool de backend, protocolo, puerto y ajustes de persistencia de sesión.

Ejemplo: Crear un Balanceador de Carga con Azure CLI

# Crear un grupo de recursos
az group create --name myResourceGroup --location eastus

# Crear una dirección IP pública
az network public-ip create --resource-group myResourceGroup --name myPublicIP

# Crear un balanceador de carga
az network lb create --resource-group myResourceGroup --name myLoadBalancer --public-ip-address myPublicIP --frontend-ip-name myFrontEnd --backend-pool-name myBackEndPool

# Crear un probe de salud
az network lb probe create --resource-group myResourceGroup --lb-name myLoadBalancer --name myHealthProbe --protocol tcp --port 80

# Crear una regla de balanceo de carga
az network lb rule create --resource-group myResourceGroup --lb-name myLoadBalancer --name myLoadBalancingRule --protocol tcp --frontend-port 80 --backend-port 80 --frontend-ip-name myFrontEnd --backend-pool-name myBackEndPool --probe-name myHealthProbe

Configuración Avanzada

Autoescalado con Conjuntos de Escalado de Máquinas Virtuales

  • Conjuntos de Escalado de Máquinas Virtuales: Aumenta o disminuye automáticamente el número de instancias de VM según la demanda.
  • Integración con Balanceador de Carga: Añade o elimina automáticamente VMs del pool de backend.

Diagnósticos y Monitoreo

  • Azure Monitor: Proporciona métricas y registros para balanceadores de carga.
  • Network Watcher: Ayuda a diagnosticar y solucionar problemas de red.

Consideraciones de Seguridad

  • Grupos de Seguridad de Red (NSGs): Controlan el tráfico entrante y saliente hacia las VMs.
  • Protección DDoS: Protege contra ataques de Denegación de Servicio Distribuidos.

Conclusión

Azure Load Balancer es una herramienta poderosa para garantizar alta disponibilidad, fiabilidad y escalabilidad de tus aplicaciones. Al entender los diferentes tipos de balanceadores de carga, configurarlos correctamente y aprovechar características avanzadas como el autoescalado y los diagnósticos, puedes optimizar el rendimiento y la resiliencia de tus servicios en Azure.
