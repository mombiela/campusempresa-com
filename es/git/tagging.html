Introducción al Etiquetado

El etiquetado es una característica en Git que te permite marcar puntos específicos en la historia de tu repositorio como importantes. Típicamente, las personas usan esta funcionalidad para marcar puntos de lanzamiento (por ejemplo, v1.0.0, v2.0.0).

Conceptos Clave

  • Etiquetas Ligeras: Punteros simples a un commit específico.
  • Etiquetas Anotadas: Objetos completos en la base de datos de Git, que contienen un nombre de etiquetador, correo electrónico, fecha y un mensaje.

Creación de Etiquetas

Etiquetas Ligeras

Las etiquetas ligeras son rápidas y simples de crear. Son solo punteros a un commit específico.

git tag v1.0.0

Este comando etiqueta el commit actual con el nombre v1.0.0.

Etiquetas Anotadas

Se recomiendan las etiquetas anotadas porque almacenan metadatos adicionales sobre la etiqueta.

git tag -a v1.0.0 -m "Release version 1.0.0"
  • -a: Crea una etiqueta anotada.
  • -m: Añade un mensaje a la etiqueta.

Visualización de Etiquetas

Para listar todas las etiquetas en tu repositorio, usa:

git tag

Para ver detalles sobre una etiqueta específica:

git show v1.0.0

Etiquetado de un Commit Específico

Puedes etiquetar un commit específico proporcionando el hash del commit:

git tag -a v1.0.0 <commit-hash> -m "Release version 1.0.0"

Compartir Etiquetas

Por defecto, las etiquetas no se envían a los repositorios remotos. Para enviar etiquetas, necesitas especificarlas explícitamente:

Enviar una Sola Etiqueta

git push origin v1.0.0

Enviar Todas las Etiquetas

git push origin --tags

Eliminación de Etiquetas

Eliminar una Etiqueta Local

git tag -d v1.0.0

Eliminar una Etiqueta Remota

Primero, elimina la etiqueta localmente:

git tag -d v1.0.0

Luego, elimina la etiqueta del repositorio remoto:

git push origin :refs/tags/v1.0.0

Ejemplos Prácticos

Ejemplo 1: Crear y Enviar una Etiqueta Anotada

  1. Crear una etiqueta anotada: 
    git tag -a v1.1.0 -m "Release version 1.1.0"
  2. Enviar la etiqueta al repositorio remoto: 
    git push origin v1.1.0

Ejemplo 2: Etiquetar un Commit Específico y Enviar Todas las Etiquetas

  1. Etiquetar un commit específico: 
    git tag -a v2.0.0 9fceb02 -m "Release version 2.0.0"
  2. Enviar todas las etiquetas al repositorio remoto: 
    git push origin --tags

Resumen

El etiquetado en Git es una forma poderosa de marcar puntos importantes en la historia de tu proyecto. Ya sea que uses etiquetas ligeras o anotadas, entender cómo crear, ver, compartir y eliminar etiquetas es esencial para un control de versiones efectivo. Al dominar estos conceptos, puedes gestionar mejor los lanzamientos y hitos de tu proyecto.
