Fusionar ramas es un concepto fundamental en Git que te permite combinar cambios de diferentes ramas en una sola rama. Esto es esencial para el desarrollo colaborativo y la gestión de diferentes características o versiones de un proyecto.

Conceptos Clave

  • Rama: Una versión paralela de un repositorio. Las ramas te permiten trabajar en diferentes características o correcciones de manera independiente.
  • Fusión: El proceso de integrar cambios de una rama en otra.
  • Conflicto: Ocurre cuando los cambios en diferentes ramas interfieren entre sí y Git no puede resolverlos automáticamente.

Tipos de Fusiones

  1. Fusión de Avance Rápido:

    • Ocurre cuando la rama que se está fusionando no ha divergido de la rama objetivo.
    • Git simplemente mueve el puntero hacia adelante.

  2. Fusión de Tres Vías:

    • Se utiliza cuando las ramas han divergido.
    • Git usa el ancestro común y las dos puntas de las ramas para crear un nuevo commit de fusión.

Fusión de Avance Rápido

Ejemplo

Supongamos que tienes una rama feature que deseas fusionar en main.

  1. Cambia a la rama main:

    git checkout main

  2. Fusiona la rama feature:

    git merge feature

Si main no ha divergido de feature, Git realizará una fusión de avance rápido.

Explicación

En una fusión de avance rápido, Git simplemente mueve el puntero de la rama main hacia adelante hasta el último commit en la rama feature. Este es el tipo de fusión más simple y no crea un nuevo commit.

Fusión de Tres Vías

Ejemplo

Supongamos que tienes una rama feature que ha divergido de main.

  1. Cambia a la rama main:

    git checkout main

  2. Fusiona la rama feature:

    git merge feature

Si main ha divergido de feature, Git realizará una fusión de tres vías.

Explicación

En una fusión de tres vías, Git usa el ancestro común de las dos ramas y las puntas de ambas ramas para crear un nuevo commit de fusión. Este nuevo commit representa los cambios combinados de ambas ramas.

Manejo de Conflictos de Fusión

Ejemplo

Supongamos que tienes un conflicto al fusionar feature en main.

  1. Git te notificará del conflicto:

    Auto-merging file.txt
CONFLICT (content): Merge conflict in file.txt
Automatic merge failed; fix conflicts and then commit the result.

  2. Abre el archivo en conflicto y resuelve el conflicto:

    <<<<<<< HEAD
Este es algún contenido de la rama main.
=======
Este es algún contenido de la rama feature.
>>>>>>> feature

  3. Después de resolver el conflicto, añade el archivo resuelto:

    git add file.txt

  4. Completa la fusión haciendo un commit:

    git commit

Explicación

Cuando ocurre un conflicto, Git marca las áreas en conflicto en el archivo. Necesitas editar manualmente el archivo para resolver el conflicto, luego añadir y hacer commit del archivo resuelto para completar la fusión.

Resumen

Fusionar ramas en Git es una habilidad crucial para gestionar cambios en un entorno colaborativo. Entender la diferencia entre fusiones de avance rápido y de tres vías, así como cómo manejar conflictos de fusión, te ayudará a mantener una base de código limpia y funcional. Practica fusionar ramas y resolver conflictos para volverte competente en estas operaciones esenciales de Git.
