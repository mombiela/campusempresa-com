Introducción al Guardado de Cambios

Guardar cambios en Git te permite guardar temporalmente los cambios en tu directorio de trabajo sin comprometerlos. Esto es particularmente útil cuando necesitas cambiar de rama o trabajar en otra cosa pero quieres volver a tus cambios actuales más tarde.

Conceptos Clave

  • Stash: Un stash es un estado guardado de tu directorio de trabajo y el índice.
  • Lista de Stash: Una lista de todos los stashes que has guardado.
  • Aplicar un Stash: Recuperar los cambios de un stash a tu directorio de trabajo.
  • Eliminar un Stash: Borrar un stash de la lista de stashes.

Comandos Básicos de Guardado de Cambios

Guardar Cambios

Para guardar tus cambios actuales, usa el comando git stash:

git stash

Este comando guardará tus cambios y revertirá tu directorio de trabajo al estado del último commit.

Listar Stashes

Para ver todos los stashes que has guardado, usa:

git stash list

Esto mostrará una lista de stashes con sus identificadores.

Aplicar un Stash

Para aplicar el stash más reciente, usa:

git stash apply

Para aplicar un stash específico, usa:

git stash apply stash@{index}

Reemplaza index con el número del stash que quieres aplicar.

Eliminar un Stash

Para borrar un stash específico, usa:

git stash drop stash@{index}

Para borrar todos los stashes, usa:

git stash clear

Técnicas Avanzadas de Guardado de Cambios

Guardar Archivos No Rastreables

Por defecto, git stash no incluye archivos no rastreables. Para guardar archivos no rastreables, usa:

git stash -u

o

git stash --include-untracked

Crear un Stash Nombrado

Puedes nombrar tus stashes para una mejor identificación:

git stash save "mensaje significativo"

Guardar Archivos Específicos

Para guardar archivos específicos, usa el comando git stash push:

git stash push -m "mensaje" ruta/al/archivo

Pop vs. Apply

  • Apply: Aplica el stash pero lo mantiene en la lista de stashes.
  • Pop: Aplica el stash y lo elimina de la lista de stashes.
git stash pop

Ejemplos

Ejemplo 1: Guardado y Aplicación Básica

  1. Realiza algunos cambios en tus archivos.
  2. Guarda los cambios: 
    git stash
  3. Lista los stashes: 
    git stash list
  4. Aplica el stash: 
    git stash apply

Ejemplo 2: Guardado con Archivos No Rastreables

  1. Realiza cambios y agrega nuevos archivos no rastreables.
  2. Guarda todos los cambios, incluyendo archivos no rastreables: 
    git stash -u
  3. Lista y aplica el stash: 
    git stash list
git stash apply

Ejemplo 3: Stash Nombrado y Pop

  1. Realiza cambios y guarda con un nombre: 
    git stash save "trabajo en progreso"
  2. Lista los stashes: 
    git stash list
  3. Aplica el stash: 
    git stash pop

Resumen

Guardar cambios es una característica poderosa en Git que te permite guardar tu trabajo temporalmente y volver a él más tarde. Es especialmente útil para gestionar trabajos en progreso cuando necesitas cambiar de contexto. Entender cómo guardar, listar, aplicar y eliminar stashes mejorará tu flujo de trabajo y te hará más eficiente en la gestión de tus cambios de código.
