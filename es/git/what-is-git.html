Introducción

Git es un sistema de control de versiones distribuido (VCS) que permite a múltiples personas trabajar en un proyecto simultáneamente sin sobrescribir los cambios de los demás. Ayuda a rastrear cambios, revertir a etapas anteriores y colaborar de manera eficiente. Creado por Linus Torvalds en 2005, Git se ha convertido en el sistema de control de versiones más utilizado en el mundo.

Conceptos Clave

  • Sistema de Control de Versiones (VCS): Una herramienta que ayuda a gestionar los cambios en el código fuente a lo largo del tiempo.
  • Distribuido: A diferencia de los VCS centralizados, Git permite que cada usuario tenga una copia completa del repositorio.
  • Repositorio (Repo): Un lugar de almacenamiento para paquetes de software, que contiene todos los archivos del proyecto y el historial de cambios.
  • Commit: Una instantánea de tu repositorio en un momento específico.
  • Branch: Una línea de desarrollo separada, que te permite trabajar en diferentes características o correcciones de manera independiente.
  • Merge: El proceso de combinar cambios de diferentes ramas.
  • Clone: Crear una copia de un repositorio desde una fuente remota.
  • Pull: Obtener cambios de un repositorio remoto y fusionarlos en tu repositorio local.
  • Push: Enviar tus cambios locales a un repositorio remoto.

Flujo de Trabajo Básico de Git

  1. Inicializar un Repositorio: Crear un nuevo repositorio Git o clonar uno existente.
  2. Hacer Cambios: Modificar archivos en tu directorio de trabajo.
  3. Preparar Cambios: Añadir cambios al área de preparación.
  4. Confirmar Cambios: Guardar una instantánea de los cambios preparados.
  5. Enviar Cambios: Enviar tus commits a un repositorio remoto.

Comandos Básicos de Git

Inicializar un Repositorio

Para iniciar un nuevo repositorio Git, navega a tu directorio de proyecto y ejecuta:

git init

Este comando crea un nuevo directorio .git en tu proyecto, que contiene todos los metadatos necesarios para el repositorio.

Clonar un Repositorio

Para clonar un repositorio existente, usa:

git clone <repository_url>

Este comando copia todo el repositorio, incluyendo su historial, a tu máquina local.

Preparar Cambios

Para añadir cambios al área de preparación, usa:

git add <file_name>

También puedes añadir todos los cambios de una vez:

git add .

Confirmar Cambios

Para confirmar los cambios preparados, usa:

git commit -m "Tu mensaje de commit"

La bandera -m te permite añadir un mensaje de commit en línea.

Enviar Cambios

Para enviar tus commits a un repositorio remoto, usa:

git push origin <branch_name>

Este comando envía tus commits locales a la rama especificada en el repositorio remoto.

Obtener Cambios

Para obtener y fusionar cambios de un repositorio remoto, usa:

git pull origin <branch_name>

Este comando actualiza tu repositorio local con los últimos cambios de la rama remota.

Ejemplo de Flujo de Trabajo

  1. Inicializar un Repositorio:

    git init

  2. Crear un Archivo:

    echo "¡Hola, Git!" > hello.txt

  3. Preparar el Archivo:

    git add hello.txt

  4. Confirmar el Archivo:

    git commit -m "Añadir hello.txt"

  5. Crear un Repositorio Remoto (por ejemplo, en GitHub) y enlazarlo:

    git remote add origin <repository_url>

  6. Enviar el Commit:

    git push -u origin master

Resumen

Git es un sistema de control de versiones poderoso y flexible que soporta el desarrollo distribuido. Entender sus conceptos y comandos básicos es esencial para una colaboración y gestión de proyectos eficientes. Al dominar Git, puedes rastrear cambios, colaborar con otros y mantener un historial del desarrollo de tu proyecto.
