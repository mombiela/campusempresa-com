Introducción

En esta sección, exploraremos cómo implementar nuevas funcionalidades y colaborar eficazmente usando Git. Esto incluye crear ramas, fusionar cambios, resolver conflictos y usar pull requests. Al final de esta sección, serás capaz de gestionar tu base de código de manera colaborativa y eficiente.

Creación de Ramas en Git

¿Qué es una Rama?

Una rama en Git es un puntero móvil y ligero a un commit. Crear ramas te permite divergir de la línea principal de desarrollo y continuar trabajando sin afectar la línea principal.

Creación de una Rama

Para crear una nueva rama, usa el comando git branch seguido del nombre de la rama.

git branch feature-branch

Cambiar a una Rama

Para cambiar a la rama recién creada, usa el comando git checkout.

git checkout feature-branch

Alternativamente, puedes crear y cambiar a una nueva rama en un solo comando:

git checkout -b feature-branch

Listar Ramas

Para listar todas las ramas en tu repositorio, usa:

git branch

Fusionar Ramas

¿Qué es Fusionar?

Fusionar es el proceso de combinar los cambios de una rama en otra. Esto se usa comúnmente para integrar ramas de funcionalidades de vuelta a la rama principal.

Fusionar una Rama

Para fusionar una rama en tu rama actual, usa el comando git merge seguido del nombre de la rama.

git checkout main
git merge feature-branch

Manejo de Conflictos de Fusión

A veces, los cambios en diferentes ramas entran en conflicto entre sí. Git te pedirá que resuelvas estos conflictos manualmente.

Ejemplo de un Conflicto de Fusión

Supongamos que tienes el siguiente archivo en ambas ramas:

rama principal (file.txt):

Hello, World!

feature-branch (file.txt):

Hello, Git!

Cuando intentas fusionar feature-branch en main, Git producirá un conflicto:

file.txt:

<<<<<<< HEAD
Hello, World!
=======
Hello, Git!
>>>>>>> feature-branch

Para resolver el conflicto, edita el archivo al estado deseado y luego añade y comete los cambios:

git add file.txt
git commit -m "Resolved merge conflict in file.txt"

Colaboración con Pull Requests

¿Qué es un Pull Request?

Un pull request (PR) es una forma de proponer cambios a un repositorio. Permite a los miembros del equipo revisar y discutir los cambios antes de que sean fusionados.

Crear un Pull Request

  1. Empuja tu rama al repositorio remoto:

    git push origin feature-branch

  2. Ve al repositorio en GitHub (u otro servicio de alojamiento Git) y navega a la pestaña "Pull requests".

  3. Haz clic en "New pull request" y selecciona tu rama.

  4. Añade un título y descripción para tu pull request, luego haz clic en "Create pull request".

Revisar y Fusionar un Pull Request

Los miembros del equipo pueden revisar el pull request, dejar comentarios y sugerir cambios. Una vez que el pull request es aprobado, puede ser fusionado en la rama principal.

Resumen

En esta sección, cubrimos:

  • Creación de Ramas: Crear, cambiar y listar ramas.
  • Fusión: Combinar cambios de diferentes ramas y resolver conflictos.
  • Pull Requests: Proponer, revisar y fusionar cambios de manera colaborativa.

Al dominar estos conceptos, puedes gestionar y colaborar efectivamente en el código en un repositorio Git, asegurando un flujo de trabajo de desarrollo suave y eficiente.
