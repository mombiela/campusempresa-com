Introducción

En esta sección, proporcionaremos una descripción general del proyecto en el que trabajarás a lo largo de este curso de Git. Este proyecto está diseñado para ayudarte a aplicar los conceptos que aprendas de una manera práctica y práctica. Al final del curso, tendrás una comprensión sólida de Git y cómo usarlo de manera efectiva en escenarios del mundo real.

Descripción del proyecto

El proyecto para este curso es una aplicación web simple que permite a los usuarios crear, leer, actualizar y eliminar (CRUD) notas. Esta aplicación se construirá utilizando HTML, CSS y JavaScript. A lo largo del curso, usarás Git para gestionar el desarrollo de este proyecto, rastrear cambios, colaborar con otros y mantener diferentes versiones de tu código.

Conceptos clave

  • Control de versiones: Comprender la importancia del control de versiones y cómo Git ayuda a gestionar los cambios en el código.
  • Repositorios: Crear y gestionar repositorios de Git.
  • Commits: Hacer commits para guardar cambios en el repositorio.
  • Ramas: Trabajar con ramas para gestionar diferentes versiones del proyecto.
  • Fusión: Combinar cambios de diferentes ramas.
  • Colaboración: Usar Git para la colaboración en equipo, incluyendo el manejo de conflictos.
  • Repositorios remotos: Trabajar con repositorios remotos en plataformas como GitHub.

Requisitos del proyecto

Para completar con éxito este proyecto, necesitarás cumplir con los siguientes requisitos:

  1. Conocimientos básicos de HTML, CSS y JavaScript: Debes tener una comprensión básica de estas tecnologías, ya que se utilizarán para construir la aplicación web.
  2. Instalación de Git: Asegúrate de que Git esté instalado en tu computadora. Puedes descargarlo desde git-scm.com.
  3. Cuenta de GitHub: Crea una cuenta de GitHub si aún no tienes una. Esto se utilizará para la gestión de repositorios remotos.
  4. Editor de texto: Usa un editor de texto o un entorno de desarrollo integrado (IDE) como Visual Studio Code, Sublime Text o Atom.

Configuración del proyecto

Sigue estos pasos para configurar el proyecto:

  1. Crear un nuevo repositorio:

    git init my-notes-app
cd my-notes-app

  2. Crear archivos iniciales:

    touch index.html styles.css app.js

  3. Agregar archivos a Git:

    git add .

  4. Hacer el primer commit:

    git commit -m "Initial commit: Set up project structure"

  5. Crear un repositorio remoto en GitHub:

    • Ve a GitHub y crea un nuevo repositorio llamado my-notes-app.
    • Sigue las instrucciones para vincular tu repositorio local al repositorio remoto: 
      git remote add origin https://github.com/tu-usuario/my-notes-app.git
git push -u origin master

Resumen

En esta sección, hemos delineado el proyecto en el que trabajarás a lo largo de este curso de Git. Discutimos los conceptos clave que aprenderás, los requisitos que necesitas cumplir y los pasos iniciales para la configuración del proyecto. Siguiendo estas pautas, estarás bien preparado para sumergirte en los aspectos prácticos del uso de Git para el control de versiones y la colaboración.
