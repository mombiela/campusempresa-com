Introducción

En esta sección, aprenderemos a construir una aplicación CRUD (Crear, Leer, Actualizar, Eliminar) simple usando SQL. Las operaciones CRUD son fundamentales para interactuar con bases de datos y son esenciales para cualquier aplicación impulsada por bases de datos.

Configurando el Entorno

Antes de comenzar, asegúrate de tener lo siguiente:

  • Un sistema de gestión de bases de datos SQL (DBMS) como MySQL, PostgreSQL o SQLite instalado.
  • Una herramienta cliente de base de datos como MySQL Workbench, pgAdmin o DBeaver.

Creando la Base de Datos y las Tablas

Primero, necesitamos crear una base de datos y las tablas necesarias.

Creando la Base de Datos

CREATE DATABASE SimpleCRUD;
USE SimpleCRUD;

Explicación:

  • CREATE DATABASE SimpleCRUD; crea una nueva base de datos llamada SimpleCRUD.
  • USE SimpleCRUD; cambia a la base de datos SimpleCRUD.

Creando la Tabla

CREATE TABLE Users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Explicación:

  • id INT AUTO_INCREMENT PRIMARY KEY crea una clave primaria autoincremental.
  • username VARCHAR(50) NOT NULL define una columna de nombre de usuario que no puede ser nula.
  • email VARCHAR(100) NOT NULL define una columna de correo electrónico que no puede ser nula.
  • created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP establece una marca de tiempo predeterminada para cuando se crea el registro.

Operaciones CRUD

Crear

Para agregar nuevos registros a la tabla Users, usamos la declaración INSERT.

INSERT INTO Users (username, email) VALUES ('john_doe', 'john@example.com');

Explicación:

  • INSERT INTO Users (username, email) especifica la tabla y las columnas en las que insertar datos.
  • VALUES ('john_doe', 'john@example.com') proporciona los valores para las columnas especificadas.

Leer

Para recuperar datos de la tabla Users, usamos la declaración SELECT.

SELECT * FROM Users;

Explicación:

  • SELECT * FROM Users; recupera todas las columnas de la tabla Users.

Actualizar

Para modificar registros existentes, usamos la declaración UPDATE.

UPDATE Users SET email = 'john_new@example.com' WHERE username = 'john_doe';

Explicación:

  • UPDATE Users SET email = 'john_new@example.com' cambia el correo electrónico del usuario.
  • WHERE username = 'john_doe'; especifica qué registro actualizar.

Eliminar

Para eliminar registros de la tabla Users, usamos la declaración DELETE.

DELETE FROM Users WHERE username = 'john_doe';

Explicación:

  • DELETE FROM Users especifica la tabla de la que eliminar.
  • WHERE username = 'john_doe'; especifica qué registro eliminar.

Operaciones CRUD Avanzadas

Usando Transacciones

Las transacciones aseguran que una serie de operaciones SQL se ejecuten como una única unidad de trabajo.

START TRANSACTION;

INSERT INTO Users (username, email) VALUES ('jane_doe', 'jane@example.com');
UPDATE Users SET email = 'jane_new@example.com' WHERE username = 'jane_doe';

COMMIT;

Explicación:

  • START TRANSACTION; comienza una nueva transacción.
  • COMMIT; guarda todos los cambios realizados durante la transacción.

Manejo de Errores con Transacciones

Para manejar errores, podemos usar ROLLBACK para deshacer cambios si algo sale mal.

START TRANSACTION;

INSERT INTO Users (username, email) VALUES ('jane_doe', 'jane@example.com');

-- Simular un error
UPDATE Users SET email = 'invalid_email' WHERE username = 'jane_doe';

ROLLBACK;

Explicación:

  • ROLLBACK; deshace todos los cambios realizados durante la transacción si ocurre un error.

Conclusión

En este tema, cubrimos los conceptos básicos de la construcción de una aplicación CRUD simple usando SQL. Aprendimos cómo:

  • Crear una base de datos y una tabla.
  • Realizar operaciones CRUD básicas: Crear, Leer, Actualizar y Eliminar.
  • Usar transacciones para asegurar la integridad de los datos.

Al dominar estas operaciones fundamentales, puedes gestionar y manipular datos de manera efectiva en cualquier base de datos basada en SQL.
