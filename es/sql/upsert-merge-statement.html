Introducción a UPSERT (MERGE)

La declaración UPSERT (MERGE) es un comando SQL poderoso que te permite insertar nuevos registros o actualizar registros existentes dentro de una tabla. Esto es particularmente útil cuando necesitas sincronizar datos entre tablas o cuando quieres asegurarte de que los datos estén actualizados sin tener que realizar operaciones INSERT y UPDATE por separado.

Conceptos Clave

  • UPSERT: Una combinación de "UPDATE" e "INSERT". Asegura que si un registro existe, se actualiza; si no existe, se inserta.
  • MERGE: El comando estándar SQL que realiza la operación UPSERT. Combina la funcionalidad de INSERT y UPDATE en una sola declaración.

Sintaxis Básica

La sintaxis básica de la declaración MERGE es la siguiente:

MERGE INTO target_table AS target
USING source_table AS source
ON target.matching_column = source.matching_column
WHEN MATCHED THEN
    UPDATE SET target.column1 = source.column1, target.column2 = source.column2
WHEN NOT MATCHED THEN
    INSERT (column1, column2) VALUES (source.column1, source.column2);

Explicación

  • MERGE INTO target_table: Especifica la tabla de destino donde se insertarán o actualizarán los datos.
  • USING source_table: Especifica la tabla fuente que contiene los nuevos datos.
  • ON target.matching_column = source.matching_column: Define la condición para hacer coincidir los registros entre las tablas de destino y fuente.
  • WHEN MATCHED THEN UPDATE: Especifica la acción a tomar cuando se encuentra una coincidencia.
  • WHEN NOT MATCHED THEN INSERT: Especifica la acción a tomar cuando no se encuentra una coincidencia.

Ejemplo

Considera dos tablas: employees (destino) y new_employees (fuente).

Estructuras de las Tablas

employees

| employee_id | name | department | |-------------|---------|------------| | 1 | Alice | HR | | 2 | Bob | IT | | 3 | Charlie | Finance |

new_employees

| employee_id | name | department | |-------------|---------|------------| | 2 | Bob | Marketing | | 4 | David | IT |

Declaración MERGE

MERGE INTO employees AS e
USING new_employees AS ne
ON e.employee_id = ne.employee_id
WHEN MATCHED THEN
    UPDATE SET e.name = ne.name, e.department = ne.department
WHEN NOT MATCHED THEN
    INSERT (employee_id, name, department) VALUES (ne.employee_id, ne.name, ne.department);

Resultado

Después de ejecutar la declaración MERGE anterior, la tabla employees se actualizará de la siguiente manera:

| employee_id | name | department | |-------------|---------|------------| | 1 | Alice | HR | | 2 | Bob | Marketing | | 3 | Charlie | Finance | | 4 | David | IT |

Uso Avanzado

Manejo de Múltiples Condiciones

Puedes manejar múltiples condiciones dentro de la declaración MERGE para realizar diferentes acciones basadas en varios criterios.

MERGE INTO employees AS e
USING new_employees AS ne
ON e.employee_id = ne.employee_id
WHEN MATCHED AND e.department != ne.department THEN
    UPDATE SET e.department = ne.department
WHEN MATCHED AND e.department = ne.department THEN
    UPDATE SET e.name = ne.name
WHEN NOT MATCHED THEN
    INSERT (employee_id, name, department) VALUES (ne.employee_id, ne.name, ne.department);

Uso de Subconsultas

También puedes usar subconsultas en la cláusula USING para generar dinámicamente los datos fuente.

MERGE INTO employees AS e
USING (SELECT employee_id, name, department FROM new_employees WHERE department = 'IT') AS ne
ON e.employee_id = ne.employee_id
WHEN MATCHED THEN
    UPDATE SET e.name = ne.name, e.department = ne.department
WHEN NOT MATCHED THEN
    INSERT (employee_id, name, department) VALUES (ne.employee_id, ne.name, ne.department);

Conclusión

La declaración UPSERT (MERGE) es una forma versátil y eficiente de gestionar la sincronización de datos y asegurar la integridad de los datos en bases de datos SQL. Al combinar la funcionalidad de las operaciones INSERT y UPDATE, simplifica el proceso de mantener registros actualizados. Entender y utilizar la declaración MERGE puede mejorar significativamente tus habilidades de programación SQL y agilizar tus tareas de gestión de bases de datos.
