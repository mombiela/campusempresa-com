Introducción a FULL OUTER JOIN

El FULL OUTER JOIN es un tipo de unión en SQL que devuelve todos los registros cuando hay una coincidencia en los registros de la tabla izquierda (table1) o derecha (table2). Combina los resultados de LEFT JOIN y RIGHT JOIN. Cuando no hay coincidencia, el resultado es NULL en el lado que no tiene coincidencia.

Conceptos Clave

  • Combina Resultados: Combina los resultados de LEFT JOIN y RIGHT JOIN.
  • Valores NULL: Devuelve NULL para las columnas de la tabla que no tienen coincidencia.
  • Conjunto de Datos Completo: Útil para obtener un conjunto de datos completo de dos tablas.

Sintaxis

SELECT column_name(s)
FROM table1
FULL OUTER JOIN table2
ON table1.column_name = table2.column_name;

Escenario de Ejemplo

Considera dos tablas, Employees y Departments.

Tabla Employees

| EmployeeID | EmployeeName | DepartmentID | |------------|--------------|--------------| | 1 | Alice | 10 | | 2 | Bob | 20 | | 3 | Charlie | 30 |

Tabla Departments

| DepartmentID | DepartmentName | |--------------|----------------| | 10 | HR | | 20 | IT | | 40 | Marketing |

Consulta de Ejemplo

Para obtener una lista completa de empleados y departamentos, incluyendo aquellos sin coincidencias en la otra tabla, puedes usar un FULL OUTER JOIN.

SELECT Employees.EmployeeID, Employees.EmployeeName, Departments.DepartmentID, Departments.DepartmentName
FROM Employees
FULL OUTER JOIN Departments
ON Employees.DepartmentID = Departments.DepartmentID;

Resultado

| EmployeeID | EmployeeName | DepartmentID | DepartmentName | |------------|--------------|--------------|----------------| | 1 | Alice | 10 | HR | | 2 | Bob | 20 | IT | | 3 | Charlie | 30 | NULL | | NULL | NULL | 40 | Marketing |

Explicación

  • Alice y Bob: Ambos tienen departamentos coincidentes, por lo que su información está completamente poblada.
  • Charlie: No tiene un departamento coincidente, por lo que DepartmentID y DepartmentName son NULL.
  • Marketing: No hay empleados asignados a este departamento, por lo que EmployeeID y EmployeeName son NULL.

Ejercicio Práctico

  1. Crear Tablas: Crea las tablas Employees y Departments e inserta los datos de ejemplo.
  2. Escribir una Consulta FULL OUTER JOIN: Escribe una consulta para recuperar todos los empleados y departamentos, incluyendo aquellos sin coincidencias.
  3. Analizar Resultados: Analiza los resultados para entender cómo funciona FULL OUTER JOIN.

Código SQL para el Ejercicio

-- Crear tabla Employees
CREATE TABLE Employees (
    EmployeeID INT,
    EmployeeName VARCHAR(50),
    DepartmentID INT
);

-- Insertar datos en la tabla Employees
INSERT INTO Employees (EmployeeID, EmployeeName, DepartmentID)
VALUES (1, 'Alice', 10), (2, 'Bob', 20), (3, 'Charlie', 30);

-- Crear tabla Departments
CREATE TABLE Departments (
    DepartmentID INT,
    DepartmentName VARCHAR(50)
);

-- Insertar datos en la tabla Departments
INSERT INTO Departments (DepartmentID, DepartmentName)
VALUES (10, 'HR'), (20, 'IT'), (40, 'Marketing');

-- Escribir una consulta FULL OUTER JOIN
SELECT Employees.EmployeeID, Employees.EmployeeName, Departments.DepartmentID, Departments.DepartmentName
FROM Employees
FULL OUTER JOIN Departments
ON Employees.DepartmentID = Departments.DepartmentID;

Conclusión

El FULL OUTER JOIN es una herramienta poderosa en SQL que te permite combinar filas de dos tablas, incluyendo todos los registros de ambas tablas, y rellena con valores NULL para las coincidencias faltantes. Este tipo de unión es particularmente útil cuando necesitas una vista completa de los datos de dos tablas relacionadas, asegurando que no se omita ninguna información. Entender cómo usar FULL OUTER JOIN de manera efectiva puede mejorar en gran medida tu capacidad para realizar análisis de datos completos.
