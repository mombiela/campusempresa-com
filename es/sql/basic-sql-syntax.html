Introducción

El Lenguaje de Consulta Estructurado (SQL) es el lenguaje estándar para gestionar y manipular bases de datos relacionales. Comprender la sintaxis básica de SQL es crucial para realizar operaciones como consultar datos, actualizar registros y gestionar estructuras de bases de datos.

Conceptos clave

  • Sentencias SQL: Comandos utilizados para comunicarse con la base de datos.
  • Cláusulas: Componentes de las sentencias SQL que especifican condiciones o acciones.
  • Palabras clave: Palabras reservadas con un significado especial en SQL.
  • Identificadores: Nombres dados a objetos de la base de datos como tablas y columnas.
  • Operadores: Símbolos utilizados para especificar condiciones en las sentencias SQL.

Sentencias SQL

Las sentencias SQL son los bloques de construcción de SQL. Aquí hay algunas de las sentencias SQL más comunes:

Sentencia SELECT

La sentencia SELECT se utiliza para consultar datos de una base de datos.

SELECT column1, column2
FROM table_name;
  • SELECT: Especifica las columnas a recuperar.
  • FROM: Especifica la tabla a consultar.

Cláusula WHERE

La cláusula WHERE se utiliza para filtrar registros.

SELECT column1, column2
FROM table_name
WHERE condition;
  • WHERE: Especifica la condición para filtrar registros.

Sentencia INSERT INTO

La sentencia INSERT INTO se utiliza para agregar nuevos registros a una tabla.

INSERT INTO table_name (column1, column2)
VALUES (value1, value2);
  • INSERT INTO: Especifica la tabla y las columnas en las que insertar datos.
  • VALUES: Especifica los valores a insertar.

Sentencia UPDATE

La sentencia UPDATE se utiliza para modificar registros existentes.

UPDATE table_name
SET column1 = value1, column2 = value2
WHERE condition;
  • UPDATE: Especifica la tabla a actualizar.
  • SET: Especifica las columnas y los nuevos valores.
  • WHERE: Especifica la condición para identificar los registros a actualizar.

Sentencia DELETE

La sentencia DELETE se utiliza para eliminar registros de una tabla.

DELETE FROM table_name
WHERE condition;
  • DELETE FROM: Especifica la tabla de la que eliminar registros.
  • WHERE: Especifica la condición para identificar los registros a eliminar.

Ejemplos y ejercicios

Ejemplo 1: Seleccionar datos

Recuperar los nombres y edades de todos los empleados de la tabla employees.

SELECT name, age
FROM employees;

Ejercicio 1: Seleccionar datos

Escribe una consulta SQL para recuperar el title y el author de la tabla books.

Ejemplo 2: Filtrar datos

Recuperar los nombres de los empleados que tienen más de 30 años.

SELECT name
FROM employees
WHERE age > 30;

Ejercicio 2: Filtrar datos

Escribe una consulta SQL para recuperar el title de los libros publicados después del año 2000 de la tabla books.

Ejemplo 3: Insertar datos

Agregar un nuevo empleado a la tabla employees.

INSERT INTO employees (name, age, department)
VALUES ('John Doe', 28, 'Marketing');

Ejercicio 3: Insertar datos

Escribe una consulta SQL para agregar un nuevo libro a la tabla books con las columnas title, author y year.

Ejemplo 4: Actualizar datos

Actualizar la edad de un empleado llamado 'John Doe' a 29.

UPDATE employees
SET age = 29
WHERE name = 'John Doe';

Ejercicio 4: Actualizar datos

Escribe una consulta SQL para cambiar el year de un libro titulado 'SQL Basics' a 2021 en la tabla books.

Ejemplo 5: Eliminar datos

Eliminar un empleado llamado 'John Doe' de la tabla employees.

DELETE FROM employees
WHERE name = 'John Doe';

Ejercicio 5: Eliminar datos

Escribe una consulta SQL para eliminar un libro titulado 'Old Book' de la tabla books.

Conclusión

Comprender la sintaxis básica de SQL es esencial para interactuar con bases de datos relacionales. Al dominar las sentencias SELECT, INSERT INTO, UPDATE y DELETE, puedes realizar una amplia gama de tareas de manipulación de datos. Practica estas sentencias y sus cláusulas para construir una base sólida en SQL.
