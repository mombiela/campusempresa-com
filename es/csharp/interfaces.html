Introducción a las Interfaces

Las interfaces en C# son un concepto fundamental que te permite definir un contrato que las clases o estructuras pueden implementar. Se utilizan para especificar qué métodos y propiedades debe tener una clase, sin proporcionar los detalles de implementación.

Conceptos Clave

  • Definición: Una interfaz se define usando la palabra clave interface.
  • Implementación: Una clase o estructura implementa una interfaz proporcionando el código para cada método y propiedad definidos en la interfaz.
  • Herencia Múltiple: C# soporta la herencia múltiple de interfaces, lo que significa que una clase puede implementar múltiples interfaces.
  • Polimorfismo: Las interfaces permiten el comportamiento polimórfico, permitiéndote usar diferentes clases de manera intercambiable si implementan la misma interfaz.

Definiendo una Interfaz

Para definir una interfaz en C#, usa la palabra clave interface seguida del nombre de la interfaz. Los nombres de las interfaces típicamente comienzan con una 'I' mayúscula.

public interface IAnimal
{
    void MakeSound();
    void Move();
}

Implementando una Interfaz

Una clase o estructura implementa una interfaz usando el símbolo : seguido del nombre de la interfaz. La clase debe proporcionar implementaciones concretas para todos los métodos y propiedades definidos en la interfaz.

public class Dog : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Bark");
    }

    public void Move()
    {
        Console.WriteLine("Run");
    }
}

Usando Interfaces

Las interfaces pueden ser usadas para lograr polimorfismo. Puedes escribir código que funcione con objetos de cualquier clase que implemente una interfaz particular.

public class Program
{
    public static void Main()
    {
        IAnimal myDog = new Dog();
        myDog.MakeSound(); // Salida: Bark
        myDog.Move();      // Salida: Run
    }
}

Herencia de Interfaces

Las interfaces pueden heredar de otras interfaces. Esto te permite crear interfaces más específicas basadas en otras más generales.

public interface IAnimal
{
    void MakeSound();
    void Move();
}

public interface IPet : IAnimal
{
    void Play();
}

public class Cat : IPet
{
    public void MakeSound()
    {
        Console.WriteLine("Meow");
    }

    public void Move()
    {
        Console.WriteLine("Jump");
    }

    public void Play()
    {
        Console.WriteLine("Chase laser pointer");
    }
}

Implementación Explícita de Interfaz

A veces, una clase puede implementar múltiples interfaces que tienen métodos con el mismo nombre. En tales casos, puedes usar la implementación explícita de interfaz para evitar ambigüedades.

public interface IPrinter
{
    void Print();
}

public interface IScanner
{
    void Print();
}

public class MultiFunctionDevice : IPrinter, IScanner
{
    void IPrinter.Print()
    {
        Console.WriteLine("Printing document");
    }

    void IScanner.Print()
    {
        Console.WriteLine("Scanning document");
    }
}

Conceptos Avanzados

Métodos de Interfaz por Defecto (C# 8.0 y posteriores)

C# 8.0 introdujo los métodos de interfaz por defecto, permitiéndote proporcionar una implementación por defecto para los métodos en una interfaz.

public interface IAnimal
{
    void MakeSound();
    void Move()
    {
        Console.WriteLine("Moving...");
    }
}

public class Bird : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Chirp");
    }
}

Interfaz vs Clase Abstracta

| Característica | Interfaz | Clase Abstracta | |----------------------------|------------------------------------|-------------------------------------| | Herencia Múltiple | Sí | No | | Implementación por Defecto | C# 8.0 y posteriores | Sí | | Modificadores de Acceso | No permitidos | Permitidos | | Campos | No permitidos | Permitidos | | Constructores | No permitidos | Permitidos |

Conclusión

Las interfaces en C# son una herramienta poderosa para definir contratos y lograr polimorfismo. Te permiten escribir código flexible y reutilizable especificando qué métodos y propiedades debe implementar una clase sin dictar cómo deben ser implementados. Entender las interfaces y sus características avanzadas, como los métodos por defecto y la implementación explícita, es crucial para escribir aplicaciones robustas y mantenibles en C#.
