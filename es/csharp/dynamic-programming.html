Gestión de Memoria y Recolección de Basura ►

La Programación Dinámica (DP) es una técnica poderosa utilizada en ciencias de la computación para resolver problemas descomponiéndolos en subproblemas más simples y almacenando los resultados de estos subproblemas para evitar cálculos redundantes. Este tema te guiará desde los conceptos básicos hasta los avanzados de la Programación Dinámica en C#.

Introducción a la Programación Dinámica

  • Definición: La Programación Dinámica es un método para resolver problemas complejos descomponiéndolos en subproblemas más simples. Es aplicable cuando el problema puede dividirse en subproblemas superpuestos que pueden resolverse independientemente.
  • Conceptos Clave:
    • Subproblemas Superpuestos: Problemas que pueden descomponerse en subproblemas que se reutilizan varias veces.
    • Estructura Óptima: La solución óptima del problema puede construirse eficientemente a partir de soluciones óptimas de sus subproblemas.
    • Memoización: Almacenar los resultados de llamadas a funciones costosas y reutilizarlos cuando los mismos inputs ocurren nuevamente.
    • Tabulación: Construir una tabla de manera ascendente para almacenar los resultados de los subproblemas.

Ejemplo Básico: Secuencia de Fibonacci

La secuencia de Fibonacci es un ejemplo clásico para ilustrar la Programación Dinámica.

Enfoque Recursivo (Ineficiente)

public int Fibonacci(int n)
{
    if (n <= 1)
        return n;
    return Fibonacci(n - 1) + Fibonacci(n - 2);
}
  • Explicación: Este enfoque es simple pero ineficiente debido a los cálculos repetidos de los mismos subproblemas.

Enfoque de Memoización

public int FibonacciMemo(int n, Dictionary<int, int> memo)
{
    if (memo.ContainsKey(n))
        return memo[n];
    if (n <= 1)
        return n;
    memo[n] = FibonacciMemo(n - 1, memo) + FibonacciMemo(n - 2, memo);
    return memo[n];
}

// Uso
var memo = new Dictionary<int, int>();
int result = FibonacciMemo(10, memo);
Console.WriteLine(result); // Salida: 55
  • Explicación: Este enfoque almacena los resultados de los subproblemas en un diccionario para evitar cálculos redundantes.

Enfoque de Tabulación

public int FibonacciTab(int n)
{
    if (n <= 1)
        return n;
    int[] fib = new int[n + 1];
    fib[0] = 0;
    fib[1] = 1;
    for (int i = 2; i <= n; i++)
    {
        fib[i] = fib[i - 1] + fib[i - 2];
    }
    return fib[n];
}

// Uso
int result = FibonacciTab(10);
Console.WriteLine(result); // Salida: 55
  • Explicación: Este enfoque construye la solución de manera ascendente usando un array para almacenar los resultados de los subproblemas.

Ejemplo Intermedio: Subsecuencia Común Más Larga (LCS)

El problema de la Subsecuencia Común Más Larga es otro ejemplo clásico de Programación Dinámica.

Enfoque Recursivo (Ineficiente)

public int LCS(string s1, string s2, int m, int n)
{
    if (m == 0 || n == 0)
        return 0;
    if (s1[m - 1] == s2[n - 1])
        return 1 + LCS(s1, s2, m - 1, n - 1);
    else
        return Math.Max(LCS(s1, s2, m - 1, n), LCS(s1, s2, m, n - 1));
}
  • Explicación: Este enfoque es simple pero ineficiente debido a la complejidad de tiempo exponencial.

Enfoque de Programación Dinámica

public int LCS(string s1, string s2)
{
    int m = s1.Length;
    int n = s2.Length;
    int[,] dp = new int[m + 1, n + 1];

    for (int i = 0; i <= m; i++)
    {
        for (int j = 0; j <= n; j++)
        {
            if (i == 0 || j == 0)
                dp[i, j] = 0;
            else if (s1[i - 1] == s2[j - 1])
                dp[i, j] = dp[i - 1, j - 1] + 1;
            else
                dp[i, j] = Math.Max(dp[i - 1, j], dp[i, j - 1]);
        }
    }
    return dp[m, n];
}

// Uso
int result = LCS("AGGTAB", "GXTXAYB");
Console.WriteLine(result); // Salida: 4
  • Explicación: Este enfoque utiliza un array 2D para almacenar las longitudes de LCS de subcadenas, resultando en una complejidad de tiempo polinomial.

Ejemplo Avanzado: Problema de la Mochila

El problema de la Mochila es un conocido problema de optimización.

Enfoque Recursivo (Ineficiente)

public int Knapsack(int[] weights, int[] values, int W, int n)
{
    if (n == 0 || W == 0)
        return 0;
    if (weights[n - 1] > W)
        return Knapsack(weights, values, W, n - 1);
    else
        return Math.Max(values[n - 1] + Knapsack(weights, values, W - weights[n - 1], n - 1),
                        Knapsack(weights, values, W, n - 1));
}
  • Explicación: Este enfoque es simple pero ineficiente debido a la complejidad de tiempo exponencial.

Enfoque de Programación Dinámica

public int Knapsack(int[] weights, int[] values, int W)
{
    int n = weights.Length;
    int[,] dp = new int[n + 1, W + 1];

    for (int i = 0; i <= n; i++)
    {
        for (int w = 0; w <= W; w++)
        {
            if (i == 0 || w == 0)
                dp[i, w] = 0;
            else if (weights[i - 1] <= w)
                dp[i, w] = Math.Max(values[i - 1] + dp[i - 1, w - weights[i - 1]], dp[i - 1, w]);
            else
                dp[i, w] = dp[i - 1, w];
        }
    }
    return dp[n, W];
}

// Uso
int[] weights = { 1, 2, 3, 8, 7, 4 };
int[] values = { 20, 5, 10, 40, 15, 25 };
int W = 10;
int result = Knapsack(weights, values, W);
Console.WriteLine(result); // Salida: 60
  • Explicación: Este enfoque utiliza un array 2D para almacenar el valor máximo que se puede obtener para cada subproblema, resultando en una complejidad de tiempo polinomial.

Conclusión

La Programación Dinámica es una técnica crucial para resolver problemas de optimización de manera eficiente. Al entender y aplicar los conceptos de subproblemas superpuestos, estructura óptima, memoización y tabulación, puedes abordar una amplia gama de problemas en C#. Practica con varios problemas para fortalecer tu comprensión y competencia en Programación Dinámica.
