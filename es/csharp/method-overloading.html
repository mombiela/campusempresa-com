Introducción

La sobrecarga de métodos es un concepto fundamental en C# que te permite definir múltiples métodos con el mismo nombre pero con diferentes parámetros. Esta característica mejora la legibilidad y usabilidad del código al permitir que los métodos manejen diferentes tipos de entrada.

Conceptos Clave

  • Firma del Método: La combinación del nombre del método y su lista de parámetros.
  • Criterios de Sobrecarga: Los métodos deben diferir en el número o tipo de parámetros.
  • Polimorfismo en Tiempo de Compilación: La sobrecarga de métodos es una forma de polimorfismo en tiempo de compilación, donde el método a ejecutar se determina en tiempo de compilación.

¿Por Qué Usar la Sobrecarga de Métodos?

  • Mejora de la Legibilidad del Código: Usar el mismo nombre de método para operaciones similares hace que el código sea más fácil de leer y entender.
  • Reutilización de Código: Reduce la necesidad de crear múltiples métodos con diferentes nombres para operaciones similares.
  • Flexibilidad: Permite que los métodos manejen diferentes tipos y cantidades de entradas.

Ejemplo Básico de Sobrecarga de Métodos

using System;

class Program
{
    // Método para sumar dos enteros
    public int Add(int a, int b)
    {
        return a + b;
    }

    // Método sobrecargado para sumar tres enteros
    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }

    // Método sobrecargado para sumar dos dobles
    public double Add(double a, double b)
    {
        return a + b;
    }

    static void Main()
    {
        Program p = new Program();

        Console.WriteLine(p.Add(1, 2)); // Llama a Add(int, int)
        Console.WriteLine(p.Add(1, 2, 3)); // Llama a Add(int, int, int)
        Console.WriteLine(p.Add(1.5, 2.5)); // Llama a Add(double, double)
    }
}

Explicación

  • El método Add está sobrecargado tres veces:
    • Add(int a, int b): Suma dos enteros.
    • Add(int a, int b, int c): Suma tres enteros.
    • Add(double a, double b): Suma dos dobles.
  • El método apropiado se llama en función de los argumentos pasados.

Conceptos Avanzados

Sobrecarga con Diferentes Tipos de Parámetros

using System;

class Program
{
    // Método para concatenar dos cadenas
    public string Concatenate(string a, string b)
    {
        return a + b;
    }

    // Método sobrecargado para concatenar una cadena y un entero
    public string Concatenate(string a, int b)
    {
        return a + b.ToString();
    }

    static void Main()
    {
        Program p = new Program();

        Console.WriteLine(p.Concatenate("Hello, ", "World!")); // Llama a Concatenate(string, string)
        Console.WriteLine(p.Concatenate("Number: ", 42)); // Llama a Concatenate(string, int)
    }
}

Uso de Parámetros Opcionales y Argumentos Nombrados

Los parámetros opcionales y los argumentos nombrados a veces pueden usarse como una alternativa a la sobrecarga de métodos.

using System;

class Program
{
    // Método con parámetros opcionales
    public void DisplayInfo(string name, int age = 0, string city = "Unknown")
    {
        Console.WriteLine($"Name: {name}, Age: {age}, City: {city}");
    }

    static void Main()
    {
        Program p = new Program();

        p.DisplayInfo("Alice"); // Usa valores predeterminados para age y city
        p.DisplayInfo("Bob", 25); // Usa valor predeterminado para city
        p.DisplayInfo("Charlie", 30, "New York"); // Usa valores proporcionados para todos los parámetros
    }
}

Consideraciones de Rendimiento

  • La sobrecarga puede llevar a un código más legible y mantenible, pero la sobrecarga excesiva puede hacer que el código sea más difícil de entender.
  • Asegúrate de que los métodos sobrecargados estén lógicamente relacionados para evitar confusión.

Resumen

La sobrecarga de métodos es una característica poderosa en C# que te permite definir múltiples métodos con el mismo nombre pero con diferentes parámetros. Mejora la legibilidad, reutilización y flexibilidad del código. Al entender y aplicar la sobrecarga de métodos, puedes escribir código más eficiente y mantenible. Recuerda usar la sobrecarga con prudencia para mantener la claridad del código y evitar confusiones.
