El polimorfismo es un concepto central en la programación orientada a objetos (OOP) que permite que objetos de diferentes clases sean tratados como objetos de una superclase común. Permite que una sola interfaz represente diferentes formas subyacentes (tipos de datos).

Conceptos Clave del Polimorfismo

  • Definición: Polimorfismo significa "muchas formas" y permite que los métodos hagan cosas diferentes según el objeto sobre el que actúan.
  • Tipos de Polimorfismo:
    • Polimorfismo en Tiempo de Compilación (Sobrecarga de Métodos)
    • Polimorfismo en Tiempo de Ejecución (Sobrescritura de Métodos)
  • Beneficios:
    • Reutilización de código
    • Flexibilidad y mantenibilidad
    • Simplifica la legibilidad del código

Polimorfismo en Tiempo de Compilación (Sobrecarga de Métodos)

La Sobrecarga de Métodos permite que múltiples métodos en la misma clase tengan el mismo nombre pero diferentes parámetros.

Ejemplo de Sobrecarga de Métodos

public class MathOperations
{
    // Método para sumar dos enteros
    public int Add(int a, int b)
    {
        return a + b;
    }

    // Método para sumar tres enteros
    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }

    // Método para sumar dos dobles
    public double Add(double a, double b)
    {
        return a + b;
    }
}

class Program
{
    static void Main()
    {
        MathOperations math = new MathOperations();
        Console.WriteLine(math.Add(2, 3)); // Salida: 5
        Console.WriteLine(math.Add(2, 3, 4)); // Salida: 9
        Console.WriteLine(math.Add(2.5, 3.5)); // Salida: 6.0
    }
}

En este ejemplo, el método Add está sobrecargado con diferentes listas de parámetros.

Polimorfismo en Tiempo de Ejecución (Sobrescritura de Métodos)

La Sobrescritura de Métodos permite que una subclase proporcione una implementación específica de un método que ya está definido en su superclase.

Ejemplo de Sobrescritura de Métodos

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Animal hace un sonido");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("El perro ladra");
    }
}

public class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("El gato maúlla");
    }
}

class Program
{
    static void Main()
    {
        Animal myDog = new Dog();
        Animal myCat = new Cat();

        myDog.MakeSound(); // Salida: El perro ladra
        myCat.MakeSound(); // Salida: El gato maúlla
    }
}

En este ejemplo, el método MakeSound está sobrescrito en las clases Dog y Cat para proporcionar implementaciones específicas.

Polimorfismo con Interfaces

Las interfaces también pueden ser utilizadas para lograr polimorfismo. Una clase puede implementar múltiples interfaces, y una interfaz puede ser implementada por múltiples clases.

Ejemplo de Polimorfismo con Interfaces

public interface IShape
{
    void Draw();
}

public class Circle : IShape
{
    public void Draw()
    {
        Console.WriteLine("Dibujando un Círculo");
    }
}

public class Square : IShape
{
    public void Draw()
    {
        Console.WriteLine("Dibujando un Cuadrado");
    }
}

class Program
{
    static void Main()
    {
        IShape shape1 = new Circle();
        IShape shape2 = new Square();

        shape1.Draw(); // Salida: Dibujando un Círculo
        shape2.Draw(); // Salida: Dibujando un Cuadrado
    }
}

En este ejemplo, ambas clases Circle y Square implementan la interfaz IShape, y el método Draw se llama polimórficamente.

Resumen

El polimorfismo en C# permite flexibilidad y reutilización en el código al permitir que los objetos sean tratados como instancias de su clase padre o interfaz. Puede lograrse mediante la sobrecarga de métodos (polimorfismo en tiempo de compilación) y la sobrescritura de métodos (polimorfismo en tiempo de ejecución). Entender y utilizar el polimorfismo es esencial para escribir código orientado a objetos eficiente y mantenible.
