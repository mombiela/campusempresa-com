Entity Framework (EF) es un mapeador objeto-relacional (ORM) de código abierto para ADO.NET, que permite a los desarrolladores trabajar con una base de datos utilizando objetos .NET. Elimina la necesidad de la mayor parte del código de acceso a datos que los desarrolladores usualmente necesitan escribir.

Introducción a Entity Framework

  • ¿Qué es Entity Framework?

    • Un marco ORM para aplicaciones .NET.
    • Simplifica el acceso a datos permitiendo a los desarrolladores trabajar con datos como objetos.
    • Soporta múltiples proveedores de bases de datos (SQL Server, SQLite, MySQL, etc.).

  • ¿Por qué usar Entity Framework?

    • Reduce la cantidad de código repetitivo.
    • Proporciona un nivel más alto de abstracción para las operaciones de base de datos.
    • Soporta consultas LINQ, haciendo la manipulación de datos más intuitiva.

Configuración de Entity Framework

  • Instalación de Entity Framework

    • Usa el Administrador de Paquetes NuGet en Visual Studio.
    • Comando: Install-Package EntityFramework

  • Creación de un Modelo

    • Define tu modelo de datos utilizando clases.
    • Ejemplo: 
      public class Product
{
    public int ProductId { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
}

  • Clase DbContext

    • Actúa como un puente entre tu modelo y la base de datos.
    • Ejemplo: 
      public class ApplicationDbContext : DbContext
{
    public DbSet<Product> Products { get; set; }
}

Operaciones CRUD Básicas

  • Crear

    • Añadir una nueva entidad a la base de datos.
    • Ejemplo: 
      using (var context = new ApplicationDbContext())
{
    var product = new Product { Name = "Laptop", Price = 999.99M };
    context.Products.Add(product);
    context.SaveChanges();
}

  • Leer

    • Recuperar entidades de la base de datos.
    • Ejemplo: 
      using (var context = new ApplicationDbContext())
{
    var products = context.Products.ToList();
    foreach (var product in products)
    {
        Console.WriteLine($"Product: {product.Name}, Price: {product.Price}");
    }
}

  • Actualizar

    • Modificar una entidad existente.
    • Ejemplo: 
      using (var context = new ApplicationDbContext())
{
    var product = context.Products.First(p => p.ProductId == 1);
    product.Price = 899.99M;
    context.SaveChanges();
}

  • Eliminar

    • Eliminar una entidad de la base de datos.
    • Ejemplo: 
      using (var context = new ApplicationDbContext())
{
    var product = context.Products.First(p => p.ProductId == 1);
    context.Products.Remove(product);
    context.SaveChanges();
}

Características Avanzadas

  • Migraciones

    • Gestionar cambios en el esquema de la base de datos.
    • Comandos:
      • Añadir Migración: Add-Migration MigrationName
      • Actualizar Base de Datos: Update-Database

  • Relaciones

    • Definir relaciones entre entidades (uno a muchos, muchos a muchos).
    • Ejemplo: 
      public class Category
{
    public int CategoryId { get; set; }
    public string Name { get; set; }
    public ICollection<Product> Products { get; set; }
}

public class Product
{
    public int ProductId { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
    public int CategoryId { get; set; }
    public Category Category { get; set; }
}

  • Consultas LINQ

    • Utiliza LINQ para consultar la base de datos.
    • Ejemplo: 
      using (var context = new ApplicationDbContext())
{
    var expensiveProducts = context.Products
                                   .Where(p => p.Price > 500)
                                   .ToList();
}

  • Operaciones Asíncronas

    • Realiza operaciones de base de datos de manera asíncrona.
    • Ejemplo: 
      using (var context = new ApplicationDbContext())
{
    var products = await context.Products.ToListAsync();
}

Consideraciones de Rendimiento

  • Carga Perezosa vs. Carga Ansiosa

    • Carga Perezosa: Carga datos relacionados bajo demanda.
    • Carga Ansiosa: Carga datos relacionados como parte de la consulta inicial.
    • Ejemplo de Carga Ansiosa: 
      using (var context = new ApplicationDbContext())
{
    var products = context.Products.Include(p => p.Category).ToList();
}

  • Consultas con Seguimiento vs. Sin Seguimiento

    • Con Seguimiento: Mantiene un seguimiento de los cambios en las entidades.
    • Sin Seguimiento: Mejor rendimiento para operaciones de solo lectura.
    • Ejemplo de Consulta Sin Seguimiento: 
      using (var context = new ApplicationDbContext())
{
    var products = context.Products.AsNoTracking().ToList();
}

Conclusión

Entity Framework es un poderoso ORM que simplifica el acceso a datos en aplicaciones .NET. Al entender sus características básicas y avanzadas, puedes gestionar eficientemente tus operaciones de base de datos con un mínimo de código. Ya sea que estés realizando operaciones CRUD básicas u optimizando el rendimiento con técnicas avanzadas, Entity Framework proporciona las herramientas que necesitas para construir aplicaciones robustas y escalables.
