La herencia es un concepto fundamental en la programación orientada a objetos (OOP) que permite a una clase heredar propiedades y métodos de otra clase. Esto promueve la reutilización del código y establece una jerarquía natural entre las clases.

Conceptos Clave

  • Clase Base (Clase Padre): La clase cuyas propiedades y métodos son heredados.
  • Clase Derivada (Clase Hija): La clase que hereda de la clase base.
  • Modificador de Acceso protected: Permite que un miembro sea accesible dentro de su propia clase y por instancias de clases derivadas.
  • Palabra Clave virtual: Indica que un método o propiedad puede ser sobrescrito en una clase derivada.
  • Palabra Clave override: Se usa para extender o modificar la implementación virtual/abstracta de un método, propiedad, indexador o evento heredado.
  • Palabra Clave sealed: Impide que una clase sea heredada o que un método sea sobrescrito.

Herencia Básica

Ejemplo

// Clase base
public class Animal
{
    public string Name { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} está comiendo.");
    }
}

// Clase derivada
public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} está ladrando.");
    }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.Name = "Buddy";
        myDog.Eat();  // Método heredado
        myDog.Bark(); // Método de la clase derivada
    }
}

Explicación

  • La clase Animal es la clase base con una propiedad Name y un método Eat.
  • La clase Dog es la clase derivada que hereda de Animal y añade un nuevo método Bark.
  • En el método Main, se crea una instancia de Dog, y se llaman tanto a los métodos heredados como a los nuevos.

Modificadores de Acceso en la Herencia

Ejemplo

public class Animal
{
    protected string Name { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} está comiendo.");
    }
}

public class Dog : Animal
{
    public void SetName(string name)
    {
        Name = name; // Accediendo al miembro protegido
    }

    public void Bark()
    {
        Console.WriteLine($"{Name} está ladrando.");
    }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.SetName("Buddy");
        myDog.Eat();
        myDog.Bark();
    }
}

Explicación

  • La propiedad Name en la clase Animal está marcada como protected, haciéndola accesible dentro de la clase Dog.
  • El método SetName en la clase Dog establece la propiedad Name, demostrando el uso del modificador de acceso protected.

Sobrescritura de Métodos

Ejemplo

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Algún sonido genérico de animal.");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("¡Ladrido!");
    }
}

class Program
{
    static void Main()
    {
        Animal myAnimal = new Animal();
        myAnimal.MakeSound(); // Salida: Algún sonido genérico de animal.

        Dog myDog = new Dog();
        myDog.MakeSound(); // Salida: ¡Ladrido!
    }
}

Explicación

  • El método MakeSound en la clase Animal está marcado como virtual, permitiendo que sea sobrescrito.
  • La clase Dog sobrescribe el método MakeSound para proporcionar una implementación específica.

Clases y Métodos Sellados

Ejemplo

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Algún sonido genérico de animal.");
    }
}

public class Dog : Animal
{
    public sealed override void MakeSound()
    {
        Console.WriteLine("¡Ladrido!");
    }
}

// Esto causará un error en tiempo de compilación
public class Bulldog : Dog
{
    // No se puede sobrescribir el método sellado
    // public override void MakeSound()
    // {
    //     Console.WriteLine("¡Ladrido de bulldog!");
    // }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.MakeSound(); // Salida: ¡Ladrido!
    }
}

Explicación

  • El método MakeSound en la clase Dog está marcado como sealed, impidiendo una mayor sobrescritura en clases derivadas como Bulldog.

Conclusión

La herencia en C# permite la creación de una jerarquía de clases donde las clases derivadas heredan y extienden la funcionalidad de las clases base. Los conceptos clave incluyen el uso de modificadores de acceso, las palabras clave virtual y override, y la palabra clave sealed para controlar la herencia y la sobrescritura de métodos. Entender estos conceptos es crucial para diseñar software orientado a objetos robusto y reutilizable.
