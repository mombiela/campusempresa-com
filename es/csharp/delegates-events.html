Los delegados y eventos son conceptos fundamentales en C# que permiten a los desarrolladores escribir código flexible y reutilizable. Este tema te guiará desde los conceptos básicos de los delegados hasta el uso avanzado de los eventos.

¿Qué son los Delegados?

Los delegados son punteros de función seguros para tipos que permiten que los métodos se pasen como parámetros. Se utilizan para definir métodos de devolución de llamada e implementar el manejo de eventos.

Conceptos Clave de los Delegados

  • Seguro para Tipos: Los delegados aseguran que la firma del método coincida con la firma del delegado.
  • Multicast: Un delegado puede contener referencias a múltiples métodos.
  • Métodos Anónimos y Expresiones Lambda: Simplifican el uso de delegados.

Sintaxis Básica de los Delegados

// Declarar un delegado
public delegate void MyDelegate(string message);

// Método que coincide con la firma del delegado
public void ShowMessage(string message)
{
    Console.WriteLine(message);
}

// Usando el delegado
MyDelegate del = ShowMessage;
del("¡Hola, Mundo!");

Ejemplo: Usando Delegados

using System;

public delegate int MathOperation(int x, int y);

public class Program
{
    public static int Add(int x, int y)
    {
        return x + y;
    }

    public static int Multiply(int x, int y)
    {
        return x * y;
    }

    public static void Main()
    {
        MathOperation addOperation = Add;
        MathOperation multiplyOperation = Multiply;

        Console.WriteLine("Adición: " + addOperation(5, 3)); // Salida: 8
        Console.WriteLine("Multiplicación: " + multiplyOperation(5, 3)); // Salida: 15
    }
}

¿Qué son los Eventos?

Los eventos son una forma para que una clase notifique a otras clases u objetos cuando ocurre algo de interés. Se construyen sobre delegados y proporcionan una forma más estructurada de manejar notificaciones.

Conceptos Clave de los Eventos

  • Modelo Publicador-Suscriptor: La clase que envía la notificación se llama publicador, y las clases que reciben la notificación se llaman suscriptores.
  • Manejadores de Eventos: Métodos que manejan el evento, típicamente siguiendo la convención de void MethodName(object sender, EventArgs e).

Sintaxis Básica de los Eventos

// Declarar un delegado para el evento
public delegate void NotifyEventHandler(object sender, EventArgs e);

// Declarar un evento basado en el delegado
public event NotifyEventHandler NotifyEvent;

// Método para desencadenar el evento
protected virtual void OnNotifyEvent()
{
    if (NotifyEvent != null)
    {
        NotifyEvent(this, EventArgs.Empty);
    }
}

Ejemplo: Usando Eventos

using System;

public class Publisher
{
    // Declarar el delegado
    public delegate void NotifyEventHandler(object sender, EventArgs e);

    // Declarar el evento
    public event NotifyEventHandler NotifyEvent;

    // Método para desencadenar el evento
    public void RaiseEvent()
    {
        Console.WriteLine("El evento está a punto de ser desencadenado.");
        OnNotifyEvent();
    }

    protected virtual void OnNotifyEvent()
    {
        if (NotifyEvent != null)
        {
            NotifyEvent(this, EventArgs.Empty);
        }
    }
}

public class Subscriber
{
    public void OnNotifyEvent(object sender, EventArgs e)
    {
        Console.WriteLine("Evento recibido por el Suscriptor.");
    }
}

public class Program
{
    public static void Main()
    {
        Publisher publisher = new Publisher();
        Subscriber subscriber = new Subscriber();

        // Suscribirse al evento
        publisher.NotifyEvent += subscriber.OnNotifyEvent;

        // Desencadenar el evento
        publisher.RaiseEvent();
    }
}

Uso Avanzado de Delegados y Eventos

Delegados Multicast

Un delegado multicast puede contener referencias a más de un método. Cuando se invoca el delegado, se llaman a todos los métodos a los que hace referencia.

public delegate void MultiDelegate();

public class Program
{
    public static void Method1()
    {
        Console.WriteLine("Method1 llamado.");
    }

    public static void Method2()
    {
        Console.WriteLine("Method2 llamado.");
    }

    public static void Main()
    {
        MultiDelegate del = Method1;
        del += Method2;

        del(); // Salida: Method1 llamado. Method2 llamado.
    }
}

Accesores de Eventos

Los accesores de eventos personalizados permiten un mayor control sobre cómo se suscriben y se desuscriben los eventos.

public class CustomEvent
{
    private EventHandler _myEvent;

    public event EventHandler MyEvent
    {
        add
        {
            Console.WriteLine("Suscriptor añadido.");
            _myEvent += value;
        }
        remove
        {
            Console.WriteLine("Suscriptor eliminado.");
            _myEvent -= value;
        }
    }

    public void RaiseEvent()
    {
        _myEvent?.Invoke(this, EventArgs.Empty);
    }
}

public class Program
{
    public static void Main()
    {
        CustomEvent customEvent = new CustomEvent();
        EventHandler handler = (sender, e) => Console.WriteLine("Evento manejado.");

        customEvent.MyEvent += handler;
        customEvent.RaiseEvent();

        customEvent.MyEvent -= handler;
    }
}

Conclusión

Los delegados y eventos son características poderosas en C# que permiten un código flexible y mantenible. Los delegados permiten que los métodos se pasen como parámetros, mientras que los eventos proporcionan una forma estructurada de manejar notificaciones entre objetos. Comprender estos conceptos es crucial para desarrollar aplicaciones robustas en C#. Practica usando delegados y eventos en varios escenarios para volverte competente en su uso.
