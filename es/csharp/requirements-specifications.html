Introducción

Entender los requisitos y especificaciones es crucial para cualquier proyecto de desarrollo de software. En esta sección, cubriremos los conceptos básicos de la recopilación de requisitos, la redacción de especificaciones y cómo estos conceptos se aplican a la programación en C#.

Conceptos Clave

  • Requisitos: Estas son las necesidades o condiciones que un software debe satisfacer o poseer. Los requisitos se recopilan típicamente de los interesados.
  • Especificaciones: Estas son descripciones detalladas de la funcionalidad, comportamiento y restricciones del software. Las especificaciones se derivan de los requisitos y sirven como un plano para el desarrollo.

Recopilación de Requisitos

Tipos de Requisitos

  • Requisitos Funcionales: Definen lo que el sistema debe hacer.
    • Ejemplo: El sistema debe permitir a los usuarios iniciar sesión.
  • Requisitos No Funcionales: Definen cómo el sistema realiza una función.
    • Ejemplo: El sistema debe responder a las entradas del usuario en menos de 2 segundos.

Técnicas para Recopilar Requisitos

  • Entrevistas: Hablar con los interesados para entender sus necesidades.
  • Encuestas/Cuestionarios: Recopilar información de un grupo más grande.
  • Talleres: Sesiones colaborativas con los interesados.
  • Observación: Observar cómo los usuarios interactúan con el sistema actual.

Redacción de Especificaciones

Componentes de una Buena Especificación

  • Claridad: Las especificaciones deben ser claras y sin ambigüedades.
  • Integridad: Todos los detalles necesarios deben estar incluidos.
  • Consistencia: No debe haber información conflictiva.
  • Verificabilidad: Cada requisito debe ser verificable a través de pruebas.

Ejemplo de un Documento de Especificación

# Especificación de la Función de Inicio de Sesión de Usuario

## Requisitos Funcionales
1. El sistema permitirá a los usuarios iniciar sesión usando su correo electrónico y contraseña.
2. El sistema validará las credenciales del usuario contra la base de datos.

## Requisitos No Funcionales
1. El proceso de inicio de sesión se completará en menos de 2 segundos.
2. El sistema bloqueará la cuenta del usuario después de 5 intentos fallidos de inicio de sesión.

## Caso de Uso
### Caso de Uso: Inicio de Sesión de Usuario
- **Actor**: Usuario
- **Precondición**: El usuario debe tener una cuenta registrada.
- **Flujo Principal**:
  1. El usuario navega a la página de inicio de sesión.
  2. El usuario ingresa correo electrónico y contraseña.
  3. El sistema valida las credenciales.
  4. El sistema redirige al usuario al panel de control.
- **Postcondición**: El usuario ha iniciado sesión y es redirigido al panel de control.
- **Flujo Alternativo**:
  - Si las credenciales son inválidas, el sistema muestra un mensaje de error.
  - Si la cuenta está bloqueada, el sistema muestra un mensaje de bloqueo.

Aplicación de Requisitos y Especificaciones en C#

Ejemplo: Implementación de Inicio de Sesión de Usuario en C#

Vamos a implementar una simple función de inicio de sesión de usuario basada en la especificación anterior.

Paso 1: Definir la Clase Usuario

public class User
{
    public string Email { get; set; }
    public string Password { get; set; }
}

Paso 2: Crear una Base de Datos Simulada

public static class MockDatabase
{
    private static List<User> users = new List<User>
    {
        new User { Email = "user@example.com", Password = "password123" }
    };

    public static User GetUserByEmail(string email)
    {
        return users.FirstOrDefault(u => u.Email == email);
    }
}

Paso 3: Implementar la Función de Inicio de Sesión

public class AuthService
{
    public string Login(string email, string password)
    {
        User user = MockDatabase.GetUserByEmail(email);
        if (user == null || user.Password != password)
        {
            return "Credenciales inválidas";
        }
        return "Inicio de sesión exitoso";
    }
}

Paso 4: Probar la Función de Inicio de Sesión

public class Program
{
    public static void Main(string[] args)
    {
        AuthService authService = new AuthService();
        string result = authService.Login("user@example.com", "password123");
        Console.WriteLine(result); // Salida: Inicio de sesión exitoso

        result = authService.Login("user@example.com", "wrongpassword");
        Console.WriteLine(result); // Salida: Credenciales inválidas
    }
}

Conclusión

En esta sección, cubrimos la importancia de los requisitos y especificaciones en el desarrollo de software. Discutimos cómo recopilar requisitos, redactar especificaciones claras y completas, y demostramos cómo implementar una función en C# basada en esas especificaciones. Entender y aplicar estos conceptos ayudará a asegurar que tu software cumpla con las necesidades de sus usuarios y funcione como se espera.
