Entender el alcance y la vida útil de las variables es crucial para escribir programas en Fortran eficientes y sin errores. Este tema te guiará a través de los conceptos de alcance y vida útil de las variables, desde niveles básicos hasta avanzados.

Alcance de las Variables

El alcance de una variable se refiere a la región del programa donde la variable es accesible.

Alcance Local

  • Definición: Las variables declaradas dentro de una subrutina, función o bloque son locales a ese alcance.
  • Ejemplo: 
    program local_scope_example
    implicit none
    call my_subroutine()
contains
    subroutine my_subroutine()
        integer :: local_var
        local_var = 10
        print *, "Valor de la variable local: ", local_var
    end subroutine my_subroutine
end program local_scope_example
    En este ejemplo, local_var solo es accesible dentro de my_subroutine.

Alcance Global

  • Definición: Las variables declaradas en el programa principal o en un módulo son accesibles en todo el programa o módulo.
  • Ejemplo: 
    module global_scope_example
    implicit none
    integer :: global_var
end module global_scope_example

program use_global_scope
    use global_scope_example
    implicit none
    global_var = 20
    call print_global_var()
contains
    subroutine print_global_var()
        print *, "Valor de la variable global: ", global_var
    end subroutine print_global_var
end program use_global_scope
    Aquí, global_var es accesible tanto en el programa principal como en la subrutina print_global_var.

Vida Útil de las Variables

La vida útil de una variable se refiere a la duración durante la cual la variable retiene su valor durante la ejecución del programa.

Variables Automáticas

  • Definición: Variables que se crean y destruyen automáticamente al entrar y salir de un alcance.
  • Ejemplo: 
    program automatic_variable_example
    implicit none
    call my_subroutine()
    call my_subroutine()
contains
    subroutine my_subroutine()
        integer :: auto_var
        auto_var = auto_var + 1
        print *, "Valor de la variable automática: ", auto_var
    end subroutine my_subroutine
end program automatic_variable_example
    Cada vez que se llama a my_subroutine, auto_var se reinicializa, por lo que la salida siempre será 1.

Variables Estáticas

  • Definición: Variables que retienen su valor entre llamadas a la subrutina o función.
  • Ejemplo: 
    program static_variable_example
    implicit none
    call my_subroutine()
    call my_subroutine()
contains
    subroutine my_subroutine()
        integer, save :: static_var = 0
        static_var = static_var + 1
        print *, "Valor de la variable estática: ", static_var
    end subroutine my_subroutine
end program static_variable_example
    Aquí, static_var retiene su valor entre llamadas, por lo que la salida será 1 y luego 2.

Resumen

  • Alcance: Determina dónde se puede acceder a una variable.
    • Alcance local: Las variables son accesibles solo dentro del bloque donde se declaran.
    • Alcance global: Las variables son accesibles en todo el programa o módulo.
  • Vida útil: Determina cuánto tiempo una variable retiene su valor.
    • Variables automáticas: Se crean y destruyen con el alcance.
    • Variables estáticas: Retienen su valor entre llamadas.

Entender estos conceptos ayuda a gestionar la memoria de manera eficiente y a evitar errores comunes de programación relacionados con la accesibilidad y persistencia de las variables.
