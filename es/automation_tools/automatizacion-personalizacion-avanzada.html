Introducción

En esta sección del curso, exploraremos cómo llevar la automatización a un nivel superior mediante la personalización avanzada. Aprenderemos a utilizar herramientas y técnicas que permiten adaptar los procesos automatizados a las necesidades específicas de cada proyecto o cliente.

Conceptos Clave

  • Automatización: Uso de tecnología para realizar tareas repetitivas sin intervención humana.
  • Personalización: Adaptación de procesos y herramientas para satisfacer necesidades específicas.
  • Scripts: Programas pequeños que automatizan tareas específicas.
  • APIs: Interfaces de programación que permiten la interacción entre diferentes sistemas.
  • Workflows: Secuencias de tareas automatizadas que se ejecutan en un orden específico.

Herramientas de Automatización

Scripts

Los scripts son una forma poderosa de personalizar la automatización. Pueden ser escritos en varios lenguajes de programación como Python, Bash, PowerShell, entre otros.

Ejemplo de Script en Python

import os

def backup_files(source_dir, backup_dir):
    if not os.path.exists(backup_dir):
        os.makedirs(backup_dir)
    for filename in os.listdir(source_dir):
        full_file_name = os.path.join(source_dir, filename)
        if os.path.isfile(full_file_name):
            os.path.copy(full_file_name, backup_dir)

source_directory = '/path/to/source'
backup_directory = '/path/to/backup'

backup_files(source_directory, backup_directory)

Explicación:

  • import os: Importa el módulo os para interactuar con el sistema operativo.
  • backup_files: Función que copia archivos de un directorio a otro.
  • os.makedirs: Crea el directorio de respaldo si no existe.
  • os.listdir: Lista los archivos en el directorio de origen.
  • os.path.isfile: Verifica si el archivo es un archivo regular.
  • os.path.copy: Copia el archivo al directorio de respaldo.

APIs

Las APIs permiten la integración y automatización entre diferentes sistemas y aplicaciones.

Ejemplo de Uso de una API en Python

import requests

def get_weather(city):
    api_key = 'your_api_key'
    base_url = 'http://api.openweathermap.org/data/2.5/weather'
    complete_url = f"{base_url}?q={city}&appid={api_key}"
    response = requests.get(complete_url)
    data = response.json()
    if data['cod'] == 200:
        main = data['main']
        temperature = main['temp']
        print(f"Temperature in {city}: {temperature}K")
    else:
        print(f"City {city} not found.")

city_name = 'London'
get_weather(city_name)

Explicación:

  • import requests: Importa el módulo requests para hacer solicitudes HTTP.
  • get_weather: Función que obtiene el clima de una ciudad usando la API de OpenWeatherMap.
  • requests.get: Hace una solicitud GET a la API.
  • response.json(): Convierte la respuesta en formato JSON.
  • data['main']: Accede a la sección principal de los datos del clima.
  • data['cod']: Verifica el código de respuesta para asegurarse de que la ciudad fue encontrada.

Workflows

Los workflows permiten la ejecución de una serie de tareas automatizadas en un orden específico. Herramientas como Jenkins, GitHub Actions y Zapier son populares para crear workflows.

Ejemplo de Workflow en GitHub Actions

name: CI

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.x'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run tests
      run: |
        pytest

Explicación:

  • name: Nombre del workflow.
  • on: Evento que desencadena el workflow (en este caso, un push a la rama main).
  • jobs: Define los trabajos a realizar.
  • runs-on: Especifica el sistema operativo en el que se ejecutará el trabajo.
  • steps: Lista de pasos a seguir en el trabajo.
  • actions/checkout@v2: Acción para hacer checkout del código.
  • actions/setup-python@v2: Acción para configurar Python.
  • pip install -r requirements.txt: Instala las dependencias del proyecto.
  • pytest: Ejecuta los tests del proyecto.

Conclusión

La automatización y personalización avanzada permiten optimizar y adaptar los procesos a las necesidades específicas de cada proyecto. Mediante el uso de scripts, APIs y workflows, es posible crear soluciones automatizadas eficientes y personalizadas. La clave está en entender las herramientas disponibles y cómo combinarlas para lograr los objetivos deseados.
