Los efectos de post-procesamiento son técnicas aplicadas a la imagen final renderizada para mejorar su calidad visual. Estos efectos se utilizan comúnmente en juegos y aplicaciones gráficas para lograr varios estilos visuales y mejorar la estética general. En esta sección, exploraremos diferentes efectos de post-procesamiento, cómo implementarlos usando DirectX, y proporcionaremos ejemplos y ejercicios para solidificar tu comprensión.

Introducción a los Efectos de Post-Procesamiento

  • Definición: Los efectos de post-procesamiento son efectos visuales aplicados a la imagen final después del proceso de renderizado inicial.
  • Propósito: Mejorar la calidad visual, crear efectos artísticos y mejorar el realismo.
  • Efectos Comunes: Bloom, Desenfoque de Movimiento, Profundidad de Campo, Corrección de Color, y más.

Configuración del Post-Procesamiento en DirectX

Antes de profundizar en efectos específicos, configuremos un marco básico para aplicar efectos de post-procesamiento en DirectX.

Renderizar a Textura

Para aplicar efectos de post-procesamiento, primero necesitamos renderizar nuestra escena a una textura en lugar de directamente a la pantalla.

// Crear una textura de objetivo de renderizado
ID3D11Texture2D* renderTargetTexture = nullptr;
D3D11_TEXTURE2D_DESC textureDesc = {};
textureDesc.Width = screenWidth;
textureDesc.Height = screenHeight;
textureDesc.MipLevels = 1;
textureDesc.ArraySize = 1;
textureDesc.Format = DXGI_FORMAT_R8G8B8A8_UNORM;
textureDesc.SampleDesc.Count = 1;
textureDesc.Usage = D3D11_USAGE_DEFAULT;
textureDesc.BindFlags = D3D11_BIND_RENDER_TARGET | D3D11_BIND_SHADER_RESOURCE;

device->CreateTexture2D(&textureDesc, nullptr, &renderTargetTexture);

// Crear una vista de objetivo de renderizado
ID3D11RenderTargetView* renderTargetView = nullptr;
device->CreateRenderTargetView(renderTargetTexture, nullptr, &renderTargetView);

// Crear una vista de recurso de shader
ID3D11ShaderResourceView* shaderResourceView = nullptr;
device->CreateShaderResourceView(renderTargetTexture, nullptr, &shaderResourceView);

Aplicando un Efecto de Post-Procesamiento

Una vez que tenemos nuestra escena renderizada a una textura, podemos aplicar efectos de post-procesamiento usando shaders.

// Vincular la textura de objetivo de renderizado como entrada al shader de píxeles
context->PSSetShaderResources(0, 1, &shaderResourceView);

// Dibujar un quad de pantalla completa para aplicar el efecto de post-procesamiento
context->Draw(6, 0);

Efectos Comunes de Post-Procesamiento

Bloom

Bloom es un efecto que hace que las áreas brillantes de la imagen brillen, creando un aspecto más vibrante y dinámico.

Pasos de Implementación

  1. Extraer Áreas Brillantes: Renderizar solo las partes brillantes de la imagen a una textura separada.
  2. Desenfocar las Áreas Brillantes: Aplicar un desenfoque Gaussiano a la textura de áreas brillantes.
  3. Combinar con la Imagen Original: Agregar las áreas brillantes desenfocadas de nuevo a la imagen original.

Ejemplo de Código

// Extraer áreas brillantes
context->PSSetShader(brightExtractShader, nullptr, 0);
context->Draw(6, 0);

// Desenfocar las áreas brillantes
context->PSSetShader(gaussianBlurShader, nullptr, 0);
context->Draw(6, 0);

// Combinar con la imagen original
context->PSSetShader(bloomCombineShader, nullptr, 0);
context->Draw(6, 0);

Desenfoque de Movimiento

El desenfoque de movimiento simula el efecto de desenfoque que ocurre cuando los objetos se mueven rápidamente a través de la pantalla.

Pasos de Implementación

  1. Calcular la Velocidad: Determinar la velocidad de cada píxel basada en la diferencia entre el cuadro actual y el anterior.
  2. Aplicar Desenfoque: Usar la velocidad para desenfocar la imagen en la dirección del movimiento.

Ejemplo de Código

// Calcular velocidad
context->PSSetShader(velocityShader, nullptr, 0);
context->Draw(6, 0);

// Aplicar desenfoque de movimiento
context->PSSetShader(motionBlurShader, nullptr, 0);
context->Draw(6, 0);

Profundidad de Campo

La profundidad de campo simula el desenfoque de objetos que no están en foco, imitando el comportamiento de una lente de cámara.

Pasos de Implementación

  1. Calcular la Profundidad: Usar el búfer de profundidad para determinar la distancia de cada píxel desde la cámara.
  2. Aplicar Desenfoque: Desenfocar la imagen basada en la información de profundidad, con objetos más cercanos siendo más nítidos y objetos distantes más desenfocados.

Ejemplo de Código

// Calcular profundidad
context->PSSetShader(depthShader, nullptr, 0);
context->Draw(6, 0);

// Aplicar desenfoque de profundidad de campo
context->PSSetShader(depthOfFieldShader, nullptr, 0);
context->Draw(6, 0);

Ejercicios

  1. Implementar Efecto Bloom: Sigue los pasos y ejemplos de código proporcionados para implementar un efecto bloom en tu aplicación DirectX.
  2. Crear Desenfoque de Movimiento: Usa el ejemplo de código proporcionado para agregar desenfoque de movimiento a tu escena. Experimenta con diferentes cálculos de velocidad.
  3. Profundidad de Campo: Implementa profundidad de campo en tu aplicación. Ajusta la intensidad del desenfoque basada en diferentes rangos de profundidad.

Conclusión

Los efectos de post-procesamiento son herramientas poderosas para mejorar la calidad visual de tus aplicaciones DirectX. Al entender e implementar efectos como bloom, desenfoque de movimiento y profundidad de campo, puedes crear experiencias más inmersivas y visualmente atractivas. Practica los ejercicios proporcionados para solidificar tu comprensión y experimenta con la combinación de diferentes efectos para lograr estilos visuales únicos.
