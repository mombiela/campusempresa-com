Introducción a los Shaders

Los shaders son pequeños programas que se ejecutan en la GPU para controlar el renderizado de gráficos. En DirectX, los shaders se escriben en HLSL (High-Level Shading Language).

Conceptos Clave

  • Vertex Shader: Procesa los datos de cada vértice.
  • Pixel Shader: Determina el color de cada píxel.
  • HLSL: El lenguaje utilizado para escribir shaders en DirectX.
  • Shader Model: Define las características y capacidades de los shaders.

Escribiendo tu Primer Shader

Para comenzar con shaders en DirectX, necesitarás escribir un shader de vértices y un shader de píxeles simples.

Ejemplo de Vertex Shader

// Simple vertex shader
struct VS_INPUT {
    float4 Pos : POSITION;
    float4 Color : COLOR;
};

struct VS_OUTPUT {
    float4 Pos : SV_POSITION;
    float4 Color : COLOR;
};

VS_OUTPUT main(VS_INPUT input) {
    VS_OUTPUT output;
    output.Pos = input.Pos;
    output.Color = input.Color;
    return output;
}

Explicación:

  • VS_INPUT: Estructura que define la entrada al vertex shader.
  • VS_OUTPUT: Estructura que define la salida del vertex shader.
  • main: La función principal del vertex shader, transformando la posición de entrada y pasando el color.

Ejemplo de Pixel Shader

// Simple pixel shader
struct PS_INPUT {
    float4 Pos : SV_POSITION;
    float4 Color : COLOR;
};

float4 main(PS_INPUT input) : SV_TARGET {
    return input.Color;
}

Explicación:

  • PS_INPUT: Estructura que define la entrada al pixel shader.
  • main: La función principal del pixel shader, que devuelve el color.

Compilando y Usando Shaders en DirectX

Para usar shaders en una aplicación DirectX, necesitas compilarlos y configurarlos en la tubería de renderizado.

Compilando Shaders

ID3DBlob* vertexShaderBlob;
ID3DBlob* pixelShaderBlob;
D3DCompileFromFile(L"shader.hlsl", nullptr, nullptr, "VS", "vs_5_0", 0, 0, &vertexShaderBlob, nullptr);
D3DCompileFromFile(L"shader.hlsl", nullptr, nullptr, "PS", "ps_5_0", 0, 0, &pixelShaderBlob, nullptr);

Explicación:

  • D3DCompileFromFile: Función para compilar shaders desde un archivo.
  • "VS" y "PS": Puntos de entrada para los vertex y pixel shaders.
  • "vs_5_0" y "ps_5_0": Modelos de shader para vertex y pixel shaders.

Configurando Shaders en la Tubería

ID3D11VertexShader* vertexShader;
ID3D11PixelShader* pixelShader;
device->CreateVertexShader(vertexShaderBlob->GetBufferPointer(), vertexShaderBlob->GetBufferSize(), nullptr, &vertexShader);
device->CreatePixelShader(pixelShaderBlob->GetBufferPointer(), pixelShaderBlob->GetBufferSize(), nullptr, &pixelShader);

context->VSSetShader(vertexShader, nullptr, 0);
context->PSSetShader(pixelShader, nullptr, 0);

Explicación:

  • CreateVertexShader y CreatePixelShader: Funciones para crear objetos shader.
  • VSSetShader y PSSetShader: Funciones para configurar los shaders en la tubería de renderizado.

Técnicas Avanzadas de Shaders

Una vez que te sientas cómodo con los shaders básicos, puedes explorar técnicas más avanzadas.

Modelos de Iluminación y Sombreado

  • Phong Shading: Un modelo para simular la forma en que la luz interactúa con las superficies.
  • Blinn-Phong Shading: Una mejora sobre el sombreado Phong para un mejor rendimiento.

Ejemplo: Phong Shading

// Phong shading vertex shader
struct VS_INPUT {
    float4 Pos : POSITION;
    float3 Normal : NORMAL;
};

struct VS_OUTPUT {
    float4 Pos : SV_POSITION;
    float3 Normal : NORMAL;
};

VS_OUTPUT main(VS_INPUT input) {
    VS_OUTPUT output;
    output.Pos = mul(worldViewProj, input.Pos);
    output.Normal = mul((float3x3)world, input.Normal);
    return output;
}

// Phong shading pixel shader
float4 main(PS_INPUT input) : SV_TARGET {
    float3 lightDir = normalize(lightPos - input.Pos);
    float3 normal = normalize(input.Normal);
    float3 viewDir = normalize(viewPos - input.Pos);
    float3 reflectDir = reflect(-lightDir, normal);

    float ambient = 0.1;
    float diffuse = max(dot(lightDir, normal), 0.0);
    float specular = pow(max(dot(viewDir, reflectDir), 0.0), shininess);

    float3 color = ambient * ambientColor + diffuse * diffuseColor + specular * specularColor;
    return float4(color, 1.0);
}

Explicación:

  • Phong Shading: Calcula componentes ambientales, difusas y especulares para una iluminación realista.

Conclusión

Trabajar con shaders en DirectX implica comprender los conceptos básicos de HLSL, escribir vertex y pixel shaders, e integrarlos en tu aplicación. A medida que avances, puedes explorar técnicas de sombreado más complejas para crear gráficos realistas y visualmente atractivos. La práctica y la experimentación son clave para dominar la programación de shaders.
