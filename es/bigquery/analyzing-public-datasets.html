Introducción

Analizar conjuntos de datos públicos es una habilidad crucial para científicos de datos y analistas. Google BigQuery proporciona una plataforma poderosa para consultar y analizar grandes conjuntos de datos de manera eficiente. Este tema te guiará a través del proceso de analizar conjuntos de datos públicos usando BigQuery, desde niveles principiantes hasta avanzados.

Comenzando con BigQuery

Configurando tu Entorno

  • Crea una cuenta en Google Cloud Platform (GCP): Regístrate para una capa gratuita si no tienes una cuenta.
  • Habilita la API de BigQuery: Navega a la biblioteca de API en la consola de GCP y habilita BigQuery.
  • Crea un nuevo proyecto: Esto te ayudará a organizar tus conjuntos de datos y consultas.

Accediendo a Conjuntos de Datos Públicos

  • Conjuntos de Datos Públicos de BigQuery: Google proporciona una variedad de conjuntos de datos públicos a los que puedes acceder directamente desde BigQuery.
  • Navegador de Conjuntos de Datos: Usa la interfaz web de BigQuery para explorar los conjuntos de datos públicos disponibles.

Ejemplo: Cargando un Conjunto de Datos Público

-- Ejemplo de SQL para explorar un conjunto de datos público
SELECT 
  name, 
  birth_year 
FROM 
  `bigquery-public-data.usa_names.usa_1910_2013` 
LIMIT 10;
  • Explicación: Esta consulta selecciona las columnas name y birth_year del conjunto de datos usa_names, limitando los resultados a 10 filas.

Técnicas Básicas de Consulta

Seleccionando Datos

  • Sentencia SELECT: Se usa para especificar las columnas que deseas recuperar.
  • Cláusula FROM: Especifica el conjunto de datos y la tabla.

Filtrando Datos

  • Cláusula WHERE: Se usa para filtrar registros basados en condiciones.
SELECT 
  name, 
  birth_year 
FROM 
  `bigquery-public-data.usa_names.usa_1910_2013` 
WHERE 
  gender = 'F' 
LIMIT 10;
  • Explicación: Esta consulta filtra el conjunto de datos para incluir solo nombres femeninos.

Agregando Datos

  • Cláusula GROUP BY: Se usa para agrupar filas que tienen los mismos valores en columnas especificadas.
  • Funciones de Agregación: Como COUNT(), SUM(), AVG(), MIN(), y MAX().
SELECT 
  state, 
  COUNT(name) AS name_count 
FROM 
  `bigquery-public-data.usa_names.usa_1910_2013` 
GROUP BY 
  state 
ORDER BY 
  name_count DESC 
LIMIT 10;
  • Explicación: Esta consulta cuenta el número de nombres por estado y ordena los resultados en orden descendente.

Técnicas Intermedias de Consulta

Uniendo Tablas

  • INNER JOIN: Combina filas de dos tablas basadas en una columna relacionada.
SELECT 
  a.name, 
  b.population 
FROM 
  `bigquery-public-data.usa_names.usa_1910_2013` AS a 
INNER JOIN 
  `bigquery-public-data.census_bureau_international.population_by_country` AS b 
ON 
  a.state = b.country_code 
LIMIT 10;
  • Explicación: Esta consulta une el conjunto de datos usa_names con el conjunto de datos population_by_country en las columnas state y country_code.

Usando Subconsultas

  • Subconsulta: Una consulta anidada dentro de otra consulta.
SELECT 
  name, 
  birth_year 
FROM 
  (SELECT 
    name, 
    birth_year 
  FROM 
    `bigquery-public-data.usa_names.usa_1910_2013` 
  WHERE 
    gender = 'M') 
WHERE 
  birth_year > 2000 
LIMIT 10;
  • Explicación: Esta consulta primero filtra por nombres masculinos y luego filtra aún más por nombres con años de nacimiento después del 2000.

Técnicas Avanzadas de Consulta

Funciones de Ventana

  • Cláusula OVER: Se usa con funciones de ventana para realizar cálculos a través de un conjunto de filas de la tabla.
SELECT 
  name, 
  birth_year, 
  COUNT(*) OVER (PARTITION BY birth_year) AS birth_year_count 
FROM 
  `bigquery-public-data.usa_names.usa_1910_2013` 
LIMIT 10;
  • Explicación: Esta consulta cuenta el número de nombres para cada año de nacimiento usando una función de ventana.

Funciones Definidas por el Usuario (UDFs)

  • UDFs en JavaScript: Te permite escribir funciones personalizadas en JavaScript.
CREATE TEMP FUNCTION MultiplyByTwo(x INT64) 
RETURNS INT64 
LANGUAGE js AS """
  return x * 2;
""";

SELECT 
  name, 
  MultiplyByTwo(birth_year) AS birth_year_multiplied 
FROM 
  `bigquery-public-data.usa_names.usa_1910_2013` 
LIMIT 10;
  • Explicación: Esta consulta crea una función temporal que multiplica un número por dos y la aplica a la columna birth_year.

Conclusión

Analizar conjuntos de datos públicos con BigQuery implica entender cómo acceder a conjuntos de datos, escribir consultas SQL desde básicas hasta avanzadas, y utilizar las poderosas características de BigQuery como uniones, subconsultas, funciones de ventana y UDFs. Al dominar estas técnicas, puedes analizar grandes conjuntos de datos de manera eficiente y derivar ideas significativas.

Resumen

  • Configuración: Crea una cuenta en GCP, habilita la API de BigQuery y crea un proyecto.
  • Consultas Básicas: SELECT, WHERE, GROUP BY.
  • Consultas Intermedias: JOINs, subconsultas.
  • Consultas Avanzadas: Funciones de ventana, UDFs.
  • Práctica: Practica regularmente consultando diferentes conjuntos de datos públicos para mejorar tus habilidades.
