Introducción

En esta sección, exploraremos los conceptos de Uniones y Uniones en BigQuery. Estas son operaciones fundamentales para combinar datos de múltiples tablas. Entender cómo usarlas efectivamente es crucial para el análisis y manipulación de datos.

Uniones

Las uniones se utilizan para combinar filas de dos o más tablas basadas en una columna relacionada entre ellas. BigQuery soporta varios tipos de uniones, cada una con diferentes propósitos.

Tipos de Uniones

  • INNER JOIN: Devuelve solo las filas que tienen valores coincidentes en ambas tablas.
  • LEFT JOIN (o LEFT OUTER JOIN): Devuelve todas las filas de la tabla izquierda y las filas coincidentes de la tabla derecha. Si no se encuentra coincidencia, se devuelven valores NULL para las columnas de la tabla derecha.
  • RIGHT JOIN (o RIGHT OUTER JOIN): Devuelve todas las filas de la tabla derecha y las filas coincidentes de la tabla izquierda. Si no se encuentra coincidencia, se devuelven valores NULL para las columnas de la tabla izquierda.
  • FULL JOIN (o FULL OUTER JOIN): Devuelve todas las filas cuando hay una coincidencia en una de las tablas. Si no hay coincidencia, el resultado es NULL en el lado que no tiene coincidencia.
  • CROSS JOIN: Devuelve el producto cartesiano de las dos tablas, es decir, todas las combinaciones posibles de filas.

Sintaxis y Ejemplos

INNER JOIN

SELECT a.column1, b.column2
FROM table1 a
INNER JOIN table2 b
ON a.common_column = b.common_column;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
INNER JOIN departments
ON employees.department_id = departments.department_id;

Explicación: Esta consulta recupera los nombres de los empleados y los nombres de sus departamentos correspondientes donde hay una coincidencia en department_id.

LEFT JOIN

SELECT a.column1, b.column2
FROM table1 a
LEFT JOIN table2 b
ON a.common_column = b.common_column;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
LEFT JOIN departments
ON employees.department_id = departments.department_id;

Explicación: Esta consulta recupera todos los nombres de los empleados y los nombres de sus departamentos. Si un empleado no pertenece a ningún departamento, el nombre del departamento será NULL.

RIGHT JOIN

SELECT a.column1, b.column2
FROM table1 a
RIGHT JOIN table2 b
ON a.common_column = b.common_column;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
RIGHT JOIN departments
ON employees.department_id = departments.department_id;

Explicación: Esta consulta recupera todos los nombres de los departamentos y los nombres de los empleados en esos departamentos. Si un departamento no tiene empleados, el nombre del empleado será NULL.

FULL JOIN

SELECT a.column1, b.column2
FROM table1 a
FULL JOIN table2 b
ON a.common_column = b.common_column;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
FULL JOIN departments
ON employees.department_id = departments.department_id;

Explicación: Esta consulta recupera todos los nombres de los empleados y los nombres de los departamentos, incluyendo aquellos empleados que no pertenecen a ningún departamento y departamentos que no tienen empleados.

CROSS JOIN

SELECT a.column1, b.column2
FROM table1 a
CROSS JOIN table2 b;

Ejemplo:

SELECT employees.name, departments.department_name
FROM employees
CROSS JOIN departments;

Explicación: Esta consulta recupera el producto cartesiano de empleados y departamentos, es decir, cada combinación posible de empleado y departamento.

Uniones

Las uniones se utilizan para combinar los resultados de dos o más consultas SELECT en un solo conjunto de resultados. Las consultas deben tener el mismo número de columnas y tipos de datos compatibles en las columnas correspondientes.

Tipos de Uniones

  • UNION: Combina los resultados de dos consultas y elimina las filas duplicadas.
  • UNION ALL: Combina los resultados de dos consultas e incluye todas las filas, incluidas las duplicadas.

Sintaxis y Ejemplos

UNION

SELECT column1, column2 FROM table1
UNION
SELECT column1, column2 FROM table2;

Ejemplo:

SELECT name FROM employees
UNION
SELECT name FROM contractors;

Explicación: Esta consulta recupera una lista de nombres únicos tanto de empleados como de contratistas.

UNION ALL

SELECT column1, column2 FROM table1
UNION ALL
SELECT column1, column2 FROM table2;

Ejemplo:

SELECT name FROM employees
UNION ALL
SELECT name FROM contractors;

Explicación: Esta consulta recupera una lista de nombres tanto de empleados como de contratistas, incluidas las duplicaciones.

Conclusión

Entender y usar Uniones y Uniones en BigQuery te permite combinar y analizar datos de múltiples tablas de manera efectiva. Las uniones ayudan a fusionar filas basadas en columnas relacionadas, mientras que las uniones ayudan a apilar resultados de múltiples consultas. Dominar estas operaciones mejorará significativamente tus habilidades de manipulación de datos en BigQuery.
