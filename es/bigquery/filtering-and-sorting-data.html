Introducción

En esta sección, exploraremos cómo filtrar y ordenar datos en BigQuery. Estas operaciones son fundamentales para el análisis de datos, permitiéndote recuperar subconjuntos específicos de datos y organizarlos en un orden significativo. Cubriremos técnicas desde básicas hasta avanzadas, asegurándonos de que tengas una comprensión completa de estas habilidades esenciales.

Filtrado de Datos

Filtrado Básico con la Cláusula WHERE

La cláusula WHERE se utiliza para filtrar registros que cumplen ciertas condiciones.

Ejemplo:

SELECT *
FROM `project.dataset.table`
WHERE age > 30;
  • Esta consulta selecciona todas las columnas de la tabla donde la columna age tiene valores mayores a 30.

Combinación de Condiciones con AND/OR

Puedes combinar múltiples condiciones usando AND y OR.

Ejemplo:

SELECT *
FROM `project.dataset.table`
WHERE age > 30 AND gender = 'M';
  • Esta consulta selecciona registros donde age es mayor a 30 y gender es 'M'.

Filtrado con IN y BETWEEN

El operador IN te permite especificar múltiples valores en una cláusula WHERE, y BETWEEN selecciona valores dentro de un rango dado.

Ejemplo:

SELECT *
FROM `project.dataset.table`
WHERE country IN ('USA', 'Canada', 'UK');
  • Esta consulta selecciona registros donde el country es 'USA', 'Canada' o 'UK'.
SELECT *
FROM `project.dataset.table`
WHERE age BETWEEN 20 AND 30;
  • Esta consulta selecciona registros donde age está entre 20 y 30, inclusive.

Uso de NULL en Filtros

Manejar valores NULL es crucial en el filtrado de datos.

Ejemplo:

SELECT *
FROM `project.dataset.table`
WHERE email IS NOT NULL;
  • Esta consulta selecciona registros donde la columna email no es NULL.

Ordenación de Datos

Ordenación Básica con ORDER BY

La cláusula ORDER BY se utiliza para ordenar el conjunto de resultados por una o más columnas.

Ejemplo:

SELECT *
FROM `project.dataset.table`
ORDER BY age;
  • Esta consulta ordena los registros por la columna age en orden ascendente.

Ordenación en Orden Descendente

Para ordenar en orden descendente, usa la palabra clave DESC.

Ejemplo:

SELECT *
FROM `project.dataset.table`
ORDER BY age DESC;
  • Esta consulta ordena los registros por la columna age en orden descendente.

Ordenación por Múltiples Columnas

Puedes ordenar por múltiples columnas especificándolas en la cláusula ORDER BY.

Ejemplo:

SELECT *
FROM `project.dataset.table`
ORDER BY age DESC, name ASC;
  • Esta consulta ordena los registros primero por age en orden descendente y luego por name en orden ascendente.

Filtrado y Ordenación Avanzados

Filtrado con Subconsultas

Las subconsultas pueden ser utilizadas dentro de la cláusula WHERE para un filtrado más complejo.

Ejemplo:

SELECT *
FROM `project.dataset.table`
WHERE age > (SELECT AVG(age) FROM `project.dataset.table`);
  • Esta consulta selecciona registros donde age es mayor que la edad promedio de todos los registros en la tabla.

Filtrado con Expresiones Regulares

BigQuery soporta el filtrado con expresiones regulares usando la función REGEXP_CONTAINS.

Ejemplo:

SELECT *
FROM `project.dataset.table`
WHERE REGEXP_CONTAINS(email, r'@gmail\.com$');
  • Esta consulta selecciona registros donde la columna email termina con '@gmail.com'.

Ordenación con Expresiones Complejas

Puedes ordenar datos usando expresiones complejas en la cláusula ORDER BY.

Ejemplo:

SELECT name, age, salary
FROM `project.dataset.table`
ORDER BY (salary / age) DESC;
  • Esta consulta ordena los registros por el resultado de la expresión (salary / age) en orden descendente.

Conclusión

El filtrado y la ordenación son operaciones esenciales en el análisis de datos con BigQuery. Al dominar estas técnicas, puedes recuperar y organizar datos de manera eficiente para obtener valiosas ideas. Practica estos conceptos con varios conjuntos de datos para volverte competente en la escritura de consultas efectivas en BigQuery.
