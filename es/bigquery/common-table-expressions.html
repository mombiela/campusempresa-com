Introducción a las CTEs

Las Expresiones de Tabla Comunes (CTEs) son una característica poderosa en SQL que te permite crear conjuntos de resultados temporales que pueden ser referenciados dentro de una declaración SELECT, INSERT, UPDATE o DELETE. En BigQuery, las CTEs pueden ayudar a simplificar consultas complejas, mejorar la legibilidad y hacer que tu código SQL sea más mantenible.

Conceptos Clave

  • Definición de CTE: Una CTE se define usando la palabra clave WITH seguida de una consulta que define el conjunto de resultados temporales.
  • Ámbito: La CTE solo está disponible dentro del ámbito de ejecución de la declaración en la que se define.
  • Múltiples CTEs: Puedes definir múltiples CTEs en una sola consulta, y pueden referenciarse entre sí.

Sintaxis Básica de las CTEs

La sintaxis básica para una CTE en BigQuery es la siguiente:

WITH cte_name AS (
    SELECT column1, column2, ...
    FROM table_name
    WHERE condition
)
SELECT column1, column2, ...
FROM cte_name;

Ejemplo

Comencemos con un ejemplo simple. Supongamos que tenemos una tabla llamada sales con las siguientes columnas: sale_id, product_id, quantity y sale_date.

WITH recent_sales AS (
    SELECT sale_id, product_id, quantity, sale_date
    FROM sales
    WHERE sale_date > '2023-01-01'
)
SELECT *
FROM recent_sales;

En este ejemplo:

  • Definimos una CTE llamada recent_sales que selecciona todas las ventas realizadas después del 1 de enero de 2023.
  • Luego seleccionamos todas las columnas de la CTE recent_sales.

Uso Avanzado de las CTEs

CTEs Recursivas

Las CTEs recursivas se utilizan para realizar operaciones que requieren procesamiento iterativo, como recorrer estructuras de datos jerárquicas.

Sintaxis

WITH RECURSIVE cte_name AS (
    initial_query
    UNION ALL
    recursive_query
)
SELECT *
FROM cte_name;

Ejemplo

Considera una tabla employees con columnas employee_id, manager_id y employee_name. Queremos listar todos los empleados y sus gerentes en un orden jerárquico.

WITH RECURSIVE employee_hierarchy AS (
    SELECT employee_id, manager_id, employee_name, 1 AS level
    FROM employees
    WHERE manager_id IS NULL
    UNION ALL
    SELECT e.employee_id, e.manager_id, e.employee_name, eh.level + 1
    FROM employees e
    JOIN employee_hierarchy eh ON e.manager_id = eh.employee_id
)
SELECT *
FROM employee_hierarchy;

En este ejemplo:

  • La consulta inicial selecciona a todos los gerentes de nivel superior (donde manager_id es NULL).
  • La consulta recursiva une la tabla employees con la CTE employee_hierarchy para encontrar empleados que reportan a cada gerente.

Aplicaciones Prácticas de las CTEs

Simplificación de Consultas Complejas

Las CTEs pueden descomponer consultas complejas en partes más simples y manejables.

Ejemplo

Supongamos que queremos encontrar las ventas totales para cada producto en la tabla sales y luego filtrar productos con ventas totales mayores a 1000.

WITH product_sales AS (
    SELECT product_id, SUM(quantity) AS total_sales
    FROM sales
    GROUP BY product_id
)
SELECT product_id, total_sales
FROM product_sales
WHERE total_sales > 1000;

Mejora de la Legibilidad

Las CTEs pueden hacer que tu código SQL sea más legible al dar nombres significativos a los conjuntos de resultados intermedios.

Ejemplo

Considera una consulta que calcula las ventas promedio por producto y luego encuentra productos con ventas superiores al promedio.

WITH avg_sales AS (
    SELECT AVG(total_sales) AS avg_total_sales
    FROM (
        SELECT product_id, SUM(quantity) AS total_sales
        FROM sales
        GROUP BY product_id
    )
),
above_avg_sales AS (
    SELECT product_id, SUM(quantity) AS total_sales
    FROM sales
    GROUP BY product_id
    HAVING SUM(quantity) > (SELECT avg_total_sales FROM avg_sales)
)
SELECT *
FROM above_avg_sales;

Conclusión

Las Expresiones de Tabla Comunes (CTEs) son una herramienta versátil en BigQuery que puede ayudar a simplificar consultas complejas, mejorar la legibilidad y gestionar datos jerárquicos. Al usar CTEs, puedes descomponer tus consultas SQL en partes más manejables, haciendo que tu código sea más fácil de entender y mantener. Ya sea que estés trabajando con CTEs simples o recursivas, dominar esta característica mejorará significativamente tus capacidades de consulta en SQL.
