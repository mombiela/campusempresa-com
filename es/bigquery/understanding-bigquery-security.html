Introducción a la Seguridad de BigQuery

BigQuery es un almacén de datos totalmente gestionado y sin servidor que permite un análisis escalable sobre petabytes de datos. La seguridad es un aspecto crítico de la gestión y el uso de BigQuery, asegurando que tus datos estén protegidos y que el acceso esté controlado.

Conceptos Clave en la Seguridad de BigQuery

  • Gestión de Identidad y Acceso (IAM): Controla quién puede hacer qué en qué recursos.
  • Cifrado de Datos: Protege los datos en reposo y en tránsito.
  • Seguridad de la Red: Asegura la comunicación segura entre clientes y BigQuery.
  • Registro de Auditoría: Rastrea el acceso y las modificaciones a los recursos de BigQuery.

Gestión de Identidad y Acceso (IAM)

IAM es un marco de políticas y tecnologías para asegurar que los usuarios correctos tengan el acceso apropiado a los recursos tecnológicos.

Roles de IAM

BigQuery utiliza roles de IAM para gestionar el acceso a los recursos:

  • Roles Primitivos: Roles básicos como Viewer, Editor y Owner.
  • Roles Predefinidos: Roles más granulares como bigquery.dataViewer, bigquery.dataEditor y bigquery.admin.
  • Roles Personalizados: Roles definidos por el usuario adaptados a necesidades específicas.

Ejemplo: Asignar un Rol a un Usuario

gcloud projects add-iam-policy-binding [PROJECT_ID] \
    --member='user:[USER_EMAIL]' \
    --role='roles/bigquery.dataViewer'

Explicación:

  • [PROJECT_ID]: Tu ID de proyecto de Google Cloud.
  • [USER_EMAIL]: El correo electrónico del usuario al que deseas asignar el rol.
  • roles/bigquery.dataViewer: El rol que se está asignando.

Cifrado de Datos

BigQuery cifra los datos en reposo y en tránsito para protegerlos del acceso no autorizado.

Cifrado en Reposo

  • Claves de cifrado gestionadas por Google: Cifrado predeterminado gestionado por Google.
  • Claves de cifrado gestionadas por el cliente (CMEK): Permite a los clientes gestionar sus propias claves de cifrado utilizando el Servicio de Gestión de Claves (KMS) de Cloud.

Ejemplo: Crear un Conjunto de Datos con CMEK

CREATE SCHEMA my_dataset
OPTIONS(
  kms_key_name="projects/[PROJECT_ID]/locations/[LOCATION]/keyRings/[KEY_RING]/cryptoKeys/[KEY]"
);

Explicación:

  • [PROJECT_ID], [LOCATION], [KEY_RING], [KEY]: Reemplaza con los detalles específicos de tu proyecto y la información de la clave.

Cifrado en Tránsito

BigQuery utiliza TLS (Transport Layer Security) para cifrar los datos mientras se mueven entre tus dispositivos y los servidores de Google.

Seguridad de la Red

La seguridad de la red en BigQuery implica asegurar los canales de comunicación y garantizar que los datos sean accedidos solo por redes autorizadas.

Controles de Servicio VPC

Los Controles de Servicio de la Nube Privada Virtual (VPC) proporcionan seguridad adicional para los datos al definir un perímetro de seguridad alrededor de los recursos de Google Cloud.

Ejemplo: Configurar Controles de Servicio VPC

  1. Crear un Perímetro de Servicio: 
    gcloud access-context-manager perimeters create [PERIMETER_NAME] 
            --title="[PERIMETER_TITLE]" 
            --resources="projects/[PROJECT_ID]"
  2. Agregar BigQuery al Perímetro: 
    gcloud access-context-manager perimeters update [PERIMETER_NAME] 
            --add-restricted-services="bigquery.googleapis.com"

Registro de Auditoría

Los registros de auditoría proporcionan un registro de las acciones realizadas en los recursos de BigQuery, lo cual es esencial para el monitoreo de seguridad y el cumplimiento.

Tipos de Registros de Auditoría

  • Registros de Actividad Administrativa: Registran acciones administrativas.
  • Registros de Acceso a Datos: Registran operaciones de lectura y escritura de datos.
  • Registros de Eventos del Sistema: Registran eventos del sistema.

Ejemplo: Habilitar Registros de Auditoría

  1. Navega al Enrutador de Registros en la Consola de Google Cloud.
  2. Crea un sumidero para enrutar los registros a un destino (por ejemplo, Cloud Storage, BigQuery o Pub/Sub).
gcloud logging sinks create [SINK_NAME] bigquery.googleapis.com/projects/[PROJECT_ID]/datasets/[DATASET_ID] \
    --log-filter='resource.type="bigquery_resource"'

Explicación:

  • [SINK_NAME]: El nombre del sumidero.
  • [PROJECT_ID], [DATASET_ID]: Tus IDs de proyecto y conjunto de datos.
  • --log-filter: Filtra los registros para incluir solo los registros de recursos de BigQuery.

Conclusión

Entender e implementar la seguridad de BigQuery es crucial para proteger tus datos y asegurar el cumplimiento con los requisitos organizacionales y regulatorios. Al aprovechar los roles de IAM, el cifrado de datos, la seguridad de la red y el registro de auditoría, puedes crear una postura de seguridad robusta para tu entorno de BigQuery.
