Introducción al Cifrado de Datos

El cifrado de datos es un aspecto fundamental de la seguridad de la información, asegurando que la información sensible esté protegida contra el acceso no autorizado. En BigQuery, el cifrado se utiliza para proteger los datos en reposo y en tránsito.

Conceptos Clave

  • Cifrado: El proceso de convertir datos en un formato codificado que solo puede ser leído por alguien que tenga la clave de descifrado.
  • Descifrado: El proceso de convertir datos cifrados de nuevo a su formato original.
  • Cifrado en Reposo: Protección de datos almacenados en disco.
  • Cifrado en Tránsito: Protección de datos mientras se mueven a través de redes.
  • Claves de Cifrado Gestionadas por el Cliente (CMEK): Claves gestionadas por el cliente para cifrar datos.
  • Claves de Cifrado Gestionadas por Google (GMEK): Claves gestionadas por Google para cifrar datos.

Cifrado en Reposo

BigQuery cifra automáticamente todos los datos en reposo utilizando claves de cifrado gestionadas por Google. Sin embargo, para un control adicional, puedes usar Claves de Cifrado Gestionadas por el Cliente (CMEK).

Uso de Claves de Cifrado Gestionadas por Google (GMEK)

Por defecto, BigQuery utiliza GMEK para cifrar datos en reposo. Este proceso es transparente para el usuario y no requiere configuración adicional.

Uso de Claves de Cifrado Gestionadas por el Cliente (CMEK)

CMEK te permite usar tus propias claves de cifrado almacenadas en Google Cloud Key Management Service (KMS).

Pasos para Usar CMEK en BigQuery

  1. Crear un Anillo de Claves y una Clave en Google Cloud KMS:

    gcloud kms keyrings create my-key-ring --location=global
gcloud kms keys create my-key --location=global --keyring=my-key-ring --purpose=encryption

  2. Conceder Acceso a la Clave a la Cuenta de Servicio de BigQuery:

    gcloud kms keys add-iam-policy-binding my-key 
            --location=global 
            --keyring=my-key-ring 
            --member=serviceAccount:your-bigquery-service-account 
            --role=roles/cloudkms.cryptoKeyEncrypterDecrypter

  3. Crear un Conjunto de Datos de BigQuery con CMEK:

    CREATE SCHEMA my_dataset
OPTIONS(
    kms_key_name="projects/my-project/locations/global/keyRings/my-key-ring/cryptoKeys/my-key"
);

Cifrado en Tránsito

BigQuery cifra los datos en tránsito utilizando TLS (Transport Layer Security) para asegurar la integridad y privacidad de los datos mientras se mueven entre tus dispositivos y los servidores de Google.

Asegurando Conexiones Seguras

  • Siempre usa HTTPS para interactuar con BigQuery.
  • Usa APIs seguras y bibliotecas cliente proporcionadas por Google Cloud.

Técnicas Avanzadas de Cifrado

Para usuarios avanzados, BigQuery soporta características adicionales de cifrado como:

  • Cifrado a Nivel de Columna: Cifrado de columnas específicas dentro de un conjunto de datos.
  • Cifrado Híbrido: Combinación de cifrado simétrico y asimétrico para una seguridad mejorada.

Ejemplo: Cifrado a Nivel de Columna

Para cifrar columnas específicas, puedes usar funciones como AEAD.ENCRYPT y AEAD.DECRYPT.

Cifrando una Columna

DECLARE key BYTES = FROM_HEX('your-hex-encoded-key');
DECLARE plaintext STRING = 'Datos Sensibles';

SET encrypted_data = AEAD.ENCRYPT(key, plaintext, '');

Descifrando una Columna

DECLARE key BYTES = FROM_HEX('your-hex-encoded-key');
DECLARE ciphertext BYTES = 'your-encrypted-data';

SET decrypted_data = AEAD.DECRYPT(key, ciphertext, '');

Resumen

El cifrado de datos en BigQuery es un componente crítico de la seguridad de datos, asegurando que tus datos estén protegidos tanto en reposo como en tránsito. Al comprender y utilizar tanto las claves de cifrado gestionadas por Google como las gestionadas por el cliente, puedes mejorar la seguridad de tus conjuntos de datos. Además, técnicas avanzadas de cifrado como el cifrado a nivel de columna proporcionan una mayor granularidad y control sobre la protección de datos.

Siguiendo las mejores prácticas y aprovechando las características de cifrado proporcionadas por BigQuery, puedes asegurarte de que tus datos permanezcan seguros y cumplan con los estándares de la industria.
