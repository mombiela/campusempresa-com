Descripción general

En esta sección, introduciremos SQL (Lenguaje de Consulta Estructurado) y su aplicación en Google BigQuery. Esto servirá como la base para entender cómo interactuar y manipular datos dentro de BigQuery.

¿Qué es SQL?

SQL es un lenguaje estándar para acceder y manipular bases de datos. Se utiliza para realizar tareas como consultar datos, actualizar registros y gestionar estructuras de bases de datos.

Conceptos clave

  • Base de datos: Una colección estructurada de datos.
  • Tabla: Una colección de entradas de datos relacionadas que consisten en filas y columnas.
  • Consulta: Una solicitud de datos o información de una base de datos.

Sintaxis básica de SQL

Comencemos con algunos comandos básicos de SQL que son esenciales para trabajar con BigQuery.

Instrucción SELECT

La instrucción SELECT se utiliza para consultar datos de una base de datos.

SELECT columna1, columna2, ...
FROM nombre_tabla;

Ejemplo:

SELECT nombre, edad
FROM empleados;

Cláusula WHERE

La cláusula WHERE se utiliza para filtrar registros.

SELECT columna1, columna2, ...
FROM nombre_tabla
WHERE condición;

Ejemplo:

SELECT nombre, edad
FROM empleados
WHERE edad > 30;

Cláusula ORDER BY

La cláusula ORDER BY se utiliza para ordenar el conjunto de resultados.

SELECT columna1, columna2, ...
FROM nombre_tabla
ORDER BY columna1 [ASC|DESC];

Ejemplo:

SELECT nombre, edad
FROM empleados
ORDER BY edad DESC;

Cláusula LIMIT

La cláusula LIMIT se utiliza para especificar el número de registros a devolver.

SELECT columna1, columna2, ...
FROM nombre_tabla
LIMIT número;

Ejemplo:

SELECT nombre, edad
FROM empleados
LIMIT 5;

SQL en BigQuery

BigQuery utiliza SQL estándar (también conocido como BigQuery SQL) para consultar datos. Aquí hay algunas características específicas y diferencias al usar SQL en BigQuery.

Características clave

  • Escalabilidad: BigQuery puede manejar grandes conjuntos de datos de manera eficiente.
  • Rendimiento: Optimizado para un rendimiento rápido de consultas.
  • Integración: Se integra fácilmente con otros servicios de Google Cloud.

Ejemplo de consulta en BigQuery

SELECT nombre, edad
FROM `proyecto.conjunto_datos.empleados`
WHERE edad > 30
ORDER BY edad DESC
LIMIT 5;

Explicación:

  • Proyecto: El ID del proyecto de Google Cloud.
  • Conjunto de datos: El conjunto de datos dentro del proyecto.
  • Tabla: La tabla dentro del conjunto de datos.

Funciones comunes de SQL en BigQuery

BigQuery proporciona una variedad de funciones integradas para realizar operaciones en datos.

Funciones agregadas

  • COUNT(): Devuelve el número de filas.
  • SUM(): Devuelve la suma de una columna numérica.
  • AVG(): Devuelve el valor promedio de una columna numérica.
  • MAX(): Devuelve el valor máximo.
  • MIN(): Devuelve el valor mínimo.

Ejemplo:

SELECT COUNT(*), AVG(edad), MAX(edad), MIN(edad)
FROM `proyecto.conjunto_datos.empleados`;

Funciones de cadena

  • UPPER(): Convierte una cadena a mayúsculas.
  • LOWER(): Convierte una cadena a minúsculas.
  • CONCAT(): Concatenar dos o más cadenas.

Ejemplo:

SELECT UPPER(nombre), LOWER(nombre), CONCAT(nombre, ' ', apellido) AS nombre_completo
FROM `proyecto.conjunto_datos.empleados`;

Funciones de fecha

  • CURRENT_DATE(): Devuelve la fecha actual.
  • DATE_ADD(): Añade un intervalo de tiempo especificado a una fecha.
  • FORMAT_DATE(): Formatea una fecha según un formato especificado.

Ejemplo:

SELECT CURRENT_DATE(), DATE_ADD(CURRENT_DATE(), INTERVAL 7 DAY) AS próxima_semana
FROM `proyecto.conjunto_datos.empleados`;

Conclusión

En esta sección, cubrimos los conceptos básicos de SQL y su aplicación en BigQuery. Discutimos comandos fundamentales de SQL como SELECT, WHERE, ORDER BY y LIMIT, y exploramos cómo usar estos comandos en BigQuery. Además, presentamos algunas funciones comunes de SQL disponibles en BigQuery. Este conocimiento fundamental será crucial a medida que avancemos a temas más avanzados en BigQuery.
