Introducción a las Funciones Definidas por el Usuario (UDFs)

Las Funciones Definidas por el Usuario (UDFs) en BigQuery te permiten extender las capacidades de SQL escribiendo funciones personalizadas usando JavaScript o SQL. Las UDFs pueden ser utilizadas para encapsular lógica compleja, realizar cálculos personalizados y reutilizar código en múltiples consultas.

Conceptos Clave

  • Tipos de UDFs: BigQuery soporta dos tipos de UDFs:
    • SQL UDFs: Escritas en SQL.
    • JavaScript UDFs: Escritas en JavaScript.
  • Sintaxis: La sintaxis para crear UDFs varía ligeramente entre SQL y JavaScript.
  • Reusabilidad: Las UDFs pueden ser reutilizadas en múltiples consultas, haciendo tu código más modular y mantenible.

Creación de SQL UDFs

Las SQL UDFs se escriben utilizando la sintaxis estándar de SQL y pueden ser usadas para encapsular lógica SQL.

Ejemplo: Simple SQL UDF

Vamos a crear una simple SQL UDF que calcula el cuadrado de un número.

CREATE OR REPLACE FUNCTION my_dataset.square(x FLOAT64) 
RETURNS FLOAT64 AS (
  x * x
);

Explicación

  • CREATE OR REPLACE FUNCTION: Esta declaración crea una nueva función o reemplaza una existente.
  • my_dataset.square: El nombre de la función, donde my_dataset es el conjunto de datos que contiene la función.
  • x FLOAT64: El parámetro de entrada y su tipo.
  • RETURNS FLOAT64: El tipo de retorno de la función.
  • AS ( x * x ): El cuerpo de la función, que contiene la expresión SQL a ejecutar.

Uso de la SQL UDF

Puedes usar la UDF en una consulta como cualquier otra función.

SELECT my_dataset.square(3) AS result;

Salida

| result | |--------| | 9 |

Creación de JavaScript UDFs

Las JavaScript UDFs te permiten escribir funciones personalizadas usando JavaScript, lo cual puede ser útil para lógica más compleja que es difícil de expresar en SQL.

Ejemplo: Simple JavaScript UDF

Vamos a crear una JavaScript UDF que invierte una cadena.

CREATE OR REPLACE FUNCTION my_dataset.reverse_string(input STRING) 
RETURNS STRING 
LANGUAGE js AS """
  return input.split('').reverse().join('');
""";

Explicación

  • CREATE OR REPLACE FUNCTION: Esta declaración crea una nueva función o reemplaza una existente.
  • my_dataset.reverse_string: El nombre de la función.
  • input STRING: El parámetro de entrada y su tipo.
  • RETURNS STRING: El tipo de retorno de la función.
  • LANGUAGE js: Especifica que la función está escrita en JavaScript.
  • AS """ ... """: El cuerpo de la función, escrito en JavaScript.

Uso de la JavaScript UDF

Puedes usar la UDF en una consulta como cualquier otra función.

SELECT my_dataset.reverse_string('BigQuery') AS reversed;

Salida

| reversed | |----------| | yreuQgiB |

Uso Avanzado de UDFs

Manejo de Múltiples Parámetros

Las UDFs pueden aceptar múltiples parámetros. Aquí hay un ejemplo de una SQL UDF que concatena dos cadenas.

CREATE OR REPLACE FUNCTION my_dataset.concat_strings(str1 STRING, str2 STRING) 
RETURNS STRING AS (
  CONCAT(str1, ' ', str2)
);

Uso de UDFs con Arrays

También puedes crear UDFs que trabajen con arrays. Aquí hay un ejemplo de una JavaScript UDF que suma los elementos de un array.

CREATE OR REPLACE FUNCTION my_dataset.sum_array(arr ARRAY<INT64>) 
RETURNS INT64 
LANGUAGE js AS """
  return arr.reduce((a, b) => a + b, 0);
""";

Consideraciones de Rendimiento

  • Tiempo de Ejecución: Las UDFs pueden introducir tiempo adicional de ejecución, especialmente si contienen lógica compleja.
  • Uso de Recursos: Ten en cuenta los recursos que tus UDFs consumen, ya que pueden impactar el rendimiento general de tus consultas.

Conclusión

Las Funciones Definidas por el Usuario (UDFs) en BigQuery proporcionan una forma poderosa de extender las capacidades de SQL con lógica personalizada. Ya sea que uses SQL o JavaScript, las UDFs pueden ayudarte a encapsular lógica compleja, realizar cálculos personalizados y hacer tus consultas más modulares y mantenibles. Al entender cómo crear y usar UDFs efectivamente, puedes aprovechar todo el poder de BigQuery para tus tareas de análisis de datos.
