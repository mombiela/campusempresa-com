En esta sección, cubriremos cómo implementar un proyecto usando Ansible de principio a fin. Esto incluirá configurar tu entorno, escribir playbooks y ejecutarlos para lograr la configuración deseada. Al final de esta sección, deberías ser capaz de implementar un proyecto con confianza usando Ansible.

Configuración de Tu Entorno

Antes de comenzar a escribir playbooks de Ansible, necesitas configurar tu entorno. Esto incluye instalar Ansible y configurar tu inventario.

Instalación de Ansible

Para instalar Ansible, puedes usar el gestor de paquetes de tu sistema operativo. Aquí están los comandos para algunos sistemas comunes:

  • Ubuntu/Debian:

    sudo apt update
sudo apt install ansible

  • CentOS/RHEL:

    sudo yum install epel-release
sudo yum install ansible

  • macOS:

    brew install ansible

Configuración de Tu Inventario

Ansible usa un archivo de inventario para gestionar los hosts que configurará. El archivo de inventario predeterminado se encuentra en /etc/ansible/hosts, pero puedes especificar un archivo diferente usando la opción -i.

Ejemplo de un archivo de inventario:

[webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com

Escribiendo Tu Primer Playbook

Un playbook es un archivo YAML que describe las tareas a ejecutar en tus hosts. Escribamos un playbook simple para instalar y arrancar el servidor web Apache en nuestros servidores web.

Ejemplo de Playbook

---
- name: Instalar y arrancar Apache
  hosts: webservers
  become: yes
  tasks:
    - name: Instalar Apache
      apt:
        name: apache2
        state: present
      when: ansible_os_family == "Debian"

    - name: Instalar Apache
      yum:
        name: httpd
        state: present
      when: ansible_os_family == "RedHat"

    - name: Arrancar Apache
      service:
        name: "{{ ansible_os_family == 'Debian' | ternary('apache2', 'httpd') }}"
        state: started

Explicación

  • name: Una descripción del playbook o tarea.
  • hosts: El grupo de hosts del archivo de inventario en el que se ejecutará este playbook.
  • become: Si se debe usar sudo para ejecutar tareas.
  • tasks: Una lista de tareas a ejecutar.

Cada tarea tiene:

  • name: Una descripción de la tarea.
  • module: El módulo de Ansible a usar (por ejemplo, apt, yum, service).
  • arguments: Los argumentos para el módulo.

Ejecutando Tu Playbook

Para ejecutar tu playbook, usa el comando ansible-playbook seguido de la ruta a tu archivo de playbook.

ansible-playbook -i inventory.ini install_apache.yml

Opciones Comunes

  • -i: Especificar el archivo de inventario.
  • -u: Especificar el usuario remoto.
  • --ask-become-pass: Solicitar la contraseña de sudo.

Características Avanzadas del Playbook

A medida que te sientas más cómodo con Ansible, puedes comenzar a usar características más avanzadas en tus playbooks.

Variables

Las variables te permiten hacer que tus playbooks sean más flexibles y reutilizables.

Ejemplo:

---
- name: Instalar y arrancar Apache
  hosts: webservers
  become: yes
  vars:
    apache_package: "{{ ansible_os_family == 'Debian' | ternary('apache2', 'httpd') }}"
  tasks:
    - name: Instalar Apache
      package:
        name: "{{ apache_package }}"
        state: present

    - name: Arrancar Apache
      service:
        name: "{{ apache_package }}"
        state: started

Manejadores

Los manejadores son tareas que solo se ejecutan cuando son notificadas por otras tareas. Son útiles para acciones como reiniciar un servicio después de un cambio de configuración.

Ejemplo:

---
- name: Instalar y configurar Apache
  hosts: webservers
  become: yes
  tasks:
    - name: Instalar Apache
      package:
        name: apache2
        state: present
      notify: Reiniciar Apache

    - name: Configurar Apache
      template:
        src: apache.conf.j2
        dest: /etc/apache2/sites-available/000-default.conf
      notify: Reiniciar Apache

  handlers:
    - name: Reiniciar Apache
      service:
        name: apache2
        state: restarted

Conclusión

En esta sección, cubrimos los pasos esenciales para implementar un proyecto usando Ansible. Comenzamos con la configuración del entorno, escribiendo un playbook básico y ejecutándolo. También tocamos características avanzadas como variables y manejadores para hacer tus playbooks más poderosos y flexibles. Con estas habilidades, ahora estás listo para comenzar a implementar tus propios proyectos usando Ansible.
