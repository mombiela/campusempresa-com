Introducción a Ansible

Ansible es una herramienta de automatización de código abierto utilizada para la gestión de configuraciones, despliegue de aplicaciones y automatización de tareas. Es sin agente, lo que significa que no requiere ningún software instalado en los nodos que gestiona. Esto lo convierte en una opción popular para la automatización de TI.

Conceptos Clave

  • Playbooks: Archivos YAML que definen una serie de tareas a ejecutar en un conjunto de hosts.
  • Inventario: Una lista de hosts o nodos que Ansible gestiona.
  • Módulos: Scripts reutilizables e independientes que Ansible ejecuta en tu nombre.
  • Roles: Una forma de agrupar múltiples tareas para organizar playbooks.
  • Tareas: Acciones que Ansible realiza, como instalar un paquete o reiniciar un servicio.

Configuración de Ansible

Antes de poder comenzar a usar Ansible, necesitas instalarlo y configurar tu entorno.

Instalación

Ansible se puede instalar en la mayoría de los sistemas tipo Unix utilizando el gestor de paquetes. Por ejemplo, en Ubuntu, puedes instalar Ansible usando los siguientes comandos:

sudo apt update
sudo apt install ansible

Para verificar la instalación, ejecuta:

ansible --version

Archivo de Inventario

El archivo de inventario es un archivo de texto simple que lista los hosts que deseas gestionar. Por defecto, Ansible busca un archivo de inventario en /etc/ansible/hosts.

Ejemplo de un archivo de inventario:

[webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com

Escribiendo tu Primer Playbook

Un playbook es un archivo YAML que describe un conjunto de tareas a ejecutar en tus hosts.

Ejemplo de Playbook

Aquí hay un playbook simple que instala Nginx en un grupo de servidores web:

---
- name: Instalar Nginx en servidores web
  hosts: webservers
  become: yes
  tasks:
    - name: Asegurarse de que Nginx esté instalado
      apt:
        name: nginx
        state: present

    - name: Asegurarse de que Nginx esté en ejecución
      service:
        name: nginx
        state: started

Explicación

  • name: Una descripción del playbook o tarea.
  • hosts: El grupo de hosts en los que se ejecutará el playbook.
  • become: Indica que las tareas deben ejecutarse con privilegios elevados (sudo).
  • tasks: Una lista de tareas a ejecutar.
    • apt: Un módulo que gestiona paquetes en sistemas basados en Debian.
    • service: Un módulo que gestiona servicios.

Ejecutando tu Playbook

Para ejecutar el playbook, usa el comando ansible-playbook seguido del nombre de tu archivo de playbook:

ansible-playbook install_nginx.yml

Conclusión

En esta sección, has aprendido los conceptos básicos de Ansible, incluyendo sus conceptos clave, cómo instalarlo y cómo escribir y ejecutar tu primer playbook. La simplicidad y potencia de Ansible lo convierten en una excelente herramienta para automatizar tareas de TI. A medida que continúes explorando Ansible, descubrirás características más avanzadas y mejores prácticas que te ayudarán a gestionar tu infraestructura de manera más eficiente.
