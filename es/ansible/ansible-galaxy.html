Introducción a Ansible Galaxy

Ansible Galaxy es un centro para encontrar, compartir y reutilizar contenido de Ansible. Permite a los usuarios descargar roles y colecciones que se pueden usar para automatizar varias tareas. Esta sección cubrirá los conceptos básicos de Ansible Galaxy, cómo usarlo y cómo contribuir a él.

Conceptos Clave

  • Roles: Unidades de contenido de Ansible preempaquetadas que se pueden reutilizar en múltiples playbooks.
  • Colecciones: Un formato de distribución para contenido de Ansible que puede incluir roles, módulos, plugins y más.
  • Galaxy CLI: Interfaz de línea de comandos para interactuar con Ansible Galaxy.

Usando Ansible Galaxy

Instalando Roles

Los roles se pueden instalar fácilmente desde Ansible Galaxy usando el comando ansible-galaxy.

ansible-galaxy install username.role_name
  • username: El nombre de usuario de Galaxy del autor del rol.
  • role_name: El nombre del rol que deseas instalar.

Ejemplo

Para instalar un rol para gestionar Nginx:

ansible-galaxy install geerlingguy.nginx

Usando Roles Instalados en Playbooks

Una vez que un rol está instalado, se puede usar en tus playbooks.

---
- hosts: webservers
  roles:
    - geerlingguy.nginx

Gestionando Roles

Puedes listar los roles instalados usando:

ansible-galaxy list

Para eliminar un rol:

ansible-galaxy remove username.role_name

Creando y Compartiendo Roles

Creando un Rol

Puedes crear un nuevo rol usando el comando ansible-galaxy:

ansible-galaxy init my_new_role

Este comando crea una estructura de directorios para tu nuevo rol.

Estructura de Directorios

El rol creado tendrá la siguiente estructura:

my_new_role/
├── defaults
│   └── main.yml
├── files
├── handlers
│   └── main.yml
├── meta
│   └── main.yml
├── tasks
│   └── main.yml
├── templates
├── tests
│   ├── inventory
│   └── test.yml
└── vars
    └── main.yml

Publicando un Rol

Para compartir tu rol en Ansible Galaxy, necesitas:

  1. Crear una cuenta en Galaxy.
  2. Vincular tu cuenta de GitHub a Galaxy.
  3. Subir tu rol a un repositorio de GitHub.
  4. Importar el rol en Galaxy usando la interfaz web de Galaxy.

Colecciones de Ansible

Instalando Colecciones

Las colecciones se pueden instalar de manera similar a los roles:

ansible-galaxy collection install namespace.collection_name

Ejemplo

Para instalar una colección para AWS:

ansible-galaxy collection install amazon.aws

Usando Colecciones en Playbooks

Las colecciones se pueden usar en playbooks especificando el nombre completo de la colección.

---
- hosts: localhost
  tasks:
    - name: Crear un bucket S3
      amazon.aws.s3_bucket:
        name: my-bucket
        state: present

Creando una Colección

Puedes crear una nueva colección usando el comando ansible-galaxy:

ansible-galaxy collection init my_namespace.my_collection

Este comando crea una estructura de directorios para tu nueva colección.

Estructura de Directorios

La colección creada tendrá la siguiente estructura:

my_namespace-my_collection/
├── docs
├── plugins
│   ├── modules
│   └── ...
├── roles
├── playbooks
└── tests

Publicando una Colección

Para compartir tu colección en Ansible Galaxy, necesitas:

  1. Crear una cuenta en Galaxy.
  2. Vincular tu cuenta de GitHub a Galaxy.
  3. Subir tu colección a un repositorio de GitHub.
  4. Importar la colección en Galaxy usando la interfaz web de Galaxy.

Conclusión

Ansible Galaxy es una herramienta poderosa para encontrar, compartir y reutilizar contenido de Ansible. Al entender cómo usar roles y colecciones, puedes agilizar significativamente tus tareas de automatización. Ya sea que estés instalando roles y colecciones existentes o creando y compartiendo los tuyos, Ansible Galaxy proporciona una plataforma robusta para mejorar tu experiencia con Ansible.
