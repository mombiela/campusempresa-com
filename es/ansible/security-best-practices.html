Introducción

La seguridad es un aspecto crítico de cualquier infraestructura de TI, y Ansible, como una poderosa herramienta de automatización, debe ser utilizada con las mejores prácticas de seguridad en mente. Esta sección te guiará a través de las prácticas esenciales de seguridad para usar Ansible, desde niveles principiantes hasta avanzados.

Prácticas Básicas de Seguridad

Usar Claves SSH en Lugar de Contraseñas

  • Por qué: Las claves SSH proporcionan una forma más segura y conveniente de autenticarse que las contraseñas.
  • Cómo: Genera claves SSH y distribuye la clave pública a los nodos gestionados.
# Generar par de claves SSH
ssh-keygen -t rsa -b 4096

# Copiar la clave pública al nodo gestionado
ssh-copy-id user@managed_node

Limitar el Acceso SSH

  • Por qué: Restringir el acceso SSH reduce la superficie de ataque.
  • Cómo: Usa firewalls y grupos de seguridad para limitar el acceso SSH a direcciones IP de confianza.

Usar Ansible Vault para Datos Sensibles

  • Por qué: Almacenar datos sensibles como contraseñas y claves API en texto plano es un riesgo de seguridad.
  • Cómo: Encripta datos sensibles usando Ansible Vault.
# Crear un archivo encriptado
ansible-vault create secrets.yml

# Editar un archivo encriptado
ansible-vault edit secrets.yml

# Usar el archivo encriptado en un playbook
ansible-playbook playbook.yml --ask-vault-pass

Prácticas Intermedias de Seguridad

Principio de Menor Privilegio

  • Por qué: Minimizar los permisos otorgados a usuarios y procesos reduce el daño potencial de brechas de seguridad.
  • Cómo: Configura el archivo sudoers para otorgar solo los permisos necesarios.
# Ejemplo de entrada sudoers para privilegios limitados
user ALL=(ALL) NOPASSWD: /usr/bin/ansible-playbook

Configuración Segura de Ansible

  • Por qué: Asegurar la configuración de Ansible previene cambios no autorizados.
  • Cómo: Establece permisos de archivo apropiados y usa control de versiones con controles de acceso.
# Establecer permisos de archivo
chmod 600 ansible.cfg
chmod 700 /etc/ansible

Usar Roles y Playbooks de Fuentes Confiables

  • Por qué: Usar roles y playbooks no confiables puede introducir vulnerabilidades.
  • Cómo: Verifica la fuente y revisa el código antes de usarlo.

Prácticas Avanzadas de Seguridad

Implementar Ansible Tower o AWX

  • Por qué: Proporciona una forma centralizada, segura y escalable de gestionar despliegues de Ansible.
  • Cómo: Instala y configura Ansible Tower o AWX, y usa sus características para control de acceso basado en roles (RBAC), registro y auditoría.

Usar Módulos Personalizados de Ansible con Cuidado

  • Por qué: Los módulos personalizados pueden introducir riesgos de seguridad si no se revisan adecuadamente.
  • Cómo: Revisa y prueba los módulos personalizados a fondo antes de su despliegue.

Auditorías de Seguridad y Chequeos de Cumplimiento Regulares

  • Por qué: Las auditorías regulares ayudan a identificar y mitigar vulnerabilidades de seguridad.
  • Cómo: Usa Ansible para automatizar chequeos de cumplimiento y generar informes de auditoría.
# Ejemplo de playbook para chequeo de cumplimiento
- name: Verificar usuarios no autorizados
  hosts: all
  tasks:
    - name: Asegurarse de que no existan usuarios no autorizados
      shell: cat /etc/passwd | grep unauthorized_user
      register: result
      failed_when: result.stdout != ""

Conclusión

La seguridad en Ansible es un proceso continuo que implica usar las mejores prácticas en cada nivel de tu flujo de trabajo de automatización. Siguiendo estas directrices, puedes reducir significativamente el riesgo de brechas de seguridad y asegurarte de que tu infraestructura permanezca segura. Mantente siempre actualizado con las últimas prácticas de seguridad y monitorea y audita continuamente tus sistemas.
