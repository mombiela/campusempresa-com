Introducción

En esta sección, exploraremos cómo usar condicionales y bucles en Ansible para crear playbooks más dinámicos y flexibles. Entender estos conceptos es crucial para escribir scripts de automatización eficientes.

Condicionales en Ansible

Los condicionales te permiten ejecutar tareas basadas en ciertas condiciones. Esto puede ser útil para tomar decisiones dentro de tus playbooks.

Conceptos Clave

  • Declaración When: Se usa para ejecutar una tarea solo si se cumple una cierta condición.
  • Hechos: Variables que Ansible recopila sobre el sistema, las cuales pueden ser usadas en condicionales.

Declaración When

La declaración when se usa para controlar si una tarea o un conjunto de tareas debe ejecutarse basado en una condición.

Ejemplo

- name: Instalar Apache en sistemas basados en Debian
  apt:
    name: apache2
    state: present
  when: ansible_os_family == "Debian"

En este ejemplo, la tarea para instalar Apache solo se ejecutará si la familia del sistema operativo es Debian.

Uso de Hechos en Condicionales

Los hechos de Ansible son variables que se descubren automáticamente cuando Ansible se conecta a un host. Estos hechos pueden ser usados en condicionales.

Ejemplo

- name: Reiniciar el servidor si está ejecutando Ubuntu
  command: /sbin/reboot
  when: ansible_distribution == "Ubuntu"

Aquí, el servidor solo se reiniciará si el sistema operativo es Ubuntu.

Bucles en Ansible

Los bucles te permiten repetir una tarea múltiples veces con diferentes elementos. Esto es útil para tareas como instalar múltiples paquetes o crear múltiples usuarios.

Conceptos Clave

  • with_items: Una directiva de bucle comúnmente usada para iterar sobre una lista de elementos.
  • loop: Una directiva más flexible introducida en Ansible 2.5, que puede reemplazar with_items.

with_items

La directiva with_items se usa para iterar sobre una lista de elementos.

Ejemplo

- name: Instalar múltiples paquetes
  apt:
    name: "{{ item }}"
    state: present
  with_items:
    - git
    - curl
    - vim

Esta tarea instalará git, curl y vim en el sistema objetivo.

loop

La directiva loop proporciona más flexibilidad y puede ser usada en lugar de with_items.

Ejemplo

- name: Agregar múltiples usuarios
  user:
    name: "{{ item.name }}"
    state: present
    groups: "{{ item.groups }}"
  loop:
    - { name: 'alice', groups: 'admin' }
    - { name: 'bob', groups: 'users' }

Esta tarea creará dos usuarios, alice y bob, con sus respectivos grupos.

Combinando Condicionales y Bucles

Puedes combinar condicionales y bucles para crear lógica más compleja en tus playbooks.

Ejemplo

- name: Instalar paquetes solo si no están ya instalados
  apt:
    name: "{{ item }}"
    state: present
  loop:
    - git
    - curl
    - vim
  when: ansible_facts.packages[item] is not defined

Esta tarea instalará git, curl y vim solo si no están ya instalados en el sistema.

Conclusión

Entender y usar condicionales y bucles en Ansible te permite crear playbooks más dinámicos y eficientes. Al aprovechar estas características, puedes automatizar tareas complejas y manejar varios escenarios con facilidad. Practica usando estos conceptos en tus playbooks para volverte competente en la automatización con Ansible.
