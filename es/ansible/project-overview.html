Introducción a Ansible

Ansible es una herramienta de automatización de código abierto utilizada para la gestión de configuraciones, despliegue de aplicaciones y automatización de tareas. Simplifica tareas complejas y asegura consistencia en toda tu infraestructura. Este curso te llevará de un nivel principiante a un nivel avanzado, proporcionándote las habilidades necesarias para usar Ansible de manera efectiva en escenarios del mundo real.

Conceptos Clave

  • Gestión de Configuraciones: Automatiza la configuración y el mantenimiento de tu infraestructura.
  • Playbooks: Archivos YAML que definen una serie de tareas a ejecutar en tus nodos gestionados.
  • Módulos: Scripts reutilizables e independientes que Ansible ejecuta en tus nodos.
  • Inventario: Una lista de nodos (servidores) que Ansible gestiona.
  • Roles: Una forma de organizar playbooks y otros archivos para facilitar la reutilización y el intercambio.
  • Variables: Utilizadas para almacenar valores que pueden ser reutilizados en todos tus playbooks.
  • Plantillas: Archivos que pueden generar dinámicamente archivos de configuración basados en variables.

Estructura del Curso

Nivel Principiante

  • Introducción a YAML: Aprende lo básico de la sintaxis YAML, esencial para escribir playbooks de Ansible.
  • Configuración de Ansible: Guía paso a paso para instalar Ansible en tu máquina local.
  • Playbooks Básicos: Escribe tu primer playbook para automatizar tareas simples.
  • Uso de Módulos: Introducción a los módulos de Ansible más comúnmente utilizados.

Nivel Intermedio

  • Playbooks Avanzados: Escribe playbooks más complejos con condicionales y bucles.
  • Roles e Inclusiones: Organiza tus playbooks para una mejor mantenibilidad.
  • Variables y Hechos: Usa variables y recopila hechos sobre tus nodos.
  • Plantillas y Archivos: Usa plantillas Jinja2 para crear archivos de configuración dinámicos.

Nivel Avanzado

  • Módulos Personalizados: Escribe tus propios módulos de Ansible para extender la funcionalidad.
  • Manejo de Errores: Implementa manejo de errores y depuración en tus playbooks.
  • Optimización del Rendimiento: Optimiza tus playbooks de Ansible para un mejor rendimiento.
  • Ansible Tower: Introducción a Ansible Tower para gestión a nivel empresarial.

Código de Ejemplo

Ejemplo para Principiantes: Playbook Simple

---
- name: Instalar y iniciar Apache
  hosts: webservers
  become: yes
  tasks:
    - name: Instalar Apache
      apt:
        name: apache2
        state: present

    - name: Iniciar Apache
      service:
        name: apache2
        state: started

Explicación:

  • hosts: webservers: Especifica el grupo de servidores en los que se ejecutará este playbook.
  • become: yes: Ejecuta tareas con privilegios elevados.
  • tasks: Una lista de tareas a ejecutar.
    • apt: Módulo para gestionar paquetes en sistemas basados en Debian.
    • service: Módulo para gestionar servicios.

Ejemplo Intermedio: Uso de Variables y Plantillas

---
- name: Desplegar Nginx con configuración personalizada
  hosts: webservers
  become: yes
  vars:
    nginx_port: 8080
  tasks:
    - name: Instalar Nginx
      apt:
        name: nginx
        state: present

    - name: Desplegar configuración de Nginx
      template:
        src: nginx.conf.j2
        dest: /etc/nginx/nginx.conf

    - name: Iniciar Nginx
      service:
        name: nginx
        state: started

Explicación:

  • vars: Define una variable nginx_port que puede ser utilizada en el playbook.
  • template: Usa una plantilla Jinja2 para crear el archivo de configuración de Nginx.

Ejemplo Avanzado: Módulo Personalizado

#!/usr/bin/python

from ansible.module_utils.basic import AnsibleModule

def run_module():
    module_args = dict(
        name=dict(type='str', required=True)
    )

    result = dict(
        changed=False,
        original_message='',
        message=''
    )

    module = AnsibleModule(
        argument_spec=module_args,
        supports_check_mode=True
    )

    result['original_message'] = module.params['name']
    result['message'] = 'Hola, {}'.format(module.params['name'])

    if module.check_mode:
        module.exit_json(**result)

    result['changed'] = True
    module.exit_json(**result)

def main():
    run_module()

if __name__ == '__main__':
    main()

Explicación:

  • Este es un módulo personalizado simple escrito en Python.
  • AnsibleModule: Una clase auxiliar para manejar parámetros de entrada y resultados de salida.
  • module_args: Define los parámetros de entrada para el módulo.
  • result: Un diccionario para almacenar los resultados de la ejecución del módulo.
