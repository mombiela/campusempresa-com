En esta sección, cubriremos la terminología fundamental utilizada en Ansible. Comprender estos términos es crucial para usar Ansible de manera efectiva en la automatización y gestión de configuraciones.

Conceptos Clave

  • Nodo de Control: La máquina donde Ansible está instalado y desde la cual se ejecutan los comandos. Este es el punto central desde el cual se gestionan todas las tareas de automatización.

  • Nodos Gestionados: Las máquinas que están siendo gestionadas por Ansible. También se conocen como hosts. Ansible se comunica con estos nodos para ejecutar tareas.

  • Inventario: Un archivo que lista todos los nodos gestionados. Puede ser un archivo de texto simple con una lista de direcciones IP o nombres de host, o puede ser un archivo más complejo con grupos y variables.

    [webservers]
web1.ejemplo.com
web2.ejemplo.com

[dbservers]
db1.ejemplo.com
db2.ejemplo.com

  • Playbook: Un archivo YAML que contiene una serie de tareas a ejecutar en los nodos gestionados. Los playbooks son el núcleo del lenguaje de configuración, despliegue y orquestación de Ansible.

    - name: Instalar y arrancar Apache
  hosts: webservers
  tasks:
    - name: Instalar Apache
      apt:
        name: apache2
        state: present

    - name: Arrancar Apache
      service:
        name: apache2
        state: started

  • Tarea: Una unidad de trabajo individual en un playbook. Cada tarea llama a un módulo de Ansible para realizar una acción específica.

  • Módulo: Las unidades de trabajo que Ansible ejecuta. Los módulos pueden gestionar recursos del sistema, instalar software y realizar varias otras tareas. Ejemplos incluyen apt, yum, service y copy.

  • Rol: Una forma de agrupar múltiples tareas, variables, archivos, plantillas y manejadores en una sola unidad. Los roles ayudan a organizar los playbooks y reutilizar código.

  • Manejador: Un tipo especial de tarea que se ejecuta solo cuando es notificado por otra tarea. Los manejadores se utilizan típicamente para reiniciar servicios.

    - name: Instalar Apache
  apt:
    name: apache2
    state: present
  notify:
    - Reiniciar Apache

handlers:
  - name: Reiniciar Apache
    service:
      name: apache2
      state: restarted

  • Variable: Se utiliza para almacenar valores que pueden ser reutilizados a lo largo de los playbooks y roles. Las variables pueden definirse en muchos lugares, incluyendo archivos de inventario, playbooks y roles.

    - name: Instalar Apache
  hosts: webservers
  vars:
    apache_package: apache2
  tasks:
    - name: Instalar Apache
      apt:
        name: "{{ apache_package }}"
        state: present

  • Hecho: Información sobre los nodos gestionados recopilada por Ansible. Los hechos se recopilan automáticamente y pueden ser utilizados en los playbooks.

  • Plantilla: Un archivo que contiene variables y es procesado por el motor de plantillas Jinja2 para producir un archivo de configuración final.

    # template.j2
ServerName {{ ansible_hostname }}

    - name: Configurar Apache
  hosts: webservers
  tasks:
    - name: Desplegar configuración de Apache
      template:
        src: template.j2
        dest: /etc/apache2/sites-available/000-default.conf

Resumen

Comprender estos términos básicos de Ansible es esencial para cualquiera que comience con Ansible. Estos conceptos forman la base sobre la cual se construyen temas y prácticas más avanzadas. Familiarizándote con estos términos, estarás mejor preparado para escribir y entender playbooks de Ansible, gestionar configuraciones y automatizar tareas de manera efectiva.
