Configuración del Entorno
Implementando el Proyecto ►

Configurar el entorno es el primer paso para comenzar con Ansible. Esta sección te guiará a través de los pasos necesarios para instalar y configurar Ansible en tu sistema, asegurando que tengas una base sólida sobre la cual construir a medida que avances en el curso.

Requisitos Previos

Antes de instalar Ansible, asegúrate de tener los siguientes requisitos previos:

  • Un sistema operativo similar a Unix (Linux, macOS o WSL en Windows).
  • Python 3.6 o posterior instalado en tu sistema.
  • Conocimientos básicos de operaciones en la línea de comandos.

Instalación de Ansible

Usando el Gestor de Paquetes

En Ubuntu/Debian

Para instalar Ansible en Ubuntu o Debian, utiliza los siguientes comandos:

sudo apt update
sudo apt install ansible -y

En CentOS/RHEL

Para CentOS o RHEL, utiliza los siguientes comandos:

sudo yum install epel-release -y
sudo yum install ansible -y

En macOS

Para los usuarios de macOS, Homebrew es la forma más fácil de instalar Ansible:

brew install ansible

Usando Pip

Alternativamente, puedes instalar Ansible usando el gestor de paquetes de Python, pip:

pip install ansible

Verificación de la Instalación

Después de instalar Ansible, verifica la instalación comprobando la versión:

ansible --version

Deberías ver una salida similar a:

ansible [core 2.10.7]
  config file = /etc/ansible/ansible.cfg
  configured module search path = ['/home/user/.ansible/plugins/modules', '/usr/share/ansible/plugins/modules']
  ansible python module location = /usr/local/lib/python3.8/dist-packages/ansible
  executable location = /usr/local/bin/ansible
  python version = 3.8.5 (default, Jul 28 2020, 12:59:40) [GCC 9.3.0]

Configuración de Ansible

Archivo de Configuración de Ansible

El comportamiento de Ansible se puede personalizar utilizando el archivo de configuración ansible.cfg. Por defecto, este archivo se encuentra en /etc/ansible/ansible.cfg. También puedes crear un archivo de configuración personalizado en tu directorio de proyecto.

Aquí tienes un ejemplo de un archivo ansible.cfg básico:

[defaults]
inventory = ./hosts
remote_user = your_username
host_key_checking = False

Archivo de Inventario

El archivo de inventario lista los hosts y grupos de hosts que Ansible gestiona. Aquí tienes un ejemplo de un archivo de inventario simple llamado hosts:

[webservers]
webserver1 ansible_host=192.168.1.10 ansible_user=ubuntu
webserver2 ansible_host=192.168.1.11 ansible_user=ubuntu

[databases]
dbserver1 ansible_host=192.168.1.20 ansible_user=ubuntu

Prueba de la Configuración

Para asegurarte de que todo esté configurado correctamente, ejecuta un comando simple de Ansible para hacer ping a todos los hosts en tu inventario:

ansible all -m ping

Deberías ver una salida similar a:

webserver1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
webserver2 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
dbserver1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}

Conclusión

En esta sección, cubrimos los pasos esenciales para configurar el entorno de Ansible, incluyendo la instalación, configuración y verificación. Con tu entorno listo, ahora estás preparado para profundizar en Ansible y comenzar a automatizar tus tareas.
