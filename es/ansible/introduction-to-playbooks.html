¿Qué es un Playbook?

Los Playbooks de Ansible son archivos YAML que definen una serie de tareas a ejecutar en hosts remotos. Son el componente central de las capacidades de configuración, despliegue y orquestación de Ansible. Los Playbooks te permiten describir el estado deseado de tus sistemas en un formato legible para humanos.

Conceptos Clave

  • Sintaxis YAML: Los Playbooks están escritos en YAML, un estándar de serialización de datos legible para humanos.
  • Tareas: Acciones individuales que Ansible ejecuta, como instalar un paquete o copiar un archivo.
  • Módulos: Scripts reutilizables e independientes que Ansible ejecuta en tu nombre. Cada tarea utiliza un módulo.
  • Handlers: Tareas especiales que son activadas por otras tareas.
  • Variables: Se utilizan para almacenar valores que pueden ser reutilizados en todo el playbook.
  • Plantillas: Archivos que contienen variables y son procesados por Ansible para producir archivos de configuración.

Estructura Básica de un Playbook

Un playbook simple consta de una o más jugadas. Cada jugada se dirige a un grupo de hosts y define tareas a ejecutar en esos hosts.

---
- name: Ensure Apache is installed
  hosts: webservers
  become: yes

  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present

    - name: Start Apache service
      service:
        name: apache2
        state: started

Explicación

  • name: Una descripción de la jugada.
  • hosts: El grupo de hosts al que se dirige la jugada.
  • become: Indica si se debe usar escalación de privilegios (por ejemplo, sudo).
  • tasks: Una lista de tareas a ejecutar.

Tareas y Módulos

Las tareas son los bloques de construcción de un playbook. Cada tarea utiliza un módulo para realizar una acción específica.

tasks:
  - name: Install Apache
    apt:
      name: apache2
      state: present

En este ejemplo:

  • name: Una descripción de la tarea.
  • apt: El módulo utilizado para gestionar paquetes en sistemas basados en Debian.
  • name: El paquete a gestionar.
  • state: El estado deseado del paquete (por ejemplo, present, absent).

Uso de Variables

Las variables te permiten hacer que tus playbooks sean más flexibles y reutilizables.

---
- name: Install software
  hosts: all
  become: yes

  vars:
    package_name: apache2

  tasks:
    - name: Install package
      apt:
        name: "{{ package_name }}"
        state: present

Explicación

  • vars: Define variables que pueden ser utilizadas en todo el playbook.
  • {{ package_name }}: La variable se referencia usando llaves dobles.

Handlers

Los handlers son tareas que se ejecutan solo cuando son notificadas por otras tareas. Se utilizan típicamente para reiniciar servicios después de cambios en la configuración.

---
- name: Configure web server
  hosts: webservers
  become: yes

  tasks:
    - name: Copy configuration file
      copy:
        src: /path/to/httpd.conf
        dest: /etc/httpd/conf/httpd.conf
      notify: Restart Apache

  handlers:
    - name: Restart Apache
      service:
        name: apache2
        state: restarted

Explicación

  • notify: Especifica un handler que se activará si la tarea cambia algo.
  • handlers: Define tareas que se ejecutan cuando son notificadas.

Plantillas

Las plantillas te permiten crear archivos de configuración dinámicos utilizando variables.

---
- name: Configure web server
  hosts: webservers
  become: yes

  vars:
    server_port: 8080

  tasks:
    - name: Deploy configuration file from template
      template:
        src: /path/to/httpd.conf.j2
        dest: /etc/httpd/conf/httpd.conf

Explicación

  • template: El módulo utilizado para procesar plantillas.
  • src: El archivo de plantilla fuente.
  • dest: La ruta de destino en el host remoto.

Conclusión

Los Playbooks de Ansible son una forma poderosa de automatizar la configuración y gestión de tus sistemas. Al entender la estructura básica y los conceptos clave como tareas, módulos, variables, handlers y plantillas, puedes comenzar a escribir playbooks efectivos para gestionar tu infraestructura. A medida que avances, aprenderás a crear playbooks más complejos que puedan manejar una amplia gama de tareas y escenarios.
