Concurrencia y Paralelismo en Ruby
Recolección de Basura ►

Introducción

La concurrencia y el paralelismo son conceptos cruciales en la programación moderna, especialmente para mejorar el rendimiento de las aplicaciones. En esta sección, exploraremos cómo Ruby maneja la concurrencia y el paralelismo, proporcionándote el conocimiento para escribir código eficiente y escalable.

Conceptos Clave

  • Concurrencia: La capacidad de un programa para avanzar en múltiples tareas simultáneamente. Esto no significa necesariamente que las tareas se estén ejecutando al mismo tiempo, sino que están avanzando sin esperar a que las otras terminen.
  • Paralelismo: La capacidad de ejecutar múltiples tareas simultáneamente, típicamente en múltiples procesadores o núcleos.

Hilos en Ruby

Creando Hilos

Ruby proporciona soporte incorporado para la concurrencia a través de hilos. Un hilo es la unidad más pequeña de ejecución dentro de un proceso.

thread = Thread.new do
  puts "¡Hola desde un nuevo hilo!"
end

thread.join # Esperar a que el hilo termine

Seguridad de Hilos

Al usar hilos, es importante asegurarse de que los recursos compartidos se accedan de manera segura para evitar condiciones de carrera.

counter = 0
mutex = Mutex.new

threads = 10.times.map do
  Thread.new do
    1000.times do
      mutex.synchronize do
        counter += 1
      end
    end
  end
end

threads.each(&:join)
puts counter # Debería imprimir 10000

Fibras en Ruby

Las fibras son una alternativa más ligera a los hilos y proporcionan una forma de implementar concurrencia cooperativa.

Creando Fibras

Las fibras te permiten pausar y reanudar la ejecución en puntos específicos.

fiber = Fiber.new do
  puts "Fibra iniciada"
  Fiber.yield
  puts "Fibra reanudada"
end

fiber.resume
puts "De vuelta al principal"
fiber.resume

Paralelismo con Procesos

Creando Procesos

Ruby también puede lograr paralelismo bifurcando procesos. Cada proceso bifurcado se ejecuta de manera independiente y puede ejecutar código en paralelo.

fork do
  puts "Este es un proceso hijo"
end

puts "Este es el proceso padre"
Process.wait # Esperar a que el proceso hijo termine

Usando la gema `Parallel`

Para un procesamiento paralelo más avanzado, puedes usar la gema parallel.

require 'parallel'

results = Parallel.map([1, 2, 3, 4, 5]) do |number|
  number * 2
end

puts results.inspect # [2, 4, 6, 8, 10]

Comparando Concurrencia y Paralelismo

| Característica | Hilos | Fibras | Procesos | |----------------|---------------------------------|---------------------------------|----------------------------------| | Concurrencia | Sí | Sí | No | | Paralelismo | Limitado (debido al GIL) | No | Sí | | Complejidad | Media | Baja | Alta | | Caso de Uso | Tareas dependientes de I/O | Tareas ligeras | Tareas dependientes de CPU |

Conclusión

La concurrencia y el paralelismo son herramientas poderosas para mejorar el rendimiento de las aplicaciones Ruby. Al entender y utilizar hilos, fibras y procesos, puedes escribir código más eficiente y escalable. Recuerda siempre considerar la seguridad de los hilos y elegir la herramienta adecuada para el trabajo según tu caso de uso específico.
