Introducción a las Pruebas Unitarias

  • Definición: Las pruebas unitarias implican probar unidades o componentes individuales de un software para asegurar que funcionan como se espera.
  • Importancia: Ayuda a identificar errores temprano, asegura la fiabilidad del código y facilita la refactorización.

Empezando con Minitest

  • ¿Qué es Minitest?: Una biblioteca de pruebas para Ruby que proporciona una suite completa de facilidades de prueba que soportan TDD, BDD, mocking y benchmarking.
  • Instalación: 
    gem install minitest

Escribiendo Tu Primera Prueba

  • Creando un Archivo de Prueba: Típicamente, los archivos de prueba se colocan en un directorio test.
  • Estructura Básica: 
    require 'minitest/autorun'

class ExampleTest < Minitest::Test
  def test_example
    assert_equal 4, 2 + 2
  end
end
    • require 'minitest/autorun': Carga Minitest y ejecuta las pruebas automáticamente.
    • class ExampleTest < Minitest::Test: Define una clase de prueba que hereda de Minitest::Test.
    • def test_example: Define un método de prueba. Los métodos de prueba deben comenzar con test_.
    • assert_equal 4, 2 + 2: Una afirmación que verifica si la expresión 2 + 2 es igual a 4.

Afirmaciones Comunes

  • assert_equal: Verifica si dos valores son iguales. 
    assert_equal expected, actual
  • assert: Verifica si una condición es verdadera. 
    assert condition
  • assert_nil: Verifica si un valor es nil. 
    assert_nil value
  • assert_raises: Verifica si se lanza una excepción específica. 
    assert_raises(ExpectedException) { code }

Técnicas Avanzadas de Prueba

  • Setup y Teardown: Métodos para ejecutar código antes y después de cada prueba. 
    class AdvancedTest < Minitest::Test
  def setup
    @value = 2
  end

  def teardown
    # Código de limpieza
  end

  def test_value
    assert_equal 2, @value
  end
end
  • Mocking y Stubbing: Simulación de objetos o métodos para aislar pruebas. 
    require 'minitest/mock'

def test_mock
  mock = Minitest::Mock.new
  mock.expect(:foo, true)

  assert mock.foo
  mock.verify
end

Organizando Pruebas

  • Estructura del Directorio de Pruebas: 
    project_root/
├── lib/
│   └── example.rb
└── test/
    ├── example_test.rb
    └── test_helper.rb
  • Test Helper: Un archivo para requerir dependencias comunes y configurar configuraciones. 
    # test/test_helper.rb
require 'minitest/autorun'

Ejecutando Pruebas

  • Línea de Comando: Ejecuta todas las pruebas con: 
    ruby -Ilib:test test/example_test.rb
  • Rake: Usando Rake para automatizar la ejecución de pruebas. 
    # Rakefile
require 'rake/testtask'

Rake::TestTask.new do |t|
  t.libs << 'test'
  t.test_files = FileList['test/**/*_test.rb']
end

task default: :test

Conclusión

Las pruebas unitarias con Minitest son una forma poderosa de asegurar que tu código Ruby sea fiable y mantenible. Al escribir pruebas, puedes detectar errores temprano, refactorizar con confianza y construir una base de código robusta. Comienza con afirmaciones simples y gradualmente incorpora técnicas más avanzadas como mocking y métodos de setup/teardown para mejorar tu suite de pruebas.
