Introducción a la Arquitectura MVC

Model-View-Controller (MVC) es un patrón de diseño utilizado para separar las preocupaciones en aplicaciones de software. Divide una aplicación en tres componentes interconectados:

  • Modelo: Gestiona los datos y la lógica de negocio.
  • Vista: Maneja la visualización y la interfaz de usuario.
  • Controlador: Gestiona la entrada del usuario e interactúa con el Modelo y la Vista.

Entendiendo los Componentes

Modelo

El Modelo representa los datos y las reglas que gobiernan el acceso y las actualizaciones de estos datos. En Ruby, esto a menudo implica clases que interactúan con la base de datos.

# app/models/article.rb
class Article < ApplicationRecord
  validates :title, presence: true
  validates :body, presence: true
end
  • Explicación: Esta clase Article hereda de ApplicationRecord, lo que significa que está conectada a la base de datos. Los métodos validates aseguran que tanto title como body deben estar presentes para que un objeto Article sea válido.

Vista

La Vista es responsable de renderizar la interfaz de usuario. En Ruby on Rails, las vistas se escriben típicamente en Embedded Ruby (ERB) u otros lenguajes de plantillas.

<!-- app/views/articles/show.html.erb -->
<h1><%= @article.title %></h1>
<p><%= @article.body %></p>
  • Explicación: Esta plantilla ERB muestra el título y el cuerpo de un artículo. Las etiquetas <%= %> se utilizan para incrustar código Ruby dentro del HTML.

Controlador

El Controlador maneja la entrada del usuario, interactúa con el Modelo y renderiza la Vista apropiada.

# app/controllers/articles_controller.rb
class ArticlesController < ApplicationController
  def show
    @article = Article.find(params[:id])
  end
end
  • Explicación: La acción show en el ArticlesController encuentra un artículo por su ID (pasado en los parámetros de la URL) y lo asigna a una variable de instancia @article, que luego es accesible en la vista.

Cómo Interactúan los Componentes MVC

  1. Interacción del Usuario: El usuario interactúa con la Vista (por ejemplo, haciendo clic en un enlace).
  2. Acción del Controlador: El Controlador recibe la entrada, la procesa (por ejemplo, obteniendo datos del Modelo) y determina la respuesta apropiada.
  3. Interacción con el Modelo: El Controlador puede interactuar con el Modelo para recuperar o actualizar datos.
  4. Renderización de la Vista: El Controlador pasa datos a la Vista, que renderiza la salida final al usuario.

Ejemplo de Flujo

  • Usuario: Hace clic en un enlace para ver un artículo.
  • Controlador: Se activa la acción show del ArticlesController.
  • Modelo: El modelo Article obtiene el artículo de la base de datos.
  • Vista: La plantilla show.html.erb se renderiza con los datos del artículo.

Conceptos Avanzados

Enrutamiento

El enrutamiento en Rails mapea URLs a acciones del controlador.

# config/routes.rb
Rails.application.routes.draw do
  resources :articles, only: [:show]
end
  • Explicación: Esta declaración de ruta crea una ruta para la acción show del ArticlesController.

Vistas Parciales

Las vistas parciales permiten reutilizar el código de la vista.

<!-- app/views/articles/_article.html.erb -->
<div class="article">
  <h2><%= article.title %></h2>
  <p><%= article.body %></p>
</div>
  • Explicación: Esta parcial puede ser renderizada en otras vistas para mostrar un artículo.
<!-- app/views/articles/index.html.erb -->
<% @articles.each do |article| %>
  <%= render partial: "article", locals: { article: article } %>
<% end %>
  • Explicación: La vista index renderiza la parcial _article para cada artículo en la colección @articles.

Parámetros Fuertes

Los parámetros fuertes proporcionan una forma de listar atributos que pueden ser asignados en masa.

# app/controllers/articles_controller.rb
class ArticlesController < ApplicationController
  def create
    @article = Article.new(article_params)
    if @article.save
      redirect_to @article
    else
      render 'new'
    end
  end

  private

  def article_params
    params.require(:article).permit(:title, :body)
  end
end
  • Explicación: El método article_params asegura que solo los atributos title y body pueden ser asignados en masa.

Conclusión

La arquitectura MVC es un poderoso patrón de diseño que ayuda a organizar el código y separar las preocupaciones en una aplicación Ruby on Rails. Al entender e implementar MVC, puedes crear aplicaciones escalables y mantenibles. Este tema cubrió los conceptos básicos de MVC, cómo interactúan los componentes y algunos conceptos avanzados para profundizar tu comprensión.
