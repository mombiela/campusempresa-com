Introducción

En Ruby, entender las variables y métodos de clase es crucial para dominar la programación orientada a objetos. Esta sección cubrirá los conceptos básicos de las variables y métodos de clase, su uso y cómo se diferencian de las variables y métodos de instancia.

Variables de Clase

Las variables de clase se comparten entre todas las instancias de una clase. Se prefijan con @@ y se utilizan para almacenar información que es común a todos los objetos de una clase.

Conceptos Clave

  • Declaración de Variables de Clase: Las variables de clase se declaran dentro del cuerpo de la clase, prefijadas con @@.
  • Estado Compartido: Todas las instancias de una clase comparten la misma variable de clase.
  • Inicialización: Las variables de clase pueden inicializarse directamente dentro del cuerpo de la clase o dentro de métodos de clase.

Ejemplo

class Car
  @@number_of_cars = 0

  def initialize(model)
    @model = model
    @@number_of_cars += 1
  end

  def self.total_cars
    @@number_of_cars
  end
end

car1 = Car.new("Toyota")
car2 = Car.new("Honda")

puts Car.total_cars # Output: 2

Explicación:

  • @@number_of_cars es una variable de clase.
  • Se incrementa cada vez que se crea un nuevo objeto Car.
  • self.total_cars es un método de clase que devuelve el valor de @@number_of_cars.

Métodos de Clase

Los métodos de clase son métodos que se llaman en la clase misma, en lugar de en instancias de la clase. Se definen usando self. antes del nombre del método.

Conceptos Clave

  • Declaración de Métodos de Clase: Usa self. antes del nombre del método.
  • Acceso a Variables de Clase: Los métodos de clase pueden acceder y modificar las variables de clase.
  • Funciones de Utilidad: A menudo se utilizan para funciones de utilidad que están relacionadas con la clase pero no con ninguna instancia específica.

Ejemplo

class Calculator
  def self.add(a, b)
    a + b
  end

  def self.subtract(a, b)
    a - b
  end
end

puts Calculator.add(5, 3)      # Output: 8
puts Calculator.subtract(5, 3) # Output: 2

Explicación:

  • self.add y self.subtract son métodos de clase.
  • Realizan operaciones que no dependen de datos específicos de la instancia.

Comparación: Variables de Clase vs Variables de Instancia

| Aspecto | Variables de Clase (@@) | Variables de Instancia (@) | |-----------------------|---------------------------|------------------------------| | Alcance | Compartido entre todas las instancias | Único para cada instancia | | Inicialización | Dentro del cuerpo de la clase o métodos de clase | Dentro de métodos de instancia (e.g., initialize) | | Acceso | Accesible por métodos de clase y de instancia | Accesible solo por métodos de instancia | | Caso de Uso | Datos comunes para todas las instancias | Datos específicos para cada instancia |

Ejercicios

Ejercicio 1: Variable de Clase

Crea una clase Library que haga un seguimiento del número total de libros en todas las instancias.

class Library
  @@total_books = 0

  def initialize(books)
    @books = books
    @@total_books += books
  end

  def self.total_books
    @@total_books
  end
end

# Crear instancias y probar la variable de clase
library1 = Library.new(100)
library2 = Library.new(200)

puts Library.total_books # Expected output: 300

Ejercicio 2: Método de Clase

Crea una clase TemperatureConverter con métodos de clase para convertir temperaturas entre Celsius y Fahrenheit.

class TemperatureConverter
  def self.celsius_to_fahrenheit(celsius)
    (celsius * 9.0 / 5) + 32
  end

  def self.fahrenheit_to_celsius(fahrenheit)
    (fahrenheit - 32) * 5.0 / 9
  end
end

# Probar los métodos de clase
puts TemperatureConverter.celsius_to_fahrenheit(0)    # Expected output: 32
puts TemperatureConverter.fahrenheit_to_celsius(32)   # Expected output: 0

Conclusión

Las variables y métodos de clase son herramientas poderosas en Ruby que te permiten gestionar datos y funcionalidades compartidas a nivel de clase. Entender cuándo y cómo usarlos te ayudará a escribir código más eficiente y organizado. Recuerda:

  • Usa variables de clase para datos compartidos entre todas las instancias.
  • Usa métodos de clase para funcionalidades que pertenecen a la clase misma en lugar de a cualquier instancia específica.

Al dominar estos conceptos, estarás bien preparado para manejar tareas de programación orientada a objetos más complejas en Ruby.
