La herencia es un concepto fundamental en la programación orientada a objetos (OOP) que permite a una clase heredar atributos y métodos de otra clase. Esto ayuda en la reutilización del código y en establecer una jerarquía natural entre las clases.

¿Qué es la Herencia?

  • Herencia permite que una nueva clase (subclase o clase derivada) herede propiedades y comportamientos (métodos) de una clase existente (superclase o clase base).
  • La subclase puede agregar sus propias propiedades y métodos o sobrescribir los heredados.

Sintaxis Básica

En Ruby, la herencia se implementa usando el símbolo <.

class Animal
  def speak
    puts "Sonido genérico de animal"
  end
end

class Dog < Animal
  def speak
    puts "¡Guau!"
  end
end

dog = Dog.new
dog.speak  # Imprime: ¡Guau!

En este ejemplo:

  • Animal es la superclase.
  • Dog es la subclase que hereda de Animal.
  • La clase Dog sobrescribe el método speak.

Conceptos Clave

  • Superclase y Subclase: La superclase es la clase de la que se hereda, y la subclase es la clase que hereda.
  • Sobrescritura de Métodos: Las subclases pueden proporcionar una implementación específica de un método que ya está definido en su superclase.
  • Palabra Clave super: Se usa para llamar a métodos de la superclase dentro de la subclase.

Sobrescritura de Métodos

Cuando una subclase define un método con el mismo nombre que un método en la superclase, el método de la subclase sobrescribe el método de la superclase.

class Animal
  def speak
    puts "Sonido genérico de animal"
  end
end

class Cat < Animal
  def speak
    puts "¡Miau!"
  end
end

cat = Cat.new
cat.speak  # Imprime: ¡Miau!

Usando `super`

La palabra clave super se usa para llamar al mismo método de la superclase.

class Animal
  def speak
    puts "Sonido genérico de animal"
  end
end

class Dog < Animal
  def speak
    super
    puts "¡Guau!"
  end
end

dog = Dog.new
dog.speak
# Imprime:
# Sonido genérico de animal
# ¡Guau!

Herencia e Inicialización

Cuando se inicializa una subclase, puede llamar al inicializador de la superclase usando super.

class Animal
  def initialize(name)
    @name = name
  end

  def display_name
    puts "Nombre: #{@name}"
  end
end

class Dog < Animal
  def initialize(name, breed)
    super(name)
    @breed = breed
  end

  def display_breed
    puts "Raza: #{@breed}"
  end
end

dog = Dog.new("Buddy", "Golden Retriever")
dog.display_name   # Imprime: Nombre: Buddy
dog.display_breed  # Imprime: Raza: Golden Retriever

Herencia y Control de Acceso

Ruby proporciona tres niveles de control de acceso para métodos:

  • Público: Los métodos son accesibles desde cualquier lugar.
  • Protegido: Los métodos son accesibles dentro de la clase y sus subclases.
  • Privado: Los métodos son accesibles solo dentro de la clase.
class Animal
  def public_method
    puts "Método público"
  end

  protected
  def protected_method
    puts "Método protegido"
  end

  private
  def private_method
    puts "Método privado"
  end
end

class Dog < Animal
  def access_methods
    public_method
    protected_method
    # private_method  # Esto generará un error
  end
end

dog = Dog.new
dog.access_methods
# Imprime:
# Método público
# Método protegido

Conclusión

La herencia es una característica poderosa en Ruby que promueve la reutilización del código y una jerarquía lógica de clases. Al entender y utilizar la herencia, puedes crear código más organizado y mantenible. Recuerda usar la sobrescritura de métodos y la palabra clave super de manera efectiva para extender y personalizar el comportamiento de los métodos heredados. El control de acceso ayuda a encapsular y proteger el estado interno de los objetos, asegurando que los métodos se usen adecuadamente dentro de la jerarquía de clases.
