Control de Acceso en Ruby
Lectura de Archivos ►

El control de acceso es un concepto fundamental en la programación orientada a objetos que ayuda a gestionar la visibilidad y accesibilidad de métodos y atributos en una clase. En Ruby, el control de acceso se gestiona utilizando tres palabras clave principales: public, protected y private.

Métodos Públicos

  • Definición: Los métodos públicos pueden ser llamados por cualquiera; no hay restricciones sobre su visibilidad.
  • Por Defecto: Por defecto, todos los métodos en una clase de Ruby son públicos a menos que se especifique lo contrario.
class Greeter
  def greet
    puts "¡Hola!"
  end
end

greeter = Greeter.new
greeter.greet  # Salida: ¡Hola!

Métodos Protegidos

  • Definición: Los métodos protegidos pueden ser llamados por cualquier instancia de la clase definitoria o sus subclases. Sin embargo, no pueden ser llamados con un receptor explícito fuera de la jerarquía de la clase.
  • Caso de Uso: Típicamente se usan para métodos que deben ser accesibles dentro de la clase y sus subclases, pero no desde fuera.
class Person
  def initialize(name)
    @name = name
  end

  def compare_name(other_person)
    self.name == other_person.name
  end

  protected

  def name
    @name
  end
end

alice = Person.new("Alice")
bob = Person.new("Bob")
puts alice.compare_name(bob)  # Salida: false

Métodos Privados

  • Definición: Los métodos privados solo pueden ser llamados dentro del contexto del objeto actual. No pueden ser llamados con un receptor explícito, incluso si ese receptor es self.
  • Caso de Uso: Se usan para métodos de utilidad interna que no deben ser expuestos fuera de la clase.
class BankAccount
  def initialize(balance)
    @balance = balance
  end

  def display_balance
    puts "Tu saldo es $#{balance}"
  end

  private

  def balance
    @balance
  end
end

account = BankAccount.new(1000)
account.display_balance  # Salida: Tu saldo es $1000
# account.balance  # Esto generará un error: private method `balance' called

Palabras Clave de Control de Acceso

  • public: Los métodos que siguen a esta palabra clave son accesibles desde cualquier lugar.
  • protected: Los métodos que siguen a esta palabra clave son accesibles dentro de la clase y sus subclases.
  • private: Los métodos que siguen a esta palabra clave solo son accesibles dentro del contexto del objeto actual.

Ejemplo de Uso de Palabras Clave de Control de Acceso

class Example
  def public_method
    puts "Este es un método público"
  end

  protected

  def protected_method
    puts "Este es un método protegido"
  end

  private

  def private_method
    puts "Este es un método privado"
  end
end

example = Example.new
example.public_method  # Salida: Este es un método público
# example.protected_method  # Genera un error: protected method `protected_method' called
# example.private_method  # Genera un error: private method `private_method' called

Resumen

El control de acceso en Ruby es un concepto crucial para gestionar la visibilidad y accesibilidad de métodos dentro de una clase. Al usar las palabras clave public, protected y private, puedes controlar cómo se acceden los métodos y asegurar que el funcionamiento interno de tu clase esté adecuadamente encapsulado. Entender y aplicar estos mecanismos de control de acceso te ayudará a escribir código más seguro y mantenible.
