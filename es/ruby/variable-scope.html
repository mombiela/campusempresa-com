Introducción al Alcance de Variables

El alcance de una variable se refiere al contexto dentro de un programa donde una variable es accesible. Entender el alcance de las variables es crucial para escribir código eficiente y libre de errores. En Ruby, el alcance de una variable está determinado por dónde se declara la variable y el tipo de variable.

Tipos de Variables en Ruby

Ruby tiene varios tipos de variables, cada una con su propio alcance:

  • Variables Locales: Accesibles solo dentro del bloque de código donde se declaran.
  • Variables de Instancia: Accesibles dentro de la instancia de una clase.
  • Variables de Clase: Compartidas entre todas las instancias de una clase.
  • Variables Globales: Accesibles desde cualquier parte del programa Ruby.
  • Constantes: Accesibles en toda la aplicación pero no deben cambiarse una vez asignadas.

Variables Locales

Las variables locales se definen dentro de un método, bloque o bucle y solo son accesibles dentro de ese contexto.

def ejemplo_metodo
  var_local = "Soy una variable local"
  puts var_local
end

ejemplo_metodo
# Salida: Soy una variable local

puts var_local
# Error: variable local o método `var_local' no definido

Puntos Clave:

  • Las variables locales comienzan con una letra minúscula o un guion bajo.
  • No son accesibles fuera del método o bloque donde se definen.

Variables de Instancia

Las variables de instancia se prefijan con @ y son accesibles dentro de la instancia de una clase.

class EjemploClase
  def initialize(nombre)
    @nombre = nombre
  end

  def mostrar_nombre
    puts @nombre
  end
end

ejemplo = EjemploClase.new("Ruby")
ejemplo.mostrar_nombre
# Salida: Ruby

Puntos Clave:

  • Las variables de instancia se usan para almacenar datos específicos del objeto.
  • Son accesibles a través de métodos dentro de la misma instancia.

Variables de Clase

Las variables de clase se prefijan con @@ y se comparten entre todas las instancias de una clase.

class EjemploClase
  @@var_clase = 0

  def initialize
    @@var_clase += 1
  end

  def self.var_clase
    @@var_clase
  end
end

ejemplo1 = EjemploClase.new
ejemplo2 = EjemploClase.new

puts EjemploClase.var_clase
# Salida: 2

Puntos Clave:

  • Las variables de clase almacenan datos que se comparten entre todas las instancias de una clase.
  • Pueden ser accedidas y modificadas por cualquier instancia de la clase.

Variables Globales

Las variables globales se prefijan con $ y son accesibles desde cualquier parte del programa Ruby.

$var_global = "Soy una variable global"

def mostrar_var_global
  puts $var_global
end

mostrar_var_global
# Salida: Soy una variable global

$var_global = "He sido cambiada"
mostrar_var_global
# Salida: He sido cambiada

Puntos Clave:

  • Las variables globales pueden ser accedidas desde cualquier parte del programa.
  • Deben usarse con moderación ya que pueden dificultar la depuración.

Constantes

Las constantes se definen con una letra mayúscula y están destinadas a permanecer sin cambios.

VAR_CONSTANTE = "Soy una constante"

def mostrar_constante
  puts VAR_CONSTANTE
end

mostrar_constante
# Salida: Soy una constante

# Intentar cambiar una constante generará una advertencia
VAR_CONSTANTE = "He sido cambiada"
# Advertencia: constante VAR_CONSTANTE ya inicializada

Puntos Clave:

  • Las constantes son accesibles en toda la aplicación.
  • No deben reasignarse una vez establecidas.

Resumen

Entender el alcance de las variables es esencial para gestionar datos y evitar errores en los programas Ruby. Aquí tienes un resumen rápido:

  • Variables Locales: Limitadas al método o bloque donde se definen.
  • Variables de Instancia: Accesibles dentro de la instancia de una clase.
  • Variables de Clase: Compartidas entre todas las instancias de una clase.
  • Variables Globales: Accesibles desde cualquier parte del programa.
  • Constantes: Deben permanecer sin cambios y son accesibles en toda la aplicación.

Al dominar el alcance de las variables, puedes escribir código Ruby más eficiente y mantenible.
