Definiendo Servicios en Docker Compose

Docker Compose es una herramienta para definir y ejecutar aplicaciones Docker de múltiples contenedores. Con Compose, utilizas un archivo YAML para configurar los servicios de tu aplicación. Luego, con un solo comando, creas y arrancas todos los servicios desde tu configuración.

Introducción a Docker Compose

Docker Compose te permite definir y gestionar múltiples contenedores como un solo servicio. Esto es particularmente útil para aplicaciones que requieren varios servicios interconectados, como un servidor web, una base de datos y una capa de caché.

Conceptos Clave

  • Servicio: Un servicio es un contenedor en tu aplicación. Se define en el archivo docker-compose.yml.
  • Red: Docker Compose crea automáticamente una red para que tus servicios se comuniquen.
  • Volumen: Los volúmenes se utilizan para persistir datos generados y utilizados por contenedores Docker.

Definiendo Servicios en `docker-compose.yml`

El archivo docker-compose.yml es donde defines tus servicios. Cada servicio es un contenedor que ejecuta una imagen específica.

Estructura Básica de `docker-compose.yml`

Aquí tienes un ejemplo simple de un archivo docker-compose.yml:

version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
  db:
    image: postgres:latest
    environment:
      POSTGRES_PASSWORD: example

Explicación

  • version: Especifica la versión del formato del archivo Docker Compose.
  • services: Define los servicios que componen tu aplicación.
    • web: Este es el nombre del servicio.
      • image: Especifica la imagen Docker a utilizar.
      • ports: Mapea el puerto del host al puerto del contenedor.
    • db: Otro servicio.
      • environment: Establece variables de entorno para el contenedor.

Añadiendo Más Configuración

Puedes añadir más configuraciones a tus servicios, como volúmenes, redes y dependencias.

version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    depends_on:
      - db
  db:
    image: postgres:latest
    environment:
      POSTGRES_PASSWORD: example
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:

Explicación

  • volumes: Monta un directorio del host o un volumen nombrado en el contenedor.
  • depends_on: Especifica dependencias entre servicios. En este caso, el servicio web depende del servicio db.
  • volumes (al final): Define volúmenes nombrados que pueden ser utilizados por los servicios.

Ejemplo: Una Aplicación Web con una Base de Datos

Vamos a crear un ejemplo más completo con una aplicación web que utiliza una base de datos.

version: '3'
services:
  app:
    image: myapp:latest
    build: .
    ports:
      - "5000:5000"
    environment:
      - DATABASE_URL=postgres://user:password@db:5432/mydatabase
    depends_on:
      - db
  db:
    image: postgres:latest
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mydatabase
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:

Explicación

  • app: El servicio de la aplicación web.
    • build: Construye la imagen Docker desde el directorio actual.
    • environment: Establece variables de entorno, incluyendo la URL de la base de datos.
  • db: El servicio de la base de datos.
    • environment: Establece variables de entorno para la base de datos PostgreSQL.
    • volumes: Utiliza un volumen nombrado para persistir los datos de la base de datos.

Conclusión

Definir servicios en Docker Compose te permite gestionar aplicaciones de múltiples contenedores de manera eficiente. Al usar un archivo docker-compose.yml, puedes especificar la configuración para cada servicio, incluyendo imágenes, puertos, volúmenes y dependencias. Esto facilita la configuración y el mantenimiento de aplicaciones complejas con múltiples servicios interconectados.
