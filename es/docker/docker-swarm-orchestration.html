Introducción a Docker Swarm

Docker Swarm es una herramienta nativa de clustering y orquestación para contenedores Docker. Te permite gestionar un clúster de nodos Docker como un único sistema virtual.

Conceptos Clave

  • Swarm: Un grupo de máquinas que están ejecutando Docker y se han unido en un clúster.
  • Nodo: Una máquina individual en un Swarm.
    • Nodo Manager: Responsable de gestionar el Swarm y sus recursos.
    • Nodo Worker: Ejecuta tareas asignadas por el Manager.
  • Servicio: Una definición de cómo ejecutar un contenedor, incluyendo la imagen, puertos y réplicas.
  • Tarea: Una instancia única de un contenedor en ejecución que es parte de un servicio.
  • Red Overlay: Una red que abarca todos los nodos en el Swarm, permitiendo que los contenedores se comuniquen de manera segura.

Configuración de un Docker Swarm

Para configurar un Docker Swarm, necesitas al menos una máquina para actuar como Nodo Manager y una o más Nodos Worker.

Guía Paso a Paso

  1. Inicializar el Swarm en el Nodo Manager:

    docker swarm init --advertise-addr <MANAGER-IP>

    Reemplaza <MANAGER-IP> con la dirección IP de tu Nodo Manager.

  2. Unir Nodos Worker al Swarm: En cada Nodo Worker, ejecuta:

    docker swarm join --token <SWARM-TOKEN> <MANAGER-IP>:2377

    Reemplaza <SWARM-TOKEN> con el token proporcionado por el comando docker swarm init y <MANAGER-IP> con la dirección IP del Nodo Manager.

Gestión de Servicios en Docker Swarm

Una vez que tu Swarm está configurado, puedes comenzar a crear y gestionar servicios.

Creación de un Servicio

Para crear un servicio, usa el comando docker service create:

docker service create --name my-service --replicas 3 -p 80:80 nginx

Este comando crea un servicio llamado my-service con 3 réplicas del contenedor nginx, exponiendo el puerto 80.

Listado de Servicios

Para listar todos los servicios en el Swarm:

docker service ls

Escalado de un Servicio

Para escalar un servicio hacia arriba o hacia abajo:

docker service scale my-service=5

Este comando escala my-service a 5 réplicas.

Actualización de un Servicio

Para actualizar un servicio (por ejemplo, cambiar la versión de la imagen):

docker service update --image nginx:latest my-service

Monitoreo y Solución de Problemas

El monitoreo y la solución de problemas son cruciales para mantener un Swarm saludable.

Visualización de Logs del Servicio

Para ver los logs de un servicio específico:

docker service logs my-service

Inspección de Nodos

Para inspeccionar los detalles de un nodo:

docker node inspect <NODE-ID>

Eliminación de un Nodo

Para eliminar un nodo del Swarm:

docker node rm <NODE-ID>

Conclusión

Docker Swarm proporciona una manera poderosa y sencilla de gestionar aplicaciones en contenedores a escala. Al entender los conceptos clave y los comandos, puedes orquestar y gestionar eficazmente tus contenedores Docker a través de múltiples nodos. Ya sea que estés desplegando un servicio web simple o una arquitectura compleja de microservicios, Docker Swarm ofrece las herramientas que necesitas para asegurar que tus aplicaciones funcionen de manera fluida y eficiente.
