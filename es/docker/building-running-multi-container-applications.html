Introducción

En esta sección, exploraremos cómo construir y ejecutar aplicaciones multi-contenedor usando Docker. Esto es esencial para crear aplicaciones escalables y mantenibles que consisten en múltiples servicios trabajando juntos.

Conceptos Clave

  • Arquitectura de Microservicios: Descomponer aplicaciones en servicios más pequeños e independientes.
  • Docker Compose: Una herramienta para definir y ejecutar aplicaciones Docker multi-contenedor.
  • Dependencias de Servicios: Gestionar cómo los servicios interactúan y dependen unos de otros.
  • Redes: Asegurar que los contenedores puedan comunicarse entre sí.

Fundamentos de Docker Compose

Docker Compose es una herramienta que te permite definir y gestionar aplicaciones Docker multi-contenedor. Utiliza un archivo YAML para configurar los servicios, redes y volúmenes de la aplicación.

Instalación de Docker Compose

Docker Compose está incluido en Docker Desktop para Windows y Mac. Para Linux, puedes instalarlo usando el siguiente comando:

sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

Creación de un Archivo `docker-compose.yml`

Un archivo docker-compose.yml define los servicios que componen tu aplicación. Aquí hay un ejemplo para una aplicación web simple con un servicio frontend y un servicio backend:

version: '3'
services:
  frontend:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./frontend:/usr/share/nginx/html
  backend:
    image: node:14
    working_dir: /app
    volumes:
      - ./backend:/app
    command: npm start

Ejecución de Docker Compose

Para iniciar la aplicación, navega al directorio que contiene tu archivo docker-compose.yml y ejecuta:

docker-compose up

Este comando iniciará todos los servicios definidos en el archivo docker-compose.yml.

Dependencias de Servicios

En una aplicación multi-contenedor, los servicios a menudo dependen unos de otros. Docker Compose te permite especificar dependencias usando la opción depends_on.

Ejemplo con Dependencias

Aquí hay un ejemplo donde el servicio backend depende de un servicio de base de datos:

version: '3'
services:
  db:
    image: postgres:latest
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mydb
  backend:
    image: node:14
    working_dir: /app
    volumes:
      - ./backend:/app
    command: npm start
    depends_on:
      - db

Redes

Docker Compose crea automáticamente una red para tu aplicación, permitiendo que los servicios se comuniquen entre sí por sus nombres de servicio.

Ejemplo con Redes

En el ejemplo anterior, el servicio backend puede conectarse al servicio de base de datos usando el nombre de servicio db:

const { Client } = require('pg');
const client = new Client({
  host: 'db',
  user: 'user',
  password: 'password',
  database: 'mydb'
});
client.connect();

Funciones Avanzadas de Docker Compose

Docker Compose ofrece varias funciones avanzadas para gestionar aplicaciones complejas.

Variables de Entorno

Puedes usar variables de entorno para configurar tus servicios dinámicamente.

version: '3'
services:
  backend:
    image: node:14
    working_dir: /app
    volumes:
      - ./backend:/app
    command: npm start
    environment:
      - NODE_ENV=production

Volúmenes

Los volúmenes te permiten persistir datos y compartirlos entre servicios.

version: '3'
services:
  db:
    image: postgres:latest
    volumes:
      - db-data:/var/lib/postgresql/data
volumes:
  db-data:

Escalado de Servicios

Puedes escalar servicios para manejar más carga especificando el número de instancias.

docker-compose up --scale backend=3

Conclusión

Construir y ejecutar aplicaciones multi-contenedor con Docker Compose simplifica la gestión de aplicaciones complejas. Al descomponer aplicaciones en servicios más pequeños, puedes lograr mejor escalabilidad y mantenibilidad. Docker Compose proporciona funciones poderosas para gestionar dependencias de servicios, redes y escalado, convirtiéndolo en una herramienta esencial para el desarrollo de aplicaciones modernas.
