Introducción a CI/CD

  • Integración Continua (CI): La práctica de integrar automáticamente los cambios de código de múltiples contribuyentes en un repositorio compartido varias veces al día.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio que pasa todas las etapas de tu pipeline de producción a producción.

Beneficios de Usar Docker en CI/CD

  • Consistencia: Docker asegura que la aplicación se ejecute de la misma manera sin importar dónde se despliegue.
  • Aislamiento: Cada aplicación se ejecuta en su propio contenedor, evitando conflictos.
  • Escalabilidad: Los contenedores Docker pueden ser fácilmente escalados hacia arriba o hacia abajo.

Configuración de un Pipeline CI/CD con Docker

Requisitos Previos

  • Conocimientos básicos de Docker.
  • Un sistema de control de versiones (por ejemplo, Git).
  • Una herramienta de CI/CD (por ejemplo, Jenkins, GitLab CI, GitHub Actions).

Guía Paso a Paso

1. Dockerizar Tu Aplicación

  • Crea un Dockerfile para definir el entorno de tu aplicación.
# Usa una imagen oficial de Python como imagen base
FROM python:3.8-slim

# Establece el directorio de trabajo en el contenedor
WORKDIR /app

# Copia el contenido del directorio actual al contenedor en /app
COPY . /app

# Instala los paquetes necesarios especificados en requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Haz que el puerto 80 esté disponible para el mundo fuera de este contenedor
EXPOSE 80

# Define una variable de entorno
ENV NAME World

# Ejecuta app.py cuando se inicie el contenedor
CMD ["python", "app.py"]

2. Construir y Probar la Imagen Docker

  • Construye la imagen Docker localmente para asegurarte de que funcione como se espera.
docker build -t my-python-app .
  • Ejecuta el contenedor Docker para probarlo.
docker run -p 4000:80 my-python-app

3. Configuración de una Herramienta de CI/CD

  • Elige una herramienta de CI/CD y configúrala para usar tu Dockerfile.
Ejemplo: GitHub Actions
  • Crea un archivo .github/workflows/main.yml en tu repositorio.
name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v1

    - name: Build Docker image
      run: docker build -t my-python-app .

    - name: Run tests
      run: docker run my-python-app pytest

4. Desplegar el Contenedor Docker

  • Sube la imagen Docker a un registro de contenedores (por ejemplo, Docker Hub).
docker tag my-python-app mydockerhubusername/my-python-app
docker push mydockerhubusername/my-python-app
  • Configura tu herramienta de CI/CD para desplegar la imagen en tu entorno de producción.
Ejemplo: GitHub Actions (continuación)
    - name: Log in to Docker Hub
      run: echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin

    - name: Push Docker image
      run: docker push mydockerhubusername/my-python-app

    - name: Deploy to production
      run: |
        ssh user@yourserver "docker pull mydockerhubusername/my-python-app && docker run -d -p 80:80 mydockerhubusername/my-python-app"

Conclusión

En este tema, cubrimos lo esencial para configurar un pipeline CI/CD usando Docker. Comenzamos con una introducción a los conceptos de CI/CD y los beneficios de usar Docker. Luego, recorrimos los pasos para Dockerizar una aplicación, construir y probar la imagen Docker, configurar una herramienta de CI/CD y desplegar el contenedor Docker. Siguiendo estos pasos, puedes asegurar un proceso de despliegue consistente, aislado y escalable para tus aplicaciones.
