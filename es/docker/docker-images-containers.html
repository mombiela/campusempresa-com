Introducción a Imágenes y Contenedores Docker

Docker es una plataforma poderosa para desarrollar, enviar y ejecutar aplicaciones dentro de contenedores. Para usar Docker de manera efectiva, es crucial entender los conceptos de imágenes y contenedores Docker.

¿Qué es una Imagen Docker?

Una imagen Docker es un paquete de software ligero, autónomo y ejecutable que incluye todo lo necesario para ejecutar una pieza de software, incluyendo el código, el tiempo de ejecución, las bibliotecas, las variables de entorno y los archivos de configuración.

Conceptos Clave de las Imágenes Docker

  • Capas: Las imágenes Docker se construyen en capas, donde cada capa representa una instrucción en el Dockerfile de la imagen.
  • Imagen Base: El punto de partida para construir una imagen Docker, a menudo una imagen oficial como ubuntu o alpine.
  • Dockerfile: Un archivo de texto que contiene una serie de instrucciones sobre cómo construir una imagen Docker.

Ejemplo de un Dockerfile

# Usar un runtime oficial de Python como imagen base
FROM python:3.8-slim

# Establecer el directorio de trabajo en el contenedor
WORKDIR /app

# Copiar el contenido del directorio actual al contenedor en /app
COPY . /app

# Instalar cualquier paquete necesario especificado en requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Hacer el puerto 80 disponible para el mundo fuera de este contenedor
EXPOSE 80

# Definir una variable de entorno
ENV NAME World

# Ejecutar app.py cuando el contenedor se inicie
CMD ["python", "app.py"]

Explicación:

  • FROM: Especifica la imagen base.
  • WORKDIR: Establece el directorio de trabajo dentro del contenedor.
  • COPY: Copia archivos del host al contenedor.
  • RUN: Ejecuta comandos en el contenedor.
  • EXPOSE: Informa a Docker que el contenedor escucha en los puertos de red especificados en tiempo de ejecución.
  • ENV: Establece variables de entorno.
  • CMD: Proporciona el comando para ejecutar dentro del contenedor.

¿Qué es un Contenedor Docker?

Un contenedor Docker es una instancia ejecutable de una imagen Docker. Los contenedores están aislados entre sí y del sistema host, pero pueden interactuar entre sí a través de canales bien definidos.

Conceptos Clave de los Contenedores Docker

  • Aislamiento: Los contenedores se ejecutan en entornos aislados, asegurando que las aplicaciones no interfieran entre sí.
  • Portabilidad: Los contenedores pueden ejecutarse en cualquier sistema que soporte Docker, haciéndolos altamente portables.
  • Efímero: Los contenedores están diseñados para ser transitorios y pueden detenerse y eliminarse fácilmente.

Ejecutando un Contenedor Docker

# Descargar la imagen desde Docker Hub
docker pull python:3.8-slim

# Ejecutar un contenedor desde la imagen
docker run -d -p 4000:80 --name my-python-app python:3.8-slim

Explicación:

  • docker pull: Descarga la imagen especificada desde Docker Hub.
  • docker run: Crea y arranca un nuevo contenedor desde la imagen especificada.
    • -d: Ejecuta el contenedor en modo desacoplado.
    • -p 4000:80: Mapea el puerto 4000 en el host al puerto 80 en el contenedor.
    • --name: Asigna un nombre al contenedor.

Comparando Imágenes y Contenedores Docker

| Característica | Imagen Docker | Contenedor Docker | |---------------|---------------------------------------------|----------------------------------------------| | Definición | Una plantilla con instrucciones para crear un contenedor | Una instancia ejecutable de una imagen Docker | | Estado | Estático (solo lectura) | Dinámico (lectura-escritura) | | Almacenamiento | Almacenado en disco como una serie de capas | Se ejecuta en memoria, con cambios almacenados en una capa de escritura | | Creación | Construido usando un Dockerfile | Creado a partir de una imagen Docker usando docker run | | Portabilidad | Puede ser compartido y reutilizado | Específico del sistema host donde se está ejecutando |

Conclusión

Entender las imágenes y contenedores Docker es fundamental para dominar Docker. Las imágenes son los planos para tus contenedores, conteniendo todos los componentes necesarios para ejecutar tus aplicaciones. Los contenedores, por otro lado, son las instancias vivas de estas imágenes, proporcionando entornos aislados para que tus aplicaciones se ejecuten.

Al comprender estos conceptos, puedes construir, compartir y ejecutar aplicaciones de manera eficiente en un modo consistente y reproducible, aprovechando todo el poder de Docker.
