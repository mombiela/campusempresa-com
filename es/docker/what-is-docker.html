Introducción

Docker es una plataforma de código abierto que automatiza la implementación, escalado y gestión de aplicaciones. Utiliza tecnología de contenedorización para empaquetar una aplicación y sus dependencias en una unidad estandarizada llamada contenedor. Esto asegura que la aplicación se ejecute de manera consistente en diferentes entornos.

Conceptos Clave

  • Contenedores: Paquetes ligeros, autónomos y ejecutables que incluyen todo lo necesario para ejecutar un software, incluyendo el código, tiempo de ejecución, bibliotecas y herramientas del sistema.
  • Imágenes: Plantillas de solo lectura utilizadas para crear contenedores. Una imagen incluye el código de la aplicación, bibliotecas y dependencias.
  • Motor Docker: La parte central de Docker, responsable de crear y ejecutar contenedores Docker.
  • Docker Hub: Un repositorio basado en la nube donde los usuarios de Docker pueden crear, probar, almacenar y distribuir imágenes de contenedores.

¿Por qué usar Docker?

  • Consistencia: Asegura que las aplicaciones se ejecuten de la misma manera en entornos de desarrollo, prueba y producción.
  • Aislamiento: Cada contenedor se ejecuta en su propio entorno aislado, lo que ayuda a evitar conflictos entre diferentes aplicaciones.
  • Eficiencia: Los contenedores comparten el kernel y los recursos del sistema anfitrión, lo que los hace más ligeros y rápidos de iniciar en comparación con las máquinas virtuales.
  • Portabilidad: Los contenedores pueden ejecutarse en cualquier sistema que soporte Docker, lo que facilita mover aplicaciones entre diferentes entornos.

Flujo de Trabajo Básico de Docker

  1. Escribir un Dockerfile: Un archivo de texto que contiene instrucciones sobre cómo construir una imagen Docker.
  2. Construir la Imagen: Usar el comando docker build para crear una imagen a partir del Dockerfile.
  3. Ejecutar el Contenedor: Usar el comando docker run para crear y iniciar un contenedor a partir de la imagen.
  4. Gestionar Contenedores: Usar varios comandos Docker para gestionar contenedores en ejecución.

Ejemplo: Creando un Contenedor Docker Simple

Paso 1: Escribir un Dockerfile

Crear un archivo llamado Dockerfile con el siguiente contenido:

# Usar una imagen oficial de Python como imagen base
FROM python:3.8-slim

# Establecer el directorio de trabajo en el contenedor
WORKDIR /app

# Copiar el contenido del directorio actual al contenedor en /app
COPY . /app

# Instalar cualquier paquete necesario especificado en requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Hacer que el puerto 80 esté disponible para el mundo fuera de este contenedor
EXPOSE 80

# Definir una variable de entorno
ENV NAME World

# Ejecutar app.py cuando el contenedor se inicie
CMD ["python", "app.py"]

Paso 2: Construir la Imagen Docker

Ejecutar el siguiente comando en la terminal:

docker build -t my-python-app .

Paso 3: Ejecutar el Contenedor Docker

Ejecutar el siguiente comando para iniciar un contenedor a partir de la imagen:

docker run -p 4000:80 my-python-app

Paso 4: Acceder a la Aplicación

Abrir un navegador web y navegar a http://localhost:4000 para ver tu aplicación en ejecución.

Conclusión

Docker es una herramienta poderosa que simplifica el proceso de desarrollo, prueba y despliegue de aplicaciones mediante el uso de contenedores. Asegura consistencia en diferentes entornos, proporciona aislamiento y mejora la eficiencia y portabilidad. Al entender los conceptos básicos y el flujo de trabajo, puedes comenzar a aprovechar Docker para optimizar tus procesos de desarrollo y despliegue.
