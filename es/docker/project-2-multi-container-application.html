Introducción

En este proyecto, construiremos una aplicación multi-contenedor utilizando Docker. Este proyecto te ayudará a entender cómo orquestar múltiples servicios, gestionar dependencias y asegurar una comunicación fluida entre contenedores.

Conceptos Clave

  • Docker Compose: Una herramienta para definir y ejecutar aplicaciones Docker multi-contenedor.
  • Servicio: Un contenedor en un archivo Docker Compose.
  • Red: Una red Docker que permite la comunicación entre contenedores.
  • Volumen: Un mecanismo de almacenamiento persistente para contenedores Docker.

Configuración del Proyecto

Para comenzar, crearemos una aplicación multi-contenedor simple que consiste en un servidor web y una base de datos.

Paso 1: Crear el Directorio del Proyecto

Primero, crea un directorio para tu proyecto:

mkdir multi-container-app
cd multi-container-app

Paso 2: Definir el Archivo Docker Compose

Crea un archivo docker-compose.yml para definir los servicios:

version: '3.8'

services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    networks:
      - app-network

  db:
    image: postgres:latest
    environment:
      POSTGRES_USER: example
      POSTGRES_PASSWORD: example
    volumes:
      - db-data:/var/lib/postgresql/data
    networks:
      - app-network

networks:
  app-network:
    driver: bridge

volumes:
  db-data:

Explicación

  • version: Especifica la versión de Docker Compose.
  • services: Define los contenedores a ejecutar.
    • web: El servicio del servidor web utilizando la imagen de Nginx.
      • ports: Mapea el puerto 80 en el host al puerto 80 en el contenedor.
      • networks: Conecta el contenedor a la app-network.
    • db: El servicio de base de datos utilizando la imagen de PostgreSQL.
      • environment: Establece variables de entorno para el usuario y la contraseña de la base de datos.
      • volumes: Monta un volumen para almacenamiento persistente.
      • networks: Conecta el contenedor a la app-network.
  • networks: Define una red personalizada llamada app-network.
  • volumes: Define un volumen llamado db-data para almacenamiento persistente.

Ejecutando la Aplicación

Para ejecutar la aplicación, utiliza el siguiente comando:

docker-compose up -d
  • La bandera -d ejecuta los contenedores en modo desacoplado.

Verificando los Servicios

Puedes verificar los servicios en ejecución utilizando:

docker-compose ps

Interactuando con los Contenedores

Accediendo al Servidor Web

Abre tu navegador y navega a http://localhost. Deberías ver la página de bienvenida predeterminada de Nginx.

Accediendo a la Base de Datos

Puedes interactuar con la base de datos PostgreSQL utilizando un cliente de PostgreSQL o la herramienta de línea de comandos psql.

Deteniendo la Aplicación

Para detener la aplicación, utiliza:

docker-compose down
  • Este comando detiene y elimina los contenedores, redes y volúmenes definidos en el archivo docker-compose.yml.

Conclusión

En este proyecto, has aprendido a:

  • Definir y ejecutar una aplicación multi-contenedor utilizando Docker Compose.
  • Configurar servicios con configuraciones específicas.
  • Gestionar redes y volúmenes para la comunicación entre contenedores y almacenamiento persistente.

Al dominar estos conceptos, ahora puedes construir y gestionar aplicaciones multi-contenedor más complejas, mejorando tus habilidades en Docker desde un nivel principiante hasta avanzado.
