Introducción a las Redes en Docker

  • Conceptos Básicos de Redes en Docker: Entiende cómo maneja Docker las redes por defecto.
  • Red de Puente (Bridge Network): Aprende sobre el tipo de red predeterminado en Docker.
  • Red de Host (Host Network): Explora cómo los contenedores pueden compartir el espacio de nombres de red del host.
  • Red Superpuesta (Overlay Network): Descubre cómo conectar múltiples demonios de Docker juntos.

Configuración de Docker Compose

  • Instalación: Pasos para instalar Docker Compose en varios sistemas operativos.
  • Comandos Básicos: Familiarízate con los comandos esenciales de Docker Compose como up, down, ps y logs.

Definición de Redes en Docker Compose

  • Estructura del Archivo Compose: Entiende la estructura de un archivo docker-compose.yml.
  • Configuración de Redes: Aprende cómo definir redes personalizadas dentro de un archivo docker-compose.yml.
version: '3'
services:
  web:
    image: nginx
    networks:
      - webnet
  db:
    image: mysql
    networks:
      - webnet

networks:
  webnet:

Explicación:

  • version: Especifica la versión del formato del archivo Docker Compose.
  • services: Define los servicios a ejecutar.
    • web: Un servicio que usa la imagen nginx.
    • db: Un servicio que usa la imagen mysql.
  • networks: Define redes personalizadas.
    • webnet: Una red personalizada que ambos servicios usarán.

Conexión de Servicios

  • Descubrimiento de Servicios: Entiende cómo los servicios pueden descubrirse entre sí usando DNS.
  • Alias de Red: Aprende cómo establecer alias de red para los servicios.
version: '3'
services:
  web:
    image: nginx
    networks:
      webnet:
        aliases:
          - myweb
  db:
    image: mysql
    networks:
      webnet:
        aliases:
          - mydb

networks:
  webnet:

Explicación:

  • aliases: Proporciona nombres alternativos para los servicios dentro de la red.

Conceptos Avanzados de Redes

  • Múltiples Redes: Aprende cómo conectar servicios a múltiples redes.
version: '3'
services:
  web:
    image: nginx
    networks:
      - frontend
      - backend
  db:
    image: mysql
    networks:
      - backend

networks:
  frontend:
  backend:

Explicación:

  • frontend: Una red para servicios de frontend.

  • backend: Una red para servicios de backend.

  • Controladores de Red: Explora diferentes controladores de red como bridge, host, overlay y macvlan.

| Controlador de Red | Descripción | |--------------------|-------------| | bridge | Controlador de red predeterminado, adecuado para contenedores independientes. | | host | Comparte el espacio de nombres de red del host. | | overlay | Conecta múltiples demonios de Docker juntos. | | macvlan | Asigna una dirección MAC a cada contenedor para acceso directo a la red. |

  2. Ejercicio 2: Modifica el archivo Docker Compose para agregar alias de red.

Conclusión

En este tema, has aprendido los fundamentos de las redes en Docker y cómo aprovechar Docker Compose para gestionar configuraciones de red complejas. Al definir redes personalizadas, establecer alias de red y conectar servicios a múltiples redes, puedes crear aplicaciones en contenedores robustas y escalables. Continúa practicando con los ejercicios proporcionados para solidificar tu comprensión y prepararte para escenarios de redes en Docker más avanzados.
