Introducción

Gestionar imágenes Docker es una habilidad fundamental para cualquiera que trabaje con Docker. Este tema cubrirá los conceptos básicos de las imágenes Docker, cómo crearlas, gestionarlas y optimizarlas, y las mejores prácticas para usar imágenes Docker en tus proyectos.

Conceptos Clave

  • Imagen Docker: Una plantilla de solo lectura que contiene un conjunto de instrucciones para crear un contenedor que puede ejecutarse en la plataforma Docker.
  • Contenedor Docker: Una instancia ejecutable de una imagen Docker.
  • Dockerfile: Un archivo de texto que contiene una serie de instrucciones sobre cómo construir una imagen Docker.

Creación de Imágenes Docker

Para crear una imagen Docker, necesitas escribir un Dockerfile. Aquí hay un ejemplo simple:

# Usar un runtime oficial de Python como imagen base
FROM python:3.8-slim

# Establecer el directorio de trabajo en el contenedor
WORKDIR /app

# Copiar el contenido del directorio actual en el contenedor en /app
COPY . /app

# Instalar cualquier paquete necesario especificado en requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Hacer que el puerto 80 esté disponible para el mundo fuera de este contenedor
EXPOSE 80

# Definir variable de entorno
ENV NAME World

# Ejecutar app.py cuando se inicie el contenedor
CMD ["python", "app.py"]

Explicación

  • FROM: Especifica la imagen base a usar.
  • WORKDIR: Establece el directorio de trabajo dentro del contenedor.
  • COPY: Copia archivos del host al contenedor.
  • RUN: Ejecuta comandos en el contenedor.
  • EXPOSE: Informa a Docker que el contenedor escucha en los puertos de red especificados en tiempo de ejecución.
  • ENV: Establece variables de entorno.
  • CMD: Proporciona el comando para ejecutar dentro del contenedor.

Construcción de Imágenes Docker

Para construir una imagen Docker a partir de un Dockerfile, usa el comando docker build:

docker build -t my-python-app .
  • -t my-python-app: Etiqueta la imagen con el nombre my-python-app.
  • .: Especifica el contexto de construcción, que es el directorio actual.

Gestión de Imágenes Docker

Listado de Imágenes Docker

Para listar todas las imágenes Docker en tu sistema, usa el comando docker images:

docker images

Eliminación de Imágenes Docker

Para eliminar una imagen Docker, usa el comando docker rmi seguido del ID o nombre de la imagen:

docker rmi my-python-app

Etiquetado de Imágenes Docker

El etiquetado es útil para versionar tus imágenes. Usa el comando docker tag:

docker tag my-python-app my-python-app:v1.0

Subida de Imágenes Docker a un Registro

Para compartir tus imágenes Docker, puedes subirlas a un registro Docker como Docker Hub. Primero, inicia sesión en Docker Hub:

docker login

Luego, sube tu imagen:

docker push my-python-app:v1.0

Mejores Prácticas

  • Minimizar el Tamaño de la Imagen: Usa imágenes base más pequeñas y limpia archivos innecesarios para reducir el tamaño de la imagen.
  • Caché de Capas: Aprovecha el caché de capas de Docker ordenando las instrucciones de tu Dockerfile de menos a más frecuentemente cambiantes.
  • Seguridad: Actualiza regularmente tus imágenes base y dependencias para incluir los últimos parches de seguridad.

Conclusión

Gestionar imágenes Docker implica crear, construir, etiquetar y subir imágenes a un registro. Siguiendo las mejores prácticas, puedes optimizar tus imágenes Docker para rendimiento y seguridad. Entender estos conceptos es crucial para un uso eficiente de Docker en cualquier entorno de desarrollo o producción.
