Introducción

En este proyecto, exploraremos cómo desplegar contenedores Docker en un clúster de Kubernetes. Kubernetes es una poderosa herramienta de orquestación que automatiza el despliegue, escalado y gestión de aplicaciones en contenedores. Al final de este proyecto, tendrás una comprensión sólida de cómo desplegar y gestionar contenedores Docker utilizando Kubernetes.

Conceptos Clave

  • Clúster de Kubernetes: Un conjunto de nodos (máquinas) que ejecutan aplicaciones en contenedores gestionadas por Kubernetes.
  • Pods: Las unidades más pequeñas desplegables en Kubernetes, que pueden contener uno o más contenedores.
  • Servicios: Abstracciones que definen un conjunto lógico de Pods y una política para acceder a ellos.
  • Despliegues: Proveen actualizaciones declarativas a aplicaciones y gestionan el despliegue de Pods.
  • ConfigMaps y Secrets: Mecanismos para inyectar datos de configuración en los Pods.

Prerrequisitos

Antes de comenzar este proyecto, asegúrate de tener:

  • Una comprensión básica de Docker y la contenedorización.
  • Docker instalado en tu máquina local.
  • Minikube o un clúster de Kubernetes configurado.

Guía Paso a Paso

Paso 1: Configurando Minikube

Minikube es una herramienta que te permite ejecutar Kubernetes localmente. Crea un clúster de Kubernetes de un solo nodo en tu máquina local.

  1. Instalar Minikube:

    • Sigue las instrucciones de instalación para tu sistema operativo en la documentación oficial de Minikube.

  2. Iniciar Minikube:

    minikube start

  3. Verificar el Estado de Minikube:

    minikube status

Paso 2: Creando una Imagen Docker

Crea una imagen Docker simple para desplegar en Kubernetes.

  1. Dockerfile:

    FROM nginx:alpine
COPY index.html /usr/share/nginx/html

  2. Construir la Imagen Docker:

    docker build -t my-nginx .

  3. Verificar la Imagen:

    docker images

Paso 3: Desplegando en Kubernetes

Despliega la imagen Docker en el clúster de Kubernetes.

  1. Crear un Despliegue:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: my-nginx:latest
        ports:
        - containerPort: 80

  2. Aplicar el Despliegue:

    kubectl apply -f deployment.yaml

  3. Verificar el Despliegue:

    kubectl get deployments
kubectl get pods

Paso 4: Exponiendo el Despliegue

Expón el despliegue para hacerlo accesible desde fuera del clúster.

  1. Crear un Servicio:

    apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer

  2. Aplicar el Servicio:

    kubectl apply -f service.yaml

  3. Obtener la URL del Servicio:

    minikube service nginx-service --url

Paso 5: Gestionando Configuraciones

Usa ConfigMaps y Secrets para gestionar configuraciones.

  1. Crear un ConfigMap:

    kubectl create configmap nginx-config --from-file=nginx.conf

  2. Crear un Secret:

    kubectl create secret generic my-secret --from-literal=password=my-password

  3. Usar ConfigMap y Secret en el Despliegue:

    spec:
  containers:
  - name: nginx
    image: my-nginx:latest
    ports:
    - containerPort: 80
    volumeMounts:
    - name: config-volume
      mountPath: /etc/nginx/nginx.conf
      subPath: nginx.conf
  volumes:
  - name: config-volume
    configMap:
      name: nginx-config

Conclusión

En este proyecto, cubrimos los pasos esenciales para desplegar contenedores Docker en un clúster de Kubernetes. Comenzamos configurando Minikube, creamos una imagen Docker, la desplegamos en Kubernetes, la expusimos a través de un servicio y gestionamos configuraciones utilizando ConfigMaps y Secrets. Siguiendo estos pasos, ahora tienes el conocimiento fundamental para desplegar y gestionar aplicaciones en contenedores utilizando Kubernetes.
