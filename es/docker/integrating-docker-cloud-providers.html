Introducción

Integrar Docker con proveedores de nube te permite desplegar, gestionar y escalar aplicaciones en contenedores sin problemas. Esta sección cubrirá los conceptos básicos de la integración de Docker con proveedores de nube populares como AWS, Google Cloud Platform (GCP) y Microsoft Azure.

Conceptos Clave

  • Orquestación de Contenedores: Herramientas como Kubernetes y Docker Swarm ayudan a gestionar contenedores en un entorno de nube.
  • Servicios en la Nube: Cada proveedor de nube ofrece servicios para ejecutar contenedores Docker, como AWS ECS, GCP GKE y Azure AKS.
  • Pipelines CI/CD: Los pipelines de Integración Continua y Despliegue Continuo pueden integrarse con Docker para automatizar el proceso de despliegue.

AWS (Amazon Web Services)

Configuración de Docker en AWS ECS (Elastic Container Service)

  1. Crear un Clúster ECS:

    • Ve al panel de ECS.
    • Haz clic en "Create Cluster".
    • Elige la plantilla de clúster adecuada (por ejemplo, EC2 Linux + Networking).
    • Configura los ajustes del clúster y créalo.

  2. Definir una Definición de Tarea:

    • Ve a la sección de Definiciones de Tareas.
    • Haz clic en "Create new Task Definition".
    • Elige el tipo de lanzamiento (EC2 o Fargate).
    • Define los ajustes del contenedor, incluyendo la imagen Docker, memoria y requisitos de CPU.

  3. Ejecutar una Tarea:

    • Ve al clúster ECS que creaste.
    • Haz clic en "Tasks" y luego en "Run new Task".
    • Selecciona la definición de tarea y configura los ajustes de la tarea.
    • Haz clic en "Run Task".
{
  "family": "my-task",
  "containerDefinitions": [
    {
      "name": "my-container",
      "image": "my-docker-image",
      "memory": 512,
      "cpu": 256,
      "essential": true
    }
  ]
}

Usando AWS Fargate

AWS Fargate te permite ejecutar contenedores sin gestionar servidores.

  • Crear un Clúster Fargate: Similar a crear un clúster ECS pero elige el tipo de lanzamiento Fargate.
  • Definir una Tarea Fargate: Similar a definir una tarea ECS pero con ajustes específicos de Fargate.

Google Cloud Platform (GCP)

Configuración de Docker en GKE (Google Kubernetes Engine)

  1. Crear un Clúster GKE:

    • Ve al panel de GKE.
    • Haz clic en "Create Cluster".
    • Configura los ajustes del clúster y créalo.

  2. Desplegar un Contenedor Docker:

    • Usa kubectl para interactuar con el clúster GKE.
    • Crea un archivo YAML de despliegue para tu contenedor Docker.
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-container
        image: my-docker-image
        ports:
        - containerPort: 80
  1. Desplegar la Aplicación:
    • Aplica el despliegue usando kubectl apply -f deployment.yaml.
    • Expón el despliegue usando kubectl expose deployment my-deployment --type=LoadBalancer --port=80.

Microsoft Azure

Configuración de Docker en Azure AKS (Azure Kubernetes Service)

  1. Crear un Clúster AKS:

    • Ve al panel de AKS.
    • Haz clic en "Add" para crear un nuevo clúster.
    • Configura los ajustes del clúster y créalo.

  2. Desplegar un Contenedor Docker:

    • Usa az aks get-credentials para configurar kubectl para usar el clúster AKS.
    • Crea un archivo YAML de despliegue similar al ejemplo de GKE.

  3. Desplegar la Aplicación:

    • Aplica el despliegue usando kubectl apply -f deployment.yaml.
    • Expón el despliegue usando kubectl expose deployment my-deployment --type=LoadBalancer --port=80.

Integración CI/CD

Usando Jenkins con Docker y Proveedores de Nube

  1. Instalar Jenkins: Configura Jenkins en un servidor o usa un servicio Jenkins gestionado.
  2. Configurar Pipeline de Jenkins:
    • Crea un Jenkinsfile en tu repositorio.
    • Define etapas para construir, probar y desplegar contenedores Docker.
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                script {
                    docker.build('my-docker-image')
                }
            }
        }
        stage('Deploy') {
            steps {
                script {
                    // Ejemplo para AWS ECS
                    sh 'aws ecs update-service --cluster my-cluster --service my-service --force-new-deployment'
                }
            }
        }
    }
}

Conclusión

Integrar Docker con proveedores de nube como AWS, GCP y Azure te permite aprovechar la escalabilidad y flexibilidad de la nube para gestionar tus aplicaciones en contenedores de manera efectiva. Al entender los servicios y herramientas específicos que ofrece cada proveedor, puedes desplegar, gestionar y escalar tus aplicaciones con facilidad. Además, integrar Docker con pipelines CI/CD puede automatizar el proceso de despliegue, asegurando que tus aplicaciones estén siempre actualizadas y funcionando sin problemas.
