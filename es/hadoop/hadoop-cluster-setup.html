Configurar un clúster de Hadoop es un paso crucial para aprovechar el poder de la computación distribuida para el procesamiento de grandes volúmenes de datos. Esta sección te guiará a través del proceso de configuración de un clúster de Hadoop desde cero, cubriendo tanto configuraciones de un solo nodo como de múltiples nodos.

Requisitos Previos

Antes de configurar un clúster de Hadoop, asegúrate de tener lo siguiente:

  • Un conocimiento básico de los comandos de Linux y del sistema de archivos.
  • Java Development Kit (JDK) instalado en todos los nodos.
  • Acceso SSH entre nodos sin contraseña.

Configuración de un Clúster de Hadoop de un Solo Nodo

Una configuración de un solo nodo es ideal para propósitos de aprendizaje y desarrollo. Te permite ejecutar Hadoop en una sola máquina.

Paso 1: Instalar Java

Hadoop requiere Java para ejecutarse. Asegúrate de que Java esté instalado y configurado correctamente.

sudo apt-get update
sudo apt-get install openjdk-8-jdk
java -version

Paso 2: Descargar y Extraer Hadoop

Descarga la última versión estable de Hadoop desde el sitio web oficial y extráelo.

wget https://downloads.apache.org/hadoop/common/hadoop-3.3.1/hadoop-3.3.1.tar.gz
tar -xzvf hadoop-3.3.1.tar.gz
mv hadoop-3.3.1 /usr/local/hadoop

Paso 3: Configurar Variables de Entorno de Hadoop

Agrega las variables de entorno de Hadoop al archivo .bashrc.

export HADOOP_HOME=/usr/local/hadoop
export HADOOP_INSTALL=$HADOOP_HOME
export HADOOP_MAPRED_HOME=$HADOOP_HOME
export HADOOP_COMMON_HOME=$HADOOP_HOME
export HADOOP_HDFS_HOME=$HADOOP_HOME
export YARN_HOME=$HADOOP_HOME
export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_HOME/lib/native
export PATH=$PATH:$HADOOP_HOME/sbin:$HADOOP_HOME/bin

Aplica los cambios:

source ~/.bashrc

Paso 4: Configurar Hadoop

Edita los siguientes archivos de configuración ubicados en el directorio $HADOOP_HOME/etc/hadoop:

  • core-site.xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
</configuration>
  • hdfs-site.xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
</configuration>
  • mapred-site.xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
</configuration>
  • yarn-site.xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
</configuration>

Paso 5: Formatear el Namenode

Formatea el sistema de archivos de Hadoop (HDFS) antes de iniciar el clúster.

hdfs namenode -format

Paso 6: Iniciar los Daemons de Hadoop

Inicia los daemons de Hadoop (Namenode, Datanode, ResourceManager y NodeManager).

start-dfs.sh
start-yarn.sh

Paso 7: Verificar la Configuración

Revisa las interfaces web de Hadoop para asegurarte de que todo esté funcionando correctamente:

  • HDFS: http://localhost:9870
  • YARN: http://localhost:8088

Configuración de un Clúster de Hadoop de Múltiples Nodos

Una configuración de múltiples nodos se utiliza para entornos de producción para manejar el procesamiento de datos a gran escala.

Paso 1: Configurar Autenticación SSH Basada en Claves

Genera una clave SSH en el nodo maestro y cópiala a todos los nodos esclavos.

ssh-keygen -t rsa
ssh-copy-id user@slave1
ssh-copy-id user@slave2

Paso 2: Configurar Hadoop en Nodos Maestros y Esclavos

Repite los pasos de configuración de un solo nodo en todos los nodos. Además, configura el archivo slaves en el nodo maestro con las direcciones IP o nombres de host de todos los nodos esclavos.

slave1
slave2

Paso 3: Configurar Archivos Principales de Hadoop

Asegúrate de que los archivos core-site.xml, hdfs-site.xml, mapred-site.xml y yarn-site.xml estén correctamente configurados en todos los nodos, similar a la configuración de un solo nodo.

Paso 4: Formatear el Namenode

Formatea el Namenode solo en el nodo maestro.

hdfs namenode -format

Paso 5: Iniciar los Daemons de Hadoop

Inicia los daemons de Hadoop en el nodo maestro.

start-dfs.sh
start-yarn.sh

Paso 6: Verificar el Clúster

Revisa las interfaces web de Hadoop en el nodo maestro para asegurarte de que todos los nodos estén conectados y funcionando correctamente.

Conclusión

Configurar un clúster de Hadoop implica varios pasos, desde la instalación de los requisitos previos hasta la configuración de varios componentes de Hadoop. Una configuración de un solo nodo es adecuada para el aprendizaje y el desarrollo, mientras que una configuración de múltiples nodos es esencial para entornos de producción. Siguiendo los pasos descritos en esta sección, puedes configurar y verificar con éxito un clúster de Hadoop, permitiéndote aprovechar el poder de la computación distribuida para el procesamiento de grandes volúmenes de datos.
