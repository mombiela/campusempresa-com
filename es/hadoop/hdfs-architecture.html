Introducción a HDFS

El Sistema de Archivos Distribuido de Hadoop (HDFS) es el sistema de almacenamiento primario utilizado por las aplicaciones de Hadoop. Está diseñado para almacenar grandes conjuntos de datos de manera confiable y para transmitir esos conjuntos de datos a alta velocidad a las aplicaciones de los usuarios.

Conceptos Clave de HDFS

  • Bloques: La unidad más pequeña de almacenamiento de datos en HDFS. El tamaño de bloque predeterminado es de 128 MB.
  • Namenode: El servidor maestro que gestiona el espacio de nombres del sistema de archivos y regula el acceso a los archivos por parte de los clientes.
  • Datanode: Los nodos de trabajo que almacenan y recuperan bloques cuando se les indica (por el Namenode).
  • Replicación: HDFS almacena múltiples copias de bloques de datos para garantizar la confiabilidad y la tolerancia a fallos. El factor de replicación predeterminado es 3.
  • Conciencia de Rack: HDFS es consciente de la topología del rack del clúster e intenta colocar réplicas de bloques de datos en diferentes racks para mejorar la tolerancia a fallos y la disponibilidad de datos.

Componentes de la Arquitectura de HDFS

Namenode

  • Responsabilidades:
    • Gestiona el espacio de nombres del sistema de archivos.
    • Mantiene el árbol del sistema de archivos y los metadatos de todos los archivos y directorios.
    • Regula el acceso a los archivos por parte de los clientes.
  • Almacenamiento de Metadatos:
    • Almacena metadatos en memoria para un acceso rápido.
    • Los metadatos incluyen información como permisos de archivos, tiempos de modificación y acceso, y ubicaciones de bloques.

Datanode

  • Responsabilidades:
    • Almacena bloques de datos reales.
    • Atiende solicitudes de lectura y escritura de los clientes.
    • Informa periódicamente la lista de bloques que almacena al Namenode.
  • Almacenamiento de Bloques:
    • Cada bloque se almacena como un archivo separado en el sistema de archivos local del Datanode.
    • Los Datanodes realizan la creación, eliminación y replicación de bloques según las instrucciones del Namenode.

Namenode Secundario

  • Responsabilidades:
    • Fusiona periódicamente la imagen del espacio de nombres con el registro de edición para evitar que el registro de edición se vuelva demasiado grande.
    • Actúa como un mecanismo de punto de control, pero no es un Namenode de respaldo.

Flujo de Datos en HDFS

Operación de Escritura

  1. Solicitud del Cliente: El cliente solicita al Namenode crear un nuevo archivo.
  2. Asignación de Bloques: El Namenode asigna un bloque y devuelve la lista de Datanodes donde el cliente debe escribir los datos.
  3. Escritura de Datos: El cliente escribe datos en el primer Datanode, que luego replica los datos al siguiente Datanode en la cadena, y así sucesivamente.
  4. Confirmación: Una vez que todas las réplicas están escritas, se envía una confirmación de vuelta al cliente.

Operación de Lectura

  1. Solicitud del Cliente: El cliente solicita al Namenode leer un archivo.
  2. Ubicación de Bloques: El Namenode devuelve la lista de Datanodes que tienen los bloques solicitados.
  3. Lectura de Datos: El cliente lee los datos directamente de los Datanodes.

Tolerancia a Fallos en HDFS

  • Replicación: Garantiza la disponibilidad de datos incluso si algunos Datanodes fallan.
  • Informes de Latidos y Bloques: Los Datanodes envían latidos e informes de bloques al Namenode para confirmar su estado y los bloques que almacenan.
  • Re-replicación Automática: Si se pierde una réplica de un bloque, el Namenode inicia automáticamente la creación de una nueva réplica.

Ejemplo: Escribir y Leer Archivos en HDFS

Escribir un Archivo en HDFS

# Comando para copiar un archivo local a HDFS
hadoop fs -put /local/path/to/file /hdfs/path/to/destination

Explicación:

  • hadoop fs -put: Comando para copiar un archivo del sistema de archivos local a HDFS.
  • /local/path/to/file: Ruta al archivo local.
  • /hdfs/path/to/destination: Ruta de destino en HDFS.

Leer un Archivo de HDFS

# Comando para leer un archivo de HDFS
hadoop fs -cat /hdfs/path/to/file

Explicación:

  • hadoop fs -cat: Comando para mostrar el contenido de un archivo en HDFS.
  • /hdfs/path/to/file: Ruta al archivo en HDFS.

Resumen

HDFS es un sistema de archivos distribuido diseñado para ejecutarse en hardware de bajo costo. Es altamente tolerante a fallos y está diseñado para ser desplegado en hardware de bajo costo. HDFS proporciona acceso de alto rendimiento a los datos de las aplicaciones y es adecuado para aplicaciones que tienen grandes conjuntos de datos. Comprender la arquitectura de HDFS, incluidos sus componentes como Namenode, Datanode y Namenode Secundario, así como su flujo de datos y mecanismos de tolerancia a fallos, es crucial para trabajar eficazmente con Hadoop.
