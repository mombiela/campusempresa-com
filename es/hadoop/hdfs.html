Introducción a HDFS

HDFS, o Hadoop Distributed File System, es el sistema de almacenamiento principal utilizado por las aplicaciones de Hadoop. Está diseñado para almacenar grandes conjuntos de datos de manera confiable y para transmitir esos conjuntos de datos a alta velocidad a las aplicaciones de los usuarios.

Conceptos Clave

  • Escalabilidad: HDFS puede escalar hasta miles de nodos y petabytes de datos.
  • Tolerancia a Fallos: Los datos se replican en múltiples nodos para garantizar la fiabilidad.
  • Alto Rendimiento: Diseñado para un alto rendimiento de datos, no para baja latencia.
  • Acceso a Datos en Streaming: Optimizado para grandes conjuntos de datos y procesamiento por lotes.
  • Hardware de Commodities: Funciona en hardware económico y de uso común.

Arquitectura de HDFS

HDFS sigue una arquitectura maestro-esclavo donde el maestro se llama NameNode y los esclavos se llaman DataNodes.

NameNode

  • Rol: Gestiona los metadatos y el espacio de nombres del sistema de archivos.
  • Responsabilidades:
    • Lleva un registro del árbol del sistema de archivos y los metadatos de todos los archivos y directorios.
    • Gestiona los DataNodes y los bloques que almacenan.
    • Maneja las solicitudes de los clientes para operaciones de archivos.

DataNode

  • Rol: Almacena los bloques de datos reales.
  • Responsabilidades:
    • Atiende las solicitudes de lectura y escritura de los clientes del sistema de archivos.
    • Realiza la creación, eliminación y replicación de bloques según las instrucciones del NameNode.

Almacenamiento de Bloques

  • Tamaño de Bloque: El tamaño de bloque predeterminado es de 128 MB.
  • Replicación: El factor de replicación predeterminado es 3.
  • Ventajas:
    • El gran tamaño de bloque minimiza el costo de las búsquedas.
    • La replicación asegura la fiabilidad y disponibilidad de los datos.

Operaciones de HDFS

Lectura y Escritura de Archivos

  • Operación de Escritura:

    1. El cliente solicita al NameNode crear un nuevo archivo.
    2. El NameNode verifica los permisos y crea los metadatos.
    3. El cliente escribe datos en los DataNodes en bloques.
    4. Los DataNodes replican los bloques a otros DataNodes.

  • Operación de Lectura:

    1. El cliente solicita al NameNode los metadatos del archivo.
    2. El NameNode proporciona la lista de DataNodes que contienen los bloques.
    3. El cliente lee los bloques directamente de los DataNodes.

Código de Ejemplo: Escritura y Lectura de Archivos

// Escribiendo un archivo en HDFS
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import java.io.OutputStream;

public class HDFSWriteExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path path = new Path("/user/hadoop/example.txt");
        OutputStream os = fs.create(path);
        os.write("Hello HDFS!".getBytes());
        os.close();
    }
}

// Leyendo un archivo de HDFS
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import java.io.InputStream;
import java.io.BufferedReader;
import java.io.InputStreamReader;

public class HDFSReadExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path path = new Path("/user/hadoop/example.txt");
        InputStream is = fs.open(path);
        BufferedReader br = new BufferedReader(new InputStreamReader(is));
        String line;
        while ((line = br.readLine()) != null) {
            System.out.println(line);
        }
        br.close();
    }
}

Características Avanzadas de HDFS

Federación

  • Propósito: Escalar el NameNode horizontalmente.
  • Cómo funciona: Múltiples NameNodes gestionan diferentes espacios de nombres, y los DataNodes almacenan bloques para múltiples NameNodes.

Alta Disponibilidad (HA)

  • Propósito: Asegurar que el NameNode no sea un único punto de fallo.
  • Cómo funciona: Utiliza un par de NameNodes en una configuración activa-en espera.

Instantáneas

  • Propósito: Capturar el estado del sistema de archivos en un punto específico en el tiempo.
  • Cómo funciona: Permite instantáneas de solo lectura o de escritura del sistema de archivos.

Conclusión

HDFS es un sistema de archivos robusto y escalable diseñado para manejar grandes conjuntos de datos con alto rendimiento. Su arquitectura asegura la fiabilidad y disponibilidad de los datos a través de la replicación y la tolerancia a fallos. Comprender las operaciones y características de HDFS es crucial para aprovechar todo el potencial de Hadoop en el procesamiento de grandes datos.
