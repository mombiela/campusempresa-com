Introducción a la Seguridad de Hadoop

La seguridad de Hadoop es crucial para proteger los datos y garantizar que solo los usuarios autorizados puedan acceder y manipular los datos almacenados en un clúster de Hadoop. Esta sección cubrirá los conceptos básicos de la seguridad de Hadoop, incluida su importancia y los componentes principales involucrados.

  • Importancia de la Seguridad de Hadoop:

    • Protege los datos sensibles del acceso no autorizado.
    • Garantiza el cumplimiento de los requisitos regulatorios.
    • Mantiene la integridad de los datos y previene las violaciones de datos.

  • Componentes Principales de la Seguridad de Hadoop:

    • Autenticación
    • Autorización
    • Cifrado
    • Auditoría

Autenticación

La autenticación es el proceso de verificar la identidad de un usuario o sistema. En Hadoop, la autenticación asegura que solo los usuarios legítimos puedan acceder al clúster.

  • Autenticación Kerberos:
    • Kerberos es un protocolo de autenticación de red diseñado para proporcionar una autenticación fuerte para aplicaciones cliente/servidor.
    • Hadoop utiliza Kerberos para autenticar usuarios y servicios.

Configuración de la Autenticación Kerberos

  1. Instalar y Configurar Kerberos:

    • Instalar paquetes de Kerberos en los nodos de Hadoop.
    • Configurar el servidor Kerberos (KDC) y crear los principales necesarios.

  2. Configurar Hadoop para Kerberos:

    • Editar el archivo core-site.xml para incluir configuraciones de Kerberos: 
      <property>
  <name>hadoop.security.authentication</name>
  <value>kerberos</value>
</property>
<property>
  <name>hadoop.security.authorization</name>
  <value>true</value>
</property>

  3. Generar Keytabs para los Servicios de Hadoop:

    • Crear archivos keytab para los servicios de Hadoop (por ejemplo, NameNode, DataNode) y distribuirlos a los nodos respectivos.

Autorización

La autorización determina lo que un usuario autenticado tiene permitido hacer. En Hadoop, la autorización controla el acceso a archivos, directorios y recursos.

  • Permisos de HDFS:
    • El Sistema de Archivos Distribuido de Hadoop (HDFS) utiliza un modelo de permisos similar a POSIX.
    • Los permisos se definen para el propietario, el grupo y otros.

Configuración de Permisos de HDFS

  1. Establecer Permisos Usando el Comando hdfs dfs:

    • Ejemplo para establecer permisos: 
      hdfs dfs -chmod 755 /user/hadoop

  2. Establecer Propiedad Usando el Comando hdfs dfs:

    • Ejemplo para establecer propiedad: 
      hdfs dfs -chown hadoop:hadoop /user/hadoop
  • Listas de Control de Acceso (ACLs):
    • Las ACLs proporcionan un control de acceso más detallado que los permisos tradicionales de HDFS.

Configuración de ACLs

  1. Habilitar ACLs en hdfs-site.xml:

    <property>
  <name>dfs.namenode.acls.enabled</name>
  <value>true</value>
</property>

  2. Establecer ACLs Usando el Comando hdfs dfs:

    • Ejemplo para establecer una ACL: 
      hdfs dfs -setfacl -m user:username:rwx /user/hadoop

Cifrado

El cifrado protege los datos convirtiéndolos en un formato que solo puede ser leído por alguien con la clave de descifrado. Hadoop admite tanto el cifrado de datos en reposo como el cifrado de datos en tránsito.

  • Cifrado de Datos en Reposo:
    • Cifra los datos almacenados en disco.
    • Utiliza el Cifrado Transparente de Datos (TDE) de Hadoop.

Configuración del Cifrado de Datos en Reposo

  1. Habilitar TDE en hdfs-site.xml:

    <property>
  <name>dfs.encryption.key.provider.uri</name>
  <value>kms://http@<KMS_HOST>:<KMS_PORT>/kms</value>
</property>

  2. Crear Zonas de Cifrado:

    • Ejemplo para crear una zona de cifrado: 
      hdfs crypto -createZone -keyName myKey -path /user/hadoop/encrypted
  • Cifrado de Datos en Tránsito:
    • Cifra los datos mientras viajan por la red.
    • Utiliza SSL/TLS para el cifrado.

Configuración del Cifrado de Datos en Tránsito

  1. Habilitar SSL/TLS en core-site.xml:

    <property>
  <name>hadoop.ssl.enabled</name>
  <value>true</value>
</property>

  2. Configurar Certificados SSL/TLS:

    • Generar y configurar certificados SSL para los servicios de Hadoop.

Auditoría

La auditoría implica rastrear y registrar el acceso a datos y recursos. La auditoría de Hadoop ayuda a monitorear las actividades de los usuarios y detectar posibles violaciones de seguridad.

  • Registros de Auditoría de HDFS:
    • HDFS genera registros de auditoría que registran eventos de acceso a archivos.

Configuración de Registros de Auditoría de HDFS

  1. Habilitar el Registro de Auditoría en hdfs-site.xml:

    <property>
  <name>dfs.namenode.audit.loggers</name>
  <value>default</value>
</property>

  2. Revisar los Registros de Auditoría:

    • Los registros de auditoría generalmente se almacenan en el directorio de registros de NameNode.
    • Ejemplo de entrada de registro: 
      2023-10-01 12:00:00,000 INFO FSNamesystem.audit: allowed=true ugi=hadoop ip=/192.168.1.1 cmd=open src=/user/hadoop/file dst=null perm=null

Resumen

La seguridad de Hadoop es un enfoque multifacético para proteger los datos y garantizar que solo los usuarios autorizados puedan acceder y manipular los datos almacenados en un clúster de Hadoop. Los componentes clave incluyen la autenticación (principalmente usando Kerberos), la autorización (usando permisos de HDFS y ACLs), el cifrado (tanto de datos en reposo como de datos en tránsito) y la auditoría (rastrear y registrar el acceso a datos y recursos). Al implementar estas medidas de seguridad, las organizaciones pueden proteger sus entornos de Hadoop contra el acceso no autorizado y las violaciones de datos.
