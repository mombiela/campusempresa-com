Introducción a Apache Oozie

Apache Oozie es un sistema de programación de flujos de trabajo para gestionar trabajos de Hadoop. Permite a los usuarios definir una secuencia de acciones para ser ejecutadas en un orden específico, facilitando la gestión de complejas canalizaciones de procesamiento de datos.

Conceptos Clave

  • Workflow: Una colección de acciones organizadas en un Grafo Acíclico Dirigido (DAG).
  • Coordinator: Gestiona flujos de trabajo basados en el tiempo y la disponibilidad de datos.
  • Bundle: Una abstracción de nivel superior que agrupa múltiples coordinadores.
  • Action: Una unidad de trabajo única en un flujo de trabajo (por ejemplo, trabajo MapReduce, script Pig, consulta Hive).

Configuración de Apache Oozie

Prerrequisitos

  • Configuración del clúster de Hadoop
  • Kit de Desarrollo de Java (JDK) instalado
  • Binarios de Apache Oozie

Pasos de Instalación

  1. Descargar Apache Oozie:

    wget http://apache.mirrors.tds.net/oozie/5.2.0/oozie-5.2.0.tar.gz
tar -xzf oozie-5.2.0.tar.gz
cd oozie-5.2.0

  2. Configurar Oozie:

    • Editar oozie-site.xml para configurar la base de datos y los ajustes de Hadoop.
    • Ejemplo de configuración: 
      <property>
  <name>oozie.service.JPAService.jdbc.url</name>
  <value>jdbc:mysql://localhost:3306/oozie</value>
</property>
<property>
  <name>oozie.service.JPAService.jdbc.username</name>
  <value>oozie</value>
</property>
<property>
  <name>oozie.service.JPAService.jdbc.password</name>
  <value>password</value>
</property>

  3. Desplegar Oozie:

    bin/oozie-setup.sh prepare-war
bin/oozie-setup.sh db-schema

  4. Iniciar Oozie:

    bin/oozied.sh start

Creación y Ejecución de Workflows

Definición de Workflow

Los workflows se definen usando XML. A continuación, se muestra un ejemplo de un workflow simple que ejecuta un script de shell.

<workflow-app name="example-wf" xmlns="uri:oozie:workflow:0.5">
    <start to="shell-node"/>
    
    <action name="shell-node">
        <shell xmlns="uri:oozie:shell-action:0.2">
            <job-tracker>${jobTracker}</job-tracker>
            <name-node>${nameNode}</name-node>
            <exec>example.sh</exec>
        </shell>
        <ok to="end"/>
        <error to="fail"/>
    </action>
    
    <kill name="fail">
        <message>Workflow failed, error message[${wf:errorMessage(wf:lastErrorNode())}]</message>
    </kill>
    
    <end name="end"/>
</workflow-app>

Ejecución del Workflow

  1. Subir Workflow a HDFS:

    hdfs dfs -put workflow.xml /user/oozie/workflows/example-wf/

  2. Enviar Workflow:

    oozie job -oozie http://localhost:11000/oozie -config job.properties -run

Monitoreo y Gestión de Workflows

  • Utiliza la consola web de Oozie para monitorear el estado de los workflows.
  • Comandos para gestionar workflows: 
    oozie job -info <job-id>
oozie job -kill <job-id>

Funciones Avanzadas

Coordinadores

Los coordinadores activan workflows basados en el tiempo o la disponibilidad de datos.

Ejemplo de definición de coordinador:

<coordinator-app name="example-coord" frequency="5" start="2023-01-01T00:00Z" end="2023-12-31T00:00Z" timezone="UTC" xmlns="uri:oozie:coordinator:0.4">
    <controls>
        <timeout>10</timeout>
    </controls>
    <datasets>
        <dataset name="example-dataset" frequency="5" initial-instance="2023-01-01T00:00Z" timezone="UTC">
            <uri-template>hdfs://localhost:9000/user/oozie/data/${YEAR}/${MONTH}/${DAY}/${HOUR}</uri-template>
        </dataset>
    </datasets>
    <input-events>
        <data-in name="input" dataset="example-dataset"/>
    </input-events>
    <action>
        <workflow>
            <app-path>hdfs://localhost:9000/user/oozie/workflows/example-wf</app-path>
        </workflow>
    </action>
</coordinator-app>

Bundles

Los bundles agrupan múltiples coordinadores para una gestión más sencilla.

Ejemplo de definición de bundle:

<bundle-app name="example-bundle" xmlns="uri:oozie:bundle:0.2">
    <coordinator name="coord1">
        <app-path>hdfs://localhost:9000/user/oozie/coordinators/example-coord</app-path>
    </coordinator>
    <coordinator name="coord2">
        <app-path>hdfs://localhost:9000/user/oozie/coordinators/another-coord</app-path>
    </coordinator>
</bundle-app>

Resumen

Apache Oozie es una herramienta poderosa para gestionar workflows, coordinadores y bundles de Hadoop. Simplifica la programación y ejecución de complejas canalizaciones de procesamiento de datos. Al comprender los conceptos básicos de la definición de workflows, configuración y funciones avanzadas como coordinadores y bundles, puedes gestionar y automatizar eficazmente tus trabajos de Hadoop.
