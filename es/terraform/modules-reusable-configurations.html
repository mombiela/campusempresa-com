Introducción a los Módulos

Los módulos en Terraform son una forma poderosa de organizar y reutilizar configuraciones. Permiten encapsular un conjunto de recursos y reutilizar esta configuración en diferentes partes de tu infraestructura.

Conceptos Clave

  • Módulo: Un contenedor para múltiples recursos que se usan juntos.
  • Módulo Raíz: El directorio principal de trabajo donde se ejecuta Terraform.
  • Módulo Hijo: Cualquier módulo que es llamado por otro módulo.
  • Registro de Módulos: Un repositorio donde los módulos pueden ser compartidos y reutilizados.

Creando un Módulo Básico

Para crear un módulo, necesitas organizar tus archivos de configuración en una estructura de directorios. Cada módulo debe tener su propio directorio.

Ejemplo de Estructura de Directorios

mi-proyecto-terraform/
├── main.tf
├── variables.tf
├── outputs.tf
└── módulos/
    └── s3_bucket/
        ├── main.tf
        ├── variables.tf
        └── outputs.tf

Ejemplo de Configuración de Módulo

Vamos a crear un módulo simple para gestionar un bucket S3.

módulos/s3_bucket/main.tf

resource "aws_s3_bucket" "example" {
  bucket = var.bucket_name
  acl    = var.acl
}

módulos/s3_bucket/variables.tf

variable "bucket_name" {
  description = "El nombre del bucket S3"
  type        = string
}

variable "acl" {
  description = "El ACL para el bucket S3"
  type        = string
  default     = "private"
}

módulos/s3_bucket/outputs.tf

output "bucket_arn" {
  description = "El ARN del bucket S3"
  value       = aws_s3_bucket.example.arn
}

Usando un Módulo

Para usar un módulo, necesitas llamarlo desde tu módulo raíz.

Ejemplo de Configuración del Módulo Raíz

main.tf

module "s3_bucket" {
  source      = "./módulos/s3_bucket"
  bucket_name = "mi-nombre-unico-de-bucket"
  acl         = "public-read"
}

output "bucket_arn" {
  value = module.s3_bucket.bucket_arn
}

Explicación

  • source: Especifica la ruta al módulo.
  • bucket_name y acl: Estas son variables definidas en el módulo a las que necesitas proporcionar valores.
  • output "bucket_arn": Esto muestra el ARN del bucket S3 creado por el módulo.

Mejores Prácticas para Módulos

  • Encapsulación: Mantén los módulos enfocados en una sola responsabilidad.
  • Versionado: Usa control de versiones para gestionar cambios en tus módulos.
  • Documentación: Documenta el propósito y uso de tus módulos.
  • Pruebas: Prueba tus módulos de manera independiente antes de integrarlos en configuraciones más grandes.

Ejemplo de Documentación del Módulo

módulos/s3_bucket/README.md

# Módulo de Bucket S3

Este módulo crea un bucket S3 con el nombre y ACL especificados.

## Variables

- `bucket_name`: El nombre del bucket S3 (requerido).
- `acl`: El ACL para el bucket S3 (opcional, el valor por defecto es "private").

## Salidas

- `bucket_arn`: El ARN del bucket S3.

Conclusión

Los módulos son una característica fundamental en Terraform que te permiten escribir código de infraestructura limpio, reutilizable y mantenible. Al encapsular recursos en módulos, puedes gestionar y escalar tu infraestructura fácilmente. Recuerda seguir las mejores prácticas como encapsulación, versionado, documentación y pruebas para asegurar que tus módulos sean robustos y confiables.
