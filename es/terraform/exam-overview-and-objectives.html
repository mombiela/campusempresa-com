Introducción

En esta sección, proporcionaremos una descripción general de la estructura y los objetivos del examen para el curso de Terraform. Esto te ayudará a entender qué esperar y cómo prepararte de manera efectiva.

Estructura del examen

El examen está diseñado para evaluar tus conocimientos y habilidades en varios niveles de Terraform, desde principiante hasta avanzado. El examen se divide en tres secciones principales:

  1. Nivel Principiante
  2. Nivel Intermedio
  3. Nivel Avanzado

Cada sección incluirá preguntas de opción múltiple, ejercicios prácticos y tareas de análisis de código.

Objetivos por nivel

Nivel Principiante

En el nivel principiante, el examen se centrará en conceptos fundamentales y el uso básico de Terraform. Los objetivos incluyen:

  • Comprender los principios de Infraestructura como Código (IaC).
  • Instalar y configurar Terraform.
  • Escribir configuraciones básicas de Terraform.
  • Gestionar archivos de estado.
  • Aplicar y destruir infraestructura.

Conceptos clave:

  • Infraestructura como Código (IaC): La práctica de gestionar y aprovisionar infraestructura informática a través de archivos de configuración legibles por máquina.
  • Configuración de Terraform: El conjunto de archivos utilizados para describir la infraestructura.
  • Archivo de Estado: Un archivo que realiza un seguimiento de los recursos gestionados por Terraform.

Ejemplo:

provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}

Este ejemplo muestra una configuración básica para crear una instancia EC2 de AWS.

Nivel Intermedio

En el nivel intermedio, el examen evaluará tu capacidad para gestionar infraestructura más compleja y utilizar características avanzadas de Terraform. Los objetivos incluyen:

  • Usar módulos para organizar configuraciones.
  • Gestionar múltiples entornos.
  • Implementar almacenamiento de estado remoto.
  • Usar fuentes de datos.
  • Escribir y usar provisionadores.

Conceptos clave:

  • Módulos: Paquetes reutilizables y autónomos de configuraciones de Terraform.
  • Estado Remoto: Almacenar el archivo de estado en una ubicación remota para la colaboración.
  • Fuentes de Datos: Permitir que Terraform obtenga datos de fuentes externas.

Ejemplo:

module "vpc" {
  source = "terraform-aws-modules/vpc/aws"
  version = "2.21.0"

  name = "my-vpc"
  cidr = "10.0.0.0/16"
  azs  = ["us-west-2a", "us-west-2b", "us-west-2c"]
}

Este ejemplo demuestra el uso de un módulo para crear una VPC.

Nivel Avanzado

En el nivel avanzado, el examen evaluará tu capacidad para manejar escenarios complejos y optimizar el uso de Terraform. Los objetivos incluyen:

  • Escribir módulos personalizados.
  • Implementar dependencias complejas.
  • Usar espacios de trabajo para la gestión de entornos.
  • Optimizar y refactorizar el código de Terraform.
  • Integrar Terraform con pipelines de CI/CD.

Conceptos clave:

  • Módulos Personalizados: Crear tus propios módulos reutilizables.
  • Espacios de Trabajo: Aislar archivos de estado para diferentes entornos.
  • Integración CI/CD: Automatizar flujos de trabajo de Terraform dentro de pipelines de CI/CD.

Ejemplo:

resource "aws_s3_bucket" "example" {
  bucket = "my-unique-bucket-name"
  acl    = "private"

  versioning {
    enabled = true
  }

  lifecycle_rule {
    id      = "log"
    enabled = true

    transition {
      days          = 30
      storage_class = "GLACIER"
    }

    expiration {
      days = 365
    }
  }
}

Este ejemplo muestra una configuración avanzada para un bucket S3 con versionado y reglas de ciclo de vida.

Conclusión

Comprender la estructura y los objetivos del examen es crucial para una preparación efectiva. Cada nivel se basa en el anterior, asegurando una comprensión integral de Terraform. Al dominar los conceptos clave y practicar con ejemplos del mundo real, estarás bien preparado para tener éxito en el examen y aplicar Terraform en tus proyectos.
