Introducción

En esta sección, te guiaremos a través de la creación de tu primera configuración de Terraform. Terraform es una herramienta de software de infraestructura como código de código abierto que proporciona un flujo de trabajo CLI consistente para gestionar cientos de servicios en la nube. Al final de esta sección, tendrás una comprensión básica de cómo escribir y aplicar una configuración de Terraform para aprovisionar infraestructura.

Prerrequisitos

Antes de comenzar, asegúrate de tener lo siguiente:

  • Terraform instalado en tu máquina. Puedes descargarlo desde el sitio web de Terraform.
  • Una comprensión básica de los servicios en la nube (por ejemplo, AWS, Azure, GCP).
  • Una cuenta con un proveedor de nube (por ejemplo, AWS).

Conceptos Clave

  • Proveedores: Plugins que permiten a Terraform interactuar con proveedores de nube, proveedores de SaaS, etc.
  • Recursos: Los componentes de tu infraestructura (por ejemplo, máquinas virtuales, cuentas de almacenamiento).
  • Variables: Parámetros que te permiten personalizar tus configuraciones.
  • Estado: Una forma de realizar un seguimiento de los recursos gestionados por Terraform.

Guía Paso a Paso

Paso 1: Configurando Tu Directorio

Crea un nuevo directorio para tus archivos de configuración de Terraform. Esto ayuda a mantener tu proyecto organizado.

mkdir my-first-terraform
cd my-first-terraform

Paso 2: Escribiendo Tu Primer Archivo de Configuración

Crea un nuevo archivo llamado main.tf. Este archivo contendrá tu configuración de Terraform.

# main.tf

# Especificar el proveedor
provider "aws" {
  region = "us-west-2"
}

# Crear una instancia EC2
resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0" # ID de Amazon Linux 2 AMI
  instance_type = "t2.micro"
}

Paso 3: Inicializando Terraform

Inicializa tu configuración de Terraform. Este comando descarga los plugins del proveedor necesarios.

terraform init

Paso 4: Planificando Tu Configuración

Genera y muestra un plan de ejecución. Este comando muestra qué acciones tomará Terraform para lograr el estado deseado.

terraform plan

Paso 5: Aplicando Tu Configuración

Aplica los cambios necesarios para alcanzar el estado deseado de la configuración.

terraform apply

Paso 6: Verificando la Infraestructura

Una vez que el comando apply se complete, puedes verificar que los recursos se hayan creado en la consola de tu proveedor de nube.

Paso 7: Limpiando

Para evitar incurrir en cargos, destruye los recursos que creaste.

terraform destroy

Ejemplo Explicado

Desglosamos la configuración del ejemplo:

  • Bloque del Proveedor: Especifica que estamos usando AWS como nuestro proveedor de nube y establece la región en us-west-2.
  • Bloque de Recursos: Define un recurso de instancia EC2 de AWS. El atributo ami especifica el ID de la Imagen de Máquina de Amazon, y el atributo instance_type especifica el tipo de instancia.

Conclusión

¡Felicidades! Has creado y gestionado con éxito tu primera configuración de Terraform. Aprendiste cómo:

  • Configurar un proyecto de Terraform.
  • Escribir un archivo de configuración básico.
  • Inicializar, planificar, aplicar y destruir infraestructura gestionada por Terraform.
