En esta sección, exploraremos cómo crear y usar proveedores personalizados en Terraform. Este tema es esencial para los usuarios avanzados de Terraform que necesitan extender las capacidades de Terraform más allá de los proveedores incorporados.

Introducción a los Proveedores Personalizados

Los proveedores personalizados en Terraform te permiten interactuar con APIs y servicios que no son compatibles con los proveedores oficiales de Terraform. Esto puede ser particularmente útil para sistemas propietarios o servicios de nicho.

Conceptos Clave

  • Proveedor: Un proveedor es un plugin que Terraform utiliza para interactuar con las APIs de proveedores de nube, proveedores SaaS y otros servicios.
  • Proveedor Personalizado: Un proveedor personalizado es un proveedor definido por el usuario que permite a Terraform gestionar recursos para servicios no compatibles con los proveedores oficiales.
  • Terraform Plugin SDK: Un conjunto de herramientas y bibliotecas proporcionadas por HashiCorp para ayudarte a construir proveedores personalizados.

Configuración de Tu Entorno de Desarrollo

Antes de comenzar a crear un proveedor personalizado, necesitas configurar tu entorno de desarrollo.

Prerrequisitos

  • Lenguaje de Programación Go: Los proveedores de Terraform están escritos en Go. Asegúrate de tener Go instalado en tu máquina.
  • Terraform Plugin SDK: Necesitarás el Terraform Plugin SDK para desarrollar tu proveedor personalizado.

Pasos de Instalación

  1. Instalar Go: Sigue las instrucciones en el sitio web oficial de Go para instalar Go.
  2. Configurar el Espacio de Trabajo de Go: Configura tu espacio de trabajo de Go estableciendo la variable de entorno GOPATH.
  3. Instalar Terraform Plugin SDK: Usa el siguiente comando para instalar el SDK: 
    go get github.com/hashicorp/terraform-plugin-sdk/v2

Creación de un Proveedor Personalizado

En esta sección, crearemos un proveedor personalizado simple para gestionar un servicio ficticio llamado "ExampleService".

Guía Paso a Paso

  1. Inicializar Tu Proyecto: Crea un nuevo directorio para tu proveedor e inicializa un módulo de Go.

    mkdir terraform-provider-exampleservice
cd terraform-provider-exampleservice
go mod init github.com/tuusuario/terraform-provider-exampleservice

  2. Crear el Archivo del Proveedor: Crea un archivo llamado main.go y define el proveedor.

    package main

import (
    "github.com/hashicorp/terraform-plugin-sdk/v2/helper/schema"
    "github.com/hashicorp/terraform-plugin-sdk/v2/plugin"
)

func main() {
    plugin.Serve(&plugin.ServeOpts{
        ProviderFunc: func() *schema.Provider {
            return &schema.Provider{
                ResourcesMap: map[string]*schema.Resource{
                    "exampleservice_item": resourceItem(),
                },
            }
        },
    })
}

  3. Definir un Recurso: Crea un archivo llamado resource_item.go y define un recurso para ExampleService.

    package main

import (
    "github.com/hashicorp/terraform-plugin-sdk/v2/helper/schema"
)

func resourceItem() *schema.Resource {
    return &schema.Resource{
        Create: resourceItemCreate,
        Read:   resourceItemRead,
        Update: resourceItemUpdate,
        Delete: resourceItemDelete,

        Schema: map[string]*schema.Schema{
            "name": {
                Type:     schema.TypeString,
                Required: true,
            },
            "value": {
                Type:     schema.TypeString,
                Required: true,
            },
        },
    }
}

func resourceItemCreate(d *schema.ResourceData, m interface{}) error {
    // Implementar la lógica para crear un ítem en ExampleService
    d.SetId("example_id")
    return resourceItemRead(d, m)
}

func resourceItemRead(d *schema.ResourceData, m interface{}) error {
    // Implementar la lógica para leer un ítem de ExampleService
    return nil
}

func resourceItemUpdate(d *schema.ResourceData, m interface{}) error {
    // Implementar la lógica para actualizar un ítem en ExampleService
    return resourceItemRead(d, m)
}

func resourceItemDelete(d *schema.ResourceData, m interface{}) error {
    // Implementar la lógica para eliminar un ítem de ExampleService
    d.SetId("")
    return nil
}

  4. Construir el Proveedor: Compila tu proveedor usando la herramienta de construcción de Go.

    go build -o terraform-provider-exampleservice

Usando Tu Proveedor Personalizado

Una vez que hayas construido tu proveedor personalizado, puedes usarlo en tus configuraciones de Terraform.

Configuración de Ejemplo

Crea un archivo de configuración de Terraform llamado main.tf para usar tu proveedor personalizado.

provider "exampleservice" {}

resource "exampleservice_item" "example" {
  name  = "example_name"
  value = "example_value"
}

Inicializar y Aplicar

  1. Inicializar Terraform: Ejecuta el siguiente comando para inicializar Terraform y descargar el proveedor.

    terraform init

  2. Aplicar la Configuración: Aplica la configuración para crear el recurso.

    terraform apply

Conclusión

Crear proveedores personalizados en Terraform te permite extender la funcionalidad de Terraform para gestionar recursos de servicios que no son compatibles de manera nativa. Siguiendo los pasos descritos en esta sección, puedes configurar tu entorno de desarrollo, crear un proveedor personalizado y usarlo en tus configuraciones de Terraform. Esta habilidad es particularmente valiosa para usuarios avanzados que necesitan integrar Terraform con servicios propietarios o de nicho.
