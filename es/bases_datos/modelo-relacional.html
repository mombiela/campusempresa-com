Modelo Relacional
Lenguaje SQL ►

Introducción

El modelo relacional es uno de los pilares fundamentales en el diseño y gestión de bases de datos. Propuesto por Edgar F. Codd en 1970, este modelo organiza los datos en tablas (o relaciones) que permiten una manipulación eficiente y una estructura clara.

Conceptos Clave

  • Relación: Una tabla en una base de datos relacional.
  • Tupla: Una fila en una tabla, que representa un registro único.
  • Atributo: Una columna en una tabla, que representa una característica del registro.
  • Clave Primaria: Un atributo o conjunto de atributos que identifican de manera única una tupla en una tabla.
  • Clave Foránea: Un atributo en una tabla que es una clave primaria en otra tabla, utilizado para establecer relaciones entre tablas.
  • Esquema de Relación: La estructura de una tabla, definida por sus atributos y sus tipos de datos.

Estructura de una Tabla

Una tabla en el modelo relacional se puede visualizar como una matriz de filas y columnas. Cada fila representa un registro único y cada columna representa un atributo del registro.

Ejemplo de una Tabla

Supongamos que tenemos una tabla llamada Clientes:

| ClienteID | Nombre | Apellido | Email | |-----------|----------|----------|----------------------| | 1 | Juan | Pérez | juan.perez@mail.com | | 2 | María | García | maria.garcia@mail.com| | 3 | Carlos | López | carlos.lopez@mail.com|

  • ClienteID: Clave primaria.
  • Nombre: Atributo que almacena el nombre del cliente.
  • Apellido: Atributo que almacena el apellido del cliente.
  • Email: Atributo que almacena el correo electrónico del cliente.

Relaciones entre Tablas

Las relaciones entre tablas se establecen mediante claves foráneas. Esto permite la normalización de datos y evita la redundancia.

Ejemplo de Relaciones

Supongamos que tenemos otra tabla llamada Pedidos:

| PedidoID | ClienteID | Fecha | Total | |----------|-----------|------------|--------| | 101 | 1 | 2023-01-15 | 150.00 | | 102 | 2 | 2023-01-16 | 200.00 | | 103 | 1 | 2023-01-17 | 100.00 |

  • PedidoID: Clave primaria.
  • ClienteID: Clave foránea que referencia a ClienteID en la tabla Clientes.
  • Fecha: Atributo que almacena la fecha del pedido.
  • Total: Atributo que almacena el total del pedido.

Ejemplos de Código SQL

Creación de Tablas

CREATE TABLE Clientes (
    ClienteID INT PRIMARY KEY,
    Nombre VARCHAR(50),
    Apellido VARCHAR(50),
    Email VARCHAR(100)
);

CREATE TABLE Pedidos (
    PedidoID INT PRIMARY KEY,
    ClienteID INT,
    Fecha DATE,
    Total DECIMAL(10, 2),
    FOREIGN KEY (ClienteID) REFERENCES Clientes(ClienteID)
);

Inserción de Datos

INSERT INTO Clientes (ClienteID, Nombre, Apellido, Email) VALUES
(1, 'Juan', 'Pérez', 'juan.perez@mail.com'),
(2, 'María', 'García', 'maria.garcia@mail.com'),
(3, 'Carlos', 'López', 'carlos.lopez@mail.com');

INSERT INTO Pedidos (PedidoID, ClienteID, Fecha, Total) VALUES
(101, 1, '2023-01-15', 150.00),
(102, 2, '2023-01-16', 200.00),
(103, 1, '2023-01-17', 100.00);

Consultas Básicas

  • Seleccionar todos los clientes:

    SELECT * FROM Clientes;

  • Seleccionar todos los pedidos de un cliente específico:

    SELECT * FROM Pedidos WHERE ClienteID = 1;

Conclusión

El modelo relacional es una herramienta poderosa para la organización y gestión de datos en bases de datos. Comprender sus conceptos básicos, como relaciones, claves primarias y foráneas, y cómo estructurar y manipular datos mediante SQL, es esencial para cualquier profesional de TI. Este conocimiento permite diseñar sistemas de bases de datos eficientes y escalables, facilitando la integridad y consistencia de los datos.
