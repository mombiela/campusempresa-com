Las operaciones de agregación en MongoDB procesan registros de datos y devuelven resultados calculados. A menudo se utilizan para agrupar valores de múltiples documentos, realizar operaciones en los datos agrupados y devolver un único resultado. Esta sección cubrirá las etapas de agregación más comunes que encontrarás, desde niveles principiantes hasta avanzados.

Introducción al Marco de Agregación

El marco de agregación en MongoDB es una herramienta poderosa para el análisis y la transformación de datos. Consiste en una tubería de etapas, cada una realizando una operación específica en los documentos de entrada.

Etapas Comunes de Agregación

$match

  • Propósito: Filtra los documentos para pasar solo aquellos que coincidan con la(s) condición(es) especificada(s) a la siguiente etapa de la tubería.
  • Uso: Similar a la consulta find, pero se usa dentro de la tubería de agregación.
db.collection.aggregate([
  { $match: { status: "A" } }
])

$group

  • Propósito: Agrupa documentos de entrada por una expresión de identificador especificada y aplica las expresiones de acumulador a cada grupo.
  • Uso: A menudo se usa para calcular agregados como sumas, promedios, conteos, etc.
db.collection.aggregate([
  { $group: { _id: "$status", total: { $sum: "$amount" } } }
])

$project

  • Propósito: Reconfigura cada documento en el flujo, como agregando nuevos campos o eliminando campos existentes.
  • Uso: Útil para incluir, excluir o calcular nuevos campos.
db.collection.aggregate([
  { $project: { item: 1, total: { $multiply: ["$price", "$quantity"] } } }
])

$sort

  • Propósito: Ordena todos los documentos de entrada y los devuelve en orden ordenado.
  • Uso: Similar al método sort pero usado dentro de la tubería de agregación.
db.collection.aggregate([
  { $sort: { total: -1 } }
])

$limit

  • Propósito: Limita el número de documentos pasados a la siguiente etapa en la tubería.
  • Uso: Útil para paginación o restringir el tamaño de la salida.
db.collection.aggregate([
  { $limit: 5 }
])

$skip

  • Propósito: Omite los primeros N documentos y pasa los documentos restantes a la siguiente etapa.
  • Uso: A menudo se usa junto con $limit para paginación.
db.collection.aggregate([
  { $skip: 10 }
])

$unwind

  • Propósito: Descompone un campo de matriz de los documentos de entrada para generar un documento por cada elemento.
  • Uso: Útil para normalizar datos almacenados en matrices.
db.collection.aggregate([
  { $unwind: "$items" }
])

$lookup

  • Propósito: Realiza una unión externa izquierda a otra colección en la misma base de datos para filtrar documentos de la colección "unida" para su procesamiento.
  • Uso: Útil para combinar datos de múltiples colecciones.
db.collection.aggregate([
  {
    $lookup: {
      from: "otherCollection",
      localField: "foreignKey",
      foreignField: "_id",
      as: "joinedData"
    }
  }
])

$addFields

  • Propósito: Agrega nuevos campos a los documentos.
  • Uso: Útil para agregar campos calculados.
db.collection.aggregate([
  { $addFields: { totalPrice: { $multiply: ["$price", "$quantity"] } } }
])

$out

  • Propósito: Escribe los documentos resultantes de la tubería de agregación en una colección especificada.
  • Uso: Útil para almacenar los resultados de una agregación para uso futuro.
db.collection.aggregate([
  { $match: { status: "A" } },
  { $out: "newCollection" }
])

Resumen

Entender y utilizar las etapas comunes de agregación en MongoDB es crucial para un análisis y transformación de datos efectivos. Cada etapa sirve a un propósito específico, desde filtrar y agrupar hasta reconfigurar y unir datos. Al dominar estas etapas, puedes realizar consultas complejas y obtener conocimientos más profundos de tus datos.

  • $match: Filtra documentos.
  • $group: Agrupa documentos y realiza agregaciones.
  • $project: Reconfigura documentos.
  • $sort: Ordena documentos.
  • $limit: Limita el número de documentos.
  • $skip: Omite documentos.
  • $unwind: Descompone matrices.
  • $lookup: Une colecciones.
  • $addFields: Agrega nuevos campos.
  • $out: Salida de resultados a una colección.

Estas etapas forman los bloques de construcción del marco de agregación de MongoDB, permitiéndote crear tuberías de procesamiento de datos poderosas y eficientes.
