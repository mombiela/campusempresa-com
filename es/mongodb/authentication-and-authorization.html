Control de Acceso Basado en Roles (RBAC) ►

La autenticación y la autorización son componentes críticos de la seguridad de la base de datos. Esta sección te guiará a través de los conceptos e implementación de la autenticación y autorización en MongoDB, desde niveles principiantes hasta avanzados.

Introducción a la Autenticación y Autorización

  • Autenticación: El proceso de verificar la identidad de un usuario o proceso.
  • Autorización: El proceso de determinar qué puede hacer un usuario o proceso autenticado.

Autenticación en MongoDB

Conceptos Básicos

  • Usuario: Una entidad que puede ser autenticada.
  • Rol: Un conjunto de privilegios que pueden ser asignados a un usuario.
  • Privilegio: Una acción específica que un usuario puede realizar en un recurso.

Habilitando la Autenticación

Por defecto, MongoDB no requiere autenticación. Para habilitar la autenticación, necesitas iniciar el servidor MongoDB con la opción --auth.

mongod --auth --dbpath /data/db

Creando Usuarios

Para crear un usuario, necesitas conectarte a la instancia de MongoDB sin autenticación y usar el método db.createUser.

use admin
db.createUser({
  user: "admin",
  pwd: "password123",
  roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
})

Autenticando Usuarios

Una vez que los usuarios son creados, pueden autenticarse usando el método db.auth.

use admin
db.auth("admin", "password123")

Autorización en MongoDB

Control de Acceso Basado en Roles (RBAC)

MongoDB usa el Control de Acceso Basado en Roles (RBAC) para gestionar la autorización. Los roles otorgan a los usuarios privilegios específicos en recursos.

Roles Integrados

MongoDB proporciona varios roles integrados, tales como:

  • read: Otorga acceso de solo lectura a una base de datos.
  • readWrite: Otorga acceso de lectura y escritura a una base de datos.
  • dbAdmin: Otorga derechos administrativos a una base de datos.
  • userAdmin: Otorga la capacidad de crear y modificar roles y usuarios en una base de datos.

Roles Personalizados

También puedes crear roles personalizados para satisfacer necesidades específicas.

db.createRole({
  role: "customRole",
  privileges: [
    { resource: { db: "test", collection: "" }, actions: [ "find", "insert" ] }
  ],
  roles: []
})

Asignando Roles a Usuarios

Los roles pueden ser asignados a los usuarios cuando son creados o modificados.

db.createUser({
  user: "appUser",
  pwd: "appPassword",
  roles: [ { role: "readWrite", db: "test" } ]
})

Ejemplos Prácticos

Ejemplo 1: Creando un Usuario Administrador

  1. Inicia MongoDB sin autenticación.
  2. Crea un usuario administrador.
use admin
db.createUser({
  user: "admin",
  pwd: "adminPass",
  roles: [ { role: "root", db: "admin" } ]
})
  1. Reinicia MongoDB con la autenticación habilitada.
mongod --auth --dbpath /data/db
  1. Autentícate como el usuario administrador.
use admin
db.auth("admin", "adminPass")

Ejemplo 2: Creando y Autenticando un Usuario Regular

  1. Autentícate como el usuario administrador.
use admin
db.auth("admin", "adminPass")
  1. Crea un usuario regular con acceso de lectura y escritura a la base de datos test.
use test
db.createUser({
  user: "testUser",
  pwd: "testPass",
  roles: [ { role: "readWrite", db: "test" } ]
})
  1. Autentícate como el usuario regular.
use test
db.auth("testUser", "testPass")

Resumen

En esta sección, cubrimos los conceptos básicos de autenticación y autorización en MongoDB. Aprendimos cómo:

  • Habilitar la autenticación.
  • Crear y autenticar usuarios.
  • Usar roles integrados y crear roles personalizados.
  • Asignar roles a usuarios.

Entender estos conceptos es crucial para asegurar tus instancias de MongoDB y asegurar que solo los usuarios autorizados puedan acceder y manipular tus datos.
