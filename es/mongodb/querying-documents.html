Introducción

Consultar documentos es un aspecto fundamental de trabajar con MongoDB. Esta sección te guiará a través de los conceptos básicos de la consulta de documentos, avanzando hacia consultas más complejas. Al final de esta sección, podrás recuperar y manipular datos almacenados en colecciones de MongoDB de manera eficiente.

Consultas Básicas

Encontrar Todos los Documentos

Para recuperar todos los documentos en una colección, usa el método find sin ningún parámetro.

db.collection.find()

Encontrar Documentos con un Filtro

Para encontrar documentos que coincidan con criterios específicos, pasa un documento de consulta al método find.

db.collection.find({ "field": "value" })

Ejemplo:

db.users.find({ "age": 25 })

Esta consulta recupera todos los documentos de la colección users donde el campo age es 25.

Operadores de Consulta

MongoDB proporciona varios operadores para construir consultas complejas.

Operadores de Comparación

  • $eq: Coincide con valores que son iguales a un valor especificado.
  • $ne: Coincide con todos los valores que no son iguales a un valor especificado.
  • $gt: Coincide con valores que son mayores que un valor especificado.
  • $gte: Coincide con valores que son mayores o iguales a un valor especificado.
  • $lt: Coincide con valores que son menores que un valor especificado.
  • $lte: Coincide con valores que son menores o iguales a un valor especificado.

Ejemplo:

db.users.find({ "age": { "$gt": 20, "$lt": 30 } })

Esta consulta recupera todos los documentos donde el campo age es mayor que 20 y menor que 30.

Operadores Lógicos

  • $and: Une cláusulas de consulta con un AND lógico.
  • $or: Une cláusulas de consulta con un OR lógico.
  • $not: Invierte el efecto de una expresión de consulta.
  • $nor: Une cláusulas de consulta con un NOR lógico.

Ejemplo:

db.users.find({
  "$or": [
    { "age": { "$lt": 20 } },
    { "age": { "$gt": 30 } }
  ]
})

Esta consulta recupera todos los documentos donde el campo age es menor que 20 o mayor que 30.

Proyección

La proyección se usa para especificar o restringir los campos a devolver en los resultados de la consulta.

Incluir Campos

Para incluir campos específicos en el resultado, usa un documento de proyección con el método find.

db.collection.find({}, { "field1": 1, "field2": 1 })

Ejemplo:

db.users.find({}, { "name": 1, "age": 1 })

Esta consulta recupera todos los documentos de la colección users, pero solo incluye los campos name y age en los resultados.

Excluir Campos

Para excluir campos específicos, establece el campo en 0 en el documento de proyección.

db.collection.find({}, { "field": 0 })

Ejemplo:

db.users.find({}, { "password": 0 })

Esta consulta recupera todos los documentos de la colección users, excluyendo el campo password de los resultados.

Ordenar

Para ordenar los resultados de una consulta, usa el método sort.

db.collection.find().sort({ "field": 1 }) // Orden ascendente
db.collection.find().sort({ "field": -1 }) // Orden descendente

Ejemplo:

db.users.find().sort({ "age": 1 })

Esta consulta recupera todos los documentos de la colección users, ordenados por el campo age en orden ascendente.

Limitar y Omitir

Para limitar el número de documentos devueltos por una consulta, usa el método limit. Para omitir un número de documentos, usa el método skip.

db.collection.find().limit(number)
db.collection.find().skip(number)

Ejemplo:

db.users.find().limit(5)
db.users.find().skip(5)

La primera consulta recupera los primeros 5 documentos de la colección users. La segunda consulta omite los primeros 5 documentos y recupera el resto.

Consultas Avanzadas

Documentos Anidados

Para consultar documentos anidados, usa la notación de puntos.

db.collection.find({ "field.subfield": "value" })

Ejemplo:

db.users.find({ "address.city": "New York" })

Esta consulta recupera todos los documentos de la colección users donde el campo city en el sub-documento address es "New York".

Consultas de Arreglos

MongoDB proporciona operadores para consultar arreglos.

  • $elemMatch: Coincide con documentos que contienen un campo de arreglo con al menos un elemento que coincide con todos los criterios de consulta especificados.
  • $size: Coincide con cualquier arreglo con el número especificado de elementos.

Ejemplo:

db.users.find({ "hobbies": { "$elemMatch": { "name": "reading", "frequency": "daily" } } })

Esta consulta recupera todos los documentos donde el arreglo hobbies contiene un elemento con name "reading" y frequency "daily".

Conclusión

Consultar documentos en MongoDB es una característica poderosa que te permite recuperar y manipular datos de manera eficiente. Al comprender y utilizar varios operadores de consulta, proyección, ordenación y técnicas de consulta avanzada, puedes realizar tareas complejas de recuperación de datos con facilidad. Practica estos conceptos con datos del mundo real para volverte competente en la consulta de colecciones de MongoDB.
