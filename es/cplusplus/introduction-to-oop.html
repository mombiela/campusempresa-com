Resumen

La Programación Orientada a Objetos (POO) es un paradigma de programación que utiliza objetos y clases para estructurar el software de una manera modular, reutilizable y fácil de mantener. En esta sección, cubriremos los conceptos fundamentales de la POO en C++, incluyendo clases, objetos, herencia, polimorfismo, encapsulación y abstracción.

Conceptos Clave

Clases y Objetos

  • Clase: Un plano para crear objetos. Define un tipo de dato agrupando datos y métodos que trabajan sobre los datos.
  • Objeto: Una instancia de una clase. Es un componente autónomo que contiene propiedades y métodos necesarios para hacer útil un cierto tipo de datos.

Ejemplo:

#include <iostream>
using namespace std;

class Car {
public:
    string brand;
    string model;
    int year;

    void displayInfo() {
        cout << "Brand: " << brand << ", Model: " << model << ", Year: " << year << endl;
    }
};

int main() {
    Car car1;
    car1.brand = "Toyota";
    car1.model = "Corolla";
    car1.year = 2020;
    car1.displayInfo();

    return 0;
}

Explicación:

  • Definimos una clase Car con tres atributos públicos: brand, model y year.
  • También definimos un método displayInfo para imprimir los detalles del coche.
  • En la función main, creamos un objeto car1 de la clase Car y establecemos sus atributos.

Encapsulación

  • Encapsulación: El concepto de envolver datos y métodos que operan sobre los datos dentro de una sola unidad (clase). Restringe el acceso directo a algunos de los componentes del objeto, lo cual es un medio para prevenir interferencias accidentales y el mal uso de los datos.

Ejemplo:

#include <iostream>
using namespace std;

class Person {
private:
    string name;
    int age;

public:
    void setName(string n) {
        name = n;
    }

    string getName() {
        return name;
    }

    void setAge(int a) {
        if (a > 0) {
            age = a;
        }
    }

    int getAge() {
        return age;
    }
};

int main() {
    Person person1;
    person1.setName("Alice");
    person1.setAge(30);
    cout << "Name: " << person1.getName() << ", Age: " << person1.getAge() << endl;

    return 0;
}

Explicación:

  • La clase Person tiene atributos privados name y age.
  • Los métodos públicos setName, getName, setAge y getAge se utilizan para acceder y modificar los atributos privados.

Herencia

  • Herencia: Un mecanismo mediante el cual una clase puede heredar propiedades y métodos de otra clase. Promueve la reutilización del código.

Ejemplo:

#include <iostream>
using namespace std;

class Animal {
public:
    void eat() {
        cout << "This animal is eating." << endl;
    }
};

class Dog : public Animal {
public:
    void bark() {
        cout << "The dog is barking." << endl;
    }
};

int main() {
    Dog dog1;
    dog1.eat();  // Método heredado
    dog1.bark(); // Método propio del perro

    return 0;
}

Explicación:

  • La clase Animal tiene un método eat.
  • La clase Dog hereda de Animal y añade su propio método bark.
  • El objeto Dog dog1 puede usar tanto el método heredado eat como su propio método bark.

Polimorfismo

  • Polimorfismo: La capacidad de diferentes clases de ser tratadas como instancias de la misma clase a través de la herencia. Permite que los métodos hagan cosas diferentes basadas en el objeto sobre el que están actuando.

Ejemplo:

#include <iostream>
using namespace std;

class Animal {
public:
    virtual void makeSound() {
        cout << "Some generic animal sound." << endl;
    }
};

class Dog : public Animal {
public:
    void makeSound() override {
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() override {
        cout << "Meow!" << endl;
    }
};

int main() {
    Animal* animal1 = new Dog();
    Animal* animal2 = new Cat();

    animal1->makeSound(); // Salida: Woof!
    animal2->makeSound(); // Salida: Meow!

    delete animal1;
    delete animal2;

    return 0;
}

Explicación:

  • La clase base Animal tiene un método virtual makeSound.
  • Las clases derivadas Dog y Cat sobrescriben el método makeSound.
  • Los punteros a Animal pueden apuntar a objetos Dog o Cat, y el método makeSound correcto se llama basado en el tipo real del objeto.

Abstracción

  • Abstracción: El concepto de ocultar los detalles complejos de implementación y mostrar solo las características necesarias de un objeto. Ayuda a reducir la complejidad y el esfuerzo de programación.

Ejemplo:

#include <iostream>
using namespace std;

class AbstractDevice {
public:
    virtual void turnOn() = 0; // Función virtual pura
    virtual void turnOff() = 0; // Función virtual pura
};

class Fan : public AbstractDevice {
public:
    void turnOn() override {
        cout << "Fan is now ON." << endl;
    }

    void turnOff() override {
        cout << "Fan is now OFF." << endl;
    }
};

int main() {
    Fan fan1;
    fan1.turnOn();
    fan1.turnOff();

    return 0;
}

Explicación:

  • AbstractDevice es una clase abstracta con funciones virtuales puras turnOn y turnOff.
  • La clase Fan hereda de AbstractDevice y proporciona implementaciones para las funciones virtuales puras.

Resumen

En esta sección, introdujimos los conceptos fundamentales de la Programación Orientada a Objetos (POO) en C++. Cubrimos:

  • Clases y Objetos: Los bloques de construcción básicos de la POO.
  • Encapsulación: Protegiendo datos restringiendo el acceso a ellos.
  • Herencia: Reutilizando código heredando propiedades y métodos de clases existentes.
  • Polimorfismo: Permitiendo que los métodos se comporten de manera diferente según el objeto sobre el que están actuando.
  • Abstracción: Simplificando sistemas complejos mostrando solo los detalles necesarios.

Entender estos conceptos es crucial para escribir código modular, reutilizable y mantenible en C++. A medida que avances en el curso, verás cómo se aplican estos principios en escenarios más complejos.
