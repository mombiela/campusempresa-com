La asignación dinámica de memoria es un concepto crucial en C++ que permite a los programas solicitar memoria en tiempo de ejecución. Esto es esencial para crear aplicaciones flexibles y eficientes que puedan manejar cantidades variables de datos.

Introducción a la Asignación Dinámica de Memoria

La asignación dinámica de memoria permite a los programas asignar memoria según sea necesario durante la ejecución, en lugar de en tiempo de compilación. Esto es particularmente útil para manejar estructuras de datos cuyo tamaño no puede determinarse de antemano.

Conceptos Clave

  • Heap vs. Stack:
    • Stack: Memoria asignada en tiempo de compilación, gestionada automáticamente y limitada en tamaño.
    • Heap: Memoria asignada en tiempo de ejecución, gestionada manualmente y generalmente más grande que la stack.
  • Punteros: Variables que almacenan direcciones de memoria, esenciales para la asignación dinámica de memoria.
  • Operadores: new y delete para objetos individuales, new[] y delete[] para arrays.

Asignación Dinámica de Memoria Básica

Asignación y Desasignación de Objetos Individuales

Para asignar memoria para un solo objeto, use el operador new. Para desasignar, use el operador delete.

int* ptr = new int; // Asignar memoria para un entero
*ptr = 10;         // Asignar un valor a la memoria asignada
delete ptr;        // Desasignar la memoria

Asignación y Desasignación de Arrays

Para arrays, use new[] y delete[].

int* arr = new int[5]; // Asignar memoria para un array de 5 enteros
for (int i = 0; i < 5; ++i) {
    arr[i] = i * 10;  // Asignar valores al array
}
delete[] arr;         // Desasignar la memoria

Asignación Dinámica de Memoria Avanzada

Eliminadores Personalizados

Al usar punteros inteligentes, se pueden especificar eliminadores personalizados para gestionar la limpieza de recursos.

#include <memory>
#include <iostream>

void customDeleter(int* ptr) {
    std::cout << "Eliminador personalizado llamado\n";
    delete ptr;
}

int main() {
    std::unique_ptr<int, void(*)(int*)> ptr(new int, customDeleter);
    *ptr = 20;
    return 0;
}

Fugas de Memoria y Gestión

La gestión adecuada de la memoria es crucial para evitar fugas de memoria, donde la memoria asignada no se desasigna correctamente.

  • Ejemplo de Fuga de Memoria:
void memoryLeak() {
    int* ptr = new int(10);
    // Olvidó desasignar ptr, causando una fuga de memoria
}
  • Evitando Fugas de Memoria:
void noMemoryLeak() {
    int* ptr = new int(10);
    delete ptr; // Desasignar la memoria correctamente
}

Punteros Inteligentes

Los punteros inteligentes (std::unique_ptr, std::shared_ptr, std::weak_ptr) ayudan a gestionar la memoria dinámica automáticamente.

  • Puntero Único:
#include <memory>

std::unique_ptr<int> uniquePtr(new int(10));
  • Puntero Compartido:
#include <memory>

std::shared_ptr<int> sharedPtr1 = std::make_shared<int>(10);
std::shared_ptr<int> sharedPtr2 = sharedPtr1; // Propiedad compartida

Resumen

La asignación dinámica de memoria en C++ permite una gestión de memoria flexible y eficiente en tiempo de ejecución. Los conceptos clave incluyen el uso del heap, punteros y los operadores new y delete. Las técnicas avanzadas implican eliminadores personalizados y punteros inteligentes para gestionar la memoria automáticamente y evitar fugas. La comprensión y aplicación adecuada de estos conceptos son esenciales para desarrollar aplicaciones robustas en C++.
