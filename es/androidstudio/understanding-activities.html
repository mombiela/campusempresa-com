Introducción a las Actividades

Las actividades son un componente fundamental de las aplicaciones Android. Representan una sola pantalla con una interfaz de usuario. Una aplicación puede tener múltiples actividades, y cada actividad es independiente de las demás.

Conceptos Clave

  • Ciclo de Vida de la Actividad: La serie de estados por los que pasa una actividad desde su creación hasta su destrucción.
  • Intenciones (Intents): Objetos de mensajería utilizados para solicitar una acción de otro componente de la aplicación.
  • Componentes de la UI: Widgets y diseños que componen la interfaz de usuario de una actividad.

Ciclo de Vida de la Actividad

Comprender el ciclo de vida de la actividad es crucial para gestionar el estado de tu aplicación y asegurar una experiencia de usuario fluida.

Métodos del Ciclo de Vida

  • onCreate(): Llamado cuando la actividad se crea por primera vez. Aquí es donde inicializas tu actividad.
  • onStart(): Llamado cuando la actividad se vuelve visible para el usuario.
  • onResume(): Llamado cuando la actividad comienza a interactuar con el usuario.
  • onPause(): Llamado cuando el sistema está a punto de reanudar otra actividad.
  • onStop(): Llamado cuando la actividad ya no es visible para el usuario.
  • onDestroy(): Llamado antes de que la actividad sea destruida.

Diagrama del Ciclo de Vida

Aquí hay un diagrama simplificado del ciclo de vida de la actividad:

onCreate() -> onStart() -> onResume()
   ^            |            |
   |            v            v
onDestroy() <- onStop() <- onPause()

Ejemplo de Código

Aquí hay una implementación básica de una actividad con métodos del ciclo de vida:

public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        Log.d("Lifecycle", "onCreate called");
    }

    @Override
    protected void onStart() {
        super.onStart();
        Log.d("Lifecycle", "onStart called");
    }

    @Override
    protected void onResume() {
        super.onResume();
        Log.d("Lifecycle", "onResume called");
    }

    @Override
    protected void onPause() {
        super.onPause();
        Log.d("Lifecycle", "onPause called");
    }

    @Override
    protected void onStop() {
        super.onStop();
        Log.d("Lifecycle", "onStop called");
    }

    @Override
    protected void onDestroy() {
        super.onDestroy();
        Log.d("Lifecycle", "onDestroy called");
    }
}

Intenciones (Intents)

Las intenciones se utilizan para iniciar actividades y comunicarse entre diferentes componentes de una aplicación.

Tipos de Intenciones

  • Intenciones Explícitas: Especifican el componente a iniciar por nombre.
  • Intenciones Implícitas: No especifican el componente directamente; en su lugar, declaran una acción general a realizar.

Ejemplo de Código

Aquí hay un ejemplo de cómo usar una intención explícita para iniciar una nueva actividad:

Intent intent = new Intent(this, SecondActivity.class);
startActivity(intent);

Y aquí hay un ejemplo de cómo usar una intención implícita para abrir una página web:

Intent intent = new Intent(Intent.ACTION_VIEW);
intent.setData(Uri.parse("http://www.example.com"));
startActivity(intent);

Componentes de la UI

Las actividades están compuestas por varios componentes de la UI, como botones, vistas de texto y diseños.

Componentes Comunes de la UI

  • TextView: Muestra texto al usuario.
  • Button: Un botón clicable.
  • EditText: Un campo de texto que permite la entrada del usuario.
  • LinearLayout: Un diseño que organiza sus hijos en una sola columna o fila.

Ejemplo de Código

Aquí hay un ejemplo de un diseño de actividad simple usando XML:

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <TextView
        android:id="@+id/textView"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello, World!" />

    <Button
        android:id="@+id/button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Click Me" />
</LinearLayout>

Conclusión

Comprender las actividades es esencial para desarrollar aplicaciones Android. Al dominar el ciclo de vida de la actividad, las intenciones y los componentes de la UI, puedes crear aplicaciones robustas y amigables para el usuario. Recuerda gestionar adecuadamente los estados del ciclo de vida para asegurar una experiencia de usuario fluida y utilizar intenciones para navegar eficazmente entre las diferentes partes de tu aplicación.
