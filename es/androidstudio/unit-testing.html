Las pruebas unitarias son una parte crucial del proceso de desarrollo de software, asegurando que las unidades individuales de código funcionen como se espera. En el desarrollo de Android, las pruebas unitarias ayudan a mantener la calidad y la fiabilidad del código. Este tema te guiará desde los conceptos básicos hasta los avanzados de las pruebas unitarias en Android Studio.

Introducción a las Pruebas Unitarias

  • Definición: Las pruebas unitarias implican probar componentes o funciones individuales de una aplicación de forma aislada.
  • Propósito: Verificar que cada unidad del software funcione según lo diseñado.
  • Beneficios:
    • Detecta errores temprano.
    • Simplifica la depuración.
    • Facilita la refactorización del código.
    • Mejora la calidad y la fiabilidad del código.

Configuración de Pruebas Unitarias en Android Studio

  1. Crear un Nuevo Proyecto:

    • Abre Android Studio y crea un nuevo proyecto.
    • Asegúrate de seleccionar el SDK y la configuración del proyecto adecuados.

  2. Agregar Dependencias de Pruebas:

    • Abre el archivo build.gradle (Módulo: app) y agrega las siguientes dependencias: 
      dependencies {
    testImplementation 'junit:junit:4.13.2'
    testImplementation 'org.mockito:mockito-core:3.9.0'
}

  3. Sincronizar el Proyecto:

    • Haz clic en "Sync Now" para descargar e integrar las dependencias.

Escribiendo Tu Primera Prueba Unitaria

  1. Crear una Clase de Prueba:

    • Navega al directorio src/test/java.
    • Haz clic derecho y selecciona New > Java Class.
    • Nombra la clase ExampleUnitTest.

  2. Escribir una Prueba Simple:

    • Agrega un método de prueba simple a la clase: 
      import org.junit.Test;
import static org.junit.Assert.*;

public class ExampleUnitTest {
    @Test
    public void addition_isCorrect() {
        assertEquals(4, 2 + 2);
    }
}

  3. Ejecutar la Prueba:

    • Haz clic derecho en la clase de prueba y selecciona Run 'ExampleUnitTest'.
    • Verifica los resultados en la ventana de ejecución.

Conceptos Avanzados de Pruebas Unitarias

Simulación con Mockito

  • Definición: La simulación es una técnica para crear una versión simulada de un objeto.
  • Propósito: Aislar la unidad de código que se está probando y controlar sus dependencias.

Ejemplo: Simulación de una Dependencia

  1. Agregar Dependencia de Mockito (ya agregada en el paso de configuración):

    testImplementation 'org.mockito:mockito-core:3.9.0'

  2. Crear una Clase con Dependencias:

    public class UserService {
    private UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public String getUserFullName(int userId) {
        User user = userRepository.getUserById(userId);
        return user.getFirstName() + " " + user.getLastName();
    }
}

public interface UserRepository {
    User getUserById(int userId);
}

public class User {
    private String firstName;
    private String lastName;

    // Constructor, getters, and setters
}

  3. Escribir una Prueba con Mockito:

    import org.junit.Before;
import org.junit.Test;
import org.mockito.Mock;
import org.mockito.MockitoAnnotations;

import static org.mockito.Mockito.*;
import static org.junit.Assert.*;

public class UserServiceTest {
    @Mock
    private UserRepository userRepository;

    private UserService userService;

    @Before
    public void setUp() {
        MockitoAnnotations.initMocks(this);
        userService = new UserService(userRepository);
    }

    @Test
    public void getUserFullName_returnsCorrectFullName() {
        User user = new User("John", "Doe");
        when(userRepository.getUserById(1)).thenReturn(user);

        String fullName = userService.getUserFullName(1);
        assertEquals("John Doe", fullName);
    }
}

Mejores Prácticas para Pruebas Unitarias

  • Escribir Pruebas Independientes: Asegúrate de que las pruebas no dependan unas de otras.
  • Usar Nombres Descriptivos: Nombra las pruebas claramente para describir su propósito.
  • Probar Casos Límite: Incluye pruebas para condiciones de límite y casos extremos.
  • Mantener las Pruebas Rápidas: Las pruebas unitarias deben ser rápidas de ejecutar.
  • Mantener las Pruebas: Actualiza regularmente las pruebas a medida que el código evoluciona.

Conclusión

Las pruebas unitarias son una práctica esencial en el desarrollo de Android, ayudando a asegurar que tu código sea fiable y mantenible. Al configurar pruebas unitarias en Android Studio, escribir pruebas básicas y avanzadas, y seguir las mejores prácticas, puedes mejorar significativamente la calidad de tus aplicaciones. Recuerda, la clave para unas pruebas unitarias efectivas es la consistencia y la exhaustividad. ¡Felices pruebas!
