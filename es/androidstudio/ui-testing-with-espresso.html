Introducción a las Pruebas de UI

  • ¿Qué son las Pruebas de UI?

    • Las pruebas de UI aseguran que la interfaz de usuario de una aplicación funcione como se espera.
    • Implican probar los elementos visuales y las interacciones del usuario.

  • ¿Por qué Usar Espresso para las Pruebas de UI?

    • Espresso es un marco de pruebas para Android que facilita la escritura de pruebas de UI confiables.
    • Es parte de la Biblioteca de Soporte de Pruebas de Android.
    • Las pruebas de Espresso se ejecutan en dispositivos reales y emuladores.

Configuración de Espresso en Android Studio

  • Agregar Dependencias

    • Abre tu archivo build.gradle y agrega las siguientes dependencias: 
      androidTestImplementation 'androidx.test.espresso:espresso-core:3.4.0'
androidTestImplementation 'androidx.test.ext:junit:1.1.3'
androidTestImplementation 'androidx.test:rules:1.4.0'

  • Sincronizar el Proyecto

    • Después de agregar las dependencias, sincroniza tu proyecto para descargar las bibliotecas necesarias.

Escribiendo tu Primera Prueba con Espresso

  • Crear una Clase de Prueba

    • Crea una nueva clase de prueba en el directorio androidTest: 
      package com.example.myapp;

import androidx.test.ext.junit.runners.AndroidJUnit4;
import androidx.test.rule.ActivityTestRule;

import org.junit.Rule;
import org.junit.Test;
import org.junit.runner.RunWith;

import static androidx.test.espresso.Espresso.onView;
import static androidx.test.espresso.action.ViewActions.click;
import static androidx.test.espresso.matcher.ViewMatchers.withId;

@RunWith(AndroidJUnit4.class)
public class MainActivityTest {

    @Rule
    public ActivityTestRule<MainActivity> activityRule =
            new ActivityTestRule<>(MainActivity.class);

    @Test
    public void testButtonClick() {
        onView(withId(R.id.my_button)).perform(click());
    }
}

  • Explicación

    • @RunWith(AndroidJUnit4.class): Especifica que la prueba debe ejecutarse usando el corredor de pruebas AndroidJUnit4.
    • ActivityTestRule: Lanza la actividad especificada antes de cada prueba y la cierra después de cada prueba.
    • onView(withId(R.id.my_button)): Encuentra la vista con el ID especificado.
    • perform(click()): Simula una acción de clic en la vista encontrada.

Conceptos Avanzados de Espresso

  • Matchers

    • Los matchers se utilizan para encontrar vistas en la UI.
    • Los matchers comunes incluyen:
      • withId(int id): Encuentra una vista por su ID.
      • withText(String text): Encuentra una vista por su texto.
      • isDisplayed(): Encuentra vistas que están actualmente mostradas.

  • Acciones de Vista

    • Las acciones son operaciones que se pueden realizar en las vistas.
    • Las acciones comunes incluyen:
      • click(): Hace clic en una vista.
      • typeText(String text): Escribe texto en una vista.
      • scrollTo(): Desplaza hasta una vista.

  • Aserciones de Vista

    • Las aserciones se utilizan para verificar el estado de las vistas.
    • Las aserciones comunes incluyen:
      • matches(Matcher<View> matcher): Verifica si una vista coincide con una condición dada.
      • doesNotExist(): Verifica si una vista no existe.

Ejemplo: Probando una Pantalla de Inicio de Sesión

  • Clase de Prueba

    package com.example.myapp;

import androidx.test.ext.junit.runners.AndroidJUnit4;
import androidx.test.rule.ActivityTestRule;

import org.junit.Rule;
import org.junit.Test;
import org.junit.runner.RunWith;

import static androidx.test.espresso.Espresso.onView;
import static androidx.test.espresso.action.ViewActions.click;
import static androidx.test.espresso.action.ViewActions.typeText;
import static androidx.test.espresso.matcher.ViewMatchers.withId;
import static androidx.test.espresso.matcher.ViewMatchers.withText;
import static androidx.test.espresso.assertion.ViewAssertions.matches;

@RunWith(AndroidJUnit4.class)
public class LoginActivityTest {

    @Rule
    public ActivityTestRule<LoginActivity> activityRule =
            new ActivityTestRule<>(LoginActivity.class);

    @Test
    public void testLoginSuccess() {
        onView(withId(R.id.username)).perform(typeText("user"));
        onView(withId(R.id.password)).perform(typeText("password"));
        onView(withId(R.id.login_button)).perform(click());
        onView(withText("Login Successful")).check(matches(isDisplayed()));
    }

    @Test
    public void testLoginFailure() {
        onView(withId(R.id.username)).perform(typeText("user"));
        onView(withId(R.id.password)).perform(typeText("wrongpassword"));
        onView(withId(R.id.login_button)).perform(click());
        onView(withText("Login Failed")).check(matches(isDisplayed()));
    }
}

  • Explicación

    • testLoginSuccess(): Prueba un escenario de inicio de sesión exitoso.
    • testLoginFailure(): Prueba un escenario de inicio de sesión fallido.
    • typeText("text"): Escribe texto en la vista especificada.
    • check(matches(isDisplayed())): Verifica que una vista esté mostrada.

Conclusión

  • Resumen

    • Espresso es una herramienta poderosa para las pruebas de UI en Android.
    • Proporciona una API simple para interactuar y verificar el estado de los componentes de la UI.
    • Al dominar Espresso, puedes asegurar que la UI de tu aplicación se comporte como se espera, proporcionando una mejor experiencia de usuario.
