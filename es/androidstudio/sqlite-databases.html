Introducción a SQLite en Android

SQLite es una base de datos ligera basada en disco que no requiere un proceso de servidor separado y permite el acceso a la base de datos utilizando una variante no estándar del lenguaje de consulta SQL. Es una opción popular para el almacenamiento local en aplicaciones Android debido a su simplicidad y eficiencia.

Configuración de SQLite en Android Studio

Para comenzar a usar SQLite en tu proyecto de Android, sigue estos pasos:

  1. Crea un nuevo proyecto Android en Android Studio.
  2. Añade una nueva clase para gestionar las operaciones de la base de datos.

Ejemplo: Creación de una clase Helper para la base de datos

public class MyDatabaseHelper extends SQLiteOpenHelper {

    private static final String DATABASE_NAME = "mydatabase.db";
    private static final int DATABASE_VERSION = 1;

    public MyDatabaseHelper(Context context) {
        super(context, DATABASE_NAME, null, DATABASE_VERSION);
    }

    @Override
    public void onCreate(SQLiteDatabase db) {
        // Crear tablas
        String CREATE_TABLE = "CREATE TABLE my_table (id INTEGER PRIMARY KEY, name TEXT)";
        db.execSQL(CREATE_TABLE);
    }

    @Override
    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
        // Lógica de actualización
        db.execSQL("DROP TABLE IF EXISTS my_table");
        onCreate(db);
    }
}
  • DATABASE_NAME: El nombre del archivo de la base de datos.
  • DATABASE_VERSION: La versión del esquema de la base de datos.
  • onCreate: Se llama cuando la base de datos se crea por primera vez.
  • onUpgrade: Se llama cuando la base de datos necesita ser actualizada.

Operaciones CRUD básicas

CRUD significa Crear, Leer, Actualizar y Eliminar. Estas son las cuatro operaciones básicas que puedes realizar en una base de datos.

Crear (Insertar)

public void addRecord(String name) {
    SQLiteDatabase db = this.getWritableDatabase();
    ContentValues values = new ContentValues();
    values.put("name", name);
    db.insert("my_table", null, values);
    db.close();
}
  • ContentValues: Una estructura de pares clave/valor para insertar datos en la base de datos.

Leer (Consultar)

public List<String> getAllRecords() {
    List<String> records = new ArrayList<>();
    SQLiteDatabase db = this.getReadableDatabase();
    Cursor cursor = db.rawQuery("SELECT * FROM my_table", null);
    if (cursor.moveToFirst()) {
        do {
            records.add(cursor.getString(1)); // 1 es el índice de la columna 'name'
        } while (cursor.moveToNext());
    }
    cursor.close();
    db.close();
    return records;
}
  • Cursor: Se usa para iterar sobre el conjunto de resultados de una consulta a la base de datos.

Actualizar

public void updateRecord(int id, String newName) {
    SQLiteDatabase db = this.getWritableDatabase();
    ContentValues values = new ContentValues();
    values.put("name", newName);
    db.update("my_table", values, "id = ?", new String[]{String.valueOf(id)});
    db.close();
}
  • update: Actualiza las filas especificadas en la base de datos.

Eliminar

public void deleteRecord(int id) {
    SQLiteDatabase db = this.getWritableDatabase();
    db.delete("my_table", "id = ?", new String[]{String.valueOf(id)});
    db.close();
}
  • delete: Elimina las filas especificadas de la base de datos.

Funciones avanzadas de SQLite

Transacciones

Las transacciones te permiten ejecutar una serie de operaciones de base de datos como una única unidad de trabajo.

public void performTransaction() {
    SQLiteDatabase db = this.getWritableDatabase();
    db.beginTransaction();
    try {
        // Realizar múltiples operaciones
        db.execSQL("INSERT INTO my_table (name) VALUES ('John')");
        db.execSQL("INSERT INTO my_table (name) VALUES ('Doe')");
        db.setTransactionSuccessful();
    } finally {
        db.endTransaction();
    }
    db.close();
}
  • beginTransaction: Inicia una nueva transacción.
  • setTransactionSuccessful: Marca la transacción actual como exitosa.
  • endTransaction: Finaliza la transacción actual.

Índices

Los índices pueden mejorar el rendimiento de tus consultas a la base de datos.

@Override
public void onCreate(SQLiteDatabase db) {
    String CREATE_TABLE = "CREATE TABLE my_table (id INTEGER PRIMARY KEY, name TEXT)";
    db.execSQL(CREATE_TABLE);
    String CREATE_INDEX = "CREATE INDEX index_name ON my_table(name)";
    db.execSQL(CREATE_INDEX);
}
  • CREATE INDEX: Crea un índice en la(s) columna(s) especificada(s).

Mejores prácticas

  • Usa consultas parametrizadas para prevenir la inyección de SQL.
  • Cierra las conexiones a la base de datos para evitar fugas de memoria.
  • Usa transacciones para operaciones en lote para asegurar la integridad de los datos.
  • Optimiza las consultas con índices y un diseño de esquema adecuado.

Conclusión

SQLite es una solución de base de datos poderosa y eficiente para aplicaciones Android. Al comprender e implementar características básicas y avanzadas, puedes gestionar eficazmente el almacenamiento de datos locales en tus aplicaciones. Practica las operaciones CRUD, usa transacciones para operaciones complejas y sigue las mejores prácticas para asegurar que la base de datos de tu aplicación sea robusta y eficiente.
