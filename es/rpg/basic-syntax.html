Introducción

RPG (Report Program Generator) es un lenguaje de programación de alto nivel utilizado principalmente para aplicaciones empresariales. Comprender la sintaxis y estructura básica de RPG es crucial para escribir programas eficientes y efectivos. Esta sección cubrirá los elementos fundamentales de la sintaxis y estructura de RPG, proporcionando una base sólida para un aprendizaje más avanzado.

Conceptos Clave

  • Especificaciones: Los programas RPG se dividen en diferentes tipos de especificaciones, cada uno con un propósito específico.

    • H-Spec: Especificación de encabezado, utilizada para configuraciones globales.
    • F-Spec: Especificación de archivo, utilizada para definir archivos.
    • D-Spec: Especificación de definición, utilizada para definir variables, constantes y estructuras de datos.
    • C-Spec: Especificación de cálculo, utilizada para código ejecutable.
    • P-Spec: Especificación de procedimiento, utilizada para definir subrutinas y procedimientos.

  • Formato Fijo vs. Formato Libre: RPG IV introdujo la codificación en formato libre, que es más flexible y legible en comparación con el formato fijo tradicional.

Especificaciones en Detalle

H-Spec (Especificación de Encabezado)

La H-spec se utiliza para establecer parámetros globales para el programa.

H dftactgrp(*no) actgrp('MYACTGRP')
  • dftactgrp(*no): Especifica que el programa no se ejecuta en el grupo de activación predeterminado.
  • actgrp('MYACTGRP'): Define el grupo de activación para el programa.

F-Spec (Especificación de Archivo)

La F-spec se utiliza para definir los archivos que el programa utilizará.

FMYFILE   IF   E           K DISK
  • FMYFILE: El nombre del archivo.
  • I: Archivo de entrada.
  • F: Archivo de procedimiento completo.
  • E: Archivo externo.
  • K: Archivo con clave.
  • DISK: Tipo de archivo.

D-Spec (Especificación de Definición)

La D-spec se utiliza para definir variables, constantes y estructuras de datos.

D CustomerName   S             50A
D OrderTotal     S              9P 2
  • CustomerName: Una variable de tipo carácter con una longitud de 50.
  • OrderTotal: Una variable decimal empaquetada con 9 dígitos y 2 decimales.

C-Spec (Especificación de Cálculo)

La C-spec es donde reside el código ejecutable.

C     Eval      OrderTotal = OrderTotal + 100.00
C     If        OrderTotal > 500.00
C     Eval      Discount = OrderTotal * 0.1
C     EndIf
  • Eval: Utilizado para asignar valores a variables.
  • If y EndIf: Utilizados para lógica condicional.

P-Spec (Especificación de Procedimiento)

La P-spec se utiliza para definir subrutinas y procedimientos.

P MyProcedure   B
D MyProcedure   PI
D InputValue    10I 0
D OutputValue   10I 0

C     Eval      OutputValue = InputValue * 2
C     Return
P MyProcedure   E
  • B y E: Inicio y fin del procedimiento.
  • PI: Interfaz del procedimiento.
  • Eval: Cálculo dentro del procedimiento.
  • Return: Fin de la lógica del procedimiento.

Formato Fijo vs. Formato Libre

Ejemplo de Formato Fijo

FMYFILE   IF   E           K DISK
D CustomerName   S             50A
C     Eval      CustomerName = 'John Doe'
C     If        CustomerName = 'John Doe'
C     Eval      Greeting = 'Hello, John!'
C     EndIf

Ejemplo de Formato Libre

dcl-f MYFILE DISK(*EXT) USAGE(*INPUT);

dcl-s CustomerName char(50);
dcl-s Greeting char(50);

CustomerName = 'John Doe';

if CustomerName = 'John Doe';
  Greeting = 'Hello, John!';
endif;
  • Formato Libre: Más legible y flexible, permitiendo prácticas de codificación modernas.

Conclusión

Comprender la sintaxis y estructura básica de RPG es esencial para escribir programas efectivos. Al dominar las diferentes especificaciones y las diferencias entre el formato fijo y el formato libre, estarás bien preparado para abordar tareas de programación más complejas en RPG. Este conocimiento fundamental servirá como un trampolín hacia temas más avanzados en la programación RPG.
