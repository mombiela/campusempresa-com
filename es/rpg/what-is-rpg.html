Introducción a RPG

RPG, que significa Report Program Generator, es un lenguaje de programación de alto nivel utilizado principalmente para aplicaciones empresariales. Fue desarrollado originalmente por IBM en la década de 1960 y ha evolucionado significativamente a lo largo de los años. RPG es conocido por su simplicidad y eficiencia en el manejo de la lógica empresarial y las tareas de procesamiento de datos.

Conceptos Clave de RPG

  • Lenguaje de Alto Nivel: RPG está diseñado para ser fácil de leer y escribir, lo que lo hace accesible para los programadores de negocios.
  • Aplicaciones Empresariales: Se utiliza ampliamente en industrias como finanzas, manufactura y comercio minorista para crear informes, gestionar datos y automatizar procesos empresariales.
  • Sistemas IBM i: RPG se utiliza predominantemente en sistemas IBM i (anteriormente AS/400), conocidos por su robustez y fiabilidad.

Evolución de RPG

RPG ha pasado por varias versiones, cada una añadiendo nuevas características y capacidades:

  • RPG I: La versión original, introducida en la década de 1960.
  • RPG II: Introducida en la década de 1970, añadió más funcionalidad y mejoró el rendimiento.
  • RPG III: Lanzada en la década de 1980, incorporó conceptos de programación estructurada.
  • RPG IV (RPGLE): La versión moderna, introducida en la década de 1990, soporta codificación en formato libre e integra con entornos de desarrollo modernos.

Estructura Básica de un Programa RPG

Un programa RPG típicamente consta de las siguientes secciones:

  1. H-Specs (Especificaciones de Cabecera): Definen configuraciones a nivel de programa.
  2. F-Specs (Especificaciones de Archivo): Definen los archivos utilizados en el programa.
  3. D-Specs (Especificaciones de Datos): Definen variables y estructuras de datos.
  4. C-Specs (Especificaciones de Cálculo): Contienen la lógica y los cálculos.
  5. O-Specs (Especificaciones de Salida): Definen el formato de salida.

Ejemplo de un Programa RPG Simple

Aquí hay un ejemplo básico de un programa RPG IV (RPGLE):

**FREE
// H-Specs
H DFTACTGRP(*NO) ACTGRP(*NEW)

// F-Specs
FMYFILE   IF   E           K DISK

// D-Specs
D Name            S             50A

// C-Specs
C     *ENTRY        PLIST
C                   PARM                    Name

C                   IF        Name = 'John'
C                   EVAL      *INLR = *ON
C                   RETURN
C                   ENDIF

C                   EVAL      *INLR = *ON
C                   RETURN

Explicación del Código

  • H-Specs: DFTACTGRP(*NO) y ACTGRP(*NEW) especifican que el programa no utiliza el grupo de activación predeterminado y crea un nuevo grupo de activación.
  • F-Specs: La línea FMYFILE define un archivo llamado MYFILE que el programa utilizará.
  • D-Specs: La línea D Name define una variable Name de tipo carácter con una longitud de 50.
  • C-Specs: Las especificaciones de cálculo contienen la lógica del programa. En este ejemplo, verifica si el Name es 'John' y establece el indicador de último registro (*INLR) a *ON, lo que termina el programa.

Conclusión

RPG es un lenguaje poderoso y eficiente para aplicaciones empresariales, particularmente en sistemas IBM i. Comprender su estructura y conceptos clave es esencial para aprovechar sus capacidades. A medida que avances en este curso, aprenderás características y técnicas más avanzadas para convertirte en un programador competente en RPG.
