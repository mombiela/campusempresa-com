Introducción

En esta sección, exploraremos el manejo de archivos y las estructuras de datos en RPG (Report Program Generator). Comprender estos conceptos es crucial para gestionar datos de manera eficiente y escribir programas robustos. Comenzaremos con los conceptos básicos del manejo de archivos y gradualmente avanzaremos hacia estructuras de datos más complejas.

Manejo de Archivos

Conceptos Básicos del Manejo de Archivos

El manejo de archivos en RPG implica leer y escribir en archivos. RPG soporta varios tipos de archivos, incluyendo archivos físicos, archivos lógicos y archivos de pantalla.

  • Archivos Físicos: Estos son los archivos reales que almacenan datos.
  • Archivos Lógicos: Estos proporcionan una vista u orden diferente de los datos almacenados en archivos físicos.
  • Archivos de Pantalla: Se utilizan para crear interfaces de usuario.

Lectura de un Archivo

Para leer datos de un archivo, necesitas definir el archivo en la Especificación de Archivos (F-spec) y usar la operación READ.

FMYFILE    IF   E           K DISK

DName            S             50A

C     READ      MYFILE
C     IF        %EOF(MYFILE)
C     ELSE
C       EVAL    Name = MYFILE.Name
C     ENDIF
C     *INLR     = *ON
  • F-spec: Define el archivo y sus atributos.
  • READ: Lee un registro del archivo.
  • %EOF: Verifica si se ha alcanzado el final del archivo.

Escritura en un Archivo

Escribir datos en un archivo implica usar la operación WRITE.

FMYFILE    O   E           K DISK

DName            S             50A

C     EVAL      MYFILE.Name = 'John Doe'
C     WRITE     MYFILE
C     *INLR     = *ON
  • WRITE: Escribe un registro en el archivo.

Actualización de un Archivo

Para actualizar un registro en un archivo, usa las operaciones CHAIN y UPDATE.

FMYFILE    UF   E           K DISK

DName            S             50A

C     CHAIN     'John Doe' MYFILE
C     IF        %FOUND(MYFILE)
C       EVAL    MYFILE.Name = 'Jane Doe'
C       UPDATE  MYFILE
C     ENDIF
C     *INLR     = *ON
  • CHAIN: Recupera un registro por clave.
  • UPDATE: Actualiza el registro recuperado.

Estructuras de Datos

Introducción a las Estructuras de Datos

Las estructuras de datos en RPG se utilizan para agrupar datos relacionados. Pueden ser simples o complejas, dependiendo de los requisitos.

Definición de Estructuras de Datos

Una estructura de datos se define usando la D-spec.

DNameDS          DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0
  • DS: Indica una estructura de datos.
  • A: Tipo de dato alfanumérico.
  • P: Tipo de dato decimal empaquetado.

Uso de Estructuras de Datos

Las estructuras de datos pueden usarse para simplificar la manipulación de datos.

DNameDS          DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0

C     EVAL      NameDS.FirstName = 'John'
C     EVAL      NameDS.LastName = 'Doe'
C     EVAL      NameDS.Age = 30

C     DSPLY     NameDS.FirstName + ' ' + NameDS.LastName
C     *INLR     = *ON

Estructuras de Datos Anidadas

Las estructuras de datos pueden anidarse para representar datos más complejos.

DAddressDS       DS
D  Street                     50A
D  City                       25A
D  ZipCode                    10A

DPersonDS        DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0
D  Address                     LIKE(AddressDS)
  • LIKE: Se utiliza para definir un subcampo con los mismos atributos que otro campo.

Arreglos en Estructuras de Datos

Los arreglos pueden usarse dentro de estructuras de datos para manejar múltiples valores.

DGradesDS        DS
D  StudentName                25A
D  Grades                     5P 0 DIM(5)

C     EVAL      GradesDS.StudentName = 'John Doe'
C     EVAL      GradesDS.Grades(1) = 85
C     EVAL      GradesDS.Grades(2) = 90
C     EVAL      GradesDS.Grades(3) = 88

C     DSPLY     GradesDS.StudentName
C     DSPLY     GradesDS.Grades(1)
C     *INLR     = *ON

Conclusión

En esta sección, cubrimos lo esencial del manejo de archivos y las estructuras de datos en RPG. Aprendimos cómo leer, escribir y actualizar archivos, así como cómo definir y usar estructuras de datos. Dominar estos conceptos te permitirá manejar datos de manera más eficiente y escribir código más organizado y mantenible. A medida que progreses, intenta implementar estos conceptos en tus proyectos para solidificar tu comprensión.
