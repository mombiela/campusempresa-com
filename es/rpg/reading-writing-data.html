La lectura y escritura de datos son operaciones fundamentales en la programación RPG (Report Program Generator). Este tema cubrirá los conceptos básicos del manejo de archivos, incluyendo la lectura y escritura en archivos, y progresará hacia técnicas más avanzadas y mejores prácticas.

Introducción al Manejo de Archivos en RPG

El manejo de archivos en RPG implica trabajar con varios tipos de archivos, como archivos físicos, archivos lógicos y archivos de pantalla. Entender cómo leer y escribir en estos archivos es crucial para la manipulación de datos y la generación de informes.

Conceptos Clave

  • Archivos Físicos: Son los archivos reales que almacenan datos.
  • Archivos Lógicos: Son vistas o índices sobre archivos físicos, utilizados para acceder a los datos en un orden o formato específico.
  • Archivos de Pantalla: Se utilizan para aplicaciones interactivas para mostrar datos a los usuarios.

Lectura de Datos desde Archivos

La lectura de datos desde archivos en RPG se puede realizar utilizando varias operaciones como READ, CHAIN y SETLL.

Operaciones Básicas de Lectura

Ejemplo: Leyendo un Registro desde un Archivo Físico

FMYFILE   IF   E           K DISK

DCL-S recordData CHAR(100);

READ MYFILE;
IF %EOF(MYFILE);
  // Fin del archivo alcanzado
ELSE;
  // Procesar el registro
  recordData = MYFILE;
ENDIF;
  • F-spec: Define el archivo MYFILE como un archivo de entrada (I) con acceso por clave (K).
  • READ: Lee el siguiente registro de MYFILE.
  • %EOF: Verifica si se ha alcanzado el fin del archivo.

Técnicas Avanzadas de Lectura

Ejemplo: Usando CHAIN para Recuperar un Registro Específico

FMYFILE   IF   E           K DISK

DCL-S key CHAR(10);
DCL-S recordData CHAR(100);

key = '12345';
CHAIN key MYFILE;
IF %FOUND(MYFILE);
  // Registro encontrado
  recordData = MYFILE;
ELSE;
  // Registro no encontrado
ENDIF;
  • CHAIN: Recupera un registro específico basado en el valor de la clave.

Escritura de Datos en Archivos

La escritura de datos en archivos implica operaciones como WRITE, UPDATE y DELETE.

Operaciones Básicas de Escritura

Ejemplo: Escribiendo un Registro en un Archivo Físico

FMYFILE   O   E           K DISK

DCL-S recordData CHAR(100);

recordData = 'New Record Data';
WRITE MYFILE recordData;
  • F-spec: Define el archivo MYFILE como un archivo de salida (O) con acceso por clave (K).
  • WRITE: Escribe un nuevo registro en MYFILE.

Actualización y Eliminación de Registros

Ejemplo: Actualizando un Registro

FMYFILE   U   E           K DISK

DCL-S key CHAR(10);
DCL-S recordData CHAR(100);

key = '12345';
CHAIN key MYFILE;
IF %FOUND(MYFILE);
  recordData = 'Updated Record Data';
  UPDATE MYFILE recordData;
ENDIF;
  • UPDATE: Actualiza el registro actual en el archivo.

Ejemplo: Eliminando un Registro

FMYFILE   D   E           K DISK

DCL-S key CHAR(10);

key = '12345';
CHAIN key MYFILE;
IF %FOUND(MYFILE);
  DELETE MYFILE;
ENDIF;
  • DELETE: Elimina el registro actual del archivo.

Mejores Prácticas para el Manejo de Archivos

  • Manejo de Errores: Siempre verifica errores después de las operaciones de archivo para asegurar la integridad de los datos.
  • Bloqueo de Registros: Usa mecanismos de bloqueo de registros para prevenir la corrupción de datos en entornos multiusuario.
  • Acceso Eficiente: Usa archivos lógicos y métodos de acceso apropiados para optimizar el rendimiento.

Conclusión

Leer y escribir datos en RPG es esencial para una gestión de datos y generación de informes efectiva. Al entender las operaciones básicas y avanzadas de manejo de archivos, puedes manipular datos de manera eficiente y asegurar la integridad y el rendimiento de tus aplicaciones RPG. Practica estos conceptos con varios tipos de archivos y operaciones para ganar competencia.
