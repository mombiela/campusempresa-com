Introducción

En RPG (Report Program Generator), las constantes y los literales son conceptos fundamentales que ayudan a definir valores fijos dentro de tus programas. Entender cómo usarlos efectivamente es crucial para escribir código claro y mantenible.

Constantes

Las constantes en RPG son valores nombrados que no cambian durante la ejecución del programa. Se definen usando la declaración DCL-C (Declarar Constante).

Conceptos Clave

  • Inmutabilidad: Una vez definida, el valor de una constante no puede cambiar.
  • Legibilidad: Las constantes hacen tu código más legible al dar nombres significativos a valores fijos.
  • Mantenibilidad: Si un valor constante necesita ser actualizado, solo necesitas cambiarlo en un lugar.

Definición de Constantes

Las constantes se definen en la sección de declaración de tu programa RPG. Aquí está la sintaxis para definir una constante:

DCL-C <nombre_constante> <valor_constante>;

Ejemplo

DCL-C PI 3.14159;
DCL-C MAX_USERS 100;
DCL-C WELCOME_MESSAGE '¡Bienvenido a la Programación RPG!';

En este ejemplo:

  • PI es una constante que representa la constante matemática π.
  • MAX_USERS es una constante que representa el número máximo de usuarios.
  • WELCOME_MESSAGE es una cadena constante usada para saludar.

Literales

Los literales son valores fijos usados directamente en el código sin ser asignados a una variable o constante. Pueden ser numéricos, de caracteres o literales especiales.

Conceptos Clave

  • Uso Directo: Los literales se usan directamente en expresiones y declaraciones.
  • Tipos: RPG soporta varios tipos de literales, incluyendo numéricos, de caracteres y literales especiales.

Tipos de Literales

  • Literales Numéricos: Representan números.
  • Literales de Caracteres: Representan cadenas o caracteres.
  • Literales Especiales: Incluyen valores especiales como *ON, *OFF, *NULL, etc.

Ejemplos

Literales Numéricos

DCL-S resultado S 10I 0;
resultado = 42; // Literal numérico 42
resultado = resultado + 3.14; // Literal numérico 3.14

Literales de Caracteres

DCL-S saludo S 50A;
saludo = '¡Hola, Mundo!'; // Literal de caracteres '¡Hola, Mundo!'

Literales Especiales

DCL-S bandera S 1A;
bandera = *ON; // Literal especial *ON
bandera = *OFF; // Literal especial *OFF

Comparación de Constantes y Literales

| Característica | Constantes | Literales | |------------------|------------------------------------|------------------------------------| | Definición | Definidas usando DCL-C | Usadas directamente en el código | | Mutabilidad | Inmutables | Inmutables | | Legibilidad | Alta (nombres significativos) | Menor (valores directos) | | Mantenibilidad | Alta (cambio en un solo lugar) | Menor (cambio en múltiples lugares)|

Conclusión

Entender y usar las constantes y los literales efectivamente puede mejorar enormemente la legibilidad y mantenibilidad de tus programas RPG. Las constantes proporcionan una forma de dar nombres significativos a valores fijos, haciendo tu código más fácil de entender y modificar. Los literales, por otro lado, son útiles para el uso rápido y directo de valores fijos en tu código. Al dominar estos conceptos, estarás mejor preparado para escribir programas RPG claros y eficientes.
