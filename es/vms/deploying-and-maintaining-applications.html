Introducción

Desplegar y mantener aplicaciones en OpenVMS requiere una comprensión sólida de las características y herramientas únicas del sistema operativo. Esta sección te guiará a través del proceso desde un nivel principiante hasta uno avanzado, cubriendo conceptos clave, ejemplos prácticos y mejores prácticas.

Entendiendo el Entorno OpenVMS

Antes de sumergirse en el despliegue, es esencial entender el entorno OpenVMS:

  • DCL (Digital Command Language): El lenguaje de comandos utilizado en OpenVMS.
  • Sistema de Archivos: OpenVMS utiliza un sistema de archivos jerárquico.
  • Cuentas de Usuario y Privilegios: Gestionar los permisos de usuario es crucial para un despliegue seguro.

Preparando la Aplicación para el Despliegue

Compilación y Enlace

  • Compilación: Convertir código fuente en código objeto.
  • Enlace: Combinar código objeto con bibliotecas para crear un ejecutable.

Ejemplo:

$ FORTRAN myprogram.for
$ LINK myprogram

Explicación:

  • FORTRAN myprogram.for compila el archivo fuente de Fortran.
  • LINK myprogram enlaza el archivo objeto compilado para crear un ejecutable.

Creación de Procedimientos de Comando

  • Procedimientos de Comando: Scripts escritos en DCL para automatizar tareas.
  • Ejemplo de Procedimiento de Comando:
$! Este es un procedimiento de comando de ejemplo
$ SET DEFAULT SYS$USER:[MYDIR]
$ RUN MYPROGRAM.EXE

Explicación:

  • SET DEFAULT SYS$USER:[MYDIR] cambia el directorio de trabajo.
  • RUN MYPROGRAM.EXE ejecuta el programa.

Desplegando la Aplicación

Copiando Archivos

  • Usando el Comando COPY: Transferir archivos al directorio de destino.
$ COPY MYPROGRAM.EXE SYS$USER:[TARGETDIR]

Explicación:

  • COPY MYPROGRAM.EXE SYS$USER:[TARGETDIR] copia el ejecutable al directorio de destino.

Configuración de Nombres Lógicos

  • Nombres Lógicos: Alias para especificaciones de archivos o dispositivos.
$ DEFINE MYAPP SYS$USER:[TARGETDIR]MYPROGRAM.EXE

Explicación:

  • DEFINE MYAPP SYS$USER:[TARGETDIR]MYPROGRAM.EXE crea un nombre lógico MYAPP para el ejecutable.

Gestión del Acceso de Usuarios

  • Concediendo Acceso: Usar la utilidad AUTHORIZE para gestionar permisos de usuario.
$ RUN SYS$SYSTEM:AUTHORIZE
UAF> ADD USERNAME /PASSWORD=secret /DEVICE=SYS$USER /DIRECTORY=[USERNAME]
UAF> EXIT

Explicación:

  • ADD USERNAME /PASSWORD=secret /DEVICE=SYS$USER /DIRECTORY=[USERNAME] crea un nuevo usuario con los atributos especificados.

Mantenimiento de la Aplicación

Monitoreo del Rendimiento

  • Usando la Utilidad MONITOR: Rastrear el rendimiento del sistema.
$ MONITOR SYSTEM

Explicación:

  • MONITOR SYSTEM muestra métricas de rendimiento del sistema.

Actualización de la Aplicación

  • Reemplazo de Ejecutables: Usar RENAME y COPY para actualizar archivos.
$ RENAME SYS$USER:[TARGETDIR]MYPROGRAM.EXE MYPROGRAM_OLD.EXE
$ COPY NEW_MYPROGRAM.EXE SYS$USER:[TARGETDIR]MYPROGRAM.EXE

Explicación:

  • RENAME renombra el ejecutable antiguo.
  • COPY coloca el nuevo ejecutable en el directorio de destino.

Respaldo y Recuperación

  • Usando la Utilidad BACKUP: Crear respaldos de archivos importantes.
$ BACKUP SYS$USER:[TARGETDIR]*.*;* SYS$USER:[BACKUPDIR]

Explicación:

  • BACKUP SYS$USER:[TARGETDIR]*.*;* SYS$USER:[BACKUPDIR] respalda todos los archivos en el directorio de destino al directorio de respaldo.

Conclusión

Desplegar y mantener aplicaciones en OpenVMS implica entender el entorno, preparar la aplicación, desplegarla correctamente y mantenerla mediante monitoreo, actualizaciones y respaldos. Siguiendo estos pasos y utilizando los ejemplos proporcionados, puedes asegurar un proceso de despliegue y mantenimiento fluido para tus aplicaciones en OpenVMS.
