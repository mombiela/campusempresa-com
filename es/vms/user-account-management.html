Introducción

La gestión de cuentas de usuario es un aspecto crítico de la administración del sistema en OpenVMS. Este tema te guiará a través de los aspectos esenciales de la creación, gestión y seguridad de las cuentas de usuario desde un nivel principiante hasta un nivel avanzado.

Nivel Principiante

Entendiendo las Cuentas de Usuario

  • Cuenta de Usuario: Una cuenta de usuario en OpenVMS es un identificador único que permite a una persona iniciar sesión y usar el sistema.
  • Nombre de Usuario: El nombre asignado a una cuenta de usuario.
  • Contraseña: Una palabra o frase secreta utilizada para autenticar al usuario.

Creación de una Cuenta de Usuario

Para crear una cuenta de usuario, necesitas usar la utilidad AUTHORIZE. A continuación, se muestra una guía paso a paso:

  1. Acceder a la Utilidad AUTHORIZE:

    $ RUN SYS$SYSTEM:AUTHORIZE

  2. Agregar un Nuevo Usuario:

    UAF> ADD nombredeusuario /PASSWORD=contraseña /UIC=[grupo,miembro] /DEVICE=dispositivo /DIRECTORY=directorio
    • nombredeusuario: El nombre del nuevo usuario.
    • contraseña: La contraseña inicial para el usuario.
    • UIC: Código de Identificación de Usuario, que es único para cada usuario.
    • dispositivo: El dispositivo de disco donde residirá el directorio del usuario.
    • directorio: La ruta del directorio para el usuario.

  3. Ejemplo:

    UAF> ADD JOHNDOE /PASSWORD=welcome123 /UIC=[200,100] /DEVICE=SYS$SYSDEVICE /DIRECTORY=[JOHNDOE]

Modificación de una Cuenta de Usuario

Para modificar una cuenta de usuario existente, usa el comando MODIFY en la utilidad AUTHORIZE:

UAF> MODIFY nombredeusuario /parameter=valor
  • Ejemplo: Cambiar la contraseña para el usuario JOHNDOE: 
    UAF> MODIFY JOHNDOE /PASSWORD=newpassword

Eliminación de una Cuenta de Usuario

Para eliminar una cuenta de usuario, usa el comando REMOVE:

UAF> REMOVE nombredeusuario
  • Ejemplo: Eliminar al usuario JOHNDOE: 
    UAF> REMOVE JOHNDOE

Nivel Intermedio

Atributos de la Cuenta de Usuario

  • Privilegios: Derechos especiales asignados a una cuenta de usuario para realizar tareas específicas.
  • Cuotas: Límites en los recursos que un usuario puede consumir.
  • Flags de Inicio de Sesión: Controlan varios aspectos del proceso de inicio de sesión.

Gestión de Privilegios

Para asignar o modificar privilegios, usa el comando MODIFY:

UAF> MODIFY nombredeusuario /PRIVILEGES=(priv1,priv2,...)
  • Ejemplo: Conceder privilegios SYSPRV y OPER a JOHNDOE: 
    UAF> MODIFY JOHNDOE /PRIVILEGES=(SYSPRV,OPER)

Establecimiento de Cuotas

Las cuotas controlan la cantidad de recursos del sistema que un usuario puede usar:

UAF> MODIFY nombredeusuario /WSQUOTA=valor /PGFLQUOTA=valor /BYTLM=valor
  • Ejemplo: Establecer cuotas para JOHNDOE: 
    UAF> MODIFY JOHNDOE /WSQUOTA=1024 /PGFLQUOTA=50000 /BYTLM=100000

Flags de Inicio de Sesión

Los flags de inicio de sesión pueden establecerse para controlar el comportamiento de inicio de sesión del usuario:

UAF> MODIFY nombredeusuario /FLAGS=(flag1,flag2,...)
  • Ejemplo: Establecer el flag DISUSER para deshabilitar la cuenta de JOHNDOE: 
    UAF> MODIFY JOHNDOE /FLAGS=DISUSER

Nivel Avanzado

Características Avanzadas de Seguridad

  • ACLs (Listas de Control de Acceso): Definen permisos de acceso detallados.
  • Auditoría: Monitorea y registra las actividades de los usuarios para propósitos de seguridad.

Gestión de ACLs

Para gestionar las ACLs, usa el comando SET ACL:

$ SET ACL /ACL=(IDENTIFIER=nombredeusuario,ACCESS=acceso) objeto
  • Ejemplo: Conceder a JOHNDOE acceso de lectura y escritura a un archivo: 
    $ SET ACL /ACL=(IDENTIFIER=JOHNDOE,ACCESS=READ+WRITE) MYFILE.TXT

Configuración de la Auditoría

Para habilitar la auditoría, modifica la configuración de auditoría del sistema:

  1. Editar la Configuración del Servidor de Auditoría:

    $ RUN SYS$SYSTEM:SYSMAN
SYSMAN> SET PROFILE /AUDIT

  2. Habilitar la Auditoría para Eventos Específicos:

    $ SET AUDIT /ENABLE=SECURITY /CLASS=LOGIN

Automatización de la Gestión de Cuentas de Usuario

  • Scripts DCL: Usa scripts de Digital Command Language (DCL) para automatizar tareas repetitivas.

  • Script de Ejemplo: Crear un script para agregar múltiples usuarios:

    $! Agregar múltiples usuarios
$ OPEN/READ userfile USERS.DAT
$ LOOP:
$   READ/END=EOF userfile line
$   nombredeusuario = F$ELEMENT(0, " ", line)
$   contraseña = F$ELEMENT(1, " ", line)
$   UIC = F$ELEMENT(2, " ", line)
$   dispositivo = F$ELEMENT(3, " ", line)
$   directorio = F$ELEMENT(4, " ", line)
$   RUN SYS$SYSTEM:AUTHORIZE
UAF> ADD 'nombredeusuario' /PASSWORD='contraseña' /UIC='UIC' /DEVICE='dispositivo' /DIRECTORY='directorio'
UAF> EXIT
$   GOTO LOOP
$ EOF:
$ CLOSE userfile
$ EXIT

Conclusión

La gestión de cuentas de usuario en OpenVMS implica la creación, modificación y eliminación de cuentas de usuario, la gestión de privilegios y cuotas, y la implementación de características avanzadas de seguridad como ACLs y auditoría. Al dominar estas tareas, puedes asegurar que tu sistema OpenVMS se mantenga seguro y eficiente.
