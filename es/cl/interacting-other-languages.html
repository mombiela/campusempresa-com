En esta sección, exploraremos cómo el Lenguaje de Control (CL) puede interactuar con otros lenguajes de programación. Esta es una habilidad crucial para aprovechar las fortalezas de diferentes lenguajes dentro de una sola aplicación o sistema.

Conceptos Clave

  • Interoperabilidad: La capacidad de CL para trabajar con otros lenguajes de programación.
  • Llamar Programas: Cómo llamar programas escritos en otros lenguajes desde un programa CL.
  • Intercambio de Datos: Métodos para intercambiar datos entre CL y otros lenguajes.
  • Manejo de Errores: Gestión de errores al interactuar con otros lenguajes.

Llamar Programas desde CL

Una de las principales formas en que CL interactúa con otros lenguajes es llamando programas escritos en esos lenguajes. Esto a menudo se hace usando el comando CALL.

Ejemplo: Llamar un Programa RPG

PGM
    DCL VAR(&RESULT) TYPE(*CHAR) LEN(10)
    CALL PGM(MYLIB/MYPGM) PARM(&RESULT)
    /* Procesar el resultado */
    SNDPGMMSG MSG('Resultado del programa RPG: ' *CAT &RESULT)
ENDPGM
  • Explicación:
    • DCL VAR(&RESULT) TYPE(*CHAR) LEN(10): Declara una variable para almacenar el resultado.
    • CALL PGM(MYLIB/MYPGM) PARM(&RESULT): Llama a un programa RPG y pasa la variable &RESULT.
    • SNDPGMMSG MSG('Resultado del programa RPG: ' *CAT &RESULT): Envía un mensaje de programa con el resultado.

Ejemplo: Llamar un Programa COBOL

PGM
    DCL VAR(&STATUS) TYPE(*CHAR) LEN(1)
    CALL PGM(MYLIB/COBOLPGM) PARM(&STATUS)
    IF COND(&STATUS *EQ '0') THEN(DO)
        SNDPGMMSG MSG('El programa COBOL se completó con éxito.')
    ENDDO
    ELSE DO
        SNDPGMMSG MSG('El programa COBOL falló.')
    ENDDO
ENDPGM
  • Explicación:
    • DCL VAR(&STATUS) TYPE(*CHAR) LEN(1): Declara una variable para almacenar el estado.
    • CALL PGM(MYLIB/COBOLPGM) PARM(&STATUS): Llama a un programa COBOL y pasa la variable &STATUS.
    • IF COND(&STATUS *EQ '0') THEN(DO): Verifica si el estado es '0' (éxito).
    • SNDPGMMSG MSG('El programa COBOL se completó con éxito.'): Envía un mensaje de éxito.
    • ELSE DO: Maneja el caso de fallo.
    • SNDPGMMSG MSG('El programa COBOL falló.'): Envía un mensaje de fallo.

Intercambio de Datos

El intercambio de datos entre CL y otros lenguajes se puede hacer a través de parámetros, áreas de datos o archivos.

Usando Parámetros

Los parámetros son la forma más sencilla de intercambiar datos.

PGM
    DCL VAR(&INPUT) TYPE(*CHAR) LEN(20)
    DCL VAR(&OUTPUT) TYPE(*CHAR) LEN(20)
    CHGVAR VAR(&INPUT) VALUE('¡Hola, Mundo!')
    CALL PGM(MYLIB/OTHERPGM) PARM(&INPUT &OUTPUT)
    SNDPGMMSG MSG('Salida del otro programa: ' *CAT &OUTPUT)
ENDPGM
  • Explicación:
    • DCL VAR(&INPUT) TYPE(*CHAR) LEN(20): Declara una variable de entrada.
    • DCL VAR(&OUTPUT) TYPE(*CHAR) LEN(20): Declara una variable de salida.
    • CHGVAR VAR(&INPUT) VALUE('¡Hola, Mundo!'): Establece la variable de entrada.
    • CALL PGM(MYLIB/OTHERPGM) PARM(&INPUT &OUTPUT): Llama a otro programa con parámetros de entrada y salida.
    • SNDPGMMSG MSG('Salida del otro programa: ' *CAT &OUTPUT): Envía un mensaje con la salida.

Usando Áreas de Datos

Las áreas de datos se pueden usar para un intercambio de datos más persistente.

PGM
    DCL VAR(&DATA) TYPE(*CHAR) LEN(50)
    RTVDTAARA DTAARA(MYLIB/MYDATA) RTNVAR(&DATA)
    CALL PGM(MYLIB/OTHERPGM)
    CHGDTAARA DTAARA(MYLIB/MYDATA) VALUE(&DATA)
ENDPGM
  • Explicación:
    • DCL VAR(&DATA) TYPE(*CHAR) LEN(50): Declara una variable para el contenido del área de datos.
    • RTVDTAARA DTAARA(MYLIB/MYDATA) RTNVAR(&DATA): Recupera datos de un área de datos.
    • CALL PGM(MYLIB/OTHERPGM): Llama a otro programa.
    • CHGDTAARA DTAARA(MYLIB/MYDATA) VALUE(&DATA): Cambia el contenido del área de datos.

Manejo de Errores

El manejo adecuado de errores es esencial al interactuar con otros lenguajes para asegurar la robustez.

PGM
    DCL VAR(&STATUS) TYPE(*CHAR) LEN(1)
    MONMSG MSGID(CPF0000) EXEC(DO)
        SNDPGMMSG MSG('Ocurrió un error al llamar al programa.')
        RETURN
    ENDDO
    CALL PGM(MYLIB/OTHERPGM) PARM(&STATUS)
    IF COND(&STATUS *NE '0') THEN(DO)
        SNDPGMMSG MSG('El otro programa devolvió un error.')
    ENDDO
ENDPGM
  • Explicación:
    • MONMSG MSGID(CPF0000) EXEC(DO): Monitorea cualquier mensaje de error.
    • SNDPGMMSG MSG('Ocurrió un error al llamar al programa.'): Envía un mensaje de error.
    • RETURN: Sale del programa.
    • IF COND(&STATUS *NE '0') THEN(DO): Verifica si el estado indica un error.
    • SNDPGMMSG MSG('El otro programa devolvió un error.'): Envía un mensaje de error.

Conclusión

Interactuar con otros lenguajes desde CL es una característica poderosa que te permite aprovechar las fortalezas de diferentes entornos de programación. Al entender cómo llamar programas, intercambiar datos y manejar errores, puedes crear aplicaciones robustas y flexibles. Practica estas técnicas con varios lenguajes para volverte competente en la interoperabilidad multilenguaje.
