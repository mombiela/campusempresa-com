Introducción al CL (Lenguaje de Control)

El Lenguaje de Control (CL) es un lenguaje de scripting utilizado en los sistemas AS/400 de IBM (ahora IBM i). Está diseñado para controlar operaciones del sistema, automatizar tareas y gestionar recursos del sistema. Este tema te guiará desde los conceptos básicos de CL hasta técnicas avanzadas de automatización.

Empezando con CL

Antes de sumergirse en la automatización, es esencial entender los conceptos básicos de CL.

Sintaxis Básica y Comandos

  • Comandos: Los comandos CL son los bloques de construcción de los scripts. Cada comando realiza una tarea específica.
  • Parámetros: Los comandos a menudo tienen parámetros que especifican detalles sobre cómo debe ejecutarse el comando.

Ejemplo: Mostrando un Mensaje

DSPMSG MSGQ(*SYSOPR)

Este comando muestra mensajes de la cola de mensajes del operador del sistema.

Creando un Programa CL Simple

Un programa CL es una secuencia de comandos CL guardados en un miembro fuente.

Ejemplo: Programa Hola Mundo

PGM
    DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
    CHGVAR VAR(&MSG) VALUE('¡Hola, Mundo!')
    SNDPGMMSG MSG(&MSG)
ENDPGM
  • PGM y ENDPGM marcan el inicio y el final del programa.
  • DCL declara una variable.
  • CHGVAR cambia el valor de una variable.
  • SNDPGMMSG envía un mensaje del programa.

Automatización de Tareas Rutinarias

La automatización en CL implica crear scripts que realicen tareas repetitivas sin intervención manual.

Programación de Trabajos

Los trabajos pueden programarse para ejecutarse en momentos específicos utilizando el comando SBMJOB.

Ejemplo: Enviando un Trabajo

SBMJOB CMD(CALL PGM(MYPGM)) JOB(MYJOB) SCDDATE('2023-10-01') SCDTIME('12:00:00')
  • CMD especifica el comando a ejecutar.
  • JOB nombra el trabajo.
  • SCDDATE y SCDTIME programan el trabajo.

Bucles y Lógica Condicional

Los bucles y las declaraciones condicionales permiten una automatización más compleja.

Ejemplo: Recorriendo Archivos

PGM
    DCLF FILE(QTEMP/MYFILE)
    DCL VAR(&EOF) TYPE(*LGL)
    DCL VAR(&COUNT) TYPE(*DEC) LEN(5 0)

    CHGVAR VAR(&COUNT) VALUE(0)
    OVRDBF FILE(QTEMP/MYFILE) TOFILE(MYLIB/MYFILE)

    DOWHILE COND(&EOF *EQ *OFF)
        RCVF
        MONMSG MSGID(CPF0864) EXEC(CHGVAR VAR(&EOF) VALUE(*ON))
        IF COND(&EOF *EQ *OFF) THEN(DO)
            CHGVAR VAR(&COUNT) VALUE(&COUNT + 1)
        ENDDO
    ENDDO

    SNDPGMMSG MSG('Total de registros: ' *CAT &COUNT)
ENDPGM
  • DCLF declara un archivo para lectura.
  • RCVF recibe un registro del archivo.
  • MONMSG monitorea mensajes específicos (por ejemplo, fin de archivo).
  • DOWHILE crea un bucle que continúa hasta que se cumple una condición.

Técnicas Avanzadas de Automatización

Las técnicas avanzadas implican integrar CL con otras herramientas del sistema y manejar errores de manera efectiva.

Manejo de Errores

El manejo adecuado de errores asegura que tus scripts puedan recuperarse de problemas inesperados.

Ejemplo: Manejo de Errores con MONMSG

PGM
    DCL VAR(&MSG) TYPE(*CHAR) LEN(50)

    CHGVAR VAR(&MSG) VALUE('Iniciando trabajo...')
    SNDPGMMSG MSG(&MSG)

    CALL PGM(MYPGM)
    MONMSG MSGID(CPF0000) EXEC(DO)
        CHGVAR VAR(&MSG) VALUE('Ocurrió un error.')
        SNDPGMMSG MSG(&MSG)
    ENDDO

    CHGVAR VAR(&MSG) VALUE('Trabajo completado.')
    SNDPGMMSG MSG(&MSG)
ENDPGM
  • MONMSG captura cualquier mensaje de error y ejecuta un bloque de recuperación.

Integración con Otras Herramientas

CL puede interactuar con otras herramientas del sistema como SQL y Qshell para una automatización más poderosa.

Ejemplo: Ejecutando SQL desde CL

PGM
    RUNSQL SQL('UPDATE MYLIB.MYTABLE SET STATUS = ''COMPLETE'' WHERE STATUS = ''PENDING''')
ENDPGM
  • RUNSQL ejecuta una declaración SQL desde dentro de un programa CL.

Conclusión

Automatizar tareas del sistema con CL puede mejorar significativamente la eficiencia y reducir la carga de trabajo manual. Al dominar los conceptos básicos, aprender a programar trabajos, implementar bucles y lógica condicional, manejar errores e integrar con otras herramientas, puedes crear scripts de automatización poderosos. Practica estas técnicas para volverte competente en la automatización de tareas en sistemas IBM i.
