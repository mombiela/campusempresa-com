Visión General

El Lenguaje de Control (CL) es un lenguaje de scripting utilizado en los sistemas de gama media de IBM, como el AS/400 (ahora IBM i). Los comandos CL se utilizan para controlar las operaciones del sistema, gestionar trabajos y manejar datos. Esta sección te introducirá a los conceptos básicos de los comandos CL, su estructura y cómo usarlos de manera efectiva.

Conceptos Clave

  • Estructura del Comando: Los comandos CL siguen una sintaxis y estructura específicas.
  • Parámetros: Los comandos a menudo requieren parámetros para especificar detalles.
  • Ejecución: Cómo ejecutar comandos CL en diferentes entornos.
  • Manejo de Errores: Manejo básico de errores en comandos CL.

Estructura del Comando

Los comandos CL están estructurados de una manera específica para asegurar claridad y consistencia. Un comando CL típico se ve así:

COMMAND PARAMETER1(VALUE1) PARAMETER2(VALUE2)

Ejemplo

CRTUSRPRF USRPRF(USER1) PASSWORD(PASS123)
  • CRTUSRPRF: Comando para crear un perfil de usuario.
  • USRPRF: Parámetro que especifica el nombre del perfil de usuario.
  • PASSWORD: Parámetro que especifica la contraseña para el perfil de usuario.

Comandos CL Comúnmente Usados

Aquí hay algunos comandos CL comúnmente usados y sus propósitos:

| Comando | Descripción | |-------------|---------------------------------------------| | CRTUSRPRF | Crear un perfil de usuario | | DLTUSRPRF | Eliminar un perfil de usuario | | WRKJOB | Trabajar con trabajos | | DSPMSG | Mostrar mensajes | | SBMJOB | Enviar un trabajo | | ENDJOB | Terminar un trabajo | | CHGJOB | Cambiar atributos del trabajo | | SNDMSG | Enviar un mensaje |

Ejemplo: Creando un Perfil de Usuario

CRTUSRPRF USRPRF(USER1) PASSWORD(PASS123) TEXT('Nuevo Perfil de Usuario')
  • TEXT: Un parámetro opcional para agregar una descripción.

Parámetros

Los parámetros son componentes esenciales de los comandos CL. Proporcionan los detalles necesarios para que el comando se ejecute correctamente.

Ejemplo: Enviando un Trabajo

SBMJOB CMD(CALL PGM(MYPGM)) JOB(MYJOB) JOBQ(QBATCH)
  • CMD: Especifica el comando a ejecutar.
  • JOB: Nombra el trabajo.
  • JOBQ: Especifica la cola de trabajos.

Ejecución

Los comandos CL pueden ejecutarse en varios entornos:

  • Línea de Comandos: Introducir directamente los comandos en la interfaz de línea de comandos.
  • Programas CL: Incrustar comandos dentro de programas CL para procesamiento por lotes.
  • Programador de Trabajos: Programar comandos para que se ejecuten en momentos específicos.

Ejemplo: Ejecutando un Comando en un Programa CL

PGM
    CRTUSRPRF USRPRF(USER1) PASSWORD(PASS123)
    SBMJOB CMD(CALL PGM(MYPGM)) JOB(MYJOB) JOBQ(QBATCH)
ENDPGM

Manejo de Errores

El manejo de errores en los comandos CL es crucial para una ejecución robusta de scripts. Las técnicas comunes incluyen:

  • Monitor de Mensajes (MONMSG): Capturar y manejar errores.
  • Manejo de Condiciones: Usar condiciones para gestionar diferentes escenarios.

Ejemplo: Usando MONMSG

PGM
    CRTUSRPRF USRPRF(USER1) PASSWORD(PASS123)
    MONMSG MSGID(CPF0000) EXEC(DO)
        /* Código de manejo de errores */
    ENDDO
ENDPGM
  • MONMSG MSGID(CPF0000): Captura cualquier mensaje de error que comience con CPF.

Conclusión

Esta introducción a los comandos CL proporciona una base para entender cómo usar CL para gestionar y controlar los sistemas IBM i. Al dominar la estructura, los parámetros, los métodos de ejecución y las técnicas de manejo de errores, puedes automatizar y optimizar eficazmente las operaciones del sistema. A medida que avances en este curso, profundizarás en temas más avanzados y escenarios complejos, mejorando tu competencia en comandos CL.
