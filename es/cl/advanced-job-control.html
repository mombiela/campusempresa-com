Introducción

En esta sección, profundizaremos en técnicas avanzadas de control de trabajos en el Lenguaje de Control (CL). Esto incluye la gestión de colas de trabajos, el control de prioridades de trabajos y el manejo de registros de trabajos. Al dominar estos conceptos, podrás optimizar el procesamiento de trabajos y asegurar un rendimiento eficiente del sistema.

Conceptos clave

  • Colas de trabajos: Comprender cómo gestionar y manipular las colas de trabajos.
  • Prioridades de trabajos: Establecer y ajustar las prioridades de trabajos para controlar el orden de ejecución.
  • Registros de trabajos: Analizar los registros de trabajos para la resolución de problemas y la optimización del rendimiento.
  • Programación de trabajos: Automatizar la ejecución de trabajos utilizando técnicas de programación.

Gestión de colas de trabajos

Las colas de trabajos son esenciales para organizar y controlar la ejecución de trabajos. Puedes crear, eliminar y gestionar colas de trabajos utilizando varios comandos CL.

Creación de una cola de trabajos

Para crear una cola de trabajos, utiliza el comando CRTJOBQ:

CRTJOBQ JOBQ(MYJOBQ) TEXT('Mi cola de trabajos personalizada')
  • JOBQ(MYJOBQ): Especifica el nombre de la cola de trabajos.
  • TEXT('Mi cola de trabajos personalizada'): Proporciona una descripción para la cola de trabajos.

Eliminación de una cola de trabajos

Para eliminar una cola de trabajos, utiliza el comando DLTJOBQ:

DLTJOBQ JOBQ(MYJOBQ)
  • JOBQ(MYJOBQ): Especifica el nombre de la cola de trabajos a eliminar.

Visualización de colas de trabajos

Para mostrar el estado y los detalles de las colas de trabajos, utiliza el comando WRKJOBQ:

WRKJOBQ JOBQ(*ALL)
  • JOBQ(*ALL): Muestra todas las colas de trabajos.

Control de prioridades de trabajos

Las prioridades de trabajos determinan el orden en el que se ejecutan los trabajos. Los trabajos de mayor prioridad se procesan antes que los de menor prioridad.

Establecimiento de la prioridad de un trabajo

Para establecer la prioridad de un trabajo, utiliza el comando SBMJOB con el parámetro JOBPTY:

SBMJOB CMD(CALL PGM(MYPGM)) JOB(MYJOB) JOBPTY(5)
  • CMD(CALL PGM(MYPGM)): Especifica el comando para ejecutar el trabajo.
  • JOB(MYJOB): Nombra el trabajo.
  • JOBPTY(5): Establece la prioridad del trabajo (1 es la más alta, 9 es la más baja).

Cambio de la prioridad de un trabajo

Para cambiar la prioridad de un trabajo existente, utiliza el comando CHGJOB:

CHGJOB JOB(MYJOB) JOBPTY(3)
  • JOB(MYJOB): Especifica el trabajo a cambiar.
  • JOBPTY(3): Establece la nueva prioridad del trabajo.

Manejo de registros de trabajos

Los registros de trabajos proporcionan información detallada sobre la ejecución de trabajos, lo cual es crucial para la resolución de problemas y el análisis del rendimiento.

Visualización de registros de trabajos

Para mostrar un registro de trabajo, utiliza el comando DSPJOBLOG:

DSPJOBLOG JOB(MYJOB)
  • JOB(MYJOB): Especifica el trabajo cuyo registro deseas mostrar.

Impresión de registros de trabajos

Para imprimir un registro de trabajo, utiliza el comando PRTJOBLOG:

PRTJOBLOG JOB(MYJOB)
  • JOB(MYJOB): Especifica el trabajo cuyo registro deseas imprimir.

Programación de trabajos

La automatización de la ejecución de trabajos puede mejorar significativamente la eficiencia. Puedes programar trabajos utilizando el comando ADDJOBSCDE.

Adición de una entrada de programación de trabajos

Para añadir una entrada de programación de trabajos, utiliza el comando ADDJOBSCDE:

ADDJOBSCDE JOB(MYJOB) CMD(CALL PGM(MYPGM)) FRQ(*WEEKLY) SCDDATE(*NONE) SCDTIME(120000)
  • JOB(MYJOB): Nombra el trabajo.
  • CMD(CALL PGM(MYPGM)): Especifica el comando para ejecutar el trabajo.
  • FRQ(*WEEKLY): Establece la frecuencia a semanal.
  • SCDDATE(*NONE): Indica que no hay una fecha de inicio específica.
  • SCDTIME(120000): Establece la hora programada a las 12:00 PM.

Eliminación de una entrada de programación de trabajos

Para eliminar una entrada de programación de trabajos, utiliza el comando RMVJOBSCDE:

RMVJOBSCDE JOB(MYJOB)
  • JOB(MYJOB): Especifica el trabajo a eliminar de la programación.

Conclusión

El control avanzado de trabajos en CL implica la gestión de colas de trabajos, el establecimiento de prioridades de trabajos, el análisis de registros de trabajos y la programación de trabajos. Al dominar estas técnicas, podrás optimizar el procesamiento de trabajos y asegurar un rendimiento eficiente del sistema. Practica estos comandos y explora sus parámetros para obtener una comprensión más profunda y control sobre la gestión de trabajos en tu entorno CL.
