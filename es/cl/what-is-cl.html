El Lenguaje de Control (CL) es un lenguaje de scripting utilizado principalmente en los sistemas de gama media de IBM, como el AS/400 (ahora conocido como IBM i). Está diseñado para controlar las operaciones del sistema, gestionar trabajos y manejar recursos del sistema. CL es esencial para los administradores de sistemas y desarrolladores que trabajan en entornos IBM.

Conceptos Clave del Lenguaje de Control (CL)

  • Propósito: CL se utiliza para automatizar y controlar tareas del sistema, como la programación de trabajos, la gestión de archivos y la configuración del sistema.
  • Entorno: Los scripts de CL se ejecutan en el entorno del sistema operativo IBM i.
  • Sintaxis: CL tiene una sintaxis única que incluye comandos, parámetros y variables.

Estructura Básica de los Programas CL

Un programa CL típico consiste en una serie de comandos que realizan tareas específicas. Cada comando se escribe en una nueva línea y puede incluir parámetros para personalizar su comportamiento.

Ejemplo de un Programa CL Simple

PGM
    DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
    CHGVAR VAR(&MSG) VALUE('¡Hola, Mundo!')
    SNDPGMMSG MSG(&MSG)
ENDPGM
  • PGM: Indica el inicio de un programa CL.
  • DCL: Declara una variable. En este caso, &MSG es una variable de tipo carácter con una longitud de 50.
  • CHGVAR: Cambia el valor de una variable. Aquí, &MSG se establece en '¡Hola, Mundo!'.
  • SNDPGMMSG: Envía un mensaje de programa. El mensaje enviado es el valor de &MSG.
  • ENDPGM: Marca el final del programa CL.

Comandos Comunes de CL

  • DCL: Declarar una variable.
  • CHGVAR: Cambiar el valor de una variable.
  • SNDPGMMSG: Enviar un mensaje de programa.
  • CALL: Llamar a otro programa.
  • IF/ELSE: Ejecución condicional de comandos.
  • DO/ENDDO: Constructos de bucle.

Ejemplo de Ejecución Condicional

PGM
    DCL VAR(&STATUS) TYPE(*CHAR) LEN(10)
    CHGVAR VAR(&STATUS) VALUE('ACTIVO')

    IF COND(&STATUS *EQ 'ACTIVO') THEN(DO)
        SNDPGMMSG MSG('El sistema está activo')
    ENDDO
    ELSE DO
        SNDPGMMSG MSG('El sistema no está activo')
    ENDDO
ENDPGM
  • IF COND: Verifica si la condición es verdadera.
  • THEN(DO): Ejecuta el bloque de comandos si la condición es verdadera.
  • ELSE DO: Ejecuta el bloque de comandos si la condición es falsa.

Características Avanzadas

A medida que avances, encontrarás características más avanzadas de CL, tales como:

  • Subrutinas: Encapsulan bloques de código reutilizables.
  • Manejo de Errores: Gestiona errores y excepciones en los programas CL.
  • Operaciones con Archivos: Leer y escribir en archivos.
  • Control de Trabajos: Gestiona colas de trabajos y atributos de trabajos.

Ejemplo de una Subrutina

PGM
    DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
    CALLSUBR SUBR(MYSUBR)
    SNDPGMMSG MSG(&MSG)
    ENDPGM

SUBR MYSUBR
    CHGVAR VAR(&MSG) VALUE('¡Hola desde la subrutina!')
    RETURN
ENDSUBR
  • CALLSUBR: Llama a una subrutina.
  • SUBR: Define una subrutina.
  • RETURN: Devuelve el control al programa que llama.

Resumen

El Lenguaje de Control (CL) es una herramienta poderosa para gestionar sistemas IBM i. Permite la automatización de tareas del sistema, el control de trabajos y la gestión de recursos. Comprender la estructura básica, los comandos comunes y las características avanzadas de CL es esencial para una administración y desarrollo efectivos del sistema en entornos IBM.

Al dominar CL, puedes crear procesos eficientes y automatizados que mejoran la funcionalidad y la fiabilidad de tus sistemas IBM i.
