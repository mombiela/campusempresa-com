Introducción a la Depuración en CL

La depuración es una habilidad esencial para cualquier programador. En Control Language (CL), entender cómo identificar y corregir errores puede ahorrarte mucho tiempo y frustración. Esta sección cubrirá varias técnicas de depuración desde niveles principiantes hasta avanzados.

Técnicas Básicas de Depuración

Entendiendo los Mensajes de Error

  • Leer los Mensajes de Error Cuidadosamente: Los mensajes de error proporcionan pistas sobre lo que salió mal. Presta atención al texto del mensaje y a cualquier código proporcionado.
  • Códigos de Error Comunes: Familiarízate con los códigos de error comunes en CL.

Uso de Comandos de Visualización

  • DSPJOBLOG: Usa el comando DSPJOBLOG para mostrar el registro del trabajo, que contiene información detallada sobre la ejecución del trabajo.
  • DSPMSG: El comando DSPMSG muestra mensajes enviados a una cola de mensajes, lo que puede ayudar a identificar problemas.

Ejemplo: Uso de DSPJOBLOG

DSPJOBLOG JOB(123456/USER/JOBNAME)

Explicación: Este comando muestra el registro del trabajo para el trabajo especificado, ayudándote a identificar dónde ocurrió el error.

Técnicas Intermedias de Depuración

Uso de Puntos de Interrupción

  • STRDBG: El comando STRDBG inicia una sesión de depuración.
  • ADDBKP: Usa el comando ADDBKP para agregar puntos de interrupción en tu programa CL.

Ejemplo: Configuración de un Punto de Interrupción

STRDBG PGM(MYPGM)
ADDBKP STMT(10)

Explicación: Esto inicia una sesión de depuración para el programa MYPGM y agrega un punto de interrupción en la declaración número 10.

Análisis de Valores de Variables

  • DSPPGMVAR: El comando DSPPGMVAR muestra los valores de las variables en el programa.
  • CHGVAR: Usa el comando CHGVAR para cambiar el valor de una variable durante la depuración.

Ejemplo: Mostrar y Cambiar Valores de Variables

DSPPGMVAR PGM(MYPGM) VAR(&MYVAR)
CHGVAR VAR(&MYVAR) VALUE('NEWVALUE')

Explicación: Esto muestra el valor de la variable &MYVAR en el programa MYPGM y cambia su valor a 'NEWVALUE'.

Técnicas Avanzadas de Depuración

Uso de Comandos de Rastreo

  • TRCJOB: El comando TRCJOB rastrea la ejecución del trabajo, proporcionando información detallada sobre cada paso.
  • ENDTRC: Usa el comando ENDTRC para finalizar la sesión de rastreo.

Ejemplo: Rastrear un Trabajo

TRCJOB JOB(123456/USER/JOBNAME)
ENDTRC JOB(123456/USER/JOBNAME)

Explicación: Esto inicia y finaliza una sesión de rastreo para el trabajo especificado, permitiéndote analizar su ejecución en detalle.

Depuración de Trabajos en Lote

  • SBMJOB: Envía un trabajo con la depuración habilitada usando el comando SBMJOB.
  • WRKSBMJOB: Usa el comando WRKSBMJOB para trabajar con trabajos enviados y monitorear su estado.

Ejemplo: Enviar un Trabajo con Depuración

SBMJOB CMD(CALL PGM(MYPGM)) JOB(MYJOB) JOBD(MYJOBD) JOBQ(MYJOBQ) LOG(4 00 *SECLVL)
WRKSBMJOB JOB(MYJOB)

Explicación: Esto envía un trabajo con la depuración habilitada y te permite monitorear su estado usando el comando WRKSBMJOB.

Conclusión

La depuración en CL implica una variedad de técnicas desde leer mensajes de error y usar comandos de visualización hasta configurar puntos de interrupción y rastrear la ejecución de trabajos. Al dominar estas técnicas, puedes identificar y resolver eficientemente problemas en tus programas CL, mejorando tu productividad y la fiabilidad de tu código.
