Variables y Constantes

Las variables y constantes son los bloques de construcción de cualquier programa en Swift.

var variableName = "¡Hola, Swift!"
let constantName = "¡Hola, Mundo!"
  • var se usa para declarar una variable que puede cambiar.
  • let se usa para declarar una constante que no puede cambiar.

Tipos de Datos

Swift tiene varios tipos de datos, incluyendo Int, Double, String y Bool.

let age: Int = 25
let pi: Double = 3.14159
let name: String = "Alice"
let isSwiftFun: Bool = true

Flujo de Control

El flujo de control incluye declaraciones condicionales y bucles.

if isSwiftFun {
    print("¡Swift es divertido!")
} else {
    print("Swift no es divertido.")
}

for i in 1...5 {
    print(i)
}

Funciones

Las funciones son bloques de código reutilizables.

func greet(name: String) -> String {
    return "¡Hola, \(name)!"
}

let greeting = greet(name: "Alice")
print(greeting)

Closures

Los closures son bloques de funcionalidad autónomos que se pueden pasar y usar en tu código.

let closure = { (name: String) -> String in
    return "¡Hola, \(name)!"
}

print(closure("Bob"))

Programación Orientada a Objetos (OOP)

La OOP implica la creación de clases y objetos.

class Person {
    var name: String
    var age: Int
    
    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }
    
    func greet() -> String {
        return "Hola, mi nombre es \(name)."
    }
}

let person = Person(name: "Alice", age: 25)
print(person.greet())

Protocolos y Extensiones

Los protocolos definen un plano de métodos, propiedades y otros requisitos.

protocol Greetable {
    func greet() -> String
}

extension Person: Greetable {}

Manejo de Errores

Swift proporciona un manejo de errores robusto usando do, try y catch.

enum CustomError: Error {
    case invalidInput
}

func checkInput(input: Int) throws {
    if input < 0 {
        throw CustomError.invalidInput
    }
}

do {
    try checkInput(input: -1)

Concurrencia

La concurrencia en Swift se gestiona usando Grand Central Dispatch (GCD) y Operation Queues.

DispatchQueue.global().async {
    // Tarea en segundo plano
    DispatchQueue.main.async {
        // Actualizar la UI en el hilo principal
    }
}

Proyecto Ejemplo: Aplicación de Lista de Tareas

Aquí hay un ejemplo simple de cómo podría ser tu proyecto final:

import UIKit

class ToDoItem {
    var title: String
    var isCompleted: Bool
    
    init(title: String, isCompleted: Bool = false) {
        self.title = title
        self.isCompleted = isCompleted
    }
}

class ToDoList {
    var items: [ToDoItem] = []
    
    func addItem(title: String) {
        let newItem = ToDoItem(title: title)
        items.append(newItem)
    }
    
    func completeItem(at index: Int) {
        items[index].isCompleted = true
    }
}

let toDoList = ToDoList()
toDoList.addItem(title: "Terminar el curso de Swift")
toDoList.completeItem(at: 0)

for item in toDoList.items {
    print("\(item.title) - Completado: \(item.isCompleted)")
}
