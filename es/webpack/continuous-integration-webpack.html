La Integración Continua (CI) es una práctica de desarrollo donde los desarrolladores integran código en un repositorio compartido con frecuencia, preferiblemente varias veces al día. Cada integración puede ser verificada mediante una compilación automatizada y pruebas automatizadas. Webpack, un popular empaquetador de módulos, puede integrarse en pipelines de CI para automatizar el proceso de compilación, asegurando que la base de código esté siempre en un estado desplegable.

Conceptos Clave de la Integración Continua

  • Compilaciones Automatizadas: Compilar y empaquetar código automáticamente cada vez que se realizan cambios.
  • Pruebas Automatizadas: Ejecutar pruebas automáticamente para asegurar la calidad del código.
  • Integración con Control de Versiones: Usar sistemas como Git para gestionar los cambios de código.
  • Servidores de Compilación: Herramientas como Jenkins, Travis CI o GitHub Actions para automatizar el proceso de compilación.

Configuración de Webpack para Integración Continua

Prerrequisitos

Antes de integrar Webpack en una pipeline de CI, asegúrate de tener lo siguiente:

  • Un proyecto con Webpack configurado.
  • Un sistema de control de versiones (por ejemplo, Git).
  • Un servicio de CI (por ejemplo, Jenkins, Travis CI, GitHub Actions).

Ejemplo de Estructura de Proyecto

mi-proyecto-webpack/
├── src/
│   └── index.js
├── dist/
├── webpack.config.js
├── package.json
└── .gitignore

Configuración de Webpack

Asegúrate de que tu webpack.config.js esté configurado correctamente tanto para entornos de desarrollo como de producción.

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  mode: 'production', // Cambiar a 'development' para compilaciones de desarrollo
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      }
    ]
  }
};

Agregar Scripts a package.json

Agrega scripts a tu package.json para facilitar el proceso de compilación.

{
  "scripts": {
    "build": "webpack --config webpack.config.js",
    "test": "echo \"Error: no test specified\" && exit 1"
  }
}

Integración de Webpack con Servicios de CI

GitHub Actions

GitHub Actions es un popular servicio de CI que se integra perfectamente con los repositorios de GitHub.

  1. Crear un Archivo de Workflow: Agrega un archivo .github/workflows/ci.yml a tu proyecto.
name: CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Build project
      run: npm run build

    - name: Run tests
      run: npm test

Travis CI

Travis CI es otro popular servicio de CI que es fácil de configurar.

  1. Crear un archivo .travis.yml en la raíz de tu proyecto.
language: node_js
node_js:
  - "14"
script:
  - npm install
  - npm run build
  - npm test

Jenkins

Jenkins es un servidor de automatización de código abierto ampliamente utilizado.

  1. Instalar Jenkins: Sigue las instrucciones de instalación en el sitio web de Jenkins.
  2. Crear un Nuevo Trabajo: Configura un nuevo proyecto Freestyle.
  3. Configurar Gestión de Código Fuente: Apunta Jenkins a tu repositorio Git.
  4. Agregar Pasos de Compilación:
    • Ejecutar Shell: Agrega un script de shell para instalar dependencias y compilar el proyecto.
npm install
npm run build
npm test

Conclusión

Integrar Webpack con servicios de Integración Continua asegura que tu base de código esté siempre en un estado desplegable. Al automatizar los procesos de compilación y prueba, puedes detectar errores temprano y mantener un alto nivel de calidad del código. Ya sea que uses GitHub Actions, Travis CI o Jenkins, los principios son los mismos: automatizar, probar y desplegar. Esta integración no solo ahorra tiempo, sino que también mejora la fiabilidad de tu ciclo de vida de desarrollo de software.
