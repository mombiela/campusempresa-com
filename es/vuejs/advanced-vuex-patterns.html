Introducción

En esta sección, exploraremos patrones avanzados en Vuex, la biblioteca de gestión de estado para Vue.js. Estos patrones te ayudarán a manejar lógica de estado compleja, mejorar el mantenimiento del código y aumentar la escalabilidad de tus aplicaciones Vue.js.

Conceptos Clave

  • Módulos: Organizar el estado, mutaciones, acciones y getters en módulos separados.
  • Namespaces: Usar namespaces para evitar conflictos de nombres en aplicaciones grandes.
  • Módulos Dinámicos: Registrar y desregistrar módulos dinámicamente.
  • API de Composición con Vuex: Integrar Vuex con la API de Composición para una mejor reutilización.
  • Plugins: Extender la funcionalidad de Vuex con plugins.
  • Pruebas de Vuex: Escribir pruebas unitarias para la tienda de Vuex.

Módulos

Los módulos te permiten dividir tu tienda en múltiples tiendas más pequeñas, cada una gestionando su propio estado, mutaciones, acciones y getters.

Ejemplo

// store/modules/user.js
const state = {
  name: '',
  age: null,
};

const mutations = {
  SET_USER(state, user) {
    state.name = user.name;
    state.age = user.age;
  },
};

const actions = {
  updateUser({ commit }, user) {
    commit('SET_USER', user);
  },
};

const getters = {
  userName: (state) => state.name,
  userAge: (state) => state.age,
};

export default {
  state,
  mutations,
  actions,
  getters,
};

Registro de Módulos

// store/index.js
import Vue from 'vue';
import Vuex from 'vuex';
import user from './modules/user';

Vue.use(Vuex);

export default new Vuex.Store({
  modules: {
    user,
  },
});

Namespaces

Los namespaces ayudan a evitar conflictos de nombres encapsulando el estado, mutaciones, acciones y getters del módulo.

Ejemplo

// store/modules/user.js
export default {
  namespaced: true,
  state: { ... },
  mutations: { ... },
  actions: { ... },
  getters: { ... },
};

Accediendo a Módulos con Namespaces

// En un componente de Vue
this.$store.dispatch('user/updateUser', user);
this.$store.commit('user/SET_USER', user);

Módulos Dinámicos

Los módulos dinámicos pueden ser registrados y desregistrados en tiempo de ejecución, lo cual es útil para aplicaciones grandes o cuando los módulos solo se necesitan bajo ciertas condiciones.

Ejemplo

// Registrando un módulo dinámico
this.$store.registerModule('dynamicModule', {
  state: { ... },
  mutations: { ... },
  actions: { ... },
  getters: { ... },
});

// Desregistrando un módulo dinámico
this.$store.unregisterModule('dynamicModule');

API de Composición con Vuex

Usar Vuex con la API de Composición permite una mejor organización y reutilización del código.

Ejemplo

// store/modules/user.js
import { reactive, toRefs } from 'vue';

export default {
  namespaced: true,
  state: () => reactive({
    name: '',
    age: null,
  }),
  mutations: { ... },
  actions: { ... },
  getters: { ... },
};

// En un componente de Vue
import { useStore } from 'vuex';
import { computed } from 'vue';

export default {
  setup() {
    const store = useStore();
    const userName = computed(() => store.state.user.name);
    const userAge = computed(() => store.state.user.age);

    return { userName, userAge };
  },
};

Plugins

Los plugins de Vuex pueden ser utilizados para extender la funcionalidad de la tienda.

Ejemplo

// Plugin de registro
const logger = (store) => {
  store.subscribe((mutation, state) => {
    console.log(mutation.type);
    console.log(mutation.payload);
  });
};

export default new Vuex.Store({
  plugins: [logger],
});

Pruebas de Vuex

Escribir pruebas unitarias para la tienda de Vuex asegura la fiabilidad de tu lógica de gestión de estado.

Ejemplo

// store/modules/user.js
import { mutations } from './user';
const { SET_USER } = mutations;

describe('mutations', () => {
  it('SET_USER', () => {
    const state = { name: '', age: null };
    SET_USER(state, { name: 'John', age: 30 });
    expect(state.name).toBe('John');
    expect(state.age).toBe(30);
  });
});

Conclusión

Patrones avanzados de Vuex como módulos, namespaces, módulos dinámicos, la API de Composición, plugins y pruebas pueden mejorar significativamente la estructura y mantenibilidad de tus aplicaciones Vue.js. Al dominar estos patrones, puedes construir aplicaciones escalables y robustas con facilidad.
