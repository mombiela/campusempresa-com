Introducción

En este proyecto, construiremos una plataforma de blog completamente funcional usando Vue.js. Este proyecto te ayudará a entender cómo crear una aplicación compleja descomponiéndola en componentes manejables. Al final de este proyecto, tendrás una comprensión sólida de Vue.js y cómo usarlo para construir aplicaciones web dinámicas.

Conceptos Clave

  • Arquitectura basada en componentes
  • Vue Router para navegación
  • Vuex para gestión de estado
  • Integración de API
  • Manejo y validación de formularios
  • Renderizado dinámico

Configuración del Proyecto

Primero, configuremos nuestro proyecto Vue.js usando Vue CLI.

# Instalar Vue CLI si aún no lo has hecho
npm install -g @vue/cli

# Crear un nuevo proyecto
vue create blog-platform

# Navegar al directorio del proyecto
cd blog-platform

# Iniciar el servidor de desarrollo
npm run serve

Configuración de Vue Router

Necesitamos configurar Vue Router para manejar la navegación entre diferentes páginas.

// src/router/index.js
import Vue from 'vue'
import VueRouter from 'vue-router'
import Home from '../views/Home.vue'
import About from '../views/About.vue'

Vue.use(VueRouter)

const routes = [
  {
    path: '/',
    name: 'Home',
    component: Home
  },
  {
    path: '/about',
    name: 'About',
    component: About
  }
]

const router = new VueRouter({
  mode: 'history',
  base: process.env.BASE_URL,
  routes
})

export default router

Gestión de Estado con Vuex

Para gestionar el estado de nuestra aplicación, usaremos Vuex.

// src/store/index.js
import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

export default new Vuex.Store({
  state: {
    posts: []
  },
  mutations: {
    setPosts(state, posts) {
      state.posts = posts
    }
  },
  actions: {
    fetchPosts({ commit }) {
      // Simular una llamada a la API
      const posts = [
        { id: 1, title: 'Primera Publicación', content: 'Esta es la primera publicación.' },
        { id: 2, title: 'Segunda Publicación', content: 'Esta es la segunda publicación.' }
      ]
      commit('setPosts', posts)
    }
  },
  getters: {
    allPosts: state => state.posts
  }
})

Integración de API

Simulemos la obtención de publicaciones desde una API.

// src/views/Home.vue
<template>
  <div>
    <Header />
    <div v-for="post in posts" :key="post.id">
      <Post :post="post" />
    </div>
  </div>
</template>

<script>
import Header from '@/components/Header.vue'
import Post from '@/components/Post.vue'
import { mapGetters, mapActions } from 'vuex'

export default {
  name: 'Home',
  components: {
    Header,
    Post
  },
  computed: {
    ...mapGetters(['allPosts'])
  },
  methods: {
    ...mapActions(['fetchPosts'])
  },
  created() {
    this.fetchPosts()
  }
}
</script>

Manejo y Validación de Formularios

Agreguemos un formulario para crear nuevas publicaciones del blog.

<!-- src/components/NewPost.vue -->
<template>
  <div>
    <h2>Crear Nueva Publicación</h2>
    <form @submit.prevent="createPost">
      <div>
        <label for="title">Título</label>
        <input type="text" v-model="title" required />
      </div>
      <div>
        <label for="content">Contenido</label>
        <textarea v-model="content" required></textarea>
      </div>
      <button type="submit">Crear</button>
    </form>
  </div>
</template>

<script>
export default {
  name: 'NewPost',
  data() {
    return {
      title: '',
      content: ''
    }
  },
  methods: {
    createPost() {
      // Simular llamada a la API para crear una nueva publicación
      const newPost = {
        id: Date.now(),
        title: this.title,
        content: this.content
      }
      this.$store.commit('setPosts', [...this.$store.state.posts, newPost])
      this.title = ''
      this.content = ''
    }
  }
}
</script>

form {

  flex-direction: column;
}

  margin-bottom: 1rem;
}
</style>
