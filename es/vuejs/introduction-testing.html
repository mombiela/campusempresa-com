Resumen

Las pruebas son una parte crucial del desarrollo de software que asegura que tu aplicación funcione como se espera. En esta sección, te introduciremos a las pruebas en Vue.js, cubriendo los conceptos básicos y avanzando gradualmente hacia conceptos más avanzados.

Por Qué las Pruebas son Importantes

  • Asegura la Calidad del Código: Ayuda a detectar errores temprano en el proceso de desarrollo.
  • Facilita la Refactorización: Hace más fácil cambiar el código sin romper la funcionalidad existente.
  • Mejora la Fiabilidad: Asegura que tu aplicación se comporte como se espera bajo diferentes condiciones.
  • Documentación: Las pruebas pueden servir como documentación de cómo se supone que debe funcionar tu código.

Tipos de Pruebas

  • Pruebas Unitarias: Prueban componentes o funciones individuales en aislamiento.
  • Pruebas de Integración: Prueban cómo diferentes partes de la aplicación funcionan juntas.
  • Pruebas de Extremo a Extremo (E2E): Prueban toda la aplicación desde la perspectiva del usuario.

Configuración de Pruebas en Vue.js

Para comenzar con las pruebas en Vue.js, necesitas configurar tu entorno. Vue CLI proporciona soporte integrado para pruebas.

Instalación de Vue CLI

Primero, asegúrate de tener Vue CLI instalado:

npm install -g @vue/cli

Creación de un Nuevo Proyecto con Soporte para Pruebas

Crea un nuevo proyecto Vue con soporte para pruebas:

vue create my-vue-app

Durante la configuración, selecciona la opción para incluir Pruebas Unitarias y Pruebas E2E.

Instalación de Bibliotecas de Pruebas

Para las pruebas unitarias, usaremos Jest, y para las pruebas E2E, usaremos Cypress.

Instala Jest:

npm install --save-dev @vue/test-utils jest

Instala Cypress:

npm install --save-dev cypress

Escribiendo Pruebas Unitarias

Las pruebas unitarias se enfocan en probar componentes individuales. Vamos a escribir una prueba unitaria simple para un componente Vue.

Componente de Ejemplo

Aquí hay un componente Vue simple HelloWorld.vue:

<template>
  <div>
    <p>{{ message }}</p>
  </div>
</template>

<script>
export default {
  name: 'HelloWorld',
  props: {
    message: {
      type: String,
      required: true
    }
  }
}
</script>

Escribiendo una Prueba Unitaria

Crea un archivo de prueba HelloWorld.spec.js:

import { shallowMount } from '@vue/test-utils';
import HelloWorld from '@/components/HelloWorld.vue';

describe('HelloWorld.vue', () => {
  it('renders props.message when passed', () => {
    const message = 'Hello, Vue!';
    const wrapper = shallowMount(HelloWorld, {
      propsData: { message }
    });
    expect(wrapper.text()).toMatch(message);
  });
});

Ejecutando Pruebas Unitarias

Para ejecutar tus pruebas unitarias, usa el siguiente comando:

npm run test:unit

Escribiendo Pruebas E2E

Las pruebas de Extremo a Extremo simulan interacciones de usuario con tu aplicación. Vamos a escribir una prueba E2E simple usando Cypress.

Ejemplo de Prueba E2E

Crea un archivo de prueba helloWorld.spec.js en la carpeta cypress/integration:

describe('HelloWorld Component', () => {
  it('displays the correct message', () => {
    cy.visit('/');
    cy.contains('Hello, Vue!');
  });
});

Ejecutando Pruebas E2E

Para ejecutar tus pruebas E2E, usa el siguiente comando:

npm run test:e2e

Conclusión

Las pruebas son una práctica esencial en el desarrollo de Vue.js que asegura que tu aplicación sea fiable, mantenible y libre de errores. Al configurar pruebas unitarias y E2E, puedes detectar problemas temprano y facilitar la refactorización. A medida que progreses, puedes explorar técnicas y herramientas de prueba más avanzadas para mejorar aún más tu estrategia de pruebas.
