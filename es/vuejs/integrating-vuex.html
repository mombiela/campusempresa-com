Introducción a Vuex

Vuex es un patrón de gestión de estado + biblioteca para aplicaciones Vue.js. Sirve como una tienda centralizada para todos los componentes en una aplicación, con reglas que aseguran que el estado solo pueda ser mutado de manera predecible.

Conceptos Clave

  • Estado: La única fuente de verdad.
  • Getters: El equivalente a las propiedades computadas para las tiendas.
  • Mutaciones: La única forma de cambiar realmente el estado en una tienda Vuex.
  • Acciones: Similar a las mutaciones, pero en lugar de mutar el estado, las acciones cometen mutaciones.
  • Módulos: Te permiten dividir la tienda en múltiples módulos, cada uno con su propio estado, mutaciones, acciones y getters.

Configuración de Vuex

Para integrar Vuex en tu proyecto Vue.js, primero necesitas instalarlo.

npm install vuex --save

Luego, crea una tienda e intégrala en tu instancia de Vue.

// store.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment(context) {
      context.commit('increment');
    }
  },
  getters: {
    count: state => state.count
  }
});
// main.js
import Vue from 'vue';
import App from './App.vue';
import store from './store';

new Vue({
  render: h => h(App),
  store
}).$mount('#app');

Gestión de Estado

El estado en Vuex es un único objeto que contiene todo el estado a nivel de aplicación.

const store = new Vuex.Store({
  state: {
    count: 0
  }
});

Accediendo al Estado

Puedes acceder al estado en tus componentes usando this.$store.state.

<template>
  <div>{{ count }}</div>
</template>

<script>
export default {
  computed: {
    count() {
      return this.$store.state.count;
    }
  }
};
</script>

Mutaciones

Las mutaciones son la única forma de cambiar realmente el estado en una tienda Vuex. Cada mutación tiene un tipo de cadena y un manejador.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  }
});

Cometiendo Mutaciones

Puedes cometer mutaciones usando store.commit.

<template>
  <button @click="increment">Incrementar</button>
</template>

<script>
export default {
  methods: {
    increment() {
      this.$store.commit('increment');
    }
  }
};
</script>

Acciones

Las acciones son similares a las mutaciones, pero en lugar de mutar el estado, las acciones cometen mutaciones. Las acciones pueden contener operaciones asincrónicas arbitrarias.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment');
      }, 1000);
    }
  }
});

Despachando Acciones

Puedes despachar acciones usando store.dispatch.

<template>
  <button @click="incrementAsync">Incrementar Asíncrono</button>
</template>

<script>
export default {
  methods: {
    incrementAsync() {
      this.$store.dispatch('incrementAsync');
    }
  }
};
</script>

Getters

Los getters son el equivalente de las propiedades computadas para las tiendas. Se utilizan para calcular el estado derivado basado en el estado de la tienda.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
    doubleCount: state => state.count * 2
  }
});

Accediendo a los Getters

Puedes acceder a los getters en tus componentes usando this.$store.getters.

<template>
  <div>{{ doubleCount }}</div>
</template>

<script>
export default {
  computed: {
    doubleCount() {
      return this.$store.getters.doubleCount;
    }
  }
};
</script>

Módulos

Los módulos te permiten dividir tu tienda en múltiples módulos, cada uno con su propio estado, mutaciones, acciones y getters.

const moduleA = {
  state: { count: 0 },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment');
    }
  },
  getters: {
    doubleCount: state => state.count * 2
  }
};

const store = new Vuex.Store({
  modules: {
    a: moduleA
  }
});

Accediendo al Estado del Módulo

Puedes acceder al estado del módulo en tus componentes usando this.$store.state.moduleName.

<template>
  <div>{{ count }}</div>
</template>

<script>
export default {
  computed: {
    count() {
      return this.$store.state.a.count;
    }
  }
};
</script>

Conclusión

Integrar Vuex en tu aplicación Vue.js proporciona una solución robusta para gestionar el estado en toda tu aplicación. Al entender y utilizar el estado, las mutaciones, las acciones, los getters y los módulos, puedes crear aplicaciones escalables y mantenibles. Practica construyendo pequeñas aplicaciones e incorporando gradualmente una gestión de estado más compleja para volverte competente en el uso de Vuex.
