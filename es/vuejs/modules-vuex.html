Usando Vuex en Componentes ►

Vuex es un patrón de gestión de estado + biblioteca para aplicaciones Vue.js. Sirve como una tienda centralizada para todos los componentes en una aplicación, con reglas que aseguran que el estado solo pueda ser mutado de una manera predecible. A medida que tu aplicación crece, puedes encontrar que la tienda se vuelve difícil de manejar. Aquí es donde los módulos de Vuex son útiles.

¿Qué son los Módulos de Vuex?

Los módulos de Vuex te permiten dividir tu tienda en piezas más pequeñas y manejables. Cada módulo puede contener su propio estado, mutaciones, acciones, getters e incluso módulos anidados. Este enfoque modular ayuda a organizar la tienda de una manera que es más fácil de mantener y escalar.

Conceptos Clave de los Módulos de Vuex

  • Estado: Cada módulo puede tener su propio estado, que está aislado del estado raíz y de otros módulos.
  • Mutaciones: Cada módulo puede definir sus propias mutaciones para modificar su estado.
  • Acciones: Cada módulo puede tener acciones para cometer mutaciones o despachar otras acciones.
  • Getters: Cada módulo puede tener getters para computar el estado derivado basado en su estado.
  • Espaciado de Nombres: Los módulos pueden tener un espacio de nombres para evitar conflictos de nombres y hacer la tienda más modular.

Creando un Módulo de Vuex

Vamos a crear un módulo de Vuex simple para gestionar la autenticación de usuarios.

Definiendo el Módulo

Primero, define el módulo en un archivo separado, auth.js:

// store/modules/auth.js
const state = {
  isAuthenticated: false,
  user: null,
};

const mutations = {
  SET_AUTH(state, payload) {
    state.isAuthenticated = payload.isAuthenticated;
    state.user = payload.user;
  },
};

const actions = {
  login({ commit }, user) {
    // Simular una llamada a la API
    setTimeout(() => {
      commit('SET_AUTH', { isAuthenticated: true, user });
    }, 1000);
  },
  logout({ commit }) {
    commit('SET_AUTH', { isAuthenticated: false, user: null });
  },
};

const getters = {
  isAuthenticated: (state) => state.isAuthenticated,
  user: (state) => state.user,
};

export default {
  state,
  mutations,
  actions,
  getters,
};

Registrando el Módulo

A continuación, registra el módulo en tu tienda de Vuex:

// store/index.js
import Vue from 'vue';
import Vuex from 'vuex';
import auth from './modules/auth';

Vue.use(Vuex);

export default new Vuex.Store({
  modules: {
    auth,
  },
});

Usando el Módulo en un Componente

Ahora, puedes usar el estado, getters, acciones y mutaciones del módulo en tus componentes Vue:

<template>
  <div>
    <p v-if="isAuthenticated">Bienvenido, {{ user.name }}</p>
    <button @click="login">Iniciar Sesión</button>
    <button @click="logout">Cerrar Sesión</button>
  </div>
</template>

<script>
import { mapState, mapGetters, mapActions } from 'vuex';

export default {
  computed: {
    ...mapState('auth', ['isAuthenticated', 'user']),
    ...mapGetters('auth', ['isAuthenticated', 'user']),
  },
  methods: {
    ...mapActions('auth', ['login', 'logout']),
  },
};
</script>

Espaciado de Nombres en Módulos

Por defecto, todos los módulos están registrados bajo el espacio de nombres global. Esto puede llevar a conflictos de nombres, especialmente en aplicaciones grandes. Para evitar esto, puedes espaciar los nombres de tus módulos.

Habilitando el Espaciado de Nombres

Para habilitar el espaciado de nombres, establece la propiedad namespaced a true en tu módulo:

// store/modules/auth.js
export default {
  namespaced: true,
  state,
  mutations,
  actions,
  getters,
};

Accediendo a Módulos con Espaciado de Nombres

Cuando uses módulos con espaciado de nombres, necesitas especificar el nombre del módulo al acceder a su estado, getters, acciones y mutaciones:

<script>
import { mapState, mapGetters, mapActions } from 'vuex';

export default {
  computed: {
    ...mapState('auth', ['isAuthenticated', 'user']),
    ...mapGetters('auth', ['isAuthenticated', 'user']),
  },
  methods: {
    ...mapActions('auth', ['login', 'logout']),
  },
};
</script>

Conclusión

Los módulos de Vuex son una característica poderosa que ayuda a organizar y gestionar el estado en grandes aplicaciones Vue.js. Al dividir la tienda en módulos más pequeños y autónomos, puedes hacer tu código más modular, mantenible y escalable. Recuerda usar el espaciado de nombres para evitar conflictos y mantener tu tienda limpia y organizada.
