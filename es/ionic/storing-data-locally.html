En esta sección, exploraremos cómo almacenar datos localmente en una aplicación Ionic. Esto es crucial para crear aplicaciones que puedan funcionar sin conexión o necesiten guardar preferencias del usuario, configuraciones u otros datos.

Conceptos Clave

  • Almacenamiento Local: Un mecanismo de almacenamiento simple de clave-valor.
  • SQLite: Un motor de base de datos SQL completo.
  • Ionic Storage: Una API unificada para diferentes motores de almacenamiento.

Almacenamiento Local

El Almacenamiento Local es un mecanismo de almacenamiento web que te permite almacenar datos en el navegador. Es simple de usar pero tiene limitaciones en términos de tamaño de almacenamiento y complejidad de datos.

Usando Almacenamiento Local

Para usar el Almacenamiento Local en una aplicación Ionic, puedes interactuar directamente con el objeto window.localStorage.

Ejemplo

// Guardar datos en el Almacenamiento Local
window.localStorage.setItem('username', 'john_doe');

// Recuperar datos del Almacenamiento Local
const username = window.localStorage.getItem('username');
console.log(username); // Salida: john_doe

// Eliminar datos del Almacenamiento Local
window.localStorage.removeItem('username');

Limitaciones

  • Capacidad de almacenamiento limitada (usualmente alrededor de 5MB).
  • Solo soporta tipos de datos de cadena.
  • API sincrónica, lo que podría bloquear el hilo principal.

SQLite

SQLite es un motor de base de datos SQL poderoso y completo que puede ser embebido en tu aplicación Ionic. Es adecuado para necesidades de almacenamiento de datos más complejas.

Configuración de SQLite

Para usar SQLite en una aplicación Ionic, necesitas instalar el plugin Cordova SQLite.

ionic cordova plugin add cordova-sqlite-storage
npm install @ionic-native/sqlite

Usando SQLite

Ejemplo

import { SQLite, SQLiteObject } from '@ionic-native/sqlite/ngx';

constructor(private sqlite: SQLite) {}

this.sqlite.create({
  name: 'data.db',
  location: 'default'
})
.then((db: SQLiteObject) => {

  db.executeSql('create table danceMoves(name VARCHAR(32))', [])
    .then(() => console.log('Executed SQL'))
    .catch(e => console.log(e));

})
.catch(e => console.log(e));

Ventajas

  • Soporta consultas y transacciones complejas.
  • Puede manejar grandes cantidades de datos.
  • API asincrónica, lo cual es mejor para el rendimiento.

Ionic Storage

Ionic Storage proporciona una API simple y unificada para diferentes motores de almacenamiento, incluyendo Almacenamiento Local, IndexedDB y SQLite. Es altamente recomendado para la mayoría de los casos de uso debido a su flexibilidad y facilidad de uso.

Configuración de Ionic Storage

Para usar Ionic Storage, necesitas instalar los paquetes necesarios.

npm install @ionic/storage-angular
npm install @ionic/storage

Configuración de Ionic Storage

Necesitas configurar Ionic Storage en tu módulo de la aplicación.

import { IonicStorageModule } from '@ionic/storage-angular';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,
    IonicModule.forRoot(),
    AppRoutingModule,
    IonicStorageModule.forRoot()
  ],
  bootstrap: [AppComponent]
})
export class AppModule {}

Usando Ionic Storage

Ejemplo

import { Storage } from '@ionic/storage-angular';

constructor(private storage: Storage) {
  this.init();
}

async init() {
  await this.storage.create();
}

// Guardar datos
this.storage.set('username', 'john_doe');

// Recuperar datos
this.storage.get('username').then((val) => {
  console.log(val); // Salida: john_doe
});

// Eliminar datos
this.storage.remove('username');

Ventajas

  • API unificada para diferentes motores de almacenamiento.
  • API asincrónica.
  • Fácil de configurar y usar.

Resumen

En esta sección, exploramos diferentes métodos para almacenar datos localmente en una aplicación Ionic:

  • Almacenamiento Local: Simple y fácil de usar pero limitado en capacidad y funcionalidad.
  • SQLite: Poderoso y adecuado para necesidades de almacenamiento de datos complejas.
  • Ionic Storage: Proporciona una API unificada para diferentes motores de almacenamiento, lo que lo hace flexible y fácil de usar.

Elegir el mecanismo de almacenamiento adecuado depende de los requisitos de tu aplicación, como la cantidad de datos, la complejidad de las consultas y la necesidad de funcionalidad sin conexión.
