En esta sección, exploraremos cómo pasar datos entre páginas en una aplicación Ionic. Esta es una habilidad crucial para crear aplicaciones móviles dinámicas e interactivas. Cubriremos varios métodos, incluyendo el uso del Router de Angular, el NavController de Ionic y servicios compartidos.

Conceptos Clave

  • Enrutamiento y Navegación: Entender cómo Ionic maneja la navegación entre páginas.
  • Pasar Datos a través de Parámetros de URL: Usar el Router de Angular para pasar datos.
  • Usar el NavController de Ionic: Aprovechar el controlador de navegación incorporado de Ionic.
  • Servicios Compartidos: Usar servicios de Angular para compartir datos entre componentes.

Enrutamiento y Navegación

Ionic usa el sistema de enrutamiento de Angular para navegar entre páginas. Entender cómo configurar rutas y navegar entre ellas es el primer paso para pasar datos entre páginas.

Configuración de Rutas

En Ionic, las rutas se definen en el archivo app-routing.module.ts. Aquí hay un ejemplo de cómo configurar rutas:

import { NgModule } from '@angular/core';
import { PreloadAllModules, RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: '',
    redirectTo: 'home',
    pathMatch: 'full'
  },
  {
    path: 'home',
    loadChildren: () => import('./home/home.module').then(m => m.HomePageModule)
  },
  {
    path: 'details/:id',
    loadChildren: () => import('./details/details.module').then(m => m.DetailsPageModule)
  }
];

@NgModule({
  imports: [
    RouterModule.forRoot(routes, { preloadingStrategy: PreloadAllModules })
  ],
  exports: [RouterModule]
})
export class AppRoutingModule { }

En este ejemplo, tenemos dos rutas: home y details/:id. La parte :id es un parámetro de ruta que podemos usar para pasar datos a la página details.

Pasar Datos a través de Parámetros de URL

Un método común para pasar datos entre páginas es usando parámetros de URL. Esto se hace usando el Router de Angular.

Enviar Datos

Para navegar a la página details y pasar un parámetro id, puedes usar el servicio Router:

import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private router: Router) {}

  goToDetails(id: number) {
    this.router.navigate(['/details', id]);
  }
}

Recibir Datos

En la página details, puedes recuperar el parámetro id usando el servicio ActivatedRoute:

import { Component, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  id: number;

  constructor(private route: ActivatedRoute) {}

  ngOnInit() {
    this.id = +this.route.snapshot.paramMap.get('id');
  }
}

Usar el NavController de Ionic

Ionic proporciona su propio controlador de navegación, NavController, que también puede ser usado para pasar datos entre páginas.

Enviar Datos

Para navegar y pasar datos usando NavController, puedes usar el método navigateForward con un objeto de estado:

import { Component } from '@angular/core';
import { NavController } from '@ionic/angular';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private navCtrl: NavController) {}

  goToDetails(id: number) {
    this.navCtrl.navigateForward('/details', {
      state: {
        id: id
      }
    });
  }
}

Recibir Datos

En la página details, puedes recuperar los datos del estado de navegación:

import { Component, OnInit } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  id: number;

  constructor(private router: Router) {}

  ngOnInit() {
    if (this.router.getCurrentNavigation().extras.state) {
      this.id = this.router.getCurrentNavigation().extras.state.id;
    }
  }
}

Servicios Compartidos

Otro método para pasar datos entre páginas es usando servicios de Angular. Esto es particularmente útil para compartir datos o estados complejos.

Crear un Servicio

Primero, crea un servicio para mantener los datos compartidos:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  private data: any;

  setData(data: any) {
    this.data = data;
  }

  getData() {
    return this.data;
  }
}

Usar el Servicio

Luego puedes usar este servicio en tus componentes para establecer y obtener datos:

import { Component } from '@angular/core';
import { Router } from '@angular/router';
import { DataService } from '../services/data.service';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private router: Router, private dataService: DataService) {}

  goToDetails(data: any) {
    this.dataService.setData(data);
    this.router.navigate(['/details']);
  }
}
import { Component, OnInit } from '@angular/core';
import { DataService } from '../services/data.service';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  data: any;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.data = this.dataService.getData();
  }
}

Conclusión

Pasar datos entre páginas en una aplicación Ionic puede lograrse a través de varios métodos, cada uno con sus propios casos de uso y ventajas. Ya sea que elijas usar parámetros de URL, el NavController de Ionic o servicios compartidos, entender estas técnicas te ayudará a crear aplicaciones más dinámicas e interactivas. Experimenta con cada método para ver cuál se adapta mejor a las necesidades de tu aplicación.
