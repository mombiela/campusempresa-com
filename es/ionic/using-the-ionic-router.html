Introducción al Router de Ionic

El Router de Ionic es una herramienta poderosa que te permite gestionar la navegación dentro de tus aplicaciones Ionic. Está construido sobre el Router de Angular, proporcionando una manera fluida de manejar la navegación y el enrutamiento en tu aplicación.

Conceptos Clave

  • Rutas: Definen los caminos y componentes que deben mostrarse.
  • Router Outlet: Actúa como un marcador de posición para los componentes enrutados.
  • Navegación: Navegación programática entre rutas.
  • Guards: Protegen las rutas del acceso no autorizado.

Configuración del Router de Ionic

Para usar el Router de Ionic, necesitas asegurarte de que tu proyecto esté configurado correctamente.

Instalación de Dependencias

Asegúrate de tener las dependencias necesarias instaladas:

npm install @ionic/angular @angular/router

Configuración de Rutas

Define tus rutas en el archivo app-routing.module.ts:

import { NgModule } from '@angular/core';
import { PreloadAllModules, RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: '',
    redirectTo: 'home',
    pathMatch: 'full'
  },
  {
    path: 'home',
    loadChildren: () => import('./home/home.module').then(m => m.HomePageModule)
  },
  {
    path: 'details',
    loadChildren: () => import('./details/details.module').then(m => m.DetailsPageModule)
  }
];

@NgModule({
  imports: [
    RouterModule.forRoot(routes, { preloadingStrategy: PreloadAllModules })
  ],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Uso de Router Outlet

La directiva RouterOutlet se usa para mostrar los componentes enrutados:

<ion-app>
  <ion-router-outlet></ion-router-outlet>
</ion-app>

Navegación Entre Rutas

Puedes navegar entre rutas usando el servicio Router o la directiva routerLink.

Uso del Servicio Router

Inyecta el servicio Router y usa el método navigate:

import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {
  constructor(private router: Router) {}

  goToDetails() {
    this.router.navigate(['/details']);
  }
}

Uso de la Directiva routerLink

Usa la directiva routerLink en tus plantillas:

<ion-button [routerLink]="['/details']">Ir a Detalles</ion-button>

Guards de Ruta

Los guards de ruta pueden usarse para controlar el acceso a ciertas rutas.

Creación de un Guard

Genera un guard usando Angular CLI:

ng generate guard auth

Implementación del Guard

Implementa la interfaz CanActivate en tu guard:

import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, UrlTree } from '@angular/router';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): Observable<boolean | UrlTree> | Promise<boolean | UrlTree> | boolean | UrlTree {
    // Añade tu lógica de autenticación aquí
    return true; // o false basado en el estado de autenticación
  }
}

Aplicación del Guard a Rutas

Aplica el guard a tus rutas en app-routing.module.ts:

const routes: Routes = [
  {
    path: 'details',
    loadChildren: () => import('./details/details.module').then(m => m.DetailsPageModule),
    canActivate: [AuthGuard]
  }
];

Conclusión

El Router de Ionic es una parte esencial para construir aplicaciones Ionic, proporcionando una manera robusta de gestionar la navegación y el enrutamiento. Al entender las rutas, los router outlets, los métodos de navegación y los guards de ruta, puedes crear aplicaciones dinámicas y seguras. Practica implementando estos conceptos en tus proyectos para obtener una comprensión más profunda y competencia en el uso del Router de Ionic.
