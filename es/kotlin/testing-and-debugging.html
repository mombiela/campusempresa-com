Introducción

Las pruebas y la depuración son aspectos cruciales del desarrollo de software. Aseguran que tu código funcione como se espera y te ayudan a identificar y corregir problemas. En esta sección, cubriremos los conceptos básicos de las pruebas y la depuración en Kotlin, progresando desde niveles principiantes hasta avanzados.

Pruebas Unitarias con JUnit

¿Qué son las Pruebas Unitarias?

  • Las pruebas unitarias implican probar componentes individuales de tu software para asegurar que funcionen correctamente.
  • JUnit es un marco popular para pruebas unitarias en Kotlin.

Configuración de JUnit

Para comenzar a usar JUnit en tu proyecto de Kotlin, necesitas agregar la dependencia de JUnit a tu archivo build.gradle:

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.7.0'
}

Escribiendo tu Primera Prueba

Aquí tienes un ejemplo simple de una prueba unitaria en Kotlin:

import org.junit.jupiter.api.Assertions.assertEquals
import org.junit.jupiter.api.Test

class CalculatorTest {

    @Test
    fun testAddition() {
        val calculator = Calculator()
        val result = calculator.add(2, 3)
        assertEquals(5, result)
    }
}

class Calculator {
    fun add(a: Int, b: Int): Int {
        return a + b
    }
}
  • La anotación @Test marca el método como un método de prueba.
  • assertEquals verifica si el resultado esperado coincide con el resultado real.

Ejecutando Pruebas

  • Puedes ejecutar pruebas directamente desde tu IDE (por ejemplo, IntelliJ IDEA) o usando herramientas de línea de comandos como Gradle.

Técnicas Avanzadas de Pruebas

Pruebas Parametrizadas

Las pruebas parametrizadas te permiten ejecutar la misma prueba con diferentes entradas.

import org.junit.jupiter.params.ParameterizedTest
import org.junit.jupiter.params.provider.CsvSource
import org.junit.jupiter.api.Assertions.assertEquals

class CalculatorTest {

    @ParameterizedTest
    @CsvSource("1, 1, 2", "2, 3, 5", "3, 5, 8")
    fun testAddition(a: Int, b: Int, expected: Int) {
        val calculator = Calculator()
        val result = calculator.add(a, b)
        assertEquals(expected, result)
    }
}

Mocking con Mockito

Mockito es un marco que te permite crear objetos simulados para pruebas.

import org.junit.jupiter.api.Test
import org.mockito.Mockito.*
import org.mockito.MockitoAnnotations

class UserServiceTest {

    @Test
    fun testGetUser() {
        MockitoAnnotations.openMocks(this)
        val userRepository = mock(UserRepository::class.java)
        val userService = UserService(userRepository)

        `when`(userRepository.findUserById(1)).thenReturn(User(1, "John Doe"))

        val user = userService.getUser(1)
        assertEquals("John Doe", user.name)
    }
}

Depuración en Kotlin

Usando el Depurador de Kotlin

  • La mayoría de los IDEs, como IntelliJ IDEA, vienen con herramientas de depuración integradas.
  • Puedes establecer puntos de interrupción en tu código para pausar la ejecución e inspeccionar variables.

Técnicas Comunes de Depuración

  • Puntos de Interrupción: Pausar la ejecución en líneas específicas.
  • Paso Sobre: Ejecutar la línea actual y pasar a la siguiente.
  • Paso Dentro: Entrar en las llamadas de método.
  • Ver Variables: Monitorear los valores de las variables durante la ejecución.

Ejemplo de Sesión de Depuración

fun main() {
    val numbers = listOf(1, 2, 3, 4, 5)
    val result = sum(numbers)
    println("Sum: $result")
}

fun sum(numbers: List<Int>): Int {
    var total = 0
    for (number in numbers) {
        total += number
    }
    return total
}
  • Establece un punto de interrupción en total += number.
  • Ejecuta el programa en modo de depuración.
  • Inspecciona el valor de total y number en cada iteración.

Conclusión

Las pruebas y la depuración son habilidades esenciales para cualquier desarrollador de Kotlin. Al dominar las pruebas unitarias con JUnit, las técnicas avanzadas de pruebas con pruebas parametrizadas y Mockito, y las estrategias efectivas de depuración, puedes asegurar que tu código sea robusto y confiable. Practica estas técnicas regularmente para volverte competente en identificar y corregir problemas en tu código.
