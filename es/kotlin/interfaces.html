Introducción a las Interfaces

Las interfaces en Kotlin son una forma poderosa de definir contratos que las clases pueden implementar. Permiten especificar métodos y propiedades que deben ser implementados por cualquier clase que implemente la interfaz.

  • Definición: Una interfaz es un plano de una clase que contiene métodos y propiedades abstractas.
  • Propósito: Proporcionar una forma de lograr abstracción y herencia múltiple en Kotlin.

Conceptos Clave

  • Métodos Abstractos: Métodos sin cuerpo que deben ser implementados por la clase.
  • Métodos por Defecto: Métodos con cuerpo que pueden ser opcionalmente sobrescritos por la clase.
  • Propiedades en Interfaces: Las interfaces también pueden contener propiedades, que pueden ser abstractas o tener implementaciones por defecto.

Definiendo una Interfaz

Para definir una interfaz en Kotlin, usa la palabra clave interface.

interface Drivable {
    fun drive() // Método abstracto
    val maxSpeed: Int // Propiedad abstracta
}

Implementando una Interfaz

Una clase que implementa una interfaz debe proporcionar implementaciones para todos los métodos y propiedades abstractas.

class Car : Drivable {
    override fun drive() {
        println("Car is driving")
    }

    override val maxSpeed: Int = 200
}

Métodos y Propiedades por Defecto

Las interfaces también pueden contener métodos y propiedades con implementaciones por defecto.

interface Flyable {
    fun fly() {
        println("Flying")
    }

    val altitude: Int
        get() = 1000
}

class Airplane : Flyable {
    // No es necesario sobrescribir fly() o altitude a menos que se necesite un comportamiento personalizado
}

Múltiples Interfaces

Una clase en Kotlin puede implementar múltiples interfaces, permitiendo un código más flexible y reutilizable.

interface Swimmable {
    fun swim()
}

class AmphibiousCar : Drivable, Swimmable {
    override fun drive() {
        println("Amphibious car is driving")
    }

    override val maxSpeed: Int = 150

    override fun swim() {
        println("Amphibious car is swimming")
    }
}

Herencia de Interfaces

Las interfaces pueden heredar de otras interfaces, permitiendo diseños más complejos y jerárquicos.

interface Vehicle {
    fun start()
}

interface ElectricVehicle : Vehicle {
    fun chargeBattery()
}

class Tesla : ElectricVehicle {
    override fun start() {
        println("Tesla is starting")
    }

    override fun chargeBattery() {
        println("Charging Tesla's battery")
    }
}

Ejemplos Prácticos

Ejemplo 1: Implementación Simple de una Interfaz

interface Animal {
    fun sound()
}

class Dog : Animal {
    override fun sound() {
        println("Bark")
    }
}

fun main() {
    val dog = Dog()
    dog.sound() // Salida: Bark
}

Ejemplo 2: Interfaz con Métodos por Defecto

interface Printer {
    fun print() {
        println("Printing document")
    }
}

class LaserPrinter : Printer

fun main() {
    val printer = LaserPrinter()
    printer.print() // Salida: Printing document
}

Ejemplo 3: Múltiples Interfaces

interface Scanner {
    fun scan()
}

class MultiFunctionPrinter : Printer, Scanner {
    override fun scan() {
        println("Scanning document")
    }
}

fun main() {
    val mfp = MultiFunctionPrinter()
    mfp.print() // Salida: Printing document
    mfp.scan()  // Salida: Scanning document
}

Conclusión

Las interfaces en Kotlin proporcionan un mecanismo robusto para definir contratos que las clases pueden implementar. Soportan abstracción, herencia múltiple, y pueden contener tanto métodos y propiedades abstractas como por defecto. Al entender y utilizar interfaces, puedes crear código más modular, reutilizable y mantenible.
