Delegación en Kotlin
Expresiones Lambda ►

La delegación es un patrón de diseño en Kotlin que permite a un objeto transferir algunas de sus responsabilidades a otro objeto. Esto puede ser particularmente útil para la reutilización de código y la composición. Kotlin proporciona soporte integrado para la delegación a través de la palabra clave by, lo que facilita la implementación de patrones de delegación.

¿Qué es la Delegación?

  • Definición: La delegación es un patrón de diseño donde un objeto delega algunas de sus responsabilidades a otro objeto.
  • Propósito: Ayuda en la reutilización de código y la composición, haciendo el código más modular y fácil de mantener.

Tipos de Delegación en Kotlin

  1. Delegación de Clases
  2. Delegación de Propiedades

Delegación de Clases

La delegación de clases permite que una clase delegue su implementación a otra clase. Esto se hace usando la palabra clave by.

Ejemplo

interface Printer {
    fun print()
}

class SimplePrinter : Printer {
    override fun print() {
        println("Printing from SimplePrinter")
    }
}

class AdvancedPrinter(printer: Printer) : Printer by printer

fun main() {
    val simplePrinter = SimplePrinter()
    val advancedPrinter = AdvancedPrinter(simplePrinter)
    advancedPrinter.print()  // Output: Printing from SimplePrinter
}
  • Explicación:
    • Printer es una interfaz con un método print.
    • SimplePrinter es una clase que implementa la interfaz Printer.
    • AdvancedPrinter delega el método print a una instancia de Printer usando la palabra clave by.

Delegación de Propiedades

La delegación de propiedades permite delegar el getter y setter de una propiedad a otro objeto. Kotlin proporciona varios delegados integrados, como lazy, observable y vetoable.

Inicialización Perezosa

La inicialización perezosa es un caso de uso común para la delegación de propiedades. El delegado lazy inicializa la propiedad solo cuando se accede a ella por primera vez.

Ejemplo

class LazyExample {
    val lazyValue: String by lazy {
        println("Computed!")
        "Hello, Kotlin!"
    }
}

fun main() {
    val example = LazyExample()
    println(example.lazyValue)  // Output: Computed! Hello, Kotlin!
    println(example.lazyValue)  // Output: Hello, Kotlin!
}
  • Explicación:
    • lazyValue es una propiedad que usa el delegado lazy.
    • La primera vez que se accede a lazyValue, se ejecuta la expresión lambda y se imprime "Computed!".
    • Los accesos posteriores a lazyValue no vuelven a ejecutar la expresión lambda.

Propiedades Observables

El delegado observable permite rastrear cambios en una propiedad.

Ejemplo

import kotlin.properties.Delegates

class User {
    var name: String by Delegates.observable("<no name>") { prop, old, new ->
        println("Property ${prop.name} changed from $old to $new")
    }
}

fun main() {
    val user = User()
    user.name = "John"  // Output: Property name changed from <no name> to John
    user.name = "Doe"   // Output: Property name changed from John to Doe
}
  • Explicación:
    • name es una propiedad que usa el delegado observable.
    • La expresión lambda se llama cada vez que la propiedad cambia, imprimiendo los valores antiguos y nuevos.

Delegados de Propiedades Personalizados

También puedes crear tus propios delegados de propiedades personalizados implementando las interfaces ReadOnlyProperty o ReadWriteProperty.

Ejemplo

import kotlin.reflect.KProperty

class CustomDelegate {
    operator fun getValue(thisRef: Any?, property: KProperty<*>): String {
        return "Custom Delegate Value"
    }

    operator fun setValue(thisRef: Any?, property: KProperty<*>, value: String) {
        println("Setting value '$value' to '${property.name}'")
    }
}

class Example {
    var customProperty: String by CustomDelegate()
}

fun main() {
    val example = Example()
    println(example.customProperty)  // Output: Custom Delegate Value
    example.customProperty = "New Value"  // Output: Setting value 'New Value' to 'customProperty'
}
  • Explicación:
    • La clase CustomDelegate implementa los operadores getValue y setValue.
    • customProperty en la clase Example usa el CustomDelegate.

Conclusión

La delegación en Kotlin es una característica poderosa que permite un código más limpio y modular. Al usar la delegación de clases y propiedades, puedes delegar fácilmente responsabilidades y comportamientos a otros objetos, promoviendo la reutilización de código y reduciendo el código repetitivo. Ya sea utilizando delegados integrados como lazy y observable o creando delegados personalizados, los mecanismos de delegación de Kotlin proporcionan una forma flexible y eficiente de gestionar las responsabilidades de los objetos.
