Introducción al Registro de Auditoría

El registro de auditoría es una característica crítica para monitorear y rastrear las actividades de los usuarios dentro de un clúster de Elasticsearch. Ayuda a mantener la seguridad, el cumplimiento y la supervisión operativa registrando eventos como inicios de sesión de usuarios, acceso a datos y cambios de configuración.

Conceptos Clave

  • Registros de Auditoría: Registros de eventos que capturan acciones de usuarios y cambios en el sistema.
  • Cumplimiento: Asegurar que el sistema cumpla con los requisitos regulatorios.
  • Seguridad: Monitorear accesos no autorizados y posibles brechas de seguridad.
  • Supervisión Operativa: Rastrear cambios y actividades para la resolución de problemas y la gestión del sistema.

Configuración del Registro de Auditoría

Para habilitar el registro de auditoría en Elasticsearch, necesitas configurar el archivo elasticsearch.yml. Aquí tienes una guía paso a paso:

Pasos de Configuración

  1. Habilitar el Registro de Auditoría:

    xpack.security.audit.enabled: true

  2. Especificar el Destino de Salida:

    • Archivo de Registro: 
      xpack.security.audit.outputs: [file]
xpack.security.audit.logfile.events.emit_request_body: true
xpack.security.audit.logfile.prefix: "audit"
xpack.security.audit.logfile.events.include: ["access_granted", "access_denied"]
    • Índice: 
      xpack.security.audit.outputs: [index]
xpack.security.audit.index.events.emit_request_body: true
xpack.security.audit.index.events.include: ["access_granted", "access_denied"]

  3. Filtrar Eventos:

    • Incluir eventos específicos: 
      xpack.security.audit.logfile.events.include: ["access_granted", "access_denied"]
    • Excluir eventos específicos: 
      xpack.security.audit.logfile.events.exclude: ["authentication_success"]

Ejemplo de Configuración

Aquí tienes un ejemplo completo de una configuración elasticsearch.yml para el registro de auditoría:

xpack.security.audit.enabled: true
xpack.security.audit.outputs: [file, index]
xpack.security.audit.logfile.events.emit_request_body: true
xpack.security.audit.logfile.prefix: "audit"
xpack.security.audit.logfile.events.include: ["access_granted", "access_denied"]
xpack.security.audit.index.events.emit_request_body: true
xpack.security.audit.index.events.include: ["access_granted", "access_denied"]

Análisis de los Registros de Auditoría

Una vez habilitado el registro de auditoría, puedes analizar los registros para monitorear las actividades de los usuarios y los cambios en el sistema.

Acceso a los Archivos de Registro

Los registros de auditoría se almacenan en el archivo de registro o índice especificado. Por ejemplo, si los registros se almacenan en un archivo, puedes acceder a ellos en la ubicación configurada:

/var/log/elasticsearch/audit.log

Consultar los Registros de Auditoría en Kibana

Si los registros de auditoría se almacenan en un índice, puedes usar Kibana para consultar y visualizar los registros:

GET /_search
{
  "query": {
    "match": {
      "event.type": "access_granted"
    }
  }
}

Ejemplo de Entrada de Registro

Aquí tienes un ejemplo de una entrada de registro de auditoría:

{
  "timestamp": "2023-10-01T12:34:56Z",
  "principal": "user1",
  "action": "access_granted",
  "indices": ["index1"],
  "request": {
    "method": "GET",
    "uri": "/index1/_search"
  }
}

Mejores Prácticas

  • Monitoreo Regular: Revisa regularmente los registros de auditoría para detectar cualquier actividad inusual.
  • Políticas de Retención: Implementa políticas de retención de registros para gestionar el almacenamiento y el cumplimiento.
  • Alertas: Configura alertas para eventos críticos como access_denied o authentication_failures.
  • Almacenamiento Seguro: Asegúrate de que los registros de auditoría se almacenen de manera segura para prevenir manipulaciones.

Conclusión

El registro de auditoría en Elasticsearch es una característica esencial para mantener la seguridad, el cumplimiento y la supervisión operativa. Al habilitar y configurar adecuadamente el registro de auditoría, puedes monitorear las actividades de los usuarios, detectar accesos no autorizados y asegurarte de que tu clúster de Elasticsearch opere dentro de los requisitos regulatorios. El análisis regular y las mejores prácticas te ayudarán a aprovechar al máximo las capacidades del registro de auditoría.
