Las plantillas de índice en Elasticsearch te permiten definir configuraciones, mapeos y alias que se aplicarán automáticamente a nuevos índices. Son una característica poderosa para gestionar y estandarizar la configuración de índices en tu clúster de Elasticsearch.

Conceptos Clave

  • Plantilla de Índice: Un conjunto de configuraciones que se pueden aplicar a nuevos índices.
  • Configuraciones: Opciones de configuración para un índice, como el número de fragmentos y réplicas.
  • Mapeos: Definen cómo se almacenan e indexan los documentos y sus campos.
  • Alias: Nombres alternativos para índices, que pueden simplificar las consultas y la gestión de índices.

Creando una Plantilla de Índice

Para crear una plantilla de índice, necesitas definirla usando un objeto JSON y enviarlo al endpoint _template. Aquí tienes un ejemplo básico:

PUT _template/my_template
{
  "index_patterns": ["log-*"],
  "settings": {
    "number_of_shards": 1
  },
  "mappings": {
    "properties": {
      "timestamp": {
        "type": "date"
      },
      "message": {
        "type": "text"
      }
    }
  },
  "aliases": {
    "logs": {}
  }
}

Explicación

  • index_patterns: Especifica los patrones de nombres de índice a los que se debe aplicar esta plantilla. En este caso, se aplica a cualquier índice que comience con "log-".
  • settings: Define las configuraciones del índice. Aquí, establecemos el número de fragmentos en 1.
  • mappings: Define los mapeos de campos. Tenemos dos campos: timestamp de tipo date y message de tipo text.
  • aliases: Define un alias llamado logs para los índices que coincidan con el patrón.

Aplicando Plantillas de Índice

Cuando creas un nuevo índice que coincide con el patrón especificado en la plantilla, Elasticsearch aplica automáticamente las configuraciones, mapeos y alias de la plantilla al nuevo índice.

Por ejemplo, creando un índice llamado log-2023-10-01:

PUT log-2023-10-01

Este índice heredará las configuraciones, mapeos y alias definidos en my_template.

Gestionando Plantillas de Índice

Listando Plantillas de Índice

Para listar todas las plantillas de índice, usa el siguiente comando:

GET _template

Recuperando una Plantilla Específica

Para recuperar una plantilla específica, usa:

GET _template/my_template

Actualizando una Plantilla de Índice

Para actualizar una plantilla existente, usa la misma solicitud PUT con el objeto JSON actualizado. Elasticsearch sobrescribirá la plantilla existente con la nueva definición.

Eliminando una Plantilla de Índice

Para eliminar una plantilla de índice, usa el siguiente comando:

DELETE _template/my_template

Ejemplo: Plantilla de Índice Avanzada

Aquí tienes un ejemplo más avanzado que incluye configuraciones y mapeos adicionales:

PUT _template/advanced_template
{
  "index_patterns": ["data-*"],
  "order": 1,
  "settings": {
    "number_of_shards": 3,
    "number_of_replicas": 2,
    "refresh_interval": "30s"
  },
  "mappings": {
    "properties": {
      "user_id": {
        "type": "keyword"
      },
      "created_at": {
        "type": "date",
        "format": "yyyy-MM-dd HH:mm:ss||epoch_millis"
      },
      "content": {
        "type": "text",
        "analyzer": "standard"
      }
    }
  },
  "aliases": {
    "data_alias": {}
  }
}

Explicación

  • order: Determina el orden de aplicación si coinciden múltiples plantillas. Los valores más altos tienen mayor precedencia.
  • refresh_interval: Establece el intervalo en el que se actualiza el índice.
  • mappings: Incluye un campo user_id de tipo keyword, un campo created_at con un formato de fecha personalizado y un campo content con un analizador estándar.

Conclusión

Las plantillas de índice en Elasticsearch son esenciales para gestionar y estandarizar la configuración de índices. Ayudan a asegurar la consistencia a través de los índices y simplifican el proceso de creación de índices. Al entender y utilizar plantillas de índice, puedes optimizar el rendimiento y la mantenibilidad de tu clúster de Elasticsearch.
