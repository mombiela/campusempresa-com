Introducción a Beats

Beats son agentes ligeros de envío de datos que instalas en tus servidores para enviar datos operativos a Elasticsearch. Están diseñados para ser simples y eficientes, facilitando la recolección y envío de datos desde diversas fuentes.

  • Filebeat: Envía archivos de registro.
  • Metricbeat: Recoge métricas del sistema operativo y servicios.
  • Packetbeat: Monitorea el tráfico de red.
  • Winlogbeat: Envía registros de eventos de Windows.
  • Auditbeat: Recoge datos de auditoría del marco de auditoría de Linux.
  • Heartbeat: Monitorea la disponibilidad de servicios.

Instalación y Configuración de Beats

Instalación de Filebeat

  1. Descargar e instalar Filebeat:

    curl -L -O https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-7.10.0-amd64.deb
sudo dpkg -i filebeat-7.10.0-amd64.deb

  2. Configurar Filebeat: Edita el archivo de configuración filebeat.yml para especificar los archivos de registro a monitorear y la instancia de Elasticsearch a la que enviar datos:

    filebeat.inputs:
- type: log
  enabled: true
  paths:
    - /var/log/*.log

output.elasticsearch:
  hosts: ["localhost:9200"]

  3. Iniciar Filebeat:

    sudo service filebeat start

Instalación de Metricbeat

  1. Descargar e instalar Metricbeat:

    curl -L -O https://artifacts.elastic.co/downloads/beats/metricbeat/metricbeat-7.10.0-amd64.deb
sudo dpkg -i metricbeat-7.10.0-amd64.deb

  2. Configurar Metricbeat: Edita el archivo de configuración metricbeat.yml para especificar los módulos a habilitar y la instancia de Elasticsearch a la que enviar datos:

    metricbeat.modules:
- module: system
  metricsets:
    - cpu
    - memory
    - network
  enabled: true
  period: 10s
  processes: ['.*']

output.elasticsearch:
  hosts: ["localhost:9200"]

  3. Iniciar Metricbeat:

    sudo service metricbeat start

Usando Beats con Elasticsearch

Enviando Datos a Elasticsearch

Beats pueden ser configurados para enviar datos directamente a Elasticsearch o a través de Logstash para procesamiento adicional.

  • Directo a Elasticsearch: Esta es la configuración más simple donde Beats envían datos directamente a Elasticsearch.
  • A través de Logstash: Esta configuración permite un procesamiento y transformación de datos más complejos antes de que los datos sean indexados en Elasticsearch.

Ejemplo: Filebeat a Elasticsearch

  1. Configurar Filebeat:

    filebeat.inputs:
- type: log
  enabled: true
  paths:
    - /var/log/*.log

output.elasticsearch:
  hosts: ["localhost:9200"]

  2. Iniciar Filebeat:

    sudo service filebeat start

Ejemplo: Metricbeat a Elasticsearch

  1. Configurar Metricbeat:

    metricbeat.modules:
- module: system
  metricsets:
    - cpu
    - memory
    - network
  enabled: true
  period: 10s
  processes: ['.*']

output.elasticsearch:
  hosts: ["localhost:9200"]

  2. Iniciar Metricbeat:

    sudo service metricbeat start

Monitoreo y Visualización de Datos

Usando Kibana

Kibana es la herramienta de visualización para Elasticsearch. Te permite crear paneles y visualizaciones basadas en los datos recolectados por Beats.

  1. Acceder a Kibana: Abre tu navegador y navega a http://localhost:5601.

  2. Crear Patrones de Índice:

    • Ve a la sección "Management".
    • Crea un patrón de índice para los datos recolectados por Beats (por ejemplo, filebeat-*, metricbeat-*).

  3. Crear Visualizaciones:

    • Usa la sección "Visualize" para crear visualizaciones basadas en los datos.
    • Combina visualizaciones en paneles para un monitoreo integral.

Ejemplo: Creando un Panel

  1. Crear Visualizaciones:

    • Crea visualizaciones para el uso de CPU, uso de memoria y tráfico de red usando datos de Metricbeat.

  2. Combinar Visualizaciones en un Panel:

    • Ve a la sección "Dashboard".
    • Añade las visualizaciones creadas en el paso anterior a un nuevo panel.

Conclusión

Beats son herramientas poderosas para recolectar y enviar datos a Elasticsearch. Son ligeras y fáciles de configurar, lo que las hace ideales para monitorear varios aspectos de tu infraestructura. Al integrar Beats con Elasticsearch y Kibana, puedes crear soluciones de monitoreo completas que proporcionan valiosos conocimientos sobre tus sistemas y aplicaciones.
