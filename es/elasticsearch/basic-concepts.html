Introducción

Elasticsearch es un motor de búsqueda y análisis poderoso que se utiliza ampliamente para una variedad de aplicaciones. Comprender sus componentes principales—nodos, clústeres e índices—es esencial para usar y gestionar Elasticsearch de manera efectiva. Esta sección cubrirá estos conceptos básicos en detalle.

Nodos

Un nodo es una instancia única de Elasticsearch. Es donde se almacena realmente la información y es responsable de realizar varias operaciones como indexar, buscar y agregar datos.

Conceptos Clave

  • Tipos de Nodos:
    • Nodo Maestro: Gestiona el clúster manejando tareas como la creación/eliminación de índices y el seguimiento de nodos.
    • Nodo de Datos: Almacena datos y realiza operaciones relacionadas con los datos como CRUD, búsqueda y agregaciones.
    • Nodo de Ingesta: Preprocesa documentos antes de indexarlos.
    • Nodo de Coordinación: Enruta solicitudes a los nodos apropiados y agrega resultados.

Ejemplo

Para iniciar un nodo, puedes usar el siguiente comando:

./bin/elasticsearch

Este comando inicia un nodo de Elasticsearch con configuraciones predeterminadas.

Clústeres

Un clúster es una colección de uno o más nodos que juntos contienen todos tus datos y proporcionan capacidades de indexación y búsqueda federadas a través de todos los nodos.

Conceptos Clave

  • Nombre del Clúster: Cada clúster se identifica por un nombre único. Los nodos con el mismo nombre de clúster forman un clúster.
  • Estado del Clúster: El estado del clúster incluye metadatos sobre el clúster, como la lista de nodos, índices y sus configuraciones.

Ejemplo

Para configurar un nodo para que se una a un clúster específico, puedes establecer el nombre del clúster en el archivo de configuración elasticsearch.yml:

cluster.name: my_cluster

Índices

Un índice es una colección de documentos que tienen características similares. En Elasticsearch, un índice es similar a una base de datos en un sistema de bases de datos relacional.

Conceptos Clave

  • Documento: La unidad básica de información que puede ser indexada. Se representa en formato JSON.
  • Fragmento: Un índice puede dividirse en múltiples piezas llamadas fragmentos. Cada fragmento es un "sub-índice" completamente funcional e independiente que puede alojarse en cualquier nodo del clúster.
  • Réplica: Cada fragmento puede tener cero o más réplicas. Una réplica es una copia del fragmento y proporciona redundancia y alta disponibilidad.

Ejemplo

Para crear un índice, puedes usar la siguiente llamada a la API REST:

PUT /my_index

Este comando crea un índice llamado my_index.

Resumen

Comprender los conceptos básicos de nodos, clústeres e índices es crucial para trabajar con Elasticsearch. Los nodos son las instancias individuales que almacenan y gestionan datos, los clústeres son grupos de nodos que trabajan juntos, y los índices son colecciones de documentos que se indexan y buscan. Al dominar estos elementos fundamentales, estarás bien preparado para profundizar en las capacidades de Elasticsearch y gestionar tus datos de manera efectiva.

En las próximas secciones, exploraremos temas más avanzados como consultas, estrategias de indexación y ajuste de rendimiento.
