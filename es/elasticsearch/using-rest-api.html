Introducción

Elasticsearch es un potente motor de búsqueda y análisis que proporciona una API RESTful para interactuar con tus datos. Esta sección cubrirá los conceptos básicos del uso de la API REST de Elasticsearch, incluyendo cómo realizar operaciones CRUD, consultas de búsqueda y funcionalidades más avanzadas.

Conceptos Clave

  • API REST: Un conjunto de servicios web que te permiten interactuar con Elasticsearch utilizando métodos HTTP.
  • Operaciones CRUD: Operaciones de Crear, Leer, Actualizar y Eliminar documentos.
  • Índice: Una colección de documentos en Elasticsearch.
  • Documento: Un registro individual en un índice de Elasticsearch.
  • Endpoint: Una ruta URL específica que corresponde a una operación en la API de Elasticsearch.

Configuración

Antes de poder usar la API REST de Elasticsearch, necesitas tener Elasticsearch instalado y en funcionamiento. Puedes interactuar con la API usando herramientas como curl, Postman, o cualquier cliente HTTP en tu lenguaje de programación preferido.

Operaciones CRUD Básicas

Creación de un Índice

Para crear un índice, utilizas el método PUT. Aquí tienes un ejemplo de cómo crear un índice llamado my_index:

PUT /my_index

Añadir un Documento

Para añadir un documento a un índice, utilizas el método POST. Aquí tienes un ejemplo de cómo añadir un documento a my_index:

POST /my_index/_doc/1
{
  "name": "John Doe",
  "age": 30,
  "occupation": "Software Engineer"
}

Recuperar un Documento

Para recuperar un documento, utilizas el método GET. Aquí tienes un ejemplo de cómo recuperar el documento con ID 1 de my_index:

GET /my_index/_doc/1

Actualizar un Documento

Para actualizar un documento, utilizas el método POST con el endpoint _update. Aquí tienes un ejemplo de cómo actualizar el documento con ID 1:

POST /my_index/_doc/1/_update
{
  "doc": {
    "age": 31
  }
}

Eliminar un Documento

Para eliminar un documento, utilizas el método DELETE. Aquí tienes un ejemplo de cómo eliminar el documento con ID 1:

DELETE /my_index/_doc/1

Búsqueda de Documentos

Búsqueda Básica

Para realizar una búsqueda básica, utilizas el endpoint _search con el método GET. Aquí tienes un ejemplo de cómo buscar documentos en my_index donde el campo name sea John Doe:

GET /my_index/_search
{
  "query": {
    "match": {
      "name": "John Doe"
    }
  }
}

Búsqueda Avanzada

Elasticsearch soporta consultas de búsqueda más avanzadas, como consultas booleanas, consultas de rango y agregaciones. Aquí tienes un ejemplo de una consulta booleana:

GET /my_index/_search
{
  "query": {
    "bool": {
      "must": [
        { "match": { "occupation": "Software Engineer" } }
      ],
      "filter": [
        { "range": { "age": { "gte": 30 } } }
      ]
    }
  }
}

Gestión de Índices

Comprobar el Estado del Índice

Para comprobar el estado de un índice, utilizas el endpoint _stats. Aquí tienes un ejemplo:

GET /my_index/_stats

Eliminar un Índice

Para eliminar un índice, utilizas el método DELETE. Aquí tienes un ejemplo:

DELETE /my_index

Conclusión

La API REST de Elasticsearch proporciona una forma poderosa y flexible de interactuar con tus datos. Al entender las operaciones CRUD básicas, las funcionalidades de búsqueda y la gestión de índices, puedes usar efectivamente Elasticsearch para almacenar, recuperar y analizar tus datos. A medida que te familiarices más con la API, puedes explorar características más avanzadas y optimizaciones para mejorar aún más tu experiencia con Elasticsearch.
