Introducción

Diseñar una base de datos es un paso crítico en la creación de una aplicación robusta y eficiente. En esta sección, cubriremos los conceptos fundamentales y las mejores prácticas para diseñar una base de datos en PostgreSQL. Esto incluirá la comprensión del modelado de datos, la normalización y la creación de tablas con tipos de datos y restricciones apropiados.

Conceptos Clave

Modelado de Datos

  • Entidades y Relaciones: Identificar las principales entidades (tablas) y sus relaciones.
  • Diagramas ER: Utilizar diagramas de Entidad-Relación para visualizar la estructura de la base de datos.

Normalización

  • Primera Forma Normal (1NF): Asegurarse de que cada columna contenga valores atómicos y cada registro sea único.
  • Segunda Forma Normal (2NF): Asegurarse de que todos los atributos no clave dependan completamente de la clave primaria.
  • Tercera Forma Normal (3NF): Asegurarse de que todos los atributos dependan únicamente de la clave primaria.

Tipos de Datos

  • Tipos de Datos Primitivos: Entero, Booleano, Texto, etc.
  • Tipos de Datos Compuestos: Arrays, JSON, etc.
  • Tipos de Datos Personalizados: Creación de tipos definidos por el usuario.

Restricciones

  • Clave Primaria: Identifica de manera única cada registro en una tabla.
  • Clave Foránea: Asegura la integridad referencial entre tablas.
  • Única: Asegura que todos los valores en una columna sean únicos.
  • No Nulo: Asegura que una columna no pueda tener valores NULL.
  • Chequeo: Asegura que todos los valores en una columna cumplan una condición específica.

Creación de Tablas

Creación Básica de Tablas

Para crear una tabla en PostgreSQL, se utiliza la instrucción CREATE TABLE. Aquí hay un ejemplo:

CREATE TABLE users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
  • user_id SERIAL PRIMARY KEY: Crea una clave primaria auto-incremental.
  • username VARCHAR(50) UNIQUE NOT NULL: Asegura que el nombre de usuario sea único y no nulo.
  • email VARCHAR(100) UNIQUE NOT NULL: Asegura que el correo electrónico sea único y no nulo.
  • created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP: Establece una marca de tiempo predeterminada para cuando se crea el registro.

Añadiendo Restricciones

Las restricciones pueden añadirse en el momento de la creación de la tabla o posteriormente usando la instrucción ALTER TABLE.

ALTER TABLE users
ADD CONSTRAINT email_check CHECK (email LIKE '%@%');

Esto asegura que la columna de correo electrónico contenga un carácter '@'.

Claves Foráneas

Las claves foráneas se utilizan para vincular dos tablas. Aquí hay un ejemplo:

CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
  • user_id INT REFERENCES users(user_id): Crea una clave foránea que referencia la columna user_id en la tabla users.

Ejemplo: Diseñando una Base de Datos Simple de Comercio Electrónico

Diagrama ER

Antes de crear tablas, es útil dibujar un diagrama ER. Para una base de datos simple de comercio electrónico, podríamos tener las siguientes entidades:

  • Usuarios
  • Productos
  • Pedidos
  • Artículos de Pedido

Creación de Tablas

CREATE TABLE products (
    product_id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    price NUMERIC(10, 2) NOT NULL,
    stock INT NOT NULL
);

CREATE TABLE orders (
    order_id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE order_items (
    order_item_id SERIAL PRIMARY KEY,
    order_id INT REFERENCES orders(order_id),
    product_id INT REFERENCES products(product_id),
    quantity INT NOT NULL,
    price NUMERIC(10, 2) NOT NULL
);

Explicación

  • Tabla de Productos: Contiene detalles sobre los productos.
  • Tabla de Pedidos: Contiene detalles sobre los pedidos realizados por los usuarios.
  • Tabla de Artículos de Pedido: Contiene detalles sobre los artículos en cada pedido, vinculando pedidos y productos.

Conclusión

Diseñar una base de datos implica una planificación cuidadosa y comprensión de los datos y sus relaciones. Siguiendo las mejores prácticas como la normalización y utilizando tipos de datos y restricciones apropiados, puedes crear una base de datos robusta y eficiente. En PostgreSQL, las instrucciones CREATE TABLE y ALTER TABLE son herramientas fundamentales para definir y modificar el esquema de tu base de datos.
