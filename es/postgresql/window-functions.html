Las funciones de ventana son una característica poderosa en PostgreSQL que te permiten realizar cálculos a través de un conjunto de filas de una tabla que están de alguna manera relacionadas con la fila actual. Son particularmente útiles para tareas como clasificación, totales acumulados, promedios móviles y más.

Introducción a las Funciones de Ventana

  • Definición: Las funciones de ventana realizan cálculos a través de un conjunto de filas de una tabla que están relacionadas con la fila actual.
  • Casos de Uso: Clasificación, totales acumulados, promedios móviles, sumas acumulativas, etc.
  • Sintaxis: La sintaxis básica para una función de ventana es: 
    function_name(expression) OVER (
  [PARTITION BY partition_expression]
  [ORDER BY sort_expression]
  [frame_clause]
)

Componentes Básicos de las Funciones de Ventana

Cláusula PARTITION BY

  • Propósito: Divide el conjunto de resultados en particiones a las que se aplica la función de ventana.
  • Ejemplo: 
    SELECT
  department,
  employee,
  salary,
  SUM(salary) OVER (PARTITION BY department) AS department_total
FROM
  employees;
    • Explicación: Esta consulta calcula el salario total para cada departamento.

Cláusula ORDER BY

  • Propósito: Define el orden de las filas dentro de cada partición.
  • Ejemplo: 
    SELECT
  department,
  employee,
  salary,
  RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank
FROM
  employees;
    • Explicación: Esta consulta clasifica a los empleados dentro de cada departamento según su salario en orden descendente.

Cláusula Frame

  • Propósito: Define un subconjunto de filas dentro de la partición para que la función de ventana opere sobre él.
  • Tipos:
    • ROWS: Especifica un marco basado en conteos físicos de filas.
    • RANGE: Especifica un marco basado en rangos lógicos de valores.
  • Ejemplo: 
    SELECT
  employee,
  salary,
  SUM(salary) OVER (ORDER BY salary ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING) AS moving_sum
FROM
  employees;
    • Explicación: Esta consulta calcula una suma móvil de salarios, incluyendo la fila actual, la fila anterior y la fila siguiente.

Funciones de Ventana Comunes

Funciones de Agregación

  • SUM(): Calcula la suma de valores.
  • AVG(): Calcula el promedio de valores.
  • MIN(): Encuentra el valor mínimo.
  • MAX(): Encuentra el valor máximo.
  • COUNT(): Cuenta el número de valores.

Funciones de Clasificación

  • RANK(): Asigna un rango a cada fila dentro de la partición de un conjunto de resultados.
  • DENSE_RANK(): Similar a RANK(), pero sin huecos en los valores de clasificación.
  • ROW_NUMBER(): Asigna un número único a cada fila dentro de la partición.

Funciones de Valor

  • LAG(): Accede a datos de una fila anterior en el mismo conjunto de resultados.
  • LEAD(): Accede a datos de una fila posterior en el mismo conjunto de resultados.
  • FIRST_VALUE(): Devuelve el primer valor en un conjunto ordenado de valores.
  • LAST_VALUE(): Devuelve el último valor en un conjunto ordenado de valores.

Ejemplos Avanzados

Total Acumulado

SELECT
  employee,
  salary,
  SUM(salary) OVER (ORDER BY employee) AS running_total
FROM
  employees;
  • Explicación: Esta consulta calcula un total acumulado de salarios ordenados por empleado.

Promedio Móvil

SELECT
  employee,
  salary,
  AVG(salary) OVER (ORDER BY employee ROWS BETWEEN 2 PRECEDING AND 2 FOLLOWING) AS moving_avg
FROM
  employees;
  • Explicación: Esta consulta calcula un promedio móvil de salarios, considerando la fila actual y dos filas anteriores y dos filas siguientes.

Distribución Acumulativa

SELECT
  employee,
  salary,
  CUME_DIST() OVER (ORDER BY salary) AS cumulative_distribution
FROM
  employees;
  • Explicación: Esta consulta calcula la distribución acumulativa de salarios.

Conclusión

Las funciones de ventana en PostgreSQL proporcionan una forma versátil y poderosa de realizar cálculos complejos a través de conjuntos de filas de una tabla. Al comprender y utilizar las cláusulas PARTITION BY, ORDER BY y frame, puedes desbloquear nuevas posibilidades para el análisis y la generación de informes de datos. Practica usando estas funciones con diferentes conjuntos de datos para volverte competente en su aplicación.
