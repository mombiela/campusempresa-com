Introducción

Las funciones definidas por el usuario (UDFs) en PostgreSQL te permiten encapsular lógica de código reutilizable dentro de la base de datos. Esto puede ayudar a mejorar la organización del código, la mantenibilidad y el rendimiento. En esta sección, cubriremos los conceptos básicos de la creación y uso de UDFs, progresando desde ejemplos simples hasta casos de uso más avanzados.

Conceptos Clave

  • Definición de Función: La sintaxis y estructura para definir una función.
  • Parámetros de Función: Cómo pasar valores de entrada a las funciones.
  • Tipos de Retorno: Diferentes tipos de valores que las funciones pueden devolver.
  • Opciones de Lenguaje: Varios lenguajes soportados para escribir funciones.
  • Manejo de Errores: Técnicas para gestionar errores dentro de las funciones.

Creando una Función Simple

Para crear una función en PostgreSQL, usas la declaración CREATE FUNCTION. Aquí hay un ejemplo básico:

CREATE FUNCTION add_numbers(a INTEGER, b INTEGER) 
RETURNS INTEGER AS $$
BEGIN
    RETURN a + b;
END;
$$ LANGUAGE plpgsql;

Explicación:

  • CREATE FUNCTION add_numbers(a INTEGER, b INTEGER): Define una función llamada add_numbers con dos parámetros enteros, a y b.
  • RETURNS INTEGER: Especifica que la función devuelve un entero.
  • AS $$ ... $$ LANGUAGE plpgsql: El cuerpo de la función está encerrado en $$ y escrito en el lenguaje plpgsql.

Uso:

SELECT add_numbers(5, 3);  -- Devuelve 8

Parámetros de Función

Las funciones pueden tomar diferentes tipos de parámetros, incluyendo valores predeterminados y parámetros variádicos.

Parámetros Predeterminados:

CREATE FUNCTION greet(name TEXT DEFAULT 'World') 
RETURNS TEXT AS $$
BEGIN
    RETURN 'Hello, ' || name || '!';
END;
$$ LANGUAGE plpgsql;

Uso:

SELECT greet();         -- Devuelve 'Hello, World!'
SELECT greet('Alice');  -- Devuelve 'Hello, Alice!'

Parámetros Variádicos:

CREATE FUNCTION sum_numbers(VARIADIC nums INTEGER[]) 
RETURNS INTEGER AS $$
DECLARE
    total INTEGER := 0;
BEGIN
    FOREACH num IN ARRAY nums
    LOOP
        total := total + num;
    END LOOP;
    RETURN total;
END;
$$ LANGUAGE plpgsql;

Uso:

SELECT sum_numbers(1, 2, 3, 4);  -- Devuelve 10

Tipos de Retorno

Las funciones pueden devolver varios tipos de valores, incluyendo valores escalares, tipos compuestos y conjuntos.

Tipo de Retorno Escalar:

CREATE FUNCTION multiply(a INTEGER, b INTEGER) 
RETURNS INTEGER AS $$
BEGIN
    RETURN a * b;
END;
$$ LANGUAGE plpgsql;

Tipo de Retorno Compuesto:

CREATE TYPE person AS (first_name TEXT, last_name TEXT);

CREATE FUNCTION get_person() 
RETURNS person AS $$
BEGIN
    RETURN ('John', 'Doe');
END;
$$ LANGUAGE plpgsql;

Tipo de Retorno de Conjunto:

CREATE FUNCTION generate_series(start INTEGER, stop INTEGER) 
RETURNS SETOF INTEGER AS $$
BEGIN
    FOR i IN start..stop LOOP
        RETURN NEXT i;
    END LOOP;
    RETURN;
END;
$$ LANGUAGE plpgsql;

Uso:

SELECT * FROM generate_series(1, 5);  -- Devuelve 1, 2, 3, 4, 5

Opciones de Lenguaje

PostgreSQL soporta múltiples lenguajes para escribir funciones, incluyendo SQL, PL/pgSQL, PL/Python, PL/Perl, y más.

Lenguaje SQL:

CREATE FUNCTION add_sql(a INTEGER, b INTEGER) 
RETURNS INTEGER AS $$
    SELECT a + b;
$$ LANGUAGE sql;

Lenguaje PL/Python:

CREATE FUNCTION add_python(a INTEGER, b INTEGER) 
RETURNS INTEGER AS $$
    return a + b
$$ LANGUAGE plpythonu;

Manejo de Errores

El manejo de errores en funciones es crucial para aplicaciones robustas. Puedes usar bloques EXCEPTION para capturar y manejar errores.

Ejemplo:

CREATE FUNCTION divide(a INTEGER, b INTEGER) 
RETURNS INTEGER AS $$
BEGIN
    RETURN a / b;
EXCEPTION
    WHEN division_by_zero THEN
        RETURN NULL;
END;
$$ LANGUAGE plpgsql;

Uso:

SELECT divide(10, 2);  -- Devuelve 5
SELECT divide(10, 0);  -- Devuelve NULL

Conclusión

Las funciones definidas por el usuario en PostgreSQL proporcionan una forma poderosa de encapsular y reutilizar lógica dentro de tu base de datos. Al entender los conceptos básicos de la creación de funciones, manejo de parámetros, tipos de retorno, opciones de lenguaje y manejo de errores, puedes escribir código de base de datos eficiente y mantenible. Practica creando y usando funciones para volverte competente en aprovechar esta característica en PostgreSQL.
