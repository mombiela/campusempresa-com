Introducción

En este estudio de caso, exploraremos cómo diseñar e implementar una base de datos de comercio electrónico utilizando PostgreSQL. Esto cubrirá todo, desde la configuración de la base de datos hasta técnicas avanzadas de consulta. Al final de este tema, deberías tener una comprensión completa de cómo gestionar una base de datos de comercio electrónico de manera efectiva.

Configuración de la Base de Datos

Creación de la Base de Datos

Primero, necesitamos crear una nueva base de datos para nuestra aplicación de comercio electrónico.

CREATE DATABASE ecommerce_db;

Conectarse a la Base de Datos

A continuación, conéctate a la base de datos recién creada.

\c ecommerce_db;

Diseño del Esquema

Tablas y Relaciones

Una base de datos de comercio electrónico típicamente incluye varias tablas clave:

  • Usuarios: Información sobre los clientes.
  • Productos: Detalles de los artículos disponibles para la venta.
  • Pedidos: Registros de compras de clientes.
  • Elementos de Pedido: Artículos incluidos en cada pedido.
  • Categorías: Categorías de productos.

Creación de Tablas

Vamos a crear estas tablas con las relaciones apropiadas.

Tabla de Usuarios

CREATE TABLE Users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Tabla de Productos

CREATE TABLE Products (
    product_id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    price DECIMAL(10, 2) NOT NULL,
    category_id INT,
    FOREIGN KEY (category_id) REFERENCES Categories(category_id)
);

Tabla de Pedidos

CREATE TABLE Orders (
    order_id SERIAL PRIMARY KEY,
    user_id INT NOT NULL,
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(50) NOT NULL,
    FOREIGN KEY (user_id) REFERENCES Users(user_id)
);

Tabla de Elementos de Pedido

CREATE TABLE OrderItems (
    order_item_id SERIAL PRIMARY KEY,
    order_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (order_id) REFERENCES Orders(order_id),
    FOREIGN KEY (product_id) REFERENCES Products(product_id)
);

Tabla de Categorías

CREATE TABLE Categories (
    category_id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

Población de la Base de Datos

Inserción de Datos

Vamos a insertar algunos datos de muestra en nuestras tablas.

Insertar Categorías

INSERT INTO Categories (name) VALUES ('Electrónica'), ('Libros'), ('Ropa');

INSERT INTO Products (name, description, price, category_id) VALUES

INSERT INTO Orders (user_id, status) VALUES

INSERT INTO OrderItems (order_id, product_id, quantity, price) VALUES

Consultas a la Base de Datos

Consultas Básicas

Recuperar todos los usuarios:

SELECT * FROM Users;

Recuperar todos los productos en la categoría 'Electrónica':

SELECT * FROM Products WHERE category_id = 1;

Consultas Avanzadas

Recuperar todos los pedidos junto con la información del usuario:

SELECT Orders.order_id, Users.username, Orders.order_date, Orders.status
FROM Orders
JOIN Users ON Orders.user_id = Users.user_id;

Recuperar las ventas totales de cada producto:

SELECT Products.name, SUM(OrderItems.quantity * OrderItems.price) AS total_sales
FROM OrderItems
JOIN Products ON OrderItems.product_id = Products.product_id
GROUP BY Products.name;
