Introducción

En esta sección, cubriremos los pasos necesarios para implementar una base de datos utilizando PostgreSQL. Esto incluye crear bases de datos, definir esquemas y poblar tablas con datos. Al final de esta sección, tendrás una comprensión sólida de cómo configurar y gestionar una base de datos PostgreSQL.

Creación de una Base de Datos

Para empezar a usar PostgreSQL, primero necesitas crear una base de datos. Aquí te mostramos cómo hacerlo:

Usando el Comando `createdb`

El comando createdb es una forma conveniente de crear una nueva base de datos desde la línea de comandos.

createdb mibasededatos

Usando Comandos SQL

También puedes crear una base de datos usando comandos SQL dentro de la interfaz psql.

CREATE DATABASE mibasededatos;
  • Nota: Asegúrate de tener los permisos necesarios para crear una base de datos.

Conectarse a una Base de Datos

Una vez que la base de datos está creada, necesitas conectarte a ella para empezar a trabajar.

Usando el Comando `psql`

Puedes conectarte a tu base de datos usando la herramienta de línea de comandos psql.

psql mibasededatos

Usando Comandos SQL

Dentro de psql, puedes conectarte a una base de datos usando el comando \c.

\c mibasededatos

Definiendo Esquemas

Los esquemas son una forma de organizar y gestionar tus objetos de base de datos. Por defecto, PostgreSQL crea un esquema llamado public.

Creación de un Esquema

Puedes crear un nuevo esquema usando el comando CREATE SCHEMA.

CREATE SCHEMA miesquema;

Configuración del Path de Búsqueda

Puedes configurar el path de búsqueda para especificar en qué esquemas PostgreSQL debe buscar primero.

SET search_path TO miesquema, public;

Creación de Tablas

Las tablas son los bloques fundamentales de una base de datos. Almacenan datos en filas y columnas.

Creación Básica de una Tabla

Aquí te mostramos cómo puedes crear una tabla simple:

CREATE TABLE usuarios (
    id SERIAL PRIMARY KEY,
    nombre_usuario VARCHAR(50) NOT NULL,
    correo VARCHAR(100) NOT NULL,
    creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
  • Explicación:
    • id SERIAL PRIMARY KEY: Crea una clave primaria auto-incremental.
    • nombre_usuario VARCHAR(50) NOT NULL: Define una columna de nombre de usuario que no puede ser nula.
    • correo VARCHAR(100) NOT NULL: Define una columna de correo que no puede ser nula.
    • creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP: Establece una marca de tiempo por defecto para cuando se crea la fila.

Población de Tablas con Datos

Una vez que tus tablas están creadas, puedes empezar a insertar datos en ellas.

Inserción de Datos

Usa el comando INSERT INTO para añadir datos a tu tabla.

INSERT INTO usuarios (nombre_usuario, correo) VALUES ('juan_perez', 'juan@ejemplo.com');

Inserción en Bloque

Para insertar múltiples filas a la vez, puedes usar múltiples cláusulas VALUES.

INSERT INTO usuarios (nombre_usuario, correo) VALUES 
('ana_garcia', 'ana@ejemplo.com'),
('maria', 'maria@ejemplo.com'),
('carlos', 'carlos@ejemplo.com');

Consulta de Datos

Recuperar datos de tus tablas se hace usando la sentencia SELECT.

Consulta Básica

Aquí tienes una consulta simple para recuperar todos los usuarios:

SELECT * FROM usuarios;

Filtrado de Datos

Usa la cláusula WHERE para filtrar resultados.

SELECT * FROM usuarios WHERE nombre_usuario = 'juan_perez';

Ordenación de Datos

Usa la cláusula ORDER BY para ordenar los resultados.

SELECT * FROM usuarios ORDER BY creado_en DESC;

Actualización de Datos

Para modificar datos existentes, usa la sentencia UPDATE.

Actualización Básica

Aquí te mostramos cómo puedes actualizar el correo de un usuario:

UPDATE usuarios SET correo = 'nuevo_correo@ejemplo.com' WHERE nombre_usuario = 'juan_perez';

Eliminación de Datos

Para eliminar datos de tu tabla, usa la sentencia DELETE.

Eliminación Básica

Aquí te mostramos cómo puedes eliminar un usuario:

DELETE FROM usuarios WHERE nombre_usuario = 'juan_perez';

Conclusión

En esta sección, cubrimos los pasos esenciales para implementar una base de datos en PostgreSQL. Aprendiste cómo crear y conectarte a una base de datos, definir esquemas, crear y poblar tablas, y realizar operaciones básicas de manipulación de datos. Con estas habilidades, ahora estás equipado para configurar y gestionar una base de datos PostgreSQL de manera efectiva.
