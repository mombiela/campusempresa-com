Los Envoltorios de Datos Externos (FDWs) son una característica poderosa en PostgreSQL que te permite acceder y manipular datos almacenados en bases de datos externas como si fueran parte de tu base de datos PostgreSQL. Este tema te guiará desde los conceptos básicos hasta el uso avanzado de los FDWs.

Introducción a los Envoltorios de Datos Externos

  • Definición: Los Envoltorios de Datos Externos son una característica de PostgreSQL que te permite consultar e interactuar con fuentes de datos externas.
  • Casos de Uso:
    • Integrar datos de diferentes bases de datos.
    • Acceder a datos de fuentes no relacionales.
    • Realizar consultas federadas a través de múltiples fuentes de datos.

Configuración de los Envoltorios de Datos Externos

Instalación de la Extensión Requerida

Para usar los FDWs, necesitas instalar la extensión apropiada. Por ejemplo, para conectarte a otra base de datos PostgreSQL, usarías la extensión postgres_fdw.

CREATE EXTENSION IF NOT EXISTS postgres_fdw;

Creación de un Servidor Externo

Un servidor externo define los detalles de conexión a la fuente de datos externa.

CREATE SERVER foreign_server
FOREIGN DATA WRAPPER postgres_fdw
OPTIONS (host 'remote_host', dbname 'remote_db', port '5432');

Creación de Mapeos de Usuario

Los mapeos de usuario definen los detalles de autenticación para acceder al servidor externo.

CREATE USER MAPPING FOR local_user
SERVER foreign_server
OPTIONS (user 'remote_user', password 'remote_password');

Creación de Tablas Externas

Las tablas externas representan los datos externos dentro de tu base de datos PostgreSQL.

CREATE FOREIGN TABLE foreign_table (
    id integer,
    name text,
    value numeric
)
SERVER foreign_server
OPTIONS (schema_name 'public', table_name 'remote_table');

Consultando Tablas Externas

Una vez que la tabla externa está configurada, puedes consultarla como cualquier otra tabla en tu base de datos PostgreSQL.

SELECT * FROM foreign_table;

Uso Avanzado

Importación de Esquema Externo

Puedes importar un esquema completo desde el servidor externo.

IMPORT FOREIGN SCHEMA public
FROM SERVER foreign_server
INTO local_schema;

Uniendo Tablas Locales y Externas

Puedes realizar uniones entre tablas locales y externas.

SELECT local_table.id, local_table.name, foreign_table.value
FROM local_table
JOIN foreign_table ON local_table.id = foreign_table.id;

Consideraciones de Rendimiento

  • Capacidades de Pushdown: Los FDWs pueden delegar ciertas operaciones (como filtros y uniones) al servidor externo para mejorar el rendimiento.
  • Latencia de Red: Ten en cuenta la latencia de red al consultar grandes conjuntos de datos desde servidores remotos.

Envoltorios de Datos Externos Soportados

PostgreSQL soporta una variedad de FDWs para diferentes fuentes de datos. Aquí hay una comparación de algunos FDWs populares:

| FDW | Fuente de Datos | Caso de Uso | |-----------------|-------------------|-------------------------------------------| | postgres_fdw | PostgreSQL | Acceder a otras bases de datos PostgreSQL | | mysql_fdw | MySQL | Integrar bases de datos MySQL | | oracle_fdw | Oracle | Acceder a bases de datos Oracle | | file_fdw | CSV/Archivos Planos| Leer datos de archivos CSV o planos | | mongodb_fdw | MongoDB | Integrar colecciones de MongoDB |

Conclusión

Los Envoltorios de Datos Externos son una herramienta versátil en PostgreSQL que permite una integración fluida con diversas fuentes de datos externas. Al entender cómo configurar y usar los FDWs, puedes extender las capacidades de tu base de datos PostgreSQL y realizar consultas complejas a través de diferentes sistemas de datos. Ya sea que estés uniendo datos de múltiples bases de datos o accediendo a datos no relacionales, los FDWs proporcionan una solución robusta para tus necesidades de integración de datos.
