Introducción a PL/pgSQL

PL/pgSQL (Lenguaje Procedural/PostgreSQL) es un lenguaje procedural soportado por PostgreSQL. Te permite escribir funciones y disparadores complejos en PostgreSQL, mejorando las capacidades de SQL.

  • Conceptos Clave:
    • Lenguaje Procedural: Extiende SQL con construcciones procedurales.
    • Funciones: Bloques de código reutilizables que pueden tomar parámetros y devolver un valor.
    • Disparadores: Tipo especial de procedimiento almacenado que se ejecuta automáticamente cuando ocurren ciertos eventos.

Configuración de PL/pgSQL

Antes de comenzar a escribir código PL/pgSQL, asegúrate de que el lenguaje esté habilitado en tu base de datos PostgreSQL.

CREATE EXTENSION IF NOT EXISTS plpgsql;

Escribiendo Funciones en PL/pgSQL

Las funciones en PL/pgSQL se definen usando la declaración CREATE FUNCTION. Pueden tomar parámetros, ejecutar comandos SQL y devolver resultados.

Ejemplo Básico de Función

Aquí hay una función simple que toma dos enteros y devuelve su suma:

CREATE OR REPLACE FUNCTION add_numbers(a INTEGER, b INTEGER)
RETURNS INTEGER AS $$
BEGIN
    RETURN a + b;
END;
$$ LANGUAGE plpgsql;
  • Explicación:
    • CREATE OR REPLACE FUNCTION: Define una nueva función o reemplaza una existente.
    • RETURNS INTEGER: Especifica el tipo de retorno de la función.
    • $$ ... $$: Delimitadores para el cuerpo de la función.
    • BEGIN ... END;: Bloque donde se escribe la lógica de la función.
    • RETURN a + b;: Devuelve la suma de los dos parámetros de entrada.

Llamando a una Función

Puedes llamar a la función usando una declaración SELECT:

SELECT add_numbers(5, 3);

Estructuras de Control

PL/pgSQL soporta varias estructuras de control como declaraciones condicionales y bucles.

Declaraciones Condicionales

Usa IF, ELSIF y ELSE para ejecutar código condicionalmente.

CREATE OR REPLACE FUNCTION check_number(n INTEGER)
RETURNS TEXT AS $$
BEGIN
    IF n > 0 THEN
        RETURN 'Positive';
    ELSIF n < 0 THEN
        RETURN 'Negative';
    ELSE
        RETURN 'Zero';
    END IF;
END;
$$ LANGUAGE plpgsql;

Bucles

PL/pgSQL proporciona varios tipos de bucles: LOOP, WHILE, FOR.

Ejemplo: Bucle FOR

CREATE OR REPLACE FUNCTION sum_series(n INTEGER)
RETURNS INTEGER AS $$
DECLARE
    total INTEGER := 0;
BEGIN
    FOR i IN 1..n LOOP
        total := total + i;
    END LOOP;
    RETURN total;
END;
$$ LANGUAGE plpgsql;
  • Explicación:
    • DECLARE: Sección para declarar variables.
    • total INTEGER := 0;: Inicializa la variable total.
    • FOR i IN 1..n LOOP: Bucle de 1 a n.
    • total := total + i;: Suma el valor actual de i a total.

Manejo de Errores

PL/pgSQL proporciona mecanismos para manejar errores usando bloques EXCEPTION.

Ejemplo: Manejo de Errores

CREATE OR REPLACE FUNCTION divide_numbers(a INTEGER, b INTEGER)
RETURNS FLOAT AS $$
BEGIN
    RETURN a / b;
EXCEPTION
    WHEN division_by_zero THEN
        RETURN NULL;
END;
$$ LANGUAGE plpgsql;
  • Explicación:
    • EXCEPTION: Bloque para capturar excepciones.
    • WHEN division_by_zero THEN: Captura errores de división por cero.
    • RETURN NULL;: Devuelve NULL si ocurre un error.

Disparadores

Los disparadores son procedimientos especiales que se ejecutan automáticamente en respuesta a ciertos eventos en una tabla.

Ejemplo: Disparador para Registrar Cambios

CREATE OR REPLACE FUNCTION log_changes()
RETURNS TRIGGER AS $$
BEGIN
    INSERT INTO changes_log(table_name, operation, changed_at)
    VALUES (TG_TABLE_NAME, TG_OP, NOW());
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER after_update
AFTER UPDATE ON your_table
FOR EACH ROW
EXECUTE FUNCTION log_changes();
  • Explicación:
    • TG_TABLE_NAME: Variable especial que contiene el nombre de la tabla.
    • TG_OP: Variable especial que contiene el tipo de operación (INSERT, UPDATE, DELETE).
    • AFTER UPDATE ON your_table: Especifica que el disparador se activa después de una actualización en your_table.

Conclusión

PL/pgSQL extiende las capacidades de PostgreSQL permitiéndote escribir código procedural. Esto incluye definir funciones, usar estructuras de control, manejar errores y crear disparadores. Dominar PL/pgSQL puede mejorar significativamente tu capacidad para gestionar y manipular datos dentro de PostgreSQL.

Al entender y practicar los conceptos cubiertos en este tema, estarás bien equipado para escribir código PL/pgSQL eficiente y robusto para diversas aplicaciones.
