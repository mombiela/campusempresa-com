Introducción a las Expresiones de Tabla Comunes (CTEs)

Las Expresiones de Tabla Comunes (CTEs) son una característica poderosa en PostgreSQL que te permite crear conjuntos de resultados temporales que pueden ser referenciados dentro de una instrucción SELECT, INSERT, UPDATE o DELETE. Las CTEs pueden simplificar consultas complejas, mejorar la legibilidad y hacer posibles las consultas recursivas.

Conceptos Clave

  • Definición de CTE: Una CTE se define usando la palabra clave WITH seguida de una consulta que genera un conjunto de resultados temporales.
  • Ámbito Temporal: El conjunto de resultados de una CTE solo está disponible dentro del ámbito de ejecución de la instrucción en la que se define.
  • CTEs Recursivas: Estas te permiten realizar consultas recursivas, que son útiles para estructuras de datos jerárquicas como organigramas o árboles genealógicos.

Sintaxis Básica de las CTEs

Para definir una CTE, utilizas la cláusula WITH seguida del nombre de la CTE y la consulta que genera el conjunto de resultados.

WITH cte_name AS (
    SELECT column1, column2
    FROM table_name
    WHERE condition
)
SELECT *
FROM cte_name;

Ejemplo: CTE Simple

Comencemos con un ejemplo simple. Supongamos que tenemos una tabla employees con las columnas id, name y department.

WITH dept_sales AS (
    SELECT id, name
    FROM employees
    WHERE department = 'Sales'
)
SELECT *
FROM dept_sales;

En este ejemplo:

  • Definimos una CTE llamada dept_sales.
  • La CTE selecciona id y name de la tabla employees donde el department es 'Sales'.
  • La consulta principal selecciona todas las columnas de la CTE dept_sales.

Uso de Múltiples CTEs

Puedes definir múltiples CTEs en una sola consulta separándolas con comas.

WITH dept_sales AS (
    SELECT id, name
    FROM employees
    WHERE department = 'Sales'
),
dept_marketing AS (
    SELECT id, name
    FROM employees
    WHERE department = 'Marketing'
)
SELECT *
FROM dept_sales
UNION ALL
SELECT *
FROM dept_marketing;

Ejemplo: Combinando Resultados de Múltiples CTEs

En este ejemplo:

  • Definimos dos CTEs: dept_sales y dept_marketing.
  • La consulta principal combina los resultados de ambas CTEs usando el operador UNION ALL.

CTEs Recursivas

Las CTEs recursivas son útiles para consultar datos jerárquicos. Consisten en dos partes: un miembro ancla y un miembro recursivo.

Sintaxis para CTEs Recursivas

WITH RECURSIVE cte_name AS (
    -- Miembro ancla
    SELECT column1, column2
    FROM table_name
    WHERE condition
    
    UNION ALL
    
    -- Miembro recursivo
    SELECT column1, column2
    FROM table_name
    JOIN cte_name ON table_name.column = cte_name.column
)
SELECT *
FROM cte_name;

Ejemplo: CTE Recursiva

Supongamos que tenemos una tabla employees con las columnas id, name y manager_id para representar una jerarquía organizacional.

WITH RECURSIVE employee_hierarchy AS (
    -- Miembro ancla: Seleccionar el gerente de nivel superior
    SELECT id, name, manager_id
    FROM employees
    WHERE manager_id IS NULL
    
    UNION ALL
    
    -- Miembro recursivo: Seleccionar empleados que reportan al nivel actual
    SELECT e.id, e.name, e.manager_id
    FROM employees e
    JOIN employee_hierarchy eh ON e.manager_id = eh.id
)
SELECT *
FROM employee_hierarchy;

En este ejemplo:

  • El miembro ancla selecciona al gerente de nivel superior (donde manager_id es NULL).
  • El miembro recursivo selecciona empleados que reportan al nivel actual de la jerarquía.
  • La consulta principal selecciona todas las columnas de la CTE employee_hierarchy.

Conclusión

Las Expresiones de Tabla Comunes (CTEs) en PostgreSQL proporcionan una manera poderosa de simplificar consultas complejas y gestionar datos jerárquicos. Al usar la cláusula WITH, puedes crear conjuntos de resultados temporales que mejoran la legibilidad y el mantenimiento de las consultas. Las CTEs recursivas extienden esta funcionalidad para manejar estructuras de datos recursivas de manera efectiva.

Resumen

  • CTEs: Conjuntos de resultados temporales definidos usando la cláusula WITH.
  • Múltiples CTEs: Pueden ser definidas en una sola consulta y combinadas usando operadores como UNION ALL.
  • CTEs Recursivas: Permiten consultar datos jerárquicos a través de consultas recursivas.

Practica usando CTEs en tus consultas para ver cómo pueden simplificar tu código SQL y hacerlo más legible.
