Introducción a la Búsqueda de Texto Completo

La búsqueda de texto completo (FTS) en PostgreSQL te permite buscar de manera eficiente documentos o texto dentro de una base de datos. A diferencia de la simple coincidencia de patrones con LIKE, FTS puede manejar consultas complejas y clasificar los resultados según su relevancia.

Conceptos Clave

  • Tokenización: Descomponer el texto en tokens (palabras o términos).
  • Normalización: Convertir los tokens a una forma estándar (por ejemplo, minúsculas, stemming).
  • Diccionarios: Se utilizan para definir cómo se procesan los tokens.
  • TSVector: Un tipo de dato que almacena texto preprocesado para una búsqueda eficiente.
  • TSQuery: Un tipo de dato que representa una consulta de búsqueda.

Configuración de la Búsqueda de Texto Completo

Creación de una Columna TSVector

Para usar FTS, necesitas crear una columna que almacene el texto preprocesado.

ALTER TABLE articles ADD COLUMN tsv tsvector;
UPDATE articles SET tsv = to_tsvector('english', title || ' ' || body);

Indexación de la Columna TSVector

Crear un índice en la columna tsvector mejora el rendimiento de la búsqueda.

CREATE INDEX idx_fts ON articles USING gin(tsv);

Realización de Búsquedas de Texto Completo

Búsqueda Básica

Puedes realizar una búsqueda básica de texto completo usando el operador @@.

SELECT title, body
FROM articles
WHERE tsv @@ to_tsquery('english', 'postgresql & search');

Clasificación de Resultados de Búsqueda

Para clasificar los resultados de búsqueda por relevancia, usa la función ts_rank.

SELECT title, body, ts_rank(tsv, to_tsquery('english', 'postgresql & search')) AS rank
FROM articles
WHERE tsv @@ to_tsquery('english', 'postgresql & search')
ORDER BY rank DESC;

Técnicas Avanzadas de Búsqueda de Texto Completo

Uso de Múltiples Diccionarios

Puedes usar múltiples diccionarios para manejar diferentes idiomas o vocabularios especializados.

CREATE TEXT SEARCH CONFIGURATION my_config (COPY = pg_catalog.english);
ALTER TEXT SEARCH CONFIGURATION my_config
    ALTER MAPPING FOR hword, hword_part, word
    WITH simple, english_stem;

Resaltar Términos de Búsqueda

Resaltar los términos de búsqueda en los resultados puede facilitar ver por qué se coincidió un documento.

SELECT title, ts_headline('english', body, to_tsquery('english', 'postgresql & search')) AS highlighted_body
FROM articles
WHERE tsv @@ to_tsquery('english', 'postgresql & search');

Conclusión

La búsqueda de texto completo en PostgreSQL es una característica poderosa que permite realizar búsquedas de texto eficientes y complejas. Al comprender y utilizar la tokenización, normalización y los tipos de datos tsvector y tsquery, puedes crear funcionalidades de búsqueda robustas en tus aplicaciones. Recuerda indexar tus columnas tsvector y considera usar técnicas avanzadas como múltiples diccionarios y resaltado de términos para mejorar tus capacidades de búsqueda.
