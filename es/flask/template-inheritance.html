La herencia de plantillas es una característica poderosa en Flask que te permite crear una plantilla base y extenderla en otras plantillas. Esto ayuda a mantener una apariencia y sensación consistentes en tu aplicación web y hace que tu código sea más modular y fácil de gestionar.

Conceptos Clave

  • Plantilla Base: Una plantilla que contiene la estructura y elementos comunes (como encabezado, pie de página, navegación) que se comparten en varias páginas.
  • Plantilla Hija: Una plantilla que extiende la plantilla base y sobrescribe bloques específicos para proporcionar contenido único para diferentes páginas.
  • Bloques: Secciones dentro de una plantilla que pueden ser sobrescritas por plantillas hijas.

Creando una Plantilla Base

Una plantilla base típicamente incluye la estructura HTML y elementos comunes como el encabezado y el pie de página. Aquí tienes un ejemplo de una plantilla base llamada base.html:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}My Flask App{% endblock %}</title>
</head>
<body>
    <header>
        <h1>Welcome to My Flask App</h1>
    </header>
    <nav>
        <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/about">About</a></li>
        </ul>
    </nav>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 My Flask App</p>
    </footer>
</body>
</html>

Explicación

  • {% block title %}My Flask App{% endblock %}: Este bloque permite que las plantillas hijas establezcan su propio título.
  • {% block content %}{% endblock %}: Este bloque será sobrescrito por las plantillas hijas para proporcionar contenido único para diferentes páginas.

Creando Plantillas Hijas

Las plantillas hijas extienden la plantilla base y sobrescriben bloques específicos para proporcionar contenido único. Aquí tienes un ejemplo de una plantilla hija llamada home.html:

{% extends "base.html" %}

{% block title %}Home - My Flask App{% endblock %}

{% block content %}
    <h2>Home Page</h2>
    <p>Welcome to the home page of my Flask app!</p>
{% endblock %}

Y otra plantilla hija llamada about.html:

{% extends "base.html" %}

{% block title %}About - My Flask App{% endblock %}

{% block content %}
    <h2>About Us</h2>
    <p>This is the about page of my Flask app.</p>
{% endblock %}

Explicación

  • {% extends "base.html" %}: Esta línea le dice a Flask que esta plantilla extiende la plantilla base.html.
  • {% block title %}Home - My Flask App{% endblock %}: Este bloque sobrescribe el bloque de título en la plantilla base.
  • {% block content %}: Este bloque sobrescribe el bloque de contenido en la plantilla base para proporcionar contenido único para las páginas de inicio y sobre nosotros.

Usando Plantillas en Rutas de Flask

Para renderizar estas plantillas en tu aplicación Flask, necesitas configurar rutas en tu archivo app.py:

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def home():
    return render_template('home.html')

@app.route('/about')
def about():
    return render_template('about.html')

if __name__ == '__main__':
    app.run(debug=True)

Explicación

  • render_template('home.html'): Esta función renderiza la plantilla home.html.
  • render_template('about.html'): Esta función renderiza la plantilla about.html.

Conclusión

La herencia de plantillas en Flask te permite crear una estructura consistente y mantenible para tu aplicación web. Al usar una plantilla base y extenderla en plantillas hijas, puedes gestionar fácilmente elementos comunes y contenido único en diferentes páginas. Este enfoque no solo ahorra tiempo, sino que también hace que tu código sea más modular y fácil de mantener.
