Introducción a Docker

Docker es una plataforma que permite a los desarrolladores automatizar el despliegue de aplicaciones dentro de contenedores ligeros y portátiles. Los contenedores incluyen todo lo necesario para ejecutar una aplicación, lo que facilita la gestión de dependencias y asegura la consistencia a través de diferentes entornos.

Conceptos Clave

  • Contenedor: Un paquete de software ligero, autónomo y ejecutable que incluye todo lo necesario para ejecutarlo.
  • Imagen: Una plantilla de solo lectura utilizada para crear contenedores. Las imágenes se construyen a partir de un conjunto de instrucciones escritas en un Dockerfile.
  • Dockerfile: Un script que contiene una serie de comandos para ensamblar una imagen.
  • Docker Hub: Un servicio de registro basado en la nube para compartir imágenes de Docker.

Configuración de Docker

Antes de usar Docker con Flask, asegúrate de que Docker esté instalado en tu máquina. Sigue las instrucciones de instalación para tu sistema operativo desde la documentación oficial de Docker.

Creación de una Aplicación Flask

Primero, vamos a crear una aplicación Flask simple. Crea un directorio para tu proyecto y añade los siguientes archivos:

`app.py`

from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello, Docker!'

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

`requirements.txt`

Flask==2.0.1

Escribir un Dockerfile

Un Dockerfile es un documento de texto que contiene todos los comandos para ensamblar una imagen. Crea un Dockerfile en tu directorio de proyecto con el siguiente contenido:

`Dockerfile`

# Usar la imagen oficial de Python desde Docker Hub
FROM python:3.9-slim

# Establecer el directorio de trabajo en el contenedor
WORKDIR /app

# Copiar el contenido del directorio actual en el contenedor en /app
COPY . /app

# Instalar cualquier paquete necesario especificado en requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Hacer el puerto 5000 disponible para el mundo fuera de este contenedor
EXPOSE 5000

# Definir variable de entorno
ENV FLASK_APP=app.py

# Ejecutar app.py cuando se inicie el contenedor
CMD ["flask", "run", "--host=0.0.0.0"]

Explicación

  • FROM python:3.9-slim: Especifica la imagen base a usar.
  • WORKDIR /app: Establece el directorio de trabajo dentro del contenedor.
  • COPY . /app: Copia el contenido del directorio actual en el contenedor.
  • RUN pip install --no-cache-dir -r requirements.txt: Instala los paquetes de Python necesarios.
  • EXPOSE 5000: Expone el puerto 5000 para permitir el acceso externo.
  • ENV FLASK_APP=app.py: Establece una variable de entorno para Flask.
  • CMD ["flask", "run", "--host=0.0.0.0"]: Especifica el comando para ejecutar la aplicación.

Construcción y Ejecución del Contenedor Docker

Para construir y ejecutar tu contenedor Docker, sigue estos pasos:

  1. Construir la Imagen Docker:

    docker build -t flask-docker-app .

  2. Ejecutar el Contenedor Docker:

    docker run -p 5000:5000 flask-docker-app

Explicación

  • docker build -t flask-docker-app .: Construye la imagen Docker y la etiqueta como flask-docker-app.
  • docker run -p 5000:5000 flask-docker-app: Ejecuta el contenedor y mapea el puerto 5000 en tu host al puerto 5000 en el contenedor.

Prueba de la Aplicación

Abre tu navegador web y navega a http://localhost:5000. Deberías ver el mensaje "Hello, Docker!".

Conclusión

Usar Docker con Flask simplifica el proceso de despliegue al asegurar que tu aplicación se ejecute de manera consistente en diferentes entornos. Al contenerizar tu aplicación Flask, puedes gestionar fácilmente las dependencias, escalar tu aplicación y desplegarla en varias plataformas.

Resumen

  • Los contenedores Docker empaquetan aplicaciones con todas sus dependencias.
  • Un Dockerfile define los pasos para crear una imagen Docker.
  • Construir y ejecutar un contenedor Docker implica usar comandos Docker simples.
  • Docker asegura consistencia y portabilidad para tus aplicaciones Flask.

Al dominar Docker, puedes mejorar tu flujo de trabajo de desarrollo con Flask y agilizar el proceso de despliegue.
