En esta sección, exploraremos cómo crear modelos en Flask. Los modelos son una parte crucial de cualquier aplicación web, ya que representan la estructura de datos y la lógica de negocio. Comenzaremos con lo básico y gradualmente avanzaremos hacia conceptos más avanzados.

Introducción a los Modelos

Los modelos en Flask se crean típicamente utilizando SQLAlchemy, un potente Mapeador Objeto-Relacional (ORM) que te permite interactuar con la base de datos usando objetos de Python.

Conceptos Clave

  • ORM (Mapeador Objeto-Relacional): Una técnica que te permite consultar y manipular datos de una base de datos utilizando un paradigma orientado a objetos.
  • SQLAlchemy: Un ORM popular para Python que proporciona un conjunto completo de patrones de persistencia a nivel empresarial bien conocidos.

Configuración de SQLAlchemy

Antes de crear modelos, necesitas configurar SQLAlchemy en tu aplicación Flask.

Instalación

Primero, instala Flask-SQLAlchemy usando pip:

pip install Flask-SQLAlchemy

Configuración

A continuación, configura SQLAlchemy en tu aplicación Flask:

from flask import Flask
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///site.db'
db = SQLAlchemy(app)
  • SQLALCHEMY_DATABASE_URI: Esta clave de configuración establece la URI de la base de datos que debe usarse para la conexión. En este ejemplo, estamos usando SQLite.

Creación de un Modelo Básico

Vamos a crear un modelo simple para un User:

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(20), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)
    password = db.Column(db.String(60), nullable=False)

    def __repr__(self):
        return f"User('{self.username}', '{self.email}')"

Explicación

  • db.Model: Esta es la clase base para todos los modelos en SQLAlchemy.
  • db.Column: Define una columna en la base de datos.
  • db.Integer, db.String: Especifica el tipo de datos que contendrá la columna.
  • primary_key=True: Marca esta columna como la clave primaria.
  • unique=True: Asegura que todos los valores en esta columna sean únicos.
  • nullable=False: Asegura que esta columna no pueda estar vacía.

Creación de Relaciones

Los modelos a menudo necesitan relacionarse entre sí. Vamos a crear un modelo Post que se relacione con el modelo User:

class Post(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(100), nullable=False)
    date_posted = db.Column(db.DateTime, nullable=False, default=datetime.utcnow)
    content = db.Column(db.Text, nullable=False)
    user_id = db.Column(db.Integer, db.ForeignKey('user.id'), nullable=False)

    def __repr__(self):
        return f"Post('{self.title}', '{self.date_posted}')"

Explicación

  • db.ForeignKey('user.id'): Esto crea una restricción de clave foránea, vinculando la columna user_id en el modelo Post a la columna id en el modelo User.

Características Avanzadas del Modelo

Añadiendo Métodos a los Modelos

Puedes añadir métodos a tus modelos para encapsular la lógica de negocio. Por ejemplo, un método para establecer el hash de la contraseña:

from werkzeug.security import generate_password_hash, check_password_hash

class User(db.Model):
    # ... campos anteriores ...

    def set_password(self, password):
        self.password = generate_password_hash(password)

    def check_password(self, password):
        return check_password_hash(self.password, password)

Usando Mixins para Reutilización

Los Mixins te permiten reutilizar campos y métodos comunes en múltiples modelos:

class TimestampMixin(object):
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)

class User(db.Model, TimestampMixin):
    # ... campos anteriores ...

class Post(db.Model, TimestampMixin):
    # ... campos anteriores ...

Resumen

En esta sección, cubrimos lo básico de la creación de modelos en Flask usando SQLAlchemy. Comenzamos con la configuración de SQLAlchemy, la creación de modelos básicos y el establecimiento de relaciones entre modelos. También exploramos características avanzadas como añadir métodos a los modelos y usar mixins para la reutilización.

Al entender estos conceptos, puedes gestionar eficazmente la capa de datos de tus aplicaciones Flask, asegurando que tus datos estén bien estructurados y sean fácilmente accesibles.
