Introducción

En esta sección, aprenderemos a configurar una máquina virtual en Amazon Web Services (AWS), una de las plataformas de Infraestructura como Servicio (IaaS) más populares. Este ejercicio práctico te permitirá comprender los pasos necesarios para desplegar y gestionar una instancia de EC2 (Elastic Compute Cloud) en AWS.

Requisitos Previos

Antes de comenzar, asegúrate de tener lo siguiente:

  • Una cuenta activa en AWS.
  • Acceso a la consola de administración de AWS.
  • Conocimientos básicos de redes y sistemas operativos.

Pasos para Configurar una Máquina Virtual en AWS

1. Iniciar Sesión en la Consola de AWS

  1. Ve a AWS Management Console.
  2. Inicia sesión con tus credenciales de AWS.

2. Navegar a EC2

  1. En la consola de AWS, busca "EC2" en la barra de búsqueda y selecciona "EC2" bajo la categoría "Compute".

3. Lanzar una Nueva Instancia

  1. Haz clic en el botón "Launch Instance".
  2. Selecciona una Amazon Machine Image (AMI):
    • Ejemplo: Elige "Amazon Linux 2 AMI (HVM), SSD Volume Type".
  3. Selecciona el tipo de instancia:
    • Ejemplo: Elige "t2.micro" (gratis elegible).
  4. Configura los detalles de la instancia:
    • Ejemplo: Usa la configuración predeterminada para este ejercicio.
  5. Añade almacenamiento:
    • Ejemplo: Usa el almacenamiento predeterminado de 8 GB.
  6. Añade etiquetas (opcional):
    • Ejemplo: Añade una etiqueta con la clave "Name" y el valor "MiPrimeraInstancia".
  7. Configura el grupo de seguridad:
    • Ejemplo: Crea un nuevo grupo de seguridad con una regla que permita el tráfico SSH desde tu IP.

4. Revisar y Lanzar

  1. Revisa todas las configuraciones.
  2. Haz clic en "Launch".
  3. Selecciona un par de claves existente o crea uno nuevo para acceder a tu instancia.
    • Ejemplo: Si creas uno nuevo, descarga el archivo .pem y guárdalo en un lugar seguro.

5. Conectar a la Instancia

  1. Una vez que la instancia esté en estado "running", selecciona la instancia y haz clic en "Connect".
  2. Sigue las instrucciones para conectar a tu instancia usando SSH:
    • Ejemplo de comando SSH: 
      ssh -i /path/to/your-key-pair.pem ec2-user@your-instance-public-dns

Ejemplo de Código

A continuación, se muestra un ejemplo de cómo conectar a tu instancia EC2 usando SSH desde una terminal:

# Navega al directorio donde guardaste tu archivo .pem
cd /path/to/your-key-pair.pem

# Cambia los permisos del archivo .pem para que solo el propietario pueda leerlo
chmod 400 your-key-pair.pem

# Conéctate a la instancia EC2
ssh -i your-key-pair.pem ec2-user@your-instance-public-dns

Conclusión

En este ejercicio, hemos configurado y lanzado una máquina virtual en AWS utilizando EC2. Este proceso incluyó la selección de una AMI, la configuración de la instancia, la configuración del almacenamiento y la seguridad, y finalmente, la conexión a la instancia mediante SSH. Este conocimiento es fundamental para cualquier profesional de TI que trabaje con servicios en la nube y desee aprovechar las capacidades de IaaS proporcionadas por AWS.
