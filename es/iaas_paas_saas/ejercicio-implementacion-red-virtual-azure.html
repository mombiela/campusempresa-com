Introducción

En este ejercicio, aprenderás a implementar una Red Virtual (VNet) en Microsoft Azure. Las Redes Virtuales son fundamentales para la infraestructura como servicio (IaaS) y permiten la comunicación segura entre recursos de Azure. Este ejercicio te guiará paso a paso en la creación y configuración de una VNet, subredes y la conexión de máquinas virtuales (VMs).

Objetivos del Ejercicio

  • Crear una Red Virtual en Azure.
  • Configurar subredes dentro de la VNet.
  • Implementar y conectar máquinas virtuales a la VNet.
  • Verificar la conectividad entre las VMs.

Requisitos Previos

  • Cuenta activa de Azure.
  • Conocimientos básicos de redes y máquinas virtuales.
  • Familiaridad con el portal de Azure.

Pasos para la Implementación

1. Crear una Red Virtual

  1. Inicia sesión en el Portal de Azure.
  2. Navega a "Crear un recurso" y selecciona "Red Virtual".
  3. Completa los detalles básicos:
    • Nombre: MyVNet
    • Región: Selecciona la región más cercana a ti.
    • Rango de direcciones IP: 10.0.0.0/16
  4. Haz clic en "Revisar y crear" y luego en "Crear".

2. Configurar Subredes

  1. Una vez creada la VNet, navega a "Subredes" en la configuración de la VNet.
  2. Agrega una nueva subred:
    • Nombre: Subred1
    • Rango de direcciones IP: 10.0.1.0/24
  3. Repite el proceso para una segunda subred:
    • Nombre: Subred2
    • Rango de direcciones IP: 10.0.2.0/24

3. Implementar Máquinas Virtuales

  1. Navega a "Máquinas Virtuales" y selecciona "Agregar".
  2. Completa los detalles básicos para la primera VM:
    • Nombre: VM1
    • Región: La misma que la VNet.
    • Imagen: Windows Server 2019 Datacenter.
    • Tamaño: Selecciona un tamaño adecuado.
    • Red virtual: MyVNet
    • Subred: Subred1
  3. Repite el proceso para una segunda VM:
    • Nombre: VM2
    • Subred: Subred2

4. Verificar la Conectividad

  1. Conéctate a VM1 usando RDP.
  2. Abre una ventana de PowerShell y ejecuta el siguiente comando para verificar la conectividad con VM2:
Test-NetConnection -ComputerName 10.0.2.4 -Port 3389
  1. Si la conectividad es exitosa, deberías ver un resultado positivo indicando que VM1 puede comunicarse con VM2.

Conclusión

En este ejercicio, has aprendido a crear y configurar una Red Virtual en Azure, incluyendo la creación de subredes y la implementación de máquinas virtuales. La verificación de la conectividad entre las VMs demuestra que la configuración de la red es correcta. Este conocimiento es esencial para gestionar y desplegar infraestructuras en la nube utilizando IaaS en Azure.
