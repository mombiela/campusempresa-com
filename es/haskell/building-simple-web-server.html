Construyendo un Servidor Web Simple en Haskell
Acceso a Bases de Datos ►

Introducción

En este tema, exploraremos cómo construir un servidor web simple usando Haskell. Esto cubrirá los conceptos básicos de la configuración de un entorno Haskell, la comprensión de HTTP y el uso de las bibliotecas wai y warp para crear un servidor web funcional.

Requisitos Previos

Antes de comenzar, asegúrate de tener:

  • Comprensión básica de la sintaxis y funciones de Haskell.
  • Haskell instalado en tu máquina (GHC y Cabal o Stack).

Configuración del Entorno

Para empezar, necesitamos configurar nuestro entorno Haskell e instalar las bibliotecas necesarias.

Instalando Haskell

  1. GHC (Glasgow Haskell Compiler): El compilador principal para Haskell.
  2. Cabal o Stack: Herramientas de construcción para gestionar proyectos Haskell.

Instalando Bibliotecas

Usaremos las bibliotecas wai (Interfaz de Aplicación Web) y warp (una biblioteca de servidor HTTP rápida).

cabal update
cabal install wai warp

O, si usas Stack:

stack update
stack install wai warp

Entendiendo los Conceptos Básicos de HTTP

Antes de sumergirnos en el código, entendamos brevemente HTTP, el protocolo que usan los servidores web para comunicarse con los clientes (navegadores).

  • Request (Solicitud): Enviada por el cliente al servidor, incluye métodos como GET, POST, etc.
  • Response (Respuesta): Enviada por el servidor al cliente, incluye códigos de estado (200, 404, etc.) y datos.

Construyendo el Servidor Web

Creando un Servidor Básico

Comencemos creando un servidor web simple que responda con "Hello, World!" a cualquier solicitud.

  1. Crea un nuevo archivo Haskell:

    touch Main.hs

  2. Escribe el siguiente código en Main.hs:

    {-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app _ respond = respond $ responseLBS status200 [("Content-Type", "text/plain")] "Hello, World!"

Explicación

  • Imports: Importamos los módulos necesarios de wai, warp y http-types.
  • Función Principal: Inicia el servidor en el puerto 8080 y usa la función app para manejar las solicitudes.
  • Función App: Define la lógica de la aplicación, que responde con un estado 200 y el mensaje "Hello, World!".

Ejecutando el Servidor

Para ejecutar el servidor, usa el siguiente comando:

runhaskell Main.hs

Abre tu navegador y navega a http://localhost:8080 para ver "Hello, World!".

Añadiendo Rutas

A continuación, añadamos algunas rutas para manejar diferentes caminos.

Ejemplo de Código

Actualiza Main.hs para incluir enrutamiento:

{-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application, pathInfo)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200, status404)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app req respond = case pathInfo req of
    ["hello"] -> respond $ responseLBS status200 [("Content-Type", "text/plain")] "Hello, World!"
    ["goodbye"] -> respond $ responseLBS status200 [("Content-Type", "text/plain")] "Goodbye, World!"
    _ -> respond $ responseLBS status404 [("Content-Type", "text/plain")] "404 - Not Found"

Explicación

  • pathInfo: Extrae el camino de la solicitud.
  • Case Matching: Coincide el camino con rutas predefinidas y responde en consecuencia.

Ejecutando el Servidor

Ejecuta el servidor nuevamente y navega a:

  • http://localhost:8080/hello para ver "Hello, World!"
  • http://localhost:8080/goodbye para ver "Goodbye, World!"
  • Cualquier otro camino devolverá "404 - Not Found".

Manejo de Parámetros de Consulta

Manejemos los parámetros de consulta para hacer nuestro servidor más dinámico.

Ejemplo de Código

Actualiza Main.hs para manejar parámetros de consulta:

{-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application, queryString)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200, status400)
import Data.Maybe (fromMaybe)
import Data.ByteString.Lazy.Char8 (pack)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app req respond = case pathInfo req of
    ["greet"] -> do
        let name = fromMaybe "World" $ lookup "name" $ queryString req
        respond $ responseLBS status200 [("Content-Type", "text/plain")] (pack $ "Hello, " ++ show name ++ "!")
    _ -> respond $ responseLBS status400 [("Content-Type", "text/plain")] "400 - Bad Request"

Explicación

  • queryString: Extrae los parámetros de consulta de la solicitud.
  • fromMaybe: Proporciona un valor predeterminado si no se encuentra el parámetro.
  • pack: Convierte una cadena a un ByteString.

Ejecutando el Servidor

Ejecuta el servidor y navega a:

  • http://localhost:8080/greet?name=Haskell para ver "Hello, Haskell!"

Conclusión

En este tema, cubrimos los conceptos básicos de la construcción de un servidor web simple en Haskell usando las bibliotecas wai y warp. Comenzamos con un servidor básico, añadimos enrutamiento y manejamos parámetros de consulta. Esta base te permite construir aplicaciones web más complejas en Haskell.

Resumen

  • Configuramos el entorno Haskell e instalamos las bibliotecas necesarias.
  • Construimos un servidor web básico que responde con "Hello, World!".
  • Añadimos enrutamiento para manejar diferentes caminos.
  • Manejamos parámetros de consulta para hacer el servidor dinámico.

Con estas habilidades, ahora puedes explorar temas más avanzados como middleware, integración de bases de datos y enrutamiento más complejo.
