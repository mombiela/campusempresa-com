¿Qué es un Functor?

En Haskell, un Functor es una clase de tipo que representa un tipo sobre el que se puede aplicar un mapeo. Es un concepto fundamental en la programación funcional y se utiliza para aplicar una función a valores envueltos en un contexto, como una lista, Maybe, u otras estructuras de datos.

Conceptos Clave

  • Clase de Tipo: Una clase de tipo define un conjunto de funciones que pueden ser implementadas por diferentes tipos.
  • Mapeo: Aplicar una función a cada elemento en una estructura de datos.
  • Contexto: La estructura que contiene los valores, como una lista o Maybe.

Clase de Tipo Functor

La clase de tipo Functor se define en el Prelude de la siguiente manera:

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • fmap es la función principal de la clase de tipo Functor.
  • Toma una función (a -> b) y un functor f a, y devuelve un nuevo functor f b.

Ejemplo: Instancia de Functor para Lista

El tipo de lista [] es un functor. Así es como funciona:

instance Functor [] where
    fmap = map
  • La función map aplica una función dada a cada elemento de una lista.

Ejemplo de Código

-- Usando fmap con una lista
fmap (+1) [1, 2, 3]  -- Resultado: [2, 3, 4]

En este ejemplo, fmap aplica la función (+1) a cada elemento de la lista [1, 2, 3], resultando en [2, 3, 4].

Leyes de los Functores

Los functores deben obedecer dos leyes para asegurar un comportamiento consistente:

  1. Ley de Identidad: fmap id == id
  2. Ley de Composición: fmap (f . g) == fmap f . fmap g

Ley de Identidad

Aplicar la función identidad id usando fmap debería devolver el functor original.

Ejemplo de Código

fmap id [1, 2, 3]  -- Resultado: [1, 2, 3]

Ley de Composición

Mapear la composición de dos funciones debería ser lo mismo que mapear una función y luego la otra.

Ejemplo de Código

fmap ((+1) . (*2)) [1, 2, 3]  -- Resultado: [3, 5, 7]
-- Equivalente a
fmap (+1) (fmap (*2) [1, 2, 3])  -- Resultado: [3, 5, 7]

Instancias de Functor para Otros Tipos

Functor Maybe

El tipo Maybe también es un functor. Representa cálculos que podrían fallar.

instance Functor Maybe where
    fmap _ Nothing  = Nothing
    fmap f (Just x) = Just (f x)

Ejemplo de Código

fmap (+1) (Just 5)  -- Resultado: Just 6
fmap (+1) Nothing   -- Resultado: Nothing

Functor Either

El tipo Either representa cálculos que podrían resultar en un error.

instance Functor (Either a) where
    fmap _ (Left x)  = Left x
    fmap f (Right y) = Right (f y)

Ejemplo de Código

fmap (+1) (Right 5)  -- Resultado: Right 6
fmap (+1) (Left "Error")  -- Resultado: Left "Error"

Casos de Uso Prácticos

Transformando Datos en una Lista

-- Duplicando cada elemento en una lista
fmap (*2) [1, 2, 3, 4]  -- Resultado: [2, 4, 6, 8]

Manejo de Valores Opcionales

-- Incrementando un valor dentro de un Maybe
fmap (+1) (Just 10)  -- Resultado: Just 11
fmap (+1) Nothing    -- Resultado: Nothing

Conclusión

Los functores son una poderosa abstracción en Haskell que te permiten aplicar funciones a valores envueltos en un contexto. Al entender y usar la clase de tipo Functor, puedes escribir código más genérico y reutilizable. Recuerda las leyes de los Functores para asegurar que tus instancias se comporten de manera consistente. Los functores son solo el comienzo; allanan el camino hacia conceptos más avanzados como los Aplicativos y los Monads.
