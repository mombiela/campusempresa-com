Introducción

En Haskell, las listas y las tuplas son estructuras de datos fundamentales que te permiten almacenar y manipular colecciones de valores. Esta sección cubrirá los conceptos básicos de las listas y las tuplas, sus diferencias y cómo usarlas efectivamente en la programación en Haskell.

Listas

Las listas en Haskell son colecciones homogéneas, lo que significa que todos los elementos deben ser del mismo tipo. Son una de las estructuras de datos más comúnmente usadas en Haskell.

Creación de Listas

Puedes crear listas usando corchetes [] y separando los elementos con comas.

-- Una lista vacía
emptyList :: [Int]
emptyList = []

-- Una lista de enteros
intList :: [Int]
intList = [1, 2, 3, 4, 5]

-- Una lista de caracteres (String)
charList :: [Char]
charList = ['H', 'e', 'l', 'l', 'o']

Operaciones con Listas

Haskell proporciona varias funciones integradas para manipular listas.

Concatenación

Puedes concatenar dos listas usando el operador ++.

list1 :: [Int]
list1 = [1, 2, 3]

list2 :: [Int]
list2 = [4, 5, 6]

combinedList :: [Int]
combinedList = list1 ++ list2  -- [1, 2, 3, 4, 5, 6]

Cabeza y Cola

La función head devuelve el primer elemento de una lista, y la función tail devuelve la lista sin su primer elemento.

firstElement :: Int
firstElement = head [1, 2, 3, 4]  -- 1

restOfList :: [Int]
restOfList = tail [1, 2, 3, 4]  -- [2, 3, 4]

Comprensiones de Listas

Las comprensiones de listas proporcionan una forma concisa de crear listas.

-- Una lista de cuadrados de números del 1 al 5
squares :: [Int]
squares = [x * x | x <- [1..5]]  -- [1, 4, 9, 16, 25]

-- Una lista de números pares del 1 al 10
evens :: [Int]
evens = [x | x <- [1..10], even x]  -- [2, 4, 6, 8, 10]

Tuplas

Las tuplas en Haskell son colecciones de tamaño fijo que pueden contener elementos de diferentes tipos. Son útiles para agrupar valores relacionados.

Creación de Tuplas

Las tuplas se crean usando paréntesis () y separando los elementos con comas.

-- Una tupla de dos enteros
pair :: (Int, Int)
pair = (1, 2)

-- Una tupla de un entero, un carácter y una cadena
mixedTuple :: (Int, Char, String)
mixedTuple = (1, 'a', "Hello")

Acceso a Elementos de la Tupla

Puedes acceder a los elementos de una tupla usando el patrón de coincidencia.

-- Patrón de coincidencia para extraer elementos de una tupla
extractElements :: (Int, Char, String) -> (Int, Char, String)
extractElements (x, y, z) = (x, y, z)

-- Ejemplo de uso
exampleTuple :: (Int, Char, String)
exampleTuple = (1, 'a', "Hello")

-- Elementos extraídos
(x, y, z) = extractElements exampleTuple  -- x = 1, y = 'a', z = "Hello"

Tuplas Anidadas

Las tuplas pueden estar anidadas entre sí.

-- Una tupla anidada
nestedTuple :: ((Int, Char), String)
nestedTuple = ((1, 'a'), "Hello")

-- Acceso a elementos en una tupla anidada
extractNested :: ((Int, Char), String) -> (Int, Char, String)
extractNested ((x, y), z) = (x, y, z)

-- Ejemplo de uso
(x, y, z) = extractNested nestedTuple  -- x = 1, y = 'a', z = "Hello"

Listas vs. Tuplas

| Característica | Listas | Tuplas | |------------------|-------------------------------|-------------------------------| | Homogeneidad | Todos los elementos deben ser del mismo tipo | Los elementos pueden ser de diferentes tipos | | Tamaño | Dinámico, puede crecer o encogerse | Tamaño fijo | | Sintaxis | Corchetes [] | Paréntesis () | | Casos de Uso Comunes | Secuencias, colecciones | Agrupación de valores relacionados |

Conclusión

Las listas y las tuplas son estructuras de datos esenciales en Haskell. Las listas son ideales para manejar secuencias de elementos del mismo tipo, mientras que las tuplas son útiles para agrupar un número fijo de elementos de tipos potencialmente diferentes. Entender cómo crear y manipular estas estructuras mejorará enormemente tu capacidad para escribir programas efectivos en Haskell.
