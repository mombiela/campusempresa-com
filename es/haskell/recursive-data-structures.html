Introducción a las Estructuras de Datos Recursivas

Las estructuras de datos recursivas son fundamentales en la programación funcional, especialmente en Haskell. Nos permiten definir tipos de datos complejos utilizando instancias más simples del mismo tipo. Esta sección introducirá el concepto y proporcionará una base para entender temas más avanzados.

Conceptos Clave

  • Recursión: Una función que se llama a sí misma.
  • Caso Base: La condición bajo la cual la recursión se detiene.
  • Caso Recursivo: La parte de la función que incluye la llamada recursiva.
  • Estructuras de Datos: Formas de almacenar y organizar datos, como listas, árboles y más.

Listas como Estructuras de Datos Recursivas

Las listas son una de las estructuras de datos recursivas más simples y comúnmente usadas en Haskell.

Definición y Construcción

Una lista en Haskell está vacía o consiste en un elemento cabeza seguido de una cola, que es en sí misma una lista.

data List a = Empty | Cons a (List a)

Ejemplo: Suma de una Lista

Aquí hay una función recursiva simple para calcular la suma de una lista de enteros.

sumList :: [Int] -> Int
sumList [] = 0
sumList (x:xs) = x + sumList(xs)
  • Caso Base: sumList [] = 0 (La suma de una lista vacía es 0).
  • Caso Recursivo: sumList (x:xs) = x + sumList(xs) (La suma de una lista es el elemento cabeza más la suma de la cola).

Árboles como Estructuras de Datos Recursivas

Los árboles son otra estructura de datos recursiva común. Consisten en nodos, donde cada nodo puede tener cero o más nodos hijos.

Definición y Construcción

Un árbol binario es un árbol donde cada nodo tiene como máximo dos hijos.

data Tree a = EmptyTree | Node a (Tree a) (Tree a)

Ejemplo: Profundidad de un Árbol

Aquí hay una función recursiva para calcular la profundidad de un árbol binario.

treeDepth :: Tree a -> Int
treeDepth EmptyTree = 0
treeDepth (Node _ left right) = 1 + max (treeDepth left) (treeDepth right)
  • Caso Base: treeDepth EmptyTree = 0 (La profundidad de un árbol vacío es 0).
  • Caso Recursivo: treeDepth (Node _ left right) = 1 + max (treeDepth left) (treeDepth right) (La profundidad de un árbol es 1 más la profundidad máxima de sus subárboles).

Estructuras de Datos Recursivas Avanzadas

A medida que te sientas más cómodo con las estructuras de datos recursivas básicas, puedes explorar otras más complejas como grafos y tipos de datos personalizados.

Ejemplo: Tipo de Dato Recursivo Personalizado

Definamos un tipo de dato recursivo personalizado para un árbol de expresiones simple.

data Expr = Val Int | Add Expr Expr | Mul Expr Expr

Ejemplo: Evaluación de un Árbol de Expresiones

Aquí hay una función para evaluar un árbol de expresiones.

eval :: Expr -> Int
eval (Val n) = n
eval (Add x y) = eval x + eval y
eval (Mul x y) = eval x * eval y
  • Caso Base: eval (Val n) = n (El valor de un nodo hoja es el entero que contiene).
  • Caso Recursivo: eval (Add x y) = eval x + eval y y eval (Mul x y) = eval x * eval y (Evalúa los subárboles y aplica la operación).

Conclusión

Las estructuras de datos recursivas son una herramienta poderosa en Haskell, permitiendo definiciones elegantes y concisas de tipos de datos y operaciones complejas. Al entender los conceptos básicos de listas y árboles, y explorar estructuras más avanzadas, puedes aprovechar la recursión para resolver una amplia gama de problemas en programación funcional.

Resumen

  • Listas: Estructura recursiva simple con un caso base (lista vacía) y un caso recursivo (cabeza y cola).
  • Árboles: Estructura más compleja con nodos y subárboles.
  • Tipos Personalizados: Define tus propias estructuras de datos recursivas para problemas específicos.
  • Recursión: Técnica clave para procesar estructuras de datos recursivas, con casos base y recursivos claros.

Al dominar estos conceptos, estarás bien equipado para enfrentar una variedad de desafíos de programación en Haskell.
