Descripción general

Cloud Functions para Firebase es un marco sin servidor que te permite ejecutar automáticamente código backend en respuesta a eventos desencadenados por características de Firebase y solicitudes HTTPS. Esta sección te guiará a través de los conceptos básicos de Cloud Functions, desde la configuración hasta casos de uso avanzados.

¿Qué son Cloud Functions?

Cloud Functions son funciones JavaScript de un solo propósito que se ejecutan en un entorno gestionado. Pueden ser desencadenadas por varios eventos, como cambios en tu base de datos Firebase, nuevos registros de usuarios o solicitudes HTTP.

Conceptos clave

  • Sin servidor: No es necesario gestionar servidores; Firebase maneja la infraestructura.
  • Basado en eventos: Las funciones se desencadenan por eventos específicos.
  • Escalable: Se escala automáticamente hacia arriba o hacia abajo según la demanda.
  • Seguro: Se ejecuta en un entorno seguro con autenticación y autorización integradas.

Configuración de Cloud Functions

Antes de que puedas comenzar a escribir Cloud Functions, necesitas configurar tu entorno de desarrollo.

Requisitos previos

  • Node.js y npm instalados
  • Firebase CLI instalado
  • Un proyecto de Firebase

Pasos para configurar

  1. Instalar Firebase CLI: 
    npm install -g firebase-tools
  2. Inicializar Firebase en tu proyecto: 
    firebase login
firebase init functions
  3. Elige tu proyecto de Firebase y configura tu directorio de funciones.

Escribiendo tu primera Cloud Function

Vamos a escribir una simple función HTTP que responda con "¡Hola, Mundo!".

Código de ejemplo

const functions = require('firebase-functions');

// Crear y desplegar una función HTTP
exports.helloWorld = functions.https.onRequest((request, response) => {
  response.send("¡Hola, Mundo!");
});

Explicación

  • functions.https.onRequest: Esto especifica que la función será desencadenada por una solicitud HTTP.
  • request y response: Estos son los objetos estándar de Express.js utilizados para manejar solicitudes y respuestas HTTP.

Desplegando Cloud Functions

Una vez que hayas escrito tu función, necesitas desplegarla en Firebase.

Pasos para desplegar

  1. Desplegar la función: 
    firebase deploy --only functions
  2. Acceder a tu función: Firebase proporcionará una URL donde tu función está alojada.

Casos de uso avanzados

Cloud Functions pueden hacer mucho más que solo responder a solicitudes HTTP. Aquí hay algunos casos de uso avanzados:

Triggers de base de datos

Desencadenar funciones en respuesta a cambios en tu Firebase Realtime Database o Firestore.

Código de ejemplo

exports.onUserCreate = functions.auth.user().onCreate((user) => {
  console.log('Un nuevo usuario se registró:', user.email);
});

Triggers de Firestore

Desencadenar funciones cuando se crean, actualizan o eliminan documentos en Firestore.

Código de ejemplo

exports.onDocumentCreate = functions.firestore
  .document('users/{userId}')
  .onCreate((snap, context) => {
    const newValue = snap.data();
    console.log('Nuevo documento de usuario creado:', newValue);
  });

Funciones programadas

Ejecutar funciones en un horario utilizando la integración de Firebase con Google Cloud Scheduler.

Código de ejemplo

exports.scheduledFunction = functions.pubsub.schedule('every 24 hours').onRun((context) => {
  console.log('Esta función se ejecuta cada 24 horas');
});

Conclusión

Cloud Functions para Firebase proporcionan una forma poderosa de extender tu aplicación Firebase con lógica backend personalizada. Son fáciles de configurar, escalables y seguras. Ya sea que estés respondiendo a solicitudes HTTP, cambios en la base de datos o eventos programados, Cloud Functions pueden ayudarte a construir aplicaciones robustas y dinámicas.

Al dominar Cloud Functions, puedes mejorar significativamente las capacidades de tus proyectos Firebase, haciéndolos más receptivos y ricos en características.
