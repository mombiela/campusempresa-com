Firestore es una base de datos flexible y escalable para el desarrollo móvil, web y de servidores de Firebase y Google Cloud Platform. Es una base de datos NoSQL de documentos que te permite almacenar, sincronizar y consultar datos fácilmente para tus aplicaciones.

Conceptos Clave

  • Base de Datos NoSQL: Firestore es una base de datos NoSQL, lo que significa que almacena datos en un formato no tabular.
  • Documentos y Colecciones: Los datos se almacenan en documentos, que están organizados en colecciones.
  • Actualizaciones en Tiempo Real: Firestore puede sincronizar datos en tiempo real entre todos los clientes.
  • Soporte Offline: Firestore soporta la persistencia de datos sin conexión, permitiendo que las aplicaciones funcionen incluso cuando no hay conexión de red.
  • Reglas de Seguridad: Firestore proporciona un lenguaje de reglas potente para controlar el acceso a tus datos.

Configuración de Firestore

  1. Crear un Proyecto de Firebase: Ve a la Consola de Firebase y crea un nuevo proyecto.
  2. Agregar Firestore a tu Proyecto: Navega a la sección de Firestore en la Consola de Firebase y habilita Firestore.
  3. Instalar el SDK de Firebase: Agrega el SDK de Firebase a tu proyecto.

Ejemplo: Agregar el SDK de Firebase a un Proyecto Web

<!-- Agregar SDKs de Firebase -->
<script src="https://www.gstatic.com/firebasejs/9.0.0/firebase-app.js"></script>
<script src="https://www.gstatic.com/firebasejs/9.0.0/firebase-firestore.js"></script>

<script>
  // Configuración de Firebase de tu aplicación web
  var firebaseConfig = {
    apiKey: "TU_API_KEY",
    authDomain: "TU_PROYECTO_ID.firebaseapp.com",
    projectId: "TU_PROYECTO_ID",
    storageBucket: "TU_PROYECTO_ID.appspot.com",
    messagingSenderId: "TU_MESSAGING_SENDER_ID",
    appId: "TU_APP_ID"
  };
  // Inicializar Firebase
  firebase.initializeApp(firebaseConfig);
  var db = firebase.firestore();
</script>

Trabajando con Firestore

Agregar Datos

Para agregar datos a Firestore, creas un documento dentro de una colección.

// Agregar un nuevo documento con un ID generado
db.collection("users").add({
    first: "Ada",
    last: "Lovelace",
    born: 1815
})
.then((docRef) => {
    console.log("Documento escrito con ID: ", docRef.id);
})
.catch((error) => {
    console.error("Error al agregar documento: ", error);
});

Leer Datos

Puedes leer datos de Firestore consultando documentos.

// Obtener una lista de usuarios
db.collection("users").get().then((querySnapshot) => {
    querySnapshot.forEach((doc) => {
        console.log(`${doc.id} => ${doc.data()}`);
    });
});

Actualizar Datos

Para actualizar un documento, puedes usar el método update.

var userRef = db.collection("users").doc("USER_ID");

// Establecer el campo "capital" de la ciudad 'DC'
return userRef.update({
    capital: true
})
.then(() => {
    console.log("Documento actualizado exitosamente!");
})
.catch((error) => {
    console.error("Error al actualizar el documento: ", error);
});

Eliminar Datos

Para eliminar un documento, usa el método delete.

db.collection("users").doc("USER_ID").delete().then(() => {
    console.log("Documento eliminado exitosamente!");
}).catch((error) => {
    console.error("Error al eliminar el documento: ", error);
});

Actualizaciones en Tiempo Real

Firestore te permite escuchar actualizaciones en tiempo real.

db.collection("users").onSnapshot((querySnapshot) => {
    var users = [];
    querySnapshot.forEach((doc) => {
        users.push(doc.data());
    });
    console.log("Usuarios actuales en la colección: ", users.join(", "));
});

Reglas de Seguridad

Las reglas de seguridad de Firestore te permiten controlar el acceso a tus datos.

service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Conclusión

Firestore es una potente base de datos NoSQL que ofrece sincronización de datos en tiempo real, soporte offline y características de seguridad robustas. Al entender los conceptos básicos de la configuración de Firestore, agregar, leer, actualizar y eliminar datos, así como implementar actualizaciones en tiempo real y reglas de seguridad, puedes gestionar eficazmente las necesidades de datos de tu aplicación.
