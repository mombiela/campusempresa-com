Las pruebas son una parte crucial de cualquier proceso de desarrollo, asegurando que tu aplicación se comporte como se espera. Firebase proporciona varias herramientas y servicios para ayudarte a ejecutar pruebas de manera efectiva. Esta sección te guiará a través de los conceptos básicos de la ejecución de pruebas en Firebase, desde la configuración de tu entorno hasta la ejecución y monitoreo de pruebas.

Configuración de Tu Entorno de Pruebas

Antes de poder ejecutar pruebas, necesitas configurar tu entorno de pruebas. Firebase ofrece varias herramientas que pueden ayudarte con esto:

  • Firebase Test Lab: Una infraestructura de pruebas de aplicaciones basada en la nube.
  • Firebase Emulator Suite: Te permite ejecutar y automatizar pruebas localmente.

Firebase Test Lab

Firebase Test Lab proporciona una infraestructura basada en la nube para probar tu aplicación en una amplia gama de dispositivos y configuraciones.

  1. Crea un Proyecto de Firebase: Si aún no lo has hecho, crea un proyecto de Firebase en la Consola de Firebase.
  2. Habilita Test Lab: Navega a la sección de Test Lab en la Consola de Firebase y habilítalo para tu proyecto.
  3. Sube Tu Aplicación: Puedes subir tu aplicación Android o iOS a Test Lab.
  4. Selecciona Dispositivos de Prueba: Elige entre una variedad de dispositivos y configuraciones para ejecutar tus pruebas.
  5. Ejecuta Pruebas: Ejecuta tus pruebas y visualiza los resultados en la Consola de Firebase.

Firebase Emulator Suite

Firebase Emulator Suite te permite ejecutar y automatizar pruebas localmente, lo que puede ser más rápido y rentable que usar soluciones basadas en la nube.

  1. Instala Firebase CLI: Asegúrate de tener instalado Firebase CLI. 
    npm install -g firebase-tools
  2. Inicializa el Emulator Suite: En el directorio de tu proyecto, inicializa el Emulator Suite. 
    firebase init emulators
  3. Configura Emuladores: Elige qué emuladores configurar (Firestore, Realtime Database, Authentication, etc.).
  4. Inicia los Emuladores: Inicia los emuladores para ejecutar tus pruebas localmente. 
    firebase emulators:start

Escribir y Ejecutar Pruebas

Una vez que tu entorno esté configurado, puedes comenzar a escribir y ejecutar pruebas. Firebase soporta varios tipos de pruebas, incluyendo pruebas unitarias, pruebas de integración y pruebas de extremo a extremo.

Pruebas Unitarias

Las pruebas unitarias se enfocan en probar componentes o funciones individuales de manera aislada.

  • Ejemplo: Probar una función de Firestore. 
    const assert = require('assert');
const firebase = require('@firebase/testing');

describe('Pruebas Unitarias de Firestore', () => {
    it('debería crear un documento en Firestore', async () => {
        const db = firebase.initializeTestApp({ projectId: 'mi-proyecto' }).firestore();
        const docRef = db.collection('usuarios').doc('alice');
        await docRef.set({ nombre: 'Alice' });
        const doc = await docRef.get();
        assert.equal(doc.data().nombre, 'Alice');
    });
});

Pruebas de Integración

Las pruebas de integración se enfocan en probar cómo diferentes partes de tu aplicación funcionan juntas.

  • Ejemplo: Probar Firestore con Firebase Authentication. 
    const assert = require('assert');
const firebase = require('@firebase/testing');

describe('Pruebas de Integración de Firestore', () => {
    it('debería permitir a usuarios autenticados escribir en Firestore', async () => {
        const db = firebase.initializeTestApp({ projectId: 'mi-proyecto', auth: { uid: 'usuario123' } }).firestore();
        const docRef = db.collection('usuarios').doc('usuario123');
        await docRef.set({ nombre: 'Alice' });
        const doc = await docRef.get();
        assert.equal(doc.data().nombre, 'Alice');
    });
});

Pruebas de Extremo a Extremo

Las pruebas de extremo a extremo simulan interacciones reales de usuarios y prueban el flujo completo de la aplicación.

  • Ejemplo: Usar Firebase Test Lab para ejecutar una prueba de Espresso en Android. 
    @RunWith(AndroidJUnit4.class)
public class MainActivityTest {
    @Rule
    public ActivityTestRule<MainActivity> activityRule = new ActivityTestRule<>(MainActivity.class);

    @Test
    public void testUserLogin() {
        onView(withId(R.id.username)).perform(typeText("usuario123"), closeSoftKeyboard());
        onView(withId(R.id.password)).perform(typeText("contraseña"), closeSoftKeyboard());
        onView(withId(R.id.login_button)).perform(click());
        onView(withId(R.id.welcome_message)).check(matches(withText("¡Bienvenido, usuario123!")));
    }
}

Monitoreo de Resultados de Pruebas

Después de ejecutar tus pruebas, es importante monitorear los resultados para asegurarte de que todo esté funcionando como se espera.

  • Consola de Firebase: Visualiza resultados detallados de las pruebas, incluyendo registros, capturas de pantalla y videos.
  • Informes de Pruebas: Genera y descarga informes de pruebas para un análisis más detallado.
  • Alertas: Configura alertas para notificarte sobre fallos o problemas en las pruebas.

Conclusión

Ejecutar pruebas es una parte esencial del desarrollo de aplicaciones robustas. Firebase proporciona herramientas poderosas como Firebase Test Lab y Firebase Emulator Suite para ayudarte a probar tu aplicación de manera efectiva. Al configurar tu entorno de pruebas, escribir varios tipos de pruebas y monitorear los resultados, puedes asegurarte de que tu aplicación sea confiable y funcione bien en diferentes escenarios.
