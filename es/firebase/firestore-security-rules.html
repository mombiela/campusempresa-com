Las Reglas de Seguridad de Firestore son esenciales para controlar el acceso a tu base de datos de Firestore. Te permiten definir cómo debe estructurarse, consultarse y actualizarse tu información. Este tema te guiará desde los conceptos básicos hasta los avanzados de las Reglas de Seguridad de Firestore.

Introducción a las Reglas de Seguridad de Firestore

  • ¿Qué son las Reglas de Seguridad de Firestore?

    • Las Reglas de Seguridad de Firestore son expresiones que controlan el acceso a documentos y colecciones en tu base de datos de Firestore.
    • Están escritas en un lenguaje similar a JavaScript.

  • ¿Por qué son importantes?

    • Ayudan a proteger tus datos de accesos no autorizados y aseguran que solo los usuarios autenticados puedan leer o escribir datos según tus especificaciones.

Conceptos Básicos

Estructura de las Reglas de Seguridad de Firestore

  • Estructura de las Reglas:

    service cloud.firestore {
  match /databases/{database}/documents {
    // Las reglas van aquí
  }
}

  • Ejemplo de Regla Básica:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
    • Esta regla niega todas las operaciones de lectura y escritura.

Permitir Operaciones de Lectura y Escritura

  • Permitir Operaciones de Lectura:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /public/{document=**} {
      allow read: if true;
    }
  }
}
    • Esta regla permite operaciones de lectura en todos los documentos de la colección public.

  • Permitir Operaciones de Escritura:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /public/{document=**} {
      allow write: if request.auth != null;
    }
  }
}
    • Esta regla permite operaciones de escritura solo si el usuario está autenticado.

Conceptos Intermedios

Control de Acceso Basado en Roles

  • Definiendo Roles:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
    }
  }
}
    • Esta regla permite a los usuarios leer y escribir su propio documento de usuario.

  • Ejemplo Basado en Roles:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /projects/{projectId} {
      allow read: if resource.data.public == true;
      allow write: if request.auth.uid == resource.data.ownerId;
    }
  }
}
    • Esta regla permite acceso de lectura a proyectos públicos y acceso de escritura al propietario del proyecto.

Condiciones Avanzadas

  • Condiciones Complejas: 
    service cloud.firestore {
  match /databases/{database}/documents {
    match /posts/{postId} {
      allow read: if request.time < timestamp.date(2023, 12, 31);
      allow write: if request.auth.token.admin == true;
    }
  }
}
    • Esta regla permite acceso de lectura a publicaciones antes de una fecha específica y acceso de escritura solo a usuarios administradores.

Conceptos Avanzados

Funciones Personalizadas

  • Creando Funciones Personalizadas: 
    service cloud.firestore {
  function isOwner(userId) {
    return request.auth.uid == userId;
  }

  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if isOwner(userId);
    }
  }
}
    • Esta regla utiliza una función personalizada para verificar si el usuario es el propietario del documento.

Validación de Datos

  • Validando Datos: 
    service cloud.firestore {
  match /databases/{database}/documents {
    match /products/{productId} {
      allow write: if request.resource.data.price > 0;
    }
  }
}
    • Esta regla asegura que el precio de un producto sea mayor que cero antes de permitir una operación de escritura.

Combinando Múltiples Condiciones

  • Combinando Condiciones: 
    service cloud.firestore {
  match /databases/{database}/documents {
    match /orders/{orderId} {
      allow read: if request.auth != null && request.auth.token.email_verified == true;
      allow write: if request.auth != null && request.auth.token.email_verified == true && request.resource.data.amount > 0;
    }
  }
}
    • Esta regla combina múltiples condiciones para asegurar que el usuario esté autenticado, su correo electrónico esté verificado y el monto del pedido sea mayor que cero.

Conclusión

Las Reglas de Seguridad de Firestore son una herramienta poderosa para asegurar tu base de datos de Firestore. Al entender e implementar estas reglas, puedes controlar el acceso a tus datos y asegurar que solo sean accesibles para usuarios autorizados. Desde permisos básicos de lectura/escritura hasta control de acceso basado en roles y validación de datos avanzada, dominar las Reglas de Seguridad de Firestore es esencial para cualquier desarrollador de Firebase.
