Introducción

Firebase Realtime Database es una base de datos NoSQL alojada en la nube que te permite almacenar y sincronizar datos entre tus usuarios en tiempo real. Esta sección te guiará a través de la configuración de la Base de Datos en Tiempo Real desde cero, cubriendo tanto configuraciones básicas como avanzadas.

Prerrequisitos

Antes de comenzar, asegúrate de tener lo siguiente:

  • Una cuenta de Google
  • Conocimientos básicos de JavaScript (para web) o Kotlin/Java (para Android)
  • Node.js instalado (para web)

Creando un Proyecto de Firebase

  1. Ve a la Consola de Firebase: Navega a Firebase Console.
  2. Añadir un Nuevo Proyecto:
    • Haz clic en "Añadir proyecto".
    • Ingresa el nombre de tu proyecto y sigue las instrucciones en pantalla.
    • Una vez creado el proyecto, serás redirigido al panel de control del proyecto.

Añadiendo la Base de Datos en Tiempo Real a Tu Proyecto

  1. Navega a la Sección de Base de Datos:
    • En la Consola de Firebase, selecciona tu proyecto.
    • Haz clic en "Base de Datos en Tiempo Real" en el menú de la izquierda.
  2. Crear una Base de Datos:
    • Haz clic en "Crear Base de Datos".
    • Elige la ubicación de la base de datos (típicamente la más cercana a tus usuarios).
    • Selecciona las reglas de seguridad (comienza en modo de prueba para propósitos de desarrollo).

Integrando la Base de Datos en Tiempo Real con Tu Aplicación

Para Web

  1. Instalar el SDK de Firebase: 
    npm install firebase
  2. Inicializar Firebase en Tu Aplicación: 
    // Importar Firebase
import firebase from 'firebase/app';
import 'firebase/database';

// Configuración de Firebase de tu aplicación web
const firebaseConfig = {
  apiKey: "TU_API_KEY",
  authDomain: "TU_PROYECTO_ID.firebaseapp.com",
  databaseURL: "https://TU_PROYECTO_ID.firebaseio.com",
  projectId: "TU_PROYECTO_ID",
  storageBucket: "TU_PROYECTO_ID.appspot.com",
  messagingSenderId: "TU_MESSAGING_SENDER_ID",
  appId: "TU_APP_ID"
};

// Inicializar Firebase
firebase.initializeApp(firebaseConfig);

// Obtener una referencia al servicio de base de datos
const database = firebase.database();

Para Android

  1. Añadir el SDK de Firebase:
    • Abre build.gradle (nivel del Proyecto) y añade lo siguiente: 
      classpath 'com.google.gms:google-services:4.3.3'
    • Abre build.gradle (nivel de la Aplicación) y añade lo siguiente: 
      implementation 'com.google.firebase:firebase-database:19.5.1'
apply plugin: 'com.google.gms.google-services'
  2. Inicializar Firebase en Tu Aplicación: 
    // Importar Firebase
import com.google.firebase.database.DatabaseReference;
import com.google.firebase.database.FirebaseDatabase;

// Inicializar Firebase
FirebaseDatabase database = FirebaseDatabase.getInstance();
DatabaseReference myRef = database.getReference("message");

// Escribir un mensaje en la base de datos
myRef.setValue("¡Hola, Mundo!");

Operaciones Básicas CRUD

Crear y Leer Datos

  • Web:

    // Crear datos
database.ref('users/1').set({
  username: "john_doe",
  email: "john@example.com"
});

// Leer datos
database.ref('users/1').once('value').then((snapshot) => {
  const data = snapshot.val();
  console.log(data);
});

  • Android:

    // Crear datos
myRef.child("users").child("1").setValue(new User("john_doe", "john@example.com"));

// Leer datos
myRef.child("users").child("1").addListenerForSingleValueEvent(new ValueEventListener() {
    @Override
    public void onDataChange(DataSnapshot dataSnapshot) {
        User user = dataSnapshot.getValue(User.class);
        System.out.println(user.username);
    }

    @Override
    public void onCancelled(DatabaseError databaseError) {
        // Manejar posibles errores.
    }
});

Actualizar y Eliminar Datos

  • Web:

    // Actualizar datos
database.ref('users/1').update({
  username: "john_doe_updated"
});

// Eliminar datos
database.ref('users/1').remove();

  • Android:

    // Actualizar datos
myRef.child("users").child("1").child("username").setValue("john_doe_updated");

// Eliminar datos
myRef.child("users").child("1").removeValue();

Configuración Avanzada

Reglas de Seguridad

  • Configuración de Reglas:
    • Navega a la pestaña "Reglas" en la sección de Base de Datos en Tiempo Real.
    • Define reglas para asegurar tus datos.
    {
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null"
  }
}

Capacidades Offline

  • Web:

    // Habilitar capacidades offline
firebase.database().goOffline();
firebase.database().goOnline();

  • Android:

    // Habilitar capacidades offline
FirebaseDatabase.getInstance().setPersistenceEnabled(true);

Conclusión

Configurar Firebase Realtime Database implica crear un proyecto de Firebase, integrar la base de datos con tu aplicación y realizar operaciones básicas CRUD. A medida que progreses, puedes configurar características avanzadas como reglas de seguridad y capacidades offline para mejorar la funcionalidad y seguridad de tu aplicación. Con estas bases, estás bien equipado para construir aplicaciones colaborativas en tiempo real.
