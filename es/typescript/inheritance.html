Herencia en TypeScript
Modificadores de Acceso ►

La herencia es un concepto fundamental en la programación orientada a objetos que permite a una clase heredar propiedades y métodos de otra clase. En TypeScript, la herencia ayuda a crear una jerarquía de clases que comparten una estructura y comportamiento comunes, promoviendo la reutilización y mantenibilidad del código.

Conceptos Clave de la Herencia

  • Clase Base (Clase Padre): La clase cuyas propiedades y métodos son heredados.
  • Clase Derivada (Clase Hija): La clase que hereda propiedades y métodos de la clase base.
  • Palabra Clave extends: Utilizada para crear una clase derivada a partir de una clase base.
  • Sobrescritura de Métodos: La capacidad de una clase derivada para proporcionar una implementación específica de un método que ya está definido en su clase base.
  • Palabra Clave super: Utilizada para llamar al constructor o métodos de la clase base desde la clase derivada.

Creando una Clase Base

Comencemos definiendo una clase base simple llamada Animal.

class Animal {
    name: string;

    constructor(name: string) {
        this.name = name;
    }

    makeSound(): void {
        console.log(`${this.name} hace un sonido.`);
    }
}

En este ejemplo:

  • La clase Animal tiene una propiedad name y un método makeSound.
  • El constructor inicializa la propiedad name.

Creando una Clase Derivada

Ahora, creemos una clase derivada Dog que hereda de Animal.

class Dog extends Animal {
    breed: string;

    constructor(name: string, breed: string) {
        super(name); // Llama al constructor de la clase base
        this.breed = breed;
    }

    makeSound(): void {
        console.log(`${this.name} ladra.`);
    }
}

En este ejemplo:

  • La clase Dog extiende la clase Animal usando la palabra clave extends.
  • La clase Dog tiene una propiedad adicional breed.
  • El constructor de Dog llama al constructor de Animal usando super(name).
  • El método makeSound es sobrescrito para proporcionar una implementación específica para Dog.

Usando la Clase Derivada

Creemos una instancia de la clase Dog y usemos sus métodos.

const myDog = new Dog('Buddy', 'Golden Retriever');
myDog.makeSound(); // Salida: Buddy ladra.
console.log(`Mi perro es un ${myDog.breed}.`); // Salida: Mi perro es un Golden Retriever.

Sobrescritura de Métodos

La sobrescritura de métodos permite a una clase derivada proporcionar una implementación específica de un método que ya está definido en su clase base. Esto es útil cuando la clase derivada necesita cambiar o extender el comportamiento del método de la clase base.

class Cat extends Animal {
    constructor(name: string) {
        super(name);
    }

    makeSound(): void {
        console.log(`${this.name} maúlla.`);
    }
}

const myCat = new Cat('Whiskers');
myCat.makeSound(); // Salida: Whiskers maúlla.

Resumen

  • La herencia permite a una clase heredar propiedades y métodos de otra clase, promoviendo la reutilización del código.
  • La palabra clave extends se utiliza para crear una clase derivada a partir de una clase base.
  • La palabra clave super se utiliza para llamar al constructor o métodos de la clase base.
  • La sobrescritura de métodos permite a una clase derivada proporcionar una implementación específica de un método definido en la clase base.

Al comprender y utilizar la herencia en TypeScript, puedes crear un código más estructurado y mantenible, aprovechando el poder de los principios de la programación orientada a objetos.
