Espacios de nombres en TypeScript
Resolución de Módulos ►

Los espacios de nombres en TypeScript proporcionan una forma de organizar y agrupar código relacionado, lo que facilita su gestión y evita conflictos de nombres. Son particularmente útiles en bases de código grandes donde varios desarrolladores pueden estar trabajando en diferentes partes de la aplicación.

Introducción a los espacios de nombres

Los espacios de nombres son una característica específica de TypeScript que te permite agrupar código relacionado. Son similares a los módulos, pero se utilizan para organizar el código dentro de un solo archivo o en varios archivos.

Conceptos clave

  • Declaración de espacio de nombres: Usa la palabra clave namespace para declarar un espacio de nombres.
  • Exportación de miembros: Usa la palabra clave export para hacer que variables, funciones, clases o interfaces estén disponibles fuera del espacio de nombres.
  • Espacios de nombres anidados: Los espacios de nombres pueden anidarse dentro de otros espacios de nombres.
  • Referencia a espacios de nombres: Usa la directiva /// <reference path="..."/> para referenciar espacios de nombres en diferentes archivos.

Creación y uso de espacios de nombres

Declaración básica de espacio de nombres

Para declarar un espacio de nombres, usa la palabra clave namespace seguida del nombre del espacio de nombres y un bloque de código.

namespace Utility {
    export function log(message: string): void {
        console.log(message);
    }

    export function error(message: string): void {
        console.error(message);
    }
}

En este ejemplo, hemos creado un espacio de nombres llamado Utility con dos funciones exportadas: log y error.

Acceso a los miembros del espacio de nombres

Para acceder a los miembros de un espacio de nombres, usa la notación de punto.

Utility.log("Este es un mensaje de registro.");
Utility.error("Este es un mensaje de error.");

Espacios de nombres anidados

Los espacios de nombres pueden anidarse para organizar aún más el código.

namespace App {
    export namespace Models {
        export class User {
            constructor(public name: string, public age: number) {}
        }
    }

    export namespace Services {
        export function getUserInfo(user: Models.User): string {
            return `${user.name} tiene ${user.age} años.`;
        }
    }
}

Acceso a los miembros del espacio de nombres anidado:

const user = new App.Models.User("Alice", 30);
console.log(App.Services.getUserInfo(user));

Espacios de nombres en varios archivos

Para usar espacios de nombres en varios archivos, necesitas referenciarlos usando la directiva /// <reference path="..."/>.

Archivo: `models.ts`

namespace App.Models {
    export class User {
        constructor(public name: string, public age: number) {}
    }
}

Archivo: `services.ts`

/// <reference path="models.ts" />

namespace App.Services {
    export function getUserInfo(user: App.Models.User): string {
        return `${user.name} tiene ${user.age} años.`;
    }
}

Archivo: `main.ts`

/// <reference path="models.ts" />
/// <reference path="services.ts" />

const user = new App.Models.User("Alice", 30);
console.log(App.Services.getUserInfo(user));

Conclusión

Los espacios de nombres en TypeScript son una herramienta poderosa para organizar y gestionar el código, especialmente en proyectos grandes. Ayudan a evitar conflictos de nombres y hacen que la base de código sea más mantenible. Al entender cómo declarar, acceder y referenciar espacios de nombres, puedes estructurar eficazmente tus aplicaciones TypeScript.

Resumen

  • Espacios de nombres: Agrupa código relacionado usando la palabra clave namespace.
  • Exportación de miembros: Usa la palabra clave export para hacer que los miembros sean accesibles fuera del espacio de nombres.
  • Espacios de nombres anidados: Organiza el código aún más anidando espacios de nombres.
  • Referencia entre archivos: Usa la directiva /// <reference path="..."/> para referenciar espacios de nombres en diferentes archivos.

Al dominar estos conceptos, estarás bien equipado para gestionar proyectos complejos de TypeScript con facilidad.
