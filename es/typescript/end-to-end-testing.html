Pruebas de extremo a extremo en TypeScript
Opciones del Compilador de TypeScript ►

Las pruebas de extremo a extremo (E2E) son una parte crucial del ciclo de vida del desarrollo de software, asegurando que todo el flujo de la aplicación funcione como se espera. Este tema te guiará a través de los conceptos básicos de las pruebas E2E en TypeScript, desde la configuración de tu entorno hasta la escritura y ejecución de tus primeras pruebas.

¿Qué son las pruebas de extremo a extremo?

Las pruebas de extremo a extremo implican probar la funcionalidad completa de una aplicación, desde la interfaz de usuario hasta el backend y la base de datos. Simulan escenarios reales de usuario para validar el comportamiento del sistema.

  • Propósito: Asegurar que la aplicación funcione como un todo.
  • Alcance: Cubre toda la aplicación, incluyendo la interfaz de usuario, APIs, bases de datos y otros servicios integrados.
  • Herramientas: Las herramientas comunes incluyen Cypress, Selenium y Playwright.

Configurando tu entorno

Antes de escribir pruebas E2E, necesitas configurar tu entorno de pruebas. Para esta guía, usaremos Cypress, un popular marco de pruebas E2E.

Instalando Cypress

Primero, instala Cypress a través de npm:

npm install cypress --save-dev

Configurando Cypress

Después de la instalación, necesitas configurar Cypress. Crea un archivo cypress.json en la raíz de tu proyecto:

{
  "baseUrl": "http://localhost:3000",
  "integrationFolder": "cypress/integration"
}
  • baseUrl: La URL donde tu aplicación está ejecutándose.
  • integrationFolder: La carpeta donde se ubicarán tus archivos de prueba.

Escribiendo tu primera prueba E2E

Vamos a escribir una simple prueba E2E para verificar si la página de inicio se carga correctamente.

Creando un archivo de prueba

Crea un nuevo archivo llamado home.spec.ts en la carpeta cypress/integration:

describe('Homepage', () => {
  it('should load the homepage', () => {
    cy.visit('/');
    cy.contains('Welcome to My App');
  });
});

Explicación

  • describe: Agrupa pruebas relacionadas.
  • it: Define un caso de prueba único.
  • cy.visit('/'): Navega a la página de inicio.
  • cy.contains('Welcome to My App'): Verifica si el texto "Welcome to My App" está presente en la página.

Ejecutando tus pruebas

Para ejecutar tus pruebas, usa el siguiente comando:

npx cypress open

Esto abrirá el Cypress Test Runner, donde puedes seleccionar y ejecutar tus pruebas.

Conceptos avanzados de pruebas E2E

A medida que te sientas más cómodo con las pruebas E2E, puedes explorar conceptos más avanzados.

Manejo de autenticación

Muchas aplicaciones requieren autenticación. Puedes manejar esto en Cypress usando comandos personalizados.

Ejemplo: Comando personalizado para iniciar sesión

Crea un nuevo archivo llamado commands.ts en la carpeta cypress/support:

Cypress.Commands.add('login', (username, password) => {
  cy.request('POST', '/api/login', {
    username,
    password
  }).then((response) => {
    window.localStorage.setItem('authToken', response.body.token);
  });
});

Usa este comando personalizado en tus pruebas:

describe('Dashboard', () => {
  beforeEach(() => {
    cy.login('testuser', 'password123');
  });

  it('should load the dashboard', () => {
    cy.visit('/dashboard');
    cy.contains('Dashboard');
  });
});

Simulación de solicitudes API

Para aislar las pruebas de frontend de las dependencias del backend, puedes simular solicitudes API.

Ejemplo: Simulación de una solicitud API

describe('User Profile', () => {
  beforeEach(() => {
    cy.intercept('GET', '/api/user', {
      statusCode: 200,
      body: { name: 'John Doe', email: 'john.doe@example.com' }
    }).as('getUser');
  });

  it('should display user profile', () => {
    cy.visit('/profile');
    cy.wait('@getUser');
    cy.contains('John Doe');
    cy.contains('john.doe@example.com');
  });
});

Conclusión

Las pruebas de extremo a extremo son esenciales para asegurar la fiabilidad y funcionalidad de tu aplicación. Usando herramientas como Cypress y escribiendo pruebas exhaustivas, puedes detectar errores temprano y mejorar la calidad general de tu software. A medida que avances, explora escenarios más complejos e integra las pruebas E2E en tu pipeline de integración continua para pruebas automatizadas.
