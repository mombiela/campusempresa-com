Opciones del Compilador de TypeScript
Archivos de Declaración ►

Introducción

TypeScript proporciona un compilador poderoso (tsc) que viene con una amplia gama de opciones para personalizar el proceso de compilación. Entender estas opciones es crucial para configurar tus proyectos de TypeScript de manera efectiva.

Opciones Básicas del Compilador

`target`

  • Especifica la versión de ECMAScript objetivo.
  • Valores comunes: ES5, ES6/ES2015, ES2016, ES2017, ES2018, ES2019, ES2020, ES2021, ES2022, ESNext.
{
  "compilerOptions": {
    "target": "ES6"
  }
}

`module`

  • Determina el sistema de módulos a utilizar.
  • Valores comunes: CommonJS, AMD, System, UMD, ES6/ES2015, ESNext.
{
  "compilerOptions": {
    "module": "CommonJS"
  }
}

`lib`

  • Especifica los archivos de biblioteca que se incluirán en la compilación.
  • Valores comunes: ES5, ES6, DOM, DOM.Iterable, ScriptHost, WebWorker.
{
  "compilerOptions": {
    "lib": ["ES6", "DOM"]
  }
}

Opciones de Verificación Estricta de Tipos

`strict`

  • Habilita todas las opciones de verificación estricta de tipos.
  • Incluye: strictNullChecks, strictFunctionTypes, strictPropertyInitialization, noImplicitAny, noImplicitThis, alwaysStrict.
{
  "compilerOptions": {
    "strict": true
  }
}

`noImplicitAny`

  • Genera un error en expresiones y declaraciones con un tipo implícito any.
{
  "compilerOptions": {
    "noImplicitAny": true
  }
}

`strictNullChecks`

  • Asegura que null y undefined solo sean asignables a ellos mismos y any.
{
  "compilerOptions": {
    "strictNullChecks": true
  }
}

Opciones de Resolución de Módulos

`baseUrl`

  • Especifica el directorio base para resolver nombres de módulos no relativos.
{
  "compilerOptions": {
    "baseUrl": "./src"
  }
}

`paths`

  • Una serie de entradas que re-mapean importaciones a ubicaciones de búsqueda relativas al baseUrl.
{
  "compilerOptions": {
    "baseUrl": "./",
    "paths": {
      "@app/*": ["src/app/*"],
      "@components/*": ["src/components/*"]
    }
  }
}

Opciones de Salida

`outDir`

  • Redirige la estructura de salida al directorio.
{
  "compilerOptions": {
    "outDir": "./dist"
  }
}

`rootDir`

  • Especifica el directorio raíz de los archivos de entrada.
{
  "compilerOptions": {
    "rootDir": "./src"
  }
}

`sourceMap`

  • Genera archivos .map correspondientes.
{
  "compilerOptions": {
    "sourceMap": true
  }
}

Opciones Avanzadas

`incremental`

  • Habilita la compilación incremental.
{
  "compilerOptions": {
    "incremental": true
  }
}

`composite`

  • Habilita la compilación de proyectos.
{
  "compilerOptions": {
    "composite": true
  }
}

`declaration`

  • Genera archivos .d.ts correspondientes.
{
  "compilerOptions": {
    "declaration": true
  }
}

Configuración de Ejemplo

Aquí tienes un archivo tsconfig.json de ejemplo que combina varias de las opciones discutidas:

{
  "compilerOptions": {
    "target": "ES6",
    "module": "CommonJS",
    "lib": ["ES6", "DOM"],
    "strict": true,
    "baseUrl": "./",
    "paths": {
      "@app/*": ["src/app/*"],
      "@components/*": ["src/components/*"]
    },
    "outDir": "./dist",
    "rootDir": "./src",
    "sourceMap": true,
    "incremental": true,
    "composite": true,
    "declaration": true
  }
}

Conclusión

Entender y configurar las opciones del compilador de TypeScript puede mejorar significativamente tu flujo de trabajo de desarrollo. Al aprovechar estas opciones, puedes adaptar el proceso de compilación para satisfacer las necesidades específicas de tu proyecto, asegurando una mejor seguridad de tipos, una salida optimizada y una mejor resolución de módulos.
