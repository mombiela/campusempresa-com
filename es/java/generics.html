Los genéricos en Java te permiten crear clases, interfaces y métodos con un marcador de posición para el tipo de datos con los que operan. Esto proporciona verificaciones de tipo más estrictas en tiempo de compilación y elimina la necesidad de conversiones de tipo.

Introducción a los Genéricos

  • Definición: Los genéricos permiten que los tipos (clases e interfaces) sean parámetros al definir clases, interfaces y métodos.
  • Propósito: Proporcionan una forma de reutilizar el mismo código con diferentes entradas y proporcionan verificaciones de tipo más estrictas en tiempo de compilación.

Sintaxis Básica

// Un ejemplo de clase genérica
public class Box<T> {
    private T t;

    public void set(T t) {
        this.t = t;
    }

    public T get() {
        return t;
    }
}

En este ejemplo, T es un parámetro de tipo que será reemplazado con un tipo concreto cuando se cree una instancia de Box.

Beneficios de Usar Genéricos

  • Seguridad de Tipo: Asegura que obtienes verificaciones en tiempo de compilación y evitas ClassCastException.
  • Reutilización de Código: Escribe un método o clase una vez y úsalo para cualquier tipo.
  • Eliminación de Conversiones de Tipo: Reduce la necesidad de conversiones de tipo explícitas.

Ejemplo Sin Genéricos

List list = new ArrayList();
list.add("Hello");
String s = (String) list.get(0); // Necesita conversión

Ejemplo Con Genéricos

List<String> list = new ArrayList<>();
list.add("Hello");
String s = list.get(0); // No necesita conversión

Métodos Genéricos

También puedes definir métodos genéricos en Java.

Sintaxis

public static <T> void printArray(T[] array) {
    for (T element : array) {
        System.out.println(element);
    }
}

Aquí, <T> antes del tipo de retorno void indica que el método es genérico.

Ejemplo de Uso

Integer[] intArray = {1, 2, 3, 4, 5};
String[] stringArray = {"Hello", "World"};

printArray(intArray); // Imprime enteros
printArray(stringArray); // Imprime cadenas

Parámetros de Tipo Acotados

A veces quieres restringir los tipos que pueden ser usados como argumentos de tipo. Esto se hace usando parámetros de tipo acotados.

Límite Superior

public <T extends Number> void printNumbers(T[] numbers) {
    for (T number : numbers) {
        System.out.println(number);
    }
}

En este ejemplo, T solo puede ser un tipo que extienda Number.

Límite Inferior

public static void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}

Aquí, ? super Integer significa cualquier tipo que sea un supertipo de Integer.

Comodines en Genéricos

Los comodines se usan para especificar un tipo desconocido.

Comodín No Acotado

public void printList(List<?> list) {
    for (Object element : list) {
        System.out.println(element);
    }
}

<?> representa un tipo desconocido.

Comodines Acotados

  • Comodín Acotado Superior: <? extends Number>
  • Comodín Acotado Inferior: <? super Integer>

Ejemplo

public void processElements(List<? extends Number> list) {
    for (Number number : list) {
        System.out.println(number);
    }
}

Clases Genéricas

Puedes crear clases que puedan trabajar con cualquier tipo.

Ejemplo

public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() {
        return key;
    }

    public V getValue() {
        return value;
    }
}

Aquí, K y V son parámetros de tipo.

Conclusión

Los genéricos en Java proporcionan una forma poderosa y flexible de escribir código seguro y reutilizable. Al comprender y utilizar los genéricos, puedes crear aplicaciones más robustas y mantenibles. Recuerda usar parámetros de tipo acotados y comodines para mejorar aún más la flexibilidad y seguridad de tu código genérico.
