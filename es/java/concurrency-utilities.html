Introducción a la Concurrencia en Java

La concurrencia en Java permite que múltiples hilos se ejecuten simultáneamente, haciendo que los programas sean más eficientes y responsivos. Java proporciona un conjunto rico de utilidades de concurrencia en el paquete java.util.concurrent para ayudar a los desarrolladores a gestionar hilos y sincronización.

Conceptos Clave

  • Hilo: La unidad más pequeña de procesamiento. Java proporciona la clase Thread y la interfaz Runnable para crear y gestionar hilos.
  • Marco de Trabajo Executor: Un reemplazo de alto nivel para trabajar directamente con hilos. Proporciona un grupo de hilos para ejecutar tareas.
  • Sincronización: Mecanismos para controlar el acceso de múltiples hilos a recursos compartidos.
  • Locks: Más flexibles y poderosos que los bloques sincronizados para controlar el acceso a recursos compartidos.
  • Variables Atómicas: Variables que soportan programación segura para hilos sin bloqueo en variables individuales.
  • Colecciones Concurrentes: Versiones seguras para hilos de colecciones estándar como List, Map y Queue.

Creación y Gestión de Hilos

Creación Básica de Hilos

public class BasicThreadExample {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("¡Hola desde un hilo!");
        });
        thread.start();
    }
}
  • Explicación: Este ejemplo crea un nuevo hilo usando una expresión lambda y lo inicia. El método start() comienza la ejecución del hilo.

Usando la Interfaz Runnable

public class RunnableExample implements Runnable {
    @Override
    public void run() {
        System.out.println("¡Hola desde un Runnable!");
    }

    public static void main(String[] args) {
        Thread thread = new Thread(new RunnableExample());
        thread.start();
    }
}
  • Explicación: Este ejemplo implementa la interfaz Runnable y pasa una instancia de RunnableExample a un nuevo objeto Thread.

Marco de Trabajo Executor

Usando Executors

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ExecutorExample {
    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        executor.submit(() -> System.out.println("Tarea 1"));
        executor.submit(() -> System.out.println("Tarea 2"));
        executor.shutdown();
    }
}
  • Explicación: Este ejemplo usa el ExecutorService para gestionar un grupo de hilos. El método submit se usa para ejecutar tareas, y shutdown se llama para detener el executor de aceptar nuevas tareas.

Sincronización

Métodos y Bloques Sincronizados

public class SynchronizedExample {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public static void main(String[] args) {
        SynchronizedExample example = new SynchronizedExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explicación: El método increment está sincronizado, asegurando que solo un hilo pueda ejecutarlo a la vez.

Locks

Usando ReentrantLock

import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class LockExample {
    private final Lock lock = new ReentrantLock();
    private int count = 0;

    public void increment() {
        lock.lock();
        try {
            count++;
        } finally {
            lock.unlock();
        }
    }

    public static void main(String[] args) {
        LockExample example = new LockExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explicación: Este ejemplo usa ReentrantLock para controlar el acceso al método increment. El método lock se llama antes de la sección crítica, y unlock se llama en un bloque finally para asegurar que siempre se ejecute.

Variables Atómicas

Usando AtomicInteger

import java.util.concurrent.atomic.AtomicInteger;

public class AtomicExample {
    private final AtomicInteger count = new AtomicInteger(0);

    public void increment() {
        count.incrementAndGet();
    }

    public static void main(String[] args) {
        AtomicExample example = new AtomicExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explicación: Este ejemplo usa AtomicInteger para realizar operaciones atómicas en una variable entera sin usar sincronización explícita.

Colecciones Concurrentes

Usando ConcurrentHashMap

import java.util.concurrent.ConcurrentHashMap;
import java.util.Map;

public class ConcurrentMapExample {
    public static void main(String[] args) {
        Map<String, Integer> map = new ConcurrentHashMap<>();
        map.put("key1", 1);
        map.put("key2", 2);

        Thread t1 = new Thread(() -> map.put("key3", 3));
        Thread t2 = new Thread(() -> map.put("key4", 4));
        t1.start();
        t2.start();
    }
}
  • Explicación: Este ejemplo usa ConcurrentHashMap para realizar operaciones seguras para hilos en un mapa.

Conclusión

La concurrencia en Java es una característica poderosa que permite a los desarrolladores escribir aplicaciones eficientes y responsivas. Al entender y utilizar las utilidades de concurrencia proporcionadas por Java, como hilos, el marco de trabajo Executor, sincronización, locks, variables atómicas y colecciones concurrentes, los desarrolladores pueden gestionar efectivamente tareas concurrentes y asegurar la seguridad de los hilos en sus aplicaciones.
