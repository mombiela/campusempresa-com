Introducción a la Encapsulación

La encapsulación es uno de los cuatro conceptos fundamentales de la Programación Orientada a Objetos (OOP). Es el mecanismo de envolver los datos (variables) y el código (métodos) que actúan sobre los datos en una sola unidad, conocida como clase. La encapsulación ayuda a proteger los datos del acceso y modificación no autorizados.

Conceptos Clave de la Encapsulación

  • Ocultación de Datos: Restringir el acceso directo a algunos de los componentes de un objeto.
  • Modificadores de Acceso: Palabras clave que establecen la accesibilidad de clases, métodos y otros miembros.
  • Getters y Setters: Métodos utilizados para acceder y actualizar el valor de variables privadas.

Modificadores de Acceso

Java proporciona cuatro tipos de modificadores de acceso para controlar la visibilidad de los miembros de la clase:

  • private: Accesible solo dentro de la misma clase.
  • default (sin modificador): Accesible solo dentro del mismo paquete.
  • protected: Accesible dentro del mismo paquete y subclases.
  • public: Accesible desde cualquier otra clase.

Ejemplo de Modificadores de Acceso

public class Example {
    private int privateVar; // acceso privado
    int defaultVar;         // acceso por defecto
    protected int protectedVar; // acceso protegido
    public int publicVar;   // acceso público

    // Getter y Setter para privateVar
    public int getPrivateVar() {
        return privateVar;
    }

    public void setPrivateVar(int privateVar) {
        this.privateVar = privateVar;
    }
}

Getters y Setters

Los getters y setters son métodos que proporcionan acceso controlado a las variables privadas de una clase.

Ejemplo de Getters y Setters

public class Person {
    private String name;
    private int age;

    // Getter para name
    public String getName() {
        return name;
    }

    // Setter para name
    public void setName(String name) {
        this.name = name;
    }

    // Getter para age
    public int getAge() {
        return age;
    }

    // Setter para age
    public void setAge(int age) {
        if (age > 0) { // Añadiendo una validación simple
            this.age = age;
        }
    }
}

Beneficios de la Encapsulación

  • Control sobre los Datos: Al usar getters y setters, puedes agregar lógica para controlar cómo se acceden o modifican los datos.
  • Mayor Flexibilidad: Puedes cambiar la implementación interna sin afectar el código externo.
  • Mejor Mantenibilidad: El código encapsulado es más fácil de mantener y entender.
  • Mayor Seguridad: La encapsulación ayuda a proteger los datos del acceso y modificación no autorizados.

Ejemplo Práctico

Vamos a crear una clase simple BankAccount que demuestre la encapsulación.

Ejemplo de la Clase BankAccount

public class BankAccount {
    private String accountNumber;
    private double balance;

    // Constructor
    public BankAccount(String accountNumber, double initialBalance) {
        this.accountNumber = accountNumber;
        this.balance = initialBalance;
    }

    // Getter para accountNumber
    public String getAccountNumber() {
        return accountNumber;
    }

    // Getter para balance
    public double getBalance() {
        return balance;
    }

    // Método para depositar dinero
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    // Método para retirar dinero
    public boolean withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }
}

Usando la Clase BankAccount

public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount("123456789", 1000.0);

        // Depositar dinero
        account.deposit(500.0);
        System.out.println("Saldo después del depósito: " + account.getBalance());

        // Retirar dinero
        if (account.withdraw(200.0)) {
            System.out.println("Retiro exitoso. Saldo restante: " + account.getBalance());
        } else {
            System.out.println("Retiro fallido. Saldo insuficiente.");
        }
    }
}

Conclusión

La encapsulación es un poderoso concepto de OOP que ayuda a proteger los datos y mantener la integridad del código. Al usar modificadores de acceso y métodos getter/setter, puedes controlar cómo se acceden y modifican los datos, lo que lleva a un código más seguro y mantenible. Entender y aplicar la encapsulación es esencial para escribir aplicaciones Java robustas.
