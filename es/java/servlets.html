Introducción a los Servlets

Los servlets son programas Java que se ejecutan en un servidor web, manejando solicitudes de clientes y generando contenido web dinámico. Son un componente clave de Java EE (Enterprise Edition) y se utilizan para crear aplicaciones web.

Conceptos Clave

  • Contenedor de Servlets: El entorno donde se ejecutan los servlets, gestionando su ciclo de vida y proporcionando servicios necesarios.
  • Protocolo HTTP: El protocolo utilizado por los servlets para comunicarse con los clientes.
  • Solicitud y Respuesta: El mecanismo mediante el cual los clientes envían solicitudes al servidor y reciben respuestas.

Configuración de Tu Entorno

Antes de sumergirte en los servlets, asegúrate de tener lo siguiente configurado:

  • Java Development Kit (JDK): Necesario para compilar y ejecutar programas Java.
  • Apache Tomcat: Un contenedor de servlets popular.
  • IDE: Entorno de Desarrollo Integrado como Eclipse o IntelliJ IDEA para un desarrollo más fácil.

Creando Tu Primer Servlet

Guía Paso a Paso

  1. Crea un Proyecto Web Dinámico en tu IDE.
  2. Añade una Clase Servlet: 
    import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/hello")
public class HelloWorldServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        response.setContentType("text/html");
        response.getWriter().println("<h1>Hello, World!</h1>");
    }
}
  3. Despliega y Ejecuta: Despliega tu proyecto en el contenedor de servlets (por ejemplo, Tomcat) y ejecútalo. Accede al servlet a través de http://localhost:8080/your_project_name/hello.

Explicación

  • Anotaciones: @WebServlet("/hello") mapea el servlet al patrón de URL /hello.
  • Método doGet: Maneja solicitudes GET del cliente. response.setContentType("text/html") establece el tipo de respuesta, y response.getWriter().println("<h1>Hello, World!</h1>") envía una respuesta HTML.

Manejo de Solicitudes HTTP

Los servlets pueden manejar varios tipos de solicitudes HTTP, como GET, POST, PUT, DELETE, etc.

Manejo de Solicitudes POST

protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
    String name = request.getParameter("name");
    response.setContentType("text/html");
    response.getWriter().println("<h1>Hello, " + name + "!</h1>");
}
  • Método doPost: Maneja solicitudes POST. request.getParameter("name") recupera datos del formulario enviados por el cliente.

Gestión de Sesiones

Las sesiones permiten a los servlets mantener el estado a través de múltiples solicitudes del mismo cliente.

Uso de HttpSession

protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
    HttpSession session = request.getSession();
    session.setAttribute("username", "JohnDoe");
    response.getWriter().println("Sesión creada con nombre de usuario: JohnDoe");
}
  • HttpSession: request.getSession() recupera la sesión actual o crea una nueva. session.setAttribute("username", "JohnDoe") almacena datos en la sesión.

Temas Avanzados

Filtros

Los filtros se utilizan para preprocesar o postprocesar solicitudes y respuestas.

import java.io.IOException;
import javax.servlet.Filter;
import javax.servlet.FilterChain;
import javax.servlet.FilterConfig;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebFilter;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebFilter("/hello")
public class LoggingFilter implements Filter {
    public void doFilter(HttpServletRequest request, HttpServletResponse response, FilterChain chain) throws IOException, ServletException {
        System.out.println("Solicitud recibida en " + new java.util.Date());
        chain.doFilter(request, response);
    }
}
  • Método doFilter: Intercepta la solicitud y la respuesta. chain.doFilter(request, response) pasa el control al siguiente filtro o servlet.

Listeners

Los listeners se utilizan para monitorear eventos en una aplicación web.

import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;
import javax.servlet.annotation.WebListener;

@WebListener
public class AppContextListener implements ServletContextListener {
    public void contextInitialized(ServletContextEvent sce) {
        System.out.println("Aplicación iniciada");
    }

    public void contextDestroyed(ServletContextEvent sce) {
        System.out.println("Aplicación detenida");
    }
}
  • ServletContextListener: Los métodos contextInitialized y contextDestroyed manejan eventos de inicio y cierre de la aplicación.

Conclusión

Los servlets son una herramienta poderosa para construir aplicaciones web dinámicas en Java. Al comprender los conceptos básicos de los servlets, manejar solicitudes HTTP, gestionar sesiones y utilizar características avanzadas como filtros y listeners, puedes crear aplicaciones web robustas y escalables. Practica creando varios servlets y experimentando con diferentes características para profundizar tu comprensión.
