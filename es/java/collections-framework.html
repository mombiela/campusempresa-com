Introducción al Marco de Colecciones

El Marco de Colecciones de Java proporciona un conjunto de interfaces y clases para manejar un grupo de objetos como una sola unidad. Es parte del paquete java.util e incluye varias estructuras de datos como listas, conjuntos y mapas.

  • Conceptos Clave:
    • Interfaz Collection: La interfaz raíz para todas las clases de colecciones.
    • Interfaz List: Una colección ordenada (también conocida como secuencia).
    • Interfaz Set: Una colección que no permite elementos duplicados.
    • Interfaz Map: Un objeto que mapea claves a valores, sin permitir claves duplicadas.

Interfaz Collection

La interfaz Collection es la interfaz raíz del Marco de Colecciones de Java. Proporciona métodos básicos que todas las colecciones deben implementar.

  • Métodos Comunes:
    • add(E e): Añade un elemento a la colección.
    • remove(Object o): Elimina un elemento especificado de la colección.
    • size(): Devuelve el número de elementos en la colección.
    • clear(): Elimina todos los elementos de la colección.
    • isEmpty(): Verifica si la colección está vacía.
Collection<String> collection = new ArrayList<>();
collection.add("Hello");
collection.add("World");
System.out.println(collection.size()); // Salida: 2

Interfaz List

La interfaz List extiende la interfaz Collection y representa una colección ordenada. Los elementos pueden ser accedidos por su índice entero.

  • Implementaciones:

    • ArrayList: Una implementación de array redimensionable.
    • LinkedList: Una implementación de lista doblemente enlazada.

  • Métodos Comunes:

    • get(int index): Devuelve el elemento en la posición especificada.
    • set(int index, E element): Reemplaza el elemento en la posición especificada.
    • add(int index, E element): Inserta el elemento en la posición especificada.
    • remove(int index): Elimina el elemento en la posición especificada.
List<String> list = new ArrayList<>();
list.add("Apple");
list.add("Banana");
list.add(1, "Cherry");
System.out.println(list.get(1)); // Salida: Cherry

Interfaz Set

La interfaz Set extiende la interfaz Collection y representa una colección que no permite elementos duplicados.

  • Implementaciones:

    • HashSet: Una implementación de tabla hash.
    • LinkedHashSet: Una implementación de tabla hash y lista enlazada.
    • TreeSet: Una implementación de árbol Rojo-Negro.

  • Métodos Comunes:

    • add(E e): Añade el elemento especificado si no está ya presente.
    • remove(Object o): Elimina el elemento especificado si está presente.
Set<String> set = new HashSet<>();
set.add("One");
set.add("Two");
set.add("One"); // Elemento duplicado, no será añadido
System.out.println(set.size()); // Salida: 2

Interfaz Map

La interfaz Map no extiende la interfaz Collection. Representa una colección de pares clave-valor.

  • Implementaciones:

    • HashMap: Una implementación de tabla hash.
    • LinkedHashMap: Una implementación de tabla hash y lista enlazada.
    • TreeMap: Una implementación de árbol Rojo-Negro.

  • Métodos Comunes:

    • put(K key, V value): Asocia el valor especificado con la clave especificada.
    • get(Object key): Devuelve el valor asociado con la clave especificada.
    • remove(Object key): Elimina la asociación para la clave especificada.
    • keySet(): Devuelve una vista de conjunto de las claves.
    • values(): Devuelve una vista de colección de los valores.
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);
map.put("B", 2);
System.out.println(map.get("A")); // Salida: 1
map.remove("A");
System.out.println(map.size()); // Salida: 1

Comparación de List, Set y Map

| Característica | List | Set | Map | |------------------|--------------------|--------------------|--------------------| | Permite Duplicados| Sí | No | Claves: No, Valores: Sí | | Ordenado | Sí | No (excepto LinkedHashSet) | No (excepto LinkedHashMap) | | Acceso por Índice| Sí | No | No | | Par Clave-Valor | No | No | Sí |

Conclusión

El Marco de Colecciones de Java es una herramienta poderosa y flexible para gestionar grupos de objetos. Entender las diferentes interfaces y sus implementaciones te permite elegir la estructura de datos adecuada para tus necesidades. Practica usando estas colecciones para volverte competente en el manejo de datos en Java.
