Introducción a la API de Streams

La API de Streams en Java proporciona una forma moderna y eficiente de procesar secuencias de elementos. Permite realizar operaciones de estilo funcional en colecciones de objetos, haciendo que el procesamiento de datos sea más legible y conciso.

Conceptos Clave

  • Stream: Una secuencia de elementos que admite operaciones agregadas secuenciales y paralelas.
  • Operaciones Intermedias: Operaciones que transforman un stream en otro stream (por ejemplo, filter, map).
  • Operaciones Terminales: Operaciones que producen un resultado o un efecto secundario (por ejemplo, collect, forEach).

Creación de Streams

Los streams se pueden crear a partir de varias fuentes de datos, como colecciones, matrices o canales de E/S.

Desde Colecciones

List<String> list = Arrays.asList("a", "b", "c");
Stream<String> stream = list.stream();

Desde Matrices

String[] array = {"a", "b", "c"};
Stream<String> stream = Arrays.stream(array);

Desde Valores

Stream<String> stream = Stream.of("a", "b", "c");

Operaciones Intermedias

Las operaciones intermedias devuelven un nuevo stream y son perezosas, lo que significa que no se ejecutan hasta que se invoca una operación terminal.

Operaciones Intermedias Comunes

  • filter: Filtra elementos basados en un predicado.
  • map: Transforma elementos usando una función.
  • sorted: Ordena elementos basados en un comparador.

Ejemplo: Filtrado y Mapeo

List<String> list = Arrays.asList("apple", "banana", "cherry");
List<String> result = list.stream()
    .filter(s -> s.startsWith("a"))
    .map(String::toUpperCase)
    .collect(Collectors.toList());

System.out.println(result); // Salida: [APPLE]

Operaciones Terminales

Las operaciones terminales producen un resultado o un efecto secundario y marcan el final del procesamiento del stream.

Operaciones Terminales Comunes

  • collect: Convierte el stream en una colección u otra estructura de datos.
  • forEach: Realiza una acción para cada elemento.
  • reduce: Combina elementos para producir un único resultado.

Ejemplo: Colección y Reducción

List<String> list = Arrays.asList("apple", "banana", "cherry");
String result = list.stream()
    .map(String::toUpperCase)
    .collect(Collectors.joining(", "));

System.out.println(result); // Salida: APPLE, BANANA, CHERRY

Streams Paralelos

Los streams paralelos permiten el procesamiento paralelo de datos, aprovechando los procesadores multinúcleo para un mejor rendimiento.

Creación de Streams Paralelos

List<String> list = Arrays.asList("apple", "banana", "cherry");
Stream<String> parallelStream = list.parallelStream();

Ejemplo: Procesamiento Paralelo

List<String> list = Arrays.asList("apple", "banana", "cherry");
list.parallelStream()
    .forEach(s -> System.out.println(s + " " + Thread.currentThread().getName()));

Mejores Prácticas

  • Usa streams paralelos con prudencia, ya que pueden introducir complejidad y sobrecarga.
  • Prefiere referencias a métodos (por ejemplo, String::toUpperCase) para un código más limpio y legible.
  • Ten en cuenta los efectos secundarios en las operaciones terminales, especialmente cuando uses streams paralelos.

Conclusión

La API de Streams en Java proporciona una forma poderosa y expresiva de procesar colecciones de datos. Al comprender y utilizar streams, puedes escribir código más conciso, legible y eficiente. Recuerda equilibrar el uso de streams paralelos con la complejidad que introducen, y siempre apunta a un código claro y mantenible.
