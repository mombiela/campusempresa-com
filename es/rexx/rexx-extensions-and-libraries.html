Introducción a las Extensiones y Bibliotecas de REXX

REXX (Restructured Extended Executor) es un lenguaje de scripting versátil que puede ser extendido con varias bibliotecas y extensiones para mejorar su funcionalidad. Esta sección cubrirá las extensiones y bibliotecas más comúnmente usadas, cómo integrarlas en tus programas REXX y ejemplos prácticos para ilustrar su uso.

Conceptos Clave

  • Extensiones: Funcionalidades adicionales que pueden ser añadidas al lenguaje REXX.
  • Bibliotecas: Módulos de código preescritos que pueden ser incluidos en programas REXX para realizar tareas específicas.
  • Integración: El proceso de incorporar extensiones y bibliotecas en tus scripts REXX.

Extensiones Comunes de REXX

1. REXXUTIL

REXXUTIL es una biblioteca que proporciona una variedad de funciones utilitarias para programas REXX, especialmente en mainframes IBM y sistemas OS/2.

Funciones Clave

  • SysFileTree: Lista recursivamente archivos en un directorio.
  • SysSleep: Pausa la ejecución por un número especificado de segundos.
  • SysGetEnv: Recupera el valor de una variable de entorno.

Ejemplo

/* Ejemplo usando REXXUTIL */
call RxFuncAdd 'SysLoadFuncs', 'REXXUTIL', 'SysLoadFuncs'
call SysLoadFuncs

/* Lista todos los archivos en el directorio actual */
call SysFileTree '*', 'fileList.', 'FO'

do i = 1 to fileList.0
    say fileList.i
end

Explicación:

  • RxFuncAdd se usa para registrar las funciones de REXXUTIL.
  • SysFileTree lista todos los archivos en el directorio actual y los almacena en el array fileList..

2. REXXLIB

REXXLIB es otra biblioteca popular que proporciona funciones adicionales para manejo de archivos, manipulación de cadenas y operaciones del sistema.

Funciones Clave

  • RxFileCopy: Copia un archivo de una ubicación a otra.
  • RxFileDelete: Elimina un archivo especificado.
  • RxStringReverse: Invierte una cadena dada.

Ejemplo

/* Ejemplo usando REXXLIB */
call RxFuncAdd 'RxFileCopy', 'REXXLIB', 'RxFileCopy'
call RxFuncAdd 'RxFileDelete', 'REXXLIB', 'RxFileDelete'

/* Copia un archivo */
call RxFileCopy 'source.txt', 'destination.txt'

/* Elimina el archivo original */
call RxFileDelete 'source.txt'

Explicación:

  • RxFuncAdd registra las funciones de REXXLIB.
  • RxFileCopy copia source.txt a destination.txt.
  • RxFileDelete elimina source.txt.

Integración de Extensiones y Bibliotecas

Para usar extensiones y bibliotecas en tus programas REXX, típicamente necesitas:

  1. Cargar la biblioteca usando RxFuncAdd.
  2. Llamar a la función de inicialización de la biblioteca si es necesario.
  3. Usar las funciones proporcionadas según sea necesario en tu script.

Ejemplo: Combinando REXXUTIL y REXXLIB

/* Cargar REXXUTIL y REXXLIB */
call RxFuncAdd 'SysLoadFuncs', 'REXXUTIL', 'SysLoadFuncs'
call SysLoadFuncs
call RxFuncAdd 'RxFileCopy', 'REXXLIB', 'RxFileCopy'
call RxFuncAdd 'RxFileDelete', 'REXXLIB', 'RxFileDelete'

/* Lista todos los archivos en el directorio actual */
call SysFileTree '*', 'fileList.', 'FO'

do i = 1 to fileList.0
    say 'Copiando archivo:' fileList.i
    call RxFileCopy fileList.i, fileList.i || '.bak'
end

say 'Todos los archivos han sido respaldados.'

Explicación:

  • Tanto REXXUTIL como REXXLIB son cargados y sus funciones registradas.
  • SysFileTree lista todos los archivos en el directorio actual.
  • Cada archivo es copiado a un nuevo archivo con una extensión .bak usando RxFileCopy.

Conclusión

Las extensiones y bibliotecas de REXX mejoran significativamente las capacidades del lenguaje REXX, permitiendo scripts más complejos y poderosos. Al entender cómo integrar y usar estas extensiones, puedes aprovechar sus funcionalidades para agilizar tus tareas de programación. Practica usando estas bibliotecas en tus proyectos para volverte competente en extender las capacidades de REXX.
