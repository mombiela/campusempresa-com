Introducción

Las subrutinas y funciones son conceptos fundamentales en la programación REXX que te permiten modularizar tu código, haciéndolo más legible, mantenible y reutilizable. Esta sección te guiará a través de los conceptos básicos de subrutinas y funciones, e introducirá gradualmente conceptos más avanzados.

Conceptos Clave

  • Subrutinas: Bloques de código que realizan una tarea específica y pueden ser llamados desde diferentes partes de un programa.
  • Funciones: Similares a las subrutinas pero diseñadas para devolver un valor.
  • Instrucción CALL: Utilizada para invocar una subrutina.
  • Instrucción RETURN: Utilizada para devolver el control al código que llama, opcionalmente con un valor.

Subrutinas

Definiendo una Subrutina

Una subrutina en REXX se define usando una etiqueta seguida de dos puntos. La subrutina puede ser llamada usando la instrucción CALL.

/* Programa Principal */
say "Iniciando el programa"
call miSubrutina
say "De vuelta en el programa principal"

/* Definición de la Subrutina */
miSubrutina:
    say "Dentro de la subrutina"
    return

Llamando a una Subrutina

Puedes llamar a una subrutina desde cualquier parte de tu programa usando la instrucción CALL.

call miSubrutina

Pasando Argumentos a Subrutinas

Los argumentos pueden ser pasados a las subrutinas y accedidos usando la instrucción ARG.

/* Programa Principal */
call miSubrutina "¡Hola, Mundo!"

/* Definición de la Subrutina */
miSubrutina:
    parse arg mensaje
    say mensaje
    return

Funciones

Definiendo una Función

Las funciones son similares a las subrutinas pero están diseñadas para devolver un valor. Se definen usando la misma sintaxis de etiqueta y dos puntos.

/* Programa Principal */
resultado = miFuncion(5, 3)
say "El resultado es" resultado

/* Definición de la Función */
miFuncion:
    parse arg a, b
    return a + b

Llamando a una Función

Las funciones se llaman usando su nombre seguido de paréntesis que contienen los argumentos.

resultado = miFuncion(5, 3)

Devolviendo Valores de Funciones

La instrucción RETURN se usa para devolver un valor de una función.

return a + b

Conceptos Avanzados

Variables Locales

Las variables en subrutinas y funciones son globales por defecto. Para crear variables locales, usa la instrucción PROCEDURE.

/* Programa Principal */
x = 10
call miSubrutina
say "Valor de x en el programa principal:" x

/* Definición de la Subrutina */
miSubrutina:
    procedure
    x = 5
    say "Valor de x en la subrutina:" x
    return

Funciones Recursivas

REXX soporta funciones recursivas, donde una función se llama a sí misma.

/* Programa Principal */
resultado = factorial(5)
say "El factorial de 5 es" resultado

/* Definición de la Función Recursiva */
factorial:
    parse arg n
    if n = 1 then return 1
    return n * factorial(n - 1)

Resumen

Las subrutinas y funciones son herramientas esenciales en REXX para crear código modular y reutilizable. Las subrutinas realizan tareas sin devolver valores, mientras que las funciones están diseñadas para devolver valores. Entender cómo definir, llamar y gestionar estos constructos, incluyendo el manejo de argumentos y variables locales, es crucial para una programación efectiva en REXX. Al dominar estos conceptos, podrás escribir código más organizado y eficiente.
