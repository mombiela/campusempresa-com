Variables y Tipos de Datos en REXX

Introducción

En esta sección, exploraremos los conceptos fundamentales de variables y tipos de datos en REXX. Comprender estos conceptos es crucial para escribir programas REXX efectivos y eficientes. Comenzaremos con lo básico y gradualmente avanzaremos hacia temas más avanzados.

Variables en REXX

¿Qué es una Variable?

  • Una variable en REXX es un nombre simbólico asociado con un valor.
  • Las variables se utilizan para almacenar datos que pueden ser manipulados y recuperados a lo largo del programa.

Declaración de Variables

  • En REXX, no es necesario declarar explícitamente las variables antes de usarlas.
  • Una variable se crea la primera vez que se le asigna un valor.
/* Ejemplo de asignación de variable */
name = "John Doe"
age = 30

Reglas de Nomenclatura de Variables

  • Los nombres de las variables deben comenzar con una letra (A-Z, a-z) o un carácter especial (como . o _).
  • Los caracteres subsiguientes pueden ser letras, dígitos (0-9) o caracteres especiales.
  • Los nombres de las variables no distinguen entre mayúsculas y minúsculas.
/* Nombres de variables válidos */
firstName = "Alice"
_lastName = "Smith"
age2 = 25

/* Nombres de variables inválidos */
// 2ndName = "Bob"  // Comienza con un dígito
// @age = 30        // Comienza con un carácter inválido

Asignación de Valores a Variables

  • Los valores se asignan a las variables usando el operador =.
  • REXX determina automáticamente el tipo de valor asignado.
/* Asignación de diferentes tipos de valores */
name = "Jane Doe"
age = 28
height = 5.7
isStudent = "Yes"

Tipos de Datos en REXX

Resumen de Tipos de Datos

  • REXX es un lenguaje de tipado dinámico, lo que significa que el tipo de una variable se determina en tiempo de ejecución.
  • Los principales tipos de datos en REXX son:
    • Cadenas
    • Números (enteros y de punto flotante)
    • Valores booleanos (representados como cadenas "0" o "1")

Cadenas

  • Las cadenas son secuencias de caracteres encerradas entre comillas dobles (").
/* Ejemplo de variables de cadena */
greeting = "Hello, World!"
name = "Alice"

Números

  • Los números en REXX pueden ser enteros o de punto flotante.
  • REXX maneja automáticamente las operaciones numéricas.
/* Ejemplo de variables numéricas */
integerNumber = 42
floatingNumber = 3.14

/* Realizando operaciones aritméticas */
sum = integerNumber + floatingNumber

Valores Booleanos

  • REXX no tiene un tipo booleano distinto. En su lugar, utiliza cadenas "0" (falso) y "1" (verdadero).
/* Ejemplo de valores booleanos */
isTrue = "1"
isFalse = "0"

Conversión de Tipos

  • REXX convierte automáticamente entre cadenas y números según sea necesario.
  • La conversión explícita se puede realizar utilizando funciones integradas.
/* Conversión automática de tipos */
numString = "123"
number = numString + 1  /* Convierte "123" a 123 y suma 1 */

/* Conversión explícita de tipos */
stringNumber = "456"
convertedNumber = VALUE(stringNumber)

Temas Avanzados

Variables Compuestas

  • Las variables compuestas (también conocidas como arreglos o arreglos asociativos) permiten estructuras de datos más complejas.
  • Están indexadas por cadenas o números.
/* Ejemplo de variables compuestas */
person.1 = "Alice"
person.2 = "Bob"
person.3 = "Charlie"

say person.1  /* Salida: Alice */

Alcance de Variables

  • Las variables en REXX pueden ser locales o globales.
  • Las variables locales se definen dentro de un procedimiento y no son accesibles fuera de él.
  • Las variables globales son accesibles en todo el programa.
/* Ejemplo de alcance de variables */
globalVar = "I am global"

procedure
  localVar = "I am local"
  say localVar
return

say globalVar  /* Salida: I am global */
// say localVar  /* Error: localVar no está definida fuera del procedimiento */

Conclusión

En esta sección, cubrimos los conceptos básicos de variables y tipos de datos en REXX. Aprendimos cómo declarar y asignar valores a variables, los diferentes tipos de datos disponibles y cómo realizar conversiones de tipos. También tocamos temas más avanzados como variables compuestas y alcance de variables. Comprender estos conceptos es esencial para escribir programas REXX efectivos.
