Introducción a los Árboles B

Los Árboles B son una estructura de datos de árbol autoequilibrado que mantiene datos ordenados y permite búsquedas, inserciones, eliminaciones y accesos secuenciales en tiempo logarítmico. Son especialmente útiles en sistemas de bases de datos y sistemas de archivos.

Características Clave de los Árboles B

  • Autoequilibrado: Los Árboles B mantienen su equilibrio automáticamente durante las operaciones de inserción y eliminación.
  • Nodos con múltiples hijos: A diferencia de los árboles binarios, los nodos en un Árbol B pueden tener más de dos hijos.
  • Altura baja: La altura de un Árbol B es baja en comparación con otros árboles, lo que permite un acceso más rápido a los datos.
  • Uso eficiente del almacenamiento: Los Árboles B están diseñados para minimizar el número de accesos a disco, lo que los hace ideales para sistemas de almacenamiento en disco.

Estructura de un Árbol B

Un Árbol B de orden m tiene las siguientes propiedades:

  • Cada nodo puede tener como máximo m hijos.
  • Cada nodo interno (excepto la raíz) tiene al menos ⌈m/2⌉ hijos.
  • La raíz tiene al menos dos hijos si no es una hoja.
  • Todos los nodos hoja están al mismo nivel.
  • Un nodo con k hijos contiene k-1 claves.

Ejemplo de un Árbol B de Orden 3

Consideremos un Árbol B de orden 3 (también conocido como un 2-3 árbol):

        [10 | 20]
       /    |    \
  [5 | 7]  [15]  [25 | 30]

En este ejemplo:

  • La raíz tiene 2 claves y 3 hijos.
  • Cada nodo hoja tiene entre 1 y 2 claves.

Operaciones en Árboles B

Inserción

La inserción en un Árbol B implica encontrar la posición correcta para la nueva clave y, si es necesario, dividir los nodos que exceden el número máximo de claves permitidas.

Ejemplo de Inserción

Supongamos que queremos insertar la clave 12 en el siguiente Árbol B de orden 3:

        [10 | 20]
       /    |    \
  [5 | 7]  [15]  [25 | 30]
  1. Encuentra la hoja adecuada para la clave 12, que es el nodo [15].
  2. Inserta 12 en el nodo [15], resultando en [12 | 15].
  3. Si el nodo excede el número máximo de claves, divídelo y ajusta el árbol.

Eliminación

La eliminación en un Árbol B puede ser más compleja que la inserción, ya que puede requerir la redistribución de claves o la fusión de nodos para mantener las propiedades del árbol.

Ejemplo de Eliminación

Supongamos que queremos eliminar la clave 20 del siguiente Árbol B de orden 3:

        [10 | 20]
       /    |    \
  [5 | 7]  [15]  [25 | 30]
  1. Encuentra la clave 20 y elimínala.
  2. Si la eliminación deja un nodo con menos de ⌈m/2⌉ claves, redistribuye o fusiona nodos para mantener el equilibrio.

Ejemplo de Código en Python

A continuación, se presenta un ejemplo básico de implementación de un Árbol B en Python:

class BTreeNode:
    def __init__(self, t, leaf=False):
        self.t = t  # Grado mínimo (orden)
        self.leaf = leaf  # True si el nodo es hoja
        self.keys = []  # Lista de claves
        self.children = []  # Lista de hijos

class BTree:
    def __init__(self, t):
        self.root = BTreeNode(t, True)
        self.t = t

    def traverse(self, node):
        for i in range(len(node.keys)):
            if not node.leaf:
                self.traverse(node.children[i])
            print(node.keys[i], end=" ")
        if not node.leaf:
            self.traverse(node.children[len(node.keys)])

    def insert(self, k):
        root = self.root
        if len(root.keys) == 2 * self.t - 1:
            temp = BTreeNode(self.t)
            self.root = temp
            temp.children.insert(0, root)
            self.split_child(temp, 0)
            self.insert_non_full(temp, k)
        else:
            self.insert_non_full(root, k)

    def split_child(self, parent, i):
        t = self.t
        y = parent.children[i]
        z = BTreeNode(t, y.leaf)
        parent.children.insert(i + 1, z)
        parent.keys.insert(i, y.keys[t - 1])
        z.keys = y.keys[t:(2 * t - 1)]
        y.keys = y.keys[0:(t - 1)]
        if not y.leaf:
            z.children = y.children[t:(2 * t)]
            y.children = y.children[0:t]

    def insert_non_full(self, node, k):
        i = len(node.keys) - 1
        if node.leaf:
            node.keys.append(0)
            while i >= 0 and k < node.keys[i]:
                node.keys[i + 1] = node.keys[i]
                i -= 1
            node.keys[i + 1] = k
        else:
            while i >= 0 and k < node.keys[i]:
                i -= 1
            i += 1
            if len(node.children[i].keys) == 2 * self.t - 1:
                self.split_child(node, i)
                if k > node.keys[i]:
                    i += 1
            self.insert_non_full(node.children[i], k)

# Ejemplo de uso
b_tree = BTree(3)
keys = [10, 20, 5, 6, 12, 30, 7, 17]
for key in keys:
    b_tree.insert(key)

b_tree.traverse(b_tree.root)

Conclusión

Los Árboles B son una estructura de datos fundamental en la informática, especialmente en el ámbito de las bases de datos y sistemas de archivos. Su capacidad para mantener el equilibrio y minimizar los accesos a disco los hace extremadamente eficientes para operaciones de búsqueda, inserción y eliminación. Comprender cómo funcionan y cómo implementarlos es esencial para cualquier profesional de TI que trabaje con grandes volúmenes de datos.
