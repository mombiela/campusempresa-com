Las variables especiales en Bash son variables predefinidas que proporcionan información útil sobre el entorno, el script o la línea de comandos. Son esenciales para escribir scripts robustos y flexibles.

Variables especiales clave

`$0`

  • Representa el nombre del script que se está ejecutando.
  • Útil para mostrar el nombre del script en mensajes o registros.
#!/bin/bash
echo "El nombre del script es: $0"

`$1, $2, ..., $N`

  • Representan los parámetros posicionales pasados al script.
  • $1 es el primer argumento, $2 es el segundo, y así sucesivamente.
#!/bin/bash
echo "Primer argumento: $1"
echo "Segundo argumento: $2"

`$#`

  • Representa el número de parámetros posicionales pasados al script.
  • Útil para verificar si se ha proporcionado el número correcto de argumentos.
#!/bin/bash
if [ $# -ne 2 ]; then
  echo "Uso: $0 arg1 arg2"
  exit 1
fi

`$@` y `$*`

  • Ambos representan todos los parámetros posicionales pasados al script.
  • $@ trata cada parámetro como una palabra separada.
  • $* trata todos los parámetros como una sola palabra.
#!/bin/bash
echo "Todos los argumentos usando \$@: $@"
echo "Todos los argumentos usando \$*: $*"

`$?`

  • Representa el estado de salida del último comando ejecutado.
  • Un valor de 0 indica éxito, mientras que cualquier otro valor indica un error.
#!/bin/bash
ls /nonexistent_directory
echo "Estado de salida del último comando: $?"

`$$`

  • Representa el ID de proceso (PID) del propio script.
  • Útil para crear archivos temporales únicos.
#!/bin/bash
tempfile="/tmp/tempfile_$$"
echo "Archivo temporal: $tempfile"

`$!`

  • Representa el ID de proceso del último comando en segundo plano.
  • Útil para rastrear procesos en segundo plano.
#!/bin/bash
sleep 60 &
echo "PID del proceso en segundo plano: $!"

`$-`

  • Representa las opciones actuales establecidas para el shell.
  • Útil para depurar y comprender el comportamiento del shell.
#!/bin/bash
echo "Opciones actuales del shell: $-"

`$_`

  • Representa el último argumento del comando anterior.
  • Útil para reutilizar el último argumento.
#!/bin/bash
echo "Primer comando"
echo "Segundo comando con el último argumento: $_"

Comparación de `$@` y `$*`

| Variable | Descripción | Ejemplo de salida | |----------|-------------|-------------------| | $@ | Trata cada parámetro como una palabra separada | "arg1" "arg2" "arg3" | | $* | Trata todos los parámetros como una sola palabra | "arg1 arg2 arg3" |

Conclusión

Las variables especiales en Bash proporcionan formas poderosas de interactuar con el entorno del script y los argumentos de la línea de comandos. Comprender y usar estas variables de manera efectiva puede mejorar significativamente la flexibilidad y robustez de tus scripts en Bash. Practica el uso de estas variables en diferentes escenarios para volverte competente en la escritura de scripts avanzados en Bash.
