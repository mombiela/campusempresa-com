Las pruebas de archivos son un aspecto esencial de la creación de scripts en Bash. Te permiten verificar las propiedades de archivos y directorios, como su existencia, tipo y permisos. Este tema cubrirá los conceptos básicos de las pruebas de archivos, varios operadores de prueba de archivos y ejemplos prácticos para ayudarte a entender cómo usar estas pruebas de manera efectiva en tus scripts.

Conceptos Clave

  • Operadores de Prueba de Archivos: Operadores especiales utilizados para probar diferentes propiedades de archivos y directorios.
  • Sentencias Condicionales: Utilizadas para realizar acciones basadas en los resultados de las pruebas de archivos.

Operadores de Prueba de Archivos

Bash proporciona varios operadores de prueba de archivos que se pueden usar dentro de sentencias condicionales para verificar diversas propiedades de archivos y directorios. Aquí están algunos de los operadores de prueba de archivos más comúnmente utilizados:

| Operador | Descripción | |----------|---------------------------------------------| | -e | Verifica si el archivo existe | | -f | Verifica si el archivo es un archivo regular| | -d | Verifica si el archivo es un directorio | | -r | Verifica si el archivo es legible | | -w | Verifica si el archivo es escribible | | -x | Verifica si el archivo es ejecutable | | -s | Verifica si el archivo no está vacío | | -L | Verifica si el archivo es un enlace simbólico| | -p | Verifica si el archivo es una tubería con nombre| | -S | Verifica si el archivo es un socket |

Uso de los Operadores de Prueba de Archivos

Los operadores de prueba de archivos se utilizan típicamente dentro de sentencias if para realizar acciones basadas en los resultados de las pruebas. Aquí hay algunos ejemplos para ilustrar su uso:

Ejemplo 1: Verificar si un Archivo Existe

#!/bin/bash

FILE="example.txt"

if [ -e "$FILE" ]; then
    echo "El archivo '$FILE' existe."
else
    echo "El archivo '$FILE' no existe."
fi

Ejemplo 2: Verificar si un Archivo es un Archivo Regular

#!/bin/bash

FILE="example.txt"

if [ -f "$FILE" ]; then
    echo "'$FILE' es un archivo regular."
else
    echo "'$FILE' no es un archivo regular."
fi

Ejemplo 3: Verificar si un Directorio Existe

#!/bin/bash

DIR="example_dir"

if [ -d "$DIR" ]; then
    echo "El directorio '$DIR' existe."
else
    echo "El directorio '$DIR' no existe."
fi

Ejemplo 4: Verificar los Permisos de un Archivo

#!/bin/bash

FILE="example.txt"

if [ -r "$FILE" ]; then
    echo "El archivo '$FILE' es legible."
else
    echo "El archivo '$FILE' no es legible."
fi

if [ -w "$FILE" ]; then
    echo "El archivo '$FILE' es escribible."
else
    echo "El archivo '$FILE' no es escribible."
fi

if [ -x "$FILE" ]; then
    echo "El archivo '$FILE' es ejecutable."
else
    echo "El archivo '$FILE' no es ejecutable."
fi

Ejemplo 5: Verificar si un Archivo No Está Vacío

#!/bin/bash

FILE="example.txt"

if [ -s "$FILE" ]; then
    echo "El archivo '$FILE' no está vacío."
else
    echo "El archivo '$FILE' está vacío."
fi

Ejercicios Prácticos

Ejercicio 1: Verificación de Existencia de Archivo

Escribe un script que verifique si un archivo llamado data.txt existe en el directorio actual. Si existe, imprime "El archivo existe." De lo contrario, imprime "El archivo no existe."

Ejercicio 2: Verificación de Directorio

Escribe un script que verifique si un directorio llamado backup existe. Si existe, imprime "El directorio existe." De lo contrario, crea el directorio e imprime "El directorio ha sido creado."

Ejercicio 3: Verificación de Permisos de Archivo

Escribe un script que verifique si un archivo llamado script.sh es legible, escribible y ejecutable. Imprime mensajes apropiados para cada verificación.

Conclusión

Las pruebas de archivos en Bash son una característica poderosa que te permite escribir scripts más robustos y flexibles. Al usar operadores de prueba de archivos, puedes verificar diversas propiedades de archivos y directorios y realizar acciones basadas en estas verificaciones. Practica usando estos operadores en tus scripts para volverte más competente en la creación de scripts en Bash.
