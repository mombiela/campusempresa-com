Gestionar procesos es una habilidad crucial cuando se trabaja con Bash. Este tema cubrirá los conceptos básicos de la gestión de procesos, incluyendo cómo iniciar, detener y monitorear procesos. Al final de esta sección, tendrás una comprensión sólida de cómo gestionar procesos de manera efectiva en un entorno similar a Unix.

Iniciando Procesos

En Bash, iniciar un proceso es tan simple como escribir un comando y presionar Enter. Sin embargo, hay diferentes maneras de iniciar procesos dependiendo de tus necesidades.

Procesos en Primer Plano

  • Definición: Un proceso en primer plano es aquel que toma el control del terminal hasta que se completa.

  • Ejemplo:

    ls -l

    Este comando lista el contenido del directorio actual en formato largo y se ejecuta en primer plano.

Procesos en Segundo Plano

  • Definición: Un proceso en segundo plano se ejecuta independientemente del terminal, permitiéndote seguir usando el terminal para otros comandos.

  • Ejemplo:

    sleep 30 &

    El & al final del comando envía el comando sleep al segundo plano, permitiéndote usar el terminal mientras el comando se ejecuta.

Monitoreando Procesos

Monitorear procesos es esencial para entender el rendimiento del sistema y gestionar recursos.

Comando `ps`

  • Definición: El comando ps muestra información sobre los procesos activos.

  • Ejemplo:

    ps aux

    Este comando muestra una lista detallada de todos los procesos en ejecución.

    | Columna | Descripción | |---------|-------------| | USER | El usuario que posee el proceso | | PID | ID del proceso | | %CPU | Porcentaje de uso de CPU | | %MEM | Porcentaje de uso de memoria | | VSZ | Tamaño de memoria virtual | | RSS | Tamaño del conjunto residente (memoria física) | | TTY | Terminal asociado con el proceso | | STAT | Estado del proceso | | START | Hora de inicio del proceso | | TIME | Tiempo acumulado de CPU | | COMMAND | Comando que inició el proceso |

Comando `top`

  • Definición: El comando top proporciona una vista dinámica y en tiempo real de los procesos en ejecución.

  • Ejemplo:

    top

    Este comando abre una interfaz interactiva que muestra los procesos que más recursos consumen.

Controlando Procesos

A veces, necesitas detener o pausar procesos. Bash proporciona varios comandos para este propósito.

Comando `kill`

  • Definición: El comando kill envía una señal a un proceso, usualmente para terminarlo.

  • Ejemplo:

    kill 1234

    Este comando envía la señal TERM por defecto al proceso con PID 1234, pidiéndole que termine de manera ordenada.

Comando `killall`

  • Definición: El comando killall envía una señal a todos los procesos que ejecutan un comando especificado.

  • Ejemplo:

    killall firefox

    Este comando envía la señal TERM por defecto a todas las instancias del proceso firefox.

Comandos `bg` y `fg`

  • Definición: El comando bg reanuda un trabajo suspendido en segundo plano, mientras que el comando fg trae un trabajo en segundo plano al primer plano.

  • Ejemplo:

    sleep 100
^Z
bg
fg

    En este ejemplo, ^Z (Ctrl+Z) suspende el comando sleep, bg lo reanuda en segundo plano, y fg lo trae de vuelta al primer plano.

Resumen

En esta sección, cubrimos los conceptos básicos de la gestión de procesos en Bash. Aprendiste cómo iniciar procesos en primer plano y en segundo plano, monitorearlos usando comandos como ps y top, y controlarlos usando comandos como kill, killall, bg y fg. Dominar estos comandos te ayudará a gestionar los recursos del sistema de manera efectiva y asegurar el funcionamiento fluido de tu entorno similar a Unix.
