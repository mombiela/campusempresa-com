Introducción

En esta sección, exploraremos los conceptos de interrupciones y llamadas al sistema en lenguaje ensamblador. Estos son cruciales para interactuar con el hardware y el sistema operativo, permitiendo que los programas realicen tareas como operaciones de entrada/salida, control de procesos y más.

Interrupciones

Las interrupciones son señales que informan al procesador que un evento necesita atención inmediata. Pueden ser activadas por hardware o software.

Conceptos Clave

  • Interrupciones de Hardware: Generadas por dispositivos de hardware para señalar que necesitan procesamiento.
  • Interrupciones de Software: Generadas por instrucciones de software para solicitar servicios del sistema operativo.
  • Tabla de Vectores de Interrupción (IVT): Una tabla que contiene las direcciones de las rutinas de servicio de interrupción (ISRs).

Ejemplo: Manejo de una Interrupción del Teclado

section .data
    msg db '¡Tecla presionada!', 0

section .bss

section .text
    global _start

_start:
    ; Habilitar interrupciones del teclado
    in al, 0x21
    and al, 0xFD
    out 0x21, al

    ; Bucle infinito para mantener el programa en ejecución
    jmp $

keyboard_interrupt:
    pusha
    ; Manejar la interrupción del teclado
    mov edx, msg
    call print_string
    popa
    iret

print_string:
    ; Imprimir la cadena apuntada por EDX
    mov eax, 4
    mov ebx, 1
    mov ecx, edx
    mov edx, 13
    int 0x80
    ret

En este ejemplo, habilitamos las interrupciones del teclado y definimos una simple rutina de servicio de interrupción (ISR) para imprimir un mensaje cuando se presiona una tecla.

Llamadas al Sistema

Las llamadas al sistema proporcionan una interfaz controlada para que los programas soliciten servicios del núcleo del sistema operativo.

Conceptos Clave

  • Interfaz de Llamada al Sistema: El mecanismo a través del cual las aplicaciones en espacio de usuario solicitan servicios del núcleo.
  • Registros: Se utilizan registros específicos para pasar argumentos a las llamadas al sistema (por ejemplo, eax para el número de llamada al sistema).

Ejemplo: Usando una Llamada al Sistema para Escribir en la Salida Estándar

section .data
    msg db '¡Hola, Mundo!', 0xA

section .text
    global _start

_start:
    ; Escribir el mensaje en la salida estándar
    mov eax, 4          ; número de llamada al sistema sys_write
    mov ebx, 1          ; descriptor de archivo 1 (stdout)
    mov ecx, msg        ; puntero al mensaje
    mov edx, 13         ; longitud del mensaje
    int 0x80            ; interrupción para invocar la llamada al sistema

    ; Salir del programa
    mov eax, 1          ; número de llamada al sistema sys_exit
    xor ebx, ebx        ; código de salida 0
    int 0x80            ; interrupción para invocar la llamada al sistema

En este ejemplo, usamos la llamada al sistema sys_write para imprimir "¡Hola, Mundo!" en la salida estándar y luego salir del programa usando la llamada al sistema sys_exit.

Comparación de Interrupciones y Llamadas al Sistema

| Característica | Interrupciones | Llamadas al Sistema | |------------------|-----------------------------------|----------------------------------| | Activadas por | Hardware o software | Software | | Propósito | Manejar eventos asíncronos | Solicitar servicios del SO | | Transferencia de Control | Tabla de vectores de interrupción (IVT) | Interfaz de llamada al sistema | | Ejemplo de Uso | Entrada de teclado, interrupciones de temporizador | Operaciones de archivo, control de procesos |

Conclusión

Entender las interrupciones y las llamadas al sistema es esencial para la programación de bajo nivel e interactuar con el hardware y el sistema operativo. Las interrupciones permiten que tu programa responda a eventos asíncronos, mientras que las llamadas al sistema proporcionan una forma de realizar operaciones privilegiadas de manera controlada. Dominar estos conceptos te permitirá escribir programas en ensamblador más eficientes y poderosos.
