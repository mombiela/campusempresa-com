Introducción al Ensamblador MIPS

MIPS (Microprocessor without Interlocked Pipeline Stages) es una arquitectura RISC (Reduced Instruction Set Computer) que se utiliza ampliamente en entornos académicos para enseñar conceptos de lenguaje ensamblador y arquitectura de computadoras.

Conceptos Clave

  • Arquitectura RISC: Instrucciones simplificadas que pueden ejecutarse en un ciclo de reloj.
  • Registros: Pequeñas y rápidas ubicaciones de almacenamiento dentro de la CPU.
  • Memoria: Organizada en bytes y palabras, accesible a través de direcciones.
  • Conjunto de Instrucciones: El conjunto de comandos que la CPU puede ejecutar.

Fundamentos del Ensamblador MIPS

Registros

MIPS tiene 32 registros de propósito general, cada uno de 32 bits de ancho. Algunos registros importantes incluyen:

  • $zero: Siempre contiene el valor 0.
  • $v0-$v1: Usados para valores de retorno de funciones.
  • $a0-$a3: Usados para argumentos de funciones.
  • $t0-$t9: Registros temporales.
  • $s0-$s7: Registros guardados.
  • $sp: Puntero de pila.
  • $ra: Dirección de retorno.

Instrucciones Básicas

  • Instrucciones Aritméticas: add, sub, mul, div
  • Instrucciones Lógicas: and, or, xor, nor
  • Instrucciones de Transferencia de Datos: lw (load word), sw (store word)
  • Instrucciones de Control de Flujo: beq (branch if equal), bne (branch if not equal), j (jump)

Ejemplo: Sumar Dos Números

.data
num1: .word 5
num2: .word 10
result: .word 0

.text
main:
    lw $t0, num1       # Cargar num1 en $t0
    lw $t1, num2       # Cargar num2 en $t1
    add $t2, $t0, $t1  # Sumar $t0 y $t1, almacenar resultado en $t2
    sw $t2, result     # Almacenar el resultado en memoria
    li $v0, 10         # Salir del programa
    syscall

Ensamblador MIPS Intermedio

Control de Flujo

  • Desvío Condicional: beq, bne
  • Salto Incondicional: j, jal (jump and link)

Ejemplo: Bucle

.data
array: .word 1, 2, 3, 4, 5
sum: .word 0

.text
main:
    li $t0, 0          # Inicializar índice a 0
    li $t1, 5          # Establecer cuenta del bucle a 5
    li $t2, 0          # Inicializar suma a 0

loop:
    beq $t0, $t1, end  # Si índice == 5, salir del bucle
    lw $t3, array($t0) # Cargar elemento del array en $t3
    add $t2, $t2, $t3  # Sumar elemento a la suma
    addi $t0, $t0, 4   # Moverse al siguiente elemento (palabra = 4 bytes)
    j loop             # Repetir bucle

end:
    sw $t2, sum        # Almacenar la suma
    li $v0, 10         # Salir del programa
    syscall

Ensamblador MIPS Avanzado

Procedimientos y Funciones

  • Convenciones de Llamada: Cómo se pasan los argumentos y se manejan los valores de retorno.
  • Gestión de la Pila: Uso de la pila para guardar y restaurar registros.

Ejemplo: Función Factorial

.data
result: .word 0

.text
main:
    li $a0, 5          # Cargar argumento (5) en $a0
    jal factorial      # Llamar a la función factorial
    sw $v0, result     # Almacenar el resultado
    li $v0, 10         # Salir del programa
    syscall

factorial:
    addi $sp, $sp, -8  # Asignar espacio en la pila
    sw $ra, 4($sp)     # Guardar dirección de retorno
    sw $a0, 0($sp)     # Guardar argumento

    li $v0, 1          # Caso base: factorial(0) = 1
    beq $a0, $zero, end_factorial

    addi $a0, $a0, -1  # Decrementar argumento
    jal factorial      # Llamada recursiva
    lw $a0, 0($sp)     # Restaurar argumento
    mul $v0, $a0, $v0  # Multiplicar resultado con argumento

end_factorial:
    lw $ra, 4($sp)     # Restaurar dirección de retorno
    addi $sp, $sp, 8   # Desasignar espacio en la pila
    jr $ra             # Retornar al llamador

Conclusión

El lenguaje ensamblador MIPS proporciona una forma clara y estructurada de entender la programación de bajo nivel y la arquitectura de computadoras. Al dominar MIPS, obtienes una visión de cómo los lenguajes de alto nivel se traducen en código máquina, cómo la CPU ejecuta instrucciones y cómo se gestionan la memoria y los registros. Este conocimiento es fundamental para optimizar el código, entender el rendimiento del sistema y desarrollar software eficiente.
