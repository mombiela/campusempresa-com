Introducción

La depuración y las pruebas son habilidades cruciales para cualquier programador de Objective-C. Esta sección cubrirá los conceptos básicos de la depuración y las pruebas, incluyendo herramientas, técnicas y mejores prácticas. Al final de esta sección, serás capaz de identificar y corregir errores en tu código y escribir pruebas para asegurar que tu código funcione como se espera.

Depuración

Conceptos Clave

  • Puntos de interrupción: Puntos en tu código donde la ejecución se detendrá, permitiéndote inspeccionar el estado de tu aplicación.
  • Saltar, Entrar, Salir: Comandos para controlar el flujo de ejecución mientras depuras.
  • Variables de vigilancia: Monitorea los valores de las variables mientras tu código se ejecuta.
  • Salida de consola: Usar NSLog para imprimir mensajes en la consola con fines de depuración.

Herramientas

  • Depurador de Xcode: La herramienta principal para depurar aplicaciones Objective-C.
  • LLDB: El depurador de línea de comandos utilizado por Xcode.

Configuración de Puntos de Interrupción

Los puntos de interrupción te permiten pausar la ejecución de tu código en líneas específicas para inspeccionar el estado de tu aplicación.

- (void)viewDidLoad {
    [super viewDidLoad];
    int a = 5;
    int b = 10;
    int suma = a + b; // Establecer un punto de interrupción aquí
    NSLog(@"Suma: %d", suma);
}

Uso de Comandos LLDB

Los comandos LLDB pueden ser utilizados en la consola de Xcode para inspeccionar y manipular el estado de tu aplicación.

(lldb) breakpoint set --file ViewController.m --line 12
(lldb) run
(lldb) print suma

Ejemplo: Depuración de una Aplicación Simple

Vamos a depurar una aplicación simple que calcula la suma de dos números.

  1. Establece un punto de interrupción en la línea donde se calcula la suma.
  2. Ejecuta la aplicación en Xcode.
  3. Cuando se alcance el punto de interrupción, usa la consola LLDB para inspeccionar los valores de a, b y suma.

Pruebas

Conceptos Clave

  • Pruebas Unitarias: Pruebas de unidades individuales de código para asegurar que funcionen como se espera.
  • Pruebas de Integración: Pruebas de cómo diferentes unidades de código funcionan juntas.
  • Desarrollo Guiado por Pruebas (TDD): Escribir pruebas antes de escribir el código real.

Herramientas

  • XCTest: El marco proporcionado por Apple para escribir y ejecutar pruebas en Xcode.

Escritura de Pruebas Unitarias

Las pruebas unitarias se escriben utilizando el marco XCTest. Aquí hay un ejemplo de una prueba unitaria simple:

#import <XCTest/XCTest.h>
#import "Calculator.h"

@interface CalculatorTests : XCTestCase
@end

@implementation CalculatorTests

- (void)testAddition {
    Calculator *calculator = [[Calculator alloc] init];
    int resultado = [calculator addNumber:5 toNumber:10];
    XCTAssertEqual(resultado, 15, @"Se esperaba que 5 + 10 fuera igual a 15");
}

@end

Ejecución de Pruebas en Xcode

  1. Abre el Navegador de Pruebas haciendo clic en el icono de diamante en la barra de herramientas de Xcode.
  2. Selecciona la prueba que deseas ejecutar.
  3. Haz clic en el botón de reproducción para ejecutar la prueba.

Ejemplo: Escritura y Ejecución de una Prueba Unitaria

Vamos a escribir una prueba unitaria para una clase de calculadora simple.

  1. Crea una nueva clase llamada Calculator con un método addNumber:toNumber:.
  2. Escribe una prueba unitaria para el método addNumber:toNumber:.
  3. Ejecuta la prueba en Xcode para asegurar que pase.

Conclusión

La depuración y las pruebas son habilidades esenciales para cualquier programador de Objective-C. Usando puntos de interrupción, comandos LLDB y pruebas unitarias, puedes identificar y corregir errores en tu código y asegurar que tu código funcione como se espera. Practica estas técnicas regularmente para volverte competente en la depuración y prueba de tus aplicaciones Objective-C.
