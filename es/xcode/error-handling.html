El manejo de errores es un aspecto crucial de la programación que asegura que tu aplicación pueda manejar situaciones inesperadas de manera elegante. En esta sección, cubriremos el manejo de errores en Xcode, desde conceptos básicos hasta técnicas avanzadas.

Introducción al Manejo de Errores

El manejo de errores permite que tu aplicación responda a condiciones inesperadas (errores) de manera controlada. En Swift, el manejo de errores se realiza utilizando el protocolo Error y bloques do-catch.

Conceptos Clave

  • Protocolo Error: Un tipo que representa un error.
  • Funciones que Lanzan Errores: Funciones que pueden lanzar un error.
  • Bloques Do-Catch: Usados para manejar errores lanzados por funciones.
  • Try, Try?, y Try!: Diferentes formas de llamar a funciones que lanzan errores.

Manejo Básico de Errores

Definiendo Errores

En Swift, defines errores creando una enumeración que se ajusta al protocolo Error.

enum FileError: Error {
    case fileNotFound
    case unreadable
    case encodingFailed
}

Lanzando Errores

Una función que puede lanzar un error debe estar marcada con la palabra clave throws.

func readFile(at path: String) throws -> String {
    // Simular un error de archivo no encontrado
    throw FileError.fileNotFound
}

Manejo de Errores con Do-Catch

Manejas errores utilizando un bloque do-catch.

do {
    let content = try readFile(at: "path/to/file")
    print(content)
} catch FileError.fileNotFound {
    print("Archivo no encontrado.")
} catch FileError.unreadable {
    print("El archivo no se puede leer.")
} catch {
    print("Ocurrió un error desconocido.")
}

Usando Try, Try?, y Try!

  • try: Usado cuando esperas que la función lance un error.
  • try?: Convierte el error en un valor opcional.
  • try!: Obliga a que la función tenga éxito o hace que el programa se bloquee.
let content = try? readFile(at: "path/to/file") // content es nil si se lanza un error
let forcedContent = try! readFile(at: "path/to/file") // se bloquea si se lanza un error

Manejo Avanzado de Errores

Propagando Errores

Las funciones que llaman a otras funciones que lanzan errores pueden propagar errores utilizando la palabra clave throws.

func processFile(at path: String) throws {
    let content = try readFile(at: path)
    // Procesar el contenido
}

Tipos de Errores Personalizados

Puedes crear tipos de errores personalizados para proporcionar más contexto sobre el error.

struct CustomError: Error {
    let message: String
    let code: Int
}

func performTask() throws {
    throw CustomError(message: "Algo salió mal", code: 404)
}

Manejo de Errores en Código Asíncrono

Manejar errores en código asíncrono requiere un enfoque diferente, a menudo utilizando manejadores de finalización o async/await.

func fetchData(completion: @escaping (Result<Data, Error>) -> Void) {
    // Simular una operación asíncrona
    DispatchQueue.global().async {
        // Simular un error
        completion(.failure(FileError.fileNotFound))
    }
}

fetchData { result in
    switch result {
    case .success(let data):
        print("Datos recibidos: \(data)")
    case .failure(let error):
        print("Error: \(error)")
    }
}

Conclusión

El manejo de errores es una habilidad esencial para cualquier desarrollador. Al comprender e implementar técnicas adecuadas de manejo de errores, puedes hacer que tus aplicaciones sean más robustas y fáciles de usar. En esta sección, cubrimos:

  • Conceptos y sintaxis básicos del manejo de errores.
  • Técnicas avanzadas como la propagación de errores y tipos de errores personalizados.
  • Manejo de errores en código asíncrono.

Con estas herramientas, puedes asegurarte de que tus aplicaciones manejen situaciones inesperadas de manera elegante, proporcionando una mejor experiencia para tus usuarios.
