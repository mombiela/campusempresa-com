Introducción al control de versiones

  • Definición: El control de versiones es un sistema que registra los cambios en un archivo o conjunto de archivos a lo largo del tiempo para que puedas recordar versiones específicas más tarde.
  • Importancia:
    • Realiza un seguimiento de cada modificación.
    • Facilita la colaboración.
    • Ayuda a gestionar versiones y lanzamientos del proyecto.

Empezando con Git

  • ¿Qué es Git?: Git es un sistema de control de versiones distribuido diseñado para manejar todo, desde proyectos pequeños hasta muy grandes, con velocidad y eficiencia.
  • Terminología básica de Git:
    • Repositorio (Repo): Un directorio que contiene el trabajo de tu proyecto, incluyendo todos los cambios realizados en los archivos.
    • Commit: Una instantánea de tu repositorio en un punto específico en el tiempo.
    • Branch: Una versión paralela de tu repositorio. Te permite trabajar en diferentes características o correcciones de errores de manera independiente.
    • Merge: Combinar cambios de diferentes ramas en una sola.

Configuración de Git en Xcode

  • Requisitos previos: Asegúrate de que Git esté instalado en tu sistema. Xcode generalmente viene con Git preinstalado.

  • Creando un nuevo proyecto con Git:

    1. Abre Xcode y crea un nuevo proyecto.
    2. En el asistente de creación de proyectos, asegúrate de que la casilla "Crear repositorio Git en mi Mac" esté seleccionada.

  • Clonando un repositorio existente:

    1. Abre Xcode.
    2. Ve a Control de Fuente > Clonar.
    3. Ingresa la URL del repositorio y elige la carpeta de destino.

Operaciones básicas de Git en Xcode

  • Realizando commits:

    1. Realiza cambios en los archivos de tu proyecto.
    2. Ve a Control de Fuente > Commit.
    3. Selecciona los archivos que deseas commitear, ingresa un mensaje de commit y haz clic en Commit.
    // Ejemplo de cambio en código Swift
func greet() {
    print("Hello, World!")
}

  • Creando y cambiando de ramas:

    1. Ve a Control de Fuente > Nueva Rama.
    2. Ingresa el nombre de la rama y haz clic en Crear.
    3. Para cambiar de rama, ve a Control de Fuente > Cambiar Rama y selecciona la rama deseada.

  • Fusionando ramas:

    1. Ve a Control de Fuente > Fusionar.
    2. Selecciona la rama que deseas fusionar con tu rama actual y haz clic en Fusionar.

Funciones avanzadas de Git en Xcode

  • Guardando cambios temporalmente (Stashing): Guarda temporalmente los cambios que no están listos para ser commiteados.

    1. Ve a Control de Fuente > Guardar Cambios.
    2. Para aplicar los cambios guardados, ve a Control de Fuente > Aplicar Guardado.

  • Resolviendo conflictos de fusión:

    1. Cuando ocurre un conflicto, Xcode resaltará los archivos en conflicto.
    2. Abre el archivo en conflicto y resuelve manualmente los conflictos.
    3. Después de resolver, marca el archivo como resuelto en Control de Fuente > Marcar como Resuelto.

  • Viendo el historial y la autoría (Blame):

    1. Para ver el historial de commits, ve a Control de Fuente > Historial.
    2. Para ver quién hizo cambios en una línea específica, usa Control de Fuente > Mostrar Autoría para Línea.

Resumen

  • Puntos clave:
    • La integración de Git en Xcode simplifica el control de versiones para proyectos de iOS/macOS.
    • Las operaciones básicas como commitear, crear ramas y fusionar se pueden realizar fácilmente dentro de Xcode.
    • Las funciones avanzadas como guardar cambios temporalmente y resolver conflictos también están soportadas, mejorando tu flujo de trabajo.

Al dominar Git en Xcode, puedes gestionar eficientemente las versiones de tu proyecto, colaborar con otros y mantener una base de código limpia y organizada.
