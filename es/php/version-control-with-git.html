Introducción al Control de Versiones

El control de versiones es un sistema que registra los cambios en un archivo o conjunto de archivos a lo largo del tiempo para que puedas recordar versiones específicas más tarde. Git es uno de los sistemas de control de versiones más populares utilizados hoy en día.

  • ¿Por qué usar control de versiones?
    • Rastrear cambios a lo largo del tiempo.
    • Colaborar con otros.
    • Revertir a versiones anteriores.
    • Hacer copias de seguridad de tu trabajo.

Empezando con Git

Antes de sumergirte en Git, asegúrate de tenerlo instalado en tu sistema. Puedes descargarlo desde git-scm.com.

Comandos Básicos de Git

Aquí hay algunos comandos fundamentales de Git para que comiences:

  • Inicializar un repositorio:

    git init

    Este comando inicializa un nuevo repositorio Git en tu directorio actual.

  • Clonar un repositorio:

    git clone <repository_url>

    Este comando crea una copia de un repositorio existente.

  • Verificar el estado de tu repositorio:

    git status

    Este comando muestra el estado de los cambios como no rastreados, modificados o en etapa.

  • Agregar archivos al área de preparación:

    git add <file_name>

    Este comando prepara archivos específicos para el próximo commit. Usa git add . para preparar todos los cambios.

  • Confirmar cambios:

    git commit -m "Tu mensaje de commit"

    Este comando confirma los cambios preparados en el repositorio con un mensaje.

  • Empujar cambios a un repositorio remoto:

    git push origin <branch_name>

    Este comando empuja tus commits locales al repositorio remoto.

  • Tirar cambios de un repositorio remoto:

    git pull origin <branch_name>

    Este comando obtiene y fusiona cambios del repositorio remoto a tu repositorio local.

Ramas y Fusiones

Las ramas son una característica poderosa en Git que te permite crear líneas de desarrollo separadas.

Creación y Gestión de Ramas

  • Crear una nueva rama:

    git branch <branch_name>

    Este comando crea una nueva rama.

  • Cambiar a una rama:

    git checkout <branch_name>

    Este comando cambia a la rama especificada.

  • Crear y cambiar a una nueva rama:

    git checkout -b <branch_name>

    Este comando crea una nueva rama y cambia a ella inmediatamente.

Fusionar Ramas

  • Fusionar una rama en la rama actual: 
    git merge <branch_name>
    Este comando fusiona la rama especificada en la rama actual.

Manejo de Conflictos de Fusión

A veces, puedes encontrar conflictos durante una fusión. Git marcará los conflictos en los archivos afectados, y necesitarás resolverlos manualmente.

  • Ver conflictos:

    git status

    Este comando te mostrará qué archivos tienen conflictos.

  • Después de resolver los conflictos, agrega los archivos resueltos:

    git add <file_name>

  • Confirmar la fusión:

    git commit

Trabajando con Repositorios Remotos

Los repositorios remotos son versiones de tu proyecto que están alojadas en internet o en una red.

Agregar un Repositorio Remoto

  • Agregar un repositorio remoto: 
    git remote add origin <repository_url>
    Este comando agrega un repositorio remoto.

Obtener y Tirar de Repositorios Remotos

  • Obtener cambios de un repositorio remoto:

    git fetch origin

    Este comando obtiene cambios del repositorio remoto pero no los fusiona.

  • Tirar cambios de un repositorio remoto:

    git pull origin <branch_name>

    Este comando obtiene y fusiona cambios del repositorio remoto.

Empujar a Repositorios Remotos

  • Empujar cambios a un repositorio remoto: 
    git push origin <branch_name>
    Este comando empuja tus commits locales al repositorio remoto.

Características Avanzadas de Git

Guardar Cambios

Guardar cambios te permite guardar tus cambios temporalmente sin confirmarlos.

  • Guardar cambios:

    git stash

    Este comando guarda tus cambios.

  • Aplicar cambios guardados:

    git stash apply

    Este comando aplica los cambios guardados.

Rebase

Rebase es una forma de integrar cambios de una rama en otra.

  • Rebase una rama: 
    git rebase <branch_name>
    Este comando hace rebase de la rama actual en la rama especificada.

Etiquetado

Las etiquetas se utilizan para marcar puntos específicos en la historia como importantes.

  • Crear una etiqueta:

    git tag <tag_name>

    Este comando crea una etiqueta.

  • Empujar etiquetas a un repositorio remoto:

    git push origin --tags

    Este comando empuja todas las etiquetas al repositorio remoto.

Conclusión

Git es una herramienta esencial para cualquier desarrollador, proporcionando características poderosas para el control de versiones y la colaboración. Al dominar Git, puedes gestionar tus proyectos PHP de manera más efectiva, colaborar con otros y mantener un historial de tu trabajo. Practica los comandos y conceptos discutidos en este tema para volverte competente en el uso de Git para tu desarrollo en PHP.
