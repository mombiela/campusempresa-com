Resumen

En esta sección, exploraremos los fundamentos de la creación de juegos multijugador en Unity. Cubriremos los conceptos básicos, herramientas y técnicas necesarias para comenzar con el desarrollo de juegos multijugador. Al final de esta sección, tendrás una comprensión sólida de cómo configurar un juego multijugador simple en Unity.

Conceptos Clave

  • Redes: El proceso de conectar múltiples dispositivos para compartir datos.
  • Modelo Cliente-Servidor: Una arquitectura común donde los clientes (jugadores) se conectan a un servidor central.
  • Modelo Peer-to-Peer: Una arquitectura donde cada jugador se conecta directamente con otros jugadores.
  • Sincronización: Asegurarse de que todos los jugadores vean el mismo estado del juego.
  • Latencia: El retraso entre una acción y su efecto en el juego, a menudo debido a los tiempos de transmisión de la red.

Herramientas y Frameworks

  • Unity Networking (UNet): La solución de red integrada de Unity (obsoleta pero aún utilizada en proyectos heredados).
  • Mirror: Una biblioteca de red impulsada por la comunidad para Unity, basada en UNet.
  • Photon Unity Networking (PUN): Una solución de red de terceros popular para Unity.

Configuración de un Juego Multijugador Básico

Paso 1: Configuración del Proyecto

  1. Crear un Nuevo Proyecto de Unity:

    • Abre Unity Hub y crea un nuevo proyecto 3D.
    • Nombra tu proyecto "MultiplayerGame".

  2. Instalar el Paquete de Red:

    • Para este ejemplo, usaremos Mirror.
    • Ve a la Tienda de Activos de Unity y busca "Mirror".
    • Importa el paquete Mirror en tu proyecto.

Paso 2: Creación del Network Manager

  1. Agregar Network Manager:
    • Crea un GameObject vacío en tu escena y nómbralo "NetworkManager".
    • Agrega los componentes NetworkManager y NetworkManagerHUD a este GameObject.
using Mirror;

public class MyNetworkManager : NetworkManager
{
    public override void OnServerAddPlayer(NetworkConnection conn)
    {
        // Llama a la funcionalidad base (agrega el jugador)
        base.OnServerAddPlayer(conn);

        // La configuración adicional para el jugador se puede hacer aquí
    }
}
  1. Configurar Network Manager:
    • Configura el Player Prefab con tu GameObject de jugador.
    • Asegúrate de que el componente NetworkManagerHUD esté habilitado para una interfaz de usuario simple para iniciar/detener el servidor y el cliente.

Paso 3: Creación del Prefab del Jugador

  1. Crear GameObject del Jugador:

    • Crea un nuevo GameObject y nómbralo "Player".
    • Agrega un componente NetworkIdentity al GameObject del Jugador.
    • Agrega un componente NetworkTransform para sincronizar la posición del jugador a través de la red.

  2. Crear Script del Jugador:

    • Crea un nuevo script llamado PlayerController y adjúntalo al GameObject del Jugador.
using UnityEngine;
using Mirror;

public class PlayerController : NetworkBehaviour
{
    public float moveSpeed = 5f;

    void Update()
    {
        if (!isLocalPlayer) return;

        float moveX = Input.GetAxis("Horizontal") * moveSpeed * Time.deltaTime;
        float moveZ = Input.GetAxis("Vertical") * moveSpeed * Time.deltaTime;

        transform.Translate(new Vector3(moveX, 0, moveZ));
    }
}
  1. Convertir Jugador a Prefab:
    • Arrastra el GameObject del Jugador a la ventana del Proyecto para crear un prefab.
    • Elimina el GameObject del Jugador de la escena.
    • Asigna el prefab del Jugador al campo Player Prefab en el NetworkManager.

Paso 4: Prueba del Juego Multijugador

  1. Ejecutar el Juego:
    • Haz clic en el botón Play en Unity.
    • Usa el NetworkManagerHUD para iniciar un Host (que actúa como servidor y cliente).
    • Abre una segunda instancia del juego y conéctate como cliente.
    • Mueve ambos jugadores y observa la sincronización.

Conclusión

En esta sección, cubrimos los conceptos básicos de la configuración de un juego multijugador en Unity usando la biblioteca de red Mirror. Discutimos conceptos clave de redes, configuramos un NetworkManager, creamos un prefab de jugador y probamos la funcionalidad multijugador. Este conocimiento fundamental servirá como un trampolín para características multijugador más avanzadas y optimizaciones en Unity.
