Introducción a la API de Redes de Unity

La API de Redes de Unity permite a los desarrolladores crear juegos multijugador gestionando objetos en red, conexiones de jugadores y sincronización de datos. Esta sección introducirá los conceptos y componentes básicos de la API de Redes de Unity.

  • NetworkManager: Gestiona el estado de la red, incluyendo el inicio y la detención de servidores y clientes.
  • NetworkIdentity: Identifica objetos en red y asegura que estén sincronizados entre los clientes.
  • NetworkBehaviour: Clase base para scripts en red, proporcionando ganchos para eventos de red.
  • NetworkTransform: Sincroniza la posición, rotación y escala de objetos en red.

Configuración de un Juego Básico en Red

Creación de un NetworkManager

El NetworkManager es el componente principal para gestionar las conexiones de red. Aquí se explica cómo configurarlo:

  1. Agregar el Componente NetworkManager:
    • Crea un GameObject vacío en tu escena.
    • Agrega el componente NetworkManager al GameObject.
    • Opcionalmente, agrega el componente NetworkManagerHUD para una interfaz de usuario simple que controle el estado de la red.
using UnityEngine;
using UnityEngine.Networking;

public class MyNetworkManager : NetworkManager
{
    // Se puede agregar lógica personalizada para la gestión de la red aquí
}
  1. Configurar el NetworkManager:
    • Establece el Player Prefab en el objeto jugador que deseas generar para cada cliente.
    • Configura la Network Address y el Network Port para el servidor.

Generación de Objetos en Red

Los objetos en red se instancian y sincronizan entre todos los clientes. Usa el componente NetworkIdentity para marcar objetos para la red.

  1. Agregar el Componente NetworkIdentity:
    • Adjunta el componente NetworkIdentity al GameObject que deseas poner en red.
    • Asegúrate de que la Local Player Authority esté marcada si el objeto debe ser controlado por el cliente.
using UnityEngine;
using UnityEngine.Networking;

public class PlayerController : NetworkBehaviour
{
    void Update()
    {
        if (!isLocalPlayer)
            return;

        // Lógica de control del jugador aquí
    }
}
  1. Generación del Objeto:
    • Usa NetworkServer.Spawn para generar objetos en el servidor y sincronizarlos con los clientes.
public class GameManager : NetworkBehaviour
{
    public GameObject playerPrefab;

    public override void OnStartServer()
    {
        GameObject player = Instantiate(playerPrefab);
        NetworkServer.Spawn(player);
    }
}

Gestión de Conexiones de Jugadores

Gestionar las conexiones de jugadores implica manejar eventos cuando los jugadores se unen o abandonan el juego.

  • OnServerConnect: Se llama cuando un cliente se conecta al servidor.
  • OnServerDisconnect: Se llama cuando un cliente se desconecta del servidor.
  • OnClientConnect: Se llama cuando el cliente se conecta al servidor.
  • OnClientDisconnect: Se llama cuando el cliente se desconecta del servidor.
public class MyNetworkManager : NetworkManager
{
    public override void OnServerConnect(NetworkConnection conn)
    {
        Debug.Log("Cliente conectado: " + conn.address);
    }

    public override void OnServerDisconnect(NetworkConnection conn)
    {
        Debug.Log("Cliente desconectado: " + conn.address);
        base.OnServerDisconnect(conn);
    }

    public override void OnClientConnect(NetworkConnection conn)
    {
        Debug.Log("Conectado al servidor");
    }

    public override void OnClientDisconnect(NetworkConnection conn)
    {
        Debug.Log("Desconectado del servidor");
        base.OnClientDisconnect(conn);
    }
}

Sincronización de Datos

La sincronización de datos asegura que todos los clientes tengan el mismo estado para los objetos en red.

SyncVars

Las SyncVars son variables que se sincronizan automáticamente desde el servidor a todos los clientes.

public class Player : NetworkBehaviour
{
    [SyncVar]
    public int health;

    void TakeDamage(int amount)
    {
        if (!isServer)
            return;

        health -= amount;
    }
}

Comandos y RPCs

Los comandos y RPCs se utilizan para enviar mensajes entre clientes y el servidor.

  • Comandos: Métodos llamados por los clientes pero ejecutados en el servidor.
  • ClientRPCs: Métodos llamados por el servidor pero ejecutados en los clientes.
public class Player : NetworkBehaviour
{
    [Command]
    void CmdTakeDamage(int amount)
    {
        health -= amount;
    }

    [ClientRpc]
    void RpcUpdateHealth(int newHealth)
    {
        health = newHealth;
    }
}

Conclusión

La API de Redes de Unity proporciona un marco robusto para crear juegos multijugador. Al comprender y utilizar componentes como NetworkManager, NetworkIdentity y NetworkBehaviour, los desarrolladores pueden gestionar conexiones de jugadores, sincronizar datos y manejar objetos en red de manera efectiva. La práctica y la experimentación con estos conceptos ayudarán a solidificar tu comprensión y capacidad para crear experiencias multijugador atractivas.
