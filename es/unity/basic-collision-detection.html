La detección de colisiones es un concepto fundamental en el desarrollo de juegos, permitiendo que los objetos interactúen entre sí en un entorno virtual. En Unity, la detección de colisiones se maneja a través de motores de física y colisionadores. Esta sección te guiará a través de los conceptos básicos de la detección de colisiones, desde la comprensión de los colisionadores hasta la implementación de respuestas simples a colisiones.

Comprendiendo los Colisionadores

Los colisionadores son componentes que definen la forma de un objeto para propósitos de colisiones físicas. No necesitan coincidir exactamente con la apariencia visual del objeto, pero deben aproximar su forma.

  • Tipos de Colisionadores:
    • Box Collider: Una forma de caja rectangular.
    • Sphere Collider: Una forma esférica.
    • Capsule Collider: Una forma de cápsula.
    • Mesh Collider: Un colisionador que usa la malla del objeto para la detección de colisiones.
    • Colisionadores 2D: Similares a los colisionadores 3D pero usados para juegos 2D (por ejemplo, Box Collider 2D, Circle Collider 2D).

Ejemplo: Añadiendo un Box Collider

Para añadir un Box Collider a un GameObject en Unity:

  1. Selecciona el GameObject en la Jerarquía.
  2. En la ventana del Inspector, haz clic en "Add Component."
  3. Escribe "Box Collider" y selecciónalo de la lista.
// Ejemplo de añadir un Box Collider mediante script
void Start() {
    gameObject.AddComponent<BoxCollider>();
}

Componente Rigidbody

Para que la detección de colisiones funcione efectivamente, al menos uno de los objetos en colisión debe tener un componente Rigidbody. El componente Rigidbody hace que el GameObject esté sujeto a la física, como la gravedad y las colisiones.

  • Añadiendo un Rigidbody:
    • Selecciona el GameObject.
    • Haz clic en "Add Component" en el Inspector.
    • Escribe "Rigidbody" y selecciónalo.
// Ejemplo de añadir un Rigidbody mediante script
void Start() {
    gameObject.AddComponent<Rigidbody>();
}

Detectando Colisiones

Unity proporciona varios métodos para detectar colisiones y desencadenar respuestas. Los métodos más comunes son OnCollisionEnter, OnCollisionStay y OnCollisionExit.

  • OnCollisionEnter: Se llama cuando el colisionador/rigidbody ha comenzado a tocar otro rigidbody/colisionador.
  • OnCollisionStay: Se llama una vez por frame para cada colisionador/rigidbody que está tocando otro rigidbody/colisionador.
  • OnCollisionExit: Se llama cuando el colisionador/rigidbody ha dejado de tocar otro rigidbody/colisionador.

Ejemplo: Usando OnCollisionEnter

void OnCollisionEnter(Collision collision) {
    // Verifica si el objeto colisionado tiene una etiqueta específica
    if (collision.gameObject.tag == "Enemy") {
        Debug.Log("¡Colisionaste con un enemigo!");
    }
}

Colisionadores de Trigger

Los colisionadores de trigger se usan para detectar superposiciones sin aplicar fuerzas físicas. Son útiles para detectar cuando un objeto entra en un área específica, como un punto de control o un objeto para recoger.

  • Configurando un Colisionador como Trigger:
    • Selecciona el GameObject con el Colisionador.
    • En el componente del Colisionador, marca la casilla "Is Trigger".

Ejemplo: Usando OnTriggerEnter

void OnTriggerEnter(Collider other) {
    // Verifica si el objeto que activa el trigger tiene una etiqueta específica
    if (other.gameObject.tag == "Pickup") {
        Debug.Log("¡Recogiste un objeto!");
        // Realiza lógica adicional, como aumentar la puntuación
    }
}

Resumen

En esta sección, cubrimos los conceptos básicos de la detección de colisiones en Unity, incluyendo:

  • Comprender y añadir diferentes tipos de colisionadores.
  • La importancia del componente Rigidbody para las interacciones físicas.
  • Detectar colisiones usando OnCollisionEnter, OnCollisionStay y OnCollisionExit.
  • Usar colisionadores de trigger para interacciones no físicas con OnTriggerEnter.

Al dominar estos conceptos, podrás crear juegos más interactivos y dinámicos. Practica añadiendo colisionadores y componentes Rigidbody a tus GameObjects e implementando lógica de detección de colisiones en tus scripts.
