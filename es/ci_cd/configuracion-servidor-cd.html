Introducción

En esta sección del curso sobre CI/CD, aprenderás a configurar un servidor de despliegue continuo. Este proceso es esencial para automatizar la integración y el despliegue de aplicaciones, mejorando la eficiencia y reduciendo los errores humanos. Utilizaremos herramientas populares como Jenkins, GitLab CI/CD, y CircleCI para ilustrar los conceptos.

Requisitos Previos

Antes de comenzar, asegúrate de tener los siguientes requisitos:

  • Conocimientos básicos de CI/CD.
  • Familiaridad con sistemas operativos basados en Unix/Linux.
  • Acceso a un servidor o máquina virtual con permisos de administrador.
  • Instalación de Docker (opcional pero recomendado).

Selección de la Herramienta de CI/CD

Existen varias herramientas para configurar un servidor de despliegue continuo. Aquí comparamos algunas de las más populares:

| Herramienta | Ventajas | Desventajas | |---------------|-----------------------------------------------|----------------------------------------------| | Jenkins | Open-source, altamente configurable | Requiere configuración manual extensa | | GitLab CI/CD | Integrado con GitLab, fácil de usar | Puede ser costoso en versiones premium | | CircleCI | Fácil de configurar, buena integración con GitHub | Limitaciones en el plan gratuito |

Instalación de Jenkins

Paso 1: Instalación de Java

Jenkins requiere Java para funcionar. Instala Java con los siguientes comandos:

sudo apt update
sudo apt install openjdk-11-jdk -y

Paso 2: Instalación de Jenkins

Añade el repositorio de Jenkins y luego instala Jenkins:

wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt update
sudo apt install jenkins -y

Paso 3: Iniciar Jenkins

Inicia el servicio de Jenkins y verifica su estado:

sudo systemctl start jenkins
sudo systemctl status jenkins

Paso 4: Configuración Inicial

Accede a Jenkins en tu navegador web en http://<tu-servidor>:8080. Sigue las instrucciones en pantalla para completar la configuración inicial.

Configuración de un Pipeline Básico en Jenkins

Paso 1: Crear un Nuevo Job

  1. En el panel de Jenkins, selecciona "Nuevo Item".
  2. Elige "Pipeline" y proporciona un nombre para tu job.

Paso 2: Definir el Pipeline

En la sección de configuración del job, define tu pipeline utilizando el siguiente script de ejemplo:

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Comandos de construcción
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Comandos de prueba
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Comandos de despliegue
            }
        }
    }
}

Paso 3: Ejecutar el Pipeline

Guarda la configuración y ejecuta el pipeline para verificar que todo funcione correctamente.

Configuración de GitLab CI/CD

Paso 1: Crear un Archivo `.gitlab-ci.yml`

En el directorio raíz de tu proyecto, crea un archivo llamado .gitlab-ci.yml con el siguiente contenido:

stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - echo "Building the project..."

test:
  stage: test
  script:
    - echo "Running tests..."

deploy:
  stage: deploy
  script:
    - echo "Deploying the project..."

Paso 2: Configurar Runners

Asegúrate de tener runners configurados en tu instancia de GitLab. Puedes hacerlo desde la sección de administración de GitLab.

Paso 3: Verificar el Pipeline

Haz un commit y push de tu archivo .gitlab-ci.yml a tu repositorio. GitLab automáticamente ejecutará el pipeline definido.

Conclusión

Configurar un servidor de despliegue continuo es un paso crucial para cualquier equipo de desarrollo que busque automatizar y mejorar su flujo de trabajo. En este tema, hemos cubierto la instalación y configuración básica de Jenkins y GitLab CI/CD, dos de las herramientas más populares en el ámbito de CI/CD. Con esta base, estarás preparado para explorar configuraciones más avanzadas y personalizadas según las necesidades de tu proyecto.

¡Sigue practicando y experimentando con diferentes configuraciones para dominar el arte del despliegue continuo!
