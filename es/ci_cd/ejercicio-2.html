Introducción

En este ejercicio, aprenderás a automatizar pruebas dentro de un pipeline de CI/CD. La automatización de pruebas es crucial para garantizar que el código nuevo no rompa funcionalidades existentes y que el software se mantenga estable y de alta calidad.

Objetivos

  • Comprender la importancia de la automatización de pruebas en CI/CD.
  • Configurar un entorno de pruebas automatizadas.
  • Integrar pruebas automatizadas en un pipeline de CI/CD.

Conceptos Clave

  • Pruebas Unitarias: Pruebas que verifican el funcionamiento de componentes individuales del código.
  • Pruebas de Integración: Pruebas que verifican la interacción entre diferentes componentes del sistema.
  • Pruebas de Funcionalidad: Pruebas que aseguran que el sistema cumple con los requisitos funcionales.
  • Pipeline de CI/CD: Un conjunto de procesos automatizados que permiten la integración continua y la entrega continua del software.

Configuración del Entorno de Pruebas

Herramientas Necesarias

  • Framework de Pruebas: JUnit (para Java), pytest (para Python), etc.
  • CI/CD Tool: Jenkins, GitLab CI, Travis CI, etc.

Ejemplo de Configuración con pytest y GitLab CI

Paso 1: Configurar pytest

  1. Instala pytest:

    pip install pytest

  2. Crea un archivo de prueba test_sample.py:

    def test_example():
    assert 1 + 1 == 2

Paso 2: Configurar GitLab CI

  1. Crea un archivo .gitlab-ci.yml en la raíz de tu proyecto:

    stages:
  - test

test:
  stage: test
  script:
    - pip install pytest
    - pytest

  2. Este archivo define una etapa de prueba que instala pytest y ejecuta las pruebas.

Integración de Pruebas en el Pipeline de CI/CD

Ejemplo de Pipeline en Jenkins

  1. Instala el plugin de Git en Jenkins.
  2. Configura un nuevo proyecto Freestyle en Jenkins.
  3. En la sección "Source Code Management", selecciona Git y proporciona la URL de tu repositorio.
  4. En la sección "Build Triggers", selecciona "GitHub hook trigger for GITScm polling".
  5. En la sección "Build", agrega un paso de ejecución de shell: 
    pip install pytest
pytest

Ejemplo de Pipeline en GitLab CI

  1. Asegúrate de que tu archivo .gitlab-ci.yml esté correctamente configurado como se muestra en la sección anterior.
  2. Cada vez que hagas un push a tu repositorio, GitLab CI ejecutará las pruebas automáticamente.

Ejercicio Práctico

  1. Configura pytest en tu proyecto: Sigue los pasos mencionados para instalar y configurar pytest.
  2. Escribe Pruebas Unitarias: Crea al menos tres pruebas unitarias para diferentes funciones en tu código.
  3. Configura GitLab CI: Crea un archivo .gitlab-ci.yml y asegúrate de que las pruebas se ejecuten automáticamente en cada push.
  4. Verifica los Resultados: Asegúrate de que las pruebas se ejecuten correctamente y verifica los resultados en la interfaz de GitLab CI.

Conclusión

La automatización de pruebas es un componente esencial en cualquier pipeline de CI/CD. Al integrar pruebas automatizadas, puedes detectar errores de manera temprana y asegurar la calidad del software. En este ejercicio, has aprendido a configurar un entorno de pruebas, escribir pruebas unitarias y automatizar su ejecución en un pipeline de CI/CD.
