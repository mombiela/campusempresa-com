Introducción

En este proyecto final, aplicaremos todos los conceptos y técnicas aprendidas a lo largo del curso para implementar una solución completa de Integración Continua (CI) y Despliegue Continuo (CD). Este proyecto servirá como una demostración práctica de tus habilidades y conocimientos en CI/CD.

Objetivos del Proyecto

  • Implementar un pipeline de CI/CD desde cero.
  • Integrar herramientas de CI/CD como Jenkins, GitHub Actions, GitLab CI, o CircleCI.
  • Configurar pruebas automatizadas y análisis de calidad de código.
  • Desplegar una aplicación en un entorno de producción.

Requisitos del Proyecto

  • Repositorio de Código: Utilizar un repositorio en GitHub, GitLab, o Bitbucket.
  • Aplicación: Puede ser una aplicación web sencilla en cualquier lenguaje de programación (por ejemplo, Node.js, Python, Java).
  • Pipeline de CI/CD: Configurar un pipeline que incluya:
    • Construcción del proyecto.
    • Ejecución de pruebas unitarias y de integración.
    • Análisis de calidad de código.
    • Despliegue automatizado a un entorno de producción (puede ser un servidor en la nube como AWS, Azure, o un contenedor Docker).

Pasos para la Implementación

1. Configuración del Repositorio

  • Crear un nuevo repositorio en la plataforma de tu elección (GitHub, GitLab, Bitbucket).
  • Clonar el repositorio en tu máquina local.
  • Añadir el código de tu aplicación al repositorio.
  • Configurar un archivo README.md con una descripción del proyecto y las instrucciones para ejecutar la aplicación localmente.

2. Configuración del Pipeline de CI/CD

Jenkins

  1. Instalación y Configuración de Jenkins:

    • Instalar Jenkins en tu máquina local o en un servidor.
    • Configurar Jenkins para que tenga acceso a tu repositorio de código.

  2. Creación del Pipeline:

    • Crear un nuevo pipeline en Jenkins.
    • Configurar el pipeline para que se ejecute automáticamente en cada commit.
    pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'npm install'
            }
        }
        stage('Test') {
            steps {
                sh 'npm test'
            }
        }
        stage('Deploy') {
            steps {
                sh 'npm run deploy'
            }
        }
    }
}

GitHub Actions

  1. Configuración del Workflow:

    • Crear un archivo .github/workflows/ci-cd.yml en tu repositorio.
    name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

    - name: Deploy
      run: npm run deploy

3. Pruebas Automatizadas y Análisis de Calidad de Código

  • Pruebas Unitarias: Asegúrate de que tu aplicación tenga pruebas unitarias configuradas.
  • Pruebas de Integración: Configura pruebas de integración si es necesario.
  • Análisis de Calidad de Código: Utiliza herramientas como ESLint, SonarQube, o CodeClimate para analizar la calidad del código.

4. Despliegue Automatizado

  • Entorno de Producción: Configura un entorno de producción para tu aplicación. Puede ser un servidor en la nube, un contenedor Docker, o una plataforma como Heroku.
  • Automatización del Despliegue: Configura el pipeline para que despliegue automáticamente la aplicación al entorno de producción después de pasar todas las pruebas.

Ejemplo Completo

A continuación, se muestra un ejemplo completo de un pipeline de CI/CD utilizando GitHub Actions para una aplicación Node.js:

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

    - name: Lint code
      run: npm run lint

    - name: Deploy
      run: npm run deploy

Conclusión

Este proyecto final te permitirá consolidar tus conocimientos en CI/CD y demostrar tu capacidad para implementar una solución completa desde cero. Asegúrate de documentar cada paso y de probar exhaustivamente tu pipeline para garantizar que funcione correctamente en todos los escenarios posibles. ¡Buena suerte!
