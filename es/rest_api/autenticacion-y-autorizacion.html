Introducción

En el desarrollo de REST APIs, la seguridad es un aspecto crucial. Dos conceptos fundamentales para garantizar la seguridad son la autenticación y la autorización. Aunque a menudo se usan de manera intercambiable, tienen significados distintos:

  • Autenticación: Proceso de verificar la identidad de un usuario o sistema.
  • Autorización: Proceso de determinar si un usuario o sistema autenticado tiene permiso para realizar una acción específica.

Autenticación

La autenticación es el primer paso para asegurar una API. Existen varios métodos para autenticar usuarios en una REST API:

Métodos de Autenticación

  1. Autenticación Básica (Basic Authentication)

    • Envia el nombre de usuario y la contraseña en cada solicitud HTTP.
    • Codifica las credenciales en Base64.
    • No es segura por sí sola; debe usarse con HTTPS.
    GET /api/resource HTTP/1.1
Host: example.com
Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=

  2. Tokens de API (API Tokens)

    • Utiliza un token único para cada usuario.
    • El token se envía en el encabezado de la solicitud.
    GET /api/resource HTTP/1.1
Host: example.com
Authorization: Bearer your_api_token

  3. OAuth 2.0

    • Protocolo de autorización que permite a las aplicaciones obtener acceso limitado a los recursos de usuario.
    • Utiliza tokens de acceso y tokens de actualización.
    GET /api/resource HTTP/1.1
Host: example.com
Authorization: Bearer your_oauth_token

  4. JWT (JSON Web Tokens)

    • Token compacto y autónomo que se utiliza para transmitir información entre dos partes.
    • Incluye una firma para verificar la integridad y autenticidad del token.
    GET /api/resource HTTP/1.1
Host: example.com
Authorization: Bearer your_jwt_token

Autorización

Una vez que un usuario está autenticado, el siguiente paso es la autorización. Esto implica verificar si el usuario tiene permisos para acceder a ciertos recursos o realizar ciertas acciones.

Métodos de Autorización

  1. Roles y Permisos

    • Asigna roles a los usuarios (por ejemplo, administrador, usuario, invitado).
    • Define permisos específicos para cada rol.
    {
  "user": {
    "role": "admin",
    "permissions": ["read", "write", "delete"]
  }
}

  2. Listas de Control de Acceso (ACL)

    • Define permisos a nivel de recurso.
    • Cada recurso tiene una lista de usuarios y sus permisos asociados.
    {
  "resource": "file123",
  "acl": [
    {"user": "user1", "permissions": ["read"]},
    {"user": "user2", "permissions": ["read", "write"]}
  ]
}

  3. Políticas Basadas en Atributos (ABAC)

    • Utiliza atributos del usuario, del recurso y del entorno para tomar decisiones de autorización.
    • Más flexible y granular que los métodos basados en roles.
    {
  "user": {
    "attributes": {"department": "sales"}
  },
  "resource": {
    "attributes": {"type": "report"}
  },
  "environment": {
    "attributes": {"time": "business_hours"}
  },
  "policy": "allow if user.department == 'sales' and resource.type == 'report' and environment.time == 'business_hours'"
}

Ejemplos de Implementación

Ejemplo de Autenticación con JWT

  1. Generación del Token

    import jwt
import datetime

def generate_token(user_id):
    payload = {
        'user_id': user_id,
        'exp': datetime.datetime.utcnow() + datetime.timedelta(hours=1)
    }
    token = jwt.encode(payload, 'secret_key', algorithm='HS256')
    return token

  2. Verificación del Token

    def verify_token(token):
    try:
        payload = jwt.decode(token, 'secret_key', algorithms=['HS256'])
        return payload['user_id']
    except jwt.ExpiredSignatureError:
        return 'Token has expired'
    except jwt.InvalidTokenError:
        return 'Invalid token'

Ejemplo de Autorización Basada en Roles

  1. Definición de Roles y Permisos

    roles_permissions = {
    'admin': ['read', 'write', 'delete'],
    'user': ['read', 'write'],
    'guest': ['read']
}

  2. Verificación de Permisos

    def check_permission(user_role, permission):
    if permission in roles_permissions.get(user_role, []):
        return True
    return False

Conclusión

La autenticación y la autorización son componentes esenciales para la seguridad de una REST API. La autenticación asegura que solo los usuarios legítimos puedan acceder a la API, mientras que la autorización garantiza que estos usuarios solo puedan realizar acciones para las que tienen permiso. Al implementar estos mecanismos, es crucial elegir el método adecuado según las necesidades y el contexto de la aplicación, y siempre utilizar HTTPS para proteger las credenciales y los tokens en tránsito.
