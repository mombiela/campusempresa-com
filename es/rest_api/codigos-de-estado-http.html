Introducción

Los códigos de estado HTTP son respuestas estándar proporcionadas por los servidores web para indicar el resultado de una solicitud HTTP. Estos códigos son esenciales para el desarrollo y la depuración de APIs RESTful, ya que informan al cliente sobre el éxito o el fracaso de una solicitud.

Categorías de Códigos de Estado HTTP

Los códigos de estado HTTP se dividen en cinco categorías principales:

  1. 1xx (Informativos): Indican que la solicitud ha sido recibida y el proceso continúa.
  2. 2xx (Éxito): Indican que la solicitud fue recibida, entendida y aceptada con éxito.
  3. 3xx (Redirección): Indican que se deben tomar acciones adicionales para completar la solicitud.
  4. 4xx (Errores del Cliente): Indican que hubo un error en la solicitud del cliente.
  5. 5xx (Errores del Servidor): Indican que el servidor falló al cumplir con una solicitud aparentemente válida.

Códigos de Estado HTTP Comunes

2xx (Éxito)

  • 200 OK: La solicitud ha tenido éxito. La respuesta depende del método HTTP utilizado. 
    HTTP/1.1 200 OK
Content-Type: application/json

{
  "message": "Solicitud exitosa"
}
  • 201 Created: La solicitud ha tenido éxito y se ha creado un nuevo recurso. 
    HTTP/1.1 201 Created
Content-Type: application/json

{
  "id": 123,
  "message": "Recurso creado exitosamente"
}
  • 204 No Content: La solicitud ha tenido éxito, pero no hay contenido que devolver. 
    HTTP/1.1 204 No Content

3xx (Redirección)

  • 301 Moved Permanently: El recurso solicitado ha sido movido permanentemente a una nueva URL. 
    HTTP/1.1 301 Moved Permanently
Location: https://nueva-url.com/recurso
  • 302 Found: El recurso solicitado se encuentra temporalmente en una URL diferente. 
    HTTP/1.1 302 Found
Location: https://temporal-url.com/recurso

4xx (Errores del Cliente)

  • 400 Bad Request: La solicitud no pudo ser entendida o fue malformada. 
    HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "error": "Solicitud malformada"
}
  • 401 Unauthorized: La solicitud requiere autenticación del usuario. 
    HTTP/1.1 401 Unauthorized
Content-Type: application/json

{
  "error": "Autenticación requerida"
}
  • 403 Forbidden: El servidor entiende la solicitud, pero se niega a autorizarla. 
    HTTP/1.1 403 Forbidden
Content-Type: application/json

{
  "error": "Acceso denegado"
}
  • 404 Not Found: El recurso solicitado no se pudo encontrar. 
    HTTP/1.1 404 Not Found
Content-Type: application/json

{
  "error": "Recurso no encontrado"
}

5xx (Errores del Servidor)

  • 500 Internal Server Error: El servidor encontró una condición inesperada que le impidió cumplir con la solicitud. 
    HTTP/1.1 500 Internal Server Error
Content-Type: application/json

{
  "error": "Error interno del servidor"
}
  • 502 Bad Gateway: El servidor, actuando como una puerta de enlace o proxy, recibió una respuesta inválida del servidor upstream. 
    HTTP/1.1 502 Bad Gateway
Content-Type: application/json

{
  "error": "Puerta de enlace incorrecta"
}
  • 503 Service Unavailable: El servidor no está disponible temporalmente, generalmente debido a mantenimiento o sobrecarga. 
    HTTP/1.1 503 Service Unavailable
Content-Type: application/json

{
  "error": "Servicio no disponible"
}

Ejemplos Prácticos

Ejemplo de una Solicitud GET Exitosa

GET /api/usuarios/1 HTTP/1.1
Host: ejemplo.com

Respuesta:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "nombre": "Juan Pérez",
  "email": "juan.perez@ejemplo.com"
}

Ejemplo de una Solicitud POST con Creación de Recurso

POST /api/usuarios HTTP/1.1
Host: ejemplo.com
Content-Type: application/json

{
  "nombre": "Ana Gómez",
  "email": "ana.gomez@ejemplo.com"
}

Respuesta:

HTTP/1.1 201 Created
Content-Type: application/json

{
  "id": 2,
  "nombre": "Ana Gómez",
  "email": "ana.gomez@ejemplo.com"
}

Conclusión

Comprender los códigos de estado HTTP es fundamental para el desarrollo y la depuración de APIs RESTful. Estos códigos proporcionan información crucial sobre el resultado de las solicitudes HTTP y ayudan a identificar y resolver problemas de manera eficiente. Al familiarizarse con los códigos de estado más comunes, los desarrolladores pueden mejorar la comunicación entre el cliente y el servidor, asegurando una experiencia de usuario más fluida y efectiva.
