Introducción a Postman

Postman es una herramienta popular para el desarrollo y prueba de APIs. Permite a los desarrolladores enviar solicitudes HTTP, guardar respuestas y automatizar pruebas. Es especialmente útil para trabajar con APIs RESTful.

Características principales de Postman

  • Interfaz amigable: Permite crear y gestionar solicitudes HTTP de manera intuitiva.
  • Colecciones: Agrupa múltiples solicitudes en un solo lugar para facilitar la organización.
  • Variables: Permite el uso de variables para reutilizar valores en diferentes solicitudes.
  • Scripts: Soporta scripts pre-solicitud y de prueba para automatizar tareas.
  • Documentación: Genera documentación automáticamente a partir de las colecciones.

Instalación y Configuración

Para comenzar a usar Postman, primero debes instalarlo y configurarlo.

Pasos para la instalación

  1. Descargar Postman: Visita Postman y descarga la versión adecuada para tu sistema operativo.
  2. Instalar: Sigue las instrucciones de instalación específicas para tu sistema operativo.
  3. Crear una cuenta: Regístrate o inicia sesión para sincronizar tus colecciones y configuraciones.

Creación de Solicitudes HTTP

Postman permite crear diferentes tipos de solicitudes HTTP, como GET, POST, PUT, DELETE, entre otras.

Ejemplo de solicitud GET

Para realizar una solicitud GET a una API, sigue estos pasos:

  1. Abrir Postman.
  2. Crear una nueva solicitud: Haz clic en "New" y selecciona "Request".
  3. Configurar la solicitud:
    • Selecciona el método HTTP (GET).
    • Introduce la URL de la API (por ejemplo, https://jsonplaceholder.typicode.com/posts).
  4. Enviar la solicitud: Haz clic en "Send".
GET /posts HTTP/1.1
Host: jsonplaceholder.typicode.com

Ejemplo de solicitud POST

Para realizar una solicitud POST, sigue estos pasos:

  1. Crear una nueva solicitud: Haz clic en "New" y selecciona "Request".
  2. Configurar la solicitud:
    • Selecciona el método HTTP (POST).
    • Introduce la URL de la API (por ejemplo, https://jsonplaceholder.typicode.com/posts).
    • En la pestaña "Body", selecciona "raw" y elige "JSON" como tipo de contenido.
    • Introduce el cuerpo de la solicitud en formato JSON.
{
  "title": "foo",
  "body": "bar",
  "userId": 1
}
  1. Enviar la solicitud: Haz clic en "Send".
POST /posts HTTP/1.1
Host: jsonplaceholder.typicode.com
Content-Type: application/json

{
  "title": "foo",
  "body": "bar",
  "userId": 1
}

Uso de Colecciones y Variables

Las colecciones y variables en Postman son herramientas poderosas para organizar y reutilizar solicitudes.

Creación de Colecciones

  1. Crear una nueva colección: Haz clic en "New" y selecciona "Collection".
  2. Agregar solicitudes a la colección: Arrastra y suelta las solicitudes existentes o crea nuevas solicitudes dentro de la colección.

Uso de Variables

  1. Definir variables: En la pestaña "Variables" de una colección, define las variables necesarias.
  2. Usar variables en solicitudes: Utiliza la sintaxis {{variableName}} para insertar variables en las URL, encabezados, cuerpos de solicitud, etc.
{
  "title": "{{postTitle}}",
  "body": "{{postBody}}",
  "userId": {{userId}}
}

Automatización de Pruebas con Scripts

Postman permite escribir scripts para automatizar pruebas y validar respuestas.

Scripts Pre-solicitud

Estos scripts se ejecutan antes de enviar la solicitud. Puedes usarlos para configurar variables o modificar la solicitud.

pm.environment.set("currentTime", new Date().toISOString());

Scripts de Prueba

Estos scripts se ejecutan después de recibir la respuesta. Puedes usarlos para validar el contenido de la respuesta.

pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});

pm.test("Response time is less than 200ms", function () {
    pm.expect(pm.response.responseTime).to.be.below(200);
});

Conclusión

Postman es una herramienta esencial para cualquier desarrollador que trabaje con APIs. Su interfaz intuitiva, junto con características avanzadas como colecciones, variables y scripts, facilita enormemente el proceso de desarrollo y prueba de APIs. Al dominar Postman, podrás mejorar la eficiencia y calidad de tus pruebas de API, asegurando que tus aplicaciones funcionen correctamente y cumplan con los requisitos esperados.
