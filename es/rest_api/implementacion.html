Introducción

En esta sección, abordaremos la implementación de una REST API, cubriendo desde la configuración inicial hasta la creación de endpoints funcionales. Utilizaremos ejemplos prácticos y explicaciones detalladas para asegurar que los conceptos sean claros y aplicables.

Configuración Inicial

Para comenzar con la implementación de una REST API, es crucial configurar el entorno de desarrollo. En este curso, utilizaremos Node.js con Express.js como framework.

Instalación de Node.js y Express.js

  1. Instalar Node.js: Descarga e instala Node.js desde nodejs.org.
  2. Crear un proyecto nuevo: 
    mkdir my-rest-api
cd my-rest-api
npm init -y
  3. Instalar Express.js: 
    npm install express

Creación del Servidor Básico

Una vez que tenemos nuestro entorno configurado, el siguiente paso es crear un servidor básico que pueda manejar solicitudes HTTP.

Código del Servidor Básico

const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(port, () => {
  console.log(`Server is running on http://localhost:${port}`);
});
  • Explicación del Código:
    • express: Importa el módulo Express.
    • app: Crea una instancia de una aplicación Express.
    • app.get('/'): Define un endpoint GET en la ruta raíz ('/').
    • res.send('Hello World!'): Envía una respuesta de texto simple.
    • app.listen(port): Inicia el servidor en el puerto especificado.

Creación de Endpoints

Una REST API se compone de múltiples endpoints que permiten realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar).

Ejemplo de Endpoints CRUD

Crear (POST)

app.post('/items', (req, res) => {
  // Lógica para crear un nuevo ítem
  res.status(201).send('Item created');
});

Leer (GET)

app.get('/items', (req, res) => {
  // Lógica para obtener todos los ítems
  res.status(200).json({ items: [] });
});

app.get('/items/:id', (req, res) => {
  const itemId = req.params.id;
  // Lógica para obtener un ítem por ID
  res.status(200).json({ item: { id: itemId } });
});

Actualizar (PUT)

app.put('/items/:id', (req, res) => {
  const itemId = req.params.id;
  // Lógica para actualizar un ítem por ID
  res.status(200).send(`Item ${itemId} updated`);
});

Eliminar (DELETE)

app.delete('/items/:id', (req, res) => {
  const itemId = req.params.id;
  // Lógica para eliminar un ítem por ID
  res.status(200).send(`Item ${itemId} deleted`);
});

Manejo de Errores

Es fundamental manejar los errores de manera adecuada para proporcionar respuestas claras y útiles a los usuarios de la API.

Ejemplo de Manejo de Errores

app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something broke!');
});
  • Explicación del Código:
    • app.use((err, req, res, next) => {...}): Middleware para manejar errores.
    • console.error(err.stack): Imprime el stack trace del error en la consola.
    • res.status(500).send('Something broke!'): Envía una respuesta de error 500 al cliente.

Conclusión

En esta sección, hemos cubierto los pasos esenciales para implementar una REST API utilizando Node.js y Express.js. Desde la configuración inicial hasta la creación de endpoints CRUD y el manejo de errores, estos fundamentos te permitirán construir APIs robustas y escalables.

Recuerda que la práctica es clave para dominar estos conceptos. Te recomendamos que experimentes con diferentes configuraciones y casos de uso para fortalecer tu comprensión y habilidades.
