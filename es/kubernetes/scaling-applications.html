El escalado de aplicaciones es un aspecto fundamental de la gestión de cargas de trabajo en Kubernetes. Este tema te guiará a través de los conceptos, herramientas y mejores prácticas para escalar aplicaciones desde un nivel principiante hasta un nivel avanzado.

Introducción al Escalado

El escalado en Kubernetes implica ajustar el número de réplicas de tu aplicación para manejar cargas variables. Esto se puede hacer de manera manual o automática.

Conceptos Clave

  • ReplicaSet: Asegura que un número especificado de réplicas de pods estén ejecutándose en cualquier momento.
  • Horizontal Pod Autoscaler (HPA): Escala automáticamente el número de pods basado en la utilización observada de la CPU u otras métricas seleccionadas.
  • Vertical Pod Autoscaler (VPA): Ajusta los límites y solicitudes de recursos para los contenedores en los pods.
  • Cluster Autoscaler: Ajusta el tamaño del clúster de Kubernetes añadiendo o eliminando nodos.

Escalado Manual

El escalado manual implica establecer explícitamente el número de réplicas para tu aplicación.

Ejemplo

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app-container
        image: my-app-image
        ports:
        - containerPort: 80

En este ejemplo, el campo replicas está establecido en 3, lo que significa que Kubernetes mantendrá 3 réplicas del pod my-app.

Escalado con kubectl

También puedes escalar tu aplicación usando el comando kubectl:

kubectl scale deployment my-app --replicas=5

Este comando escala el despliegue my-app a 5 réplicas.

Horizontal Pod Autoscaler (HPA)

HPA escala automáticamente el número de pods en un despliegue o conjunto de réplicas basado en métricas observadas.

Ejemplo

Primero, asegúrate de que el Metrics Server esté instalado en tu clúster:

kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

Luego, crea un recurso HPA:

apiVersion: autoscaling/v1
kind: HorizontalPodAutoscaler
metadata:
  name: my-app-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  minReplicas: 1
  maxReplicas: 10
  targetCPUUtilizationPercentage: 50

Este HPA escalará el despliegue my-app entre 1 y 10 réplicas para mantener una utilización promedio de CPU del 50%.

Aplicando el HPA

kubectl apply -f my-app-hpa.yaml

Vertical Pod Autoscaler (VPA)

VPA ajusta las solicitudes y límites de recursos para los contenedores en tus pods.

Ejemplo

Primero, instala los componentes de VPA:

kubectl apply -f https://github.com/kubernetes/autoscaler/releases/latest/download/vertical-pod-autoscaler.yaml

Luego, crea un recurso VPA:

apiVersion: autoscaling.k8s.io/v1
kind: VerticalPodAutoscaler
metadata:
  name: my-app-vpa
spec:
  targetRef:
    apiVersion: "apps/v1"
    kind:       Deployment
    name:       my-app
  updatePolicy:
    updateMode: "Auto"

Este VPA ajustará automáticamente las solicitudes y límites de recursos para el despliegue my-app.

Aplicando el VPA

kubectl apply -f my-app-vpa.yaml

Cluster Autoscaler

Cluster Autoscaler ajusta el número de nodos en tu clúster basado en los requisitos de recursos de tus pods.

Ejemplo

Cluster Autoscaler se configura típicamente a través de tu proveedor de nube. Por ejemplo, en Google Kubernetes Engine (GKE), puedes habilitarlo a través de la consola de GKE o usando comandos gcloud.

Habilitando Cluster Autoscaler en GKE

gcloud container clusters update my-cluster --enable-autoscaling --min-nodes=1 --max-nodes=10 --zone=us-central1-a

Resumen

Escalar aplicaciones en Kubernetes es esencial para mantener el rendimiento y la eficiencia. Puedes escalar manualmente o usar características de autoescalado como HPA, VPA y Cluster Autoscaler para automatizar el proceso. Entender estas herramientas y cómo aplicarlas te ayudará a gestionar tus cargas de trabajo de manera efectiva.

Al dominar estas técnicas de escalado, puedes asegurar que tus aplicaciones sean resilientes, rentables y capaces de manejar cargas variables.
