Introducción

Kubernetes, a menudo abreviado como K8s, es una plataforma de código abierto diseñada para automatizar el despliegue, escalado y operación de contenedores de aplicaciones. Comprender los conceptos y la terminología clave es crucial para dominar Kubernetes desde niveles principiantes hasta avanzados.

Conceptos Básicos

Clúster

  • Definición: Un conjunto de nodos (máquinas) que ejecutan aplicaciones en contenedores gestionadas por Kubernetes.
  • Componentes:
    • Nodo Maestro: Gestiona el clúster y coordina todas las actividades.
    • Nodos Trabajadores: Ejecutan las aplicaciones en contenedores.

Nodo

  • Definición: Una sola máquina en el clúster de Kubernetes, que puede ser una máquina física o virtual.
  • Tipos:
    • Nodo Maestro: Maneja el plano de control.
    • Nodo Trabajador: Ejecuta las cargas de trabajo de las aplicaciones.

Pod

  • Definición: El objeto más pequeño y simple de Kubernetes. Un pod representa una sola instancia de un proceso en ejecución en tu clúster.
  • Características:
    • Puede contener uno o más contenedores.
    • Comparte recursos de almacenamiento y red.
apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx

Servicio

  • Definición: Una abstracción que define un conjunto lógico de Pods y una política para acceder a ellos.
  • Tipos:
    • ClusterIP: Expone el servicio en una IP interna del clúster.
    • NodePort: Expone el servicio en la IP de cada Nodo en un puerto estático.
    • LoadBalancer: Expone el servicio externamente usando un balanceador de carga del proveedor de la nube.
apiVersion: v1
kind: Service
metadata:
  name: example-service
spec:
  selector:
    app: example
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376
  type: ClusterIP

Despliegue

  • Definición: Proporciona actualizaciones declarativas a las aplicaciones, asegurando que el número deseado de Pods esté en ejecución.
  • Características:
    • Actualizaciones continuas.
    • Reversiones.
apiVersion: apps/v1
kind: Deployment
metadata:
  name: example-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

Espacio de Nombres

  • Definición: Una forma de dividir los recursos del clúster entre múltiples usuarios.
  • Casos de Uso:
    • Separación de entornos (por ejemplo, desarrollo, pruebas, producción).
    • Gestión de cuotas de recursos.
apiVersion: v1
kind: Namespace
metadata:
  name: example-namespace

Conceptos Avanzados

StatefulSet

  • Definición: Gestiona el despliegue y escalado de un conjunto de Pods, y proporciona garantías sobre el orden y la unicidad de estos Pods.
  • Casos de Uso:
    • Aplicaciones con estado como bases de datos.
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: example-statefulset
spec:
  serviceName: "example"
  replicas: 3
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

DaemonSet

  • Definición: Asegura que todos (o algunos) Nodos ejecuten una copia de un Pod.
  • Casos de Uso:
    • Monitoreo de nodos.
    • Recopilación de registros.
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: example-daemonset
spec:
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

ConfigMap y Secret

  • ConfigMap:
    • Definición: Se utiliza para almacenar datos no confidenciales en pares clave-valor.
    • Casos de Uso: Configuración de ajustes.
apiVersion: v1
kind: ConfigMap
metadata:
  name: example-configmap
data:
  example.property: "exampleValue"
  • Secret:
    • Definición: Se utiliza para almacenar información sensible, como contraseñas, tokens OAuth y claves ssh.
    • Casos de Uso: Almacenamiento seguro de datos sensibles.
apiVersion: v1
kind: Secret
metadata:
  name: example-secret
type: Opaque
data:
  password: cGFzc3dvcmQ=

Conclusión

Comprender estos conceptos y terminología clave es esencial para usar Kubernetes de manera efectiva. Desde los bloques de construcción básicos como Pods y Servicios hasta constructos más avanzados como StatefulSets y DaemonSets, cada componente juega un papel crítico en el ecosistema de Kubernetes. El dominio de estos conceptos proporcionará una base sólida para una exploración y uso avanzado de Kubernetes.
