Introducción a GitOps

  • Definición: GitOps es un conjunto de prácticas para gestionar la infraestructura y las configuraciones de aplicaciones utilizando Git.
  • Principios Básicos:
    • Descripciones Declarativas: Todas las configuraciones se describen de manera declarativa.
    • Control de Versiones: Las configuraciones se almacenan en un sistema de control de versiones (por ejemplo, Git).
    • Aplicación Automatizada: Los cambios en las configuraciones se aplican automáticamente al sistema.
    • Reconciliación Continua: El sistema asegura continuamente que el estado actual coincida con el estado deseado.

Introducción a ArgoCD

  • Definición: ArgoCD es una herramienta de entrega continua declarativa, GitOps, para Kubernetes.
  • Características Clave:
    • GitOps Declarativo: Utiliza repositorios Git como la fuente de verdad para el estado deseado de las aplicaciones.
    • Sincronización Automática: Sincroniza automáticamente el estado del clúster con el estado definido en Git.
    • Auto-Curación: Corrige automáticamente la desviación entre el estado deseado y el estado actual.
    • Soporte Multi-Clúster: Gestiona aplicaciones en múltiples clústeres de Kubernetes.

Configuración de ArgoCD

  • Requisitos Previos:

    • Un clúster de Kubernetes en funcionamiento.
    • La herramienta de línea de comandos kubectl configurada para interactuar con tu clúster.
    • Un repositorio Git con manifiestos de Kubernetes.

  • Pasos de Instalación:

    1. Instalar ArgoCD en Kubernetes: 
      kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
    2. Acceder al Servidor API de ArgoCD: 
      kubectl port-forward svc/argocd-server -n argocd 8080:443
    3. Iniciar Sesión en ArgoCD: 
      # Obtener la contraseña inicial del administrador
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
# Abrir la interfaz de usuario de ArgoCD en tu navegador en https://localhost:8080

Configuración de ArgoCD

  • Conectar un Repositorio Git:

    • Crear una Aplicación: 
      argocd app create my-app 
            --repo https://github.com/my-org/my-repo.git 
            --path path/to/manifests 
            --dest-server https://kubernetes.default.svc 
            --dest-namespace default
    • Sincronizar la Aplicación: 
      argocd app sync my-app

  • Configuración de la Aplicación:

    • Ejemplo de Manifiesto de Aplicación: 
      apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: guestbook
  namespace: argocd
spec:
  project: default
  source:
    repoURL: 'https://github.com/argoproj/argocd-example-apps'
    targetRevision: HEAD
    path: guestbook
  destination:
    server: 'https://kubernetes.default.svc'
    namespace: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true

Características Avanzadas de ArgoCD

  • Políticas de Sincronización Automatizadas:

    • Auto-Curación: Corrige automáticamente cualquier desviación entre el estado deseado y el estado actual.
    • Poda: Elimina automáticamente los recursos que ya no están definidos en el repositorio Git.

  • Gestión Multi-Clúster:

    • Agregar un Clúster: 
      argocd cluster add <context-name>

  • Comprobaciones de Salud Personalizadas:

    • Definir Comprobaciones de Salud Personalizadas: 
      apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: guestbook
  namespace: argocd
spec:
  project: default
  source:
    repoURL: 'https://github.com/argoproj/argocd-example-apps'
    targetRevision: HEAD
    path: guestbook
  destination:
    server: 'https://kubernetes.default.svc'
    namespace: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
  healthChecks:
    - apiVersion: apps/v1
      kind: Deployment
      name: guestbook-ui
      health:
        status: Healthy
        message: 'Guestbook UI is healthy'

Conclusión

  • Resumen:
    • GitOps con ArgoCD proporciona una forma poderosa de gestionar aplicaciones de Kubernetes de manera declarativa utilizando Git como la fuente de verdad.
    • ArgoCD automatiza el proceso de despliegue, asegurando que el estado actual del clúster coincida con el estado deseado definido en Git.
    • Las características avanzadas como las políticas de sincronización automatizadas, la gestión multi-clúster y las comprobaciones de salud personalizadas hacen de ArgoCD una herramienta versátil para la entrega continua en Kubernetes.

Al dominar GitOps con ArgoCD, puedes lograr despliegues más confiables y consistentes, reducir la intervención manual y mejorar la estabilidad general de tus entornos de Kubernetes.
