Introducción a las Definiciones de Recursos Personalizados (CRDs)

Las Definiciones de Recursos Personalizados (CRDs) son una característica poderosa en Kubernetes que te permite extender la API de Kubernetes con tus propios recursos personalizados. Esto te permite crear y gestionar recursos que no son parte de la API predeterminada de Kubernetes.

Conceptos Clave

  • Recursos Personalizados (CRs): Son extensiones de la API de Kubernetes que te permiten almacenar y recuperar datos estructurados.
  • Definiciones de Recursos Personalizados (CRDs): Definen el esquema para tus recursos personalizados, permitiendo que Kubernetes los entienda y gestione.
  • Controlador: Un bucle de control que observa el estado de tus recursos personalizados y realiza cambios para mover el estado actual hacia el estado deseado.

Creando una Definición de Recurso Personalizado

Para crear una CRD, necesitas definirla en un archivo YAML y aplicarla a tu clúster de Kubernetes.

Ejemplo de YAML de CRD

apiVersion: apiextensions.k8s.io/v1
kind: CustomResourceDefinition
metadata:
  name: crontabs.example.com
spec:
  group: example.com
  versions:
    - name: v1
      served: true
      storage: true
      schema:
        openAPIV3Schema:
          type: object
          properties:
            spec:
              type: object
              properties:
                cronSpec:
                  type: string
                image:
                  type: string
                replicas:
                  type: integer
  scope: Namespaced
  names:
    plural: crontabs
    singular: crontab
    kind: CronTab
    shortNames:
    - ct

Explicación

  • apiVersion: Especifica la versión de la API.
  • kind: Especifica el tipo de recurso, que es CustomResourceDefinition.
  • metadata: Contiene el nombre de la CRD.
  • spec: Define el grupo, versiones, alcance y nombres para el recurso personalizado.
    • group: Un agrupamiento lógico para tus recursos personalizados.
    • versions: Lista las versiones del recurso personalizado.
    • scope: Puede ser Namespaced o Cluster.
    • names: Define los nombres para el recurso personalizado.

Creando Recursos Personalizados

Una vez que la CRD se aplica, puedes crear instancias de tu recurso personalizado.

Ejemplo de YAML de Recurso Personalizado

apiVersion: example.com/v1
kind: CronTab
metadata:
  name: my-crontab
spec:
  cronSpec: "* * * * */5"
  image: my-cron-image
  replicas: 3

Explicación

  • apiVersion: Especifica la versión de la API y el grupo.
  • kind: Especifica el tipo de recurso personalizado.
  • metadata: Contiene el nombre del recurso personalizado.
  • spec: Define el estado deseado del recurso personalizado.

Gestionando Recursos Personalizados

Puedes gestionar tus recursos personalizados usando comandos estándar de kubectl.

Comandos de Ejemplo

  • Crear un Recurso Personalizado: 
    kubectl apply -f my-crontab.yaml
  • Obtener Recursos Personalizados: 
    kubectl get crontabs
  • Describir un Recurso Personalizado: 
    kubectl describe crontab my-crontab
  • Eliminar un Recurso Personalizado: 
    kubectl delete crontab my-crontab

Temas Avanzados

Validación y Valores Predeterminados

Puedes agregar validación y valores predeterminados a tus CRDs para asegurar la integridad de los datos y proporcionar valores predeterminados.

Ejemplo de Esquema de Validación

schema:
  openAPIV3Schema:
    type: object
    properties:
      spec:
        type: object
        properties:
          cronSpec:
            type: string
            pattern: '^[0-9*/,-]+ [0-9*/,-]+ [0-9*/,-]+ [0-9*/,-]+ [0-9*/,-]+$'
          image:
            type: string
          replicas:
            type: integer
            minimum: 1
            maximum: 10

Explicación

  • pattern: Asegura que el cronSpec siga un formato cron válido.
  • minimum y maximum: Aseguran que el valor de replicas esté entre 1 y 10.

Controladores Personalizados

Para gestionar el ciclo de vida de tus recursos personalizados, puedes escribir controladores personalizados usando frameworks como Kubebuilder o Operator SDK.

Conclusión

Las Definiciones de Recursos Personalizados (CRDs) proporcionan una forma flexible de extender la API de Kubernetes con tus propios recursos personalizados. Al definir CRDs, puedes crear, gestionar y validar recursos personalizados, permitiéndote adaptar Kubernetes a tus necesidades específicas. Las características avanzadas como la validación, los valores predeterminados y los controladores personalizados mejoran aún más las capacidades de las CRDs, convirtiéndolas en una herramienta poderosa para los desarrolladores de Kubernetes.
