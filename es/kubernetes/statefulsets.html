StatefulSets son un recurso de Kubernetes diseñado para gestionar aplicaciones con estado. A diferencia de los Deployments, que se utilizan para aplicaciones sin estado, los StatefulSets se utilizan para aplicaciones que requieren almacenamiento persistente e identidades de red estables.

Conceptos Clave

  • Aplicaciones con Estado: Aplicaciones que requieren almacenamiento persistente e identidades de red estables.
  • Pods: Los StatefulSets gestionan el despliegue y escalado de un conjunto de Pods, y proporcionan garantías sobre el orden y la unicidad de estos Pods.
  • Persistent Volume Claims (PVCs): Los StatefulSets trabajan con PVCs para proporcionar almacenamiento persistente para cada Pod.
  • Identidades de Red Estables: Cada Pod en un StatefulSet tiene una identidad de red única y estable.
  • Despliegue y Escalado Ordenado: Los Pods se crean, eliminan y escalan en un orden específico.

Fundamentos de StatefulSet

Creando un StatefulSet

Para crear un StatefulSet, necesitas definirlo en un archivo YAML. A continuación, se muestra un ejemplo de una definición simple de StatefulSet:

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: web
spec:
  serviceName: "nginx"
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: k8s.gcr.io/nginx-slim:0.8
        ports:
        - containerPort: 80
          name: web
  volumeClaimTemplates:
  - metadata:
      name: www
    spec:
      accessModes: [ "ReadWriteOnce" ]
      resources:
        requests:
          storage: 1Gi

Explicación

  • apiVersion: Especifica la versión de la API.
  • kind: Especifica que esto es un StatefulSet.
  • metadata: Contiene el nombre del StatefulSet.
  • spec: Define el estado deseado del StatefulSet.
    • serviceName: El nombre del servicio que gobierna este StatefulSet.
    • replicas: El número de Pods deseados.
    • selector: Una consulta de etiquetas sobre los Pods que deben coincidir con el recuento de réplicas.
    • template: La plantilla del Pod.
      • metadata: Etiquetas para los Pods.
      • spec: La especificación para los Pods.
        • containers: Las especificaciones del contenedor.
          • name: El nombre del contenedor.
          • image: La imagen del contenedor.
          • ports: Los puertos expuestos por el contenedor.
    • volumeClaimTemplates: Define los PVCs para los Pods.

Desplegando el StatefulSet

Para desplegar el StatefulSet, usa el siguiente comando:

kubectl apply -f statefulset.yaml

Verificando el StatefulSet

Para verificar que el StatefulSet está en ejecución, usa el siguiente comando:

kubectl get statefulsets

Deberías ver una salida similar a:

NAME   READY   AGE
web    3/3     1m

Conceptos Avanzados

Actualizaciones Graduales

Los StatefulSets soportan actualizaciones graduales, permitiéndote actualizar los Pods de manera controlada. Para realizar una actualización gradual, simplemente actualiza la imagen del contenedor en la definición del StatefulSet y aplica los cambios.

spec:
  template:
    spec:
      containers:
      - name: nginx
        image: k8s.gcr.io/nginx-slim:0.9

Aplica los cambios:

kubectl apply -f statefulset.yaml

Escalando un StatefulSet

Para escalar un StatefulSet, usa el siguiente comando:

kubectl scale statefulset web --replicas=5

Eliminando un StatefulSet

Para eliminar un StatefulSet sin eliminar los PVCs asociados, usa el siguiente comando:

kubectl delete statefulset web --cascade=orphan

Este comando elimina el StatefulSet pero deja los PVCs intactos.

Resumen

StatefulSets son un recurso poderoso de Kubernetes para gestionar aplicaciones con estado. Proporcionan identidades de red estables, almacenamiento persistente y despliegue y escalado ordenado. Al comprender y utilizar StatefulSets, puedes gestionar eficazmente aplicaciones complejas con estado en tu clúster de Kubernetes.

  • StatefulSets gestionan aplicaciones con estado.
  • Persistent Volume Claims (PVCs) proporcionan almacenamiento persistente.
  • Identidades de Red Estables aseguran que cada Pod tenga una identidad única.
  • Despliegue y Escalado Ordenado aseguran que los Pods se creen y eliminen en un orden específico.

Al dominar estos conceptos, puedes aprovechar los StatefulSets para gestionar aplicaciones con estado de manera eficiente en Kubernetes.
