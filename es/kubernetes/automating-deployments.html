Introducción

Automatizar los despliegues en Kubernetes es crucial para mantener la consistencia, reducir errores y mejorar la velocidad de lanzamiento de nuevas características. Esta sección te guiará desde las técnicas básicas hasta las avanzadas de automatización de despliegues utilizando Kubernetes.

Conceptos Clave

  • Despliegue: Un recurso de Kubernetes que proporciona actualizaciones declarativas a las aplicaciones.
  • ReplicaSet: Asegura que un número especificado de réplicas de pods estén ejecutándose en cualquier momento.
  • Actualizaciones Graduales: Reemplaza gradualmente las versiones antiguas de los pods con nuevas.
  • Despliegue Blue-Green: Mantiene dos entornos idénticos, uno activo y otro inactivo, para alternar entre ellos.
  • Despliegue Canary: Desvía gradualmente el tráfico de la versión antigua a la nueva.
  • Pipelines CI/CD: Los pipelines de Integración Continua y Despliegue Continuo automatizan el proceso de construir, probar y desplegar código.

Automatización Básica de Despliegues

Creando un Despliegue

Un despliegue simple puede ser creado usando un archivo YAML. A continuación, un ejemplo de una configuración básica de despliegue:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app-container
        image: my-app-image:1.0
        ports:
        - containerPort: 80
  • apiVersion: Especifica la versión de la API.
  • kind: El tipo de recurso de Kubernetes.
  • metadata: Metadatos sobre el despliegue, incluyendo el nombre.
  • spec: La especificación del despliegue, incluyendo el número de réplicas y la plantilla del pod.

Aplicando el Despliegue

Para aplicar el despliegue, usa el siguiente comando:

kubectl apply -f my-app-deployment.yaml

Este comando creará el despliegue e iniciará el número especificado de réplicas.

Estrategias Intermedias de Despliegue

Actualizaciones Graduales

Las actualizaciones graduales permiten actualizar la aplicación sin tiempo de inactividad. Kubernetes maneja el proceso de actualización reemplazando gradualmente los pods antiguos con nuevos.

Para actualizar el despliegue, modifica la versión de la imagen en el archivo YAML:

spec:
  template:
    spec:
      containers:
      - name: my-app-container
        image: my-app-image:2.0

Aplica los cambios:

kubectl apply -f my-app-deployment.yaml

Despliegue Blue-Green

El despliegue Blue-Green implica mantener dos entornos. Aquí hay un enfoque simplificado:

  1. Despliega la nueva versión (verde) junto a la versión antigua (azul).
  2. Cambia el servicio para apuntar a la nueva versión.

Ejemplo:

apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app-green
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

Cambiar el selector del servicio a app: my-app-green redirigirá el tráfico a la nueva versión.

Técnicas Avanzadas de Despliegue

Despliegue Canary

Los despliegues Canary desvían gradualmente el tráfico a la nueva versión. Esto puede lograrse usando múltiples despliegues y servicios.

  1. Crea un nuevo despliegue para la versión canary.
  2. Usa un service mesh o un controlador de ingreso para enrutar un pequeño porcentaje de tráfico al despliegue canary.

Ejemplo usando Istio:

apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: my-app
spec:
  hosts:
  - my-app.example.com
  http:
  - route:
    - destination:
        host: my-app
        subset: v1
      weight: 90
    - destination:
        host: my-app
        subset: v2
      weight: 10

Pipelines CI/CD

Integra los despliegues de Kubernetes en tu pipeline CI/CD usando herramientas como Jenkins, GitLab CI o GitHub Actions.

Ejemplo de workflow de GitHub Actions:

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout code
      uses: actions/checkout@v2
    - name: Build Docker image
      run: docker build -t my-app-image:${{ github.sha }} .
    - name: Push Docker image
      run: docker push my-app-image:${{ github.sha }}
    - name: Deploy to Kubernetes
      run: |
        kubectl set image deployment/my-app my-app-container=my-app-image:${{ github.sha }}

Conclusión

Automatizar los despliegues en Kubernetes mejora la fiabilidad y eficiencia del proceso de entrega de aplicaciones. Al comprender e implementar diversas estrategias de despliegue como actualizaciones graduales, despliegues blue-green y despliegues canary, puedes asegurar lanzamientos suaves y controlados. Integrar estas prácticas en pipelines CI/CD agiliza aún más el proceso, permitiendo el despliegue continuo y automatizado de nuevas características y actualizaciones.
