Configurar un clúster de Kubernetes es una habilidad fundamental para cualquiera que busque gestionar aplicaciones en contenedores a gran escala. Esta sección te guiará a través del proceso de configuración de un clúster de Kubernetes desde cero, cubriendo tanto conceptos teóricos como pasos prácticos.

Requisitos Previos

Antes de configurar un clúster de Kubernetes, asegúrate de tener lo siguiente:

  • Comprensión básica de la contenedorización (Docker).
  • Una máquina con al menos 2 CPUs y 2GB de RAM.
  • Un sistema operativo basado en Linux (se recomienda Ubuntu 20.04).
  • Conectividad a Internet para descargar los paquetes necesarios.

Conceptos Clave

  • Clúster de Kubernetes: Un conjunto de nodos (máquinas) que ejecutan aplicaciones en contenedores gestionadas por Kubernetes.
  • Nodo Maestro: El nodo que controla y gestiona el clúster.
  • Nodos Trabajadores: Nodos que ejecutan las aplicaciones en contenedores.
  • kubectl: La herramienta de línea de comandos para interactuar con el clúster de Kubernetes.

Guía Paso a Paso

1. Instalar Docker

Kubernetes utiliza Docker para gestionar contenedores. Instala Docker en todos los nodos (tanto en el nodo maestro como en los nodos trabajadores).

sudo apt-get update
sudo apt-get install -y docker.io
sudo systemctl start docker
sudo systemctl enable docker

2. Instalar kubeadm, kubelet y kubectl

Estos son los componentes esenciales para configurar y gestionar un clúster de Kubernetes.

sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl
sudo curl -fsSL https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
sudo bash -c 'cat <<EOF >/etc/apt/sources.list.d/kubernetes.list
deb https://apt.kubernetes.io/ kubernetes-xenial main
EOF'
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

3. Inicializar el Nodo Maestro

Inicializa el nodo maestro usando kubeadm.

sudo kubeadm init --pod-network-cidr=192.168.0.0/16

Después de la inicialización, verás un comando para unir los nodos trabajadores al clúster. Guarda este comando para usarlo más tarde.

4. Configurar kubectl para el Nodo Maestro

Configura kubectl para el nodo maestro.

mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

5. Instalar un Complemento de Red para Pods

Una red de pods permite la comunicación entre nodos. Instala un complemento de red como Calico.

kubectl apply -f https://docs.projectcalico.org/manifests/calico.yaml

6. Unir Nodos Trabajadores al Clúster

Ejecuta el comando de unión en cada nodo trabajador. Este comando fue proporcionado al final del proceso kubeadm init.

sudo kubeadm join <ip-nodo-maestro>:6443 --token <token> --discovery-token-ca-cert-hash sha256:<hash>

7. Verificar el Clúster

Verifica el estado de los nodos para asegurarte de que todos son parte del clúster.

kubectl get nodes

Solución de Problemas

  • Problema: Los nodos trabajadores no se unen al clúster.

    • Solución: Asegúrate de que el token y el hash sean correctos y que los nodos trabajadores puedan alcanzar el nodo maestro.

  • Problema: Los pods no se inician.

    • Solución: Verifica la instalación del complemento de red para pods y asegúrate de que todos los nodos estén en estado Ready.

Conclusión

Configurar un clúster de Kubernetes implica varios pasos, desde instalar los componentes necesarios hasta configurar la red y unir los nodos trabajadores. Siguiendo esta guía, deberías tener un clúster de Kubernetes funcional listo para desplegar y gestionar aplicaciones en contenedores. Recuerda verificar regularmente el estado de tus nodos y solucionar cualquier problema que surja para mantener un clúster saludable.
