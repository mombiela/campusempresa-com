Introducción

En Kubernetes, gestionar datos de configuración e información sensible es crucial para el despliegue y operación de aplicaciones. ConfigMaps y Secrets son dos recursos fundamentales que ayudan en este sentido. Esta sección cubrirá los conceptos básicos y el uso avanzado de ConfigMaps y Secrets, proporcionando explicaciones claras, ejemplos y ejercicios.

ConfigMaps

¿Qué es un ConfigMap?

  • Un ConfigMap es un objeto API utilizado para almacenar datos no confidenciales en pares clave-valor.
  • Los ConfigMaps te permiten desacoplar los artefactos de configuración del contenido de la imagen para mantener las aplicaciones en contenedores portátiles.

Creando un ConfigMap

Puedes crear un ConfigMap usando un archivo YAML o directamente desde la línea de comandos.

Usando un Archivo YAML

apiVersion: v1
kind: ConfigMap
metadata:
  name: example-config
data:
  database_url: "mongodb://localhost:27017"
  log_level: "DEBUG"

Para crear el ConfigMap:

kubectl apply -f configmap.yaml

Usando la Línea de Comandos

kubectl create configmap example-config --from-literal=database_url="mongodb://localhost:27017" --from-literal=log_level="DEBUG"

Usando ConfigMaps en Pods

Puedes usar ConfigMaps en Pods a través de variables de entorno o montajes de volumen.

Variables de Entorno

apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    env:
    - name: DATABASE_URL
      valueFrom:
        configMapKeyRef:
          name: example-config
          key: database_url

Montajes de Volumen

apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    volumeMounts:
    - name: config-volume
      mountPath: /etc/config
  volumes:
  - name: config-volume
    configMap:
      name: example-config

Mejores Prácticas para ConfigMaps

  • Usa ConfigMaps para datos de configuración no sensibles.
  • Mantén los ConfigMaps pequeños y manejables.
  • Versiona tus ConfigMaps para rastrear cambios.

Secrets

¿Qué es un Secret?

  • Un Secret es un objeto API que almacena datos sensibles como contraseñas, tokens OAuth y claves SSH.
  • Los Secrets están codificados en base64 para evitar la exposición accidental, pero no están encriptados por defecto.

Creando un Secret

Puedes crear un Secret usando un archivo YAML o directamente desde la línea de comandos.

Usando un Archivo YAML

apiVersion: v1
kind: Secret
metadata:
  name: example-secret
type: Opaque
data:
  username: YWRtaW4=  # codificado en base64 'admin'
  password: MWYyZDFlMmU2N2Rm  # codificado en base64 '1f2d1e2e67df'

Para crear el Secret:

kubectl apply -f secret.yaml

Usando la Línea de Comandos

kubectl create secret generic example-secret --from-literal=username=admin --from-literal=password=1f2d1e2e67df

Usando Secrets en Pods

Puedes usar Secrets en Pods a través de variables de entorno o montajes de volumen.

Variables de Entorno

apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    env:
    - name: USERNAME
      valueFrom:
        secretKeyRef:
          name: example-secret
          key: username

Montajes de Volumen

apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx
    volumeMounts:
    - name: secret-volume
      mountPath: /etc/secret
  volumes:
  - name: secret-volume
    secret:
      secretName: example-secret

Mejores Prácticas para Secrets

  • Usa Secrets para datos sensibles.
  • Habilita la encriptación en reposo para los Secrets.
  • Limita el acceso a los Secrets usando RBAC (Control de Acceso Basado en Roles).

Conclusión

ConfigMaps y Secrets son herramientas esenciales en Kubernetes para gestionar datos de configuración e información sensible. Los ConfigMaps son adecuados para datos no confidenciales, mientras que los Secrets deben usarse para información sensible. El uso adecuado y las mejores prácticas aseguran que tus aplicaciones sean tanto flexibles como seguras. Al entender e implementar estos recursos, puedes mejorar la portabilidad y seguridad de tus despliegues en Kubernetes.
