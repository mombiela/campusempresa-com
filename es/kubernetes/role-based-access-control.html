Introducción a RBAC

El Control de Acceso Basado en Roles (RBAC) es un método para regular el acceso a recursos informáticos o de red basado en los roles de los usuarios individuales dentro de una empresa. En Kubernetes, RBAC se utiliza para controlar quién puede acceder a la API de Kubernetes y qué acciones pueden realizar.

Conceptos Clave de RBAC

  • Rol: Define un conjunto de permisos. Los roles están en un espacio de nombres, lo que significa que se aplican a recursos dentro de un espacio de nombres específico.
  • ClusterRole: Similar a un Rol, pero los ClusterRoles no están en un espacio de nombres y se pueden usar para otorgar acceso a recursos en todo el clúster.
  • RoleBinding: Otorga los permisos definidos en un Rol a un usuario o conjunto de usuarios dentro de un espacio de nombres.
  • ClusterRoleBinding: Otorga los permisos definidos en un ClusterRole a un usuario o conjunto de usuarios en todo el clúster.

Creación de Roles y RoleBindings

Creación de un Rol

Un Rol en Kubernetes es una colección de permisos. Aquí hay un ejemplo de un Rol que permite leer pods en un espacio de nombres:

apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
- apiGroups: [""] # "" indica el grupo de API principal
  resources: ["pods"]
  verbs: ["get", "watch", "list"]

Creación de un RoleBinding

Un RoleBinding otorga los permisos definidos en un Rol a un usuario o conjunto de usuarios. Aquí hay un ejemplo de un RoleBinding que otorga el Rol pod-reader a un usuario llamado jane:

apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: read-pods
  namespace: default
subjects:
- kind: User
  name: jane
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io

Creación de ClusterRoles y ClusterRoleBindings

Creación de un ClusterRole

Un ClusterRole se puede usar para definir permisos que no son específicos de un espacio de nombres. Aquí hay un ejemplo de un ClusterRole que permite leer nodos:

apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: node-reader
rules:
- apiGroups: [""]
  resources: ["nodes"]
  verbs: ["get", "watch", "list"]

Creación de un ClusterRoleBinding

Un ClusterRoleBinding otorga los permisos definidos en un ClusterRole a un usuario o conjunto de usuarios en todo el clúster. Aquí hay un ejemplo de un ClusterRoleBinding que otorga el ClusterRole node-reader a un usuario llamado jane:

apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: read-nodes
subjects:
- kind: User
  name: jane
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: ClusterRole
  name: node-reader
  apiGroup: rbac.authorization.k8s.io

Comparación de Roles y ClusterRoles

| Característica | Rol | ClusterRole | |-----------------|-------------------------------|--------------------------------| | Alcance | Específico del espacio de nombres | En todo el clúster | | Caso de Uso | Otorgar permisos dentro de un espacio de nombres | Otorgar permisos en todo el clúster | | Recurso Ejemplo | Pods, Servicios, ConfigMaps | Nodos, PersistentVolumes, Recursos de todo el clúster |

Mejores Prácticas para Usar RBAC

  • Principio de Mínimos Privilegios: Siempre otorga los permisos mínimos necesarios para que un usuario realice sus tareas.
  • Aislamiento de Espacios de Nombres: Usa Roles y RoleBindings para limitar el acceso a espacios de nombres específicos.
  • Auditoría y Revisión: Audita y revisa regularmente las políticas de RBAC para asegurarte de que estén actualizadas y sigan las mejores prácticas de seguridad.

Conclusión

RBAC es una característica poderosa en Kubernetes que ayuda a gestionar el acceso a recursos de manera segura y organizada. Al comprender e implementar adecuadamente Roles, RoleBindings, ClusterRoles y ClusterRoleBindings, puedes asegurar que tu clúster de Kubernetes sea tanto seguro como eficiente. Siempre sigue las mejores prácticas para mantener un entorno seguro y manejable.
