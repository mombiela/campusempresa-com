Introducción

En MUMPS (M), entender la diferencia entre variables locales y globales es crucial para una programación efectiva. Esta sección cubrirá las definiciones, uso y diferencias entre variables locales y globales, proporcionando ejemplos y ejercicios para solidificar tu comprensión.

Variables Locales

Las variables locales en MUMPS son temporales y existen solo dentro del contexto del proceso o rutina actual. No son accesibles fuera de la rutina o proceso en el que se definen.

Conceptos Clave

  • Ámbito: Las variables locales están limitadas a la rutina o proceso.
  • Duración: Existen solo durante la duración de la rutina o proceso.
  • Sintaxis: Las variables locales típicamente se nombran sin ningún prefijo especial.

Ejemplo

MiRutina
    NEW X,Y
    SET X=10
    SET Y=20
    WRITE "X: ", X, " Y: ", Y, !
    QUIT

En este ejemplo:

  • X y Y son variables locales.
  • Se definen y usan dentro de la rutina MiRutina.
  • Dejan de existir una vez que la rutina termina su ejecución.

Variables Globales

Las variables globales en MUMPS son persistentes y pueden ser accedidas por cualquier rutina o proceso. Se almacenan en la base de datos y permanecen disponibles incluso después de que el proceso que las creó haya terminado.

Conceptos Clave

  • Ámbito: Las variables globales son accesibles a través de diferentes rutinas y procesos.
  • Duración: Persisten hasta que se eliminan explícitamente.
  • Sintaxis: Las variables globales se prefijan con un acento circunflejo (^).

Ejemplo

MiRutinaGlobal
    SET ^GlobalX=100
    SET ^GlobalY=200
    WRITE "^GlobalX: ", ^GlobalX, " ^GlobalY: ", ^GlobalY, !
    QUIT

En este ejemplo:

  • ^GlobalX y ^GlobalY son variables globales.
  • Se almacenan en la base de datos y pueden ser accedidas por cualquier rutina.

Comparación de Variables Locales y Globales

| Característica | Variables Locales | Variables Globales | |----------------|----------------------------|----------------------------| | Ámbito | Limitado a rutina/proceso | Accesible entre rutinas | | Duración | Duración de rutina/proceso | Persistente hasta eliminar | | Sintaxis | Sin prefijo especial | Prefijado con ^ | | Almacenamiento | En memoria | En base de datos |

Ejemplo de Código: Usando Tanto Variables Locales como Globales

MiRutinaMixta
    NEW LocalA, LocalB
    SET LocalA=5
    SET LocalB=10
    SET ^GlobalA=15
    SET ^GlobalB=20
    WRITE "LocalA: ", LocalA, " LocalB: ", LocalB, !
    WRITE "^GlobalA: ", ^GlobalA, " ^GlobalB: ", ^GlobalB, !
    QUIT

En este ejemplo:

  • LocalA y LocalB son variables locales.
  • ^GlobalA y ^GlobalB son variables globales.
  • La rutina demuestra el uso simultáneo de variables locales y globales.

Ejercicios

  1. Ejercicio 1: Crea una rutina que defina y use variables locales para realizar un cálculo simple (por ejemplo, suma o multiplicación).
  2. Ejercicio 2: Crea una rutina que defina y use variables globales para almacenar y recuperar datos.
  3. Ejercicio 3: Modifica la rutina del Ejercicio 1 para que también use variables globales, y compara los resultados.

Conclusión

Entender las diferencias entre variables locales y globales en MUMPS es esencial para una programación efectiva. Las variables locales son temporales y limitadas en ámbito, mientras que las variables globales son persistentes y accesibles entre rutinas. Al dominar estos conceptos, puedes escribir código MUMPS más eficiente y organizado.
