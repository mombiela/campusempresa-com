Introducción

En MUMPS (M), las funciones y los procedimientos son bloques de construcción fundamentales que te permiten encapsular y reutilizar código. Esta sección te guiará desde los conceptos básicos hasta los avanzados, asegurándote de entender cómo crear, llamar y gestionar funciones y procedimientos de manera efectiva.

Conceptos Clave

  • Funciones: Devuelven un valor y pueden ser usadas en expresiones.
  • Procedimientos: Realizan acciones pero no devuelven un valor.
  • Argumentos: Valores pasados a funciones o procedimientos.
  • Variables Locales: Variables definidas dentro de una función o procedimiento.
  • Variables Globales: Variables accesibles a lo largo del programa.

Creación de Funciones

Las funciones en MUMPS se definen usando la sintaxis FunctionName^RoutineName. Devuelven un valor usando el comando QUIT.

Ejemplo

Sum(a, b)
    QUIT a + b

En este ejemplo, Sum es una función que toma dos argumentos a y b, y devuelve su suma.

Llamada a Funciones

Las funciones se llaman usando su nombre seguido de los argumentos entre paréntesis.

Ejemplo

SET result = $$Sum(3, 5)
WRITE result  ; Salida: 8

Aquí, $$Sum(3, 5) llama a la función Sum con los argumentos 3 y 5, y almacena el resultado en result.

Creación de Procedimientos

Los procedimientos son similares a las funciones pero no devuelven un valor. Se definen usando la sintaxis ProcedureName^RoutineName.

Ejemplo

PrintSum(a, b)
    NEW sum
    SET sum = a + b
    WRITE "La suma es: ", sum, !
    QUIT

En este ejemplo, PrintSum es un procedimiento que calcula la suma de a y b e imprime el resultado.

Llamada a Procedimientos

Los procedimientos se llaman usando el comando DO seguido del nombre del procedimiento y los argumentos.

Ejemplo

DO PrintSum(3, 5)
; Salida: La suma es: 8

Aquí, DO PrintSum(3, 5) llama al procedimiento PrintSum con los argumentos 3 y 5.

Variables Locales y Globales

  • Variables Locales: Definidas dentro de una función o procedimiento usando el comando NEW.
  • Variables Globales: Accesibles a lo largo del programa y no requieren el comando NEW.

Ejemplo

SumWithGlobal(a, b)
    NEW sum
    SET sum = a + b
    SET ^GlobalSum = sum
    QUIT sum

En este ejemplo, sum es una variable local, mientras que ^GlobalSum es una variable global.

Conceptos Avanzados

Recursión

Las funciones y procedimientos en MUMPS pueden llamarse a sí mismos, permitiendo algoritmos recursivos.

Ejemplo

Factorial(n)
    IF n=0 QUIT 1
    QUIT n * $$Factorial(n-1)

Este ejemplo define una función recursiva Factorial que calcula el factorial de n.

Manejo de Errores

Usa los comandos TRY y CATCH para manejar errores dentro de funciones y procedimientos.

Ejemplo

SafeDivide(a, b)
    NEW result
    TRY {
        IF b=0 THROW "DivideByZero"
        SET result = a / b
    } CATCH {
        WRITE "Error: División por cero", !
        SET result = "indefinido"
    }
    QUIT result

En este ejemplo, SafeDivide maneja los errores de división por cero de manera elegante.

Resumen

  • Las funciones y procedimientos son esenciales para la reutilización y organización del código en MUMPS.
  • Las funciones devuelven valores, mientras que los procedimientos realizan acciones sin devolver valores.
  • Usa variables locales para datos temporales dentro de funciones y procedimientos, y variables globales para datos que necesitan ser accesibles a lo largo del programa.
  • Conceptos avanzados como la recursión y el manejo de errores pueden mejorar la robustez y funcionalidad de tu código.

Al dominar estos conceptos, estarás bien equipado para escribir código MUMPS eficiente y mantenible.
