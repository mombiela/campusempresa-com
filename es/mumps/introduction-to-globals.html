¿Qué son los Globales?

Los globales en MUMPS (M) son un concepto fundamental que permite el almacenamiento y la recuperación de datos persistentes. A diferencia de las variables locales, los globales se almacenan en disco y pueden ser accedidos por diferentes procesos, lo que los hace esenciales para las operaciones de bases de datos.

  • Almacenamiento Persistente: Los globales se almacenan en disco, asegurando la persistencia de los datos entre sesiones.
  • Acceso Multi-Proceso: Los globales pueden ser accedidos y modificados por múltiples procesos, permitiendo operaciones concurrentes de datos.
  • Estructura Jerárquica: Los globales están organizados en una estructura en forma de árbol, permitiendo relaciones de datos complejas.

Sintaxis y Estructura de los Globales

Sintaxis Básica

Los globales se identifican por un acento circunflejo (^) seguido de un nombre. Pueden tener subíndices para representar datos jerárquicos.

^NombreGlobal(subíndice1, subíndice2, ...)

Ejemplo

; Estableciendo un valor global
SET ^Paciente(12345, "Nombre") = "Juan Pérez"

; Recuperando un valor global
WRITE ^Paciente(12345, "Nombre")

En este ejemplo:

  • ^Paciente es el nombre del global.
  • 12345 es el primer subíndice, representando un ID de paciente.
  • "Nombre" es el segundo subíndice, representando el nombre del paciente.
  • El valor "Juan Pérez" se almacena en esta ubicación.

Conceptos Clave

  • Nodos Globales: Cada combinación de subíndices en un global representa un nodo.
  • Arreglos Globales: Los globales pueden considerarse como arreglos multidimensionales.
  • Recorrido de Globales: Puedes recorrer los globales usando comandos especiales como ORDER y NEXT.

Ejemplo de Recorrido

; Estableciendo múltiples valores globales
SET ^Paciente(12345, "Nombre") = "Juan Pérez"
SET ^Paciente(12345, "Edad") = 30
SET ^Paciente(67890, "Nombre") = "Ana Gómez"
SET ^Paciente(67890, "Edad") = 25

; Recorriendo nodos globales
SET idPaciente = ""
FOR  SET idPaciente = $ORDER(^Paciente(idPaciente)) QUIT:idPaciente=""  DO
. WRITE "ID del Paciente: ", idPaciente, !
. WRITE "Nombre: ", ^Paciente(idPaciente, "Nombre"), !
. WRITE "Edad: ", ^Paciente(idPaciente, "Edad"), !

Uso Avanzado

Bloqueo de Globales

Para asegurar la integridad de los datos cuando múltiples procesos acceden a los globales, MUMPS proporciona mecanismos de bloqueo.

; Bloqueando un nodo global
LOCK ^Paciente(12345)

; Realizar operaciones
SET ^Paciente(12345, "Nombre") = "Juan Pérez Actualizado"

; Desbloqueando el nodo global
UNLOCK ^Paciente(12345)

Arreglos Globales e Indirección

Los globales pueden ser manipulados usando la indirección, permitiendo el acceso dinámico a los nodos globales.

; Usando indirección para establecer un valor global
SET nombreGlobal = "^Paciente"
SET @nombreGlobal@(12345, "Dirección") = "Calle Principal 123"

; Usando indirección para recuperar un valor global
WRITE @nombreGlobal@(12345, "Dirección")

Resumen

Los globales son una característica poderosa en MUMPS que proporcionan almacenamiento persistente, jerárquico y accesible por múltiples procesos. Entender cómo usar efectivamente los globales, incluyendo establecer, recuperar, recorrer y bloquearlos, es crucial para desarrollar aplicaciones robustas en MUMPS.

  • Almacenamiento Persistente: Los globales aseguran que los datos se guarden entre sesiones.
  • Estructura Jerárquica: Permite relaciones de datos complejas.
  • Concurrencia: Los globales pueden ser accedidos por múltiples procesos, con mecanismos de bloqueo para asegurar la integridad de los datos.

Al dominar los globales, podrás manejar tareas complejas de almacenamiento y recuperación de datos de manera eficiente en MUMPS.
