Configurar el entorno MUMPS (Massachusetts General Hospital Utility Multi-Programming System) es el primer paso para comenzar a programar en MUMPS. Esta sección te guiará a través del proceso de instalación y configuración del entorno MUMPS en tu sistema.

Instalación de MUMPS

Requisitos Previos

Antes de instalar MUMPS, asegúrate de que tu sistema cumpla con los siguientes requisitos:

  • Un sistema operativo compatible (Linux, Windows, macOS).
  • Conocimientos básicos de la interfaz de línea de comandos (CLI).

Pasos de Instalación

En Linux

  1. Actualiza tu lista de paquetes: 
    sudo apt-get update
  2. Instala GT.M (una implementación popular de MUMPS): 
    sudo apt-get install -y fis-gtm
  3. Verifica la instalación: 
    gtm
    Deberías ver el prompt de GT.M (GTM>).

En Windows

  1. Descarga el instalador de GT.M desde el sitio web oficial.
  2. Ejecuta el instalador y sigue las instrucciones en pantalla.
  3. Configura las variables de entorno:
    • Añade el directorio binario de GT.M a tu PATH.
    • Configura la variable de entorno gtm_dist al directorio de instalación de GT.M.
  4. Verifica la instalación abriendo un símbolo del sistema y escribiendo: 
    gtm
    Deberías ver el prompt de GT.M (GTM>).

En macOS

  1. Instala Homebrew si aún no lo has hecho: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  2. Instala GT.M usando Homebrew: 
    brew install fis-gtm
  3. Verifica la instalación: 
    gtm
    Deberías ver el prompt de GT.M (GTM>).

Configuración del Entorno MUMPS

Configuración de Directorios

Crea una estructura de directorios para tus proyectos MUMPS. Por ejemplo:

mkdir -p ~/mumps/projects
mkdir -p ~/mumps/data

Variables de Entorno

Configura las variables de entorno necesarias para asegurar que GT.M pueda encontrar tus proyectos y archivos de datos.

En Linux/macOS

Añade las siguientes líneas a tu archivo .bashrc o .zshrc:

export gtm_dist=/usr/lib/fis-gtm/V6.3-003
export gtmroutines="~/mumps/projects"
export gtmgbldir="~/mumps/data/global.gld"

Luego, aplica los cambios:

source ~/.bashrc

En Windows

Configura las variables de entorno a través de las Propiedades del Sistema:

  1. Abre Propiedades del Sistema.
  2. Ve a la pestaña Avanzado y haz clic en Variables de entorno.
  3. Añade nuevas variables:
    • gtm_dist con la ruta al directorio de instalación de GT.M.
    • gtmroutines con la ruta a tu directorio de proyectos.
    • gtmgbldir con la ruta a tu directorio global.

Ejecutando Tu Primer Programa MUMPS

Creación de un Programa Simple

  1. Crea un nuevo archivo llamado hello.m en tu directorio de proyectos: 
    nano ~/mumps/projects/hello.m
  2. Escribe un programa MUMPS simple: 
    hello
write "¡Hola, MUMPS!",!
quit
  3. Guarda y cierra el editor.

Ejecución del Programa

  1. Inicia el entorno GT.M: 
    gtm
  2. Ejecuta tu programa: 
    GTM>do ^hello
¡Hola, MUMPS!
GTM>

Conclusión

Configurar el entorno MUMPS implica instalar la implementación GT.M, configurar los directorios y variables de entorno necesarios, y ejecutar tu primer programa MUMPS. Con el entorno configurado, ahora estás listo para profundizar en la programación MUMPS. Esta configuración fundamental asegura que tengas un entorno estable y consistente para desarrollar y probar tus aplicaciones MUMPS.
