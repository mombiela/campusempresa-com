Introducción a la Infraestructura Global de AWS

Amazon Web Services (AWS) proporciona una plataforma de computación en la nube robusta y escalable. Comprender la infraestructura global de AWS es crucial para aprovechar todo su potencial. Esta sección cubrirá los componentes y conceptos clave de la infraestructura global de AWS.

Componentes Clave de la Infraestructura Global de AWS

Regiones

  • Definición: Una Región es una ubicación física alrededor del mundo donde AWS agrupa centros de datos.
  • Propósito: Las regiones te permiten desplegar aplicaciones y servicios más cerca de tus usuarios finales para reducir la latencia.
  • Ejemplo: us-east-1 (Virginia del Norte), eu-west-1 (Irlanda).

Zonas de Disponibilidad (AZs)

  • Definición: Las Zonas de Disponibilidad son ubicaciones aisladas dentro de una Región.
  • Propósito: Las AZs proporcionan redundancia y tolerancia a fallos al aislar fallos a una sola zona.
  • Ejemplo: us-east-1a, us-east-1b.

Ubicaciones de Borde

  • Definición: Las Ubicaciones de Borde son puntos finales para los servicios de AWS utilizados para almacenar en caché contenido.
  • Propósito: Mejoran la latencia al almacenar en caché contenido más cerca de los usuarios.
  • Ejemplo: Utilizado por Amazon CloudFront para entregar contenido.

Regiones y Zonas de Disponibilidad

Elegir una Región

  • Factores a Considerar:
    • Latencia: Elige una región cercana a tus usuarios.
    • Cumplimiento: Algunos datos deben residir en ubicaciones geográficas específicas.
    • Costo: Los precios pueden variar entre regiones.

Alta Disponibilidad y Tolerancia a Fallos

  • Despliegues Multi-AZ: Distribuye aplicaciones a través de múltiples AZs para asegurar alta disponibilidad.
  • Recuperación ante Desastres: Utiliza múltiples regiones para estrategias de recuperación ante desastres.

Ejemplo: Desplegar una Instancia EC2 en una Región y AZ Específica

aws ec2 run-instances --image-id ami-0abcdef1234567890 --count 1 --instance-type t2.micro --key-name MyKeyPair --region us-west-2 --placement AvailabilityZone=us-west-2b
  • Explicación: Este comando lanza una instancia EC2 en la región us-west-2 y la AZ us-west-2b.

Ubicaciones de Borde y Entrega de Contenido

Amazon CloudFront

  • Definición: Un servicio de red de entrega de contenido (CDN) que utiliza ubicaciones de borde.
  • Propósito: Distribuye contenido a los usuarios con baja latencia.
  • Ejemplo: Servir contenido web estático y dinámico.

Ejemplo: Crear una Distribución de CloudFront

{
  "CallerReference": "unique-string",
  "Aliases": {
    "Quantity": 1,
    "Items": ["example.com"]
  },
  "DefaultRootObject": "index.html",
  "Origins": {
    "Quantity": 1,
    "Items": [
      {
        "Id": "S3-example-bucket",
        "DomainName": "example-bucket.s3.amazonaws.com",
        "S3OriginConfig": {
          "OriginAccessIdentity": ""
        }
      }
    ]
  },
  "DefaultCacheBehavior": {
    "TargetOriginId": "S3-example-bucket",
    "ViewerProtocolPolicy": "redirect-to-https",
    "AllowedMethods": {
      "Quantity": 2,
      "Items": ["GET", "HEAD"]
    },
    "ForwardedValues": {
      "QueryString": false,
      "Cookies": {
        "Forward": "none"
      }
    },
    "MinTTL": 0
  },
  "Comment": "Example CloudFront Distribution",
  "Enabled": true
}
  • Explicación: Esta configuración JSON crea una distribución de CloudFront que sirve contenido desde un bucket S3.

Resumen

Comprender la infraestructura global de AWS es fundamental para diseñar aplicaciones escalables, confiables y eficientes. Los componentes clave incluyen Regiones, Zonas de Disponibilidad y Ubicaciones de Borde, cada uno sirviendo propósitos específicos como reducir la latencia, asegurar alta disponibilidad y entregar contenido eficientemente. Al elegir estratégicamente las regiones y aprovechar los despliegues multi-AZ, puedes optimizar el rendimiento y la confiabilidad. Además, usar servicios como Amazon CloudFront puede mejorar significativamente la entrega de contenido a los usuarios finales.
