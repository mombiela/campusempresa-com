Introducción a la Red de AWS
Amazon Route 53 ►

Resumen

Amazon Web Services (AWS) proporciona una plataforma de computación en la nube robusta y escalable que incluye una variedad de servicios de red. Esta sección te introducirá a los conceptos fundamentales de la red de AWS, incluyendo Virtual Private Cloud (VPC), subredes, tablas de rutas y grupos de seguridad.

Conceptos Clave

Virtual Private Cloud (VPC)

  • Definición: Una VPC es una red virtual dedicada a tu cuenta de AWS. Está lógicamente aislada de otras redes virtuales en la nube de AWS.
  • Propósito: Te permite lanzar recursos de AWS en una red virtual que has definido.
  • Componentes:
    • Bloque CIDR: Define el rango de direcciones IP para la VPC.
    • Subredes: Segmentos del rango de direcciones IP de la VPC donde puedes colocar grupos de recursos aislados.

Subredes

  • Definición: Una subred es un rango de direcciones IP en tu VPC.
  • Tipos:
    • Subred Pública: Accesible desde internet.
    • Subred Privada: No accesible desde internet.
  • Propósito: Organizar y asegurar tus recursos dentro de la VPC.

Tablas de Rutas

  • Definición: Un conjunto de reglas, llamadas rutas, que se utilizan para determinar hacia dónde se dirige el tráfico de red.
  • Componentes:
    • Destino: El rango de direcciones IP hacia donde se dirige el tráfico.
    • Objetivo: La puerta de enlace, interfaz de red o conexión a través de la cual se envía el tráfico.

Puerta de Enlace a Internet

  • Definición: Un componente de VPC escalado horizontalmente, redundante y altamente disponible que permite la comunicación entre instancias en tu VPC y el internet.
  • Propósito: Permite que las instancias en una VPC se conecten a internet.

Puerta de Enlace NAT

  • Definición: Un servicio de traducción de direcciones de red (NAT) para tus instancias en una subred privada para acceder a internet.
  • Propósito: Permite el tráfico de internet saliente desde instancias en una subred privada mientras se previene el tráfico entrante.

Grupos de Seguridad

  • Definición: Cortafuegos virtuales que controlan el tráfico entrante y saliente hacia los recursos de AWS.
  • Componentes:
    • Reglas de Entrada: Definen el tráfico entrante permitido.
    • Reglas de Salida: Definen el tráfico saliente permitido.

Ejemplo: Creación de una VPC con Subredes Públicas y Privadas

Guía Paso a Paso

  1. Crear una VPC:

    import boto3

ec2 = boto3.client('ec2')

response = ec2.create_vpc(
    CidrBlock='10.0.0.0/16'
)
vpc_id = response['Vpc']['VpcId']
print(f'Created VPC with ID: {vpc_id}')

  2. Crear Subredes:

    # Subred Pública
public_subnet = ec2.create_subnet(
    CidrBlock='10.0.1.0/24',
    VpcId=vpc_id
)
public_subnet_id = public_subnet['Subnet']['SubnetId']
print(f'Created Public Subnet with ID: {public_subnet_id}')

# Subred Privada
private_subnet = ec2.create_subnet(
    CidrBlock='10.0.2.0/24',
    VpcId=vpc_id
)
private_subnet_id = private_subnet['Subnet']['SubnetId']
print(f'Created Private Subnet with ID: {private_subnet_id}')

  3. Crear una Puerta de Enlace a Internet y Adjuntar a la VPC:

    igw = ec2.create_internet_gateway()
igw_id = igw['InternetGateway']['InternetGatewayId']
ec2.attach_internet_gateway(
    InternetGatewayId=igw_id,
    VpcId=vpc_id
)
print(f'Created and attached Internet Gateway with ID: {igw_id}')

  4. Crear Tabla de Rutas y Añadir Rutas:

    route_table = ec2.create_route_table(VpcId=vpc_id)
route_table_id = route_table['RouteTable']['RouteTableId']
ec2.create_route(
    RouteTableId=route_table_id,
    DestinationCidrBlock='0.0.0.0/0',
    GatewayId=igw_id
)
print(f'Created Route Table with ID: {route_table_id} and added route to Internet Gateway')

  5. Asociar Tabla de Rutas con Subred Pública:

    ec2.associate_route_table(
    RouteTableId=route_table_id,
    SubnetId=public_subnet_id
)
print(f'Associated Route Table with Public Subnet')

  6. Crear Grupos de Seguridad:

    security_group = ec2.create_security_group(
    GroupName='my-security-group',
    Description='My security group',
    VpcId=vpc_id
)
security_group_id = security_group['GroupId']
print(f'Created Security Group with ID: {security_group_id}')

# Añadir Regla de Entrada para permitir SSH
ec2.authorize_security_group_ingress(
    GroupId=security_group_id,
    IpPermissions=[
        {
            'IpProtocol': 'tcp',
            'FromPort': 22,
            'ToPort': 22,
            'IpRanges': [{'CidrIp': '0.0.0.0/0'}]
        }
    ]
)
print('Added inbound rule to allow SSH')

Resumen

En esta sección, cubrimos los conceptos básicos de la red de AWS, incluyendo VPCs, subredes, tablas de rutas, puertas de enlace a internet, puertas de enlace NAT y grupos de seguridad. También recorrimos un ejemplo práctico de creación de una VPC con subredes públicas y privadas, adjuntando una puerta de enlace a internet y configurando grupos de seguridad. Entender estos conceptos fundamentales es crucial para diseñar y gestionar entornos de AWS seguros y eficientes.
