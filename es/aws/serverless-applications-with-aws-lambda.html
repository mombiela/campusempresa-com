Introducción a la Computación sin Servidor

  • Definición de la computación sin servidor.
  • Beneficios de la arquitectura sin servidor.
  • Descripción general de AWS Lambda y su papel en la computación sin servidor.

Comenzando con AWS Lambda

  • Configuración de una cuenta de AWS.
  • Introducción a la Consola de Administración de AWS.
  • Creación de tu primera función Lambda. 
    import json

def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': json.dumps('¡Hola desde Lambda!')
    }
    • Explicación:
      • lambda_handler: La función principal que AWS Lambda llama para iniciar la ejecución.
      • event: Contiene los datos del evento.
      • context: Proporciona información de tiempo de ejecución al controlador.

Conceptos de AWS Lambda

  • Funciones: La unidad básica de implementación en AWS Lambda.
  • Fuentes de Eventos: Servicios que pueden activar funciones Lambda (por ejemplo, S3, DynamoDB, API Gateway).
  • Rol de Ejecución: Rol de IAM que otorga a la función permisos para acceder a los servicios de AWS.
  • Variables de Entorno: Pares clave-valor que se pueden usar para pasar configuraciones a tu función.

Creación e Implementación de Funciones Lambda

  • Usando la Consola de Administración de AWS.
  • Usando la CLI de AWS. 
    aws lambda create-function --function-name my-function 
        --runtime python3.8 --role arn:aws:iam::account-id:role/execution_role 
        --handler lambda_function.lambda_handler --zip-file fileb://function.zip
    • Explicación:
      • --function-name: El nombre de la función Lambda.
      • --runtime: El entorno de ejecución para la función Lambda.
      • --role: El ARN del rol de IAM que Lambda asume cuando ejecuta la función.
      • --handler: La función dentro de tu código que Lambda llama para iniciar la ejecución.
      • --zip-file: La ruta al paquete de implementación.

Fuentes de Eventos y Disparadores

  • Amazon S3: Activación de funciones Lambda en la creación de objetos. 
    import json

def lambda_handler(event, context):
    # Registrar el evento
    print(json.dumps(event))
    return {
        'statusCode': 200,
        'body': json.dumps('Evento procesado')
    }
  • Amazon DynamoDB: Activación de funciones Lambda en actualizaciones de tablas.
  • Amazon API Gateway: Creación de APIs RESTful con Lambda.

Funciones Avanzadas de Lambda

  • Concurrencia y Escalado: Entender cómo Lambda escala con las solicitudes entrantes.
  • Manejo de Errores y Reintentos: Configuración de reintentos y manejo de errores.
  • Monitoreo y Registro: Uso de CloudWatch para registros y métricas. 
    import logging
logger = logging.getLogger()
logger.setLevel(logging.INFO)

def lambda_handler(event, context):
    logger.info('Evento: %s', event)
    return {
        'statusCode': 200,
        'body': json.dumps('Evento registrado')
    }

Integración con Otros Servicios de AWS

  • AWS Step Functions: Orquestación de múltiples funciones Lambda.
  • Amazon SNS y SQS: Uso de Lambda con servicios de mensajería.
  • AWS RDS y DynamoDB: Acceso a bases de datos desde funciones Lambda.

Mejores Prácticas de Seguridad

  • Roles y Políticas de IAM: Otorgar acceso de menor privilegio.
  • Variables de Entorno: Almacenamiento seguro de información sensible.
  • Integración con VPC: Ejecución de funciones Lambda dentro de una VPC.

Conclusión

  • Resumen de conceptos clave: computación sin servidor, funciones Lambda, fuentes de eventos e integraciones.
  • Motivación para explorar temas más avanzados y servicios de AWS.
