Introducción a Amazon EC2

Amazon Elastic Compute Cloud (EC2) es un servicio web que proporciona capacidad de cómputo redimensionable en la nube. Está diseñado para facilitar la computación en la nube a escala web para los desarrolladores.

Conceptos Clave

  • Instancias: Servidores virtuales para ejecutar aplicaciones en la infraestructura de AWS.
  • AMI (Amazon Machine Image): Una plantilla que contiene la configuración de software (sistema operativo, servidor de aplicaciones y aplicaciones) necesaria para lanzar una instancia.
  • Tipos de Instancia: Varias configuraciones de CPU, memoria, almacenamiento y capacidad de red para tus instancias.
  • Regiones y Zonas de Disponibilidad: Ubicaciones físicas alrededor del mundo donde se encuentran los centros de datos de AWS.
  • Grupos de Seguridad: Cortafuegos virtuales que controlan el tráfico para una o más instancias.
  • Direcciones IP Elásticas: Direcciones IP estáticas diseñadas para la computación en la nube dinámica.
  • Pares de Claves: Información de inicio de sesión segura para tus instancias.

Empezando con EC2

Lanzando una Instancia EC2

  1. Inicia sesión en la Consola de Administración de AWS.
  2. Navega al Panel de EC2.
  3. Haz clic en "Launch Instance".
  4. Elige una Amazon Machine Image (AMI):
    • Selecciona una AMI de la lista (por ejemplo, Amazon Linux 2 AMI).
  5. Elige un Tipo de Instancia:
    • Selecciona el tipo de instancia que se ajuste a tus necesidades (por ejemplo, t2.micro para el nivel gratuito).
  6. Configura los Detalles de la Instancia:
    • Establece el número de instancias, configuraciones de red y otras configuraciones.
  7. Agrega Almacenamiento:
    • Especifica el tamaño y tipo de almacenamiento (por ejemplo, SSD de Propósito General de 8 GiB).
  8. Agrega Etiquetas:
    • Agrega pares clave-valor para ayudar a gestionar tus instancias.
  9. Configura el Grupo de Seguridad:
    • Crea un nuevo grupo de seguridad o selecciona uno existente para definir reglas de cortafuegos.
  10. Revisa y Lanza:
    • Revisa tus configuraciones y haz clic en "Launch".
  11. Selecciona un Par de Claves:
    • Elige un par de claves existente o crea uno nuevo para SSH en tu instancia.

Ejemplo: Lanzando una Instancia EC2 usando AWS CLI

aws ec2 run-instances \
    --image-id ami-0abcdef1234567890 \
    --count 1 \
    --instance-type t2.micro \
    --key-name MyKeyPair \
    --security-group-ids sg-0123456789abcdef0 \
    --subnet-id subnet-6e7f829e

Explicación:

  • --image-id: El ID de la AMI.
  • --count: Número de instancias a lanzar.
  • --instance-type: El tipo de instancia.
  • --key-name: El nombre del par de claves.
  • --security-group-ids: El ID del grupo de seguridad.
  • --subnet-id: El ID de la subred.

Gestión de Instancias EC2

Monitoreo y Métricas

  • Amazon CloudWatch: Monitorea tus instancias EC2 y configura alarmas.
  • Verificaciones de Estado de la Instancia: Realizadas automáticamente por AWS para verificar la salud de tus instancias.

Escalado de Instancias EC2

  • Elastic Load Balancing (ELB): Distribuye automáticamente el tráfico de la aplicación entrante entre múltiples instancias.
  • Auto Scaling: Ajusta automáticamente el número de instancias EC2 en respuesta a la demanda cambiante.

Ejemplo: Configurando un Grupo de Auto Scaling

aws autoscaling create-auto-scaling-group \
    --auto-scaling-group-name my-asg \
    --launch-configuration-name my-launch-config \
    --min-size 1 \
    --max-size 5 \
    --desired-capacity 2 \
    --vpc-zone-identifier subnet-6e7f829e

Explicación:

  • --auto-scaling-group-name: Nombre del grupo de Auto Scaling.
  • --launch-configuration-name: Nombre de la configuración de lanzamiento.
  • --min-size: Número mínimo de instancias.
  • --max-size: Número máximo de instancias.
  • --desired-capacity: Número deseado de instancias.
  • --vpc-zone-identifier: ID de la subred.

Características Avanzadas de EC2

Instancias Spot

  • Instancias Spot: Compra capacidad EC2 no utilizada a una tarifa reducida.
  • Spot Fleet: Colección de Instancias Spot y opcionalmente Instancias On-Demand.

Instancias Reservadas

  • Instancias Reservadas: Comprométete a usar EC2 durante un término de 1 o 3 años para recibir un descuento significativo.

Ejemplo: Comprando una Instancia Reservada

aws ec2 purchase-reserved-instances-offering \
    --instance-count 1 \
    --reserved-instances-offering-id e-12345678

Explicación:

  • --instance-count: Número de instancias a reservar.
  • --reserved-instances-offering-id: ID de la oferta de instancia reservada.

Conclusión

Amazon EC2 proporciona una forma escalable, flexible y rentable de ejecutar aplicaciones en la nube. Al comprender los conceptos básicos de lanzar y gestionar instancias, así como características avanzadas como Auto Scaling e Instancias Spot, puedes optimizar tu infraestructura en la nube para satisfacer tus necesidades específicas. Ya seas un principiante o un usuario avanzado, dominar EC2 es una habilidad crucial para aprovechar al máximo el poder de AWS.
