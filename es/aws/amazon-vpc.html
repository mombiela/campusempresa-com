Introducción a Amazon VPC

Amazon Virtual Private Cloud (VPC) te permite lanzar recursos de AWS en una red virtual lógicamente aislada que tú defines. Esta sección cubrirá los conceptos básicos de VPC, sus componentes y cómo se integra con otros servicios de AWS.

Conceptos Clave

  • VPC: Una red virtual dedicada a tu cuenta de AWS.
  • Subnets: Segmentos del rango de direcciones IP de una VPC donde puedes colocar grupos de recursos aislados.
  • Tablas de Rutas: Reglas que determinan hacia dónde se dirige el tráfico de red.
  • Gateway de Internet: Un gateway que permite la comunicación entre instancias en tu VPC y el internet.
  • NAT Gateway: Un gateway que permite a las instancias en una subred privada conectarse a internet u otros servicios de AWS, pero evita que internet inicie una conexión con esas instancias.
  • Grupos de Seguridad: Cortafuegos virtuales que controlan el tráfico entrante y saliente para tus instancias.
  • ACLs de Red: Capa opcional de seguridad para tu VPC que actúa como un cortafuegos para controlar el tráfico que entra y sale de una o más subredes.

Creando Tu Primera VPC

En esta sección, recorreremos los pasos para crear una VPC usando la Consola de Administración de AWS.

Pasos para Crear una VPC

  1. Abre el Tablero de VPC: Navega al Tablero de VPC en la Consola de Administración de AWS.

  2. Crear una VPC:

    • Haz clic en "Crear VPC".
    • Ingresa un nombre para tu VPC.
    • Especifica un bloque CIDR IPv4 (por ejemplo, 10.0.0.0/16).
    • Elige un bloque CIDR IPv6 si es necesario.
    • Selecciona la tenencia (por defecto o dedicada).
    • Haz clic en "Crear VPC".

  3. Crear Subredes:

    • Ve a la sección de Subredes.
    • Haz clic en "Crear Subred".
    • Selecciona tu VPC.
    • Ingresa un nombre para tu subred.
    • Especifica la Zona de Disponibilidad.
    • Ingresa un bloque CIDR IPv4 (por ejemplo, 10.0.1.0/24).
    • Haz clic en "Crear".

  4. Crear un Gateway de Internet:

    • Ve a la sección de Gateways de Internet.
    • Haz clic en "Crear Gateway de Internet".
    • Ingresa un nombre para el gateway.
    • Haz clic en "Crear".
    • Adjunta el Gateway de Internet a tu VPC.

  5. Actualizar Tablas de Rutas:

    • Ve a la sección de Tablas de Rutas.
    • Selecciona la tabla de rutas asociada con tu VPC.
    • Haz clic en la pestaña "Rutas".
    • Haz clic en "Editar rutas".
    • Agrega una ruta con destino 0.0.0.0/0 y objetivo como tu Gateway de Internet.
    • Haz clic en "Guardar rutas".

  6. Configurar Grupos de Seguridad:

    • Ve a la sección de Grupos de Seguridad.
    • Haz clic en "Crear Grupo de Seguridad".
    • Ingresa un nombre y una descripción.
    • Selecciona tu VPC.
    • Agrega reglas entrantes y salientes según sea necesario.
    • Haz clic en "Crear".

Ejemplo: Creando una VPC usando AWS CLI

# Crear una VPC
aws ec2 create-vpc --cidr-block 10.0.0.0/16

# Crear una subred
aws ec2 create-subnet --vpc-id vpc-12345678 --cidr-block 10.0.1.0/24

# Crear un Gateway de Internet
aws ec2 create-internet-gateway

# Adjuntar el Gateway de Internet a la VPC
aws ec2 attach-internet-gateway --vpc-id vpc-12345678 --internet-gateway-id igw-12345678

# Crear una tabla de rutas
aws ec2 create-route-table --vpc-id vpc-12345678

# Crear una ruta hacia el Gateway de Internet
aws ec2 create-route --route-table-id rtb-12345678 --destination-cidr-block 0.0.0.0/0 --gateway-id igw-12345678

# Asociar la tabla de rutas con la subred
aws ec2 associate-route-table --subnet-id subnet-12345678 --route-table-id rtb-12345678

# Crear un grupo de seguridad
aws ec2 create-security-group --group-name my-security-group --description "My security group" --vpc-id vpc-12345678

# Agregar una regla para permitir acceso SSH
aws ec2 authorize-security-group-ingress --group-id sg-12345678 --protocol tcp --port 22 --cidr 0.0.0.0/0

Configuraciones Avanzadas de VPC

Esta sección cubrirá temas más avanzados como Peering de VPC, Conexiones VPN y Endpoints de VPC.

Peering de VPC

  • Definición: Una conexión de red entre dos VPCs que te permite enrutar tráfico entre ellas usando direcciones IP privadas.
  • Casos de Uso: Compartir recursos entre VPCs, acceso entre cuentas.

Conexiones VPN

  • Definición: Una conexión segura entre tu red on-premises y tu VPC de AWS.
  • Casos de Uso: Arquitecturas de nube híbrida, transferencia segura de datos.

Endpoints de VPC

  • Definición: Te permite conectar tu VPC de manera privada a servicios de AWS soportados y servicios de endpoint de VPC impulsados por AWS PrivateLink.
  • Tipos: Endpoints de Interfaz, Endpoints de Gateway.

Mejores Prácticas para VPC

  • Diseño de Subredes: Usa múltiples Zonas de Disponibilidad para alta disponibilidad.
  • Seguridad: Implementa acceso de menor privilegio usando Grupos de Seguridad y ACLs de Red.
  • Monitoreo: Usa VPC Flow Logs para monitorear el tráfico.
  • Gestión de Costos: Optimiza el uso de NAT Gateway para reducir costos.

Conclusión

Amazon VPC es un servicio poderoso que te permite crear una red segura y escalable para tus recursos de AWS. Al entender los componentes básicos y configuraciones, puedes diseñar e implementar una arquitectura de red robusta adaptada a tus necesidades. Configuraciones avanzadas como Peering de VPC, Conexiones VPN y Endpoints de VPC extienden aún más las capacidades de tu VPC, permitiendo escenarios de red complejos y seguros. Siempre sigue las mejores prácticas para asegurar la seguridad, disponibilidad y eficiencia de costos.
