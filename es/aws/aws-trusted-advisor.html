Introducción a AWS Trusted Advisor

AWS Trusted Advisor es un servicio que proporciona orientación en tiempo real para ayudarte a aprovisionar tus recursos siguiendo las mejores prácticas de AWS. Las verificaciones de Trusted Advisor cubren cinco categorías: optimización de costos, rendimiento, seguridad, tolerancia a fallos y límites de servicio.

Conceptos Clave

  • Optimización de Costos: Identifica oportunidades para reducir tus costos generales de AWS.
  • Rendimiento: Ayuda a mejorar el rendimiento de tus servicios de AWS.
  • Seguridad: Mejora la seguridad de tu entorno de AWS.
  • Tolerancia a Fallos: Aumenta la disponibilidad y redundancia de tus aplicaciones de AWS.
  • Límites de Servicio: Monitorea tu uso contra los límites de servicio para prevenir interrupciones.

Accediendo a AWS Trusted Advisor

Para acceder a AWS Trusted Advisor:

  1. Inicia sesión en la Consola de Administración de AWS.
  2. Navega al panel de Trusted Advisor desde la Consola de Administración de AWS.

Usando AWS Trusted Advisor

Resumen del Panel

El panel de Trusted Advisor proporciona un resumen de los resultados de tus verificaciones, categorizados en las cinco áreas clave. Cada categoría muestra el número de verificaciones que están en estado "verde" (sin problemas), "amarillo" (se recomienda investigación) o "rojo" (se recomienda acción).

Ejemplo: Verificación de Optimización de Costos

Una de las verificaciones más comunes es la verificación de optimización de costos. Esta verificación te ayuda a identificar recursos infrautilizados que puedes detener o reducir para ahorrar dinero.

# Ejemplo: Identificando Instancias EC2 Infrautilizadas
import boto3

# Inicializa una sesión usando Amazon EC2
session = boto3.Session(
    aws_access_key_id='TU_CLAVE_DE_ACCESO',
    aws_secret_access_key='TU_CLAVE_SECRETA',
    region_name='us-west-2'
)

# Inicializa el recurso EC2
ec2 = session.resource('ec2')

# Lista todas las instancias
for instance in ec2.instances.all():
    print(instance.id, instance.instance_type, instance.state)

Este script lista todas las instancias EC2, que luego puedes analizar para identificar instancias infrautilizadas.

Ejemplo: Verificación de Seguridad

Las verificaciones de seguridad te ayudan a identificar posibles vulnerabilidades de seguridad en tu entorno de AWS. Por ejemplo, puede verificar si hay buckets de S3 con acceso público.

# Ejemplo: Verificando Buckets S3 Públicos
import boto3

# Inicializa una sesión usando Amazon S3
s3 = boto3.client('s3')

# Lista todos los buckets
response = s3.list_buckets()

# Verifica cada bucket para acceso público
for bucket in response['Buckets']:
    bucket_name = bucket['Name']
    acl = s3.get_bucket_acl(Bucket=bucket_name)
    for grant in acl['Grants']:
        if grant['Grantee']['Type'] == 'Group' y 'AllUsers' in grant['Grantee']['URI']:
            print(f"El bucket {bucket_name} es público.")

Este script verifica si hay buckets S3 con acceso público e imprime sus nombres.

Uso Avanzado

Personalizando Verificaciones de Trusted Advisor

Para usuarios avanzados, AWS Trusted Advisor permite la personalización de verificaciones para ajustarse mejor a necesidades específicas. Esto se puede hacer a través de la Consola de Administración de AWS o programáticamente usando los SDKs de AWS.

Integrando Trusted Advisor con Herramientas de Automatización

Puedes integrar Trusted Advisor con herramientas de automatización como AWS Lambda para remediar automáticamente problemas. Por ejemplo, puedes crear una función Lambda que detenga instancias EC2 infrautilizadas basándose en las recomendaciones de Trusted Advisor.

# Ejemplo: Función Lambda para Detener Instancias EC2 Infrautilizadas
import boto3

def lambda_handler(event, context):
    ec2 = boto3.client('ec2')
    trusted_advisor = boto3.client('support')
    
    # Obtén los resultados de la verificación de Trusted Advisor
    response = trusted_advisor.describe_trusted_advisor_check_result(
        checkId='eW7HH0l7J9',  # ID de verificación de ejemplo para instancias infrautilizadas
        language='en'
    )
    
    # Detén instancias infrautilizadas
    for resource in response['result']['flaggedResources']:
        instance_id = resource['resourceId']
        ec2.stop_instances(InstanceIds=[instance_id])
        print(f"Instancia {instance_id} detenida")

# Nota: Asegúrate de que la función Lambda tenga los permisos necesarios de IAM.

Conclusión

AWS Trusted Advisor es una herramienta poderosa que te ayuda a optimizar tu entorno de AWS en cuanto a costos, rendimiento, seguridad, tolerancia a fallos y límites de servicio. Al revisar regularmente las recomendaciones de Trusted Advisor e integrarlas en tus prácticas operativas, puedes asegurarte de que tus recursos de AWS se usen de manera eficiente y segura.
