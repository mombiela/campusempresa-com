Amazon CloudWatch es un servicio de monitoreo y observabilidad diseñado para ingenieros DevOps, desarrolladores, ingenieros de confiabilidad del sitio (SRE) y gerentes de TI. Proporciona datos e información procesable para monitorear aplicaciones, responder a cambios de rendimiento a nivel del sistema, optimizar la utilización de recursos y obtener una vista unificada de la salud operativa.

Introducción a Amazon CloudWatch

  • ¿Qué es Amazon CloudWatch?

    • Un servicio de monitoreo y observabilidad.
    • Recoge datos de monitoreo y operativos en forma de registros, métricas y eventos.
    • Proporciona una vista unificada de los recursos, aplicaciones y servicios de AWS.

  • Características clave:

    • Métricas: Recoge y rastrea métricas de recursos de AWS y aplicaciones personalizadas.
    • Registros: Recoge, monitorea y analiza archivos de registro.
    • Alarmas: Configura alarmas para reaccionar automáticamente a cambios en tus métricas.
    • Tableros: Crea tableros personalizados para visualizar métricas y registros.
    • Eventos: Responde a cambios en tu entorno de AWS.

Configuración de Amazon CloudWatch

  • Requisitos previos:

    • Cuenta de AWS.
    • Comprensión básica de los servicios de AWS.

  • Pasos para habilitar CloudWatch:

    1. Navega a CloudWatch en la Consola de Administración de AWS:

      • Ve a la Consola de Administración de AWS.
      • Busca y selecciona "CloudWatch."

    2. Crea un Tablero de CloudWatch:

      • Haz clic en "Tableros" en el menú de la izquierda.
      • Haz clic en "Crear tablero."
      • Ingresa un nombre para tu tablero y haz clic en "Crear tablero."

    3. Agrega Widgets a tu Tablero:

      • Elige el tipo de widget (por ejemplo, Línea, Área Apilada, Número).
      • Selecciona las métricas que deseas mostrar.
      • Configura los ajustes del widget y haz clic en "Agregar al tablero."

Monitoreo con Métricas de CloudWatch

  • Comprensión de las Métricas:

    • Las métricas son puntos de datos recogidos a lo largo del tiempo.
    • Cada métrica tiene un espacio de nombres, un nombre y una o más dimensiones.

  • Métricas Comunes de AWS:

    • EC2: CPUUtilization, DiskReadOps, NetworkIn.
    • RDS: CPUUtilization, DatabaseConnections, FreeStorageSpace.
    • S3: BucketSizeBytes, NumberOfObjects.

  • Creación de Métricas Personalizadas:

    import boto3
import time

# Crear cliente de CloudWatch
cloudwatch = boto3.client('cloudwatch')

# Poner datos de métrica personalizada
response = cloudwatch.put_metric_data(
    Namespace='MyCustomNamespace',
    MetricData=[
        {
            'MetricName': 'MyCustomMetric',
            'Dimensions': [
                {
                    'Name': 'InstanceType',
                    'Value': 'm5.large'
                },
            ],
            'Value': 1.0,
            'Unit': 'Count',
            'Timestamp': time.time()
        },
    ]
)
print(response)

Trabajando con Registros de CloudWatch

  • Grupos de Registros y Flujos de Registros:

    • Grupos de Registros: Un grupo de flujos de registros que comparten la misma retención, monitoreo y configuraciones de control de acceso.
    • Flujos de Registros: Una secuencia de eventos de registro de la misma fuente.

  • Creación de un Grupo de Registros:

    import boto3

# Crear cliente de CloudWatch Logs
logs = boto3.client('logs')

# Crear un grupo de registros
response = logs.create_log_group(
    logGroupName='MyLogGroup'
)
print(response)

  • Envío de Registros a CloudWatch:

    import boto3

# Crear cliente de CloudWatch Logs
logs = boto3.client('logs')

# Crear un flujo de registros
response = logs.create_log_stream(
    logGroupName='MyLogGroup',
    logStreamName='MyLogStream'
)

# Poner eventos de registro
response = logs.put_log_events(
    logGroupName='MyLogGroup',
    logStreamName='MyLogStream',
    logEvents=[
        {
            'timestamp': int(time.time() * 1000),
            'message': 'Este es un mensaje de registro'
        },
    ]
)
print(response)

Configuración de Alarmas de CloudWatch

  • Creación de una Alarma:

    1. Navega a Alarmas en la Consola de CloudWatch:

      • Haz clic en "Alarmas" en el menú de la izquierda.
      • Haz clic en "Crear alarma."

    2. Selecciona una Métrica:

      • Elige una métrica de la lista o busca una métrica específica.
      • Haz clic en "Seleccionar métrica."

    3. Configura la Alarma:

      • Establece las condiciones para la alarma (por ejemplo, umbral, período).
      • Configura acciones (por ejemplo, enviar notificación, ejecutar una política de Auto Scaling).
      • Haz clic en "Crear alarma."

Tableros de CloudWatch

  • Creación y Personalización de Tableros:

    • Los tableros proporcionan una página de inicio personalizable en la consola de CloudWatch.
    • Puedes agregar widgets para mostrar métricas, registros y alarmas.

  • Ejemplo de Configuración de un Tablero:

    {
    "widgets": [
        {
            "type": "metric",
            "x": 0,
            "y": 0,
            "width": 6,
            "height": 6,
            "properties": {
                "metrics": [
                    [ "AWS/EC2", "CPUUtilization", "InstanceId", "i-1234567890abcdef0" ]
                ],
                "period": 300,
                "stat": "Average",
                "region": "us-east-1",
                "title": "CPU de Instancia EC2"
            }
        }
    ]
}

Eventos de CloudWatch

  • Comprensión de los Eventos:

    • Los eventos se generan cuando ocurren cambios en tu entorno de AWS.
    • Puedes crear reglas para coincidir con eventos y dirigirlos a destinos como funciones Lambda, temas SNS o colas SQS.

  • Creación de una Regla de Evento:

    import boto3

# Crear cliente de CloudWatch Events
events = boto3.client('events')

# Crear una regla
response = events.put_rule(
    Name='MyRule',
    EventPattern='{"source": ["aws.ec2"]}',
    State='ENABLED'
)
print(response)

Conclusión

Amazon CloudWatch es una herramienta poderosa para monitorear y gestionar tus recursos de AWS. Al comprender sus características clave y cómo configurar y usar métricas, registros, alarmas, tableros y eventos, puedes obtener información valiosa sobre tus aplicaciones e infraestructura. Este conocimiento te permite optimizar el rendimiento, asegurar la confiabilidad y responder rápidamente a cualquier problema que surja.
