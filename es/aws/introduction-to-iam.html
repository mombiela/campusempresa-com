¿Qué es IAM?

AWS Identity and Access Management (IAM) es un servicio web que te ayuda a controlar de manera segura el acceso a los servicios y recursos de AWS para tus usuarios. IAM te permite gestionar permisos para usuarios y grupos para asegurar que solo los usuarios autorizados puedan acceder a recursos específicos.

Conceptos Clave

  • Usuarios: Cuentas individuales que representan a una persona o servicio.
  • Grupos: Colecciones de usuarios que comparten permisos comunes.
  • Roles: Credenciales temporales que pueden ser asumidas por usuarios o servicios.
  • Políticas: Documentos que definen permisos y se adjuntan a usuarios, grupos o roles.

¿Por qué Usar IAM?

  • Seguridad: Controla quién puede acceder a tus recursos.
  • Permisos Granulares: Define permisos específicos para diferentes usuarios y servicios.
  • Gestión Centralizada: Gestiona el acceso desde una única ubicación.

Creación de un Usuario IAM

Para crear un usuario IAM, sigue estos pasos:

  1. Inicia sesión en la Consola de Administración de AWS.
  2. Abre la consola de IAM en https://console.aws.amazon.com/iam/.
  3. En el panel de navegación, elige "Usuarios" y luego "Agregar usuario".
  4. Ingresa los detalles del usuario y selecciona el tipo de acceso (Acceso programático, Acceso a la Consola de Administración de AWS).
  5. Adjunta políticas al usuario para otorgar permisos.
  6. Revisa y crea el usuario.

Ejemplo: Creación de un Usuario IAM con AWS CLI

aws iam create-user --user-name JohnDoe
aws iam attach-user-policy --user-name JohnDoe --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

En este ejemplo:

  • El primer comando crea un usuario llamado JohnDoe.
  • El segundo comando adjunta la política AmazonS3ReadOnlyAccess a JohnDoe, otorgando acceso de solo lectura a Amazon S3.

Gestión de Políticas IAM

Las políticas IAM son documentos JSON que definen permisos. Aquí hay una estructura básica de una política:

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:ListBucket",
      "Resource": "arn:aws:s3:::example_bucket"
    }
  ]
}

Elementos Clave de una Política

  • Versión: La versión del lenguaje de la política.
  • Declaración: Una o más declaraciones individuales.
    • Efecto: Ya sea Allow o Deny.
    • Acción: La acción específica (por ejemplo, s3:ListBucket).
    • Recurso: El ARN del recurso (por ejemplo, arn:aws:s3:::example_bucket).

Roles IAM

Los roles IAM te permiten delegar acceso con credenciales temporales. Los roles son útiles para:

  • Otorgar permisos a servicios de AWS.
  • Permitir que usuarios de otra cuenta de AWS accedan a tus recursos.
  • Habilitar aplicaciones que se ejecutan en instancias EC2 para acceder a recursos de AWS.

Ejemplo: Creación de un Rol IAM para EC2

  1. Abre la consola de IAM.
  2. En el panel de navegación, elige "Roles" y luego "Crear rol".
  3. Selecciona "Servicio de AWS" y luego "EC2".
  4. Adjunta las políticas necesarias.
  5. Completa el proceso de creación del rol.

Ejemplo: Creación de un Rol IAM con AWS CLI

aws iam create-role --role-name EC2S3AccessRole --assume-role-policy-document file://trust-policy.json
aws iam attach-role-policy --role-name EC2S3AccessRole --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

En este ejemplo:

  • El primer comando crea un rol llamado EC2S3AccessRole con una política de confianza definida en trust-policy.json.
  • El segundo comando adjunta la política AmazonS3ReadOnlyAccess al rol.

Mejores Prácticas para IAM

  • Menor Privilegio: Otorga solo los permisos necesarios para que los usuarios realicen sus tareas.
  • Usa Grupos: Gestiona permisos para múltiples usuarios asignándolos a grupos.
  • Habilita MFA: Usa Autenticación Multifactor para mayor seguridad.
  • Auditorías Regulares: Revisa y actualiza periódicamente los permisos.

Conclusión

IAM es un servicio fundamental en AWS que te ayuda a gestionar el acceso a tus recursos de manera segura. Al comprender y utilizar las características de IAM, como usuarios, grupos, roles y políticas, puedes asegurar que tu entorno de AWS sea tanto seguro como eficiente. Recuerda seguir las mejores prácticas para mantener una postura de seguridad robusta.
