Introducción a AWS Config

AWS Config es un servicio que te permite evaluar, auditar y evaluar las configuraciones de tus recursos de AWS. Te ayuda a monitorear y registrar las configuraciones de tus recursos de AWS y automatizar la evaluación de las configuraciones registradas contra las configuraciones deseadas.

Conceptos Clave

  • Elemento de Configuración (CI): Un registro de la configuración de un recurso en un momento dado.
  • Grabador de Configuración: Registra las configuraciones de los recursos compatibles en tu cuenta.
  • Canal de Entrega: Especifica dónde AWS Config entrega las instantáneas de configuración y los archivos de historial de configuración.
  • Instantánea de Configuración: Una colección de elementos de configuración para los recursos compatibles en tu cuenta.
  • Historial de Configuración: Una colección de elementos de configuración para un recurso durante un período de tiempo.
  • Reglas: Las reglas de AWS Config representan tus configuraciones deseadas para tus recursos de AWS.

Configuración de AWS Config

Para comenzar a usar AWS Config, necesitas configurar el grabador de configuración y el canal de entrega.

Guía Paso a Paso

  1. Abre la Consola de Administración de AWS.
  2. Navega a AWS Config.
  3. Configura el Grabador de Configuración:
    • Elige los recursos que deseas registrar.
    • Especifica la frecuencia de grabación (por ejemplo, cada 1 hora).
  4. Configura el Canal de Entrega:
    • Elige un bucket de S3 para almacenar las instantáneas de configuración y el historial.
    • Opcionalmente, configura un tema de SNS para notificaciones.

Ejemplo

import boto3

# Crear un cliente de Config
config = boto3.client('config')

# Configurar el grabador de configuración
config.put_configuration_recorder(
    ConfigurationRecorder={
        'name': 'default',
        'roleARN': 'arn:aws:iam::123456789012:role/config-role',
        'recordingGroup': {
            'allSupported': True,
            'includeGlobalResourceTypes': True
        }
    }
)

# Configurar el canal de entrega
config.put_delivery_channel(
    DeliveryChannel={
        'name': 'default',
        's3BucketName': 'my-config-bucket',
        'snsTopicARN': 'arn:aws:sns:us-east-1:123456789012:config-topic'
    }
)

# Iniciar el grabador de configuración
config.start_configuration_recorder(
    ConfigurationRecorderName='default'
)

Reglas de AWS Config

Las reglas de AWS Config te permiten definir directrices de cumplimiento para tus recursos. Puedes usar reglas administradas por AWS o crear reglas personalizadas.

Reglas Administradas

AWS proporciona un conjunto de reglas predefinidas que puedes usar para evaluar si tus recursos de AWS cumplen con las mejores prácticas comunes.

Reglas Personalizadas

Puedes crear reglas personalizadas usando funciones de AWS Lambda para evaluar tus recursos según criterios específicos.

Ejemplo: Regla Administrada

config.put_config_rule(
    ConfigRule={
        'ConfigRuleName': 's3-bucket-public-read-prohibited',
        'Description': 'Verifica que tus buckets de S3 no permitan acceso de lectura pública.',
        'Scope': {
            'ComplianceResourceTypes': [
                'AWS::S3::Bucket',
            ]
        },
        'Source': {
            'Owner': 'AWS',
            'SourceIdentifier': 'S3_BUCKET_PUBLIC_READ_PROHIBITED'
        }
    }
)

Ejemplo: Regla Personalizada

config.put_config_rule(
    ConfigRule={
        'ConfigRuleName': 'custom-rule',
        'Description': 'Regla personalizada para verificar el cumplimiento específico.',
        'Scope': {
            'ComplianceResourceTypes': [
                'AWS::EC2::Instance',
            ]
        },
        'Source': {
            'Owner': 'CUSTOM_LAMBDA',
            'SourceIdentifier': 'arn:aws:lambda:us-east-1:123456789012:function:custom-rule-function'
        }
    }
)

Monitoreo y Notificaciones

AWS Config proporciona varias formas de monitorear y recibir notificaciones sobre el estado de cumplimiento de tus recursos.

Monitoreo

  • Consola de Administración de AWS: Ver el estado de cumplimiento de tus recursos.
  • AWS CLI/SDK: Usar comandos y llamadas a la API para recuperar información de cumplimiento.

Notificaciones

  • Notificaciones de SNS: Configura temas de SNS para recibir notificaciones sobre cambios de configuración y estado de cumplimiento.

Ejemplo: Recuperar Estado de Cumplimiento

response = config.get_compliance_details_by_config_rule(
    ConfigRuleName='s3-bucket-public-read-prohibited'
)

for result in response['EvaluationResults']:
    print(result['ComplianceType'], result['EvaluationResultIdentifier']['EvaluationResultQualifier']['ResourceId'])

Conclusión

AWS Config es una herramienta poderosa para gestionar y auditar las configuraciones de tus recursos de AWS. Al configurar grabadores de configuración, canales de entrega y reglas, puedes asegurarte de que tus recursos cumplan con las políticas y mejores prácticas de tu organización. El monitoreo regular y las notificaciones te ayudan a mantenerte informado sobre el estado de cumplimiento de tus recursos, permitiéndote tomar acciones correctivas cuando sea necesario.
