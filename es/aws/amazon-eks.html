Introducción a Amazon EKS

Amazon Elastic Kubernetes Service (EKS) es un servicio de Kubernetes gestionado que facilita la ejecución de Kubernetes en AWS sin necesidad de instalar y operar tu propio plano de control o nodos de Kubernetes. EKS está certificado como conforme a Kubernetes, por lo que puedes usar herramientas y complementos existentes de la comunidad de Kubernetes.

Conceptos Clave

  • Kubernetes: Un sistema de código abierto para automatizar el despliegue, escalado y gestión de aplicaciones en contenedores.
  • Plano de Control de EKS: Gestionado por AWS, incluye el servidor API de Kubernetes y la base de datos etcd.
  • Nodos de Trabajo: Instancias EC2 que ejecutan tus aplicaciones en contenedores.
  • Cluster de EKS: Una colección de instancias EC2 gestionadas por EKS para ejecutar cargas de trabajo de Kubernetes.

Configuración de un Cluster de EKS

Configurar un cluster de EKS implica varios pasos, incluyendo la creación del propio cluster de EKS, la configuración de los nodos de trabajo y la configuración de los roles y políticas de IAM necesarios.

Guía Paso a Paso

  1. Crear un Cluster de EKS:

    aws eks create-cluster --name my-cluster --role-arn arn:aws:iam::123456789012:role/EKS-Role --resources-vpc-config subnetIds=subnet-6782e71e,subnet-e7e761ac,securityGroupIds=sg-6979fe18

  2. Crear un Grupo de Nodos:

    aws eks create-nodegroup --cluster-name my-cluster --nodegroup-name my-nodegroup --subnets subnet-6782e71e subnet-e7e761ac --node-role arn:aws:iam::123456789012:role/EKS-Node-Role --scaling-config minSize=1,maxSize=3,desiredSize=2

  3. Actualizar kubeconfig:

    aws eks update-kubeconfig --name my-cluster

  4. Desplegar una Aplicación de Ejemplo:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

    Aplicar el despliegue:

    kubectl apply -f nginx-deployment.yaml

Gestión de Clusters de EKS

Gestionar un cluster de EKS implica escalar, actualizar y monitorear el cluster y sus cargas de trabajo.

Escalado del Cluster

  • Autoscaler Horizontal de Pods: Escala automáticamente el número de pods en un despliegue basado en la utilización observada de la CPU.

    apiVersion: autoscaling/v1
kind: HorizontalPodAutoscaler
metadata:
  name: nginx-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: nginx-deployment
  minReplicas: 1
  maxReplicas: 10
  targetCPUUtilizationPercentage: 50

    Aplicar el autoscaler:

    kubectl apply -f nginx-hpa.yaml

Actualización del Cluster

  • Actualizaciones Graduales: Kubernetes soporta actualizaciones graduales para actualizar la versión de la aplicación sin tiempo de inactividad. 
    kubectl set image deployment/nginx-deployment nginx=nginx:1.16.1

Monitoreo del Cluster

  • CloudWatch: Usa Amazon CloudWatch para monitorear tu cluster de EKS.
  • Prometheus y Grafana: Herramientas de código abierto populares para monitorear clusters de Kubernetes.

Características Avanzadas de EKS

Integración con Fargate

EKS soporta AWS Fargate, lo que te permite ejecutar pods de Kubernetes sin gestionar las instancias EC2 subyacentes.

Malla de Servicios con AWS App Mesh

AWS App Mesh proporciona redes a nivel de aplicación para facilitar la comunicación entre tus servicios a través de múltiples tipos de infraestructura de cómputo.

Mejores Prácticas de Seguridad

  • Roles de IAM para Cuentas de Servicio (IRSA): Permisos granulares para cuentas de servicio de Kubernetes.
  • Políticas de Red: Controla la comunicación entre pods.

Conclusión

Amazon EKS simplifica la ejecución de Kubernetes en AWS gestionando el plano de control e integrándose con otros servicios de AWS. Al entender cómo configurar, gestionar y utilizar características avanzadas de EKS, puedes desplegar y escalar efectivamente aplicaciones en contenedores en la nube. Ya seas un principiante o un usuario avanzado, EKS proporciona las herramientas y la flexibilidad necesarias para ejecutar cargas de trabajo de Kubernetes de grado de producción.
