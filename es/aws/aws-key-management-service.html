Introducción a AWS KMS

AWS Key Management Service (KMS) es un servicio gestionado que facilita la creación y el control de las claves de cifrado utilizadas para cifrar tus datos. KMS está integrado con otros servicios de AWS, lo que hace sencillo cifrar los datos que almacenas en estos servicios y controlar el acceso a las claves que los descifran.

Conceptos Clave

  • Customer Master Keys (CMKs): Los recursos principales en AWS KMS. Las CMKs se pueden usar para cifrar y descifrar hasta 4 KB de datos directamente o para generar claves de datos que pueden cifrar cantidades mayores de datos.
  • Data Keys: Claves generadas por las CMKs que se utilizan para cifrar datos. Las claves de datos se pueden usar fuera de AWS KMS.
  • Key Policies: Políticas basadas en JSON que definen quién puede usar y gestionar las CMKs.
  • Grants: Permiten permisos temporales para usar las CMKs.
  • Encryption Context: Datos opcionales y no secretos que se pueden asociar con datos cifrados para proporcionar seguridad adicional.

Creación y Gestión de CMKs

Creación de una CMK

Para crear una CMK, puedes usar la Consola de Administración de AWS, AWS CLI o los SDKs de AWS.

Usando la Consola de Administración de AWS

  1. Abre la consola de AWS KMS.
  2. Elige Create key.
  3. Selecciona el tipo de clave (simétrica o asimétrica).
  4. Configura los ajustes de la clave, incluyendo los permisos de uso de la clave.
  5. Revisa y crea la clave.

Usando AWS CLI

aws kms create-key --description "My CMK" --key-usage ENCRYPT_DECRYPT --origin AWS_KMS

Gestión de CMKs

  • Habilitar/Deshabilitar CMKs: Puedes habilitar o deshabilitar las CMKs para controlar su uso.
  • Rotación de CMKs: AWS KMS puede rotar automáticamente las CMKs cada año.
  • Eliminación de CMKs: Programa una CMK para su eliminación si ya no es necesaria.

Cifrado y Descifrado de Datos

Cifrado de Datos

Puedes cifrar datos usando CMKs directamente o utilizando claves de datos.

Cifrado con AWS CLI

aws kms encrypt --key-id <CMK_ID> --plaintext fileb://example.txt --output text --query CiphertextBlob

Descifrado de Datos

Para descifrar datos, necesitas tener los permisos necesarios para usar la CMK.

Descifrado con AWS CLI

aws kms decrypt --ciphertext-blob fileb://encrypted_file --output text --query Plaintext

Políticas de Claves y Control de Acceso

Políticas de Claves

Las políticas de claves son documentos JSON que especifican quién puede usar y gestionar una CMK. Son similares a las políticas de IAM pero son específicas de KMS.

Ejemplo de Política de Clave

{
  "Version": "2012-10-17",
  "Id": "key-default-1",
  "Statement": [
    {
      "Sid": "Enable IAM User Permissions",
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::123456789012:root"
      },
      "Action": "kms:*",
      "Resource": "*"
    }
  ]
}

Grants

Los grants proporcionan permisos temporales para usar una CMK. Son útiles para delegar acceso sin modificar la política de la clave.

Creación de un Grant

aws kms create-grant --key-id <CMK_ID> --grantee-principal <IAM_ROLE_ARN> --operations Encrypt Decrypt

Temas Avanzados

Uso del Encryption Context

Un encryption context es un dato adicional y no secreto que puedes asociar con datos cifrados para proporcionar seguridad adicional.

Ejemplo con Encryption Context

aws kms encrypt --key-id <CMK_ID> --plaintext fileb://example.txt --encryption-context Purpose=Test --output text --query CiphertextBlob

Claves Asimétricas

AWS KMS soporta claves asimétricas para cifrado/descifrado y firma/verificación.

Creación de una CMK Asimétrica

aws kms create-key --description "My Asymmetric CMK" --key-usage ENCRYPT_DECRYPT --key-spec RSA_2048

Conclusión

AWS KMS es un servicio poderoso y flexible para gestionar claves de cifrado en la nube. Al comprender los conceptos clave, cómo crear y gestionar CMKs, y cómo cifrar y descifrar datos, puedes asegurar efectivamente tus datos en AWS. Las características avanzadas como el encryption context y las claves asimétricas proporcionan capacidades adicionales para casos de uso más complejos.
