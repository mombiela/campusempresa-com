Introducción a las Políticas de IAM

Las políticas de AWS Identity and Access Management (IAM) son documentos JSON que definen permisos para acciones sobre recursos de AWS. Entender las políticas de IAM es crucial para gestionar el control de acceso en AWS.

  • Políticas de IAM: Documentos JSON que definen permisos.
  • Permisos: Permiten o deniegan acciones sobre recursos de AWS.
  • Principales: Usuarios, grupos o roles a los que se adjuntan las políticas.

Estructura Básica de una Política de IAM

Una política de IAM consta de varios elementos:

  • Versión: Especifica la versión del lenguaje de la política.
  • Declaración: Contiene los elementos principales de la política.
    • Efecto: Puede ser "Allow" o "Deny".
    • Acción: Lista de acciones que están permitidas o denegadas.
    • Recurso: Especifica los recursos de AWS a los que se aplican las acciones.
    • Condición: (Opcional) Especifica las condiciones bajo las cuales la política está en efecto.

Ejemplo de una Política de IAM Simple

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:ListBucket",
      "Resource": "arn:aws:s3:::example-bucket"
    }
  ]
}
  • Explicación:
    • Esta política permite la acción s3:ListBucket en el example-bucket.

Tipos de Políticas de IAM

Existen varios tipos de políticas de IAM:

  • Políticas Administradas: Políticas predefinidas creadas y gestionadas por AWS o tu organización.
  • Políticas Inline: Políticas que creas e incrustas directamente en un único usuario, grupo o rol.
  • Políticas Administradas por el Cliente: Políticas que creas y gestionas en tu cuenta de AWS.

Políticas Administradas vs. Políticas Inline

| Característica | Políticas Administradas | Políticas Inline | |----------------------|----------------------------------|--------------------------------| | Creación | Creado por AWS o por ti | Creado por ti | | Gestión | Más fácil de gestionar y reutilizar | Incrustado directamente en un usuario, grupo o rol | | Reutilización | Puede ser adjuntado a múltiples entidades | Específico para una sola entidad |

Creación y Adjunto de Políticas de IAM

Creación de una Política Administrada

  1. Navega a la consola de IAM.
  2. Selecciona "Policies" en la barra lateral.
  3. Haz clic en "Create policy".
  4. Usa el editor visual o el editor JSON para definir la política.
  5. Revisa y crea la política.

Adjuntar una Política a un Usuario, Grupo o Rol

  1. Navega a la consola de IAM.
  2. Selecciona "Users", "Groups" o "Roles" en la barra lateral.
  3. Elige el usuario, grupo o rol específico.
  4. Haz clic en la pestaña "Permissions".
  5. Haz clic en "Add permissions" y adjunta la política deseada.

Conceptos Avanzados de Políticas de IAM

Condiciones de la Política

Las condiciones te permiten especificar cuándo una política está en efecto. Utilizan pares clave-valor para definir las condiciones.

Ejemplo con Condiciones

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::example-bucket/*",
      "Condition": {
        "IpAddress": {
          "aws:SourceIp": "192.0.2.0/24"
        }
      }
    }
  ]
}
  • Explicación:
    • Esta política permite la acción s3:GetObject en objetos en example-bucket solo si la solicitud proviene del rango de IP 192.0.2.0/24.

Lógica de Evaluación de Políticas

IAM evalúa las políticas basándose en la siguiente lógica:

  1. Denegación Explícita: Si alguna política deniega la acción, la solicitud es denegada.
  2. Permiso Explícito: Si no hay una denegación explícita y al menos una política permite la acción, la solicitud es permitida.
  3. Denegación por Defecto: Si ninguna política permite la acción, la solicitud es denegada por defecto.

Mejores Prácticas para Políticas de IAM

  • Principio de Menor Privilegio: Otorga solo los permisos necesarios para la tarea.
  • Usa Políticas Administradas: Prefiere las políticas administradas por AWS para casos de uso comunes.
  • Auditorías Regulares: Revisa y audita regularmente las políticas y permisos de IAM.
  • Usa Condiciones: Aplica condiciones para añadir una capa extra de seguridad.

Conclusión

Las políticas y permisos de IAM son fundamentales para asegurar tu entorno de AWS. Al entender la estructura, tipos y mejores prácticas para las políticas de IAM, puedes gestionar efectivamente el acceso a tus recursos de AWS. Siempre sigue el principio de menor privilegio y revisa regularmente tus políticas para asegurarte de que cumplen con tus requisitos de seguridad.
