Amazon Route 53 es un servicio web de Sistema de Nombres de Dominio (DNS) escalable y altamente disponible diseñado para enrutar a los usuarios finales a aplicaciones de Internet traduciendo nombres de dominio en direcciones IP. Este tema te guiará desde los conceptos básicos hasta las características avanzadas de Amazon Route 53.

Introducción a Amazon Route 53

  • ¿Qué es Amazon Route 53?

    • Un servicio web de DNS escalable.
    • Diseñado para alta disponibilidad y fiabilidad.
    • Puede ser utilizado para enrutar usuarios a servicios de AWS y recursos externos.

  • Características Clave:

    • Registro de dominios.
    • Enrutamiento DNS.
    • Verificación y monitoreo de salud.
    • Gestión de tráfico.

Configuración de Amazon Route 53

Registro de un Dominio

  • Pasos para Registrar un Dominio:
    1. Abre la consola de Route 53.
    2. Haz clic en "Registered Domains" en el panel de navegación.
    3. Haz clic en "Register Domain."
    4. Ingresa el nombre de dominio que deseas registrar y verifica su disponibilidad.
    5. Sigue las indicaciones para completar el proceso de registro.
# Ejemplo: Registro de un dominio usando AWS CLI
aws route53domains register-domain --domain-name example.com --duration-in-years 1 --admin-contact file://admin-contact.json --registrant-contact file://registrant-contact.json --tech-contact file://tech-contact.json

Creación de una Zona Alojada

  • Pasos para Crear una Zona Alojada:
    1. Abre la consola de Route 53.
    2. Haz clic en "Hosted Zones" en el panel de navegación.
    3. Haz clic en "Create Hosted Zone."
    4. Ingresa el nombre de dominio y un comentario opcional.
    5. Haz clic en "Create."
# Ejemplo: Creación de una zona alojada usando AWS CLI
aws route53 create-hosted-zone --name example.com --caller-reference unique-string

Políticas de Enrutamiento DNS

Enrutamiento Simple

  • Política de Enrutamiento Simple:
    • Mapea un dominio a un solo recurso.
    • Adecuado para consultas DNS directas.
{
  "Comment": "Ejemplo de política de enrutamiento simple",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ]
      }
    }
  ]
}

Enrutamiento Ponderado

  • Política de Enrutamiento Ponderado:
    • Distribuye el tráfico entre múltiples recursos basándose en pesos asignados.
    • Útil para balanceo de carga y pruebas de nuevas versiones de aplicaciones.
{
  "Comment": "Ejemplo de política de enrutamiento ponderado",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Primary",
        "Weight": 70,
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ]
      }
    },
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Secondary",
        "Weight": 30,
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.45"
          }
        ]
      }
    }
  ]
}

Enrutamiento Basado en Latencia

  • Política de Enrutamiento Basado en Latencia:
    • Enruta el tráfico al recurso con la menor latencia para el usuario.
    • Mejora el rendimiento de la aplicación reduciendo la latencia.
{
  "Comment": "Ejemplo de política de enrutamiento basado en latencia",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "US-East",
        "Region": "us-east-1",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ]
      }
    },
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "EU-West",
        "Region": "eu-west-1",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.45"
          }
        ]
      }
    }
  ]
}

Verificaciones de Salud y Monitoreo

  • Verificaciones de Salud:

    • Monitorean la salud de los recursos.
    • Enrutan automáticamente el tráfico lejos de recursos no saludables.

  • Creación de una Verificación de Salud:

    1. Abre la consola de Route 53.
    2. Haz clic en "Health Checks" en el panel de navegación.
    3. Haz clic en "Create Health Check."
    4. Configura los ajustes de la verificación de salud.
    5. Haz clic en "Create."
# Ejemplo: Creación de una verificación de salud usando AWS CLI
aws route53 create-health-check --caller-reference unique-string --health-check-config file://health-check-config.json

Gestión de Tráfico

Enrutamiento por Geolocalización

  • Política de Enrutamiento por Geolocalización:
    • Enruta el tráfico basado en la ubicación geográfica del usuario.
    • Útil para servir contenido localizado.
{
  "Comment": "Ejemplo de política de enrutamiento por geolocalización",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "US",
        "GeoLocation": {
          "CountryCode": "US"
        },
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ]
      }
    },
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "EU",
        "GeoLocation": {
          "CountryCode": "EU"
        },
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.45"
          }
        ]
      }
    }
  ]
}

Enrutamiento por Conmutación por Error

  • Política de Enrutamiento por Conmutación por Error:
    • Enruta el tráfico a un recurso primario a menos que esté no saludable, luego enruta a un recurso secundario.
    • Asegura alta disponibilidad y recuperación ante desastres.
{
  "Comment": "Ejemplo de política de enrutamiento por conmutación por error",
  "Changes": [
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Primary",
        "Failover": "PRIMARY",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.44"
          }
        ],
        "HealthCheckId": "health-check-id"
      }
    },
    {
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "example.com",
        "Type": "A",
        "SetIdentifier": "Secondary",
        "Failover": "SECONDARY",
        "TTL": 300,
        "ResourceRecords": [
          {
            "Value": "192.0.2.45"
          }
        ]
      }
    }
  ]
}

Conclusión

Amazon Route 53 es un poderoso servicio web de DNS que proporciona una variedad de políticas de enrutamiento y capacidades de verificación de salud para asegurar alta disponibilidad y rendimiento para tus aplicaciones. Al entender y utilizar estas características, puedes gestionar y optimizar efectivamente el tráfico hacia tus recursos. Ya sea que estés comenzando con el registro de dominios o implementando estrategias avanzadas de gestión de tráfico, Amazon Route 53 ofrece las herramientas que necesitas para tener éxito.
