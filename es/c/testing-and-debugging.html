Introducción

Las pruebas y la depuración son habilidades cruciales para cualquier programador. En esta sección, cubriremos los conceptos básicos de las pruebas y la depuración en C, progresando desde técnicas para principiantes hasta técnicas avanzadas. Al final de esta sección, podrás identificar y corregir errores en tu código y escribir pruebas para asegurar que tu código funcione como se espera.

Conceptos Básicos de la Depuración

¿Qué es la Depuración?

  • La depuración es el proceso de identificar y eliminar errores del software de computadora.
  • Los errores en el software se conocen comúnmente como "bugs".

Tipos Comunes de Bugs

  • Errores de Sintaxis: Errores en el código que violan las reglas del lenguaje C.
  • Errores en Tiempo de Ejecución: Errores que ocurren mientras el programa se está ejecutando, a menudo causando que el programa se bloquee.
  • Errores Lógicos: Errores que hacen que el programa funcione incorrectamente pero no causan que el programa se bloquee.

Herramientas para la Depuración

  • Declaraciones printf: Simples y efectivas para programas pequeños.
  • Herramientas de Depuración: Herramientas más avanzadas como GDB (GNU Debugger).

Uso de printf para la Depuración

#include <stdio.h>

int main() {
    int a = 5;
    int b = 0;
    int result;

    printf("Antes de la división\n");
    result = a / b; // Esto causará un error en tiempo de ejecución (división por cero)
    printf("Después de la división\n");

    return 0;
}
  • En este ejemplo, el programa se bloqueará antes de imprimir "Después de la división" debido a un error de división por cero.

Uso de GDB para la Depuración

  • Iniciando GDB: gdb ./tu_programa
  • Estableciendo Puntos de Interrupción: break main
  • Ejecutando el Programa: run
  • Inspeccionando Variables: print nombre_variable
  • Avanzando a Través del Código: step o next

Conceptos Básicos de las Pruebas

¿Qué es la Prueba?

  • La prueba es el proceso de ejecutar un programa con la intención de encontrar errores.
  • Asegura que el software se comporte como se espera.

Tipos de Pruebas

  • Pruebas Unitarias: Pruebas de unidades o componentes individuales de un programa.
  • Pruebas de Integración: Pruebas de partes combinadas de una aplicación para determinar si funcionan correctamente juntas.
  • Pruebas del Sistema: Pruebas del sistema completo como un todo.

Escribir Pruebas Unitarias en C

  • Usa bibliotecas como CUnit o Unity para escribir pruebas unitarias.

Ejemplo de una Prueba Unitaria Simple

#include <assert.h>

int add(int a, int b) {
    return a + b;
}

void test_add() {
    assert(add(2, 3) == 5);
    assert(add(-1, 1) == 0);
    assert(add(-1, -1) == -2);
}

int main() {
    test_add();
    printf("¡Todas las pruebas pasaron!\n");
    return 0;
}
  • En este ejemplo, la función assert se usa para verificar si la función add devuelve los resultados esperados.

Técnicas Avanzadas de Depuración

Depuración de Memoria

  • Valgrind: Una herramienta para detectar fugas de memoria y errores de memoria.
  • Uso: valgrind ./tu_programa

Ejemplo de Detección de Fuga de Memoria

#include <stdlib.h>

int main() {
    int *ptr = (int *)malloc(sizeof(int) * 10);
    // Se olvidó liberar la memoria asignada
    return 0;
}
  • Ejecutar este programa a través de Valgrind detectará la fuga de memoria.

Depuración de Fallos de Segmentación

  • Causas Comunes: Desreferenciar punteros nulos o no inicializados, acceder a memoria fuera de los límites.
  • Uso de GDB: Establece puntos de interrupción y avanza a través del código para encontrar la línea exacta que causa el fallo de segmentación.

Técnicas Avanzadas de Pruebas

Desarrollo Guiado por Pruebas (TDD)

  • Escribe pruebas antes de escribir el código real.
  • Asegura que el código siempre sea probado y funcione como se espera.

Mocking y Stubbing

  • Mocking: Crear una versión falsa de una función o módulo para probar otra parte del código.
  • Stubbing: Reemplazar una función con una versión simplificada que devuelve valores fijos.

Ejemplo de Mocking

#include <assert.h>

int get_data_from_db() {
    // Función original que obtiene datos de una base de datos
    return 100;
}

// Función mock
int mock_get_data_from_db() {
    return 42; // Devuelve un valor fijo para pruebas
}

void test_function_using_db() {
    int (*original_function)() = get_data_from_db;
    get_data_from_db = mock_get_data_from_db; // Reemplazar con mock

    assert(get_data_from_db() == 42);

    get_data_from_db = original_function; // Restaurar función original
}

int main() {
    test_function_using_db();
    printf("¡Todas las pruebas pasaron!\n");
    return 0;
}

Conclusión

Las pruebas y la depuración son habilidades esenciales para cualquier programador. Al dominar estas técnicas, puedes asegurar que tus programas en C sean confiables, eficientes y libres de errores. Comienza con la depuración básica usando printf y GDB, luego pasa a escribir pruebas unitarias y usar herramientas avanzadas como Valgrind. Finalmente, explora técnicas avanzadas como TDD y mocking para mejorar aún más tus capacidades de prueba y depuración.
