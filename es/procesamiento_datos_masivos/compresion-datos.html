Introducción a la Compresión de Datos

La compresión de datos es una técnica esencial en el procesamiento de datos masivos, ya que permite reducir el tamaño de los datos para facilitar su almacenamiento y transmisión. En este tema, exploraremos los conceptos fundamentales de la compresión de datos, los algoritmos más comunes y cómo aplicarlos en el contexto de grandes volúmenes de datos.

Conceptos Clave

  • Compresión sin pérdida: Técnica de compresión que permite recuperar los datos originales exactamente después de la descompresión. Ejemplos incluyen Huffman Coding y Lempel-Ziv-Welch (LZW).
  • Compresión con pérdida: Técnica de compresión que reduce el tamaño de los datos eliminando parte de la información, lo que resulta en una pérdida de calidad. Ejemplos incluyen JPEG y MP3.
  • Ratio de compresión: Medida de la eficacia de un algoritmo de compresión, calculada como el tamaño original dividido por el tamaño comprimido.
  • Entropía: Medida de la cantidad de información contenida en un conjunto de datos, que influye en la capacidad de compresión.

Algoritmos de Compresión

Compresión sin pérdida

  1. Huffman Coding

    • Algoritmo de compresión basado en la frecuencia de aparición de los caracteres.
    • Construye un árbol binario donde los caracteres más frecuentes tienen códigos más cortos.
    import heapq
from collections import defaultdict, Counter

def huffman_encoding(data):
    frequency = Counter(data)
    heap = [[weight, [symbol, ""]] for symbol, weight in frequency.items()]
    heapq.heapify(heap)
    while len(heap) > 1:
        lo = heapq.heappop(heap)
        hi = heapq.heappop(heap)
        for pair in lo[1:]:
            pair[1] = '0' + pair[1]
        for pair in hi[1:]:
            pair[1] = '1' + pair[1]
        heapq.heappush(heap, [lo[0] + hi[0]] + lo[1:] + hi[1:])
    return sorted(heapq.heappop(heap)[1:], key=lambda p: (len(p[-1]), p))

data = "this is an example for huffman encoding"
huff = huffman_encoding(data)
print("Symbol\tWeight\tHuffman Code")
for p in huff:
    print(f"{p[0]}\t{frequency[p[0]]}\t{p[1]}")

  2. Lempel-Ziv-Welch (LZW)

    • Algoritmo de compresión basado en diccionarios que reemplaza subcadenas repetidas con referencias a un diccionario.
    def lzw_compress(uncompressed):
    dict_size = 256
    dictionary = {chr(i): i for i in range(dict_size)}
    w = ""
    result = []
    for c in uncompressed:
        wc = w + c
        if wc in dictionary:
            w = wc
        else:
            result.append(dictionary[w])
            dictionary[wc] = dict_size
            dict_size += 1
            w = c
    if w:
        result.append(dictionary[w])
    return result

data = "TOBEORNOTTOBEORTOBEORNOT"
compressed = lzw_compress(data)
print("Compressed data:", compressed)

Compresión con pérdida

  1. JPEG (Joint Photographic Experts Group)

    • Algoritmo de compresión de imágenes que reduce el tamaño eliminando detalles que son menos perceptibles para el ojo humano.
    • Utiliza transformadas de coseno discreto (DCT) y cuantización.

  2. MP3 (MPEG-1 Audio Layer III)

    • Algoritmo de compresión de audio que elimina sonidos inaudibles para el oído humano.
    • Utiliza transformadas de Fourier y cuantización perceptual.

Comparación de Algoritmos

| Algoritmo | Tipo | Uso Común | Ventajas | Desventajas | |-----------------|---------------|------------------|-----------------------------------|----------------------------------| | Huffman Coding | Sin pérdida | Textos, archivos | Simple y eficiente | No siempre logra alta compresión | | LZW | Sin pérdida | Textos, imágenes | Buena compresión para datos repetitivos | Puede ser ineficiente para datos no repetitivos | | JPEG | Con pérdida | Imágenes | Alta compresión, buena calidad visual | Pérdida de calidad irreversible | | MP3 | Con pérdida | Audio | Alta compresión, buena calidad auditiva | Pérdida de calidad irreversible |

Conclusión

La compresión de datos es una herramienta poderosa en el procesamiento de datos masivos, permitiendo reducir significativamente el tamaño de los datos para su almacenamiento y transmisión. Comprender los diferentes algoritmos de compresión y sus aplicaciones es crucial para seleccionar la técnica adecuada según el tipo de datos y los requisitos específicos del proyecto.
