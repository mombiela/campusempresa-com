Introducción

En esta sección del curso sobre DevOps Básico, aprenderemos a configurar un pipeline de Integración Continua (CI). La Integración Continua es una práctica esencial en DevOps que permite a los equipos de desarrollo integrar su trabajo frecuentemente, lo que facilita la detección temprana de errores y mejora la calidad del software.

Conceptos Clave

  • Pipeline de CI: Un conjunto de procesos automatizados que permiten la integración continua del código.
  • Repositorio de Código: Lugar donde se almacena el código fuente, comúnmente en plataformas como GitHub, GitLab o Bitbucket.
  • Build: Proceso de compilación del código fuente.
  • Test: Ejecución de pruebas automatizadas para asegurar la calidad del código.
  • Deploy: Despliegue del código en un entorno de prueba o producción.

Herramientas Comunes para CI

| Herramienta | Descripción | |--------------|-----------------------------------------------------------------------------| | Jenkins | Una de las herramientas de CI más populares y ampliamente utilizada. | | Travis CI | Servicio de CI basado en la nube que se integra bien con GitHub. | | CircleCI | Plataforma de CI/CD que permite la automatización de todo el ciclo de vida. | | GitLab CI/CD | Integrado en GitLab, permite configurar pipelines directamente en el repositorio. |

Configuración Básica de un Pipeline de CI con Jenkins

Paso 1: Instalación de Jenkins

  1. Descargar Jenkins: Visita la página oficial de Jenkins y descarga la versión adecuada para tu sistema operativo.
  2. Instalar Jenkins: Sigue las instrucciones de instalación específicas para tu sistema operativo.
  3. Iniciar Jenkins: Una vez instalado, inicia Jenkins y accede a la interfaz web a través de http://localhost:8080.

Paso 2: Configuración Inicial

  1. Desbloquear Jenkins: Utiliza la contraseña inicial proporcionada durante la instalación.
  2. Instalar Plugins Sugeridos: Jenkins te sugerirá una lista de plugins esenciales. Instálalos para tener una configuración básica funcional.
  3. Crear un Usuario Administrador: Configura un usuario administrador para gestionar Jenkins.

Paso 3: Crear un Nuevo Job

  1. Nuevo Item: En la interfaz de Jenkins, selecciona "Nuevo Item".
  2. Nombre del Job: Asigna un nombre a tu job y selecciona "Freestyle project".
  3. Configuración del Repositorio: En la sección "Source Code Management", selecciona "Git" y proporciona la URL de tu repositorio.

Paso 4: Configuración del Pipeline

  1. Build Triggers: Configura los triggers para que el pipeline se ejecute automáticamente. Por ejemplo, puedes usar "GitHub hook trigger for GITScm polling".
  2. Build Steps: Añade pasos de construcción. Por ejemplo, si estás usando Maven, selecciona "Invoke top-level Maven targets" y proporciona los objetivos de Maven como clean install.
  3. Post-build Actions: Añade acciones post-construcción como "Publish JUnit test result report" para publicar los resultados de las pruebas.

Ejemplo de Configuración de un Job en Jenkins

pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/tu-repositorio.git'
            }
        }
        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Deploy') {
            steps {
                sh 'mvn deploy'
            }
        }
    }
    post {
        always {
            junit 'target/surefire-reports/*.xml'
        }
    }
}

Conclusión

La configuración de un pipeline de CI es un paso crucial en la implementación de prácticas DevOps. Utilizando herramientas como Jenkins, puedes automatizar la integración y despliegue de tu código, mejorando la eficiencia y calidad de tu desarrollo. En esta sección, hemos cubierto los conceptos básicos y proporcionado un ejemplo práctico para que puedas comenzar a configurar tu propio pipeline de CI.
