Introducción

En esta sección, exploraremos cómo construir APIs REST utilizando Scala. Comenzaremos con lo básico y gradualmente avanzaremos hacia temas más complejos. Al final de este tema, tendrás una comprensión sólida de cómo crear, gestionar y desplegar APIs REST en Scala.

Requisitos previos

Antes de sumergirnos en la construcción de APIs REST, asegúrate de tener:

  • Comprensión básica de la sintaxis de Scala y conceptos de programación funcional.
  • Familiaridad con los conceptos de HTTP y principios REST.
  • Scala y sbt (Scala Build Tool) instalados en tu máquina.

Configuración del proyecto

Para comenzar a construir una API REST, necesitamos configurar un proyecto Scala utilizando sbt.

  1. Crea un nuevo directorio para tu proyecto:

    mkdir scala-rest-api
cd scala-rest-api

  2. Inicializa un nuevo proyecto sbt:

    sbt new scala/scala-seed.g8

  3. Actualiza build.sbt para incluir las dependencias necesarias:

    name := "scala-rest-api"

version := "0.1"

scalaVersion := "2.13.6"

libraryDependencies ++= Seq(
  "com.typesafe.akka" %% "akka-http" % "10.2.6",
  "com.typesafe.akka" %% "akka-stream" % "2.6.16",
  "de.heikoseeberger" %% "akka-http-circe" % "1.37.0",
  "io.circe" %% "circe-generic" % "0.14.1"
)

Creación de una API REST simple

Comencemos creando una API REST simple que responda con un mensaje "¡Hola, Mundo!".

  1. Crea un nuevo objeto Scala para el servidor:

    // src/main/scala/com/example/HelloWorldServer.scala
package com.example

import akka.actor.ActorSystem
import akka.http.scaladsl.Http
import akka.http.scaladsl.server.Directives._
import akka.stream.ActorMaterializer

object HelloWorldServer {
  def main(args: Array[String]): Unit = {
    implicit val system: ActorSystem = ActorSystem("hello-world-system")
    implicit val materializer: ActorMaterializer = ActorMaterializer()
    implicit val executionContext = system.dispatcher

    val route =
      path("hello") {
        get {
          complete("¡Hola, Mundo!")
        }
      }

    Http().newServerAt("localhost", 8080).bind(route)
    println("Servidor en línea en http://localhost:8080/")
  }
}

  2. Ejecuta el servidor:

    sbt run

  3. Prueba la API navegando a http://localhost:8080/hello en tu navegador o usando curl:

    curl http://localhost:8080/hello

Manejo de datos JSON

Para manejar datos JSON, utilizaremos Circe, una biblioteca JSON para Scala.

  1. Actualiza la ruta para manejar respuestas JSON:

    // src/main/scala/com/example/JsonServer.scala
package com.example

import akka.actor.ActorSystem
import akka.http.scaladsl.Http
import akka.http.scaladsl.server.Directives._
import akka.stream.ActorMaterializer
import de.heikoseeberger.akkahttpcirce.FailFastCirceSupport._
import io.circe.generic.auto._

object JsonServer {
  case class Greeting(message: String)

  def main(args: Array[String]): Unit = {
    implicit val system: ActorSystem = ActorSystem("json-system")
    implicit val materializer: ActorMaterializer = ActorMaterializer()
    implicit val executionContext = system.dispatcher

    val route =
      path("greet") {
        get {
          complete(Greeting("¡Hola, Mundo!"))
        }
      }

    Http().newServerAt("localhost", 8080).bind(route)
    println("Servidor en línea en http://localhost:8080/")
  }
}

  2. Prueba la API JSON:

    curl http://localhost:8080/greet

Operaciones CRUD

Implementemos una API CRUD (Crear, Leer, Actualizar, Eliminar) simple para gestionar una lista de elementos.

  1. Define el modelo de datos y el almacenamiento en memoria:

    // src/main/scala/com/example/ItemServer.scala
package com.example

import akka.actor.ActorSystem
import akka.http.scaladsl.Http
import akka.http.scaladsl.model.StatusCodes
import akka.http.scaladsl.server.Directives._
import akka.stream.ActorMaterializer
import de.heikoseeberger.akkahttpcirce.FailFastCirceSupport._
import io.circe.generic.auto._

import scala.collection.mutable
import scala.concurrent.Future

object ItemServer {
  case class Item(id: Int, name: String, description: String)

  var items: mutable.Map[Int, Item] = mutable.Map()

  def main(args: Array[String]): Unit = {
    implicit val system: ActorSystem = ActorSystem("item-system")
    implicit val materializer: ActorMaterializer = ActorMaterializer()
    implicit val executionContext = system.dispatcher

    val route =
      pathPrefix("items") {
        concat(
          pathEnd {
            concat(
              get {
                complete(items.values.toList)
              },
              post {
                entity(as[Item]) { item =>
                  items += (item.id -> item)
                  complete(StatusCodes.Created, item)
                }
              }
            )
          },
          path(IntNumber) { id =>
            concat(
              get {
                items.get(id) match {
                  case Some(item) => complete(item)
                  case None => complete(StatusCodes.NotFound)
                }
              },
              put {
                entity(as[Item]) { item =>
                  items.update(id, item)
                  complete(StatusCodes.OK, item)
                }
              },
              delete {
                items.remove(id) match {
                  case Some(_) => complete(StatusCodes.NoContent)
                  case None => complete(StatusCodes.NotFound)
                }
              }
            )
          }
        )
      }

    Http().newServerAt("localhost", 8080).bind(route)
    println("Servidor en línea en http://localhost:8080/")
  }
}

  2. Prueba la API CRUD usando curl:

    # Crear un elemento
curl -X POST -H "Content-Type: application/json" -d '{"id":1,"name":"Item1","description":"First item"}' http://localhost:8080/items

# Obtener todos los elementos
curl http://localhost:8080/items

# Obtener un elemento específico
curl http://localhost:8080/items/1

# Actualizar un elemento
curl -X PUT -H "Content-Type: application/json" -d '{"id":1,"name":"Updated Item1","description":"Updated description"}' http://localhost:8080/items/1

# Eliminar un elemento
curl -X DELETE http://localhost:8080/items/1

Temas avanzados

Autenticación y autorización

Implementar autenticación y autorización es crucial para asegurar tu API. Puedes usar bibliotecas como akka-http con OAuth2 o JWT para este propósito.

Manejo de errores

El manejo adecuado de errores asegura que tu API proporcione mensajes de error y códigos de estado significativos. Usa ExceptionHandler y RejectionHandler en akka-http para gestionar errores.

Pruebas

Probar tu API es esencial para asegurar su fiabilidad. Usa bibliotecas como akka-http-testkit y ScalaTest para escribir pruebas unitarias e integrales para tu API.

Despliegue

El despliegue de tu API REST en Scala se puede hacer utilizando varios métodos, como Docker, Kubernetes o servicios en la nube como AWS y Heroku.

Conclusión

En este tema, cubrimos los conceptos básicos de la construcción de APIs REST en Scala utilizando akka-http. Comenzamos con la configuración de un proyecto, creando una API simple, manejando datos JSON e implementando operaciones CRUD. También tocamos temas avanzados como autenticación, manejo de errores, pruebas y despliegue. Con este conocimiento, ahora estás equipado para construir y desplegar APIs REST robustas en Scala.
