Introducción a SBT

SBT (Simple Build Tool) es una herramienta de construcción popular para proyectos en Scala. Se utiliza para compilar, ejecutar y probar código Scala, así como para gestionar dependencias y automatizar tareas.

Conceptos Clave

  • Definición de Construcción: La definición de construcción se escribe en un archivo llamado build.sbt.
  • Tareas: Acciones que SBT puede realizar, como compilar código o ejecutar pruebas.
  • Configuraciones: Opciones de configuración para tu construcción, como la versión de Scala o las dependencias de la biblioteca.
  • Plugins: Extensiones que añaden funcionalidad a SBT.

Configuración de SBT

Para comenzar con SBT, necesitas instalarlo y configurar una estructura básica de proyecto.

Instalación

  1. Descargar e Instalar SBT: Sigue las instrucciones en el sitio web oficial de SBT.
  2. Verificar Instalación: Abre una terminal y ejecuta: 
    sbt sbtVersion
    Esto debería imprimir la versión de SBT instalada.

Creación de un Nuevo Proyecto

  1. Crear Directorio del Proyecto: 
    mkdir my-scala-project
cd my-scala-project
  2. Crear Archivo de Definición de Construcción: Crea un archivo llamado build.sbt con el siguiente contenido: 
    name := "MyScalaProject"

version := "0.1"

scalaVersion := "2.13.6"
  3. Crear Directorio de Fuentes: 
    mkdir -p src/main/scala
mkdir -p src/test/scala

Gestión de Dependencias

Las dependencias son bibliotecas externas que tu proyecto necesita. Puedes agregarlas a tu archivo build.sbt.

Agregar Dependencias

  1. Encontrar la Dependencia: Busca la biblioteca en Maven Central.
  2. Agregar a la Definición de Construcción: Agrega la dependencia a tu archivo build.sbt: 
    libraryDependencies += "org.scalatest" %% "scalatest" % "3.2.9" % Test

Compilación y Ejecución de Código

SBT proporciona comandos para compilar y ejecutar tu código Scala.

Compilación de Código

  1. Comando de Compilación: En la terminal, ejecuta: 
    sbt compile
    Esto compila el código en el directorio src/main/scala.

Ejecución de Código

  1. Crear Objeto Principal: Crea un archivo src/main/scala/Main.scala: 
    object Main extends App {
  println("¡Hola, Scala!")
}
  2. Comando de Ejecución: En la terminal, ejecuta: 
    sbt run

Pruebas con SBT

Las pruebas son una parte esencial del desarrollo de software. SBT se integra bien con marcos de pruebas como ScalaTest.

Escritura de Pruebas

  1. Crear Archivo de Pruebas: Crea un archivo src/test/scala/MainSpec.scala: 
    import org.scalatest.flatspec.AnyFlatSpec

class MainSpec extends AnyFlatSpec {
  "El objeto Main" debería "imprimir ¡Hola, Scala!" en {
    assert(Main.main(Array()) == "¡Hola, Scala!")
  }
}

Ejecución de Pruebas

  1. Comando de Pruebas: En la terminal, ejecuta: 
    sbt test

Automatización de Tareas

SBT te permite automatizar tareas repetitivas usando tareas personalizadas y plugins.

Tareas Personalizadas

  1. Definir Tarea Personalizada: Agrega lo siguiente a tu build.sbt: 
    lazy val hello = taskKey[Unit]("Imprime '¡Hola, Mundo!'")

hello := {
  println("¡Hola, Mundo!")
}
  2. Ejecutar Tarea Personalizada: En la terminal, ejecuta: 
    sbt hello

Conclusión

SBT es una herramienta poderosa para gestionar proyectos en Scala. Simplifica el proceso de compilar, ejecutar y probar tu código, así como la gestión de dependencias y la automatización de tareas. Al dominar SBT, puedes mejorar significativamente tu productividad y optimizar tu flujo de trabajo de desarrollo.
