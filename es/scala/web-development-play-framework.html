Ejemplo: Configuración de un Nuevo Proyecto Play

sbt new playframework/play-scala-seed.g8
cd play-scala-seed
sbt run
  • Explicación: Esta secuencia de comandos inicializa un nuevo proyecto Play utilizando la plantilla seed de Play Scala y ejecuta el servidor de desarrollo.

Conceptos Básicos

  • Comprensión de la Arquitectura MVC

    • Modelo: Datos y lógica de negocio
    • Vista: Interfaz de usuario
    • Controlador: Manejo de solicitudes y generación de respuestas

  • Enrutamiento

    • Definición de rutas en conf/routes
    • Ejemplo: 
      GET     /                           controllers.HomeController.index
POST    /submit                     controllers.HomeController.submit
    • Explicación: Este archivo de rutas mapea solicitudes HTTP a acciones del controlador.

  • Controladores

    • Creación de un controlador básico
    • Ejemplo: 
      package controllers

import javax.inject._
import play.api.mvc._

@Singleton
class HomeController @Inject()(val controllerComponents: ControllerComponents) extends BaseController {

  def index() = Action { implicit request: Request[AnyContent] =>
    Ok("¡Hola, Play Framework!")
  }
}
    • Explicación: Este controlador maneja una solicitud GET a la URL raíz y responde con un mensaje de texto simple.

Trabajando con Vistas

  • Uso de Plantillas Twirl

    • Creación de una vista HTML básica
    • Ejemplo: 
      @(message: String)

<!DOCTYPE html>
<html>
  <head>
    <title>Bienvenido a Play</title>
  </head>
  <body>
    <h1>@message</h1>
  </body>
</html>
    • Explicación: Esta plantilla Twirl toma un parámetro message y lo muestra dentro de una estructura HTML.

  • Integración de Vistas con Controladores

    • Ejemplo: 
      def index() = Action { implicit request: Request[AnyContent] =>
  Ok(views.html.index("¡Bienvenido a Play Framework!"))
}
    • Explicación: Esta acción del controlador renderiza la vista index.scala.html con un mensaje.

Temas Avanzados

  • Manejo de Formularios

    • Creación y procesamiento de formularios
    • Ejemplo: 
      import play.api.data._
import play.api.data.Forms._

case class UserData(name: String, age: Int)

val userForm = Form(
  mapping(
    "name" -> nonEmptyText,
    "age" -> number(min = 0, max = 100)
  )(UserData.apply)(UserData.unapply)
)

  • Integración con Bases de Datos

    • Uso de Slick para operaciones de base de datos
    • Ejemplo: 
      import slick.jdbc.H2Profile.api._

case class User(id: Long, name: String, age: Int)

class Users(tag: Tag) extends Table[User](tag, "USERS") {
  def id = column[Long]("ID", O.PrimaryKey, O.AutoInc)
  def name = column[String]("NAME")
  def age = column[Int]("AGE")
  def * = (id, name, age) <> (User.tupled, User.unapply)
}

val users = TableQuery[Users]
    • Explicación: Esto define un esquema de tabla User utilizando Slick.

  • Programación Asíncrona

    • Uso de Futures en Play
    • Ejemplo: 
      import scala.concurrent.Future
import scala.concurrent.ExecutionContext.Implicits.global

def asyncAction() = Action.async {
  val futureResult: Future[String] = Future {
    // Simular una computación de larga duración
    Thread.sleep(1000)
    "Resultado Asíncrono"
  }
  futureResult.map(result => Ok(result))
}
    • Explicación: Esta acción del controlador demuestra cómo manejar operaciones asíncronas utilizando Futures de Scala.
