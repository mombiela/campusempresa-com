Introducción

En esta sección, exploraremos dos características poderosas de Scala: Clases Caso y Coincidencia de Patrones. Estas características son esenciales para escribir código conciso y legible, especialmente cuando se trata de estructuras de datos inmutables y manipulaciones de datos complejas.

Clases Caso

¿Qué son las Clases Caso?

Las Clases Caso son un tipo especial de clase en Scala que están optimizadas para su uso en coincidencia de patrones. Son inmutables por defecto y vienen con varias características integradas que las hacen muy convenientes de usar.

Características Clave de las Clases Caso

  • Inmutabilidad: Los campos son inmutables por defecto.
  • toString, equals y hashCode Automáticos: Estos métodos se generan automáticamente.
  • Coincidencia de Patrones: Las clases caso pueden descomponerse usando coincidencia de patrones.
  • Método Copy: Proporciona una forma conveniente de crear copias modificadas de instancias.
  • Método apply Predeterminado: Permite la instanciación sin la palabra clave new.

Definiendo una Clase Caso

case class Person(name: String, age: Int)

En este ejemplo, Person es una clase caso con dos campos: name y age.

Usando Clases Caso

val alice = Person("Alice", 25)
val bob = Person("Bob", 30)

// Accediendo a los campos
println(alice.name)  // Salida: Alice
println(bob.age)     // Salida: 30

// Copiando con modificaciones
val olderAlice = alice.copy(age = 26)
println(olderAlice)  // Salida: Person(Alice,26)

Coincidencia de Patrones

¿Qué es la Coincidencia de Patrones?

La Coincidencia de Patrones es un mecanismo para verificar un valor contra un patrón. Es una versión más poderosa de la declaración switch que se encuentra en muchos otros lenguajes. Permite descomponer estructuras de datos y ejecutar código basado en su forma y contenido.

Sintaxis Básica

value match {
  case pattern1 => result1
  case pattern2 => result2
  // más casos
  case _ => defaultResult
}

Ejemplo con Clases Caso

def greet(person: Person): String = person match {
  case Person("Alice", _) => "¡Hola, Alice!"
  case Person(_, age) if age < 18 => "¡Hola, jovencito!"
  case Person(name, age) => s"¡Hola, $name! Tienes $age años."
}

val alice = Person("Alice", 25)
val bob = Person("Bob", 17)
val charlie = Person("Charlie", 30)

println(greet(alice))    // Salida: ¡Hola, Alice!
println(greet(bob))      // Salida: ¡Hola, jovencito!
println(greet(charlie))  // Salida: ¡Hola, Charlie! Tienes 30 años.

Coincidencia de Patrones con Rasgos Sellados

Los rasgos sellados aseguran que todas las implementaciones se conozcan en tiempo de compilación, lo que hace que la coincidencia de patrones sea exhaustiva y segura.

sealed trait Shape
case class Circle(radius: Double) extends Shape
case class Rectangle(width: Double, height: Double) extends Shape

def area(shape: Shape): Double = shape match {
  case Circle(radius) => Math.PI * radius * radius
  case Rectangle(width, height) => width * height
}

val circle = Circle(5.0)
val rectangle = Rectangle(4.0, 6.0)

println(area(circle))      // Salida: 78.53981633974483
println(area(rectangle))   // Salida: 24.0

Ejercicios

Ejercicio 1: Definir y Usar una Clase Caso

  1. Define una clase caso Book con los campos title (String) y author (String).
  2. Crea una instancia de Book.
  3. Imprime el title y author del libro.

Ejercicio 2: Coincidencia de Patrones con Clases Caso

  1. Define un rasgo sellado Animal y las clases caso Dog y Cat que extienden Animal.
  2. Escribe una función describe que tome un Animal y devuelva una descripción basada en el tipo de animal.
  3. Prueba la función con instancias de Dog y Cat.

Conclusión

Las Clases Caso y la Coincidencia de Patrones son características fundamentales en Scala que permiten un código conciso y expresivo. Las Clases Caso proporcionan una forma conveniente de definir estructuras de datos inmutables, mientras que la Coincidencia de Patrones permite una elegante descomposición de datos y flujo de control. Al dominar estas características, puedes escribir código Scala más legible y mantenible.
