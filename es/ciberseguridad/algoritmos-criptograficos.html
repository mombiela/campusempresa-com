Introducción a los Algoritmos Criptográficos

Los algoritmos criptográficos son fundamentales en la ciberseguridad, ya que permiten proteger la información mediante técnicas de cifrado y descifrado. Estos algoritmos aseguran la confidencialidad, integridad y autenticidad de los datos.

Tipos de Algoritmos Criptográficos

Algoritmos Simétricos

  • Definición: Utilizan la misma clave para cifrar y descifrar la información.
  • Ventajas:
    • Rápidos y eficientes en términos de procesamiento.
    • Menor consumo de recursos.
  • Desventajas:
    • Distribución segura de la clave es un desafío.
    • Menos seguros si la clave es comprometida.
  • Ejemplos:
    • AES (Advanced Encryption Standard): 
      from Crypto.Cipher import AES
from Crypto.Random import get_random_bytes

data = b'Secret Message'
key = get_random_bytes(16)  # 128-bit key
cipher = AES.new(key, AES.MODE_EAX)
nonce = cipher.nonce
ciphertext, tag = cipher.encrypt_and_digest(data)

print(f"Ciphertext: {ciphertext}")

Algoritmos Asimétricos

  • Definición: Utilizan un par de claves, una pública y una privada.
  • Ventajas:
    • No es necesario compartir la clave privada.
    • Mejor para la distribución de claves.
  • Desventajas:
    • Más lentos y requieren más recursos.
  • Ejemplos:
    • RSA (Rivest-Shamir-Adleman): 
      from Crypto.PublicKey import RSA
from Crypto.Cipher import PKCS1_OAEP

key = RSA.generate(2048)
public_key = key.publickey().export_key()
private_key = key.export_key()

cipher = PKCS1_OAEP.new(RSA.import_key(public_key))
ciphertext = cipher.encrypt(b'Secret Message')

print(f"Ciphertext: {ciphertext}")

Comparación de Algoritmos Simétricos y Asimétricos

| Característica | Algoritmos Simétricos | Algoritmos Asimétricos | |------------------------|-----------------------|------------------------| | Claves | Una clave compartida | Par de claves (pública y privada) | | Velocidad | Rápidos | Lentos | | Seguridad | Menos seguros si la clave es comprometida | Más seguros para la distribución de claves | | Uso común | Cifrado de grandes volúmenes de datos | Intercambio de claves, firmas digitales |

Algoritmos de Hash

  • Definición: Transforman datos de entrada en una cadena de longitud fija, que actúa como una "huella digital".
  • Propiedades:
    • Determinismo: La misma entrada siempre produce la misma salida.
    • Resistencia a colisiones: Es difícil encontrar dos entradas diferentes que produzcan la misma salida.
    • Resistencia a preimagen: Es difícil revertir el hash para obtener la entrada original.
  • Ejemplos:
    • SHA-256 (Secure Hash Algorithm 256-bit): 
      import hashlib

data = b'Secret Message'
hash_object = hashlib.sha256(data)
hex_dig = hash_object.hexdigest()

print(f"SHA-256 Hash: {hex_dig}")

Ejercicios Prácticos

  1. Cifrado y Descifrado con AES:

    • Crea un script que cifre y descifre un mensaje utilizando AES.
    • Asegúrate de manejar correctamente la clave y el nonce.

  2. Generación de Claves RSA:

    • Escribe un programa que genere un par de claves RSA y las guarde en archivos.
    • Implementa el cifrado y descifrado de un mensaje utilizando las claves generadas.

  3. Generación de Hash con SHA-256:

    • Desarrolla un script que tome una entrada de usuario y genere su hash utilizando SHA-256.
    • Verifica que el mismo mensaje siempre produzca el mismo hash.

Conclusión

Los algoritmos criptográficos son esenciales para proteger la información en el ámbito de la ciberseguridad. Comprender las diferencias entre los algoritmos simétricos y asimétricos, así como la importancia de los algoritmos de hash, es crucial para implementar soluciones de seguridad efectivas. A través de la práctica con ejemplos de código y ejercicios, los profesionales de TI pueden fortalecer sus habilidades en criptografía y mejorar la seguridad de los sistemas que administran.
