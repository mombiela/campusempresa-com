Introducción

El Proyecto de Seguridad de Aplicaciones Web Abiertas (OWASP, por sus siglas en inglés) es una organización sin fines de lucro que se dedica a mejorar la seguridad del software. OWASP proporciona una serie de recursos, incluyendo documentación, herramientas y estándares, que son esenciales para los profesionales de TI. Este curso está diseñado para ofrecer una comprensión profunda de los conceptos y prácticas de seguridad promovidos por OWASP.

Objetivos del Curso

  • Comprender los principios fundamentales de la seguridad de aplicaciones web.
  • Familiarizarse con las herramientas y recursos proporcionados por OWASP.
  • Aprender a identificar y mitigar las vulnerabilidades comunes en aplicaciones web.
  • Aplicar buenas prácticas de seguridad en el desarrollo y mantenimiento de software.

Contenido del Curso

1. Introducción a OWASP

  • Historia y Misión de OWASP
    • Fundada en 2001.
    • Misión: Mejorar la seguridad del software.
  • Principales Proyectos de OWASP
    • OWASP Top Ten.
    • OWASP ASVS (Application Security Verification Standard).
    • OWASP ZAP (Zed Attack Proxy).

2. OWASP Top Ten

  • Descripción General
    • Lista de las diez vulnerabilidades más críticas en aplicaciones web.
  • Vulnerabilidades Comunes
    • Inyección (SQL, NoSQL, OS, etc.).
    • Autenticación Rota.
    • Exposición de Datos Sensibles.
    • Entidades Externas XML (XXE).
    • Control de Acceso Roto.
    • Configuración Incorrecta de Seguridad.
    • Cross-Site Scripting (XSS).
    • Deserialización Insegura.
    • Uso de Componentes con Vulnerabilidades Conocidas.
    • Registro y Monitoreo Insuficientes.

3. Herramientas de OWASP

  • OWASP ZAP (Zed Attack Proxy)
    • Herramienta de prueba de penetración.
    • Ejemplo de uso: 
      zap.sh -daemon -port 8080 -config api.key=12345
  • OWASP Dependency-Check
    • Herramienta para identificar vulnerabilidades en dependencias.
    • Ejemplo de uso: 
      dependency-check --project "My Project" --scan /path/to/project
  • OWASP ASVS (Application Security Verification Standard)
    • Estándar para verificar la seguridad de aplicaciones.
    • Niveles de verificación: Nivel 1, Nivel 2, Nivel 3.

4. Talleres Prácticos

  • Taller de Identificación de Vulnerabilidades
    • Uso de OWASP ZAP para identificar y explotar vulnerabilidades.
  • Taller de Mitigación de Vulnerabilidades
    • Aplicación de parches y configuraciones de seguridad.
    • Ejemplo de mitigación de SQL Injection: 
      # Uso de consultas preparadas en Python
cursor.execute("SELECT * FROM users WHERE username = %s", (username,))
  • Taller de Implementación de Buenas Prácticas
    • Implementación de controles de acceso.
    • Ejemplo de control de acceso en una aplicación web: 
      @PreAuthorize("hasRole('ADMIN')")
public String adminPage() {
    return "admin";
}

Conclusión

Este curso proporciona una base sólida en los principios y prácticas de seguridad promovidos por OWASP. Al finalizar, los participantes estarán equipados con el conocimiento y las habilidades necesarias para identificar y mitigar vulnerabilidades en aplicaciones web, utilizando las herramientas y recursos proporcionados por OWASP. La seguridad es un proceso continuo, y este curso es un paso importante hacia la creación de aplicaciones más seguras y robustas.
