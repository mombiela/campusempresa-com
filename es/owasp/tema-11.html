Introducción

La inyección es una de las vulnerabilidades más críticas y comunes en aplicaciones web. Ocurre cuando un atacante puede enviar datos no confiables a un intérprete como parte de un comando o consulta. Los datos maliciosos pueden engañar al intérprete para ejecutar comandos no intencionados o acceder a datos sin autorización.

Tipos Comunes de Inyección

  • SQL Injection (SQLi): Inyección de comandos SQL en consultas a bases de datos.
  • Command Injection: Inyección de comandos del sistema operativo.
  • LDAP Injection: Inyección de comandos en consultas LDAP.
  • XPath Injection: Inyección en consultas XPath para XML.
  • NoSQL Injection: Inyección en bases de datos NoSQL.

Ejemplo de SQL Injection

Código Vulnerable

# Ejemplo en Python con una consulta SQL vulnerable
import sqlite3

def get_user_info(user_id):
    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()
    query = f"SELECT * FROM users WHERE id = {user_id}"
    cursor.execute(query)
    return cursor.fetchall()

Explicación

En el ejemplo anterior, la variable user_id se inserta directamente en la consulta SQL sin ninguna validación o sanitización. Un atacante podría explotar esta vulnerabilidad pasando un valor malicioso como 1 OR 1=1, lo que resultaría en la ejecución de la consulta:

SELECT * FROM users WHERE id = 1 OR 1=1

Esto devolvería todos los registros de la tabla users.

Prevención de Inyección

Uso de Consultas Preparadas

Las consultas preparadas (también conocidas como consultas parametrizadas) son una técnica efectiva para prevenir inyecciones SQL. Aquí hay un ejemplo de cómo modificar el código anterior para usar consultas preparadas:

# Ejemplo en Python usando consultas preparadas
import sqlite3

def get_user_info(user_id):
    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()
    query = "SELECT * FROM users WHERE id = ?"
    cursor.execute(query, (user_id,))
    return cursor.fetchall()

Validación y Sanitización de Entradas

  • Validación: Asegúrate de que los datos de entrada cumplan con los formatos esperados antes de procesarlos.
  • Sanitización: Limpia los datos de entrada para eliminar caracteres peligrosos.

Uso de ORM (Object-Relational Mapping)

Los ORM pueden ayudar a prevenir inyecciones SQL al abstraer las consultas SQL y manejar automáticamente la sanitización de entradas.

# Ejemplo en Python usando SQLAlchemy ORM
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from models import User

engine = create_engine('sqlite:///example.db')
Session = sessionmaker(bind=engine)
session = Session()

def get_user_info(user_id):
    user = session.query(User).filter(User.id == user_id).first()
    return user

Ejercicios Prácticos

  1. Identificación de Inyección: Dado un fragmento de código, identifica si es vulnerable a inyección y explica por qué.
  2. Corrección de Vulnerabilidades: Modifica un código vulnerable para que sea seguro utilizando consultas preparadas.
  3. Implementación de Validación: Implementa funciones de validación y sanitización para entradas de usuario en un formulario web.

Conclusión

La inyección es una vulnerabilidad crítica que puede tener consecuencias devastadoras para la seguridad de una aplicación. Es esencial que los desarrolladores comprendan cómo identificar y mitigar estas vulnerabilidades mediante el uso de consultas preparadas, validación y sanitización de entradas, y el uso de ORM. La práctica constante y la revisión de código son claves para mantener aplicaciones seguras.
