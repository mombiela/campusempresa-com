Introducción

En esta sección del curso sobre OWASP, nos enfocaremos en ejercicios prácticos de seguridad. Estos ejercicios están diseñados para ayudar a los profesionales de TI a identificar, entender y mitigar vulnerabilidades comunes en aplicaciones web. A través de ejemplos y ejercicios prácticos, los participantes podrán aplicar los conceptos aprendidos en situaciones del mundo real.

Ejercicio 1: Inyección SQL

Descripción

La inyección SQL es una de las vulnerabilidades más comunes y peligrosas en aplicaciones web. Este ejercicio te ayudará a identificar y mitigar esta vulnerabilidad.

Objetivos

  • Comprender cómo funciona la inyección SQL.
  • Aprender a identificar puntos vulnerables en una aplicación web.
  • Implementar medidas de mitigación para prevenir inyecciones SQL.

Ejemplo de Código

# Código vulnerable a inyección SQL
import sqlite3

def get_user_data(user_id):
    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()
    query = f"SELECT * FROM users WHERE id = {user_id}"
    cursor.execute(query)
    return cursor.fetchall()

# Uso del código vulnerable
user_data = get_user_data("1 OR 1=1")
print(user_data)

Ejercicio

  1. Identifica el problema en el código anterior.
  2. Modifica el código para usar consultas parametrizadas y prevenir la inyección SQL.

Solución

# Código seguro usando consultas parametrizadas
import sqlite3

def get_user_data(user_id):
    conn = sqlite3.connect('example.db')
    cursor = conn.cursor()
    query = "SELECT * FROM users WHERE id = ?"
    cursor.execute(query, (user_id,))
    return cursor.fetchall()

# Uso del código seguro
user_data = get_user_data(1)
print(user_data)

Ejercicio 2: Cross-Site Scripting (XSS)

Descripción

El Cross-Site Scripting (XSS) permite a los atacantes inyectar scripts maliciosos en páginas web vistas por otros usuarios. Este ejercicio te ayudará a identificar y mitigar esta vulnerabilidad.

Objetivos

  • Comprender cómo funciona el XSS.
  • Aprender a identificar puntos vulnerables en una aplicación web.
  • Implementar medidas de mitigación para prevenir XSS.

Ejemplo de Código

<!-- Código vulnerable a XSS -->
<html>
<body>
    <form method="GET">
        <input type="text" name="user_input">
        <input type="submit" value="Submit">
    </form>
    <div>
        <?php echo $_GET['user_input']; ?>
    </div>
</body>
</html>

Ejercicio

  1. Identifica el problema en el código anterior.
  2. Modifica el código para escapar adecuadamente el input del usuario y prevenir XSS.

Solución

<!-- Código seguro usando escape de caracteres -->
<html>
<body>
    <form method="GET">
        <input type="text" name="user_input">
        <input type="submit" value="Submit">
    </form>
    <div>
        <?php echo htmlspecialchars($_GET['user_input'], ENT_QUOTES, 'UTF-8'); ?>
    </div>
</body>
</html>

Ejercicio 3: Autenticación y Gestión de Sesiones

Descripción

La autenticación y la gestión de sesiones son cruciales para la seguridad de una aplicación web. Este ejercicio te ayudará a implementar prácticas seguras para la autenticación y la gestión de sesiones.

Objetivos

  • Comprender los principios de la autenticación segura.
  • Aprender a gestionar sesiones de manera segura.
  • Implementar medidas de mitigación para prevenir ataques relacionados con la autenticación y las sesiones.

Ejemplo de Código

# Código vulnerable a ataques de sesión
from flask import Flask, request, session

app = Flask(__name__)
app.secret_key = 'super_secret_key'

@app.route('/login', methods=['POST'])
def login():
    username = request.form['username']
    password = request.form['password']
    if authenticate(username, password):
        session['username'] = username
        return "Logged in"
    return "Login failed"

def authenticate(username, password):
    # Autenticación simple (no segura)
    return username == "admin" and password == "password"

if __name__ == '__main__':
    app.run()

Ejercicio

  1. Identifica los problemas en el código anterior.
  2. Modifica el código para usar prácticas seguras de autenticación y gestión de sesiones.

Solución

# Código seguro usando prácticas seguras de autenticación y gestión de sesiones
from flask import Flask, request, session, redirect, url_for
from werkzeug.security import check_password_hash, generate_password_hash

app = Flask(__name__)
app.secret_key = 'super_secret_key'

# Simulación de una base de datos de usuarios
users = {
    "admin": generate_password_hash("password")
}

@app.route('/login', methods=['POST'])
def login():
    username = request.form['username']
    password = request.form['password']
    if username in users and check_password_hash(users[username], password):
        session['username'] = username
        return redirect(url_for('dashboard'))
    return "Login failed"

@app.route('/dashboard')
def dashboard():
    if 'username' in session:
        return f"Welcome {session['username']}!"
    return redirect(url_for('login'))

if __name__ == '__main__':
    app.run()
