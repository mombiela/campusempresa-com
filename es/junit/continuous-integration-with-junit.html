Integración Continua con JUnit
Escribiendo Pruebas Mantenibles ►

La Integración Continua (CI) es una práctica de desarrollo donde los desarrolladores integran código en un repositorio compartido frecuentemente, idealmente varias veces al día. Cada integración puede ser verificada por una compilación automatizada y pruebas automatizadas, permitiendo a los equipos detectar problemas temprano.

En esta sección, exploraremos cómo integrar pruebas JUnit en una tubería de CI.

Conceptos Clave de la Integración Continua

  • Compilaciones Automatizadas: Compilar y construir automáticamente el proyecto cada vez que se realicen cambios.
  • Pruebas Automatizadas: Ejecutar pruebas automáticamente para asegurar que los nuevos cambios en el código no rompan la funcionalidad existente.
  • Sistemas de Control de Versiones (VCS): Herramientas como Git para gestionar cambios en el código.
  • Servidores de CI: Herramientas como Jenkins, Travis CI o GitHub Actions para automatizar el proceso de compilación y prueba.

Configuración de una Tubería de CI con JUnit

Paso 1: Sistema de Control de Versiones

Asegúrate de que tu proyecto esté bajo control de versiones. Para este ejemplo, usaremos Git.

  1. Inicializa un repositorio Git:

    git init

  2. Añade los archivos de tu proyecto y haz un commit:

    git add .
git commit -m "Commit inicial"

Paso 2: Escribir Pruebas JUnit

Asegúrate de tener pruebas JUnit en tu proyecto. Aquí hay un ejemplo simple:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        assertEquals(5, calculator.add(2, 3));
    }
}

Paso 3: Elegir un Servidor de CI

Hay varios servidores de CI disponibles. Usaremos GitHub Actions para este ejemplo.

Paso 4: Configuración de GitHub Actions

  1. Crea un directorio .github/workflows en tu proyecto.

  2. Crea un nuevo archivo de flujo de trabajo, por ejemplo, ci.yml:

    name: Java CI with Maven

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:

    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up JDK 11
      uses: actions/setup-java@v2
      with:
        java-version: '11'

    - name: Build with Maven
      run: mvn clean install

    - name: Run tests
      run: mvn test

Paso 5: Subir Cambios para Activar CI

  1. Añade el archivo de flujo de trabajo a Git:

    git add .github/workflows/ci.yml
git commit -m "Añadir flujo de trabajo de CI"

  2. Sube los cambios al repositorio:

    git push origin main

GitHub Actions ahora ejecutará automáticamente la tubería de CI cada vez que se suban cambios a la rama main o se haga una solicitud de extracción.

Conclusión

Integrar pruebas JUnit en una tubería de CI asegura que tu código sea probado continuamente, proporcionando retroalimentación inmediata sobre cualquier problema introducido por nuevos cambios. Esta práctica ayuda a mantener la calidad del código y reduce el riesgo de errores en producción.

Siguiendo los pasos descritos anteriormente, puedes configurar una tubería de CI usando GitHub Actions para construir y probar automáticamente tu proyecto Java con JUnit. Esta configuración puede adaptarse a otros servidores de CI como Jenkins o Travis CI con modificaciones menores.
