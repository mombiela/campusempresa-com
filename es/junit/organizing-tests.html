En esta sección, exploraremos cómo organizar eficazmente tus pruebas cuando usas JUnit. La organización adecuada de las pruebas es crucial para mantener una base de código limpia, comprensible y escalable. Cubriremos las mejores prácticas, estructuras de directorios, convenciones de nombres y el uso de características de JUnit para mantener tus pruebas bien organizadas.

Mejores Prácticas para Organizar Pruebas

  • Separación de Responsabilidades: Mantén tu código de prueba separado de tu código de producción.
  • Convenciones de Nombres Consistentes: Usa nombres claros y consistentes para las clases y métodos de prueba.
  • Pruebas Modulares: Escribe pruebas pequeñas y enfocadas que prueben una cosa a la vez.
  • Uso de Anotaciones: Aprovecha las anotaciones de JUnit para gestionar el ciclo de vida y la organización de las pruebas.

Estructura de Directorios

Una estructura de directorios bien organizada ayuda a localizar y gestionar fácilmente los archivos de prueba. Una estructura común es:

project-root/
├── src/
│   ├── main/
│   │   └── java/
│   │       └── com/
│   │           └── example/
│   │               └── MyClass.java
│   └── test/
│       └── java/
│           └── com/
│               └── example/
│                   └── MyClassTest.java
  • src/main/java: Contiene tu código de producción.
  • src/test/java: Contiene tu código de prueba.

Convenciones de Nombres

  • Nombres de Clases de Prueba: Deben reflejar la clase que están probando. Por ejemplo, MyClass debería tener una clase de prueba correspondiente llamada MyClassTest.
  • Nombres de Métodos de Prueba: Deben describir el comportamiento que se está probando. Usa un patrón como nombreDelMétodo_condición_resultadoEsperado.

Ejemplo

public class MyClassTest {

    @Test
    public void add_twoPositiveNumbers_returnsPositiveSum() {
        MyClass myClass = new MyClass();
        int result = myClass.add(2, 3);
        assertEquals(5, result);
    }

    @Test
    public void add_negativeAndPositiveNumber_returnsCorrectSum() {
        MyClass myClass = new MyClass();
        int result = myClass.add(-2, 3);
        assertEquals(1, result);
    }
}

Uso de Anotaciones de JUnit

JUnit proporciona varias anotaciones para ayudar a organizar y gestionar tus pruebas:

  • @Test: Marca un método como un método de prueba.
  • @BeforeEach: Se ejecuta antes de cada método de prueba.
  • @AfterEach: Se ejecuta después de cada método de prueba.
  • @BeforeAll: Se ejecuta una vez antes de todos los métodos de prueba en la clase.
  • @AfterAll: Se ejecuta una vez después de todos los métodos de prueba en la clase.
  • @Disabled: Desactiva temporalmente un método o clase de prueba.

Ejemplo

public class MyClassTest {

    private MyClass myClass;

    @BeforeAll
    public static void initAll() {
        System.out.println("Antes de todas las pruebas");
    }

    @BeforeEach
    public void init() {
        myClass = new MyClass();
    }

    @Test
    public void add_twoPositiveNumbers_returnsPositiveSum() {
        int result = myClass.add(2, 3);
        assertEquals(5, result);
    }

    @Test
    public void add_negativeAndPositiveNumber_returnsCorrectSum() {
        int result = myClass.add(-2, 3);
        assertEquals(1, result);
    }

    @AfterEach
    public void tearDown() {
        System.out.println("Después de cada prueba");
    }

    @AfterAll
    public static void tearDownAll() {
        System.out.println("Después de todas las pruebas");
    }
}

Agrupación de Pruebas

JUnit 5 te permite agrupar pruebas usando clases anidadas y etiquetas:

  • Clases Anidadas: Organiza pruebas relacionadas dentro de clases anidadas.
  • Etiquetas: Usa etiquetas para categorizar pruebas y ejecutar grupos específicos de pruebas.

Ejemplo

@Tag("fast")
public class MyClassTest {

    @Nested
    @Tag("addition")
    class AdditionTests {

        @Test
        public void add_twoPositiveNumbers_returnsPositiveSum() {
            MyClass myClass = new MyClass();
            int result = myClass.add(2, 3);
            assertEquals(5, result);
        }

        @Test
        public void add_negativeAndPositiveNumber_returnsCorrectSum() {
            MyClass myClass = new MyClass();
            int result = myClass.add(-2, 3);
            assertEquals(1, result);
        }
    }

    @Nested
    @Tag("subtraction")
    class SubtractionTests {

        @Test
        public void subtract_twoPositiveNumbers_returnsPositiveDifference() {
            MyClass myClass = new MyClass();
            int result = myClass.subtract(5, 3);
            assertEquals(2, result);
        }
    }
}

Conclusión

Organizar pruebas en JUnit es esencial para mantener una base de código limpia y eficiente. Siguiendo las mejores prácticas, usando una estructura de directorios consistente y aprovechando las anotaciones y características de JUnit, puedes crear un conjunto de pruebas bien organizado que sea fácil de entender y mantener. Recuerda mantener tus pruebas pequeñas, enfocadas y modulares para asegurar que sean efectivas y confiables.
