En esta sección, exploraremos el concepto de ejecutores de pruebas personalizados en JUnit. Los ejecutores de pruebas son una parte integral de JUnit, responsables de ejecutar pruebas y reportar sus resultados. Aunque JUnit proporciona varios ejecutores de pruebas integrados, hay escenarios en los que podrías necesitar crear tu propio ejecutor de pruebas personalizado para cumplir con requisitos específicos.

¿Qué es un Ejecutor de Pruebas?

Un ejecutor de pruebas es un componente en JUnit que orquesta la ejecución de pruebas. Es responsable de:

  • Descubrir métodos de prueba.
  • Ejecutar métodos de prueba.
  • Reportar resultados de pruebas.

JUnit proporciona varios ejecutores de pruebas integrados, tales como:

  • BlockJUnit4ClassRunner: El ejecutor predeterminado para pruebas JUnit 4.
  • Parameterized: Un ejecutor para pruebas parametrizadas.
  • Suite: Un ejecutor para ejecutar múltiples clases de prueba juntas.

¿Por Qué Crear un Ejecutor de Pruebas Personalizado?

Crear un ejecutor de pruebas personalizado puede ser útil en varios escenarios, tales como:

  • Personalizar el proceso de ejecución de pruebas.
  • Integrar con otras herramientas o marcos.
  • Agregar registro o reporte personalizado.
  • Implementar políticas específicas de ejecución de pruebas.

Creando un Ejecutor de Pruebas Personalizado

Para crear un ejecutor de pruebas personalizado, necesitas extender la clase org.junit.runner.Runner y sobrescribir sus métodos. Aquí tienes una guía paso a paso:

  1. Extender la Clase Runner:

    import org.junit.runner.Runner;
import org.junit.runner.notification.RunNotifier;
import org.junit.runners.model.InitializationError;

public class CustomRunner extends Runner {
    private final Class<?> testClass;

    public CustomRunner(Class<?> testClass) throws InitializationError {
        this.testClass = testClass;
    }

    @Override
    public Description getDescription() {
        return Description.createTestDescription(testClass, "Custom Test Runner");
    }

    @Override
    public void run(RunNotifier notifier) {
        // Lógica personalizada de ejecución de pruebas
    }
}

  2. Sobrescribir el Método getDescription:

    • Este método proporciona una descripción de las pruebas que se están ejecutando.
    • Es utilizado por JUnit para mostrar nombres y estructura de pruebas.

  3. Sobrescribir el Método run:

    • Este método contiene la lógica para ejecutar las pruebas.
    • Puedes agregar lógica personalizada aquí para controlar cómo se ejecutan y reportan las pruebas.

Ejemplo: Ejecutor Personalizado con Registro

Vamos a crear un ejecutor personalizado que registre el inicio y el fin de cada método de prueba:

import org.junit.runner.Description;
import org.junit.runner.Runner;
import org.junit.runner.notification.RunNotifier;
import org.junit.runner.notification.RunListener;
import org.junit.runners.model.InitializationError;
import org.junit.runners.model.Statement;

public class LoggingRunner extends Runner {
    private final Class<?> testClass;

    public LoggingRunner(Class<?> testClass) throws InitializationError {
        this.testClass = testClass;
    }

    @Override
    public Description getDescription() {
        return Description.createTestDescription(testClass, "Logging Test Runner");
    }

    @Override
    public void run(RunNotifier notifier) {
        notifier.addListener(new RunListener() {
            @Override
            public void testStarted(Description description) throws Exception {
                System.out.println("Starting: " + description.getMethodName());
            }

            @Override
            public void testFinished(Description description) throws Exception {
                System.out.println("Finished: " + description.getMethodName());
            }
        });

        try {
            Object testInstance = testClass.getDeclaredConstructor().newInstance();
            for (java.lang.reflect.Method method : testClass.getMethods()) {
                if (method.isAnnotationPresent(org.junit.Test.class)) {
                    Statement statement = new Statement() {
                        @Override
                        public void evaluate() throws Throwable {
                            method.invoke(testInstance);
                        }
                    };
                    statement.evaluate();
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Usando el Ejecutor Personalizado

Para usar el ejecutor personalizado, anota tu clase de prueba con @RunWith:

import org.junit.Test;
import org.junit.runner.RunWith;

@RunWith(LoggingRunner.class)
public class ExampleTest {
    @Test
    public void testMethod1() {
        System.out.println("Executing testMethod1");
    }

    @Test
    public void testMethod2() {
        System.out.println("Executing testMethod2");
    }
}

Conclusión

Los ejecutores de pruebas personalizados en JUnit proporcionan una forma poderosa de controlar y extender el proceso de ejecución de pruebas. Al crear un ejecutor personalizado, puedes integrar con otras herramientas, agregar registro personalizado o implementar políticas específicas de ejecución de pruebas. Entender cómo crear y usar ejecutores de pruebas personalizados puede mejorar significativamente tus capacidades de prueba y permitirte adaptar el proceso de prueba a tus necesidades específicas.
