Introducción

La anotación @Test es una parte fundamental de JUnit, un popular marco de pruebas para Java. Marca un método como un método de prueba, permitiendo que JUnit lo reconozca y lo ejecute como parte del conjunto de pruebas. Entender cómo usar la anotación @Test es crucial para escribir pruebas unitarias efectivas.

Conceptos Clave

  • Anotación: Una forma de metadatos que proporciona datos sobre un programa pero no es parte del programa en sí.
  • Método de Prueba: Un método que contiene código para probar una funcionalidad específica de la aplicación.
  • JUnit: Un marco de pruebas utilizado para escribir y ejecutar pruebas automatizadas repetibles en Java.

Uso Básico

Para usar la anotación @Test, necesitas importarla desde la biblioteca de JUnit. Aquí hay un ejemplo simple:

import org.junit.Test;
import static org.junit.Assert.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
}

Explicación:

  • Declaraciones de Importación: import org.junit.Test; e import static org.junit.Assert.assertEquals; son necesarias para usar la anotación @Test y los métodos de aserción.
  • Método de Prueba: testAddition está marcado con @Test, indicando que es un método de prueba.
  • Aserción: assertEquals(5, result); verifica si el resultado de la suma es el esperado.

Uso Avanzado

Excepciones Esperadas

Puedes especificar que un método de prueba debe lanzar una excepción usando el parámetro expected.

@Test(expected = ArithmeticException.class)
public void testDivisionByZero() {
    Calculator calculator = new Calculator();
    calculator.divide(1, 0);
}

Explicación:

  • Excepción Esperada: La prueba pasará si se lanza una ArithmeticException durante la ejecución de testDivisionByZero.

Tiempo de Espera

Puedes especificar un tiempo de espera para un método de prueba para asegurar que se complete dentro de un cierto período.

@Test(timeout = 1000)
public void testPerformance() {
    Calculator calculator = new Calculator();
    for (int i = 0; i < 1000000; i++) {
        calculator.add(i, i);
    }
}

Explicación:

  • Tiempo de Espera: La prueba fallará si toma más de 1000 milisegundos en ejecutarse.

Combinando Excepciones Esperadas y Tiempo de Espera

Puedes combinar ambos parámetros expected y timeout en un solo método de prueba.

@Test(expected = IllegalArgumentException.class, timeout = 500)
public void testInvalidInput() {
    Calculator calculator = new Calculator();
    calculator.add(null, 5);
}

Explicación:

  • Parámetros Combinados: La prueba pasará si se lanza una IllegalArgumentException y el método se completa dentro de los 500 milisegundos.

Resumen

La anotación @Test es una herramienta poderosa en JUnit que te permite definir métodos de prueba fácilmente. Al entender su uso básico y avanzado, puedes escribir pruebas más efectivas y eficientes. Las características clave incluyen:

  • Marcar métodos como métodos de prueba.
  • Especificar excepciones esperadas.
  • Establecer tiempos de espera para los métodos de prueba.

Dominar la anotación @Test mejorará significativamente tu capacidad para escribir pruebas unitarias robustas, asegurando que tu código sea confiable y mantenible.
