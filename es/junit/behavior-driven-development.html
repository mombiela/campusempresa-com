El Desarrollo Guiado por Comportamiento (BDD) es una extensión del Desarrollo Guiado por Pruebas (TDD) que enfatiza la colaboración entre desarrolladores, QA y participantes no técnicos o de negocio en un proyecto de software. BDD fomenta la escritura de pruebas en un lenguaje natural que los no programadores puedan leer.

Introducción al BDD

  • Definición: BDD es un proceso de desarrollo de software que involucra ejemplos escritos en un lenguaje específico del dominio para describir el comportamiento de un sistema.
  • Propósito: Mejorar la comunicación entre las partes interesadas del proyecto y asegurar que el software cumpla con los requisitos del negocio.

Conceptos Clave del BDD

  • Historias de Usuario: Descripciones de características del software desde la perspectiva del usuario final.
  • Escenarios: Ejemplos específicos de cómo una característica debe comportarse en diferentes situaciones.
  • Sintaxis Dado-Cuando-Entonces: Una forma estructurada de escribir escenarios.

Sintaxis Dado-Cuando-Entonces

  • Dado: Describe el contexto inicial o estado del sistema.
  • Cuando: Describe la acción o evento.
  • Entonces: Describe el resultado o desenlace esperado.

BDD con JUnit

JUnit es un marco de pruebas popular para Java, y puede ser utilizado para implementar BDD escribiendo pruebas que sigan la estructura Dado-Cuando-Entonces.

Configuración de JUnit para BDD

  1. Agregar Dependencia de JUnit: Asegúrate de que JUnit esté incluido en tu proyecto. 
    <!-- Para Maven -->
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.7.0</version>
    <scope>test</scope>
</dependency>
  2. Crear Clases de Prueba: Organiza tus clases de prueba para reflejar las características y escenarios.

Escribir Pruebas BDD con JUnit

Consideremos un ejemplo de una aplicación bancaria simple donde queremos probar el comportamiento de una característica de depósito.

Escenario de Ejemplo

Historia de Usuario: Como cliente del banco, quiero depositar dinero en mi cuenta para poder aumentar mi saldo.

Escenario: Depósito exitoso

  • Dado: El saldo de la cuenta es $100.
  • Cuando: El cliente deposita $50.
  • Entonces: El saldo de la cuenta debería ser $150.

Ejemplo de Código

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;

class BankAccountTest {
    private BankAccount account;

    @BeforeEach
    void setUp() {
        account = new BankAccount(100); // Dado: El saldo de la cuenta es $100
    }

    @Test
    void testDeposit() {
        account.deposit(50); // Cuando: El cliente deposita $50
        assertEquals(150, account.getBalance()); // Entonces: El saldo de la cuenta debería ser $150
    }
}

class BankAccount {
    private int balance;

    public BankAccount(int initialBalance) {
        this.balance = initialBalance;
    }

    public void deposit(int amount) {
        this.balance += amount;
    }

    public int getBalance() {
        return this.balance;
    }
}

Prácticas Avanzadas de BDD con JUnit

  • Pruebas Parametrizadas: Usa las pruebas parametrizadas de JUnit para ejecutar la misma prueba con diferentes entradas.
  • Aserciones Personalizadas: Crea métodos de aserción personalizados para mejorar la legibilidad y reutilización.
  • Simulación de Dependencias: Usa marcos de simulación como Mockito para aislar la unidad bajo prueba.

Ejemplo de Prueba Parametrizada

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

class BankAccountParameterizedTest {
    @ParameterizedTest
    @CsvSource({
        "100, 50, 150",
        "200, 100, 300",
        "50, 25, 75"
    })
    void testDeposit(int initialBalance, int depositAmount, int expectedBalance) {
        BankAccount account = new BankAccount(initialBalance);
        account.deposit(depositAmount);
        assertEquals(expectedBalance, account.getBalance());
    }
}

Conclusión

El Desarrollo Guiado por Comportamiento (BDD) con JUnit ayuda a cerrar la brecha entre las partes interesadas técnicas y no técnicas al escribir pruebas en un formato claro y comprensible. Siguiendo la estructura Dado-Cuando-Entonces, puedes asegurarte de que tus pruebas sean fáciles de leer y mantener, lo que en última instancia lleva a una mejor calidad del software y colaboración.
