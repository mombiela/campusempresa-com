En esta sección, exploraremos cómo crear y usar mocks en pruebas de JUnit. Los mocks son esenciales para aislar la unidad de trabajo y asegurar que las pruebas sean tanto confiables como rápidas. Cubriremos los conceptos básicos de la creación de mocks, cómo crear mocks usando bibliotecas populares como Mockito, y técnicas avanzadas para escenarios más complejos.

¿Qué son los Mocks?

Los mocks son objetos que simulan el comportamiento de objetos reales de manera controlada. Se utilizan en pruebas unitarias para aislar el código que se está probando de sus dependencias.

  • Aislamiento: Los mocks ayudan a aislar la unidad de trabajo de las dependencias externas.
  • Control: Los mocks permiten controlar el comportamiento de las dependencias.
  • Verificación: Los mocks pueden verificar interacciones entre la unidad de trabajo y sus dependencias.

¿Por qué Usar Mocks?

  • Velocidad: Los mocks son más rápidos que los objetos reales, especialmente cuando se trata de sistemas externos como bases de datos o servicios web.
  • Confiabilidad: Los mocks proporcionan un comportamiento consistente, haciendo que las pruebas sean más confiables.
  • Enfoque: Los mocks permiten centrarse en la unidad de trabajo sin preocuparse por el comportamiento de sus dependencias.

Creación de Mocks con Mockito

Mockito es una biblioteca popular para crear mocks en Java. Se integra perfectamente con JUnit y proporciona una API simple para crear y usar mocks.

Añadiendo Mockito a Tu Proyecto

Primero, necesitas añadir Mockito a tu proyecto. Si estás usando Maven, añade la siguiente dependencia a tu pom.xml:

<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>4.0.0</version>
    <scope>test</scope>
</dependency>

Creación Básica de Mocks

Aquí tienes un ejemplo simple de cómo crear y usar un mock con Mockito:

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        // Crear una instancia mock de UserRepository
        UserRepository mockRepository = mock(UserRepository.class);

        // Definir el comportamiento del mock
        when(mockRepository.findUserById(1)).thenReturn(new User(1, "John Doe"));

        // Usar el mock en el servicio
        UserService userService = new UserService(mockRepository);
        User user = userService.getUser(1);

        // Verificar el resultado
        assertEquals("John Doe", user.getName());
    }
}

Verificación de Interacciones

Mockito te permite verificar interacciones con el mock para asegurar que los métodos esperados fueron llamados:

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        UserRepository mockRepository = mock(UserRepository.class);
        when(mockRepository.findUserById(1)).thenReturn(new User(1, "John Doe"));

        UserService userService = new UserService(mockRepository);
        userService.getUser(1);

        // Verificar que findUserById fue llamado con el argumento correcto
        verify(mockRepository).findUserById(1);
    }
}

Técnicas Avanzadas de Mocking

Argument Matchers

Mockito proporciona argument matchers para permitir una verificación y stubbing más flexible:

import static org.mockito.Mockito.*;
import static org.mockito.ArgumentMatchers.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        UserRepository mockRepository = mock(UserRepository.class);
        when(mockRepository.findUserById(anyInt())).thenReturn(new User(1, "John Doe"));

        UserService userService = new UserService(mockRepository);
        User user = userService.getUser(1);

        assertEquals("John Doe", user.getName());
    }
}

Captura de Argumentos

A veces necesitas capturar los argumentos pasados a un método mock para hacer más afirmaciones:

import static org.mockito.Mockito.*;
import static org.mockito.ArgumentCaptor.*;
import org.junit.jupiter.api.Test;
import org.mockito.ArgumentCaptor;

public class UserServiceTest {

    @Test
    public void testGetUser() {
        UserRepository mockRepository = mock(UserRepository.class);
        UserService userService = new UserService(mockRepository);

        userService.getUser(1);

        ArgumentCaptor<Integer> captor = ArgumentCaptor.forClass(Integer.class);
        verify(mockRepository).findUserById(captor.capture());

        assertEquals(1, captor.getValue().intValue());
    }
}

Resumen

En esta sección, hemos cubierto los conceptos básicos de la creación y uso de mocks en pruebas de JUnit con Mockito. Los mocks son herramientas poderosas para aislar la unidad de trabajo, controlar el comportamiento de las dependencias y verificar interacciones. Al dominar estas técnicas, puedes escribir pruebas unitarias más confiables y enfocadas.

  • Mocks: Simulan el comportamiento de objetos reales.
  • Mockito: Una biblioteca popular para crear mocks.
  • Técnicas Básicas: Crear mocks, definir comportamiento y verificar interacciones.
  • Técnicas Avanzadas: Usar argument matchers y capturar argumentos.

Al incorporar estas prácticas en tu estrategia de pruebas, puedes mejorar la calidad y mantenibilidad de tu código.
