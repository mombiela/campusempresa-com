Introducción a las Extensiones de JUnit 5

Las extensiones de JUnit 5 te permiten extender la funcionalidad de JUnit 5 creando comportamientos personalizados que pueden ser reutilizados en múltiples clases de prueba. Esta característica es poderosa para crear lógica de prueba reutilizable, anotaciones personalizadas y más.

Conceptos Clave

  • Puntos de Extensión: JUnit 5 proporciona varios puntos de extensión que te permiten engancharte en el ciclo de vida de la prueba.
  • Modelo de Extensión: Las extensiones se implementan creando clases que implementan una o más de las interfaces de extensión proporcionadas por JUnit 5.
  • Registro: Las extensiones pueden registrarse de manera declarativa a través de anotaciones o programáticamente.

Puntos de Extensión Comunes

  • Post-Procesamiento de Instancias de Prueba: Modificar instancias de prueba después de que se crean.
  • Condición de Ejecución de Pruebas: Controlar si las pruebas deben ejecutarse o saltarse.
  • Resolución de Parámetros: Proporcionar parámetros personalizados a los métodos de prueba.
  • Callbacks de Ciclo de Vida: Engancharse en varias etapas del ciclo de vida de la prueba (por ejemplo, antes de todas las pruebas, antes de cada prueba).

Creando una Extensión Simple

Vamos a crear una extensión simple que registre mensajes antes y después de cada método de prueba.

Ejemplo Paso a Paso

  1. Definir la Clase de Extensión:

    import org.junit.jupiter.api.extension.BeforeEachCallback;
import org.junit.jupiter.api.extension.AfterEachCallback;
import org.junit.jupiter.api.extension.ExtensionContext;

public class LoggingExtension implements BeforeEachCallback, AfterEachCallback {
    @Override
    public void beforeEach(ExtensionContext context) {
        System.out.println("Antes del método de prueba: " + context.getDisplayName());
    }

    @Override
    public void afterEach(ExtensionContext context) {
        System.out.println("Después del método de prueba: " + context.getDisplayName());
    }
}

  2. Registrar la Extensión:

    • De manera Declarativa: Usar la anotación @ExtendWith.

      import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;

@ExtendWith(LoggingExtension.class)
public class MyTests {
    @Test
    void testExample() {
        System.out.println("Ejecutando testExample");
    }
}

    • Programáticamente: Usar la anotación @RegisterExtension.

      import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.RegisterExtension;

public class MyTests {
    @RegisterExtension
    static LoggingExtension loggingExtension = new LoggingExtension();

    @Test
    void testExample() {
        System.out.println("Ejecutando testExample");
    }
}

Explicación

  • Clase LoggingExtension: Implementa BeforeEachCallback y AfterEachCallback para registrar mensajes antes y después de cada método de prueba.
  • Anotación @ExtendWith: Registra la extensión de manera declarativa a nivel de clase.
  • Anotación @RegisterExtension: Registra la extensión programáticamente a nivel de campo.

Extensiones Avanzadas

Ejecución Condicional de Pruebas

Puedes crear extensiones para ejecutar pruebas condicionalmente basadas en lógica personalizada.

import org.junit.jupiter.api.extension.ExecutionCondition;
import org.junit.jupiter.api.extension.ExtensionContext;
import org.junit.jupiter.api.extension.ConditionEvaluationResult;

public class CustomCondition implements ExecutionCondition {
    @Override
    public ConditionEvaluationResult evaluateExecutionCondition(ExtensionContext context) {
        boolean condition = // tu lógica de condición personalizada
        return condition ? ConditionEvaluationResult.enabled("Condición cumplida") :
                           ConditionEvaluationResult.disabled("Condición no cumplida");
    }
}

Resolución de Parámetros

Puedes crear extensiones para proporcionar parámetros personalizados a los métodos de prueba.

import org.junit.jupiter.api.extension.ParameterContext;
import org.junit.jupiter.api.extension.ParameterResolver;
import org.junit.jupiter.api.extension.ExtensionContext;

public class CustomParameterResolver implements ParameterResolver {
    @Override
    public boolean supportsParameter(ParameterContext parameterContext, ExtensionContext extensionContext) {
        return parameterContext.getParameter().getType() == String.class;
    }

    @Override
    public Object resolveParameter(ParameterContext parameterContext, ExtensionContext extensionContext) {
        return "Parámetro Personalizado";
    }
}

Resumen

Las extensiones de JUnit 5 proporcionan una manera poderosa de extender la funcionalidad de tus pruebas. Al implementar varios puntos de extensión, puedes crear lógica de prueba reutilizable y modular. Las extensiones pueden registrarse de manera declarativa o programática, dándote flexibilidad en cómo las aplicas. Ya sea que necesites registrar mensajes, ejecutar pruebas condicionalmente o proporcionar parámetros personalizados, las extensiones de JUnit 5 ofrecen una solución robusta para mejorar tu marco de pruebas.
