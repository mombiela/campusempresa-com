La Integración Continua (CI) y el Despliegue Continuo (CD) son prácticas esenciales en el desarrollo de software moderno. Ayudan a garantizar que los cambios en el código se prueben y desplieguen automáticamente, lo que lleva a un software más confiable y mantenible. Este tema te guiará desde los conceptos básicos hasta los avanzados de CI/CD en el contexto del desarrollo de JavaScript.

Introducción a la Integración y Despliegue Continuos

  • Integración Continua (CI): La práctica de integrar frecuentemente cambios de código en un repositorio compartido, donde se ejecutan compilaciones y pruebas automatizadas.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio que pasa las pruebas automatizadas a producción.

Configuración de una Canalización CI/CD

Elección de una Herramienta CI/CD

Hay varias herramientas CI/CD disponibles, cada una con sus propias fortalezas. Algunas populares incluyen:

  • Jenkins
  • Travis CI
  • CircleCI
  • GitHub Actions

Ejemplo: Configuración de GitHub Actions para un Proyecto de JavaScript

  1. Crear un Repositorio en GitHub: Inicializa un nuevo repositorio en GitHub para tu proyecto de JavaScript.
  2. Agregar un Archivo de Flujo de Trabajo: Crea un directorio .github/workflows en la raíz de tu proyecto y agrega un archivo de flujo de trabajo, por ejemplo, ci.yml.
name: CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

Explicación del Archivo de Flujo de Trabajo

  • name: El nombre del flujo de trabajo.
  • on: Especifica los eventos que desencadenan el flujo de trabajo (por ejemplo, push, pull_request).
  • jobs: Define los trabajos a ejecutar.
    • runs-on: Especifica el entorno (por ejemplo, ubuntu-latest).
    • steps: Enumera los pasos a ejecutar en el trabajo.
      • Checkout code: Usa la acción actions/checkout para verificar el repositorio.
      • Set up Node.js: Usa la acción actions/setup-node para configurar el entorno de Node.js.
      • Install dependencies: Ejecuta npm install para instalar las dependencias del proyecto.
      • Run tests: Ejecuta npm test para ejecutar el conjunto de pruebas.

Escribir Pruebas para la Integración Continua

Pruebas Unitarias con Jest

Jest es un marco de pruebas popular para JavaScript. Aquí se explica cómo configurarlo y escribir una prueba simple.

  1. Instalar Jest:

    npm install --save-dev jest

  2. Agregar un Script de Prueba a package.json:

    {
  "scripts": {
    "test": "jest"
  }
}

  3. Escribir una Prueba Simple: Crea un archivo sum.js:

    function sum(a, b) {
  return a + b;
}
module.exports = sum;

    Crea un archivo de prueba sum.test.js:

    const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

  4. Ejecutar las Pruebas:

    npm test

Integrar Pruebas en el Flujo de Trabajo CI

Asegúrate de que tu archivo de flujo de trabajo CI incluya un paso para ejecutar las pruebas, como se muestra en el ejemplo de GitHub Actions anterior.

Despliegue Continuo

Desplegar en un Servicio de Alojamiento

Para aplicaciones JavaScript, los servicios de alojamiento populares incluyen:

  • Netlify
  • Vercel
  • Heroku

Ejemplo: Desplegar en Netlify

  1. Crear una Cuenta en Netlify: Regístrate en Netlify.
  2. Conectar tu Repositorio: Vincula tu repositorio de GitHub a Netlify.
  3. Configurar Configuraciones de Compilación: Establece el comando de compilación (por ejemplo, npm run build) y el directorio de publicación (por ejemplo, build).
  4. Desplegar: Netlify desplegará automáticamente tu aplicación cada vez que empujes cambios al repositorio.

Conclusión

La Integración y Despliegue Continuos son cruciales para mantener aplicaciones JavaScript de alta calidad. Al automatizar los procesos de prueba y despliegue, puedes asegurarte de que tu código esté siempre en un estado desplegable y que las nuevas características y correcciones se entreguen de manera rápida y confiable. Comienza configurando una canalización CI simple, escribe pruebas exhaustivas y avanza gradualmente hacia un despliegue continuo completo para optimizar tu flujo de trabajo de desarrollo.
