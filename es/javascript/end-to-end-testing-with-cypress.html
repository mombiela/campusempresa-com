Introducción a las pruebas de extremo a extremo

Las pruebas de extremo a extremo (E2E) son una metodología utilizada para probar si el flujo de una aplicación funciona como se diseñó de principio a fin. El propósito de las pruebas E2E es identificar dependencias del sistema y asegurar que la integridad de los datos se mantenga entre varios componentes y sistemas del sistema.

  • Definición: Probar el flujo completo de una aplicación de principio a fin.
  • Propósito: Asegurarse de que la aplicación se comporte como se espera en un escenario del mundo real.

Introducción a Cypress

Cypress es un marco de pruebas de extremo a extremo poderoso y moderno que facilita la escritura y ejecución de pruebas para tus aplicaciones web.

  • Características:
    • Viaje en el tiempo: Cypress toma instantáneas mientras se ejecutan tus pruebas.
    • Depurabilidad: Mensajes de error detallados y trazas de pila.
    • Espera automática: No es necesario agregar esperas o pausas a tus pruebas.
    • Recargas en tiempo real: Se recarga automáticamente cada vez que haces cambios en tus pruebas.
    • Control del tráfico de red: Simula y maqueta solicitudes de red.

Configuración de Cypress

Antes de que puedas comenzar a escribir pruebas con Cypress, necesitas configurarlo en tu proyecto.

Instalación

Para instalar Cypress, necesitas tener Node.js y npm instalados en tu máquina.

npm install cypress --save-dev

Apertura de Cypress

Después de la instalación, puedes abrir Cypress usando el siguiente comando:

npx cypress open

Este comando abrirá el Cypress Test Runner, donde puedes ver y ejecutar tus pruebas.

Escribiendo tu primera prueba

Vamos a escribir una prueba simple para verificar si una página web se carga correctamente.

Prueba de ejemplo

Crea un nuevo archivo en la carpeta cypress/integration, por ejemplo, example_spec.js.

describe('Mi primera prueba', () => {
  it('Visita el Kitchen Sink', () => {
    cy.visit('https://example.cypress.io')
    cy.contains('type').click()
    cy.url().should('include', '/commands/actions')
    cy.get('.action-email').type('fake@email.com')
      .should('have.value', 'fake@email.com')
  })
})

Explicación

  • describe: Agrupa pruebas relacionadas.
  • it: Define una sola prueba.
  • cy.visit: Navega a una URL.
  • cy.contains: Encuentra un elemento que contiene texto específico.
  • cy.url: Verifica la URL.
  • cy.get: Obtiene un elemento basado en un selector.
  • type: Escribe en un campo de entrada.
  • should: Realiza una afirmación.

Técnicas avanzadas de prueba

A medida que te sientas más cómodo con Cypress, puedes comenzar a usar características más avanzadas.

Comandos personalizados

Puedes crear comandos personalizados para reutilizar código y hacer tus pruebas más limpias.

Cypress.Commands.add('login', (email, password) => {
  cy.visit('/login')
  cy.get('input[name=email]').type(email)
  cy.get('input[name=password]').type(password)
  cy.get('button[type=submit]').click()
})

Fixtures

Los fixtures se utilizan para almacenar y gestionar datos de prueba.

// cypress/fixtures/user.json
{
  "email": "user@example.com",
  "password": "password123"
}
// Usando el fixture en una prueba
describe('Prueba de inicio de sesión', () => {
  beforeEach(() => {
    cy.fixture('user').as('userData')
  })

  it('Inicia sesión con credenciales válidas', function() {
    cy.visit('/login')
    cy.get('input[name=email]').type(this.userData.email)
    cy.get('input[name=password]').type(this.userData.password)
    cy.get('button[type=submit]').click()
    cy.url().should('include', '/dashboard')
  })
})

Mejores prácticas

  • Mantén las pruebas aisladas: Cada prueba debe ser independiente.
  • Usa nombres significativos: Nombra tus pruebas y comandos claramente.
  • Evita el hardcoding: Usa fixtures y variables de entorno.
  • Limpia el estado: Asegúrate de que las pruebas no dejen datos residuales.

Conclusión

Las pruebas de extremo a extremo con Cypress te permiten asegurarte de que tu aplicación web funcione como se espera desde la perspectiva del usuario. Al configurar Cypress, escribir pruebas básicas y avanzadas, y seguir las mejores prácticas, puedes crear pruebas robustas y confiables que ayuden a mantener la calidad de tu aplicación.

Cypress proporciona un conjunto rico de características que hacen que escribir y ejecutar pruebas sea sencillo y eficiente, lo que lo convierte en una excelente opción para los desarrolladores que buscan implementar pruebas E2E en sus proyectos.
