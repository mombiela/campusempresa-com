Introducción a la Concurrencia en JavaScript

JavaScript es un lenguaje de un solo hilo, lo que significa que tiene una pila de llamadas y un montón de memoria. Sin embargo, puede manejar operaciones concurrentes utilizando su modelo de concurrencia no bloqueante y basado en eventos. Esta sección te introducirá a los conceptos básicos de la concurrencia en JavaScript.

  • Naturaleza de un solo hilo: JavaScript ejecuta una pieza de código a la vez.
  • Bucle de eventos: Gestiona la ejecución de múltiples piezas de código a lo largo del tiempo, manejando operaciones asíncronas.
  • Cola de callbacks: Almacena callbacks que están listos para ser ejecutados.
  • APIs web: APIs proporcionadas por el navegador que manejan tareas asíncronas como solicitudes HTTP, temporizadores, etc.

El Bucle de Eventos

El bucle de eventos es el núcleo del modelo de concurrencia de JavaScript. Permite a JavaScript realizar operaciones no bloqueantes delegando tareas al entorno del navegador o Node.js.

  • Pila de llamadas: Donde se ejecuta el código.
  • Cola de callbacks: Contiene mensajes (callbacks) para ser procesados.
  • Bucle de eventos: Revisa continuamente la pila de llamadas y la cola de callbacks, empujando callbacks a la pila de llamadas cuando está vacía.

Ejemplo: Bucle de Eventos Básico

console.log('Start');

setTimeout(() => {
  console.log('Timeout callback');
}, 1000);

console.log('End');

Explicación:

  1. console.log('Start') se ejecuta y se registra inmediatamente.
  2. setTimeout programa un callback para ser ejecutado después de 1 segundo.
  3. console.log('End') se ejecuta y se registra inmediatamente.
  4. Después de 1 segundo, el callback de setTimeout se empuja a la cola de callbacks y eventualmente se ejecuta.

Programación Asíncrona

La programación asíncrona permite a JavaScript realizar tareas de larga duración sin bloquear el hilo principal.

  • Callbacks: Funciones pasadas como argumentos a otras funciones, ejecutadas después de la finalización de una tarea asíncrona.
  • Promesas: Objetos que representan la eventual finalización (o fallo) de una operación asíncrona.
  • Async/Await: Azúcar sintáctico sobre promesas, haciendo que el código asíncrono parezca sincrónico.

Ejemplo: Callback

function fetchData(callback) {
  setTimeout(() => {
    callback('Data received');
  }, 1000);
}

fetchData((data) => {
  console.log(data);
});

Explicación:

  1. fetchData se llama con una función de callback.
  2. setTimeout simula una operación asíncrona.
  3. Después de 1 segundo, el callback se ejecuta, registrando 'Data received'.

Ejemplo: Promesas

function fetchData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('Data received');
    }, 1000);
  });
}

fetchData().then((data) => {
  console.log(data);
});

Explicación:

  1. fetchData devuelve una promesa.
  2. setTimeout simula una operación asíncrona.
  3. Después de 1 segundo, la promesa se resuelve, y se llama a then, registrando 'Data received'.

Ejemplo: Async/Await

function fetchData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('Data received');
    }, 1000);
  });
}

async function getData() {
  const data = await fetchData();
  console.log(data);
}

getData();

Explicación:

  1. fetchData devuelve una promesa.
  2. getData es una función async que espera a que la promesa se resuelva.
  3. Después de 1 segundo, await obtiene el valor resuelto, registrando 'Data received'.

Conceptos Avanzados de Concurrencia

A medida que avanzas, entender patrones de concurrencia más complejos y herramientas se vuelve esencial.

  • Web Workers: Permiten ejecutar scripts en hilos de fondo.
  • Canales de Mensajes: Facilitan la comunicación entre diferentes contextos (por ejemplo, hilo principal y web workers).
  • Atomics y SharedArrayBuffer: Proporcionan primitivas de concurrencia de bajo nivel para memoria compartida.

Ejemplo: Web Workers

// worker.js
self.onmessage = function(e) {
  const result = e.data * 2;
  self.postMessage(result);
};

// main.js
const worker = new Worker('worker.js');
worker.onmessage = function(e) {
  console.log('Result from worker:', e.data);
};
worker.postMessage(10);

Explicación:

  1. worker.js escucha mensajes y procesa datos.
  2. main.js crea un worker y le envía datos.
  3. El worker procesa los datos y envía el resultado de vuelta al hilo principal.

Conclusión

El modelo de concurrencia de JavaScript, centrado en el bucle de eventos, le permite manejar operaciones asíncronas de manera eficiente a pesar de ser de un solo hilo. Entender el bucle de eventos, callbacks, promesas y async/await es crucial para escribir código eficiente y no bloqueante. A medida que progresas, explorar conceptos avanzados como Web Workers y memoria compartida puede mejorar aún más tu capacidad para gestionar la concurrencia en aplicaciones JavaScript.
