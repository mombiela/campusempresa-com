Introducción a Clases y Objetos

En Python, las clases y los objetos son conceptos fundamentales de la Programación Orientada a Objetos (OOP). Ayudan a organizar el código y a crear componentes reutilizables.

Conceptos Clave

  • Clase: Un plano para crear objetos. Define un conjunto de atributos y métodos que los objetos creados tendrán.
  • Objeto: Una instancia de una clase. Se crea usando la clase y puede usar los atributos y métodos definidos en la clase.
  • Atributos: Variables que pertenecen a una clase o a un objeto.
  • Métodos: Funciones que pertenecen a una clase y típicamente operan sobre sus atributos.

Definiendo una Clase

Para definir una clase en Python, usa la palabra clave class seguida del nombre de la clase y dos puntos. Dentro de la clase, define atributos y métodos.

class Dog:
    # Atributo de clase
    species = "Canis familiaris"

    # Inicializador / Atributos de instancia
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Método
    def description(self):
        return f"{self.name} tiene {self.age} años"

Explicación

  • class Dog:: Define una nueva clase llamada Dog.
  • species: Un atributo de clase compartido por todas las instancias de la clase.
  • __init__: El método inicializador que se llama cuando se crea un objeto. Inicializa los atributos de instancia name y age.
  • description: Un método que devuelve una cadena describiendo al perro.

Creando Objetos

Para crear un objeto (instancia) de una clase, llama a la clase usando su nombre y pasa los argumentos requeridos.

# Creando una instancia de la clase Dog
my_dog = Dog("Buddy", 3)

# Accediendo a atributos y métodos
print(my_dog.name)  # Salida: Buddy
print(my_dog.age)   # Salida: 3
print(my_dog.description())  # Salida: Buddy tiene 3 años

Atributos de Instancia vs. Clase

Los atributos de instancia son específicos de cada objeto, mientras que los atributos de clase son compartidos por todas las instancias de la clase.

class Dog:
    species = "Canis familiaris"  # Atributo de clase

    def __init__(self, name, age):
        self.name = name  # Atributo de instancia
        self.age = age    # Atributo de instancia

# Creando instancias
dog1 = Dog("Buddy", 3)
dog2 = Dog("Lucy", 5)

# Accediendo al atributo de clase
print(dog1.species)  # Salida: Canis familiaris
print(dog2.species)  # Salida: Canis familiaris

# Accediendo a los atributos de instancia
print(dog1.name)  # Salida: Buddy
print(dog2.name)  # Salida: Lucy

Métodos en Clases

Los métodos son funciones definidas dentro de una clase y se usan para realizar operaciones usando los atributos de la clase.

class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def speak(self, sound):
        return f"{self.name} dice {sound}"

# Creando una instancia
my_dog = Dog("Buddy", 3)

# Llamando a un método
print(my_dog.speak("Guau Guau"))  # Salida: Buddy dice Guau Guau

Herencia

La herencia permite que una clase herede atributos y métodos de otra clase. Esto promueve la reutilización de código y la jerarquía lógica.

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        raise NotImplementedError("La subclase debe implementar este método")

class Dog(Animal):
    def speak(self):
        return f"{self.name} dice Guau Guau"

class Cat(Animal):
    def speak(self):
        return f"{self.name} dice Miau"

# Creando instancias
dog = Dog("Buddy")
cat = Cat("Whiskers")

# Llamando a métodos
print(dog.speak())  # Salida: Buddy dice Guau Guau
print(cat.speak())  # Salida: Whiskers dice Miau

Polimorfismo

El polimorfismo permite que los métodos se usen de manera intercambiable entre diferentes clases, siempre que compartan el mismo nombre de método.

class Bird:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} dice Pío"

# Creando instancias
animals = [Dog("Buddy"), Cat("Whiskers"), Bird("Tweety")]

# Llamando al mismo método en diferentes objetos
for animal in animals:
    print(animal.speak())
# Salida:
# Buddy dice Guau Guau
# Whiskers dice Miau
# Tweety dice Pío

Encapsulación

La encapsulación restringe el acceso directo a algunos de los componentes de un objeto, lo que puede prevenir la modificación accidental de datos.

class Car:
    def __init__(self, make, model, year):
        self.__make = make  # Atributo privado
        self.__model = model  # Atributo privado
        self.__year = year  # Atributo privado

    def get_info(self):
        return f"{self.__year} {self.__make} {self.__model}"

# Creando una instancia
my_car = Car("Toyota", "Corolla", 2020)

# Accediendo a los datos a través de un método
print(my_car.get_info())  # Salida: 2020 Toyota Corolla

# Intentar acceder a los atributos privados directamente resultará en un error
# print(my_car.__make)  # AttributeError: 'Car' object has no attribute '__make'

Resumen

  • Las clases y los objetos son el núcleo de la OOP en Python.
  • Las clases definen el plano, mientras que los objetos son instancias de las clases.
  • Los atributos pueden ser específicos de la instancia o compartidos entre todas las instancias (atributos de clase).
  • Los métodos definen el comportamiento de los objetos.
  • La herencia permite que las clases hereden atributos y métodos de otras clases.
  • El polimorfismo permite el uso de una interfaz unificada para diferentes formas subyacentes (tipos de datos).
  • La encapsulación protege el estado interno de un objeto restringiendo el acceso a ciertos atributos.

Entender estos conceptos es crucial para escribir código organizado, reutilizable y mantenible en Python.
