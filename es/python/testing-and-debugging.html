Introducción

Las pruebas y la depuración son habilidades cruciales para cualquier programador. Aseguran que tu código funcione como se espera y te ayudan a identificar y corregir errores. Esta sección cubrirá los conceptos básicos de pruebas y depuración en Python, progresando desde niveles principiantes hasta avanzados.

Pruebas Unitarias con `unittest`

Las pruebas unitarias implican probar componentes individuales de tu código para asegurarse de que funcionen correctamente.

Conceptos Básicos de `unittest`

  • ¿Qué es unittest?
    • unittest es un módulo incorporado en Python para crear y ejecutar pruebas.
  • Creación de un Caso de Prueba
    • Un caso de prueba es una unidad única de prueba. Verifica una respuesta específica a un conjunto particular de entradas.
import unittest

def add(a, b):
    return a + b

class TestMathOperations(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(1, 2), 3)
        self.assertEqual(add(-1, 1), 0)

if __name__ == '__main__':
    unittest.main()

Ejecución de Pruebas

  • Línea de Comando
    • Ejecuta tus pruebas desde la línea de comando usando: 
      python -m unittest test_module.py
  • Descubrimiento de Pruebas
    • Descubre y ejecuta pruebas automáticamente: 
      python -m unittest discover

Pruebas Avanzadas con `pytest`

pytest es un marco de pruebas más poderoso que simplifica la escritura y ejecución de pruebas.

Conceptos Básicos de `pytest`

  • Instalación
    • Instala pytest usando pip: 
      pip install pytest
  • Escritura de Pruebas
    • Las pruebas se escriben como funciones con el prefijo test_.
def add(a, b):
    return a + b

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0

Ejecución de Pruebas

  • Línea de Comando
    • Ejecuta tus pruebas usando: 
      pytest

Fixtures

  • ¿Qué son los Fixtures?
    • Los fixtures se utilizan para configurar un estado conocido antes de que se ejecuten las pruebas y limpiar después.
import pytest

@pytest.fixture
def sample_data():
    return [1, 2, 3, 4, 5]

def test_sum(sample_data):
    assert sum(sample_data) == 15

Depuración con `pdb`

pdb es el depurador incorporado de Python, que te permite recorrer el código, inspeccionar variables y evaluar expresiones.

Conceptos Básicos de `pdb`

  • Iniciar el Depurador
    • Inserta pdb.set_trace() en el punto donde deseas comenzar a depurar.
import pdb

def add(a, b):
    pdb.set_trace()
    return a + b

print(add(1, 2))

Comandos Comunes

  • n (next)
    • Moverse a la siguiente línea dentro de la misma función.
  • s (step)
    • Entrar en una llamada de función.
  • c (continue)
    • Continuar la ejecución hasta el siguiente punto de interrupción.
  • p (print)
    • Imprimir el valor de una expresión.

Técnicas Avanzadas de Depuración

Uso de `logging`

  • ¿Por qué Usar Logging?
    • El registro proporciona una forma de rastrear eventos que ocurren cuando se ejecuta un software. Puede ser más flexible e informativo que las declaraciones print.
import logging

logging.basicConfig(level=logging.DEBUG)

def add(a, b):
    logging.debug(f'Sumando {a} y {b}')
    return a + b

print(add(1, 2))

Perfilado de Código

  • ¿Por qué Perfilar?
    • El perfilado te ayuda a entender dónde tu programa gasta su tiempo e identificar cuellos de botella.
import cProfile

def slow_function():
    total = 0
    for i in range 10000):
        total += i
    return total

cProfile.run('slow_function()')

Conclusión

Las pruebas y la depuración son prácticas esenciales para escribir código Python confiable y mantenible. Al dominar herramientas como unittest, pytest, pdb y logging, puedes asegurarte de que tu código sea robusto y eficiente. Recuerda, el objetivo no es solo encontrar errores, sino entender por qué ocurren y cómo prevenirlos en el futuro.
