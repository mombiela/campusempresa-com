Introducción a PyTest

PyTest es un potente marco de pruebas para Python que facilita la escritura de casos de prueba simples y escalables. Es ampliamente utilizado debido a su simplicidad y flexibilidad.

  • Facilidad de uso: PyTest es fácil de comenzar y requiere una configuración mínima.
  • Flexibilidad: Soporta fixtures, pruebas parametrizadas y plugins.
  • Compatibilidad: Funciona con suites de pruebas existentes de unittest o nose.

Instalando PyTest

Para comenzar con PyTest, necesitas instalarlo. Puedes instalar PyTest usando pip:

pip install pytest

Escribiendo tu primera prueba

Comencemos con un ejemplo simple para entender cómo funciona PyTest.

Ejemplo: Probando una función simple

Considera una función simple que suma dos números:

# my_module.py
def add(a, b):
    return a + b

Para probar esta función usando PyTest, crea un nuevo archivo llamado test_my_module.py:

# test_my_module.py
from my_module import add

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0
    assert add(-1, -1) == -2

Para ejecutar la prueba, simplemente ejecuta el siguiente comando en tu terminal:

pytest

Entendiendo las afirmaciones

Las afirmaciones son la columna vertebral de cualquier prueba. En PyTest, usas la declaración assert para verificar si una condición es verdadera. Si la condición es falsa, la prueba falla.

  • Afirmación básica: assert expresión
  • Con mensaje: assert expresión, "mensaje"

Ejemplo: Usando afirmaciones

def test_example():
    assert 1 == 1
    assert "hello".upper() == "HELLO", "La conversión de cadena a mayúsculas falló"

Fixtures

Los fixtures son una forma de proporcionar una base fija sobre la cual las pruebas pueden ejecutarse de manera confiable y repetida. Ayudan a configurar y desmantelar el entorno para las pruebas.

Ejemplo: Usando Fixtures

import pytest

@pytest.fixture
def sample_data():
    return {"key1": "value1", "key2": "value2"}

def test_sample_data(sample_data):
    assert sample_data["key1"] == "value1"
    assert sample_data["key2"] == "value2"

Pruebas parametrizadas

PyTest te permite ejecutar una función de prueba varias veces con diferentes argumentos usando el decorador @pytest.mark.parametrize.

Ejemplo: Pruebas parametrizadas

import pytest

@pytest.mark.parametrize("a, b, expected", [
    (1, 2, 3),
    (-1, 1, 0),
    (-1, -1, -2)
])
def test_add(a, b, expected):
    assert add(a, b) == expected

Ejecutando pruebas

Puedes ejecutar pruebas de varias maneras usando PyTest:

  • Ejecutar todas las pruebas: pytest
  • Ejecutar un archivo de prueba específico: pytest test_my_module.py
  • Ejecutar una función de prueba específica: pytest test_my_module.py::test_add

Características avanzadas

Marcadores

Los marcadores se utilizan para categorizar pruebas y ejecutarlas selectivamente.

@pytest.mark.slow
def test_slow_function():
    pass

# Ejecutar pruebas marcadas como lentas
pytest -m slow

Plugins

PyTest soporta una amplia gama de plugins para extender su funcionalidad. Puedes encontrar plugins en PyPI.

Ejemplo: Usando un Plugin

pip install pytest-cov

Agrega lo siguiente a tu comando de prueba para generar un informe de cobertura:

pytest --cov=my_module

Conclusión

PyTest es un marco de pruebas versátil y fácil de usar que puede manejar pruebas unitarias simples así como pruebas funcionales complejas. Aprovechando los fixtures, las pruebas parametrizadas y los plugins, puedes crear suites de pruebas robustas y mantenibles.

  • Comienza simple: Empieza con afirmaciones básicas y pruebas simples.
  • Usa Fixtures: Simplifica los procesos de configuración y desmantelamiento.
  • Parametriza: Ejecuta pruebas con múltiples conjuntos de datos.
  • Explora Plugins: Extiende la funcionalidad según sea necesario.

Al dominar PyTest, puedes asegurar que tu código Python sea confiable y esté libre de errores.
