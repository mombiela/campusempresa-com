Introducción a las APIs RESTful

  • Definición: REST (Transferencia de Estado Representacional) es un estilo arquitectónico para diseñar aplicaciones en red. Se basa en un protocolo de comunicación sin estado, cliente-servidor y almacenable en caché: el HTTP.
  • Conceptos Clave:
    • Recursos: Objetos o representaciones de datos que se pueden acceder a través de una URL.
    • Métodos HTTP: Métodos comunes incluyen GET, POST, PUT, DELETE.
    • Sin Estado: Cada solicitud de un cliente al servidor debe contener toda la información necesaria para entender y procesar la solicitud.
    • Operaciones CRUD: Operaciones de Crear, Leer, Actualizar, Eliminar que se mapean a métodos HTTP.

Configurando Tu Entorno

  • Instalación de Python: Asegúrate de que Python esté instalado en tu sistema.
  • Bibliotecas Requeridas: Flask para crear APIs, requests para consumir APIs. 
    pip install Flask requests

Creando una API RESTful Simple con Flask

Configurando Flask

  • Aplicación Básica de Flask: 
    from flask import Flask, jsonify, request

app = Flask(__name__)

@app.route('/')
def home():
    return "¡Bienvenido a la API RESTful con Flask!"

if __name__ == '__main__':
    app.run(debug=True)

Definiendo Endpoints

  • Solicitud GET:

    @app.route('/api/v1/resource', methods=['GET'])
def get_resource():
    data = {"message": "Esta es una solicitud GET"}
    return jsonify(data)

  • Solicitud POST:

    @app.route('/api/v1/resource', methods=['POST'])
def create_resource():
    new_data = request.get_json()
    response = {"message": "Recurso creado", "data": new_data}
    return jsonify(response), 201

  • Solicitud PUT:

    @app.route('/api/v1/resource/<int:id>', methods=['PUT'])
def update_resource(id):
    updated_data = request.get_json()
    response = {"message": f"Recurso {id} actualizado", "data": updated_data}
    return jsonify(response)

  • Solicitud DELETE:

    @app.route('/api/v1/resource/<int:id>', methods=['DELETE'])
def delete_resource(id):
    response = {"message": f"Recurso {id} eliminado"}
    return jsonify(response), 204

Consumiendo APIs RESTful con `requests`

  • Solicitud GET:

    import requests

response = requests.get('http://localhost:5000/api/v1/resource')
print(response.json())

  • Solicitud POST:

    import requests

data = {"name": "Nuevo Recurso"}
response = requests.post('http://localhost:5000/api/v1/resource', json=data)
print(response.json())

  • Solicitud PUT:

    import requests

data = {"name": "Recurso Actualizado"}
response = requests.put('http://localhost:5000/api/v1/resource/1', json=data)
print(response.json())

  • Solicitud DELETE:

    import requests

response = requests.delete('http://localhost:5000/api/v1/resource/1')
print(response.status_code)

Temas Avanzados

Autenticación y Autorización

  • Autenticación Básica: 
    from flask import request, jsonify

@app.route('/api/v1/secure-resource', methods=['GET'])
def secure_resource():
    auth = request.authorization
    if auth and auth.username == 'user' and auth.password == 'pass':
        return jsonify({"message": "Autenticado"})
    return jsonify({"message": "Autenticación fallida"}), 401

Usando Flask-RESTful

  • Instalación:

    pip install Flask-RESTful

  • Creando una API RESTful con Flask-RESTful:

    from flask import Flask
from flask_restful import Resource, Api

app = Flask(__name__)
api = Api(app)

class HelloWorld(Resource):
    def get(self):
        return {'hello': 'world'}

api.add_resource(HelloWorld, '/')

if __name__ == '__main__':
    app.run(debug=True)

Conclusión

  • Resumen:
    • Las APIs RESTful son una forma poderosa de habilitar la comunicación entre diferentes partes de una aplicación web o entre diferentes aplicaciones.
    • Flask es un framework ligero y fácil de usar para crear APIs RESTful en Python.
    • Entender cómo crear y consumir APIs RESTful es esencial para el desarrollo web moderno.
