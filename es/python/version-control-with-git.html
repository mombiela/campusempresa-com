Control de Versiones con Git

Introducción al Control de Versiones

  • Definición: El control de versiones es un sistema que registra los cambios en un archivo o conjunto de archivos a lo largo del tiempo para que puedas recordar versiones específicas más tarde.
  • Importancia: Ayuda a rastrear cambios, colaborar con otros y mantener un historial de tu proyecto.

¿Qué es Git?

  • Definición: Git es un sistema de control de versiones distribuido diseñado para manejar todo, desde proyectos pequeños hasta muy grandes, con velocidad y eficiencia.
  • Características Clave:
    • Distribuido: Cada desarrollador tiene una copia completa del repositorio.
    • Rendimiento: Manejo rápido y eficiente de grandes proyectos.
    • Ramificación y Fusión: Fácil de crear, gestionar y fusionar ramas.

Configuración de Git

Instalación

  • Windows: Descarga desde git-scm.com y sigue las instrucciones de instalación.
  • Mac: Usa Homebrew: brew install git
  • Linux: Usa el gestor de paquetes: sudo apt-get install git (basado en Debian) o sudo yum install git (basado en Red Hat)

Configuración

  • Configura tu nombre de usuario y correo electrónico: 
    git config --global user.name "Tu Nombre"
git config --global user.email "tu.correo@ejemplo.com"

Comandos Básicos de Git

Inicializando un Repositorio

  • Comando: git init
  • Explicación: Inicializa un nuevo repositorio Git en el directorio actual.

Clonando un Repositorio

  • Comando: git clone <repository-url>
  • Explicación: Crea una copia de un repositorio existente.

Verificando el Estado

  • Comando: git status
  • Explicación: Muestra el estado de los cambios como no rastreados, modificados o en etapa.

Añadiendo Cambios

  • Comando: git add <file>
  • Explicación: Añade cambios en el archivo especificado al área de preparación.
  • Ejemplo: 
    git add script.py

Confirmando Cambios

  • Comando: git commit -m "mensaje del commit"
  • Explicación: Registra los cambios en el repositorio con un mensaje.
  • Ejemplo: 
    git commit -m "Commit inicial"

Viendo el Historial de Commits

  • Comando: git log
  • Explicación: Muestra el historial de commits del repositorio.

Ramificación y Fusión

Creando una Rama

  • Comando: git branch <branch-name>
  • Explicación: Crea una nueva rama.
  • Ejemplo: 
    git branch feature-xyz

Cambiando de Rama

  • Comando: git checkout <branch-name>
  • Explicación: Cambia a la rama especificada.
  • Ejemplo: 
    git checkout feature-xyz

Fusionando Ramas

  • Comando: git merge <branch-name>
  • Explicación: Fusiona la rama especificada en la rama actual.
  • Ejemplo: 
    git merge feature-xyz

Trabajando con Repositorios Remotos

Añadiendo un Remoto

  • Comando: git remote add <name> <url>
  • Explicación: Añade un repositorio remoto.
  • Ejemplo: 
    git remote add origin https://github.com/user/repo.git

Enviando Cambios

  • Comando: git push <remote> <branch>
  • Explicación: Envía cambios al repositorio remoto y rama especificados.
  • Ejemplo: 
    git push origin main

Obteniendo Cambios

  • Comando: git pull <remote> <branch>
  • Explicación: Obtiene y fusiona cambios del repositorio remoto y rama especificados.
  • Ejemplo: 
    git pull origin main

Manejo de Conflictos

  • Definición: Los conflictos ocurren cuando los cambios en diferentes ramas se contradicen entre sí.
  • Resolución:
    1. Identifica el conflicto usando git status.
    2. Abre el archivo en conflicto y resuelve manualmente el conflicto.
    3. Añade el archivo resuelto usando git add <file>.
    4. Confirma los cambios usando git commit.

Características Avanzadas de Git

Guardando Cambios Temporalmente

  • Comando: git stash
  • Explicación: Guarda temporalmente los cambios que no están listos para ser confirmados.
  • Ejemplo: 
    git stash

Rebase

  • Comando: git rebase <branch>
  • Explicación: Vuelve a aplicar commits sobre otra punta base.
  • Ejemplo: 
    git rebase main

Conclusión

El control de versiones con Git es una habilidad esencial para cualquier programador. Te permite rastrear cambios, colaborar con otros y gestionar el historial de tu proyecto de manera eficiente. Al dominar los comandos de Git y comprender sus características, puedes mejorar significativamente tu flujo de trabajo y productividad.
