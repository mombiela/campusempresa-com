Introducción a la Calidad del Código

  • Definición: La calidad del código se refiere al grado en que el código se adhiere a los estándares y mejores prácticas acordados, haciéndolo legible, mantenible y eficiente.
  • Importancia:
    • Legibilidad: Más fácil de leer y entender para otros (y para ti mismo).
    • Mantenibilidad: Simplifica el proceso de actualizar y corregir el código.
    • Eficiencia: Asegura que el código se ejecute de manera óptima.

¿Qué es PEP 8?

  • Definición: PEP 8 es la Propuesta de Mejora de Python que describe las convenciones para escribir código Python legible y consistente.
  • Propósito: Mejorar la legibilidad y consistencia del código Python.

Principales Directrices de PEP 8

Indentación

  • Usa 4 espacios por nivel de indentación.
def example_function():
    if True:
        print("¡Sigue PEP 8!")

Longitud de Línea

  • Limita todas las líneas a un máximo de 79 caracteres.
# Este es un comentario que tiene exactamente 79 caracteres de largo para demostrar la directriz.

Líneas en Blanco

  • Usa líneas en blanco para separar definiciones de funciones y clases de nivel superior, y dentro de las funciones para separar secciones lógicas.
class MyClass:
    def __init__(self):
        self.value = 0

    def increment(self):
        self.value += 1

Importaciones

  • Las importaciones generalmente deben estar en líneas separadas.
  • Ordena las importaciones: biblioteca estándar, bibliotecas de terceros, importaciones específicas de la aplicación/biblioteca local.
import os
import sys

import requests

from mymodule import myfunction

Convenciones de Nombres

  • Variables y Funciones: Usa lower_case_with_underscores.
  • Clases: Usa CapWords.
  • Constantes: Usa ALL_CAPS_WITH_UNDERSCORES.
# Nombres de variables y funciones
my_variable = 10

def my_function():
    pass

# Nombre de clase
class MyClass:
    pass

# Nombre de constante
MAX_LIMIT = 100

Comentarios

  • Usa comentarios para explicar por qué se hace algo, no qué se hace.
  • Usa # para comentarios en línea y comentarios de bloque.
# Este es un comentario de bloque
x = 42  # Este es un comentario en línea

Docstrings

  • Usa comillas triples para docstrings en funciones, clases y módulos.
def my_function():
    """
    Este es un docstring.
    Explica lo que hace la función.
    """
    pass

Herramientas para Hacer Cumplir PEP 8

  • pycodestyle: Una herramienta para verificar tu código Python contra algunas de las convenciones de estilo de PEP 8.
  • autopep8: Una herramienta que formatea automáticamente el código Python para que cumpla con la guía de estilo PEP 8.
  • flake8: Una herramienta que combina pycodestyle, pyflakes y otros plugins para verificar el estilo y la calidad del código Python.

Ejemplo de Código Cumpliente con PEP 8

import os
import sys

class ExampleClass:
    """Esta es una clase de ejemplo que sigue las directrices de PEP 8."""

    def __init__(self, value):
        self.value = value

    def increment(self):
        """Incrementa el valor en uno."""
        self.value += 1

def main():
    """Función principal para ejecutar el ejemplo."""
    example = ExampleClass(0)
    example.increment()
    print(example.value)

if __name__ == "__main__":
    main()

Conclusión

  • Resumen: Adherirse a las directrices de PEP 8 asegura que tu código Python sea legible, mantenible y consistente. Utilizar herramientas como pycodestyle, autopep8 y flake8 puede ayudar a hacer cumplir estos estándares.
  • Próximos Pasos: Practica escribir código conforme a PEP 8 y usa las herramientas mencionadas para verificar y mejorar la calidad de tu código.
