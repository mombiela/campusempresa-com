Introducción al Desarrollo Guiado por Pruebas

El Desarrollo Guiado por Pruebas (TDD) es un proceso de desarrollo de software en el que escribes pruebas para tu código antes de escribir el código real. Este enfoque ayuda a garantizar que tu código sea confiable, mantenible y libre de errores.

Conceptos Clave del TDD

  • Ciclo Rojo-Verde-Refactorizar: La práctica central del TDD implica tres pasos:
    1. Rojo: Escribir una prueba que falle.
    2. Verde: Escribir la mínima cantidad de código necesaria para que la prueba pase.
    3. Refactorizar: Mejorar el código asegurándose de que todas las pruebas sigan pasando.
  • Pruebas Unitarias: Pruebas pequeñas que verifican el comportamiento de una sola función o método.
  • Aserciones: Declaraciones en las pruebas que verifican si una condición es verdadera.

Configuración de Tu Entorno

Antes de comenzar con TDD, necesitas configurar tu entorno Python con las herramientas necesarias.

Instalando pytest

pytest es un marco de pruebas popular para Python. Puedes instalarlo usando pip:

pip install pytest

Escribiendo Tu Primera Prueba

Comencemos con un ejemplo simple para ilustrar el proceso TDD.

Ejemplo: Función de Suma

  1. Rojo: Escribir una prueba que falle porque la función aún no existe.
  2. Verde: Implementar la función para que la prueba pase.
  3. Refactorizar: Limpiar el código si es necesario.

Paso 1: Escribir una Prueba que Falle

Crea un archivo llamado test_sum.py:

def test_sum():
    assert sum([1, 2, 3]) == 6

Ejecuta la prueba usando pytest:

pytest test_sum.py

Deberías ver un fallo porque la función sum no está definida.

Paso 2: Implementar la Función

Crea un archivo llamado sum.py:

def sum(numbers):
    return sum(numbers)

Actualiza test_sum.py para importar la función sum:

from sum import sum

def test_sum():
    assert sum([1, 2, 3]) == 6

Ejecuta la prueba nuevamente:

pytest test_sum.py

La prueba debería pasar ahora.

Paso 3: Refactorizar

En este ejemplo simple, puede que no haya mucho que refactorizar, pero siempre verifica si el código puede mejorarse sin romper las pruebas.

Conceptos Avanzados de TDD

A medida que te sientas más cómodo con TDD, puedes explorar temas más avanzados.

Mocking y Patching

A veces, necesitas probar código que interactúa con sistemas externos (por ejemplo, bases de datos, servicios web). Puedes usar mocking para simular estas interacciones.

Ejemplo: Mocking de una Llamada a la Base de Datos

from unittest.mock import patch

def get_user_from_db(user_id):
    # Simular una llamada a la base de datos
    pass

@patch('your_module.get_user_from_db')
def test_get_user(mock_get_user):
    mock_get_user.return_value = {'id': 1, 'name': 'John Doe'}
    user = get_user_from_db(1)
    assert user['name'] == 'John Doe'

Pruebas Parametrizadas

Puedes ejecutar la misma prueba con diferentes entradas usando pruebas parametrizadas.

Ejemplo: Prueba Parametrizada

import pytest

@pytest.mark.parametrize("input,expected", [
    ([1, 2, 3], 6),
    ([4, 5, 6], 15),
    ([7, 8, 9], 24),
])
def test_sum(input, expected):
    assert sum(input) == expected

Conclusión

El Desarrollo Guiado por Pruebas es una metodología poderosa que puede ayudarte a escribir código mejor y más confiable. Siguiendo el ciclo Rojo-Verde-Refactorizar, aseguras que tu código cumpla con los requisitos y esté libre de errores. A medida que avances, puedes incorporar técnicas más sofisticadas como mocking y pruebas parametrizadas para manejar escenarios complejos. ¡Feliz codificación!
