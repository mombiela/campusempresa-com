Introducción

La sincronización y comunicación entre procesos son aspectos fundamentales en los sistemas operativos. Estos conceptos permiten que múltiples procesos trabajen de manera coordinada, evitando conflictos y garantizando la integridad de los datos. En esta sección, exploraremos los mecanismos y técnicas utilizados para lograr una sincronización y comunicación efectiva entre procesos.

Sincronización de Procesos

La sincronización de procesos es crucial para evitar condiciones de carrera y asegurar que los procesos compartan recursos de manera segura.

Conceptos Clave

  • Condición de carrera: Ocurre cuando dos o más procesos acceden y manipulan datos compartidos simultáneamente, lo que puede llevar a resultados inconsistentes.
  • Sección crítica: Parte del código donde el proceso accede a recursos compartidos.
  • Mutex (Mutual Exclusion): Mecanismo que asegura que solo un proceso pueda acceder a la sección crítica a la vez.
  • Semáforos: Variables utilizadas para controlar el acceso a recursos compartidos.

Ejemplo de Mutex en C

#include <pthread.h>
#include <stdio.h>

pthread_mutex_t lock;

void *thread_function(void *arg) {
    pthread_mutex_lock(&lock);
    // Sección crítica
    printf("Thread %d está en la sección crítica\n", *(int *)arg);
    pthread_mutex_unlock(&lock);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    pthread_mutex_init(&lock, NULL);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    pthread_mutex_destroy(&lock);
    return 0;
}

Ejemplo de Semáforo en C

#include <pthread.h>
#include <semaphore.h>
#include <stdio.h>

sem_t sem;

void *thread_function(void *arg) {
    sem_wait(&sem);
    // Sección crítica
    printf("Thread %d está en la sección crítica\n", *(int *)arg);
    sem_post(&sem);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    sem_init(&sem, 0, 1);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    sem_destroy(&sem);
    return 0;
}

Comunicación entre Procesos (IPC)

La comunicación entre procesos (IPC) permite que los procesos intercambien datos y señales. Existen varios mecanismos de IPC, cada uno con sus propias ventajas y desventajas.

Mecanismos de IPC

  • Pipes: Permiten la comunicación unidireccional entre procesos relacionados.
  • FIFOs (Named Pipes): Similar a los pipes, pero permiten comunicación bidireccional y entre procesos no relacionados.
  • Sockets: Permiten la comunicación entre procesos en diferentes máquinas a través de una red.
  • Memoria compartida: Permite que múltiples procesos accedan a una región de memoria común.
  • Mensajes: Permiten el intercambio de mensajes entre procesos.

Ejemplo de Pipe en C

#include <stdio.h>
#include <unistd.h>

int main() {
    int fd[2];
    pipe(fd);
    pid_t pid = fork();

    if (pid == 0) {
        // Proceso hijo
        close(fd[0]);
        char message[] = "Hola desde el proceso hijo";
        write(fd[1], message, sizeof(message));
        close(fd[1]);
    } else {
        // Proceso padre
        close(fd[1]);
        char buffer[100];
        read(fd[0], buffer, sizeof(buffer));
        printf("Mensaje recibido: %s\n", buffer);
        close(fd[0]);
    }

    return 0;
}

Ejemplo de Memoria Compartida en C

#include <stdio.h>
#include <stdlib.h>
#include <sys/ipc.h>
#include <sys/shm.h>
#include <sys/types.h>

int main() {
    key_t key = ftok("shmfile", 65);
    int shmid = shmget(key, 1024, 0666 | IPC_CREAT);
    char *str = (char *)shmat(shmid, (void *)0, 0);

    if (fork() == 0) {
        // Proceso hijo
        printf("Escribiendo en memoria compartida\n");
        sprintf(str, "Hola desde el proceso hijo");
        shmdt(str);
    } else {
        // Proceso padre
        wait(NULL);
        printf("Leyendo de memoria compartida: %s\n", str);
        shmdt(str);
        shmctl(shmid, IPC_RMID, NULL);
    }

    return 0;
}

Conclusión

La sincronización y comunicación entre procesos son componentes esenciales en los sistemas operativos modernos. La sincronización asegura que los procesos no interfieran entre sí al acceder a recursos compartidos, mientras que la comunicación entre procesos permite el intercambio de datos y señales. Comprender estos conceptos y sus implementaciones prácticas es crucial para el desarrollo de aplicaciones robustas y eficientes en entornos multitarea.
