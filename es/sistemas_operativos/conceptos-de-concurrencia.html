Introducción a la concurrencia

La concurrencia es un concepto fundamental en los sistemas operativos que permite la ejecución de múltiples procesos o hilos de manera simultánea. Esto es crucial para mejorar la eficiencia y el rendimiento de los sistemas modernos. En esta sección, exploraremos los conceptos básicos de concurrencia, sus beneficios y los desafíos que presenta.

Procesos e hilos

  • Proceso: Un proceso es una instancia de un programa en ejecución. Cada proceso tiene su propio espacio de direcciones y recursos asignados.
  • Hilo: Un hilo es la unidad más pequeña de ejecución dentro de un proceso. Los hilos de un mismo proceso comparten el mismo espacio de direcciones y recursos.

Ejemplo de creación de hilos en Python

import threading

def print_numbers():
    for i in range(5):
        print(i)

# Crear dos hilos
thread1 = threading.Thread(target=print_numbers)
thread2 = threading.Thread(target=print_numbers)

# Iniciar los hilos
thread1.start()
thread2.start()

# Esperar a que los hilos terminen
thread1.join()
thread2.join()

En este ejemplo, se crean dos hilos que ejecutan la función print_numbers. Ambos hilos se ejecutan concurrentemente, imprimiendo números del 0 al 4.

Beneficios de la concurrencia

  • Mejor utilización de la CPU: Permite que la CPU trabaje en múltiples tareas al mismo tiempo, mejorando la eficiencia.
  • Mejor rendimiento: Reduce el tiempo de espera para los usuarios al permitir que múltiples tareas se ejecuten simultáneamente.
  • Mejor respuesta del sistema: Mejora la capacidad de respuesta del sistema, especialmente en aplicaciones interactivas.

Desafíos de la concurrencia

  • Condiciones de carrera: Ocurren cuando dos o más hilos acceden y modifican un recurso compartido simultáneamente, causando resultados impredecibles.
  • Interbloqueo (Deadlock): Ocurre cuando dos o más procesos esperan indefinidamente por recursos que están siendo retenidos por ellos mismos.
  • Inanición (Starvation): Ocurre cuando un proceso no puede acceder a los recursos necesarios para su ejecución debido a que otros procesos acaparan esos recursos.

Ejemplo de condición de carrera en Python

import threading

counter = 0

def increment_counter():
    global counter
    for _ in range(100000):
        counter += 1

# Crear dos hilos
thread1 = threading.Thread(target=increment_counter)
thread2 = threading.Thread(target=increment_counter)

# Iniciar los hilos
thread1.start()
thread2.start()

# Esperar a que los hilos terminen
thread1.join()
thread2.join()

print(f"Counter: {counter}")

En este ejemplo, dos hilos incrementan una variable compartida counter. Debido a la condición de carrera, el valor final de counter puede no ser el esperado (200000).

Mecanismos de sincronización

Para evitar los problemas de concurrencia, se utilizan mecanismos de sincronización. Algunos de los más comunes son:

  • Bloqueos (Locks): Permiten que solo un hilo acceda a un recurso compartido a la vez.
  • Semáforos (Semaphores): Generalizan los bloqueos permitiendo un número específico de hilos acceder a un recurso.
  • Monitores (Monitors): Combina bloqueos y variables de condición para gestionar el acceso a los recursos compartidos.

Ejemplo de uso de bloqueo en Python

import threading

counter = 0
lock = threading.Lock()

def increment_counter():
    global counter
    for _ in range(100000):
        with lock:
            counter += 1

# Crear dos hilos
thread1 = threading.Thread(target=increment_counter)
thread2 = threading.Thread(target=increment_counter)

# Iniciar los hilos
thread1.start()
thread2.start()

# Esperar a que los hilos terminen
thread1.join()
thread2.join()

print(f"Counter: {counter}")

En este ejemplo, se utiliza un bloqueo (lock) para asegurar que solo un hilo pueda incrementar la variable counter a la vez, evitando la condición de carrera.

Conclusión

La concurrencia es una herramienta poderosa en los sistemas operativos que permite la ejecución simultánea de múltiples procesos e hilos, mejorando la eficiencia y el rendimiento del sistema. Sin embargo, también introduce desafíos como condiciones de carrera, interbloqueos e inanición. El uso adecuado de mecanismos de sincronización como bloqueos, semáforos y monitores es crucial para gestionar estos desafíos y asegurar un comportamiento correcto y predecible en aplicaciones concurrentes.
