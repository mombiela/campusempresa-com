Encapsulación en Delphi/Object Pascal
Interfaces ►

La encapsulación es un concepto fundamental en la Programación Orientada a Objetos (OOP) que ayuda a agrupar los datos (atributos) y métodos (funciones o procedimientos) que operan sobre los datos en una sola unidad, conocida como clase. También restringe el acceso directo a algunos de los componentes del objeto, lo cual es un medio para prevenir interferencias accidentales y el mal uso de los datos.

Conceptos Clave de la Encapsulación

  • Clase: Un plano para crear objetos. Encapsula datos para el objeto y métodos para manipular esos datos.
  • Objeto: Una instancia de una clase.
  • Especificadores de Acceso: Palabras clave que establecen la accesibilidad de clases, métodos y otros miembros. Los especificadores de acceso comunes en Delphi/Object Pascal son private, protected, public y published.

Especificadores de Acceso

  • Private: Los miembros declarados como privados son accesibles solo dentro de la misma unidad.
  • Protected: Los miembros declarados como protegidos son accesibles dentro de la misma unidad y por clases descendientes.
  • Public: Los miembros declarados como públicos son accesibles desde cualquier código que pueda ver la clase.
  • Published: Similar a public, pero también permite que las propiedades sean transmitidas y examinadas en tiempo de ejecución.

Ejemplo de Encapsulación en Delphi/Object Pascal

Vamos a crear una clase simple para demostrar la encapsulación:

unit UPerson;

interface

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
    procedure SetName(const Value: string);
    procedure SetAge(Value: Integer);
  public
    constructor Create(Name: string; Age: Integer);
    function GetDetails: string;
    property Name: string read FName write SetName;
    property Age: Integer read FAge write SetAge;
  end;

implementation

{ TPerson }

constructor TPerson.Create(Name: string; Age: Integer);
begin
  FName := Name;
  FAge := Age;
end;

procedure TPerson.SetName(const Value: string);
begin
  if Value <> '' then
    FName := Value;
end;

procedure TPerson.SetAge(Value: Integer);
begin
  if (Value > 0) and (Value < 150) then
    FAge := Value;
end;

function TPerson.GetDetails: string;
begin
  Result := Format('Name: %s, Age: %d', [FName, FAge]);
end;

end.

Explicación

  • Declaración de la Clase: La clase TPerson se declara con campos privados FName y FAge para almacenar el nombre y la edad de una persona.
  • Constructor: El constructor Create inicializa los campos.
  • Métodos de Propiedad: SetName y SetAge son métodos privados para establecer los valores de FName y FAge con validación.
  • Propiedades: Las propiedades Name y Age proporcionan acceso controlado a los campos privados.
  • Método Público: GetDetails es un método público para recuperar los detalles de la persona.

Ejercicios

  1. Crear una Nueva Clase: Crea una clase TCar con campos privados para Make, Model y Year. Implementa propiedades y métodos para encapsular estos campos.
  2. Modificar la Clase: Añade validación al método SetYear para asegurar que el año esté dentro de un rango razonable (por ejemplo, de 1886 al año actual).
  3. Usar la Clase: Escribe un programa que cree una instancia de TCar, establezca sus propiedades y imprima los detalles.

Conclusión

La encapsulación es una característica poderosa en Delphi/Object Pascal que ayuda a organizar el código, proteger los datos y mejorar la mantenibilidad. Al usar clases y especificadores de acceso, puedes controlar cómo se accede y modifica la información, asegurando que tus objetos se usen correctamente y de manera segura.
