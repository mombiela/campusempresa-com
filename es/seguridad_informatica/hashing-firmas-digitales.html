Introducción

En esta sección del curso sobre Fundamentos de Seguridad Informática, exploraremos dos conceptos fundamentales: el hashing y las firmas digitales. Ambos son esenciales para garantizar la integridad y autenticidad de la información en sistemas informáticos.

Hashing

El hashing es un proceso que transforma una entrada de datos (de cualquier tamaño) en una salida de tamaño fijo, generalmente una cadena de caracteres alfanuméricos. Esta salida se conoce como hash o resumen criptográfico.

Características del Hashing

  • Determinístico: La misma entrada siempre produce el mismo hash.
  • Rápido de calcular: El hash de cualquier entrada debe ser calculable en un tiempo razonable.
  • Resistencia a colisiones: Es difícil encontrar dos entradas diferentes que produzcan el mismo hash.
  • Resistencia a preimágenes: Es difícil revertir el hash para obtener la entrada original.
  • Resistencia a la segunda preimagen: Es difícil encontrar una segunda entrada que produzca el mismo hash que una entrada dada.

Algoritmos de Hashing Comunes

  • MD5: Produce un hash de 128 bits. No es seguro para aplicaciones criptográficas debido a vulnerabilidades conocidas.
  • SHA-1: Produce un hash de 160 bits. También considerado inseguro para aplicaciones criptográficas.
  • SHA-256: Parte de la familia SHA-2, produce un hash de 256 bits y es ampliamente utilizado en aplicaciones de seguridad.

Ejemplo de Código en Python

import hashlib

# Texto de entrada
data = "Seguridad Informática"

# Crear un objeto hash SHA-256
hash_object = hashlib.sha256(data.encode())

# Obtener el hash en formato hexadecimal
hex_dig = hash_object.hexdigest()

print("Hash SHA-256:", hex_dig)

Aplicaciones del Hashing

  • Verificación de integridad de archivos: Comparar el hash de un archivo descargado con el hash proporcionado por el proveedor.
  • Almacenamiento de contraseñas: Guardar el hash de las contraseñas en lugar de las contraseñas en sí.
  • Firmas digitales: Utilizar el hash de un mensaje como parte del proceso de firma digital.

Firmas Digitales

Las firmas digitales son un mecanismo criptográfico que permite verificar la autenticidad y la integridad de un mensaje, software o documento digital. Utilizan criptografía asimétrica, que involucra un par de claves: una clave privada y una clave pública.

Proceso de Firma Digital

  1. Generación del Hash: Se calcula el hash del mensaje o documento.
  2. Firma del Hash: El hash se cifra con la clave privada del remitente, creando la firma digital.
  3. Envío del Mensaje y la Firma: El mensaje y la firma digital se envían al destinatario.
  4. Verificación de la Firma: El destinatario descifra la firma digital usando la clave pública del remitente y compara el hash resultante con el hash del mensaje recibido.

Ejemplo de Código en Python

from Crypto.PublicKey import RSA
from Crypto.Signature import pkcs1_15
from Crypto.Hash import SHA256

# Generar un par de claves RSA
key = RSA.generate(2048)
private_key = key.export_key()
public_key = key.publickey().export_key()

# Mensaje a firmar
message = b"Seguridad Informática"

# Crear el hash del mensaje
hash_object = SHA256.new(message)

# Firmar el hash con la clave privada
signature = pkcs1_15.new(key).sign(hash_object)

# Verificar la firma con la clave pública
try:
    pkcs1_15.new(key.publickey()).verify(hash_object, signature)
    print("La firma es válida.")
except (ValueError, TypeError):
    print("La firma no es válida.")

Aplicaciones de las Firmas Digitales

  • Autenticación de mensajes: Garantizar que el mensaje proviene del remitente legítimo.
  • Integridad de documentos: Asegurar que el documento no ha sido alterado.
  • Certificados digitales: Verificar la identidad de entidades en comunicaciones seguras.

Conclusión

El hashing y las firmas digitales son componentes cruciales en la seguridad informática. El hashing permite verificar la integridad de los datos, mientras que las firmas digitales aseguran la autenticidad y la integridad de los mensajes y documentos. Comprender estos conceptos y su implementación es esencial para cualquier profesional de TI que trabaje en el campo de la seguridad informática.
