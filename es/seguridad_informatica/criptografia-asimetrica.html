Introducción

La criptografía asimétrica, también conocida como criptografía de clave pública, es un método de cifrado que utiliza un par de claves: una clave pública y una clave privada. Este tipo de criptografía es fundamental para asegurar la comunicación en redes y sistemas informáticos.

Conceptos Clave

  • Clave Pública: Una clave que puede ser distribuida abiertamente y utilizada por cualquier persona para cifrar mensajes destinados al propietario de la clave privada correspondiente.
  • Clave Privada: Una clave que debe mantenerse en secreto y es utilizada para descifrar los mensajes cifrados con la clave pública correspondiente.
  • Cifrado: El proceso de convertir información legible en un formato codificado que solo puede ser leído por alguien que posea la clave privada correspondiente.
  • Descifrado: El proceso de convertir la información cifrada de vuelta a su formato original utilizando la clave privada.
  • Firma Digital: Un mecanismo que utiliza criptografía asimétrica para verificar la autenticidad e integridad de un mensaje o documento.

Funcionamiento de la Criptografía Asimétrica

  1. Generación de Claves: Se genera un par de claves (pública y privada).
  2. Distribución de la Clave Pública: La clave pública se distribuye a cualquier persona que desee enviar un mensaje seguro.
  3. Cifrado del Mensaje: El remitente cifra el mensaje utilizando la clave pública del destinatario.
  4. Transmisión del Mensaje: El mensaje cifrado se envía al destinatario.
  5. Descifrado del Mensaje: El destinatario utiliza su clave privada para descifrar el mensaje.

Ejemplo de Código en Python

A continuación, se muestra un ejemplo básico de cómo se puede implementar la criptografía asimétrica utilizando la biblioteca cryptography en Python.

from cryptography.hazmat.primitives.asymmetric import rsa
from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.primitives.asymmetric import padding
from cryptography.hazmat.primitives import hashes

# Generación de claves
private_key = rsa.generate_private_key(
    public_exponent=65537,
    key_size=2048,
)

public_key = private_key.public_key()

# Serialización de la clave pública
public_pem = public_key.public_bytes(
    encoding=serialization.Encoding.PEM,
    format=serialization.PublicFormat.SubjectPublicKeyInfo
)

# Mensaje a cifrar
message = b"Este es un mensaje secreto"

# Cifrado del mensaje
ciphertext = public_key.encrypt(
    message,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None
    )
)

# Descifrado del mensaje
plaintext = private_key.decrypt(
    ciphertext,
    padding.OAEP(
        mgf=padding.MGF1(algorithm=hashes.SHA256()),
        algorithm=hashes.SHA256(),
        label=None
    )
)

print("Mensaje original:", plaintext.decode())

Comparación entre Criptografía Simétrica y Asimétrica

| Característica | Criptografía Simétrica | Criptografía Asimétrica | |------------------------|---------------------------------|-----------------------------------| | Claves | Una sola clave compartida | Par de claves (pública y privada) | | Velocidad | Más rápida | Más lenta | | Seguridad | Menos segura si la clave es comprometida | Más segura debido a la separación de claves | | Uso | Cifrado de datos en masa | Intercambio seguro de claves, firmas digitales |

Conclusión

La criptografía asimétrica es una herramienta poderosa en la seguridad informática, proporcionando un método seguro para el intercambio de información y la verificación de la autenticidad de los mensajes. Aunque es más lenta que la criptografía simétrica, su capacidad para utilizar pares de claves la hace indispensable para muchas aplicaciones modernas, como el correo electrónico seguro y las transacciones en línea.
