Introducción a la Concurrencia y Paralelismo

La concurrencia y el paralelismo son conceptos esenciales en la programación moderna que permiten la ejecución eficiente de tareas. En esta sección, exploraremos cómo se implementan estos conceptos en ALGOL, un lenguaje de programación de alto nivel que fue influyente en el desarrollo de muchos lenguajes modernos.

Conceptos Clave

  • Concurrencia: La capacidad de un programa para gestionar múltiples tareas al mismo tiempo.
  • Paralelismo: La capacidad de un programa para ejecutar múltiples tareas simultáneamente, típicamente en múltiples procesadores o núcleos.
  • Procesos: Unidades independientes de ejecución.
  • Hilos: Unidades más pequeñas de ejecución dentro de un proceso.

Concurrencia en ALGOL

ALGOL no tiene soporte incorporado para la concurrencia como se ve en los lenguajes modernos, pero puede ser simulada usando ciertos constructos y técnicas.

Simulando Concurrencia

Una forma de simular la concurrencia en ALGOL es utilizando corutinas. Las corutinas permiten que múltiples rutinas cooperen cediendo el control entre ellas en puntos específicos.

Ejemplo: Corutina en ALGOL

begin
  integer procedure coroutine1;
  integer procedure coroutine2;
  integer i;

  coroutine1 := begin
    for i := 1 step 1 until 5 do
      begin
        outinteger(i);
        yield;
      end;
    0
  end;

  coroutine2 := begin
    for i := 6 step 1 until 10 do
      begin
        outinteger(i);
        yield;
      end;
    0
  end;

  call coroutine1;
  call coroutine2;
end
  • Explicación:
    • coroutine1 y coroutine2 se definen como procedimientos.
    • Cada corutina imprime una secuencia de enteros y luego cede el control.
    • La instrucción yield se usa para simular la transferencia de control entre corutinas.

Paralelismo en ALGOL

El paralelismo en ALGOL se puede lograr mediante el uso de bibliotecas externas o interfacing con lenguajes de bajo nivel que soportan la ejecución paralela.

Usando Bibliotecas Externas

Aunque ALGOL en sí no soporta el paralelismo directamente, se pueden usar bibliotecas externas o sistemas que proporcionen capacidades de ejecución paralela.

Ejemplo: Pseudocódigo para Ejecución Paralela

begin
  integer procedure task1;
  integer procedure task2;

  task1 := begin
    outstring("Task 1 is running");
    0
  end;

  task2 := begin
    outstring("Task 2 is running");
    0
  end;

  parallel begin
    call task1;
    call task2;
  end;
end
  • Explicación:
    • task1 y task2 se definen como procedimientos.
    • El bloque parallel es una representación conceptual de la ejecución paralela.
    • En la práctica, esto requeriría un sistema o biblioteca externa para gestionar la ejecución paralela.

Comparación de Concurrencia y Paralelismo

| Característica | Concurrencia | Paralelismo | |----------------|---------------------------------------|--------------------------------------| | Definición | Gestionar múltiples tareas simultáneamente | Ejecutar múltiples tareas simultáneamente | | Implementación | Corutinas, multitarea cooperativa | Multihilos, multiprocesamiento | | Caso de Uso | Tareas dependientes de I/O, interfaces de usuario | Tareas dependientes de CPU, computación científica |

Ejercicios

  1. Simular Concurrencia: Escriba un programa en ALGOL que simule la concurrencia usando corutinas para alternar entre dos tareas.
  2. Ejecución Paralela: Investigue y escriba un breve informe sobre cómo podría integrar ALGOL con una biblioteca o sistema moderno de ejecución paralela.

Conclusión

La concurrencia y el paralelismo son técnicas poderosas para mejorar la eficiencia y el rendimiento de los programas. Aunque ALGOL no soporta nativamente estos conceptos, pueden ser simulados o logrados mediante medios externos. Entender estos conceptos en el contexto de ALGOL proporciona una base para explorar implementaciones más avanzadas en lenguajes de programación modernos.
