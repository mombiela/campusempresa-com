La herencia de plantillas es una característica poderosa en Django que te permite crear una plantilla base y extenderla para crear plantillas más específicas. Esto ayuda a mantener una apariencia y sensación consistentes en tu sitio web y facilita la gestión y actualización de tus plantillas.

Conceptos Clave

  • Plantilla Base: Una plantilla general que contiene elementos comunes como encabezados, pies de página y barras de navegación.
  • Plantilla Hija: Una plantilla más específica que extiende la plantilla base y sobrescribe ciertas secciones.
  • Bloques: Secciones en la plantilla base que pueden ser sobrescritas por plantillas hijas.

Creando una Plantilla Base

La plantilla base es la base de tu estructura de herencia de plantillas. Contiene el diseño y los elementos comunes que se compartirán en múltiples páginas.

Ejemplo: `base.html`

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}Mi Sitio Web{% endblock %}</title>
</head>
<body>
    <header>
        <h1>Bienvenido a Mi Sitio Web</h1>
        <nav>
            <ul>
                <li><a href="/">Inicio</a></li>
                <li><a href="/about/">Acerca de</a></li>
                <li><a href="/contact/">Contacto</a></li>
            </ul>
        </nav>
    </header>
    <main>
        {% block content %}
        <!-- El contenido predeterminado va aquí -->
        {% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 Mi Sitio Web</p>
    </footer>
</body>
</html>

En este ejemplo:

  • {% block title %} y {% block content %} son marcadores de posición que pueden ser sobrescritos por plantillas hijas.

Creando una Plantilla Hija

Una plantilla hija extiende la plantilla base y proporciona contenido específico para los marcadores de posición definidos en la plantilla base.

Ejemplo: `home.html`

{% extends "base.html" %}

{% block title %}Inicio - Mi Sitio Web{% endblock %}

{% block content %}
    <h2>Página de Inicio</h2>
    <p>Bienvenido a la página de inicio de mi sitio web.</p>
{% endblock %}

En este ejemplo:

  • {% extends "base.html" %} le dice a Django que esta plantilla debe usar base.html como su base.
  • {% block title %} y {% block content %} sobrescriben los bloques correspondientes en la plantilla base.

Usando la Herencia de Plantillas en Vistas

Para renderizar una plantilla que usa herencia, simplemente especificas la plantilla hija en tu vista.

Ejemplo: `views.py`

from django.shortcuts import render

def home(request):
    return render(request, 'home.html')

En este ejemplo:

  • La vista home renderiza la plantilla home.html, que a su vez extiende base.html.

Uso Avanzado

Bloques Anidados

Puedes anidar bloques dentro de otros bloques para crear plantillas más complejas.

Ejemplo: Bloques Anidados

{% extends "base.html" %}

{% block content %}
    <h2>Página de Inicio</h2>
    <p>Bienvenido a la página de inicio de mi sitio web.</p>
    {% block extra_content %}
    <!-- Las plantillas hijas pueden agregar contenido adicional aquí -->
    {% endblock %}
{% endblock %}

Ejemplo: Sobrescribiendo Bloques Anidados

{% extends "home.html" %}

{% block extra_content %}
    <p>Este es un contenido extra agregado a la página de inicio.</p>
{% endblock %}

Conclusión

La herencia de plantillas en Django es una herramienta poderosa para crear aplicaciones web mantenibles y escalables. Al usar plantillas base y plantillas hijas, puedes asegurar una apariencia y sensación consistentes en tu sitio mientras facilitas la actualización y gestión de tus plantillas. Comprender y utilizar la herencia de plantillas mejorará significativamente tu eficiencia y la calidad de tus proyectos en Django.
