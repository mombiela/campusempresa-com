Las pruebas unitarias son una parte crucial del desarrollo de aplicaciones Django robustas y confiables. Esta sección te guiará desde los conceptos básicos de las pruebas unitarias en Django hasta técnicas más avanzadas.

Introducción a las Pruebas Unitarias

  • Definición: Las pruebas unitarias implican probar componentes individuales de tu aplicación para asegurar que funcionen como se espera.
  • Importancia: Ayuda a detectar errores temprano, asegura la confiabilidad del código y facilita la refactorización.

Configuración de Pruebas Unitarias en Django

  • Framework de Pruebas de Django: Django viene con un framework de pruebas integrado basado en el módulo unittest de Python.
  • Creación de Archivos de Prueba: Por convención, los archivos de prueba se nombran test_*.py y se colocan en el mismo directorio que el módulo que están probando.

Ejemplo: Caso de Prueba Básico

# tests.py
from django.test import TestCase
from .models import MyModel

class MyModelTestCase(TestCase):
    def setUp(self):
        MyModel.objects.create(name="test1", value=10)
        MyModel.objects.create(name="test2", value=20)

    def test_my_model_values(self):
        """MyModel values are correctly identified"""
        test1 = MyModel.objects.get(name="test1")
        test2 = MyModel.objects.get(name="test2")
        self.assertEqual(test1.value, 10)
        self.assertEqual(test2.value, 20)
  • Explicación:
    • TestCase es una clase de django.test que proporciona un framework para escribir pruebas.
    • El método setUp se usa para crear datos iniciales.
    • test_my_model_values es un método de prueba que verifica si los valores se asignan correctamente.

Ejecutando Pruebas

  • Comando: Usa python manage.py test para ejecutar todas las pruebas en tu proyecto Django.
  • Salida: El comando mostrará los resultados de las pruebas, indicando cuáles pasaron y cuáles fallaron.

Escribiendo Pruebas Más Complejas

  • Pruebas de Vistas: Puedes probar vistas simulando solicitudes HTTP.
  • Pruebas de Formularios: Asegúrate de que los formularios validen y procesen los datos correctamente.
  • Pruebas de Modelos: Verifica métodos y propiedades del modelo.

Ejemplo: Prueba de una Vista

# tests.py
from django.test import TestCase, Client
from django.urls import reverse

class MyViewTestCase(TestCase):
    def setUp(self):
        self.client = Client()

    def test_my_view(self):
        response = self.client.get(reverse('my_view'))
        self.assertEqual(response.status_code, 200)
        self.assertContains(response, "Expected Content")
  • Explicación:
    • Client se usa para simular un navegador web.
    • reverse se usa para obtener la URL de una vista por su nombre.
    • assertContains verifica si la respuesta contiene el contenido esperado.

Técnicas Avanzadas de Pruebas

  • Mocking: Usa la biblioteca unittest.mock para simular dependencias externas.
  • Fixtures: Conjuntos de datos predefinidos que pueden cargarse en la base de datos para pruebas.
  • Cobertura de Pruebas: Mide cuánto de tu código está probado usando herramientas como coverage.py.

Ejemplo: Usando Mocking

# tests.py
from django.test import TestCase
from unittest.mock import patch
from .external_service import ExternalService

class ExternalServiceTestCase(TestCase):
    @patch('myapp.external_service.ExternalService.get_data')
    def test_external_service(self, mock_get_data):
        mock_get_data.return_value = {'key': 'value'}
        response = ExternalService().get_data()
        self.assertEqual(response, {'key': 'value'})
  • Explicación:
    • patch se usa para reemplazar el método get_data de ExternalService con un mock.
    • mock_get_data.return_value establece el valor de retorno del método simulado.

Conclusión

Las pruebas unitarias son una práctica esencial para mantener la calidad y confiabilidad de tus aplicaciones Django. Comenzando con pruebas básicas e incorporando gradualmente escenarios más complejos, puedes asegurar que tu base de código permanezca robusta y mantenible. Recuerda ejecutar tus pruebas con frecuencia y apunta a una alta cobertura de pruebas para detectar problemas temprano en el proceso de desarrollo.
