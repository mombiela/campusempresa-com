Introducción

En esta sección, exploraremos cómo Django maneja el enrutamiento de URL y las vistas. El enrutamiento de URL es el mecanismo por el cual Django empareja una URL solicitada con la función de vista apropiada. Las vistas son funciones de Python que toman solicitudes web y devuelven respuestas web. Comprender estos conceptos es crucial para construir aplicaciones web con Django.

Enrutamiento de URL

Conceptos Clave

  • URLconf: Un módulo que mapea patrones de URL a vistas.
  • Función Path: Una función utilizada para definir patrones de URL.
  • Expresiones Regulares: Utilizadas para un emparejamiento de URL más complejo.

Enrutamiento Básico de URL

En Django, el enrutamiento de URL se define en un archivo urls.py. Aquí hay un ejemplo simple:

# mysite/urls.py
from django.contrib import admin
from django.urls import path
from myapp import views

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', views.home, name='home'),
    path('about/', views.about, name='about'),
]

Explicación

  • path('admin/', admin.site.urls): Enruta la URL /admin/ al sitio de administración de Django.
  • path('', views.home, name='home'): Enruta la URL raíz (/) a la vista home en myapp/views.py.
  • path('about/', views.about, name='about'): Enruta la URL /about/ a la vista about en myapp/views.py.

Enrutamiento Avanzado de URL

Para patrones de URL más complejos, puedes usar expresiones regulares con la función re_path:

# mysite/urls.py
from django.urls import re_path
from myapp import views

urlpatterns = [
    re_path(r'^articles/(?P<year>[0-9]{4})/$', views.year_archive),
]

Explicación

  • re_path(r'^articles/(?P<year>[0-9]{4})/$', views.year_archive): Enruta URLs como /articles/2023/ a la vista year_archive, capturando el año como un parámetro.

Vistas

Conceptos Clave

  • Vistas Basadas en Funciones (FBVs): Simples funciones de Python que manejan solicitudes.
  • Vistas Basadas en Clases (CBVs): Vistas que se representan como clases, proporcionando más estructura y reutilización.
  • HttpResponse: El objeto de respuesta devuelto por las vistas.

Vistas Basadas en Funciones

Aquí hay una simple vista basada en funciones:

# myapp/views.py
from django.http import HttpResponse

def home(request):
    return HttpResponse("¡Bienvenido a la Página Principal!")

Explicación

  • home(request): Una función que toma una solicitud HTTP y devuelve una respuesta HTTP con el texto "¡Bienvenido a la Página Principal!".

Vistas Basadas en Clases

Las vistas basadas en clases proporcionan más funcionalidad y son más reutilizables. Aquí hay un ejemplo:

# myapp/views.py
from django.views import View
from django.http import HttpResponse

class HomeView(View):
    def get(self, request):
        return HttpResponse("¡Bienvenido a la Página Principal!")

Explicación

  • HomeView(View): Una clase que hereda de la clase View de Django.
  • get(self, request): Un método que maneja solicitudes GET y devuelve una respuesta HTTP.

Usando Plantillas en Vistas

Las vistas a menudo renderizan plantillas para generar HTML. Aquí te mostramos cómo hacerlo:

# myapp/views.py
from django.shortcuts import render

def home(request):
    return render(request, 'home.html')

Explicación

  • render(request, 'home.html'): Renderiza la plantilla home.html y devuelve el HTML generado como respuesta.

Resumen

En esta sección, cubrimos los conceptos básicos del enrutamiento de URL y las vistas en Django. Aprendimos cómo definir patrones de URL usando las funciones path y re_path, y cómo crear tanto vistas basadas en funciones como vistas basadas en clases. También vimos cómo renderizar plantillas en vistas. Comprender estos conceptos es esencial para construir aplicaciones web dinámicas con Django.
