Introducción

En este módulo, aprenderemos a construir una API REST usando Django. Cubriremos los principios básicos de RESTful, la configuración de Django y la creación de endpoints. Al final de este módulo, serás capaz de crear una API REST completamente funcional.

Prerrequisitos

Antes de comenzar, asegúrate de tener:

  • Conocimientos básicos de Python y Django.
  • Python instalado en tu máquina.
  • Django instalado (pip install django).

Configuración del Proyecto

  1. Crear un Proyecto Django:

    django-admin startproject myapi
cd myapi

  2. Crear una Aplicación Django:

    python manage.py startapp api

  3. Agregar la Aplicación a INSTALLED_APPS: Edita myapi/settings.py:

    INSTALLED_APPS = [
    ...
    'api',
    'rest_framework',  # Agregar Django REST framework
]

Creación de Modelos

Define la estructura de datos para tu API usando modelos de Django.

  1. Crear un Modelo: Edita api/models.py:

    from django.db import models

class Item(models.Model):
    name = models.CharField(max_length=100)
    description = models.TextField()
    price = models.DecimalField(max_digits=10, decimal_places=2)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.name

  2. Aplicar Migraciones:

    python manage.py makemigrations
python manage.py migrate

Serializadores

Los serializadores convierten tipos de datos complejos, como querysets e instancias de modelos, en tipos de datos nativos de Python que luego pueden ser fácilmente renderizados en JSON o XML.

  1. Crear un Serializador: Edita api/serializers.py: 
    from rest_framework import serializers
from .models import Item

class ItemSerializer(serializers.ModelSerializer):
    class Meta:
        model = Item
        fields = '__all__'

Vistas

Las vistas manejan la lógica de tus endpoints de la API.

  1. Crear Vistas: Edita api/views.py: 
    from rest_framework import generics
from .models import Item
from .serializers import ItemSerializer

class ItemListCreate(generics.ListCreateAPIView):
    queryset = Item.objects.all()
    serializer_class = ItemSerializer

class ItemDetail(generics.RetrieveUpdateDestroyAPIView):
    queryset = Item.objects.all()
    serializer_class = ItemSerializer

URLs

Define los patrones de URL para tus endpoints de la API.

  1. Crear Patrones de URL: Edita api/urls.py:

    from django.urls import path
from .views import ItemListCreate, ItemDetail

urlpatterns = [
    path('items/', ItemListCreate.as_view(), name='item-list-create'),
    path('items/<int:pk>/', ItemDetail.as_view(), name='item-detail'),
]

  2. Incluir URLs de la API en URLs del Proyecto: Edita myapi/urls.py:

    from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('api/', include('api.urls')),
]

Pruebas de la API

Usa herramientas como Postman o curl para probar tus endpoints de la API.

  1. Listar Items:

    curl -X GET http://127.0.0.1:8000/api/items/

  2. Crear un Item:

    curl -X POST http://127.0.0.1:8000/api/items/ -d '{"name": "Item1", "description": "Description1", "price": "10.00"}' -H "Content-Type: application/json"

  3. Recuperar un Item:

    curl -X GET http://127.0.0.1:8000/api/items/1/

  4. Actualizar un Item:

    curl -X PUT http://127.0.0.1:8000/api/items/1/ -d '{"name": "Updated Item", "description": "Updated Description", "price": "20.00"}' -H "Content-Type: application/json"

  5. Eliminar un Item:

    curl -X DELETE http://127.0.0.1:8000/api/items/1/

Conclusión

En este módulo, cubrimos los conceptos básicos de la construcción de una API REST usando Django. Pasamos por la configuración de un proyecto Django, la creación de modelos, serializadores, vistas y la definición de patrones de URL. También probamos nuestra API usando comandos curl. Con estos fundamentos, ahora puedes expandir tu API para incluir características más complejas y manejar varios tipos de datos.
