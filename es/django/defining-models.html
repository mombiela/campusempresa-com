En esta sección, exploraremos cómo definir modelos en Django. Los modelos son una parte crucial de cualquier aplicación Django, ya que definen la estructura de tu base de datos. Cubriremos los conceptos básicos de la creación de modelos, la adición de campos y la comprensión de los métodos del modelo.

¿Qué es un Modelo?

Un modelo en Django es una clase de Python que representa una tabla en tu base de datos. Cada atributo de la clase corresponde a un campo de la base de datos. Los modelos de Django proporcionan una capa de abstracción que te permite interactuar con tu base de datos usando código Python en lugar de SQL puro.

Creando un Modelo Básico

Para crear un modelo, necesitas definir una clase que herede de django.db.models.Model. Comencemos con un ejemplo simple:

from django.db import models

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.CharField(max_length=100)
    published_date = models.DateField()
    isbn = models.CharField(max_length=13)

Explicación:

  • title: Un CharField para almacenar el título del libro. El parámetro max_length especifica la longitud máxima de la cadena.
  • author: Otro CharField para el nombre del autor.
  • published_date: Un DateField para almacenar la fecha de publicación.
  • isbn: Un CharField para el número ISBN del libro.

Tipos de Campo

Django proporciona varios tipos de campo para manejar diferentes tipos de datos. Aquí hay algunos tipos de campo comúnmente usados:

| Tipo de Campo | Descripción | |------------------|---------------------------------------------------| | CharField | Un campo de cadena, para cadenas de tamaño pequeño a grande.| | TextField | Un campo de texto grande. | | IntegerField | Un campo de entero. | | FloatField | Un campo de número de punto flotante. | | BooleanField | Un campo de verdadero/falso. | | DateField | Un campo de fecha. | | DateTimeField | Un campo de fecha y hora. | | EmailField | Un campo para direcciones de correo electrónico. | | URLField | Un campo para URLs. |

Añadiendo Métodos a los Modelos

Puedes añadir métodos a tus modelos para encapsular la lógica de negocio. Por ejemplo, podrías querer añadir un método para obtener el nombre completo de un autor:

class Author(models.Model):
    first_name = models.CharField(max_length=50)
    last_name = models.CharField(max_length=50)

    def full_name(self):
        return f"{self.first_name} {self.last_name}"

Explicación:

  • full_name: Este método concatena los campos first_name y last_name para devolver el nombre completo del autor.

Opciones Meta

Los modelos de Django vienen con una clase Meta que te permite especificar varias opciones, como el orden y los nombres descriptivos. Aquí hay un ejemplo:

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.CharField(max_length=100)
    published_date = models.DateField()
    isbn = models.CharField(max_length=13)

    class Meta:
        ordering = ['published_date']
        verbose_name = 'Libro'
        verbose_name_plural = 'Libros'

Explicación:

  • ordering: Especifica el orden predeterminado para el modelo. En este caso, los libros se ordenarán por su published_date.
  • verbose_name: Un nombre legible para el modelo.
  • verbose_name_plural: La forma plural del nombre legible.

Conclusión

En esta sección, cubrimos los conceptos básicos de la definición de modelos en Django. Aprendimos cómo crear un modelo, añadir campos y usar varios tipos de campo. También exploramos cómo añadir métodos a los modelos y usar la clase Meta para opciones adicionales. Entender cómo definir modelos es esencial para trabajar con Django, ya que forma la base de la estructura de datos de tu aplicación.
