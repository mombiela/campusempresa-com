En esta sección, exploraremos algunos de los paquetes más útiles de Django que pueden ayudarte a mejorar tus proyectos de Django. Estos paquetes pueden simplificar tareas comunes, mejorar el rendimiento y añadir nuevas funcionalidades a tus aplicaciones.

1. Django Rest Framework (DRF)

Django Rest Framework es un conjunto de herramientas poderoso y flexible para construir APIs web. Es ampliamente utilizado para crear APIs RESTful con Django.

  • Características Clave:

    • Serialización que soporta tanto fuentes de datos ORM como no ORM.
    • Autenticación y permisos.
    • Conjuntos de vistas y enrutadores para un desarrollo rápido de APIs.
    • API navegable para pruebas e interacción fáciles.

  • Instalación:

    pip install djangorestframework

  • Ejemplo Básico:

    # serializers.py
from rest_framework import serializers
from .models import MyModel

class MyModelSerializer(serializers.ModelSerializer):
    class Meta:
        model = MyModel
        fields = '__all__'

# views.py
from rest_framework import viewsets
from .models import MyModel
from .serializers import MyModelSerializer

class MyModelViewSet(viewsets.ModelViewSet):
    queryset = MyModel.objects.all()
    serializer_class = MyModelSerializer

# urls.py
from django.urls import path, include
from rest_framework.routers import DefaultRouter
from .views import MyModelViewSet

router = DefaultRouter()
router.register(r'mymodel', MyModelViewSet)

urlpatterns = [
    path('', include(router.urls)),
]

2. Django Allauth

Django Allauth es un conjunto integrado de aplicaciones de Django que abordan la autenticación, el registro, la gestión de cuentas y la autenticación de cuentas de terceros (sociales).

  • Características Clave:

    • Registro y autenticación de usuarios.
    • Autenticación de cuentas sociales (por ejemplo, Facebook, Google).
    • Verificación de correo electrónico y gestión de contraseñas.

  • Instalación:

    pip install django-allauth

  • Configuración Básica:

    # settings.py
INSTALLED_APPS = [
    ...
    'django.contrib.sites',
    'allauth',
    'allauth.account',
    'allauth.socialaccount',
    'allauth.socialaccount.providers.google',
    ...
]

SITE_ID = 1

AUTHENTICATION_BACKENDS = (
    'django.contrib.auth.backends.ModelBackend',
    'allauth.account.auth_backends.AuthenticationBackend',
)

# urls.py
from django.urls import path, include

urlpatterns = [
    ...
    path('accounts/', include('allauth.urls')),
    ...
]

3. Django Debug Toolbar

Django Debug Toolbar es un conjunto configurable de paneles que muestran varias informaciones de depuración sobre la solicitud/respuesta actual.

  • Características Clave:

    • Consultas SQL y su tiempo de ejecución.
    • Uso de caché.
    • Tiempos de renderizado de plantillas.
    • Encabezados de solicitud y respuesta.

  • Instalación:

    pip install django-debug-toolbar

  • Configuración Básica:

    # settings.py
INSTALLED_APPS = [
    ...
    'debug_toolbar',
    ...
]

MIDDLEWARE = [
    ...
    'debug_toolbar.middleware.DebugToolbarMiddleware',
    ...
]

INTERNAL_IPS = [
    '127.0.0.1',
]

# urls.py
from django.urls import path, include

urlpatterns = [
    ...
    path('__debug__/', include('debug_toolbar.urls')),
    ...
]

4. Django Crispy Forms

Django Crispy Forms es una aplicación de Django que te ayuda a gestionar formularios de Django. Te permite controlar el comportamiento de renderizado de tus formularios de una manera muy elegante y DRY.

  • Características Clave:

    • Formularios DRY.
    • Soporte para Bootstrap, Foundation y otros frameworks CSS.
    • Diseños de formularios personalizados.

  • Instalación:

    pip install django-crispy-forms

  • Ejemplo Básico:

    # settings.py
INSTALLED_APPS = [
    ...
    'crispy_forms',
    ...
]

CRISPY_TEMPLATE_PACK = 'bootstrap4'

# forms.py
from django import forms
from crispy_forms.helper import FormHelper
from crispy_forms.layout import Submit

class MyForm(forms.Form):
    name = forms.CharField(max_length=100)
    email = forms.EmailField()

    def __init__(self, *args, **kwargs):
        super(MyForm, self).__init__(*args, **kwargs)
        self.helper = FormHelper()
        self.helper.form_method = 'post'
        self.helper.add_input(Submit('submit', 'Submit'))

5. Django Celery

Django Celery es una cola de tareas/job asíncrona basada en el paso de mensajes distribuidos. Está enfocada en la operación en tiempo real pero también soporta la programación.

  • Características Clave:

    • Ejecución de tareas asíncronas.
    • Programación de tareas.
    • Integración con el ORM y caché de Django.

  • Instalación:

    pip install celery

  • Configuración Básica:

    # settings.py
CELERY_BROKER_URL = 'redis://localhost:6379/0'
CELERY_RESULT_BACKEND = 'redis://localhost:6379/0'

# celery.py
from __future__ import absolute_import, unicode_literals
import os
from celery import Celery

os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'myproject.settings')

app = Celery('myproject')
app.config_from_object('django.conf:settings', namespace='CELERY')
app.autodiscover_tasks()

# tasks.py
from celery import shared_task

@shared_task
def add(x, y):
    return x + y
