Introducción

Entender la estructura de un proyecto Django y cómo gestionar aplicaciones Django es crucial para desarrollar aplicaciones web escalables y mantenibles. Esta sección cubrirá los conceptos básicos de la estructura de proyectos y aplicaciones en Django, cómo crear y gestionar aplicaciones, y las mejores prácticas para organizar tu código.

Estructura del Proyecto Django

Cuando creas un nuevo proyecto Django, Django configura una estructura de directorios por defecto. Comencemos creando un nuevo proyecto Django para ver esta estructura en acción.

Creando un Nuevo Proyecto Django

Para crear un nuevo proyecto Django, usa el siguiente comando:

django-admin startproject myproject

Este comando crea un directorio llamado myproject con la siguiente estructura:

myproject/
    manage.py
    myproject/
        __init__.py
        settings.py
        urls.py
        asgi.py
        wsgi.py

Archivos y Directorios Clave

  • manage.py: Una utilidad de línea de comandos que te permite interactuar con tu proyecto Django.
  • myproject/: El directorio interno con el mismo nombre que tu proyecto.
    • init.py: Un archivo vacío que le dice a Python que este directorio debe ser considerado un paquete.
    • settings.py: Configuraciones de configuración para tu proyecto Django.
    • urls.py: Declaraciones de URL para tu proyecto Django; una "tabla de contenidos" de tu sitio impulsado por Django.
    • asgi.py: Punto de entrada para servidores web compatibles con ASGI para servir tu proyecto.
    • wsgi.py: Punto de entrada para servidores web compatibles con WSGI para servir tu proyecto.

Aplicaciones Django

Un proyecto Django puede contener múltiples aplicaciones. Cada aplicación debe hacer una cosa y hacerla bien. Por ejemplo, podrías tener una aplicación para la autenticación de usuarios, otra para un blog y otra para un foro.

Creando una Nueva Aplicación Django

Para crear una nueva aplicación dentro de tu proyecto, usa el siguiente comando:

python manage.py startapp myapp

Este comando crea un directorio llamado myapp con la siguiente estructura:

myapp/
    __init__.py
    admin.py
    apps.py
    models.py
    tests.py
    views.py
    migrations/
        __init__.py

Archivos y Directorios Clave en una Aplicación

  • admin.py: Registra tus modelos con el sitio de administración de Django.
  • apps.py: Configuración para la aplicación.
  • models.py: Define tus modelos de base de datos aquí.
  • tests.py: Escribe tus pruebas aquí.
  • views.py: Define tus vistas aquí.
  • migrations/: Almacena las migraciones de la base de datos aquí.

Integrando Aplicaciones en un Proyecto

Una vez que has creado una aplicación, necesitas decirle a tu proyecto sobre ella. Esto se hace añadiendo la aplicación a la lista INSTALLED_APPS en settings.py.

Ejemplo

# myproject/settings.py

INSTALLED_APPS = [
    ...
    'myapp',
]

Mejores Prácticas para la Estructura de Proyectos y Aplicaciones

  • Mantén las aplicaciones pequeñas y enfocadas: Cada aplicación debe tener una sola responsabilidad.
  • Usa nombres significativos: Nombra tus aplicaciones y proyectos claramente para reflejar su propósito.
  • Organiza plantillas y archivos estáticos: Usa directorios dentro de cada aplicación para mantener las plantillas y archivos estáticos organizados.

Ejemplo de Estructura de Directorios

myproject/
    manage.py
    myproject/
        __init__.py
        settings.py
        urls.py
        asgi.py
        wsgi.py
    myapp/
        __init__.py
        admin.py
        apps.py
        models.py
        tests.py
        views.py
        migrations/
            __init__.py
        templates/
            myapp/
                template1.html
        static/
            myapp/
                style.css

Conclusión

Entender la estructura de proyectos y aplicaciones en Django es fundamental para desarrollar aplicaciones web eficientes y mantenibles. Siguiendo las mejores prácticas y organizando tu código adecuadamente, puedes crear aplicaciones escalables y modulares.
