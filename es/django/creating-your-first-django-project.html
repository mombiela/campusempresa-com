Introducción

En esta sección, te guiaremos a través del proceso de creación de tu primer proyecto Django. Django es un framework web de alto nivel para Python que fomenta el desarrollo rápido y un diseño limpio y pragmático. Al final de esta sección, tendrás un proyecto básico de Django en funcionamiento.

Prerrequisitos

Antes de comenzar, asegúrate de tener lo siguiente:

  • Python instalado (versión 3.6 o superior).
  • Pip (instalador de paquetes de Python).
  • Virtualenv (opcional pero recomendado para gestionar dependencias).

Configurando Tu Entorno

  1. Instalar Django: Primero, necesitas instalar Django. Puedes hacerlo usando pip. 
    pip install django
  2. Crear un Entorno Virtual: Es una buena práctica crear un entorno virtual para tu proyecto. 
    python -m venv myenv
source myenv/bin/activate  # En Windows usa `myenv\Scripts\activate`

Creando un Proyecto Django

  1. Iniciar un Nuevo Proyecto: Usa el comando django-admin para iniciar un nuevo proyecto.

    django-admin startproject myproject

    Este comando crea un nuevo directorio llamado myproject con la siguiente estructura:

    myproject/
    manage.py
    myproject/
        __init__.py
        settings.py
        urls.py
        wsgi.py

  2. Entendiendo la Estructura del Proyecto:

    • manage.py: Una utilidad de línea de comandos que te permite interactuar con este proyecto Django.
    • myproject/: El directorio interno es el paquete Python real para tu proyecto.
      • __init__.py: Un archivo vacío que le dice a Python que este directorio debe ser considerado un paquete.
      • settings.py: Configuraciones de configuración para el proyecto.
      • urls.py: Declaraciones de URL para el proyecto.
      • wsgi.py: Un punto de entrada para servidores web compatibles con WSGI para servir tu proyecto.

Ejecutando el Servidor de Desarrollo

  1. Navegar al Directorio de Tu Proyecto: 
    cd myproject
  2. Ejecutar el Servidor: 
    python manage.py runserver
    Deberías ver una salida similar a: 
    Watching for file changes with StatReloader
Performing system checks...

System check identified no issues (0 silenced).
October 10, 2023 - 14:00:00
Django version 3.2, using settings 'myproject.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.
  3. Acceder al Servidor: Abre tu navegador web y navega a http://127.0.0.1:8000/. Deberías ver la página de bienvenida de Django.

Creando una Aplicación Django

  1. Crear una Aplicación: Dentro de tu directorio de proyecto, crea una nueva aplicación.

    python manage.py startapp myapp

    Este comando crea un nuevo directorio llamado myapp con la siguiente estructura:

    myapp/
    __init__.py
    admin.py
    apps.py
    models.py
    tests.py
    views.py
    migrations/

  2. Entendiendo la Estructura de la Aplicación:

    • admin.py: Configuración para la interfaz de administración de Django.
    • apps.py: Configuración para la propia aplicación.
    • models.py: Modelos de base de datos para la aplicación.
    • tests.py: Casos de prueba para la aplicación.
    • views.py: Vistas para la aplicación.
    • migrations/: Migraciones de base de datos para la aplicación.

Configurando la Aplicación

  1. Agregar la Aplicación a Configuraciones: Abre myproject/settings.py y agrega tu aplicación a la lista INSTALLED_APPS. 
    INSTALLED_APPS = [
    ...
    'myapp',
]

Creando una Vista Simple

  1. Definir una Vista: Abre myapp/views.py y agrega una vista simple.

    from django.http import HttpResponse

def home(request):
    return HttpResponse("¡Hola, mundo!")

  2. Mapear la Vista a una URL: Abre myapp/urls.py y crea un patrón de URL.

    from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
]

  3. Incluir las URLs de la Aplicación en el Proyecto: Abre myproject/urls.py e incluye las URLs de la aplicación.

    from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('myapp.urls')),
]

Conclusión

¡Felicidades! Has creado con éxito tu primer proyecto Django y una aplicación simple con una vista básica. Ahora tienes una comprensión fundamental de cómo están estructurados los proyectos Django y cómo crear y configurar aplicaciones dentro de un proyecto. En las siguientes secciones, profundizaremos en las características de Django, incluyendo modelos, plantillas y vistas más avanzadas.
