Desplegar tu aplicación Django en Heroku es un proceso sencillo que te permite hacer que tu aplicación web sea accesible para el mundo. Esta sección te guiará a través de los pasos necesarios para desplegar tu proyecto Django en Heroku, desde configurar tu entorno hasta subir tu código en vivo.

Requisitos previos

Antes de comenzar, asegúrate de tener lo siguiente:

  • Una cuenta de Heroku: Regístrate en Heroku.
  • Heroku CLI instalado: Sigue las instrucciones de instalación en Heroku CLI.
  • Git instalado: Puedes descargarlo desde Git.

Paso 1: Prepara tu proyecto Django

Instalar paquetes requeridos

Heroku requiere algunos paquetes adicionales para ejecutar tu aplicación Django. Instala estos paquetes usando pip:

pip install gunicorn dj-database-url psycopg2-binary
  • gunicorn: Un servidor HTTP WSGI para UNIX en Python.
  • dj-database-url: Te permite usar la URL de la base de datos proporcionada por Heroku.
  • psycopg2-binary: Adaptador de base de datos PostgreSQL para Python.

Actualizar `requirements.txt`

Asegúrate de que todas tus dependencias estén listadas en requirements.txt:

pip freeze > requirements.txt

Crear un `Procfile`

Heroku usa un Procfile para determinar cómo ejecutar tu aplicación. Crea un archivo llamado Procfile en el directorio raíz de tu proyecto con el siguiente contenido:

web: gunicorn your_project_name.wsgi

Reemplaza your_project_name con el nombre de tu proyecto Django.

Configurar ajustes de la base de datos

Modifica tu settings.py para usar dj-database-url para la configuración de la base de datos. Agrega el siguiente código:

import dj_database_url

DATABASES = {
    'default': dj_database_url.config(default='postgres://localhost/your_db_name')
}

Reemplaza your_db_name con el nombre de tu base de datos local.

Configuración de archivos estáticos

Heroku requiere algunos ajustes adicionales para servir archivos estáticos. Agrega lo siguiente a tu settings.py:

import os

BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))

STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')
STATIC_URL = '/static/'

# Lugares adicionales para que collectstatic encuentre archivos estáticos.
STATICFILES_DIRS = (
    os.path.join(BASE_DIR, 'static'),
)

# Servir archivos estáticos de manera simplificada.
# https://warehouse.python.org/project/whitenoise/
MIDDLEWARE = [
    'whitenoise.middleware.WhiteNoiseMiddleware',
    # ... otros middleware ...
]

STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'

Instala whitenoise:

pip install whitenoise

Actualiza requirements.txt nuevamente:

pip freeze > requirements.txt

Paso 2: Desplegar en Heroku

Inicializar un repositorio Git

Si aún no lo has hecho, inicializa un repositorio Git en tu directorio de proyecto:

git init

Agrega todos tus archivos y haz un commit:

git add .
git commit -m "Initial commit"

Crear una aplicación en Heroku

Inicia sesión en Heroku usando la CLI:

heroku login

Crea una nueva aplicación en Heroku:

heroku create your-app-name

Reemplaza your-app-name con un nombre único para tu aplicación.

Configurar variables de entorno

Configura las variables de entorno SECRET_KEY y DEBUG:

heroku config:set SECRET_KEY='your_secret_key'
heroku config:set DEBUG=False

Reemplaza your_secret_key con una clave segura para tu aplicación.

Subir tu código a Heroku

Despliega tu código en Heroku:

git push heroku master

Ejecutar migraciones

Ejecuta las migraciones de la base de datos en Heroku:

heroku run python manage.py migrate

Recopilar archivos estáticos

Recopila archivos estáticos:

heroku run python manage.py collectstatic --noinput

Conclusión

¡Felicidades! Has desplegado exitosamente tu aplicación Django en Heroku. Tu aplicación ahora debería estar en vivo y accesible a través de la URL proporcionada por Heroku. Este proceso implica configurar tu proyecto Django para producción, configurar los paquetes necesarios y usar Git y Heroku CLI para desplegar tu código. Con estos pasos, puedes desplegar y gestionar fácilmente tus aplicaciones Django en Heroku.
