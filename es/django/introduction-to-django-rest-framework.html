Visión General

En esta sección, presentaremos el Django REST Framework (DRF), un conjunto de herramientas poderoso y flexible para construir APIs web en Django. Al final de esta sección, comprenderás los conceptos básicos de DRF, sus componentes clave y cómo crear una API simple.

Conceptos Clave

  • API Web: Una interfaz que permite que diferentes sistemas de software se comuniquen entre sí a través de la web.
  • Django REST Framework (DRF): Un conjunto de herramientas para construir APIs web en Django, que proporciona características como serialización, autenticación y conjuntos de vistas.

Configuración del Django REST Framework

Para comenzar con DRF, necesitas instalarlo en tu proyecto de Django. Sigue estos pasos:

  1. Instalar DRF:

    pip install djangorestframework

  2. Agregar DRF a las Aplicaciones Instaladas: Abre tu archivo settings.py y añade 'rest_framework' a la lista INSTALLED_APPS:

    INSTALLED_APPS = [
    ...
    'rest_framework',
]

Creando una API Simple

Vamos a crear una API simple para gestionar una lista de libros.

Paso 1: Crear una Aplicación de Django

Primero, crea una nueva aplicación de Django llamada books:

python manage.py startapp books

Paso 2: Definir el Modelo de Libro

En books/models.py, define un modelo simple de Book:

from django.db import models

class Book(models.Model):
    title = models.CharField(max_length=100)
    author = models.CharField(max_length=100)
    published_date = models.DateField()

    def __str__(self):
        return self.title

Paso 3: Crear un Serializador

Los serializadores en DRF se utilizan para convertir tipos de datos complejos, como los modelos de Django, en JSON y viceversa. Crea un serializador para el modelo Book en books/serializers.py:

from rest_framework import serializers
from .models import Book

class BookSerializer(serializers.ModelSerializer):
    class Meta:
        model = Book
        fields = '__all__'

Paso 4: Crear Vistas

En books/views.py, crea vistas para manejar las solicitudes de la API. DRF proporciona varias clases de vistas, pero comenzaremos con APIView:

from rest_framework.views import APIView
from rest_framework.response import Response
from rest_framework import status
from .models import Book
from .serializers import BookSerializer

class BookList(APIView):
    def get(self, request):
        books = Book.objects.all()
        serializer = BookSerializer(books, many=True)
        return Response(serializer.data)

    def post(self, request):
        serializer = BookSerializer(data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

Paso 5: Configurar URLs

En books/urls.py, define los patrones de URL para la API:

from django.urls import path
from .views import BookList

urlpatterns = [
    path('books/', BookList.as_view(), name='book-list'),
]

Incluye las URLs de books en el urls.py principal del proyecto:

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('api/', include('books.urls')),
]

Probando la API

Ejecuta el servidor de desarrollo de Django:

python manage.py runserver

Ahora puedes probar los endpoints de la API usando herramientas como Postman o curl:

  • Solicitud GET a /api/books/ para recuperar la lista de libros.
  • Solicitud POST a /api/books/ para agregar un nuevo libro.

Conclusión

En esta sección, cubrimos los conceptos básicos del Django REST Framework, incluyendo su configuración, la creación de una API simple y su prueba. DRF simplifica el proceso de construir APIs robustas y escalables en Django, convirtiéndolo en una herramienta esencial para el desarrollo web moderno. En las próximas secciones, profundizaremos en características más avanzadas de DRF, como conjuntos de vistas, enrutadores y autenticación.
