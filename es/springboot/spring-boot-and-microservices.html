Introducción a los Microservicios

  • Definición: La arquitectura de microservicios es un método de desarrollo de sistemas de software que intenta centrarse en construir módulos de función única con interfaces y operaciones bien definidas.
  • Beneficios:
    • Escalabilidad
    • Flexibilidad
    • Resiliencia
    • Despliegue independiente

Configuración de Spring Boot para Microservicios

  • Descripción general de Spring Boot: Spring Boot simplifica el desarrollo de nuevas aplicaciones Spring proporcionando valores predeterminados para el código y la configuración de anotaciones.
  • Creación de un proyecto Spring Boot:
    • Usa Spring Initializr (https://start.spring.io/) para generar un nuevo proyecto Spring Boot.
    • Incluye dependencias como Spring Web, Spring Data JPA y H2 Database.
@SpringBootApplication
public class MicroserviceApplication {
    public static void main(String[] args) {
        SpringApplication.run(MicroserviceApplication.class, args);
    }
}

Construyendo un Microservicio Simple

  • Creación de un Controlador REST:
    • Define un endpoint REST simple para manejar solicitudes HTTP.
@RestController
@RequestMapping("/api")
public class SimpleController {
    
    @GetMapping("/hello")
    public String sayHello() {
        return "¡Hola, Mundo!";
    }
}
  • Ejecutando la Aplicación:
    • Usa mvn spring-boot:run para iniciar la aplicación.
    • Accede al endpoint en http://localhost:8080/api/hello.

Comunicación entre Servicios

  • REST Template: Un cliente sincrónico para realizar solicitudes HTTP.
  • Feign Client: Un cliente de servicios web declarativo que facilita la escritura de clientes de servicios web.

Usando RestTemplate

@RestController
@RequestMapping("/api")
public class CommunicationController {

    @Autowired
    private RestTemplate restTemplate;

    @GetMapping("/fetch-data")
    public String fetchData() {
        String response = restTemplate.getForObject("http://other-service/api/data", String.class);
        return response;
    }
}

Usando Feign Client

@FeignClient(name = "other-service")
public interface OtherServiceClient {

    @GetMapping("/api/data")
    String getData();
}

@RestController
@RequestMapping("/api")
public class CommunicationController {

    @Autowired
    private OtherServiceClient otherServiceClient;

    @GetMapping("/fetch-data")
    public String fetchData() {
        return otherServiceClient.getData();
    }
}

Descubrimiento de Servicios con Eureka

  • Eureka Server: Actúa como un registro de servicios.
  • Eureka Client: Se registra con el Eureka Server.

Configuración de Eureka Server

@SpringBootApplication
@EnableEurekaServer
public class EurekaServerApplication {
    public static void main(String[] args) {
        SpringApplication.run(EurekaServerApplication.class, args);
    }
}

Configuración de Eureka Client

# application.yml
eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka/

API Gateway con Spring Cloud Gateway

  • Propósito: Actúa como un punto de entrada único para todos los microservicios.
  • Configuración:
    • Define rutas a diferentes microservicios.
# application.yml
spring:
  cloud:
    gateway:
      routes:
        - id: simple-service
          uri: http://localhost:8081
          predicates:
            - Path=/api/simple/**

Circuit Breaker con Resilience4j

  • Propósito: Manejar fallos de manera elegante y proporcionar mecanismos de respaldo.
  • Implementación:
    • Añade la dependencia de Resilience4j.
    • Anota métodos con @CircuitBreaker.
@RestController
@RequestMapping("/api")
public class CircuitBreakerController {

    @Autowired
    private RestTemplate restTemplate;

    @GetMapping("/fetch-data")
    @CircuitBreaker(name = "otherService", fallbackMethod = "fallback")
    public String fetchData() {
        return restTemplate.getForObject("http://other-service/api/data", String.class);
    }

    public String fallback(Throwable t) {
        return "Respuesta de respaldo";
    }
}

Seguridad en Microservicios

  • OAuth2 y JWT: Usa OAuth2 para autenticación y JWT para seguridad basada en tokens.
  • Spring Security: Integra Spring Security para asegurar endpoints.
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
            .antMatchers("/api/public/**").permitAll()
            .anyRequest().authenticated()
            .and()
            .oauth2Login();
    }
}

Conclusión

En este tema, cubrimos lo esencial para construir microservicios usando Spring Boot. Comenzamos con los conceptos básicos de la arquitectura de microservicios y pasamos a crear microservicios simples. Exploramos la comunicación entre servicios usando RestTemplate y Feign Client, el descubrimiento de servicios con Eureka, el API Gateway con Spring Cloud Gateway y la resiliencia con Resilience4j. Finalmente, tocamos la seguridad en microservicios usando OAuth2 y Spring Security. Con estos conceptos, estás bien equipado para construir y gestionar arquitecturas de microservicios robustas usando Spring Boot.
