Introducción

En esta sección, cubriremos cómo manejar los métodos HTTP principales (GET, POST, PUT, DELETE) usando Spring Boot. Entender estos métodos es crucial para construir servicios web RESTful. Comenzaremos con los conceptos básicos y gradualmente avanzaremos hacia conceptos más avanzados.

Método GET

El método GET se utiliza para recuperar información del servidor. Es idempotente, lo que significa que múltiples solicitudes idénticas deberían tener el mismo efecto que una sola solicitud.

Conceptos Clave

  • Se usa para obtener datos.
  • No debería alterar el estado del servidor.
  • Puede ser almacenado en caché.

Ejemplo

@RestController
@RequestMapping("/api")
public class ExampleController {

    @GetMapping("/items")
    public List<Item> getAllItems() {
        // Lógica para recuperar ítems
        return itemService.getAllItems();
    }
}

En este ejemplo, la anotación @GetMapping mapea las solicitudes HTTP GET al método getAllItems.

Método POST

El método POST se utiliza para enviar datos a un recurso especificado para ser procesados. A menudo resulta en un cambio en el estado del servidor o efectos secundarios.

Conceptos Clave

  • Se usa para crear nuevos recursos.
  • Puede alterar el estado del servidor.
  • No es idempotente.

Ejemplo

@RestController
@RequestMapping("/api")
public class ExampleController {

    @PostMapping("/items")
    public Item createItem(@RequestBody Item item) {
        // Lógica para crear un ítem
        return itemService.createItem(item);
    }
}

Aquí, la anotación @PostMapping mapea las solicitudes HTTP POST al método createItem. La anotación @RequestBody se usa para vincular el cuerpo de la solicitud al parámetro del método.

Método PUT

El método PUT se utiliza para actualizar un recurso existente o crear un nuevo recurso si no existe.

Conceptos Clave

  • Se usa para actualizar recursos existentes.
  • También puede crear recursos si no existen.
  • Idempotente.

Ejemplo

@RestController
@RequestMapping("/api")
public class ExampleController {

    @PutMapping("/items/{id}")
    public Item updateItem(@PathVariable Long id, @RequestBody Item item) {
        // Lógica para actualizar un ítem
        return itemService.updateItem(id, item);
    }
}

En este ejemplo, la anotación @PutMapping mapea las solicitudes HTTP PUT al método updateItem. La anotación @PathVariable se usa para extraer el id de la URL.

Método DELETE

El método DELETE se utiliza para eliminar un recurso especificado.

Conceptos Clave

  • Se usa para eliminar recursos.
  • Idempotente.

Ejemplo

@RestController
@RequestMapping("/api")
public class ExampleController {

    @DeleteMapping("/items/{id}")
    public void deleteItem(@PathVariable Long id) {
        // Lógica para eliminar un ítem
        itemService.deleteItem(id);
    }
}

Aquí, la anotación @DeleteMapping mapea las solicitudes HTTP DELETE al método deleteItem.

Resumen

En esta sección, hemos cubierto los cuatro métodos HTTP principales (GET, POST, PUT, DELETE) y cómo manejarlos usando Spring Boot. Cada método sirve para un propósito específico en los servicios web RESTful:

  • GET: Recuperar datos.
  • POST: Crear nuevos recursos.
  • PUT: Actualizar recursos existentes o crear nuevos.
  • DELETE: Eliminar recursos.

Entender estos métodos y sus casos de uso apropiados es esencial para construir servicios web robustos y eficientes.
