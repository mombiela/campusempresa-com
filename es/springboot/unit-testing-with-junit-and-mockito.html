Introducción a las Pruebas Unitarias

  • Definición: Las pruebas unitarias implican probar componentes individuales o funciones de una aplicación de software para asegurar que funcionen como se espera.
  • Importancia: Ayuda a identificar errores temprano, simplifica la integración y asegura la fiabilidad del código.

Configuración de JUnit y Mockito en Spring Boot

  • Dependencias: Añade las dependencias necesarias en tu archivo pom.xml o build.gradle.

Maven

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <scope>test</scope>
</dependency>

Gradle

testImplementation 'org.springframework.boot:spring-boot-starter-test'
testImplementation 'org.mockito:mockito-core'

Escribiendo Pruebas Unitarias con JUnit

  • Estructura Básica: Una clase de prueba típica de JUnit incluye métodos de configuración, métodos de prueba y métodos de limpieza.

Ejemplo

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    public void setUp() {
        calculator = new Calculator();
    }

    @Test
    public void testAddition() {
        int result = calculator.add(2, 3);
        assertEquals(5, result);
    }
}
  • Explicación:
    • @BeforeEach: Inicializa el entorno de prueba antes de cada prueba.
    • @Test: Marca un método como un método de prueba.
    • assertEquals: Asegura que el resultado esperado coincida con el resultado real.

Introducción a Mockito

  • Definición: Mockito es un marco de trabajo para la creación de objetos simulados para pruebas unitarias.
  • Propósito: Permite probar componentes en aislamiento simulando dependencias.

Uso de Mockito en Pruebas Unitarias

  • Anotaciones: Las anotaciones clave incluyen @Mock, @InjectMocks y @RunWith(MockitoJUnitRunner.class).

Ejemplo

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
import org.mockito.InjectMocks;
import org.mockito.Mock;
import org.mockito.MockitoAnnotations;

import static org.mockito.Mockito.when;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class UserServiceTest {

    @Mock
    private UserRepository userRepository;

    @InjectMocks
    private UserService userService;

    @BeforeEach
    public void setUp() {
        MockitoAnnotations.openMocks(this);
    }

    @Test
    public void testGetUserById() {
        User mockUser = new User(1, "John Doe");
        when(userRepository.findById(1)).thenReturn(Optional.of(mockUser));

        User user = userService.getUserById(1);
        assertEquals("John Doe", user.getName());
    }
}
  • Explicación:
    • @Mock: Crea una instancia simulada de la dependencia.
    • @InjectMocks: Inyecta las dependencias simuladas en la clase que se está probando.
    • MockitoAnnotations.openMocks(this): Inicializa los simulacros.
    • when(...).thenReturn(...): Define el comportamiento del simulacro.

Temas Avanzados

  • Capturadores de Argumentos: Capturar argumentos pasados a métodos simulados.
  • Espías: Simulación parcial de objetos reales.
  • Matchers Personalizados: Crear matchers de argumentos personalizados para escenarios más complejos.

Ejemplo de Capturador de Argumentos

import org.mockito.ArgumentCaptor;
import static org.mockito.Mockito.verify;

@Test
public void testArgumentCaptor() {
    ArgumentCaptor<String> captor = ArgumentCaptor.forClass(String.class);
    userService.updateUserName(1, "Jane Doe");

    verify(userRepository).updateName(captor.capture());
    assertEquals("Jane Doe", captor.getValue());
}

Mejores Prácticas

  • Aislamiento: Prueba una unidad a la vez.
  • Legibilidad: Escribe pruebas claras y comprensibles.
  • Mantenibilidad: Mantén las pruebas actualizadas con los cambios en el código.
  • Cobertura: Apunta a una alta cobertura de código pero prioriza pruebas significativas.

Conclusión

Las pruebas unitarias con JUnit y Mockito en Spring Boot son esenciales para asegurar la calidad y fiabilidad del código. Al entender los conceptos básicos y avanzar a escenarios más complejos, los desarrolladores pueden escribir pruebas efectivas que detecten errores temprano y faciliten una integración fluida. Recuerda seguir las mejores prácticas para mantener una suite de pruebas robusta.
