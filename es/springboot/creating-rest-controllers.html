En esta sección, exploraremos cómo crear controladores REST en Spring Boot. Comenzaremos con lo básico y gradualmente avanzaremos hacia conceptos más avanzados. Al final de esta sección, deberías ser capaz de crear y gestionar servicios web RESTful utilizando Spring Boot.

Introducción a los Controladores REST

REST (Representational State Transfer) es un estilo arquitectónico para diseñar aplicaciones en red. Las aplicaciones RESTful utilizan solicitudes HTTP para realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar).

Conceptos Clave

  • Métodos HTTP: GET, POST, PUT, DELETE, PATCH
  • Endpoints: URLs que representan recursos
  • Códigos de Estado: Indican el resultado de una solicitud HTTP (por ejemplo, 200 OK, 404 Not Found)
  • Cuerpo de la Solicitud: Datos enviados con solicitudes POST, PUT y PATCH
  • Cuerpo de la Respuesta: Datos devueltos desde el servidor

Configuración de Spring Boot

Antes de crear controladores REST, asegúrate de tener un proyecto Spring Boot configurado. Puedes crear un nuevo proyecto Spring Boot utilizando Spring Initializr o tu IDE preferido.

Dependencias

Asegúrate de tener las siguientes dependencias en tu archivo pom.xml o build.gradle:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

Creación de un Controlador REST Simple

Comencemos creando un controlador REST simple que maneje solicitudes HTTP GET básicas.

Ejemplo: Controlador Hola Mundo

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/api")
public class HelloWorldController {

    @GetMapping("/hello")
    public String sayHello() {
        return "¡Hola, Mundo!";
    }
}

Explicación

  • @RestController: Indica que esta clase es un controlador REST.
  • @RequestMapping("/api"): URL base para todos los endpoints en este controlador.
  • @GetMapping("/hello"): Mapea las solicitudes HTTP GET a /api/hello al método sayHello.

Manejo de Diferentes Métodos HTTP

Spring Boot facilita el manejo de diferentes métodos HTTP utilizando anotaciones.

Ejemplo: Operaciones CRUD para un Recurso

import org.springframework.web.bind.annotation.*;

import java.util.ArrayList;
import java.util.List;

@RestController
@RequestMapping("/api/items")
public class ItemController {

    private List<String> items = new ArrayList<>();

    @GetMapping
    public List<String> getAllItems() {
        return items;
    }

    @PostMapping
    public void addItem(@RequestBody String item) {
        items.add(item);
    }

    @PutMapping("/{index}")
    public void updateItem(@PathVariable int index, @RequestBody String item) {
        items.set(index, item);
    }

    @DeleteMapping("/{index}")
    public void deleteItem(@PathVariable int index) {
        items.remove(index);
    }
}

Explicación

  • @GetMapping: Recupera todos los elementos.
  • @PostMapping: Añade un nuevo elemento.
  • @PutMapping("/{index}"): Actualiza un elemento en el índice especificado.
  • @DeleteMapping("/{index}"): Elimina un elemento en el índice especificado.
  • @RequestBody: Vincula el cuerpo de la solicitud HTTP a un parámetro del método.
  • @PathVariable: Extrae valores de la URI.

Temas Avanzados

Manejo de Excepciones

Manejar excepciones de manera elegante es crucial para una API REST robusta.

Ejemplo: Manejador de Excepciones Personalizado

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(IndexOutOfBoundsException.class)
    public ResponseEntity<String> handleIndexOutOfBounds(IndexOutOfBoundsException ex) {
        return new ResponseEntity<>("Elemento no encontrado", HttpStatus.NOT_FOUND);
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<String> handleGeneralException(Exception ex) {
        return new ResponseEntity<>("Ocurrió un error", HttpStatus.INTERNAL_SERVER_ERROR);
    }
}

Explicación

  • @RestControllerAdvice: Permite manejar excepciones globalmente.
  • @ExceptionHandler: Especifica el tipo de excepción a manejar.

Validación de Solicitudes

La validación asegura que los datos enviados por el cliente cumplan con ciertos criterios.

Ejemplo: Validando un Cuerpo de Solicitud

import javax.validation.Valid;
import javax.validation.constraints.NotEmpty;

import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/validated-items")
public class ValidatedItemController {

    @PostMapping
    public void addItem(@Valid @RequestBody ValidatedItem item) {
        // Procesar el elemento
    }

    public static class ValidatedItem {
        @NotEmpty(message = "El nombre no puede estar vacío")
        private String name;

        // Getters y setters
    }
}

Explicación

  • @Valid: Activa la validación en el cuerpo de la solicitud.
  • @NotEmpty: Asegura que el campo no esté vacío.
