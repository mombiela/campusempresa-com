Introducción a la Inyección de Dependencias (DI)

La Inyección de Dependencias (DI) es un patrón de diseño utilizado para implementar IoC, permitiendo la creación de objetos dependientes fuera de una clase y proporcionando esos objetos a una clase de varias maneras.

  • Definición: La Inyección de Dependencias es una técnica donde un objeto recibe otros objetos de los que depende.
  • Propósito: Desacoplar la creación de una dependencia de la clase que la utiliza, haciendo el código más modular y testeable.

Tipos de Inyección de Dependencias

  1. Inyección por Constructor: Las dependencias se proporcionan a través del constructor de una clase.
  2. Inyección por Setter: Las dependencias se proporcionan a través de métodos setter.
  3. Inyección por Campo: Las dependencias se inyectan directamente en los campos.

Ejemplo: Inyección por Constructor

public class Service {
    private final Repository repository;

    // Inyección por Constructor
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Inversión de Control (IoC)

La Inversión de Control es un principio en la ingeniería de software donde el control de objetos o partes de un programa se transfiere a un contenedor o framework.

  • Definición: IoC es un principio de diseño en el cual el flujo de control de un programa se invierte en comparación con la programación tradicional.
  • Propósito: Aumentar la modularidad y hacer el código más fácil de gestionar y probar.

IoC en Spring Boot

Spring Boot utiliza el principio de IoC para gestionar el ciclo de vida y las dependencias de los beans.

  • Bean: Un objeto que es instanciado, ensamblado y gestionado por un contenedor IoC de Spring.
  • Contenedor IoC: El núcleo del Framework Spring, responsable de instanciar, configurar y ensamblar beans.

Ejemplo: Contenedor IoC en Spring Boot

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ApplicationContext;

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(Application.class, args);
        Service service = context.getBean(Service.class);
        service.performAction();
    }
}

Combinando DI e IoC en Spring Boot

Spring Boot aprovecha tanto DI como IoC para gestionar las dependencias y el ciclo de vida de los beans.

Ejemplo: Implementación Completa

  1. Definir una Interfaz Repository y su Implementación:

    public interface Repository {
    void action();
}

@Service
public class RepositoryImpl implements Repository {
    @Override
    public void action() {
        System.out.println("Action performed!");
    }
}

  2. Definir una Clase Service con Inyección por Constructor:

    @Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

  3. Ejecutar la Aplicación:

    @SpringBootApplication
public class Application {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(Application.class, args);
        Service service = context.getBean(Service.class);
        service.performAction();
    }
}

Temas Avanzados

Calificadores y Beans Primarios

Cuando existen múltiples beans del mismo tipo, puedes usar @Qualifier o @Primary para especificar qué bean debe ser inyectado.

  • @Qualifier: Se utiliza para especificar qué bean inyectar cuando existen múltiples beans del mismo tipo.
  • @Primary: Se utiliza para designar un bean como el candidato principal para la inyección.

Ejemplo: Usando @Qualifier

@Service
@Qualifier("repository1")
public class RepositoryImpl1 implements Repository {
    @Override
    public void action() {
        System.out.println("Repository1 action performed!");
    }
}

@Service
@Qualifier("repository2")
public class RepositoryImpl2 implements Repository {
    @Override
    public void action() {
        System.out.println("Repository2 action performed!");
    }
}

@Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(@Qualifier("repository1") Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Ejemplo: Usando @Primary

@Service
@Primary
public class PrimaryRepositoryImpl implements Repository {
    @Override
    public void action() {
        System.out.println("Primary repository action performed!");
    }
}

@Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Conclusión

La Inyección de Dependencias y la Inversión de Control son conceptos fundamentales en Spring Boot que ayudan a crear código desacoplado, modular y testeable. Al entender y aplicar estos principios, puedes aprovechar todo el poder del Framework Spring para construir aplicaciones robustas y mantenibles.
