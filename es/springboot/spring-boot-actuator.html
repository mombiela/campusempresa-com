Spring Boot Actuator es un conjunto de herramientas poderoso que proporciona características listas para producción para ayudarte a monitorear y gestionar tu aplicación Spring Boot. Incluye una variedad de endpoints integrados que pueden ser utilizados para recopilar métricas, entender el tráfico, o incluso cambiar configuraciones sin reiniciar la aplicación.

Conceptos Clave

  • Endpoints: Actuator proporciona varios endpoints que exponen diferentes aspectos de la aplicación.
  • Métricas: Recopila y expone métricas sobre la aplicación.
  • Verificaciones de Salud: Monitorea la salud de la aplicación y sus dependencias.
  • Endpoints Personalizados: Crea tus propios endpoints para exponer información personalizada.
  • Seguridad: Asegura los endpoints del actuator para prevenir el acceso no autorizado.

Comenzando con Spring Boot Actuator

Añadiendo Actuator a Tu Proyecto

Para empezar a usar Spring Boot Actuator, necesitas añadir la dependencia spring-boot-starter-actuator a tu archivo pom.xml:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

Habilitando Endpoints

Por defecto, solo unos pocos endpoints están habilitados. Puedes habilitar más endpoints configurando tu archivo application.properties:

management.endpoints.web.exposure.include=*

Esta configuración expondrá todos los endpoints disponibles.

Endpoints Comunes de Actuator

Aquí hay algunos de los endpoints de Actuator más comúnmente usados:

  • /actuator/health: Proporciona información de salud sobre la aplicación.
  • /actuator/info: Muestra información arbitraria de la aplicación.
  • /actuator/metrics: Muestra información de métricas.
  • /actuator/env: Expone propiedades del Environment de Spring.

Ejemplo: Accediendo al Endpoint de Salud

Una vez que hayas añadido la dependencia de Actuator y habilitado los endpoints, puedes acceder al endpoint de salud navegando a http://localhost:8080/actuator/health en tu navegador o usando una herramienta como curl:

curl http://localhost:8080/actuator/health

La respuesta se verá algo así:

{
    "status": "UP"
}

Personalizando Endpoints de Actuator

Indicadores de Salud Personalizados

Puedes crear indicadores de salud personalizados para incluir verificaciones de salud adicionales. Aquí hay un ejemplo de un indicador de salud personalizado:

import org.springframework.boot.actuate.health.Health;
import org.springframework.boot.actuate.health.HealthIndicator;
import org.springframework.stereotype.Component;

@Component
public class CustomHealthIndicator implements HealthIndicator {

    @Override
    public Health health() {
        // Lógica personalizada de verificación de salud
        boolean isHealthy = checkCustomHealth();
        if (isHealthy) {
            return Health.up().withDetail("CustomService", "Available").build();
        } else {
            return Health.down().withDetail("CustomService", "Not Available").build();
        }
    }

    private boolean checkCustomHealth() {
        // Implementa aquí tu lógica personalizada de verificación de salud
        return true;
    }
}

Métricas Personalizadas

También puedes crear métricas personalizadas para monitorear aspectos específicos de tu aplicación. Aquí hay un ejemplo:

import io.micrometer.core.instrument.MeterRegistry;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

import javax.annotation.PostConstruct;

@Component
public class CustomMetrics {

    @Autowired
    private MeterRegistry meterRegistry;

    @PostConstruct
    public void init() {
        meterRegistry.counter("custom.metric.counter").increment();
    }
}

Asegurando Endpoints de Actuator

Para asegurar tus endpoints de Actuator, puedes configurar Spring Security. Aquí hay un ejemplo de configuración:

import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;

@Configuration
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
            .antMatchers("/actuator/**").hasRole("ADMIN")
            .anyRequest().authenticated()
            .and()
            .httpBasic();
    }
}

En esta configuración, solo los usuarios con el rol ADMIN pueden acceder a los endpoints de Actuator.

Conclusión

Spring Boot Actuator es una herramienta esencial para monitorear y gestionar tus aplicaciones Spring Boot. Proporciona una amplia gama de endpoints integrados para ayudarte a obtener información sobre la salud, métricas y entorno de tu aplicación. Al personalizar y asegurar estos endpoints, puedes adaptar el Actuator a tus necesidades específicas y asegurar que tu aplicación permanezca robusta y segura en entornos de producción.
