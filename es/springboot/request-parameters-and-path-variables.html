En esta sección, exploraremos cómo manejar parámetros de solicitud y variables de ruta en Spring Boot. Estos son conceptos esenciales para construir servicios web RESTful y manejar solicitudes de clientes de manera efectiva.

Parámetros de Solicitud

Los parámetros de solicitud se utilizan para enviar datos del cliente al servidor como parte de la cadena de consulta de la URL. Se utilizan típicamente para filtrar, ordenar o especificar opciones adicionales para una solicitud.

Conceptos Clave

  • Cadena de Consulta: La parte de la URL después del símbolo ?, que contiene pares clave-valor.
  • Anotación @RequestParam: Se utiliza para vincular parámetros de solicitud a parámetros de método en Spring Boot.

Ejemplo

Vamos a crear una aplicación simple de Spring Boot para demostrar cómo usar parámetros de solicitud.

  1. Crear un Proyecto de Spring Boot: Utiliza Spring Initializr para generar un nuevo proyecto de Spring Boot con las dependencias necesarias (Spring Web).

  2. Clase Controladora: Crea una clase controladora para manejar solicitudes HTTP.

package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class GreetingController {

    @GetMapping("/greet")
    public String greet(@RequestParam(value = "name", defaultValue = "World") String name) {
        return "Hello, " + name + "!";
    }
}

Explicación

  • @RestController: Indica que esta clase es un controlador REST.
  • @GetMapping("/greet"): Mapea solicitudes HTTP GET al endpoint /greet.
  • @RequestParam: Vincula el parámetro de solicitud name al parámetro de método name. Si no se proporciona el parámetro name, se establece por defecto en "World".

Probando el Endpoint

  • Accede al endpoint a través de http://localhost:8080/greet?name=John.
  • La respuesta será Hello, John!.
  • Si se omite el parámetro name, la respuesta será Hello, World!.

Variables de Ruta

Las variables de ruta se utilizan para capturar valores de la ruta de la URL. Se utilizan típicamente para identificar recursos específicos.

Conceptos Clave

  • Ruta de la URL: La parte de la URL que especifica la ubicación del recurso.
  • Anotación @PathVariable: Se utiliza para vincular variables de ruta a parámetros de método en Spring Boot.

Ejemplo

Vamos a extender nuestra aplicación de Spring Boot para demostrar cómo usar variables de ruta.

  1. Clase Controladora: Actualiza la clase controladora para manejar variables de ruta.
package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class GreetingController {

    @GetMapping("/greet")
    public String greet(@RequestParam(value = "name", defaultValue = "World") String name) {
        return "Hello, " + name + "!";
    }

    @GetMapping("/greet/{name}")
    public String greetWithPathVariable(@PathVariable String name) {
        return "Hello, " + name + "!";
    }
}

Explicación

  • @GetMapping("/greet/{name}"): Mapea solicitudes HTTP GET al endpoint /greet/{name}.
  • @PathVariable: Vincula la variable de ruta name al parámetro de método name.

Probando el Endpoint

  • Accede al endpoint a través de http://localhost:8080/greet/John.
  • La respuesta será Hello, John!.

Conclusión

Entender cómo manejar parámetros de solicitud y variables de ruta es crucial para construir servicios web RESTful robustos y flexibles en Spring Boot. Los parámetros de solicitud son ideales para datos opcionales y filtrado, mientras que las variables de ruta son perfectas para identificar recursos específicos. Al dominar estos conceptos, puedes crear APIs más dinámicas y amigables para el usuario.
