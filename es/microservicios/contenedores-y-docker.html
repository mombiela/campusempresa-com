Introducción a los Contenedores

Los contenedores son una tecnología que permite empaquetar y ejecutar aplicaciones de manera aislada del sistema operativo subyacente. A diferencia de las máquinas virtuales, los contenedores comparten el mismo núcleo del sistema operativo, lo que los hace más ligeros y eficientes.

Conceptos Clave

  • Aislamiento: Los contenedores proporcionan un entorno aislado para ejecutar aplicaciones, lo que mejora la seguridad y la estabilidad.
  • Portabilidad: Los contenedores pueden ejecutarse en cualquier sistema que tenga un motor de contenedores compatible, como Docker.
  • Eficiencia: Los contenedores son más ligeros que las máquinas virtuales porque comparten el núcleo del sistema operativo.

Introducción a Docker

Docker es una plataforma de contenedorización que permite a los desarrolladores empaquetar aplicaciones y sus dependencias en contenedores. Docker facilita la creación, despliegue y ejecución de aplicaciones en contenedores.

Conceptos Clave

  • Docker Engine: El motor de Docker es el componente principal que permite la creación y ejecución de contenedores.
  • Docker Images: Las imágenes de Docker son plantillas de solo lectura que contienen el sistema operativo y las aplicaciones necesarias para ejecutar un contenedor.
  • Docker Containers: Los contenedores de Docker son instancias en ejecución de imágenes de Docker.
  • Docker Hub: Un registro público donde los usuarios pueden almacenar y compartir imágenes de Docker.

Instalación de Docker

Para instalar Docker en diferentes sistemas operativos, siga los pasos a continuación:

Instalación en Linux

sudo apt-get update
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io

Instalación en Windows

  1. Descargue Docker Desktop desde Docker Hub.
  2. Ejecute el instalador y siga las instrucciones en pantalla.
  3. Reinicie su computadora si es necesario.

Instalación en macOS

  1. Descargue Docker Desktop desde Docker Hub.
  2. Abra el archivo .dmg y arrastre Docker a la carpeta de Aplicaciones.
  3. Ejecute Docker desde la carpeta de Aplicaciones.

Uso Básico de Docker

Comandos Esenciales

  • docker run: Ejecuta un contenedor a partir de una imagen.
docker run hello-world
  • docker ps: Lista los contenedores en ejecución.
docker ps
  • docker images: Lista las imágenes disponibles en el sistema.
docker images
  • docker stop: Detiene un contenedor en ejecución.
docker stop <container_id>

Crear una Imagen de Docker

Para crear una imagen de Docker, se utiliza un archivo Dockerfile. A continuación se muestra un ejemplo de un Dockerfile simple:

# Use una imagen base de Python
FROM python:3.8-slim

# Establezca el directorio de trabajo
WORKDIR /app

# Copie los archivos necesarios
COPY . /app

# Instale las dependencias
RUN pip install --no-cache-dir -r requirements.txt

# Comando para ejecutar la aplicación
CMD ["python", "app.py"]

Para construir la imagen, use el siguiente comando:

docker build -t my-python-app .

Ejemplo Práctico: Desplegar una Aplicación Web

Paso 1: Crear el `Dockerfile`

FROM node:14

WORKDIR /usr/src/app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 8080

CMD ["node", "server.js"]

Paso 2: Construir la Imagen

docker build -t my-node-app .

Paso 3: Ejecutar el Contenedor

docker run -p 8080:8080 -d my-node-app

Conclusión

Docker es una herramienta poderosa para la contenedorización de aplicaciones, proporcionando portabilidad, eficiencia y aislamiento. Con Docker, los desarrolladores pueden empaquetar aplicaciones y sus dependencias en contenedores que se pueden ejecutar en cualquier entorno compatible. En este tema, hemos cubierto los conceptos básicos de los contenedores y Docker, incluyendo la instalación, comandos esenciales y un ejemplo práctico de despliegue de una aplicación web.
