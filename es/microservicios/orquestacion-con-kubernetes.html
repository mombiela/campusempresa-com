Introducción a Kubernetes

Kubernetes es una plataforma de orquestación de contenedores que automatiza el despliegue, la gestión y la escalabilidad de aplicaciones en contenedores. Es una herramienta esencial en el desarrollo y operación de microservicios debido a su capacidad para manejar aplicaciones complejas y distribuidas.

Conceptos Clave

  • Nodo: Una máquina física o virtual en el clúster de Kubernetes.
  • Pod: La unidad más pequeña y básica de despliegue en Kubernetes, que puede contener uno o más contenedores.
  • Cluster: Un conjunto de nodos que ejecutan aplicaciones en contenedores gestionados por Kubernetes.
  • Namespace: Un mecanismo para dividir recursos de clúster entre múltiples usuarios.
  • Deployment: Un recurso que define cómo desplegar y gestionar un conjunto de Pods.
  • Service: Un recurso que define una política para acceder a un conjunto de Pods.

Arquitectura de Kubernetes

Kubernetes sigue una arquitectura de clúster maestro-esclavo. El clúster maestro gestiona el estado del clúster y los nodos esclavos ejecutan las aplicaciones.

Componentes del Maestro

  • API Server: Punto de entrada para todas las operaciones del clúster.
  • etcd: Almacén de datos distribuido que guarda la configuración del clúster.
  • Controller Manager: Gestiona los controladores que regulan el estado del clúster.
  • Scheduler: Asigna Pods a los nodos disponibles.

Componentes del Nodo

  • Kubelet: Agente que asegura que los contenedores se ejecuten en un Pod.
  • Kube-proxy: Gestiona la red de servicios en cada nodo.
  • Container Runtime: Software que ejecuta los contenedores (por ejemplo, Docker).

Despliegue de Aplicaciones con Kubernetes

Creación de un Pod

Un Pod es la unidad básica de despliegue en Kubernetes. Aquí hay un ejemplo de un archivo YAML para crear un Pod:

apiVersion: v1
kind: Pod
metadata:
  name: my-pod
spec:
  containers:
  - name: my-container
    image: nginx
    ports:
    - containerPort: 80

Para desplegar este Pod, se usa el comando:

kubectl apply -f my-pod.yaml

Creación de un Deployment

Un Deployment gestiona la creación y actualización de Pods. Aquí hay un ejemplo de un archivo YAML para un Deployment:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-container
        image: nginx
        ports:
        - containerPort: 80

Para desplegar este Deployment, se usa el comando:

kubectl apply -f my-deployment.yaml

Creación de un Service

Un Service expone un conjunto de Pods como un servicio de red. Aquí hay un ejemplo de un archivo YAML para un Service:

apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80
  type: LoadBalancer

Para desplegar este Service, se usa el comando:

kubectl apply -f my-service.yaml

Escalabilidad y Gestión de Estados

Escalado Horizontal

Kubernetes permite escalar aplicaciones horizontalmente (añadiendo más réplicas de Pods) de manera sencilla:

kubectl scale deployment my-deployment --replicas=5

Actualizaciones y Rollbacks

Kubernetes facilita las actualizaciones y rollbacks de aplicaciones:

  • Actualización:
kubectl set image deployment/my-deployment my-container=nginx:1.19.0
  • Rollback:
kubectl rollout undo deployment/my-deployment

Monitoreo y Logging

Monitoreo

Kubernetes proporciona métricas a través de herramientas como Prometheus y Grafana para monitorear el rendimiento de las aplicaciones.

Logging

Los logs de los contenedores pueden ser accedidos usando:

kubectl logs my-pod

Conclusión

Kubernetes es una herramienta poderosa para la orquestación de contenedores, esencial para el despliegue y gestión de microservicios. Con su capacidad para automatizar el escalado, las actualizaciones y la recuperación ante fallos, Kubernetes facilita la gestión de aplicaciones complejas y distribuidas. A través de este módulo, hemos cubierto los conceptos básicos, la arquitectura, y las operaciones esenciales para trabajar con Kubernetes.
