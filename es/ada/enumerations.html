Enumeraciones en Ada
Tipos de Acceso (Punteros) ►

Las enumeraciones en Ada son un tipo de dato fundamental que te permite definir un conjunto de valores nombrados. Estos valores se llaman literales de enumeración y son útiles para representar una colección de constantes relacionadas de una manera legible y mantenible.

Conceptos Clave

  • Tipo de Enumeración: Un tipo definido por el usuario que consiste en un conjunto de valores nombrados.
  • Literales de Enumeración: Los valores nombrados que pertenecen a un tipo de enumeración.
  • Seguridad de Tipo: Las enumeraciones en Ada proporcionan seguridad de tipo al asegurar que solo valores válidos sean asignados a variables del tipo de enumeración.

Definiendo Enumeraciones

Para definir un tipo de enumeración en Ada, usas la palabra clave type seguida del nombre del tipo y la lista de literales de enumeración encerrados entre paréntesis.

Ejemplo

type Day is (Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday);

En este ejemplo, Day es un tipo de enumeración con siete literales: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, y Sunday.

Usando Enumeraciones

Una vez que has definido un tipo de enumeración, puedes declarar variables de ese tipo y asignarles literales de enumeración.

Ejemplo

procedure Enumerations_Example is
   type Day is (Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday);
   Today : Day;
begin
   Today := Monday;
   if Today = Monday then
      Put_Line("¡Es lunes!");
   end if;
end Enumerations_Example;

En este ejemplo, la variable Today es del tipo Day, y se le asigna el valor Monday. El programa luego verifica si Today es Monday y muestra un mensaje en consecuencia.

Atributos de Enumeración

Ada proporciona varios atributos para tipos de enumeración que pueden ser útiles en diversas situaciones.

  • 'Pos: Devuelve el número de posición de un literal de enumeración.
  • 'Val: Convierte un número de posición en el literal de enumeración correspondiente.
  • 'Succ: Devuelve el sucesor de un literal de enumeración.
  • 'Pred: Devuelve el predecesor de un literal de enumeración.

Ejemplo

procedure Enumeration_Attributes is
   type Day is (Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday);
   Today : Day := Monday;
begin
   Put_Line("Posición de Monday: " & Integer'Image(Day'Pos(Today)));
   Put_Line("Literal en la posición 2: " & Day'Image(Day'Val(2)));
   Put_Line("Sucesor de Monday: " & Day'Image(Day'Succ(Today)));
   Put_Line("Predecesor de Wednesday: " & Day'Image(Day'Pred(Wednesday)));
end Enumeration_Attributes;

En este ejemplo, se usan varios atributos para obtener información sobre los literales de enumeración.

Uso Avanzado

Cláusulas de Representación de Enumeración

Puedes especificar la representación interna de los literales de enumeración usando cláusulas de representación. Esto puede ser útil para interactuar con hardware u otros lenguajes de programación.

Ejemplo

type Color is (Red, Green, Blue);
for Color use (Red => 1, Green => 2, Blue => 3);

En este ejemplo, los literales de enumeración Red, Green, y Blue se asignan los valores internos 1, 2, y 3, respectivamente.

Enumeración y Sentencias Case

Las enumeraciones se usan a menudo con sentencias case para manejar diferentes valores de una manera estructurada.

Ejemplo

procedure Case_Statement_Example is
   type Traffic_Light is (Red, Yellow, Green);
   Signal : Traffic_Light := Red;
begin
   case Signal is
      when Red =>
         Put_Line("¡Detente!");
      when Yellow =>
         Put_Line("¡Precaución!");
      when Green =>
         Put_Line("¡Adelante!");
   end case;
end Case_Statement_Example;

En este ejemplo, se usa una sentencia case para mostrar diferentes mensajes basados en el valor de la variable Signal.

Conclusión

Las enumeraciones en Ada proporcionan una manera poderosa de definir y trabajar con un conjunto de constantes relacionadas. Mejoran la legibilidad y mantenibilidad del código mientras aseguran la seguridad de tipo. Al entender cómo definir, usar y manipular enumeraciones, puedes escribir programas en Ada más robustos y claros.
