Los objetos protegidos en Ada proporcionan un mecanismo para la programación concurrente segura y eficiente. Se utilizan para proteger los datos compartidos del acceso concurrente, asegurando que solo una tarea pueda acceder a los datos protegidos a la vez. Esto es crucial en sistemas en tiempo real y embebidos donde la integridad de los datos y la sincronización son primordiales.

Conceptos Clave

  • Objeto Protegido: Un constructo que encapsula datos y operaciones, asegurando la exclusión mutua y la sincronización.
  • Tipo Protegido: La definición de un objeto protegido, especificando sus datos y operaciones.
  • Procedimientos Protegidos: Operaciones que modifican los datos protegidos.
  • Funciones Protegidas: Operaciones que leen los datos protegidos.
  • Entradas Protegidas: Operaciones que incluyen mecanismos de sincronización, como esperar a que una condición sea verdadera.

Definiendo un Tipo Protegido

Un tipo protegido se define usando la palabra clave protected. Incluye la declaración de datos y operaciones (procedimientos, funciones y entradas).

protected type Shared_Counter is
   procedure Increment;
   function Value return Integer;
private
   Counter : Integer := 0;
end Shared_Counter;

Explicación

  • Declaración del Tipo Protegido: El tipo Shared_Counter incluye un procedimiento Increment y una función Value.
  • Sección Privada: La variable Counter es privada, lo que significa que solo puede ser accedida por los procedimientos y funciones dentro del tipo protegido.

Implementando un Tipo Protegido

La implementación del tipo protegido especifica el comportamiento de sus operaciones.

protected body Shared_Counter is
   procedure Increment is
   begin
      Counter := Counter + 1;
   end Increment;

   function Value return Integer is
   begin
      return Counter;
   end Value;
end Shared_Counter;

Explicación

  • Procedimiento Increment: Incrementa el Counter en 1.
  • Función Value: Devuelve el valor actual de Counter.

Usando Objetos Protegidos

Para usar un objeto protegido, se declara una instancia del tipo protegido y se llaman sus operaciones.

procedure Main is
   Counter : Shared_Counter;
begin
   Counter.Increment;
   Ada.Text_IO.Put_Line("Counter Value: " & Integer'Image(Counter.Value));
end Main;

Explicación

  • Declaración del Counter: Se declara una instancia de Shared_Counter.
  • Llamadas a Operaciones: Se llama al procedimiento Increment para incrementar el contador, y se llama a la función Value para recuperar e imprimir el valor del contador.

Entradas Protegidas

Las entradas protegidas se utilizan para operaciones que requieren sincronización, como esperar a que una condición sea verdadera.

protected type Synchronized_Buffer is
   entry Write(Item : in Integer);
   entry Read(Item : out Integer);
private
   Buffer : Integer;
   Full : Boolean := False;
end Synchronized_Buffer;

protected body Synchronized_Buffer is
   entry Write(Item : in Integer) when not Full is
   begin
      Buffer := Item;
      Full := True;
   end Write;

   entry Read(Item : out Integer) when Full is
   begin
      Item := Buffer;
      Full := False;
   end Read;
end Synchronized_Buffer;

Explicación

  • Entrada Write: Espera hasta que el buffer no esté lleno antes de escribir un ítem.
  • Entrada Read: Espera hasta que el buffer esté lleno antes de leer un ítem.
  • Condiciones de Sincronización: La cláusula when especifica la condición que debe ser verdadera para que la entrada proceda.

Conclusión

Los objetos protegidos en Ada son una herramienta poderosa para gestionar el acceso concurrente a datos compartidos. Proporcionan una forma estructurada de asegurar la integridad de los datos y la sincronización, lo cual es esencial en sistemas en tiempo real y embebidos. Al encapsular datos y operaciones dentro de un tipo protegido, Ada asegura que solo una tarea pueda acceder a los datos protegidos a la vez, previniendo condiciones de carrera y asegurando una programación concurrente segura.
