Resumen

La Programación Orientada a Objetos (POO) es un paradigma de programación que utiliza "objetos" para diseñar aplicaciones y programas informáticos. Ada, un lenguaje de programación tipado estáticamente, estructurado e imperativo, soporta principios de POO como encapsulación, herencia y polimorfismo.

Conceptos Clave de la POO

  • Encapsulación: Agrupación de datos y métodos que operan sobre los datos dentro de una unidad, por ejemplo, una clase.
  • Herencia: Mecanismo por el cual una clase puede heredar atributos y métodos de otra clase.
  • Polimorfismo: Capacidad de procesar objetos de manera diferente según su tipo de datos o clase.

Encapsulación en Ada

La encapsulación en Ada se logra utilizando paquetes y tipos privados. Aquí hay un ejemplo simple:

-- Definir un paquete para una Pila
package Stack_Pkg is
    type Stack is private;
    procedure Push(S : in out Stack; Element : Integer);
    procedure Pop(S : in out Stack; Element : out Integer);
    function Is_Empty(S : Stack) return Boolean;
private
    type Stack is record
        Elements : array(1..100) of Integer;
        Top : Integer := 0;
    end record;
end Stack_Pkg;

package body Stack_Pkg is
    procedure Push(S : in out Stack; Element : Integer) is
    begin
        S.Top := S.Top + 1;
        S.Elements(S.Top) := Element;
    end Push;

    procedure Pop(S : in out Stack; Element : out Integer) is
    begin
        Element := S.Elements(S.Top);
        S.Top := S.Top - 1;
    end Pop;

    function Is_Empty(S : Stack) return Boolean is
    begin
        return S.Top = 0;
    end Is_Empty;
end Stack_Pkg;

Explicación:

  • Especificación del Paquete: Define la interfaz de la pila.
  • Tipo Privado: La implementación real de la pila está oculta para el usuario.
  • Cuerpo del Paquete: Implementa los métodos definidos en la especificación del paquete.

Herencia en Ada

La herencia permite definir un nuevo tipo basado en un tipo existente. Aquí hay un ejemplo:

package Shape_Pkg is
    type Shape is tagged record
        X, Y : Integer;
    end record;

    procedure Move(S : in out Shape; New_X, New_Y : Integer);

    type Circle is new Shape with record
        Radius : Integer;
    end record;

    procedure Resize(C : in out Circle; New_Radius : Integer);
end Shape_Pkg;

package body Shape_Pkg is
    procedure Move(S : in out Shape; New_X, New_Y : Integer) is
    begin
        S.X := New_X;
        S.Y := New_Y;
    end Move;

    procedure Resize(C : in out Circle; New_Radius : Integer) is
    begin
        C.Radius := New_Radius;
    end Resize;
end Shape_Pkg;

Explicación:

  • Registro Etiquetado: Shape es un registro etiquetado, lo que significa que puede ser extendido.
  • Tipo Derivado: Circle se deriva de Shape y añade un nuevo atributo Radius.
  • Sobrescritura de Métodos: Métodos como Move y Resize pueden ser definidos tanto para Shape como para Circle.

Polimorfismo en Ada

El polimorfismo en Ada se logra a través de la distribución dinámica. Aquí hay un ejemplo:

with Ada.Text_IO; use Ada.Text_IO;

package Shape_Pkg is
    type Shape is tagged record
        X, Y : Integer;
    end record;

    procedure Move(S : in out Shape; New_X, New_Y : Integer) is abstract;
    procedure Display(S : in Shape) is abstract;

    type Circle is new Shape with record
        Radius : Integer;
    end record;

    procedure Move(S : in out Circle; New_X, New_Y : Integer);
    procedure Display(S : in Circle);
end Shape_Pkg;

package body Shape_Pkg is
    procedure Move(S : in out Circle; New_X, New_Y : Integer) is
    begin
        S.X := New_X;
        S.Y := New_Y;
    end Move;

    procedure Display(S : in Circle) is
    begin
        Put_Line("Circle at (" & Integer'Image(S.X) & ", " & Integer'Image(S.Y) & ") with radius " & Integer'Image(S.Radius));
    end Display;
end Shape_Pkg;

with Shape_Pkg; use Shape_Pkg;

procedure Main is
    C : Circle := (X => 0, Y => 0, Radius => 10);
begin
    C.Move(5, 5);
    C.Display;
end Main;

Explicación:

  • Métodos Abstractos: Move y Display son métodos abstractos en Shape.
  • Distribución Dinámica: Cuando Move y Display se llaman en un Circle, se utilizan las implementaciones específicas para Circle.

Conclusión

En este tema, introdujimos los conceptos fundamentales de la Programación Orientada a Objetos en Ada, incluyendo encapsulación, herencia y polimorfismo. Proporcionamos ejemplos para ilustrar cómo se implementan estos conceptos en Ada. Entender estos principios es crucial para diseñar sistemas de software robustos y mantenibles. A medida que avances, verás cómo se aplican estos conceptos en escenarios más complejos.
