El polimorfismo es un concepto fundamental en la programación orientada a objetos que permite tratar objetos de diferentes tipos como objetos de un tipo supercomún. En Ada, el polimorfismo se logra mediante el uso de tipos etiquetados y despacho dinámico. Este tema te guiará desde los conceptos básicos hasta el uso avanzado del polimorfismo en Ada.

Introducción al Polimorfismo

  • Definición: El polimorfismo permite que los métodos operen sobre objetos de diferentes tipos a través de una interfaz común.
  • Conceptos Clave:
    • Tipos Etiquetados: Tipos especiales en Ada que soportan herencia y polimorfismo.
    • Despacho Dinámico: El mecanismo por el cual se llama al método correcto para un objeto, basado en su tipo real en tiempo de ejecución.

Tipos Etiquetados

Los tipos etiquetados son la base del polimorfismo en Ada. Permiten la creación de una jerarquía de tipos donde los tipos derivados heredan propiedades y métodos de sus tipos padres.

Definiendo un Tipo Etiquetado

package Shapes is
   type Shape is tagged record
      X, Y : Float;
   end record;

   procedure Draw(S : in Shape) is abstract;
end Shapes;
  • Explicación:
    • Shape es un tipo etiquetado con dos campos, X y Y.
    • El procedimiento Draw se declara como abstracto, lo que significa que debe ser sobrescrito en los tipos derivados.

Derivando de un Tipo Etiquetado

package Shapes.Circles is
   type Circle is new Shape with record
      Radius : Float;
   end record;

   procedure Draw(C : in Circle);
end Shapes.Circles;
  • Explicación:
    • Circle es un nuevo tipo derivado de Shape, añadiendo un campo Radius.
    • El procedimiento Draw se sobrescribe para proporcionar un comportamiento específico para Circle.

Despacho Dinámico

El despacho dinámico permite que se llame al método correcto basado en el tipo real del objeto en tiempo de ejecución.

Ejemplo de Despacho Dinámico

with Shapes; use Shapes;
with Shapes.Circles; use Shapes.Circles;

procedure Main is
   S : Shape'Class := Circle'(X => 0.0, Y => 0.0, Radius => 5.0);
begin
   Draw(S); -- Llama al procedimiento Draw de Circle
end Main;
  • Explicación:
    • S se declara como Shape'Class, que es un tipo de clase que puede contener cualquier tipo derivado de Shape.
    • El procedimiento Draw llamado en S se despachará dinámicamente al procedimiento Draw para Circle.

Polimorfismo Avanzado

Operaciones de Clase

Las operaciones de clase permiten escribir procedimientos que pueden operar sobre cualquier tipo dentro de una jerarquía de clases.

procedure Move(S : in out Shape'Class; DX, DY : Float) is
begin
   S.X := S.X + DX;
   S.Y := S.Y + DY;
end Move;
  • Explicación:
    • El procedimiento Move puede operar sobre cualquier objeto de tipo Shape o cualquiera de sus derivados.

Tipos Abstractos e Interfaces

Los tipos abstractos e interfaces proporcionan una forma de definir comportamientos comunes sin proporcionar una implementación completa.

package Shapes is
   type Shape is abstract tagged record
      X, Y : Float;
   end record;

   procedure Draw(S : in Shape) is abstract;
end Shapes;
  • Explicación:
    • Shape se declara como un tipo abstracto, lo que significa que no puede ser instanciado directamente.
    • El procedimiento Draw también es abstracto, requiriendo que los tipos derivados proporcionen una implementación.

Conclusión

El polimorfismo en Ada es una característica poderosa que permite un código flexible y reutilizable. Mediante el uso de tipos etiquetados y despacho dinámico, puedes crear una jerarquía de tipos que comparten comportamientos comunes mientras permiten implementaciones específicas en tipos derivados. Entender y utilizar el polimorfismo te permitirá escribir programas en Ada más modulares y mantenibles.
