Introducción a los Sistemas en Tiempo Real

Los sistemas en tiempo real son sistemas que requieren una respuesta oportuna a eventos externos. Estos sistemas se utilizan a menudo en aplicaciones críticas donde los retrasos pueden llevar a fallos del sistema o una degradación significativa del rendimiento.

Conceptos Clave

  • Determinismo: La capacidad de un sistema para producir la misma salida dado el mismo input dentro de un marco de tiempo predecible.
  • Latencia: El retraso de tiempo entre la ocurrencia de un evento y la respuesta del sistema a este.
  • Jitter: La variabilidad en el tiempo de respuesta.
  • Planificación: El método por el cual las tareas se asignan para ejecutarse en el procesador.

Ada y los Sistemas en Tiempo Real

Ada es un lenguaje de programación diseñado con soporte incorporado para sistemas en tiempo real. Proporciona características que ayudan a escribir aplicaciones en tiempo real confiables y eficientes.

Características Clave de Ada para Sistemas en Tiempo Real

  • Tareas: El modelo de concurrencia de Ada permite la creación de tareas que pueden ejecutarse en paralelo.
  • Objetos Protegidos: Mecanismos para asegurar el acceso seguro a recursos compartidos.
  • Reloj en Tiempo Real: Facilidades para manejar el tiempo y los retrasos.
  • Planificación por Prioridad: Soporte para asignar prioridades a las tareas.

Tareas en Ada

Las tareas en Ada son similares a los hilos en otros lenguajes de programación. Permiten la ejecución concurrente de código.

Ejemplo: Creación y Ejecución de Tareas

with Ada.Text_IO; use Ada.Text_IO;

procedure Main is
   task type Worker is
      entry Start;
   end Worker;

   task body Worker is
   begin
      accept Start;
      Put_Line("La tarea Worker está en ejecución.");
   end Worker;

   Worker_Task : Worker;
begin
   Worker_Task.Start;
   Put_Line("La tarea principal está en ejecución.");
end Main;
  • Explicación: Este ejemplo demuestra la creación de una tarea (Worker) y su interacción con el programa principal. La tarea Worker tiene un punto de entrada Start que es llamado por el programa principal.

Objetos Protegidos

Los objetos protegidos en Ada proporcionan una forma de compartir datos de manera segura entre tareas.

Ejemplo: Uso de Objetos Protegidos

with Ada.Text_IO; use Ada.Text_IO;

protected Counter is
   procedure Increment;
   function Value return Integer;
private
   Count : Integer := 0;
end Counter;

protected body Counter is
   procedure Increment is
   begin
      Count := Count + 1;
   end Increment;

   function Value return Integer is
   begin
      return Count;
   end Value;
end Counter;

procedure Main is
begin
   Counter.Increment;
   Put_Line("Valor del contador: " & Integer'Image(Counter.Value));
end Main;
  • Explicación: Este ejemplo muestra un objeto protegido Counter con un procedimiento Increment y una función Value. El objeto protegido asegura que la variable Count sea accedida y modificada de manera segura.

Reloj en Tiempo Real y Retrasos

Ada proporciona facilidades para manejar relojes en tiempo real y retrasos, los cuales son esenciales para los sistemas en tiempo real.

Ejemplo: Uso de Retrasos

with Ada.Text_IO; use Ada.Text_IO;
with Ada.Real_Time; use Ada.Real_Time;

procedure Main is
   Start_Time : Time := Clock;
   Delay_Duration : constant Duration := 0.5; -- 500 milisegundos
begin
   Put_Line("Iniciando retraso...");
   delay Delay_Duration;
   Put_Line("Retraso completado.");
   Put_Line("Tiempo transcurrido: " & Duration'Image(To_Duration(Clock - Start_Time)));
end Main;
  • Explicación: Este ejemplo demuestra cómo usar el reloj en tiempo real y los retrasos en Ada. El programa se retrasa por 500 milisegundos y luego imprime el tiempo transcurrido.

Planificación por Prioridad

Ada soporta la planificación por prioridad, permitiendo que las tareas sean asignadas con diferentes prioridades.

Ejemplo: Establecimiento de Prioridades de Tareas

with Ada.Text_IO; use Ada.Text_IO;
with Ada.Real_Time; use Ada.Real_Time;
with Ada.Dispatching; use Ada.Dispatching;

procedure Main is
   task type Worker(Priority : Integer) is
      pragma Priority(Priority);
      entry Start;
   end Worker;

   task body Worker is
   begin
      accept Start;
      Put_Line("La tarea Worker con prioridad " & Integer'Image(Priority) & " está en ejecución.");
   end Worker;

   High_Priority_Task : Worker(10);
   Low_Priority_Task : Worker(5);
begin
   High_Priority_Task.Start;
   Low_Priority_Task.Start;
end Main;
  • Explicación: Este ejemplo muestra cómo crear tareas con diferentes prioridades usando la directiva pragma Priority. Las tareas High_Priority_Task y Low_Priority_Task se asignan con prioridades 10 y 5, respectivamente.

Conclusión

Los sistemas en tiempo real requieren un control preciso sobre el tiempo y la ejecución de tareas. Ada proporciona características robustas como tareas, objetos protegidos, manejo de relojes en tiempo real y planificación por prioridad para apoyar el desarrollo de aplicaciones en tiempo real confiables. Al aprovechar estas características, los desarrolladores pueden crear sistemas que cumplan con los estrictos requisitos de rendimiento y fiabilidad en tiempo real.
