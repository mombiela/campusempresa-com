Introducción

En este estudio de caso, exploraremos cómo diseñar e implementar un sistema de control en tiempo real utilizando Ada. Los sistemas en tiempo real son cruciales en varios dominios como la aeroespacial, automotriz y automatización industrial, donde las respuestas oportunas y predecibles son esenciales. Ada, con su tipado fuerte, soporte de concurrencia y capacidades en tiempo real, es una excelente opción para tales sistemas.

Conceptos Clave

  • Sistemas en Tiempo Real: Sistemas que deben responder a entradas o eventos dentro de una estricta restricción de tiempo.
  • Concurrencia: La capacidad de ejecutar múltiples secuencias de operaciones simultáneamente.
  • Tareas en Ada: Soporte incorporado de Ada para la programación concurrente a través de tareas.
  • Planificación en Tiempo Real: Técnicas para asegurar que las tareas cumplan con sus plazos.

Requisitos del Sistema

Antes de profundizar en la implementación, delineemos los requisitos para nuestro sistema de control en tiempo real:

  1. Adquisición de Datos del Sensor: Leer periódicamente datos de los sensores.
  2. Procesamiento de Datos: Procesar los datos adquiridos para tomar decisiones de control.
  3. Control del Actuador: Enviar comandos a los actuadores basados en los datos procesados.
  4. Restricciones de Tiempo: Asegurar que todas las tareas cumplan con sus plazos.

Diseño del Sistema

Estructura de Tareas

Diseñaremos nuestro sistema utilizando tareas de Ada para manejar diferentes componentes:

  • Tarea del Sensor: Lee datos de los sensores a intervalos regulares.
  • Tarea de Procesamiento: Procesa los datos del sensor.
  • Tarea del Actuador: Controla los actuadores basados en los datos procesados.

Comunicación entre Tareas

Las tareas se comunicarán usando objetos protegidos para asegurar la consistencia de los datos y evitar condiciones de carrera.

Implementación

Tarea del Sensor

La tarea del sensor leerá datos de un sensor cada 100 milisegundos.

task Sensor_Task is
    period : constant Duration := 0.1; -- 100 milisegundos
end Sensor_Task;

task body Sensor_Task is
    sensor_data : Integer;
begin
    loop
        -- Simular la lectura de datos del sensor
        sensor_data := Read_Sensor();
        -- Almacenar datos en un recurso compartido
        Sensor_Data_Protected.Store(sensor_data);
        delay period;
    end loop;
end Sensor_Task;

Objetos Protegidos

Se usarán objetos protegidos para almacenar y recuperar datos de manera segura entre tareas.

protected Sensor_Data_Protected is
    procedure Store(data : in Integer);
    function Retrieve return Integer;
private
    sensor_data : Integer := 0;
end Sensor_Data_Protected;

protected body Sensor_Data_Protected is
    procedure Store(data : in Integer) is
    begin
        sensor_data := data;
    end Store;

    function Retrieve return Integer is
    begin
        return sensor_data;
    end Retrieve;
end Sensor_Data_Protected;

protected Processed_Data_Protected is
    procedure Store(data : in Integer);
    function Retrieve return Integer;
private
    processed_data : Integer := 0;
end Processed_Data_Protected;

protected body Processed_Data_Protected is
    procedure Store(data : in Integer) is
    begin
        processed_data := data;
    end Store;

    function Retrieve return Integer is
    begin
        return processed_data;
    end Retrieve;
end Processed_Data_Protected;
