Introducción

En esta sección del curso sobre Infraestructuras TI, exploraremos la configuración y gestión de almacenamiento. El almacenamiento es un componente crítico en cualquier infraestructura de TI, ya que afecta directamente el rendimiento, la disponibilidad y la seguridad de los datos. Aprenderemos sobre los diferentes tipos de almacenamiento, cómo configurarlos y gestionarlos de manera eficiente.

Tipos de Almacenamiento

Almacenamiento Directo (DAS)

  • Definición: El almacenamiento directo (Direct Attached Storage, DAS) se conecta directamente a un servidor o estación de trabajo sin pasar por una red.
  • Ventajas:
    • Baja latencia.
    • Fácil de configurar.
  • Desventajas:
    • No es escalable.
    • Difícil de compartir entre múltiples servidores.

Almacenamiento en Red (NAS)

  • Definición: El almacenamiento en red (Network Attached Storage, NAS) es un dispositivo de almacenamiento que se conecta a una red y permite el acceso a múltiples usuarios y dispositivos.
  • Ventajas:
    • Fácil de compartir.
    • Escalable.
  • Desventajas:
    • Puede tener latencia debido a la red.
    • Requiere configuración de red.

Almacenamiento en Red de Área de Almacenamiento (SAN)

  • Definición: Una red de área de almacenamiento (Storage Area Network, SAN) es una red dedicada de dispositivos de almacenamiento que proporciona acceso a almacenamiento consolidado.
  • Ventajas:
    • Alta escalabilidad.
    • Alta disponibilidad y rendimiento.
  • Desventajas:
    • Costoso.
    • Complejo de configurar y gestionar.

| Tipo de Almacenamiento | Ventajas | Desventajas | |------------------------|-----------------------------------|------------------------------------| | DAS | Baja latencia, fácil de configurar| No escalable, difícil de compartir | | NAS | Fácil de compartir, escalable | Latencia de red, requiere configuración de red | | SAN | Alta escalabilidad, alto rendimiento | Costoso, complejo de gestionar |

Configuración de Almacenamiento

Configuración de DAS

  1. Conexión Física: Conectar el dispositivo de almacenamiento directamente al servidor mediante cables (SATA, SAS).
  2. Configuración del BIOS/UEFI: Asegurarse de que el dispositivo sea reconocido en el BIOS/UEFI.
  3. Particionado y Formateo: 
    # Listar discos disponibles
sudo fdisk -l

# Crear una nueva partición
sudo fdisk /dev/sdX

# Formatear la partición
sudo mkfs.ext4 /dev/sdX1

Configuración de NAS

  1. Conexión a la Red: Conectar el NAS a la red mediante un cable Ethernet.
  2. Configuración de Red: Asignar una dirección IP estática o dinámica.
  3. Configuración de Comparticiones:
    • Crear carpetas compartidas.
    • Asignar permisos de acceso.
    • Ejemplo de configuración en un NAS Synology: 
      # Crear una carpeta compartida
synoshare --create MyShare --path /volume1/MyShare

# Asignar permisos
synoshare --set MyShare --user admin --read-write

Configuración de SAN

  1. Configuración de la Red SAN: Configurar switches y cables de fibra óptica.
  2. Configuración de LUNs (Logical Unit Numbers):
    • Crear LUNs en el dispositivo de almacenamiento.
    • Asignar LUNs a servidores específicos.
  3. Configuración del Servidor:
    • Instalar y configurar el software de cliente SAN.
    • Mapear LUNs a dispositivos de bloque en el servidor.
    # Ejemplo de mapeo de LUN en Linux
sudo iscsiadm --mode discovery --type sendtargets --portal 192.168.1.100
sudo iscsiadm --mode node --targetname iqn.2023-10.com.example:storage --login

Gestión de Almacenamiento

Monitoreo

  • Herramientas de Monitoreo:
    • Nagios
    • Zabbix
    • Prometheus
  • Métricas Clave:
    • Utilización del espacio.
    • Rendimiento (IOPS, latencia).
    • Estado de los discos.

Backup y Recuperación

  • Estrategias de Backup:
    • Completo.
    • Incremental.
    • Diferencial.
  • Herramientas de Backup:
    • Bacula
    • Veeam
    • Acronis
  • Ejemplo de Script de Backup en Linux: 
    #!/bin/bash
tar -czvf /backup/full_backup_$(date +%F).tar.gz /data

Seguridad

  • Cifrado de Datos:
    • Cifrado en reposo.
    • Cifrado en tránsito.
  • Control de Acceso:
    • Implementar políticas de acceso basadas en roles (RBAC).
    • Utilizar autenticación multifactor (MFA).

Conclusión

La configuración y gestión de almacenamiento es esencial para garantizar la disponibilidad, rendimiento y seguridad de los datos en una infraestructura de TI. Conocer los diferentes tipos de almacenamiento y cómo configurarlos adecuadamente permite a los profesionales de TI diseñar soluciones eficientes y escalables. Además, la gestión continua mediante monitoreo, backup y medidas de seguridad asegura la integridad y disponibilidad de los datos a lo largo del tiempo.
