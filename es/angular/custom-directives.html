Introducción a las Directivas

Las directivas en Angular son clases que añaden comportamiento adicional a los elementos en tus aplicaciones Angular. Con las directivas, puedes gestionar formularios, listas, estilos y lo que los usuarios ven. Hay tres tipos de directivas en Angular:

  • Directivas de Componentes: Estas son las directivas más comunes y se utilizan para crear componentes.
  • Directivas de Atributo: Estas cambian la apariencia o el comportamiento de un elemento, componente u otra directiva.
  • Directivas Estructurales: Estas cambian el diseño del DOM añadiendo y eliminando elementos del DOM.

En este tema, nos enfocaremos en crear directivas personalizadas de atributo y estructurales.

Creación de Directivas de Atributo Personalizadas

Las directivas de atributo se utilizan para cambiar la apariencia o el comportamiento de los elementos del DOM. Vamos a crear una simple directiva de atributo que cambia el color de fondo de un elemento cuando se pasa el ratón por encima.

Guía Paso a Paso

  1. Generar la Directiva: Usa Angular CLI para generar una nueva directiva.

    ng generate directive highlight

  2. Implementar la Directiva: Modifica la directiva generada para cambiar el color de fondo al pasar el ratón por encima.

    import { Directive, ElementRef, HostListener, Input } from '@angular/core';

@Directive({
  selector: '[appHighlight]'
})
export class HighlightDirective {
  @Input() defaultColor: string = 'yellow';
  @Input('appHighlight') highlightColor: string;

  constructor(private el: ElementRef) {}

  @HostListener('mouseenter') onMouseEnter() {
    this.highlight(this.highlightColor || this.defaultColor);
  }

  @HostListener('mouseleave') onMouseLeave() {
    this.highlight(null);
  }

  private highlight(color: string) {
    this.el.nativeElement.style.backgroundColor = color;
  }
}

  3. Usar la Directiva en un Componente: Aplica la directiva a un elemento en tu plantilla.

    <p appHighlight="lightblue">Pasa el ratón sobre este texto para ver un efecto de resaltado.</p>

Explicación

  • ElementRef: Un servicio que otorga acceso directo al elemento del DOM.
  • @HostListener: Decorador que escucha eventos del DOM y desencadena el método especificado.
  • @Input: Decorador que permite que la directiva acepte valores de entrada.

Creación de Directivas Estructurales Personalizadas

Las directivas estructurales cambian el diseño del DOM añadiendo o eliminando elementos. Vamos a crear una directiva estructural personalizada que incluya condicionalmente una plantilla basada en un valor booleano.

Guía Paso a Paso

  1. Generar la Directiva: Usa Angular CLI para generar una nueva directiva.

    ng generate directive unless

  2. Implementar la Directiva: Modifica la directiva generada para incluir condicionalmente una plantilla.

    import { Directive, Input, TemplateRef, ViewContainerRef } from '@angular/core;

@Directive({
  selector: '[appUnless]'
})
export class UnlessDirective {
  private hasView = false;

  constructor(
    private templateRef: TemplateRef<any>,
    private viewContainer: ViewContainerRef
  ) {}

  @Input() set appUnless(condition: boolean) {
    if (!condition && !this.hasView) {
      this.viewContainer.createEmbeddedView(this.templateRef);
      this.hasView = true;
    } else if (condition && this.hasView) {
      this.viewContainer.clear();
      this.hasView = false;
    }
  }
}

  3. Usar la Directiva en un Componente: Aplica la directiva a un elemento en tu plantilla.

    <div *appUnless="isVisible">Este texto se muestra solo cuando isVisible es falso.</div>

Explicación

  • TemplateRef: Representa una plantilla incrustada que puede ser utilizada para instanciar vistas incrustadas.
  • ViewContainerRef: Representa un contenedor donde se pueden adjuntar una o más vistas.
  • @Input: Decorador que permite que la directiva acepte valores de entrada.

Resumen

En este tema, cubrimos cómo crear directivas de atributo y estructurales personalizadas en Angular. Las directivas de atributo se utilizan para cambiar la apariencia o el comportamiento de los elementos, mientras que las directivas estructurales se utilizan para cambiar el diseño del DOM añadiendo o eliminando elementos. Al entender y utilizar estas directivas, puedes crear aplicaciones Angular más dinámicas e interactivas.

Puntos Clave

  • Las directivas son clases que añaden comportamiento a los elementos en las aplicaciones Angular.
  • Las directivas de atributo cambian la apariencia o el comportamiento de los elementos.
  • Las directivas estructurales cambian el diseño del DOM añadiendo o eliminando elementos.
  • Usa Angular CLI para generar directivas y modificarlas según tus necesidades.
  • Decoradores como @Input y @HostListener son esenciales para crear directivas personalizadas.

Al dominar las directivas personalizadas, puedes mejorar significativamente la funcionalidad y la experiencia del usuario en tus aplicaciones Angular.
