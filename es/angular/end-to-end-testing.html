Las pruebas de extremo a extremo (E2E) son una parte crucial del ciclo de vida del desarrollo de software, asegurando que todo el flujo de la aplicación funcione como se espera. En Angular, las pruebas E2E se realizan típicamente usando Protractor, un marco de pruebas específicamente diseñado para aplicaciones Angular.

Introducción a las pruebas de extremo a extremo

Las pruebas de extremo a extremo simulan escenarios reales de usuario para validar la funcionalidad de la aplicación de principio a fin. Prueban toda la pila de la aplicación, incluyendo el frontend, backend y la base de datos.

Conceptos clave

  • Escenarios de usuario: Simulación de interacciones de usuario del mundo real.
  • Automatización: Automatización del proceso de prueba para asegurar consistencia y eficiencia.
  • Protractor: Un marco de pruebas para aplicaciones Angular.
  • WebDriver: Una herramienta para automatizar pruebas de aplicaciones web.

Configuración de Protractor

Para comenzar con las pruebas E2E en Angular, necesitas configurar Protractor.

Instalación

Primero, instala Protractor globalmente usando npm:

npm install -g protractor

Luego, actualiza el administrador de WebDriver:

webdriver-manager update

Configuración

Crea un archivo protractor.conf.js en el directorio raíz de tu proyecto:

exports.config = {
  framework: 'jasmine',
  seleniumAddress: 'http://localhost:4444/wd/hub',
  specs: ['e2e/**/*.e2e-spec.js'],
  capabilities: {
    browserName: 'chrome'
  },
  onPrepare: function() {
    browser.ignoreSynchronization = true;
  }
};

Escribiendo tu primera prueba E2E

Vamos a escribir una prueba E2E simple para verificar si la página de inicio se carga correctamente.

Ejemplo de prueba

Crea un nuevo archivo e2e/homepage.e2e-spec.js:

describe('Homepage', function() {
  it('should have a title', function() {
    browser.get('http://localhost:4200');
    expect(browser.getTitle()).toEqual('My Angular App');
  });
});

Explicación

  • describe: Define un conjunto de pruebas.
  • it: Define un caso de prueba.
  • browser.get: Navega a la URL especificada.
  • expect: Asegura que la aplicación se comporte como se espera.

Técnicas avanzadas de pruebas E2E

A medida que te sientas más cómodo con las pruebas E2E, puedes comenzar a usar técnicas más avanzadas.

Objetos de página

Los objetos de página ayudan a organizar tus pruebas separando el código de prueba del código específico de la página.

Ejemplo

Crea un nuevo archivo e2e/pages/homepage.po.js:

class HomePage {
  constructor() {
    this.title = element(by.css('app-root h1'));
  }

  getTitle() {
    return this.title.getText();
  }
}

module.exports = new HomePage();

Actualiza tu prueba para usar el objeto de página:

const HomePage = require('./pages/homepage.po');

describe('Homepage', function() {
  it('should have a title', function() {
    browser.get('http://localhost:4200');
    expect(HomePage.getTitle()).toEqual('Welcome to My Angular App!');
  });
});

Manejo de operaciones asincrónicas

Las aplicaciones Angular a menudo involucran operaciones asincrónicas. Protractor proporciona formas de manejarlas.

Ejemplo

it('should wait for an element to be visible', function() {
  const EC = protractor.ExpectedConditions;
  const button = element(by.css('.my-button'));
  browser.wait(EC.visibilityOf(button), 5000);
  button.click();
});

Ejecutando pruebas E2E

Para ejecutar tus pruebas E2E, inicia el administrador de WebDriver y ejecuta Protractor:

webdriver-manager start
protractor protractor.conf.js

Conclusión

Las pruebas de extremo a extremo son una práctica esencial para asegurar la fiabilidad y funcionalidad de tus aplicaciones Angular. Usando Protractor, puedes automatizar escenarios de usuario, manejar operaciones asincrónicas y organizar tus pruebas usando objetos de página. A medida que avances, puedes integrar las pruebas E2E en tu pipeline de CI/CD para mantener altos estándares de calidad a lo largo del ciclo de vida del desarrollo.
