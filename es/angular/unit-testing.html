Introducción a las Pruebas Unitarias

Las pruebas unitarias son un método de prueba de software donde unidades o componentes individuales de un software se prueban en aislamiento del resto de la aplicación. En Angular, las pruebas unitarias se escriben típicamente usando Jasmine y se ejecutan con Karma.

  • Jasmine: Un marco de desarrollo dirigido por comportamiento para probar código JavaScript.
  • Karma: Un ejecutor de pruebas que te permite ejecutar tus pruebas en varios navegadores.

Configuración de Pruebas Unitarias en Angular

Los proyectos Angular vienen preconfigurados con Jasmine y Karma. Cuando creas un nuevo proyecto Angular usando el Angular CLI, los archivos de configuración necesarios y las dependencias se incluyen automáticamente.

ng new my-angular-app
cd my-angular-app
ng test

Escribiendo Tu Primera Prueba Unitaria

Comencemos escribiendo una prueba unitaria simple para un servicio de Angular.

Creando un Servicio

Primero, crea un nuevo servicio usando el Angular CLI:

ng generate service my-service

Esto generará un archivo de servicio (my-service.service.ts) y un archivo spec correspondiente (my-service.service.spec.ts).

Escribiendo una Prueba para el Servicio

Abre el archivo my-service.service.spec.ts y escribe una prueba simple:

import { TestBed } from '@angular/core/testing';
import { MyService } from './my-service.service';

describe('MyService', () => {
  let service: MyService;

  beforeEach(() => {
    TestBed.configureTestingModule({});
    service = TestBed.inject(MyService);
  });

  it('should be created', () => {
    expect(service).toBeTruthy();
  });

  it('should return correct value from getValue', () => {
    expect(service.getValue()).toBe('some value');
  });
});

Explicación

  • TestBed: La API principal de Angular para configurar e inicializar el entorno para pruebas unitarias.
  • describe: Una función de Jasmine para agrupar pruebas relacionadas.
  • it: Una función de Jasmine para definir casos de prueba individuales.
  • expect: Una función de Jasmine para hacer afirmaciones.

Pruebas de Componentes

Las pruebas unitarias de componentes implican probar la clase del componente y la plantilla.

Creando un Componente

Genera un nuevo componente usando el Angular CLI:

ng generate component my-component

Escribiendo una Prueba para el Componente

Abre el archivo my-component.component.spec.ts y escribe una prueba:

import { ComponentFixture, TestBed } from '@angular/core/testing';
import { MyComponent } from './my-component.component';

describe('MyComponent', () => {
  let component: MyComponent;
  let fixture: ComponentFixture<MyComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ MyComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(MyComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should render title', () => {
    const compiled = fixture.nativeElement;
    expect(compiled.querySelector('h1').textContent).toContain('My Component Title');
  });
});

Explicación

  • ComponentFixture: Un contenedor alrededor de un componente y su plantilla.
  • compileComponents: Compila el componente y su plantilla.
  • fixture.detectChanges: Activa la detección de cambios y actualiza la vista.

Técnicas Avanzadas de Pruebas

Mocking de Dependencias

En aplicaciones del mundo real, los componentes y servicios a menudo dependen de otros servicios. Simular estas dependencias es crucial para aislar la unidad bajo prueba.

import { TestBed } from '@angular/core/testing';
import { MyService } from './my-service.service';
import { MyComponent } from './my-component.component';

class MockMyService {
  getValue() {
    return 'mock value';
  }
}

describe('MyComponent', () => {
  let component: MyComponent;
  let service: MyService;

  beforeEach(() => {
    TestBed.configureTestingModule({
      declarations: [ MyComponent ],
      providers: [
        { provide: MyService, useClass: MockMyService }
      ]
    });

    service = TestBed.inject(MyService);
    component = TestBed.createComponent(MyComponent).componentInstance;
  });

  it('should use the mock service', () => {
    expect(service.getValue()).toBe('mock value');
  });
});

Pruebas de Código Asíncrono

Las pruebas de código asíncrono se pueden hacer usando las utilidades async y fakeAsync proporcionadas por Angular.

import { fakeAsync, tick } from '@angular/core/testing';

it('should handle async code', fakeAsync(() => {
  let flag = false;
  setTimeout(() => {
    flag = true;
  }, 1000);

  tick(1000);
  expect(flag).toBe(true);
}));

Conclusión

Las pruebas unitarias son una parte esencial del desarrollo de aplicaciones robustas en Angular. Al escribir pruebas para servicios y componentes, y usar técnicas avanzadas como la simulación y las pruebas de código asíncrono, puedes asegurarte de que tu aplicación se comporte como se espera. Comienza con pruebas simples y gradualmente avanza a escenarios más complejos a medida que te sientas cómodo con las herramientas y metodologías de prueba.
