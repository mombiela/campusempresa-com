Introducción a las Rutas Hijas

En Angular, las rutas hijas se utilizan para crear rutas anidadas dentro de una ruta principal. Esto permite escenarios de enrutamiento más complejos y ayuda a organizar la aplicación en partes más pequeñas y manejables. Las rutas hijas son particularmente útiles cuando se crean aplicaciones de múltiples vistas donde se muestran diferentes vistas dentro de un componente principal.

Conceptos Clave

  • Ruta Principal: La ruta principal que contiene una o más rutas hijas.
  • Ruta Hija: Una ruta que está anidada dentro de una ruta principal.
  • Módulo de Enrutador: Módulo de Angular que proporciona los servicios y directivas necesarios para el enrutamiento.

Configuración de Rutas Hijas

Para configurar rutas hijas, necesitas definirlas dentro de la propiedad children de una ruta principal en tu módulo de enrutamiento.

Ejemplo

Vamos a crear una aplicación Angular simple con una ruta principal y dos rutas hijas.

  1. Crear Componentes: Primero, crea los componentes necesarios para las rutas principal e hijas.

    ng generate component parent
ng generate component child1
ng generate component child2

  2. Definir Rutas: A continuación, define las rutas en tu archivo app-routing.module.ts.

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { ParentComponent } from './parent/parent.component';
import { Child1Component } from './child1/child1.component';
import { Child2Component } from './child2/child2.component';

const routes: Routes = [
  {
    path: 'parent',
    component: ParentComponent,
    children: [
      { path: 'child1', component: Child1Component },
      { path: 'child2', component: Child2Component }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

  3. Actualizar la Plantilla del Componente Principal: En la plantilla del componente principal, agrega un <router-outlet> para mostrar los componentes hijos.

    <!-- parent.component.html -->
<h2>Componente Principal</h2>
<nav>
  <a routerLink="child1">Hijo 1</a>
  <a routerLink="child2">Hijo 2</a>
</nav>
<router-outlet></router-outlet>

  4. Ejecutar la Aplicación: Inicia tu aplicación Angular y navega a /parent. Deberías ver el componente principal con enlaces a los componentes hijos. Al hacer clic en los enlaces se mostrarán los respectivos componentes hijos dentro del componente principal.

Carga Diferida de Rutas Hijas

La carga diferida es una técnica para cargar rutas hijas solo cuando se necesitan, lo que puede mejorar el rendimiento de tu aplicación.

Ejemplo

  1. Crear un Módulo para Rutas Hijas:

    ng generate module child --route child --module app.module

  2. Definir Rutas Hijas en el Módulo:

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { Child1Component } from './child1/child1.component';
import { Child2Component } from './child2/child2.component';

const routes: Routes = [
  {
    path: '',
    component: Child1Component,
    children: [
      { path: 'child2', component: Child2Component }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forChild(routes)],
  exports: [RouterModule]
})
export class ChildModule { }

  3. Actualizar el Módulo de Enrutamiento Principal:

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: 'parent',
    loadChildren: () => import('./child/child.module').then(m => m.ChildModule)
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Conclusión

Las rutas hijas en Angular permiten un enfoque más organizado y modular para el enrutamiento dentro de una aplicación. Al anidar rutas, puedes crear estructuras de enrutamiento complejas que son más fáciles de gestionar y mantener. Además, aprovechar la carga diferida para las rutas hijas puede mejorar significativamente el rendimiento de tu aplicación al cargar solo las partes necesarias cuando se necesitan. Entender e implementar rutas hijas es una habilidad crucial para construir aplicaciones Angular escalables y eficientes.
