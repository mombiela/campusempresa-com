Introducción a la Arquitectura de Angular

Angular es una plataforma y un framework para construir aplicaciones cliente de una sola página utilizando HTML y TypeScript. Angular está escrito en TypeScript y proporciona un conjunto de herramientas y bibliotecas para construir aplicaciones robustas. Entender la arquitectura de Angular es crucial para desarrollar aplicaciones eficientes y mantenibles.

Conceptos Clave de la Arquitectura de Angular

  • Módulos
  • Componentes
  • Plantillas
  • Servicios
  • Inyección de Dependencias
  • Enrutamiento

Módulos

Los módulos son los bloques de construcción fundamentales de una aplicación Angular. Ayudan a organizar una aplicación en bloques cohesivos de funcionalidad.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }
  • NgModule: Decorador que marca una clase como un módulo de Angular.
  • declarations: Array de componentes, directivas y pipes que pertenecen a este módulo.
  • imports: Array de módulos para importar en este módulo.
  • providers: Array de servicios que el módulo contribuye a la colección global de servicios.
  • bootstrap: Array de componentes para inicializar cuando este módulo se inicializa.

Componentes

Los componentes son los principales bloques de construcción de una aplicación Angular. Cada componente consta de una plantilla HTML y una clase TypeScript.

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'my-angular-app';
}
  • @Component: Decorador que marca una clase como un componente de Angular.
  • selector: Selector CSS que identifica este componente en una plantilla.
  • templateUrl: URL a un archivo externo que contiene la plantilla del componente.
  • styleUrls: Array de URLs a hojas de estilo para este componente.

Plantillas

Las plantillas definen la vista para un componente. Utilizan la sintaxis de plantillas de Angular para enlazar datos y manejar eventos de usuario.

<h1>{{ title }}</h1>
<button (click)="onClick()">Click me</button>
  • Interpolación: {{ title }} enlaza la propiedad title del componente al elemento <h1>.
  • Enlace de Eventos: (click)="onClick()" enlaza el evento de clic del botón al método onClick en la clase del componente.

Servicios

Los servicios se utilizan para encapsular la lógica de negocio y el acceso a datos. Normalmente se inyectan en componentes y otros servicios utilizando el sistema de inyección de dependencias de Angular.

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class DataService {
  getData() {
    return ['Data1', 'Data2', 'Data3'];
  }
}
  • @Injectable: Decorador que marca una clase como disponible para ser inyectada como una dependencia.
  • providedIn: Especifica que el servicio debe ser proporcionado en el inyector raíz.

Inyección de Dependencias

La Inyección de Dependencias (DI) es un patrón de diseño utilizado para implementar IoC (Inversión de Control). Permite que una clase reciba dependencias de una fuente externa en lugar de crearlas por sí misma.

import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  templateUrl: './data.component.html',
})
export class DataComponent implements OnInit {
  data: string[];

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.data = this.dataService.getData();
  }
}
  • constructor(private dataService: DataService): El DataService se inyecta en el DataComponent.

Enrutamiento

El enrutamiento permite la navegación entre diferentes vistas o componentes en una aplicación Angular.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent },
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }
  • RouterModule.forRoot(routes): Configura el enrutador en el nivel raíz de la aplicación.
  • routes: Array de definiciones de rutas.

Conclusión

Entender la arquitectura de Angular es esencial para construir aplicaciones escalables y mantenibles. Al dominar módulos, componentes, plantillas, servicios, inyección de dependencias y enrutamiento, puedes crear aplicaciones robustas en Angular. Cada uno de estos conceptos juega un papel crítico en la estructura y funcionalidad general de una aplicación Angular.
