Introducción

Los formularios basados en plantillas son una forma simple y poderosa de manejar la entrada del usuario en aplicaciones Angular. Se basan en la vinculación bidireccional de datos de Angular y directivas para crear y gestionar formularios. Este enfoque es adecuado para formularios más simples y es más fácil de configurar en comparación con los formularios reactivos.

Conceptos Clave

  • Directiva NgModel: Vincula los controles del formulario a las propiedades del modelo.
  • FormsModule: Módulo de Angular necesario para usar formularios basados en plantillas.
  • Validación de Formularios: Validadores integrados y validadores personalizados.
  • Envío de Formularios: Manejo de datos del formulario al enviarlo.

Configuración de Formularios Basados en Plantillas

Importando FormsModule

Para usar formularios basados en plantillas, necesitas importar FormsModule en tu módulo de Angular.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { FormsModule } from '@angular/forms';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, FormsModule],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Creando un Formulario Básico

Aquí tienes un ejemplo simple de un formulario basado en plantillas:

<!-- app.component.html -->
<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Nombre:</label>
    <input type="text" id="name" name="name" ngModel required>
  </div>
  <div>
    <label for="email">Correo Electrónico:</label>
    <input type="email" id="email" name="email" ngModel required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>

Manejo del Envío del Formulario

En tu componente, puedes manejar el envío del formulario y acceder a los datos del formulario:

// app.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  onSubmit(form: any) {
    console.log('Datos del Formulario:', form.value);
  }
}

Validación de Formularios

Validadores Integrados

Angular proporciona varios validadores integrados como required, minlength, maxlength, email, etc.

<div>
  <label for="password">Contraseña:</label>
  <input type="password" id="password" name="password" ngModel required minlength="8">
  <div *ngIf="userForm.controls.password?.invalid && userForm.controls.password?.touched">
    <small *ngIf="userForm.controls.password?.errors?.required">La contraseña es obligatoria.</small>
    <small *ngIf="userForm.controls.password?.errors?.minlength">La contraseña debe tener al menos 8 caracteres.</small>
  </div>
</div>

Validadores Personalizados

También puedes crear validadores personalizados para lógica de validación más compleja.

// app.component.ts
import { Component } from '@angular/core';
import { NgForm } from '@angular/forms';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  onSubmit(form: NgForm) {
    console.log('Datos del Formulario:', form.value);
  }

  validateUsername(control: any): { [key: string]: boolean } | null {
    const forbiddenUsernames = ['admin', 'user'];
    if (forbiddenUsernames.includes(control.value)) {
      return { 'forbiddenUsername': true };
    }
    return null;
  }
}
<div>
  <label for="username">Nombre de Usuario:</label>
  <input type="text" id="username" name="username" ngModel #username="ngModel" [ngModelOptions]="{ updateOn: 'blur' }" [ngModel]="validateUsername">
  <div *ngIf="username.invalid && username.touched">
    <small *ngIf="username.errors?.forbiddenUsername">Este nombre de usuario no está permitido.</small>
  </div>
</div>

Temas Avanzados

Vinculación Bidireccional de Datos

Los formularios basados en plantillas aprovechan la vinculación bidireccional de datos de Angular para sincronizar los controles del formulario con las propiedades del modelo.

<!-- app.component.html -->
<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Nombre:</label>
    <input type="text" id="name" name="name" [(ngModel)]="user.name" required>
  </div>
  <div>
    <label for="email">Correo Electrónico:</label>
    <input type="email" id="email" name="email" [(ngModel)]="user.email" required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Enviar</button>
</form>
// app.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  user = {
    name: '',
    email: ''
  };

  onSubmit(form: any) {
    console.log('Datos del Formulario:', form.value);
  }
}

Estado del Formulario y Clases CSS

Angular agrega automáticamente clases CSS a los controles del formulario según su estado (por ejemplo, ng-touched, ng-dirty, ng-valid, ng-invalid).

<div>
  <label for="name">Nombre:</label>
  <input type="text" id="name" name="name" ngModel required>
  <div *ngIf="userForm.controls.name?.invalid && userForm.controls.name?.touched">
    <small *ngIf="userForm.controls.name?.errors?.required">El nombre es obligatorio.</small>
  </div>
</div>

Conclusión

Los formularios basados en plantillas en Angular proporcionan una forma sencilla de crear y gestionar formularios utilizando las poderosas directivas y la vinculación bidireccional de datos de Angular. Son ideales para formularios más simples y ofrecen validación integrada, validación personalizada y fácil gestión del estado del formulario. Al comprender y utilizar estos conceptos, puedes manejar eficientemente la entrada del usuario y la validación de formularios en tus aplicaciones Angular.
