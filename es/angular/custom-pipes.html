Introducción a los Pipes

Los pipes en Angular son una forma de transformar datos en plantillas. Son funciones simples que aceptan un valor de entrada y devuelven un valor transformado. Angular proporciona varios pipes integrados, como DatePipe, UpperCasePipe, LowerCasePipe y CurrencyPipe.

¿Por Qué Usar Pipes Personalizados?

Aunque los pipes integrados cubren muchos casos de uso comunes, a veces necesitas transformar datos de una manera específica para tu aplicación. Los pipes personalizados te permiten crear lógica de transformación de datos reutilizable que se puede aplicar directamente en tus plantillas.

Creando un Pipe Personalizado

Crear un pipe personalizado en Angular implica los siguientes pasos:

  1. Generar un nuevo pipe usando Angular CLI.
  2. Implementar la interfaz PipeTransform.
  3. Definir la lógica de transformación.
  4. Usar el pipe personalizado en tus plantillas.

Paso 1: Generar un Nuevo Pipe

Usa Angular CLI para generar un nuevo pipe:

ng generate pipe custom

Este comando creará dos archivos: custom.pipe.ts y custom.pipe.spec.ts.

Paso 2: Implementar la Interfaz `PipeTransform`

Abre el archivo custom.pipe.ts e implementa la interfaz PipeTransform:

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'custom'
})
export class CustomPipe implements PipeTransform {
  transform(value: any, ...args: any[]): any {
    // La lógica de transformación va aquí
    return value;
  }
}

Paso 3: Definir la Lógica de Transformación

Añade tu lógica de transformación personalizada dentro del método transform. Por ejemplo, vamos a crear un pipe que convierta una cadena a mayúsculas y minúsculas:

export class CustomPipe implements PipeTransform {
  transform(value: string): string {
    if (!value) return value;
    return value.replace(/\w\S*/g, (txt) => {
      return txt.charAt(0).toUpperCase() + txt.substr(1).toLowerCase();
    });
  }
}

Paso 4: Usar el Pipe Personalizado en Plantillas

Una vez que tu pipe personalizado está definido, puedes usarlo en tus plantillas como cualquier otro pipe:

<p>{{ 'hello world' | custom }}</p>

Esto mostrará:

Hello World

Pasando Parámetros a Pipes Personalizados

Los pipes personalizados también pueden aceptar parámetros. Por ejemplo, vamos a crear un pipe que formatee un número a un decimal especificado:

@Pipe({
  name: 'decimal'
})
export class DecimalPipe implements PipeTransform {
  transform(value: number, decimalPlaces: number): string {
    return value.toFixed(decimalPlaces);
  }
}

Uso en plantilla:

<p>{{ 123.4567 | decimal:2 }}</p>

Esto mostrará:

123.46

Pipes Puros vs Impuros

  • Pipes Puros: Estos pipes solo se llaman cuando Angular detecta un cambio en el valor de entrada. Son más eficientes y deben usarse en la mayoría de los casos.
  • Pipes Impuros: Estos pipes se llaman en cada ciclo de detección de cambios, lo que puede afectar el rendimiento. Úsalos solo cuando sea necesario.

Para crear un pipe impuro, establece la propiedad pure a false en el decorador @Pipe:

@Pipe({
  name: 'impurePipe',
  pure: false
})
export class ImpurePipe implements PipeTransform {
  transform(value: any): any {
    // Lógica de transformación
    return value;
  }
}

Conclusión

Los pipes personalizados en Angular son una forma poderosa de encapsular y reutilizar la lógica de transformación de datos en tus plantillas. Siguiendo los pasos descritos en este tema, puedes crear tus propios pipes personalizados para satisfacer las necesidades específicas de tu aplicación. Recuerda considerar las implicaciones de rendimiento al decidir entre pipes puros e impuros.
