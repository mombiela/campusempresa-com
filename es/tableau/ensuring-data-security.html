Usando Extensiones de Tableau ►

En este tema, cubriremos los aspectos esenciales para garantizar la seguridad de los datos en Tableau. Esto incluye comprender la importancia de la seguridad de los datos, implementar permisos de usuario y utilizar las funciones de seguridad integradas de Tableau para proteger la información sensible.

Importancia de la Seguridad de los Datos

La seguridad de los datos es crucial en cualquier herramienta de visualización de datos, incluido Tableau. Garantizar que la información sensible esté protegida contra accesos no autorizados y brechas es esencial para mantener la confianza y el cumplimiento de las normativas.

  • Confidencialidad: Garantizar que los datos sean accesibles solo para aquellos que están autorizados.
  • Integridad: Mantener la precisión y consistencia de los datos a lo largo de su ciclo de vida.
  • Disponibilidad: Asegurar que los datos estén disponibles para los usuarios autorizados cuando se necesiten.

Permisos de Usuario

Tableau proporciona configuraciones robustas de permisos de usuario que permiten a los administradores controlar quién puede acceder e interactuar con los datos.

Tipos de Permisos

  1. Permisos de Libro de Trabajo: Controlan el acceso a los libros de trabajo.
  2. Permisos de Fuente de Datos: Controlan el acceso a las fuentes de datos.
  3. Permisos de Proyecto: Controlan el acceso a los proyectos, que pueden contener múltiples libros de trabajo y fuentes de datos.

Configuración de Permisos

Los permisos se pueden establecer en diferentes niveles para asegurar que solo los usuarios autorizados tengan acceso a datos específicos.

Ejemplo: Configuración de Permisos de Libro de Trabajo

  1. Navegar al Libro de Trabajo: Ve al libro de trabajo para el que deseas establecer permisos.
  2. Abrir Permisos: Haz clic en la pestaña 'Permisos'.
  3. Agregar Usuarios/Grupos: Agrega los usuarios o grupos para los que deseas establecer permisos.
  4. Establecer Permisos: Elige los permisos apropiados (por ejemplo, Ver, Editar, Eliminar).
| Tipo de Permiso | Descripción                  |
|-----------------|------------------------------|
| Ver             | Permite a los usuarios ver el libro de trabajo. |
| Editar          | Permite a los usuarios editar el libro de trabajo. |
| Eliminar        | Permite a los usuarios eliminar el libro de trabajo. |

Seguridad de la Fuente de Datos

Asegurar las fuentes de datos es fundamental para garantizar que los datos sensibles no se expongan.

Credenciales Incrustadas

Al publicar una fuente de datos, puedes optar por incrustar credenciales. Esto asegura que la fuente de datos pueda ser accesible sin requerir que los usuarios ingresen credenciales cada vez.

Ejemplo: Incrustar Credenciales

  1. Publicar Fuente de Datos: Al publicar una fuente de datos, selecciona la opción 'Credenciales Incrustadas'.
  2. Ingresar Credenciales: Ingresa las credenciales necesarias para la fuente de datos.
  3. Guardar: Guarda la fuente de datos con las credenciales incrustadas.

Seguridad a Nivel de Fila

La seguridad a nivel de fila te permite restringir el acceso a filas específicas en una fuente de datos según los roles de usuario.

Ejemplo: Implementar Seguridad a Nivel de Fila

  1. Crear un Filtro de Usuario: Crea un campo calculado que filtre los datos según el rol del usuario.
  2. Aplicar el Filtro: Aplica el filtro de usuario a la fuente de datos.
IF [User Role] = 'Manager' THEN [Data]
ELSEIF [User Role] = 'Employee' THEN [Data] END

Funciones de Seguridad de Tableau Server

Tableau Server ofrece funciones de seguridad adicionales para proteger tus datos.

Cifrado SSL

El cifrado SSL (Secure Sockets Layer) asegura que los datos transmitidos entre Tableau Server y los clientes sean seguros.

Habilitar Cifrado SSL

  1. Obtener un Certificado SSL: Compra o genera un certificado SSL.
  2. Configurar Tableau Server: Configura Tableau Server para usar el certificado SSL.
  3. Reiniciar Tableau Server: Reinicia Tableau Server para aplicar los cambios.

Cifrado de Datos en Reposo

Tableau Server puede cifrar los datos en reposo para protegerlos contra accesos no autorizados.

Habilitar Cifrado de Datos en Reposo

  1. Navegar a Configuración: Ve a la configuración de Tableau Server.
  2. Habilitar Cifrado: Habilita la opción de cifrado de datos en reposo.
  3. Guardar Configuración: Guarda la configuración para aplicar los cambios.

Conclusión

Garantizar la seguridad de los datos en Tableau implica una combinación de permisos de usuario, seguridad de la fuente de datos y el aprovechamiento de las funciones de seguridad integradas de Tableau Server. Al comprender e implementar estas medidas de seguridad, puedes proteger la información sensible y mantener la integridad, confidencialidad y disponibilidad de tus datos.

En resumen:

  • Comprende la importancia de la seguridad de los datos.
  • Establece permisos de usuario apropiados para libros de trabajo, fuentes de datos y proyectos.
  • Usa credenciales incrustadas y seguridad a nivel de fila para proteger las fuentes de datos.
  • Utiliza las funciones de cifrado SSL y cifrado de datos en reposo de Tableau Server.

Siguiendo estas mejores prácticas, puedes asegurar que tus datos permanezcan seguros mientras usas Tableau.
