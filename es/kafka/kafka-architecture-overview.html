Introducción

Apache Kafka es una plataforma de transmisión de eventos distribuida capaz de manejar billones de eventos al día. Se utiliza para construir canalizaciones de datos en tiempo real y aplicaciones de transmisión. Comprender la arquitectura de Kafka es crucial para aprovechar eficazmente sus capacidades.

Componentes Clave de la Arquitectura de Kafka

1. Temas y Particiones

  • Tema: Una categoría o nombre de feed en el que se almacenan y publican registros.
  • Partición: Un tema se divide en particiones, que son la unidad básica de paralelismo en Kafka.
Tema: Pedidos
Particiones: [Partición 0, Partición 1, Partición 2]

2. Productores

  • Los productores son clientes que publican (escriben) registros en temas de Kafka.
  • Los productores pueden elegir a qué partición enviar registros, a menudo basándose en una clave.
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

KafkaProducer<String, String> producer = new KafkaProducer<>(props);
ProducerRecord<String, String> record = new ProducerRecord<>("Pedidos", "pedido123", "Detalles del pedido");
producer.send(record);
producer.close();

3. Consumidores

  • Los consumidores son clientes que leen registros de temas de Kafka.
  • Los consumidores pertenecen a grupos de consumidores, lo que permite el balanceo de carga y la tolerancia a fallos.
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "consumidores-pedidos");
props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Arrays.asList("Pedidos"));

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
    for (ConsumerRecord<String, String> record : records) {
        System.out.printf("Registro consumido con clave %s y valor %s%n", record.key(), record.value());
    }
}

4. Brokers

  • Los brokers son servidores de Kafka que almacenan datos y sirven a los clientes.
  • Un clúster de Kafka está compuesto por múltiples brokers.

5. Zookeeper

  • Zookeeper gestiona y coordina los brokers de Kafka.
  • Maneja tareas como la elección de líderes para particiones y la gestión de configuraciones.

6. Replicación

  • Kafka asegura la tolerancia a fallos replicando particiones en múltiples brokers.
  • Cada partición tiene un líder y múltiples seguidores.

| Concepto | Descripción | |----------------------|-----------------------------------------------------------------------------| | Líder | El broker responsable de todas las lecturas y escrituras para la partición. | | Seguidor | Replica los datos del líder y toma el control si el líder falla. | | Factor de Replicación| El número de copias de una partición en todo el clúster de Kafka. |

7. Desplazamiento

  • Un desplazamiento es un identificador único para cada registro dentro de una partición.
  • Los consumidores usan desplazamientos para llevar un seguimiento de su posición en el registro.

Resumen

La arquitectura de Kafka está diseñada para un alto rendimiento, tolerancia a fallos y escalabilidad. Los componentes clave incluyen temas, particiones, productores, consumidores, brokers, Zookeeper, replicación y desplazamientos. Comprender estos componentes y sus interacciones es esencial para construir aplicaciones robustas basadas en Kafka.

Al dominar la arquitectura de Kafka, puedes diseñar e implementar eficientemente soluciones de transmisión de datos que satisfagan las necesidades de tu organización.
