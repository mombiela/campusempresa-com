En esta sección, cubriremos cómo enviar mensajes usando Apache Kafka. Comenzaremos con los conceptos básicos y gradualmente avanzaremos hacia conceptos más avanzados. Al final de esta sección, deberías ser capaz de enviar mensajes a temas de Kafka de manera eficiente y comprender los mecanismos subyacentes.

Introducción a los Productores de Kafka

Un Productor de Kafka es un cliente que envía registros a un clúster de Kafka. Los productores son responsables de elegir a qué partición dentro de un tema asignar cada registro.

Conceptos Clave

  • Productor: La aplicación cliente que envía registros a Kafka.
  • Registro: Los datos que se envían a Kafka, consistentes en una clave, un valor y metadatos opcionales.
  • Tema: Una categoría o nombre de feed al que se envían los registros.
  • Partición: Una división de un tema, que permite el procesamiento en paralelo.

Configuración de un Productor de Kafka

Antes de enviar mensajes, necesitas configurar un productor de Kafka. Esto implica configurar las propiedades del productor e inicializar la instancia del productor.

Configuración Básica

Aquí hay algunas propiedades esenciales que necesitas configurar:

  • bootstrap.servers: Una lista de pares host/puerto para usar al establecer la conexión inicial con el clúster de Kafka.
  • key.serializer: Clase serializadora para la clave que implementa la interfaz org.apache.kafka.common.serialization.Serializer.
  • value.serializer: Clase serializadora para el valor que implementa la interfaz org.apache.kafka.common.serialization.Serializer.

Ejemplo de Configuración

import org.apache.kafka.clients.producer.KafkaProducer;
import org.apache.kafka.clients.producer.ProducerConfig;
import org.apache.kafka.common.serialization.StringSerializer;

import java.util.Properties;

public class ProducerConfigExample {
    public static void main(String[] args) {
        Properties properties = new Properties();
        properties.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        properties.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class.getName());
        properties.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class.getName());

        KafkaProducer<String, String> producer = new KafkaProducer<>(properties);
        // Usa el productor para enviar mensajes
    }
}

Envío de Mensajes

Una vez que el productor está configurado, puedes comenzar a enviar mensajes a temas de Kafka.

Envío de un Mensaje Simple

Para enviar un mensaje, necesitas crear un ProducerRecord y usar el método send del productor.

import org.apache.kafka.clients.producer.ProducerRecord;

public class SimpleMessageProducer {
    public static void main(String[] args) {
        // Asumiendo que el productor ya está configurado como se muestra arriba
        KafkaProducer<String, String> producer = new KafkaProducer<>(properties);

        String topic = "my-topic";
        String key = "my-key";
        String value = "Hello, Kafka!";

        ProducerRecord<String, String> record = new ProducerRecord<>(topic, key, value);
        producer.send(record);

        producer.close();
    }
}

Manejo de Callbacks

También puedes manejar callbacks para ser notificado cuando un mensaje se envía con éxito o si ocurre un error.

import org.apache.kafka.clients.producer.Callback;
import org.apache.kafka.clients.producer.RecordMetadata;
import org.apache.kafka.clients.producer.ProducerRecord;

public class CallbackProducer {
    public static void main(String[] args) {
        KafkaProducer<String, String> producer = new KafkaProducer<>(properties);

        String topic = "my-topic";
        String key = "my-key";
        String value = "Hello, Kafka with Callback!";

        ProducerRecord<String, String> record = new ProducerRecord<>(topic, key, value);
        producer.send(record, new Callback() {
            @Override
            public void onCompletion(RecordMetadata metadata, Exception exception) {
                if (exception == null) {
                    System.out.println("Mensaje enviado con éxito a " + metadata.topic() + " partición " + metadata.partition());
                } else {
                    exception.printStackTrace();
                }
            }
        });

        producer.close();
    }
}

Configuraciones Avanzadas

Configuración de Acknowledgment

Kafka te permite configurar el nivel de acknowledgment para asegurar la fiabilidad de la entrega de mensajes.

  • acks=0: No se requiere acknowledgment.
  • acks=1: Se requiere acknowledgment del líder.
  • acks=all: Todos los réplicas en sincronía deben dar acknowledgment.

Ejemplo con Acknowledgment

properties.put(ProducerConfig.ACKS_CONFIG, "all");

Tamaño de Lote y Tiempo de Espera

Puedes configurar el tamaño del lote y el tiempo de espera para optimizar el rendimiento.

  • batch.size: El tamaño del lote en bytes.
  • linger.ms: El tiempo de espera antes de enviar un lote.

Ejemplo con Tamaño de Lote y Tiempo de Espera

properties.put(ProducerConfig.BATCH_SIZE_CONFIG, 16384); // 16 KB
properties.put(ProducerConfig.LINGER_MS_CONFIG, 10); // 10 ms
