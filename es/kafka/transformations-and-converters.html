Introducción

Apache Kafka es una herramienta poderosa para construir pipelines de datos en tiempo real y aplicaciones de streaming. Una de las características clave que hacen a Kafka tan versátil es su capacidad para transformar y convertir datos a medida que fluyen a través del sistema. Esta sección cubrirá los conceptos básicos de las transformaciones y convertidores en Kafka, desde niveles principiantes hasta avanzados.

Conceptos Clave

  • Transformaciones: Operaciones que modifican los datos a medida que se mueven a través de Kafka.
  • Convertidores: Herramientas que cambian el formato de los datos para hacerlos compatibles con diferentes sistemas.

Transformaciones

Las transformaciones en Kafka son típicamente manejadas por Kafka Streams o Kafka Connect. Estas transformaciones pueden variar desde modificaciones simples de datos hasta agregaciones y uniones complejas.

Transformaciones de Kafka Streams

Kafka Streams es una biblioteca cliente para construir aplicaciones y microservicios, donde los datos de entrada y salida se almacenan en clústeres de Kafka. Permite el procesamiento y la transformación de datos en tiempo real.

Transformaciones Básicas

  • mapValues: Transforma los valores de los registros.
  • filter: Filtra registros basados en una condición.
  • flatMap: Transforma cada registro en cero o más registros.
KStream<String, String> source = builder.stream("input-topic");

// Ejemplo de mapValues
KStream<String, String> uppercased = source.mapValues(value -> value.toUpperCase());

// Ejemplo de filter
KStream<String, String> filtered = source.filter((key, value) -> value.length() > 5);

// Ejemplo de flatMap
KStream<String, String> expanded = source.flatMap((key, value) -> {
    List<KeyValue<String, String>> result = new ArrayList<>();
    for (String word : value.split(" ")) {
        result.add(new KeyValue<>(key, word));
    }
    return result;
});

Transformaciones Avanzadas

  • groupByKey: Agrupa registros por clave.
  • aggregate: Agrega registros por clave.
  • join: Une dos streams basados en una clave.
KStream<String, String> source1 = builder.stream("input-topic1");
KStream<String, String> source2 = builder.stream("input-topic2");

// Ejemplo de groupByKey y aggregate
KGroupedStream<String, String> grouped = source.groupByKey();
KTable<String, Long> counts = grouped.count();

// Ejemplo de join
KStream<String, String> joined = source1.join(source2, 
    (value1, value2) -> value1 + "-" + value2,
    JoinWindows.of(Duration.ofMinutes(5))
);

Convertidores

Los convertidores se utilizan para cambiar el formato de los datos. Esto es particularmente útil al integrar Kafka con otros sistemas que pueden usar diferentes formatos de datos.

Convertidores de Kafka Connect

Kafka Connect proporciona convertidores integrados para formatos de datos comunes como JSON y Avro.

Convertidor JSON

El convertidor JSON se utiliza para convertir datos hacia y desde el formato JSON.

key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter

Convertidor Avro

El convertidor Avro se utiliza para convertir datos hacia y desde el formato Avro. Requiere un registro de esquemas para gestionar los esquemas Avro.

key.converter=io.confluent.connect.avro.AvroConverter
value.converter=io.confluent.connect.avro.AvroConverter
key.converter.schema.registry.url=http://localhost:8081
value.converter.schema.registry.url=http://localhost:8081

Convertidores Personalizados

En algunos casos, puede ser necesario implementar convertidores personalizados para manejar formatos de datos o transformaciones específicas.

public class CustomConverter implements Converter {
    @Override
    public byte[] fromConnectData(String topic, Schema schema, Object value) {
        // Lógica de serialización personalizada
    }

    @Override
    public SchemaAndValue toConnectData(String topic, byte[] value) {
        // Lógica de deserialización personalizada
    }
}

Conclusión

Las transformaciones y los convertidores son componentes esenciales de cualquier pipeline de datos basado en Kafka. Permiten modificar y formatear datos a medida que fluyen a través del sistema, haciendo posible integrar Kafka con una amplia variedad de otros sistemas y aplicaciones. Al comprender y utilizar estas herramientas, puedes construir soluciones de procesamiento de datos más flexibles y poderosas.
