Kafka Streams es una poderosa biblioteca para construir aplicaciones de procesamiento de flujos y microservicios. Te permite procesar datos en tiempo real con facilidad, aprovechando el poder de Apache Kafka. Este tema te introducirá a los conceptos básicos de Kafka Streams, su arquitectura y cómo crear una aplicación simple de Kafka Streams.

¿Qué es Kafka Streams?

Kafka Streams es una biblioteca cliente para construir aplicaciones y microservicios, donde los datos de entrada y salida se almacenan en clústeres de Kafka. Combina la simplicidad de escribir y desplegar aplicaciones estándar de Java y Scala en el lado del cliente con los beneficios de la tecnología de clúster del lado del servidor de Kafka.

Conceptos Clave

  • Stream: Una secuencia de registros sin límites y en constante actualización.
  • Table: Una vista de un flujo como un registro de cambios, que representa el estado más reciente de cada clave.
  • Topology: Un gráfico de nodos de procesamiento de flujos que representa la lógica de computación.
  • Processor: Un nodo en la topología que procesa registros.
  • State Store: Una base de datos local que almacena el estado de una aplicación de procesamiento de flujos.

Arquitectura de Kafka Streams

Las aplicaciones de Kafka Streams se construyen alrededor de algunos conceptos clave:

  • Streams y Tables: Los flujos son la abstracción central en Kafka Streams, representando una secuencia ilimitada de registros. Las tablas se derivan de los flujos y representan el estado más reciente de cada clave.
  • Topology: La topología define la lógica de procesamiento de flujos, que consiste en fuentes, procesadores y sumideros.
  • Stateful Processing: Kafka Streams soporta el procesamiento con estado, permitiéndote mantener y consultar el estado dentro de tu aplicación.

Ejemplo de Arquitectura

| Componente | Descripción | |------------------|-----------------------------------------------------------------------------| | Source Processor | Lee datos de los temas de Kafka. | | Stream Processor | Procesa los datos, aplicando transformaciones, filtros y agregaciones. | | Sink Processor | Escribe los datos procesados de vuelta a los temas de Kafka o sistemas externos. | | State Store | Mantiene el estado para el procesamiento con estado, como agregaciones o uniones. |

Creando una Aplicación Simple de Kafka Streams

Vamos a recorrer los pasos para crear una aplicación simple de Kafka Streams que lea datos de un tema de Kafka, los procese y escriba los resultados en otro tema de Kafka.

Paso 1: Configura tu Proyecto

Primero, crea un nuevo proyecto Maven o Gradle y añade la dependencia de Kafka Streams a tu archivo pom.xml o build.gradle.

Maven:

<dependency>
    <groupId>org.apache.kafka</groupId>
    <artifactId>kafka-streams</artifactId>
    <version>3.0.0</version>
</dependency>

Gradle:

implementation 'org.apache.kafka:kafka-streams:3.0.0'

Paso 2: Define la Topología de Procesamiento de Flujos

Crea una clase para definir tu topología de procesamiento de flujos. En este ejemplo, leeremos de un tema de entrada, convertiremos los valores a mayúsculas y escribiremos los resultados en un tema de salida.

import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.StreamsConfig;
import org.apache.kafka.streams.kstream.KStream;

import java.util.Properties;

public class SimpleKafkaStreamsApp {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(StreamsConfig.APPLICATION_ID_CONFIG, "streams-example");
        props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, "org.apache.kafka.common.serialization.Serdes$StringSerde");
        props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, "org.apache.kafka.common.serialization.Serdes$StringSerde");

        StreamsBuilder builder = new StreamsBuilder();
        KStream<String, String> sourceStream = builder.stream("input-topic");
        KStream<String, String> uppercasedStream = sourceStream.mapValues(value -> value.toUpperCase());
        uppercasedStream.to("output-topic");

        KafkaStreams streams = new KafkaStreams(builder.build(), props);
        streams.start();

        Runtime.getRuntime().addShutdownHook(new Thread(streams::close));
    }
}

Paso 3: Ejecuta tu Aplicación

Compila y ejecuta tu aplicación. Asegúrate de que tu clúster de Kafka esté en funcionamiento y que los temas de entrada y salida existan.

mvn clean package
java -cp target/your-app-jar-with-dependencies.jar com.yourpackage.SimpleKafkaStreamsApp
