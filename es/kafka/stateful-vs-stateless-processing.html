Procesamiento con Estado vs Sin Estado
Introducción a Kafka Connect ►

Apache Kafka es una herramienta poderosa para construir canalizaciones de datos en tiempo real y aplicaciones de streaming. Entender la diferencia entre el procesamiento con estado y sin estado es crucial para diseñar aplicaciones Kafka eficientes y escalables. Este tema cubrirá los conceptos clave, proporcionará ejemplos y ofrecerá ejercicios para solidificar tu comprensión.

Conceptos Clave

Procesamiento Sin Estado

  • Definición: El procesamiento sin estado significa que cada evento se procesa de manera independiente sin ningún conocimiento de eventos anteriores.
  • Características:
    • No es necesario mantener estado entre eventos.
    • Más fácil de escalar horizontalmente.
    • Normalmente implica transformaciones simples o filtrado.
  • Casos de Uso:
    • Filtrar mensajes no deseados.
    • Transformaciones simples como convertir formatos de datos.

Procesamiento con Estado

  • Definición: El procesamiento con estado implica mantener estado a través de múltiples eventos, permitiendo que la aplicación recuerde información entre eventos.
  • Características:
    • Requiere mantener estado, a menudo en un almacén local o base de datos externa.
    • Más complejo de escalar y gestionar.
    • Permite operaciones más complejas como agregaciones, uniones y cálculos en ventanas de tiempo.
  • Casos de Uso:
    • Contar ocurrencias de eventos.
    • Agregar datos sobre ventanas de tiempo.
    • Unir flujos de datos.

Procesamiento Sin Estado en Kafka Streams

Ejemplo: Filtrado de Mensajes

Comencemos con un ejemplo simple de procesamiento sin estado: filtrado de mensajes.

import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.KStream;
import org.apache.kafka.streams.kstream.KTable;
import org.apache.kafka.streams.kstream.Consumed;
import org.apache.kafka.streams.kstream.Produced;

public class StatelessExample {
    public static void main(String[] args) {
        StreamsBuilder builder = new StreamsBuilder();
        KStream<String, String> sourceStream = builder.stream("input-topic");

        KStream<String, String> filteredStream = sourceStream.filter(
            (key, value) -> value.contains("important")
        );

        filteredStream.to("output-topic", Produced.with(Serdes.String(), Serdes.String()));

        KafkaStreams streams = new KafkaStreams(builder.build(), new Properties());
        streams.start();
    }
}

Explicación

  • Flujo de Origen: Lee desde "input-topic".
  • Operación de Filtrado: Mantiene solo los mensajes que contienen la palabra "important".
  • Flujo de Salida: Escribe los mensajes filtrados en "output-topic".

Procesamiento con Estado en Kafka Streams

Ejemplo: Contar Ocurrencias

Ahora, veamos un ejemplo de procesamiento con estado: contar ocurrencias de palabras.

import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.KStream;
import org.apache.kafka.streams.kstream.KTable;
import org.apache.kafka.streams.kstream.Consumed;
import org.apache.kafka.streams.kstream.Produced;
import org.apache.kafka.streams.kstream.Materialized;
import org.apache.kafka.streams.state.Stores;

public class StatefulExample {
    public static void main(String[] args) {
        StreamsBuilder builder = new StreamsBuilder();
        KStream<String, String> sourceStream = builder.stream("input-topic");

        KTable<String, Long> wordCounts = sourceStream
            .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
            .groupBy((key, word) -> word)
            .count(Materialized.<String, Long, KeyValueStore<Bytes, byte[]>>as("counts-store"));

        wordCounts.toStream().to("output-topic", Produced.with(Serdes.String(), Serdes.Long()));

        KafkaStreams streams = new KafkaStreams(builder.build(), new Properties());
        streams.start();
    }
}

Explicación

  • Flujo de Origen: Lee desde "input-topic".
  • Operación FlatMapValues: Divide cada mensaje en palabras.
  • Operación GroupBy: Agrupa las palabras.
  • Operación Count: Cuenta las ocurrencias de cada palabra, manteniendo el estado en "counts-store".
  • Flujo de Salida: Escribe los conteos de palabras en "output-topic".

Comparación

| Característica | Procesamiento Sin Estado | Procesamiento con Estado | |------------------------|----------------------------------------|----------------------------------------| | Gestión de Estado | No se requiere gestión de estado | Requiere gestión de estado | | Complejidad | Más simple de implementar y escalar | Más complejo, requiere gestión cuidadosa del estado | | Casos de Uso | Filtrado, transformaciones simples | Agregaciones, uniones, cálculos en ventanas de tiempo | | Escalabilidad | Más fácil de escalar horizontalmente | Más desafiante de escalar |

Ejercicios

  1. Ejercicio de Procesamiento Sin Estado:

    • Modifica el ejemplo sin estado para filtrar mensajes que contengan la palabra "urgent".
    • Escribe los mensajes filtrados en un nuevo tema llamado "urgent-messages".

  2. Ejercicio de Procesamiento con Estado:

    • Modifica el ejemplo con estado para contar las ocurrencias de cada carácter en lugar de palabras.
    • Escribe los conteos de caracteres en un nuevo tema llamado "character-counts".

Conclusión

Entender la diferencia entre el procesamiento con estado y sin estado es esencial para diseñar aplicaciones Kafka efectivas. El procesamiento sin estado es más simple y fácil de escalar, lo que lo hace adecuado para tareas sencillas como el filtrado y las transformaciones simples. El procesamiento con estado, aunque más complejo, permite operaciones poderosas como agregaciones y uniones, que son cruciales para muchas aplicaciones de procesamiento de datos en tiempo real. Al dominar ambos enfoques, puedes aprovechar Kafka Streams para construir aplicaciones de streaming robustas y escalables.
