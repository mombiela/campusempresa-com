En este tema, te guiaremos a través del proceso de construir una aplicación simple usando PL/SQL. Esto incluirá la creación de tablas, la escritura de código PL/SQL para manipular datos y la comprensión de cómo integrar PL/SQL con otros objetos de la base de datos.

Introducción a PL/SQL

PL/SQL (Procedural Language/Structured Query Language) es la extensión procedimental de Oracle Corporation para SQL y la base de datos relacional Oracle. Te permite escribir código que combina el poder de SQL con constructos procedimentales.

Conceptos Clave

  • Estructura del Bloque PL/SQL: La unidad básica de un programa PL/SQL es un bloque, que puede ser anónimo o nombrado.
  • Estructuras de Control: PL/SQL soporta estructuras de control como bucles, condicionales y manejo de excepciones.
  • Cursores: Se utilizan para recuperar múltiples filas de una consulta.
  • Procedimientos y Funciones: Encapsulan código reutilizable.
  • Paquetes: Agrupan procedimientos, funciones y otros elementos relacionados.

Configuración del Entorno

Antes de comenzar a construir nuestra aplicación, asegúrate de tener acceso a una base de datos Oracle y una herramienta como SQL*Plus o Oracle SQL Developer.

Creación del Esquema de la Base de Datos

Crearemos un esquema simple para un sistema de gestión de bibliotecas. Esto incluye tablas para libros, autores y préstamos.

CREATE TABLE authors (
    author_id NUMBER PRIMARY KEY,
    name VARCHAR2(100)
);

CREATE TABLE books (
    book_id NUMBER PRIMARY KEY,
    title VARCHAR2(100),
    author_id NUMBER,
    FOREIGN KEY (author_id) REFERENCES authors(author_id)
);

CREATE TABLE loans (
    loan_id NUMBER PRIMARY KEY,
    book_id NUMBER,
    loan_date DATE,
    return_date DATE,
    FOREIGN KEY (book_id) REFERENCES books(book_id)
);

Escribiendo Código PL/SQL

Bloques Anónimos

Un bloque anónimo es un bloque PL/SQL que no tiene nombre. Es útil para operaciones rápidas y de una sola vez.

BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello, PL/SQL!');
END;
/

Procedimientos Almacenados

Los procedimientos almacenados son bloques PL/SQL nombrados que pueden ser reutilizados. Vamos a crear un procedimiento para prestar un libro.

CREATE OR REPLACE PROCEDURE loan_book(p_book_id NUMBER, p_loan_date DATE) IS
BEGIN
    INSERT INTO loans (loan_id, book_id, loan_date) 
    VALUES (loans_seq.NEXTVAL, p_book_id, p_loan_date);
    COMMIT;
END;
/

Funciones

Las funciones son similares a los procedimientos pero devuelven un valor. Vamos a crear una función para verificar si un libro está disponible.

CREATE OR REPLACE FUNCTION is_book_available(p_book_id NUMBER) RETURN BOOLEAN IS
    v_count NUMBER;
BEGIN
    SELECT COUNT(*) INTO v_count 
    FROM loans 
    WHERE book_id = p_book_id AND return_date IS NULL;

    RETURN v_count = 0;
END;
/

Uso de Cursores

Los cursores te permiten obtener múltiples filas de una consulta. Aquí hay un ejemplo de uso de un cursor explícito.

DECLARE
    CURSOR book_cursor IS
        SELECT title FROM books;
    v_title books.title%TYPE;
BEGIN
    OPEN book_cursor;
    LOOP
        FETCH book_cursor INTO v_title;
        EXIT WHEN book_cursor%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_title);
    END LOOP;
    CLOSE book_cursor;
END;
/

Conceptos Avanzados de PL/SQL

Manejo de Excepciones

El manejo de excepciones te permite gestionar errores de manera elegante.

BEGIN
    -- Alguna operación que podría causar un error
    INSERT INTO loans (loan_id, book_id, loan_date) VALUES (1, 1, SYSDATE);
EXCEPTION
    WHEN DUP_VAL_ON_INDEX THEN
        DBMS_OUTPUT.PUT_LINE('Error de valor duplicado.');
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('Ocurrió un error inesperado.');
END;
/

Paquetes

Los paquetes agrupan procedimientos, funciones y otros elementos relacionados. Aquí hay un ejemplo de un paquete simple.

CREATE OR REPLACE PACKAGE library_pkg IS
    PROCEDURE loan_book(p_book_id NUMBER, p_loan_date DATE);
    FUNCTION is_book_available(p_book_id NUMBER) RETURN BOOLEAN;
END library_pkg;
/

CREATE OR REPLACE PACKAGE BODY library_pkg IS
    PROCEDURE loan_book(p_book_id NUMBER, p_loan_date DATE) IS
    BEGIN
        INSERT INTO loans (loan_id, book_id, loan_date) 
        VALUES (loans_seq.NEXTVAL, p_book_id, p_loan_date);
        COMMIT;
    END loan_book;

    FUNCTION is_book_available(p_book_id NUMBER) RETURN BOOLEAN IS
        v_count NUMBER;
    BEGIN
        SELECT COUNT(*) INTO v_count 
        FROM loans 
        WHERE book_id = p_book_id AND return_date IS NULL;

        RETURN v_count = 0;
    END is_book_available;
END library_pkg;
/
