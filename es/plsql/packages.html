Introducción a los Paquetes

Los paquetes en PL/SQL son objetos de esquema que agrupan tipos, variables y subprogramas (procedimientos y funciones) de PL/SQL relacionados lógicamente. Ayudan a organizar y gestionar grandes programas de PL/SQL proporcionando modularidad, encapsulación y reutilización.

Conceptos Clave

  • Modularidad: Los paquetes permiten agrupar procedimientos, funciones, variables y otros constructos de PL/SQL relacionados.
  • Encapsulación: Los paquetes pueden ocultar los detalles de implementación de los subprogramas y exponer solo las partes necesarias.
  • Reutilización: Una vez creado un paquete, sus componentes pueden ser reutilizados en múltiples aplicaciones.

Estructura de un Paquete

Un paquete en PL/SQL consta de dos partes:

  1. Especificación del Paquete: Declara los elementos públicos que pueden ser accedidos por otros bloques de PL/SQL.
  2. Cuerpo del Paquete: Define la implementación de los subprogramas declarados en la especificación del paquete.

Especificación del Paquete

La especificación del paquete es la interfaz del paquete. Declara los tipos, variables, constantes, excepciones, cursores y subprogramas que son accesibles desde fuera del paquete.

CREATE OR REPLACE PACKAGE nombre_paquete IS
  -- Declaraciones de tipos públicos
  TYPE nombre_tipo IS RECORD (...);

  -- Declaraciones de variables públicas
  nombre_variable tipo_dato;

  -- Declaraciones de constantes públicas
  nombre_constante CONSTANT tipo_dato := valor;

  -- Declaraciones de excepciones públicas
  nombre_excepcion EXCEPTION;

  -- Declaraciones de cursores públicos
  CURSOR nombre_cursor IS sentencia_select;

  -- Declaraciones de procedimientos y funciones públicas
  PROCEDURE nombre_procedimiento(lista_parametros);
  FUNCTION nombre_funcion(lista_parametros) RETURN tipo_retorno;
END nombre_paquete;

Cuerpo del Paquete

El cuerpo del paquete contiene la implementación de los subprogramas declarados en la especificación del paquete. También puede contener declaraciones privadas que no son accesibles fuera del paquete.

CREATE OR REPLACE PACKAGE BODY nombre_paquete IS
  -- Declaraciones de tipos privados
  TYPE nombre_tipo_privado IS RECORD (...);

  -- Declaraciones de variables privadas
  nombre_variable_privada tipo_dato;

  -- Declaraciones de constantes privadas
  nombre_constante_privada CONSTANT tipo_dato := valor;

  -- Declaraciones de excepciones privadas
  nombre_excepcion_privada EXCEPTION;

  -- Declaraciones de cursores privados
  CURSOR nombre_cursor_privado IS sentencia_select;

  -- Implementación de procedimientos y funciones públicas
  PROCEDURE nombre_procedimiento(lista_parametros) IS
  BEGIN
    -- Implementación del procedimiento
  END nombre_procedimiento;

  FUNCTION nombre_funcion(lista_parametros) RETURN tipo_retorno IS
  BEGIN
    -- Implementación de la función
    RETURN valor;
  END nombre_funcion;

  -- Procedimientos y funciones privadas
  PROCEDURE nombre_procedimiento_privado IS
  BEGIN
    -- Implementación del procedimiento
  END nombre_procedimiento_privado;

  FUNCTION nombre_funcion_privada RETURN tipo_retorno IS
  BEGIN
    -- Implementación de la función
    RETURN valor;
  END nombre_funcion_privada;
END nombre_paquete;

Ejemplo: Creación y Uso de un Paquete

Vamos a crear un paquete simple que gestione datos de empleados.

Especificación del Paquete

CREATE OR REPLACE PACKAGE emp_pkg IS
  -- Declaración de tipo público
  TYPE emp_rec IS RECORD (
    emp_id NUMBER,
    emp_name VARCHAR2(100),
    emp_salary NUMBER
  );

  -- Declaración de procedimiento público
  PROCEDURE add_employee(p_emp_id NUMBER, p_emp_name VARCHAR2, p_emp_salary NUMBER);

  -- Declaración de función pública
  FUNCTION get_employee(p_emp_id NUMBER) RETURN emp_rec;
END emp_pkg;

Cuerpo del Paquete

CREATE OR REPLACE PACKAGE BODY emp_pkg IS
  -- Variable privada para almacenar datos de empleados
  emp_data emp_rec;

  -- Implementación del procedimiento add_employee
  PROCEDURE add_employee(p_emp_id NUMBER, p_emp_name VARCHAR2, p_emp_salary NUMBER) IS
  BEGIN
    emp_data.emp_id := p_emp_id;
    emp_data.emp_name := p_emp_name;
    emp_data.emp_salary := p_emp_salary;
  END add_employee;

  -- Implementación de la función get_employee
  FUNCTION get_employee(p_emp_id NUMBER) RETURN emp_rec IS
  BEGIN
    IF emp_data.emp_id = p_emp_id THEN
      RETURN emp_data;
    ELSE
      RETURN NULL;
    END IF;
  END get_employee;
END emp_pkg;

Uso del Paquete

BEGIN
  -- Añadir un empleado
  emp_pkg.add_employee(1, 'John Doe', 50000);

  -- Recuperar datos del empleado
  DECLARE
    emp emp_pkg.emp_rec;
  BEGIN
    emp := emp_pkg.get_employee(1);
    DBMS_OUTPUT.PUT_LINE('Nombre del Empleado: ' || emp.emp_name);
  END;
END;

Conclusión

Los paquetes en PL/SQL proporcionan una forma poderosa de organizar y gestionar tu código PL/SQL. Al agrupar procedimientos, funciones y otros elementos relacionados, los paquetes mejoran la modularidad, encapsulación y reutilización. Entender cómo crear y usar paquetes es esencial para desarrollar aplicaciones PL/SQL robustas y mantenibles.
