Introducción a las Funciones

Las funciones en PL/SQL son subprogramas que devuelven un único valor. Se utilizan para encapsular lógica reutilizable, haciendo que el código sea modular y más fácil de mantener.

Conceptos Clave

  • Declaración de Función: La estructura de una función en PL/SQL.
  • Cuerpo de la Función: Contiene la parte ejecutable de la función.
  • Tipo de Retorno: Especifica el tipo de dato del valor devuelto por la función.
  • Parámetros: Valores de entrada que la función puede aceptar.

Creación de Funciones

Para crear una función en PL/SQL, se utiliza la sentencia CREATE FUNCTION.

Sintaxis

CREATE [OR REPLACE] FUNCTION nombre_funcion 
   (parametro_1 tipo_dato, parametro_2 tipo_dato, ...)
RETURN tipo_dato_retorno 
IS
   -- Sección de declaración
BEGIN
   -- Sección ejecutable
   RETURN valor;
EXCEPTION
   -- Sección de manejo de excepciones
END nombre_funcion;

Ejemplo

Vamos a crear una función simple que calcula el cuadrado de un número.

CREATE OR REPLACE FUNCTION calcular_cuadrado (p_numero IN NUMBER)
RETURN NUMBER
IS
   v_resultado NUMBER;
BEGIN
   v_resultado := p_numero * p_numero;
   RETURN v_resultado;
EXCEPTION
   WHEN OTHERS THEN
      RETURN NULL;
END calcular_cuadrado;

Explicación

  • Nombre de la Función: calcular_cuadrado
  • Parámetro: p_numero de tipo NUMBER
  • Tipo de Retorno: NUMBER
  • Variable: v_resultado para almacenar el resultado
  • Lógica: Multiplica p_numero por sí mismo y devuelve el resultado
  • Manejo de Excepciones: Devuelve NULL si ocurre algún error

Llamada a Funciones

Las funciones pueden ser llamadas desde bloques PL/SQL, sentencias SQL u otros subprogramas PL/SQL.

Ejemplo

DECLARE
   v_cuadrado NUMBER;
BEGIN
   v_cuadrado := calcular_cuadrado(5);
   DBMS_OUTPUT.PUT_LINE('El cuadrado de 5 es: ' || v_cuadrado);
END;

Explicación

  • Declaración: Declara una variable v_cuadrado para almacenar el resultado.
  • Llamada a la Función: Llama a calcular_cuadrado con 5 como argumento.
  • Salida: Muestra el resultado usando DBMS_OUTPUT.PUT_LINE.

Características Avanzadas de las Funciones

Funciones con Múltiples Parámetros

Las funciones pueden aceptar múltiples parámetros, tanto de entrada (IN) como de salida (OUT).

Ejemplo

CREATE OR REPLACE FUNCTION calcular_area (p_longitud IN NUMBER, p_ancho IN NUMBER)
RETURN NUMBER
IS
   v_area NUMBER;
BEGIN
   v_area := p_longitud * p_ancho;
   RETURN v_area;
EXCEPTION
   WHEN OTHERS THEN
      RETURN NULL;
END calcular_area;

Explicación

  • Parámetros: p_longitud y p_ancho de tipo NUMBER
  • Lógica: Multiplica p_longitud por p_ancho para calcular el área

Uso de Funciones en Sentencias SQL

Las funciones pueden ser usadas directamente en sentencias SQL.

Ejemplo

SELECT calcular_area(10, 20) AS area FROM dual;

Explicación

  • Sentencia SQL: Llama a calcular_area con 10 y 20 como argumentos y selecciona el resultado.

Funciones Recursivas

Las funciones pueden llamarse a sí mismas recursivamente para resolver problemas que pueden descomponerse en tareas más pequeñas y repetitivas.

Ejemplo

CREATE OR REPLACE FUNCTION factorial (p_numero IN NUMBER)
RETURN NUMBER
IS
BEGIN
   IF p_numero = 0 THEN
      RETURN 1;
   ELSE
      RETURN p_numero * factorial(p_numero - 1);
   END IF;
EXCEPTION
   WHEN OTHERS THEN
      RETURN NULL;
END factorial;

Explicación

  • Caso Base: Devuelve 1 si p_numero es 0.
  • Caso Recursivo: Se llama a sí misma con p_numero - 1.

Conclusión

Las funciones en PL/SQL son herramientas poderosas para crear código reutilizable y modular. Pueden aceptar múltiples parámetros, manejar excepciones e incluso llamarse a sí mismas recursivamente. Entender cómo crear y usar funciones de manera efectiva es esencial para escribir programas PL/SQL eficientes. Practica creando y llamando funciones para volverte competente en su uso.
