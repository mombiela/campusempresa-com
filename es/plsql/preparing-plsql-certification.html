Introducción

Prepararse para una certificación PL/SQL puede ser un esfuerzo gratificante que valida tus habilidades y conocimientos en el Lenguaje Procedural de Oracle para SQL. Esta sección te guiará a través de las áreas clave en las que necesitas enfocarte, proporcionará ejemplos y ofrecerá ejercicios para asegurarte de que estés bien preparado para el examen de certificación.

Conceptos Clave a Dominar

Para sobresalir en el examen de certificación PL/SQL, debes ser competente en las siguientes áreas:

  • Conceptos Básicos de PL/SQL

    • Escribir sentencias ejecutables

  • Estructuras de Control

    • Sentencias condicionales (IF, CASE)
    • Bucles (FOR, WHILE, LOOP)

  • Cursores

    • Cursores implícitos y explícitos

    • Bucles FOR de cursores

  • Manejo de Excepciones

    • Excepciones predefinidas
    • Excepciones definidas por el usuario
    • Propagación de excepciones

  • Procedimientos y Funciones Almacenados

    • Crear e invocar procedimientos y funciones
    • Modos de parámetros (IN, OUT, IN OUT)
    • Sobrecarga

  • Paquetes

    • Crear paquetes
    • Especificación y cuerpo del paquete
    • Uso de variables y cursores de paquetes

Conceptos Básicos de PL/SQL

Comprender la Sintaxis y Estructura de PL/SQL

PL/SQL es un lenguaje estructurado en bloques. Cada bloque consta de tres secciones: declaración, ejecución y manejo de excepciones.

DECLARE
  -- Sección de declaración
  v_example VARCHAR2(100);
BEGIN
  -- Sección de ejecución
  v_example := '¡Hola, PL/SQL!';
  DBMS_OUTPUT.PUT_LINE(v_example);
EXCEPTION
  -- Sección de manejo de excepciones
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Ocurrió un error.');
END;

Declarar Variables y Constantes

Las variables y constantes se declaran en la sección de declaración.

DECLARE
  v_name VARCHAR2(50);
  c_max CONSTANT NUMBER := 100;
BEGIN
  v_name := 'John Doe';
  DBMS_OUTPUT.PUT_LINE('Nombre: ' || v_name);
  DBMS_OUTPUT.PUT_LINE('Valor máximo: ' || c_max);
END;

Estructuras de Control

Sentencias Condicionales

Las sentencias condicionales te permiten ejecutar código basado en ciertas condiciones.

DECLARE
  v_score NUMBER := 85;
BEGIN
  IF v_score >= 90 THEN
    DBMS_OUTPUT.PUT_LINE('Calificación: A');
  ELSIF v_score >= 80 THEN
    DBMS_OUTPUT.PUT_LINE('Calificación: B');
  ELSE
    DBMS_OUTPUT.PUT_LINE('Calificación: C');
  END IF;
END;

Bucles

Los bucles se usan para ejecutar un bloque de código múltiples veces.

BEGIN
  FOR i IN 1..5 LOOP
    DBMS_OUTPUT.PUT_LINE('Iteración: ' || i);
  END LOOP;
END;

Cursores

Cursores Implícitos y Explícitos

Los cursores se usan para manejar resultados de consultas fila por fila.

DECLARE
  CURSOR c_employees IS
    SELECT first_name, last_name FROM employees;
  v_first_name employees.first_name%TYPE;
  v_last_name employees.last_name%TYPE;
BEGIN
  OPEN c_employees;
  LOOP
    FETCH c_employees INTO v_first_name, v_last_name;
    EXIT WHEN c_employees%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE(v_first_name || ' ' || v_last_name);
  END LOOP;
  CLOSE c_employees;
END;

Manejo de Excepciones

Excepciones Predefinidas y Definidas por el Usuario

Las excepciones manejan errores en tiempo de ejecución en PL/SQL.

DECLARE
  v_number NUMBER;
BEGIN
  v_number := 10 / 0; -- Esto causará una excepción predefinida
EXCEPTION
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('No se permite la división por cero.');
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Ocurrió un error inesperado.');
END;

Procedimientos y Funciones Almacenados

Crear e Invocar Procedimientos y Funciones

Los procedimientos y funciones encapsulan código reutilizable.

CREATE OR REPLACE PROCEDURE greet_user(p_name IN VARCHAR2) IS
BEGIN
  DBMS_OUTPUT.PUT_LINE('¡Hola, ' || p_name || '!');
END;
/

BEGIN
  greet_user('Alice');
END;

Paquetes

Crear Paquetes

Los paquetes agrupan procedimientos, funciones y otros elementos relacionados.

CREATE OR REPLACE PACKAGE my_package IS
  PROCEDURE greet(p_name IN VARCHAR2);
END my_package;
/

CREATE OR REPLACE PACKAGE BODY my_package IS
  PROCEDURE greet(p_name IN VARCHAR2) IS
  BEGIN
    DBMS_OUTPUT.PUT_LINE('¡Hola, ' || p_name || '!');
  END greet;
END my_package;
/

BEGIN
  my_package.greet('Bob');
END;

Disparadores

Crear Disparadores

Los disparadores se ejecutan automáticamente en respuesta a ciertos eventos.

CREATE OR REPLACE TRIGGER trg_before_insert
BEFORE INSERT ON employees
FOR EACH ROW
BEGIN
  :NEW.created_at := SYSDATE;
END;

Colecciones PL/SQL

Arrays Asociativos

Las colecciones almacenan múltiples elementos en una sola variable.

DECLARE
  TYPE t_numbers IS TABLE OF NUMBER INDEX BY PLS_INTEGER;
  v_numbers t_numbers;
BEGIN
  v_numbers(1) := 10;
  v_numbers(2) := 20;
  DBMS_OUTPUT.PUT_LINE('Primer número: ' || v_numbers(1));
  DBMS_OUTPUT.PUT_LINE('Segundo número: ' || v_numbers(2));
END;

SQL Dinámico

Usar EXECUTE IMMEDIATE

El SQL dinámico te permite construir y ejecutar sentencias SQL en tiempo de ejecución.

DECLARE
  v_table_name VARCHAR2(30) := 'employees';
  v_sql VARCHAR2(100);
BEGIN
  v_sql := 'SELECT COUNT(*) FROM ' || v_table_name;
  EXECUTE IMMEDIATE v_sql INTO v_count;
  DBMS_OUTPUT.PUT_LINE('Número de empleados: ' || v_count);
END;

Conclusión

Prepararse para la certificación PL/SQL requiere una comprensión sólida de varios conceptos de PL/SQL y la capacidad de aplicarlos efectivamente. Al dominar las áreas clave delineadas en esta sección y practicar con los ejemplos y ejercicios proporcionados, estarás bien equipado para enfrentar el examen de certificación con confianza. Recuerda revisar cada tema a fondo y practicar escribiendo y depurando código PL/SQL para reforzar tu aprendizaje. ¡Buena suerte!
