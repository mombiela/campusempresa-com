Introducción al Manejo de Excepciones

El manejo de excepciones en PL/SQL es un aspecto crítico para escribir código robusto y libre de errores. Permite a los desarrolladores gestionar errores de manera elegante y asegurar que el programa pueda manejar situaciones inesperadas sin fallar.

Conceptos Clave

  • Excepción: Una condición de error durante la ejecución de un programa.
  • Manejo de Excepciones: El proceso de responder a la ocurrencia de excepciones.
  • Excepciones Predefinidas: Excepciones estándar proporcionadas por PL/SQL.
  • Excepciones Definidas por el Usuario: Excepciones personalizadas definidas por el programador.

Tipos de Excepciones

PL/SQL soporta dos tipos de excepciones:

  1. Excepciones Predefinidas
  2. Excepciones Definidas por el Usuario

Excepciones Predefinidas

PL/SQL proporciona varias excepciones predefinidas que son automáticamente levantadas por el motor de tiempo de ejecución. Algunas excepciones predefinidas comunes incluyen:

| Nombre de la Excepción | Descripción | |------------------------|-------------| | NO_DATA_FOUND | Se levanta cuando una declaración SELECT INTO no devuelve filas. | | TOO_MANY_ROWS | Se levanta cuando una declaración SELECT INTO devuelve más de una fila. | | ZERO_DIVIDE | Se levanta cuando se intenta dividir un número por cero. | | INVALID_CURSOR | Se levanta cuando se intenta una operación ilegal con un cursor. |

Excepciones Definidas por el Usuario

Las excepciones definidas por el usuario son excepciones personalizadas que puedes definir en tu código PL/SQL. Estas excepciones son útiles para manejar condiciones de error específicas que no están cubiertas por las excepciones predefinidas.

Declaración y Levantamiento de Excepciones

Declaración de Excepciones Definidas por el Usuario

Para declarar una excepción definida por el usuario, se utiliza la palabra clave EXCEPTION en la sección declarativa de un bloque PL/SQL.

DECLARE
  my_custom_exception EXCEPTION;
BEGIN
  -- Bloque PL/SQL
END;

Levantamiento de Excepciones

Para levantar una excepción, se utiliza la declaración RAISE.

DECLARE
  my_custom_exception EXCEPTION;
BEGIN
  -- Alguna condición a verificar
  IF some_condition THEN
    RAISE my_custom_exception;
  END IF;
EXCEPTION
  WHEN my_custom_exception THEN
    DBMS_OUTPUT.PUT_LINE('Se levantó una excepción personalizada.');
END;

Manejo de Excepciones

Estructura de Manejo de Excepciones

El manejo de excepciones en PL/SQL se realiza utilizando el bloque EXCEPTION, que sigue al bloque BEGIN.

BEGIN
  -- Bloque PL/SQL
EXCEPTION
  WHEN exception_name1 THEN
    -- Código de manejo de excepciones
  WHEN exception_name2 THEN
    -- Código de manejo de excepciones
  WHEN OTHERS THEN
    -- Manejo de todas las demás excepciones
END;

Ejemplo: Manejo de Excepciones Predefinidas

DECLARE
  v_number NUMBER;
BEGIN
  v_number := 10 / 0; -- Esto levantará la excepción ZERO_DIVIDE
EXCEPTION
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('No se puede dividir por cero.');
END;

Ejemplo: Manejo de Excepciones Definidas por el Usuario

DECLARE
  my_custom_exception EXCEPTION;
  v_number NUMBER := 10;
BEGIN
  IF v_number > 5 THEN
    RAISE my_custom_exception;
  END IF;
EXCEPTION
  WHEN my_custom_exception THEN
    DBMS_OUTPUT.PUT_LINE('Se levantó una excepción personalizada: El número es mayor que 5.');
END;

Manejo Avanzado de Excepciones

Propagación de Excepciones

Las excepciones pueden propagarse al entorno de llamada si no se manejan en el bloque actual.

DECLARE
  my_custom_exception EXCEPTION;
  PROCEDURE test_proc IS
  BEGIN
    RAISE my_custom_exception;
  END;
BEGIN
  test_proc;
EXCEPTION
  WHEN my_custom_exception THEN
    DBMS_OUTPUT.PUT_LINE('Excepción propagada desde el procedimiento.');
END;

Uso de PRAGMA EXCEPTION_INIT

La directiva PRAGMA EXCEPTION_INIT asocia una excepción definida por el usuario con un número de error de Oracle.

DECLARE
  my_custom_exception EXCEPTION;
  PRAGMA EXCEPTION_INIT(my_custom_exception, -20001);
BEGIN
  RAISE_APPLICATION_ERROR(-20001, 'Mensaje de error personalizado');
EXCEPTION
  WHEN my_custom_exception THEN
    DBMS_OUTPUT.PUT_LINE('Se capturó una excepción personalizada con el número de error -20001.');
END;

Conclusión

El manejo de excepciones en PL/SQL es esencial para crear aplicaciones robustas y confiables. Al comprender y utilizar excepciones predefinidas y definidas por el usuario, puedes gestionar errores de manera efectiva y asegurar que tus programas puedan manejar situaciones inesperadas de manera elegante. Recuerda siempre manejar excepciones donde sea apropiado y propagarlas cuando sea necesario para mantener un código limpio y mantenible.
