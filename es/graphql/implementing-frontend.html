En esta sección, exploraremos cómo implementar un frontend que interactúe con una API GraphQL. Cubriremos los conceptos básicos de la configuración de un proyecto frontend, la realización de consultas y mutaciones GraphQL, y el manejo de respuestas. Al final de esta sección, serás capaz de crear una aplicación frontend simple que se comunique con un servidor GraphQL.

Configuración del Proyecto Frontend

Para comenzar, necesitamos configurar un proyecto frontend. Usaremos React para este ejemplo, pero los conceptos pueden aplicarse a otros frameworks frontend también.

Prerrequisitos

  • Node.js y npm instalados
  • Conocimientos básicos de React

Pasos para Configurar

  1. Crear un nuevo proyecto React:

    npx create-react-app graphql-frontend
cd graphql-frontend

  2. Instalar Apollo Client:

    Apollo Client es una biblioteca popular para interactuar con APIs GraphQL en una aplicación React.

    npm install @apollo/client graphql

  3. Configurar Apollo Client:

    Crea un nuevo archivo src/apolloClient.js y configura Apollo Client.

    import { ApolloClient, InMemoryCache, ApolloProvider } from '@apollo/client';

const client = new ApolloClient({
  uri: 'http://localhost:4000/graphql', // Reemplaza con la URL de tu servidor GraphQL
  cache: new InMemoryCache()
});

export default client;

  4. Envuelve tu aplicación con ApolloProvider:

    Modifica src/index.js para incluir ApolloProvider.

    import React from 'react';
import ReactDOM from 'react-dom';
import './index.css';
import App from './App';
import client from './apolloClient';
import { ApolloProvider } from '@apollo/client';

ReactDOM.render(
  <ApolloProvider client={client}>
    <App />
  </ApolloProvider>,
  document.getElementById('root')
);

Realizando Consultas GraphQL

Ahora que hemos configurado nuestro proyecto, hagamos una consulta GraphQL para obtener datos del servidor.

Ejemplo de Consulta

  1. Define la consulta:

    Crea un nuevo archivo src/queries.js y define tu consulta GraphQL.

    import { gql } from '@apollo/client';

export const GET_USERS = gql`
  query GetUsers {
    users {
      id
      name
      email
    }
  }
`;

  2. Usa la consulta en un componente:

    Modifica src/App.js para usar la consulta.

    import React from 'react';
import { useQuery } from '@apollo/client';
import { GET_USERS } from './queries';

function App() {
  const { loading, error, data } = useQuery(GET_USERS);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error :(</p>;

  return (
    <div>
      <h1>Users</h1>
      <ul>
        {data.users.map(user => (
          <li key={user.id}>
            {user.name} - {user.email}
          </li>
        ))}
      </ul>
    </div>
  );
}

export default App;

Realizando Mutaciones GraphQL

Además de las consultas, también podemos realizar mutaciones para modificar datos en el servidor.

Ejemplo de Mutación

  1. Define la mutación:

    Agrega una mutación a src/queries.js.

    export const ADD_USER = gql`
  mutation AddUser($name: String!, $email: String!) {
    addUser(name: $name, email: $email) {
      id
      name
      email
    }
  }
`;

  2. Usa la mutación en un componente:

    Modifica src/App.js para incluir un formulario para agregar un nuevo usuario.

    import React, { useState } from 'react';
import { useQuery, useMutation } from '@apollo/client';
import { GET_USERS, ADD_USER } from './queries';

function App() {
  const { loading, error, data } = useQuery(GET_USERS);
  const [addUser] = useMutation(ADD_USER);
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');

  const handleSubmit = async (e) => {
    e.preventDefault();
    await addUser({ variables: { name, email } });
    setName('');
    setEmail('');
  };

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error :(</p>;

  return (
    <div>
      <h1>Users</h1>
      <ul>
        {data.users.map(user => (
          <li key={user.id}>
            {user.name} - {user.email}
          </li>
        ))}
      </ul>
      <h2>Add User</h2>
      <form onSubmit={handleSubmit}>
        <input
          type="text"
          placeholder="Name"
          value={name}
          onChange={(e) => setName(e.target.value)}
        />
        <input
          type="email"
          placeholder="Email"
          value={email}
          onChange={(e) => setEmail(e.target.value)}
        />
        <button type="submit">Add User</button>
      </form>
    </div>
  );
}

export default App;

Manejo de Respuestas y Errores

Manejar respuestas y errores es crucial para una aplicación robusta.

Ejemplo

  1. Estados de Carga y Error:

    Asegúrate de manejar los estados de carga y error en tus componentes.

    if (loading) return <p>Loading...</p>;
if (error) return <p>Error :(</p>;

  2. Actualizaciones Optimistas de la UI:

    Para una mejor experiencia de usuario, puedes usar actualizaciones optimistas de la UI al realizar mutaciones.

    const [addUser] = useMutation(ADD_USER, {
  optimisticResponse: {
    addUser: {
      id: -1, // ID temporal
      name,
      email,
      __typename: 'User'
    }
  },
  update: (cache, { data: { addUser } }) => {
    const { users } = cache.readQuery({ query: GET_USERS });
    cache.writeQuery({
      query: GET_USERS,
      data: { users: users.concat([addUser]) }
    });
  }
});

Conclusión

En esta sección, cubrimos los conceptos básicos de la implementación de un frontend que interactúa con una API GraphQL. Configuramos un proyecto React con Apollo Client, realizamos consultas y mutaciones GraphQL, y manejamos respuestas y errores. Con estas habilidades, puedes construir aplicaciones más complejas que aprovechen el poder de GraphQL para una obtención y manipulación de datos eficiente.
