Introducción

En GraphQL, los tipos de entrada se utilizan para definir la estructura de los datos que se pueden enviar al servidor. Son cruciales para crear mutaciones y consultas que requieren entradas complejas. Comprender los tipos de entrada es esencial para construir APIs de GraphQL robustas y flexibles.

Conceptos Clave

  • Tipos de Entrada: Tipos especiales utilizados para definir la estructura de los datos de entrada.
  • Escalares: Tipos de datos básicos como Int, Float, String, Boolean y ID.
  • Tipos de Entrada Personalizados: Tipos definidos por el usuario que se pueden usar para encapsular estructuras de entrada complejas.
  • Modificadores de No Nulo y Lista: Mejoras a los tipos de entrada para especificar campos requeridos y matrices.

Escalares

Los escalares son la forma más simple de tipos de entrada. Representan tipos de datos primitivos.

Ejemplo

type Query {
  getUser(id: ID!): User
}

type User {
  id: ID!
  name: String!
  age: Int
}

En este ejemplo, ID, String e Int son tipos escalares.

Tipos de Entrada Personalizados

Los tipos de entrada personalizados te permiten definir estructuras más complejas para tus entradas. Se definen usando la palabra clave input.

Ejemplo

input UserInput {
  name: String!
  age: Int
}

type Mutation {
  createUser(input: UserInput!): User
}

Aquí, UserInput es un tipo de entrada personalizado que encapsula los campos name y age.

Modificadores de No Nulo y Lista

Los modificadores de no nulo (!) y lista ([]) se pueden usar con tipos de entrada para imponer campos requeridos y matrices de entradas.

Ejemplo

input UserInput {
  name: String!
  age: Int
  hobbies: [String!]
}

type Mutation {
  createUser(input: UserInput!): User
}

En este ejemplo:

  • name es un campo requerido.
  • age es opcional.
  • hobbies es una lista opcional de cadenas requeridas.

Uso de Tipos de Entrada en Mutaciones

Los tipos de entrada se utilizan más comúnmente en mutaciones para pasar datos estructurados al servidor.

Ejemplo

input PostInput {
  title: String!
  content: String!
  tags: [String!]
}

type Mutation {
  createPost(input: PostInput!): Post
}

type Post {
  id: ID!
  title: String!
  content: String!
  tags: [String!]
}

Aquí, PostInput se usa para encapsular los datos necesarios para crear una nueva publicación.

Comparación de Escalares y Tipos de Entrada Personalizados

| Característica | Escalares | Tipos de Entrada Personalizados | |------------------|-------------------------|---------------------------------| | Definición | Tipos primitivos integrados | Estructuras definidas por el usuario | | Complejidad | Simple | Puede ser complejo | | Casos de Uso | Campos básicos | Entradas complejas para mutaciones y consultas |

Conclusión

Los tipos de entrada en GraphQL son esenciales para definir la estructura de los datos que se pueden enviar al servidor. Los escalares proporcionan tipos de datos básicos, mientras que los tipos de entrada personalizados permiten estructuras más complejas. Comprender cómo usar los modificadores de no nulo y lista puede ayudar a imponer la integridad de los datos y la flexibilidad en tu API de GraphQL. Al dominar los tipos de entrada, puedes crear esquemas de GraphQL más robustos y flexibles.
