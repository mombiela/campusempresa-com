Introducción a las Variables de GraphQL

Las variables de GraphQL te permiten pasar argumentos dinámicamente a tus consultas y mutaciones, haciendo que tus operaciones de GraphQL sean más flexibles y reutilizables. En lugar de codificar valores directamente en tus consultas, puedes usar variables para pasar diferentes valores en tiempo de ejecución.

Conceptos Clave

  • Variables: Marcadores de posición en tus consultas y mutaciones de GraphQL que pueden ser reemplazados con valores reales en tiempo de ejecución.
  • Definiciones de Variables: La parte de la consulta de GraphQL donde declaras las variables que planeas usar.
  • Uso de Variables: La parte de la consulta de GraphQL donde usas las variables declaradas.

Definiendo y Usando Variables

Para usar variables en una consulta o mutación de GraphQL, sigue estos pasos:

  1. Declara las variables en la consulta o mutación.
  2. Pasa las variables al ejecutar la consulta o mutación.

Ejemplo: Consulta con Variables

Paso 1: Declara las Variables

En la consulta, declara las variables con un prefijo $ y especifica sus tipos.

query GetUser($id: ID!) {
  user(id: $id) {
    id
    name
    email
  }
}
  • $id: ID! declara una variable llamada id de tipo ID que no es anulable (!).

Paso 2: Pasa las Variables

Al ejecutar la consulta, pasa las variables en un objeto variables separado.

{
  "query": "query GetUser($id: ID!) { user(id: $id) { id name email } }",
  "variables": {
    "id": "123"
  }
}

Ejemplo: Mutación con Variables

Paso 1: Declara las Variables

En la mutación, declara las variables con un prefijo $ y especifica sus tipos.

mutation CreateUser($name: String!, $email: String!) {
  createUser(name: $name, email: $email) {
    id
    name
    email
  }
}
  • $name: String! y $email: String! declaran variables llamadas name y email de tipo String que no son anulables.

Paso 2: Pasa las Variables

Al ejecutar la mutación, pasa las variables en un objeto variables separado.

{
  "query": "mutation CreateUser($name: String!, $email: String!) { createUser(name: $name, email: $email) { id name email } }",
  "variables": {
    "name": "John Doe",
    "email": "john.doe@example.com"
  }
}

Beneficios de Usar Variables

  • Reutilizabilidad: Puedes reutilizar la misma consulta o mutación con diferentes valores de variables.
  • Seguridad: Ayuda a prevenir ataques de inyección al separar la estructura de la consulta de los datos.
  • Legibilidad: Hace que las consultas y mutaciones sean más limpias y fáciles de leer.

Tipos Comunes de Variables

  • Escalares: Tipos de datos básicos como Int, Float, String, Boolean y ID.
  • Enums: Tipos escalares personalizados que representan un conjunto de valores nombrados.
  • Objetos de Entrada: Objetos complejos que pueden ser pasados como una sola variable.

Ejemplo: Objeto de Entrada

Paso 1: Define el Tipo de Objeto de Entrada

En tu esquema de GraphQL, define un tipo de objeto de entrada.

input UserInput {
  name: String!
  email: String!
}

Paso 2: Usa el Objeto de Entrada en una Mutación

Declara la variable del objeto de entrada y úsala en la mutación.

mutation CreateUser($input: UserInput!) {
  createUser(input: $input) {
    id
    name
    email
  }
}

Paso 3: Pasa el Objeto de Entrada

Al ejecutar la mutación, pasa el objeto de entrada en el objeto variables.

{
  "query": "mutation CreateUser($input: UserInput!) { createUser(input: $input) { id name email } }",
  "variables": {
    "input": {
      "name": "Jane Doe",
      "email": "jane.doe@example.com"
    }
  }
}

Conclusión

Las variables de GraphQL son una característica poderosa que mejora la flexibilidad, seguridad y legibilidad de tus operaciones de GraphQL. Al entender cómo declarar, usar y pasar variables, puedes crear consultas y mutaciones más dinámicas y reutilizables. Practica usando variables en diferentes escenarios para volverte competente en aprovechar esta característica en tus aplicaciones de GraphQL.
