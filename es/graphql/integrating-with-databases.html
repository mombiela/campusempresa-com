Integración con Bases de Datos
Microservicios con GraphQL ►

Integrar GraphQL con bases de datos es una habilidad crucial para construir aplicaciones eficientes y escalables. Este tema te guiará a través del proceso de conectar GraphQL a varios tipos de bases de datos, desde configurar la conexión hasta realizar operaciones CRUD.

Introducción a la Integración de Bases de Datos

  • Entendiendo el Rol de las Bases de Datos: Las bases de datos almacenan y gestionan datos para aplicaciones. Integrar GraphQL con bases de datos te permite consultar y manipular estos datos de manera eficiente.
  • Tipos de Bases de Datos:
    • Bases de Datos SQL: Bases de datos relacionales como PostgreSQL, MySQL.
    • Bases de Datos NoSQL: Bases de datos no relacionales como MongoDB, Redis.

Configuración del Entorno

Prerrequisitos

  • Conocimientos básicos de GraphQL.
  • Familiaridad con JavaScript (Node.js) u otro lenguaje de backend.
  • Base de datos instalada (por ejemplo, PostgreSQL, MongoDB).

Instalación de Paquetes Necesarios

Para Node.js, normalmente usarás paquetes como graphql, express, apollo-server-express y un cliente de base de datos (por ejemplo, pg para PostgreSQL, mongoose para MongoDB).

npm install graphql express apollo-server-express pg mongoose

Conectando a una Base de Datos SQL

Configuración de PostgreSQL

  1. Instalar PostgreSQL: Sigue la guía de instalación para tu sistema operativo.
  2. Crear una Base de Datos: Usa la línea de comandos de PostgreSQL o una herramienta GUI como pgAdmin.
CREATE DATABASE mydatabase;

Conectando GraphQL a PostgreSQL

  1. Crear una Conexión: Usa el paquete pg para conectar a PostgreSQL.
const { Pool } = require('pg');
const pool = new Pool({
  user: 'yourusername',
  host: 'localhost',
  database: 'mydatabase',
  password: 'yourpassword',
  port: 5432,
});
  1. Definir el Esquema y los Resolutores de GraphQL:
const { ApolloServer, gql } = require('apollo-server-express');
const express = require('express');

const typeDefs = gql`
  type Query {
    users: [User]
  }

  type User {
    id: ID!
    name: String!
    email: String!
  }
`;

const resolvers = {
  Query: {
    users: async () => {
      const res = await pool.query('SELECT * FROM users');
      return res.rows;
    },
  },
};

const server = new ApolloServer({ typeDefs, resolvers });
const app = express();
server.applyMiddleware({ app });

app.listen({ port: 4000 }, () =>
  console.log(`Server ready at http://localhost:4000${server.graphqlPath}`)
);

Conectando a una Base de Datos NoSQL

Configuración de MongoDB

  1. Instalar MongoDB: Sigue la guía de instalación para tu sistema operativo.
  2. Crear una Base de Datos: Usa la shell de MongoDB o una herramienta GUI como MongoDB Compass.

Conectando GraphQL a MongoDB

  1. Crear una Conexión: Usa el paquete mongoose para conectar a MongoDB.
const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/mydatabase', {
  useNewUrlParser: true,
  useUnifiedTopology: true,
});
  1. Definir el Esquema y los Resolutores de GraphQL:
const { ApolloServer, gql } = require('apollo-server-express');
const express = require('express');

const UserSchema = new mongoose.Schema({
  name: String,
  email: String,
});

const User = mongoose.model('User', UserSchema);

const typeDefs = gql`
  type Query {
    users: [User]
  }

  type User {
    id: ID!
    name: String!
    email: String!
  }
`;

const resolvers = {
  Query: {
    users: async () => {
      return await User.find();
    },
  },
};

const server = new ApolloServer({ typeDefs, resolvers });
const app = express();
server.applyMiddleware({ app });

app.listen({ port: 4000 }, () =>
  console.log(`Server ready at http://localhost:4000${server.graphqlPath}`)
);

Realizando Operaciones CRUD

Crear (Insertar)

  • Ejemplo SQL:
const resolvers = {
  Mutation: {
    addUser: async (_, { name, email }) => {
      const res = await pool.query(
        'INSERT INTO users (name, email) VALUES ($1, $2) RETURNING *',
        [name, email]
      );
      return res.rows[0];
    },
  },
};
  • Ejemplo NoSQL:
const resolvers = {
  Mutation: {
    addUser: async (_, { name, email }) => {
      const user = new User({ name, email });
      await user.save();
      return user;
    },
  },
};

Leer (Seleccionar)

  • Ejemplo SQL:
const resolvers = {
  Query: {
    users: async () => {
      const res = await pool.query('SELECT * FROM users');
      return res.rows;
    },
  },
};
  • Ejemplo NoSQL:
const resolvers = {
  Query: {
    users: async () => {
      return await User.find();
    },
  },
};

Actualizar

  • Ejemplo SQL:
const resolvers = {
  Mutation: {
    updateUser: async (_, { id, name, email }) => {
      const res = await pool.query(
        'UPDATE users SET name = $1, email = $2 WHERE id = $3 RETURNING *',
        [name, email, id]
      );
      return res.rows[0];
    },
  },
};
  • Ejemplo NoSQL:
const resolvers = {
  Mutation: {
    updateUser: async (_, { id, name, email }) => {
      return await User.findByIdAndUpdate(id, { name, email }, { new: true });
    },
  },
};

Eliminar

  • Ejemplo SQL:
const resolvers = {
  Mutation: {
    deleteUser: async (_, { id }) => {
      const res = await pool.query('DELETE FROM users WHERE id = $1 RETURNING *', [id]);
      return res.rows[0];
    },
  },
};
  • Ejemplo NoSQL:
const resolvers = {
  Mutation: {
    deleteUser: async (_, { id }) => {
      return await User.findByIdAndDelete(id);
    },
  },
};

Conclusión

Integrar GraphQL con bases de datos te permite aprovechar el poder de las consultas y mutaciones de GraphQL para interactuar con tus datos de manera eficiente. Ya sea que estés usando bases de datos SQL o NoSQL, los principios siguen siendo los mismos: establecer una conexión, definir tu esquema y resolutores, y realizar operaciones CRUD. Al dominar estas habilidades, puedes construir aplicaciones robustas y escalables que puedan manejar interacciones de datos complejas.
