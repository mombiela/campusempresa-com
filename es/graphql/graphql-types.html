Introducción a los Tipos de GraphQL

GraphQL es un lenguaje de consulta para APIs que permite a los clientes solicitar exactamente los datos que necesitan. Uno de los conceptos centrales en GraphQL es su sistema de tipos, que define la estructura de los datos que se pueden consultar. Entender los tipos de GraphQL es esencial tanto para definir tu esquema como para escribir consultas efectivas.

Tipos Básicos

GraphQL viene con un conjunto de tipos escalares básicos que representan los tipos de datos más comunes.

  • Int: Un entero de 32 bits con signo.
  • Float: Un valor de punto flotante de doble precisión con signo.
  • String: Una secuencia de caracteres UTF-8.
  • Boolean: Un valor verdadero o falso.
  • ID: Un identificador único, a menudo utilizado para volver a buscar un objeto o como clave para una caché.

Ejemplo

type Query {
  id: ID
  name: String
  age: Int
  isActive: Boolean
  score: Float
}

Tipos de Objeto

Los tipos de objeto son el tipo más común en un esquema de GraphQL. Representan una colección de campos, cada uno de los cuales puede ser otro tipo.

Ejemplo

type User {
  id: ID!
  name: String!
  age: Int
  email: String
}

En este ejemplo:

  • id y name son campos no anulables (denotados por !).
  • age y email son campos anulables.

Tipos de Consulta y Mutación

En GraphQL, el esquema define dos tipos especiales: Query y Mutation.

  • Query: Define los puntos de entrada para las operaciones de lectura.
  • Mutation: Define los puntos de entrada para las operaciones de escritura.

Ejemplo

type Query {
  user(id: ID!): User
  users: [User]
}

type Mutation {
  createUser(name: String!, email: String!): User
}

En este ejemplo:

  • El tipo Query tiene dos campos: user y users.
  • El tipo Mutation tiene un campo: createUser.

Tipos Enum

Los tipos enum son un tipo especial de escalar que está restringido a un conjunto particular de valores permitidos.

Ejemplo

enum Role {
  ADMIN
  USER
  GUEST
}

type User {
  id: ID!
  name: String!
  role: Role!
}

En este ejemplo:

  • El enum Role puede tener uno de tres valores: ADMIN, USER o GUEST.

Tipos de Lista y No Nulo

GraphQL también soporta tipos de lista y no nulo para definir estructuras más complejas.

  • Lista: Representa un array de un cierto tipo.
  • No Nulo: Asegura que un campo no puede ser nulo.

Ejemplo

type Query {
  users: [User!]!
}

En este ejemplo:

  • users es una lista no anulable de objetos User no anulables.

Tipos de Entrada

Los tipos de entrada se utilizan como argumentos en consultas y mutaciones. Permiten pasar objetos complejos como argumentos.

Ejemplo

input UserInput {
  name: String!
  email: String!
  age: Int
}

type Mutation {
  createUser(input: UserInput!): User
}

En este ejemplo:

  • UserInput es un tipo de entrada utilizado en la mutación createUser.

Tipos de Interfaz

Los tipos de interfaz son tipos abstractos que definen un conjunto de campos que múltiples tipos deben incluir.

Ejemplo

interface Character {
  id: ID!
  name: String!
}

type Human implements Character {
  id: ID!
  name: String!
  homePlanet: String
}

type Droid implements Character {
  id: ID!
  name: String!
  primaryFunction: String
}

type Query {
  characters: [Character]
}

En este ejemplo:

  • Character es una interfaz implementada por Human y Droid.

Tipos de Unión

Los tipos de unión son similares a las interfaces pero no requieren que los tipos compartan ningún campo.

Ejemplo

union SearchResult = Human | Droid

type Query {
  search(text: String!): [SearchResult]
}

En este ejemplo:

  • SearchResult es un tipo de unión que puede ser Human o Droid.

Conclusión

Entender los tipos de GraphQL es fundamental para diseñar un esquema de GraphQL robusto y eficiente. Al aprovechar los diversos tipos disponibles, puedes crear un esquema que represente con precisión tus datos y soporte consultas y mutaciones complejas. A medida que progreses, descubrirás que dominar estos tipos te permitirá construir APIs más poderosas y flexibles.
