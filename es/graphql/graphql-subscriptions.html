Introducción a las Suscripciones de GraphQL

Las Suscripciones de GraphQL son una característica poderosa que permite a los clientes recibir actualizaciones en tiempo real desde el servidor. A diferencia de las consultas y mutaciones, que son operaciones de una sola vez, las suscripciones mantienen una conexión activa y envían actualizaciones al cliente cada vez que ocurren eventos específicos en el servidor.

Conceptos Clave

  • Comunicación en Tiempo Real: Las suscripciones permiten la comunicación en tiempo real entre el cliente y el servidor.
  • WebSockets: Las suscripciones generalmente usan WebSockets para mantener una conexión persistente.
  • Mecanismo Pub/Sub: Las suscripciones a menudo dependen de un mecanismo de publicación/suscripción para transmitir actualizaciones a los clientes suscritos.

Configuración de Suscripciones de GraphQL

Para usar Suscripciones de GraphQL, necesitas configurar un servidor WebSocket y configurar tu servidor GraphQL para manejar operaciones de suscripción.

Configuración del Lado del Servidor

  1. Instalar Dependencias:

    • graphql-subscriptions: Proporciona una implementación simple de pub/sub.
    • subscriptions-transport-ws: Transporte WebSocket para suscripciones de GraphQL.
    npm install graphql-subscriptions subscriptions-transport-ws

  2. Configurar el Servidor WebSocket:

    • Configura un servidor WebSocket para manejar solicitudes de suscripción.
    const { createServer } = require('http');
const { execute, subscribe } = require('graphql');
const { SubscriptionServer } = require('subscriptions-transport-ws');
const { makeExecutableSchema } = require('@graphql-tools/schema');

const typeDefs = `
  type Query {
    _: Boolean
  }
  type Subscription {
    messageSent: String
  }
`;

const resolvers = {
  Subscription: {
    messageSent: {
      subscribe: (_, __, { pubsub }) => pubsub.asyncIterator('MESSAGE_SENT')
    }
  }
};

const schema = makeExecutableSchema({ typeDefs, resolvers });

const server = createServer((req, res) => {
  res.writeHead(404);
  res.end();
});

server.listen(4000, () => {
  new SubscriptionServer({
    execute,
    subscribe,
    schema
  }, {
    server,
    path: '/subscriptions',
  });
  console.log('El Servidor WebSocket está funcionando en ws://localhost:4000/subscriptions');
});

  3. Publicar Eventos:

    • Usa un mecanismo de pub/sub para publicar eventos a los que los clientes pueden suscribirse.
    const { PubSub } = require('graphql-subscriptions');
const pubsub = new PubSub();

// Ejemplo de publicación de un evento
setInterval(() => {
  pubsub.publish('MESSAGE_SENT', { messageSent: '¡Hola Mundo!' });
}, 1000);

Configuración del Lado del Cliente

  1. Instalar Dependencias:

    • subscriptions-transport-ws: Transporte WebSocket para suscripciones de GraphQL.
    • apollo-client y apollo-link-ws: Apollo Client y enlace WebSocket para manejar suscripciones.
    npm install subscriptions-transport-ws @apollo/client

  2. Configurar Apollo Client:

    • Configura Apollo Client para usar el enlace WebSocket para suscripciones.
    import { ApolloClient, InMemoryCache, split } from '@apollo/client';
import { WebSocketLink } from '@apollo/client/link/ws';
import { getMainDefinition } from '@apollo/client/utilities';

const wsLink = new WebSocketLink({
  uri: 'ws://localhost:4000/subscriptions',
  options: {
    reconnect: true
  }
});

const client = new ApolloClient({
  link: wsLink,
  cache: new InMemoryCache()
});

  3. Suscribirse a Eventos:

    • Usa Apollo Client para suscribirse a eventos.
    import { gql, useSubscription } from '@apollo/client';

const MESSAGE_SENT_SUBSCRIPTION = gql`
  subscription OnMessageSent {
    messageSent
  }
`;

function Messages() {
  const { data, loading } = useSubscription(MESSAGE_SENT_SUBSCRIPTION);

  if (loading) return <p>Cargando...</p>;
  return <p>Nuevo mensaje: {data.messageSent}</p>;
}

Conclusión

Las Suscripciones de GraphQL proporcionan una forma robusta de implementar características en tiempo real en tus aplicaciones. Al aprovechar WebSockets y un mecanismo de pub/sub, puedes enviar actualizaciones a los clientes tan pronto como ocurran. Configurar suscripciones implica configurar tanto el servidor como el cliente para manejar conexiones WebSocket y gestionar eventos de suscripción. Con este conocimiento, ahora puedes implementar funcionalidad en tiempo real en tus aplicaciones GraphQL, mejorando la experiencia del usuario con actualizaciones en vivo.
