Redis Sentinel es un sistema diseñado para ayudar a gestionar instancias de Redis, proporcionando alta disponibilidad y capacidades de monitoreo. Este tema cubrirá los conceptos básicos de Sentinel, su configuración y cómo usarlo para asegurar que tus instancias de Redis sean resilientes y altamente disponibles.

Introducción a Redis Sentinel

Redis Sentinel proporciona varias características clave:

  • Monitoreo: Sentinel verifica constantemente si tus instancias maestro y réplica están funcionando como se espera.
  • Notificación: Sentinel puede notificar a los administradores del sistema u otros sistemas a través de una API cuando algo está mal con una de las instancias de Redis monitoreadas.
  • Failover Automático: Si un maestro no está funcionando como se espera, Sentinel puede iniciar un proceso de failover donde una réplica es promovida a maestro.
  • Proveedor de Configuración: Sentinel actúa como una fuente de autoridad para que los clientes descubran el maestro y las réplicas actuales.

Configuración de Redis Sentinel

Para configurar Redis Sentinel, necesitas configurar instancias de Sentinel y asegurarte de que puedan comunicarse entre sí y con las instancias de Redis que están monitoreando.

Archivo de Configuración de Sentinel

Un archivo típico de configuración de Sentinel (sentinel.conf) incluye los siguientes ajustes clave:

# Ejemplo de sentinel.conf

# Nombre del maestro monitoreado
sentinel monitor mymaster 127.0.0.1 6379 2

# Autenticación para el maestro
sentinel auth-pass mymaster your_password

# Con qué frecuencia hacer ping al maestro y a las réplicas
sentinel down-after-milliseconds mymaster 5000

# Cuánto tiempo esperar antes de hacer failover
sentinel failover-timeout mymaster 60000

# Número de réplicas a reconfigurar después de un failover
sentinel parallel-syncs mymaster 1
  • sentinel monitor <master-name> <ip> <port> <quorum>: Define el maestro a monitorear. El quórum es el número de Sentinels que necesitan estar de acuerdo antes de iniciar un failover.
  • sentinel auth-pass <master-name> <password>: Establece la contraseña para autenticarse con el maestro.
  • sentinel down-after-milliseconds <master-name> <milliseconds>: Tiempo de espera antes de considerar que el maestro está caído.
  • sentinel failover-timeout <master-name> <milliseconds>: Tiempo de espera antes de iniciar un nuevo failover.
  • sentinel parallel-syncs <master-name> <number>: Número de réplicas a reconfigurar en paralelo durante un failover.

Ejecutando Sentinel

Para iniciar una instancia de Sentinel, usa el siguiente comando:

redis-server /path/to/sentinel.conf --sentinel

Monitoreo y Notificación

Sentinel monitorea continuamente la salud de las instancias de Redis. Si una instancia maestra se considera caída, Sentinel notificará a los administradores y a otros sistemas configurados para recibir notificaciones.

Ejemplo de Monitoreo

# Salida del log de Sentinel
[12345] 01 Jan 01:00:00.000 * +monitor master mymaster 127.0.0.1 6379 quorum 2
[12345] 01 Jan 01:00:05.000 * +sdown master mymaster 127.0.0.1 6379
[12345] 01 Jan 01:00:10.000 * +odown master mymaster 127.0.0.1 6379 #quorum 2
[12345] 01 Jan 01:00:15.000 * +new-epoch 1
[12345] 01 Jan 01:00:20.000 * +try-failover master mymaster 127.0.0.1 6379

Failover Automático

Cuando un maestro se considera caído, Sentinel iniciará un proceso de failover. Esto implica promover una de las réplicas a maestro y reconfigurar las otras réplicas para replicar desde el nuevo maestro.

Ejemplo de Failover

# Salida del log de Sentinel durante el failover
[12345] 01 Jan 01:00:20.000 * +failover-state-select-slave master mymaster 127.0.0.1 6379
[12345] 01 Jan 01:00:25.000 * +selected-slave slave 127.0.0.1:6380
[12345] 01 Jan 01:00:30.000 * +failover-state-send-slaveof-noone slave 127.0.0.1:6380
[12345] 01 Jan 01:00:35.000 * +failover-end master mymaster 127.0.0.1 6379
[12345] 01 Jan 01:00:40.000 * +switch-master mymaster 127.0.0.1 6379 127.0.0.1 6380

Proveedor de Configuración

Los clientes pueden usar Sentinel para descubrir el maestro y las réplicas actuales. Esto es particularmente útil para aplicaciones que necesitan conectarse al maestro para operaciones de escritura y a las réplicas para operaciones de lectura.

Ejemplo de Uso de Sentinel para Descubrir el Maestro

import redis
from redis.sentinel import Sentinel

sentinel = Sentinel([('localhost', 26379)], socket_timeout=0.1)
master = sentinel.master_for('mymaster', socket_timeout=0.1)
slave = sentinel.slave_for('mymaster', socket_timeout=0.1)

# Escribir en el maestro
master.set('foo', 'bar')

# Leer de la réplica
print(slave.get('foo'))

Conclusión

Redis Sentinel es una herramienta poderosa para asegurar la alta disponibilidad y el monitoreo de instancias de Redis. Al entender su configuración y capacidades, puedes configurar un entorno de Redis resiliente que pueda manejar fallos de manera elegante. La capacidad de Sentinel para monitorear, notificar y realizar failover automático lo convierte en un componente esencial para cualquier despliegue de Redis en producción.
