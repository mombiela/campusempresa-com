Introducción a las Cadenas de Redis

Las cadenas de Redis son el tipo de valor más básico que puedes almacenar en Redis. Son seguras para datos binarios, lo que significa que pueden contener cualquier tipo de dato, incluyendo texto, números u objetos serializados.

  • Seguras para Datos Binarios: Las cadenas pueden contener cualquier dato, como texto, números o incluso datos binarios como imágenes u objetos serializados.
  • Longitud Máxima: Una cadena de Redis puede tener hasta 512 MB de longitud.

Operaciones Básicas con Cadenas

Redis proporciona una variedad de comandos para trabajar con cadenas. Aquí están algunos de los comandos más comúnmente usados:

Establecer y Obtener Cadenas

  • SET: Establece el valor de una clave.
  • GET: Obtiene el valor de una clave.
SET mykey "¡Hola, Redis!"
GET mykey

Explicación:

  • SET mykey "¡Hola, Redis!" almacena la cadena "¡Hola, Redis!" en la clave mykey.
  • GET mykey recupera el valor almacenado en mykey.

Modificar Cadenas

  • APPEND: Añade un valor a una clave.
  • STRLEN: Devuelve la longitud de la cadena almacenada en una clave.
APPEND mykey " ¿Cómo estás?"
GET mykey
STRLEN mykey

Explicación:

  • APPEND mykey " ¿Cómo estás?" añade " ¿Cómo estás?" al valor existente de mykey.
  • GET mykey recupera el valor actualizado.
  • STRLEN mykey devuelve la longitud de la cadena almacenada en mykey.

Incrementar y Decrementar

  • INCR: Incrementa el valor entero de una clave en uno.
  • DECR: Decrementa el valor entero de una clave en uno.
  • INCRBY: Incrementa el valor entero de una clave en una cantidad especificada.
  • DECRBY: Decrementa el valor entero de una clave en una cantidad especificada.
SET counter 10
INCR counter
GET counter
INCRBY counter 5
GET counter
DECR counter
GET counter
DECRBY counter 3
GET counter

Explicación:

  • SET counter 10 inicializa la clave counter con el valor 10.
  • INCR counter incrementa el valor de counter en 1.
  • INCRBY counter 5 incrementa el valor de counter en 5.
  • DECR counter decrementa el valor de counter en 1.
  • DECRBY counter 3 decrementa el valor de counter en 3.

Operaciones Avanzadas con Cadenas

Redis también soporta operaciones más avanzadas con cadenas, que pueden ser útiles en varios escenarios.

Establecer con Expiración

  • SETEX: Establece el valor de una clave con un tiempo de expiración.
  • PSETEX: Establece el valor de una clave con un tiempo de expiración en milisegundos.
SETEX tempkey 10 "Esto expirará en 10 segundos"
GET tempkey

Explicación:

  • SETEX tempkey 10 "Esto expirará en 10 segundos" establece el valor de tempkey y hace que expire después de 10 segundos.

Establecimiento Condicional

  • SETNX: Establece el valor de una clave solo si la clave no existe.
SETNX newkey "Esto solo se establecerá si newkey no existe"
GET newkey

Explicación:

  • SETNX newkey "Esto solo se establecerá si newkey no existe" establece el valor de newkey solo si newkey no existe.

Obtener y Establecer en un Solo Comando

  • GETSET: Establece el valor de una clave y devuelve su valor antiguo.
SET mykey "Valor inicial"
GETSET mykey "Nuevo valor"
GET mykey

Explicación:

  • GETSET mykey "Nuevo valor" establece mykey a "Nuevo valor" y devuelve el valor antiguo "Valor inicial".

Resumen

Las cadenas de Redis son versátiles y pueden almacenar una amplia gama de tipos de datos. Soportan varias operaciones, desde las básicas de establecer y obtener hasta características más avanzadas como el establecimiento condicional y la expiración. Entender estas operaciones es crucial para usar Redis efectivamente en tus aplicaciones.

Al dominar estos comandos, puedes gestionar y manipular eficientemente los datos de cadenas en Redis, convirtiéndolo en una herramienta poderosa para tus necesidades de desarrollo.
