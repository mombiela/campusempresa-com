Entendiendo las Pipelines de Jenkins ►

En esta sección, cubriremos cómo crear y configurar trabajos en Jenkins. Los trabajos son los bloques fundamentales en Jenkins, permitiéndote definir tareas a ejecutar, como construir software, ejecutar pruebas o desplegar aplicaciones. Al final de esta sección, podrás crear y configurar trabajos para automatizar tu flujo de trabajo de desarrollo.

Creación de un Nuevo Trabajo

Crear un nuevo trabajo en Jenkins es sencillo. Sigue estos pasos para crear tu primer trabajo:

  1. Abrir el Panel de Jenkins: Navega a tu instancia de Jenkins e inicia sesión.
  2. Crear un Nuevo Elemento: Haz clic en el enlace "New Item" en el lado izquierdo del panel.
  3. Ingresar Nombre del Trabajo: Proporciona un nombre único para tu trabajo.
  4. Seleccionar Tipo de Trabajo: Elige el tipo de trabajo que deseas crear. Para principiantes, "Freestyle project" es un buen punto de partida.
  5. Hacer Clic en OK: Haz clic en el botón "OK" para proceder a la página de configuración del trabajo.

Configuración de un Trabajo

Una vez que hayas creado un trabajo, necesitas configurarlo para definir lo que debe hacer. Aquí están las secciones clave que encontrarás en la página de configuración del trabajo:

General

  • Descripción: Proporciona una descripción significativa de lo que hace el trabajo.
  • Descartar Construcciones Antiguas: Configura Jenkins para descartar construcciones antiguas y ahorrar espacio en disco.
  • Proyecto GitHub: Si tu proyecto está alojado en GitHub, puedes enlazarlo aquí.

Gestión de Código Fuente

  • Ninguno: Selecciona esto si tu trabajo no involucra código fuente.
  • Git: Configura los detalles del repositorio Git si tu trabajo involucra código fuente de un repositorio Git. 
    Repository URL: https://github.com/your-repo.git
Credentials: (Agregar credenciales si es necesario)
Branches to build: */main

Disparadores de Construcción

  • Construir Periódicamente: Programa el trabajo para que se ejecute a intervalos específicos usando sintaxis cron. 
    H/15 * * * *
  • Sondear SCM: Configura Jenkins para sondear el repositorio de código fuente en busca de cambios. 
    H/5 * * * *

Entorno de Construcción

  • Eliminar espacio de trabajo antes de que comience la construcción: Asegura un espacio de trabajo limpio para cada construcción.
  • Usar texto(s) o archivo(s) secreto(s): Gestiona información sensible de manera segura.

Pasos de Construcción

  • Ejecutar Shell: Ejecuta comandos shell como parte del proceso de construcción. 
    echo "Construyendo el proyecto..."
./build.sh
  • Invocar Script de Gradle: Si estás usando Gradle, puedes invocar tareas de Gradle. 
    Tasks: clean build

Acciones Post-construcción

  • Archivar los artefactos: Guarda los artefactos de construcción para referencia futura. 
    Files to archive: **/target/*.jar
  • Publicar informe de resultados de pruebas JUnit: Publica los resultados de las pruebas si tu construcción incluye pruebas. 
    Test report XMLs: **/target/test-*.xml

Ejemplo: Creación de un Trabajo Simple de Freestyle

Vamos a crear un trabajo simple de Freestyle que imprima "Hello, Jenkins!" en la consola.

  1. Crear un Nuevo Elemento: Nómbralo "HelloJenkins" y selecciona "Freestyle project".
  2. General: Añade una descripción como "Un trabajo simple para imprimir Hello, Jenkins!".
  3. Pasos de Construcción: Añade un paso de construcción "Ejecutar Shell" con el siguiente comando: 
    echo "Hello, Jenkins!"
  4. Guardar: Haz clic en "Save" para crear el trabajo.

Ejecutar el Trabajo

  • Construir Ahora: Ve a la página del trabajo y haz clic en "Build Now".
  • Salida de Consola: Haz clic en el número de construcción en el "Build History" y luego en "Console Output" para ver el resultado.

Resumen

En esta sección, cubrimos los conceptos básicos de la creación y configuración de trabajos en Jenkins. Aprendimos cómo crear un nuevo trabajo, configurar varios aspectos del trabajo y ejecutar un trabajo simple de Freestyle. Entender cómo crear y configurar trabajos es esencial para automatizar tu flujo de trabajo de desarrollo con Jenkins. A medida que progreses, puedes explorar tipos de trabajos y configuraciones más avanzadas para satisfacer las necesidades de tu proyecto.
