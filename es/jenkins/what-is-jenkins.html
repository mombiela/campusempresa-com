Introducción

Jenkins es un servidor de automatización de código abierto que permite a los desarrolladores construir, probar y desplegar su software de manera confiable. Es una herramienta crucial en el ciclo de vida de DevOps, facilitando la integración continua (CI) y la entrega continua (CD). Jenkins es altamente extensible, con una vasta biblioteca de plugins que soportan varias etapas del ciclo de vida del desarrollo de software.

Conceptos Clave

  • Integración Continua (CI): La práctica de fusionar todas las copias de trabajo de los desarrolladores en una línea principal compartida varias veces al día.
  • Entrega Continua (CD): La práctica de asegurar que el software pueda ser lanzado de manera confiable en cualquier momento.
  • Pipeline: Un conjunto de plugins que soportan la implementación e integración de pipelines de entrega continua en Jenkins.
  • Plugins: Extensiones que añaden funcionalidad a Jenkins, permitiéndole integrarse con una amplia variedad de herramientas y plataformas.
  • Arquitectura Maestro-Esclavo: Jenkins puede distribuir cargas de trabajo a través de múltiples máquinas, mejorando el rendimiento y la fiabilidad.

Características de Jenkins

  • Extensibilidad: Jenkins soporta más de 1,500 plugins que se integran con varias herramientas en el ciclo de vida del desarrollo de software.
  • Construcciones Distribuidas: Jenkins puede distribuir tareas de construcción a través de múltiples máquinas, mejorando el rendimiento y la fiabilidad.
  • Fácil Configuración: Jenkins proporciona una interfaz web simple para configuración y gestión.
  • Soporte para Varios Sistemas de Control de Versiones: Jenkins se integra con Git, SVN, Mercurial y muchos otros sistemas de control de versiones.
  • Pruebas Automatizadas: Jenkins puede ejecutar pruebas automáticamente y reportar resultados, asegurando la calidad del código.

Instalación de Jenkins

Requisitos Previos

  • Kit de Desarrollo de Java (JDK) instalado en tu máquina.
  • Privilegios administrativos para instalar software.

Pasos de Instalación

En Windows

  1. Descarga el instalador de Jenkins para Windows desde el sitio oficial de Jenkins.
  2. Ejecuta el instalador y sigue las instrucciones en pantalla.
  3. Una vez instalado, Jenkins se iniciará automáticamente y se puede acceder a él vía http://localhost:8080.

En Linux

  1. Añade el repositorio de Jenkins a tu sistema: 
    wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
  2. Actualiza tu índice de paquetes: 
    sudo apt-get update
  3. Instala Jenkins: 
    sudo apt-get install jenkins
  4. Inicia Jenkins: 
    sudo systemctl start jenkins
  5. Accede a Jenkins vía http://localhost:8080.

Configuración Básica

Configuración Inicial

  1. Abre un navegador web y navega a http://localhost:8080.
  2. Se te pedirá desbloquear Jenkins usando una contraseña de administrador inicial, que se puede encontrar en el directorio secrets de tu instalación de Jenkins.
  3. Sigue las instrucciones en pantalla para completar la configuración, incluyendo la instalación de plugins recomendados y la creación de un usuario administrador.

Creando Tu Primer Trabajo

  1. Desde el panel de control de Jenkins, haz clic en "Nuevo Ítem".
  2. Ingresa un nombre para tu trabajo y selecciona "Proyecto de estilo libre".
  3. Configura el trabajo especificando el repositorio de código fuente, los disparadores de construcción y los pasos de construcción.
  4. Guarda el trabajo y haz clic en "Construir Ahora" para ejecutarlo.

Ejemplo: Trabajo de Construcción Simple

Aquí hay un ejemplo de un trabajo simple de Jenkins que extrae código de un repositorio Git y ejecuta un script de shell.

  1. Crear un Nuevo Trabajo:

    • Nombre: Trabajo-Construcción-Simple
    • Tipo: Proyecto de estilo libre

  2. Configurar Gestión de Código Fuente:

    • URL del Repositorio: https://github.com/your-repo/sample-project.git

  3. Añadir Paso de Construcción:

    • Ejecutar Shell:
    echo "Construyendo el proyecto..."
./build.sh

  4. Guardar y Construir:

    • Haz clic en "Guardar" y luego en "Construir Ahora".

Resumen

Jenkins es un servidor de automatización poderoso y flexible que soporta la integración continua y la entrega continua. Es altamente extensible a través de plugins y puede ser configurado para automatizar varias tareas en el ciclo de vida del desarrollo de software. Al entender los conceptos básicos de Jenkins, incluyendo su instalación, configuración y creación de trabajos, puedes comenzar a aprovechar sus capacidades para optimizar tus procesos de desarrollo.
