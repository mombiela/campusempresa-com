Introducción

Integrar Jenkins con Git es un paso crucial en la configuración de una pipeline de integración continua/despliegue continuo (CI/CD). Esta sección te guiará a través del proceso de conectar Jenkins con un repositorio Git, permitiendo compilaciones y despliegues automatizados.

Prerrequisitos

Antes de comenzar, asegúrate de tener lo siguiente:

  • Jenkins instalado y en funcionamiento.
  • Un repositorio Git (GitHub, GitLab, Bitbucket, etc.).
  • Conocimiento básico de Jenkins y Git.

Configuración de Jenkins con Git

Instalación del Plugin de Git en Jenkins

Para integrar Jenkins con Git, necesitas instalar el plugin de Git.

  1. Abre Jenkins y navega a Manage Jenkins.
  2. Haz clic en Manage Plugins.
  3. Ve a la pestaña Available y busca Git Plugin.
  4. Selecciona el plugin y haz clic en Install without restart.

Configuración de Git en Jenkins

Después de instalar el plugin de Git, necesitas configurar Jenkins para usar Git.

  1. Ve a Manage Jenkins > Global Tool Configuration.
  2. Desplázate hacia abajo hasta Git y haz clic en Add Git.
  3. Proporciona un nombre (por ejemplo, Default Git) y especifica la ruta al ejecutable de Git si es necesario.
  4. Haz clic en Save.

Creación de un Trabajo de Jenkins para un Proyecto Git

Ahora, vamos a crear un trabajo de Jenkins que extraiga código de un repositorio Git y lo compile.

  1. Desde el panel de Jenkins, haz clic en New Item.
  2. Ingresa un nombre para tu trabajo y selecciona Freestyle project.
  3. Haz clic en OK.

Configuración de la Gestión del Código Fuente

  1. En la página de configuración del trabajo, desplázate hacia abajo hasta Source Code Management.
  2. Selecciona Git.
  3. Ingresa la URL del repositorio (por ejemplo, https://github.com/username/repository.git).
  4. Si el repositorio es privado, proporciona las credenciales necesarias haciendo clic en Add e ingresando tus credenciales de Git.
  5. Especifica la rama a compilar (por ejemplo, main).

Agregar Disparadores de Compilación

Para automatizar el proceso de compilación, puedes configurar disparadores de compilación.

  1. Desplázate hacia abajo hasta Build Triggers.
  2. Selecciona Poll SCM e ingresa un horario (por ejemplo, H/5 * * * * para sondear cada 5 minutos).
  3. Alternativamente, puedes usar GitHub hook trigger for GITScm polling si has configurado webhooks en tu repositorio de GitHub.

Definir Pasos de Compilación

  1. Desplázate hacia abajo hasta Build.
  2. Haz clic en Add build step y selecciona Execute shell.
  3. Ingresa los comandos de compilación (por ejemplo, ./build.sh).

Acciones Post-compilación

  1. Desplázate hacia abajo hasta Post-build Actions.
  2. Agrega cualquier paso post-compilación necesario, como archivar artefactos o enviar notificaciones.

Ejemplo de Configuración

Aquí tienes un ejemplo de un script shell simple que podría usarse en el paso de compilación:

#!/bin/bash
echo "Building the project..."
# Comando de compilación de ejemplo
make
echo "Build completed successfully."

Conclusión

Integrar Jenkins con Git es un paso fundamental para automatizar tu flujo de trabajo de desarrollo. Siguiendo los pasos descritos anteriormente, puedes configurar Jenkins para extraer código de un repositorio Git, compilarlo y desencadenar compilaciones automáticamente basadas en cambios en el repositorio. Esta integración no solo ahorra tiempo, sino que también asegura que tu código esté siempre en un estado desplegable.

En las próximas secciones, exploraremos temas más avanzados como la configuración de pipelines, el manejo de procesos de compilación complejos y la integración con otras herramientas y servicios.
