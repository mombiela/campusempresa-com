Introducción

Configurar la seguridad en Jenkins es crucial para proteger tus pipelines de CI/CD, datos sensibles e infraestructura. Esta sección te guiará a través de los conceptos básicos de la seguridad en Jenkins, desde la configuración de la autenticación de usuarios hasta configuraciones de seguridad avanzadas.

Autenticación de Usuarios

La autenticación de usuarios es la primera línea de defensa para asegurar tu instancia de Jenkins. Garantiza que solo los usuarios autorizados puedan acceder y realizar acciones en Jenkins.

Configuración de la Seguridad en Jenkins

  1. Habilitar Seguridad:

    • Navega a Manage Jenkins > Configure Global Security.
    • Marca la opción Enable security.

  2. Elegir el Reino de Seguridad:

    • Base de datos de usuarios propia de Jenkins: Simple y adecuado para equipos pequeños.
    • LDAP: Integrar con el servidor LDAP de tu organización.
    • Active Directory: Usar Active Directory para la autenticación.
    • SAML: Usar SAML para Single Sign-On (SSO).

  3. Configurar Autorización:

    • Seguridad basada en matriz: Control granular sobre los permisos.
    • Estrategia de Autorización basada en Matriz de Proyectos: Controlar permisos a nivel de proyecto.
    • Estrategia basada en roles: Definir roles y asignar permisos a roles.

Ejemplo: Configuración de la Base de Datos de Usuarios Propia de Jenkins

# Navega a Manage Jenkins > Configure Global Security
security:
  enable: true
  realm: jenkins
  authorization: matrix
  users:
    - username: admin
      password: admin_password
      roles: [admin]
    - username: developer
      password: dev_password
      roles: [developer]

Asegurando Jenkins con SSL

Asegurar Jenkins con SSL garantiza que los datos transmitidos entre el servidor Jenkins y los usuarios estén encriptados.

Pasos para Habilitar SSL

  1. Generar un Keystore:

    keytool -genkey -alias jenkins -keyalg RSA -keystore jenkins.keystore

  2. Configurar Jenkins para Usar SSL:

    • Edita el archivo de configuración de Jenkins (por ejemplo, jenkins.xml o jenkins.service).
    • Añade los siguientes argumentos: 
      <arguments>--httpPort=-1 --httpsPort=8443 --httpsKeyStore=/path/to/jenkins.keystore --httpsKeyStorePassword=your_password</arguments>

  3. Reiniciar Jenkins:

    sudo systemctl restart jenkins

Control de Acceso Basado en Roles (RBAC)

RBAC te permite definir roles con permisos específicos y asignar estos roles a usuarios o grupos.

Configuración de RBAC

  1. Instalar el Plugin de Estrategia de Roles:

    • Navega a Manage Jenkins > Manage Plugins.
    • Instala el Role Strategy Plugin.

  2. Configurar Roles:

    • Navega a Manage Jenkins > Manage and Assign Roles > Manage Roles.
    • Define roles y asigna permisos.

  3. Asignar Roles a Usuarios:

    • Navega a Manage Jenkins > Manage and Assign Roles > Assign Roles.
    • Asigna roles a usuarios o grupos.

Ejemplo: Definir y Asignar Roles

roles:
  - name: admin
    permissions: [Overall/Administer, Job/Configure, Job/Delete]
  - name: developer
    permissions: [Job/Read, Job/Build, Job/Cancel]

assignments:
  - user: admin
    roles: [admin]
  - user: developer
    roles: [developer]

Mejores Prácticas de Seguridad

  • Actualizar Regularmente Jenkins: Mantén Jenkins y los plugins actualizados para mitigar vulnerabilidades.
  • Usar Contraseñas Fuertes: Imponer políticas de contraseñas fuertes.
  • Limitar el Uso de Plugins: Instalar solo los plugins necesarios para reducir la superficie de ataque.
  • Hacer Copias de Seguridad Regularmente: Realizar copias de seguridad regulares de las configuraciones y datos de Jenkins.
  • Auditar Logs: Revisar regularmente los logs de auditoría en busca de actividades sospechosas.

Conclusión

Configurar la seguridad en Jenkins es esencial para proteger tus pipelines de CI/CD y datos sensibles. Al configurar la autenticación de usuarios, habilitar SSL, implementar RBAC y seguir las mejores prácticas de seguridad, puedes mejorar significativamente la seguridad de tu instancia de Jenkins. Revisa y actualiza regularmente tus configuraciones de seguridad para adaptarte a nuevas amenazas y vulnerabilidades.
