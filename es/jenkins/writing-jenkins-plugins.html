Introducción a los Plugins de Jenkins

Jenkins es un poderoso servidor de automatización que soporta la construcción, despliegue y automatización de cualquier proyecto. Una de sus características más poderosas es su extensibilidad a través de plugins. Esta sección te guiará a través de los conceptos básicos de escribir plugins para Jenkins, desde configurar tu entorno de desarrollo hasta crear y desplegar tu primer plugin.

Conceptos Clave

  • Plugin de Jenkins: Un componente de software que añade capacidades específicas a Jenkins.
  • Arquitectura de Plugins de Jenkins: Jenkins utiliza una arquitectura modular donde los plugins pueden extender su funcionalidad principal.
  • Maven: Una herramienta de automatización de construcción utilizada principalmente para proyectos Java, esencial para el desarrollo de plugins de Jenkins.

Configurando Tu Entorno de Desarrollo

Antes de comenzar a escribir plugins para Jenkins, necesitas configurar tu entorno de desarrollo.

Prerrequisitos

  • Java Development Kit (JDK): Asegúrate de tener instalado JDK 8 o posterior.
  • Apache Maven: Maven es necesario para construir plugins de Jenkins.
  • Entorno de Desarrollo Integrado (IDE): IntelliJ IDEA, Eclipse, o cualquier otro IDE para Java.

Pasos para Configurar

  1. Instalar JDK: Descarga e instala el JDK desde el sitio web de Oracle.
  2. Instalar Maven: Descarga e instala Maven desde el sitio web de Apache Maven.
  3. Configurar Tu IDE: Configura tu IDE para usar el JDK y Maven instalados.

Creando Tu Primer Plugin para Jenkins

En esta sección, crearemos un simple plugin para Jenkins que añade un nuevo paso de construcción.

Guía Paso a Paso

  1. Generar el Esqueleto del Plugin: Usa el arquetipo de Maven para generar una estructura básica de plugin.

    mvn archetype:generate -Dfilter=io.jenkins.archetypes:

    Sigue las indicaciones para seleccionar el arquetipo io.jenkins.archetypes:empty-plugin.

  2. Entender la Estructura Generada:

    • pom.xml: El archivo descriptor del proyecto Maven.
    • src/main/java: Contiene el código fuente Java.
    • src/main/resources: Contiene los recursos del plugin, como archivos de configuración.

  3. Implementar la Lógica del Plugin:

    • Crea una nueva clase Java en src/main/java que extienda Builder.
    • Anota la clase con @Extension para registrarla como una extensión de Jenkins.
    package com.example.jenkins;

import hudson.Extension;
import hudson.model.AbstractBuild;
import hudson.model.BuildListener;
import hudson.tasks.Builder;
import hudson.tasks.BuildStepDescriptor;
import hudson.tasks.BuildStepMonitor;
import hudson.tasks.Publisher;
import org.kohsuke.stapler.DataBoundConstructor;

public class HelloWorldBuilder extends Builder {

    private final String name;

    @DataBoundConstructor
    public HelloWorldBuilder(String name) {
        this.name = name;
    }

    @Override
    public boolean perform(AbstractBuild build, BuildListener listener) {
        listener.getLogger().println("¡Hola, " + name + "!");
        return true;
    }

    @Extension
    public static final class DescriptorImpl extends BuildStepDescriptor<Builder> {
        @Override
        public boolean isApplicable(Class<? extends AbstractBuild> aClass) {
            return true;
        }

        @Override
        public String getDisplayName() {
            return "Decir Hola Mundo";
        }
    }
}

  4. Construir el Plugin:

    mvn clean install

  5. Desplegar el Plugin:

    • Copia el archivo .hpi generado desde el directorio target al directorio de plugins de Jenkins.
    • Reinicia Jenkins para cargar el nuevo plugin.

Desarrollo Avanzado de Plugins

Una vez que te sientas cómodo creando plugins básicos, puedes explorar temas más avanzados.

Conceptos Clave Avanzados

  • Puntos de Extensión de Jenkins: Interfaces o clases abstractas que los plugins pueden implementar para extender la funcionalidad de Jenkins.
  • API de Jenkins: El conjunto de clases e interfaces Java proporcionadas por Jenkins para el desarrollo de plugins.
  • Pruebas Unitarias: Escribir pruebas para tu plugin usando JenkinsRule y otras utilidades de prueba.

Ejemplo: Añadiendo una Página de Configuración

Para añadir una página de configuración a tu plugin, necesitas crear un archivo config.jelly en el directorio src/main/resources.

<j:jelly xmlns:j="jelly:core" xmlns:st="jelly:stapler" xmlns:f="/lib/form">
    <f:entry title="Nombre" field="name">
        <f:textbox />
    </f:entry>
</j:jelly>

Conclusión

Escribir plugins para Jenkins te permite extender las capacidades de Jenkins para satisfacer tus necesidades específicas. Al entender los conceptos básicos del desarrollo de plugins, configurar tu entorno de desarrollo y seguir un enfoque estructurado para crear y desplegar plugins, puedes aprovechar al máximo el potencial de Jenkins. A medida que avances, explorar los puntos de extensión y las APIs de Jenkins te permitirá crear plugins más sofisticados y poderosos. ¡Feliz codificación!
