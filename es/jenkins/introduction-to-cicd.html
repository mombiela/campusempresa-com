¿Qué es CI/CD?

Integración Continua (CI) y Despliegue Continuo (CD) son prácticas en el desarrollo de software destinadas a mejorar la calidad y velocidad de entrega del software.

  • Integración Continua (CI): La práctica de fusionar todas las copias de trabajo de los desarrolladores en una línea principal compartida varias veces al día.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio que pasa todas las etapas del pipeline de producción al entorno de producción.

Beneficios de CI/CD

  • Tiempo de Comercialización Más Rápido: Automatiza el proceso de integración y despliegue de código, reduciendo el tiempo que lleva lanzar nuevas características.
  • Mejor Calidad de Código: Las pruebas automatizadas se ejecutan con cada cambio, detectando errores temprano en el ciclo de desarrollo.
  • Reducción del Esfuerzo Manual: La automatización reduce la necesidad de intervención manual, liberando a los desarrolladores para que se concentren en escribir código.

Conceptos Clave en CI/CD

  • Automatización de Construcción: Compilar y construir el código automáticamente.
  • Pruebas Automatizadas: Ejecutar pruebas automáticamente para asegurar la calidad del código.
  • Integración Continua: Integrar frecuentemente los cambios de código en un repositorio compartido.
  • Despliegue Continuo: Desplegar automáticamente los cambios de código a producción.
  • Sistemas de Control de Versiones (VCS): Herramientas como Git que gestionan los cambios en el código fuente.

Descripción General de Jenkins

Jenkins es un servidor de automatización de código abierto que permite a los desarrolladores construir, probar y desplegar su software.

  • Plugins: Jenkins tiene un rico ecosistema de plugins que extienden su funcionalidad.
  • Pipeline: Jenkins Pipeline es un conjunto de plugins que soportan la implementación e integración de pipelines de entrega continua.
  • Pipelines Declarativos y Escritos: Jenkins soporta dos tipos de pipelines, Declarativos y Escritos.

Configuración de Jenkins

Requisitos Previos

  • Kit de Desarrollo de Java (JDK)
  • Navegador Web
  • Conexión a Internet

Pasos de Instalación

  1. Descargar Jenkins: Visita el sitio web de Jenkins y descarga el instalador apropiado para tu sistema operativo.
  2. Instalar Jenkins: Sigue las instrucciones de instalación para tu sistema operativo.
  3. Iniciar Jenkins: Ejecuta Jenkins y ábrelo en tu navegador web en http://localhost:8080.

Configuración Inicial

  • Desbloquear Jenkins: Usa la contraseña de administrador inicial proporcionada durante la instalación.
  • Instalar Plugins Sugeridos: Jenkins recomendará un conjunto de plugins para instalar.
  • Crear Usuario Administrador: Configura la cuenta de usuario administrador inicial.

Creando tu Primer Trabajo en Jenkins

Guía Paso a Paso

  1. Crear un Nuevo Trabajo: Haz clic en "Nuevo Elemento" en el panel de Jenkins.
  2. Ingresar Nombre del Trabajo: Proporciona un nombre para tu trabajo.
  3. Seleccionar Tipo de Trabajo: Elige "Proyecto de estilo libre" y haz clic en "OK".
  4. Configurar Gestión de Código Fuente: Conéctate a tu sistema de control de versiones (por ejemplo, Git). 
    git 'https://github.com/tu-repo.git'
  5. Agregar Pasos de Construcción: Agrega pasos para compilar y construir tu código. 
    mvn clean install
  6. Agregar Acciones Post-Construcción: Agrega acciones como enviar notificaciones o archivar artefactos.
  7. Guardar y Construir: Guarda la configuración del trabajo y haz clic en "Construir Ahora".

Pipeline de Jenkins

Ejemplo de Pipeline Declarativo

Un ejemplo simple de Pipeline Declarativo de Jenkins:

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                sh 'mvn clean install'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                sh 'mvn test'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                sh 'scp target/my-app.jar user@server:/path/to/deploy'
            }
        }
    }
}

Explicación

  • agent any: Ejecuta el pipeline en cualquier agente disponible.
  • stages: Define las etapas del pipeline.
  • steps: Los pasos individuales a ejecutar en cada etapa.
