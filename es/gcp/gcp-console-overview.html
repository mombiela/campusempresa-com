Introducción

La consola de Google Cloud Platform (GCP) es una interfaz web que te permite gestionar tus recursos de GCP. Esta sección proporcionará una descripción general de la consola de GCP, sus características clave y cómo navegarla de manera efectiva.

Características clave de la consola de GCP

  • Tablero: Proporciona un resumen de tus recursos de GCP, información de facturación y actividad reciente.
  • Menú de navegación: Te permite acceder a varios servicios de GCP como Compute Engine, Cloud Storage, BigQuery, etc.
  • Cloud Shell: Una interfaz de línea de comandos integrada para gestionar tus recursos de GCP.
  • IAM y administración: Gestiona permisos y roles de usuarios.
  • Facturación: Monitorea y gestiona tu gasto en GCP.
  • Soporte: Accede a recursos de soporte y documentación de GCP.

Navegando la consola de GCP

Tablero

El tablero es la primera pantalla que ves cuando inicias sesión en la consola de GCP. Proporciona una visión general de alto nivel de tus proyectos y recursos.

  • Información del proyecto: Muestra el nombre e ID del proyecto actual.
  • Resumen de recursos: Muestra un resumen de los recursos que se están utilizando en el proyecto actual.
  • Actividad reciente: Lista las acciones recientes realizadas en el proyecto.

Menú de navegación

El menú de navegación está ubicado en el lado izquierdo de la consola y proporciona acceso a todos los servicios de GCP.

  • Cómputo: Accede a servicios como Compute Engine, Kubernetes Engine y App Engine.
  • Almacenamiento: Gestiona Cloud Storage, Filestore y otras soluciones de almacenamiento.
  • Big Data: Accede a BigQuery, Dataflow y otras herramientas de big data.
  • IA y aprendizaje automático: Usa AI Platform, AutoML y otros servicios de IA.

Cloud Shell

Cloud Shell es una interfaz de línea de comandos integrada disponible directamente en la consola de GCP. Proporciona una VM temporal con la herramienta de línea de comandos gcloud preinstalada.

# Ejemplo: Listar todos los proyectos de GCP
gcloud projects list

IAM y administración

IAM (Gestión de identidades y accesos) te permite gestionar permisos para usuarios y cuentas de servicio.

  • Roles: Conjuntos predefinidos de permisos.
  • Políticas: Asigna roles a usuarios o grupos.
  • Cuentas de servicio: Cuentas especiales para aplicaciones y servicios.

Facturación

La sección de facturación te ayuda a monitorear y gestionar tu gasto en GCP.

  • Cuenta de facturación: La cuenta que paga por tu uso de GCP.
  • Presupuestos y alertas: Establece presupuestos y recibe alertas cuando el gasto supera los umbrales.
  • Informes: Ve informes de facturación detallados.

Soporte

Accede a recursos de soporte de GCP, incluyendo documentación, foros comunitarios y soporte directo de Google.

Ejemplo: Crear una instancia de VM

Vamos a crear una instancia de VM usando la consola de GCP.

  1. Navegar a Compute Engine: Haz clic en el menú de navegación y selecciona Compute Engine > Instancias de VM.
  2. Crear instancia: Haz clic en el botón "Crear instancia".
  3. Configurar instancia:
    • Nombre: Ingresa un nombre para tu instancia.
    • Región y zona: Selecciona la región y zona deseadas.
    • Tipo de máquina: Elige el tipo de máquina (por ejemplo, n1-standard-1).
    • Disco de arranque: Selecciona el sistema operativo y el tamaño del disco.
  4. Firewall: Opcionalmente, permite el tráfico HTTP y HTTPS.
  5. Crear: Haz clic en el botón "Crear" para lanzar la instancia.
# Ejemplo: Crear una instancia de VM usando gcloud
gcloud compute instances create my-instance \
    --zone=us-central1-a \
    --machine-type=n1-standard-1 \
    --image-family=debian-9 \
    --image-project=debian-cloud

Conclusión

La consola de GCP es una herramienta poderosa para gestionar tus recursos de Google Cloud. Entender sus características clave y navegación te ayudará a gestionar tus proyectos de manera eficiente. Ya sea que estés creando instancias de VM, gestionando permisos o monitoreando la facturación, la consola de GCP proporciona una interfaz fácil de usar para realizar tus tareas.
