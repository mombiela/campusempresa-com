Introducción

En esta sección, profundizaremos en conceptos avanzados de redes en Google Cloud Platform (GCP). Esto incluye comprender las configuraciones de Virtual Private Cloud (VPC), conectividad híbrida, balanceo de carga y medidas de seguridad. Al final de este tema, deberías ser capaz de diseñar e implementar soluciones de red complejas en GCP.

Virtual Private Cloud (VPC)

Una Virtual Private Cloud (VPC) es una red virtual dedicada a tu proyecto de Google Cloud. Es un recurso global que consiste en una lista de subredes regionales, una tabla de enrutamiento y reglas de firewall.

Conceptos Clave

  • Subredes: Recursos regionales que dividen tu red VPC en segmentos más pequeños.
  • Rutas: Definen caminos para que el tráfico viaje dentro de tu red VPC.
  • Reglas de Firewall: Controlan el tráfico hacia y desde las instancias basadas en reglas especificadas.

Ejemplo: Creando una VPC con Subredes

gcloud compute networks create my-vpc --subnet-mode=custom
gcloud compute networks subnets create subnet-us-central1 \
    --network=my-vpc \
    --region=us-central1 \
    --range=10.0.0.0/24

En este ejemplo, creamos una VPC en modo personalizado llamada my-vpc y una subred llamada subnet-us-central1 en la región us-central1 con un rango CIDR de 10.0.0.0/24.

Conectividad Híbrida

La conectividad híbrida te permite conectar tu red on-premises a tu red VPC de GCP. Esto se puede lograr utilizando varios métodos como VPN, Interconnect y Peering.

Conceptos Clave

  • Cloud VPN: Establece una conexión segura a través de internet.
  • Cloud Interconnect: Proporciona una conexión física directa entre tu red on-premises y GCP.
  • VPC Peering: Conecta dos redes VPC para habilitar la comunicación utilizando direcciones IP internas.

Ejemplo: Configurando una Cloud VPN

gcloud compute vpn gateways create my-vpn-gateway \
    --network=my-vpc \
    --region=us-central1

gcloud compute vpn tunnels create my-vpn-tunnel \
    --peer-address=203.0.113.1 \
    --region=us-central1 \
    --ike-version=2 \
    --shared-secret=my-secret \
    --target-vpn-gateway=my-vpn-gateway

En este ejemplo, creamos una puerta de enlace VPN y un túnel VPN para establecer una conexión segura a una red on-premises con una dirección de par 203.0.113.1.

Balanceo de Carga

El balanceo de carga distribuye el tráfico entrante entre múltiples instancias para asegurar alta disponibilidad y fiabilidad.

Conceptos Clave

  • Balanceo de Carga HTTP(S): Distribuye tráfico HTTP y HTTPS.
  • Balanceo de Carga TCP/UDP: Distribuye tráfico TCP y UDP.
  • Balanceo de Carga Interno: Distribuye tráfico dentro de una VPC.

Ejemplo: Configurando un Balanceador de Carga HTTP

gcloud compute instance-groups unmanaged create my-instance-group --zone=us-central1-a

gcloud compute instance-groups unmanaged add-instances my-instance-group \
    --instances=instance-1,instance-2 --zone=us-central1-a

gcloud compute backend-services create my-backend-service \
    --protocol=HTTP \
    --port-name=http \
    --health-checks=my-health-check \
    --global

gcloud compute url-maps create my-url-map \
    --default-service=my-backend-service

gcloud compute target-http-proxies create my-http-proxy \
    --url-map=my-url-map

gcloud compute forwarding-rules create my-http-rule \
    --global \
    --target-http-proxy=my-http-proxy \
    --ports=80

En este ejemplo, creamos un grupo de instancias no gestionado, añadimos instancias a él y configuramos un balanceador de carga HTTP para distribuir el tráfico a estas instancias.

Medidas de Seguridad

La seguridad es un aspecto crítico de las redes. GCP proporciona varias herramientas y características para asegurar tu red.

Conceptos Clave

  • Reglas de Firewall: Controlan el tráfico hacia y desde las instancias.
  • Cloud Armor: Proporciona protección contra DDoS y capacidades de WAF.
  • Acceso Privado a Google: Permite a las instancias acceder a las APIs y servicios de Google utilizando direcciones IP internas.

Ejemplo: Creando una Regla de Firewall

gcloud compute firewall-rules create allow-http \
    --network=my-vpc \
    --allow=tcp:80 \
    --source-ranges=0.0.0.0/0 \
    --target-tags=http-server

En este ejemplo, creamos una regla de firewall para permitir tráfico HTTP (puerto 80) desde cualquier fuente a instancias etiquetadas con http-server.

Conclusión

En este tema, cubrimos conceptos avanzados de redes en GCP, incluyendo configuraciones de VPC, conectividad híbrida, balanceo de carga y medidas de seguridad. Al dominar estos conceptos, puedes diseñar e implementar soluciones de red robustas y seguras en Google Cloud Platform. Continúa practicando con escenarios del mundo real para solidificar tu comprensión y habilidades.
