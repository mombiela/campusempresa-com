Introducción a las Arquitecturas Serverless

La computación serverless te permite construir y ejecutar aplicaciones y servicios sin pensar en servidores. Con serverless, el proveedor de la nube provisiona, escala y gestiona automáticamente la infraestructura necesaria para ejecutar tu código.

Conceptos Clave

  • Computación Serverless: Un modelo de ejecución en la nube donde el proveedor de la nube gestiona dinámicamente la asignación de recursos de la máquina.
  • Función como Servicio (FaaS): Una categoría de servicios de computación en la nube que proporciona una plataforma que permite a los clientes desarrollar, ejecutar y gestionar funcionalidades de aplicaciones sin la complejidad de construir y mantener la infraestructura.
  • Arquitectura Basada en Eventos: Un patrón de arquitectura de software que promueve la producción, detección, consumo y reacción a eventos.

Comenzando con Google Cloud Functions

Google Cloud Functions es una solución de computación ligera, basada en eventos y asíncrona que te permite crear funciones pequeñas y de un solo propósito que responden a eventos en la nube sin la necesidad de gestionar un servidor o un entorno de ejecución.

Creando Tu Primera Cloud Function

  1. Configura Tu Proyecto GCP:

    • Asegúrate de tener una cuenta de GCP y crea un nuevo proyecto.
    • Habilita la API de Cloud Functions.

  2. Escribe Tu Función:

    • Crea una función simple que responda a solicitudes HTTP.
// index.js
exports.helloWorld = (req, res) => {
  res.send('¡Hola, Mundo!');
};
  1. Despliega Tu Función:
    • Usa la herramienta de línea de comandos gcloud para desplegar tu función.
gcloud functions deploy helloWorld --runtime nodejs14 --trigger-http --allow-unauthenticated
  1. Prueba Tu Función:
    • Después del despliegue, recibirás una URL. Abre esta URL en tu navegador para ver el resultado.

Funciones Avanzadas de Cloud Functions

Variables de Entorno

  • Configurando Variables de Entorno: Usa variables de entorno para configurar tus funciones sin cambiar el código.
gcloud functions deploy helloWorld --set-env-vars GREETING="¡Hola, GCP!"
  • Accediendo a Variables de Entorno:
exports.helloWorld = (req, res) => {
  const greeting = process.env.GREETING || '¡Hola, Mundo!';
  res.send(greeting);
};

Conectando a Otros Servicios de GCP

  • Cloud Storage: Desencadena funciones en cambios de archivos en Cloud Storage.
exports.processFile = (event, context) => {
  const file = event.name;
  console.log(`Procesando archivo: ${file}`);
};
  • Cloud Pub/Sub: Desencadena funciones en mensajes de Pub/Sub.
exports.subscribe = (message, context) => {
  const data = Buffer.from(message.data, 'base64').toString();
  console.log(`Mensaje recibido: ${data}`);
};

Google Cloud Run

Google Cloud Run te permite ejecutar contenedores sin estado que pueden ser invocados mediante solicitudes HTTP. Abstrae la gestión de la infraestructura, permitiéndote enfocarte en escribir código.

Desplegando un Contenedor en Cloud Run

  1. Crea un Dockerfile:
# Usa la imagen oficial de Node.js.
FROM node:14

# Crea y cambia al directorio de la aplicación.
WORKDIR /usr/src/app

# Copia los manifiestos de dependencias de la aplicación a la imagen del contenedor.
COPY package*.json ./

# Instala las dependencias.
RUN npm install

# Copia el código local a la imagen del contenedor.
COPY . .

# Ejecuta el servicio web al iniciar el contenedor.
CMD [ "node", "index.js" ]

# Documenta que el servicio escucha en el puerto 8080.
EXPOSE 8080
  1. Construye y Sube la Imagen del Contenedor:
gcloud builds submit --tag gcr.io/PROJECT-ID/helloworld
  1. Despliega en Cloud Run:
gcloud run deploy --image gcr.io/PROJECT-ID/helloworld --platform managed
  1. Prueba Tu Servicio:
    • Después del despliegue, recibirás una URL. Abre esta URL en tu navegador para ver el resultado.

Mejores Prácticas para Arquitecturas Serverless

  • Funciones Sin Estado: Asegúrate de que tus funciones sean sin estado para permitir una fácil escalabilidad.
  • Gestión Eficiente de Recursos: Usa variables de entorno y gestión de secretos para manejar la configuración.
  • Monitoreo y Registro: Utiliza las herramientas de monitoreo y registro de GCP para mantener un seguimiento del rendimiento y errores de tus funciones.

Conclusión

Las arquitecturas serverless en GCP proporcionan una forma poderosa de construir aplicaciones escalables y eficientes sin la sobrecarga de gestionar la infraestructura. Al aprovechar servicios como Google Cloud Functions y Google Cloud Run, puedes enfocarte en escribir código y entregar valor a tus usuarios. Recuerda seguir las mejores prácticas para asegurar que tus aplicaciones sean robustas y mantenibles.
