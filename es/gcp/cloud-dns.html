Introducción a Cloud DNS

Google Cloud DNS es un servicio de Sistema de Nombres de Dominio (DNS) autoritativo, escalable, confiable y gestionado que funciona en la misma infraestructura que Google. Traduce nombres de dominio como www.example.com en direcciones IP como 192.0.2.1, que son utilizadas por las computadoras para comunicarse entre sí.

Conceptos Clave

  • DNS: Sistema de Nombres de Dominio, un sistema jerárquico para traducir nombres de dominio amigables para los humanos en direcciones IP.
  • DNS Gestionado: Un servicio DNS proporcionado por un tercero, en este caso, Google Cloud, que maneja la infraestructura y la escalabilidad.
  • Zonas: Una zona DNS es una porción del espacio de nombres DNS gestionada por una organización o administrador específico.
  • Registros: Los registros DNS son entradas en una zona DNS que asignan nombres de dominio a direcciones IP u otros recursos.

Configuración de Cloud DNS

Creación de una Zona DNS

Para crear una zona DNS en Google Cloud:

  1. Ve a la página de Cloud DNS en la Consola de Google Cloud.
  2. Haz clic en "Crear zona".
  3. Rellena los siguientes campos:
    • Nombre de la zona: Un nombre único para la zona.
    • Nombre DNS: El nombre de dominio para la zona (por ejemplo, example.com).
    • Descripción: Una descripción opcional para la zona.
    • DNSSEC: Habilitar o deshabilitar DNSSEC (Extensiones de Seguridad del Sistema de Nombres de Dominio).
gcloud dns managed-zones create my-zone \
    --description="Mi primera zona DNS" \
    --dns-name="example.com." \
    --visibility="public"

Añadir Registros DNS

Una vez creada la zona, puedes añadir registros DNS:

  1. Ve a la página de Cloud DNS en la Consola de Google Cloud.
  2. Haz clic en la zona que creaste.
  3. Haz clic en "Añadir conjunto de registros".
  4. Rellena los siguientes campos:
    • Nombre DNS: El subdominio para el registro (por ejemplo, www).
    • Tipo de registro de recurso: El tipo de registro DNS (por ejemplo, A, CNAME, MX).
    • TTL: Tiempo de vida, la duración en segundos que el registro es almacenado en caché por los resolutores DNS.
    • Datos: La dirección IP u otros datos para el registro.
gcloud dns record-sets transaction start --zone=my-zone
gcloud dns record-sets transaction add --zone=my-zone \
    --name="www.example.com." --ttl=300 --type=A "192.0.2.1"
gcloud dns record-sets transaction execute --zone=my-zone

Configuración Avanzada

DNSSEC

DNSSEC añade una capa adicional de seguridad a tu DNS habilitando firmas criptográficas en tus registros DNS.

  • Habilitar DNSSEC: Esto se puede hacer durante la creación de la zona DNS o más tarde editando la configuración de la zona.
  • Gestión de Claves: Google Cloud DNS maneja la gestión de claves automáticamente, pero también puedes gestionar las claves manualmente si es necesario.

Zonas DNS Privadas

Las zonas DNS privadas te permiten gestionar registros DNS para tu red interna, que solo son accesibles desde dentro de tu VPC (Nube Privada Virtual).

  1. Crear una zona DNS privada:

    gcloud dns managed-zones create my-private-zone 
           --description="Mi zona DNS privada" 
           --dns-name="internal.example.com." 
           --visibility="private" 
           --networks="my-vpc-network"

  2. Añadir registros a la zona privada como lo harías con una zona pública.

Monitoreo y Registro

Google Cloud proporciona herramientas para monitorear y registrar consultas y respuestas DNS:

  • Cloud Logging: Habilita el registro para tus zonas DNS para capturar registros de consultas DNS.
  • Cloud Monitoring: Configura alertas y paneles para monitorear la salud y el rendimiento de tus zonas DNS.
gcloud dns managed-zones update my-zone --logging

Conclusión

Google Cloud DNS es un servicio DNS poderoso y flexible que puede manejar tanto necesidades DNS públicas como privadas. Al comprender los conceptos básicos de DNS, configurar zonas y registros, y aprovechar características avanzadas como DNSSEC y zonas privadas, puedes gestionar eficazmente la infraestructura DNS de tu dominio. Las capacidades de monitoreo y registro mejoran aún más tu capacidad para mantener un entorno DNS robusto y seguro.
