Introducción a Cloud Run

Cloud Run es una plataforma de computación completamente gestionada que escala automáticamente tus contenedores sin estado. Abstrae toda la gestión de infraestructura, permitiéndote enfocarte en escribir código.

  • Completamente Gestionado: No necesitas gestionar servidores o clústeres.
  • Escalable: Escala automáticamente hacia arriba y hacia abajo según el tráfico.
  • Contenedores Sin Estado: Ejecuta cualquier contenedor sin estado.

Configuración de Cloud Run

Antes de desplegar en Cloud Run, necesitas configurar tu entorno de Google Cloud Platform (GCP).

  1. Crear un Proyecto GCP:

    • Ve a la Consola de GCP.
    • Haz clic en el desplegable de proyectos y selecciona "Nuevo Proyecto".
    • Nombra tu proyecto y haz clic en "Crear".

  2. Habilitar Facturación:

    • Ve a la sección de Facturación en la Consola de GCP.
    • Vincula una cuenta de facturación a tu proyecto.

  3. Habilitar la API de Cloud Run:

    • Navega a la sección de API y Servicios.
    • Busca "Cloud Run" y habilita la API.

  4. Instalar Google Cloud SDK:

    • Sigue las instrucciones de instalación para tu sistema operativo desde la documentación de Google Cloud SDK.

Desplegando un Contenedor en Cloud Run

Guía Paso a Paso

  1. Escribe Tu Aplicación:

    • Crea un servidor web simple usando tu lenguaje preferido. Aquí tienes un ejemplo en Python usando Flask:
    # app.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return "¡Hola, Mundo!"

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)

  2. Crea un Dockerfile:

    • Dockerfile se usa para contenerizar tu aplicación.
    # Dockerfile
FROM python:3.8-slim

WORKDIR /app

COPY requirements.txt requirements.txt
RUN pip install -r requirements.txt

COPY . .

CMD ["python", "app.py"]

  3. Construye y Sube la Imagen del Contenedor:

    • Usa Google Container Registry (GCR) para almacenar tus imágenes de contenedor.
    # Construye la imagen Docker
docker build -t gcr.io/[ID-PROYECTO]/[NOMBRE-IMAGEN]:[ETIQUETA] .

# Sube la imagen Docker a GCR
docker push gcr.io/[ID-PROYECTO]/[NOMBRE-IMAGEN]:[ETIQUETA]

  4. Despliega en Cloud Run:

    • Usa la herramienta de línea de comandos gcloud para desplegar tu contenedor.
    gcloud run deploy --image gcr.io/[ID-PROYECTO]/[NOMBRE-IMAGEN]:[ETIQUETA] --platform managed
    • Sigue las indicaciones para especificar el nombre del servicio y la región.

Funcionalidades Avanzadas de Cloud Run

Dominios Personalizados

  • Mapeo de Dominios Personalizados: Puedes mapear dominios personalizados a tus servicios de Cloud Run.
    • Ve a la página de detalles del servicio de Cloud Run.
    • Haz clic en "Gestionar dominios personalizados" y sigue las instrucciones.

Gestión de Tráfico

  • División de Tráfico: Puedes dividir el tráfico entre diferentes revisiones de tu servicio.
    • Navega a la página de detalles del servicio de Cloud Run.
    • Haz clic en "Gestionar tráfico" y asigna porcentajes a diferentes revisiones.

Autenticación y Autorización

  • Roles de IAM: Controla quién puede desplegar e invocar tus servicios usando roles de IAM.
    • Asigna roles como roles/run.admin y roles/run.invoker a usuarios o cuentas de servicio.

Monitoreo y Registro

  • Monitoreo en la Nube: Usa Cloud Monitoring para rastrear el rendimiento de tus servicios.
    • Configura paneles de control y alertas personalizadas.
  • Registro en la Nube: Recoge automáticamente los registros de tus servicios.
    • Visualiza los registros en la sección de Cloud Logging de la Consola de GCP.

Mejores Prácticas

  • Servicios Sin Estado: Asegúrate de que tus servicios sean sin estado para una mejor escalabilidad.
  • Variables de Entorno: Usa variables de entorno para gestionar la configuración.
  • Verificaciones de Salud: Implementa verificaciones de salud para asegurar que tu servicio esté funcionando correctamente.

Conclusión

Cloud Run simplifica el despliegue y la gestión de aplicaciones contenerizadas al abstraer la infraestructura subyacente. Ofrece funcionalidades potentes como escalado automático, dominios personalizados, gestión de tráfico y un robusto monitoreo y registro. Siguiendo las mejores prácticas y aprovechando estas funcionalidades, puedes desplegar y gestionar eficientemente tus aplicaciones en GCP.

Al final de este tema, deberías tener una comprensión sólida de cómo configurar, desplegar y gestionar aplicaciones usando Cloud Run en Google Cloud Platform.
