Overfitting y Underfitting

Introducción

En Machine Learning, uno de los desafíos más comunes es encontrar el equilibrio adecuado entre el ajuste del modelo a los datos de entrenamiento y su capacidad para generalizar a datos no vistos. Este equilibrio se ve afectado por dos problemas principales: el overfitting (sobreajuste) y el underfitting (subajuste).

Definiciones

Overfitting

  • Definición: Ocurre cuando un modelo se ajusta demasiado bien a los datos de entrenamiento, capturando tanto las tendencias generales como el ruido y las anomalías. Esto resulta en un rendimiento excelente en el conjunto de entrenamiento, pero pobre en el conjunto de prueba o en datos nuevos.
  • Síntomas:
    • Alta precisión en el conjunto de entrenamiento.
    • Baja precisión en el conjunto de prueba.
  • Causas comunes:
    • Modelo demasiado complejo (muchos parámetros).
    • Demasiadas iteraciones de entrenamiento.
    • Datos de entrenamiento con ruido o anomalías.

Underfitting

  • Definición: Ocurre cuando un modelo es demasiado simple para capturar las tendencias subyacentes en los datos de entrenamiento. Esto resulta en un rendimiento pobre tanto en el conjunto de entrenamiento como en el conjunto de prueba.
  • Síntomas:
    • Baja precisión en el conjunto de entrenamiento.
    • Baja precisión en el conjunto de prueba.
  • Causas comunes:
    • Modelo demasiado simple (pocos parámetros).
    • Insuficiente tiempo de entrenamiento.
    • Datos de entrenamiento insuficientes o no representativos.

Ejemplos de Código

Overfitting

import numpy as np
import matplotlib.pyplot as plt
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

# Generar datos de ejemplo
np.random.seed(0)
X = np.sort(np.random.rand(20, 1) * 10, axis=0)
y = np.sin(X).ravel() + np.random.randn(20) * 0.5

# Dividir los datos en conjuntos de entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=0)

# Ajustar un modelo polinomial de grado 15 (overfitting)
poly = PolynomialFeatures(degree=15)
X_poly_train = poly.fit_transform(X_train)
X_poly_test = poly.transform(X_test)
model = LinearRegression()
model.fit(X_poly_train, y_train)

# Predicciones
y_train_pred = model.predict(X_poly_train)
y_test_pred = model.predict(X_poly_test)

# Evaluación
print("Error en el conjunto de entrenamiento:", mean_squared_error(y_train, y_train_pred))
print("Error en el conjunto de prueba:", mean_squared_error(y_test, y_test_pred))

# Visualización
plt.scatter(X, y, color='black', label='Datos')
plt.plot(X, model.predict(poly.transform(X)), color='blue', label='Modelo polinomial (grado 15)')
plt.legend()
plt.show()

Underfitting

# Ajustar un modelo lineal simple (underfitting)
model_simple = LinearRegression()
model_simple.fit(X_train, y_train)

# Predicciones
y_train_pred_simple = model_simple.predict(X_train)
y_test_pred_simple = model_simple.predict(X_test)

# Evaluación
print("Error en el conjunto de entrenamiento (modelo simple):", mean_squared_error(y_train, y_train_pred_simple))
print("Error en el conjunto de prueba (modelo simple):", mean_squared_error(y_test, y_test_pred_simple))

# Visualización
plt.scatter(X, y, color='black', label='Datos')
plt.plot(X, model_simple.predict(X), color='red', label='Modelo lineal simple')
plt.legend()
plt.show()

Comparación

| Característica | Overfitting | Underfitting | |--------------------|--------------------------------------|--------------------------------------| | Complejidad del modelo | Alta (muchos parámetros) | Baja (pocos parámetros) | | Rendimiento en entrenamiento | Alto | Bajo | | Rendimiento en prueba | Bajo | Bajo | | Causa principal | Captura de ruido y anomalías | Incapacidad para capturar la tendencia subyacente |

Cómo Mitigar Overfitting y Underfitting

Técnicas para Mitigar Overfitting

  • Regularización: Añadir una penalización por complejidad del modelo (L1, L2).
  • Cross-validation: Utilizar validación cruzada para evaluar el rendimiento del modelo.
  • Pruning: En el caso de árboles de decisión, podar ramas innecesarias.
  • Early stopping: Detener el entrenamiento antes de que el modelo se ajuste demasiado a los datos de entrenamiento.
  • Aumentar datos: Incrementar el tamaño del conjunto de datos de entrenamiento.

Técnicas para Mitigar Underfitting

  • Aumentar la complejidad del modelo: Utilizar modelos más complejos o agregar más parámetros.
  • Más características: Incluir más características relevantes en el conjunto de datos.
  • Más tiempo de entrenamiento: Aumentar el número de iteraciones o épocas de entrenamiento.
  • Reducir la regularización: Disminuir la penalización por complejidad del modelo.

Conclusión

El equilibrio entre overfitting y underfitting es crucial para el éxito de un modelo de Machine Learning. Comprender las causas y síntomas de cada uno, así como las técnicas para mitigarlos, es esencial para desarrollar modelos que generalicen bien a datos no vistos. Practicar con diferentes modelos y técnicas en diversos conjuntos de datos ayudará a mejorar la intuición y habilidades para manejar estos desafíos.
