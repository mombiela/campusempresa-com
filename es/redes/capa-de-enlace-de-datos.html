Introducción

La Capa de Enlace de Datos es la segunda capa del modelo OSI (Open Systems Interconnection) y es fundamental para la transmisión de datos entre dispositivos en una red local. Esta capa se encarga de la transferencia fiable de datos a través de un enlace físico, gestionando errores y controlando el flujo de información.

Funciones Principales

La Capa de Enlace de Datos tiene varias funciones clave:

  • Encapsulación de Datos: Los datos se encapsulan en tramas para su transmisión.
  • Control de Acceso al Medio (MAC): Gestiona el acceso al medio físico compartido.
  • Detección y Corrección de Errores: Asegura la integridad de los datos mediante técnicas de detección y corrección de errores.
  • Control de Flujo: Regula la velocidad de transmisión de datos entre dispositivos para evitar la saturación.

Subcapas de la Capa de Enlace de Datos

La Capa de Enlace de Datos se divide en dos subcapas:

  1. Subcapa de Control de Enlace Lógico (LLC):

    • Proporciona un enlace lógico entre los dispositivos.
    • Maneja la multiplexación de protocolos de red.
    • Controla la sincronización y el control de flujo.

  2. Subcapa de Control de Acceso al Medio (MAC):

    • Gestiona el acceso al medio físico.
    • Define las direcciones MAC para identificar dispositivos en la red.
    • Implementa protocolos como Ethernet, Wi-Fi, etc.

Direcciones MAC

Las direcciones MAC (Media Access Control) son identificadores únicos asignados a las interfaces de red para la comunicación en la Capa de Enlace de Datos. Una dirección MAC tiene 48 bits y se representa en formato hexadecimal.

Ejemplo de Dirección MAC

00:1A:2B:3C:4D:5E
  • Primeros 24 bits: Identificador único del fabricante (OUI).
  • Últimos 24 bits: Identificador único asignado por el fabricante.

Protocolos de la Capa de Enlace de Datos

Algunos de los protocolos más comunes en la Capa de Enlace de Datos incluyen:

  • Ethernet: Utilizado en redes cableadas.
  • Wi-Fi (IEEE 802.11): Utilizado en redes inalámbricas.
  • PPP (Point-to-Point Protocol): Utilizado en conexiones punto a punto.

Ejemplo de Trama Ethernet

+-------------------+-------------------+-------------------+-------------------+
|  Dirección MAC    |  Dirección MAC    |  Tipo/Longitud    |       Datos       |
|    Destino        |    Origen         |                   |                   |
+-------------------+-------------------+-------------------+-------------------+
|       6 bytes     |      6 bytes      |      2 bytes      |    46-1500 bytes  |
+-------------------+-------------------+-------------------+-------------------+
|     CRC (FCS)     |
|      4 bytes      |
+-------------------+
  • Dirección MAC Destino: Dirección del dispositivo receptor.
  • Dirección MAC Origen: Dirección del dispositivo emisor.
  • Tipo/Longitud: Indica el tipo de protocolo de red o la longitud de los datos.
  • Datos: Información a ser transmitida.
  • CRC (FCS): Código de redundancia cíclica para la detección de errores.

Ejercicio Práctico

Ejercicio 1: Identificación de Direcciones MAC

  1. Encuentra la dirección MAC de tu dispositivo.
  2. Identifica el OUI y el fabricante correspondiente.

Ejercicio 2: Análisis de Tramas Ethernet

  1. Captura tramas Ethernet utilizando una herramienta como Wireshark.
  2. Analiza los campos de una trama capturada y explica su contenido.

Conclusión

La Capa de Enlace de Datos es esencial para la comunicación fiable y eficiente en redes locales. A través de sus subcapas LLC y MAC, gestiona la encapsulación de datos, el control de acceso al medio, y la detección y corrección de errores. Comprender esta capa es crucial para cualquier profesional de TI que trabaje con redes, ya que proporciona la base para la transmisión de datos en una red local.
