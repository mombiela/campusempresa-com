Introducción

Los protocolos de transporte son fundamentales en la arquitectura de redes, ya que se encargan de gestionar la transmisión de datos entre sistemas finales. En esta sección, exploraremos los principales protocolos de transporte, sus características y cómo se utilizan en la práctica.

Principales Protocolos de Transporte

TCP (Transmission Control Protocol)

TCP es uno de los protocolos de transporte más utilizados en las redes de datos. Proporciona una conexión confiable y orientada a la conexión entre dos sistemas finales.

  • Características de TCP:

    • Confiabilidad: Garantiza la entrega de los datos sin errores y en el orden correcto.
    • Control de flujo: Gestiona la cantidad de datos que se envían para evitar la congestión de la red.
    • Control de congestión: Ajusta la velocidad de transmisión en función de la capacidad de la red.
    • Orientado a la conexión: Establece una conexión antes de transmitir datos.

  • Estructura del Segmento TCP: | Campo | Descripción | |--------------------|------------------------------------| | Puerto de origen | Puerto del emisor | | Puerto de destino | Puerto del receptor | | Número de secuencia| Número de secuencia del segmento | | Número de acuse | Número de acuse de recibo | | Longitud de cabecera| Longitud de la cabecera TCP | | Reservado | Reservado para uso futuro | | Flags | Indicadores de control | | Tamaño de ventana | Tamaño de la ventana de recepción | | Checksum | Verificación de errores | | Puntero urgente | Indica datos urgentes | | Opciones | Opciones adicionales | | Datos | Datos de la aplicación |

  • Ejemplo de Código TCP en Python:

    import socket

# Crear un socket TCP
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# Conectar al servidor
s.connect(('www.example.com', 80))

# Enviar una solicitud HTTP GET
request = "GET / HTTP/1.1\r\nHost: www.example.com\r\n\r\n"
s.sendall(request.encode('utf-8'))

# Recibir la respuesta del servidor
response = s.recv(4096)
print(response.decode('utf-8'))

# Cerrar la conexión
s.close()

UDP (User Datagram Protocol)

UDP es un protocolo de transporte que proporciona un servicio de transmisión de datos sin conexión y sin garantía de entrega.

  • Características de UDP:

    • No confiable: No garantiza la entrega de los datos ni el orden de los mismos.
    • Sin conexión: No establece una conexión antes de transmitir datos.
    • Baja sobrecarga: Tiene una cabecera más pequeña y menos control que TCP.
    • Ideal para aplicaciones que requieren velocidad y pueden tolerar pérdida de datos (e.g., streaming de video, juegos en línea).

  • Estructura del Datagram UDP: | Campo | Descripción | |--------------------|------------------------------------| | Puerto de origen | Puerto del emisor | | Puerto de destino | Puerto del receptor | | Longitud | Longitud del datagrama UDP | | Checksum | Verificación de errores | | Datos | Datos de la aplicación |

  • Ejemplo de Código UDP en Python:

    import socket

# Crear un socket UDP
s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# Dirección del servidor
server_address = ('localhost', 10000)

# Mensaje a enviar
message = 'Este es un mensaje UDP'

try:
    # Enviar el mensaje
    print(f'Enviando: {message}')
    sent = s.sendto(message.encode('utf-8'), server_address)

    # Recibir respuesta
    print('Esperando respuesta...')
    data, server = s.recvfrom(4096)
    print(f'Recibido: {data.decode("utf-8")}')
finally:
    # Cerrar el socket
    s.close()

Comparación entre TCP y UDP

| Característica | TCP | UDP | |----------------------|------------------------------------|------------------------------------| | Confiabilidad | Alta (garantiza entrega y orden) | Baja (no garantiza entrega ni orden)| | Control de flujo | Sí | No | | Control de congestión| Sí | No | | Orientado a conexión | Sí | No | | Sobrecarga | Alta (cabecera más grande) | Baja (cabecera más pequeña) | | Uso típico | Transferencia de archivos, correos electrónicos | Streaming de video, juegos en línea |

Conclusión

En esta sección, hemos explorado los dos principales protocolos de transporte: TCP y UDP. Ambos tienen sus propias ventajas y desventajas, y se utilizan en diferentes escenarios según las necesidades de la aplicación. TCP es ideal para aplicaciones que requieren confiabilidad y orden, mientras que UDP es más adecuado para aplicaciones que necesitan velocidad y pueden tolerar la pérdida de datos. Entender estas diferencias es crucial para diseñar y gestionar redes de manera efectiva.
