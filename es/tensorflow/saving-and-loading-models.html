Guardar y Cargar Modelos
Introducción a las CNNs ►

En esta sección, cubriremos cómo guardar y cargar modelos en TensorFlow. Esta es una habilidad crucial para cualquier practicante de aprendizaje automático, ya que te permite preservar tus modelos entrenados y reutilizarlos más tarde sin necesidad de reentrenarlos.

¿Por qué Guardar y Cargar Modelos?

  • Preservación: Guarda el modelo para evitar reentrenarlo.
  • Despliegue: Carga el modelo para su despliegue en entornos de producción.
  • Compartir: Comparte el modelo con otros.
  • Experimentación: Guarda modelos intermedios durante el entrenamiento para experimentación.

Métodos para Guardar y Cargar Modelos

TensorFlow proporciona varios métodos para guardar y cargar modelos:

  1. Checkpointing: Guarda y carga los pesos del modelo.
  2. SavedModel: Guarda y carga toda la arquitectura del modelo, los pesos y el estado del optimizador.
  3. Formato HDF5: Guarda y carga modelos usando el formato HDF5.

Checkpointing

Checkpointing es útil para guardar y cargar solo los pesos del modelo.

Guardar Pesos

import tensorflow as tf

# Define un modelo simple
model = tf.keras.Sequential([
    tf.keras.layers.Dense(10, activación='relu'),
    tf.keras.layers.Dense(1)
])

# Compila el modelo
model.compile(optimizer='adam', loss='mean_squared_error')

# Guarda los pesos
model.save_weights('model_checkpoint')

Cargar Pesos

# Carga los pesos
model.load_weights('model_checkpoint')

Formato SavedModel

El formato SavedModel es el formato de serialización estándar de TensorFlow. Incluye la arquitectura, los pesos y el estado del optimizador.

Guardar el Modelo

# Guarda todo el modelo
model.save('saved_model/my_model')

Cargar el Modelo

# Carga todo el modelo
loaded_model = tf.keras.models.load_model('saved_model/my_model')

Formato HDF5

El formato HDF5 es otra forma de guardar todo el modelo, incluyendo la arquitectura, los pesos y el estado del optimizador.

Guardar el Modelo

# Guarda todo el modelo en formato HDF5
model.save('my_model.h5')

Cargar el Modelo

# Carga todo el modelo desde el formato HDF5
loaded_model = tf.keras.models.load_model('my_model.h5')

Comparación de Métodos

| Método | Guarda Arquitectura | Guarda Pesos | Guarda Estado del Optimizador | Formato | |--------------|---------------------|--------------|-------------------------------|---------------| | Checkpointing| No | Sí | No | TensorFlow | | SavedModel | Sí | Sí | Sí | TensorFlow | | HDF5 | Sí | Sí | Sí | HDF5 |

Ejemplo Práctico

Vamos a recorrer un ejemplo práctico de entrenamiento de un modelo, guardarlo y luego cargarlo para inferencia.

import tensorflow as tf
import numpy as np

# Genera datos ficticios
x_train = np.random.random((1000, 32))
y_train = np.random.random((1000, 1))

# Define un modelo simple
model = tf.keras.Sequential([
    tf.keras.layers.Dense(64, activación='relu', input_shape=(32,)),
    tf.keras.layers.Dense(1)
])

# Compila el modelo
model.compile(optimizer='adam', loss='mean_squared_error')

# Entrena el modelo
model.fit(x_train, y_train, epochs=10)

# Guarda el modelo
model.save('my_model.h5')

# Carga el modelo
loaded_model = tf.keras.models.load_model('my_model.h5')

# Realiza predicciones con el modelo cargado
predictions = loaded_model.predict(x_train)
print(predictions)

Conclusión

En esta sección, cubrimos la importancia de guardar y cargar modelos en TensorFlow. Exploramos diferentes métodos, incluyendo checkpointing, el formato SavedModel y el formato HDF5. Cada método tiene sus propios casos de uso y beneficios. Al dominar estas técnicas, puedes asegurarte de que tus modelos estén preservados, sean fácilmente desplegables y compartibles.
