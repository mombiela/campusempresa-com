Introducción a TensorFlow Serving

TensorFlow Serving es un sistema de servicio flexible y de alto rendimiento para modelos de aprendizaje automático, diseñado para entornos de producción. Permite desplegar nuevos algoritmos y experimentos manteniendo la misma arquitectura de servidor y APIs.

Conceptos Clave

  • Servidor de Modelos: Un servidor diseñado para servir modelos de aprendizaje automático.
  • Versionado de Modelos: Soporta múltiples versiones de modelos, permitiendo actualizaciones sin interrupciones.
  • Agrupamiento: Combina múltiples solicitudes para mejorar el rendimiento.
  • Monitoreo: Proporciona herramientas para monitorear la salud y el rendimiento de los modelos.

Configuración de TensorFlow Serving

Para comenzar con TensorFlow Serving, necesitas instalarlo y configurar tu entorno.

Instalación

Puedes instalar TensorFlow Serving usando Docker o directamente en tu sistema.

Usando Docker

docker pull tensorflow/serving

Instalación Directa

Para Ubuntu:

echo "deb [arch=amd64] http://storage.googleapis.com/tensorflow-serving-apt stable tensorflow-model-server tensorflow-model-server-universal" | sudo tee /etc/apt/sources.list.d/tensorflow-serving.list && \
curl https://storage.googleapis.com/tensorflow-serving-apt/tensorflow-serving.release.pub.gpg | sudo apt-key add -
sudo apt-get update && sudo apt-get install tensorflow-model-server

Sirviendo un Modelo de TensorFlow

Una vez que TensorFlow Serving está instalado, puedes servir un modelo de TensorFlow.

Exportando un Modelo

Primero, necesitas exportar tu modelo entrenado en un formato que TensorFlow Serving pueda usar.

import tensorflow as tf

# Define un modelo simple
model = tf.keras.Sequential([
    tf.keras.layers.Dense(10, activación='relu', input_shape=(784,)),
    tf.keras.layers.Dense(10, activación='softmax')
])

# Guarda el modelo
model.save('my_model/1')

Iniciando el Servidor de Modelos

Usa el siguiente comando para iniciar el Servidor de Modelos de TensorFlow:

docker run -p 8501:8501 --name=tfserving_my_model --mount type=bind,source=$(pwd)/my_model,target=/models/my_model -e MODEL_NAME=my_model -t tensorflow/serving

Haciendo Predicciones

Puedes hacer predicciones enviando solicitudes HTTP POST a la API de TensorFlow Serving.

Ejemplo de Solicitud

import requests
import json

data = json.dumps({"signature_name": "serving_default", "instances": [[0.0]*784]})
headers = {"content-type": "application/json"}
json_response = requests.post('http://localhost:8501/v1/models/my_model:predict', data=data, headers=headers)
predictions = json.loads(json_response.text)['predictions']
print(predictions)

Características Avanzadas

TensorFlow Serving ofrece varias características avanzadas para optimizar y gestionar tus modelos.

Versionado de Modelos

TensorFlow Serving soporta servir múltiples versiones de un modelo simultáneamente. Esto permite desplegar nuevas versiones sin tiempo de inactividad.

Ejemplo de Estructura de Directorios

/models
    /my_model
        /1
        /2

Agrupamiento

El agrupamiento combina múltiples solicitudes en un solo lote para mejorar el rendimiento.

Habilitando el Agrupamiento

Crea un archivo batching_parameters.txt:

max_batch_size { value: 32 }
batch_timeout_micros { value: 10000 }

Inicia el servidor con el agrupamiento habilitado:

docker run -p 8501:8501 --name=tfserving_my_model --mount type=bind,source=$(pwd)/my_model,target=/models/my_model -e MODEL_NAME=my_model -t tensorflow/serving --enable_batching=true --batching_parameters_file=/models/my_model/batching_parameters.txt

Monitoreo

TensorFlow Serving proporciona capacidades de monitoreo para rastrear el rendimiento y la salud de tus modelos.

Integración con Prometheus

TensorFlow Serving puede exportar métricas en un formato que Prometheus puede recolectar.

Inicia el servidor con el monitoreo habilitado:

docker run -p 8501:8501 -p 8502:8502 --name=tfserving_my_model --mount type=bind,source=$(pwd)/my_model,target=/models/my_model -e MODEL_NAME=my_model -t tensorflow/serving --monitoring_config_file=/models/my_model/monitoring_config.txt

Conclusión

TensorFlow Serving es una herramienta poderosa para desplegar modelos de aprendizaje automático en producción. Soporta versionado de modelos, agrupamiento y monitoreo, haciéndolo adecuado para aplicaciones de aprendizaje automático de alto rendimiento y escalables. Siguiendo los pasos descritos en este tema, puedes configurar TensorFlow Serving, desplegar tus modelos y aprovechar sus características avanzadas para optimizar tus flujos de trabajo de aprendizaje automático.
