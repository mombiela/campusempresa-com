La Ejecución Ansiosa es un entorno de programación imperativa que evalúa operaciones inmediatamente, sin construir gráficos. Esto facilita el inicio con TensorFlow y la depuración de modelos, ya que se comporta más como el código estándar de Python.

Introducción a la Ejecución Ansiosa

  • Programación Imperativa: La Ejecución Ansiosa permite que las operaciones de TensorFlow se ejecuten inmediatamente cuando se llaman desde Python.
  • Facilidad de Depuración: Dado que las operaciones se ejecutan inmediatamente, puedes usar herramientas estándar de depuración de Python como pdb.
  • Desarrollo Interactivo: La Ejecución Ansiosa es particularmente útil en entornos interactivos como los cuadernos de Jupyter.

Habilitando la Ejecución Ansiosa

Por defecto, TensorFlow 2.x tiene la Ejecución Ansiosa habilitada. Sin embargo, si estás usando TensorFlow 1.x, necesitas habilitarla explícitamente.

TensorFlow 2.x

import tensorflow as tf

# La Ejecución Ansiosa está habilitada por defecto en TensorFlow 2.x
print(tf.executing_eagerly())  # Debería imprimir: True

TensorFlow 1.x

import tensorflow as tf

# Habilitar la Ejecución Ansiosa
tf.enable_eager_execution()

print(tf.executing_eagerly())  # Debería imprimir: True

Operaciones Básicas con Ejecución Ansiosa

Con la Ejecución Ansiosa, las operaciones de TensorFlow devuelven valores concretos en lugar de construir un gráfico computacional.

Ejemplo: Operaciones Aritméticas Básicas

import tensorflow as tf

# Definir constantes
a = tf.constant(2)
b = tf.constant(3)

# Realizar operaciones
add = tf.add(a, b)
multiply = tf.multiply(a, b)

print(f"Suma: {add.numpy()}")  # Salida: 5
print(f"Multiplicación: {multiply.numpy()}")  # Salida: 6

Ejemplo: Operaciones de Matrices

import tensorflow as tf

# Definir matrices
matrix1 = tf.constant([[1, 2], [3, 4]])
matrix2 = tf.constant([[5, 6], [7, 8]])

# Realizar multiplicación de matrices
product = tf.matmul(matrix1, matrix2)

print(f"Producto de Matrices:\n{product.numpy()}")
# Salida:
# [[19 22]
#  [43 50]]

Beneficios de la Ejecución Ansiosa

  • Sintaxis Intuitiva: El código se ve y se siente como Python estándar.
  • Ejecución Inmediata: No es necesario compilar un gráfico antes de ejecutar operaciones.
  • Fácil Depuración: Usa directamente las herramientas de depuración de Python.
  • Desarrollo Interactivo: Ideal para entornos interactivos como los cuadernos de Jupyter.

Limitaciones de la Ejecución Ansiosa

  • Rendimiento: La Ejecución Ansiosa puede ser más lenta que la ejecución de gráficos para modelos a gran escala.
  • Despliegue: Algunas herramientas y entornos de despliegue pueden seguir prefiriendo la ejecución de gráficos.

Transición de la Ejecución Ansiosa a la Ejecución de Gráficos

Para aplicaciones críticas en rendimiento, puede que desees convertir tu código de Ejecución Ansiosa a ejecución de gráficos usando tf.function.

Ejemplo: Usando `tf.function`

import tensorflow as tf

@tf.function
def add(a, b):
    return a + b

a = tf.constant(2)
b = tf.constant(3)

result = add(a, b)
print(f"Resultado: {result.numpy()}")  # Salida: 5

Resumen

La Ejecución Ansiosa en TensorFlow proporciona una manera intuitiva e interactiva de desarrollar y depurar modelos. Está habilitada por defecto en TensorFlow 2.x y puede habilitarse explícitamente en TensorFlow 1.x. Aunque ofrece muchos beneficios, incluyendo facilidad de uso y ejecución inmediata, puede no ser adecuada para todas las aplicaciones críticas en rendimiento. Para tales casos, la transición a la ejecución de gráficos usando tf.function puede proporcionar las mejoras de rendimiento necesarias.
