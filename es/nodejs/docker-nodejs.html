Introducción a Docker

  • ¿Qué es Docker?

    • Docker es una plataforma para desarrollar, enviar y ejecutar aplicaciones dentro de contenedores.
    • Los contenedores son ligeros, portátiles y aseguran consistencia a través de diferentes entornos.

  • ¿Por qué usar Docker con Node.js?

    • Simplifica el proceso de configuración.
    • Asegura consistencia a través de entornos de desarrollo, prueba y producción.
    • Facilita la gestión de dependencias y versiones.

Configuración de Docker

  • Instalación de Docker

    • Sigue la guía oficial de instalación de Docker para tu sistema operativo: Instalación de Docker

  • Comandos básicos de Docker

    • docker --version: Verificar la versión de Docker.
    • docker run: Ejecutar un contenedor.
    • docker build: Construir una imagen desde un Dockerfile.
    • docker ps: Listar contenedores en ejecución.
    • docker stop: Detener un contenedor en ejecución.

Creando una aplicación simple de Node.js

  • Aplicación básica de Node.js

    // app.js
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});

  • Package.json

    {
  "name": "docker-nodejs-app",
  "version": "1.0.0",
  "description": "Una aplicación simple de Node.js para demostrar Docker",
  "main": "app.js",
  "scripts": {
    "start": "node app.js"
  },
  "dependencies": {
    "http": "^0.0.1-security"
  }
}

Dockerizando la aplicación de Node.js

  • Creando un Dockerfile

    # Usar la imagen oficial de Node.js desde Docker Hub
FROM node:14

# Crear y cambiar al directorio de la aplicación
WORKDIR /usr/src/app

# Copiar package.json y package-lock.json
COPY package*.json ./

# Instalar dependencias
RUN npm install

# Copiar el resto del código de la aplicación
COPY . .

# Exponer el puerto en el que se ejecuta la aplicación
EXPOSE 3000

# Definir el comando para ejecutar la aplicación
CMD ["node", "app.js"]

  • Construyendo la imagen de Docker

    docker build -t docker-nodejs-app .

  • Ejecutando el contenedor de Docker

    docker run -p 3000:3000 docker-nodejs-app

Gestión de contenedores Docker

  • Ver contenedores en ejecución

    docker ps

  • Detener un contenedor

    docker stop <container_id>

  • Eliminar un contenedor

    docker rm <container_id>

Conceptos avanzados de Docker

  • Docker Compose

    • Docker Compose es una herramienta para definir y ejecutar aplicaciones Docker de múltiples contenedores.

    • docker-compose.yml

      version: '3'
services:
  app:
    build: .
    ports:
      - "3000:3000"

    • Ejecutando Docker Compose

      docker-compose up

  • Volúmenes de Docker

    • Los volúmenes se utilizan para persistir datos generados y utilizados por contenedores Docker.

    • Creando un volumen

      docker volume create my_volume

    • Usando un volumen en Docker Compose

      version: '3'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - my_volume:/usr/src/app

Conclusión

Usar Docker con Node.js simplifica el proceso de desarrollo y despliegue al asegurar consistencia a través de diferentes entornos. Al contenerizar tus aplicaciones de Node.js, puedes gestionar fácilmente las dependencias, escalar tus aplicaciones y mantener una configuración de desarrollo limpia y reproducible. Docker Compose mejora aún más esto al permitirte gestionar aplicaciones de múltiples contenedores con facilidad.
