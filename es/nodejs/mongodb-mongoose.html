Introducción a MongoDB y Mongoose

  • MongoDB: Una base de datos NoSQL que almacena datos en documentos similares a JSON.
  • Mongoose: Una biblioteca de Modelado de Datos de Objetos (ODM) para MongoDB y Node.js, que proporciona una solución basada en esquemas para modelar los datos de tu aplicación.

¿Por qué usar Mongoose?

  • Simplifica la validación de datos.
  • Proporciona un esquema para estructurar tus datos.
  • Ofrece conversión de tipos incorporada, construcción de consultas y ganchos de lógica de negocio.

Configuración de MongoDB y Mongoose

Instalación de MongoDB

  1. Descarga e instala MongoDB desde el sitio web oficial.
  2. Inicia el servidor de MongoDB: 
    mongod

Instalación de Mongoose

  1. Inicializa un proyecto de Node.js si no lo has hecho ya: 
    npm init -y
  2. Instala Mongoose: 
    npm install mongoose

Conexión a MongoDB con Mongoose

Conexión básica

const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/mydatabase', {
  useNewUrlParser: true,
  useUnifiedTopology: true
}).then(() => {
  console.log('Conectado a MongoDB');
}).catch(err => {
  console.error('Error de conexión', err);
});
  • Explicación:
    • mongoose.connect: Conecta al servidor de MongoDB.
    • useNewUrlParser y useUnifiedTopology: Opciones para evitar advertencias de deprecación.

Definición de esquemas y modelos

Creación de un esquema

const { Schema } = mongoose;

const userSchema = new Schema({
  name: String,
  age: Number,
  email: String
});
  • Explicación:
    • Schema: Define la estructura del documento.
    • name, age, email: Campos del documento con sus respectivos tipos de datos.

Creación de un modelo

const User = mongoose.model('User', userSchema);
  • Explicación:
    • model: Compila el esquema en un modelo, una clase con la que construimos documentos.

Operaciones CRUD

Crear

const newUser = new User({
  name: 'John Doe',
  age: 30,
  email: 'john.doe@example.com'
});

newUser.save().then(() => console.log('Usuario guardado'));
  • Explicación:
    • new User: Crea una nueva instancia del modelo User.
    • save: Guarda el documento en la base de datos.

Leer

User.find({ age: { $gte: 18 } }).then(users => {
  console.log(users);
});
  • Explicación:
    • find: Recupera documentos que coinciden con la consulta.

Actualizar

User.updateOne({ name: 'John Doe' }, { age: 31 }).then(() => {
  console.log('Usuario actualizado');
});
  • Explicación:
    • updateOne: Actualiza un único documento que coincida con la consulta.

Eliminar

User.deleteOne({ name: 'John Doe' }).then(() => {
  console.log('Usuario eliminado');
});
  • Explicación:
    • deleteOne: Elimina un único documento que coincida con la consulta.

Temas avanzados

Middleware

  • Pre Middleware: Se ejecuta antes de un cierto evento.

    userSchema.pre('save', function(next) {
  console.log('Antes de guardar');
  next();
});

  • Post Middleware: Se ejecuta después de un cierto evento.

    userSchema.post('save', function(doc) {
  console.log('Después de guardar', doc);
});

Virtuales

  • Propiedades virtuales: Campos que no se almacenan en MongoDB pero que pueden usarse como cualquier otro campo. 
    userSchema.virtual('fullName').get(function() {
  return `${this.firstName} ${this.lastName}`;
});

Población

  • Poblar referencias: Reemplaza las rutas especificadas en el documento con documento(s) de otra(s) colección(es). 
    const postSchema = new Schema({
  title: String,
  author: { type: Schema.Types.ObjectId, ref: 'User' }
});

const Post = mongoose.model('Post', postSchema);

Post.find().populate('author').then(posts => {
  console.log(posts);
});

Conclusión

  • Resumen:
    • Mongoose simplifica las interacciones con MongoDB proporcionando una solución basada en esquemas.
    • Las operaciones CRUD básicas son sencillas con Mongoose.
    • Funcionalidades avanzadas como middleware, virtuales y población mejoran la funcionalidad y flexibilidad de tu aplicación.

Al dominar Mongoose, puedes gestionar eficientemente tus datos en MongoDB y construir aplicaciones robustas con Node.js.
