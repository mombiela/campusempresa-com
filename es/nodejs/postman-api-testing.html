Usando Postman para Pruebas de API
Despliegue de Aplicaciones Node.js ►

Introducción a Postman

Postman es una herramienta popular utilizada para pruebas de API. Permite a los desarrolladores enviar solicitudes HTTP a APIs, inspeccionar respuestas y automatizar flujos de trabajo de pruebas. Esta sección cubrirá los conceptos básicos de Postman y cómo puede ser utilizado junto con aplicaciones Node.js.

Conceptos Clave

  • Métodos HTTP: GET, POST, PUT, DELETE, etc.
  • Endpoints: URLs donde las APIs son accesibles.
  • Solicitudes y Respuestas: Enviar datos a y recibir datos de APIs.
  • Colecciones: Agrupación de solicitudes de API relacionadas.
  • Variables de Entorno: Almacenamiento y reutilización de valores en diferentes solicitudes.

Configuración de Postman

Para comenzar con Postman, sigue estos pasos:

  1. Descargar e Instalar Postman: Visita el sitio web de Postman y descarga la versión apropiada para tu sistema operativo.
  2. Crear una Cuenta: Regístrate para obtener una cuenta gratuita para guardar tu trabajo y acceder a funciones adicionales.
  3. Crear una Nueva Solicitud: Abre Postman y haz clic en el botón "New" para crear una nueva solicitud.

Realizando tu Primera Solicitud de API

Hagamos una simple solicitud GET a una API pública.

  1. Abre Postman.
  2. Crear una Nueva Solicitud:
    • Haz clic en "New" > "Request".
    • Nombra tu solicitud y agrégala a una colección si lo deseas.
  3. Configurar el Tipo de Solicitud y URL:
    • Selecciona "GET" del menú desplegable.
    • Ingresa la URL: https://jsonplaceholder.typicode.com/posts/1.
  4. Enviar la Solicitud:
    • Haz clic en el botón "Send".
    • Inspecciona la respuesta en la sección inferior de la interfaz de Postman.

Ejemplo de Respuesta

{
  "userId": 1,
  "id": 1,
  "title": "sunt aut facere repellat provident occaecati excepturi optio reprehenderit",
  "body": "quia et suscipit\nsuscipit..."
}

Probando una API de Node.js con Postman

Ahora, probemos una API de Node.js. Supongamos que tienes un servidor Express simple ejecutándose localmente.

Ejemplo de Servidor Node.js

const express = require('express');
const app = express();
const port = 3000;

app.use(express.json());

app.get('/api/users', (req, res) => {
  res.json([{ id: 1, name: 'John Doe' }]);
});

app.post('/api/users', (req, res) => {
  const newUser = req.body;
  res.status(201).json(newUser);
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});

Probando el Endpoint GET

  1. Configurar el Tipo de Solicitud y URL:
    • Selecciona "GET" del menú desplegable.
    • Ingresa la URL: http://localhost:3000/api/users.
  2. Enviar la Solicitud:
    • Haz clic en el botón "Send".
    • Inspecciona la respuesta.

Probando el Endpoint POST

  1. Configurar el Tipo de Solicitud y URL:
    • Selecciona "POST" del menú desplegable.
    • Ingresa la URL: http://localhost:3000/api/users.
  2. Configurar el Cuerpo de la Solicitud:
    • Haz clic en la pestaña "Body".
    • Selecciona "raw" y elige "JSON" del menú desplegable.
    • Ingresa el siguiente JSON: 
      {
  "id": 2,
  "name": "Jane Doe"
}
  3. Enviar la Solicitud:
    • Haz clic en el botón "Send".
    • Inspecciona la respuesta.

Usando Colecciones y Variables de Entorno

Las colecciones y variables de entorno de Postman pueden ayudar a gestionar y organizar tus pruebas de API.

Creando una Colección

  1. Crear una Nueva Colección:
    • Haz clic en "New" > "Collection".
    • Nombra tu colección y agrega una descripción si lo deseas.
  2. Agregar Solicitudes a la Colección:
    • Arrastra y suelta solicitudes existentes en la colección o crea nuevas dentro de ella.

Usando Variables de Entorno

  1. Crear un Entorno:
    • Haz clic en el ícono de engranaje en la esquina superior derecha y selecciona "Manage Environments".
    • Haz clic en "Add" y nombra tu entorno.
  2. Agregar Variables:
    • Agrega pares clave-valor, por ejemplo, baseUrl = http://localhost:3000.
  3. Usar Variables en Solicitudes:
    • Reemplaza partes de tu URL con variables, por ejemplo, {{baseUrl}}/api/users.

Automatizando Pruebas con Postman

Postman te permite escribir pruebas usando JavaScript para automatizar tus pruebas de API.

Escribiendo Pruebas

  1. Abrir la Pestaña de Pruebas:
    • Haz clic en la pestaña "Tests" en tu solicitud.
  2. Agregar Scripts de Prueba:
    • Escribe JavaScript para validar respuestas.
    pm.test("Status code is 200", function () {
  pm.response.to.have.status(200);
});

pm.test("Response has userId", function () {
  const jsonData = pm.response.json();
  pm.expect(jsonData).to.have.property('userId');
});

Conclusión

Postman es una herramienta poderosa para probar APIs, y se integra perfectamente con aplicaciones Node.js. Al entender cómo configurar solicitudes, usar colecciones y variables de entorno, y automatizar pruebas, puedes mejorar significativamente tu flujo de trabajo de desarrollo y pruebas de API. Practica usando Postman con tus proyectos Node.js para volverte competente en pruebas de API.
