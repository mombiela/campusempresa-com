El manejo de errores es una parte crucial para construir aplicaciones robustas. En esta sección, cubriremos cómo manejar errores en Express, un popular framework web para Node.js. Comenzaremos con lo básico y avanzaremos hacia técnicas más avanzadas.

Introducción al Manejo de Errores

El manejo de errores en Express implica capturar y gestionar errores que ocurren durante la ejecución de tu aplicación. Esto asegura que tu aplicación pueda manejar situaciones inesperadas de manera elegante y proporcionar retroalimentación significativa a los usuarios.

Conceptos Clave

  • Errores Sincrónicos: Errores que ocurren durante la ejecución de código sincrónico.
  • Errores Asincrónicos: Errores que ocurren en código asincrónico, como callbacks, promesas o async/await.
  • Middleware: Funciones que tienen acceso a los objetos de solicitud y respuesta y pueden modificarlos o finalizar el ciclo de solicitud-respuesta.

Manejo Básico de Errores

Errores Sincrónicos

En Express, los errores sincrónicos pueden ser capturados usando bloques try-catch. Aquí hay un ejemplo:

const express = require('express');
const app = express();

app.get('/', (req, res) => {
  try {
    // Simular un error
    throw new Error('¡Algo salió mal!');
  } catch (err) {
    res.status(500).send('Error Interno del Servidor');
  }
});

app.listen(3000, () => {
  console.log('El servidor está corriendo en el puerto 3000');
});

Errores Asincrónicos

Para el código asincrónico, puedes usar promesas o async/await junto con bloques try-catch:

app.get('/async', async (req, res) => {
  try {
    // Simular un error asincrónico
    await Promise.reject(new Error('¡Error asincrónico!'));
  } catch (err) {
    res.status(500).send('Error Interno del Servidor');
  }
});

Middleware de Manejo de Errores

Express proporciona un tipo especial de middleware para el manejo de errores. Este middleware tiene cuatro parámetros: err, req, res, y next.

Ejemplo

const express = require('express');
const app = express();

// Ruta normal
app.get('/', (req, res) => {
  throw new Error('¡Ups!');
});

// Middleware de manejo de errores
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('¡Algo se rompió!');
});

app.listen(3000, () => {
  console.log('El servidor está corriendo en el puerto 3000');
});

Explicación

  • err: El objeto de error.
  • req: El objeto de solicitud.
  • res: El objeto de respuesta.
  • next: La siguiente función de middleware en la pila.

Manejo Avanzado de Errores

Clases de Errores Personalizadas

Crear clases de errores personalizadas puede ayudarte a gestionar diferentes tipos de errores de manera más efectiva.

class NotFoundError extends Error {
  constructor(message) {
    super(message);
    this.name = 'NotFoundError';
    this.statusCode = 404;
  }
}

app.get('/custom-error', (req, res, next) => {
  next(new NotFoundError('Este recurso no fue encontrado'));
});

app.use((err, req, res, next) => {
  res.status(err.statusCode || 500).send(err.message);
});

Manejo Centralizado de Errores

Centralizar el manejo de errores puede hacer que tu código sea más limpio y fácil de mantener.

const errorHandler = (err, req, res, next) => {
  console.error(err.stack);
  res.status(err.statusCode || 500).send(err.message);
};

app.use(errorHandler);

Conclusión

El manejo de errores en Express es esencial para construir aplicaciones confiables. Al entender e implementar tanto técnicas básicas como avanzadas de manejo de errores, puedes asegurar que tu aplicación pueda manejar situaciones inesperadas de manera elegante y proporcionar retroalimentación significativa a los usuarios.

Resumen

  • Errores Sincrónicos: Usa bloques try-catch.
  • Errores Asincrónicos: Usa promesas o async/await con bloques try-catch.
  • Middleware de Manejo de Errores: Usa middleware con cuatro parámetros (err, req, res, next).
  • Clases de Errores Personalizadas: Crea clases de errores personalizadas para tipos específicos de errores.
  • Manejo Centralizado de Errores: Usa un manejador de errores centralizado para gestionar errores en un solo lugar.

Al dominar estas técnicas, estarás bien equipado para manejar errores de manera efectiva en tus aplicaciones Express.
