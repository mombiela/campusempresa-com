Introducción a las Promesas

Las promesas son una forma moderna de manejar operaciones asíncronas en JavaScript, proporcionando un enfoque más legible y manejable en comparación con los callbacks tradicionales.

Conceptos Clave

  • Operaciones Asíncronas: Operaciones que no bloquean la ejecución del programa.
  • Estados de la Promesa:
    • Pendiente: Estado inicial, ni cumplida ni rechazada.
    • Cumplida: Operación completada con éxito.
    • Rechazada: Operación fallida.
  • Métodos de la Promesa:
    • .then(): Maneja el cumplimiento.
    • .catch(): Maneja el rechazo.
    • .finally(): Se ejecuta independientemente del resultado.

Ejemplo Básico

const myPromise = new Promise((resolve, reject) => {
  let success = true; // Simular éxito o fracaso
  if (success) {
    resolve('¡La operación fue exitosa!');
  } else {
    reject('La operación falló.');
  }
});

myPromise
  .then((message) => {
    console.log(message); // Salida: ¡La operación fue exitosa!
  })
  .catch((error) => {
    console.error(error);
  });

Creación y Uso de Promesas

Creación de una Promesa

Una Promesa se crea usando el constructor Promise, que toma una función con dos parámetros: resolve y reject.

const promise = new Promise((resolve, reject) => {
  // Operación asíncrona
  setTimeout(() => {
    resolve('¡Datos obtenidos con éxito!');
  }, 2000);
});

Manejo de Promesas con `.then()`, `.catch()`, y `.finally()`

  • .then(): Se usa para manejar el valor resuelto de la Promesa.
  • .catch(): Se usa para manejar cualquier error o rechazo.
  • .finally(): Ejecuta código después de que la Promesa se ha resuelto, independientemente de su resultado.
promise
  .then((data) => {
    console.log(data); // Salida: ¡Datos obtenidos con éxito!
  })
  .catch((error) => {
    console.error(error);
  })
  .finally(() => {
    console.log('Operación completada.');
  });

Encadenamiento de Promesas

Las promesas pueden encadenarse para manejar múltiples operaciones asíncronas en secuencia.

Ejemplo de Encadenamiento

const fetchData = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Datos obtenidos');
  }, 1000);
});

fetchData
  .then((data) => {
    console.log(data); // Salida: Datos obtenidos
    return 'Procesando datos';
  })
  .then((processMessage) => {
    console.log(processMessage); // Salida: Procesando datos
    return 'Datos procesados';
  })
  .then((finalMessage) => {
    console.log(finalMessage); // Salida: Datos procesados
  })
  .catch((error) => {
    console.error(error);
  });

Patrones Avanzados de Promesas

Promise.all

Promise.all se usa para manejar múltiples Promesas concurrentemente. Se resuelve cuando todas las Promesas en el array se han resuelto.

const promise1 = Promise.resolve('Promesa 1 resuelta');
const promise2 = Promise.resolve('Promesa 2 resuelta');

Promise.all([promise1, promise2])
  .then((results) => {
    console.log(results); // Salida: ['Promesa 1 resuelta', 'Promesa 2 resuelta']
  })
  .catch((error) => {
    console.error(error);
  });

Promise.race

Promise.race se resuelve o rechaza tan pronto como una de las Promesas en el array se resuelva o rechace.

const promise1 = new Promise((resolve) => setTimeout(resolve, 500, 'Primero'));
const promise2 = new Promise((resolve) => setTimeout(resolve, 100, 'Segundo'));

Promise.race([promise1, promise2])
  .then((result) => {
    console.log(result); // Salida: Segundo
  })
  .catch((error) => {
    console.error(error);
  });

Manejo de Errores en Promesas

El manejo adecuado de errores es crucial cuando se trabaja con Promesas para asegurar que los errores sean capturados y gestionados apropiadamente.

Ejemplo de Manejo de Errores

const faultyPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    reject('¡Algo salió mal!');
  }, 1000);
});

faultyPromise
  .then((data) => {
    console.log(data);
  })
  .catch((error) => {
    console.error(error); // Salida: ¡Algo salió mal!
  });

Conclusión

Las promesas proporcionan una forma poderosa y flexible de manejar operaciones asíncronas en Node.js. Al entender y utilizar las Promesas, los desarrolladores pueden escribir código más limpio, legible y mantenible. Los puntos clave incluyen:

  • Entender los estados y métodos de las Promesas.
  • Crear y usar Promesas efectivamente.
  • Encadenar Promesas para operaciones secuenciales.
  • Usar patrones avanzados como Promise.all y Promise.race.
  • Manejar adecuadamente los errores en las Promesas.

Con estos conceptos, puedes manejar con confianza las operaciones asíncronas en tus aplicaciones Node.js.
