El middleware es un concepto fundamental en Node.js, especialmente cuando se trabaja con frameworks como Express.js. Las funciones de middleware son funciones que tienen acceso al objeto de solicitud (req), al objeto de respuesta (res) y a la siguiente función de middleware en el ciclo de solicitud-respuesta de la aplicación. El middleware puede ejecutar código, modificar los objetos de solicitud y respuesta, finalizar el ciclo de solicitud-respuesta y llamar a la siguiente función de middleware en la pila.

¿Qué es el Middleware?

  • Las funciones de middleware son funciones que tienen acceso al objeto de solicitud (req), al objeto de respuesta (res) y a la siguiente función de middleware en el ciclo de solicitud-respuesta de la aplicación.
  • El middleware puede realizar las siguientes tareas:
    • Ejecutar cualquier código.
    • Hacer cambios en los objetos de solicitud y respuesta.
    • Finalizar el ciclo de solicitud-respuesta.
    • Llamar a la siguiente función de middleware en la pila.

Tipos de Middleware

  1. Middleware a nivel de aplicación
  2. Middleware a nivel de enrutador
  3. Middleware de manejo de errores
  4. Middleware incorporado
  5. Middleware de terceros

Middleware a nivel de aplicación

El middleware a nivel de aplicación está vinculado a una instancia del objeto de la aplicación usando app.use() y app.METHOD().

const express = require('express');
const app = express();

// Middleware a nivel de aplicación
app.use((req, res, next) => {
  console.log('Hora:', Date.now());
  next();
});

app.get('/', (req, res) => {
  res.send('¡Hola Mundo!');
});

app.listen(3000, () => {
  console.log('El servidor está corriendo en el puerto 3000');
});

Middleware a nivel de enrutador

El middleware a nivel de enrutador funciona de la misma manera que el middleware a nivel de aplicación, excepto que está vinculado a una instancia de express.Router().

const express = require('express');
const app = express();
const router = express.Router();

// Middleware a nivel de enrutador
router.use((req, res, next) => {
  console.log('URL de la solicitud:', req.originalUrl);
  next();
});

router.get('/', (req, res) => {
  res.send('¡Hola desde el Enrutador!');
});

app.use('/router', router);

app.listen(3000, () => {
  console.log('El servidor está corriendo en el puerto 3000');
});

Middleware de manejo de errores

El middleware de manejo de errores se define de la misma manera que otras funciones de middleware, excepto que tiene cuatro argumentos en lugar de tres: (err, req, res, next).

const express = require('express');
const app = express();

// Middleware de manejo de errores
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('¡Algo se rompió!');
});

app.get('/', (req, res) => {
  throw new Error('ROTO'); // Express lo capturará por sí solo.
});

app.listen(3000, () => {
  console.log('El servidor está corriendo en el puerto 3000');
});

Middleware incorporado

Express viene con algunas funciones de middleware incorporadas, como express.static, express.json y express.urlencoded.

const express = require('express');
const app = express();

// Middleware incorporado
app.use(express.json()); // Para analizar application/json
app.use(express.urlencoded({ extended: true })); // Para analizar application/x-www-form-urlencoded

app.post('/data', (req, res) => {
  res.send(req.body);
});

app.listen(3000, () => {
  console.log('El servidor está corriendo en el puerto 3000');
});

Middleware de terceros

El middleware de terceros se puede instalar a través de npm y luego requerir en tu aplicación. Middleware de terceros popular incluye morgan, body-parser y cookie-parser.

const express = require('express');
const morgan = require('morgan');
const app = express();

// Middleware de terceros
app.use(morgan('tiny'));

app.get('/', (req, res) => {
  res.send('¡Hola Mundo!');
});

app.listen(3000, () => {
  console.log('El servidor está corriendo en el puerto 3000');
});

Conclusión

El middleware es una característica poderosa y flexible en Node.js que te permite manejar varios aspectos del procesamiento de solicitudes y respuestas. Comprender cómo usar diferentes tipos de middleware de manera efectiva puede mejorar en gran medida la funcionalidad y mantenibilidad de tus aplicaciones Node.js. Ya sea que estés registrando solicitudes, analizando datos entrantes, manejando errores o sirviendo archivos estáticos, el middleware proporciona una forma estructurada de gestionar estas tareas.
