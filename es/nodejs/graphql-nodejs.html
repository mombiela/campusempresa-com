Introducción a GraphQL

GraphQL es un lenguaje de consulta para APIs y un tiempo de ejecución para ejecutar esas consultas utilizando un sistema de tipos que defines para tus datos. Proporciona una alternativa más eficiente, poderosa y flexible a REST.

  • Conceptos Clave:
    • Esquema: Define la estructura de tu API, incluyendo tipos y relaciones.
    • Consulta: Solicita datos específicos y da forma a la respuesta.
    • Mutación: Modifica datos en el servidor.
    • Resolver: Funciones que manejan la obtención de datos para una consulta o mutación.

Configuración de un Proyecto Node.js con GraphQL

Para comenzar con GraphQL en Node.js, necesitas configurar un proyecto básico de Node.js e instalar los paquetes necesarios.

Configuración Paso a Paso

  1. Inicializar un Proyecto Node.js:

    mkdir graphql-nodejs
cd graphql-nodejs
npm init -y

  2. Instalar Paquetes Requeridos:

    npm install express express-graphql graphql

  3. Crear un Servidor Básico:

    const express = require('express');
const { graphqlHTTP } = require('express-graphql');
const { buildSchema } = require('graphql');

// Definir un esquema
const schema = buildSchema(`
  type Query {
    hello: String
  }
`);

// Definir un resolver
const root = {
  hello: () => {
    return '¡Hola mundo!';
  },
};

const app = express();
app.use('/graphql', graphqlHTTP({
  schema: schema,
  rootValue: root,
  graphiql: true,
}));

app.listen(4000, () => console.log('Servidor corriendo en http://localhost:4000/graphql'));

Definiendo un Esquema GraphQL

El esquema es el núcleo de cualquier servidor GraphQL. Define los tipos y las relaciones entre ellos.

Ejemplo de Esquema

type Query {
  book(id: ID!): Book
  books: [Book]
}

type Book {
  id: ID!
  title: String!
  author: Author!
}

type Author {
  id: ID!
  name: String!
  books: [Book]
}
  • Tipos:
    • Query: Punto de entrada para operaciones de lectura.
    • Book: Representa un libro con un ID, título y autor.
    • Author: Representa un autor con un ID, nombre y lista de libros.

Escribiendo Resolvers

Los resolvers son funciones que manejan la lógica para obtener los datos definidos en tu esquema.

Ejemplo de Resolvers

const books = [
  { id: '1', title: '1984', authorId: '1' },
  { id: '2', title: 'Un Mundo Feliz', authorId: '2' },
];

const authors = [
  { id: '1', name: 'George Orwell' },
  { id: '2', name: 'Aldous Huxley' },
];

const resolvers = {
  Query: {
    book: (_, { id }) => books.find(book => book.id === id),
    books: () => books,
  },
  Book: {
    author: (book) => authors.find(author => author.id === book.authorId),
  },
  Author: {
    books: (author) => books.filter(book => book.authorId === author.id),
  },
};

Integración con una Base de Datos

Para hacer tu API GraphQL más dinámica, puedes integrarla con una base de datos.

Ejemplo con MongoDB

  1. Instalar Mongoose:

    npm install mongoose

  2. Conectar a MongoDB:

    const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/graphql', { useNewUrlParser: true, useUnifiedTopology: true });

const BookSchema = new mongoose.Schema({
  title: String,
  authorId: String,
});

const Book = mongoose.model('Book', BookSchema);

const AuthorSchema = new mongoose.Schema({
  name: String,
});

const Author = mongoose.model('Author', AuthorSchema);

  3. Actualizar Resolvers para Usar Mongoose:

    const resolvers = {
  Query: {
    book: async (_, { id }) => await Book.findById(id),
    books: async () => await Book.find(),
  },
  Book: {
    author: async (book) => await Author.findById(book.authorId),
  },
  Author: {
    books: async (author) => await Book.find({ authorId: author.id }),
  },
};

Temas Avanzados

Autenticación y Autorización

Implementar autenticación y autorización en GraphQL se puede hacer usando middleware.

Ejemplo de Middleware

const jwt = require('jsonwebtoken');

const authMiddleware = (req, res, next) => {
  const token = req.headers.authorization;
  if (token) {
    try {
      const user = jwt.verify(token, 'your_secret_key');
      req.user = user;
    } catch (err) {
      console.error(err);
    }
  }
  next();
};

app.use(authMiddleware);

Suscripciones

Las suscripciones de GraphQL te permiten suscribirte a actualizaciones en tiempo real.

Ejemplo con Apollo Server

  1. Instalar Apollo Server y WebSocket:

    npm install apollo-server apollo-server-express graphql-subscriptions

  2. Configurar Suscripciones:

    const { ApolloServer, gql, PubSub } = require('apollo-server');
const pubsub = new PubSub();

const typeDefs = gql`
  type Query {
    _: Boolean
  }

  type Subscription {
    bookAdded: Book
  }

  type Book {
    id: ID!
    title: String!
    author: Author!
  }

  type Author {
    id: ID!
    name: String!
    books: [Book]
  }
`;

const resolvers = {
  Subscription: {
    bookAdded: {
      subscribe: () => pubsub.asyncIterator(['BOOK_ADDED']),
    },
  },
};

const server = new ApolloServer({ typeDefs, resolvers });

server.listen().then(({ url }) => {
  console.log(`🚀 Servidor listo en ${url}`);
});

Conclusión

GraphQL con Node.js proporciona una forma poderosa y flexible de construir APIs. Al comprender los conceptos clave, configurar un servidor básico, definir esquemas, escribir resolvers e integrar con una base de datos, puedes crear APIs robustas y eficientes. Los temas avanzados como autenticación, autorización y suscripciones mejoran aún más las capacidades de tu servidor GraphQL.
