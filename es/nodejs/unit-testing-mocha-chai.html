Introducción a las Pruebas Unitarias

  • Definición: Las pruebas unitarias implican probar unidades individuales o componentes de un software para asegurar que funcionen como se espera.
  • Importancia: Ayuda a identificar errores temprano, simplifica la integración y asegura la fiabilidad del código.

Configuración de Mocha y Chai

  • Mocha: Un marco de pruebas de JavaScript rico en características que se ejecuta en Node.js, haciendo que las pruebas asincrónicas sean simples y divertidas.
  • Chai: Una biblioteca de aserciones BDD / TDD para Node.js que se puede combinar con cualquier marco de pruebas de JavaScript.

Instalación de Mocha y Chai

Para comenzar, necesitas instalar Mocha y Chai usando npm:

npm install --save-dev mocha chai

Configuración de Mocha

Crea un directorio test en la raíz de tu proyecto y añade una configuración en tu package.json:

{
  "scripts": {
    "test": "mocha"
  }
}

Escribiendo Tu Primera Prueba

Estructura Básica de la Prueba

Crea un archivo llamado test.js dentro del directorio test:

const assert = require('chai').assert;

describe('Array', function() {
  it('should start empty', function() {
    let arr = [];
    assert.equal(arr.length, 0);
  });
});
  • describe(): Agrupa pruebas relacionadas.
  • it(): Define un caso de prueba único.
  • assert: Proporciona métodos de aserción para verificar los resultados de las pruebas.

Ejecutando Pruebas

Ejecuta tus pruebas usando el siguiente comando:

npm test

Pruebas Avanzadas con Mocha y Chai

Pruebas Asincrónicas

Mocha soporta pruebas asincrónicas usando callbacks, promesas o async/await.

Usando Callbacks

describe('Async Test', function() {
  it('should complete asynchronously', function(done) {
    setTimeout(function() {
      assert.isTrue(true);
      done();
    }, 1000);
  });
});

Usando Promesas

describe('Async Test', function() {
  it('should complete with a promise', function() {
    return new Promise((resolve) => {
      setTimeout(function() {
        assert.isTrue(true);
        resolve();
      }, 1000);
    });
  });
});

Usando Async/Await

describe('Async Test', function() {
  it('should complete with async/await', async function() {
    await new Promise((resolve) => setTimeout(resolve, 1000));
    assert.isTrue(true);
  });
});

Usando Aserciones de Chai

Chai proporciona tres estilos de aserciones: Assert, Expect y Should.

Estilo Assert

assert.equal(3, 3, '3 equals 3');

Estilo Expect

const expect = require('chai').expect;
expect(3).to.equal(3);

Estilo Should

const should = require('chai').should();
(3).should.equal(3);

Probando APIs HTTP con Chai HTTP

Instala chai-http para probar endpoints HTTP:

npm install --save-dev chai-http

Ejemplo de Prueba

const chai = require('chai');
const chaiHttp = require('chai-http');
const app = require('../app'); // Tu aplicación Express
chai.use(chaiHttp);

describe('GET /api', function() {
  it('should return status 200', function(done) {
    chai.request(app)
      .get('/api')
      .end(function(err, res) {
        chai.assert.equal(res.status, 200);
        done();
      });
  });
});

Mocking y Stubbing con Sinon

Introducción a Sinon

Sinon es una biblioteca utilizada para crear espías, mocks y stubs.

Instalación de Sinon

npm install --save-dev sinon

Ejemplo de Stubbing

const sinon = require('sinon');
const myModule = require('../myModule');

describe('myModule', function() {
  it('should call the callback', function() {
    const callback = sinon.stub();
    myModule.doSomething(callback);
    sinon.assert.calledOnce(callback);
  });
});

Conclusión

Las pruebas unitarias son una parte crucial del proceso de desarrollo, asegurando que los componentes individuales de tu aplicación funcionen como se espera. Mocha y Chai proporcionan una combinación poderosa para escribir y ejecutar pruebas en Node.js. Al dominar estas herramientas, puedes escribir código confiable, mantenible y robusto. Recuerda:

  • Comienza con pruebas simples y gradualmente avanza a escenarios más complejos.
  • Usa técnicas de pruebas asincrónicas según sea necesario.
  • Aprovecha los diversos estilos de aserciones de Chai para hacer tus pruebas más legibles.
  • Utiliza Sinon para mocking y stubbing avanzados.
