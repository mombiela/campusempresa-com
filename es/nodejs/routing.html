Enrutamiento en Node.js
Servir Archivos Estáticos ►

El enrutamiento es un concepto fundamental en el desarrollo web, que te permite definir cómo responde tu aplicación a diferentes solicitudes HTTP. En Node.js, el enrutamiento se maneja típicamente usando frameworks como Express.js. Esta sección te guiará desde los conceptos básicos del enrutamiento hasta técnicas más avanzadas.

Introducción al Enrutamiento

El enrutamiento es el proceso de determinar cómo una aplicación responde a una solicitud del cliente a un endpoint particular, que es un URI (o ruta) y un método de solicitud HTTP específico (GET, POST, etc.).

Conceptos Clave

  • Endpoint: Una combinación específica de ruta URL y método HTTP.
  • Métodos HTTP: Métodos comunes incluyen GET, POST, PUT, DELETE, etc.
  • Manejador de Rutas: Una función que se ejecuta cuando se coincide con una ruta específica.

Configuración de Express.js

Para manejar el enrutamiento en Node.js, utilizaremos el framework Express.js. Primero, configuremos una aplicación básica de Express.

Instalación

npm install express

Configuración Básica

const express = require('express');
const app = express();
const port = 3000;

app.listen(port, () => {
  console.log(`Server is running on http://localhost:${port}`);
});

Enrutamiento Básico

Definiendo Rutas

Las rutas se definen usando métodos del objeto app que corresponden a los métodos HTTP. Por ejemplo:

app.get('/', (req, res) => {
  res.send('Hello, World!');
});

app.post('/submit', (req, res) => {
  res.send('Form submitted!');
});

Parámetros de Ruta

Los parámetros de ruta son segmentos de URL nombrados que se utilizan para capturar valores en posiciones específicas en la URL.

app.get('/user/:id', (req, res) => {
  const userId = req.params.id;
  res.send(`User ID: ${userId}`);
});

Parámetros de Consulta

Los parámetros de consulta se utilizan para pasar datos al servidor a través de la URL.

app.get('/search', (req, res) => {
  const query = req.query.q;
  res.send(`Search query: ${query}`);
});

Enrutamiento Avanzado

Middleware

Las funciones middleware son funciones que tienen acceso al objeto de solicitud (req), al objeto de respuesta (res) y a la siguiente función middleware en el ciclo de solicitud-respuesta de la aplicación.

const logger = (req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

app.use(logger);

Grupos de Rutas

Los grupos de rutas te permiten agrupar rutas que comparten un prefijo de ruta común.

const router = express.Router();

router.get('/profile', (req, res) => {
  res.send('User profile');
});

router.get('/settings', (req, res) => {
  res.send('User settings');
});

app.use('/user', router);

Manejo de Errores

Las funciones middleware de manejo de errores se definen de la misma manera que otras funciones middleware, excepto que tienen cuatro argumentos en lugar de tres: err, req, res y next.

app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something broke!');
});

Resumen

El enrutamiento es una parte crucial de cualquier aplicación web, definiendo cómo responde tu aplicación a varias solicitudes HTTP. En Node.js, Express.js proporciona una forma simple y poderosa de manejar el enrutamiento. Cubrimos:

  • Enrutamiento básico con diferentes métodos HTTP.
  • Uso de parámetros de ruta y de consulta.
  • Técnicas avanzadas como middleware, grupos de rutas y manejo de errores.

Al dominar estos conceptos, puedes construir aplicaciones web robustas y escalables con Node.js y Express.js.
