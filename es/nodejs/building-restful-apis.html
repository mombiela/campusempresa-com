Introducción a las APIs RESTful

  • Definición: REST (Transferencia de Estado Representacional) es un estilo arquitectónico para diseñar aplicaciones en red.
  • Principios:
    • Sin estado
    • Arquitectura Cliente-Servidor
    • Interfaz Uniforme
    • Sistema en Capas
    • Cacheabilidad
    • Código bajo demanda (opcional)

Configuración de tu Entorno

  • Instalación de Node.js: Asegúrate de que Node.js y npm (Node Package Manager) estén instalados.
  • Inicialización del Proyecto: 
    mkdir rest-api-example
cd rest-api-example
npm init -y
  • Instalación de Dependencias:
    • Express.js: Un marco de aplicación web minimalista y flexible para Node.js.
    • Nodemon: Una herramienta que ayuda a desarrollar aplicaciones Node.js reiniciando automáticamente la aplicación cuando se detectan cambios en los archivos.
    npm install express
npm install --save-dev nodemon

Creación de un Servidor Básico con Express

  • Configuración Básica del Servidor: 
    const express = require('express');
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
  res.send('¡Hola Mundo!');
});

app.listen(PORT, () => {
  console.log(`El servidor está corriendo en http://localhost:${PORT}`);
});
  • Ejecutando el Servidor: 
    node index.js

Rutas RESTful

  • Operaciones CRUD:
    • Crear: POST
    • Leer: GET
    • Actualizar: PUT
    • Eliminar: DELETE
  • Rutas de Ejemplo: 
    app.post('/items', (req, res) => {
  // Código para crear un ítem
});

app.get('/items', (req, res) => {
  // Código para obtener todos los ítems
});

app.get('/items/:id', (req, res) => {
  // Código para obtener un ítem por id
});

app.put('/items/:id', (req, res) => {
  // Código para actualizar un ítem por id
});

app.delete('/items/:id', (req, res) => {
  // Código para eliminar un ítem por id
});

Middleware y Análisis del Cuerpo

  • Middleware: Funciones que tienen acceso al objeto de solicitud, objeto de respuesta y la siguiente función de middleware.
  • Body Parser: Middleware para analizar los cuerpos de las solicitudes entrantes. 
    npm install body-parser

    const bodyParser = require('body-parser');
app.use(bodyParser.json());

Conexión a una Base de Datos

  • MongoDB y Mongoose:
    • Instalación: 
      npm install mongoose
    • Conexión a MongoDB: 
      const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost:27017/rest-api-example', { useNewUrlParser: true, useUnifiedTopology: true });

Definición de Esquemas y Modelos de Mongoose

  • Definición del Esquema: 
    const itemSchema = new mongoose.Schema({
  name: String,
  quantity: Number,
  price: Number
});

const Item = mongoose.model('Item', itemSchema);

Implementación de Operaciones CRUD con Mongoose

  • Operación de Creación: 
    app.post('/items', async (req, res) => {
  const newItem = new Item(req.body);
  try {
    const savedItem = await newItem.save();
    res.status(201).send(savedItem);
  } catch (error) {
    res.status(400).send(error);
  }
});
  • Operaciones de Lectura: 
    app.get('/items', async (req, res) => {
  try {
    const items = await Item.find();
    res.status(200).send(items);
  } catch (error) {
    res.status(500).send(error);
  }
});

app.get('/items/:id', async (req, res) => {
  try {
    const item = await Item.findById(req.params.id);
    if (!item) {
      return res.status(404).send();
    }
    res.status(200).send(item);
  } catch (error) {
    res.status(500).send(error);
  }
});
  • Operación de Actualización: 
    app.put('/items/:id', async (req, res) => {
  try {
    const item = await Item.findByIdAndUpdate(req.params.id, req.body, { new: true, runValidators: true });
    if (!item) {
      return res.status(404).send();
    }
    res.status(200).send(item);
  } catch (error) {
    res.status(400).send(error);
  }
});
  • Operación de Eliminación: 
    app.delete('/items/:id', async (req, res) => {
  try {
    const item = await Item.findByIdAndDelete(req.params.id);
    if (!item) {
      return res.status(404).send();
    }
    res.status(200).send(item);
  } catch (error) {
    res.status(500).send(error);
  }
});

Manejo de Errores

  • Middleware Centralizado de Manejo de Errores: 
    app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('¡Algo salió mal!');
});

Autenticación y Autorización

  • JWT (JSON Web Tokens):
    • Instalación: 
      npm install jsonwebtoken
    • Generación de Tokens: 
      const jwt = require('jsonwebtoken');
const token = jwt.sign({ _id: user._id }, 'secretKey', { expiresIn: '1h' });
    • Verificación de Tokens: 
      const auth = (req, res, next) => {
  const token = req.header('Authorization').replace('Bearer ', '');
  try {
    const decoded = jwt.verify(token, 'secretKey');
    req.user = decoded;
    next();
  } catch (error) {
    res.status(401).send({ error: 'Por favor autentíquese.' });
  }
};

Temas Avanzados

  • Limitación de Tasa: Prevenir abusos limitando el número de solicitudes que un cliente puede hacer.
  • Validación de Datos: Usar bibliotecas como Joi para validar los datos de las solicitudes.
  • Documentación de la API: Usar herramientas como Swagger para documentar tu API.

Conclusión

Construir APIs RESTful con Node.js implica entender los principios de REST, configurar un servidor con Express, definir rutas, conectar a una base de datos e implementar operaciones CRUD. Temas avanzados como la autenticación, manejo de errores y limitación de tasa son cruciales para crear APIs robustas y seguras. Siguiendo este enfoque estructurado, puedes desarrollar APIs escalables y mantenibles que se adhieren a los estándares de la industria.
