Introducción al Despliegue

Desplegar una aplicación Node.js implica mover tu aplicación desde un entorno de desarrollo a un entorno de producción donde pueda ser accesible para los usuarios. Esta sección cubrirá los conceptos básicos del despliegue, incluyendo los pasos y herramientas necesarios.

Conceptos Clave

  • Entorno de Desarrollo: La configuración local donde escribes y pruebas tu código.
  • Entorno de Producción: El entorno en vivo donde tu aplicación se ejecuta y es accesible para los usuarios.
  • Integración Continua/Despliegue Continuo (CI/CD): Prácticas que automatizan el proceso de despliegue.

Preparando Tu Aplicación Node.js para el Despliegue

Antes de desplegar, necesitas asegurarte de que tu aplicación esté lista para producción.

Pasos para Preparar

  1. Variables de Entorno: Usa variables de entorno para gestionar configuraciones. 
    // config.js
module.exports = {
  port: process.env.PORT || 3000,
  dbUrl: process.env.DB_URL,
};
  2. Manejo de Errores: Implementa un manejo de errores robusto para gestionar problemas inesperados. 
    // app.js
process.on('uncaughtException', (err) => {
  console.error('Excepción No Capturada:', err);
  process.exit(1);
});
  3. Registro de Logs: Usa bibliotecas de registro como winston o morgan para seguir el comportamiento de la aplicación. 
    const winston = require('winston');
const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' }),
  ],
});

Opciones de Despliegue

Hay varias maneras de desplegar una aplicación Node.js. Aquí, cubriremos algunas opciones populares.

Desplegando en un Servidor Privado Virtual (VPS)

  1. Elige un Proveedor de VPS: Ejemplos incluyen DigitalOcean, AWS EC2 y Linode.
  2. Configura el Servidor: Instala Node.js, npm y otras dependencias. 
    sudo apt update
sudo apt install nodejs npm
  3. Transfiere Tu Aplicación: Usa scp o un sistema de control de versiones como Git. 
    scp -r tu-carpeta-de-app usuario@tu-ip-vps:/ruta/al/destino
  4. Instala Dependencias: Navega a tu directorio de aplicación e instala dependencias. 
    cd /ruta/al/destino
npm install
  5. Inicia la Aplicación: Usa un gestor de procesos como pm2 para mantener tu aplicación en ejecución. 
    npm install -g pm2
pm2 start app.js

Desplegando en Plataforma como Servicio (PaaS)

  1. Heroku: Una PaaS popular para desplegar aplicaciones Node.js.

    • Instala Heroku CLI: Sigue las instrucciones en el sitio web de Heroku.
    • Crea una App en Heroku: 
      heroku create tu-nombre-de-app
    • Despliega Tu Código: 
      git push heroku main
    • Gestiona Variables de Entorno: 
      heroku config:set NODE_ENV=production

  2. Otras Opciones de PaaS: Ejemplos incluyen Vercel, Netlify y Google App Engine.

Integración Continua/Despliegue Continuo (CI/CD)

Automatizar el proceso de despliegue asegura que tu aplicación esté siempre actualizada con los últimos cambios.

Configurando CI/CD con GitHub Actions

  1. Crea un Archivo de Workflow: Añade un archivo .github/workflows/deploy.yml a tu repositorio. 
    name: Deploy Node.js App

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test

    - name: Deploy to Heroku
      env:
        HEROKU_API_KEY: ${{ secrets.HEROKU_API_KEY }}
      run: |
        git remote add heroku https://git.heroku.com/tu-nombre-de-app.git
        git push heroku main

Conclusión

Desplegar una aplicación Node.js implica varios pasos, desde preparar tu aplicación para producción hasta elegir la plataforma de despliegue adecuada. Ya sea que optes por un VPS, una PaaS o una canalización CI/CD, entender estos conceptos te ayudará a asegurar que tu aplicación sea confiable y escalable. Siguiendo las mejores prácticas y usando las herramientas adecuadas, puedes agilizar el proceso de despliegue y enfocarte en construir grandes aplicaciones.
