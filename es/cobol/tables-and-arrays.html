Introducción a Tablas y Arreglos

Las tablas y arreglos en COBOL se utilizan para almacenar colecciones de elementos de datos que son del mismo tipo. Son esenciales para manejar estructuras de datos repetitivas de manera eficiente.

  • Tablas: A menudo se usan indistintamente con arreglos en COBOL, las tablas son una forma de almacenar múltiples ocurrencias de elementos de datos.
  • Arreglos: Una forma estructurada de almacenar elementos de manera secuencial.

Definiendo Tablas y Arreglos

Definición Básica

En COBOL, las tablas (arreglos) se definen usando la cláusula OCCURS en la DATA DIVISION.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 STUDENT-GRADES.
   05 STUDENT-GRADE PIC 99 OCCURS 10 TIMES.
  • STUDENT-GRADES: El nombre de la tabla.
  • STUDENT-GRADE: El elemento de datos que se repite.
  • OCCURS 10 TIMES: Especifica que hay 10 ocurrencias de STUDENT-GRADE.

Arreglos Multidimensionales

COBOL también soporta arreglos multidimensionales usando cláusulas OCCURS anidadas.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 CLASS-GRADES.
   05 STUDENT-GRADES OCCURS 10 TIMES.
      10 STUDENT-GRADE PIC 99 OCCURS 5 TIMES.
  • CLASS-GRADES: El nombre del arreglo multidimensional.
  • STUDENT-GRADES: La primera dimensión con 10 ocurrencias.
  • STUDENT-GRADE: La segunda dimensión con 5 ocurrencias.

Accediendo a Elementos de la Tabla

Usando Subíndices

Los subíndices se utilizan para acceder a elementos específicos en una tabla.

PROCEDURE DIVISION.
MOVE 85 TO STUDENT-GRADE (3).
DISPLAY STUDENT-GRADE (3).
  • STUDENT-GRADE (3): Accede al tercer elemento del arreglo STUDENT-GRADE.
  • MOVE 85 TO STUDENT-GRADE (3): Asigna el valor 85 al tercer elemento.
  • DISPLAY STUDENT-GRADE (3): Muestra el valor del tercer elemento.

Usando Índices

Los índices proporcionan una forma más eficiente de acceder a los elementos de la tabla, especialmente en tablas grandes.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 STUDENT-GRADES.
   05 STUDENT-GRADE PIC 99 OCCURS 10 TIMES INDEXED BY IDX.

PROCEDURE DIVISION.
SET IDX TO 1.
MOVE 90 TO STUDENT-GRADE (IDX).
DISPLAY STUDENT-GRADE (IDX).
  • INDEXED BY IDX: Declara un índice llamado IDX para la tabla.
  • SET IDX TO 1: Inicializa el índice a 1.
  • MOVE 90 TO STUDENT-GRADE (IDX): Asigna el valor 90 al elemento en el índice 1.
  • DISPLAY STUDENT-GRADE (IDX): Muestra el valor del elemento en el índice 1.

Realizando Operaciones en Tablas

Iterando Sobre Tablas

Se utilizan bucles para iterar sobre los elementos de la tabla.

PROCEDURE DIVISION.
PERFORM VARYING IDX FROM 1 BY 1 UNTIL IDX > 10
   DISPLAY STUDENT-GRADE (IDX)
END-PERFORM.
  • PERFORM VARYING IDX FROM 1 BY 1 UNTIL IDX > 10: Bucle de 1 a 10.
  • DISPLAY STUDENT-GRADE (IDX): Muestra cada elemento del arreglo STUDENT-GRADE.

Buscando en Tablas

COBOL proporciona las sentencias SEARCH y SEARCH ALL para buscar en tablas.

PROCEDURE DIVISION.
SEARCH STUDENT-GRADES
   AT END DISPLAY 'NOT FOUND'
   WHEN STUDENT-GRADE (IDX) = 85
      DISPLAY 'FOUND AT INDEX ' IDX
END-SEARCH.
  • SEARCH STUDENT-GRADES: Busca en la tabla STUDENT-GRADES.
  • AT END: Se ejecuta si la búsqueda se completa sin encontrar el elemento.
  • WHEN STUDENT-GRADE (IDX) = 85: Condición para verificar cada elemento.
  • DISPLAY 'FOUND AT INDEX ' IDX: Muestra el índice si se encuentra el elemento.

Temas Avanzados

Arreglos Dinámicos

COBOL permite la asignación dinámica de arreglos usando las sentencias ALLOCATE y FREE.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 DYNAMIC-TABLE.
   05 DYNAMIC-ELEMENT PIC 99 OCCURS 0 TO 100 TIMES DEPENDING ON TABLE-SIZE.

PROCEDURE DIVISION.
MOVE 50 TO TABLE-SIZE.
ALLOCATE DYNAMIC-TABLE.
  • OCCURS 0 TO 100 TIMES DEPENDING ON TABLE-SIZE: Define un arreglo dinámico.
  • MOVE 50 TO TABLE-SIZE: Establece el tamaño del arreglo a 50.
  • ALLOCATE DYNAMIC-TABLE: Asigna memoria para el arreglo.

Ordenando Tablas

COBOL proporciona la sentencia SORT para ordenar los elementos de la tabla.

PROCEDURE DIVISION.
SORT STUDENT-GRADES ON ASCENDING KEY STUDENT-GRADE.
  • SORT STUDENT-GRADES ON ASCENDING KEY STUDENT-GRADE: Ordena la tabla STUDENT-GRADES en orden ascendente basado en STUDENT-GRADE.

Conclusión

Las tablas y arreglos son herramientas poderosas en COBOL para gestionar colecciones de datos. Entender cómo definir, acceder y manipular estas estructuras es crucial para una programación eficiente en COBOL. Con práctica, puedes aprovechar estos constructos para manejar tareas complejas de procesamiento de datos de manera efectiva.
