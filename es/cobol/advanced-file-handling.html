Introducción

En esta sección, profundizaremos en técnicas avanzadas de manejo de archivos en COBOL. Esto incluye la comprensión de archivos indexados, archivos relativos y operaciones avanzadas de archivos como el acceso dinámico y el manejo del estado de los archivos. Al final de esta sección, deberías ser capaz de gestionar y manipular archivos en COBOL de manera eficiente para aplicaciones complejas.

Conceptos Clave

  • Archivos Indexados: Archivos que permiten el acceso a registros basados en valores clave.
  • Archivos Relativos: Archivos que permiten el acceso a registros basados en su posición relativa.
  • Acceso Dinámico: La capacidad de leer, escribir y actualizar registros en un archivo de manera dinámica.
  • Códigos de Estado de Archivos: Códigos que indican el éxito o fracaso de las operaciones de archivos.

Archivos Indexados

Los archivos indexados en COBOL te permiten acceder a registros utilizando una clave. Esto es particularmente útil para bases de datos y aplicaciones donde es necesario recuperar registros rápidamente.

Definiendo un Archivo Indexado

Para definir un archivo indexado, necesitas especificar la cláusula ORGANIZATION como INDEXED y definir la clave.

FILE-CONTROL.
    SELECT EMPLOYEE-FILE ASSIGN TO 'employee.dat'
        ORGANIZATION IS INDEXED
        ACCESS MODE IS DYNAMIC
        RECORD KEY IS EMP-ID
        FILE STATUS IS WS-FILE-STATUS.

DATA DIVISION.
FILE SECTION.
FD  EMPLOYEE-FILE.
01  EMPLOYEE-RECORD.
    05  EMP-ID         PIC X(5).
    05  EMP-NAME       PIC X(20).
    05  EMP-DEPT       PIC X(10).

Accediendo a Registros

Puedes acceder a registros en un archivo indexado utilizando las sentencias READ, WRITE, REWRITE y DELETE.

PROCEDURE DIVISION.
    OPEN I-O EMPLOYEE-FILE.
    MOVE '12345' TO EMP-ID.
    READ EMPLOYEE-FILE
        INVALID KEY DISPLAY 'Record not found'.
    IF WS-FILE-STATUS = '00'
        DISPLAY 'Employee Name: ' EMP-NAME
        DISPLAY 'Employee Dept: ' EMP-DEPT.
    CLOSE EMPLOYEE-FILE.

Archivos Relativos

Los archivos relativos te permiten acceder a registros basados en su posición relativa en el archivo.

Definiendo un Archivo Relativo

Para definir un archivo relativo, usa la cláusula ORGANIZATION como RELATIVE y especifica la clave relativa.

FILE-CONTROL.
    SELECT CUSTOMER-FILE ASSIGN TO 'customer.dat'
        ORGANIZATION IS RELATIVE
        ACCESS MODE IS DYNAMIC
        RELATIVE KEY IS CUST-REL-KEY
        FILE STATUS IS WS-FILE-STATUS.

DATA DIVISION.
FILE SECTION.
FD  CUSTOMER-FILE.
01  CUSTOMER-RECORD.
    05  CUST-REL-KEY   PIC 9(4).
    05  CUST-NAME      PIC X(20).
    05  CUST-ADDRESS   PIC X(50).

Accediendo a Registros

Puedes acceder a registros en un archivo relativo utilizando las sentencias READ, WRITE, REWRITE y DELETE.

PROCEDURE DIVISION.
    OPEN I-O CUSTOMER-FILE.
    MOVE 1001 TO CUST-REL-KEY.
    READ CUSTOMER-FILE
        INVALID KEY DISPLAY 'Record not found'.
    IF WS-FILE-STATUS = '00'
        DISPLAY 'Customer Name: ' CUST-NAME
        DISPLAY 'Customer Address: ' CUST-ADDRESS.
    CLOSE CUSTOMER-FILE.

Acceso Dinámico

El acceso dinámico te permite leer, escribir y actualizar registros en un archivo sin cerrar y reabrir el archivo.

Ejemplo de Acceso Dinámico

PROCEDURE DIVISION.
    OPEN I-O EMPLOYEE-FILE.
    MOVE '12345' TO EMP-ID.
    READ EMPLOYEE-FILE
        INVALID KEY DISPLAY 'Record not found'.
    IF WS-FILE-STATUS = '00'
        MOVE 'John Doe' TO EMP-NAME
        REWRITE EMPLOYEE-RECORD
        INVALID KEY DISPLAY 'Update failed'.
    CLOSE EMPLOYEE-FILE.

Códigos de Estado de Archivos

Los códigos de estado de archivos proporcionan retroalimentación sobre el éxito o fracaso de las operaciones de archivos. Es crucial manejar estos códigos para asegurar un manejo robusto de archivos.

Códigos de Estado de Archivos Comunes

| Código de Estado | Significado | |------------------|---------------------------| | 00 | Operación exitosa | | 10 | Fin de archivo | | 23 | Registro no encontrado | | 34 | Error de I/O permanente | | 92 | Error lógico |

Manejo de Códigos de Estado de Archivos

PROCEDURE DIVISION.
    OPEN I-O EMPLOYEE-FILE.
    IF WS-FILE-STATUS NOT = '00'
        DISPLAY 'Error opening file: ' WS-FILE-STATUS
        STOP RUN.
    MOVE '12345' TO EMP-ID.
    READ EMPLOYEE-FILE
        INVALID KEY DISPLAY 'Record not found: ' WS-FILE-STATUS.
    CLOSE EMPLOYEE-FILE.

Conclusión

El manejo avanzado de archivos en COBOL implica comprender y utilizar archivos indexados y relativos, acceso dinámico y manejar adecuadamente los códigos de estado de archivos. Dominar estos conceptos te permitirá crear aplicaciones COBOL eficientes y robustas que puedan manejar operaciones complejas de archivos. Practica estas técnicas con varios ejemplos para solidificar tu comprensión y mejorar tus habilidades de manejo de archivos en COBOL.
