SQL Embebido en COBOL ►

Visión General

En esta sección, introduciremos COBOL (Common Business-Oriented Language) y su interacción con bases de datos. COBOL es un lenguaje de programación de alto nivel utilizado principalmente en sistemas de negocios, finanzas y administración para empresas y gobiernos. Entender cómo COBOL interactúa con las bases de datos es crucial para gestionar datos en aplicaciones empresariales a gran escala.

Conceptos Clave

  • Fundamentos de COBOL: Entender la estructura y sintaxis de COBOL.
  • Bases de Datos: Introducción a las bases de datos, enfocándose en bases de datos relacionales.
  • COBOL y Bases de Datos: Cómo COBOL interactúa con las bases de datos, incluyendo SQL embebido.

Fundamentos de COBOL

COBOL es conocido por su legibilidad y sintaxis directa. Aquí hay algunos puntos clave:

  • Estructura de Divisiones: Los programas COBOL se dividen en cuatro divisiones:
    1. División de Identificación: Metadatos del programa.
    2. División de Entorno: Configuración de ajustes.
    3. División de Datos: Declaraciones de variables.
    4. División de Procedimientos: Lógica de negocios y operaciones.

Ejemplo: Programa Simple en COBOL

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.

PROCEDURE DIVISION.
    DISPLAY 'Hello, World!'.
    STOP RUN.

Explicación:

  • IDENTIFICATION DIVISION: Especifica el nombre del programa.
  • PROCEDURE DIVISION: Contiene el código ejecutable.

Introducción a las Bases de Datos

Las bases de datos son colecciones organizadas de datos. Las bases de datos relacionales usan tablas para almacenar datos en filas y columnas.

  • Tablas: Consisten en filas (registros) y columnas (campos).
  • SQL (Structured Query Language): Se usa para interactuar con bases de datos relacionales.

Ejemplo: Consulta SQL Simple

SELECT * FROM Employees WHERE Department = 'Sales';

Explicación:

  • SELECT: Recupera datos de la base de datos.
  • FROM Employees: Especifica la tabla.
  • WHERE Department = 'Sales': Filtra los resultados.

COBOL y Bases de Datos

COBOL puede interactuar con bases de datos usando SQL embebido, lo que permite incluir sentencias SQL dentro del código COBOL.

Ejemplo: Programa COBOL con SQL Embebido

IDENTIFICATION DIVISION.
PROGRAM-ID. EmployeeReport.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 EMP-ID PIC 9(5).
01 EMP-NAME PIC X(20).
01 EMP-DEPT PIC X(10).

PROCEDURE DIVISION.
    EXEC SQL
        DECLARE C1 CURSOR FOR
        SELECT EMP_ID, EMP_NAME, EMP_DEPT
        FROM EMPLOYEES
        WHERE EMP_DEPT = 'Sales'
    END-EXEC.

    EXEC SQL
        OPEN C1
    END-EXEC.

    PERFORM UNTIL SQLCODE = 100
        EXEC SQL
            FETCH C1 INTO :EMP-ID, :EMP-NAME, :EMP-DEPT
        END-EXEC
        DISPLAY 'ID: ' EMP-ID ' Name: ' EMP-NAME ' Dept: ' EMP-DEPT
    END-PERFORM.

    EXEC SQL
        CLOSE C1
    END-EXEC.

    STOP RUN.

Explicación:

  • EXEC SQL: Marca el inicio de una sentencia SQL embebida.
  • DECLARE C1 CURSOR: Declara un cursor para la consulta SQL.
  • OPEN C1: Abre el cursor.
  • FETCH C1 INTO: Recupera datos del cursor en variables COBOL.
  • PERFORM UNTIL SQLCODE = 100: Itera hasta que no se recuperen más filas (SQLCODE 100 indica que no hay más datos).
  • CLOSE C1: Cierra el cursor.

Resumen

En esta sección, cubrimos los fundamentos de COBOL y su interacción con bases de datos. Aprendimos sobre la estructura de los programas COBOL, los fundamentos de las bases de datos relacionales y cómo usar SQL embebido dentro de COBOL para interactuar con una base de datos. Entender estos conceptos es esencial para desarrollar y mantener aplicaciones a nivel empresarial que gestionen grandes volúmenes de datos.
