COBOL (Common Business-Oriented Language) está estructurado en cuatro divisiones principales. Cada división tiene un propósito específico y ayuda a organizar el código de manera lógica y legible. Entender estas divisiones es crucial para escribir y mantener programas en COBOL.

1. División de Identificación

La División de Identificación es la primera división en un programa COBOL. Proporciona metadatos sobre el programa, como el nombre del programa y los detalles del autor.

Conceptos Clave:

  • PROGRAM-ID: Especifica el nombre del programa.
  • AUTHOR: Proporciona el nombre del programador.
  • INSTALLATION: Describe el sitio de instalación.
  • DATE-WRITTEN: Indica cuándo se escribió el programa.
  • DATE-COMPILED: Muestra cuándo se compiló el programa.

Ejemplo:

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.
AUTHOR. John Doe.
INSTALLATION. XYZ Corporation.
DATE-WRITTEN. 2023-10-01.
DATE-COMPILED. 2023-10-02.

2. División de Entorno

La División de Entorno especifica el entorno informático en el que se ejecutará el programa. Incluye información sobre la configuración y los archivos utilizados por el programa.

Conceptos Clave:

  • CONFIGURATION SECTION: Describe la configuración del sistema.
  • INPUT-OUTPUT SECTION: Define los archivos y dispositivos utilizados por el programa.

Ejemplo:

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
SOURCE-COMPUTER. IBM-370.
OBJECT-COMPUTER. IBM-370.

INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'.
    SELECT OutputFile ASSIGN TO 'output.dat'.

3. División de Datos

La División de Datos es donde se definen todas las variables y estructuras de datos utilizadas en el programa. Se divide en varias secciones, pero las más comúnmente utilizadas son la Sección de Almacenamiento de Trabajo y la Sección de Archivos.

Conceptos Clave:

  • FILE SECTION: Define la estructura de los archivos.
  • WORKING-STORAGE SECTION: Declara variables y constantes.
  • LINKAGE SECTION: Se utiliza para los parámetros pasados entre programas.

Ejemplo:

DATA DIVISION.
FILE SECTION.
FD  InputFile.
01  InputRecord.
    05  InputField1  PIC X(10).
    05  InputField2  PIC 9(5).

FD  OutputFile.
01  OutputRecord.
    05  OutputField1  PIC X(10).
    05  OutputField2  PIC 9(5).

WORKING-STORAGE SECTION.
01  WS-Variable1  PIC X(20) VALUE 'Hello, COBOL!'.
01  WS-Variable2  PIC 9(4) VALUE 2023.

4. División de Procedimientos

La División de Procedimientos contiene las instrucciones ejecutables del programa. Aquí es donde se implementa la lógica del programa a través de varias sentencias COBOL.

Conceptos Clave:

  • Párrafos: Bloques lógicos de código.
  • Secciones: Grupo de párrafos.
  • Sentencias: Instrucciones individuales.

Ejemplo:

PROCEDURE DIVISION.
Main-Procedure.
    OPEN INPUT InputFile.
    OPEN OUTPUT OutputFile.
    PERFORM Read-Process-Write UNTIL End-Of-File.
    CLOSE InputFile.
    CLOSE OutputFile.
    STOP RUN.

Read-Process-Write.
    READ InputFile INTO InputRecord
        AT END
            SET End-Of-File TO TRUE
        NOT AT END
            MOVE InputField1 TO OutputField1
            MOVE InputField2 TO OutputField2
            WRITE OutputRecord.

Resumen

Entender las cuatro divisiones principales en COBOL es esencial para escribir programas estructurados y mantenibles. Cada división tiene un rol específico:

  • División de Identificación: Proporciona metadatos sobre el programa.
  • División de Entorno: Especifica el entorno del sistema y el uso de archivos.
  • División de Datos: Define las estructuras de datos y variables.
  • División de Procedimientos: Contiene la lógica ejecutable del programa.

Al dominar estas divisiones, podrás crear programas COBOL bien organizados que sean más fáciles de leer, depurar y mantener.
