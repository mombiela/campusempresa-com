Introducción a COBOL

COBOL (Common Business-Oriented Language) es un lenguaje de programación de alto nivel diseñado para aplicaciones empresariales. Ha sido ampliamente utilizado en diversas industrias, particularmente en instituciones financieras y agencias gubernamentales.

  • Historia de COBOL: Desarrollado en 1959 por CODASYL (Conference on Data Systems Languages).
  • Características Clave:
    • Sintaxis similar al inglés
    • Fuertes capacidades de procesamiento de datos
    • Portabilidad a través de diferentes sistemas

Configuración de Tu Entorno COBOL

Antes de sumergirte en la programación COBOL, necesitas configurar tu entorno de desarrollo.

  • Compiladores COBOL:

    • GnuCOBOL (código abierto)
    • Micro Focus COBOL
    • IBM COBOL para z/OS

  • Instalación:

    • Sigue las instrucciones específicas para el compilador elegido.
    • Asegúrate de tener acceso a un mainframe o a un emulador de mainframe si es necesario.

Sintaxis y Estructura Básica de COBOL

Los programas COBOL se dividen en cuatro divisiones principales:

  1. Identification Division: Contiene metadatos sobre el programa.
  2. Environment Division: Especifica el entorno en el que se ejecutará el programa.
  3. Data Division: Define las variables y estructuras de datos utilizadas.
  4. Procedure Division: Contiene el código ejecutable.

Programa de Ejemplo

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.

ENVIRONMENT DIVISION.

DATA DIVISION.

PROCEDURE DIVISION.
    DISPLAY 'Hello, World!'.
    STOP RUN.
  • Explicación:
    • IDENTIFICATION DIVISION: Declara el nombre del programa.
    • PROCEDURE DIVISION: Contiene la instrucción DISPLAY para imprimir "Hello, World!" y STOP RUN para finalizar el programa.

Tipos de Datos y Variables

COBOL soporta varios tipos de datos, principalmente enfocados en datos numéricos y alfanuméricos.

  • Tipos de Datos Numéricos:

    • PIC 9(n): Datos numéricos con n dígitos.
    • PIC S9(n): Datos numéricos con signo.

  • Tipos de Datos Alfanuméricos:

    • PIC X(n): Datos alfanuméricos con n caracteres.

Ejemplo

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM-VAR PIC 9(5).
01 ALPHA-VAR PIC X(10).

PROCEDURE DIVISION.
    MOVE 12345 TO NUM-VAR.
    MOVE 'Hello' TO ALPHA-VAR.
    DISPLAY NUM-VAR.
    DISPLAY ALPHA-VAR.
    STOP RUN.
  • Explicación:
    • NUM-VAR es una variable numérica con 5 dígitos.
    • ALPHA-VAR es una variable alfanumérica con 10 caracteres.
    • Las instrucciones MOVE asignan valores a las variables.
    • Las instrucciones DISPLAY imprimen los valores.

Estructuras de Control

COBOL proporciona varias estructuras de control para el control de flujo, incluyendo declaraciones condicionales y bucles.

  • Declaraciones IF:

    IF condition
    THEN
        statements
    ELSE
        statements
END-IF.

  • Bucles PERFORM:

    PERFORM UNTIL condition
    statements
END-PERFORM.

Ejemplo

DATA DIVISION.
WORKING-STORAGE SECTION.
01 COUNTER PIC 9(2) VALUE 0.

PROCEDURE DIVISION.
    PERFORM UNTIL COUNTER > 10
        ADD 1 TO COUNTER
        DISPLAY 'Counter: ' COUNTER
    END-PERFORM.
    STOP RUN.
  • Explicación:
    • COUNTER es una variable numérica inicializada a 0.
    • El bucle PERFORM UNTIL incrementa COUNTER hasta que exceda 10.
    • DISPLAY imprime el valor de COUNTER.

Manejo de Archivos

COBOL es conocido por sus robustas capacidades de manejo de archivos, esenciales para aplicaciones empresariales.

  • Control de Archivos:

    • Declaración SELECT para asociar un archivo con un descriptor de archivo.
    • Cláusula ASSIGN para especificar el archivo físico.

  • Operaciones de Archivos:

    • OPEN, READ, WRITE, CLOSE.

Ejemplo

IDENTIFICATION DIVISION.
PROGRAM-ID. FileExample.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT IN-FILE ASSIGN TO 'input.txt'
    ORGANIZATION IS LINE SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD IN-FILE.
01 IN-RECORD PIC X(100).

WORKING-STORAGE SECTION.

PROCEDURE DIVISION.
    OPEN INPUT IN-FILE.
    READ IN-FILE INTO IN-RECORD
        AT END
            DISPLAY 'End of file reached.'
    END-READ.
    DISPLAY IN-RECORD.
    CLOSE IN-FILE.
    STOP RUN.
  • Explicación:
    • La sección FILE-CONTROL asocia IN-FILE con input.txt.
    • FD (Descripción de Archivo) define la estructura del archivo.
    • Las operaciones OPEN, READ y CLOSE gestionan el acceso al archivo.

Temas Avanzados

A medida que avances, encontrarás conceptos y características más complejas en COBOL.

  • Subprogramas: Modulariza el código usando las declaraciones CALL y PERFORM.
  • Manejo de Errores: Usa las cláusulas INVALID KEY y AT END.
  • Acceso a Bases de Datos: Integra con bases de datos usando SQL embebido (EXEC SQL).

Ejemplo de un Subprograma

IDENTIFICATION DIVISION.
PROGRAM-ID. MainProgram.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 RESULT PIC 9(2).

PROCEDURE DIVISION.
    CALL 'SubProgram' USING RESULT.
    DISPLAY 'Result from SubProgram: ' RESULT.
    STOP RUN.

IDENTIFICATION DIVISION.
PROGRAM-ID. SubProgram.

DATA DIVISION.
LINKAGE SECTION.
01 RESULT PIC 9(2).

PROCEDURE DIVISION USING RESULT.
    MOVE 42 TO RESULT.
    EXIT PROGRAM.
  • Explicación:
    • MainProgram llama a SubProgram y pasa RESULT.
    • SubProgram establece RESULT en 42 y devuelve el control a MainProgram.

Conclusión

COBOL sigue siendo un lenguaje vital en el mundo empresarial, particularmente para sistemas heredados en mainframes. Comprender su sintaxis, estructuras de control y capacidades de manejo de archivos es crucial para mantener y desarrollar aplicaciones empresariales. A medida que progreses, dominar temas avanzados como subprogramas y acceso a bases de datos te permitirá manejar tareas más complejas de manera eficiente.
