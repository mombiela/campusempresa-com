Introducción

El monitoreo del sistema y el ajuste de rendimiento son habilidades críticas para los administradores de Linux. Este tema cubrirá las herramientas y técnicas esenciales para monitorear el rendimiento del sistema y hacer los ajustes necesarios para asegurar una operación óptima.

Monitoreo del Sistema

Conceptos Clave

  • Carga del Sistema: La cantidad de trabajo computacional que realiza un sistema.
  • Uso de la CPU: El porcentaje de capacidad de la CPU que se está utilizando.
  • Uso de la Memoria: La cantidad de RAM que se está utilizando.
  • E/S de Disco: La tasa a la que se leen y escriben datos en el disco.
  • Uso de la Red: La cantidad de datos que se transmiten y reciben a través de las interfaces de red.

Herramientas de Monitoreo

  • top: Muestra información resumida del sistema en tiempo real y una lista de tareas que actualmente está gestionando el kernel.
  • htop: Un visor de procesos interactivo para sistemas Unix.
  • vmstat: Informa sobre procesos, memoria, paginación, E/S de bloques, trampas y actividad de la CPU.
  • iostat: Informa sobre estadísticas de la CPU y estadísticas de entrada/salida para dispositivos y particiones.
  • netstat: Muestra conexiones de red, tablas de enrutamiento, estadísticas de interfaces, conexiones de enmascaramiento y membresías de multidifusión.
  • sar: Recoge, informa o guarda información de actividad del sistema.
  • dstat: Herramienta versátil de estadísticas de recursos.

Ejemplos

Usando top

top
  • Explicación: El comando top proporciona una vista dinámica y en tiempo real de un sistema en ejecución. Muestra información resumida del sistema y una lista de tareas que actualmente está gestionando el kernel.

Usando htop

htop
  • Explicación: htop es similar a top, pero proporciona una interfaz más amigable y características adicionales como desplazamiento vertical y horizontal.

Usando vmstat

vmstat 5
  • Explicación: El comando vmstat informa sobre procesos, memoria, paginación, E/S de bloques, trampas y actividad de la CPU. El 5 indica que el informe debe actualizarse cada 5 segundos.

Usando iostat

iostat -x 5
  • Explicación: El comando iostat informa sobre estadísticas de la CPU y estadísticas de entrada/salida para dispositivos y particiones. La opción -x proporciona estadísticas extendidas, y 5 actualiza el informe cada 5 segundos.

Usando netstat

netstat -tuln
  • Explicación: El comando netstat muestra conexiones de red, tablas de enrutamiento, estadísticas de interfaces, conexiones de enmascaramiento y membresías de multidifusión. La opción -tuln muestra puertos TCP y UDP en modo de escucha.

Ajuste de Rendimiento

Conceptos Clave

  • Parámetros del Kernel: Configuraciones que controlan el comportamiento del kernel de Linux.
  • Límites de Recursos: Restricciones sobre la cantidad de recursos del sistema que un proceso puede usar.
  • Swappiness: Un parámetro del kernel que controla la tendencia del kernel a mover procesos fuera de la memoria física y hacia el disco de intercambio.
  • Planificador de E/S: Determina el orden en que las operaciones de E/S de disco se envían a los dispositivos de almacenamiento.

Técnicas de Ajuste

  • Ajuste de Parámetros del Kernel: Usar sysctl para modificar parámetros del kernel.
  • Establecimiento de Límites de Recursos: Usar ulimit para establecer límites en los recursos del sistema.
  • Configuración de Swappiness: Modificar el parámetro vm.swappiness para controlar el uso del intercambio.
  • Elección de un Planificador de E/S: Seleccionar el planificador de E/S adecuado para su carga de trabajo.

Ejemplos

Ajuste de Parámetros del Kernel

sysctl -w net.ipv4.ip_forward=1
  • Explicación: El comando sysctl se usa para modificar parámetros del kernel en tiempo de ejecución. La opción -w se usa para escribir un valor en un parámetro. En este ejemplo, se habilita el reenvío de IP configurando net.ipv4.ip_forward a 1.

Establecimiento de Límites de Recursos

ulimit -n 4096
  • Explicación: El comando ulimit se usa para establecer límites de recursos a nivel de usuario. La opción -n establece el número máximo de descriptores de archivos abiertos. En este ejemplo, el límite se establece en 4096.

Configuración de Swappiness

sysctl -w vm.swappiness=10
  • Explicación: El parámetro vm.swappiness controla la tendencia del kernel a mover procesos fuera de la memoria física y hacia el disco de intercambio. Un valor más bajo reduce el uso del intercambio. En este ejemplo, la swappiness se establece en 10.

Elección de un Planificador de E/S

echo noop > /sys/block/sda/queue/scheduler
  • Explicación: El planificador de E/S determina el orden en que las operaciones de E/S de disco se envían a los dispositivos de almacenamiento. El planificador noop es una cola FIFO simple. En este ejemplo, se selecciona el planificador noop para el dispositivo sda.

Conclusión

El monitoreo del sistema y el ajuste de rendimiento son habilidades esenciales para mantener un sistema Linux saludable y eficiente. Al comprender y utilizar las diversas herramientas de monitoreo y técnicas de ajuste, los administradores pueden asegurar que sus sistemas funcionen de manera óptima. El monitoreo regular y el ajuste proactivo pueden ayudar a prevenir problemas de rendimiento y asegurar que los recursos se utilicen de manera efectiva.
