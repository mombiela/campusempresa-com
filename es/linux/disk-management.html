Introducción

La gestión de discos es un aspecto crucial de la administración de sistemas en Linux. Involucra tareas como particionamiento, formateo, montaje y monitoreo del uso del disco. Esta sección te guiará desde técnicas básicas hasta avanzadas de gestión de discos en Linux.

Conceptos Básicos

Entendiendo las Particiones

  • Partición: Una división de un disco duro (HDD) o unidad de estado sólido (SSD) en secciones separadas e independientes.
  • Tipos de Particiones:
    • Partición Primaria: Puede ser utilizada para arrancar el sistema operativo.
    • Partición Extendida: Un tipo especial de partición primaria que puede contener múltiples particiones lógicas.
    • Partición Lógica: Subdivisiones dentro de una partición extendida.

Sistemas de Archivos

  • Sistema de Archivos: Un método para almacenar y organizar archivos en un disco.
  • Sistemas de archivos comunes en Linux:
    • ext4: Cuarto sistema de archivos extendido, ampliamente utilizado.
    • XFS: Sistema de archivos de alto rendimiento.
    • Btrfs: Sistema de archivos con características avanzadas como instantáneas.

Dispositivos de Disco

  • Los dispositivos de disco están representados en Linux como archivos en el directorio /dev.
    • Ejemplo: /dev/sda (primer disco SATA), /dev/sdb (segundo disco SATA).

Comandos Básicos de Gestión de Discos

Visualización de Información del Disco

  • lsblk: Lista información sobre todos los dispositivos de bloque disponibles o especificados. 
    lsblk
  • fdisk -l: Lista todas las particiones en todos los discos. 
    sudo fdisk -l

Creación de Particiones

  • fdisk: Una utilidad de línea de comandos para crear y manipular tablas de particiones. 
    sudo fdisk /dev/sda
    • Sigue las indicaciones interactivas para crear una nueva partición.

Formateo de Particiones

  • mkfs: Utilizado para crear un sistema de archivos en una partición. 
    sudo mkfs.ext4 /dev/sda1

Montaje y Desmontaje de Particiones

  • mount: Monta un sistema de archivos. 
    sudo mount /dev/sda1 /mnt
  • umount: Desmonta un sistema de archivos. 
    sudo umount /mnt

Gestión Intermedia de Discos

Montajes Persistentes

  • Para asegurar que un sistema de archivos se monte al iniciar, agrega una entrada en /etc/fstab. 
    sudo nano /etc/fstab
    • Ejemplo de entrada: 
      /dev/sda1  /mnt  ext4  defaults  0  2

Cuotas de Disco

  • Cuota: Limita la cantidad de espacio en disco y el número de archivos que un usuario o grupo puede usar.
  • Habilitar cuotas en un sistema de archivos: 
    sudo mount -o remount,usrquota,grpquota /mnt
sudo quotacheck -cug /mnt
sudo quotaon /mnt

LVM (Gestor de Volúmenes Lógicos)

  • LVM: Permite una gestión flexible del disco.
    • Crear un volumen físico: 
      sudo pvcreate /dev/sda1
    • Crear un grupo de volúmenes: 
      sudo vgcreate myvg /dev/sda1
    • Crear un volumen lógico: 
      sudo lvcreate -L 10G -n mylv myvg
    • Formatear y montar el volumen lógico: 
      sudo mkfs.ext4 /dev/myvg/mylv
sudo mount /dev/myvg/mylv /mnt

Gestión Avanzada de Discos

RAID (Matriz Redundante de Discos Independientes)

  • RAID: Combina múltiples unidades de disco en una sola unidad para redundancia o rendimiento.
  • Crear una matriz RAID 1 usando mdadm: 
    sudo mdadm --create --verbose /dev/md0 --level=1 --raid-devices=2 /dev/sda1 /dev/sdb1

Ajuste del Sistema de Archivos

  • tune2fs: Ajusta parámetros del sistema de archivos en sistemas de archivos ext2/ext3/ext4. 
    sudo tune2fs -m 1 /dev/sda1  # Establecer el porcentaje de bloques reservados al 1%

Monitoreo del Uso del Disco

  • df: Reporta el uso del espacio en disco del sistema de archivos. 
    df -h
  • du: Estima el uso del espacio de archivos. 
    du -sh /path/to/directory

Conclusión

La gestión de discos en Linux abarca desde tareas básicas como particionamiento y formateo hasta técnicas avanzadas como LVM y RAID. Dominar estas habilidades asegura una gestión eficiente y confiable del almacenamiento, lo cual es crucial tanto para entornos personales como empresariales. Al entender y utilizar las herramientas y comandos discutidos, puedes gestionar efectivamente los recursos de disco en un sistema Linux.
