Introducción a la Generación de Sitios Estáticos (SSG)

La Generación de Sitios Estáticos (SSG) es un método de pre-renderizado de páginas web en el momento de la construcción, lo que resulta en archivos HTML estáticos. Esto puede mejorar significativamente el rendimiento y el SEO.

  • Pre-renderizado: Generación de HTML en el momento de la construcción.
  • Rendimiento: Cargas de página más rápidas ya que el HTML está pre-generado.
  • SEO: Mejor indexación en motores de búsqueda debido al contenido estático.

¿Por qué Usar Next.js para SSG?

Next.js es un popular framework de React que proporciona soporte integrado para SSG, facilitando la creación de sitios web estáticos.

  • Facilidad de Uso: Simplifica la configuración y el ajuste.
  • Enrutamiento Integrado: Enrutamiento automático basado en el sistema de archivos.
  • Obtención de Datos: Soporta varios métodos de obtención de datos para SSG.

Configuración de un Proyecto Next.js

Para comenzar con Next.js, necesitas configurar un nuevo proyecto.

  1. Instalar Next.js: 
    npx create-next-app@latest my-nextjs-site
cd my-nextjs-site
npm run dev
  2. Estructura del Proyecto:
    • pages/: Contiene todos los componentes de página.
    • public/: Activos estáticos como imágenes.
    • styles/: Archivos CSS.

Creación de una Página Estática

Next.js te permite crear páginas estáticas simplemente exportando un componente de React desde un archivo en el directorio pages/.

  1. Crear una nueva página:

    // pages/index.js
import React from 'react';

const HomePage = () => {
  return (
    <div>
      <h1>Bienvenido a Mi Sitio Estático</h1>
      <p>Esta página está generada estáticamente.</p>
    </div>
  );
};

export default HomePage;

  2. Ejecutar el servidor de desarrollo:

    npm run dev

    Abre http://localhost:3000 para ver tu página estática.

Obtención de Datos con `getStaticProps`

Para obtener datos en el momento de la construcción, Next.js proporciona la función getStaticProps.

  1. Ejemplo con getStaticProps: 
    // pages/index.js
import React from 'react';

export async function getStaticProps() {
  // Obtener datos de una API o sistema de archivos
  const data = await fetch('https://api.example.com/data').then(res => res.json());

  return {
    props: {
      data,
    },
  };
}

const HomePage = ({ data }) => {
  return (
    <div>
      <h1>Bienvenido a Mi Sitio Estático</h1>
      <p>Datos obtenidos en el momento de la construcción:</p>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
};

export default HomePage;

Rutas Dinámicas con `getStaticPaths`

Para páginas que requieren enrutamiento dinámico, puedes usar getStaticPaths junto con getStaticProps.

  1. Ejemplo con rutas dinámicas: 
    // pages/posts/[id].js
import React from 'react';

export async function getStaticPaths() {
  // Obtener lista de IDs de una API o sistema de archivos
  const posts = await fetch('https://api.example.com/posts').then(res => res.json());
  const paths = posts.map(post => ({
    params: { id: post.id.toString() },
  }));

  return { paths, fallback: false };
}

export async function getStaticProps({ params }) {
  // Obtener datos para una sola publicación
  const post = await fetch(`https://api.example.com/posts/${params.id}`).then(res => res.json());

  return {
    props: {
      post,
    },
  };
}

const PostPage = ({ post }) => {
  return (
    <div>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </div>
  );
};

export default PostPage;

Comparación de SSG con Otros Métodos de Obtención de Datos

| Método | Descripción | Caso de Uso | |-----------------|----------------------------------------------|--------------------------------------------------| | Generación de Sitios Estáticos (SSG) | Pre-renderiza páginas en el momento de la construcción. | Contenido estático que no cambia con frecuencia. | | Renderizado del Lado del Servidor (SSR) | Renderiza páginas en cada solicitud. | Contenido dinámico que cambia frecuentemente. | | Renderizado del Lado del Cliente (CSR) | Obtiene datos en el lado del cliente después de la carga. | Contenido altamente interactivo o específico del usuario. |

Conclusión

La Generación de Sitios Estáticos con Next.js es una forma poderosa de crear sitios web rápidos y amigables para el SEO. Al aprovechar getStaticProps y getStaticPaths, puedes obtener datos en el momento de la construcción y generar páginas estáticas de manera eficiente. Este método es ideal para contenido que no cambia con frecuencia, proporcionando un gran equilibrio entre rendimiento y facilidad de uso.

Al dominar SSG en Next.js, puedes construir sitios web estáticos robustos y de alto rendimiento que son fáciles de mantener y escalar.
