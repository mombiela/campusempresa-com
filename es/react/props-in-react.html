Introducción a Props

Props (abreviatura de "properties") son un concepto fundamental en React que te permite pasar datos de un componente a otro. Se utilizan para personalizar y configurar componentes, haciéndolos más reutilizables y dinámicos.

  • Props son de solo lectura: No deben ser modificados por el componente receptor.
  • Props se pasan de padre a hijo: Los datos fluyen en una dirección, de componentes padres a componentes hijos.

Uso de Props en Componentes Funcionales

Los componentes funcionales son más simples y concisos. Aquí se muestra cómo puedes usar props en un componente funcional:

import React from 'react';

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

export default Greeting;

En el ejemplo anterior:

  • Greeting es un componente funcional que toma props como argumento.
  • props.name se usa para mostrar el nombre pasado al componente.

Ejemplo de Uso

import React from 'react';
import ReactDOM from 'react-dom';
import Greeting from './Greeting';

ReactDOM.render(<Greeting name="Alice" />, document.getElementById('root'));

Aquí, el componente Greeting se renderiza con la prop name establecida en "Alice".

Uso de Props en Componentes de Clase

Los componentes de clase también pueden usar props. Aquí hay un ejemplo:

import React, { Component } from 'react';

class Greeting extends Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

export default Greeting;

En este ejemplo:

  • Greeting es un componente de clase que accede a las props usando this.props.

Ejemplo de Uso

import React from 'react';
import ReactDOM from 'react-dom';
import Greeting from './Greeting';

ReactDOM.render(<Greeting name="Bob" />, document.getElementById('root'));

Aquí, el componente Greeting se renderiza con la prop name establecida en "Bob".

Props por Defecto

Puedes definir valores por defecto para las props en caso de que no sean proporcionadas por el componente padre.

Props por Defecto en Componentes Funcionales

import React from 'react';

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

Greeting.defaultProps = {
  name: 'Stranger'
};

export default Greeting;

Props por Defecto en Componentes de Clase

import React, { Component } from 'react';

class Greeting extends Component {
  static defaultProps = {
    name: 'Stranger'
  };

  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}

export default Greeting;

Tipos de Props

Los tipos de props se utilizan para imponer el tipo y la estructura de las props pasadas a un componente. Esto ayuda a detectar errores y asegura que los componentes se usen correctamente.

Uso de PropTypes

import React from 'react';
import PropTypes from 'prop-types';

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

Greeting.propTypes = {
  name: PropTypes.string
};

export default Greeting;

En este ejemplo:

  • PropTypes.string asegura que la prop name sea una cadena de texto.

Pasando Múltiples Props

Puedes pasar múltiples props a un componente incluyéndolas como atributos:

import React from 'react';

function UserCard(props) {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>Age: {props.age}</p>
      <p>Email: {props.email}</p>
    </div>
  );
}

export default UserCard;

Ejemplo de Uso

import React from 'react';
import ReactDOM from 'react-dom';
import UserCard from './UserCard';

ReactDOM.render(
  <UserCard name="Charlie" age={30} email="charlie@example.com" />,
  document.getElementById('root')
);

Resumen

Props son una característica poderosa en React que te permite pasar datos y configuraciones de componentes padres a componentes hijos. Son de solo lectura y no deben ser modificados por el componente receptor. Al usar props por defecto y tipos de props, puedes hacer que tus componentes sean más robustos y fáciles de usar. Entender cómo usar efectivamente las props es esencial para construir componentes de React dinámicos y reutilizables.
