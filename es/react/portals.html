Introducción a los Portales

Los portales proporcionan una forma de renderizar hijos en un nodo del DOM que existe fuera de la jerarquía del componente padre. Esto es particularmente útil para escenarios donde necesitas salir de la estructura típica de DOM padre-hijo, como modales, tooltips o cualquier elemento superpuesto.

Conceptos Clave

  • Portales: Mecanismo para renderizar componentes fuera de la jerarquía DOM de su padre.
  • ReactDOM.createPortal: El método utilizado para crear un portal.

Creando un Portal

Para crear un portal, utilizas el método ReactDOM.createPortal. Este método toma dos argumentos:

  1. El JSX que quieres renderizar.
  2. El nodo del DOM donde quieres renderizar el JSX.

Ejemplo

import React from 'react';
import ReactDOM from 'react-dom';

class Modal extends React.Component {
  render() {
    // Crear un portal
    return ReactDOM.createPortal(
      this.props.children,
      document.getElementById('modal-root')
    );
  }
}

class App extends React.Component {
  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <Modal>
          <div className="modal">
            <p>This is a modal!</p>
          </div>
        </Modal>
      </div>
    );
  }
}

// Suponiendo que tienes un div con id 'modal-root' en tu HTML
// <div id="modal-root"></div>

En este ejemplo:

  • El componente Modal usa ReactDOM.createPortal para renderizar sus hijos en el nodo del DOM con el id modal-root.
  • El componente App renderiza un componente Modal, que a su vez renderiza sus hijos (el contenido del modal) fuera de la jerarquía DOM normal.

Casos de Uso para Portales

  • Modales/Diálogos: Mostrar contenido en un modal que se superpone al resto de la aplicación.
  • Tooltips: Mostrar tooltips que necesitan escapar de las restricciones de overflow o z-index de sus elementos padres.
  • Menús Desplegables: Renderizar menús desplegables que necesitan aparecer sobre otro contenido.

Manejo de Eventos en Portales

Los eventos que ocurren dentro de un portal aún se propagan a los ancestros en el árbol de React, incluso si esos elementos no son ancestros en el árbol del DOM. Esto significa que puedes manejar eventos en el componente padre como de costumbre.

Ejemplo

import React from 'react';
import ReactDOM from 'react-dom';

class Modal extends React.Component {
  render() {
    return ReactDOM.createPortal(
      <div className="modal">
        <button onClick={this.props.onClose}>Close Modal</button>
      </div>,
      document.getElementById('modal-root')
    );
  }
}

class App extends React.Component {
  handleClose = () => {
    console.log('Modal closed');
  };

  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <Modal onClose={this.handleClose} />
      </div>
    );
  }
}

En este ejemplo:

  • El componente Modal renderiza un botón que, cuando se hace clic, llama al método onClose pasado desde el componente App.
  • El método handleClose en el componente App registra un mensaje en la consola.

Conclusión

Los portales en React proporcionan una forma poderosa de renderizar componentes fuera de la jerarquía DOM de su padre, haciéndolos ideales para casos de uso como modales, tooltips y menús desplegables. Al usar ReactDOM.createPortal, puedes asegurarte de que tus componentes se rendericen en el lugar correcto en el DOM mientras mantienes la estructura del árbol de componentes de React y el manejo de eventos.

Entender y utilizar portales puede mejorar enormemente la flexibilidad y usabilidad de tus aplicaciones React, especialmente cuando se trata de elementos de UI complejos que necesitan salir de sus contenedores padres.
