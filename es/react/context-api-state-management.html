Introducción a la API de Contexto

La API de Contexto es una característica en React que te permite compartir estado a lo largo de toda la aplicación (o parte de ella) sin pasar props manualmente en cada nivel. Esto es particularmente útil para la gestión de estado global.

Conceptos Clave

  • Contexto: Una forma de pasar datos a través del árbol de componentes sin tener que pasar props manualmente en cada nivel.
  • Proveedor: Un componente que proporciona el estado a sus hijos.
  • Consumidor: Un componente que consume el estado proporcionado por el Proveedor.

Creando un Contexto

Para crear un contexto, utilizas la función React.createContext. Esta función devuelve un objeto Contexto.

import React from 'react';

const MiContexto = React.createContext();

Usando el Proveedor

El componente Proveedor se usa para envolver la parte de tu aplicación donde quieres que el contexto esté disponible. Acepta una prop value, que es el dato que quieres compartir.

import React, { useState } from 'react';
import MiContexto from './MiContexto';

const App = () => {
  const [estado, setEstado] = useState('¡Hola, Mundo!');

  return (
    <MiContexto.Provider value={{ estado, setEstado }}>
      <MiComponente />
    </MiContexto.Provider>
  );
};

Consumiendo el Contexto

Hay dos formas principales de consumir contexto: usando el hook useContext o el componente Consumer.

Usando el Hook `useContext`

El hook useContext es la forma más directa de consumir contexto en componentes funcionales.

import React, { useContext } from 'react';
import MiContexto from './MiContexto';

const MiComponente = () => {
  const { estado, setEstado } = useContext(MiContexto);

  return (
    <div>
      <p>{estado}</p>
      <button onClick={() => setEstado('¡Hola, React!')}>Cambiar Estado</button>
    </div>
  );
};

Usando el Componente `Consumer`

El componente Consumer es una forma alternativa de consumir contexto, útil en componentes de clase.

import React from 'react';
import MiContexto from './MiContexto';

class MiComponente extends React.Component {
  render() {
    return (
      <MiContexto.Consumer>
        {({ estado, setEstado }) => (
          <div>
            <p>{estado}</p>
            <button onClick={() => setEstado('¡Hola, React!')}>Cambiar Estado</button>
          </div>
        )}
      </MiContexto.Consumer>
    );
  }
}

Uso Avanzado

Contextos Anidados

Puedes anidar múltiples contextos para gestionar diferentes piezas de estado de manera independiente.

const ContextoTema = React.createContext();
const ContextoUsuario = React.createContext();

const App = () => {
  return (
    <ContextoTema.Provider value={{ tema: 'oscuro' }}>
      <ContextoUsuario.Provider value={{ usuario: 'John Doe' }}>
        <MiComponente />
      </ContextoUsuario.Provider>
    </ContextoTema.Provider>
  );
};

Actualizando Contexto desde Componentes Anidados

Para actualizar el contexto desde componentes profundamente anidados, puedes pasar funciones a través del contexto.

const MiContexto = React.createContext();

const App = () => {
  const [estado, setEstado] = useState('¡Hola, Mundo!');

  return (
    <MiContexto.Provider value={{ estado, setEstado }}>
      <ComponenteAnidado />
    </MiContexto.Provider>
  );
};

const ComponenteAnidado = () => {
  const { estado, setEstado } = useContext(MiContexto);

  return (
    <div>
      <p>{estado}</p>
      <button onClick={() => setEstado('¡Hola, Mundo Anidado!')}>Cambiar Estado</button>
    </div>
  );
};

Conclusión

La API de Contexto es una herramienta poderosa para la gestión de estado en React, especialmente para casos de uso más simples y aplicaciones más pequeñas. Te permite evitar el paso de props en cadena y hace que tu código sea más limpio y mantenible. Para necesidades de gestión de estado más complejas, podrías considerar otras soluciones como Redux o MobX.
