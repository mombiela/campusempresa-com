Introducción a las Pruebas de Extremo a Extremo

Las pruebas de extremo a extremo (E2E) son una metodología utilizada para probar si el flujo de una aplicación se está ejecutando como se diseñó de principio a fin. El objetivo es simular escenarios reales de usuarios para validar el sistema en su totalidad.

  • Propósito de las Pruebas E2E: Asegurarse de que los componentes integrados de una aplicación funcionen como se espera.
  • Beneficios:
    • Detecta problemas en la interfaz de usuario.
    • Valida el flujo general del sistema.
    • Asegura que la aplicación funcione en escenarios del mundo real.

Introducción a Cypress

Cypress es un marco de pruebas E2E moderno y de código abierto construido para la web. Proporciona una experiencia de prueba robusta, rápida y confiable para cualquier cosa que se ejecute en un navegador.

  • Características Clave:
    • Viaje en el tiempo: Depuración facilitada al ver instantáneas de tus pruebas.
    • Recargas en tiempo real: Recarga automáticamente las pruebas al guardar.
    • Espera automática: No es necesario agregar esperas o pausas a tus pruebas.
    • Espías, stubs y relojes: Controla e inspecciona el comportamiento de funciones, respuestas del servidor o temporizadores.

Configuración de Cypress en un Proyecto React

Para comenzar a usar Cypress en tu proyecto React, sigue estos pasos:

  1. Instalar Cypress:

    npm install cypress --save-dev

  2. Abrir Cypress:

    npx cypress open

  3. Estructura del Proyecto: Cypress creará un directorio cypress en la raíz de tu proyecto con la siguiente estructura:

    cypress/
  ├── fixtures/
  ├── integration/
  ├── plugins/
  └── support/

Escribiendo Tu Primera Prueba

Vamos a escribir una prueba simple para verificar si nuestra aplicación React se carga correctamente.

  1. Crear un Archivo de Prueba: Crea un nuevo archivo en el directorio cypress/integration, por ejemplo, app_spec.js.

  2. Escribir la Prueba:

    describe('Mi Primera Prueba', () => {
  it('Visita la URL raíz de la aplicación', () => {
    cy.visit('http://localhost:3000');
    cy.contains('Welcome to React');
  });
});

  3. Ejecutar la Prueba: Abre Cypress y ejecuta la prueba. Deberías ver que la prueba pasa si tu aplicación muestra "Welcome to React" en la página de inicio.

Técnicas Avanzadas de Pruebas

Interacción con Elementos

Aprende cómo interactuar con varios elementos en la página.

  • Hacer Clic en un Botón:

    cy.get('button').click();

  • Escribir en un Input:

    cy.get('input[name="username"]').type('miusuario');

Aserciones

Las aserciones se utilizan para validar el estado de tu aplicación.

  • Verificar la Visibilidad de un Elemento:

    cy.get('button').should('be.visible');

  • Verificar el Texto de un Elemento:

    cy.get('h1').should('have.text', 'Welcome to React');

Simulación de Solicitudes API

Cypress te permite simular y mockear solicitudes API para probar diferentes escenarios.

  • Simular una Solicitud GET: 
    cy.server();
cy.route('GET', '/api/users', 'fixture:users.json').as('getUsers');
cy.visit('/users');
cy.wait('@getUsers');

Mejores Prácticas

  • Mantén las Pruebas Independientes: Cada prueba debe poder ejecutarse independientemente de las demás.
  • Usa Fixtures para Datos de Prueba: Almacena y reutiliza datos de prueba usando fixtures.
  • Limpia el Estado Entre Pruebas: Asegúrate de que el estado de la aplicación se restablezca entre pruebas para evitar pruebas inestables.

Conclusión

Las pruebas de extremo a extremo con Cypress proporcionan una manera poderosa de asegurar que tu aplicación React funcione como se espera en escenarios del mundo real. Al configurar Cypress, escribir pruebas básicas y aprovechar técnicas avanzadas, puedes crear un conjunto de pruebas robusto que ayude a mantener la calidad y confiabilidad de tu aplicación.

Recuerda seguir las mejores prácticas para mantener tus pruebas mantenibles y confiables. ¡Felices pruebas!
