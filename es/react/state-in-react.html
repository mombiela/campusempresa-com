Introducción al Estado

El estado es un concepto fundamental en React que permite a los componentes crear y gestionar sus propios datos. Entender el estado es crucial para construir aplicaciones dinámicas e interactivas.

  • Estado: Un objeto que contiene información que puede cambiar a lo largo de la vida del componente.
  • setState: Un método para actualizar el estado y volver a renderizar el componente.

Usando Estado en Componentes Funcionales

Con la introducción de los Hooks en React versión 16.8, gestionar el estado en componentes funcionales se ha vuelto sencillo.

Hook useState

El hook useState se utiliza para añadir estado a los componentes funcionales.

import React, { useState } from 'react';

function Counter() {
  // Declarar una variable de estado llamada "count" con un valor inicial de 0
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Has hecho clic {count} veces</p>
      <button onClick={() => setCount(count + 1)}>
        Haz clic en mí
      </button>
    </div>
  );
}

export default Counter;
  • useState: Inicializa el estado y devuelve un array con el estado actual y una función para actualizarlo.
  • count: El valor actual del estado.
  • setCount: Función para actualizar el estado.

Usando Estado en Componentes de Clase

Antes de los hooks, el estado se gestionaba en componentes de clase usando this.state y this.setState.

import React, { Component } from 'react';

class Counter extends Component {
  constructor(props) {
    super(props);
    // Inicializar el estado
    this.state = { count: 0 };
  }

  render() {
    return (
      <div>
        <p>Has hecho clic {this.state.count} veces</p>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          Haz clic en mí
        </button>
      </div>
    );
  }
}

export default Counter;
  • this.state: Inicializa el estado en el constructor.
  • this.setState: Método para actualizar el estado y desencadenar una re-renderización.

Estado vs. Props

Entender la diferencia entre estado y props es esencial.

| Característica | Estado | Props | |----------------|--------|-------| | Definición | Contiene datos que pueden cambiar | Contiene datos pasados desde el componente padre | | Mutabilidad | Mutable | Inmutable | | Uso | Gestionado dentro del componente | Pasado al componente |

Mejores Prácticas para Usar Estado

  • Mantén el estado mínimo: Solo almacena lo que es necesario.
  • Eleva el estado: Si múltiples componentes necesitan el mismo estado, elévalo a su ancestro común.
  • Evita la mutación directa del estado: Siempre usa setState o la función actualizadora de useState.

Patrones Comunes

Elevación del Estado

Cuando múltiples componentes necesitan compartir estado, elévalo a su ancestro común más cercano.

import React, { useState } from 'react';

function Parent() {
  const [sharedState, setSharedState] = useState(0);

  return (
    <div>
      <Child1 sharedState={sharedState} />
      <Child2 setSharedState={setSharedState} />
    </div>
  );
}

function Child1({ sharedState }) {
  return <p>Estado Compartido: {sharedState}</p>;
}

function Child2({ setSharedState }) {
  return (
    <button onClick={() => setSharedState(prevState => prevState + 1)}>
      Incrementar Estado Compartido
    </button>
  );
}

export default Parent;

Renderizado Condicional Basado en el Estado

Renderiza componentes o elementos condicionalmente basado en el estado.

import React, { useState } from 'react';

function Toggle() {
  const [isToggled, setIsToggled] = useState(false);

  return (
    <div>
      <button onClick={() => setIsToggled(!isToggled)}>
        {isToggled ? 'Ocultar' : 'Mostrar'} Mensaje
      </button>
      {isToggled && <p>¡Este es un mensaje alternado!</p>}
    </div>
  );
}

export default Toggle;

Conclusión

La gestión del estado es un aspecto central de React que permite que los componentes sean dinámicos e interactivos. Ya sea usando componentes funcionales con hooks o componentes de clase, entender cómo gestionar y actualizar correctamente el estado es esencial para construir aplicaciones robustas en React. Recuerda mantener el estado mínimo, elevar el estado cuando sea necesario y evitar mutaciones directas para una experiencia de desarrollo más fluida.
