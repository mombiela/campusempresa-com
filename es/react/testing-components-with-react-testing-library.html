Introducción a React Testing Library

React Testing Library (RTL) es una utilidad de prueba popular para aplicaciones React. Se centra en probar componentes desde la perspectiva del usuario, asegurando que tus pruebas sean más fiables y mantenibles.

  • Conceptos Clave:
    • Pruebas centradas en el usuario: RTL fomenta probar los componentes tal como un usuario interactuaría con ellos.
    • Consultas: RTL proporciona varios métodos de consulta para encontrar elementos en el DOM.
    • Aserciones: Usar aserciones para verificar los resultados esperados.

Configuración de React Testing Library

Para comenzar a usar RTL, necesitas instalarlo y configurar tu entorno de pruebas.

  • Instalación:

    npm install --save-dev @testing-library/react @testing-library/jest-dom

  • Configuración: Asegúrate de tener Jest configurado, ya que RTL funciona perfectamente con Jest. Crea un archivo setupTests.js para incluir configuraciones personalizadas si es necesario.

    // setupTests.js
import '@testing-library/jest-dom/extend-expect';

Escribiendo tu Primera Prueba

Vamos a escribir una prueba simple para un componente React usando RTL.

  • Componente de Ejemplo:

    // Greeting.js
import React from 'react';

const Greeting = ({ name }) => {
  return <h1>Hello, {name}!</h1>;
};

export default Greeting;

  • Prueba para el Componente:

    // Greeting.test.js
import React from 'react';
import { render, screen } from '@testing-library/react';
import Greeting from './Greeting';

test('renders greeting message', () => {
  render(<Greeting name="John" />);
  const greetingElement = screen.getByText(/hello, john!/i);
  expect(greetingElement).toBeInTheDocument();
});

    Explicación:

    • render: Renderiza el componente en el DOM virtual.
    • screen.getByText: Encuentra un elemento por su contenido de texto.
    • expect: Asegura que el elemento esté en el documento.

Consultas Comunes en React Testing Library

RTL proporciona varios métodos de consulta para encontrar elementos en el DOM.

  • Tipos de Consultas:

    • getBy: Lanza un error si no se encuentra ningún elemento.
    • queryBy: Devuelve null si no se encuentra ningún elemento.
    • findBy: Devuelve una promesa y espera a que el elemento aparezca.

  • Ejemplos:

    // ExampleComponent.js
import React from 'react';

const ExampleComponent = () => (
  <div>
    <button>Click me</button>
    <input placeholder="Enter text" />
  </div>
);

export default ExampleComponent;

    // ExampleComponent.test.js
import React from 'react';
import { render, screen } from '@testing-library/react';
import ExampleComponent from './ExampleComponent';

test('finds elements by text, role, and placeholder', () => {
  render(<ExampleComponent />);

  const buttonElement = screen.getByText(/click me/i);
  expect(buttonElement).toBeInTheDocument();

  const inputElement = screen.getByPlaceholderText(/enter text/i);
  expect(inputElement).toBeInTheDocument();
});

Técnicas Avanzadas de Pruebas

A medida que avances, es posible que necesites probar interacciones más complejas y cambios de estado.

  • Pruebas de Eventos de Usuario:

    // Counter.js
import React, { useState } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
};

export default Counter;

    // Counter.test.js
import React from 'react';
import { render, screen, fireEvent } from '@testing-library/react';
import Counter from './Counter';

test('increments counter', () => {
  render(<Counter />);

  const buttonElement = screen.getByText(/increment/i);
  fireEvent.click(buttonElement);

  const countElement = screen.getByText(/count: 1/i);
  expect(countElement).toBeInTheDocument();
});

    Explicación:

    • fireEvent.click: Simula un evento de clic en el botón.

Simulación de Llamadas a API

A veces, necesitas probar componentes que hacen llamadas a API. Puedes simular estas llamadas usando Jest.

  • Ejemplo con Simulación:

    // FetchData.js
import React, { useEffect, useState } from 'react';

const FetchData = () => {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch('/api/data')
      .then(response => response.json())
      .then(data => setData(data));
  }, []);

  if (!data) return <div>Loading...</div>;
  return <div>Data: {data}</div>;
};

export default FetchData;

    // FetchData.test.js
import React from 'react';
import { render, screen, waitFor } from '@testing-library/react';
import FetchData from './FetchData';

beforeAll(() => {
  global.fetch = jest.fn(() =>
    Promise.resolve({
      json: () => Promise.resolve('mock data'),
    })
  );
});

test('fetches and displays data', async () => {
  render(<FetchData />);

  await waitFor(() => screen.getByText(/data: mock data/i));
  expect(screen.getByText(/data: mock data/i)).toBeInTheDocument();
});

    Explicación:

    • global.fetch: Simula la función global fetch.
    • waitFor: Espera a que la operación asincrónica se complete.

Conclusión

React Testing Library es una herramienta poderosa para probar componentes React. Al centrarse en las interacciones y resultados del usuario, ayuda a crear pruebas más fiables y mantenibles. A medida que te sientas más cómodo con RTL, puedes explorar técnicas avanzadas como la simulación de llamadas a API y la prueba de interacciones complejas para asegurar que tu aplicación funcione como se espera.
