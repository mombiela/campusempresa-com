Introducción a las Pruebas Unitarias

Las pruebas unitarias son un método de prueba de software donde se prueban unidades o componentes individuales de un software. El propósito es validar que cada unidad del software funcione como se espera.

  • Unidad: La parte más pequeña de cualquier software que se puede probar.
  • Pruebas: El proceso de evaluar un sistema o sus componentes para verificar si cumple con los requisitos especificados.

¿Qué es Jest?

Jest es un encantador marco de pruebas de JavaScript con un enfoque en la simplicidad. Funciona con proyectos que usan Babel, TypeScript, Node.js, React, Angular, Vue.js, y más.

  • Características de Jest:
    • Cero configuración.
    • Pruebas de instantáneas.
    • Ejecución de pruebas aisladas y en paralelo.
    • Cobertura de código incorporada.

Configuración de Jest en un Proyecto React

Para comenzar a usar Jest en un proyecto React, necesitas instalarlo y configurarlo adecuadamente.

Instalación

Puedes instalar Jest usando npm o yarn:

npm install --save-dev jest

o

yarn add --dev jest

Configuración

Agrega el siguiente script a tu package.json para ejecutar Jest:

"scripts": {
  "test": "jest"
}

Escribiendo tu Primera Prueba

Vamos a escribir una prueba simple para entender cómo funciona Jest.

Ejemplo: Probando una Función de Suma

  1. Crea un archivo llamado sum.js:
function sum(a, b) {
  return a + b;
}
module.exports = sum;
  1. Crea un archivo de prueba llamado sum.test.js:
const sum = require('./sum');

test('suma 1 + 2 para igualar 3', () => {
  expect(sum(1, 2)).toBe(3);
});
  1. Ejecuta la prueba:
npm test

Probando Componentes React

Probar componentes React implica renderizar el componente y hacer afirmaciones sobre su salida.

Ejemplo: Probando un Componente Simple

  1. Crea un archivo llamado Button.js:
import React from 'react';

function Button({ label }) {
  return <button>{label}</button>;
}

export default Button;
  1. Crea un archivo de prueba llamado Button.test.js:
import React from 'react';
import { render } from '@testing-library/react';
import Button from './Button';

test('renderiza botón con etiqueta', () => {
  const { getByText } = render(<Button label="Click Me" />);
  const buttonElement = getByText(/Click Me/i);
  expect(buttonElement).toBeInTheDocument();
});
  1. Ejecuta la prueba:
npm test

Pruebas de Instantáneas

Las pruebas de instantáneas son una forma de probar la interfaz de usuario de tu aplicación. Aseguran que la interfaz de usuario no cambie inesperadamente.

Ejemplo: Pruebas de Instantáneas de un Componente

  1. Crea un archivo llamado Header.js:
import React from 'react';

function Header() {
  return <h1>Welcome to My App</h1>;
}

export default Header;
  1. Crea un archivo de prueba llamado Header.test.js:
import React from 'react';
import renderer from 'react-test-renderer';
import Header from './Header';

test('se renderiza correctamente', () => {
  const tree = renderer.create(<Header />).toJSON();
  expect(tree).toMatchSnapshot();
});
  1. Ejecuta la prueba:
npm test

Mocking en Jest

Mocking es una técnica para reemplazar una función o un módulo con una implementación falsa para controlar su comportamiento durante las pruebas.

Ejemplo: Mocking de una Función

  1. Crea un archivo llamado fetchData.js:
export const fetchData = () => {
  return Promise.resolve('data');
};
  1. Crea un archivo de prueba llamado fetchData.test.js:
import { fetchData } from './fetchData';

jest.mock('./fetchData');

test('obtiene datos', async () => {
  fetchData.mockResolvedValue('datos simulados');
  const data = await fetchData();
  expect(data).toBe('datos simulados');
});
  1. Ejecuta la prueba:
npm test

Conclusión

Las pruebas unitarias con Jest en un proyecto React son cruciales para asegurar la fiabilidad y mantenibilidad de tu código. Al escribir pruebas para tus funciones y componentes, puedes detectar errores temprano y asegurar que tu aplicación se comporte como se espera. Jest proporciona un marco poderoso y fácil de usar para escribir y ejecutar pruebas, lo que lo convierte en una excelente opción para los desarrolladores de React.
