Introducción

En React, hay dos formas principales de manejar datos de formularios: componentes controlados y componentes no controlados. Esta sección se centrará en los componentes no controlados, que te permiten gestionar los datos del formulario utilizando el propio DOM en lugar del estado de React.

Conceptos Clave

  • Componentes No Controlados: Estos componentes dependen del DOM para gestionar los datos del formulario.
  • Refs: Un atributo especial que te permite acceder directamente a los nodos del DOM dentro de React.

¿Por Qué Usar Componentes No Controlados?

  • Simplicidad: Para formularios simples, los componentes no controlados pueden ser más fáciles de configurar.
  • Rendimiento: Pueden ser más eficientes para formularios grandes ya que no requieren actualizaciones de estado en cada cambio de entrada.

Uso de Refs en Componentes No Controlados

Las refs proporcionan una forma de acceder a los nodos del DOM o a los elementos de React creados en el método render. Aquí tienes cómo puedes usar refs para gestionar los datos del formulario en un componente no controlado.

Ejemplo: Formulario Básico No Controlado

import React, { useRef } from 'react';

function UncontrolledForm() {
  const inputRef = useRef(null);

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Input Value: ${inputRef.current.value}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" ref={inputRef} />
      </label>
      <button type="submit">Submit</button>
    </form>
  );
}

export default UncontrolledForm;

Explicación

  • Hook useRef: El hook useRef se usa para crear una referencia al elemento input.
  • Accediendo al Valor: Cuando se envía el formulario, se usa inputRef.current.value para acceder al valor actual del campo de entrada.

Manejo de Múltiples Entradas

Puedes gestionar múltiples entradas en un componente no controlado creando múltiples refs.

Ejemplo: Múltiples Entradas

import React, { useRef } from 'react';

function UncontrolledFormMultiple() {
  const nameRef = useRef(null);
  const emailRef = useRef(null);

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Name: ${nameRef.current.value}, Email: ${emailRef.current.value}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" ref={nameRef} />
      </label>
      <br />
      <label>
        Email:
        <input type="email" ref={emailRef} />
      </label>
      <br />
      <button type="submit">Submit</button>
    </form>
  );
}

export default UncontrolledFormMultiple;

Explicación

  • Múltiples Refs: Se crean refs separadas para cada campo de entrada.
  • Accediendo a los Valores: Cada ref se usa para acceder al valor correspondiente de la entrada cuando se envía el formulario.

Comparación: Componentes Controlados vs. No Controlados

| Característica | Componentes Controlados | Componentes No Controlados | |-----------------------------|----------------------------------------|---------------------------------------| | Gestión del Estado | Gestionado por el estado de React | Gestionado por el DOM | | Rendimiento | Puede ser más lento para formularios grandes | Generalmente más rápido para formularios grandes | | Complejidad | Más complejo debido a la gestión del estado | Más simple para formularios básicos | | Validación en Tiempo Real | Más fácil de implementar | Más desafiante de implementar | | Valores Iniciales | Establecidos a través del estado de React | Establecidos a través de defaultValue o defaultChecked |

Conclusión

Los componentes no controlados en React ofrecen una forma más simple y, a veces, más eficiente de manejar los datos del formulario aprovechando el DOM. Son particularmente útiles para formularios básicos donde la validación en tiempo real y la gestión del estado no son críticas. Entender tanto los componentes controlados como los no controlados te dará la flexibilidad para elegir el mejor enfoque para tu caso de uso específico.
