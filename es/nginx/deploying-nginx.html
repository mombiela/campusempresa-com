Desplegar NGINX es un paso crucial en la configuración de un servidor web o proxy inverso. Esta sección te guiará a través del proceso de desplegar NGINX, desde la instalación hasta la configuración y finalmente ejecutarlo en un entorno de producción.

Instalación

Instalando NGINX en Diferentes Sistemas Operativos

  • Ubuntu/Debian:

    sudo apt update
sudo apt install nginx

  • CentOS/RHEL:

    sudo yum install epel-release
sudo yum install nginx

  • Windows:

    1. Descarga el archivo zip de NGINX desde el sitio web oficial.
    2. Extrae el archivo zip en un directorio de tu elección.
    3. Abre el Símbolo del sistema y navega al directorio extraído.
    4. Ejecuta nginx.exe para iniciar el servidor.

Verificando la Instalación

Después de la instalación, verifica que NGINX esté funcionando visitando http://localhost en tu navegador web. Deberías ver la página de bienvenida predeterminada de NGINX.

Configuración Básica

Entendiendo el Archivo de Configuración

El archivo de configuración principal de NGINX se encuentra típicamente en /etc/nginx/nginx.conf en sistemas Linux. Aquí hay un ejemplo básico de cómo podría verse este archivo:

worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;

    sendfile        on;
    keepalive_timeout  65;

    server {
        listen       80;
        server_name  localhost;

        location / {
            root   /usr/share/nginx/html;
            index  index.html index.htm;
        }

        error_page  500 502 503 504  /50x.html;
        location = /50x.html {
            root   /usr/share/nginx/html;
        }
    }
}

Conceptos Clave

  • worker_processes: Define el número de procesos de trabajo.
  • events: Contiene directivas que afectan el procesamiento de conexiones.
  • http: Contexto principal para manejar tráfico HTTP.
  • server: Define un servidor virtual.
  • location: Especifica cómo procesar solicitudes para una ubicación específica.

Configuración Avanzada

Configurando un Proxy Inverso

Se puede configurar un proxy inverso para reenviar solicitudes de clientes a otro servidor. Aquí hay un ejemplo de configuración:

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend_server;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Balanceo de Carga

NGINX también se puede usar para balanceo de carga. Aquí hay un ejemplo de configuración:

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
    }

    server {
        listen 80;
        server_name example.com;

        location / {
            proxy_pass http://backend;
        }
    }
}

Configuración SSL/TLS

Para asegurar tu servidor NGINX con SSL/TLS, necesitas obtener un certificado y configurar NGINX para usarlo:

server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/nginx.crt;
    ssl_certificate_key /etc/nginx/ssl/nginx.key;

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
    }
}

Ejecutando NGINX en Producción

Iniciando y Deteniendo NGINX

  • Iniciar NGINX:

    sudo systemctl start nginx

  • Detener NGINX:

    sudo systemctl stop nginx

  • Reiniciar NGINX:

    sudo systemctl restart nginx

Habilitando NGINX para Iniciar al Arrancar

  • Habilitar:

    sudo systemctl enable nginx

  • Deshabilitar:

    sudo systemctl disable nginx

Conclusión

Desplegar NGINX implica varios pasos, desde la instalación hasta la configuración y ejecutarlo en un entorno de producción. Al entender las configuraciones básicas y avanzadas, puedes usar NGINX efectivamente como servidor web, proxy inverso o balanceador de carga. Practica estos pasos y configuraciones para volverte competente en desplegar NGINX en varios escenarios.
