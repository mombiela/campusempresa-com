Introducción

En esta sección, exploraremos cómo se puede usar NGINX como un balanceador de carga. El balanceo de carga es un aspecto crítico de las aplicaciones web modernas, asegurando alta disponibilidad y confiabilidad al distribuir el tráfico entrante entre múltiples servidores.

Conceptos Clave

  • Balanceo de Carga: El proceso de distribuir el tráfico de red o de aplicaciones entre múltiples servidores.
  • NGINX: Un servidor HTTP de alto rendimiento y proxy inverso, así como un servidor proxy IMAP/POP3.
  • Proxy Inverso: Un tipo de servidor proxy que recupera recursos en nombre de un cliente desde uno o más servidores.

Tipos de Balanceo de Carga en NGINX

NGINX soporta varios métodos de balanceo de carga:

  • Round Robin: Distribuye las solicitudes de manera uniforme entre todos los servidores.
  • Least Connections: Envía solicitudes al servidor con menos conexiones activas.
  • IP Hash: Distribuye las solicitudes basándose en la dirección IP del cliente.

Ejemplo de Configuración: Balanceo de Carga Round Robin

Comencemos con un ejemplo básico de configuración de NGINX como balanceador de carga utilizando el método Round Robin.

Paso 1: Instalar NGINX

Primero, asegúrese de que NGINX esté instalado en su sistema. Puede instalarlo usando los siguientes comandos:

# Para Debian/Ubuntu
sudo apt update
sudo apt install nginx

# Para CentOS/RHEL
sudo yum install epel-release
sudo yum install nginx

Paso 2: Configurar NGINX

Edite el archivo de configuración de NGINX para configurar el balanceo de carga. El archivo de configuración generalmente se encuentra en /etc/nginx/nginx.conf.

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

Explicación

  • upstream backend: Define un grupo de servidores (backend1, backend2, backend3) a los que NGINX distribuirá las solicitudes.
  • proxy_pass: Reenvía las solicitudes al grupo upstream definido.
  • proxy_set_header: Establece varios encabezados para pasar junto con la solicitud.

Paso 3: Reiniciar NGINX

Después de realizar los cambios de configuración, reinicie NGINX para aplicarlos.

sudo systemctl restart nginx

Configuración Avanzada: Balanceo de Carga Least Connections

Para un balanceo de carga más avanzado, puede usar el método Least Connections. Este método envía solicitudes al servidor con menos conexiones activas.

Ejemplo de Configuración

Modifique el bloque upstream para usar la directiva least_conn.

http {
    upstream backend {
        least_conn;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

Explicación

  • least_conn: Instruye a NGINX para usar el método de balanceo de carga Least Connections.

Monitoreo y Chequeos de Salud

Para asegurar que su balanceador de carga esté distribuyendo el tráfico a servidores saludables, puede configurar chequeos de salud.

Ejemplo de Configuración

Agregue la directiva health_check dentro del bloque upstream.

http {
    upstream backend {
        least_conn;
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
        health_check;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

Explicación

  • health_check: Habilita chequeos de salud para los servidores upstream.

Conclusión

En este estudio de caso, hemos cubierto los conceptos básicos de la configuración de NGINX como balanceador de carga utilizando diferentes métodos como Round Robin y Least Connections. También tocamos la importancia del monitoreo y los chequeos de salud para asegurar la confiabilidad de su balanceador de carga. Al entender estos conceptos y configuraciones, puede distribuir el tráfico de manera efectiva y mantener alta disponibilidad para sus aplicaciones web.
