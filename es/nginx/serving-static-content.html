Introducción a HTTP en NGINX ►

Introducción

Servir contenido estático es uno de los usos más comunes y fundamentales de NGINX. El contenido estático incluye archivos como HTML, CSS, JavaScript, imágenes y videos que no cambian dinámicamente. Esta sección te guiará desde los conceptos básicos hasta técnicas avanzadas para servir contenido estático usando NGINX.

Conceptos Clave

  • Contenido Estático: Archivos que no cambian dinámicamente y se sirven tal cual.
  • Directiva Root: Especifica el directorio raíz desde el cual se sirven los archivos.
  • Directiva Index: Define el archivo predeterminado a servir cuando se solicita un directorio.
  • Bloque Location: Se utiliza para hacer coincidir URLs específicas y aplicar configuraciones.

Configuración Básica

Configuración del Directorio Raíz

Para servir contenido estático, necesitas especificar el directorio raíz donde se almacenan tus archivos. Esto se hace usando la directiva root dentro de un bloque server.

server {
    listen 80;
    server_name example.com;

    root /var/www/html;

    location / {
        try_files $uri $uri/ =404;
    }
}
  • listen 80;: NGINX escucha en el puerto 80 para solicitudes HTTP.
  • server_name example.com;: Especifica el nombre de dominio.
  • root /var/www/html;: Establece el directorio raíz en /var/www/html.
  • location / { ... }: Coincide con todas las solicitudes e intenta servir el archivo correspondiente.

Archivo Index Predeterminado

Puedes especificar un archivo predeterminado a servir cuando se solicita un directorio usando la directiva index.

server {
    listen 80;
    server_name example.com;

    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
  • index index.html;: Establece index.html como el archivo predeterminado a servir.

Configuración Avanzada

Servir Diferentes Tipos de Contenido Estático

Puedes crear múltiples bloques location para servir diferentes tipos de contenido estático desde diferentes directorios.

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        try_files $uri $uri/ =404;
    }

    location /images/ {
        root /var/www/media;
    }

    location /css/ {
        root /var/www/styles;
    }
}
  • location /images/ { root /var/www/media; }: Sirve imágenes desde /var/www/media.
  • location /css/ { root /var/www/styles; }: Sirve archivos CSS desde /var/www/styles.

Caché de Contenido Estático

Para mejorar el rendimiento, puedes habilitar la caché para contenido estático.

server {
    listen 80;
    server_name example.com;

    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 30d;
        add_header Cache-Control "public, no-transform";
    }
}
  • location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ { ... }: Coincide con solicitudes para imágenes, CSS y archivos JavaScript.
  • expires 30d;: Establece la expiración de la caché a 30 días.
  • add_header Cache-Control "public, no-transform";: Añade un encabezado Cache-Control a la respuesta.

Compresión

Puedes habilitar la compresión gzip para reducir el tamaño de los archivos que se sirven.

http {
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 256;
}
  • gzip on;: Habilita la compresión gzip.
  • gzip_types ...;: Especifica los tipos MIME a comprimir.
  • gzip_min_length 256;: Solo comprime archivos mayores de 256 bytes.

Conclusión

Servir contenido estático con NGINX es sencillo pero poderoso. Al entender y utilizar las directivas root e index, configurar bloques location, habilitar la caché y la compresión, puedes servir archivos estáticos de manera eficiente a tus usuarios. Estas técnicas no solo mejoran el rendimiento, sino que también mejoran la experiencia del usuario al entregar contenido de manera rápida y confiable.
