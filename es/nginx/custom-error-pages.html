Las páginas de error personalizadas son una parte esencial del desarrollo web, proporcionando una mejor experiencia de usuario cuando algo sale mal. En esta sección, exploraremos cómo configurar páginas de error personalizadas en NGINX, desde configuraciones básicas hasta configuraciones más avanzadas.

Introducción a las Páginas de Error Personalizadas

Las páginas de error personalizadas te permiten:

  • Proporcionar un mensaje amigable cuando ocurre un error.
  • Mantener una apariencia y sensación consistentes en tu sitio web.
  • Ofrecer información útil u opciones de navegación a los usuarios.

Configuración Básica

Para comenzar con las páginas de error personalizadas en NGINX, necesitas modificar tu archivo de configuración de NGINX. Los errores más comunes a manejar son 404 (No Encontrado) y 500 (Error Interno del Servidor).

Ejemplo: Página de Error 404 Básica

  1. Crear una Página de Error Personalizada: Crea un archivo HTML para tu página de error personalizada, por ejemplo, 404.html.
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Página No Encontrada</title>
</head>
<body>
    <h1>404 - Página No Encontrada</h1>
    <p>Lo sentimos, la página que buscas no existe.</p>
</body>
</html>
  1. Modificar la Configuración de NGINX: Edita tu archivo de configuración de NGINX (usualmente ubicado en /etc/nginx/nginx.conf o /etc/nginx/sites-available/default).
server {
    listen 80;
    server_name tudominio.com;

    location / {
        try_files $uri $uri/ =404;
    }

    error_page 404 /404.html;
    location = /404.html {
        root /usr/share/nginx/html;
        internal;
    }
}
  • error_page 404 /404.html;: Esta directiva le dice a NGINX que use 404.html para errores 404.
  • location = /404.html { ... }: Este bloque especifica dónde se encuentra el archivo 404.html y lo hace interno, para que no pueda ser accedido directamente por los usuarios.

Manejo de Múltiples Códigos de Error

Puedes manejar múltiples códigos de error especificándolos en la directiva error_page.

Ejemplo: Manejo de Errores 404 y 500

  1. Crear Páginas de Error Personalizadas: Crea 404.html y 500.html.
<!-- 404.html -->
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Página No Encontrada</title>
</head>
<body>
    <h1>404 - Página No Encontrada</h1>
    <p>Lo sentimos, la página que buscas no existe.</p>
</body>
</html>
<!-- 500.html -->
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Error Interno del Servidor</title>
</head>
<body>
    <h1>500 - Error Interno del Servidor</h1>
    <p>Lo sentimos, algo salió mal en nuestro lado.</p>
</body>
</html>
  1. Modificar la Configuración de NGINX:
server {
    listen 80;
    server_name tudominio.com;

    location / {
        try_files $uri $uri/ =404;
    }

    error_page 404 /404.html;
    error_page 500 502 503 504 /500.html;

    location = /404.html {
        root /usr/share/nginx/html;
        internal;
    }

    location = /500.html {
        root /usr/share/nginx/html;
        internal;
    }
}
  • error_page 500 502 503 504 /500.html;: Esta directiva maneja múltiples códigos de error del servidor con una sola página de error personalizada.

Páginas de Error Personalizadas Avanzadas

Para configuraciones más avanzadas, podrías querer incluir contenido dinámico o usar scripts del lado del servidor.

Ejemplo: Usar un Script PHP para Páginas de Error Personalizadas

  1. Crear un Script PHP: Crea error.php para manejar errores dinámicamente.
<?php
$error_code = $_GET['code'];
?>
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Error <?php echo $error_code; ?></title>
</head>
<body>
    <h1>Error <?php echo $error_code; ?></h1>
    <p>Lo sentimos, ocurrió un error: <?php echo $error_code; ?></p>
</body>
</html>
  1. Modificar la Configuración de NGINX:
server {
    listen 80;
    server_name tudominio.com;

    location / {
        try_files $uri $uri/ =404;
    }

    error_page 404 = /error.php?code=404;
    error_page 500 502 503 504 = /error.php?code=500;

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
    }
}
  • error_page 404 = /error.php?code=404;: Esta directiva usa un script PHP para manejar el error, pasando el código de error como un parámetro.

Conclusión

Las páginas de error personalizadas en NGINX mejoran la experiencia del usuario proporcionando mensajes de error informativos y estéticamente agradables. Siguiendo los pasos descritos en esta sección, puedes crear páginas de error personalizadas básicas a avanzadas adaptadas a las necesidades de tu sitio web. Recuerda probar tus configuraciones a fondo para asegurarte de que funcionen como se espera.
