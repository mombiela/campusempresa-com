Introducción a la Caché

La caché es una técnica utilizada para almacenar copias de archivos o datos en una caché, o ubicación de almacenamiento temporal, para reducir el tiempo que lleva acceder a los datos y mejorar el rendimiento. En el contexto de NGINX, la caché puede mejorar significativamente el rendimiento de las aplicaciones web al reducir la carga en los servidores backend y disminuir el tiempo de respuesta para los usuarios.

Beneficios de la Caché en NGINX

  • Mejora del Rendimiento: Reduce la latencia y acelera la entrega de contenido.
  • Reducción de la Carga del Servidor: Disminuye el número de solicitudes a los servidores backend.
  • Eficiencia de Costos: Reduce el uso de ancho de banda y el consumo de recursos del servidor.
  • Escalabilidad: Ayuda a manejar más usuarios concurrentes al servir contenido en caché.

Tipos de Caché en NGINX

  • Caché de Contenido Estático: Almacena en caché archivos estáticos como imágenes, CSS y JavaScript.
  • Caché de Proxy: Almacena en caché respuestas de servidores backend para servir solicitudes futuras más rápido.
  • Microcaché: Caché a corto plazo para contenido dinámico para manejar alto tráfico.

Configuración de Caché de Contenido Estático

La caché de contenido estático implica configurar NGINX para almacenar en caché archivos estáticos. Esto se hace típicamente usando la directiva expires.

Ejemplo de Configuración

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html;
    }

    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 30d;
        add_header Cache-Control "public, no-transform";
    }
}
  • Explicación:
    • location ~* \.(jpg|jpeg|png|gif|ico|css|js)$: Coincide con solicitudes de archivos estáticos.
    • expires 30d: Establece el tiempo de expiración de la caché a 30 días.
    • add_header Cache-Control "public, no-transform": Agrega un encabezado Cache-Control a la respuesta.

Configuración de Caché de Proxy

La caché de proxy implica almacenar en caché respuestas de servidores backend. Esto es útil para contenido dinámico que no cambia frecuentemente.

Ejemplo de Configuración

http {
    proxy_cache_path /data/nginx/cache levels=1:2 keys_zone=my_cache:10m max_size=10g inactive=60m use_temp_path=off;

    server {
        listen 80;
        server_name example.com;

        location / {
            proxy_pass http://backend_server;
            proxy_cache my_cache;
            proxy_cache_valid 200 302 10m;
            proxy_cache_valid 404 1m;
            add_header X-Cache-Status $upstream_cache_status;
        }
    }
}
  • Explicación:
    • proxy_cache_path: Define la ruta de almacenamiento de la caché y los parámetros.
    • proxy_pass http://backend_server: Reenvía solicitudes al servidor backend.
    • proxy_cache my_cache: Habilita la caché usando la zona de caché definida.
    • proxy_cache_valid 200 302 10m: Almacena en caché respuestas 200 y 302 por 10 minutos.
    • proxy_cache_valid 404 1m: Almacena en caché respuestas 404 por 1 minuto.
    • add_header X-Cache-Status $upstream_cache_status: Agrega un encabezado personalizado para indicar el estado de la caché.

Microcaché

La microcaché es una técnica donde el contenido dinámico se almacena en caché por períodos muy cortos (por ejemplo, unos pocos segundos) para manejar el alto tráfico de manera eficiente.

Ejemplo de Configuración

http {
    proxy_cache_path /data/nginx/microcache levels=1:2 keys_zone=microcache:10m max_size=1g inactive=1m use_temp_path=off;

    server {
        listen 80;
        server_name example.com;

        location / {
            proxy_pass http://backend_server;
            proxy_cache microcache;
            proxy_cache_valid 200 1s;
            proxy_cache_valid 404 1s;
            add_header X-Cache-Status $upstream_cache_status;
        }
    }
}
  • Explicación:
    • proxy_cache_valid 200 1s: Almacena en caché respuestas 200 por 1 segundo.
    • proxy_cache_valid 404 1s: Almacena en caché respuestas 404 por 1 segundo.
    • Esta configuración es ideal para escenarios de alto tráfico donde incluso una corta duración de la caché puede reducir significativamente la carga del servidor.

Purga de Caché

La purga de caché permite eliminar manualmente el contenido almacenado en caché. Esto es útil cuando el contenido se actualiza y necesitas asegurarte de que los usuarios reciban la versión más reciente.

Ejemplo de Configuración

http {
    proxy_cache_path /data/nginx/cache levels=1:2 keys_zone=my_cache:10m max_size=10g inactive=60m use_temp_path=off;

    server {
        listen 80;
        server_name example.com;

        location / {
            proxy_pass http://backend_server;
            proxy_cache my_cache;
            proxy_cache_valid 200 302 10m;
            proxy_cache_valid 404 1m;
            add_header X-Cache-Status $upstream_cache_status;
        }

        location /purge {
            allow 127.0.0.1;
            deny all;
            proxy_cache_purge my_cache "$scheme$request_method$host$request_uri";
        }
    }
}
  • Explicación:
    • location /purge: Define una ubicación para purgar la caché.
    • allow 127.0.0.1; deny all;: Restringe el acceso a la ubicación de purga a localhost.
    • proxy_cache_purge my_cache "$scheme$request_method$host$request_uri";: Purga la caché especificada.

Conclusión

La caché en NGINX es una característica poderosa que puede mejorar enormemente el rendimiento y la escalabilidad de las aplicaciones web. Al entender y configurar la caché de contenido estático, la caché de proxy, la microcaché y la purga de caché, puedes optimizar tu servidor NGINX para manejar más tráfico de manera eficiente y proporcionar una mejor experiencia de usuario.
