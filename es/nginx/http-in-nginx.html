Resumen

NGINX es un servidor web poderoso que también puede ser usado como proxy inverso, balanceador de carga y caché HTTP. Entender cómo NGINX maneja las solicitudes HTTP es fundamental para configurar y optimizar tu servidor web.

Conceptos Clave

  • Protocolo HTTP: La base de la comunicación de datos en la web.
  • Archivo de Configuración de NGINX: El archivo principal donde defines cómo NGINX maneja las solicitudes HTTP.
  • Directivas: Instrucciones en el archivo de configuración que controlan el comportamiento de NGINX.
  • Contextos: Secciones en el archivo de configuración que agrupan directivas relacionadas.

Estructura del Archivo de Configuración de NGINX

El archivo de configuración de NGINX típicamente se encuentra en /etc/nginx/nginx.conf. Consiste en directivas y contextos.

Ejemplo de Archivo de Configuración

user www-data;
worker_processes auto;
pid /run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    sendfile on;
    keepalive_timeout 65;

    server {
        listen 80;
        server_name example.com;

        location / {
            root /var/www/html;
            index index.html index.htm;
        }
    }
}

Explicación

  • Directivas Globales: user, worker_processes, y pid son configuraciones globales.
  • Contexto events: Contiene directivas relacionadas con las conexiones de red.
  • Contexto http: Contiene directivas para manejar el tráfico HTTP.
  • Bloque server: Define un servidor virtual.
  • Bloque location: Especifica cómo procesar solicitudes para una ubicación específica.

Manejo de Solicitudes HTTP

NGINX procesa las solicitudes HTTP en una serie de fases, incluyendo:

  1. Fase de Conexión: Estableciendo una conexión con el cliente.
  2. Fase de Solicitud: Recibiendo y analizando la solicitud HTTP.
  3. Fase de Respuesta: Generando y enviando la respuesta HTTP.

Ejemplo: Configuración Básica de un Servidor HTTP

server {
    listen 80;
    server_name mywebsite.com;

    location / {
        root /usr/share/nginx/html;
        index index.html;
    }

    location /images/ {
        root /data;
    }
}

Explicación

  • listen: Especifica el puerto en el que NGINX escucha.
  • server_name: Define el nombre de dominio.
  • location /: Maneja solicitudes a la URL raíz.
  • location /images/: Maneja solicitudes a la ruta de URL /images/.

Directivas Comunes

  • listen: Especifica la dirección IP y el puerto.
  • server_name: Define los nombres de dominio.
  • root: Establece el directorio raíz para las solicitudes.
  • index: Especifica el archivo de índice.

Ejemplo: Uso de Directivas

server {
    listen 8080;
    server_name example.org;

    location / {
        root /var/www/example;
        index index.html index.htm;
    }
}

Explicación

  • listen 8080: NGINX escucha en el puerto 8080.
  • server_name example.org: El servidor responde a solicitudes para example.org.
  • root /var/www/example: El directorio raíz para las solicitudes es /var/www/example.
  • index index.html index.htm: Los archivos de índice son index.html y index.htm.

Conclusión

Entender cómo NGINX maneja las solicitudes HTTP y cómo configurarlo usando directivas y contextos es crucial para configurar un servidor web confiable y eficiente. Al dominar estos conceptos básicos, puedes avanzar a configuraciones y optimizaciones más avanzadas.
