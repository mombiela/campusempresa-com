Introducción a SSL/TLS

  • SSL (Secure Sockets Layer) y TLS (Transport Layer Security) son protocolos para establecer enlaces autenticados y encriptados entre computadoras en red.
  • SSL es el predecesor de TLS. TLS es más seguro y es el estándar moderno.
  • Usar SSL/TLS con NGINX asegura que los datos transmitidos entre el servidor y los clientes estén encriptados y seguros.

Requisitos previos

  • Comprensión básica de la configuración de NGINX.
  • Un nombre de dominio apuntando a tu servidor.
  • Acceso administrativo a tu servidor.
  • OpenSSL instalado en tu servidor.

Obtener un certificado SSL

  • Certificado autofirmado: Útil para pruebas y desarrollo.
  • Certificado firmado por una Autoridad de Certificación (CA): Requerido para entornos de producción para asegurar la confianza.

Generar un certificado autofirmado

  1. Generar una clave privada: 
    openssl genpkey -algorithm RSA -out /etc/nginx/ssl/private.key
  2. Generar una Solicitud de Firma de Certificado (CSR): 
    openssl req -new -key /etc/nginx/ssl/private.key -out /etc/nginx/ssl/certificate.csr
  3. Generar el certificado autofirmado: 
    openssl x509 -req -days 365 -in /etc/nginx/ssl/certificate.csr -signkey /etc/nginx/ssl/private.key -out /etc/nginx/ssl/certificate.crt

Obtener un certificado firmado por una CA

  1. Generar una clave privada: 
    openssl genpkey -algorithm RSA -out /etc/nginx/ssl/private.key
  2. Generar una Solicitud de Firma de Certificado (CSR): 
    openssl req -new -key /etc/nginx/ssl/private.key -out /etc/nginx/ssl/certificate.csr
  3. Enviar la CSR a una Autoridad de Certificación (CA): Sigue las instrucciones de la CA para obtener tu certificado firmado.

Configurar NGINX para SSL/TLS

  1. Editar el archivo de configuración de NGINX:

    • Abre el archivo de configuración de NGINX (usualmente ubicado en /etc/nginx/nginx.conf o /etc/nginx/sites-available/default).

  2. Agregar la configuración SSL:

    server {
    listen 443 ssl;
    server_name tu_dominio.com;

    ssl_certificate /etc/nginx/ssl/certificate.crt;
    ssl_certificate_key /etc/nginx/ssl/private.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}

  3. Redirigir HTTP a HTTPS:

    server {
    listen 80;
    server_name tu_dominio.com;

    location / {
        return 301 https://$host$request_uri;
    }
}

  4. Probar la configuración:

    sudo nginx -t

  5. Recargar NGINX:

    sudo systemctl reload nginx

Configuración avanzada de SSL/TLS

  • Soporte HTTP/2: Mejora el rendimiento al permitir que múltiples solicitudes y respuestas se multiplexen sobre una sola conexión. 
    server {
    listen 443 ssl http2;
    ...
}
  • OCSP Stapling: Mejora el rendimiento de SSL/TLS al permitir que el servidor proporcione al cliente una respuesta OCSP en caché. 
    ssl_stapling on;
ssl_stapling_verify on;
ssl_trusted_certificate /etc/nginx/ssl/ca-certs.pem;
resolver 8.8.8.8 8.8.4.4 valid=300s;
resolver_timeout 5s;
  • HSTS (HTTP Strict Transport Security): Instruye a los navegadores a interactuar con tu sitio solo usando HTTPS. 
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

Conclusión

Configurar SSL/TLS con NGINX es crucial para asegurar el tráfico web. Siguiendo los pasos descritos, puedes asegurar que tu servidor se comunique de manera segura con los clientes. Recuerda mantener tu configuración SSL/TLS actualizada para protegerte contra vulnerabilidades y proporcionar el mejor rendimiento y seguridad para tus usuarios.
