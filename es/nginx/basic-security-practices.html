Introducción

En esta sección, cubriremos prácticas esenciales de seguridad para configurar y gestionar NGINX. Estas prácticas son cruciales para proteger tu servidor web y las aplicaciones que sirve de amenazas y vulnerabilidades comunes.

Conceptos Clave

  • Encabezados de Seguridad: Encabezados HTTP que ayudan a proteger contra ataques.
  • SSL/TLS: Encriptar datos en tránsito para prevenir la interceptación.
  • Control de Acceso: Restringir el acceso a ciertas partes de tu servidor.
  • Limitación de Tasa: Prevenir abusos limitando el número de solicitudes.
  • Registro y Monitoreo: Mantener un registro de la actividad del servidor para análisis de seguridad.

Encabezados de Seguridad

Los encabezados de seguridad pueden ayudar a proteger tu servidor de varios tipos de ataques, como XSS, clickjacking y detección de MIME.

Ejemplo: Añadiendo Encabezados de Seguridad

Añade las siguientes líneas a tu archivo de configuración de NGINX para establecer encabezados de seguridad:

server {
    listen 80;
    server_name example.com;

    add_header X-Content-Type-Options nosniff;
    add_header X-Frame-Options SAMEORIGIN;
    add_header X-XSS-Protection "1; mode=block";
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • X-Content-Type-Options: Previene la detección de tipos MIME.
  • X-Frame-Options: Previene el clickjacking.
  • X-XSS-Protection: Habilita el filtrado XSS.
  • Strict-Transport-Security: Impone HTTPS.

SSL/TLS

SSL/TLS encripta los datos entre el cliente y el servidor, asegurando la privacidad y la integridad de los datos.

Ejemplo: Habilitando SSL/TLS

Primero, obtén un certificado SSL de una Autoridad de Certificación (CA) de confianza. Luego, configura NGINX para usarlo:

server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • ssl_certificate: Ruta al certificado SSL.
  • ssl_certificate_key: Ruta a la clave del certificado SSL.
  • ssl_protocols: Especifica los protocolos SSL/TLS a usar.
  • ssl_ciphers: Define los conjuntos de cifrado a usar.

Control de Acceso

Restringe el acceso a ciertas partes de tu servidor solo a usuarios autorizados.

Ejemplo: Restringiendo Acceso por IP

Puedes restringir el acceso a una dirección IP específica o rango:

server {
    listen 80;
    server_name example.com;

    location /admin {
        allow 192.168.1.0/24;
        deny all;
        root /var/www/html/admin;
        index index.html index.htm;
    }
}
  • allow: Permite el acceso desde la dirección IP o rango especificado.
  • deny: Niega el acceso a todas las demás direcciones IP.

Limitación de Tasa

La limitación de tasa ayuda a prevenir abusos limitando el número de solicitudes que un cliente puede hacer en un período de tiempo determinado.

Ejemplo: Implementando Limitación de Tasa

Configura la limitación de tasa en el archivo de configuración de NGINX:

http {
    limit_req_zone $binary_remote_addr zone=mylimit:10m rate=1r/s;

    server {
        listen 80;
        server_name example.com;

        location / {
            limit_req zone=mylimit burst=5;
            root /var/www/html;
            index index.html index.htm;
        }
    }
}
  • limit_req_zone: Define la zona de limitación de tasa.
  • limit_req: Aplica la limitación de tasa a una ubicación específica.

Registro y Monitoreo

El registro y monitoreo son esenciales para detectar y responder a incidentes de seguridad.

Ejemplo: Configurando Registros de Acceso

Habilita el registro de acceso en tu configuración de NGINX:

server {
    listen 80;
    server_name example.com;

    access_log /var/log/nginx/access.log main;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • access_log: Especifica la ruta al archivo de registro de acceso.

Conclusión

Al implementar estas prácticas básicas de seguridad, puedes mejorar significativamente la seguridad de tu servidor NGINX. Mantente siempre actualizado con los últimos parches de seguridad y mejores prácticas para mantener tu servidor seguro.
