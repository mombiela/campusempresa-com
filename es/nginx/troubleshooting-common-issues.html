Introducción

NGINX es un potente servidor web y servidor proxy inverso, pero como cualquier software, puede encontrar problemas. Esta sección te guiará a través de la solución de problemas comunes que puedes enfrentar al trabajar con NGINX, desde niveles principiantes hasta avanzados.

Problemas Comunes y Soluciones

1. NGINX No Inicia

  • Verificar Sintaxis de Configuración: Antes de iniciar NGINX, asegúrate de que tu archivo de configuración sea sintácticamente correcto.

    sudo nginx -t

    Si hay errores, NGINX proporcionará detalles sobre lo que necesita ser corregido.

  • Conflictos de Puerto: Asegúrate de que el puerto al que NGINX intenta enlazarse (generalmente el puerto 80 o 443) no esté ya en uso por otro servicio.

    sudo lsof -i :80

    Si otro servicio está usando el puerto, es posible que necesites detener ese servicio o cambiar el puerto que usa NGINX.

  • Problemas de Permisos: Asegúrate de que NGINX tenga los permisos necesarios para acceder a sus archivos de configuración y a los directorios que sirve.

    sudo chown -R www-data:www-data /var/www/html

2. Error 403 Prohibido

  • Permisos de Archivos: Asegúrate de que los archivos y directorios servidos por NGINX tengan los permisos correctos.

    sudo chmod -R 755 /var/www/html

  • Configuración de NGINX: Verifica el nginx.conf o los archivos de configuración específicos del sitio para asegurarte de que la directiva root apunte al directorio correcto.

    server {
    listen 80;
    server_name example.com;
    root /var/www/html;
    index index.html;
}

3. Error 404 No Encontrado

  • Verificar Rutas de Archivos: Asegúrate de que las rutas de archivos especificadas en tu configuración coincidan con las ubicaciones reales de los archivos.
  • Directiva Root Correcta: Verifica que la directiva root en tu bloque de servidor apunte al directorio correcto.

4. Error 502 Puerta de Enlace Incorrecta

  • Servicio Backend Caído: Asegúrate de que el servicio backend (por ejemplo, un servidor PHP-FPM o Node.js) esté en funcionamiento.

    sudo systemctl status php7.4-fpm

  • Configuración de Upstream Incorrecta: Verifica que la configuración del servidor upstream en NGINX sea correcta.

    upstream backend {
    server 127.0.0.1:3000;
}

server {
    listen 80;
    location / {
        proxy_pass http://backend;
    }
}

5. Error 504 Tiempo de Espera de la Puerta de Enlace Agotado

  • Aumentar Configuraciones de Tiempo de Espera: Si el servicio backend tarda más en responder, es posible que necesites aumentar las configuraciones de tiempo de espera en tu configuración de NGINX. 
    server {
    listen 80;
    location / {
        proxy_connect_timeout 600;
        proxy_send_timeout 600;
        proxy_read_timeout 600;
        send_timeout 600;
        proxy_pass http://backend;
    }
}

6. Problemas de SSL/TLS

  • Verificar Archivos de Certificado: Asegúrate de que el certificado SSL y los archivos de clave estén correctamente especificados y accesibles.

    server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;
}

  • Permisos Correctos: Asegúrate de que los archivos de certificado tengan los permisos correctos.

    sudo chmod 600 /etc/nginx/ssl/example.com.key

7. Alto Uso de CPU/Memoria

  • Optimizar Configuración: Revisa y optimiza las configuraciones de NGINX, como los procesos de trabajo y las conexiones.

    worker_processes auto;
events {
    worker_connections 1024;
}

  • Caché: Implementa caché para reducir la carga en el servidor.

    location / {
    proxy_cache my_cache;
    proxy_cache_valid 200 1h;
    proxy_pass http://backend;
}

Conclusión

Solucionar problemas en NGINX puede ser un desafío, pero entender los problemas comunes y sus soluciones puede facilitar significativamente el proceso. Siempre comienza verificando la sintaxis de la configuración, verificando rutas de archivos y permisos, y asegurándote de que los servicios backend estén funcionando correctamente. Con estos pasos, puedes resolver la mayoría de los problemas comunes de NGINX de manera eficiente.
