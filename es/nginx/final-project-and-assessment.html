Proyecto Final y Evaluación

Descripción General

El proyecto final y la evaluación de este curso de NGINX están diseñados para consolidar tu aprendizaje desde niveles principiantes hasta avanzados. Este proyecto evaluará tu comprensión de los conceptos, configuraciones y mejores prácticas de NGINX. Se te pedirá que configures un entorno de servidor NGINX integral que incluya varias características y optimizaciones cubiertas a lo largo del curso.

Requisitos del Proyecto

Tu proyecto final implicará configurar un servidor NGINX que cumpla con los siguientes requisitos:

  • Configuración Básica:

    • Instalar NGINX en un servidor Linux.
    • Configurar un bloque de servidor HTTP básico para servir contenido estático.

  • Proxy Inverso:

    • Configurar NGINX como un proxy inverso para un servidor de aplicaciones backend (por ejemplo, Node.js, Python Flask).

  • Balanceo de Carga:

    • Implementar balanceo de carga con múltiples servidores backend.

  • SSL/TLS:

    • Configurar SSL/TLS para conexiones seguras usando un certificado autofirmado o un certificado de una CA de confianza.

  • Caché:

    • Implementar caché para mejorar el rendimiento.

  • Seguridad:

    • Configurar medidas de seguridad básicas, como limitación de tasa y control de acceso.

  • Registro y Monitoreo:

    • Configurar registros y configurar monitoreo básico usando herramientas como nginx_status.

Instrucciones Paso a Paso

1. Configuración Básica

  • Instalar NGINX:

    sudo apt update
sudo apt install nginx

  • Configurar Bloque de Servidor HTTP:

    server {
    listen 80;
    server_name example.com;
    root /var/www/html;

    location / {
        try_files $uri $uri/ =404;
    }
}

2. Proxy Inverso

  • Configurar Proxy Inverso: 
    server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

3. Balanceo de Carga

  • Implementar Balanceo de Carga: 
    upstream backend {
    server backend1.example.com;
    server backend2.example.com;
}

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

4. SSL/TLS

  • Configurar SSL/TLS: 
    server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/nginx.crt;
    ssl_certificate_key /etc/nginx/ssl/nginx.key;

    location / {
        try_files $uri $uri/ =404;
    }
}

5. Caché

  • Implementar Caché: 
    server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend;
        proxy_cache my_cache;
        proxy_cache_valid 200 1h;
    }
}

proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=1g;

6. Seguridad

  • Configurar Limitación de Tasa: 
    http {
    limit_req_zone $binary_remote_addr zone=mylimit:10m rate=1r/s;

    server {
        listen 80;
        server_name example.com;

        location / {
            limit_req zone=mylimit burst=5;
            proxy_pass http://backend;
        }
    }
}

7. Registro y Monitoreo

  • Configurar Registro:

    server {
    listen 80;
    server_name example.com;

    access_log /var/log/nginx/access.log;
    error_log /var/log/nginx/error.log;

    location / {
        try_files $uri $uri/ =404;
    }
}

  • Configurar Monitoreo:

    server {
    listen 80;
    server_name example.com;

    location /nginx_status {
        stub_status;
        allow 127.0.0.1;
        deny all;
    }
}

Conclusión

Este proyecto final es una oportunidad para aplicar todo lo que has aprendido sobre NGINX, desde configuraciones básicas hasta características avanzadas. Al completar este proyecto, tendrás una comprensión integral de cómo configurar y gestionar un servidor NGINX en un escenario del mundo real. ¡Buena suerte!
