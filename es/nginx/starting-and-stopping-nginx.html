En esta sección, cubriremos las operaciones esenciales de iniciar y detener el servidor web NGINX. Estas son tareas fundamentales que todo administrador de NGINX debe conocer.

Conceptos Clave

  • Iniciar NGINX: Iniciar el servicio NGINX para comenzar a manejar el tráfico web.
  • Detener NGINX: Detener el servicio NGINX para dejar de manejar el tráfico web.
  • Reiniciar NGINX: Detener y luego iniciar el servicio NGINX, a menudo utilizado para aplicar cambios de configuración.
  • Recargar NGINX: Aplicar cambios de configuración sin detener el servicio.

Iniciar NGINX

Para iniciar NGINX, necesitas usar la línea de comandos. El comando puede variar ligeramente dependiendo de tu sistema operativo.

En Linux (Systemd)

sudo systemctl start nginx

En Linux (SysVinit)

sudo service nginx start

En macOS (Homebrew)

brew services start nginx

Explicación

  • sudo: Ejecuta el comando con privilegios de superusuario.
  • systemctl start nginx: Usa systemd para iniciar el servicio NGINX.
  • service nginx start: Usa SysVinit para iniciar el servicio NGINX.
  • brew services start nginx: Usa Homebrew para iniciar el servicio NGINX en macOS.

Detener NGINX

Para detener NGINX, puedes usar comandos similares a los de inicio, pero con la palabra clave stop.

En Linux (Systemd)

sudo systemctl stop nginx

En Linux (SysVinit)

sudo service nginx stop

En macOS (Homebrew)

brew services stop nginx

Explicación

  • systemctl stop nginx: Usa systemd para detener el servicio NGINX.
  • service nginx stop: Usa SysVinit para detener el servicio NGINX.
  • brew services stop nginx: Usa Homebrew para detener el servicio NGINX en macOS.

Reiniciar NGINX

Reiniciar NGINX es útil cuando necesitas aplicar cambios de configuración. Esta operación detiene y luego inicia el servicio NGINX.

En Linux (Systemd)

sudo systemctl restart nginx

En Linux (SysVinit)

sudo service nginx restart

En macOS (Homebrew)

brew services restart nginx

Explicación

  • systemctl restart nginx: Usa systemd para reiniciar el servicio NGINX.
  • service nginx restart: Usa SysVinit para reiniciar el servicio NGINX.
  • brew services restart nginx: Usa Homebrew para reiniciar el servicio NGINX en macOS.

Recargar NGINX

Recargar NGINX es una forma más elegante de aplicar cambios de configuración sin detener el servicio. Esto es a menudo preferido en entornos de producción para evitar tiempos de inactividad.

En Linux (Systemd)

sudo systemctl reload nginx

En Linux (SysVinit)

sudo service nginx reload

En macOS (Homebrew)

nginx -s reload

Explicación

  • systemctl reload nginx: Usa systemd para recargar el servicio NGINX.
  • service nginx reload: Usa SysVinit para recargar el servicio NGINX.
  • nginx -s reload: Envía directamente una señal al proceso NGINX para recargar la configuración.

Resumen

En esta sección, cubrimos los comandos básicos para iniciar, detener, reiniciar y recargar el servicio NGINX. Estas operaciones son esenciales para gestionar el servidor web NGINX y asegurar que los cambios de configuración se apliquen correctamente. Entender estos comandos te ayudará a mantener un entorno de servidor web estable y receptivo.
