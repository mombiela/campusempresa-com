Introducción

En NGINX, las directivas y los contextos son conceptos fundamentales que dictan cómo se comporta el servidor. Las directivas son instrucciones que configuran el servidor, mientras que los contextos definen dónde se pueden aplicar estas directivas. Comprender estos conceptos es crucial para configurar y gestionar eficazmente un servidor NGINX.

Directivas

Las directivas son declaraciones de configuración que controlan el comportamiento de NGINX. Pueden ser pares clave-valor simples o estructuras más complejas.

  • Sintaxis: Las directivas generalmente siguen el formato nombre_de_directiva valor;.
  • Tipos:
    • Directivas Simples: Consisten en un nombre y un solo valor. 
      worker_processes 4;
    • Directivas de Bloque: Encierran múltiples directivas dentro de llaves {}. 
      http {
    server {
        listen 80;
        server_name example.com;
    }
}

Contextos

Los contextos definen el alcance dentro del cual las directivas son válidas. Son jerárquicos y pueden estar anidados.

  • Contexto Principal: El contexto de nivel superior, que incluye configuraciones globales. 
    user nginx;
worker_processes auto;
  • Contexto de Eventos: Define configuraciones para conexiones de red. 
    events {
    worker_connections 1024;
}
  • Contexto HTTP: Contiene directivas para manejar el tráfico HTTP. 
    http {
    include       mime.types;
    default_type  application/octet-stream;
}
  • Contexto de Servidor: Define configuraciones para un servidor virtual específico. 
    server {
    listen 80;
    server_name example.com;
}
  • Contexto de Ubicación: Especifica cómo procesar solicitudes para URIs específicos. 
    location / {
    root   /usr/share/nginx/html;
    index  index.html index.htm;
}

Ejemplo de Configuración

Veamos un ejemplo completo para entender cómo las directivas y los contextos trabajan juntos.

# Contexto Principal
user nginx;
worker_processes auto;

# Contexto de Eventos
events {
    worker_connections 1024;
}

# Contexto HTTP
http {
    include       mime.types;
    default_type  application/octet-stream;

    # Contexto de Servidor
    server {
        listen 80;
        server_name example.com;

        # Contexto de Ubicación
        location / {
            root   /usr/share/nginx/html;
            index  index.html index.htm;
        }

        # Otro Contexto de Ubicación
        location /images/ {
            root /data;
        }
    }
}

Explicación

  • Contexto Principal: Establece el usuario y el número de procesos de trabajo.
  • Contexto de Eventos: Configura el número máximo de conexiones simultáneas.
  • Contexto HTTP: Incluye tipos MIME y establece un tipo de contenido predeterminado.
  • Contexto de Servidor: Configura un servidor virtual que escucha en el puerto 80 para example.com.
  • Contexto de Ubicación: Define cómo servir solicitudes para el URI raíz / y el URI /images/.

Conclusión

Comprender las directivas y los contextos es esencial para configurar NGINX de manera efectiva. Las directivas son los bloques de construcción que definen el comportamiento del servidor, mientras que los contextos determinan dónde se aplican estas directivas. Al dominar estos conceptos, puedes crear configuraciones NGINX robustas y eficientes adaptadas a tus necesidades específicas.
