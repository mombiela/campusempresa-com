Introducción al Proxy Inverso

Un proxy inverso es un servidor que se sitúa entre los dispositivos cliente y los servidores backend, reenviando las solicitudes de los clientes al servidor backend apropiado y devolviendo la respuesta del servidor al cliente. NGINX es una opción popular para configurar un proxy inverso debido a su alto rendimiento y flexibilidad.

Conceptos Clave

  • Proxy Inverso: Actúa como intermediario para las solicitudes de los clientes que buscan recursos de los servidores.
  • Servidor Backend: El servidor que procesa las solicitudes de los clientes reenviadas por el proxy inverso.
  • Balanceo de Carga: Distribuir las solicitudes de los clientes entre múltiples servidores backend para asegurar que ningún servidor se sature.
  • Terminación SSL: Desencriptar el tráfico cifrado SSL en el servidor proxy antes de pasarlo al servidor backend.

Configuración Básica de un Proxy Inverso

Para configurar un proxy inverso básico con NGINX, sigue estos pasos:

  1. Instalar NGINX:

    sudo apt update
sudo apt install nginx

  2. Editar el Archivo de Configuración de NGINX: Abre el archivo de configuración predeterminado ubicado en /etc/nginx/sites-available/default:

    sudo nano /etc/nginx/sites-available/default

  3. Configurar el Proxy Inverso: Añade la siguiente configuración al bloque del servidor:

    server {
    listen 80;

    server_name tu_dominio.com;

    location / {
        proxy_pass http://ip_servidor_backend:puerto_servidor_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

  4. Reiniciar NGINX:

    sudo systemctl restart nginx

Explicación de la Configuración

  • listen 80;: NGINX escucha en el puerto 80 para solicitudes HTTP entrantes.
  • server_name tu_dominio.com;: Especifica el nombre de dominio para el servidor.
  • location / { ... }: Define el bloque de ubicación para manejar las solicitudes.
    • proxy_pass http://ip_servidor_backend:puerto_servidor_backend;: Reenvía las solicitudes al servidor backend especificado.
    • proxy_set_header: Establece varios encabezados para pasar junto con la solicitud al servidor backend.

Opciones de Configuración Avanzada

Balanceo de Carga

Para distribuir el tráfico entre múltiples servidores backend, usa la siguiente configuración:

http {
    upstream backend {
        server backend1.ejemplo.com;
        server backend2.ejemplo.com;
        server backend3.ejemplo.com;
    }

    server {
        listen 80;
        server_name tu_dominio.com;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}

Terminación SSL

Para manejar la terminación SSL en el servidor NGINX, usa la siguiente configuración:

server {
    listen 443 ssl;
    server_name tu_dominio.com;

    ssl_certificate /etc/nginx/ssl/tu_dominio.com.crt;
    ssl_certificate_key /etc/nginx/ssl/tu_dominio.com.key;

    location / {
        proxy_pass http://ip_servidor_backend:puerto_servidor_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name tu_dominio.com;

    return 301 https://$host$request_uri;
}

Explicación de la Configuración SSL

  • listen 443 ssl;: NGINX escucha en el puerto 443 para solicitudes HTTPS.
  • ssl_certificate y ssl_certificate_key: Rutas a los archivos del certificado SSL y la clave.
  • El segundo bloque del servidor escucha en el puerto 80 y redirige el tráfico HTTP a HTTPS.

Conclusión

Configurar un proxy inverso con NGINX implica configurar NGINX para reenviar las solicitudes de los clientes a los servidores backend. Esta configuración puede mejorarse con balanceo de carga y terminación SSL para mejorar el rendimiento y la seguridad. Al comprender y aplicar estas configuraciones, puedes gestionar eficazmente el tráfico y asegurar un entorno robusto para el servidor web.
