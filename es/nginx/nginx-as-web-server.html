Introducción a NGINX

NGINX es un servidor web potente y de alto rendimiento que también puede ser utilizado como proxy inverso, balanceador de carga y caché HTTP. Es conocido por su estabilidad, conjunto de características ricas, configuración simple y bajo consumo de recursos.

Instalando NGINX

Para comenzar a usar NGINX, primero necesitas instalarlo en tu sistema. El proceso de instalación varía dependiendo de tu sistema operativo.

En Ubuntu/Debian

sudo apt update
sudo apt install nginx

En CentOS/RHEL

sudo yum install epel-release
sudo yum install nginx

En Windows

Para Windows, puedes descargar los binarios de NGINX desde el sitio web oficial de NGINX y seguir las instrucciones de instalación proporcionadas.

Configuración Básica

Una vez instalado, necesitas configurar NGINX para servir tu contenido web.

Archivo de Configuración Predeterminado

El archivo de configuración principal de NGINX generalmente se encuentra en /etc/nginx/nginx.conf.

Ejemplo de Configuración Básica

A continuación, se muestra un ejemplo simple de un archivo de configuración de NGINX para servir contenido estático desde /var/www/html:

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • listen 80; - Indica a NGINX que escuche en el puerto 80.
  • server_name example.com; - Especifica el nombre de dominio.
  • location / { ... } - Define cómo manejar las solicitudes a la URL raíz.

Iniciar y Detener NGINX

Puedes controlar el servicio NGINX usando los siguientes comandos:

En Ubuntu/Debian

sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx

En CentOS/RHEL

sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx

Servir Contenido Estático

NGINX se utiliza a menudo para servir contenido estático como archivos HTML, CSS y JavaScript.

Ejemplo de Configuración

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html;
    }

    location /images/ {
        root /var/www/html;
    }
}
  • El bloque location /images/ especifica que las solicitudes a /images/ deben ser servidas desde /var/www/html/images/.

Manejo de Contenido Dinámico

NGINX también puede ser configurado para manejar contenido dinámico pasando solicitudes a un servidor de aplicaciones como PHP-FPM, Node.js o Python.

Ejemplo con PHP-FPM

server {
    listen 80;
    server_name example.com;

    root /var/www/html;
    index index.php index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
    }
}
  • try_files $uri $uri/ =404; - Intenta servir el archivo solicitado, si no se encuentra, devuelve un error 404.
  • location ~ \.php$ { ... } - Maneja archivos PHP pasándolos al servicio PHP-FPM.

Mejores Prácticas de Seguridad

Para asegurar que tu servidor NGINX sea seguro, sigue estas mejores prácticas:

  • Deshabilitar Tokens del Servidor: Evita que NGINX muestre su número de versión.

    server_tokens off;

  • Limitar el Tamaño de las Solicitudes: Evita solicitudes grandes que podrían dañar tu servidor.

    client_max_body_size 1M;

  • Usar HTTPS: Siempre usa HTTPS para cifrar los datos entre el cliente y el servidor.

    server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;

    location / {
        root /var/www/html;
        index index.html;
    }
}

Conclusión

Usar NGINX como servidor web implica instalar el software, configurarlo para servir contenido estático y dinámico, y aplicar las mejores prácticas de seguridad. Al dominar estos conceptos básicos, puedes aprovechar las potentes características de NGINX para construir aplicaciones web robustas y eficientes.
