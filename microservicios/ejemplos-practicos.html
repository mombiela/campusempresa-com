Microservicios y Serverless ►

En esta sección, exploraremos ejemplos prácticos que te ayudarán a entender cómo implementar y gestionar microservicios en un entorno real. Estos ejemplos están diseñados para ser claros y comprensibles, incluso si eres un principiante en el tema.

Ejemplo 1: Creación de un Microservicio Básico

Descripción del Ejemplo

En este primer ejemplo, crearemos un microservicio básico utilizando Spring Boot, un framework popular en el ecosistema de Java. Este microservicio será responsable de gestionar información de usuarios.

Pasos para Crear el Microservicio

  1. Configuración del Proyecto:

    • Crear un nuevo proyecto Spring Boot utilizando Spring Initializr.
    • Seleccionar las dependencias necesarias: Spring Web, Spring Data JPA, y H2 Database.

  2. Definición del Modelo:

    • Crear una clase User que represente la entidad de usuario.
@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private String email;

    // Getters and Setters
}
  1. Creación del Repositorio:
    • Crear una interfaz UserRepository que extienda JpaRepository.
public interface UserRepository extends JpaRepository<User, Long> {
}
  1. Implementación del Controlador:
    • Crear un controlador UserController para manejar las solicitudes HTTP.
@RestController
@RequestMapping("/users")
public class UserController {
    @Autowired
    private UserRepository userRepository;

    @GetMapping
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    @PostMapping
    public User createUser(@RequestBody User user) {
        return userRepository.save(user);
    }
}

Ejecución y Pruebas

  • Ejecutar la aplicación y utilizar herramientas como Postman para probar los endpoints /users (GET y POST).

Ejemplo 2: Comunicación entre Microservicios

Descripción del Ejemplo

En este ejemplo, veremos cómo dos microservicios pueden comunicarse entre sí utilizando REST API. Tendremos un microservicio de Order que se comunicará con el microservicio de User para obtener información del usuario.

Pasos para Configurar la Comunicación

  1. Creación del Microservicio de Orden:

    • Configurar un nuevo proyecto Spring Boot para el microservicio de Order.

  2. Definición del Modelo de Orden:

    • Crear una clase Order que represente la entidad de orden.
@Entity
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private Long userId;
    private String product;
    private int quantity;

    // Getters and Setters
}
  1. Implementación del Cliente REST:
    • Crear una clase UserClient para comunicarse con el microservicio de User.
@Service
public class UserClient {
    @Autowired
    private RestTemplate restTemplate;

    public User getUserById(Long userId) {
        return restTemplate.getForObject("http://localhost:8080/users/" + userId, User.class);
    }
}
  1. Configuración del Controlador de Orden:
    • Crear un controlador OrderController que utilice UserClient.
@RestController
@RequestMapping("/orders")
public class OrderController {
    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private UserClient userClient;

    @PostMapping
    public Order createOrder(@RequestBody Order order) {
        User user = userClient.getUserById(order.getUserId());
        if (user != null) {
            return orderRepository.save(order);
        } else {
            throw new RuntimeException("User not found");
        }
    }
}

Ejecución y Pruebas

  • Ejecutar ambos microservicios (User y Order).
  • Utilizar Postman para crear órdenes y verificar la comunicación entre los microservicios.

Conclusión

En esta sección, hemos visto dos ejemplos prácticos que ilustran cómo crear y gestionar microservicios. El primer ejemplo se centró en la creación de un microservicio básico, mientras que el segundo mostró cómo los microservicios pueden comunicarse entre sí. Estos ejemplos son fundamentales para entender los conceptos clave de los microservicios y te servirán como base para proyectos más complejos.
