Subarchivos y Colas de Datos
Principios de Diseño Modular ►

Introducción

En esta sección, exploraremos dos características poderosas en la programación RPG: Subarchivos y Colas de Datos. Estas herramientas son esenciales para crear aplicaciones interactivas y eficientes en la plataforma IBM i.

Subarchivos

Los subarchivos son una forma de manejar listas de datos en programas RPG. Permiten mostrar múltiples registros en una pantalla e interactuar con ellos.

Conceptos Clave

  • Registro de Control de Subarchivo: Gestiona el subarchivo, incluyendo la carga y visualización de registros.
  • Registro de Subarchivo: Representa registros individuales dentro del subarchivo.
  • Palabras Clave de Subarchivo: Palabras clave DDS especiales utilizadas para definir el comportamiento del subarchivo, como SFL, SFLCTL, SFLDSP, SFLDSPCTL y SFLEND.

Ejemplo

Vamos a crear un subarchivo simple para mostrar una lista de clientes.

DDS para Subarchivo

A          R SFLRCD
A            CUSTID         5A  O  2  2
A            CUSTNAME      20A  O  2  8

A          R SFLCTL
A  90                                  SFLDSP
A  91                                  SFLDSPCTL
A                                      SFLEND(*MORE)
A                                      SFLPAG(10)
A                                      SFLSIZ(100)
A            SFLRCDNBR      4S 0H      SFLRCDNBR(CURSOR)
A            SFLRRN         4S 0H
A            SFLEND         1A  H

Código RPG para Cargar Subarchivo

Dcl-F CUSTOMERS Usage(*Input) Keyed;
Dcl-F DISPLAY   WorkStn;

Dcl-S RRN       Int(10);
Dcl-S EndOfFile Ind;

Read CUSTOMERS;
DoW Not %EOF(CUSTOMERS);
  RRN += 1;
  SFLRCD.CUSTID = CUSTOMERS.CUSTID;
  SFLRCD.CUSTNAME = CUSTOMERS.CUSTNAME;
  Write SFLRCD;
  Read CUSTOMERS;
EndDo;

SFLCTL.SFLDSP = *On;
SFLCTL.SFLDSPCTL = *On;
Write SFLCTL;

ExFmt SFLCTL;

Explicación

  • DDS: Define los formatos del subarchivo y del registro de control.
  • Código RPG: Lee los registros de clientes y los carga en el subarchivo. El subarchivo se muestra luego usando ExFmt.

Colas de Datos

Las Colas de Datos se utilizan para la comunicación asincrónica entre programas. Permiten que un programa envíe datos a otro programa sin esperar una respuesta.

Conceptos Clave

  • Cola de Datos: Una estructura de almacenamiento que contiene entradas de datos.
  • Enviar Cola de Datos: Añade una entrada a la cola de datos.
  • Recibir Cola de Datos: Recupera una entrada de la cola de datos.

Ejemplo

Vamos a crear un ejemplo simple donde un programa envía un mensaje a una cola de datos y otro programa lo recibe.

Programa de Envío

Dcl-F DATAQ Usage(*Output) ExtDesc('MYDATAQ');

Dcl-S Message Char(100);

Message = 'Hola desde el Programa A';
Write DATAQ Message;

Programa de Recepción

Dcl-F DATAQ Usage(*Input) ExtDesc('MYDATAQ');

Dcl-S Message Char(100);

Read DATAQ Message;
Dsply Message;

Explicación

  • Programa de Envío: Escribe un mensaje en la cola de datos.
  • Programa de Recepción: Lee el mensaje de la cola de datos y lo muestra.

Conclusión

Los Subarchivos y las Colas de Datos son herramientas esenciales en la programación RPG para manejar listas de datos y permitir la comunicación asincrónica entre programas. Los subarchivos permiten la visualización y manipulación interactiva de datos, mientras que las Colas de Datos facilitan la comunicación eficiente entre programas. Comprender y utilizar estas características mejorará significativamente tu capacidad para crear aplicaciones robustas y fáciles de usar en la plataforma IBM i.
