Introducción a la POO

La Programación Orientada a Objetos (POO) es un paradigma de programación que utiliza "objetos" para diseñar aplicaciones y programas informáticos. Simplifica problemas complejos dividiéndolos en objetos más pequeños y manejables.

Conceptos Clave de la POO

  • Clase: Un plano para crear objetos (una estructura de datos particular).
  • Objeto: Una instancia de una clase.
  • Herencia: Un mecanismo donde una nueva clase hereda propiedades y comportamientos (métodos) de otra clase.
  • Encapsulación: Agrupar los datos (variables) y los métodos (funciones) que operan sobre los datos en una sola unidad o clase.
  • Polimorfismo: La capacidad de presentar la misma interfaz para diferentes tipos de datos subyacentes.
  • Abstracción: Ocultar los detalles de implementación complejos y mostrar solo las características necesarias de un objeto.

Clases y Objetos

Definiendo una Clase

Una clase en PHP se define usando la palabra clave class.

<?php
class Car {
    // Propiedades
    public $color;
    public $model;

    // Métodos
    public function __construct($color, $model) {
        $this->color = $color;
        $this->model = $model;
    }

    public function message() {
        return "Mi coche es un " . $this->color . " " . $this->model . ".";
    }
}
?>

Creando un Objeto

Un objeto se crea usando la palabra clave new.

<?php
$myCar = new Car("negro", "Volvo");
echo $myCar->message();
?>

Herencia

La herencia permite que una clase herede propiedades y métodos de otra clase.

<?php
class ElectricCar extends Car {
    public $batteryCapacity;

    public function __construct($color, $model, $batteryCapacity) {
        parent::__construct($color, $model);
        $this->batteryCapacity = $batteryCapacity;
    }

    public function batteryStatus() {
        return "La capacidad de la batería es " . $this->batteryCapacity . " kWh.";
    }
}

$myElectricCar = new ElectricCar("rojo", "Tesla", 100);
echo $myElectricCar->message();
echo $myElectricCar->batteryStatus();
?>

Encapsulación

La encapsulación se logra utilizando modificadores de acceso: public, protected y private.

<?php
class Person {
    private $name;
    private $age;

    public function setName($name) {
        $this->name = $name;
    }

    public function getName() {
        return $this->name;
    }

    public function setAge($age) {
        if($age > 0) {
            $this->age = $age;
        }
    }

    public function getAge() {
        return $this->age;
    }
}

$person = new Person();
$person->setName("John");
$person->setAge(30);
echo $person->getName();
echo $person->getAge();
?>

Polimorfismo

El polimorfismo permite que los métodos hagan cosas diferentes según el objeto sobre el que actúan.

<?php
class Animal {
    public function makeSound() {
        echo "Algún sonido genérico de animal";
    }
}

class Dog extends Animal {
    public function makeSound() {
        echo "Ladrido";
    }
}

class Cat extends Animal {
    public function makeSound() {
        echo "Maullido";
    }
}

$animals = [new Dog(), new Cat(), new Animal()];

foreach ($animals as $animal) {
    $animal->makeSound();
    echo "\n";
}
?>

Abstracción

La abstracción se logra utilizando clases abstractas e interfaces.

Clase Abstracta

<?php
abstract class Shape {
    abstract protected function getArea();
}

class Circle extends Shape {
    private $radius;

    public function __construct($radius) {
        $this->radius = $radius;
    }

    public function getArea() {
        return pi() * pow($this->radius, 2);
    }
}

$circle = new Circle(5);
echo $circle->getArea();
?>

Interfaz

<?php
interface Logger {
    public function log($message);
}

class FileLogger implements Logger {
    public function log($message) {
        echo "Registrando mensaje en un archivo: " . $message;
    }
}

$logger = new FileLogger();
$logger->log("Ocurrió un error.");
?>

Conclusión

La Programación Orientada a Objetos en PHP proporciona un marco robusto para construir aplicaciones escalables y mantenibles. Al comprender y aplicar los principios de clases, objetos, herencia, encapsulación, polimorfismo y abstracción, puedes crear aplicaciones complejas que sean más fáciles de gestionar y ampliar. Practica estos conceptos con ejemplos del mundo real para profundizar tu comprensión y competencia en POO con PHP.
