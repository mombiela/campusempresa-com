Introducción

En esta sección, introduciremos el concepto de la arquitectura MVC (Modelo-Vista-Controlador) y explicaremos por qué es beneficioso para construir aplicaciones web. También proporcionaremos una visión general de lo que aprenderás en este tema.

  • Arquitectura MVC: Un patrón de diseño que separa la aplicación en tres componentes principales: Modelo, Vista y Controlador.
  • Beneficios: Mejora la organización del código, facilita el mantenimiento y mejora la separación de responsabilidades.

Configurando el Entorno

Antes de comenzar a construir nuestro marco de trabajo MVC, necesitamos configurar nuestro entorno de desarrollo.

  • Instalación de PHP: Asegúrate de que PHP esté instalado en tu sistema.
  • Servidor Web: Usa Apache o Nginx.
  • Composer: Administrador de dependencias de PHP.

Ejemplo: Instalando Composer

# En Windows
php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
php composer-setup.php
php -r "unlink('composer-setup.php');"
# En macOS/Linux
curl -sS https://getcomposer.org/installer | php

Creando la Estructura de Directorios

Crearemos una estructura de directorios que siga el patrón MVC.

  • Directorio Raíz: Contiene el punto de entrada y archivos de configuración.
  • Directorio App: Contiene los componentes MVC.
    • Modelos: Lógica de negocio y manipulación de datos.
    • Vistas: Capa de presentación.
    • Controladores: Manejan la entrada del usuario y las interacciones.

Ejemplo de Estructura de Directorios

/my_mvc_framework
    /app
        /controllers
        /models
        /views
    /public
        index.php
    /config
        config.php
    /vendor
    composer.json

Configurando la Aplicación

Configuraremos archivos de configuración para gestionar ajustes como conexiones a bases de datos y configuraciones de la aplicación.

Ejemplo: config/config.php

<?php
return [
    'db' => [
        'host' => 'localhost',
        'dbname' => 'my_database',
        'user' => 'root',
        'password' => '',
    ],
];

Construyendo el Modelo

El Modelo representa los datos y la lógica de negocio de la aplicación.

  • Conexión a la Base de Datos: Establecer una conexión a la base de datos.
  • Operaciones CRUD: Crear, Leer, Actualizar, Eliminar operaciones.

Ejemplo: app/models/User.php

<?php
class User {
    private $db;

    public function __construct($db) {
        $this->db = $db;
    }

    public function getAllUsers() {
        $stmt = $this->db->query("SELECT * FROM users");
        return $stmt->fetchAll(PDO::FETCH_ASSOC);
    }
}

Construyendo la Vista

La Vista es responsable de renderizar la interfaz de usuario.

  • Plantillas: Archivos HTML con PHP incrustado para contenido dinámico.
  • Separación de Lógica: Mantener la lógica de negocio fuera de las vistas.

Ejemplo: app/views/users.php

<!DOCTYPE html>
<html>
<head>
    <title>Users</title>
</head>
<body>
    <h1>Users List</h1>
    <ul>
        <?php foreach ($users as $user): ?>
            <li><?php echo htmlspecialchars($user['name']); ?></li>
        <?php endforeach; ?>
    </ul>
</body>
</html>

Construyendo el Controlador

El Controlador maneja la entrada del usuario y actualiza el Modelo y la Vista en consecuencia.

  • Enrutamiento: Mapear URLs a controladores y acciones específicas.
  • Manejo de Solicitudes: Procesar la entrada del usuario e interactuar con el Modelo.

Ejemplo: app/controllers/UserController.php

<?php
class UserController {
    private $userModel;

    public function __construct($userModel) {
        $this->userModel = $userModel;
    }

    public function index() {
        $users = $this->userModel->getAllUsers();
        include '../app/views/users.php';
    }
}

Configurando el Enrutamiento

El enrutamiento es esencial para dirigir las solicitudes a las acciones del controlador apropiadas.

Ejemplo: public/index.php

<?php
require '../config/config.php';
require '../app/models/User.php';
require '../app/controllers/UserController.php';

$config = require '../config/config.php';
$db = new PDO('mysql:host=' . $config['db']['host'] . ';dbname=' . $config['db']['dbname'], $config['db']['user'], $config['db']['password']);

$userModel = new User($db);
$userController = new UserController($userModel);

if ($_SERVER['REQUEST_URI'] == '/users') {
    $userController->index();
} else {
    echo "404 Not Found";
}

Conclusión

En este tema, hemos construido un marco de trabajo MVC simple desde cero. Hemos cubierto los conceptos básicos de la configuración del entorno, la creación de la estructura de directorios y la implementación de los componentes Modelo, Vista y Controlador. Este marco puede ser ampliado y personalizado para adaptarse a requisitos más complejos a medida que avances en tus habilidades de programación en PHP.

  • Próximos Pasos: Considera agregar características como manejo de formularios, validación y autenticación para mejorar tu marco de trabajo MVC.
