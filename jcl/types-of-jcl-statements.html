Tipos de Declaraciones JCL
Declaración JOB ►

El Lenguaje de Control de Trabajos (JCL) es un lenguaje de scripting utilizado en los sistemas operativos mainframe de IBM para instruir al sistema sobre cómo ejecutar un trabajo por lotes o iniciar un subsistema. Comprender los diferentes tipos de declaraciones JCL es crucial para gestionar y ejecutar trabajos de manera efectiva. Esta sección cubrirá los tipos principales de declaraciones JCL, proporcionando explicaciones claras, ejemplos y ejercicios para ayudarte a comprender su uso.

1. Declaración Job

La declaración JOB es la primera declaración en un JCL y define el inicio de un trabajo. Proporciona información sobre el trabajo, como el nombre del trabajo, información contable y opciones de procesamiento del trabajo.

Conceptos Clave:

  • Nombre del Trabajo: Identifica el trabajo.
  • Información Contable: Utilizada para facturación y seguimiento.
  • Opciones de Procesamiento del Trabajo: Especifica parámetros como prioridad, clase y notificación.

Ejemplo:

//MYJOB JOB (ACCT),'MY JOB',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
  • MYJOB: Nombre del trabajo.
  • (ACCT): Información contable.
  • 'MY JOB': Descripción del trabajo.
  • CLASS=A: Clase del trabajo.
  • MSGCLASS=X: Clase de mensaje de salida.
  • NOTIFY=&SYSUID: Notificar al usuario que envió el trabajo.

2. Declaración Exec

La declaración EXEC especifica el programa o procedimiento a ejecutar. También puede pasar parámetros al programa.

Conceptos Clave:

  • Nombre del Programa: El nombre del programa a ejecutar.
  • Nombre del Procedimiento: El nombre del procedimiento a ejecutar.
  • Parámetros: Parámetros opcionales para el programa o procedimiento.

Ejemplo:

//STEP1 EXEC PGM=MYPROG,PARM='INPUT1'
  • STEP1: Nombre del paso.
  • PGM=MYPROG: Nombre del programa.
  • PARM='INPUT1': Parámetro pasado al programa.

3. Declaración DD

La declaración de Definición de Datos (DD) describe los conjuntos de datos utilizados en el trabajo. Especifica los archivos de entrada y salida, sus atributos y cómo deben ser manejados.

Conceptos Clave:

  • Nombre del Conjunto de Datos: El nombre del conjunto de datos.
  • Disposición: Especifica el estado del conjunto de datos (por ejemplo, nuevo, viejo, eliminar).
  • Tipo de Dispositivo: Indica el tipo de dispositivo (por ejemplo, disco, cinta).

Ejemplo:

//INPUT1 DD DSN=MY.DATA.SET,DISP=SHR
  • INPUT1: Nombre DD.
  • DSN=MY.DATA.SET: Nombre del conjunto de datos.
  • DISP=SHR: Disposición (compartido).

4. Declaración de Comentario

Las declaraciones de comentario se utilizan para incluir comentarios en el JCL. Son ignoradas por el sistema y están destinadas a fines de documentación.

Conceptos Clave:

  • Indicador de Comentario: Comienza con //*.

Ejemplo:

//* Este es un comentario

5. Declaración PROC

La declaración PROC se utiliza para definir un procedimiento, que es un conjunto de declaraciones JCL que se pueden reutilizar. Los procedimientos ayudan a reducir la redundancia y simplificar la gestión de trabajos.

Conceptos Clave:

  • Nombre del Procedimiento: El nombre del procedimiento.
  • Parámetros: Parámetros opcionales para el procedimiento.

Ejemplo:

//MYPROC PROC
//STEP1 EXEC PGM=MYPROG
//INPUT1 DD DSN=MY.DATA.SET,DISP=SHR
  • MYPROC: Nombre del procedimiento.
  • STEP1: Paso dentro del procedimiento.
  • PGM=MYPROG: Nombre del programa.
  • DSN=MY.DATA.SET: Nombre del conjunto de datos.
  • DISP=SHR: Disposición.

6. Declaración INCLUDE

La declaración INCLUDE se utiliza para incluir un miembro de un conjunto de datos particionado (PDS) en el JCL. Esto es útil para incluir segmentos comunes de código JCL.

Conceptos Clave:

  • Nombre del Miembro: El nombre del miembro del PDS a incluir.

Ejemplo:

//INCLUDE MEMBER=MYCOMMON
  • MEMBER=MYCOMMON: Nombre del miembro a incluir.

Resumen

En esta sección, cubrimos los tipos principales de declaraciones JCL:

  • Declaración JOB: Define el inicio de un trabajo.
  • Declaración EXEC: Especifica el programa o procedimiento a ejecutar.
  • Declaración DD: Describe los conjuntos de datos utilizados en el trabajo.
  • Declaración de Comentario: Incluye comentarios para documentación.
  • Declaración PROC: Define procedimientos reutilizables.
  • Declaración INCLUDE: Incluye segmentos comunes de código JCL.

Comprender estas declaraciones es fundamental para escribir y gestionar scripts JCL de manera efectiva. Practica escribir y ejecutar estas declaraciones para volverte competente en JCL.
