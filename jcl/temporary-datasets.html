Introducción a los datasets temporales

Los datasets temporales en JCL son datasets que existen solo durante la duración del trabajo. Se eliminan automáticamente una vez que el trabajo se completa. Estos datasets son útiles para el almacenamiento intermedio durante la ejecución del trabajo y ayudan a gestionar el espacio en disco de manera eficiente.

Conceptos clave

  • Nombrado de datasets temporales: Los datasets temporales se identifican por nombres que comienzan con dos signos de ampersand (&&).
  • Alcance: Solo son accesibles dentro del paso del trabajo o en todo el trabajo, dependiendo de cómo se definan.
  • Eliminación automática: Se eliminan automáticamente al final del trabajo, reduciendo la necesidad de limpieza manual.

Definición de datasets temporales

Para definir un dataset temporal en JCL, se utiliza el parámetro DSN con un nombre que comienza con &&.

Ejemplo

//STEP1    EXEC PGM=MYPROG
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(NEW,PASS),
//         SPACE=(TRK,(5,5)),UNIT=SYSDA
  • DSN=&&TEMPFILE: Define un dataset temporal llamado TEMPFILE.
  • DISP=(NEW,PASS): Especifica que el dataset es nuevo y debe pasarse al siguiente paso.
  • SPACE=(TRK,(5,5)): Asigna espacio para el dataset.
  • UNIT=SYSDA: Especifica el tipo de dispositivo.

Uso de datasets temporales entre pasos

Los datasets temporales pueden pasarse entre pasos utilizando el parámetro DISP=PASS.

Ejemplo

//STEP1    EXEC PGM=MYPROG1
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(NEW,PASS),
//         SPACE=(TRK,(5,5)),UNIT=SYSDA
//STEP2    EXEC PGM=MYPROG2
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(OLD,DELETE)
  • STEP1: Crea un dataset temporal TEMPFILE y lo pasa al siguiente paso.
  • STEP2: Utiliza el dataset temporal TEMPFILE creado en STEP1 y lo elimina después de su uso.

Uso avanzado de datasets temporales

Los datasets temporales también pueden usarse con parámetros más complejos y en conjunto con otras características de JCL como GDGs (Grupos de Datos Generacionales).

Ejemplo con GDG

//STEP1    EXEC PGM=MYPROG1
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(NEW,PASS),
//         SPACE=(TRK,(5,5)),UNIT=SYSDA
//STEP2    EXEC PGM=MYPROG2
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(OLD,DELETE)
//GDGOUT   DD   DSN=MYGDG(+1),DISP=(NEW,CATLG),
//         SPACE=(TRK,(5,5)),UNIT=SYSDA
  • GDGOUT: Define una nueva generación de un GDG utilizando el dataset temporal.

Conclusión

Los datasets temporales en JCL son una característica poderosa para gestionar datos intermedios dentro de un trabajo. Ayudan en la gestión eficiente del espacio en disco y simplifican el proceso de limpieza del trabajo. Al entender cómo definir, usar y pasar datasets temporales entre pasos, puedes escribir scripts JCL más eficientes y mantenibles.
