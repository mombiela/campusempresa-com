Introducción

La Integración Continua (CI) es una práctica de desarrollo de software donde los desarrolladores integran su código en un repositorio compartido varias veces al día. Cada integración es verificada por una construcción automatizada (incluyendo pruebas) para detectar errores lo más pronto posible. En esta sección, aprenderemos cómo configurar un servidor de Integración Continua.

Conceptos Clave

  • Integración Continua (CI): Proceso de automatización que permite a los desarrolladores integrar su código frecuentemente en un repositorio compartido.
  • Servidor de CI: Herramienta que automatiza la construcción y prueba del código cada vez que se realiza una integración.
  • Pipeline: Secuencia de pasos que el servidor de CI sigue para construir, probar y desplegar el código.

Herramientas Populares de CI

  • Jenkins
  • Travis CI
  • CircleCI
  • GitLab CI/CD

Configuración de Jenkins como Servidor de CI

Requisitos Previos

  • Java Development Kit (JDK) instalado.
  • Servidor web (por ejemplo, Apache Tomcat) o contenedor Docker.
  • Acceso administrativo al servidor donde se instalará Jenkins.

Instalación de Jenkins

  1. Descargar Jenkins:

    • Visita la página oficial de Jenkins y descarga el archivo .war o el paquete para tu sistema operativo.

  2. Instalar Jenkins:

    • Si usas Docker: 
      docker run -p 8080:8080 -p 50000:50000 jenkins/jenkins:lts
    • Si usas el archivo .war: 
      java -jar jenkins.war

  3. Acceder a Jenkins:

    • Abre un navegador web y navega a http://localhost:8080.
    • Sigue las instrucciones en pantalla para completar la configuración inicial.

Configuración Inicial

  1. Desbloquear Jenkins:

    • Durante la primera ejecución, Jenkins solicitará una contraseña de administrador que se encuentra en el archivo initialAdminPassword.

  2. Instalar Plugins Recomendados:

    • Jenkins sugerirá una lista de plugins recomendados. Instálalos para obtener funcionalidades básicas.

  3. Crear un Usuario Administrador:

    • Configura un usuario administrador para gestionar Jenkins.

Crear un Pipeline Básico

  1. Crear un Nuevo Proyecto:

    • Navega a "New Item" y selecciona "Pipeline".

  2. Configurar el Pipeline:

    • En la sección "Pipeline", selecciona "Pipeline script" y añade el siguiente script de ejemplo: 
      pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Aquí irían los comandos de construcción
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Aquí irían los comandos de prueba
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Aquí irían los comandos de despliegue
            }
        }
    }
}

  3. Guardar y Ejecutar:

    • Guarda la configuración y ejecuta el pipeline para verificar que todo funcione correctamente.

Ejemplo de Configuración de Pipeline en Jenkins

pipeline {
    agent any
    environment {
        // Definir variables de entorno si es necesario
        NODE_ENV = 'production'
    }
    stages {
        stage('Checkout') {
            steps {
                // Clonar el repositorio
                git 'https://github.com/tu-repositorio.git'
            }
        }
        stage('Build') {
            steps {
                // Comandos de construcción
                sh 'npm install'
                sh 'npm run build'
            }
        }
        stage('Test') {
            steps {
                // Comandos de prueba
                sh 'npm test'
            }
        }
        stage('Deploy') {
            steps {
                // Comandos de despliegue
                sh 'npm run deploy'
            }
        }
    }
    post {
        success {
            echo 'Pipeline completado exitosamente!'
        }
        failure {
            echo 'Pipeline falló. Revisa los logs para más detalles.'
        }
    }
}

Conclusión

Configurar un servidor de Integración Continua como Jenkins es un paso crucial para automatizar el proceso de construcción, prueba y despliegue de aplicaciones. Con una configuración adecuada, puedes detectar errores rápidamente y mejorar la calidad del software. En este tema, hemos cubierto los conceptos básicos, la instalación y configuración de Jenkins, y cómo crear un pipeline básico. Practica estos pasos y adapta el pipeline a las necesidades específicas de tu proyecto para obtener los mejores resultados.
