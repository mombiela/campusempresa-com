Git LFS (Large File Storage)

Introducción a Git LFS

Git Large File Storage (LFS) es una extensión para Git que te permite gestionar archivos grandes de manera eficiente. Reemplaza archivos grandes con punteros de texto dentro de Git, mientras almacena el contenido de los archivos en un servidor remoto.

  • Propósito: Gestionar archivos grandes en un repositorio Git sin inflar el tamaño del repositorio.
  • Casos de Uso: Proyectos con archivos multimedia grandes, conjuntos de datos, o cualquier otro archivo binario grande.

Configuración de Git LFS

Para usar Git LFS, necesitas instalarlo y configurar tu repositorio para rastrear archivos grandes.

Instalación

Instala Git LFS usando el gestor de paquetes para tu sistema operativo.

  • macOS:

    brew install git-lfs

  • Windows: Descarga y ejecuta el instalador desde Git LFS Releases.

  • Linux:

    sudo apt-get install git-lfs

Inicializando Git LFS en tu Repositorio

Después de instalar Git LFS, inicialízalo en tu repositorio:

git lfs install

Este comando configura los hooks de Git LFS en tu repositorio.

Rastreando Archivos Grandes

Especifica qué archivos deben ser gestionados por Git LFS usando el comando git lfs track.

Ejemplo: Rastreando Archivos de Imagen

Para rastrear todos los archivos .png en tu repositorio:

git lfs track "*.png"

Este comando crea un archivo .gitattributes (si no existe ya) y añade una entrada para los archivos .png.

Cometiendo Cambios

Después de rastrear archivos, añade y comete los cambios:

git add .gitattributes
git commit -m "Track PNG files with Git LFS"

Trabajando con Git LFS

Una vez que los archivos están siendo rastreados por Git LFS, puedes trabajar con ellos como de costumbre. Git LFS maneja el almacenamiento y la recuperación de archivos grandes detrás de escena.

Añadiendo Archivos Grandes

Añade un archivo grande a tu repositorio:

git add largefile.zip
git commit -m "Add large file"

Pushing a Remoto

Cuando empujas cambios a un repositorio remoto, Git LFS sube los archivos grandes al servidor LFS:

git push origin main

Gestionando Archivos de Git LFS

Git LFS proporciona varios comandos para gestionar archivos grandes en tu repositorio.

Listando Archivos Rastreado

Para listar todos los archivos rastreados por Git LFS:

git lfs ls-files

Fetching Archivos LFS

Para obtener todos los archivos LFS en la rama actual:

git lfs fetch

Comprobando el Estado de LFS

Para comprobar el estado de los archivos LFS:

git lfs status

Mejores Prácticas

  • Rastreo Selectivo: Solo rastrea archivos que sean realmente grandes y binarios.
  • Limpieza Regular: Usa git lfs prune para eliminar archivos LFS antiguos y no utilizados.
  • Monitorea el Tamaño del Repositorio: Mantén un ojo en el tamaño de tu repositorio para asegurarte de que sigue siendo manejable.

Conclusión

Git LFS es una herramienta poderosa para gestionar archivos grandes en un repositorio Git. Al rastrear archivos grandes con Git LFS, puedes mantener el tamaño de tu repositorio bajo control y mejorar el rendimiento. Recuerda seguir las mejores prácticas para el rastreo selectivo y la limpieza regular para mantener un flujo de trabajo eficiente.
