Cherry-Picking en Git
Guardando Cambios (Stashing) ►

Cherry-picking en Git te permite aplicar los cambios introducidos por algunos commits existentes en tu rama actual. Esto puede ser particularmente útil cuando deseas incorporar cambios específicos de una rama a otra sin fusionar toda la rama.

¿Qué es Cherry-Picking?

  • Definición: Cherry-picking es el proceso de seleccionar commits específicos de una rama y aplicarlos a otra.
  • Casos de Uso:
    • Aplicar correcciones de errores de una rama a otra.
    • Incorporar características o cambios específicos sin fusionar toda la rama.
    • Retroportar cambios a versiones anteriores del código.

Cómo Hacer Cherry-Picking

Cherry-Picking Básico

Para hacer cherry-pick de un commit, necesitas el hash del commit (SHA) que deseas aplicar.

  1. Encontrar el Hash del Commit:

    git log

    Este comando mostrará el historial de commits. Copia el hash del commit que deseas hacer cherry-pick.

  2. Hacer Cherry-Pick del Commit:

    git cherry-pick <commit-hash>

    Reemplaza <commit-hash> con el hash real del commit.

Ejemplo

Supongamos que tienes el siguiente historial de commits:

commit 1a2b3c4d5e6f7g8h9i0j
Author: John Doe <john@example.com>
Date:   Mon Oct 4 12:34:56 2021 -0700

    Fix bug in user authentication

commit 0a9b8c7d6e5f4g3h2i1j
Author: Jane Smith <jane@example.com>
Date:   Sun Oct 3 11:23:45 2021 -0700

    Add new feature to dashboard

Si deseas aplicar la corrección de errores del commit 1a2b3c4d5e6f7g8h9i0j a tu rama actual, ejecutarías:

git cherry-pick 1a2b3c4d5e6f7g8h9i0j

Resolución de Conflictos

A veces, hacer cherry-pick puede resultar en conflictos si los cambios en el commit que estás haciendo cherry-pick entran en conflicto con los cambios en tu rama actual.

  1. Identificar Conflictos: Git te notificará de los conflictos y marcará los archivos en conflicto.

  2. Resolver Conflictos: Abre los archivos en conflicto y resuelve manualmente los conflictos. Busca los marcadores de conflicto (<<<<<<<, =======, >>>>>>>).

  3. Marcar Conflictos como Resueltos: Después de resolver los conflictos, márcalos como resueltos:

    git add <resolved-file>

  4. Continuar con el Cherry-Picking:

    git cherry-pick --continue

Cherry-Picking de Múltiples Commits

Puedes hacer cherry-pick de un rango de commits especificando los hashes de los commits:

git cherry-pick <start-commit-hash>^..<end-commit-hash>

Por ejemplo, para hacer cherry-pick de los commits desde 0a9b8c7d6e5f4g3h2i1j hasta 1a2b3c4d5e6f7g8h9i0j, ejecutarías:

git cherry-pick 0a9b8c7d6e5f4g3h2i1j^..1a2b3c4d5e6f7g8h9i0j

Mejores Prácticas

  • Evitar Cherry-Picking de Cambios Grandes: El cherry-picking es más adecuado para cambios pequeños y aislados. Para cambios más grandes, considera fusionar o rebasar.
  • Documentar los Cherry-Picks: Lleva un registro de los commits cherry-picked para evitar confusiones y posibles conflictos en el futuro.
  • Probar a Fondo: Siempre prueba los cambios después de hacer cherry-pick para asegurarte de que se integren bien con la rama actual.

Conclusión

Cherry-picking es una herramienta poderosa en Git que te permite aplicar selectivamente cambios de una rama a otra. Al entender cómo hacer cherry-pick de commits, resolver conflictos y seguir las mejores prácticas, puedes gestionar e incorporar eficazmente cambios específicos en tu código. Esta técnica es particularmente útil para aplicar correcciones de errores, retroportar cambios e integrar características específicas sin fusionar ramas enteras.
