Introducción

Git es un poderoso sistema de control de versiones que ayuda a los desarrolladores a gestionar los cambios en su base de código. Entender la terminología básica de Git es esencial para usar Git de manera efectiva. Esta sección cubrirá términos y conceptos clave que forman la base de Git.

Conceptos Clave

Repositorio (Repo)

  • Definición: Un repositorio es un espacio de almacenamiento donde vive tu proyecto. Puede ser local en una carpeta en tu computadora o alojado en un servidor remoto como GitHub.
  • Ejemplo: 
    git init my-repo
    Este comando inicializa un nuevo repositorio Git en el directorio my-repo.

Commit

  • Definición: Un commit es una instantánea de tu repositorio en un momento específico. Registra cambios en los archivos y directorios.
  • Ejemplo: 
    git commit -m "Initial commit"
    Este comando crea un nuevo commit con el mensaje "Initial commit".

Rama

  • Definición: Una rama es una versión paralela de tu repositorio. Te permite trabajar en diferentes características o correcciones de manera independiente.
  • Ejemplo: 
    git branch new-feature
    Este comando crea una nueva rama llamada new-feature.

Merge

  • Definición: Hacer merge es el proceso de combinar cambios de diferentes ramas en una sola.
  • Ejemplo: 
    git merge new-feature
    Este comando hace merge de la rama new-feature en la rama actual.

Clonar

  • Definición: Clonar es el proceso de crear una copia de un repositorio remoto en tu máquina local.
  • Ejemplo: 
    git clone https://github.com/user/repo.git
    Este comando clona el repositorio ubicado en la URL dada a tu máquina local.

Pull

  • Definición: Hacer pull es el proceso de obtener cambios de un repositorio remoto y hacer merge de ellos en tu repositorio local.
  • Ejemplo: 
    git pull origin main
    Este comando obtiene y hace merge de los cambios de la rama main del repositorio remoto llamado origin.

Push

  • Definición: Hacer push es el proceso de enviar tus commits locales a un repositorio remoto.
  • Ejemplo: 
    git push origin main
    Este comando envía tus commits locales a la rama main del repositorio remoto llamado origin.

Área de Staging (Índice)

  • Definición: El área de staging es un lugar donde puedes agrupar cambios antes de hacer commit de ellos.
  • Ejemplo: 
    git add file.txt
    Este comando añade file.txt al área de staging.

Remoto

  • Definición: Un remoto es una versión de tu repositorio alojada en internet u otra red.
  • Ejemplo: 
    git remote add origin https://github.com/user/repo.git
    Este comando añade un repositorio remoto llamado origin con la URL dada.

Resumen

Entender estos términos básicos de Git es crucial para usar Git de manera efectiva en tu flujo de trabajo de desarrollo. Aquí tienes un resumen rápido:

  • Repositorio: Espacio de almacenamiento para tu proyecto.
  • Commit: Instantánea de tu repositorio.
  • Rama: Versión paralela de tu repositorio.
  • Merge: Combinación de cambios de diferentes ramas.
  • Clonar: Copiar un repositorio remoto localmente.
  • Pull: Obtener y hacer merge de cambios de un repositorio remoto.
  • Push: Enviar commits locales a un repositorio remoto.
  • Área de Staging: Agrupar cambios antes de hacer commit.
  • Remoto: Una versión de tu repositorio alojada en una red.

Dominando estos términos, estarás bien equipado para navegar y utilizar Git de manera efectiva en tus proyectos.
