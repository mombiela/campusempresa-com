Revisión Final y Envío

Resumen

En esta sección final, revisaremos los conceptos y comandos clave cubiertos a lo largo del curso. Esto asegurará que tengas una comprensión sólida de Git desde niveles principiantes hasta avanzados. También cubriremos las mejores prácticas para enviar tu proyecto final o repositorio de código.

Revisión de Conceptos Clave

Comandos Básicos de Git

  • git init: Inicializa un nuevo repositorio Git.
  • git clone: Clona un repositorio existente.
  • git status: Verifica el estado de tu directorio de trabajo y área de preparación.
  • git add: Prepara cambios para el próximo commit.
  • git commit: Confirma los cambios preparados en el repositorio.
  • git push: Envía los commits locales a un repositorio remoto.
  • git pull: Obtiene y fusiona cambios desde un repositorio remoto.
# Inicializa un nuevo repositorio Git
git init

# Clona un repositorio existente
git clone https://github.com/user/repo.git

# Verifica el estado de tu directorio de trabajo
git status

# Prepara un archivo para commit
git add filename.txt

# Confirma los cambios preparados con un mensaje
git commit -m "Initial commit"

# Envía los commits locales al repositorio remoto
git push origin main

# Obtiene y fusiona cambios desde el repositorio remoto
git pull origin main

Ramas y Fusiones

  • git branch: Lista, crea o elimina ramas.
  • git checkout: Cambia de rama o restaura archivos del directorio de trabajo.
  • git merge: Fusiona ramas en la rama actual.
# Lista todas las ramas
git branch

# Crea una nueva rama
git branch feature-branch

# Cambia a la nueva rama
git checkout feature-branch

# Fusiona feature-branch en main
git checkout main
git merge feature-branch

Comandos Avanzados de Git

  • git rebase: Reaplica commits sobre otra punta base.
  • git stash: Guarda cambios en un directorio de trabajo sucio.
  • git cherry-pick: Aplica los cambios introducidos por algunos commits existentes.
# Reaplica la rama actual sobre main
git checkout feature-branch
git rebase main

# Guarda cambios
git stash

# Aplica cambios guardados
git stash apply

# Aplica un commit específico
git cherry-pick commit_hash

Colaboración y Flujo de Trabajo

  • Forking: Crea una copia personal del proyecto de otra persona.
  • Pull Requests: Propón cambios a un repositorio.
  • Revisiones de Código: Revisa y discute cambios en el código.

Mejores Prácticas para el Envío

  • Historial de Commits Limpio: Asegúrate de que tu historial de commits sea limpio y significativo. Usa git rebase para combinar commits innecesarios.
  • Mensajes de Commit Descriptivos: Escribe mensajes de commit claros y descriptivos.
  • Convenciones de Nombres de Ramas: Usa nombres de ramas consistentes y descriptivos.
  • Calidad del Código: Asegúrate de que tu código siga los estándares y directrices del proyecto.
  • Documentación: Incluye la documentación y comentarios necesarios en tu código.

Ejemplo de Flujo de Trabajo para el Envío

  1. Haz un Fork del Repositorio: Crea una copia personal del repositorio al que quieres contribuir.
  2. Clona el Fork: Clona tu repositorio bifurcado en tu máquina local.
  3. Crea una Rama: Crea una nueva rama para tu característica o corrección de errores.
  4. Haz Cambios: Implementa tus cambios y confiérmalos con mensajes descriptivos.
  5. Envía al Remoto: Envía tu rama a tu repositorio bifurcado.
  6. Crea un Pull Request: Abre un pull request al repositorio original.
# Haz un fork del repositorio en GitHub

# Clona tu repositorio bifurcado
git clone https://github.com/your-username/repo.git

# Navega al directorio del repositorio
cd repo

# Crea una nueva rama
git checkout -b feature-branch

# Haz cambios y confiérmalos
git add .
git commit -m "Add new feature"

# Envía la rama a tu repositorio bifurcado
git push origin feature-branch

# Abre un pull request en GitHub

Conclusión

¡Felicidades por completar el curso de Git! Ahora deberías tener una comprensión completa de Git, desde comandos básicos hasta flujos de trabajo avanzados. Recuerda seguir las mejores prácticas para una colaboración limpia y eficiente. ¡Feliz codificación!
