Introducción

En esta sección, describiremos el proyecto en el que trabajarás a lo largo de este curso de Scala. Este proyecto te ayudará a aplicar los conceptos que aprendas de una manera práctica y práctica. Al final del curso, tendrás una aplicación completamente funcional que mostrará tus habilidades de programación en Scala.

Descripción del proyecto

El proyecto que desarrollarás es una Aplicación de Gestión de Tareas. Esta aplicación permitirá a los usuarios crear, actualizar, eliminar y ver tareas. También incluirá características como la categorización de tareas, fechas de vencimiento y niveles de prioridad.

Características clave

  • Autenticación de usuarios: Los usuarios pueden registrarse, iniciar sesión y cerrar sesión.
  • Operaciones CRUD de tareas: Los usuarios pueden crear, leer, actualizar y eliminar tareas.
  • Categorización de tareas: Las tareas se pueden categorizar en diferentes grupos.
  • Fechas de vencimiento y niveles de prioridad: Las tareas pueden tener fechas de vencimiento y niveles de prioridad asignados.
  • Búsqueda y filtrado: Los usuarios pueden buscar y filtrar tareas según varios criterios.

Requisitos

Para completar con éxito este proyecto, deberás cumplir con los siguientes requisitos:

Requisitos técnicos

  • Versión de Scala: Utiliza Scala 2.13 o posterior.
  • Herramienta de construcción: Utiliza SBT (Simple Build Tool) para gestionar dependencias y construir el proyecto.
  • Base de datos: Utiliza una base de datos en memoria como H2 para el desarrollo y las pruebas.
  • Frameworks y bibliotecas:
    • Play Framework: Para construir la aplicación web.
    • Slick: Para la interacción con la base de datos.
    • ScalaTest: Para escribir y ejecutar pruebas.

Requisitos funcionales

  • Autenticación de usuarios:
    • Los usuarios deben poder registrarse con un nombre de usuario y una contraseña.
    • Los usuarios deben poder iniciar y cerrar sesión.
  • Gestión de tareas:
    • Los usuarios deben poder crear nuevas tareas con un título, descripción, fecha de vencimiento y nivel de prioridad.
    • Los usuarios deben poder ver una lista de todas sus tareas.
    • Los usuarios deben poder actualizar los detalles de una tarea.
    • Los usuarios deben poder eliminar una tarea.
  • Categorización y filtrado:
    • Los usuarios deben poder categorizar las tareas en diferentes grupos.
    • Los usuarios deben poder filtrar las tareas según la categoría, la fecha de vencimiento y el nivel de prioridad.

Código de ejemplo

Aquí tienes un ejemplo simple de cómo podrías definir una clase Task en Scala:

case class Task(
  id: Long,
  title: String,
  description: String,
  dueDate: Option[java.time.LocalDate],
  priority: PriorityLevel,
  category: Option[String]
)

sealed trait PriorityLevel
case object High extends PriorityLevel
case object Medium extends PriorityLevel
case object Low extends PriorityLevel

Explicación

  • Clase Case: Task es una clase case que representa una tarea con varios atributos.
  • Tipo Option: dueDate y category son campos opcionales, representados por el tipo Option.
  • Trait Sellado: PriorityLevel es un trait sellado con tres valores posibles: High, Medium y Low.

Conclusión

En esta sección, hemos descrito el proyecto en el que trabajarás a lo largo del curso. Al comprender los requisitos del proyecto y las características clave, estarás mejor preparado para aplicar los conceptos de Scala que aprendas de manera práctica. A medida que avances en el curso, irás construyendo gradualmente esta Aplicación de Gestión de Tareas, mejorando tus habilidades desde niveles principiantes hasta avanzados.
