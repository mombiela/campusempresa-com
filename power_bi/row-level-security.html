Introducción a la Seguridad a Nivel de Fila (RLS)

La Seguridad a Nivel de Fila (RLS) en Power BI te permite restringir el acceso a los datos para usuarios específicos. Los filtros restringen los datos a nivel de fila, y puedes definir filtros dentro de roles.

Conceptos Clave

  • Roles: Un conjunto de reglas que definen a qué datos puede acceder un usuario.
  • Filtros DAX: Las Expresiones de Análisis de Datos (DAX) se utilizan para crear filtros para roles.
  • RLS Estática: Los roles y filtros están predefinidos y no cambian.
  • RLS Dinámica: Los roles y filtros se determinan en tiempo de ejecución basados en atributos del usuario.

Configuración de la Seguridad a Nivel de Fila

Creación de Roles en Power BI Desktop

  1. Abre Power BI Desktop y carga tu modelo de datos.
  2. Navega a la pestaña de Modelado.
  3. Selecciona Administrar Roles.
  4. Crea un Nuevo Rol y define el filtro DAX.

Ejemplo

Supongamos que tienes una tabla llamada Ventas con una columna Región. Quieres crear un rol que solo permita el acceso a los datos donde la Región sea "Norte".

[Región] = "Norte"

Asignación de Usuarios a Roles

  1. Publica tu informe en el servicio de Power BI.
  2. Navega al conjunto de datos en el servicio de Power BI.
  3. Selecciona Seguridad.
  4. Agrega miembros a los roles que creaste.

Seguridad a Nivel de Fila Dinámica

Uso de las Funciones USERNAME() o USERPRINCIPALNAME()

La RLS dinámica te permite filtrar datos basados en el usuario que ha iniciado sesión.

Ejemplo

Supongamos que tienes una tabla llamada Ventas y una tabla llamada Usuarios con las columnas CorreoUsuario y Región. Quieres filtrar Ventas basado en el correo del usuario.

  1. Crea una relación entre las tablas Ventas y Usuarios.
  2. Define el filtro DAX en el rol:
[CorreoUsuario] = USERPRINCIPALNAME()

Implementación de RLS Dinámica

  1. Crea una tabla que mapee a los usuarios con sus respectivos datos.
  2. Usa funciones DAX para filtrar datos dinámicamente.

Ejemplo

Ventas[Región] = LOOKUPVALUE(Usuarios[Región], Usuarios[CorreoUsuario], USERPRINCIPALNAME())

Prueba de la Seguridad a Nivel de Fila

Prueba en Power BI Desktop

  1. Navega a la pestaña de Modelado.
  2. Selecciona Ver como Roles.
  3. Elige el rol que quieres probar.

Prueba en el Servicio de Power BI

  1. Ve al conjunto de datos.
  2. Selecciona Seguridad.
  3. Prueba como Rol para ver los datos como un usuario específico.

Mejores Prácticas para RLS

  • Minimiza el número de roles: Demasiados roles pueden complicar la gestión.
  • Usa RLS dinámica cuando sea posible: Reduce la necesidad de múltiples roles.
  • Prueba exhaustivamente: Asegúrate de que los roles y filtros funcionen como se espera.
  • Documenta tu configuración de RLS: Mantén un registro de los roles y sus filtros para el mantenimiento.

Conclusión

La Seguridad a Nivel de Fila (RLS) en Power BI es una característica poderosa que te ayuda a controlar el acceso a los datos a un nivel granular. Al comprender e implementar tanto RLS estática como dinámica, puedes asegurarte de que los usuarios solo vean los datos que se supone deben ver. Recuerda probar exhaustivamente tus configuraciones de RLS y seguir las mejores prácticas para mantener un entorno de datos seguro y eficiente.
