Configuración de Remoting
Usando Invoke-Command ►

PowerShell Remoting es una característica poderosa que te permite ejecutar comandos en sistemas remotos. Este tema te guiará a través del proceso de configuración y uso de PowerShell Remoting, desde la configuración básica hasta configuraciones avanzadas.

Introducción a PowerShell Remoting

PowerShell Remoting permite a los administradores gestionar múltiples sistemas desde una sola consola. Utiliza el protocolo WS-Management y puede configurarse para usar HTTP o HTTPS para una comunicación segura.

Conceptos Clave

  • Protocolo WS-Management: Un protocolo estándar para la gestión remota.
  • WinRM (Windows Remote Management): La implementación de Microsoft del WS-Management.
  • Endpoints: Configuraciones que definen qué comandos se pueden ejecutar de forma remota.
  • Sesiones: Conexiones a sistemas remotos donde se ejecutan los comandos.

Habilitando PowerShell Remoting

Antes de que puedas usar PowerShell Remoting, necesitas habilitarlo en los sistemas que deseas gestionar.

Habilitando Remoting en una Sola Máquina

Para habilitar PowerShell Remoting en una sola máquina, usa el cmdlet Enable-PSRemoting:

Enable-PSRemoting -Force
  • -Force: Omite los avisos del usuario y fuerza la ejecución del comando.

Habilitando Remoting en Múltiples Máquinas

Para habilitar remoting en múltiples máquinas, puedes usar Group Policy o un script que ejecute el cmdlet Enable-PSRemoting en cada máquina.

Usando Group Policy

  1. Abre la Consola de Administración de Políticas de Grupo (GPMC).
  2. Crea un nuevo Objeto de Política de Grupo (GPO) o edita uno existente.
  3. Navega a Configuración del equipo -> Plantillas administrativas -> Componentes de Windows -> Administración remota de Windows (WinRM) -> Servicio WinRM.
  4. Habilita la política Permitir la administración remota del servidor a través de WinRM.

Usando un Script

Puedes usar un script para habilitar remoting en múltiples máquinas. Aquí tienes un ejemplo usando PowerShell:

$computers = @("Computer1", "Computer2", "Computer3")
foreach ($computer in $computers) {
    Invoke-Command -ComputerName $computer -ScriptBlock {
        Enable-PSRemoting -Force
    }
}

Configurando Remoting para HTTPS

Para una comunicación segura, se recomienda configurar PowerShell Remoting para usar HTTPS.

Pasos para Configurar HTTPS

  1. Crear u Obtener un Certificado: Necesitas un certificado para el servidor que sea confiable para las máquinas cliente.
  2. Configurar el Listener: Usa el cmdlet New-WSManInstance para crear un nuevo listener.

Ejemplo: Configurando Listener HTTPS

# Crear un nuevo listener HTTPS
$certThumbprint = "YOUR_CERT_THUMBPRINT"
New-WSManInstance -ResourceURI winrm/config/Listener -SelectorSet @{Address="*";Transport="HTTPS"} -ValueSet @{CertificateThumbprint=$certThumbprint}
  • CertificateThumbprint: La huella digital del certificado que estás usando.

Verificando el Listener

Para verificar que el listener está configurado correctamente, usa el cmdlet Get-WSManInstance:

Get-WSManInstance -ResourceURI winrm/config/Listener -SelectorSet @{Address="*";Transport="HTTPS"}

Usando PowerShell Remoting

Una vez que el remoting está habilitado y configurado, puedes comenzar a usarlo para ejecutar comandos en sistemas remotos.

Ejecutando Comandos en una Sola Máquina Remota

Usa el cmdlet Invoke-Command para ejecutar comandos en una máquina remota:

Invoke-Command -ComputerName "RemoteComputer" -ScriptBlock { Get-Process }
  • -ComputerName: Especifica la computadora remota.
  • -ScriptBlock: Los comandos a ejecutar en la computadora remota.

Ejecutando Comandos en Múltiples Máquinas Remotas

También puedes ejecutar comandos en múltiples máquinas remotas especificando una matriz de nombres de computadoras:

$computers = @("Computer1", "Computer2", "Computer3")
Invoke-Command -ComputerName $computers -ScriptBlock { Get-Process }

Entrando en una Sesión Remota

Para entrar en una sesión interactiva con una computadora remota, usa el cmdlet Enter-PSSession:

Enter-PSSession -ComputerName "RemoteComputer"
  • Exit-PSSession: Para salir de la sesión remota, simplemente escribe Exit-PSSession.

Configuraciones Avanzadas

Personalizando Endpoints

Puedes crear endpoints personalizados para controlar qué comandos y scripts se pueden ejecutar de forma remota.

Ejemplo: Creando un Endpoint Personalizado

$endpoint = @{
    Name = "CustomEndpoint"
    ConfigurationName = "CustomConfig"
    RunAsUser = "Domain\User"
}
Register-PSSessionConfiguration @endpoint
  • Name: El nombre del endpoint.
  • ConfigurationName: La configuración a usar.
  • RunAsUser: La cuenta de usuario bajo la cual se ejecutarán los comandos.

Gestionando Sesiones

Puedes gestionar sesiones remotas usando los cmdlets New-PSSession, Get-PSSession y Remove-PSSession.

Ejemplo: Creando y Gestionando Sesiones

# Crear una nueva sesión
$session = New-PSSession -ComputerName "RemoteComputer"

# Ejecutar un comando en la sesión
Invoke-Command -Session $session -ScriptBlock { Get-Process }

# Obtener todas las sesiones activas
Get-PSSession

# Eliminar la sesión
Remove-PSSession -Session $session

Conclusión

Configurar PowerShell Remoting es una habilidad crucial para gestionar múltiples sistemas de manera eficiente. Al habilitar remoting, configurar una comunicación segura y usar cmdlets para ejecutar comandos y gestionar sesiones, puedes agilizar las tareas administrativas y mejorar la productividad. Practica estos pasos para volverte competente en el uso de PowerShell Remoting en tus operaciones diarias.
