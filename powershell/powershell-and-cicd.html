PowerShell y la Integración/Despliegue Continuo (CI/CD)
Infraestructura como Código con PowerShell ►

Introducción a CI/CD

  • Integración Continua (CI): La práctica de fusionar todas las copias de trabajo de los desarrolladores en una línea principal compartida varias veces al día.
  • Despliegue Continuo (CD): La práctica de desplegar automáticamente cada cambio que pasa todas las etapas de tu pipeline de producción a producción.

¿Por qué usar PowerShell para CI/CD?

  • Automatización: Los scripts de PowerShell pueden automatizar tareas repetitivas.
  • Integración: PowerShell puede interactuar con varias herramientas de CI/CD como Jenkins, Azure DevOps y GitHub Actions.
  • Flexibilidad: PowerShell se puede usar en diferentes entornos (Windows, Linux, macOS).

Configurando tu Entorno

Prerrequisitos

  • Instalar PowerShell Core (si no está ya instalado).
  • Familiaridad con comandos básicos de PowerShell.
  • Acceso a una herramienta de CI/CD (por ejemplo, Jenkins, Azure DevOps, GitHub Actions).

Instalando Módulos de PowerShell

# Instalar módulo de Azure DevOps
Install-Module -Name AzureDevOps -Scope CurrentUser

# Instalar módulo de Jenkins
Install-Module -Name Jenkins -Scope CurrentUser

Tareas Básicas de CI/CD con PowerShell

Ejecutando Pruebas Unitarias

  • Usa PowerShell para ejecutar pruebas unitarias como parte del pipeline de CI.
# Ejemplo: Ejecutando pruebas de Pester
Invoke-Pester -Path .\Tests

Construyendo Proyectos

  • Automatiza el proceso de construcción usando scripts de PowerShell.
# Ejemplo: Construyendo un proyecto .NET
dotnet build .\MyProject.csproj

Desplegando Aplicaciones

  • Usa PowerShell para desplegar aplicaciones en diferentes entornos.
# Ejemplo: Desplegando una aplicación web en Azure
az webapp deploy --resource-group MyResourceGroup --name MyWebApp --src-path .\MyApp.zip

Integrando PowerShell con Herramientas de CI/CD

Jenkins

  • Usa scripts de PowerShell en pipelines de Jenkins.
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                powershell 'dotnet build .\\MyProject.csproj'
            }
        }
        stage('Test') {
            steps {
                powershell 'Invoke-Pester -Path .\\Tests'
            }
        }
        stage('Deploy') {
            steps {
                powershell 'az webapp deploy --resource-group MyResourceGroup --name MyWebApp --src-path .\\MyApp.zip'
            }
        }
    }
}

Azure DevOps

  • Usa tareas de PowerShell en Azure Pipelines.
trigger:
- main

pool:
  vmImage: 'windows-latest'

steps:
- task: PowerShell@2
  inputs:
    targetType: 'inline'
    script: 'dotnet build .\MyProject.csproj'

- task: PowerShell@2
  inputs:
    targetType: 'inline'
    script: 'Invoke-Pester -Path .\Tests'

- task: AzureCLI@2
  inputs:
    azureSubscription: 'MyAzureSubscription'
    scriptType: 'ps'
    scriptLocation: 'inlineScript'
    inlineScript: 'az webapp deploy --resource-group MyResourceGroup --name MyWebApp --src-path .\MyApp.zip'

GitHub Actions

  • Usa PowerShell en workflows de GitHub Actions.
name: CI/CD Pipeline

on: [push]

jobs:
  build:
    runs-on: windows-latest

    steps:
    - uses: actions/checkout@v2

    - name: Build
      run: dotnet build .\MyProject.csproj
      shell: pwsh

    - name: Test
      run: Invoke-Pester -Path .\Tests
      shell: pwsh

    - name: Deploy
      run: az webapp deploy --resource-group MyResourceGroup --name MyWebApp --src-path .\MyApp.zip
      shell: pwsh

CI/CD Avanzado con PowerShell

Módulos Personalizados de PowerShell para CI/CD

  • Crea módulos personalizados de PowerShell para encapsular la lógica de CI/CD.
# Ejemplo: Módulo personalizado para despliegue
function Deploy-WebApp {
    param (
        [string]$ResourceGroup,
        [string]$WebAppName,
        [string]$SourcePath
    )
    az webapp deploy --resource-group $ResourceGroup --name $WebAppName --src-path $SourcePath
}

Export-ModuleMember -Function Deploy-WebApp

Manejo de Errores y Registro

  • Implementa un manejo robusto de errores y registro en scripts de PowerShell.
try {
    # Ejemplo: Proceso de construcción
    dotnet build .\MyProject.csproj
} catch {
    Write-Error "La construcción falló: $_"
    exit 1
}

try {
    # Ejemplo: Proceso de despliegue
    Deploy-WebApp -ResourceGroup "MyResourceGroup" -WebAppName "MyWebApp" -SourcePath ".\MyApp.zip"
} catch {
    Write-Error "El despliegue falló: $_"
    exit 1
}

Conclusión

PowerShell es una herramienta poderosa para automatizar pipelines de CI/CD. Al integrar scripts de PowerShell con herramientas de CI/CD como Jenkins, Azure DevOps y GitHub Actions, puedes optimizar tus procesos de desarrollo y despliegue. Ya sea que estés ejecutando pruebas, construyendo proyectos o desplegando aplicaciones, PowerShell proporciona la flexibilidad y el control necesarios para gestionar tus flujos de trabajo de CI/CD de manera efectiva.
