Parámetros de Script
Modularizando Código con Funciones ►

En esta sección, exploraremos cómo usar parámetros de script en PowerShell. Los parámetros de script te permiten pasar datos a tus scripts, haciéndolos más flexibles y reutilizables. Cubriremos los conceptos básicos para principiantes y avanzaremos hacia conceptos más avanzados.

Introducción a los Parámetros de Script

Los parámetros de script son una forma de hacer que tus scripts sean más dinámicos al permitirte pasar diferentes valores cada vez que ejecutas el script. Esto es particularmente útil para tareas de automatización y scripting que requieren entradas variables.

Sintaxis Básica

Para definir parámetros en un script de PowerShell, usas la palabra clave param seguida de una lista de parámetros encerrados entre paréntesis.

param (
    [string]$Name,
    [int]$Age
)

Write-Output "Name: $Name"
Write-Output "Age: $Age"
  • [string]$Name: Esto define un parámetro llamado Name de tipo string.
  • [int]$Age: Esto define un parámetro llamado Age de tipo int.

Ejecutar el Script con Parámetros

Para ejecutar el script con parámetros, simplemente proporcionas los valores después del nombre del script.

.\MyScript.ps1 -Name "Alice" -Age 30

Valores Predeterminados

También puedes establecer valores predeterminados para los parámetros, que se usarán si no se proporciona ningún valor.

param (
    [string]$Name = "John Doe",
    [int]$Age = 25
)

Write-Output "Name: $Name"
Write-Output "Age: $Age"

Parámetros Obligatorios

Para hacer que un parámetro sea obligatorio, puedes usar el atributo Mandatory.

param (
    [Parameter(Mandatory=$true)]
    [string]$Name,
    [int]$Age
)

Write-Output "Name: $Name"
Write-Output "Age: $Age"

Si ejecutas el script sin proporcionar el parámetro Name, PowerShell te pedirá que lo ingreses.

Parámetros de Script Avanzados

Validación de Parámetros

PowerShell proporciona varios atributos para validar los valores de los parámetros.

  • ValidateSet: Asegura que el valor del parámetro sea uno de un conjunto predefinido de valores.
  • ValidateRange: Asegura que el valor del parámetro esté dentro de un rango especificado.
  • ValidatePattern: Asegura que el valor del parámetro coincida con un patrón de expresión regular.
param (
    [ValidateSet("Red", "Green", "Blue")]
    [string]$Color,

    [ValidateRange(1, 100)]
    [int]$Percentage,

    [ValidatePattern("^[a-zA-Z]+$")]
    [string]$Name
)

Write-Output "Color: $Color"
Write-Output "Percentage: $Percentage"
Write-Output "Name: $Name"

Alias de Parámetros

Puedes definir alias para los parámetros para proporcionar nombres alternativos.

param (
    [Alias("FN")]
    [string]$FirstName,

    [Alias("LN")]
    [string]$LastName
)

Write-Output "First Name: $FirstName"
Write-Output "Last Name: $LastName"

Uso de `CmdletBinding`

El atributo CmdletBinding convierte tu script en una función avanzada, permitiéndote usar parámetros comunes y otras características avanzadas.

[CmdletBinding()]
param (
    [string]$Name,
    [int]$Age
)

Write-Output "Name: $Name"
Write-Output "Age: $Age"

Conjuntos de Parámetros

Los conjuntos de parámetros te permiten definir diferentes conjuntos de parámetros que pueden usarse en diferentes escenarios.

[CmdletBinding()]
param (
    [Parameter(Mandatory=$true, ParameterSetName="Set1")]
    [string]$Name,

    [Parameter(Mandatory=$true, ParameterSetName="Set2")]
    [int]$Age
)

if ($PSCmdlet.ParameterSetName -eq "Set1") {
    Write-Output "Name: $Name"
} elseif ($PSCmdlet.ParameterSetName -eq "Set2") {
    Write-Output "Age: $Age"
}

Conclusión

Los parámetros de script son una característica poderosa en PowerShell que pueden mejorar enormemente la flexibilidad y reutilización de tus scripts. Al entender y utilizar características básicas y avanzadas de los parámetros, puedes crear scripts más dinámicos y robustos. Ya sea que estés estableciendo valores predeterminados, haciendo parámetros obligatorios, o usando validación avanzada y conjuntos de parámetros, dominar los parámetros de script es esencial para una escritura efectiva de scripts en PowerShell.
