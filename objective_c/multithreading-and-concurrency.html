Multithreading y Concurrencia en Objective-C
Redes ►

Introducción a Multithreading y Concurrencia

  • Multithreading: La capacidad de una CPU o un solo núcleo en un procesador multinúcleo para proporcionar múltiples hilos de ejecución concurrentemente.
  • Concurrencia: La ejecución de varias secuencias de instrucciones al mismo tiempo.

¿Por qué Usar Multithreading?

  • Mejora del Rendimiento: Utilizar eficientemente los recursos de la CPU realizando múltiples operaciones simultáneamente.
  • Responsividad: Mantener la interfaz de usuario receptiva realizando tareas de larga duración en segundo plano.
  • Compartición de Recursos: Compartir recursos como memoria y datos entre múltiples hilos.

Conceptos Básicos de Multithreading en Objective-C

  • NSThread: Una forma básica de crear y gestionar hilos.
  • GCD (Grand Central Dispatch): Una API de alto nivel para gestionar operaciones concurrentes.
  • NSOperationQueue: Una forma orientada a objetos de gestionar operaciones.

Usando NSThread

// Crear y comenzar un nuevo hilo
NSThread *myThread = [[NSThread alloc] initWithTarget:self selector:@selector(myThreadMainMethod) object:nil];
[myThread start];

// Método principal del hilo
- (void)myThreadMainMethod {
    @autoreleasepool {
        // Realiza tu tarea aquí
        NSLog(@"El hilo está corriendo");
    }
}
  • Explicación:
    • NSThread se inicializa con un objetivo (self), un selector (método a ejecutar) y un objeto opcional.
    • El método start comienza la ejecución del hilo.
    • El método myThreadMainMethod contiene el código a ejecutar en el nuevo hilo.

Usando GCD (Grand Central Dispatch)

// Despachar una tarea a una cola de fondo
dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
    // Realiza tu tarea en segundo plano aquí
    NSLog(@"La tarea en segundo plano está corriendo");
    
    // Actualizar la UI en el hilo principal
    dispatch_async(dispatch_get_main_queue(), ^{
        // Realiza actualizaciones de la UI aquí
        NSLog(@"UI actualizada en el hilo principal");
    });
});
  • Explicación:
    • dispatch_async programa un bloque de código para ejecutarse de manera asíncrona.
    • dispatch_get_global_queue obtiene una cola concurrente global.
    • dispatch_get_main_queue obtiene la cola principal, utilizada para actualizaciones de la UI.

Usando NSOperationQueue

// Crear una cola de operaciones
NSOperationQueue *operationQueue = [[NSOperationQueue alloc] init];

// Crear una operación
NSBlockOperation *operation = [NSBlockOperation blockOperationWithBlock:^{
    // Realiza tu tarea aquí
    NSLog(@"La operación está corriendo");
}];

// Agregar la operación a la cola
[operationQueue addOperation:operation];
  • Explicación:
    • NSOperationQueue gestiona una cola de objetos NSOperation.
    • NSBlockOperation es una subclase concreta de NSOperation que ejecuta un bloque de código.
    • addOperation agrega la operación a la cola para su ejecución.

Conceptos Avanzados

  • Sincronización de Hilos: Asegurar que múltiples hilos puedan acceder a recursos compartidos sin conflictos.
  • Deadlock: Una situación donde dos o más hilos están bloqueados para siempre, esperando unos por otros.
  • Condición de Carrera: Una condición donde el resultado depende de la secuencia o el tiempo de eventos incontrolables.

Sincronización de Hilos con @synchronized

- (void)criticalSection {
    @synchronized(self) {
        // Código que accede a recursos compartidos
        NSLog(@"Accediendo a recurso compartido");
    }
}
  • Explicación:
    • @synchronized asegura que solo un hilo pueda ejecutar el bloque de código a la vez.

Evitando Deadlocks

  • Mejores Prácticas:
    • Siempre adquirir bloqueos en un orden consistente.
    • Evitar bloqueos anidados cuando sea posible.
    • Usar mecanismos de tiempo de espera.

Manejo de Condiciones de Carrera

  • Propiedades Atómicas: Asegurar que una propiedad sea accedida de manera segura en hilos.
@property (atomic, strong) NSString *sharedString;
  • Explicación:
    • atomic asegura que la propiedad sea accedida de manera segura en hilos, previniendo condiciones de carrera.

Conclusión

Multithreading y concurrencia son herramientas poderosas en Objective-C que pueden mejorar significativamente el rendimiento y la responsividad de tus aplicaciones. Al entender y utilizar NSThread, GCD y NSOperationQueue, puedes gestionar eficientemente tareas concurrentes. Siempre ten en cuenta la sincronización de hilos, deadlocks y condiciones de carrera para asegurar que tus aplicaciones multihilo funcionen sin problemas.
