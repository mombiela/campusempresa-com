Polimorfismo en Objective-C
Categorías y Extensiones ►

El polimorfismo es un concepto central en la programación orientada a objetos que permite tratar objetos como instancias de su clase padre en lugar de su clase real. Esto permite que una sola interfaz represente diferentes formas subyacentes (tipos de datos). En Objective-C, el polimorfismo se logra a través de la sobrescritura de métodos y la tipificación dinámica.

Conceptos Clave del Polimorfismo

  • Sobrescritura de Métodos: Las subclases pueden proporcionar una implementación específica de un método que ya está definido en su superclase.
  • Tipificación Dinámica: Objective-C permite enviar mensajes a objetos sin conocer su tipo exacto en tiempo de compilación.
  • Tipo id: Un tipo de objeto genérico que puede contener cualquier objeto.

Sobrescritura de Métodos

La sobrescritura de métodos permite que una subclase proporcione una implementación específica de un método que ya está definido en su superclase. Este es un aspecto fundamental del polimorfismo.

Ejemplo

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Animal *myAnimal = [[Animal alloc] init];
        [myAnimal speak]; // Output: Animal speaks
        
        Dog *myDog = [[Dog alloc] init];
        [myDog speak]; // Output: Dog barks
    }
    return 0;
}

En este ejemplo:

  • La clase Animal tiene un método speak.
  • La clase Dog sobrescribe el método speak para proporcionar su propia implementación.
  • Cuando se llama a speak en un objeto Animal, utiliza la implementación de Animal.
  • Cuando se llama a speak en un objeto Dog, utiliza la implementación de Dog.

Tipificación Dinámica

Objective-C soporta la tipificación dinámica, lo que permite enviar mensajes a objetos sin conocer su tipo exacto en tiempo de compilación. Esto es particularmente útil para el polimorfismo.

Ejemplo

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

@interface Cat : Animal
@end

@implementation Cat
- (void)speak {
    NSLog(@"Cat meows");
}
@end

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        NSArray *animals = @[[[Dog alloc] init], [[Cat alloc] init], [[Animal alloc] init]];
        
        for (Animal *animal in animals) {
            [animal speak];
        }
    }
    return 0;
}

En este ejemplo:

  • Se crea un array de objetos Animal, que contiene instancias de Dog, Cat y Animal.
  • Se llama al método speak en cada objeto del array.
  • Se llama a la implementación correcta de speak para cada objeto, demostrando el polimorfismo.

Tipo id

El tipo id es un tipo de objeto genérico en Objective-C que puede contener cualquier objeto. Esto es útil para el polimorfismo porque permite escribir código que puede trabajar con objetos de cualquier tipo.

Ejemplo

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

@interface Cat : Animal
@end

@implementation Cat
- (void)speak {
    NSLog(@"Cat meows");
}
@end

void makeAnimalSpeak(id animal) {
    [animal speak];
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        id myDog = [[Dog alloc] init];
        id myCat = [[Cat alloc] init];
        
        makeAnimalSpeak(myDog); // Output: Dog barks
        makeAnimalSpeak(myCat); // Output: Cat meows
    }
    return 0;
}

En este ejemplo:

  • La función makeAnimalSpeak toma un parámetro de tipo id, permitiéndole aceptar cualquier objeto.
  • Se llama a la implementación correcta de speak para cada objeto pasado a makeAnimalSpeak.

Conclusión

El polimorfismo en Objective-C permite un código flexible y reutilizable al permitir que los objetos sean tratados como instancias de su clase padre. A través de la sobrescritura de métodos, la tipificación dinámica y el uso del tipo id, Objective-C proporciona herramientas poderosas para implementar el polimorfismo. Comprender y utilizar estos conceptos te ayudará a escribir código más eficiente y mantenible.
