Introducción

Integrar Node.js con tecnologías frontend es una habilidad crucial para los desarrolladores full-stack. Esta sección te guiará a través del proceso de conectar tu backend de Node.js con varios frameworks y bibliotecas frontend. Al final de este tema, comprenderás cómo servir archivos estáticos, usar motores de plantillas y crear APIs que puedan ser consumidas por aplicaciones frontend.

Servir Archivos Estáticos

Node.js puede servir archivos estáticos como HTML, CSS y JavaScript. Este es a menudo el primer paso para integrar tu backend con el frontend.

Ejemplo: Servir Archivos Estáticos con Express

const express = require('express');
const path = require('path');
const app = express();

// Servir archivos estáticos desde el directorio 'public'
app.use(express.static(path.join(__dirname, 'public')));

app.get('/', (req, res) => {
  res.sendFile(path.join(__dirname, 'public', 'index.html'));
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`El servidor está corriendo en el puerto ${PORT}`);
});
  • Explicación:
    • El middleware express.static se usa para servir archivos estáticos.
    • path.join ayuda a construir la ruta a los archivos estáticos.
    • res.sendFile envía el archivo HTML al cliente.

Uso de Motores de Plantillas

Los motores de plantillas te permiten generar HTML dinámicamente. Los motores de plantillas populares incluyen EJS, Pug y Handlebars.

Ejemplo: Usar EJS con Express

const express = require('express');
const app = express();

// Configurar EJS como el motor de plantillas
app.set('view engine', 'ejs');

app.get('/', (req, res) => {
  res.render('index', { title: 'Página Principal', message: '¡Bienvenido a Node.js con EJS!' });
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`El servidor está corriendo en el puerto ${PORT}`);
});
  • Explicación:
    • app.set('view engine', 'ejs') configura EJS como el motor de plantillas.
    • res.render renderiza la plantilla EJS y la envía al cliente.

Creación de APIs para Consumo Frontend

Las APIs son esenciales para la comunicación frontend-backend. Puedes crear APIs RESTful usando Node.js y Express.

Ejemplo: Crear una API REST Simple

const express = require('express');
const app = express();

app.use(express.json());

let items = [
  { id: 1, name: 'Item 1' },
  { id: 2, name: 'Item 2' }
];

// Obtener todos los ítems
app.get('/api/items', (req, res) => {
  res.json(items);
});

// Obtener un ítem por ID
app.get('/api/items/:id', (req, res) => {
  const item = items.find(i => i.id === parseInt(req.params.id));
  if (!item) return res.status(404).send('Ítem no encontrado');
  res.json(item);
});

// Crear un nuevo ítem
app.post('/api/items', (req, res) => {
  const newItem = {
    id: items.length + 1,
    name: req.body.name
  };
  items.push(newItem);
  res.status(201).json(newItem);
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`El servidor está corriendo en el puerto ${PORT}`);
});
  • Explicación:
    • El middleware express.json se usa para analizar cuerpos JSON.
    • app.get define rutas para obtener datos.
    • app.post define una ruta para crear nuevos datos.

Integración con Frameworks Frontend

Los frameworks frontend modernos como React, Angular y Vue.js pueden consumir APIs creadas con Node.js.

Ejemplo: Obtener Datos en React

import React, { useEffect, useState } from 'react';

function App() {
  const [items, setItems] = useState([]);

  useEffect(() => {
    fetch('/api/items')
      .then(response => response.json())
      .then(data => setItems(data));
  }, []);

  return (
    <div>
      <h1>Ítems</h1>
      <ul>
        {items.map(item => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
}

export default App;
  • Explicación:
    • useEffect se usa para obtener datos de la API cuando el componente se monta.
    • fetch hace una solicitud HTTP a la API de Node.js.
    • setItems actualiza el estado con los datos obtenidos.

Conclusión

Integrar Node.js con tecnologías frontend implica servir archivos estáticos, usar motores de plantillas, crear APIs y consumir esas APIs en aplicaciones frontend. Al dominar estas habilidades, puedes construir aplicaciones full-stack robustas que proporcionen una experiencia de usuario fluida.
