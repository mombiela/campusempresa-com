Servir Archivos Estáticos
Introducción a las Bases de Datos ►

En esta sección, exploraremos cómo servir archivos estáticos usando Node.js. Los archivos estáticos incluyen HTML, CSS, JavaScript, imágenes y otros recursos que no cambian dinámicamente. Servir estos archivos de manera eficiente es crucial para construir aplicaciones web.

Conceptos Clave

  • Archivos Estáticos: Archivos que no cambian dinámicamente y se sirven tal cual al cliente.
  • Node.js: Un entorno de ejecución de JavaScript construido sobre el motor de JavaScript V8 de Chrome, comúnmente utilizado para el desarrollo del lado del servidor.
  • Express.js: Un marco de aplicación web minimalista y flexible para Node.js que proporciona características robustas para aplicaciones web y móviles.

Configuración del Proyecto

Antes de comenzar a servir archivos estáticos, configuremos un proyecto básico de Node.js.

  1. Inicializar un Proyecto de Node.js:

    mkdir static-files-example
cd static-files-example
npm init -y

  2. Instalar Express.js:

    npm install express

  3. Crear la Estructura del Proyecto:

    static-files-example/
├── public/
│   ├── css/
│   │   └── styles.css
│   ├── js/
│   │   └── script.js
│   └── index.html
├── server.js
└── package.json

Servir Archivos Estáticos con Express.js

Express.js hace que sea sencillo servir archivos estáticos usando el middleware express.static.

Ejemplo Básico

  1. Crear server.js:

    const express = require('express');
const path = require('path');
const app = express();

// Servir archivos estáticos desde el directorio 'public'
app.use(express.static(path.join(__dirname, 'public')));

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});

  2. Crear public/index.html:

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Static Files Example</title>
    <link rel="stylesheet" href="css/styles.css">
</head>
<body>
    <h1>Hello, World!</h1>
    <script src="js/script.js"></script>
</body>
</html>

  3. Crear public/css/styles.css:

    body {
    font-family: Arial, sans-serif;
    background-color: #f0f0f0;
    text-align: center;
    padding: 50px;
}

  4. Crear public/js/script.js:

    document.addEventListener('DOMContentLoaded', () => {
    console.log('JavaScript is working!');
});

Explicación

  • express.static: Este middleware sirve archivos estáticos desde el directorio especificado. En este caso, sirve archivos desde el directorio public.
  • path.join: Este método une los segmentos de ruta dados en una sola ruta, asegurando la compatibilidad entre plataformas.

Configuración Avanzada

Servir Múltiples Directorios

Puedes servir archivos estáticos desde múltiples directorios llamando a express.static varias veces.

app.use(express.static(path.join(__dirname, 'public')));
app.use(express.static(path.join(__dirname, 'assets')));

Personalizar el Control de Caché

Puedes personalizar los encabezados de control de caché para los archivos estáticos pasando opciones a express.static.

app.use(express.static(path.join(__dirname, 'public'), {
    maxAge: '1d', // Cachear archivos por un día
    etag: false   // Deshabilitar encabezados ETag
}));

Manejar Errores 404 para Archivos Estáticos

Para manejar errores 404 para archivos estáticos, puedes agregar una función de middleware después del middleware de archivos estáticos.

app.use((req, res, next) => {
    res.status(404).send('File not found');
});

Conclusión

Servir archivos estáticos es un aspecto fundamental de la construcción de aplicaciones web con Node.js y Express.js. Al entender cómo configurar y configurar el servicio de archivos estáticos, puedes asegurarte de que tu aplicación web entregue recursos de manera eficiente y efectiva. Este conocimiento forma la base para temas más avanzados en desarrollo web, como la optimización del rendimiento y la implementación de medidas de seguridad.
