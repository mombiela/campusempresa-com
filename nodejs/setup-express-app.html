Introducción

Express es un framework de aplicaciones web minimalista y flexible para Node.js que proporciona un conjunto robusto de características para aplicaciones web y móviles. En esta sección, cubriremos cómo configurar una aplicación Express desde cero, incluyendo la instalación, configuración básica y la creación de tu primera ruta.

Prerrequisitos

Antes de comenzar, asegúrate de tener lo siguiente instalado:

  • Node.js (versión 12 o superior)
  • npm (Node Package Manager)

Guía Paso a Paso

1. Inicializar un Nuevo Proyecto Node.js

Primero, crea un nuevo directorio para tu proyecto y navega dentro de él usando tu terminal o símbolo del sistema.

mkdir my-express-app
cd my-express-app

A continuación, inicializa un nuevo proyecto Node.js usando npm:

npm init -y

Este comando creará un archivo package.json con configuraciones predeterminadas.

2. Instalar Express

Para instalar Express, ejecuta el siguiente comando:

npm install express

Esto añadirá Express a tu proyecto y actualizará el archivo package.json para incluir Express como una dependencia.

3. Crear el Archivo de la Aplicación

Crea un nuevo archivo llamado app.js en tu directorio de proyecto. Este archivo será el punto de entrada para tu aplicación Express.

// app.js
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('¡Hola Mundo!');
});

app.listen(port, () => {
  console.log(`Aplicación de ejemplo escuchando en http://localhost:${port}`);
});

4. Ejecutar la Aplicación

Para iniciar tu aplicación Express, ejecuta el siguiente comando en tu terminal:

node app.js

Abre tu navegador web y navega a http://localhost:3000. Deberías ver el mensaje "¡Hola Mundo!".

Conceptos Clave

  • Aplicación Express: El componente principal que maneja solicitudes y respuestas.
  • Middleware: Funciones que se ejecutan durante el ciclo de vida de una solicitud al servidor Express.
  • Ruteo: Definir endpoints (URIs) y cómo la aplicación responde a las solicitudes del cliente.

Ejemplo: Añadiendo Middleware y Rutas

Añadiendo Middleware

Las funciones middleware son funciones que tienen acceso al objeto de solicitud (req), al objeto de respuesta (res) y a la siguiente función middleware en el ciclo de vida de la solicitud-respuesta de la aplicación.

// app.js
const express = require('express');
const app = express();
const port = 3000;

// Función middleware para registrar solicitudes
app.use((req, res, next) => {
  console.log(`${req.method} solicitud para '${req.url}'`);
  next();
});

app.get('/', (req, res) => {
  res.send('¡Hola Mundo!');
});

app.listen(port, () => {
  console.log(`Aplicación de ejemplo escuchando en http://localhost:${port}`);
});

Definiendo Rutas

Las rutas definen los endpoints de tu aplicación y especifican cómo debería responder a las solicitudes del cliente.

// app.js
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('¡Hola Mundo!');
});

app.get('/about', (req, res) => {
  res.send('Página Acerca de');
});

app.get('/contact', (req, res) => {
  res.send('Página de Contacto');
});

app.listen(port, () => {
  console.log(`Aplicación de ejemplo escuchando en http://localhost:${port}`);
});

Resumen

En esta sección, cubrimos los conceptos básicos de la configuración de una aplicación Express. Comenzamos inicializando un nuevo proyecto Node.js, instalando Express y creando un archivo de aplicación básico. También discutimos conceptos clave como middleware y ruteo, y proporcionamos ejemplos para ilustrar cómo añadir middleware y definir rutas en tu aplicación.

Siguiendo estos pasos, ahora deberías tener una comprensión básica de cómo configurar y ejecutar una aplicación Express. En las próximas secciones, profundizaremos en temas más avanzados como el manejo de errores, el uso de plantillas y la conexión a bases de datos.
