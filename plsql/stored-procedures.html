Procedimientos Almacenados en PL/SQL
Funciones ►

Los procedimientos almacenados son una característica poderosa en PL/SQL que te permiten encapsular y reutilizar código. Este tema te guiará desde los conceptos básicos hasta los avanzados de los procedimientos almacenados en PL/SQL.

Introducción a los Procedimientos Almacenados

Los procedimientos almacenados son subprogramas almacenados en la base de datos que pueden ser ejecutados para realizar una tarea específica. Ayudan a mejorar el rendimiento, mantener la consistencia y mejorar la seguridad.

Conceptos Clave

  • Definición: Un procedimiento almacenado es un bloque PL/SQL nombrado que puede aceptar parámetros y ser invocado para realizar una tarea específica.
  • Ventajas:
    • Rendimiento: Reducción del tráfico de red y mejor velocidad de ejecución.
    • Reusabilidad: El código puede ser reutilizado en múltiples aplicaciones.
    • Seguridad: El acceso puede ser controlado a través de permisos.
    • Mantenibilidad: Más fácil de gestionar y actualizar el código.

Creación de Procedimientos Almacenados

Para crear un procedimiento almacenado, se utiliza la sentencia CREATE PROCEDURE. Aquí está la sintaxis básica:

CREATE [OR REPLACE] PROCEDURE nombre_procedimiento 
  [ (nombre_parametro [IN | OUT | IN OUT] tipo_dato [, ...]) ]
IS
  -- Declaraciones
BEGIN
  -- Sentencias ejecutables
EXCEPTION
  -- Manejo de excepciones
END nombre_procedimiento;

Ejemplo: Procedimiento Almacenado Simple

Vamos a crear un procedimiento almacenado simple que imprime "¡Hola, Mundo!".

CREATE OR REPLACE PROCEDURE decir_hola IS
BEGIN
  DBMS_OUTPUT.PUT_LINE('¡Hola, Mundo!');
END decir_hola;

Para ejecutar este procedimiento, puedes usar:

BEGIN
  decir_hola;
END;

Parámetros en Procedimientos Almacenados

Los procedimientos almacenados pueden aceptar parámetros para hacerlos más flexibles y dinámicos. Hay tres tipos de parámetros:

  • IN: Pasa un valor al procedimiento.
  • OUT: Devuelve un valor del procedimiento.
  • IN OUT: Pasa un valor al procedimiento y devuelve un valor actualizado.

Ejemplo: Procedimiento con Parámetros

Aquí hay un ejemplo de un procedimiento almacenado con parámetros IN y OUT:

CREATE OR REPLACE PROCEDURE saludar_usuario (
  p_nombre IN VARCHAR2,
  p_saludo OUT VARCHAR2
) IS
BEGIN
  p_saludo := '¡Hola, ' || p_nombre || '!';
END saludar_usuario;

Para ejecutar este procedimiento:

DECLARE
  v_saludo VARCHAR2(50);
BEGIN
  saludar_usuario('Alicia', v_saludo);
  DBMS_OUTPUT.PUT_LINE(v_saludo);
END;

Manejo de Excepciones en Procedimientos Almacenados

El manejo de excepciones es crucial para gestionar errores y asegurar la robustez de tus procedimientos almacenados.

Ejemplo: Procedimiento con Manejo de Excepciones

CREATE OR REPLACE PROCEDURE dividir_numeros (
  p_num1 IN NUMBER,
  p_num2 IN NUMBER,
  p_resultado OUT NUMBER
) IS
BEGIN
  p_resultado := p_num1 / p_num2;
EXCEPTION
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('Error: División por cero.');
    p_resultado := NULL;
END dividir_numeros;

Para ejecutar este procedimiento:

DECLARE
  v_resultado NUMBER;
BEGIN
  dividir_numeros(10, 0, v_resultado);
  DBMS_OUTPUT.PUT_LINE('Resultado: ' || v_resultado);
END;

Temas Avanzados

Uso de Cursores en Procedimientos Almacenados

Los cursores te permiten obtener y manipular datos fila por fila.

Ejemplo: Procedimiento con Cursor

CREATE OR REPLACE PROCEDURE listar_empleados IS
  CURSOR emp_cursor IS
    SELECT nombre, apellido FROM empleados;
  v_nombre empleados.nombre%TYPE;
  v_apellido empleados.apellido%TYPE;
BEGIN
  OPEN emp_cursor;
  LOOP
    FETCH emp_cursor INTO v_nombre, v_apellido;
    EXIT WHEN emp_cursor%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE(v_nombre || ' ' || v_apellido);
  END LOOP;
  CLOSE emp_cursor;
END listar_empleados;

SQL Dinámico en Procedimientos Almacenados

El SQL dinámico te permite construir y ejecutar sentencias SQL dinámicamente en tiempo de ejecución.

Ejemplo: Procedimiento con SQL Dinámico

CREATE OR REPLACE PROCEDURE ejecutar_sql_dinamico (
  p_nombre_tabla IN VARCHAR2
) IS
  v_sql VARCHAR2(200);
BEGIN
  v_sql := 'SELECT COUNT(*) FROM ' || p_nombre_tabla;
  EXECUTE IMMEDIATE v_sql INTO v_count;
  DBMS_OUTPUT.PUT_LINE('Cantidad: ' || v_count);
END ejecutar_sql_dinamico;

Conclusión

Los procedimientos almacenados en PL/SQL son una herramienta poderosa para encapsular y reutilizar código. Mejoran el rendimiento, la seguridad y la mantenibilidad. Al entender cómo crear, usar parámetros, manejar excepciones e implementar características avanzadas como cursores y SQL dinámico, puedes aprovechar los procedimientos almacenados para construir aplicaciones de base de datos robustas y eficientes.
