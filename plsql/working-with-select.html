Introducción

La sentencia SELECT es uno de los comandos más fundamentales y frecuentemente utilizados en SQL y PL/SQL. Te permite recuperar datos de una o más tablas en una base de datos. Esta sección cubrirá los conceptos básicos de la sentencia SELECT y gradualmente avanzará hacia un uso más avanzado.

Sentencia SELECT Básica

La forma más simple de una sentencia SELECT recupera todas las columnas de una sola tabla.

SELECT * FROM employees;
  • SELECT: Especifica las columnas a recuperar.
  • *: Un carácter comodín que significa "todas las columnas."
  • FROM: Especifica la tabla de la cual recuperar datos.
  • employees: El nombre de la tabla.

Seleccionando Columnas Específicas

También puedes especificar qué columnas deseas recuperar.

SELECT first_name, last_name, salary FROM employees;
  • first_name, last_name, salary: Las columnas a recuperar.

Filtrando Datos con la Cláusula WHERE

La cláusula WHERE te permite filtrar las filas que se devuelven.

SELECT first_name, last_name, salary 
FROM employees 
WHERE department_id = 10;
  • WHERE: Especifica la condición para filtrar filas.
  • department_id = 10: La condición que debe cumplirse para que una fila se incluya en el resultado.

Usando Operadores de Comparación

Puedes usar varios operadores de comparación en la cláusula WHERE.

  • =: Igual a
  • <>: No igual a
  • >: Mayor que
  • <: Menor que
  • >=: Mayor o igual que
  • <=: Menor o igual que
SELECT first_name, last_name, salary 
FROM employees 
WHERE salary > 50000;

Ordenando Datos con ORDER BY

La cláusula ORDER BY te permite ordenar el conjunto de resultados.

SELECT first_name, last_name, salary 
FROM employees 
ORDER BY salary DESC;
  • ORDER BY: Especifica la(s) columna(s) por la(s) que ordenar.
  • salary: La columna por la que ordenar.
  • DESC: Ordena el resultado en orden descendente. Usa ASC para orden ascendente (predeterminado).

Agrupando Datos con GROUP BY

La cláusula GROUP BY se usa para agrupar filas que tienen los mismos valores en columnas especificadas en filas de resumen.

SELECT department_id, AVG(salary) AS average_salary 
FROM employees 
GROUP BY department_id;
  • GROUP BY: Especifica la(s) columna(s) por la(s) que agrupar.
  • AVG(salary): Una función de agregación que calcula el salario promedio.
  • AS average_salary: Un alias para la columna calculada.

Funciones de Agregación Comunes

  • AVG(): Calcula el valor promedio.
  • COUNT(): Cuenta el número de filas.
  • SUM(): Calcula la suma de valores.
  • MAX(): Encuentra el valor máximo.
  • MIN(): Encuentra el valor mínimo.

Filtrando Grupos con HAVING

La cláusula HAVING se usa para filtrar grupos basados en una condición.

SELECT department_id, AVG(salary) AS average_salary 
FROM employees 
GROUP BY department_id 
HAVING AVG(salary) > 60000;
  • HAVING: Especifica la condición para filtrar grupos.
  • AVG(salary) > 60000: La condición que debe cumplirse para que un grupo se incluya en el resultado.

Uniendo Tablas

Puedes recuperar datos de múltiples tablas usando uniones.

INNER JOIN

Un INNER JOIN devuelve filas cuando hay una coincidencia en ambas tablas.

SELECT e.first_name, e.last_name, d.department_name 
FROM employees e 
INNER JOIN departments d ON e.department_id = d.department_id;
  • INNER JOIN: Especifica el tipo de unión.
  • ON: Especifica la condición para la unión.

LEFT JOIN

Un LEFT JOIN devuelve todas las filas de la tabla de la izquierda y las filas coincidentes de la tabla de la derecha.

SELECT e.first_name, e.last_name, d.department_name 
FROM employees e 
LEFT JOIN departments d ON e.department_id = d.department_id;

RIGHT JOIN

Un RIGHT JOIN devuelve todas las filas de la tabla de la derecha y las filas coincidentes de la tabla de la izquierda.

SELECT e.first_name, e.last_name, d.department_name 
FROM employees e 
RIGHT JOIN departments d ON e.department_id = d.department_id;

Subconsultas

Una subconsulta es una consulta anidada dentro de otra consulta.

Subconsulta Simple

Una subconsulta simple devuelve un solo valor.

SELECT first_name, last_name 
FROM employees 
WHERE department_id = (SELECT department_id FROM departments WHERE department_name = 'Sales');

Subconsulta Correlacionada

Una subconsulta correlacionada hace referencia a columnas de la consulta externa.

SELECT first_name, last_name 
FROM employees e 
WHERE salary > (SELECT AVG(salary) FROM employees WHERE department_id = e.department_id);

Conclusión

La sentencia SELECT es una herramienta poderosa para consultar datos en PL/SQL. Al dominar sus diversas cláusulas y opciones, puedes recuperar y manipular datos de manera eficiente. Practica usando diferentes tipos de sentencias SELECT para volverte competente en la recuperación y análisis de datos.
