Control de Transacciones en PL/SQL
Procedimientos Almacenados ►

El control de transacciones es un aspecto crucial de la gestión de bases de datos, asegurando la integridad y consistencia de los datos. En PL/SQL, las sentencias de control de transacciones te permiten gestionar los cambios realizados por sentencias DML (Lenguaje de Manipulación de Datos) como INSERT, UPDATE y DELETE.

Conceptos Clave

  • Transacción: Una secuencia de una o más sentencias SQL que se ejecutan como una sola unidad de trabajo.
  • Commit: Guarda todos los cambios realizados durante la transacción actual.
  • Rollback: Deshace todos los cambios realizados durante la transacción actual.
  • Savepoint: Establece un punto dentro de una transacción al cual puedes volver más tarde.

Sentencias de Control de Transacciones

COMMIT

La sentencia COMMIT se utiliza para guardar todos los cambios realizados durante la transacción actual. Una vez que una transacción se confirma, los cambios son permanentes y visibles para otros usuarios.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (101, 'John', 'Doe');
    
    COMMIT;
END;

ROLLBACK

La sentencia ROLLBACK deshace todos los cambios realizados durante la transacción actual. Esto es útil si ocurre un error y necesitas revertir la base de datos a su estado anterior.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (102, 'Jane', 'Smith');
    
    ROLLBACK;
END;

SAVEPOINT

La sentencia SAVEPOINT establece un punto dentro de una transacción al cual puedes volver más tarde. Esto permite reversiones parciales dentro de una transacción.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (103, 'Alice', 'Johnson');
    
    SAVEPOINT savepoint1;
    
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (104, 'Bob', 'Brown');
    
    ROLLBACK TO savepoint1;
    
    COMMIT;
END;

En este ejemplo, la inserción de 'Alice Johnson' se guarda, pero la inserción de 'Bob Brown' se revierte.

Ejemplos Prácticos

Ejemplo 1: Control Básico de Transacciones

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (105, 'Charlie', 'Davis');
    
    UPDATE employees
    SET last_name = 'Davison'
    WHERE employee_id = 105;
    
    COMMIT;
END;

En este ejemplo, tanto las sentencias INSERT como UPDATE son parte de una sola transacción. La sentencia COMMIT asegura que ambos cambios se guarden.

Ejemplo 2: Usando SAVEPOINT

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (106, 'Eve', 'White');
    
    SAVEPOINT savepoint1;
    
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (107, 'Frank', 'Black');
    
    SAVEPOINT savepoint2;
    
    DELETE FROM employees
    WHERE employee_id = 106;
    
    ROLLBACK TO savepoint1;
    
    COMMIT;
END;

En este ejemplo:

  • La inserción de 'Eve White' y 'Frank Black' son inicialmente parte de la transacción.
  • La eliminación de 'Eve White' se revierte a savepoint1, por lo que 'Eve White' permanece en la base de datos.
  • Luego, la transacción se confirma, guardando la inserción de 'Frank Black'.

Conclusión

El control de transacciones en PL/SQL es esencial para mantener la integridad y consistencia de los datos. Al usar COMMIT, ROLLBACK y SAVEPOINT, puedes gestionar las transacciones de manera efectiva, asegurando que tus operaciones de base de datos sean fiables y sin errores. Entender estos conceptos y cómo aplicarlos mejorará significativamente tu capacidad para escribir código PL/SQL robusto.
