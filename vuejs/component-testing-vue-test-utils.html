Introducción a Vue Test Utils

Vue Test Utils es la biblioteca oficial de utilidades para pruebas unitarias de Vue.js. Proporciona métodos para montar e interactuar con componentes de Vue, facilitando la prueba de su comportamiento y salida.

Conceptos Clave

  • Montaje de Componentes: Renderizar un componente en aislamiento para pruebas.
  • Encontrar Elementos: Consultar el componente renderizado para encontrar elementos.
  • Simulación de Eventos: Desencadenar eventos como clics o cambios de entrada.
  • Aserciones: Verificar si el componente se comporta como se espera.

Configuración de Vue Test Utils

Antes de comenzar a probar, necesitas configurar tu entorno.

Instalación

Para instalar Vue Test Utils, puedes usar npm o yarn:

npm install @vue/test-utils --save-dev

o

yarn add @vue/test-utils --dev

También necesitarás un ejecutor de pruebas como Jest o Mocha. Para este curso, usaremos Jest:

npm install jest --save-dev

Escribiendo tu Primera Prueba

Vamos a escribir una prueba simple para un componente de Vue.

Componente de Ejemplo

Considera un componente Counter simple:

<template>
  <div>
    <p>{{ count }}</p>
    <button @click="increment">Incrementar</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      count: 0
    };
  },
  methods: {
    increment() {
      this.count++;
    }
  }
};
</script>

Escribiendo la Prueba

Crea un archivo de prueba Counter.spec.js:

import { mount } from '@vue/test-utils';
import Counter from '@/components/Counter.vue';

describe('Counter.vue', () => {
  it('incrementa el conteo cuando se hace clic en el botón', async () => {
    const wrapper = mount(Counter);
    
    expect(wrapper.text()).toContain('0');
    
    await wrapper.find('button').trigger('click');
    
    expect(wrapper.text()).toContain('1');
  });
});

Explicación

  • Montaje: mount(Counter) renderiza el componente Counter.
  • Aserciones: expect(wrapper.text()).toContain('0') verifica el conteo inicial.
  • Simulación de Eventos: await wrapper.find('button').trigger('click') simula un clic en el botón.
  • Aserciones: expect(wrapper.text()).toContain('1') verifica el conteo después del clic.

Técnicas Avanzadas de Pruebas

A medida que te sientas más cómodo con las pruebas básicas, puedes explorar técnicas más avanzadas.

Prueba de Props

Puedes probar cómo se comportan los componentes con diferentes props.

import { mount } from '@vue/test-utils';
import Message from '@/components/Message.vue';

describe('Message.vue', () => {
  it('renderiza props.msg cuando se pasa', () => {
    const msg = '¡Hola, Vue!';
    const wrapper = mount(Message, {
      propsData: { msg }
    });
    expect(wrapper.text()).toContain(msg);
  });
});

Prueba de Eventos Emitidos

Puedes probar si un componente emite eventos correctamente.

import { mount } from '@vue/test-utils';
import CustomButton from '@/components/CustomButton.vue';

describe('CustomButton.vue', () => {
  it('emite un evento cuando se hace clic', async () => {
    const wrapper = mount(CustomButton);
    
    await wrapper.find('button').trigger('click');
    
    expect(wrapper.emitted()).toHaveProperty('custom-event');
  });
});

Conclusión

Las pruebas de componentes con Vue Test Utils son una forma poderosa de asegurar que tus componentes de Vue funcionen como se espera. Al dominar los conceptos básicos de montar componentes, simular eventos y hacer aserciones, puedes escribir pruebas robustas que ayuden a mantener la calidad de tu base de código. A medida que avances, explorar props y eventos emitidos mejorará aún más tus capacidades de prueba. ¡Felices pruebas!
