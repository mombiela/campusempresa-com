Introducción a Vuex
Estado, Getters, Mutaciones y Acciones ►

Vuex es un patrón de gestión de estado + biblioteca para aplicaciones Vue.js. Sirve como una tienda centralizada para todos los componentes en una aplicación, con reglas que aseguran que el estado solo pueda ser mutado de manera predecible. Vuex también se integra profundamente con el sistema de reactividad de Vue para hacer la gestión del estado simple y eficiente.

Conceptos Clave

  • Estado: La única fuente de verdad que contiene los datos de la aplicación.
  • Getters: Propiedades computadas para el estado de la tienda, permitiendo que los componentes accedan al estado.
  • Mutaciones: La única forma de cambiar el estado en Vuex, y deben ser sincrónicas.
  • Acciones: Similares a las mutaciones, pero pueden ser asincrónicas. Cometen mutaciones.
  • Módulos: Permiten que la tienda se divida en piezas más pequeñas y manejables.

Configuración de Vuex

Para comenzar a usar Vuex en tu proyecto Vue.js, primero necesitas instalarlo:

npm install vuex --save

Luego, puedes crear una tienda e integrarla en tu instancia de Vue:

import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment(context) {
      context.commit('increment');
    }
  },
  getters: {
    count: state => state.count
  }
});

new Vue({
  el: '#app',
  store,
  render: h => h(App)
});

Estado

El estado es la fuente de datos de tu aplicación. Es un único objeto que contiene todo el estado a nivel de aplicación y sirve como la "única fuente de verdad."

const store = new Vuex.Store({
  state: {
    count: 0
  }
});

Accediendo al Estado en Componentes

Puedes acceder al estado en tus componentes usando this.$store.state:

export default {
  computed: {
    count() {
      return this.$store.state.count;
    }
  }
};

Getters

Los getters son como propiedades computadas para las tiendas. Te permiten definir propiedades que dependen del estado de la tienda.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
    doubleCount: state => state.count * 2
  }
});

Usando Getters en Componentes

Puedes usar getters en tus componentes de manera similar a como usas el estado:

export default {
  computed: {
    doubleCount() {
      return this.$store.getters.doubleCount;
    }
  }
};

Mutaciones

Las mutaciones son la única forma de cambiar el estado en Vuex. Deben ser sincrónicas.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  }
});

Cometiendo Mutaciones

Puedes cometer mutaciones usando this.$store.commit:

export default {
  methods: {
    increment() {
      this.$store.commit('increment');
    }
  }
};

Acciones

Las acciones son similares a las mutaciones, pero pueden ser asincrónicas. Cometen mutaciones.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment');
      }, 1000);
    }
  }
});

Despachando Acciones

Puedes despachar acciones usando this.$store.dispatch:

export default {
  methods: {
    incrementAsync() {
      this.$store.dispatch('incrementAsync');
    }
  }
};

Módulos

Los módulos te permiten dividir tu tienda en piezas más pequeñas y manejables. Cada módulo puede tener su propio estado, mutaciones, acciones y getters.

const moduleA = {
  state: () => ({ count: 0 }),
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment');
    }
  },
  getters: {
    doubleCount: state => state.count * 2
  }
};

const store = new Vuex.Store({
  modules: {
    a: moduleA
  }
});

Conclusión

Vuex es una poderosa herramienta de gestión de estado que se integra perfectamente con Vue.js. Al entender y utilizar sus conceptos básicos—estado, getters, mutaciones, acciones y módulos—puedes gestionar el estado de tu aplicación de una manera más estructurada y predecible. Esta introducción debería darte una base sólida para comenzar a usar Vuex en tus proyectos Vue.js.
