Redis Streams es una estructura de datos poderosa introducida en Redis 5.0, diseñada para gestionar y procesar flujos de datos. Esta sección cubrirá desde los conceptos básicos hasta el uso avanzado de Redis Streams, incluyendo conceptos clave, comandos y ejemplos prácticos.

Introducción a Redis Streams

Redis Streams es una estructura de datos de registro que te permite almacenar múltiples campos y valores de cadena con una secuencia automática basada en el tiempo. Es particularmente útil para el procesamiento de datos en tiempo real, el event sourcing y los sistemas de mensajería.

Conceptos Clave

  • Stream: Una secuencia de entradas, cada una con una ID única.
  • Entry: Un solo registro en el stream, que consiste en una ID única y uno o más pares campo-valor.
  • Consumer Group: Un grupo de consumidores que pueden leer del stream de manera coordinada.
  • Consumer: Un solo cliente que lee entradas del stream.

Comandos Básicos

Añadiendo Entradas a un Stream

Para añadir una entrada a un stream, usa el comando XADD. La sintaxis básica es:

XADD stream_name * field1 value1 field2 value2 ...

Ejemplo:

XADD mystream * sensor-id 1234 temperature 19.8

Leyendo Entradas de un Stream

Para leer entradas de un stream, usa el comando XRANGE. La sintaxis básica es:

XRANGE stream_name start end [COUNT count]

Ejemplo:

XRANGE mystream - +

Este comando lee todas las entradas del stream mystream.

Leyendo Nuevas Entradas

Para leer nuevas entradas añadidas al stream, usa el comando XREAD. La sintaxis básica es:

XREAD COUNT count STREAMS stream_name last_id

Ejemplo:

XREAD COUNT 2 STREAMS mystream $

Este comando lee las dos próximas nuevas entradas de mystream.

Uso Avanzado

Grupos de Consumidores

Los grupos de consumidores permiten que múltiples consumidores lean del mismo stream sin duplicar el trabajo. Usa el comando XGROUP para gestionar grupos de consumidores.

Creando un Grupo de Consumidores

XGROUP CREATE stream_name group_name $ MKSTREAM

Ejemplo:

XGROUP CREATE mystream mygroup $ MKSTREAM

Leyendo Entradas con Grupos de Consumidores

Usa el comando XREADGROUP para leer entradas como parte de un grupo de consumidores.

XREADGROUP GROUP group_name consumer_name COUNT count STREAMS stream_name last_id

Ejemplo:

XREADGROUP GROUP mygroup consumer1 COUNT 2 STREAMS mystream >

Acknowledging Entries

Los consumidores deben reconocer las entradas que han procesado usando el comando XACK.

XACK stream_name group_name entry_id

Ejemplo:

XACK mystream mygroup 1526569495631-0

Gestionando Entradas Pendientes

Para ver las entradas pendientes, usa el comando XPENDING.

XPENDING stream_name group_name [start end count [consumer]]

Ejemplo:

XPENDING mystream mygroup

Ejemplos Prácticos

Ejemplo: Productor-Consumidor Simple

Productor:

XADD mystream * sensor-id 1234 temperature 19.8
XADD mystream * sensor-id 1235 temperature 20.1

Consumidor:

XREAD COUNT 2 STREAMS mystream 0

Ejemplo: Usando Grupos de Consumidores

Crear un grupo de consumidores:

XGROUP CREATE mystream mygroup $ MKSTREAM

Productor:

XADD mystream * sensor-id 1234 temperature 19.8
XADD mystream * sensor-id 1235 temperature 20.1

Consumidor 1:

XREADGROUP GROUP mygroup consumer1 COUNT 2 STREAMS mystream >

Consumidor 2:

XREADGROUP GROUP mygroup consumer2 COUNT 2 STREAMS mystream >

Reconocer entradas procesadas:

XACK mystream mygroup 1526569495631-0

Resumen

Redis Streams es una estructura de datos versátil para manejar flujos de datos en tiempo real. Soporta añadir y leer entradas, gestionar grupos de consumidores y reconocer entradas procesadas. Al entender y utilizar estos comandos, puedes gestionar y procesar eficientemente flujos de datos en tus aplicaciones.
