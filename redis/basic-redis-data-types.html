Tipos de Datos Básicos de Redis
Conceptos Básicos de Redis CLI ►

Redis es un almacén de estructuras de datos en memoria utilizado como base de datos, caché y corredor de mensajes. Entender los tipos de datos básicos en Redis es crucial para aprovechar todo su potencial. Esta sección cubrirá los tipos de datos fundamentales en Redis, incluyendo Cadenas, Listas, Conjuntos, Hashes y Conjuntos Ordenados.

Cadenas

Las cadenas son el tipo de valor más básico que puedes almacenar en Redis. Son seguras para datos binarios, lo que significa que pueden contener cualquier dato, como una imagen JPEG o un objeto Ruby serializado.

  • Conceptos Clave:

    • Las cadenas pueden almacenar hasta 512 MB de datos.
    • Las operaciones comunes incluyen establecer, obtener y agregar valores.

  • Ejemplos:

SET mykey "¡Hola, Redis!"
GET mykey

Explicación:

  • SET mykey "¡Hola, Redis!" almacena la cadena "¡Hola, Redis!" bajo la clave mykey.
  • GET mykey recupera el valor almacenado en mykey.

Listas

Las listas son colecciones de valores ordenados. Están implementadas como listas enlazadas, lo que significa que insertar un elemento en la cabeza o en la cola es muy eficiente.

  • Conceptos Clave:

    • Las listas pueden contener hasta 2^32 - 1 elementos (más de 4 mil millones).
    • Las operaciones comunes incluyen empujar y sacar elementos de ambos extremos.

  • Ejemplos:

LPUSH mylist "Mundo"
LPUSH mylist "Hola"
LRANGE mylist 0 -1

Explicación:

  • LPUSH mylist "Mundo" empuja "Mundo" a la lista mylist.
  • LPUSH mylist "Hola" empuja "Hola" a la lista mylist.
  • LRANGE mylist 0 -1 recupera todos los elementos en la lista mylist.

Conjuntos

Los conjuntos son colecciones de valores únicos y desordenados. Son útiles para almacenar elementos únicos y realizar operaciones de conjuntos como uniones e intersecciones.

  • Conceptos Clave:

    • Los conjuntos no permiten valores duplicados.
    • Las operaciones comunes incluyen agregar, eliminar y verificar la pertenencia.

  • Ejemplos:

SADD myset "Hola"
SADD myset "Mundo"
SADD myset "Hola"  # Esto no agregará "Hola" de nuevo
SMEMBERS myset

Explicación:

  • SADD myset "Hola" agrega "Hola" al conjunto myset.
  • SADD myset "Mundo" agrega "Mundo" al conjunto myset.
  • SMEMBERS myset recupera todos los miembros del conjunto myset.

Hashes

Los hashes son mapas entre campos de cadena y valores de cadena. Son perfectos para representar objetos.

  • Conceptos Clave:

    • Los hashes son ideales para almacenar pequeñas piezas de datos.
    • Las operaciones comunes incluyen establecer, obtener y eliminar campos.

  • Ejemplos:

HSET myhash field1 "Hola"
HSET myhash field2 "Mundo"
HGET myhash field1
HGETALL myhash

Explicación:

  • HSET myhash field1 "Hola" establece el campo field1 en el hash myhash a "Hola".
  • HSET myhash field2 "Mundo" establece el campo field2 en el hash myhash a "Mundo".
  • HGET myhash field1 recupera el valor de field1 en el hash myhash.
  • HGETALL myhash recupera todos los campos y valores en el hash myhash.

Conjuntos Ordenados

Los conjuntos ordenados son similares a los conjuntos pero con una puntuación asociada para cada miembro. Los miembros están ordenados por sus puntuaciones.

  • Conceptos Clave:

    • Los conjuntos ordenados son útiles para sistemas de clasificación.
    • Las operaciones comunes incluyen agregar elementos con puntuaciones, recuperar elementos por puntuación y eliminar elementos.

  • Ejemplos:

ZADD myzset 1 "uno"
ZADD myzset 2 "dos"
ZRANGE myzset 0 -1 WITHSCORES

Explicación:

  • ZADD myzset 1 "uno" agrega el miembro "uno" con una puntuación de 1 al conjunto ordenado myzset.
  • ZADD myzset 2 "dos" agrega el miembro "dos" con una puntuación de 2 al conjunto ordenado myzset.
  • ZRANGE myzset 0 -1 WITHSCORES recupera todos los miembros en el conjunto ordenado myzset junto con sus puntuaciones.

Conclusión

Entender los tipos de datos básicos en Redis es esencial para usar efectivamente este poderoso almacén de estructuras de datos en memoria. Cada tipo de dato sirve para diferentes casos de uso, desde el almacenamiento simple de clave-valor con Cadenas hasta estructuras de datos más complejas como Listas, Conjuntos, Hashes y Conjuntos Ordenados. Dominar estos conceptos básicos proporcionará una base sólida para características y operaciones más avanzadas de Redis.
