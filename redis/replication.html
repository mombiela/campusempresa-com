Replicación en Redis
Sentinel ►

La replicación es una característica fundamental en Redis que permite que los datos de un servidor Redis (el maestro) se repliquen en uno o más servidores Redis (las réplicas). Esto asegura alta disponibilidad, balanceo de carga y redundancia de datos.

Conceptos Clave

  • Maestro: El servidor Redis principal donde se realizan todas las operaciones de escritura.
  • Réplica: Un servidor Redis secundario que recibe datos del maestro y puede servir operaciones de lectura.
  • Enlace de Replicación: La conexión entre los servidores maestro y réplica.
  • Retraso de Replicación: El retraso entre la sincronización de datos del maestro y la réplica.

Configuración de la Replicación

Configuración del Maestro

El servidor maestro no requiere ninguna configuración especial para habilitar la replicación. Simplemente necesita estar en funcionamiento y ser accesible por las réplicas.

Configuración de la Réplica

Para configurar una réplica, necesitas configurarla para que se conecte al maestro. Esto se puede hacer modificando el archivo redis.conf o usando el comando SLAVEOF.

Usando redis.conf

Agrega las siguientes líneas al archivo redis.conf de la réplica:

replicaof <master-ip> <master-port>

Por ejemplo:

replicaof 127.0.0.1 6379

Usando el Comando SLAVEOF

También puedes configurar una réplica en tiempo de ejecución usando el comando SLAVEOF:

SLAVEOF 127.0.0.1 6379

Ejemplo

Aquí hay un ejemplo simple para ilustrar la configuración de un maestro y una réplica:

  1. Iniciar el Servidor Maestro

    redis-server --port 6379

  2. Iniciar el Servidor Réplica

    redis-server --port 6380

  3. Configurar la Réplica para Seguir al Maestro

    redis-cli -p 6380 SLAVEOF 127.0.0.1 6379

Verificación de la Replicación

Para verificar que la replicación está funcionando, puedes usar el comando INFO replication tanto en el maestro como en la réplica.

En el Maestro

redis-cli INFO replication

Salida:

# Replication
role:master
connected_slaves:1
slave0:ip=127.0.0.1,port=6380,state=online,offset=12345,lag=0

En la Réplica

redis-cli -p 6380 INFO replication

Salida:

# Replication
role:slave
master_host:127.0.0.1
master_port:6379
master_link_status:up

Manejo de Failover

El failover es el proceso de promover una réplica a maestro si el maestro falla. Redis no maneja el failover automático por defecto, pero puedes usar Redis Sentinel para este propósito.

Redis Sentinel

Redis Sentinel es un sistema diseñado para gestionar instancias de Redis, proporcionando alta disponibilidad y monitoreo.

Configuración de Sentinel

  1. Crear un Archivo de Configuración de Sentinel

    sentinel monitor mymaster 127.0.0.1 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 60000
sentinel parallel-syncs mymaster 1

  2. Iniciar Sentinel

    redis-sentinel /path/to/sentinel.conf

Ejemplo

Aquí hay un ejemplo de una configuración simple de Sentinel:

  1. Crear Configuración de Sentinel

    sentinel monitor mymaster 127.0.0.1 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 60000
sentinel parallel-syncs mymaster 1

  2. Iniciar Sentinel

    redis-sentinel sentinel.conf

Resumen

La replicación en Redis es una característica poderosa que mejora la disponibilidad de datos, el balanceo de carga y la redundancia. Al configurar un maestro y una o más réplicas, puedes asegurar que tus datos siempre estén disponibles y puedan manejar cargas de lectura aumentadas. Para el failover automático y la alta disponibilidad, Redis Sentinel puede ser usado para monitorear y gestionar tus instancias de Redis. Entender e implementar la replicación es crucial para construir aplicaciones robustas y escalables basadas en Redis.
