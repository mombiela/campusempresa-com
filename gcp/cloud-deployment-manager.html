Introducción a Cloud Deployment Manager

Google Cloud Deployment Manager es un servicio de gestión de infraestructura que automatiza la creación y gestión de recursos de Google Cloud. Te permite definir todos los recursos necesarios para tu aplicación en un formato declarativo utilizando plantillas YAML, Python o Jinja2.

Conceptos Clave

  • Plantillas: Definen los recursos y sus propiedades.
  • Configuraciones: Especifican los recursos a desplegar y sus configuraciones.
  • Despliegues: Una colección de recursos creados y gestionados juntos.

Empezando con Cloud Deployment Manager

Configurando tu Entorno

  1. Instalar Google Cloud SDK: Asegúrate de tener instalado el Google Cloud SDK.
  2. Autenticar: Usa gcloud auth login para autenticar tu cuenta.
  3. Configurar Proyecto: Configura tu proyecto activo usando gcloud config set project [PROJECT_ID].

Creando tu Primer Despliegue

Vamos a crear un despliegue simple que incluya una instancia de Google Compute Engine.

Paso 1: Crear un Archivo de Configuración

Crea un archivo llamado vm.yaml con el siguiente contenido:

resources:
- name: my-vm
  type: compute.v1.instance
  properties:
    zone: us-central1-a
    machineType: zones/us-central1-a/machineTypes/n1-standard-1
    disks:
    - deviceName: boot
      type: PERSISTENT
      boot: true
      autoDelete: true
      initializeParams:
        sourceImage: projects/debian-cloud/global/images/family/debian-9
    networkInterfaces:
    - network: global/networks/default

Paso 2: Desplegar la Configuración

Usa el siguiente comando para desplegar la configuración:

gcloud deployment-manager deployments create my-first-deployment --config vm.yaml

Verificando el Despliegue

Para verificar el despliegue, usa:

gcloud deployment-manager deployments describe my-first-deployment

Conceptos Intermedios

Usando Plantillas

Las plantillas te permiten reutilizar y parametrizar tus configuraciones.

Creando una Plantilla

Crea un archivo llamado vm-template.jinja:

resources:
- name: {{ properties["vmName"] }}
  type: compute.v1.instance
  properties:
    zone: {{ properties["zone"] }}
    machineType: {{ properties["machineType"] }}
    disks:
    - deviceName: boot
      type: PERSISTENT
      boot: true
      autoDelete: true
      initializeParams:
        sourceImage: {{ properties["sourceImage"] }}
    networkInterfaces:
    - network: global/networks/default

Usando la Plantilla en una Configuración

Crea un archivo llamado vm-config.yaml:

imports:
- path: vm-template.jinja

resources:
- name: my-vm-from-template
  type: vm-template.jinja
  properties:
    vmName: my-vm
    zone: us-central1-a
    machineType: zones/us-central1-a/machineTypes/n1-standard-1
    sourceImage: projects/debian-cloud/global/images/family/debian-9

Despliega la configuración:

gcloud deployment-manager deployments create my-template-deployment --config vm-config.yaml

Temas Avanzados

Gestionando Despliegues Complejos

Para despliegues más complejos, puedes usar múltiples plantillas y configuraciones.

Ejemplo: Aplicación Multi-Capa

Crea una red, una regla de firewall y dos instancias de VM (una para el servidor web y otra para la base de datos).

  1. Plantilla de Red (network-template.jinja):
resources:
- name: {{ properties["networkName"] }}
  type: compute.v1.network
  properties:
    autoCreateSubnetworks: true
  1. Plantilla de Firewall (firewall-template.jinja):
resources:
- name: {{ properties["firewallName"] }}
  type: compute.v1.firewall
  properties:
    network: global/networks/{{ properties["networkName"] }}
    allowed:
    - IPProtocol: tcp
      ports:
      - 80
  1. Plantilla de VM (vm-template.jinja):
resources:
- name: {{ properties["vmName"] }}
  type: compute.v1.instance
  properties:
    zone: {{ properties["zone"] }}
    machineType: {{ properties["machineType"] }}
    disks:
    - deviceName: boot
      type: PERSISTENT
      boot: true
      autoDelete: true
      initializeParams:
        sourceImage: {{ properties["sourceImage"] }}
    networkInterfaces:
    - network: global/networks/{{ properties["networkName"] }}
  1. Archivo de Configuración (multi-tier-config.yaml):
imports:
- path: network-template.jinja
- path: firewall-template.jinja
- path: vm-template.jinja

resources:
- name: my-network
  type: network-template.jinja
  properties:
    networkName: my-network

- name: my-firewall
  type: firewall-template.jinja
  properties:
    firewallName: my-firewall
    networkName: my-network

- name: web-server
  type: vm-template.jinja
  properties:
    vmName: web-server
    zone: us-central1-a
    machineType: zones/us-central1-a/machineTypes/n1-standard-1
    sourceImage: projects/debian-cloud/global/images/family/debian-9
    networkName: my-network

- name: db-server
  type: vm-template.jinja
  properties:
    vmName: db-server
    zone: us-central1-a
    machineType: zones/us-central1-a/machineTypes/n1-standard-1
    sourceImage: projects/debian-cloud/global/images/family/debian-9
    networkName: my-network

Despliega la configuración:

gcloud deployment-manager deployments create multi-tier-deployment --config multi-tier-config.yaml

Conclusión

Google Cloud Deployment Manager es una herramienta poderosa para gestionar tu infraestructura en la nube de manera declarativa. Usando plantillas y configuraciones, puedes automatizar el despliegue de aplicaciones complejas, asegurando consistencia y reduciendo el potencial de errores humanos. A medida que te familiarices más con Deployment Manager, podrás aprovechar sus capacidades completas para gestionar despliegues a gran escala de manera eficiente.
