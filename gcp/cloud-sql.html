Introducción a Cloud SQL

Cloud SQL es un servicio de base de datos relacional completamente gestionado para MySQL, PostgreSQL y SQL Server. Te ayuda a configurar, mantener, gestionar y administrar tus bases de datos relacionales en Google Cloud Platform.

Conceptos Clave

  • Servicio Completamente Gestionado: Cloud SQL automatiza las copias de seguridad, la replicación y la conmutación por error para garantizar alta disponibilidad y fiabilidad.
  • Escalabilidad: Escala fácilmente los recursos de cómputo y almacenamiento de tu base de datos con un tiempo de inactividad mínimo.
  • Seguridad: Integrado con Google Cloud IAM y VPC para acceso seguro y cifrado de datos.
  • Compatibilidad: Soporta MySQL, PostgreSQL y SQL Server.

Configuración de Cloud SQL

Creación de una Instancia de Cloud SQL

  1. Navegar a Cloud SQL en la Consola de GCP:

    • Ve a la Consola de GCP.
    • Selecciona "SQL" del menú de la izquierda.

  2. Crear una Nueva Instancia:

    • Haz clic en "Crear Instancia".
    • Elige tu motor de base de datos deseado (MySQL, PostgreSQL o SQL Server).

  3. Configurar Ajustes de la Instancia:

    • Establece el ID de la instancia, la contraseña de root y la región.
    • Configura el tipo de máquina y las opciones de almacenamiento.

  4. Finalizar y Crear:

    • Revisa tus configuraciones y haz clic en "Crear".

Ejemplo: Creación de una Instancia MySQL

gcloud sql instances create my-instance \
    --database-version=MYSQL_5_7 \
    --tier=db-n1-standard-1 \
    --region=us-central1

Conectándose a Cloud SQL

  • Usando Cloud Shell: 
    gcloud sql connect my-instance --user=root
  • Usando un Cliente Local:
    • Instala el proxy de autenticación de Cloud SQL.
    • Inicia el proxy: 
      ./cloud_sql_proxy -instances=<INSTANCE_CONNECTION_NAME>=tcp:3306
    • Conéctate usando tu cliente MySQL local: 
      mysql -u root -p --host 127.0.0.1 --port 3306

Gestión de Instancias de Cloud SQL

Copias de Seguridad y Restauraciones

  • Copias de Seguridad Automatizadas: Habilita copias de seguridad automatizadas para asegurar la recuperación de datos.
  • Copias de Seguridad Manuales: Crea copias de seguridad manuales para puntos específicos en el tiempo.

Ejemplo: Habilitando Copias de Seguridad Automatizadas

gcloud sql instances patch my-instance --backup-start-time 23:00

Monitoreo y Registro

  • Integración con Stackdriver: Usa Stackdriver para monitoreo y registro.
  • Query Insights: Analiza el rendimiento de las consultas y optimiza tu base de datos.

Escalado y Optimización del Rendimiento

  • Escalado Vertical: Aumenta el tipo de máquina o el tamaño de almacenamiento.
  • Escalado Horizontal: Usa réplicas de lectura para balanceo de carga.

Ejemplo: Escalando Almacenamiento

gcloud sql instances patch my-instance --storage-size=100

Funciones Avanzadas de Cloud SQL

Alta Disponibilidad y Replicación

  • Alta Disponibilidad (HA): Configura HA para conmutación por error automática.
  • Réplicas de Lectura: Crea réplicas de lectura para distribuir el tráfico de lectura.

Ejemplo: Creación de una Réplica de Lectura

gcloud sql instances create my-replica --master-instance-name=my-instance

Mejores Prácticas de Seguridad

  • Roles de IAM: Asigna roles de IAM apropiados para el control de acceso.
  • Emparejamiento de VPC: Usa el emparejamiento de VPC para conexiones seguras y privadas.
  • SSL/TLS: Habilita SSL/TLS para conexiones cifradas.

Ejemplo: Habilitando SSL/TLS

gcloud sql instances patch my-instance --require-ssl

Conclusión

Cloud SQL en GCP proporciona una solución robusta, escalable y segura para gestionar bases de datos relacionales. Al aprovechar sus servicios completamente gestionados, copias de seguridad automatizadas, alta disponibilidad y funciones avanzadas de seguridad, puedes centrarte en desarrollar tus aplicaciones sin preocuparte por la administración de bases de datos. Ya seas un principiante configurando tu primera instancia o un usuario avanzado optimizando el rendimiento y la seguridad, Cloud SQL ofrece las herramientas y la flexibilidad para satisfacer tus necesidades.
