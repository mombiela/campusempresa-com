Introducción a los Proyectos de GCP

Google Cloud Platform (GCP) organiza recursos en proyectos, los cuales proporcionan una manera de agrupar y gestionar recursos relacionados. Comprender los proyectos de GCP es fundamental para usar eficazmente los servicios de GCP.

Conceptos Clave

  • Proyecto: Un contenedor para recursos como máquinas virtuales, bases de datos y almacenamiento.
  • ID del Proyecto: Un identificador único para tu proyecto.
  • Nombre del Proyecto: Un nombre legible para humanos para tu proyecto.
  • Número del Proyecto: Un número único asignado por GCP.
  • Cuenta de Facturación: Vinculada a un proyecto para gestionar costos.
  • IAM (Gestión de Identidad y Acceso): Controla el acceso a los recursos dentro de un proyecto.

Creando un Proyecto de GCP

Para empezar a usar GCP, necesitas crear un proyecto. Esto se puede hacer a través de la Consola de GCP, la CLI de gcloud o la API.

Usando la Consola de GCP

  1. Navega a la Consola de GCP.
  2. Haz clic en el menú desplegable de proyectos en la parte superior de la página.
  3. Selecciona "Nuevo Proyecto."
  4. Ingresa el nombre del proyecto, el ID y selecciona una cuenta de facturación.
  5. Haz clic en "Crear."

Usando la CLI de gcloud

# Establecer un ID de proyecto predeterminado
PROJECT_ID="mi-proyecto-gcp"

# Crear un nuevo proyecto
gcloud projects create $PROJECT_ID

# Vincular el proyecto a una cuenta de facturación
BILLING_ACCOUNT=$(gcloud beta billing accounts list --format="value(name)" --limit=1)
gcloud beta billing projects link $PROJECT_ID --billing-account=$BILLING_ACCOUNT

Usando la API

from googleapiclient import discovery
from oauth2client.client import GoogleCredentials

credentials = GoogleCredentials.get_application_default()
service = discovery.build('cloudresourcemanager', 'v1', credentials=credentials)

project_body = {
    'name': 'Mi Proyecto GCP',
    'projectId': 'mi-proyecto-gcp'
}

request = service.projects().create(body=project_body)
response = request.execute()

print(response)

Gestionando Proyectos de GCP

Una vez que un proyecto está creado, puedes gestionarlo añadiendo recursos, estableciendo permisos y monitoreando el uso.

Añadiendo Recursos

Recursos como instancias de Compute Engine, buckets de Cloud Storage y conjuntos de datos de BigQuery pueden ser añadidos a tu proyecto.

Estableciendo Permisos

Usa IAM para controlar quién tiene acceso a tu proyecto y qué pueden hacer.

  • Roles: Definen qué acciones están permitidas.
  • Miembros: Usuarios, grupos o cuentas de servicio.
  • Políticas: Vinculan roles a miembros.

Ejemplo de añadir un rol a un miembro:

gcloud projects add-iam-policy-binding $PROJECT_ID \
    --member="user:ejemplo@gmail.com" \
    --role="roles/viewer"

Monitoreo y Facturación

Usa la Consola de GCP para monitorear el uso de recursos y gestionar la facturación.

  • Stackdriver Monitoring: Monitorea el rendimiento y el tiempo de actividad.
  • Informes de Facturación: Rastrea costos y establece presupuestos.

Mejores Prácticas

  • Organizar Proyectos: Usa proyectos para separar entornos (por ejemplo, desarrollo, pruebas, producción).
  • Usar Etiquetas: Etiqueta recursos con etiquetas para una mejor organización y seguimiento de costos.
  • Establecer Presupuestos y Alertas: Evita costos inesperados estableciendo presupuestos y alertas.

Conclusión

Comprender y gestionar proyectos de GCP es crucial para usar eficazmente Google Cloud Platform. Al organizar recursos en proyectos, establecer permisos apropiados y monitorear el uso, puedes asegurar operaciones en la nube eficientes y seguras.
