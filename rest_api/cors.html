Introducción a CORS

Cross-Origin Resource Sharing (CORS) es un mecanismo que utiliza cabeceras HTTP adicionales para permitir que un servidor indique cualquier origen (dominio, esquema o puerto) desde el cual un navegador debería permitir la carga de recursos. CORS es una especificación del W3C que permite solicitudes HTTP entre sitios web de diferentes orígenes.

¿Por qué es importante CORS?

  • Seguridad: CORS es crucial para la seguridad web, ya que previene ataques como Cross-Site Request Forgery (CSRF).
  • Interoperabilidad: Permite que las aplicaciones web realicen solicitudes a dominios diferentes al de origen, facilitando la integración de servicios y APIs de terceros.

Funcionamiento de CORS

Cabeceras HTTP de CORS

  • Access-Control-Allow-Origin: Especifica los orígenes permitidos para acceder al recurso.
  • Access-Control-Allow-Methods: Indica los métodos HTTP permitidos (GET, POST, PUT, DELETE, etc.).
  • Access-Control-Allow-Headers: Lista de cabeceras permitidas en la solicitud.
  • Access-Control-Allow-Credentials: Indica si las credenciales (cookies, cabeceras de autenticación) están permitidas.
  • Access-Control-Expose-Headers: Lista de cabeceras que pueden ser expuestas al cliente.
  • Access-Control-Max-Age: Tiempo en segundos que el resultado de una solicitud preflight puede ser almacenado en caché.

Solicitudes Preflight

Las solicitudes preflight son solicitudes de verificación que el navegador envía antes de la solicitud real para asegurarse de que el servidor permitirá la solicitud de origen cruzado.

Ejemplo de solicitud preflight:

OPTIONS /api/data HTTP/1.1
Host: api.example.com
Origin: http://example.com
Access-Control-Request-Method: POST
Access-Control-Request-Headers: X-Custom-Header

Ejemplo de respuesta preflight:

HTTP/1.1 200 OK
Access-Control-Allow-Origin: http://example.com
Access-Control-Allow-Methods: POST, GET, OPTIONS
Access-Control-Allow-Headers: X-Custom-Header
Access-Control-Max-Age: 86400

Implementación de CORS en una REST API

Ejemplo en Node.js con Express

Para habilitar CORS en una aplicación Node.js usando Express, puedes utilizar el middleware cors.

  1. Instalar el paquete cors:
npm install cors
  1. Configurar CORS en tu aplicación Express:
const express = require('express');
const cors = require('cors');
const app = express();

// Configuración básica de CORS
app.use(cors());

// Configuración avanzada de CORS
const corsOptions = {
  origin: 'http://example.com',
  methods: 'GET,POST,PUT,DELETE',
  allowedHeaders: 'Content-Type,Authorization',
  credentials: true,
  optionsSuccessStatus: 200 // Para navegadores antiguos
};

app.use(cors(corsOptions));

app.get('/api/data', (req, res) => {
  res.json({ message: 'CORS enabled!' });
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});

Ejemplo en Python con Flask

Para habilitar CORS en una aplicación Flask, puedes utilizar la extensión flask-cors.

  1. Instalar el paquete flask-cors:
pip install flask-cors
  1. Configurar CORS en tu aplicación Flask:
from flask import Flask, jsonify
from flask_cors import CORS

app = Flask(__name__)
CORS(app)

@app.route('/api/data')
def data():
    return jsonify(message='CORS enabled!')

if __name__ == '__main__':
    app.run(port=3000)

Conclusión

CORS es una parte esencial del desarrollo de aplicaciones web modernas que interactúan con APIs de diferentes orígenes. Comprender cómo funciona y cómo implementarlo correctamente es crucial para garantizar la seguridad y funcionalidad de tus aplicaciones. Asegúrate de configurar adecuadamente las cabeceras CORS en tu servidor para permitir solo los orígenes y métodos necesarios, minimizando así los riesgos de seguridad.
