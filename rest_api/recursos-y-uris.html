Recursos y URIs
Métodos HTTP ►

En este tema, exploraremos los conceptos fundamentales de recursos y URIs (Uniform Resource Identifiers) en el contexto de las APIs REST. Comprender estos conceptos es crucial para diseñar y consumir APIs de manera efectiva.

¿Qué es un Recurso?

En REST, un recurso es cualquier tipo de objeto, dato o servicio que puede ser accedido y manipulado a través de la API. Los recursos son las entidades principales que se exponen a través de la API y se identifican mediante URIs.

  • Ejemplos de recursos:
    • Usuarios
    • Productos
    • Pedidos
    • Comentarios

Cada recurso tiene una representación que puede ser en formato JSON, XML, HTML, etc.

¿Qué es un URI?

Un URI (Uniform Resource Identifier) es una cadena de caracteres que identifica de manera única un recurso en la web. En el contexto de REST, los URIs son utilizados para acceder a los recursos.

  • Componentes de un URI:
    • Esquema: Indica el protocolo a utilizar (e.g., http, https).
    • Host: El dominio donde reside el recurso (e.g., api.ejemplo.com).
    • Ruta: La ubicación específica del recurso (e.g., /usuarios/123).
    • Query Parameters: Parámetros adicionales para filtrar o modificar la solicitud (e.g., ?page=2&limit=10).

Ejemplo de URI

https://api.ejemplo.com/usuarios/123?include=detalles
  • Esquema: https
  • Host: api.ejemplo.com
  • Ruta: /usuarios/123
  • Query Parameters: include=detalles

Diseño de URIs

El diseño de URIs es fundamental para la claridad y la usabilidad de una API. Aquí hay algunas mejores prácticas:

  • Utilizar sustantivos en plural: Los URIs deben representar colecciones de recursos.
    • Correcto: /usuarios
    • Incorrecto: /usuario
  • Utilizar jerarquías claras: Organizar los recursos de manera jerárquica.
    • Correcto: /usuarios/123/pedidos
    • Incorrecto: /pedidos/usuario/123
  • Evitar verbos: Los URIs deben centrarse en los recursos, no en las acciones.
    • Correcto: /productos/456
    • Incorrecto: /obtenerProducto/456
  • Utilizar guiones en lugar de guiones bajos: Los guiones son más legibles en URIs.
    • Correcto: /productos-nuevos
    • Incorrecto: /productos_nuevos

Ejemplo de Código

A continuación, se muestra un ejemplo de cómo se puede consumir una API REST utilizando JavaScript y la función fetch para obtener un recurso específico.

// URL del recurso
const url = 'https://api.ejemplo.com/usuarios/123';

// Realizar una solicitud GET para obtener el recurso
fetch(url)
  .then(response => {
    if (!response.ok) {
      throw new Error('Error en la solicitud');
    }
    return response.json();
  })
  .then(data => {
    console.log('Datos del usuario:', data);
  })
  .catch(error => {
    console.error('Error:', error);
  });

Conclusión

Comprender los conceptos de recursos y URIs es esencial para trabajar con APIs REST. Los recursos son las entidades que se manipulan a través de la API, y los URIs son las direcciones que permiten acceder a estos recursos. Un buen diseño de URIs mejora la claridad y la usabilidad de la API, facilitando su consumo por parte de los desarrolladores.

En el próximo tema, profundizaremos en los métodos HTTP y cómo se utilizan para interactuar con los recursos en una API REST.
