Introducción

TypeScript es un superconjunto de JavaScript con tipado estático que se compila a JavaScript puro. Entender cómo compilar TypeScript es crucial para aprovechar sus potentes características de verificación de tipos y asegurar que tu código funcione en cualquier entorno JavaScript.

Conceptos Clave

  • Compilador de TypeScript (tsc): El compilador de TypeScript (tsc) es una herramienta de línea de comandos que compila archivos TypeScript (.ts) en archivos JavaScript (.js).
  • Archivo de Configuración (tsconfig.json): Un archivo de configuración que especifica los archivos raíz y las opciones del compilador necesarias para compilar el proyecto.

Instalando TypeScript

Antes de poder compilar TypeScript, necesitas instalarlo. Puedes instalar TypeScript globalmente usando npm (Node Package Manager).

npm install -g typescript

Verifica la instalación comprobando la versión:

tsc --version

Compilando un Archivo Único

Para compilar un solo archivo TypeScript, usa el comando tsc seguido del nombre del archivo.

tsc filename.ts

Este comando generará un archivo JavaScript correspondiente (filename.js) en el mismo directorio.

Ejemplo

Considera el siguiente archivo TypeScript example.ts:

let message: string = "Hello, TypeScript!";
console.log(message);

Compílalo usando:

tsc example.ts

Esto producirá example.js:

var message = "Hello, TypeScript!";
console.log(message);

Usando tsconfig.json

Para proyectos más grandes, gestionar las configuraciones de compilación a través del archivo tsconfig.json es más eficiente. Este archivo te permite especificar varias opciones del compilador y los archivos que se incluirán en la compilación.

Creando tsconfig.json

Puedes crear un archivo tsconfig.json usando el siguiente comando:

tsc --init

Este comando genera un archivo tsconfig.json predeterminado con varias opciones comentadas.

Ejemplo de tsconfig.json

Aquí tienes un ejemplo simple de un archivo tsconfig.json:

{
  "compilerOptions": {
    "target": "es5",
    "module": "commonjs",
    "outDir": "./dist",
    "strict": true
  },
  "include": ["src/**/*"]
}
  • target: Especifica la versión de JavaScript a la que se compilará.
  • module: Especifica el sistema de módulos a usar.
  • outDir: Especifica el directorio de salida para los archivos JavaScript compilados.
  • strict: Habilita todas las opciones de verificación de tipos estricta.
  • include: Especifica los archivos o directorios a incluir en la compilación.

Compilando con tsconfig.json

Una vez que tienes un archivo tsconfig.json, simplemente ejecuta:

tsc

Este comando compilará todos los archivos especificados en la sección include de tsconfig.json usando las opciones definidas en compilerOptions.

Opciones Comunes del Compilador

Aquí tienes algunas opciones del compilador comúnmente usadas:

| Opción | Descripción | |----------------|-----------------------------------------------------------------------------| | target | Especifica la versión objetivo de ECMAScript (e.g., es5, es6, es2015). | | module | Especifica el sistema de módulos (e.g., commonjs, amd, es6). | | outDir | Especifica el directorio de salida para los archivos compilados. | | rootDir | Especifica el directorio raíz de los archivos de entrada. | | strict | Habilita todas las opciones de verificación de tipos estricta. | | noImplicitAny| Genera un error en expresiones y declaraciones con un tipo any implícito. | | sourceMap | Genera archivos .map correspondientes para depuración. |

Conclusión

Compilar TypeScript es un proceso sencillo que puede gestionarse usando el compilador de TypeScript (tsc). Para proyectos más grandes, usar un archivo tsconfig.json simplifica la gestión de las opciones del compilador y los archivos incluidos. Entender estos conceptos básicos te ayudará a compilar y gestionar eficientemente tus proyectos TypeScript, asegurando un código JavaScript robusto y con tipado seguro.
