Introducción a las Pruebas Unitarias

Las pruebas unitarias son un método de prueba de software donde se prueban unidades o componentes individuales de un software. El propósito es validar que cada unidad del software funcione como se espera.

  • Unidad: La parte más pequeña testeable de cualquier software.
  • Prueba Unitaria: Un fragmento de código escrito por un desarrollador que ejecuta una pequeña parte del código que se está probando y verifica si los resultados son los esperados.

¿Por Qué Usar Jest para Pruebas Unitarias?

Jest es un marco de pruebas de JavaScript encantador con un enfoque en la simplicidad. Funciona con proyectos que usan Babel, TypeScript, Node.js, React, Angular, Vue.js, y más.

  • Configuración cero: Jest pretende funcionar sin configuración.
  • Instantáneas: Captura el estado de tu UI para asegurarte de que no cambie inesperadamente.
  • Aislado: Las pruebas se ejecutan en paralelo en sus propios procesos para maximizar el rendimiento.

Configuración de Jest con TypeScript

Para comenzar con Jest en un proyecto TypeScript, sigue estos pasos:

  1. Instala Jest y sus dependencias de TypeScript:

    npm install --save-dev jest ts-jest @types/jest

  2. Configura Jest: Crea un archivo jest.config.js en la raíz de tu proyecto:

    module.exports = {
  preset: 'ts-jest',
  testEnvironment: 'node',
};

  3. Agrega un script de prueba a package.json:

    "scripts": {
  "test": "jest"
}

Escribiendo Tu Primera Prueba

Vamos a escribir una función simple y una prueba correspondiente.

Función de Ejemplo

Crea un archivo llamado sum.ts:

export function sum(a: number, b: number): number {
  return a + b;
}

Prueba de Ejemplo

Crea un archivo llamado sum.test.ts:

import { sum } from './sum';

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Ejecutando la Prueba

Ejecuta el siguiente comando para ejecutar tus pruebas:

npm test

Técnicas Avanzadas de Pruebas

Mocking

El mocking es una técnica para reemplazar objetos complejos con versiones simplificadas que simulan el comportamiento de los objetos reales.

Ejemplo con Mocking

Supongamos que tienes una función que obtiene datos de una API:

import axios from 'axios';

export async function fetchData(url: string): Promise<any> {
  const response = await axios.get(url);
  return response.data;
}

Puedes hacer mock de axios en tu prueba:

import axios from 'axios';
import { fetchData } from './fetchData';

jest.mock('axios');
const mockedAxios = axios as jest.Mocked<typeof axios>;

test('fetches successfully data from an API', async () => {
  const data = { name: 'John Doe' };
  mockedAxios.get.mockResolvedValue({ data });

  const result = await fetchData('https://api.example.com/user');
  expect(result).toEqual(data);
});

Pruebas de Instantáneas

Las pruebas de instantáneas son una forma de probar la UI comparando el estado actual de la UI con un estado guardado previamente.

Ejemplo con Pruebas de Instantáneas

import renderer from 'react-test-renderer';
import MyComponent from './MyComponent';

test('renders correctly', () => {
  const tree = renderer.create(<MyComponent />).toJSON();
  expect(tree).toMatchSnapshot();
});

Conclusión

Las pruebas unitarias son una parte esencial del proceso de desarrollo de software, asegurando que los componentes individuales funcionen como se espera. Jest proporciona un marco robusto para escribir y ejecutar pruebas en proyectos TypeScript, ofreciendo características como configuración cero, mocking y pruebas de instantáneas. Al integrar Jest en tus proyectos TypeScript, puedes mejorar la calidad y mantenibilidad del código, haciendo tu proceso de desarrollo más eficiente y confiable.
