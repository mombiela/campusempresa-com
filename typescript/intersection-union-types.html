Tipos de Intersección y Unión en TypeScript
Guardias de Tipo ►

Introducción

En TypeScript, entender los tipos de intersección y unión es crucial para crear sistemas de tipos flexibles y robustos. Estos tipos te permiten combinar múltiples tipos en uno, lo que posibilita definiciones de tipos más precisas y un código más seguro.

Tipos de Unión

Los tipos de unión permiten que una variable sea uno de varios tipos. Esto es útil cuando un valor puede ser de más de un tipo.

Conceptos Clave

  • Definición: Un tipo de unión se define usando el símbolo | (barra vertical).
  • Uso: Los tipos de unión se usan cuando un valor puede ser uno de varios tipos.

Ejemplos

let value: string | number;

value = "Hola"; // válido
value = 42;     // válido
value = true;   // inválido, TypeScript lanzará un error

Explicación

En el ejemplo anterior, value puede ser un string o un number. Asignar un valor booleano a value resultará en un error de TypeScript porque boolean no es parte del tipo de unión.

Casos de Uso Prácticos

  • Parámetros de Función: Cuando una función puede aceptar múltiples tipos.
  • Respuestas de API: Cuando una API puede devolver diferentes tipos de datos.
function printId(id: string | number) {
    console.log(`ID: ${id}`);
}

printId("abc123"); // válido
printId(123);      // válido

Tipos de Intersección

Los tipos de intersección combinan múltiples tipos en uno. Esto significa que un valor debe satisfacer todos los tipos combinados.

Conceptos Clave

  • Definición: Un tipo de intersección se define usando el símbolo & (ampersand).
  • Uso: Los tipos de intersección se usan cuando un valor debe tener todas las propiedades de los tipos combinados.

Ejemplos

interface Person {
    name: string;
}

interface Employee {
    employeeId: number;
}

type EmployeePerson = Person & Employee;

const employee: EmployeePerson = {
    name: "John Doe",
    employeeId: 1234
};

Explicación

En el ejemplo anterior, EmployeePerson es un tipo de intersección que combina Person y Employee. El objeto employee debe tener tanto las propiedades name como employeeId.

Casos de Uso Prácticos

  • Combinación de Interfaces: Cuando un objeto necesita satisfacer múltiples interfaces.
  • Mejora de Tipos: Cuando se extienden tipos con propiedades adicionales.
interface Address {
    street: string;
    city: string;
}

type EmployeeWithAddress = Employee & Address;

const employeeWithAddress: EmployeeWithAddress = {
    employeeId: 1234,
    street: "123 Main St",
    city: "Anytown"
};

Comparación de Tipos de Unión e Intersección

| Característica | Tipos de Unión (|) | Tipos de Intersección (&) | |------------------------|---------------------------------|----------------------------------| | Definición | type AorB = A \| B | type AandB = A & B | | Uso | El valor puede ser uno de los tipos | El valor debe satisfacer todos los tipos | | Ejemplo | string \| number | Person & Employee | | Casos de Uso Prácticos | Parámetros de función, respuestas de API | Combinación de interfaces, mejora de tipos |

Conclusión

Entender y utilizar los tipos de intersección y unión en TypeScript puede mejorar enormemente la flexibilidad y seguridad de tu código. Los tipos de unión permiten definiciones de variables más versátiles, mientras que los tipos de intersección aseguran que los objetos cumplan con múltiples requisitos de tipo. Dominar estos conceptos te ayudará a escribir código TypeScript más robusto y mantenible.
