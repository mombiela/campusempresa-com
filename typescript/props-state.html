Introducción

En esta sección, exploraremos los conceptos de Props y Estado en TypeScript, particularmente en el contexto de React. Entender estos conceptos es crucial para construir aplicaciones web dinámicas e interactivas.

Props

Props (abreviatura de propiedades) son atributos de solo lectura que se pasan de un componente padre a un componente hijo. Se utilizan para pasar datos y controladores de eventos a lo largo del árbol de componentes.

Conceptos Clave

  • Inmutabilidad: Los props son inmutables, lo que significa que no pueden ser cambiados por el componente hijo.
  • Seguridad de Tipos: TypeScript te permite definir los tipos de los props, proporcionando una mejor seguridad de tipos y autocompletado.

Ejemplo

Vamos a crear un componente React simple que use props en TypeScript.

import React from 'react';

// Definir el tipo para los props
interface GreetingProps {
  name: string;
  age: number;
}

// Componente funcional usando props
const Greeting: React.FC<GreetingProps> = ({ name, age }) => {
  return (
    <div>
      <h1>Hola, {name}!</h1>
      <p>Tienes {age} años.</p>
    </div>
  );
};

// Uso
const App: React.FC = () => {
  return (
    <div>
      <Greeting name="Alice" age={30} />
    </div>
  );
};

export default App;

En este ejemplo:

  • Definimos una interfaz GreetingProps para especificar los tipos de name y age.
  • El componente Greeting usa estos props para renderizar un mensaje.
  • El componente App pasa los props a Greeting.

Estado

El estado es una forma de gestionar datos que pueden cambiar con el tiempo dentro de un componente. A diferencia de los props, el estado es mutable y puede ser actualizado por el propio componente.

Conceptos Clave

  • Mutabilidad: El estado puede ser actualizado usando la función setState.
  • Local al Componente: El estado es local al componente y no puede ser accedido por otros componentes a menos que se pase como props.

Ejemplo

Vamos a crear un componente React simple que use estado en TypeScript.

import React, { useState } from 'react';

// Componente funcional usando estado
const Counter: React.FC = () => {
  // Definir estado con valor inicial
  const [count, setCount] = useState<number>(0);

  return (
    <div>
      <h1>Cuenta: {count}</h1>
      <button onClick={() => setCount(count + 1)}>Incrementar</button>
      <button onClick={() => setCount(count - 1)}>Decrementar</button>
    </div>
  );
};

// Uso
const App: React.FC = () => {
  return (
    <div>
      <Counter />
    </div>
  );
};

export default App;

En este ejemplo:

  • Usamos el hook useState para crear una variable de estado count con un valor inicial de 0.
  • La función setCount se usa para actualizar el estado.
  • El componente renderiza botones para incrementar y decrementar la cuenta.

Comparación entre Props y Estado

| Característica | Props | Estado | |----------------|--------------------------------|--------------------------------| | Mutabilidad | Inmutable | Mutable | | Alcance | Pasado de padre a hijo | Local al componente | | Método de Actualización | No puede ser actualizado por el hijo | Actualizado usando setState | | Uso | Pasar datos y controladores de eventos | Gestionar datos dinámicos dentro del componente |

Conclusión

Entender los props y el estado es fundamental para construir aplicaciones React con TypeScript. Los props te permiten pasar datos y funciones a lo largo del árbol de componentes, mientras que el estado te permite gestionar datos dinámicos dentro de un componente. Aprovechando la seguridad de tipos de TypeScript, puedes crear código más robusto y mantenible. Practica usando props y estado en tus proyectos para solidificar estos conceptos.
