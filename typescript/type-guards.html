Guardias de Tipo en TypeScript
Alias de Tipo ►

TypeScript es un lenguaje poderoso que se basa en JavaScript añadiendo tipos estáticos. Una de las características clave que hacen que TypeScript sea robusto es su capacidad para realizar comprobaciones de tipos en tiempo de compilación. Los Guardias de Tipo son una forma de asegurar que tu código se comporte correctamente al reducir los tipos dentro de un bloque condicional.

¿Qué son los Guardias de Tipo?

Los Guardias de Tipo son expresiones que realizan comprobaciones en tiempo de ejecución para asegurar que una variable es de un tipo específico. Ayudan a TypeScript a entender el tipo de una variable dentro de un bloque, permitiendo una comprobación de tipos más precisa y reduciendo el riesgo de errores en tiempo de ejecución.

Conceptos Clave

  • Reducción de Tipo: El proceso de refinar el tipo de una variable dentro de un bloque condicional.
  • Predicados de Tipo: Funciones que devuelven un booleano y se utilizan para afirmar el tipo de una variable.
  • Operador typeof: Utilizado para comprobar el tipo de una variable.
  • Operador instanceof: Utilizado para comprobar si un objeto es una instancia de una clase específica.
  • Guardias de Tipo Definidos por el Usuario: Funciones personalizadas que ayudan en la comprobación de tipos.

Uso de `typeof` para Guardias de Tipo

El operador typeof se utiliza comúnmente para comprobar tipos primitivos como string, number, boolean, etc.

function printId(id: number | string) {
    if (typeof id === "string") {
        console.log(`ID es una cadena: ${id.toUpperCase()}`);
    } else {
        console.log(`ID es un número: ${id.toFixed(2)}`);
    }
}

En este ejemplo, typeof id === "string" reduce el tipo de id a string dentro del bloque, permitiendo que se utilicen métodos específicos de cadenas como toUpperCase().

Uso de `instanceof` para Guardias de Tipo

El operador instanceof se utiliza para comprobar si un objeto es una instancia de una clase específica.

class Dog {
    bark() {
        console.log("¡Guau!");
    }
}

class Cat {
    meow() {
        console.log("¡Miau!");
    }
}

function makeSound(animal: Dog | Cat) {
    if (animal instanceof Dog) {
        animal.bark();
    } else {
        animal.meow();
    }
}

Aquí, animal instanceof Dog reduce el tipo de animal a Dog, permitiendo que el método bark() se llame de manera segura.

Guardias de Tipo Definidos por el Usuario

Los guardias de tipo definidos por el usuario son funciones que devuelven un predicado de tipo. Son útiles para comprobaciones de tipos complejas.

interface Fish {
    swim: () => void;
}

interface Bird {
    fly: () => void;
}

function isFish(pet: Fish | Bird): pet is Fish {
    return (pet as Fish).swim !== undefined;
}

function move(pet: Fish | Bird) {
    if (isFish(pet)) {
        pet.swim();
    } else {
        pet.fly();
    }
}

En este ejemplo, isFish es un guardia de tipo definido por el usuario que comprueba si pet es un Fish. El tipo de retorno pet is Fish es un predicado de tipo que le dice a TypeScript el tipo de pet dentro del bloque if.

Conclusión

Los Guardias de Tipo son una característica poderosa en TypeScript que ayudan a asegurar que tu código sea seguro en cuanto a tipos y esté libre de errores en tiempo de ejecución. Al usar typeof, instanceof y guardias de tipo definidos por el usuario, puedes reducir los tipos dentro de bloques condicionales, haciendo tu código más robusto y fácil de mantener.

Entender y usar efectivamente los Guardias de Tipo mejorará significativamente tus habilidades de programación en TypeScript, permitiéndote escribir aplicaciones más seguras y confiables.
