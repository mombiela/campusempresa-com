Introducción

El archivo config.xml es un componente crucial en los proyectos de Apache Cordova. Sirve como el archivo de configuración central donde defines varias configuraciones y preferencias para tu aplicación. Este archivo te permite personalizar el comportamiento, la apariencia y los permisos de tu aplicación en diferentes plataformas.

Conceptos Clave

  • Preferencias Globales: Configuraciones que se aplican a todas las plataformas.
  • Preferencias Específicas de Plataforma: Configuraciones que se aplican a plataformas específicas como Android o iOS.
  • Plugins: Bibliotecas externas que añaden funcionalidad a tu aplicación.
  • Control de Acceso: Configuración de los recursos externos a los que tu aplicación puede acceder.

Estructura de config.xml

El archivo config.xml es un documento XML que sigue una estructura específica. Aquí hay un ejemplo básico:

<?xml version='1.0' encoding='utf-8'?>
<widget id="com.example.hello" version="1.0.0" xmlns="http://www.w3.org/ns/widgets" xmlns:cdv="http://cordova.apache.org/ns/1.0">
    <name>HelloWorld</name>
    <description>
        Una aplicación de muestra de Apache Cordova.
    </description>
    <author email="dev@example.com" href="http://example.com">
        Equipo de Apache Cordova
    </author>
    <content src="index.html" />
    <access origin="*" />
    <preference name="Fullscreen" value="true" />
    <platform name="android">
        <preference name="AndroidLaunchMode" value="singleTop" />
    </platform>
    <platform name="ios">
        <preference name="BackupWebStorage" value="none" />
    </platform>
</widget>

Preferencias Globales

Las preferencias globales son configuraciones que se aplican a todas las plataformas. Se definen directamente bajo el elemento <widget>.

Ejemplo

<preference name="Fullscreen" value="true" />
  • Fullscreen: Esta preferencia hace que la aplicación se ejecute en modo de pantalla completa.

Preferencias Globales Comunes

  • Fullscreen: true o false
  • Orientation: portrait o landscape
  • BackgroundColor: Código de color hexadecimal (por ejemplo, #FFFFFF para blanco)

Preferencias Específicas de Plataforma

Las preferencias específicas de plataforma son configuraciones que se aplican solo a una plataforma en particular. Se definen dentro de un elemento <platform>.

Ejemplo

<platform name="android">
    <preference name="AndroidLaunchMode" value="singleTop" />
</platform>
  • AndroidLaunchMode: Esta preferencia establece el modo de lanzamiento para Android.

Preferencias Específicas de Plataforma Comunes

  • Android:
    • AndroidLaunchMode: singleTop, singleTask, etc.
    • AndroidPersistentFileLocation: internal o compatibility
  • iOS:
    • BackupWebStorage: none, local, o cloud
    • DisallowOverscroll: true o false

Plugins

Los plugins son bibliotecas externas que añaden funcionalidad a tu aplicación Cordova. Se definen usando el elemento <plugin>.

Ejemplo

<plugin name="cordova-plugin-camera" spec="^4.1.0" />
  • cordova-plugin-camera: Este plugin permite que la aplicación acceda a la cámara del dispositivo.

Añadiendo Plugins

Para añadir un plugin, puedes usar la CLI de Cordova:

cordova plugin add cordova-plugin-camera

Control de Acceso

Las configuraciones de control de acceso definen a qué recursos externos puede acceder tu aplicación. Se definen usando el elemento <access>.

Ejemplo

<access origin="*" />
  • origin="*": Esto permite que la aplicación acceda a cualquier recurso externo.

Restringiendo el Acceso

Puedes restringir el acceso a dominios específicos:

<access origin="https://example.com" />

Conclusión

El archivo config.xml es esencial para configurar tu aplicación Apache Cordova. Te permite establecer preferencias globales y específicas de plataforma, añadir plugins y controlar el acceso a recursos externos. Entender cómo configurar este archivo de manera efectiva te permitirá personalizar el comportamiento y la apariencia de tu aplicación en diferentes plataformas.

Dominando el archivo config.xml, puedes asegurarte de que tu aplicación Cordova cumpla con tus requisitos específicos y proporcione una experiencia de usuario consistente en todas las plataformas soportadas.
