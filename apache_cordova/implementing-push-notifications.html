Implementación de Notificaciones Push
Optimizando el Rendimiento de la Aplicación ►

Las notificaciones push son una forma poderosa de mantener a los usuarios comprometidos con tu aplicación al entregar información oportuna y relevante directamente a sus dispositivos. Este tema te guiará a través del proceso de implementación de notificaciones push en una aplicación Apache Cordova, desde niveles principiantes hasta avanzados.

Introducción a las Notificaciones Push

Las notificaciones push permiten que las aplicaciones notifiquen a los usuarios sobre nuevos mensajes o eventos incluso cuando el usuario no está utilizando activamente la aplicación. Son una característica esencial para muchos tipos de aplicaciones, incluidas las de redes sociales, noticias y comercio electrónico.

Conceptos Clave

  • Servicio de Notificaciones Push: Un servicio que envía notificaciones a dispositivos. Ejemplos incluyen Firebase Cloud Messaging (FCM) para Android y Apple Push Notification Service (APNs) para iOS.
  • Token de Dispositivo: Un identificador único asignado a cada dispositivo por el servicio de notificaciones push, utilizado para enviar notificaciones a ese dispositivo específico.
  • Carga Útil: Los datos enviados en la notificación, que pueden incluir un mensaje, título y datos adicionales.

Configuración de tu Entorno

Antes de que puedas implementar notificaciones push, necesitas configurar tu entorno de desarrollo.

Prerrequisitos

  • Node.js y npm: Asegúrate de tener Node.js y npm instalados.
  • Apache Cordova: Instala Cordova globalmente usando npm. 
    npm install -g cordova
  • Proyecto Cordova: Crea un nuevo proyecto Cordova o navega a tu proyecto existente. 
    cordova create MyApp
cd MyApp

Añadiendo el Plugin de Push

Para manejar las notificaciones push en Cordova, necesitas añadir un plugin que proporcione la funcionalidad necesaria.

Instalando el Plugin

  • Usa el Cordova Push Plugin, que soporta tanto FCM como APNs. 
    cordova plugin add phonegap-plugin-push

Configuración de Notificaciones Push

Configuración para Android

  1. Configuración de Firebase: Crea un proyecto Firebase y añade tu aplicación Android a él.
  2. Descargar google-services.json: Coloca este archivo en el directorio platforms/android/app.
  3. Modificar config.xml: Añade las siguientes líneas a tu archivo config.xml. 
    <platform name="android">
    <resource-file src="google-services.json" target="app/google-services.json" />
</platform>

Configuración para iOS

  1. Cuenta de Desarrollador de Apple: Asegúrate de tener una cuenta de desarrollador de Apple.
  2. Configuración de APNs: Crea una clave APNs y descarga el archivo .p8.
  3. Modificar config.xml: Añade las siguientes líneas a tu archivo config.xml. 
    <platform name="ios">
    <config-file platform="ios" target="*-Info.plist" parent="UIApplicationExitsOnSuspend">
        <false/>
    </config-file>
</platform>

Implementación de Notificaciones Push

Inicializando el Plugin

  • Añade el siguiente código a tu index.js o archivo principal de JavaScript para inicializar el plugin de push. 
    document.addEventListener('deviceready', function () {
    var push = PushNotification.init({
        android: {
            senderID: "YOUR_SENDER_ID"
        },
        ios: {
            alert: "true",
            badge: "true",
            sound: "true"
        }
    });

    push.on('registration', function (data) {
        console.log("Dispositivo registrado con token:", data.registrationId);
        // Envía el ID de registro a tu servidor
    });

    push.on('notification', function (data) {
        console.log("Notificación recibida:", data.message);
        // Maneja la notificación
    });

    push.on('error', function (e) {
        console.error("Error de notificación push:", e.message);
    });
}, false);

Enviando Notificaciones

  • Usa tu servidor backend para enviar notificaciones. Aquí hay un ejemplo usando Node.js y FCM. 
    const admin = require('firebase-admin');
const serviceAccount = require('./path/to/serviceAccountKey.json');

admin.initializeApp({
    credential: admin.credential.cert(serviceAccount)
});

const registrationToken = 'DEVICE_REGISTRATION_TOKEN';
const message = {
    notification: {
        title: 'Hola',
        body: 'Mundo'
    },
    token: registrationToken
};

admin.messaging().send(message)
    .then((response) => {
        console.log('Mensaje enviado exitosamente:', response);
    })
    .catch((error) => {
        console.error('Error enviando mensaje:', error);
    });

Temas Avanzados

Manejo de Notificaciones en Segundo Plano

  • Modifica tu aplicación Cordova para manejar notificaciones cuando la aplicación está en segundo plano. 
    push.on('notification', function (data) {
    if (data.additionalData.foreground) {
        // La aplicación está en primer plano
        console.log("Notificación en primer plano:", data.message);
    } else {
        // La aplicación está en segundo plano
        console.log("Notificación en segundo plano:", data.message);
    }
});

Acciones Personalizadas en Notificaciones

  • Añade acciones personalizadas a tus notificaciones, como botones para respuestas rápidas. 
    const message = {
    notification: {
        title: 'Nuevo Mensaje',
        body: 'Tienes un nuevo mensaje'
    },
    android: {
        notification: {
            clickAction: 'OPEN_ACTIVITY_1'
        }
    },
    token: registrationToken
};

Conclusión

Implementar notificaciones push en una aplicación Apache Cordova implica configurar tu entorno, configurar los servicios necesarios y usar el Cordova Push Plugin para manejar las notificaciones. Siguiendo esta guía, puedes asegurarte de que tu aplicación pueda enviar y recibir notificaciones push de manera efectiva, manteniendo a tus usuarios comprometidos e informados.
