Introducción a los Closures

Los closures son bloques de funcionalidad autónomos que pueden ser pasados y utilizados en tu código. En Swift, los closures pueden capturar y almacenar referencias a variables y constantes del contexto en el que se definen. Esta característica se conoce como "cerrar sobre" esas constantes y variables, de ahí el nombre "closures."

Conceptos Clave

  • Expresiones de Closure: Closures sin nombre escritos en una sintaxis ligera.
  • Captura de Valores: Los closures pueden capturar y almacenar referencias a variables y constantes del contexto circundante.
  • Sintaxis de Closure de Trailing: Una conveniencia sintáctica para pasar un closure como el último argumento a una función.
  • Closures Escapando: Closures que se llaman después de que la función a la que fueron pasados retorna.
  • Autoclosures: Closures que se crean automáticamente para envolver una expresión pasada como argumento a una función.

Sintaxis Básica de los Closures

Los closures en Swift tienen una sintaxis limpia y concisa. Aquí hay un ejemplo básico de un closure:

let greetingClosure = {
    print("¡Hola, Mundo!")
}

greetingClosure()  // Salida: ¡Hola, Mundo!

Expresiones de Closure

Las expresiones de closure son una forma de escribir closures en línea de manera breve y clara. Aquí hay un ejemplo de una expresión de closure utilizada para ordenar un array:

let names = ["Chris", "Alex", "Ewa", "Barry", "Daniella"]

let sortedNames = names.sorted(by: { (s1: String, s2: String) -> Bool in
    return s1 < s2
})

print(sortedNames)  // Salida: ["Alex", "Barry", "Chris", "Daniella", "Ewa"]

Inferencia de Tipo Desde el Contexto

Swift puede inferir los tipos de parámetros y el tipo de retorno cuando el closure se usa como argumento de una función:

let sortedNames = names.sorted(by: { s1, s2 in return s1 < s2 })

Nombres de Argumentos Abreviados

Swift proporciona nombres de argumentos abreviados para hacer los closures más concisos:

let sortedNames = names.sorted(by: { $0 < $1 })

Sintaxis de Closure de Trailing

Si un closure es el último argumento de una función, puedes usar la sintaxis de trailing closure:

let sortedNames = names.sorted { $0 < $1 }

Captura de Valores

Los closures pueden capturar y almacenar referencias a variables y constantes del contexto circundante:

func makeIncrementer(incrementAmount: Int) -> () -> Int {
    var total = 0
    let incrementer: () -> Int = {
        total += incrementAmount
        return total
    }
    return incrementer
}

let incrementByTwo = makeIncrementer(incrementAmount: 2)
print(incrementByTwo())  // Salida: 2
print(incrementByTwo())  // Salida: 4

Closures Escapando

Un closure escapando es un closure que se llama después de que la función a la que fue pasado retorna. Declaras un closure escapando usando la palabra clave @escaping:

var completionHandlers: [() -> Void] = []

func someFunctionWithEscapingClosure(completionHandler: @escaping () -> Void) {
    completionHandlers.append(completionHandler)
}

func someFunction() {
    someFunctionWithEscapingClosure {
        print("Este es un closure escapando.")
    }
}

someFunction()
completionHandlers.first?()  // Salida: Este es un closure escapando.

Autoclosures

Un autoclosure es un closure que se crea automáticamente para envolver una expresión pasada como argumento a una función. No toma ningún argumento y retorna el valor de la expresión:

func serve(customer customerProvider: @autoclosure () -> String) {
    print("Ahora sirviendo a \(customerProvider())!")
}

serve(customer: "John Doe")  // Salida: Ahora sirviendo a John Doe!

Conclusión

Los closures son una característica poderosa en Swift que te permiten encapsular funcionalidad y pasarla en tu código. Pueden capturar y almacenar referencias a variables y constantes del contexto en el que se definen, haciéndolos increíblemente flexibles. Entender los closures es esencial para dominar Swift, ya que se usan extensamente en muchas APIs y patrones de Swift.
