Declaraciones de Transferencia de Control en Swift
Opcionales y Desempaquetado ►

Las declaraciones de transferencia de control son esenciales en la programación ya que te permiten cambiar el flujo de ejecución en tu código. En Swift, estas declaraciones te ayudan a gestionar bucles, condicionales y otras estructuras de control de manera efectiva. Esta sección cubrirá las diversas declaraciones de transferencia de control disponibles en Swift, desde el uso básico hasta el avanzado.

Declaración Break

La declaración break se usa para salir de un bucle o una declaración switch inmediatamente.

Uso en Bucles

for number in 1...10 {
    if number == 5 {
        break
    }
    print(number)
}
  • Explicación: Este bucle imprime números del 1 al 4. Cuando number es igual a 5, la declaración break sale del bucle.

Uso en Declaraciones Switch

let fruit = "Apple"

switch fruit {
case "Apple":
    print("Es una manzana.")
    break
case "Banana":
    print("Es un plátano.")
default:
    print("Fruta desconocida.")
}
  • Explicación: La declaración break aquí es redundante porque Swift sale automáticamente del caso del switch después de ejecutar el caso coincidente. Sin embargo, puede ser útil para la legibilidad o modificaciones futuras.

Declaración Continue

La declaración continue salta la iteración actual de un bucle y procede a la siguiente iteración.

Uso en Bucles

for number in 1...10 {
    if number % 2 == 0 {
        continue
    }
    print(number)
}
  • Explicación: Este bucle imprime solo números impares del 1 al 10. Cuando number es par, la declaración continue salta la iteración actual.

Declaración Fallthrough

La declaración fallthrough se usa en casos de switch para continuar la ejecución al siguiente caso.

Uso en Declaraciones Switch

let number = 3

switch number {
case 1:
    print("Uno")
case 2:
    print("Dos")
case 3:
    print("Tres")
    fallthrough
case 4:
    print("Cuatro")
default:
    print("Otro número")
}
  • Explicación: Cuando number es 3, el caso del switch imprime "Tres" y luego "Cuatro" debido a la declaración fallthrough.

Declaración Return

La declaración return sale de una función y opcionalmente devuelve un valor.

Uso en Funciones

func greet(name: String) -> String {
    return "¡Hola, \(name)!"
}

let greeting = greet(name: "Alice")
print(greeting)
  • Explicación: La declaración return sale de la función greet y devuelve la cadena de saludo.

Declaración Throw

La declaración throw se usa para lanzar un error en Swift.

Uso en Funciones

enum CustomError: Error {
    case invalidInput
}

func validate(input: Int) throws {
    if input < 0 {
        throw CustomError.invalidInput
    }
}

do {
    try validate(input: -1)
} catch {
    print("Se atrapó un error: \(error)")
}
  • Explicación: La declaración throw lanza un error CustomError.invalidInput si la entrada es negativa. El bloque do-catch maneja el error.

Declaración Guard

La declaración guard se usa para transferir el control fuera de un ámbito si no se cumple una condición.

Uso en Funciones

func process(input: Int?) {
    guard let validInput = input else {
        print("Entrada inválida")
        return
    }
    print("Procesando \(validInput)")
}

process(input: nil)
process(input: 10)
  • Explicación: La declaración guard verifica si input no es nulo. Si es nulo, imprime "Entrada inválida" y sale de la función. De lo contrario, continúa procesando la entrada válida.

Resumen

Las declaraciones de transferencia de control en Swift son herramientas poderosas para gestionar el flujo de tu código. Te permiten:

  • Salir de bucles y declaraciones switch (break).
  • Saltar iteraciones en bucles (continue).
  • Continuar la ejecución al siguiente caso del switch (fallthrough).
  • Salir de funciones y devolver valores (return).
  • Lanzar y manejar errores (throw).
  • Asegurarte de que se cumplan condiciones antes de proceder (guard).

Entender y usar estas declaraciones de manera efectiva te ayudará a escribir código Swift más robusto y legible.
