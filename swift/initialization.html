Inicialización en Swift
Herencia ►

La inicialización es un concepto fundamental en Swift que implica preparar una instancia de una clase, estructura o enumeración para su uso. Este tema cubrirá los conceptos básicos de la inicialización, incluidos los inicializadores predeterminados, los inicializadores personalizados y las técnicas avanzadas de inicialización.

Inicializadores Predeterminados

Cuando creas una nueva instancia de una clase o estructura, Swift proporciona automáticamente un inicializador predeterminado si no has definido ningún inicializador personalizado.

Ejemplo

struct Person {
    var name: String
    var age: Int
}

let person = Person(name: "Alice", age: 30)
print(person.name) // Output: Alice
print(person.age)  // Output: 30

En este ejemplo, la estructura Person tiene un inicializador predeterminado miembro a miembro porque no se define ningún inicializador personalizado.

Inicializadores Personalizados

Puedes definir inicializadores personalizados para configurar tus instancias con valores iniciales específicos o realizar cualquier configuración necesaria antes de que una instancia esté lista para su uso.

Ejemplo

struct Person {
    var name: String
    var age: Int

    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }
}

let person = Person(name: "Bob", age: 25)
print(person.name) // Output: Bob
print(person.age)  // Output: 25

En este ejemplo, se define un inicializador personalizado para inicializar las propiedades name y age.

Parámetros del Inicializador

Los inicializadores pueden tomar parámetros al igual que las funciones, lo que te permite pasar valores al crear una instancia.

Ejemplo

struct Rectangle {
    var width: Double
    var height: Double

    init(width: Double, height: Double) {
        self.width = width
        self.height = height
    }

    func area() -> Double {
        return width * height
    }
}

let rectangle = Rectangle(width: 10.0, height: 5.0)
print(rectangle.area()) // Output: 50.0

Aquí, la estructura Rectangle tiene un inicializador que toma width y height como parámetros.

Inicializadores Fallables

A veces, la inicialización puede fallar. Swift te permite definir inicializadores fallables que pueden devolver nil si la inicialización falla.

Ejemplo

struct Temperature {
    var celsius: Double

    init?(fahrenheit: Double) {
        guard fahrenheit >= -459.67 else {
            return nil
        }
        self.celsius = (fahrenheit - 32) * 5 / 9
    }
}

if let temp = Temperature(fahrenheit: -500) {
    print(temp.celsius)
} else {
    print("Invalid temperature")
}
// Output: Invalid temperature

En este ejemplo, el inicializador devuelve nil si el valor de Fahrenheit proporcionado está por debajo del cero absoluto.

Delegación de Inicializadores para Tipos de Valor

Para tipos de valor como estructuras y enumeraciones, puedes delegar la inicialización a otro inicializador dentro del mismo tipo.

Ejemplo

struct Color {
    var red: Double
    var green: Double
    var blue: Double

    init(white: Double) {
        self.red = white
        self.green = white
        self.blue = white
    }

    init(red: Double, green: Double, blue: Double) {
        self.red = red
        self.green = green
        self.blue = blue
    }
}

let gray = Color(white: 0.5)
print(gray.red, gray.green, gray.blue) // Output: 0.5 0.5 0.5

En este ejemplo, la estructura Color tiene dos inicializadores, y uno de ellos delega al otro.

Herencia y Sobrescritura de Inicializadores

Las clases pueden heredar inicializadores de su superclase y también pueden sobrescribirlos para proporcionar un comportamiento de inicialización personalizado.

Ejemplo

class Vehicle {
    var speed: Double

    init(speed: Double) {
        self.speed = speed
    }
}

class Car: Vehicle {
    var model: String

    init(speed: Double, model: String) {
        self.model = model
        super.init(speed: speed)
    }
}

let car = Car(speed: 120.0, model: "Sedan")
print(car.speed) // Output: 120.0
print(car.model) // Output: Sedan

En este ejemplo, la clase Car hereda de la clase Vehicle y sobrescribe el inicializador para incluir una propiedad adicional model.

Conclusión

La inicialización es una parte crucial de la creación de instancias en Swift. Al comprender los inicializadores predeterminados, los inicializadores personalizados, los inicializadores fallables, la delegación de inicializadores y la herencia de inicializadores, puedes gestionar eficazmente la configuración de tus instancias. Este conocimiento te ayudará a escribir código Swift más robusto y flexible.
