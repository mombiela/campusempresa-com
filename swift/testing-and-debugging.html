Introducción

Las pruebas y la depuración son habilidades cruciales para cualquier desarrollador de Swift. Esta sección te guiará a través de los conceptos básicos de las pruebas y la depuración en Swift, desde escribir tu primer caso de prueba hasta usar herramientas avanzadas de depuración.

Pruebas Unitarias con XCTest

¿Qué son las Pruebas Unitarias?

  • Las pruebas unitarias implican probar componentes individuales de tu código para asegurarte de que funcionen como se espera.
  • En Swift, el framework XCTest se usa comúnmente para pruebas unitarias.

Configuración de XCTest

  1. Crear un Target de Prueba:
    • Cuando creas un nuevo proyecto en Xcode, puedes agregar un target de prueba seleccionando la opción "Include Unit Tests".
  2. Importar XCTest:
    • Importa el framework XCTest en tus archivos de prueba.
    import XCTest

Escribir tu Primer Caso de Prueba

  • Un caso de prueba es una clase que hereda de XCTestCase.
  • Cada método de prueba debe comenzar con la palabra test.
import XCTest
@testable import YourProjectName

class MyFirstTests: XCTestCase {
    func testAddition() {
        let sum = 2 + 3
        XCTAssertEqual(sum, 5, "La suma debería ser 5")
    }
}

Ejecutar Pruebas

  • Puedes ejecutar pruebas presionando Cmd+U en Xcode o seleccionando el navegador de pruebas y haciendo clic en el botón de ejecución junto a tu caso de prueba.

Desarrollo Guiado por Pruebas (TDD)

  • TDD es un proceso de desarrollo de software donde escribes pruebas antes de escribir el código real.
  • El ciclo es: Escribir una prueba -> Ejecutar la prueba (debería fallar) -> Escribir el código -> Ejecutar la prueba (debería pasar) -> Refactorizar.

Ejemplo de TDD

  1. Escribir una Prueba: 
    func testMultiplication() {
    let product = multiply(2, 3)
    XCTAssertEqual(product, 6, "El producto debería ser 6")
}
  2. Ejecutar la Prueba (falla):
    • La función multiply aún no existe.
  3. Escribir el Código: 
    func multiply(_ a: Int, _ b: Int) -> Int {
    return a * b
}
  4. Ejecutar la Prueba (pasa):
    • Ahora la prueba debería pasar.
  5. Refactorizar:
    • Limpia el código si es necesario.

Depuración en Xcode

Puntos de Interrupción

  • Los puntos de interrupción te permiten pausar la ejecución de tu código en una línea específica.
  • Puedes agregar un punto de interrupción haciendo clic en el margen junto al número de línea en Xcode.

LLDB (Depurador de Bajo Nivel)

  • LLDB es un depurador poderoso integrado en Xcode.
  • Puedes usar comandos de LLDB en la consola de depuración para inspeccionar variables, controlar la ejecución y más.

Comandos Comunes de LLDB

  • po: Imprimir la descripción del objeto. 
    po myVariable
  • p: Imprimir el valor de una variable. 
    p myVariable
  • bt: Imprimir el backtrace del hilo actual. 
    bt

Consejos de Depuración

  • Usa Declaraciones Print: A veces, simples declaraciones print pueden ayudarte a entender qué está fallando. 
    print("Valor de myVariable: \\(myVariable)")
  • Verifica Nil: Asegúrate de que las variables opcionales no sean nil antes de usarlas. 
    if let value = myOptional {
    // Usa value
} else {
    print("myOptional es nil")
}

Técnicas Avanzadas de Pruebas

Mocking y Stubbing

  • Mocking: Crear objetos que simulan el comportamiento de objetos reales.
  • Stubbing: Proporcionar respuestas predefinidas a llamadas de métodos.

Ejemplo de Mocking

  • Usa protocolos para definir el comportamiento de tus objetos.
  • Crea clases mock que se ajusten a estos protocolos.
protocol NetworkService {
    func fetchData() -> String
}

class MockNetworkService: NetworkService {
    func fetchData() -> String {
        return "Datos Mock"
    }
}

Pruebas de Rendimiento

  • XCTest también te permite escribir pruebas de rendimiento para medir la eficiencia de tu código.
func testPerformanceExample() {
    self.measure {
        // Código para medir el tiempo
        let _ = (0..<1000).map { $0 * $0 }
    }
}

Conclusión

Las pruebas y la depuración son habilidades esenciales para cualquier desarrollador de Swift. Al dominar las pruebas unitarias con XCTest, adoptar prácticas de TDD y utilizar las poderosas herramientas de depuración de Xcode, puedes escribir código más confiable y mantenible. Recuerda, el objetivo no es solo encontrar y corregir errores, sino entender mejor tu código y asegurarte de que se comporte como se espera bajo diversas condiciones.
