Introducción a la Herencia

La herencia es un concepto fundamental en la programación orientada a objetos (OOP) que permite a una clase heredar propiedades y métodos de otra clase. Esto promueve la reutilización del código y establece una jerarquía natural entre las clases.

  • Clase Superclase (Clase Padre): La clase cuyas propiedades y métodos son heredados.
  • Clase Subclase (Clase Hija): La clase que hereda de la superclase.

Herencia Básica en Groovy

En Groovy, la herencia se implementa usando la palabra clave extends. Comencemos con un ejemplo simple para ilustrar la herencia básica.

Ejemplo: Herencia Básica

// Superclase
class Animal {
    String name

    void eat() {
        println("${name} está comiendo")
    }
}

// Subclase
class Dog extends Animal {
    void bark() {
        println("${name} está ladrando")
    }
}

// Creando una instancia de Dog
Dog dog = new Dog()
dog.name = "Buddy"
dog.eat()  // Método heredado
dog.bark() // Método de la subclase

Explicación

  • La clase Animal es la superclase con una propiedad name y un método eat().
  • La clase Dog es la subclase que extiende Animal y añade un nuevo método bark().
  • Una instancia de Dog puede acceder tanto al método eat() de Animal como al método bark() definido en Dog.

Sobrescribir Métodos

Una subclase puede sobrescribir métodos de su superclase para proporcionar una implementación específica.

Ejemplo: Sobrescribir Métodos

class Animal {
    String name

    void eat() {
        println("${name} está comiendo")
    }
}

class Dog extends Animal {
    @Override
    void eat() {
        println("${name} está comiendo comida para perros")
    }
}

Dog dog = new Dog()
dog.name = "Buddy"
dog.eat()  // Salidas: Buddy está comiendo comida para perros

Explicación

  • La clase Dog sobrescribe el método eat() de la clase Animal.
  • La anotación @Override se usa para indicar que el método está siendo sobrescrito.

Usando la Palabra Clave `super`

La palabra clave super se usa para llamar a los métodos y constructores de la superclase desde la subclase.

Ejemplo: Usando la Palabra Clave `super`

class Animal {
    String name

    Animal(String name) {
        this.name = name
    }

    void eat() {
        println("${name} está comiendo")
    }
}

class Dog extends Animal {
    Dog(String name) {
        super(name)  // Llamando al constructor de la superclase
    }

    @Override
    void eat() {
        super.eat()  // Llamando al método de la superclase
        println("${name} está comiendo comida para perros")
    }
}

Dog dog = new Dog("Buddy")
dog.eat()
// Salidas:
// Buddy está comiendo
// Buddy está comiendo comida para perros

Explicación

  • El constructor de Dog llama al constructor de Animal usando super(name).
  • El método eat() en Dog llama al método eat() de Animal usando super.eat() antes de añadir su propio comportamiento.

Clases Abstractas

Las clases abstractas no pueden ser instanciadas y están destinadas a ser subclasificadas. Pueden contener métodos abstractos que deben ser implementados por las subclases.

Ejemplo: Clases Abstractas

abstract class Animal {
    String name

    Animal(String name) {
        this.name = name
    }

    abstract void makeSound()  // Método abstracto
}

class Dog extends Animal {
    Dog(String name) {
        super(name)
    }

    @Override
    void makeSound() {
        println("${name} está ladrando")
    }
}

Dog dog = new Dog("Buddy")
dog.makeSound()  // Salidas: Buddy está ladrando

Explicación

  • La clase Animal es abstracta y contiene un método abstracto makeSound().
  • La clase Dog extiende Animal y proporciona una implementación para el método makeSound().

Resumen

La herencia en Groovy permite a las clases heredar propiedades y métodos de otras clases, promoviendo la reutilización del código y estableciendo una jerarquía. Los conceptos clave incluyen:

  • Herencia básica usando la palabra clave extends.
  • Sobrescribir métodos para proporcionar implementaciones específicas.
  • Usar la palabra clave super para llamar a los métodos y constructores de la superclase.
  • Clases abstractas que definen métodos abstractos para ser implementados por las subclases.

Al comprender y utilizar la herencia, puedes crear código más modular, mantenible y reutilizable en Groovy.
