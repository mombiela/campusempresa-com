Introducción

En esta sección del curso, nos enfocaremos en la práctica de conceptos de ciberseguridad a través de simulaciones y ejercicios. Estas actividades están diseñadas para reforzar el conocimiento teórico y proporcionar experiencia práctica en la identificación y mitigación de amenazas de seguridad.

Objetivos

  • Aplicar conceptos teóricos de ciberseguridad en situaciones prácticas.
  • Desarrollar habilidades para identificar y mitigar amenazas de seguridad.
  • Mejorar la capacidad de respuesta ante incidentes de seguridad.

Simulaciones de Seguridad

Simulación de un Ataque de Phishing

El phishing es una técnica común utilizada por los atacantes para obtener información confidencial. En esta simulación, los estudiantes aprenderán a identificar correos electrónicos de phishing y a tomar las medidas adecuadas para protegerse.

Pasos de la Simulación

  1. Recepción del Correo Electrónico: Los estudiantes recibirán un correo electrónico simulado que contiene un enlace sospechoso.
  2. Análisis del Correo: Se les pedirá que analicen el correo en busca de señales de phishing, como errores gramaticales, URLs sospechosas y remitentes desconocidos.
  3. Respuesta al Correo: Los estudiantes deberán decidir si reportan el correo, lo eliminan o toman alguna otra acción.

Ejemplo de Código para Análisis de URLs

import re

def is_suspicious_url(url):
    # Expresión regular para detectar URLs sospechosas
    pattern = re.compile(r'https?://(?!www\.)')
    return bool(pattern.search(url))

# Ejemplo de uso
url = "http://malicious-site.com"
print("URL sospechosa" if is_suspicious_url(url) else "URL segura")

Simulación de un Ataque de Fuerza Bruta

En esta simulación, los estudiantes aprenderán cómo los atacantes utilizan ataques de fuerza bruta para obtener acceso no autorizado a sistemas y cómo protegerse contra estos ataques.

Pasos de la Simulación

  1. Configuración del Entorno: Se proporcionará un entorno seguro donde los estudiantes puedan realizar ataques de fuerza bruta sin causar daño.
  2. Ejecución del Ataque: Los estudiantes utilizarán herramientas como Hydra o John the Ripper para intentar romper contraseñas.
  3. Implementación de Medidas de Seguridad: Después del ataque, los estudiantes implementarán medidas de seguridad como el bloqueo de cuentas después de múltiples intentos fallidos.

Ejemplo de Código para Bloqueo de Cuentas

class UserAccount:
    def __init__(self, username):
        self.username = username
        self.failed_attempts = 0
        self.is_locked = False

    def login(self, password):
        if self.is_locked:
            return "Cuenta bloqueada"
        if password == "correct_password":
            self.failed_attempts = 0
            return "Inicio de sesión exitoso"
        else:
            self.failed_attempts += 1
            if self.failed_attempts >= 3:
                self.is_locked = True
                return "Cuenta bloqueada"
            return "Contraseña incorrecta"

# Ejemplo de uso
user = UserAccount("usuario1")
print(user.login("wrong_password"))
print(user.login("wrong_password"))
print(user.login("wrong_password"))
print(user.login("correct_password"))

Ejercicios de Seguridad

Ejercicio de Análisis de Vulnerabilidades

Los estudiantes realizarán un análisis de vulnerabilidades en un sistema simulado utilizando herramientas como Nmap y OpenVAS.

Pasos del Ejercicio

  1. Escaneo de Puertos: Utilizar Nmap para identificar puertos abiertos y servicios en ejecución.
  2. Detección de Vulnerabilidades: Utilizar OpenVAS para detectar vulnerabilidades en los servicios identificados.
  3. Informe de Vulnerabilidades: Redactar un informe detallado de las vulnerabilidades encontradas y las posibles soluciones.

Ejemplo de Comando Nmap

nmap -sV -p 1-65535 target_ip

Ejercicio de Configuración de Firewall

Los estudiantes configurarán reglas de firewall para proteger un sistema de ataques externos.

Pasos del Ejercicio

  1. Identificación de Servicios Críticos: Determinar qué servicios deben estar accesibles desde el exterior.
  2. Configuración de Reglas: Configurar reglas de firewall para permitir solo el tráfico necesario.
  3. Prueba de Configuración: Probar la configuración del firewall para asegurarse de que solo el tráfico permitido puede pasar.

Ejemplo de Configuración de Firewall con iptables

# Permitir tráfico SSH
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Permitir tráfico HTTP
iptables -A INPUT -p tcp --dport 80 -j ACCEPT

# Bloquear todo el tráfico restante
iptables -A INPUT -j DROP

Conclusión

Las simulaciones y ejercicios de seguridad son esenciales para desarrollar habilidades prácticas en ciberseguridad. A través de estas actividades, los estudiantes pueden aplicar conceptos teóricos en situaciones reales, mejorando su capacidad para identificar y mitigar amenazas de seguridad. La práctica constante y la actualización de conocimientos son clave para mantenerse al día en el campo de la ciberseguridad.
