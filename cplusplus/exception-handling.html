Introducción al Manejo de Excepciones

El manejo de excepciones es un aspecto crucial del desarrollo de software robusto. Permite que un programa maneje situaciones inesperadas (excepciones) de manera elegante, en lugar de colapsar o producir resultados incorrectos.

Conceptos Clave

  • Excepción: Un evento que interrumpe el flujo normal del programa.
  • Bloque Try: Un bloque de código que se prueba para detectar excepciones.
  • Bloque Catch: Un bloque de código que maneja la excepción.
  • Sentencia Throw: Utilizada para señalar la ocurrencia de una excepción.

Manejo Básico de Excepciones

Bloques Try y Catch

En C++, las excepciones se manejan utilizando bloques try y catch. El bloque try contiene el código que podría lanzar una excepción, y el bloque catch contiene el código para manejar la excepción.

#include <iostream>
using namespace std;

int main() {
    try {
        int num1 = 10;
        int num2 = 0;
        if (num2 == 0) {
            throw "¡Error de división por cero!";
        }
        cout << num1 / num2 << endl;
    } catch (const char* e) {
        cout << "Excepción: " << e << endl;
    }
    return 0;
}

En este ejemplo:

  • El bloque try contiene código que podría lanzar una excepción.
  • La sentencia throw lanza una excepción si num2 es cero.
  • El bloque catch captura la excepción y la maneja imprimiendo un mensaje de error.

Lanzamiento de Excepciones

Puedes lanzar excepciones de cualquier tipo, incluyendo tipos integrados y tipos definidos por el usuario.

Lanzamiento de Tipos Integrados

#include <iostream>
using namespace std;

int main() {
    try {
        throw 404;
    } catch (int e) {
        cout << "Excepción: Código de error " << e << endl;
    }
    return 0;
}

Lanzamiento de Tipos Definidos por el Usuario

#include <iostream>
using namespace std;

class MyException {
public:
    const char* what() const {
        return "¡Ocurrió una excepción personalizada!";
    }
};

int main() {
    try {
        throw MyException();
    } catch (MyException& e) {
        cout << "Excepción: " << e.what() << endl;
    }
    return 0;
}

Múltiples Bloques Catch

Puedes tener múltiples bloques catch para manejar diferentes tipos de excepciones.

#include <iostream>
using namespace std;

int main() {
    try {
        throw 3.14;
    } catch (int e) {
        cout << "Capturó un entero: " << e << endl;
    } catch (double e) {
        cout << "Capturó un doble: " << e << endl;
    } catch (...) {
        cout << "Capturó una excepción desconocida." << endl;
    }
    return 0;
}

En este ejemplo:

  • Los bloques catch manejan diferentes tipos de excepciones.
  • El bloque catch(...) es un manejador de captura general para cualquier tipo de excepción no capturada por los manejadores anteriores.

Clases de Excepción Estándar

C++ proporciona un conjunto de clases de excepción estándar definidas en el encabezado <stdexcept>.

Excepciones Estándar Comunes

| Clase de Excepción | Descripción | |--------------------|-------------| | std::exception | Clase base para todas las excepciones estándar | | std::runtime_error | Representa errores en tiempo de ejecución | | std::logic_error | Representa errores lógicos | | std::out_of_range | Se lanza cuando ocurre un error fuera de rango | | std::invalid_argument | Se lanza cuando se pasa un argumento inválido |

Uso de Excepciones Estándar

#include <iostream>
#include <stdexcept>
using namespace std;

int main() {
    try {
        throw runtime_error("Ocurrió un error en tiempo de ejecución");
    } catch (runtime_error& e) {
        cout << "Excepción: " << e.what() << endl;
    }
    return 0;
}

Manejo Avanzado de Excepciones

Clases de Excepción Personalizadas

Puedes crear tus propias clases de excepción heredando de std::exception.

#include <iostream>
#include <exception>
using namespace std;

class MyException : public exception {
public:
    const char* what() const noexcept override {
        return "Mi excepción personalizada";
    }
};

int main() {
    try {
        throw MyException();
    } catch (MyException& e) {
        cout << "Excepción: " << e.what() << endl;
    }
    return 0;
}

Especificaciones de Excepción (Obsoleto)

Las especificaciones de excepción se utilizaban para especificar qué excepciones podría lanzar una función, pero están obsoletas en C++11 y eliminadas en C++17.

Especificador noexcept

El especificador noexcept indica que una función no lanza ninguna excepción.

#include <iostream>
using namespace std;

void safeFunction() noexcept {
    cout << "Esta función garantiza no lanzar excepciones." << endl;
}

int main() {
    safeFunction();
    return 0;
}

Conclusión

El manejo de excepciones en C++ es una característica poderosa que ayuda a crear programas robustos y resistentes a errores. Al usar try, catch y throw, puedes manejar errores de manera elegante. Comprender las clases de excepción estándar y crear excepciones personalizadas mejora aún más tu capacidad para manejar errores de manera efectiva. Recuerda usar el especificador noexcept para funciones que garantizan no lanzar excepciones, mejorando la fiabilidad y el rendimiento del código.
