Introducción a Cron

Cron es un programador de tareas basado en el tiempo en sistemas operativos tipo Unix. Los usuarios pueden programar trabajos (comandos o scripts) para que se ejecuten periódicamente en momentos, fechas o intervalos fijos. Es una herramienta poderosa para automatizar tareas repetitivas.

Entendiendo la Sintaxis de Cron

Los trabajos de Cron se definen en un archivo llamado crontab. Cada línea en un archivo crontab representa una tarea programada y sigue una sintaxis específica.

Sintaxis de Crontab

Una línea típica de crontab consta de cinco campos de tiempo y fecha seguidos del comando a ejecutar:

* * * * * comando_a_ejecutar

Los cinco campos representan:

  1. Minuto (0-59)
  2. Hora (0-23)
  3. Día del mes (1-31)
  4. Mes (1-12)
  5. Día de la semana (0-7, donde tanto 0 como 7 representan el domingo)

Ejemplo

Para ejecutar un script todos los días a las 2:30 AM:

30 2 * * * /ruta/al/script.sh

Gestión de Archivos Crontab

Ver Entradas de Crontab

Para ver tus entradas actuales de crontab, usa:

crontab -l

Editar Entradas de Crontab

Para editar tu archivo crontab, usa:

crontab -e

Esto abre el archivo crontab en el editor de texto predeterminado, permitiéndote agregar, modificar o eliminar trabajos cron.

Eliminar Entradas de Crontab

Para eliminar todas las entradas de crontab para el usuario actual, usa:

crontab -r

Cadenas Especiales en Cron

Cron también admite cadenas especiales para simplificar la programación:

  • @reboot – Ejecutar una vez, al iniciar.
  • @yearly o @annually – Ejecutar una vez al año, es decir, 0 0 1 1 *.
  • @monthly – Ejecutar una vez al mes, es decir, 0 0 1 * *.
  • @weekly – Ejecutar una vez a la semana, es decir, 0 0 * * 0.
  • @daily o @midnight – Ejecutar una vez al día, es decir, 0 0 * * *.
  • @hourly – Ejecutar una vez por hora, es decir, 0 * * * *.

Ejemplo

Para ejecutar un script en cada reinicio del sistema:

@reboot /ruta/al/script.sh

Uso Avanzado de Cron

Redirigir Salida

Los trabajos cron pueden producir salida, la cual puede ser redirigida a un archivo o enviada por correo electrónico.

Redirigir a un Archivo

Para registrar la salida de un trabajo cron en un archivo:

* * * * * /ruta/al/script.sh >> /ruta/al/archivo_de_registro 2>&1

Esto añade tanto la salida estándar como los errores estándar a archivo_de_registro.

Enviar Salida por Correo Electrónico

Por defecto, cron envía la salida al correo electrónico del usuario. Asegúrate de que la variable de entorno MAILTO esté configurada:

MAILTO="usuario@ejemplo.com"
* * * * * /ruta/al/script.sh

Uso de Variables de Entorno

Puedes configurar variables de entorno directamente en el archivo crontab:

SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin

Ejecutar Múltiples Comandos

Para ejecutar múltiples comandos en un solo trabajo cron, usa el operador &&:

* * * * * /ruta/al/primer_script.sh && /ruta/al/segundo_script.sh

Errores Comunes y Solución de Problemas

  • Problemas de Ruta: Asegúrate de especificar la ruta completa al comando o script.
  • Permisos: Verifica que el script tenga permisos de ejecución.
  • Diferencias de Entorno: Los trabajos cron se ejecutan en un entorno shell mínimo. Configura las variables de entorno necesarias dentro del script o crontab.

Ejemplo de un Script de Solución de Problemas

Para depurar un trabajo cron, puedes redirigir la salida a un archivo de registro:

* * * * * /ruta/al/script.sh >> /ruta/al/archivo_de_registro 2>&1

Revisa archivo_de_registro para errores o salida.

Conclusión

Cron es una herramienta esencial para automatizar tareas en Linux. Al entender su sintaxis, gestionar archivos crontab y utilizar funciones avanzadas, puedes programar y gestionar eficientemente tareas repetitivas. Siempre recuerda solucionar problemas comunes verificando rutas, permisos y variables de entorno.
