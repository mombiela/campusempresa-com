Introducción

Las tuberías y la redirección son conceptos fundamentales en Linux que permiten a los usuarios manipular la entrada y salida de comandos. Estas técnicas son esenciales para un uso eficiente de la línea de comandos y la creación de scripts.

Redirección

La redirección es el proceso de cambiar los dispositivos de entrada/salida estándar al ejecutar comandos.

Entrada, Salida y Error Estándar

  • Entrada Estándar (stdin): La fuente predeterminada de datos de entrada para un comando, típicamente el teclado.
  • Salida Estándar (stdout): El destino predeterminado para los datos de salida de un comando, típicamente la pantalla del terminal.
  • Error Estándar (stderr): El destino predeterminado para los mensajes de error, típicamente la pantalla del terminal.

Redirigiendo la Salida

  • Redirigir stdout a un archivo: Usa el operador > para redirigir la salida de un comando a un archivo, sobrescribiendo el archivo si existe. 
    ls > file_list.txt
  • Agregar stdout a un archivo: Usa el operador >> para agregar la salida al final de un archivo. 
    echo "New line" >> file_list.txt
  • Redirigir stderr a un archivo: Usa el operador 2> para redirigir los mensajes de error a un archivo. 
    ls non_existing_file 2> error_log.txt
  • Redirigir tanto stdout como stderr a un archivo: Usa el operador &> para redirigir tanto la salida estándar como los errores al mismo archivo. 
    ls existing_file non_existing_file &> combined_log.txt

Redirigiendo la Entrada

  • Redirigir stdin desde un archivo: Usa el operador < para tomar la entrada desde un archivo en lugar del teclado. 
    wc -l < file_list.txt

Tuberías

Las tuberías permiten que la salida de un comando se use como entrada para otro comando, habilitando la concatenación de comandos.

Tuberías Básicas

  • Operador de tubería (|): El operador de tubería toma la salida estándar de un comando y la pasa como entrada estándar a otro comando. 
    ls | grep "pattern"

Ejemplos de Tuberías

  • Combinando comandos: Usa tuberías para combinar múltiples comandos para operaciones más complejas.

    cat file_list.txt | sort | uniq
    • cat file_list.txt: Muestra el contenido de file_list.txt.
    • sort: Ordena la salida alfabéticamente.
    • uniq: Elimina las líneas duplicadas de la salida ordenada.

  • Filtrando y buscando: Usa tuberías con comandos como grep para filtrar y buscar datos.

    dmesg | grep "error"

Tuberías y Redirección Avanzadas

Combinando Tuberías y Redirección

  • Redirigiendo la salida de comandos encadenados: Combina tuberías con redirección para guardar la salida final en un archivo. 
    cat file_list.txt | sort | uniq > sorted_unique_list.txt

Usando Múltiples Tuberías

  • Encadenando múltiples comandos: Usa múltiples tuberías para crear secuencias de comandos complejas. 
    cat file_list.txt | grep "pattern" | sort | uniq | wc -l

Redirigiendo stderr en Comandos Encadenados

  • Separando stdout y stderr: Usa diferentes operadores de redirección para manejar stdout y stderr por separado en comandos encadenados. 
    ls existing_file non_existing_file 2> error_log.txt | grep "existing" > output_log.txt

Resumen

Las tuberías y la redirección son herramientas poderosas en Linux que permiten a los usuarios gestionar eficientemente la entrada y salida de comandos. Al dominar estas técnicas, los usuarios pueden crear secuencias de comandos complejas, automatizar tareas y manejar errores de manera efectiva. Entender cómo redirigir stdout, stderr y stdin, así como cómo usar tuberías para encadenar comandos, es esencial para cualquier usuario de Linux.
