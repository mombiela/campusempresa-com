Proyecto Final: Desarrollo de un Producto Completo

Introducción

En esta sección del curso, los estudiantes aplicarán todo lo aprendido para desarrollar un producto completo desde la concepción hasta la implementación. Este proyecto final permitirá a los estudiantes integrar conocimientos de diseño, desarrollo, pruebas y despliegue de software.

Objetivos del Proyecto

  • Integrar conocimientos adquiridos: Aplicar conceptos de diseño, desarrollo, pruebas y despliegue.
  • Desarrollar habilidades prácticas: Trabajar en un proyecto realista que simule un entorno profesional.
  • Fomentar el trabajo en equipo: Colaborar con compañeros para simular un entorno de desarrollo real.

Fases del Proyecto

1. Concepción y Planificación

  • Definición del problema: Identificar una necesidad o problema que el producto resolverá.
  • Análisis de requisitos: Documentar los requisitos funcionales y no funcionales.
  • Planificación del proyecto: Crear un plan de proyecto con hitos y entregables.

Ejemplo de Documento de Requisitos

# Requisitos Funcionales
1. El sistema debe permitir a los usuarios registrarse y autenticarse.
2. Los usuarios deben poder crear, leer, actualizar y eliminar publicaciones.

# Requisitos No Funcionales
1. El sistema debe ser escalable para soportar hasta 10,000 usuarios concurrentes.
2. El tiempo de respuesta del sistema debe ser inferior a 2 segundos.

2. Diseño

  • Arquitectura del sistema: Seleccionar la arquitectura adecuada (monolítica, microservicios, etc.).
  • Diseño de la base de datos: Crear el esquema de la base de datos.
  • Diseño de la interfaz de usuario: Crear wireframes y mockups.

Ejemplo de Esquema de Base de Datos

| Tabla Usuarios | Campo | Tipo | |----------------|-------------|------------| | | id | INT | | | nombre | VARCHAR(50)| | | email | VARCHAR(50)| | | contraseña | VARCHAR(50)|

3. Desarrollo

  • Implementación del backend: Desarrollar las APIs y lógica del servidor.
  • Implementación del frontend: Desarrollar la interfaz de usuario.
  • Integración: Asegurar que el frontend y backend funcionen juntos correctamente.

Ejemplo de Código Backend (Node.js)

const express = require('express');
const app = express();
const port = 3000;

app.use(express.json());

let users = [];

app.post('/register', (req, res) => {
    const { name, email, password } = req.body;
    users.push({ name, email, password });
    res.status(201).send('User registered');
});

app.listen(port, () => {
    console.log(`Server running at http://localhost:${port}`);
});

4. Pruebas

  • Pruebas unitarias: Escribir y ejecutar pruebas unitarias para el código.
  • Pruebas de integración: Asegurar que los diferentes módulos del sistema funcionen juntos.
  • Pruebas de aceptación: Validar que el sistema cumple con los requisitos del usuario.

Ejemplo de Prueba Unitaria (Jest)

const request = require('supertest');
const app = require('./app'); // Suponiendo que el servidor está en app.js

describe('POST /register', () => {
    it('should register a new user', async () => {
        const response = await request(app)
            .post('/register')
            .send({ name: 'John Doe', email: 'john@example.com', password: '123456' });
        expect(response.statusCode).toBe(201);
        expect(response.text).toBe('User registered');
    });
});

5. Despliegue

  • Configuración del entorno: Preparar el entorno de producción.
  • Despliegue del sistema: Implementar el sistema en un servidor o servicio de nube.
  • Monitoreo y mantenimiento: Configurar herramientas de monitoreo y planificar el mantenimiento.

Conclusión

El proyecto final es una oportunidad para que los estudiantes demuestren su capacidad para desarrollar un producto completo. Al finalizar este proyecto, los estudiantes habrán adquirido una experiencia valiosa que les preparará para enfrentar desafíos reales en el desarrollo de software.
