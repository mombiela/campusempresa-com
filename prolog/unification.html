Unificación en Prolog
Retroceso ►

La unificación es un concepto fundamental en Prolog, esencial para entender cómo el lenguaje procesa consultas y coincide patrones. Este tema te guiará desde los conceptos básicos hasta los aspectos avanzados de la unificación, proporcionando explicaciones claras, ejemplos y ejercicios.

¿Qué es la Unificación?

La unificación es el proceso de hacer que dos términos sean idénticos encontrando una sustitución adecuada para las variables. Es el mecanismo central que Prolog utiliza para hacer coincidir objetivos con hechos y reglas en la base de datos.

Conceptos Clave

  • Término: El bloque básico en Prolog, que puede ser un átomo, número, variable o término compuesto.
  • Variable: Un marcador de posición que puede unificarse con diferentes términos.
  • Sustitución: El proceso de reemplazar una variable con un término.
  • Coincidencia: El proceso de comparar dos términos para ver si pueden hacerse idénticos mediante sustitución.

Unificación Básica

En Prolog, la unificación se utiliza para determinar si dos términos pueden hacerse idénticos. Comencemos con algunos ejemplos simples.

Ejemplo 1: Unificando Átomos

?- apple = apple.
true.

En este ejemplo, los dos átomos apple y apple son idénticos, por lo que la unificación tiene éxito.

Ejemplo 2: Unificando Variables con Átomos

?- X = apple.
X = apple.

Aquí, la variable X se unifica con el átomo apple. La variable X ahora tiene el valor apple.

Ejemplo 3: Unificando Variables con Variables

?- X = Y.
X = Y.

En este caso, tanto X como Y son variables. Pueden unificarse entre sí, lo que significa que pueden tomar el mismo valor.

Unificación con Términos Compuestos

La unificación también funciona con términos compuestos, que son términos que tienen un functor y argumentos.

Ejemplo 4: Unificando Términos Compuestos

?- point(X, Y) = point(1, 2).
X = 1,
Y = 2.

Aquí, point(X, Y) se unifica con point(1, 2). Las variables X y Y se unifican con 1 y 2, respectivamente.

Ejemplo 5: Fallo de Unificación

?- point(X, Y) = point(1, 2, 3).
false.

En este ejemplo, la unificación falla porque los términos tienen estructuras diferentes. point(X, Y) tiene dos argumentos, mientras que point(1, 2, 3) tiene tres.

Unificación Avanzada

Exploremos algunos escenarios más complejos que involucran listas y estructuras anidadas.

Ejemplo 6: Unificando Listas

?- [H|T] = [1, 2, 3].
H = 1,
T = [2, 3].

En este ejemplo, la lista [H|T] se unifica con [1, 2, 3]. La cabeza H se unifica con 1, y la cola T se unifica con [2, 3].

Ejemplo 7: Estructuras Anidadas

?- person(name(First, Last), age(Age)) = person(name(john, doe), age(30)).
First = john,
Last = doe,
Age = 30.

Aquí, la estructura anidada person(name(First, Last), age(Age)) se unifica con person(name(john, doe), age(30)). Las variables First, Last y Age se unifican con john, doe y 30, respectivamente.

Ejercicios

  1. Unifica los siguientes términos y proporciona las sustituciones resultantes:

    • ?- X = Y, Y = 3.
    • ?- f(a, X) = f(Y, b).
    • ?- [A, B | T] = [1, 2, 3, 4].

  2. Determina si las siguientes unificaciones tienen éxito o fallan:

    • ?- g(X, Y) = g(a, b).
    • ?- h(X, X) = h(a, b).
    • ?- [H|T] = [1].

Conclusión

La unificación es un concepto poderoso y esencial en Prolog, que permite al lenguaje coincidir patrones y resolver consultas. Al entender y practicar la unificación, puedes aprovechar todo el potencial de las capacidades de programación lógica de Prolog. Recuerda experimentar con diferentes términos y estructuras para profundizar tu comprensión de cómo funciona la unificación en varios escenarios.
