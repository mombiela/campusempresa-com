Introducción a Prolog y Bases de Datos

Prolog es un lenguaje poderoso para trabajar con bases de datos debido a su soporte inherente para consultas lógicas y coincidencia de patrones. Esta sección introducirá los conceptos básicos de usar Prolog con bases de datos, incluyendo cómo representar y consultar datos.

Representación de Datos en Prolog

En Prolog, los datos se representan típicamente usando hechos y reglas. Aquí, cubriremos cómo representar datos relacionales usando hechos de Prolog.

  • Hechos: Afirmaciones básicas sobre el mundo.
  • Reglas: Declaraciones lógicas que definen relaciones entre hechos.

Ejemplo: Representación de una Base de Datos Simple

Considera una base de datos simple de empleados y departamentos.

% Hechos
employee(john, sales).
employee(mary, marketing).
employee(susan, sales).
employee(james, it).

department(sales, 101).
department(marketing, 102).
department(it, 103).

En este ejemplo:

  • employee(Name, Department) representa que un empleado trabaja en un departamento específico.
  • department(Name, Code) representa que un departamento tiene un código específico.

Consultas de Datos en Prolog

Las consultas de Prolog te permiten recuperar información de la base de datos haciendo coincidir patrones contra hechos y reglas.

Consultas Básicas

Para encontrar todos los empleados en el departamento de ventas, puedes usar la siguiente consulta:

?- employee(Name, sales).

Esta consulta devolverá:

Name = john ;
Name = susan.

Consultas Complejas

También puedes realizar consultas más complejas combinando múltiples hechos y reglas. Por ejemplo, para encontrar el código del departamento para los empleados en el departamento de ventas:

?- employee(Name, sales), department(sales, Code).

Esta consulta devolverá:

Name = john,
Code = 101 ;

Name = susan,
Code = 101.

Operaciones Avanzadas en Bases de Datos

A medida que avances, podrás realizar operaciones más sofisticadas como uniones, agregaciones y actualizaciones.

Uniones

Unir datos de múltiples relaciones es sencillo en Prolog. Por ejemplo, para encontrar los códigos de departamento para todos los empleados:

?- employee(Name, Department), department(Department, Code).

Agregaciones

Prolog también puede realizar agregaciones usando predicados incorporados como findall/3. Por ejemplo, para encontrar todos los empleados en una lista:

?- findall(Name, employee(Name, _), Employees).

Esta consulta devolverá:

Employees = [john, mary, susan, james].

Actualizaciones

Actualizar datos en Prolog implica retractar hechos antiguos y afirmar nuevos. Por ejemplo, para actualizar el departamento de John a IT:

?- retract(employee(john, sales)), assert(employee(john, it)).

Ejercicios Prácticos

Para solidificar tu comprensión, intenta los siguientes ejercicios:

  1. Ejercicio 1: Agrega nuevos empleados y departamentos a la base de datos y consúltalos.
  2. Ejercicio 2: Escribe una consulta para encontrar todos los empleados que trabajan en un departamento con un código específico.
  3. Ejercicio 3: Actualiza el departamento de un empleado y verifica el cambio con una consulta.

Conclusión

La base lógica de Prolog lo convierte en una excelente opción para trabajar con bases de datos. Al entender cómo representar, consultar y manipular datos, puedes aprovechar las fortalezas de Prolog para construir aplicaciones de bases de datos poderosas y flexibles. Practica con los ejercicios proporcionados para profundizar tu comprensión y prepararte para temas más avanzados.
