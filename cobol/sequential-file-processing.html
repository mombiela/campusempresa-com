Procesamiento de Archivos Secuenciales en COBOL
Procesamiento de Archivos Indexados ►

El procesamiento de archivos secuenciales es un concepto fundamental en la programación COBOL, esencial para manejar datos almacenados en archivos. Este tema te guiará desde los conceptos básicos hasta técnicas avanzadas, asegurando que entiendas cómo leer y escribir en archivos secuenciales de manera efectiva.

Introducción a los Archivos Secuenciales

Los archivos secuenciales son archivos donde los datos se almacenan en un orden específico y los registros se acceden secuencialmente, uno tras otro. Esta es la organización de archivos más común en COBOL.

Conceptos Clave

  • Control de Archivos: Especifica la organización del archivo y el modo de acceso.
  • Descripción del Archivo (FD): Describe la estructura de los registros del archivo.
  • Sentencia OPEN: Abre el archivo para lectura, escritura o ambos.
  • Sentencia READ: Lee registros del archivo.
  • Sentencia WRITE: Escribe registros en el archivo.
  • Sentencia CLOSE: Cierra el archivo después del procesamiento.

Control de Archivos y Descripción del Archivo

Control de Archivos

Las cláusulas SELECT y ASSIGN en el párrafo FILE-CONTROL especifican la organización del archivo y el nombre externo del archivo.

IDENTIFICATION DIVISION.
PROGRAM-ID. SeqFileExample.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
        ORGANIZATION IS SEQUENTIAL.
    SELECT OutputFile ASSIGN TO 'output.dat'
        ORGANIZATION IS SEQUENTIAL.

Descripción del Archivo

La entrada FD en la DATA DIVISION describe la estructura de los registros del archivo.

DATA DIVISION.
FILE SECTION.
FD  InputFile.
01  InputRecord.
    05  CustomerID    PIC X(10).
    05  CustomerName  PIC X(30).
    05  CustomerBalance PIC 9(7)V99.

FD  OutputFile.
01  OutputRecord.
    05  CustomerID    PIC X(10).
    05  CustomerName  PIC X(30).
    05  CustomerBalance PIC 9(7)V99.

Apertura y Cierre de Archivos

Sentencia OPEN

La sentencia OPEN se usa para abrir archivos para su procesamiento.

PROCEDURE DIVISION.
    OPEN INPUT InputFile.
    OPEN OUTPUT OutputFile.

Sentencia CLOSE

La sentencia CLOSE se usa para cerrar archivos después del procesamiento.

    CLOSE InputFile.
    CLOSE OutputFile.

Lectura y Escritura de Registros

Sentencia READ

La sentencia READ lee el siguiente registro del archivo.

    READ InputFile INTO InputRecord
        AT END
            MOVE 'YES' TO EndOfFile
    END-READ.

Sentencia WRITE

La sentencia WRITE escribe un registro en el archivo.

    MOVE InputRecord TO OutputRecord.
    WRITE OutputRecord.

Programa de Ejemplo

Aquí hay un ejemplo completo que lee registros de un archivo de entrada y los escribe en un archivo de salida.

IDENTIFICATION DIVISION.
PROGRAM-ID. SeqFileExample.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
        ORGANIZATION IS SEQUENTIAL.
    SELECT OutputFile ASSIGN TO 'output.dat'
        ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD  InputFile.
01  InputRecord.
    05  CustomerID    PIC X(10).
    05  CustomerName  PIC X(30).
    05  CustomerBalance PIC 9(7)V99.

FD  OutputFile.
01  OutputRecord.
    05  CustomerID    PIC X(10).
    05  CustomerName  PIC X(30).
    05  CustomerBalance PIC 9(7)V99.

WORKING-STORAGE SECTION.
01  EndOfFile        PIC X VALUE 'NO'.

PROCEDURE DIVISION.
    OPEN INPUT InputFile.
    OPEN OUTPUT OutputFile.

    PERFORM UNTIL EndOfFile = 'YES'
        READ InputFile INTO InputRecord
            AT END
                MOVE 'YES' TO EndOfFile
        END-READ

        IF EndOfFile = 'NO'
            MOVE InputRecord TO OutputRecord
            WRITE OutputRecord
        END-IF
    END-PERFORM.

    CLOSE InputFile.
    CLOSE OutputFile.

    STOP RUN.

Técnicas Avanzadas

Manejo del Estado del Archivo

Usar la cláusula FILE STATUS para manejar errores durante las operaciones de archivo.

FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
        ORGANIZATION IS SEQUENTIAL
        FILE STATUS IS WS-FileStatus.

WORKING-STORAGE SECTION.
01  WS-FileStatus    PIC XX.

Verificación del Estado del Archivo

Verificar el estado del archivo después de cada operación para asegurar una ejecución exitosa.

    READ InputFile INTO InputRecord
        AT END
            MOVE 'YES' TO EndOfFile
    END-READ.

    IF WS-FileStatus NOT = '00'
        DISPLAY 'Error reading file: ' WS-FileStatus
        STOP RUN
    END-IF.

Conclusión

El procesamiento de archivos secuenciales en COBOL implica entender cómo definir, abrir, leer, escribir y cerrar archivos. Al dominar estas técnicas básicas y avanzadas, puedes manejar eficientemente los datos almacenados en archivos secuenciales, un requisito común en muchas aplicaciones COBOL. Practica con diferentes estructuras de archivos y mecanismos de manejo de errores para volverte competente en el procesamiento de archivos secuenciales.
