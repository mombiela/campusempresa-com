Integración y Entrega Continua
Mantenimiento y Actualización de tu Aplicación ►

La Integración Continua (CI) y la Entrega Continua (CD) son prácticas esenciales en el desarrollo de software moderno. Ayudan a garantizar que tu código esté siempre en un estado desplegable, reducen los problemas de integración y permiten una entrega más rápida de características y correcciones. En este tema, exploraremos cómo implementar CI/CD para una aplicación Ionic.

Introducción a la Integración y Entrega Continua

  • Integración Continua (CI): La práctica de integrar automáticamente los cambios de código de múltiples contribuyentes en un repositorio compartido varias veces al día.
  • Entrega Continua (CD): La práctica de desplegar automáticamente cada cambio de código en un entorno de staging o producción después de pasar por el pipeline de CI.

Configuración de la Integración Continua para Ionic

Elegir una Herramienta de CI

Hay varias herramientas de CI disponibles, cada una con sus propias fortalezas. Algunas opciones populares incluyen:

  • GitHub Actions
  • Travis CI
  • CircleCI
  • Jenkins

Para este curso, utilizaremos GitHub Actions debido a su integración perfecta con los repositorios de GitHub.

Configuración de GitHub Actions

  1. Crear un Repositorio en GitHub: Si aún no lo has hecho, crea un repositorio para tu proyecto Ionic en GitHub.
  2. Agregar un Archivo de Workflow: En tu repositorio, crea un directorio llamado .github/workflows y agrega un archivo YAML (por ejemplo, ci.yml).
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Build the app
      run: npm run build

    - name: Run tests
      run: npm test

Explicación del Archivo de Workflow

  • name: El nombre del workflow.
  • on: Especifica los eventos que desencadenan el workflow (por ejemplo, push, pull_request).
  • jobs: Define los trabajos a ejecutar en el workflow.
    • runs-on: Especifica el sistema operativo para el trabajo.
    • steps: Enumera los pasos a ejecutar en el trabajo.
      • Checkout repository: Utiliza la acción actions/checkout para clonar el repositorio.
      • Set up Node.js: Utiliza la acción actions/setup-node para configurar el entorno de Node.js.
      • Install dependencies: Ejecuta npm install para instalar las dependencias del proyecto.
      • Build the app: Ejecuta npm run build para construir la aplicación Ionic.
      • Run tests: Ejecuta npm test para ejecutar la suite de pruebas.

Configuración de la Entrega Continua para Ionic

Desplegar en Firebase Hosting

Firebase Hosting es una opción popular para desplegar aplicaciones web. Para desplegar tu aplicación Ionic en Firebase Hosting, sigue estos pasos:

  1. Instalar Firebase CLI: Si aún no lo has hecho, instala Firebase CLI.
npm install -g firebase-tools
  1. Inicializar Firebase en Tu Proyecto: Ejecuta el siguiente comando y sigue las indicaciones para configurar Firebase Hosting.
firebase init
  1. Agregar Pasos de Despliegue a GitHub Actions: Actualiza tu archivo de workflow para incluir pasos para desplegar en Firebase Hosting.
name: CI/CD

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout repository
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Build the app
      run: npm run build

    - name: Run tests
      run: npm test

    - name: Deploy to Firebase Hosting
      env:
        FIREBASE_TOKEN: ${{ secrets.FIREBASE_TOKEN }}
      run: |
        npm install -g firebase-tools
        firebase deploy --only hosting

Explicación de los Pasos de Despliegue

  • Deploy to Firebase Hosting: Este paso instala Firebase CLI y despliega la aplicación en Firebase Hosting.
    • env: Establece la variable de entorno FIREBASE_TOKEN, que es necesaria para la autenticación. Necesitas crear un token de Firebase y agregarlo a los secretos de tu repositorio en GitHub.

Conclusión

En este tema, cubrimos los conceptos básicos de la Integración Continua y la Entrega Continua para una aplicación Ionic. Configuramos un pipeline de CI utilizando GitHub Actions y lo configuramos para construir, probar y desplegar nuestra aplicación en Firebase Hosting. Al implementar CI/CD, puedes asegurarte de que tu código esté siempre en un estado desplegable y entregar nuevas características y correcciones de manera más rápida y confiable.
