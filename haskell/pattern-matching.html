Coincidencia de Patrones en Haskell
Inmutabilidad ►

La coincidencia de patrones es una característica poderosa en Haskell que te permite verificar un valor contra un patrón. Es un concepto fundamental que se utiliza extensamente en la programación en Haskell para desestructurar estructuras de datos, controlar el flujo y más.

Introducción a la Coincidencia de Patrones

La coincidencia de patrones en Haskell se usa para:

  • Desestructurar estructuras de datos.
  • Simplificar el código al manejar diferentes casos.
  • Mejorar la legibilidad y mantenibilidad.

Coincidencia de Patrones Básica

En su forma más simple, la coincidencia de patrones puede usarse en definiciones de funciones para coincidir con valores específicos.

-- Función para verificar si un número es cero
isZero :: Int -> Bool
isZero 0 = True
isZero _ = False

En este ejemplo:

  • isZero 0 = True coincide con la entrada 0 y devuelve True.
  • isZero _ = False usa el patrón comodín _ para coincidir con cualquier otro valor y devuelve False.

Coincidencia de Patrones con Listas

Las listas son una estructura de datos común en Haskell, y la coincidencia de patrones puede usarse para desestructurarlas.

-- Función para obtener la cabeza de una lista
head' :: [a] -> a
head' [] = error "Lista vacía"
head' (x:_) = x

En este ejemplo:

  • head' [] = error "Lista vacía" coincide con una lista vacía y lanza un error.
  • head' (x:_) = x coincide con una lista no vacía, vincula la cabeza de la lista a x y la devuelve.

Coincidencia de Patrones con Tuplas

Las tuplas también pueden desestructurarse usando la coincidencia de patrones.

-- Función para sumar dos pares de enteros
addPairs :: (Int, Int) -> (Int, Int) -> (Int, Int)
addPairs (a, b) (c, d) = (a + c, b + d)

En este ejemplo:

  • addPairs (a, b) (c, d) = (a + c, b + d) coincide con dos pares de enteros y devuelve un nuevo par con sus sumas.

Coincidencia de Patrones Avanzada

Guardas

Las guardas proporcionan una forma de agregar condiciones más complejas a la coincidencia de patrones.

-- Función para clasificar un número
classifyNumber :: Int -> String
classifyNumber n
  | n < 0     = "Negativo"
  | n == 0    = "Cero"
  | otherwise = "Positivo"

En este ejemplo:

  • Las guardas (|) se usan para verificar múltiples condiciones.
  • otherwise es una guarda de captura que coincide con cualquier valor no capturado por las guardas anteriores.

Patrones As

Los patrones as te permiten mantener una referencia al valor completo mientras coincides con sus componentes.

-- Función para duplicar el primer elemento de una lista
duplicateHead :: [a] -> [a]
duplicateHead xs@(x:_) = x : xs
duplicateHead [] = []

En este ejemplo:

  • xs@(x:_) coincide con una lista no vacía, vincula la lista completa a xs y la cabeza a x.
  • La función devuelve una nueva lista con la cabeza duplicada.

Coincidencia de Patrones en Expresiones Case

Las expresiones case proporcionan una forma de usar la coincidencia de patrones dentro de expresiones.

-- Función para describir una lista
describeList :: [a] -> String
describeList xs = case xs of
  []      -> "Lista vacía"
  [x]     -> "Lista de un solo elemento"
  (x:xs)  -> "Lista más larga"

En este ejemplo:

  • case xs of se usa para coincidir la lista xs contra diferentes patrones.

Resumen

La coincidencia de patrones es una característica versátil y poderosa en Haskell que simplifica el código y mejora la legibilidad. Al dominar la coincidencia de patrones, puedes escribir programas en Haskell más concisos y expresivos. Los conceptos clave incluyen:

  • Coincidencia de patrones básica en definiciones de funciones.
  • Desestructuración de listas y tuplas.
  • Uso de guardas para condiciones complejas.
  • Patrones as para referenciar valores completos.
  • Expresiones case para coincidencia de patrones en línea.

Entender y usar efectivamente la coincidencia de patrones mejorará significativamente tus habilidades de programación en Haskell.
