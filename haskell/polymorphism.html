Polimorfismo en Haskell
Estructuras de Datos Recursivas ►

El polimorfismo es un concepto central en Haskell que permite a las funciones operar sobre diferentes tipos. Mejora la reutilización y flexibilidad del código. Este tema te guiará a través de la comprensión del polimorfismo en Haskell, desde niveles básicos hasta avanzados.

Introducción al Polimorfismo

El polimorfismo en Haskell se puede categorizar ampliamente en dos tipos:

  • Polimorfismo Paramétrico: Funciones o tipos de datos que pueden operar sobre cualquier tipo.
  • Polimorfismo Ad-hoc: Funciones que pueden operar sobre diferentes tipos pero se comportan de manera diferente según el tipo.

Polimorfismo Paramétrico

El polimorfismo paramétrico te permite escribir funciones genéricas que pueden manejar valores uniformemente sin depender de su tipo.

Ejemplo: Una Función de Identidad Genérica

identity :: a -> a
identity x = x
  • identity es una función que toma un valor de cualquier tipo a y devuelve un valor del mismo tipo.
  • La variable de tipo a puede representar cualquier tipo, haciendo que identity sea una función polimórfica.

Ejemplo: Una Función de Par Genérica

makePair :: a -> b -> (a, b)
makePair x y = (x, y)
  • makePair toma dos valores de tipos potencialmente diferentes a y b y devuelve una tupla que contiene ambos valores.
  • Esta función es polimórfica en ambos de sus argumentos.

Polimorfismo Ad-hoc

El polimorfismo ad-hoc se logra a través de clases de tipos, permitiendo que las funciones operen de manera diferente según el tipo de sus argumentos.

Ejemplo: La Clase de Tipo `Eq`

La clase de tipo Eq se usa para tipos que soportan pruebas de igualdad.

class Eq a where
    (==) :: a -> a -> Bool
    (/=) :: a -> a -> Bool
  • Los tipos que son instancias de Eq deben implementar las funciones (==) y (/=).

Ejemplo: Haciendo un Tipo una Instancia de `Eq`

data Color = Red | Green | Blue

instance Eq Color where
    Red == Red = True
    Green == Green = True
    Blue == Blue = True
    _ == _ = False
  • Aquí, Color se convierte en una instancia de Eq, definiendo cómo se determina la igualdad para los valores de Color.

Ejemplo: La Clase de Tipo `Show`

La clase de tipo Show se usa para tipos cuyos valores pueden ser convertidos a cadenas.

class Show a where
    show :: a -> String
  • Los tipos que son instancias de Show deben implementar la función show.

Ejemplo: Haciendo un Tipo una Instancia de `Show`

data Color = Red | Green | Blue

instance Show Color where
    show Red = "Red"
    show Green = "Green"
    show Blue = "Blue"
  • Aquí, Color se convierte en una instancia de Show, definiendo cómo los valores de Color se convierten a cadenas.

Polimorfismo Avanzado

Tipos de Clase Superior

Los tipos de clase superior te permiten abstraer sobre constructores de tipos. Esta es una forma más avanzada de polimorfismo.

Ejemplo: Clase de Tipo Functor

La clase de tipo Functor es para tipos que pueden ser mapeados.

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • f es un constructor de tipo que toma un argumento de tipo.
  • fmap aplica una función a cada elemento dentro de un functor.

Ejemplo: Haciendo un Tipo una Instancia de `Functor`

instance Functor Maybe where
    fmap _ Nothing = Nothing
    fmap f (Just x) = Just (f x)
  • Aquí, Maybe se convierte en una instancia de Functor, definiendo cómo funciona fmap para Maybe.

Conclusión

El polimorfismo en Haskell te permite escribir código flexible y reutilizable. Al entender el polimorfismo paramétrico y el polimorfismo ad-hoc, puedes crear funciones y tipos que funcionan en una amplia gama de escenarios. A medida que avances, los tipos de clase superior y las clases de tipos como Functor te permitirán abstraer patrones aún más complejos, haciendo que tu código Haskell sea tanto poderoso como elegante.
