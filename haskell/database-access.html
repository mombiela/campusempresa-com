Introducción al Acceso a Bases de Datos

El acceso a bases de datos es un aspecto crucial de muchas aplicaciones, permitiendo el almacenamiento, recuperación y manipulación de datos. En Haskell, el acceso a bases de datos se puede lograr utilizando varias bibliotecas y frameworks. Esta sección te guiará a través de los conceptos básicos del acceso a bases de datos en Haskell, desde la configuración de una conexión hasta la realización de operaciones CRUD (Crear, Leer, Actualizar, Eliminar).

Configuración del Entorno

Antes de sumergirnos en el código, configuremos el entorno necesario.

  1. Instalar GHC y Cabal: Asegúrate de tener instalado el Glasgow Haskell Compiler (GHC) y Cabal.
  2. Crear un Nuevo Proyecto: Usa Cabal para crear un nuevo proyecto en Haskell. 
    cabal init
  3. Agregar Dependencias: Agrega las bibliotecas necesarias a tu archivo cabal. Para el acceso a bases de datos, usaremos persistent y persistent-sqlite para SQLite como ejemplo. 
    build-depends: base >=4.7 && <5,
               persistent,
               persistent-sqlite,
               persistent-template,
               text

Conectándose a una Base de Datos

Para conectarse a una base de datos, necesitas definir tu esquema de base de datos y configurar la conexión.

Definiendo el Esquema

Define tu esquema de base de datos usando la sintaxis de Template Haskell de Persistent.

{-# LANGUAGE GADTs #-}
{-# LANGUAGE OverloadedStrings #-}
{-# LANGUAGE QuasiQuotes #-}
{-# LANGUAGE TemplateHaskell #-}
{-# LANGUAGE TypeFamilies #-}

import Database.Persist
import Database.Persist.Sqlite
import Database.Persist.TH
import Control.Monad.IO.Class (liftIO)

share [mkPersist sqlSettings, mkMigrate "migrateAll"] [persistLowerCase|
Person
    name String
    age Int
    deriving Show
|]

Configurando la Conexión

Configura la conexión a la base de datos SQLite y ejecuta la migración para crear las tablas.

main :: IO ()
main = runSqlite "test.db" $ do
    runMigration migrateAll
    liftIO $ putStrLn "Base de datos conectada y esquema migrado."

Realizando Operaciones CRUD

Una vez que la conexión está configurada, puedes realizar operaciones CRUD.

Crear

Inserta un nuevo registro en la base de datos.

addPerson :: String -> Int -> IO ()
addPerson name age = runSqlite "test.db" $ do
    personId <- insert $ Person name age
    liftIO $ putStrLn $ "Persona insertada con ID: " ++ show personId

Leer

Recupera registros de la base de datos.

getPeople :: IO ()
getPeople = runSqlite "test.db" $ do
    people <- selectList [] []
    liftIO $ mapM_ (print . entityVal) people

Actualizar

Actualiza un registro existente.

updatePersonAge :: PersonId -> Int -> IO ()
updatePersonAge personId newAge = runSqlite "test.db" $ do
    update personId [PersonAge =. newAge]
    liftIO $ putStrLn $ "Persona actualizada con ID: " ++ show personId

Eliminar

Elimina un registro de la base de datos.

deletePerson :: PersonId -> IO ()
deletePerson personId = runSqlite "test.db" $ do
    delete personId
    liftIO $ putStrLn $ "Persona eliminada con ID: " ++ show personId

Temas Avanzados

Para usuarios avanzados, Haskell ofrece operaciones de bases de datos más sofisticadas y optimizaciones.

Transacciones

Realiza múltiples operaciones dentro de una transacción para asegurar atomicidad.

transactionExample :: IO ()
transactionExample = runSqlite "test.db" $ do
    personId <- insert $ Person "Alice" 30
    update personId [PersonAge =. 31]
    delete personId
    liftIO $ putStrLn "Transacción completada."

Optimización de Consultas

Usa índices y consultas optimizadas para mejorar el rendimiento.

-- Ejemplo de creación de un índice
share [mkPersist sqlSettings, mkMigrate "migrateAll"] [persistLowerCase|
Person
    name String
    age Int
    deriving Show
    UniqueName name
|]

Conclusión

En esta sección, hemos cubierto los conceptos básicos del acceso a bases de datos en Haskell usando la biblioteca Persistent. Aprendiste cómo configurar una conexión, definir un esquema y realizar operaciones CRUD. Para usuarios avanzados, tocamos temas de transacciones y optimización de consultas. Con estas habilidades, puedes gestionar eficientemente las operaciones de bases de datos en tus aplicaciones Haskell.
