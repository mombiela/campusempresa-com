Introducción a los Mónadas
Mónadas Comunes: Maybe, Either, List ►

Los mónadas son un concepto fundamental en Haskell que puede ser desafiante para los principiantes. Este tema te guiará a través de los conceptos básicos de los mónadas, su propósito y cómo usarlos efectivamente en la programación en Haskell.

¿Qué es un Mónada?

Los mónadas son una clase de tipo en Haskell utilizada para manejar efectos secundarios y gestionar cálculos de manera funcional. Proporcionan una forma de estructurar programas de manera genérica.

Conceptos Clave

  • Clase de Tipo: Una clase de tipo define un conjunto de funciones que pueden ser implementadas por diferentes tipos.
  • Bind (>>=): Una operación fundamental que aplica una función a un valor envuelto.
  • Return: Envuelve un valor en un mónada.

Clase de Tipo Mónada

La clase de tipo Mónada se define en Haskell de la siguiente manera:

class Applicative m => Monad m where
    (>>=)  :: m a -> (a -> m b) -> m b
    return :: a -> m a

Explicación

  • Applicative m => Monad m: Indica que cada Mónada también debe ser un Applicative.
  • (>>=) :: m a -> (a -> m b) -> m b: La función bind toma un valor monádico y una función que devuelve un valor monádico, luego aplica la función al valor desenvuelto.
  • return :: a -> m a: La función return toma un valor y lo envuelve en un mónada.

Mónadas Comunes

Mónada Maybe

El mónada Maybe se utiliza para manejar cálculos que podrían fallar.

instance Monad Maybe where
    Nothing >>= _ = Nothing
    Just x  >>= f = f x
    return = Just

Ejemplo

safeDivide :: Double -> Double -> Maybe Double
safeDivide _ 0 = Nothing
safeDivide x y = Just (x / y)

result = Just 10 >>= (\x -> safeDivide x 2) >>= (\y -> safeDivide y 2)
-- result será Just 2.5

Mónada List

El mónada List se utiliza para cálculos no determinísticos.

instance Monad [] where
    xs >>= f = concat (map f xs)
    return x = [x]

Ejemplo

pairs :: [a] -> [b] -> [(a, b)]
pairs xs ys = xs >>= \x -> ys >>= \y -> return (x, y)

result = pairs [1, 2] ['a', 'b']
-- result será [(1,'a'), (1,'b'), (2,'a'), (2,'b')]

Mónada IO

El mónada IO se utiliza para manejar operaciones de entrada/salida.

instance Monad IO where
    (>>=)  = bindIO
    return = returnIO

Ejemplo

main :: IO ()
main = do
    putStrLn "Introduce tu nombre:"
    name <- getLine
    putStrLn ("Hola, " ++ name)

Notación Do

Haskell proporciona azúcar sintáctico para las operaciones monádicas llamada "notación do", que hace que el código monádico sea más legible.

Ejemplo con Mónada Maybe

safeDivide :: Double -> Double -> Maybe Double
safeDivide _ 0 = Nothing
safeDivide x y = Just (x / y)

result = do
    x <- Just 10
    y <- safeDivide x 2
    safeDivide y 2
-- result será Just 2.5

Ejemplo con Mónada IO

main :: IO ()
main = do
    putStrLn "Introduce tu nombre:"
    name <- getLine
    putStrLn ("Hola, " ++ name)

Resumen

Los mónadas son una poderosa abstracción en Haskell que ayudan a manejar efectos secundarios y estructurar programas funcionales. Al entender la clase de tipo Mónada, mónadas comunes como Maybe, List y IO, y usar la notación do, puedes escribir código en Haskell más expresivo y mantenible. Practicar con estos conceptos profundizará tu comprensión y competencia en la programación en Haskell.
