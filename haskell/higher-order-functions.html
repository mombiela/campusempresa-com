Funciones de Orden Superior en Haskell
Expresiones Lambda ►

Las funciones de orden superior son un concepto fundamental en Haskell y en la programación funcional en general. Son funciones que pueden tomar otras funciones como argumentos y/o devolver funciones como resultados. Este tema te guiará desde los conceptos básicos hasta los usos más avanzados de las funciones de orden superior en Haskell.

Introducción a las Funciones de Orden Superior

  • Definición: Una función de orden superior es una función que:

    • Toma una o más funciones como argumentos.
    • Devuelve una función como resultado.

  • Importancia: Las funciones de orden superior permiten un código más abstracto y flexible. Habilitan patrones poderosos como la composición de funciones, el currying y más.

Ejemplos Básicos

Comencemos con un ejemplo simple para ilustrar el concepto:

-- Una función que toma una función y un valor, y aplica la función al valor
applyTwice :: (a -> a) -> a -> a
applyTwice f x = f (f x)

-- Ejemplo de uso
double :: Int -> Int
double x = x * 2

main = print (applyTwice double 3)  -- Salida: 12

En este ejemplo:

  • applyTwice es una función de orden superior porque toma una función f como argumento.
  • double es una función simple que multiplica su entrada por 2.
  • applyTwice double 3 aplica double a 3 dos veces, resultando en 12.

Funciones de Orden Superior Comunes

`map`

  • Descripción: Aplica una función a cada elemento de una lista.
  • Firma de Tipo: (a -> b) -> [a] -> [b]
-- Ejemplo de uso de map
increment :: Int -> Int
increment x = x + 1

main = print (map increment [1, 2, 3, 4])  -- Salida: [2, 3, 4, 5]

`filter`

  • Descripción: Selecciona elementos de una lista que satisfacen un predicado.
  • Firma de Tipo: (a -> Bool) -> [a] -> [a]
-- Ejemplo de uso de filter
isEven :: Int -> Bool
isEven x = x `mod` 2 == 0

main = print (filter isEven [1, 2, 3, 4, 5, 6])  -- Salida: [2, 4, 6]

`foldl` y `foldr`

  • Descripción: Reduce una lista a un solo valor combinando elementos usando una función binaria.
  • Firma de Tipo:
    • foldl: (b -> a -> b) -> b -> [a] -> b
    • foldr: (a -> b -> b) -> b -> [a] -> b
-- Ejemplo de uso de foldl
sumList :: [Int] -> Int
sumList = foldl (+) 0

main = print (sumList [1, 2, 3, 4])  -- Salida: 10

Composición de Funciones

  • Descripción: Combinar dos o más funciones para formar una nueva función.
  • Operador: (.)
  • Firma de Tipo: (b -> c) -> (a -> b) -> a -> c
-- Ejemplo de uso de la composición de funciones
addOne :: Int -> Int
addOne x = x + 1

multiplyByTwo :: Int -> Int
multiplyByTwo x = x * 2

-- Componer addOne y multiplyByTwo
composedFunction = addOne . multiplyByTwo

main = print (composedFunction 3)  -- Salida: 7

Currying

  • Descripción: Transformar una función que toma múltiples argumentos en una serie de funciones que cada una toma un solo argumento.
  • Ejemplo:
-- Una función que suma dos números
add :: Int -> Int -> Int
add x y = x + y

-- Versión curried
addFive :: Int -> Int
addFive = add 5

main = print (addFive 3)  -- Salida: 8

Uso Avanzado

Creación de Funciones de Orden Superior Personalizadas

  • Ejemplo: Una función que toma una función y la aplica tres veces.
-- Una función que aplica una función dada tres veces
applyThrice :: (a -> a) -> a -> a
applyThrice f x = f (f (f x))

-- Ejemplo de uso
main = print (applyThrice double 2)  -- Salida: 16

Uso de Funciones de Orden Superior con Estructuras de Datos

  • Ejemplo: Uso de map con una estructura de datos personalizada.
-- Definir una estructura de datos personalizada
data Tree a = Leaf a | Node (Tree a) (Tree a) deriving Show

-- Definir una función map para el Tree
treeMap :: (a -> b) -> Tree a -> Tree b
treeMap f (Leaf x) = Leaf (f x)
treeMap f (Node left right) = Node (treeMap f left) (treeMap f right)

-- Ejemplo de uso
main = print (treeMap increment (Node (Leaf 1) (Leaf 2)))  -- Salida: Node (Leaf 2) (Leaf 3)

Conclusión

Las funciones de orden superior son una característica poderosa de Haskell que permiten una programación más abstracta y flexible. Al entender y utilizar funciones de orden superior, puedes escribir código más conciso, reutilizable y expresivo. Este tema cubrió los conceptos básicos de las funciones de orden superior, funciones de orden superior comunes como map, filter y fold, así como conceptos más avanzados como la composición de funciones y el currying. Practica usando estos conceptos para volverte competente en programación funcional con Haskell.
