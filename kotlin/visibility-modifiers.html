Modificadores de Visibilidad en Kotlin
Clases de Datos ►

Los modificadores de visibilidad en Kotlin controlan la accesibilidad de clases, objetos, interfaces, constructores, funciones, propiedades y sus setters. Entender estos modificadores es crucial para diseñar un código robusto y mantenible.

Principales Modificadores de Visibilidad

Kotlin proporciona cuatro modificadores de visibilidad:

  • public
  • private
  • protected
  • internal

Public

  • Modificador de visibilidad por defecto: Si no se especifica ningún modificador de visibilidad, el miembro es public por defecto.
  • Accesibilidad: Los miembros públicos son accesibles desde cualquier lugar del proyecto.
class PublicExample {
    public val publicProperty: String = "Soy público"
    
    public fun publicFunction() {
        println("Esta es una función pública")
    }
}

fun main() {
    val example = PublicExample()
    println(example.publicProperty) // Accesible
    example.publicFunction() // Accesible
}

Private

  • Accesibilidad: Los miembros privados son accesibles solo dentro del archivo o la clase en la que se declaran.
class PrivateExample {
    private val privateProperty: String = "Soy privado"
    
    private fun privateFunction() {
        println("Esta es una función privada")
    }
    
    fun accessPrivateMembers() {
        println(privateProperty) // Accesible dentro de la clase
        privateFunction() // Accesible dentro de la clase
    }
}

fun main() {
    val example = PrivateExample()
    // println(example.privateProperty) // No accesible
    // example.privateFunction() // No accesible
    example.accessPrivateMembers() // Accesible
}

Protected

  • Accesibilidad: Los miembros protegidos son accesibles dentro de la clase y sus subclases.
open class ProtectedExample {
    protected val protectedProperty: String = "Soy protegido"
    
    protected fun protectedFunction() {
        println("Esta es una función protegida")
    }
}

class SubclassExample : ProtectedExample() {
    fun accessProtectedMembers() {
        println(protectedProperty) // Accesible dentro de la subclase
        protectedFunction() // Accesible dentro de la subclase
    }
}

fun main() {
    val example = SubclassExample()
    example.accessProtectedMembers() // Accesible
    // println(example.protectedProperty) // No accesible
    // example.protectedFunction() // No accesible
}

Internal

  • Accesibilidad: Los miembros internos son accesibles dentro del mismo módulo.
internal class InternalExample {
    internal val internalProperty: String = "Soy interno"
    
    internal fun internalFunction() {
        println("Esta es una función interna")
    }
}

fun main() {
    val example = InternalExample()
    println(example.internalProperty) // Accesible dentro del mismo módulo
    example.internalFunction() // Accesible dentro del mismo módulo
}

Conclusión

Los modificadores de visibilidad en Kotlin son esenciales para controlar el acceso a diferentes partes de tu código. Al usar public, private, protected e internal adecuadamente, puedes asegurarte de que tu código sea tanto seguro como mantenible. Entender estos modificadores te ayudará a diseñar mejores APIs y encapsular tu código de manera efectiva.
