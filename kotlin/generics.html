Genéricos en Kotlin
Extensiones ►

Los genéricos en Kotlin te permiten escribir código flexible y reutilizable. Te permiten crear clases, interfaces y funciones que funcionan con cualquier tipo de dato mientras proporcionan seguridad de tipos. Este tema te guiará desde los conceptos básicos de los genéricos hasta conceptos más avanzados.

Introducción a los Genéricos

Los genéricos son una característica poderosa en Kotlin que te permite definir clases, interfaces y funciones con parámetros de tipo. Esto significa que puedes escribir código que funcione con diferentes tipos sin sacrificar la seguridad de tipos.

¿Por Qué Usar Genéricos?

  • Seguridad de Tipos: Los genéricos aseguran que captures errores de tipo en tiempo de compilación en lugar de en tiempo de ejecución.
  • Reutilización: Puedes escribir código más flexible y reutilizable.
  • Rendimiento: Los genéricos pueden ayudar a evitar la necesidad de conversiones de tipo, lo que puede mejorar el rendimiento.

Sintaxis Básica

Para definir una clase o función genérica, usas corchetes angulares (<>) para especificar el parámetro de tipo.

// Clase genérica
class Box<T>(val value: T)

// Función genérica
fun <T> printItem(item: T) {
    println(item)
}

En los ejemplos anteriores, T es un parámetro de tipo que puede ser reemplazado con cualquier tipo.

Clases Genéricas

Las clases genéricas te permiten crear clases que pueden manejar diferentes tipos de datos.

Ejemplo: Clase Genérica Box

class Box<T>(val value: T)

fun main() {
    val intBox = Box(1)
    val stringBox = Box("Hola")

    println(intBox.value)  // Salida: 1
    println(stringBox.value)  // Salida: Hola
}

En este ejemplo, Box puede contener cualquier tipo de valor, lo que la convierte en un contenedor versátil.

Múltiples Parámetros de Tipo

Puedes definir una clase con múltiples parámetros de tipo.

class Pair<A, B>(val first: A, val second: B)

fun main() {
    val pair = Pair(1, "Uno")
    println(pair.first)  // Salida: 1
    println(pair.second)  // Salida: Uno
}

Funciones Genéricas

Las funciones genéricas te permiten escribir funciones que pueden operar sobre diferentes tipos.

Ejemplo: Función Genérica Print

fun <T> printItem(item: T) {
    println(item)
}

fun main() {
    printItem(123)  // Salida: 123
    printItem("Hola")  // Salida: Hola
}

En este ejemplo, printItem puede imprimir cualquier tipo de ítem.

Restricciones en Parámetros de Tipo

Puedes restringir los tipos que pueden ser usados como parámetros de tipo usando restricciones.

Límites Superiores

Puedes especificar un límite superior para un parámetro de tipo usando el símbolo :.

fun <T : Number> sum(a: T, b: T): Double {
    return a.toDouble() + b.toDouble()
}

fun main() {
    println(sum(1, 2))  // Salida: 3.0
    println(sum(1.5, 2.5))  // Salida: 4.0
    // println(sum("1", "2"))  // Error: Incompatibilidad de tipos
}

En este ejemplo, T está restringido a ser un subtipo de Number.

Variancia

La variancia te permite definir cómo la subtipificación entre tipos más complejos se relaciona con la subtipificación entre sus componentes.

Covarianza

La covarianza permite que un tipo sea un subtipo de otro tipo. Usas la palabra clave out para declarar un parámetro de tipo como covariante.

class Box<out T>(val value: T)

fun main() {
    val intBox: Box<Int> = Box(1)
    val numberBox: Box<Number> = intBox  // La covarianza permite esto
    println(numberBox.value)  // Salida: 1
}

Contravarianza

La contravarianza permite que un tipo sea un supertipo de otro tipo. Usas la palabra clave in para declarar un parámetro de tipo como contravariante.

class Box<in T> {
    fun put(value: T) { /*...*/ }
}

fun main() {
    val numberBox: Box<Number> = Box()
    val intBox: Box<Int> = numberBox  // La contravarianza permite esto
    intBox.put(1)  // Esto está permitido
}

Proyecciones de Tipo

Las proyecciones de tipo te permiten usar tipos genéricos de una manera más flexible.

Proyecciones Estrella

Las proyecciones estrella (*) te permiten usar un tipo genérico sin especificar el parámetro de tipo exacto.

fun printBox(box: Box<*>) {
    println(box.value)
}

fun main() {
    val intBox = Box(1)
    val stringBox = Box("Hola")

    printBox(intBox)  // Salida: 1
    printBox(stringBox)  // Salida: Hola
}

Resumen

Los genéricos en Kotlin proporcionan una manera poderosa de escribir código flexible, reutilizable y seguro en cuanto a tipos. Al entender los conceptos básicos de clases y funciones genéricas, restricciones, variancia y proyecciones de tipo, puedes crear aplicaciones más robustas y versátiles. Practica usando genéricos en tus proyectos para comprender completamente su potencial y beneficios.
