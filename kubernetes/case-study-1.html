Introducción

Migrar una aplicación a Kubernetes puede ser un proceso complejo pero gratificante. Este estudio de caso te guiará a través de los pasos para migrar una aplicación web simple a Kubernetes, destacando conceptos clave, mejores prácticas y errores comunes.

Conceptos Clave

  • Contenedores: Paquetes ejecutables ligeros, autónomos, que incluyen todo lo necesario para ejecutar un software.
  • Kubernetes: Una plataforma de código abierto diseñada para automatizar el despliegue, escalado y operación de contenedores de aplicaciones.
  • Pods: Las unidades de despliegue más pequeñas en Kubernetes, que pueden contener uno o más contenedores.
  • Servicios: Abstracciones que definen un conjunto lógico de Pods y una política para acceder a ellos.
  • Despliegues: Proporcionan actualizaciones declarativas a las aplicaciones, asegurando que se mantenga el estado deseado.

Requisitos Previos

Antes de comenzar la migración, asegúrate de tener:

  • Un conocimiento básico de Docker y la contenedorización.
  • Un clúster de Kubernetes en funcionamiento (por ejemplo, Minikube para desarrollo local o un proveedor en la nube como GKE, EKS o AKS).
  • La herramienta de línea de comandos kubectl instalada y configurada para interactuar con tu clúster.

Migración Paso a Paso

Paso 1: Contenerizar la Aplicación

Primero, necesitamos contenerizar nuestra aplicación web usando Docker.

Ejemplo de Dockerfile

# Usar una imagen oficial de Node.js como imagen base
FROM node:14

# Establecer el directorio de trabajo
WORKDIR /usr/src/app

# Copiar package.json e instalar dependencias
COPY package*.json ./
RUN npm install

# Copiar el resto del código de la aplicación
COPY . .

# Exponer el puerto en el que se ejecuta la aplicación
EXPOSE 8080

# Definir el comando para ejecutar la aplicación
CMD ["node", "app.js"]
  • FROM: Especifica la imagen base.
  • WORKDIR: Establece el directorio de trabajo dentro del contenedor.
  • COPY: Copia archivos del host al contenedor.
  • RUN: Ejecuta comandos en el contenedor.
  • EXPOSE: Informa a Docker que el contenedor escucha en los puertos de red especificados.
  • CMD: Proporciona el comando predeterminado para ejecutar la aplicación.

Paso 2: Construir y Subir la Imagen Docker

Construir la imagen Docker y subirla a un registro de contenedores (por ejemplo, Docker Hub).

# Construir la imagen Docker
docker build -t tuusuario/tuapp:latest .

# Subir la imagen a Docker Hub
docker push tuusuario/tuapp:latest

Paso 3: Crear Manifiestos de Kubernetes

Crear manifiestos de Kubernetes para el Despliegue y el Servicio.

Manifiesto de Despliegue (deployment.yaml)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: webapp-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: webapp
  template:
    metadata:
      labels:
        app: webapp
    spec:
      containers:
      - name: webapp
        image: tuusuario/tuapp:latest
        ports:
        - containerPort: 8080
  • replicas: Número de réplicas de pods.
  • selector: Identifica los pods gestionados por el despliegue.
  • template: Define la plantilla del pod.

Manifiesto de Servicio (service.yaml)

apiVersion: v1
kind: Service
metadata:
  name: webapp-service
spec:
  selector:
    app: webapp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 8080
  type: LoadBalancer
  • selector: Coincide con los pods a exponer.
  • ports: Define las asignaciones de puertos.
  • type: Especifica el tipo de servicio (por ejemplo, LoadBalancer para acceso externo).

Paso 4: Desplegar en Kubernetes

Aplicar los manifiestos para desplegar la aplicación en el clúster de Kubernetes.

# Aplicar el despliegue
kubectl apply -f deployment.yaml

# Aplicar el servicio
kubectl apply -f service.yaml

Paso 5: Verificar el Despliegue

Verificar el estado del despliegue y del servicio.

# Obtener el estado del despliegue
kubectl get deployments

# Obtener el estado de los pods
kubectl get pods

# Obtener el estado del servicio
kubectl get services

Conclusión

Migrar a Kubernetes implica varios pasos, incluyendo contenerizar la aplicación, crear manifiestos de Kubernetes y desplegar en el clúster. Siguiendo este enfoque estructurado, puedes asegurar un proceso de migración fluido. Kubernetes proporciona herramientas poderosas para gestionar aplicaciones contenerizadas, facilitando el escalado y mantenimiento de tus servicios.

En este estudio de caso, cubrimos:

  • Contenerizar una aplicación web con Docker.
  • Crear manifiestos de Kubernetes para despliegue y servicio.
  • Desplegar y verificar la aplicación en un clúster de Kubernetes.

Este conocimiento fundamental te ayudará a abordar escenarios más complejos de Kubernetes en el futuro.
