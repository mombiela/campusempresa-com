Introducción a las Redes de Clúster

La red de clúster en Kubernetes es un concepto fundamental que asegura la comunicación entre varios componentes dentro de un clúster de Kubernetes. Esta sección cubrirá los conceptos básicos de la red de clúster, incluyendo cómo los pods se comunican entre sí, servicios y redes externas.

Conceptos Clave

  • Red de Pod: Cada pod en un clúster de Kubernetes obtiene su propia dirección IP, lo que permite la comunicación directa entre pods.
  • Red de Servicio: Los servicios proporcionan una dirección IP estable y un nombre DNS a un conjunto de pods, permitiendo una comunicación confiable.
  • ClusterIP: El tipo de servicio predeterminado que expone el servicio en una IP interna del clúster.
  • NodePort: Expone el servicio en la IP de cada Nodo en un puerto estático.
  • LoadBalancer: Expone el servicio externamente usando un balanceador de carga del proveedor de nube.
  • Políticas de Red: Definen cómo se permite que los pods se comuniquen entre sí y con otros puntos finales de la red.

Comunicación Pod a Pod

En Kubernetes, cada pod obtiene su propia dirección IP, y los pods pueden comunicarse entre sí usando estas direcciones IP.

Ejemplo

apiVersion: v1
kind: Pod
metadata:
  name: pod-a
spec:
  containers:
  - name: container-a
    image: nginx
---
apiVersion: v1
kind: Pod
metadata:
  name: pod-b
spec:
  containers:
  - name: container-b
    image: busybox
    command: ["sh", "-c", "sleep 3600"]
  • Explicación: En este ejemplo, pod-a y pod-b son dos pods que se ejecutan en el mismo clúster. pod-b puede comunicarse con pod-a usando la dirección IP de pod-a.

Servicios

Los servicios en Kubernetes proporcionan un punto final estable para acceder a un conjunto de pods.

Tipos de Servicios

  • ClusterIP: Expone el servicio en una IP interna del clúster.
  • NodePort: Expone el servicio en la IP de cada Nodo en un puerto estático.
  • LoadBalancer: Expone el servicio externamente usando un balanceador de carga del proveedor de nube.

Ejemplo

apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: MyApp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376
  type: ClusterIP
  • Explicación: Este servicio selecciona pods con la etiqueta app: MyApp y reenvía el tráfico desde el puerto 80 al puerto 9376 en los pods seleccionados.

Políticas de Red

Las políticas de red te permiten controlar la comunicación entre pods.

Ejemplo

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-nginx
spec:
  podSelector:
    matchLabels:
      app: nginx
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: frontend
  • Explicación: Esta política de red permite el tráfico entrante a pods con la etiqueta app: nginx solo desde pods con la etiqueta app: frontend.

Comparación de Tipos de Servicios

| Tipo de Servicio | Acceso Interno | Acceso Externo | Caso de Uso | |------------------|----------------|----------------|-------------| | ClusterIP | Sí | No | Comunicación interna entre pods | | NodePort | Sí | Sí | Exponer servicios en un puerto estático en cada nodo | | LoadBalancer | Sí | Sí | Exponer servicios externamente usando un balanceador de carga del proveedor de nube |

Conclusión

La red de clúster en Kubernetes es esencial para asegurar una comunicación fluida entre varios componentes dentro de un clúster. Entender la comunicación pod a pod, servicios y políticas de red es crucial para gestionar y asegurar tus aplicaciones en Kubernetes. Al dominar estos conceptos, puedes diseñar y desplegar aplicaciones escalables y seguras en un entorno de Kubernetes.
