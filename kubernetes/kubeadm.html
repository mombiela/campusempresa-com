Kubeadm
Minikube ►

Kubeadm es una herramienta que te ayuda a iniciar un clúster de Kubernetes mínimamente viable que se ajusta a las mejores prácticas. Realiza las acciones necesarias para poner en marcha un clúster seguro y mínimamente viable de una manera fácil de usar.

Introducción a Kubeadm

  • ¿Qué es Kubeadm?

    • Kubeadm es una herramienta proporcionada por Kubernetes para simplificar el proceso de configuración de un clúster de Kubernetes.
    • Maneja la instalación y configuración de los componentes del plano de control y los nodos de trabajo.

  • ¿Por qué usar Kubeadm?

    • Simplifica el proceso de configuración del clúster.
    • Asegura las mejores prácticas y configuraciones de seguridad.
    • Proporciona una forma consistente y repetible de crear clústeres.

Instalación de Kubeadm

Requisitos Previos

  • Sistema Operativo: Linux (Ubuntu, CentOS, etc.)
  • Requisitos de Hardware:
    • 2 GB o más de RAM por máquina (menos dejará poco espacio para tus aplicaciones).
    • 2 CPUs o más.
    • Conectividad de red completa entre todas las máquinas en el clúster (red pública o privada está bien).
    • Nombre de host único, dirección MAC y product_uuid para cada nodo.
  • Requisitos de Software:
    • Docker (u otro runtime de contenedores).
    • Kubeadm, kubelet y kubectl instalados en todos los nodos.

Pasos de Instalación

  1. Instalar Docker:

    sudo apt-get update
sudo apt-get install -y docker.io
sudo systemctl enable docker
sudo systemctl start docker

  2. Instalar Kubeadm, Kubelet y Kubectl:

    sudo apt-get update && sudo apt-get install -y apt-transport-https curl
curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
cat <<EOF | sudo tee /etc/apt/sources.list.d/kubernetes.list
deb https://apt.kubernetes.io/ kubernetes-xenial main
EOF
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

Iniciar un Clúster con Kubeadm

Inicializando el Plano de Control

  • Comando:

    sudo kubeadm init --pod-network-cidr=10.244.0.0/16

  • Explicación:

    • --pod-network-cidr=10.244.0.0/16: Esta bandera especifica el CIDR para la red de pods. Este ejemplo usa el CIDR predeterminado de Flannel.

  • Pasos Post-Inicialización:

    mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

Configurando una Red de Pods

  • Instalar Flannel: 
    kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml

Uniendo Nodos de Trabajo

  • Comando:

    • Después de inicializar el plano de control, kubeadm generará un comando kubeadm join. Ejecuta este comando en cada nodo de trabajo para unirlos al clúster.

  • Ejemplo:

    sudo kubeadm join <control-plane-host>:<control-plane-port> --token <token> --discovery-token-ca-cert-hash sha256:<hash>

Gestión del Clúster

Comandos Comunes

  • Verificar el Estado de los Nodos:

    kubectl get nodes

  • Verificar el Estado de los Pods:

    kubectl get pods --all-namespaces

  • Desplegar una Aplicación:

    kubectl create deployment nginx --image=nginx

  • Exponer el Despliegue:

    kubectl expose deployment nginx --port=80 --type=NodePort

Solución de Problemas

  • Problemas Comunes:

    • Pods no inician: Verifica los logs del pod usando kubectl logs <pod-name>.
    • Nodo no está listo: Asegúrate de que el servicio kubelet esté en ejecución y verifica el estado del nodo usando kubectl describe node <node-name>.

  • Comandos Útiles:

    • kubectl describe pod <pod-name>: Proporciona información detallada sobre un pod.
    • kubectl logs <pod-name>: Obtiene los logs de un pod.

Conclusión

Kubeadm es una herramienta poderosa que simplifica el proceso de configuración de un clúster de Kubernetes. Siguiendo las mejores prácticas y proporcionando un proceso de configuración consistente, asegura que tu clúster sea seguro y confiable. Ya sea que estés configurando un pequeño clúster de desarrollo o un gran entorno de producción, Kubeadm proporciona las herramientas que necesitas para comenzar rápida y eficientemente.
