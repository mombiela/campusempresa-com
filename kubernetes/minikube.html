Minikube
Estudio de Caso 1: Migración a Kubernetes ►

Minikube es una herramienta que facilita la ejecución de Kubernetes localmente. Ejecuta un clúster de Kubernetes de un solo nodo dentro de una máquina virtual (VM) en tu portátil. Esto es particularmente útil para propósitos de desarrollo, pruebas y aprendizaje.

Conceptos Clave

  • Clúster de Kubernetes de un solo nodo: Minikube ejecuta un clúster de Kubernetes de un solo nodo, lo que significa que incluye tanto las funcionalidades del nodo maestro como del nodo trabajador en una sola VM.
  • Máquina Virtual: Minikube utiliza una VM para ejecutar el clúster de Kubernetes. Soporta varios controladores de VM como VirtualBox, VMware, Hyper-V y Docker.
  • Desarrollo Local: Minikube está diseñado para el desarrollo y pruebas locales, facilitando la experimentación con Kubernetes sin necesidad de un clúster a gran escala.

Instalación

Prerrequisitos

Antes de instalar Minikube, asegúrate de tener lo siguiente:

  • Un hipervisor instalado (por ejemplo, VirtualBox, VMware, Hyper-V o Docker).
  • kubectl instalado. kubectl es la herramienta de línea de comandos para interactuar con clústeres de Kubernetes.

Instalando Minikube

  1. Descargar Minikube:

    • Para macOS: 
      brew install minikube
    • Para Linux: 
      curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube
    • Para Windows: Descarga el instalador desde la página de lanzamientos de Minikube.

  2. Iniciar Minikube:

    minikube start

    Este comando inicia un clúster de Kubernetes local. Puedes especificar el controlador de VM usando la bandera --driver, por ejemplo:

    minikube start --driver=virtualbox

Uso Básico

Interactuando con Minikube

  • Verificar el Estado del Clúster:

    minikube status

  • Acceder al Tablero de Kubernetes: Minikube proporciona un tablero de Kubernetes incorporado para gestionar el clúster.

    minikube dashboard

Desplegando Aplicaciones

  1. Crear un Despliegue:

    kubectl create deployment hello-minikube --image=k8s.gcr.io/echoserver:1.4

  2. Exponer el Despliegue:

    kubectl expose deployment hello-minikube --type=NodePort --port=8080

  3. Acceder a la Aplicación: Obtén la URL para acceder a la aplicación:

    minikube service hello-minikube --url

Gestionando Minikube

  • Detener Minikube:

    minikube stop

  • Eliminar el Clúster de Minikube:

    minikube delete

Características Avanzadas

Configuración de Recursos

Puedes configurar la cantidad de CPU y memoria asignada a la VM de Minikube:

minikube start --cpus=4 --memory=8192

Complementos

Minikube soporta varios complementos para extender su funcionalidad. Algunos complementos populares incluyen:

  • Ingress:

    minikube addons enable ingress

  • Metrics Server:

    minikube addons enable metrics-server

Almacenamiento Persistente

Minikube soporta almacenamiento persistente, permitiéndote crear recursos PersistentVolume y PersistentVolumeClaim.

  1. Crear un PersistentVolume:

    apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv0001
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadWriteOnce
  hostPath:
    path: "/mnt/data"

  2. Crear un PersistentVolumeClaim:

    apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: pvc0001
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi

  3. Aplicar las configuraciones:

    kubectl apply -f pv.yaml
kubectl apply -f pvc.yaml

Conclusión

Minikube es una excelente herramienta para que desarrolladores y aprendices obtengan experiencia práctica con Kubernetes. Simplifica el proceso de configuración de un entorno local de Kubernetes, facilitando el desarrollo, las pruebas y la experimentación con las características de Kubernetes. Al entender cómo instalar, configurar y usar Minikube, puedes simular efectivamente un clúster de Kubernetes en tu máquina local y obtener valiosos conocimientos sobre las operaciones de Kubernetes.
