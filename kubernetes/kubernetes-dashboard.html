Introducción al Tablero de Kubernetes

El Tablero de Kubernetes es una interfaz de usuario basada en la web que te permite gestionar y monitorear tus clústeres de Kubernetes. Proporciona una representación visual de los recursos y cargas de trabajo del clúster, lo que facilita la comprensión y gestión de tus aplicaciones.

Conceptos Clave

  • Gestión del Clúster: Ver y gestionar recursos del clúster como nodos, espacios de nombres y cargas de trabajo.
  • Gestión de Cargas de Trabajo: Desplegar, escalar y solucionar problemas de aplicaciones.
  • Monitoreo de Recursos: Monitorear el uso de recursos y la salud del clúster.
  • Gestión de Usuarios: Gestionar el control de acceso y permisos.

Instalación del Tablero de Kubernetes

Para usar el Tablero de Kubernetes, necesitas instalarlo en tu clúster. Sigue estos pasos para instalar el Tablero.

Instalación Paso a Paso

  1. Desplegar el Tablero:

    kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.3.1/aio/deploy/recommended.yaml

  2. Crear una Cuenta de Servicio:

    kubectl create serviceaccount dashboard-admin-sa
kubectl create clusterrolebinding dashboard-admin-sa --clusterrole=cluster-admin --serviceaccount=default:dashboard-admin-sa

  3. Obtener el Token Bearer:

    kubectl get secret $(kubectl get serviceaccount dashboard-admin-sa -o jsonpath="{.secrets[0].name}") -o jsonpath="{.data.token}" | base64 --decode

  4. Acceder al Tablero:

    kubectl proxy

    Abre la siguiente URL en tu navegador web:

    http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy/

Usando el Tablero de Kubernetes

Una vez que el Tablero esté instalado y funcionando, puedes comenzar a usarlo para gestionar tu clúster.

Navegando por el Tablero

  • Resumen: Proporciona un resumen del estado del clúster, incluyendo nodos, espacios de nombres y cargas de trabajo.
  • Cargas de Trabajo: Gestiona despliegues, pods, conjuntos de réplicas y trabajos.
  • Servicios e Ingress: Gestiona servicios, recursos de ingress y políticas de red.
  • Configuración y Almacenamiento: Gestiona ConfigMaps, secretos, reclamaciones de volúmenes persistentes y clases de almacenamiento.
  • Clúster: Ver y gestionar nodos, espacios de nombres y roles.
  • Configuraciones: Configura los ajustes del Tablero y gestiona el acceso de usuarios.

Ejemplo: Desplegando una Aplicación

  1. Crear un Despliegue:

    • Navega a la sección "Cargas de Trabajo".
    • Haz clic en "Despliegues" y luego en "Crear".
    • Rellena los detalles necesarios como el nombre de la aplicación, la imagen del contenedor y el número de réplicas.
    • Haz clic en "Desplegar".

  2. Exponer el Despliegue:

    • Navega a la sección "Servicios e Ingress".
    • Haz clic en "Servicios" y luego en "Crear".
    • Selecciona el despliegue que creaste y configura el tipo de servicio (por ejemplo, ClusterIP, NodePort, LoadBalancer).
    • Haz clic en "Crear".

Monitoreo de Recursos

  • Uso de Recursos: Ver el uso de CPU y memoria para nodos y pods.
  • Registros: Acceder a los registros de pods individuales para solucionar problemas.
  • Eventos: Ver eventos relacionados con los recursos del clúster para entender qué acciones se han tomado.

Conclusión

El Tablero de Kubernetes es una herramienta poderosa para gestionar y monitorear tus clústeres de Kubernetes. Proporciona una interfaz fácil de usar para desplegar aplicaciones, monitorear recursos y solucionar problemas. Siguiendo los pasos de instalación y explorando las características del Tablero, puedes gestionar eficazmente tu entorno de Kubernetes.

Resumen

  • El Tablero de Kubernetes es una interfaz de usuario basada en la web para gestionar clústeres de Kubernetes.
  • Te permite desplegar aplicaciones, monitorear recursos y gestionar el acceso de usuarios.
  • La instalación implica desplegar el Tablero, crear una cuenta de servicio y acceder al Tablero a través de un proxy.
  • El Tablero proporciona varias secciones para gestionar cargas de trabajo, servicios, configuraciones y el clúster en sí.

Dominando el Tablero de Kubernetes, puedes mejorar tu capacidad para gestionar y optimizar tus clústeres de Kubernetes de manera eficiente.
