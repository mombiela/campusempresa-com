Introducción a los Controladores de Ingress

Los Controladores de Ingress son un componente crítico en Kubernetes para gestionar el acceso externo a los servicios dentro de un clúster. Proporcionan una forma de exponer rutas HTTP y HTTPS desde fuera del clúster a servicios dentro del clúster.

Conceptos Clave

  • Recurso de Ingress: Una colección de reglas que permiten que las conexiones entrantes lleguen a los servicios del clúster.
  • Controlador de Ingress: Un demonio que observa el servidor API de Kubernetes en busca de actualizaciones en los recursos de Ingress y configura el balanceador de carga en consecuencia.

Configuración de un Controlador de Ingress

Para usar un recurso de Ingress, necesitas tener un Controlador de Ingress ejecutándose en tu clúster. Hay diferentes Controladores de Ingress disponibles, como NGINX, Traefik y HAProxy.

Instalación del Controlador de Ingress NGINX

NGINX es uno de los Controladores de Ingress más populares. Aquí se explica cómo instalarlo:

kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/cloud/deploy.yaml

Este comando desplegará el Controlador de Ingress NGINX en tu clúster.

Creación de un Recurso de Ingress

Una vez que el Controlador de Ingress está instalado, puedes crear un recurso de Ingress para definir cómo el tráfico externo debe ser enrutado a tus servicios.

Ejemplo de Recurso de Ingress

Aquí hay un ejemplo de un recurso de Ingress:

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: example-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  rules:
  - host: example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: example-service
            port:
              number: 80

Explicación

  • apiVersion: Especifica la versión de la API.
  • kind: Define el tipo de recurso, que es Ingress.
  • metadata: Contiene metadatos sobre el recurso de Ingress, como su nombre.
  • annotations: Proporcionan opciones de configuración adicionales para el Controlador de Ingress.
  • spec: Define el estado deseado del Ingress.
    • rules: Una lista de reglas para enrutar el tráfico.
      • host: El nombre de dominio para la regla.
      • http: Especifica que la regla se aplica al tráfico HTTP.
        • paths: Una lista de rutas a coincidir.
          • path: La ruta URL a coincidir.
          • pathType: El tipo de coincidencia de ruta (por ejemplo, Prefix).
          • backend: Especifica el servicio backend al que enrutar el tráfico.
            • service: El nombre y puerto del servicio.

Funciones Avanzadas de Ingress

Los Controladores de Ingress ofrecen funciones avanzadas como terminación SSL, reescritura de URL y balanceo de carga.

Terminación SSL

Para habilitar la terminación SSL, necesitas crear un secreto TLS y referenciarlo en tu recurso de Ingress.

Creación de un Secreto TLS

kubectl create secret tls example-tls --cert=path/to/tls.crt --key=path/to/tls.key

Actualización del Recurso de Ingress

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: example-ingress
spec:
  tls:
  - hosts:
    - example.com
    secretName: example-tls
  rules:
  - host: example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: example-service
            port:
              number: 80

Reescritura de URL

La reescritura de URL puede configurarse usando anotaciones. Por ejemplo, para reescribir todas las rutas a la raíz:

metadata:
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /

Solución de Problemas de los Controladores de Ingress

Los problemas comunes con los Controladores de Ingress incluyen configuraciones incorrectas y problemas de red. Aquí hay algunos consejos para solucionar problemas:

  • Verificar los Logs del Controlador de Ingress: Usa kubectl logs para ver los logs del Controlador de Ingress.
  • Validar el Recurso de Ingress: Asegúrate de que el recurso de Ingress esté correctamente definido.
  • Políticas de Red: Verifica que las políticas de red no estén bloqueando el tráfico.

Conclusión

Los Controladores de Ingress son esenciales para gestionar el acceso externo a los servicios en un clúster de Kubernetes. Al entender cómo configurar y configurar los Controladores de Ingress, puedes enrutar eficazmente el tráfico a tus aplicaciones y aprovechar funciones avanzadas como la terminación SSL y la reescritura de URL.
