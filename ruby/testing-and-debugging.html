Introducción

Las pruebas y la depuración son habilidades esenciales para cualquier programador. En esta sección, cubriremos los conceptos básicos de las pruebas y la depuración en Ruby, progresando desde niveles principiantes hasta avanzados. Al final de esta sección, podrás escribir pruebas para tu código Ruby y depurar eficazmente cualquier problema que surja.

Pruebas en Ruby

Por qué las Pruebas son Importantes

  • Aseguran la corrección del código.
  • Facilitan el mantenimiento del código.
  • Ayudan a detectar errores temprano.
  • Proporcionan documentación sobre el comportamiento del código.

Tipos de Pruebas

  • Pruebas Unitarias: Prueban unidades o componentes individuales.
  • Pruebas de Integración: Prueban la interacción entre diferentes componentes.
  • Pruebas de Sistema: Prueban el sistema completo en su totalidad.
  • Pruebas de Aceptación: Validan la funcionalidad de extremo a extremo contra los requisitos.

Marcos de Pruebas en Ruby

  • Minitest: Una biblioteca de pruebas ligera y rápida.
  • RSpec: Un marco de pruebas más expresivo y con más características.

Escribiendo tu Primera Prueba con Minitest

# Instala Minitest si no lo has hecho ya
# gem install minitest

require 'minitest/autorun'

class TestExample < Minitest::Test
  def test_addition
    assert_equal 4, 2 + 2
  end
end
  • Explicación: Esta simple prueba verifica si la suma de 2 y 2 es igual a 4 usando Minitest.

Escribiendo tu Primera Prueba con RSpec

# Instala RSpec si no lo has hecho ya
# gem install rspec

# Crea un archivo llamado example_spec.rb
require 'rspec'

RSpec.describe 'Addition' do
  it 'adds 2 and 2 to get 4' do
    expect(2 + 2).to eq(4)
  end
end

# Ejecuta la prueba con el comando: rspec example_spec.rb
  • Explicación: Esta prueba de RSpec verifica si la suma de 2 y 2 es igual a 4.

Pruebas Avanzadas con RSpec

  • Matchers: Usa diferentes matchers para probar varias condiciones.
  • Hooks: Configura y desmonta el código usando hooks before y after.
  • Ejemplos Compartidos: Reutiliza el código de prueba para múltiples contextos.
RSpec.describe 'Advanced Testing' do
  before(:each) do
    @array = [1, 2, 3]
  end

  it 'checks array length' do
    expect(@array.length).to eq(3)
  end

  it 'includes a specific element' do
    expect(@array).to include(2)
  end
end
  • Explicación: Este ejemplo demuestra el uso de hooks before y matchers en RSpec.

Depuración en Ruby

Técnicas Comunes de Depuración

  • Declaraciones Print: Usa puts o p para imprimir valores de variables.
  • IRB (Interactive Ruby): Usa IRB para probar fragmentos de código de manera interactiva.
  • Depurador: Usa el depurador incorporado de Ruby o herramientas como pry.

Usando Declaraciones Print

def add(a, b)
  result = a + b
  puts "Adding #{a} and #{b} to get #{result}"
  result
end

add(2, 3)
  • Explicación: Este código usa puts para imprimir los valores de las variables durante la ejecución.

Usando IRB

  • Inicia IRB escribiendo irb en tu terminal.
  • Prueba fragmentos de código de manera interactiva.
irb(main):001:0> 2 + 2
=> 4
irb(main):002:0> def greet(name)
irb(main):003:1>   "Hello, #{name}!"
irb(main):004:1> end
=> :greet
irb(main):005:0> greet("Alice")
=> "Hello, Alice!"
  • Explicación: Este ejemplo muestra cómo usar IRB para probar código Ruby de manera interactiva.

Usando el Depurador

  • Byebug: Un depurador popular para Ruby.
  • Pry: Una alternativa a Byebug con más características.

Usando Byebug

# Instala Byebug si no lo has hecho ya
# gem install byebug

require 'byebug'

def add(a, b)
  byebug
  a + b
end

add(2, 3)
  • Explicación: Este código usa Byebug para pausar la ejecución e iniciar una sesión de depuración.

Usando Pry

# Instala Pry si no lo has hecho ya
# gem install pry

require 'pry'

def add(a, b)
  binding.pry
  a + b
end

add(2, 3)
  • Explicación: Este código usa Pry para pausar la ejecución e iniciar una sesión interactiva.

Conclusión

Las pruebas y la depuración son cruciales para escribir código Ruby confiable y mantenible. Al dominar estas habilidades, puedes asegurarte de que tu código funcione como se espera y puedes identificar y solucionar rápidamente cualquier problema que surja. Ya sea que uses Minitest, RSpec, declaraciones print, IRB, Byebug o Pry, la clave es integrar estas prácticas en tu flujo de trabajo de desarrollo para mejorar la calidad de tu código.
