Bloques, Procs y Lambdas
Arreglos ►

Introducción

En Ruby, los bloques, procs y lambdas son herramientas poderosas para manejar fragmentos de código. Permiten pasar código como variables, haciendo que tus programas sean más flexibles y concisos. Esta sección te guiará para entender y usar estas características de manera efectiva.

Bloques

¿Qué es un Bloque?

  • Un bloque es un fragmento de código anónimo que puede ser pasado a métodos.
  • Los bloques no son objetos, pero pueden ser convertidos en objetos usando Proc.

Sintaxis

Los bloques pueden ser definidos usando do...end o llaves {...}.

# Usando do...end
[1, 2, 3].each do |number|
  puts number
end

# Usando llaves
[1, 2, 3].each { |number| puts number }

Cediendo a un Bloque

  • Los métodos pueden ceder el control a un bloque usando la palabra clave yield.
def greet
  yield "Hello"
end

greet { |greeting| puts greeting }  # Salida: Hello

Parámetros de Bloque

  • Los bloques pueden tomar parámetros, que se definen entre barras verticales | |.
def greet
  yield "Hello", "World"
end

greet { |greeting, name| puts "#{greeting}, #{name}!" }  # Salida: Hello, World!

Procs

¿Qué es un Proc?

  • Un Proc es un objeto que encapsula un bloque de código, que puede ser almacenado en una variable, pasado a métodos, o llamado.

Creando un Proc

  • Usa Proc.new o el método proc para crear un Proc.
# Usando Proc.new
my_proc = Proc.new { |name| puts "Hello, #{name}!" }

# Usando proc
my_proc = proc { |name| puts "Hello, #{name}!" }

Llamando a un Proc

  • Llama a un Proc usando el método call.
my_proc.call("Alice")  # Salida: Hello, Alice!

Proc vs Bloque

| Característica | Bloque | Proc | |-------------------|--------------------------------|--------------------------------| | Definición | Anónimo, no es un objeto | Objeto, puede ser almacenado en una variable | | Pasar a Métodos | Implícitamente | Explícitamente | | Múltiples Bloques | No | Sí |

Lambdas

¿Qué es una Lambda?

  • Una lambda es un tipo especial de Proc con una verificación de argumentos más estricta y un comportamiento diferente para return.

Creando una Lambda

  • Usa el método lambda o la sintaxis ->.
# Usando el método lambda
my_lambda = lambda { |name| puts "Hello, #{name}!" }

# Usando la sintaxis ->
my_lambda = ->(name) { puts "Hello, #{name}!" }

Llamando a una Lambda

  • Llama a una lambda usando el método call.
my_lambda.call("Bob")  # Salida: Hello, Bob!

Lambda vs Proc

| Característica | Proc | Lambda | |-------------------|--------------------------------|--------------------------------| | Verificación de Argumentos | Laxa | Estricta | | Comportamiento de Return | Retorna del método que lo contiene | Retorna de la lambda misma |

Conclusión

Entender los bloques, procs y lambdas es esencial para escribir código Ruby flexible y conciso. Los bloques son útiles para pasar fragmentos de código a métodos, mientras que los procs y lambdas proporcionan más control y flexibilidad al permitir almacenar y reutilizar código. Las lambdas ofrecen una verificación de argumentos más estricta y un comportamiento de retorno diferente en comparación con los procs. Al dominar estos conceptos, podrás escribir programas Ruby más poderosos y mantenibles.
