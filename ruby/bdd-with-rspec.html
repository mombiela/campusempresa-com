Introducción al Desarrollo Guiado por el Comportamiento (BDD)

  • Definición: BDD es un proceso de desarrollo de software que fomenta la colaboración entre desarrolladores, QA y participantes no técnicos o de negocio en un proyecto de software.
  • Propósito: Asegurar que todas las partes interesadas tengan una comprensión clara del comportamiento del software antes de que comience el desarrollo.
  • Conceptos Clave:
    • Historias de Usuario: Descripciones de las características del software desde la perspectiva del usuario final.
    • Escenarios: Ejemplos específicos de cómo una característica debe comportarse en diferentes situaciones.
    • Given-When-Then: Una plantilla para escribir escenarios.

Configuración de RSpec

  • Instalación: 
    gem install rspec
  • Inicialización: 
    rspec --init
    Este comando crea el directorio spec necesario y los archivos de configuración.

Escribiendo tu Primera Prueba con RSpec

  • Creando un Archivo de Especificación: 
    # spec/calculator_spec.rb
require 'calculator'

RSpec.describe Calculator do
  it 'suma dos números' do
    calculator = Calculator.new
    result = calculator.add(5, 10)
    expect(result).to eq(15)
  end
end
  • Ejecutando Pruebas: 
    rspec

Entendiendo la Sintaxis de RSpec

  • Bloques Describe: Usados para agrupar pruebas relacionadas. 
    RSpec.describe 'Un grupo de pruebas' do
  # las pruebas van aquí
end
  • Bloques It: Definen casos de prueba individuales. 
    it 'hace algo' do
  # código de prueba aquí
end
  • Expectativas: Aserciones para verificar si el código se comporta como se espera. 
    expect(actual).to eq(expected)

Características Avanzadas de RSpec

  • Hooks: Código que se ejecuta en ciertos puntos del ciclo de vida de la prueba.
    • before: Se ejecuta antes de cada prueba. 
      before(:each) do
  # código de configuración aquí
end
    • after: Se ejecuta después de cada prueba. 
      after(:each) do
  # código de limpieza aquí
end
  • Ejemplos Compartidos: Casos de prueba reutilizables. 
    RSpec.shared_examples 'un ejemplo compartido' do
  it 'hace algo' do
    expect(subject).to be_something
  end
end

RSpec.describe MyClass do
  it_behaves_like 'un ejemplo compartido'
end

Mocking y Stubbing

  • Mocks: Objetos que simulan el comportamiento de objetos reales. 
    let(:mock_object) { double('MockObject') }
  • Stubs: Métodos que devuelven respuestas predefinidas. 
    allow(mock_object).to receive(:method_name).and_return('value')

Integrando RSpec con Rails

  • Generando Archivos RSpec: 
    rails generate rspec:install
  • Especificaciones de Modelos: 
    RSpec.describe User, type: :model do
  it 'es válido con atributos válidos' do
    user = User.new(name: 'John Doe', email: 'john@example.com')
    expect(user).to be_valid
  end
end
  • Especificaciones de Controladores: 
    RSpec.describe UsersController, type: :controller do
  describe 'GET #index' do
    it 'devuelve una respuesta exitosa' do
      get :index
      expect(response).to be_successful
    end
  end
end

Conclusión

El Desarrollo Guiado por el Comportamiento con RSpec permite un enfoque más colaborativo y claro para el desarrollo de software. Al escribir pruebas que describen el comportamiento deseado de la aplicación, los desarrolladores pueden asegurarse de que el software cumpla con las necesidades de todas las partes interesadas. RSpec proporciona un marco poderoso y flexible para escribir estas pruebas, convirtiéndolo en una herramienta esencial para los desarrolladores de Ruby.
