Introducción a las Excepciones

En Ruby, las excepciones son una forma de manejar errores que ocurren durante la ejecución de un programa. En lugar de hacer que el programa se bloquee, Ruby proporciona mecanismos para capturar y manejar estos errores de manera elegante.

  • Excepción: Un evento que interrumpe el flujo normal de un programa.
  • Manejo de Excepciones: El proceso de responder a las excepciones cuando ocurren.

Manejo Básico de Excepciones

Ruby utiliza las palabras clave begin, rescue y end para manejar excepciones.

Sintaxis

begin
  # Código que podría generar una excepción
rescue
  # Código que se ejecuta si ocurre una excepción
end

Ejemplo

begin
  puts "Introduce un número:"
  number = gets.to_i
  result = 10 / number
  puts "El resultado es #{result}"
rescue
  puts "Ocurrió un error: No se permite la división por cero."
end

En este ejemplo:

  • El bloque begin contiene código que podría generar una excepción.
  • El bloque rescue contiene código que se ejecuta si ocurre una excepción.

Manejo de Excepciones Específicas

Puedes rescatar tipos específicos de excepciones especificando la clase de excepción.

Sintaxis

begin
  # Código que podría generar una excepción
rescue SpecificExceptionClass
  # Código que se ejecuta si ocurre una excepción específica
end

Ejemplo

begin
  puts "Introduce un número:"
  number = gets.to_i
  result = 10 / number
  puts "El resultado es #{result}"
rescue ZeroDivisionError
  puts "Ocurrió un error: No se permite la división por cero."
end

En este ejemplo:

  • El bloque rescue ZeroDivisionError solo captura excepciones de tipo ZeroDivisionError.

Asegurando la Limpieza con `ensure`

El bloque ensure se ejecuta tanto si ocurre una excepción como si no, lo que lo hace útil para tareas de limpieza.

Sintaxis

begin
  # Código que podría generar una excepción
rescue
  # Código que se ejecuta si ocurre una excepción
ensure
  # Código que siempre se ejecuta
end

Ejemplo

begin
  file = File.open("example.txt", "r")
  # Realizar operaciones con el archivo
rescue
  puts "Ocurrió un error al abrir el archivo."
ensure
  file.close if file
  puts "Archivo cerrado."
end

En este ejemplo:

  • El bloque ensure asegura que el archivo se cierre tanto si ocurre una excepción como si no.

Generando Excepciones

Puedes generar tus propias excepciones usando la palabra clave raise.

Sintaxis

raise "Mensaje de error"

Ejemplo

def divide(a, b)
  raise "No se permite la división por cero" if b == 0
  a / b
end

begin
  result = divide(10, 0)
  puts "El resultado es #{result}"
rescue => e
  puts "Ocurrió un error: #{e.message}"
end

En este ejemplo:

  • La palabra clave raise se usa para generar una excepción si b es cero.
  • El bloque rescue captura la excepción e imprime el mensaje de error.

Clases de Excepción Personalizadas

Puedes definir tus propias clases de excepción heredando de StandardError.

Ejemplo

class MyCustomError < StandardError; end

def risky_method
  raise MyCustomError, "¡Algo salió mal!"
end

begin
  risky_method
rescue MyCustomError => e
  puts "Se capturó un error personalizado: #{e.message}"
end

En este ejemplo:

  • MyCustomError es una clase de excepción personalizada.
  • El bloque rescue MyCustomError captura excepciones de tipo MyCustomError.

Conclusión

Manejar excepciones es una parte crucial de escribir programas Ruby robustos y resistentes a errores. Usando begin, rescue, ensure y clases de excepción personalizadas, puedes gestionar errores de manera elegante y asegurar que tu programa pueda manejar situaciones inesperadas sin bloquearse. Practica estos conceptos con varios escenarios para volverte competente en el manejo de excepciones en Ruby.
